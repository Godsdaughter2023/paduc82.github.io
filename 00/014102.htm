Notes: 

1. You will need to change the path to your template folder.

2. SaveSetting writes the pointer to the ribbon memory location to a registry key:
HKCU\Software\VB  and VBA Program Settings

3.  I used a standard module named modCK_RC

 

The conditional compiling in this case is just to ensure the code will work in Windows\Office 64 bit.

 

Option Explicit
Public rxIRibbonUI As IRibbonUI
#If VBA7 Then
  Public Declare PtrSafe Sub CopyMemory Lib "kernel32" Alias "RtlMoveMemory" ( _
       ByRef destination As Any, ByRef source As Any, ByVal Length As Long)
#Else
  Public Declare Sub CopyMemory Lib "kernel32" Alias "RtlMoveMemory" ( _
       ByRef destination As Any, ByRef source As Any, ByVal Length As Long)
#End If
#If VBA7 Then
  Function GetRibbon(ByVal lRibbonPointer As LongPtr) As Object
#Else
  Function GetRibbon(ByVal lRibbonPointer As Long) As Object
#End If
  Dim objRibbon As Object
  CopyMemory objRibbon, lRibbonPointer, LenB(lRibbonPointer)
  Set GetRibbon = objRibbon
  Set objRibbon = Nothing
End Function
Sub onLoad(ribbon As IRibbonUI)
  'Callback for onLoad to capture RibbonUI  -
  Set rxIRibbonUI = ribbon
  SaveSetting "KenyonForms.dotm", "Wd2010", "Ribbon Ref", ObjPtr(rxIRibbonUI)
End Sub

Sub ButtonOnAction(control As IRibbonControl)
  Select Case control.ID
    Case "rxbtn1Refresh"
      If rxIRibbonUI Is Nothing Then RefreshRibbon
      rxIRibbonUI.InvalidateControl ("dmDemo")
    Case Else
     Documents.Add control.Tag
    End Select
End Sub

Sub RefreshRibbon()
Dim strRibRef As String
  strRibRef = GetSetting("KenyonForms.dotm", "Wd2010", "Ribbon Ref")
  Beep
  If strRibRef <> vbNullString Then
    Set rxIRibbonUI = GetRibbon(strRibRef)
    Call onLoad(rxIRibbonUI)
  End If
 End Sub

Sub GetContent(control As IRibbonControl, ByRef content)
Dim objFSO As Object
Dim objTemplateFolder As Object
Dim oFile As Object
Dim sXML As String
Dim lBtnCount As Long
Dim strPath As String
  MsgBox "Loading or refreshing"
  'Open the XML string
  sXML = "<menu xmlns=""" & "http://schemas.microsoft.com/office/2009/07/customui"">"
  'sXML = "<menu xmlns=""" & "http://schemas.microsoft.com/office/2006/01/customui"">"
 
  'Create FSO object and set to templates folder
  Set objFSO = CreateObject("Scripting.FileSystemObject")
  strPath = Application.Options.DefaultFilePath(wdWorkgroupTemplatesPath) & "\Templates\"
  Set objTemplateFolder = objFSO.getFolder(strPath)
  'Add template files
  If objTemplateFolder.Files.Count > 0 Then
    For Each oFile In objTemplateFolder.Files
      'Check if file is a temporary file
      If Not Left(oFile.Name, 2) = "~$" Then
        'File is not a temp file, so check extension
        Select Case LCase(Right(oFile.Name, 4))
          Case ".dot", "dotx", "dotm"
            'Word template.
            If Not LCase(Left(oFile.Name, 6)) = "normal" Then
              sXML = sXML & _
                "<button id=""rxbtn1Dyna" & lBtnCount & """ " & _
                "label=""" & oFile.Name & """ " & _
                "imageMso=""FileSaveAsWord97_2003"" " & _
                "tag=""" & oFile.Path & """ " & _
                "onAction=""ButtonOnAction""/>" & vbCrLf
              lBtnCount = lBtnCount + 1
            End If
          Case Else
             'Unknown format.  Ignore.
        End Select
      End If
    Next oFile
  End If
  'Release the FSO objects
  Set oFile = Nothing
  Set objTemplateFolder = Nothing
  Set objFSO = Nothing
  'Check if any items buttons were created
  'Create a "No Templates" button if not
  If lBtnCount = 0 Then _
    sXML = sXML & "<button id=""rxbtn1Dyna0"" label=""No Templates Found""/>"
  'Add Refresh button & close the menu tags
  sXML = sXML & _
    "<button id=""rxbtn1Refresh"" " & _
    "label=""Refresh List"" " & _
    "imageMso=""RecurrenceEdit"" " & _
    "onAction=""ButtonOnAction""/>" & _
    "</menu>"
  'Feed the XML back to the Ribbon
  content = sXML
End Sub

Microsoft Office 2010 Starter

jay freedman say:
When you say "Are my Microsoft Office files still there?" if you mean the documents you created with the Office Starter programs, then the answer is yes -- uninstalling Office does not remove any documents. They will still be in the folders where you saved them, usually the My Documents folder.

If the "problem" with your C drive didn't make it impossible to read the files from the drive, then you can copy the documents to a flash drive.

Most manufacturers include a recovery partition on the drive that will restore the main part of the drive to its condition when it left the factory, and that should include a fresh copy of Office 2010 Starter. If you intend to install Office 2007 (if you have the disk and product key for that version), you should then uninstall Office 2010 Starter before installing 2007.

One other tip: when the documents are on the flash drive, do not open them in Word directly from the flash drive. That has a high risk of corrupting the documents so that they can't be opened again. Instead, copy them back to the hard drive and edit them there.

jay freedman say:
You have two completely separate jobs to do.

First, use Windows Explorer to copy all the existing Office documents (spreadsheets, Word documents, PowerPoint slide decks if there are any) to a flash drive or other location that isn't a permanent part of the laptop. You don't have to have any Office programs available when you do this -- and if any Office program is open, you should close it first.

Also make copies of any template files (.dotx, .dotm, .xltx, .xltm, .potx, .potm) from Office, and any database files (.mdb, .accdb). If there are other files that you or your friend created on the hard drive -- pictures, music, videos, etc. -- copy them off, too.

After all the documents and other things are safely off the laptop to someplace where they won't be deleted, then you can think about getting Office back. What Windows calls a "system restore" won't replace the files you deleted. If you have the disk and product key for some version of Office, you can install that. If you don't have them, you can (maybe) use the factory recovery partition if there is one. I can't tell you how to do that, because each manufacturer does it differently. Try looking up the specific laptop model in the Support pages of the manufacturer's website.
