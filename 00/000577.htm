Sub OnEntryText1()

> "I don't understand why it is only selecting certain records and not all. "

if you want ALL the records from JHC regardless of whether there is a corresponding record in [Master Tracking Log], then change this:
FROM JHC INNER JOIN [Master Tracking Log] ON JHC.[Equip Code No] = [Master Tracking Log].[EPI Code]
to this:
FROM JHC LEFT JOIN [Master Tracking Log] ON JHC.[Equip Code No] = [Master Tracking Log].[EPI Code]

by default, Access uses an INNER join to relate records.  This means that only records where the key field is in both tables will be shown.  Changing the join type to LEFT or RIGHT (depending on how the SQL is constructed) is called an "outer" join and sounds like it may be what you want.

~~~~

> "How do I make this form open to a blank display with only my headers/footers, NO data."

On the form LOAD event (which runs when the data is loaded -- do not use the Open event because data is not yet available):

'~~~~~~~~~~~~~~~~~~

   if not me.NewRecord then

      DoCmd.GoToRecord , , acNewRec

   end if

'~~~~~~~~~~~~~~~~~~

HOWEVER! It also appears you might want to be using a mainform/subform scenario so that you can EDIT and ADD records.  If you do not know what a subform is, then read the "Form and Report Design" in my free Access Basics document (link in siggy)

What I often do is use unbound controls in the form header to filter a form and then use the form BeforeInsert event to move data in the unbound header controls to a new record.  Then you can use a combobox to pick values that are actually in your table (and limit to the first 9 chars if you wish) as opposed to having to fill out a parameter each time -- just cuts out some typing and ensures what is being entered is there
