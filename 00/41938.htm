sagb - mcet say:

Touchpad gestures not working properly on Chuwi Hi 10 Air tablet

The CHUWI Hi10 Air's keyboard contains a generic touchpad that is gesture capable, but Windows 10 designates it as an HID Generic Mouse.  Even with this driver designation the gesture capabilities are active.

Unfortunately, the touchpad sensitivity causes misreading of the user's finger, sometimes detecting three points of contact when the user is only using one finger in contact with the pad.  This leads to the spontaneously hiding of all open programs to display the desktop (and other such random interpretations).

The mouse designation creates a problem in that the options to disable the gestures to help avoid this issue are not available.

The current only known way to fix this problem is to download a third party application called "AutoHotKey" and create a script that effectively disables the misread gestures.

The commands for the script are as follows:

#d::return
#a::return
#tab::return
#B::return
#Down::return
#Up::return

The script must then be placed in the "C:\ProgramData\Microsoft\Windows\Start Menu\Programs\StartUp" folder in order to disable the features on startup.

If anyone knows of a driver that fixes this issue please comment.

If anyone at Microsoft knows how to forcibly enable the touchpad options menu so that the gesture options can be changed, please comment.
