a. user say:

Hi Mike,

 

Physical Address Extension (PAE) is a processor feature that enables x86 processors to access more than 4 GB of physical memory on capable versions of Windows. Certain 32-bit versions of Windows Server running on x86-based systems can use PAE to access up to 64 GB or 128 GB of physical memory, depending on the physical address size of the processor.

 

Windows automatically enables PAE if DEP is enabled on a computer that supports hardware-enabled DEP, or if the computer is configured for hot-add memory devices in memory ranges beyond 4 GB. If the computer does not support hardware-enabled DEP or is not configured for hot-add memory devices in memory ranges beyond 4 GB, PAE must be explicitly enabled.

 

You can try the following steps to explicitly enable PAE.

 

a.       Press Windows key +X and select Command Prompt(Admin).

b.      Type BCDedit /set PAE forceenable at the command prompt and press "Enter."

 

If DEP is enabled, PAE cannot be disabled. Use the following commands to disable both DEP and PAE:

 

bcdedit /set [{ID}] nx AlwaysOff

bcdedit /set [{ID}] pae ForceDisable

 

Hope this helps.

Possibly the data is not getting copied properly to the external hard drive, or there is some disk error or bad sector on external hard disk which is preventing the data from being copied.

Let’s try these methods below to check if that solves the problem.

Method 1:

The first possible reason for the issue could be hard drive errors. To check if your hard drive is causing the issue we can try checking the disk for errors and repair bad sectors if any.

To run CHKDSK from command prompt.

  1. Click the Start, type cmd, cmd.exe will be shown above - right click on cmd.exe and select Run as administrator.
  2. OK the UAC prompt if shown. That will open an elevated command prompt. (It will show Administrator in the caption.)
  3. Then enter the chkdsk command
    chkdsk d: /f (d: the external drive letter or whichever letter your external drive is assigned with)
  4. Confirm to run on next startup and reboot.

Computer can’t run a disk check while actively using Windows, but you can schedule it to start automatically the next time your computer is turned on.

When the disk check is finished, your computer will automatically reboot. The disk check will fix any problems it encounters in your hard

Important: While performing chkdsk on the hard drive if any bad sectors are found on the hard drive when chkdsk tries to repair that sector if any data available on that might be lost

Check copying the files again to the external drive.

Method 2:

We can try to format the drive once again with Disk Part Command Line utility.

In order to format a drive through Command Prompt lets open the command prompt.

  1. Click Start > Run and type “CMD”, pressing Enter should open a new terminal. cmd.exe will be shown above - right click on cmd.exe and select Run as administrator.
  2. OK the UAC prompt if shown. That will open an elevated command prompt. (It will show Administrator in the caption.)
  3. Type “DISKPART” into the Command Line and press Enter to open Disk part tool.
  4. Type LIST DISK and press ENTER to get the list of all the disks available.
  5. Type SELECT DISK X and press ENTER (Where x is your external hard disk) to select the external drive disk for further action.
  6. Type LIST VOLUME to list all the volumes on the selected disk.
  7. Type SELECT VOLUME X and press ENTER, (where x is the volume you wish to focus on).
  8. Type “CLEAN” and press Enter. This removes any and all partition or volume formatting from the disk in focus.
  9. Type “CREATE PARTITION PRIMARY” and press ENTER. This creates a primary partition on the current disk. After you create the partition, the focus automatically shifts to the new partition. It is possible to create multiple partitions if you like. To do this you would add the option size=<x> where <x> is the size in megabytes (MB) you would like the partition to be. If no size is specified, the partition continues until there is no more unallocated space.
  10. Type “format fs=ntfs quick ” and press ENTER. This will now perform a quick format, using a ntfs file system.
  11. Lastly type ASSIGN LETTER X and press ENTER, where x is the letter you would like to assign the drive.

shinmila h say:

Hey easternguy,

Check if you can ping the website by its name.

To do this:

a.    Go to Start and type cmd.

b.    Press Enter.

c.    Type the following command:

ping Host name  

Ping uses name resolution to resolve a computer name into an IP address. Therefore, if you successfully ping an IP address but you cannot ping a computer name, there is a problem with host name resolution, not with network connectivity.

Case 1: If you cannot ping the website using its address, follow these steps.

In this case, try editing your host file.

Hosts files in the C:\Windows\System32\Drivers\etc may be damaged or corrupted. 

Examine the content of your HOSTS file, and compare it to the screenshot below. We do not need to worry about any line that begins with an # because is ignored by Windows. Also, the line "127.0.0.1 localhost" can be safely ignored, because it is a standard entry.

Anything else that appears in your HOSTS file without an # at the beginning, apart from the "127.0.0.1 localhost" line, should be viewed with suspicion when we are trying to diagnose the cause of the problem. The quickest way to test for HOSTS file involvement is to right rename the host file. By changing the name of the HOSTS file, we stop Internet Explorer from using it, and therefore resolve any issues caused by the file.

Case 2:

If you could ping external sites by both IP address and Name, but cannot browse the web, likely your web browser is misconfigured.

Double check that you don't have an incorrect or not needed Proxy server assigned.

For the Internet Explorer, follow these steps. 

a.    Open Internet Explorer.

b.    Go to Tools / Internet Options.

c.    Click on Connections tab.

d.    Click on LAN settings button.

e.    Uncheck both "Automatically detect settings" and "Use a proxy server for your LAN".

 

 That should be it.
