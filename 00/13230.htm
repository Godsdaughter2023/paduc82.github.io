In Windows 10, Windows Defender has been rapidly revamped for corporate use - to the point where it’s actually become a bit unfriendly for home users. The best way to get a sense of what’s happened is to page through the current documentation:

Block at First Sight (Seen):

Windows Defender can now immediately block a suspicious or unknown file; upload a sample for analysis; and generate a signature – all within a matter of seconds. And we can use PowerShell to upgrade the default level of protection for this feature: 

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-antivirus/configure-block-at-first-sight-windows-defender-antivirus

https://blogs.technet.microsoft.com/mmpc/2017/07/18/windows-defender-antivirus-cloud-protection-service-advanced-real-time-defense-against-never-before-seen-malware/  

Windows Defender Exploit Guard:

The exploit protection features that were previously provided by EMET are now integrated into Windows 10; and most users shouldn’t need to modify the default settings for these:

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/exploit-protection-exploit-guard

https://blogs.technet.microsoft.com/srd/2017/08/09/moving-beyond-emet-ii-windows-defender-exploit-guard/

Attack Surface Reduction:

We also have the ability to add Attack Surface Reduction rules in Version 1709, but the only practical way to add these rules in Windows 10 Home is with the PowerShell Set-MpPreference cmdlet:

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/enable-attack-surface-reduction

https://blogs.technet.microsoft.com/mmpc/2017/10/23/windows-defender-exploit-guard-reduce-the-attack-surface-against-next-generation-malware/

Controlled Folder Access:

Controlled Folder Access is turned off by default; so you’ll need to turn it on in the Windows Defender Security Center app > Virus & threat protection > Virus & threat protection settings. Once Controlled Folder Access is turned on, standard Windows document folders will be protected by default, and you’ll also be able to add ransomware protection for additional folders, as well as whitelist trusted applications in order to allow them access to your protected folders. If you have trouble whitelisting a friendly app, then you can set this feature to run in Audit Mode, where it will identify access events, but won’t block them.

There’s already a lot of confusion about allowing apps through Controlled Folder Access, and some “Windows experts” are responding to this with a just-turn-it-off “solution”. Now I’ll admit that a dialog with “Block” and “Allow” buttons would make this a whole lot friendlier – but if you just jot down the blocked app’s file path that appears in the “Unauthorized changes blocked” notification; and then click on the notification, this will bring up the “Allow an app through Controlled folder access” window, where you can quickly add the blocked app to the whitelist:

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/exploit-protection-exploit-guard

https://blogs.technet.microsoft.com/mmpc/2017/10/23/stopping-ransomware-where-it-counts-protecting-your-data-with-controlled-folder-access/?ocid=cx-blog-mmpc

PUA Protection:

Windows Defender has actually been able to detect and block Potentially Unwanted Applications for some time now, but many people still don’t know that this feature is disabled by default and needs to be enabled by running this command line at the Administrator PowerShell prompt:

Set-MpPreference -PUAProtection 1

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-antivirus/detect-block-potentially-unwanted-apps-windows-defender-antivirus  

Windows Defender Advanced Threat Protection

The Advanced Threat Protection service is also an available option for enterprise environments:

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-atp/windows-defender-advanced-threat-protection

http://chart.av-comparatives.org/chart1.php?chart=chart2&year=2017&month=11&sort=0&zoom=2

jllatham say:

Certainly you can.  We will set up some code that will sort columns A and B every time you enter a new value into column B.  This is worksheet event code and it's easy to put into place:

Open the workbook, choose the sheet you will be wanting this to work on and right-click on its name tab and choose [View Code] from the popup list.  Copy the code below and paste it into the code module presented to you.  Press [Ctrl]+[Q] to close the VB Editor.

Rules for the code: if you change more than one cell in column B at one time, nothing happens, if you [Del] an entry in column B, nothing happens, if you change anything in some other column, nothing happens.  If there is no data in column B under row 1, nothing happens.

The code is to set up to put the highest seniority people at the top.  Then when 2 or more have the same amount of seniority, they are sub-ordered alphabetically.  If you want lowest seniority at the top, change order1:=xlDescending to order1:=xlAscending

The code:

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim sortRange As Range
  Dim sortKey1 As Range
  Dim sortKey2 As Range
   
  If Target.Cells.Count > 1 _
   Or Target.Column <> 2 _
   Or IsEmpty(Target) Then
    Exit Sub
  End If
  If Range("B" & Rows.Count).End(xlUp).Row < 2 Then
    Exit Sub ' no data to sort
  End If
  Set sortRange = Range("A1:" & _
   Range("B" & Rows.Count).End(xlUp).Address)
  Set sortKey1 = Range("B1")
  Set sortKey2 = Range("A1")
  Application.ScreenUpdating = False
  sortRange.Sort key1:=sortKey1, order1:=xlDescending, _
   key2:=sortKey2, order2:=xlAscending, Header:=xlYes
  Set sortRange = Nothing
  Set sortKey1 = Nothing
  Set sortKey2 = Nothing
End Sub

jllatham say: But what about JOBS: ?  It is probably going to need to be sorted with the other 4 fields also?

So we are faced with 2 options for automatically triggering - either we trigger when you enter the Seniority and you don't enter any JOBS entries until all employees other information is entered and then go back and enter the jobs, or
We trigger on the entry of a JOBS entry, knowing that one entry has been completed.

The code below is set up for your layout and hopefully I've commented it enough so that you can make any changes required in the future.  Just replace the previous code with this:

Private Sub Worksheet_Change(ByVal Target As Range)
  'this is the row that your 5 LABELS are on, not
  'the first row with an employee name in it
  Const labelRow = 9
  'this is the column that you want this to run
  'when a change is made to it, currently set
  'up as F so that we do this when a JOBS entry
  'is made, assuming a whole row is completed
  ' change to E if you want to do it when the
  ' Seniority is entered and come back and do
  ' jobs later (although it is going to sort
  ' jobs at the same time no matter what).
  Const changeCol = "F" ' JOBS Column
  'these are used in setting up and defining the sort
  Const firstColToSort = "B" ' just what it says
  Const lastColToSort = "F" ' just what it says
  Const seniorityCol = "E"
  Const nameCol = "B"
 
  Dim sortRange As Range
  Dim sortKey1 As Range
  Dim sortKey2 As Range
  
  If Target.Column <> Range(changeCol & 1).Column _
   Or Target.Row < labelRow + 1 _
   Or Target.Cells.Count > 1 _
   Or IsEmpty(Target) Then
    Exit Sub
  End If
  Set sortRange = Range(firstColToSort & labelRow & _
   ":" & lastColToSort & Range(changeCol & Rows.Count).End(xlUp).Row)
  Set sortKey1 = Range(seniorityCol & labelRow)
  Set sortKey2 = Range(nameCol & labelRow)
  Application.ScreenUpdating = False
  sortRange.Sort key1:=sortKey1, order1:=xlDescending, _
   key2:=sortKey2, order2:=xlAscending, Header:=xlYes
  Set sortRange = Nothing
  Set sortKey1 = Nothing
  Set sortKey2 = Nothing
End Sub
