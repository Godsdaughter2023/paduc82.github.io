- I find it difficult to wade through walls of text, and it makes it hard to re-reference parts of the post, but...

- Win10 does have a different way of expressing the amount of system RAM that it allocates to the gpu than Win7.

Win10 also does a better job of using some of the excess RAM installed on so many modern machines than previous Windows versions.

My Win7 used around 1.5+ at idle, while Win10 manages to make use of  2GB+ atm.

- My dedicated gpu with 4GB VRAM, currently has 7.9GB RAM Win10 describes as 'shared' with the gpu.

It also shows a total of 11.9 GB 'gpu memory' (of 16GB total) showing in Processes.

Seems pretty close % to your experience.

Nothing at all to be concerned about.

Win10 will reallocate system RAM on a needs basis to whatever requires the system RAM at the time.

The 'share' is just the amount of system RAM that Win10 loosely 'dedicates' to  graphics after discounting the RAM Windows

uses at idle, with a bit of headroom left for whatever else may need some when the graphics are using the RAM.

- You may also notice that the system RAM tab does not show more RAM in use than what should be atm, 'committed'

is not a large amount, and 'available' coincides closely with what is in use.

- Before reading your post, I had no idea Win10 dedicated so much system RAM was allocated to the gpu.

Why ? Because I've never even come close to using my 16GB RAM when running various programs.

I do however have a couple of games that, with what Windows is also using at the time, manage to just break through 8GB system RAM use.

If everything is running smooth... No worries.

- I believe that my Win7 also 'shared' system RAM with the gpu, even without my Intel integrated graphics in use,

but I'll have to boot into Win7 to verify that.

- btw - What configuration or RAM are you using to = 18GB RAM ? 

Shouldn't you be using 12 or 24GB factory matched kits of 3 sticks to properly run a 3 channel RAM configuration

without it running in the less efficient single channel mode vs triple channel ?

(As would be the case with dual channel machines using an odd RAM configuration. )

Hi There,

METHOD TO FIX PIN LOGIN PASSWORD:

1. Log in to your Windows 10.

2. Open File Explorer then navigate to  

C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft

3. Then Right Click on the Ngc Folder, Then go to Properties.

4. Navigate to the Security Tab and click on Advanced.

5. Then locate the Owner Section at the top and click on Change.

6. In Enter the object name to select enter Administrators, Go To Advanced Tab.

7. Then Click On Find Now, Search results will be Shown Below.

8. Then Select The ADMINISTRATORS option not the ADMINISTRATOR.[In My Case It's 2nd Option From Top], Double Click On It, It'll Get Selected.

9. To Verify or to check click on Check Names, Then click Ok.

10. Then Under Owner Section,There will be a option Replace owner on subcontainers and objects select it.

11. Click Apply, Then in Windows Security Tab click on YES then OK.

12. Close all windows and now go back to Ngc folder now you'll be able to Access that folder, open it.

13. Now Shift+Delete all folders in it.

14. All Done, Now you can ADD New PIN Password without any problem.

davem121 say:
Hi Johnny,

Remove the USB Media

Unplug your PC and if it is a laptop, remove the battery

Press and hold the Power button for 30 seconds

Plug your PC back in and if it is a laptop, replace the Battery

Start your PC and see if it boots into Windows

If not, let's see if we can get into the Windows Recovery Environment:

Start your PC, just as Windows attempts to load (spinning dots), press and hold Power Button for 5 - 10 seconds to perform a Hard Shut Down

Do this twice

On the third start Windows will boot into the Recovery Environment and from there you can access System Repair, Safe Mode, Command Prompt . . . etc.

Please try each option one at a time, then see of Windows will boot . . .

Go to Troubleshoot - Advanced Option - Startup Repair
See if Windows startup can repair itself

Go to Troubleshoot - Advanced Option - System Restore
Check to see if you have a restore point you can go back to

If you do not have a Restore Point, go to Troubleshoot - Advanced Option - Startup Settings and click Restart
Upon restart, press 4 to enter Safe Mode
Once in Safe Mode, backup your data if you do not already have a backup
Open Device Manager and check to make sure all your Drivers are up to date
Reboot to see if Windows will start Normally

If the above fails, go to Troubleshoot - Advanced Options - Reset this PC
Choose to keep your files and apps, just your files or to keep nothing
Then start the Reset
