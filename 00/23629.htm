john sr wilson say:

Put a DoEvents statement inside the loop.

In 2010 the screen refresh lines should not be needed.

Declare PtrSafe Sub Sleep Lib "kernel32" (ByVal dwMilliseconds As Long)

Sub WhereSheStopsNobodyKnows()
' Only works in slideshow mode.
'
    Dim sld As Slide
    Dim shpRng As ShapeRange
    Dim shp As Shape
    Dim lnMin As Long, lnMax As Long, i As Long, lnRand As Long, lnPrev As Long
    Dim ssw As SlideShowWindow
   
    'Random = Int((lnMax - (lnMin - 1)) * Rnd) + lnMin
    lnMin = 1
    lnMax = 4
   
     'Check we are in slideshow mode:
    On Error Resume Next
    Set ssw = ActivePresentation.SlideShowWindow
    On Error GoTo 0
    If ssw Is Nothing Then
        Exit Sub
    Else
        Set sld = ssw.View.Slide
    End If
   
    Set shpRng = sld.Shapes.Range(Array("Rectangle 1", "Rectangle 2", "Rectangle 3", "Rectangle 4"))
    For i = 1 To 30
        Do
            Randomize
            lnRand = Int((lnMax - (lnMin - 1)) * Rnd) + lnMin 'get rand num.
        Loop While lnRand = lnPrev 'make sure rand num is different from previous rand num
        lnPrev = lnRand
        Sleep 100 '0.1s delay
       
        shpRng.Fill.ForeColor.RGB = RGB(0, 0, 255) 'set all shapes to blue
        sld.Shapes("Rectangle " & lnRand).Fill.ForeColor.RGB = RGB(255, 255, 0) 'set a random shape to yellow
        DoEvents
'         'Refresh the slide: (Not needed in 2010??)
        ssw.View.GotoSlide (sld.SlideIndex)
        Set shp = sld.Shapes.AddLine(0, 0, 0, 1)
        shp.Delete
    Next i
End Sub

joeu2004 say: MyBlueBuddha wrote:

You seem to have several problems.

You wrote:
they are both formatted as dates.
One date in cell b3 (2/1/2011) and
one date in c3 (2/16/2011)
In d3 I type =c3-b3 hit enter and get the message "#VALUE!"

Despite the numeric format (Date), it appears that the cell contents are interpreted as text.  But even then, =C3-B3 would have a numeric result.

So additionally, it seems that there are spaces or other non-printing characters before and possibly after the dates.  You might try =TRIM(C3)-TRIM(B3).

But I suspect the "spaces" include HTML non-breaking spaces (&nbsp; -- ASCII 160).  So you might try =SUBSTITUTE(C3,CHAR(160),"")-SUBSTITUTE(B3,CHAR(160),"").

And you might need to combine the solutions, e.g. TRIM(SUBSTITUTE(C3,CHAR(160),"")).

Of course, it would be better to convert C3 and B3 rather than do the conversions whenever you reference them as above.

PS:  Based on the following problem, I suspect the cells might have been formatted as Text when you initially entered the data, then changed to Date.  See the additional remedy below.

You wrote:
I have tried using DATEDIF() and get no result at all it just displays the equation in the cell.

You do not need to use DATEDIF if all you want is a count of days.

However, this problem suggests that the cell format was Text when you initially entered the formula.  Even if you change the cell format to General or Number afterwards, the data is still text.  After changing the format, you also need to "edit" the cell by pressing F2, then Enter.
