I think the subform's Current event is way too late to affect itself in all situations.

 

Try setting the property in the main form's Current event:

 

With Me.VegCommunityType.Form

End With

 

You did not provide the name of the subform control, so I guessed it's the same as the form object.

marshall barton say:

Is the main form a Navigation form?

 

If so, there is only one subform control so only one of the subforms is open at any point in time.   Referring to a subform that is not the selected one will then cause problems.

 

OTOH, maybe you're talking about a tab control with the subform on a tab page.  In this situation, there would be no referencing problems.

 

If the tab is not really involved, then it could be that you have left the form's AllowAdditions in the False state when it should be true.  To check for this, I would check the form's property to see and compare that to the number of records in the subform.

 

If none of that helps, chack if you have any code that moves to the next/previous record, deletes a record or something else that can set the subform's position to a nonexistent record.
