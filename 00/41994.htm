Open Registry Editor (Regedit.exe) and navigate to:

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\Desktop\NameSpace]

Backup the key by exporting to a REG file.

Expand the above branch, and delete the following subkey:

{21EC2020-3AEA-1069-A2DD-08002B30309D}
(The above is the GUID for "All Control Panel Items" Namespace)

Similarly, delete the entry here (if found)

[HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\Desktop\NameSpace]

I can think of no good reason to use the Diagnostic mode of msconfig, but I could be missing something.  I don't think I will ever even recommend it because Microsoft says if/when you invoke it:

...you permanently delete all restore points for the System Restore utility.

That might be an unpleasant surprise if you later discover that all your RPs are suddenly missing.

However what you are seeing is a fairly common problem to people that actually know how XP works and I doubt it has anything to do with a corrupt Administrator account but you can sure try that if it is your desire.

Usually you see this when folks are tying to change/disable/enable XP Services from msconfig and when you try to enable Diagnostic mode, behind the scenes, msconfig is trying to change alot of XP Services and is unable to do so, and thus it complains.

You could simplify the issue some by just disabling/enabling one XP Service to see if you get the same thing and also determine if the change "sticks" in spite of the annoying message.

Here are some notes about what you might be seeing:


Trying to change something in msconfig you get access denied error (but the change seems to work):

An Access Denied error was returned while attempting to change a service. You may need to log on using an Administrator account to make the specified changes.

The msconfig changes sometimes seem to work, but the error message is annoying.

Describe your current antivirus and anti malware situation:  McAfee, Symantec, Norton, Spybot, AVG, Avira!, MSE, Panda, Trend Micro, CA, Defender, ZoneAlarm, PC Tools, Comodo, etc.

Remember that msconfig is not a startup manager - it is a troubleshooting tool.  If you want a startup manager, you will have to download some third party tool that is designed to be a startup manager and not use msconfig for purposes for which it is not intended.

Unless you are troubleshooting or testing, you should manipulate your Services the correct way - via the Services applet.

Depending on the antivirus/antimalware applications you have installed, some (such as Norton) essentially put a "lock" on the parts of the registry that would be adjusted from the change and as long as those applications are running, you will not be able to make changes to your system using the msconfig program.  You may  be able to make your changes by booting into Safe Mode to make the changes.

It is a well documented fact that Norton/Symantec products (like Norton Internet Security 2011) will not allow you to manipulate XP Services via msconfig while it is "protecting" your system.   You can read about it in their user forums.

There is a possibility you will lose communications with your HP printer if you make the changes necessary to prevent the error message.  This may not be a problem if you don't care about the communications or never use those HP features.

Here is an article to get you started about running chkdsk:

http://support.microsoft.com/kb/315265

If you want chkdsk to run on boot up, you need to invoke it with some kind of error checking and you can do that from My Computer, select/right click the volume of interest, Properties, Tools, and choose the level of error checking you want and respond in the affirmative to have the chkdsk run on the next reboot.

Or you can do something like click Start, Run and in the box enter (assuming XP is installed on your C drive):

chkdsk  c:  /r

Click OK and respond in the affirmative to have the chkdsk run on the next reboot.

It may take a long time for chkdsk to complete depending on the size of the volume, the amount of data on the volume and what chkdsk finds to do and sometimes it may appear to be 'stuck'.  Be patient.  If the HDD light is still flashing, chkdsk is doing something.  Keep an eye on the percentage amount to be sure it is still making progress.  It may even appear to go backwards sometimes.

When chkdsk runs automatically on a reboot either because XP thinks it needs to (perhaps your NTFS volume has been marked as dirty) or you chose to do it yourself, the results are shown in the Event Viewer Application log.  You need to look there for the chkdsk log and any problems after chkdsk has run.

To see the Event Viewer logs, click Start, Settings, Control Panel, Administrative Tools, Event Viewer.

A shortcut to Event Viewer is to click Start, Run and in the box enter:

%SystemRoot%\system32\eventvwr.msc

Click OK to launch the Event Viewer.

Look in the Application log for an event sourced by Winlogon, something like:

Event Type:    Information
Event Source:    Winlogon
Event Category:    None
Event ID:    1001
Description:

Checking file system on C:

The type of the file system is NTFS.
A disk check has been scheduled.
Windows will now check the disk.

  25151976 KB in 78653 files.
     48256 KB in 10264 indexes.
         0 KB in bad sectors.
    237080 KB in use by the system.
     65536 KB occupied by the log file.
  13632736 KB available on disk.
Windows has finished checking your disk.


If you see any errors - even errors that are corrected, run chkdsk /r again until you see no errors.  You should have no errors.
