From the looks of it, this might be a simple OS version info error.

Deep inside of the Windows operating system is a structure called OSVERSIONINFOEX.  

From the manual, it seems as if any application not specifically compiled for Windows 10 will see the OS version as Windows 8.0 ... Platform targeting is important... and it's possible they took support for Windows 8.0 completely out of their product ... and since Windows 10 reports itself as Windows 8.0 ... that's probably the most likely explanation, here.

ronrosenfeld say:

Hi:

 

I do have my settings on the computer set to dd-mmm-yy, but it won't accept the date when I enter it.  It's extremely frusting to have to enter the entire date when the format is set up.  Kinda makes the formatting useless. :)



Formatting in Excel is useful if you understand that it's purpose is to DISPLAY the date in the desired format.  However, Excel date formatting has NO affect on how the date is parsed when you enter it.

If you want to ENTER dates in the format of  Month Day  (e.g. 2/22)  and have Excel append the current year, then you will need to change your computer short date settings to reflect that order.

Since your computer short date settings are in the order of dmy, Excel interprets your date input in that manner according to it's parsing rules, and you get the result you see.

You can still have the Excel cell format set to dd-mm-yy, but you need to have your windows computer regional settings set differently to enter dates in the manner you wish.

If you do not want to change your computer regional settings, then you will need to enter dates into Excel as d-mm.   e.g. instead of entering 2-22,  enter 22-2  to get 22 Feb 2012

ipellant say:

This seems to be what I called the Beep Beep Lockup.

Moving the mouse or pressing a button causes a beep. Nothing on the screen will respond to the mouse. Keyboard does work, so it is possible to Ctrl Alt Del and Tab to shutdown.

It started happening after the update to 10041. In desperation I downloaded the ISO, burnt a DVD and did a clean install. Problem persisted. Built a desktop with dual monitors then noticed that the lock up mouse beep was triggered by moving the mouse at the time the Theme was swapping the background picture. I tested again a few times and sure enough.... it's the background picture swap being done by the Desktop Theme that is locking the mouse cursor in limbo.

Workaround:

Go to Personalisation.

Click on "Desktop Background"

In the "Picture Location" drop list, select "Solid Colors" and pick a colour.

So far, I've had no repeats of the problem when using a solid colour background.

Ironically, changing Themes can trigger the lockup when the background picture changes and you are moving the mouse at the wrong millisecond.
