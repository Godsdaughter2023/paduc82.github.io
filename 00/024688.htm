m h g say:

Not really. The CELL function originated back in the 1980s, and it was meant to mimic Lotus 1-2-3 Release 2.2's @CELL and @CELLPOINTER functions. The number formats CELL returns are Lotus 1-2-3 number format codes. Excel itself doesn't use 'em, so there's no direct way to use them to produce number formats.

I'm guessing you want numbers formatted but still numbers so they're included in calculations like =COUNT(range), =SUM(range), =AVERAGE(range), =MAX(range), etc. but appear with the intended formats. There's no way to do this using formulas in Excel. User-defined functions (udfs) called by worksheet formulas can't change Excel's state beyond changing the formula's return value.

Other than conditional formatting using formula criteria to access the format codes in other worksheet cells, the closest you could get would be a MYFORMAT udf which stored cell addresses and format codes in a module-level Nx2 array then returned the passed number value. Use Application.OnTime to run a macro in the same module periodically which would format the cells in the array.
