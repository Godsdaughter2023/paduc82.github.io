    graham mayor say:

    Do you need it to be a single document as opposed to separate documents for each defendant (given that presumably these orders are sent to the defendants, so will need to be separated at some point) in order to deliver them?

    With separate documents it is a relatively straightforward task to start a new document from a template, which would contain the cover page and a page break before the case data. Then loop through the worksheet to get the defendant's data and insert it sequentially.  There should be no reason to have blank pages, (unless there are extra breaks are in the inserted text) as long as you set a range where the text is to be inserted.

    Where you insert the fixed texts from is really immaterial. If it is in the cells of an Excel worksheet then you can read that worksheet into an array using ADODB and process the array to get the data in the document. A Word table you can read from directly, if you open the document, and it works well, but is slower.

    Building blocks are already in the template so are inserted quickly, but are more fiddly to set up.

    If you want it all in one document then having processed the first defendant, you insert a section break, and start to compile again. Otherwise you save and close then start again,

    sky pei msft say:

    Hi Markus,

    Sorry for my late response.

    After checking the tenant information and screenshots you provided , finally, I have figured it out the cause of the issue you experienced.

    ===========================

    Issue1:

    You can't deploy add-ins referring to: Publish task pane and content add-ins to a SharePoint catalog.

    Cause:

    You don't have SharePoint Online license, so when you create the catalog site , after loading for a while, you are returned to the "Manage site collection" page,  but the site you entered does not appear.

    Resolution:

    To use this method to deploy add-in centrally, please subscribe a SharePoint Online subscription.

    Issue2:

    You can't deploy addin using Office 365 admin center. Refer to: Deploy Office Add-ins in the Office 365 admin center.

    Cause:

    Please check this article: Determine if Centralized Deployment of add-ins works for your Office 365 organization. To deploy add-ins centrally, you need to subscribe a plan which includes Office ProPlus, while you are using Office 365 Business.


    Resolution:

    To use this method, you need to subscribe a plan including Office 365 ProPlus.

    If you don't have the licenses I mentioned above, you are not able to centrally deploy add-ins. By the way,  I have tested both methods with your add-ins and the method works. You can register a trial plan, such as E3, to do a test(Both of the two methods apply to this plan). If you have any other questions, feel free to start a new thread, and we will keep assisting your from there.

    ============================
