Is there a way to change the margins to 3.5" without doing it - Microsoft Community

mikebookmark say:

Is there a way to change the margins to 3.5" without doing it manually?

Change Margins

When wrote my screenplay in MS Word I set my dialogue margins at 3".  Is there a way to change the margins to 3.5" without doing it manually?
jay freedman say:
The answer depends on two factors:

  • Do you want the margin change to apply to all the text in the document? That is, are there parts of the text that are not dialogue and that should keep the 3" margins?
  • If the change should apply only to specific parts of the document, are those parts formatted with one specific style, different from the style(s) applied to the parts that should not change?

If the change applies to everything, then click the Margins button on the Page Layout ribbon and choose Custom Margins, and use the Left and Right boxes to set the values you want.

If it applies only to the dialogue paragraphs, and if those are formatted with a specific style not used elsewhere, then right-click the style in the Home ribbon and choose Modify; click the Format button and choose Paragraph; and set the Left and Right boxes under Indentation to 0.5" (the additional amount to indent those paragraphs from the 3" margins).

Be careful to understand the difference between "margins" and "indentations". In Word, margins are the distances from the edges of the paper to the edges of the text area, and they apply to the entire document (or to the current section, if you've used section breaks). Indentations (or indents) are the distances from the left and right edges of the text area to the maximum extent of the visible text, and they apply to the current paragraph (initially set by the paragraph's style and may be overridden manually).

It's possible that the 3" margins you referred to are really manually-set indents, in which case you'll need to clear the formatting -- select all the paragraphs and press Ctrl+Q to set the paragraph formatting back to the base style. Then you can modify the style and be confident that the paragraphs will show the style's formatting.

sue mosher say:

When you import the calender in .ics format it stores the data as a separate folder under the default calender in Outlook. After you import the calender as .ics format export it as .csv format from Outlook and save to a location. Later import it back to Outlook so that all the calender items will be mergered with the default Outlook calender. Note: The recurring appointments will not be exported or imported when you do it via .CSV format.

That's way too much trouble. There are two more direct approaches that shouldn't mangle any recurring appointments. 

Method #1: Save the .ics file to the local hard drive, rather than opening it from the web site. Then use the File | Import and Export command to import the .ics file. You should see a prompt asking you whether you want to create a new calendar folder or import into your (default) calendar. 

Method #2: Open the .ics file from the web site and let it create the separate folder. In that folder, switch to a table view, then select all the items and move them to the default Calendar folder.

For your question, the answer is yes. You can use transport rules with connectors if you choose this option within the connector setup process. The image here shows what to select during connector setup.

The problem with a fully updated WDO reporting that definitions are out of date is perplexing. What I believe is happening is that the boot environment is assigning your hard drive as the System Drive instead of your boot drive, so once it starts, it can't find the definitions as it isn't looking at the USB drive for it, but your hard drive. And, it can't establish a network connection, so it reports the out of date situation. All of that is speculation on my part and based on observation of these reports over the last months in the forum.

 

I agree with Le Boule that using a different scanner may be the sensible solution.

Alternatively, you might want to pull the drive from the infected system, attach it as a slave on the good system, and run a scan of that drive using the updated a/v of the clean system.
