cmdrkeene say:

That's some progress.  Normally the desktop folder works the same way, where it will have a "location" tab at the top of the properties box.  This is a function that you'll normally see on special "shell folders" like this (shell being the part of the operating system you see, or the UI).

If it's lost its specialness, you can usually restore it quickly in the registry.

  1. First, just make sure you have a folder named "Desktop" inside your personal "username" folder. You'll be setting the registry to use this folder as the desktop storage location.


  2. Hold Windows Logo and type R to show the Run box, type REGEDIT and click OK, and give administrator authorization if needed.


  3. Navigate the folder list (registry keys) on the left to find this location:
    HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders


  4. Find the "Desktop" entry on the right (registry value) and double-click it to edit the data to be:
    %USERPROFILE%\Desktop


    The %userprofile% is a variable that is wherever your actual profile is stored.  Administrators and power users can use this to move their entire profile without having to set all these locations individually.


  5. Click OK to save it, then close Registry Editor.


  6. Lot out of Windows completely and log back in to re-initialize your user profiles and folder locations. Or just reboot your computer, either way :)

At this point, right-click and properties of the Desktop folder should now have the expected Location tab (let me know if not).

If you do ever want to move these locations, a word of advice:  Make sure you have a folder name specified.  In other words, don't set your Documents to store in just the G: drive. Instead, make a folder on the target drive and be sure to include that in the location (like G:\Docs).  One last thing: don't try to set two "special" folders to be stored in the same single actual location (don't set two to the same location).

Here's a photo of this Registry location on Windows 10, but these steps are the same all the way back to Windows 2000 (maybe earlier).

Let me know how it goes, I hope this gets it all cleared up.
