Hello,

Please try these steps:

1. Open Command Prompt.

4. Or use net use > drives.txt command and then save the command output to a text file. Open it with Notepad and then copy the full path from there.

lastwraith say: While some of these suggestions are nice, here is what worked for me after fiddling with some other possible solutions.

1) Locate the windows update log file and rename (or delete) it - c:\windows\softwaredistribution\datastore\logs\edb.log
2) Either stop the Windows update service, use something like Unlocker, or use safe mode to manipulate the file.
3) I didn't even have to reboot once renaming the file, but it's possible someone might need to.
4) Check for updates and hopefully you are good to go!

Note: Use the correct version of Unlocker for your OS, 32bit did not work on 64bit Win7.

Original post (follows) from Paul Woods on another site

Hi solution works,

but a bit of a sledge hammer approach: the actual cause of the pain is the following file is corrupt

c:\windows\softwaredistribution\datastore\logs\edb.log

if you rename or delete that file (as your solution ultimately does) then reboot the pc, auto updates will resume.

Background:

This file keeps track of the update history and status and if the status cannot be determined then updates won’t run. If the file does not exist (deleted or renamed) it is recreated and it is assumed that updates have never been run.
