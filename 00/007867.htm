albert d. kallal say:

Ok, a bit of confusing here.

 

In general, you never needed some “open database” command in your previous VBA code.

 

It is assumed that you had linked tables to some access back end, and you migrated the access back end (data file) to sql server.

 

At this point in time, then you now have to link your front end (application part) to sql server.

 

Once you done this, then all of your forms, reports and for the most part near everything should now just work as before. You not really see or notice any difference here, and you not in general need to execute some “open database” command.

 

If you have such code in your VBA, then the question would be what “tables” are you attempting to use or open? You as a general rule don’t need to execute any “open database” command, and existing VBA code that operates on linked tables should work as before.

 

About the only change is that you might have say VBA code like this:

 

   Dim rst     As DAO.Recordset

  

   Dim strSQL  As String

  

   strSQL = "select * from dbo_tblHotels where HotelName = 'DAYS INN'"

   Set rst = CurrentDb.OpenRecordset(strSQL)

 

 

The last line in general has to be changed to:

 

   Set rst = CurrentDb.OpenRecordset(strSQL, dbOpenDynaset, dbSeeChanges)

 

So you as a general rule don’t need some opendata base command here

 

You might want to provide a bit more details, but as a “normal” approach here, your existing forms, and even VBA code really should work the same as before, and additional commands such as “open

 

If you need to use say more than “one” database on the server, such as a “test” database, and a “real live” database, then you can run the linked table manger to flip between the two databases (your testing, and then actual live production).

 

Do you have a existing “open database” command that is failing or needs to be changed, or are you attempting to introduce a open database command here?

 

All you need is the linked tables – the code to use and setup reocrdsets etc. in VBA should not require the use of openDatabase.

 

Simply use the above currentDB + open Record set.

bill manville say:

If you create a bookmark (e.g. MyBookmark) in a document then you can go to that bookmark (e.g. MyBookmark) with a hyperlink such as

C:\Temp\MyDoc.doc#MyBookmark

Word creates internal bookmarks for section headings which you can discover by creating a reference to the heading within the document and viewing the field code.  In my case { REF_Ref282698749 /r /h } 

If I then create a hyperlink to

C:\Temp\MyDoc.doc#_Ref282698749

it takes me to the required heading.

I would prefer to use explicit bookmark names unless it is a document you don't control.

    Safe Sender doesn't work for one particular HOTMAIL account - Microsoft Community

    budget programmer say:

    Safe Sender doesn't work for one particular HOTMAIL account

    Hello,

    I have a PHP application that generates an automatic eMail notification whenever someone enrolls on a particular web site.  Three recipients have Hotmail accounts.  Two of the Hotmail recipients receive the mail fine.  One does not.  I've logged into each of the recipients accounts in OutlookLive and checked their settings by navigating to:

    • Settings Gear
    • Options
    • Mail
    • Junk email
    • Safe senders

    I've compared the senders with the problem account with the two that receive fine.  The Safe Senders contain exactly the same name.  None of the recipients have Inbox and sweep rules.

    The Blocked Senders list for all three recipients does not contain a reference to the sender.

    What would cause an eMail to go to the Junk folder for this one particular Hotmail account?

    Thanks for looking at this.

    budget programmer say:

    I figured out a solution.  I was wondering if the entry somehow got corrupted.  I did the following:

    1. Navigated to Settings Cog / Options / Junk email / Safe Senders
    2. Deleted the entry that wasn't being recognized
    3. Signed out of OutlookLive
    4. Closed the browser
    5. Signed in again
    6. Navigated to Settings Cog / Options / Junk email / Safe SendersAdded the email address again

    Everything worked fine after that.

    It seems the entries in this list can get corrupted.

    Hopes this helps someone else struggling with this.
