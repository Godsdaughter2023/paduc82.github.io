lorby say:

Ok, so this one took a little bit of time but I have hopefully sorted it. It is not an issue with IE11 but windows itself.

My standard install is US keyboard, locale en-AU. Sysprep looks a little like this:

            <UserLocale>en-AU</UserLocale>
            <UILanguageFallback>en-US</UILanguageFallback>
            <SystemLocale>en-AU</SystemLocale>
            <InputLocale>0c09:00000409</InputLocale>

Under region and keyboard under preferred language English Australia is the only option there. Unfortunately even though spell check is enabled it doesn't appear to function  with only English Australia set. However adding english United States as a second preference seems to fix this. So I added to current user in the registry using reg add:

"HKCU\Keyboard Layout\Preload" /v 2 /t REG_SZ /d 00000409 /f
"HKCU\Software\Microsoft\TabletTip\1.7" /v EnableAutocorrection /t REG_DWORD /d 1 /f
"HKCU\Software\Microsoft\TabletTip\1.7" /v EnableSpellchecking /t REG_DWORD /d 1 /f
"HKCU\Control Panel\International\User Profile\en-US" /v 0409:00000409 /t REG_DWORD /d 1 /f
"HKCU\Software\Microsoft\CTF\SortOrder\Language" /v 00000001 /t REG_SZ /d 00000409 /f
"HKCU\Control Panel\International\User Profile" /v Languages /t REG_MULTI_SZ /d en-AU\0en-US /f

Adding the second keyboard (and a couple of checks to make sure spellcheck is set to 1) seems to have rectified the issue. Spell check begins underlining words immediately in IE. I have added this to our sccm deployment to the default reg hive so all new machines get these changes from the get go.
