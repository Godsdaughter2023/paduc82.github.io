You cannot customize the built-in formats [Traditional, Classic, Modern, etc.]. bob jones aka: cybertaz say:

Re: "I went to my Normal document and made the changes directly.  I know they 'took' because when I open a new, blank document I do see the changed fields (such as some of the boilerplate I have in my standard footer). "

I believe this is the crux of the deception :-) A template is a "cookie cutter" that stamps out new documents based on its specifications, but...

Once you create a document it is no longer dependent on the template on which it was based, so modifying styles in Normal after-the-fact is proverbially closing the barn door after the cows run off. Styles from the template are encoded within the documents, so existing documents will continue to use the styles as they were defined at the time the document was created. This carries over to the tables builders as well -- you may have modified the TOC styles in your Normal.dotm template, but when you update a table it uses the TOC styles as they are defined in the document, the newly defined versions in the template are not drawn upon.

In order to revamp the styles used in a document you must either do it in the document [that's why I recommended against Add to template - to avoid changes in Normal.dotm] or copy styles of the same name into the document using Organizer.

FWIW, I also subscribe to the school of thought that Normal paragraph style is best left unmodified beyond simple changes [i.e., 'default' font & line spacing] to accommodate routine use of the program. Virtually every built-in style is based on Normal, so changes to it can have some far-reaching & perplexing consequences. Instead {IMHO] it's best to use other built-in styles (such as Body Text) or create custom styles specific to the document.

bob jones aka: cybertaz say:

Your understanding (a) is correct, but (b) not so much :-)

Think of a template as a set of instructions that tells Word how to build a new document... period. The template includes default specifications for page size, margins, etc., as well as styles, macros, keyboard shortcuts, menu/toolbar customizations, Preference settings & even graphic & text content. When a new document based on the template is created, all of that is replicated into the document. From that point on, however, the template has no further influence on the document. [Not to confuse the issue, but a template can be Attached to a document, but that does not happen automatically & is not relevant to this concern.]

To restate (b) more accurately, documents don't have templates as an ongoing element -- The document was based on a template at the time it was created, but it no longer has any association with that template... It is an independent file which contains everything pertinent to it. The original template no longer has any influence on the document. Even if you knew what/where that template was, changes made to it now would not have any impact on existing documents that had been based on it.

The bottom line is that any modifications to the document should be made in the document, itself. If you create/modify a style & check the Add to template box, 1 of 2 things will occur:

  • If Word can locate the original template that template will be updated but your current Normal.dotm will not be changed, but
  • If the original template is not available, Normal.dotm will be updated

charlesyoe say:

Unwanted Space in Word Documents

I am using Word 2016 with Windows 10.  When I paste a figure or some anchored object at the yop of a page the bottom is left blank. I want to wrap text tight around the object but the usual fixes are not working. Wrap text, Paragraph (Keep with next) and other options do not work.

If I place my cursor after the figure title, hit enter to get to a new line then hit delete the cursor jumps to the top of the next page. How can i remove unwated spaces following anchored objects?

doug robbins - mvp office apps & services (word) say:

For the benefit of others, there were a couple of issues:

  1. paragraphs formatted to be kept with next
  2. after removing that attributed from those paragraphs, there was an image on the second page that was anchored to the remaining paragraph at the top of that page and was set to have the text wrap around it.  As the paragraph and the image would not fit in the remaining space on the first page, they were both on the second page.   As it was not necessary for the image to be set to have the text wrap around it, changing that setting so that the image was in-line with the text, then "freed up" the paragraph to which it had been anchored so that it could be located in the space available on the first page.

Here's a somewhat silly suggestion that might provide a way to work around this issue:

 

Create a new workbook with a macro that creates a NEW instance of Excel, and runs your existing chart export macro in that NEW instance.  After every 100 (or whatever) charts are exported, close the NEW instance of Excel, set it to Nothing, and then start over.

jim gordon mvp say:
Excel's default behavior is to store an email address as link, which includes mailto as part of the link. You might be able to prevent excel from adding the mailto command by going to Excel preferences. In the AutoCorrect group uncheck Replace Internet and Network paths with hyperlinks.
