jay freedman say:

The creation of the two headers is covered in http://wordfaqs.ssbarnhill.com/LetterheadRibbon.htm, with additional information at http://wordfaqs.ssbarnhill.com/HeaderFooterRibbon.htm.

Making a footer appear only on the last page is a bit more complicated, because Word doesn't have an option for "last page" the way it does for "first page". The first question that must be answered is whether there is any footer on the pages other than the last one. 

If there is no footer except on the last page, the simplest method would be to put the address at the end of the body text, with a large enough "Space Before" setting on its first line to push it to the bottom of the page.

The alternative (discussed in http://wordfaqs.ssbarnhill.com/HeaderFooterRibbon.htm#CleverTricks) is to enclose the three lines in an IF field like this:

{ IF {PAGE} = {NUMPAGES} "<the 3-line address here>" "" }

(This assumes that the document doesn't contain any section breaks that restart the page numbering. That would require a somewhat different approach.) Note that each of the three pairs of braces must be inserted by pressing Ctrl+F9. You can't just type them on the keyboard.

If there is some other footer on the pages before the last one, then the same kind of IF field can be used; just put the before-the-last-page footer's text inside the last pair of quote marks.

chip pearson say: Strictly speaking, CTRL SHIFT 8 selects what is called the CurrentRegion of the active cell. The CurrentRegion of a cell is that rectangular range of cells containing the active cell and that is bounded on all sides by empty cells (or the edge of the sheet).

There is no way, without resorting to VBA programming, to omit the first line of the region from the selection.  You could use the following code, attach it to a keyboard shortcut to do what you are looking for:

Sub SelectRegionLessGTop()
Range(ActiveCell.CurrentRegion(2, 1), _
    ActiveCell.CurrentRegion.Cells(ActiveCell.CurrentRegion.Cells.Count)).Select
End Sub

j.e. mcgimpsey say: Your macro doesn't tell Word to just find bold characters, it finds ALL characters. To limit the replacement to bold characters you need to set the Find object's .Font.Bold property to True, e.g.:

    With Selection.Find
        .ClearFormatting
        .Font.Bold = True
        .Text = "(?)"
        With .Replacement
            .ClearFormatting
            .Style = ActiveDocument.Styles("bold")
            .Text = "\1"
        End With
        .Forward = True
        .Wrap = wdFindContinue
        .Format = True
        .MatchCase = False
        .MatchWholeWord = False
        .MatchSoundsLike = False
         .MatchAllWordForms = False
        .MatchWildcards = True
        .Execute Replace:=wdReplaceAll
    End With

The use of a connected outlook data file is not supported on a Lan/Network/OneDrive as it will lead to corruption of the data file.

2) File>Open Outlook data File.......... open the file you copied......... Then in the Outlook Tree View pane you can deal with its contents as required
