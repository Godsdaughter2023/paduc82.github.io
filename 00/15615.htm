Macro to automatically hide empty rows in Excel 2010 - Microsoft Community

ooglethepoogle say:

Macro to automatically hide empty rows in Excel 2010

Hi!

I have several worksheets (Let's call  them 'Worksheet_1' to 'Worksheet_4') whose first column is populated as a result of making a yes/no selection on a central worksheet ('Overview') using the simple formula:

=IF('Overview'!$B7="yes", 'Overview'!$A7,"")

This results in a number of rows being empty on the worksheets ('empty' apart from the formula in column A)

I would like to write a Macro that would hide those empty rows, but did so automatically whenever a user selects "yes" or "no" on the overview page (which would update the worksheets).

Would be really grateful for some help on this.

andreas killer say:

Copy the code below into the code module of the sheet "Overview".

Andreas.

Option Explicit
Option Compare Text

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim Ws As Worksheet
  Dim All As Range, R As Range
  'Has something changed in column B?
  If Not Intersect(Target, Me.Columns("B")) Is Nothing Then
    'Check all sheets
    For Each Ws In Worksheets
      Select Case Ws.Name
        'And in this ones
        Case "Worksheet_1", "Worksheet_2", "Worksheet_3", "Worksheet_4"
          'Get all used cells in column A in that sheet
          Set All = Intersect(Ws.Columns("A"), Ws.UsedRange)
          'Show them
          All.EntireRow.Hidden = False
          'Find all cells which contain and empty string
          Set R = FindAll(All, "")
          'Hide them if any
          If Not R Is Nothing Then R.EntireRow.Hidden = True
      End Select
    Next
  End If
End Sub

Function FindAll(ByVal Where As Range, ByVal What, _
    Optional ByVal After As Variant, _
    Optional ByVal LookIn As XlFindLookIn = xlValues, _
    Optional ByVal LookAt As XlLookAt = xlWhole, _
    Optional ByVal SearchOrder As XlSearchOrder = xlByRows, _
    Optional ByVal SearchDirection As XlSearchDirection = xlNext, _
    Optional ByVal MatchCase As Boolean = False, _
    Optional ByVal SearchFormat As Boolean = False) As Range
  'Find all occurrences of What in Where
  Dim FirstAddress As String
  Dim c As Range
  'From FastUnion:
  Dim Stack As Object 'Dictionary
  Dim Temp() As Variant
  Dim i As Long, j As Long

  If Where Is Nothing Then Exit Function
  If SearchDirection = xlNext And IsMissing(After) Then
    'Set After to the last cell in Where to return the first cell in Where in front if _
      it match What
    Set c = Where.Areas(Where.Areas.Count)
    'BUG in XL2010: Cells.Count produces a RTE 6 if C is the whole sheet
    'Set After = C.Cells(C.Cells.Count)
    Set After = c.Cells(c.Rows.Count * CDec(c.Columns.Count))
  End If

  Set c = Where.Find(What, After, LookIn, LookAt, SearchOrder, _
    SearchDirection, MatchCase, SearchFormat:=SearchFormat)
  If c Is Nothing Then Exit Function

  'Initialize our internal stack
  Set Stack = CreateObject("Scripting.Dictionary")

  FirstAddress = c.Address
  Do
    Stack.Add Stack.Count, c
    If SearchFormat Then
      'If you call this function from an UDF and _
        you find only the first cell use this instead
      Set c = Where.Find(What, c, LookIn, LookAt, SearchOrder, _
        SearchDirection, MatchCase, SearchFormat:=SearchFormat)
    Else
      If SearchDirection = xlNext Then
        Set c = Where.FindNext(c)
      Else
        Set c = Where.FindPrevious(c)
      End If
    End If
    'Can happen if we have merged cells
    If c Is Nothing Then Exit Do
  Loop Until FirstAddress = c.Address

  'FastUnion algorithm © Andreas Killer, 2011:
  'Get all cells as fragments
  Temp = Stack.Items
  'Combine each fragment with the next one
  j = 1
  Do
    For i = 0 To UBound(Temp) - j Step j * 2
      Set Temp(i) = Union(Temp(i), Temp(i + j))
    Next
    j = j * 2
  Loop Until j > UBound(Temp)
  'At this point we have all cells in the first fragment
  Set FindAll = Temp(0)
End Function

brian tillman [outlook mvp 2007-2019] say:

You cannot rename the default Inbox folder.

The interface was tweaked somewhat recently and "Options" replaced "More mail settings".

Visit the "Manage Rules" link and add a rule to put the messages addressed the Hotmail address in a different folder and remove the rule that moves the messages addressed to the Outlook.com address.

The Outlook.com and the Hotmail.com addresses are aliases of each other.  This means that no matter which address people use, both will deliver to the same mailbox.  There is no forwarding needed.

In the mail merge facility, change the type of document from a mail merge document to a Normal Word document, which will strip the data source information from it, then change it back to whatever mail merge main document type that it was and then attach the data source to the main document and save it.

If that does not solve the issue then I would think that there is some more fundamental problem with the way that you have the network setup.
