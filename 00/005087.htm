jim gordon mvp say:

I can't say for sure about your specific case, but some likely culprits:


1. Along the say, Apple stopped support PICT file format, and Microsoft Office followed suit. Copy/Paste of images used to use PICT file format as the format for the pasted images. Since PICT is no longer supported by Mac OS, Office can't display PICT format pictures any more.  So depending on what version of Mac OS and Office was being used, some older files may contain PICT format pictures that can no longer be displayed.


You should be saving documents in the current standard file formats, not the deprecated (.doc) file format. This link is for Word 2011, but the same formats are supported by Word 2008.

Office 2011 for Mac: Compatibility among Word File Formats


2. Word doesn't change the resolution of pictures that are embedded. Copies of the picture files are kept within the Word file. In the current standard file formats, in Finder you can change the file extension to (.zip) and then double-click the (.zip) file. You can then navigate the folder structure and see the media files. You could use this method to try to recover those pictures displayed with a red X. If you can get at the original files this way, use a graphic editor to change the file formats from PICT to something more usable such as PNG or JPEG. Then reinsert the updated pictures into your Word document.

bob jones aka: cybertaz say:
Ahhh... that discloses a great deal.

The red X means that the graphic is "missing" so Word cannot recreate the image. One of the primary causes is that the image was inserted as a Linked image but the document and/or the image file has been moved from its original location. As a result, Word cannot locate the image file because of the path being inaccurate -- essentially the link is broken.

Another major cause is Copy/Paste, because only screen pixels are being pasted -- and a good bit depends on the source of the copying as well. Not to dismiss your experience, but pasted images are quite prone to eventual failure regardless of which software is used to open the document containing them. It's a form of corruption, as you surmised.

What I'd first suggest is that you use discreet image files by way of the Insert> Picture command whenever possible. [The wording varies from one version of Word to another, but I'm sure you'll recognize it. There's typically a button, palette or Ribbon section containing it as well.] That enables Word to actually read the file from its source using filters built into the program, which, in turn, allows the file to actually store a copy of the image rather than an assortment of pixels.

Understandably, you have no control over the methods used by others & Copy/Paste is one of the most commonly used 'conveniences' on any OS... Unfortunately, it's also one of the most misunderstood, misused and over-rated as well. The utility never was meant to be used for anything but text. However, it got out of control as graphics capabilities evolved & most developers have done nothing to dissuade users from it.

I think you're being a little harsh wrt the users :-) but the fact is that with Office 2007, 2010 [PC] & Office 2008, 2011 [Mac] .dotx is the default format, so it's pretty much transparent to basic users. Also, Office 2003 {PC] & 2004 [Mac] do the conversion on the fly so there is no extra step involved. Those earlier versions cannot create OOXML files, but opening, editing & saving the work shouldn't be a problem. I'm not trying to sell you on using OOXML file formats -- Just saying that at this point in time it's pretty much the defacto format for virtually anyone who's bought a new computer or a new version of Office in the last 5 years.

Bob J.

hansv mvp say:

You mention "blank records" (plural). There should be only one blank record at the bottom of the table - the new record with an asterisk * in its record selector on the left hand side.
