jon perkins say:

RESOLVED: While the bug in the driver still remains in build 14352, there is a workaround. The driver in question is purely to mount an Apple volume, so in my case where I only have the Windows OS installed this is not needed and so can be disabled - leading to a clean reboot after the update.

To do this, open a Command Prompt window with Admin privileges and type:

sc config AppleMNT start=disabled

and then let the Windows update proceed. All worked well from here on.

Just posting this update as a reference to anybody else having the problem.
