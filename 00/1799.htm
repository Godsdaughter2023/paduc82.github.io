The basic problem has to deal with the different locks taken by the Word Web App and Microsoft Word 2010.  

 

The Word Web App takes an exclusive lock on the file.  This prevents all other users from editing the file until the lock is removed.  The lock should get removed every time you leave your editing session, or 30 minutes without a lock refresh.  (The lock is refreshed periodically during the editing session to make sure that it doesn't expire in the middle of your work.)  The lock is necessary to ensure that nobody is changing the file out from under you, while you are working on it.  When the Office Web Apps try to open a Word document containing this exclusive lock they show the error message you described:  “Word Web App cannot open this document for editing because it is currently being edited by another user.”  If Microsoft Word 2010 tries to open a file containing this lock, it will offer to open this file in read-only mode.


Microsoft Word 2010 takes a shared lock on the file.  This allows multiple instances of Microsoft Word 2010 to be editing the file simultaneously, but still prevents the Word Web App from opening the document.  This can be done because the Microsoft Word 2010 support co-authoring but the Word Web App does not.  Under normal circumstances, this lock should be removed when you close Microsoft Word 2010.  When the Word Web App tries to open a file that currently has this shared lock applied to it, it shows an error message along the lines of "This file is currently being edited in Microsoft Word 2010. You can edit this file along with that user by opening the document in Microsoft Word 2010"

 

Since you are describing the first of these two error messages, it does not sound like Microsoft Word 2010 is failing to release its lock.  Let's try getting to the bottom of this together though, ok?  I was wondering if you could try a handful of experiments and tell me whether they succeed or fail.  If they fail, any error information that you can provide will be helpful.

 

The first test is to open the file in the Office Web Apps twice.  Create a new document in the Word Web App on SkyDrive using the online interface that SkyDrive provides.  Open this document in the Word Web App, make a change, save your change, and leave.  Then try to re-open this document in the Word Web App for editing.

 

The second test is to open the file in Microsoft Word 2010 twice.  Create a new document using Microsoft Word 2010, and save it (through Word) to SkyDrive.  This can be done in the backstage view (click the file button) under "Save & Send". Now, close Microsoft word, re-open Microsoft Word and try to directly open the document that you just saved to SkyDrive.

 

 

The third test is to open the file in the Office Web Apps after creating it in Microsoft Word 2010.  Create a new document using Microsoft Word 2010, and save it (through Word) to SkyDrive.  This can be done in the backstage view (click the file button) under "Save & Send". Now, close Microsoft word.  Then try to re-open this document in the Word Web App for editing by logging into SkyDrive.

 

The forth test is to open the file in Microsoft Word 2010 after creating it in the Office Web Apps.  Create a new document in the Word Web App on SkyDrive using the online interface that SkyDrive provides.  Open this document in the Word Web App, make a change, save your change, and leave.  Open Microsoft Word and try to directly open the document that you just saved to SkyDrive.  This can be done most easily by copying the URL path that you were previously saving to (when using the "Save & Send" feature earlier) and pasting it in the Open dialog.

 

 

These four tests will help me isolate if the issue you are running into is with the Word Web App, Microsoft Word 2010, or a specific interaction between these two applications.

xl_geek say:

Hi,

I don't think this would be possible with the inbuilt excel "clear filer".

However this can be achieved using one line of VBA easily.

Just paste this code in a new module in VBA

Sub ShowData()

ActiveSheet.AutoFilter.ShowAllData

End Sub

In order to add the code:

  • Press Alt + F11 to open vba
  • Click Insert > Module
  • Paste this code
  • Unprotect the sheet in question
  • On the worksheet , insert any shape (for a button), add a label .. say "Clear Filter"
  • Right click on the shape> Assign Macro > Select "ShowData"
  • Protect the sheet

On clicking this new button "Clear Filter", filter will be cleared. (no need to unprotect sheet)

Hope this helps.
