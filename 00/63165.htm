HP OfficeJet 6500 wireless will not print since I changed my wireless - Microsoft Community

pamela0803 say:

HP OfficeJet 6500 wireless will not print since I changed my wireless router 2 days ago

Original Title : Printer Print printing

 

My HP OfficeJet 6500 wireless will not print since I changed my wireless router 2 days ago. The printer is out of warranty and I'm using Windows 7 OS. I

lemp say:
Assuming that there is a wireless connection to the printer, the most likely cause of the problem is that the printer is configured to obtain its IP address automatically and the new router has assigned the printer an IP address that is different than the one that was originally assigned.

To check if the above is the issue:

Use the printer's front panel controls to print a network configuration page:
From the Home screen, touch right arrow, and then select Setup.
Touch Reports, and then select Network Configuration Page.

Find the printer's IP address on the Network Configuration page.

On your computer, go to Devices and Printers
  • Right click on the icon for your printer and select "Printer properties"
  • Click the "Ports" tab
  • WITHOUT clicking anywhere in the list of ports other than the scroll bar on the right, find the highlighted line (the one with the check mark in the box)
  • Is the "Port" the same as the printer's IP address that you noted on the Network Configuration page?
  • Assuming that the port in the printer's properties dialog is not the same as that shown on the Network Configuration page, click the "Configure Port" button and enter the correct IP address in the "Printer name or IP address" box
  • OK your way out

To prevent this problem from happening again, read your printer's User Guide and configure the printer to use a fixed or static IP address.  Choose an address that is in the same subnet as the router but outside the range of addresses used by the router's DHCP server.  You can find this information from your router's configuration utility.


For example, most Linksys/Cisco routers use IP addresses of the form 192.168.1.x and allocate addresses in the range 192.168.1.100 to 192.168.1.150.  In this situation, you should pick a static IP address such as 192.168.1.20  (20 is outside the 100 to 150 range).

Hi,

Stopping services to gain speed is actually a myth. You might see some improvement
using some benchmarking tools however seldom any difference in the real world.

Services have dependencies which affect not only that chain but others that share even
one of the dependencies. So there is no way we can tell you specifically which service
(or services) need to be running. So setting back to the defaults is the best move you
can make.
