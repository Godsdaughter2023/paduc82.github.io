teylyn say:

Hello,

what is in row 1? Do the numbers start in row 1 or do the cells contain text?

If row 1 contains text (i.e. labels) and column A contains the X values, temporarily delete the text in A1, then plot the chart, then put the text back in. 

If row 1 contains the first data point, insert a blank row before row 1, enter some text in cell B1 and leave A1 empty. Then create the chart.

Explanation: When Excel encounters numbers while trying to build a chart, it will first try to interpret these numbers as values for a data series. Whenever you have numerical data that needs to be plotted on the X axis (X values in an XY chart, dates or numbers as the categories in a column chart, etc), make sure to have an empty cell as the column header in the source data. Then Excel will treat this data correctly.

Screenshot: the correct example is on the left. Excel treats column A as X axis values and will set the text in B1 as the Series name.

The chart on the right is the result when both data columns in the source data contain labels in the first row, or if the data has no labels. Excel will interpret both data columns as series values and will build a chart with two series.

rvdd_let say:

These registry setting solved my problem:

Set the DelayOleSrvParseDisplayName registry entry and the NoActivateOleLinkObjAtOpen registry entry to enable the hotfix. To do this, follow these steps:

  1. Start Registry Editor.
  2. Locate and then select the following registry subkey:
    HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Word\Options
  3. On the Edit menu, point to New, and then click DWORD Value.
  4. Type DelayOleSrvParseDisplayName, and then press ENTER.
  5. Right-click DelayOleSrvParseDisplayName, and then click Modify.
  6. In the Value data box, type 1, and then click OK.
  7. On the Edit menu, point to New, and then click DWORD Value.
  8. Type NoActivateOleLinkObjAtOpen, and then press ENTER.
  9. Right-click NoActivateOleLinkObjAtOpen, and then click Modify.
  10. In the Value data box, type 1, and then click OK.
  11. On the File menu, click Exit to exit Registry Editor.

ioanirimia say:

Is there a way to select multiple numbers in Microsoft Word and change them all by a certain value or mathematical equation?

Say, I select multiple numbers, 25, 63, 94 and I want them all +4, so they will be 29, 67, 98 in Microsoft Word. Can I do that? Or do I have to change them manually?
macropod say:

This kind of process could be done with a macro. For example:

Sub Demo()
Application.ScreenUpdating = False
Dim i As Long, StrVal As String
With ActiveDocument.Range
  StrVal = InputBox("What is the amount to increment all numbers by", "Number Incrementer", 0)
  If Not IsNumeric(StrVal) Then GoTo ErrExit
  i = CLng(StrVal)
  With .Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .Text = "<[0-9]{1,}>"
    .Replacement.Text = ""
    .Forward = True
    .Wrap = wdFindStop
    .Format = False
    .MatchWildcards = True
    .Execute
  End With
  Do While .Find.Found
    .Text = CLng(.Text) + i
    .Collapse wdCollapseEnd
    .Find.Execute
  Loop
End With
ErrExit:
Application.ScreenUpdating = True
End Sub

The page numbers in all TOCs in all versions of Word are hyperlinked by default (and there is no way, AFAIK, to disable this hyperlinking). Private Function TimerFunc(ByVal hWnd As Long, ByVal wMsg As Long, _
    ByVal nIDEvent As Long, ByVal dwTime As Long) As Long
  'Called from the windows timer?
  If wMsg = WM_TIMER Then
    Debug.Print "Windows " & Now & ": " & hWnd & "," & nIDEvent

    Debug.Print "OurSelf " & Now & ": " & hWnd & "," & wMsg & "," & nIDEvent
  End If
End Function

  hWnd = GetForegroundWindow
