jay freedman say:

With the requirements you've listed -- particularly the need to use the form on Android -- there's only one approach that will work. All the others (https://gregmaxey.com/word_tip_pages/content_controls_do_not_print_pht.html) require macros.

Click the Design Mode button on the Developer tab, select each content control in turn, and replace the placeholder text "Click here..." with underlined nonbreaking spaces (Ctrl+Shift+spacebar). Then turn off Design mode. The underlines will appear, indicating where the user should enter text. If the user wants to remove the underlines from unused controls, it's necessary to click in the control and type a space, or delete the control itself.

I've tested this on an Android phone, and the underlines do appear. The controls behave mostly as they do in Windows Word, but the bounding box and title tag don't appear when the control is selected, which is why you need underlines (or some other means of indicating where the control is).
