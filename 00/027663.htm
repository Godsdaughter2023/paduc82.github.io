Re:  textboxes - more
I do see what you mean in that copying and pasting (textboxes) in xl2003 placed the tb in the same location relative to the cell corner.  XL2010 places the tb at the top left corner of the cell.

You can use the arrow keys to move a selected textbox.  It is very precise.
You can also use VBA to determine the position of a textbox and use those measurements to add a textbox to additional sheets...
'---
Option Explicit
Public L As Double
Public t As Double
Public w As Double
Public h As Double

Sub GetSpecs()
  L = ActiveSheet.Shapes("Textbox 1").Left   '<<<<TB name must be correct
  t = ActiveSheet.Shapes("Textbox 1").Top
  w = ActiveSheet.Shapes("Textbox 1").Width
  h = ActiveSheet.Shapes("Textbox 1").Height
 'MsgBox L & vbCr & t & vbCr & w & vbCr & h
End Sub
'---

Sub MoreOfTheSame() 'left,top,width,height
  ActiveSheet.Shapes.AddTextbox msoTextOrientationHorizontal, L, t, w, h
End Sub

Hi Brandon,

I had a case few weeks ago about free/busy problems from the Microsoft Cloud to Onprem.

The envirionment was mixed with Exchange2013/2010 servers,  Exchange 2013 Hybrid.

I have a few questions about the issue

Did you the check sharing policy?

Are you able to obtain free/busy information from the remote connectivity analyzer?

The solution was:

get-WebServicesVirtualDirectory |Set-WebServicesVirtualDirectory -WSSecurityAuthentication $true

get-autodiscoverVirtualDirectory  |set-autodiscoverVirtualDirectory  -WSSecurityAuthentication $true

The WSSecurityAuthentication property was already set, but for some reason it doesn't work after installing updates.

One I set the WSSecurityAuthentication property again, Free/busy was working immediately.

* Open File Manager and navigate to the file.
