The most common cause of the problem is that the Outgoing (SMTP) server has not been set to authenticate with your POP3 server

kergga say:

Is the data in column L numeric? If so, you can return that as a single value within your sumproduct;

 

=SUMPRODUCT(--(A:A=D2),--(B:B=E2),--(F2=C:C),L:L)

 

And if it isn't numeric, then just return the row number and wrap Index around the whole statement:

 

=SUMPRODUCT(--(A:A=D2),--(B:B=E2),--(F2=C:C), Row(L:L))

then

=index(L:L,SUMPRODUCT(--(A:A=D2),--(B:B=E2),--(F2=C:C), Row(L:L)))

bernie deitrick say: If you are positive that the value will be unique:

=INDEX(L:L,SUMPRODUCT((A:A=D2)*(B:B=E2)*(C:C=F2)*ROW(C:C)))

Though it will be a faster formula if you don't use entire columns:
=INDEX(L:L,SUMPRODUCT((A1:A100=D2)*(B1:B100=E2)*(C1:C100=F2)*ROW(C1:C100)))

and if it is at all possible that the values will be duplicated:
=INDEX(L:L,SUMPRODUCT(MAX((A1:A100=D2)*(B1:B100=E2)*(C1:C100=F2)*ROW(C1:C100))))

per jessen say:

This should do it:

 

Sub ConsolidateSheets()
Dim TargetSh As Worksheet
Dim DestCell As Range
Dim LastRow As Long
Dim sh As Worksheet

With Application
    .ScreenUpdating = False
    .Calculation = xlCalculationManual
End With

On Error Resume Next
Set TargetSh = Worksheets("Consolidated")
On Error GoTo 0
If TargetSh Is Nothing Then
    Set TargetSh = Worksheets.Add(before:=Sheets(1))
    TargetSh.Name = "Consolidated"
Else
    TargetSh.Cells.Clear
End If
Set DestCell = TargetSh.Range("A1")
Sheets(2).Range("A1:H1").Copy DestCell 'copy header
Set DestCell = DestCell.Offset(1)

For Each sh In ThisWorkbook.Sheets
    If sh.Name <> "Consolidated" Then
        LastRow = sh.Cells.SpecialCells(xlCellTypeLastCell).Row
        sh.Range("A2", sh.Range("H" & LastRow)).Copy Destination:=DestCell
        Set DestCell = DestCell.Offset(LastRow - 1)
    End If
Next

With Application
    .ScreenUpdating = True
    .Calculation = xlCalculationAutomatic
End With
End Sub

teylyn say:
Hello,

by default, Windows hides the file extension. To my mind this is a very stupid setting, because it takes information away from the user.

If you want to always show the file extension, so you can see if you are dealing with an .xls file or an .xlsx file, do this (using Windows 8)

- Open Windows Explorer
- On the View ribbon, click "Options" at the very right
- This will open the Folder Options dialog with three tabs.
- Select the "View" tab
- in about the 10th row there is an option called "Hide extensions for known file types". By default, this setting is ticked.
- untick the setting and click OK.
- now you will always see the file extension and can use your knowledge of file extensions to identify files.
