john korchok say:

For setting margins for the whole table, Word normally works as you expect it to. Your steps 1-5 work here. However, this can be defeated by fixed-height rows (Table Tools>Layout>Properties>Row>Row height is: Exactly). If that's not the issue, you could try running a repair on Office. Let me know if you need the steps to do that.

When you start setting cell margins, there are some gotchas. You discovered one: if two or more cells have different cell margins, selecting them and setting the margins for both at the same time does nothing. You can select and change margins if the cells already have the same margins, but not if they're different.

Another peculiarity is that a row will have a top or bottom margin equal to the largest custom cell margin in a row. So if one cell has a 0.5" top margin, but the table and all the other cells have a 0" top margin, all the cells in the row will have a 0.5" margin. This, combined with the inability to select multiple cells to change margins means you have to go through the row cell by cell until you find the one with different settings.

When I run into a document that has problem tables, I'll often record a simple macro that sets a cell to the margins that I need. Then I assign a keyboard shortcut and tab through each cell, hitting the shortcut key every time. That helps. It's not much more work to write a general purpose macro to go through each cell in a selected table, then I've got the fix down to 1/2 a second.

jim gordon mvp say:
If you install Outlook 2016 you can import your data from the 2011 identities. Once you are satisfied all your account information, mail, etc. has been ported successfully, you can safely delete the 2011 user data. Outlook 2016 does not use the 2011 files except to use it as a source to copy to a new database for 2016. Once copied, the source can be discarded.
