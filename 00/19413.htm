scottlutes say: I am not sure which version of Excel you are using, but after doing a little research, it looks like there are two ways to handle adding pictures within Excel 2007.  I have modified my original code and have tested it on my 2007 environment.  With this, I give you a caveat.  All of the locally saved pictures should be in the same format (i.e. jpg, bmp, png, etc), and the ".jpg" portion of the code modified to match.  Leaving off the extension does not appear to work properly for me and changing between format does not work without changing the code.  Take a look at it and see if you can use it, or modify it to what you need.

[CODE]
Option Explicit

Private Sub Worksheet_Activate()
   
ActiveSheet.Pictures.Delete

On Error Resume Next
Dim cell As Range

For Each cell In Range("B7")

If InStr(1, cell, "/", 1) Then

    Dim u As Variant
    Dim p As Picture
    u = Range("B7").Value
    p = ActiveSheet.Pictures.Insert(u)

        With ActiveSheet.Shapes(ActiveSheet.Shapes.Count)
            .Top = Range("B7").Top
            .Left = Range("B7").Left
            .Height = 240
        End With
Else

If InStr(1, cell, "\", 1) Then
    Dim strPicName As String
    strPicName = Range("B7").Value
   
        With ActiveSheet.Pictures.Insert(strPicName & ".jpg")
            .Left = Range("B7").Left
            .Top = Range("B7").Top
            .ShapeRange.LockAspectRatio = msoTrue
            .ShapeRange.Height = 240
        End With
End If
End If
Next
End Sub

jay freedman say:
The problem is that you named the macro -- Sub Dir() -- with the same name as the function you're trying to call -- Dir("C:\WINDOWS\WIN.INI"). The Sub takes zero arguments, but the call has one argument. In the VBA interpreter, the Sub gets "precedence", that is, it assumes that the call refers to the macro and not to the built-in function.

The solution is to change the name of the macro to something that isn't already built into VBA, such as MiDir().

djsteele, mvp say:
  • The concept of "Trusted Locations" does not apply to versions of Access older than Access 2007
  • Unfortunately, the location in the registry required to set the trusted location is different for each version of Office that requires it be set, so no, it's not possible to have a fix that will work for each version of Office. If using a simple .REG file to update the registry, you'd have to include entries for each version of Office. If using code, you could check to see which version(s) of Office are already installed, and only make the necessary entries.
  • AFAIK, the packaging wizard includes the necessary update to the registry. This makes sense: since the user has already confirmed that they want the wizard to run, they've granted implicit permission to make the necessary updates to the registry.

Please be informed that the emails in our Outlook.com page still appears in the web interface because the POP setting in your Microsoft account is not properly configured. I will give you the steps to change it however, I'd like to remind you that changing your POP settings might cause you to be unable to recover emails that you may lose or accidentally delete from the account, as the messages will not leave a copy from the servers after they were downloaded in your device. Here are the steps:

  1. Go to your Inbox.
  2. Click the gear icon next to your profile name and then select More mail settings.
  3. Under Managing your account select Pop and deleting downloaded messages.
  4. Tick on Do what my other program says-if it says to delete messages, then delete them.
  5. Click Save.
