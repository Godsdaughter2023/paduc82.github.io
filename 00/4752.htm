mike middleton say:

If you are using a Line chart type, the x-axis data (whether the data are numeric or text) are displayed as equally-spaced text labels. That is, there will be the same horizontal spacing between 2003 & 2004 as there is between 2030 & 2035.

If you use an XY Scatter chart type, the x-axis data must be numeric, and the horizontal spacing will depend on the numeric values. That is, the horizontal spacing between 2030 & 2035 will be five times the distance between 2003 &2004.

I no longer have Mac Excel 2008 available for testing, but with an XY Scatter chart type the Format Axis > Scale command (or similar command) should show options for setting the Minimum, Maximum, and Major Unit numerical values, in which case you would specify Minimum = 2000, Maximum = 2040, and Major Unit = 20.

hansv mvp say:

In Access, press Alt+F11 to activate the Visual Basic Editor.

Select Insert > Module to create a new code module.

Select and copy the code from Public Function WorkingDays2(... up to and including End Sub.

Paste it into the module window in Access.

You mentioned that you had a table named holidays with a field named holiday date.

That means the code must be edited slightly: change the line

Set rst = DB.OpenRecordset("SELECT [HolidayDate] FROM tblHolidays", dbOpenSnapshot)

to

Set rst = DB.OpenRecordset("SELECT [Holiday Date] FROM Holidays", dbOpenSnapshot)

and change the line

rst.FindFirst "[HolidayDate] = #" & StartDate & "#"

to

rst.FindFirst "[Holiday Date] = #" & StartDate & "#"

If the database may be used by someone who does not use US date format m/d/yyyy, it's better to use

rst.FindFirst "[Holiday Date] = #" & Format(StartDate, "yyyy-mm-dd") & "#"

That should work regardless of the user's system date format.

That's it. Press Alt+F4 to close the Visual Basic Editor.

In your query, you can calculate the number of business days between date received and date completed as

WorkingDays2([date received],[date completed])

just like you would use NETWORKDAYS in Excel.

hansv mvp say:

Here is a version that should return -1 if something is wrong.

Public Function WorkingDays2(StartDate As Variant, EndDate As Variant) As Long
    '....................................................................
    ' Name: WorkingDays
    ' Inputs: StartDate As Date
    ' EndDate As Date
    ' Returns: Long
    ' Author: Arvin Meyer
    ' Date:     May 5,2002
    ' Comment: Accepts two dates and returns the number of weekdays between them
    ' Note that this function has been modified to account for holidays. It requires a table
    ' named Holidays with a field named Holiday Date.
    ' Modified by HansV to account for missing/non-date values
    '....................................................................

    Dim lngCount As Long
    Dim dtmCurr As Date
    Dim rst As DAO.Recordset
    Dim dbs As DAO.Database

    On Error GoTo ErrHandler
   
    If Not IsDate(StartDate) Or Not IsDate(EndDate) Then
        WorkingDays2 = -1
        Exit Function
    End If

    If EndDate < StartDate Then
        WorkingDays2 = -1
        Exit Function
    End If

    Set dbs = CurrentDb
    Set rst = dbs.OpenRecordset("Holidays", dbOpenSnapshot)

    ' Use the following line to include StartDate
    dtmCurr = StartDate
    ' Use the following line to exclude StartDate
    'dtmCurr = StartDate + 1

    lngCount = 0

    Do While dtmCurr <= EndDate
        If Weekday(dtmCurr, vbMonday) < 6 Then
            rst.FindFirst "[Holiday Date] = #" & Format(dtmCurr, "mm/dd/yyyy") & "#"
            If rst.NoMatch Then
                lngCount = lngCount + 1
            End If
        End If
        dtmCurr = dtmCurr + 1
    Loop

    WorkingDays2 = lngCount

ExitHandler:
    On Error Resume Next
    rst.Close
    Set rst = Nothing
    Set dbs = Nothing
    Exit Function

ErrHandler:
    WorkingDays2 = -1
    Resume ExitHandler
End Function
