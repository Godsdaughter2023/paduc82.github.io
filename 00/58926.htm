I want to limit folder sharing on a Windows XP Pro home network - Microsoft Community

ironyooper say:

I want to limit folder sharing on a Windows XP Pro home network (workgroup) to a specific user on another computer.

 I can't get that username entered successfully because I'm being limited to only users on the computer I'm working on.  I have Windows XP simple file sharing disabled.  Both computers are running XP Pro and I can share the folder, but it is open to EVERYONE.  Is there a way to do this?
wunders say: Let me attempt to explain how all this works.  You have XP Pro on Computer 1 and have disabled Simple File Sharing.  This is a step in the right direction.  The facts are that the file exists on Computer 1 and it is up to Computer 1 to control which external connection gets access to this file.  Computer 1 can only validate usernames/password combinations that it knows about.  Not being in a Domain environment, Computer 1 only understands usernames/passwords that exist on Computer 1.  It cannot know or trust any username/password combinations that do not exist within itself.  The normal method most people use is to create the same user on Computer 1 that exists on the computer that is connecting and give that user the same password that is used on the computer that is connecting.  If this is the case, the external user will connect seamlessly and will have access if Computer 1 has given that user permission to that file.  Access can be controlled by right-clicking on the file or folder, selecting "Properties", and then use the "Security" tab to generate an access matrix.

Authentication:  When connecting to Computer 1 from another computer,  Computer 1 first looks to see if the username on the connecting computer exists on Computer 1.  If the username of the user on the connecting computer does exist on Computer 1 and the passwords on both computers match, the connection is made.  If the passwords do not match, the connection is denied.  If the username of the connecting user does not match an existing user on Computer 1, then the "Guest" user is attempted.  If the Guest user is not passworded, then the connecting user will look like the user "Guest" and be granted appropriate "Guest" access.  If the Guest user is disabled on Computer1, then the connecting computer will display a Username/Login screen requesting that the connecting user provide credentials (username/password) that Computer 1 can use and recognize to verify that user.  If a username/password that computer 1 recognizes is entered, then Computer 1 authenticates that user with that credential.

Of course, once authenticated, that user will have access to those files to which permissions have been granted to that authenticated user as per KB307874.  XP cannot grant permissions to users it does not know about (obviously) beyond that of "Guest".

Bottom line is as you have discovered....  in a non-domain (workgroup) environment, you MUST authenticate as a legitimate user that exists (and is not disabled) on Computer 1.

Hey einnaj,

Transmit Buffers:

When a port is opened, receive and transmit buffers are created. To manage these buffers, the Communications control provides you with a number of properties that can be set at design time using the control’s Property Pages.

transmit buffers are used to transmit outgoing data. The Communications control allows you to manage these buffers by providing you with a number of properties that are used to place and retrieve data, return the size of each buffer, and handle both text and binary data.
