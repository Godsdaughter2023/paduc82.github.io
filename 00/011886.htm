    Hi Francisco Gochez,

     

    Thanks for your detailed description about this issue. As I understand based on your description, you’re unable to add an alias for a user in Office 365 portal. Given this situation, I first would like to confirm whether you’re the Global admin of this tenant. If so, I suggest you try to clear the cache of your browser or switch to another browser and try again.

     

    If the issue persists, I suggest you try to add the alias in the Exchange Admin center by following the steps below:

     

    In Exchange Admin Center > recipients > under mailboxes tab double click the mailbox you want to add an alias > email address > click Add button > choose SMTP email address type > and type the alias@verifieddomain in the Email address filed

    Note: Don’t check the Make this the reply address option. If you check this option, the new email address will be your primary email address.

     

    In addition, you can also try to add the alias using PowerShell command:

     

    1. Connect to Exchange Online PowerShell first.
    2. To check current exchange mailbox aliases: Get-Mailbox mailboxname | select -expand emailaddresses alias
    3. To add an email alias to an exchange mailbox: Set-Mailbox mailboxname -EmailAddresses @{Add='alias@ verifieddomain'}
