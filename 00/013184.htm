Cutting to the chase here is some general advice on sorting. Where you have sparse data such as you are describing you are always best off to first select the entire area you want to sort. If you do not do this step XL will make it's best guess as to what you want sorted which can be problematic.

ken sheridan say: There are a number of ways you can do this:

1.  The simplest approach is not to open a separate form based on the referencing table, but to embed a subform in the parent form, linking it on the keys.  The linking mechanism will not only restrict the subform to the rows referencing the parent form's current record. but will also automatically insert the correct value into the foreign key column in the subform's table. when a new row is inserted in the subform.  The subform can be placed on a separate page of a tab control in the main form if space is limited.

2.  Open a separate 'linked' form in dialogue mode, filtered to the current record's referencing rows, via a button on the main form.  This forces the user to close the linked form before returning to the original form.  The primary key is passed to the linked form via the OpenArgs mechanism and is then used in the linked form's Open event procedure to set the DefaultValue property of the foreign key column's bound control in the linked form.

3.  Open a separate 'linked' form in normal mode, again filtered to the current record's referencing rows, via a button on the main form.  Synchronize the linked form with the main form by means of code in the latter's Current event procedure which re-filters the linked form to the rows referencing the parent form's current record.  If the user then moves focus back to the parent form while the linked form is still open, the latter will be re-filtered to show the relevant rows if the user navigates to a different record in the main form.  Again the primary key is passed to the linked form via the OpenArgs mechanism and is then used in the linked form's Open event procedure to set the DefaultValue property of the foreign key column's bound control in the linked form.  Additionally the DefaultValue property of the foreign key column's bound control in the linked form is reset when the linked form is re-filtered.

You'll find examples of all three of the above approaches in FormsDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.
