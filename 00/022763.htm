tom ogilvy say:

Without knowing what you are doing, I will assume seeing the unique indicator is not necessary when you are refreshing pivot tables.   so

Instead of doing the copy and paste values, how about modifying your formulas like this

=if(Z1=1,IF(COUNTIFS($F$2:$F2,F2)>1,0,1),"")    for all formulas.  Then when you want to see uniques (with the time penalty, put a 1 in Z1. When not, leave it blank or put in anything that is not a 1.

The other approach would be premised on the fact that unique identification once performed is a done deal.  Then I would write a macro to do that.  Generally the macro can be written to be much faster than 1000 formulas in a column and it only runs when you run it.

Another approach might be to create your pivot tables in a second workbook that references the data in the first.  I believe the first workbook does not have to be open when you are refreshing the pivot tables and the closed workbook would not be calculating. 

If it turns out you do need to have the data "present", then you could hard code links to the first workbook.  Then source the pivot table from the local data or do a copy and paste special on the local data before refreshing the pivot tables.  ( don't know if a refresh would cause the links to be updated and even if it did whether this would be a big time penalty - thus if it is, you can use the copy paste special approach and recreate the formulas when you need them. )

Again, these linking approaches I haven't tested so I am just throwing them out as Ideas I would try if I were faced with this situation.

Another idea,  if the max number of duplicates for any single value is say 20, then SORT your data on the column with duplicate values.  Now starting in row 21 your formula would be changed to:  IF(COUNTIFS($F2:$F21,F21)>1,0,1) and then drag fill it down.  

Now none of your formulas will ever refer to more than 20 cells and this should remove a lot of the overhead.   

another idea

copy your column of duplicates to another column.  then use the new Remove duplicates feature to get a unique list.  Next to the unique list put in  (assume the column containing duplicates in your dat is column D:D and the unique list is in X:X starting in row 1) in Y1 put in  =Match(X1,D:D,0) and drag fill down.  This gives you the row with first occurance of of each value in your original data.

Now in your column where you had your formula you can do  =if(isnumber(match(row(),Y:Y,0)),1,0)  and drag fill down.  

for these simpler formulas, they should copy and paste special values pretty rapidly if required.

a. user say:

Hi,

How to get all occurrences of lookup value (duplicates)
As mentioned above, the Excel VLOOKUP function cannot get duplicate instances of the lookup value. To do this, you would need a far more complex array formula consisting of several Excel functions such as INDEX, SMALL and ROW.

For instance, the below formula finds all instances of the value in cell F2 in the lookup range B2:B16, and returns values from column C in the same rows:

{=IFERROR(INDEX($C$2:$C$16, SMALL(IF($F$2=B2:B16, ROW(C2:C16)-1,""), ROW()-3)),"")}

Copy the below formula to several adjacent cells, e.g. cells F4:F8 as demonstrated in the screenshot below. The number of cells where you copy the formula should be equal to or larger than the maximum number of possible duplicate entries. Also, remember to press Ctrl + Shift + Enter to enter an array formula correctly.
Let me know if that helps.

Engr. Jamshed Muhammad

myall_blues say: The general form of VLOOKUP is:
=VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])

lookup_value can only be a single reference or value, not a range like you have done.

It's hard to tell from the question exactly what is required without seeing some of the data, but the VLOOKUP part of the formula is likely to be something like:

daniel g. grau say:
It is possible that parts of your text have the "Do not check spelling or grammar" attribute enabled. To solve this, do a Select All (Command A), then go to Tools > Language,  select the proper language and make sure the check box at the bottom is blank.

If you are using styles, make sure their definitions have the proper language attributes set as well.

If yes, please be informed that in the current design of Outlook.com, you cannot drag a word or a sentence while composing an email. What I can suggest to you instead of highlighting and dragging is to use the Cut and Paste option. To do this, please follow the steps below:

Using your mouse:

  1. Highlight the word or sentence you would like to move, then right click and select Cut. Paste it on your desired location.

Using your keyboard and mouse:

  1. Highlight the word or sentence you would like to move, press Ctrl+X (for cut) then Ctrl+V (for paste) to your desired location.

I hope these steps help.
