ossiemac say:

Hello Phil,

 

Firstly the answer by Hans works well. However, I thought that you might like some further explanation of dates in AutoFilter.

 

Generally AutoFilter can handle the m/d/y date format.

 

With Regional date set to d/m/y format, if you record code to set the filter then the recorded code returns the correct d/m/y format like the following for dates between 2 Jan 2011 and 2 Mar 2011 and filter is set correctly during recording the code.

   

ActiveSheet.Range("A1:J1000").AutoFilter Field:=2, Criteria1:= _

       ">=2/01/2011", Operator:=xlAnd, Criteria2:="<=2/03/2011"

 

However, if you then run that recorded code, Excel sees them as m/d/y format and returns 1 Feb 2011 to 3 Feb 2011.

 

After running the above in Excel 2007, if you then click the filter and select Date Filters -> Between then the dates in the fields are 1/2/2011 and 3/2/2011 and if you click the calendar icon to the right of the field then you will see that it is 1 Feb and 3 Feb.

 

Now if you edit the code above to m/d/y format as per the following then you will get the correct filters.

 

   ActiveSheet.Range("$A$1:$J$486").AutoFilter Field:=2, Criteria1:= _

       ">=1/02/2011", Operator:=xlAnd, Criteria2:="<=3/02/2011"

 

Alternatively you can use the alpha literal name for the month and use dd mmm yyyy format and Excel can understand them.

 

ActiveSheet.Range("A1:J1000").AutoFilter Field:=2, Criteria1:= _

       ">=2 Jan 2011", Operator:=xlAnd, Criteria2:="<=2 Mar 2011"

 

An alternative to Hans’ answer for your code, you could use the following to include the literal month name:

 

ActiveSheet.Range("A1:J1000").AutoFilter Field:=2, _

     Criteria1:=">=" & Format(StartDate, "dd mmm yyyy"), _

     Operator:=xlAnd, _

     Criteria2:="<=" & Format(EndDate, "dd mmm yyyy")

Regards, OssieMac

This is from "Excel 2002 VBA Programmer's Reference"
Written by John Green, Stephen Bullen, Rob Bovey and Robert Rosenberg

http://www.oaltd.co.uk:80/ExcelProgRef/Ch22/ProgRefCh22.htm
Search for "Range.AutoFilter" and you'll see this note:

Range.AutoFilter

The AutoFilter method of a Range object is a very curious beast. We are forced
to pass it strings for its filter criteria and hence must be aware of its string
handling behaviour. The criteria string consists of an operator (=, >, <, >=
etc.) followed by a value.

If no operator is specified, the "=" operator is assumed. The key issue is that
when using the "=" operator, AutoFilter performs a textual match, while using
any other operator results in a match by value. This gives us problems when
trying to locate exact matches for dates and numbers.

If we use "=", Excel matches on the text that is displayed in the cell, i.e. the
formatted number. As the text displayed in a cell will change with different
regional settings and Windows language version, it is impossible for us to
create a criteria string that will locate an exact match in all locales.

There is a workaround for this problem. When using any of the other filter
criteria, Excel plays by the rules and interprets the criteria string according
to US formats. Hence, a search criterion of ">=02/01/2001" will find all dates
on or after 1st Feb, 2001, in all locales.

We can use this to match an exact date by using two AutoFilter criteria. The
following code will give an exact match on 1st Feb, 2001 and will work in any
locale:

   Range("A1:D200").AutoFilter 2, ">=02/01/2001", xlAnd, "<=02/01/2001"

PowerPoint gets its font defaults from several places.

For placeholder text (the "CLICK HERE" shapes you see on some layouts) the font formatting comes from the slide master or the specific layout itself.  To change these, choose View | Slide Master and change the font of the placeholders.  You might need to change a specific placeholder or the slide master (the one at the top of the collection of layouts).

For other text, add a text box, format it in the font/style/size you want as the default, then right-click the border of the text box and choose "Set as Default Text Box" (or whatever the equivalent is in Spanish, of course.)

bill mosca say: I wonder if this database is on a server and the Compact on Close feature is turned on. Access creates a copy of the database and names it db1.accdb (or db1.mdb if you are using An earlier version of Access). That file is deleted if the compacting is successful.

With the Compact on Close feature turned on, every time you close the database it will try to compact. At times, the network connection is such that the compacting is not successful so you end up with another database named db1.accdb or db1.mdb. The number increments with each failure so you end up with multiple copies.

To prevent this, turn off the Compact on Close and move the database to your local drive and compact it. This usually can be done once or twice a month. Then move it back to the server.

Better still, split the database with the back end containing only the tables on the server and the front end with the forms, reports, queries, macros, etc. on your local drive. Splitting is really simple if you use the database splitting wizard.

bernie deitrick say:

You need to loop through the areas of the visible cells and the cells to copy to process them, but the specifics of the macro depend on the layout of your sheets, which you do not describe:

Sub TestMacro()
    Dim rngC As Range
    Dim rngA As Range
    Dim rngS As Range
    Dim W1 As Worksheet
    Dim W2 As Worksheet
    
    Set W1 = Worksheets("Sheet1")
    Set W2 = Worksheets("Sheet2")
    
    Set rngC = W1.Range("A1").CurrentRegion
    Set rngC = rngC.Offset(1, 1).Resize(rngC.Rows.Count - 1, rngC.Columns.Count - 3)
    Set rngC = rngC.SpecialCells(xlCellTypeVisible)
    
    Set rngS = W2.Range("A1")
    
    For Each rngA In rngC.Areas
        Set rngS = rngS.Offset(rngS.Rows.Count).Resize(rngA.Rows.Count, rngC.Columns.Count - 3)
        rngS.Copy rngA
    Next
End Sub
