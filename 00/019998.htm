Hi Keith:

It's the same thing in Mac, but you need to take care of a couple of details:

  • Ensure the file is in .docx format.
  • Ensure the picture is not "inline with text".
There's a misperception out there that the file formats .doc and .docx are equivalent.  They never were, and are becoming increasingly divergeant. If the file is in the obsolete .doc format, the internal picture format is different: Word cannot rotate the picture unless its Wrapping style is anything BUT "inline with text".

If the file is in the current .docx format, the instructions will work perfectly no matter what the picture's wrapping format is.

There are possibly 2 things going on here.

One is that Office programs apply antialiasing to placed photos, so they always appear blurrier after being placed in a document or presentation than when you looked at them in PhotoShop. This is a display-only effect and printing the files should show their true resolution.

The other issue is image compression, where placed photos are re-rasterized to 220 d.p.i. or lower.
