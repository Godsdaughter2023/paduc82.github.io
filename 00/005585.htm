jjqltx say:

Filter Pivot Table based on array

I have a pivot table with hundreds of options.  I have an array with a subset of those options; how do I filter my pivot table in VBA with the items in the array.  Also some items in the array may not be in the pivot table since the data changes often.
tasos_k say:

Hi,

Assuming that you have 5 items in an array

v = Array("abc", "def", "ghi", "klm", "xyz")

you want visible only the above items

pivot table in active sheet, field name is 'DATA'

[edit..]

Sub PivotTable_Field_VisibleItems()
'May 31, 2016
On Error Resume Next
Const sField$ = "DATA" '<< field name, change as needed
Dim ws As Worksheet
Set ws = ActiveSheet

Application.ScreenUpdating = False

For Each pf In ws.PivotTables(1).PivotFields
pf.ClearAllFilters
Next pf
Dim v As Variant
v = Array("abc", "def", "ghi", "klm", "xyz")
ws.PivotTables(1).ManualUpdate = True
For Each pi In ws.PivotTables(1).PivotFields(sField).PivotItems
Select Case pi.Value
Case v(0), v(1), v(2), v(3), v(4)
Case Else
pi.Visible = False
End Select
Next pi
ws.PivotTables(1).ManualUpdate = False
Application.ScreenUpdating = True
End Sub

Sub PivotTable_filters()

glenprouty say:

Hi Kristina,

Typically, the message that you are seeing, is the result of a policy change

that has been made when Windows Defender senses another antivirus. Or

the fragments of one.

If you have recently uninstalled another antivirus, you should be sure to run

the removal tool for it. A list of removal tools is supplied in Le Boule"s first

link, at step two.

This policy change can also be the result of malware. You should scan your PC

with a good antivirus scanner, such as MalwareBytes or the ESET online scanner.

The location of both are also mentioned in Le Boule's link.

When you are satisfied that no other antivirus or fragments are still present, and

that you are free from malware, Enter the Registry an observe the following Key.

Enter the Registry by using "regedit" and "Run as Administrator"

Navigate through the following branch.

Hkey_Local_Machine\Software\Policies\Microsoft and select the folder

Windows Defender

The only Key that should be present is Default. Any other keys, are those that

have been set by Another Antivirus, Malware, or You (if W10 Pro).

You may find keys for "DisableAntispyware" or "DisableAntivirus". If you do you

should modify their DWord from 1 to 0.

To modify the key, "right-click" on the "DWord" and select "Modify".

Type 0 and the highlighted 1 should change. When finished click "OK".

You should back out of the Registry by retracing the steps you used to enter.

This will prevent your Registry from becoming cluttered. Once you reach "Hkey

_Local_Machine" click on "File" and then "Exit".

Restart your PC and then try accessing Windows Defender again.

Your "policy" message should be gone, however, you may have to re-enable

Windows Defender by setting "Real Time Protection" to "ON".

Good luck,  Glen

I was having this same problem and found the solution. I would either get a crash or a black rectangle in the upper left and no pdf would load.

(Another issue that is caused by this in IE 9 and 10 is the inability to drag separate tabs to create new windows).

 

Go to your Start menu and open the "Run" command or you can hit the Windows Key and R at the same time. Type in regedit and then click "Run". You can also find it by opening the start menu and just typing regedit into the search box. A little green block image should come up to open.

 

Then browse down through the expansions to get to

 HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Main

 

Don't expand Main and there should be a registry key on the right that is called "TabProcGrowth"

Double click the value and change it from a 0 to a 1. Click OK.

 

Close regedit and IE and then go back and click on the PDF link. It should now work like normal.

 

TabProcGrowth =1 instead of  0.

Closed IE and reopened it. Opened up a pdf...IT WORKED! Didn't reinstall anything.

 

Hopefully this information helps somebody out!

Several parts of the registry are probably corrupted and cannot be fixed by attempting to repair the Adobe Reader installation.

 

Try the following work-around, which works well enough for me until I can do a complete re-install of the operating system (Windows 8) and all applications:

 

1. Start Control Panel.
2. Click on Programs.
3. Click on “Set your default programs” under the "Default Programs" section.
4. Click on “Reader” in the list of Programs (this is the "Microsoft Corporation" entry near the bottom of the list), then click on “Choose defaults for this program”.
5. Check (turn on) the entry for “PDF File” in the list, then click Save. (This will uncheck the "PDF File" entry for "Adobe Reader XI (11.0.05)". You can only "check" entries for a program, not "uncheck" them!)
6. Close out of Control Panel by clicking the “X” in the upper-right corner of the window.

 

NOTE: You must also disable the "Adobe PDF Reader" add-on in Internet Explorer 10.

 

1. In IE 10, choose "Manage add-ons" from the Tools menu.

2. Show "All add-ons" in the drop-down combo box.

3. Select the "Adobe PDF Reader" add-on, then click Disable.

4. Click Close.

 

Now, when attempting to open a PDF file while in Internet Explorer 10, an “Open, Save, Cancel” message will display at the bottom of the browser window.

 

If Open is clicked, the PDF file will open in the Adobe Reader app for Windows 8. Otherwise, click the down arrow next to Save, choose “Save as” and save the file somewhere on your system.

 

At the Start screen, right-click on the Adobe Reader XI icon (not the one for the Windows version of the Reader), then “Run as administrator” to open, read, and print the document as usual.
