Cannot open Word document. Extension is .docx but appears to be .doc - Microsoft Community

paul schellens say:

Cannot open Word document. Extension is .docx but appears to be .doc in notepad. Changing extension doesn't help.

When opening previously saved .docx files, I and my work colleagues regularly get the error message: Word cannot open the file because the file format does not match the extension.

We change the extension to .doc and get:
Microsoft Word cannot open this file because it is an unsupported file type.

We cannot recover the files.
We are using Office 2013. The file gets saved with a .docx extension, but it doesn't seem to be a docx. It is not a zip file. Changing the extension to .doc, as mentioned, doesn't work either.
When we open the files in notepad, it looks very much like the old .doc format, but with one notable difference:
The last line of old .doc files opened in notepad have: Microsoft Word 97-2003 Document MSWordDoc Word.Document.8
These files have: Microsoft Word Document MSWordDoc Word.Document.12
It's almost like Word tried to save it as a new version, but didn't complete the zipping process to .docx.
It happens regularly.
Any thoughts on what steps we could take to investigate further?

paul schellens say:

Solved.

The document BeforeSave event was running when autorecovery save was occurring.  The code in there was examining the document for certain attributes - nothing too weird - but for some reason that had an effect on the auto-recovery save.

Following a further save by the user, the document would then only save (despite the extension remaining .docx) in Word's autorecovery format.

It's probably a Word bug, but I've worked around it by cutting down the amount of code in the BeforeSave event handler until it no longer has the effect.

The old documents that seemed 'corrupt' were able to be fully recovered by changing the extension to .asd and (importantly) placing the document in Word's autorecovery location and then using Open and Repair.
