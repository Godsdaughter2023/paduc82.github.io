lemp say:

ActiveX doesn't have anything to do with running gpedit.msc

ActiveX -- or more accurately ActiveX controls -- are plug-ins that add capability to a program.  ActiveX controls are primarily used in Internet Explorer, but they are also used in other Microsoft programs, such as some versions of Microsoft Office. ActiveX controls are often a vector for malware.  See https://www.howtogeek.com/162282/what-activex-controls-are-and-why-theyre-dangerous/

My guess is the something in the EQMOD package interfaces with Internet Explorer and that's why it involves ActiveX.  Furthermore, it seems that EQMOD starts up automatically.  You should look carefully at your star gazing program to see if you can change any setting that would prevent it from starting when Windows starts.

Other than that, the way to troubleshoot this sort of problem is to start Windows with the absolute minimum number of processes needed to run Windows (this is called a "clean boot") and then systematically add the other processes that normally start at bootup until the problem comes back.  This is a tedious process because it requires repeated restarts of your computer, but if you want to track down the problem, it's what you'll have to do.

STEP A

  1. Click Start and type msconfig in the search box.

  2. Press Enter.

  3. On the General tab, click Selective startup.

  4. Clear the Load startup items check box.

  5. Click the Services tab.

  6. Select the Hide all Microsoft services check box (at the bottom).

  7. Click Disable all.

  8. Click OK.

  9. Click Restart.

Confirm that the problem does not occur.  In other words, do whatever it is you do that causes the problem to appear and confirm that it does not appear.  If the problem DOES appear at this point, you have something more serious going on.  Skip to STEP D and reply here explaining what you did and what happened.

STEP B

  1. Click Start and type msconfig in the search box.

  2. Press Enter.

  3. Click the Startup tab

  4. Clear half of the items.  I suggest clicking "Startup Item" in the header row, which will sort the items alphabetically.  This should make it easier to remember which items you've cleared.

  5. Click OK
  6. Click Restart
  • STEP C
  • Check to see whether the problem occurs.
    • If the problem DOES occur, then you know that it is caused by one of the programs that remained checked.  In this case, repeat STEP B and this time clear half of the items that had remained checked.
    • If the problem DOES NOT occur, then you know that it is caused by one of the programs that you cleared in the last iteration.  In this case, repeat STEP B and this time re-check half of the items that you cleared in the last iteration.

    Repeat STEPS B and C until you have identified the item that causes the problem

  • STEP D

    To return to normal

    1. Click Start and type msconfig in the search box.

    2. Press Enter.

    3. On the General tab, select Normal startup.

    4. Click OK.

    5. Click Yes when asked to restart your computer.

    lehenryjr say: Taking a Snapshot of a video…
    Windows Live Movie Maker has the option of grabbing a screenshot during the previewing of your video.
    Play the video in Windows Live Movie Maker, and during the preview [on the HOME tab] you'll see the option for Snapshot.
    You click on Snapshot and then the application will capture the current video position and then ask you to save the Snapshot as a PNG file.

    There are no other options to export, other than PNG.

    The PNG file can be saved anywhere on your system [like your desktop].

    Link:
    http://su.pr/28z00C
