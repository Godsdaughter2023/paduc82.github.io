Cell Value Greater than or equal to =$A$19

Sorry guys, but it looks like the official documentation took a wrong turn once again. Automatic Maintenance is already triggering the Windows Defender Scheduled Scan task – and that’s probably still set to happen on a daily basis. You schedule the Windows Defender Scheduled Scan task in the Automatic Maintenance settings, just exactly as CJ. Eichholz (MSFT) suggested in the thread referenced above. But it’s very important to realize that this scan will run reliably even if you don’t change the default schedule (unlike the MSE scheduled scan). You don’t have to do anything to get the Windows Defender scheduled scan going. It’s fully automatic.

Now, if you add a second trigger to the existing scheduled scan task, as the documentation suggests, you’ll just be adding a second Quick Scan on a different schedule, so you’ll just be duplicating the Automatic Maintenance scan. But if you decide to go ahead and change this task’s “action” in order to get it to do a Full Scan, as some of the Web gurus have suggested, you’re ipso facto converting the Automatic Maintenance scan into a Full Scan, and good luck with getting Automatic Maintenance to get that job done on a daily basis. I’ve covered this many times over for Windows 8, and I think it’s pretty much the same in Windows 10.

zorvek say:

If worksheet cells do not seem to be calculating when changing worksheet cell values or copying cells containing formulas then workbook or worksheet calculation is turned off. To turn on automatic calculation in Excel 2007 and later, click on the Office Button or File tab, select Options, select Formulas, and set Workbook Calculation to Automatic. In Excel 2003 and earlier the menu command Tools->Options, navigate to the Calculation tab, select Automatic, and click OK.

Note that in some cases manual calculation may make more sense such as when there are many calculations, complex calculations, or many UDF references which cause every calculation to take more than a second or two.

Note that a workbook can be manually calculated at any time by pressing F9.

As far as the formula being displayed, perhaps the worksheet is displaying formulas instead of values. To toggle displaying formulas, press CTRL + ` (grave accent). In this more, formatted values such as dates and numbers are displayed in their raw form, not in their formatted form.

Kevin

ossiemac say:

I am guessing that some or all of the cells are in Text format, including the cell in which you are typing the formula. The cells need to be reformatted to General or numeric and then convert the text numbers to real numeric.

Select the cells and then select Number format and format to either General or Number. This prepares the cells to accept numeric rather than text numbers but does not convert the existing numeric numbers. For the conversion we need perform a mathematically operation on them so we multiply them all by 1 as per the following. (Multiplying a number by 1 does not change its value). 

Type the number 1 in any blank cell.

Select and Copy the cell with 1 in it.

Select all of your other cells with numbers in them and then PasteSpecial -> Multiply.
