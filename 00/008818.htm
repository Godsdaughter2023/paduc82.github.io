Capitals in Index - Microsoft Community

stuart raymond say:

Capitals in Index

I like all my index terms to begin with a capital letter. Is it possible to capitalise initial letters in an index without having to capitalise each entry individually?
jay freedman say:
It's "sort of" possible.

Toggle the Index field so you can see its code. Insert a \* Caps switch at the end of the code, so it looks something like this:

{ INDEX \c "2" \z "1033" \* Caps }

That will capitalize the first letter of every word in the index. If all your entries are one word each, that will work as you want. If any of them are multiple words, though, you'll get entries that look like

Turks And Caicos Islands, 115


Although "and" should not be capitalized, you won't be able to change it unless you unlink the Index field (so it's no longer a field but plain text, which can't be updated).

Shortcut using Event Viewer.

 

1.  Open Event Viewer and go to Windows Logs / System.

 

2.  On the right, click Filter Current Log.

 

3. Click the arrow to the right of Event Sources and scroll to Microsoft Antimalware and click it to select it.

 

4. The log should now reduce to a manageable size with only those events showing.

 

5. Go to the date that the Approval was authorized/listed and see if anything is shown.  If you needed to approve it, the detection at least should appear even if the conversion to Approved doesn't show for some reason.  While I have no approved items (and no real way to add them) myself to check, I was able to see all of the detections with all of the detailed information.  For example, for one that got quarantined, I saw the following detail:
