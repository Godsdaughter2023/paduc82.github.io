techfreak_ say:

I wrote this response in regards to zoom on the C920, in the old Community.  The same should apply for the C930 as well.  Take the following into consideration, based off my observations:

I have tested the zoom on my C920 and the zoom appears to work properly using the Logitech drivers or the generic drivers.  Zoom appears to scale more on lower resolutions and less at higher resolutions.  When the call was negotiated at 1080p, I could only zoom in once, regardless of how far I moved the slider.  When the video call was negotiated at a lower resolution, similar to the 4:3 Skype video preview, the zoom scaled several times without issue.  I have tested this behavior with other software and it functions exactly the same whether the camera is in use or not in use.  

Based off these observations, zoom more than likely uses real hardware available resolutions to virtually scale to, similar to  some monitors and projectors.  There are a total of 19 resolutions the C920 supports and only 2 of them are above 1080p vs 18 above at the lowest available resolution.  The 2 resolutions above 1080p have the same X resolution but differing Y resolutions.  This appears inline with the ability to only zoom in once, since changes in both X and Y would probably have to exist, to fully satisfy a zoomable condition.

If you are no longer able to zoom as deep as before, then it is probably due to your video calls being negotiated at higher resolutions.  Other possibilities may be issues with your camera drivers/Logitech software (as pointed out by Allen).

If your call is at 1080p and you have a 4K camera (and the resolution remains at 1080p) then you should have more resolution overhead to allow more in-depth zooming at this higher resolution.  If I owned a 4K camera I would surely test this theory out.

In regards to specifically the Skype App for Windows 10:

Windows Apps in general do not have access to the standard desktop webcam settings you are used to in the desktop version.  In order to control zoom the software provided to you by Logitech (similar to their defunct controller software), would have to give you those options or possibly a 3rd party Windows App.  I have no idea if the Skype App will ever create generic built-in controls to do this.

How to Resolve Problems Installing the Visual C++ Redistributable

The Visual C++ Redistributable is a DLL (Dynamic Link Library) file required by programs or games built using Microsoft’s Visual Studio software development environment. When a program requires a DLL or some other supporting file to run, this is called a dependency. The MSVCR.DLL stands for:

MS – Microsoft
V – Visual
C – C++
R – Redistributable.

Browsing your Apps and Features Setting or Programs and Features Control Panel, you might see multiple versions of the redistributable installed. A particular app installed might be utilizing one of them, so, removing any version can cause an application to malfunction. Some apps might even require updated versions of the Visual C++ Redistributable such as 2010 SP1 or 2012 Update 4.

A DLL helps app creators efficiently reuse common components that make a program work. Instead of reinventing the wheel and bloating their code each time, developers can reuse the libraries in the MSVCR.dll. This in return can reduce the size of a program and provide better performance and memory utilization. Some applications are written using specific versions of the Visual C++ Redistributable. If your computer does not have the full Visual C++ development program installed (which you likely won’t unless you are a software developer), a program will come with what is called a Runtime Component to facilitate that particular program. The Runtime Components are just the essential pieces of code that make the software work.

The error indicates that a particular version of the Visual C++ Redistributable required by the program you are attempting to install is either corrupt or missing. In this case, MSVCR110 represents the Visual C++ 2010 SP1 Redistributable. Other versions include:

MSVCR71.dll, MSVCR08.dll, MSVCR09.dll (2008 – 2010)
MSVCR120.dll (2012)
MSVCR130.dll (2013)
MSVCR140.dll (2015)

you can download the version required by the application. The program will indicate the version of the file required if an error pops up during or when executing the app. Users should also be aware that some programs might require either the 32 or 64-bit architecture.
