bobgreenblatt say:

You don't need VBA, you just need to separate the street numbers from the street names. Make a separate column for each like this:

Assuming the street number and name is in column C, starting in row 2, then in one spare column to the right, enter this formula:

=left(c2,search(" ",c2)) 

This will give you the street numbers.Fill the formula down for the number of rows you have.

Then to get the street names, in the next column to the right, use this formula:

=mid(c2,search(" ",c2)+1,255)

This will give the street names. Likewise fill this down also. Then select the entire range of addresses, including these 2 columns. Sort the whole range first on the street names, then on the numbers.

Leaving OL'2010 C2R issue aside - just a comment about "........as this would leave me totally unprotected" as it applies to any Outlook A/V addin - not picking on McAfee in particular..................

If you have a system level A/V in place which should be the case from what I've read about Total Protection - then you are not "more or less" protected with an Outlook addin. Before you can open any attachment, the file is saved and at that point the system level A/V kicks in so all that is happening is that the same file is being scanned twice. Similarly, really no point in scanning "outbound" messages since you can't receive/send a virus via text data and if you attach a file - it will be scanned again by the system level A/V. If you have something pro-actively scanning the websites visited - how (or more specifically - from where) the URL is invoked is not going to make a difference.

So just going from experience - the only thing that Outlook addin A/V's have caused is a whole lot of aggravation on a variety of levels. Some seriously slow down Outlook's operation (or cause problems within Outlook) and/or interfere with other 3rd party programs that integrate with Outlook or access Outlook data (ContactGenie programs included) causing program crashes or resulting in intolerable performance hits. When the A/V addin is disabled or removed - the universe returns to being completely normal.

IMHO - Outlook A/V addins are a hangover from days gone by and have far outlived their usefulness in today's operating environment at least as is applicable with all the major mainstream system-level A/V vendors.

kathleen orland say:

If there is any old data, it should be located here:

 

C:\Users\%username%\Documents\Outlook Files

 

However, if Outlook was upgraded to 2010 from 2007 the PST would be in the default location for Outlook 2007 and would not have been moved:

 

:\Users\%username%\AppData\Local\Microsoft\Outlook\

 

In order to find the PST, you must use Windows Explorer to unhide hidden files and folders, and also to show known file extensions. This is usually not turned on, by default. When you open Windows Explorer, click on Tools > Folder Options > View and make your changes there. Then you can search for the PST file.

 

It might be easier to do a search for *.pst rather than to hunt for it once you unhide files and snow known file extensions.

 

If you do find it, you can easily add it back to your Exchange mailbox. Instructions are for Outlook 2010.  

 

  1. Open Outlook.
  2. File tab -> Info -> click Account Settings (Outlook 2007 Account Settings option is available in Tool menu, in Outlook 2003 and 2002 Tool -> Options).
  3. E-mail tab-> click your email account. (in Outlook 2003 and 2002 Mail Setup -> Email Accounts -> View or change existing e-mail accounts - > Next)
  4. In the "Account Settings" dialogue box Click "Change Folder".
  5. Click New Outlook Data File-> click Office Outlook Personal Folders File-> click OK.
  6. Browse and locate the .pst file of old profile-> click OK.
