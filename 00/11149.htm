And Act II, scene i, page 1 would display: I-ii-1

I have been trying to use a multi-level list, but I am having a bit of trouble getting that to work.  Ultimately, the problem is getting the document to run 3 counters so that when I start the second scene of an act, the page number resets to 1, but the scene counter goes up to ii; same for the act differences.

Is there any way possible to convince word to run the page numbers in this manner?  It's basically like paginating a multi-sub-chapter document, with Chapter 1, scetion 1, page 1 being displayed as 1:1:1... -Keith
graham mayor say:
You can have as many counters as you wish using SEQ fields to provide the numbering and if you format the SEQ field in the document with a unique character style that applies only to the specific sequence number you can reproduce the sequence in the header using a Styleref field. You will need a character style for each sequence e.g. ActNum and SceneNum and thus. the character styles are the same as the underlying style, they merely add a name to cross reference in the Styleref field.

Act { Seq Act \*Roman }
Scene { SEQ Scene \*Roman \*Lower}

will give you your formatted numbers and

{ Styleref ActNum }-{ Styleref SceneNum }-{ Page }
 will reproduce the values in the header/footer

The only minor issue with SEQ fields is that they do not automatically update if you change the order, so you would need to force an update after making changes. You could use the example macro at http://www.gmayor.com/installing_macro.htm to do that.

dianeoforegon say: DNR are a useless feature which gives people false assurances, always a bad thing:

1) It doesn't work when the receiving email client does not have an automatic service that sends a reply. (Not only Entourage/Outlook - there are many, many other other email clients which don't reply.) Therefore, if you don't get a reply you may incorrectly think it wasn't received,

2) It doesn't tell you that the recipient read the message anyway, only that his email client received it. Therefore when you get the reply you may incorrectly believe that he read it when he hasn't.

So - bad both ways round. It's a really feeble protocol, and good that Entourage/Outlook does not (pretend to) implement it. The fact that Outlook for Windows implements it is an indication that the Outlook developers, unlike the Entourage developers, seem to think that everybody else in the world uses Outlook, which they don't. The MacBU developers, being on a "minority" platform, know better, and are smarter in not implementing it. To be a bit fairer, the Outlook approach probably reflects the fact that, originally, back in Office 95 for PCs, Outlook was an extension of an earlier client that worked only as intranet - something you used only with other people on your own Exchange server. Outlook started life doing the same thing - there used to be two modes - one for "Workgroups" on an Exchange server, and another for "Internet".

There were some poor decisions made when Outlook was extended to the Internet, that shouldn't be there. This is one of them - it's inaccurate and unreliable, since it assumes that everyone can do it, which they can't. You're much better off just not using it, since it cannot be relied upon.

Now that you know this, guess what the chances are of it being added to Outlook when we have so many features that need work or to be included?
