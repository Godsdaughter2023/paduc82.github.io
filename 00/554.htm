mike h.. say:

Hi,

 

The reason <12 doesn't get counted is that COUNTIF is treating the < as an operator and looking for cells less than 12 and finding none.

 

The ressaon =COUNTIF('2012'!R1562:R1612,"Over 96 hours")/B2  does work is it's counting the instances of that text.

 

I'd use SUMPRODUCT for both like this

 

=SUMPRODUCT(--('2012'!R1562:R1612="<12"))/B2

 

=SUMPRODUCT(--('2012'!R1562:R1612="Over 96 hours"))/B2

stefan blom say:

The behavior you describe indicates that the indentation hasn't been properly defined in the first place. Once the indentations are correct, restarts will work.

Here's how to modify left indents in a numbered (or bulleted) paragraph: Click the actual number (you'll see that all numbers of the list will be highlighted). Right-click and choose Adjust List Indents. Adjust the indentation as desired.

"Number position" determines the position of the number, "Text indent" determines where the second and subsequent text lines start, and "Add tab stop at" defines the location for the text after the number (it's only available if you set "Follow number with" to "Tab character"). All measurements are from the left margin (of the page, text box, table cell, whatever applies).

If what you have is a multilevel list, the Adjust List Indents command will display the Multilevel List dialog box instead. See http://shaunakelly.com/word/numbering/numbering20072010.html.
