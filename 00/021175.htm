Sub Find_Duplicate()
 
    Dim myDataRng As Range
    Dim cell As Range
    Dim strV As String
   
    ' Use column A
    Set myDataRng = Range(Range("A5"), Cells(Rows.Count, "A").End(xlUp))
   
    For Each cell In myDataRng
        If Application.CountIf(Range(Range("A5"), cell), cell.Value) > 1 Then
            strV = cell.Value
             Interaction.Beep
            If MsgBox("""" & strV & """ has already appeared above. Delete duplicates?", vbYesNo) = vbYes Then
                cell.ClearContents
                While Application.CountIf(myDataRng, strV) > 1
                    myDataRng.Find(strV, cell, xlValues, xlWhole, , xlNext).ClearContents
                Wend
            End If
        End If
    Next cell
    Interaction.Beep
      If Not boolDupes Then MsgBox "No duplicates were found!"
  End Sub

In case it is useful, I have found this workaround for similar problems. In order to force Access to present the dialog, the following steps work before you try to open your query to the SharePoint resource.

1) In Access, click External Data tab

3) Click "SharePoint List"

4) Select the SharePoint site to which you will query

5) Click "Next"

6) Enter your credentials in the dialog

6) Click "Cancel" or "OK". It does not matter at this point which you click.
