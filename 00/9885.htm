 karl timmermans (outlook mvp 2012-2018)  say:

    
    

            
            
            
            >>

Do I have to sync my OUTLOOK calendar and such through GOOGLE in order to sync my Microsoft product? 

<<



For the sake of accuracy, you don't sync your Microsoft product - in this case Outlook which is an <email client>. Multiple devices sync with an <Email account> to the extent that the protocol for that account allows it. Email protocols are industry defined standards and not defined by Microsoft.



There are two primary industry protocols (POP3 and IMAP) and a third - Microsoft Exchange - which has also become an accepted standard.



POP3 - does not allow the sync'ing of any data across devices. It only allows for the one-way download of <Inbox> data from the server to the desktop/device. 



IMAP - provides for a 2 way sync of email data between server and device. In short, what happens on one gets sync'd with the other. Contact/Calendar data is not sync'd with the server and is local to the device where the contact/calendar items are created



Exchange - allows for the full two-way synchronization of all data (email, contacts, calendar etc) between server and device. An Outlook.com account is by default configured as an Exchange account on desktops and either EAS or EWS on mobile  devices depending on what has been implemented.



Every device accesses <email accounts> via an <email client>. <IF> the email client on a given device sync's Google contact/calendar items, it means that there is an extension incorporated into the <email client> to allow that. FWIW, you require a 3rd party addin (such as Google App Sync) to sync Google contact/calendar data with Outlook desktop - part of the Office 365 suite of programs.



So to get back on point, if you are asking about contact/calendar data created locally via the Outlook desktop program - you will have to save that contact/calendar data in an email account such as either a free Outlook.com account or a hosted Exchange account which can then be configured on all devices.

 standards and not defined by Microsoft. 



There are two primary industry protocols (POP3 and IMAP) and a third - Microsoft Exchange - which has also become an accepted standard.




POP3 - does not allow the sync'ing of any data across devices. It only allows for the one-way download of <Inbox> data from the server to the desktop/device.




IMAP - provides for a 2 way sync of email data between server and device. In short, what happens on one gets sync'd with the other. Contact/Calendar data is not sync'd with the server and is local to the device where the contact/calendar items are created




Exchange - allows for the full two-way synchronization of all data (email, contacts, calendar etc) between server and device. An Outlook.com account is by default configured as an Exchange account on desktops and either EAS or EWS on mobile devices depending
 on what has been implemented. 



Every device accesses <email accounts> via an <email client>. <IF> the email client on a given device sync's Google contact/calendar items, it means that there is an extension incorporated into the <email client> to allow that. FWIW, you require a 3rd party
 addin (such as Google App Sync) to sync Google contact/calendar data with Outlook desktop - part of the Office 365 suite of programs.




So to get back on point, if you are asking about contact/calendar data created locally via the Outlook desktop program - you will have to save that contact/calendar data in an email account such as either a free Outlook.com account or a hosted Exchange account
 which can then be configured on all devices.

             techfreak_  say:

    


            
            
            
            Skype for Windows Desktop allows you to control the history duration in the Skype options.  If you are using a version of Skype like the Skype for Windows 10 App, then history is set to the equivalent of forever.  A forever setting on either client means
 that the client will not delete your local history on exit.  Any history greater than 30 days would remain stored in your local history.  You cannot increase the history capacity in the cloud.  Even if Microsoft decides to increase the history located in the
 cloud, as soon as today, the history would not go back any further than it is today.  If you use a client like Skype for Web, the only history you have access to, is what is in the cloud.   Be aware that even if you use a client to has history set forever,
 this does not stop the history from being deleted through corruption,  or full uninstalls and reinstalls of the client.  For this reason it is wise to frequently make backups of your history fi
