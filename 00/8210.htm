rich michaels - mvp (greatcirclelearning.com) say:

Follow the advice that Tim Liu has given you about updating to the most current version of Office for the Mac. You are entitled to those updates and it's not going to cost you more money to apply them.

Regarding your other questions and I believe a few misunderstandings ...

The option to "Always create a backup ..." only runs when you initiate a File Save function. If you click Cmd + S, or choose File Save from the menu, etc... it does not automatically run on an AutoRecovery save.

Relying on the AutoRecovery function for saving is unwise in my opinion. I find the reliability of the function to be very poor. It doesn't seem to operate consistently on all system crashes. Personally, I never count on it working and instead, I try to make it a practice to press Cmd + S often.

AutoSaving is a new feature, which you can take advantage of but not from that old version you are currently running ... so upgrade. Then, you will have the feature but only if you also work from OneDrive. OneDrive files that you are editing are automatically backed up, and automatically saved.

Enabling Time Machine is also another recommended safety feature on a Mac. With Time Machine running, a backup of version of your file might be 1-hour old, but at least you have one.

If you do look into 3rd party helpers on this, do a web search on Got-Ur-Back from Great Circle Learning. It's a free add-in for file overwrite protection. It initiates when you click Save and it keeps up to 5 previous versions of the file you are editing. But again like Word's built-in "Always Create a Backup" feature, Got-Ur-Back only runs when you click Save.

Borders automatically close between pages - Microsoft Community

ann s e say:

Borders automatically close between pages

Windows 7, Office 2010, Word document:   I often put one border ('box') around 2 or 3 consecutive paragraphs to show that they are related.  If automatic pagination happens to fall in the midst of a paragraph, no problem:  The box stays open, i.e., no border at the bottom of one page or the top of the next, correctly indicating the the contents of the box continue from one page to the next.  The PROBLEM occurs when automatic pagination breaks *between* consecutive paragraphs in a single box, in which case Word closes the boxes (at the bottom of one page and the top of the next), making it seem like two discrete boxes when it's meant to 'read' as a single continuing box.  Any way to adjust the settings to do this?

suzanne s. barnhill say:

When you apply a box border, you're applying Top, Bottom, Left, and Right borders. Unless you also add an Inside Horizontal Border, you won't get borders between paragraphs except when the bottom of a paragraph falls at the bottom of a page. The only way around this is, on an ad hoc basis, to remove the bottom border from that bottom paragraph. To do this, click in or select the paragraph and click on the Bottom Border button in the Paragraph group on the Home tab (if you've previously used another border, you'll need to click the arrow beside the button to drop down the menu and then select Bottom Border). You may need to click twice to turn it off.

 

The likelihood is that you're also getting a top border on the next page; you'll need to use the same technique to remove it.

ann s e say: I appreciate the suggestion.  The problem is that I'm dealing with a dynamic 400+ page document that undergoes frequent edits (insertions and deletions).  As a result, the auto-pagination (page break) locations change constantly.  With such a long and dynamic document, ad hoc 'fixing' of each affected box one-by-one just isn't feasible.   I'm looking for a setting that will override the troublesome default box closure that occurs whenever the bottom of a paragraph falls at the bottom of the page.       

suzanne s. barnhill say:
I don't think there is a solution. This is just the way Word works (and always has). You could define a series of otherwise identical styles that have, sequentially, top and side borders, only side borders, and side and bottom borders, and apply these to the first, following, and last paragraphs of your boxed text.

ann s e say: Thank you for your creative suggestion.  I am still seeking (perhaps in vain) a WORD 'settings' solution so I don't have to reconfigure each box for 400 pages.  This problematic box closure which, as you know, occurs by default when page breaks fall between paragraphs has been frustrating.  But thanks.

suzanne s. barnhill say:
It's always possible that I'm wrong, but I'm not aware of any setting that would change this. Some users would prefer having the box closed just for neatness of appearance, even though it is not "logical."

daniel g. grau say:
Try manually transferring them:
  • Word 2004 ACLs reside in YOUR_USER_NAME/Library/Preferences/Microsoft
  • Word 2011's are here: YOUR_USER_NAME/Library/Application Support/Microsoft/Office/Preferences/Office 2011
If that doesn't work, take a look at Rich Michael's macros in this thread.

daniel g. grau say:
Yes, those files need to be in their specific folders and have exactly those names, as that's where and what Word will be looking for.

Rather than renaming, create a folder Original ACLs to store any existing ACL files. I did that myself when I moved from 2004 to 2011, but since then it has become a memento, as that folder has remained untouched for years.
