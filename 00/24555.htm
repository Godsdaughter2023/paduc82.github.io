jllatham say:

Yes, it is possible.  I will provide code to do so.

First, the setup on the sheet with the months that are to be moved:  The code assumes that you have actual dates in A1 through L1 and have probably formatted them to just display the month name.  It is important that actual dates be in those cells, like 1/1/2015 for January 2015 although you may just display Jan or January (custom format of mmm will display Jan, Feb, etc).

This code is worksheet event code, it will go into the code module for the sheet where you will be entering the new data (sheet1).  To put it there, choose that sheet and right-click on its name tab and choose [View Code] from the list and then copy the code below and paste it into the code module presented to you, make any edits to it that you need to (the places you might need to edit are in bold type).   You must now save the file as a macro enabled workbook, type .xlsm or .xlsb to preserve the code.

Try entering various dates into the cell on Sheet 1 to test it.

Private Sub Worksheet_Change(ByVal Target As Range)
  'change this to be the name of the sheet with the months in columns
  Const monthWSName = "Sheet2"
  'change this to be the address of the cell where you enter the
  'date into on THIS sheet
  Const newDateCell = "A1"

'working variables
  Dim monthWS As Worksheet ' will reference full months worksheet
  Dim monthToMove As Range
  Dim colPtr As Integer
 
  'did a change of the cell with the date in it take place?
  If Target.Address(False, False) <> newDateCell Then
    Exit Sub ' NO, so do not do anything
  End If
  'YES: set reference to the months sheet
  Set monthWS = Worksheets(monthWSName)
  'yes, get the month from the date as "Jan", "Feb" etc.
  'this assumes that row 1 on the other sheet has
  'date entries as 1/1/15, 2/1/15, 3/1/15 in row 1 and
  'you have custom formatted them so that
  'they show the month names like Jan or January
  '
  'insert a new column as column A
  'shifting all old columns to the right 1 column
  monthWS.Range("A1").EntireColumn.Insert

  'find where the 1st regular month is now starting at column B
  For colPtr = 2 To 13 ' testing columns B through M
    If Month(monthWS.Cells(1, colPtr)) = Month(Target) Then
      'this makes it faster & prevents screen flicker
      Application.ScreenUpdating = False
      With monthWS
        Set monthToMove = .Range(.Cells(1, colPtr), _
         .Cells(Rows.Count, colPtr).End(xlUp))
        monthToMove.Copy
        .Range("A1").PasteSpecial xlPasteAll
        'delete the old column that has been copied
        Application.DisplayAlerts = False
        monthToMove.EntireColumn.Delete
        Application.DisplayAlerts = True
        'release assigned resource back to the system
        Set monthToMove = Nothing
        'all done
        Exit Sub
      End With
    End If
  Next ' end colPtr loop
End Sub

hansv mvp say:

Let's say the column is column B.

Select this column. B1 should be the active cell within the selection.

On the Home tab of the ribbon, click Conditional Formatting > New Rule...

Select 'Use a formula to determine which cells to format'.

Enter the formula   =AND(B1<>"",MID(B1,4,1)<>"-")

Click Format...

Activate the Fill tab.

Select a highlight color.

Click OK twice.

Anomalous cells will be highlighted.
