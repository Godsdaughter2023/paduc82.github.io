    chip pearson say:

    Dates are just numbers, the number of days since 0-Jan-1900 (1 = 1-Jan-1900, 2 = 2-Jan-1900, ...41067 = 7-June-2012), and therefore you can just subtract the start date from the end date to get the number of days between the two dates.

    =EndDate-StartDate

     

    Excel may try to help you by formatting the result cell as a date. Be sure to format it as General or a numeric format.

     

    If you need to ignore weekends and, optionally, holidays, use the NETWORKDAYS function.

     

    =NETWORKDAYS(StartDate, EndDate, Holidays)

     

    where Holidays, which is optional, refers to a list of holiday dates you want to exclude from the count.

     

    The DAYS360 function works by assuming that every month has 30 days. It is used in bond calculations but does not return the actual number of days between two dates.

    In Excel 2003 and earlier, you need to load the Analysis ToolPak add-in to get the NETWORKDAYS function. In Excel 2007 and later, NETWORKDAYS is built into Excel and the add-in is not needed.

    NETWORKDAYS has some significant limitatins. See http://www.cpearson.com/Excel/betternetworkdays.aspx for a better approach to NETWORKDAYS.

    jllatham say:

    First add this code to your UserForm's code module:

    Private Sub CboOffice_Change()
      Dim varResult As Variant
      Dim findWhat As String
     
      findWhat = Me.CboOffice.Text
      On Error Resume Next ' if no match, generates an error.
      varResult = _
       Application.WorksheetFunction.VLookup(findWhat, Range("ActCode"), 2, False)
      If Err <> 0 Then
        'same as #N/A error
        'just make the text box blank
        varResult = ""
        'and clear the error
        Err.Clear
      End If
      On Error GoTo 0 'reset error trapping
      Me.txtOffice = varResult
    End Sub

     

    Second, add a call to that routine to your _Activate() process so that things get synch'd up when the form is opened/reactivated by just putting the name of the process inside of it.  Using earlier sample, it might look like this:

    Private Sub UserForm_Activate()

    Me!Txt_FromCountCell = Worksheets("Sheet3").Range("X4").Value
    'and to show the current date in a textbox on the form
    Me!TextBox3 = Format(Now(),"dd-mmm-yyyy") ' change "dd-mmm-yyyy" as desired
     CboOffice_Change ' will call that routine to sync things up if needed

    End Sub

     

    jllatham say:

    The next step?

    Figure you might be wondering about moving data from your userform back to worksheets - it's just the reverse of one line we used earlier:
    Me!Txt_FromCountCell = Worksheets("Sheet3").Range("X4").Value

    would become something like

    Worksheets("Sheet3").Range("X4") = Me!Txt_FromCountCell

    the .Value property is the default, so we really don't have to specify it here.

     

    You may have noticed that one time I used

    Me.controlName

    and I've also used

    Me!controName

    6 of 1, half-dozen of the other type of notation.  If memory serves me right, I think the Me!controlName form is the more proper of the two.  Whichever you choose, be consistent within your code to keep confusion to a minimum.
