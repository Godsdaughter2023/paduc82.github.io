brian tillman [outlook mvp 2007-2019] say:

WLM uses the subject to determine if a message is part of a conversation.  In your example, if two people send you messages with subject "AAA", they are placed in the same conversation by WLM, even though they are not in the same conversation.  Outlook uses more than just the subject to determine if a message is part of a conversation.  It correctly does not include the second "AAA" message in the original conversation, since it's not related to that conversation at all.

Outlook messages have a conversation attribute that helps determine if it is part of a conversation.  Outlook uses not only the subject, but reply and forwarding information as well to determine if a message is part of a particular conversation.  You can't change Outlook's behavior for classifying messages in conversations, other than to turn them off or on, using the UI.  You probably could write some VBA code to modify the conversation attribute to make it behave more like you want, though.

jeeped say:
I use LEN() as a measure to determine whether there is any content in the cell. A boolean FALSE is equal to 0 (i.e. zero) and a boolean TRUE is anything that is not zero. Excel comes with an ISBLANK() function but that does not take into account a cell that might have "" (i.e. empty string) returned by a formula. IF(LEN(A1),<true>,<false>) is the same as IF(A1<>"",<true>,<false>) as both correctly determine truly blank calls and those containing an empty string. I just prefer the LEN() method.

hansv mvp say:

The name of the current sheet is returned by

MID(CELL("filename",A1),FIND("]",CELL("filename",A1))+1,255)

But if you want to refer to the active sheet, you don't need INDIRECT, you can simply use

=IFERROR(VLOOKUP($A$1,$B$2:$AO$2300,MATCH("Ratio",$B$1:$AZ$1,0),FALSE),"")
