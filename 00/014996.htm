Here is another macro approach that you might find easier to use. Simply select the cells in the column (blank cells and all), run the macro, select the cell you want to copy the parsed list to (when asked) and click OK.

Sub MoveAndCondense()
  Dim CellValues As String, Destination As Range
  With WorksheetFunction
    CellValues = .Trim(Join(.Transpose(Selection)))
    Set Destination = Application.InputBox("Select cell to start copied list at.", Type:=8)
    Destination.Resize(UBound(Split(CellValues)) + 1) = .Transpose(Split(CellValues))
  End With
End Sub

In cases where the DisableAntiSpyware and DisableAntivirus DWORDs either don’t get added to the Windows Defender registry key, or where the existing values aren’t changed from 0 to 1 (disabled False to disabled True), then the screen will look like this when you try to start Windows Defender:

This will probably turn out to be a harmless issue in most cases; and I want to stress, once again, that there's no need for any immediate action. Of course if we look at the other side of this coin, where a third-party AV app is being uninstalled, then the consequences of not properly changing the value of these DWORDs  from 1 to 0 (disabled True to disabled False) might not be nearly so benign.

[Edit for review and update, 4/26/2016]

I first observed incorrect values for the DisableAntiSpyware and DisableAntiVirus DWORDs (in the Windows Defender registry key) in this thread – where Windows Defender should have been enabled, assuming that no other AV provider was registered with the Security Center, but where it was in fact disabled:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender-is-switched-off-and-i-cannot-get/ecdfdc6f-6fb4-42f0-af57-58e0f524deef

This led me to hypothesize that in the opposite case, where an AV app had been installed, these same DWORD values  might not be getting properly updated to the value of 1 (disabled true) and that this might explain some very prevalent issues with Windows Defender. So I looked at the effects of directly changing these values to the incorrect value (0) when Avast Free was installed, and I posted the results here:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-defender-error-code-0x8007139f/93096739-16ce-471b-8716-1776f0c69f89?page=2

Searching the forum for confirmation, I turned up these threads, which do seem to confirm the hypothesis:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_updating/windows-defender-conflict-in-windows-10-with-other/36fc885f-d2b7-4ca4-b2c8-c91cd9533a7b

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/definition-update-for-windows-defender-kb2267602/39c69902-5580-4634-8ebf-4d2da4048888

The registry query suggested by CJ. Eichholz (MSFT) in the first thread provides an easy way to check the value of the DisableAntiSpyware DWORD and thus confirm or disconfirm that this is the source of the issue. Run this command line at the command prompt:

reg query "hklm\software\microsoft\windows defender" /v DisableAntispyware

If you have a third-party AV app installed, then the return value should be 0x1 (disabled True). In cases where 0x0 (disabled False) is returned, you should probably disable Windows Defender with Group Policy, since it appears that uninstalling and running the removal tool for the third-party AV app, and then reinstalling it, didn’t correct the problem. For the Home edition of Windows 10, you can disable Windows Defender with the 10 Forums Turn_Off_Windows_Defender.reg  file:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

In some cases it might be necessary to first uninstall the third-party AV app and run the removal tool, and then turn off Windows Defender with Group Policy before reinstalling the third-party app, as has previously been suggested for resolving the Windows Defender definition update issue.

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_updating/definition-update-for-windows-defender-fails-over/02927e61-14b5-4255-b4d7-e8c93114fdd0

A possible confounding factor would be the presence of the DisableAntiSpyware and DisableAntiVirus DWORDs in the HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\WinDefend registry key:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-defender-keeps-turning-off-by-itself/20ea213d-50d0-4bd3-942e-9b577570ca1b

[End Edit]
