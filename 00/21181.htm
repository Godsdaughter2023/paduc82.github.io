ruel mayoya msft say:

Hi DaveTQM,

From my experience you receive this error whenever an Office 365 user tries to share his/her calendar with an on-premises user or an external user. Or, an on-premises user tries to share his/her calendar with an Office 365 user or an external user. In this case, I would like to confirm if one of the users has tried to share their calendar with the cross-premises user.

If yes, then this problem occurs if the sharing policy doesn't allow the user to share the level of detail that the user sets in the sharing invitation.  

To fix this problem, please follow these steps (as an Admin):

1. On the on-premises server, open the Exchange Management Shell.

2. Connect to Exchange Online using Remote PowerShell, determine the sharing policy that’s assigned to the user and note the policy returned, by running this commandlet:

Get-Mailbox User1 | fl *sharing*

(change “User1” to the alias or email address of the user)

3. Update the sharing policy for the user. Go to Exchange admin center > Organization > Individual Sharing > d.click the policy assigned to the user (policy result from step 2) > Sharing Rule > Update the information you want to share > Save.

If you are not sharing the calendar with an on-premises server (internal sharing), then this issue could occur when an internal recipient is also in your Contacts list in Outlook. When you add this recipient to a Calendar Sharing message, the autocomplete feature of Outlook adds the recipient to the recipient list by using the recipient's external email address.

To work around this issue, select internal recipients directly from the global address list. Please follow these steps:

  1. In the Sharing invitation, click the To button to open the Select names window.
  2. In the Address, select Global Address List.
  3. In the Search box, type the recipient's name.
  4. Select the recipient to whom you want to send the invitation, and then click To.
  5. Repeat steps 3 and 4 for each internal Exchange recipient to whom you want to send the invitation.
  6. Click OK, and then click Send.

Feel free to post back if you have further questions.

Best Regards,

Ruel

You don't say WhatOSX version you use. More Recent version Lion on up at least Show The following in the User & Groups Control panel in the System Preferences.

  1. the annotations Shown have nothing to do with your question

  7. If made no difference then you have some type of corruption either in the

windows seems to forget modem drivers installed

with win 10 tech you can install zoom pci modem driver but with in 3 min windows seems to forget these and programs such as phone tree doesn't work and you can a nag every 20 mins that there no driver installed. How to fix this

a. user say:

Hi,

 

Thank you for posting your query in Windows 10 Insider Preview.

 

I understand that while using Phone Tree software you are getting a prompt that the drivers are not installed.

 

If you have installed a driver in the computer, it should be there unless and until you remove it.

 

To check if the driver is installed, please follow the steps below:

 

  1. Press Windows key + R
  2. Type devmgmt.msc
  3. You will get the list of the device installed or present on the computer, please check if Zoom PCI is a part of it.

 

The issue could have occurred if the Phone Tree software is not installed properly.

 

Please try installing the driver and the software in compatibility mode and let us know if the issue persists.

 

To install a driver and software in compatibility mode, please follow these steps:

 

  1. Right click on the setup file of the program/driver and select Properties
  2. Select Compatibility Tab
  3. Place a check mark next to Run this program in Compatibility mode and select the operating system accordingly from the drop down list
  4. Click OK and run the setup.

 

Hope this helps, do let us know if you need further assistance with Windows, we will be happy to assist you.
