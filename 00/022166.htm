Function GetRef() As String

'GYY-DD/MM/###

Const TABLE_LOCKED = 3262

 

On Error GoTo Error_Handler

 

Dim rs As DAO.Recordset

Dim tmpNum As Integer

Dim Y As String

Dim D As String

Dim M As String

Y = Format(Date, "yy")

ken sheridan say: This raises a number of issues:

1.  Two or more attributes should not be stored in a single column.  Store the formatted date value in one column of short text data type, and the serial number in a second, of integer number data type.  I would recommend storing the date in the ISO standard format for date notation of YYYY-MM-DD as this is sortable and internationally unambiguous, but the choice is yours.  You can generate the formatted date value in the form's BeforeInsert event procedure with:

    Me.FileDate = Format(VBA.Date,"yyyy-mm-dd")

2.  The SerialNumber column is a candidate key, so should either be defined as the primary key, or, if a separate surrogate key autonumber column is used, should be uniquely indexed.

3.  In a multi-user environment conflicts might arise in obtaining the serial number.  For means of preventing this take a look at Concat.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates two methods of generating  sequential numbers.  One generates a 'pseudo autonumber' by retrieving the number from a separate table, but in my case the table is in a separate database which is opened exclusively to avoid conflicts when retrieving the number.  This method is based on one originally published by Ken Getz entity type al in the Access Developer's Handbook many years ago.  The other method, for 'sequential numbering', is a development of one published by Roger Carlson, which uses error handling to correct conflicts.  I'd recommend the latter approach, which is simpler.  In both cases I've added functionality to allow the next number to be 'seeded'.

Whenever you need to see the structured file number, you can do so by concatenating the values of the two columns, e.g. in an unbound text box on the form with a ControlSource property of:

    =[FileDate] & "/" & Format([SerialNumber],"000")

I know personally there is a problem under Vista (for only some PC owners) that occurs during login when MSE is turned On (turned On means MSE is actively monitoring for Virus and Malware)

MSE 4.9 includes a new Administrator setting named "Turn off this app" that allows a user to Turn OFF MSE (i.e. monitoring for virus and malware) by default this setting is turned ON.

If a black screen occurs (sometimes after a message "Configuring Desktop" is displayed) and nothing happens, no logon to the Desktop. Press control-alt-Del to bring up a menu page. Near the right bottom of the page there is a restart or power down button.

You can then restart and boot to Safe Mode by pressing F8. While in Safe Mode open MSE and uncheck the Administer setting "Turn on this app". This will prevent MSE from actively monitoring from a regular boot login.

After another restart from Safe Mode and normal Login you can then turn ON MSE by opening the Security Center from the Control Panel and enabling Virus and Malware protection. This essentially changes the MSE's Administrator setting and checks "Turn on this app"

If you perform a Windows restart you would either need to uncheck "Turn on this app" essentially turning off MSE or  boot to Safe Mode to turn off MSE.

I'm not certain exactly what the conflict is, one user suggested it may have something to do with the ShellHWDetection service.

Under Windows Vista, you can view different types of errors and conflicts  by going to the Control Panel > System > Performance (under see also) > Problem Reports and Solutions (under see also)
