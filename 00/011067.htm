To prevent Word from tracking format changes in a particular file, you will have to clear the "Track formatting" option in the Advanced Track Changes Options dialog box:

Changing the slide start number in Powerpoint 2013 - Microsoft Community

algebe say:

Changing the slide start number in Powerpoint 2013

This is an answer rather than a question. I thought the information might save someone else the time it took me to figure it out.

I had to submit a PowerPoint presentation to a client in three parts, and I wanted the slide numbers in each part to start where the previous one left off. There is a command for this. Go to Design. On the right, you'll see a button called "Slide Size." Click that and a drop-down box for "Standard" and "Widescreen" will appear.  At the bottom of that is "Custom Slide Size." Click that and the "Slide Size" drop down box will appear. "Number slides from" is at the bottom of that box.

The question I'd really like answered is what does slide numbering have to do with slide size? Is this what passes for intuitiveness in the new Office paradigm? In PowerPoint 2003 the same command was two clicks away in File-Page Setup. In the new paradigm it's four clicks away in a totally illogical place. Unfortunately that's typical.

 I did that, but I got annoyed seeing the "Slide Size" hint bubble, which cannot be renamed because it's a native command. So I wrote the following macro and put that on the QAT instead with "Change slide number" as the hint. Anyone is welcome to use it, but no warranty is provided or implied.

Now I need to find a way to make a meaningful icon for it, because Icon editing is another thing that the MS geniuses wrecked. In the meantime, I've chosen the butterfly in MS corporate colors from the miniscule selection of meaningless icons thoughtlessly provided by our friends in Richmond. It seems to symbolize the amount of thought that went into the Office 2013 interface design.

Sub pnumber()
message = "New page number"
  pnum = InputBox(message)
    With ActivePresentation.PageSetup
           .FirstSlideNumber = pnum
        End With
End Sub
