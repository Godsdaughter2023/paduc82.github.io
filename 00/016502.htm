rblake50 say:

I just ran into the OneNote sync error code: 0xE000004A for a notebook stored on a SharePoint server.

After some investigation, I discovered that because the SharePoint server is configured to retain the version history for stored files, and OneNote/SharePoint were automatically generating a new minor revision on the file whenever a OneNote folder was changed.

Once the file version got to 0.511, I hit the minor-version limit of SharePoint. (I don't know if this is a universal limit, or if perhaps it was configured by our SharePoint administrator.)

I solved the problem by choosing the option to 'Publish a new Major Version of this file' in the SharePoint view of the file that contains the OneNote folder. It bumped the version from 0.511 to 1.0, and now OneNote is once again successfully syncing the notebook.

(I was impatient, and just closed / re-opened OneNote as soon as I created the new version of the file, so I don't know if Syncing would restart on it's own without a restart.)

suzanne s. barnhill say:

I think you are misunderstanding "mirror indents." Like "mirror margins," "mirror indents" alternates margins left and right. So if you have a 1.5" left indent and a 1" right indent, these will become the Inside and Outside margins, alternating on odd and even pages ("mirror margins" does the same for the entire document).

If what you want is just to have the bulleted paragraphs indented on both sides, you should just add a right indent as well as a left indent. The left indents (left, first-line, hanging) must be set in the Define New Multilevel List dialog, but the right indent can be set in the Paragraph dialog.

fishal say:

access 2007: compare data between 2 exact databases

We have created an access 2007 database for initial data entry.  Afterwhich we created an identical database for the second entry of the same data.  We want to compare the data entered between the two databases to clean the data. How do we complete this process?

jeeped say:

How many fields are there in the table common to both databases? How many fields would have to be examined and compared between the two in order to definitively define uniqueness? Is there a PK (e.g. Primary Key) that should be left behind?

It could be as simple as something like this,

INSERT INTO tblOne ( fld1, fld2, fld3, fld4 )
    SELECT t.fld1, t.fld2, t.fld3, t.fld4
        FROM tblTwo AS t
        WHERE t.fld1 Not In (SELECT fld1 FROM tblOne);

 ... or significantly more difficult depending upon what it takes to guarantee that only unique values cross over.

  • If this proposed solution has resolved your issue(s), please return and mark it as Answered for others to consider.

gina whipp say:

As posted by Allen Browne...

1. Create a query that inner-joins the 2 tables on all fields. Save as (say) qryMatch.

2. Create another query, using the 'unmatched query wizard' (in the first dialog when you create a new query) to get the records from table A that are not in qryMatch.

3. Repeat step 2 for table B.

4. (Optional) create another new query to UNION ALL the results of step 2 and step 3.
