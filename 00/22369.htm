Password protect back end files - Microsoft Community

waltejr say:

Password protect back end files

I would like to password protect the back end files of a split Access 2007 database so that users cannot open them and modify data (potentially destroy a file).  When I tried to encrypt just the back end, the program wouldn't run - an error message indicating an unrecognized password came up.  Is there any way to encrypt just the back end files?
daniel pineault say:

You need to open the back-end file in exclusive mode and password protect it (see http://office.microsoft.com/en-us/access-help/encrypt-a-database-by-using-a-database-password-HA010096299.aspx#BM2).

Then you will need to open you Front-end, delete all your linked tables and then reconnect them (and provide the password when prompted).

Do keep in mind that database password protection is very limited!  It is a good security feature to prevent most users from messing around, but it can easily be cracked (many tools to do so available freely online).  But for most office scenarios, it is enough to stop those nosy people.  That said, your best friend should always be the BACKUP!  Setup a nightly automated backup process, and this way no matter what ever happens, you will always be safe.

daniel g. grau say:
@ dr ap:

There are three parameters involved:

  1. Window size (controlled by grabbing the bottom-right corner)
  2. Zoom percentage (controlled through the slider at the bottom or the numerical options in View > Zoom). This does not re-dimension the document's window.
  3. Number of pages viewed (controlled through the non-numerical options Page width, Whole page, Manu pages in View > Zoom). This does not re-dimension the document's window either.

The last two items act within the available space, i.e., the document's window:

  • Page width will show the document as wide as possible within the current window. That is, it changes the zoom percentage so the document takes up all the horizontal space in the window. If the window is large, it might show it at 150% and if it's small, at 75%.
  • Whole page will show a full page within the window, scaling the zoom view as needed. As in Page width, for a large window size, View > Zoom > Many pages might show them at 80%, and for a smaller window, they will appear at 50%. In addition, the pages will show in a column.
  • Many pages will display as many pages as possible, horizontally, within the current window's width, given the current zoom percentage. It does not change the scaling.

daniel g. grau say:
This can happen if your document has more than one section. If so, when deleting that first page, you are deleting the whole section where you've specified "Different first page."

Go to the header (View > Header and Footer) and take a look at the blue tab under the header. It should say "Header | Close" rather than "Header -Section 1 | Close."

To better locate the problem spots, click on the ¶ button in the "Standard" toolbar to toggle the display of non-printing characters (if you don't see this toolbar, select View > Toolbars > Standard). You will see "====== Section Break ======" at the point where there is a new section. Delete that blue symbol just like you would any character, then redefine the different first page.
