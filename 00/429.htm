    jake gamlieli say: unfortunately this is an issue with the ATT setup putting in a proxy entry in the registry which prevents it from connecting.  There is a fix, however it involves registry edits.  I am enclosing a note below from the person who discovered the fix, and it does work.  Alternately, to doing the registry edits there is a cab that will supposedly do it for you.  there is another one to undo the change as well.  You can get both here :  http://forum.xda-developers.com/showpost.php?p=4728750&postcount=1035

    If the above cab does not work, i suggest that you use the restore proxy cab which will undo it, and then manually do the regedits below:
    The Tilt 2 OEM ROM includes a pretty cool YouTube app located in the Windows directory. However, sometimes it would work, and sometimes it would time out on attempting to connect to play the movie.

    <<<<Courtesy of Vendetta from ATT forums and XDA developers website>>>>>
    Here's what I learned while getting the HTC YouTube application included in the OEM AT&T ROM to work perfectly.


    The value in the registry needs to be modified:
    HKEY_LOCAL_MACHINE\Software\HTC\YouTube
    ProxyName = wireless.cingular.com:80 (default value) needs to be replaced with [BLANK].


    This value is intermittently restored, so it is nececssary to change a few others to prevent that.


    The value at
    HKEY_LOCAL_MACHINE\Software\SUN\JVM\PRELOADAPP
    proxy = wireless.cingular.com:80 (default value) needs to be replaced with [BLANK].


    The value at
    HKEY_LOCAL_MACHINE\Comm\ConnMgr\Providers\{EF097F4 C-DC4B-4c98-8FF6-AEF805DC0E8E}\HTTP-{436EF144-B4FB-4863-A041-8F905A62C572}
    Proxy = wireless.cingular.com:80 (default value) needs to be replaced with [BLANK].


    The value at
    HKEY_LOCAL_MACHINE\Software\Apps\Esmertec Java\Properties
    com.sun.midp.io.http.proxy = wireless.cingular.com:80 (default value) needs to be replaced with [BLANK].


    Changing these three values will prevent the YouTube proxy from being restored. (However, this will prevent the Opera Browser from connecting when the phone is connected via Wi-Fi. In order to fix this, the value at
    HKEY_LOCAL_MACHINE\Software\Opera\Defaults
    ByPassProxyForNonGPRS = 1 (default value) needs to be changed to 0.)


    EDIT: The greyed out text above in my original post is mostly likely unrelated to the YouTube Proxy issue. For some reason, my Opera browser seems to have trouble connecting over WiFi when the "Data Connection" is turned on in the CommManager.


    These changes have been in place on my phone now for about a week now and have survived several resets, power cycles, and tower changes.


    Good luck and enjoy the included YouTube app without a hitch!


    -Vendetta

    It sounds like your problem is that your network hasn't released their version of the firmware - I'm on Voda UK and I have the same version of firmware you do.

    The way this works is Nokia develop a firmware upgrade. They release this to SIM-free customers and to the networks.

    The networks SHOULD then apply their customisations -- branding / pre-configuration of network services like sms, voicemail etc / removal of some features -- to the firmware and then pass this revised firmware back to Nokia who make it available to NSU and the service centres.

    Unfortunately, some networks are not so good at doing their bit - voda and o2 are probably the worst offenders in the UK. Orange are not far behind.

    There is no legal way for you to get your firmware upgraded unless and until it your network releases it. Any illegal ways will void your warranty.
