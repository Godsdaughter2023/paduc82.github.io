Denial of Service attacks are entirely external and are not dependent on any form of program on your computer itself. They are used specifically because they don't need anything on the other side of the firewall. Hackers are continually probing the internet looking for interesting targets and would have been trying to break the firewall of the router to see what is behind it. I see fairly short lived attacks commonly in router logs.

There have been some (not many) examples of weaknesses it router firmware but whereas users mostly accept keeping their computer up to date they rarely think of the router that is the 1st firewall in your network. It is a good idea to periodically check to see if there are any firmware updates for the router itself. Netgear did have that function as an option whenever somebody logged in to manage the router. Some ISP may do this externally for the equipment they provide (UK provider British Telecom do)

bytemaster say:

NTFS Errors can occur in sections of the hard drive that were previously unused but are now being used.  An operating system update requires a lot of files to be updated, copied, and moved around on the physical hard drive.  If one of these previously unused areas is damaged on the hard drive, it will appear as though everything is running fine until the system uses this space, which then causes read errors that don't line up with the integrity checks in place to prevent errors - thus the NTFS error.

On physical, magnetic hard drives drives this is more likely then on solid state drives.  Physical drives will try to reuse open areas and only use the new areas if necessary.  This has to do with spin patterns and faster retrieval on some hard drives.  Solid state drives wear-level the writes so as not to burn out an individual area from too much use - which is not a big issue on magnetic drives.

In short, it is most likely that your system had an existing problem that is only surfacing due to the size of the new installation.
