john w. vinson say:

There are two ways to do "remote desktop". 

One is to use a remote terminal server so that the computer you're typing into and looking at is acting like a "dumb terminal": you're logging on to a session on the host computer. That host computer has Access installed, a copy of the Access frontend database, and links (over a fast stable LAN) to the backend database. Your computer in this case does not need Access or any database files installed - just the remote terminal software.

THIS WORKS.

Another way tries to have an Access frontend database on your computer communicating over the Internet - over a remote, package-switched network - to a backend database on the server computer containing the data.

THIS DOES NOT WORK and will cause no end of trouble.

Which are you trying to use?

albert d. kallal say:

Ok, this “sounds” like the correct set.

 

So you should able to log into that remote system. Now on the remote system, you launch Access. You then use Access to open your front end with the linked tables. (And if code and start up forms are trying to run, hold down the shift key to prevent application start up).

 

At this point, you have your application open, and now can choose the linked table manager, and re-link your tables to the back end data file that is also on that remote system.

 

So far, it sounds like you have a workable setup, you just need to use the Access linked table manager to re-connect your front end to the data file back end. As long as your launching Access on that remote system, and doing everything remote, then your setup should work.
