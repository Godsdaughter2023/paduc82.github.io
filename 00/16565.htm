jw aldridge say:

conditional format adjacent cell based on value

jllatham say:

I'm not sure what you meant by "the adjacent cell in B" unless you mean adjacent to column C.

How I looked at it: Looking for "Deleted" in B3:B100000 and if found, then make C on same row font red and bold (otherwise black and not bold).  I've also set it up so if last used row in B is less than 100,000 then that's all that will be examined.  But if you have formulas in B, then it may always do 99,998 anyhow - and it takes a bit of time to do it: about 6 seconds on a 3GHz dual-core system.

Why not just use standard conditional formatting (other than that you'd have 99,998 volatile formulas recalculating every time you changed something/anything on the sheet)?

Sub MakeItRed()
'worksheet with data to test must be active when run
  Const ColToMakeRed = "C" ' change as required
  Const keyPhrase = "DELETED" ' all uppercase for testing
  Const testCol = "B"
  Const firstRow = 3
  Const maxRow = 100000
 
  Dim lastRow As Long
  Dim offset2RedCol As Integer
  Dim testRange As Range
  Dim anyTestCell As Range
 
  'see if we can do fewer than 99,998
  lastRow = Range(testCol & Rows.Count).End(xlUp).Row
  'set test range based on fewest rows to have to work with
  'if lastRow > 100,000 then reduce to 100,000
  If maxRow < lastRow Then
    lastRow = maxRow
  End If
  Set testRange = Range(testCol & firstRow & ":" _
   & testCol & lastRow)
 
  offset2RedCol = _
   Range(ColToMakeRed & 1).Column - Range(testCol & 1).Column
  Application.ScreenUpdating = False
  For Each anyTestCell In testRange
    'assume not a match, and to clean up anything
    'that was made red before but should not be now
    With anyTestCell.Offset(0, offset2RedCol)
      .Font.Bold = False
      .Font.ColorIndex = xlAutomatic
      If UCase(Trim(anyTestCell)) = keyPhrase Then
        .Font.Bold = True
        .Font.Color = vbRed
      End If
    End With
  Next
  Set testRange = Nothing ' good housekeeping
End Sub

sam wang msft say: Hi Phil,

Thanks for your confirmation.

I have checked the whole thread and would like to share two more workarounds with you since the user is not able to use cached mode in the Outlook client.
1. You can refer to Method 2 in the article mentioned above and run Set-Mailbox <mailbox name> -MessageCopyForSentAsEnabled $True for the shared mailbox, so a copy of sent message will be preserved in the sent items folder of the shared mailbox. However, here are a few things you may need to be aware of:
1) Like described in the article, the cmdlet only works if the Outlook DelegateSentItemsStyle registry value is disabled. Therefore, please disable the registry key you’ve set on the user’s desktop first before running this PowerShell cmdlet.
2) The reason why I say it’s just a workaround is that although it works in either cached mode or online mode of the Outlook client, the sent message will still be saved in the sent items folder of the primary mailbox. The only difference is that a copy of that message will be retained in the sent items folder of the shared mailbox also.
3) Please connect to Exchange Online PowerShell before running the cmdlet.
2. Based on my experience, if you want the sent items of the shared mailbox to be saved in the sent items folder of the shared mailbox only, you can just open the shared mailbox via opening another mailbox in the Outlook Web App. Then you can manage the sent and received items of the shared mailbox in a separate browser and all items will no longer be moved to the folders of the user’s primary mailbox. In this case, you don’t need to take any action on the shared mailbox. For the detailed steps, please refer to the Open another person’s mailbox in a separate browser window in Outlook Web App part in the article below:
https://support.office.com/en-us/article/Access-another-person-s-mailbox-A909AD30-E413-40B5-A487-0EA70B763081?ui=en-US&rs=en-US&ad=US

suzanne s. barnhill say:

On the Margins tab of Page Setup, what is selected for "Multiple pages"? If it is "Book fold" or "2 pages per sheet," change it to "Normal."

If it is already "Normal," go to the Columns dialog (Page Layout | Page Setup) and see how many columns are selected. If it's more than one, change it to one (using the One preset).

Alternatively, when you click in the text, is the contextual Table Tools tab added to the Ribbon? If so, the text is in a table; it should be possible to get the text out intact, but instructions will depend on the number of columns (and rows) in the table.

Another possibility is that the text is in text boxes or frames. If you click in the text, do you get a dotted line around it? Does the border have white or black sizing handles?
