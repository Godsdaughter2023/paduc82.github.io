WORD Custom Document Properties - Microsoft Community

rocksaltlife say:

WORD Custom Document Properties

Hi All,

Recently I have written an bit of code that takes variables from one program and inserts them, using custom document properties, into a word document. However some of the variables used are filled by users with some detail. I have learnt recently that word is limited to 255 characters using the custom document properties and was wondering if this can be increased in any way so that the blocks of text copy over fully?

Any help with this would be greatly appreciated.

a. user say:

The 255 is a built-in limitation.

You have two options - use Document Variables instead, or use mapped content controls instead. Document Variables are simpler (simpler than Custom Document Properties IMO). The main difference from Document Properties is that there is no built-in user interface that would allow the user to change the values. But that may suit you in this case.

To create or set a DOcument Variable, use

ActiveDocument.Variables("thevariablename") = "the text value"

To insert the value in the document, use a field code (ctrl-F9 for the special field code braces { })

{ DOCVARIABLE thevariablename }

They have to be updated using select then F9, same as DOCPROPERTY fields.

Custom XML Parts + mapped content controls have some advantages, e.g. modifying the data in the part results in an immediate update to Content Controls linked to it, although if you have any Mac Word users not all of the functionality.

You need a Custom XML Part with an XML element for each property, and you need a Plain Text Content Control that is mapped to each part.

Some sample VBA for use with an empty document that shows how to create a Custom XML Part and some mapped Content Controls, and what happens when you update the data in it...

With an empty document open, run the first sub. Look at the resulting document. 

Run the second Sub.

Notice that you can go over 255 characters *and* have multiple lines.

' This should be set to a string that should ideally be
' a unique name @owned@ by you or your organisation
' Some people use a domain name or web address that they
' own as a "stem", then a qualifier
' www.somedomainiown.com/samples/20180103"
' (The web address does not actually have to exist)
Const UniqueURI As String = "YourUniqueID"
Sub recreateandlinkcxps()
Dim cc As Word.ContentControl
Dim cxp As Office.CustomXMLPart
Dim i As Integer
Dim myprefix As String
Dim s As String
s = "<?xml version = '1.0'?>" & vbCrLf
s = s & "<myns:root xmlns:myns='" & UniqueURI & "'>" & vbCrLf
s = s & "<myns:property1/>" & vbCrLf
s = s & "<myns:property2/>" & vbCrLf
s = s & "</myns:root>" & vbCrLf

With ActiveDocument

  For Each cc In .ContentControls
    cc.LockContentControl = False
  Next
  .Content.Delete
  .Content.InsertParagraphAfter
  .Content.InsertParagraphAfter
  .Content.InsertParagraphAfter
  .Content.InsertParagraphAfter
  
  With .CustomXMLParts
    ' remove non-builtins
    For i = .Count To 1 Step -1
      If Not .Item(i).BuiltIn Then
        .Item(i).Delete
      End If
    Next
    Set cxp = .Add(s)
    'Debug.Print cxp.XML
  End With

  myprefix = cxp.NamespaceManager.LookupPrefix(UniqueURI)
  mapping = "/" & myprefix & ":root[1]/" & myprefix & ":property"
  With .Content.Paragraphs(1).Range.ContentControls
    .Add wdContentControlText
    With .Item(1)
      .XMLMapping.SetMapping mapping & "1[1]", , cxp
      .LockContentControl = True
      .MultiLine = True
    End With
  End With
  
  With .Content.Paragraphs(2).Range.ContentControls
    .Add wdContentControlText
    With .Item(1)
      .XMLMapping.SetMapping mapping & "2[1]", , cxp
      ' make the control undeletable
      .LockContentControl = True
      .MultiLine = True
    End With
  End With

  With .Content.Paragraphs(3).Range.ContentControls
    .Add wdContentControlText
    With .Item(1)
      .XMLMapping.SetMapping mapping & "1[1]", , cxp
      .LockContentControl = True
      .LockContents = True
      .MultiLine = True
    End With
  End With
  
  ' two read-only versions connected to the same elements
  With .Content.Paragraphs(4).Range.ContentControls
    .Add wdContentControlText
    With .Item(1)
      .XMLMapping.SetMapping mapping & "2[1]", , cxp
      .LockContentControl = True
      .LockContents = True
      .MultiLine = True
    End With
  End With

End With
Set cxp = Nothing
End Sub

Sub populateCXP()
Dim pre As String
Dim cxps As Office.CustomXMLParts
Dim s As String
' build a string >> 255 characters long
s = ""
s = s & String(100, "a") & vbCrLf
s = s & String(100, "b") & vbCrLf
s = s & String(51, "c") & vbCrLf
s = s & String(49, "d")
Set cxps = ActiveDocument.CustomXMLParts.SelectByNamespace(UniqueURI)
If cxps.Count > 0 Then
  With cxps(1)
    .SelectSingleNode("//ns0:property1[1]").Text = "abc"
    .SelectSingleNode("//ns0:property2[1]").Text = s
  End With
End If
Set cxps = Nothing
End Sub
