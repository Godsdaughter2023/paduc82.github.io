EDGE WebVIEW: Indexeddb writes are cached and not written to disk

We've been building a hybrid Universal application with cordova and noticed that the latest summer update has introduced a defect in the EDGE WebView Indexeddb writes that is causing loss of data.

Basically writes to the db are being cached and not written in a timely manner. Meaning we can wait upto 15 minutes turn off the tablet restore the table start the app and see that the data we "wrote" to the db is now gone. This did NOT happen with Windows 10 1511, but is definately reproducible on 1607. 

Unfortunately, this "data" corruption is serious enough that we may need to consider moving away from windows to provide our health care solutions. FWIW: Android does not have this flaw with indexeddb.

If this was an intentional change, we would accept having the ability to force a flush of the cache on demand, or as a different transaction mode.

wcalderw say:

I've been able to reproduce this issue using the Microsoft Provided Universal Application Sample for Indexeddb

https://github.com/Microsoft/Windows-universal-samples/tree/master/Samples/IndexedDB

On Windows 10 1607 (and insider 1700) if you exit the app and then power down your app after committing writes (receiving the oncomplete event) to the database. The data will be lost.

On windows 10 1511 if you power down right after committing (receiving the oncomplete event) the writes the data will still be saved (which is what you want from a db).

jay freedman say:

Click the Review tab, and click the Protect Document button at the right end of the ribbon. In the pane that opens on the right side of the window, if you see a Stop Protection button at the bottom, click it. Depending on what was done when the document was protected, you may be prompted to enter a password.

If the document requests a password to stop protection, but you don't have the password, you can work around it. (Note: This works for an editing restriction password, but not for a  password required to open the document.) Open a new blank document. On the Insert tab of the ribbon, click the down arrow on the Object button and choose Text from File. In the dialog that opens, select the protected document and click the Insert button. This will put the text of the protected file into the new document, without any protection.
