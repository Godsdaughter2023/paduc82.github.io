¡firedog say:

Because mail servers only understand ASCII text, email messages have to be converted to plain text before transmission. This will not make any difference if the message consists only of ASCII text anyway, but as soon as other characters get involved, encoding has to take place.

Quoted-Printable encoding replaces the odd out-of-range character as an equals sign followed by its two-digit hex code. This covers the characters used in most Latin languages. Because of this use of the equals sign, any equals sign in the text has to be encoded, too, which is why you see =3D everywhere when HTML has been QP-encoded.

Any text containing more than a few non-ASCII characters - e.g. in a Far Eastern language - can be encoded using QP, but this would be very inefficient. Base 64 uses three ASCII characters to encode any character, so this is used in these cases. It is also used to encode binary files like, say, a JPG image file. The program does this regardless of your encoding preference.

For ordinary North American users, the choice you make in the encoding dialogue is immaterial. If you do choose Base 64, recipients won't be able to read the message source without putting it through a decoder first. Using B64 has been known to solve some problems caused by bad QP encoding, so it's also another weapon in the troubleshooter's armoury.

These are the characters that mail servers understand, borrowed from Wikipedia's ASCII article:
