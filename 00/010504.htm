It depends on the width of the Excel application window.

If the window is very narrow, the entire Styles group will be collapsed to one dropdown button.

stefan blom say:

See if some Spacing After (or Spacing Before) has been added to one or more of your paragraph styles. The specified value will be added between paragraphs (but not between text lines within a paragraph).

Note that Spacing Before/After may be suppressed for two or more consecutive paragraphs. This is controlled by the "Don't add space between paragraphs of the same style" option (Paragraph dialog box).

Excel data entry, transaction recording. - Microsoft Community

tenison say:

Excel data entry, transaction recording.

I have a large worksheet in which data is entered for a group of suppliers. I would like an automatic entry recorded with each new data line entered to record the date and time of each entry. The purpose is to be able to report the time taken on various dates for each supplier's data.

Any help would be greatly appreciated.

rajesh c. say:

Try the following:

1. Assume that your data entry is from columns A thru H.

2. Assume that you wish to capture the date in column-J and the system time in column-K.

3. Click on FILE >> OPTIONS >> FORMULAS >> Enable iterative calculation by

    tick-marking the box adjacent to the title "Enable iterative calculation"

4. Format the Column-J as the Date format of your wish

5. Format the Column-K as the Time format of your wish

6. Insert the following formula in cell J2

                         =IF(A2<>"",IF(J2="",now(),J2),"")

7. Insert the following formula in cell K2

                         =IF(A2<>"",IF(K2="",now(),K2),"")

You can drag the above formulas up till the range you wish in those respective columns.

NOTE:   Here, you have to ensure that column-A will be a mandatory data field

Hope this Helps.

I would like to be able to play with the slicers and have the graphs update automatically when I do.

I cannot use a Pivot table because some tables are based on data crossing different questions, and a pivot works on a single table at a time. Create a macro that copy pastes the table as values into a new sheet, to be activated after applying the filters on the slicers. The graphs' data sources are linked to the values table.

Complaints:

tom ogilvy say:

Floriana GUARDINI,

Subtotal is designed to work with the data that is visible in the table.  So Countif functionality is inherent to the function if you are counting the data that meets the criteria for the visible rows of the table. 

--

Regards,

Tom Ogilvy

  

bernie deitrick say:

SUBTOTAL does include COUNTIF - it COUNTS the values IF they match all of the filter requirements.

If you want other COUNTIF filtering, then you need to have a column of formulas in the filtered table that will change with the filtering, like

=COUNTIF(3,A2)

Copied down.  That will give you a 0/1 value that you can then feed to your other formula.

tom ogilvy say:

Floriana GUARDINI,

so you are correct that for what you are asking for, subtotal will not work.  A method that will work is the one you found and cited on Chandoo.org   ( http://chandoo.org/wp/2012/06/07/formula-forensics-023/ )

As he explains in his forensic review of the formula, the whole subtotal part of the formula is used to identify which rows are visible and which are hidden.  It will produce an array across you data the has a 0 for each row that is hidden and a 1 for each row that is visible.  The remainder of the function can be used to apply whatever condition you want to evaluate such as how many A's that are male (filtering identifies the male subset).

That formula or some derivation of it is pretty much the common way to do it.

You're not ready yet for calculations; it has to start with a correct Relational Database design that follows the rules of normalization.

Calculated values don't belong in a database; rather they are calculated on the fly by queries.

tblCountries

CountryID Autonumber PK

CountryName text(50) required uniqueidx

tblCallTypes

CallTypeName text(50) required uniqueidx

tblCallData

CountryID long int required

MinutesCalled long int required

Then use the Relationships window to draw relations between the corresponding fields and in the dialog that comes up check the box to enforce the relation.
