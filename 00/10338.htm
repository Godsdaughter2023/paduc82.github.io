For example:

we shared a master folder"总经办-测试" setting "can read" for specific members so that they can access to the master folder, and then we set sub-folders with different unique accesses for different persons so that we can allow specific people to read only or edit the shared document. Then, like I said, the master folder and sub-folders both show(not just in my "shared by me", but also show the same under specific person's "shared with me"  field, which will make shared folders seem loads, contrary to our idea of making folder sharing be neat.

When master folder and sub folders are sharing the same access setting, like "总经办-测试3", "总经办测试-4", whose subfolders are not seperately shown, which is exactly what we want to achieve. So is there any setting available for us to make/set our master folders shown only, even with different access settings for sub-folders?

zixuan yuan msft say:

Hi Tina,

You are correct that OneDrive sharing doesn't work in that way. If you shared folder 1(Read permission) and subfolder 1(edit permission) to User B, he will see two records in his OneDrive for Business. This is an expected behavior. Please submit your feedback in our OneDrive UserVoice forum like we discussed above.

Meanwhile, as for sharing, the subfolder will inherit its parent's permission. After you shared folder 1 to User A&B&C, all of them will be able to view all the content in folder1 including its subfolders. Therefore, to achieve your requirement that let User A & B & C be able to view folder 1 and User B has special permission to edit subfolder 1, it only needs to share twice: share folder 1 to User A & B & C without edit permission, share subfolder 1 to User B with edit permission. After that, the following should be the results:

  1. You can see two records in Shared by me in your OneDrive for Business.
  2. User A&C can see folder 1 in Shared with me in his own OneDrive for Business. And they can only view all the contents in folder 1.
  3. User B can see folder 1 and subfolder 1 in Shared with me in his OneDrive for Business. He can edit the content in subfolder 1, but for other contents in folder 1, he can only view.

You may stop sharing and share again to see the outcome. Another good option is to create a new folder to share and test.

Regards,

Zixuan

jimr1 say:
If MSe does not appear in add/remove programs (uninstall a program in Vista and Windows 7) Try the manual uninstall method provided by MVP Aiscer:

1. Download the MSE installer for your Operating System from the Home Page:http://microsoft.com/security_essentials

2. Save the file to your Desktop. This will ensure that you can find it easily after the download is done.

3. Now press the Windows Key and R all at the same time. This will open the RUN dialog box.

4. Make sure that  you delete any entries inside RUN. Now drag the MSE installer into the RUN dialog box.

5. Depending on your Operating System, after dragging the file into RUN it should appear as any of the three below: 

         "C:\Documents and Settings\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-xp.exe"

          C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-vista-win7

          C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-amd64fre-en-us-vista-win7.exe

6. Now put a space in between the whole line and put /U as seen below:

        "C:\Documents and Settings\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-xp.exe" /U

         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-vista-win7 /U

         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-amd64fre-en-us-vista-win7.exe /U

7. Hit OK.

 If you are unable to uninstall MSE using this method you can submit a support case here -https://support.microsoftsecurityessentials.com/
