cody c. say: Hi CCG123,

Thanks for using Microsoft Answers!

I understand you're trying to use Remote Desktop to connect to a Windows 7 PC from outside your network. When you connect within your network, RDP works correctly, but when you connect from outside the network, you get an error.

This is very commonly a firewall issue and shouldn't take too much work to fix. By default, RDP uses port 3389. It's critical that this port is open on your router's firewall, server's firewall, desktop's firewall, and any other firewall between your Windows 7 PC and the internet. 

A tool that can really help this process is an online Port Scanner. The following link is to one that others have used in the forums and have had good luck with: http://www.t1shopper.com/tools/port-scanner/

Once the page loads, check the box for Microsoft Remote Desktop vulnerable (port 3389) and then press the Scan Ports button at the bottom. RDP from outside the network will not work until that test says the port is open. If the test fails, that means there is still a firewall blocking traffic to that port

If you have multiple systems on the same network and you want to use RDP on all of them, they need to listen for RDP on different port numbers. Have a look at the following Microsoft KB article on how to chance the listening port for Remote Desktop: http://support.microsoft.com/kb/306759. Once all your systems have different listening ports, open and forward the ports on each firewall. 

If you need any assistance configuring Windows Server 2003 to allow RDP access, I'd recommend posting on the TechNet Windows Server Forums here: http://social.technet.microsoft.com/Forums/en-US/winserverNAP/threads.

Let me know if I can be of further assistance!

Edit: Wanted to also recommend looking at the following Microsoft How-to: http://windows.microsoft.com/en-US/windows7/allow-remote-desktop-connections-from-outside-your-home-network
