suat.cini say:

There is a registry setting that controls Fast Startup.

HiberbootEnabled

If you set the value to 0, it disables Fast Startup on next reboot. It comes as 1 by default.

I don't know if upgrading to a new build over the current one resets the registry back to 1.

Clean-install sets the value to 1 for sure.

If you prepare a .reg file and run it, it makes things easy.

--------------------------------------------------------------------------------------------------

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\Power]
"HiberbootEnabled"=dword:00000000

----------------------------------------------------------------------------------------------------

Copy and paste the code between dashed lines into a text file and change the extension to .reg, someting like fastStart.reg

Hope this helps.

don cuthbert say:

If you have initialized Windows Sandbox, it may have placed the Microsoft Hypervisor, used for Hyper-V, in the BCD that runs during boot.

To check, run bcdedit at a Command Prompt and look for hypervisorlaunchtype Auto.

On my PC, I disable the Hypervisor in my BCD with the following and then restart, if I want to run VirtualBox or VMware:

bcdedit /set hypervisorlaunchtype off

To enable the Hypervisor:

bcdedit /set hypervisorlaunchtype Auto
