The only problem with the way the Required property works is the nature of the messages that it displays, which aren't very user-friendly.  I completely agree that it *must* be used to ensure database integrity, but I like to intercept the missing-data situation earlier and display a
friendlier message.  I do this by using each control's Tag property to indicate that it is required by putting the string "Required" in there, and then using the form's BeforeUpdate event to call a function to check whether any of the controls marked "Required" are empty:

'------ start of code in form module ------Private Sub Form_BeforeUpdate(Cancel As Integer)
   
    Cancel = fncRequiredFieldsMissing(Me)
   
End Sub'------ end of code ------

The function displays a friendlier message, listing the unfilled controls by "friendly name" and sending the focus to the first missing control in the tab order:

'------ start of code in standard module ------Function fncRequiredFieldsMissing(frm As Form) As Boolean

    ' Check form (passed as <frm> for controls marked as  "Required"
    ' in their Tag properties.  If any required control is empty, return
    ' True, display an error message listing all the "missing" fields,
    ' and set the focus to the first missing control (where "first" is
    ' based on the controls' tab order).  In the list of missing controls,
    ' use each control's caption if possible, else use the name of the
    ' control after stripping off the expected object-type prefix (if
    ' present).

    On Error Resume Next
    
    Dim ctl As Access.Control
    Dim strErrCtlName As String
    Dim strErrorMessage As String
    Dim strMsgName As String
    Dim lngErrCtlTabIndex As Long
    Dim blnNoValue As Boolean
    
    lngErrCtlTabIndex = 99999999  'more than max #controls
    
    For Each ctl In frm.Controls
        With ctl
            Select Case .ControlType
                Case acTextBox, acComboBox, acListBox, acCheckBox
                    If .Tag = "Required" Then
                        blnNoValue = False
                        If IsNull(.value) Then
                            blnNoValue = True
                        Else
                            If .ControlType = acTextBox Then
                                If Len(.value) = 0 Then
                                    blnNoValue = True
                                End If
                            End If
                        End If
                        
                        If blnNoValue Then
                        
                            strMsgName = vbNullString
                            If .Controls.Count = 1 Then
                                strMsgName = .Controls(0).Caption
                                If Right$(strMsgName, 1) = ":" Then
                                    strMsgName = Trim$(Left$(strMsgName, Len(strMsgName) - 1))
                                End If
                            End If
                            If Len(strMsgName) = 0 Then
                                strMsgName = .Name
                                Select Case Left$(strMsgName, 3)
                                    Case "txt", "cbo", "lst", "chk"
                                        strMsgName = Mid(strMsgName, 4)
                                End Select
                            End If
                            
                            strErrorMessage = strErrorMessage & vbCr & _
                                        "   " & strMsgName
                                        
                            If .TabIndex < lngErrCtlTabIndex Then
                                strErrCtlName = .Name
                                lngErrCtlTabIndex = .TabIndex
                            End If
                            
                        End If
                    End If
                Case Else
                    ' Ignore this control
            End Select
        End With
    Next ctl

    If Len(strErrorMessage) > 0 Then
        MsgBox "The following fields are required:" & vbCr & _
                strErrorMessage, _
                vbInformation, "Required Fields Are Missing"
        frm.Controls(strErrCtlName).SetFocus
        fncRequiredFieldsMissing = True
    Else
        fncRequiredFieldsMissing = False
    End If

End Function
'------ end of code ------

ken sheridan say: You need to bear in mind that a user might attempt to save a record by closing the form.  In addition to setting the Required property of  the relevant columns in the table to True (Yes), which disallows Nulls, and, in the case of columns of text data type, setting the AllowZeroLength property to False (No), which disallows zero-length strings, you might try to anticipate the error in the form's BeforeUpdate event procedure with code like this:

    Dim ctrl As Control
    Dim strMessage As String
    
    For Each ctrl In Me.Controls
        If ctrl.Tag = "EnforceMe" Then
            If IsNull(ctrl) Then
                strMessage = "A value must be entered in " & ctrl.Name & "."
                MsgBox strMessage, vbExclamation, "Invalid Opeartion"
                ctrl.SetFocus
                Cancel = True
                Exit For
            End If
        End If
    Next ctrl

The Tag property of each mandatory control should first be set to EnforceMe for this code to work.

This will work fine if the user tries to save the record by moving to another record in the form or otherwise explicitly saves the record.  The record will not be saved until the user has entered values into all mandatory controls.  It will also work if the user tries to close the form, but they will also be presented with a system error message informing them that the record can't be saved.  The dialogue gives them the opportunity to abort the closure of the form, but system messages can be confusing to users, and they could easily end up closing the form and losing the record completely.  So it would be better if this could be handled more elegantly.

To prevent a form closing it is necessary to set the return value of the form's Unload event procedure's Cancel argument to True.  This means that  there has to be some way of telling the Unload event procedure that the validation criteria have not been met.  This is usually done by setting the value of a module level Boolean variable to True or False as appropriate elsewhere in the form's module.  This is quite tricky to do, and I've always found the best solution in this sort of situation is to force the user to save a record or close the form by means of command buttons.  You'll find an example as SaveDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file, code in the form's module not only validates the data in the form's BeforeUpdate event procedure, but also enables/disables the Save, Undo and Close buttons in the form on the basis of the current state of the record.  The form also includes validation at control level, which forces the user to insert data into the controls in a specific order.
