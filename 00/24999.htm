ken sheridan say: Why not use the simpler approach of a bound form which includes an unbound navigational combo box with a row such as <New Ticket>.  You'll find an example as FindRecord.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes a number of options for navigating to records in a form.  The first option in the menu form opens a form which includes such a combo box.  Its RowSource property is:

SELECT ContactID, FirstName & " " & LastName, 1 As SortColumn,
LastName,Firstname  
FROM Contacts  
UNION
SELECT 0, "<New Contact>", 0,"",""  
FROM Contacts
ORDER BY SortColumn, LastName, FirstName;

and the code for its AfterUpdate event procedure is:

    Const MESSAGETEXT = "No matching record"
    Dim ctrl As Control
    
    Set ctrl = Me.ActiveControl
    
    If Not IsNull(ctrl) Then
        If ctrl = 0 Then
            ' go to new record and move focus to FirstName control
            DoCmd.GoToRecord acForm, Me.Name, acNewRec
            Me.FirstName.SetFocus
        Else
            With Me.RecordsetClone
                .FindFirst "ContactID = " & ctrl
                If Not .NoMatch Then
                    ' go to record by synchronizing bookmarks
                    Me.Bookmark = .Bookmark
                Else
                    MsgBox MESSAGETEXT, vbInformation, "Warning"
                End If
            End With
        End If
    End If


PS:  The combo box's NotInList event also takes the user to a new record, so you can dispense with the UNION operation in the RowSource property to show the <New Ticket> line if you wish.  The code in my case parses the value entered into first and last names, which you would not need of course:

    Dim ctrl As Control
    Dim strMessage As String
    Dim strFirstName As String, strLastName As String
    
    Set ctrl = Me.ActiveControl
    strMessage = "Add " & NewData & " to list?"
    
    strFirstName = Left(NewData, InStr(NewData, " ") - 1)
    strLastName = Mid(NewData, InStr(NewData, " ") + 1)
    
    'if user confirms insert move form to new record
    ' and insert values into first and last name controls
    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        Response = acDataErrContinue
        ctrl.Undo
        DoCmd.GoToRecord acForm, Me.Name, acNewRec
        Me.FirstName = strFirstName
        Me.LastName = strLastName
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If

Numbering rows in a Word Table - Microsoft Community

rich from nc say:

Numbering rows in a Word Table

I have a table that has over a thousand rows.  Is there a way number each row automatically?  I am using MS Word 2007 and Windows 7 on an IBM compatible.  Thanks!

 

Rich from NC

greg maxey say:

Rich,

 

If you execute this macro each time you add or remove a row it will renumber them for you:

Sub ScratchMacro()
 Dim oTbl As Word.Table, oRow As Row, oCell As Cell
 Dim oRng As Range
   Set oTbl = Selection.Tables(1)
   For Each oRow In oTbl.Rows
     Set oCell = oRow.Cells(1)
     Set oRng = oCell.Range
     If oRng.Fields.Count > 0 Then
       If oRng.Fields(1).Type = wdFieldSequence Then
         oRng.Fields(1).Delete
         oRng.Collapse wdCollapseStart
         oRng.MoveEnd wdCharacter, 2
         oRng.Delete
       End If
     End If
     oRng.Collapse wdCollapseStart
     oRng.InsertAfter ". "
     oRng.Collapse wdCollapseStart
     oRng.Fields.Add oRng, wdFieldSequence, "Numbered"
   Next oRow
 End Sub
