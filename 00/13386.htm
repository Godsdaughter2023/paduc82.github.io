Your description suggests you may have embedded the document's contents inside a field. Easily-enough done by accidentally pressing Ctrl-A, Ctrl-F9 instead of Ctrl-A, Shift-F9 to update fields, for example.

rich007a say:

Hi,

OK, here's what you need to do.


In Excel, make a new Workbook and save it, then close it.

 

Next, right-click on the ribbon and Customize Ribbon.  At the bottom right click Import/Export > Export All Customizations.
 

Save the file to your desktop.  Open the file in Notepad.

 

The code you will see will include all the controls on your QAT as well as your custom tab.
First you will see:

 

    <mso:cmd app="Excel" dt="1" /><mso:customUI xmlns:mso="http://schemas.microsoft.com/office/2009/07/customui"><mso:ribbon>

 

then you will see

 

    <mso:qat>

 

and a load of code representing your quick access toolbar buttons.  Look for the bit of code:

 

    </mso:qat>

 

(note the slash at the start of that).  Ignore all the code up to that point.  The next bit of code should start:

 

    <mso:tabs>

 

copy the text BETWEEN BUT NOT INCLUDING <mso:tabs>.....</mso:tabs>

The text you copy should start <mso:tab ...    and end with     ...</mso:tab>

 

Paste that text into a new notepad (close the original exported file without making any changes).  Do a find and replace to find all "mso:" and replace with nothing (i.e. we are removing all isntances of "mso:").

 

Now copy that text.

 

Open the Custom UI Editor.  In the Custom UI Editor, open your workbook that you saved earlier.

Do: Insert > Sample > Custom Tab.
The following code should appear:


<customUI xmlns="http://schemas.microsoft.com/office/2009/07/customui">
 <ribbon startFromScratch="false">
  <tabs>
   <tab id="customTab" label="Custom Tab">
    <group id="customGroup" label="Custom Group">
     <button id="customButton" label="Custom Button" imageMso="HappyFace" size="large" onAction="Callback" />
    </group>
   </tab>
  </tabs>
 </ribbon>
</customUI>

 
Replace the <tab....  </tab> (highlighted bold above) with the text you copied earlier.

 

Click the validate button on the toolbar.  If all is ok, save the file and close the Custom UI Editor. 

 

Now switch back to Excel.  Your custom tab that appears in all workbooks will need to be removed (right-click on it in the Customize Ribbon dialog > Remove).  You'll be able to bring it back later if you need to by importing the file that you exported previously.

 

Now open your workbook in Excel.  You should now only see your custom tab when that workbook has focus.

 

You may find that you get error messages either when you validate the xml in the Custom UI Editor, or when you first open your workbook in Excel.  Make a note of those error messages and post your xml code here if you need more help.

 

Hope that helps.
Cheers
Rich
