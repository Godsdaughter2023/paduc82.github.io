greg maxey say:

You could use an array.  BTW font red appears in two places in your example "idi" and "Dre" and these instances are not in the list of intances tht you enumerate:

Sub FRUsingAnArray2()
Dim SearchArray As Variant
Dim myRange As Range
Dim i As Long
Dim TermString As String
SearchArray = Array("im", "oo", "th")
Set myRange = ActiveDocument.Range
For i = 0 To UBound(SearchArray)
  TermString = SearchArray(i)
    With myRange.Find
      .Text = TermString
      .Replacement.Font.Color = wdColorRed
      .Replacement.Text = TermString
      .MatchWholeWord = False
      .Execute Replace:=wdReplaceAll
    End With
Next
End Sub

graham mayor say:

As there are likely to be more items and more colours, I would suggest an alternative approach.
Create a two column table with the strings in the first column and the colours in the second column e.g.

tri

Red

trí

Red

ert

Red

ért

Red

tri

Red

tré

Red

ch

Green
etc

Save the document and put its name and path in the following macro where indicated. Note that the macro caters for only four colours. If you want more you would need to add appropriate case statements to reflect the available colours from the Word colour palette.

Sub FontColoring()
Dim oChanges As Document, oDoc As Document
Dim oTable As Table
Dim oRng As Range
Dim oColor As Range
Dim sColor As String
Dim rFindText As Range
Dim i As Long
Dim sFname As String
On Error GoTo CleanUp
'***************************************
sFname = "D:\My Documents\Test\Changes.doc"
'***************************************
Set oDoc = ActiveDocument
Set oChanges = Documents.Open(FileName:=sFname, Visible:=False)
Set oTable = oChanges.Tables(1)
For i = 1 To oTable.Rows.Count
    Set oRng = oDoc.Range
    Set rFindText = oTable.Cell(i, 1).Range
    Set oColor = oTable.Cell(i, 2).Range
    rFindText.End = rFindText.End - 1
    oColor.End = oColor.End - 1
    With oRng.Find
            .ClearFormatting
            .Replacement.ClearFormatting
            Do While .Execute(findText:=rFindText, _
                MatchWholeWord:=False) = True
                Select Case oColor.Text
                    Case Is = "Red": oRng.Font.Color = wdColorRed
                    Case Is = "Blue": oRng.Font.Color = wdColorBlue
                    Case Is = "Green": oRng.Font.Color = wdColorGreen
                    Case Is = "Yellow": oRng.Font.Color = wdColorYellow
                End Select
            Loop
    End With
Next i
CleanUp:
oChanges.Close wdDoNotSaveChanges
End Sub

davidf2 say:

Backup you backup...the odds of you having problems with 120 pages of images is high. Newer versions of Publisher especially have problems with high resolution pictures and/or too many pictures. If you have access to Pub 2007 then use it as it still had the 'fast resize and zoom' option for rendering the pictures in your publication. For that matter if you had success with the MAC then I would keep doing what you are doing. If ain't broke, don't fix it.

Some things you might try to minimize the chances of problems:

If all your pictures are going to be the same size then using a third party image editor resize and optimize the images before you insert them into the Publisher page. Do some test pages to determine what minimum resolution you need. It will probably be no more than 180 dpi for a  local printer and certainly no more than 300 dpi even if you send it to a commercial printer to print. Make sure you keep a copy of the original pictures in case you need to start over. Probably use PNG format instead of JPG or TIF.

Consider linking your images to the picture boxes rather than embedding them. This will help reduce your overall file size. Keep all your presized and optimized images in a separate folder from the originals. Again you really need to protect the original images in case of problems. Once a Pub file goes corrupt you will NOT be able to salvage any images from those files!

As you had a way of working with individual pages using the MAC, then consider the same using Publisher; create multiple Publisher files. Combining Pub files is a pain but you can use Primopdf , a freebie PDF tool to combine them by printing them to the PDF driver; it has the ability to append a PDF file. If you are going to have a commercial printer print the final document then consult with them in advance to see what file format they want.

Check in the Word forum whether it would be possible to produce this publication using Word. In general Word is better for books and other long documents and might be a better choice than Publisher.

Good luck.

DavidF
