n95guru say: Instructions for a Nokia mobile phone user using Nokia Map Loader to download maps, which produces the following error message "Connection to the storage medium has been interrupted. Attach your storage medium and retry."

The interruption connection problem is caused by another application on your PC - ATI Catalyst Control Center, which requires uninstalling for the Map Loader to work.

It took me 3 days of searching the Internet forums to find this solution –thanks to a user called ‘Galazar’ I thought it would be a good idea to post a complete instruction list to help everyone out.

Instructions:

1 Right-click the ATI symbol on the System Tray next to the digital clock. Choose ‘Exit’ to exit the ATI Catalyst Control Center programme from your PC memory.
2 Control Panel –Click on Programs & Features (Classic View) or just Programs (std view)
3 Click on ATI Catalyst Install Manager then ‘Change’ button in between ‘Views’ and ‘Repair’.
4 Click ‘Next’ button to continue.
5 Click on ‘Uninstall Manager’ radio button and then ‘Next’ button.
6 On the Welcome screen choose ‘Custom’ radio button then ‘Next’ button.
7 On next screen ‘Component Selection’, click on tick-box for ‘ATI Catalyst Control Center’ then click on ‘Next’ button.
8 Once the uninstall has completed and the application exited I recommend your restarting Windows to completely remove it from the System Registry (automatically).

Ok, instructions on how to update your phone. Read this first then attempt!

1 First you’ll need to backup your phone using Nokia PC Suite. The latest version of this software (v6.84) is available here –
http://mea.nokia.com/A4346277
2 Download and install the software. Connect your phone via USB and choose ‘PC Suite’ on the phone. Backup your phone memory as well as the removable card memory.
3 Once completed, exit PC Suite and disconnect your phone USB cable.
4 Next, you’ll need to update your phone to the latest firmware. To do this you’ll need to download the latest version of Nokia Software Updater (v12.0.0.49974) available here -
http://nds1.nokia.com/files/support/global/phones/software/NokiaSoftwareUpdaterSetup_en.exe
5 Install the software.
6 Connect your phone via USB and choose ‘Mass Storage’.
7 Run Nokia Software Updater and update your phone firmware. For example, a Nokia N95 updates to v12.0.013. As this wipes your phone’s memory it’s vital you follow step 1 first to backup you phone!
8 Once completed your phone will reboot and should return to Standby mode.
9 Disconnect your phone from USB.
10 Next, you’ll need to install Nokia Map Loader. The latest version of this software (v1.2.1) is available here –
http://europe.nokia.com/A4509291
11 Install the software.
12 Connect your phone via USB and choose ‘Mass Storage’.
13 Run Nokia Map Loader and choose the maps you require. Note the arrows are used to expand the selection (eg Europe) to show the individual countries. Likewise, Expanding ‘United Kingdom’ then shows ‘England’, ‘Northern Ireland’, ‘Scotland’ & ‘Wales’. Use the + key to select, then ‘Download’ to install.
14 The maps are installed and spread over many subfolders of the hidden folder on your memory card –
Private/20001f63/diskcache

That’s it! Good luck.

The ability to manipulate the content of merge fields is very limited.  

As a result, you may need to use the data source with a Directory type mail merge main document in which you have the fields inside the cells of a one row table.  When you execute that merge to a new document, that document will include a row of data for each record in the data source.  You would then be able to manipulate the data in the particular field - one way to do that would be with a Wildcard replace.

a. user say:

<<
I have no access to the data source 

>>

What is the data source (Excel, Access, etc.) ? You may not have the ability to modify the data source, but in cases where Word connects via a dialect of SQL that can do substrings, you may be able to get what you need using a VBA OpenDataSource call.


e.g. for an Excel file where the data is in Sheet1 and the relevant field is called myfield, you might be able to use something like this

ActiveDocument.OpenDataSource _
  Name:="your xls/xlsx path and name", _
  SQLStatement:=_

    " SELECT s.*, 

    sqlstatement:= _
      " SELECT s.*, instr(s.myfield,'(') as x1, instrrev(s.myfield,')') as x2, iif(x1=0,'',mid(s.myfield,x1+1,x2-1-x1)) as mytext" & _
      " FROM [Sheet1$] s"

then use the merge field "mytext" 

In fact, once you have run the VBA, the Word document remembers the SQL so in fact you would not necessarily have to distribute the VBA. But there are also constraints...

a. user say:

I read your original post and skimmed the thread.

Keep in mind Windows 10's new "hybrid/fast boot" facility can shut the file system down dirty, as it is NOT a true shutdown, but something akin to what used to be called "Hibernation".

Rebooting the system (vs. shutting it down) does NOT exercise the hybrid/fast boot facility, but DOES execute a true shutdown.  Hence argmarovski's suggestion above.

Reconfigure your Windows 10 system to disable the feature, and I'll bet the problems go away entirely, whether you shut down or reboot.

I don't have specific instructions for Windows 10, but it should be similar to that for Windows 8.

http://superuser.com/questions/495240/how-to-instruct-windows-8-not-to-perform-a-fast-shutdown
