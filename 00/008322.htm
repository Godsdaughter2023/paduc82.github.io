myall_blues say:

Make the formula in C1:

=((($B1*C$7)*$D$10)+($D$11*C$7)) and copy it down/across.

Adding a $ symbol to a cell reference 'locks' it, changing it from a relative to an absolute cell reference.

In general terms when copying/filling formulas:

A1: Both the column and row will change

$A1: The column is locked, the row changes

A$1: The column changes, the row is locked

$A$1: Neither the column nor row change.

Macro to find text string in attachments

graham mayor say:

It should be possible if your copy of Word does not have the file types blocked in the security settings and there are no autorunning macros in the DOC format files, though I haven't tested it. The main macro needs to be changed as follows. (The Functions are unchanged).

Sub MoveMessages()
Const strFindText As String = "10000 SMS- HS"        'The exact search string
Const strFileType As String = "doc|docx|rtf"        'The document type
Const strPath As String = "C:\Outlook Attachments\"        'The root folder
Dim vFileType As Variant
Dim strFilename As String
Dim strName As String
Dim wdApp As Object
Dim wdDoc As Object
Dim olItems As Outlook.Items
Dim olItem As Outlook.MailItem
Dim olAttach As Outlook.Attachment
Dim strFolder As String
Dim bStarted As Boolean
Dim bFound As Boolean
Dim i As Long, i_V As Long
    On Error Resume Next
    Set wdApp = GetObject(, "Word.Application")
    If Err Then
        Set wdApp = CreateObject("Word.Application")
        bStarted = True
    End If
    On Error GoTo 0
    wdApp.Visible = True

    If Not FolderExists(strPath & strFindText) Then
        CreateFolders strPath & strFindText
    End If

    Set olItems = Session.GetDefaultFolder(olFolderInbox).Items
    For i = olItems.Count To 1 Step -1
        Set olItem = olItems(i)
        If olItem.Attachments.Count > 0 Then
            vFileType = Split(strFileType, "|")
            For Each olAttach In olItem.Attachments
                For i_V = 0 To UBound(vFileType)
                    If Right(LCase(olAttach.Filename), Len(vFileType(i_V))) = vFileType(i_V) Then
                        strFilename = strPath & Format(olItem.ReceivedTime, "yyyymmdd-HHMMSS") & _
                                      Chr(32) & olAttach.Filename
                        olAttach.SaveAsFile strFilename
                        Set wdDoc = wdApp.Documents.Open(strFilename)
                        With wdDoc.Content.Find
                            bFound = False
                            Do While .Execute(strFindText)
                                bFound = True
                                Exit Do
                            Loop
                            strName = wdDoc.Name
                            wdDoc.Close 0
                            If bFound Then
                                Name strFilename As strPath & strFindText & Chr(92) & strName
                            End If
                        End With
                    End If
                Next i_V
            Next olAttach
        End If
    Next i
    If bStarted Then wdApp.Quit
    Set wdDoc = Nothing
    Set wdApp = Nothing
    Set olItem = Nothing
    Set olItems = Nothing
End Sub

Joe,Frank,Bill,Bob,Tom,Mary,Julie
will not wrap, but

will wrap because a space appears before "Bob"

If you have your account configured as POP3 one system without the "Leave Messages on Server" option checked, the Inbox will be emptied anytime that system connects to the account - having it configured as an Imap account on a different system is irrelevant in that scenario.

Re: "When I set up my email it pull ALL  of my emails off of webmail.  And, it gives me NO option to leave the mails on the server! " - there's a contradiction in there somewhere.......

If the account is configured as POP3 - the option to leave messages on server is located in the Account Settings for the POP account

--> File --> Account Settings --> Email Tab --> highlight the account --> Change

     --> click the <More Settings> button --> Advanced tab

The contradiction is in the "it pulls all email off of webmail" - are you actually saying that your WebMail inbox is empty after the emails are downloaded? If that's the case, you likely have a POP account but if that were the case, you would also have the option to keep things on the server described above.

An Imap account would have no such option as Palcouk has already described BUT if you are operating under the original scenario with multiple devices accessing this account - certainly something that could occur. If you've successfully downloaded all your email to connecting via IMAP <AND THEN> access the account via POP3 afterwards which is set to remove all items from the server, the next time you access this account via Imap - everything in the IMAP OST data file will be deleted since it will reflect the current status of the <Inbox> contents.

albert d. kallal say:

I would suggest you create the IncSEQ column as a number type.

While Access does a good job of “auto casting” data types, web based is far more sensitive to “mixing” of data types such as number and strings.

I suspect the condition:

[Counter].[IncDate]= Year(Now())

Is failing.

IncDate is text type, and year() will return a number. This likely works client side, but not server side.

I would suggest making both IncDate, and incSEQ number data types.

Of couse the above many “well” cause issues when you attempt ot use the columns in string expressions . If memory serves me correct, then at form level (UI macro) you can convert the int number to a string by using say left$(<expr>, 999).

So I would get the two columns working as number types. and "then" work on using the columns in expresions - you may well need the left$() trick to convert the numbers to stirngs before they can be used in expressions.

jay freedman say:

Also, since attachments were mentioned, it's worth noting that when you "open" an attachment directly in Word, the email program creates a temporary file and opens that. Possibly one of those is the temporary file causing the problem.

It's much safer to use the Save As command to save the attachment as a regular document file, and then open that file normally in Word. This is especially true if you expect to edit the document, because the edits get saved in the temporary file, which will then be deleted automatically. See http://www.gmayor.com/outlook_attachments.htm.
