PROBLEM WITH THE LOOKUP WIZARD IN ACCESS - Microsoft Community

hawra al-khaz'aly say:

PROBLEM WITH THE LOOKUP WIZARD IN ACCESS

Hello,

So I used the Lookup Wizard for this, and now the problem is that what only shows in the drop-down menu are the numbers following L and not L itself. In case you might need to know, I did enable data integrity when using the Lookup Wizard.

So, my question: is there any way that I can keep the L there in Table1 so that I don't have to rewrite it, but still have the L in the drop-down menu along with the numbers following it in Table2?

Thank you,

john w. vinson say:

If the L is JUST for display purposes, don't put it in the field at all: just use a Format to display it.

And most developers prefer to avoid the Lookup Field type altogether; it's of very little benefit, and has many problems (like the one you're seeing). See the Access Web critique at http://www.mvps.org/access/lookupfields.htm for a discussion.

ken sheridan say: The "L" will not be included in the actual values in the table, as you have found.

As regards the column (field) in a referencing table, as the link which John has given you describes, the 'lookup field' wizard should be avoided like the plague.  You can achieve the same result by using a combo box in a bound form (which is the only place data should be entered, not in a table's datasheet), setting up the control as follows.  I've called the column in question MyCode for this example:

ControlSource:   MyCode

RowSource:     SELECT [MyCode], "L" & [MyCode]  FROM [Table1] ORDER BY [MyCode];

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero.

By concatenating the "L" to the value of the column in the second column of the RowSource property this is what you'll see it in the list, but the value of the control, and hence the value in the table, will be that of the first column, which is hidden by setting the ColumnWidths property to zero.

Whenever you need to see the leading "L". e.g. in the result table of a query or in a report, you would simply concatenate it to the value in a computed column in a query, or a computed text box control in a report.
