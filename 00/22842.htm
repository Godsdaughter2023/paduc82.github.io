Have installed Office 2019 (Professional Plus) - previously had Office 2016 but uninstalled.

Every time I close Outlook, it prompts to save NormalEmail.dotm (wants to save to my user "Documents" folder).  The file is already there so no idea why it's prompting to save every time on close.   I don't have any special add-ins I think.

I notice Word is doing similar - when I open it with no document, create New, then close again, it is prompting to save Normal.dotm (in Documents folder)  for some reason.

Any suggestions how to stop this?

wrd313 say:

I think I figured out ... I installed Office 2019 with admin account privileges (of course), but then I use a different standard account for launching Office applications.  I think as a standard user, both Outlook and Word were looking for normalemail/normal.dotm  in the location %appdata%/Microsoft/Templates, but neither file was there.  So every time I closed Outlook or Word - it prompted to save the template - except it prompted to save in my Documents folder - not the %appdata%/Microsoft/Templates folder.

Once I figured this out, I let them save each in the standard user Templates folder ... and now they no longer prompt to save each time they are closed.

sandy johnson say:

I don't have Visio installed on this computer to test, but maybe you can try the following:

Copy the Visio file, then Paste As a Windows Media File onto your PowerPoint slide. Then Ungroup the WMF until you get all of the individual shapes. Delete the artifacts that result (blank boxes, usually the size of the full image). Each shape might also be surrounded by an extra line, making the shapes thicker than the original, so change the shapes' properties to No Outline.

Anyway, now you should be left with individual shapes that can be Grouped and Animated.

Note that depending on the complexity of your Visio file, the number of shapes can be overwhelming and difficult to work with, so this approach may not be perfect.
