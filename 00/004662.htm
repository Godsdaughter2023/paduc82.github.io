bob jones aka: cybertaz say:

Phillip is correct that the 'document' is Protected, but you might first try going to the Tools menu to see whether Unprotect Document is an available command. You still may be prompted for a Password to turn the Protection off.

With Protection removed, what type of editing you can do further depends on exactly what types of controls have been used in the document. Word for Windows provides some that are not supported in the Mac version. However....

The further question is "Is this file a document (.docx, .docm) or is it a template (.dotx, .dotm)?"... it isn't clear because you appear to use the terms interchangeably. The 2 terms have specific, separate & distinct meanings in Word as well as most other programs.

If it is a template, its entirely possible that you are not intended to open & modify the template, itself. A template is a 'cookie cutter' that stamps out new documents based on the design & content of the template file. It isn't intended to be opened [File> Open] & edited as a matter of routine. Usually, the creator of a template will protect the template for exactly that reason -- to prevent the template from being altered, so that additional new documents generated from it start off as what the template is intended to produce.

When new documents are generated from a protected template, you still may be able to edit the areas intended to be edited. In fact, certain areas may be completed automatically when the new document is generated. Also, when you Close or Save the document the Save As dialog will open so that the file name & location for saving the new document can be specified. The template file, itself, will not be affected by the changes you made.

To create a new document from a template, either;

  • Double-click the template file icon in a Finder window, or
  • Move the file to your My Templates folder [~:UserName:Library:Application Support:Microsoft:Office:User Templates] then use File> New from Template rather than File> Open.

ruwim.b say:

The option to re-enable contact delete confirmation pop-up is not available in Skype interface, but you can enable this pop-up by editing your config.xml file.

 

Make sure that Skype is not running (Quit Skype).

Go to Windows Start and in the Search/Run box type %appdata%\skype and then press Enter or OK button. Windows File Explorer will pop up. Locate now the folder that is named as your Skype Name. Open this folder. You will find there the config.xml file. You may see only config but with XML Document in the Type column.

Open the config.xml in any text editor (e.g. Notepad). Find there a section starting with <General>. Within this section find the line <ConfirmRemoveBuddy>1</ConfirmRemoveBuddy>. Change the value 1 to 0. This will restore the Confirmation option. Save the file and restart Skype.

j.e. mcgimpsey say: If your file(s) were saved with the Read Only option set (done by choosing File/Save As/Options/ and checking the Read-only recommended checkbox), then you'll need to re-save the file using File/Save As/Options/ and unchecking the checkbox.

Otherwise, if you're getting normally saved files coming up as read-only, try, with all other apps closed, running Disk Utility to repair your disk permissions.
