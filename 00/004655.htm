daniel pineault say:

Yes, this is a common annoyance/problem.  I too never have as many problems occur, as with the newer version of Access.  Previously, the solution below wasn't necessary, but now it is.

 

You'd have to switch your code to use word automation and then integrate code to bring the word object to the foreground before performing the mailmerge.  See: http://www.everythingaccess.com/tutorials.asp?ID=Bring-an-external-application-window-to-the-foreground

 

 

Here is a simple example to get you going:

 

Function test()
    Dim oApp         As Object
    Dim oDoc         As Object
    Dim LWordDoc  As String
    
    LWordDoc = "MailMergeMain.docx"

    On Error Resume Next
    Set oApp = GetObject(, "Word.Application")
    If Err.Number <> 0 Then    'Word isn't running so start it
        Set oApp = CreateObject("Word.Application")
    End If
    On Error GoTo 0

    Set oDoc = oApp.Documents.Open(sPath & LWordDoc)
    oApp.Visible = True

    Call FnFindWindowLike("*" & LWordDoc & "*")
   
    'continue your code here for the mailmerge should you wish to automate this further
   
    Set oDoc = Nothing
    Set oApp = Nothing
End Function

scottgem say:

First, you don't have fields on a form, you have Controls that may or may not be bound to fields in a table. This is a subtle but important distinction. 

The next question is are all these Yes/No fields represented by checkbox controls on your form? it is much more difficult to tell the datatype of the field bound to a control then it is to determine the type of control.

farey say: Its all checkboxes. And i filter this subform based on cbo on mainform. So basically what i want achieve is when i filter the subform, i want show all controls (checkboxes) that has value "true". 

scottgem say:

Well first let me say that a table with a lot of Yes/No fields is likely not normalized properly. Given what you want, your subform would be better as a continuous form with Comboboxes to select a value. For example: Let's say you have a Resort hotel app. And you want to indicate the amenities for each hotel. Have a table of Hotel Amenities with a subform of comboxes.

then you don't have to worry about hiding the unchecked boxes, the form will only show those values selected.
