How to close all OneNote Notebooks in Office 2016 - Microsoft Community

itxpress say:

How to close all OneNote Notebooks in Office 2016

If you have the need to close all open OneNote notebooks (because by default they never 'close' they stay open in the background), there is way! Because Microsoft removed safeboot as an option for OneNote, then they changed the cache directory behavior so if you deleted it, it would reopen ALL the notebooks that were open (instead of starting over), it was impossible to close all notebooks that were open and syncing in the background. 

We have a client whose employees create a notebook in a shared folder for every customer they have to track things. But they would just keep opening notebooks and we got into a situation where we had employees with 800+ notebooks open in the background because they never knew how to 'close' a notebook all the way (you have to right click the notebook in the pulldown and select 'Close this notebook' or add a Close This Notebook button in the ribbon (right click ribbon, select Customize Ribbon, add the Close This Notebook from 'All Commands' to an existing or new group in a menu). When they got too many notebooks open, OneNote would start chewing up 10-20% of the terminal server CPU PER USER. It was insane. And when it got like this - closing a notebook took forever so they didn't want to do it (one person spent literally six hours doing nothing but closing notebooks on an otherwise idle terminal server just trying to get out of the Catch-22). 

I've spent close to two years trying to find a way to mass close notebooks for the few employees still in this situation as MS has made it harder and harder to do so. I guess I understand why - because this CAN result in data loss.

So let me be VERY clear - this CAN result in data loss. If you have changes made to a shared notebook that are NOT synced back to the shared folder when you do this procedure - the changes WILL be lost. OneNote tends to be pretty aggressive in syncing changes from current notebooks, but you may want to make sure OneNote has sat open for a while to get all changes from the local cache synced out to the shared notebook folder. With OneNote overloaded, syncs may take much longer - so better to be safe than sorry - give it time over a weekend to sit open and sync!

Now to mass close the notebooks...

1) Close OneNote

2) Delete the OneNote cache directory for the user - C:\Users\userid\AppData\Local\Microsoft\OneNote\16.0\cache (16.0 is Office 2016. Might be different if office is older)

3) Open regedit and browse to the following key:

HKEY\CURRENT_USER\Software\Microsoft\Office\16.0\OneNote\OpenNotebooks

Select the '1' key and then scroll down to the bottom of the list and Shift-select the last key to select all but the '(Default)' key. Delete the entries. Again 16.0 is for Office 2016 - yours may be lower/older version of Office

4) Re-open OneNote and no notebooks should reopen. 

Hope this helps someone else as well! I've spent far too long trying to find a way to do this once safeboot and the cache delete no longer worked to accomplish this.

It is certainly possible to automate this as well since you can access all user's OpenNotebooks entry via HKEY_USERS, but again I'd caution against that given the potential for data to be lost. We've finally gotten the employees in the habit of using the 'Close This Notebook' button we put on their ribbons and only needed this process for the handful of users facing days of mindless right click, close, wait...

But I'm getting Object doesn't support that.

graham mayor say:

Your syntax is wrong. The following should work and checks if there is something selected to print. If there is a Word document to print from then you need to get the running Word object not create a new one. For all parameters prefixed wd e.g. wdPrintSelection, you need to use the numeric equivalent when using Late Binding to the Word object.

Sub PrintWordSelection()
Dim wd As Object
    On Error Resume Next
    Set wd = GetObject(, "Word.Application")
    If Err Then
        MsgBox "Word is not running"
        GoTo lbl_Exit
    End If
    If Len(wd.Selection.Range) > 0 Then
        wd.PrintOut _
                Filename:="", Range:=1, _
                Item:=7, Copies:=1, Pages:="", PageType:=0, _
                Collate:=True, Background:=False, PrintToFile:=False, _
                PrintZoomColumn:=0, PrintZoomRow:=0, PrintZoomPaperWidth:=0, _
                PrintZoomPaperHeight:=0
    Else
        MsgBox "Nothing selected"
    End If
lbl_Exit:
    Exit Sub
End Sub
