macropod say:

You could find potential adjectives with code like:

Sub Demo()
Dim wdSynInfo As SynonymInfo, wdSynList As Variant
Dim StrTxt As String, StrOut As String, StrTmp As String
Dim i As Long, j As Long
StrTxt = " ": StrOut = "Potential Adjectives:"
With ActiveDocument.Range
  For i = 1 To .Words.Count
    StrTmp = Trim(.Words(i))
    If InStr(1, StrTxt, " " & StrTmp & " ", vbTextCompare) = 0 Then
      StrTxt = StrTxt & StrTmp & " "
    End If
  Next
  StrTxt = Trim(StrTxt)
  For i = 0 To UBound(Split(StrTxt, " "))
    StrTmp = Split(StrTxt, " ")(i)
    Set wdSynInfo = SynonymInfo(Word:=StrTmp, LanguageID:=wdEnglishUS)
    If wdSynInfo.MeaningCount <> 0 Then
      wdSynList = wdSynInfo.PartOfSpeechList
      For j = 1 To UBound(wdSynList)
        If wdSynList(j) = wdAdjective Then
          StrOut = StrOut & Chr(11) & StrTmp
          Exit For
        End If
      Next
    End If
  Next
  .InsertAfter vbCr & StrOut
End With
End Sub

The above macro outputs a list of potential adjectives at the end of the document.

Once you start analysing the output, you'll find the code demonstrates a fundamental problem; many words exist in more than one part of speech. The part of speech a word belongs to depends on its context and there is no definitive solution apart from one that employs contextual analysis - which you can't do with a macro.

mr don keebles say:

I have had a similar problem with Word 2008 over the last couple of years (I haven't used Word 2011), but I've found a satisfactory workaround.


The problem: when editing with tracked changes, Activity Monitor shows Real Memory usage starts at the usual 180mb or so and gradually increases with each change tracked to 1.5gb - 1.7gb before Word becomes unresponsive/crashes. Docs may be corrupted but can usually be reopened. I have just installed OS 10.8.2 and the latest update of Word 2008 on a clean hard drive and this problem still occurs, and it is reproducible on both my 2.5 GHz intel mini with 4gb ram and my iBook (1GHz PowerPC with 1.25gb of ram running OS 10.5).


The solution that works for me is to go to Preferences, select the Track Changes pref pane, and then uncheck the "Include author, time stamp and action buttons" check-box. Real Memory usage no longer increases with each change, and Word is accordingly much more stable. The information on the author and time of changes is still collected; you just need to hover over a balloon to see it. If required, you can reselect the checkbox at the end to display the colour-coded time-stamped balloons while reviewing changes.


There may be another variable at work here (I also turn autosave off), but I'd be interested to hear if this solves the problem for anyone else.

=INDEX(B:B,MATCH(C1,A:A,0)+1)

To get the first name you can use this as an alternative to the VLOOKUP that you already have:

=INDEX(B:B,MATCH(C1,A:A,0))

aaron zhu msft say:

Hi Andy,

Based on my personal experience, Exchange Koisk Plan doesn’t support Outlook Anyway(MAPI), it supports POP way for the Outlook client. It’s not feasible to configure the Exchange Koisk accounts onto the Outlook client using Exchange way(Autodiscover). As a result, your scenario should be an expected behavior.

Additionally, if you want to access the shared mailboxes in your Outlook client using the Exchange way and in your iOS device as user mailboxes, you need to assign at least Office 365 Business Premium licenses to them.

However, from my personal experience, if your goal is to access the shared mailboxes on your Outlook client and iOS devices, you can assign the Full Access permission of the shared mailboxes to your own mailboxes, for the Outlook client, the shared mailboxes will be configured under your main mailboxes automatically due to Auto-Mapping, for the iOS devices, you can add the shared mailboxes in OWA(Outlook Web App) and then access them using the application OWA for iOS. 

Feel free to post back for any further concern.
