Footnotes aren't continuous - Microsoft Community

michaelmovn say:

Footnotes aren't continuous

So I received an article from someone to edit, with footnotes already included (starting from the 2nd page on). I want to add a footnote on the 1st page, but when I do so, the 1st footnote has the number "1", but the 2nd footnote (originally the 1st) still shows "1" next to it. I went to "Insert Footnotes", then "Apply to entire document", but nothing changed. Does this mean the previous writer put a page break between the 1st and 2nd pages?
john mcghie say:

Track Changes usually causes the footnote numbering to be "wrong", not to "restart".  If you have tracked deletions in the sequence, then the remaining footnotes are numbered too high, because the tracked deletions have not yet been physically removed from the file.  In which case, the numbering will come right when you accept all changes.

I don't think that's your problem here.

The footnotes are a "list".  Either one list on each page, or one list for the entire document.

If they are set to "Restart Each Page" and you add a new one to a page that doesn't have any, then you will indeed get a note numbered "1".  But in that case, the original footnote on that page will now be numbered "2".

What I suspect has happened is that someone has opened that file in one of the other applications out there that impersonate Word, such as LibreOffice.  That will have converted the footnotes to text, so you will have to re-insert them ALL.

Immediately beside each old footnote, insert a new footnote.  Copy the text of the old footnote to the new footnote, then delete the old.  When you get to the end of the document, your numbering should be correct (if it isn't, you've missed a few...)

Hope this helps

marshall_morris say: Not sure if you ever got your answer but if I'm reading this right you're actually describing the width of the slider on the horizontal scroll bar.  Normally it is the width of the first to last columns of data, but sometimes when it gets reset it narrows.  When you slide the narrow slider, it jumps dozens of columns at a time instead of small increments.

The fix for me was as follows - in Excel 2010, navigate under the Microsoft logo to Options > Advanced.  Now scroll down the right pane to "Display options for this workbook:"  and UNCHECK "Show Horizontal Scroll Bar."

Hit OK. At this point, I closed the document and reopened it before reactivating the scrollbar but it will probably work if you just save the document.

Now your horizontal scroll bar is hidden - just reactivate it by following the same steps above. Microsoft logo > Options > Advanced - RECHECK the "Show Horizontal Scroll Bar" and it should reappear, this time in the correct width.

If that didn't work, and you haven't already done so, close and reopen the document.
