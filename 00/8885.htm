fastfr3 say:

inserting clip art with mail merge

Am Trying to insert clip art on labels with mail merge. Clip art inserted into Excel does not show up as available field. Is this possible?
macropod say:

Hi fastfr3,

Your field code appears to be missing the mergefield! Depending on how you want the results to appear upon completion of the merge, you need to use coding along the lines of:
{INCLUDEPICTURE "C:\\users\\fred\\desktop\\icons\\«Image»" \d}
or:
{INCLUDEPICTURE {IF TRUE "C:\\users\\fred\\desktop\\icons\\«Image»"} \d}
or:
{IF {INCLUDEPICTURE {IF TRUE "C:\users\\fred\desktop\icons\«Image»"} \d} {INCLUDEPICTURE {IF TRUE "C:\users\fred\desktop\icons\«Image»"} \d}}

The first construction will require you to manually update the field display (Ctrl-A then F9) after the merge has executed and, if you don't want the images permanently linked to the files on your system, to then manually unlink them (Ctrl-A then Ctrl-Shift-F11).

The second construction automatically updates the field display when the merge has executes but, if you don't want the images permanently linked to the files on your system, you'll still have to manually unlink them (Ctrl-A then Ctrl-Shift-F11).

The third construction automatically updates the field display and unlinks the images when the merge has executes.

Cheers,

 It's the classic "Top N by Group" query, and you already have the classic
solution, so I don't have an immediate answer.  Google shows a number of
articles on optimizing this sort of query, but a quick scan of them suggests
that the first set at least are directed at SQL Server, and based on SQL
Server features that are not available in Access.  Still you might have a
look at them for ideas.
 
One possibility would be to create a temporary table to hold the desired
output, and loop through a recordset (opened on a sorted query) to populate
it.  Usually, recordset processing is much slower than straight-SQL
implementations, but maybe in this case it would be faster.  I don't know.
The idea would be to have an input and an output recordset open, and loop
through the input recordset, copying only the first three records for each
[Method Analyte Company id] to the output recordset.
 
As I said, that may well be slower than what you have now, but you might
give it a try to find out.

john p spencer say:

Is there any way you can pre-filter the Test2 records?   If your other queries are filtering their results are any of the criteria relevant to the Test2 query?  If so, you might try creating a query to filters the records as far as possible AND THEN use that query as a means to get the top 3 per [Method Analyte Company id].

For instance, if your report is only for companies whose names start with A, you might have something like

SELECT Test2.*
FROM Test2 INNER JOIN CompaniesTable
ON Test2.[Method Analyte Company id] = CompaniesTable.CompanyID
WHERE CompaniesTable.CompanyName Like "A*"

Now using that as the source for your TOP N query should be much faster.
