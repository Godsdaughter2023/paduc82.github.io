danielco say:

Use the following function to count colored cells. Note that, if you change the number of colored cells, the function will not update until the sheet is calculated.

The first argument is the considered range and the second one is the cell which contains the color sample :

Function ColoredCellsCount(rgRange As Range, c As Range)

Application.Volatile

For Each cell In rgRange

    If cell.Interior.Color = c.Interior.Color Then

        ColoredCellsCount = ColoredCellsCount + 1

    End If

Next cell

End Function

the following function adds merged cells :

Function MergedCellsCount(rgRange As Range)

Application.Volatile

For Each cell In rgRange

    If cell.MergeCells Then

        MergedCellsCount = MergedCellsCount + 1

    End If

Next cell

End Function

In Word for Mac, they put the Grid Options in a weird place: Preferences>Edit, click on the Grid Options button about halfway down. Uncheck To grid in Print Layout View. You can still snap to the grid if you hold down the Option/Alt or Command keys.

What will happen if install Access Runtime on Full Version? - Microsoft Community

a2010learner say:

What will happen if install Access Runtime on Full Version?

Hi Gurus,

 

If I have full Access 2010 installed on a PC, will I overwrite the full version if I install a 2010 Runtime?

 

Thanks,

albert d. kallal say:

Actually, in theory nothing will happen here.

 

The reason is runtime install routines will detect that you already have Access installed. And you only allowed to install one version of Access of the same release. So in fact what occurs is the runtime part is not installed at all.

 

If you in fact built the install with the package wizard, then your setup and install of your application will NOT install runtime, but will copy + setup your application and this almost always includes a shortcut to your application. That shortcut will (and always does to my knowledge) include a /runtime switch.

 

So your application will run like runtime, but in fact any installing of access is ignored and in fact does not occur if the full version is present.

 

So, in the nutshell? Nothing happens if you attempt to do this. 

 

And while one would not by intention do this, it is VERY common for this to occur. I mean if you send a packaged application on disk to a user then they will just pop in the disk and follow your instructions.  They are thus rather likely to just install this as you ask them to. However they may not realize that they already have the full version of Access installed.

 

So in effect your question is quite reasonable.

 

At the end of a day installing by accident or in a typical environments such an install attempt could and would be quite common. Just keep in mind thus the simple act of sending a packaged application to a "end user" will NEVER overwrite their full edition of Access and in fact the runtime install will LEAVE the full edition alone and forces YOUR application to use the full edition with /runtime switch.

 

So if the same edition of Access is already installed (full or runtime) then  your install of runtime is ignored. So if you send 10 packaged applications with each the runtime then again each additional install "detects" that runtime (or as noted full edition) is already installed and the runtime part of your package is ignored.
