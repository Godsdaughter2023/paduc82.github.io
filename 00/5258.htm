4 (optional): The type of .DMP files located in the Minidump folder are known as Small Memory Dumps. In %systemroot% there will be what is known as a Kernel Memory Dump (if your system is set to generate). It is labeled MEMORY.DMP. The difference between Small Memory Dumps and Kernel Memory Dumps in the simplest definition is a Kernel Memory Dump contains much more information at the time of the crash, therefore allowing further debugging of your issue. If your upload speed permits it, and you aren't going against any strict bandwidth and/or usage caps, etc, the Kernel Memory Dump is the best choice. Do note that Kernel Memory Dumps are much larger in size due to containing much more info, which is why I mentioned upload speed, etc.

What is the difference between MSE and Safety Scanner? - Microsoft Community

moferk say:

What is the difference between MSE and Safety Scanner?

Difference between the two softwares.
kosh vorlon say:

Hi Moferk,

 

The short answer is that MSE is a program you install on your computer that provides real-time protection against malware, viruses and spyware (and also does scheduled and manual scans) while Safety Scanner is on Online Scanning program that does not provide real-time protection, is not actually installed on your computer, and runs basically the same type of scans (but it considered by some to be just slightly more potent in terms of detection and removal of threats during those scans even though they are based on exactly the same definition database. 

 

It's not a matter of choosing one or the other.  If you need real-time protection on your computer, MSE is the choice.  MSE will also do a great job in terms of detecting and removing threats in real-time and during scans.  Some suggest if you suspect you're infected but MSE either isn't detecting it or is detecting it but can't remove it, that Safety Scanner might sometimes do the trick.  MSE is something that runs all the time on your computer.  Safety Scanner is something you do manually, if you choose to use it, every once in a while as a backup for MSE (or for any other primary AV program you are using).  Also, while you can have only one primary real-time AV program installed on the computer at the same time, you can use Safety Scanner in addition to another AV program and thus if you're using an AV program other than MSE, Safety Scanner allows you to scan using Microsoft's definition database when you would otherwise not be able to do that without uninstalling whatever you are using and installing MSE, then updating and running the MSE scan, then reversing that to get back to normal - making that a whole lot easier.  Personally, since I use MSE, I do not use Safety Scanner as a backup because while some say it may be slightly better at removal or even detection, that difference is not very significant and so I prefer a different Online Scanning program so I'm more likely to cover a different set of threats (and thus I use ESET Online Scanner for that).

 

For more details and far better explanations, please see the following:

 

For MSE, see: http://windows.microsoft.com/en-US/windows/products/security-essentials.

 

For Safety Scanner, see: http://www.microsoft.com/security/scanner/en-us/default.aspx.

 

I hope this helps.

 

Good luck!

How to select object by name-Visio 2007 Standard - Microsoft Community

jameszd say:

How to select object by name-Visio 2007 Standard

Have been trying to find a way to select objects by name so that I can automate connections in VBA based on data supplied from other departments.

I have been able to drop the object and rename it based on columns in a text file (hope to be able to use excel in next revision) but cannot find a way to select individual objects based on name in order to run a connectors subroutine.

jameszd say:

Finally found a method that will work to connect Visio shapes by name (syntax for the next guy that searches for this).

It's not elegant but it works. Any help or suggestions for efficiency, code cleanup, or layout would still be appreciated.

 


'' Select and connect shapes section
    Dim Fieldx As String
    Dim Fieldy As String
    Dim Fieldz As String
    Dim vConnector As Visio.Shape
    Dim vConnectorMaster As Visio.Master
    Dim vStencil As Visio.Document
    Set vStencil = Application.Documents.OpenEx("Basic Flowchart Shapes.vss", visOpenDocked)
    Set vConnectorMaster = vStencil.Masters("Dynamic Connector")
   
    Dim vsoCell1 As Visio.Cell
    Dim vsoCell2 As Visio.Cell
    Dim ShpConn01 As Visio.Shape
    Dim ShpConn02 As Visio.Shape
    Dim ShpConn03 As Visio.Shape
    Dim storedShape01 As Integer
    Dim storedShape02 As Integer
    Dim storedShape03 As Integer

Open "x:\<path>\<filename.txt" For Input As #1

Do While Not EOF(1) ' Loop until end of file.
Line Input #1, TextLine ' Read line into variable.
Field1 = Trim(Mid(TextLine, 1, 10)) ' -jd stencil shape/master name
Field2 = Trim(Mid(TextLine, 11, 10)) ' -jd Object's name
Field3 = Trim(Mid(TextLine, 21, 10)) ' -jd Object to connect to name

Fieldx = Field2
Fieldy = Field3
'Fieldz = Field4

ActiveWindow.DeselectAll
ActiveWindow.Select Application.ActiveWindow.Page.Shapes.ItemU(Fieldx), visSelect
Set ShpConn01 = Application.ActiveWindow.Page.Shapes.ItemU(Fieldx)
    Set vsoSelection = ActiveWindow.Selection
    Set vConnector = ActivePage.Drop(vConnectorMaster, 0, 0)
     Set vsoCell1 = vConnector.CellsU("BeginX")
     Set vsoCell2 = ShpConn01.CellsU("PinX")

        vsoCell1.GlueTo vsoCell2
            
ActiveWindow.Select Application.ActiveWindow.Page.Shapes.ItemU(Fieldy), visSelect
Set ShpConn02 = Application.ActiveWindow.Page.Shapes.ItemU(Fieldy)

    Set vsoCell1 = vConnector.CellsU("EndX")
    Set vsoCell2 = ShpConn02.CellsU("PinX")
       
        vsoCell1.GlueTo vsoCell2
       
     Set vsoCell1 = Nothing
Loop
Close #1
ActiveWindow.DeselectAll

Visio.Application.ScreenUpdating = True


' Call LayoutSubroutine - jd find a better one that will not let lines cross over each other.

End Sub
