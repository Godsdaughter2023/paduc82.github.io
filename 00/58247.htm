cmdrkeene say:

There's at least 5 places to check for timeout settings (don't get me started...).  Probably one of these just needs to be edited.  Probably the 1st one, since it will dump you to the login screen (just like you are seeing) even if you don't actually have any password.

  1. Right-click Desktop > Personalize > Screensaver
    Even if you don't use a screensaver, this timer value is used to lock the screen if the checkbox is on.  So, make sure the checkbox is offso it doesn't go to the lock screen.

    I see this happen dozens of times a week - someone has the screensaver set to "none", but they forgot to uncheck the box and so it keeps locking their screen every 1 minute.

  2. Power Options > Change when to turn off the display (set to never)

  3. Power Options > Change when the computer sleeps (set to never)

  4. PC Settings App > Lock Screen Settings (set to "never")

  5. PC Settings App > Accounts > Sign in options
    this one is harder -- you can't change the option if you are currently receiving email from a secure server like Office 365, school, work, university, or government accounts.  So for example, my computer forces me to always unlock it since I have a company email account.  Your administrator gets to decide this, not you.  Your only option otherwise is to stop using work/school email on that computer.  Example:

¡firedog say:

The errors you're seeing are the result of a corruption of at least the calendar database (DbTimeCorrupted). How to deal with it depends rather on how your friend uses the calendar.

  • If she uses the calendar and she usually signs in in Windows Live Mail, or
  • If she doesn't use the calendar at all 

then the first thing to try is to navigate to the Store folder (\AppData\Local\Microsoft\Windows Live Mail) and rename the Calendars folder to Agenda. This should cause a new Calendars folder to be created next time the program starts - assuming it will now start. If the user normally signs in, then a new instance of the account calendar will be downloaded at the next sign-in.

  • If she uses the calendar but she doesn't usually sign in in Windows Live Mail

things get a little more fraught and she may have to resign herself to the fact that the calendar is no good. There are procedures to try and mend the corruption, but it may not be worth the effort.
