OK now I see how to solve the problem ... although I'm not sure that I have the right explanation for the source.  Apparently, some tweak was made so that in Build 10122, the .NET Framework 3.5 cannot be installed with the Windows Live Essentials setup file.  However, if you go to Control Panel->All Control Panel Items and then click on the "Turn Windows features on or off" link, you get a list of options and the

.NET Framework 3.5 is at the top of the list.  If your select it and then close that window by clicking OK, then you are taken to the same "downloading" window that you get when you attempt to download

and install using the wlsetup-web.exe approach.  Only now it works, and after some minutes, you get

a message indicating that the installation was successful.

You then try the wlsetup-web.exe again, and this time, it takes you immediately to the window where you decide whether to install all components or just some.  I chose all, and the installation went as expected.

Now Windows Live Mail seems to run fine.

So the only issue is that the control panel approach has to be taken if you want to install Windows Live Essentials on build 10122.  Of course, this step is not necessary if you did the install in an earlier build and simply go to 10122 via an upgrade.
