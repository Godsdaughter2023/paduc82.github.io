hatifnatt say:

Yesterday I had exactly same issue, all custom names are lost.
Information about renames saved in SQLite 3 database
C:\Users\<YourUsename>\AppData\Roaming\Skype\<YourSkypeLogin>\main.db
and also synced to Skype servers, it's not too hard to edit database but I still can't find the way how to sync updated data to Skype servers. Also this database held almost all other Skype data, history, groups etc. so you can't just restore it from backup, you will loose all history, "Recent contacts" will reorder and so on.

You can extract all your lost custom names from this database with this SQL query

SELECT id, skypename, fullname, displayname, given_displayname FROM Contacts WHERE given_displayname <> '';

but first of all you need a copy of main.db file before data loss.

If you have backup you can save DB from it. If you don't have backup you can save old DB with "Previous Version" Windows feature.

Short info below if you had never been using this feature before.

  1. Open folder with main.db file
  2. Right click on main.db
  3. Click on "Restore previous versions"
  4. Choose version you need (before data lost)
  5. Click to "Copy..." and save file to your desired location.

More info about "Previous Version" here.

Now you can extract data with sqlite3 tools. Download binaries here https://sqlite.org/download.html extract somewhere, run

sqlite3.exe -header -csv  \location\of\main.db "SELECT id, skypename, fullname, displayname, given_displayname FROM Contacts WHERE given_displayname <> '';" > skypenames.csv

tom ogilvy say:

If you want to do a find or replace, there are built in Excel vba commands to do this.  You don't need to try to execute the dialog box or the ribbon.

for find as an example:

Sub Macro1()

Dim rng as Range

Set rng = Selection.Find(What:="B", _
 After:=ActiveCell, _
 LookIn:=xlFormulas, _
 LookAt:=xlPart, _
 SearchOrder:=xlByRows, _
 SearchDirection:=xlNext, _
 MatchCase:=False)

If not rng is nothing then

  rng.select

  msgbox "Found at " & rng.Address

End if

End Sub
