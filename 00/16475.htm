macropod say:

I tried a rich or plain text content control, which would have been idea as it can be made undeletable, but it didn't automatically update the cross reference, even when printing.  I had to ctrl+F9.

You could use content controls with bookmarks & cross-referencing, but you wouldn't want to make the content control undeletable; otherwise the bookmark will refuse to update after the first time you do so. For auto-updating of bookmarks at print time, all the users need do is ensure Word's 'update fields before printing' option is set. Alternatively, you could use a ContentControlOnExit macro to either: a) update cross-references; or b) populate subsidiary content control(s). With the latter approach, both sets of content controls can be protected against deletion and, with proper coding, against editing of the subsidiary content control(s).

jay freedman say:
If your "negation sign" looks like this   ¬   then it isn't ascii 170 (that's a superscript a, like this  ª ) but 172.

If searching with ^172 doesn't find them, there are two other possibilities. One is that you're seeing "conditional hyphens", the kind that you can insert with Ctrl+hyphen, but those should be visible only when you have displayed nonprinting characters. The other possibility is that they're enclosed in fields, for which you need the macro in http://www.word.mvps.org/FAQs/MacrosVBA/FindReplaceSymbols.htm.

a. user say:

<<
I found the original ascii character in a table saying it was 170 so obviously there are some incorrect tables around or else I am misunderstanding something, - not unusual!
>>

"NOT SIGN" is 172 in Unicode. 

The reason ^170 finds it is because (roughly speaking) the encoding that these ^ codes were based on was the old IBM  PC DOS encoding, either "Code page 437" or something similar. This is also the encoding used if you entered characters using Alt-numeric-keypad-170. I think if you entered Alt-0172 you would get the same thing, but I can't verify that here.
