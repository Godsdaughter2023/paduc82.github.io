Those symbols (and a number of others) do have default keyboard shortcuts. The copyright symbol is assigned to Alt+Ctrl+C, and the registered symbol is assigned to Alt+Ctrl+R. It's not so easy to build the correct syntax strings for this using VBA, you must follow the same rules as the PAGE.SETUP would be a real macro. That means e.g. any string must be surrounded by "" chars.

Try the sub below.

Andreas.


Sub PAGE_SETUP( _
    Optional ByVal CenterHeader, _
    Optional ByVal CenterFooter, _
    Optional ByVal LeftMargin, _
    Optional ByVal RightMargin, _
    Optional ByVal TopMargin, _
    Optional ByVal BottomMargin, _
    Optional ByVal PrintHeadings, _
    Optional ByVal PrintGridlines, _
    Optional ByVal ChartSize, _
    Optional ByVal CenterHorizontally, _
    Optional ByVal CenterVertically, _
    Optional ByVal Orientation, _
    Optional ByVal PaperSize, _
    Optional ByVal Zoom, _
    Optional ByVal FirstPageNumber, _
    Optional ByVal Order, _
    Optional ByVal BlackAndWhite, _
    Optional ByVal PrintQuality, _
    Optional ByVal HeaderMargin, _
    Optional ByVal FooterMargin, _
    Optional ByVal PrintNotes, _
    Optional ByVal Draft)
  'Execute the Excel4Macro PAGE.SETUP on the active sheet
 
  Dim Arg, Args() As String, Parse()
  Dim PS(0 To 2) As String, Temp() As String
  Dim i As Integer, j As Integer, k As Integer
 
  'Setup an array with all arguments used in PAGE.SETUP
  Parse = Array("head", "foot", _
    "left", "right", "top", "bot", _
    "hdng", "grid", "size", _
    "h_cntr", "v_cntr", "orient", "paper_size", _
    "scale", "pg_num", "pg_order", _
    "bw_cells", "bw_chart", _
    "quality", "head_margin", "foot_margin", _
    "notes", "draft")
  'Create an array for the real arguments
  ReDim Args(0 To UBound(Parse)) As String
  'Convert all arguments passed to our sub
  For i = 0 To UBound(Parse)
    Select Case Parse(i)
      Case "head": Arg = CenterHeader
      Case "foot": Arg = CenterFooter
      Case "left": Arg = LeftMargin
      Case "right": Arg = RightMargin
      Case "top": Arg = TopMargin
      Case "bot": Arg = BottomMargin
      Case "hdng": Arg = PrintHeadings
      Case "grid": Arg = PrintGridlines
      Case "size": Arg = ChartSize
      Case "h_cntr": Arg = CenterHorizontally
      Case "v_cntr": Arg = CenterVertically
      Case "orient": Arg = Orientation
      Case "paper_size": Arg = PaperSize
      Case "scale": Arg = Zoom
      Case "pg_num": Arg = FirstPageNumber
      Case "pg_order": Arg = Order
      Case "bw_cells": Arg = BlackAndWhite
      Case "bw_chart": Arg = BlackAndWhite
      Case "quality": Arg = PrintQuality
      Case "head_margin": Arg = HeaderMargin
      Case "foot_margin": Arg = FooterMargin
      Case "notes": Arg = PrintNotes
      Case "draft": Arg = Draft
    End Select
    GoSub CreateArg
  Next

  'Syntax 1
  'Worksheets and macro sheets
  PS(0) = "head, foot, left, right, top, bot, " & _
    "hdng, grid, " & _
    "h_cntr, v_cntr, orient, paper_size, scale, pg_num, " & _
    "pg_order, bw_cells, " & _
    "quality, head_margin, foot_margin, " & _
    "notes, " & _
    "draft"
  'Syntax 2
  'Charts
  PS(1) = "head, foot, left, right, top, bot, " & _
    "size, " & _
    "h_cntr, v_cntr, orient, paper_size, scale, pg_num, " & _
    "bw_chart, " & _
    "quality, head_margin, foot_margin, " & _
    "draft"
  'Syntax 3 (actually not used, maybe in the future)
  'Visual Basic Modules and the Info Window
  PS(2) = "head, foot, left, right, top, bot, " & _
    "orient, paper_size, scale, " & _
    "quality, head_margin, foot_margin, " & _
    "pg_num"
    
  'Which string should we use?
  j = IIf(TypeOf ActiveSheet Is Worksheet, 0, 1)
  'Remove all blanks
  Do While InStr(PS(j), " ") > 0
    PS(j) = Replace(PS(j), " ", "")
  Loop
 
  'Patch the arguments into the string
  Temp = Split(PS(j), ",")
  i = 0
  For k = 0 To UBound(Temp)
    For i = i To UBound(Parse)
      If StrComp(Temp(k), Parse(i), vbTextCompare) = 0 Then
        Temp(k) = Args(i)
        i = i + 1
        Exit For
      End If
    Next
  Next
  PS(j) = Join(Temp, ",")
 
  'Debug.Print "PAGE.SETUP(" & PS(j) & ")"
  Application.ExecuteExcel4Macro "PAGE.SETUP(" & PS(j) & ")"
  Exit Sub
 
CreateArg:
  Select Case VarType(Arg)
    Case vbSingle, vbDouble
      Args(i) = Replace(Arg, ",", ".")
    Case vbString
      Args(i) = """" & Arg & """"
    Case vbBoolean, vbByte, vbInteger, vbLong
      Args(i) = CLng(Arg)
  End Select
  Return
End Sub

dirk goldgar say:

GovPOCID = DLookup("[GovernmentTaskLead]", "qry_Current_POC", "[WBS] = '" & [WBS] & "'")   //   where GovPOCID = empty
GovPoc_Fname = DLookup("[FirstName]", "Personnel", "[PersonnelID] = " & GovPOCID)    //      where GovPoc_Fname = empty, GovPOCID is null

Boiling that down, that tells us that after executing the first DLookup, GovPOCd is Null.  There are two ways that could happen:  either (1) there was no record in the result set of qry_Current_POC that had WBS = to the value from the form, or (2) there was a matching record, but that record had the GovernmentTaskLead field = Null.

The thing to do now is to find out which of those two possibilities is the case, and correct it.  Verify what the value of WBS was when the code executed -- you can do that by stepping through the code again and hovering your mouse pointer over the second "[WBS]" in this line: 

    GovPOCID = DLookup("[GovernmentTaskLead]", "qry_Current_POC", "[WBS] = '" & [WBS] & "'")

Then, once you are sure what value it contains, open query "qry_Current_POC" and look for the record matching that WBS value.  If you don't find one, and you think there should be one, go back to the table from which that value should come and create the necessary record.  If you do find a matching record, make sure that its GovernmentTaskLead value is not Nulll.  If it's Null and it shouldn't be, enter the appropriate value.

Note that it's conceivable there could be more than one record in the query results that have the same value for WBS.  If there are more than one, there's no way of knowing which one of them the DLookup will look at for the GovernmentTaskLead, so you ought to ensure either that all of them have an appropriate value, or determine that they will all always have the same value so it's not a problem, or determine that there ought to be only one record and correct whatever is wrong that is giving you multiple matching records.
