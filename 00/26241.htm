ken sheridan say: Row in this context is the more strictly correct term for a record in a relational database table, though the two tend to be used interchangeably.

In the past Access could not lock a record currently being edited by one user to prevent conflicts without locking the 'page', which meant that not only would the current record be locked, but other records on the same page.  For some time now 'row level locking', i.e. locking the edited record only has been supported.

With 'page level locking' the strategy adopted was usually 'optimistic locking', which meant that more than one user could edit a record and conflicts then had to be resolved when attempts were made to save the record.  This avoided users being prevented from editing any records which were not currently being edited by another user.  With the advent of  'row level locking' a 'pessimistic locking' strategy became more feasible as it only locked a record currently being edited.  Wearing my manager's hat rather than my developer's hat I have favoured this strategy as I did not want staff to be wasting time and money editing records fruitlessly and then having to abandon the edit if another user's edit took priority, or overriding another user's edit.  With the sort of scientific data with which my team were concerned this could not be ruled out.

In your case, as the database is opened by only one user, 'page level locking', which is apparently de rigueur in this context, is no disadvantage of course as there can be no conflicts between users, so you have no need to be concerned.  My guess is that your Access installation is set up to use 'row level locking' by default.

myall_blues say: The general form of the IF statement is:
=IF(condition,do if true, do if false)
These can be nested together to deal with multiple conditions and combined with AND and OR statements to meet multiple criteria.

Looking at your example, something along these lines might be:
=IF(condition1,do if true,IF(AND(condition2,condition3),do if true,do if all conditions are false))

For example:
IF(A1>=5,"Green",IF(AND(A1>=2,A1<5),"Amber","Red"))

My general rule of thumb is that if you end up with more than about 4 or 5 nested conditions it's time to start looking for an alternative way, as they get difficult to read and maintain.

The trick is to nest If statements, create a statement initially, then keep replacing the false part with a new IF statement to create a more complex statement

=IF(A1=2, A1, IF(A1=3, A1-1, IF(A1=4,A1-2,"FALSE:}))
