I find it easiest to employ a User Defined Function.

Function ConCatRange(CellBlock As Range) As String
'usage =concatrange(A1:D10)
'for non-contiguous cells  =ccr((a1:a10,c4,c6,e1:e5))
'note double parens
Dim Cell As Range
Dim sbuf As String
    For Each Cell In CellBlock
    'comma de-limited. . . . .alter to suit
    If Len(Cell.text) > 0 Then sbuf = sbuf & Cell.text & ","
        Next
    ConCatRange = Left(sbuf, Len(sbuf) - 2)
End Function

me loffy say:

CREATING DATE ORDINALS WITH WORD 2016

Hi

Having searched many websites I find it is not possible to create a single field to be used across any new word document by saving something in the standard date entry/fields (under "insert and date/time" tabs of any word doc).  I have however created  field formula(s) to insert a format with ordinal dates e.g. 1st January 2000.  I created new field(s) with the text string(s) below and used it by copying it into the word document required (then change font style/size as appropriate) and finally saved it as a template.  Hope this may help?

{ DATE  \@ "dd"\*ordinal \* MERGEFORMAT } {  DATE  \@ "MMMM yyyy"  \* MERGEFORMAT }

NOTE:  use Ctrl and F9 to create the correct field parenthesis first and insert the text only inside them.  Then click "update field option" to show actual ordinal date/month/year.  Also note that there are two field(s) with a space between them to show a gap between "day" and "month/year".

All I do is copy / paste the formatted date from the template into whichever doc I want; and/or keep saving template with new file name.

Make sure the database (both FE and BE) are in Trusted Locations. The "Macro security warning" is the key here - if you have not setup those locations as Trusted Locations, you'll continue to get that prompt.

The macro name should be autoexec no spaces. When Access loads, it looks to see if there is a macro with that name and will then run it.
