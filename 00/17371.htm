Public Function get_emps()
Set objConn = CreateObject("ADODB.Connection")
objConn.ConnectionString = "DSN=MYODBCTOMSSQL"
objConn.Open

'Clear Previous Data
Sheets("DATA").Range("A3:C50").ClearContents

'create a recordset
var_sql = "select * from ts_emp where emp_status = 'A' ORDER BY emp_name"
Set rs1 = CreateObject("adodb.Recordset")
Set rs1 = objConn.Execute(var_sql)

i = 2
While Not rs1.EOF
i = i + 1
Sheets("DATA").Range("A" & i).Value = rs1("emp_id")
Sheets("DATA").Range("B" & i).Value = rs1("emp_name")
Sheets("DATA").Range("C" & i).Value = rs1("emp_type")
rs1.MoveNext
Wend

Set rs1 = Nothing
Set objConn = Nothing
End Function

superfranky say:

Hi Richard,

Retention policy is to keep the emails for a certain time and automatically delete/archive them after the certain time.

It’s not feasible to prevent users from being able to manually deleting emails from their mailboxes.

After the emails are permanently deleted by the end users, they are not able to access them.

However, we have Litigation Hold which can preserve mailbox items deleted by users or automatic deletion processes, and it’s a feature in standalone Exchange Online Plan 2, Enterprise E3 or higher plans.

After litigation hold is enabled, the user still can delete the emails, but the administrator can find the deleted items by In-Place eDiscovery.

For more information about Litigation Hold, please refer to the following article
https://technet.microsoft.com/en-us/library/ff637980(v=exchg.150).aspx
