Likely the first evaluation of the sql, parsing and creating some kind of query plan is the reason. And the data engine also may well pull a "blank" or fake row to determine the table/data structure before it attempts to pull the actual data. Or it pulls a blank row to create an internal memory structure to hold the data that the data engine will eventually pull. So there likely a dozen reasons for this.

So sure, agree that your "use case" is different then what I outlined, but you still don't have a lot of control over what is going to occur.

So in your case, likely the data engine is pulling the data structure, pre-parsing the structure and expressions for syntax errors, and then again for errors AFTER the syntax been verified. And then the data engine also has to build a query plan (and it may build more then one way to try and execute the sql).

greginmich say:

MSE can be reconfigured to present you with a choice of remediation options rather than taking automatic action, if that’s what you mean by “send only Alerts”. If you want the option to choose the action that MSE takes in response to an on-access detection, then you have to do 2 things:

1.) Turn off automatic remediation by unchecking the “Apply recommended actions: Help protect your PC by applying these actions after Security Essentials detects potential threats.” option in the “Default actions” settings.

2.) After making that change, you’ll get a more persistent popup when a detection is made: “This app detected and suspended a potential threat. Click Clean PC to remove this threat”. This popup only has one button, but it’s really giving you two choices; either “Clean computer” or “Show details”. What you have to do here is click on the “Show details” link in order to see the information provided for the threat, and also the options for alternative actions.

I always have automatic remediation disabled, not only so I can see what’s been detected in real-time, but also in order to make sure that I actually get notified when a detection is made. The automatic remediation popup – “Detected threats are being cleaned. No action needed”, disappears so quickly that it’s easy to miss.
