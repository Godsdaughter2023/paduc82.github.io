That version will appear if an auto-recover version of the document was not created (because the interval between the last user save and the crash was less than the auto-recover interval, which is set under File>Options>Save "Save AutoRecover information every ## minutes"

rob koch say:

Since Windows Defender is now very aggressive about the way it protects a system, it's difficult to either disable the scanning or control it dependably.

However, since your intent is to avoid the potential that an automated quick scan might occur during your game play, there's a relatively simple option that should work.  Just run a manual quick scan before you begin playing your game, since if an equivalent quick scan has already been run within the last 24 hours, a second should be automatically aborted when Windows Maintenance attempts to run another.

Note that I'm making this assumption based on past rules of operation for Defender and haven't tested it myself, so you'll need to try it to confirm it still operates the same way.

It's also possible to configure your own automated run of a quick scan using Task Scheduler, but this would still depend upon the automatic quick scan run by Windows Maintenance aborting, so I don't especially see this as valuable if your gaming times are variable.

Another option I just realized might be to change the start time for automatic maintenance itself to a time when you would typically have the computer on, but not be gaming.  I believe this still defaults to something like 3 am, but can be configured to a different hourly start time or even to wake from sleep if the system isn't typically turned completely off or disconnected from the electrical power grid.

Rob

greginmich say:

It’s very unlikely that an Automatic Maintenance scan is interfering with your gaming experience, since this scan runs with an only-on-system-idle priority, and has also been very dysfunctional since the Anniversary update. But you can turn this automatic scan off just by right-clicking on the "Windows Defender Scheduled Scan" entry in Task Scheduler and choosing “disable”.

To resolve the issue, though, you’ll most likely need to turn off Defender’s real-time protection as line that will make switching the Group Policy setting for Defender's real-time protection quick and easy. We never actually need to use the Group Policy Editor, or merge a downloaded REG file with the registry, in order to make these Group Policy changes.

GreginMich

The Antimalware Engine runs from startup to shutdown providing real-time (on-access) protection. It consumes CPU as it continuously scans all files involved in file system read/write operations, with the CPU utilization of the Antimalware Engine being proportional to the level of file system read/write activity.

Automatic Maintenance triggers the Windows Defender Scheduled Scan task on a “daily” baseline frequency, but this task normally fails to start. When it does start, it runs with an only-on-system-idle priority. Because this scan is terminated by any user activity, it’s very difficult to monitor, and the simplest way to make sure that it’s not running is to disable the Windows Defender Scheduled Scan task in Task Scheduler.

You should try just turning off real-time protection in the Windows Defender Security Center app before you resort to a “permanent” solution. If you feel compelled to use Group Policy in order to turn off Defender’s real-time protection (and turn it back on again later), then you can do that very quickly with these commands:

Right-click on the Start button and select Windows Powershell (Admin) and then copy, paste, and enter the command:

Turn off Windows Defender’s real-time protection:

REG ADD "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" /v DisableRealtimeMonitoring /t REG_DWORD /d 1 /f

Turn on Windows Defender’s real-time protection:

REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" /v DisableRealtimeMonitoring

But the only indication that you’ve just turned off all Antivirus/Antimalware protection will be this: 

This setting is managed by your administrator:

In case it isn't clear, you can't just leave real-time protection turned off; and you really shouldn't disable Defender's real-time protection unless you're offline. So, depending on the circumstances, you might be better off looking for a free AV solution that comes with a built-in gaming mode.

Hi Mat,

Why don't you try putting this formula into your "On Target %" text field:

Round(Switch([Start]>[Status Date],0,[Finish]<[Status Date],100,ProjDateDiff([Start],[Status Date])/[Duration]*100<100,ProjDateDiff([Start],[Status Date])/[Duration]*100),1) & "%"

This is the value that Project would insert in %Complete if you were to "Mark on Track" as Trevor indicated.  There is no dividing by 3600 needed because both the numerator and the denominator are durations in minutes.   Obviously this formula presumes that you use Status Date.  If not, then you need to replace "Status Date" with "Current Date" in the formula.

karl timmermans (outlook mvp 2012-2018) say:
>>
Is there a way to sync calendar and contacts from outlook 2013 to outlook.com
<<

Hello Pete

All that is required is to configure the Outlook.com account in Outlook 2013 by just entering your email address/password and the account will be automatically configured as an Exchange account where all folders (email, contact, calendar) will be sync'd with the server.

If you already have the Outlook.com account configured as either an IMAP or POP3 account, there are some additional steps you're going to have to take first which I'll provide if this is applicable. Just advise.

Just in case you already have the Outlook.com account configured and not sure of how it was configured (POP3 or IMAP), go to File --> Account Settings --> Email tab (from within Outlook desktop)

Lastly, in case this is also applicable, if you have other email accounts configured, it would be useful to know which data file is configured as the "default data file" since that's where the default contact/calendar folders are stored. To check that, go to File --> Account Settings --> Data Files tab (also from within Outlook desktop_

brian tillman [outlook mvp 2007-2019] on directly say:
As Karl said, you can sync between desktop Outlook and Outlook.com by using Exchange. What I gather from what you say is that you log into Outlook.com using a gmail address. If that's the case, add an @outlook.com alias to the account. Log into https://account.live.com/names/Manage and click the "Add email" link. Add an address ending in @outlook.com and then use that address and your existing password when you create the account in desktop Outlook.
