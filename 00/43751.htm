These normally refer to the Visual C++ Runtime.

The Visual C++ Redistributable is a DLL (Dynamic Link Library) file required by programs or games built using Microsoft’s Visual Studio software development environment. When a program requires a DLL or some other supporting file to run, this is called a dependency. The MSVCR.DLL stands for:

MS – Microsoft
V – Visual
C – C++
R – Redistributable.

Browsing your Apps and Features Setting or Programs and Features Control Panel, you might see multiple versions of the redistributable installed. A particular app installed might be utilizing one of them, so, removing any version can cause an application to malfunction. Some apps might even require updated versions of the Visual C++ Redistributable such as 2010 SP1 or 2012 Update 4.

Source: https://www.groovypost.com/howto/fix-visual-c-p...

Install both of the following:

Microsoft Visual C++ 2010 SP1 Redistributable (x86)

    martmcd say:

    Is there any way to express my fury at my Windows 10 automatic update settings? 

    yes, use the feedback app (click Start type feedback) you can search for feedback already left and upvote/add you own comment or you can send new feedback.

    setting the connection as metered is probably a good idea, I believe that windows defender will still get definitions but windows updates will not be downloaded until you either switch it back from metered or connect to an unmetered network (or plug in an Ethernet cable)

    Or

    you could disable the windows update service all together.

    However the following would also prevent

    1) updates for any other software that uses Windows update service, including windows defender.

    2) the ability to do a  manual search for updates, during active (working) hours.

    Updates are important, not just to keep up with the new features, but for security, I do not recommend that you disable the service without the provision to run windows update at least once a day. note Windows defender definitions are released daily, security updates are released when the risk becomes apparent, they are not left until 'update Tuesday'

    first check that Active Hours are set to cover the time that you need the computer working.

    Settings ->Update & Security -> windows Update, click the 'Change active hours link'

    next create a directory to put the batch files, (where you save them is up to you, as long as they are easy to find and accessible)

    right click start and select command prompt (admin)
    type     cd\ and press enter
    type     md BatchFiles and press enter
    close the command prompt.

    write a batch file to stop and disable the Windows Update Service.

    Open Notepad and type (or copy and paste) the following

    @echo off

    net stop wuauserv

    sc config "wuauserv" start= disabled

    quick explanation, @echo off - prevents the process being displayed on the screen, net stop - stops the service, wuauserv - is the windows automatic update service. sc config "wuauserv" Start=disabled - disables the service, stops it restarting.

    disabling the service while it is still running does not stop the service

    save the file as UpdateOff.bat in C:\BatchFiles.Note - the extension (the bit after the dot) needs to be bat in order for the system to see it as a batch file.

     

    write a batch file to Start the Windows Update Service and check for updates.

    Open Notepad and type (or copy and paste) the following

    @echo off

    sc config "wuauserv" start= auto

    powershell -(New-Object -ComObject Microsoft.Update.AutoUpdate).DetectNow()

    save the file as ForceUpdate.bat in C:\BatchFiles.Note - the extension (the bit after the dot) needs to be bat in order for the system to see it as a batch file.

     

     

     

    Use task scheduler to run the batch files to get windows updates.

    Open Task Scheduler and from the right hand pane, select Create Task.

    Give it a name (that means something) and description.

    Under Security options select a user with administrative permissions, run whether user is logged on or not,  Run with Highest privileges and configure for Windows 10.

     

    Click on the Triggers Tab and select New

    this is the 'Start the Service and Force Update' schedule so make sure to select a time that you want it to get Updates.

    Select Daily, select a time that you are happy to start the Windows Update service and click Ok, in this example I have set it to start shortly after the end of the Active Hours.

    Click on the Actions tab and click New.

    type C:\BatchFiles\ForceUpdate.bat into the program/script box..

    Click Ok

    Click on the Conditions tab and tick the 'wake the computer to run this task' to ensure that sleep does not prevent updates from being installed.

    I will let you decide on other Conditions and Settings - they will only affect this task if you change anything.

    Click OK

    it will ask for User Password - give it the details of an administrator account.

     

    Back in Task Scheduler, from the right hand pane, select Create Task.

    Give it a name (that means something) and description.

    Under Security options select a user with administrative permissions, run whether user is logged on or not,  Run with Highest privileges and configure for Windows 10.

    Click on the Triggers Tab and select New

    this is the stop and disable the Service' schedule so make sure to select a time that you want to prevent any Updates.

    Select Daily, select a time that you are happy to prevent Windows Updates and click Ok, in this example I have set it to stop updates at the start  of the Active Hours.

    Click on the Actions tab and click New.

    type C:\BatchFiles\UpdateOff.bat into the program/script box..

    Click Ok

    Click on the settings tab and tick the 'Run the task as soon as possible if a scheduled start is missed' to ensure that it will run even if it misses the schedule

    I will let you decide on other Conditions and Settings - they will only affect this task if you change anything.

    Click OK

    it will ask for User Password - give it the details of an administrator account.

    Is it worth upgrading to Windows 8 from Windows 7?

    This is ultimately a personal decision, Windows 7 is a modern operating system that supports all the advances available today in hardware and software. It also prepares for a future that is centred around mobility and touch. Windows 7 also supports many of the significant changes that have occurred on the Internet in the past 10 years. What more could you really want in a desktop OS?

    Windows 8 does introduce some specific advances where Windows 7 does fall short and I will list some of them here:

    • Fast Boot – Windows 8 boots significantly faster than Windows 7, in fact on an SSD, I can have Windows 8 up and running in 5 to 7 seconds.
    • Fully Touch Ready – This has been a miss on on prior versions of Windows going back to Windows XP Tablet PC edition. Windows 8 features a smart, fluid and fast interface that is significantly engineered for Touch called the Start Screen. There is just no disadvantage when using it with Touch. You can swipe, pan, zoom and do all the key actions expected on a Touch device. Microsoft has also created an environment that will encourage third party developers to create touch ready applications just for Windows 8. These applications will not run on Windows 7.
    • Fully compatible with existing hardware and software. If it works with Vista or Windows 7, it is guaranteed to work with Windows 8.
    • Tighter Internet integration – Windows 8 features a holistic integration with services that deliver information right to your screen without much need for you to find and discover it.
    • The ability to sync information across multiple devices makes Windows 8 a truly Internet ready operating system.

    These are just some of the key areas where Windows 8 advances over Windows 7. They may or may not entice you.
