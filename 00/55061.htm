lemp say:
As Xircal said, you're more likely to find the Office experts in the Office forum, but perhaps I can offer a few suggestions.

By your use of "ribbon," I assume that you have Word and Excel 2007 or 2010.  Please confirm your version(s).  In 2007/2010, click File > Help; in 2003 and earlier, Help > About Word (or About Excel).

At least in my installation of Word 2010, when I select File, the default selection is "Recent," which gives a list of recently-opened files and folders.  Can you see your files here?  If so, can you open them?

If you click the "Open folder" option on the File menu, you should get a dialog box like this:



To the right of the box for "File name" is a drop-down control.  In the figure above, the drop-down control shows "All Files (*)" -- I suspect that your Open dialog box shows something else.  The two most common settings here are probably "Word documents (*.docx)" and "All Word  documents (xxx,yyy,zzz)" where the xxx, yyy, zzz are a long list of file types.

If you have an old computer that used to have an earlier version of Word installed (2003 or earlier), you probably have a large number of *.doc files.  You can open and edit these files with Word 2007 or 2010, and when you save them, they will remain *.doc files.  If you have the drop-down control in the Open dialog set to "Word documents (*.docx)" you will not see any of files of this type.

Files that are newly-created in Word 2007 or 2010 are saved as *.docx files.

Because it would be surprising if you hadn't created some files from scratch, you should have some *.docx files (unless the default for saving files has been changed).  You should see these files if the drop-down control in the Open dialog is set to "Word documents (*.docx)." 

The fact that you don't see any files is a bit odd ... so what is your setting of the drop-down control in the Open dialog?  If you want the most flexibility -- at the expense of possibly having to sort through some non-Word documents when you are looking for a file to open -- set the control to "All Files (*)" as in the image above.

a. user say:

A couple of the common error messages associated with a failed launch Windows Live Mail are:

 

·         Windows Live Mail could not be started. The application was unable to open the Windows Live Mail message store. Your computer may be out of memory or your disk is full.

·         The application was unable to open the message store   

Typically with these types of errors, the Mail’s message store is unable to locate the database and it is quite possible that the message store has become corrupt.  To resolve this issue, try deleting the message store file of your account.

 

     1.  Click on Start.

     2.   In the search programs and files field (or Run) copy and paste the default location of the Windows Live

          Mail message store found below:

 

          %USERPROFILE%\Local Settings\Application Data\Microsoft\Windows Live Mail

 

     3.   Look for Mail.MSMessageStore file name and delete it (right click, delete).

Note:  The Mail.MSMessageStore does have a back-up folder and will work to recover your messages but may not be able to view some recently received messages.  If you have web based e-mail, Windows Live Mail will sync with the mail server so they are made available again.

 

If this does not resolve your issue, please Ask A Question on our answers forum and an agent will assist you. We’d really appreciate if you let us know where the steps failed and/or you are receiving a different error.
