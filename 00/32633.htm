Autoexec.bat went out of fashion with Windows ME, more than 10 years ago. It was never used under Windows NT. There was an autoexec.nt but I doubt that this would help you. Here is what you could do instead.
  1. Boot into Repair Mode.
  2. Locate the correct System Drive (Could be any of the lower letters of the alphabet. Can't be X:.)
  3. Create a backup of all registry files.
  4. Run regedit.
  5. Load the System hive (the one located on the correct System Drive).
  6. Navigate to this location: HKEY_LOCAL_MACHINE\{HiveName}\SYSTEM\MountedDevices
  7. If the value \DosDevices\D: exists (bottom, right-hand pane), rename it to \DosDevices\Q:
  8. Rename one of the other values to \DosDevices\D:. Which one it should be I have no idea. It's trial and error.
  9. Unload the System hive.
  10. Reboot and check things out.
  11. If things do not work, try a different value in Step 8.

I had this problem.  This is what I did, but I haven't tested it on any other Windows 10 PC's, so click on the "Solved my problem" ONLY if it really did solve your problem.  I have tried to be as detailed as possible.

Today is January 16, 2017, and I'm running Windows 10 Pro 64-bit Anniversary Edition version 1607 (OS Build 14393.693).  I installed the Japanese Microsoft IME to add support for my Japanese keyboard, but I couldn't switch from the "A" icon, and I only had the English alphabet.  When I right clicked on the "A", a menu showed up (all in Japanese, in spite of my system language being English and no Japanese language pack installed).  I tried rebooting, but that didn't seem to help.  You should probably reboot anyway.  It can't hurt.

Sdlion mentioned that the "A" menu was all in Japanese.  The first thing I did was to set the menu to English.  Right click on the "A".  Click on the 12th line that has an (R) next to it (Properties).  Click on the box that has an (A) next to it (Advanced).  At the top, select the tab that is the farthest to the right (Others).  In the top drop-down menu, select the third option (English).  At the bottom, click the third button from the left with an (A) next to it (Apply).  At the bottom, click the first button from the left (OK).  Now, if you right-click the IME "A", you should get this menu (I have added line numbers):

1-Hiragana
2-Full-width Katakana
3-Full-width Alphanumeric
4-Half-width Katakana
5-Half-width Alphanumeric
6-IME Pad
7-Add Word
8-User Dictionary Tool
9-Add-on Dictionary
10-Search
11-Mis-conversion report
12-Properties
13-Input Method
14-Conversion mode
15-Private mode (Off)  Ctrl + Shift + F10)
16-Troubleshoot problems

Note: By default, the "A" menu language is set to "Auto-select", and Windows 10 assumes that anyone installing a Japanese IME probably prefers Japanese (or it's a bug).

When I had BrennanSanford's problem, "5-Half-width Alphanumeric" was always selected.  I went into "12-Properties" -> Advanced -> Input settings -> Input method and changed it from "Romaji Input" to "Kana Input" and hit "Apply".  Then I set "13-Input Method" to "Kana Input" (the second option).  After this, I could select "1-Hiragana".  I typed a "t" on my keyboard, and the Hiragana "ka" (か)showed up in WordPad.  Now I can right-click on the IME and select Hiragana, Katakana or Alphanumeric.

If you don't have a Japanese keyboard, you can set the "13-Input Method" to "Romaji Input" (the first option), and then type "k" followed by "a" to get the same result.

Be sure to use a font that supports Japanese, such as MS Mincho.  The IME seems to pick the font automatically.

Let me know if this worked for you.  Reply if there's something I forgot to say, or if there's something that is unnecessary.

FYI: You can have multiple physical keyboards plugged into the same computer.  All will work, but you have to manually select the keyboard type in the language bar to get the proper mapping.
