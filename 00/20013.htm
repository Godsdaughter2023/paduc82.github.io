It has been my experience that IE will eventually (perhaps for no visible reason) sort

Favorites by name.

 

Workaround:

Rename individual Internet Shortcuts so that sort by name works for you.

Example: 1yahoo.com will  be listed before bing.com. Or aYahoo.com before bBing.com.

 

Preventive measure:

After organizing Favorites in your preferred order, export one or both of these registry keys every day. Double-clicking (merging) either key will

restore the value (sort order) when the key was exported.

 

Favorites:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\MenuOrder\Favorites

 

Favorites Bar (Links)

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\MenuOrder\Favorites\Favorites Bar

Export/backup the entire registry (for safety)

Drill down, by clicking the + to

HKEY_CURRENT_USER

   Software

      Microsoft

         Windows

            CurrentVersion

               Explorer

                  MenuOrder

                     Favorites 
Select Favorites, right-click> Export.

Repeat for Favorites Bar.

 
Exit Registry Editor.

sven johannsen say:

General reminders/appointments are stored in the phone adjusted to a universal time, and displayed based on the time zone you have set, or is set for you. What that winds up doing is changing the time the appointment appears on the calendar. If you place an appointment on your calendar at noon (12PM) and travel west two times zones, and the phone picks up that time zone change, the appointment will show up at 10AM. 10AM and 12PM are actually the same time. If one person called the other and asked the time, one would say 10AM, the other 12 noon.

You used be able to select your Time Zone and change it when you wanted to on PDAs, but with the advent of cell phone PDAs, the time zone is fairly universally set by the carrier. Great for the clock, the calendar however depends on the individual on whether they like this to happen if it is good or not.

All day appointments are a bit trickier. Haven't looked real hard at what happens to them in WP. The hope is that they are designated as a single day event, keyed to a day/date. Some systems have erroneously stored them as a 24hour appointment for midnight to midnight. Moving time zones with these can have the affect of the 'day' now spanning two days, from like 10PM one day to 10PM the next. That can erroneously result in the same appointment being shown on adjacent days, because of an all-day tag being assigned to the entry on two different days.

Sometimes these things happen not because the system you are using has a bug, but one you sync with has a different interpretation, or methodology,  which messes it up and syncs that mistake back.

Windows 10 UI Design Submissions & Discussion

Among those participating in the Windows 10 Technical preview are a number of designers - some are professionals (such as Padwick19, who inspired this discussion) while some are novices (such as myself).  Rather than simply complaining about certain UI elements and design choices, those of us with a modicum of design skills should try putting forth our ideas and see if the rest of the community thinks they're any good.

Even if they aren't, they should at least help Microsoft get a better feel for what users want from Windows 10 design-wise.

Although the design concept presented in this post focuses on the Start Menu/Destop UI, Metro UI submissions are equally welcome.


Just to get the ball rolling.....


Desktop UI Concept

The following design is my attempt to 'modernize' the classic look of Windows XP/7 for those who prefer the traditional Start Menu/Desktop paradigm.  It utilizes 70% opacity for background fields (Taskbar, Start Menu, Windows, etc) to mimic the look of Windows 7 Aero, though this can be turned off to resemble the flat colors of Windows 8/8.1.

The Start Menu field is entirely modular and tile-based - 4 wide and 6 high by default, though it automatically adjusts to contain the user-configured content.  Default content includes:

  • Traditional Program Listing - 3 tiles wide; defaults to 6 tiles high, but can be made taller or shorter
  • User Profile Picture - single tile; left-click brings up the User Accounts section of the Control Panel
  • My Documents, My Pictures, My Music, and My Computer Icons - single tiles; self-explanatory
  • Control Cluster - single tile; contains:
    • Help and Support shortcut button
    • Control Panel shortcut button
    • Power Button w/context option

Users can drag Tiles and Icons (program shortcuts) and hover over the Start Button to open the Start Menu and drop them in a manner similar to the current system of pinning shortcuts to the Taskbar.  Users can also right-click in any vacant area of the Start Menu to bring up a configuration menu.  This configuration menu functions similar to the customization menus in Microsoft Office products.  Configuration menu would also contain a checkbox to lock Tiles/Icons to the Start Menu to prevent accidental removal.

Tiles and Icons pinned to the Start Menu feature simple drop shadows though, as with transparency, users are free to turn off this feature.

Default (Desktop View) w/Docked Tiles/Shortcuts  Search Entry & Results          Tray/Desktops


Start Button and Taskbar Icons are updated to present a more polished appearance than the flat Windows 8/8.1 style presently in Windows 10.

Search feature has been relocated to the Taskbar.  Left-clicking on the Icon brings up the Search Menu, while right-clicking brings up a context menu to select between searching the user's computer, the internet, or everything (default).

The Desktop Selector has been relocated to the far right of the Taskbar.  Hovering over (or left-clicking on) this Icon brings up all current desktops similar to hovering over an item on the Taskbar.  Right-clicking on the Icon brings up the context menu, including Show Desktop.

I don't mean to derail your input, Padwick19.  I don't like Metro and I don't use Metro, so I don't feel I'm in a position to critique changes to its operation.

That being said, I can say the following:

1 ) I wholeheartedly agree with you that being thrown to the desktop while running in Metro mode all-but defeats the purpose of using Metro to begin with.  This problem will be further exacerbated by running desktop-based software on mobile devices - as Microsoft seems to have made clear as part of their goals for Windows 10.

I mean, SERIOUSLY?!?!?  Most of my 'design experience' comes from the realms of video games and web development, but come on!!  It's rather nauseating to hear people laud Metro as "sleek and modern" when, in reality, it's a throwback to the Dumb Days of web design - back when the internet was so new that we could barely even grasp its potential, let alone think things through design-wise.

AOL is the laughing stock of the internet, yet people somehow thought it was a good idea at the time.  Metro is barely two years old and it's following the same pattern as AOL.  While the mockery it receives is often dismissed as 'schoolyard banter', kids in grade school are old enough to distinguish good from bad but not yet old enough to convincingly lie about it.

Let's just be honest, shall we?

In the realm of mobile OSes, Microsoft fell asleep at the switch and is desperately trying to play catch-up.  Personally, I don't think they can ever overcome this liability but, for the moment, let's assume that they can.

So let's look at the evolution of mobile OSes...

For the first decade (1996 to 2006) PalmOS (native to the PalmPilot) reigned supreme.  Microsoft tried to challenge with Pocket PC in 2000 and Windows Mobile in 2003 to no avail.  This scenario is highly reminiscent of the early 'Home Computer Wars' of the mid-to-late 1980s - lots of competing standards and OSes were mostly wedded to specific devices.  This held true even when iOS (iPhone, 2007) took the world by storm.

Android came out back in 2005 and, as recently as 2010, was little more than an 'also ran' in the world of mobile OSes.  So how did they manage to become the dominant mobile OS within 18 months?

We could argue some of the finer details til we're blue in the face, but all those pale to the simple fact that Android was released as open-source.  This measure put UI design in the hands of the masses - countless UIs were developed while the core Android team incorporated the most popular aspects into the 'standard' UI.

The bottom line to all this is that Android is the dominant mobile OS (some sources report an 85% market share) because it features the most-widely preferred mobile UI extant.  In short, it managed to out-iOS iOS!

If Microsoft wants to gain dominance in the mobile OS market, they have two options:

1 ) Try to establish a whole new operational paradigm.  This is what they tried - and failed - with Metro.  Mobile users don't want it, and computer users sure as hell don't want it!  Windows 8 and the various mobile incarnations have proven this in spades.

2 ) Try to out-Android Android (which, as an open-source OS vs Microsoft's proprietary OS, I don't believe possible).  Microsoft does, however, have three major trump cards - 95, XP, and Windows 7.  iOS succeeded by adapting the long-established Mac OS experience to mobile devices (desktop > mobile).  Android succeeded by adapting the long-established Linux experience to mobile devices (desktop > mobile).  

The ironic fine print in this equation is that Android/Linux and Mac/iOS both did this by preserving most of the conventions established by Microsoft and their best Windows OSes - 95, XP, and Windows 7.  In short, Android and iOS managed to out-Windows Windows!  In fact, Android is currently making headway in competing against Windows in the computer OS market!!

I simply cannot understand why Microsoft is trying to reinvent the wheel when they invented the wheel in the first place!  In the chaos of the early days of personal computers, Microsoft emerged as the standard to which the entire world has conformed ever since - and they're the only ones refusing to conform.  It's like the role the US played in establishing the Metric System as the international standard system of measure then refusing to abandon the old SAE system.

Again, I don't fault Microsoft for trying to become a serious player among mobile OSes.  What I do fault is their doing so by apparently abandoning their utter dominance of Windows as a computer OS.
