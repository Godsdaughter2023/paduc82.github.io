I was hoping you could take me one step further with this?  I was able to apply the same concept with Lunches, but now I need to get a daily total (sum) of receipts (payments received) by MealType.

It’s much the same but this time you multiply the value by the return value of the IIf function:

SELECT OrderDate,
SUM(PaymentReceived *IIF(MealID = 1,1,0)) AS [Free Breakfast],
SUM(PaymentReceived *IIF(MealID = 2,1,0)) AS [Reduced Breakfast],
SUM(PaymentReceived *IIF(MealID = 3,1,0)) AS [Full Price Breakfast]
FROM Orders INNER JOIN [Order Details] ON Orders.OrderID = [Order Details].OrderID
GROUP BY OrderDate;

ken sheridan say: You don't need a crosstab query for this.  As you are counting only three fixed categories of meals you can use simple conditional aggregation:

SELECT OrderDate,
SUM(IIF(MealID = 1,1,0)) AS [Free Breakfast],
SUM(IIF(MealID = 2,1,0)) AS [Reduced Breakfast],
SUM(IIF(MealID = 3,1,0)) AS [Full Price Breakfast]
FROM Orders INNER JOIN [Order Details] ON Orders.OrderID = [Order Details].OrderID
GROUP BY OrderDate;

Each computed column sums the return value of an expression which returns 1 or 0.  Summing the ones is therefore the same as counting the instances of rows which meet the criterion.  Note that you do not need to include the table which contains the text meal type values as the meal types are returned as column headings here not data.

      Exit Sub
    End If

doug robbins - mvp office apps & services (word) say:

It is NOT be necessary to remove all the page numbers and start again!!!  Save yourself the trouble.

If you have Continuous Section Breaks in your document, it can be difficult via the user interface to access all of the Sections to set the page numbering format.

In those circumstances, running a macro containing the following code can help:

Dim i As Long, j As Long

With ActiveDocument
    For i = 2 To .Sections.Count
        With .Sections(i)
            For j = 1 To .Footers.Count
                .Footers(j).PageNumbers.RestartNumberingAtSection = False
            Next j
        End With
    Next i
End With

charles kenyon say:

The Continuous Page Numbering Add-In has the equivalent code attached to a button.

Here is the code:

Sub ContinuousPageNumbers1(control As IRibbonControl)

' Jay Freedman

' http://answers.microsoft.com/en-us/office/forum/office_2007-word/page-numbers-are-all-fouled-up-in-my-large/d188687e-9663-43e0-a450-1dbadc47f09f

' modified to use with Ribbon button and add message boxes by Charles Kenyon

'

    Dim secNum As Long

    Dim btnCancel ' give user chance to cancel

    btnCancel = MsgBox(prompt:="Do you want to reset all of the page numbers in this document to number continuously?", _

        Title:="Are you sure?", _

        Buttons:=vbYesNo)

    If btnCancel = vbNo Then

        MsgBox prompt:="Reset of continuous page numbering cancelled by user!", Buttons:=vbExclamation, Title:="Page Number Reset Cancelled!"

        Exit Sub

    End If

'   Proceed with reset

    With ActiveDocument

        For secNum = 2 To .Sections.Count

            .Sections(secNum).Headers(wdHeaderFooterPrimary) _

                 .PageNumbers.RestartNumberingAtSection = False

        Next

    End With

    MsgBox prompt:="The Continuous Page Numbers macro has run.", Title:="Page number reset macro finished!"

End Sub

To modify this so it runs as a macro without using the button, you would delete “control as IRibbonControl” from the first line. This macro adds a check to make sure that this is what  the user wants (because it is easy to click a button by mistake) and a confirming message when done.
