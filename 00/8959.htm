Five 2" rows are consuming the entire 10" of space between top and bottom margins, so the table is filling the page. What you had not taken into account is the fact that Word requires an empty paragraph following the table. For more, including workarounds, see http://wordfaqs.mvps.org/blankpage.htm#TableAtEnd

In addition, if your table has borders, they will be added to the row height; if you have a one-point border between rows (and at the top and bottom of the table), then you've added six points to the table height.

john sr wilson say:
No way from the GUI. You would need to add images of the previous and next slides. Could be done with a macro.

Sub pics()
Dim i As Integer
Dim s As Integer
Call zap
ActivePresentation.SaveCopyAs Environ("TEMP"), ppSaveAsJPG
For i = 1 To ActivePresentation.Slides.Count
Select Case i
Case Is = 1
With ActivePresentation.Slides(i).Shapes.AddPicture(FileName:=Environ("TEMP") & "\Slide" & i + 1 & ".JPG", _
 LinkToFile:=False, SaveWithDocument:=True, Left:=ActivePresentation.PageSetup.SlideWidth - 200, Top:=5)
.ScaleHeight 0.15, True
.Left = ActivePresentation.PageSetup.SlideWidth - .Width - 5
.Line.Visible = True
.Line.ForeColor.RGB = vbBlack
.ActionSettings(ppMouseClick).Hyperlink.SubAddress = "-1,-1,NEXT"
.Tags.Add "DEL", "OK"
End With
Case Is = ActivePresentation.Slides.Count
With ActivePresentation.Slides(i).Shapes.AddPicture(FileName:=Environ("TEMP") & "\Slide" & i - 1 & ".JPG", _
 LinkToFile:=False, SaveWithDocument:=True, Left:=5, Top:=5)
.ScaleHeight 0.15, True
.Line.Visible = True
.Line.ForeColor.RGB = vbBlack
.ActionSettings(ppMouseClick).Hyperlink.SubAddress = "-1,-1,PREV"
.Tags.Add "DEL", "OK"
End With
Case Else
With ActivePresentation.Slides(i).Shapes.AddPicture(FileName:=Environ("TEMP") & "\Slide" & i - 1 & ".JPG", _
 LinkToFile:=False, SaveWithDocument:=True, Left:=5, Top:=5)
.ScaleHeight 0.15, True
.Line.Visible = True
.Line.ForeColor.RGB = vbBlack
.ActionSettings(ppMouseClick).Hyperlink.SubAddress = "-1,-1,PREV"
.Tags.Add "DEL", "OK"
End With
With ActivePresentation.Slides(i).Shapes.AddPicture(FileName:=Environ("TEMP") & "\Slide" & i + 1 & ".JPG", _
 LinkToFile:=False, SaveWithDocument:=True, Left:=ActivePresentation.PageSetup.SlideWidth - 200, Top:=5)
.ScaleHeight 0.15, True
.Line.Visible = True
.Line.ForeColor.RGB = vbBlack
.ActionSettings(ppMouseClick).Hyperlink.SubAddress = "-1,-1,NEXT"
.Tags.Add "DEL", "OK"
.Left = ActivePresentation.PageSetup.SlideWidth - .Width - 5
End With
End Select
Next i
End Sub

Sub zap()
Dim s As Integer
Dim osld As Slide
For Each osld In ActivePresentation.Slides
For s = osld.Shapes.Count To 1 Step -1
If osld.Shapes(s).Tags("DEL") = "OK" Then osld.Shapes(s).Delete
Next s
Next osld
End Sub

steve rindsberg say:

I think so. 

If you want to match the rectangle to the position/size of the SHAPE that the user's typing into, you'd:

Look at ActiveWindow.Selection.Type to see if the user's selected a shape or is typing text.  If shape, you match your new shape's coordinates to ActiveWindow.Selection.ShapeRange(1).Top (.left etc)

If text, you'd walk the textrange's parent tree to get to the shape, then match its coordinates.

If you want to work with the current text cursor position, you can use the .BoundTop, .BoundLeft etc properties of the current textrange, but that won't help if the users' typing because there'll be no selected text.  The .Selection.Type will be ppSelectionText but the .Selection.TextRannge.Text will be "" (null) so .BoundTop/etc will throw errors.
