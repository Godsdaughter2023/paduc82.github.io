Use this if you want the answer blank if either C2 or C4 are blank:

=IF(OR(C2="",C4=""),"",IF(C2<6,IF(C4="yes","Worksheet","Form"),IF(C4="yes","Nothing","Memo")))

Use this if you want the answer blank if both C2 and C4 are blank:

=IF(AND(C2="",C4=""),"",IF(C2<6,IF(C4="yes","Worksheet","Form"),IF(C4="yes","Nothing","Memo")))

randy k. say: Len -

   I used tables a lot in an old version of Word for Mac (version X) and had a problem which was similar to what you describe with the last row on a page.  What worked for me was to change the row so it would not flow across pages.  This works the same in my new Word for Mac 2011, so I assume its the same for all intervening versions.

    I position my cursor somewhere in the visible portion of the last row at the bottom of the page.  I then go to the top menu and click on "Table".  On the drop down menu at the very bottom I click on "Table Properties".  In the "Table Properties" box that opens up I click the second choice at the top - "ROW".  There are two sections in the "Row" box, "Size" and "Options".  In the "Options" section there is a check box "Allow row to break across pages."  Unchecking this box causes the page break to occur at the end of the last row that can fully display.  The partial row now appears on the next page and all text shows up there.

    I'm not sure if this is exactly your problem, but it corrected text missing from the bottom row on a page for me.  Hope this helps.

   - Randy
