Hello,

not sure if or how the default choices in the Accounting format drop-down can be changed, but I have an alternative suggestion.

If you only use a limited number of different currency formats, create a sample for each format. Then select one of the cells, click the Cell Styles drop down on the Styles group in the Home ribbon and click "New Cell Style".  Uncheck all options other than "Number", give the style a name and hit OK.

Repeat for the other currency formatted cells.

The next time you want to apply a currency format, don't use the Accounting Number Format drop-down, but use the Cell Styles drop-down instead. The formats you have just created will be listed at the top in the "Custom" category.

You can, of course, include other cell formatting in the style, for instance, if you want to highlight each currency with a specific colour. The cell styles can be just a number format, but are a lot more powerful than just that.

Outlook Running Slow/Not Responding Constantly - Solved

I've been getting several reports from my users that their Outlook is running incredibly slow, and is constantly hanging up while trying to use it. We are in an Exchange 2010 environment, and the majority of users are running Outlook 2010. I tried everything I could possibly think of; Installing all updates, Repair installation, ScanPST.exe, Outlook in Safe Mode, Running Logging mode, using the /Cleanviews switch, disabling add-ins, restarting several times. None of them solved the problem at all. 

Then my coworker mentioned that he created a new mail profile for one of the users, and that this seemed to clear up the problem. When I tried it for myself, I noticed that it took away the additional mailboxes that the user had, so I went to add them back in. Several of them could not resolve the name. I switched back to the original profile, removed the offending mailboxes, and that solved all the issues. I tried it on a couple other machines that had the same issue, and it solved their problem as well. The issue seemed to be that the mailboxes that had been deleted were trying to pull information, but with the mailbox being gone, there was nothing for it to find. 

I wanted to share this as a possible solution for other people that may be experiencing this issue. This sort of issue is incredibly difficult to troubleshoot because there are so many different reasons why it may be happening. I hope that this can save someone some time when they run into this problem.

As default, outlook checks for changes to data loss prevention policies one time every 24 hours. But if we clear the LastDownloadTimesPerAccount, we can force the outlook client to download the newest data loss prevention policies from server side, its’ equal to let the data loss prevention policies take effect earlier. 

1.            Right-click Windows Start button, then select Run. 
2.            Enter regedit in the open box and select OK.
3.            In Registry Editor, locate the following subkeys: 
HKEY_Current_User\Software\Microsoft\Office\16.0\Outlook\PolicyNudges
4.            Delete the LastDownloadTimesPerAccount registry entry.
5.            Exit Registry Editor.
6.           restart your outlook.

If you are in different situation or this method can’t meet your requirement, please correct me and tell me the specific policy type you want to configure.
