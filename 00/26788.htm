MSE does not block websites.  Why are you so certain that MSE is the cause of the block?  It may block a file download if it detects a threat within that file (and that may be the case here even if it isn't right to do so).  The most likely problem is that the download is being blocked by some security setting in your browser and not by MSE.

 

Please tell me the entire and exact word-for-word error message you get when the problem occurs and exactly when the problem occurs (when you go to the site, when you press the download button, when the download is done or almost done and is being scanned, or when)?  What is the name and version of your browser (are you using Internet Explorer 9.0.8112.16421IC with update version RTM or something else)?  What version of Windows are you using (e.g., 32-bit Vista Business SP2 - and yes, that much detail)?

 

Assuming it is IE (or that you can do the same with other browsers if not), go to Tools / Internet Options / Security / Trusted Sites / Sites / and add the Adobe website to the list of Trusted Sites.  Close all instances of IE and open a new window and see if that helps.

 

If not and you're absolutely certain MSE is causing it because of the message - I would prefer you wait until after you have replied to the above questions and received my reply first and do not recommend it because it leaves the system vulnerable while MSE is disabled, but the decision and the risk are yours (I have MSE and have downloaded both programs and updates many times and do not have this problem), then go to the Adobe site (type in the address yourself so you know that's where you are) and have only ONE open tab in only one open Window in your browser.  Have no other programs open that connect to the internet (like an email program or whatever).  Select the first file you want to download and choose to save it to your desktop but don't start it yet.  Open MSE and go to Settings and then Real Time Protection and then uncheck the box that enables it.  Now go back to your browser and start the download.  If it works, do the second download and make sure it too works.  When done, immediately go to MSE Settings Real-time protection and recheck the box to enable real-time protection.  Now right click on each downloaded file and click to scan with MSE just to make sure the files are safe.

I hope this helps.

So you have 10 worksheets...There are two ways..

1. Either apply the below formula in cell A1 of Sheets 2 to . In Sheet2 apply the below formula. =Sheet1!A1+IF(WEEKDAY(Sheet1!A1)=6,3,1)

In Sheet3 =Sheet2!A1+IF(WEEKDAY(Sheet2!A1)=6,3,1)

Please note that the sheetnames are referred to the previous sheet everytime....

2. If you dont want to have a different formula for each sheet then try the below on asaved workbook.  Sheets should be renamed as 1 to 10 instead of Sheet1 to Sheet10

=INDIRECT(REPLACE(CELL("Filename",A1),1,FIND("]",CELL("filename",A1)),"")-1 & "!A1")+

To spell it out, assuming your date wants to go in cell A1:

Sheet4, cell A1: enter "=Sheet1!A1+3"

Sheet8, cell A1: enter "=Sheet1!A1+9"

Sheet9, cell A1: enter "=Sheet1!A1+10"

Sheet10, cell A1: enter "=Sheet1!A1+11"

Save your workbook.

Powerpoint action button

Is there a way to run a macro in the coding of a action button (activeX control). When I click on the action button I want it to run one of my macros. Is this possible?

john korchok say:

An ActiveX button is called a Command button. and it's inserted from the Developer>Controls group. An Action button is inserted from Insert>Shapes. Both can run macros. Which one are you using?

justinpf say: I'm using a command button (activex)

john korchok say:

To store a macro in the button code:

Insert a command button on the slide using Developer>Controls>Command Button.

Click on Developer>Controls>View Code.

 

Paste your macro between the lines. They look like this:

Private Sub CommandButton1_Click()

End Sub

Private Sub CommandButton1_Click()

End Sub
