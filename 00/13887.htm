Office 365 - Outlook / Calendar

Hello All,

     I will be running Windows 10 Enterprise and Office 365 and solely deploy to over 5000 workstations in New York City.  Within Outlook/Calendar, the default geolocation is Washington D.C. Is there a way to change the default to New York instead if Washington D.C. with a registry or GPO?

Thanks to all in advance,

nyc_al say:

Again thanks for the response, but I was able to resolve the issue with the first bullet, the second to control the update intervals and the third is for those who wish to disable the feature.

 User Configuration > Admin Templates > Microsoft Office 2016 > Outlook Options > Preferences > Calendar Options > Weather Service URL

Enable the policy and set the url to:

http://weather.service.msn.com/data.aspx?weasearchstr=newyork,ny&src=outlook

                        NOTE: If you disable or do not configure this policy setting, Outlook uses the default weather service URL.

User Configuration -> Administrative Templates -> Microsoft Outlook 2016 -> Outlook Options -> Preferences -> Calendar Options -> Weather Bar Update Frequency

NOTE: If you disable or do not configure this policy setting, Outlook uses the default value of 120 mins.

User Configuration -> Administrative Templates -> Microsoft Outlook 2016 -> Outlook Options -> Preferences -> Calendar Options -> "Disable Weather Bar" is set to       "Enabled”

gobluetwo say:

Visio 2002 SP2: "Unexpected end of file" error during action "Copy"

I've opened an org chart from a client in Visio 2002, but can't copy anything.  I get the "Unexpected end of file" error when I try to copy anything.  Not sure what version of visio the client used to create, but must not be anything newer than 2003 b/c I can open it. 

End goal is to be able to copy into a more "common" format (powerpoint, word) for my colleagues who do not have a visio license.  Any ideas on how to resolve/work around the error?

sekar r - microsoft support say:

In case if you encounter this issue in future, please try following options and see if it will fix the corruption:

1) Remove unused master shapes from the document stencil in either Visio 2000,
2002, or 2007. In Visio 2000 and 2002 you can use the Clean It utility to remove
unused master shapes (Visio 2007 has built-in option to remove unused shapes).

Clean It 2.0 can be downloaded from,
http://download.microsoft.com/download/visio5/utility/2.0/win98/en-us/cleanit2.exe

*Microsoft does not provide support for this utility.

After cleaning out the unused master shapes, attempt to open the drawing in
Visio 2003.

2) WORKAROUND:

Delete all master shapes from the document stencil. The drawback to this approach
is that it will break the links to the objects on the drawing page, which can
enlarge the file size of the drawing.

Hope this helps.
