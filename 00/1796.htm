Try this adaptation for your newer sample layout. I had to guess a bit at where the HPageBreaks were to be placed.

Sub mcrFillCarts()
    Dim c As Range, hdrs As Variant, ws1 As Worksheet, ws2 As Worksheet
    Dim iCart As Long, sCart As String, app As Application
    Set app = Application
    app.ScreenUpdating = False
    Set ws1 = Sheets("Sheet1")
    Set ws2 = Sheets("Sheet2")
    With Sheets("Sheet1")
        hdrs = Array(.Cells(1, 1), .Cells(1, 2), "QTY")
        With .Cells(1, 1).CurrentRegion
            .Cells.Sort Key1:=.Columns(3), Order1:=xlAscending, _
                        Key2:=.Columns(2), Order2:=xlAscending, _
                        Key3:=.Columns(1), Order3:=xlAscending, _
                    Orientation:=xlTopToBottom, Header:=xlYes
        End With
        iCart = 0
        sCart = app.Proper(.Cells(2, 3))
        ws2.Cells(Rows.Count, 1).End(xlUp) = sCart
        ws2.Cells(Rows.Count, 1).End(xlUp).Offset(1, 0).Resize(1, 3) = hdrs
        For Each c In .Cells(2, 3).Resize(.Cells(Rows.Count, 3).End(xlUp).Row - 1, 1)
            If (iCart = 5 And app.Proper(c.Value) = sCart) Or _
              app.Proper(c.Value) <> sCart Then
                iCart = 0
                ws2.HPageBreaks.Add Before:= _
                  ws2.Cells(Rows.Count, 1).End(xlUp).Offset(2, 0)
                ws2.Cells(Rows.Count, 1).End(xlUp).Offset(2, 0) = app.Proper(c.Value)
                ws2.Cells(Rows.Count, 1).End(xlUp).Offset(1, 0).Resize(1, 3) = hdrs
            End If
            ws2.Cells(Rows.Count, 1).End(xlUp).Offset(1, 0) = c.Offset(0, -2)
            ws2.Cells(Rows.Count, 1).End(xlUp).Offset(0, 1) = c.Offset(0, -1)
            iCart = iCart + 1
            sCart = app.Proper(c.Value)
        Next c
    End With
    Set ws2 = Nothing
    Set ws1 = Nothing
    app.ScreenUpdating = True
    Set app = Nothing
End Sub

The Windows 10 update for Lumias - Microsoft Community

stranger_hanyo say:

The Windows 10 update for Lumias

Hello,

The last few days were pretty exciting for us all insiders as windows 10 mobile finally rolled out to many phones over the world. We, insiders played an important role in shaping up windows 10 mobile through the insider program. Many devices around the world had been enrolled into the insider program through the windows insider app, and users got a taste of what would become the next great mobile OS. All Lumias which came pre-installed  with windows phone 8 could sign up for the insider program and receive preview builds and provide feedback based upon the users experience. This feedback played a great role in shaping up the present release of windows 10 mobile.

As for every software update, not all devices can enjoy all the new features and so it was for the windows 10 mobile update. While all Lumias with the denim update could run insider builds, not all devices made cut for the official update. Why this decision though?

Simply, even if all devices could run the insider builds, not all of them had a buttery experience. See this link, http://answers.microsoft.com/en-us/mobiledevices/forum/mdlumia-mdupdate/announcement-windows-10-mobile-upgrade-for-windows/dc50c9a5-f509-4892-9da5-fd85d71ba514.

Consider the ever popular Lumia 520. It received builds upto 10586.164(which is the public release) and the end user experience was not as smooth as using the same build on a Lumia 735. Though both ran the same OS before, the underlying hardware was not at all same.  Hardware plays an important part in determining the software experience and simply the old hardware could not provide a satisfactory experience for us, insiders. Now testing preview builds is quite a different thing than the usage of a consumer. Insiders, know that a preview build may have some quirks here and there but the average consumer is not much of a tech geek. He will evidently be displeased with the lags and all the quirks. Thus the decision of not releasing the update to the older devices may be considered to be a fair one.

Now the question may arise, "What for the unsupported phones then? Didn't we provide feedback based upon them? Then where is my update?" The answer is that you can still use windows 10 on your phones upto build 10586. Builds from this branch will still be available for these devices through the insider program and believe me, they run fairly well with a few quirks here and there. 

So fellow insiders who may have grudges against Microsoft regarding the update decision, calm yourselves and continue enjoying windows 10 on your devices through the insider program. The experience is stable enough for insiders and hey, you still get windows 10 on your phones right?

Thanks.
