Issue when exporting to CSV in Excel 2016 - Microsoft Community

noonand2 say:

Issue when exporting to CSV in Excel 2016

Hello,

As per request from @msexcel I'm posting this here. I initially tweeted about this here: https://twitter.com/noonand/status/693118028632956929

Attached image showing simple repro:

teylyn say:

Hello,

Do you mind explaining the issue in this question instead of only on Twitter?

It looks like you don't want semicolons as the separator in your csv file. 

When saving as a CSV, Excel will use the Windows list separator to separate values. Depending on your regional settings, this may very well be the semicolon, not the comma. Most European settings have the semicolon and will use the semicolon in formulas, where US or UK regional settings use the comma. The comma does not work well as the list separator in European formats, where the comma also serves as the decimal character. 

Therefore, European settings use the semicolon in CSV files. 

I normally use the comma, but have changed my regional settings to use the semicolon. After that I saved an Excel file as a CSV and it has semicolons.

You have a few options, but be aware of the potential conflict with commas as decimal characters in numbers.

- if it is vital that the CSV is comma separated, either

-- change the list separator to a comma in the regional settings, then save as CSV and change back, -- OR --

-- save as CSV, open with Notepad and replace each semicolon with a comma.

- if you use the resulting CSV in another application, the application may well have an option to define what the delimiter character is. The Excel import wizard lets you select any character as the delimiter when you import a CSV file.
