Sub Nested_Revisions()
  Dim R As Revision
  Dim Pos() As Long
  Dim i As Long, j As Long, Start As Long, Ende As Long
 
  With ActiveDocument
    .Content.HighlightColorIndex = wdAuto
    ReDim Pos(1 To .Revisions.Count, 1 To 2)
    For i = 1 To .Revisions.Count
      Set R = .Revisions(i)
      Pos(i, 1) = R.Range.Start
      Pos(i, 2) = R.Range.End
      For j = 1 To i - 1
        If (Pos(i, 1) >= Pos(j, 1) And Pos(i, 1) < Pos(j, 2)) Or _
          (Pos(i, 2) > Pos(j, 1) And Pos(i, 2) <= Pos(j, 2)) Then
          'Overlaps
          Start = Pos(i, 1)
          Ende = Pos(i, 2)
          'Get the shortest range
          If Pos(j, 1) > Start Then Start = Pos(j, 1)
          If Pos(j, 2) < Ende Then Ende = Pos(j, 2)
          .Range(Start, Ende).HighlightColorIndex = wdBrightGreen
        End If
      Next
    Next
  End With
End Sub

Can smartphones get viruses and spread them to the network?

Since a smartphone will typically be automatically mounted as a USB drive by the PC when attached via the power cable, you need to avoid making this type of connection the default via the PC Autorun ability and instead only perform this action manually at your request.

daniel pineault say:

You break your databases into 2 components:

  • the Back-End (BE) which houses your tables
  • the Front-End (FE) which houses everything else: Queries, Forms, Reports, Macros, VBA

You place the BE on a central server to which all your users can access (Read/Write/Delete).

You Relink the FE to the new location of the BE

You distribute a copy of the FE to each user that they locate on their PC (Desktop, Documents, ... wherever they would like).  Each user need to have their own copy, do not share a common copy between user as this can cause corruption.

Whenever a user Inserts/Edits/Deletes records through the FE, they are in fact inserting/editing/deleting the data contained in the BE through the linked tables.  Thus, when you interact with the BE as the administrator, you are actually interacting with the live data from your users.

Backing up the BE is a crucial aspect of database administration.  Depending on your business, and IT, setup this can easily be automated as can Compact and Repair of the BE to ensure optimum performance.

Lastly, with a split database (which they all should be) the improve performance you are highly advised to setup a persistent connection between the FE and the BE.

      ol-black.css

4.   Open Internet Explorer and go to your inbox at http://mail.live.com
      The message list will probably look something like this:

5.   Click the Internet Explorer settings cog in the top right-hand corner of the window
      (not the identical outlook.com one below it). Select Internet options.

6.   Click the Accessibility button on the General tab.

7.   Select Format documents using my style sheet and click Browse.

8.   Navigate to the file you saved earlier, select it and click Open.

a. user say:

> but was hoping that someone already had some usable code I could utilize...

Yes, who wouldn't? :-)

Unfortunately, dealing with this in a general way is non-trivial, because it's difficult to open a Mail Merge Main Document using VBA when the data source does not exist, without either having to deal with pop-up dialogs that you cannot reliably dismiss, or without losing the data source details in the process.

There may be code or services out there that can help, but I can't point you to any. I am sure that many organisations have been through exactly the same process, so someone has probably put the effort in to create tools that can at least ease the transition.

In the absence of such code/services, if the number of merge documents that need to be converted is relatively small, it may in fact be easier (although incredibly tedious) to do the conversion by hand.

Even that is non-trivial, at least for certain types of data source, because if the user follows the prompts that allow them to connect to a new Excel source,
 a. they will typically be shown the OLE DB datalink dialog box, which is typically less familiar than the usual file dialog
 b. even after they have chosen a new, valid data source, if they save the document, it is not updated with the new data source information. They have to disconnect the data source and start again.

If you can recreate a situation where your mail merge main documents can find the existing data sources, it does make the business of updating the locations significantly simpler. At the very least, you would be able to extract the relevant information for every Mail Merge Main Document programmatically.

It sounds like database corruption. I suggest moving to a new Identity using the steps in the following link, but with a slight modification. Rebuilding will most likely not fix this type of problem and you'll do the same amount of work and get a stable Identity by moving rather than rebuilding.

Instead of trying to delete all accounts and since it appears you don't have any mail stored under "On My Computer" folders, do a File > Export and uncheck Mail but export all other items as .olm file.

How to move your data to a new Identity

Under Outlook in the Menu bar select About Outlook to view version info. Please verify that you are using the latest version of Outlook, v 14.3.6
