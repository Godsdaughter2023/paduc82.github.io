A classic problem, I'm afraid. I suspect that the fault doesn't lie with you or your mail program, but with Mark's Outlook. I think he is sending messages using an encoding method (RTF) that is intended only for other Outlook users. An attachment to a message sent from Office Outlook using that program's proprietary RTF format will be encoded in such a way that it requires special software to decode it. To avoid this, ask the sender to resend not using RTF.

An alternative solution may be to access the message in webmail and download the attachment from there; some webmail services (e.g. Gmail and Outlook.com) do have the special software required.

This is all explained in excruciating detail in this article: How e-mail message formats affect Internet e-mail messages in Outlook. Diane Poremsky's excellent Slipstick site also has an explanation and suggested remedies: Outlook is Sending Winmail.dat Attachments
