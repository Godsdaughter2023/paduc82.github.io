Running printui /s /t2  will show the installed printer drivers (among other things).

 

What would be strange if a driver for some particular printer WAS listed there but Windows kept trying to install it anyway.

 

You say that the printer was used in the past but is not now connected.  How was it uninstalled (if it was uninstalled)?

 

If you don't see an odd entry in the Startup tab of msconfig (as suggested by John Ado), you can do a more through look by using Autoruns:  https://docs.microsoft.com/en-us/sysinternals/downloads/autoruns  After the Autoruns scan completes, select the "Logon" tab and then closely examine the "Image Path" column.

 

You should also make sure to empty the Temp folder.  Click Start, type %temp% into the Search box, and press Enter.  Select all (press CTRL+A) and then press the DELete key.  If you get any popups complaining about deleting a file, just click "Skip" and delete as many as you can.

 

If all else fails, you should consider running the script behind the two now-retired Microsoft "FixIt" routines described here:  https://blogs.technet.microsoft.com/askperf/2012/02/24/microsoft-fixit-for-printing/

 

Running the "Light" mode should fix things, but if not, you can run the "Full" mode, which essentially resets the printing subsystem to its default state.  Doing the latter requires a computer restart and will remove all printers other than the Microsoft XPS Writer and Microsoft Fax (if installed).  You'll have to reinstall any printers you want to use.

Because the FixIt routines have been "retired" by Microsoft, you need to run the PowerShell script that powered those routines.   PowerShell 2.0 is built into Windows 7 and is sufficient for this task.

  1. Go to https://github.com/jdickson289/Print-Reset-Tool/ and download print-reset.ps1 and make a note of where you save the file
  2. Click Start, type PowerShell into the Search box, right-click on the link to Windows PowerShell, select "Run as administrator" and respond as needed to the UAC prompt 
  3. Type the following command into the PowerShell window and press Enter
    Set-ExecutionPolicy Unrestricted
  4. In response to the "security risks" question, type Y and press Enter
  5. Type the following command into the PowerShell window and press Enter.  In typing the command, use the full path of the location where you saved the print-reset.ps1 file.  In the example shown below, print-reset.ps1 was saved in C:\Temp
      C:\Temp\print-reset.ps1 -light
  6. Type R to run the script

This is what the PowerShell window should look like as you proceed through steps 3 through 6 (although the screenshot shows use of the -full switch rather than -light and I didn't enter R because I didn't want to reset my own printing system).  Don't forget to use the actual location where you saved the print-reset.ps1 file rather than C:\Temp as shown below.

Note that in step 5 if you omit the -light switch and run the script with no switches, you'll get a complete description of what the script does.  See the screenshot below.

If running with -light doesn't fix the problem, run the script again using -full.  Just realize that you'll have to reboot after the script completes and you should have all the necessary installer files for your printers handy.
