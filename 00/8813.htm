jeffrey min say: I agree that the "once a day" setting is very vague and maybe could be designed better.

The account will be synchronized every 24 hours based on the last time it was synchronized.  So if you create the account at 9AM and it finishes synchronizing at 9:05AM then the next time an automatic send/receive will take place will be at 9:05AM the next day.  If, prior to that, you kick off a manual send/receive at 3PM, then the next automatic send/receive will take place at 3PM the next day, and so on.

Here are some Sshots and the detail of the names.

Last_name Cobox  [Combo0]

Company Cobox [Combo1]

ken sheridan say: As the combo box draws its list from the same table as the subform, to filter the subform you should make the bound column of each combo box the column being referenced, not the key of the table:

1. Change the RowSource property of Combo0 to the following.  Note the use of the DISTINCT predicate as names can legitimately be duplicated:

SELECT DISTINCT last_name FROM phones ORDER BY last_name;

2.  Change its ColumnCount property to 1, delete the entry in the ColumnWidths property, and set its ListWidth property to Auto.

3. Change the RowSource property of Combo 2 to the following.  In this case it's assumed that company names are distinct:

SELECT Company FROM phones ORDER BY Company.  

4.  Change its ColumnCount property to 1, delete the entry in the ColumnWidths property, and set its ListWidth property to Auto.

5.  In the AfterUpdate event procedure of each combo box requery the subform control, i.e. the control in the parent form's Controls collection which houses the subform, with:

    Me.[list subform].Requery

6.  Change the RecordSource property of the subform to:

SELECT *
FROM Phones
WHERE (Company=Forms!Mainform!Combo2
OR Forms!Mainform!Combo2 IS NULL)
AND (Last_Name=Forms!Mainform!Combo0
OR Forms!Mainform!Combo0 IS NULL)
ORDER BY Phones.Company, Phones.Last_Name;

The above is written in SQL view, not in query design view.  If you do the latter Access will move things around.  With only a few parameters like this the underlying logic will be unaffected, but will be less apparent.  With more complex queries it is possible that the changes made when design view is used could result in the query being too complex to open.
