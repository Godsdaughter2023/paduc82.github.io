As stated - the right approach to permanently save data locally that has been deleted "server side" is to copy the folders to a separate data store (create a PST file). Storing data in an .ost file that no longer resides on the server is just asking for problems. If an .ost file is separated from the email account/Outlook profile used to create it - it will no longer be accessible without using a 3rd party OST2PST conversion utility.

Just be very aware of something - backing up (or more specifically, making a copy of) an .ost file is not going to serve any purpose long term and is definitely not the right approach to use.
