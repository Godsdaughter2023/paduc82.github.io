tricky300 say:

Hi,

Some ISPs will not permit you to send via their server if you are connected via a different ISP. Sometimes a solution to this problem is to use outgoing authentication (user name and password)...

  • Click the Tools dropdown menu then click Account Settings...
  • Click the E-mail tab and highlight (click) the account that you want to provide authentication for then click Change...
  • Click the More Settings... button
  • Click the Outgoing Server tab
  • Tick the box for My outgoing server (SMTP) requires authentication and select Use same settings as my incoming mail server
  • Click OK the click Next then click Finish

Repeat for all your e-mail accounts. This assumes that you are set up to use a different SMTP server for each e-mail address, i.e. the one that would normally be assocated with that e-mail address.

Alternatively, you might be able to use the same SMTP server for all e-mail addresses in which case you would need to specify the user name and password for that SMTP server.

For example, if you had two e-mail addresses, say abc @ xyz . com and abc @ 123 . com you could have servers pop3.xyz.com and smtp.xyz.com for the first e-mail address and pop3.123.com and smtp.123.com for the second e-mail address, both using the same settings as the incoming server.

Alternatively, you might for example, be able use smtp.abc.com for both e-mail addresses using the same as the incoming server for the first e-mail address but specifying the pop3.abc.com username/password for the second e-mail address.
