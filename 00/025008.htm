max, singapore say:

.. > As I have formula's that reference all worksheets, I need my formula's to also update with the new worksheet name

To quickly "update" the latest list of sheetnames, 
you could run this sub below to list all sheetnames starting in A1 down
(try running it in a new sheet first, then simply copy n paste over to overwrite the previous list of sheetnames that you have)

Sub ListSheetNames()
    Dim wkSht As Worksheet
    Range("A1").Select
    For Each wkSht In Worksheets
       Selection = wkSht.Name
       ActiveCell.Offset(rowOffset:=1, columnOffset:=0).Activate
    Next wkSht
End Sub

max, singapore say:

Here's some easy steps to groove you into installing and running the sub ...

In Excel,
Press Alt+F11 to go to VBE
In VBE,

Press Alt+Q to get back to Excel

In Excel, insert a new sheet
Press Alt+F8 to surface the macro dialog
Double click on "ListSheetNames" (or select it, then click Run button)
The sub will list all the "current" sheetnames starting in A1 down

john - project say:

Barry,

The reason only one row is getting font characteristics set is because the short form code Rod provided and your code for "Color_red" are running in different modes. Rod's code is background code while the font coloring code is foreground code. Background code does not require any selection of the active view elements to do it's thing while foreground code does. So even though Rod's code loops through all task objects in the file, the "SelectRow" statement only selects the current active cell, over and over.

Unfortunately unlike Excel VBA , Project VBA can only operate on font characteristics (e.g. color, bold, etc.) in the foreground. You would need to tell the macro which row in the active view to select and to keep it interesting, the row callout in the SelectRow Method is NOT the task ID. Rather the row designator is the physical row number in the current view. If the current view is fully exploded, unfiltered and ungrouped, the row number and task ID will be in lock step, offset by 1 since there is no ID "0" (unless you display the Project Summary Task). And here you thought this was going to be simple, right?

However, rather than trying to synchronize row numbers with task ID let me suggest a simpler approach, one that I have used many times. Go ahead and set Flag2 using Rod's code. Then create a filter to isolate all tasks with the flag set. SelectAll and do your font formatting. Remove the filter and boom! you're done.

John
