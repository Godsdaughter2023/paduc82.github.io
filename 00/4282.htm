Runtime Broker is using too much memory

-Runtime Broker is a Windows process in Task Manager that helps manage permissions on your PC for apps from Windows Store. It should only use a few megabytes of memory, but in some cases, a faulty app might cause Runtime Broker to use up to a gigabyte of RAM or more.

- If your RAM use is high and your PC is running slowly, an app may be the cause of the problem. Press Ctrl+Shift+Esc to open Task Manager and then, on the Processes tab, check to see how much memory Runtime Broker is using. If it’s using more than 15% of your memory, you probably have an issue with an app on your PC. To stop Runtime Broker from using so much memory, select Runtime Broker in the list, select End task to close Runtime Broker, and then restart your computer.

If the user enters a value <500 in Total Paid, then I want the Strata field to populate with <500. 

If you enter a value between 500-2500 in Total Paid, then I want the Strata field to populate with 500-2500. 

If you enter a value >2500 in Total Paid, then I want the Strata field to populate with 2500. 

Any help would be appreciated.  Thank you in advance.

[Moved from Access/ Windows 10/ Office 365 for Business]

hansv mvp say:
Do you want the user to be able to edit the Strata field, or should it strictly depend on the value of Total Paid?

scacci say: The Strata field will not be edited. It will depend on the value of the Total Paid field. Thank you!

hansv mvp say:

In that case, you don't need a Strata field in the table. You can compute it in a query or on the form itself.

On the form: set the Control Source of the Strata text box to

=IIf(IsNull([Total Paid]), Null, IIf([Total Paid]<500, "<500", IIf([Total Paid]<2500, "500-2500", "2500")))

hansv mvp say:

You could create a Before Update event procedure for the form:

Private Sub Form_BeforeUpdate(Cancel As Integer)
    If Me![Error] = 1 And IsNull(Me![Error Reason]) Then
        Me![Error Reason].SetFocus
        MsgBox "You must supply a reason for the error!", vbExclamation
        Cancel = True
    End If
End Sub
