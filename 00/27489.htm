rob koch say:

You've confused yourself by concentrating on the fact that both the virus and the file that contains it are "programs" or more specifically, "executable code".

If you simply examine the portions of the Microsoft Security Essentials settings that relate to allowing files manually in advance, these exclusion settings for files, locations, file types and processes are all selecting the items which are being scanned or monitored for malicious activity.

On the other hand, the detections themselves occur using definitions containing signatures, which are really just a database of known snippets of code, activity or behavior that have been identified as [potentially] malicious in the past.

When blocking, removing or allowing an item identified as possible malware, the offering includes identification of the potential malicious item via path for a file or other discerning characteristics.  The named malware associated with this item or activity is based upon the definition information used to identify it.

Nothing within any of these descriptions allows for the selection of an item by it's definition or signature, they instead are always selected via the criteria that identify the file's location, type or process name.  So all of this implies that it is the item which contains the potential malicious item that is allowed, rather than the malicious portions themselves.

Rob

COUNTIF doesn't accept a 3D range. You could use SUMPRODUCT combined with COUNTIF, or add three COUNTIFS to capture each sheet.

Your formula could be like this:

=COUNTIF('October '!E8:M8,"P")+COUNTIF(November!E8:M8,"P")+COUNTIF(December!E8:M8,"P")

Also, based on your formula above, your October sheet has a space after the name - not sure if you intended that or not.
