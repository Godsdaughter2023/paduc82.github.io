Make Table Query Not Working - Microsoft Community

jasonjacobs1 say:

Make Table Query Not Working

Hey All,

In my database I have 3 table that contain room conditions data. The device that collects this data stores it in 3 separate text files, all formatted identically and all named the same. Each file is named by the date it was created. I have created 3 tables in the database (Zone1, Zone2, Zone3) and made an import macro that gets the data from the text files and drops it into the appropriate tables.

What I'm trying to do is use a Make Table query that pulls the data I want from each of the tables, and combines it into one table (Conditions), So I can use that table to create a chart and reports.

In each table are Date, Time, Zone X Temp, Zone X Humidity.  (X being whichever zone it belongs to).

The system records the values at 10 minute intervals. So multiple values list per day. Access sets primary autonumber key.

I need a table that shows the Temp and Humidity for each zone, for the given date and time.

I can make the query, and it runs fine. Click view and I get my compiled list of Dates, Times, and the various zone data for said date and time. But when I change this query to Make Table, access does the 'not responding thing' for a bit, and when it finally comes back gives me an error saying it can't connect to the database.

In my limited research, I've seen it suggested that the database might be to large (2Gb limit apparently). I dismissed this as an option at first because this is a very simply database, and it's new. Only 7 days worth of data stored there. After failing to find a solution, I decided to watch the file size before and after I tried the 'Make Table'. Suprisingly, the file size before was 3.2Kb.  After the 'Make Table' it jumps to 2Gb!!

I'm a novice with Access, and my gut says I'm missing some simple step in this to be able to merge 3 tables into one.

Can anyone tell me what I'm missing here? Or maybe a better way to do this? Please note: I have no control over how the device records this data.

jasonjacobs1 say:

Tom,

I got it. Figured out how to JOIN them and it works properly now. Below is the SQL for the query.  Thanks for the help!

SELECT [Zone 1].Date, [Zone 1].Time, [Zone 1].[Zone Temp 1] AS [Zone 1 °F], [Zone 1].[Zone Humid 1] AS [Zone 1 %Rh], [Zone 2].[Zone Temp 2] AS [Zone 2 °F], [Zone 2].[Zone Humid 2] AS [Zone 2 %Rh], ([Zone 1]![Dewpoint 1]+[Zone 2]![Dewpoint 1]+[Zone 3]![Dewpoint 1])/3 AS [AVG DP] INTO Conditions
FROM ([Zone 1] INNER JOIN [Zone 2] ON ([Zone 1].Time = [Zone 2].Time) AND ([Zone 1].Date = [Zone 2].Date)) INNER JOIN [Zone 3] ON ([Zone 2].Time = [Zone 3].Time) AND ([Zone 2].Date = [Zone 3].Date);

Hi Jeremy,

For the Insert tab to appear when replying to an email or forwarding an email, do the following:

  1. Double-click the email you want to reply or forward.
  2. After double-clicking the email, the email will open in a new window. On that window, click reply or forward.
  3. After clicking reply or forward, the Insert tab will appear.

If you do not access the email in a new window (by double-clicking the email), and select the reply or forward right away, you are performing quick reply or quick forward. In quick reply or quick forward, you can only see the basic settings like changing the font size, changing the font color, or attaching a file. For you to see all of the tabs and options whenever you want to reply or forward an email, you need to access the email itself (by double-clicking the email).

The way Outlook search strings work in rules is that all punctuation is treated equally as string terminators. It doesn't matter if you were to use a plus, a minus, a space, an asterisk, or any other special character. Outlook considers them all as string terminators and does not include them in the comparisons.
