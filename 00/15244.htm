Assuming that you are using replication, the data in the replica will not affect the data in the master until such time as you synchronize the two.

If you mean you made a copy of an existing database, then changing data in the copy will not affect the data in the original.  That assumes that you made a copy of the database that contains the tables. 

If you are using the strategy of a front end with all the forms, reports, code, macros and a backend with just the tables, then if you copied the front end and did not change the table links you will change the data in the backend.

You should keep Drafts, Sent folder and Trash - they sync with the gmail server - drafts and deleted items go into those folder and sent items sync down (outlook doesn't save sent items in gmail accounts because gmail saves them automatically), but not important and starred or others. The IMAP folders in Outlook are labels in Gmail.  The [Gmail] folders are the defaults that Gmail creates - the folders at the same level as the Inbox are Gmail labels.  Important and Starred are new folders in Gmail for things you marked as starred or important using the web browser.
