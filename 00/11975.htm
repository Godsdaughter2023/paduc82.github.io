Here are the steps I took, in case it helps others.

  1. Removed .NET Framework 3.5 as mentioned in the release notes and other posts.

  2. Reboot. However, when I rebooted to finish the removal, Windows Update had already gotten Build 16226 ready enough that it wanted to reboot to attempt the install again. So I let it try.

  5. When it asked if I wanted it to delete all currently downloaded Windows Updates so that it could re-download it all, I said Yes.

  6. I also said Yes to all other repair questions that it asked so that it would do it all.

  7. Then I rebooted, brought up Windows Update again, and let it download Build 16226 from scratch. I told it to Install and sat here and waited for it to painstakingly download it all and go through the install step in the Windows Update window (Downlevel stage?).
