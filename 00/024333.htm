ken sheridan say: To express the logic required for making multiple parameters optional in query design view is rather more complex than the way you have attempted.  Moreover, it is not really practical to do so in query design view as, even if you get the logic right, Access will move things around significantly on saving the query.  At best the logic will be obscured, but the resulting query might well be too complex to open.  It should be done in SQL view therefore, and it is important that the query is saved in SQL view, not switched to design view before doing so.  The logic for the query's WHERE clause is very simple:

WHERE (SomeField = Forms!SomeForm!SomeControl
OR Forms!SomeForm!SomeControl IS NULL)
AND (SomeOtherField = Forms!SomeForm!SomeOtherControl
OR Forms!SomeForm!SomeOtherControl IS NULL)

As many controls as necessary can be referenced in this way simply by adding additional AND operations in which the OR operation is parenthesized as above to force it to evaluate independently.

However, it seems that you are not in fact testing for equality, but attempting to use a set of BETWEEN….AND operations, which presumably each reference two controls as parameters, so the WHERE clause would be like this:

WHERE (SomeField BETWEEN Forms!SomeForm!SomeControl1
AND Forms!SomeForm!SomeOtherControl1
OR Forms!SomeForm!SomeControl1 IS NULL
OR Forms!SomeForm!SomeOtherControl1 IS NULL)
AND (SomeOtherField BETWEEN Forms!SomeForm!SomeControl2
AND Forms!SomeForm!SomeOtherControl2
OR Forms!SomeForm!SomeControl2 IS NULL
OR Forms!SomeForm!SomeOtherControl2 IS NULL)

This will require both parameters per BETWEEN….AND operation to have values to implement the restriction.  If, on the other hand, you want to allow for one or the other parameter per BETWEEN….AND operation to be left Null, i.e. for each range to be open ended in either or both directions, then you would need to do so like this:

WHERE (SomeField >= Forms!SomeForm!SomeControl1
OR Forms!SomeForm!SomeControl1 IS NULL)
AND (SomeField <= Forms!SomeForm!SomeOtherControl1
OR Forms!SomeForm!SomeOtherControl1 IS NULL)
AND (SomeOtherField >= Forms!SomeForm!SomeControl2
OR Forms!SomeForm!SomeControl2 IS NULL)
AND (SomeOtherField <= Forms!SomeForm!SomeOtherControl2
OR Forms!SomeForm!SomeOtherControl2 IS NULL)

Of greater importance than this issue, however, is the fundamental point identified by Scott that data is being encoded as table names.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.  The correct way to model this would be to import all the worksheets into a single table which contains a foreign key column which references the primary key of a table in which each test is represented by a single row.

Use OneNote 2016 to convert audio to text - Microsoft Community

rhea steele say:

Use OneNote 2016 to convert audio to text

Can I use One Note 2016 to import an audio file and translate the file into text?
swischner say:

Wrong.

OneNote (Desktop versions only) includes an audio indexing mechanism, that part is correct, but

  • It cannot transcribe (convert into written text) audio recordings, just point to the location inside a recording where it found a search term you entered in OneNotes search box.
  • It requires the audio recording to be indexed first which can take quite a while (even hours) after the recording. OneNote has to be running for the indexing to take place.
  • It only works on audio recordings you took from inside OneNote (using it's record audio function) and for audio files in the expected format (OneNote 2016 still uses the outdated WMA9 for this)
  • It does NOT work with audio files you imported in a note (even when they are in WMA9) nor with audio you recorded with other versions of OneNote (e.g. on Android, iOS, Mac, even Win 10 app).

This is a "forgotten" function of OneNote which has not been maintained or developed any further since OneNote 2007 (I think) and will most likely not get any attention in the future. Maybe MS is working on something similar, but server based.

I once even tried to "reroute" the audio playback of OneNote through Dragons's speech recognition to perform a transcription...no success.

Maybe Microsoft should think about something, at least the speech recognition is there in Win 10 (Cortana, APIs for Microsoft's server based "AI" services...)
