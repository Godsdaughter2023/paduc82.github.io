What you are talking about is called an audit trail. There are many examples of audit trails out there including one in my blog that works using data macros. 

ken sheridan say: You'll find an example of logging changes to a table in ChangedRecord.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the logging simply records the date/time of the insertion of a new row into a table and by whom the row was inserted/changed, but you could easily amend it to store the complete data from the changed records in a history table as follows:

1.  Create a History table which mirrors the structure of the Contacts table, but with the ContactID column as a straightforward long integer data type rather than an autonumber, and make its primary key a composite one of the ContactID  and DateTimeStamp columns.

2.  Amend the code in the frmUpdateLog form's module as follows:

Private Sub Form_AfterInsert()

    Dim dbs As DAO.Database
    Dim strSQL As String

    Set dbs = CurrentDb
    
    ' insert row into UpDateLog table
    strSQL = "INSERT INTO UpdateLog(ContactID,DateTimeStamp,UpdatedBy) " & _
        "VALUES(" & Me.ContactID & ", #" & Format(Me.DateTimeStamp, "yyyy-mm-dd hh:nn:ss") & _
        "#,""" & GetUser() & """)"
    dbs.Execute strSQL, dbFailOnError
           
    ' insert row into History table
    strSQL = "INSERT INTO History " & _
        "SELECT * FROM Contacts WHERE ContactID = " & Me.ContactID
    dbs.Execute strSQL, dbFailOnError

End Sub


Private Sub Form_AfterUpdate()

 On Error GoTo Err_Handler

    Dim dbs As DAO.Database, rst As DAO.Recordset
    Dim strSQL As String
       
    strSQL = "SELECT * FROM Contacts WHERE ContactID = " & Me.ContactID
       
    Set dbs = CurrentDb
    Set rst = dbs.OpenRecordset(strSQL)
    
    StoreNewVals rst
       
    If Not blnNewRecord Then
        If RecordHasChanged() Then
            MsgBox "Record has changed"
            ' insert values into DateTimeStamp and UpdateBy columns
            strSQL = "UPDATE Contacts SET DateTimeStamp = #" & _
                Format(Now(), "yyyy-mm-dd hh:nn:ss") & _
               "#, UpdatedBy = """ & GetUser() & _
               """ WHERE ContactID = " & Me.ContactID
            dbs.Execute strSQL, dbFailOnError
            ' insert row into History table
            strSQL = "INSERT INTO History " & _
                "SELECT * FROM Contacts WHERE ContactID = " & Me.ContactID
            dbs.Execute strSQL, dbFailOnError
        End If
    End If
    
    blnNewRecord = False
    
Exit_Here:
   Exit Sub
   
Err_Handler:
   MsgBox Err.Description, vbExclamation, "Error"
   Resume Exit_Here
   
End Sub

The key thing to note about this demo file is that it logs actual changes to data, not merely updates.  A row can be updated without any of the values in its columns changing.

jllatham say:

Assuming your first row with Y/N entries is row 2, and the 7 columns are B:H, then select cell I2 and go into conditional formatting and use a FORMULA to evaluate the condition.  You'll need 2 formulas, first the one for all "N" entries, and type all of this in as the formula to use in the conditional format dialog:
=COUNTIF(B2:H2,"N")=7
and set up your formatting (red?) for all N's.
Then go back into conditional formatting and add the second formatting formula, for all Y entries:
=COUNTIF(B2:H2,"Y")=7
and set up formatting (green?) for all Y's.

Set up the default formatting for column I to be amber, for mixed/incomplete entries.  That should take care of all 3 conditions.

In the Options dialog, I went to Advanced-> Send and Receive, to bring up the Send/Receive Groups dialog.  There was one group, "All Accounts".  Clicking that brings up the "Send/Receive Settings - All Accounts".  Clicking the old Q.com address, I un-checked "Send mail Items", "Receive Mail Items", and finally "Include the selected account...".  (I did not want to delete the account because when I began to do that (through what dialog I don't recall), I got a warning to the effect that all email from that account would disappear as well.  That was a much unwanted side-effect, so I did not continue.)

I closed and reopened Outlook after all changes.  There is nothing on the "outbox" of the Q.com account.

I don't think i could document this more thoroughly.  I hope you can offer an effective solution.

An afterthought: could I edit the Q account to provide server name & pw for the cable account, just so whatever is making the query would be happy and shut up?

doug firr say:

Please read the post.  I worked hard to provide detailed information.

The Q.com account was closed by the ISP because we cancelled their services.  An excellent reason, btw, not to use your ISP's email service.

Up until that time, Outlook worked well with the Q.com mail service.  Understandably, it does not work now.  Incidentally, it is a POP3 server.  The account was not deleted in Outlook because when I began to do so, I got a warning that all mail associated with the account would disappear; I did not want that outcome.

The question is not "Why does the q.com account not work".  The question is "Why does Outlook continue to display a dialog, at approximately 60 second intervals, even though I have configured Outlook NOT to send/receive to/from the Q.com account?"

The answer appear to be this: if there is any mail in the Outbox of a particular account, Outlook will try to send it (or at least log on to a POP3 server) EVEN IF OUTLOOK HAS BEEN CONFIGURED NOT TO SEND/RECIEVE THAT ACCOUNT.  In other words, Outlook does not behave as the configuration specifies.

In addition, on failure OUTLOOK DOES NOT WAIT THE TIME SPECIFIED IN THE CONFIGURATION BEFORE REATTEMPTING.  It waits between 45 and 60 seconds following dismissal of the dialog.  As far as i can tell, that is an unconfigurable delay hardwired into Outlook.

I conclude this because deleting two emails in the outbox of the "inactivated" account made the problem go away.
