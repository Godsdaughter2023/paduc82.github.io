brant gurganus say: The Windows Update drivers are no different than any other drivers. Windows selects the driver based on signing, preferring logo'd drivers to non-logo'd drivers; hardware compatibility preferring specific compatibility to generic compatibility; and versioning, preferring newer versions to older versions. It may also look at driver date in addition. Anyhow, that algorithm of how the drivers are selected is documented, so if Windows is picking a driver that does not work, it is not generally Windows at fault but the driver manufacturer. I've investigated a few cases of Windows Update-distributed drivers causing an issue, and it has generally been the working driver not specifying the hardware specifically, so Windows sees correctly a newer driver that is just as specific to the hardware and chooses to install it. What the manufacturer should do in those situations is create a new version of the working driver for that hardware and package it with a more specific hardware ID for the compatible har

ossiemac say:

Before entering the dates use NumberFormat and format the column as Text and then enter the dates with just the day and month. Excel then sees what you enter as text and will save it as text in the csv file.

If you already have the dates and need to convert them then insert a column immediately to the right of the dates (but do NOT use NumberFormat) and then use the following formula to convert the dates to text. (Assuming you have column headers and are commencing on row 2 and the dates are in column D)

=TEXT(D2,"dd/mm")

Copy the formula down and then select the column and Copy -> Paste Special -> Values

You should now be able to delete your original column D of dates and will be left with text dates in dd/mm format in the column and it will save that way in a csv file.
