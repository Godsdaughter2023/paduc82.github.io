Hello Mike,

 

I changed my default file association for .pdf to use Noted and went through a number of tests. I found that a .pdf hyperlink in Access, Excel and Word will open in Adobe Reader. The hyperlink from Outlook opens in Notepad.

 

Using Process Monitor, I observed the registry calls from Access and noted this sequence which ends with locating the CLSID for Adobe Reader and subsequently launching it.

 

HKCR\.pdf\(Default)                                        AcroExch.Document

HKCR\AcroExch.Document\CLSID\(Default)   {B801CA65-A1FC-11D0-85AD-444553540000}

 

Again with Process Monitor, I observed the calls for Outlook which appears to locate Notepad by calling a current user hive for explorer.

 

HKCR\.pdf\(Default)                                                                                                                            AcroExch.Document

HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.PDF\UserChoice\Progid            Applications\notepad.exe

 

I don't know the specifics of why these paths are taken but it appears that Outlook is treating the process more like using Windows Explorer and the other Office applications locate and rely upon the application associated with the CLSID for that type of object.

 

The only suggestion I would have is to use VBA code with the shell command to open the file and it will then use the default file association to open it.

 

Public Function OpenPDF(strFile)

    Shell ("CMD.exe /c" & Chr(34) & strFile & Chr(34)), vbNormalFocus

End Function

a. user say:

This feature is usually controlled by a Windows Registry setting that is shared by the Office programs. In the registry, the Value is called

HKEY_CURRENT_USER\Software\Microsoft\Office\11.0\Common\Toolbars\AdaptiveMenus

The value should be set to 0 if you want the full menus.

If your system is maintained in a network environment - or perhaps was originally set up by someone else, there could also be a "Grooup Policy" Setting at

HKEY_CURRENT_USER\Software\Policies\Microsoft\Office\11.0\Common\Toolbars\AdaptiveMenus

If this value exists, and is set to 1, the system will work the way you describe - i.e. the setting will work until you (say) close Word and restart it. Then Word restores the setting from the Policy setting - or something like that.

I don't have a good link for "how to modify the registry", but here is /a/ link:

http://support.microsoft.com/kb/256986

If that is not the problem, there are other possibilities, for example...

 a. you might not have write access to the relevant part of the registry (unlikely IMO, unless you are on a network)

 b. an addin might be resetting the registry entry

 c. Word (for example) only writes the updated registry entry when it exits, not when you make the change. So if Word is not closing correctly, that might cause this behaviour. But IMO that is also unlikely, and in all probability you would be able to change the setting in /one/ of the Office programs.

Hi Barry,

Yes, Office has its own builtin fonts that are not available on the system through Font Book.

It is not feasible to delete these fonts such as Abadi MT and Angsana New as you mentioned.

But you can delete other styles of a font.

When you put the cursor on the font, you can see some extra styles under the font. For example:

1. Abadi MT font: You can delete the Condensed Light and Condensed Extra Bold styles.

2. Angsana New font: You can delete the Regular, Italic, Bold and Bold Italic styles.

Following are the steps to delete them:

1. Finder> Applications> Right click Microsoft Word (or other Office applications)> Show Package Contents> Contents> Resources> Fonts.
2. In the Fonts folder, find and delete following six .ttf files, close Word application and reopen it, you will see that these styles disappear.

Abadi MT Condensed Light: AbadiMTCondensedLight.ttf

Abadi MT Condensed Extra Bold: AbadiMTCondensedExtraBold.ttf

Angsana New Regular: angsa.ttf

Angsana New Italic: angsai.ttf

Angsana New Bold: angsab.ttf

Try the simplified code below.

Note:
The Change event does not fire if a cell contains a formula!

Andreas. 

Option Explicit
Option Compare Text
Private Sub Worksheet_Change(ByVal Target As Range)
  Dim State As Boolean
  Select Case Target.Address(0, 0)
    Case "D21"
      State = Target.Value = "Yes"
      Range("24:38").EntireRow.Hidden = State
      Range("40:54").EntireRow.Hidden = Not State
    Case "AH2"
      State = Target.Value = "Yes"
      Range("AJ:AM").EntireColumn.Hidden = State
      Range("AO:AR").EntireColumn.Hidden = Not State
  End Select
End Sub
