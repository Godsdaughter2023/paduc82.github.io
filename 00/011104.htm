hansv mvp say:

In the Anniversary Update to Windows 10, Microsoft inadvertently increased the width of digits in Calibri 11, the default font for recent versions of Excel. As a result, the default column width of columns increased too.

The cumulative update for Windows 10 version 1607 released on 25 or 26 October 2016 (KB3197954) restores digits in Calibri 11 to their original width, hence reducing the default column width.

Pictures that have "Move and size with cells" set will have decreased in width too. Pictures that have "Move but don't size with cells" or "Don't move or size with cells" should not be affected.

If you have a large number of pictures, you might create a macro to increase their width:

Sub ResizePictures()
    Dim wsh As Worksheet
    Dim shp As Shape
    Application.ScreenUpdating = False
    For Each wsh In Worksheets
        For Each shp In wsh.Shapes
            If (shp.Type = msoLinkedPicture Or shp.Type = msoPicture) _
                    And shp.Placement = xlMoveAndSize Then
                shp.LockAspectRatio = False
                shp.Width = Application.CentimetersToPoints(3.6)
                shp.Placement = xlMove
            End If
        Next shp
    Next wsh
    Application.ScreenUpdating = True
End Sub

As far as the original question that Terry had, it looks like you may be looking in the wrong spot for the drivers. On a 64 bit machine, by default, when you open the ODBC administrator it will open the 64 bit version of it. This will then only display the 64 bit drivers on the machine. In order to see the 32 bit drivers on a 64 bit machine, you would need to launch the 32 bit ODBC administrator window by doing the following:

+>Double click on odbcad32.exe

+>Select the driver you want to use and create the DSN.
