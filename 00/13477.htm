trevor rabey say:

Carlos,

the plan is not a one off exercise. It evolves throughout the planning period and all the way on through the duration of the project.

You want to preserve the history of the development of the plan.

You may never need to refer back to any of it. But in construction there is often the potential for disputes or even litigation, and if you have the record of whatt he plan was

Sure, you end up with 200 files. So what? They all fit onto one memory stick or backup hard disk. The only problem is being careful to come up with a file naming system which clearly identifies them, and then be careful to not edit them. Write protection is a good idea.

As well as all of the archived files and the most current best one, you may also have a need for other versions from time to time to try out "what if?" scenarios. Plan A, Plan B, Plan C etc. For example, you may want to see the effect of increasing the duration estimate of every task or some selected tasks by 2 days. You make the change and save the file because having the answer you don't want to lose it. Sure, you could print it out or save it as a PDF, but that never captures the whole story.

Analysis and optimisation starts with asking questions such as "what if this?" or "what if that?".

Imagine a time before we had such tools. People simply did not have the luxury of being able to have different plans in order to compare them or improve them. The calculations are so much work and so time consuming. They had to have one plan and go with that.

Now the calculations are instantaneous. We just have to build the network model. We have so much opportunity to use the tools and make so much better plans. Unfortunately, many people do not recognise this opportunity.

Another reason for suggesting you get help from AKO is that ordinary people here, civilians, many not even US citizens, can not try to replicate whatever it is that you are seeing, at least without getting intimidating error messages.

However, if your problem really represents an ordinary slow DNS case I may be able to help.   E.g. my procedure for testing/working around such cases is:

  1. do an  ipconfig  /flushdns  before starting to the  Networking troubleshooter, e.g. before clicking on the  Diagnose connection problems  button in your error message.
  2. Use the troubleshooter and after it tells you it couldn't find anything wrong (but before clicking OK to close it), do an  ipconfig  /displaydns
  3. if your host name is represented there only by an alias, e.g. a CNAME record,   ping -n 1 -w 1  the CNAME.
  4. Keep repeating 2 and 3 until you have a demonstrably fully cached lookup (and enough time left in their Time To Live values to let IE try to use them).
  5. Only then click OK to close the troubleshooter and allow an automatic refresh to occur on the problem request with IE.
  6. If the first request would have resulted in a redirect with its own set of alias and CNAME records you may have to anticipate that before doing step 5 (e.g. do steps 2 to 5 for them too.)

Finally, since you are using  IE9  you could have its  Developer Tools, Network Capture tool active to try to capture more details to help refine your symptom description.   However, that will only help if you are not having a problem with the first lookup.  I can't tell from what you have given us so far where your problem cause would be.   The DNS lookup/redirect issue is the mostly like cause, so let's make sure you have eliminated it first before going on to try to analyze your symptoms further.

charles kenyon say:
You should Not put any text (including a header or footer) in the Normal template. Instead you should create a separate template that has your text and use that to create new documents. The only exception to this blanket statement would be if you never want to use Word to create labels or envelopes.

That said, any text in the normal.dotm template should be in any new documents. Unless you have multiple users or multiple versions of Word, there should only be one normal template on your computer. It is quite possible, though, to have multiple files with that name.

I would suggest finding all of them and renaming them. The default location for the normal.dotm template is a hidden system file that is not searched (by default) by Windows. Here is how to find the one that Word thinks of as its normal.dotm template:
Normal Template in Microsoft Word - How to Open

Finally, to nit-pick, you do not open a new document but create one. You open an existing document. No text you put in normal.dotm will show up in an existing document that you open.
