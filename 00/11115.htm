1) What is the purpose of GoToControl here? I'm not editing a field or anything in this macro, I'm just running a MenuCommands later on in the first three If statements. Is this just guaranteeing that the [Form] object later on will have the correct scope, and since we know that the button pressed (the GoToControl simply goes to the deleteRecord button that was pressed to run this macro in the first place...) was on this form by construction, its a guaranteed way to set [Form] to the current form?

Bear in mind that this is a canned macro that might be used in a number of different situations.  So long as the macro is called from the form whose record you want to delete, you would not need GoToControl, because that form would always be the context object.  But if the macro is called from some other place -- a command button on an unbound floating toolbar, perhaps, or a main form when the record to be deleted is on a subform -- the focus needs to be placed back on the object that has the record to be deleted.  Setting the focus to the previous control ought to do that (though I can think of circumstances where it wouldn't).

2) The ClearMacroError action right after GoToControl: what is the point of it? Nothing has happened yet, and I can't imagine how GoToControl could throw an error (or perhaps it could?). Is it just clearing any errors that might be queued up before this macro was started?

It is possible that there *is* no previous control, in which case "GoToControl Screen.PreviousControl" will raise an error.  In that case, we would want to ignore the error completely, and not have it show up later on when we test "If [MacroError]<>0".

SharePoint Online – Troubleshooting Sharing and Permissions - Microsoft Community

alexandra craita say:

SharePoint Online – Troubleshooting Sharing and Permissions

Technical Level: 0-100

Description:

SharePoint Online is offering you the possibility to share contents with your co-workers or to collaborate with external users. Due to this, the sharing capabilities include different settings that can be adjusted based on your organization’s needs.

Frequently asked questions

Q: How can the sharing options be adjusted for the entire SharePoint Online environment?

A: If you navigate to the SharePoint Online Admin Center and select from the left menu the “Sharing” option, you will notice four main sharing options, as follows:

  1. Don't allow sharing outside your organization – by selecting this option, the external sharing will be disabled.
  2. Allow sharing only with the external users that already exist in your organization's directory – if you want to only allow access to external users which previously accepted an invitation that you shared for your SharePoint online environment, this is the suitable option to be selected.
  3. Allow users to invite and share with authenticated external users – you will be able to share content with external users, but they will have to go through the authentication process, before being able to access any of the shared contents.
  4. Allow sharing to authenticated external users and using anonymous access links – This sharing options includes the experience from point 3 and additionally you will be able to generate anonymous access links which can be shared with anyone outside the organization.

Based on the sharing option you will choose, the additional settings that will appear for customizing the sharing experience will be different. Let’s take as example the option Allow sharing to authenticated external users and using anonymous access links.

Once selected, the additional options you will see will be very diverse, such as establishing the number of days during which the anonymous link will be valid, who can share with external users or which external domanins should have access to your SharePoint data.

These settings are going to have global impact on the entire SharePoint Online environment.

Q: How can the sharing options be adjusted for a specific site collection?

A: If you external users to have access to a specific site collection, here are the steps that you need to perform:

  • From the SharePoint Online Admin Center, select the site collection for which you want to adapt the sharing options.
  • Enable or disable the external sharing – the available options will be the same with the ones we presented previously, for the entire SharePoint Online environment. 

Q: How can we check which users already have access to the site collection?

A:

  • Click on the sharing option, displayed in the upper-right corner.
  • A new window will be displayed, giving you the possibility to mention the user with whom you want to share the site.
  • Click on Show options to change the default settings, if needed. By doing so, you will be able to:
    • Select whether you want the user to receive an email notification.
    • Assign the desired permission for the user. 

In the same window, you can see the users/groups with whom the site was already shared, by selecting from the left menu the option Shared with.

A list will all the users which have received access to the site will be displayed. If you want to see all the details of the assigned permissions, you can click on Advanced.

Another way to access this information would be to navigate to Site Settings – Site Permissions in the classc sites or click on the wheel - Site permissions – Advanced permission seettings in the modern sites.

Q: Is it possible for a user, which is not the owner for a site collection, to grant permissions to SharePoint content?

A: There is an option, which can allow users that are not owners for the site collection to grant permission to files, folders or sites and sub-sites without requiring owner approval. You can choose to turn off sharing for non-owners on all sites in the current site collection, by clicking directly on the displayed option.

To make the setting reversible, navigate to Site Settings – Site Permissions – Access Request Settings and select all the three options displayed here:

  • Allow members to share the site and individual files and folders.
  • Allow members to invite others to the site members group, <Group Name>. This setting must be enabled to let members share the site.
  • Allow access requests.

Q: Which are the standard SharePoint Online permission levels?

A: Once you access the Site Permissions, you will notice that by default permissions groups are displayed:

  • Members
  • Owners
  • Visitors

The purpose of the above listed groups is only to assign permissions. They will not be email enabled, so once a user will be added to one of the groups, he will automatically receive the permission accoridingly.

More information: https://support.office.com/en-us/article/Understanding-permission-levels-in-SharePoint-87ecbb0e-6550-491a-8826-c075e4859848

Q: How can we customize the permissions for a document library?

A: When it comes to modern libraries, the sharing behavior is similar with the one we discussed regarding the site collection.

A major difference for customizing the permissions for a document library might be the place where you can find the permission settings.

When you click on Documents you will see that the modern view is automatically enabled. In order to find the permissions for the library and adjust them based on the organisation’s needs, click on the wheel - Library Setting – Permissions for this document library.

Newly created libraries (either you have just created a new library, or this is the first time you are opening the default library of the site collection) are inheriting all the permissions from the site collection. This means that if you assign certain permissions to a user or a group, this one will have the same rights when browsing through the library.

More information: https://support.office.com/en-us/article/Edit-and-manage-permissions-for-a-SharePoint-list-or-library-02d770f3-59eb-4910-a608-5f84cc297782

Q: How can we customize the permissions for a modern library?

A:

  • Select the document for which you want to check the permissions.
  • In the right part of the screen, you will see an information pane being displayed automatically.
  • The option Manage Access will allow you to see which users or groups can manage the document.
  • If you scroll at the bottom of the list, you will notice an option called Advanced.
  • Once you click on this one, you will be re-directed to the page where you can manage the entire access for the selected document.
  • The same steps as for the library needs to be performed if you want to break the inheritance of the permissions from the library level to the item level.

Q: How can we enable the external sharing for Office 365 group site collection?

A: Please check the following article: https://support.office.com/en-us/article/Turn-external-sharing-on-or-off-for-SharePoint-Online-6288296a-b6b7-4ea4-b4ed-c297bf833e30#ID0EAABAAA=Office_365_Groups
