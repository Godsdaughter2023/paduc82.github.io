marco_mi. say:

Hi majica(2),

 

Let me help you with your concern regarding “Be careful! This sender failed our fraud detection checks.” error message. Please be informed that this warning usually occurs if the email came from a third party domain. If the sender’s domain does not have SPF record configured/set up in the domain’s registrar, Outlook.com detects the email as potential fraud since it was forwarded from 3rd party IPs.

 

What is SPF Record?

Sender Policy Framework (SPF) is an email validation system designed to prevent email spam by detecting email spoofing, a common vulnerability, by verifying sender IP addresses. It is a security standard that has been implemented to stop forgery of sender address.

SPF allows administrators to specify which hosts are allowed to send mail from a given domain by creating a specific SPF record (or TXT record) in the Domain Name System (DNS).

 

To resolve this, the sender’s domain administrator should add the SPF record in the domain registrar.

john w. vinson say:

Access is telling you it's a number because it IS a number.

The "Lookup Wizard" has been infuriating experienced database developers since the misfeature was first introduced. See http://www.mvps.org/access/lookupfields.htm for a critique. What you see in the table is a "Combo Box" control, displaying the looked up value (the company name); but what's actually stored in the table is a numeric ID, a foreign key pointing to the appropriate record in the lookup table.

The lookup field has one minor advantage - it lets you use table and query datasheets a bit more easily for editing data. The cost for this minor advantage is the sort of confusion you're now experiencing. And a good quality Access application will let the users interact with data in Forms, not in Tables; and it's perfectly easy to put a combo box on a Form to display the company name while storing the company ID.
