If your Help and Support is complaining like this:

Windows cannot open Help and Support because a system service is not running.
To fix this problem, start the service named 'Help and Support'.

There are lots of things that can break Help and Support, but start with easy stuff first.

Look in the Services applet and check to be sure the Help and Support Service is listed, it is set to Startup type of Manual or Automatic and it has a Status of Started. 

To do that, click Start, Run and in the box enter:

services.msc

Click OK to launch the Services Applet and locate the Help and Support Service.

Highlight the Help and Support Service, right click, Properties and set the Startup type to 'Manual' or  'Automatic', click OK to save the changes.

Right click the Help and Support Service again and choose 'Start' and the Status column should change to 'Started'

If that all works out, see if you can get into Help and Support now.

I don't know how that might change your uninstall Silverlight (mine uninstalled just fine) or MSE problem, but you can look in the Event Viewer System log after you see one of those messages and look for Events like this after I disabled my Help and Support Service and tried to use Help and Support (and saw the error above):

Event Type:    Error
Event Source:    DCOM
Event Category:    None
Event ID:    10005
Description:
DCOM got error "The service cannot be started, either because it is disabled or because it has no enabled devices associated with it. " attempting to start the service helpsvc with arguments "" in order to run the server:
{833E4010-AFF7-4AC3-AAC2-9F24C1457BCE}

To see the Event Viewer logs, click Start, Settings, Control Panel, Administrative Tools, Event Viewer.

A shortcut to Event Viewer is to click Start, Run and in the box enter:

%SystemRoot%\system32\eventvwr.msc

Click OK to launch the Event Viewer.

The most interesting logs are usually the Application and System.  Some logs may be almost or completely empty.

Not every event is a problem, some are informational messages that things are working okay and some are warnings.

No event should defy reasonable explanation.

Each event is sorted by Date and Time.  Errors will have red Xs, Warnings will have yellow !s.   Information messages have white 'i's.  Not every Error or Warning event means there is a serious issue.   Some are excusable at startup time when Windows is booting.  Try to find just the events at the date and time around your problem.

If you double click an event, it will open a Properties windows with more information.  On the right are black up and down arrow buttons to scroll through the open events. The third button that looks like two pages on top of each other is used to copy the event details to your Windows clipboard.

When you find an interesting event that occurred around the time of your issue, click the third button under the up and down arrows to copy the details and then you can paste the details (right click, Paste or CTRL-V) the detail text back here for analysis.

To get a fresh start on any Event Viewer log, you can choose to clear the log (backing up the log is offered),  then reproduce your issue, then look at just the events around the time of your issue.

OK.  Since I opened your link on a Windows 10 machine I was directed to:

https://www.microsoft.com/en-us/software-download/windows10, which is the media creation tool.

But looking at your question I think I got sidetracked, so let's see if I can answer everything you asked.

  1. As far as I know the Home and Pro installs can't be used interchangeably.  I actually created an ISO for both because have both kinds of systems.  I could be wrong about this, I certainly have not tested trying it using the "wrong one".  I created my ISOs with the media creation tool and Home vs Pro are different.
  2. There is no system to automatically pick them in the ISO install, the media creation tool on the other hand has a "update this system" option which would pick the right one to download and install.
  3. You can tell what you need by seeing what your existing operating system is.   Basically Home to Home and Pro to Pro.  The table is given here: http://www.microsoft.com/en-us/windows/windows-10-specifications#upgrade under Upgrade editions.
  4. Note that you have to do the first upgrade from the running system (running the old version of Windows).  You run setup, you do not boot to the install media/ISO.  After the initial update you can do a clean install using the install media.  And in that case you would skip the prompts for the product key and when you get to the point where you log into your Microsoft account, Windows will activate automatically.  This is possible because on that first update when it activates automatically it registers the device with the Microsoft register servers.  And when you install for the second time it uses that information to validate activating again.
  5. I don't know what would happen if you used the wrong one.  For instance trying to install a Pro version on an existing Home version.  I'm sure it would block you at one point, which would hopefully be right at the start, but it might go all the way to a point where you are installed, but can't activate.

a_user_here say:

P.S. There is a lot of confusion about the old version's product key, and the new Windows 10 product key for that matter.  The Windows 10 free upgrade never uses them.  It doesn't use the Windows 7/8/8.1 product key.  By requiring the update to be done from within a running system, that system has already validated the old key.

As for the Windows 10 product key.  For the case of the free upgrade everyone is getting the same product key, so it is not useful for installing.  It is the fact that your device has been register during the update, that allows it to be automatically activated when you do another install on the same device.  It activates when you get to the point where you log in with your Microsoft account.

Run a test of your RAM with memtest86+ (I know it is boring and will cost you a CD).

Sometimes, reseating the RAM in the slots will relieve the error but a failure is still cause for suspicion.

joshualutz say:

Is your intent to mirror the Surface to the Dell monitor? If so, the aspect ratios are completely different and one or the other will need to display "black bars" to match up.

The SurfaceBook will need top and bottom black bars, or the Dell monitor will require left and right side black bars.

If instead you want to extend your desktop onto the Dell, you can select that in display settings. Then the Surface keeps its display, and the Dell is an extension of the desktop. Both monitors would allow full screen use at that point.
