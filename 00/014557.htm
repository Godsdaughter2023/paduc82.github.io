#1 - Archive/clean-up your existing 14gb PST file to remove whatever it is that you don't want located on the server

#3 - Organize your existing PST file data to reflect what you want to see in each of the Imap accounts after step #1. Ergo - create the folders (either in a new PST file or the existing PST file) you want to eventually see in each account populated with the applicable data.

#4 - Once step #3 is done, configure your Imap email accounts and create the folders you want in each Imap folder tree

#5 - This steps assumes that nothing exists on the server

a) "copy" (not "move") all the items from each applicable PST based folder to the corresponding Imap folder. (reason for using "copy" is so you don't lose anything should anything go awry)

b) when done, do a manual S/R on each account and let it do it's thing

Backgrounder to copying/moving things easily in case it's of interest (ignore the <import> reference in the title)

How to Copy-Move Folders-Items in Outlook (a PST Import alternative)

#7 - Things to remember about IMAP and OL'2013/'2016

a) When you add your Imap accounts to your profile, if any other email account has already been configured, it will in turn set the "default data file" for the profile which contains your contact/calendar file - important to keep that in the back of your mind.

Understanding Outlook Default Folders versus Default Account
http://www.contactgenie.info/understanding-outlook-default-folder-and-account-differences

b) If you configure OL'2016 only with IMAP accounts and add the PST file you want to copy the data from, one of the IMAP Accounts is going to set as the default data file which in turn will contain the default contact/calendar folders. How to Protect Outlook '2013/'2016 Imap Contact/Calendar Data

http://www.contactgenie.info/how-to-protect-outlook-this-computer-only-data

In your scenario, the objective is to maintain your calendar (and contacts?????) in an Exchange account which would be accomplished by simply adding the EX account keeping in mind that you can only have one data store set to be the "default" for the profile and the "default" applies equally to the contacts folder. Ergo, if you set the EX data file as the default store - it will be for both contacts/calendar.

Make sure you backup any and all PST files before you start which is done by simply making a copy of the necessary PST files (with Outlook closed)

How to Find & Copy the Right Outlook PST Files
http://www.contactgenie.info/how-to-find-right-outlook-pst-files
