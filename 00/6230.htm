j.e. mcgimpsey say: MacBU for some unknown reason left Text Box off the Chart Menu Bar as well as not putting an icon on the Ribbon. They also made it hard, but not impossible, to customize the Chart Menu Bar with the Text Box command.

One way (a bit slow, but it works):

 - select a workbook cell. Choose View/Toolbars/Customize Toolbars & Menus...
 -  in the Customize Toolbars and Menus dialog's Toolbars and Menus pane, create a new toolbar.
  - In the small menu bar underneath the main menu bar, select Insert.
   - Holding the Option key down, drag the Text Box icon to the new toolbar. Click OK to exit the dialog.
    - Select your chart (to activate the Chart Menu rather than the Workbook Menu)
    - Choose View/Toolbars/Customize Toolbars & Menus...
    - Drag the Text Box command from your custom toolbar to the small menu bar's Insert menu.
    - In the Customize Toolbars & Menus dialog, delete the custom menu bar. Click OK.

You should now be able to insert a text box in your Chart. Close XL to save the Toolbar preferences.

Alternatively (and much faster):

- Type Opt-F11 to open the Visual Basic Editor (VBE).
 - Choose View/Immediate Window
  -in the Immediate Window, type or paste this line, followed by the Return key:

       Commandbars("Chart Menu Bar").Controls("Insert").Controls.Add id:=10865

Type  Opt-F11 to exit the VBE and return to XL. Close XL to save the preference.

talking moose say:
When I say "This is not typical behavior for Outlook for Mac" I mean that something else is causing your problem. Outlook for Mac will not exhibit this behavior on a fully updated Mac OS X system where no other software has been installed and where system settings have not been changed.

Two things might cause this to happen:
  1. Something has corrupted your Microsoft User Identity or preferences.
  2. You've introduced a third party piece of software that is conflicting with Outlook for Mac.
The first option is easy to test. Go to Apple menu --> System Preferences... --> Accounts and create a new Mac OS X user account. Log out of your current Mac OS X user account and log in to the new one. Launch Outlook (no need to set up an email account) and open a new mail message. Test the problem. Does it still occur?

If the problem still occurs then you've more than likely installed a third party piece of software that affects all users. Antivirus software would be my first guess but it could be anything from some Finder enhancement software to a new System Preferences pane to some backup software.

If the problem does not occur then you may have corruption in your original Microsoft User Data folder that holds your emails. Or you may have a third party piece of software that you've installed only for your original Mac OS X user account.

You're the only one who can see your computer. No one here can possibly know what you've installed or changed. If you're uncomfortable troubleshooting your own computer then the Genius Bar at a local Apple Store is a good place to go for help. Or check with a local Mac OS X consultant. A second pair of eyes is often helpful when you've been looking at a problem for too long.

Hope this helps!
