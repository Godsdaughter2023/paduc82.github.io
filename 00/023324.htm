tom ogilvy say:

I would take Hiderows out of the worksheet B code module and go into the VBE.  with your workbook as the active project (selected in project explorer on the left), I would go to the menu and insert a module.

I would put this code in that module:

Sub Hiderows()    '<====  Note it is not Private
Dim LastRow As Long, c As Range
Application.EnableEvents = False
With Worksheets("B")   '<================Change "B" to the name of the sheet you

' have been referring to as sheet B

LastRow = .Cells(.Cells.Rows.Count, "E").End(xlUp).Row
On Error Resume Next
For Each c In .Range("D1:D" & LastRow)
    If c.Value = 0 Then
        c.EntireRow.Hidden = True
    ElseIf c.Value = 1 Then
        c.EntireRow.Hidden = False
    ElseIf c.Value = "" Then
        c.EntireRow.Hidden = False
    End If
Next
On Error GoTo 0
End With
Application.EnableEvents = True
End Sub


Now you can put in your change event in the Code Module for the Sheet you call A.  That is where the change activity is taking place

Private Sub Worksheet_Change(ByVal Target As Range)
If Target.Column = 3 Then
Call Hiderows()
End if
End Sub

When you make a change in sheet A, unless you are changing sheet B in reaction with a macro, only a calculate event will occur in sheet B - a change event won't be kicked of.   So putting the change event in Sheet A will and having it call the hiderows macro which has been updated to refer specifically to Sheet B, that should work.

Hi Anna,

Usually, this error occurs if you have a corrupted installer download. To see if this is the case, launch Terminal, change to the folder where Microsoft_Office_2016_15.30.17010700_Installer.pkg exists (probably the ~/Downloads folder if you use Safari), and run the following command:

shasum Microsoft_Office_2016_15.30.17010700_Installer.pkg

After 20 seconds or so, you should see the following file hash: 9d2c3454329cb6e2bb8ac7af5316c6af173f3b9e

If you see any other value, then the file is corrupt, in which case you should delete it, then open Safari and navigate to https://go.microsoft.com/fwlink/?linkid=525133 at which point the installer will download again. Once it's done (the file is around 1.6GB in size), try the shasum command again to verify that it is now the correct value.
