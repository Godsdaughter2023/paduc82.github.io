I actually was able to sort this out.

First, the onenote add-in for WORD/POWERPOINT must be enabled (just select it from the REVIEW tab) and point to the onenote folder.  Now a docked onenote will create the link for WORD and POWERPOINT when entries are made while the application is the active window.

Only the IE browser is supported for docked onenote (Chrome does not work).

empower it say:

How to get McAfee and Office 365 to play nicely together

Add McAfee SaaS Email Protection & Continuity to Office 365

Log on to the Office365 Console as an administrator of the account.

Click on Exchange, This will open the Exchange Admin Center

Click on Mail Flow

Click Rules then the + symbol and

Choose Option to Create a new Rule

Name = Mxlogic Connector

Click the More Options button at the bottom of the screen

Apply this rule if = The sender is located outside the organization

Add condition = Sender’s IP address is in the range of 208.65.144.0/21,208.81.64.0/21 (Put on separate lines)

(These addresses are from the Mxlogic console under Mxrecords.)

You can now use the two addresses at the top of the page that end with the /21.

This should be 208.81.64.2/21

Once you have created the connector you have to the click the ENFORCE option

Add a checkmark to stop processing more rules

Add a comment at the bottom of the screen so you will know what the rule is for if anyone looks at it later.

This rule controls inbound filtering for messages received via the Partner connector ‘MXLogic Connector’. Messages received via this connector are excluded from processing by inbound rules migrated from the FOPE Admin Center.

Here is a screen shot of what the final screen should look like:

Now Open the Mxlogic/McAfee Console.

Under Setup

Inbound Servers

Wait one hour before changing the information below. This allows Forefront to be configured and for mail to flow properly. If there is an issue, log in to the McAfee SaaS site, and check to see if mail flow is paused.

Inbound Server

This is under your Office 365 profile, under the DNS settings.  Should look something like Yourdomain.mail.protection.outlook.com (Leave the page open so you can test mail flow)

Do not enable TLS

MX records should be point to McAfee (Change these at the registrar. Only thing that should be necessary is to change the MX record. All else stays the same.)

While still on the McAfee page

If mail is not flowing, check to see if the spooling has been activated on the account. If so make sure the changes setup in Forefront have been implemented for more than 1 hour and then set to Automatic.
