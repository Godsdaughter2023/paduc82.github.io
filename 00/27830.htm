john - project say:

BillSmith1222,

It's not so much a simple "radio" button as it is understanding how Project works. First let me explain that Project stores all date related information in minutes (actually tenths of a minute) regardless of how it may be displayed. Second Project uses working days to schedule as opposed to calendar days. Third, the Duration field is determined by the project calendar (e.g. standard 8 hour day for 5 days a week, 24 hour calendar, etc.).

Given that knowledge, if your option setting for the definition of a day is the default 8 hours but you have set a 24 hour calendar, that 24 hour period will actually show as 3 days in the Duration field. To show that, try this. Set up 3 tasks all with a default 1 day duration. Link the three tasks finish-to-start. Now set the Gantt display timescale bottom tier to be in hours. You will see that the first task starts at 8:00 AM and finishes at 3:00 PM. The second task will then start at 3:00 PM and go to 11:00 PM. The third task will start at 11:00 PM and finish at 8:00 AM the next day.

However, I'm guessing that's not what you want. I'm assuming the classes do not run round the clock so therefore you do NOT want to use a 24 hour calendar. If your classes are shorter or longer than the default 8 hours, you can create your own custom working time calendar based on the standard and then set that as the Project calendar.

Ordinarily I would suggest you simply enter duration values in elapsed days (e.g. 21edays), however, when doing so the definition of a "day" now automatically expands to 24 hours so if you set a start date of 13 Jan. 2016 and enter 21 edays in the Duration field, the finish date will be 3 Feb. 2016 at 8:00 AM. Depending on how you have the Gantt display set (i.e. hours, days, etc.) the Gantt bar may or may not show the subtle difference. You may find it useful to include the time element when displaying dates. That is done under File/Options/General group/Date format.

However, another option is available. First create a 7 day calendar based on the Standard calendar except make Saturday and Sunday working days. Then make that new calendar the Project calendar. Now when you enter the number of days in the Duration field Project will automatically set the finish date to be that number of calendar days from the start date. If the class participants do indeed attend classes 7 days a week, then you need do nothing more. However, if the classes are only held 5 or 6 days a week, you will need to create a 5 or 6 day calendar for each class participant (i.e. resource). By doing so Project will only schedule the resources during their working days. The screen shot below shows a simple task with a 7 calendar day duration but the resource only works a standard 5 day week.

So, it depends on what you are trying to portray in your plan.

The TOC field must also include the \f switch. You can edit the TOC field manually to add it, or you can choose to include it when you insert the TOC.

If you bypass the "Automatic Table of Contents" selections at References | Table of Contents | Table of Contents and instead select Insert Table of Contents..., you'll get the Table of Contents dialog. If you prefer to use one of the built-in "automatic" TOCs, then you have no choice but to edit the TOC field manually to add the \f switch.
