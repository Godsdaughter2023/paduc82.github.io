marshall barton say:

Somehow you got the calculated field's alias name in the Order By clause, but that's why you were prompted for it.  I suspect the reason is because you used the alias name in the Count function and I am surprised that it even ran at all.  Technically, you should not use a field's alias name anywhere else in the query, but sometimes(?), with no rules about why, it does work.  To sort by the calculated field, you can add an ORDER BY clause at the end of the query, but before the semi colon:

SELECT [Class] & " " & [Type] AS [Original text], Changes.Predicted_text,
         Count(IIf([Class] & " " & [Type] <> [Predicted_text],1,Null)) AS [# Wrong]
FROM Changes
WHERE Changes.Property) Like "bach*" AND Changes.Month Is Not Null
    AND Changes.Year="2010"
GROUP BY [Class] & " " & [Type], Changes.Predicted_text
ORDER BY [Class] & " " & [Type] DESC

If you Paste that into an new query's SQL view and switch to datasheet view, it should work.  Once you are satisfied it's ok, switch to Design View to see how the query designer represents it.

Note that the semi colon is optional, so it won't make any difference if you leave it out.
