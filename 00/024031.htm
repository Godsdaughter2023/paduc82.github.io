There has been a package missing from the Install.wim for several builds. DISM Repair looks in Install.wim for a package with a name that includes "Microsoft-TestRoot-and-FlightSigning". When it does not find this, it gives an error indicating that it cannot find the repair Source, even if it has already successfully used that same source to repair other problems, or if there is nothing wrong in the first place (i.e. sfc /scannow completes without error). The specific error shows up in the CBS.log after running DISM /RestoreHealth.

When I first installed build 15063, I had copied it to a separate partition, by simply using DISM to apply Index:4 from the ESD file to that partition. Before I had time to boot to it, I ran sfc /scannow and DISM /RestoreHealth on it to see if it came up clean and it did. After booting to it I then ran DISM /Online /RestoreHealth and it gave the same error about missing the FlightSigning package as was happening on previous builds. So something causes Windows to look for that missing package in Install.wim, after install, when running DISM. Another user explained why it looks for it in another thread some time back. I expect that it will be resolved once version 1703 becomes public.
