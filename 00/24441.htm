Once a Word or Excel doc is corrupted, how can you recover it? - Microsoft Community

cheyenne manornursing home say:

Once a Word or Excel doc is corrupted, how can you recover it?

Split from this thread.

This did not answer the question.  The question was how to recover these corrupted files, not how to save them as a converted document.

Once a Word or Excel doc is corrupted, how can you recover it?

jay freedman say:

If the file won't open at all (from inside the Word or Excel program, rather than double-clicking the document in Windows Explorer), click once on the file in the File > Open dialog*; then click the down arrow on the right side of the Open button and choose "Open and Repair".

* If you have Office 2013/Office 365, first go to File > Options > Save and check the box for "Don't show the Backstage when opening and saving files". Then the Ctrl+O shortcut will open the old File > Open dialog instead of the newer Open page.

If the file opens but shows signs of corruption (chunks that are missing, inability to scroll through certain sections, etc.) then copy everything except the final paragraph mark and paste it into a new blank document. If the corrupt document has section breaks and you can isolate which section is corrupt, do the copy/paste but don't copy the section break at the end of the corrupted section. (Easier to do this when nonprinting characters are displayed by clicking the Show/Hide button.)

There are some other techniques for files that don't work with these procedures, but they're more laborious and require some special knowledge.

gord dibben say:

To automatically sort based on a re-calculation will require VBA worksheet event code.

I do not normally recommend auto-sorting because when an incorrect value is entered a sort takes place and it can be difficult to find the incorrect entry after the sort.

But if the data is being auto-updated I guess human error is not a factor??

Copy and paste this event code to the worksheet module by right-click on the Sheet Tab and select the item  "View Code"

A blank module will open for you to paste into.

Alt + Esc to return to Excel sheet.

The code will sort A8:Px   where x is last entry at bottom of column M

Private Sub Worksheet_Calculate()
      Dim lastrow As Long
      Application.EnableEvents = False
      lastrow = Cells(Rows.Count, "M").End(xlUp).Row
      Range("A8:P" & lastrow).Sort key1:=Range("M8:M" & lastrow), _
           order1:=xlAscending, Header:=xlNo
Application.EnableEvents = True
End Sub

You are quite welcome Nhaz!  Thank you for the kind comment. I was surprised to see so many folks jumped in to repeat what I had originally said.

Clearly in the past, folks wanted to completely close apps to preserve valuable resources but it is not a concern anymore.  When you leave an app, it is off and NOT using resources.  It is in a saved state that will jump you right back into it (where you left off) if you choose to go back to it before it eventually scrolls off the list of apps you see when you long press on the back button.

 

toddallcock say:

As others have stated, WP7's multitasking is nothing like that on a PC.  Apps don't have the ability to keep running in the background. Instead, they can use the device's notifications system to periodically look for information updates and the system will alert you via a live tile or "toast" popup that there is new information and invite you to reopen the app.  So, for example, the app can tell the notification system to monitor a particular service and tell you there's a new weather alert, or Facebook message, but the app itself won't have that information until you open it and it fetches it itself.

 

You can control what apps are allowed to use the notification system (IIRC, to conserve resources, only ten apps are allowed to do this) in Settings > Applications > Background Tasks. It's a nice way to balance the need for fresh information vs. the power and resource sapping method of "real" multitasking used by OSes like Android or old-school Windows Mobile 6.x and earlier.
