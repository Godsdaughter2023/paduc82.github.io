stefan blom say:

Table styles are a bit trickier to deal with because, unfortunately, they are never visible in the Styles pane (Ctrl+Alt+Shift+S), but if you click inside a table and then look at the table gallery on the Table Tools Design tab you can tell which table style is applied to the current table. Just rest the mouse pointer on the highlighted table format and see which name shows in the screentip.

It is best to create a table style with the settings that you want. Give the style a descriptive name (the built-in table styles have names that are very difficult to remember, I think).

The paragraph styles applied to text in table cells won't change if you apply a different table style.

If you want to, you can use a macro to quickly apply your new table style to all tables in a document.

AutoText entries are always saved to templates. To share AutoText entries with other people, you will have to share the template that contains them; that is correct. jay freedman say:
Word 2010 doesn't have a way to assign keyboard shortcuts directly to specific font colors -- but neither did Word 2003. (I looked, so yes, I'm sure.) [Edit: There is a way to assign keyboard shortcuts to font colors. See the other post marked as an answer to this question.] That's why Suzanne suggested either macros or character styles, either of which can be assigned shortcut keys.

Step by step for character styles:
  1. To make it easier to get to the right dialog, go to Word Options > Quick Access Toolbar. Set the "Choose commands from" dropdown to
  2. On the Quick Access Toolbar (which may be above or below the ribbon), click the New Style button you just added. A dialog opens.
  3. In this dialog, enter a name for the style (for example, Blue).
  4. Change the Style Type dropdown to Character.
  5. Under the "Formatting" heading, open the dropdown on the right and pick the color you want (you can use any of the colors in the display, or click More Colors to open another dialog).
  6. Click the option button near the bottom for New documents based on this template.  
  7. Click the Format button below that and choose Shortcut key, then assign the shortcut you want. (Don't forget to click the Assign button before you click Close.)
  8. Click Close or OK in each open dialog to close it.
Repeat steps 2 through 8 for the other colors.

If you don't want to keep the New Style button on the Quick Access Toolbar, you can right-click it and choose
You can now select any portion of text and press the shortcut key to change its color. To remove the color, re-select the text and press Ctrl+spacebar.

jay freedman say:

The Check Issues function has nothing to do with whether documents open in Compatibility Mode, so that's not surprising.

The fact that new documents start in Compatibility Mode indicate that Word thinks your Normal.dotm template is from an earlier version. I'm not sure how that would happen on a new Office installation on a new computer, but it should be easy to fix.

The Normal.dotm template by default lives in a hidden folder. Although there's a way to make it visible in File Explorer, there's an easier way to open it:

  • Press Alt+F11 to open the macro editor.
  • Press Ctrl+G to open the Immediate window at the bottom of the editor.
  • Type or paste this into the Immediate window and press Enter: 

NormalTemplate.OpenAsDocument

  • Close the macro editor. The Normal.dotm template will be open in the main Word window, as indicated in the title bar.

If the title bar also says [Compatibility Mode], go to File > Info and click the Convert button. Then save and close the template. That will immediately fix the issue with new documents. Any existing documents that still say they're in Compatibility Mode can be updated by the same Convert button, although it isn't necessary to update them in most cases -- the only difference is whether they'll allow you to add content that uses new features.

jaiminip say: This is a problem with MS word.

The Save As PDF/XPS Add-in takes the bookmarks based on the headings displayed on Document Map (View > Show/Hide > Document Map) of MS word. If you can get the required heading added to the document map, this should be resolved. But i'm afraid word does not have the flexibility to do it. Some macros might work though, I have not explored that possibility.

You can try one of the following options:
  • Create PDF using PDFMaker for word. This comes along with Acrobat Professional installation. You will get the bookmarks, but on the levels, you might still need to work. 
  • Edit the PDF file with a PDF editor like Acrobat Pro. You can manually add bookmarks to a PDF doc wherever you please.
  • Keep the headings outside the table and give outside borders (available in Home > Paragraph). 
Using white font color for an extra heading is a good option, but the text will be used in all versions made henceforth. Even if someone copies the content, the white text will also go with it. When used a paste special command, it will become visible as well. So be sure whether it serves your purpose.
