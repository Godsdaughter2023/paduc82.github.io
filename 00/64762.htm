If the olecnv32.dll file is corrupt, you will see an error like this:

STOP: c0000221 {Bad Image Checksum}
The image olecnv32.dll is possibly corrupt.  The header checksum does not match the computed checksum.

Curiously, if the file is completely missing or has a length of zero bytes XP will boot just fine without it.

SInce it is one of the 3498 files that Windows File Protection looks after, there should be another copy of the file on your system is the dllcache folder.

Like Wunders says, you can make a Hiren's boot CD and from the Mini XP mode, see if you can find a copy of the file in one of these folders (assuming XP is installed on your C drive):

c:windows\system32\dllcache
c:\windows\ServicePackFiles\i386

If you can find a copy, rename your suspicious one and replace it with the copy and see how things look.

If you can't find a copy, there is one on my SkyDrive you can download and sneakernet it to the afflicted system using a USB thumb drive or floppy.  You will still have to use Hiren's for that method.

If that all seems to complicated and you do not have a genuine bootable XP installation CD from which you can boot into the XP Recovery Console, you can make a bootable XP Recovery Console CD and see if you can replace the replace the file from there.  I can send you instructions if you need to know how to do that.

I can hear the Support Engineer "experts" engaged by Microsoft suggesting you run sfc /scannow to replace the missing file but that will not work since that is not what sfc /scannow does, that is not how it works and that is not what it is for.  I know it will not replace the file because I just tried it.

If you need to download the file, here's how:

I uploaded a copy of the XP Pro SP3 file you need to my SkyDrive (everybody has a SkyDrive for sharing files).

You can download it and when you do, put a copy of the file in these folders (assuming Windows is installed on your C drive):

When you download the file (especially if you use Internet Explorer), when you get a chance to save the file, your browser may not save the file with an extension (like .exe, .dll, .cpl, .sys, .zip, etc.) so you will have to manually add the proper extension to the file when you download the file before you save it.
