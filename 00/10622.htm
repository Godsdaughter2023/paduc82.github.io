brian tillman [outlook mvp 2007-2019] say:

I don't recall agreeing to those permissions, either.

Just because the permissions are requested, that doesn't mean they'll be used. Take the "delete all mails" permission, for example. Exchange and Exchange ActiveSync are two-way synching protocols. If you delete a message on the phone or using a web browser online, the message will be deleted both places. This means that whatever is deleting the message needs permission to do so. And if it can delete one message, it can delete them all. There's no permission to allow deleting just single messages, so it has to request permission to delete them all.

I'm pretty sure the same type of thing applies to the reset permission. While I don't know what function may be asking, in order to perform a needed function, that permission must be granted because whatever function it is is lumped in with the reset permission.

By the way, you don't need any extra apps to connect an Android device to your Exchange mailbox. Add an Exchange or Corporate account.

macropod say:

The previous code won't work with references like 'Figure 17.1' - only with plain numbers (e.g. 'Figure 1').

The following code should work with captions that may or may not include chapter references and non-numeric sequences. 

Sub Demo()
Application.ScreenUpdating = False
Dim i As Long, j As Long, ArrCptn, StrCptn As String, StrTmp As String
ArrCptn = Array("Equation", "Figure", "Table")
For i = LBound(ArrCptn) To UBound(ArrCptn)
  With ActiveDocument.Range
    StrCptn = "||"
    For j = 1 To .Fields.Count
      With .Fields(j)
        If .Type = wdFieldSequence Then
          If InStr(.Code.Text, ArrCptn(i)) > 0 Then
            StrCptn = StrCptn & .Result & "|"
          End If
        End If
      End With
    Next
    If StrCptn <> "||" Then
      With .Find
        .ClearFormatting
        .Replacement.ClearFormatting
        .Text = ArrCptn(i) & " [A-Za-z0-9^+^=.:\-]{1,}"
        .Replacement.Text = ""
        .Forward = True
        .Format = False
        .Wrap = wdFindStop
        .MatchWildcards = True
        .Execute
      End With
      Do While .Find.Found
        If Len(.Text) > 0 Then
          If .Style <> "Caption" Then
            Do While .Characters.Last Like "[.:\-" & Chr(150) & Chr(151) & "]"
              .MoveEnd wdCharacter, -1
            Loop
            StrTmp = Split(.Duplicate.Text, " ")(1)
            StrTmp = Replace(Replace(Replace(Replace(StrTmp, Chr(150), "."), Chr(151), "."), "-", "."), ":", ".")
            If InStr(StrTmp, ".") > 0 Then StrTmp = Split(StrTmp, ".")(1)
            If InStr(StrCptn, "|" & StrTmp & "|") > 0 Then
              j = UBound(Split(Split(StrCptn, "|" & StrTmp & "|")(0), "|"))
              .InsertCrossReference ReferenceType:=Split(.Text, " ")(0), ReferenceKind:=wdOnlyLabelAndNumber, _
                ReferenceItem:=j, InsertAsHyperlink:=True
              .End = .End + 1
              .End = .Fields(1).Result.End
            End If
          End If
        End If
        .Collapse wdCollapseEnd
        .Find.Execute
      Loop
    End If
  End With
Next
Application.ScreenUpdating = True
End Sub

rod gill say:

Would have to know a lot more to help, especially about any date constraints.

 

I recommend not having a must finish date on the last task: too confusing for most people. Some management especially will see the last task finishing on time and be happy, completely ignoring the negative slack.

 

In addition, the last tasks in a project are rarely time constrained. Instead:

  1. Add a task called project closed and force a date that is at least 10d after the last task to finish. Now all but this task should be non-critical
  2. Work with your Project Sponsor and Project manager and key stakeholders and agree completion dates for all time critical deliverables
  3. Enter these dates on the relevant tasks as Deadline dates.

Now your critical path is all tasks critical to completing the key deliverables on time. Where tasks are finishing late, they show as having overshot and that therefore something needs to change to recover lost time. In addition any task that overshoots its deadline date has a red diamond appear in the Indicators column. You now have an automatic exception report!

 

Finally I would set an extra Baseline and reset it every week before updating progress. Now each week after updating you can see progress compared to previous week and identify the key areas that are losing or gaining time.

 

Good Luck!
