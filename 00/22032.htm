Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

This little demo file illustrates a number of methods for restricting a report's output by means of values selected or entered in dialogue forms.  The first option, to 'Filter Report on One Field by WhereCondition Argument' would be an appropriate model to follow.  This uses a multi-select list box to select one or more items, in my case employees, by which the report is filtered.  The underlying model is a conventional one for a many-to-many relationship type, as described by Duane, in my case:



The code behind the button in the dialogue form loops through the list box's ItemsSelected collection and cumulatively builds a string expression , which is then used to filter the report by means of the WhereCondition argument of the OpenReport method:

Private Sub cmdOpenReport_Click()

    Dim varItem As Variant
    Dim strEmployeeIDList As String
    Dim strCriteria As String
    Dim ctrl As Control
    
    Set ctrl = Me.lstEmployees
    
    If ctrl.ItemsSelected.Count > 0 Then
        For Each varItem In ctrl.ItemsSelected
            strEmployeeIDList = strEmployeeIDList & "," & ctrl.ItemData(varItem)
        Next varItem
        
        ' remove leading comma
        strEmployeeIDList = Mid(strEmployeeIDList, 2)
        
        strCriteria = "EmployeeID In(" & strEmployeeIDList & ")"
