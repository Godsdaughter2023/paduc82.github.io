From what you've said, I believe the code you've posted above is fundamentally correct, but I see the probably cause of the problem: The query is using a number of controls on the form as selection criteria.  These then would be seen by Access as parameters to the query, and need to be resolved.  Access does this automatically when *it* runs the query, for example, when using the query as recordsource for the form, but it *doesn't* do it when you run the query yourself using the DAO Execute method.

However, we can make it resolve the parameters, using either of two methods: (1) use DoCmd.RunSQL instead of CurrentDb.Execute to run the query, or (2) manually resolve the parameters in code, via a DAO querydef object.  For myself, I would use the second method, but it's more complicated, so I will give you code for the first method:

'------ start of code ------
Private Sub CmdClearCheckBox_Click()

    On Error GoTo Err_Handler

    If Me.Dirty Then Me.Dirty = False ' save current record if dirty
    DoCmd.SetWarnings False    ' Temporarily turn warnings off

    DoCmd.RunSQL "UPDATE qryFilter_for_Transmittals SET traSELECT = False"

    Me.Refresh

Exit_Point:
    DoCmd.SetWarnings True    ' Turn warnings back on!
    Exit Sub

Err_Handler:
    MsgBox Err.Description, vbExclamation, "Error " & Err.Number
    Resume Exit_Point

End Sub
'------ end of code ------

Note, by the way, that this will only clear the checkbox for the records that are returned by the query "qryFilter_for_Transmittals".  If there are other records in the underlying table that are *not* returned by the query (because they don't meet the filter criteria), those records will not be touched.  I *think* that's what you want, but I could be wrong.

john korchok say:

Please do your students and all future users of the manual by applying correct styles to it. Styles vastly improve a documents useability and future updateability. From the sounds of it, the current manual is all in Normal style with local formatting applied to create fake headings. These type of documents are often created by users with no training in Word.

For the Table of Contents, you need to be modifying Styles, specifically the styles TOC1, TOC2, etc. that are used for styling the Table of Contents. Please follow these steps:

  1. In Word, choose Format>Styles.
  2. Click on TOC 1, then on Modify.
  3. Click on Format, then Tabs.
  4. Enter a tab stop position (normally the distance from left to right margin).
  5. Select Right Alignment.
  6. Select 2, the dot leader.
  7. Click on Set. OK out.
  8. Repeat for TOC 2 and up, depending on the number of levels in your Table of Contents.

bob jones aka: cybertaz say:

Well, let's start at the end of your message & perhaps that will suffice :-)...

Use of Word's built-in TOC generator does not require the use of heading styles. There are at least several other means by which to have TOC entries included;

  • Any Paragraph Style can have a TOC Level included in the Style's definition,
  • Even Paragraph Styles that don't have a Level assigned can be incorporated via Insert> Index & Tables> Table of Contents - Options,
  • TC fields can be manually inserted to mark TOC content even if the content has no Style applied,
  • Use a combination of all the above

Although PC-centric, this Tutorial on MVP Suzanne Barnhill's site applies nearly verbatim to Mac Word.
