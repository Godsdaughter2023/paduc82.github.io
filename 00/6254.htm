Reason that your data is in an OST file is because MS changed the data file type that stores IMAP information which has some caveats specific to O'2013 and IMAP accounts

#1 - To restate what Palcouk mentioned, the OST file is a mirror of what's on the server so whenever the IMAP account is configured, a new OST file will be created and the OST file will contain the same info as found on the server (after the first account sync) BUT.........................

#2 - If the IMAP account is the first account configured in the Outlook profile - it will have folders in its folder tree with the additional description of "This Computer Only" (i.e. Contacts/Calendar folders). If the data in these folders is not backed up to a PST file on a regular basis - the data in those folders will be lost if the email account is ever deleted from the Outlook profile. This data is not stored on the server and you cannot open an OST file separately - can only be accessed using the Outlook profile/email account specifically used to originally create it. Reconfiguring the same IMAP email account in the same profile will NOT re-use a previously created .OST file.

Hi Weshenbach,

Based on my understanding, you want to sign out of an old Office 365 account and use Office applications with a new Office 365 account.

If my understanding is correct, first, let us check whether the new Office 365 account is assigned a valid license. Please go to Office 365 subscriptions page with your new Office 365 account and check whether you have the latest desktop version of Office license, it is necessary to use Office applications.

If you don't have that license, please contact your IT department to let them assign one to you.

If you do have that license, please quit all of your Office applications first and follow the steps below to remove the old Office 365 account credentials.

A.Remove Office 365 license for Subscription based installs

1).  Find where the ospp.vbs file locates, generally it should be in this folder: C:\Program Files\Microsoft Office\Office16 or C:\Program Files (x86)\Microsoft Office\Office16  (the file path depends on your versions of Windows and Office, here I use the former one as example.)

2).  Run C:\program files\Microsoft office\office16>cscript ospp.vbs /dstatus

The above command line will generate a report of the currently installed/activated license.

NOTE: You might see multiple licenses in the /dstatus report.

3).  Make note of value for “Last 5 characters of installed product key”, Run  C:\program files\Microsoft office\office16>cscript ospp.vbs /unpkey:“Last 5 of installed product key”   For example: C:\program files\Microsoft office\office16>cscript ospp.vbs /unpkey:G8YH4 (See Below) Repeat the step above if necessary until all keys are removed.


B.  Remove cached identities in HKCU registry:

Open Registry Editor, navigate to Computer\HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Common\Identity\Identities and remove all of the identities under \Identities.

C.  Remove the stored Credentials in the Credential Manager:

Open Control Panel > Credential Manager.  Remove all Windows credentials listed for MicrosoftOffice

After you finish the steps above, open any Office applications, it should prompt you activate and sign in, enter your new Office 365 account and password and all is done.

Joanne
