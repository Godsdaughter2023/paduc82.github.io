Hi.

 

You could use Outlook's object model to generate an html email. You would just need to craft the html and insert it into the htmlbody property of the mail message.

 

Here's some sample code:

Function CreateHTMLMail()
    'Creates a new e-mail item and modifies its properties.
    Dim appOutLook As Outlook.Application
    Set appOutLook = CreateObject("Outlook.Application")
    Dim objMail As Outlook.MailItem
 
    'Create e-mail item
    Set objMail = appOutLook.CreateItem(olMailItem)
    With objMail
       'Set body format to HTML
       .BodyFormat = olFormatHTML
       .HTMLBody = "<HTML><BODY>Enter the message text here. </BODY></HTML>"
       .Display
    End With
    Set OutlookApplication = Nothing
End Function

 

This was built from the sample here:

MailItem.HTMLBody Property (Outlook)

http://msdn.microsoft.com/en-us/library/ff868941.aspx

 

*Remember to add a reference to the Microsoft Outlook xx.x Object Library under Tools > References in the Visual Basic editor.

chip pearson say:

Due to the lack of parentheses, your sample equation is ambiguous. I've provided 3 possible interpretations. With the base value in cell A1, you can use

 

=(A1*(1-42.25%)+((A1*10%)))*6

If A1 is 100, this returns 406.5

 

Another way is

 

=A1*(1-42.25%)+((A1*10%)*6)

If A1 is 100, this returns 117.75

 

The difference is that the first formula multiplies everything by 6, while the second formula multiplies only 10% of A1 by 6.  In both formulas, the base amount is multiplied by 10%. If you want the 10% to apply to the discounted value of A1 as in (A1*(1-42.25%)), use

 

=A1*(1-42.25%)+(((A1*(1-42.25%)*10%)*6))

If A1 is 100, this returns 92.4.

 

Pick the formula that returns the result you expect. If none of these are correct, rewrite the equation using parentheses and provided input values with expected results.
