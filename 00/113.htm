Given the standard calendar and having a task start on Jan 1, 2015 with a duration of 261 days, the task will end on Dec 31, 2015, when the above resource is assigned to the task, the total work content is 2,088 hours. ken sheridan say: The enabling/disabling of controls will only work if the tab order of controls common to each subset of controls dependent on the value of 3 is constant, which was the case in your original post, and on which my initial reply was predicated.  If the required tab order per of controls common to each subset will change, then this approach won't work.

An alternative way of doing it would be to create three subsets of duplicate controls, and superimpose each trio of duplicates on each other.  The tab order of each of the subsets can then be set independently in the properties sheet.  Then, depending on the value of 3 the relevant subset would be made visible and the other two hidden.

This could be done most efficiently in the code by first setting the Tag property of the controls in each subset to a different value.  The code would then loop through the form's Controls collection and, at each iteration of the loop, within a Select Case Me.3 construct and, within it a nested Select Case ctrl.Tag construct, the Visible property of the control would be set to True or False as appropriate to the value of 3.  In the same code3 you could also disable any controls in the visible subset which you don't want the user to inadvertently move to with the mouse.

ken sheridan say: It should be easy enough to adapt the code from the demo:  

1.  In the demo the code is in the Click event procedure of an option group.  In your case the code would need to execute in two places: (i) the form's Current event procedure, to allow for existing records, and (ii) the 3 control's AfterUpdate event procedure to allow for setting the tab ordered when a value in 3 is entered in a new record.

2.  The code in the demo uses an If....Else....End If construct as it caters for only two values.  As, to judge by your latest code, you need to cater for the value being '-', '*' or something else I'd recommend a Select Case construct:

    Select Case Me.3
        Case "-"
        ' code to set TabIndex property of each control goes here
        Case "*"
        ' code to set TabIndex property of each control goes here
        Case Else
        ' code to set TabIndex property of each control goes here
    End Select

The important thing is that you must set the TabIndex property of all the relevant controls in each Case statement, not just the ones you want to tab to first if the value in question is selected

3.  Rather than repeating the code in each of the two event procedures You can put it in a private function in the form's module:

Private Function SetTabOrder()

    Select Case Me.3
        Case "-"
        ' code to set TabIndex property of each control goes here
        Case "*"
        ' code to set TabIndex property of each control goes here
        Case Else
        ' code to set TabIndex property of each control goes here
    End Select

End Function

In the form's properties sheet you'd call the function as the On Current event property with:

    =SetTabOrder()

In the 3 control's properties sheet you'd do the same as the After Update event property.

4.  As well as setting the TabIndex property for each control in each Case statement you could, if you wish, enable/disable the controls as well so the user can't access those controls irrelevant to the current value of 3.  If you do this you must first move focus to a currently enabled control, so as the first line in each Case statement enable/unlock the control you want to be first in the tab order, then in the second line set focus to it.  The lines to set the TabIndex property of each control would then follow this.
