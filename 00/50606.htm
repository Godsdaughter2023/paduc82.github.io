Bootsqm.dat file is automatically created when check disk is either automatically or manually started on your hard drive. It is basically a scratchpad for use by this command so that it can keep track of what, if any, corrections it needs to apply to the disk. 1.     Open Folder Options by clicking the Start button, clicking Control Panel, clicking Appearance and Personalization, and then clicking Folder Options.

2.     Click the View tab.

Additionally, you may boot into safe mode and check the status of the disk to verify if the data in the drive is indeed present or not.

 

Safe mode is a troubleshooting option for Windows that starts your computer in a limited state. Only the basic files and drivers necessary to run Windows are started. The words Safe Mode appear in the corners of your monitor to identify which Windows mode you're using.

All of your Windows 7 Scheduled Tasks are corrupted - not just the backup task and without them your system may seem functional but will be crippled.

Fixing problems like this are not everybody's cup of tea so you may need to take your system to a professional or somebody you trust that will fix it and fix it right (not using the "try" word).

Let me be bleak:


As many times as this problem has come up, the Microsoft engaged Support Engineer "experts" have repeatedly demonstrated that they do not know what has happened to your system, they don't know how to fix it and have never successfully fixed this popular problem.  All you get are things to "try" and they don't work (as you have seen).

The problem is that when reverting back to Windows 7 the registry parts of the Scheduled Tasks are restored but the Windows 10 tasks files themselves are not restored so now there is a conflict and hence the corruption - things are not matching up like they should.  The problem usually surfaces first when folks try to set their backup schedule up again.

simstim_og say:

"Always ask before opening this file" - why can I not just open my pictures, jpg? How to disable

Why does windows now require me to pass a security warning if I want to open jpg files? I open a lot of these and don't have time for windows to ask me to uncheck a box for each one. Help in disabling would be great so I can open pictures freely.
cmdrkeene say:

This isn't from UAC nor WebRoot.  The file is correctly marked with the "mark of the web" bit, but that should not be causing the warning you see.  The warning is only intended to happen with files that are potentially dangerous (jpg is not one that is set to show the message usually).

I suspect you could fix this by resetting your security settings for downloads to default.  Here's how:

  1. Open the Run box (+R) and run the command inetcpl.cpl
  2. On the Security tab, click "reset all zones to default", then click OK.
  3. Restart your computer, then try downloading a picture again and opening it

If that doesn't work, I'd like to know.  But my next suggestion just in case is still as follows (only if the above doesn't work).

  1. From the Run box, open gpedit.msc
  2. Go to User Config > Admin Templates > Windows Components > Attachment Manager
  3. Open the Low Risk item and set JPG as a low-risk file type
  4. Restart your computer, then try downloading a picture again and opening it

If you don't have Pro edition, you won't be able to use GPEDIT to make the change. If that's the case, go to this website for instructions on how to do the same thing using Notepad: http://www.webtlk.com/2010/04/12/how-to-turn-off-and-disable-do-you-want-to-run-this-file-warning-in-windows-7-and-vista/

Let me know how it goes.
