a. user say:

When you reset your PC, your hard drive is erased and a fresh copy of Windows is installed. If you did a 'keep my files' type of reset, apps that you got from Microsoft Store will be reinstalled. Apps that you got elsewhere are not - more on that below. Windows will put an HTML file on your desktop that shows you all the non-Microsoft Store apps that you need to install.

The apps in that HTML file are taken from the list in Start > Settings > Apps > Apps and features, but that may not, in fact, be everything that you need to re-install. For example, it may not include all your drivers, and it may not include apps that don't need to be installed (so-called 'portable apps.')

If you have apps that were installed from a disk, you can re-install them from the disk. For apps that were obtained from the internet, visit the developer's website and follow their re-installation instructions. If you're re-installing apps that you paid for, you'll need the license key and any other registration information that you used to install them in the first place.

Even with all that, there are still some apps that you may not be able to re-install, e.g., apps from developers that no longer exist, and apps that were pre-installed by the computer manufacturer. The license for the latter sometimes states that they cannot be re-installed once removed.

This is one more reason that I advise against a reset unless it's truly necessary. There are 'gotchas' that you don't find out about until it's too late.

It's not that Microsoft doesn't want to re-install apps that you got from outside Microsoft Store. Apps from Microsoft's own Microsoft Store are easy to find and easy to re-install because Microsoft keeps track of them in your Microsoft account, including your configuration choices and progress (e.g., your progress in a game.)

For apps that you got from other places, Microsoft doesn't have a master list it can consult for every app ever released that tells them where to find an installer (if one still exists,) and even if they did, they don't know what configuration options you chose during installation. And they certainly don't know your license key for apps you purchased.

    Where do I find the exit codes after chkdsk has run on restart? - Microsoft Community

    okieseeker say:

    Where do I find the exit codes after chkdsk has run on restart?

    Searching Microsoft gives me tons of information about running chkdsk and the various options and meanings of exit codes. I ran it today on my D drive with the /f option which required a restart. I thought Windows might display a screen or something after the restart showing the exit code but no such luck. None of the MS pages I've seen says where to find the exit code. I tried the Event Viewer but no luck. Is there a log file somewhere?
    wunders say: Exit codes are transitory in nature and are typically only available to the next instruction after the chkdsk instruction when it is included in a script or .bat file.  The Event ID has nothing to do with the exit code.  Chkdsk only has four exit codes that are described here:
       "Microsoft Windows XP - Chkdsk"
       < http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/chkdsk.mspx?mfr=true >

    It only makes sense to use these codes if you are writing a script that checks drives other than the Windows OS drive as the Windows drive is always in use and can only be checked before Windows (and ergo any script) is executed.  The information that you retrieved from the Application log tells you much more than the exit codes do.  If you need to programatically check the result of a power-on Chkdsk operation, you will need a tool such as "psloglist" to dump the entry from the log and parse it from there.
