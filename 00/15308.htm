bob jones aka: cybertaz say:
Re: "When  I open it to use it, it appears not as the labeled template, but a new workbook."

I honestly don't know what you're expecting, but that is exactly what a template is supposed to do :-) A template is basically a 'cookie cutter' used for the purpose of stamping out file after file, each of which starts off identically based on the design of the template.

If your need is for an Excel workbook file that you can continuously open, revise & update you do not want to save it as a template... Save it as a regular Excel Workbook (.xlsx)  file.

Although some may understand what Phillip means there are a few discrepancies in how he actually expresses it, so just for clarification;

  • "it’s a specialized file that can not normally be overwritten.": Well, yes it can be 'overwritten' just like any other type of file. I believe the point Phillip is trying to make is that changes made in new workbook files generated from the template do not change the template file, itself. [See #2 below]
  • "When a Template is opened it will always be titled as untitled  until given a name when saving.": This is inaccurately phrased in 2 ways...
  1. When a template file actually is opened the name displays in the title bar just as it does with any other file -- the name with extension is shown as [e.g.] MyTemplate.XLTX to indicate that the template file is the active document. In that case, any changes made are saved to the template unless the user specifically selects the Save As command. If the user does opt to Save As he can pick any file format he prefers -- IOW, You can open one template then make changes & use Save As to create another template based on the first but incorporating the differences.
  2. It's when a new document is generated from a template that the title bar displays a generic name [e.g.] Workbook4 (with no extension). It also forces the Save As dialog when saved just as does any new file.
