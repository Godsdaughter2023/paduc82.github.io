Yes, there is a limit to how far the automatic context-sensitive list feature will search. I have never seen a published spec, but your conclusion seems to be rather accurate based on some limited testing.

There are at least 2 options to consider;

  1. Have the users right-click the cell & select Pick From List. The list provided will comprise the unique entirety of the column. There may be a limit to this as well, but I've tested successfully on ~1,000 rows.
  2. Store your list of Category Codes elsewhere in the workbook (it also makes it easier if you assign a name to that range of cells). Apply Data> Validation - Allow: List to the cells in the column where the codes are to be entered. Specify the range of Category Codes as the Source: for the list. This method also effectively prevents erroneous entries. See Apply data validation to cells & Create a drop-down list to enter data

Just as information based on experience: Providing a list of that length for user's to pick from has a tendency to backfire on you :-) The optimal recommendation is that a selection list should be kept to ~20 items.

hansv mvp say:

Right-click one of the problem subforms in the navigation pane.

Select Design View from the context menu.

Activate the Data tab of the Property Sheet. (Press F4 if you don't see the Property Sheet)

Make note of the Record Source property. This can be one of the following:

  1. The name of a table. If so, the subform should be updatable.
  2. The name of a query. If so:
    • Right-click that query in the navigation pane.
    • Select Design View from the context menu.
    • Select SQL from the View dropdown on the left hand side of the ribbon.
    • Copy the SQL text and paste it into a reply.
  3. A SQL string. If so, you can copy this directly from the Record Source property and paste it into a reply.
