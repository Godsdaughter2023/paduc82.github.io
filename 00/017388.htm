lz. say:

@Chris

With a "big database" (always good to mention the rough # rows/recors otherwise you run the risk to get inappropriate proposals...) a non-formula solution might be more efficient.

If you're looking for an exact match (no wildcard) you could use a VBA UDF (User Defined Function) like (quick & dirty):

Option Explicit

                           Optional ByVal colIndex As Long = 1) As Variant
'
    Dim valToMatch      As Variant
    Dim vArray          As Variant
    Dim l               As Long
    Dim r               As Long
   
    vArray = vlookupRange.Columns(1)
    valToMatch = LCase(target.Value)
   
    For l = UBound(vArray) To LBound(vArray) Step -1
        If LCase(vArray(l, 1)) = valToMatch Then
            r = l + vlookupRange.Row - 1
            Exit For
        End If
    Next l

        ReverseVLookup = CVErr(xlErrNA)
    Else
        With vlookupRange
            ReverseVLookup = Cells(r, .Column + colIndex - 1).Value
        End With
    End If
End Function

Syntax to use it is the same as VLOOKUP but there's no last argument (TRUE/FALSE for Approx./Exact match), ie.:

  • With a range: =ReverseVLookup(A2, C$2:C$14, 3)
  • With a Table: =ReverseVLookup(A2, tblData, 3)

Back to a "big database". If you perform several VLOOKUPs to return differents values from the same row (if you're not already aware) there are ways to make something more efficient

hal hostetler - mvp-outlook say:
Not necessarily, Outlook, like many POP clients, can be set to "Leave a copy of messages on the server" and, in this day of ISP supplied webmail interfaces to POP accounts, its fairly common for account setups to default that way.  'Tools|E-mail Accounts', select your POP account, click "Change", cilck the "More Settings" button and on the "Advanced" tab, make sure the "Leave a copy" box is unchecked.  If your ISP provides a webmail interface, use it to check your server "mailbox" to be sure its empty.
