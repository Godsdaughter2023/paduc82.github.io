leongoudikian say: Here is the way I attached my beloved HP Laserjet 1012 to my  Win 7 64bit laptop.  It is a slight rewrite of the steps from ryoceullar.

You will first need to install a printer driver.  Do a google of hp laserjet 2200 pcl 5 and you should see a download page on the HP site.  You will download the executable and let it run.

Then:

1. Plug in power to the printer and attach it to the computer's USB port. 

2. Go to Control Panel and open the Devices and Printers window.

3. You should see the printer under the heading "unspecified" and it should be named "hp Laserjet 1012."   If it is not there try replugging in the USB port or rebooting the computer.  It needs to show up there to show the computer has detected the hardware.

4. From the Devices and Printers window click "Add a printer."

5. On the next screen pick "Add a local printer."

6. On the next screen pick  "Use an existing port."  That will activate a drop-down menu to the right.

7.  From the drop-down menu of existing ports, choose "DOT4."

8.  On the next screen you should see two windows, one to choose the Mfr and one to choose the model.  Choose HP and then choose the model "Laserjet 2200 Series PCL 5." 

9. Let the OS find the driver.  After it is done, recheck the main Devices and Printers screen and you should see the newly installed Laserjet 2200 PCL 5 in the list of printers. 

I only did this recently and printed a dozen or so pages perfectly, but I don't know how finicky this will be.  At worst I expect I may need to cycle power on the printer once in a while but we'll see.

¡firedog say:

Windows Live Mail 2011 isn't supported on Windows 10. Upgrading to version 2012 may help.

Much of what happens when Windows Live Mail is launched involves an Internet connection. If there is no connection, then those parts of the start-up process will be bypassed and you may find that the program can open normally. You would then be in a position to conduct some troubleshooting.

A lot of things happen more-or-less simultaneously when you launch the program. Any one of them might be interfering with the launch, so to find out which, disable as many of them as possible and see whether the problem goes away. Here are some of them:

  1. The message database is loaded.
  2. The calendar is loaded.
  3. The contacts database is loaded.
  4. The program connects to the Internet.
  5. The program conducts a version check.
  6. You sign in to Windows Live services.
  7. Your calendar is updated.
  8. Your contacts are updated.
  9. Your feeds are updated.
  10. The currently selected message is retrieved and displayed.
  11. Your mail accounts are polled for new messages.


You can't do anything about the first three, because they're essential to the functioning of the program. You can stop some of the others, though.

 

When Windows Live Mail is running and you're viewing your message list,

  1. Press Ctrl-Shift-O for Options. On the Connection tab, click Change.
    For most users, there will be no active entries on the page that opens.
    Click     LAN settings. There should normally be no entries here, either. If Automatically detect settings is selected, deselect it.
  2. Click OK twice, then if Stop signing in is active, click it. This will disable 6, 7 and 8 at the next program start. Confirm.
  3. On the General tab, deselect
  4. Deselect Check for new messages every ... and click OK.
  5. On the Home tab, click Work offline. This will disable 4, 5 and (I think) 9 at the next program start.
  6. Close down Windows Live Mail and wait a few minutes while the program does its housekeeping.


Now launch and see how it goes. If there's no improvement, the problem is probably with process 1, 2 or 3, most likely 1 - a corrupt message database.

 

If the launch is normal, start re-enabling things systematically one at a time until the fault recurs at launch.

 

It would also help troubleshooting to have logging enabled. Do this by selecting all the boxes at the foot of the Maintenance page (Options > Advanced > Maintenance). While it takes an expert and probably special tools to decipher the log, the event timings might indicate where any undue delay is occurring.

A program crash should raise a crash report in Event Viewer. Have you looked there?

bitsmasher say:

I found a solution for my issue with Windows10 getting error 80070543 when installing updates.

To resolve this issue, follow these steps:

  1. Click Start, click Run, type dcomcnfg.exe, and then click OK.
  2. Click OK if you receive the UAC prompt.
  3. In the console tree, expand Component Services, and then expand Computers.
  4. Right-click My Computer, and then click Properties.
  5. Click the Default Properties tab.
  6. Select Connect in the Default Authentication Level list if it is set to None.

    Note If the Default Authentication Level item is not set to None, do not change it. It may have been set by administrator.
  7. Select Identify in the Default Impersonation Level list.
  8. Click OK, and then click Yes to confirm the selection.
  9. Close Component Services console.
  10. Retry updates ( may have to reboot first)
