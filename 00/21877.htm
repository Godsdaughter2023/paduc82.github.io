joeu2004 say:

Aswin wrote:

when i'm trying to save it again as CSV, those format changes are getting lost. Is there some way to sort this out

-----

No, other than the work-arounds that you have already tried (or manually formatting the cells afterwards). That is just the way that Excel processes CSV files.

When the CSV file is saved, Excel writes the data to the file exactly as they

However, a CSV file is plain text file. It contains no formatting information itself.

When Excel opens the CSV file, normally it interprets the data exactly as it would if you entered it manually. Even putting double-quotes around the data does not matter.

Consequently, most numeric strings are interpreted as numbers, which are entered into a cell formatted as General. There are a few exceptions, just as there are when we enter data manually. For example, data that looks like a percentage (1.23%), currency ($1.23) or date (1/23/45) are formatted accordingly.

The only way to avoid the interpretation is to import (not open) the CSV file and assign the Text format to the column, as you did. But bear in mind that the type of the data is text, even if looks numeric. In some contexts, Excel treats the text data as non-numeric. For example, comparisons with numeric values will have unexpected results.

Alternatively, after opening (or importing) the CSV file, you can assign the appropriate format to the cells.  However, beware that if the CSV data includes numeric strings with more than 15 significant digits, Excel will interpret only the first 15 digits, substituting zeros for any digits to the right, effectively truncating the numeric value after 15 significant digits.

The only other way to work around this Excel behavior is to write a VBA subroutine to read the CSV file and interpret the data however you wish.

It would unrealistic to be reliant on technology alone to prevent Phishing attempts.

For example, a particular sender may have his credentials stolen due to poor PC security (eg using jail broken device, using it on compromised public computers..etc). In which case, there is no way for anyone to tell that the sender is really someone else with a stolen password.

We have also seen poorly designed attempts eg Applesupport123(a)gmail.com as a sender email.

Office365 already has built in anti-spoofing technology, including the use of SPF and DKIM

In addition if you sign up with Advance Threat Protection (ATP) you get Spoof Intelligence and other advanced capabilities.

ATP will also give you additional features like sandboxing and safe links, which will also enhance your corporate protection.

While ATP will enhance your anti-phishing protection, you will also need to educate your users to -

1. Always check on who the sender is. This will identify simple attacks that do not modify sender credentials.

2. Always be wary of unsolicited files - double check with sender

3. Always be wary of unsolicited instructions - double check with relevant department

4. Always be wary of unusual requests from management or collegues, especially asking for payment or information - double check with sender
