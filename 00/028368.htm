Storing the description in the second table is redundant, since it depends entirely on the Feedstock Preparation ID.

You can display the description as follows:

  • Activate the Create tab of the ribbon.
  • Click Query Design.
  • Select the first table and click Add.
  • Select the second table and click Add.
  • Click Close in the Show Table dialog.
  • If you have already created a relationship between the tables, Access will have "joined" the tables by drawing a line from Feedstock Preparation ID in the first table to the field of the same name in the second table.
  • Otherwise: click on Feedstock Preparation ID in the first table and drag it to the second table; drop it onto the field of the same name there.
  • Double-click the line that joins the two tables.
  • Select the option to include ALL records from the second table and only related records from the first table, then click OK.
  • Add all fields from the second table that you need from the list of fields to the query grid below (you can drag them, or double-click them).
  • Add the description field from the first table to the query grid below.
  • Save the

    When you open this query in datasheet view, you'll see the description from the first table alongside the fields from the second table. You can use this query as record source for forms and reports.

    mike h.. say:

    Hi,

    You have an answer to trapping cancel but but it will only work if you have a default value in the input box and if you do I don't see any sense in the DO loop.

    You're seeding the input box with a default value of 1 so if a user presses OK accidentally then the DO loop evaluates as TRUE and on we go. If you don't use a default value then pressing OK evaluates the same as pressing cancel so your not really trapping cancel at all. I suggest you try something like this with no default value

    Sub Test_For_Cancel()
    Do
    ReportLength = Application.InputBox("LengthMsg", "Heading")
    If ReportLength = False Then
    MsgBox "User pressed Cancel"
    Exit Sub
    End If

    Loop Until ReportLength >= 1 And ReportLength <= 12
    End Sub

    The "default contact folder" is contained in the data file designated as the "default data file" for the Outlook profile. Each email account will be associated with a specific data file. This will also change the "default calendar folder" for the profile

    To change the default data file, from OL menu, go to File --> Account Settings --> Data Files tab --> highlight which data file you want to set as the default and click on "Set as default" button.

    Changing the default data file is completely separate from the "default email account" setting.
