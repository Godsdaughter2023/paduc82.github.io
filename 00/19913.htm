brian tillman [outlook mvp 2007-2019] say:

It doesn't matter much at all how much memory your current processes take.  In fact, you WANT as many processes as possible to be in memory.  Memory sitting idle is a sign of bottlenecks in other areas of your PC, like CPU or disk I/O.  It's expensive to page programs into physical ememory when you want to schedule them.  It's cheap for the scheduler to switch between processes already in memory, thereby allowing processes to run without having to wait for them to be loaded from disk first.

 

Windows is a virtual memory system.  As processes require physical memory, if there's not enough memory to hold the next process scheduled for execution, Windows will swap out processes that don't require the CPU and allocate the memory to those processes that are ready to run.  Don't worry about any particular process taking up memory.

 

With regard to "only MSN PPL can post pics here", that's simply not true.  Anyone can post a picture.  See http://answers.microsoft.com/en-us/feedback/wiki/feedback_other-na/how-to-embed-images-on-microsoft-community/f74dac61-38d6-492e-8e71-286ab2645b2a
