To boot a device in UEFI BIOS mode, the USB must be formatted FAT32, not NTFS. So creating it on another PC, that is not setup to match your new one, could be the issue.

If you are able to download the ISO file for Windows 10, which is one Media Creation Tool option, you can use another utility called Rufus, to create the USB stick. Rufus has options to make the resulting USB stick bootable on a Legacy BIOS device, or on a UEFI device.  The same ISO file can be used to create either version of a bootable USB stick (plus you can recreate the USB stick as often as you need to).  A new PC ought to be setup as UEFI, which means a GPT-initialized hard disk, as opposed to an MBR-initialized hard disk like older PCs used.

https://rufus.akeo.ie

In your PC BIOS/Firmware settings, enable UEFI with Secure Boot enabled.  Then boot from the USB stick that has been prepared to be bootable by a UEFI-enabled device.

You're seeing what AOL call a 'temporary system error.' This is usually the result of some glitch at the server, but I think that in your case and similar ones, it's a bit more serious and a bit less temporary. For some years now, AOL has not really been supporting POP3, but there were so many Verizon customers used to it who transferred across when Verizon buried their own email service that they had to relax their requirements a bit. Now it looks as if they've stopped supporting POP3 altogether, at least for some customers. The remedy would be to start using IMAP instead. 

iPhone and iPad haven't used POP3 for a long time - if they ever did - so when you set up the account on these Apple devices, it uses IMAP by default. That's why you don't experience the same problems there. It could well be that AOL's servers detect that an IMAP client is accessing the account, and so disable POP3 access for it to prevent the problems that this 'hybrid access' would cause.

The instructions for setting up an IMAP account are pretty clearly explained on this page: Configure a Verizon.net account using IMAP There is a pretty basic explanation of the difference on this page: What is the difference between POP3 and IMAP?

jose est say:

Hi,

If you have the Windows "Home" version, you won't be able to turn off automatic updates. The only thing we can do is suspend or stop Windows Update from automatically rebooting the computer.

Maybe you don’t mind the automatic downloads, but you just don’t want Windows to restart while you’re in the middle of something. Windows 10 lets you set a 12-hour window called “Active Hours” in which it won’t automatically reboot.

To set Active Hours, go to Settings > Update & Security > Windows Update. Click “Change Active Hours” under Update Settings. You will be able to set the times you don’t want Windows to restart automatically.

You can also override the active hours. To do this, go to Settings > Update & Security > Windows Update > Restart Options. You can set a custom restart time when your device will restart to install updates.

NOTE: This is a one time setting, and you can only set a custom restart time if your device needs to restart to install updates.

If Windows needs to restart, use the “Restart Now” button on the Settings > Update & Security > Windows Update screen to restart immediately.

Let us know if you need anything else.
