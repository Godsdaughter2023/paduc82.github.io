john w. vinson say:

First off... consider that Microsoft's so-called "Lookup Field" is often considered a mistake in design. See http://www.mvps.org/access/lookupfields.htm for a critique.

Lookup tables are fine, and universally used; but they should be used on Forms, not in tables. On a Form, a Combo Box control has a "Not In List" event which is designed for just this purpose; you can use it to add a new value to the lookup table programmatically whenever needed. See the VBA Help for the event for an example or post back.

Note that you should not be using table datasheets for routine interaction with data in any case; they're limited (no Not In List event among many other limitations) and inflexible. Forms are the way to go!

    Note If your phone is currently stuck at the OEM splash screen, you may have to remove the battery to turn it off.
