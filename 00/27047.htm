mike h.. say:

Hi,

There are a couple of things wrong here.

This line puts a load of zeroes in the range because J has not been assigned a value.

Range("H2:H" & lr).Value = J

This line will fail because the lookup range is a single column and your asking for the result from column 2 to be returned.

J = Application.WorksheetFunction.Vlookup(Range("G2"), Sheets("TaskCode").Range("C:C"), 2, 0)

Also in Vlookup in VBA I would always error trap in case the lookup value can't be found. I would also qualify all ranges with a worksheet name. Is this close to what you want

Sub VlookupJobAdmin()
 Dim TimeT As Worksheet
 Dim lr As Long
 Dim J As Long
 Set TimeT = ThisWorkbook.Sheets("TimeTicket")
 lr = TimeT.Cells(Rows.Count, 2).End(xlUp).Row
On Error Resume Next
J = Application.WorksheetFunction.VLookup(TimeT.Range("G2"), Sheets("TaskCode").Range("C:D"), 2, 0)
On Error GoTo 0
TimeT.Range("H2:H" & lr).Value = J
 
 End Sub

jeeped say:

You seem to be confusing two things. In one, you are putting a formula into a number of cells in column H. All of the formulas are slightly different (e.g. G2, G3, G4, etc). In the other, you are using VLOOKUP() to get a single value and then putting that single value into all of the cells in column H. The value is not going to change. You are going to get the same value for all of the cells in column H.

Try this,

TimeT.Range("H2:H" & lr).Formula = "=VLOOKUP(G2,Table24[[TaskCode]:[JobAdmin]],2,FALSE)"

TimeT.Range("H2:H" & lr) = TimeT.Range("H2:H" & lr).Value

Task Scheduler is lying to us. The Windows Defender Scheduled Scan task isn’t actually completing successfully – as evidenced by the 0x2 Last Run Result in Task Scheduler; and by this entry in the Windows/Temp/MpCmdRun.log file:

-------------------------------------------------------------------------------------

MpCmdRun: Command Line: "C:\Program Files\Windows Defender\\MpCmdRun.exe" Scan -ScheduleJob

 Start Time: ‎Thu ‎Feb ‎09 ‎2017 19:43:02

Starting RunCommandScan.

This is a known issue with Defender scheduled scans in Version 1607: Any scheduled scan that runs under the System user account will only run intermittently. That includes the daily Automatic Maintenance scans; any scans that are scheduled by adding a trigger to the Windows Defender Scheduled Scan task; and also any scans that are scheduled with the Create Basic Task Wizard and edited to run under the System account. Changing the user account for the Windows Defender Scheduled Scan task to the local administrator has been reported to resolve the issue for Automatic Maintenance scans, and also for scans scheduled with the Create Basic Task Wizard. It should also resolve the issue for scans scheduled by adding a trigger to the Windows Defender Scheduled Scan task, although I haven’t confirmed that. As far as I know, the classic issue of scans only running during system-idle states still hasn’t been resolved with the add-a-trigger method, and I generally recommend against that method.
