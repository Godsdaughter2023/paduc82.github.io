OK, not quite as straightforward as I anticipated :(

There are two main problems.
1. You cannot grab a selection from a message body without additional software
2. The format of the date you select must be convertible in vba to a recognised date format and with random dates styles that may not be possible.

Taking 1 first - you will need to download Outlook Redemption and install the personal developers version. Instructions are included in the download file. Then having registered the DLL as instructed, from the Outlook VBA editor Tools > References, locate and check 'Redemption Outlook and MAPI COM Library'. The following macro uses Redemption to recover the selected text.

2. I have done my best to correct likely date formats to an acceptable format, so see how you get on with

Sub CreateAppt()
Dim olAppointment As Outlook.AppointmentItem
Dim strDate As String
Dim datOutlookDate As Date
Dim olInsp As Object
Dim olMsg As MailItem
Dim strAsk As String
Set olMsg = ActiveExplorer.Selection.item(1)
Set olInsp = CreateObject("Redemption.SafeInspector")
olInsp.item = Application.ActiveInspector
strDate = olInsp.SelText
strDate = Replace(strDate, ",", "")
strDate = Replace(strDate, "st", "")
strDate = Replace(strDate, "ST", "")
strDate = Replace(strDate, "nd", "")
strDate = Replace(strDate, "ND", "")
strDate = Replace(strDate, "rd", "")
strDate = Replace(strDate, "RD", "")
strDate = Replace(strDate, "th", "")
strDate = Replace(strDate, "TH", "")
strDate = Replace(strDate, "Monday", "")
strDate = Replace(strDate, "Tuesday", "")
strDate = Replace(strDate, "Wednesday", "")
strDate = Replace(strDate, "Thursday", "")
strDate = Replace(strDate, "Friday", "")
strDate = Replace(strDate, "Saturday", "")
strDate = Replace(strDate, "Sunday", "")
strDate = Replace(strDate, "MONDAY", "")
strDate = Replace(strDate, "TUESDAY", "")
strDate = Replace(strDate, "WEDNESDAY", "")
strDate = Replace(strDate, "THURSDAY", "")
strDate = Replace(strDate, "FRIDAY", "")
strDate = Replace(strDate, "SATURDAY", "")
strDate = Replace(strDate, "SUNDAY", "")
strDate = Trim(strDate)
If IsDate(strDate) Then
    datOutlookDate = CDate(strDate)
    Set olAppointment = CreateItem(olAppointmentItem)
    With olAppointment
        .Start = datOutlookDate
        .ReminderSet = True
        .AllDayEvent = True
        .Subject = olMsg.Subject
        .BusyStatus = olBusy
        .Display
    End With
Else
    MsgBox "Operation cancelled - " & vbCr + vbCr & _
           strDate & vbCr + vbCr & _
           "is not a valid appointment date format." & vbCr & _
           "Select the date and re-run the macro", _
           vbInformation, "Invalid date selection"
    GoTo CleanUp
End If
CleanUp:
Set olAppointment = Nothing
Set olInsp = Nothing
Set olMsg = Nothing
End Sub

The macro will run on a date selected in the dody of a message either in the open message or the message previewer. If there is no year associated with the date, the current year is used. Add the macro to the Outlook 2010 ribbon or QAT.

 Hi Willy,

Thank you for posting your query on Microsoft Community.

Basic storage uses normal partition tables supported by all Windows versions right from MS-Dos, while a Dynamic disk storage has the feature that basic disks do not have, such as support for volumes spanning multiple disks, RAID 1 or RAID 5 disk arrays.

The downside of use Dynamic storage is that once the disk converted to Dynamic and being partitioned and used it cannot be converted back to Basic without losing data, as the Convert to Basic Disk option will be greyed out once it’s been used.

Note: I would suggest you to back up all your data before taking any steps.

Follow these steps to convert Dynamic partition into basic partition:

1. Open the computer management console. To open this console, click the "Start" button, then "Control Panel," then double-click the "Administrative Tools" icon, and then double-click "Computer Management."
2. Click the "Disk Management" heading in the leftmost pane of the Computer Management window.
3. Right click all volumes contained in the dynamic disk you wish to convert.
4. Select "Delete" from the context menu that appears.
5. Right click the disk after all volumes are deleted and select "Convert to basic disk." This will convert the disk and set it up for normal usage.

rohn007 say:

I don't know about bookmarks, but the "old school" way to do what you are asking for is to use the built in heading styles and document map.

You can change the look of the default heading styles as much as you want to do what you want. Then apply the heading styles, say Heading 1 for Chapter, and heading 2 ... for lower subdivisions in the chapter you would like to break it up into.

Turn on the document map, sorry it is now called the "Navigation Pain" (sic, pane): View tab / Show group / Navigation Pane check box

NOTE: the "old" short cut <alt> <V> <D> still works to toggle it on and off. Or you can start right away by using the "Right", ie the new combo <ALT> <W> <K> (even though it makes absolutely no logical sense ... why W for view when there is no other tab names starting with "V" ?  And why "K" for "Navigation Pane"? I want some of what whoever assigned these shortcuts was "on" when they did it, Wheeeeee!

In the 2010 Navigation Pane there is the "Find" input box at the top and 3 mini tab buttons below it. What used to be call the "Document Map" is the leftmost tab, "Browse the headings in your document". (The middle one is a "snapshot" view of the document pages, and the rightmost one highlights snips of text around each "Find" hit)

In the Document map view you will see a text version of your headings. If the heading has sub headings there is a little grey triangle to the left of the heading text. You can click on this triangle to expand and compress that headings below that heading level.

The right context menu displays some heading related editing options. Check them out. The bottom three are relevant to this discussion: Expand All, Compress All (down to Heading 1's only), and Show Heading Levels ddl. The Show Heading Levels is handy if you only want to see headings down to a specified level, ie H1 & H2, or H1 H2 H3 & H4.

2010 has also introduced a new feature. Just like Outline view, in the navigation pane you can click on a heading and drag it, and it's associated text, to a new location in the doc.

brian tillman [outlook mvp 2007-2019] say:

I'm unsure why you marked my previous post as the answer when, apparently, it did not solve the issue.  Please unmark it and, in the future, mark as answers only those posts that actually solve your problem.

 

The best way to handle junk mail on Outlook.com is to change your junk filter to "Exclusive".  Click the Options gear, then "More mail settings".  Click the link labeled "Filters and reporting" in the right-hand column.  Select the radio button labeled "Exclusive".  Click Save.  Subsequently, only messages from people in your Contacts or Safe Sender list will be delivered to your Inbox.  Periodically (every few days), review the subjects and senders of messages in your Junk folder.  If you see any that should have been delivered to Inbox (false positives), then mark them as "not junk" and put the senders (or the senders' domains) in your Safe Sender list.  From then on, those messages will make it to the Inbox.  After a while, the vast majority of good messages will be delivered properly and the vast majority of junk will arrive in the Junk folder, to be automatically deleted after ten days if you don't empty the folder sooner.
