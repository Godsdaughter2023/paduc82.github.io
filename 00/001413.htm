1. Login your EASI account in Outlook.com.

2. Click the gear icon at the upper right and then select "Options".

3. Under "Managing your account", click "Import email accounts".

1. Login your EASI account in Outlook.com.

3. Under "Managing your account", click "Your email accounts"

4. Under "Add an email account", click "Add a send-and-receive account" button.

5. In the next screen, click "Advanced options".

6. Enter all needed information specially the POP and SMTP settings of your Gmail account. To get the correct POP and SMTP settings of your Gmail account, I suggest to get in touch with Gmail support.

hansv mvp say:

Add an error handler:

 

Private Sub Command1_Click()
    Dim stDocName As String
    Dim stLinkCriteria As String

    stDocName = "frmPPHShowCheckOut"    
    stLinkCriteria = "[ShowID]=" & Me![Combo0]
    On Error GoTo ErrHandler
    DoCmd.OpenForm stDocName, , , stLinkCriteria
    Exit Sub

ErrHandler:
    If Err = 2501 Then
        ' Opening the form was cancelled, we can ignore this
    Else
        ' Something else went wrong, inform the user
        MsgBox Err.Description, vbCritical
    End If
End Sub

So the default behavior of filtering the data only based on the current employee is great... but once in a while I want to override the default behaviour and show all the values.

hansv mvp say:

When you add a subform to a form, Access tries to guess on which field or fields the subform should be linked to the main form, and it enters the name of that field / those fields in the Link Master Fields and Link Child Fields properties of the subform control on the main form (in the Data tab of the Property Sheet).

If you don't want a subform to be linked to the main form, clear its Link Master Fields and Link Child Fields properties. It will then display ALL records regardless of which record is active in the main form.
