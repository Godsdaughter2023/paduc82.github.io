Just to let you all know that I have found the registry key to turn off connected standby. So you can all stop raging and take a sigh of relief.

Go to:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Power. There is a DWORD called "CsEnabled" which you should find set to 1 and changing it to 0 will turn connected standby off.

I have a Surface Pro 3 and can confirm that since making the change my battery life has dramatically improved. Also on the battery meter, previously it just showed the percentage remaining but now it shows the time remaining (e.g 8Hrs, 30 mins) as well.

a. user say:

Usually that message means that XP just can't find the file - probably due to some corruption in the NT File System (NTFS).  File System corruption can occur from things like power interruptions, using the power button on your system while XP is running, aborted restarts, ungraceful shutdowns, etc.

When your system is running normally, if it crashes or you locks up and you have to use the power button or unplug it to get it going again, you need to stop doing things like that and figure out what the real problem behind the crashes is and fix that or you will just keep having this problem.

When you do see it, you can see if running a chkdsk with error correction (chkdsk  /r) from the XP Recovery Console will straighten it out.  That is alot easier than reinstalling XP, but until you find out why it is happening and fix the real problem, you will probably just see it again.

If you check the HDD and are finding bad sectors, some could be for the reason listed above or it could be that your HDD is just starting to fail or has already failed and needs to be replaced.

I don't think your message CpuID 0f49 patch id :0003 has anything to do with XP - it has to do with your Gigabyte GA-945PL-S3 motherboard so you might want to look in the Gigabyte help forums for ideas on that.
