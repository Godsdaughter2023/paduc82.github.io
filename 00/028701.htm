barb way say:

There isn't a way to 'justify' page output in Visio.  When saving a drawing, it thinks of each page as an image, and so it is centered for visibility.  There is no concept in Visio for page level justification.

As a work around, you could create a template with a standard page outline that users could use to 'standardize' page output size and location, perhaps.  I have even seen drawings which have a rectangle that is 'drawn' with white lines (on the normal white background), and which is pushed to the back of the z-order, etc.  This creates an invisible 'page' outline so that the saved Visio page images appear in the same part of the window.

Other than that, there isn't a way to manage location on the rendered page in the browser when using Save as HTML from Visio.

james1509 say:

 To add to what the others posted ...

 Windows 10 did not originally accept the product keys for earlier versions.  If you upgrade a Windows 7 (or 8.1) installation, Windows 10 recognizes that fact, and no key should be required.  If your previous Windows 10 installation on that computer was activated, then you can hopefully just skip the product key and the new installation will automatically be activated.

 Otherwise, the recently released preview build 10565 includes the ability to use Windows 7/8.1 keys during installation, so that would be an option, if you don't mind using a preview build.  You can read more about that here:

http://answers.microsoft.com/en-us/insider/forum/insider_wintp-insider_install/activation-faq-for-insider-preview-build-10565/68e759a2-abcc-4b16-8758-92a720a6efe9
