sachinshakya say:

Give read access across all site collection and sub-sites wtih unique permission

We have requirement to give users read access to all site collection and sites with unique permission in sharepoint online. While this was easily achievable in on prmise through web application user policy, how can this be implemented in SharePoint online?

jamesyan_0708 say:

Hi Sachinshakya,

We can follow these steps:
1. Go to the sub-site > site settings > site permissions > Stop Inheriting Permissions > OK.

2. Go back to the Edit permissions page > select the user(s) > click Remove User Permissions > click Check Permissions > make sure the result is None, if not, we may need to remove the user from the specific group according to the Check Permissions result.

3. Click Grant Permissions > type the user's name, select site name Visitors [Read] permission level > Share.
Or we can just add the user to the site name Visitors group.

If we want to perform these steps for multiple users, we can also create a group with Read permission and add the users to it.

1. Stop Inheriting Permissions following the steps above.

2. Site settings > People and groups > More... > New Group > select Read permission level > Create.

3. Go the site permissions page > select the user(s) > click Remove User Permissions > click Check Permissions > make sure the result is None.

4. Go back to the People and groups page > select the group we just created > New > Add Users > type the users' names > Share.

Moreover, I understand this is inconvenient if we have lots of site collections to manage.For whether we can achieve this via PowerShell scripts, we would suggest you post in our SharePoint forum as that is the specific support channel for SharePoint customization related queries.

john korchok say:

You may have to do a lot of preformatting with your text to make Text to Table work. The Table to Text conversion relies on having a consistent separator for the items that are to become individual columns in each row. If you want to use the custom character ; (a semicolon), then you enter that character in the Separate text at>Other field in Convert Text to Table options and you also must have that character between every piece of information that is to form a new column.

Your sample only has one semicolon after 1.1, so using Text to Table with a semicolon would put 1.1 into the first column and everything else into the second column. If you want your sample turned into 1 row with separate columns for each item, put each item on separate line and set Separate text at to Paragraphs.

As I mentioned in my previous message, Text to Table can take a lot of formatting before you convert, to get the table you are seeking. In addition, each line must have the same number of separators to get a table that works:

1.1;;

STATE;STIME;FMRI

 file solarisappsummary.txt:

 file solarisdbsummary.txt:

Everywhere there is a separator (in this case a semicolon), you will get a new column. If there is only one item of text in a row, it must be followed by the same number of separators as there will be additional columns in the table, or Word will not know how to create the table.
