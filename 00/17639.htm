When faced with this kind of transcription, I use an acronym, and either place it in autocorrect, or, if it's related to the current transcription and not worth saving to autocorrect, after I'm done with the transcription, I'll go through and replace all instances of each acronym with desired text.

Example:  you know     = .yk

                so, like          = .sl

 in the office of the Recorder of Deeds in and for New Castle County, Delaware = .rod

in my humble opinion = .ho

and is this recording true and correct according to your information and belief = .truerecord

                                  

For my work, in order to not interfere with words that contain the letter combinations, I preface it with a period.  And in auto correct and in search and replace, I include the blank space before the period so it looks for text that begins with a space, a period, and some letter combination. 

I really dislike reentering all the "unh-unh", the "um", "so then I said", "like I said", "thank you", "yes, ma'am", "yes sir", etc. when transcribing interviews or depositions.

I'm not saying this is the correct or accepted way to handle these things, it's just a suggestion that works for me.

john p spencer say:

Try the following.  It should report all fields in the table that are null or blank and a count of the number of records that are affected.

Public Sub fFindNulls(strTableName As String)
Dim rstAny As Recordset
Dim iFldCount As Long
Dim iLoop As Long
Dim strFldName As String
Dim strSQL As String
Dim dbAny As DAO.Database
Dim iRecCount As Long
Dim strMsg As String

   Set dbAny = CurrentDb()
   Set rstAny = dbAny.OpenRecordset(strTableName)
   iFldCount = rstAny.Fields.Count
  
   For iLoop = 0 To iFldCount - 1
      strFldName = rstAny.Fields(iLoop).Name
      strSQL = "SELECT COUNT(*) FROM [" & strTableName & _
         "] WHERE Trim([" & strFldName & "] & '') = ''"
      iRecCount = dbAny.OpenRecordset(strSQL).Fields(0)
      If iRecCount > 0 Then
         strMsg = strMsg & iRecCount & " null values in field " & strFldName & vbCrLf
      End If
   Next iLoop

   MsgBox strMsg

End Sub

Sub Test()
  Dim CO As ChartObject
  Dim S As Series
  Dim P As Point
  Dim i As Long
  Dim Data
 
  'Create some data
  Worksheets.Add
  Range("A1") = "Nr"
  With Range("A2:A20")
    .Formula = "=RAND()"
    .Value = .Value
  End With
 
  'Create a chart above this cells
  With Range("C2:H20")
    Set CO = ActiveSheet.ChartObjects.Add(.Left, .Top, .Width, .Height)
  End With
  With CO.Chart
    'Set chart type and associate the data
    .ChartType = xlLineMarkers
    .SetSourceData Range("A1").CurrentRegion
    'Get the 1st series
    Set S = .SeriesCollection(1)
  End With
 
  With S
    'Read the Y-Values (Data becomes a 1D array)
    Data = .Values
    For i = 1 To UBound(Data)
      'The points are in the same order 1..x
      Set P = .Points(i)
      'Change the color of the markers based on the value
      If Data(i) < 0.5 Then
        P.MarkerBackgroundColor = vbRed
      Else
        P.MarkerBackgroundColor = vbGreen
      End If
    Next
  End With
End Sub

daniel g. grau say:

The extra "paper" you see is a fictitious area added to display comments shown as balloons. I guess they should have suppressed the misleadingly added white space.

Your screenshot depicts an 8.5 x 11 page. The ruler indicates a 7.5 wide text area, with 1/2-inch margins on each side and 1 inch at top and bottom. The ends of the rulers (in light blue) always coincide with the boundaries of the page.

Go to Word > Preferences > Track Changes and take the checkmark next to "Use balloons to display changes" and you'll see the addition disappear.

To change the page settings, use File > Page Setup and click on the orientation button.
