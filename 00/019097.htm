danielco say:

Assuming the pivot table name is PT1, paste the following code into the sheet module. Use the named range "rgPT" as the array :

Private Sub Worksheet_PivotTableUpdate(ByVal Target As PivotTable)

    With ActiveSheet.PivotTables("PT1")

        ThisWorkbook.Names.Add "rgPT", RefersTo:=.TableRange1

    End With

End Sub

Try this:

Public Function Lib_Forms() As Boolean
    Dim Filename As String
    Dim DestName As String
    Dim OrigPath As String
    Dim OrigFile As String
    Dim DestFile As String
 
    On Local Error GoTo Out
    OrigPath = "L:\legalper\lib\forms\"

    If Forms!DA!DA = 2 Then
        DestFile = "merge_li.docx"

    DestName = Tmppath & DestFile
    Filename = OrigPath & OrigFile

john - project say:

Tom,

A resource pool does not define "generic" resources. A resource pool is a convenient way to have a separate pool file with all the resources that will be assigned to multiple separate projects (sharer files). A generic resource is equivalent to your "imaginary resources". For example, a generic resource might be an "engineer" or "accountant" or "MTS" (member of technical staff). With generic resources you will not need to re-assign them, yet you will be able to see overallocations and do resource costing, assuming you can define a rate for each type of generic resource.

With regard to operating a linked structure over a LAN/WAN, it has a chance or working okay if the LAN/WAN is in the same building where it is likely to be more stable. Attempting to operate a linked structure over a network that has nodes in different locations is inviting trouble. Any glitch in the network, and those do occur, can corrupt the structure in a heartbeat. If you haven't seen a problem at this point, you are lucky.

Have you looked at Project Server or Project Online? Those are specifically structured for enterprise environments.

With regard to moving or re-saving existing linked files. The problem with moving or re-saving a linked file either creates link fragments or duplicate links (to the old structure and to the new structure). If you need to move a file that was part of a linked structure, first un-link it from the old structure, move it and then link it to the new structure. When links are broken, it is imperative that both files (i.e. source and destination) be saved in order to "capture" the change, otherwise a link fragment will remain in one of the files and that's a fragment you don't want to have.

If you are going to use a linked structure it is prudent to have a single individual maintain all files because properly maintaining that structure requires considerable discipline.

So to answer your question, use generic resources and you won't need a way to "batch change resource names".

John

private void editResourceRibbonButton_OnClick(object sender,
    IRibbonControl control, bool pressed)
{
    MSProject.Project currProject = null;

    MSProject.Resource resource = null;

        currProject = MSProjectApp.ActiveProject;

        if (resources != null) Marshal.ReleaseComObject(resources);
        if (currProject != null) Marshal.ReleaseComObject(currProject);
    }
}
