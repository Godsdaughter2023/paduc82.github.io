ossiemac say:

The original workbook containing the code can be referenced as ThisWorkbook.

When opening a workbook it is best to assign the workbook to a workbook variable and then that variable can be used for all future references to the opened workbook.

Ensure that you have a backup of your workbook before testing the code below.

It is almost never necessary to select ranges when working with VBA. Simply reference the range as per the code below.

Sub test()
    Dim ws1 As Workbook

    Sheets("Sheet 1").Select      'This select OK to ensure starting at correct worksheet.
    Range("A1:D100").ClearContents
    Set ws1 = Workbooks.Open(Filename:=Range("Q1").Value)
    Range("A1:D100").Copy
    ThisWorkbook.Activate
    Range("A1").PasteSpecial Paste:=xlPasteValues, _
               Operation:=xlNone, SkipBlanks:=False, _
               Transpose:=False
    Application.CutCopyMode = False
    ws1.Close SaveChanges:=False
    Range("A1").Select
End Sub

If Word is showing a difference in the section number, then you do have section breaks. Do you have a multi-column section somewhere on that page? That will require a section break before and after.

doug robbins - mvp office apps & services (word) say:

Have you tried to execute the replace?  While it does show the Find and Replace dialog does show Indent First line : 0 ch, behind the scenes, what is searched for is:

    With Selection.Find.ParagraphFormat
        .SpaceBeforeAuto = False
        .SpaceAfterAuto = False
        .FirstLineIndent = InchesToPoints(0.24)
        .CharacterUnitFirstLineIndent = 0
    End With

and if you execute the replace, you should find that the style has been applied.  Note that if you don't specify the SpaceBefore and SpaceAfter it will probably default to the values shown above and if the actual spacing is different, the execute will not work.
