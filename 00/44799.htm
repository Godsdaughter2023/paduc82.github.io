cmdrkeene say:

Use question marks instead of asterisks and it should work.

copy <filepath>\spreadsheet ??.??.2014 <destination folder>

An asterisk is a wildcard for infinity number of characters.  Question mark is a wildcard placeholder for one (and must be one) space.  So ??.?? guaruntees it only matches files that have two characters, then a dot, then two characters, then a dot with 2014.

Whether or not you can delete previous backups and still have viable backups in the remaining files depends entirely what backup program you use and how you have it configured, neither of which you've mentioned.  You can configure backup programs to make full backups or incremental backups.  If you make full backups, each backup is completely self-contained.  If you make incremental backups, then a restore will require the most recent full backup plus all of the incremental backups since that full backup to reproduce the current state of the disk.
