john w. vinson say:

It may just be me, but I avoid ever using input masks for date/time. They justprevent users from entering data by forcing them to enter too much. You can set the Format property of the field to "mm/dd/yyyy hh:nn:ss am/pm" to display the data as you wish to see it; if the user types "8/4 10pm" the field will be filled with a double float date/time numeric value, which will be displayed as "08/04/2010 10:00:00 PM" with no further action required and at the savings of quite a few unnecessary keystrokes.
