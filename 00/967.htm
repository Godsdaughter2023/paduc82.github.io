a. user say:

The following process works OK for me - I am assuming that your chapter headings are Heading 1 headings so that a Figure caption would be like this:

Figure { STYLEREF \s 1 }.{ SEQ Figure \* ARABIC \s 1 }

(If you are using Heading 2 then the "\s 1" would be "\s 2". Also, if the spacing and/or capitalisation is different, e.g. you have \*Arabic instead of \* ARABIC, adapt the below instructions accordingly.

1. Use Alt-F9 (Option-F9, or fn-option-F9 or fn-alt-F9 depending on your keyboard preferences) to display the field codes.

2. Click Edit->Find->Advanced Find and Replace...->Replace

3. Find

^d STYLEREF \s 1

and replace it with

@@@@@

(By which I mean a text string that you do not have in your document)

4. Find

@@@@@.

and replace it with nothing.

5. Find

SEQ Figure \* ARABIC \s 1

and replace it with

SEQ Figure \* ARABIC

Then update all the fields and verify that it has worked.

If so, repeat with appropriate changes for the tables and maps.

a. user say:

Word, out of the box, doesn't have a multiselect combo box content control, and the legacy form field control does not allow multi-select either. So although you say you don't understand VB etc., unfortunately you do end up having to do "programming" in either VBA or VB.NET+VSTO to do something along these lines. Typically, using VBA you would actually create a "UserForm" with a multiselect control in that. Userforms are "popups", i.e. their content does not appear in the body of the document.

Except that if you happen to be using a Sharepoint server, you can define a Sharepoint column as a multiselect list. Then you can make multiple selections, but only in the "Document Information Panel (DIP) - such controls cannot be inserted in the body of the document. Further, with this particular type of control, I do not see a way to access the values you chose either in the document or even using VBA.

Just to be sure, when you say "Word 2010-form maker", I am assuming that you are referring to the facilities in the Developer tab in Word. If you are actually referring to a third-party tool for creating forms in Word, you probably need to consult the provider of that tool.

My final question is what result(s) you would hope to see as a consequence of a multiselect - would you hope to see each selected item in the document, either one after the other or perhaps each on a separate line? Or what? I ask in case someone could think of a different approach that would achieve what you needed.
