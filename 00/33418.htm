Firstly, let me paste a screenshot to show off my success.

My solution:

1. Open C:\Windows\Logs\CBS\CBS.log

2. Search for "Error STATUS_INVALID_PARAMETER originated in function ComponentStore::CRawStoreLayout::RecursivelyRegenerateComponentPayload"

3. You will find something like

2017-07-06 20:17:15, Info                  CSI    000007c9 Regenerating payload files from delta files on component: [l:85 ml:140]'amd64_microsoft-windows-shell32_31bf3856ad364e35_10.0.15063.296_none_010a97c27872cb6a'
2017-07-06 20:17:16, Error                 CSI    000007ca@2017/7/7:03:17:16.01 (F) onecore\base\wcp\componentstore\deltastore.cpp(3320): Error STATUS_INVALID_PARAMETER originated in function ComponentStore::CRawStoreLayout::RecursivelyRegenerateComponentPayload expression: Parameter check failed
[gle=0x80004005]

Pay attention on the bold part, that is the name of corrupted component, copy & paste, we will use it later.

4. Open the Registry Editor (press Win+R, enter "regedit"), find the key "HKEY_LOCAL_MACHINE\COMPONENTS\DerivedData\Components\amd64_microsoft-windows-shell32_31bf3856ad364e35_10.0.15063.296_none_010a97c27872cb6a". (The bold part comes from the previous step)

NOTE: If "HKEY_LOCAL_MACHINE\COMPONENTS" is missing on your computer, please follow these steps:

    a. Open "File Explorer", paste "Control Panel\All Control Panel Items\Programs and Features" to the address bar, press Enter.

    b. On the left side, you can find something like "Turn Windows features on or off", open it and leave it alone.

    c. Go back to Registry Editor, press F5 to refresh, now you should see our fancy dude "HKEY_LOCAL_MACHINE\COMPONENTS".

5. If you successfully find the path, check the value of the key "CF", we expect to see a hexadecimal value, e.g. 0xA80. Open the Windows calculator or find an online calculator, calculate the result of the value minus 0x200, e.g. 0xA80 - 0x200 = 0x880. Then, change the value of key "CF" to the result.

6. Reinstall the KB4022725 update in Windows Update, if it still fails, go back to step 1 and we should be able to find out another corrupted component in CBS.log. I repeated the whole process twice, you may expect to have the same workload.

WHAT HAPPENED BEHIND THIS BUG

This is a really stupid bug that shouldn't happen on Microsoft. That 0x200 value in "CF" key we subtracted, is a flag which indicates that "this component has been compressed", but actually it's not. When we install the update, Windows system will check the CF flag and try to unzip the component, ummm, we expect a failure here, I mean, what else you can expect when you are trying to unzip a non-zip file?

A tricky thing about this bug is, those CF flags can be broken even on a fresh installed Win10. I've reinstalled my system for several times to get rid of this bug, but it doesn't work! Truth be told, this is the time when you wish yourself bought a Mac.

I'm a usability specialist and I understand your frustration.

In the instructions below, square brackets refer to information that is variable from machine to machine, and where you are told to type something, don't type the square brackets. Also, do not type the outer double quotes, but rather, type what's between them.

Right-click the Windows button at the lower left of the tray.

Select "Run". (If there is an option "Run as an Administrator", select that instead.)

In the window titled "Run", in the textbox type "cmd" and press Enter.

A new window will open, titled something like "C:\WINDOWS\system32\cmd.exe". It has a flashing DOS prompt next to "C:\Users\[your_name_here]>"

Type "diskpart" and press Enter.

A new window will open that looks like the window that is already open, but the title will be instead "C:\WINDOWS\system32\diskpart.exe"" It will have a flashing DOS prompt next to "DISKPART >".

Type "list volume" and press Enter.

Text will appear on several lines: A header with "Volume ### Ltr Label Fs Type Size Status Info" and a line of dashes below that, then a set of rows with "Volume [numeral]" at the beginning of each row.

Type "select volume [x]" where [x] is the "Volume" number on the same line with the offending disk drive under the heading "Ltr", then press Enter. E.g. if the offending disk drive is "E", to the left of it will be an item such as "Volume 3" in which case you want to type "select volume 3" and press Enter.

Type "remove letter="[x]"" using the same letter as in the previous step, then press Enter. In this case, DO type the inner double quotes.
