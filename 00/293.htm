scott mcdaniel say:

I've always found it easiest to import the data from Excel into a "staging" table in Access, and then use SQL and VBA to eliminate duplicates.

You can then create a query to get a distinct combination of DoID and LtID:

Dim rst As DAO.Recordset

Set rst = Currentdb.OpenRecordset("SELECT DISTINCT DoID, LtID FROM YourStagingTable")

Do Until rst.EOF

  Currentdb.Execute "INSERT INTO YourLiveTable(Col1, Col2, etc) VALUES(" & rst("Col1") & "," & rst("Col2") & ", etc)

  rst.MoveNext

Loop

You could also use a Recordset in the loop, if it's easier for you to understand:

Dim rstNew As DAO.Recordset

  

Do Until rst.EOF

  rstNew.AddNew

  rstNew("Col1") = rst("Col1")

  rstNew("Cod2") = rst("Col2")

  etc etc

  rstNew.Update

  rst.MoveNext

Loop

ken sheridan say: You should be able to delete the redundant rows after the import with:

DELETE *
FROM YourTable AS T1
WHERE LtID IS NULL
OR LtID = ""
OR LtType <>
     (SELECT MAX(LtType)
      FROM YourTable AS T2
      WHERE T2.DoID = T1.DoID
      AND T2.LtID = T1.LtID);

This should retain all rows with the LtType which sorts first alphabetically per matching pair and where the LtID column is not Null or a zero-length string.

karl timmermans [outlook mvp 2012-2018] say:

Outlook is continually downloading emails back 6 months.


POP email issue fixed - see first post in the following thread

FIXED 3/4/16: Problems with POP accounts in Outlook 2016: Email is deleted from server or duplicated in Inbox
http://answers.microsoft.com/en-us/office/forum/office_2016-outlook/fixed-3416-problems-with-pop-accounts-in-outlook/79e8177f-b04e-491e-9a82-ae5b6565d3c0 

Pay special attention to the following in MS KB 3145116 - https://support.microsoft.com/en-us/kb/3145116

Note: After the fix is applied, Outlook may download all email one final time. If this occurs and email is duplicated in your Inbox, use the method described in the "More Information" section.

The operative word in the above is "may". If you have a lot of data in your <Inbox> and want to take some additional precautionary steps avoiding newly downloaded dupes as an alternative to what's suggested - the process is as follows <before> connecting to the POP account the first time with the updated build

#1 - Create a new mail folder
#2 - Highlight your Inbox
     --> move focus to <Inbox> items
     --> CTRL+A to highlight all items
     --> right-click 
     --> select <Move> 
     --> Other folder 
     --> select the newly created folder
#3 - If the update results in everything being downloaded again, you will have a clean inbox without dupes. If the items are not downloaded, then, you will need to reverse the process and copy everything back to the <Inbox> 

Also, for those using <rules> to move inbox items to other folders, you will probably want to turn those off during the first connection to your account in the event that everything gets downloaded again - else you will have dupes in all the other folders.

stefan blom say:

To force a particular zoom on all documents, you can make use of a macro:

You can run a macro such as the following:

Sub SetOnePageDisplay()
'Macro created by Stefan Blom, MVP, January 2017
With ActiveWindow.View
.Type = 3  'Print Layout view
.Zoom.PageColumns = 1
.Zoom.Percentage = 125  'Zoom to 125%
End With
End Sub

See also

On the other hand, if you want to store the zoom with a document, you'll need the SaveView2013.zip add-in by MVP Jay Freedman. Go to https://jay-freedman.info/.
