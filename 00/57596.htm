@echo off

subinacl /subkeyreg HKEY_CURRENT_USER /grant=system=f

subinacl /subkeyreg HKEY_CLASSES_ROOT /grant=system=f

subinacl /subdirectories %SystemDrive% /grant=system=f

 

@Echo =========================

@Echo Finished.

@Echo =========================

@pause

Sometimes with those errors about missing files the problem is not that the file(s) are actually missing but the problem is that Windows just can't find the files because of what is hopefully minor corruption of the NT File System (NTFS).  

An ungraceful shutdown like power off/plug/pulling/reset button etc. are great ways to cause some NTFS corruption.

Windows can't find the file so it reports the file(s) as "missing".

It could also be the reason there are no Restore Points - Windows can't find them.

From the F8 Advanced Boot Options menu if you choose to disable restart on system failure what does the entire error message say?

My notes say for those C0000135 errors to always start with a chkdsk with error correction (chkdsk  /r) which you can run from a Command Prompt window.

aric williams say:

I fixed it!
And I even suggest adding this to your repertoire of Wi-Fi Fixes.

What I did;

It appears that, for whatever reason, The Default Gateway didn't change or update or 'Something' when we got Comcast. 

1. Go to Network and Sharing center by whatever means you do.

2. Choose "Change Adapter Settings"

3. Double-click on your internet connection, for me it was my Intel Wifi. 

(Or; Just Right-click and click 'Properties')

4. Click 'Properties'.

5. Go to Internet Protocol Version 4 (TCP/IPv4) and again click 'Properties'.

6. Click 'Advanced' and Under Gateway click 'Add'.

7. Type in the same Number you would type into your Address bar to log in to your router.

8. Restart.

Boom. Hasn't given me trouble since.
