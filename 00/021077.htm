wwein say:

I mostly agree with Steven.

 

Explanation

What we see by going to "C:\Windows\Downloaded Program Files" is not in fact a list of files. Microsoft, in their infinite wisdom, decided to make this and a small handful of other folders act as though they're showing files, but actually list something slightly different - the contents of a section of the registry. The list of entries you see when visiting this folder is simply the list found here in the registry:

Computer\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Code Store Database\Distribution Units

You can also prove to yourself that these aren't the actual contents of the folder by opening a command prompt (Start ->Run , type:cmd , hit enter), navigating to the folder (type the following and hit enter:cd "c:\windows\Downloaded Program Files" ), and then displaying the contents (type the following and hit enter:dir )

Normally when an ActiveX control is installed it tells Windows what name it goes by, and then Windows references that name against the unique number (hex) it gives for each program. You'll notice that most of the ActiveX controls in your troublesome folder have sensible names, while the one in question is written in hex - ie some might say "Java Runtime" or "Facebook Photo Uploader" rather than "{E2883E8F-472F-4FBO-9522-AC9BF37916A7}". This either means the ActiveX didn't give Windows a name for it to go by (usually means it's malware), or as in your case, it didn't install successfully.

The solutions several people above have mentioned is to reinstall and then uninstall GetPlus, as{E2883E8F-472F-4FBO-9522-AC9BF37916A7} is indeed the hex name for that activeX control. This would normallywork because doing so would correctly install the ActiveX control and create the files it needs, and then in the process of uninstalling would not only remove these missing files but would remove this reference in the registry. The GetPlus control is the software that Adobe uses to help you download Flash, Adobe Acrobat etc, rather than being an actual part of Flash. Uninstalling/reinstalling Flash won't therefore help. I think someone above mentioned uninstalling GetPlus via a utility that removes it (rather than using Add/Remove Programs) but I can't find that post, so maybe not.

 

Solution

In any case, here is the solution to your problem, which does involve the registry. But as you mentioned being worried about messing with the registry, I've included a method to undo all our changes to reassure you.

1. Go to Start , then Run

2. Type: regedit

3. Hit enter

4. In the window that opens, double click on the folders on the left to navigate to:

Computer\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Code Store Database\Distribution Units

5. Double Click on 'Distribution Units ' to show the contents.

6. You'll see several entries, all with names like {8f0011....} etc. Right click the one marked:

{E2883E8F-472F-4FBO-9522-AC9BF37916A7}

7. Click Export . A window will then pop up asking you where to export (ie save/backup) this registry value. Pick any folder and any name you want and clickSave when ready. You can skip this step altogether if you like, it's just for peace of mind.

8. Again, right click :

{E2883E8F-472F-4FBO-9522-AC9BF37916A7}

9. Now select Delete . You can close the Registry Editor now.

10. Download and install/update Flash (to fix your problem at Youtube) from here:

http://get.adobe.com/flashplayer/

11. You don't need to agree to download the Google Toolbar, but do agree to the ActiveX control and everything else in the windows that follows.

12. Finally open a windows explorer window and navigate to:

C:\Windows\Downloaded Program Files

Not only will the entry {E2883E8F-472F-4FBO-9522-AC9BF37916A7} finally be gone, but you should now correctly haveget_atlcom Class in it's place. For a final confirmation, right click it, select properties, and look at the ID.

Look familiar? :)
