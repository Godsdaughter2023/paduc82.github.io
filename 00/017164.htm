anna_82 say:

How to password protect a folder in Microsoft Office 2003

graham mayor say:
Office 2003 has no facility to password protect folders. You can individually password protect the files in the folders; and through the operating system restrict access to the folders.

If your Buffalo drive is a NAS drive in addition you can probably restrict who accesses that drive also, through the NAS file sharing.

To protect the files in the folder, run the following macro on the folder and don't forget the password!!! or you won't be able to open them again! http://www.gmayor.com/installing_macro.htm

Public Sub PasswordAll()
Dim strFileName As String
Dim sPassword As String
Dim bPassword As String
Dim strPath As String
Dim oDoc As Document
Dim bBack As Boolean
Dim Response As Long
Dim fDialog As FileDialog
Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)
With fDialog
    .Title = "Select the folder containing the documents to protect and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , "Add Password To All"
        Exit Sub
    End If
    strPath = fDialog.SelectedItems.Item(1)
    If Right(strPath, 1) <> "\" Then strPath = strPath + "\"
End With
OpenPass:
sPassword = InputBox("Enter the Password", "Password to Open")
bPassword = InputBox("Re-enter Password", "Password to Open")
If sPassword <> bPassword Then
    MsgBox "The password and check do not match", _
    vbCritical, "Password Error!"
    GoTo OpenPass
End If
On Error Resume Next
Documents.Close SaveChanges:=wdPromptToSaveChanges
bBack = Options.CreateBackup
Options.CreateBackup = False
strFileName = Dir$(strPath & "*.doc")
While strFileName <> ""
        WordBasic.DisableAutoMacros 1
        Set oDoc = Documents.Open(strPath & strFileName)
        With oDoc
        .Password = sPassword
        .SaveAs .FullName, AddtoRecentFiles:=False
        .Close SaveChanges:=wdDoNotSaveChanges
    End With
    strFileName = Dir$()
    WordBasic.DisableAutoMacros 0
Wend
Options.CreateBackup = bBack
End Sub

marshall barton say:

Notes:

1) Reports with anything in their Sorting and Grouping ignore the query's ORDER By clause.

2) You can not reset the query's SQL property immediately after opening the report. There is no guarantee that the report is done with the query so quickly.

Putting that information together, I strongly recommend that you use a separate query just for this subreport. You can then base this separate query on your original query simply by using:

CurrentDb.QueryDefs("new query").SQL = "SELECT * FROM [original query] WHERE " & strWhere

This way, you don't have to parse out the original query, worry about the ORDER BY clause, restore the original query to its previous SQL and it's still just the one line of code. Note that it doesn't make much difference how you create the new query because you will be replacing it with all new SQL

marshall barton say:

You said before that both the main and sub report have the same record source query, so I'm not sure what your question implies.

Be that as it may, the answer to 1) is Yes, the name of the query for the subreport.

For 2)  Use the main report name.  OpenReport does not know about what is inside a main report so there is no choice here.

For 3)  Yes, the line of code that sets the query def's SQL property goes right before the OpenRepprt line you have now.

All you're doing is inserting one line to modify the QueryDef's SQL property.  If the main report's record source is a different query than the subreport's, that's all you do.  If the main report uses the exact same query as the subreport, then remove the where condition argument from the OpenReport line.

Use the keyboard shortcut of Command+A to select all on the page. With everything on the page selected, the embedded containers should be obvious. Click the header of an embedded container and only it will now display as selected.

Next perform a manual click and drag selection of all content that is within the embedded container and press Command+X to cut it. This places all content onto the clipboard but not its container.
