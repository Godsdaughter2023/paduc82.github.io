henkvan der zee say:

I had the same problem and found a solution here:
https://social.technet.microsoft.com/Forums/windows/en-US/315841e1-b8b2-4084-8224-580ef3d13420/upgrade-to-windows-10-1709-iis-fails?forum=win10itprosetup#f9c2d9fd-22f1-424a-a807-f4d9a941a29a

The problem is that some files that WAS uses do not have SYSTEM as user anymore.

You have to open the properties of files with administrator elevation. That seems not to be possible with Windows Explorer. I am doing it from Total Commander run as administrator. There may be other ways.

The steps are:

  1.  Go to c:\ProgramData\Microsoft\Crypto\RSA\MachineKeys\
  2. You should see 3 files there:
    76944fb33636aeddb9590521c2e8815a_*
    6de9cb26d2b98c01ec4e9e8b34824aa2_*
  3. Do the following steps for each of these files:
  4. Go to properties > Security tap
  5. You will not see properties because you do not have read-access.
  6. Click the Advanced button
  7. Change ownership to yourself
  8. Then you can add SYSTEM to the list (ACL) and give it full access.

After these steps you should be able to start WAS again.
