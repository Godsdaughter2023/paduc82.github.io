jeeped say:

For x = 1 To Worksheets.Count

    Worksheets(x).Select


This cycles through all of the worksheets in your workbook.

    'Set the data range object:

    Range("a7").Select
    Set mydata = ActiveCell.CurrentRegion

This selects one or more cells that are 'connected' to A7. It is the equivalent of selecting A7 and tapping Ctrl+A. If there is nothing in A7 then it selects the entire worksheet. .CurrentRegion should only be used when you are absolutely sure there will be data in the first cell.

    Set mytotal = mydata.Offset(mydata.Rows.Count).Rows(2) 'WHAT'S THIS ONE DOING?


This looks at range of cells offset to A7 by the number of rows in A7's .CurrentRegion. No number of columns is provided so it is defaulted to offset 0 columns. From that offset range, the second row is assigned to the mytotal range variable.

Use,
    Debug.Print mydata.Offset(mydata.Rows.Count, 0).Rows(2).Address
... before assigning or,
    Debug.Print mytotal.address
.. after the assignment to see what was passed over. The Debug.Print output will go to the Immediate Windows (e.g. Ctrl+G).

    mytotal.Formula = "=sum(" & mydata.Columns(1).Address(False, False) & ")"


This could be a problem if the .CurrentRegion was more than a single column and shouldn't be left ambiguous like this. Use something like,

mytotal.Cells(1,1).Formula = "<your formula>"
... or earlier as,
Set mytotal = mydata.Offset(mydata.Rows.Count).Rows(2).Cells(1,1)

To make sure that mytotal is a single cell.

mike h.. say:

Hi,


The range mydata is set with this


Set mydata = ActiveCell.CurrentRegion


Now let's say that the current region for this was A1:I20 so in that range there are 20 rows. This line then set as new range called mytotal



 Set mytotal = mydata.Offset(mydata.Rows.Count).Rows(2)


The range will be the same width as the mydata range but offset from it by 20 rows from rows(2) of that range.


So 20 rows down from rows(2) is row 22 so this second range becomes A22:I22

Each of the rows in a defined range can be referred to with rows(n)


However doing that way isn't clear, I would write it like this


Set mytotal = mydata.Rows(2).Offset(mydata.Rows.Count)

jeeped say:
I looked a little deeper and realized that you wanted the SUM() formula across a number of cells, not just the first. However, while .Rows() can be used sometimes, it cannot be used at others and this will come back and bite you in the rear at some point. Better to define the range with .Rows().Cells instead of relying on VBA's fallback. Here is how I would write that.

Sub AddTotals()
    Dim x As Integer, mydata As Range, mytotal As Range
    For x = 1 To Worksheets.Count
        With Sheets(x)
            Set mydata = .Cells(7, 1).CurrentRegion
            Set mytotal = mydata.Offset(mydata.Rows.Count, 0).Rows(2).Cells
            mytotal.Formula = "=sum(" & mydata.Columns(1).Address(False, False) & ")"
            mytotal.Cells(1, 1) = "Total"
            mytotal.Range("B1:C1").ClearContents '(or mytotal.cells(1,2).resize(1,2).clearcontents)
        End With
        Set mytotal = Nothing
        Set mydata = Nothing
    Next x
End Sub
