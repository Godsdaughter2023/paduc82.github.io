How much RAM Can Microsoft Security Essentials use? - Microsoft Community

dreamer77dd say:

How much RAM Can Microsoft Security Essentials use?

I am just wondering.

Would MSE benefit from using more RAM and how much can it us? Lets say i had the top of the line CPU,
Would it make it faster if it used more RAM?



kosh vorlon say:

So lets say i have Windows 64 Ultimate with 192gig of RAM will MSE be able to use all that RAM to improve speed? How much of that would it use?

 

Hi dreamer,

 

The problem here is that in order to specifically answer your question for your system, we would need to determine where the bottleneck is now in terms of MSE scans.  In essence, if scans are currently taking 1 hour, what is the primary reason it takes that long?

 

If the reason is that the hard drive is running at maximum capacity, then you can add 100,000 TB of RAM and it won't matter because in that case the problem is that the hard drive is the limiting factor and so more RAM isn't going to make a difference because that's not going to make the hard drive move any quicker.  Hence the answer to your question would be that it would not use all the RAM to improve speed but I haven't a clue how to tell how much it would use and it would vary each time unless everything was exactly the same each time (and that's very unlikely to be the case).

 

In terms of how much more MSE would use to improve speed the answer is it would use as much as it could - but at some point something would become a bottleneck.  Perhaps you might be able to use even more RAM.  Perhaps it would be something else.  At some point, something will cause the scanning speed to max out in which case that's as fast as it will go.   If the cause isn't RAM, then adding more RAM will not make MSE go any faster because it is not limiting the speed.

 

As I said earlier, how much MSE uses is "as much as it can given how much is available and other factors that are limiting the scanning speed."  There's no specific number.  Chances are that if you really did have that much RAM, then the limiting factor would become the hard drive or the processor (probably the hard drive).  At that point, more RAM would not help.  As to how much it would use, that depends on what else is happening at the same time (and other things will be happening so 100% of RAM will never be available to the MSE scan), but it would use as much as it could.  If whatever was available after things other than the scan could be used, it would be used - so it would use as much of it as possible to improve speed.

 

If it helps, nothing in MSE limits how much RAM can be used to run scans.  It's just that something will eventually place a limit on the scanning speed and at that point, no additional RAM will be needed (assuming that something isn't RAM) since the scan already is moving at the top possible speed all other things considered.  There's no way to tell in advance what that number will be.  With that much RAM, I think the realistic answer is that MSE will not be able to use all of it to improve speed because something else will limit scanning speed before more RAM can be helpful (and I'd imagine it would be the hard drive but it could be other things as well).

 

I hope this clarifies things.

 

Good luck!

For the most part design objects are placed on the page in a static position. If you want the ability to 'bump down' then you will need to use Word.

"Object Position: Exact" is one of the options for wrapping text around a picture. If you drag a picture into a text box it will become active. But this has nothing to do with what you are trying to do.

Two options you might explore is to set the text boxes to Shrink Text to Fit or Best Fit. These options will allow the user to type in an 'excess' of characters in a fixed size text box without forcing the text box to expand and 'bump into' other design elements.
