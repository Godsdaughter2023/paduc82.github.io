bossdweeb say:

I believe you are wanting to increase the amount of VRAM on your AMD HD7000 series graphics card.

Can't be done.

I expect the 1792MB is the true amount of VRAM on a '2GB' graphics card. It's not unknown for mfg's to

short cut the advertised value. Nvidia did the same with their GTX970, labelling it as 4GB when it had only 3.75GB VRAM.

- You can only specify a set amount of system RAM set aside for use by integrated graphics. This does not apply

to dedicated graphics cards because they have their own VRAM.

Even if  you could 'dedicate' 6GB your gpu is not likely up to the task of processing that much data.

The game (via cpu/gpu), and Windows deals with the amount of data being fed to the system RAM while on it's way

to the VRAM/gpu on the graphics card. This is dynamic, so the numbers you see in Task Manager > Performance

in relation to graphics are not fixed.

- If you were using a laptop with integrated graphics the amount of system RAM allocated to graphics is often fixed,

even on machines that also have a dedicated gpu.Desktop machines generally have an option in BIOS where you can select Auto, or from several quantities starting from as

low as 64MB, to a maximum of 2GB, depending on the machine/motherboard. Some 'gaming' motherboards have a

fairly low limit. My current MB only allows 512MB for the Intel HD 4600 graphics.

a. user say:

Hello,

Thank you for bringing up your query on Microsoft Community.

We sincerely regret for the inconvenience caused.

Please be assured that we will do our best to help you.

This issue occurs because the Splwow64.exe process stays in the memory for additional time to increase system performance after a print job is finished. Therefore, the Splwow64.exe process does not end as expected when a print job is finished.

You can configure this time-out value by changing the registry value. To configure the time-out value, follow these steps:

  • Press Windows + R keys.

  • Type in regedit into the text box and press Enter.

  • Follow the path: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Print

  • On the Edit menu, point to New, and then tap or click DWORD Value.
  • Type SplWOW64TimeOutSeconds.
  • Press and hold or right-click SplWOW64TimeOutSeconds, and then tap or click Modify.
  • In the Value data box, enter a value, and then tap or click OK.
  • Exit Registry Editor.

Note: You must enter a value that is greater than zero in the Value data box. The value that you enter is the seconds that the system periodically waits while it checks whether the Splwow64.exe process is unloaded. This behavior occurs when that same number of seconds elapses after the last 32-bit print process has closed. Therefore, when a service process prints, the print capability is not released until the service closes, and at least the time that you specify in the Value data box has passed. For example, if you type 15 in the Value data box, and the last 32-bit process takes two minutes to print and then close, the print capability is released between two minutes 15 seconds and two minutes 30 seconds. If no time-out value is specified, the default time-out value is two minutes.
