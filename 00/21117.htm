Ok, so lets say i have office 2010 professional plus installed on my pc and i want to co-author a word or powerpoint document with my friend and i don't have sharepoint 2010 but i do have SharePoint Workspace 2010 as part of office 2010 can i do this- do i need to use the "Save to web"(aka to skydrive) function or something and will this scenario be enabled when the office web apps are available?

First take SharePoint Workspace out of the picture. :-) As previously noted, SharePoint Workspace enables you to create a workspace for a SharePoint 2010 site so you can work on files stored on SharePoint 2010 Server files offline.

Second, it's the application and the server the file is stored that are the keys to co-authoring. The servers that enable you to co-author are SkyDrive, SharePoint 2010, or a server running SharePoint Foundation 2010. You can co-author in the Word or PPT client but you can't co-author in Word Web App or PPT Web App on either SharePoint 2010 or Windows Live (your SkyDrive). Only Excel Web App and OneNote Web app support co-authoring.

Additionally, the only application that supports co-authoring from on either the web or client is OneNote. For example, you can be working on a notebook in the client and your friend can be working on the same notebook in OneNote Web App. This scenario isn't supported for any other application.

Essentially, all co-authoring does is enable two people to access the same file at the same time and see each other's updates. In order to co-author, all authors must open the same file stored in the same location and that location must support co-authoring. So you'd both need to open the file from the same folder on SkyDrive or open the file from the same document library on SharePoint. If you don't have access to a SharePoint site then you can use SkyDrive when co-authoring is eventually enabled.

Does this help make more sense of co-authoring?

bob jones aka: cybertaz say:

No offense, but the Compress Pictures dialog window you're referring to does not reflect the actual resolution of an image. Nor does Mac Word automatically compress images provided the document is saved in the .docx or .dotx file format*.

The File> Reduce File Size command simply is an available feature if the user chooses to invoke it, just like applying a shadow, cropping, etc. If not, images are stored at the size/resolution at which they are inserted. IOW, a 2MB 600 ppi TIFF is stored as a 2MB 600 ppi TIFF. The setting of 220 ppi is nothing more than the default option applied if one clicks OK... there is no means in the program for determining the resolution of an image.

*Please note that I have not been able to get definitive confirmation of this from anyone at Microsoft - but nobody has contradicted me, either :-) My interpretation is based on having dismantled .docx files & extracting the images contained within them.
