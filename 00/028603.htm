steve ikard say:

Word 2010 Mailmerge Labels - but ask operator to pick which label position to start with

I have created a Word 2010 Mailmerge template for Avery 5164 Shipping labels (2x3 per page).  It works fine when the operator has 6 or more data records.  We don't like wasting label stock.  So when the operator wants to print, say 2 labels, starting at position 3, how can I prompt him/her to indicate which position to start with?  Thanks for any advice.

doug robbins - mvp office apps & services (word) say:

If the data source is a table in a Word document, if you run a macro containing the following code when the mail merge main document is the active document, it will insert the necessary number of blank rows at the beginning of the data source so that the first row with data will be used for the label that is specified.

Macro to set the first label on a part sheet of labels for a label type mailmerge.

Dim MMMDoc As Document

Dim dsource As Document

Dim dtable As Table

Dim i As Long, j As Long

Set MMMDoc = ActiveDocument

With MMMDoc.MailMerge

    If .MainDocumentType = wdMailingLabels Then

        If .State = wdMainAndDataSource Then

            Set dsource = Documents.Open(.DataSource.Name)

            Set dtable = dsource.Tables(1)

            i = InputBox("Enter the number of labels that have already been used on the sheet.", "Set Starting Label")

            If IsNumeric(i) Then

                With dtable

                    For j = 1 To i

                        .Rows.Add BeforeRow:=.Rows(2)

                    Next j

                End With

            End If

            .Destination = wdSendToNewDocument

            .Execute

        End If

    End If

End With

dsource.Close wdDoNotSaveChanges

If the data source is in some other format, you can use it with a Directory type mail merge main document to produce a Word document that you can use as the data source.

macropod say:

One way of achieving your goal without the need for a macro  is to insert as many empty records as are needed in the data source to pad out the data.

Alternatively, if you use a Directory merge, as per the instructions below, then send the output to a document before printing, it is then a simple matter to insert however many empty cells are needed into the mailmerge output document (e.g. via the insertion of column breaks and/or by however many cells you need for the blanks then choosing Table Tools | Layout | Insert Above). Do note that this won’t be especially useful for using left-over labels from a conventional label merge, but it does facilitate the use of its own left-over labels.

First, a bit of background:

Word Label merges output the records using an across-then-down arrangement. The technique described below uses a Directory merge that generates a down-then-across arrangement

The required steps to generate a down-then-across arrangement label are:

1. In the Labels dialog, click New Document to create a new document based on the label definition you have chosen.
2. Display table gridlines (Table Tools | Layout | Table | View Gridlines) and nonprinting characters (Ctrl+* or Home | Paragraph | Show/Hide ¶) so you can see what you're doing.
3. Delete all cells except the first. If there is a spacer column, note its width beforehand.
4. On the Page Layout tab, in the Page Setup group, click Columns and choose More Columns...
5. In the Columns dialog, select however many labels across your label stationery had.
6. In the Spacing box, set the width to the original spacer column width or, if it had none, 0.
7. If necessary, set the column width to your label width.
8. Change the mailmerge type to 'Directory'.
9. Run the mailmerge.
