jimreisert say:

This was a pain in the butt to fix, but I finally got the TH2 update to install.  The problem was directly related to UEFI/GPT.  It seems Microsoft created the problem.  And left it up to me to fix it.

When a Windows 10 installation begins on a UEFI/GPT system, it creates a couple of partitions above and beyond the partition into which Windows is to be installed:

    100 MB EFI System partition

     16 MB MRS partition

Because I had a full backup, I started out by wiping out the original partitioning on my boot drive used the clean and "convert GPT" options in diskpart.  Then I installed TH2 (build 10586) on this newly cleaned drive.  I wanted to see what the partition setup looked like, compared to how it looked originally.  It was the same.  So that eliminated one possible error source.

Here's where the problem is.  Refer this tech note:

    Configure UEFI/GPT-Based Hard Drive Partitions 

This is what it says under "Drive Partition Rules":

Microsoft® Reserved Partition (MSR)

The size of an MSR is 128 MB.

Well, the Windows installer didn't think so, because it only created a 16 MB MSR!  It was the same size on my original boot disk, as well as as the newly-created Windows 10 build 10586 installation.

Assuming this to be the problem, I needed to enlarge the MSR from 16 MB to 128 MB, or add an additional 112 MB (I really wanted to use the word "embiggen").  Because the MSR and the installation partition were adjacent, I used EASEUS Partition Magic to downsize my installation partition slightly to free up 112 MB of space adjacent to the MSR.  But PM would not let me merge the existing MSR and the now unallocated 112 MB!

I had to use diskpart instead.  First, I examined the disk and partition layout.  I deleted the existing MSR and kept my fingers crossed.  In fact, I believe I rebooted at this point, just to be sure that deleting the MSR didn't have any unintended consequences.  I then tried to create a new MSR of 128 MB at offset 551 MB (where the smaller one originally lived), but diskpart said the size I was trying to allocate was too big (not 100% sure of the error message anymore).  So instead, I just told it to create a 128 MB MSR, regardless of starting offset, and lo and behold, it created the new MSR in the same place as the old one, just bigger.

I then rebooted, and restored my C:\ (Windows boot) partition from my last Acronis backup.  I rebooted again, and my Windows desktop looked like it did before this whole mess started.  Then, I tried the TH2 (build 10596) installation again. I was pleasantly surprised (and relieved!) when I finally saw the big progress circle on my monitor after the update installation rebooted for the first time.

So in summary, on a UEFI/GPT system, an initial installation of Windows creates a MSR (Microsoft Reserved Partition) that is too small!  This is something that ought to be fixed in Windows.

I hope I don't have to go through this again when the next update is released!  I also hope this helps someone else, because NONE of the fixes previously described apply to this situation.
