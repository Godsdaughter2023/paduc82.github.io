considerate_guy say:

First, a .wlmp is actually a *project* file, not a movie. It's really intended to be used on the original PC while editing but then you are supposed to create an actual video/audio file from it using File > Save movie. The resulting file is a .mp4 file that can be put on any PC.

A project file (.wlmp) on the other hand relies on dynamically loading photos, videos and music to *preview* the items before you actually save the movie. If you try to transfer a project file to another location, you may not be able to make it work unless you also move all the associated media files (photos/videos/music) and into the exact same folders names.

Okay, that aside, let's at least get you a copy of Movie Maker so that you can try opening the project (.wlmp) file to see what you are able to view. <removed>

Once you are able to load the project file into Movie Maker, you'll probably see a lot of yellow triangles with exclamation points indicating it can't find the source media files. If you know where they now are, you can double-click on a problem clip and navigate to its new location. Then you should be able to preview the project again.

If you get that all working, be sure to use File > Save movie and then you won't have to ever worry about keeping all the project and media files in sync. You'll create a single movie that you can view on any most any PC or mobile device with that single file.

Finding files with Search in Vista is different than it was in XP - you need to do a special process to avoid being limited by the index.  Here's the procedure:

To search the entire C: drive in Vista go to Start / Search Box and type in ‘<filename> or <foldername>’.  As you're typing you'll see two hyperlinks appear just above where you're typing and one will say Search Everywhere.  Click on that.  That will do a quick indexed search and bring up a dialog box.  Click on Advanced Search.  Click on the location drop down menu and find Drive C: and click on it. Check the box "include non-indexed, hidden, and system files (may be slow)."  Then click on Search.  You will now be searching your entire hard drive for the specified file.  And yes, it's not a very efficient way of searching the entire drive for a file but that's how it's done in Vista.  All you need is the names of the files and/or folders you're searching for (you can't escape that) - you could search for files by date and only look for those created in the last 24 hours (using essentially the same process but searching for *.* by date).  This will probably give you more than you expect, but what you're looking for should be included.

Remember to convert Mbits to MegaBytes.

Check this :

WiFi Speeds vs Real-World Throughput
http://www.wifihowto.net/wifi-speeds-vs-real-world-throughput

You must be using N as G would not work anywhere near as fast as you are seeing.

N = 300 Mbits  dual stream 150 Mbit single stream or 37.5/18.7 MB and real world 18.5/9.3 MB.

Interference, distance, and other environmental conditions can further reduce WiFi throughput.

So it seems like your WiFi is getting pretty good N dual stream thoughput.
