don cuthbert say:

It sounds like the partitions on your hard disk are not configured as Windows requires.  The "reserve" partition, probably means the MSR or Microsoft Reserve partition, which is required on a GPT disk in a UEFI BIOS mode device.

On a UEFI device, there should be an EFI (FAT32) partition, which is the System partition, an MSR (unformatted) partition, as well as the Boot/Windows (NTFS) partition and often a Recovery (NTFS) partition.

If yours is missing an MSR partition, you can create one, by shrinking one of the other partitions by 16MB:  Go to an Administrator Command prompt and run diskpart.

In diskpart, do the following:

Find which partition number is type System (e.g. Partition 2)

I suspect the MSR is just missing, but if you already have an MSR partition, it will show in diskpart, under the Type column of "display disk" as Reserved.  If it is there, normally it is around 128MB.  Select the Reserved partition number.  List partition to be sure that this partition is selected with an asterisk beside it.  Type "delete partition override".  Type "create partition MSR" to recreate it.

Now you should have a fresh MSR partition and the Windows upgrade will proceed via the ISO or Windows Update method.

A clean install of Windows 10 from the official ISO on a UEFI device sets up partitions as below:

C:\Windows\system32>diskpart

Your concatenation returns a text value while the table entry is a number

Change formula to =--CONCATENATE("2015","0","1")

The double negation  (--) converts text to number

Another way would be to use the cells with year and month to generate a number: year*100 + month*10+1 for example

BTW you can avoid CONCATENATE with =--("2015"&"0"&"1")

john sr wilson say:

The problem is that MSFT have set the bullet alignment default to AUTO instead of the probably more useful CENTER.

AFAIK this cannot be accessed except with code.

This would set the current master to centered bullets

Sub fixBulletAlignment()

   Dim oDes As Design

   Dim oMast As Master

   Dim L As Long

   Dim oCust As CustomLayout

   Dim oshp As Shape

   For Each oDes In ActivePresentation.Designs

      For Each oCust In oDes.SlideMaster.CustomLayouts

         For Each oshp In oCust.Shapes

            If oshp.Type = msoPlaceholder Then

               Select Case oshp.PlaceholderFormat.Type

               Case Is = ppPlaceholderBody, ppPlaceholderVerticalBody, ppPlaceholderObject

                  For L = 1 To oshp.TextFrame.TextRange.Paragraphs.Count

                     With oshp.TextFrame.TextRange.Paragraphs(L).ParagraphFormat

                     .BaseLineAlignment = ppBaselineAlignCenter

                     End With

                  Next L

               End Select

            End If

         Next oshp

      Next oCust

   Next oDes

   MsgBox "Masters and layouts fixed", vbInformation, "PowerPoint Alchemy"

End Sub

steven. s say:

Hey James,

 

TabProcGrowth=0 is a registry edit that will disable the Loosely-Coupled IE (LCIE) function in Internet Explorer 8. Essentially what this means is that all tabs in Internet Explorer will be handled by one process of iexplore.exe. This also means that Protected Mode will be Off and that if one of your tabs crash, Internet Explorer will crash.

 

As stated by PA Bear above, even if you didn’t have the TabProcGrowth=0 registry edit you do have both of those options. A more detailed guide to the requirements essential to using Protected Mode can be found here:

Protected Mode for IE7 in Windows Vista – Is it On or Off? (still applicable to Internet Explorer 8 and/or Windows 7)

 

More information on LCIE:

IE8 and Loosely-Coupled IE (LCIE)

 

More information on TabProcGrowth:

Opening a New Tab may launch a New Process with Internet Explorer 8.0

Till Excel 2007, validation across sheets is not permitted.  To workarounf this issue, you may wither work with named ranges or try the following:

 

1. Suppose the sheet name of the validation source list is Base data.  On this sheet, validation source list is in A5:A10

2. Suppose you are trying to validate cell G5 on the sheet named Master

3. While on cell G5 of Master sheet, go to Data > Validation > Allow > List

4. In the Source box, enter

 

='Base data'!A5:A10
