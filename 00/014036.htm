ken sheridan say: A distinct count can be returned by counting the rows returned by a subquery in which the DISTINCT predicate is used, e.g.

PARAMETERS Forms!frmParameter05!txtSvcMo DATETIME;
SELECT COUNT(*) AS AccountCount
FROM (SELECT DISTINCT Account
            FROM Accounts
            WHERE Svc_Mo=DATEADD("m",-1,Forms!frmParameter05!txtSvcMo));

So if you create such a query, and, rather than adding the values of the first two text boxes in the report, you call the DLookup function in a text box's ControlSource property, e.g.

=DLookup("AccountCount","NameOfTheQuery")

this should eliminate the duplication from the count.  This does assume that you are returning the counts only once in the report, in the report header or footer.  If multiple counts are being returned in a group header of footer, the subquery would need to be further restricted by means of a parameter which references the current group identifier.  You might also have to take account of any restriction on the report overall, either in its query's WHERE clause, or as the WhereCondition argument of the OpenReport method.

suzanne s. barnhill say:

You don't need a macro. You just need a building block. Here's how:

  1. Double-click in the header area to open the header.
  2. Assuming you don't want any other text in the header, press Ctrl+R to right-align the paragraph.
  3. Type your last name and a space, then press Alt+Shift+P to insert a PAGE field.
  4. Select the text you just typed and press Alt+F3.
  5. In the Create New Building Block dialog, type a name for your header (such as "myheader").
  6. Without changing any of the other settings, click OK.
  7. When you want to use this header, double-click in the header area and start typing "myheader."
  8. When you've typed "myhe," Word will pop up an AutoComplete suggestion; press Enter to accept it.

Alternatively, you could create a template that includes this header to begin with.
