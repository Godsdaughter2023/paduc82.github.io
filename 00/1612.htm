dianeoforegon say: Select to save sent messages to "On My Computer" Sent folder.

GMAIL'S RECOMMENDED IMAP CLIENT SETTINGS

Sending:

Do NOT save sent messages on the server. If your client is sending mail
through Gmail's SMTP2 server, your sent messages will be automatically
copied to the [Gmail]/Sent Mail folder.

DO save draft messages on the server. If you want your drafts in your mail
client to sync correctly with Gmail's web interface, set your client to save
drafts to the [Gmail]/Drafts folder.

Deleting:

Do NOT save deleted messages on the server. Messages that are deleted from
an IMAP folder (except for those in [Gmail]/Spam or [Gmail]/Trash) only have
that label removed and still exist in All Mail. Hence, your client doesn't
need to store an extra copy of a deleted message.

Do NOT save deleted messages to your [Gmail]/Trash folder because this will
delete a message in all folders.

Do NOT save deleted messages to your [Gmail]/All Mail folder as some clients
will try to empty this folder and ultimately fail. This can lead to delayed
mail access or excessive battery consumption on a mobile device.

Junk mail and spam:

Do NOT enable your client's junk mail filters. Gmail's spam filters also
work in your IMAP client, and we recommend turning off any additional
anti-spam or junk mail filters within your client. Your client's filter will
attempt to download and classify all of your existing messages, which may
slow down your client until the process is complete.

>>MS upgrade wiped out my outlook email accounts. MS restored outlook but not my address book.

Can you clarify exactly what you mean by the above. Specifically,

#1 - Are you saying that you're email accounts were restored but
#2 - Your AddressBook was not?

If the answer to the above is yes, then the next question is "What type (POP, Imap, Exchange etc) of email account(s) are involved?

For the purposes of this response, going to assume that you are using IMAP in which case, the next assumption is that your contact/calendar data was stored in folders labeled as "This Computer Only".

From your description, sounds like the .ost file for the Imap account was recreated in which case, the data in the "This Computer Only" folders would have been lost - hence the AddressBook being empty. The AB is a specialized view of the contact folders set to be a member of the AB.

If you have not backed up (exported) your contacts to a PST file, then unfortunately, that data has been permanently lost unless the old ost file still exists on your system, in which case, you would need to use a paid 3rd party OST2PST conversion program.

While the following info will not help with your current situation, it will help mitigate the problem going forward (keeping data in folders labeled as "This Computer Only" is not recommended since if you delete an IMAP account, the OST file associated with that account gets summarily deleted. Also, if Outlook cannot find the OST file, a new one will be created and email folders will be sync'd but the contact/calendar folders will be empty.

The way to determine what type of email accounts are in use is by going to Outlook menu --> File --> Account Settings --> Email tab

- POP3 accounts will use a PST file which can be <either> one PST file for each POP3 account or one PST file for all POP3 accounts
- Imap accounts will use one OST file per email account

To determine your data files (including the exact file folder path/name, switch to the Data file tab in Account Settngs

#1 - If you are using POP3 accounts, then the next question is, do your contacts exist in any contact folder contained in the PST file(s)?
#2 - More specifically, how contact folders exist in your Outlook profile? (either switch to the People view in the Outlook Navigation pane or press CTRL+6 to view the complete folder list in the Nav Pane
