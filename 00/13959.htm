scottgem say:

Karen, the previous response given uses a feature called parameter prompts. It allows you to put in a prompt within a query so the user can enter some criteria when the query is run.

However using parameter prompts (while it works) is not the best method. I almost never use them. There are three reasons against using them in your case. The first is that it means the report can only report on a single record, it can't be used with other criteria. The second is because users may not know a record number but can identify a record by other means. the third is there is no validation of the recordnumber entered.

The way I would do this is use a form to generate the report. Create an unbound form with a combobox on it that prompts the user to select a record. Either a list of valid record IDs. or choosing from some other list depending on what the report is about (for example a person or company name). Then have a button on the form to launch the report. The code behind that button is simple:

DoCmd.OpenReport "reportname", acViewPreview,,"keyfield = " & Me.comboboxname

If you give us more information about the report and its Recordsource we can be more specific.
