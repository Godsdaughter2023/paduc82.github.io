albert d. kallal say:

Do you know what line of code this is failing?

The fact of this working with a accDE, or mde suggests that some error is occurring, and the INSTANT a un-handled error occurs in a accDB/mdb, then ALL LOCAL and ALL global VBA variables are blowen out, and re-set.

If you use a accDE (or older mde), then any and all un-handled errors NEVER blow up VBA vars, and NEVER re-set them.

So I not convinced this is a reference issue, since if you had a broken reference, then you could not compile your accDB into a accDE, and it would spit out compile errors and not produce the accDE.

However, I always "strong" recommend using a accDE, since such applications are FAR MORE robust, and they NEVER re-set VBA vars, and they NEVER loose VBA values - even when you have unhandled errors. However, with a accDB, then ANY unhandled error will blow out and re-set all local and global vars. So a accDB is rather "flakey" compared to a accDE in this regards unless you have really aggressive error handling code. (that means in just about every routine - but is far less effort and work to just compile to a accDE).

So based on your post, the only reason why a accDE would work, and not the accDB, is some VBA value in a expression is void, incorrect or has been re-set. (and it even could be the actual expression the nz() is operating on.

I would post/check the exact line the VBA code is failing on - it is unlikely a reference issue, and you note that you are able to compile the accDB to a accDE (you are using the compile option in the VBA editor to ensure no syntax errors). You then use the file save-as, and choose the compiled accDE option.

So from VBA editor - ensure debug-compile works.

Then (and only then), you create (compile) the accDB to a access executable (a accDE).

derks say:

Ericsch, what you need to do is access the Notifications Queue, where you can see all pending events on your device and when they are set to trigger.  In your case you would want to find any Calendar.exe events and remove the ones you don't want.

 

I use SK Tools, which isn't free, but does allow you to delete the events you don't want, as well as a whole host of other tweaky stuff with the potential to cause more damage to your device   A quick search should uncover a few apps that can do this, some of which may have trial versions that you can use to do what you want.

 

Back to the original problem.  It isn't only all day events that have this issue.  I set up an annual recurring appointment and gave it a 3.5 day reminder, which worked, and then I got another reminder at the start time.  So it looks like it could be to do with having reminders set that are farther ahead the appointment than a certain period of time, or something to do with recurring appointments.  Rubbish!
