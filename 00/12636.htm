Hi Morten,

Firstly, I want to explain the Modify Date as below:
1.The Modify Date in our personal site and the Modify Date in the synced folder are not completely the same. A good example is that when we sync personal site at the first time, the local synced folder's Modify Date is when the folder created. It can be different from the cloud folder's modify date. All these new created folders will have the same local Modify Date.

2.They are both for the Modify Date of the files, but one is for the modify date when cloud files are changed and the other one is the date when the synced local files are changed. We can say not the modify date has been directly synced but the sync makes the files changed, so the Modify Dates which related to them changed. Especially for the files which are not in folders, the Modify Date for cloud and local always keep the same.

3.For the folder and the files in the folder. The situation is a little different. Because not all the changes after syncing can make the Modify Date changed. For example, if you change a file's name in the cloud folder, the name change will be synced to the local file, however, the local Modify Date for this file may be not changed during the sync.

This is a little confusing. In summary, for the folder and the files in the folder, the Modify Date can be different. The most important thing is that there should be no sync issue like files can't be synced or the content change in a file can't be synced.
