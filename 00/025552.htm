steven _fyiguy_ hughes say:

Have you checked to see if port 3389 is open/forwarded on your home router for the IP address of the computer you are trying to connect to?

This generally listed under port forwarding on your home Router (generally you can access this by typing in 198.192.1.1 in a browser and enter in the password you used to set it up) and make sure to set the start/end port to3389 for the IP address of your computer you wish to connect to.

This will allow you to connect locally via the IP on your home network if you enter in the local IP number in the computer name. You can find the IP of your computer at the Windows run command line typeCMD ->then run IPCONFIG. This should show your local IP address.

To connect via your phone off your network you will have  to use the non-local IP address as the computer name. This can be generally found by going to sites likehttp://www.whatismyip.com/ on your desktop computer to find your internet provider's IP address for your home.

If you don't want to use that port you can change the default port in the Windows Mobile RDP to something else say 3398 or any other by applying a simple registry hack.

  1. Open the device registry using a registry editor
  2. Open the key HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Default\
  3. Create an entry of type DWORD, called Server Port
  4. Set the value to the required port number (3398 as menitioned above)
