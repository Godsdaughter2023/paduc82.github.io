nathan o. - microsoft support say:

Hi Dan,

 

For performance reasons, the maximum number of values listed in the drop down filter list is controlled by a current database option.  If the number of values exceeds this option for a specific column, then you are no longer able to see all the individual values for filter options.  To see / change this option you can do the following:

 

+>Open your database.

+>Click on the Office button and then choose Access Options.

+>Click on the Current Database tab in the Options dialog.

+>Scroll to the bottom of the Current Database options and you will see a section called “Filter lookup option for <database name>”.  In this section there will be an option called “Don’t display lists where more than this number of records read”. 

+>Change this value to a higher number.

 

 

Changing this option to a higher number will allow you to see more values in your datasheet filter drop down lists.  But as mentioned if you make this number too large, you will start to see performance problems with your database.

rick rothstein, mvp - excel say:

You have been told why it happens in other messages in this thread, so controlling whether you can see all of the text or not cannot be managed completely. However, you might find these two companion macros handy (especially if you assign them to two buttons which you install on a toolbar somewhere, The first will AutoFit the column the ActiveCell is in so that any hidden text is visible... the second will return the column width to the original value. Note that to make this work, I am storing the existing ColumnWidth property value in a Comment attached to the first cell in the ActiveColumn (that way you could close the workbook with the column expanded and still return it to its default width when you next open that workbook); so, it is important to realize there cannot be any existing comments attached to the first cell in any column you expect to use the macros with. The names of each macro should tell you what they do...

Sub AutoFitActiveColumn()
  Static CurrentWidth As Double
  CurrentWidth = ActiveCell.ColumnWidth
  With ActiveCell.EntireColumn
    .AutoFit
    If ActiveCell.ColumnWidth > CurrentWidth Then
      .Cells(1).AddComment CStr(CurrentWidth)
    Else
      ActiveCell.ColumnWidth = CurrentWidth
    End If
  End With
End Sub

Sub ReturnActiveColumnToOriginalWidth()
  On Error Resume Next
  With ActiveCell.EntireColumn.Cells(1).Comment.Shape.OLEFormat.Object
    ActiveCell.ColumnWidth = .Text
    .Delete
  End With
  On Error GoTo 0
End Sub

a. user say:

Hello,

This might be happening because of empty cells in the column/row that you are trying to select using CTRL+SHIFT+Arrow.

CTRL+SHIFT+Arrow Key/s shortcut in MS Excel is used to select a whole set of data to the left, right, up or down. It will only select the cells which has data in it otherwise it will stop at an empty cell. However, if you tap it again, it will continue selecting over the blank cell.

For example:
Column A contains Dates and column B contains Names.

In column B, there's a blank cell, for a specific date. Let us say you want to select all of the names under column B, you may use CTRL+SHIFT+Down arrow. However, since there is a blank cell in B5, the selection will stop at B4. Should you wish to continue selecting the rest of the names, you may press CTRL+SHIFT+Down arrow again.

      A                    B
   1.  DATES           NAMES
   2.  5/1/2017      Alpha
   3.  5/2/2017      Bravo
   4.  5/3/2017      Charlie
   5.  5/4/2017      
   6.  5/5/2017      Echo
   7.  5/6/2017      Foxtrot

You may also use CTRL+A to Select All of the data set, pressing it twice at while at any cell will also select the entire sheet.
In addition, you may also use, CTRL+SHIFT+End while at a cell where you want to begin the selection, to select the data set (columns and rows) regardless of the blank cells in-between.
