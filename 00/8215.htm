Note the dynamic names

Xdata      =OFFSET(Sheet1!$A$2,0,0,COUNTA(Sheet1!$A:$A))

Ydate     =OFFSET(Sheet1!$B$2,0,0,COUNTA(Sheet1!$B:$B))

These are used in Select Data dialog (see screenshot)

The subroutine added to Sheet1's module (click Sheet1's tab and use View Data) is:

Private Sub Worksheet_Change(ByVal Target As Range)
     If Intersect(Target, Range("B2:B100")) Is Nothing Or Target.Cells.Count > 1 Then Exit Sub
     MsgBox "Formatting chart"
     Call ScheduleAdherence
End Sub
Sub ScheduleAdherence()
    With Charts(1).SeriesCollection(1)
      pntValues = .Values
      For chtPoint = 1 To UBound(pntValues)
       Select Case pntValues(chtPoint)
        Case Is >= 0.9
            With .Points(chtPoint).Border
              .ColorIndex = 4
              .Weight = xlThick
              .LineStyle = xlContinuous
            End With
        Case Is <= 0.9
             With .Points(chtPoint).Border
               .ColorIndex = 3
               .Weight = xlThick
               .LineStyle = xlContinuous
             End With
       End Select
      Next
End With
End Sub

Enter a date at end of column A and next to in the B cell enter a percentage value.

The chart will automatically update

=SUMPRODUCT(--(OFFSET(DownTime[Month],0,0,,1)=EOMONTH(B$2,0)),--(OFFSET(DownTime[Machine],0,0,,1)=B$3),--(OFFSET(DownTime[Category],0,0,,1)=$A4),(OFFSET(DownTime[Minutes Down],0,0,,1)))

=SUMPRODUCT(--(OFFSET(DownTime[Production Day],0,0,,1)=EOMONTH(B$2,0)),--(OFFSET(DownTime[Machine],0,0,,1)=B$3),--(OFFSET(DownTime[Category],0,0,,1)=$A4),(OFFSET(DownTime[Minutes Down],0,0,,1)))

bernie deitrick say:

No need to use OFFSET rather than just the table column, (though I'm not sure what you want to do with the dates).

=SUMPRODUCT((MONTH(Downtime[Date])=MONTH(B$2))*(Downtime[Machine]=B$3)*(Downtime[Category]=$A4)*Downtime[Minutes Down])

You could also use an array formula like this, entered using Ctrl-Shift-Enter:

=SUM(IF(MONTH(Downtime[Production Day])=MONTH(B$2), IF(Downtime[Machine]=B$3,IF(Downtime[Category]=$A4,Downtime[Minutes Down]))))

so I found 

"amd64_microsoft-windows-servicingstack_31bf3856ad364e35_6.1.7601.17592_none_672ce6c3de2cb17f"

 

I opened Regedit => HKLM\Software\Microsoft\Windows\CurrentVersion\Component Based Servicing\

I added a new key called "version", and under "version" I added a Expandable Value

Name = "6.1.7601.17592"

after reboot I started  "Windows Modules Installer" in "services.msc" and it worked, then I also tried the command "sfc /scannow" and it also worked.
