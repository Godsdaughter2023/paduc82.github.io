hansv mvp say:

The column headings in a crosstab query are dynamic: they depend on the values of the field specified as Column Heading in the design of the query. In your example, the quarters for which data are available change from year to year. This makes it impossible for the Query Wizard to know which fields to use.

To get around this, specify the values of the column heading field explicitly:

  • Open the crosstab query in design view.
  • Click in an empty part of the upper half of the query window.
  • Click in the Column Headings property in the Property Sheet.
  • Enter the column headings:

    Qtr 1;Qtr 2;Qtr 3;Qtr 4

  • Access will automatically place quotes around the values:

    "Qtr 1","Qtr 2","Qtr 3","Qtr 4"

  • Save and close the crosstab query.

You will now be able to use the Report Wizard. Also, opening a report based on the crosstab query in design view will be much faster, because Access doesn't have to determine the column headings any more.
