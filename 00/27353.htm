suzanne s. barnhill say:
Word automatically suppresses Spacing Before at the top of a page following a "soft" (automatic) page break. If you want to retain the spacing, you will need to apply "Page break before" formatting to the heading or (less desirable) insert a manual page break before it. There is really no way around this.
 
The Compatibility Option to "Suppress extra line spacing at top of page" applies to line spacing within a paragraph, not Spacing Before, and is irrelevant here.

bill manville say:

The message indicates that the attempt to remove subtotals from the selected range failed.

That suggests that there were no subtotals within the selected range.
Possibly because your previous run of the macro had already removed them.
Or possibly because the range (of cells) you had selected was not the range containing the subtotals.

If the first macro added the subtotals then you might want to run it again before running the second macro.

You probably used the macro recorder to generate the macro?

It is a useful tool, but in order for the macro to do the same job on a second run it generally has to be started from the same initial state (what sheet is active, and what range is selected). 

With a little extra care you can make it work from a wider range of initial states (e.g. by starting the macro recorder with a different sheet active, it will record the additional steps of you activating the sheet you need and selecting the range you want to work on.

If you want to work on the entirety of a table, regardless of how big it has become it is wise to start the macro by selecting the top left corner and then using Ctrl+* to select the region of occupied cells surrounding that initial cell.
