vincentchoy say:

Yammer if a fairly flat architecture.

Network -> Group -> Contents (Chat, Files, Notes..etc) -> Topics (spans across groups)

In the File Server days, we index items by Filename and Folders c:>Proposals\ProjectA\Pool\Quotationv1.xls You do this because the traditional File Operating System like DOS or Windows doesn't allow you tag index data to a file, so you index it by how you store it.

If this level of indexing is required I would suggest using Sharepoint. However in Sharepoint, it is better to tag metadata to the file so its easy to search, rather than use folders & filename, as a file placed into the wrong folder of a massive file structure will be most difficult to find later.

You also seem to be using One Drive as a departmental file share. One Drive is designed as a personal file share. Using it as a departmental file share with many people accessing may result in a lot of files and a very complex folder structure which may lead you to be using One Drive beyond what it was design and tested for. It would be good to understand OD4B limits and limitations to avoid issues as this library grows.

If you use Yammer, I would suggest you categorise discussion and contents by Groups & Topics.

For example, Safety Training Video -> Goes into the safety group, with topics #Safety #SafetyVideo #TrainingVideo #NewEmployee #Mustwatchvideo
