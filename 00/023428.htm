a. user say:

Hi,

 

I appreciate your detailed description on the query.
 
I would like to enlighten you with a few facts about running Internet Explorer 11 on a Windows 7 32-bit system.
 
Enhanced Protected Mode (EPM) adds additional security to Protected Mode and includes AppContainer and 64-bit tabs. With AppContainer is turned on, IE11 checks the add-on registration for an AppContainer compatible flag. If it's present, the BHO or toolbar loads and runs. If it isn't present, the BHO or toolbar is blocked, and the user is notified of the EPM incompatibility.
 
Now, tabs running in Enhanced Protected Mode on Windows 8 run inside an AppContainer. On Windows 7 and Windows Server 2008 R2, AppContainer does not exist, so EPM only enables 64bit tabs on a 64bit OS. (That also means that enabling EPM on a 32bit Windows 7 system doesn’t do anything, because a 32bit Windows 7 system supports neither 64bit nor AppContainer).
 
[Refer to the blog: http://blogs.msdn.com/b/ieinternals/archive/2012/03/23/understanding-ie10-enhanced-protected-mode-network-security-addons-cookies-metro-desktop.aspx]
 
On the other hand, Enhanced Protected Mode (EPM) and AppContainer turned on by default in Internet Explorer 11.

nuculerman say:

Try this:

Sub Upload()

Dim wkb As Workbook, sht As Worksheet
Dim m As Range, myrange As Range, d As Range

Set sht = ActiveWorkbook.Worksheets(1) 'change worksheet number if needed
Set wkb = Workbooks.Open(Filename:="<directory goes here>")

mylastcell = sht.Cells(Cells.Rows.Count, "A").End(xlUp).Row
Set d = wkb.Worksheets(1).Range("A2")

Set myrange = sht.Range("A2").Resize(mylastcell - 1, 1)
For Each m In myrange
    rownum = 0
    Do Until m.Offset(0, 13).Value <= d.Offset(rownum + 1, 13).Value And m.Offset(0, 13).Value > d.Offset(rownum, 13)
        rownum = rownum + 1
    Loop
    d.Offset(rownum + 1, 0).EntireRow.Insert Shift:=xlDown
    m.EntireRow.Copy Destination:=d.Offset(rownum + 1, 0)
Next

wkb.Close SaveChanges:=True

End Sub

Change the "A2"'s if needed.  I assumed your data started on row 2.  If you change A2, for every row you add, subtract one from mylastcell.

 

EDIT:  OF course, there's really not a whole lot of point to that.  You can just copy and paste entries at the bottom of the master file and Sort column N by date after the fact... Which begs the question why you aren't just editing the master file directly.  All you'd have to do is scroll to the bottom of it and enter any new data, put a filter in row one, and sort column N by date.  If you ever need data just from a specific month... that's what AutoFilter is for.

    graham mayor say:
    If the content of the cells is bookmarked with the names R2C3 and R2C4 respectively then

    { IF { R2C3 } = "No" "0" "{IF { { R2C4 } = "High" "100" "{ If { R2C4 } = "Medium" "50" 10" }" }" } }

    would work, but the problem is the ease with which bookmarks can be overwritten when changing the values in the cells.

    A macro may prove a better bet. Insert a docvariable field "varNum" where you want the value to appear. i.e. { DocVariable varNum }

    Run the following macro (which assumes the first table in the document contains the values) to put the value in the docvariable. If you don't know what to do with the code - see http://www.gmayor.com/installing_macro.htm  

    Sub GetValue()
    Dim oTable As Table
    Dim oCell1 As Range, oCell2 As Range
    Dim iNum As Integer
    Dim oFld As Field
        Set oTable = ActiveDocument.Tables(1)
        Set oCell1 = oTable.Cell(2, 3).Range
        oCell1.End = oCell1.End - 1
        Set oCell2 = oTable.Cell(2, 4).Range
        oCell2.End = oCell2.End - 1
        If LCase(oCell1.Text) = "no" Then
            iNum = 0
        Else
            Select Case LCase(oCell2.Text)
                Case "high": iNum = 100
                Case "medium": iNum = 50
                Case "low": iNum = 10
            End Select
        End If
        ActiveDocument.Variables("varNum").Value = iNum
        For Each oFld In ActiveDocument.Fields
            If InStr(1, oFld.Code, "varNum") > 0 Then
                oFld.Update
                Exit For
            End If
        Next oFld
    End Sub

    sharepointisneedlesslycomplex say:

    Ah ok figured this out. Its a bit of a weird one for me and has to do with the "integrity level" of the file itself.

    My fist clue was that this did not happen to network files. So whats the difference? on the PST file that i was working with, under the advanced security tab, I noticed a field i had never seen on a file before:  Integrity level. This was set to "high mandatory level".

    So off to DDG i go to search it out, however except for some slight mentions in programming forums, there is really no admin level documentation of this. Just documentation for programmers who are developing applications. Then i found this document which is a good read and explains a bit about the levels, what they are for and why they are there. All part of the UAC system:

    http://www.tmurgent.com/TmBlog/?p=2386

    So I would encourage you to read it, but a lot is very programmer specific. The key point i got was that i can use icalcs to set this!  So using that post, i determined to test this hypothesis that this was causing the problem by resetting the ACL with icalcs. Luckily in the icalcs documentation this is easily done with the following command:

    icacls *.pst /setintegritylevel m

    M is for medium, which proved enough de-elevation for me to now successfully add the pst file to the users outlook!

    HOW the PST file, which did not have this property on the network, gained it when being copied to the machine is a bit of a mystery to me. I am just happy that i can now manipulate that attribute and get the user to be able to access their old mail. This manipulation does survive after a reboot, so i think its a good work around for the problem and really a solution at this point.

    Just don't know the WHY and the HOW this file became modified in this fashion. Perhaps someone else can explain that one, as i have never seen that property on a file before. And cant really find any other files with it either.
