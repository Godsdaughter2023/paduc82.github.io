garym123 say:

The standard is the power output of USB-C is 20 volts at 5 amps = 100 watts.  

This does not mean the device will charge at that rate. There is no standard for charging.  The device controls the charging rate.  A type C charger has to be 20 volts but it does not have to be 5 amps.

Current USB chargers can put out 5 volts at 2 amps. The charger that comes with current phones are based on what the phone can support. That is why there are 5 volt 1.2 amp chargers

So, connecting to a higher amperage charger will not necessarily increase the charging rate but it won't hurt the device.

jeeped say:
You can take a hint from the following line,

Range("B1:B" & lr).Formula = "=A1*2"

This dynamically creates a formula in column B from the first row to the row specified by the variable lr , adjusting the row number to suit. This tells us that we can use the same behavior to create a dynamic formula across columns B through E with,

Range("B" & lr + 1 & ":E" & lr + 1).Formula = "=SUM(B1:B" & lr & ")"
   ... or,
Range("B" & lr + 1).Resize(1, 4).Formula = "=SUM(B1:B" & lr & ")"
   ... or in R1C1 terms,
Range("B" & lr + 1).Resize(1, 4).FormulaR1C1 = "=SUM(R1C:R" & lr & "C)"

Some of your formatting code ignores the fact that is is within a With ... End With, and non-recorded code should avoid the use of .Select to get its point across. Here is my version of the cleaned up procedure.

Sub Test()
    Dim lr As Long
    lr = Range("A" & Cells.Rows.Count).End(xlUp).Row
    Range("B1:B" & lr).Formula = "=A1*2"
    Range("A" & lr + 1).Value = " Total :"
    Range("B" & lr + 1).Resize(1, 4).Formula = "=SUM(B1:B" & lr & ")"
    With Range("A" & lr + 1).Resize(1, 5)
        .Font.Color = 65535
        .Font.Bold = True
        .Interior.Color = -16677886
        .Borders.Weight = xlThin
        .Borders(xlInsideVertical).LineStyle = xlNone
        .Borders(xlInsideHorizontal).LineStyle = xlNone
    End With
End Sub

In an effort to put my money where my mouth (or profile page idiom) is, there is no reason to keep using With ... WEnd with unless you are saving more code lines than you create unless you are getting paid for each line of code you write. If you were assigning multiple border attributes, the With ... End With might be valid, but I don't see that here.

Edit¹: you can use .Borders since you are assigning the same attributes to all edges.
Edit²: in case you do not want the the interior vertical border, I've added the last two lines. Comment then or delete them if you want to keep the vertical interior border of the A:E group.

rob koch say:

Cyber stated all of this already, but I'm going to try and simplify this by explaining the primary purpose for each scanner to exist.

First, the Malicious Software Removal Tool (MSRT) was created many years ago to be automatically distributed once a month to all supported Windows systems via Windows Update, scanning only for the most common malware currently known to be attacking these devices.  This smaller package downloads faster and detects and removes most malware, but since it's only usually updated once a month, it's not really valuable unless your system has a well known common infection, often due to the fact that the existing AV product either isn't working properly or is missing entirely.

The Safety Scanner was created later and contains the same detections that are currently being delivered to the full Microsoft AV based security products, but of course only performs scans on demand.  This is the program that should be used by anyone trying to verify whether there's any significant malware infection, though it is still only aware of existing malware and doesn't attempt to detect via behavioral or other advanced techniques.

The complete AV security products like Windows Defender or MSE for Windows 7 obviously include these real-time, behavioral and cloud protections in addition to signature based AV detection.

So MSRT isn't really useful unless it's for some reason not being properly "installed" monthly by Windows Update.  Safety Scanner is the best tool for on demand scans for older systems, since it's easy to download and use and covers most of the current malware spectrum.

The Windows Defender offline is more capable than Safety Scanner and can operate outside the normal operating system in order to detect items using rootkit or other techniques to hide from built-in AV.  However, it's also typically more difficult to use, though in theory the built-in version with Windows 10 should be easy.  Unfortunately it's still possible that some malware might manage to damage the operation of Defender Offline or it won't run for some other reason, so it's still valuable to have other manual tools like Safety Scanner or even the manual download version of Windows Defender Offline available in case these are needed for a second opinion.

Guess I didn't end up making it simpler, but I do hope I've made the purpose for each more clear.

rob koch say:

I saw that in your initial post, but forgot about it when writing my response.

Actually there are a number of possible reasons that these numbers might be different and without lots of experimentation or research I can't be certain of which apply in each case, but I'll list them so you can understand why they probably don't matter.

The large difference for the full scans is most likely due to the setting to scan archive files, which I'm not even certain still exits in Windows 10 Defender.  Since the scan engines count the number of individual files opened, this number will inflate dramatically if the individual files contained within all zip, cab or other compressed files including iso packages are unpacked and scanned as well.  The requirement to unpack these files during a scan is less important now that the real-time protection included in Windows 10 is so fast that it can scan them as they are opened instead.

Another common reason for different scan counts within Windows Defender itself is that both it and Microsoft Security Essentials maintain a file scan cache in order to keep track of which files have already been scanned since the last system reboot.  This is done to reduce the amount of repetitive scanning performed each time a new scan runs, since re-scanning files that haven't changed according to both their last modified file date and the information contained in the cache is obviously pointless.  Of course, this should result in a smaller number of files being scanned after the first, at least until another system reboot requires that the cache be reset as well, since nothing can be guaranteed if the operating system and so Defender itself has stopped running.

The last difference that comes to mind is the function of the individual scanners themselves, since as you noted their purposes and function are different.  The most important thing to remember is that any malware that's well known and operating will always be detected by a quick scan, since that's what this type of scan is defined to do.  Full scans are designed to scan the entire set of disk and other mass storage devices for potential malware, whether it's operating or just sitting there in a zip or individual file waiting to be opened and launched.

The number of items that each type of quick scan needs to look at varies, even between individual sessions close together, both because of the cache operation I already mentioned as well as things like what programs have been run recently and what temporary or other files may be open, created or deleted within the most recent session.

The other aspect of this is that since the purpose for an offline scan is to check for items that might be hidden, it focuses on a more limited set of startup and other areas than the more general Windows Defender scan that operates when Windows is fully running, which also inherently opens even more files and functions.  For this reason, it's not surprising that this number is slightly smaller, since you'd expect additional items to exist or be opened once all of the Windows drivers, startups and other application functions are running.

The point here is that the number of items scanned in truth means basically nothing, since it only has meaning in the context of the version of Windows, specific hardware and drivers and other applications included in startup on that specific machine, with that specific configuration and at that one single moment in time.  This is one of the most confusing things to consumers, since they believe there are a fixed set of rules and the reality is that a computer is a complex system of many overlapping rules and status, that at any give moment is performing millions of individual operations.

So in truth, the number of files scanned is simply a relative indication of the amount of work the scanner has performed and nothing more.  It's value is only to the user viewing it in order to determine whether the scan is nearing completion based in the results of this count in previous sessions.  In fact, Microsoft only added the display of these numbers after lots of complaints from "techie" type users who were used to having these displayed in other 3rd-party scanners in the past.  Microsoft as with all of us having deep backgrounds in security knows that this number has no intrinsic value to PC security and in fact most often just ends up in extended conversations like this one.

Note that I'm not saying the numbers are pointless, but since very few ever pay attention to them and fewer still would recognize when they've changed or why, they serve little true purpose other than to confuse in most cases.
