a. user say:

Setting Up Windows Explorer Folders, Views, Options and Settings Tutorial

Customize your folders for the full potential of your PC experience.

ONLY HAVE THIS ONE FOLDER OPEN YOU ARE TRYING TO SET.

Select the bottom arrow and select View Tab

Now you will have this view with only a couple columns of info.

this is what you do to add and save the columns for similar type files in folders. (Shown in Photos)

Select sort by, and then select choose columns. Now here you want to select the columns as you want them displayed from left to right in the window. You may have to open and close choose columns several times to set it how you want it because they are listed in alphabetical order in the selections box.

Now after you have selected all that you want now select this, size all columns to fit.

Now you need to set it to stay this way. To do this select FILE TAB>>Change folder and search options. Go to View Tab

Now this is the important part. If the Apply to folders button is enabled to use, GREAT. (Mine shows disabled because I already made the change) Just select Apply on bottom, then Apply to Folders on top and OK.

If the Apply to folders is disabled like mine shows, then you have to get it enabled. I usually select to unhide (Uncheck) protected operating system files and then Apply and confirm YES. Then check it back to hide and select Apply and then the Apply to folders is usually enabled. Now select to Apply to folders, Apply again if enabled and OK.

Close out the folder.

Now log off desktop and log back on.

If you do not select to Apply to folders, then your selections may not stay as you set them or may not apply the settings to all folders with similar files.

Also, the bottom right of windows explorer has default thumbnail icon size you have chosen and details views to switch between that has these columns you created. And these setting will hold true for any other drives you have connected when you made these changes.

Also, once you do this, if you like the Icon thumbnail view by default when opening windows over the details view by default, all you have to do is on bottom right is switch to icon view, open file explorer options and do the same procedure in View Tab to APPLY, APPLY TO FOLDERS, OK. Now your will get Icon view by default and you have already set your details view for columns to switch to anytime and both settings are stored. There are a few Windows and users folders that you may have to manually set default view as they may even be small icons view when you have large icons by default view and of course your details view you setup. An example of this is Music Folder\Playlists, usually every time you open music folder, all the icons will be small. But most all folders will hold true your default view for open Windows.

tryx3 say:

1  It's up to you but a phone call to the activation center seems easier than going through the rigmarole of reinstalling Windows 8.1 and then re-upgrading to Windows 10.

2  I'm astounded that changing the HDD should necessitate reactivation [this was not the case with Windows 7].  Arul's post is corroborated by the article he linked to so it's not simply a typo.

3  You can prepare for the change by

3.1  Checking that your current Windows 10 installation is activated.

3.2  Creating a Recovery drive with the Include system files option [see Control panel, View by - Small icons if necessary, Recovery, Create Recovery drive] because that is a reinstallation tool [not an installation tool, which is why you need to check that your current activation status is correct].  Then boot from it to test it works so you don't find out the hard way after you've started swapping disks about.

3.3  Please note that a Recovery drive will, apparently, overwrite any existing partitions on a hard drive so if you intend to partition it you'll have to leave that until afterwards.

steve cochran say:
OE tends to self-destruct, which means it will often fill the inbox with zeroes and then you have no more messages.  That appears to have happened in your case.  Check your Recycle bin for an inbox.bak file that is a backup of the dbx file if present.  See www.oehelp.com/OETips.aspx#2 and also #4 on that page.  If there is no bak file then search for *.dbx (hidden) and see if you find a variation on the inbox.dbx file (inbox (1) or somesuch) and see if you can import that manually into OE (seewww.oehelp.com/backup.aspx#imp1).  You can determine which file OE is using for the inbox by right clicking on the inbox folder in the OE folder list and going to Properties.
 
If you still can't get the messages from all that, then your only recourse is the extract from disk feature of my DBXpress program (www.oehelp.com/DBXpress/) unless you have a backup.
