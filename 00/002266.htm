graham mayor say:

The date picker uses a 32 bit object library - MSCOMCT2.OCX to provide the DatePicker function. Essentially this is no longer used in Windows 64 bit operating systems, and does not work at all with 64 bit versions of Office.

While 32 bit Office users can probably get the OCX file to work, by moving it to the required folder and registering it, that's not going to be of much help to others who you may wish to use your code. You should therefore consider a date picker created completely in VBA.

There are several code samples around, but I especially like Trevor Eyre's version which you can find at http://bit.ly/1ossyMA

cenedd say:

Just to confirm this fixed the issue for me.  Printer is an HP LaserJet 3330 using Windows 7's own driver.  Disabling Mopier mode (which had been enabled by default) fixed the problem and allowed multiple copies to be printed.   Steps to fix:

1. Start > Devices and Printers
2. Double-click printer
3. Double-click "Customize your printer"
4. Click "Device Settings" tab
5. Scroll to bottom of page and change "Mopier Mode" to Disabled under the "Installable Options" group.
