Suppose you have a series of lines & fields, thus:
Name¶
No of Adults: «Adults»¶
No of Boys Age 6: «M6»¶
No of Boys Age 7: «M7»¶
No of Boys Age 8: «M8»¶
No of Boys Age 9: «M9»¶
No of Boys Age 10: «M10»¶
No of Girls Age 10: «F6»¶
No of Girls Age 10: «F7»¶
No of Girls Age 10: «F8»¶
No of Girls Age 10: «F9»¶
No of Girls Age 10: «F10»¶

Given that you're using a PC, you could use the mergefield \b switch to suppress the data & lines when the corresponding mergefield is empty. For example to suppress:
¶
No of Adults: «Adults»
when «Adults» is empty, you could select the «Adults» field and press Shift-F9 to expose the field code, which would look like:

After 'Adults, type '\b ""' so you get:
{ MERGEFIELD Adults \b ""}
then drag all of:
'¶
No of Adults: ' into the field between the double-quotes so you end up with:
{ MERGEFIELD Adults \b "¶
No of Adults: "}
repeat the process for the other fields.

If, however, the records with no data return 0 instead of a blank, you'll need to use an IF test. To do this:
• input two of the «Adults» fields
• select both «Adults» fields and press Ctrl-F9 to enclose them in another field, thus:
{«Adults» «Adults»}
• Edit the field code so you get:
{IF«Adults»<> 0 "«Adults»"}
• drag all of:
'¶
No of Adults: ' into the field after the first double-quotes so you end up with:
{IF«Adults»<> 0 "¶
No of Adults: «Adults»"}
repeat the process for the other fields.

Note: The field brace pairs (i.e. '{ }') for the above examples are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. Likewise, you can't type or copy & paste the chevrons (i.e. '« »') - they're part of the actual mergefields, which you can insert from the 'Insert Merge Field' dropdown. The spaces represented in the field construction are all required. Instead of the ¶ symbols, you should use real tabs and line/paragraph breaks, respectively.

For more, see the Mailmerge Tips and Tricks thread at:
http://www.msofficeforums.com/mail-merge/21803-mailmerge-tips-tricks.html
or:

http://windowssecrets.com/forums/showthread.php/163017-Word-Mailmerge-Tips-amp-Tricks

doug robbins - mvp office apps & services (word) say:

If you want to add a MERGEFIELD as code, you can either do it via Insert>Quick Parts>Fields, or by pressing CTRL+F9 to insert a pair of field delimiters { } inside of which you type MERGEFIELD and the field name.

However, it would be easier to use the Insert Merge Field facility in the Write & Insert Fields section of the Mailings tab of the ribbon and then press ALT+F9 to toggle on the display of the field codes in the document.

That would change a «Name» field to { MERGEFIELD Name }

Without knowing the exact names of the fields in your data source, it is not possible to give you precise instructions for how to control what appears in the document, but assuming that the fields are

Boy6

Girl6

Boy7

Girl7

etc.

You would use  a field construction like

{ IF { MERGEFIELD Boy6 } > 0 "No of Boys Age 6: { MERGEFIELD Boy6 }¶

" }{ IF { MERGEFIELD Girl6 } > 0 "No of Girls Age 6: { MERGEFIELD Girl6 }¶

" }{ IF { MERGEFIELD Boy7 } > 0 "No of Boys Age 7: { MERGEFIELD Boy7 }¶

" }{ IF { MERGEFIELD Girl7 } > 0 "No of Girls Age 7: { MERGEFIELD Girl7 }¶

" }{ IF { MERGEFIELD Boy8 } > 0 "No of Boys Age 8: { MERGEFIELD Boy8 }¶

" }{ IF { MERGEFIELD Girl8 } > 0 "No of Girls Age 8: { MERGEFIELD Girl8 }¶

" }{ IF { MERGEFIELD Boy9 } > 0 "No of Boys Age 9: { MERGEFIELD Boy9 }¶

" }{ IF { MERGEFIELD Girl9 } > 0 "No of Girls Age 9: { MERGEFIELD Girl9 }¶

" }

etc.

Where the ¶ appears in the above, you need to press the ENTER key or use SHIFT+ENTER and you must use CTRL+F9 to insert each pair of field delimiters { }. The field construction will NOT work if they are added using the { and } keys on the keyboard.
