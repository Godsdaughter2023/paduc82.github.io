word 2011 for mac switching between documents takes a long time - Microsoft Community

jorge olenewa say:

word 2011 for mac switching between documents takes a long time

I can't figure out why switching between documents in Word 2010 for Windows is quick whereas the same in Word 2011 for Mac takes a long time. I am working on a dual-core Intel Mac, 2.0 GHz with 8GB of RAM. My Windows PC is 32-bits and only has 4GB of RAM. On the Mac there are no Page outs, so the document should be in RAM at all times. The same thing happens on my MacBook Pro with 2.4 GHz dual-core CPUs and 8 GB of RAM.
john mcghie say:
First, go to Word>About Word and make sure you are up at Version 14.1.4.  If you're not, that may be your problem right there.  Run Help>Check for Updates.

OK: Next, a cold dose of reality...  Word 2010 is quite considerably quicker than Word Mac 2011.  Microsoft is not happy with their current performance, and they are working to improve it.  But for now, PC Word is a lot faster, and if you are going to tackle big documents, you may need to do so on the PC.

However, I find the performance on my Macbook Pro quite acceptable for documents less than a thousand pages.  For really heavy stuff, I also have a Mac Pro.  You may want to consider getting one if you make a living doing this -- time is money :-)  If you are using a Mac as a tool of trade, there are some "adjustments" you can make to speed things up.

I guess the first thing is that a 2.0 GHz dual-core is a little light on for serious work.  If you can, invest in a Core i7 with a solid-state disk.  The speed improvement will be spectacular.  8 GB of RAM is plenty, of course.  However, Word rarely holds a whole document in RAM: generally it stores only the chunk you are currently working on.  You can't change that.  An SSD will help a lot.

Next thing to look at is tracked changes.  To use a simplistic analogy, for each tracked change in a document, Word has to read the whole file from top to bottom each time you make a change: it will get VERY slow if the document is full of tracked changes.  Do not use tracked changes.  Instead, keep a copy of the document you started with, and use Compare Documents to mark the changes at the end.  Accept All Changes in Document in the current document, then save and close it.  When you re-open it, the speed improvement will be dramatic.

Next, make sure you have cleaned up your fonts:  http://word.mvps.org/Mac/fontweeding.html

Next, pay attention to file size.  If you have a normal hard disk, any document bigger than 32 MB will handle very slowly.  Instead of embedding your pictures, try "linking" them.  Insert>Photo>Picture From File...  Check "Link to File" then UNcheck "Save With document".  Again, this makes a dramatic speed difference in documents that have lots of pictures.  But learn to store all the pictures in the same folder as the document (not a sub-folder of it) and move the entire folder as a single unit if you want to move the document.  Otherwise, you will break all your picture links.

Learn to use Draft View.  Draft View is a special view of the document built specifically for easier editing in large, complex documents like books.  It shows you more of the control characters, so you have a better view of what is happening.  And it uses about a thousand times less CPU power than Print Layout View.

And lastly, pay attention to what "else" is running while you are working.  Every running application is sucking a little CPU, even if it is doing "nothing".  It keeps up a constant chatter interrupting the CPU to see if there is anything it should be doing!  Quit applications you are not currently using: if they're only minimised, they're still sucking power.  Music applications, web browsers, Instant messaging, Twitter, Facebook, Spaces, Download Torrents, Live TV: these are notorious power hogs, sitting in the background sucking power and RAM out of the box.

I hope your answer lies in one or more of the above :-)

Cheers

Supress text wrapping in chart titles? - Microsoft Community

pollard911 say:

Supress text wrapping in chart titles?

Is there a way?

teylyn say:
Hello,

The obvious way would be to resize the chart title box, but unfortunately it cannot be resized. Excel determines the width of the chart title in relation to the chart area. Although it shows sizing handles when selected, these cannot be dragged. As a workaround, increase the width of the chart area until the chart title fits. Then select the plot area and reduce its width to the desired representation. 

cheers, teylyn
