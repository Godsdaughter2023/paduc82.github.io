a. user say:

Hi,

 

Are you using any third party firewall program?

 

As the error message states, this issue could be caused if the port through which the computer communicates with the printer is blocked by the firewall. If you are using Windows firewall, refer to these steps to unblock the UDP port 427:

 

a)      Press Windows key +F to bring up the search Charms bar.

b)      Type Windows Firewall and press enter; click on the Settings tab below the search box.

c)      Click on the Windows firewall tab and click on Allow an app or feature through firewall.

h)      Click on New rule on the right pane, select the port radio button in the wizard and click Next.

i)       In the protocols and ports window, select UDP and type 427 in the specific local ports box.

j)       In the Action window, click on Allow Connection and click next.

k)      In the Profile Window, select the appropriate network and follow the prompts.

l)       Repeat the same for outbound rules.

For many incidents the setup is something like this:

An image backup is stored on some external device.

A new target HDD/SDD is installed in the system.

A bootable Windows USB thumb drive is used to start Windows to get into the Recovery Process and get to here.

Everything goes fine until the point of clicking the 'Finish' button and the error message pops up.

The theory is that the Windows recovery is so stupid that it wants to restore the image onto (in this case) the USB thumb drive used to boot Windows - which is unlikely to work.

So before clicking the 'Finish' button, physically remove/yank/pull the USB thumb drive from the system and then click the 'Finish' button.

That is a common solution to many similar incidents to maybe that will work for Windows 10 too.

Since the Windows recovery stuff is already loaded there is no need to have the USB thumb drive anyway and removing it will make that drive letter (D, E, F, etc.) unavailable so then Windows recovery wont see it anymore.
