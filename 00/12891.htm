dirk goldgar say:
I've done a little further testing, and I find I have to amend my earlier
post.  It turns out that the LOOP version of the function will not report an
AccessField as a member of the Controls collection, so it will only find
controls that have actually been added to the form by the designer.  On the
other hand, the INDEX version as originally written will give a type mismtch
error if the name given is an AccessField (in the form's recordsource but
not a control), because the function wants to return an Access.Control type.
 
Here's a version of the fncFindControl_INDEX function that doesn't have that
problem, and that only returns true controls, not AccessField objects:
 
'------ start of code ------
Function fncFindControl_INDEXControlName As String, frm As Access.Form) As
Access.Control
    ' Return a reference to the control with name <ControlName>
   ' on form <frm>, if it exists, or return Nothing if it doesn't.
    On Error GoTo Err_Handler
    Dim objctl As Object
    Set objctl = frm.Controls(ControlName)
    If TypeOf objctl Is Access.Control Then
       Set fncFindControl_INDEX = objctl
   Else
       Set fncFindControl_INDEX = Nothing
   End If
 
Exit_Point:
   Exit Function
 
Err_Handler:
   Select Case Err.Number
       Case 2465, 3265, 2482
           Set fncFindControl_INDEX = Nothing
           Resume Exit_Point
       Case Else
           Err.Raise Err.Number, Err.Source, Err.Description, Err.HelpFile,
Err.HelpContext
   End Select
 
End Function
'------ end of code ------

This sounds like an odd resquest.  If you are programming a form, you already know whether or not it has a given control?!

Anyways, you can loop through the controls of a form by doing something along the lines of:

Dim ctl As Control

For Each ctl In Me.Controls
        If ctl.Name = "ZipCode" Then
            ctl.Value = "Whatever value you want" 'Change the value of the control
            ctl.Visible = False 'Makes the control invisible
        End If
Next ctl

Be sure your update level of PowerPoint is 14.3.1 by going to the PowerPoint menu and choosing About PowerPoint. If you have an older version, head to the PowerPoint Help menu and choose Check for Updates.

1. Use Disk Utility or other font utility to verify the startup volume and repair permissions.

2. Completely remove Office 2011
