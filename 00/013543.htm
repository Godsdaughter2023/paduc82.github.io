{ = 1 + {DATE \@ "yyyy"} }

This document is set up to print file folder labels.

john korchok say:

It's working here. You must create a field within a field, which means you don't actually type any of those braces:

  1. Insert a Date field. The field codes will look like this { DATE  \@ "MMM d, yyyy"  \* MERGEFORMAT }
  2. Edit the field to show just the year: { DATE  \@ "yyyy"}
  3. Press Command + F9 to create a new blank field. (You may have been reading Windows help pages that use the Ctrl + F9 shortcut that doesn't work on a Mac): {  }
  4. In the blank field, type = 1 + The result should look like this: { = 1 + }
  5. Finally, cut the date field and paste it into the new field just after the plus sign: { = 1 + {DATE \@ "yyyy"} }
  6. Toggle the field codes and update the field to see 2019.

If you still can't get it working, here's a sample doc with the compound field, just copy and paste the field: Next Year Date Field

jay freedman say:

There is a fundamental difference between character styles and paragraph styles, and that will explain why you're seeing "underlying properties".

In the Styles pane (Ctrl+Alt+Shift+S), paragraph styles are marked with the ¶ symbol and character styles are marked with a letter a. Styles that have both symbols are called "linked" styles, meaning that they behave as paragraph styles if applied to an entire paragraph, or as character styles if applied to only part of a paragraph.

A paragraph style determines the properties for an entire paragraph -- not just font, size, color, but also line spacing, left and right indents, justification, paragraph borders and shading, and so forth. A character style determines only the properties that are applicable to individual characters -- font, size, color, italic, bold, underlined, etc. -- only the things that can be set in the Font dialog; plus borders and shading of characters or words, and the proofing language.

You can apply a paragraph style, such as Normal or Heading 1, to an entire paragraph, and then you can apply a character style to a word or phrase within that paragraph. The "underlying properties" are the ones that the paragraph style determined and that are not overridden by the character style. As an example, the Emphasis character style applies only the italic formatting -- every other property of the characters with Emphasis style is set by the paragraph style at that point.

In a sense, a character style is always independent of the Normal style, but there is no way to make it independent of whatever paragraph style is "underneath" it in the text.

Probably the best way to see concrete examples is to open the Reveal Formatting pane by pressing Shift+F1, and checking the box at the bottom for "Distinguish style source". Apply a paragraph style to some text, and then apply a character style to part of that text. The Reveal Formatting pane will show which properties of the text at the cursor location are determined by which style.

To answer your earlier question about the torn edge effect, it's one of many available in the Snagit program from TechSmith.
