-----

Ostensibly, array-enter (press

=TRIMMEAN(IF(B10:BH10<>0, B10:BH10), 4%)

TRIMMEAN (and AVERAGE) automatically ignores text data in a range.

However:

1. TRIMMEAN has nothing to do with excluding statistical outliers, if that is what you mean.  It simply exlcudes a percentage of the lowest and highest data, whether they are statistical outliers or not.

2. "A standard deviation of 4%" is a meaningless term, AFAIK.  If I were to give it meaning, I would guess that it means the standard deviation is 4% of the mean.

3. Trimming 4% of the data does not provide a "standard deviation of 4%", whatever that means to you.

4. "Standard deviation" is most meaningful with data that is normally distributed. Is your data normally distributed?

(The STDEVP calculation can be used as a measure of dispersion with any distribution. But there might be better measurements for non-normal distributions.)

joeu2004,
re:  TrimFunctions Stdev.P

For any percent specified, the size of the data set is altered using the same criteria as the TrimMean function.
(the number of cells excluded is always an even number - rounding down if necessary)

For a data set of 100 numbers (at 4%), the two smallest and two largest numbers are excluded from the data set.
To do that, the size of the data set is reduced by sorting it and removing the first two and last two elements from the array used by the function.

A check is made comparing the TrimMean function results from the original data set against the average of the new data set. The specified function (Stdev.P) is then applied against the array.
Do you see any problems with the above?

FIXED 3/4/16: Problems with POP accounts in Outlook 2016: Email is deleted from server or duplicated in Inbox

If you have a lot of data in your <Inbox> and want to take some additional precautionary steps avoiding newly downloaded dupes as an alternative to what's suggested - the process is as follows <before> connecting to the POP account the first time with the updated build

#1 - Create a new mail folder

     --> CTRL+A to highlight all items
