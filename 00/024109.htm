In Access 2010 where are the object properties stored - Microsoft Community

davekstlmo say:

In Access 2010 where are the object properties stored

I am trying to find the object properties like Table, Macro & Queries Description fields.  I want to be able to write a query to view and export these properties.

Is this possible?

Dave

 

peter_the_oak say:
You can read (and basically write) all the MS Access objects by VBA: tables, menus, queries, forms, reports, macros, even VBE modules. I recommend to do so because I have seen that the JET SQL has limits in dealing the DDL which forces you to evide and use the objects, even if you try hard to get it right by SQL. 

Of great value is that all these objects are in collections, and the properties of each object are in a collection too, which is the door opener for tasks like import and export. You can simply explore the objects in the VBE editor.

However, it will not be as easy as "just writing a nested loop". If you start working your object exports, you will see that every object has its little exceptions and the concept itself needs exceptions too:
  • Forms have Sections (which is hardly documented), may be 3 or 5, depends on if you have switched on/off form/page headers/footers.
  • Most objects are object trees, which means you have to parse the children and their properties too.
  • Forms may have subforms, which disturb the export process if they are opened in design mode.
  • The properties of a form may be of a variety of types, even big OLE objects of a big size. In that case, you would have to cast the bytes into a e.g. a comma separated string. Or MS Access cannot decide the value due to the current object state.
  • Tables must be distincted between their definition and their data. Experience shows that there are some tasks of creating a table which can only be done by direct object manipulation, not by the JET SQL.
and so on. Queries are the easiest objects to export.

To give you a start, here is a detailed example of code that exports all forms into XML files. Please be aware that this code is in proofing-of-concept state, it is not meant to be an example of nice code. Further, some lines that need other ressources are commented out. But it will give you a clear understanding and should even work. You want to add the XML library reference in the VBE editor. 

8< -------------------------


' Export all forms, each form results in one XML file.

Public Function FormsToXML() As DOMDocument60
    Dim objXML As DOMDocument
    Dim xmlRoot As IXMLDOMElement
    Dim xmlNode As IXMLDOMElement
    Dim xmlNode2 As IXMLDOMElement
    Dim xmlNode3 As IXMLDOMElement
    
    Const MAX_FORM_SECTIONS = 5
    
    
    Dim frmObj As AccessObject
    Dim frm As Form
    Dim strValue As String
    Dim blnAttriValid As Boolean
    

    ' Loop all forms.

    For Each frmObj In CurrentProject.AllForms
        Set objXML = CreateObject("Microsoft.XMLDOM")
        Set xmlRoot = objXML.createElement("forms")
        objXML.appendChild xmlRoot
        
        Set xmlNode = objXML.createElement("form")
        xmlNode.setAttribute "name", frmObj.name
        
        On Error GoTo propertyErrorHandler
        
        DoCmd.OpenForm frmObj.name, acDesign, , , , acHidden
        Set frm = Forms(frmObj.name)
                
        
         ' Export properties of the current form.

        Dim p
        For Each p In frm.Properties
            'Debug.Print p.Name
            If IsNull(p.Value) Then
                strValue = "[null]"
            Else
                strValue = CStr(p.Value)
            End If
            xmlNode.setAttribute p.name, strValue
        Next
        
        xmlRoot.appendChild xmlNode
        
        Debug.Print frm.name
        
        ' Export Section, each with properties
        ' Important: Each position of section(0 to 4) may be invalid or a ref
        Dim i As Integer
        Dim S As Section
        Dim blnSectionValid As Boolean
        For i = 0 To MAX_FORM_SECTIONS - 1
            blnSectionValid = True
            Set S = frm.Section(i)  ' !implicit error handler branch!
            If blnSectionValid Then
                

                ' Export properties of the current section
                
                Set xmlNode2 = objXML.createElement("section")
                xmlNode2.setAttribute "fioSectionNumber", CStr(i)
                For Each p In S.Properties
                    If IsNull(p.Value) Then
                        strValue = "[null]"
                    Else
                        strValue = CStr(p.Value)
                    End If
                    xmlNode2.setAttribute p.name, strValue
                Next
                xmlNode.appendChild xmlNode2
                
                ' Export controls of a section
                Dim ctr As control
                For Each ctr In S.controls
                    Set xmlNode3 = objXML.createElement("control")
                    xmlNode2.appendChild xmlNode3
                    
                    ' export properties of a control
                    ' add parent
                    
                    xmlNode3.setAttribute "fioParent", ctr.parent.name
                    
                    Dim knock
                    For Each p In ctr.Properties
                        
                        blnAttriValid = True

                        Set knock = p.Value ' ! implicit error handler branch jump !

                        If blnAttriValid Then
                            If IsNull(p.Value) Then
                                strValue = "[null]"
                            ElseIf p.name = "PictureData" Or p.name = "ObjectPalette" Then
                                ' Create a comma separated string as this is just a byte array. It's not implemented here.
                                ' ByteArray
                                'Dim cast() As Byte
                                'cast = p.Value
                                'todo: use lib_arraySize
                                'If UBound(p.Value) - LBound(p.Value) < 20000 Then ' XXXX
                                '    strValue = CStrByteArray(cast)
                                'Else
                                '    strValue = Array(0)
                               ' End If
                            Else
                                strValue = p.Value ' most properties are just strings.
                            End If
                            
                            xmlNode3.setAttribute p.name, strValue
                        
                        End If
                    Next
                Next
                
            End If
        Next
        
        DoCmd.Close acForm, frmObj.name, acSaveNo

        DoEvents

        objXML.Save "C:\" & frmObj.name & ".xml"
        
    Next
    Exit Function
    
propertyErrorHandler:
    Select Case err.number
        Case 13: ' wrong type, it's ok for just knocking at the door
            blnAttriValid = True
            Resume Next
        
        Case 2186: ' Not available in design mode
            blnAttriValid = False
            Resume Next
        Case 2462: ' Bereichsnummer nicht gültig
            blnSectionValid = False
            Resume Next
        Case 2196: ' Cannot read the value of this property
            blnAttriValid = False
            Resume Next
        Case 7751: ' In der Entwurfsansicht ist es nicht möglich, für ein OLE-Objekt,
                   ' das in einem gebundenen Objektfeld enthalten ist, den Wert der
                   ' Objektpalette abzurufen.
            blnAttriValid = False
            Resume Next
        Case 7784: ' Subform bereits im Entwurfsmodus geöffnet.
            Resume
        Case Else:
            MsgBox err.number & err.Description
    End Select
End Function

Troubleshoot Error Code c101a006 While Downloading/Updating Apps

Sometimes the people see an Error Code c101a006 while installing or updating apps. This is not a big problem. It's just a server error, however, please do the following step:

  1. Restart your phone. Press and hold the Power and Volume down keys simultaneously for 10-15 seconds. Release and the phone will restart.
  2.  If the issue still occurs, please wait for 1-2 weeks. This will automatically resolve this issue.
  3. If you want to immediately fix this issue reset the phone using the following method:
Reset your phone
Note

This erases all personal content on your phone and restores the factory settings.



Before continue with the reset, make sure your phone’s battery level is above 50%.
To check the remaining battery life, go to Settings > System > Battery saver (Lumia with Windows Phone 8: Settings > battery saver). If you are not able to check it, or the level is too low, charge your phone for at least 20 minutes.

Reset your phone by using the phone settings.
Select Settings > System > About > Reset your phone (Lumia with Windows Phone 8: Settings> About > Reset your phone).

However, if you cannot access the phone settings, follow the steps below:
  1. Disconnect the charger. Press and hold the Power and Volume down keys simultaneously. Release the keys when the phone vibrates and then immediately press and hold the volume down key until the phone displays an exclamation mark (!) on the screen. Release the volume down key.

  2. Press the volume and power keys in the following order: volume up, volume down, power, and volume down.

    Lumia 950 XL users: the physical key combinations differ slightly compared to other Lumias.

  3. Wait for your phone to reset. During the reset you will see gears rotating on the screen for up to 5 minutes, after which the screen goes blank for ~30 seconds and then your phone will restart.
  4. through the initial phone setup. Make sure that you have the date and time set correctly. Select Settings > Time & language > Date & time (Lumia with Windows Phone 8: Settings > date+time)

brian tillman [outlook mvp 2007-2019] say:

The Address Book is just a view of those Contacts folders you want Outlook to use for resolving names to addresses.  You can have all your contacts in either the Contacts folder associated with the Hotmail account, the default Contacts folder, which is probably the one used by the POP account, or both.  For whichever COntacts folder you want to appear in the address book, right-click that Contacts folder, choose Properties, select the Outlook Address Book tab, and check the box labeled "Show this folder as an e-mail Address Book".  The contents of that folder (if the contacts have email addresses or fax numbers) will then appear in the address book interface and will be use by Outlook when you resolve a recipient name.
