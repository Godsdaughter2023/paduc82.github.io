Sub SortRows()
'Tom Ogilvy macro
Dim r As Long
Dim lRow As Long
Dim lCol As Long

    Application.ScreenUpdating = False
    Application.Calculation = xlCalculationManual
    lRow = ActiveSheet.Cells(Rows.Count, "A").End(xlUp).Row
    lCol = ActiveSheet.Cells(ActiveCell.Row, _
    Columns.Count).End(xlToLeft).Column
    For r = 1 To lRow
        With Cells(r, 1).Resize(1, lCol)
            .Sort Key1:=Cells(r, 2), Order1:=xlAscending, _
            Header:=xlGuess, _
            Orientation:=xlLeftToRight, _
            DataOption1:=xlSortNormal
        End With
    Next r

    Application.ScreenUpdating = True
    Application.Calculation = xlCalculationAutomatic

End Sub

Sub AutoOpen()

use the following userdefined function. Copy it into module:

Function Dec2Bin(Number As Double) As String
Dim arrNumbers()
Dim dblNumber As Double
Dim dblDummy As Double
Dim dblMod As Double
  dblNumber = Int(Number)
  arrNumbers = Array("0", "1")
  Do
    dblMod = dblNumber - Int(dblNumber / 2) * 2
    dblDummy = (dblNumber - dblMod) / 2
    Dec2Bin = arrNumbers(dblMod) & Dec2Bin
    dblNumber = dblDummy
  Loop Until dblNumber = 0
End Function

Formula is: Dec2Bin(A1)
Result: 10011010010

rick rothstein, mvp - excel say:

How would a UDF (user defined function) capable of returning up to 94 bits (max decimal value of 9999999999999999999999999999) be for you? The following UDF works for positvie decimal values only)...

Function BigDec2Bin(ByVal DecimalIn As Variant, Optional NumberOfBits As Variant) As String
  BigDec2Bin = ""
  DecimalIn = CDec(DecimalIn)
  If DecimalIn < 0 Then
    BigDec2Bin = "Error - Positive values only!"
    Exit Function
  End If
  Do While DecimalIn <> 0
    BigDec2Bin = Trim$(Str$(DecimalIn - 2 * Int(DecimalIn / 2))) & BigDec2Bin
    DecimalIn = Int(DecimalIn / 2)
  Loop
  If Not IsMissing(NumberOfBits) Then
    If Len(BigDec2Bin) > NumberOfBits Then
      BigDec2Bin = "Error - Number too large for bit size"
    Else
      BigDec2Bin = Right$(String$(NumberOfBits, "0") & _
      BigDec2Bin, NumberOfBits)
    End If
  End If
End Function

If you are new to UDFs, here is how to install this function. Press ALT+F11 from any worksheet in order to get into the VB editor, click Insert/Module from the VB editor's menu bar once there and then copy/paste the above code into the code window that opened up. That is it. Go back to any worksheet and type your decimal number into any cell (I'll assume you used A2)... if your number is bigger than 99999999999 (that is, if it displays in scientific notation), you will need to format the cell as Text or precede this large "number" with an apostrophe... and use this formula to get the binary equivalent of it...

=BigDec2Bin(A2)

Note that you can optionally set the number of bits to be returned (in case you want to see leading zeroes) by specifying a second number showing the number of bits for the function to use. To return 20-bit numbers only (larger bit values will display a "too large" type error message doing it this way), use this...

=BigDec2Bin(A2,20)

doug robbins - mvp office apps & services (word) say:

Both the DollarText and CardText switches can only handle numbers up to 999999.  The easiest way to do what you want will be to split the number in the data source so that in the case where you have a number of 2150000.50, you have 2 in one field, 150000 in second field and 50 in a third field. Then in your mail merge main document use

The If...then...Else fields have been included to deal with the case where the number is less than 1,000,000 and\or it is a whole number.

macropod say:

Without splitting the data, you can use a field coded as:

{QUOTE{SET Val {MERGEFIELD Amount}}"ringit {=INT(Val/10^6) \# "'{=INT(Val/10^6) \* CardText} million ';;"}{=MOD(INT(Val),10^6) \* CardText}{=MOD(Val,1)*100 \# "' and sen {=MOD(Val,1)*100 \* CardText}';;"} only" \* Upper}

where 'Amount' is your field name.

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. The spaces represented in the field constructions are all required. For a macro to convert the above into a working field code, see Convert Text Representations of Fields to Working Fields in the Mailmerge Tips and Tricks thread at:
http://www.msofficeforums.com/mail-merge/21803-mailmerge-tips-tricks.html
or:
http://windowssecrets.com/forums/showthread.php/163017-Word-Mailmerge-Tips-amp-Tricks
