themase_100 say:

Open to specific record

hello all 

i am having trouble with access 2007 opening a form to specific records this is usually straight forward however i have two tables client and expenses on the client form i have a clientid field (auto number) on the expenses field i have clientID (number) i am trying to input data into the client record then click the cmd button and open the expense record showing the clientid filled in with a new record waiting to be populated but it is just coming up with a new record 

macro
FrmExpenses, Form, , ="[clientid]=" & [clientid], , Normal 

any ideas
ken sheridan say: You need to pass the current clientid value to the second form, which you can do via the OpenArgs mechanism.  I'd also suggest that the expenses form is opened in dialogue mode as this forces the user to close it ( or strictly speaking to at least hide it) before returning to the clients form.  Like most developers I rarely use macros, but as code in the button's Click event procedure it would be:

DoCmd.OpenForm "FrmExpenses", _
    DataMode:=acFormAdd, _
    WindowMode:=acDialog, _
    OpenArgs:=(Me.clientid)

If you are unfamiliar with entering code in event procedures, this is how it's done in form design view:

1.  Select the button and open its properties sheet if it's not already open.

2.  Select the On Click event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the above code as new lines between these.

You also have to enter some code in the Open event procedure of the FrmExpenses form to assign the value as its clientid control's DefaultValue property:

If Not IsNull(Me.OpenArgs) Then
    Me.clientid.DefaultValue = """" & Me.OpenArgs & """"
End If

Note that the DefaultValue property is always a string expression, regardless of the data type of the column to which the control is bound, so should be wrapped in literal quotes characters as above.

Another option of course would be to embed the FrmExpenses form in the clients form as a subform, linked on clientid.  That way you don't need any code at all as the linking mechanism passes the correct value to the subform.  To economise on space the main form's controls and the subform can be placed on separate pages of a tab control.
