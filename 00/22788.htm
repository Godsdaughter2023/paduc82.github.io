hi_there_2080 say:

here's the thing

I'm making a system for recording employees days off
so there's a combobox with all employees names (based on a table), and another for his substitute (based on the same table). Logically, the person who's having the day off cannot be in the list of substitutes, so how to do this?

I know it has something to do with validation rule, but cant figure out the rule. or maybe with beforeupdate, dunno.

hansv mvp say:

Let's say you have cboEmployee and cboSubstitute on a form frmDaysOff.

The table listing the employees is tblEmployees, with a primary key EmployeeID. The bound column of cboEmployee corresponds to EmployeeID.

Create a query with SQL

 

SELECT * FROM tblEmployees WHERE EmployeeID <> Nz([Forms]![frmDaysOff]![cboEmployee], 0)

 

Save this query, and set the Row Source of cboSubstitute to the name of this query.

 

Create an After Update event for cboEmployee:

 

Private Sub cboEmployee_AfterUpdate()
    Me.cboSubstitute = Null
    Me.cboSubstitute.Requery
End Sub

 

and also an On Current event procedure for the form:

 

Private Sub Form_Current()
    Me.cboSubstitute.Requery
End Sub
