If the document contains lots of fields, and your Word preferences are set to highlight fields as "Always", those fields in the document will show with a grey highlight.

The option can be changed from the below location:

Word>Preferences>View>Field shading: "Always"

Or

cbarnhorst say:

"Safe" is a relative term.  It depends on what you have to lose.  If this is your main (primary or production) computer then Microsoft advises against using previews on it.  It is possible for a preview build to cause an issue that could render the whole device unusable, causing a rebuild of the entire software system.  Rare?  Yes.  I have had it happen to me, though not with Windows 10.  If you decide to go ahead, at a minimum you should be in the habit of making system and data backups.

Personally, I don't consider any of my devices with important data "safe" for pre-release software.  Why don't you get a two or three year old laptop off of eBay and use it for testing?

rick-m say: I've been multi-booting my system with Windows 10 throughout the Insiders Program along with having Windows 7, Windows XP and Linux being my other operating systems. I haven't had any real issues when accessing and using my various Windows partitions. With my Linux partition, I find that every so often, I need to restore my GRUB boot manager to allow me access back into my Linux system as Windows 10 tends to disable this when it is doing a major update to it's system. Having a good understanding of how disk partitioning works is beneficial here. As a precaution, it is also advisable to keep good back-ups of your data when doing dual or multiboot systems. Data loss tends to happen if you are not careful when you are reconfiguring your system for multi-booting.
