a. user say:

Hi Ali,

The problem is that that the UAC (user account control) related program "consent.exe" will not start properly. This is the program that pops up on your screen(alternatively dimming it) to ask you if you want to let programs make changes to your computer.  

The following seems to only happen when "windows update" tries to update third party applications, such as graphics drivers. 

When "consent.exe" fails to start properly the prompt to accept changes to your computer never appears, halting the update you just started indefinitely. You can identify that this is the problem by opening your task manager and going through "processes" and "details" to see if "consent.exe" shows up. If it does, this is most likely your issue.

This is the very easy work around when you understand what the problem is:

  1. Open your "control panel" this is most easily done by right clicking the windows icon/start menu and selecting "control panel" 
  2. In the control panel window, next to the address bar, use the search field and search for "UAC" without quotations. 
  3. In the results select "change user account control settings"
  4. Drag the slider to the lowest setting. Finnish by selecting "Ok" in the bottom right corner and then "yes" when prompted. 

Note: To do this this you must be logged into Admin account.

This will disable any "consent.exe" prompts and let you update smoothly. If you feel unsafe not having these security prompts pop up, simply redo the steps after the update and put the slider to the level of security you want again.

Keep us posted to help you better.
