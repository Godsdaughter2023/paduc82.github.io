Evolution of AutoPlay and AutoRun in Windows - Microsoft Community

sushovon sinha say:

Evolution of AutoPlay and AutoRun in Windows

AutoRun is a legacy Windows feature that automatically runs a autorun.inf batch file stored in an external storage device such as a CD, DVD, Pendrive when the device is connected to a PC. Typically the autorun.inf file will install a software program contained in the CD/DVD or open up a menu screen as in a DVD movie or play the music files in a music CD.

AutoPlay is a later inclusion which opens a Windows application such as open Windows Media Player or Window Explorer or Burn Disc Application or Import pictures from a camera whenever an external storage device is connected to the PC. This is to provide an appropriate user interface to start using an external storage. Other AutoPlay options are "Take no action / Ask me everytime".

Thus AutoRun is effected by the device software whereas AutoPlay is effected by the operating system.

In Windows, AutoPlay action needs to be user configured from a list of choices, for Media (Optical Discs types) and Devices (Camera, Music Player, Mobile Phone etc). The AutoPlay action for Media applies generic to CD, DVD, Blue Ray whereas AutoPlay action for Devices are specific to the Model Name defined by the manufacturer. The model registration process takes place everytime a new Device is connected to the PC. Henceforth we shall refer to Media and Device together as External Storage.

Upon connecting an external storage to a PC, Windows detects the media type OR device and runs the AutoPlay action defined for that storage type.

AutoPlay can be enabled/disabled globally for all external storages through Control Panel > AutoPlay > de-selecting the option on top "Use AutoPlay for all media and devices". To disable AutoPlay for a specific external storage, AutoPlay should be enabled for all media and devices, and "Take no action" should be the defined choice for the storage type.

In Windows editions prior to Windows 7, the AutoRun feature runs independently and with higher priority than AutoPlay, and thus could install a program automatically without user acknowledgement even when AutoPlay is disabled for that particular storage type. This led to installation of malwares through PenDrives.

In Windows 7, AutoPlay controls the AutoRun program if present in the device. If AutoPlay is disabled, then both AutoPlay and AutoRun are disabled. This could apply either for all external storages Or specific to a storage type depending upon how AutoPlay is configured to disable at a global Or storage type level. If AutoPlay is enabled, then an AutoPlay Dialog Box appears on connecting an external storage which would list the AutoPlay action defined for that storage type and the AutoRun program if any present in the storage device. Thus the user acknowledges to execute an AutoRun program by explicitly clicking it, which introduces a level of security. For example if a device has a SETUP.EXE program to run through an autorun.inf file, and "Ask me everytime" is defined as the AutoPlay setting for that storage type, then the AutoPlay Dialog Box will list the possible actions as Run SETUP.EXE, Open Windows Explorer, Play using Media player etc for the user to make the choice.

However there are restrictions which apply only to PenDrives. Here AutoRun is altogether disabled to prevent self-execution of malwares and AutoPlay does not exist for user configuration. Instead AutoPlay for PenDrive is system managed through an AutoPlay Dialog excluding the AutoRun option. The system defined AutoPlay dialog options are "Open folder to view files" And "Speed up my system using Windows ReadyBoost". If AutoPlay is disabled for all Media and Devices, there will be no AutoPlay dialog on plugging a pendrive to a PC. There is no way to disable AutoPlay separately for PenDrive.

Windows 8 introduces AutoPlay for PenDrives via user configuration under a new external storage category termed "Removable Drive" while the AutoRun feature remains disabled. The AutoPlay action is executed directly without a AutoPlay Dialog, and this reduces an un-necessary overhead to select the action from the AutoPlay Dialog. This is also logical as the AutoPlay Dialog was originally created to include the AutoRun option which anyway has been excluded for PenDrives. This change makes use of PenDrives more friendly as it is more frequently used than any other external storage type. To enable AutoPlay Dialog for PenDrives, the AutoPlay action for Removable Drive should be set as "Ask me everytime". You can disable AutoPlay only for PenDrive by choosing "Take no action" as the AutoPlay action for Removable Drive.

In Windows 8, the AutoPlay for Media and Device category and it's sub-categories are replaced with distinct categories as Removable Drives, Camera Storage (Memory Card), DVDs, Blue-ray discs, CDs, Software and Devices.
