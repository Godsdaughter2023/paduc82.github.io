microcompsunltd say:

I figured it out:

You can not run the version that is part of the installation media you are trying to make your USB drive with on a EFI or non-PC/AT firmware system.

Bootsect is part of my Windows 8.1 UEFI systems, and is part of Windows 7 (maybe Vista) on up.  So what I did was opened command prompt running as Administrator, which takes me to C:\Windows\System32.  Note 'C' might be different, and will depend on which drive you have your current Windows OS on.

From there, without changing directories, I typed:

bootsect /nt60 f:

(f is the location of my USB drive and will be different for most people, just substitute your USB drive letter for it)

Voila, done.

My best guess is that the problem is the router.  When you ask for a connection, you are asking the router to recognize your PC.  It should require a password, which is only asked for once and stored in your PC, if you have made that initial choice.  The PC wireless adapter is then communicating back and forth with the router asking for an internal IP address.  If there is a long delay, it could be a problem with the router.

  4. Manage wireless networks

If the uninstall program is properly written then it will do all of this. If not then you have to do it manually.
