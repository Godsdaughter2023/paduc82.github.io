Display nonprinting characters (Ctrl+* or click the Show/Hide ¶ button). Place the insertion point at the beginning of your heading (at the top of the column) and press Enter. I can see 2 problems in

            Workbooks.Open filename:=filename
   
            Set DstWkb = ThisWorkbook
            Set SrcWkb = Workbooks.Open(filename:=filename, ReadOnly:=True)
a) you are opening filename twice

b) DstWkb should be the active workbook when the macro starts, not the workbook containing the code

So change the above to

            Set DstWkb = ActiveWorkbook
            Set SrcWkb = Workbooks.Open(filename:=filename, ReadOnly:=True)
and you will make some progress.

 

By the way, I would not try to put code into the user's personal.xlsb.  That file is really for users' own macros.
Consider the user who records some new macros into personal.xlsb and then you come along with a new version of your code and form.  How will he integrate the two?  It would be better for your code to be in a separate workbook which you install as an add-in.

You can most certainly add a button to the form to allow deleting. You have to remove the check and systems in place that prevents deleting of the record first.

So you simply have to remove the code that prevents the deletion of a quote when it been submitted.

Open up the table called quotes. Click on the table tab in the ribbon, and you see the "Before Delete" event in the ribbon (it is highlighted orange since there code for that event). Just click on that, and the code editor will launch. Now remove all of the code in that event (ctrl-a select all and then hit the delete key). Close + save. This is the code that is preventing the delete.

Once you remove all of that, then you can usually delete most invoices using the cancel invoice.

Of course I think the delete button on the form does have some additional checking. You can consider also adding an another button called delete that does no checking.  So try the above and this may suffice.

DeleteRecord

The editor will convert the above to
Runncommand
      DeleteRecord

So, placing a delete button on the form will now work. You could also add a close window command to the above to close the form after the record delete, but those kinds of things are "finishing" touches that are up to you.
