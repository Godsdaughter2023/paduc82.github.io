suzanne s. barnhill say:
I'm not clear what you're trying to achieve--limiting the amount of text or allowing unlimited text without wrapping. There are only two ways to constrain the amount of text in a cell--cell width and cell height. If you want the cell to grow so that text doesn't have to wrap, then you set the row to Auto height (no height setting). If you want text to extend beyond the cell, then, on the Cell tab of Table Properties, click Options and clear the check box for "Wrap text" in the Cell Options.

It could have to do with the vertical alignment of the cell. What you're describing isn't possible UNLESS you inadvertently removed access for users/groups from the parent site. If for example, you remove a user from a parent level group, you're changing the composition of the group AT THE PARENT LEVEL. One of my developers once removed "Everyone" from the parent site Visitors group, after breaking inheritance at the sub site level - thinking that would only affect the sub site. But that group lives at the parent level so it actually removed access to all my users. It took about 100 access request emails (and fortunately only 10 minutes) for me to realize what had happened and to fix it.  

Because permissions are one of the many poorly designed, unituitive features in SP Online, the best practice is to run a development/test site collection and try things out before committing them to a live site.
