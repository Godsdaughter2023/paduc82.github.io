If you want +/-25%, which includes 0%, then you want to multiply by a random value between 1-25% = 0.75 and 1+25% = 1.25.  Note that that is a range of 50% = 0.50.  So you might want:

=IFERROR(VLOOKUP($A2,sheet2!$A:$Y,9,0)*(0.75+RAND()*0.50),"")

e.j. gun say:

=IFERROR(VLOOKUP($A2,sheet2!$A:$Y,9,0)*(0.75+RAND()*0.25),"")

 

This function is multiplying the result of the VLOOKUP by a number between 0.75 and 1.00.  Is that what you want it to do?  The RAND function produces a number between 0.0 and 1.0, so you are adding a number between 0.0*0.25 and 1.0*0.25 to 0.75.

 

If you are trying to randomly increase or decrease your value by a certain amount, try something like the formula below:

 

=IFERROR(VLOOKUP($A2,sheet2!$A:$Y,9,0)*(1.0 + RANDBETWEEN(-25,25)/100.)

 

This formula will randomly increase/decrease your value by 25%.

Why doesn't Excel 2010 allow me to save a CSV file?

2. When exporting to a CSV file it does not quote the text fields.

joeu2004 say: Samir wrote:
1. When exporting a spreadsheet to a CSV file it is using a semicolon as a field separator instead of the by all means standard comma! 

Look at the List Separator setting in the Regional and Language Options control panel.  It is probably a semicolon.  Perhaps that is standard for your language setting.  Perhaps it was changed manually.  In either case, you should be able set it to comma.  But beware:  that might have other implications for your computer usage overall.

Samir wrote:
2. When exporting to a CSV file it does not quote the text fields.

Nothing you can do about that, other than write your own save procedure using VBA.  Excel quotes fields only when certain conditions arise; for example, when the cell value contains the field separator.

Samir wrote:
3. When a number in a cell is stored as text (with apostrophe prefix) Excel automatically converts it to a number upon export to CSV and approximates it, like so: 5465198745513540 into 5,4652E+15.

First, Excel writes values to the CSV field the way that they are displayed.  So if the original value was the text "5465198745513549" (with or without an apostrophe), Excel will write 5465198745513549 to the file.

You can see that by opening the file using Notepad or an equivalent text editor.

Second, even if the field were enclosed in parentheses, Excel will still try to interpret it as a number.  So even if you manually wrote (or used VBA to write) "5465198745513549" to the CSV file, Excel will still treat that as it would any 16-digit numeric input.  In this example, the last 9 would be ignored and replaced with zero.

The only work-around is use the Import Text File wizard instead of opening the CSV file directly.  And that will work only if you can select the entire column and set the Column Data Format to Text in the last dialog box.

Otherwise, you would need to write a VBA procedure to input the file and treat individual fields in the manner that you require.

Third, however, if the original value saved to the CSV file is actually "5465198745513540" -- that is, up to 15 digits followed by 1 or more zeros -- Excel's treatment of that as a number will not adversely alter the value.  That is, it will indeed be input as 5465198745513540.

The appearance of 5,4652E+15 in the cell is just a function of formatting.  Excel sets the format to General for most numeric input.  (Some exceptions:  dates and percentages.)

So all you might need to do is set the format to Number with zero decimal places.

If you want the cell to be treated as text, it is best to use the Text To Columns wizard, selecting Text for the Column Data format in the last dialog box.

Caveat:  Alternatively, you could set the cell format to Text.  And it might appear that the number was converted to text; for example, it will be left-adjusted, the default for text.  But ISTEXT(A1) will demonstrate that the cell value really was not changed to text.  It will become truly text only after you "edit" the cell; for example, by pressing F2, then Enter.

scheissschiesser say:

I want to insert today's date as static text each time I insert a new slide

Hello,

I use a PP presentation to keep track my iterations of a design process as I make changes to said design. Each time I insert a slide, I would like to have today's date appear somewhere on the slide, but as static text so that it doesn't change when I open/edit the presentation tomorrow (when I continue with the design process, for instance). Is this possible? It's just quite annoying to have to copy/paste or manually type the date into a slide every time I insert it. Using a masterslide with the date automatically updating doesn't work, because that will change tomorrow. And static text doesn't work, because that will change all of the slides. Any ideas?

Much Appreciated.
steve rindsberg say:
Here's a little macro that'll do it.  
You can adjust the position, size, font name etc as desired.

See this if you're not familiar with running VBA in PPT:
How do I use VBA code in PowerPoint
http://www.pptfaq.com/FAQ00033_How_do_I_use_VBA_code_in_PowerPoint.htm



Sub InsertDate()

Dim oSl As Slide
Dim oSh As Shape

Set oSl = ActiveWindow.Selection.SlideRange(1)
With oSl
    Set oSh = .Shapes.AddTextbox(msoTextOrientationHorizontal, 10, 10, 100, 100)
    With oSh
        .TextFrame.TextRange.Text = Date
        ' or this to add the time and date:
        ' .TextFrame.TextRange.Text = Date & vbTab & Time
        With .TextFrame.TextRange.Font
            .Size = 12
            .Name = "Arial"
        End With
    End With
End With



End Sub
