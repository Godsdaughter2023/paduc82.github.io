daniel g. grau say:
As usually advised, you first need to eliminate duplicates and bad fonts.

While in Font Book, also do Edit > Select All, followed by File > Validate Fonts.

If that fails to solve your problem, continue below:

There are three places that fonts reside (full paths from the top of your hard disk):

  1. /System/Library/Fonts
  2. /Library/Fonts (this one contains a Microsoft folder, with their official fonts)
  3. /Users/USER_NAME/Library/Fonts (sometimes called ~/Library/Fonts or "Home/Library/Fonts)

1) Fonts in the first one are supplied by Apple. You would seldom modify this folder

2) Fonts in the second folder are available to all users/all accounts in the computer

3) Fonts here are only seen by user USER_NAME. They are private.

Open those three folders side by side. (If you are on Lion, you will have to use the Finder's Go menu and paste or type those paths in the field.) Verify where the problem font resides. There should be no duplicates.

Note that the file name does not indicate the font's name as shown in font menus. Say you see "Times New Roman Bold" and also "TNR bold" or "Times Roman Bd," that's surely a duplication. Move the duplicate items to the Desktop, for instance, and test. (Word must be re-launched each time you modify the available fonts.)

Other users have aldo had problesm with TNR. If changing the TNR font's location does not work (it's default place is the Microsoft folder mentioned above), download this four-in-one font: TNR. The format is .hqx, so perhaps you'll need to download Stuffit Expander.

cmb1974 say: Resolved the issue.  If you delete  VBE7.DLL and then launch Excel with an .xlsm file it will reinstall VBA and the Macros will work again.

Here is a script that I wrote to have users run to find which version they are using and delete the file.  

remove_vbs.vbs

On Error Resume Next

Dim WshShell
Dim OsType

Set WshShell = CreateObject("WScript.Shell")
If WScript.Arguments.length =0 Then
  Set objShell = CreateObject("Shell.Application")
  'Pass a bogus argument with leading blank space, say [ uac]
  objShell.ShellExecute "wscript.exe", Chr(34) & _
  WScript.ScriptFullName & Chr(34) & " uac", "", "runas", 1
Else


OsType = WshShell.RegRead("HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Environment\PROCESSOR_ARCHITECTURE")

If OsType = "x86" then
Delete32
elseif OsType = "AMD64" then
Delete64
end if
End if

Function Delete32
Set objFSO = CreateObject("Scripting.FileSystemObject")
objFSO.DeleteFile("C:\Program Files\Common Files\Microsoft Shared\VBA\VBA7\VBE7.DLL")
WScript.Echo "Deleted Visual Basic for Applications on a (X86) system" 
End Function

Function Delete64
Set objFSO = CreateObject("Scripting.FileSystemObject")
objFSO.DeleteFile("C:\Program Files (x86)\Common Files\Microsoft Shared\VBA\VBA7\VBE7.DLL")
WScript.Echo "Deleted Visual Basic for Applications on a (X64) system" 
End Function

If you have a typo and you meant to write =SUM(E2:F3) for the "next row" -- that is, the cumulative sum of two columns starting with row 2 -- try copying the following formula in G2 and pasting it down column G:

=SUM($E$2:F2)


Note the use of reference types.  Only the first term is an absolute reference ($E$2), which does not change.  The second term is a relative reference (F2), which changes as you paste down the column.

[EDIT] Alternatively, if you meant to write =SUM(E2:F3) for the "next row", meaning the sum of two columns in the last 2 rows -- try copying the following in G3 and pasting it down column G:

=SUM(E2:F3)

Note that the formula in G2, if any, is still =SUM(E2:F2), since there is only the one row to sum.  (KISS!)

When a browser runs an applet, the Java Runtime Environment (JRE) stores the downloaded files into its cache folder for quick execution later and better performance. Both legitimate and malicious applets (malicious Java class files) are stored in the Java cache directory and many anti-virus programs may detect them as threats. The detection can indicate the presence of malicious code which could attempt to exploit a vulnerability in the JRE, specially if an out-dated/vulnerable version of Java is installed. For more information about Java exploits, please refer to this article: Anti-Virus has detected a virus. Is it related to Java?

A detection of these files as a threat does not always mean that a machine has been infected; it indicates that a program included the viral class file but this does not necessarily mean that it used the malicious functionality. In some cases, it might have been a failed attack attempt, or may be remnants of an old inactive file now discovered. We can only go by what was detected and your description of whatever signs or symptoms of infection you are experiencing.

Now, please note that in order to help avoid potential threat exploits (or attack attempts) like the one you're probably seeing, MS would surely recommend (as in this article) you keep, not only Windows and IE, but every third-party software (including Java) always up-to-date.

Also see: How to protect your computer against dangerous Java Applets.

You should be aware, however, that it's a known issue that even at the latest and current version of the Java plugin

Please note that the JRE (same as for the Adobe Flash Player) has always been the most exploited piece of software - affecting your general browsing experience - and certainly being a high infection vector for many threats. It is easily exploited.

Unless you know that you need it, I recommend that you uninstall it completely.

I would do as most recommended by many reputable security researchers: If you don't need it, stay away from it!

To 'help' completing the removal of suspected threat, I would suggest you try the following:

- Delete the Contents of your Temporary Internet Files Folder
- Clear your Java Cache
- Remove ALL VERSIONS of the Java Plugin from your Control Panel, and DO NOT reinstall it back!

- Reboot the OS, update your resident AV software, and run a full scan. You can also run the MSS again if you wish. Running some additional scanners such as AdwCleaner and MBAM as suggested above by J is also a good idea.

TIP: If you find out later on that you 'genuinely need' to continuously visit a 'legitimate website' that requires Java to display their content properly, you can always reinstall it (obviously the latest and current version - with ALL security settings applied) back from Java.com. Enable it only on an alternate 'updated' browser of your choice (e.g., Mozilla Firefox, for instance), which to be used for you to visit those Java-based website(s) only! And stick to your default Java-disabled main browser (IE) for everything else.
To learn how to disable the Java web plug-in in Internet Explorer, please see: KB2751647, and this article from the MR&D's Team: Java: A Fix it for when you cannot let go.

Disclosure: The actual macro I use is some mighty  fine code written by someone other than myself. I

http://windowssecrets.com/forums/showthread.php/75402-Macro-to-Remove-Duplicate-Entries-(MS-Word-2003)

 

Sub RemoveDupEntries()
Dim para As Paragraph
Dim doc As Document
Dim vEntries() As Variant
Dim vDupEntries() As Variant
Dim sParaText As String
ReDim vEntries(0)
ReDim vDupEntries(0)
Set doc = ActiveDocument
For Each para In doc.Paragraphs
sParaText = Left(para.Range.Text, para.Range.Characters.Count - 1)
If IsMember(vEntries, sParaText) Then
If IsMember(vDupEntries, sParaText) = False Then
Push vDupEntries, sParaText
End If
para.Range.Delete
Else
Push vEntries, sParaText
End If
Next para
' Now vDupEntries contains any items that were duplicated.
' You can insert it at the end of doc 3 as needed
' For example:
' Documents("Doc3").Content.InsertAfter Join(vDupEntries, vbCr)
End Sub
'
Function Push(ByRef vArray As Variant, ByVal str As String)
ReDim Preserve vArray(UBound(vArray) + 1)
vArray(UBound(vArray)) = str
End Function
'
Function IsMember(vArray As Variant, ByVal str As String)
Dim v As Variant
For Each v In vArray
If v = str Then
IsMember = True
Exit Function
End If
Next v
