doug robbins - mvp office apps & services (word) say:
Hi there,

I am using MS Word 2010 for writing my dissertation. I have mastered templates for headings, headers, fonts etc. However, I cannot seem to find a way to change the default alignment settings of a table, i.e. for every new table created in the document have the alignment centered (vertically and horizontally). Is there a way to do this?

cheers,
J


Modify the formatting of the Table Grid Style.  The easy way to get access to it is to use Ctrl+Shift+s to display the Apply Styles dialog and when you have the selection inside a cell of a table, the Table Grid style will appear in that dialog and you can then click on the Modify button.

In the Modify Style dialog, click on the Format button and then on the Table Properties item and go to the Cell tab of the Table Properties dialog and select Center for the Vertical alignment.  Then, back in the Modify Style dialog, click on the Format button again and then on Paragraph to set the horizonatl alignment of the paragraphs to Center.

Finally, before exiting from the Modify Style dialog, select the radio button for "New documents based on this template".

chip pearson say:

When you use the name of the userform in code (other than in a variable declaration), an instance of that userform is created if it does not already exist.  The term "UserForm1" is both a variable name and a class name. Behind the scenes, VBA is doing something like

 

Dim UserForm1 As New UserForm1

 

Using the "New" keyword makes the variable an "auto-instancing" variable and the runtime will automatically create it when it is encountered in code. This is why for example, the following will always return FALSE.

Debug.Print UserForm1 Is Nothing

Just using "UserForm1" in the code causes it to be loaded. The logic, were it written in VBA, is something like

If UserForm1 Is Nothing Then
    Set UserForm1 = New UserForm1
End If
Debug.Print UserForm1 Is Nothing

Here, UserFrom is not Nothing because the runtime creates it when you are testing it.

Without using another variable to hold the form reference, you can never test whether a form is loaded because doing so loads the form.

 

The way to work around all this is to create a variable of the userform's type, one variable for each form. For example,

Dim UF1 As UserForm1
Dim UF2 As UserForm2
Dim UF3 As UserForm3
Dim UF4 As UserForm4

Then to find whether a form is loaded, use the IS operator in a For Each loop, iterating through the VBA.Globals.UserForms collection. E.,g,

Dim FormToFind As UserForm
Set FormToFind = UF3 ' which form to test if loaded
For Each UF In VBA.Global.UserForms
    If UF Is FormToFind Then
        MsgBox "Found"
        Exit For
    End If
Next UF

You can also loop through VBA.Globals.UserForms with an integer index, testing the Caption property to see each form is the one you are looking for. E.g.,

 

Dim N As Long
With VBA.Global.UserForms
    For N = 0 To .Count - 1
        Debug.Print N, .Item(N).Name
        If .Item(N).Caption = "My Caption" Then
            Debug.Print "My Caption Loaded"
        End If
    Next N
End With

 

I hope this will get you going in the right direction.

tom ogilvy say:

if the tab named with the value of the sheetname variable (or if you were indicating that a sheet name would be there) is not the activesheet in the activeworkbook, then this line will raise an error:

Set rg1 = ActiveWorkbook.Worksheets(sheetname).Range(Cells(j + 26, 19), Cells(j + 86, 19))

you can thwart this with

With ActiveWorkbook.Worksheets(sheetname)

  set rg1 = .Range(.Cells(j + 26, 19), .Cells(j + 86, 19))

End With

the second caution is that you close a workbook.  If it is the workbook that was the activeworkbook, then rg1 will loose its reference in that workbook.  

Dim bk as workbook

' set a reference to the currently activeworkbook

set bk = ActiveWorkbook

Workbooks.Open filename:=filepath & filename

With ActiveWorkbook.Worksheets(sheetname)

  set rg1 = .Range(.Cells(j + 26, 19), .Cells(j + 86, 19))

End With             

rg1.copy bk.Worksheets("Capacities").Range("K3")           
' the copy is complete now close the workbook

Workbooks(filename).Close

I certainly don't know what workbooks you have open and so forth so this is just a guess at what  your situation is and why you might get this error.
