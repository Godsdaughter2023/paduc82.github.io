    john john __ say:

    "Make Active"...there is no such action available at the disk level.  What you mean is "Mark Partition as Active".  This isn't what you or the other poster might think it is, it doesn't make a disk "available or unavailable", this is applied at the partition level and all that this does is tell the MBR which partition to boot when it takes control of the booting process.  In other word, the Master Boot Record has a bit of code in it telling it to load the boot sector of the active partition, that is all.

    There need not be any active partition on the disk at all, data disks often do not have an active partition, active partitions are only needed to boot operating systems, otherwise there is no need to have an active partition and this doesn't make a hill of beans as to the ability to use the disk to store data.

    There can only be one active partition on a given disk at any one time, if one partition is already marked as active and if you select to mark another partition on the same disk as active the active flag will be be removed from the already active partition and the other partition will be marked as active, the active flag will be "toggled" between the partitions.  If you remove the active flag from the partition being used to boot your Windows installation then MBR will not be able to boot Windows and it will give you an error message to the effect that it couldn't find an operating system to load or the the boot loader is missing.
