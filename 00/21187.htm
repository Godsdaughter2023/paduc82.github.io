scott ladewig say:

This is old and you probably have the answer by now, but in case anyone else comes across the question...

Jay Freedman answered this in another query as follows (my summary, not his exact text):

Office 2013 stores it in "HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Word\User MRU". Each user for Office will have their own key, and under that key will be "File MRU". Each pinned file has values named "Item 1", "Item 2", etc. Office 2016 is same except under the 16.0 key of course.

Value looks like this: [F00000000][T01CFA783D052DEC0][O00000000]*C:\test1\test.docx

If the item is not pinned, the first section will be [F00000000]. If it is pinned it will start with [F00000001].

One other note, lairee:
 
If you want the routine to be case-sensitive, you can set OPTION COMPARE BINAR at the top (see below).
You could also have it flag invalid entries within the range, if you wish.
 
In the code below, I also cleaned up another line and removed some debug code.  It will stop testing at an invalid entry, so a set of entries such as
 
XX00    XX01    END
 
will NOT show a "skip"
 
===============================
Option Explicit
Option Compare Binary
Function IsSkip(rg As Range) As String
    Dim c As Range
    Dim i As Long, j As Long
    Dim n1 As Long, n2 As Long
    Dim sChar1 As String, sChar2 As String
i = 1
 
If rg.Rows.Count <> 1 Then
    MsgBox ("Data must be in a single row")
    Exit Function
End If
 
Do While rg(1, i + 1).Text Like "[A-Z][A-Z][0-9][0-9]" _
And (i + 1) <= rg.Count
    n1 = 0: n2 = 0
    For j = 1 To 2
        sChar1 = Mid(rg(1, i).Text, j, 1)
        sChar2 = Mid(rg(1, i + 1).Text, j, 1)
            n1 = n1 + (Asc(sChar1) - 65) * 26 ^ (4 - j)
            n2 = n2 + (Asc(sChar2) - 65) * 26 ^ (4 - j)
    Next j
            n1 = n1 + Val(Mid(rg(1, i).Text, 3))
            n2 = n2 + Val(Mid(rg(1, i + 1).Text, 3))
   
    If (n2 - n1) <> 1 Then
        IsSkip = "Skip"
        Exit Function
    End If
    i = i + 1
    Loop
End Function
