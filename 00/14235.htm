formula 3 ::(((entry time - exit time))-int((entry time - exit time)))/0.041666667

Example :

How to I account for the Seconds in the entry and exit time such that the sucm is accurate ?

P.S : Please post back with the status of the issue. If this post was helpful to you, please vote for my post If this post has helped you resolve your issue, please mark my post as answered.
ronrosenfeld say: 1.  Your formulas will return negative numbers.  If that is not what you want, you should be using (exit time - entry time) and not the reverse.

2.  Assuming your entries are proper "time" entries,   (ExitTime - EntryTime) will return the difference, including seconds, as a decimal where 1 = 24 hours (1 day).

3.   If you want some other decimal, you need to use the appropriate multiplier.
            For hours:   24  (hours in a day)
            For minutes:  1440
            For seconds:  86400

So your five seconds elapsed time in your example is, using the formula (ExitTime-EntryTime) * Multiplier
      0.0000578703703703702 Days
      0.00138888888888889      Hours
      0.0833333333333331         Minutes
     4.99999999999999               Seconds

Floating point arithmetic is subject to some rounding errors which is the reason the formula for seconds does not return exactly 5, but you can always ROUND to the appropriate level of precision.

This registry fix may get Defender working . . .

Click your Start Button, type regedit and hit Enter to open the Registry Editor

In the left pane, navigate to:
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SecurityHealthService

In the right hand pane, double click Start

Set its value to 2

In the left pane, navigate to:

HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows Defender

In the right hand pane, if DisableAntiSpyware and DisableAntiVirus exist, double click both DisableAntiSpyware and DisableAntiVirus

Set their values to 0

Set their values to 0

Clikc OK and close the Registry Editor

Restart your PC

No, of course it can be built as a multiuser database. Access was designed initially to be a multi-user database from version 1.0. By default it supports multiple users.
