Anyone experience this or have a solution?

jan karel pieterse say:

Assuming the number of COLUMNS (which -after transposing- become rows) exceeds 65.536 I'm afraid you are right, the code below fails too:

Sub test()
    Const NumCols As Long = 65537
    Dim vArr(1 To 4, 1 To NumCols) As Long
    Dim lct As Long
    For lct = 1 To NumCols
        vArr(1, lct) = lct
    Next
    Range(Cells(1, 1), Cells(NumCols, 4)).Value = Application.Transpose(vArr)
End Sub

The only way I know to resolve this is to do the transpose yourself:

 

Public Function MyTranspose(vArr As Variant) As Variant
    Dim lRow As Long
    Dim lCol As Long
    Dim vNewArray() As Variant
    ReDim vNewArray(LBound(vArr, 2) To UBound(vArr, 2), LBound(vArr, 1) To UBound(vArr, 1))
    For lRow = LBound(vArr, 1) To UBound(vArr, 1)
        For lCol = LBound(vArr, 2) To UBound(vArr, 2)
            vNewArray(lCol, lRow) = vArr(lRow, lCol)
        Next
    Next
    MyTranspose = vNewArray
End Function

graham mayor say:

The short answer is no, but if you open the message in Outlook (as opposed to previewing it) then you could use an Outlook macro to print the selected text using Word to do the printing e.g.

Option Explicit

Sub PrintSelected()
Dim wdApp As Object
Dim wdDoc As Object
Dim bWordWasNotRunning As Boolean
    On Error Resume Next
    If TypeName(ActiveWindow) = "Inspector" Then
        If ActiveInspector.IsWordMail And ActiveInspector.EditorType = olEditorWord Then
            If Len(ActiveInspector.WordEditor.Application.Selection) > 0 Then
                ActiveInspector.WordEditor.Application.Selection.Copy
                bWordWasNotRunning = False
                Set wdApp = GetObject(, "Word.Application")
                If Err Then
                    Set wdApp = CreateObject("Word.Application")
                    bWordWasNotRunning = True
                End If
                On Error GoTo err_Handler
                wdApp.Visible = True
                Set wdDoc = wdApp.documents.Add
                With wdDoc
                    .Range.Paste
                    .PrintOut
                    .Close 0
                End With
                If bWordWasNotRunning Then wdApp.Quit
            End If
        End If
    End If
lbl_Exit:
    Set wdApp = Nothing
    Set wdDoc = Nothing
    Exit Sub
err_Handler:
    Err.Clear
    GoTo lbl_Exit
End Sub

Word does not have this capability without even more drastic work-arounds like tables.

You can have a line break rather than a paragraph break by pressing Shift-Enter.

You can have a hanging indent.

This is more like a graphics function than a Word processing function.

In Excel you would be able to click in cells below your last typing. You could get the same effect using a table in Word.
