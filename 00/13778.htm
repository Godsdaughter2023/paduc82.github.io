INSERT INTO has two forms:


1) INSERT INTO table (field1, field2, ...) VALUES (value1, value2, ....)

2) INSERT INTO table (field1, field2, ...) SELECT field1, field2, ... FROM othertable


The SELECT statement in 2) can contain a WHERE clause, an ORDER BY clause etc.


The first form inserts one record only. There is no way to add multiple records. You'd have to create a separate INSERT INTO statement for each record.


The second form can insert multiple records in one go.

andreas killer say:
Is there a way to quickly copy groups of yes/no radio buttons so that the group name automatically changes to something new? 
AFAIK no. But you can change the GroupName property afterwards, select some text so that the Check-/Optionbuttons are inside the selection and execute the macro below.

Andreas.

Sub SetGroupNameInSelection()
  Dim oIS As InlineShape
  Dim GroupName As String
  For Each oIS In Selection.Range.InlineShapes
    If oIS.Type = wdInlineShapeOLEControlObject Then
      Select Case oIS.OLEFormat.ClassType
        Case "Forms.CheckBox.1", "Forms.OptionButton.1"
          If GroupName = "" Then
            GroupName = InputBox("GroupName", "SetGroupNameInSelection", _
              oIS.OLEFormat.Object.GroupName)
            If GroupName = "" Then Exit Sub
          End If
          oIS.OLEFormat.Object.GroupName = GroupName
      End Select
    End If
  Next
End Sub

Hmm, this is another reason I stopped using Groups... no easy way to see actual name of the Object without right mouse click on each Object and selecting Properties, a real pain.

Side note, I will disagree with Scott in this one instance... the purpose of Groups was so you could name your Objects with spaces and the like for easier identification.  It's like the Shortcut on your Desktop, you would not name that the name of the file, you would name it something you would recognize.  Moving on...

SELECT MSysNavPaneGroupCategories.Filter AS GroupName, MSysNavPaneGroups.Name AS ObjectName, MSysNavPaneGroupToObjects.Name AS ShortcutName
FROM (MSysNavPaneGroups INNER JOIN MSysNavPaneGroupToObjects ON MSysNavPaneGroups.ObjectID = MSysNavPaneGroupToObjects.ObjectID) INNER JOIN MSysNavPaneGroupCategories ON MSysNavPaneGroupToObjects.Id = MSysNavPaneGroupCategories.SelectedObjectID;

Copy/paste the above to a query (SQL) and run.  IMPORTANT NOTE: Do not change any data in this query as they are System tables and you could easily destroy your database.
