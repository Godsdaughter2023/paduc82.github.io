stet7 say:

Problem with wildcard brackets

Hi,

I wrote the macro pasted below to delete leading and trailing spaces before and after various characters. The result: almost all the spaces in the doc are deleted. As you'll see, I crammed a bunch of search terms inside brackets, and VBA is apparently treating that as a search for any character (as in the ? wildcard). So are there some rules applicable here to using wildcard brackets that I'm not aware of? Is there a limit to how many characters you can use? Do they have to be given in a certain order? 

************************

Sub LeadingAndTrailing()

Dim Rng As Range
For Each Rng In ActiveDocument.StoryRanges
With Rng.Find
  .ClearFormatting
  .Replacement.ClearFormatting
  .MatchWildcards = True
  .Replacement.Text = "\1"

  'leading
  .Execute FindText:=" ([,;:-^=^+\?\!\)\]^02^0146^0148])", Replace:=wdReplaceAll

  'trailing
  .Execute FindText:="([-^=^+$\(\[^0145^0147]) ", Replace:=wdReplaceAll

End With
Next Rng
End Sub

macropod say:

Try:

Sub LeadingAndTrailing()
Dim Rng As Range
For Each Rng In ActiveDocument.StoryRanges
  With Rng.Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .MatchWildcards = True
    .Replacement.Text = "\1"

    'leading
    .Text = " ([,;:\-^=^+\?\!\)\]^02^0146^0148])"
    .Execute Replace:=wdReplaceAll

    'trailing
    .Text = "([\-^=^+$\(\[^0145^0147]) "
    .Execute Replace:=wdReplaceAll

  End With
Next Rng
End Sub

I would not expect resetting the view to fix it - IMAP accounts tend to have this behavior. Check your Send /Receive settings (Ctrl+Alt+S) - outlook should be checking for new mail about every 5 min - less than that can cause problems.
