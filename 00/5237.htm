ken sheridan say: Reference the year combo box as a parameter in the WHERE clauses of the RowSource properties for each list box, e.g.

WHERE YEAR([DateCompleted]) = Form!cboYear

and:

WHERE YEAR([DateAssigned]) = Form!cboYear

where DateCompleted and DateAssigned are columns of date/time data type in the relevant table on which the list box is based, and cboYear is the name of the year combo box.  Note the use of the Form property to return a reference to the current form.  There is no need to fully reference it as a member of the Forms collection.

In the AfterUpdate event procedure of the year requery each list box with:

Me.[lstRequirementsCompleted].Requery
Me.[lstRequirementsOutstanding].Requery

where lstRequirementsCompleted and lstRequirementsOutstanding are the names of the list boxes.

The kicker is that I don't want to return those things that match but I want to return those requirements from the Total List that do not appear on the Completed List.

ken sheridan say: In essence your suggested approach is correct, but you don't need two separate queries.  You need to firstly return all requirements for the selected student on the basis of their expert, basic or inactive classification, regardless of the year or whether they have completed the requirement or not and then restrict the result set to those which are not completed in the selected year, which I gather means that no row would exist in whatever table models the returns between students, requirements and years.  This can be done by means of an outer query which is restricted by means of the NOT EXISTS predicate against a subquery which returns all requirements completed by the student in the year in question.  I'm not clear on which tables you would base the outer query, however, but let's assume you have a query named qry_StudentRequirements which includes columns StudentID and Requirement.  You appear already to have a query which returns the requirements completed as qry_StudentReqStatus, so this can be used as the basis for the subquery.  So the query would be:

SELECT Requirement
FROM qry_StudentRequirements
WHERE StudentID = [Forms]![frm_StudentReqStatus]![cboSelectStudent]
AND NOT EXISTS
    (SELECT *
     FROM qry_StudentReqStatus
     WHERE qry_StudentReqStatus.StudentID = qry_StudentRequirements.SudentID
     AND  qry_StudentReqStatus.Reqirement = qry_StudentRequirements.StudentID
     AND YEAR([ClassDate]) =[Forms]![frm_StudentReqStatus]![cboYear]);

You could almost certainly cut out the middlemen by referencing the relevant base tables rather than queries in both the outer query and subquery, but without more information on the base tables which make up your model I can't say exactly how.
