    The issue is always related to the file path which exceeding 256 characters. For this kind of issue, we’d like to share following information here with you:


    Symptom:
    When syncing the library to the local computer, the local folder name is “T” instead of “[Web site title] - [Document library title]”

    Cause:
    The path length of some files in the local synced folder exceeds 256 if using “[Web site title] - [Document library title]”.

    Resolution:
    Reduce the length file path, stop sync the library, then resync the library   

                        a. Reduce the length file path on the SharePoint online site
                        b. Stop sync the library according to the KB https://support.office.com/en-us/article/Stop-syncing-a-library-with-the-OneDrive-for-Business-app-a7e41f1f-3a98-4ca7-9443-f10250688330
                        c. Resync the library

    How to find the files whose path length exceeds 256?
    1. Copy the following scripts to a file.
    2. Change the value of $RootUrl to the folder name of your synced library.
    3.  Save the scripts file to a local folder. We name it Pull-files256.ps1 here as an example.

    ===================================================

                                   

       

        if ($root

        {

            return;                                                                                         

        }

     

        $items = Get-ChildItem

     

        Foreach ($item in $items)

        {

            [string]$itemName

            [string]$itemFullName

     

            # Folder

            if ($item.Attributes

            {

                RecurseDirectory $itemFullName;                 

            }

     

            # Documents

            else

            {

                # Print/Change Name for Document

                if (($itemName -ne "") -and ($itemName

                {

                    if ($itemFullName.Length

                    {

                        Add-Content c:\filepath.txt  "$itemFullName";

                    }

                }  

            }

        }

    }

     

     

    4. Open PowerShell by “Run as Administrator”.
    5. Change the directory to the folder where you put Pull-files256.ps1 to and run “.\Pull-files256.ps1”.

    6. Then open c:\filepath.txt to find the files whose length of path is greater than or equal to 256.
