andreas killer say:

e.g. 16-May-1862, returns fro me #value!

The issue is the date limit in Excel, resp. this is no valid date within Excel.

Make a new file and write my birthday 14-March-1964 into A1.

Widen column A by at least the length of the date on your screen. As you see the date inside the cell is right aligned.Write the formula =VALUE(A1) into B1. You'll see the number 23450.

Excel use numbers internally for dates and the min. date is the 1/1/1900 which value is 1. Means any date before this limit is not valid.

Now change the date in A1 to 14-March-1864. As you see the content of A1 is now left aligned, which means you have a string in A1, not a date. And in B1 you can see an error.
There is no way to set the date limit in Excel to a previous date as 1/1/1900. And that has fundamental reasons.

BTW, there are AddIn's out there that allows you to calculate with such dates, these are more or less correct and work only within certain limits.

But for most cases, there is an easy way without:

Our Gregorian calendar is regular and repeats every 400 years. Means you can shift the date 400 years into the future, make your calculations and shift the result 400 years back in the past, e.g.:

A1:  =DATE(1864+400,3,14)
A2:  =A1+23
A3:  =MONTH(A2)&"/"&DAY(A2)&"/"&YEAR(A2)-400

That is correct also for the weekdays.

But you should note that this is not valid for all mankind, especially not in the past, because we have different calendars on our planet and in our Christian region there is also a calendar switch:

From the ancient times till the day 4-Oct-1582 we have had the Julian calendar .
The next day after is the 15-Oct-1582 in the Gregorian calendar.
Means in our "time line" the dates between doesn't exists.

And before the Julian calendar there was the Roman calendar, the switch was in 47 BC.

And also our Gregorian calendar is not valid forever, because all this has to do how long the earth needs around the sun and how precisely we can measure that.

Andreas.

Excel formulas will not support dates less than 1-Jan-1900. For dates below that, you will have to use VBA function which will work for both dates less than and greater than 1-Jan-1900

1. Make a backup of your workbook. 
2. Open your workbook and ALT+F11
3. Locate your Workbook name in Project Explorer Window
4. Right click on your workbook name > Insert > Module 

5. Copy Paste the below code

In you cell, you can call this function a

=DateAddYears(A2,B2)

You can also put values in place of B2, =DateAddYears(A2,21)

The returned value will be in Text format if date is less than 1-Jan-1900

********** Start of Code **************

Function DateAddYears(Cell, Years)
    If DateValue(Cell) > DateValue("1-Jan-1900") Then
        DateAddYears = DateAdd("yyyy", Years, Cell)
        Else
        DateAddYears = CStr(DateAdd("yyyy", Years, Cell))
    End If
End Function

brookzhang_msft8047 say:

hi morgen,

there’s a contact public folder that may be your reference.

it’s possible to let the contact public folder show up in outlook desktop. please refer to the following steps:

  1. firstly, create a public folder.
  2. by default, there are no contact lists under public folders. so we need to create a contact list and then create some contact items in the contact list. permissions are required to create a contact list under public folder. here’re the detailed steps:
    1. login to exchange admin center with admin credentials.
    2. navigate to public folders > public folders > choose the public folder, in the right panel > folder permission> manage > add > add the one user as the folder owner and then save.
  3. use outlook for windows to create a contact list. in outlook client > folders, create a contact list under the public folder

once the contact list is created, every normal users in the organization can create a contact in the contact list.

      4. then in owa (outlook web app) > mail > favorites > right > add public folder to favorites > choose the contact you created > add to favorites.

  

     5. after that, the contact list will show up under people > other contacts and you can create contact lists in it.

     6. in outlook desktop, the contact public folder also shows up.

     7. for iphone, the pf contacts cannot be seen and edit. a present solution to meet your requirement is not available.

note: the #4 may be different from you side as it’s the feature of the new public folder picker for owa. see this thread for more information:

https://community.office365.com/en-us/f/158/p/422961/1063107#1063107
