Hi Karl,

I finally resolved my issue and can finally connect to the work exchange server from home after 4 months. Here is what I found:

The problem is not with email configuration. It works fine outside the home where we have Uverse. Connect to public wifi at an Airport or a hotel or at a friend's place with a different ISP and there is no issue. I tried the AT&T test tools but the connection failed every time. Finally I went into the router settings, checked a few configurations and then made the following changes:

1. Open Network and Sharing Center

2. Manage wireless networks

3. Select the Network at your home that is Uverse

4. Click on Adapter Properties

5. Make sure to check: Internet Protocol Version 4 (TCP/IPv4)

6. Now select it and click on the 'Properties' button

7. Ensure that 'Obtain an IP address automatically' option is checked by default

8. Now Select the option: Use the following DNS server addresses:

a. Preferred DNS Server: Type 8-8-8-8

b. Alternate DNS Server: Type 4-4-4-4

9. Click 'Ok'

Now you should be able to connect to your Exchange server. Mine did after 4 months of not working or giving the message: 'Trying to connect..'.

Note: Should you have problem connecting to your exchange server outside of home, you will have to go back to step 7 and select: Obtain DNS server address automatically.

SFC is a function built into Windows known as System File Checker.  A high level overview would be that it checks the integrity of certain system files and "fixes" any problems it finds.  Off the top of my head I don't really know much more about how it works.

To run it, you would go to Start, type cmd, and choose the command prompt.  Type sfc /scannow into that window and press enter.  It will count up to 100% and let you know what it finds when it is finished.
