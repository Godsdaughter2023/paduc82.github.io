Windows 10 Technical Preview, Issues downloading updates

Open Windows services (run -> services.msc)
Stop the Windows Update Service (You have to try couple of times to stop it successfully)
Using explorer or run command %windir% to access “Windows” folder
Locate “SoftwareDistribution” folder
Open the folder and delete all the content (Please note, this will possibility remove all the log entries for previously installed updates). Administrator privileges will be required for completing this act
Once the files are deleted, restart the windows update service.
Return to Windows update page, retry the update

That should fix the annoyance until next occurrence!

mr b frankiebee say:

Problem Windows 10 Update from Original 10041 Version fails before Finishing at 30% point

Computer Used was a HP Pavilion m7557c AMD Processor with 2 Gig Byte of Memory, Hard drive was 80 Gigabyte.
In Fast ring Update mode Update always failed at 30% when trying to update to Version 10049 and reverted to Version 10041.
Same Problem when trying to update to version 10061 except update never got past the download of version 10061.
There were postings that it might be due to insufficient hard drive space remaining. This did not seem to be the problem as the remaining hard drive space was about 40 gigabyte.

After some study I found that it was not the hard drive space remaining but the Usable memory space remaining that was the problem.
I found that with the 2 gigabyte of Ram and allowing windows to manage the Swap file size, windows was only allowing the swap file to be about 640 Kbyte which it seems is not enough memory for windows to process the update properly.

I changed allowing windows from managing the swap file size to a custom swap file size of minimum 2048 and a maximum 8192 swap file size and rebooted the computer.
I then restarted the update to 10061 versions.
Although at first the update did not seem that much faster it no longer stalled at the download process nor at the 30% point and windows rebooted several times and completed the 10061 update in a reasonable amount of time.
Therefore it seems that the memory problem is really the total memory both RAM and virtual memory [RAM] Windows has to work with.
Microsoft testing probably was done with more modern computers which more than likely had 8 Gig to 16 Gig of RAM memory installed in the computer.

Get watermark to appear on every page

I have created a document in Word 2013 that I want to issue in draft form, so want to put the "Draft" watermark on to every page of the document.  However, I suspect that because I have a "Different First Page" header set, I can only either get the watermark on to the first page, or into the body of the document, but not on to both (ie, not on to every page).  Could some-one help, please?
charles kenyon say:
A Watermark in Word is a graphic that is placed in a header. It follows the rules for the header except it also responds to the Insert Watermark commands. If you have different headers, inserting one in a single header will not put it in the others, in that section or in other sections that do not have linked headers. AFAIK this has been the same for all Ribbon versions of Word. Since Word 97, at least it has been a graphic in a headers.

To put a Watermark on every page of a multi-section document, it needs to be put in every header in that document. Each section can have up to three different headers.

If you go into the header containing the Watermark, you can select and copy that graphic. It can then be pasted into any header that does not contain it.

If you do this, and later change your mind, you can remove the Watermark from the entire document with the Remove Watermark command.

Cross-references to paragraph numbers, in Mail Merge: how can I - Microsoft Community

dave morehous say:

Cross-references to paragraph numbers, in Mail Merge: how can I stop them converting to static text during merge?

I write contracts, and I use cross-reference fields to refer to other numbered sections ("Section 7 of this Agreement"), etc. I've started using Mail Merge to run form contracts - or at least the first drafts. However, after the first draft is finished, and the other side is reviewing and revising, I need the cross-references to remain dynamic.  But when I "merge to new document", Word converts them to static text.  Is there a way to prevent this?
john mcghie say:
Hi Dave:

The answer to your question is "No."

*  A cross-reference is composed of an Anchor and a Target.  The Target is a a bookmark.  The name of the bookmark is normally automatically generated by Word, and it's a unique number.  The anchor points specifically at that number.

The problem is that if we were to transport the target to another document (which is what we are doing in a Merge) we cannot guarantee the number will be unique when it gets there.  You and I know it will be, but the software doesn't.  So it doesn't do it, in case it breaks something.

You could conceivably write a piece of VBA to work around this.  But it would be a lot of coding, and quite slow.  And you would have to run it on the merge source before you merged, and on the merge results afterwards.  Effectively, you would need to read each Anchor, retrieve the name of the bookmark, remove the cross-reference, insert a tag containing the name of the bookmark (as plain text).  Then you would have to find each bookmark, remove the bookmark and insert a tag containing its name (again, as plain text).

After the merge, you would have to go through the merge result document and find each tag, create a bookmark or an anchor as required, and enter the correct identifying name in each, then remove the tag.

Such code would not be for the faint-hearted: it's quite a serious coding exercise.  Without doing a fair bit of investigating, I am not sure whether Mac Word 2011 VBA will do this: I have a sneaking suspicion there are distinct limits to what it will enable you to do with fields.

And I wouldn't go anywhere near the Microsoft Developer Network on this one: the politest answer you are likely to get is "You're on a Mac, you're on your own!"  Sorry, but they're a bit Xenophobic over there :-)

If you could somehow avoid including the cross-references in the merge, this would be easy.  If the cross-references were in the main document, the fields would survive (because the main document is simply a copy of the source document).

However, I suspect what you are doing is merging CLAUSES in from your data source, and the cross-references are imbedded in the clauses.  Am I right?  Is there any chance of constructing the main document so that the cross-references are sitting outside the merge fields?

You can use the "Nested Fields" technique to send the bookmark name into a pre-built {Ref NameOfClause } field, as explained here: http://support.microsoft.com/kb/909132

So you would end up with a field similar to { Ref { MERGEFIELD "NameOfClause" } \* MERGEFORMAT }

But this means that you would have to know the names of all the bookmarks and create them all in advance.  Which means you would have to know the cross reference target in advance.  It's a lot of work, but it would function provided that you updated all fields after merging.  The document would also be a bit fragile: you would get broken references if the other end was not careful about protecting the bookmarks.

The issue is that you would have to know when you did the merge which clauses were going to survive the editing process.  And you can't know that.

Sorry, but I think for an affordable amount of effort/coding, the answer is "No".

Cheers
