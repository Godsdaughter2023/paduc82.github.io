I would not necessarily equate the loss of a system cached icon as definitely proving any sort of partial or full account deletion. All Skype accounts are Microsoft accounts whether you manually completed a migration or not. According to the document,That Microsoft Account doesn't exist ,inactive accounts are deleted as follows.

If the last time you signed in was... Your account would expire... 
Before June 2014 After 1 year of inactivity
June 2014–August 2015 After 2 years of inactivity
After August 2015 After 5 years of inactivity

You can check your old Skype name folder under %appdata%\Skype on the last machine you successfully logged into, to provide you with a possible date. Files like main.db, statistics.db, config.xml, keyval.db, msn.db, and rclib.db, will have these date/time stamps. Of course, if you signed successfully into something like Skype for web or Skype.com, after these dates, that date would be newer.

I believe access activity is supposed to be based on successful login, not failed login attempts. Beyond that, you are relying on the automated inactivity deletion policy to do the work for you. This process may or may not be flawed. There are variables like: not knowing what is considered activity by that account, the definite exact dates involved, if anyone else gained access to the account, if the completed migration process (~2016) reset the now 5 year inactivity counter, or if they choose whether or not to fully purge inactive accounts from the directory. scottgem say:

It the field is a number datatype, then you cannot enter leading zeros. Number datatypes should only be used when the value is going to be used in a calculation. That's why data like zip codes and SSNs should be text datatypes because they might include leading zeros.

So you have a couple of options here:

1) Use a text datatype. This will allow you to enter leading zeros or you can use code (like the format function, to add leading zeros.

2) Display the value using format. You can keep the field as a number but display it using:

=Format([fieldname],"000")
