rick rothstein, mvp - excel say:

Here is, perhaps, a more complete UDF function which will report the empty string if the referenced cell has no text in it, the font name if all character's in the cell have the same font name and a semi-colon delimited list of all the font names in use if the cell's text is composed of characters with different (multiple) font names...

Function FontName(Rng As Range) As String
  Dim X As Long, FN As String
  If Rng.Count > 1 Then Exit Function
  If Len(Rng.Value) = 0 Then Exit Function
  FontName = Rng.Font.Name & ""
  If Len(FontName) Then Exit Function
  For X = 1 To Len(Rng.Value)
    FN = Rng.Characters(X, 1).Font.Name
    If InStr(FontName, FN) = 0 Then FontName = FontName & FN & "; "
  Next
  If InStr(FontName, "; ") Then FontName = Left(FontName, Len(FontName) - 2)
End Function

john mcghie say:

Steven:

Before you go, that's a classic symptom of the "Renamed Server" problem.

If a server has been re-named or moved somewhere on the network, when Word opens a document that contains a reference to an attached template or printer that was on the old server, it will be unable to connect to it.  It will wait for a lengthy time-out before opening the document.

If that's the case, call up the document, and when you get it open, go to Developer tab (which you may have to reveal), click the Template button, choose Templates tab and select the Normal template, then click Attach.

OK your way out.  Now, print a page or two of the document to the local printer.

Now Save and Close the document.

Re-open the same document.  If that was the problem, it will open instantly this time.

So now you need to figure out what was moved, and notionally put it back (an alias is fine...).  If you install an alias to the Template share so the computers "think" they have their templates back, your problem will go away.

Maddening, isn't it!

Hope this helps
