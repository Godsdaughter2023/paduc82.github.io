If by "move" you mean to make a duplicate entry of the row's data on another sheet AND delete that row on the original sheet: then it cannot be done without VBA.

 

In point of fact, there's no way to automatically capture a 'snapshot in time' of a value in Excel without VBA.  So if your desire is to capture the snapshot of the information just after the update, it is not going to happen without VBA.

 

Scenario:  You have data updated in cell A2 of Sheet1. 

To "copy" it to another sheet without VBA you might simply have this formula on another sheet:

=Sheet1!A2

But the instant that the value in A2 of Sheet1 changes through updating, so does its copy on the other sheet.

 

So the only way to capture the value at a particular point in time without VBA is to do it by hand (copy --> paste special with 'value' selected).

bkreft say:

assign a running count in a query

I have a Part Number record that may have one or more alternate part number(s). If the record has an alternate Part Number there will be a value in a field called APN (alternate part number) which links all related part numbers.

 

I am attempting to number each Part Number record within the APN group so I can list in a report or show on a form all related part numbers. I have already determined through a query how many alternate numbers exist for a given part number. Right now the maximum any part number alternates is four (4), most have two (2) or three (3).

 

By numbering each record (starting with one (1) for each group) within the group I will be able to write a query that will display all associated information (this will help choose the best vendor and pricing for a given part number) on one line.

 

I am not sure how to assign a running count value to this grouping, I suspect a sub query but am not sure as to how to begin, any suggestions?

 

Bruce
ken sheridan say: Firstly I'd recommend against using the DCount function unless you need the query to be updatable, i.e. as the RecordSource of a form in which you need to be able to edit the data.  A join is the most efficient method.  To use the DCount function however, the expression needs to count those rows where the APN value is the same as the current row's  and the part number value is less than or equal to that of the current row.  The data types of the columns are also important as these determine whether the values need to be delimited with literal quotes characters or not.  So if we assume that both columns are of text data type the expression would be:

DCount("*", "[Part Header]", "[APN] = """ & [APN] & """ And [PartNo] <= """ & [PartNo] & """")

If both are of number data type the literal quotes are unnecessary, so it would be:

DCount("*", "[Part Header]", "[APN] = " & [APN] & " And [PartNo] <= " & [PartNo])

The query needs to be ordered by APN then PartNo, so would be like this:

SELECT APN, PartNo,
DCount("*", "[Part Header]", "[APN] = """ & [APN] & """ And [PartNo] <= """ & [PartNo] & """")
AS RowNumber
FROM [Part Header]
ORDER BY APN, PartNo;

Using a join the query does the same thing, but uses two instances of the table, aliased as PH1 and PH2 below, rather than looking up the table independently:

SELECT COUNT(*) AS RowNumber, PH1.APN, PH1.PartNo
FROM [Part Header] AS PH1 INNER JOIN [Part Header] AS PH2
ON (PH2.APN = PH1.APN)
AND (PH2.PartNo <= PH1.PartNo)
GROUP BY PH1.APN, PH1.PartNo;

This is the most efficient solution, but the query is not updatable.  Note that in this case the data types of the columns are immaterial.

hansv mvp say:

Activate the footer, for example by double-clicking on it, then place the insertion point where you want the page number.

On the Design tab of the ribbon, in the Header & Footer group, click Page Number > Current Position > Plain Number.

Alternatively, press Ctrl+F9 to insert a pair of field brackets { }, type the word PAGE between the brackets and press F9 to hide the field code and display the result.

suzanne s. barnhill say:

When you use a Page Number building block from the gallery, that building block (as you have seen) replaces the existing header or footer, whether it is a building block or a custom creation. One solution, as Hans has indicated, is to use the Current Position setting instead of one of the gallery page numbers. Another is to insert a PAGE field either directly (using Hans's instructions) or with Insert | Text | Quick Parts | Field.

A third alternative for inserting a plain page number is to place the insertion point where you want the number and press Alt+Shift+P, which inserts a PAGE field. You can then format the page number as desired.
