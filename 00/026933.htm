I did a test of the suggestion that saving the file would clear out the Undo cache and speed things up.  The code below allowed me to run my test case (build 50 slides, each with 250 random shapes) with varying periodic saves of the file during the run.  I first ran the code without saving the file at all - this was the slowest case.  The amazing thing is, the fastest case is the one where the file is saved after creating each slide.  There was a factor of almost 20 speedup in total time required between the slowest case (don't save at all) and the fastest case (save after every slide is created).  I guess the file save action is pretty fast for subsequent saves, because it does not seem to increase the time per slide at all.

PowerPoint Code (paste into a blank PPT file VBA module):

Option Explicit

Sub Random_Shapes()
    Dim i As Long, j As Long, k As Long
    Dim tL As Single, tT As Single, tW As Single, tH As Single
    Dim theSeed As Long
    Dim tVal1 As Double, tVal2 As Double
    Dim tShape As Shape
    Dim tSlide As Slide
    Dim saveInterval As Integer
'
    saveInterval = 1
'
    Randomize (18392084)
    For k = 1 To 50    ' Create 50 slides total
        Set tSlide = ActivePresentation.Slides.AddSlide(ActivePresentation.Slides.Count + 1, ActivePresentation.Slides(1).CustomLayout)
        tVal1 = Timer
        For j = 1 To 250    ' Each slide will have 250 randomized shapes
            i = Round(Rnd() * 9 + 1, 0)
            tL = Rnd() * 700#
            tT = Rnd() * 550#
            tW = Rnd() * 250#
            tH = Rnd() * 250#
            Select Case i
            Case 1
                Set tShape = tSlide.Shapes.AddShape(msoShape16pointStar, tL, tT, tW, tH)
            Case 2
                Set tShape = tSlide.Shapes.AddShape(msoShapeBalloon, tL, tT, tW, tH)
            Case 3
                Set tShape = tSlide.Shapes.AddShape(msoShapeCloud, tL, tT, tW, tH)
            Case 4
                Set tShape = tSlide.Shapes.AddShape(msoShapeDecagon, tL, tT, tW, tH)
            Case 5
                Set tShape = tSlide.Shapes.AddShape(msoShapeDoubleWave, tL, tT, tW, tH)
            Case 6
                Set tShape = tSlide.Shapes.AddShape(msoShapeExplosion2, tL, tT, tW, tH)
            Case 7
                Set tShape = tSlide.Shapes.AddShape(msoShapeHeart, tL, tT, tW, tH)
            Case 8
                Set tShape = tSlide.Shapes.AddShape(msoShapeFlowchartPunchedTape, tL, tT, tW, tH)
            Case 9
                Set tShape = tSlide.Shapes.AddShape(msoShapeIsoscelesTriangle, tL, tT, tW, tH)
            Case 10
                Set tShape = tSlide.Shapes.AddShape(msoShapeSwooshArrow, tL, tT, tW, tH)
            Case Else
                Set tShape = tSlide.Shapes.AddShape(msoShapeSun, tL, tT, tW, tH)
            End Select
            With tShape
                .Fill.ForeColor.RGB = RGB(123, 234, 45)
                .Line.Weight = 1.25
                .Line.ForeColor.RGB = RGB(45, 234, 123)
            End With
            Set tShape = Nothing
        Next j
        tVal2 = Timer - tVal1
        Debug.Print "Slide " & k & " took " & tVal2 & " seconds."
        If (k Mod saveInterval = 0) Then
            ActivePresentation.Save
        End If
    Next k
End Sub

steve rindsberg say:

PowerPoint's slide size (page setup) and the printer (or PDF software) page size are two different things.

You'll need to create a custom printer page size that matches your PowerPoint slide size  OR adjust your PowerPoint page size so that it's the same as, or at least proportional to, the printer's page size, which is probably letter (8.5x11) or A4, depending on where you live and your computer's settings.

Once the two match up, choose the "Scale to fit paper" option in PowerPoint's Prnit dialog box, pick any other options you like and print.

When you do that, PowerPoint will print your slides as large as it can w/o distortion on the printer driver's current page size.  If the two page sizes (PPT's and the printer's) match, your border will be preserved.

That should do ya, I think.

Here is a Macro to overcome the problem with propagation of mail merge field
fields with a label type mail merge when using a Tablet PC and some other
Computers that have Tablet PC Functionality available:
 
Sub MailMergePropagateLabel()
Dim atable As Table
Dim i As Long, j As Long
Dim source As Cell, target As Cell
Dim myrange As Range
Set atable = ActiveDocument.Tables(1)
Set source = atable.Cell(1, 1)
Set myrange = source.Range
myrange.Collapse wdCollapseStart
ActiveDocument.Fields.Add Range:=myrange, Text:="NEXT", _
PreserveFormatting:=False
source.Range.Copy
For j = 2 To atable.Columns.Count
Set target = atable.Cell(1, j)
If target.Range.Fields.Count > 0 Then
target.Range.Paste
End If
Next j
For i = 2 To atable.Rows.Count
For j = 1 To atable.Columns.Count
Set target = atable.Cell(i, j)
If target.Range.Fields.Count > 0 Then
target.Range.Paste
End If
Next j
Next i
atable.Cell(1, 1).Range.Fields(1).Delete
End Sub
 
It is suggested that you put this code into a template that you save in the
Word Startup folder so that it becomes an add-in and will be available
whenever you need it. As the macro has the same name as the Word Command
that is supposed to do the job, this macro will run when the Update Labels
button is used when setting up the mail merge label main document.
