A routine Windows Defender Full Scan picked up a Trojan.  I've deleted it, but I want to check it is not getting reinstalled.  So I am running a custom scan on the disc area where it appeared.  However, WD appears to be scanning a much wider area.  According to Explorer, the directory I am scanning contains about 8000 files.  Running a custom scan on that folder, WD reports that it has scanned 4 million files, and it's still going, with a steadily increasing estimate of how much time it will need (it's been around the 20 minute mark, rising gently, for the past 3 hours).  I realise that WD might be looking inside zip files, but this looks an impossibly large expansion: a full scan of my system (which tends to take about 24 hours) only finds about 12 million files.  Can anyone explain what is going on, or is this simply a bug in WD?  Thanks for any suggestions.
rob koch say:

There are other types of packed, compressed archive files which contain many individual files other than zip, including ISO images, Windows Installer (.msi) file, or Windows Installer Patch (.msp), as well as 3rd-party archives like RAR.

If you're scanning a folder that contains any of these, such as one you use for downloaded app installers, it's highly likely that the ratio of files listed by Windows Explorer, which only counts the archives as a single file, versus those scanned by Defender will be greatly different.

This is because both the Full and Custom Defender scans assume that you wish to scan everything within the storage area(s) chosen, while a Quick scan only looks in the operating system areas known to have been affected by truly active malware.
