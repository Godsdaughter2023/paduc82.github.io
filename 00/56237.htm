slance310 say:

Hi,

Are the drives over 2TB Boot drives ?

My understanding is that Windows will not allow a partition over 2TB without using GPT... You can use the drive as a MBR, but nothing over 2TB will be recognized...

And that is for DATA drives... boot drives are even more strict, and can vary a bit depending on the machines BIOS capabilities...

There are utilities that will enable a over 2TB drive to be utilized in Legacy Bios, but I'm certain that won't work with a boot drive...

As far as your current drive... If it was working before, it may be that the BIOS setting got changed somehow... You would know that better than I...

If the 4TB has your windows installation on it, you have a bit of a problem...

UEFI requires a EFI boot partition as well as the OS partition... If the drive has never been UEFI it won't have one...

One thing you might try is to make a image of the drive or the OS partition, and possibly Recovery partition... EFI as well if it exists...

Then delete what is on the 4TB drive, re-initialize it as a GPT, set your BIOS to UEFI, restore the image to the 4TB drive...

You will then probably need to do some Automatic Startup Repairs, Boot Repairs, and\or etc...

Let me know more of your setup and I may be able to help more...

Hi,

The Computer Name you are seeing was assigned By Windows 10... probably...

The thing is that this type of name is usually only applied when doing a clean install...

You shouldn't be seeing it if this was cloned from another drive.... Unless of course it happened on the original drive before the cloning...

You will need to think about your upgrades\installations to try to determine when\how this happened...

One thing I see is that the working computer has two "Recovery" partitions... One is not listed as Recovery, but is 450mb and Hidden... Most likely some type of upgrade process created the 2nd.. A lot of times just updating to a newer build will create a additional Recovery for some reason...

I don't have a clue as to what might have created that...

You may want to delete it... If your boot repair needs to create a new partition for some reason, it will not be possible... You have four partitions, and that is the max on a MBR\Legacy drive... at least as far as primary partitions go...

Before you do something like that though... Try this sequence of Commands...

You mentioned something like this, but I'm not sure you did the entire sequence...

I didn't mention it previously because these command are not effective on a GPT  drive, which I was thinking you must have for 4TB... Do all three in the sequence listed...

BOOTREC /FIXMBR, and then press ENTER.

BOOTREC /FIXBOOT, and then press ENTER.
