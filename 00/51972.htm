cmdrkeene say:

If the item that is running is coming from the "HKEY Local Machine" (global) registry branch, or from the /All Users "Startup" folder, then it can't be changed from the user.  This is similar to desktop icons for example.  If an administrator installed Adobe Reader, and the shortcut icon got copied to the /All Users desktop folder, then no individual user would ever be able to delete that icon.

mouneshwar r say: Hi code8000,

I would suggest you to re-install the WAN miniport driver by following the steps below.

1. Go to device manager and select "Show Hidden Devices" under the
View menu.

2. Expand Network adapters. You should see the WAN Miniport (PPTP)

3. Try right clicking on it and uninstalling it. More than likely it
won't allow you to. To get around this update the driver to something
"uninstallable" like the MAC Bridge. To do this right click on it and
select update driver. Choose no to look on windows update. Then choose
to specify a location. Next choose dont search. Next unselect show
compatible hardware and find the MAC Bridge under Microsoft. Force it
to install it. After thats done you should be able to uninstall your
new 'MAC Bridge Miniport."

4. Now that it's gone, lets reinstall it. Now get the devcon.exe
package from the link below http://support.microsoft.com/kb/311272 and extract
it desktop or c:drive.

5. Go to the command prompt (Run it in administrator mode) / console (cmd.exe) and get to the folder with devcon.exe in it.

6. Run this command: devcon.exe install c:\windows\inf\netrasa.inf
MS_PptpMiniport

7. Go back to the device manager. You should see the WAN Miniport
(PPTP) back.

This should do it.

Troubleshoot problems finding wireless networks
http://windows.microsoft.com/en-US/windows-vista/Troubleshoot-problems-finding-wireless-networks

Post your results back.

It sounds like you are pressing 'R' to Repair a Windows XP installation...  after booting on the XP installation CD and that will attempt to load the XP Recovery Console and that is not how you perform a Repair Install (in spite of what the message says).  You can also resolve the issue of not being able to get past the password prompt in Recovery Console, but one does not perform a Repair install from the Recovery Console.

To perform a Repair install you need to instead press 'Enter' to 'set up Windows XP now' and then proceed. 

Performing a Repair install of XP is a pretty drastic measure - the next to the last thing before a complete reinstall of XP and only after all other troubleshooting methods have been exhausted (unless you are just practicing) so unless there is a really good reason to do a Repair install, you might want to consider other alternatives.

And you will not be doing a Repair install if the genuine bootable XP installation CD you have is not the same Service Pack as your installed Service Pack.  i.e.  If you have Service Pack 3 installed and your CD is less than SP3, you will not be doing a Repair install and will see a message like this:

Setup cannot continue because the version of Windows on your computer is newer than the version on the CD.

The fact is that on the planet called earth most people don't have a matching genuine bootable XP installation CD or they have no genuine bootable XP installation CD at all.  All the more reason not to suggest or consider a Repair install.

Do you have your XP Product Key handy?  You are going to need that too.

Then even if it were to work, you would have to reinstall all your drivers (probably), get back on the Internet and download and reinstall all the perhaps hundreds of Microsoft updates that have come out since you first installed SP3 (definitely) and probably reintstall all the third party programs you have installed.  You have all that stuff ready, right?

That all sounds like too much work to me.

If you want to tell us what the problem is that you think a Repair install will resolve, perhaps you can just fix what you have.
