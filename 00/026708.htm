mikew333 say:

Allowing a user to add a row to a table when completing a form through MS Word

I am creating a questionnaire, sort of profile form, for my company's distributors to complete. One of the questions asks them to provide a list of the locations they have branch offices in. I do not want to have 20 blank rows to give them the option to fill out (they could have 2 or they could have 32). 

So I am wondering if there is a way they could click a button (possibly similar to the + symbol that I see when creating a table), or any other way? Thank you.

jay freedman say:

When you say it's a form, I'll guess that you have applied forms protection to the document or template through the Restrict Editing pane. There is no way to add rows to a table in a protected document while it's protected.

The only solution -- other than requiring the user to unprotect, edit, and reprotect the document -- is to provide a macro that performs that same sequence while not giving the user direct control.

The way the macro gathers the information to put into the table depends on several factors, but the general answer is to display a userform (a custom dialog box) for data entry. If the possible locations consist of a known list, the userform can contain a list box for the user to click their selection(s). If the entries aren't known in advance, then a text box or combo box can be used instead. The macro code (part of the userform) for the OK button can count the number of locations the user has selected/entered, create that number of rows in the table, and insert the text in the rows.

If the document isn't a protected form, then please give more details about how it's set up.

charles kenyon say:
I would urge you to use Content Controls and one of the other protection methods. It will give you a lot more flexibility unless you need compatibility with Word 2003 or below or need legacy form fields for some other reason.
