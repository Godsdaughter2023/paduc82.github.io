tom van stiphout say:

Daniel's suggestion works. Here is an elegant alternative using regular expressions:

Public Function ConvertPhoneToNewFormat(ByVal strPhone As String) As String
    'Requires reference to Microsoft VBScript Regular Expressions 5.5
    Dim re As New VBScript_RegExp_55.RegExp
    re.Pattern = "(\d{3})-(\d{3})-(\d{4})"
    ConvertPhoneToNewFormat = re.Replace(strPhone, "($1) $2-$3")
    Set re = Nothing
End Function

The pattern says: Group 1 is a collection of three digits, then a dash, then group 2 which is a collection of 3 digits, then a dash, then group 3 which is a collection of 4 digits.

The Replace instruction says to replace this with an open_parenthesis, group 1, a close_parenthesis, a space, group 2, a dash, and group 3.

If you make "re" a global object, the function is >50% faster than Daniel's.

daniel pineault say:

Make a backup of your database, just in case.  Better safe than sorry!

Paste the following function into a module

Public Function StripAllChars(strString As String) As String
'Strips the alphabetical characters from a string to leave only a numeric string
Dim lngCtr As Long
Dim intChar As Integer

    For lngCtr = 1 To Len(strString)
        intChar = Asc(Mid(strString, lngCtr, 1))
        If intChar >= 48 And intChar <= 57 Then
            StripAllChars = StripAllChars & Chr(intChar)
        End If
    Next lngCtr
End Function

Then create an update query with only the phone number field.  In the Update To input somthing like: Format(StripAllChars([Your Phone Number Field Name]),"(@@@) @@@-@@@@") and then run the query.  Everything should be fixed.
