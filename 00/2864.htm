how can I force "export to PDF" to embed the my non-standard - Microsoft Community

matthias265 say:

how can I force "export to PDF" to embed the my non-standard font in the PDF,

hello

how can i force the "export to PDF" or "save as PDF" to embed the font in the file or create bitmaps so i can share it to systems that don't have the font installed. this is an issue as clients won't have the font on their system

john korchok say:

Turning the font to bitmaps to save will make your PDFs larger, not smaller. In fact, that could be what is happening to make your files large. When you Save As>PDF, click on the Options button in the Save As dialog. At the bottom you'll see Bitmap text when fonts may not be embedded. If that is checked and the font you're using has permissions set to forbid embedding (many fonts do), then the text is being turned to bitmaps and you get a large file.

You can test for this if you have a copy of Adobe Illustrator or Acrobat. In Illustrator, you can open the PDF to see if you can edit text in the non-standard font. If you can't it's been bitmapped. In Acrobat, if you open the PDF and check File>Properties>Fonts and your font is not listed, that means it wasn't embedded and the fonts were bitmapped. Ubuntu has a flexible license that looks like it may allow you to send the fonts with the PDF. You would have to instruct users to install it before viewing the PDF.

If bitmapping is not the issue and the fonts are being embedded, you could get a better result using Adobe Acrobat. It has a font control panel that allows you to subset fonts, which embeds only the characters actually used in the file. This can reduce file size significantly.

Not all fonts are suitable for all purposes. Using the standard fonts included with Office is almost always easier.

zvido1 say: Okay, I think I figured out the problem.

After some further digging, I realized the fonts that were giving me problems were .ttf fonts that were file version 1.00 fonts (I guess that is what they call type1 fonts which Office no longer supports).

What happened was, I have a collection of fonts that I have saved over the years and whenever I upgrade to a new computer or do a new Windows installation I copy and install all those old fonts to the new system.

The problem was that many of these fonts were type 1 fonts and I assume they were erasing newer versions of the same font. For example, the font David that comes with Windows 7 is version 5.02 (see here http://www.microsoft.com/typography/fonts/font.aspx?FMID=1884) yet I had version 1.00 installed.

The solution was to reinstall all the original Windows 7 fonts and delete the original versions they replaced. (Here is an easy way to do it - http://www.withsteps.com/404/how-i-restored-my-windows-7-default-fonts.html).
