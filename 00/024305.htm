scottgem say:
Hi Blue Angel, I'm an independent advisor and will try to help

What you are referring to is called synchronized or cascading comboboxes. It is well documented with lots of support articles. It appears your tables are set up OK. So your RowSources should look like this:
The Builder combo should be
SELECT BuilderID, Builder
FROM Builders
ORDER BY Builder;

For the Subdivision it should be:
SELECT SubdivisionID, Subdivision
FROM Subdivisions
WHERE BuiderId = Forms!formname!cboBuilder
ORDER BY Subdivision

For floorplans it should be:
SELECT FloorplanID, Floorplan
FROM Floorplans
WHERE SubdivionId = Forms!formname!cboSubdivision
ORDER BY Floorplan

Then in the After Update event of Builders and Subdivision you want to requery the next combo.

Note that BuilderID and SubdivisionID are superfluous in your Jobs table. Since (and I'm assuming this) a floorplan is only associated with one subdivision and a subdivision with one builder then if you know the floorplan you can find the Builder and subdivision by your relations.

ken sheridan say: You need to model the many-to-many relationship type between floorplans and subdivisions by a further table which resolves it into two one-to-many relationship types.  The relationship type between subdivisions and builders and is one-to-many, so is modelled by a BuilderID foreign key in Subdivisions, as you have it.  This means that BuilderID is functionally determined by SubdivisionID, so the Jobs table should not have a BuilderID.

Once you have selected a subdivision in the first combo box, bound to SubdivisionID in Jobs, the next combo box should be restricted to the subset of floor plans which relate to the subdivision determined by the selected subdivision in the new table which models the relationship type between floorplans and subdivisions, i.e. its RowSource query should be a query which joins Builders, the new table, and FloorPlans, and is restricted on SubdivisionID by referencing the first combo box as a parameter.  The combo box should be bound to FloorPlanID in Jobs, and show the FloorPlan and Builder values in the visible columns.  The builder can be shown in an unbound text box once a selection is made by referencing the relevant column of the second combo box in its ControlSource property.  Remember that the Column property is zero-based, so the first column is Column(0), the second Column(1) and so on.

If I am understanding your question correctly, I believe you are wanting to take a group of emails for a specific company and have them added to one distribution list saved as the companies name.

 

I was attempting the same thing and possibly came up with a solution.

 

I am making the assumption that you have the email addresses in a program such as Word or Excel.  Make sure if they are in a table, you change the Table to Text prior to following the below instructions.

 

  • First copy the email addresses you want included in the same distribution list.
  • Go to your Outlook contacts and right-click in the white area next to your contact cards/names, then select New Contact Group.
  • Select Add Member, then select From Outlook Contacts
  • A dialog box will come up with your current contacts - Do not choose any.
  • Place your cursor in the field at the bottom , next to the Members button and paste the email addresses in the field.  Select OK. It doesn't matter if you have any blank rows between the email addresses.  Outlook will clean these up.
  • In the Name field, enter the name you would like to call your distribution list and select the Save & Close button.

I hope this helps!
