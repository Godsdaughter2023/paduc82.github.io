joeu2004 say:

This is a common anomaly of 64-bit binary floating-point, which Excel uses internally to represent numerical values.  Most decimal fractions cannot be represented exactly; and the binary approximation of a decimal fraction might differ depending on the magnitude of the integer part.  So binary arithmetic often results in infinitesimal differences with the decimal arithmetic that we might do manually.

Explicitly rounding the original data is not sufficient.  It is prudent to do that as well, if the "original data" are formulas.  But you do not need to do that if the "original data" are entered as constants, not formulas.

But we must (also?) explicitly round the calculation in the Pivot Table.

Unfortunately, that is normally calculated automated (and internally?) by Excel, which does not know that the calculation should be rounded.

I am not a PT person.  But I believe there is a way to replace the automatic PT calculation with a formula of your own.  You would explicitly round appropriately in that formula.

"Appropriately" means:  if you expect a calculation to be accurate to n decimal places, explicitly round to

For example, IF(10.01-10 = 0.01, TRUE) returns FALSE(!).  Fix:  IF(ROUND(10.01-10,2) = 0.01, FALSE.

joeu2004 say:

PS....  The difference between SUM in column D and the totals in column B and C (about -1.82E-12) is due to a "trick" that Excel plays with arithmetic

Presumably, =SUM(B2:B5) displays 0.00E+00, which is exact zero.  So does =B2+B3+B4+B5.  But note that =B2+B3+B4+B5-0 (!) displays -1.82E-12 when formatted as General or Scientific.

Of course, adding or subtracting zero -- even putting parentheses around the expression, to wit =(B2+B3+B4+B5) -- should not make any difference mathematically.

But they do.  The difference arises because of a dubious heuristic that Excel applies inconsistently, which is poorly and

(You might get the impression that I do not think highly of this "trick". ;->)

Essentially, when the last operation of a formula (not an expression in a formula) is subtraction or addition of oppositely signed operands, and Excel deems the operands to be "close enough", Excel arbitrarily replaces the actual arithmetic result with exact zero.

Most Excel functions do not implement that heuristic.  For example, SUMPRODUCT(B2:B5) returns -1.82E-12.  And the SUM function is inconsistent.  For example, =SUM(--B2,--B3,--B4,--B5) returns -1.82E-12.
