When you run an .exe file on a Windows XP, Windows Vista or Windows 7-based computer, the file may start a different program
http://support.microsoft.com/kb/950505

 

Method 2:
I would suggest you to reset the icon cache and then if that does not help rebuild icon cache.

Step 1: Reset Icon cache
a. Right click on empty area of the desktop and click on Personalize.
b. Click on the Display Properties link.
c. Under Colors, select Medium (16-bit) and click on Apply.
NOTE: If you already have Medium (16-bit) selected, then select Highest (32-bit) instead. Sometimes you will need to do a restart first, and then proceed to step.
d. Try both ways.
Click yes to confirm the new settings.
e. Now, select Highest (32-bit) and click on OK.

NOTE: If you already have Highest (32-bit) selected, then select Medium (16-bit) instead.

Click on Yes to confirm the new settings.

Step 2: Rebuild icon cache
a. Click Start, then control panel and in the search box write Folder options.

b. Now click Folder Options to open and select (dot) Show hidden files and folders.
c. Open a Windows Explorer window.
d. Go to C:\Users\ (User Name)\AppData\Local
e. Right click on IconCache.db and click on Delete.
f. Click on Yes to confirm the deletion.

NOTE: This deletes the file to the Recycle Bin. It is safe to empty the Recycle Bin when finished.

g. Close the window.
h. Empty the Recycle Bin.
i. Restart the computer.
j. When you go back, you will notice the Size of the IconCache.db file is smaller, and the Date Modified is now the current date.

NOTE: If for some reason IconCache.db is not there or the size has not changed much then you may just restart the computer again. You may need to restart a couple of times in some cases.

k. The icon cache will get rebuilt.

 

Hope it helps. For any further Windows related assistance, feel free to contact us and we will be happy to help.

dena b say:

Hi Dressi,

Since Startup Repair and System Restore did not resolve the issue, try the following steps:

1.      Boot from the Windows 7 DVD, select your keyboard and language options, select Repair your computer, and then select the Command Prompt. 

2.       From the Command Prompt run the following command:
Dism /image:c:\ /get-packages

This will produce a list of installed updates and packages.  They will look like this:

Package Identity: Package_for_KB978262~31bf3856ad364e35~x86~~6.1.1.0
State: Installed
Release Type : Security Update
Install Time : 2/11/2010 1:01 PM

3.      Find the most recent one and then make a note of the package identity.

4.      Run:

dism /image:c:\ /remove-package /PackageName:Package_for_KB978262~31bf3856ad364e35~x86~~6.1.1.0 (name of update from previous step)

You will get a status of the removal until it is complete. 

5.    Then reboot your PC.

 You will want to start with the most recent and remove them one at a time to find the problem update.

Repeatable: I would suggest a full system image BEFORE you do anything at all though.  Safety in backups!

  

Fair warning: 

  • All system restore points are deleted.
  • All event logs and personalizations are removed.

Using an administrative (elevated) command prompt . . . 
Use this command:
%windir%\System32\Sysprep\sysprep.exe
  

Choose "Enter System Out-of-Box Experience (OOBE)".

Check the "Generalize" box.

Choose "Shutdown".

  

(**NOTE** -- you can just open the elevated command prompt and use this command:
   %windir%\system32\sysprep\sysprep.exe /generalize /oobe /shutdown
 . . . which does all the selection I walked you thru the GUI interface; I suggest doing the GUI method.)

  

Let it do its thing and shut down.  Change motherboard, etc.  Start it up.

  

That method has been the same for a long time in the Windows world.  You will have to create a new user (make temporary one) at first start up, but just make a temporary one (e. TEMPUSER) and once you get fully in, reboot and log in with your other users to make sure they are all good and ... maybe enable the built-in administrator and get rid of the temporary user.

  

Download and install all the proper device drivers from the manufacturers web pages for your OS version, edition and architecture and ... You should be good to go.

  

Repeatable: I would suggest a full system image BEFORE you do anything at all though.  Safety in backups!
