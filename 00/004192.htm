grschinon say: Setting up the laptop as a hotspot isn't going to help. You also need both a client and a server "talking" the same protocol for exchange of files to take place. That protocol can be SMB, FTP, SCP/SFTP etc.

Not only that but the laptop doesn't need to be set up as a hotspot. If you already have a wireless router or access point then the laptop will be visible to the N82 if all devices are in infrastructure mode. Setting the laptop up with a static IP address will help.

The obvious solution would have been to set up filesharing on the computer and install SymSMB on the N82, but SymSMB seems to have disappeared off the face of the earth after some kind of disagreement between Nokia and Telexy (the publisher of SymSMB).

Next best would be to install SICFTP! on the N82, but that then involves setting up an FTP server on the laptop. Good luck doing that in a secure manner under Windows.

So, I stand by what I wrote. Unless you're an experienced sysadmin who knows how to lock down an FTP server correctly and securely, you'll save yourself a lot of trouble by using the USB connection. There is no "step by step" guide to doing this, it takes years of experience and a knowledge of concepts such as chroot jails and other measures to curtail attacks.

  2. Type Troubleshooter and click on Troubleshooting; click on the Settings tab below the search box.

  4. Click on the Windows update troubleshooter and run it.

  3. If not, right click on the service and click start to start the service.
