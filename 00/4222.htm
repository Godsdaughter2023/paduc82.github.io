Why? Are there multiple default fonts sections in Word? How do I fix this?

suzanne s. barnhill say:

Are you saying that the font of the heading changes when you press Enter or that the font of the next paragraph is TNR rather than Calibri?

If the latter, then, as Guessed pointed out, the issue is that the "Style for following paragraph" of all the built-in heading styles is Normal, so when you press Enter at the end of a Heading 1 or Heading 2, the next paragraph is Normal.

Now the question is, why is Normal using TNR instead of Calibri? Possibly the font has been explicitly defined for the Normal style separately from the document defaults.

  1. Press Alt+Ctrl+Shift+S to open the Styles pane.
  2. Right-click on Normal and choose Modify...
  3. In the Modify Style dialog, you should see "Calibri (Body)" for the font. Don't change it there.
  4. Instead, click Format, then Font...
  5. In the Font dialog, scroll the Font list to the top and choose +Body, which will ensure that the Normal style uses the default Body font. This can change depending on what theme is applied, but when you are using the Office theme, the fonts will be Cambria for headings and Calibri for body.

This message usually means any Identity saved with a newer version will not open in an earlier version. We see this a lot when users do a re-install of Office then fail to upgrade.

There was a database upgrade in 14.1.0 and again in 14.1.3. Until you update the Identity will not open. It's OK to stop at 14.1.3 but you'll have to download the file from Microsoft downloads since it's not available under Autoupdate now. 14.1.4 has some issues so stopping at 14.1.3 might not be a bad idea if you do a re-install.

IMHO 4GB RAM with Lion is not enough. I have 8GB and wish I had more and I don't do any intensive graphics works.

Open Activity Monitor in Applications/Utilities and under Memory, check for Page ins/outs as well as swap files. This will help determine if adding more RAM would help. this link provides more info on how to do this.

Problems from Insufficient RAM and Free Hard Disk Space

The other issue is the number of messages in folders. I'm assuming you have Exchange.

Exchange Basics

The Inbox syncs more frequently than any other and should be kept as clean as possible. (this is the number one cause of problems with sync). Do NOT make subfolders under the Inbox. When Outlook is connecting, it will slow things down. I've seen typing show o n e   l e t t e r at a time while a schedule is running.

Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.
