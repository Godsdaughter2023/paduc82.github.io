gerry c j cornell say:

Here are some notes about Event Viewer Reports which may help. When you have a lot of errors you need to concentrate on system errors and warnings, even if it is applications that are giving you grief. Sorting system problems first can make resolving application problems easier. Note the time the computer is last booted and deal with those at the beginning of the boot first. Correcting the earlier errors can resolve later ones.

1. Normally when an error occurs on your computer looking in Event Viewer should be your starting point for finding a solution. Most system related errors are logged and getting an exact copy of the relevant report is important. Unfortunately understanding the reports is not easy and most computer users need help with their interpretation. I have more to say later on interpretation.

2. Event Viewer comprises three main Windows logs. These are Application, Security and System. For troubleshooting purposes System is by far the most important.

3. To access the System log select Start, Control Panel, Administrative Tools, Event Viewer, from the list in the left side of the window select Windows Logs and System. Place the cursor on System, right click and select Filter Current Log. Check the box before Error and click on OK and you see only Error reports. Click on the Date and Time Column Header to sort. You may need to click a second time to see the latest Report at the top.

4. A tip for posting copies of Error Reports! Run Event Viewer and double click on the error you want to copy. Click on the Copy button on the General tab to place a copy on your Clipboard and close Event Viewer. Now start your message and paste into the body of the message. Make sure this is the first paste after exiting from Event Viewer.

5. There are three types of Report, being Information, Warning and Error reports. In most situations it is Error Reports that offer the best information but occasionally Warning Reports provide useful clues.

6. All reports have date and time stamps and when troubleshooting it is important to concentrate on more recent reports. Study reports since the point when the computer was last booted and then check whether a similar report appeared in the previous session. If errors do not repeat investigation as to why they occurred is wasted effort.

7. Within individual reports the more important information is Event ID and Source as these help when looking for help on the internet. The description is equally important and copying the exact text for use as the search criteria greatly helps getting better results when using Google. Do not paraphrase descriptions when asking others for help.

Hi Jabber,

Thank you for writing us.

I understand that you are system is getting crashed and you are getting error message that boot manager is corrupt or missing.

This issue is caused if the system files become corrupt.

If the boot manager becomes corrupt or points to an operating system that no longer exists, you must use the host operating system to repair and rebuild your boot manager, entering simple commands in the command prompt.

Please follow the steps below and check the issue.

  1. Insert your Windows installation CD/DVD into a drive on your computer and restart the computer.
  2. Press any key on your keyboard when you receive a prompt on your screen telling you to do so.
  3. Click Next when you get to the first window.
  4. Click the Repair Your Computer link on the next dialog.
  5. Select your main operating system and click Next.
  6. Click Command Prompt.
  7. Type bootrec /FixBoot and press Enter.
  8. Type bootrec /FixMbr and press Enter.
  9. Type bootrec /RebuildBcd and press Enter.
  10.  Restart your computer. This should completely repair any issue encountered in the boot manager.

If the issue doesn’t get fix try to follow the suggestions provided by April 21, 2012.

bossdweeb say:

- One thing I don't see on your Startup image is you main AV. This should load with Windows and be set for automatic updates of the definitions. ( I have my AV inform me of a program update but not install automatically).

No good having four AV's and none of them running.

- Run only one AV in real time. Running more than one AV can cause conflicts. It can also slow the system down

due to the constant scanning.

"Junkware Removal tool, and CCleaner frequently... how to speed up your PC/laptop with programs   "

Optimising programs are not really necessary. The link provided by Ruth has all of what you need to keep

Windows happy.

Windows Disk Cleanup and defraging the HDD are the two most important functions for keeping

your machine running smoothly.

(Disk Cleanup can be found in All Apps > Windows Admin Tools. Pin it to the Start Menu for quicker access.)

- I also keep the number of Startup programs at a minimum. I have ten active atm.

- I have CCleaner, but only use it for the uninstaller, which will remove entries that sometimes

get stuck in Programs and Features.

- Most of this optimising stuff is a hangover from the dim dark distant past of 128MB system RAM and 1GB HDD's.

Too much focus on 'optimising' and security, can slow how you do things.

- Cleaning cookies all the time can wipe your login info which can keep you from being logged in automatically

on sites you visit frequently.

- You also need to be careful of other things which these programs 'clean' such as commands you enter into

'Run' (if you use that frequently it's nice to have the commands you use there already)

- Registry 'cleaners' don't really help with performance, and the risk of something going wrong because of

a deleted key, while rare with the registry tool in CCleaner because it only does a shallow scan, is greater than

the perceived benefit. Perceived being the operative word.

- Aside from that, go through Programs and Features and uninstall software you no longer use (No! Not any of

the many C++ installs, or anything you are not sure of. Keep these), and trawl through your User folder and

delete stuff you  know you don't need (being careful you know what it is you are deleting, particularly

in the AppData folder).

- Yes you may have too many programs running at Startup, but what you don't need is up to you.

There is really no reason to have a some of that stuff running in the background if you don't use it

all the time. Even your browser can be launched  from a shortcut pinned to the Task bar.

The only auto updaters I allow are my AV and Adobe Flash Player, but Flash Player is set to inform so I

know when it's happening.

.

ken blake say:

Two points:

1. What you have starting automatically might affect your boot time, but does not affect the speed of your computer unless it continues to run after booting.

2. Despite what many people tell you, you should be concerned, not with how many programs you have running, but which. Some of them can hurt performance severely, but others have no effect on performance.

Don't just stop programs from running willy-nilly. What you should do
is determine what each program is, what its value is to you, and what
the cost in performance is of its running all the time. You can get
more information about these with internet searches and asking about
specifics here.

Once you have that information, you can make an intelligent informed
decision about what you want to keep and what you want to get rid of.
