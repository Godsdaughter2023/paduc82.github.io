Unable to attach a Soft Deleted mailbox back to original user account - Microsoft Community

alastairbailey say:

Unable to attach a Soft Deleted mailbox back to original user account

We had a 'human' problem with directory sync and a number of users were removed from the sync.

There mailboxes for these users were automatically placed in to the soft delete selection of Exchange Online.

The original users account were then added back into the sync and back into 365.

The original mailboxes will not attach back onto the original accounts.

They see the same 'ExchangeGUID' on the user account and the mailbox as a conflict.

Is there a way to force the connection between mailbox and account?

Thank you.

alastairbailey say:

Hello,

Just to let anyone know, after much back and forth with Microsoft support they confirmed that although the mailbox existed as 'Soft Deleted' items it was not possible to re-attach them back onto the original AD user account due to seeing the original user and the original mailbox (now soft deleted) as conflicting items.

It was also not possible to create a new mailbox on the original account and do a restore as, again, it conflicted with the original soft deleted mailbox.

In the end we had to do the following:

1. Suspend the ADSyncScheduler.

2. Delete the original user accounts from Azure AD (Remove-MsolUser)

3. Delete the original user account from Azure AD recycle bin (Remove-MsolUser -RemoveFromRecycleBin)

4. Create temporary user accounts with 'Exchange Online' licenses to create new mailboxes.

5. Use the 'ExchageGUID' of the soft deleted mailbox and the 'ExchageGUID' of the new mailbox to run a restore (New-MailboxRestoreRequest)

6. Change the temporary user accounts 'UserPrincipalName' and 'ImmutableId' to the original users details.

7. Restore the ADSyncScheduler.

Thanks,

Alastair.
