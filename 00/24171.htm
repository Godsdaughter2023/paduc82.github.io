andreas killer say:
The answer to your question is no, that is not possible, because the Change event fires if the change is done.

You can use the Selection_Change event and preserve the contents... whatever that means in your sense.

BTW, below is a little optimized and bug fixed version of your code.

Andreas. 

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim KeyCells As Range
  Dim Current As Date
  
  ' The variable KeyCells contains the cells that will cause an alert when they are changed.
  Set KeyCells = Range("w21:w23")
  If Intersect(KeyCells, Target) Is Nothing Then Exit Sub
  
  ' Copy and Paste Values ONLY if there are entries prior to todays date
  Current = Now
  If Current <= Range("aa6") Then Exit Sub
  ' Identify the Dynamic Range
  Dim R As Range
  Dim LastDiff As Date
  Dim LastCell As Range
  LastDiff = DateSerial(9999, 1, 1)
  For Each R In Range("Aa6:Aa154") '<<< CHANGE
    If Abs(R.Value - Current) < LastDiff Then
      Set LastCell = R
      LastDiff = Abs(R.Value - Current)
    End If
  Next
  'Be sure we found a cell
  If LastCell Is Nothing Then Exit Sub
  ' Extend the Range
  With Range("AB6:AG" & LastCell.Row)
    Application.EnableEvents = False
    .Value = .Value
    Application.EnableEvents = True
  End With
End Sub

Try this formula in cell C3 in Sheet2:

=IF(ROWS($A$3:$A3)>SUMPRODUCT((Sheet1!$A$1:$A$100=$A$3)*(Sheet1!$I$1:$I$100<=Sheet1!$J$1)),"",INDEX(Sheet1!B:B,SMALL(IF((Sheet1!$A$1:$A$100=$A$3)*(Sheet1!$I$1:$I$100<=Sheet1!$J$1),ROW($1:$100),999999),ROWS($A$3:$A3))))

If your data in Sheet1 extends below row 100, change all five occurances of 100 to a number big enough to cover you data.

As you see, this formula does not refer to cell B3 or to column O in Sheet1 so the formulas you have there can be removed if they are not used for other purposes.

andreas killer say:

You can have the same result without a macro.

1) 2) If there is no data in that rows, there is no need to clear anything. To setup the cells with a white background to make the gridlines unvisible on screen makes no sense, because if we leave them empty they are not shown on printout.

3) Change your conditional formatting in a way that it ignores empty rows and format only the data.

4) If this 3 rows are always the same why not use a footer in page setup?

5) It is not possible to detect the wide of the printout and place some text in a cell that it is at the center of the printout. But you can use a header in the page setup instead, the page setup supports this.

6) If you adjust the columns, there is no need to do this again.

7) Makes no sense, because we print all data on that sheet. If you have more data to the right of column V, use the page setup and set the print area to A:V

8) 9) Also this must be done only one time.

If you prepare a file with all this settings, you can save it as template (.xltx) and import/copy the data into that sheet.
