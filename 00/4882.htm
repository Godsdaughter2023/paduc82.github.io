The main thing that Mobile Office 6.1 gives you is compatibility with Office 2007, that is, you can open Word 2007, Excel 2007, or Powerpoint 2007 files on your mobile device.

I think you are thinking about WM6.1 - that would update the operating system.  You'll need to look to the OEM for any kind of update to the operating system.  Updates to the operating system are not usually available to end users.  Microsoft makes these kinds of updates available to the OEM, and the OEM customizes it for its devices.  It is entirely up to the discretion of the OEM as to whether or not an OS update will be delivered to end users.

Resizing Excel table with one single row using VBA - Microsoft Community

martinandersen67 say:

Resizing Excel table with one single row using VBA

When resizing a table with multible rows to table with one row, the DataBodyRange of the table becomes Nothing, which means that it is not possible to write to the single row:

Set rngNonActiveUsers = tblNonActiveUsers.Range

IsNonActive  = 1

With rngNonActiveUsers
     Set rngNonActiveUsers = .Resize(IsNonActive + 1, .Columns.Count)
End With
Debug.Print rngNonActiveUsers.Address

tblNonActiveUsers.Resize Range(rngNonActiveUsers.Address)
Set rngDataNonActiveUsers = tblNonActiveUsers.DataBodyRange
Debug.Print rngDataNonActiveUsers.Address

 The code line Debug.Print rngNonActiveUsers.Address returns the expected two rows range address. After tblNonActiveUsers.Resize Range(rngNonActiveUsers.Address), the DataBodyRange of tblNonActiveUsers is Nothing, which means that the line Debug.Print rngDataNonActiveUsers.Address returns error 91; Object variable or With block not set.

One way of solving this problem is to hard code the single row range address and insert the values in this range.

But is there at better way?

mr andrew say:

1. Open the modify dialog for your style and check that "Style for following paragraph" is set to the style itself.  If it is something else (e.g. Normal) then that will explain what is happening to you.

2. (On the assumption that your style has been set up in a document based on the Normal template...)  Again, while modifying your style, choose the option that says "New documents based on this template" rather than "Only in this document".

(a) Click the "Manage Styles" button at the bottom of the styles drop-down menu (bottom right of the styles gallery on the Home tab)

(b) Click "Import/Export"

(c) Use the dialog to copy the style from the template you are working with to the Normal template

(d) Make sure you save modifications to Normal if prompted when you close Word
