Hello aatnveer,

 

  Thanks for posting to the Microsoft Answers forums!  Are you receiving a specific error code along with this message?  Ordinarily, seeing a message that there is not enough memory available to download would require deleting content on the device.  If you have deleted all music and video content, there is a chance that application or application data is using the remaining space.  Another possibility is that the amount of reserved space on your phone is larger than you expected, we can check this by connecting your phone to the Zune software on your computer and viewing the memory usage.

 

  With the phone connected to your PC, open the Zune software and (on your computer) click Settings > Phone > Reserved Space.  This will display a bar along the bottom of the screen that indicates the memory usage on your phone.  One thing to point out is that the amount of reserved space you have set aside affects the free space for downloading music and video on the phone.  This reserved space is typically used for app and game downloads, and content synced to your phone from other computers.

 

  If necessary, you can reduce the amount of reserved space by adjusting the space slider on this menu page.  More details about space and memory usage can be found on our support site:  http://support.xbox.com/en-US/music-and-video/zune/player/less-space-on-device-than-expected

The forms don't really depend on the table styles. From what you have, there should be a form bound to the Assets tables. On that form, you should use comboboxes to select all the FKs (EmployeeID, Item, RackID).

I would add a lookup table for Status as well.

scottgem say:

I suggest you read up on Normalization. You will find a good beginners reading list at utteraccess.com.

Your database is not constructed properly, which is one reason you don't see the solution. There is no reason to have a separate table for each type of object, especially when the tables have the same construction. You should have one table with a field for object type. based on what you have said you need several tables here: 1) Employee table, 2) Item table; ItemID (PK), SerialNumber, OjectTypeID, RackID, 3) ItemStatus table; ItemStatusID, (PK), ItemID (FK), StatusID (FK), StatusDate, etc. 4 Lookup table for Object Type, Rack, Status, etc.

As you are aware you shouldn't be duplicating data. except to include foreign keys (FK) that refer to the record in a related table. This way, you don't need to duplicate data.

rich michaels - mvp (greatcirclelearning.com) say:
Macros are stored either in Documents or Templates. Documents containing macros have a format extension of ".docm" and Templates have a ".dotm" extension.

Word's global macro enabled template is Normal.dotm and is located in this directory location:
~/Library/Application Support/Microsoft/Office/User Templates

User defined macro enabled templates are in this directory location:
~/Library/Application Support/Microsoft/Office/User Templates/My Templates

Macro enabled documents are all user defined so they will be located where ever the user saved them.
