in the sticks say:

Attempting network boot is not a "possible", it is definite that your PC is actually doing just that as per your BIOS settings. Furthermore, it is attempting network boot after having already tried booting to an optical drive. Both boot device attempts failed, in all probability after having already failed the hard disk drive boot. Either there is no message to indicate an attempt to boot to hard drive, or it leaves the screen too quickly for you to see or you have not recorded it in your original post.

My advice on diagnostics is based on having researched your PC's actual model, not generic advice for all Dell PCs with Vista installed.

  1. Your computer has already finished POST (Power On Self Test) otherwise you would not be seeing any boot attempt at all.
  2. You have already supplied specific error messages.
  3. I have already advised the best and most approriate diagnostics for your PC.
  4. Your PC will not be booting in any mode if this ever becomes a "hard" fault, rather than the intermittent fault your original post implies. Boot in any mode you can and do a backup asap.
  5. Unless those recent changes include BIOS settings adjustments, they are irrelevant, this is a boot failure that cannot be caused by anything done within Vista.
  6. Removing external devices is never a bad troubleshooting step if you are experiencing problems. I doubt it is going to assist in this instance but it will do no harm to try, especially if they are bootable devices.
  7. Startup repair is not going to solve a hardware problem (by far the most likley scenario).
  8. System restore is only operable if you can actually boot to Vista an In all probability this is what you are going to have to do but to a replacement hard disk drive.

I repeat that a failing or failed hard drive is the most likely scenario. Other more remote possibilities include

  • A failed or failing disk drive controller (almost certainly on the motherboard)
  • Hard drive boot is disabled in BIOS (very unlikely if this fault is intermittent)
  • Some weird kind of BIOS failure
  • Option 3 mentioned in the previous post (unlikely unless your PC has been internally inspected/modified or subjected to vibration by moving/knocking over/dropping etc.)

frederik long say: No problem. Now try this:

1. Log on under an admin account.
2. Click Start.
3. Type the three letters cmd into the Search box.
4. Press Ctrl+Shift+Enter
5. Click "Run as Administrator".
6. Type the following commands and press Enter after each:
    takeown  /f   c:\Users\Tom   /a   /r   /d y
    icacls c:\Users\Tom  /t /c /grant Administrators:F  System:F  Tom:F
   (The command assumes that "Tom" is the logon name of the account
    that owns the profile folder c:\Users\Tom)
    (Answer "yes" when prompted "Are you sure?")
    cd  /d  "c:\users\Tom"
    for  /d  %a  in (*.*)  do  attrib -h -r  /d  /s "%a"

The account "Tom" will now have full acces to all files in the profile folder c:\Users\Tom. All previously hidden files will be unhidden.

Accurate typing of the above commands is essential, particularly when it comes to spaces, double quotes and the two types of slashes.
