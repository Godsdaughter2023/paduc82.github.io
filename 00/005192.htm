hansv mvp say:

See for example http://bytes.com/topic/access/insights/601315-vba-data-types. It mentions Access but applies to VBA in other applications too.

As you can see there, the Integer data type can hold values from -32,768 to + 32,767 (inclusive). If you assign a value to a variable of type Integer outside this range, you get an overflow error. Example:

Dim m As Integer

Dim n as Integer

m = 30000 ' OK, no problem

n = m + 10000 ' Oops, overflow

In this example, you can avoid the error by declaring m and n as Long instead of as Integer. Long can be from -2,147,483,648 to +2,147,483,647 (inclusive), so 40, 000 is not a problem.

john korchok say:

It sounds like the arrows may be graphics rather than symbol characters. Since PowerPoint can't include graphics in tables, they wander. PowerPoint graphics always sit above or below the table and are anchored in relation to the page edges rather than the table cells.

If your Excel sheets used characters like the up and down arrows in Wingdings, they would stay in the cells when transferred to PowerPoint.
