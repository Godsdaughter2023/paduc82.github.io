dirk goldgar say:

I don't have an appropriate table handy to test this, but I think because you are outputting an open, filtered, table datasheet to the Excel workbook, Access must be placing the data on the clipboard for transfer to Excel.  111350 records is a lot of data to put on the clipboard, even though Excel can easily handle that many rows in a worksheet.

If I were doing this, I'd use a somewhat more roundabout method.  I would create a dedicated QueryDef (stored query) object for the purpose of exporting data.  Maybe I'd call it "qryExport", and initially set it to this SQL:

    SELECT * FROM tblScope

You'll notice that the query doesn't yet apply the criterion, "Plant='56'".

Then, when I wanted to export to Excel, I'd dynamically modify the query's SQL, and use DoCmd.TransferSpreadsheet to export it:

    CurrentDb.QueryDefs("qryExport").SQL = _

        "SELECT * FROM tblScope WHERE Plant='56'"

    DoCmd.TransferSpreadsheet acExport, acSpreadsheetTypeExcel12Xml, "qryExport", "C:\Users\Desktop\Scope_56.xlsx", True

And if I want to then open it in Excel, I'd add the line,

    Application.FollowHyperlink "C:\Users\Desktop\Scope_56.xlsx"

Note: TransferSpreadsheet doesn't do some of the formatting that OutputTo does.  If you want to programmatically set formatting on the exported spreadsheet, you would have to automate an instance of Excel to open the workbook, do the formatting, and save the workbook.
