I found that the problem with just copying in new font files is that it does not delete the old font files, it just adds the new fonts to C:\windows\fonts.  You cannot "see" the old font files by navigating to C:\windows\fonts as it is a "special" folder and just shows the installed fonts (r-click a font and Properties shows which font file is being used for that font).  To see the other files, I needed to use a cmd window and issue a dir command.

Second, asume you have arial.ttf in C:\windows\fonts.  When you copy in a new arial.ttf file it is renamed arial_1.ttf and the registry key HKLM\Software\Microsoft\WindowsNT\Current Version\Fonts is updated to point Arial to the new file C:\windows\fonts\arial_1.ttf.  I had over 1GB of fonts, with many duplicated or triplicated fonts like arial.ttf, arial_1.ttf, arial_2.ttf, arial_3.ttf where I had been copying in fonts.  W7 asks the user if he wants to overwrite the existing font - when the user says yes, W7 adds the font but does not overwrite the existing file.  This is sensible as it means a rogue font does not destroy a good font.  However, the Windows message prompt is wrong.

I fixed it be using cmd with administrator privilege,navigating to C:\windows\fonts, and deleting all copies of the font files I did not want, working always with just a few fonts at a time as I did not want to be in a position where no fonts were installed - I didn't know what W7 would use to display things.  I used del fred*.ttf /P as it allowed me to confirm each file deleted.  I then deleted the registry keys, and dragged the new font files into  C:\windows\fonts with Explorer to create the new registry keys, which could only be seen by restarting regedit.  I then rebooted so that I could see the new fonts when I opened C:\windows\fonts. 
 
I always ensured I had plenty of working fonts, and I left Martlett well alone - this is apparently the font used by Windows for things like the minimise icon at the top right of every window title bar.

I had thought that merely editing the key to change it away from arial_3.ttf to arial.ttf would force Windows to use the arial.ttf file for Arial.  But it didn't - I had to delete the key and get it recreated when the font was dragged in. 

This seems to have worked for me although I am still concerned that editing the registry is not a sensible thing to do.

I reset all the Ready for archive attrributes with attrib -A *.* in the cmd window.  Sometimes Windows refused to let me delete a font file, but renaming it to something else allowed it to be deleted.

a. user say:

The different versions of Windows can be legally used in any situation - home, business, non-profit. The difference among them is their feature sets. For example: A business can use the Home version if it wants to, but the Home version lacks features that businesses typically need.

Some versions of Windows are only sold to specific customers. For example: The Education version is only sold to schools because schools have unique use cases. A business can use the Education version, but why would it want to? A home user can use the Enterprise version if they want, but that version's feature set benefits organizations with huge numbers of users, so it's only sold with a volume license. If a home user wants to pay for a volume license, Microsoft is glad to take their money, but what's the point?
