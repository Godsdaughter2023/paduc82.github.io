ken sheridan say: Apologies for the late reply; I've been heavily committed to other matters of late, so did not see your post until today.

You misunderstand what the Answers table is modelling.  It models the set of all possible answers, not a respondent's answers to a question.  That is done by the Responses table.  If there can be only one administrator response to each respondent's response then you can add columns to the Responses table.

If you have a DATE field in a merge document, when merged to a new document that new document no longer has date fields but fixed dates.

macropod say:

No, unless it's in a QUOTE field or an IF field, DATE fields in a mailmerge are preserved.

So, the simplest way to have a mailmerge convert the output of a DATE field to static text is to embed the DATE field in a QUOTE field. To do this, assuming a DATE field displaying '12/12/2012':
• Open the mailmerge main document
• Select the DATE field
• Press Ctrl-F9 to create a pair for field braces around the DATE field, thus { 12/12/2012 }
• Type 'QUOTE' inside the left field brace, thus {QUOTE 12/12/2012 }
• Press F9 to update the display
• run the mailmerge.
