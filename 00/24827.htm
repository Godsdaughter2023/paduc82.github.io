a. user say:

Before taking my word for it, it's worth seeing if anyone else comes up with a bright idea. But...

The ActiveX controls on WIndows Word are regarded as "legacy controls" - although they still function it's probably not such a good idea to use them for anything that needs to be maintained longer term. These days, you probably need to save as a .doc/.docm/.dotm (i.e. a macro-enabled document) to use them. As far as I know, Word for Windows is the only environment where these ActiveX controls will actually do anything useful (e.g. scroll or modify the text). Everywhere else they just don't work or just display text or a graphic. 

The same would in fact be true if you tried to use Windows Word+VSTO+Windows forms controls. You also have to create a .NET project (e.g. in Visual Studio) to do that. They would only work on a Windows platform with Word present. They actually rely on the same mechanism as ActiveX to function, but being a .NET-related technology that doesn't purport to be cross-platform, they probably aren't regarded as quite such a "legacy" option.

As for other software, nothing I know is an obvious choice. If it has to be "a feature of a word processor", I'd probably be looking at just the same options as you. Personally, I don't think the IT industry has ever managed to come up with a truly standard, rich (in the sense that it could do stuff that word processing packages do), preferably open, set of forms standards and technologies - we still seem to be in the "hire an individual who knows how to do the right things in HTML5/XML territory".

Having the confirmation that you work or know how to work using the PowerShell Module, I can recommend the following commands to identify users / aliases / distribuxion groups, duplicates.

Start-transcript
$ Duplicateuser = "*** Email address is removed for privacy ***"
get-contact | where {$ _. WindowsEmailAddress -match "$ Duplicateuser"} | ft identity
get-msoluser -All | where {$ _. ProxyAddresses -match "$ Duplicateuser"} | select UserPrincipalName

Get-Recipient -ResultSize Unlimited | where {$ _. EmailAddresses -match "$ DuplicateUser"} | Select -Expand EmailAddresses Aliases
Get-msoluser | Where-Object {$ _. AlternateEmailAddresses -eq $ Duplicateuser}
Get-Mailbox -ResultSize unlimited | Where-Object {$ _. WindowsLiveID -eq $ Duplicateuser} | select userprincipalname, identity
Stop-transcript
