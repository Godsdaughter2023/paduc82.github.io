daniel g. grau say:
A section can have it's own set of headers & footers, and they can be separately linked or not to those of the preceding section (this is done through the "Same as previous" setting).

The main criteria for inserting a new section is: "Does this page require a change in the header/footer, margins or page numbering?" Another criterion is a change in number of text columns (which in normal documents is one).

In your case, the answer is "yes" just once, and that's the point where you need to insert a section break. If you need to start a new page within the same section, insert a page break.

So first get rid of all section breaks. Your document needs to look like this:

         Page 1, no page # in header or footer
         [we are in Section 1]
         
     ----------Page break(s), if required------

         Pages 2-6, no page # in header or footer
         [it's still Section 1]
         
     =======Section break============

         Page 7, page # formatted as "Start at 1" in header or footer
         [this is Section 2]
         In addition, "Link to previous" needs to be disabled
         to make the headers and footers independent.
         
It will be easier to follow the above structure if you display nonprinting characters (press the Â¶ button in the Standard toolbar):

  • Place your cursor at the start of Page 7
  • Insert a "Section break (next page)"
  • Go to the header or footer
  • Click on the Header and footer purple tab in the Ribbon
  • Unlink the header (in Section 2) from the previous one (in Section 1) by taking the checkmark off in "Link to Previous"


To modify a page number after it has been inserted, click on its header, select Insert > Page Numbers and click on the Format button.

If you don't mind dabbling with a bit of VBA code, my normal way of dealing with CF is to insert an event macro into my workbooks that resets all the CF conditions whenever the workbook is saved. The easiest way is to record the CF steps, then paste them into a macro shell that does what you want.

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
    With Range("F1:F3") ' Macro recorder will record statements like next one as 
        .FormatConditions.Delete
        .FormatConditions.Add Type:=xlExpression, Formula1:="=D2=1"
        With .FormatConditions(1).Interior
            .PatternColorIndex = xlAutomatic
            .Color = 255
            .TintAndShade = 0
        End With
        .FormatConditions(1).StopIfTrue = False
    End With
End Sub

I've also managed to repair my system files by using 'sfc /scannow'. This is how to do it for those interested in it. Note: you will need the ISO for build 10041. If you don't know how to get it, see Andre's thread: Instructions: Create a bootable ISO file using ESD Image for Windows 10 Build 10041

Now follow the next steps.

1. Mount the .ISO file by double clicking on it.

2. Open 'Windows Powershell' or 'Command Prompt' with Admin privileges (right click -> Run as Administrator)

3. Let's check the System Health first, by running these commands:

  3.1 dism /online /cleanup-image /scanhealth

  3.2 dism /online /cleanup-image /checkhealth

  3.3 dism /online /cleanup-image /restorehealth

4. I'm sure that 3.3 will give you an error stating that it could not perform the task. And now it's when the mounted ISO comes into play. Let's specify the file from the ISO so that we can fix it. Run the following command: (Notice that X must be the drive letter on which your system has mounted the ISO)

  4.1 DISM /Online /Cleanup-Image /RestoreHealth /source:WIM:X:\Sources\Install.wim:1 /LimitAccess

5. Now let's repair any damage in the system files, shall we?

  5.1 sfc /scannow

Now it should work for you, guys! I hope this will help. My system seems more responsive since then. Please report if you encounter any problems following these instructions.
