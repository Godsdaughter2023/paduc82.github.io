I have investigated this (major) software bug and found the following:

 

When you have called a contact that is stored in your phone using different numbers (eg called them on home number AND mobile number recenty), the call log will display all recent multiple calls.

The same applies if you have called the same contact's number but using a different notation (eg perhaps once using the standard national number, and another time using #31# to hide your own number sending).

 

HOWEVER, the problem still remains that if calling a contact / receiving a call multiple times from the exact same number (as is often the case), multiple calls do not show up in the call log.

 

### THIS IS A MAJOR GLITCH that no one seems to be aware of and needs to be fixed urgently!!! ####

 

In the meantime, if you need to see your prvious call history, you can simply call the contact with the #31# prefix and then hang up straigth away, when you go back to your call log you will see the last 5 calls made and/or received from that contact, not just the last one.

jim gordon mvp say:
Your experience shows that open formats are not always the most desirable way to save files. The old format (.xls) is binary, meaning it is optimized for computer efficiency, not human readability. The open XML format (.xlsx) is designed so that humans can read the contents. If you change the file extension from .xlsx to .zip and double-click the zip file in Finder you can readily see and read the XML text files of which the workbook is comprised, and see the media files (if any) that are in your workbook, such as pictures and movies.

The trouble is, text is a horrible way for computers to store complex files if speed is important to you, which seems to be the case here.

When you open a file saved in the old (.xls) format in Excel 2011, many commands are grayed out and there is a notice at the top of the workbook window that you are in "compatibility mode." Essentially, all features added to Excel since 1998 are turned off in compatibility mode. So saving in the old format may not be the best way around this problem.

There is another new file format in Excel 2011 you may wish to try out. It lets you save in an efficient binary format, but doesn't disable any commands because this new format supports all the new features. Try saving your file using File > Save As and in the Format pop-up choose Excel Binary Workbook (.xlsb).  This format might give you the even better speed opening and saving as the old (.xls) format while avoiding compatibility mode.

Try saving your workbook in each of these formats. I expect you will find the following:
.xlsx - largest file size, slowest to open and save
.xls   - smaller file size, faster to open and save
.xlsb - smallest file size, fastest to open and save

If your findings match my predictions, you can change your Excel preferences so that any new workbooks you make default to .xlsb. Go to Excel > Preferences and click on Compatibility. In the Compatibility preferences dialog, in the Transition section, where it says Save files in this format, click the pop-up menu and choose Excel Binary Workbook.xlsb.

Where I work we have standardized on .xlsb for Excel workbooks.
