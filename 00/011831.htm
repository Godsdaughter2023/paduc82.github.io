Hi Lisa,

This feature has always worked this way, this is based on my own experience as well as some research I found on old threads. When there is a required column, after you successfully uploaded an item, it will always redirect to the EditForm.aspx, which is why your users are getting the Access Denied error.

I can't confirm why you experienced a different behavior a few weeks ago, especially since it is not happening anymore.

However, as you have let us know regarding your requirements, let's focus on that. I have found a few solutions to the issue that you can consider.

1. Create a default public view that only shows items the owner creates, delete the All Items view, and assign the Edit permission to users. This means that users can only edit their own items. You can add a personal All Items view. Users will not be able to create their own view if they are not given the Manage List permissions.

Here are the steps for that:

a. Open doc library. Click Settings > Create View > Standard View.

b.  Specify the name for the view - for example filter - and choose to make this the default view.

c.  Choose ‘Show items only when the following is true’.

d.  Set show the items when the column ‘Created by’ ’ is equal to  ‘[me]’.

e.  Click OK  to save the view.

2. Create a template that contains the custom property by creating a new document inside the library and downloading it. This will make the document contain the custom property, which the users can fill in via Word before uploading to SharePoint. Please note that if you are using a managed metadata column type, this won't work.

3. Use a custom solution that requires coding, something similar to what is explained here:

Permission difference between "Edit Document" and "Edit Properties"

If you wish to do this, we suggest posting to the MSDN SharePoint 2013 forum for assistance.

ken sheridan say: Why not use the OutputTo method of the DCmd object to create the PDF file.  You first need to get the report name and the path to which it is to be saved.  There are a million and one solutions for the latter, but for many years now I've used Bill Wilson's simple and freely distributed class module.  You can find a demo of this as BrowseDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

You might have to copy and paste the text of the above (not copy the link location) into your browser; for some reason it sometimes doesn't seem to work as a hyperlink.

With the module in place in your database you'd open the dialogue to save the report as a PDF file like so:

    Dim OpenDlg As New BrowseForFileClass
    Dim strFullPath As String, strReport AsString
   
    OpenDlg.DialogTitle = "Select or Enter File Name"
    OpenDlg.InitialDir = <get default folder path from somewhere>
    OpenDlg.DefaultType = "*.pdf"
    strFullPath = OpenDlg.GetFileSpec
    Set OpenDlg = Nothing

    strReport = <get report name from somewhere>

    DoCmd.OutputTo acOutputReport, strReport, acformatPDF, strFullPath

If you want the file to open immediately in Adobe Reader use:

    DoCmd.OutputTo acOutputReport, strReport, acformatPDF, strFullPath, True

rob koch say: Yes, Microsoft Security Essentials works the same, but the browsers themselves are actually quite different in their abilities to protect you from things like Phishing, which really have little or nothing to do with the malware that MSE is designed to stop.

For more about these browser protection abilities, see these tests by a third-party organization that show Internet Explorer 8 and Firefox 3.5+ have far better protection than the others you mention, though most have improved to at least some extent.

http://nsslabs.com/browser-security

Note that MSE also integrates with both Internet Explorer 7/8 and recent Firefox (3.x?) versions to protect from malware downloads, since both of these are designed to work directly with antivirus products to protect you.  I don't believe any of the other browsers currently do this, so consider this when choosing them to perform file downloads and even browsing in general.

The common misconception is that all your protection is left to the antivirus program, while in truth much of your security depends on your choice of browser.  Also realize that each browser you add to a PC is another entirely different program that requires you to maintain updates and upgrades, so in reality each one simply increases your risk and reduces your overall security.

Rob

don cuthbert say:

There is no way to activate Windows 10 free upgrade by Product Key.  It will only activate after an upgrade from a qualifying, activated prior OS (Windows 7, 8 or Windows 10 Insider Preview).  There is no unique Product Key for the free upgrades.  After an upgrade install it auto-activates based on detecting a valid upgrade.

If you can install and activate Build 10130 or an earlier Insider Preview Build to upgrade from, it could work, but none of them are currently available from the Insider Program.  The most recent Build, which was released to Windows Insiders before July 29th, is Build 101240, so the same base Build as the RTM and public release.

Of course, if you can activate a genuine Windows 7 or 8 on your computer, you can use the Windows 10 ISO to upgrade to an activated Windows 10.
