To avoid confusion for others coming here: Stefan's screen shot is from Word 2010. There are two differences in Word 2013:

  • There is no Color dropdown for change lines, because the change lines in 2013 are red when the document displays Simple Markup or grey when it displays All Markup (and clicking a change line switches from one to the other).
  • There is no Use Balloons dropdown in the large Track Changes Options dialog (renamed "Advanced" Track Changes Options). Instead, it's in the new small Track Changes Options dialog that appears first when you click the dialog launcher in the Tracking group.

Subtotals functions ignore values returned by other subtotal functions, so that the grandtotal works out:

Type Value
Type 1 1
Type 1 2
Type 2 7
Type 2 8

With Subtotals applied:

Type Value
Type 1 1
Type 1 2
Type 1 Total 3
Type 2 7
Type 2 8
Type 2 Total 15
Grand Total 18  <<<< ignores the 3 and 15, adding 1,2,7,&8

=SUM(U683,AC683,AK683,AS683,BA683,BI683,BQ683,BY683,CG683,CO683,CW683,DE683)

hansv mvp say:

Create an event procedure for the command button that looks like this:

 

Private Sub cmdReport_Click()
    On Error GoTo ErrHandler
    DoCmd.OpenReport ReportName:="rptMyReport", View:=acViewPreview, WhereCondition:="ID=" & Me.ID
    Exit Sub

ErrHandler:
    If Err <> 2501 Then
        MsgBox Err.Description, vbExclamation
    End If
End Sub

 

where cmdReport is the name of the command button, rptMyReport is the name of the report, and ID is the name of a field that uniquely identifies the current record in the form. I have assumed that it is a number field. If it is a text field, use

 

    ..., WhereCondition:="ID=" & Chr(34) & Me.ID & Chr(34)

 

Chr(34) is the double quote character.

graham mandeno say:

Hi Emily

 

I think there was a syntax error.  Access is much fussier about JOIN syntax than other SQL implementations.

 

The correct syntax is to "nest" the JOINs (which actually makes it harder to read):

table1 INNER JOIN
(table2 INNER JOIN table3
ON table2.field2=table3.field3)
ON table1.field1=table2.field2

 

What you had was this:

(table2 INNER JOIN table3 ON table2.field2=table3.field3)
INNER JOIN table1 ON table1.field1=table2.field2

 

-- Graham
