I don't know of any MS utility other, of course, than the DIR command itself.

You can print the output of the DIR command by opening a Command Prompt window and using the following command

     dir {parameters} > filename.txt

There are a lot of possible parameters that can be used with the dir command:

DIR [drive:][path][filename] [/A[[:]attributes]] [/B] [/C] [/D] [/L] [/N] [/O[[:]sortorder]] [/P] [/Q] [/R] [/S] [/T[[:]timefield]] [/W] [/X] [/4]

For details, type

    dir /?

in a Command Prompt window

If you don't include a complete pathname for filename.txt, it will be saved in whatever directory appears at the command prompt (default C:\Users\<yourusername>\

If you use the following, it will show up on your desktop:

   "%userprofile%\Desktop\directorylist.txt"

techie bear say:

Hello Alan,

There are 2 different things going on. First, you need to set your display resolution to 125%. To do this go to Control Panel, click on Display, and when that opens a new window, click on the left-hand side menu, "Set custom text size (DPI)". That should lock in your custom DPI setting @ 125% (or whatever you set it at).

 

The second setting is more problematical, and is something you need to set in each application that is a problem for you. I have a similar problem with my desktop computer, and have to set my high def monitor @ 150% to be able to see text clearly. The problem is that many applications aren't "DPI aware", meaning they don't automatically adjust for high definition monitors.

 

To get around this problem, right-click on a program's desktop icon and select properties. When that window opens, select the compatibility tab from the top row.  Scroll down the list of check boxes till the last one and mark it, "disable display scaling on high DPI settings", then click Apply, OK, and OK.

 

Try that and let me know how you fare.
