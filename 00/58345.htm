0x8007007B - Microsoft Community

fsalha say:

0x8007007B

what is this error mean?

charlesjones32 say:
Hi. Fsalha

I'm Charles, an Independent advisor trying to help.

The error code 0x8007007B simply means that the code used has a limit on the number of times it can be activated or installed.

It is an error that occurs when you try to activate Windows 10, Windows Server 2008, Windows Vista, Windows 7, Windows 8 or Windows Server 2012.
Common symptoms include:

    • You will see an error message like this: "Error 0x8007007B 'Syntax for a file name, directory name or volume label is incorrect'".

    • If you are using volume-licensed media to install: Windows 7, Windows 8, Windows Vista Business, Windows Vista Enterprise, Windows Server 2008 R2, Windows Server 2008, Windows Server 2012, Windows 10.

    • Your computer is not connecting the activation wizard to a KMS host computer (Key Management Service).

To fix it, it is recommended to download the automatic repair utility to correct this and other PC performance problems.

It is important to know the cause in order to effectively find the solution.

Method one:

    1. From the desktop screen, press the Windows key and S, type "Command Prompt". Then, right-click on Command Prompt in the results. Choose Run as administrator. Click Yes when asked about the permission request.

    2. In the command box, type the command: slmgr.vbs -ipk xxxxx-xxxxx-xxxxx-xxxxx-xxxxx

    1. The x represents the key to the product. Replace them with your unique product key.

    1. Type the following command in the "Command Prompt" window: slmgr.vbs -ato

    2. Click on Enter

    3. Restart the computer and try again to activate Windows. It should not show the error code anymore.

Method two:

    1. While on the desktop, press the Windows key and the R key.

    2. A window that says Run should open.

    3. Type the following command: Slui 3

    4. Press ENTER and a window will appear.

    5. This window asks you to enter the activation product key of the operating system.

    6. Enter this key in the space provided.

    7. After entering this, click Activate.

    8. Restart your computer now.

    9. Verify activation. The error code should disappear now.

Method Three:

    1. Open the command prompt with administrator access as in Method One and enter the following command: sfc / scannow

    2. The command will complete the tasks. This will take some time so just let the computer do its thing and work. Let the sfc finish the scan.

    3. When the scan is complete, restart the computer.

    4. Check the activation. The error code should disappear at this time.

    5. If you have left, review Method One or move Method Four.

Method Four:

Call the Microsoft Support Team and explain the problem with all the details you can give. Give the team the error code 0x8007007B and ask that you have a different product key.

When you are given the new product key, repeat Method 2. The Microsoft Support Team will change your product key or reset your current product key so that it can be used again for activation.

Best Regards,
Charles.

Download and run XXClone. It is small, free and can clone an IDE, SATA or SCSI hard disk to either an internal disk or a USB disk. Only the professional version will do cumulative backups but this is perfect if an exact clone of the drive is needed. Get it here http://www.xxclone.com/index.htm Click ‘Download’ in the left pane and select the freeware package version. It is intended for single installations that run Windows 95 through to XP, 32-bit only, not 64-bit. Note that a separate disk or separate partition must be used as the target (destination) drive because all the data on it will be destroyed by the cloning.

 

Here are a few tips on running the program:

Close down any other running programs before starting. Make sure you select the correct Target Volume. If you intend to boot from the cloned drive, before clicking Start to do the cloning process, you need to go to the Cool Tools Tab > Make Bootable and tick/run all three ‘options’ (Write MBR, Write Boot Sector and Write BOOT.INI). Do not continue with the cloning if you receive any error messages, e.g. NTLDR or NTDETECT.COM is missing.

 

If an internal drive is used, boot.ini on the source drive will add the cloned drive as a second bootup choice, i.e. it becomes dual-boot.
