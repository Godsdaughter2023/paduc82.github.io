ken sheridan say:
I am still puzzled, however, as to why the command button wizard doesn't seem to open a new form and carry over the related field.

When a form is filtered it restricts the rows returned to the existing rows which match the criteria.  It does not set the value for the foreign key column in a new record.  In fact a form's Filter property need not necessarily refer to the key at all; it could restrict the rows returned on something completely different.

To set the value for the foreign key column when a new record is inserted a different property is used, the DefaultValue property, so to achieve what you are aiming at it is necessary to set this in addition to filtering the form.  This only applies where the form's are independent of each other, what I've termed 'linked' forms in my demo.  If a form is embedded in another form as a subform and linked to the parent form on the key, then when a new record is inserted into the subform the value of the foreign key column is automatically set by the linking mechanism to that of the primary key of the parent form's current record.

chanchan05 say:

All files sent through bluetooth to the phone would go to the memory for which you selected to save your messages. To alter this, go to Menu>Messaging>Options>Settings>Other>Memory in Use

sidharth_sahni say:

You must be having your phone memory [C] as your message memory. Change your message memory to mass memory. That should do it.

 

You are facing this problem because the size of the songs you are trying to transfer on to your cell phone must be greater than 9.6MB i.e. the space available in your phone memory. Whenever you transfer songs or any other thing using bluetooth the data is received by the cell phone as a message and thus automatically goes to the message memory, in your case the phone memory, which is left with just 9.6 MB of space

 

Alternatively, you can transfer some of your data from the phone memory to the mass memory which will free up some space in the phone memory and you will be able to receive your songs.

 

But prefer the first alternative.

 

Good luck
