aaaargh say:

Here is a VBA script that I wrote to cycle through every appointment item in the default calendar and remove the "Copy: " prefix.  It's a horrible kluge, but unfortunately Outlooks locks us out of any straightforward ability to control how Calendar items are copied or restored:

 

Sub FixCopy()
Dim calendar As MAPIFolder
Dim calItem As Object
   
Set calendar = Application.GetNamespace("MAPI").GetDefaultFolder(olFolderCalendar)
       
Dim iItemsUpdated As Integer
Dim strTemp As String

iItemsUpdated = 0
For Each calItem In calendar.Items
    If Mid(calItem.Subject, 1, 6) = "Copy: " Then
      strTemp = Mid(calItem.Subject, 7, Len(calItem.Subject) - 6)
      calItem.Subject = strTemp
      iItemsUpdated = iItemsUpdated + 1
    End If
    calItem.Save
Next calItem

MsgBox iItemsUpdated & " of " & calendar.Items.count & " Items Updated"

End Sub

Make sure your version of Office is fully up-to-date (14.3.2) and you have run the Font Weeding and Repair Disk Permission procedures.

Next I recommend a full system restart. It could be as simple as that to fix the problem.

If the above doesn't help then quit Word completely (not just closing the open file) and while holding down the Shift Key, start Word. This disables all plug-ins such as copy/paste helpers. 

Now check the behavior as you open new documents and move between them. If the problem goes away, then you know it's one of those "helpers" and you need to either get rid of the "non-helper" or live with it.

quietman7 say:

You are most likely dealing with CryptoWall 2.0 which uses its own TOR gateways. At this time there is no way to retrieve the private key that can be used to decrypt your files without paying the ransom....your alternative is to restore from backup if you routinely performed that task. The only reliable way to effectively protect your data and limit the loss with this type of infection is user education and to have an effective backup strategy.

There is a lengthy ongoing discussion in this topic where Nathan (DecrypterFixer), a Security Colleague who specializes in crypto malware research has been assisting numerous victims with this and similar infections.
