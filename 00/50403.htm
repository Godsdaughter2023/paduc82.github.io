ralf_g say:

Homegroups and Workgroups are completely different systems that aren't compatible with each other. All computers in your LAN should be connected to the same private network which assigns all of their IPs within the same address range.  If you have been using the homegroup system and your homegroup disappears over the Wifi connection then it could  be due to an addressing problem or to network restrictions on the WiFi connected computer itself.

If you're using two routers there could be a number of problematic scenarios that you've inadvertently created.

Each router is capable of creating a different local network. These networks would be independently able to access the internet while computers attached to either router might not be able to connect with each other. Only one router should have routing and DNS addressing enabled to ensure that whichever hardware connection is being used, all of your computers are always connected to the same local network. The second router/repeater should be configured as nothing more than a switch which passes routing/DNS info through from the first router. 

Windows automatic networking configuration tends to default WiFi or secondary network connections in general  to use Public rather than Private network settings. Allowing that setting will prevent the computer in question from accessing your network shares whenever you use the WiFi. Similarly if both routers have WiFi active they will be treated as different networks even if you use the same password on both.  Ensure that your WiFi connection in the computer is set as a "Private" or home network and that it is receiving an IP address in the same range as it normally would via the Ethernet connection.

Please follow the steps to resolve the RPC problem

Step 1 : Start the RPC service

  1.  Press Windows key + X and select Command Prompt .
  2. In the Command prompt window Type net start RpcSs and hit enter.
  3. Type exit to close the command prompt.
  4.   Restart the computer and check the result.
Step 2: Check the registry entry for the service
 a. Start the computer using Windows DVD to boot to Windows Recovery Environment (WinRE) and select command prompt.
 b. On the command prompt, type regedit.exe and hit enter.
 c. Look for the following entries:
 i. Entry for Remote Procedure Call (RPC):
 Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\RpcSs
 ii. Entry for Remote Procedure Call (RPC) Locator:
 Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\RpcLocator
Important:
Set REG_DWORD value to 2 for automatic start on RpcSs
 Set REG_DWORD value to 3 for manual start on RPCLocator
 

Very important: Modifying the registry incorrectly will cause serious damage to the computer. Backup the registry before you make any changes.

http://windows.microsoft.com/en-us/windows/back-up-registry#1TC=windows-7
