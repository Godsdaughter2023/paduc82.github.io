john w. vinson say:

The argument to FindFirst needs to be a text string containing a valid SQL WHERE clause. If you're not (yet... you will want to be!) familiar with SQL you can create a query in the query grid and view the SQL to help learn the syntax.

In this case you just need a couple of OR clauses:

    rs.FindFirst "[UniqueRecNum] = " & Str(Nz(Me![SearchLeavingDocIDs], 0)) & " OR [CaseNumber] = " & Str(Nz(Me![SearchLeavingDocIDs], 0)) & " OR [CourtCase] = " & Str(Nz(Me![SearchLeavingDocIDs], 0))

  all on one line (or use a blank followed by an underscore _ as a line continuation signal).

marshall barton say:

A  data type mismatch error means that the values you are searching for are a different type than the Data Type of the field in the table.  You only put put quotes around a search value when the field is a Text or Memo field.  E.g.
   "TextField ='" & Me!SearchText & "' "
or
   "TextField = """ & Me!SearchText & """ "

But it the field you are searching is a Number Data Type in the table, you can not use quotes around the search value:
   "NumberField = " & Me!SearchNumber

And for Searching a Date/Time field, the search value must be in an unambiguous style and enclosed in # signs:
   "DateField = " & Format(SearchDate, "\#yyyy-m-d\#")

Another point here is that you need to be aware of what data library you are using.  If you are using a staandard Access/Jet database for your tables, you should be using the DAO library.  In that case, the wizard generated code you are tryng to modify is wrong.  It should be:

Private Sub SearchField_AfterUpdate()
    ' Find the record that matches the control.
    Dim rs As DAO.Recordset
    Set rs = Me.RecordsetClone
    rs.FindFirst "[DCSS_Case] = '" & Me![SearchField] & "' OR [DocIDLeaving]='" & Me![SearchField] & "' OR [Court_Case]='" & Me![SearchField] & "'"
    Me.DocIDReturning.SetFocus
    If Not rs.NoMatch Then Me.Bookmark = rs.Bookmark

End Sub

OTOH, If you are using the ADO library, the wizard generated code is ok, but that library's FindFirst can only search for a single condition. You would be getting other errors, so I don't think you are using the ADO library.

To do what you asked in this question, use additional code something like this untested air code:

Private Sub SearchField_AfterUpdate()
    ' Find the record that matches the control.
    Dim rs As DAO.Recordset
    Set rs = Me.RecordsetClone
    rs.FindFirst "[DCSS_Case] = '" & Me![SearchField] & "' OR [DocIDLeaving]='" & Me![SearchField] & "' OR [Court_Case]='" & Me![SearchField] & "'"
    Me.DocIDReturning.SetFocus
    If Not rs.NoMatch Then
        Me.Bookmark = rs.Bookmark
    Else
        MsgBox "No records match the search value"
        Exit Sub
    End If

    Do
       If MsgBox("Is this the record you want?", vbYesNo) = vbYes Then Exit Sub
       rs.FindNext "[DCSS_Case] = '" & Me![SearchField] & "' OR [DocIDLeaving]='" & Me![SearchField] & "' OR [Court_Case]='" & Me![SearchField] & "'"
       If Not rs.NoMatch Then
           Me.Bookmark = rs.Bookmark
       Else
           MsgBox "There are no more records that match the search value"
           Exit Sub
       End If       
    Loop While True
          

End Sub

can i block sharepoint system administrator from seeing libraries that - Microsoft Community

evren baser say:

can i block sharepoint system administrator from seeing libraries that i choose?

Hi,

I am admin in sharepoint with full authority. can i block sharepoint system administrator from seeing libraries that i choose?

jashion say:

hi evren,

we can’t prevent sharepoint service administrators from accessing one single library. however, we can prevent them from accessing a site collection via the steps below:
1. go to sharepoint admin center > site collections.
2. select the specific site collection, then on the ribbon click owners > manage administrators.
3. in the site collection administrators section, remove sharepoint service administrator. then click ok.
 

regarding the requirement, you can try the following workaround:
1. remove sharepoint service administrator from the site collection administrators.
2. go to the root page of the specific site collection, click share on the top-right corner to share the site collection with your sharepoint service administrators. give them full control permission.
 
3. go the library you’re trying to block access. click library > shared with > advanced > stop inheriting permissions.
4. then remove their full control permission that are inherited from the parent.

doug robbins - mvp office apps & services (word) say:

With a document that is "Protected for filling in forms", you cannot place the selection into text that is not located inside a FormField.

If you want to be able to edit such text, use Content Controls instead of Legacy FormFields.

stefan blom say:

In a document with form fields protection, you can't select existing content or add new content outside the text form fields. Editing will be a bit more free if you add a section and leave it unprotected.

However, if you want the document to be more flexible, consider using content controls instead. These are designed to be used without document protection.
