need a formula to count cells in a range that are shaded any color and - Microsoft Community

terielad say:

need a formula to count cells in a range that are shaded any color and return a value of a figure

Hi,

 

I need a formula to count cells in a range that are shaded any color and return a vale of a figure.

 

So if A1:A10 were all colored it would return 112.5

 

Many thanks

 

Mark

jeeped say:
Right click the worksheet name tab at the bottom and select View Code. Paste this into the pane on the right titled something like Book1 - Sheet1 (Code),

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    a = 11.25
    b = 0
    For Each cell In Range("A1:A20")
        If cell.Interior.ColorIndex > 0 Then b = b + a
    Next cell
    Range("C1").Value = b
End Sub

Modify the Range("A1:A20") line to address your own range, then tap Alt+Q to close the VBE and go back to Excel.
This function will fire every time that you move from one cell to another. The routine is not intensive so I do not think that should be a problem. Other solutions have relied upon recalculation which is not a factor in back-filling a cell's color.
You will have to save this workbook as macro-enabled.

jeeped say:

... Can this be repeated as I have 21 of these separate formulas to do...


Yes, you can either hard code the 21 routines or use a loop as JLLatham has suggested. The only change I would make would be to use the .Offset() method rather than constructing the cell ranges and addresses.

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    a = 11.25
    For i = 0 To 20
        b = 0
        For Each Cell In Range("A1:A20").Offset(i * 20, 0)
            If Cell.Interior.ColorIndex > 0 Then b = b + a
        Next Cell
        Range("C3").Offset(i, 0).Value = b
    Next i
End Sub

This works because you have a repeating pattern to the collection and display of the 11.25 values and totals. You may find the .Offset() method easier to follow than the constructs. This is starting to perform a lot more calculations, but it shouldn't pose any unrealistic burden on a computer capable of running Office 2010 under Windows 7.

You've mentioned displaying the totals stating in C1 and in C3. The above formula uses C3 as the starting point.
