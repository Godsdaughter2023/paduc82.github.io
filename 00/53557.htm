myronh say:

OK, problem defined, solution coming. There are several options available to you, some cheap (or free) some not so much. All the solutions come with a varying degree of complication and may not be practical to your situation. I am going to list them from easiest to hardest. Only you can decide which is the best based on your resources.

1) Upgrade all of your 16 bit software to 32 bit versions. Depending on the number and type of applications this can easily get to be very expensive, but it’s by far the simplest solution. Some of your 16 bit applications may not have updated versions though. As well, some of the updated versions may not have the functionality you want. Therefore this solution, while simple, may be wildly impractical.

 

2) Reload your OS as 32 bit. If you upgraded, simply restore your computer to factory default and reload Windows 7 as 32 bit. If you purchased a computer with 64 bit pre-loaded, you will need to go to the manufacturer and see if they will provide a 32 bit disk. If you have more than 4 GB of RAM and want to keep using that extra RAM, or you have 64 bit applications you need to use, this is not a viable option. However, if you don’t have either of those problems this is a fairly simple solution.

 

3) If you have a full version of 32-bit XP, Vista, or even Windows 7, you can either dual boot or use Virtual Box to run a virtual environment for your 32 bit OS. Virtual Box is free, open source software so the cost for this solution is free. You can see the instructions for dual boot above and you can get Virtual Box and instructions on that at http://www.virtualbox.org/. If you do Virtualization you will essentially be running both OS’s at the same time. More resources, but easier to switch back and forth,.

 

4) If you don’t have another full version of a 32bit OS, you can use anytime upgrade (just time “anytime” in the search box and it’ll pop up for running) to upgrade to Professional or Ultimate. Professional is the more practical if you are fine with what is available on Home Premium because it’s much cheaper. Before doing this though, make sure you can support Hardware Virtualization. The link on my first response talking about XP mode includes a link to a Hardware Assisted Virtualization (HAV) detection tool. However, for simplicity here is the direct link to the tool: http://www.microsoft.com/downloads/details.aspx?FamilyID=0ee2a17f-8538-4619-8d1c-05d27e11adb2&displaylang=en. I just ran it on mine and it correctly detected my state in about 2 seconds (and I also run Home Premium). If you meet the hardware requirement, the total cost to get XP mode will be about $90 to upgrade to Professional. Everything else is free.

 

5) If you don’t meet the HAV requirement and don’t have a full version of any other OS, then you can buy one. Won’t be cheap, but it will likely be cheaper and more practical than upgrading all of your software. Once you have that, return to “3)” above.

 

6) This is by far the cheapest and easiest option, but it’s not first because it’s not a solution. Ditch the 16 bit applications. I know this sounds flippant and I apologize for that, but at some point you likely will have to anyway. I have a bunch of software that I sometimes miss, but can’t load anyway (even with all of the options above) because I don’t have floppy drives anymore. However, there are enough options above that you don’t need to do this unless the software isn’t worth the time an expense of running any of the above.

 

BTW, if any of your applications (or all of those 16 bit apps) are DOS, those solutions aren’t the most practical or cheapest (although running a virtual XP or dual booting DOS will probably work). The best bet in this case would be to run a DOS emulator such as DOSBox, which can be found at http://www.dosbox.com/

 

Good luck to you in your endeavors. One of these solutions should be fairly practical and easy for you to run and get all the applications you want running.
