peter atherton say:

I looked at this again and think that you want the start (Date & Time) in one cell and the End date and time in another so calculations can be easily made. This macro might be what you need. Follow TasosK directions for macro.

 

Sub ExtractStrtEnd()
  Dim c As Range
  Dim dStart As Variant
  Dim str As String
  Dim dEnd As Variant
  Dim vx As Variant
  Dim i As Integer
 
  str = "M"
  For Each c In Selection
    c = Trim(c)
    vx = Split(c, " ")

    dStart = vx(0) & " " & vx(2)
    dEnd = vx(4) & " " & vx(6)
   
    If Right(dStart, 1) = str Then
      dStart = Left(dStart, Len(dStart) - 2)
      If Right(dEnd, 1) = str Then
        dEnd = Left(dEnd, Len(dEnd) - 2)
      End If
    End If

    c.Offset(0, 1).Value = dStart
    c.Offset(0, 2) = dEnd
  Next c

End Sub

 

The values are placed in adjacent column so make sure you have two empty columns.

hansv mvp say:

Try this (test it on a copy of your document first):

 

Sub ReplaceEm()
    Dim p1 As Long, p2 As Long, p3 As Long
    Dim s As String
    Application.ScreenUpdating = False
    ' First, the article numbers
    Selection.HomeKey Unit:=wdStory
    With Selection.Find
        .ClearFormatting
        .Replacement.ClearFormatting
        .Wrap = wdFindStop
        .MatchWildcards = True
        .Text = "art. [0-9]{1,}/[0-9]{1,}"
        Do While .Execute
            p1 = Selection.Start
            s = Selection.Text
            p2 = InStr(s, "/")
            p3 = Len(s)
            ActiveDocument.Range(Start:=p1 + p2, End:=p1 + p3).Font.Superscript = True
            ActiveDocument.Range(Start:=p1 + p2 - 1, End:=p1 + p2).Delete
            Selection.Collapse Direction:=wdCollapseEnd
        Loop
    End With
    ' Then, the dates
    Selection.HomeKey Unit:=wdStory
    With Selection.Find
        .Wrap = wdFindStop
        .MatchWildcards = True
        .Text = "([0-9]{1,})(/)([0-9]{1,})"
        .Replacement.Text = "\1 din \3"
        .Execute Replace:=wdReplaceAll
    End With
    Application.ScreenUpdating = True
End Sub

Hi Nitin,

 

I’d like to explain that the OneDrive for Business is designed for personal use and the OneDrive for Business site is actually belongs to the user himself.

 

Regarding your requirement, we cannot directly set the OneDrive For Business site to read only mode. As a workaround, we can remove the user’s site administrator permission and grant another user (such as the Office 365 admin) the site administrator permission and change the user’s Full control permission to Read or other permissions we need. 1. Add another user’s site administrator permission and remove the user’s own site administrator permission.

2. Go to Site settings > Site permissions under Users and Permissions, change the user’s own permission level to Read.

doug robbins - mvp office apps & services (word) say:

There is no difference in the way templates\macros behave\are used between 2007 and 2013.

 

If templates are saved either in the User Templates Folder or the Workgroup Templates Folder, both of which are considered, or can be set as Trusted Locations via Options>Trust Centre>Trust Center Settings>Trusted Locations, then the templates will be trusted and macros in them should run.

 

Starting with Word\Office 2010 however a Protected View facility was introduced which allows potentially dangerous files to be opened, without security prompts, in a restricted mode to help minimize harm to the computer.  There are options to have that mode apply to:

 

Files originating from the Internet

Files located in potentially unsafe locations

Files received as attachments in Outlook.

 

While Protected View can be disabled on a global basis, for individual documents\templates, the "blocking" can be removed by accessing the Properties of the file in Windows Explorer and clicking on the Unblock button.

 

A decision on the best method for you to achieve what you are after would depend upon:

 

The type of documents that you want to have created from the forms\templates

The frequency of use that each recipient will make of the forms\templates

The relationship that you have with the recipients - that is, if you are working in a corporate environment and the recipients are fellow employees, or if there is only a casual relationship.
