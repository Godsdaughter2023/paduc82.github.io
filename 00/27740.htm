rohn007 say:

If you don't get the prompt to save changes to normal (when it is turned on) then it is possible the changes you made are no longer saved to normal. Some settings have been moved to the registry, and some to other places (also not NORMAL).

Although the "Change Default" buttons USUALLY work, I have never trusted them. I prefer to make my changes by explicitly opening NORMAL.DOT(M). That goes along with an extra security precaution I like to take, I also turn on the READ ONLY file attribute.That is in addition to the notify before saving changes to normal prompt.

I started doing it this way years ago after I had some unexpected software installed that caused problems. (It was in a corporate shop and the installation was done at the network level.)

So for me, saving updates to NORMAL is a very defined process"

  • shut down word
  • remove the read only file attribute on NORMAL
  • make a copy of NORMAL, so I can fall back to a known good copy with most of my customizations
  • restart word
  • make my changes
  • exit word, saving the changes
  • (optionally, make another backup copy of NORMAL)
  • turn on the read only file attribute on NORMAL
  • restart word

This way if I get a prompt to save normal I decide if I am aware of changes that need saving. If yes, I can save to an alternate name, then rename it .  The extra ReadOnly comes in handy when I'm in a hurry and OK through all of the saves without looking.

Granted, my approach doesn't work if you constantly save autotext/building blocks, but I don't, so no problem for me. Now that I think of it, I'm not sure what effect the prompt to save changes to normal has to saving changes to buildingblocks template ...
