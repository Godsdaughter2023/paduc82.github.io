jay freedman say:

Unfortunately, your terminology doesn't match up with anything used by Microsoft, so I'm guessing what kind of "text box" you're working with. It sounds like an ActiveX text box, although that doesn't require any protection of the document.

Look at the following picture and let us know which button you used to create the text boxes.

If they are indeed ActiveX text boxes, click the Design Mode button on the Developer tab. Right-click on one of the text boxes and click Properties to open the pane shown below, and double-click the AutoSize item to change its value from False to True. That will allow the box to expand downward (but not sideways) as more text is entered in it. Repeat setting the AutoSize item to True for the other text boxes. Finally, click the Design Mode button again to turn it off, and save the document.

jay freedman say:

So basically what you are telling me is that, there is no way to make auto fit text type sideways? 

That's generally correct; the box's width will stay at whatever size you (the creator of the form) set it to. The text in the box will wrap at that point, and the height of the box will increase to accept the next line of text.

I think there may be a way to use macro code to expand the box's width, but you would have to specify the conditions that would cause an expansion and at what point to stop expanding. The coding is likely to become complicated.

It would be better to design the form with the text boxes set to the maximum width that you want to allow.  Short entries will "waste" space, but that's less disturbing than boxes that grow and shrink.

At the top of the page list there should be an EDIT button. If you tap that you get another list view. Every page entry now has a "handle" (three horizontal lines) to the right. Touch and hold that handle to move pages to another list position. Press DONE at the end.
