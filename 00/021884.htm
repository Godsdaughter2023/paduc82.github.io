suzanne s. barnhill say:
And then, assuming that it does, use the Replace dialog to search for footnote references and apply the Footnote Reference style. To do this: 
  1. Press Ctrl+H to open the Replace dialog.
  2. In the "Find what" box, type ^f (which is the code for a footnote reference mark). 
  3. Click More>> to expand the dialog. 
  4. With the insertion point in the (empty) "Replace with" box, click Format, then Style, then Footnote Reference, and click OK. 
  5. Click Replace All.
You may need to do this once in the document body and once in the footnote story if references are incorrectly formatted in both.

john w. vinson say:

That's correct. It doesn't store the value in the table, nor should it! Storing a calculated value is rarely a good idea; it's redundant, it wastes space, and worst of all it risks inconsistancy (such as a DOB of 5/16/1946 and an Age of 2). Just do the calculation when you need to see it; the Age field should simply not exist in the table.

Note that this expression is not quite correct. DateDiff() counts the number of year (or other interval) boundaries, so if DOB were 12/31/2013 and you ran this on January 3, it would show an age of 1 for this four day old baby. A better expression is:

=DateDiff("yyyy", [DOB], Date()) - IIF(Format([DOB], "mmdd") > Format(Date(), "mmdd", 1, 0)

This will subtract 1 from the age if the person's birthday has not yet happened.
