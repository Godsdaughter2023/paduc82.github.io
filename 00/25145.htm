I believe I have found what could be the issue:

If you are working in any office file in Excel/Powerpoint that is linked to and saved on sharepoint (via autosave and with collaboration on) and you close the file, Onedrive will synchronise the file from sharepoint too. chitrahaas nalla msft say:

Hi Niek,

 

Thanks for writing back and letting us know your experience.

 

Based on our test, what I can conclude is, for suppose two users A and B open the file and works together. If user A make the changes to file with user B, if the user A completes the making changes and close the files before user B (as user B still need to edit some content). And user B makes some changes after user A closing his file, in the version history the last modified name will be the user B (because he is the last person to edit the file) and the version history treats the both users as one user.

 

For example, if the user A opens the files before the user B and make some changes before the user B open the files, then there will be the version history of user A (the time at which user B is doesn’t open the file) because the changes made by the user A is saved when user B is not opened the file.

 

Hope the above scenarios will clarify the problem and let us know if you have any queries.
