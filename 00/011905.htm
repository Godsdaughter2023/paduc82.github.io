Hi,

Windows Defender may crash or close instantly upon launch if the program itself is corrupted or outdated. For us better assist you, we'd like to ask the following:

  • Have you made any changes on your device prior to the issue? When did the issue begin?
  • Did you encounter any error messages after the crash?
  • What have you done so far to troubleshoot the issue?

Meanwhile, we suggest that you check for the latest Windows update on your device. To check for updates now, select the Start button, and then go to Settings>Update & security>Windows Update, and select Check for updates. If Windows Update says your device is up to date, you have all the updates that are currently available. Check if the issue persists.

If the issue persists after updating Windows, we suggest that you check for corrupted Windows system files. System File Checker (SFC) is a utility in Windows that scans for corrupted Windows system files. If it finds a problem, it will try to replace the problematic system files from a cached version that’s on your PC. If SFC can’t replace the files, the Deployment Image Service and Management Tool (DISM) can check the corrupted files then download and replace the files through Windows Update. To run the tools on your device, follow these steps:

  1. Open an elevated command prompt. To do this, right click the Windows button, and then select Command Prompt (Admin). If you are prompted for an administrator password or for a confirmation, type the password, or click Allow or Yes.
  2. Type the following command, and then press Enter.

DISM.exe /Online /Cleanup-image /Restorehealth

Note It might take several minutes for the command operation to be completed.

3. Type the following command, and then press Enter.

sfc /scannow

Note It might take several minutes for the command operation to be completed.

        4. Close the command prompt, and then run Windows Update again.

For more information about the SFC and DISM and how to run them, see ZigZag3143 (MS -MVP) System file check (SFC) Scan and Repair System Files & DISM to fix things SFC cannot.

macropod say:

There are various ways of doing this, including the use of sequential IF tests, as in:

{IF{MERGEFIELD Year} = 2012 {MERGEFIELD Year \* Charformat}} 

{IF{MERGEFIELD Year} = 2013 {MERGEFIELD Year \* Charformat}} 

{IF{MERGEFIELD Year} = 2014 {MERGEFIELD Year \* Charformat}}

where the M is formatted in the colour you require.

 

Another way is to use field maths & formatted numeric picture switches to control the output. In the following construction, 12 values (1-12) can be handled. The '3;1;2', '6;4;5', etc. strings show the positions of each of the 12 outputs, which you could format individually with the colour of your choice; in reality, you'd replace those digits with formatted 0s - or you could use formatted text strings, like "'March';'January';'February'", etc. (note the combination of single & double quotes).

{QUOTE{SET Val {=MOD({MergeField MyData \# 0}-1,12)+1}}{IF{=INT(Val/4)}= 0 {=Val-2 \# 3;1;2} {IF{=INT(Val/7)}= 0 {=Val-5 \# 6;4;5} {IF{=INT(Val/10)}= 0 {=Val-8 \# 9;7;8} {=Val-11 \# 12;10;11}}}}}

No, inserting the mergefield twice just gives you the start for creating the IF field. The first thing to do is to press Alt+F9 so that you are seeing the actual fields. So what you entered as «Year» is displayed as { MERGEFIELD Year }. Then you add text so that it looks like this:

{ IF { MERGEFIELD Year } = 2015 { MERGEFIELD Year \* Charformat } }
