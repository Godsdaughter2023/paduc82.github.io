   =SUMPRODUCT('macHD:Users:mike:Desktop:[log.xlsx]Sheet1'!$B:$B, --('macHD:Users:mike:Desktop:[log.xlsx]Sheet1'!$C:$C="y"), --('macHD:Users:mike:Desktop:[log.xlsx]Sheet1'!$D$D="yes"))

Message: "Insufficient memory. Save document now." - Microsoft Community

richarddunn2 say:

Message: "Insufficient memory. Save document now."

Running Word 2011 for Mac on a 2012 Mac Mini and OS 10.12.6 Sierra.  I am working on a document that is about 150 pages long.  I just now got this message about "insufficient memory."

Can I make memory larger?  How?

Or should I break up the document into Chapters?

bob jones aka: cybertaz say:

In OS X, memory management is done by the operating system. A user cannot allocate memory to individual programs.

How much Random Access Memory [RAM] does the Mac currently have? You may be able to add more physical RAM to your Mac, but it depends on the model & its currently installed amount. IIRC, 2012 Mini came standard with 4 GB, expandable to 8 GB. IMO, however, even 8 GB is a bare minimum based on the requirements of the OS X version & Office version you're running.

The number of 'pages' usually is not an issue... Pagination is arbitrary based on a number of variables. It's a matter of what is contained in the document. Word readily can handle properly created documents with reasonable graphic content of 1500-2000 pages or more. Graphic objects - especially hi-res images - are typically major memory hogs, but it's usually system limitations which produce the issue.

Without adding more RAM some suggestions for conserving memory:

  • Before launching Word, Quit all programs & processes that are not required (including browser, email, messaging, etc.)
  • Open only the one document
  • Save frequently
  • Restrict the number of graphic objects & effects if possible
  • Avoid hi-res images [Link rather than embed if needed for commercial printing]
  • Make sure to use Styles for formatting rather than applying direct formatting changes
  • Minimize the use of such features as Track Changes

The message also could be referring to limited free space [storage capacity, not memory] on the drive. That is managed by the OS as well, but it helps to:

  • Delete any unneeded files (old email, downloads, etc.)
  • Remove any unused programs
  • Move files to other storage devices if they don't need to be accessed regularly
  • Replace the HD with one of larger capacity
  • Invest in an external HD
  • Take advantage of cloud storage for your user files

In Word, you can start creating macros in Normal.dotm immediately:

  • Press Alt+F8 to activate the Macros dialog.
  • Enter a name for the macro.
  • Make sure that "Normal.dotm (global template)" is selected in the "Macros in" dropdown.
  • Click Create.

 

In Excel, you have to create the personal macro workbook first:

  • Click the Record Macro button on the status bar (next to the word Ready).
  • Enter a name for the macro.
  • Select "Personal Macro Workbook" from the "Store macro in" dropdown.
  • Click OK.
  • Record a few actions, it doesn't really matter what. For example, make the active cell bold.
  • Then click the button in the task bar (it has now become a Stop Recording button).
  • Press Alt+F11 to activate the Visual Basic Editor.
  • You should see VBAProject (PERSONAL.XLSB) on the left hand side.
  • Expand it, then expand Modules.
  • Double-click the module you see there (probably Module1).
  • You can now edit the existing macro and create new ones.
  • When you quit Excel, you'll be asked to save changes to your personal macro workbook. Make sure that you answer Yes!
