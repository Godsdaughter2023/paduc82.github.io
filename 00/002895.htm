jllatham say:

Russell, So as not to get your hopes up with this reply, I don't think I have a solution for you, just some more discussion.  You've already covered the only idea that immediately came to mind: deleting all conditional formatting in the copy before beginning the copying of formatting from the master book.

I agree that it appears you're being "let down by Excel's implementation".  When I did the little testing to come up with the code I posted earlier, one thing I noticed was that the two conditions I'd set up in the test workbook were reported in reverse order of application!  Now that wouldn't be troublesome if all areas had just one set of conditions, but with multiple conditions and overlapping areas, it probably does create some issues in your effort.

I suppose that one possible solution, that would take some effort to set up initially, would be to go ahead and use Andreas' solution as the basis for getting basic formatting for the sheets set up, although that might not be needed if your copy is based on an original that already has basic formatting set up.  But if the 3rd party tool is not even copying basic formatting, it would be a quick way to get over that part of the effort.

For the second part of the setup of the copy, getting conditional formatting in place, you might consider simply recording a macro as you manually apply the needed conditional formatting of ranges on the worksheets in the proper order.  At this point you'd have a couple of options: once the copy of the workbook has been made, simply run that macro to fairly quickly apply the needed conditional formatting; or, use the code the macro generates as the 'spec' to rewrite it within your custom code.

If your 3rd party tool does not copy code modules also, there are a couple of ways to attack the issue so you could use the recorded macro:
1) adapt the recorded macro so that it would work from a copy of the workbook with the code in it to apply the changes to the copy of the workbook made with your 3rd party application, or
2) once the macro has been recorded and tested, export the code module as a .bas file and then when you make a new workbook using the 3rd party app, import the .bas module into it long enough to run the recorded macro - you could then even delete the code module since it would presumably only be needed for the one-time operation.

It's that second part, manually recording the macro, that's going to be somewhat labor intensive.  You'll probably want to sit down and go through the process once or twice, making detailed notes of the steps involved so that you'll have a script to follow when you finally record the macro for use.  But at least once you got the macro right, you don't have to futz with it any more.

How do I remove contacts - Microsoft Community

2mowgli2 say:

How do I remove contacts

Simple action and simple question. Incredible not to have an answer.
marites_p say:

Hi,

To remove your contacts, kindly follow the steps below:

For Classic Skype version (7.xx):

  • Sign into Skype.
  • In your contact list, find the contact you want to remove. Right-click the contact name and select Remove from Contacts. Then confirm by clicking Remove.
  • The contact is removed from your contact list but will still appear in the Recent tab with a question mark. Right-click the contact name to see the options: Add to Contacts or Block This Person.

For Skype for Windows 10 (12.xx):  

  • Sign in to Skype for Windows 10.
  • In your contact list, find the contact you want to remove. Right-click the contact name and select View Profile > x Remove Contact.

 For New Skype (8.xx):

  • Sign in to Skype.
  • Find the person's profile you want to remove/delete.
  • Right-click their name, and then select View profile. Alternatively, from within a chat, tap the chat header to view their profile. 
  • Select to delete this person from your Skype contacts. 

Should you have further questions, do not hesitate to contact us back.
