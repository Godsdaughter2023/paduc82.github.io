jllatham say:

That's fairly easy and you can take care of it in the UserForm_Initialize() routine - but first we need to 'move' our variable cellChosen

Delete the line of code where it is declared:

Dim cellChosen As Range

Then put this declaration in the same code module at the top ahead of the any Sub or Function declaration:

Public cellChosen As Range

This makes it a variable that can be 'seen' and used by any piece of code in your project, most importantly by your UserForm code.

In your UserForm_Initialize() routine you are going to have a bunch of statements that get the information from the row pointed to by cellChosen something like this:

You'll eventually want code to move the revised data back onto the worksheet, and I've shown a sample of that also.

Private Sub UserForm_Initialize()
  Dim whatRow As Long
 
  'cellChosen is Public variable
  whatRow = cellChosen.Row ' so we only interpret it 1 time
  'move data from the worksheet into text boxes
  'on the userform:
  With ActiveSheet
    Me.TextBox1 = .Range("A" & whatRow)
    Me.TextBox2 = .Range("G" & whatRow)
    Me.TextBox3 = .Range("H" & whatRow)
  End With
End Sub

Private Sub CommandButton1_Click()
  'this is for the button set up to save the data
  Dim whatRow As Long
 
  'cellChosen is Public variable
  whatRow = cellChosen.Row ' so we only interpret it 1 time
  'move data from the userform text boxes
  'back onto the userform:
  With ActiveSheet
    .Range("A" & whatRow) = Me.TextBox1
    .Range("G" & whatRow) = Me.TextBox2
    .Range("H" & whatRow) = Me.TextBox3
  End With
  'and if you want to close the form
  Unload Me  ' must use Unload and not Me.Hide so that Initialize is run again next time you use the form
End Sub

jay freedman say:
First, take a deep breath.
 
Although you were using Word, this question doesn't really have anything to do with Word, but instead with Windows.
  • Start up "My Computer" (a stupid name for the file manager program).
  • Click the Tools menu and choose Folder Options.
  • Click the View tab of the dialog.
  • Click the item "Show hidden files and folders".
  • Click OK.
Now you can see all the hidden folders, including the one you hid. Right-click the icon for that folder, choose Properties, and uncheck the Hidden box.

With that setting, you'll also be able to see any hidden files inside the folder, but there probably aren't any.

If the Word program had the option enabled to save AutoRecover information every so many minutes, and if your document was open long enough for the Auto save to happen, then you should see the recovered document when you restart Word. If that doesn't happen, then I'm afraid your document has just evaporated.

hi patrick,

here are the answers to your questions:

if we have domains abc.co.uk and xyz.co.uk and have these set up in o365 under 'domains' and apply the necessary records in the respective dns zones will exchange accept mail for either domain?

if all the relevant dns records are added such as mx record and the user is assigned with an exchange online license, exchange online will accept mail for either domain.

 

if i then have a user 'jane'. in her profile i assign two email addresses *** Email address is removed for privacy *** and *** Email address is removed for privacy *** will jane be able to receive messages sent to both addresses?

the answer is yes. for example, if jane’s primary email address is *** Email address is removed for privacy ***, you can set *** Email address is removed for privacy *** as her secondary email address, so she can receive emails from both of the accounts. to do it, please sign in to portal (https://portal.office.com) with the admin account, navigate to users – active users, double click the mailbox. under email address, click add new and add the email address *** Email address is removed for privacy ***. click save at last.

 

i understand that users can only send emails addressed from their 'primary' email address but does this mean that i could have some users that have abc.co.uk as their primary address and other users with xyz.co.uk as their primary address.

the answer is yes. you can set the user’s primary email with whatever domain you want. when you first create the user, you can select the domain you want. a screenshot for your reference:

 

you can also change the user’s primary address after it is created. just go to users – active users, double click the mailbox. under details, you can change the user’s primary emails address.
