If the file actually is saved as file type Excel 2004 XML Spreadsheet or XML Spreadsheet 2003, there should be no problem. Files of that type use a built-in standardized schema supported in all Mac/PC versions from [IIRC] Office 2003 forward.

Publisher produces fixed width, static websites which limits your options. You can consider keeping the page width somewhat narrow to minimize the horizontal scrolling and use a default font size of 12 or larger which those of us with old eyes appreciate in regular sites.

 

You could create another website specifically designed in terms of width and font size for smartphones, and provide a link to that site from the home page. You would just upload that site to a subfolder and link to it from your main homepage. We can talk specifics if you want, but here is an example of a website that offers an English and a Greek version. MSFT has made Expression Web a freebie now. It requires you learn basic CSS and HTML coding but it will give you the ability to code your site for smart phones or big screens. You can code your pages to automatically render a different page if viewed with a smartphone.
