lemp say:

Yes, that's exactly what I suspected.

The "problem computer" is not obtaining a valid IP address from your wireless router.  Because DHCP is enabled for your computer's wireless network adapter (which is the default) and because the other computers on your network are able to obtain valid IP address from the router and successfully connect to the Internet, the problem is almost certainly that the "problem computer" is failing to authenticate to the router.

That you "configured the router for a long time" is beside the point.  The problem is not in the router but in the computer.  For some reason, the computer is no longer presenting the proper security credentials to the router.

The easiest way to fix this is to delete the saved wireless profile on the computer and then connect again, being sure to enter the correct password.

1.  Open the Network and Sharing Center (right-click on the wifi link in the Notification Area (near the clock display) and select "Open Network and Sharing Center")

2.  In the left pane of the Network and Sharing Center, click on the link to "Manage Wireless Networks"

3.  Select the connection to your own router and click "Remove" (or right-click the connection and select "Remove network"

4.  Click OK in the "Manage Wireless Networks - Warning" dialog box

5.  Go back to the main Network and Sharing Center and click on the link in the left pane to "Change adapter settings"

6.  Right-click on "Wireless Network Connection 2" and click "Connect/Disconnect"

7.  Click the refresh button to view available wireless networks. Double-click your network or select it and then click the Connect button.

8.  Enter the password carefully.

Summary of above with pictures:  http://www.tp-link.com/us/faq-281.html

Firstly, try running the System File Checker in case it can repair any corruptions:

- Right-click Command Prompt and run it as administrator.
- Type sfc /scannow
- Press Enter.

- Click Start and type "cmd".
- Right-click Command Prompt and run it as administrator.
- Type "D:" (without quotes) and press Enter. If your external drive isn't D, change it to the appropriate letter.
- Type "chkdsk /x /f /r" (without quotes) and press Enter.

It'll ask you to reboot so it can check the disk. Press Y and your computer will reboot, check your disk and repair any errors.

frederik long say: You need to be aware of a couple of points:
  • Installing Windows by using the downloaded file will completely wipe your existing installation. You would lose all programs and settings.
  • Storing back-up data on CD/DVDs is not a good idea. Because of "DVD rot", it is likely that some of your DVDs will become unreadable after a year or two.
Here is what you need to do:
  • The low-cost method: 
    - Purchase a 2.5" disk in a USB case.
    - Create an image of your installation via the Control Panel / Backup and Restore.
    - Burn a Windows Repair CD.
    - Boot the machine with this CD and check if you can see the image.
    This method usually works but it is temperamental and can fail you when you most need it.
  • The reliable method:
    - Purchase a 2.5" disk in a USB case.
    - Purchase an image creation tool, e.g. TrueImage from Acronis.
    - Create an image of your installation with TrueImage.
    - Burn a rescue CD.
    - Boot the machine with this CD and check if you can see the image.
    TrueImage is far more reliable and flexible than the native image creation tool.
Neither of these tools will have any effect on the hidden recovery partition.

How to copy a Library from one user account to another in Windows 7 - Microsoft Community

mikeosborn_844 say:

How to copy a Library from one user account to another in Windows 7

In order to solve an problem in installing a new printer, the only solution Microsoft Support could find was to create another user account. They certainly intended to copy all the Libraries to that account - My Documents is certainly there - but My Pictures is not. Would welcome advice on how to correct this. My intention is to delete the original user account once I am satisfied I will not be losing anything.

MTIA

lemp say:

Log into an account with "administrator" privileges.  You'll be able to access both the old account's files and your new account's files.

It's not clear if you're saying that your new account has no "My Pictures" folder or if the My Pictures folder in your new account is empty.

If your new account does NOT have My Pictures folder, then do the following:


  1. Click the Start Orb and then click Computer.
  2. Navigate to C:\Users\<old account name>
  3. You should see something like the following (you probably won't see all of these folders):
  4. Click once on My Pictures to select it
  5. Click the Edit menu and then click "Copy to folder"
  6. In the "Copy items" window that pops up, navigate to C:\Users\<new account name> and select that so that your the folder with your user name shows in the "Folder" window
  7. Click the "Copy" button
  8. After the copy operation has finished, confirm that your new account now has a My Pictures folder and that all of your pictures are in it.  Then go back and delete the contents of the My Pictures folder in your old account.

If your new account DOES have a My Pictures folder, but that folder is empty:

  1. Click the Start Orb and then click Computer
  2. Navigate to C:\Users\<old account name>\My Pictures
  3. Click somewhere in the right-pane of Windows Explorer (where the picture files/folders are) and then click Edit > Select All (or press CTRL+A) and Edit > Copy (or press CTRL+C)
  4. Navigate to C:\Users\<new account name>\My Pictures
  5. Click somewhere in the right-pane of Windows Explorer (the empty window) and then click Edit > Paste (or press CTRL+V)
  6. After you confirm that all of your pictures are in the My Pictures folder of your new account, go back and delete the contents of the My Pictures folder in your old account.
