andreas killer say:
So my question is, is it possible to create “private” instance of Excel that will not be used by Windows Explorer or hyperlinks to open other Excel files?
No.

If you double click on a file in the windows explorer, the file is always opened in the last created instance.

I have a similar problem with an Excel file, that special file creates a tray icon, hides the instance and if you click on the tray icon, a menu is shown.

So if anyone opens an Excel file, my special instance becomes visible and finally crashes.

There are 2 possible ways, first the easiest. Add a class module to your PP-AddIn and take over the application events from Excel, especially the Workbook_Open event.

If a workbook is opened, your instance becomes visible, there is nothing to prevent that. But then hide your instance, close the file immediately and open it in a new instance of Excel again.

This code in a regular module in PP:

--- schnipp ---
Option Explicit

Dim MyExcel As New MyExcelhandler

Sub Init()
  Set MyExcel.App = CreateObject("Excel.Application")
End Sub
--- schnapp ---

And this code in a class module, named "MyExcelhandler".

--- schnipp ---
Option Explicit

Public WithEvents App As Excel.Application

Private Sub App_WorkbookOpen(ByVal Wb As Excel.Workbook)
  Dim FName As String
  'Get the filename from that workbook
  FName = Wb.FullName
  'Close it
  Wb.Close
  'Hide our instance again
  App.Visible = False
  'Open the file in a new instance
  'Note: The SHELL command forces Excel to load AddIns etc.
  Shell "excel.exe """ & FName & """"
End Sub
--- schnapp ---

This works, but is not very nice, because the process is slow, the file is indeed loaded 2 times.

The other way is more complicated, you must create 2 instances of Excel (first one is hidden, second one is visible and has all AddIns loaded). Then take over the events of the second (visible) instance. Now you must detect if that application tries to close and if so, reopen it hidden again.

Both solutions have a lot of effort for slightly effect only. I recommend to close the instance, as Graham has already shown.

Andreas.

        On Error GoTo 0
        If lnXlBuild = 0 Then

            Set xl = Nothing
        End If
    End If
   
    If Not wb Is Nothing Then 'Check if Workbook was closed some how
        On Error Resume Next
        strWbName = wb.Name
        On Error GoTo 0
        If strWbName = vbNullString Then
            Set wb = Nothing
        End If
    End If

        Set xl = CreateObject("Excel.Application")
        xl.IgnoreRemoteRequests = True
        xl.ScreenUpdating = False
        xl.DisplayAlerts = False
    End If

The global variable "MyExcel" keeps the instance of Excel alive.

MyFieldIdDate: IsDate([MyField])
