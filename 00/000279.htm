Well, perhaps this in C33:  =--TEXT(MOD($C$3-B33,1),"hh:mm").

The need for MOD depends on assumptions about the value in C3.

I assumed that C3 contains a date as well as time.  In that case, MOD is not needed.

But if C3 contains only time, MOD is one way to avoid negative time in the examples of 00:00 - 03:00 (subject line) and 00:00 - 04:00 (description).

In either case, what you wrote above might seem to work only by coincidence, depending on the value in C3.

There should be no difference in the binary representation of 01:00 and --TEXT("01:00","hh:mm").  So using --TEXT(...) as you did in B33 is unnecessary, and it serves no useful purpose.

As for the need for --TEXT(...) in C33, consider the following example, each subsequent step taken in order; that is, each step depends on all the previous steps.

A8:  00:00
B8:  07:00
C8:  =MOD(B8-A8,1)
D8:  17:00
E8:  =MATCH(C8,D8,0)

1. E8 display the #N/A error, indicating no exact match.
2. If we change A8 to =TODAY() (00:00 is implied), E8 still displays #N/A.
3. If we then change C8 to =--TEXT(B8-A8,"hh:mm"), E8 displays 1, indicating an exact match.
4. If we then change A8 back to 00:00, C8 displays the #VALUE error because B8-A8 is negative time.
5. If we then change C8 to =--TEXT(MOD(B8-A8,1),"hh:mm"), E8 displays 1, indicating an exact match.

Compare the results in E8 in step #5 with step #1.

In your original posting, you said you only need to subtract 01:00 through 04:00 (in B8).  In that case, concidentally, MOD(...) is sufficient in C8 when A8 is 00:00.  But if A8 is 03:00 and B8 is 02:00, the problem with E8 returning #N/A (not an exact match) reappears.  Again, --TEXT(...) corrects the problem.

Hi,

 

Try this ARRAY formula and see below on how to enter it.

 

=INDEX(Sheet2!B1:B20,MATCH(TRUE,EXACT(A1,Sheet2!A1:A20),0))

 

The equivalent VLOOKUP that isn't case sensitive would be

=VLOOKUP(A1,Sheet2!A1:B20,2,FALSE)

 

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

peter atherton say:

Searches in text are not usually case dependent so ap = AP. Try a helper column with the formula

=CODE(A2)+CODE(RIGHT(A2))

and use this formula

=LOOKUP(CODE(D2)+CODE(RIGHT(D2)),B2:B5,A2:A5)

 

Lookup can use columns to the right of a lookup value. The abve just returns the code.

 

Note: this is of no use if codes include 'aP or Ap as the sum of codes is the same in each case.

steven. s say:

Hey MarinaPincus,

 

Try changing your window color to a setting that is brighter in color. The window color ties into a fade on the coloring of the tabs. You can change these settings here: (assuming you have Windows 7)

  • Open Internet Explorer
  • Open a few tabs (both as color organized groups and by using the new tab button for the default grey color)
  • Move the Internet Explorer window to one side of the screen
  • Right click your desktop
  • Click Personalize
  • Click Window Color (toward the bottom of the window)
  • Click Show color mixer
  • Adjust the color settings (especially Brightness) to find a setting that allows you to easily see the text displayed in the tabs

 

Note: I find it easier to read windows in general with the "Enable transparency" feature disabled or set toward the right side of the scale, but that is a matter of preference.

 

Hope that helps, look forward to hearing back from you.
