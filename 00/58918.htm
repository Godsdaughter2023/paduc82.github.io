The save game could have been corrupted console side.  There are many ways this could happen, and the vast majority of them are preventable or would be caused by bugs.  The only ones that wouldn't is user turning off the console while saving (I didn't, but this shouldn't corrupt a cloud saved game anyway) and physical hardware failure (my hardware seems ok, but that's not always obvious).

When the file is uploaded to cloud saved games the system can create a hash of the file for CRC (en.wikipedia.org/.../Cyclic_redundancy_check).  It sends the hash along with the file and the server can then verify the file's integrity before overwriting the currently stored save game.

In addition, the data is undoubtedly sent over TCP which has built in error detection (en.wikipedia.org/.../Transmission_Control_Protocol).

This is how data is protected during transfer.  It's used everywhere.  I'm sure Microsoft employs these practices in cloud saved games too.  It's just somewhere along the line, something went wrong, and that something is almost definitely a bug.  Please don't make assumptions before understanding the underlying technologies completely.
