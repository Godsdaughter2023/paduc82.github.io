al jarvi say: Use Remote Desktop [RDC]...

Since you already have it running internally on your work LAN the next step is to forward/open TCP Port 3389 through any firewall/router the RDC host PC is behind. You then call from a remote location using the public IP of the router. If your business router is assigned a static IP from your ISP use that. If not look at using a fully qualified domain name [FQDN] from a source like No-IP.com or DynDNS or similar service. The way it works is you install as small program on one of your PCs, ie. the RDC host for example. The program contacts the No-IP.com servers on a time scheduled basis. The servers then map your FQDN to your current IP and propogate that information over the public internet. You call via the FQDN. Some routers include built-in support for DynDNS or No-IP.com. Check the users manual for help with that.

Here is a general help page for XP and Vista...

http://theillustratednetwork.mvps.org/RemoteDesktop/RemoteDesktopSetupandTroubleshooting.html

http://theillustratednetwork.mvps.org/RemoteDesktop/RDP6ConfigRecommendations.html

You can access multiple PCs with RDC if you wish...

http://theillustratednetwork.mvps.org/RemoteDesktop/Multiple_PC_RD.html

Some folks also do that via a VPN or SSH tunnel, ie. multiple PC access using RDC by only opening one hole in your firewall/router.

Keep in mind some DSL modems are also routers. You may need to contact your ISP for help forwarding/opening TCP Port 3389 through the device. Who is your ISP? Do you also run a router behind the DSL modem?
