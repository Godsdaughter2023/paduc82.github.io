charles kenyon say:

See Style Separator.

As you can see in the Reveal Formatting Pane, the style separator puts two Word paragraphs into one logical paragraph. Those two can have different paragraph styles, one of which shows up in the table of contents. The formatting for the non-heading style is shown in the Reveal Formatting Pane (Shift+F1).

suzanne s. barnhill say:
You have to break the heading into two paragraphs and apply a different style to the second paragraph (it should be a nonheading style (with an outline level of Body Text) but can otherwise be formatted the same as the heading. Then click in the first paragraph and press Alt+Ctrl+Enter to insert a style separator. The paragraphs will appear to become one, but only the first part will be in the TOC.

I used External Import with the macro recorder on and modified the code slightly:

 

Sub Macro1()
'
' Macro1 Macro
'

Dim r As Range
Set r = Range("A" & Cells(Rows.Count, "A").End(xlUp).Row + 1)
    With ActiveSheet.QueryTables.Add(Connection:= _
        "TEXT;C:\Users\James\My Documents\My Spreadsheets\Book1.csv", Destination:=r)
        .Name = "Book1"
        .FieldNames = True
        .RowNumbers = False
        .FillAdjacentFormulas = False
        .PreserveFormatting = True
        .RefreshOnFileOpen = False
        .RefreshStyle = xlInsertDeleteCells
        .SavePassword = False
        .SaveData = True
        .AdjustColumnWidth = True
        .RefreshPeriod = 0
        .TextFilePromptOnRefresh = False
        .TextFilePlatform = 437
        .TextFileStartRow = 1
        .TextFileParseType = xlDelimited
        .TextFileTextQualifier = xlTextQualifierDoubleQuote
        .TextFileConsecutiveDelimiter = False
        .TextFileTabDelimiter = False
        .TextFileSemicolonDelimiter = False
        .TextFileCommaDelimiter = True
        .TextFileSpaceDelimiter = False
        .TextFileColumnDataTypes = Array(1)
        .TextFileTrailingMinusNumbers = True
        .Refresh BackgroundQuery:=False
    End With
End Sub

 

It puts the data after the last value in column A

Wordpad can save documents in RTF format (with the extension DOC or RTF) or three flavours of plain text.

It cannot open Word 2007 DOCX format documents
Word 6 DOC format is RTF so Wordpad shouldn't need any additional filter to handle that format. It can do so natively.
If you are opening RTF files and they are not opening correctly, it seems they may be corrupt.

The first question is why are you adding a record with default values? Using placeholder data is not recommended.

But if that's what you need to do, don't do it the way you are doing it. Create a named query that appends the default record to the table. Then run that query from the macro. That will add the record to the table.

INSERT INTO table (field1, field2) SELECT 'data1' AS field1, 'data2' AS field2 FROM table.
