Let's say that your table is named tblData and that the treeview control is named TreeView0.

You could use the following code in the On Load event of the form with the treeview control:

 

Private Sub Form_Load()
  Dim dbs As DAO.Database
  Dim rst1 As DAO.Recordset
  Dim rst2 As DAO.Recordset
  Dim nod As MSComctlLib.Node
  Set dbs = CurrentDb
  Set rst1 = dbs.OpenRecordset("SELECT DISTINCT ClassNumber FROM tblData", dbOpenForwardOnly)
  Do While Not rst1.EOF
    Set nod = Me.TreeView0.Nodes.Add(Key:="C" & rst1!ClassNumber, _
      Text:="Class " & rst1!ClassNumber)
    Set rst2 = dbs.OpenRecordset("SELECT ID, FullName FROM tblData WHERE ClassNumber=" & _
      rst1!ClassNumber & " ORDER BY FullName", dbOpenForwardOnly)
    Do While Not rst2.EOF
      Me.TreeView0.Nodes.Add Relative:=nod, Relationship:=tvwChild, _
        Key:="M" & rst2!ID, Text:=rst2!FullName
      rst2.MoveNext
    Loop
    rst2.Close
    rst1.MoveNext
  Loop
  Set rst2 = Nothing
  rst1.Close
  Set rst1 = Nothing
  Set dbs = Nothing

Is it possible to introduce a drag and drop into your codes so I could move students from class to class, or is this very complex?  This is another thing I have been working on?

 


Right-click the treeview control (in design view) and select TreeCtrl Object > Properties from the context menu.

Set the OLEDragMode property to 1 - ccOLEDragAutomatic and the OLEDropMode property to 1 - ccOLEDropManual.

Click OK.

 

Create the following event procedures in the form module (note: these events are not listed in the Property Sheet, you have to create the code in the Visual Basic Editor).

 

' -----------

Private Sub TreeView0_OLEDragDrop(Data As Object, Effect As Long, _
        Button As Integer, Shift As Integer, x As Single, y As Single)
    Dim strKey As String, strText As String
    Dim nodDragged As Node, nodClass As Node
    Dim lngClass As Long, lngID As Long

    On Error GoTo ErrHandler

    With Me.TreeView0.Object
        If Not .SelectedItem Is Nothing Then
            'Reference the selected node as the one being dragged.
            Set nodDragged = .SelectedItem
            If Left(nodDragged.Key, 1) = "M" Then
                If Not .DropHighlight Is Nothing Then
                    Set nodClass = .DropHighlight
                    If Left(nodClass.Key, 1) = "M" Then
                        Set nodClass = nodClass.Parent
                    End If
                    If nodDragged.Parent.Index <> nodClass.Index Then
                        Set nodDragged.Parent = nodClass
                        lngClass = Mid(nodClass.Key, 2)
                        lngID = Mid(nodDragged.Key, 2)
                        'Set the drop target as the selected node's parent.
                        'Locate the record in the Employees table and update it.
                        CurrentDb.Execute "UPDATE tblData SET ClassNumber=" & _
                            lngClass & " WHERE ID=" & lngID, dbFailOnError
                    End If
                End If
            End If
        End If
        'Deselect the node
        Set nodDragged = Nothing
        'Unhighlight the nodes.
        Set .DropHighlight = Nothing
    End With
    Exit Sub

ErrHandler:
    MsgBox "An error occurred while trying to move the node.  " & _
        "Please try again." & vbCrLf & Err.Description, vbExclamation
End Sub

Private Sub TreeView0_OLEDragOver(Data As Object, Effect As Long, _
        Button As Integer, Shift As Integer, _
        x As Single, y As Single, State As Integer)

Display "No Records Found" when particular data is not found - Microsoft Community

dextermum say:

Display "No Records Found" when particular data is not found

I currently have a report with several subreports linked to the person's ID. Since they are academics, they may have publications but it might not be consistent every year (1 subreport being their publications).

In the scenario where Academic A has publications in 2016 but not in 2017, how do I prompt a message indicating "No Records Found" rather than the subreport just skipping the year entirely (subreport is grouped by publication year). I don't want to resort to brute force and creating fake publications where I link the academics to them because it would be a nightmare to remove. 

ken sheridan say: I notice that you have a column named PubYear which suggests that it is an integer number data type, whereas my example has a column PublicationDate of Date/Time data type.  This is why the YEAR function is called to return the year of the date.  This won't raise an error as, in Access, the date/time data type is implemented as a 64 bit floating point number.  The number 2017 for instance represents 9 July 1905, because that date is 2017 days after 30 December 1899, which is 'day zero' in Access's date/time implementation.

The column in your tblYearCalendar is named CalDate.  That would suggest that it is a date/time data type.  In my example it is CalYear and is an integer number data type with values like 2015, 2016 etc.

Assuming PubYear and CalDate  are numbers such as 2015 etc therefore, the YEAR function call should again be omitted:

    AND tblPublication.PubYear = tblYearCalendar.CalDate)

Similarly in the first part of the UNION the JOIN criterion should omit the YEAR function call:

    ON tblYearCalendar.CalDate = tblPublication.PubYear

ken sheridan say: Thinking a little more about your problem it occurred to me that this can be done more simply with a UNION operation in which the first part returns the authors and their publications for each year in which they have published, and the second part returns the year, the authors and the "No Records Found" string for those years in which an author has not published.  The latter is achieved by the use of the NOT EXISTS predicate and a subquery.

The query would thus be along these lines:

SELECT CalYear, Authors.AuthorID, PublicationTitle, PublicationDate
FROM YearCalendar
INNER JOIN (Publications
INNER JOIN (AuthorPublications
INNER JOIN Authors
ON AuthorPublications.AuthorID = Authors.AuthorID)
ON Publications.PublicationID = AuthorPublications.PublicationID)
ON YearCalendar.CalYear = YEAR(Publications.PublicationDate)
WHERE CalYear BETWEEN 2015 AND 2017
UNION
SELECT CalYear, AuthorID, "No Records Found", NULL
FROM YearCalendar,Authors
WHERE CalYear BETWEEN 2015 AND 2017
AND NOT EXISTS
    (SELECT * FROM AuthorPublications INNER JOIN Publications
     ON AuthorPublications.PublicationID = Publications.PublicationID
     WHERE AuthorPublications.AuthorID = Authors.AuthorID
     AND YEAR(Publications.PublicationDate) = YearCalendar.CalYear);

With this query as its RecordSource the subreport can be linked to the parent report on AuthorID, and controls bound to the columns in the usual way, with no need to use an expression in an unbound control.
