Language NEDERLANDS for Word 2016 on MAC - Microsoft Community

jan_de_boer say:

Language NEDERLANDS for Word 2016 on MAC

My Word 2016 (for Mac) version does not find DUTCH words in the dictionary. Everything is red-underlined. Why?

bob jones aka: cybertaz say:

Spelling & Grammar Checking is based on the Language attribute applied to the text in the document... IOW, you can type Dutch words perfectly, but if the Language attribute is German the words will be flagged as misspelled. In order for the process to use the Dutch proofing tools the text must be identified as Dutch.

Select the text, then go to Tools> Language & select Dutch. The better approach is to include the preferred Language attribute in the Styles used to format the document.

For an in-depth explanation of how the mechanism works you may want to review this article:

Spelling, Language, Dictionary

jan_de_boer say:
Thanks.
But that I have to do for every document. I’am looking for an setting that all my document are checked in Dutch default. Every new document comes up in English(VS).

bob jones aka: cybertaz say:

In order to change the Default Language for new documents based on Normal.dotm;

  1. Launch Word
  2. Go to Tools> Language
  3. Select Dutch
  4. Click the Default button
  5. Confirm update of the Normal template

Note that this will have absolutely no influence on existing documents. As I wrote, Language is an attribute of the text, comparable to Bold, Italic, etc. It is not a 'per document' setting or a program preference for anything other than new documents you create.. Each existing document whose text is in the 'wrong' language will have to be changed individually... That's why modifying the Styles used usually is the preferable way to go, especially if the document contains a mixture languages.

Also, if you generate documents from templates other than Normal.dotm you'll need to specify the Language in those templates as well.

doug robbins - mvp office apps & services (word) say:

Try these steps posted by Vinay G in response to a similar problem:

Try the steps below and check if it helps:

Step 1:**Clear Print spooler

=========

  1. In Windows Vista/Windows 7, click*Start*, type *Command*, Right click *Command Prompt* and select*Run as Administrator*.
    In Windows XP, click*Start > Run*, type *command prompt* and press *Enter*.
  2. Type the following commands and press*Enter* one by one:
    net stop spooler
    del %systemroot%\System32\spool\printers\* /Q
    net start spooler
    Check again if we can print. If not, go to next step.

*Step 2: Print by starting the Office program in safe mode*

==============

  1. Click *Start*, point to *All Programs*, and then point to *Microsoft Office*.
  2. Press and hold the*CTRL* key, and then click the Office program you want to start.
  3. Create a new document or open an existing document in the program and try to print again.

If the problem does not occur in the safe mode, this issue might be related to some third-party add-ins in the Office program, we can try to disable them. Normally, you could do the following to disable the conflict add-ins in your Office program:

*Disable add-ins*

***************************

  1. Click on office button, click word Options >  Add-in, click *Go* button in the Manage: Com-in Add*.*
  2. Check if there are any add-ins, clear the checkbox to disable them.
  3. Close the Office program and restart it.
  4. Add one check back each time to the list of Add-In,restart the Office program, and repeat the above procedure. Once the issue reappears again, we can determine which add-in causes this problem and then disable it.

teylyn say:

Hi,

a starting point could be a formula that shows which SKUs to remove in column C and another one that shows the SKUs to add in column D.

In cell C2 use

=IF(ISERROR(MATCH(A2,B:B,0)),A2,"")

In cell D2 use

=IF(ISERROR(MATCH(B2,A:A,0)),B2,"")

Copy down as far as required.
