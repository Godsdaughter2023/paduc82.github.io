·         Do you receive any print spooler errors while printing?

 

I would suggest you to try the steps mentioned below to see if that help you to fix the issue.

 

Step 1:Check the Spool folder to see if there are any old files in the folder

When printing is working properly, the files in the Spool folder are deleted as the jobs are printed. The default Spool folder is:

 

C:\windows\System32\Spool\Printers

 

Move any old files that are in the Spool folder to see if the problem still occurs. Corrupt files in the Spool folder can cause Print Spooler service problems. You may need to stop the Print Spooler service to move the files from the Spool folder. To stop the print spooler service, follow the steps below.

 

a.    Click on Start; in the start search type services.msc

b.    Select Print Spooler service and click on Stop.

 

Once the old print jobs are moved, then restart the print spooler service and try to print.

 

Step 2:Check the Spooler Dependency Services

The Print Spooler service is, by default, dependent only upon the Remote Procedure Call (RPC) service, RPCSS. Confirm that the dependent services are started.

 

a.    Click on Start; in the start search type services.msc

b.    Right click on Print Spooler Service and click on Properties.

c.    Switch to Dependencies tab.

d.    Confirm that the dependent services are started.

 

Step 3:Configure the installed antivirus application to exclude scanning the Spoolfolder

There can be contention between the antivirus application and the Print Spooler service that may cause intermittent printing problems. You can also manually disable the anti-virus software and then try to print.

Note: Do not forget to re-enable the anti-virus software once you’re done with the troubleshooting.
