I have diagnosed my own issue and for anyone else who has this issue the following was casuing the problems.

Since the anniversary update 1607 n windows 10 this is what was happening.

Word crashing randomly and restarting resulting in work being lost.

Cause:

Checked event logs of time winword.exe was crashing and cross referenced it with the system logs. After each crash i was getting Group policy updated sucessfully in system event logs roughly 30 seconds after each crash.

In my group policy i have printers under the printer preferences set to apply to users of a certain OU. Each time the policy weas applying the printers were being removed from the system and being re-added. This was set via the replace and update commands. So each time the printer was being removed word was crashing.

I verified this by manually removing a printer from devices and printers whilst having word open. AND BEHOLD word crashed when i removed our dell laser printer. Upon checking the driver installed for this printer the last update to the driver was back in July 2014 and none since. Therfore my hunch is that since the update to Win 10 1607 the driver may no longer be fully compatible with the OS.

So for anyone else having random crashing issues with word and have group policy that applies printers!! take a look. This only happened on the printer with the outdated driver. For our HP this issue didnt happen.

Word Crashing = Check printer policies/ manually remove your printers to see if word crashes.

jllatham say:

Let's change the code just a bit and work from there:

Sub NewGroupAdded()
     Dim NextEmptyCell As Range
     Set NextEmptyCell = Range("B" & Rows.Count).End(xlUp).Offset(1, 0)
     '.... more code to follow
End Sub

At the end of that Set statement, NextEmptyCell will represent the blank cell in column B - in other words it would have been the same as Cells(NextRow, 2)


You can now work with cells on that sheet by using .Offset(row, column) from that cell.

NextEmptyCell.Offset(-3, 0) would reference the cell in column B that is 3 rows above the empty cell.
NextEmptyCell.Offset(0, -1) would reference the cell in column A on the same row with the empty cell.
NextEmptyCell.Offset(-3, 1) would reference the cell in column C 3 rows above the empty cell's row.

and

NextEmptyCell.Offset(1, 0) would reference the cell in column B right below where we found the empty cell.

You could use NextEmptyCell.Offset(-2, 0).ClearContents to remove the date from the cell in column B that is 2 rows above the found empty cell.

Examples of copying some information:
NextEmptyCell.Offset(0,-1) = NextEmptyCell.Offset(-3,-1)
would be like (assuming B50 is where the empty cell is as): Range("A50")=Range("A47")
NextEmptyCell= NextEmptyCell.Offset(-3, 0) ' same as Range("B50") = Range("B47")
NextEmptyCell.Offset(0, 1) = NextEmptyCell.Offset(-3, 1) ' same as Range("C50")=Range("C47")

NextEmptyCell.Offset(1, -1) = NextEmptyCell.Offset(-2, -1) ' same as Range("A51") = Range("A48")

 

Does that help, or do you need more specific instruction?
