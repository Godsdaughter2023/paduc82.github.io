johnmartinjr. say:

No problem Paul.  See the below.

Please note that you have to be careful when editing the registry.  Also, when you navigate in the registry to the location mentioned in the below steps, be certain you are opening up the "HKEY_CURRENT_USER" section as mentioned in below steps, and not another registry branch!

1. Login to the Windows account that has the problem.

2. Make sure all Office applications are closed.

3. Open the registry editor (regedit).

4. On the left pane, navigate to and select the "HKEY_CURRENT_USER\Software\Microsoft\Office" folder.

5. (Optional but recommended) Right-click on the "Office" folder, choose export, then name and save the .reg file to your desktop (or some other location if you wish).

Note: Name the file whatever you want (I named mine as "HKCU_SW_MS_Office.reg" which is short for the registry path).  Keep in mind that when you do the next step (step 6) you will loose any personal office application settings (i.e. it will be like the first time you opened each app).  Also, the same applies if you have other Office apps like Visio, Project etc. that were purchased separately.

Note added on 8/21/2018: "IF" you use Outlook, see my 8/21/2018 reply to "WordsBySusan" on how to backup and restore Outlook profiles PRIOR to performing the next step (step 6).

6. Right click on the "HKEY_CURRENT_USER\Software\Microsoft\Office" folder and choose delete.

7. Close the registry editor.

8. Open your Office application(s) and hopefully you will no longer get the SIGN IN REQUIRED prompt.

Added Note: The registry file you created in step 5 above can be used to restore your original current user office settings (including the "sign in required" problem)  if for some reason you wish to do so (but should be unnecessary unless you had some custom app settings that you can't live without, or recreate etc.).

Hi Daryl,

The "My outgoing server (SMTP) requires authentication" option may not be enabled and that is why you're seeing the error message whenever you send an email. We recommend that you check the "Outgoing Server" in "Internet E-mail Settings"  to see if the option is enabled or not. To do this, follow these steps:

  1. In Outlook, click File.
  2. On the left pane, click Info.
  3. On the right pane, select Account Settings.
  4. Select Account Settings... again from the drop-down box.
  5. Click your email account, and then click Change.
  6. Select More Settings ....
  7. choose the Outgoing Server tab.
  8. Make sure that the "My outgoing server (SMTP) requires authentication" checkbox is checked.
  9. Click Use same settings as my incoming mail server.
  10. Click OK to close the "Internet E-mail Settings" window.
  11. Click Next.
  12. Check if you can send emails again.
