a. user say:

That is because Windows is showing a drive that is suppose to be hidden from users.

Follow these steps now to remove it:

1. Select the Start button, type cmd and then right-click Command Prompt in the search results and select Run as administrator.

2. In the Administrator: Command Prompt window, type diskpart and press Enter.

3. Type list volume and press Enter.

4. Make note of the volume number for the drive letter you want to remove. For example, if the drive letter you want to remove is Drive “D”, note the volume number for Drive D.

5. Type the following command, and press Enter.

select volume <volume number>

When typing this command, in place of <volume number>, substitute the volume number you made note of in Step 4. In the example above, if the volume number for drive G is 5, you would type: select volume 5

6. Type the following command and press Enter.

remove letter=<drive letter>

When typing this command, in place of <drive letter>, substitute the drive letter you want to remove. In this example, you would type: remove letter=G

7. Close the Command Prompt window.

I hope this helps. Please let me know if that resolves the issue or you have any other questions.

Snipping tool Windows 7 Emailing trouble. - Microsoft Community

dunntorrey say:

Snipping tool Windows 7 Emailing trouble.

I just installed window7 on my laptop.  I am trying to email an image using the snipping tool provided but I recieve no image in the email.  I do not want to save the file and add it as an attachment.  Shouldnt i be able to simply email an image without navigating away from the snipping tool window?  I use IE and yahoo mail as my defaults
k.c.s. say: Many Web-based email programs that run inside of IE (like Yahoo Mail) don't have access to your local clipboard by default (for security reasons) and/or do not support pasting files or images directly into the email message you are composing.  It is not a problem with the Snipping Tool, it is the email program/method you are using that requires you to save the image to a file first and then attach it in your Web browser.

By default the image you snip with the Snipping Tool is saved to your local Windows 7 clipboard (a temporary storage place that holds things to copy/paste).
You can test that this is the case by taking a snip of something, click Start, go to All Programs, Choose Accessories, and click WordPad.

When WordPad opens you can push Ctrl+V on your keyboard or simply push the "Paste" button in the top-left corner of WordPad and you will see your snipped image appear in the document without having to save it first.  WordPad has access to your local clipboard (where your unsaved image is), IE/Yahoo Mail doesn't.

One way around this limitation is to use an email client program that runs on your local computer and does not need a Web-based email server to store/save the email message before it is sent.  Microsoft Outlook is one such email program that can do this.  Microsoft provides the free "Microsoft Office Outlook Connector" here:
http://office.microsoft.com/en-us/outlook/HA102225181033.aspx

This lets you use Outlook to create emails you can paste into without saving and manage your Web-based Windows Live Email account.  This gives you the power of a local email program (Outlook) with the convenience of connecting it to a Web-based email solution (Windows Live Mail).

borate say:

Update, as of 12/5/15

  --- revised 2/18/16 --- No further instances of lag (which can also stop macros from completing) since SUPERFETCH was disabled.  However, some users report occasional lag despite this measure, and the underlying problem is extant.

It appears to have nothing to do with hardware, keyboard drivers, SFC or services other than SF.

After considerable experimentation it appears that the lag will occur when a STORE app (Calculator, for example) or "store" itself is listed in Task Manager background processes.  Kill the process, and the lag vanishes.  When the app is actually running, it is listed under APPS, not background processes.  Then at random times, often after a sleep cycle or at boot, it will spontaneously appear as a background process - Calculator, (System) Settings, Store and possibly others.


Speculated causes of this annoyance  are discussed in this article and its embedded links... https://superuser.com/questions/426947/slow-windows-desktop-keyboard-shortcuts/957210#957210 where "Helen" posted a workaround...

Disable SUPERFETCH.  Click START, type "services", <enter>, double-click on SUPERFETCH.  Stop it.  Disable it.

No performance hit has been noted by doing so, and shortcuts launched via keyboard no longer lag.
