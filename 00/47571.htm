lemp say:
It might be malware (you haven't identified your a/v application or indicated if you've scanned with MalwareBytes AntiMalware or SUPERAntiSpyware), but considering that you seem to be working with Java development, it might also be some Java application gone wild.

In any event, one of the following techniques ought to work for you:


Even though we don't usually see it, each file/folder with a Long File Name
(LFN) also has a computer-created Short File Name (SFN), also referred to as
an 8.3 filename, since it is limited to 8 characters plus and optional
extension of up to 3 characters. (This goes back in computer history to
MS-DOS and before.)

To see the 8.3 filename of your folder, open a Command Prompt and enter:
Dir /x

That command should present the usual listing of all the files and folders
in that directory, as with the plain Dir command. But the /x switch will
create an additional column, before the LFN, showing the SFN for any file or
folder whose LFN does not also meet the 8.3 requirements. Perhaps there is
a space or other "illegal" character in the filename. The SFN for a LFN
will probably have 6 characters, followed by ~1, or some other digit.

Use the Del or RD command with that SFN to delete your file or remove your
folder. Using RD <SFN> /s for the top-level folder should remove that whole
directory tree.

(As always in a Command Prompt window, type the name of any command followed
by /? to see all the switches and parameters available with that command,
such as: Dir /? )



You said that you had tried renaming the folders to make the path shorter but as soon as you got the path shorter, a new subdirectory was created.  You need to stop that from happening.  You can either uninstall your Java development software, as John Rubdy suggests, or pick one of the following methods:

(a) Restart in Safe Mode or (b)  Clean Boot mode or (c) open both a Command Prompt windows and Task Manager and stop explorer.exe using Task Manager (do not close Task Manager). 

Whichever of these 3 modes you are in, use Method A above.  You also might be interested in reading here --> http://superuser.com/questions/256105/how-do-i-delete-a-folder-which-is-nested-quite-deep-and-avoid-file-name-too-lon

If you selected (c), when you are done with the deletion, click File > New Task (Run ...) in Task Manager to restart explorer.exe.


Start the computer using a live Linux distro (e.g., Knoppix) or Hiren's BootCD or similar boot cd and delete the directory tree using the available tools.

I was a Software victim too (in my case, QuickBooks 215) having incompatible features with WIN10 settings.  After an update, I was no longer able to export reports to Excel... and the only export feature available was to CSV files.  While QuickBooks Software was not running I removed .NET 4.6 settings (See image) and was then able to export to Excel again.  I do a lot of importing and exporting and often find WIN 10 updates mess with two main areas:  ActiveX (resets to Disable and I have to go back and set to Enable) and .NET Framework (giving new issues to not-so-old Software).  Hope this helps somewhat...
