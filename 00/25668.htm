ASUS G73Sw Geforce GTX 560M Driver fix for Hardware undetectablility - Microsoft Community

devonnissen say:

ASUS G73Sw Geforce GTX 560M Driver fix for Hardware undetectablility and invalid hash

One of the computers I've recently installed windows 10 on was an older G73Sw ASUS laptop.


Hardware =
Intel Core i7 2630QM 2.0GHz

GeForce GTX 560M

8GB Mem 

When the computer booted after initial install and began getting its drivers. It failed to find drivers for the video card. Turns the hardware was not compatible with any driver I found.

I went to the ASUS' support site and downloaded both of the video drivers for the model and both failed to install and gave the same error. 

"Could not find Compatible hardware!"

Which led me to explore the possibilities.

Turns out that the GPU's ID was not in any of the device list in the INF files for the display drivers.

I'm not going to tell you about the Journey on the way to the solution just the fix.

1st - Verify your GPU
I was absolutely in the dark as to the exact hardware specs when I began so when I went to get the drivers from NVIDIA I used their Java app to identify the hardware.
Or you can open the laptop and read the id number off of the card.

2nd - Get the Hardware id
Open Control Panel Go to System & Security Click on System Click Device Manager (should be located in the left column) Here it should be under either Display Adapters or Others click  Microsoft Basic Display Adapter click the Details tab click the drop down and select "Device Instance Path" or "Hardware ID" here you should see this number or a number very similar.

PCI\VEN_10DE&DEV_1251&SUBSYS_125110DE...ETC

3rd - Go to Nvidia and download the appropriate driver.

4th - Run the installer so that it builds the directory

5th - open explorer and go the this pc/ local disk (c:)/ NVIDIA/ displaydriver/xxx.xx/win9_win7_64/international/display.driver

6th - open nvami

Here is where it gets a little fun.
Page down until you see this
[Manufacturer]%NVIDIA_A% = NVIDIA_SetA_Devices,

NTamd64.6.0,NTamd64.6.1,NTamd64.6.2,NTamd64.6.3,NTamd64.10.0

[NVIDIA_SetA_Devices.NTamd64.10.0]>%NVIDIA_DEV.0FD1.10B8.1462% = Section002, PCI\VEN_10DE&DEV_0FD1&SUBSYS_10B81462 

%NVIDIA_DEV.0FD1.10C7.1462% = Section002, PCI\VEN_10DE&DEV_0FD1&SUBSYS_10C71462 %NVIDIA_DEV.0FD1.10CD.1462% = Section006, PCI\VEN_10DE&DEV_0FD1&SUBSYS_10CD1462 %NVIDIA_DEV.0FD2.10B7.1462% = Section002, PCI\VEN_10DE&DEV_0FD2&SUBSYS_10B71462 %NVIDIA_DEV.0FD2.10B8.1462% = Section011, PCI\VEN_10DE&DEV_0FD2&SUBSYS_10B81462 %NVIDIA_DEV.0FD2.10BA.1462% = Section006, PCI\VEN_10DE&DEV_0FD2&SUBSYS_10BA1462 


So your model number for the laptop and the manufacture code are the important part. Youre going to create the id as you see the multiple ids above so that when you run the installer it can see the hardware. 

so like all those above you will type it like so

%NVIDIA_DEV.1251.0000.0001% = Section(just pick the three numbers associated with one of the other ids next to wher this is going. on my friends it was 038)
so

%NVIDIA_DEV.1251.0000.0001% = Section038, PCI\VEN_1251&SUBSYS_125110DE
Now that you've built your id, youre going to scroll down through the list and stick in to the area it would numerically fit. I have not seen if it being out of order really matters but I like keeping things neat.
I did this for all the sections. >example

[NVIDIA_SetA_Devices.NTamd64.10.0]

[NVIDIA_SetA_Devices.NTamd64.10.1]

[NVIDIA_SetA_Devices.NTamd64.9.0] etc


when finished scroll down to the bottom of the file where you will find STRINGS


Here you will see a list of video cards and IDs You will now make you gpu id again but a bit differently


NVIDIA_DEV.1251.0000.0001 = "NVIDIA GeForce GTX 560M"


Save the file and youre done with this step

Next run the installer again so that it can remove the "windows basic display adapter"

Then click the start button and click settings

go to Update and Recovery

click the Recovery tap

Click Advanced Start up

Click Trouble shoot

Click advanced options

click startup settings

Click restart

When it boots press 7, but read first because what you want to do is disable signature enforcement so make sure you select the correct option

When the computer boots up, and you log into windows

Go back to Control panel/ system security/ system/ device manager/select the display adapter which at this point may just be called display, it may be under Other or display adapter so you need to pay attention and look.

click the device and go to Driver

click update driver

select browse my computer for driver software

go to C/Nvidia/displaydriver

select include subfolder check box underneath the directory box

then click next

it will start installing, then it will ask if you are sure you want to install the software, say yes.

Youre done you now can use your gpu.

I'm not sure if this is the best possible way but as far as I am aware after hours of looking this is the only fix available as of now.

I hope it was helpful.

FWIW the 0xC0000022 happens on a completely new install of 1803.  Resetting the machine makes it go away.  So you can mitigate this by:

  1. Syncing all files (make sure you don't delete anything you need)
  2. Uninstall OneDrive.
  3. Make sure you delete your OneDrive folder from disk (this is why step #1 is important)
  4. Reinstall OneDrive

HOWEVER... this issue can be reproduced very easily, especially on new installs.

  1. Install OneDrive
  2. Ensure Desktop is protected and syncing.
  3. Create a new folder on your desktop to store your links/shortcuts.
  4. Install a new application, like Chrome browser.
  5. Chome will install a shortcut on your desktop.
  6. Move this shortcut into the folder created in #3

On another machine:

  1. Repeat steps 1-6 above.
  2. After step 6 you will get prompted to overwrite as the shortcut has already been created and placed from the previous sync.  Accept and overwrite.
  3. OneDrive will think VERY HARD FOR A VERY LONG TIME. 😉 (this can only mean one thing, of course):
  4. Behold the mighty error.

Not so much a new way as the old tried and true:

taylodl say:

The paste option 'Keep Source Formatting' worked! 

Here's what I was trying to do: insert a set of slides from one presentation (source) into another (target) while preserving the formatting of the slides from the source. The problem I was experiencing was when I chose 'New Slide/Reuse Slides...' it would insert all the source slides but use the target format, which isn't what I wanted.

So here's what I did:

  • Open target presentation
  • Open source presentation
  • Copy first slide from source presentation
  • Paste into target presentation where desired, using the 'Keep Source Formatting' option (you have to use the Paste drop-down from the ribbon bar on the Home ribbon)
  • Choose 'New Slide/Reuse Slides...'
  • Choose the source presentation
  • All the slides will be inserted into the target and preserving the source presentation format

I agree I liked how it worked better prior to the 2016 version, but at least we can recover this functionality!

corey riley 76 say:

Doug, I've been fighting these issues for months in our organization.  Unfortunately there are few different fixes depending on what Outlook version you are running.  I've found in many cases on version 1803 and 1809 that the either the credential manager object get hosed up which it's just a matter of clearing out the credential manager entries (everything except the ones that have "teams" in the name) and opening Outlook back up, or the "identity" registry key gets goofed up.  Since you mentioned that a new windows profile fixes it, then these are probably the two fixes you should try if you haven't yet.

To reset the "Identity" registry key, open regedit and go to "HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Common\Identity".  Right-click and delete the "Identity" key.  If you are uncomfortable deleting registry keys, you can export that key to make a backup before deleting. 

When you open Outlook back up and go through the login process, these keys are recreated.  I use multiple O365 accounts in my Outlook profile and have found that the entries will get crossed up where the "SigninName" or "ProviderId" entries will list the incorrect account information.
