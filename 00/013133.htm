I am no expert in VBA web queries and HTML parsing.

But I can tell you that with a virgin (new) workbook on my computer, more than 99% of the macro time is consumed by the .Refresh statement, which actually performs the network transactions and stores the parsed data into the worksheet.

So I would be surprised if there is much you can do to improve your example macro's performance as long as you use the web query paradigm.  (Perhaps there is a better way altogether.)

But that is with virgin (new) workbook.  The following points might be useful.

  • If your workbook includes formulas that use volatile functions or formulas that reference the range of cells that are filled in by the web query, you might benefit from disabling screen updates, auto calculation and events.

  • Unrelated to performance concerns, your algorithm is implemented incorrectly with respect to retries.  The .Connection statement does not initiate any network transactions.  Instead, it is the .Refresh statement that does.

  • Since your macro is designed to store the web query only in worksheet "A", you can avoid some unneeded worksheet referencing by putting the macro in the Sheet A code module instead of a standard code module.

    However, because that is much less than 1% of the total macro time, there is little to be gained other than simplifying the implementation a little.

The following implementation addresses those points.
