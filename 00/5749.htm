<5% showing green

bernie deitrick say:

Easiest with colour fill.  Select both columns of data, and use the formula option three with formulas like

Green fill

=$C5<($B5*1.05)

Amber fill

=AND($C5>=($B5*1.05),$C5<($B5*1.10))

Red fill

=$C5>=($B5*1.10)

where 5 is the row with the first value, and you want to highlight columns B and C based on C being higher than B.

Is there a way to run the code before an error is generated? Is the solution to package the accdb without the reference and somehow trap for an error?

thedbguy say:
Hi Jim. See if this old page has anything you can use. Good luck!

jimp46 say:

Thanks, I will check out the article.

I may have found a solution in the meantime (it appears to work but don't know if it is the best solution )

1. Ship the application without the type library registered

2. During the startup routine, run the following.code

     Set refs = Application.References
     Set ref = refs("MyTypeLibrary")

3. If the reference is missing, then an error is generated, and run the code to register the library and set the reference

scott mcdaniel say:

Note that one issue with this sort of process is that the very fact that you are missing a reference could cause your app to NOT run VBA correctly (or at all). While it may very well work in most cases, if you're deploying this to a lot of people you might find that you issues with the method.

IMO, the right way to do this is to package you application with all needed dependent objects, and use a Windows Installer to handle the installation needs. Any decent installer will be able to determine if the target environment contains the necessary prerequisites and advise the user on what to do from there. InnoSetup is free and widely used, and has a very robust support community. I don't use it any longer - I use InstallMate now, which gives me a few more features that I like, plus it's fairly cheap at @100 for a small office license - but when I did i found it to be more than capable of handling Access deployments.
