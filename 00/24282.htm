vba macro insert pictures in a word table - Microsoft Community

a. user say:

vba macro insert pictures in a word table

I have a macro for a two column table in a word document (no excel involved) with text in the right hand column cells and would like to insert pictures into the left hand column cells. what does the vba macro code look like for inserting a jpeg file into a certain cell? ie cell 1,1 / 2,1 / 3,1 / etc.

Thank you for your help!

jay freedman say:

This is about the simplest sample code (more on this after the code):

                    

Sub x()
    Dim objTbl As Table
    Dim nRow As Long
    Dim rg As Range

    With ActiveDocument
        If .Tables.Count = 0 Then Exit Sub

        Set objTbl = .Tables(1)
        objTbl.AutoFitBehavior wdAutoFitFixed

        For nRow = 1 To objTbl.Rows.Count
            Set rg = objTbl.Cell(Row:=nRow, Column:=1).Range
            rg.Collapse wdCollapseStart
            .InlineShapes.AddPicture _
                FileName:="D:\temp\" & CStr(nRow) & ".png", _
                Range:=rg
        Next nRow
    End With
End Sub

In this code, the image files in the D:\temp folder are all PNG format with names 1.png, 2.png, etc. You may need an array of known file names, or code to let the user pick a sequence of files; or there could be a way to pick the filename out of the text in column 2. The AutoFitBehavior wdAutoFitFixed will force any picture wider than the column to be reduced to fit; otherwise the column width will be adjusted to fit the largest picture and column 2's width will be reduced.

suzanne s. barnhill say:

To include the appendix number as a "chapter number," it must be the paragraph number of an autonumbered paragraph in one of Word's built-in heading styles (see http://www.ShaunaKelly.com/word/numbering/NumberingAppendixes.html). Creating each appendix as a separate document complicates the task, but you can still accomplish it by setting the starting number of the heading to B, C, etc., for the appendixes after A. If you plan ultimately to combine the documents, you'll need to make sure that the numbering is sorted after they're assembled.

You could do this manually with a StyleRef field if you apply a character style to the appendix number, but that would be more work.

If so, you should include Mailchimp as a valid origin in your SPF record.

Your SPF should look like this :

v=spf1 ip4:111.222.333.44 include:servers.mcsv.net -all

where "servers.mcsv.net" is mailchimp

SPF record contains the possible origins of the mails of certain domain.

If your domain's SPF is not declaring mailchimp as a source, those mails probably end up as unwanted or rejected due phishing .
