chip pearson say:

Assume you have data in a rectangular range of cells. VLOOKUP works by scanning down the left-most column of that range until a match is found. It then scans to the right on that row for the specified number of columns and returns the value in that cell. E.g., suppose you have the following data in cells A1:D3.

11  22  33  44
55  aa  bb  cc
66  xx  yy  zz

and a formula like

=VLOOKUP(55,A1:D4,3,FALSE)

This will return the value "bb". This is because VLOOKUP looks down the cells A1:A3 and finds a match to 55. It then moves three columns to the right (starting at 1 = left most column, A) and gets "bb" in the 3rd column. The last parameter here is FALSE, which indicates that an exact match is required when scanning the left-most column. In this case, the first column need not be in any sorted order. If you use TRUE as the last parameter to VLOOKUP, the function will scan until it finds a value in the first column that is less than or equal to the specified search value. An exact match is not required.  In this case, the first column must be in ascending sorted order. Otherwise, the results will not be what you expect.

 

Cordially, Chip Pearson Microsoft MVP, Excel Pearson Software Consulting, LLC www.cpearson.com

jllatham say:

Chip Pearson has given a very good explanation of how VLOOKUP() works.  One thing I might add is that the 3rd parameter, the number of the column to return data from, is relative to the columns included in your rectangular area.  In Chip's example he uses a rectangle that includes columns A, B, C and D and so the ,3, parameter pulls data from the 3rd column which happens to be column C.  IF he had set up an example where the rectangular data was in columns R, S, T and U then the column to match data in would be R and the 3rd column of the rectangle would be T. 

Your example only showed 1 column of data, and if there is no associated data to make up a rectangle, then you do not need to use VLOOKUP().  Let's say that the sheet with the information you've show us in your example is named Sheet1, and it is laid out like this:

       A
1    BOX
2         1
3         2
4         3
5         4

Over on Sheet2, you could put this formula in a cell:  =Sheet1!A2  to display the "1" in your list on Sheet1.  You can even let Excel do a lot of the typing for you: 
Select a cell on Sheet2 and then
type in the = symbol
Select Sheet1 and
click in the cell you want to echo over onto Sheet2
press the [Enter] key.  Excel will build the formula for you.  (This same type of automatic building of references can even be extended to cells on worksheets in other workbooks that are open at the time you need to set them up.)

Now, in my example, Excel will have built a formula that looks like  =Sheet1!$A$2.  If you edit that formula and remove the $ symbols from it, you can even use Excel's "fill" feature to fill the formula down on Sheet2 to pick up the other values from Sheet1.

If I've added too much here and have just added to your confusion, I apologize.  It just looks to me, from the information you gave us, that VLOOKUP() may be overkill.  If neither suggestion is helping you any, then post a little more detail about what you are trying to do and give examples of at least a good sized (3 or 4 rows and columns) of information you're dealing with.  Sometimes showing us what you have and also showing us what you want things to look like after you apply a solution provided here really helps to get a good fix without a lot of back-and-forth message exchanging.

charles kenyon say:

You should be able to do what you want with something like this:

=(IFERROR(B6/C6,0)+IFERROR(D6/E6,0)+A6)

The IFERROR functions will give a value of 0 if a division throw an error and otherwise will give the result of the division. The A6, above was the cell that will always have a value.

If you can, please mark this one as the answer and unmark the other one. It will help others who might find this thread.

hjc3 say:

I think the assumption was that you would want to disregard the term if you had no value for the denominator.  However, if that is not correct, you simply need to change the terms from IFERROR( G5/H5, 0 ), for example, to IFERROR( G5/H5, G5 ).  The second argument represents the value you want to use as a replacement if an error is encountered when evaluating the formula given as the first argument, and it may be a single value (such as zero) or another formula, depending on what you need.
