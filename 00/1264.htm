    Changing all hyperlinks in a MS Publisher Intranet webpage - Microsoft Community

 wannabe_448  say:


            Changing all hyperlinks in a MS Publisher Intranet webpage

                    Recently our IT department changed the name of the server that stores our Intranet and Intranet files.  Because of this change, all of the hyperlinks in the document need to change to the new name of the server.  Is there any way of doing this systematically
 or by script or is this a manual process where each link needs to be recreated?
                



            



            



            
            
 davidf2  say:
            
            
            
            
Assuming you are using Publisher 2007 when you produce web pages you do so by Publish to the web and if you don't change the defaults Publisher will produce an index.htm file (your home page) and an index_files folder which contains all supporting grahpics
 and any additional pages/ .htm files. Then normally you would upload both the index.htm file and the index_files folder to the root directory of the web server. Then the link to the home page is:
http://yourdomain.com/index.htm and the links to the other pages would be
http://yourdomain.com/index_files/Page###.htm .



How are your links named?


If you insert links within your Publisher publication to other pages then in some cases those links will be relative links and you won't need to change them.


So to answer your question, if your link points to the index.htm file in the root directory of your newly named server then it should work. If it points to the old root directory you will have to change it/them.


There is no script to change all links.
