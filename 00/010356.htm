Application.Volatile
Dim oset As Long

For Each c In rng
    If c.Interior.ColorIndex = clr.Interior.ColorIndex And _

    End If
Next

Hi Tom,

There may be many reasons causing messages are going to spam. We need to confirm SPF record added firstly.

The SPF record should be like this: 

v=spf1 include:spf.protection.outlook.com ip4: <IP address> include: <domain name> -all

Tips: 1. "v=spf1" and "-all" are required to all SPF TXT records end with this value.

2. Where the value for <IP address> is the IP address of the other mail system. From the information you provided, it should be 167.89.**.***.

3. <domain name> is the domain name of the other mail system that sends mail on behalf of your domain.

Additionally, you need to add the SPF record to the sender's domain. For example, if you send messages from "another company" to your email account, you need to add this SPF record to the domain of "another company".

After that, you can add this domain to "allow lists" in receiver's part if your received email account is Office 365 account: use your admin account to Exchange Admin Center> navigate to Protection> Spam filter> edit default spam filter policy> allow lists> add this domain to sender allow lists.
