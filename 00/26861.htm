paulerith say:

How can i link/mirror more than one cell with the same data

How can i link/mirror three cells on the same worksheet eg a9,c18 ,g55 so that i can add data in any one of the three cells and it updates all three cells.

Thanks Paul

jllatham say:

Can't do it without a macro (VBA code).  A cell can either have a formula in it or it can have a value in it - it cannot have both.  Also, if you set something like this up:

Cell          Formula

A8             =C15

C15           =G55

G55           =A8

You create a circular reference, most days that's not a good thing to do.

 

Either pick one of the three to be the 'master' source and link the other two to it, or ask for a macro solution - and be specific about all cell addresses involved for that to be done properly.  With the macro solution, most likely any existing formulas in the cells are going to be wiped out - meaning that it's not going to work well unless all cells involved normally get their contents from user data entry from the keyboard.

 

Also, if you ask for a macro solution, if any of the cells involved are on different worksheets in the workbook, be sure to provide both the worksheet names and cell addresses involved, like:

Sheet1 A8,  Sheet1 C15,  Sheet3 G55

 

rich007a say:

Hi Paul,

Right-click on the sheets "tab" at the bottom (where it says "Sheet1") > View Code

Paste the following code on the right:

 

Option Explicit

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim rng As Range, c As Range
   
    On Error GoTo ExitPoint
    Set rng = Me.Range("A9,C18,G55")
   
    Application.EnableEvents = False
    For Each c In rng
        If Not Intersect(Target, c) Is Nothing Then
            rng.Value = c.Value
            Exit For
        End If
    Next c
ExitPoint:
    Application.EnableEvents = True
End Sub

 

Close the VBE to return to Excel.

Save your file as Macro Enabled .xlsm.

Done.

bill manville say:

The problem is that the query you are creating contains the literal string tttype which means nothing to the database.  It is a variable local to your VBA procedure and you want the value of that variable to be part of the string.  You do that by concatenating the variable with the rest of the string, and surrounding it with ' ' for the sake of the SQL syntax so:

"WHERE (HS_FILE.TR_TYPE='" & ttype & "') AND (HS_FILE.ZNE_CODE='" & zcode & "')"

jllatham say:

This code looks at entries in column F of Sheet1 (row 21 and after) to determine whether or not to transfer data.  It ignores all other columns, including G and H in this decision making.  I've made the code about as adaptable as I know how to - and even if more columns become involved, hopefully you would be able to add constants and instructions to accommodate them based on this process as a roadmap.

 

Sub TransferData()
  'these all refer/define the sheet with the data to be copied
  Const ws1Name = "Sheet1" ' change as required
  Const ws1FirstDataRow = 21
  Const ws1FirstDataCol = "F"
  Const ws1SecondDataCol = "G"
  Const ws1ThirdDataCol = "H"
  'these all refer/define the 'copy to' sheet
  Const ws2Name = "Sheet2" ' change as required
  Const ws2FirstDataRow = 4 ' ignore rows 1-3
  Const ws2FirstDataCol = "B" ' where F from sheet1 goes
  Const ws2SecondDataCol = "C" ' where G from sheet1 goes
  Const ws2ThirdDataCol = "G" ' where H from sheet 1 goes
  Const ws2SpecialCol = "F" ' where to put GBP
  Const ws2FixedData = "GBP" 'to be put into ws2SpecialCol
'end of user re-definable constants

  Dim ws1 As Worksheet
  Dim ws1DataList As Range ' based on ws1FirstDataCol content
  Dim anyDataEntry As Range
  Dim ws2 As Worksheet
  Dim anyLastRow As Long
 
  Set ws1 = ThisWorkbook.Worksheets(ws1Name)
  anyLastRow = ws1.Range(ws1FirstDataCol & Rows.Count).End(xlUp).Row
  If anyLastRow < ws1FirstDataRow Then
    'actually could just Exit Sub since no real work to do
    anyLastRow = ws1FirstDataRow
  End If
  Set ws1DataList = ws1.Range(ws1FirstDataCol & ws1FirstDataRow & ":" _
   & ws1FirstDataCol & anyLastRow)
 
  Set ws2 = ThisWorkbook.Worksheets(ws2Name)
  Application.ScreenUpdating = False
  For Each anyDataEntry In ws1DataList
    anyLastRow = ws2.Range(ws2FirstDataCol & Rows.Count).End(xlUp).Row + 1
    If anyLastRow < ws2FirstDataRow Then
      anyLastRow = ws2FirstDataRow
    End If
    ws2.Range(ws2FirstDataCol & anyLastRow) = anyDataEntry ' from col F
    ws2.Range(ws2SecondDataCol & anyLastRow) = _
     ws1.Range(ws1SecondDataCol & anyDataEntry.Row)
    ws2.Range(ws2ThirdDataCol & anyLastRow) = _
     ws1.Range(ws1ThirdDataCol & anyDataEntry.Row)
    ws2.Range(ws2SpecialCol & anyLastRow) = ws2FixedData
  Next ' end anyDataEntry loop
  'housekeeping cleanup
  Set ws1DataList = Nothing
  Set ws1 = Nothing
  Set ws2 = Nothing
  MsgBox "Data Transfer Finished", vbOKOnly + vbInformation, "Task Completed"
End Sub

Please try the following steps to uninstall OneDrive completely and reinstall it.

1. Open Command Prompt as Administrator (Right-click on the Windows icon in the taskbar and select Command Prompt (Admin)).

2. Type in taskkill /f /im OneDrive.exe  to terminate any OneDrive processes and hit Enter.

3. Then type in either  %SystemRoot%\System32\OneDriveSetup.exe /uninstall  if you are using 32-bit Windows 10 or  %SystemRoot%\SysWOW64\OneDriveSetup.exe /uninstall  if you are using 64-bit Windows 10 and hit Enter.

4. Then reinstall OneDrive by downloading new OneDrive sync client.
