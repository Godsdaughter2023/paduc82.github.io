A simple troubleshooting step to perform is to do a Safe Boot. It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot in El Capitan Apple's Disk Utility runs and fixes any troubles with your Mac's directory and file system. Permissions are repaired. The font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. 

 

Once you reach the desktop in Safe Mode try running Excel. Is the crash still there? If not, then maybe simply performing a Safe Boot fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart. Try Excel again. Is the crash still there?

 

If the crash was gone in Safe Mode but returns during a regular boot then there are more things to try. The first would be to use a font utility to validate ALL your fonts. Mac OS comes with a free font utility called Font Book. Get rid of any font that doesn't pass as clean and green, restart your Mac and then empty the trash. If you've gotten this far and still have a crash, then there are more things to explore. But if you get this far you've probably gotten rid of the crash. Either way, please reply with your results.

You can make use of the \f switch in both the XE (index entry) and INDEX fields to decide which specific entries should be included in your second index. graham mayor say:
Well they would say that :) The fact is that historically Epson printer drivers have frequently got the envelope feed orientation wrong. I have a web page about it http://www.gmayor.com/envelope_orientation.htm.

My guess is that the feed is 180 degrees out and so the address is printing on the wrong part of the envelope. It is either that or the Envelope Address paragraph style in the template the document from which the envelope is created from is in the wrong place.

You can only have one Envelope Address style in a template so if it is right for one envelope, chances are that it won't be for another without modification. For this reason alone it is better to use an envelope template. http://www.gmayor.com/changing_envelope_layout.htm  explains how.

You may however find http://www.gmayor.com/Envelope_Label_Add_In.htm useful. It comes with a sample envelope template in #10 format (which is close enough to DL to be used without alteration, though by all means change the page size if you wish). The page also has instructions for using other envelope sizes.
