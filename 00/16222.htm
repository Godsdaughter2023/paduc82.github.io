karl timmermans [outlook mvp 2012-2018] say:

(2). Can anyone suggest a way of merging these .pst folders so that my Gmail address doesn't show up as *** Email address is removed for privacy *** with a number in parentheses attached to it?

Most important thing to understand is how Imap data files work in OL'2010 (or earlier) - which is different then how it's implemented in OL'2013/2016 for anyone else reading this.

#1 - For sake of explanation - if you were to configure a single IMAP account in a new Outlook profile - there would be TWO .pst files created

a) one would be designated as the <default data file> for the profile which would contain the default contact/calendar folders

b) the other would be for the IMAP email folders. A new pst file gets automatically created each time you configure the IMAP account. The number you see in brackets merely means that a pst file with the email address already existed so it appends a number to the newly created file.

There is nothing to merge since the PST file related to the IMAP email folders is a mirror of what exists on the server - the last file created is the one you want to keep or you can delete all of the files and then reconfigure the IMAP account again.

Given your description of the back and forth configuration of accounts - suggest you create a new Outlook profile via the Control Panel <Mail> app and configure your accounts. The only thing that you need to pay attention to is your contact/calendar data (if applicable) in this case. You'll have to determine where that data currently resides in terms of which data .pst file actually contains those folders (it will NOT be in any of the IMAP pst files involving email folders).

    hansv mvp say:

    By default, userforms are displayed modally, i.e. the code after UserForm1.Show won't be executed until the user has closed the form. So it's useless to have

    UserForm1.Show
    UserForm1.Label1.Caption = ...

    Instead, set the caption in the UserForm_Initialize event procedure in the userform's code module:

    Private Sub UserForm_Initialize()
        Me.Label1.Caption = Now
    End Sub

    or

    Private Sub UserForm_Initialize()
        Me.Label1.Caption = Time
    End Sub

    or

    Private Sub UserForm_Initialize()
        Me.Label1.Caption = Format(Time, "hh:mm")
    End Sub

    rich michaels - mvp (greatcirclelearning.com) say:

    You probably have Odd and Even headers in the document.  To test put in 3 page breaks so you cover a Different First Page option, an Even Page option and an Odd Page option. And your changes to one are overridden by another.

    Regarding updating all fields here is some code out of one of my apps. Put it into your template and assign a key sequence or a toolbar icon.

    Sub UpdateAllFields()
        Dim TOC As Word.TableOfContents
        Word.ActiveDocument.Fields.Update
        For Each TOC In Word.ActiveDocument.TablesOfContents
            TOC.Update
            TOC.TabLeader = wdTabLeaderSpaces
        Next TOC
    End Sub

    It will update all fields in the document as well as update your TOC if the document has one. You can obviously comment out the TOC part if you don't want that to be automated.

    If you need help inserting the macro into your template let me know.
