I produce reports containing 8 photos per page, each photo i import has to be resized, so that all the photos are the same size and spaced out equally. i want to be able to place the photo into its location whereby they they self size.
jay freedman say:
Insert a table with eight cells. Select the whole table.

Drag the columns to the maximum width you want for the photos.

If you don't want the lines between cells, go to the Table Tools | Design tab on the ribbon, click the down-arrow on the Borders button, and click No Border.

On the Table Tools | Layout tab of the ribbon, click AutoFit and choose Fixed Column Width.

On the same tab, click Properties. In the Properties dialog, click the Row tab. Check the box for Specify Height, and enter the maximum height you want for the photos. Then set the Row Height Is box to Exactly, and click OK.

Save the table as a building block, so you can insert a copy of it on each new page. It's a good idea to go to the Table Tools | Layout tab and click the View Gridlines button so you can see where the table cells are; you can turn it off later.

When you insert a picture into a cell of the table, it will automatically be sized to fit within the fixed dimensions of the cells.

greginmich say:

I realize that not many people use exclusions, and that those who do aren’t likely to bother with periodically scanning the excluded folders. But for the few people who might want to do this, here’s another approach that might prove useful – and this one doesn’t require writing registry values. If we simply run a custom scan of an excluded folder with remediation disabled, this causes the scan to ignore the exclusion, and any threats that MSE detects will be identified in the C:\Users\All Users\Microsoft\Microsoft Antimalware\Support\MPLog-<date-code>.log file. Even though this approach would require removing the exclusion and rescanning in the event that malware was actually found, the very low likelihood of finding malware makes this a viable alternative for monitoring excluded folders (especially if the batch file is automated with Task Scheduler). This sample batch file runs a custom scan with remediation disabled and responds to a detection (return code of “2”) by echoing a warning and displaying the relevant MpCmdRun.log file. The “timeout” command holds the command prompt window open long enough to let us see the results:

 

 

@echo off

 

cd C:\Program Files\Microsoft Security Client

 

MpCmdRun.exe -Scan -ScanType 3 -File C:\Users\Greg\Downloads -DisableRemediation

 

if errorlevel 2 (

 

echo Warning! MALWARE DETECTED but no action taken!

 

Start Notepad C:\Users\Greg\AppData\Local\Temp\MpCmdRun.log

 

)

 

timeout /t 500
