ken sheridan say: Rather than ranking the total scores in the report by means of the RunningSum mechanism, you can so in the report's RecordSource query.  You'll find examples of such queries in RowNumbering.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates a number of queries for computing row numbers and ranks.  While that using a join is the most efficient, that using a subquery is perhaps easier to understand:

SELECT T1.TransactionDate, T1.TransactionAmount, CustomerID,
    (SELECT COUNT(*)+1
     FROM Transactions AS T2
     WHERE T2.TransactionAmount > T1.TransactionAmount) AS Rank
FROM Transactions AS T1
ORDER BY T1.TransactionAmount DESC;

If it is necessary to group the ranks, e.g. per customer in my demo, then it is just a question of correlating the subquery with the outer query on CustomerID also:

SELECT T1.TransactionDate, T1.TransactionAmount, CustomerID,
    (SELECT COUNT(*)+1
     FROM Transactions AS T2
     WHERE T2.CustomerID = T1.CustomerID
     AND T2.TransactionAmount > T1.TransactionAmount) AS Rank
FROM Transactions AS T1
ORDER BY T1.CustomerID, T1.TransactionAmount DESC;

In these the Transactions table would be the equivalent of your query which returns the total scores, and the TransactionAmount would be equivalent to the TotalScore column returned by the query.  CustomerID might be equivalent to something like an event in your case, to return separate rankings per event.  A report based on such a query can then be sorted by the Rank column.

Note that when using such queries as the RecordSource property for a report, the ORDER BY clause should be removed from the query, and the report ordered by means of its internal sorting and grouping mechanism.

How do I transfer (copy) the entire, formatted Style Pane from - Microsoft Community

How do I transfer (copy) the entire, formatted Style Pane from one document into a new document in MS Word 2013?

In order to not have to individually recreate the various customized style settings of one document (the original document) to be used in a different document (the new document); is there a way to copy in one step (or just a few steps) the customized style settings of the original document?  For example, the original document has 5 levels of headings, 2 bullet list settings, paragraph settings, table settings, text-block settings, and more.  How do I recreate (copy) those settings from the original to the new document prior to starting to enter content into my new document?  Is it also possible to apply the style settings of the original document to another existing document (ultimately having the original document and the new existing document be governed by the same style settings)?
john korchok say:

This is a job for Templates! Use Word's File>Save As command and change the Save as type to Word Template (*.dotx). Double-clicking on a template does not open it: instead it creates a new document that includes all the typestyles of your originals.

To apply a template to an existing document, open the document in Word and follow these steps:

  1. From the File tab, choose Options.
  2. Click on Add-Ins.
  3. At the bottom of the dialog, change the Manage dropdown to Templates and click on the Go button.
  4. Click on the Attach button.
  5. Navigate to your template, select it and click on OK.
  6. Check the box Automatically update document styles.
  7. Click on OK. The styles in the document are updated.
  8. Highly recommended: repeat the above steps. When you get to the Templates and Add-Ins dialog, UNcheck Automatically update document styles. This will will keep the new styles constant in the document even if you later update the styles in the template.

A note about how styles update when you attach a new template:

  • For styles that have the same name in both document and template, the document styles will be updated.
  • For styles that are in the template, but not in the document, the styles will be added to the document.
  • For styles that are in the document, but not in the template, the styles will remain unchanged.
  • For all text that has been locally formatted, the styles will be ignored and the local formatting retained. Local formatting is selecting text and change the font and paragraph, but not applying a style. The majority of Word documents are locally formatted.

Private Sub cmdCreateInvoices_MultiFiles_Click()

    On Error GoTo Err_Handler
    
    Const FOLDER_EXISTS = 75
    Const MESSAGE_TEXT_1 = "No folder set for storing PDF files."
    Const MESSAGE_TEXT_2 = "No invoice(s) selected."
    Dim strFullPath As String
    Dim varFolder As Variant
    Dim varFolder_1 As Variant
    Dim varItem As Variant
    
    ' build path to save PDF file
    varFolder = DLookup("Folderpath", "pdfFolder")
    If IsNull(varFolder) Then
        MsgBox MESSAGE_TEXT_1, vbExclamation, "Invalid Operation"
    Else
        With Me.lstInvoices
            If .ItemsSelected.Count > 0 Then
                For Each varItem In .ItemsSelected
                    Me.txtInvoiceList = .ItemData(varItem)
                    ' create folder if does not exist
                    varFolder_1 = varFolder & "\" & .Column(1, varItem)
                    MkDir varFolder
                    strFullPath = varFolder_1 & "\" & .Column(1, varItem) & " " & .ItemData(varItem) & ".pdf"
                    DoCmd.OutputTo acOutputReport, "rptInvoiceMultiple", acFormatPDF, strFullPath, True
                Next varItem
            Else
                MsgBox MESSAGE_TEXT_2, vbExclamation, "Invalid Operation"
            End If
        End With

Exit_Here:
    Exit Sub
    
Err_Handler:
    Select Case Err.Number
        Case FOLDER_EXISTS
        Resume Next
        Case Else
        MsgBox Err.Description
        Resume Exit_Here
    End Select

End Sub

2.  To identify selected items in a multi-select list box you need to loop through the control's ItemsSelected collection.  This is a collection of variants, each of which identifies a selected row.  The value in the bound column can then be retrieved by means of the ItemData property, a value in another column by means of the Column property, with the value of the variant identifying the row in each case.

With the MkDir, if the directory already exists it throws an error: # 75. I don't remember the exact description but it was something about a file/path error. Just trap it and Resume Next.


Before I did the query rebuilding part I tried using the command code that gathered the filter parts and ran the reports. I figured since the reports would pop up in Preview it might work. I lucked out and it does. This is a cut-down version of the big report app so there are less filters and choices but it may work for you if you have a solid report tool already.


My reports have suffixes depending on what you are looking at, how grouped and sorted, etc. The base is what the suffix is added to. Be aware that you have to get the base on each time through the loop if the base is the same but the suffix changes (instead of before the loop) or the appended suffix on the variable makes a report name that doesn't exist. Crash.

After the report is run, printed and saved, close it. What I ran into was if not closed you end up printing the same report over and over without the new filtering.

I cut out all those Echo False lines. I liked the way the user not only sees the list boxes change, but also sees the pop-up that shows printing and the pop-up that shows saving.

Notice that there are 2 Print commands. One pops up the Print dialog, the other doesn't.

Finally, before this is all put to bed I will make constants that define the loop indexes. That makes it easier, not having to go back and forth between the lookup queries (sources for the lists) and the code to see which one is next in the list when you get back in here next year.

MondayPack_done:
    DoCmd.Hourglass False
    DoCmd.Echo True
    MsgBox "Finished running the Monday Pack reports." & vbCrLf & vbCrLf & _
        "   * Your reports can be found on the printer that is set up as your Default Printer." & vbCrLf & _

    Exit Function
    
MondayPack_err:
    DoCmd.Hourglass False
    DoCmd.Echo True
    iMondayPackReports = 0
    
    Select Case Err.Number
        Case 0, 20      ' not really an error
            Resume Next
        Case 75         ' file/path error; when folder already created, MkDir raises this error
            Resume Next
        Case Else
            ' just move on to the regular error handler
    End Select
            
    strLocale = "iMondayPackReports: RptID " & CStr(lngRptID)
    
    fErr = ErrHandler(Err.Number, strLocale)
    
    If fErr = False Then
        Debug.Print "Error was not logged to Error table.  It occurred in " _
            & strLocale & "."
    End If

    Resume MondayPack_done
Exit Function
