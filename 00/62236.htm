After upgrading PC RAM, waking from sleep resulted in a black screen in Windows 10 - here's my fix

I regularly put my Windows 10 desktop PC to sleep during the day when I'm not using it. I hadn't had any PC sleep related issues until I recently replaced and upgraded my PC RAM from 8GB to 16GB. After changing the RAM I began to notice that every time I woke my Windows 10 PC back up from sleep it would usually result in a black screen (i.e. no display). The keyboard and mouse seemed to be functioning but the display wasn't waking up. Without any display I would press the sleep button again on my keyboard to put the PC back to sleep once more. Waking up the PC after the second or third time would finally bring the display back again. It was weird. I was running the latest BIOS version, I checked all my RAM timings and voltages were correct, tested the RAM, reinstalled the nVidia display drivers, disconnected all USB devices, updated Intel system drivers, and even tried swapping out the RAM for another brand - but it all resulted in the same sleep problem of no display after waking from sleep until the second or third attempt of waking it up.

Then today I stumbled across the fix.

Firstly, I needed to have Hibernate turned on as an option in Windows 10 - I don't use hibernate but just needed to have it available as an option. Hibernate is enabled by default, but I had disabled it months ago. I re-enabled hibernate by running "powercfg.exe /hibernate on" in an elevated Command Prompt. With hibernate now enabled a new advanced setting in the Control Panel > Power Settings appeared, named Sleep > Allow Hybrid Sleep. I found that this "Allow Hybrid Sleep" setting needs to be set to "On".

With hibernate re-enabled and Allow Hybrid Sleep turned on, my screen now displays correctly when waking from sleep. I don't know why my new RAM required this setting when my old RAM didn't but I wanted to share the above with you in case you are experiencing something similar.

Your original question asked if there was a "VNC server" for the Surface platform.  Then when DPil replied that there was no such animal for the Surface RT, you mentioned the VNC Viewer app for iPad.

Please note, the VNC Viewer is the VNC *Client* not the VNC *Server*

A VNC Server is a software component that allows a remote (client) computer to view and control the local (server) computer.

A VNC Client is a software component that allows the local (client) computer to view and control a remote (server) computer.
