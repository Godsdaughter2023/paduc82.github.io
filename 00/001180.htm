One of the most common behavior of Windows Defender is that if it detects a third-party antivirus software, then Windows Defender will shut itself off. Windows is designed to detect the presence of security software installed, and we often we get to see the notification in the Action Center. One of the common issues we have seen in the past, is when there are multiple security software installed. It often creates a conflict between them and in the end, in creates more trouble than doing any good. Frequent freezing, crashing to applications are some of the common issues which occur when you install more than one antivirus software.

So to avoid this situation Windows Defender will turn itself off when it detects another security program. Likewise if this security application has a Firewall then Windows Firewall will also stop working. So in case you want Windows Defender to work, then you will have to uninstall your third-party security software and reboot the system.

john w. vinson say:

I'm guessing that the Athlete field is a Lookup Field - right? If so see http://www.mvps.org/access/lookupfields.htm for a critique of what a lot of us consider a misfeature. The Athlete field appears to contain a name, but it actually contains the numeric ID; and when you do almost anything with the table - such as a Union query or an export - you can't see the name anymore!

The solution is to include the lookup table of athletes in your query, joining it to both [Athlete] and [Other Athlete]; include the name field from the two instances of the lookup table.

 Can I do this directly out of Office 2010 or do I need to purchase another program?

steve rindsberg say:

This is what we've been saying.  Maybe we haven't said it clearly so:

PowerPoint will NOT make a standalone DVD that plays back on any PC w/o the user having to do something and it won't make DVDs at all, not on its own.

If you want a DVD that'll play back as a movie on most computers, use Windows Movie Maker (included with XP and later versions of Windows) to make a movie as I suggested earlier.  I'd start with a short version, maybe just a few slide images, and experiment with the different Save Movie options.

One option it offers is to save to CD in something called HiMat format; the Help file says that this'll play on a variety of computers/players.   Tried that here for giggles.  No joy.  Doesn't play at all on my ancient $25 dirtball DVD player, and doesn't seem to do much on the computers other than put a WMV file on the CD;  doubleclick that and it starts in Windows Media Player.  OTOH, Windows Movie Maker did burn the CD automatically when asked.  And who knows ... maybe these HiMat thingies work better in newer or more upscale DVD players.  That'd be "any DVD player on the planet but mine". ;-)

To make a regular DVD with a menu and all that, you need to be able to convert the movie file from Movie Maker into the movie format required for DVDs.  Neither Powerpoint nor Movie Maker will do that.

I've used Nero (a third-party commercial product) for this in the past.  It lets you add multiple movies to a single DVD, create menu screens so the user can choose which movie to watch, and so on.  I already had the movie made in MovieMaker, so I didn't try this, but I think you can also use it directly to create slide shows ... ie, to take the images you exported from PPT and make a movie from them, include that on the DVD.
