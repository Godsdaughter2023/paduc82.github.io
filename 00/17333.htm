So rather than getting the Device I/O error while trying to open the file, you are getting it after it has actually loaded from the drive and is presumably then running some automatic Macro code?

Since it was apparently a corrupt file that you've eliminated, I would imagine that somehow the corruption was generating an error that the VBA engine was interpreting as an I/O Error.  Sometimes the VBA engine does get these things wrong (or maybe actually correct, depending on the nature of the corruption of the file).

The most likely code to be running would be Workbook_Open() event code.  You could locate that (it would be part of the ThisWorkbook object in the VBA Project browser) and put a STOP statement right at the start of it and then work through it all step-by-step using the [F8] key until the error was encountered.  That would show you where the error was being generated.

doug robbins - mvp office apps & services (word) say:

I doubt that http://social.technet.microsoft.com/Forums/en/word/threads is likely to provide better suggestions than here or Answers Microsoft Office Programming Forum at http://answers.microsoft.com/en-us/office/forum/customize?page=1&tab=no&sort=LastReplyDate&dir=Desc&tm=1373166544565

 

The following may do what MikeGray is after:

 

Dim i As Long
Dim strbmname As String
Dim ffrange As Range
Dim cc As ContentControl
With ActiveDocument
    For i = .FormFields.Count To 1 Step -1
        Set ffrange = .FormFields(i).Range
        strbmname = .FormFields(i).Name
        .FormFields(i).Delete
        ffrange.Select
        Set cc = .ContentControls.Add(wdContentControlText, Selection.Range)
        Set ffrange = cc.Range
        ffrange.End = ffrange.End + 1
        ffrange.Start = ffrange.Start - 1
        .Bookmarks.Add strbmname, ffrange
        Selection.Collapse wdCollapseStart
    Next i
End With

The following should populate the content control with the data that was entered into the formfield:

 

Dim i As Long
Dim strbmname As String
Dim ffrange As Range
Dim ffresult As String
Dim cc As ContentControl
With ActiveDocument
    For i = .FormFields.Count To 1 Step -1
        Set ffrange = .FormFields(i).Range
        ffresult = ffrange.Text
        strbmname = .FormFields(i).Name
        .FormFields(i).Delete
        ffrange.Select
        Set cc = .ContentControls.Add(wdContentControlText, Selection.Range)
        cc.Range.Text = ffresult
        Set ffrange = cc.Range
        ffrange.End = ffrange.End + 1
        ffrange.Start = ffrange.Start - 1
        .Bookmarks.Add strbmname, ffrange
        Selection.Collapse wdCollapseStart
    Next i
End With

Note, the document containing the formfields would need to be protected for filling in forms to allow the formfields to be used, but the protection would need to be removed to allow the code that replaces the formfields with content controls to be used.

When you copy a text that has a different font style and paste it to your Excel file without using your preferred paste option , it will retain the original font style. If you will delete the text and use the "Match Destination Formatting" option to paste it again in the same cell, it will still retain the original style.
