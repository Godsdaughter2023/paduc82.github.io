chip pearson say:

Excel VBA code is contained within a workbook, so you must send the containing workbook or export the code to a text file and send that text file. Just to be sure the macros reside where you think they are (in the workbook to be sent), open the VBA Editor (ALT F11) and then open the Project windows (CTRL R) if it is not visible on the left side of the screen. You should see a project for each open workbook and one for Personal.xlsb. If the macros reside in one of the workbooks, they will travel with that workbook, so you just send the workbook. If they are in another workbook, you can drag the module "folders" from one book to another to get them into the workbook to sent. If the code is in your Personal.xlsb wokrkbook, drag the modules to the workbook to send. Code in the Personal.xlsb file does not travel with any other workbook.

 

You can also export the code to text files and send those files to your user. Select the appropriate module, go to the File menu, and choose Export. This will open a standard Windows Save As dialog, so navigate to the appropriate folder where you want to save the text files, and click Export. VBA will create the module text files with a "bas" extension (e.g., Module1.bas) . Many mail systems will not allows "bas" attachments to get through, so change the extension from "bas" to "txt" and instruct the user to do the reverse when he receives the file(s). The user should open a workbook in VBA, click the Import item on the File menu. This will display a standard Windows File Open dialog. The user should select the bas files to import. New modules will be created, one for each bas file.

It is worth noting that the bas and txt files are just text containers for the code and nothing more. You cannot execute code directly from a bas or txt file. The code must first be imported into the VBA project to enable it to run..

mike h.. say:

Hi,


So you have a simple link to another worksheet that returns zero of the cell is empty and you want to hide all rows that have a zero, is that correct? Yes, then try this code.



Private Sub Worksheet_Calculate()
Dim c As Range
For Each c In Range("D3:D100")
    If c.Value = 0 Then
        c.EntireRow.Hidden = True
    Else
        c.EntireRow.Hidden = False
    End If
Next
End Sub

To setup an @outlook.com mailbox as pop3/smtp:

 

Start the new mailbox guide

Select "Other"

Input your @outlook.com email address and password

Decline the Nokia terms when asked - so it is possible to edit the server names

 

Go to the mailbox settings, advanced mailbox settings, and change

 

Server incoming: pop3.live.com

Safety: On (SSL/TLS)

Port 995

 

Server outgoing: smtp.live.com

Safety: On (StartTLS)

Port 587

 

Adjust the rest of the settings to your preferences, mailbox name, your name, signature, fetch intervals ...
