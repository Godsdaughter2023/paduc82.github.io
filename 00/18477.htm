rhetoric101 say: OK, I was able to get that approach to work, Suzanne! Here's what I did:

  1. Created a multilevel numbering style.
  2. Set the first level to the "Normal" style.
  3. Set the second level to the "List Number" style.
  4. Set the third level to the "List Number 2" style.
  5. Set the fourth level to the "List Number 3" style.
  6. Set the fifth level to the "List Number 4" style.
  7. Set the six level to the "List Number 5" style.

What I noticed after I set this up is that the renumbering worked, but only if I stayed within the level one after I finished the list.

Problem is that when I expand a month in 2013 it also expands the same month in 2012.  Doesn't seem to know the difference between a month in 2012 and a month in 2013.

Any ideas? 

bernie deitrick say:

When Excel groups by date, it splits the date into the values for groups that you selected: Month and Year.  Month no longer has a year associated with it - it is just the month. So by showing detail by month, you show all the details for that month, no matter the year. If you don't want to see the other year's months, then filter by year to only show the year of interest. Otherwise, you can create you own custom date groupings by using a formula of columns added to your data source, like

=TEXT(A2,"YY-MM")

where column A has the dates. Using that column to group will give you the same 'group-ability' as Excel's grouping, but will only expand the specific month when you show details.
