rich michaels - mvp (greatcirclelearning.com) say:

Hi Nathaniel,

Try this code, it should run a bit more efficiently. When you are removing objects from a collection by a loop method starting with object 1 and until you find the last object (x), the system can at times get confused about the existence of a current object held in memory. The method can also be quite inefficient because in each successive iteration it must recreate the collection... the original code is saying "for each individual revision in the full collection of revisions" however, in the previous iteration one of the collection members was just removed via the accept/reject command... thus membership in a new collection must first be determined.

A more appropriate method for working with the revision collection in this case is to work backwards thru the collection.

Hope this helps

Sub RemoveRevisionsExceptHighlights()
    Dim checkingFor As String
    Dim rev As Word.Revision
    Dim i As Integer
    
    checkingFor = "Formatted: Highlight"
    If ActiveDocument.Revisions.Count > 0 Then
        For i = ActiveDocument.Revisions.Count To 1 Step -1
            Set rev = ActiveDocument.Revisions(i)
            If InStr(1, rev.FormatDescription, checkingFor, 1) > 0 Then
                rev.Accept
            Else
                rev.Reject
            End If
        Next
    End If
    
    ActiveDocument.SaveAs fileName:=ActiveDocument.Name & ".pdf", FileFormat:=Word.WdSaveFormat.wdFormatPDF
    
End Sub

sky pei msft say:

Hi DellT,

 

It is not feasible to do that with the links feature. However, you may try the following workaround to see if it meet your needs.

 

Wordaround1: Content query webpart:

In your subsite -> edit page -> add a content query webpart-> edit webpart -> under source, select show items from the following list in the following screenshot -> you can select the library of the parent site which stores the documents -> save the changes. Then you can display the document stored in root site on a subsite as below.

A similar article is for your reference: Access parent site library content from child site

 


 

 

 

Workaround2: Change search center URL

In you subsite -> click the gear on the upper right corner -> site settings -> search settings ->next to Enter a search center URL, type into your search center URL: https://domain.sharepoint.com/search -> ok. (It may take up to 30 minutes to take effect). In this situation, when users search in this site, actually they are search through search center, so they are able to find the documents stored in the parent site.
