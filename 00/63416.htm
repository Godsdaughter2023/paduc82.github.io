greg carmack - mvp 2010-2020 say:

Hi Benjamin -

Unfortunately, without reinstalling you can't change from Legacy (CSM) to UEFI install, taking full advantage of the modern replacement for BIOS. 

This is because to install to UEFI one must boot the installation media (created by Media Creation Tool to be sure) as a UEFI device, then delete all partitions down to Unallocated Space to reset the the disk formatting and allow creating GPT partitions needed for UEFI install. 

This is all illustrated in Clean Install Windows 10 for when you get ready to reinstall.  If you'll follow those steps closely you'll have the best install possible which will stay that way as long as you do.  Then save a System Image Backup so you never need to reinstall again, but if Windows becomes irreparable you can restore to new or replacement HD/SSD in 20 minutes.

A good time to do the reinstall would be with the Anniversary Update.

Hi,

Thank you for posting your response, we appreciate the time and effort that you have invested in to fix the issue.  

I would suggest that you place your computer in Safe Mode and check if you are able to put your computer to Sleep. To do so follow the steps below:

  1. Open Run Command (Windows button +R)
  2. A dialog box will open type msconfig
  3. Locate and select the Boot tab and select Safe with Minimal as option.
  4. Restart Computer.

Note: Follow the same procedure to undo Safe Mode, uncheck the Safe option and restart your system.

 

If the problem persist, then it is possible that the issue is caused by an incompatible or corrupt driver. When the computer goes to sleep mode, Windows sends a request to all hardware parts asking to enter sleep mode. But if the driver for a particular hardware part (like a network adapter or graphic card) is corrupt, it might not respond to these requests and may prevent the computer from entering sleep mode.

To resolve this, I would suggest you to run Windows update to make sure that you have installed the latest drivers for all the devices. Refer to the section ‘To check for updates’ in this article:

 Note: (Also applicable to 10)

http://windows.microsoft.com/en-us/windows-8/windows-update-faq
