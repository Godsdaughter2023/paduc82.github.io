You must use option buttons (a.k.a. radio buttons).

 

If you use Form Controls, you can divide them into groups by creating group boxes (also form Form Controls) and placing option buttons within a group box.

Archive folder appears under Inbox instead of being at the same level - Microsoft Community

alexandru cazacu say:

Archive folder appears under Inbox instead of being at the same level with it in Outlook and OWA

Hello again!

I recently came up across a case in which a user was complaining about having the default mailbox Archive folder located under the Inbox instead of being at the same level with it, right under the Top of Information Store. Although we have an official KB article that speaks about issues similar to this one, it doesn't actually describe the exact steps that we should perform to fix this particular scenario, so I invite you to keep on reading below:

The solution:

Please follow Method 4/Option 2 from article https://support.microsoft.com/en-us/help/2826855/folder-names-are-incorrect-or-displayed-in-an-incorrect-language-in-ou, but modified as described below:

IMPORTANT: Apply these steps exactly like this, only if the Archive folder is located under Inbox and not under the Top of Information Store (this is not the same issue as with Inbox folder being displayed as Archive or the issue with Archive01 folder).

IMPORTANT 2: Before following below steps, make sure that Outlook and OWA are setup to show their interfaces (and default mailbox folders) in the native language of the user. I am saying this because when executing Step 8, you might end up having default mailbox folders in 2 different languages.

1. Right-click the mailbox name at the top of the folder hierarchy in Outlook, select New Folder, and then create a folder that’s named Archive (use the name according to the language which is set on the mailbox). 

​

2. Start MFCMapi in Online mode. To make sure that MFCMapi is in Online mode, select Tools > Options, and then make sure that the Use the MDB_ONLINE flag and Use the MAPI_NO_CACHE check boxes are selected.​ You can download MFC MAPI from here:

• for Office 32bit: https://github.com/stephenegriffin/mfcmapi/releases/download/19.2.19007.645/MFCMAPI.exe.19.2.19007.645.zip

• for Office 64bit: https://github.com/stephenegriffin/mfcmapi/releases/download/19.2.19007.645/MFCMAPI.exe.x64.19.2.19007.645.zip

​

3. Expand Root Container, and then expand Top of Information Store.​

​

4. Select the new Archive folder, and then find the property that's named PR_ENTRYID.​

​

5. Double-click to open the PR_ENTRYID property, and then copy the property value.​

​

6. Select the Inbox folder, and then double-click to open the 0x35FF0102 property tag. This is the old Archive Entry ID.​

​

7. Paste the value of the Entry ID of the new Archive folder that you copied in step 5.​

​

8. Run Outlook.exe /resetfolders. 

9. Move the content from the old Archive folder to the new one and then delete it from under the Inbox.

Hope this helps.

Create a log of when databases are opened - Microsoft Community

rosa_nubie say:

Create a log of when databases are opened

We have a group of databases in a folder which we'd like to clean up. we'd like to create a log that shows:

  1. The database name
  2. Who opens database
  3. The date it's opened

I know that when a db is opened a .laccdb file which contains the username, date, filename is created.  But, I'm not an advanced user and i'm not sure how to handle this. One thought was to create an autoexec macro that outputs the date to a table somewhere. If it was a small number of databases I could add the db name to the query.  But, there are too many databases to do this. any help is appreciated.

scottgem say:

Create a table like so:

tblAccessLog

AccessLogID (PK Autonumber)

UserID

Database

OpenWhen

In the default form (the one that opens with the database) use the On Open event to run code like:

Dim strSQL As String

strSQL = "INPUT INTO tblAccessLog IN 'S:\AccesLog\Accesslog.accdb' (UserID, Database, OpenWhen) " & _

      "VALUES('" & fOSUsername() & "', '" & CurrentDB.Name & "', #" & Now() & "#);"

CurrentDB.Execute strSQL, dbFailOnError

Where S:\AccesLog\Accesslog.accdb is the full path and filename to the file where the accesslog table resides (alternatively you just link this to all your front ends.) 

fOSusername(0 is a function that will return the user's Windows login name and can be found here: http://access.mvps.org/access/api/api0008.htm  Copy this function into a Global module as instructed. 

So every time the default form is opened, a record will be written to this table with the username. database name and date and time.

jay freedman say:
The "space" you refer to is really the paragraph mark at the end of the line, which you can see if you turn on the nonprinting marks by clicking the ¶ button on the Home tab. (Click it again to turn it off.)

In the Editing Options section of the Advanced Options dialog, clear the check box for "Use smart paragraph selection" to stop automatically selecting the paragraph mark.

julie fka profjulie say: On many WM5.0 devices the OEMINFO.xml file was missing and so Microsoft Reader activation could not be completed on the device.  The OEMINFO.xml file is unique to the device and so the manufacturer of each device was responsible for providing the file for the affected devices they manufactured.  There isn't a generic OEMINFO.xml file you could use for this - you need to get one that was created for the XV6900.  The message you get ("the previous version of MS activation.....") is a common message that users got who were missing this file on their device.

This used to be a pretty hot topic in the Microsoft Reader forum.  There was a person there by the name of Christine who was with the Microsoft Reader Team (she posted as msreadergirl).  Christine was extremely helpful and would sent the file to users who sent her an email.  It was quite a long time ago, but perhaps you can still get in touch with Christine on that forum....here's a link to that forum to help you out.
