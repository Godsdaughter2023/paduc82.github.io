Indirect formula to link to new tabs created by a macro - Microsoft Community

kelly_996 say:

Indirect formula to link to new tabs created by a macro

Hi,

I have a macro that creates new tabs and splits some info onto each one. I want to use an Indirect formula to pull info from a new tab onto an existing tab, and I got the formula to work, but it does not adjust as I copy the formula down or over. Which makes me assume I have the formula incorrect.

=INDIRECT("'Sheet8'!A1")

To pull the cell contents of A1 on the newly created Sheet8. But when I copy it down, it stays as A1. If I manually change it to A2 it works, but my sheet is huge and I would rather copy it down if possible?

jeeped say:

... But when I copy it down, it stays as A1. If I manually change it to A2 it works, but my sheet is huge and I would rather copy it down if possible?


The textual representation of a cell address (e.g. "Sheet8!A1") you've constructed is not going to change with a fill right, copy or fill down because it is static text with the emphasis on static. If you are using the INDIRECT() function because you are constructing the cell address piece-by-piece you need to use something that will advance when filled or copied to a new position. For vertical advancement, you can use ROWS() within an ADDRESS() function. Something like,

=INDIRECT(ADDRESS(ROWS($1:1),1,4,TRUE,"Sheet8"))

You should be able to adjust that so your macro contributes the "Sheet8" portion.

To ensure the end user's privacy, Microsoft uses a one-way mathematical algorithm to create the hardware hash used by Product Activation to create the Installation ID. Once created, the hash information cannot be calculated back to its original values. Hardware information is sent through the algorithm in the software on the PC-not at Microsoft-to create the hash. The raw hardware information is not known or sent to Microsoft. Ensuring end user privacy is a No. 1 design goal for Microsoft with Product Activation.

 

Product Activation keeps users from changing or upgrading their hardware.

Not true at all. Users can change or upgrade their hardware. One of the forms of piracy that Product Activation guards against is hard disk imaging. Not all forms of hard disk imaging are illegal. In the case where a pirate copies data from one PC hard drive to another to illegally run the software on two PCs, Product Activation stops that by forcing the copied software to be reactivated. It does so by comparing the hardware on which it was activated to the hardware on which it is now being booted. If the hardware is substantially different, then reactivation is required. If it is the same or similar, then the software will continue to work. Those who upgrade their PC's hardware substantially may be asked to reactivate. Reactivation for this reason is easy and can be completed by contacting Microsoft to obtain another confirmation ID.

The table of contents does not automatically update in any version of Word in response to a subheader being added to the text.  Nor has there been any change in its use/behavior between Word versions 2007 and 2010.

To get the table of contents to update, you need to select it and press F-9 and then check the appropriate radio button to update the complete table rather than just the page numbers.

To modify the table of contents, click on the Insert Table of Contents item in the Table of Contents pulldown in the Table of Contents section of the References tab of the ribbon.
