gord dibben say:

With all the cells available to you on a sheet why do you want to use one cell to input a number to accumulate in the other cell?

You will have no paper trail to follow in the event of an error in one of the entries.

An accumulator cell is possible using the CELL function

Goes like this:  =IF(CELL("address")="$C$4",

Enter this in cell D4 and then in Options>Calculation set Iterations to 1.

Now when you change the number in C4, D4 will accumulate.

Note 1.  If C4 is selected and a calculation takes place anywhere in
the Application D4 will update even if no new number is entered in
C4.  NOT GOOD.
Note 2.  This operation is not recommended because you will have no
"paper trail" to follow.  Any mistake in entering a new number in C4
cannot be corrected.  NOT GOOD.

If you must have an accumulator cell best bet is VBA

Laurent Longre's MOREFUNC add-in has a Function RECALL
which does what you want without the re-calculation problem, but
again there is no "paper trail" for back-checking in case of errors
in data input.

http://ccm.net/download/download-24205-morefunc-macro-add-in-for-excel

John McGimpsey also has VBA code to create an accumulator in a single cell

http://www.mcgimpsey.com/excel/accumulator.html

Dim installer, state

' Connect to Windows Installer object
set installer = CreateObject("WindowsInstaller.Installer")
state = installer.featurestate ("{90150000-0011-0000-0000-0000000FF1CE}","ACCESSMiscFullFiles")

If ( state = -2 ) then
  MsgBox "INSTALLSTATE_INVALIDARG"
 elseif (state = -1) then
  MsgBox "INSTALLSTATE_UNKNOWN"
 elseif (state = 2) then
  MsgBox "INSTALLSTATE_ABSENT"
 elseif (state = 2) then
  MsgBox "INSTALLSTATE_ADVERTISED"
 elseif (state = 3) then
  MsgBox "INSTALLSTATE_LOCAL"
 elseif (state = 4) then
  MsgBox "INSTALLSTATE_SOURCE"
End If

Technical Level : Intermediate

Summary

Below are the various ways you can disable or delete a mailbox in Exchange Online, along with the expected results of these actions.

In cases where you as an Administrator of Office 365 tenant or your customers are doing this or seeking advice on developing a process or strategy, use these tables for guidance.

For information on recipient creation, see Provisioning Recipients vs Normal Users.


Details

Current Behavior:

  • When a user account is permanently removed in AzureAD (remove-msoluser -removefromRecycleBin) the EXO mailbox is no longer permanently deleted.
    • The mailbox becomes soft-deleted with a blank externalDirectoryObjectID
    • Clean up using Remove-mailbox -permanentlyDelete if the account is not on litigation/inPlace Hold. This cleans up the archive mailbox as well.
      • Mailbox and archive are not recoverable after this cleanup action.

  • When a license is removed from a user account, the mailbox will become disconnected.
    • Login is not rejected for the user, but mailbox will no longer be accessible.
    • At this point, to regain access to the mailbox a new license has to be added to the AzureAD user.
