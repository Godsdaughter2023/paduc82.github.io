Okay

step1

Save As, your Workbook with extension .xlsm (macros enabled)

step2

1) press ALT+F11 to open Visual Basic

2) select: Insert > Module and paste the code below on the right 

[...new edit...]

Sub Add_Chart_LineMarkers() ' << start vba ###
'May 10, 2017

Const xAxis1 As String = "B1:I1" '<< labels for primary x-axis /row1
Const xAxis2 As String = "B7:I7" '<< labels for secondary x-axis /row7
Dim ws As Worksheet
Set ws = Sheets("Sheet1") 'data / source sheet name

On Error Resume Next
ws.ChartObjects(1).Delete

Dim N As Integer
N = ws.Range(xAxis1).Columns.Count
Dim v1 As Variant, v2 As Variant
ReDim v1(1 To N)
ReDim v2(1 To N)

Dim t As Integer

t = 1
For Each r In Range(xAxis1)
v1(t) = CStr(Format(r.Value, "dd.mm.yyyy"))
t = t + 1
Next
t = 1
For Each r In Range(xAxis2)
v2(t) = CStr(r.Value)
t = t + 1
Next
Dim cel As Range
Set cel = [A9] 'chart/ top left cell
Dim obj As ChartObject

'chart width =[range("A1:I1") width]  / chart height =[250 points]

Set obj = ActiveSheet.ChartObjects.Add(cel.Left, cel.Top, ws.Range("A1:I1").Width, 250)
Dim c As Chart
Set c = obj.Chart
'set primary x-axis 
c.HasAxis(xlCategory) = True
'<STEP-1 modify primary Y-axis series>
'series1
With c.SeriesCollection.NewSeries
.ChartType = xlLineMarkers
.Name = ws.Range("A4")' series1 name
.Values = ws.Range("B4:I4") 'series1 range
.XValues = v1 'primary category x-axis
.AxisGroup = 1
.Format.Line.ForeColor.RGB = vbRed
For x = 1 To N
With .Points(x) 'modify points/labels
.HasDataLabel = True
.ApplyDataLabels Type:=xlValue
.DataLabel.Font.Size = 11
.DataLabel.Font.Color = vbRed
.Format.Fill.ForeColor.RGB = vbRed
End With
Next
End With
'series3
With c.SeriesCollection.NewSeries
.ChartType = xlLineMarkers
.Name = ws.Range("A6") 'series3 name
.Values = ws.Range("B6:I6") 'series3 range
.XValues = v1 'primary category x-axis
.AxisGroup = 1
.Format.Line.ForeColor.RGB = vbBlue
For x = 1 To N
With .Points(x) 'modify points/labels
.HasDataLabel = True
.ApplyDataLabels Type:=xlValue
.DataLabel.Position = xlAbove
.DataLabel.Font.Size = 11
.DataLabel.Font.Color = vbBlue
.Format.Fill.ForeColor.RGB = vbBlue
End With
Next
End With
'<modify secondary Y-axis series>
'series2
With c.SeriesCollection.NewSeries
.ChartType = xlLineMarkers
.Name = ws.Range("$A5")'series2 name
.Values = ws.Range("B5:I5") 'series2 data
.XValues = v2 'array / secondary category x-axis
.AxisGroup = 2
.Format.Line.ForeColor.RGB = RGB(0, 128, 0)
.Format.Line.DashStyle = msoLineSysDash
For x = 1 To N
With .Points(x) 'modify points/labels
.HasDataLabel = True
.ApplyDataLabels Type:=xlValue
.DataLabel.Font.Size = 11
.DataLabel.Font.Color = RGB(0, 128, 0)
.Format.Fill.ForeColor.RGB = RGB(0, 128, 0)
End With
Next
End With
With c
'set secondary x-axis
.HasAxis(xlCategory, xlSecondary) = True
'set legend
.SetElement (msoElementLegendBottom)
'format chart/plot area
.ChartArea.Interior.Color = RGB(240, 255, 255)
.PlotArea.Interior.Color = RGB(240, 255, 255)
End With
End Sub '<< end vba ###

langware say:

Do you use an email client on a device to review your outlook mail (such as an email app on your phone)?

If so, and if that email app is configured to use the POP protocol to fetch mail from the outlook.com server, then your email on the outlook.com server is being marked as READ whenever it is fetched by your email app.

Microsoft is aware of this problem (for several months) and refuses to fix it.

beth23456 say:

Thanks,

This just started happening a few months ago.  I have had the email app for a while now, way before this started happening.  Thanks for your reply.

langware say:

Microsoft is converting user's outlook.com accounts to a new web interface, and new servers (based on Microsoft Exchange).

This problem only starts happening when ones account is converted.

When an email app (such as one on your phone), that is configured to use the POP protocol, fetches your mail from your outlook.com account (that resides on the Microsoft Exchange server), then Microsoft Exchange marks that fetched mail as READ ... even though you have never opened that mail.

This did not happen on the old outlook.com servers .... it only happens to outlook.com accounts that have been moved to the new servers (and it only happens when the email app that is used to fetch mail from the outlook.com servers is configured to use the POP protocol).

This explains why your email app used to work fine and only recently (when your outlook.com account was moved to the Microsoft Exchange servers) did this problem start happening.

Unfortunately, Microsoft does not provide any way to go back to the old servers.

beth23456 say: So what is the resolution?  Switch email to gmail or cox?  I understand the why, now how about the fix??

langware say:

Switching your email to gamil, cox, yahoo, etc is one alternative.

If you want to keep using outlook.com, and you are comfortable making changes to the email client app on your phone, you could delete the profile in the phone's email app (the one that uses POP protocol to fetch mail from your outlook.com account) and then create a new profile (on your phone's email app) that uses the IMAP protocol.

This assumes that the email app on your phone supports IMAP and that you feel comfortable making the changes and making the necessary settings in your new IMAP profile.

IMAP is more powerful than POP so be careful in making the settings .... dependibg on what settings you enable/disable, actions you take on your phone's email app may affect mail on the outlook.com server.

You should be able to set the IMAP profile so that fetched messages will not be marked as READ on the outlook.com server.
