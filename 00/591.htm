AutoFit to Contents clears any width settings for table columns and adjusts the width based on cell content.

Ok - in more detail ....

There are several ways to do this, but here's one way:

Say (for example), that you downloaded Sounds.exe to C:\Downloads

Open a Command Prompt / Dos Window

It will probably display something like 

"C:\users\Tom>"

- this tells us that the Current Directory (or Folder) is "\Tom", in the "\users" directory, on the "C:" drive

Although we can issue the commands from the current directory, it makes the commands longer, so I find it easier to change to the directory where Sounds.exe is.

So, (in the Command prompt window), type "cd \Downloads" - this will change directory up to the root, and then down to the Downloads directory

Your Command window should now show "C:\Downloads>"

Now, (in the Command prompt window), type "sounds /c /t:c:\Downloads" - this will extract "sounds.msi", from "sounds.exe", and put sounds.msi in the C:\Downloads 

Then start Windows Explorer,

Double-click on the C: drive

Double-click on C:\Downloads  

Double-click on Sounds.msi  (not Sounds.exe), and it Sounds.msi should install, and then be available in the MS Office settings 

That's about as easy as I can explain it  - I hope it makes sense for you {;>))

How to solve strange behavior of self-created styles with bullets - Microsoft Community

a. user say:

How to solve strange behavior of self-created styles with bullets

Everyone who uses bullets in Word knows, when you hit enter after an empty bullet (so no text after the bullet), when using List Paragraph style, the bullet disappears and the active style will become the Normal style.

However, try the following:

  • Start a bulleted list
  • Create a new style and call it, for example, New List Paragraph
  • Hit enter.

You will see that, as you would expect, the bullet disappears. However, the active style remains New List Paragraph and the indentation is incorrect. The active style doesn't switch to Normal. You have to select the Normal style manually to fix this.

Of course, this is not what you want. Especially when you are creating a nice template for everyone in the office to use.

Hope someone has a fix for this bug or a nice solution/elegant workaround.

You can also follow the discussion on SuperUser.

stefan blom say:

Which style is applied as you press Enter at the very end of a paragraph is determined by the "Style for following paragraph" setting in the Modify Style dialog box for the paragraph style that you are using.

To access the Modify Style dialog for any paragraph style, do the following: In the Apply Styles pane (Ctrl+Shift+S), type in the style name and then click Modify.

The best approach to bullets (and numbering) in Word is to make consistent use of paragraph styles. For more on creating bulleted paragraph styles, see https://shaunakelly.com/word/bullets/controlbullets20072010.html (the article applies to Word 2007 and later).

karl timmermans [outlook mvp 2012-2018] say:

have office 2007 and many outlook folders of outlook..........need to transfer all to windows 10 new computer

What you need to transfer from one computer to another depends entirely on the type (POP3, Imap, Exchange etc) email account(s) that are configured

If POP3 - you need to copy the PST file associated with that account

How to Find & Copy the Right Outlook PST Files
https://www.contactgenie.info/how-to-find-right-outlook-pst-files

If IMAP - you will need to copy the PST file set as the default data file for the profile to so that you have your contact/calendar data. IMAP accounts utilize two PST files in OL'2010 and earlier. One for the email folders and one for contact/calendar data. There's no need to copy the PST file containing the email folders since Outlook will re-create it and re-sync with the server. Note, if you are intending on upgrading to Office 365 (Office 2016) - how data is stored differs. One OST file is created for the IMAP account.

If Exchange - nothing to be copied. Outlook will recreate the OST file for the account and resync all data with the server (email and contacts)
