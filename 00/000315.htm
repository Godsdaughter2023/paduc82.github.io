PS....  I wrote:

The remedy is to round expressions that you expect to be accurate to 2 decimal places (dollars and cents), namely the formulas in columns G and R.  For example:

The suggestion to use ROUND applies to all expressions that you expect to be accurate to specific number of decimal places.

For example, in D69 and D70 (and elsewhere), you have formulas that use AVERAGE and AVERAGEIF, and you format the result to display 3 decimal places.

However, note that the underlying value is more precise.  For example, D69 displays 0.259; but it is actually about 0.258733333333333.  So if you reference D69 in an arithmetic expression elsewhere, you are using 0.258733333333333, not 0.259.

That might be exactly what you want.  There is no "right" or "wrong" here.

But if your intent is to use the rounded result (0.259), it would be prudent to compute =ROUND(AVERAGE(D53:D67),3).

Alternatively, if you find yourself rounding every arithmetic expression to the number of decimal places specified in the format, you might consider setting the "Precision as displayed" calculation option by clicking on File, Options, Advanced and paging down.

Usually I deprecate the use of "Precision of displayed" because it can change constants irreversibly if you are not careful to set the cell format before entering the constant or setting "Precision as displayed".

But if your entire workbook is as simple as the example that you uploaded, it might be safe for your purposes.

 

Caveat:  Be sure to make a backup copy of the workbook before setting "Precision as displayed", just in case you discover some undersirable side-effects.


joeu2004 say: WarmZombiesRCool wrote:
I manipulated the data to produce the problematic results I described, which you can see in array I5:I19 (two values tie for #14 in the rank despite the fact that they are different values and the formula is supposed to prevent ties)

Actually, your formula is of the form:

=RANK(G5,$G$5:$G$19,0)+COUNTIF($G$5:G5,G5)-1

and it is COUNTIF that is returning the misleading results, not RANK.



[EDIT] Or more simply:  =ROUND(D5*100 + E5*0.5 + F5*0.2, 2).

Explanation....

If we separate RANK and COUNTIF into separate columns, we see that there are 2 pairs of numbers that COUNTIF considers equal:  G6 & G14, and G12 & G18.

And if we format column G with 13 or more decimal places, we see that each pair does indeed
But appearances can be deceiving.

If we compute =G14-G6-0 formatted as Scientific [1], the difference is indeed exactly zero (0.00E+00).

But if we compute =G18-G12-0 formatted as Scientific, the difference is actually about -3.55E-15, an infinitesimal amount.  So in fact, they are not exactly equal. [2]

It appears that RANK uses the true difference, whereas COUNTIF uses the difference based on the arbitrary "correction" due to the heuristic described in KB 78113.

In any case, the difference between RANK and COUNTIF interpretations go away when the amounts are rounded to the expected precision, namely 2 decimal places in this case.


WarmZombiesRCool wrote:
in array I53:I67 (all sorts of problems, including a #16 ranking despite there being only 15 values).

The explanation is similar.  And so is the remedy:  round the expressions in G53:G67 along with all other formulas in columns G and R.

In this case, COUNTIF thinks that G55, G66 and G67 are all equal.  And indeed, G67 and G55 are.

But =G67-G66-0 formatted as Scientific reveals an infinitesimal difference between G67 and G66; ergo, there is an infinitesimal difference between G55 and G66.

So RANK treats them as different.

[1] The redundant -0 avoids the dubious heuristic poorly description under the misleading title "Example When a Value Reaches Zero" at http://support.microsoft.com/kb/78113.  Because of this heuristic, Excel sometimes forces differences to be exactly zero and comparisons to be exactly equal when in fact they are not.  I suspect that COUNTIF implements the same heuristic.

I format results as Scientific so that we can see the infinitesimal differences that sometimes arise in calculations due to the way that Excel represents number and performs arithmetic, called 64-bit binary floating-point.  See KB 78113 for details.


[2] We do not see such small differences when formatting to 2 decimal places, as you do.  And sometimes not even when formatting to 13 or more decimal places because Excel formats only up to the first 15 significant digits, rounding the 16th.

ken sheridan say: Your syntax for this line is wrong:

strWhere = "[MemberID] = "" & Me.[MemberID]"""

If MemberID is a text data type it  should be:

strWhere = "[MemberID] = """ & Me.[MemberID] & """"

More commonly IDs are a number data type however, often an autonumber for convenience.  In which case the syntax would be:

strWhere = "[MemberID] = " & Me.[MemberID]

HDMI cables do support audio signals, but not all HDMI-enabled video cards support audio. If your HDTV has an audio input, you might be able to connect a separate audio cable from your computer's sound card directly to the TV. Otherwise, you'll need to connect the audio signal to a different output device, such as external computer speakers or your home stereo system. Follow these methods and check if it helps:

Method 1:

By default Windows does not route sound over the HDMI output. To change this we need to change the settings in control panel.

Note: This will turn off sound from the speakers or normal audio output on the laptop. To get that back you will have to reverse this process.

a) Press Windows key + X
b) Click on "Control Panel"
c) With the Control Panel in "Control Panel Home" view please click on "Hardware and Sound"
d) Where it says "Sound" in green letters there are three options below that are in blue click on "Manage Audio Devices"
e) This will bring up a window labeled "Sound"
f) On the "Playback" tab there is listed the sound options available to you. One should read "Digital Output Device (HDMI)" Click on that.
g) There should now be a check mark on that option, make HDMI sound output as default .
h) The Sound should then play through your HDMI hookups to the TV.

If the issue persists then I suggest you to update the sound card driver and check if it helps.

Method 2 :

You can try the following steps to update the sound card driver.

a) Press Windows key + X key when you are at desktop.
b) Select Device Manager.
c) Double-click Sound driver from the left panel to Update.
d) Right-click the device, and click Update driver Software
e) Windows will prompt you to confirm the device’s removal. Click OK to remove the driver.
