wunders say: Well, when I tried it last night, it worked like a charm... but it was a simple case.  When I tried it with a more normal case (i.e. trying to start a program in the Program Files folder), I got the same result as you.  After a bit of playing around, I found that the problem is that when using the "Start" command, the command that you give it can contain no "space" characters or quotation characters in it.  There was no way that I could escape out the spaces so that it would recognize the executable.  I finally found a way to get around this....  and that is to use "8.3" filenames for the path to the executable.  If you bring up a command prompt window (Start -> Run -> "cmd"), Then change directory to the top directory by typing the command:
     cd  \
and then perform a directory command with the "/x" option:
     dir /x
you will see that another name for the "Program Files" directory is "Progra~1" (no spaces!).  If you then change directory into the program files directory:
     cd  progra~1
and repeat the "dir /x" command, you will see the name of the directory containing your executable and the corresponding 8.3 filename for it that contains no spaces.

So if, from a command prompt, you enter the following command:
    start  /abovenormal  c:\progra~1\outloo~1\msimn.exe
for example, it will start Outlook Express with higher priority.

So if you can back this into your shortcut, I think it will work.

Hi William,


You may disable the debugger application by editing the Windows registry. Here's how:

  1. Press the Windows + R keys from the keyboard to launch the Run dialog box.
  2. Type regedit, then press the Enter key to launch the registry editor application window.
  3. Use this registry search function to find the key: HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\AeDebug.
