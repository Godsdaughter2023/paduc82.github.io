rick rothstein, mvp - excel say:

Just to point out... if the cell color is due to Conditional Formatting, the code you posted will not work. I mention that because I really do not see anything wrong with the code (aside from the fact it is not testing for yellow), but what is the Worksheet_Change event header line doing there (it is hard to tell because you did not post an End Sub but you do have an extra End Function)?  CTest is a function and, as such, you call it from code (or a worksheet as a UDF), but you do not just include it within another procedure. If you were trying to run it inside that Worksheet_Change event, the proper way would be to use code something like this...

Private Sub Worksheet_Change(ByVal Target As Excel.Range)
    MsgBox "Is target cell yellow? - " & CTest(Target)
End Sub

Public Function CTest(targetCell As Range) As Boolean
    Dim colorIndex As Long
    colorIndex = targetCell.Interior.Color
    If colorIndex = vbYellow Then  'VB has some predefined constants for certain colors... yellow is one of them
       CTest = TRUE
     Else
         CTest = FALSE
    End If
End Function

Since you want a True/False return value, the function can be simplified...

Public Function CTest(targetCell As Range) As Boolean
    CTest = (TargetCell.Interior.Color = vbYellow)
End Function

Note that I changed the return type for the function to Boolean. Also, if your "yellow" color is not a **pure** yellow, then the above code won't work because the vbYellow constant is for pure yellow. To find the code number for your particular yellow, you can execute this line of code in the Immediate Window inside the VB editor...

? Range("F5").Interior.Color

where I have assumed F5 is the cell that is colored yellow (change to suit your actual conditions). Then you would just use that number in place of the vbYellow constant in the code I posted.

Calculating Age - Microsoft Community

helenwm say:

Calculating Age

I am using Access 2010 to enter data for a Free Store at our church.  I have a table that includes each member of the family group and his/her age the date the  family joined.   Since I don't have birthdates I know the ages will be approximate.  We need to know when they become approximately 18 so they can receive adult items.     I assume I need to write a query showing ages for the current year so I can print the table for Free Store day.    I am not good at this so I need an elementary explanation. 

 

Thank you in advance.

ken sheridan say: If you know the date when someone joined and their age at the time you can compute their approximate age now by adding the difference in years between the date joined and the current date to their age when they joined.  If you have columns named  DateJoined and Age for instance in a table you can see how it would work in the debug window:

DateJoined = #2010-07-04#
Age = 64
? Age + DateDiff("yyyy",DateJoined,Date())
 70

This is my data and it gives my current age correctly, but it would give my age at 70 any time during 2016, so it would be incorrect prior to my recent birthday as I would then have been 69, and it would be incorrect next year as it would make me 71 from 1 January onwards, whereas I'd actually be 70 until November.

John's expression will do the same.

If you can obtain people's dates of birth you can easily compute their age accurately with the following function:

Public Function GetAge(varDoB As Variant, Optional varAgeAt As Variant) As Variant

    If IsMissing(varAgeAt) Then varAgeAt = VBA.Date
    
    GetAge = DateDiff("yyyy", varDoB, varAgeAt) - _
        IIf(Format(varAgeAt, "mmdd") < Format(varDoB, "mmdd"), 1, 0)

End Function

Using my DoB you can see how it works in the debug window:

DoB = #1946-11-01#
? GetAge(Dob)
 70

john w. vinson say:

Well, a conservative estimate would be to calculate their age assuming they were born on January 1. This can easily be done if you have a Long Integer Age field in the table (the age at joining) and a JoinDate field:

[Age] + Year(Date()) - Year([JoinDate])

So if the were 14 when they joined in 2011, they would now be 14 + 2016 - 2011 = 19.

alan wu msft say:
Hi Hawkins,

To troubleshoot the issue, please make sure the following 4 points:
1. On the owner’s Outlook delegate setting, the delegate (admin assistants) must be listed to have correct delegate permission. To set the permission in the Outlook for Windows (this cannot be set up on the Outlook for Mac client):
a. Configure the owner’s account on the Outlook for Windows client, click File > Account Settings > Delegate Access > Add.
b. Find and add the name of the person who is going to be the delegate.
c. Click the Calendar menu, and select Editor right.
2. On the owner’s machine, Outlook must be running default profile that align with the current sign-in Skype for Business account.
3. Both owner and delegate must be running Skype for Business full client, basic client does not support this feature. A full Skype for Business application can only be downloaded from Office 365 ProPlus, Office 365 Enterprise E3, E4 and E5 subscriptions’ portal.
4. On the owner’s Skype for Business client, the delegate must be listed, this will happen after ExchangeDelegateSync completed. (This is an automatic sync, performed at backend)

If the issue persists, it is recommended that you provide the detailed steps that you perform the task so that we can check if there is any clue.

Moreover, since the assistant has the full access to the manager’s mailbox, you can let the assistant organize the Skype for Business Online meeting in Outlook Web App (OWA) as a workaround. The detailed steps are:
1.    Sign into OWA with the assistant’s account.
2.    Click user profile> open another mailbox> type the manager’s mailbox address> Calendar> create a new meeting> Skype meeting.

Regards,
Alan

It’s not possible to stop syncing contacts with Outlook Hotmail Connector. If you don’t want your contacts to sync with your Outlook, then you can set up your Hotmail as a POP3 account. POP3 will only download the inbox folder from the server.

http://support.microsoft.com/kb/287532

POP server: pop3.live.com (Port 995)  

POP SSL required? Yes

User name: Your Windows Live ID

Password: The password you usually use to sign in to Hotmail or Windows Live

SMTP server: smtp.live.com (Port 25 or 587)  

Authentication required? Yes (this matches your POP username and password)

TLS/SSL required? Yes
