In my task list, all of them look correct and have links to their respective subproject files in the task information tab.

john - project say:

Rachael,

Unfortunately you violated one or more of the "do not do" when working with linked structures in Project. To clarify, you must never rename, move, overwrite, or save off to another location any of the files in a linked structure. You should also not operate the structure over a network and all files should reside in a single directory on your local drive.

But, now that you are in this position, what can you do? Here is what I suggest

1. Open the master with write access for each subproject

2. In the master select the subproject summary line insertion point for a subproject

3. Go to Task > Properties group > Task Information > advanced tab

4. Uncheck the "link to project" option and hit "OK"

5. Repeat steps 2 through 4 for each subproject summary line insertion point

6. Once everything is unlinked, save all files. Note, if you do not have enough memory to have all files open at once, periodically do a "save all files" and then start again to pick up the remaining files.

7. Delete the master, you'll need to create a new one after each of the individual subproject files are further cleaned up

As an added measure of cleanup I'd do the following

8. Save each subproject as XML

9. Re-open each XML file in Project. Note, you will lose any custom formatting with this process

Once you have all subprojects cleaned up, you can then re-create a new master and insert them.

Question. Do you really need a dynamic master? If you really only need the master for periodic reporting, then a much safer approach is to create a static master. A static master is created by unchecking the "link to project" option in the lower right corner of the Insert Project window. A static master is a snapshot in time of all the inserted subprojects and you will need to re-create a new static master any time you need to do an all-up report, but that process can be automated by recording a macro the first time, then run the macro after that.

Hope this helps.

John

suzanne s. barnhill say:

Another difference between the two panes that I had forgotten about is that, while the Styles pane can be set to show or not show a preview of the style, the Apply Styles pane always shows a simple listing.

An even less obtrusive solution for styles is to add the classic Styles dropdown to the QAT. It will always display the style in use at the insertion point and, when opened, will display whatever styles are selected in the Style Pane Options. Unlike the Styles pane, it always shows the styles with formatting (there is no option to get a simple list), and it has the further drawback that it lists all the table styles, but it definitely takes up less real estate than either of the panes; unfortunately, unlike in Word 2003 and earlier, its width is not configurable, so sometimes style names are truncated, though of course the full name is displayed when the list is opened.

yoga yao msft say:

Hi Cyndie,

You can open Apply Styles toolbar: Home> Styles> Apply Styles (or via the keyboard shortcut Ctrl+Shift+S). It displays the current style; applying another style requires using the dropdown.

You can open Style Task Pane: Home> Styles> Style dialog launcher (or via the keyboard shortcut Alt+Ctrl+Shift+S).

The advantage of the Styles task pane over the Apply Styles toolbar in applying styles is that a full list of styles is displayed at all times. If you can see the style you want in the task pane, then you can click on it without having to first click to drop down a list.

The disadvantage of the Styles task pane in seeing what style is currently applied is that, when the list is longer than the task pane, it may be necessary to scroll the list to see what style is selected.

suzanne s. barnhill say:

Bullets are generally not taken from the default paragraph font, however. Most commonly, a Symbol or Wingdings symbol is used for the bullets. If you click the arrow beside the Bullets button and choose Define New Bullet, the Define New Bullet dialog opens with the current bullet format selected (by default this will be the small round bullet). If you then click on Symbol, you'll see that the Symbol dialog opens with the Symbol font displayed.

If you first change the bullet style to the check (tick) mark and open Define New Bullet, the Symbol dialog will display the symbols in the Wingdings font. The bullet style that uses colors is actually a "picture bullet." You get the idea!

It's still a long shot, but it just might be the answer.

Hi LL One Holding Sdn Bhd

Thanks for posting in our forum.

When exporting a ppt file into a pdf file, pdf may involve more info than the original file, such as font info, the quality of the pictures. As a result, you may get a pdf file which is larger than the original ppt file.

You can't set the PDF file to smaller on PowerPoint, you can set the print quality of the PDF to High, Medium, or Lowâ€”depending on your need. By default, the print quality is set to Highâ€”which is the optimal choice for printing a PDF.

Please follow these instructions to set the print quality of the PDF:

  1. Click the PowerPoint menu > Preferences.
  2. In the PowerPoint Preferences dialog box, click General.
  3. Under Print Quality (Paper/PDF), set the Desired Quality to an option of your choice.

For more information, you can refer to the part "set the print quality of the PDF" in this article Save PowerPoint presentations as PDF files.

doug robbins - mvp office apps & services (word) say:

You have the "white space" at the top/bottom of the document hidden.

You can either

Click on the Office Button and then on Word Options and then on Display and check the box for "Show white space between pages in Print Layout view"

or

Move the mouse cursor to the top of the page until it changes into two vertically opposed arrows and a tool tip "Double click to show white space" appears and then do that.
