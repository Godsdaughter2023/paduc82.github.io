rob koch say:

PID - Process ID

TID - Thread ID

Microsoft antimalware products would need to scan the processes and threads in memory to confirm they don't already contain malware, since cleaning the malicious code out of the files on disk will do no good if those already running malicious programs in memory put them right back.

The following thread on the MSDN - Microsoft Developer Network discusses a bit more about the PID and TID in the answers.
