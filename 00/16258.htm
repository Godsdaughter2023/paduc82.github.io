The question is why do edits to the contacts document library at the root of the site show up in the activity report for these two sub-sites?

cliff gu msft say:

Hi Rob,

From your description, I understand as follows.

1. You set unique permissions on site 1,  site 2 and the HR site. To site 1 and site 2, the HR site is the root site.

2. One test user has the Read permissions on the HR site and on the top-level root site.

3. Another test user has the Edit permission on the site 1 and the Read permission on the top-level root site.

If the permissions of the files in the contact library in the top-level root site inherit from the top-level root site, the two test users will still see the files showed in the Site Activity web part. But they can’t edit them as they only have the Read permission on the top-level root site. If you don’t want the test users to see the files in the Site Activity web part, you need to set unique permissions on the files. Please go to the top-level root site>go to the contact library>click on the files you don’t want the test users to see>Share>Manage Access>Advanced to set the unique permissions. You can do a test and let us know if you need further help at your convenience.

About the PDF issue, I’d also like you to check if you set the unique permission on the PDF file. If yes, the test user without the permission on the file shouldn’t be able to see the file in the Site Activity web part.

Setting a Default Language affects only new presentations and Placeholders/Text Boxes created after the Default is specified. It does not have any effect on existing slide content.

stefan blom say:

There is no ready-made macro, if that's what you're asking, but copying styles is supported in Word's object model. The following example copies a style named TestStyle from the current document to a template called testtemplate.dotx.

Sub CopyAStyle()
    Application.OrganizerCopy Source:=ActiveDocument.FullName, Destination:= _
        "C:\Users\JohnDoe\AppData\Roaming\Microsoft\Templates\testtemplate.dotx", Name:= _
        "TestStyle", Object:=wdOrganizerObjectStyles
End Sub

Of course, you'll have to adjust template paths as appropriate.

For the basics on using macros in Word, take a look at http://www.gmayor.com/installing_macro.htm.
