    We don't know exactly how you moved the user folders but it certainly involved creating junction points pointing to the new location.  Most backup applications do not traverse NTFS junction points or symbolic links as this may cause the application to recursively back up the data (backing up the same data over and over again until the application chokes or until the backup media runs out of available space).  Your best bet is to backup the user data under a separate backup.

    John

    1)  Terabyte Image for Windows  not free but probably the best $40 you'll ever spend!

    4)  Yes, the operating system will look for the profiles at their present location (as recorded at the ProfileList registry key), if you restore the profiles to the same location the operating system will find them there, however if the Windows backup utility cannot back them up you will need another method to reliably backup and restore them when the operating system is not available.

    Please note that drive letter assignments are persistent on Windows installations, this is recorded in the mount manager's database at HKEY_LOCAL_MACHINE\SYSTEM\MountedDevices, when you restore the system image the operating system will keep the same drive letter assignments providing that the disk signatures have not been erased.  The disk signature is stored in the MBR, if you wipe the MBR on your disk E: it may not be assigned the same drive letter when Windows discovers it and Windows will not then be able to find the restored profile on the disk.  If nothing has changed in the pc (no additional disks were added) Windows will most likely assign the same drive letter to the your second disk but don't take it for granted that it will as Windows drive letter assignments can be unpredictable at times.  You can read more about this here: http://www.multibooters.com/articles/hard-drive-disk-signature.html  Of course, if the whole disk is imaged this is moot as the original MBR & signature will be restore during bare metal restore.  (Note that GPT styled hard disks (GUID partition Table) use a different method to identify disks from one another so this information does not apply to GPT disks).
