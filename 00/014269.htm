john korchok say:

Here's a work method I've used to create Word tables with linked Excel cells that have a different layout than the Excel original:

  1. Use Home>Paste Special to paste a cell into a table. Choose the Paste link option. The default HTML format is fine.
  2. With the pasted data selected, hold down the Option/Alt key and press F9. The link's field codes are displayed.
  3. Copy and paste the link, manually editing the R1C1 value to point to an Excel cell.
  4. When you're done, select the table and press Option/Alt + F9 to display of the field values.
  5. Update each link to display the correct values.

Office 365 License Transfer from one Tenant to another Tenant - Microsoft Community

dagonr say:

Office 365 License Transfer from one Tenant to another Tenant

I was looking for an official document on the process to transfer Office 365 licenses from one tenant to another tenant.  While I could not find an official document, I did find a lot of misinformation on this topic so thought I would clear this up.  While I could not find an official document, I did find a lot of misinformation on this topic so thought I would clear this up. 

Transferring all Office 365 licenses from one tenant to another is absolutely possible - at least when an Enterprise agreement is involved.  You do not need to purchase new licenses for the target tenant as has been stated in several other posts.  They call this a remap.   Ownership if the licenses remain with the original party but the license usage is remapped to the new tenant.

The Office 365 Support team can help start the remap process.   The following information is asked to verify the account:

1. Microsoft Online Services ID.  This will be the ID used by the Administrator on your account to sign in to the Microsoft Online Services Portal.
2. The name of your company as it displays on your Microsoft Online Services account.
3. The full name of the primary contact for your company.
4. The complete address for your company.

They will then ask for the following information:

1. Enrollment Number:
2. All subscriptions moving to the new tenant: <Subscription type and seat count>
3. Are there any subscriptions on the existing tenant that you do not intend to move to the new tenant?
4. Incorrect/Current tenant Domain: <*.onmicrosoft.com domain the subscription(s) is/are currently provisioned on>
5. Correct/Destination tenant Domain: <*.onmicrosoft.com domain the customer wants the subscription(s) provisioned under.>
6. Reason the move is being requested:

Once completed, they will then place the services in the Enterprise Agreement on hold in the previous tenant and will then be moved to the preferred account immediately which means that it should be updated in real time.  After the licenses are transferred, they can still be accessed in the old account for around 3 months as they will just be placed in an "in grace period" state for migration purposes which means that the users will not be affected once we have the agreement moved.

henrynavigator say:

append data to a cell

In Excel for Mac, how can you append a data string to a cell?

j.e. mcgimpsey say: Should one assume that there is text in the cell to begin with? If so...

For one cell, just select the cell, click in the formula bar, and type or paste the string. Hit return.

For multiple cells (I'll assume in A1:A100 for this example):
  • in an empty column (say, J), put this in Row 1
                =A1 & "<data string here>"
  • Copy that cell down to J100.
  • Copy J1:J100
  • Select A1:A100
  • Choose Edit/Paste Special, selecting the Values radio button
  • Delete/Clear J1:J100
