sent items not being saved in correct sent folder when another aplication creates and sends an email through outlook

We have the correct IMAP settings to create and send email, and the email messages that are created within outlook are saved to the correct sent folder on the server. The problem we seem to be running into is when another application uses outlook to create and send an email, then outlook automatically saves the copy of the message created in the local outlook data file and not on the server that the email account is setup to save sent items.

mukherjee ashutosh say:

When sending emails using the right-click menu (Windows Shell Context menu) on a file, it calls the default email client using MAPI (Messaging Application Programming Interface). When MAPI is in use to send emails it always uses the default settings for the email client on the computer. Since Outlook is set as the default mail client, therefore it’ll use all default settings for Outlook to send emails and in your case the default storage location is the local PST.

 

Possible workaround: drag and drop the target emails from the Local PST Sent Items folder to the IMAP Sent Items folder.
