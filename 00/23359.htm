steve rindsberg say:

This is one of the differences between PowerPoint and Word.  In Word, everything is normally in one big long stream of words/graphics/etc.  Add something at the top, everything below moves downward.

 

PowerPoint is more of a freeform collection of shapes on each slide.  There's no overflow from slide to slide (as there would be in Word, from page to page) and when you move one shape around or change its size, the other shapes (e.g. your table) don't move.

 

You'll have to move the table manually.

The method described in my post is all about "building blocks". Word divides its building blocks into groups, which it calls "galleries". There's one for built-in headers, one for footers, and so on. This is described in more detail at https://gregmaxey.com/word_tip_pages/building_blocks_autotext.html.

When you select something in your document and press Alt+F3, it opens a dialog box:

The second item in the dialog is a dropdown where you can select which gallery you want to add your new building block to. The default is AutoText, but there are lots more in the dropdown:

The bottom half of that list is a series of "custom" galleries that you can use for whatever you want.

The point of this is that when you insert a Building Block Content Control in your document, you set one of its properties to the name of the gallery where you stored your building blocks, and then that content control will offer only the building blocks that are in that gallery.

If you don't need a group of special graphics, you could instead use a regular dropdown content control with characters from the Segoe UI Symbol font or another symbol font. For example,

jay freedman say:

To make the Alt+F3 keystroke do anything, you must first select the image or text that you want to save as a building block. Fill in a name for the building block, and choose a gallery (I suggest using the Custom 1 gallery, as shown here); leave the Category as "General".

Important: Leave the "Save in" set to Normal.dotm if this form will be used only on your computer. Entering a Name for the control is optional; it will replace the "Quick Parts" label. If you don't enter a name, the gallery name will display in the label instead.

Click the down arrow on the control's label and choose the icon that you want users to see first.

Inserting PDF document into a word document - Microsoft Community

terrynorthcutt say:

Inserting PDF document into a word document

I want to insert a pdf file into a word document and then be able to print the entire document.  So far, when I insert the pdf, I only get a link to click on.
jay freedman say:

If you drag/drop a PDF file into a Word document or click the main part of the Insert > Object button in Word, you will get an "object" in the Word document that acts like a link to the file (you can choose whether to embed the object in the document or just link to it).

Instead, click the down arrow on the right end of the Insert > Object button and click Text from File, then select the PDF file. Word will then "open" the PDF and convert it to text if possible, just as it would if you used the File > Open command to open the PDF. The Insert > Text from File command adds the content of the file to the current document at the cursor location.

dwade100 say:

How do you change the width of charts columns?

Hello, 

In the most recent chart I've made the columns of my chart look like pin strips. I'd like to make them much wider. 

My previous chart does not have this problem. 

The difference between the two is the previous chart only has 1 unit for the horizontal axis while the current chart has 5 units. I have no idea how to change the number of units as neither has chart has x-axis coordinates. 

I'd appreciate any help you can offer.

Thanks!
mike middleton say: The usual way to change the width of the vertical bars in a Column chart type is to change the gap width (in all versions of Excel, Windows and Mac).

Select the data series by clicking on one of the bars (just to be sure the data series is selected).

Right-click or control-click, and choose Format Data Series from the context menu.

In the Format Data Series dialog box, select Options.

Then change the Gap width.

For wider bars, make the gap width smaller.

For narrower bars, make the gap width larger.

-  Mike Middleton, www.TreePlan.com

bob jones aka: cybertaz say:
As is often the case, your problem is being caused by merged cells. Fortunately, this is one of the lesser evils inflicted by that abominable "feature" :-) If you unmerge the cells & recreate the chart you should be fine.

As an alternative, you could create formula links elsewhere that refer to cells C50, C51, E50 & E51 then generate the chart based on the cells containing the formula links. Just be sure to switch the Rows/Columns plotting.
