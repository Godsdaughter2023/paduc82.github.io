How to Create a Tool Tip Text for a Check Box in Excel using Macro/VBA

Hi All,

I want a message to be displayed when the move cursor is move over the command button or check box.

how to write the VBA codes/ Macro for this.

I'm using Excel 2013 on Windows 7 platform.

andreas killer say:

On a UserForm you can use the ControlTipText property of the control.

But in a sheet it is not so easy:

  1. You need a hidden / invisible control for the tool tip.
  2. Each "user" control must have a MouseMove event routine. Inside that routine make the "tool tip" control visible and move to an appropriate point.
  3. Unfortunately the sheet hasn't a MouseMove event routine, so you can not hide the "tool tip" control when the mouse leave the "user" control. As workaround, use the Worksheet_SelectionChange event routine and hide the "tool tip" control when called.

Thanks Roady.

That Outlook considers the " modified" date rather than " received" date threw me off track as well. The work around was to :

1. Make a new set  archive folders.

2. Delete files from the "Inbox" which are within a desired range date and move them into the respective IN and SEND subfolders of the Archive Folders.

3. Archive the "Archive folders" containing all its subfolders by selecting it .

4. On the pop up screen on archiving requirements, indicate the current date.  That is most practical since not sure  as to when any of those selected emails were last modified. This is the key!.

5. Once that is done and a file name assigned, click OK. The archive will produce a pst file with all your emails in their respective named subfolders . In fact the archived folder will appear in the folder list. 

6. To check for the pst file, just go to File - Open - Outlook Data File. You should find it there. Works for me.

Hi Linh,

We need to connect to Exchange online first in powershell. And here is the command to edit the allow list:

set the allow list users addresses:

Set-DistributionGroup "group name" –AcceptMessagesOnlyFrom user1@domain.com,user2@domain.com

add the specific users to allow list:

Set-DistributionGroup "group name" –AcceptMessagesOnlyFrom @{add="user3@domain.com","user4@domain.com"}

remove the specific users to allow list:

Set-DistributionGroup "group name" –AcceptMessagesOnlyFrom @{remove="u3@domain.com","user4@domain.com"}

use the following command to check the allow list users:

Get-distributiongroup -identity "group name" | fl acceptmessagesonlyfrom

Reminder: we need to type "" in the command in powershell.

How do I numerate formulas in MS Word 2013? - Microsoft Community

cup_of_coffee say:

How do I numerate formulas in MS Word 2013?

If possible, with no use of tables. It's not practical to me.
suzanne s. barnhill say:

You can number equations without using a table, but if you number them with a caption using References | Captions | Insert Caption and then want to cross-reference the caption, you'll find that you get the equation with it, even if you choose "Only label and number."

Assuming a display equation (centered in its own paragraph), set a center tab stop at the center of the line and a  right-aligned tab stop at the right margin. Tab to the first stop and insert the equation, then tab to the second stop and use References | Captions | Insert Caption to insert an Equation caption. If you want just the number, check the box for "Exclude label from caption." If you want parentheses around the number, you'll have to add them yourself.

Another way to number the equations is just to use a manually generated SEQ field: { SEQ Eq }, but you will not be able to cross-reference this unless you bookmark it manually.

If you have used the native Equation Editor in Word 2013 (as opposed to the old Microsoft Equation 3.0 object), you will find that adding a caption to a paragraph containing an equation applies the Caption style, which in Word 2013 is italic and smaller than the body text. You can get around this by modifying the style, or you can just use a table as described below.

If you need to cross-reference the equations, a table will be required. The usual suggestion, if you want the equation centered, is to insert a three-column, single-row borderless table. Make the third column just wide enough to accommodate the caption and the first column the same width, leaving the center column for the equation. Insert a dummy equation and caption and then select the entire table and save it as an AutoText entry (Quick Part, building block). When you want to insert another equation and caption, insert the AutoText entry, then replace the equation.

charles kenyon say:

You specified without macros. I'm tossing one out, anyway.


The following macros, if placed in your document will append a date to whatever is in the Title Property. Note, no provision is made for more than one version in a day.

These should run automatically whenever the document is saved.


To see what to do with a macro, see Graham Mayor's Installing Macros.


    ' Run as substitute for FileSave to add date to document title

    '
    On Error Resume Next
    '
    Dim strName As String, dlgSave As Dialog
    Set dlgSave = Dialogs(wdDialogFileSaveAs)
    strName = ActiveDocument.BuiltInDocumentProperties("Title").Value

    strName = strName & " " & Format((Year(Now() + 1) Mod 100), _

        Format((Month(Now() + 1) Mod 100), "0#") & "-" & _
        Format((Day(Now()) Mod 100), "0#")
    With dlgSave
        .Name = strName
        .Show
        
    End With
End Sub

Sub FileSaveAs()
    Call FileSave
End Sub
