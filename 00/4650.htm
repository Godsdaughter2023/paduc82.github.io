bernie deitrick say: If you want to search formulas, then you have to use VBA to return the formula first, like this in cell B1:

=CFormula(A1)

with this code in a standard codemodule in the same workbook, saved as a macro-enabled .xlsm

Function CFormula(C As Range) As String
CFormula = C.Formula
End Function

Then you can use a formula like

=IF(NOT(ISERROR(SEARCH("~*",B1))), "EST","P")

You can also combine the two into one cell:

=IF(NOT(ISERROR(SEARCH("~*",CFormula(A1)))), "EST","P")

jeeped say:
... what if I wanted to average the 30 most recent entries from column B by date with the same scenario already mentioned and also add the 30 most recent entries from column B by date ...

You can hard code the 30 into the formula as you mentioned or put it into a cell and reference the cell. Let's assume that you put 30 into Z1. Your two formulas would be,

=AVERAGEIF(A:A,">="&LARGE(A:A,MIN(Z1,COUNT(A:A))),B:B)
... and,
=SUMIF(A:A,">="&LARGE(A:A,MIN(Z1,COUNT(A:A))),B:B) 

You may want to tighten up those full column references (e.g. A:A, B:B to A2:A999, B2:B999) to save some processing power. You can reference cells that are blank or contain text and they will be happily skipped over byt the SUMIF() and AVERAGEIF() functions. I've added a MIN(COUNT()) function to make sure that you are not asking for more values than are available.

How to anchor a table and have it repeat on all pages - Microsoft Community

chrisyl say:

How to anchor a table and have it repeat on all pages

Hi,

 

I’m trying to figure out how to allow users of a Word 2007 document, which will be password-protected, to enter data into a small, one-row table containing text form field boxes.  I need the table anchored in place just below the header on each page, and I’d like it to repeat when a new page is added, like the header and footer do.  I’ve been experimenting for hours, trying text boxes and frames, but I cannot figure this out.  I don’t have any experience with macros either.  Any help would be greatly appreciated!

 

Chris

jay freedman say:
As often happens in Word, you have mutually exclusive requirements so you can't do what you want in a straightforward way.

In this case, the only reasonable way to repeat a table on every page is to anchor it in the header and make it "float" below the header area. However, there is no way in Word to put legacy (protected) form fields in a header/footer, or in anything anchored there. That means you must use something other than text form fields.

The best alternative is to use document variables (invisible storage managed by a macro) to hold the information, and to use DocVariable fields in the table to display the values of the variables. Greg Maxey's article http://gregmaxey.mvps.org/word_tip_pages/Old_Tip_Pages/Create_and_employ_a_UserForm.htm is an excellent tutorial on all the steps needed to make a userform that accepts information from the user and stores it in document variables.

The one other thing you'll need is a way to run the macro (Greg calls it Sub CallUF()) that displays the userform. There are a couple of possibilities. One way is to change the name of the macro to Sub AutoNew(), so it runs automatically when anyone creates a new document based on this template. (By the way, make this a macro-enabled template, *.dotm, and not just a document. That will prevent people from accidentally overwriting the original with a filled-in version, a common mistake.)

A better way is to add a button to the ribbon that runs the macro when clicked. Modifying the ribbon isn't terribly difficult (http://gregmaxey.mvps.org/word_tip_pages/Old_Tip_Pages/Customize_Ribbon.htm), although Word 2010 has made it a lot easier. If you don't want to do that, though, you can add a simple MacroButton field at the beginning of the document (http://word.mvps.org/FAQs/MacrosVBA/AssignMacroToButton.htm).

For S60/Symbian based models from 2009 (like the Nokia 2007c) Real Audio/Video support is mentioned, and RTSP is originally by Real. As Real is not mentioned for the 2007c, I suspect it might not have a Real Player built-in, and thus it might not support RTSP.

 

Easy enough to try, if you elect to use your network operator's (carrier's) Internet access point, and then try to view or listen to an RTSP stream (where the audio/video encoding has been made so that also those are supported by the 2700c). If it works, then you know.

 

I don't know if it is a common problem, or if it can be classified as a problem at all (just a feature that's either there or isn't there).

 

If it isn't there, there may be available a user-installable audio/video player that supports RTSP, but as the 2700c is based on Nokia's Series 40 platform, it'd have to be written in Java, and implementing a streming protocol handler, as well as audio/video decoders in Java will probably perform unacceptably badly, that isn't likely to exist.

 

The other solution, if you really need RTSP support, and the 2700c doesn't have it, is to switch to some other phone model.
