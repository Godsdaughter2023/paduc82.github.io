andrea jones - all about resources say: You could just use Find & Replace.  Press Ctrl+H to display the Replace window and click in the Find What box.  Now click the More button at the bottom of the window and then click the Format button at the very bottom of the window and select Paragraph.  Select Right from the Alignment drop-down box and click OK, the Find What box should now have Right written underneath it (leave the box itself blank).  Now click in the Replace With box and click the Format button again and select Paragraph.  This time set the Alignment drop-down box and select Justified.  Click OK and the Replace With box should now have Justified written underneath it.  Now click the Replace All button and your paragraphs will be changed.

Hey Glen,

You need to be aware that the post by GreginMich that you referenced is from late 2015, which though some of it still applies, the operation of Windows Defender has changed significantly since that time.  For example, the Periodic Scanning feature and the related underlying design which keeps portions of Defender operating regardless of these settings all occurred sometime later, possibly around the time of the first Creator's update though that' just a guess.

I won't try to explain the possible ramifications of all of these changes, since I was never interested in trying to modify any of this and so hadn't kept up as well as GreginMich to the potential side effects.  I'll just say that in the years since, the design of Defender has become far more aggressive in both monitoring for attacks as well as protecting itself from being disabled, so it's not as easy now as it was even at the time that particular post was written.

GreginMich had complained when many of those older threads were locked and he was unable to maintain or update them, but since he continued to use future threads rather than the Wiki many of us suggested, the corrections and updates are just as difficult to find.

Overall, I'd recommend that if this particular person wishes to attempt to disable Defender, they use the only documented method by Malwarebytes that uses the support provided by Microsoft in Windows Defender for precisely this purpose.

https://www.malwarebytes.com/support/guides/mbam/Settings.html

Scroll down to the bottom of that page and see the section titled Windows Action Center settings that define how MBAM registers with Action Center.  Though the descriptions there don't make this clear, the registration of MBAM defines how Defender operates as well, placing Defender into its "sleep" mode if MBAM is set to "always register".

Though I haven't seen this tested lately, at the time MBAM 3.0 released I had tested it in the free trial mode, with Defender having tuned off its real-time protection in favor of MBAM.

However, as noted in the Malwarebytes Labs blog article I referenced above, MBAM isn't truly a replacement for antivirus, since it only contains protections for the most prevalent viruses included in the current WildList as I recall.  Instead, most people still recommend leaving both enabled via the default setting for MBAM, which allows both to operate in tandem as Malwarebytes designed.  Of course, this still might lead to occasional conflicts when a new version of Defender is released, which would be up to MBAM to remedy since Microsoft doesn't officially support this.

OneNote Screen Clip from Browser not Working | Windows 10 | - Microsoft Community

exhalo say:

OneNote Screen Clip from Browser not Working | Windows 10 | Office 2016

I had two issues with Screen clippings not working for OneNote.

I could Screen Clip but it would just go into the general copy/paste part of the OS, and not be send to OneNote.

From some research on the web it looks like Win 10 Creators was intercepting  the "Command Shift S" first and preventing OneNote from ever seeing it.

I found the fix from the following:

https://www.top-password.com/blog/fix-onenote-screen-clipping-shortcut-not-working-in-windows-10/

3 steps:

1. Open a command windows

2. reg.exe add HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced /v DisabledHotkeys /t REG_SZ /d S /f

3. reboot

After rebooting this functionality is enabled.

I then encountered an issue with the "Include link to source when pasting from the Web" not working. After some testing it looks like it works for one at least one browser, IE (v.11.309) but not Edge or Firefox.

jasperuk say: To revert back to the old style messageing you'll need to use a Reg Editing tool.


HKEY_CURRENT_USER\Software\Microsoft\Inbox\Settings

Under this key you'll need to add a new DWORD value called "SMSInboxThreadingDisabled" and set it to 1.
Soft reset the device and the old style message view will be used.
Set this registry value back to 0 or delete it to restore the default threaded view.

As always ensure you backup your data before using any Reg Editing tools.


I also had the problem of the word completion not working, in the end I disabled the T9 to fix the problem.
