lars-Åke aspelin say:

I am doing an election analysis. There are three parties with their vote shares (SAD, INC, AAP) given in excel above on each seat. I want to create a formula for 'which party won the elections'. The party with highest vote share wins.

Try this formula in cell D2:

=INDEX(B$1:D$1,MATCH(MAX(B2:D2),B2:D2,0))

If you want to be informed when there is more than one party with the highest vote, try this formula in cell D2:

=IF(COUNTIF(B2:D2,MAX(B2:D2))=1,INDEX(B$1:D$1,MATCH(MAX(B2:D2),B2:D2,0)),"No unique winner")

how to add a number within a circle as a numbered list or separately - Microsoft Community

alwaysquestions say:

how to add a number within a circle as a numbered list or separately

I would like to know how to add a number in a circle to a word document.  Not sure if it was used as 'bullet' or 'number' list or separately.

 

Example:

 

The number 1 within a circle - comments

 

The number 2 within  a circle - comments

a. user say:

Yes, but it turns out to be one of the things that is only enabled when an "East Asian" language such as Japanese/Chinese is listed in the Office 2010 Language Preferences. (There is another option to "circle a number" when such an editing language is enabled, but that actually uses an EQ field to do the work).

I suppose it is possible that your boss had such an option enabled. Once the numbering style is in the numbered list "gallery," it will appear even when the option is disabled.

Here, I was able to set up the same thing by
 a. starting with a new document
 b. inserting a numbered list paragraph
 c. clicking in that paragraph
 d. selecting a simple numbering style such as "1."
 e. running the following VBA:

Sub setCircledNumbering()

With ListGalleries(wdNumberGallery).ListTemplates(1).ListLevels(1)

  .NumberStyle = wdListNumberStyleNumberInCircle

End With

Selection.Range.ListFormat.ApplyListTemplateWithLevel _

  ListTemplate:=ListGalleries(wdNumberGallery).ListTemplates(1), _

  DefaultListBehavior:=wdWord10ListBehavior

End Sub

Otherwise, I don't know how your boss did it - possibly 

 a. using the Arial Unicode MS characters from 2460 (hex) (9312 decimal), or the Wingdings that get you circled numbers from 0 to 9 - but that would not give you an /automatically/ numbered list

 b. a nested SYMBOL field, e.g.

{ SYMBOL \u { SEQ circle \r 9312 } } 

for the first number, then

{ SYMBOL \u { SEQ circle } }

for subsequent ones, or

 c. (maybe) a nested EQ field such as

{ eq \o \ac(O,{ SEQ x })

where the O needs to be a circle character (Word uses Unicode (decimal) 9675 for this), and the { SEQ x } is formatted as a smaller, raised font.

I think it is unlikely that your boss used (b) or (c) - I do not think (b) works on Word 2003 or 2007, for example.

hansv mvp say:

Using military time hhmm is going to require a lot of extra computations because it is not a "native" Windows format. If at all possible, I'd use a standard Windows format, e.g. dd-mmm-yyyy hh:mm (example: 23-Nov-2010 21:09) or dd-mmm-yy hh:mm (example: 23-Nov-10 21:09).

Say that you have date/time fields dtmStart and dtmEnd (with the above format), you can then calculate the elapsed time simply by subtracting dtmStart from dtmEnd. In a query:

Elapsed: [dtmEnd]-[dtmStart]

or in the control source of a text box on a form or report:

=[dtmEnd]-[dtmStart]

Format this field as hh:mm

Note: this will display times up to 23:59. If you need to display longer durations, you can use

Elapsed: 24*[dtmEnd]-[dtmStart])

and format as a number to display decimal hours.

marshall barton say:

Put two text boxes (named txtStart and txtEnd) on your form where you can enter the start and end date time.  Set their Format property to mm/dd/yyyy hh:nn 

Add a third text box to display the result by setting its ControlSource to an expression.  If the elapsed time will never be more than 24 hours, you can use the expression:
   =txtEnd - txtStart
and set this text box's Format property to h:nn

But if the elapsed time can be more than 24 hours and you only want to see it in hous and minutes, then you need an expression like:
   =DateDiff("n", txtStart, txtEnd) \ 60 & Format(DateDiff("n", txtStart, txtEnd) Mod 60, "\:00")
or the same thiing another way:
   =Int((txtEnd - txtStart) * 24) & Format(txtEnd - txtStart, ":nn")
Choose the one that is easiest for you to understand why it works.

ken sheridan say: The following function will return the time duration formatted either as hh:nn:ss or d:hh:nn:ss if the optional second argument is passed as True. 

Public Function TimeElapsed(dtmTime As Date, _
            Optional blnShowdays As Boolean = False) As String

    ' Returns a date/time value as a duration
    ' in format hh:nn:ss, or d:hh:nn:ss if optional
    ' blnShowDays argument is True.
   
    Dim lngDays As Long
    Dim strDays As String
    Dim strHours As String
   
    ' get whole days
    lngDays = Int(dtmTime)
    strDays = CStr(lngDays)
    ' get hours
    strHours = Format(dtmTime, "hh")
   
    If blnShowdays Then
        TimeElapsed = lngDays & ":" & strHours & Format(dtmTime, ":nn:ss")
    Else
        TimeElapsed = Format((Val(strDays) * 24) + Val(strHours), "00") & _
            Format(dtmTime, ":nn:ss")
    End If

End Function

Paste the function into a standard module in the database and add an unbound text box to the form, with a ControlSourse property of:

=TimeElapsed([dtmEnd]-[dtmStart])

or if you want to show days:

=TimeElapsed([dtmEnd]-[dtmStart], True)

Note that the result is a string, so you cannot do mathematical operations on it.  If you need to do that, e.g. to sum a set of durations, then sum the difference between the two date/time values first and then apply the function to the result, e.g.

=TimeElapsed(Sum([dtmEnd]-[dtmStart]))

or to show days:

=TimeElapsed(Sum([dtmEnd]-[dtmStart]),True)
