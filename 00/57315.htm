a. user say:

What is the problem you are having that you think SR will resolve (or are you just practicing)?

Go back to where and for what?

The default for the VSS Service is Manual - as it should be.

Every Service/Process running on your system consumes CPU cycles and memory (maybe not much) so it makes sense that the fewer unnecessary thing things you have running the better your system will perform so your system will use those CPU cycles and memory on things that are more important to you.  There is no need for VSS to be starting and running all day and night with nothing to do.

When looking at your RPs, be sure this box is checked:

There are lots of reasons Windows might not create an automatic restore point.

The default is an automatic RP every 7 days but only when the system has been idle for 10 minutes.  Idle means idle. 

Idle means no mouse, no keyboard, no surfing, no scanning, no instant messenger, no Skype, no movies, no Magicjack, no XP Indexing Service, etc.  Anything that uses 10% of the CPU even for a split second can prevent RPs from being created "naturally".  It could be some third party background process or service you have installed that prevents idleness.

If Windows needs to create a RP it will wait for idle time, but if you then shutdown your system for the day, you will not see a RP until the next time the system is idle.

In my case, I don't think my system is ever idle for 10 minutes so I might not see an automatic RP created every 7 days but I know I should see one at least every 7 days or so and if there is not, there is a good reason for it.

There are also no RPs created if your system is running on battery power (this is also the default setting) so if your system is running on battery power most of the time, when the time comes to create and automatic RP Windows will not create one.

If something else creates a RP (like a Windows Update), that counts as a RP so Windows will not create another one for 7 more days.

You can adjust all these settings, but if you think your SR is afflicted, see if you can manually create a RP, the reboot and immediately see if you can restore your system to the RP you just created.

If your system can't restore, there are lots of reasons for that too usually some third party antivirus/antimalware application that is "protecting" your system from changes.  Those can be temporarily disabled or perform the restore in Safe Mode when those things are not running.

System Restore is also not a time machine.  It only cares about some Windows specific files.

System Restore does not know or care about any of your personal files like emails, documents, music, images, pictures, etc.

If you can't' make a manual RP, reboot and restore your system to the RP you just made, something is wrong (or something is getting in the way) and you should figure out why and fix it.
