The Problem

 

Windows 7 Taskbar icon preview not working with MS Excel 2007

 

Symptom

 

When you have one or more excel spreadsheets (aka workbooks) opened and hover over Excel icon on taskbarwith your mouse,popup only displaysgray box with the Excel logo in the center and name of files on top, and does not show preview of opened spreadsheets.

 

 

Reason for the problem

 

Excel is what you call an MDI interface.  Basically, each opened spreasheet or workbook window is the equivalent of what a tab would be in your internet browser, where the browser window is parent to all the opened tabs, in this case, excel workbooks.  To get it's child tabs (workbooks) to appear as separate application instances on the taskbar, Excel does some tricky stuff that isn't really optimal.  It's basically tricking the taskbar into showing them.  Because of this, no preview of child documents, unless you launch an entire new instance of Excel (so that the two documents are cousins (or step children?), rather than siblings of the same 'parent' window).  In other words, what you want is for excel to open each spreadsheet as its own window, and not as a tab like on your internet browser.

julie sheets say:
Hi Chris,

I've tried a couple of experiments and I have to say, I too am getting inconsistent results.

I've done the following:

Project start date: March 26th 2012

I've created an Excel 2010 file with a simple list of 7 tasks listing just the values you note: Name, Start, Finish, Resource Names, Work.

As expected, on appending tasks into an existing Project file all tasks are constrained with a Finish No Earlier than constraint. 

However, like you when importing tasks without start or finish dates, the tasks are scheduled as if there was a relationship with the task above.  In my case I was seeing the finish date align with the finish of the task above.

I've experimented with saving the Excel file as an earlier version (*.xls), importing into custom fields (Start1, Finish1), as well as importing into a project will all tasks set as manually scheduled.

The constraints is what is setting dates -- but why they seem to "inherit" dates from tasks above, I'm not sure.

I'm guessing your LOB program does not have predecessor/successor information contained in the data. That would certainly affect dates as well.  Of course removing constraints sets the tasks back to the project start date.

So the work around I did:
In the Excel file I used the IsBlank function to test whether a task had a start date.  If it had no start date, the IsBlank function returns yes.

I appended to the new project (in order to be able to set the Project Start date in the past) mapping the column to the Flag1 field.  After importing, I filtered for Flag1=yes and removed the constraints from those tasks.

I hope this helps.
