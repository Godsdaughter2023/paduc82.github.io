ken sheridan say: If you are only recording each patient's progress in relation to one survey, i.e. you don't also have a SurveyID or similar column so that multiple surveys per patient can be tracked, then it's just a question of returning the MAX(Round) value and adding 1:

SELECT PatientID, MAX(Round)+1 AS NextRoundUp
FROM [Tracking Table - Patient]
GROUP BY PatientID;

BTW be sure to make the two columns the composite primary key of the table to ensure no inadvertent duplication of Round values per patient.

ken sheridan say:

What you need to do is create a Rounds table as follows:

 

Rounds

....RoundNumber (Number – primary key)

....Round (Text – indexed uniquely)

 

Fill it like so:

 

RoundNumber               Round

 

0                                  BL

1                                  1

2                                  2

3                                  3

4                                  4

 

...and so on as high as will conceivably be necessary, then add a few more as a buffer.

 

In your Tracking Table - Patient table add a RoundNumber column and in each row insert the corresponding RoundNumber value with the following update query:

 

UPDATE [Tracking Table - Patient] INNER JOIN Rounds

ON [Tracking Table - Patient].Round = Round.Round

SET [Tracking Table - Patient].RoundNumber = Rounds.RoundNumber;

 

Once you are happy that it's been updated correctly you can delete the original Round column from [Tracking Table - Patient].

 

For data entry in a form – and remember data should only ever be entered via a form, never in a table in datasheet view – use a combo box set up as follows;

 

ControlSource:  RoundNumber

 

RowSource:     SELECT RoundNumber, Round  FROM Rounds ORDER BY RoundNumber;

 

BoundColumn:   1

ColumnCount:    2

ColumnWidths:  0cm

 

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the dimension is zero to hide the first column.

 

In any reports using this data base these on a query which joins the two tables on RoundNumber so that you can return the text Round values in the report.

 

In the relationship between the tables be sure to enforce referential integrity.

 

You'll now be able to compute the next round per patient with:

 

SELECT PatientID, MAX(RoundNumber)+1 AS NextRoundUp
FROM [Tracking Table - Patient]
GROUP BY PatientID;

 

The fact that the BL round is represented by zero in the RoundNumber column means of course that for patients who have only progressed as far the BL round a 1 will be returned.  If it's possible that there could be one or more patients in the Patients table, but not yet represented at all in the Tracking Table - Patient table, for whom you'd want to return BL as the next round up, you'd need something like this:

 

SELECT Patients.PatientID, Firstname, LastName,

NZ(MAX(RoundNumber)+1,"BL") AS NextRoundUp

FROM Patients LEFT JOIN [Tracking Table - Patient]

ON Patients.PatientID = [Tracking Table - Patient].PatientID

GROUP BY Patients.PatientID;

 

One sting in the tail:  I'd recommend that you avoid Round as a column name.  It's the name of a built in function in Access, so as a 'reserved' word is best not used as an object name.  Something more specifically descriptive like SurveyRound would be better.
