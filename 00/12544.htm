hansv mvp say:

You can format the merge field in Word:

  • Click anywhere in the merge field.
  • Press Shift+F9 to view the field code. You'll see something like

    { MERGEFIELD FieldName }

    or

    { MERGEFIELD FieldName \* MERGEFORMAT }

    where FieldName is the name of the merge field.
  • Change this to

    { MERGEFIELD FieldName \# "$,#.00" }

  • Press F9 to hide the field code and to update the result.

scottgem say:

No it doesn't. The fact of the matter is that the ENVNO is assigned by a manual process when they are sorted. That's fine. But, since you are already designating the Payment Type in the ENVNO, why do you think you need to do it again? Why waste a field that is redundant?

If you want to DISPLAY a different Payment Type code you can do that with an expression anywhere you want to display it. 

For example:

=SWITCH(Left(ENVNO,1) = "M","C",Left(ENVNO,1) = "Q","Q",Left(ENVNO,1) = "V","V")

Or

=IIF(Left(ENVNO,1)="M","C",LEFT(ENVNO,1))

Either of those two expressions will return the Payment Type code you want. But THERE IS NO REASON TO STORE IT since it can be calculated on the fly where needed.
