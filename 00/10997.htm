Sub Move_Specific_Comments()

'### 26 Jan. 2019 oAnast ##

Dim r As Range

Dim v As Variant, vv As Variant

v = Array("A5", "A7", "A9")

Application.ScreenUpdating = False

For Each vv In v 

Set r = Range(vv).Offset(, 3)

If Not Range(vv).Comment Is Nothing Then

With r.Comment

.Visible = False

End With

End If

Next

Application.ScreenUpdating = True

End Sub

ADD BY COLORS IN EXCEL

Function SumarColor (color As range, range As range)
    'color: The cell that contains the color to add
    'range: The range of cells to be considered in the sum
        
    Dim result 'Will store the result of the sum
    Dim cell As range
    
    'Scroll through each cell in the range
    For Each cell In range
        'Add if the color of the cell is equal to the specified color
        If cell.Interior.ColorIndex = color.Interior.ColorIndex Then
            result = result + cell.Value
        End If
    Next cell
    
    SumarColor = result
End Function

rod gill say:

Yes. Linked master files are prone to file corruption if you ever rename, move or over-write any of the files. Storing them on a disk physically not in your building is not a good idea either.

So try creating unlinked master files (uncheck the Link option each insert).

This now creates a master with all task data, consolidates all resource info and creates a great snapshot for reporting and audit purposes.

I use a macro to recreate a new master each week. Record a macro of you doing one manually to get the basic code. make sure File, New is the first recorded action.

John,

I have a couple additions to Rod's excellent advice;

  1. As long as the original files share a common folder path, I've had success (been lucky) saving the whole group/folder into a single zip/rar archive, then sharing the archive.  The links typically survive the file extraction at the other end as long as all the files are extracted together.

tom_bpc say:

John,

I'm pretty sure that CrossLinks Pro does a lot more than consolidate linked project files, though that's a very handy functionality.  The problem is that the links are still needed.

As Rod highlighted above, dynamic logic links between project files in a file system are simply not stable enough for routine use without some very strict procedural (and physical) controls.  Unfortunately, such links are still necessary when the "projects" are separately-managed parts of an integrated mega-project or program - i.e. when the strict file controls are not possible.  I have the impression that CrossLinks Pro provides a robust alternative to dynamic linking between files.  (It reminds me of the Interface Milestone Lists that I've used to mediate among multiple contractors on very large projects, but with some additional automation.)

ProjectPro makes a few other Add-ins that have impressed me as useful for some very specific analyses.  (I'm not a user, though - I developed my own Add-ins for the same purposes.)  The price of this one - an order of magnitude higher than the rest - implies that this is custom-built for a very small (but sophisticated) market segment with large costs at risk.  For some in that segment, I imagine its considered a very good buy.
