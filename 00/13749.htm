a. user say:

1. Date and Close are both "reserved words" in Access/Jet so it is best to avoid them as names of tables, columns, etc., If you have to use them, you have to quote them, e.g. using []:

sSQL = "SELECT Symbol, [Date], [Close] FROM stock WHERE Symbol = 'DJIA'; "

See e.g. 

http://office.microsoft.com/en-us/access-help/sql-reserved-words-HP001032249.aspx

for other reserved words (I do not know if the list is complete).

Date literals can be enclosed in # characters, so e.g. the following should work:

sSQL = "SELECT * FROM stock WHERE [Date] = #9/9/2010# "

However, AFAICR remember when using that syntax the dates are interpreted as US Date format M/D/Y unless they are unambiguously in (say) D/M/Y  sequence, whatever the date format in your database. You can use e.g. #YYYY-MM-DD# if you prefer to avoid that, e.g. #2010-09-30#

hanyali1 say:

Three identifiers for each customer

1st if he selected user if it populates atomaically in user name and he presses filter button and the related info should appear in the report

given that the two options should reset the other if selected 

any idea how i could acheive that i already have all my boxes prepared but cant get them to depend on each other

ken sheridan say:

The first two combo boxes should only differ in that in the second the UserID column is hidden be setting the ColumnWidths property to zero and the ColumnCount property to 2.  In each case the BoundColumn property should be 1.  The two can be synchronized by setting the value of the other combo box to the value of the active combo box in the control's AfterUpdate event procedure.

However, selecting a user solely by name is not very reliable as personal names can legitimately be duplicated.  I worked with two Maggie Taylors for instance.  For this reason it is better to include other data in one or more other columns in the list to differentiate between users of the same name.

The third combo box can be synchronized with the first two by referencing the first combo box as a parameter in its RowSource property, e.g.

SELECT DISTINCT UserCategories.UserCategoryID, UserCategory
FROM UserCategories INNER JOIN Users
ON UserCategories.UserCategoryID = Users.UserCategoryID
WHERE (UserID = Form!cboUserID OR Form!cboUserID IS NULL)
ORDER BY UserCategory;

Note the use of the Form property to return a reference to the current form, rather than referencing the form by name as a member of the Forms collection.  The third combo box should be requeried in the AfterUpdate event procedures of both of the first two combo boxes.

The first two combo boxes can similarly be synchronized with the third by referencing it as a parameter in their RowSource properties, e.g. in the case of the second:

SELECT UserID, UserName
FROM Users
WHERE (UserCategoryID = Form!cboUserCategories OR Form!cboUserCategories IS NULL)

ORDER BY UserName;

The first two combo boxes should be requeried in the AfterUpdate event procedure of the third.

The multi-select list box can be synchronized with the third combo box in the same way:

SELECT UserID, UserName
FROM Users
WHERE (UserCategoryID = Form!cboUserCategories OR Form!cboUserCategories IS NULL)
ORDER BY UserName;

The list box should be requeried in the AfterUpdate event procedure of the third combo box.

To open a report filtered to the user selected in either of the first two combo boxes is simply a matter of doing so in the WhereCondition argument of the OpenReport method, e.g.

Dim strCriteria As String

strCriteria = "UserID = " & Me.cboUserID

DoCmd.OpenReport "rptUsers", _
    View:=acViewPreview, _
    WhereCondition:=strCriteria

To open the report filtered to those users selected in the multi-select list box requires code which loops through the controls ItemsSelected collection, e.g.

    Dim varItem As Variant
    Dim strUserIDList As String
    Dim strCriteria As String
    Dim ctrl As Control

    Set ctrl = Me.lstUsers

    If ctrl.ItemsSelected.Count > 0 Then
        For Each varItem In ctrl.ItemsSelected
            strUserIDList = strUserIDList & "," & ctrl.ItemData(varItem)
        Next varItem

        ' remove leading comma
        strUserIDList = Mid(strUserIDList, 2)

        strCriteria = "UserID In(" & strUserIDList & ")"

        DoCmd.OpenReport "rptUsers", _
            View:=acViewPreview, _
            WhereCondition:=strCriteria
    Else
        MsgBox "No Users selected", vbInformation, "Warning"
    End If

You'll find examples of the use of controls in dialogue forms to filter or restrict reports in various ways in MultiSelect.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.
