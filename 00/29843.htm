What you perceive as random layout changes are not random: they result from pressing some key combinations (Ctrl+Shift or left Alt+Shift) as it can happen when we are typing text. I consider that the management of keyboard layouts, input languages and input methods in Windows is close to a complete failure for many people like you and me (just search this website for "keyboard layout" and see the number of threads), or at least half-baked. Please, read further, long post. 

I have the same problem too, but my (physical) layout is fr_FR on a single physical keyboard, be it on the laptop computer or on my desktop system. I have used MS Keyboard Layout Creator to create a custom layout that includes the otherwise-missing accented characters for French and also a few characters for Portuguese. One input language (French), one keyboard layout; you would think everything would work fine, and you'd be right. Until an application (see further below) changes the list of languages and adds a keyboard layout. That's when problems start (I also mean that MS's way would have been correct if programs had not been allowed to mess with languages and layouts) and there are two possible outcomes but each is undesirable. 

First outcome: if you have deactivated the key combinations aforementioned, then, you are screwed because you can't change back to your "normal" language, i.e. by using the keyboard. You can only go the Control Panel way. Good luck if you're on the login screen!

Second outcome: if you have not deactivated the key combinations, you can then try using them. I write "try" because somehow, Windows has lost track of my custom keyboard layout because once I've passed the login screen, I can see that the Control Panel says something like "input method unavailable". And the key combinations are ineffective since no "input method" is available, which explains why the layout at the login screen is en_US.  So yes, en_US is built-in the system and used as a layout of last resort. 

I'm typing this message after ending up uninstalling my custom layout, just so that I could have some peace of mind for a few hours.

Some applications DO change your current keyboard layout. Typically, if you have Steam, a game like Left 4 Dead (the Web says other games built using Valve's Source engine behave the same) WILL set your layout to en_US just because you've chosen "English" for the voice and text language inside the game. I've read that some Adobe applications do the same. This baffles me but I read last week that a MS engineer asserted that this is how applications should work. When an application alters the keyboard layouts, to add insult to injury, the language icon/bar in the task bar shows fr_FR and en_US but the list of languages in Control Panel/Language does not show en_US. Yay! I can still add en_US to the list and remove it immediately. BUT, the next time I'm on the login screen, somehow, the language activated is... English US! Yes, because Windows has lost track of my custom layout :-)
