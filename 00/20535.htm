Deleted occurrences of a recurring event reappear if I edit the time

Hello internet.  This has annoyed me for ages (several versions of Outlook and Outlook for Mac on several computers) so I'm finally going to ask for help.  Whenever I make changes to the time of a recurring event (start time, end time, or time zone) it will regenerate all of the previously deleted occurrences of that event.  As a work around, I keep a list of deleted occurrences in the notes section so I can re-delete the right occurrences after they regenerate but this is really annoying.  The deleted occurrences are not regenerated if I make changes to the event name, notes, category, reminder, etc.  Is there any way to make Outlook remember which occurrences should be deleted?  Thanks :)
brian tillman [outlook mvp 2007-2019] say:
This is normal. Events, even recurring events, are single items in the calendar.  The individual recurrences are calculated based on the start date and the recurrence pattern.  When you edit an individual recurrence, such as by deleting it or moving it to a different time, Outlook adds an exception to the event record and modifies what you see on the calendar.  Certain edit events, such as changing the start or end dates for the entire series or the edit you made, causes Outlook to recalculate the individual recurrences, wiping out any exceptions, since the recurrences are time-based.

bob jones aka: cybertaz say:

I'm not sure exactly how you're going about this or what specific attributes you want as a part of the Table Style but the process is remarkably simple in this version of Word :-) First, though, I think it might be good to clarify a few points. The way the terminology is used by Microsoft can be confusing.

The default Paragraph Style used in Word documents is a style by the name of Normal. It determines the font formatting, paragraph formatting, Tab settings, Language... even Borders & Shading, etc. used in new blank documents [those based on Word's Normal.dotm template].

The standard specifications for Normal, of course, are very basic, and although Normal can be modified it's generally recommended that it not be changed drastically... It is the underlying Style on which numerous other built-in styles (including Table Styles) are based. Modifying Normal can lead to some very interesting results further down the road :-)

Pursuant to that, when you use the Default button in the Format Font dialog you are actually updating the Default Paragraph Font [Character Style] which is used by the Normal Paragraph Style, so Default & Normal really are one in the same. You're correct that there is no Default button in the Format Paragraph dialog... changes to default paragraph formatting are done by modifying the Normal style -- or preferably by creating your own styles based on Normal, some other style (such as Body Text), or no style.

For a user-defined Table Style, though, there is no reason to alter any of the built-in Paragraph or Character styles. Generally, they are not used in a table unless there is certain content that needs to deviate from what the Table Style specifications call for.

As for creating the Table Style, the procedure I suggest is:

First, make sure you've saved your template as a template file [.dotx/.dotm]

  1. While working in the template file go to Format> Style - New
  2. Name the style
  3. Select Table as the Style type
  4. Select Table Normal (or other of your choice) as the Style based on
  5. Use the tools available on that same page if you wish and/or
  6. Click the Format button to select the Font, Paragraph, & other attributes you want
  7. Check the Add to template box
  8. Include Add to Quick Style list if you wish
  9. Click OK then click Close
  10. Save the template

The user-defined Table Style will be added at the front of the Table Styles Gallery in the Ribbon. If you want it to be the default for all new tables created in documents based on that template:

  1. Create a table so the Tables tab activates in the Ribbon
  2. Right-click the style's icon & select Set as Default
  3. Delete the table you created [unless you have some reason to include it as a part of the template]
  4. Save again

bob jones aka: cybertaz say:

When you first create a table it will be formatted based on the default for Normal Paragraph Style until you apply your Table Style. When you apply your style to a table the table will be reformatted based on what that style calls for. Applying your Table Style essentially tells the table to deviate from Normal in the manner you've specified in the Table style. IOW, Normal Paragraph Style is the built-in default used by the Table Normal Table Style. It isn't until you've done the 2nd set of steps to set your Table Style as the default that new tables will be created from the start using your style.

Please don't be offended if I seem blunt but I just want to be clear -- Defining a Table Style does not & should not involve direct modification to Normal Paragraph Style in any way. The only reason to modify Normal Paragraph Style is if you want to set global defaults that apply to all text in all new documents that are based on the Normal.dotm template. As I tried to explain earlier, changes made to Normal Paragraph Style have far-reaching effects because so many other styles are based on it. In a nutshell, every time you modify Normal you're redefining the formatting of everything to which Normal is applied.

It's true that Style management is handled somewhat more extensively in Word for Windows, but that's not really a factor here. The main difference about the Format> Paragraph dialog in Mac Word is that there is no Default button. However, the button in Win Word's dialog simply updates the Normal Paragraph Style as though you had gone into the Modify Style dialog to make the changes. Similarly, Set Defaults in Win Word's Manage Styles dialog is nothing more than a limited express route for making a few commonly modified Character & Paragraph formatting modifications to Normal.
