in the sticks say:

To install the way you want would require booting the PC from the installation CD rather than starting Windows and then autoplaying the CD. Then you will be able to select drive and/or partition to install to. To boot from the CD there might be an F key depression at boot time to select the desired boot device. If not you could check BIOS settings to make sure that the optical drive will boot before the hard drive or just disconnect the existing hard drive (see next paragraph). The latter will only work if the new hard drive does not currently have an established OS. If the new drive does have an established OS that you want to overwrite, you will need to make sure that you boot from the CD via F key or BIOS settings.

However, I would caution against installing XP to the new drive whilst the current XP hard drive is attached, i.e. powered and on a ribbon cable attached to the drive controller. You are liable to end up installing the new XP to the D drive rather than what I suspect you want, which would be another C drive. Worse still you might accidentally overwrite the original XP rather than create a new XP.

You talk about master/slave, so I am assuming that your drives are IDE (PATA) not SATA. If the drives are IDE then you need to make sure via jumpers that one is designated master and the other is designated slave if you want to use both together after the new OS is installed. If you heed my advice in the second paragraph, both drives would individually boot as the C drive and see the other OS as a later letter, probably E because the optical drive would be D to both XP installations. If the IDE assumption is incorrect, there might be other complications for SATA drives, so let us know if that is the case.

Because your hard drives are IDE I suspect that your optical drive is also IDE. Hopefully you are using a different ribbon cable for it? Assuming that it is the only drive on that ribbon cable, make sure its jumper is set to master and that if it is a dual connection ribbon cable that it is on the end connector (probably black). If it is a 80 strand cable make sure the blue connector is inserted to the mainboard.

Very often, to boot from a CD you might see a message at boot time saying something like "Hit any key to boot from CD...". Does such a messgae appear? If it does then you will need to do so. It might be that your display resolution at boot time is not capable of showing this message, so try hitting (for example) space bar once a second immediately after turning on the PC until it either boots from the CD or gives the same error message you are currently getting.

Is there a message about hitting an F key to select boot device? If there is then use that F key to obtain a boot device menu.

You could maybe try moving the optical drive to the same cable as the HDD as a temporary measure. Configure the HDD as master and the optical drive as slave with their jumpers. Put the HDD on the end connection (probably black) and the CD on the 'middle' connection (probably grey). This should only be a temporary measure to get installation completed. Ideally optical drives should be on a different cable to hard drives to avoid slowing down data transfer from the HDD.

You could check out the mainboard by booting to your old disk on the same ribbon cable on the same mainboard connector. Configure the old disk as master and put it on the end connector of the ribbon cable.

I know you say that you tried my suggestions but your description of what you have tried does not specifically state that you have tried the optical drive on the same cable as the HDD. Specifically, you need to try this on the mainboard connector that does appear to at least attempt to boot a HDD OK. Normally this mainboard connector would be designated "Primary IDE". The secondary IDE is normally used for optical drives. My theory is that the "Secondary IDE" connector (or the Primary if the normal roles of Primary and Secondary are reversed) might be bad for some reason. So, if you have not done this yet, please try putting the new IDE and an optical disk drive on the same ribbon cable which is connected to the mainboard connector normally used for the HDD. Set HDD = master and optical drive = slave.

Other things to check if you haven't already done so...

  • Bent pins on the mainboard IDE connectors.
  • Pin 1 is correctly located for the ribbon cable at both ends. The ribbon cable will have a colour (normally red) running down the side that is pin 1. Pin 1 should be marked on the mainboard probably by "1" but I have also seen it maked by an arrow head. Pin 1 on the optical drive is on the right, nearest the power connector when viewing the rear of the drive. This only applies if your ribbon cable(s) do not force the correct orientation (I have seen some that do not do this).
  • Both Primary IDE and Secondary IDE are enabled in BIOS.

BIOS update would also be a last resort. To assist with that you will need to provide the hardware information already requested.
