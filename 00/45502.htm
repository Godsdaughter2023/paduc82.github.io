brianl1 say:

Disconnect your One from your network, power cycle both your modem and router. That is, disconnect the power from both, then power on the modem.

Wait two minutes, Make sure your modem indicates 'Online', this means it has Block sync with you ISP's servers. Make sure none of your other devices on your network are on, then power on the router.

Wait two minutes while it syncs with your modem, then connect the Ethernet to your XB1 and power it on.

This should actually fix the issue, unless you're actually having ISP issues... a 300ms ping is usually a sign of server side issues. running a 'tracert' command in your computer's command prompt can reveal issues with servers your provider is using.

For instance, if you're with Comcast, you would type tracert comcast.com and hit enter. The server with the highest ping, in the resulting list, would be the one causing you issues, especially if you see one of them just timeout.

john_barnett say:

Hello JosephWilson7.1,

 

You can write protect a floppy disk by ensuring that the hole at the top left of the disk is open. If this hole is closed, in other words the piece of plastic has been slid over the hole, then the  floppy is not write protected. If the window is open, you can see through it, then the floppy is write protected. It is just a case of sliding the plastic tab down so that the hole is closed.

 

see this illustrated link: http://oit.ua.edu/oit/services/it-service-desk/write-protect-a-floppy-diskette/

sarahkong say:
Hi SireeshaR ... yes this is expected behavior.
It's not that it changes it. It is more that your BIOS always looks for the primary active partition.
Windows when installed will set the partition that windows is on the primary and active partition. When that happens the BIOS will look for that and find and set that to load first.
This is your BCDEDIT file that contains the windows load.
I am assuming you removed the PXE boot disk after you installed and then the BIOS loaded windows.
