yangzheng_msft6300 say:

Hi Ryan,

I have read all the posts above and my understanding is as below:
1. You want to add your two custom domain: main domain and secondary domain. And you want to use the two domains in your Office 365 tenant.
2. The main domain is used as the suffix of normal user mailboxes and the secondary domain is used as the suffix of shared mailboxes.
3. You want to access the shared mailboxes in OWA and Outlook client.

Commonly, the process to achieve your requirement is as below:
1. After purchasing an Office 365 subscription, a global admin account will be automatically created with the onmicrsoft.com domain as the suffix.
2. To use the main domain and secondary domain in your Office 365 tenant, it is required to verify these two domains referring to article below.
https://support.office.com/en-us/article/Add-your-users-and-domain-to-Office-365-6383f56d-3d09-4dcb-9b41-b5f5a5efd611.
And it doesn’t matter whether you verify the main domain first or the secondary domain first. Also if you have added the primary global admin account into Outlook 2016, it is not required to change or remove this account.
3. Add or change the DNS records in your DNS hosting provider of the main and secondary domains.
Create DNS records for Office 365 at any DNS hosting provider
After the DNS records especially the MX record are pointed to Office 365, the IMAP/POP3 users still pointing to the old mail server will not receive emails and can only send emails.
4. Create the user mailboxes with the main domain as the suffix and create the shared mailboxes with the secondary domain as the suffix.
5. Assign the Full Access permission of the shared mailboxes to the normal user mailboxes. Then when adding the normal Office 365 user mailboxes into Outlook 2016 client, the shared mailboxes will automatically appear under the user mailboxes. Then if you want to send emails using the email addresses of the shared mailboxes, it is required to assign the Send As or Send on Behalf permission to the user mailboxes. Please refer to the article below to manage the permissions of the shared mailboxes:
https://technet.microsoft.com/en-us/library/jj150570(v=exchg.150).aspx

yangzheng_msft6300 say:

Hi Ryan,

Thanks for your reply.

It is required to change the DNS records after you verifying the domains in Office 365. Only after changing the MX record to point to Office 365, the emails sent to this domain could be sent to Office 365. And the SPF record will ensure that the destination email systems trust messages sent from your domain in Office 365. So if you don’t change the DNS records in your custom domains, you would suffer issues when sending or receiving emails. So if you just want the normal user mailbox with the main domain as suffix to deliver emails well, you will need to firstly change the DNS records of the main domain. If you want the normal mailboxes and shared mailboxes will all send or receive emails well, you will need to change the DNS records of both the two domains.

Commonly, you can use Exchange mode to configure your accounts into Outlook client which can sync your emails, calendar, contacts and tasks with Office 365 Exchange Online. The IMAP mode can only sync emails with Exchange Online. If your requirement is to use IMAP for your users and disable POP3, you can do this with PowerShell.
1. Connect to Exchange Online with PowerShell.
2. Run the command below.
Get-Mailbox | Set-CASMailbox –POPEnabled $False

Then, if an account is not existed in your Office 365 tenant, emails can’t be sent to the address of this non-existed account. So it is not feasible to set a redirection to the emails sent to this address.

Yang

luke_jones say:

Make "Sort" Simple!

Here I was using Windows 10 Technical Preview on my desk this morning and suddenly this idea came to my mind about the way it uses the “Sort” function. I have this issue and I am guessing many of us as well. Unfortunately this issue is so neglected that I have not been aware about it until this morning!

Issue:

When we use “Sort by” in Windows Explorer, we are offered many options to choose from. Such as Names, Date and so on. I was looking for a file having a “PDF” format this morning and since there are lots of them I always use the “Sort” function to find them quickly. Problem is as soon as I select another sorting category it begins once again from the start, sorting everything again!

Solution:

Well, I want to suggest this idea where we are offered, a “Sequential Sort.” This is how the sorting function should work:

  1. Sort by Names (or whatever you like) to begin with.
  2. Keeping the order of previously sorted entries, the next sort is introduced.
  3. Search is finished when result is found otherwise further sorting takes place.

So what I am trying to say is if I am searching for a file named “ABC.pdf” in any Window Explorer window, I get a continuous sorting process to find it! So first, I will sort all files by Format, then keeping those results intact, I will continue sorting, this time, by Names. This process of consecutive sorting is like a loop, which is halted when file, “ABC.pdf” is found!

I hope I have helped!

Luke

Or you can refer to the following steps to give it a shot. 

（Export public Folder calendar data）
1. In Outlook 2013 click file - Open & Export
2. Click Import/Export

(Import Public Folder calendar data to shared mailbox）
1. Expand the shared mailbox such that the Calendar is viewable

2. Click File - Open and Export
3. Click Open Outlook Data File
4. Verify that the PST is mounted under "Outlook Data File"
5. Expand IPM_SUBTREE/Path/TO/Calendar until the calendar view is visible.
6. Click View - Change View - List
7. Select all items (Control+A)
8. Drag and drop items to the Shared Mailbox calendar folder.
