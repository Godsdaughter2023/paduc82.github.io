mike hall... always say:

If I MUST have a pagefile on C: why does Windows allow it to be turned off?

Because the controls for the page file are basic and global. If Windows finds the need to create a page file on C, it will do it and it has always been that way..

I have to ask, why isn't this in the help information?  I shouldn't have to go to an online forum because something doesn't work in the way Windows insists it does.

Creation of help files is a huge job, and if help files included every conceivable configuration, you would find it hard to navigate them to find out what you wanted and would be squeaking about that too.. :-)

Surely with the hundreds of mb of updates I have done they could have included an update of the tooltips and help info for this feature.

Yes, they could, but there are no hard and fast rules re page files other than the fact that ALL operating systems, Windows and others, require one.

Unless you disable virtual memory completely Windows makes no mention of this requirement and when it does it indicates a minimum 800mb file.

Virtual memory should not be disabled completely. Each source has it's own algorithm for working out the optimum size for a page file. The thinking was that at least 500mb was needed for dump files. That may have changed, and in any case, it is always best to allow for a little more than the absolute minimum..

I would also add that in the two previous generations of Windows this was not a requirement and I have to wonder at the stupidity of introducing it.

It has ALWAYS been a requirement, but Microsoft may have made changes to allow Windows 7 greater freedom. Thinking does change over time and what was good x years ago may not be relevant today..

Sad to say this is typical Microsoft and only the squeaky wheel gets the oil, so now I'm squeaking.

It is no more typical of Microsoft than it is of others. You just haven't come across similar issues on other platforms..  :-)

So why do you want to move your page file away from the SSD? With the amount of RAM you have, page file read/writes will be minimum and an SSD is the absolute best place for a page file anyway. Wear on SSDs may be overstated, especially as SSDs are getting better, and in all likelihood, you will need a new computer before your SSD wears out
