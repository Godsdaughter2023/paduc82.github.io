Basic excell question on formula - Microsoft Community

patrickharrington3333 say:

Basic excell question on formula

I am trying to create a schedule in excel.  I need a cell to calculate the difference between 2 numbers in the cell... example.. Employee scheduled in one cell 11-2  or 11:00AM-2:00PM I need to calculate for 3 hours in that cell so I can total the number projected at the end of the sheet for each employee's weekly scheduled total. Example

 

Employee   Scheduled   11-2. 11-2. 5-9.30,    Total for week 10.5

jllatham say:

You have a basic design problem.  Each of those entries needs to have the start time in one column and the ending time in another column.  Then it is very easy to do, or at least much easier.  Excel is not designed to handle 2 values in a single cell.  You'd need a complicated formula to get the results, or a custom User Defined Function (UDF) to do the same thing - a UDF is a special type of macro using VBA code to get the results.

 

Additionally, the cells you put the start/end times into need to be formatted as date/time entries to do the math properly, and the final cell where you want to show the total for the week needs to be formatted as [h]:mm.  Finally if any scheduled time period goes across midnight, such as from 11 p.m. on one day to 2 a.m. on the next day, the entries need to include the date along with the time, although they can be formatted to only show the hours and minutes.

 

Here's a basic example:

The formula in the Total Hours column for Bill (row 3) is:

=SUM(C3-B3,E3-D3,G3-F3,I3-H3,K3-J3,M3-L3,O3-N3)

with the cell custom formatted as [h]:mm

The start/end times all include the date, and note that entries like Bill's Monday entry - midnight is in the 'next' day:  so the 2 entries for Monday were:

12/30/13 17:00   and   12/31/13 24:00

jllatham say:

If you absolutely MUST put both times into one cell, and I really strongly urge you not to, then the following UDF should deal with things for you.  You would put a formula referencing it where you want the total for the week to show up at.  The cells to be evaluate must be adjacent to one another (although it could be just 1 cell).  It's not real "robust" in that it doesn't test for possible error conditions such as a cell without proper pair of numbers in it, so if your entries are in error, it will probably bomb on you.

 

For example, lets say your 3 cells were in B2, C2 and D2, you would enter the formula like this:

=gettotaltime(B2:D2)

 

The code goes into a regular code module.  See this page for instructions on putting it there:

http://www.contextures.com/xlvba01.html#videoreg

 

Once you have added the code to the workbook, you must save it as a Macro Enabled workbook, type .xlsm or .xlsb.

 

Here is the code.  Read the comments at the beginning of it to see how it treats minutes entered.

 

Function GetTotalTime(scheduleDays As Range) As Single
'call from worksheet as
'   =GetTotalTime(B2:H2)
'     where B2 is the first cell containing scheduled times
'     and H2 is the last cell containing scheduled times
'NOTE: you may separate hour and minutes with either
'       a colon ( : ) or a decimal ( . ), however
'       in either case the minutes are treated as
'       minutes, not decimal portions of an hour, so
'       11:30 and 11.3 are the same thing.
'
  Const hrSeparator = "-"
  Const minSeparator = ":"
  Dim currentDay As Range
  Dim splitAt As Integer
  Dim splitHM As Integer
  Dim totalHours As Single
  Dim txtEHour As String
  Dim txtEMin As String
  Dim txtSHour As String
  Dim txtSMin As String
  Dim endHour As Single
  Dim startHour As Single
  
  For Each currentDay In scheduleDays
    If Not IsEmpty(currentDay) Then
      splitAt = InStr(currentDay, hrSeparator)
      If splitAt > 0 Then
        txtSHour = Left(currentDay, splitAt - 1)
        txtSHour = Replace(txtSHour, ".", minSeparator)
        splitHM = InStr(txtSHour, minSeparator)
        If splitHM > 0 Then
          If Right(txtSHour, 1) = minSeparator Then
            txtSHour = txtSHour & "0"
            splitHM = InStr(txtSHour, minSeparator)
          End If
          startHour = Val(Left(txtSHour, splitHM - 1)) _
           + (Val(Mid(txtSHour, splitHM + 1, Len(txtSHour) - splitHM)) / 60)
        Else
          startHour = Val(txtSHour)
        End If
       
        txtEHour = Mid(currentDay, splitAt + 1, Len(currentDay) - splitAt)
        txtEHour = Replace(txtEHour, ".", minSeparator)
        splitHM = InStr(txtEHour, minSeparator)
        If splitHM > 0 Then
          If Right(txtEHour, 1) = minSeparator Then
            txtEHour = txtEHour & "0"
            splitHM = InStr(txtEHour, minSeparator)
          End If
          endHour = Val(Left(txtEHour, splitHM - 1)) _
           + (Val(Mid(txtEHour, splitHM + 1, Len(txtEHour) - splitHM)) / 60)
        Else
          endHour = Val(txtEHour)
        End If
        If endHour < startHour Then
          endHour = endHour + 24 ' allows for crossing midnight once
        End If
        totalHours = totalHours + (endHour - startHour)
      End If
    End If
  Next ' end Each currentDay loop
  GetTotalTime = totalHours
End Function

The file name:

And the path:

 

=LEFT(CELL("filename"),FIND("[",CELL("filename"))-1)

 

These formulas will only work in a workbook that has been saved to disk, not in a newly created workbook that hasn't been saved yet.

macropod say:

As Peter indicated, you can get the date format you want by adding a formatting picture switch in Word. Here's a slightly different approach, using your existing mergefield:
. select the mergefield;
. press Shift-F9 to expose the field coding. It should look something like {MERGEFIELD MyDate} where 'MyDate' is your mergefield's name;
. delete anything appearing after the mergefield's name and add '\@ "d MMMM yyyy"' to the field, as in {MERGEFIELD MyDate \@ "d MMMM yyyy"}. With this switch your date will come out as '2 August 2008'. Other possible date formatting switches include:
  . \@ "dddd, d MMMM yyyy";
  . \@ "ddd, d MMMM yyyy";
  . \@ "d MMM yyyy";
  . \@ "dd/MMM/yyyy";
  . \@ "d-MM-yy";
    Note: Note: you can swap the d, M, y expressions around, but you must use uppercase 'M's for months - lowercase 'm's are for minutes.
. position the cursor anywhere in this field and press F9 to update it;
. run your mailmerge.

j.e. mcgimpsey say: I gave you the path:

In the Finder:

Starting in your home directory (conventionally represented by ~, since it's different for every user), open the Library folder. In the Library folder open the Preferences Folder. Alternatively, in any Finder window, enter

    com.microsoft.excel.plist

and click the File Name button (vs Contents - it'll still find it, but there will likely be more).

Likewise, in a Finder search box, enter 

Excel Toolbars

to find that file.
