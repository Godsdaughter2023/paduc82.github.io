viejo1000 say:

Please follow the below steps to troubleshoot with the issue:

Step 1. Modify TCPIPv6 Parameters through Windows Registry.

1. Open Windows Registry Editor. To do that:

  1. Press “Windows” + “R” keys to load the Run dialog box.
  2. Type “regedit” and press Enter.

2. Inside Windows Registry, navigate (from the left pane) and highlight this key.

Can Security Essential run during "sleep" mode? - Microsoft Community

donna m evans say:

Can Security Essential run during "sleep" mode?

Will Security Essentials run automatically while my computer is in "sleep" mode?  I always put it in "sleep" to save energy.  Also, when I opened up Security Essentials, a scan was in progress.  I have the scan set to run automatically at 3a.m. every day.  Right now it is 11p.m. and a scan is running.  What is up with that? 
kosh vorlon say:

Hi Donna,

 

MSE will not scan while the computer is shutdown, in sleep mode, in hybrid sleep mode, or in hibernation.  It will not awaken the computer to perform the scan even if it is scheduled at that time.  It will simply run at the next available time.

 

Now, it the computer awakens for some OTHER reason (I don't know how you have it configured in terms of events which will awaken it from sleep) and a scan is in the queue, then because the computer is now awake, it may begin the scan (but only because the computer was otherwise awake and not because it awoke it itself).

 

Under Settings / Scheduled Scan, do you have the box checked to start the scan only if the computer is on but not in use?  If so, and you are always working on the computer and when you step away you put it to sleep, then no idle time occurs and so the scan never has a chance to start - so it carries forward.  The scan running at 11PM could be the 3AM scan from the prior day that never occurred because you never used the computer or because this box was checked and there was no idle time for it to automatically start.  Also, remember that the timing is "about" in terms of time.  While 4 hours is much longer than I think would ever occur, I have had mine start 25 minutes earlier than the scheduled time (but I can't remember it ever being longer than that). 

 

Also, do you have the box checked to run an update before a scheduled scan?  If so, it needs an internet connection to begin in order to do the update.  If the connection is turned on and off, this can affect when it is able to start.  I leave my connection on at all times so it's not an issue for me, but if you have one where you pay for connect time and so have it disconnect after a certain idle period, this can prevent a scan from starting because it can't do the update first if that's how it is configured.  Then it can possibly be a long time before the scheduled scan is actually able to start - maybe even until the next night.  I don't know if these are applicable to you, but present them as options to examine to see if you see something that brings to mind a reason it might be occurring.

 

Also remember, in normal circumstances MSE starts automatically at startup.  If after that you decide to actually open the MSE program to check it out, it may have just started to run the scan automatically (unless you're saying when you awaken it it shows the scan has been going on for a long time - way beyond when you awoke it).  I would normally tell you that was impossible if you weren't telling me it was happening. 

 

Again, I strongly suspect the computer awoke for some other reason and found it essentially idle and had an overdue scan scheduled in the queue and so it started it and whatever other reason the computer awoke either wasn't visible to you or had already completed, but the scan had now started so it was going to keep the computer awake while it finished.

 

If none of the above works, I suggest you open a support case at: https://support.microsoftsecurityessentials.com/ where Microsoft Support experts will be happy to assist you with this problem.

 

I hope this helps.

 

Good luck!
