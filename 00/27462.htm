Code               Account

1148930        =VLOOKUP(A2,Sheet1!A2:B6,2,FALSE)

 

Note that the cells in Column A on Sheet2 are formatted as Text.

 

The VLOOKUP should return "Smith".  Instead we get #N/A.  If I manually add the apostrophe to the data in column A on Sheet2, then the VLOOKUP is correct.  Why is there a difference between text entered using an apostrophe, and cells formatted as text?

ashish mathur say:

Hi,

 

Merely formatting something to text doe snot change the characteristic of the entry from number to text - formatting is a mere display change/eye candy.  The number converts to text by putting a single quote.

 

On sheet 2, you may use the following

 

=VLOOKUP(A2&"",Sheet1!A2:B6,2,FALSE)

 

By concatenating a number with a null string, the number converts to text.

marshall barton say:

THe problem is that you are changing property values in design view (which sets a flag that the form's design needs to be saved) and then switching to form view where your code makes other property settings.  When you finally get around to closing the form, the save saves both sets of changes.

The only way to deal with these two different ways of setting properies is to close the form both before making design changes and save the changes after design changes so the save triggered by the internal design modified flag only has real design changes to deal with.

If you use DoCmd.Close Me.Name. acSaveNo, you will not save any property chaged via code, but you will also lose the changes made while in design view.

Your workaround of using the form's Open event to initialize all the properties that you change in code is a valid alternative, even if it requires additional code that is only needed because of your casual approach to making design changes.  Personally, I find it too difficult to remember to add a line of code in the Open event whenever I add code to change another property.

john p spencer say:

Did you use VBA code to set the AllowDesignChanges property to false?   You can do so in the immediate window with

    Forms!FormName.Properties("AllowDesignChanges") = False

Or you could do all forms with code like the following.

Public Sub sSetAllowDesignChanges()
' Set the AllowDesignChanges property to false
' on all forms so changes can only be made in
' design view
Dim dbAny As DAO.Database
Dim docAny As Document
Dim strFormName As String
Dim iFormCount As Long

   Set dbAny = CurrentDb()
  
   For Each docAny In dbAny.Containers("Forms").Documents

      strFormName = docAny.Name
      DoCmd.OpenForm strFormName, acDesign
      With Forms(strFormName)
         If .AllowDesignChanges = True Then
            .AllowDesignChanges = False
            DoCmd.Close acForm, strFormName, acSaveYes
            iFormCount = iFormCount + 1
         Else
            DoCmd.Close acForm, strFormName, acSavePrompt
         End If
      End With
   Next docAny
  
   MsgBox "Set AllowDesignChanges to False on " & iFormCount & " forms"
End Sub
