yoga yao msft say:

Hi DebK0404,

I would like to explain the combined formula firstly.

1. CELL function: Returns information about the formatting, location, or contents of a cell.

CELL("address",G2) indicates the cell G2 in reference, as text. For example, if Cell G2=Hello, then Hello=CELL("address",G2).

2. MID function: Returns the characters from the middle of a text string, given a starting position and length.

MID(CELL("address",G2),2,2) indicates extraction from the substring begins at position 2 and ends after 2 characters, such as el=MID(Hello,2,2).

3. INDIRECT function: Returns the reference specified by a text string.

INDIRECT(MID(CELL("address",G2),2,2) & $AR$8) indicates combination "el" with the value in AR8. For example, If AR8=10, then Cell EL10=INDIRECT(el&AR8).

Note: If you click AR8, and press F4, it would display $AR$8.

4. SUBTOTAL function: Returns a subtotal in a list or database.

SUBTOTAL(3,G9:INDIRECT(MID(CELL("address",G2),2,2) & $AR$8)) indicates it will count cells including hidden values. Then this formula counts the cells G9:EL10.

hansv mvp say:

The Calendar control came with Microsoft Access. It was always troublesome because it was not a 'native' control, and because users who don't have Access don't have the control by default, they had to download it separately.

 

So Microsoft decided to introduce a built-in date picker in Access 2007 (an excellent idea), and to discontinue the separate calendar control in Access 2010 (not a good idea).

 

It'd be best to replace the calendar control with a calendar userform. This will work in all versions. See for example http://www.vbaexpress.com/kb/getarticle.php?kb_id=543 (with free download).
