Inserting a value if cell is NULL or BLANK - Microsoft Community

bornrockwall say:

Inserting a value if cell is NULL or BLANK

Without referencing ANY cells, can a formula be placed in a column that checks to see if any cell is blank; and if so insert the value ".00"?

jllatham say:

I believe the question is self-contradictory: "Without referencing ANY cells" ... "see if any cell is blank..."

You certainly could not do it with a worksheet formula/function.  You could do it using a macro associated with the worksheet's _Change() event.  But the macro still has to reference a range of cells although you wouldn't see that on a sheet.

Open a new workbook, right-click on a sheet's name tab and choose [View Code] from the list, then copy the code below and paste it into the code module presented to you.  Close the VB Editor (use [Alt]+[Q] or the red-x).

Type something into column F, anything in any row (except 1 for this test) - notice that nothing appears in F1.  Now delete the entry you made: 0.00 should show up in F1.  You could actually do this with any cell on the sheet - placing 0.00 into it when, in this case, no entries or formulas were in column F.

Private Sub Worksheet_Change(ByVal Target As Range)
  'test column F for entries and if there are
  'currently none in it, put 0.00 into cell F1
  If Application.WorksheetFunction.CountA(Range("F:F")) = 0 Then
    With Range("F1")
      .NumberFormat = "0.00"
      .Value = 0
    End With
  End If
End Sub

jllatham say:

Now, THAT you could do with a formula as long as it's not in same column you are evaluating.

Assuming Excel 2007 or later, with 1,048,576 rows per column, then this formula would do it - we will examine column A and put this formula in cell B1

=IF(COUNTBLANK(A:A)=1048576,0,"something in a")

Format that cell to display numbers with 2 decimal places.  If you can put a limit for the COUNTBLANK() it would be more efficient, say something like COUNTBLANK(A2:A10000)=ROWS(A2:A10000)  where 10,000 would contain all data you might reasonably expect to enter into column A.  We use ROWS() with the same range to get that row count.

You have to decide what you are going to display when there are not 1,048,576 blank cells in column A.  And formulas in column A that return "" will count as blanks.  For example, put this formula into any cell in column A:

=IF(1=2,"OOPS!","")

It still counts as a blank cell -- it's not empty, but it is blank.

hansv mvp say:

Since there should be only one record in tbldata1 per employee, you effectively have a one-to-one relationship between tblmaster and tbldata1. Similar for tbldata2 etc.

This means that tblinforneeded is actually superfluous - it would be needed for a many-to-many relationship.

If each of the subjects (Education, Experience etc.) will have different questions, I think it'd be easier to place the responses in tblmaster itself. That way, you don't have to worry about creating records. For each of the subjects, you fill in the relevant fields in the employee record.

You wouldn't need the subform and the popup forms.

You could create a tab control on the main form, with a tab page for each subject.
