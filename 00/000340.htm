pete__b say:

What I am doing, I have a vacation tracking database that I need to see all the days that are still available for the year.  In the Criteria of one column field I already am using

In (SELECT VTDate FROM  tblVacationTaken GROUP BY VTDate HAVING Count(*)>2)

to get the days that are available that have but I need to get the other days as well.  The tblVacationTaken has all the scheduled days of the people and is also set with the criteria of the year.

In the end I hope to make it smart enough to weekends as they are not a given workday but for now I just need to know where to begin.   If it helps I have a number table with 366 numbers for future use.

ken sheridan say: The use of auxiliary calendar tables is a tried and trusted approach to problems of this nature in relational database applications.  The result set can be restricted to a specific leave year, or any date range for that matter, in a query's WHERE clause.  As regards weekends and public holidays, these can easily be omitted from the calendar table, and my demo, to which I referred you, includes the facility for this.

Let's assume you create a table named WorkDaysCalendar, which excludes weekends and public holidays, and your tblVacationTaken table includes a foreign key employeeID column referencing the primary key of an Employees table, then to return all available days for each employee in the current calendar year would require a query as follows:

SELECT employeeID,calDate AS AvailableDate
FROM WorkdaysCalendar, Employees
WHERE calDate BETWEEN #2018-01-01# AND #2018-12-31#
AND NOT EXISTS
    (SELECT *
     FROM tblVacationTaken
     WHERE tblVacationTaken.VTDate =WorkdaysCalendar.calDate)
ORDER BY employeeID,calDate;

The outer query returns the Cartesian product of the Employees table and a subset of rows for the current year from the WorkdaysCalendar table, i.e. it returns each non-working, non-public holiday date in the year for each employee.  This is restricted by the subquery, which is correlated with the outer query on the date columns, to those days per employee on which no leave has been taken.
