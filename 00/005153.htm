mike middleton say:

When you post a reply on this system, the reply may be seen by anyone.

It is important to distinguish between the underlying content of a worksheet cell and the way that the content is displayed in the worksheet cell.

The underlying content of most of your worksheet cells should be numbers (or formulas that return numbers). When a cell is selected, the formula bar should show only a numerical value or a formula. If the formula bar contains any text, then the content is not numerical. You should not try to SUM cells that contain text.

The way that the numbers are displayed in worksheet cells depends on the format applied to that cell. One way to see the possible formats is to select a cell, and in the top menu choose Format > Cells > Number. Many of the cells in your worksheet use an Accounting format with a currency symbol of KRW or $. You can apply the appropriate format to each of your cells.

Cell C13 contains text. Select C13, type the six digits 615000, and press Return. Then cell C13 contains a number, and the cell is already formatted to use an Accounting format, specifically "KRW - South Korean Won." In the Format Cells > Number > Accounting > Currency symbol dropdown list, this entry is far down towards the bottom of the list.

mike middleton say:

In the "Aug 2015" worksheet, cell B71 has the formula =SUM(C13:C22,C24:C30,C32:C34,C36:C48,C50,C51,C53,C55:C61)

In cell F13 I entered the function =ISNUMBER(C13), and I dragged the fill handle down to copy the function into F13:F61.

The ISNUMBER function returns FALSE for the following cells (referred to in the B71 formula): C13:C15, C25, C27, C36:C37, C44, C47, C51, and C55.

Those cells contain text (that looks like a number) that is evaluated as zero in the SUM function.

Using the same approach for column D, the ISNUMBER function returns TRUE for the cells referred to in the C71 formula.

ossiemac say:

I am assuming that only one shape by the required name will be found in the workbook. ie each shape name is unique throughout the workbook. If not, then the code below will not do what you want.

Suggest that you make a backup copy of your workbook before running the code or you could be going through all of your shapes and re-setting colors manually.

Sub CurtainwallProgress()
    
    Dim CurtainWall As String
    Dim Progress As String
    Dim lngOffset As Long
    Dim ws As Worksheet
    Dim shp As Shape
    Dim sglRGB As Single
                 
    Application.ScreenUpdating = False
   
    'Offsetting by zero is valid code but no offset occurs on first loop
    lngOffset = 0
   
    Do
        'Sheet1 is the CodeName; not the Tab name
        CurtainWall = Sheet1.Range("A2").Offset(lngOffset, 0).Value
        Progress = Sheet1.Range("A2").Offset(lngOffset, 7).Value
       
        For Each ws In Worksheets
            Set shp = Nothing   'Initialize to nothing
            On Error Resume Next    'If shape not found then code errors
            Set shp = ws.Shapes(CurtainWall)    'Assign shape to a variable
            On Error GoTo 0
            If Not shp Is Nothing Then Exit For     'Not nothing so is something (Found)
        Next ws
       
        If shp Is Nothing Then
           MsgBox "Did not find shape " & CurtainWall
           GoTo PastColorSet
        End If
       
        Select Case Progress
           Case "Nothing"
               sglRGB = rgbWhite   'Assigns color to a variable
           Case "Fabricated"
               sglRGB = rgbBlue
           Case "Shipped"
               sglRGB = rgbYellow
           Case "Delivered"
               sglRGB = rgbFuchsia
           Case "Installed"
               sglRGB = rgbLime
           Case Else
               sglRGB = rgbWhite
        End Select
       
        shp.Fill.ForeColor.RGB = sglRGB    'Use variable to assign color to shape

PastColorSet:
   
    lngOffset = lngOffset + 1  'Increment offset ready for next loop
   
    Loop Until Sheet1.Range("A2").Offset(lngOffset, 0).Value = ""
   
    Application.ScreenUpdating = True
   
End Sub

It's possible that the settings in the Options screen aren't being written or updated in the registry, which is where Word looks when it's starting. You can try repairing Office; if that doesn't help, it could be a problem with permissions in the registry, which can be difficult to solve.

Since you will probably only want to change the setting once, you can do it directly in the registry.

There is one registry entry that controls all Office programs' Start screens, and a separate one for each Office program. Where you need to look depends on what behavior you see. For all of these, use the registry editor (regedit.exe) running as Administrator to enable changes.

  • If only Word doesn't display the Start screen, while Excel and other programs do display it, then go to the key HKEY_CURRENT_USER\SOFTWARE\Microsoft\Office\16.0\Word\Options and set the value DisableBootToOfficeStart to 0.
  • If none of the Office programs display their Start screens, then go to the key HKEY_CURRENT_USER\SOFTWARE\Microsoft\Office\16.0\Common\General and set the value DisableBootToOfficeStart to 0.
    • If the value doesn't exist yet in this key, you can create it. Right-click the key name, click New > DWORD, name it DisableBootToOfficeStart, and set it to 0.

Record a macro for that? Forget it, that's to difficult.

The first step is to make the macro work in one sheet, after that you can execute the same macro on (part of) all sheets.

Okay, the simplest basic sort is that, please read the help for Range.Sort:

Sub Test()
  Range("B16:AK69").Sort Range("B16")
End Sub

But that's static, you need a dynamic one:

Sub Test2()
  Dim R As Range
 
  'Find the last row in column B
  Set R = Range("B" & Rows.Count).End(xlUp)
  'Expand to the data rows
  Set R = Range("B16", R)
  'Expand to the columns
  Set R = Intersect(Range("B:AK"), R.EntireRow)
  'Sort
  R.Sort Range("B16")
End Sub

Please read the help on Range.End, Intersect, Range.EntireRow

dale howard [mvp] say:

Delta Dawn 007 --

Apply any task view, such as the Gantt Chart view.  Right-click on the Duration column and select the Insert Column item on the shortcut menu.  In the list of columns, select the column named Marked.  Notice that the Marked columns is a Yes/No or Flag field.  This is the column you will use to hide random rows using a filter.  Set the Marked column value to Yes for every task row you want to hide.

To create the filter, click the View tab to display the View ribbon.  In the Data section of the View ribbon, click the New Filter item.  In the Filter Definition dialog, enter a name for the filter, such as Hide Rows.  In the data grid of the dialog, enter the following filter criteria:

Marked          Equals          No

Select the Show Related Summary Rows checkbox if you want to see summary tasks displayed when filtering.  Click the Save button to save this new filter.  Microsoft Project 2010 will save the filter automatically in both the project in which you created it, and in your Global.mpt file to make it available to all current and future project.

Each time you want to hide rows, set the value to Yes in the Marked column for the rows you want to hide and then run this custom filter.  Hope this helps.
