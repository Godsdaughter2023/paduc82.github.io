brian tillman [outlook mvp 2007-2019] say:

IMAP accounts leave message on the server always.  What you see in the IMAP folders is what you should see if you log into the account using a web browser.  Moreover, all of the folders are synched two-way.  While it's not necessary, you can also configure the account to put sent messages on the server.

 

POP3 accounts, however, don't operate that way.  First, the can access only the server-side Inbox.  No other folders on the server are detectable   Second, after messages are downloaded from the server, usually they are deleted from the server so that the only copy is on your PC in the Outlook folders.  Of course, you can configure the account so that it leaves copies of message on the server so that you can log in via a web browser and see them there as well as on your PC.  Third, the access on one-way, fro the server to the client.  Nothing can return back to your mailbox.

 

So, something to keep in mind: messages sent via the POP3 account can never get stored on the server, since mail cannot flow to the mailbox that way.  Only your IMAP account can access the server's folders.  So, your PST folders may contain messages not on the server and not in your IMAP folders because you removed them from the server when you downloaded them.  If you delete the POP3 PST, you will lose those messages and you will not be able to recover them.  However, removing the POP3 account should not delete the PST, so you can still access those folders.  And since the IMAP folders can't be your default folders, you will always have to two folder sets: one to contain your calendar, contacts, tasks, and all the other non-mail folders in Outlook, and your IMAP folders. will be dedicated to mail.

 

You can also, or instead, if you wish, move your PST data back to the server by simply dragging the contents of the folders (or, in fact, the folders themselves if they are not the same as the IMAP folders) from the PST to the IMAP data file.  This should push them back to the server.  If your mailbox is capacious, with a large storage limit, this shouldn't be a problem if it's what you'd like to do.

SKYPE For Business - How to keep a common area phone presence to - Microsoft Community

robgaul say:

SKYPE For Business - How to keep a common area phone presence to "Available"

We have added a number of common area phones to response groups for our student workers. Our problem is that they phones drop out from the response group. we determined that the phones were going to a presence setting of "unavailable" and of course the response groups calls would not carry through to that phone.

We created a new user account for the phone, carried over the SIP address, added the phone gave it a login. We set the presence in the phone to available but after the default of 15 min it went unavailable. We changed the time to our max 30 min but of course that just delayed the inevitable.

During the writing of the question I came about the answer after reading through the admin config for the Polycom VVX410.

I changed the master config file on our tftp server and added the following:

<pres pres.idleTimeout.officeHours.enabled="0"
pres.idleTimeout.offHours.enabled="0" />

This disabled the time out settings for the presence and the phone has been available now for several hours. The best thing about this is now I do not need to edit each phone.

I hope this helps others as now we can have any volunteer use the phone and represent our college with out a login.

Rob Gaul
