macropod say:
This would normally only occur if the column has a mix of data types in the column (e.g. empty cells, text or numbers, plus the dates), especially in the first 8 rows. This includes text strings that are formatted to look like dates. If practicable, you should ensure there are only dates in that column. To check whether there is anything other than dates in this column, temporarily change the number format to, say, General. Do all the dates change to 5-digit numbers? macropod say:

The number formatting is easily handled via numeric picture switches. See 'Mailmerge Number & Currency Formatting' at:

http://www.msofficeforums.com/mail-merge/21803-mailmerge-tips-tricks.html

or:

http://windowssecrets.com/forums/showthread.php/163017-Word-Mailmerge-Tips-amp-Tricks

The Date & 'pending' problems are the result of having mixed data types, just as my previous post describes. There's some discussion of this in the above links also, under the 'Mailmerge Data Format Problems' heading. There are various ways these could be handled, including via a DDE connection or the 'Test Whether a Text Mergefield Contains Numeric Data' test described in the links. For example, with the latter, to manage:

• 'pending' in the data, you might have a field in the mailmerge main document coded as

{QUOTE{SET Val {MERGEFIELD Data}}{IF{REF Val}= {=Val} "{=Val \# $,0.00}" "Pending"}}

• 'Net 30'

{QUOTE{SET Val {MERGEFIELD Data}}{IF{REF Val}= 0 "Net 30" "{Val \@ "DD MMM YYYY"}"}}

See 'Note 1' in each of the above links about constructing these fields.

There is a limit to what can be achieved, though, and you really shouldn't be mixing data types the way you are. Having true dates and 'text' dates in the same column is especially poor practice and there's probably not much that can be done with those.

beth melton say:

That's really the only solution if you also want to use the new graphics engine (Escher 2 graphics) in Word 2010 and still be able to make changes to the objects. When you save the file in the legacy format (97-2003 .doc) grouping functions as it did before because you're also converting the objects to an earlier graphics engine (Escher 1 graphics). However, as you noted, the new features aren't available.

Now, if the objects are final you can copy/paste the grouped objects as an image if you want to make sure they're treated as a single object.

What I do for my artwork in Word documents is to use PowerPoint. I'll create diagrams and such there and when I'm finished, I'll copy the group in PPT and in Word use Paste Special and paste it as an .png. (Actually I have a macro for this.) This way I still have my original artwork if I need to make a modification. Note there are several reasons I use this method. 1) I started using it before Word had the new graphics engine and could do more with graphics in PPT. 2) I don't want anyone modifying my artwork and pasting as an image is an easy way to prevent this from happening. 3) Many times I need keep my artwork in individual files and PPT has the ability to save the grouped objects as a picture. (Granted, Word 2010 finally has this option now as well...)

Personally, I like the ability to move/resize/delete objects that are grouped and find it saves me a lot of time and didn't like needing to ungroup objects to make these types of changes. Like any feature, each user's preferences are bound to vary. :-)

graham mayor say:

Your macro deletes UK English formatted text (the second of the language statements) and there is no reason why it should not delete UK English text in a table provided the table is in the body of the document.

If you want to delete US English wherever it appears then you need to process all the ranges e.g.

Sub ProcessDoc()
Dim oStory As Range
    For Each oStory In ActiveDocument.StoryRanges
        DeleteUS oStory
        'DeleteUK oStory 'Optional
        If oStory.StoryType <> wdMainTextStory Then
            While Not (oStory.NextStoryRange Is Nothing)
                Set oStory = oStory.NextStoryRange
                DeleteUS oStory
                'DeleteUK oStory 'Optional
            Wend
        End If
    Next oStory
    Set oStory = Nothing
lbl_Exit:
    Exit Sub
End Sub


Sub DeleteUS(oRng As Range)
    With oRng.Find
        .ClearFormatting
        .Text = ""
        .LanguageID = wdEnglishUS
        With .Replacement
            .ClearFormatting
            .Text = ""
        End With
        .Forward = True
        .Wrap = wdFindContinue
        .Format = True
        .MatchCase = False
        .MatchWholeWord = False
        .MatchWildcards = True
        .MatchSoundsLike = False
        .MatchAllWordForms = False
        .Execute Replace:=wdReplaceAll
    End With
End Sub

If you want to delete both UK and US English then you could create a second sub to DeleteUK and call both macros from the first macro

Sub DeleteUK(oRng As Range)
    With oRng.Find
        .ClearFormatting
        .Text = ""
        .LanguageID = wdEnglishUK
        With .Replacement
            .ClearFormatting
            .Text = ""
        End With
        .Forward = True
        .Wrap = wdFindContinue
        .Format = True
        .MatchCase = False
        .MatchWholeWord = False
        .MatchWildcards = True
        .MatchSoundsLike = False
        .MatchAllWordForms = False
        .Execute Replace:=wdReplaceAll
    End With
End Sub
