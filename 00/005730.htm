Splitting a table in Access - Microsoft Community

eskimojnr say:

Splitting a table in Access

I imported an excel spreadsheet with many records made up of the following notable fields:

- First Name

- Last Name

- House Number

- Street

- City

I would like to split this table into a table with just names, and a child table with address information as there is a 1:many relationship of people to address.

Can someone suggest how I might best proceed?

ken sheridan say: Import the Excel worksheet into a temporary holding table, and then create the following empty tables:

Contacts
….ContactID (autonumber PK)
….FirstName
….LastName

Addresses
….AddressID (autonumber PK)
….Street
….CityID (FK)
….ContactID  (FK)

Cities
….CityID (autonumber PK)
….City

Then execute the following 'append' queries:

INSERT INTO Contacts(FirstName, LastName)
SELECT DISTINCT [First Name], [Last Name]
FROM [HoldingTable];

INSERT INTO Cities(City)
SELECT DISTINCT City
FROM [Holding Table];

INSERT INTO Addresses(Street, CityID, ContactID)
SELECT DISTINCT Street, CityID, ContactID)
FROM ([Holding Table] INNER JOIN Contacts
ON [Holding Table].[First Name] = Contacts.FirstName
AND [Holding Table].[Last Name] = Contacts.LastName)
INNER JOIN Cities ON [Holding Table].City = Cities.City;

The above makes the following assumptions:

1.  Each contact can be distinctly identified by their first and last name.  This might be a risky assumption.

2.  Each contact can have one or more addresses, but each address can be occupied by one contact only, i.e. the relationship type between contacts and addresses is one-to-many as you said.

3.  Each city can be identified distinctly by its City name.  Again this might be a risky assumption.

Note that in the final Access tables I have removed the spaces from the imported column names like First Name.  It is not good practice to include spaces or other special characters in object names.  Use either CamelCase as I've done, or represent a space by an underscore character, e.g. First_Name.  I've assumed that the original column names with the spaces have been retained in the temporary holding table.

Note also that I have included a Cities table.  Cities are a separate entity type and therefore should modelled by a separate table.

You'll find an example of how to import and decompose data like this in DecomposerDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file imports a simple Excel worksheet into Access and executes the queries to decompose it into the predefined related tables, with a brief explanation of the query at each step of the operation.

jeeped say:

You have run into the Last_Cell property of an Excel worksheet. Some cell outside of your currently used data range either contains data or contained data at one time.

You say you have 15 rows of data. Assuming you have a number of columns as well (say 10 columns) you have a currently used range of data of A1:J15. I'll use that for an example. You will have to modify J15 to suit your own data.

You can test the current extents of Last_Cell by pressing CTRL+End.

If you are absolutely sure that J15 is the last cell with data in it, walk through these steps,

  1. Press F5, type in K16 and hit <enter>. That is K16, one down and one right of J15.
  2. Press CTRL+Shift+<right arrow>. Release and press CTRL+<spacebar>. Release and press ALT+E then D or right click the highlighted columns and select Delete.
  3. Press Home, then press CTRL+Shift+<down arrow>. Release and press Shift+<spacebar>. Release and press ALT+E then D or right click the highlighted rows and select Delete.
  4. Press CTRL+Home to return to A1. Immediately Save or Save As if you don't trust your keystrokes.
  5. Press CTRL+End and you should be in the Last_Cell of your current used range (e.g. J15).

It might seem like a lot, but it takes a lot more time to type it than it does to actually do it.

keith- say:

First, check to see if required services for Microsoft Security Essentials are set to Automatic:

1.      PressWindows Key + R on your keyboard

2.      In theOpen: field type services.msc

3.      ClickOK

4.      Make sure the following services are set to Automatic (not Automatic (Delayed Start)):

·        Windows Installer Service

·        Cryptographic Services

·        Background Intelligent Transfer Service

5.      If you need to change the Startup Type:

·        Right click the service in question

·        ClickProperties

·        ForStartup type:, select Automatic

·        ClickOK

This can also be caused by a corrupted user profile. Instructions to test/fix this can be found by clicking the appropriate link:

Windows 7 – Fix a corrupted user profile

Windows Vista – Fix a corrupted user profile

How to copy data from a corrupted user profile to a new profile in Windows XP

 

If neither of the two fixes above solved the problem for you, the next step will be to uninstall and then reinstall Microsoft Security Essentials:

 

Download the installer for Microsoft Security Essentials. Then follow the applicable uninstall instructions for you operating system. After the restart use the installer you downloaded to reinstall Microsoft Security Essentials.

 

DownloadMicrosoft Security Essentials (this link would be changed to the location of the new product’s download)

For Windows Vista or Windows 7:

  1. Close all programs.
  2. ClickStart, and then clickControl Panel.
  3. ClickUninstall a Program under thePrograms category
  4. In the list of currently installed programs, clickMicrosoft Security Essentials, and then clickUninstall.
  5. Follow the instructions to uninstall Microsoft Security Essentials.
  6. When the uninstall program is finished, restart your computer.

 

For Windows XP:

  1. Close all programs.
  2. ClickStart, and then clickControl Panel.
  3. ClickAdd or Remove Programs.
  4. In the list of currently installed programs, clickMicrosoft Security Essentials, and then click Remove.
  5. Follow the instructions to uninstall Internet Explorer 8.
When the uninstall program is finished, restart your computer.
