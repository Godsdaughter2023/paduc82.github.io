Outlook.com does not use UTF-8 encoding, thus non-latin characters are not displayed properly in many clients

When sending a message that contains non-latin, characters in the Subject and/or Message Body, the "charset" used by Outlook.com is not "UTF-8" (the most widely used & accepted standard for non-latin characters), hence many different clients (offline and web-based), cannot properly display the message content.
For example, a message that has Hebrew characters, is sent using the charset: "iso-8859-8-i", instead of "UTF-8" and many 
different clients (offline and web-based), cannot properly display these characters, hence rendering Outlook.com unuseful and forcing customers to tranfer to a different email service provider, such as Google (Gmail), which uses UTF-8.

Here is the relevant header portion of such an email message using Outlook.com, which many clients cannot properly display:
Content-Type: text/html; charset="iso-8859-8-i"
Content-Transfer-Encoding: quoted-printable

Here is the relevant header portion of the same email message, sent using gmail.com, which probably ALL clients (tested at least 20) display properly:
Content-Type: text/html; charset=UTF-8
Content-Transfer-Encoding: quoted-printable
