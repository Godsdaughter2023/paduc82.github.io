You can use the StrComp function in VBA code or in an expression to compare strings:

 

StrComp(string1, string2, vbBinaryCompare) will return 0 if the strings are exactly equal (case sensitive), and a non-zero value otherwise. So you could use code like

 

    If StrComp(string1, string2, vbBinaryCompare) = 0 Then
        ' OK, password matches
        MsgBox "Collect $200"
    Else
        ' Password does not match
       MsgBox "Go to Jail"
    End If

 

Alternatively, you can change the default

 

Option Compare Database

 

at the top of the code module to

 

Option Compare Binary

 

This will make ALL string comparisons in that module case sensitive.

jay freedman say:
Andreas,

Office 2010 doesn't allow any changes within the built-in groups, either by custom UI or through the Customize Ribbon dialog.

What you can do, though, is to use the Customize Ribbon dialog to create a custom group that contains copies of all (or just some) of the controls in the built-in group plus whatever macros you want to add; and then you can hide the built-in group.

If you want to see the XML for this, you can use the Import/Export button in the dialog to export it to a *.exportedUI file. When I created a custom Index group on the References tab, repeated all three standard Index buttons on it, and added one for an UpdateAllFields macro, the exported UI contained this XML just before the </mso:tabs> tag:

<mso:tab idQ="mso:TabReferences">
  <mso:group id="mso_c1.14291F3C" label="Index" insertBeforeQ="mso:GroupTableOfAuthorities" autoScale="true">
    <mso:control idQ="mso:IndexMarkEntry" visible="true"/>
    <mso:control idQ="mso:IndexInsert" visible="true"/>
    <mso:control idQ="mso:IndexUpdate" visible="true"/>
    <mso:button idQ="x1:UpdateAllFields_0_142B809D" label="Update All Fields" imageMso="TagMarkComplete" onAction="UpdateAllFields" visible="true"/>
  </mso:group>
  <mso:group idQ="mso:GroupIndex" visible="false"/>
</mso:tab>

As far as I know, this file can't be used by Word unless you use the Import part of the Import/Export button -- it isn't like the customUI in Word 2007.
