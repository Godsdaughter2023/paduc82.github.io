doug robbins - mvp office apps & services (word) say:

When a merge is executed, the data from the fields in each record is inserted into the new document as ordinary text so there is no way of locking it.

If the information that you want to lock can be indentified in some way, then it would be possible to run a macro that inserted a Text Input FormField in place of the information and set the .Result of the FormField to the information and set the enabled property of the FormField to false BUT, for that to be effective, you would have to protect the whole document for Filling in Forms and the user would not be able to edit any other part of the document, so you might just as well protect the document with a password in the first place.

So, do you want the user to be able to edit any part of the document?

If not, you could modify the code in the addin that you can download from fellow MVP Graham Mayor's website at:

http://www.gmayor.com/individual_merge_letters.htm

so that each document is protected with a password as it is created, or use the option to create .pdf files in that Add-in.  Or if all of the documents produced by the mail merge were in the one folder, you could run a batch process that opened each document in turn and then saved it with a password by modifying the code in the article at:

http://www.word.mvps.org/FAQs/MacrosVBA/BatchFR.htm


-- Hope this helps.

Windows Defender on Windows 8 integrates a more robust version of Windows Defender (and uses that name) for its anti-virus and anti-malware protection. If you're currently using Windows Defender on Windows 8, then you shouldn't be using Norton which most likely came as a trial when your purchased your computer.

TOC shows content instead of heading

Any thoughts why and hat to do?

Thanks a lot in advance

pamcaswell say:

It sounds like your headings run into the regular text paragraph.  If you are using Word's built-in headings (or any linked paragraph style), applying a style to a portion of a paragraph  (in W2007-2013) should work .  But it is not compatible with autonumbered headings.  (I haven't tried this feature in W2013 since the evaluation stage, so that may  have changed.)


You can also use the style separator between the two styles. The style separator is compatible with autonumbered headings.   Insert a paragraph mark after the heading.  Move the cursor to before the  paragraph mark and press Ctrl+Alt+Enter.  Word will apply special formatting to the paragraph mark at the end of the heading.   Apply the paragraph styles you want before and after the style separator.  After you have created one style separator, you can copy and paste it into the other places.


Another possibility is that there are only  line breaks (down & left pointing arrow) instead of paragraph breaks (¶) after the headings.  Make sure the headings end with ¶s.
