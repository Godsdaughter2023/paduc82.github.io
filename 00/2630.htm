ken sheridan say: If you wish to use VBA code for a control's event property and have it copy with the control, then rather than using an event procedure, wrap the code in a function declared as Public in a standard module.  Then set the event property of the control in its properties sheet to the function:

=NameOfFunctionGoesHere()

When you copy and paste the control into another form or report, the event property is copied with it, and the same code in the function will be executed when the event is triggered in either object.  If you wish to vary the code on the basis of which object calls it, then you can pass the object's name as an argument into the function, e.g. in the case of a form:

=NameOfFunctionGoesHere(Form.Name)

and reference the argument in a Select Case statement in the function's code.

What's the point of In-Place Archive?

Going to Exchange Online Plan 2 you get unlimited mailbox space via In-Place Archiving (a second mailbox).  Why not just have an unlimited size primary mailbox?  Seems like this adds confusion and unnecessary complication.

alan wu msft say:

Hi Esark33,

Archive mailboxes (called an In-Place Archive in Exchange Online) help people in your Office 365 organization take control of messaging data by providing additional email storage. Using Outlook or Outlook Web App, people can view messages in their archive mailbox and move or copy messages between their primary and archive mailboxes. In-Place Archive can only be used to archive mail for a single user or entity for which a license has been applied. That is to say, only the user’s primary mailbox has been assigned with the Exchange online license, the user’s Archive mailbox can be enabled.

In general, you can regard an Archive mailbox is a supplementary mailbox of a user's primary mailbox. 


Regards,
Alan

trevor j lawrence say:

As Entegy says, the primary MS account you use on the phone must sync contacts/calendar.  However any other MS account you add to the phone can optionally sync contacts/calendar.  Therefore one solution is to use two MS accounts.  The first is the primary and used ONLY for purchasing apps.  The second one you add can be set as the default for calendar/contact additions and you can configure it not to sync these  but to sync emails.

 

It's a workaround and not without issues.  your current MS account is the one you'll want to use as the "2nd" account, and a new dummy one as the primary.  If you've bought many apps on your current acct, they would not be available on the new setup.  Also you'd need to do a hard reset on the phone to set this up.

 

From my point of view, what is wrong with contacts/calendar syncing to the cloud?  I couldn't operate without it!

entegy say:
If you're worried about security, you might want to look into turning on two-factor authentication (2FA) on your account. When this is on, even if someone gets a hold of your password, they cannot access your account without a special code your phone generates.

I will admit this up front, yes, 2FA adds a bit more hassle to your life. You will always have to have your phone to log into your account on non-trusted devices. Devices that aren't compatible with 2FA require will require a special app password to log in. But in my opinion, the security it offers is worth the extra hassle.
