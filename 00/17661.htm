dhookom say: I would suggest using DLookup() which is mostly the same as a query that returns a single value. The DLookup() and other domain aggregate functions have an optional where condition which allows you to filter the results.
=Dlookup("[Your Field Name]","[Your Query Name]","StudentID=" & [StudentID])

The specific syntax depends on your query and field names as well as data types.

scottgem say:
"I don't know how to refer to the result of a query in VBA."

That's because you can't do it. Since a query can return multiple rows and columns, there is no way to place the results of a query into a variable. What you would do is use a DLookup function for that:

DLookup("[Balance]","qryBalances","[StudentID] = " & Me.StudentID)

unclehefty say:

create a3 booklet in word

I may have confuse everyone with a previous question, for that I apologise.  I have a 36 side A4 document on word and want to publish it as a booklet, stapled down the centre fold in a3 format.  I have looked at publisher format and that only appears to have a4 single side.  So what I want to end up with is a document that, when folded, will hold 18 leaves, each printed on two sides.  Does that make more sense?

Uncle H
daniel g. grau say:
What you need to do is:
  • Load A3 paper and select Print. In Layout, specify 2 pages per sheet (the document's page size should be A4), then manually enter (or paste) this page range: 36, 1, 34, 3, 32, 5, 30, 7, 28, 9, 26, 11, 24, 13, 22, 15, 20, 17
  • Flip the stack of printed pages over, then feed into the input tray.
  • Now print with this range: 2, 35, 4, 33, 6, 31, 8, 29, 10, 27, 12, 25, 14, 23, 16, 21, 18, 19
  • Depending on how your printer stacks printed pages (face down or face up), you'll need to specify the second sequence in reverse order by pairs. You can do this by selecting "print in reverse order" (if available), or use this range: 18, 19, 16, 21, 14, 23, 12, 25, 10, 27, 8, 29,  6, 31,4, 33, 2, 35
You will likely not be successful on the first or second tries, until you deduce how to properly feed the pages into your printer. Also, try printing the second set of pages first if the first sequence does not result in a final output will all pages properly ordered.

Finally, write the method down for the next time you need it, because it's not exactly intuitive.
