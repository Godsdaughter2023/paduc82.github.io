That could be a symptom of a problem with the DNS lookup and having search from the Address bar enabled.   Can you cache the lookup before trying to make IE use it?  E.g. do a  ping -n 1 -w 1  nameofsite  
(and ping also any associated host names if it is an alias).   Then use  ipconfig /displaydns  to see how much time (Time To Live in seconds)  IE would have to make use of that lookup.   Since the host name is a single word and not a recognizable domain name you must make sure that search from the Address bar is disabled at least for that case.  Actually, I'm surprised that the protocol prefix would be ignored (assuming it is being given, otherwise that makes this lookup problem explanation even more likely IMO.)   You could also use  Fiddler2  to trace what is going on in terms of requests.   In terms of lookup you would have to use  NetMon or WireShark  but Fiddler can give you some clearer clues about whether a lookup has been done.   In addition you could run ProcMon to see what is going on in terms of registry and file access but you would probably want to have two identical traces, one of it working and one of it not working to be able to use them for comparison purposes.  (However, I'm not even sure that there would be enough significant registry or file accesses to make it worthwhile to acquire or analyze this diagnostic.)

karl timmermans [outlook mvp 2012-2018] say:
going to try and simplify the explanation by going through where things were and where they are now

#1 - The fact that you have folders with the label "This Computer Only" means that the underlying data file (an .ost file) belongs to an account configured as an IMAP account. The folders labeled as "This Computer Only" are for contact/calendar/task folders which will be addressed separately for the moment

#2 - If you did nothing else other than configure the same email account in a new Outlook profile (as part of the reconstruction process if replacing the hard drive) - all email folders would have been created and sync'd with the server. The .ost file in terms of email folders is a mirror image of what exists on the server. Nothing to import from old to new

#3 - Re duplicate "This Computer Only" contact folders. That would ONLy occur if you added a PST file that was the result of exporting the primary .ost file for the IMAP account. it would never occur by configuring multiple IMAP accounts in the same Outlook profile

#4 - Re empty "This Computer Only" contact folders

That's only going to happen if Outlook could not locate/open the primary .ost file for the IMAP account in which case, a new .ost file would be created and folders labeled as "This Computer Only" would be empty. Translated, it means that if the contacts were originally contained in a contact folder marked as "This Computer Only" and the folder was not manually exported (backed up) to a PST file, that data is lost.

#5 - So what to do going forward?

First, determine if you have contacts located somewhere on one of the contact folders. Then do the same for calendar items. If none exist, then it would seem you're out of luck) but assuming these exist somewhere in one or more of the folders..................

Second, I would recommend creating a new PST file (from Home Ribbon --> New Items --> More Items --> Outlook Data File

Third, in the newly created PST file create contact & calendar folders as required. Be sure to set the folder type to reflect the type of items to be stored in the folder (i.e. contacts / calendar etc)

Fourth, move the required data to the appropriate folder(s)

Fifth, take note of the name/location of the newly created PST file

#6 - With all the above done and verified, close Outlook, open the Control Panel <Mail> app, create a new Outlook profile and configure the IMAP email account. Make sure to set the newly created Outlook profile as the default for the next time Outlook is opened

#7 - Once #6 is completed, open the PST file that was created which contains the contact/calendar data (go to the Account Settings --> Data file tab --> add the PST file --> set it as the "default data file for the profile". This is where the default contact/calendar folders will be located

#8 - Assuming a single email account, you should be all set to go. Anything else will require additional info

Step #7 is a summary of what's contained in

How to Protect Outlook '2013/'2016 Imap Contact/Calendar Data
https://www.contactgenie.info/how-to-protect-ou...

There's an alternative to the above and that is by creating a free Outlook.com account, configure it in Outlook, make the .ost data file associated with the account the "default data file" for the profile and copy the contacts/calendar data to the corresponding folders in the outlook.com account. That way, the contact/calendar data will be accessible by any device where the Outlook.com account is configured

No doubt there will be other questions so will stop this extended explanation here at this point
