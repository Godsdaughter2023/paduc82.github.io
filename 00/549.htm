stevedig say:

OK,

 

Figured it out.

 

First you have to change the install location when you do the initial install. It will try to install into your profile rather than a location available to all users. I changed the install location to c:\program files (x86)\microsoft\Fuzzy Lookup Add-In for Excel\

 

Then, under the install account,.export the following registry key and have the user import it when they log in:

 

HKEY_CURRENT_USER\Software\Microsoft\Office\Excel\Addins\Microsoft.FuzzyLookupAddInForExcel

 

 

They will have rights to do the registry key import as it goes into HKCU

section page and document page on same page? - Microsoft Community

cyj_978 say:

section page and document page on same page?

I have a lengthy Word 2010 document with many sections.  I want to put the section page numbers in the headers as "Page X of Y" and the document page number in the footer.  I have already inserted section breaks and numbered the pages of each section.  I want to add the document page number to each page.  how do I do this?
doug robbins - mvp office apps & services (word) say:
I have a lengthy Word 2010 document with many sections.  I want to put the section page numbers in the headers as "Page X of Y" and the document page number in the footer.  I have already inserted section breaks and numbered the pages of each section.  I want to add the document page number to each page.  how do I do this?


You will need to insert a bookmark at the end of each Section (I suggest using the bookmark names S1, s2, S3, etc.

and then starting with the second section, you need to use the following type of field construction

{ = { PAGE } + { PAGEREF S1 } }

In the third Section, you would use

{ = { PAGE } + { PAGEREF S1 } + { PAGEREF S2} }

etc.

In the first section, you can just use { PAGE }

Note, you must use Ctrl+F9 to insert each pair of field delimiters { } and you use Alt+F9 to toggle off the display of the field codes.


does this give me the section page numbers or the document page numbers?


It assumes that you have the page numbering in each Section set to restart at 1.  The { Page } field by itself will then give you the Section Page Numbers and the above field constructions will give the Document Page Numbers.

Thinking about it more, a simpler method would be to have the numbering in each Section continue from that in the previous section in which case the { Page } field will give the Document Page Number and to get the Section Page Number use the following field construction in each section starting with the second Section

{ = { PAGE } - { PAGEREF Sn } }

where n is the Section Number of the previous Section

i.e., in the Second Section 

{ = { PAGE } - { PAGEREF S1 } }

In the third

{ = { PAGE } - { PAGEREF S2 } }

etc.
