macropod say:

Hi maloneandbanks,

Embedded or linked Excel objects cannot span more than one page in a Word document. There is also a limit to how many rows (about 70, IIRC) an Excel object can display in a Word document. However, if you change the paste link format to, say HTML or formatted text, you'll get a field that can span multiple pages. The downside is that the formatting is likely to be different to what you get with the Excel object.

The alternative, is to use two (or more) links as Excel objects, each spanning no more than one page, so that you end up with the complete set of Balance Sheets and Income Statements.

marvin ma msft say:

Hi Odin99,

If you saw the message when you initiated a search in Outlook, it's related with the search methods of Outlook. There're two types of search in Outlook 2016 and later versions. One is Windows Desktop Search (WDS), the other is FAST search.

  • WDS

After you added an Exchange account to Outlook and enabled Cached Exchange Mode for this account, Outlook will download a copy of your mailbox data from Exchange server to local. Then indexes are created on the local disk. When you initiate a search, Outlook uses the same WDS architecture to search local indexes in the copy. This type of search is based on Windows Search service.

  • FAST search

If you're connected to Exchange server 2016 or Exchange online, when you initiate a search, Outlook 2016 and the later versions will directly submit a search query to Exchange server. The search results will be returned from server. However, this type of search only works when you select the following search scopes: Current Folder and Current Mailbox. To learn more details, you can refer to How Outlook 2016 utilizes Exchange Server 2016 FAST Search.

So you can see when you search by WDS, it will be influenced by Windows search service. And generally, if it's your personal computer, you can turn on Windows search service in your computer by the following steps.

1. Open"Services" app. You can search this app in Start menu.

2. Find "Windows Search" entry and double click it.

3. Navigate to Log On tab and tick "Allow service to interact with desktop".

However, based on your error message, it seems like an office computer in your organization. If your computer is domain-joined and managed by admins in your organization, admins can use Group Policy (a tool) to control some services for all domain-joined computers. In this scenario, you can try to contact the admins in your organization to feedback and modify the Group Policy to turn on Windows search service for the whole domain-joined computers.

1. Press Win + R keys to open "Run".

2. Type "gpedit.msc" to open Group Policy editor.

3. Go to Computer Configuration > Windows Settings > Security Settings > System Services to turn on the Windows search service.

e.j. gun say:

The code below should be a good start.  It relies on a function to find unique values (Division names) from your data, which can be downloaded from this link:

Chip Pearson's Distinct Values Function

Option Explicit

Sub SplitIntoTabsByDivision()
    Dim i As Long, j As Long
    Dim nRows As Long
    Dim masterWB As Workbook
    Dim masterWS As Worksheet
    Dim newWS As Worksheet, oldWS As Worksheet
'
    Dim inRange As Range
    Dim uniqueDivisions As Variant
'
    Set masterWB = ActiveWorkbook
    Set masterWS = masterWB.Worksheets("Master Sheet") ' change the name as required...
    masterWS.Activate
'
Application.ScreenUpdating = False
    nRows = masterWS.Range("A1").End(xlDown).Row - 1 ' Assume a single header row
    Set inRange = masterWS.Range("A2:A" & nRows + 1)
'
' Get a list of unique division names
'
    uniqueDivisions = DistinctValues(inRange, False)
'
' Create a worksheet for each division that does not already exist and populate it with data.
' If the sheet already exists, just update the data.
'
    Set oldWS = masterWS
    On Error Resume Next
    For i = 1 To UBound(uniqueDivisions)
        If (Not WorksheetExists(uniqueDivisions(i))) Then
            Set newWS = masterWB.Worksheets.Add(after:=oldWS)
            newWS.Name = uniqueDivisions(i)
            masterWS.Range("A1:G1").Copy Destination:=newWS.Range("A1:G1")
        End If
    Next i
    On Error GoTo 0
'
' Return to Master Sheet
'
    masterWS.Activate
'
' Update the data on each division sheet using data from Master Sheet
'
    If (Not masterWS.AutoFilterMode) Then
        Call TurnOnAutoFiltering(masterWS)
    End If
'
' For each unique division name, filter the Master Sheet data, copy the
' filtered data and paste into the division's worksheet.
'
    For i = 1 To UBound(uniqueDivisions)
        Set newWS = masterWB.Worksheets(uniqueDivisions(i))
        nRows = newWS.Range("A1").End(xlDown).Row
        If (nRows > 1) Then
            newWS.Range("A2:G" & nRows).Clear
        End If
        masterWS.Range("$A$1:$G$" & masterWS.Range("$A$1").End(xlDown).Row).AutoFilter Field:=1, Criteria1:="" & uniqueDivisions(i) & ""
        masterWS.Range("$A$2:$G" & masterWS.Range("$A$1").End(xlDown).Row).Select
        Selection.SpecialCells(xlCellTypeVisible).Select
        Selection.Copy
        newWS.Select
        newWS.Range("$A$2").Select
        Selection.PasteSpecial Paste:=xlPasteColumnWidths, Operation:=xlNone, _
            SkipBlanks:=False, Transpose:=False
        Selection.PasteSpecial Paste:=xlPasteFormats, Operation:=xlNone, _
            SkipBlanks:=True, Transpose:=False
        Selection.PasteSpecial Paste:=xlPasteValues, Operation:=xlNone, SkipBlanks _
            :=True, Transpose:=False
        masterWS.Activate
        masterWS.Range("$A$1:$G$" & masterWS.Range("$A$1").End(xlDown).Row).AutoFilter Field:=1
        masterWS.Range("A1").Select
        Application.CutCopyMode = False
    Next i
Application.ScreenUpdating = True
'
End Sub
'
' WorksheetExists Function
'
Public Function WorksheetExists(ByVal WorksheetName As String) As Boolean

 WorksheetExists = (Sheets(WorksheetName).Name <> "")
 On Error GoTo 0

'
' TurnOnAutoFiltering Sub
'
Sub TurnOnAutoFiltering(whichWS As Worksheet)
    whichWS.Rows("1:1").AutoFilter
End Sub
