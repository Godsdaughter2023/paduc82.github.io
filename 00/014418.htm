Sub Reassign_Names()
    Dim rw As Long, CellRange As String, WS As Worksheet, xlOldCalc As Long
    xlOldCalc = Application.Calculation
    Application.Calculation = xlManual
    Application.EnableEvents = False

    Do Until IsEmpty(WS.Cells(rw, 1))

        ActiveWorkbook.Names.Add Name:=WS.Cells(rw, 1), RefersTo:=CellRange
        rw = rw + 1
    Loop
    Application.EnableEvents = True
    Application.Calculation = xlOldCalc
End Sub

I contacted Microsoft Support about this and this is what I was told:

 

Unfortunately there is no way to avoid the InfoPath 2010 users getting that prompt to “Save and Update Form”.   Even with code it is not possible.
The users will only get the prompt the initial time to Save and Update Form as you already know, and then any subsequent openings will not prompt the users for any updates.  The only way around this option is to have the InfoPath 2010 users clear their InfoPath cache first, before accessing the forms.  This will delete the currently saved template information that is in their cache, so it will not prompt them to save and update the form since it doesn’t have any record of there being an open form.

The way to have users empty their InfoPath cache is by doing this:
Have them go to Start then either launch “Run” or in their “Start Search” bar (right above the Start button) have them type the following command:
   InfoPath  /cache clearall
(this will launch the InfoPath client on the user’s machine and they will now have empty InfoPath cache)

joeu2004 say: Stu wrote:

The formula in cell E23 is  =IF(ISBLANK(E32),D23,E23)  which fails with a circular reference error.  How should I set this up?


Perhaps:

=IF(ISBLANK(E32),D23,"")

However, I would suggest:

=IF(E32="",D23,"")

The point is:  ISBLANK is something of a misnomer.  It returns TRUE if E32 is truly empty:  no constant and no formula.  But suppose you wanted to write elsewhere IF(ISBLANK(E23),...).  ISBLANK would return FALSE even if the formula in E23 returned the null string ("").

IF(E32="",...) is TRUE if E32 is empty or appears empty because its value is the null string.

I wrote:
You wrote IF(ISBLANK(E32),...), but your description says "If E23 is blank".  If you meant to write IF(ISBLANK(E23),...), normally you cannot do that.  As noted, ISBLANK(E23) is always FALSE because there is a formula in E23.

What is it that you are trying to accomplish with a formula like that?  For example, do you want to preserve the
If so, you might consider the following event macro:

Private Sub Worksheet_Change(ByVal Target As Range)
If IsEmpty(Range("e23")) Then
    If Not Application.Intersect(Target, Range("d23")) Is Nothing Then
        Application.EnableEvents = False
        Range("e23") = Range("d23")
        Application.EnableEvents = True
    End If
End If
End Sub

Create the event macro as follows.  Right-click on the worksheet tab; click on View Code.  That opens the VBA editing pane.  Copy-and-paste the VBA code above into the VBA editing pane.  You can now close the VBA window.  Note:  When you save the Excel file, be sure to save it as macro-enabled (*.xlsm").

Caveat:  If the location of E23 and/or D23 change due to inserting rows or columns, you would need to change the macro.  It would be better to name those cells, e.g. Ecell and Dcell, then substituted those names in the macro, e.g. Range("Ecell") and Range("Dcell").
