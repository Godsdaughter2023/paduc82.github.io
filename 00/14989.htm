How to autofill column with IF condition with VBA code

Hi, I would like to know how to autofill the entire column with this IF statement with VBA code :

=(IF($Q2="NETWORK",1,IF($Q2="ADJACENT",0.75,IF($Q2="LOCAL",0.25,0))))

The reason why I prefer the VBA code is because I've tried to autofill by pressing shirt + enter + arrow down + ctrl D but it makes the entire file very large and it lags a lot.

Thank you!

hansv mvp say:

In the macro below, I have assumed that you want to start at row 2, and that the column you want to fill is column R. You can adjust the constants at the beginning of the macro to suit your wishes.

Sub FillData()
    Const FirstRow = 2
    Const SourceCol = "Q"
    Const TargetCol = "R"
    Dim CurRow As Long
    Dim LastRow As Long
    Application.ScreenUpdating = False
    LastRow = Range(SourceCol & Rows.Count).End(xlUp).Row
    Range(Cells(FirstRow, TargetCol), Cells(LastRow, TargetCol)).Value = 0
    For CurRow = FirstRow To LastRow
        Select Case Cells(CurRow, SourceCol).Value
            Case "NETWORK"
                Cells(CurRow, TargetCol).Value = 1
            Case "ADJACENT"
                Cells(CurRow, TargetCol).Value = 0.75
            Case "LOCAL"
                Cells(CurRow, TargetCol).Value = 0.25
        End Select
    Next CurRow
    Application.ScreenUpdating = True
End Sub

hansv mvp say:

You complained that the formulas caused the workbook to become slow.

The macro fills column R with the result of the formulas instead of with the formulas, so that the cells don't have to be recalculated all the time. Also, it doesn't fill the entire column (1048576 rows) but only the part corresponding to the used part of column Q.

Hi Alan,

 

Yes. You can use one Azure AD Connect with multiple forests connected through Azure AD Connect without a trust.

 

For example: In domain A, assume you have a user user1@A.com, and in domain B you have a user user2@B.com. These users are specific to their domain and do not have copies in other domains. Therefore, when configuring AADSync, we enter and add forest and enterprise administrator credentials in the forest, and on the other hand (next step), we provide Azure AD credentials (tenant GA). Word 2013 - Is there a way to replace tracked changes with different - Microsoft Community

jgreg7 say:

Word 2013 - Is there a way to replace tracked changes with different font

I have a document with many tracked changes, and wish to replace (e.g. accept the change) but have the change show up in a different font so that it can still be distinguished from the earlier version.

This can be a different font, color, type style, etc. just so that it is not the same as the original text.

Is there any way to do this?

doug robbins - mvp office apps & services (word) say:

Run a macro containing the following code:

Dim arev As Revision
For Each arev In ActiveDocument.Revisions
    With arev
        If .Type = wdRevisionDelete Then
            With .range.Font
                .ColorIndex = wdRed
                .StrikeThrough = True
            End With
            .Reject
        Else
            .range.Font.ColorIndex = wdDarkBlue
            .Accept
        End If
    End With
Next

Note the deleted text is retained in the document, but colored red and struck through.  Otherwise it would be very had to tell if something had been deleted.

Revision Tracking should of course be turned off before running the macro.
