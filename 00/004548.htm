jllatham say:

Until we have enough information by way of examples to help with a function using regular worksheet functions, I'll offer what is known as a UDF (User Defined Function) that may work for you.  I call this QuasiVLookup because it works much like VLOOKUP in that it will stop at the first apparent match in the list in column F and return the value from column N related to that apparent match.

You will generally be better off with a "native worksheet functions" using formula because they are generally faster than equivalent VBA code and you won't be forced to save the workbook as a .xlsm workbook just to keep this function available.  But...

To put it to work, you have to put it into a regular code module in the workbook.  To do that: open the workbook, press [Alt]+[F11] to open the VB Editor (VBE) and in the VBE use Insert --> Module.  Copy the code below and paste it into the blank code module presented to you.  If the phrase Option Explicit shows up in the code module, paste the code below that statement.  Close the VBE by clicking its red-x or pressing [Alt]+[F11] again to return to your worksheet. You will have to save the workbook as .xlsm type to keep the code in it.

Using the UDF:  The UDF is now available for use just like a built-in worksheet function and what you do is put a formula like this where you want to return the value of column N on the sheet:

=quasivlookup(A2)

Just that simple.  Now, here's the code:

Public Function QuasiVlookup(searchFor As Variant) As Variant
  Dim dataWS As Worksheet
  Dim searchList As Range
  Dim anySearchItem As Range
  Dim foundItem As Range

  If searchFor = "" Then
    QuasiVlookup = ""
    Exit Function
  End If
  Set dataWS = ThisWorkbook.Worksheets("Data") ' change to exact name
  Set searchList = dataWS.Range("F1:" & _
   dataWS.Range("F" & Rows.Count).End(xlUp).Address)
  searchFor = CStr(searchFor)
  For Each anySearchItem In searchList
    'stops at first apparent match!
    'as written, is not case sensitive
    'to make it case sensitive, change vbTextCompare to vbBinaryCompare
    If InStr(1, anySearchItem, searchFor, vbTextCompare) > 0 Then
      QuasiVlookup = dataWS.Range("N" & anySearchItem.Row)
      Set searchList = Nothing
      Set dataWS = Nothing
      Exit Function
    End If
  Next
  Set searchList = Nothing
  Set dataWS = Nothing
  QuasiVlookup = "No Match"
End Function

Move a macro - Microsoft Community

robertlobo say:

Move a macro

 How do you macros from one workbook to another?

chip pearson say: Open the VBA editor and then open the Project Window (CTRL R) if it is not already visible, typically on the left side of the screen. Open the "modules" folder of the source workbook and find the module containing the procedure in question. Then, in the destination project, insert a new module (Edit menu, Insert, Module) and then copy and paste the procedure from the source project to the destination project. If you want to move the entire module and all the procedures it contains, you can drag and drop the module from the source project to the destination project. The is no (practical) limit on the number of procedures contained within a single module, so you can copy multiple procedures from the source project to the same module in the destination project.

gord dibben say:
