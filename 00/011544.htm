Hi,

 

You can use this array formula to ignore zeroes. See below on how to enter it

 

=MEDIAN(IF(J4:J101<>0,J4:J101))

 

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

townfield say:

Phil,

Try this...

=LARGE(j4:j101,COUNTIF(j4:j101,">0")/2)

You may need to refine this if the number of non-zero values is odd.

From Excel help "LARGE returns the k-th largest value in a data set. You can use this function to select a value based on its relative standing. For example, you can use LARGE to return the highest, runner-up, or third-place score."

The above formula counts the number of non-zero values, halves it and asks for that relative value standing in the data set.

karl timmermans (outlook mvp 2012-2018) say:
What type (POP, Imap, Exchange) of email account do your contacts belong to?

If Imap, (in a contact folder labeled as "This Computer Only") categories are not applicable. Also, if Imap, as an aside, having contacts in a "This Computer Only" folder is not recommended, the reason being that if you ever delete the Imap account from the profie, the associated OST file for the account would also be deleted and all data in "This Computer Only" folders will be lost. Similarly, if Outlook cannot find the .ost file, it will re-create one, re-sync the email folders but the contact/calendar "This Computer Only" folders will be empty.

Again, assuming this involves an IMAP account, the way to get categories to work and mitigate the potential loss of contact data would be as follows

#1 - Create a new PST file
(Select <New Items> in Outlook ribbon --> <More Items> --> Outlook Data File)
#2 - From Outlook menu --> File --> Account Settings --> Data Files tab
- set the newly created PST file as the default data file which will then contain the default contact/calendar data files for the profile. This assumes that the .ost file for the IMAP Account was originally set as the default for the profile
#3 - Returning to the Outlook Nav Pane, click on the contact folder in the Imap folder tree
#4 - Press CTRL+A to highlight all the contacts in the folder
#5 - Right-click on highlighted contacts --> Move --> Copy to Folder --> select the Contact folder in the newly created PST file. The reason for selecting <Copy to folder> instead of <Other Folder> is safeguard against anything going wrong during the Move/Copy process. The <Other Folder> option will delete the contacts from the source folder whereas the <Copy to Folder> will do exactly that - make a copy leaving the original contact items. Once you've verified that the items have been copied completely/correctly, you can then delete the items from the source folder

The contact's Category option will now be available in the PST based contact folder.
