kosh vorlon say:

Hi Michal,

 

You can never be 100% certain no corruption will occur no matter what you're doing - even if the computer is only on and idle.  But there's no specific problem running defragmentation while MSE is running real-time protection (but don't try to run a scan at the same time).  I atually defragment in the background while otherwise working as normal - but it is better if you close as much as possible first so it can do a better job.  Remember to do a Disk Cleanup before defragmenting so it doesn't include files that you will soon be deleting and thus create earlier opportunities for fragmentation to start building again.

 

I hope this helps.

 

Good luck!

jeeped say:
... Name Conflict. Name cannot be the same as a Built in name ...

With the expanded number of columns in Excel 2007/2010, you can make the mistake of attempting to name a cell or range of cells to an actual cell address. Let's say you wanted to name a couple of ranges of data as Win1, Win2 and Win3. You cannot do this because WIN1, WIN2 and WIN3 are actual cell addresses in the expanded worksheet (WIN is the 15,796th column in a worksheet, rows 1, 2 and 3). Solution: select a name that is not an actual cell address. Excel 2007/2010 cell addresses extend from A1 to XFD1048576. In the scenario described above, one option would be WinA, WinB and WinC.

martin crosbie say:

It's not the phone's fault, it's the operators. not everyone uses the same time. Some operators transmit the correct time signal, some don't use it at all.

This facility is more about updating the phone time for daylight savings changes in your home country/network (where that network supports it), rather than changing your phones time to reflect time zone changes, since it changes the time, not the time zone.

the best way to change the time when travelling is to manually update the time zone, not changing the time on the clock.
