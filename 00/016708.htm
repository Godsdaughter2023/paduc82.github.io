jay freedman say:

That's just the way legacy forms ("Filling in forms") protection works. If any section of the document is protected, many functions are disabled, even in the unprotected sections.

There are two alternatives:

  • If you aren't using legacy form fields -- if the protection is only to prevent changes in section 1 -- then open t he template for editing, reopen the Restrict Editing pane, stop protection, and change the dropdown to "No changes (Read only)". Now select everything in section 2, and check the Everyone box in the Exceptions section of the pane. Restart the protection.
  • If you do have legacy form fields in section 1, replace them with content controls. Then select all of section 1, and click Developer > Group > Group. That will prevent changes to anything in section 1 that isn't inside a content control.

If your co-worker doesn't have Myriad Pro installed, then, as Stefan says, another font is being substituted for display (you can check on this at Tools | Options | Compatibility: Font Substitution). If she adds text in the middle of a block of Myriad Pro, it should still have that formatting when it comes back to you.

This will be true unless you embedded the font. There are various levels of permission for font embedding. Installable embedding is no longer honored by Word but causes the font to be installed automatically on the user's system. Editable embedding would allow your co-worker to use the font in the document. Print & Preview embedding would allow her to read and print the document but not edit it. Fonts with a Restricted license cannot be embedded. The embedding license varies from one font to another and can't be changed by the user.

Hi eryan,

Emptying the Deleted items folder is very slow, especially if you have a lot of mail to delete.

1. Empty the recovery folder linked at the top of the Deleted items folder. (The recovery folder has a different, smaller quota, but emptying it can help speed up the processes.)

brian tillman [outlook mvp 2007-2019] say:
No one using the exact same domain as you are can have the same username you do.  As long as the domain is different, however, the accounts are different.  For example, someone @domain.com and someone @domain.org are completely separate addresses even if "someone" in each is identical.  So, even if this person in Canada has exactly the same name as you do, the person sending the message is using the wrong address and reaching you instead of the person expected because they're using something like ".com" instead of ".ca" on the domain.  Perhaps the person even gave out the wrong address (which happens to be yours) because they used the wrong domain.
