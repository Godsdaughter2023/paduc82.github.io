david (nobby) barnes say:

Hi Old Schoolie,

 

I know this is a tad old, but thought I'd add my thoughts to also aid any searchers/visitors in the future.

 

Contacts are derived from 2 places:

  1.  the nick-name list which 'remembers' e-mail addresses you type in, even incorrect ones and then tries to present you with a prompt list as you start to type in someones email.
    Outlook versions prior to 2010 used a ".NK2" file to store these in.
      The NK2 data is NOT stored in the Outlook PST file and often gets missed on backups.
    Outlook versions 2010+ use a stream file inside the data file or mailbox to store this data.
      In theory this should then move with the user as they move their mail file (PST) about.
    - you can get editors to 'import' these lost contacts into your true 'contacts' folder
    See http://www.nirsoft.net/utils/outlook_nk2_edit.html for an editor/importer
    See http://www.slipstick.com/outlook/suggested-contacts-and-auto-complete-lists/ for more info.
  2. The outlook "Contacts" folder.
    To access the list of contacts in your contact folder, either
     - Type some of the persons name or email address in the 'to' box and press <CTRL>+K
     - click on the 'TO' button and select the contact.

The point is, the 'nick-name' list is dangerous, not stored in outlook, prone to getting lost/corrupted and remembers incorrect addresses even if you have the correct address in your contacts folder. Almost all Outlook 'usage' issues we get called about are to do with the nick-name list being wrong, lost or other such problem.

 

Your 'contacts' that you are referring to are stored on your computer in the 'nick-name' file (not in outlook) and you need to use a tool to 'import' these to your contacts folder.
