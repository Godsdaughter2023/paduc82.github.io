Macro for new month on new worksheet. - Microsoft Community

bubba070 say:

Macro for new month on new worksheet.

Can someone help me with a macro please, I am new to macros so please be patient with me. On the first of every month I would like to start a new worksheet with the same template from worksheet one but with a new month in column A. I would also like it to rename the worksheet to the month and year of that particular sheet. On Sheet 1 October 1st is in A3 and goes to the end of the month.


 October
Oct 1, 2012    
Oct 2, 2012    
Oct 3, 2012    
Oct 4, 2012    
Oct 5, 2012    

hansv mvp say:

See if you can adapt the following macro for your purposes:

 

Sub AddMonth()
    Dim wsh As Worksheet
    Dim dtm As Date
    Dim strName As String
    Dim lngDay As Long
    Dim lngDays As Long
    ' First of current month
    dtm = Date - Day(Date) + 1
    ' Number of days in current month
    lngDays = Day(DateSerial(Year(dtm), Month(dtm) + 1, 0))
    ' Name of worksheet
    strName = Format(dtm, "mmmm yyyy")
    ' Check whether worksheet exists
    On Error Resume Next
    Set wsh = Worksheets(strName)
    On Error GoTo 0
    If Not wsh Is Nothing Then
        MsgBox "The worksheet for the current month " & _
            "has already been created!", vbInformation
        Exit Sub
    End If
    ' Create worksheet
    Set wsh = Worksheets.Add(After:=Worksheets(Worksheets.Count))
    wsh.Name = strName
    ' Fill days
    wsh.Range("A3:A" & (lngDays + 2)).NumberFormat = "m/d/yyyy"
    For lngDay = 1 To lngDays
        wsh.Range("A" & (lngDay + 2)) = dtm + lngDay - 1
    Next lngDay
    wsh.Range("A1").EntireColumn.AutoFit
End Sub

dianeoforegon say: It could be a bad preference file or a problem with your Identity.

Try deleting the com.microsoft.entourage.plist file in your User's Library/Preferences folder.

/Users/YOU/Library/Preferences/com.microsoft.Entourage.plist

If this fails to fix the issue, let's next test in a new User. Not only will we be testing a new blank Identity but it will eliminate any other corrupt preference file or conflict in your User's folder.

CREATE A NEW USER

Go to System Preferences --> Create a New User in Accounts. Switch to the New User by logging out/in or use Fast User Switching.

Enter your account info. If POP account set to "leave on server"  for this test.

Do you still see the issue?

    If yes, then the problem is with your base files.
    If no, then the problem is in  your User's folder.
 
If this works, we next need to eliminate your Identity. In the new User, drag the newly created Test Identity to Shared folder.

Your Hard Drive/Users/Shared

Log back into your main User. Drag from Shared (it actually copies) your Test Identity and place it in the Microsoft User Data folder > Office 2008 Identities folder.

 ~/Documents/Microsoft User Data/Office 2008 Identities/your identity 

Use "Switch Identity" to switch to the Test Identity. Does the problem go away in the new Identity in your regular User account? If yes, you'll need to export all your data and import into the new Identity.

How to manually move your data. (when import fails and/or you need to move to a new Identity same version or revert to an older version)

If the new Identity fails in your main User's folder it's corruption or conflict with a file in your User's folder. This can be difficult to track down if it's not a usual suspect.

Let me know what your testing shows.
