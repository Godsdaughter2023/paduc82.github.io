If you mean online videos played through the browser, then the reason they stop has to do with the speed of the internet connection you are using and how fast your computer is buffering the videos. Basically, the video is being played faster then the data is being downloaded and stored, so it freezes while the downloading catches up. On Youtube and similar sites, you may be able get around this by pausing the video for a few minutes at the begining, to give it a head start on downloading. 1. You can't just copy/paste that program's folder to another computer and expect it to work.

When a program is installed it not only resides in the programs folder but is spread throughout the operating system in the Registry, in file associations with other programs so that it can work with them, etc., etc..

_________________________________________

If you move the hard drive to another computer the chances are that it won't Boot due to the fact that the installed XP operating system on the XP hard drive also will have the drivers installed for the Motherboard in the XP computer.

If XP was preinstalled (Dell, HP, etc.) the license was OEM and can't be transferred.

You reinstall programs on a new computer from the original installation media

a. user say:

Hi,

This issue might occur due to bad sectors on the drive that couldn’t be repaired. Also, the disk might be in use when the toll starts checking for errors.

I would suggest you to schedule a check disk on the drive and then check if scanning the drive on reboot fixes the issue. Follow the methods:

Method 1:

Follow the steps to enable check disk to run on startup (after setting the disrty bit on the drive):

  • Click on Start menu and type “cmd”. Right Click on Command Prompt and select Run as administrator.
  • Type the following command and hit Enter:

chkdsk c: /r

  • Type “Y” to confirm and then “Exit” to close Command Prompt.
  • Restart the computer and check if it resolves the issue. This should take you through 5 stages of the chkdsk scan and will unset that dirty bit. This can take an hour or more depending on the size of your hard drive.
  • Once booted back to Windows, open another Command Prompt and query the dirty bit again by entering fsutil dirty query c: and Windows will confirm that the dirty bit is not set on that drive. If it shows that the dirty bit is set, type “chkntfs /x c:” without quotes. The X tells Windows to NOT check that particular drive on the next reboot.

Important:  Data available on bad sectors might be lost when chkdsk tries to repair your hard disc.

If the issue still persists, proceed to the next method.

Method 2:

I would suggest you to run check disk from Windows Recovery Environment using the System Recovery options. Refer to the section “To open the System Recovery Options menu using the Windows 7 installation disc or a USB flash drive, or a system repair disc” from the link given below to do so:

http://windows.microsoft.com/en-in/windows/what-are-system-recovery-options#what-are-system-recovery-options=windows-7  

When you reach system recovery options, choose Command Prompt and run the command:

“chkdsk  /F /R”

Important:  Data available on bad sectors might be lost when chkdsk tries to repair your hard disc.
