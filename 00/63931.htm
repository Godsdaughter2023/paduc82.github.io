      The prohibition is not because the programmers don't want you to see what behind, but to make sure you can't change anything until you dismiss the dialog. This is usually because it would "break" the consistency of the data the application uses if you could change things while the dialog is open.

      For example, if you are changing a password and deleted the account while the change dialog was open there is no consistent way to apply the changed password when you close the dialog. You are effectively saying "change the password on the account that doesn't exist". That's not possible so the programmers make impossible to even attempt that action by locking you into the dialog.
