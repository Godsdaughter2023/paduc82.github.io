john w. vinson say:

Don't confuse datatypes with formats: they are different things. A field's Datatype can be Number, Text, Date/Time etc.; this controls how the value is stored on the disk (i.e. a Long Integer number is stored as a 32-bit bitstring). The Datatype is indeed critical for compatibility between Access tables and SQL/Server tables (what you need on line).

The format of a field is much less intrusive; it doesn't affect what's stored, only how that value is displayed. For example, you can have a Date/Time field stored, and can choose to display it with lots of different formats - 06/26/2014, 2014-06-26, June 26, 2014 10:24:13 AM for example. You can display the same field in a dozen different formats on different forms, reports or datasheets; it's just cosmetic and shouldn't affect compatibility.

So... what is the actual Datatype of the field that you want to have leading zeros? What is the purpose of the field? If it's some sort of identifier (like an Employee Number or a Part Number) then even though it consists of just digits, you should use a Text datatype rather than any sort of Number. More of concern is the date field - did you perhaps originally set up the table with a Text field in which you're storing dates? That would require a change of Datatype, not just Format.

More info please!

scottgem say:

Yes, this can greatly affect relationships. Both sides of a relationship MUST be the same datatype. So if you change a field from a number to text, then you break the relationships with all foreign keys. As John said don't confuse formatting with data types. 

Also, numerically datatypes perform better in joins, so key fields should be a number datatype, especially with an online database.
