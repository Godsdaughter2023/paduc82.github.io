john mcghie say:
You're another victim of an advanced Word feature called "Automatically screw up my document" :-)

The formal name is "AutoCorrect", and you will find it in Word>Preferences>Autocorrect.  Under "Autoformat as you type" I would turn off as much as you can.  At least turn off the top and bottom sections in there, so you are back in control.  If you want to turn off everything until you get the hang of Word, that's a good move.

 What has happened is that Word has correctly deduced that you wanted a "Line" so it has converted what you typed into a "Border".  A border is not a physical thing you can select, it's a "property": in this case, of a paragraph.  Which you can't see, because by default Word hides the paragraph marks.

In your case, the pain gets worse: because you did this in the Running Header of the document (I bet you didn't even know you had one of those...) Word is now replicating it on every page for you.  That's what "Headers" are for, but only when you want to!

OK, to get back from there:

1)  Double-click the "line".  A new view will open up, allowing you to edit the headers and footers.

2)  Click in the paragraph above the line.  To see the paragraph marks, click the "Show all non-printing characters" button on the Standard toolbar.  It looks like this: ¶

3)  Go to Format>Borders and Shading and click "None".  If None is already selected, your cursor is in the wrong paragraph: move it, and try again.

Next time you want a line, you know how to get one: but this time, use Format>Borders and Shading yourself, so you remain in control of the result :-)

And learn about "paragraphs".  Everything in Word starts with or is built from a paragraph.  They do moe that contain text: they also contain almost all of the formatting.

Hope this helps

LastOrientation = -1
With ActiveDocument
        Set ThisP = .Paragraphs(i + j)
        With ThisP.Range.PageSetup
          If .Orientation <> LastOrientation Then
            MaxTabPos = .PageWidth - .LeftMargin - .RightMargin - .Gutter - ThisP.RightIndent
            ThisP.Range.InsertBefore String(255, "X") & vbCr
            Set Temp = ThisP.Previous.Range
            Temp.Collapse wdCollapseStart
            F = Temp.Information(wdFirstCharacterLineNumber)
            t = Temp.Start
            Do
              Temp.Move wdCharacter, 1
            Loop Until Temp.Information(wdFirstCharacterLineNumber) <> F
            MaxLineLen = Temp.End - t
            ThisP.Previous.Range.Delete
            LastOrientation = .Orientation
          End If
        End With
End With

diane poremsky mvp (slipstick.com) say:

You need to use mfcmapi to edit the properties of the folders.

To fix:

Get MFCMAPI . If you have 64 bit office, you need 64 bit MFCMAPI.

  • Open MFCMAPI, go to Session, Logon
  • Select your profile
  • Double-click on the affected data store.
  • Expand Root - Mailbox
  • Expand Top of Information Store or IPM_Subtree
  • Select Inbox
  • Look for PR_CONTAINER_CLASS in the right pane, double-click on it.
  • Change IPF.IMAP to IPF.Note then click OK to close the dialog.

  • You'll need to do this to each folder that is affected.
