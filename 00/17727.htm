willy denoyette say:

But there were files in the download folder right?

This is how UUP seems to work for now.

The process starts with some file downloads In SoftwareDistribution , no large files, the largest is ~1MB, the others are smaller,  After that the Delivery Optimization service (dosvc) kicks in and starts building the DeliveryOptimization folder subfolders tree.

When whole tree is filled with data files coming from the cab file downloads (+600 small files). When the whole tree is built, the bulk of the build (containing the ESD files and others +- 3.7GB) should get downloaded.

This is where the process fails most of the time, the dosvc keeps consuming + 10%  CPU cycles without any other activity, no network no disk IO, the bulk of the download never arrives. And that's the time to stop this processing; or by killing the dosvc service host or by stopping the WU service followed by a reboot. A cold reboot might help as well.

After the reboot and a restart of the update, the process continues where it has left and the bulk download start & succeeds.  If the bulk download doesn't start within a few minutes, you are possibly stuck again and you need to repeat the above killing things.

gsoto say:

I think I got the progress bar to work in  my Insider build 15019, I did a disk cleanup including system files, and the bar seems to be moving again. Before that, I stopped the Windows Update service, restarted, and rebooted Windows. Hopefully that helps somebody!

My solution

Open Task Manager
Choose More Details
Stay or go to the Processes Tab
Under Windows Processes look for: Service Hosts: Delivery Optimization
Right Click it and choose End Task
Select Abandon Unsaved Data and Shutdown
The Shut Down button will be now active Click it

If you go to Windows Update you will see and error, you will have to Click Retry.
Maximum Two times I have to click Retry and the updates will begin to download almost immediately.
