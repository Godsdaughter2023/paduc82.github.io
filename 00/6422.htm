How do I stop an empty field leaving a space when mail merging? - Microsoft Community

charliebyrne say:

How do I stop an empty field leaving a space when mail merging?

I need to merge from an excel that has street number and street name as different fields (they are pulled from our database like this) but not all have numbers so I am left with a space in the address. I need to enter the fields <streetnumber> and <streetname> with a space between them so that if there is a number it doesn't look like 9Hillpark as opposed to 9 Hillpark?


suzanne s. barnhill say:

The usual approach is to use an IF field on this order:

     { IF { MERGEFIELD streetnumber } <> "" "{ MERGEFIELD streetnumber } " }

This says that if the «streetnumber» field is not blank, Word will insert the contents of the field with a following space. If the field is blank, nothing will be inserted, and there will be no space.

Word makes it unnecessarily difficult to insert IF fields containing merge fields. Indeed, it does not appear that Word 2007 and 2010 even has the "Insert Word Field" feature (which allows you to select an IF field) as part of the Mailings tab any more. So you need to do this "by hand."

  1. Press Ctrl+F9 to insert a field. It will be displayed as two curved brackets with two spaces between, the insertion point between the spaces.
  2. Type IF and then use the Insert Merge Field button on the Mailings tab to insert the appropriate merge field.
  3. Type <> "" " and then insert the merge field again, followed by a space and another quote.
  4. F9 to update the field, if desired.

    Location:    C:\Documents and Settings\<username>\Local Settings\Application Data\Microsoft\Outlook

    2.       Frmcache.dat

    Location:  C:\Documents and Settings\<username>\Application Data\Microsoft\Outlook

    3.        Extend.dat

    Location: C:\Documents and Settings\<username>\Application         Data\Microsoft\Outlook

    Note: If you are not able to find the files use search option as follow.

     

    Search for the files (Win XP)

     

    On file name type the name of the file to be searched.

     

    1.       Click Start, and then click Search.

    2.       In the Search Companion dialog box, click All files and folders.

    3.       Type <File name to be searched> in all or part of the file name box.

    4.       In the Look in box, click local hard drives C

    5.       Click more advanced options

     

                                                                               i.            For type of File name: Choose All files and folders

                                                                              ii.            Check the boxes for search system folders search hidden files and folders and search subfolders.

    Click on search.

    Removing addins

     

    1.      Click on Start and then in the Run box type “regedit” (without quotation marks) click on okay.

    2.      Now in the registry, take a backup by clicking on File > Export > (under the Export Range select ALL) and then click on Save.

    3.      Now please delete the following keys (Ignore if you don’t find or if you are not able to delete).

     

    HKEY_CURRENT_USER \Software\Microsoft\Office\Outlook\Addins (right click on addins and click Delete to remove addins)

    HKEY_LOCAL_MACHINE \Software\Microsoft\Office\Outlook\Addins (remove addins)

    HKEY_LOCAL_MACHINE \Software\Microsoft\Exchange\Client\Extensions (remove Extensions)

    Restart and try working in outlook.
