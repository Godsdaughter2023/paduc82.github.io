peanuts4me say:

Excel 2008 keeps reformatting large numbers into scientific notation

Original Title: "reformatting my numbers"
I am trying to enter large numbers into a field, and excel keeps reformatting them into scientific notation.  This becomes problematic when they are things like UPC codes, or item tracking numbers.  I've tried formatting the cells as text, and using the number field, but it keeps giving me error messages, and then if someone clicks on the error, it reformats to scientific notation.
j.e. mcgimpsey say: XL stores all numbers as double precision floating point values, which means there's about 15 digits of precision.

You can format the cells with Format/Cells/Number/Custom 000000000000000 and you'll see 15 digits, including leading zeros if the number is less than 15 digits long.

Alternatively, use ##############0 to get up to 15 digits.

If your numbers are greater than 15 digits long, you're out of luck treating them as numbers.

Of course, UPC codes and item tracking numbers aren't truly numbers - you don't do math on them - they're simply unique x-character strings, which are treated more appropriately as text. 

As you've noted, you can preformat the cells as text and everything works fine, but it's hard to fix stupid with your users unless you protect the worksheet from making changes to the format. If you've only got a few users, you might try to educate them...

Dim isect as Range

ken sheridan say: As the underlying column is a Boolean (Yes/No) data type it can only have one of two values , TRUE or FALSE, so when the user updates it the previous value will be NOT the new value, so:

Private Sub Box_AfterUpdate()

    Dim ctrl As control

    Set ctrl = Me.ActiveControl
    globalVar = Not ctrl

End Sub
Ken Sheridan, Stafford, England

And as far as I know unbound controls don't have old values.

You could simulate the old value of an unbound control storing the value of the control in a variable when the control gets the focus.  The problem is that if you change the value, exit the control, and then set the focus to the control again, the variable will get updated.
