You can save the document as a template and then delete all or most of the text. The styles will remain. Customizations you have made to the QAT are global and should be retained for all documents. suzanne s. barnhill say:

You misunderstood me. I said to save the document as a template and then delete all or most of the text from the template, with the understanding that you then resave the template without the text; you don't want to delete the text before creating the template, as you would then be dealing with your original document, which is risky. You then create new documents based on the template; even though you have deleted the text, the styles remain.

Note that when I say "template," I mean a .dotx (or .dotm), saved in Word's Templates folder. When you save the document, you select "Word Template (*.dotx)" or "Word Macro-Enabled Template (*.dotm)" as the file type.

boblarson say:

Is there any reason why you have to look at the client ID and not just the client name?  I would just use a combo box bound to the ID field in the table and have as its row source a query which pullls the client ID and client name from the client table and then you have the first column (the ID) hidden and the name showing.  But if you need to have the ID showing as well as the name you can still have the combo box which is bound to the field and still have the ID column be the bound column and then you can display both by setting the column widths to 1";1" or whatever works for you and then you can have a text box on the form (Unbound) and set the control source to

 

=[ComboBoxNameHere].[Column](1)

 

where 1 is the second column (because it is zero based) and then whenever you update the combo, the text box will follow.  And it is better than a DLookup.
