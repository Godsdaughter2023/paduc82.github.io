ken sheridan say: Another thing you need to be aware of is that this expression will only work when initially inserting a new record.  Otherwise it will fail as it will detect itself as a duplicate.   If, for instance, a user begins to enter a new record, and saves it before inserting the CustomerID, which is easily done inadvertently as all it needs is a stray click on the record selector, the current record will be detected as a 'duplicate' of itself when the code executes.  The same would apply when editing an existing record of course.

The way to avoid this is to bring the primary key into play:

Dim strCriteria As String

strCriteria = "CustomerID = """ & Me,.CustomerID and """ " & _
    "And YourPrimaryKey <> " & Me.YourPrimaryKey

If Not IsNull(DLookup("CustomerID","YourTable",strCriteria)) Then
     MsgBox "Your message goes here",vbInformation, "Warning"
End If

This assumes the key is a number data type.  If it's text use:

    "And YourPrimaryKey <> """ & Me.YourPrimaryKey & """"

1l year law student say:

Customize Quick access toolbar to include legal symbols

How do I insert symbols into my custom quick access toolbar vs always having to use the insert symbol on the quick access toolbar?

In particular, would like to enter the legal symbols for plaintiff (phi), defendant (delta), section, etc..with just one click or code.

omer atay say:

Here are macros you can use to insert Phi and Delta:

 

<?xml version="1.0" encoding="utf-16"?>
<Macro name="Φ" category="Auto text" description="Insert Φ (Phi)">
  <For first="Text">
    <That hasProp="selected" op="eq" value="true" />
    <ModifyProp name="value" op="set" value="Φ " />
  </For>
  <For each="Text">
    <That hasProp="selected" op="eq" value="true" />
    <That hasProp="value" op="eq" value="Φ" />
    <ModifyProp name="selected" op="set" value="false" />
  </For>
</Macro>

 

 
<?xml version="1.0" encoding="utf-16"?>
<Macro name="Δ" category="Auto text" description="Insert Δ (Delta)">
  <For first="Text">
    <That hasProp="selected" op="eq" value="true" />
    <ModifyProp name="value" op="set" value="Δ " />
  </For>
  <For each="Text">
    <That hasProp="selected" op="eq" value="true" />
    <That hasProp="value" op="eq" value="Δ" />
    <ModifyProp name="selected" op="set" value="false" />
  </For>
</Macro>

 

Here is how to import them:

  1. Install Onetastic
  2. Click on New Macro on the Home tab
  3. Click Import
  4. Paste the macro XML above (one of them)
  5. Click OK
  6. Click Save
  7. Close Macro Editor

Now you will have an Auto Text drop down on the Macro group in the Home tab which will have a single button that will insert Phi (or Delta whichever you imported). You can try it and check if it does the right thing. Now repeat to add the second one there too.

 

The buttons under drop down menus are not available to add to QAT (you can only add the whole dropdown to QAT). To add these buttons separately to QAT, you need to first move the macros to their separate tab by clicking on Settings > Show in a Separate Tab. When you do that ribbon will switch to Macros tab and you will have all the macros there. The newly added macros will show up under All Other Macros, and they are still not available for QAT. The last thing to do here is to restart OneNote, so that these new macros will move to their own group (Auto Text). (Sorry for stupid complications here, this doesn't happen without a restart due to some technical detail).

 

Once you restart and go to Macros tab, you should see the Δ and Φ buttons at the begining. You can right click to either of them and choose "Add to Quick Access Toolbar".

 

Let me know if you have any trouble and need any help.

graham mandeno say: When you open a Recordset or Execute a SQL statement from VBA, you are bypassing the Access user interface and going directly to the database engine.  The engine knows nothing about the Access forms that are open on the screen, so you can't include anything like this:

...WHERE [WO Details].[Customer Name]=[Forms]![frmCustParam]![cboCustParam] AND ...

Instead, break the string and use concatenation operators (&) to insert the VALUE into the string:

... WHERE [WO Details].[Customer Name]='" & [Forms]![frmCustParam]![cboCustParam] & "' AND ...

Note that since (I assume) [Customer Name] is a text field, you need to add extra single quotes on either side.  Does your Customers table have a numeric primary key?  If so they you would be better to use that.

Best wishes,
Graham

scottgem say:
As Graham said, that's why that error. You need to concatenate in the control reference. One of the reasons to write the SQL to a string variable is that you can then use Debug.Print to display the results in the Immediate window. And even copy and paste into SQL View of Query Design mode to debug.
