brian tillman [outlook mvp 2007-2019] say:

In Outlook, recurring events are single entries, not multiple ones.  The recurrences are calculated using the initial event and the recurrence pattern.  Outlook shows you the generated recurrences in any day/week/month view, but if you use a table view like Events, you'll see that it is only a single entry.


You can certainly delete individual occurrences or make some modifications to individual occurrences without modifying the entire series (i.e., the original event), but changes that apply from a particular point onward are modifications to the original event and Outlook will regenerate the entire series for that type of modification.


One way to deal with this is to create a new folder containing calendar items, display your main calendar in a table view, copy the event to that new caledar folder, then export that event from that folder.  Since you cannot export open-ended events (or the exported file would be infinitely large), Outlook will ask you for the time range of the export.  Specify a beginning time of when the event started and specify for an ending time the last date before you want to change the event.  Export it as a CSV (comma-separated values) file.  This file will contain all of the occurrences of the event up to the change point as individual entries, including any exceptions or occurrence-specific changes you made over the life of the event.


Now, go back the the original calendar and delete the event.  Import the CSV file and you'll have all of the occurrences back in your calendar up to the change point.  Copy one of the occurrences to the new time, make the changes you wish, create the new recurrence and you should have what you want from the change point forward.

Reverting is fraught with danger.  I have seen far too many of them go bad.

Reverting can only be done if you installed within the last 30 days (if you created a new user it must be removed before you revert)

surfacem say:

+1

Issue remains.  It may be a Windows 10 Mobile issue.  When phone is displaying via the WDA, if the phone goes to sleep or lock screen mode, it disconnects from the WDA.  Many apps let the phone go to lock screen, so the phone disconnects from the WDA at very inconvenient times.

Simply put, whenever the phone goes to the lock screen (or black/glance screen), WDA disconnects and then one has to re-connect manually.

The solution is to prevent the OS from locking when connected to the WDA (like other apps do), or to let the phone remain connected to the WDA even when locked (like Bluetooth).  These could be default and user settings.
