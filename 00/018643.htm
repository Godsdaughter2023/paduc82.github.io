One day you'll find that the method I gave you doesn't work because as well as being text values there are also invisible characters; usually char 160,  that stop this method converting them. I became so fed up with solving this on an ad hoc basis I now keep the macro below in my personal workbook to clean data before converting.

Sub CleanMe()
Dim c As Range
    For Each c In Selection
    If Not c.HasFormula Then
    With c
        .Value = WorksheetFunction.Trim(.Value)
        .Value = Replace(.Value, Chr(160), "")
    End With
    End If
    Next
End Sub

pete_uk say:

Times are stored by Excel as fractions of a 24-hour day. So to get the difference between two times as decimal hours, you just need:

 

=(later time - earlier time)*24

 

and format as a number with 2 dp.

 

Dates are stored by Excel as the number of elapsed days since some reference date (1st Jan 1900), and so the date part of a date/time cell will always be a whole number, whereas the time part will be fractional. So, if your arrival time also includes the date you will need to extract the fractional part to use it above. You can do this like the following:

 

arrival time - INT(arrival time)

 

or like this:

 

MOD(arrival time,1)

 

Hope this helps.

 

Pete

TIME-TO-FRACTION -- IT WORKS, BUT…

A number of people suggested the following formula (or a variation of it). While it will work on hours that are within the same day, it does not calculate correctly when you have time that spans across days. Why? Because it ignores the date portion of the date/time. It simply extracts the hours and minutes.

=HOUR(End-Start)+(MINUTE(End-Start)/60)

Suppose that cell B2 contains a start time of 2:00 and cell C2 contains an end time of 2:30. The formula would be

=HOUR(C2-B2)+(MINUTE(C2-B2)/60)

and the result, when formatted with a General or Number format, would be .5. Now (using a 24 hour clock) if cell B2 contains a start day/time of Dec/6/99 8:00 and cell C2 contains an end day/time of Dec/7/99 15:30, the result using the above formula (when formatted properly) will be 7.5, when clearly it is 31.5 hours (7.5+24).

Naturally you could modify the above formula to account for this:

=(DAY(C2-B2)*24)+HOUR(C2-B2)+MINUTE(C2-B2)/60

But there is no need for such a complex formula. There is a far more simple way to convert time to fractions... read on!

TIME-TO-FRACTION -- EASY and FLEXIBLE

Using the following example, you simply multiply the subtraction (End-Start) by 24. When you consider how Excel gets the fractional serial numbers for date and time -- they are a decimal fraction of a day -- therefore if you multiply the subtracted time by 24 (hours in a day), you'll have a decimal fraction of an hour.

B2(start)     C2(end)        Formula          Results

2:00          2:30           =(C2-B2)*24      0.5

Dec/6/99 8:00 Dec/7/99 15:30 =(C2-B2)*24      31.5

FORMATTING NOTE

If you perform the subtraction in one cell and the multiplication in another cell, you will have to format the results using the General format (Ctrl+Shift+~) or using the Number format. However, if you combine the formula into one cell as shown above, the result will display as a fraction without you having to apply a separate format.

CROSS-DAY/MIDNIGHT TIME CALCULATION NOTE

If you perform the subtraction shown above (Dec/7/99 15:30 - Dec/6/99 8:00) without the multiplication, the result appears as either 1/1/00 7:30 or just 7:30 depending on how you created/copied the formula.

HOWEVER, when you perform the multiplication on the 1/1/00 7:30 subtracted result (or include the multiplication in the formula), the correct answer of 31.5  (thirty-one and a half hours) appears.

TIME-TO-FRACTION -- CALCULATING MINUTES or SECONDS AS A DECIMAL

Likewise if you want the result to appear in minutes instead of hours/minutes, you would multiply by 1440 (24*60 = minutes per day).

B2(start)     C2(end)        Formula          Results

2:00          2:30           =(C2-B2)*1440    30

Dec/6/99 8:00 Dec/7/99 15:30 =(C2-B2)*1440    1890

To get seconds, you would multiply by 86400 (24*60*60)

OTHER TIME-TO-FRACTION FORMULA SUGGESTIONS

One reader suggested the following User-Defined function.

Function numHours(TimeVal) As Double

    numHours=Hour(TimeVal)+(Minute(TimeVal)+Second(TimeVal)/60)/60

End Function

You can use the function as is, IF the time is all in the same day. This function does not calculate properly when the date/time crosses over midnight, as can be seen in the examples below.

B2(start)        C2(end)        Formula                  Results

2:00             2:30       =numHours(C2-B2)     .5

 

B2(start)        C2(end)             Formula                 Results

Dec/6/99 8:00    Dec/7/99 15:30  =numHours(C2-B2)     7.5

To create a User-Defined function, start by creating a new Excel workbook. Then in the VBA Editor (Tools, Macro, Visual Basic Editor), create a new module (Insert, Module). Copy/paste the code you want to use (such as the code above) into the VBA module. Then save the workbook into the XLStart folder. This folder is typically located under C:\Program Files\Microsoft Office\Office. If you cannot locate the XLStart folder, use the Find command in Windows Explorer (Tools, Find) to locate the XLStart folder. When you start Excel, the workbook containing the VBA module always loads and the function will be available to you regardless of the workbook your working in. You should be able to invoke the function through the Paste Function/Function Wizard button on the Standard toolbar. It will be listed in the User Defined Functions category in the dialog box.

CONVERTING A SINGLE DATE TO A FRACTION

Suppose you have a cell that contains both a date and time, such as Dec/11/99 11:30 and you simply want to see the time represented as a decimal. Using the multiply by 24 technique doesn't work on just a single date. John T. suggested this formula

=(Date-INT(Date))*24

If the date you have is 12/11/99 4:30 PM, this formula will display 16.5 as the result.
