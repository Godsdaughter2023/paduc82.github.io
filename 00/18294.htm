How to incorporate resource constraints - Microsoft Community

sankhadeeprakshit say:

How to incorporate resource constraints

Hi, I am using project 2010. I have a problem where the company will charge $200 per week for each worker that is used. If it happens to need an additional worker, there is one available at $300 per week. ($100 being an additional cost for coordinating between the workers.) There is also a constraint on a number or worker working on any task at any given point of time and that is maximum 2. Also the maximum number employed in the tasks at a given week is 5.

How will I be able to put these constraints in the Resource sheet?

Thanks

dale howard [mvp] say:

SankhadeepRakshit --

You can use Cost Rate tables for the $200 and $300 costs for your resources.  Put the $200/week in the Standard Rate column for every resource, which will appear automatically on Cost Rate Table A for the resources.  Then enter the $300/week value on Cost Rate Table B for each resource.  If you add a second resource to a task, you will need to set the Cost Rate Table value to B for the helper's resource assignment (do this in either the Task Usage or Resource Usage view).

Regarding the other constraints, such as no more than 2 resources per tasks, and no more than 5 resources working in a week, must be handled using a methodology or process.  There is no setting in Microsoft Project to enforce the resource constraints you describe.  Hope this helps.

john - project say:

Sankhadeep,

A question, is the $200/wk and $300/wk a flat rate? In other words, does a resource get paid $200 for a week whether he/she works 1 day or 5 days? If it is a flat rate, then simply entering $200/wk into the rate table will not be what you need because Project converts all rates to rate/minute. For example, if the rate is $200/wk but a task only takes 30 minutes, the cost for that task will be calculated as $2.50. Likewise, if a task takes 1.5 days, the cost will be $60.

In order to apply a flat rate in Project, a macro is required.
