rohn007 say:

I agree with everything that Jay said.  As far as I can see, MS has done everything it considers it needs to reveal codes, so I also think it is extremely unlikely that MS will do more.   The article does a good job of explaining what Word does to reveal formatting. Just one additional point. Although the article does suggest looking at the ribbon to see formatting I think they could put more emphasis the idea that when you apply something in the ribbon there is a subtle highlight to show the button is "turned on".  In Office 2016 the specific highlight color depends on the "Office Theme" you are using. You just have to look for it. I mention this because it was over a year before I learned about this feature when I started using Word 2007.

The thing is from day one Word and WP had fundamentally different design philosophies for their document files.  WP approached formatting like HTML.  The file is a long text string with "codes" embedded to turn formatting on and off. So "reveal codes" was natural for them. You could open their files in a text editor and easily understand what is going on.  On the other hand Word created a "binary" format file that was structured like a compiled program file that can only be read by a program specifically designed to decrypt the file structure. One key feature in Word is the use of styles.  All of the formatting information is stored in one place, at the start or end of the file. When you apply the formatting Word uses a "pointer" to point to the style information. Remember, Word's paragraph style information is stored in the "paragraph mark" at the end of each paragraph.  Even when MS switched to the new text / XML based file formats in Office 2007, they still maintained the same design strategy, styles defined in one place, applied using pointers. 

The point of this explanation is that in Word, there effectively are no "codes" to reveal!

That being said, I tried CrossEyes a couple of years ago. It does an incredibly good job of translating Word style based formatting into a WP-like "Reveal Codes" window.  I have not tried it with 2016, but expect it still works.
