john - project say:

Daniel,

The only way to have the master automatically updated when something changes in a subproject is to have a dynamic master. However, as you are apparently aware, linked structures (e.g. master/subproject) in Project are prone to corruption especially when multiple users are involved.

Let me suggest an alternate and safer method. Instead of creating a dynamic master, any time you need to see all workstream plans rolled into one file, go ahead and insert each of the 22 separate plans into a master but uncheck the "link to project" option in the lower right corner of the Insert Project window. That will create a single project file that is a snapshot in time of all the inserted projects. Since it is not dynamic, you will need to create a new static master each time you want an update. One easy way to do that is to record a macro the first time you create a static master. After that, you can just run the macro. Note however that if you record a macro, none of the individual files can be moved to a different location, otherwise the macro code won't be able to find them. With a little more sophisticated macro (i.e. not a recorded macro) that requirement could go away.

For reference you cannot create a dynamic master with files on SharePoint. I do not use SharePoint so I don't know if you can create a static master with the individual files on SharePoint but it is easy to do a quick test. If you find that you cannot use SharePoint, then your best bet is to set up a separate directory on a shared server for all the user files.

Hope this helps.

John

john - project say:

Daniel,

Inter-project links and dynamic master links are two separate features of Project so yes, you can have inter-project links without ever creating a master. The downside is that inter-project links have the same fragility as dynamic masters, that is, unless you adhere to the "rules" for linked structures, you court corruption. Never rename, move, overwrite or "save off" to another location any of the files in the structure. Ideally, all files should reside in the same directory on a local drive and not be operated over a network.

For reference, inter-project links will NOT carry over to a static master. That can be done, but it requires a macro which I happen to have but it is not freeware. Given rigorous file management discipline and a macro to convert inter-project links into normal predecessors/successors when creating a static master, the idea does work.

Hope this helps.

John

I had some success with this, although it isn't a total solution and I imagine there's a better way. I suspect that desktop Outlook 2016 may give more options too.

  5. Use the "9-squares" icon to navigate to Calendar (for the room this time)
  6. Right-click on Calendar down the left-hand side and use Share or Permissions to give other people access.
  7. However I don't know how to set it so that Everyone in the organisation can see it read-only, but someone else may know how to do that...
