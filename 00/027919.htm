andreas killer say:
The error is not due to the code itself, the issue might be how the user has copied the code.

In the past I've often user that uses Microsoft Edge in here and if they copied code from this forum directly into the VBA editor they've seen that some lines are red.

After my investigation I found that some invisible chars (ASCII 160) are copied from the browser window too. So my suggestion to the users was to paste the code into Notepad first, then reselect and copy & paste into the VBA editor. That removes the invisible chars.

For the regional formats: VBA speaks English, means the English date delimiter / is converted to the local date delimiter automatically. The call of the Format function is superfluous because Excel converts the text into a date automatically (except the cell is formatted as date).

So the only point is the am/pm part, you can check and remove that very easy: 

Sub clockInOut()
  With Selection
    .NumberFormat = "m/d/yyyy h:mm" & IIf(Application.International(xl24HourClock), "", " am/pm")
    .Value2 = Now
    .EntireColumn.AutoFit
  End With
End Sub

mike h.. say:

Hi,

 

 

Attach this code to a button. The code toggles the columns hidden/visible

 

 

Sub somesub()
Dim FirstCol As Long, LastCol As Long
LastCol = Cells.Find(What:="*", After:=[A1], _
SearchOrder:=xlByColumns, _
SearchDirection:=xlPrevious).Column

'Hide columns to right of data
LastCol = Cells.Find(What:="*", After:=[A1], _
SearchOrder:=xlByColumns, _
SearchDirection:=xlPrevious).Column + 1

Cells(1, LastCol).Resize(, (Columns.Count - LastCol + 1)).EntireColumn.Hidden = _
Not Cells(1, LastCol).Resize(, (Columns.Count - LastCol + 1)).EntireColumn.Hidden

'Hide columns before data
FirstCol = Cells.Find(What:="*", After:=[A1], _
SearchOrder:=xlByColumns, _
SearchDirection:=xlNext).Column
If FirstCol = 1 Then Exit Sub
Cells(1, 1).Resize(, FirstCol - 1).EntireColumn.Hidden = _
Not Cells(1, 1).Resize(, FirstCol - 1).EntireColumn.Hidden


End Sub

renegade technologies say: Check the System Event Log when you try opening a document that gives you this problem. If you see these DistributedCOM errors:

The application-specific permission settings do not grant Local Activation permission for the COM Server application with CLSID
{A0A8C0AC-FC70-4EE2-93A8-4A2257AE8619}
 and APPID
{38AFE312-B8E5-4354-A11F-9224307B28AC}

and

The machine-default permission settings do not grant Local Activation permission for the COM Server application with CLSID
{000C101C-0000-0000-C000-000000000046}
 and APPID
{000C101C-0000-0000-C000-000000000046}

then here's how to (supposedly) fix the problem (on Server 2003, 2008, and 2008 R2 at least):
1) Start the registry editor
2) Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Classes\AppID\{000C101C-0000-0000-C000-000000000046}
3) Change the permissions on the branch so that the local Administrators group OWNS the branch, then add the Local Admin group to the users who can modify the branch.
- You may also need to do this for the other CLSID mentioned in the Event Log messages.
4) Start Administrative Tools -> Component Services
5) Expand Computers -> My Computer -> DCOM Config
6) Switch to Detailed view
7) Find the component with the same CLSID as mentioned in the Event Log messages (stinks that you can't sort by CLSID.)
8) Right-click it, click the security tab, then under Launch and Activation Permissions, click Edit...
9) Add the group/user as appropriate and check "Local Activation" under the Allow column.
10) Repeat for the other CLSID.

(I got these steps from https://kb4sp.wordpress.com/2011/06/30/fixing-the-dcom-error-the-application-specific-permission-settings-do-not-grant-local-activation-permission-for-the-com-server-application-with-clsid/)
