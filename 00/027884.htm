Currently don't have access to a test system with OL'2013 on it and don't remember if you can still add the same email account using a different protocol if it already exists in the profile (i.e. adding the same account using IMAP if it already exists as a POP account) - you can't do that with OL'2016 so going to cover both scenarios

One thing that you will need to check with your email provider is the amount of space you have available for your email data so that you don't run into any server capacity limits

Scenario #1 - If you can add the same email account using a different profile

#1 - Add the email account by just entering the email address/password - that will default to it being added as an IMAP account
#2 - Highlight the POP3 email account and select <Remove>
#3 - From the Outlook <Send/Receive> Ribbon - set Outlook to work "Offline" (just click on the "Work Offline" to toggle it
*** the reason for doing this is to prevent Outlook from starting to sync data before you've finish copying everything
#4 - Copy each of your custom folders to the same location in the IMAP folder tree
(right-click on the folder --> Copy folder --> select the destination in the IMAP folder tree
#5 - For your <Inbox> items, select the <Inbox> folder --> press CTRL+A to select all the items --> right-click on the selected items --> select <Move> --> select <Copy to folder> --> select <Inbox> in the IMAP folder tree
*** reason for selecting <Copy to folder> is so that the original items don't get deleted from the source <Inbox> folder in case something goes wrong during the copy process - if you select <Other Folder> the source data gets deleted after being copied to the destination folder
#6 - Set Outlook to work <Online> and initiate a manual <Send/Receive> on the IMAP Account. This starts the sync with the server and could take a while depening on how much data has to be copied up to the server (again, be sure that you have enough capacity on the server for the amount of data being copied to it)

At this point, you will need to keep the data file from the POP3 account since it contains your contact/calendar folders. The PST file will still be set as the default data file for the profile which is where the default contact/calendar folders are stored (will cover sharing contact/calendar data separately)

Scenario #2 - You can't add the IMAP account to the same profile where the account is configured as a POP3 account

#1 - Purely as a precaution, create a new Outlook profile by <copying> the existing one. Open the Control Panel <Mail> app --> <Show Profiles> --> <Copy>
#2 - Next --> Select your existing <Profile> --> select <Properties> --> select <Email Accounts> --> select the POP3 account --> select <Remove>
#3 - After the POP3 account is removed --> <Add> the account using IMAP (just entering email address/password
#4 - When done, close Control Panel and open Outlook
#5 - Follow steps #3 thru #6 from Scenario #1

Scenario #3 - Sharing contact/calendar data

#1 - Done by creating a free Outlook.com account - go to https://www.outlook.com and follow the steps to create a new account
#2 - Add this account to your existing profile by entering the email address/password - it will be configured as an Exchange account by default (there is no requirement that the account is used for email purposes)
#3 - Select your existing contact folder --> press CTRL+A to select all your contacts --> right-click on the selected contacts --> select <Copy to Folder> (for the same reasons as previously mentioned
#4 - Select your <Calendar> folder --> change the <View> for this folder to be a <List> view by clicking on the <View> tab --> <Change View> --> select <List>
#5 - Do the same as item #3
#6 - From Outlook menu --> File --> Account Settings --> Data Files tab --> select the .ost file for the Outlook.com account --> set it as the default for the profile which is now where your default contact/calendar folders will be stored
#7 - If you're comfortable that all your contact/calendar data has been copied correctly, highlight the PST file from the original POP3 account --> select <Remove>

You can now add the Outlook.com account to your other devices allowing you to have the same contact/calendar data on all of them
