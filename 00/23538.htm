bill manville say:

I think you would need to use a Worksheet_Change macro to detect the change in the currency designator.

I would select one of the currency cells and define a new Style, say VarCur, with Number among the properties of the Style.
(Format > Style > VarCur > Add, uncheck all except Number - unless you want to standardise other aspects)
I would then apply the style to all the other variable currency cells...
(select the cells, Format > Style > VarCur > OK)

Then in the module of the worksheet containing the designator, let's say it's in a cell named "Currency"

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim stFormat As String
  If Application.Intersect(Target, Me.Range("Currency")) Is Nothing Then Exit Sub
  Select Case Me.Range("Currency").Value
    Case "USD", "AUD", "NZD"
       stFormat = "\$#,##0.00;-\$#,##0.00;\$0.00"  ' \ prevents it being translated into the local currency format
    Case "UKP":
       stFormat = "£#,##0.00;-£#,##0.00;£0.00"
    Case "EUR"
       stFormat = "€#,##0.00;-€#,##0.00;€0.00"
    ' etc
  End Select
  ThisWorkbook.Styles("VarCur").NumberFormat = stFormat
End Sub

The \ only makes one following character literal.  S has meaning in format strings whereas Z does not.

I would think you would need

stFormat = "\U\S\$#,##0.00;-\U\S\$#,##0.00;\U\S\$0.00"
or
stFormat = """US$""#,##0.00;-""US$""#,##0.00;""US$""0.00"

The only way to share an MS Access database is to split the database into 2 parts: the Back-End -> stores all the tables (the data) and the Front-End -> stores queries, forms, reports, vba.  You place the back-end in a central location, relink the Front-end to this new location and then give a copy of the Front-End to each end-user.  They need their own copy, they should never share a copy!
