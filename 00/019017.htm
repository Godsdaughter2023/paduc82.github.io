albert d. kallal say:

The “switching” you not is not due to the 32/64 issue, but likely due to a DIFFERENT version of office/runtime being used.

 

So the “configuring” screen they see each time is because you’re running a DIFFERENT version of office.

 

If you are distributing a compiled accDE (as opposed to an accDB), then your application can run x32 or x64.

 

However, if you are distributing compiled accDE (and I highly recommend you do since un-handled errors will NOT shut down the runtime – if you use a accDB, then ANY un-handled error WILL shut down and terminate your application (thus your application tends to be un-reliable unless you go crazy on error handing in every  small VBA routine).

 

If you choose to distribute a compiled accDE, then you MUST compile the accDB to the accDE with an x64 version of Access for the x64 machines. And for the office x32 you again MUST compile the accDB to an accDE with an x32 version of office.

 

So the basic trade off issue is:

 

Distribute an accDB – will (can) work on ether version, but any un-handled errors will terminate your application.

 

Distribute an accDE – you must compile the accDE for the correct given bit size. So an x64 accDE will not work/run on an x32 version of Access, and of course visa versa.

 

Of course both office x32 and x64 run on x64 OS machines. However, the current recommend is to ALWAYS run x32 office – even on x64 machines. As you note, you may well not have control over this issue.

 

The other note is you can NOT install or MIX versions of office applications for the SAME version of office. So if a customer is using Excel 2010 (x64), then you cannot install Access 2010 x32. (nor word x32). You can (could) install say Access 2013, or 2016 x32 versions. So as long as you not mixing the SAME version of office, you can install mixed x32/x64 applications from the office suite

 

Last but not least:

You can have ONE copy of your application, and it can work/run on x32 or x64. If you use any API calls, then such code needs some modifications. While you can have this "one" master copy of your application, when using a compiled accDE, you have to compile on the correct given version of office. If you distribute the master accDB (non compiled), then it can run on both x32/x64 versions of Access.
