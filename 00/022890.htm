There are no 'pages' in a Word document. Word is not a page layout application. What data types do the fields have? They should all have a number typ
not text. If they are text/strings the + operator may result in string
concatenation instead of addition. The same applies to your expression:
 
IIf([NSetUpCharge] Is Null,"0",[NSetUpCharge]/[NQty1])
 
By using quotes in "0" you get a string instead of a number.
So this should be:
 
IIf([NSetUpCharge] Is Null,0,[NSetUpCharge]/[NQty1])
 
As a general rule delete the quotes around the numbers in
all involved expressions.
If you can't change all data types to numbers then you have to cast
the characters to numbers before you can use them for calculations
e.g. with CDbl(FieldName) or CCur(FieldName).
If you don't succeed, then post the SQL string of your query.

karl timmermans (outlook mvp 2012-2018) say:
If you are upgrading to '2016 from '2010 on the same machine (an in-place upgrade) then the info from the '2010 profile will be used by '2016 with the following difference when it comes to any IMAP account

- OL'2010 used 2 PST files for an Imap account - (1) for the email folder data and (2) the default data file for the profile which would contain the default contact/calendar folders
- on a <new> install for '2016 and the IMAP account is freshly configured, the Imap data is stored in an .OST file which would include contact/calendar folders with the added words "This Computer Only". When an "in-place" upgrade is done, Outlook creates an .ost file for the email folders and re-syncs with the server. The contact/calendar folders remain in the original PST file as defined in '2010.

For an in-place upgrade, there is nothing to copy to accomplish the upgrade. OL'2016 will create a new Outlook profile that is used by OL'2016 only leaving the original OL'2010 profile in tact. However, it's always a good idea to have a backup of local data (i.e. the default data file containing your contact/calendar info). The PST file for email folders is really a non-issue since that is always a mirror of the data stored on the server.
