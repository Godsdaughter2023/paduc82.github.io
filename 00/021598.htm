stet7 say:

Macro for deleting unwanted styles, keeping custom styles?

Hi, I need some help with clearing out unwanted styles. I receive Word docs from all over the world, and they are often chock-full of garbage paragraph and character styles. I've searched online for solutions, and all I've been able to find are methods of deleting all the "unused" styles, like this:

http://word.tips.net/T001337_Removing_Unused_Styles.html

The problem is that I want to keep all my custom styles, whether they're currently in use or not. Up till now, my workaround has been to use a macro that does the following:

1. Insert the text from a style-palette doc on my hard drive. This palette includes all my custom styles; inserting the text makes them "in use."

2. Run the code above from Word Tips, clearing out unused styles.

3. Delete the text inserted from the style palette.

This solution works, but it seems unnecessarily clunky. Is there a better way to do this? I'd appreciate your advice. 

jay freedman say:

The macro below will do what you want. For each style that is not built-in, it compares the style's name to a list of names in the variable strKeepers. If the style's name isn't in the list, the InStr function returns the value 0, so the macro deletes the style.

You'll have to construct the list in strKeepers with the names of the styles in your palette document, separated with pipe ( | ) characters; also place a pipe character after the last name. The list is case-sensitive (although that could be changed if you prefer). Style names may have spaces within them, but don't put spaces before or after names in the list.

Sub DeleteCustomStylesExceptMine()
    Dim strKeepers As String
    Dim oStyle As Style
        
    strKeepers = "MyBig|MyBold|"
    
    For Each oStyle In ActiveDocument.Styles
        If Not oStyle.BuiltIn Then
            If InStr(strKeepers, oStyle.NameLocal & "|") = 0 Then
                oStyle.Delete
            End If
        End If
    Next oStyle
End Sub

Your problem may be caused by a combination of factors such as another Word add-in. The way to verify this is to see if spell-check works in “safe mode”. Safe Mode is a reduced functionality state where Microsoft Word loads without add-ins.

Hold down your Ctrl key and the press the icon or menu option for Word.

Open up an existing document.

It could be that your problem is tied to your global template which is either normal.dot or normal.dotm for Word 2007. These files are usually found in your MicrosoftTemplates folder. When you rename the template, Word will reset itself to the default settings.

Because your global template file has your settings, I would rename the file to something like normalPRV.dot or normalPRV.dotm so you can change the name back if this suggestion doesn’t work.

Solution 4: Try Detect and Repair
Every now and then, I have the spell-check feature go wonky after some other software update or hardware change. I suspect some registry entry has become corrupted. In these rare cases, you may be able to solve the problem using the built-in Detect and Repair… feature. In Word 2007, this is called Diagnose. Before running this utility, make sure you have your installation media as you may be prompted for a CD.

Type regedit
Click OK
Expand the left pane to HKEY_CURRENT_USER | Software | Microsoft | Shared Tools | Proofing Tools

Close regedit.
Restart Word.

brian tillman [outlook mvp 2007-2019] say:
Your impression is incorrect.  Accessing your Outlook.com account via Exchange ActiveSync with Outlook 2013 is a two-way connection.  Anything you do on Outlook 2013 happens on the server and vice versa.  If you delete something in Outlook 2013, it's deleted from the server and if you delete something via Outlook.com, it will be deleted from Outlook as well.  The Deleted Items folder still works, though.  Delete something and it will be moved to Deleted Item in Outlook 2013 or the Deleted folder on Outlook.com.  Simply move it back from that folder to its original folder to recover it.

Hey everyone,

 

Try clearing out your Cookies and Temporary Internet Files:

·        Open Internet Explorer

·        Press Ctrl + Shift + Del on your keyboard

·        Uncheck all boxes

·        Click Delete

·        Allow the deletion process to finish

·        Close all windows of Internet Explorer windows

·        Open Internet Explorer

 

Previous versions of Internet Explorer can delete the Cookies and Temporary Internet files by clicking Tools > Internet Options and selecting the appropriate option under the General tab.

 

Next check to see if you have the same problem if you disable all add-ons:

·        Click Start

·        Click All Programs

·        Click Accessories

·        Click System Tools

·        Click Internet Explorer (No add-ons)

·        Click Tools

·        Click Manage Add-ons

·        Click Toolbars and Extensions

·        Under Show: click All add-ons

·        Click any add-on

·        Press CTRL + A on your keyboard (this should select all add-ons)

·        Click Disable all

·        Click Close

·        Close Internet Explorer (No add-ons)

·        Open Internet Explorer

 

Now all add-ons are disabled in Internet Explorer, test to see if the same problem occurs without add-ons. Also try accessing the website I linked above that allows for the selection of the local version of MSN. If the problem is fixed then the next step is to enable the add-ons one by one to find which is causing the problem. (an alternative method is provided below these instructions)

·        Click Tools

·        Click Manage Add-ons

·        Click Toolbars and Extensions

·        Under Show: click All add-ons

·        Click the first disabled add-on

·        Click Enable

·        Click Close

·        Surf a few pages

·        Close Internet Explorer

·        Open Internet Explorer

 

If Internet Explorer still opens/works correctly, the add-on you enabled is good to stay. Repeat the enabling process for the next disabled add-on and continue this process until the problem returns. Then the last enabled add-on will need to be disabled/uninstalled/updated to avoid further problems.

 

Alternative method:

Instead of enabling add-ons one at a time, you could enable the first half of the add-ons and then test. If the problem is still gone, the add-ons you enabled are good to stay. Then enable half of the remaining disabled add-on and so on until the problem returns. Once the problem returns, you will need to test the last group of add-ons you enabled one by one.
