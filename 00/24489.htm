For some reason that I have never really been able to understand some properties need Object in front of them. By declaring a variable for the control you get the intellisence dropdown that gives you a clue for the correct syntax.

Sub test()
    Dim i
    Dim nRows
    Dim cbxColumn
    Dim tmpColumn
    Dim objOLE As OLEObject
   
    cbxColumn = 5       'Dummy value for testing
    nRows = 10          'Dummy value for testing
    tmpColumn = 4       'Dummy value for testing
   
    With ActiveSheet.OLEObjects
             For i = .Count To 1 Step -1
                 .Item(i).Delete
             Next i
    End With
   
    For i = 2 To nRows  ' Assumes row 1 is a header row
        Set objOLE = ActiveSheet.OLEObjects.Add(ClassType:="Forms.CheckBox.1")
        With objOLE
            .Left = ActiveSheet.Cells(i, cbxColumn).Left + 1
            .Top = ActiveSheet.Cells(i, cbxColumn).Top + 1
            .Width = ActiveSheet.Cells(i, cbxColumn).Width - 2
            .Height = ActiveSheet.Cells(i, cbxColumn).Height - 2
            .Name = "cbx" & i
            .LinkedCell = ActiveSheet.Name & "!" & Cells(i, tmpColumn).Address
            .Object.TripleState = True
            .Object.Caption = ""
        End With
    Next i

End Sub

Further, if each of IsNull([Myfield]) or IsEmpty([Myfield]) is used alone as a validation rule, neither work.

Finally, why does [Myfield]="Z" work if Z is keyed in but [Myfield]=" " not work if a single space is keyed in?

I am clearly doing something silly!

john w. vinson say:

NULL is a funny beast. It is emphatically NOT the same as a blank, or a zero-length string.

In fact, Access automatically trims trailing blanks; and defaults to prohibit zero length strings - so (by default) your field can contain either "Z" or a Null, but nothing else.

You can if you (unwisely in my opinion) want to allow either "" or Null to be entered change the field's Properties in Table Design; set the Allow Zero Length string property to Yes. A validation rule of

IS NULL OR = "Z" OR = ""

should allow these entries. The NULL and the zero length string will be visually indistinguishable; a query with a criterion of 

IS NULL 

or 

IS NOT NULL

will distinguish them, but you'll need some code to give the user a visual indication of which is which.

I BELIEVE there is a way to change the table definition to keep it from truncating trailing blanks, allowing entry of " ", but I think only by creating the table using SQL DDL and the Char() datatype - which is fixed length, so if you want to enter a single blank you would prohibit either zls or NULL.

Could you explain the purpose of this field and why you want to store any of "" or " " or NULL?
