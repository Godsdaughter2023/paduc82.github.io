Hi Eric,

I can feel your pain. That is an extremely annoying problem. After waiting for two a week without a reply, I decided to surf the web to find the answer. This problem is not uncommon. It appears to stem from a driver issue between HP and Windows 10. I found out that disabling 'Printer Status Notifications' seemed to do the trick.

Here's how I did it:

Right click on the windows start icon,

Select control panel,

Select 'View Devices and Printers'

Right clicked on my HP laser printer,

Select 'Printer Properties',

Disable 'Printer Status Notifications'

Clicked 'Apply'

and reboot the computer.

This fixed my problem with my HP printer. I hope this helps.

Salvador wrote:

Application.OnTime TimeValue ("23:31:00"), "update"


It would be better to have one OnTime statement in Auto_Open.  (But I would use Workbook_Open.)

And put the following at the beginning of "update":

' global declaration
Dim lastOnTime as Double

Sub update()
lastOnTime = lastOnTime + TimeSerial(0,30,0)
Application.OnTime lastOnTime, "update"
' ... the rest of "update" ...
End Sub

 

' execute manually
Sub stopUpdate()
Application.OnTime lastOnTime, "update",, False
End Sub

 

Start the timer as follows:

 

Private Sub Workbook_Open()
Dim n As Double, d As Long, h As Long
' call "update" at next hh:01:00 or hh:31:00
n = Now    ' call Now only once, in case we are near next minute
d = Int(n)
h = Hour(n)
lastOnTime = d + TimeSerial(h, 1, 0)
If n > lastOnTime Then
    lastOnTime = d + TimeSerial(h, 31, 0)
    If n > lastOnTime Then
        lastOnTime = d + TimeSerial(h + 1, 1, 0)
    End If
End If
' call "update" immediately after we exit Workbook_Open.
' remove next line if that is not what you want
lastOnTime = lastOnTime - TimeSerial(0, 30, 0)
Application.OnTime lastOnTime, "update"
' ... the rest of Workbook_Open ...
End Sub

 

Note that calling "update" immediately might result in two calls in quick succession if Now is near the next hh:01:00 or hh:31:00.

Courtesy of forum user bettisph:

 

Many suggested solutions recommend to delete

[HKEY_CLASSES_ROOT\Installer\UpgradeCodes\1F69ACF0D1CF2B7418F292F0E05EC20B].

 

What they fail to mention is that on current systems, there might be "Micosoft Security Essentials Client Update Package" installed. To reinstall MSE, after going through the usual MSE v2 removal instructions (http://support.microsoft.com/kb/2483120), also delete this key:

 

[HKEY_CLASSES_ROOT\Installer\UpgradeCodes\26D13F39948E1D546B0106B5539504D9]

(which contains 77B248F0AE65FAA42859180A2253B0E5)
