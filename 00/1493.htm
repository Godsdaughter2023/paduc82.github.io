scott mcdaniel say:

You can split your tables, but as Mimmo suggests an Access file that grows rapidly is a sign that there is a problem with the file. If that's the case, then you should take steps to correct that problem, and not simply try to find a way around it.

To try and correct the problem, do this:

MAKE A BACKUP OF YOUR DATABASE. This is as simple as locating the file in Windows Explorer and pressing Ctrl+C and Ctrl+V

1. Compact the database

2. Compile the database - from the VBA Editor click Debug - Compile. Fix any errors, and continue doing this until the menuitem is disabled.

3. Compact again.

Try to decompile the database. To do this, create a Shortcut with this as the Target:

"full path to msaccess.exe" /decompile

Run that shortcut, then open your database. Access will decompile it when opening your database.

To finish the task you should create a new, blank database (as Mimmo suggested) and import everything into that database. When doing this, be sure to select to import relationships and such, otherwise you may have some issue. Access generally does a good job of importing relationships, but in some cases you may have to remake them.

After doing that, it's a good idea to run the 3 steps above again.

Finally, be sure your Office and Windows installations are fully up to date.

With all that said ...

If you still must split individual tables into separate files, you can do so like this:

1. Create a new, blank database to hold your table

2. In that new database, use the External Data - Import & Link - Access ribbon item to access your current database.

3. Make sure to select the "Import" option on the wizard screen

4. Select your current database using the Browse button

5. Select the table you want to import

6. Finish the wizard

After doing that, return to your original database and delete the table you just imported. Now use the same Ribbon item to LINK to that new table in the new database.

But as suggested earlier, you really, really should fix the issue of that unexpected, rapid growth in your db.

jeeped say:
If you must use Select and Selection then try,

    Range("A3").Select
    ActiveCell.Value = "X"
    Selection.AutoFill Destination:=Selection.Resize(Cells(Rows.Count, _
      "C").End(xlUp).Row - Selection.Row + 1, 1)

... or more succinctly as,

    Range("A3").Resize(Cells(Rows.Count, "C").End(xlUp).Row - 2, 1) = "X"

... or even,

    Range("A3:A" & Cells(Rows.Count, "C").End(xlUp).Row) = "X"

Note that the 2 in Cells(Rows.Count, "C").End(xlUp).Row - 2 is 1 less than the starting row. This code assumes that there is always data in column C that extends below row 3. If there is any possibility of that not being true, error control should be applied.
