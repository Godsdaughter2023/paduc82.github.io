First of all, what version of Project are you using? Is it fully updated with the latest cumulative updates?

Here's a little reference information about subprojects and master files. First, be advised that dynamically linked structures in Project are prone to corruption. You must never rename, move, overwrite or save off to another location any of the files in the structure. Ideally, all files should reside in a single directory on a local drive and not be operated over a network. As far as I know, you cannot created a linked structure via SharePoint.

Second, unless setup appropriately, the subproject start date and the master start date could be different. For example, if you created a subproject with a start date of 5/22/17 and then decided to create a master today (5/12/17), when you insert the subproject into the master any tasks in the subproject that are not constrained either with a predecessor or a start-no-earlier-than constraint, will shift over to the start date of the master. Perhaps that is what you are experiencing.

John

john - project say:

NewProjectUser,

In Project there are two types of master/subproject structures, dynamic and static. The former is the default and allows dynamic interaction between the files, however as I noted previously, a dynamic master is prone to corruption and cannot be used with SharePoint.

On the other hand a static master is created by unchecking the "link to project" option in the lower right corner of the Insert Project window. That will effectively copy the subproject into the master thus creating a snapshot in time of the subproject. Although a static master is not dynamically linked to the subproject (it is a separate independent file), it is not subject to corruption and can be uploaded to SharePoint. The disadvantage of a static master is that you need to create a new one any time you want an update. This can be automated by recording a macro the first time you create the static master and then just run the macro thereafter.

As far as other ways to provide visibility, you might want to look into Project Online or Project Server. They are designed for enterprise configurations.

Hope this helps.

John

azjames52o say:

Temp workaround is working for me , registry key that AdrianFeregotto mentioned below.  

Disable Windows Desktop Search Service for Outlook:

Path: Computer\HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\Windows Search
DWORD: PreventIndexingOutlook
Value: 1

NOTE: Please backup the registry key before modifying it.

  1. Click Start, click Run, type regedit in the Open box, and then click OK.
  2. Locate and then click the following subkey in the registry:

       HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows

  1. On the Edit menu, point to New, and then click New Key and name the key Windows Search.
  2. Click on the new key Windows Search.
  3. On the Edit menu, point to New, and then click DWORD Value.
  4. Type PreventIndexingOutlook for the name of the DWORD, and then press Enter.
  5. Right-click PreventIndexingOutlook, and then click Modify.
  6. In the Value data box, type 1 to enable the registry entry, and then click OK.
  7. Note to disable the PreventIndexingOutlook setting, type 0 (zero), and then click OK.
  8. Exit Registry Editor, and then restart Outlook.

john mcghie say:

Hi Nicola:

No.  Word uses/requires the pagination engine to generate the PRINT view.  If you could turn off repagination, Word could not produce the print view.

The old version of the operating system will not be contributing noticeably to the problem.  The latest versions are quicker, but Word is not quicker in them. It is possible that you are a little short of CPU power on an old computer, and certainly of RAM and hard disk speed.

Word 2016 was designed for 8 GB of RAM, and benefits from 16 GB for long and complex documents. Especially in PRINT view :-)  Similarly, if you are low on RAM, Word will try to compensate by using part of your disk as "virtual memory". And THAT will really make the system crawl.

Of course, if you were loaded with money, you wouldn't have the problem!  However, you may find that you can add a SSD hard disk to that computer (you can add one to most computers).  That will improve your speed close to 100 times.  If you can slot in two 8GB sticks of RAM, you will get an extra speed boost from that (depending how much you have now).

Of course, a new computer would fix all your troubles...

In the meantime, try to get used to Draft view: it really is faster.  And it's better for creating and editing: it is subtly enhanced to make it easier to see the things you need to see when creating text.

And of course: the only time you NEED to see how a document "looks" is just before you print it!  In the meantime, you can rely on the fact that if you have the style names correct, the document will be correct.  So you don't need to see it.  That's literally true!

I'm a long-document specialist (a Technical Writer producing 2 to 5 thousand pages...) and I very rarely work on such a document in print view: it's too slow.  But I do assign distinctive colours to my heading styles so I can see at a glance which one has been applied in Draft view.  Just before I publish, I set all the colours to the final palette.

Hope this helps
