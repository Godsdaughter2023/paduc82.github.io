Mark

Can you double check the Pop & smtp settings?

If you’d like to manage your Hotmail account using an email app, you’ll need to set up your POP3 or SMTP server and port settings in your email app. You’ll need the following info:

    POP3 server: pop3.live.com (port 995)
    SMTP server: smtp.live.com (port 25)

If port 25 has been blocked in your network or by your ISP, you can set the SMTP port to 587 with TLS or SSL Encryption (depending on which email app you're using).

Choose the Secure Sockets Layer (SSL) option for both POP3 and SMTP connections.

Choose the SMTP authentication for SMTP connections.

 Choose the outgoing server requires authentication option. (In most email apps, this option isn't checked by default.)

    User name: Enter your full email address.
    Password: Enter your Microsoft account password.

    If you don’t want email messages to be permanently deleted from your inbox after you download them to your mail program, select the Leave a copy of messages on the server option.

Prevent losing email in Hotmail when you've set up POP3

Hotmail has a feature that prevents losing email messages if you accidentally set up POP3 to delete them after reading them on the server. Here’s how it works:

    The first time that Hotmail receives a POP3 instruction to delete email messages after reading from an email app, Hotmail won't delete the messages. Instead, Hotmail moves the email messages that were successfully downloaded by the email app into a POP folder and out of the inbox. The POP folder is available in the Folders list.

    Hotmail keeps the email messages in the POP folder until you sign in to your Hotmail account and verify whether Hotmail should respect or ignore the POP3 delete command.

To verify your POP3 settings:

    In the upper-right of the Hotmail window, click Options, and then click More options.

    Under Managing Your Account, click Pop and deleting downloaded messages, and then select one of the following:

        Don’t let another program delete messages from Hotmail. (If your other program is set to delete messages from the server, we’ll simply move them to a special POP folder. They won’t be deleted.)

        Do what my other program says—if it says to delete messages, then delete them.

    Click Save.

toddalbertson say:

Using Advanced Query Syntax (AQS), you can specify the date a file was created.

Go to Start>File Explorer.  It will open this window.  The search box is located at the arrow:

On the left side of the window, specify where you'd like to search.  If you want to search the whole PC, select This PC.

Now, in the search box, type the following:  kind:=document created:2/16/2016

(Of course, you want to insert the appropriate date, but that is the format to use.)

It will then show you all documents created on the date specified (and it may take some time if you specify the whole PC).  For more information on advanced searching with AQS, go to: 

https://msdn.microsoft.com/en-us/library/aa965711%28v=vs.85%29.aspx

brian tillman [outlook mvp 2007-2019] say:

If you are accessing the "webmail" mailbox using a mail client like Outlook, then, by definition, it is not webmail.  "Webmail" means you use a web browser to access the mailbox.

 

Who is your non-Exchange mail provider?  I'm guessing Outlook.com, since otherwise you would not be able to designate the non-Exchange data file as the default data file.  The fact that outbound messages are being placed in the Exchange account's Outbox leads me to believe you have not designated the non-Exchange data file as the default.  Perhaps you mean you designated the non-Exchange account as the default account.  If that's the case, then the behavior you describe is explainable.  Outlook will place outgoing mail in the default data file's Outbox regardless of the default account.  Here's an explanation and a solution: http://www.slipstick.com/outlook/outlook-2010/multiple-accounts-and-the-default-account/

 

You can remotely connect to Exchange only if the Exchange admins have implemented Outlook Anywhere.  See http://www.slipstick.com/exchange/configuring-outlook-anywhere-overview/.  Otherwise, a VPN connection would be necessary or no connection would be allowed.
