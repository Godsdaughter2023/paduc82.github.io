teylyn say:

Hello,

the approach you link to uses text boxes to overlay cells. That's a bit clunky, but if it works for you, fine.

If you want the appearance of the lower table, you need to enter a line feed character after each comma in column B.

You can do a Find and Replace to replace all commas that are followed by a space, with commas that are followed by a line feed. To do that

- Select column B and hit Ctrl-H to open the Find and Replace dialog

- in the "Find What" box enter a comma and a space

- in the "Replace with" box enter a comma and then hold down the Alt key and type 010 on the number pad of the keyboard.

- Use "Replace All" to replace in one go.

Now each entry in column B should be on its own line and you can start building the text boxes for the hyperlinks. 

teylyn say:

The URL that you use to make the text box a clickable link must be entered directly int the hyperlink dialog. You cannot refer to a cell for the hyperlink address. That's why I feel the text box approach is clunky.  You still need to manually edit and define each hyperlink. It is not dynamic. It cannot be linked to a cell that contains the URL. 

You may want to re-think your data architecture and find a way that each CVE is in its own cell, so it can be assigned a hyperlink with a worksheet formula instead of using a text box.

Back in the old days, Visio had two supersets to Visio Standard. One was Visio Technical aimed at draughting and the other was Visio Pro aimed at IT guys. Eventually, Visio Technical was rolled in to Visio Pro, so what you had in Technical is now available in the Pro version.

The ribbon does take a bit of getting used to, but I find it far better than the old menus. With the menus, you had all the menus opened that you used, whether you needed them or not (or whether they were relavent to the current situation). The other issue I had was that when you changed the screen size (by adding a second monitor, the position of the menus could move leaving gaps between the mens). There was no capability to tell the menus to snuggle up and remove the gaps. I have a photo somewhere showing a dual monitor setup and almost 50% of the screen used by the menus is gaps.

The big issue is actually finding where a menu item exists on the ribbon. If you go to File -> Options -> Customize Ribbon, you can look up the location of where items are on the ribbon. Change the Choose Commands from drop down to say All Commands.

As to opening older versions, I have been able to open Visio 1 drawings in Visio 2010. The only caveate is if the drawing used an add-on. The drawing will open, but the add-on will not be there to run.
