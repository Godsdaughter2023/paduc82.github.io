%LOCALAPPDATA%

don cuthbert say:

It is hard to say how far into the install your PC got.  I believe that there is a point where it initially boots to the new installation files and then applies those to your existing Windows folders, in order to upgrade the OS while leaving most things unchanged from the previous version.  If you launched the upgrade by running setup.exe from the install media, by the time it restarts the PC, it is done with the install media and is working with what it has copied to your hard disk.

The commands listed simply ensure that the existing System partition is marked Active to enable it to boot on a Legacy BIOS device.  They also copy the required files to the System partition.  Bcdboot points to the install of Windows that you want it to boot from, in order to create the appropriate entries in the BCD (Boot Configuration Database) which is also on the System partition.  You could actually format the System partition and still use these commands to make the PC boot, but only if the Windows folder and partition are actually intact and functional. However, the System volume also contains a copy of Windows Recovery Environment, which would be erased with a format and would need to be replaced.

The function of C: is to host the MBR (Master Boot Record) of the disk and the System files.  If you run dir /a /s on the System volume (which at the moment is C:, but normally has no letter) you should see a lot of files and folders, but some are normally hidden files.  On Legacy BIOS devices, this has to be on the first partition of the disk and it has to be marked Active. It then looks to Windows (on the "Boot" partition / your usual C: drive) for the Operating System. The disk could be setup with a single NTFS partition, which could then function as a System, Boot and also have the Recovery files all in one place, but this is not standard. I guess keeping them separate allows for Repair efforts that would not be as easy otherwise.

Your approach to remove the disks and backup your data elsewhere is good.  Repairing what you have is only intended to make it operational more quickly than what you are suggesting.  Because you appear to be in mid-install, it makes it more difficult to anticipate the exact results, but the commands only affect the System partition.

If for some reason you had no other way to recover your data, you could even install a separate copy of Windows 10 to a small partition on the existing disks (and then use that install of Windows to recover your data) but that doesn't seem to be the case, since you are willing to replace the disks to do a clean install.

teylyn say:

as I understand your question, you would like a macro that prompts for a date, then opens a file that has such a date as the file name, and then copies and pastes specific cells from the just opened workbook to the already open workbook. 

It will be a lot easier, if the date is not entered as a date but in the format that is used in the file name, i.e. with - signs instead of / signs between month, day and year.

The following macro can be stored in your Personal macro workbook and you can run it from any file.

Sub CopyFromPreviousYearDate()

Dim sDate As String
Dim sSplit() As String
Dim fName As String
Dim iYear As Integer
Dim sExt As String
Dim sPath As String
Dim sSheet As String

Dim wbTarget As Workbook
Dim wbSource As Workbook

Set wbTarget = ActiveWorkbook
Debug.Print wbTarget.Name

' You need to change these three variables to suit your environment
' if you are using compatibility mode, the file extension will be .xls
sExt = ".xlsx"
' enter the full path to the folder where the daily files are stored
sPath = "C:\Users\UserName\Downloads\"
' enter the name of the Worksheet that has the numbers you need
sSheet = "MySheet"

On Error GoTo ErrHandler

sDate = InputBox("Enter a date in the format mm-dd-yy")

fName = sDate & sExt

Workbooks.Open Filename:=sPath & fName

Set wbSource = ActiveWorkbook

wbSource.Worksheets(sSheet).Range("A1").Copy wbTarget.Worksheets(sSheet).Range("A1")
wbSource.Worksheets(sSheet).Range("B2").Copy wbTarget.Worksheets(sSheet).Range("B2")
wbSource.Worksheets(sSheet).Range("C3").Copy wbTarget.Worksheets(sSheet).Range("C3")
wbSource.Worksheets(sSheet).Range("D4").Copy wbTarget.Worksheets(sSheet).Range("D4")
wbSource.Worksheets(sSheet).Range("E5").Copy wbTarget.Worksheets(sSheet).Range("E5")

' don't show prompt to save workbook
Application.DisplayAlerts = False
' close without saving
wbSource.Close
Application.DisplayAlerts = False

Exit Sub

ErrHandler:
MsgBox "Something went wrong. Maybe you entered a wrong date or the file for that date does not exist."

End Sub

There are a few places in the code where you need to enter your specific values. You need to set the values for the variables sExt, sPath and sSheet and you need to edit the cells that you want to copy and paste.

Hope that helps.
