jay freedman say:

It's possible to create "something like it", with some reservations.

You can put the following code in a module in Normal.dotm (see http://www.gmayor.com/installing_macro.htm if needed):

Dim IsCounting As Boolean
Dim IsPaused As Boolean
Dim StartCount As Long
Dim StartPause As Long
Dim PauseCount As Long

Sub StartCounting()
StartPause = 0
PauseCount = 0
StartCount = ActiveDocument.ComputeStatistics(wdStatisticWords)
IsCounting = True
IsPaused = False
StatusBar = "Word counting is started."
End Sub

Sub StopCounting()
Dim HowMany As Long
If IsCounting Then
IsCounting = False
HowMany = ActiveDocument _
.ComputeStatistics(wdStatisticWords) - StartCount - PauseCount
MsgBox HowMany & " words", , "Since Start"
End If
End Sub

Sub TogglePausing()
If Not IsPaused Then ' start a pause
StartPause = ActiveDocument.ComputeStatistics(wdStatisticWords)
StatusBar = "Word counting is paused."
Else ' end a pause
PauseCount = PauseCount + _
ActiveDocument.ComputeStatistics(wdStatisticWords) - StartPause
StatusBar

Put three buttons on the Quick Access Toolbar, one for each macro. Each time you click the StartCounting button and, later, the StopCounting button, you'll get a messagebox with the count of words added in that interval -- minus any added during one or more pauses.

The drawback is that there is no way to make QAT buttons for macros "depressed" to show whether counting is currently turned on, off, or paused.  The statusbar messages are the best I can do. If you want to do better, you'll need to add a custom group to the ribbon (see  http://gregmaxey.mvps.org/Customize_Ribbon.htm ).

macropod say:

The conditional output is easily achieved via field coding in the document itself; simply ensure the dropdowns have their 'calculate on exit' property checked. The field coding would be something along the lines of:

{IF{REF ddModel_07}= "Model-1" "Part# for Model-1"}

{IF{REF ddModel_07}= "Model-2" "Part# for Model-2"}

{IF{REF ddModel_07}= "Model-3" "Part# for Model-3"}

{IF{REF ddModel_07}= "Model-4" "Part# for Model-4"}

for the part #s, and:

{IF{REF ddModel_07}= "Model-1" "$1.00"}

{IF{REF ddModel_07}= "Model-2" "$2.00"}

{IF{REF ddModel_07}= "Model-3" "$3.00"}

{IF{REF ddModel_07}= "Model-4" "$4.00"}

for the costs (use your own Part #s and amounts). No VBA code required.

Although I've shown each set of fields on separate lines, that's just for clarity; in practice they'd all flow one after the other on the same line. FWIW, the output for a given dropdown selection could even include multiple paragraphs, tables and pictures.

Note: The field brace pairs (i.e. '{ }') for the above examples are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. The spaces represented in the field constructions are all required.
