    remove letter=X

    Close Command Prompt and restart your PC

    This is one of the known issues with the update as Recovery partition should be hidden and is accidentally assigned a drive letter.

    list volume

    remove letter="G"
    Replace G with the Recovery drive letter.

    Don't worry, removing a drive letter would not cause any harm to your device.

    "Logical partition" doesn't sound right for Windows.  You probably mean "Logical drive."


    Windows has "primary partitions" and "extended partitions." 


    A primary partition that has the operating system on it is an "active partition." 

    Windows permits up to 4 primary partitions on a "basic disk."

    An extended partition can be subdivided into "logical drives."  There can be an "unlimited" number of logical drives on an extended partition (until you run out of drive letters).

    See http://www.theeldergeek.com/hard_drives_01.htm

    As I understand your description, the system has a single hard drive that has both a primary partition (C:) and an extended partition.  The extended partition contains one logical drive (D:).

    Putting aside the issue of saving the data that is now in the D: drive, when you delete the D: logical drive, you just create "free space" within the extended partition (which doesn't have a name).  You then have to delete the extended partition in order to create "unallocated space" into which you can then expand the C: partition.

    Because you have not created the necessary space by deleting the D drive volume first.

    Here's a full procedure for you:-

    Repartitioning your hard drive

    You’ll need admin permissions for most of these tasks so it’s easiest to just log on as Admin now.

    I had no problems with this stage but it is a risky one.  I had a high level of confidence because of the preparations I’d made [installation USB made & tested, full system image made, all user files copied to an external drive].

    If anything goes wrong you can try again or can just get out of it all then boot from your Repair Disk / Windows 7 installation disk & restore the entire system from one of your images then start again.

    Type “management” into the Start menu search box then right-click on “Computer management” & select Run as Admin.  You need to do this even though you have logged in to an Admin account when starting Windows [this might be why you reported options being greyed out before].

    Click on the help icon on the toolbar [this is an additional Help file not the standard Windows Help - this file can also be found at C:\Windows\Help\mui\0409\diskmgt.CHM without opening Computer Management].  In the Help file, select Disk management, Manage basic volumes - read Extend a basic volume and Shrink a basic volume [including the final part about “Additional considerations”].  I could not find a section on deleting a volume but reading these other two sections made me comfortable enough to proceed. 

    1.     Partition the hard drive

    In Computer management itself, click on Disk management, then select your D:\ drive.  Right-click on it, select Delete volume....  This all went so smoothly that I’ve made no notes about any further dialog boxes for this stage of the procedure. 

    Still in Computer management, Disk management, select your C:\ drive.  Right-click on the C:\ drive & select Extend volume...  The system will then calculate what it can extend into & offer you the chance to do it.  It will show you the maximum you can extend it by.  Note that it will ask you how much to extend it by and it will calculate the resultant size – it’s not asking you what to extend it to.  I suggest that you extend it by enough to get the C drive to 100GB since you have the space to do that [I made mine 70GB but many people say that can be risky even though I have had no problems].  This all went so smoothly that I’ve made no notes about any further dialog boxes for this stage of the procedure. 

    Still in Computer management, Disk management, select the unused space where your D:\ drive used to be.  Right-click on it & select Create a new partition & let it use the whole of the available space.  Once again, this all went so smoothly that I’ve made no notes about any further dialog boxes for this stage of the procedure – I think that formatting the new partition then assigning a drive letter happened as part of this stage but I might have had to do them separately straight afterwards [again just by right-clicking then selecting from the context menu].  There is a guide at Partition or Volume - Create Newthat seems to say it is all done as part of  the creation stage.

    Close Computer management.

    You’ve now done all the hard work and have completed the stage that’s regarded as risky.

    Reboot.  This proves that the system has not been messed up.

    2.     Setup the data drive & your datafiles

    You are now going to create a set of folders on D:\ for your personal folders then tell Windows to use them. 

    Open Windows explorer twice.  Arrange the windows side-by-side.

    For every visible user folder within C:\Users\YourUsername -

    1  Select the folder, click in an empty area of the Windows explorer Address bar to see the actual folder name [rather than the name it appears to be –you will spot differences such as C:\Users\YourUsername\Documents appearing to be C:\Users\YourUsername\My Documents]

    2  Create an actual folder name on D:\ corresponding to the actual folder name you found in the Address bar.  Purely to suit my planned backup procedures, I also created D:\SystemSettings & put some of the new folders in there [see below] but still took care to make the folder name the actual name.

    If your system has Multiple users then you will want to do all this for each set of C:\Users\YourUsername folders possibly including the C:\Users\YourUsername\Puublic set.  You would have to have folder structures like D:\EachUserName\Documents & so on for this to be managed decently. 

    I decided on

    Documents            D:\Documents,

    Music                     D:\Documents\Music,

    Pictures                  D:\Documents\Pictures,

    Videos                    D:\Documents\Videos,

    Favorites                 D:\SystemSettings*\Favorites [these are IE Favorites],

    Links                    D:\SystemSettings*\Links [these are the entries within Windows Explorer, Favorites - the bit at the top-left of the Windows explorer navigation pane],

    Searches                 D:\SystemSettings*\Searches.

    [* It just suited my backup plan to separate these settings into a different folder group.]

     3 “Relocate” the user folders at C:\Users\YourUsername by right-clicking on each [e.g. C:\Users\YourUsername\Documents] selecting Properties, selecting the Location tab & pointing it at the new folder concerned.  The system will offer to move the existing contents of each user folder into the new Location but I chose not to do that as I intended to copy the user files back from my external drive later on.

    Do note that I did nothing to any of the hidden folders within C:\Users\YourUsername or to any of the things that look like shortcuts [they are not].

    4 Return to the folders on the D:\ drive.  Select each of Documents, Desktop, SystemSettings & so on then right-click, Share with..., Specific people, delete Everybody, set MyUserName – Full control, keep admin account as owner, consider any other permissions you want other accounts to have.

    I run my system with my own name as a Standard user and also have two Admin accounts – I considered repeating the folder creation procedure for the Admin accounts but there are never any datafiles in those accounts so I didn’t bother.

    Make a system image of the C:\ drive only.  Windows backup cannot do this if you also have a restore-to-factory partition - it will also include this other partition so your image will be unnecessarily bloated.  Also note that your previous system images or the restore-to-factory utility would put you right back at the start & would overwrite the partitioning work you have just done.

    Using Paragon Backup & Recovery Free Editionor another image-making backup application, make a system image of the D:\ drive only.  Windows backup cannot do this.  Whilst I’ve never had to use this image it is very small since the D:\ drive is virtually empty at this stage so I’ve kept it anyway.

    Then copy back the user files into their new location manually from the external drive you copied them to.

    3.     Future operations - backing up & other system safeguards

    3.1       Backing up in general

    How you do it is up to you but there is some advice here  Back up your programs, system settings, and files

    Many people reject Windows backup as being clumsy or unreliable & use third-party applications instead.  See any of the many threads in this forum to see people’s recommendations.

    In addition to conventional backups, I use the Win7 utility RoboCopy to make a mirror image of my D:\Documents & D:\SystemSettings folders.  I find this the most convenient form to have the files in as they are simply files and are completely free of any proprietary backup application formats.

    3.2       Weekly / monthly system images

    My Win7 Home Premium x64 system image takes up 40GB depending what mood it is in.  Each image takes about 20-30 minutes to complete. 

    I make images on my two external drives alternately & keep a couple of the latest images on each as well as images at key stages [such as those made immediately after completing the partitioning task].  Being so small, I have enough room not to be faced with a decision to delete an image I think of as being important.

    I always make an image before running Windows Update as I have seen so many people in this forum report problems afterwards.

    3.3       Admin accounts

    User profiles do get corrupted including Admin user profiles.  Just look at the number of questions in this forum from people whose login is failing & who cannot get into their systems.  I run daily tasks under a standard user account.  I have two Admin accounts [both with password protection].  I have UAC [see Control panel, User accounts] to the highest setting.  When I need to authorise an installation or other protected task, I alternate the admin account & password I enter.  I believe that this routine establishes very good protection against profile corruption since at least one of the admin accounts is liable to be working even if the other is corrupted so I can log into that one and sort out whatever problems have appeared.

    3.4       Restoring from a system image

    I have restored from images made since the partitioning task, restoration also takes about 20-30 minutes. 

    My images are made using Windows Backup System image & I can restore them to just that partition without disrupting the D:\ drive at all.  [I do my routine backup before this every time just in case but there’s never been a problem]. 

    I experimented with using another application to make my images but it did not have this partition-specific capability [it was a free edition that came with my new hard drive and I understand that if I’d bought the full version I could have this capability].
