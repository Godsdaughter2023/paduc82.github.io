gary's student say:

It does not loop because you are actually looping over columns rather than cells.  For example:

 

Sub RangeRover()
Range("N1:T7").Name = "MyNamedRange"
Dim c As Range
For Each c In Range("MyNamedRange").Columns(3)
    MsgBox c.Address
Next
End Sub

will produce a single output:  "P1:P7"

 

To loop over individual cells in that column:

 

Sub RangeRover2()
Range("N1:T7").Name = "MyNamedRange"
Dim r As Range
Set r = Range("MyNamedRange").Columns(3)
Set r = Intersect(Range("MyNamedRange"), r)

Dim c As Range
For Each c In r
    MsgBox c.Address
Next
End Sub

EX149947 - SMTP duplicate mail send issue (WORKAROUND) - use - Microsoft Community

ajhopen say:

EX149947 - SMTP duplicate mail send issue (WORKAROUND) - use CC and not TO if possible.

Hi Folks:

Apparently Microsoft is working on this problem (EX149947) but in testing my ASP.NET application I discovered that if you change the address object from the TO to CC you will NOT get duplicates.  

For example, this will give you duplicates under the current system state:

emailmessage.TO.Add(New MailAddress(ListBox2.Items(x).ToString, ListBox2.Items(x).ToString))

However, change the "TO" to "CC"

emailmessage.CC.Add(New MailAddress(ListBox2.Items(x).ToString, ListBox2.Items(x).ToString))

We no longer see duplicates (e.g., 2 email addresses in the TO field = 2 duplicates; 3 email addresses in the TO field = 3 duplicates).  

Hope this helps as a temporary fix. 

Thanks!
