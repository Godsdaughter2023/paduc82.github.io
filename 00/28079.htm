17063/74 (and up?): legacy file system filter - Microsoft Community

rseiler say:

17063/74 (and up?): legacy file system filter drivers breaking apps and causing GSODs

The purpose of this post is to tie together what's breaking for this specific reason and to hopefully learn from Microsoft whether the change is intentional or not. This way, vendors will know whether they have a long-term problem requiring them to rewrite drivers, or if this is an acknowledged problem and is something due to be fixed before 1803 goes wide.

Example 1:

Macrium Reflect (https://aka.ms/Mjtaqd):
-Users attempting to update to 17063+ will GSOD with INACCESSIBLE BOOT DEVICE.
-Reason: Macrium Reflect Changed Block Tracker (CBT) driver (MrCbt.sys), a "legacy" file system filter driver.
-Workaround (for 17063-untested on 17074).

Example 2:
Mozy Backup (https://aka.ms/Wxbijz):
-Users attempting to update to 17063+ will GSOD with SYSTEM_SCAN_AT_RAISED_IRQL_CAUGHT_IMPROPER_DRIVER_UNLOAD.
-Reason: Mozy Change Monitor Filter driver (mozyfilter.sys), a  "legacy" file system filter driver.
-Workaround: None that will allow continued use of the product. Uninstall it in the meantime, or use Autoruns to uncheck mozyfilter.sys.

Example 3:
Raxco PerfectDisk 14 Pro:
-Doesn't interfere with Windows installation, but no longer works
-Reason: Most likely PerfectDisk OptiWrite Filter Driver (PDFsFilter.sys), a "legacy" file system filter driver.
-Workaround: The same as Example 1, which is why I'm supposing that this is really the same issue.

Apparent reason this is all happening: As mentioned in the "Workaround" link above:

Windows 10 version 1607, introduced a new registry entry to block legacy drivers - IoBlockLegacyFsFilters, and this registry entry had to be explicitly set to exclude legacy drivers from loading.  However, the default value for this new registry entry has been changed from '0' (Not blocked) to '1' (blocked) in the Insider Preview 17063 build of Windows 10.  This means that that no 'Legacy' kernel drivers, except for Microsoft's own legacy drivers, are loaded at boot time. Unfortunately this causes a fatal error when Windows starts if any legacy driver is designated as a boot start upper filter driver.

Way to test if your system has legacy filter drivers (from admin CMD*):

fltmc filters

*If a particular driver is legacy, you'll see <Legacy> in the "Frame" column.

Does anyone have further examples?

hansv mvp say:

Applying the custom format 00\:00 does NOT make the values you enter time values. They are still ordinary numbers such as 438 and 616. 616 - 438 = 178.

If you want the difference in minutes, you can use

=60*(INT(B1/100)-INT(A1/100))+MOD(B1,100)-MOD(A1,100)

Or see http://www.cpearson.com/excel/datetimeentry.htm on how to enter 438 and have it converted automatically to a real time value 04:38.

gord dibben say:

Formatting simply changes how data looks.

It does not change the underlying value of the data.

Time must be entered as time 06:16 for 6:16 AM or  16:45 for 4:45 PM

The only way to skip entering the colon is to use VBA

Chip Pearson has such code.

http://www.cpearson.com/excel/DateTimeEntry.htm
