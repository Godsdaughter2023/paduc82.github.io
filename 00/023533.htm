    john - project say:

    CathiB,

    Why bother with VBA? When you create your custom view table, use the Organizer to transfer it to your Global. Then if someone messes with your custom view fields, you can simply use the Organizer to delete the modified version in the file and either replace it with the one in your Global, or simply select your custom view from the ribbon and Project will automatically use the one from your Global.

    However, if you still like the VBA idea, here is a stripped down, more efficient version of your code. Basically create all your fields with TableEditEx methods in sequence with the last one catching any common elements like "ShowInMenu" and "LockFirstColumn". Once you've defined all the fields, then apply the new table. And note, you only need to specify a date format for fields that are date fields, which none of your are.

    Sub NewTbl()

    TableEditEx Name:="MEL Gantt Table", TaskTable:=True, overwriteexisting:=True, _
        FieldName:="Name", Width:=50, Align:=0, RowHeight:=2, ColumnPosition:=0, AlignTitle:=0
    TableEditEx Name:="MEL Gantt Table", TaskTable:=True, overwriteexisting:=True, _
        FieldName:="Resource Names", Width:=13, Align:=0, RowHeight:=2, ColumnPosition:=1, AlignTitle:=0
    TableEdit Name:="MEL Gantt Table", TaskTable:=True, overwriteexisting:=True, _
        FieldName:="Duration", Width:=10, Align:=2, RowHeight:=2, ColumnPosition:=2, AlignTitle:=2, _
        ShowInMenu:=True
    TableApply Name:="MEL Gantt Table"
    End Sub

    Hope this helps.
