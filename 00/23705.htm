Switching Microsoft Accounts with paid services in them may be 'tricky'. Many consumers had their own share of troubles contacting the right support for this. Before you fully decide to move all those services to another Microsoft Account, I can suggest an alternative on what you want to do.

If you want to delete the old Hotmail account because you do not like the name anymore, or you registered the email address to 3rd party services that you no longer need, you can simply rename the account instead.

Microsoft features the use of an "Alias". It's basically another name for your account. You can have multiple email address, but it still refers to one Microsoft Account, that's used to access the online services. It makes it very convenient for users who already owns an email address of their own, but don't really want to get a new one just to access the services that Microsoft provides.

By adding in an Alias for your existing Hotmail account, you'll essentially be getting a new name for it. Then if you delete the old Hotmail address, you've finished renaming the account. Any service that sends an email to the old address will be told that the account does not exist anymore. And you can have a professional name that you want for it, if you had created it with a 'silly' name.

1. Login your account at https://account.live.com/

2. Click on Your Info on top of the page.

3. It will display a brief description of your account. Click on Manage your sign-in email or phone number

4. You'll be at the Manage your account aliases page. Click on Add email alias.

5. You have two options at this point. You can either: Get a new address provided by Microsoft that ends in @outlook.com, or use an existing email address that you already have that you have not yet registered as a Microsoft Account.

verification email sent to the original inbox of the account.

7. Click on Remove next to the old Hotmail address to fully remove it on your account.

The new Win 8 'Refresh" feature is good, when you learn ALL about it. Unfortunately MS has not bothered to publicize the second half of the Refresh process, and most of the reporters have sheepishly followed in MS's footsteps. 

Refresh actually has a complimentary program called RecImg.EXE  RecImg is used to create a new "image file" that is used in the Refresh process.  Ideally what you should do is after you install a new "traditional desktop" type application (like MS Office!) you should apply all of the outstanding Windows Updates and any configuration personalization you normally do.  Then you should run RecImg to create a new image file that includes the newly installed and configured program.  That way the next time you use Refresh the application (like Office), will also be restored!

The problem you probably encountered is that you used the factory default "image" file which only included the factory trial of 365, not your installed copy.

Win 8 Total Backup System: File History + RecImg + Refresh + SlimImage

Win 8 has refined your backup and restore options. Before Win 8 you could create an Image Copy backup of the full drive (or selected directories), but it was hard to separate backups of the OS, Applications and personal Data unless you created a separate

Out of the box, MS has been advertising the “Refresh” and “Reset” options. Personally, I think they missed a valuable selling point by not mentioning RecImg.EXE in conjunction with Refresh. It removes my main concern/objection to Refresh.

Reset is equivalent to using the “Recovery Partition” on previous versions of Windows. The factory would create a “Recovery Partition” that you could use to “reset the computer to factory fresh”. It would typically include all of the factory “crapware” and it would wipe out all of your data files and installed applications.  A less than desirable outcome but an appropriate reaction to a bad malware infection.  The “recovery partition” type of restore typically had to be accessed at boot time while Reset can be accessed from within Windows 8.

Refresh is roughly equivalent to an image copy backup.  A Reset will restore a backup of the OS and Metro applications. It will wipe out “legacy” / x86 desktop type applications BUT it will not affect your personal data files, which is much better than a Reset.  But there is still that issue of wiping out your installed desktop applications. I do not have documented confirmation, but I’m guessing that Metro apps are treated like personal data and those directories/folders are simply not reset back to the old condition.

RecImg.EXE is a backup tool that ships with Windows 8. It creates the backup “image” that is used by the Refresh tool. This image that was created at the factory is why using Refresh will wipe out installed desktop programs in the “C:\Program files” and “C:\Program Files (x86)” folders because they haven’t been backed up. However, using RecImg.EXE you can create new images that will include recently installed desktop applications.

RecImg Manager is a third party tool the provides a GUI to access the RecImg.EXE command line tool and backup files.

File History is a new Win 8 feature. It is an extension to the “Previous Versions” and “Volume Shadow Copy” features. File History continuously monitors files stored in Libraries, Desktop, Contacts, and Favorites folders. It runs on a predetermined schedule, and when it detects changes in any file, it makes a backup of that file to an External drive. It isn’t enabled by default.
