dale howard [mvp] say:

Adam --

You are describing the Task Type feature in Microsoft Project.  The default Task Type for every task in the software is Fixed Units, which means that Microsoft Project "locks" or fixes the Units value you enter for each resource assigned to tasks.  If you change the Duration of a task, the Units value is locked, therefore, the software MUST recalculate the Work because it is the only number left that can be recalculated.  What I believe you want is Fixed Work tasks in your projects.  By using the Fixed Work task type, Microsoft Project will "lock" or fix the Work value you enter for each resource assigned to tasks.  To change the Task Type for every task to Fixed Work, click the Select All button to select every task in the project and then click the Information button in the Task ribbon.  Click the Advanced tab in the Multiple Task Information dialog, click the Task Type pick list and select Fixed Work, and then click the OK button.  You can also change the default Task Type for all new tasks in each project and in all new projects by clicking File > Options > Schedule.  In the "Scheduling options for this project" section, set the Default Task Type value to Fixed Work.  This will change the setting for the current project.  Then click the "Scheduling options for this project" pick list and select All New Projects.  Set the Default Task Type value to Fixed Work as well, and then click the OK button.

By the way, one bad habit you are already forming is you are manually typing Start dates and Finish dates.  If you need to enter when a task actually finishes, apply the Tracking table and then enter the date in the Actual Finish column.  You can also enter Actual Start dates in the table as well, along with Percent Complete values.  Hope this helps.

scottgem say:
Why not use a DSum()?

tom van stiphout say:

The reason would be that you have not set a reference to that ADODB library.

Fix this by selecting the reference in Code window > Tools > References > Microsoft ActiveX Data Objects [version] Library

ordnance1 say: Thank you

ken sheridan say: The error is almost certainly that identified by Tom, but even if you create a reference to the ADO object library, the code won't do anything as it is not assigning the string expression as the Source of the recordset object's Open method.

But as Scott implies, you can simply, and more efficiently, call the DSum function, e.g.

    Dim TotalDaysOff As Integer

    TotalDaysOff = DSum("Change","tblDaysOff")
    Debug.Print TotalDaysOff
