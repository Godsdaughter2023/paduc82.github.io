jay freedman say:
Check boxes are really the wrong kind of object for this, because it would require a macro to enforce the "only one per group" rule. That gets you into all sorts of difficulties you don't want to deal with.

If everybody who uses these documents will have Word 2007 or later, then you can use dropdown content controls, which don't involve restricting or protecting the documents. If the documents have to be used in Word 2003 or earlier, you must use dropdown form fields instead, because those versions (even with the Compatibility Pack) can't use content controls. If you use form fields, the document must be protected to make them work, but you can leave other parts of the document unprotected.

In either case, the first thing to do is to open the Word Options dialog, go to the Popular tab of the dialog, and check the box for "Show Developer tab in the Ribbon". The buttons for inserting content controls and form fields are in the Controls group on the Developer tab.

If you're making Word 2007-only documents, click the leftmost button in the second row in the Controls group, whose tag says "Drop-Down List". Then, in the right-hand half of the Controls group, click the Properties button. In the Properties dialog, click the Add button and type in the word Yes. Repeat this for No, Maybe, and Discuss. Finally, click OK.

If you're making Word 2003-compatible documents, click the fourth button in the second row of the Controls group, whose tag says "Legacy Tools", and click the third button in the top row of the menu, the Drop-Down Form Field. Click the Properties button. In the dialog, click the the "Drop-down item" box and type the word Yes, then click the Add button. Repeat this for No, Maybe, and Discuss. Finally, click OK.

To make protected areas in the document around the form fields and other areas that are not protected, you must insert section breaks between the areas. The smallest part of a document that can be either protected or not protected is a section, which is the area between section breaks (or at the beginning or end of the document). Usually you'll want to use "continuous" section breaks, which don't force a new page.

Once all the form fields and section breaks are in place, click the Protect Document button on the Developer ribbon (and there's another one on the Review ribbon that does the same thing). This opens the Restrict Formatting and Editing pane on the right. Click the check box under Editing Restrictions. Set the dropdown under the check box to "Filling in forms". Then click the "Select sections..." link below the dropdown, which opens a dialog with a list of the document's sections. Uncheck the sections that will be unprotected, leaving the checks for the sections to be protected. Finally, click the Start Enforcing button and click OK in the password dialog (you can leave the boxes blank if you don't want a password).

allan mej say:

Hi,

It is possible that your PC's Print Spooler Service is not working correctly, preventing you from printing. To troubleshoot this, we recommend clearing spooler files and restarting the Spooler Service. To do so, kindly follow these steps:

  1. In the search box on the taskbar, type Services, then select the search result of the same name.
  2. In the Services list, double-tap (or double-click) Print Spooler. Select Stop > OK.
  3. In the search box on the taskbar, type %WINDIR%\system32\spool\printers, select the file folder in the search results, and then delete all files in the folder.
  4. Search for and open Services again
  5. In the Services list, double-tab (or double-click) Print Spooler.
  6. Select Start, and in the Startup Type list, make sure that Automatic is selected, and then select OK.

Let us know how things go.
