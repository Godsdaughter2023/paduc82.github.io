As rightly mentioned by you, when you run the macro, it will open windows explorer for the user to select the "Common Parts.xlsx" file   (which the macro finds that it is not yet opened).

Ideally you must first open the "Common Parts.xlsx" file through the macro

you may use the following code for this...

Set fs = CreateObject("Scripting.FileSystemObject")
ThePath = fs.GetAbsolutePathName(CustPartsFFLNm)
FileToOpen = CustPartsFFLNm
Set CustPartsWRKBK = Workbooks.Open(FileName:=FileToOpen, UpdateLinks:=xlUpdateLinksNever)

Here CustPartsWRKBK is a variable name that we are setting for the "Common Parts.xlsx" workbook while opening it. This name can be used in the entire macro for referring the xlsx file.

Here CustPartsFFLNm is a variable name used to store the file  "Common Parts.xlsx" with its complete path.

The complete path for this workbook can be like 

C:\Users\285015\Desktop\AA

Here 285015 is the user-name (or the user-id) which gets created when a system is formatted or created.  "AA" is a folder name on the desktop.

An alternative approach is to use a function to read the desktop folder location from the PC e.g. as follows

SpecFolder is the Desktop path

Option Explicit
Public Declare Function SHGetSpecialFolderLocation _
                          Lib "shell32" (ByVal hwnd As Long, _
                                         ByVal nFolder As Long, ppidl As Long) As Long

Public Declare Function SHGetPathFromIDList _
                          Lib "shell32" Alias "SHGetPathFromIDListA" _
                              (ByVal Pidl As Long, ByVal pszPath As String) As Long

Public Declare Sub CoTaskMemFree Lib "ole32" (ByVal pvoid As Long)
Public Const CSIDL_DESKTOP = &H0        'Desktop
Public Const MAX_PATH = 260
Public Const NOERROR = 0

Public Function SpecFolder(Optional ByVal lngFolder As Long = &H0) As String
Dim lngPidlFound As Long
Dim lngFolderFound As Long
Dim lngPidl As Long
Dim strPath As String

    strPath = Space(MAX_PATH)
    lngPidlFound = SHGetSpecialFolderLocation(0, lngFolder, lngPidl)
    If lngPidlFound = NOERROR Then
        lngFolderFound = SHGetPathFromIDList(lngPidl, strPath)
        If lngFolderFound Then
            SpecFolder = Left$(strPath, _
                               InStr(1, strPath, vbNullChar) - 1) & Chr(92)
        End If
    End If
    CoTaskMemFree lngPidl
lbl_Exit:
    Exit Function
End Function

Note that OWA will only show the PF folder structure when you are browsing for a folder to view.  Once you select a folder, it will only ever show the contents (items) in that folder, if any exist.  OWA won't show subfolders in the view.

It's just the way it works :(
