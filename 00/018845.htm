daniel pineault say:

The error is merely complaining about the use of MS Access Constants that VBS does not recognize.  So you need to either define the constants or change them for their numerical values.

 

Const acExport = 1

Const acSpreadsheetTypeExcel12 = 9

 

As you can also see, the acSpreadsheet is a class and not a valid input variable.  You need to use a valid member of the class:

 

...

Const acSpreadsheetTypeExcel9 = 8

Const acSpreadsheetTypeExcel12 = 9

Const acSpreadsheetTypeExcel12Xml = 10 '(xlsx)

...

 

So you could do something like:

 

appAccess.DoCmd.TransferSpreadsheet 1, 8, "Inv_W_Expiry_Dates", "S:\Japan-Acct\Transfer\" & strFileName, True, ""

 

or (taking some code from the net and tweaking it to your description)

 

Dim appAccess
Dim strDB
Const acExport = 1
Const acSpreadsheetTypeExcel12Xml = 10

 ' strings for database path and database name

 strDB = "C:\Databases\tests.mdb"

 ' Create new instance of Microsoft Access.
 Set appAccess = CreateObject("Access.Application")

 appAccess.OpenCurrentDatabase strDB
 'export
 appAccess.DoCmd.TransferSpreadsheet acExport , acSpreadsheetTypeExcel12Xml, "Switchboard Items", "C:\Databases\testme2.xlsx",

True
 appAccess.CloseCurrentDatabase
 Set appAccess = Nothing

jay freedman say:

Going back to your original post, the jargon term "custom UIXML" refers to a file containing XML (eXtended Markup Language) statements about customization of the user interface (UI). That should be in a file named Word16.customUI in the folder C:\Users\<your name>\AppData\Local\Microsoft\Office.

The AppData folder is marked as hidden, so in Windows Explorer you'll need to go to the Tools > Folder Options dialog, click the View tab of the dialog, and click the "Show hidden files, folders, and drives" option.

Shut down Word. Go to the folder named above, and move (not delete) the Word16.customUI file to another folder where you can find it later. Then go to Control Panel > Programs and Features and do an online repair of Office 365, which should write a new, correct file in its place.

If that doesn't help, it may be possible to make manual repairs to the existing customUI file (this is not a simple matter), or to replace it with one from an undamaged copy of Office 365.
