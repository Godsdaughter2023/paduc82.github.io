Windows Backup was interrupted by a Windows Update.

I've started another backup and was instructed to label the backup disc as #1. My two questions are: Is there a way to tell how many discs i'm going to need before i run a backup. and 2. Is this picking up where the failed backup left off or am i going to need another 42 DVDs?

Thanks for your help! Lisa

a. user say:

Vista doesn't tell you how many disks will be required - that isn't one of its features (but it's a good idea).  You just have to check out what you've selected to backup, total it, divide it by the size each disk can hold, and come up with your best guess.

I believe it will start over from the beginning - not pick up where it left off (I don't think that is possible with any version of Vista or any type of backup you choose).  Vista doesn't keep a record of what has and hasn't been backed up that can be used to pickup where it left off (well, it has a record but it can't be accessed for this purpose) - this is especially true if you've already started another backup (though I don't think it would have been possible to pick up where you left off in any case).  You may need more than another 42 disks because it hadn't finished and we don't know how many it will ultimately required - we can say you will need AT LEAST another 42.

If you're using that many disks, may I suggest you get an external USB hard drive and use that instead for your backups - it will be a lot easier and you won't need to keep changing disks.  They aren't that expensive these days - just make sure it is large enough to save everything you want to backup and give yourself a lot of extra space because you will use it up quickly.

Sorry if that isn't what you wanted to hear, but Vista's Backup program is not the best available and has many limitations and problems (but it's free and it basically works so we use it).

BTW, next time, turn off Automatic Updates AND deactivate your security software before you do the next backup.  Turn them back on when the backup is done.

Good luck!
