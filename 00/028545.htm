john - project say:

Lu,

Just for reference, damaged branches should be cut off at the branch crown and not "pulled off" :-)

Okay, time for a little primer on master projects. There are two kinds of master projects, dynamic and static. By default Project creates a dynamic master when a user inserts subprojects into a master. The option I was referring to is in the lower right corner of the Insert Project window (i.e. "link to project"). What that means is, the subprojects do not actually become part of the master, rather, the master contains pointers (i.e. links) to each individual subproject. Changes made at master level are reflected in the subproject and vice versa (i.e. the files are dynamically linked). However to preserve those changes, the user must save both master and subprojects when closing.

If the user de-selects the option "link to project" when the master is created, a static master will be created. A static master basically copies the contents of each subproject into the master such that the master is a snapshot in time of each subproject when the master was created. The structure is not linked so changes made in either the master or the original subproject files are not reflected in the other. A static master is a single independent file and must be recreated any time the user desires to "update" the subproject data. This is easy to do by recording a macro the first time the static master is created.

Creating a dynamic master does not create any links between tasks in the inserted subprojects. To create links between tasks in one subproject with tasks in another subproject, a separate process is required and we discussed that in a previous thread. The option for "links between projects" applies to these inter-project task links and not to the master/subproject link structure.

It is easy to tell if the master is dynamic or static by looking at the ID field. If the ID field is continuous throughout the master, then it is a static master. If the ID sequence repeats for each inserted subproject, then it is a dynamic master.
