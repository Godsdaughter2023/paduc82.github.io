Bug: Snap grid spacing is incorrect & does not match visible grid pitch (roundoff error)

There is a bug in the implementation of the "snap to" grid function In powerpoint 2010.  The spacing of the snap grid is slightly incorrect due to rounding of the grid pitch to three decimal places, and so does not match the visibly displayed grid (the one shown when "display grid on screen" is enabled.)  The visible grid is correct.

For example, when grid spacing is set to 1/24", the snap grid pitch is rounded from 0.04166... to 0.042.  This is a relative error of one part in 125, so for every 126 visible grid pitches there are only 125 snap grid pitches.

Steps to reproduce:   (I have a slide that shows this problem, but there doesn't appear to be any way to upload it here.)

1) Set grid to 1/24, set grid visible
2) Create 126 short vertical lines, on adjacent snap grid points
3) Group them into a "ruler", and turn snap to grid off.
4) Align the "ruler" to a major division of the visible grid.  Notice that there are only 125 visible grid points spanning the same length as the 126 lines in the ruler.
