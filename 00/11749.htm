pamcaswell say:

Use the style separator and two paragraphs in two different  paragraph styles styles.   After you have created  the caption and are back in the text, type the explanation in a separate paragraph and  apply  a different paragraph style to it. — You can create one that  looks just like the caption style if you want it to look the same.

mike h.. say:

Hi,

 

I'd adopt a different approach and extract all the names with NO to a new list and then use that list for your data validation list like this.

 

Say the above columns are in A, B and C. use this ARRAY formula to extract all the names with a no in Col C

 

 

=IFERROR(INDEX($B$2:$B$11,MATCH(0,IF($C$2:$C$11="Yes","",COUNTIF($H$1:H1,$B$2:$B$11)),0)),"")

 

Put this in a cell and ARRAY enter it and drag down as far as you want and it will extract all the NO names and when that's done it will return blanks cells.

 

Note the underlined bit in the formula. This reference must be the cell above the cell with the formula in so in this case I used H2.

 

 

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

In your very first post you said, "I sent an email out to a group of recipients".  First you say it was a group then you say it wasn't.  Please make up your mind.

The GAL is the "Global Address List".  In an Exchange environment, it's usually what you see when you open the Address Book interface.  It's the company-wide list of addresses from the Active Directory where people's accounts are defined.

You don't say whether or not it's an outside address (outside of your domain), but I think you should enlist the aid of your Exchange admins.  In an Exchange environment, it is often Exchange that sends messages to the outside world and Exchange that stores the messages n the Sent Items folder.  Perhaps they can help you trace the flow of the message.  I have never seen the behavior you describe and I've bounced many messages in my time.
