fraser xu msft say:

Hi Wed532,

We may have some misunderstanding. To ensure we are on the same page, I’d like to confirm whether you are referring the Labels in this article: Implement Records Management.

If so, please note that this label feature is based on Document Library. Although the labels will move with documents, but the requirement of adding labels are only stay in the Document Library which have enable this feature.

Let’s say there is a Document Library A that has enabled the Labels feature and a Document Library B that doesn’t. When a user edits a document in Library A, it will require the user add a Label in the document and user cannot delete the label. After that, if the user moved the document to Library B, the label will remain. However, the label can be deleted now.

After you have declare a document as record, please go to Site Settings> Site Collection Administration> Record declaration settings> In Record Restrictions section> Select “No Additional Restrictions”. After that, you can move the record to other place with Workflow/ Custom scripts. Even more, you can configure Content Organizer feature to move the record: Configure the Content Organizer to route documents.

steve rindsberg say:

I'm about to release an add-in for this, but it'll only be available for Windows PPT, at least for now.

But you can manually do a lot of what it does:

Save a copy of your original presentation.  Let's call the original O and the copy C.

Open C

Change it to 16:9

Leave C open while you also open O and arrange the view so the two presentations are side by side.

Step through the presentations a slide at a time.  On each slide, delete any shape/picture in C that's gotten distorted, select the same shape in O and copy/paste it into C.

A bit tedious but you get into a rhythm and it goes by pretty quickly.  And definitely it beats re-creating/recropping all those images.
