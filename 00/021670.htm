Current event fires twice in Subform - Microsoft Community

jguen say:

Current event fires twice in Subform

I have a Form with a Subform on it.  There is one linking field.  The Form data source is a Table.  The Subform data source is a single-table query.  Whenever the Form's OrderBy property is set, moving to a new record on the Form causes the Subform's Current event to happen twice.  The Subform is also redrawn twice.  It is very noticeable because there are picture fields on the Subform.  Is there a way to get the Subform to only redraw once when moving to a new record on the Form if the Form's OrderBy property is set?  I have tested using different combinations of Tables and Queries as the data sources, and it doesn't make a difference.

Jay

ken sheridan say: Have you tried basing the parent form on an ordered query rather than using the form's OrderBy property to sort it?  If you want to dynamically change the form's sort order at runtime you can always amend its RecordSource property in code.

I've always used a query to order a form rather than its OrderBy property, but note that you should not do this for a report, whose internal sorting and grouping mechanism should always be used.
Ken Sheridan, Stafford, England

jguen say:

Intercepting the ApplyFilter event on the Form and changing the RecordSource property instead of the OrderBy property does prevent the double redraw of the Subform.  It is kind of a pain, though, even for one OrderBy field, because it sets the OrderBy property to [FormName].[FieldName].  Therefore, you have to do some string manipulation to get just the field name before you append it to the RecordSource query.  Ordering on multiple fields would get even more complicated.  And, you have to create a way for the user to remove the sort order.

Jay

Important: Please back up the registry before making changes to registry.

1. Press Win + R > type regedit and hit OK.

Important This procedure will cause you to lose all local changes. Therefore, we recommend that you save a local copy of all pending files before you perform these steps.

1. Check for pending files. To do this, click the Office Upload Center icon in the notification area of the Task Bar.
2. Wipe the entire Office file cache. To do this, follow these steps:

  a. Close all Office programs.

  b. In the notification area, right-click the OneDrive icon, and then click Exit.

  c. Press Ctrl+Alt+Del to start Task Manager.

  d. On the Details tab, select all running processes that have any of the following image names, and then click End Process:

MsoSync.exe
CsiSyncClient.exe
MsoUC.exe
Groove.exe
Lync.exe
OneDrive.exe

  e. In Windows Explorer, locate the following folder:

%SystemDrive%\Users\%UserName%\AppData\Local\Microsoft\Office\16.0\

  f. Rename the OfficeCacheFolder folder. For example, rename it as OfficeCacheFolder_old.

Note By renaming the folder, you preserve document data. However, you can also delete the folder to recover disk space if the folder is extremely large. Both methods reset the document sync state.

  g. Restart OneDrive and other Office programs.
