scott mcdaniel say:

You can use any field in the Report's recordsource for filtering. If you're using the Openreport method to open your report, you could do this:

dim sql as String

sql = "[Assigned Date}=#" & Me.txYourDate & "# OR [Approved Date]=#" & Me.txYourDate & "# OR [Completed Date]=#" & Me.txYourDate & "#"

DoCmd.OpenReport "YourReportName", acViewPreview, , sql

You use the OR keyword to show any record that matches ANY of the criteria.

You would use the AND keyword (instead of OR) to show records which match ALL of the criteria.

ken sheridan say:
sql = "[Assigned Date}=#" & Me.txYourDate & "# OR [Approved Date]=#" & Me.txYourDate & "# OR [Completed Date]=#" & Me.txYourDate & "#"

Just one comment.  This would fail in many parts of the world, which do not use the US short date format.  I would always recommend the use of the ISO standard for date notation when building a date literal:

Dim strISODate As String
Dim strCriteria As String

strISODate = "#" & Format(Me.txYourDate,"yyyy-mm-dd") & "#"

strCriteria = "[Assigned Date] = " & strISODate  & _
    " Or [Approved Date] = " & strISODate  & _
    " Or [Completed Date] = " & strISODate

DoCmd.OpenReport "YourReportName", View:=acViewPreview, WhereCondition:=strCriteria

Many developers use a simple function which accepts a date in the local short date format as a string, and returns a string expression for a date literal formatted to the ISO standard, though I tend to do it on the fly as above.

ken sheridan say:

You don't need to create a new report; you can filter a report to those dates which fall between two dates entered as parameters, or within a particular week or month.

 

1.  To filter by means of two combo boxes txtStartdate and txtEndDate say, the code would be like this:

 

Dim strRange As String

Dim strCriteria As String

strRange = "Between "#" & Format(txtStartDate ,"yyyy-mm-dd") & _

    "# And #" & Format(txtEndDate ,"yyyy-mm-dd") & "#"

strCriteria = "[Assigned Date] " & strRange & _
    " Or [Approved Date] " & strRange & _
    " Or [Completed Date] " & strRange

DoCmd.OpenReport "YourReportName", View:=acViewPreview, WhereCondition:=strCriteria

 

2.  To filter by a week I'd normally use a combo box in which to select a week by a 'week-starting' date.  You'll find an example in Timesheet.zip in my public databases folder at:

 

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

 

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

 

In this little demo file the frmWeeklyTimeSheet form has a combo box in which to select a week, whose RowSoutrce is:

 

SELECT DISTINCT CDate(WeekStart(1,[WeekStartingCalendar].[WeekStarting])) AS WeekStarting

FROM WeekStartingCalendar

ORDER BY CDate(WeekStart(1,[WeekStartingCalendar].[WeekStarting]));

 

This uses a table WeekStartingCalendar, which is simply a table of all Sunday dates.  You'll find a means of building an auxiliary calendar table like this (and others) in Calendar.zip in my same OneDrive folder.

 

The TimeSheet demo also contains the following function:

 

Public Function WeekStart(intStartDay As Integer, Optional varDate As Variant)

 

    ' Returns 'week starting' date for any date

   

    ' Arguments:

    ' 1. intStartDay - weekday on which week starts, 1-7 (Sun - Sat)

    ' 2. vardate - optional date value for which week starting

    '   date to be returned.  Defaults to current date

   

    If IsMissing(varDate) Then varDate = VBA.Date

   

    If Not IsNull(varDate) Then

        WeekStart = DateValue(varDate) - Weekday(varDate, intStartDay) + 1

    End If

   

End Function

 

So by adding this to a database you can use a form containing a combo box of week-starting dates to filter a report to rows in the selected week, assuming you've used Sundays for the week-starting dates, with:

 

Dim strCriteria As String

Dim strWeekStarting

strWeekStarting = "#" & Format(cboWeekStarting ,"yyyy-mm-dd") &  "#"

strCriteria = "WeekStart(1,[Assigned Date]) = " & strWeekStarting & _
    " Or WeekStart(1,[Approved Date]) = " & strWeekStarting & _
    " Or WeekStart(1,[Completed Date]) = " & strWeekStarting

DoCmd.OpenReport "YourReportName", View:=acViewPreview, WhereCondition:=strCriteria

 

3.  To filter by a month you need to specify the Year and Month, so I'd use two combo boxes, cboYear and cboMonth.  The former would simply be a list of years; the latter would have the numeric month values as its hidden first column, and the text month names as its visible column, so its RowSource would be a Value List of:

 

1;January;2;February;3;March;4;April;5;May;6;June;7;July;8;August;9;September;10;October;11;November;12;December

 

Its other properties would be:

 

BoundColumn:   1

ColumnCount:    2

ColumnWidths:  0cm

 

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

 

You can then filter the report with:

 

Dim strCriteria As String


strCriteria = "(Year([Assigned Date]) = " & Me.cboYear & _

    " And Month([Assigned Date]) = " & Me.cboMonth & ")"& _

    " Or (Year([Approved Date]) = " & Me.cboYear & _

    " And Month([Approved Date]) = " & Me.cboMonth & ")" & _

    " Or (Year([Completed Date]) = " & Me.cboYear & _

    " And Month([Completed Date]) = " & Me.cboMonth & ")"


DoCmd.OpenReport "YourReportName", View:=acViewPreview, WhereCondition:=strCriteria

 

An alternative way to filter by Month is to compute the dates of the first and last days of the month form the Year and Month values by means of the DateSerial function and use a Between....And Operation in the filter.

 

marshall barton say:

The standard way to filter a report according to various criteria specified by the use is to have a form with two text boxes for the date range start and end dates and another text box for the name.  Then the form can have a command button with VBA code to open the report in its Click event procedure.

There should be only one report based on the data table (or a query if more than one table is involved).

All the filtering will be done using the OpenReport method's WhereCondition argument that is constructed using VBA code like this untested typo infested air code:

Dim strWhere As String

Dim strStart As String

Dim strEnd As String

If Not IsNull(Me.txtStart) And Not IsNull(Me.txtEnd) Then

   strStart = Format(Me.txtStart, "\#yyyy-m-d\#")   ' both dates entered

   strEnd =  Format(Me.txtEnd, "\#yyyy-m-d\#")

   strWhere = " And ([Assigned Date] Between " & strStart & "And "& strEnd _

                     & " Or [Approved Date] Between " & strStart & "And "& strEnd  _
                     & " Or [Completed Date] Between " & strStart & "And "& strEnd & ")"

ElseIf Not IsNull(Me.txtStart) Then

   strStart = Format(Me.txtStart, "\#yyyy-m-d\#")  ' only start date entered

   strWhere = " And ([Assigned Date] = " & strStart _

                     & " Or [Approved Date] = " & strStart _
                     & " Or ([Completed Date] = " & strStart & ")"

End If

If Not IsNull(Me.txtmanager) Then

    strWhere = " And (Assigned = """ & Me.txtmanager & """ " _

                      & " Or Approved = """ & Me.txtmanager & """ " _

                      & "Or Completed = """ & Me.txtmanager & """ " & ")"

End If

DoCmd.OpenReport "name of the report",acviewPreView, WhereCondition:= Mid(strWhere, 6)

And that should take care of everything I think you wanted.  If not, maybe you have detected a pattern to this kind of code and might be able to adjust it for more criteria.  But, if you need help with more complicated filtering come on back.

doug robbins - mvp office apps & services (word) say:

A text box has internal margins, which by default are:

Left and Right 0.25 cm
Top and Bottom 0.13 cm

They can be modified via the Text Box tab of the Format Text Box dialog.

The dimensions (Height and Width) of the text box can be set to Absolute values via the Size tab of the same dialog.

The precise position of text box can be set in the Layout dialog which is accessed by clicking on the Advanced button on the Layout tab of the Format Text Box dailog.
