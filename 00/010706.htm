You should be able to do the following:

bob jones aka: cybertaz say:

It sound to me like you're referring to a Heading, not a Header. It may seem pedantic, but there is a definite distinction between the two. In either case, though, the best suggestion is to avoid the use of  Text Box, altogether. As with any other type of object, Text Boxes are foreign bodies interjected into or attached to the text flow, which makes them difficult to control. PowerPoint is a totally different type of program which is object-oriented.

If it truly is a Header it should be contained the Header/Footer layer of the document. That would prevent it from moving in the first place.

If it's actually a Heading there are 2 other approaches;

  1. For a Heading that pertains to a portion of the document containing the Table, the Heading should be regular text in a paragraph which precedes the Table. Format it using a Heading Style as you see fit. It will remain in its respective location regardless of how long the Table may be.
  2. For a Heading that pertains specifically to the Table the Heading should be in the first row of the Table, itself. If necessary, Merge the cells on that row & format the Heading as necessary. Also, if the Table will extend beyond 1 page you also have the option of applying the Table property to Repeat as header row [yeah, even Microsoft misuses the terminology :-)] at the top of each page so that the Table's caption will do so automatically.

Footnote: In my 30 years with Word I've found that the fewer objects one inserts into a document, the better. Inserting them when there is no need to do so is just asking for trouble.
