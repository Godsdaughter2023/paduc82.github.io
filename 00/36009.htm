This is very common currently, caused by Windows Update installing an incorrect display driver, and the current fix is rolling back your Display driver rather than updating it . . .

My original work around solution to my specific issue was as follows;

  1. Open File Explorer and Select your SkyDrive account
  2. Open the Directory and select the file causing you problems.
  3. The select the File Properties (either right-click or from the ribbon). The File Properties dialog box is displayed. Please note there is NO Security Tab displayed in the File Properties dialog box.
  4. Now Close this dialog box and highlight (select) the file again.
  5. Then select the Properties Option from the Ribbon. This will display two (2) options;
    Properties
    Remove Properties
  6. Select the Remove Properties option.
  7. After some processing the Remove Properties dialog box is displayed.
  8. Select to Cancel this operation on the dialog box.
  9. Now repeat Steps (1-3) to display the file properties.

Note: The Security tab is now displayed and the file will now attach to the email.


Recent discovery

I have discussed this issue with one of my work colleagues who is a Systems Architect and we were able to discover that the issue related to the 'Availability' status of the file;

  • Available offline
  • Online only

 If the file is 'Online only' then we were unable to attach this to an email, either in OWA or Outlook.


Switching the Availability flag to 'Available offline' corrected this, however I would suggest that this creates a file 'locally' therefore allowing it to be attached.


Interestingly, trying to switch the setting back to 'Online only' generates an error dialog box;


'Windows cannot access the specific device, path or file. You may not have the appropriate permissions to access the file'
