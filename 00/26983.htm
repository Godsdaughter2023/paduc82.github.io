If you have Excel 2007 you can try this formula in cell D11.

=IFERROR(INDEX(IF(COUNTBLANK(OFFSET($E$6:$P$6,ROW($A$1:$A$5)-1,))=COLUMNS(E6:P6),"",$D$6:$D$10), _
SMALL(IF(IF(COUNTBLANK(OFFSET($E$6:$P$6,ROW($A$1:$A$5)-1,))=COLUMNS(E6:P6),"",$D$6:$D$10) _
<>"",ROW($D$1:$D$5),1E+99),ROW()-ROW(D$10))),"")

Note: This is an array formula that must be confirmed with CTRL+SHIFT+ENTER rather than just ENTER.

Copy the formula down to cell D15 and use the range D11:D15 as the list in your data validation.

(You can choose any range that suits better than D11:D15 should that be busy).

If you have Excel before 2007 you have to use =IF(ISERROR(stuff),"",stuff) instead of =IFERROR(stuff,"")

Hope this helps / Lars-Åke

XFD equals 16384 ? - Microsoft Community

jpdas say:

XFD equals 16384 ?

Hi, the max no of columns in excel is XFD = 16384 as declaired by microsoft.

But when i convert XFD (26 base)  into decimal (10base) the result comes 16357,

why this mismatch?

javed a say:

Excel column numbering is not exactly a positional number system. It calculates column putting the value of A....Z as 1...26 and multiplying power of 26 with it's offset from right

X is in 2nd offset from right - X will be multiplied by 26*26*24. 24 is positional value of X

F is in 1st offset from right - F will be multiplied by 26*6. 6 is positional value of F

D is in 0th Offset from right - D will be multiplied with 1*4. 4 is positional value of F
