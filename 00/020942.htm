No. The attachments are stored within the .mdb or .accdb or other database file. And they contribute to the 2GByte limit on the size of the database.

Multilevel List Using Mixed Numbering Scheme - Microsoft Community

d-piddy2k say:

Multilevel List Using Mixed Numbering Scheme

I'm using Word 2010 and trying to implement the following multilevel list using a mixed numbering scheme:

1  Section Level-1

R:1-1  [Reqts]

R:1-2  [Reqts]

1.1  Section Level-2

R:1.1-1  [Reqts]

R:1.1-2  [Reqts]

R:1.1-3  [Reqts]

1.2  Section Level-2

R:1.2-1  [Reqts]

...

2  Section Level-1

R:2-1  [Reqts]

...

2.1 Section Level-2

R:2.1-1  [Reqts]

...

2.1.1  Section Level-3

R:2.1.1-1  [Reqts]

R:2.1.1-2  [Reqts]

...

As you can see, there are 2 numbering schemes involved:

a. Section-level numbering scheme ("#.#.#...")

b. Requirements-level numbering scheme ("R:<section-number>-#") - this number format contains the section level number it belongs to and counts within that section only.  The numbering sequence is reset for each section back to 1.

Any help with this would be greatly appreciated.

Thanks.

doug robbins - mvp office apps & services (word) say:

The Section Level numbering (1, 1.1, 1.1.1, etc) can be done by following the method on the following page of the Late Shauna Kelly's website: 

http://www.ShaunaKelly.com/word/numbering/OutlineNumbering.html

For the [Reqts] numbering however, I believe that the only way will be to make use of a combination of StyleRef fields with a \n switch and Seq fields with a \s switch followed by the appropriate Heading Level number (1 for 1, 2 for 1.1, 3 for 1.1.1)

For example, if you have

1  Section Level-1

followed by the fields

R:{ STYLEREF 1 \n }-{ SEQ L1 \s 1 }

R:{ STYLEREF 1 \n }-{ SEQ L1 \s 1 }

the result of those fields will be

1  Section Level-1

Those same fields following

2  Section Level-1

will give

R:2-1
R:2-2

Note that in those field constructions, the numeral 1 is a shortcut that you can use instead of "Heading 1" and is independent of the language being used.

Where necessary, you would use similar fields with the numeral 2.

To facilitate their use, you could make autocorrect entries of the field constructions.
