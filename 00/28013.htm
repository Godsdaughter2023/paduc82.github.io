Moving clip art on a document - Microsoft Community

wendyharvey say:

Moving clip art on a document

I have tried using drag and drop editing to move clip art on a document and nothing happens.  What am I doing wrong?
I've tried "help," followed the suggestions, and still the object does not move.
john mcghie say:

I generally choose NOT to use drag-and-drop to move graphics around, because of the difficulties in doing so.

 

This problem can be caused by several things: I am going to take a wild guess that your specific instance is caused by 'the wrong kind of selection'.

 

A graphic in Word has as many as three "clicks" to select it.  Each click gives a different kind of selection.  If you think of a graphic as a "container", the outer container is the graphics canvas; inside that may be the graphic itself, and the third click may open the text inside the graphic (if it has any).

 

If you click in the text of the document so you have no graphic selected, then click on the very edge of the graphic, the first click will give you the blue "handles" around the graphic.  That's the "Object Selection" which is the one you want for moving it.  The next click gives you different handles (hollow squares, from memory) that's the "Graphics" selection you use for setting properties of the graphic.  The third click will take the handles away: that will be the "Text" selection, if the shape you clicked contains a text field.

 

All too confusing for me!  I use Command + x on the first click to cut the graphic out, then click where I want it, and Command + v to paste it back in.  That is faster, easier, and safer: it is less likely to result in file corruption in the document.

 

Now we get into the knotty question of whether the graphic "will" move or not.  That is determined by its "Layout" setting.

 

Right-click the graphic and choose Properties>Layout.

 

If that is set to "Inline with text", the graphic behaves like a large character.  You can drag it or paste it into a different paragraph, but it will always be *in* a paragraph.

 

If the layout is set to anything else, then you can drag the graphic around the page, and the text will move around as you do, to make room for it.  You can use the "Wrapping" properties to determine if the text surrounds the graphic on the left, on the right, both sides, or whether the text leaves a space right across the page in which the graphic sits.

 

You can then use the "Order" property to determine whether the graphic sits "in front" of the text and obscures it, or "behind" the text and the text appears over the top.  If you have several graphics on the page, you can move them forward or backwards with respect to each other until you get things the way you want them.  A Word document has about 256 "layers", of which the text is the middle one.

 

Hope this helps

You can't force any font to display characters it doesn't contain :-) Also, the program doesn't fetch fonts on a character-by-character basis. It's only if the on metrics, not volume/variety of characters.

If the code called for is used for a different glyph by the font in use, that's what will display. But if the code is not used at all a placeholder symbol is used.

Calibri is a nice - but somewhat limited - font. It contains fewer than 1130 glyphs, whereas Arial Unicode MS provides in excess of 50,000 glyphs. Perhaps the OS X Character Viewer utility will help you find an acceptable alternative [IMHO, MS Mincho is rather nice & may work for you.]

How to use ComboBox AfterUpdate to fill other fields? - Microsoft Community

frank oquendo say:

How to use ComboBox AfterUpdate to fill other fields?

I'm creating an Access 2010 web database to track some information on our clients. On the contact details form, I replaced the Company textbox control with a combobox control named cboCompany. The combobox displays a list of company names, filtered by a SELECT DISTINCT clause to populate it with unique values.

What I'd like to do is have other fields populated once a value is selected from the combobox. Values would include Address (txtAddress), Business Phone (txtBusinessPhone), etc. for the selected company. The same company may be present multiple times in the table so I'd only need the very first result whose company name matches the value in cboCompany.

How would I do this?

marshall barton say:

Create a query that retrieves all the fields you want to see on the form (along with the companyID and company name).  I don't understand how your companies table can have multiple records for the same company.  It kind of sounds like your contacts table has all the company information, and, if so, would be a serious mistake.  I don't want to propose how to write the query until I have a better understanding of how your tables are organized.

Anyway, once you have the query doing what you need, use it as the combo box's row source.  Set the combo box properties:
   ColumnCount   <the number of fields in the query>
   BoundColumn   <the number of the companyID field in the query>
   ColumnWidths  0;;0;0;0;0;...
where the position of the missing 0 corresponds to the company name field in the query.

Then each of the text boxes that you want to display the other company data can use an expression like:
   =thecombobox.Colummn(2)
to display the third field in the query.

The ".bak" file is merely a copy of the original ".pst" file made by ScanPST before it starts to run purely as a precaution in the event there's a requirement to go back to the original file. Can be moved anywhere. If you've been running with the .pst file after running ScanPST on it for a while without any problems - it's questionable as to whether or not you need to keep the .bak file altogether.
