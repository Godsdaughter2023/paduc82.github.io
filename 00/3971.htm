Which label template are you using? None that I've seen include a printable Border. Since all are based on table structures they often display Gridlines as Daniel indicated, but Gridlines do not print.

My second issue was with the Ethernet adapter.  I noticed it when I was working on an old router.  It would not show any adapter settings under Ipconfig.  After a bit of troubleshooting I went to check if there was an IP set manually.  What I found instead was every protocol and option had been deselected.  After making sure the usual had their checkmarks, the Ethernet adapter was working normally.

I believe both issues were due to upgrading with Hyper-V enabled and setup with the Virtual Switch.

I just wanted to pass this along to anyone who may be experiencing similar issues.

amit tandon say:

To Combine SumIfs with SubTotal so that formula ignores hidden cells, you may use below:

=SUMPRODUCT(--(F1:F1000=L1),SUBTOTAL(109,OFFSET(H1,ROW(H1:H1000)-ROW(H1),0)))

This will SUM ONLY the non-hidden cells in the range H1:H1000 where the range F1:F1000 = L1.

Or if you want to include hidden cells also in SUM, you may use:

=SUMPRODUCT(--(F1:F1000=L1),SUBTOTAL(9,OFFSET(H1,ROW(H1:H1000)-ROW(H1),0)))

john korchok say:

The best way for that to work is if you place all the linked files in the same folder as the presentation. Then create a hyperlink that is only the file name of the linked document. Otherwise, PowerPoint will include the entire path to the linked files, and that path will not exist on the other computer.

Here's what the dialog should look like:

In the Address field, only the file name displays. In the Current Folder window, you can see both the linked file and the presentation containing the link.

Linking can also work if you put the files to be linked to in a folder at the root of your C: drive. That's more work, because you colleague must first copy that folder to the right spot. This methods keeps the file path constant on both computers as C:\foldername\filename.
