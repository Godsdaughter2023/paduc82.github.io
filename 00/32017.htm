It sounds as though, since the data on the NTFS drive was created with another user (your username may be the same, but the security ID will have been regenerated when you reinstalled the OS), that you'll need set the permissions so that you have full access to all the files.

Assuming your account has Administrator privileges:

1.  Right-click on the root folder, choose Properties
2.  Go to the Security tab.
3.  Click the [ Advanced ] button.
4.  In the Permissions tab, click the [ Change Permissions... ]  button.
5.  Give Administrators Full control.  I'm not sure whether you'll have to Add one or Edit one that's already there.
6.  Check the [ ] Replace all child object permissions with inheritable permissions from this object.
7.  Press [ OK ]. This may take a while to complete.

For reference, this is what that the permissions and ownership look like on a properly set up system (this is on a Raid-0 drive).  I suggest setting your system to match this, at least with regard to Administrators and SYSTEM.

If you have problems booting from a boot disc, you may have UEFI BIOS:

1. Disable Secure Boot
2. Enable Legacy Boot
3. If Available enable CSM
4. If Required enable USB Boot
5. Move the device with the bootable disc to the top of the boot order
6. Save BIOS changes, restart your System and it should boot from the Installation Media

HI! Alex,

Sorry that your recovery failed to work. As many experts already tell you how to install a clean win10 in details. I just come to give some explanations on your case.

First of all, the "recovery" is located somewhere inside your hard disk's small partition (usually hidden). And DISKPART is a powerful and dangerous tool to re-assign and manage the hard disk partition. As result, your "reset key" may not work anymore even after you have fully re-install a brand new win 10.
