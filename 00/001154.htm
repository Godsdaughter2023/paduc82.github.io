cbarnhorst say:

The reason your copy of Windows 10 "complained and stopped working" was that the build expired.  All Windows 10 evaluation copies do have an expiration date.

The 25 character keys used in the early days of the program are now blocked by Microsoft's activation servers.  The old method of providing each build with a special key ended in the Summer of 2015 and the old keys blocked to prevent their use in activating new installations with them.  Since 2015 all Insider previews require an activated copy of Windows 10, whether by prior upgrade or new purchase.

Try installing without providing a product key by clicking on the link that says "I don't have a product key" on the lower right of the product key page.  Your QEMU virtual machine is probably not recognized as a Registered Windows 10 Device under the newer, permanent registration system if you did not upgrade to Windows 10 from either Windows 7 or 8.1 at some point as instructed by Microsoft in 2015.  All users were warned about this change back then.  Microsoft warned that such old copies of Windows 10 previews could continue to be upgraded to new ones only if the user (1) kept up with new builds as they became available, and (2) did not attempt a clean re-installation of Windows.

It is likely, from your description, that you need a permanent license for Windows 10 to continue using the VM as intended.  Insider preview builds will still have expiration dates but the VM itself will be permanently registered as a Windows 10 device and activation of each new Insider build will be automatic.

don cuthbert say:

There is a good chance that the Product Key you had activated was just the generic Product Key for that Edition that most other Windows 10 devices have.  Activation is based on the device's hardware (or a Virtual Machine) being recognized, not the Product Key.  The generic Home key ends in 8HVX7.  The generic Pro key ends in 3V66T.

Each Virtual Machine requires its own Windows license, independent of whatever Windows license the Host may or may not have.  The files that represent a Virtual Machine are usually very tiny (if you exclude the file that represents the hard disk).  It is those tiny files which have the "hardware signature" that is recognized as either licensed or not by Microsoft Activation servers.

Do you have any copy or backup of the VM files from when it was still activated?  Those tiny files are the activated device that was licensed for Windows 10, so if you deleted them, it is like tossing a licensed physical computer in the trash bin.  Creating a new VM on the same host will have no license for Windows, if you have deleted the previous VM files, instead of reusing them (or replicating them in some way) for the new VM.

Activation of a clean install of Windows to an already licensed physical or virtual device does not require a Microsoft Account Windows login.  However, the Activation troubleshooter can help to activate Windows after a hardware change, such as a new motherboard, if you had linked to a Microsoft Account while activated.

To summarize:  If you deleted the licensed VM files and cannot recover them, you deleted the Windows 10 licensed device.  The large hard disk file does not matter, but the other VM files do.

It may be worth noting that a stand alone Windows 10 license is transferable to another device, even to one that you do not own.  Legally, you might have the right to transfer the Windows 10 license from the deleted VM to a new physical computer or Virtual Machine, but only if the original Windows license was a stand alone license.

Also, ensure that you are installing the same Edition of Windows 10, such as Pro or Home, as you had activated.
