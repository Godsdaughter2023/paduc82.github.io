a. user say:

Unfortunately, Word does not deal with this datetime format well. 

The simplest solution if it is possible in your situation is to change the cell formatting of the duration column to Numeric. Then Word should instead see a number in days and fractions of days.

If you do not want to disturb the data in that column, the next simplest thing is probably to copy the column (or use formulas to reference the existing column, perhaps), and apply numeric formatting to that.

In either case if your column is called Duration, you could use { =INT({ MERGEFIELD Duration }*24) } to get a whole number of hours, and (say) { QUOTE "{ SET dur { =24*{ MERGEFIELD Duration } }{ =INT(dur) }:{ =60*INT(dur-INT(dur)) \#00 }" } for h:mm format.

 
Or you could make a new column with formulas that created the exact format you need in Word. But I am not going to go there.

The problem with trying to deal with this entirey in Word is that Word treats the durations that come in without a date ( <24 hours, e.g. "4:00:00 AM") as a datetime on *today's* date. Let's say your field is called Duration. Then { MERGEFIELD Duration \@h } will extract the number of hours in that case.

But Word treats the ones that come in as a date followed by a time (e.g. 1/2/1900 4:00:00 AM" as a regular piece of text. If you try

{ MERGEFIELD Duration \@h } on that, you get the same text back (i.e. 1/2/1900 4:00:00 AM )

As far as I know, the same is true regardless of locale, i.e. when there is a date, it is in "US" format (M/D/Y), but having your regional settings to US format doesn't make Word recognise it as a date.

If you cannot deal with it in Excel, other possibilities include:

  a. use a collection of IF fields and other field codes to try to extract the day number, hour number and minutes (if you need that). This will be get harder the larger the number of hours can be, especially if they can be so large that the date part goes beyond a January date.

 b. use Word VBA and SQL to get the data and return an extra "pseudo-column" with the data you need

 c. use a DATABASE field in your Word document and a small external file to extract the necessary data from the string with a similar formula to the one you would need in (b).


If you need one of those options, post back!
