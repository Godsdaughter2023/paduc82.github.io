Hello Kaufmann,

Thank you for your post.

In the groups.csv, updated_at column means if there is some changes done by group administrator by changing the group name or image.

In the messages.csv if you want to see by which particular user posting has been done than kindly refer to the sender_name or sender email address column, because participants means the sender has send the private message to some other user or user has been added in the CC of a post.

1. Admin.csv- In this file you will find all the current active admins of the network.

2. Groups.csv- You will find the columns as (i) id, (ii) name, (iii) private, (iv)moderated, (v) api_url, (vi) created_by_id, (vii) created_by_type, (viii) created_at, (ix)updated_at, (x) deleted, (xi)external
 i. Id:- that is the group id for that group
 ii. Name:- This will indicate name of group
 iii. Private:- If the group is private than the value will be true under this column
 iv. Moderated :- If any changed done than value will be there
 v. Api_url:- This contains the URL which you can use in the API.
 vi. Created_by_id:- ID of the user will be mentioned who created the group, and in the users.csv we can find the same user who created the group
 vii. Create_at:- Time of the group creation
 viii. Deleted :- if the group is deleted then the value will be true and the group is active than value will be false.
 ix. External:- if the group is external then the value will be true and if the value is false than the group is false.
   
3. Message.csv :- There will be all the messages with the message id and sender name and if its deleted then in deleted at column.
 i. Id:- this is the message ID
 ii. Thread_id:- This column is relate to thread of a post
 iii. Group_name:- In this column group name will be shown in which group the message is posted 
 iv. In_private_group:- If the posted message group is private then value will be TRUE in this coulmn, for private group will be FALSE
 v. In_private_conversation:- If its a private message then value will be TRUE in this coulmn, otherwise as FALSE
 vi. Sender_id:- ID of the user who posted the messages 
 vii. Sender_name:- Name of the sender
 viii. Sender_email:- Email ID of the sender
 ix. Body:- This coulmn contains entire message which is posted
 x. Attachments:- If any file attached to the post then it will have file value
 xi. Deleted_by_id:- If the message is deleted then this coulmn will have ID of the deleted user
 xii. Created_at:- This shows creation time of the message
 xiii. Deleted_at:- If the message is deleted then it will show the time when its deleted

4. Network.csv:- List of all network with there ID will be mentioned in this file.
  
5. Users.csv:- In this file all users email id and joined at date with the name, email address, suspended at, deleted at and deleted by with state of the users mentioned.
  
 • We can differentiate in users.csv by Filter with column “state” and select soft_delete. Uncheck the box of “active”. Applying this filter would give us only soft_delete users.
  ○ Filter “joined_at” column which contains only blank value.
  ○ Filter “deleted_at” column which contains only blank value.
  ○ Filter “suspended_at” column which contains only blank value.
  ○ By doing above steps we can differentiate the users are currently in deleted state if they have “deleted_at” time stamp.
  ○ If the user with “suspended_at”  than that user is suspended or in block state.
  ○ If the user is having none of the states “deleted_at” and “suspended_at” then the user is in pending state.
