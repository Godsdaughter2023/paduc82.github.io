rocaw say:

Hi Paul,

Upon checking the issue you’re having with importing appointments using .CSV file, from Excel to Outlook, it seems that this feature you’re trying to perform is not possible in Outlook.com. However, on an Outlook desktop app it is. I just want to verify which Outlook are you using? Is it the Outlook desktop app or Outlook.com? If it is Outlook desktop app what is the build and version? Can you tell us the error message, if there is any? You can also send a screenshot for us to have a clear idea of the issue is all about. Because you are using .CSV file to import appointment to Outlook, I suggest trying these simple steps:

Using Excel, you can create a file and save it as a .CSV file to do that follow these steps:

  1. Go to File and choose Save As in the left pane.
  2. You can change the file name if necessary. Excel won't write over your Excel workbook file; it will create a new .csv file.
  3. You will see from the File Type drop down, choose the CSV (Comma delimited) (*.csv) option.
  4. Choose Save. You can ignore the possible data loss error if shown. Be sure to note the location of the file if there's any chance you won't remember where it is.
  5. Close the .csv file you just created.

To import the .CSV file to Outlook follow these steps:

Click the File tab, choose Open & Export in the left pane, and then click Import/Export from the Open section. Now, run the wizard as follows:

  1. Choose Import From Another Program Or File (if necessary) and click Next.
  2. Choose Comma Separated Values and click Next.
  3. Click the Browse button and work through the folder structure to select the .csv file you created earlier. Click OK after selecting that file. This pane also lets you determine how Outlook deals with duplicate data. There won't be any in our example, but when applying this to your own work, you'll want to choose appropriately. For now, accept the default option, Allow Duplicates to Be Created, and click Next.
  4. Select Calendar and click Next. If you have more than one calendar, make sure that you select correct Calendar.
  5. Click Map Custom Fields. If necessary, check Import "yourfilename" to enable the Map Custom Fields option.
  6. At this point, you must identify the Excel values to import and specify where to put them. The left pane lists the Excel fields. The right pane lists the Calendar items and shows the default mapping .The Task field isn't mapped because there's no Calendar field by that name.
