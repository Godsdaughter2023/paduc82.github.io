brian tillman [outlook mvp 2007-2019] say:
The reason I said what I did is that it could be counter-productive to try to keep the PST's size close the the in-use data size.  Databases work more efficiently if there is white space in the file so that it can simply reuse free space without having to request additional allocation from the file system.  Expanding a file is "expensive" in terms of disk activity and file system activity, whereas reusing white space within a file is "inexpensive".  There is no need to request additional file space, just to write the data and change a few pointers. I/O is reduced, fragmentation lessened, and processing kept to a minimum.
Brian Tillman [MVP-Outlook]
