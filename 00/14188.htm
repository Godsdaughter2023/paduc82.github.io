scottgem say:
You can't filter a Value List. The only thing you can do is replace the list. Value lists are not appropriate here. The only time to use a Value list is for a short, static list of items. For example months of the year. 

As Ken pointed out, using a query on MSysObjects will do the job for you. However, I tend to create a reports table for this functionality. For 3 reasons. One, because I may have reports that I don't want to expose to the users. Though I can use the report name to filter those out. Two, because I sometimes use less than obvious names for reports so having a table with a description field makes it easier for the user to find a report. Three, because I will often use a form to provide criteria for the report. So my table has a field for formname. I test to see if that field is populated when the report is selected and open the form instead of the report. 


ken sheridan say: If you use a query on the MSysObjects table as I suggested in your other thread you can restrict the RowSource query by referencing the text box as a parameter:

PARAMETERS Forms!YourForm!txtReportSearch TEXT(50);
SELECT [Name]
FROM MSysObjects
WHERE [Type] = -32764
AND [Name] LIKE "*" & Forms!YourForm!txtReportSearch & "*"
ORDER BY [Name];

You then simply need to requery the list box in the text box's Change event procedure to progressively restrict the list.

ghotchkiss say:

I rebuilt my database profile and now my inbox is missing

I rebuilt my database profile and now my inbox is missing. Other folders (Sent emails, Calendar, Deleted Emails) showed up but no inbox. I have other accounts in the same profile and their inboxes showed up.


dianeoforegon say: Without knowing What type of account(s) (POP, IMAP, Exchange), it's hard to give a complete answer. If it's IMAP or Exchange just delete the account and let it download from the server. For help recovering data under "On My Computer" folders see:

How to recover your data when rebuilding, restoring and upgrading fails


Rebuild Basics:

  • Do Not Rebuild Identities with Exchange and IMAP accounts. Delete the accounts (this deletes the cache files in Outlook). Do the rebuild then add back accounts. Better yet start over in a new Identity. If the rebuild does not work then you could sync unwanted changes or deletions to your server and lose data.
  • Make sure you have plenty of free space on your drive. Make sure that you have at least three times as much hard disk space available as the current size of your Outlook identity PLUS 10% free for virtual memory.
  • Rebuilding does not fix all issues. It can make some problems worse. Even though the Database Utility will make a backup, I advise making a copy in the Finder first.
  • You can delete the rebuild version after you are satisfied with the results. This copy will be named like this: Main Identity [Backed up 1-21-2012 13.35]   

Tip: If your Identity will open, before you rebuild, check "On My Computer"Inbox and Sent folders to be sure they are under the 2GB limit. Generally this is less than 10,000 messages but with large attachments could be less. Make subfolders e.g. Inbox 2011, Inbox 2010, Sent 2011, Sent 2010 etc to help reduce the number of messages in a folder. (Check for any large archive folders you might have in addition to Inbox and Sent).
