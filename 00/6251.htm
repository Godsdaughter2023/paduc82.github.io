The originator of the documents should be using a CreateDate or SaveDate field.

Probably the best thing for you to do is to add a SaveDate field to the letter and before saving it, use F9 to update that field and then CTRL+SHIFT+F9 to unlink it so that it is converted into ordinary text.

blechler say:

Mac Excel 2016 opens old version of OneDrive file

Hi everybody,

I have a mac user who is editing an Excel document from a shared OneDrive folder (she is not the owner).  When she selects "Edit in Excel" it opens a version of the file from yesterday.  The likely assumption is that there is a cached version of this file on her system that is not being replaced by the newer version from the cloud.  How do I get rid of that cached version if that is the case, and how do I prevent this from repeating?

Thanks in advance.

bironology_001 say:

SOLVED!

I used Activity Monitor to look at the Microsoft Excel's process open files and ports, and copied that to a txt file, then I opened the problem file, took another snapshot saved to another txt file, and diff'd the files.

This illuminated the problem file was open from location: 

/Users/<me>/Library/Containers/com.microsoft.Excel/Data/Library/Application Support/Microsoft/AppData/Microsoft/Office/16.0/OfficeFileCache/

So I quit Excel, toasted all of the files in that folder, restarted Excel and opened the file, and voila! The fresh update from OneDrive appeared.

Phew. Hope this helps someone else.

In summary, just quite Excel, delete files in 

/Users/<you>/Library/Containers/com.microsoft.Excel/Data/Library/Application Support/Microsoft/AppData/Microsoft/Office/16.0/OfficeFileCache/

and restart Excel. No system restart was necessary.

shannon_007 say:

Excel Formula Help

Hi,

 

I'm trying to write a formula that will only count cells D3:D104 from Sheet 1 if they contain a file number, if cells B3:B104 from Sheet1 contain Jan, Feb, Mar  first.  This is a summary formula for a quarter on a separate sheet (Sheet2)...I've tried varius versions of COUNTIF and COUNTIFS, but cannot get it to work...frustrating!!

 

Please help.

 

Thanks!

edwin tam say:

You can think of the array function has inside the memory an array (matrix) of values:

 

For example, this will produce a vertical list of result of TRUE (when a cell is non-empty) and FALSE (when a cell is empty):

'Data Entry'!B3:B104<>""

 

While TRUE = 1 and FALSE =0

 

Similarly, the second part:

IFERROR(FIND(UPPER('Data Entry'!B3:B104),"JANFEBMAR")>0,0)

produces another list of TRUE/FALSE. TRUE if a cell contains JAN, FEB, MAR. (The Find function checks whether the cell content can be found within "JANFEBMAR".)

 

Now, you got 2 lists of TRUE/FALSE. By multiplying the 2 lists across, you got something like this:

TRUE *  TRUE   =  TRUE

TRUE *  FALSE  =  FALSE

FALSE * TRUE   = FALSE

FALSE * FALSE  =  FALSE

 

This is the logic for AND. It also mean that, only situations where ALL criteria are satisfied, a TRUE will be returned in the result array.

 

At the end, by SUM up all the 1 and 0 of the array, I got a COUNT of all those TRUE's.

 

 

Using this approach, you can test for as many criteria as you need, by keep multiplying TRUE/FASLE arrays. Only cells which pass all test will end up having a TRUE (or 1).

 

For example, if you now want to test column G also. You can add another test to the function, like this:

=SUM(('Data Entry'!B3:B104<>"")*(IFERROR(FIND(UPPER('Data Entry'!B3:B104),"JANFEBMAR")>0,0)) *('Data Entry'!D3:D104<>"")*('Data Entry'!D3:D104<>"None")*('Data Entry'!G3:G104="Concluded"))

 

 

The array function of Excel is a very powerful feature worth learning. There are many other powerful applications as well. :-)
