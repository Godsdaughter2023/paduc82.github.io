ken sheridan say: If you want to set the RecordSource property of the report you can pass the string expression to it via the OpenArgs mechanism.  Then in the report's Open event procedure set the RecordSource property to the string expression.  So to open the report the code would be along these lines

Dim strSQL As String

strSQL = "SELECT * FROM MyTable " & _
    "WHERE MyDate >= #" & Format(Me.txtDateStart),"yyyy-mm-dd") & _
    "# AND MyDate < #" & Format(Me.txtDateStart),"yyyy-mm-dd") & "#+1"

DoCmd.OpenReport "MyReport", View:=acViewPreview, OpenArgs:=strSQL

In the report's Open event procedure put:

If Not IsNull(Me.OpenArgs) Then
    Me.RecordSource = Me.OpenArgs
End If

You'll find an example of this method of restricting a report's results, along with other methods, in the file MultiSelect.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

The form which uses this method is opened by the button captioned 'Restrict Report on One or More Fields by Building RecordSource in Code'.

What is the size of Outlook data file?
 
Have you tried repairing the Outlook data file?
 
This error is due to the following reasons:
1.       Antivirus scanning of the file
2.       Forced Microsoft Windows shutdown before the Outlook.exe process shuts down completely
3.       Outlook add-ins
 
 
Try working in Outlook safe and check if it helps.
Click on Start > All Programs > Accessories > Run > Copy and Paste the following command in the open box and press Enter.
 
Outlook /safe
 
If Outlook safe works as expected, disable the add-ins:
 
http://office.microsoft.com/en-us/outlook-help/enable-or-disable-add-ins-in-office-programs-HA010034127.aspx#BM9
 
If the issue persists, repair the Outlook data file and check if it helps:
 
http://office.microsoft.com/en-us/outlook-help/repair-outlook-data-files-pst-and-ost-HA010075831.aspx
