As with the built in Replace function the following function, which I posted in the other current thread on a similar topic, will replace all instances of the character, whatever its position in the string, and also allow the replacement of multiple characters simultaneously:

Public Function MultiReplace(varInput, ParamArray varReplacements())

    ' call like this:
    ' MultiReplace("abcdefghijk","c","V","e","X","j","Y","m","Z")
    ' to return this:
    ' abVdXfghiYk

    Const MESSAGETEXT = "Uneven number of replacements parameters."
    Dim n As Integer
    Dim varOutput As Variant
    Dim intParamsCount As Integer
    
    If Not IsNull(varInput) Then
        intParamsCount = UBound(varReplacements) + 1
        If intParamsCount Mod 2 = 0 Then
            varOutput = varInput
            For n = 0 To UBound(varReplacements) Step 2
                varOutput = Replace(varOutput, varReplacements(n), varReplacements(n + 1))
            Next n
        Else
            MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"
        End If
    End If
            
    MultiReplace = varOutput
    
End Function

e.g. to remove the page break (ASCII 12) and carriage return/line feed characters (ASCII 13 + ASCII 10)

MyString = Chr(12) & "ABC" & Chr(13) & Chr(10)

? MultiReplace(MyString,Chr(12),"",Chr(13),"",Chr(10),"")
ABC

swischner say:

I am somewhat surprised that pages are being duplicated not sections which would be the usual result of this type of fault.

Anyway I am certain this is the problem:

You are NOT supposed to copy .ONE, .ONEPKG or the OneNote TOC file to OneDrive, neither your local OneDrive Sync folder nor directly upload them to OneDrive.

I explained the reason in another thread in the German answers forum and in an article on my OneNote-Blog. But as I suppose you don't speak German here comes a brief summary:

The very brief version:

Never copy .ONE files to OneDrive manually but use the SHARE mechanism of OneNote to move them to OneDrive.

The somewhat more in-depth explanation:

OneDrive, like Dropbox, Google Drive and other Cloud file services, ist designed to sync whole files between a server and several clients. Meaning: If you change anything in a file (say a Word document) on a device the whole file on the server is replaced with the new version. If you change (or just leave open for editing) the file from two or more devices this will of course result in conflicting versions (the ones temporarily stored locally on the devices and the old version on the server). OneDrive can not MERGE those changes / versions. So it goes the safe way and uploads/stores both versions. That is where your duplicates are coming from.

OneNote files consist of .ONE files holding sections which contain the pages in a XML format. If you are trying to do the syncing with the file exchanging mechanism that I explained before, the same thing will happen as to word files if you just opened the notebook on different devices.

Especially for OneNote notebooks OneDrive is using a different syncing mechanism instead, which is able to only sync the changes of an object (e.g. a text paragraph or picture) inside a notebook page. For this mechanism to work, the OneNote notebook has to be stored "the right way".

Although OneNote on the "server side" does use the .ONE file structure, those files are supposed to be hidden. You should only see a symbolic link file in your OneDrive folder if the notebook is stored the proper way.

If the proper way to store the notebook on OneDrive is NOT to copy .ONE files over, what is then?

Simple: From inside OneNote use the FILE menu and SHARE the file. You will be alerted that the "local" (as when you store OneNote files in a folder of your choice, even on OneDrive, they are considered local) notebook has to be transferred to OneDrive first. The OneNote desktop application will do that for you. You don't have to (should) copy any files over manually.

So I would suggest that you first copy the .ONE files and their folder structure back to a local drive, open them from there and then transfer it to OneDrive using the proper way as described. I bet your duplicating problem will be gone.

For the future: If you create a new notebook, either create it on OneDrive selecting the right location in the OneNote program or create it locally (OneNote from MS Office only). If you later decide to transfer it to OneDrive DO NOT just copy the files but use the internal SHARE function instead.

Yes, this is a very poorly implemented and "not well" documented feature. It comes from the fact that originally (OneNote 2003) OneNote notebooks were not meant to be shared via a cloud service or the like and the developers somewhat glued that option on top of what was already there (the folder/.ONE file structure).

Hi,

 

So long as your data are laid out as above then this macro will do what you require

 

Sub test()
Dim x As Long, y As Long
Dim LastRow As Long
y = 1
LastRow = Cells(Cells.Rows.Count, "A").End(xlUp).Row
For x = 1 To LastRow Step 5
    Cells(x, 1).Resize(4).Copy
    Cells(y, 2).PasteSpecial Paste:=xlPasteAll, Operation:=xlNone, SkipBlanks:= _
        False, Transpose:=True
    y = y + 1
Next
End Sub
