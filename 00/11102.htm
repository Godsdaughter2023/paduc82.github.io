This should do it ;
=================
Sub HideRows()
On Error Resume Next
Application.ScreenUpdating = False
ltrw = Cells(Rows.Count, "Q").End(xlUp).Row
For i = 2 To ltrw
 If Cells(i, 17).Value = "Exclude" Then
    Cells(i, 1).EntireRow.Hidden = True
      End If
      Next i
Application.ScreenUpdating = True
End Sub

Sub UnhideRows()
On Error Resume Next

      End If
      Next i
Application.ScreenUpdating = True
End Sub

MS Word: Update Form Field after e-signature - Microsoft Community

thomasthuene say:

MS Word: Update Form Field after e-signature

I am having an issue that probably many others in highly compliant industries have, too. So perhaps there is a solution out there. 

I am trying to develop a form that allows me to first approve the validity of the form itself, and then, after the form is filled in, approve the content in that form. 

EXAMPLE: I have form A. This design of this form A requires approval from Quality Assurance that it is compliant with our standards about documents. Thus it requires a an e-Signature. This e-Signature would typically go into a signature box the footer of that document. It indicates to the user, together with the effective date, that he/she is using a valid form for entry of the information.

After the user has entered the information, Quality Assurance will need to sign again - in a different signature box - that the added content is correct.

Can this be done in Word? And if so, how?

Thanks, TT

eli qian msft say:

Hi Thomas, 

Thanks for the detailed explanation. Currently, A signature confirms that the information originated from the signer and has not been altered. Once the document is modified, the signature will be expired. So if you want to keep both signatures in the process, to my knowledge, it might not be achieved.

As a workaround, we would suggest you try using Restrict Editing when the design of this form A requires approval from Quality Assurance. After allowing users to edit the corresponding field, you could add the original both signatures in the same fields and remove the restriction to achieve the same goal.

Regards,

Eli

thomasthuene say:

Eli,

Thanks for the response. I am not entirely sure that I understand the sentence: "you could add the original both signatures in the same fields and remove the restriction to achieve the same goal."

Of course you would use Restrict Editing for the form content, so that the user could use the form field, but not change anything else in the form template. 

The point is that there are two totally independent review processes at play here. There is one that determines that the form template is ok. That review is only done once before the form becomes useable. The form template then is used multiple times over a longer period of time. the second signature really only deals with the content of the form fields. That is a different review process.

Any other ideas?

I cannot possibly imagine that this issue hasn't surfaced elsewhere. Do I have to go the PDF route?

Regards, TT

eli qian msft say:

Hi Thomas,

Thanks for detailed explanation again.

No, it might not achieve this by the process mentioned when we just use Word. Because the signature prevent any medication in the document otherwise, the signature will be expired. So the another workaround is using PDF. We can try the following steps below to check if it might help:

1.Finish the first edition

2.Make the signature in the first review.

3.Export the file as PDF file and save the first review process.

4.Send the document and PDF file to the user in the second process.

5.When the user confirm the content in PDF, he/she can make the update in the document.

6.Finish the second review with the PDF file and make two signatures together in different fields if necessary. (Because the first signature will disappear and we can make multiple signatures in the single document continuously.)

In the meantime, we encourage and suggest you vote and comment Improve digital signatures or deliver the feedback in Word UserVoice regarding this to involve more attention from the development department. Thank you for your understanding and feedback.
