scottl msft say:

How to: Install Windows 7 side by side with XP

Problem: You can't upgrade directly from XP to Windows 7.  You want to move your files, and you're okay with reinstalling your applications.  Problem is, you don't have another computer or USB drive large enough.  However, you do have plenty of free space on your existing harddrive.

Possible solution:  Install Windows 7 side by side with XP (dual boot).  Use the diskpart utility to free up space on the harddisk to install Windows 7.

This solution will also work for those that wish to go from Vista 32-bit to Windows 7 x64.

Required:
Harddisk should have a minimum of 30GB free.  Ideally, the disk should also be less than 50% full.  A good example is a 120GB drive with 70GB available.

Detailed Steps:
1) First, clean up your existing drive.  Suggested possibilities (these are not required if there is adequate free space):
    a. Run the disk cleanup agent (right-click on the volume, properties, cleanup)
    b. Remove all temporary files (c:\windows\temp,<users>\appdata\local\temp)
    c. Empty "Recycle Bin"
    d. Consider removing programs that are not used
    e. Consider removing pagefile and/or hibernate file
2) Run defrag, even if defrag is not required.  This could take a while.
3) Reboot with Windows 7 install CD.
4) After clicking through and agreeing to the license agreement, press SHIFT+F10 to launch a WinPE cmd window.
5) In the cmd window, type DISKPART and press enter
6) Type LIST VOLUME and make note of the volume with the previous install of Windows.  It will generally be NTFS and the largest partition on the disk.
7) Type SELECT VOLUME n, where n is the volume number noted above (e.g., SELECT VOLUME 1)
8) Type SHRINK QUERYMAX to see how much space will be available on the disk by shrinking the partition.  If the amount of space is adequate for installing Windows 7, and to be able to copy your files between partitions, then proceed to Step 9.  Otherwise, this procedure may not be right for you.
9) If you are happy with the amount of space, then type SHRINK.  You can also specify the minimum and desired space for Windows 7.  For example, this command will attempt to shrink the XP volume by at least 30GB, and try to reclaim as much as 70GB:
SHRINK DESIRED=70000 MINIMUM=40000
10) When this completes, type EXIT and proceed with the Windows install.  Choose CUSTOM from the install types and select the UNALLOCATED SPACE as the destination.

Windows 7 boot manager will do all the necessary preparation.  Once setup completes, you will have two options, Windows 7 or "Previous version of Windows".  You can copy your files easily from the previous install and the new version.  You can also go back to XP if you find an older program or driver that is not compatible.  After a while, if you have all the files and programs moved over, you could also free up the space, remove the old partition, and EXTEND the new volume.

Hope this helps someone!

BCCode: 50     0x00000050

Cause

The BCCode 50 usually occurs after the installation of faulty hardware or in the event of failure
of installed hardware (usually related to defective RAM, be it main memory, L2 RAM cache, or
video RAM).

Another common cause is the installation of a faulty system service.

Antivirus software can also trigger this error, as can a corrupted NTFS volume.

Resolving the Problem

Resolving a faulty hardware problem: If hardware has been added to the system recently,
remove it to see if the error recurs. If existing hardware has failed, remove or replace
the faulty component. You should run hardware diagnostics supplied by the system manufacturer.
For details on these procedures, see the owner's manual for your computer.

Resolving a faulty system service problem: Disable the service and confirm that this
resolves the error. If so, contact the manufacturer of the system service about a possible
update. If the error occurs during system startup, restart your computer, and press F8 at
the character-mode menu that displays the operating system choices. At the resulting
Windows Advanced Options menu, choose theLast Known Good Configuration option.
This option is most effective when only one driver or service is added at a time.

Resolving an antivirus software problem: Disable the program and confirm that this resolves
the error. If it does, contact the manufacturer of the program about a possible update.

Resolving a corrupted NTFS volume problem:  Run Chkdsk /f /r to detect and repair disk
errors. You must restart the system before the disk scan begins on a system partition. If
the hard disk is SCSI, check for problems between the SCSI controller and the disk.

Finally, check the System Log in Event Viewer for additional error messages that might help
pinpoint the device or driver that is causing the error. Disabling memory caching of the BIOS
might also resolve it.
