davidf2 say:

Publisher effectively produces two sets of code. One for IE and one for all other browsers. In some circumstances it can choke and kill the link in the code for FireFox.

On your  http://www.hisgracecreations.com/index_files/therelationalplace.htm page almost everything on the page including all the dead links you cite has been combined into one image:  http://www.hisgracecreations.com/index_files/image2456.png  When Publisher converts text to an image it kills the links. If you left click and drag to try to select any of the text you can tell the text has been converted. You can also look at the files in your index_files folder in thumbnail view and spot the converted text throughout the site files. Anyway, this can be caused by using some fill colors and gradiants, some fancy borders or in this case I am guessing that you have grouped all the design elements together when you layed out the page. The fix is to select the group > Arrange > Ungroup and your links should work.

Note that you have the same problem with the other pages of your site: 

http://www.hisgracecreations.com/index_files/image2461.png ,
http://www.hisgracecreations.com/index_files/image377.png  , http://www.hisgracecreations.com/index_files/image2455.png , and http://www.hisgracecreations.com/index_files/image2538.png

Most of these pages have been converted to single image killing the links. In fact it appears that only your home page has escaped this design error.

Note also that grouped items do not render in IE8, so it is important to ungroup each page. You can do this all at one time if you want by going to File > Select All and then to Arrange > Ungroup for each page. Be aware that that will ungroup any wizard built navbars. If later you do want to add a page and a link to the main navbar you can insert a new one on each page that works with the wizard. (Insert > navigation bar > existing) This manual fix is required for Pub 2003. If you have Pub 2007 then you can fix the compatibility issue with IE8 by installing the Office SP2 and don't have to ungroup the navbars.

Reference: Navigation bars and other content is missing from Publisher HTML output in Internet Explorer 8:  http://support.microsoft.com/kb/969705

Reference: Description of 2007 Microsoft Office Suite Service Pack 2 (SP2) and of Microsoft Office Language
Pack 2007 SP2:  http://support.microsoft.com/?kbid=953195

As per your home page:  http://www.hisgracecreations.com/ the dead links can probably be attributed to the use of the callouts which like the use of "fancy" borders results in an image. The fix is to remove the text from the callout, create a small plain text box with the text and the link and layer it over the empty callout. You can use the Arrange > Order > bring to front or back to make sure the text is on top. Now when you publish new web files you will get the same effect but the text will not be converted to an image and thereby killing the link.

If you find other links not working and can not figure out why, post back with a link to the page where you are having the problem.
