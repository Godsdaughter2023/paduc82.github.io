    arzi(2) say:

    This issues started for me when Microsoft started their migration to Outlook Mail. I've spent a couple of weeks with Level 2 technicians. This is a known and emerging issue since the migration. The final resolution was to go to my Microsoft account and create a new alias with "@outlook.com" at the end of the email address. Then in Control Panel, a new profile was created to this new account. Then Outlook 2016 was opened and the new account added and pst Data File connected, and all is now well. Here are more specific steps, but if you contact Microsoft support, you should mention my resolution as the other things they tried absolutely didn't work:

    Created an alias at account.live.com: [username]@outlook.com.

    With Outlook 2016 closed, open Control Panel, Mail; from Mail Setup window, General tab, click Add; from New Profile window, type new profile name, Create new profile. Add credentials [username]@outlook.com with same password as your original account; Finish

    Show Profiles; under Always use this profile, click Outlook [new profile]. OK.

    Open Outlook 2016, let sync finish, and close Outlook.

    With Outlook 2016 closed, open Control Panel, Mail; from Mail window, click the new profile, Properties; from the Mail Setup—New Profile window, click Data Files; from the Account Settings window, Data Files tab, click My Outlook Data File; browse to the correct *.pst; close everything and relaunch Outlook 2016. All should be well.

    Hope this helps--this is incredibly frustrating and incredibly complicated.

    dhubba say: got it working -- here's the deal.

    Apparently, Dreamhost uses one domain for non-encrypted POP3/SMTP connections and a different domain for TLS/SSL POP3/SMTP.

    For example, for non-encrypted connections, it's - sub4.homie.mail.dreamhost.com

    For TLS/SSL, it's - sub4.mail.dreamhost.com

    If your email is hosted at Dreamhost, you can get this info at https://sabrextreme.com/mailserver/

    As far as I can tell, this isn't documented and Dreamhost tech support didn't know it. Dropping them an email right now.

    robert aldwinckle on forums say:

    The favorites list used to be easily accessible on the left 


    You still can have it there and if you have a wide screen monitor you may want to leave it on.   Press Ctrl-Shift-I  (the sixth letter in Favorites)  to pin the Favorites Center (Explorer bar) there.

     

    and there was a favorites bar that I used for favorites that were going to be temporary.

    That's still there too.   In fact, it is a "subfolder" of Favorites called Favorites Bar, so if you wanted to save some vertical space you could use it on the left too.   Notice the handy  Add to Favorites bar (sic) button?   That should really be labeled Add to Favorites Bar  IMO  but what it really does is just creates a new Internet shortcut (aka .URL file) out of the current contents of the Address bar and puts it in there.  In fact, it is just an ambiguous relabeling of old implementation because if you look at a directory list of Favorites in a cmd window you won't see a subdirectory called Favorites Bar but LInks is still there just the same as it ever was.   So, in fact, the Favorites Bar for its buttons is just the same as the old Links bar.   Another difference is that there is no longer a label for the for the bar that we can doubleclick on to contract the Links bar buttons out of the way.

    Another way to access your Favorites which could also be done more to the left is to use the Menu bar's Favorites menu,  e.g. press Alt-a  if you don't see that to click on.   So, to access your Favorites Bar (Links bar) temporary links you could use a keystroke sequences like  Alt-a, F, CursorRight  to see them.
