Mail Merge recognizes Excel file as Access Database - Microsoft Community

falcios say:

Mail Merge recognizes Excel file as Access Database

I have an Excel file that has only 2 sheets and each contain 1 column of names only.  I am trying to do a mail merge with this file.  Each time I select the file a Data Link Properties box comes up and it says select or enter a database name.  Somehow it thinks it is an Access data file.

I have done many mail merges before but never received this message. 

Why am I getting this and how can I correct it?

Thanks in advance.

 

a. user say:

FWIW here, with a very simple example, I do not get this (Windows XP, Office XP). However

 a. Word does sometimes have difficulty recognising single-column data sources properly. Mostly, AFIAK, that's when they are text sources and Word can't work out the field delimiter (because there isn't one).

 b. When the Data Link Properties dialog comes up, it is usually because Word has recognised the file as an Excel file, but the OLE DB provider that it uses to get the data cannot open the file. For example, if you take a delimited text file and give it a .xls extension instead of .txt, .csv etc., Word thinks it's an Excel file, but the OLE DB provider does not. The reason you see "Access" is because a single provider ("Jet") is used to get data from Access and a number of other types of data source, including Excel.

 c. In this case, you're clearly dealing with an Excel file (because if it was a .csv or some such, there would only be one sheet). The trouble is that I do not have a definitive list of problems that will cause OLE DB not to recognise that. Possibilities include:

    - the Excel document is password protected. (Then you have to use DDE to connect)

    - You have created a database/list in Excel (this sometimes seems to cause problems, even after you remove the database/list)

    - The file is corrupt (but obviously not in a way that stops Excel from opening it)

    (- it's also possible that the 1-column thing is causing problems, but my experiment suggests that it cannot be /just/ that)

So all I can suggest is that you

 a. workaround by using a different type of data source (e.g. copy/paste into a Word document and use that) or

 b. try a few experiments with a copy of your .xls, e.g.

    - does it work if the .xls is open in Excel (in that case I would hope that the OLE DB provider would behave correctly)

    - add a column. Does that fix it?

    - recreate the workbook from scratch. in that case, I would probably

      1. export each sheet to .csv
