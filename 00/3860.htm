RM-505 N97 vs. RM-507 N97

mccbleue say:

The main reason for there being different versions of the phones for the Americas is the differences in frequencies used in mobile communications. In a quadband phone like the N97, there is no difference for GSM services since you have all of the world's GSM frequencies usable in the phone, but the standard version will have US 3G frequencies rather than the 3G frequencies used in most of the rest of thew world, so if you use a network that offers 3G services you would need to have the American version.

 

The primary reason that US phones are often lacking in updates is that, like in parts of Europe, a significant proportion of the phones sold are network-variant phones, which means the software is customised to the needs of the network and therefore they need to approve updates before Nokia can release them, and networks are frequently slow to do this. Some US networks even prevent phones that they did not sell accessing their services - AT&T has some history of doing this. So much for the "Land of the Free", eh?

grschinon say: Also, Nokia USA are just slow at releasing updates too. Even SIM-free phones such as the N95-3/4 for example are seriously lagging behind their global N95-1/2 cousins.

So, if 3G isn't an issue for you then by all means go for a standard N97-1. You will not be able to use 3G services in Northern America (or in parts of Latin America and Australia) with this phone. If you do need 3G in these areas then, as mccbleue stated, you'll need to have the N97-3 (NAM version) and you'll get access to updates a lot later than everyone else.

shido151 say:

I figured it out, Here is the answer to my question for any one who might have the same problem:

Outlook will configure the account as IMAP if you let it Auto configure the account. To set it up as a POP account you have to delete the IMAP version and click the "do it manually" bubble. But in order to do that you need to;

have the email account on the server set to POP

know the incoming and outgoing mail severs.

Go into the advanced setting tab to set the correct incoming and outgoing ports.

The best way to learn how to do this is search for a tutorial on how to set up a POP gmail account in outlook.

The reason I wasn't getting more than two weeks of emails on the server is because Windows live account on the old computer (which was still in use for emails) was set to delete emails off the server after 14 days.

I have setup the POP account in Outlook and migrated all the old emails from Windows Live to Outlook.

tom ogilvy say:

This worked for me. 

Sub abc()
Workbooks("newfsimacro.xlsx").Activate
Lastrow = Cells(Rows.Count, "b").End(xlUp).Row
Workbooks("newfsitest.xlsm").Activate
Range("c3").Select
ActiveCell.Formula = "=match(d3,[newfsimacro.xlsx]sheet1!B1:B" & Lastrow & ",0)"
End Sub

if you are using a version of excel older than Excel 2007 then it would be: 

Sub abc()
Workbooks("newfsimacro.xls").Activate
Lastrow = Cells(Rows.Count, "b").End(xlUp).Row
Workbooks("newfsitest.xls").Activate
Range("c3").Select
ActiveCell.Formula = "=match(d3,[newfsimacro.xls]sheet1!B1:B" & Lastrow & ",0)"
End Sub

It is always best to use the extension when referring to a workbook  (sometimes it is not necessary to use the extension, but that is dependent on a windows setting about known extensions - if it changes, then the code could suddenly stop working).

also, you don't need single quotes if there are no spaces or other special characters in the workbook name and sheet name.
