mister wizard say:

Hello Clay,

The "unknown" account does not indicate a security breach.  Each account has two names, the "user name" (normally shown), and the SID number (an internal code).  There are various things that can cause the SID to appear in a file's Properties dialog.  For example, after you delete a user profile, its SID may appear because file ownership was assigned to the deleted profile name.  This is normal. 

In that case, you can delete the SID reference from the Properties tabs that contain it and reassign the file ownership or permission to an active profile name.  With desktop shortcuts, you can simply recreate the shortcuts.

To see the active profile names and their corresponding SID numbers, you can use the RegEdit utility.  Navigate to the key below and look for the value ProfileImagePath under each SID number:  

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\WindowsNT\CurrentVersion\ProfileList\

>Could this have anything to do with the unknown account?

The SID number could perhaps be a symptom of whatever has damaged your user profile, but it cannot be the cause.  More likely, as Lorien said, your user profile needs to be rebuilt.
