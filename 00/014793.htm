greg maxey say:

That is the way it works.  Out of the box the heading styles are cascading.  Heading 2 is based on Heading 1, Heading 3 is based on Heading 2 and so on.


If you want to change that behavior, you will need to modify Heading 2 to not be based on Heading 1.

The cause should be the same.  If you look at the Heading 2 style in a document and if it is based on Heading 1, then changes to Heading 1 will cascade to Heading 2.  Same same for TOC2.  If it is based on TOC1 then changes to TOC1 will cascade to TOC2.

    Dim FilterCriteria As String
    Dim CCount As Long

        End With

ossiemac say:

Not sure if you mean that you want to copy individual columns or multiple columns after the data is filtered.

There are 2 example subs below. (Not related to your code but you should be able to get the idea).

The first one copies only an individual column by only assigning that column to the range to be copied. The second one copies multiple columns by hiding the columns that are NOT to be copied.

Note the method of testing if any actual rows in the filtered data by counting the cells in the first column of the AutoFiltered.Range. If more than one row then it is more than just the column headers.

First Example to copy a single column:

Sub CopyVisibleRowsAndOneColumn()
    Dim rngSource As Range
    Dim rngDestin As Range
   
    'Note that AutoFilter.Range is an inbuilt object variable _
     for the entire range covered by the AutoFilter
    With Worksheets("Sheet1").AutoFilter.Range
        'Set the required filters
        .AutoFilter Field:=2, Criteria1:="2"
        .AutoFilter Field:=4, Criteria1:="4"
       
        'Following tests if any cells (other than column header) are visible _
         in the AutoFilter.Range. 'If > 1 then more than just column headers
        If .Columns(1).SpecialCells(xlCellTypeVisible).Cells.Count > 1 Then
       
            'Assign the visible data of one column only to a range variable
            Set rngSource = .Columns(3).Offset(1, 0) _
                        .Resize(.Rows.Count - 1, 1) _
                        .SpecialCells(xlCellTypeVisible)
                       
            'Set the destination
            With Worksheets("Sheet2")
                Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
            End With
           
            'Copy the source data and paste to destination
            rngSource.Copy Destination:=rngDestin
        Else
            MsgBox "No visible data to copy"
        End If
   
    End With
   
End Sub

Second Example to copy multiple columns:

Sub CopyVisibleRowsAndColumns()
    Dim rngSource As Range
    Dim rngDestin As Range
   
    'Note that AutoFilter.Range is an inbuilt object variable _
     for the entire range covbered by the AutoFilter
    With Worksheets("Sheet1").AutoFilter.Range
        'Set the required filters
        .AutoFilter Field:=2, Criteria1:="2"
        .AutoFilter Field:=4, Criteria1:="4"
       
        'Following tests if any cells (other than column header) are visible _
         in the AutoFilter.Range. 'If > 1 then more than just column headers
        If .Columns(1).SpecialCells(xlCellTypeVisible).Cells.Count > 1 Then
       
            'Hide the columns that are NOT to be copied
             Range("C:C,F:F,G:G").EntireColumn.Hidden = True
           
            'Assign the visible data to a range variable
            Set rngSource = .Offset(1, 0) _
                        .Resize(.Rows.Count - 1, .Columns.Count) _
                        .SpecialCells(xlCellTypeVisible)
                       
            'Set the destination
            With Worksheets("Sheet2")
                Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
            End With
           
            'Copy the source data and paste to destination
            rngSource.Copy Destination:=rngDestin
        Else
            MsgBox "No visible data to copy"
        End If
   
    End With
   
End Sub

jay freedman say:

"Inactive Items" is completely different from "Disabled Items".

In the Word Options > Add-ins dialog, look at the line for the EndNote item in the Inactive Items list. Note in the "Type" column what kind of add-in it is (probably COM Add-in or Template). Then select that type in the Manage dropdown and click the Go button. In the next dialog box, you should see EndNote among the listed add-ins, but its check box will be empty. Click the check box to put a check in it and click the OK button.

Some add-ins remain inactive until you do something in the document to require their services. Others are loaded into Word only while Word is starting up, so you would have to restart Word to get them to appear in the Active Items list. I don't know whether EndNote is one of these.
