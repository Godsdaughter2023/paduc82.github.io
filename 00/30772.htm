1. Type "cmd" in windows search bar

6. Type "DISM /Online /Cleanup-Image /RestoreHealth" without quote and press ENTER

Windows 7 Backup fails or hangs (workaround)

Problem: The Windows 7 backup and restore utility fails or hangs when it tries to backup "user files" directly to an external USB DVD-writer.  It appears to be successful when it creates "System Image" disks and "system recovery" disks directly to a USB DVD-writer.

Not an issue with bad DVD's or third party drivers. I tried several different types (DVD-R, DVD+R) of name brand DVDs (Magnavox and TDK).  They work OK with "System Image", "recovery", and plain old file copy but fail with "user file" backup.  I restored my hard disk to a factory fresh copy of Windows 7, tried the "user file" backup, and it failed (hung for several hours before I aborted).

What happens: The backup utility starts up, gathers files, requests a disk to backup to, FORMATS the disk, and then starts to write the files and hangs (the DVD is spinning, lights flashing, and the DVD has tracks burned into the disk).  I also tried to pre-format the DVD disk using both the CD/DVD (Mastered) and the USB Flash functions.  The pre-formating didn't work because the utility tried to reformat the disk them anyway.

Workaround: Run the backup utility to output to a USB Flash drive.  After it finishes you can copy (burn) the backup directory and the MediaID.bin file from the Flash drive to the DVD-writer (formatted as CD/DVD (Master)).  The retore function will work directly from the DVD.

Real fix needed:  I don't know what will happen if your backup is bigger than the Flash drive so we still need a real fix from Microsoft.  It appears to me that the backup utility is not correctly formating the external USB DVD-writer disk for the backup utility.
