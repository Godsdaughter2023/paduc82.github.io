Do you wish to close the @outlook.com account and change that address to be an alias of the @hotmail.com account?  What has worked for this in the past is to visit https://account.live.com/names/Manage, for the [prefix1]@outlook.com account, add another @outlook.com address, like [prefix2]@outlook.com, as an alias, make that new alias the primary, then remove the original address.  Then after 30 days, that previous address should become available to add as an alias to the [prefix1]@hotmail.com account.

If you don't want to close the @outlook.com account, I think you should just forward it to the @hotmail.com address and set it up as a send-only account in the @hotmail.com account so you can send messages using the @outlook.com address from that account.

ron sommer mvp 2009-2017 say:

Here are the steps needed:

1) Move emails and addresses to the other account.

2)Remove the address or close the account.

3)Wait 30 or 60 days.

4)Try to add the address as a alias.

Moving the addresses just requires exporting from one account as a csv file and importing to the other account.

Moving the emails requires setting up both accounts in Windows Live Mail and moving emails between accounts.

If you remove an alias address, the wait is 30 days.  If you close an account, the wait may be 60 days before the address is available to add as an alias.

No emails will be received or sent during the 30 or 60 days.

The address will be available to anyone after the waiting period.

My preference would be to keep the separate accounts.

Each account can be the alternate email address for account security.

joeu2004 say: sross002 wrote:

First, use TODAY(), not NOW().  The difference is:  NOW() includes current time, which might create some confusing results, if not with this formula, then with others in the future.

If 6/20/2010 is in A1 and =TODAY() is in A2, try:

=DATEDIF(A1,A2,"m") & "m " & A2 - EDATE(A1,DATEDIF(A1,A2,"m")) & "d"

I suspect that is what you mean by "months and days", i.e. civil time as used in legal circles.

julie fka profjulie say: The correct registry entries for a WM6.5 device are:
  • For EMAIL bodies :  HKLM\System\Inbox\Settings\PropertyPath - create the PropertyPath value, if necessary, and input the path you want to use (i.e., \Storage Card\Mail)(REG_SZ string)
  • For Email attachments:  HKLM\System\Inbox\Settings\AttachPath  - create the AttachPath value, if necessary, and input the path you want to use for attachments (i.e., \Storage Card\Mail\Attachments) (REG_SZ string)
Or you can use SKTools to make these changes (this is how I made these changes on my WM6.5 HTC Touch Diamond2).

Be sure to perform a soft reset after you make these changes.
