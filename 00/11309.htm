b j murray say:

Tisky,

SOLVED.  Thanks for your help.  Yes, it appears that Word, or Windows, whichever, only shows the tail portion of an over-sized file name, including the full file path.  That size limit is supposedly 255 characters and, supposedly, there should be a warning or error if that size is attempted to be exceeded.   It appears that truncation of the file name is the only 'warning' although it was also sometimes giving filenames from other documents in that folder instead of the current document truncated name and that was causing overwrite havoc.

I have complex iterations of legal patent documents that need to show its precedent action in the file name along with a rather extensive folder / sub-folder arrangement.  This was evidently causing the full path name to exceed 255 although, again, there was no warning of that.   It's likely few folks have touched that limit to experience this issue..

Here is a helpful hint I found that states that limitation well.

http://word.tips.net/T006397_Limits_on_Path_Length_in_Word.html

Case closed.  Thanks for your and the others' help.
