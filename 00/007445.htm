Export all distribution group with delivery management setting Office365

ruel mayoya msft say:

Hi LinhNguyen,

Yes, you need to add RequireSenderAuthenticationEnabled parameter, to show the Delivery restriction status of a DG or mail-enabled security group.

Connect to Exchange online powershell and run this cmdlet:

Get-DistributionGroup | FT Name, RequireSenderAuthenticationEnabled

This will list all the Distribution Group and mail-enabled Security Group with their corresponding message delivery restriction status. True means external users can send messages to the DG.

There are other parameters which shows the delivery restriction to specific users (AcceptMessageOnlyFrom) and also which rejects messages from specific users (RejectMessagesFrom).

To show all these message delivery restrictions, run this cmdlet in PowerShell:

Get-DistributionGroup | FT AcceptMessagesOnlyFrom,AcceptMessagesOnlyFromDLMembers,RejectMessagesFrom,RejectMessagesFromDLMembers,RequireSenderAuthenticationEnabled

Now you can export these result to a CSV format by using the Export-Csv cmdlet.

Feel free to post back if you have further questions.

guessed say:

Templates (.dotx, and .dotm files) can be stored in Sharepoint but they don't work from there. Word needs any templates (or addins) to be sitting on C:\ or a shared network drive in order for the template to work. If you put the quick parts into your Building Blocks template, they will only be available to you because that file is sitting on your C:\ drive and is not accessible to other users.

Only a template can contain quick parts so saving it to a document (.docx file) immediately removes any quick parts you may have embedded in the doc. BUT, if that user's Word has access to the template that contains the quick parts then the docx file can still show the quick parts if the template is attached to the doc or loaded as an addin.

The recommended setup would be to save the template (which contains the quick parts) onto each computer in the User Templates path OR place it on a network drive and have each user point their Word Workgroup Templates path to that location. Then you create new documents based on your template and each will access the 'attached template' automatically. Those new documents can be placed into Sharepoint and each user can open those docs with the template functionality.

Hi Chad,

I have checked the logs you provided. The NDR message was generated in Office 365 to block the detected malware. The internal private IP addresses 10.x.x.x belong to the Office 365 mail servers that locate in the same site. The message is directly and internally delivered to the target server which holds your mailbox data.

Therefore, the public IP address is not displayed as it is an internal mail flow from Office 365 [postmaster@mydomain] to you [*** Email address is removed for privacy ***]. Also, internal mail flow bypasses the SPF checks in Office 365. So it is an expected behavior that the NDR message is delivered to your Inbox.
