PowerPoint: how to prevent the embedded video from re-showing - Microsoft Community

sean on ms community say:

PowerPoint: how to prevent the embedded video from re-showing frame 1

Several discussions out here are too old so I cannot update them with what I did to solve the problem.

THE PROBLEM: During a PowerPoint presentation, an embedded video starts fine, but right after it plays, it displays a glimpse of frame 1, as if the video is re-queued/re-wound, right before transitioning to the next slide.  What you want is a seamless, smooth transition to the next slide.

I discovered these articles while searching for a solution:

PowerPoint 2013 Video Playing Problem

How can I stop video rewind with a fade smoothly transition

Set the 'Play' options for a video in your presentation

A couple of those articles may have referenced the Poster Frame (when your video object is selected, under Format you'll see this button), and that got me thinking...what IS the Poster Frame?  That's the image you want to show when the video isn't running (like the default image on YouTube when the video isn't running).  It turns out, PowerPoint shows this image at the beginning and end of the video's run.

SOLUTION: Set your Poster Frame to be the same as your slide's background.  In my case, I set the background of my master slide deck to a custom PNG, but if you have something different like a stock image from one of Microsoft's themed deck templates, you can just right-click the slide to then save the background image somewhere on disk.

Then, while your video object is selected, go to the Video, Format menu, select Poster Frame, and locate that background image.  I also set Fade In and Fade Out of the video to 0.25 or something appropriate for a nice transition from the background image to the video and back again.  This way, the initial frame and final frame will MATCH your slide's background, preventing that annoying glimpse of the first frame once the video is done running.

CAUTION: This will give the appearance in your deck that the slide is empty, so I added a comment (under the Review menu) so there is at least a dialog bubble to remind me that the slide is NOT blank.

Anyway, I hope this is helpful to someone encountering the same problem.

clevsparrow say:

Open the Application folder

Right click on one of the MS applications (Word for example) and select Show Package Contents.

Open Contents / SharedSupport / Proofing Tools and delete all the language-related files not needed.

Back at the top of the Show Package Contents level, open the Resources folder.  Delete all the language folders (have .lproj suffix). Optionally open the DFonts folder and delete all the fonts there (you may want to leave the fonts in this folder for Outlook and Notes since there are fewer there.  I've experienced some rendering issues if I deleted all the fonts in DFonts in Outlook.). 

Repeat for each MS application.

This is really a new steroidal level of bloatware on Microsoft's part.  Supposedly this is so each MS Office application can be downloaded and run independently via the Apple App Store.   So now we have the replication of multiple language support files (lproj files and language font files) in EACH Office app not just in a central installer to install in centralized fonts folders. etc..

Essentially there are 400mb at minimum in bloat for each Office app (400 x 5 = 2GB) and for each update of the app when MS releases an update to Office.  MS dutifully re-adds ALL the deleted resources if you removed them so you will need to remove them again or just not apply the updates until absolutely needed.

techfreak_ say:

The Skype for Windows 10 UWP App comes package with the OS and loads more like a background service. You can stop it from loading and components will remain in a suspended stated utilizing little to no resources. If you restrict the startup background processes and exit out, I believe SkypeBackgroundHost is all that remains suspended until the next reboot. If you are looking for a totally detached version, I suggest you uninstall the packaged App and utilize the desktop version. Its behavior and controls mirror the classic version.

Keeping Skype for Windows 10

Stop it from loading at startup. If you do not want it to startup, you can stop if from launching by signing out every time. This will ensure that even if components of Skype are loaded in the background, they will load in a suspended state. If you do not want any components of that version to load, you can try to stop it from Startup Apps or disable the Skype App in your Background apps located in your Windows 10 Privacy settings.


Removing Skype for Windows 10

You can uninstall it from Apps & Features. You can also try to uninstall it by right clicking on it and selecting Uninstall. If the program keeps reinstalling when new users sign on or something specific to the build of Windows 10, you can try my removal tool (SRT (.NET 4.0 version)[pcdust.com]) by selecting Skype for Windows App and clicking remove. This will keep the Skype App from reinstalling until the next major update to Windows 10. Since the Skype App is packaged with Windows 10, it will be reinstalled with every major update.

suzanne s. barnhill say:

It would appear that what you have is not a table but a set of tabbed paragraphs in tabular form. You might find it more satisfactory to convert the text to an actual table, which would allow you to copy the entire table as a table. But the reason the "table formatting went lost" is probably that you have pasted the content into a paragraph that did not have the custom tab stops that the original paragraph used. You can rectify this by selecting the original table and pressing Ctrl+Shift+C (Copy Format) and then selecting the pasted material and pressing Ctrl+Shift+V (Paste Format), which will reproduce the formatting of the original "table."

Edit: From your other posting (http://answers.microsoft.com/en-us/office/forum/office_2007-word/copy-paste-not-working-properly/63853b02-1353-4c99-a984-5492f58f75d7) I see that what you are dealing with is a table and that Paste is not working properly for you, so I suppose the above will not be helpful.
