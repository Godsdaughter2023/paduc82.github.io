1. Connect your computer to the Internet, if it's not already connected.

If you have an Internet connection, the Zune software can then download useful details, such as track names and album art, for the CD that you're ripping.

2. Insert the CD into the CD tray of your PC.

Your Zune auto-detects the CD, and an album screen appears.

3. To begin ripping the CD, click the Start Rip button in the lower-right of the window.

Zune begins to rip the CD.

In a few minutes, your music is copied to your library, and you can remove the CD from your PC. The next time you sync, the newly ripped songs get copied onto your Zune. Now you can enjoy your music anytime, whether it's on your Zune or your PC.

The default file format for ripped music is Windows Media Audio (WMA), but you can tweak the default setting in a couple of ways:

Manage add-ons in Internet Explorer

http://windows.microsoft.com/en-in/internet-explorer/manage-add-ons#ie=ie-11-win-7

Follow these steps and try to restore the Favorites:

c. Right click on Favorites folder and click onProperties.

d. Under the Location tab, click on Restore Default.

e. Test the issue.

 

 

Method 3:

Reset Internet Explorer settings

a@work say:

Awesome. Thanks, guys! Below is what I have so far. I was able to remove the strip return carriage function, since using this method of copying the title didn't add the extra return carriage that I get doing it manually. This works well so long as there is a title placeholder on the active layout.

 

Sub mkTitle()
Dim osld As Slide
Dim oshp As Shape
Dim oTxR As TextRange

If ActiveWindow.Selection.Type = ppSelectionNone Or _
ActiveWindow.Selection.Type = ppSelectionSlides Then Exit Sub

If ActiveWindow.Selection.ShapeRange.Count <> 1 Then Exit Sub


Set oshp = ActiveWindow.Selection.ShapeRange(1)
Set osld = oshp.Parent
Set oTxR = oshp.TextFrame.TextRange

'is there a title placeholder?
If osld.Shapes.HasTitle Then
osld.Shapes.Title.TextFrame.TextRange = oTxR.Characters
oshp.Delete
Else
'Reset slide - is ther now a title?
CommandBars.ExecuteMso ("SlideReset")
DoEvents
If osld.Shapes.HasTitle Then
osld.Shapes.Title.TextFrame.TextRange = oTxR.Characters
oshp.Delete
End If
End If
'still no title
If Not osld.Shapes.HasTitle Then MsgBox "No Title Found", vbCritical
End Sub


 

I also created this variation that will copy the formatting and position of the selected object to the title rather than using the layout's title placeholder format. For some reason, it isn't working as well if the original object selected is a place holder object - It doesn't copy the aligment and the original placeholder gets a little buggy - So if anyone has any ideas on that one, please let me know.

 

Sub SwapTitle()
Dim osld As Slide
Dim oshp As Shape
Dim oTxR As TextRange

Dim L As Single
Dim T As Single
Dim W As Single
Dim H As Single


If ActiveWindow.Selection.Type = ppSelectionNone Or _
ActiveWindow.Selection.Type = ppSelectionSlides Then Exit Sub

If ActiveWindow.Selection.ShapeRange.Count <> 1 Then Exit Sub


Set oshp = ActiveWindow.Selection.ShapeRange(1)
     L = oshp.Left
     T = oshp.Top
     W = oshp.Width
     H = oshp.Height
Set osld = oshp.Parent
Set oTxR = oshp.TextFrame.TextRange

oshp.PickUp
'is there a title placeholder?
If osld.Shapes.HasTitle Then
osld.Shapes.Title.TextFrame.TextRange = oTxR.Characters
osld.Shapes.Title.Apply
With osld.Shapes.Title
    .Top = T
    .Left = L
    .Width = W
    .Height = H
End With
oshp.Delete
Else
'Reset slide - is ther now a title?
CommandBars.ExecuteMso ("SlideReset")
DoEvents
If osld.Shapes.HasTitle Then
osld.Shapes.Title.TextFrame.TextRange = oTxR.Characters
osld.Shapes.Title.Apply
With osld.Shapes.Title
    .Top = T
    .Left = L
    .Width = W
    .Height = H
End With
oshp.Delete
End If
End If
'still no title
If Not osld.Shapes.HasTitle Then MsgBox "No Title Found", vbCritical
End Sub

This issue seems to have occurred since the time you have change the alias for your account. In order to resolve this issue I suggest you to configure your account again using the new alias this time, and the same password in Outlook 2013. Delete the previously configured account and reconfigure the account with the new alias.

In case the above step does not work then I suggest you to setup your account with the new alias in a new Outlook Profile.

To create a new Outlook Profile you may refer to the following Microsoft article.

http://support.microsoft.com/kb/829918

Check if the issue resolves in the new Outlook Profile.

In case you face the same issue, then in order to resolve this issue setup the email account manually in the new Outlook Profile. Follow these steps: 

  1. To add the account manually in the new Outlook Profile click on File > Add Account.

  2. Select Manual setup or additional server types and click on Next.

  3. Select the 2nd option: Outlook.com or Exchange ActiveSync compatible service and click on Next. Complete all the fields with the relevant information. Note: In the Mail Server field under Server Information type: m.hotmail.com. User Name under Logon Information should be your complete Outlook.com email address including @outlook.com.

  4. Click on Next to complete the account settings.

  5. Close and reopen Outlook in the new Profile.  

Check if this resolves the issue.
