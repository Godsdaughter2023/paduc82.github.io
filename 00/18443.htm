ken sheridan say: It's very important as a newcomer to Access that you have a good understanding of the principles of the database relational model which govern the design of a database.  In particular you need to understand Normalization, which is the formal process by which redundancy in a database is eliminated.  The elimination of redundancy is crucial to the integrity of the database, as it leaves the door open to update anomalies.

In your case, repeating the non-key columns (fields) in more than one table introduces redundancy.  As Duane has explained, it is only necessary to store the foreign key value in a referencing table.  By virtue of the relationship with the referenced table the values of the non-key columns in the referenced table are known, without any need to repeat them.  Where you need to show values from both tables, e.g. in a form or report, the form or report's RecordSource property can be a query which joins the tables.

For a simple introduction to some of the basic principles and methodologies in the design of a relational database in Access you might like to take a look at DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

As its name suggests, this little demo file is a simple introduction to the topic.  For an introduction to Normalization see the Normalization.zip file in the same OneDrive folder.

Hi,

If you want to pull out the unique names to a new column you can use this array formula, see below for how to enter it. ARRAY enter the formula in cell E2 and drag down

=IFERROR(INDEX($A$1:$A$20,MATCH(0,IF(ISBLANK($A$1:$A$20),"",COUNTIF(E$1:$E1,$A$1:$A$20)),0)),"")

Note if you put the formula into a different cell then the bold bit in the formula must refer to the cell above where you put it.

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

rik_uk say:

It is possible to over ride, but only with a little snippet of code...

Place the following into a standard module of the work book:

Function TestColor(MyRange As Range) As Boolean
    If Range(MyRange.Address).Interior.Pattern = xlNone Then TestColor = True
End Function

Then in the conditional formatting add a formula rule [change the cell ref to suit] with no formatting using this equation...

=NOT(TestColor($A$1))

Make sure it is the first rule in the list and the stop if true is selected. Now if you color a cell it will default to the selected color.

Hope this helps, worked for me when tested.
