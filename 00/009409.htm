john w. vinson say:

Access simply isn't designed to be shared over VPN. It requires a FAST, High Capacity network connection. Even a cheap LAN is 100MBps - my internet connection is often 3% of that.

 

That said, a frontend and a backend are inherently and by their very nature two separate databases. The data is stored in one (shared) database, and the user interface in a different, separate database. The frontend doesn't have any data; the backend doesn't (usually) have any forms or reports! For Microsoft's take on it see http://office.microsoft.com/en-us/access-help/ways-to-share-an-access-database-HA010279159.aspx; for other takes see http://allenbrowne.com/ser-01.html or http://www.kallal.ca/Articles/split/. These links may look old but that's just because this split database architecture has been standard operating procedure for at least a decade!

 

If you must have users at different locations connect to the same data over a Wide Area Network, then you'll need to consider using Sharepoint (as suggested in the Microsoft link) or putting the data in a SQL/Server instance and linking to it: not trivial but not as big a job as it once was.

graham mayor say:
The problem is that collective nouns like 'staff' can be singular or plural and grammatical rules of this type are not as selective as you might wish. This is not a programming oversight, but an intentional reminder that there is a possibility that you may have used the wrong word form. Your sentence would be better written as 'the staff members are annoyed'. Then there can be little confusion as to your intended meaning.

Grammar checking can only ever be advisory. If you feel the version you wish to use is correct in the circumstances then ignore the grammar checker ... or turn off the checking as you type altogether.

Alternate user locations for tablets - Microsoft Community

donald howard - pgh say:

Alternate user locations for tablets

Here is a trick for users using a tablet or ultra mobile device with limited internal storage memory.

First thing I have found is windows 10 when rebooting will not recognize an external storage device (SD / Micro SD Etc...) when you first login to windows unless you add a shortcut and change the way you shut down your device. Full directions below:

First if you want to change your storage default locations (IE: Desktop, Download, My Music, My Pictures, Etc...) you will need to create a directory on your C drive called SD (Or you can name it anything you want, Just easier to identify later)

Next go to Control Panel, Administrative Tools, Computer Management, Disk Management. Right click on your SD/External storage location, Change Drive Letter and Paths, Add, Check Mount in the following NTFS folder, then type in the name/location of the folder you just created on your C drive. (Ex; C:\SD)

Now go to File Explorer, This PC, C, Users, (User Name), (Create a folder on your external device for each folder you see there), Right click on each of the folder names in the C:\Users\(User Name) and change the location to C:\SD\(Folder Name)

Now there is an issue with Windows 10 that it does not use a Hybrid Shutdown as Windows 8 does by default. You need to create this shortcut and use this to shutdown your device or your external storage locations will not be available before the desktop loads.

Right click on your desktop and create a new shortcut. type in "%windir%\System32\shutdown.exe -s -hybrid -f -t 00" in the location of the shortcut and name it "Hybrid Shutdown" or anything you want. If you use THIS shortcut to shut down your device, your external location will be available when the desktop loads and your locations will be available. This should help with the limited storage abilities of most Tablets and Ultra mobile devices.

This also comes in VERY HANDY if you have a problem with windows and need to reload your system, or converting to the Beta/Preview versions as all of your Documents, Downloads, Pictures, Etc... are saved to an external location and you can easily reset these locations after your install and your files are still there without having to reload everything. All I need to reload are my actual program files. I even save my Outlook.pst file to the external storage so that all I need to do is just reload my e-mail account and change the data file location. 

I hope this helps many people and make more tech savy people use the preview on more devices   :)
