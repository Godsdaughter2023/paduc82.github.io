doug robbins - mvp office apps & services (word) say:

Most definitely, the way in which a document is laid out can be affected by different Printer Drivers, even though the same physical printer may be used.  In fact a common source of printing problems in Word is a miss-match between the Word version and the printer driver, especially in the case of HP printers.

macropod say:

As Doug says, different printer drivers do affect the layout. And that can even be the case where different drivers are used for the same printer, as can occur if some users have Win 7, others Win 8 and yet others on a Mac, because each OS might use a different driver.

In situations where it's important everyone works with the same layout and they're not all using the same printers and drivers, you may need to distribute the document in PDF format.

First to cover some basics

#1 - Outlook 2016 (part of Office Home & Business) is completely separate and distinct from the Win 10 Mail app which does read a PST file which is only used by Outlook). There is no need to delete the Win10 Mail app

--> enter your email address,
--> click on <Advanced Options) immediately underneath the email address box
--> check the box for "Let me set up my account manually
--> Click Connect
--> enter the appropriate info (server, port settings etc) for the POP email account you want to configure. If you don't know what those settings are, either check the settings from your old PC or check your email provider's website for the info
--> be sure to browse to the PST file you copied instead of letting Outlook create a new PST file for the account

*** if you let Outlook configure the account automatically, it will do so using IMAP

One thing that you need to make sure of is that you copied the right PST file from the old system which if it only had one POP account and only one PST file, all info (email, contact, calendar etc) will be contained in that PST file.

john w. vinson say:

What constitutes a "match"? One field or more than one? Is the pluginid field unique, or might you have six (possibly different) records in the import table and eleven (possibly different) records with the same pluginID? If so you'll get all 66 possible matchings.

The field need not be the primary key, but - to prevent combinatorial explosions like this - it should be unique.

john w. vinson say:

That should be pretty simple, then. You would just create a new Query addng both the imported table and the issues table. You would create a Join line by dragging the IssueID field from the Issues table (hopefully this ID is the Primary Key of that table!) to the IssuesID field of the imported table.

You will now have all of the fields in each table available to be included in the query. You will be able to display them on a report, export them to a text or XML or spreadsheet external file, whatever you wish.

You should almost certainly NOT store the "findings" text redundantly in both tables. You're using a relational database - use it relationally! Each field/piece of information should be stored once, and once only; you would use Queries linking the tables in order to work with data from multiple tables together.

hazerd say:

You can revert the start menu/start screen with a simple registry edit.

  1. press [Windows Key] + R
  2. type "regedit" in the box and click run or enter
  3. On the left, expand the folders to reach HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced
  4. Make sure "Advanced" is selected then do Edit>New>DWORD(32-bit) Value
  5. It will create a new entry, rename it to "EnableXamlStartMenu" and leave the default value of 0, if the value happens to default to something different than 0 for you, make sure to set it to 0.
  6. Either restart your computer, sign out of your account and back in, or restart Windows Explorer from the Task Manager
