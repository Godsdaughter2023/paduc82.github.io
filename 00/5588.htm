marshall barton say:

Also, your code is incorrect because SourceObject is not a property of the form object, it is a property of the subform control:

   Me!Subform1.SourceObject = "Name"

boblarson say:

You don't use the Got Focus event.  You would check the value of the toggle.  But if you set up an Option Group with Toggle Buttons as the option to use, you can just check the value of the toggle button in the After Update event.

 

Private Sub YourOptionGroupFrameNameHere_AfterUpdate()

    Select Case YourOptionGroupFrameNameHere

            Case 1

                Me.Subform1.Form.SourceObject = "sbfrmName"

            Case 2 

                Me.Subform1.Form.SourceObject = "sbfrmID"

            Case 3

                Me.Subform1.Form.SourceObject = "sbfrmPosition"

            Case 4

                Me.Subform1.Form.SourceObject = "sbfrmDateStarted"

   End Select

End Sub

 

 

Also, I put in different names because, for one - you should NOT use NAME as the name of any object or field as that is an Access Reserved Word and can cause problems when you do use it like that.  Change the form name, now.

 

Also, you need to make sure that Subform1 is the name of your subform CONTROL (the control on the parent form which houses the subform).

lance cai msft say:

Hi KarizySt,

You can add a second time zone on your Outlook for Mac if you are an Office 365 subscription user and on version 16.11.0 (180311) +. Please upgrade to the latest version to get the feature (with Office 365 subscription).

To turn on dual time zones from the calendar:

  1. Right-click on the time scale in the day or week views.
  2. Select the zone and enter an short name for the zone
  3. Once you close the dialog, the second zone will appear in the calendar

You can also enable or change the second zone in Outlook's Preferences, Calendar. Note that the default zone for new appointments has nothing to do with the default zone used in the time scale. If you want to change the primary time zone, you'll need to change the Time Zone on the Mac.

However, if your boss is using Outlook for iOS, it is not feasible to use the timezone feature.
