a. user say:

Re:  out of memory

Excel has its own internal memory limits, so it is unable to use all computer memory available in many cases.
You have an array with ~1,500,000 elements.
I've found that exceeding ~300,000-500,000 elements is self defeating.
Fwiw, numeric arrays are less burdensome.
I believe MS put ~1mm rows on worksheets to boost sales; they are not there for actual use.

Suggestions...
1.  You use only column 1 and column 3 of the array, so maybe two single column arrays might work better.

2.  You also could use a Range object for Column 6 on Sheet 3...
      Dim rng As Excel.Range
      Set rng = Sheet3.Range(Sheet3.Cells(1, 6), Sheet3.Cells(500000, 6)).Cells
      And use it like this...  rng(i).Value = "OPC"

3.  [Edit-added] Breakup your operation into parts and run only 100,000 rows at a time

4.  If you get the code to run, execute it during lunch hour. <grin>
