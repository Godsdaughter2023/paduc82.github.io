bernard liengme say:

A simple approach would be to use Gord's method with a second sheet to do the conversion - much easier than hiding. Sample Excel workbook on my OneDrive site: https://onedrive.live.com/redir?resid=162119FF97286948!20444&authkey=!AKAROQsgMmDSvpM&ithint=file%2cxlsx

Or use VBA. Sample worksheet shown below; the VBA code for the User Defined Function is also shown

Function gradeavg(Grange)
  For j = Grange.Count To 1 Step -1
    If Grange(j) <> "" Then
      LastCol = j
      Exit For
    End If
  Next j
  
  For Each grade In Grange
    Select Case grade
     Case "A+":         mark = 15
     Case "A":           mark = 14
     Case "A-":          mark = 13
     Case "B+":         mark = 12
     Case "B":           mark = 11
     Case "B-":          mark = 10
     Case "C+":         mark = 9
     Case "C":           mark = 8
     Case "C-":          mark = 7
     Case "D+":         mark = 6
     Case "D":           mark = 5
     Case "D-":          mark = 4
     Case "E+":          mark = 3
     Case "E":            mark = 2
     Case "E-":           mark = 1
     Case Else:           mark = 0
    End Select
    TotalMark = TotalMark + mark
  Next
  gradeavg = TotalMark / LastCol
End Function

gord dibben say:

Maybe this event code will suffice.  Edit to suit.

Enter letter grades in A1:A20 to return score 1-15

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim r As Range
    Dim rr As Range
    Dim iVal
    Set r = Range("A1:A20")
    If Intersect(Target, r) Is Nothing Then
        Exit Sub
    End If
    vals = Array(15, 13, 11, 9, 7, 5, 3, 1)
    nums = Array("A", "B", "B-", "C", "C-", "D", "D-", "E")
    For Each rr In r
        iVal = ""
        For i = LBound(nums) To UBound(nums)
            If UCase(rr.Value) = nums(i) Then
                iVal = vals(i)
            End If
        Next
        If iVal <> "" Then
            rr.Value = iVal
        End If
    Next
End Sub

hansv mvp say:

I thought that the payments would have to be matched not only on the company and bill number, but also on the site name. The data in your original post seem to suggest this. But in your sample database that won't work since the site hasn't been filled in in the payments table.

Therefore you must delete the join on SiteName in he Balance1 query.

 

Also, you may want to make Balance2 a Totals query that sums the balance amount:

 

SELECT Payments.Company, 0 AS BillsID, Payments.SiteName, Sum(-[PaymentAmount]) AS Balance
FROM Payments
WHERE (((Payments.AgainstBill) Is Null))
GROUP BY Payments.Company, 0, Payments.SiteName;

 

Finally, you can add the contacts and sitedetail tables in a new query based on Balance3:

 

SELECT Contacts.Company, Balance3.BillsID, SiteDetail.SiteName, Balance3.Balance
FROM (Balance3 LEFT JOIN Contacts ON Balance3.Company = Contacts.ContactsID) LEFT JOIN SiteDetail ON Balance3.SiteName = SiteDetail.SiteID;

hansv mvp say:

You could create the following queries:

 

Balance1:

 

SELECT Bills.Company, Bills.BillsID, Bills.SiteName, Sum([BillAmount])-Nz(Sum([PaymentAmount]),0) AS Balance
FROM Bills LEFT JOIN Payments ON (Bills.SiteName = Payments.SiteName) AND (Bills.BillsID = Payments.AgainstBill) AND (Bills.Company = Payments.Company)
GROUP BY Bills.Company, Bills.BillsID, Bills.SiteName
HAVING (Sum([BillAmount])-Nz(Sum([PaymentAmount]),0))>0

 

Balance2:

 

SELECT Payments.Company, 0 AS BillsID, Payments.SiteName, -[PaymentAmount] AS Balance
FROM Payments
WHERE Payments.AgainstBill Is Null

 

The 0 value of BillsID represents Advance (we can't enter a text value since BillsID is a number field)

 

Balance:

 

SELECT * FROM Balance1
UNION SELECT * FROM Balance2
ORDER BY Company, SiteName, BillsID
