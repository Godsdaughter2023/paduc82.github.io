Can I change the user mailbox to a shared mailbox but enable the Active Directory account (Disabled by default on a Shared Mailbox)? dileepakariyawasam say:

Hi Christopher,

As Alex suggested you can stick to converting the shared mailbox and removing the license.

If you want the user account associated to the converted shared mailbox to use a mailbox yes you need a license as shared mailboxes cannot be mapped to a AD User Account for login. Shared mailboxes do not have login accounts.

But if you want to keep the AD user account synced to cloud without using a license, you can very well do that too. Assigning a license to the Service account will let the service account have access to the features based on the license you assign to it. But if you don't you should be completely okay to operate the service account synced via DirSync on Office 365 without a license.

Hope this clarifies your concerns.

I suspect the issue is due to a new feature we just shipped a couple weeks ago: support for Freeze Panes. Please see this forum thread for more info. 

What I would suggest you do is open the file in the Excl desktop app and remove all freeze panes, then set your freeze pane to be smaller than the viewport of the Excel Web App.

Here is what you can do:

 ·         Open the file in Excel desktop app

 ·         Go the sheet in your workbook where you are having issues viewing

 ·         In the ribbon do ‘View | Freeze Panes’

 ·         Click ‘Unfreeze Panes’ to clear out the settings

 ·         If you still want to have Freeze Panes, then add them back by placing your mouse where you want the freeze pane to be anchored, but choose only a few rows or columns so the freeze pane area is  smaller than your browser viewport

 ·         Save the file and then view it in Excel Web App

 

Let me know if this works for you.   Feel free to get back to us with additional questions.

 

Thanks for using Excel Web App.
