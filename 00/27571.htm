dirk goldgar say:

Thanks for sending me the database copy by e-mail.  I've sent back a version that works.   The only real problem was that the unbound text box MAXft, in the absence of any format, was being interpreted by Access as having a Text data type, and so the comparison of the calculated number in WOlinealFT with that text value was failing.  All I had to do fix it was set the Format property of MAXft to "General Number", and now Access interprets it as a number data type and performs the comparison correctly.

I also commented out the lines of code in the form's AfterUdate procedure that were doing this validation, since we are now doing it in the check box control's AfterUpdate event.

I added Option Explicit at the top of all the modules.  That directive is very helpful in avoiding bugs caused by undeclared variables (though you didn't have any -- good for you!).  You'll want to set the "Require Variable Declaration" option in your VB Editor's options dialog, so that all new modules will automatically have that directive set.
