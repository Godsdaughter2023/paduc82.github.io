In case any runs across this post outside of TechNet, here was the fix:

Our domain server is on 2008 R2. I have heard that this applies to people running up to 2012 as well.

Create the following registry entry:

[HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\System] "AllowDomainPINLogon"=dword:00000001

Set the Group Policy items in Computer Configuration>>Administrative Tools>>Windows Components>>Windows Hello for Business to "Not Configured"

Fixing this issue is very simple. Here was what I've done to start the Print Spooler:

 

1. Click Start, type “services.msc” and hit Enter.
2. Scroll down to Print spooler.

3. Right-click Print spooler, then select Properties.

4. Select the Log On tab, and uncheck the Allow service to interact with desktop setting.

(Microsoft recommends that you do not change the Allow service to interact with desktop setting. If you allow the service to interact with the desktop, any information that the service displays on the desktop will also be displayed on an interactive user's desktop. A malicious user could then take control of the service or attack it from the interactive desktop).

 

5. Once you have uncheck this setting, select the General tab and click "Start" (the Print Spooler should be able to start without popping up the Error 1053 again).

Balloon tips - Microsoft Community

arpeggioshredder say:

Balloon tips

How do I enable balloon tips in the taskbar? I can hear the sound but there is no balloon. Thanks
davelkp say: As I understand it balloon tips are enable or disabled for the whole computer; it's all or nothing.  They are controlled by a setting in the registry.  To enable balloon tips do the following:

  1. Click Start, click Run, type regedit, and then press ENTER.
  2. Locate the following subkey:
    HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced
  3. Right-click the right side pane, create a new DWORD value, and then name it EnableBalloonTips.
  4. Double-click EnableBalloonTips, and then give it a value of 1.
  5. Close Registry Editor.
  6. Log off Windows, and then log back on

Editing the registry is dangerous if you don't know what you are doing.  If you aren't comfortable Microsoft will do it for you using one of their Fixit tips.  It's Fixit Tip 50048.
