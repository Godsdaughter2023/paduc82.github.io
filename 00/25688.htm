jllatham say:

GIVEN: That my assumption about the layout of your data in my previous post is correct, you can set up a table using formulas like this.  This example assumes the table would be on the same sheet.

       G                         H                I                      J                       K                   L
1   From                  To             1-Way          Tickets              Round Trip    Tickets
2    London    Manchester      357                   8                      889                  20
3    Manchester   Londong     532                 12                     889                   20

Now for the formulas for row 2, columns I, J, K and L (and once they are entered into row 2 you can fill them down the sheet as far as you need, You just need to change the end reference to row $100 in them to the row number that your lists in column A-D occupy.  They must be the same row # in all formulas).

I2:    =SUMPRODUCT(--($A$2:$A$100=$G2),--($B$2:$B$100=$H2),($C$2:$C$100))

J2:   =SUMPRODUCT(--($A$2:$A100=$G2),--($B$2:$B$100=$H2),($D$2:$D$100))

K2:
=SUMPRODUCT(--($A$2:$A100=$G2),--($B$2:$B$100=$H2),($C$2:$C$100)) + SUMPRODUCT(--($A$2:$A100=$H2),--($B$2:$B$100=$G2),($C$2:$C$100))

L2:

=SUMPRODUCT(--($A$2:$A100=$G2),--($B$2:$B$100=$H2),($D$2:$D$100))  +  SUMPRODUCT(--($A$2:$A100=$H2),--($B$2:$B$100=$G2),($D$2:$D$100))

 

All you have to do is enter start/to locations into columns G and H and the formulas will compute the one-way and round trip costs/ticket values for you.

Now, for routes like London - Paris - Leeds - London, well that's yet another problem, but the solution can be derived using similar logic.

Unable to sign in from a different device when traveling - Microsoft Community

melindawaltner say:

Unable to sign in from a different device when traveling

Why do I get a message telling me that Microsoft detected "unusual activity" on my hotmail account when using a different device while traveling? Very inconvenient. I want to switch this "service" off.

zigzag3143 (ms -mvp) say:

We have recently implemented a security update called Familiar location protection to keep our customers’ accounts secure. The message "You haven't signed in from this location recently." will occur if you are signing in from a location more than 300 miles from your normal location. The Microsoft account service will prompt you for an additional challenge to complete the log in. Codes can be the date of birth, or receiving a code sent to your SMS number or alternate email if configured.

 

To prevent this you will need to configure your computer as your trusted PC and other security information for added protection to your account.

 

With the new system update, you can remove old inaccessible security information on your account automatically after 30 days. To do that:

 

1.       Log in to this link.  

2.       Click Delete on the security information you do not have access to. (Alternate email, Trusted PC, Mobile number)

3.       Select  "No, I Don't have access" button. (The date on when the security information will be removed will appear )

4.       Click Delete Security Info.

5.       Update the account with new Security Info.

6.       Click Save.

 

You just need to wait for 30 days for all the information to be removed and the new security information to be automatically confirmed.
Text by Ann_N

jashion say:

Hi Iwishiweresmarter,

Yes, there is.

You can try the following command:

Get-Content c:\test\users.txt |  ForEach {Set-MsolUser -UserPrincipalName $_ –UsageLocation “US”; Set-MsolUserLicense -UserPrincipalName $_ –AddLicenses “tenant:ENTERPRISEPACK”; Set-MailboxRegionalConfiguration -Identity  $_ -Language 1033 -TimeZone "Eastern Standard Time" }

Note:
1) In you users.txt file, you need to put each UserPrincipalName in a separate line.
2) When assigning licenses to the users with AddLicenses, we can Use the Get-MsolAccountSku command to retrieve a list of valid licenses.
3) To configure the mailbox, we may need wait a moment for the license provisioning to take effect. So I suggest you run this part separately after that.
Get-Content c:\test\users.txt |  ForEach {Set-MailboxRegionalConfiguration -Identity  $_ -Language 1033 -TimeZone "Eastern Standard Time" }

4) About the UserLandingPageIdentifierForO365Shell parameter, we can only use it with Get-MsolUser to display the settings of the landing page, rather than change it.

In fact, we have a dedicated forum focusing on PowerShell scripting. To ensure you get more professional support, I would suggested you post in the Windows PowerShell forum.
