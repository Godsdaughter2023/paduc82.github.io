Ignore the previous reply.

The 1 KB files are AUTOMATICALLY generated by Word when you edit a file. They are known as "LOCK FILES". They are NOT a symptom of malware! They are supposed to be deleted when you close the file normally.

There are also some other "temp" files created while editing. Files that start with

~$filename.docx

~WRD####.TMP

~WRL####.TMP

They are also normal.

If you see any of these files, they are an indication that Word has abended and they may be used to recover some of the work in the file.  If you have not had a recent Word crash you can delete them to free up space

Description of how Word creates temporary files: KB 211632

daniel g. grau say:
It seems that this black-box phenomenon is the modern equivalent of an old hat—opening the original .doc with Word 2004 or Word v.X renders the image as the well-known "red X". This problem only happens with .doc documents. Here are several solutions:

  • Open the file in OpenOffice and resave, then open in Word 2011
  • Open the file in OpenOffice, then cut and paste the pictures into Word 2011
  • Open the file in Word 2011 and resave as .docx
  • Open the file in Word 2011, then cut the picture and paste it into a text box

However, if you save the document back into .doc format (or repaste it there from a .docx), the black box will reappear. In fact, graphic programs like Preview and GraphicConverter will display the pasted image as black. This is how I fixed the problem:

  • Paste the image into a blank Word document (you can leave it unsaved)
  • Copy it from there and paste into a TextEdit blank document
  • Do Select All (or Command A), copy and paste back into the original document
