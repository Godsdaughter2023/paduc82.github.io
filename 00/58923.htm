robin r. carlin say:

IP,s address taken by another user__

Hello,

How to be shure that my IP(s) address are not being used by another parson(computer) without my authorization, If they are Is this considered fraud? Who do i contact in this case?
steve winograd say: There are two types of IP addresses:

  • A computer on a local area network has a private IP address .  This address isn't accessible from the Internet, and the same one can be assigned on any number of local area networks.  It's typically assigned by a home broadband router.
  • A computer or router that connects directly to the Internet has a public IP address .  This address is assigned by your Internet service provider.  Unless you've specifically arranged with your ISP to have a static IP address (one that never changes), you get a dynamic IP address, which can change at any time.

So the only way that someone could be using your IP address on the Internet would be if you have a static IP address.  In that case, you can check with your ISP to see how it's being used.

Unfortunately, this is not actually the case, except in limited circumstances.
Every machine from a Large OEM (Dell, HP, Toshiba, etc.) comes with two working Keys - one with which the machine is preinstalled at the factory (the OEM_SLP Key), and which uses data from the machine's BIOS to self-activate; and one on the COA sticker (the COA_SLP Key) which should only be used if the OEM_SLP Key fails for some reason. Note that all suppliers who use the OEM_SLP system MUST, according to their contract with MS, also provide the COA sticker, and attach it to the case (MS used to insist on it being on the outside of the case, but since the rise of the laptop, that restriction appears to have been lifted, and some manufacturers have taken to putting the stickers inside the battery compartment, or even inside the machine's case in desktops)
While Key Finders will discover (usually) the key in use, they do not tell the user what type of key it is, as they merely read the relevant bits in the registry and decode them. You may be able to find out what sort of Key it is by doing a Google search on it.
Using the Manufacturer's Recovery Disks will almost always automatically reinstall using the OEM_SLP Key.
The OEM_SLP Key is not activatable unless the BIOS is the correct one - it will not activate over the internet, or by phone.
The only sure way to find which key type is in use is to use the MGADiag utility from MS - which will also give details as to the activation and validation status of Windows, and whether an activation/validation problem is caused by an incorrect Key, a software problem, or a BIOS problem (or counterfeiting).
The chances are that your problem was caused by an OEM_SLP Key not being verified by the BIOS data, either because you used the wrong disk, or had a BIOS update which was not properly adjusted for Win7 activation, or because the system couldn't properly read the BIOS SLIC table which verifies the install.
 
Recovery disks never, in my experience, use a Default Key (which is another beast altogether) as (quite apart from anything else) it would cause too many calls to their helplines, and hence cost too much to support!
Default Keys are used when installing from OEM System Builder or Retail disks - when the user opts not to input their own Key during the install. The system uses a Default Key (more properly named a 'Keyless Install Key' by MS) which allows the user to play with the system for 30 days - but which cannot be activated under any circumstances. he only way to activate the system is to change the Key to a valid one, and go through the activation process.
 
I hope that helps explain the process a little?
