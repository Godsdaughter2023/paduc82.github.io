lemp say:

The symptoms you describe can be caused by:

  1. a bad port on the modem/router
  2. a bad Ethernet cable
  3. a bad network adapter in your computer
  4. a bad driver for the network adapter
  5. a misconfigured router
  6. a misconfigured firewall or antivirus program on the computer

and probably a number of other things that I can't think of at the moment.

Starting with the easy ones first, disconnect the Ethernet cable from the working desktop (but leave the other end connected to the modem/router) and connect it to the laptop.  If you still can't connect, you know it isn't numbers 1 and 2 (because the port and cable work with the other computer).

Numbers 3 and 4 are more difficult to diagnose.  You can open Device Manager (Start > Run > devmgmt.msc > OK) and double-click on the entry for the adapter.  Look at Device status and Device usage.  What these should say is "This device is working properly" and "Use this device (enable)."  If that's what is shown are your system, you can try the "Troubleshoot" button, but in my experience, that's pretty useless.

Look closely at the Ethernet port on your computer.  There should be two small LED lights.  With the computer connected to your router, is either light lit or blinking?  If so, what color(s)?

With the laptop connected to the router, please do the following:

  • Open a Command Prompt window (Start > Run > cmd > OK)    
  • Type the following in the black Command Prompt window and press Enter after each line
         ipconfig /all > "%userprofile%\Desktop\ipinfo.txt"
         exit
  • Because the laptop can't connect to the Internet, get a USB flash drive (thumb drive) and copy the file ipinfo.txt from your Desktop to the flash drive
  • Bring the flash drive to the computer that can connect to the Internet, open ipinfo.txt and copy/paste the data from that file into your Reply here.

To investigate number 5, you will need to access your modem/router's web configuration utility using the desktop computer that works properly.  If you don't know how to do this, please provide the make, model, and hardware version, if any, from the label on the back or bottom of the modem/router.

You need to look at the router's settings for its DHCP server and the router's "filtering" settings.  Most routers allow you to configure the number of IP address that may be allocated by the router's DHCP server.  It's a bit unusual to have that number set to only a single IP address, but it's possible.  Most home routers can also be configured to "filter" -- i.e., allow or disallow -- connections, generally based on MAC address. 

With respect to number 6, what is the name and version of the antivirus program installed on the laptop?  What other security software is installed?  If all you have is the Windows firewall, temporarily turn it off and see if you can access the router's configuration utility.  To turn off the Windows firewall:

  • Click Start > Run > firewall.cpl > OK
  • Check the "Off (not recommended)" box
  • Click OK

As long as you're behind your router, you're safe from intrusion because the router will act as a firewall.  If you're nervous doing this, disconnect the modem/router from the cable to the outside.  When you have completed your test to see if you can access the router with the firewall turned off, don't forget to turn it back on.

lemp say:

1.  The IP address you saw in the network connection details (169.254.115.153) is an "Automatic Private IP Address" (APIPA) that by default is assigned to the NIC by Windows when (a) the NIC is configured to obtain its IP address automatically from a DHCP server but (b) the NIC does not, in fact, get an address from the DHCP server.  

2.  I can't follow your description of your attempts to use different drivers. 

Does "When i tried win7 64bit, i tried dell's vista 64bit driver (falied) and then i used Broadcoms win7 64bit driver from their website (failed)" mean that you installed Windows 7 x64 on the laptop and then attempted to install these two drivers?  In what way did the installations fail?  That is, did the NIC just appear to not work or did you get some sort of error message?  If the latter, what was the message?

You also said that the Dell XP driver failed.  Same question:  does this mean just that the NIC doesn't appear to work or that an error message appeared?

Did you ever actually look at the NIC's status in Device Manager as I suggested earlier?

3.  I think that I know what the answer to this will be, but ...

  • Take your second (working) router and disconnect all cables to it except for one known-good Ethernet cable from one of the router's LAN ports to the NIC on your laptop
  • Use either ipconfig /all in a Command Prompt window or simply click once on the icon for the Local Area Connection in "Network Connections" and look at the "Details"
  • Do you get an IP address OTHER THAN 169.254.x.y?

Assuming that the answer to my question in #3 above is "no" and based on my understanding of your descriptions of your various tests, I don't think you have a problem with your modem/router ... but I can't tell you what your problem actually is.  My guess would be that it's a failed NIC.  The Dell XPS M1710 is supposed to have wi-fi capability.  Have you attempted to connect to your modem/router using wi-fi, and if so, with what result?

The usual reasons that a NIC does not obtain an IP address from the DHCP server (see # 1 above) are:

  • The DHCP server is disabled or broken.  We know that this is not the case for you because you have another computer that works properly (unless, of course, you had assigned a static IP address to that other computer)
  • The data connection between the DHCP server and the NIC is not functional.  In wi-fi situations, this is typically caused by a failure to authenticate (i.e., you don't have the correct password or other security credentials).  In a wired situation, the problem is mostly likely physical.  That is, a bad port on the modem/router, a bad cable, or a bad port on the NIC.  Again, we know that it's not the port on the modem/router or the cable because these two items work with your other computer.  It doesn't seem to be a bad port on your NIC because the NIC's LED status lights appear normal.
  • The DHCP server is refusing to allocate an IP address to the NIC.  This could be because there's a filter configured in the modem/router or because the DHCP's server has already allocated the maximum configured number of IP addresses.  Although you didn't check for either of these, if the second (working) router also doesn't allocate an IP address to the NIC (see question 3above), the odds are pretty low that both routers have the same filter configured.

The only other think I can think of is some failure internal to the NIC.  You say that the NIC passed a Dell diagnostic, but it's hard to know how comprehensive that diagnostic is.  The failure could be with a driver, but I can't determine from your descriptions whether you have installed a functional driver or not.  I think you probably have or the results of some of the tests you've tried would have been different.
