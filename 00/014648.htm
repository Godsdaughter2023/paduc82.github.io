There is no need to export in order to save the data for use on a new PC.

Exporting loses some data & settings, as does any Import.

Your problem has nothing to do with heat. If a component has failed due to heat, the PC will be unusable.

With outlook closed you simply copy the pst files to an external source, eg thumb drive.

Then on the new PC you copy those pst files to a documents location.

Then when configuring outlook on the new PC, for first use, once you have added the mail account, manually with the settings provided by your mail provider, in the data files tab you add the copied primary pst file, probably named outlook.pst, and set this as the default, remove any created data file, then continue.

Macro bug in Excel with Cells.Find() - Microsoft Community

l0basetom say:

Macro bug in Excel with Cells.Find()

There is a bug in Excel 2010 that I can reproduce every time.

Steps to reproduce:

  1. Open a new spreadsheet.
  2. Record a macro.
  3. Press Ctrl+F, then Enter.
  4. Stop the macro.
  5. Play back the macro.

You will get this error:


Run-time error '91':
Object variable or With block variable not set


Here is the macro code:


Sub Macro1()
'
' Macro1 Macro
'

'
    Cells.Find(What:="*", MatchCase:=False _
        , SearchFormat:=False).Activate

End Sub


As soon as you enter something in a cell, then the macro runs without incident. It appears the bug only reveals itself when Excel 'Find' cannot find anything.


UPDATE:

I found out you get this error because Cells.Find() returns Nothing if no match is found, and Nothing has no method .Activate. This should be taken into account when recording macros.

0 people found this helpful

jp ronse say: Hi,

It is not a bug in Excel 2010. This behavior exists in all versions AFAIK.

To catch this "error" use following code

Sub Macro1()
   Dim rng As Range
  
   Set rng = Cells.Find(What:="JP", MatchCase:=False _
        , SearchFormat:=False)
   If Not rng Is Nothing Then
      MsgBox Prompt:="found"
   Else
      MsgBox Prompt:="not found"
   End If
End Sub
