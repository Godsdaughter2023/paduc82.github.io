DateValue Function and the Data Mismatch error - Microsoft Community

kejo63 say:

DateValue Function and the Data Mismatch error

I am trying to run the following query:

UPDATE tbl_Hospitalization_Emergency_Import INNER JOIN tbl_Temp_Hospitalizations ON (tbl_Hospitalization_Emergency_Import.SITE_CD = tbl_Temp_Hospitalizations.HOSPITAL_ID) AND (tbl_Hospitalization_Emergency_Import.VISIT_DT = tbl_Temp_Hospitalizations.ADMITTED_DATE) AND (tbl_Hospitalization_Emergency_Import.SAC_ID = tbl_Temp_Hospitalizations.SAC_ID) SET tbl_Temp_Hospitalizations.RESPONSIBLE_DX = 15, tbl_Temp_Hospitalizations.DIAGNOSIS_CLASS = "ICD10CM", tbl_Temp_Hospitalizations.ADMITTED_DATE = DateValue([ADMITTED_DATE]), tbl_Temp_Hospitalizations.DISCHARGED_DATE = DateValue([DISCHARGED_DATE])
WHERE (((tbl_Temp_Hospitalizations.DIAGNOSIS_ID)=[tbl_Hospitalization_Emergency_Import].[SEC_DIAG_15_COD]));

The problem is that I am getting the Data Mismatch error. I only get the error when I include the DateValue function. It is essential that I truncate the date/time value and only enter the date!

Any help will be appreciated.

marshall barton say:

Check to make sure your date fields are really Date data type.  If they are Text fields, you will probably get that error trying to set a text field to a date value.  Also make sure that there are no records that do not have a valid date in the date fields.  If the date fields are really Text fields that just look like a date, DateValue will choke on anything that can not be converted to a date.

So, first check to make sure they are not Text field.  If they are, then run a query to find the records with invalid dates.
SELECT tbl_Hospitalization_Emergency_Import.*, tbl_Temp_Hospitalizations.*
FROM tbl_Hospitalization_Emergency_Import INNER JOIN tbl_Temp_Hospitalizations ON (tbl_Hospitalization_Emergency_Import.SITE_CD = tbl_Temp_Hospitalizations.HOSPITAL_ID) AND (tbl_Hospitalization_Emergency_Import.VISIT_DT = tbl_Temp_Hospitalizations.ADMITTED_DATE) AND (tbl_Hospitalization_Emergency_Import.SAC_ID = tbl_Temp_Hospitalizations.SAC_ID)
WHERE (((tbl_Temp_Hospitalizations.DIAGNOSIS_ID)=[tbl_Hospitalization_Emergency_Import].[SEC_DIAG_15_COD])) AND ( Not (IsDate(tbl_Temp_Hospitalizations.ADMITTED_DATE)
         AND IsDate(tbl_Temp_Hospitalizations.DISCHARGED_DATE))

If that finds any problem dates, fix them before trying to proceed.

If you must use Text fields for the dates, then use the Format function to convert the DateValue results to an acceptable style date.

If any records have a Null in your date fields, DateValue would complain about invalid use of Null.  So I don't see how you could get a data type mismatch from real Date type values.

Open CCleaner Applications Tab, Under Windows uncheck MS Antimalware...that should prevent CCleaner from deleting the MS Antimalware scanning history files.  The actions of CCleaner  should not have any effect on the antimalware protection provided by MSE.

Per suggestion of MSE User MartinHampton you may also need to uncheck MS Management Console.

You may need to run a "Quick Scan" and possibly re-boot the computer to re-establish a scan history.

Scroll to Quick Care, Privacy Sweep, go to Windows, uncheck MS Antimalware and MS Management Console.
