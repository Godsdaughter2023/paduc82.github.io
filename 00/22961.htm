I don't see how because reapplying the notes master (even manually) will destroy bold etc.

If you just need to change the font size you can just say:

Sub Notes_Text()
Dim osld As Slide
Dim oshp As Shape
For Each osld In ActivePresentation.Slides
For Each oshp In osld.NotesPage.Shapes
If oshp.Type = msoPlaceholder Then
If oshp.PlaceholderFormat.Type = 2 Then

End If
End If
Next oshp
Next osld
End Sub

With that kind of useage you should expect two days life with this model. The battery life quoted is in ideal conditions and are never a true reflection on how someone uses a mobile.

Doing the following things will drain battery power quickly:

1) Bluetooth always set to on.
2) Data connection set to "always on"
3) Screen savers with long time outs
4) Listening to music (mp3 or radio) with the headset or loudspeaker
5) Constantly using it so the screen is lit for a long period of time.
6) Being in a poor signal area which causes the phone to constantly search for a network.

You should also check that any power saving options are enabled in the settings menu.

Is there a VBA that could be applied the first time Word starts that could make changes to this field?

charles kenyon say:

I would recommend putting the following code into the "ThisDocument" part of each template:

Option Explicit

Private Sub Document_New()
    ' Update AutoText fields in document including headers and footers

    On Error Resume Next ' in case no fields
    For Each oStory In ActiveDocument.StoryRanges
    ' This goes into headers and footers as well as the regular document
        Do
            For Each oField In oStory.Fields
                If oField.Type = wdFieldAutoText Then
                    oField.Locked = False
                    oField.Update
                    oField.Locked = True
                End If
            Next oField
            Set oStory = oStory.NextStoryRange
        Loop Until oStory Is Nothing
    Next oStory
    On Error GoTo 0
End Sub

This will update any AutoText fields in the new document and then "lock" them so that they will keep the content from the user who created the document. You do not want to run this on the template, itself, because you do not want the AutoText fields to be locked when the document is created until after they are updated.
