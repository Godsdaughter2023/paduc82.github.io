md5hash say: You won't be able to get it to work in safe mode. I'm having the same problem. The problem is that there is a Windows Service called Office Software Protection, or osppsvc.exe, that needs to run briefly every time an Office program is started up to make sure you have not tampered with the Office files.

This service, as with most services, does not and can not run in Safe Mode.

Basically, it seems that Microsoft will not allow users to run Office programs in Safe Mode anymore - in order for osppsvc.exe to work, you can't be in Safe Mode, and in order for Office to work, osppsvc.exe needs to work.

It's a vicious cycle, and I disagree with Microsoft's handling of things. I need to diagnose a client's machine for 24 hours in Safe Mode, but she still needs to work on her files. Why shouldn't she be able to use Word in Safe Mode? Microsoft, you need to change the osppsvc.exe service so that it's not required for Word/Office to function while in Safe Mode.

Sorry Wendy, not the answer you wanted I'm sure.

md5hash say: I take back my previous response. It seems there is a way to trick the Office Protection Service into running in Safe Mode - you add the service file into the whitelist. Read the instructions here.

In my case, adding a single key named osppsvc with a string value of "service" allows me to leave Word 2010 to run without issue for the past 5 minutes.
