jay freedman say:

Hi Craig,

 

I wouldn't say that you've boxed yourself in by concentrating on legacy form fields, but you should learn and use content controls as an additional (and often better) tool. Here's a short (and highly subjective) summary of the strengths and weaknesses of each type.

 

Legacy Form Fields

Pluses: 

  • Are compatible with all versions of Word for Windows at least back to 97, and Word for Mac.
  • Can be used with formula fields to do computations (totals, averages, etc.) without macros, via "Calculate on exit" setting.
  • Have unique bookmark names that can be used in IF and REF fields, or as names of the fields in macros.

Minuses:

  • Require protection "for filling in forms" to make them active, which disables a number of features of Word (spell checking, inserting pictures, etc.) even in sections that are excluded from protection.
  • Don't have some field types that are available with content controls (see below).

Content Controls

Pluses:

  • Offer rich text, picture, date picker, building block, and repeating section types that aren't available in legacy form fields.
  • Other content controls, tables, and other objects can be inserted inside a rich text control or a repeating section control.
  • Don't require protection of any kind to make them active. Protection similar to forms protection, but without disabling other functions, is available by using the Group button on the Developer tab or by using the "No changes" with exceptions in the Restrict Editing pane.
  • Default to having "placeholder text" to show where to fill in.
  • Can be "mapped" to XML nodes in the document file, enabling immediate updates of repetitions throughout the text (see http://gregmaxey.mvps.org/Repeating_Data.htm).

Minuses:

  • Instead of entry and exit macros that can be called from specific form fields, all content controls in the document are handled by a single OnEntry event and a single OnExit event, which pass the control object as a parameter. That object then must be tested to see whether it's one that needs attention.
  • Because the Title and Tag values of content controls aren't necessarily unique, the methods for selecting a specific control in a macro are more complex than the addressing of legacy form fields.
  • Certain content control types (check box, repeating section) are not supported when opened in earlier versions of Word for Windows than the version that created them.
  • There is no support for content controls in Word for Windows 2003 or earlier, or in Word for Mac 2011 or earlier.
  • Macro support for content controls in macros in Word for Mac 2016 is limited (see this thread).

There's a lot of information about working with content controls in the articles at https://gregmaxey.com/word_tips.html.

jrsyangl say:

Some entries in my index are incorrect.

Original Title: index has wrong page

 Some entries in my index are incorrect. I think I have a clue to what is going on. My text is mostly in tables (so I can have independent columns of text). Some of these tables span multiple pages. The index seems to be pointing at the page of the top of the table in which the text appears. So if the text is on page 6, but the top of the table is on page 5, the index points to page 5. But how do I fix this? Microsoft probably calls this a feature, it seems like a bug to me. Any help would be appreciated.

suzanne s. barnhill say:

There are two ways to do this. In each case I would suggest working on a copy of the document.

  1. The first method is manual and tedious. Select the first non-heading row of the table. Right-click and choose Insert Rows. You'll get a new row above the existing one. Drag the first paragraph from each column into the corresponding cell in the new row above. If you see a paragraph mark, delete it so that you have only the end-of-cell marker in the cell. Then repeat the process, adding a new row above the original one and dragging paragraphs. New rows are by default added above the selected one, so this is the easiest way to proceed. This will preserve your existing formatting.
  2. The second method is a little scarier but more efficient. Use Convert Table to Text, separating with paragraphs. This will give you all four columns' worth as a long continuous text. Select each block and convert it to a single-column table, separating at paragraphs (to get one row per paragraph), then combine the tables as previously described. This may well not preserve your existing formatting, depending on to what extent you've used paragraph and table styles.

stefan blom say:
The end-of-row markers show at the very end of each row, outside the table, at the right hand side (that is, in the page margin if the table extends across the full width of the page). End-of-cell markers, and possibly also end-of-row markers, may be difficult to spot depending on the chosen values for cell margins.
