dlookup value in table based on two criteria in different table - Microsoft Community

julie_0506 say:

dlookup value in table based on two criteria in different table

I want to build a query with a dlookup expression.  The query is using a table, "TblCurrAllocation" and this table contains a field "CurrPrice".  CurrPrice will change daily.  I have created another table (TblAllPricing) that has three fields.  "Product", "PassThrough" and "Price".  "Price" will be the CurrPrice in TblCurrAllocation.  I want to lookup the CurrPrice in TblAllPricing based on the "Product" AND "PassThrough" fields in TblCurrAllocation.  I'm successful if the dlookup is based on one criteria but not both.  BTW, "Product" is text and "PassThrough" and "Price" are standard numbers in both tables.  Thank you for taking a look at this for me.

 

Julie

ken sheridan say: There are a number of possible scenarios here:

1.  Is the Text10 control bound to a column in the form's underlying table?  If not, but an unbound control, then you should not be assigning a value to it in code, but calling the DLookup function as an expression in its ControlSource property:

= DLookup("[Price]", "TblAllPricing", "[PassThrough] = " & Forms![FrmQryLiquidityAsAllocated]![PassThroughRate] & " And [Product] = '" & Forms![FrmQryLiquidityAsAllocated]![LoanType] & "'")

This should be entered as a single line in the control's property sheet.  If the control is in the FrmQryLiquidityAsAllocated form then you don't need to reference the form, just the control, so can shorten it to:

= DLookup("[Price]", "TblAllPricing", "[PassThrough] = " & [PassThroughRate] & " And [Product] = '" & [LoanType] & "'")

2.  If Text10 is unbound, however, a more efficient method than that above would be not to call the DLookup function at all, but base the form on a query which joins the form's current underlying table to TblAllPricing:

ON YourCurrentTable.PassThroughRate = TblAllPricing.PassThrough
AND  YourCurrentTable.[LoanType]  = TblAllPricing.Product

and returns the Price column in addition to the columns from the current table.  You can then include a text box in the form bound to the Price column, setting its Enabled property to False (No) and its Locked property to True (Yes) to make it read only.

3.  If, on the other hand, the Text10 control is bound to a column in the form's underlying table (which I doubt from what you've told us), then where is the code being called?  In this scenario you'd probably have to call it in a number of places, again assuming the Text10 control is in the FrmQryLiquidityAsAllocated form:

(i) In the AfterUpdate event procedure of the [PassThroughRate] control.
(ii) In the AfterUpdate event procedure of the [LoanType] control.

But you'd need to allow for the PassThroughRate being Null when called in the LoanType control's event procedure:

Me.[Text10] = DLookup("[Price]", "TblAllPricing", "[PassThrough] = " & Nz([PassThroughRate],0) & " And [Product] = '" & [LoanType] & "'")

4.  Whether Text10 should be a bound or unbound control depends on what you want.  If you want all records in the form to dynamically show the current price from TblAllPricing then it should be an unbound control, and one or other of the methods described in 1 or 2 above should be adopted.  If you want the current price to be assigned to each record, and to remain fixed for that record regardless of changes in the Price in TblAllPricing, then it should be bound to a column in the underlying table and the method described in 3 above should be adopted.  This is analogous to a classic invoicing operation, where you want invoices to retain the price of a product from the time the invoice was raised, not reflect the current price.
