This question shouldn't have been moved to malware from the Windows forum where you originally posted it, since the answer has nothing to do with malware and relates instead to a Windows security function called User Account Control (UAC).

The prompt you mentioned is included a few pages down in the following [relatively complex] document and is called the "consent prompt".

How User Account Control works (Windows 10)  Microsoft Docs

The consent prompt is presented when a user attempts to perform a task that requires a user's administrative approval, in other words allowing the program named by the prompt administrative access to your computer's operating system and other programs or

The problem is that once you've allowed this access, there's no way to "take it back".  Since you've already allowed the program this access, it has already done whatever it was designed to do, so if this was malicious then it's already performed those malicious actions and can't be reversed by changing any settings related to the original prompt, since there actually aren't any.

However, if the item were actually malware, then any security program running on the PC should have notified you of this fact and offered you the chance to stop it from doing any actual damage.

Since you stated that you don't currently have an antivirus, then the built-in Windows Defender security application that Microsoft includes in Windows 10 should be operating.  If it's not, then either some other antivirus is installed and running or there's a more critical issue affecting the operation of Defender which needs to be resolved.

Mail Merge Rules in Office 365 - Microsoft Community

geert-jan schutte say:

Mail Merge Rules in Office 365

How can I see, edit, change Rules I inserted in a Office 365 Word document? The Rules function works fine, but in the document they are invisible. And, unlike former versions of MS-Word it seems impossible to use Fields in the Rules.

a. user say:

Seeing them is probably a matter of using Alt-F9/fn-alt-F9 (or option-F9/fn-option-F9)  (depending on your keyboard setup) to view the field codes rather than the field results, so you see e.g. { IF { MERGEFIELD x } = 5 "abc" "def" }.

As far as I know, there has never been a particularly good mechanism for editing some existing field types such as { IF } fields - going in manually and using Alt-F9 to toggle backwards and forward, using cmd-F9/fn-cmd-F9 to insert new fields, F9/fn-F9 to update them and so on is probably the best way once you get used to it. I'm not sure any of the other field types let you do that, although in principle they should do.

As far as the IF field is concerned, it is still possible to insert field codes in the "Insert this text" and "Otherwise insert this text" boxes using cmd-F9/fn-cmd-F9 to insert a pair of the special field code braces {  }, then typing your own field code (such as "MERGEFIELD myfield") between them. But there is no opportunity to insert fields from a drop-down or via drag-and-drop at that point.
