Okay, I found the answer to my problem within another post in the Microsoft Community.

On examining the contents of four of the C:\Users\User Profile\AppData\Roaming\Microsoft\Windows\SendTo folders on my laptop (including Guest but not Default or Public) I found that there was not only the correct "Bluetooth device" item (along with other correct items) but after that there was an extraneous 0 byte "Bluetooth" entry.

Apparently it is this extraneous 0 byte "Bluetooth" entry that causes the other items in this folder to not show on the Send To context menu (it "truncates" the menu).

Deleted this 0 byte "Bluetooth" item (not "Bluetooth device") from each of these folders and now the Send To context menu for each user profile is fine.

Tech background:

Belkin has some of the worst "User Guides" for its routers that I've seen in a while, so I can't determine if your model still has a "wireless mode" setting.  The three commonly-used types of home wifi equipment are wireless-b (oldest), wireless-g (old), and wireless-n (newest). 

Wireless-n equipment is backwards-compatible with both wireless-b and wireless-g.  This means that a wireless-n router can communicate with a wireless-n adapter in a computer, a wireless-g adapter, or a wireless-b adapter.  When wireless-n products were first released, there were reports of problems when wireless-n routers were used in a network with both wireless-n adapters and wireless-g (and/or wireless-b) adapters.  Thus, wireless-n routers had a configuration setting that permitted them to be used with "n-only" devices or in a "mixed" network of N, G, and B devices.  I suspect that this setting is still available in your router, even though it isn't discussed in the on-line documentation that I could find.
