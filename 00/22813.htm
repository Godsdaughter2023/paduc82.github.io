Going to guess that this involves an IMAP account where the contact folder has the added words "This Computer Only" in the folder description. If I'm correct, you're contacts are likely lost. With IMAP accounts, if Outlook cannot find the .ost file, a new one is automatically created and the email folders are once again sync'd with the server. The "This Computer Only" folders are empty and any previous data is lost. This is equally true if the IMAP account is deleted and then re-added to the profile. Contacts in an Exchange account are stored on the server so would always be re-sync'd should the OST file need to be recreated.

As for downloading a 3rd party OST2PST conversion utility, unless you have a copy of the OST file prior to the problem arising in terms of "lost contacts" - would be a pointless exercise.

The following is not going to solve your current issue but will mitigate this from happening again in the future in terms of lost contact/calendar info associated with an IMAP account

1) create a PST file in the profile (from the ribbon --> New Items --> More Items --> Outlook Data File)
2) from OL menu --> File --> Account Settings --> Data Files tab --> set the PST file as the default data file for the profile, the "default data file" is where the "default contact/calendar folders" are stored. PST files never get deleted by Outlook.

The Outlook Nav pane (the left pane) is comprised of files and folders. The "files" are the Outlook data files configured in the Outlook profile as listed in the Data FIles tab in Account Settings (go to File --> Account Settings --> Data Files tab).
