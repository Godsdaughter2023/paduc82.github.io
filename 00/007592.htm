Yes! I misunderstood. I thought you were individually entering the formulas in each of the cells G3:J13 and the method I provided lets you enter one formula in G3 and then copy to the remainder of the range.

Following guidelines for the highlighting issue

  • Select the entire range where you want the highlighting (C3:G13) (Or extend to C3:J13 if you want)
  • Select Conditional Formatting -> New Rule ->  Use a formula to determine which cells to format
  • Enter the following formula

      =C3=SMALL($C3:$F3,1)

  • Then click OK each screen until finished.

If you want then you can add additional rules to highlight the remaining cells in the selection with different colors depending on the position of 2, 3 or 4 as follows.

  • Select the entire range where you want the highlighting (Same as used for first condition range above)
  • Select Conditional Formatting -> Manage rules
  • Select "New rule" -> Use a formula to determine which cells to format
  • Enter the following formula to highlight the 2nd position.

       =C3=SMALL($C3:$F3,2)

  • Select Format button and select the required color from the Fill tab for the 2nd position and OK and OK
  • Click "Apply" button
  • Repeat previous to add rules for position 3 and 4.

rob koch say:

"Right now it appears that the malware is only quarantined, but there is no obvious way to flush it away. I just have to wait for it to expire from the cache? There is an "Allow" button, but nothing about the crucial distinction there. Would I be allowing Windows Defender to delete it, or would I be allowing the malware to run?"

First, to address these earlier questions, as already mentioned the quarantined item(s) are blocked and will be automatically removed by Defender, so there's no need to remove them manually, though you could typically still do this by flushing the browser's cache using its own controls.  Since all such actions as remove and allow are made relative to the threat (malware), the choice of allowing would result in allowing the malware to operate in the future.

As for your later question regarding the lack of information related to the website where the malware was actually encountered, this is entirely the fault of the browser.  The Microsoft security applications have included tight integration with the Internet Explorer and now Edge browsers for many years, including the ability for these to call an API that examines the URL information prior to download in order to detect potential malware as it's being downloaded to the cache.  This also provides Windows Defender with the additional information required to display this website URL within the information logged relating to such a detection.

In the case of 3rd-party browsers, though they've been provided access to the same API sets related to browser security for many years, they've typically chosen not to support these.  So the Microsoft security applications such as Defender are unable to gain access to these items until they've already been placed into the browser cache, the first point at which the built-in APIs related to the filing system are encountered.

Since those helping here are typically consumer volunteer helpers, regardless of their various official looking tags like the MVP and even those with Microsoft badges are typically provided minimal training related mostly to basic use of the Microsoft products, there's often little depth of knowledge.  Only a small number of those helping truly understand how these products work at a deeper level and so you need to expect that the answers provided will often seem limited as well.

This type of limited information is to be expected with a consumer product, since that subset of Microsoft's customer population wouldn't usually have the background to understand even the relatively basic explanation I just provided above.

So no one is trying to indicate you are ignorant, expect maybe in your understanding of the purpose behind these Community forums themselves, which had been designed from the beginning as a free support avenue for consumers.  I'm aware of this due to having been one of the first 2 people present when the original Answers community forum software was demonstrated at an MVP Summit side-session and the Microsoft Security Essentials product we helped with soon became the first application supported on this new platform.

FileSearch is not available in recent VBA versions. Peter's approach is simple for this application, but for a universal solution you could consider

Sub BatchProcess()
Dim strFilename As String
Dim strPath As String
Dim fDialog As FileDialog
Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)
Dim iCount As Integer
iCount = 0
With fDialog
    .Title = "Select folder and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
        "List Folder Contents"
        Exit Sub
    End If
    strPath = fDialog.SelectedItems.Item(1)
    If Right(strPath, 1) <> "\" _
    Then strPath = strPath + "\"
End With
If Left(strPath, 1) = Chr(34) Then
    strPath = Mid(strPath, 2, Len(strPath) - 2)
End If
strFilename = Dir$(strPath & "*.xls")
While Len(strFilename) <> 0
    iCount = iCount + 1
    'Do what you want with strFilename e.g.
    MsgBox strFilename
    strFilename = Dir$()
Wend
If iCount = 0 Then MsgBox "No files of this type found"
End Sub

n0madtanks say:

Outlook Web For Mobile

I have seen a few post where people had asked how do you get a more mobile friendly view when using the outlook website on a mobile phone. Many of the posts claimed to be resolved with answers that consisted of clearing cache or cookies which was not true in my case so I continued investigating and found there is a string you can add to the url to force a mobile friendly view. In fact there are 2! I thought I would share.

Phone Friendly view:

If you want to view a phone friendly version of outlook via the browser on your phone simply type in your web address but add "/?layout=tnarrow" (without the quotes).

It should look like this (fake url of course) www.[outlookaddressgoeshere].com/owa/?layout=tnarrow

Tablet Friendly view:

If you want to view a tablet friendly version of outlook via the browser on your phone simply type in your web address but add "/?layout=twide" (without the quotes).

It should look like this (fake url of course) www.[outlookaddressgoeshere].com/owa/?layout=twide

From there just favorite the URL and if an android device (maybe other devices do this too) you can even add it as an icon to you home screen from your mobile browser settings (which is awesome).

I hope someone else finds this helpful.
