joanne li msft say:

Hi Andy,

Thanks for the detailed explanation.

I understand that you want to use Save a Copy feature to copy a document to the location where it is stored such as OneDrive for Business or SharePoint Online without making members exist the collaborating. I'd like to explain that it is not feasible to do this.

We suggest using Copy to feature to create a copy of the document and save to the same location. If you'd like to use Save a Copy feature to achieve this, we invite you to post this idea in UserVoice to submit this feature request to help improve our products and services. UserVoice is actively monitored by related team and many features have been developed based on users' comments.

As a workaround, you can try using major and minor version to publish a version of the document for manager review, in the meantime, users can continue collaborating on the documents:

1.  Go to library settings>Versioning settings, set to Create major and minor (draft) versions for Document version history. Set Only users who can edit items for Draft Item Security.

2.  Grant your manager view permission to the document.

3.  When you think it's time to publish a version for manger review, in the library, right-click on the document>More>Publish to publish it. By doing this, your manager can only view the published version of the document and your colleagues can continue working on the draft.
