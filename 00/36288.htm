Step 1: Find program "Run" and run it
Step 2: Type "cmd" (no quotes)
Step 3: Type the bold commands (64Bit Windows) | OR | Type the italic commands (32Bit Windows)

Uninstall: %windir%\system32\msiexec.exe /unregister (and) %windir%\syswow64\msiexec.exe /unregister

And, if your MSInstaller is broken and you want in REINSTALLED etc:

Reinstall: %windir%\system32\msiexec.exe /regserver (and) %windir%\syswow64\msiexec.exe /regserver

If you have 32Bit Windows:

msiexec /unregister

(And, if you want to reinstall)msiexec /regserver

The attached DMP file is of the MANUALLY_INITIATED_CRASH1 (deaddead) bug check.

This indicates that the user deliberately initiated a crash dump from either the kernel debugger or the keyboard.

This is very interesting to see, as this as I said is a MANUALLY initiated bug check only. Did you set up key-crash registry keys and perform the crash yourself?

  1. With PS/2 keyboards, you must enable the keyboard-initiated crash in the registry. In the registry key HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\i8042prt\Parameters, create a value named CrashOnCtrlScroll, and set it equal to a REG_DWORD value of 0x01.

  2. With USB keyboards, you must enable the keyboard-initiated crash in the registry. In the registry key HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\kbdhid\Parameters, create a value named CrashOnCtrlScroll, and set it equal to a REG_DWORD value of 0x01.
