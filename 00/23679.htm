We can test on pagebreaks, and copy the range to a temporary sheet and then save it as PDF...

 

I assume that the macro should only look for manual pagebreaks row by row. This should get you started:

 

Sub SaveCellsAsPDF()
Dim LastRow As Long
Dim FirstRow As Long
Dim c As Long
Dim SavePath As String
Dim TempSh As Worksheet

Set TempSh = Worksheets.Add
SavePath = "C:\Temp\"
LastRow = Range("C" & Rows.Count).End(xlUp).Row
FirstRow = 16
For r = 16 To LastRow
    If Rows(r).PageBreak = xlPageBreakManual Then
        Range("C" & FirstRow, Range("G" & r-1)).Copy TempSh.Range("A1")
        FirstRow =r

        c = c + 1
        TempSh.ExportAsFixedFormat Type:=xlTypePDF, Filename:= _
        SavePath & "Form " & c & ".pdf", Quality:= _
        xlQualityStandard, IncludeDocProperties:=True, IgnorePrintAreas:=False, _
        OpenAfterPublish:=False
        TempSh.Cells.Clear
    End If
Next

Range("C" & FirstRow, Range("G" & LastRow)).Copy TempSh.Range("A1")
c = c + 1
TempSh.ExportAsFixedFormat Type:=xlTypePDF, Filename:= _
     SavePath & "Form " & c & ".pdf", Quality:= _
     xlQualityStandard, IncludeDocProperties:=True, IgnorePrintAreas:=False, _
     OpenAfterPublish:=False


Application.DisplayAlerts = False
TempSh.Delete
Application.DisplayAlerts = True
End Sub

Sub SaveCellsAsPDF()
Dim LastRow As Long
Dim FirstRow As Long
Dim c As Long
Dim SavePath As String
Dim RngToHide As Range
Dim RngToPrint As Range
Dim PageBreakRng As Range

Application.ScreenUpdating = False
SavePath = "C:\Temp\"
LastRow = Range("C" & Rows.Count).End(xlUp).Row
FirstRow = 16
Set RngToHide = Range("A16:A" & LastRow).EntireRow
RngToHide.Hidden = True
Set RngToPrint = Range("A16")
For r = 16 To LastRow + 1
  
    If Rows(r).PageBreak = xlPageBreakManual Or Rows(r).PageBreak = xlPageBreakAutomatic Then
        c = c + 1
        If PageBreakRng Is Nothing Then
            Set PageBreakRng = Range("A" & r)
        Else
             Set PageBreakRng = Union(PageBreakRng, Range("A" & r))
        End If
        RngToHide.Hidden = True
        Set RngToPrint = Range(RngToPrint, Range("A" & r - 1)).EntireRow
        RngToHide.Hidden = True
        RngToPrint.Hidden = False
        ActiveSheet.ExportAsFixedFormat Type:=xlTypePDF, Filename:= _
        SavePath & "Form " & c & ".pdf", Quality:= _
        xlQualityStandard, IncludeDocProperties:=True, IgnorePrintAreas:=True, _
        OpenAfterPublish:=False
        Range("A" & r).EntireRow.PageBreak = xlNone
        Set RngToPrint = Range("A" & r)
         RngToHide.Hidden = False
    End If
Next
RngToHide.Hidden = False
For Each cell In PageBreakRng
    cell.EntireRow.PageBreak = xlPageBreakManual
Next
Application.ScreenUpdating = True
End Sub

macropod say:

You can use a macro for this, but not the standard Find/Replace dialogue. For example, to find paragraphs that are not double-spaced:

Sub Demo()
Dim Prgph As Paragraph
For Each Prgph In ActiveDocument.Paragraphs
  With Prgph
    If .LineSpacingRule <> wdLineSpaceDouble Then
      MsgBox .Range.Text
    End If
  End With
Next
End Sub

 

For font validation, the process is rather different. For example, the following macro finds which fonts other than Arial are used in a document. The macro works by first making all text hidden, un-hiding the Arial text, then going through whatever's left to determine the font in use (un-hiding it too). The process is slow on lengthy documents with lots of fonts.

 

Sub TestDocFonts()
Dim StrFnt As String, StrNoFnt As String
Application.ScreenUpdating = False
With ActiveDocument.Range
  .Font.Hidden = True
  With .Find
    .ClearFormatting
    .Text = "*"
    .Font.Hidden = True
    With .Replacement
      .ClearFormatting
      .Text = "^&"
      .Font.Hidden = False
    End With
    .Format = True
    .Forward = True
    .Wrap = wdFindContinue
    .MatchWildcards = True
    .Font.NameAscii = "Arial"
    .Execute Replace:=wdReplaceAll
    .Font.NameAscii = ""
    .Execute
  End With
  Do While .Find.Found
    StrFnt = .Characters.First.Font.NameAscii
    StrNoFnt = StrNoFnt & vbTab & StrFnt
    With .Find
      .Font.NameAscii = StrFnt
      .Execute Replace:=wdReplaceAll
    End With
    .Collapse wdCollapseEnd
    .Find.Execute
  Loop
End With
Application.ScreenUpdating = False
MsgBox "The following non-standard fonts were found in the document:" _
  & vbCr & Right(StrNoFnt, Len(StrNoFnt) - 1)
End Sub

 

I agree with Mark123, though, that it's better to manage the process via Styles. Note that in addition to restricting users' Style choices, Word offers the option to automatically update the document Styles each time the document is opened. Provided the template is available, this will re-set any over-ridden Styles to their template specifications.
