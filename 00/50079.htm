3.  Being logged into a different account than where the credit resides.

4.  Selective blindness possibly not aided by the existing layout.

5.  Other website or account issues.  Skype Customer Service may need to contacted to rectify the problem.

Destiny 2 is a Xbox One X enhanced game, which means, that this particular game uses a higher resolution and more detailed textures than most "normal" games. This also means, that the One X has to "work harder" for the game to run smoothly with the added detail, which results in more heat. 

Usually, the console can handle that heat, if the area around the console is well ventilated. For example my One X doesn't have any issues with Destiny 2. Since you don't say which exact Raid you're having issues with, I can't say if I also experienced that part (I haven't done all chambers yet).

Both can have an adverse effect of cooling the console enough.

Using Driver Verifier is an iffy proposition. Most times it'll crash and it'll tell you what the driver is. But sometimes it'll crash and won't tell you the driver. Other times it'll crash before you can log in to Windows. If you can't get to Safe Mode, then you'll have to resort to offline editing of the registry to disable Driver Verifier.

So, I'd suggest that you first backup your stuff and then make sure you've got access to another computer so you can contact us if problems arise. Then make a System Restore point (so you can restore the system using the Vista/Win7 Startup Repair feature).

Then, here's the procedure:
- Go to Start and type in "verifier" (without the quotes) and press Enter
- Select "Create custom settings (for code developers)" and click "Next"
- Select "Select individual settings from a full list" and click "Next"
- Select everything EXCEPT FOR "Special Pool", "Force Pending I/O Requests" and "Low Resource Simulation" and click "Next"
- Select "Select driver names from a list" and click "Next"
Then select all drivers NOT provided by Microsoft and click "Next"
- Select "Finish" on the next page.

Reboot the system and wait for it to crash to the Blue Screen. Continue to use your system normally, and if you know what causes the crash, do that repeatedly. The objective here is to get the system to crash because Driver Verifier is stressing the drivers out.  If it doesn't crash for you, then let it run for at least 36 hours of continuous operation (an estimate on my part).

Reboot into Windows (after the crash) and turn off Driver Verifier by going back in and selecting "Delete existing settings" on the first page, then locate and zip up the memory dump file and upload it with your next post.

If you can't get into Windows because it crashes too soon, try it in Safe Mode.
If you can't get into Safe Mode, try using System Restore from your installation DVD to set the system back to the previous restore point that you created.

If that doesn't work, post back and we'll have to see about fixing the registry entry off-line:

You should do this:
  1. Create, test and document two admin accounts. One of them is your back-up account.
  2. Log on under one of the new accounts.
  3. Right-click the iTunes installation program then left-click "Run as Administrator".
I had done this previously, and I ran into the exact same problem regardless. I actually knocked heads with my dad, and we discovered that Windows throws a hissy fit when you drop the first account you make down to a standard user, then try to make them an Admin again.

Before anyone tries to use my method for fixing their problem, I broke it down into easier to understand steps near the bottom of my reply.

So we reset the defaults of Windows (Also creating a backup of the programs I had managed to install and other files I had transferred), named the initial account Root (That would be my Linux upbringing) and let that stand as the Admin account while I created a second account under my name, and put my computer back together there. I have yet to hit the same snag, and it was definitely NOT worth $60- $100 to have Acer or Windows tell me not to drop the first account from Admin to Standard user.

My suggestion to anyone coming across this problem?
1) If it's a preloaded OS like mine, find the instructions to reset Windows to factory settings and making a backup file, and do it.
2) When the reset is all said and done, create your Administrator/ Root account, and immediately create a standard user account
3) Log out of Admin and into your Standard user account
4) Install your programs. Just be ready to CONSTANTLY put in your password.

You could probably make a second Admin account instead of the Standard user, but I didn't want to push my luck, so I just went for the Standard user account.
