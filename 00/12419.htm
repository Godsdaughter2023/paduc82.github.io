The Source Object property is not the same as the Recordscource property. If you need a count, then you have to count the Records in the Recordsourc:

=DCOUNT("*", "query or tablename"," filter ")

ken sheridan say:

The Source Object property is not the same as the Recordscource property.

However, a base table or querydef object can be the SourceObject property of a subform control by preceding the table name with Table. or Query.  This places a simple datasheet in the control.  I think this is what the OP is doing here.  I did manage to get this to work in a form of my own temporarily set up like this by putting a text box in the parent form and setting its ControlSource property to:

=DCount("*",Mid([sfcEmployers].[SourceObject],7),"ContactID = " & [ContactID])

where sfcEmployers is the name of the subform control and ContactID is the setting for the LinkMasterFields and LinkChildFields properties of the subform control.

stefan blom say:

In addition to the other responses offered here, there is also a macro (originally created by Doug Robbins) which resets the font formatting of each numbering level.

Sub ResetAllLists()
'Reset font settings for all numbered list items
'Originally created by MVP Doug Robbins
'This version provided by MVP Stefan Blom, May 2018
Dim lt As ListTemplate
Dim i As ListLevel

For Each lt In ActiveDocument.ListTemplates
For Each i In lt.ListLevels
i.Font.Reset
Next i
Next lt
End Sub
