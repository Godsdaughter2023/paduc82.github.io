hairyfool say:

That is more a router issue than Windows and the solution depends on a number of components.


1st you have to allow inbound connection for the RDP protocol which is port 3389. Then you have tell the router to send any connection on this port to your Windows 8 Pro PC. These two tasks may be linked together in the router as one and obviously depends entirely on the model of router so you are going to need the manual.


This tells the router to pass the RDP connection to the PC so you may have to fix the IP address of the PC to one value so it does not change. However when you are outside the network you don't connect to the PC but to the external port of the router. You can test the connection by checking the ADSL port address from a PC on the network then connecting the RDP session to that address. Providing that works that's great but most home ISPs don't keep your external address the same so when away from home you need to know what the address currently is. The simplest solution is to ask your ISP for a fixed address but most charge and some will not provide it for a home account, only business users. Having a fixed address is more reliable so if the cost is OK go for it.


If that is not an option then you need to set up dynamic DNS. This configures you're router to tell a service what your current address is and the new one if it changes. The main one was DynDns.org but they have become less cooperative for their free service, I think there may be an alternative from D-Link. So having set it up instead of connecting to the address of the router you now connect to gmakb.dnsservice.name. As with standard web DNS this converts to your current external address and the connection is made.


Incidentally if you have a NAS box which provides a web server you can connect to outside the network then that may be using dynamic DNS. I have WHS so if I browse to myserver.homeserver.com I get the website but if I RDP to the same name the router directs the connection to the PC.

    ken blake say: Are you running 32-bit Windows or 64-bit Windows?

    josepena1 say: It is a 32bit Windows.

    josepena1 say: 32 bit windows no changes made before this occured, not aware when it actually happened.

    ken blake say:

    You can not use the entire 16GB, not unless you run 64-bit Windows.


    All 32-bit client versions of Windows (not just XP/Vista/7/8/10) have a
    4GB address space (64-bit versions can use much more). That's the
    theoretical upper limit beyond which you can not go.

    But you can't use the entire  address space. Even though you have a  4GB address space, most people can only use *around* 3.1GB of RAM. That's because some of that space is used by hardware and is not available to the operating system and applications. The amount you can
    use varies, depending on what hardware you have installed, but can range from as little as 2GB to as much as 3.5GB. It's usually around 3.1GB.

    You are getting somewhat less than 3.1 GB. Possibly because of what hardware you have, but perhaps for some other reason. I don't know.

    Note that the hardware is using the address *space*, not the actual RAM itself. If you have a greater amount of RAM, the rest of the RAM goes unused because there is no address space to map it to.

    cmdrkeene say:

    After an installation or the twice-yearly full updates (like 1709 or the upcoming 1803), the CPU will be heavily utilized and the system will be sluggish for a while as it finishes up background work.  I have a post that talks a bit about this at PSA: Windows 10 High CPU Usage.  But in general if you set your power to "do not sleep when plugged in" and let it run overnight, it'll usually finish up.

    If it doesn't, try looking in Task Manager and sort by the CPU column to find out which program is working so hard.  That'll let you figure out what it is, and hopefully either learn what it is doing or stop it.  Let me know what you see.
