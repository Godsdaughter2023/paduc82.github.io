brian tillman [outlook mvp 2007-2019] say:

Typically, PSTs, such as your Archive PST, are stored in hidden folders and by default Windows Explorer won't let you see the contents of hidden folders.  However, the dialog you used to turn archiving off should have shown you the path to the archive PST.  Copy that path (probably C:\Documents and Settings\username\Local Settings\Application Data\Microsoft\Outlook, where "username" is your Windows username) to Windows Explorer's Address field and press Enter.  You should see the PST then.  if you click File>Open>Outlook Data File, the default folder that will open should be that folder and you should see "Archive.pst" there.  Select it and click OK to open it in Outlook.  Select the Calendar folder and change to a table view like By Category or Events.  You should be able to see all the events in the archive's Calendar folder.  You can select them all by pressing Ctrl+A, then drag the selection to your defail Calendar folder to reestore them.  Once you've done that, you can right-click the root of the archive folder set and choose Close or you can click Tools>Account Settings, select the Data Files tabe, select the Archive PST, and choose Remove.

dale howard [mvp] say:

Jo --

Unless I am losing my mind, it appears that custom Duration fields are not available for inclusion in tables in the new Dashboard Reports feature.  However, the good news is that custom Number fields ARE available.  So, what you would need to do is to create a new custom Number field, name it something like Baseline Finish Diffs, and then use the following formula in the field:

(ProjDateDiff([Baseline9 Estimated Finish],[Baseline10 Estimated Finish],"Standard"))/[Minutes Per Day]

In the preceding formula, I recommend using the BaselineX Estimated Finish fields just in case you are using Manually Scheduled tasks in your project.  Also, I recommend using the ProjDateDiff formula, as it will return an answer in working days, not calendar days.  If your are not using the Standard calendar as your Project Calendar, please substitute the correct calendar in the formula.

At this point, you will be able to include your custom Number field in the new Dashboard Report.  Hope this helps.
