evaldas tamasauskas say:

Excel - breaking columns into smaller ones

Hello,

I have an excel file with two columns A and B (one with names another with numbers) both have more than 5000 rows.

In order to print it all more efficiently, I want to split it 50 rows per column. (So, A(1-50) B(1-50); D(51-100) E(51-100);  etc.)

Is there any way to do so?

I tried using VBA code i found online, but it worked with one column only and I couln't figure out how to change the code to my needs.

Although, maybe theres an easier way that you know?

Or if you could help me with the code, here it is:

Thanks in advance!

Sub split_data()
Dim lastRow As Long, copynumRow As Long
Dim cRow As Long, cCol As Long

Application.ScreenUpdating = False

copynumRow = 50
cCol = 2
cRow = 1 + copynumRow

With ActiveSheet
    lastRow = .Range("A" & .Rows.Count).End(xlUp).Row

    Do While cRow <= lastRow
        .Range("A" & cRow).Resize(copynumRow, 1).Cut _
            Destination:=.Cells(1, cCol).Resize(copynumRow, 1)

        cRow = cRow + copynumRow
        cCol = cCol + 1
    Loop
End With

Application.ScreenUpdating = True

End Sub

This should do what you ask for.

Sub Move_2_to_8()

'blank column between pairs

Dim iSource As Long
Dim iTarget As Long
    iSource = 1
    iTarget = 1

    Do

mike middleton say: Both data tables have F36 as (what I call) the output formula.

The first data table is =TABLE(F22,F8). Cell F36 does depend directly on column input cell F8 and indirectly on row input cell F22, so the table appears to be working.

The second data table is =TABLE(F23,F8). Cell F36 does not depend on row input cell F23, so the output values in the body of the table do not depend on the Trade Spending % values in the row at the top of the table.

As you noted in an earlier post in this thread, I usually apply the Data Table feature to a model that has one or more input cells and one output cell, and I recommend that the input cells of the model contain values, not formulas. If a model cell contains a formula, strictly speaking it's either an output cell or an intermediate cell. If the model input cells contain values, it's easy to do manual what-ifs to verify that the output is changing appropriately when you change input values (and avoid overwriting formulas or references) before creating the Data Table.

How can get powerpoint　animation path data via c# or vba - Microsoft Community

sampson amir say:

How can get powerpoint　animation path data via c# or vba

How can i get the animation path from powerpoint via C# or vba?
With project need, we must get each ppt shape's animation path data. Now, we can get the animation path by manual through convert ppt into zip, then extract the packed file to the slide.xml file. The xml files contain all animations information.  


Is there a better way to get those path data  via C# or vba? 
john sr wilson say:
Maybe


Sub getPaths()
Dim oeff As Effect
Dim osld As Slide
Dim strReport As String
On Error Resume Next
Set osld = ActiveWindow.View.Slide
If Not osld Is Nothing Then
For Each oeff In osld.TimeLine.MainSequence
If oeff.Behaviors(1).Type = msoAnimTypeMotion Then
strReport = strReport & "Path for Effect " & oeff.Index & " Shape= " & oeff.Shape.Name _
& ":-" & oeff.Behaviors(1).MotionEffect.Path & vbCrLf & vbCrLf
End If
Next
End If
If strReport <> "" Then MsgBox strReport
End Sub
