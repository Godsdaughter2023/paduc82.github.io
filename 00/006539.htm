File history security issue? - Microsoft Community

ray.g say:

File history security issue?

If you use file history it exposes ALL the files protected by different users to everyone on the system because it does not set up per user permissions on the directories in file history location to ensure any user can ONLY see files that belong to that user

Steps

1). Create a volume to hold the history (or of course use an existing one if you have to)

2). Then via control panel turn on file history for more than one user

3). Let the initial run(s) complete

4). Browse the contents of the file history location - look at files from other users apart from the one you are logged in as.

IMHO all other histories should be 100% invisible to the current user.
