suzanne s. barnhill say:

You could insert an empty Heading 1 at the beginning of the document. Of course, this would then cause problems in the TOC.

If the number of pages before the first Heading 1 is known, you could use an IF field:

{ IF { PAGE } > x { STYLEREF "Heading 1" } }

where x is the number of pages.

In a finished document, an IF field could include a cross-reference to the page number of the first heading:

{ IF { PAGE } < { PAGEREF _Ref440433583 } "" "{ STYLEREF "Heading 1" }" }

where { PAGEREF _Ref440433583 } is an example of a Word-inserted cross-reference.

Either of these solutions, however, would require more user expertise than inserting a section break and unlinking the headers.

joeu2004 say:

VBA: How to manually calculate multicell array formula?

EDIT....  I should note that the following question is regarding VBA used in Excel 2003.  I had purposely posted the question in an Excel-specific forum, so that fact was obvious.  A moderator moved my question to a more general forum.  And in the process of doing so, the context information that I had entered seems to have been lost.  I am unhappy with the moderator's decision.

 

I selected A1:E1 and entered an array formula of the form =foobar(), where "foobar" is a UDF that returns a 5-cell row of numbers.  Then I copied A1:E1 down through A30:E30.

For debug purposes, I instrumented foobar() with the statement:

MsgBox Application.Caller.Address

All that works fine.  When I press ctrl+alt+F9, foobar() is called one time for each 5-cell range, as intended.

The only problem is:  for debug purposes, I want to evaluate each row in order, but Excel has its own ideas about the order ;-).

So I have the following macro to control the order of evaluation:

Application.Calculation = xlCalculationManual
For i = 1 To 30
   Cells(i, 1).Resize(1, 5).Calculate
Next
Application.Calculation = xlCalculationAutomatic

That does call foobar() for each 5-cell range in the order of rows 1 through 30, as intended.

But after that, when .Calculation is set to xlCalculationAutomatic, I see calls to foobar() for each individual cell -- a situation that foobar() does not tolerate.

First, of course, what is the right way to manually calculate a multicell array formula without incurring these side-effects, namely the single-cell calls?

As I noted, Excel does not seem to do that normally, at least not when ctrl+alt+F9 forces recalculation.

Second, if they are unavoidable, how should I make foobar() tolerant of the situation without screwing up the worksheet?

I added the following at the beginning of the function:

If InStr(Application.Caller.Address, ":") = 0 Then
   foobar = Range(Application.Caller.Address)
   Exit Function
End If

Without the assignment to foobar, simply exiting the function would cause zero to be returned, overwriting the legitimate value returned when foobar was called with the range of the array formula.

The assignment statement is suitable for debug purposes.  But it causes a circular reference error.  Surprisingly, that is only the first time.  Subsequent calls either fail silently or do the assignment silently; I'm not bothering to determine which.  In either case, it has the "desired" effect of not overwriting the value in the caller cell.  But it's a kludge.

suzanne s. barnhill say:
Although you can try doing this with a left Page Border, that will not stretch from the top of the page to the bottom. The pleading templates previously supplied with Word instead used a drawing line anchored to the page header. While this can be drawn the length of the page, most printers will not be able to print it clear to the edge because they have irreducible unprintable areas required for paper handling. The advantage of a drawing line over a Page Border, however, is that you can place it exactly where you want it, irrespective of where the page margins may be.
