jasles say:
That would be done with a VLOOKUP function.
On that separate list, have the common name in column A, the scientific name in column B, and then in the first sheet, use this formula (assuming column D on the first sheet has the common name) in cell E2 and drag down the column:

=VLOOKUP(D2,Sheet2!A:B,2,FALSE)

john mcghie say:
Hi Rolando:

OK, I think the error you are getting contains the phrase "... may be in use."?

If so: it's one of those "Deadly combination" issues.  MacKeeper, OS X, Time Machine, and Word are all involved.

You have not said which OS you are using, but if it's OS 10.7, there's a new feature that enables it to "lock old files after x number of days of inactivity".

If that's turned on, protection software such as MacKeeper will play up, because they need to open the files to check them out, and they are prevented from doing so.

This mechanism also prevents Word from opening files.

So the first thing to do is check that if you are on OS 10.7, this feature is switched off.  http://osxdaily.com/2012/01/24/turn-off-file-locking-in-os-x-lion/

Time machine briefly locks files while checking to see if they need to be backed up.

Word needs to lock each file before it can open it.

Your first move is always to ensure that both OS X and Microsoft Office are FULLY up-to-date.

Then make sure File Locking is turned off.

Then re-start the computer (not just close the lid, all the way down until the lights go off).

Then come back and tell us what is happening now.  We're going to need more detail such as "which files", what else works/doesn't work, etc.

Cheers
