gary's student say: Select the cells you want to convert and run this small macro:

Sub NameChange()
Dim wf As WorksheetFunction
Set wf = Application.WorksheetFunction
For Each r In Selection
    v = wf.Proper(r.Value)
    If Left(v, 2) = "Mc" Then
        v = "Mc" & wf.Proper(Right(v, Len(v) - 2))
    End If
    If Left(v, 2) = "O'" Then
        v = "O'" & wf.Proper(Right(v, Len(v) - 2))
    End If
    r.Value = v
Next
End Sub

How can I flag an excel spreadsheet - Microsoft Community

tssst say:

How can I flag an excel spreadsheet

How can I flag an excel spreadsheet in a given timeframe with a hyperlink to that data? So if I say I want to review this workbook or spreadsheet in a month or a year it will notify me and provide a hyperlink to the data.
bill manville say:

I guess you will need to create a list somewhere of the filenames of the items you want to review and the dates on which you want to review them.

This list could be in a worksheet.

You could sort the list into review date order.

You could use conditional formatting to highlight rows which are due or overdue for review.

When you enter a filename into the worksheet you can manually turn it into a hyperlink using Insert > Hyperlink

Or you could have a Worksheet_Change event procedure that turns any text entered into column B of the worksheet into a hyperlink:

Private Sub Worksheet_Change(ByVal Target As Excel.Range)
  Dim C As Range
  If Intersect(Me.Columns(2), Target) Is Nothing Then Exit Sub
  For Each C In Intersect(Me.Columns(2), Target)
    If C.Value = "" Then
      C.Hyperlinks.Delete
    ElseIf C.Hyperlinks.Count = 1 Then
      C.Hyperlinks(1).Address = C.Value
    Else
      C.Hyperlinks.Add C, Address:=C.Value
    End If
  Next
End Sub

How to set the default table style when opening an HTML file?

I am using Autocad to create an HTML file of a report.  The HTML file is setup in multiple tables.  When I open the file in Word 2010, the tables are not consistent in column width and, as a result, some entries "wrap" within the cell and create a row that is 2 lines high.  If I change the "preferred" table width  to something larger, the problem is solved.  However, in the report there may be 20 or 30 tables to deal with.  How can I set the default table settings in Word so that when I open an HTML file, the tables created will be consistent??

pamcaswell say: This may help:
The default cell margins in Word (0.08", which is almost 6 points) is probably larger than the cell padding in your HTML tables.  You can modify the Table Grid table style in normal.dotm.  With your cursor in a table, go to Table Tools > Design tab.  Hover the cursor over the plain tables until you find Table Grid.   Right click then left click Modify table style.  Make sure Whole table is selected for Apply formatting to.  Click Format button > Table Properties > Options button. Change the left and right margins to something smaller, say, 0.03".  If you know what the padding is in the  HTML, you can use that value (but translated in inch units).

The tables of new documents created from the revised normal template will have smaller cell margins.  You can import via the Organizer the modified Table Grid style into existing documents.


stefan blom say:

You'll probably find that Pam's observation regarding cell margins will be helpful to you.

However, if you need to adjust column widths as well, note that you can't do that in a table style. You will have to create a macro and run it after the tables have been imported.
