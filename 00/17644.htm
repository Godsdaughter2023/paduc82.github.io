john mcghie say:
Hi Ellard:

This is a bug in Word for Mac that has been with us for some time.

Depending on which version of Word opens the file, it may have to down-grade the image to display it.  If it does, you will notice these unwelcome changes.

Word stores three versions of an image in a document:
1)  The original
2)  The Printable version
3)  The display version.

Word's limits are RGB colour only, in 24-bit colour depth.  Anything outside that range must be re-processed.  This is where you will get colour shifts, sometimes quite dramatic.  If the original is an AI format, depending on what version of AI, Word may not be able to read it at all, although it can print it.  For display, it will fall back to the AI placeable header image, which is 72 dpi black-and-white.

When it re-processes the file, Word stores the original, then converts it to a "printable" image in Word's native format (either PNG or EMF).  It stores that in the document as well. Conversion from some graphics formats may exceed the abilities of EMF (especially, the colour depth, an sometimes, the line weights and vector sizes).  This can result in unwanted changes also.

If the original image is very high resolution, Word often also produces a "placable image" at very low resolution (72 dpi) which is quicker to handle when paginating the document on screen.  If Word falls back to displaying this version, that's where the very blurry images come from.  This down-grade usually happens if the document is in the obsolete format.

There is not much you can do about this, but there are some things that will lessen the impact.

The first is to upgrade your software: Word is currently at update level 14.2.4, and there have been many fixes since 14.1.0, some of which improved graphics handling.  And there is another major fix on the way soon.  Go to the Help menu and choose "Check for Updates" to get your copy of Office up to date.  Best to leave it set to Automatic Update.

Make sure your OS X is fully up-to-date too, of course, as there were some fixes on the Apple side as well.

Second:  Try to prepare your images in RGB format, with a colour depth of 24 bits or lower.  If you are using AI or PDF, flatten the thing to a single layer.

These two suggestions will give you a "better" result, but you are up against designed limits in Microsoft Office, so they won't solve all the problems.

The PC version of Word is a couple of levels ahead of the Mac version, and in some circumstances can handle 48-bit colour.

Sometimes, people have better success by first inserting the image into PowerPoint, then copying from PowerPoint and pasting into Word.  Again, don't expect miracles, but it may get you closer.

Hope this helps

neha singh msft say:

Hi Tara,

Please make sure you are using the new OneDrive sync client to sync your OneDrive documents. If you are sharing a folder and you want other users with whom you’ve shared the folder to sync on their local computer, then they must go to Shared with me folder in their OneDrive on browser.

Once the user enters the Shared with me folder, open the shared folder and click Sync on the ribbon.

 

If you are syncing the shared folder using the new sync client, any sub-folder/file residing in the shared folder will sync automatically.

In file Explorer, the shared folder will appear under SharePoint with original user’s name. For example, user A shared a folder with user B, user B syncs the shared folder on to this local drive. On user B’s computer, in file explorer it will appear under SharePoint with the name “User A-Sharedfoldername”.

In addition, if you wish to work on documents with your team and the documents are the collaborative effort related to a project, then saving to a SharePoint team site might be a better choice.
