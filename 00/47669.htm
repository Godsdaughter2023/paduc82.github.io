Windows Update version 1803 KB4103721 causing Windows not booting - Microsoft Community

massimilianobernini say:

Windows Update version 1803 KB4103721 causing Windows not booting

I discovered that if you installed a program from Iobit (like Driver Booster, for example) it creates a service called "LiveUpdate" with full name "LiveUpdateSvc".

Even if you remove the program, the service will remain and this cause the KB4103721 update to fail to boot Windows after the update.
Uninstall all Iobit programs and remove the service from the administrator command prompt typing "sc delete LiveUpdateSvc" and then you will be updating Windows without any problem.

If you already experienced the issue the only way to boot again into Windows is to load a sistem restore point. You may try also booting into safe mode but it will be probably not working. If you can boot a system restore point then remove Iobit software and the service "LiveUpdateSvc", then you will be updating Windows without any problem.

What's the difference between paid and free HEVC Media Extensions - Microsoft Community

sanghyeon yeo(enusan) say:

What's the difference between paid and free HEVC Media Extensions app?

When I try to open HEVC video file, Movies app linked me to Store to install codec, which is $0.99 paid app (https://www.microsoft.com/en-us/store/p/hevc-video-extensions/9nmzlz57r3t7) but there is a free one which is named  'HEVC Video Extensions from Device Manufacturer' (https://www.microsoft.com/en-us/store/p/hevc-video-extension/9n4wgh0z6vhq). Is there any difference between those two apps?

greenreaper say:

The free one will only work if your device already provides a way to process HEVC-encoded content. High-end video-processing devices released in 2015 and most (but not necessarily all) such devices sold in 2016 should support this.

Per the description:

Compatible devices support HEVC in hardware, including Intel 7th Generation Core processors and other modern GPUs.

In practice it means that the device manufacturer has paid the licensing fees for you, and probably advertised support for the codec, either on purchase or as an update. Usually the "device" in question is the GPU, but might be the case even if the actual implementation is partly or completely software-based (e.g. Xbox One).

For Intel this means GPUs included with its consumer "Kaby Lake", "Coffee Lake" and future "Cannon Lake" and "Ice Lake" CPUs. "Skylake" is probably included but will not handle 10-bit/wide colour videos (I think there is even a review complaining about this). "Broadwell" (for which Intel has offered partial hardware support via drivers) and "Haswell" (entirely done in software) might be included as well, but it's unclear whether this is the case.

NVIDIA's 9xx and 10xx GPUs and AMD's R9/Fury/Vega GPUs and Carizzo/Bristol Ridge/Raven Ridge APUs should also support it. Of course you will need updated drivers for all of these.

As it says, you should be able to install it even if it won't work:

If you are not using a compatible device, this extension will have no effect on your Windows 10 video experience.
