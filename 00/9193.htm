Apparently some SMTP servers require the HELO to be sent along with the FQDN of the source server as a way to stop spamming. That means the client (your outlook machine) should be named machine1.domainname.com and not just machine1.

You may follow the steps below to resolve the issue:

 

     

  1. Get the properties for My Computer, next click the tab; Network Identification.

     

  2. Click the properties button

     

  3. Under Computer name specify the HOST / Computer (example; mail)

     

  4. Next click the More button

     

  5. Where it says, "Primary DNS suffix of this computer; enter your primary domain name (actually can be any domain name :) ).

     

  6. Click OK and OK again

The normal sequence would look like:

  • UserForm1 is invoked somewhere in your code using a line like "UserForm1.Show vbModal"
  • Code within UserForm1 invokes UserForm2 in a similar manner.
  • When the user clicks on the "OK" button for UserForm2, if the UserName and Password are correct, UserForm2 closes itself.  This returns the program to the code inside UserForm1.

If this is the way you are doing it, after you perform the "Unload Me" line (I am assuming that line is inside UserForm2), you should add a line to UserForm1, right after the "UserForm2.Show" line, that Unloads UserForm1.  You cannot close UserForm1 from UserForm2 if both forms are being shown modally (which means you cannot switch from that form to any other window until that form is closed).

shyamal debnath msft support say:

Hi Ryan,

It’s not feasible to assign read-only permission to the delegate via using Admin portal.

When you assign “Read and manage email to this mailbox”, the Full Access permission allows a delegate to open a user’s mailbox and access/delete the contents of the mailbox behave as the mailbox owner. However, assigning the Full Access permission doesn't allow the delegate to send mail from the mailbox. You have to assign the delegate the Send As or the Send on Behalf permission to send mail.
