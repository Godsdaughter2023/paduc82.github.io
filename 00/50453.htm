a. user say:

All we know is that you have some kind of computer that is probably giving you a message like this:

Windows XP could not start because the following file is missing or corrupt: \WINDOWS\SYSTEM32\CONFIG\SYSTEM

Usually (and I mean 99.999% of the time), there is nothing really wrong with the file - the message just means that XP can't find the file...  probably due to some corruption in the NT File System (NTFS). 

Things like that can be caused by a power interruption, aborted restart, or improper shutdown (this includes plug pulling, power buttons, removing the battery, etc.).  If you ever want to recreate the problem so you can actually see it (listen all your Microsoft engaged Support Engineer "experts"), while you system is running, pull the power plug on it and restart.  If you do that about 5 times (or less), you will see the message :-), then you can learn how to fix it.

A chkdsk with error correction (chkdsk  /r) will usually fix things right up sometimes you have to run it more than once, but depending on your hardware (about which we know nothing), you may have to do some other things to get that to work so you might have to run the chkdsk with error correction from something like a Hiren's Boot CD that you can also use to hopefully copy your personal data files off to some external device.

Some antivirus/antimalware programs (read Norton) can also cause corruption in the file system when they load.  Your system will boot and run, but the the AV has already corrupted the file system when it loaded so the next time you try to boot, you won't - until you run the chkdsk  /r  and that will fix it one more time...  until you boot again.

Thanks to the MS Answers forums implementation, we don't know anything about what you might be running in that area either.

If it keeps happening for none of the above reasons, it could just be that your HDD is failing, and needs to be replaced - you could have a bad/loose cable.  You could run some diagnostics on it, but we still don't know anything about your hardware to make any suggestions.  Could be less frustrating to just replace it.

john john __ say: If this keeps on happening it would suggest faulty hardware or that something is writing to the registry while the system is shutting down.  On the hardware side you would often have other indications that problems exist, files other than the registry hives might also experience corruption.  If chkdsk is reporting bad sectors it means that the drive's spare sector pool is depleted... the drive is kaput, back it up and replace it ASAP!  If it is a hardware problem and if no other files seem affected my first inclination would be to suspect bad RAM as the culprit.
