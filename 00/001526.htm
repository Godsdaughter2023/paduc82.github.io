vlookup won't lookup a value on a different sheet - Microsoft Community

johninsing say:

vlookup won't lookup a value on a different sheet

I input =VLOOKUP(A440,Main_Data,13,FALSE) where Main_Data is on a different sheet, and I get #N/A

If in type the exact same formula on the same sheet as Main_Data then I get the correct answer

How do I make vlookup function across sheets ?

pete_uk say:

To add to Mike's comment, if Main_Data was not being recognised you would get a #NAME error. VLOOKUP is sensitive to different formats, so if A440 contains a number and there looks to be an exact match within Main_Data, then it might be that the "numbers" in Main_Data are actually text values that just look like a number (or vice-versa). You can try these to overcome this situation:

=VLOOKUP(A440&"",Main_Data,13,FALSE)

if you have a numeric value in A440 with text values in the lookup table, or:

=VLOOKUP(A440*1,Main_Data,13,FALSE)

if you have a text value in A440 which looks like a number and proper numbers in your lookup table.

You can also use:

=IF(ISNA(vlookup_formula),"",vlookup_formula)

or:

=IFERROR(vlookup_formula,"")

to trap the #NA error.

Hope this helps.

Pete

Add bulk users in distribution group

I have closed to around 100 external contacts whom I want to make a part of my distribution group.

ynnhoj gnahz say:

Hi Ashishsrkhan,

To add bulk users to a distribution group, we need to use PowerShell. Here are the detailed steps:

1.Prepare a CSV file to contain each user’s information, including Display name, Alias, E-mail address, and then store it in local disk (for example, D:\list.csv ).

For example:
Display name, Alias, E-mail address
Member1, member1, *** Email address is removed for privacy ***
Member2, member2, *** Email address is removed for privacy ***

2. Connect to Exchange Online PowerShell.

3. Import the CSV file to Exchange Online via PowerShell.
Run the cmdlet:
Import-Csv “D:\list.csv” | foreach{Add-DistributionGroupMember -Identity “GroupName” -Member $_.alias}

Please replace “GroupName” with your actual distribution group email address.

hansv mvp say:

In recent versions of Access, you can't use FileDateTime directly in an expression, it is considered to be "unsafe". You can use it in VBA code though.

You could create a custom function to act as "wrapper":

Function GetFileDateTime(FileName As String) As Variant
    On Error GoTo ErrHandler
    GetFileDateTime = FileDateTime(FileName)
    Exit Function
ErrHandler:
    ' Something went wrong, return Null
    GetFileDateTime = Null
End Function

and use it like this in an expression:

=GetFileDateTime("C:\Access\MyDatabase.mdb")

Alternatively, you can fill a text box using code, for example:

Me.txtFileDate = FileDateTime("C:\Access\MyDatabase.mdb")
