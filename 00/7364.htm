djsteele, mvp say:

The error indicates that you haven't set a reference to Outlook (under Tools | References in the VB Editor). However, that's not actually recommended, because your application won't work if the user doesn't have the exact same version of Outlook installed.

Your subject line mentions you're using CreateObject("Outlook.Application"). That implies you could be using Late Binding, so simply declare Dim objOutlook As Object. This means it will work for anyone who has Outlook installed, regardless of what version. However, it does mean you'll need to explicitly declare any of the intrinsic Outlook constants you're using (they start with ol, such as olAttachment). To do this, set the reference, go into the Object Browser (F2 while in the VB editor), determine the values for all constants you're using, declare them yourself, then remove the reference.

hughburns say:

So we ended up getting support from MS. After they connected to server and diagnosed problem, here's the fix they came up with...

Problem

=======

IE opens a blank page when clicking a link from an email or typing the address on a run command

 

Root Cause

=========

Process Monitor log shows: HKCR\IE.HTTP\shell\open\command\(Default) value was "C:\Program Files\Internet Explorer\iexplore.exe" –nohome in problematical machine:

9:54:38.2084168 AM         Explorer.EXE         17704    RegQueryValue    HKCR\IE.HTTP\shell\open\command\(Default)                SUCCESS Type: REG_EXPAND_SZ, Length: 116, Data: "C:\Program Files\Internet Explorer\iexplore.exe" –nohome

9:54:38.2131120 AM         Explorer.EXE         17704    Process Create     C:\Program Files\Internet Explorer\iexplore.exe                SUCCESS PID: 18276, Command line: "C:\Program Files\Internet Explorer\iexplore.exe" –nohome

 

The right value should be "C:\Program Files\Internet Explorer\iexplore.exe" %1, then URL parameter can pass in when IE open.

 

Solution

=======

Changed HKEY_CLASSES_ROOT\ IE.HTTP\shell\open\command\(Default) value from "C:\Program Files\Internet Explorer\iexplore.exe" –nohome to "C:\Program Files\Internet Explorer\iexplore.exe" %1
