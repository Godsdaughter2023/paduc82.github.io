a. user say:

Found cause and a work around.

When installing 1709 pro, after installation the windows edition changes from pro to pro education.

When installing 1803 pro, the edition remains pro. It doesn't change to pro education. To solve this, I grep the license key from the uefi enviroment and in the activation window i enter this pro edu license key. After activation the edition is changed to Pro edu.

When sysprepping the system and making an image of the system, the product key is replaced by the general pro edu key.

When deploying this image to another system it activates its digital licence key.

davesofranko1 say:

We have a common image that has both Pro and Pro Education systems. I found a workaround to automate this here: https://docs.microsoft.com/en-us/windows/deployment/windows-10-enterprise-subscription-activation

copy the below into a .cmd file and run as administrator:

@echo off
FOR /F "skip=1" %%A IN ('wmic path SoftwareLicensingService get OA3xOriginalProductKey') DO  ( 
SET "ProductKey=%%A"
goto InstallKey
)
:InstallKey
IF [%ProductKey%]==[] (
echo No key present
) ELSE (
echo Installing %ProductKey%
changepk.exe /ProductKey %ProductKey%
)

the slmgr.vbs command we used before doesn't seem to work for activating product keys in 1803, but this works.

Go to Start / Default programs / Associate a file or protocol with a program and scroll to .pdf.  It should indicate Adobe Reader as the default program but it probably doesn't.  Click on change and then see if Adobe Reader is one of the listed options.  If so, choose it and save your work.  You will now be able to open PDF files.  If Adobe Reader is not one of the options, click on browse and locate the Adobe Reader executable file (it will probably be somewhere in a sub-directory of the Program Files sub-directory) and then select that as the default program.  Again, once done you should once again be able to open PDF files.  Chances are good that the Microsoft Word Viewer set itself up as the default for PDF files and now there's noplace for them to go (they have no valid association).  The reason you have no options to open with is that there are probably no programs setup as associations for PDF files.
