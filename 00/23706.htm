If Formula returning as false instead of a number - Microsoft Community

dianarapoza say:

If Formula returning as false instead of a number

How do I turn this formula into a number instead of it reading false:

=IF(COUNTIF('Refernce Table'!$A:$A,$G3)=1,VLOOKUP($G3,'Refernce Table'!$A$2:$C$1311,3,FALSE))

I have tried adding 0, 1, even 2 where is says false.  It works on some cells but not others and I do not understand why.

Please help.

jeeped say:

While the IFERROR() wrapper suggested by JLLatham is the better method, I thought I would offer some insight into why the results from your original formula seemed sporadic.

Excel treats a boolean FALSE as 0 (zero). By strict definition, anything that is not FALSE is TRUE. By extension, if the COUNTIF(...) returned 0 (e.g. not found) then it is considered FALSE all by itself. If it returned anything other than 0 then it will be considered TRUE. You didn't need to compare it to a number. Your formula would have worked if it was,

=IF(COUNTIF('Refernce Table'!$A:$A,$G3),VLOOKUP($G3,'Refernce Table'!$A$2:$C$1311,3,FALSE))

  ... or,

=IF(COUNTIF('Refernce Table'!$A:$A,$G3),VLOOKUP($G3,'Refernce Table'!$A$2:$C$1311,3,FALSE), "")

  ... or (and this one is unnecessarily redundant),

=IF(COUNTIF('Refernce Table'!$A:$A,$G3)>0,VLOOKUP($G3,'Refernce Table'!$A$2:$C$1311,3,FALSE), "")

It may be important to note that conversely when TRUE is converted to a number¹ it always converts to 1 (one)., not some random number that is not 0 (zero).

I also want to clarify that the FALSE in your original formula had nothing to do with FALSE being returned by the IF(). The FALSE in VLOOKUP(..., FALSE) dictates that an exact match must be found. You were receiving FALSE as a result because nothing was specified for what should be returned by the IF() if the criteria was not true. You probably knew that but I didn't want any confusion.

(Don't know if that was a typo or intentional but Refernce in 'Refernce Table' is spelled wrong.)

¹ In VBA, True equates to -1, not 1.

jllatham say:

What are you trying to do exactly?  I am thinking that your logic with the

COUNTIF('Refernce Table'!$A:$A,$G3)=1

portion may be incorrect or not exactly what you want.

Your formula is kind of saying:

If G3 appears just one time in 'Refernce Table' column A, then do the VLOOKUP()

But if G3 does not appear just once (could be 2, 10, 1000 or 0) then return FALSE.

If you just want to return the results of the VLOOKUP() when there is a match for G3, then try this, which requires Excel 2007 or later to use.

=IFERROR(VLOOKUP($G3,'Refernce Table'!$A$2:$C$1311,3,False),"")

That will either return the VLOOKUP() result or an empty looking cell, you could change the ,"") part to a number if you need to, as ,1)  or ,2)  for when G3 is not in column A of the lookup table sheet.

steve rindsberg say:

First:  THANK YOU for the very complete/detailed explanation of the situation.  

You mention this:

"cannot link to a url which it see the unc path as?? "

PPT can link to files on UNC paths but you might be running into other issues here.  Some versions of PPT used to let you create links that included bookmarks (ie, a # character, followed by the location of a specific page within the linked document).  But they'd drop the # and everything after it once you saved and reopened the file.  Is it possible that it's dropped some characters or whole chunks of them from the link path?  See below for a way of testing this.

Or, and my eyes aren't good enough to answer this from the screenshot, but the length of the path might be an issue.  For some things, PPT doesn't deal well with full file paths > 128 characters or so.  The OS itself may limit you to 255 characters.  If you would, count the number of characters in the link path and if it's much over 100, try the same operation on a file in a location with a shorter path as a test.

Another oddity is that the total amount of space available to PPT for storing links is somewhat limited.  Normally it's not something you bump into but with very long paths like yours, you could be pushing the edge, especially if there are lots of similar links in the presentation.  If it's just a few like this, we can dismiss this possibility though.

Something that might shed some light:  I have a PPT add-in called FixLinks.  It's a commercial add-in but when you download and install it, you get the free demo version, which includes a link reporting tool that'll give you the link path to all of the links in your presentation.  It might be useful for you to install it, then

- Create a simple presentation with one of the links you're having trouble with, but don't save it yet

- Run a FixLinks report; save the resulting notepad txt file

- Save the presentation

- Quit PowerPoint, restart PPT, open your file again, verify that the link now fails

- Run another FixLinks report; save and compare it to the first one.

- Let us know what you discover

More info on FixLinks:  http://www.pptools.com/fixlinks/

Car manufacturers are notoriously bad at updating their car kits, so they are often behind the times. Most phones have a production life of 18-24 months, and when you look at compatibility lists for some factory-fitted car kits, the phones that work with them are anient history. Ford actually fits in some cars a kit that was specially developed for them a number of years ago by Nokia, but they didn't maintain a relationship with Nokia and so the kits are out of date when the car leaves the production line.
