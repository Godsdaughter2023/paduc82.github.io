steve rindsberg say:

You're not getting extra spaces, though ... the width of the one space is changing.  That's one of the ways justification works.

Again, I can understand how you *want* it to work, but non-breaking space means just that ... a space that forces the words on either side of it to remain together.  As far as I know, there's no implied promise that the width of the space will never change.

A clumsy workaround that might help:  Use a normal character rather than the non-breaking space, then set the normal character's fill to No Fill.  Now the twoawords (two words with a normal character between them) look like one single word to the justification algorithm but look like two normal words to the casual observer.

Your English is actually quite good, but I'm not 100% certain what you are asking -- you have asked a number of questions here.

The first question is about the detection and removal of a threat. The location is the Temporary Internet Files folder, which is named in a confusing manner. The folder name does not mean that IE is not up to date -- the folder name remained the same for many versions.

If MSE detects a threat, the default actions to take will vary based on the severity of the threat. You can modify these actions in settings, but there is no need to do so. Serious threats will always be removed or quarantined, but you will still receive a prompt to confirm. Low level threats may be allowed or quarantined.

I'm not sure what you mean by the "file package location" or the open door and how it relates to reinstalling MSE or a System Restore Point. if a threat was allowed, you can remove it from the allow list (History, select the allowed items radio button, delete the item previously allowed, then run a full scan).

I don't know what Pinyin 2010 is.

As for space on the system drive, a clean install of Vista on an empty hard drive will consume a fair amount of space -- 12 gigs or so would not be out of the question. Add programs to that and it will quickly use up more. Additionally, temporary files will consume space. A 40 gig drive is a bit tight for a modern system with Vista. Personally, I usually use a single partition for Vista or Windows 7 these days on large drives. Having a partition (drive) for data is valid and useful if you multiple physical drives. Partitioning a single drive doesn't really safeguard your data.

I can't answer the Vista search concern except that my experience with Search in Vista was that it can be slow, Crashing isn't something that I've encountered. This may indicate some other problem, perhaps hardware such as memory or a faulty drive.

Questions about IE should really be posted in the IE forums.
