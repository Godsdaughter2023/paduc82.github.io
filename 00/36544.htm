Changing compatibility mode as suggested doesn't work but I found this fix elsewhere in another discussion. You need to change the version number of Internet Explorer in the registry from 9.11.10240.16384 to 9.11.10240.0. As with all registry edits a backup is useful if you're not confident.

This is for 64 bit version of build 10240

[HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Internet Explorer]
"Version"="9.11.10240.0" (originally: 9.11.10240.16384)

32 bit version:

The registry entry is not the same on the 32 bit version of build 10240; there is no Wow6432Node sub folder. Instead the entry is here:

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Internet Explorer]
"Version"="9.11.10240.0" (originally: 9.11.10240.16384)

I assume it will work for the 32 bit version in the same way.

As a work around, I suggest you to check if the Volume Shadow Copy service (VSS) is running properly.

Steps:

  1. Press Windows + R to open Run command, type Services.msc and press Enter.
  2. Search for Volume Shadow Copy Service, right click on it and start the service.
  3. Also, set the Startup type to Automatic.
  4. Log-out from the current user account, restart, re-log-in and check if the issue persists.
