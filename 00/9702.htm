charles kenyon say:

Here is a macro that should set the proofing language for all text in a document to EngliskUK.

Sub ProofingLanguageEnglishUKAllStory()    ' based on field updater by Greg Maxey
    ' https://gregmaxey.com/word_tip_pages/word_fields.html
    ' Charles Kenyon 6 November 2018
    ' https://answers.microsoft.com/en-us/msoffice/forum/all/force-all-documents-to-be-edited-in-uk-english/df6d1f8e-5426-49d9-bea0-5620d0208294    ' Changes proofing language to English UK in all stories of document
    Dim rngStory As Word.range
    Dim oShp As Shape
    Dim oTOC As TableOfContents, oToa As TableOfAuthorities, oTof As TableOfFigures
    For Each rngStory In ActiveDocument.StoryRanges
      'Iterate through all linked stories
      Do
        On Error Resume Next
        rngStory.LanguageID = wdEnglishUK
        Select Case rngStory.StoryType
          Case 6, 7, 8, 9, 10, 11
            If rngStory.ShapeRange.Count > 0 Then
              For Each oShp In rngStory.ShapeRange
                If oShp.TextFrame.HasText Then
                   oShp.TextFrame.TextRange.LanguageID = wdEnglishUK
                End If
              Next
            End If
          Case Else
            'Do Nothing
        End Select
        On Error GoTo 0
        'Get next linked story (if any)
        Set rngStory = rngStory.NextStoryRange
      Loop Until rngStory Is Nothing
      Next
End Sub

Editing returns in wdEndnotesStory with array macro? - Microsoft Community

stet7 say:

Editing returns in wdEndnotesStory with array macro?

Hi,


Following a template given here--

http://answers.microsoft.com/en-us/office/forum/office_2013_release-word/find-replace-array-macro-in-story-ranges/c3ae0515-ff8b-48cf-9b6a-66fecdbc1d83

--I put together the following macro to do a quick cleanup on endnotes:

Dim i As Long, ArrFnd(4) As Variant, ArrRep(4) As Variant

'multiple returns
ArrFnd(1) = "^13{2,}"
ArrRep(1) = "^p"

'leading space, return
ArrFnd(2) = " ^13"
ArrRep(2) = "^p"

'trailing space, return
ArrFnd(3) = "^13 "
ArrRep(3) = "^p"

'add period at end of note if there is no other end punc
ArrFnd(4) = "([!.\?\!^0148])(^13^02)"
ArrRep(4) = "\1.\2"

With ActiveDocument.StoryRanges(wdEndnotesStory).Find
  .ClearFormatting
  .Replacement.ClearFormatting
  .Forward = True
  .Wrap = wdFindContinue
  .MatchWildcards = True
  For i = 1 To UBound(ArrRep)
    .Text = ArrFnd(i)
    .Replacement.Text = ArrRep(i)
    .Execute Replace:=wdReplaceAll
  Next
End With

Only the 2nd and 3rd arrays work properly. This is apparently because the returns at the ends of endnotes are special somehow, having the power to defy the replace function. 

I've found a macro for overcoming that problem--

http://editorium.com/archive/deleting-extraneous-carriage-returns-in-footnotes-and-endnotes/

--but it uses the selection method, and I have no idea how to adapt it to what I have.

Many thanks, as usual, for any help you give on this.

stet7 say:

OK, I think I've solved this, in case anyone else finds it useful. The following macro knocks out the following problems in endnotes: leading spaces ahead of paragraph returns, multiple paragraph returns, and endnotes without end punctuation (that is, it adds a period at the end of each note if it doesn't already end with a period or a question mark, exclamation point, or closing double quotes). It's adapted from the Editorium macro in the link above, with the advantage that you don't have to click in the endnotes pane to trigger it.

Sub EndnoteCleanup()

ActiveDocument.StoryRanges(wdEndnotesStory).Select

NoteCount = ActiveDocument.Endnotes.Count

With Selection.Find
  .ClearFormatting
  .Text = " ^p"
  .Replacement.ClearFormatting
  .Replacement.Text = "^p"
  .Forward = True
  .Wrap = wdFindContinue
  .Format = False
  .MatchWildcards = False
End With
Selection.Find.Execute
On Error GoTo TrapTheError
While Selection.Find.Found
  Selection.MoveLeft
  Selection.Delete
  Selection.Find.Execute
Wend
GoTo NextReplace1

NextReplace1:
With Selection.Find
  .Text = "^p^p"
  .Replacement.Text = "^p"
  .Forward = True
  .Wrap = wdFindContinue
  .MatchWildcards = False
End With
Selection.Find.Execute
On Error GoTo TrapTheError
While Selection.Find.Found
  Selection.MoveLeft
  Selection.Delete
  Selection.Find.Execute
Wend
GoTo NextReplace2

NextReplace2:
With Selection.Find
  .Text = "[!.\!\?^0148]^13"
  .Forward = True
  .Wrap = wdFindContinue
  .MatchWildcards = True
End With
Selection.Find.Execute
Do While Selection.Find.Found
  Selection.Collapse Direction:=wdCollapseStart
  Selection.MoveRight Unit:=wdCharacter, Count:=1
  Selection.TypeText Text:="."
  Selection.Find.Execute
Loop
GoTo TheEnd

TrapTheError:
ErrorCount = ErrorCount + 1
Selection.MoveRight
Selection.Delete
If ErrorCount < NoteCount Then Resume Next

TheEnd:
Selection.HomeKey Unit:=wdStory
End Sub

john - project say:

ICCarl,

With regard to your first issue, a resource's time off IS highlighted when you look at that resource's calendar. For example, the the following screen shot, Joe is off from 6/26/17 through 6/30/17. However there is no way to explicitly show this time off in any other calendar view or on any timeline. You can see a resource's work availability on the Resource Usage view.

With regard to your second "issue", you shouldn't be getting an overallocation message if you try to assign a resource to a task during a period when that resource is off. Unless you have selected the option to "ignore resource calendars", Project will re-schedule the task to the first available time for that resource (i.e. when the resource is available). Overallocation is a different "animal". Overallocation applies if you assign a resource beyond his/her max unit level. For example, if you have two simultaneous (or overlapping) tasks and assign a resource with a max unit value of "1" to both tasks, that resource will be flagged as overallocated.

Hope this helps.

John
