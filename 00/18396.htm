You need to create an access point (i.e. connection profile on the phone settings) matching the name and security settings of your home WLAN network.

 

Do following configuration on the phone while you are within range of your home WLAN network.

 

1. Check your WLAN router settings (refer to it's user manual / getting started guide etc.) for the name of the WLAN network and what WLAN security mode and keys have been defined on the router. Take note of these since you need to configure the same on the phone.

 

2. On the phone, Go to Tools (or Control Panel) -> Settings -> Connection -> Destinations -> Internet

 

3. Your "Internet" destination probably already has at least one "access point" which refers to your cellular packet data GPRS / 3G connection (iCellcom). Leave iCellcom access point as it is because otherwise you will lose the cellular data connectivity as well. In order to make your phone to "use" your home WLAN you need to add an access point also for the home WLAN within the "Internet" destination.

 

4. If your "Internet" destination already has one or more access points refering to the name of your home WLAN network and your phone still fails to connect via this, then it's probably best to remove those home WLAN "access points" and try to create a new one from the scratch.

 

5. While you are still inside the "Internet" destination menu, select Options -> New access point

 

6. Allow phone to "Automatically check for  available access points" and take a look at the list of WLAN networks that your phone was able to find. If you are within reach of your WLAN network and your WLAN router is up and running properly you should see the "name" of your home WLAN network on the list. 

 

7. Select the network and your phone should ask you to enter WLAN security parameters, unless your WLAN network runs without any security. Typically security of the WLAN router defaults to either a WEP Key or WPA Pre-Shared key. Enter the correct security key when prompted by the phone and note that this must be entered absolutely correctly (Pre-shared key is case sensitive and WEP keys are sensitive to lenght). So if you are unsure of the correct security key's then check this from the router's user manual or you can usually chahge the default key's via routers web configuration interface. Note that sometimes default security key's are also written on the sticker under/behind the router.

 

8. Now you should have a new "access point" defined within the Internet destination. Just make sure that your WLAN access point is higher than your cellular data access point on "list" of access points so that phone prefers the home WLAN connection whenever it is available. If needed you the change the priorities of access points via Options-> Organize -> Change Priority.

 

9. Go one level back in the menu so that you are in the list of Destinations (Internet, WAP Services, MMS etc.). Here you can define what is the default Destination (group of access points) that some of the applications (without their own connectivity selection settings) will utilize. Select Options -> "Default Connection" and you will be shown a list of Destinations to choose from as well an option of "Always Ask". You can choose "Internet" destination as your default connection for automatic usage of Internet destination or you can choose "Always Ask" which allows you to choose manually which Destination to utilize when (some) applications attempt to open up a connection.

 

10. Most likely your phone's built-in Web browser has it's own connectivity selection settings which you can control via browser application's settings menu. Start the browser and go to Options -> Settings -> General -> Access Point. Here you can choose either choose the "Internet" destination to be used (attempts to connect with highest priority access point available at the time within the "Internet" destination) or you can select "Always Ask" which should allow you to manually choose the connection method everytime Web browser attempts to connect.

 

Note that when browser is set to "Always Ask" you will be shown a list of those Destinations where at least one access point is currently available. If multiple access points are available within a single Destination phone will try to use the one with highest priority.  If you ever need to temporarily override the access point priorities within Destiantion (without going to Destinations and changing the priorities permanently) you can highlight the given Destination (at choose connection dialog) and select Options -> "Select Access Point". This is where you can temporarily select the "lower priority" access point to be used instead of the highest priority available access point.

 

11. Java based applications (like Opera Mini) and some of the older native applications,are not utilizing the automatic connection selection logic provided by the Destinations and these applications may either have their own individual "access point" selection setting within their own "settings" menus or they will revert to the "Default Connection".

 

12. For Opera Mini and other Java based applications,  there usually a way to control their access point selection via to the Application Manager (the one that shows a list and details of all installed applications). I'm not sure where exactly Application Manager is located on N86, but usually it's something like Tools -> Application Manager or Settings -> Data Manager -> App.manager. 

 

When you are at the Application Manager, go to Installed Apps -> find and highlight the Opera Mini -> select Options -> Settings -> Access Point -> Default access point. In case you leave default access point as "None" your Opera Mini should always prompt you to select which access point (like your home WLAN or iCellcom) it should use while connecting this time. Most likely your Opera Mini is by default set to utilize the cellular packet data conneciton (iCellcomm) so you won't be provided an option to choose your home WLAN network even if it would be available and you have successfully defined the access point for WLAN. 

 

Note that If you specifically select e.g.your home WLAN access point for Opera Mini then it will completely fail connecting whenever you are out of range of your home WLAN. Thus it's probably better to set Opera Mini so that default access point is set to "None" and select the connection method manually each time starting to browse.

 

Another tip related to Java applications connectivity is that under that very same Application Manager menu, you might also be able to control whether paricular Java application will ask for user's "permission" before it connecting i.e. allowed to use networking functionality. See Network Access option which has option like Ask Every Time / Ask first time only / Always allowed. Note that "always allowed" might not be available for all Java applications but even choosing "Ask first time only" might reduce the number of "permission" queries.

It's been a while since I've used Opera Mini and just realized that in order to get Opera Mini to prompt which connection method to use you need to set your phone's Default Connection to Always Ask (via Settings -> Connection -> Destinations -> Options -> Default Connection) AND also configure Opera Mini's default access point  (via Application Manager) to "None".

 

I don't have any experience on eBuddy or Nimbuzz IM clients but based on your description it sounds that those are most likely native Symbian / S60 applications (not Java apps, like Opera Mini) thus they don't have the "Settings" item visible on the Application Manager, like Java based applications have. Most native applications either have their own access point / connectivity settings built-in to their own menus or they simply use the "Default Connection" (i.e. connect with "Internet" Destination or allow you to choose in case Always Ask).

 

You don't have to go through all steps between 2-9 in case you wan't to use other WLAN networks. In these cases you can choose "Search for WLAN" when e.g. your built-in Web browser asks you to choose the connection method. This should show you a list of currently available WLAN networks. When you choose a network and enter the security key (if required by the network) your browser should connect through this WLAN network.

 

This situation where phone connects to a new, previously undefined WLAN network, is where the Internet Connectivity Test comes to play (which you had set to "Ask every time" ). When you connect to a new WLAN network and allow the Internet Connectivity test to be run your phone will try to determine if this connection actually works properly, providing successful internet connection.  If Internet Connectivity Test succeeds your phone will automatically create a new "access point" for this particular WLAN network and places it under "Internet" destination. So next time you would like use this WLAN network your phone already has an access point defined and you can use the connection without entering the security key again (or phone will automatically utilize this connection in case you have set Internet destination as your Default Connection).

 

If you know for sure that you won't be using this particular WLAN network ever again and/or don't want it to be automatically saved as an access point, then you can simply skip the Internet Connectivity Test and your Web browser will still connect using the network you chose via "Search for WLAN" but permanent access point will not get automatically created. If Internet Connectivity Test is set to "Run Automatically" then you can't skip the test and phone will always create an access point to Internet destination assuming that connection via Search for WLAN did provide successful internet connection.

e. Confirm the Uninstall.

f. Close device manager.

a. Locate the executable file (.exe file) for the driver installation program.
