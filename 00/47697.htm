    Games with dedicated servers don't generally work by your console settings, instead they use IP or ping results to pick the best server for you. Both Titanfall and Battlefield use the ping results to your best datacenter which is either going to be in the Netherlands for you or London.

    Windows 7 activation error 0x80070005 access is denied - Microsoft Community

    ben_userdn say:

    Windows 7 activation error 0x80070005 access is denied

    We had an existing machine, Windows 7 64-bit ultimate, which was already activated and running fine. We added more memory to it, and it then started to continually pop up a message saying activation failed.  The error code displayed is 0x80070005, access denied. Trying to run slmgr.vbs also gives an access denied error.

    So I Googled it: ran "sfc /scannow" to check all the system files (found no issues), checked the registry permissions (they were all fine), group policy settings (machine has never been on a domain so there is nothing there), nothing. The machine is also protected by Kaspersky, and is behind a hardware firewall, so no virus issues.

    Trying to activate by phone also does not work because running "slui 4" also gives an access denied error message and then is unable to display any information that they need to supply for phone activation.

    Eventually the phone activation let me get through to a real life Microsoft support person (level 1 support).  They pretty much checked this:

    http://support.microsoft.com/kb/2008385

    They also ran MGADaig which said everything was fine. They also rebuilt some security file (the licensing store). None of this helped, other than that the activation error which had kept saying you had 21 days left for some time now, now started to count down, and changed the screen background to black. Still gives the same error trying to activate.

    So a level 2 support person took charge of this. They suggested that I rerun the Windows install DVD and choose the "upgrade" option. That DVD did not have SP1 included, so I had to uninstall that first (took three hours at least, and this is a decent machine), and then it started the upgrade. At the very end of the long upgrade (hours and hours later) it said "The upgrade failed, restoring your original operating system." So after spending far over ten hours on this (would have taken less time to rebuild the machine), I was back to square one, except now it was counting down too, and complaining with larger windows.

    So the only option left was to either reinstall clean or try a system restore.

    I decided to try and do a system restore. I had made a full system image backup in February (6 months ago), and restored that one. It took about an hour to restore that image, and that one was able to automatically reactivate once it started! Yai! I have since reinstalled about 98 critical updates (nearly 1 GB downloaded), and also reinstalled service pack 1, and updated all the other software, and if I right click on "My Computer", "Properties" it still shows the machine as activated. <phew>

    So thank you Microsoft for finally giving us a backup and restore with Windows 7 that actually works (as opposed to what was in earlier Windows versions). Most third party backup and restore solutions are not worth the money you pay, and the Apple people always brag about their wonderful backup options. Only the operating system can properly back up and restore itself. Trust me on that.

    Microsoft does not know what causes this problem, other than suspect some patch or installation that did not go through fully. The level 2 support lady helping was very kind and patient and understanding and apologetic. Towards the end the support person said "Thank you for choosing Microsoft"...

    So lessons from this are:
    - Make periodic full system image backups using the Windows 7 backup feature. Make them to another physical hard drive in your machine. System restore does not seem to list backups made on network locations, it only list the default backup locations it would suggest.
    - If your machine gets stuck wanting to activate but does not count the days down, consider leaving it be!
    - Configure your system so critical files and data are stored on a separate hard drive (I moved my Documents to another physical drive). However, the system restore did tell me that my documents will remain intact and it did in fact, and even some newer licence and other files on the C drive were still there after the restore. And after the restore I have had no issues yet, it seems to work well. (Contrary to server 2003 restores where nothing will work, blue screen, you name it).
    - Search MSDN / support / KB for solutions. If nothing on there works or applies, consider that rebuilding or restoring an earlier backups may be your fastest "solution". I went through Microsoft support hoping it may be just some permission setting they can easily fix, but no such luck. From what the support person told me, everything they use and try and know is on the MSDN / knowledge base / KB support web site (i.e. the stuff you find Googling the errors), so if it is not there, do not think that phoning support is necessarily going to be able to fix it for you and save time.
