graham mayor say:

The following macro should do the trick. http://www.gmayor.com/installing_macro.htm You will have to choose whether you want to save the changed document near the end of the macro and enter the name of your PDF printer (as shown in Word when you select it) between the quotes where indicated. Try it on a test folder containing a handful of copy documents first!

Sub BatchProcess()
Dim strFilename As String
Dim strPath As String
Dim oDoc As Document
Dim fDialog As FileDialog
Dim sPrinter As String

With Dialogs(wdDialogFilePrintSetup)
    sPrinter = .Printer
    '*****************************************
    .Printer = "Put your PDF printer name here"
    '*****************************************
    .DoNotSetAsSysDefault = True
    .Execute
End With

Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)
With fDialog
    .Title = "Select folder and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
               "List Folder Contents"
        Exit Sub
    End If
    strPath = fDialog.SelectedItems.Item(1)
    If Right(strPath, 1) <> "\" _
       Then strPath = strPath + "\"
End With

If Documents.Count > 0 Then
    Documents.Close savechanges:=wdPromptToSaveChanges
End If
If Left(strPath, 1) = Chr(34) Then
    strPath = Mid(strPath, 2, Len(strPath) - 2)
End If
strFilename = Dir$(strPath & "*.doc?")

While Len(strFilename) <> 0
    WordBasic.DisableAutoMacros 1
    Set oDoc = Documents.Open(strPath & strFilename)
    With ActiveWindow.View
        .ShowRevisionsAndComments = False
        .RevisionsView = wdRevisionsViewFinal
    End With

    'Do what you want with oDoc
    oDoc.PrintOut
    oDoc.Close savechanges:=wdDoNotSaveChanges
    'OR
    'oDoc.Close savechanges:=wdSaveChanges
    WordBasic.DisableAutoMacros 0
    'End If
    strFilename = Dir$()
Wend
With Dialogs(wdDialogFilePrintSetup)
    .Printer = sPrinter
    .Execute
End With
End Sub

rich michaels - mvp (greatcirclelearning.com) say:

I don't recall in any previous version of Word, maybe someone else does, that pasting images into a document ever resulted in an automatic capture into Alt Text of the path of the pasted image. If an image is inserted as a linked image then the path is of course saved into an XML node of the document bundle.

Since you mentioned macro, VBA, or add-in code that might help you here are a few to perform a web search on and try out.

1. AuthorTec Insert - an add-in that does image insertions (embedded, linked, or the special combo of both) and does save the image path in Alt Text.

2. AuthorTec Accessibility - an add-in that make it very easy to Review & Edit all objects that require Alt Text in a Word document. It saves a bunch of clicking back and forth.

3. AuthorTec Link - an add-in that is used to change the path on linked images, which is very helpful when the master location needs to be updated. It also converts linked images to embedded or embedded to linked as an option.

4. For a custom VBA solution you will need the following for the core of your macro:

    Dim iShp As Word.InlineShape, shp As Word.Shape

    On Error Resume Next

    For Each iShp In ActiveDocument.InlineShapes

        If iShp.LinkFormat.Type = wdLinkTypePicture Then

            Debug.Print iShp.LinkFormat.SourceFullName

        End If

    Next

    For Each shp In ActiveDocument.Shapes

        If shp.LinkFormat.Type = wdLinkTypePicture Then

            Debug.Print shp.LinkFormat.SourceFullName

        End If

    Next

Hope this helps

doug robbins - mvp office apps & services (word) say:

As indicated in the Alt Text pane, the purpose of the Alt Text was really to enable screen readers to describe the image to a person who is blind

And as the presence of the filename in the Alt Text interfered with its purpose and in view of the security issues that had been raised with Microsoft when the Alt Text revealed the source of the image, a decision was made to remove the source details from the Alt Text.

If the images are linked to the document, the following modification of Rich's code will assign the source of each image to its Alt Text

 Dim iShp As Word.InlineShape, shp As Word.Shape
    On Error Resume Next
    For Each iShp In ActiveDocument.InlineShapes
        If iShp.LinkFormat.Type = wdLinkTypePicture Then
            iShp.AlternativeText = iShp.LinkFormat.SourceFullName
        End If
    Next
    For Each shp In ActiveDocument.Shapes
        If shp.LinkFormat.Type = wdLinkTypePicture Then
            shp.AlternativeText = shp.LinkFormat.SourceFullName
        End If
    Next
End Sub

As a work-around for new documents or when inserting images and at the same time assign the source of the image to its AltText, you can use

Dim FD As FileDialog

Dim strFileName As String
Dim ILS As InlineShape
Set FD = Application.FileDialog(msoFileDialogFilePicker)
With FD
    .Title = "Select the image that you want to insert."
    .Filters.Clear
        .Filters.Add "All Pictures", _

"*.emf;*.wmf;*.jpg;*.jpeg;*.jfif;*.jpe;*.png;*.bmp;*.dib;*.rle;*.gif;*.emz;*.wmz;*.pcz;*.tip;*.tiff;*.svg;*.pct;*.pict"
        .AllowMultiSelect = False
        If .Show = -1 Then
            strFileName = .SelectedItems(1)
         Else
            MsgBox "You did not select an image."
            Exit Sub

    End With
Set ILS = ActiveDocument.InlineShapes.AddPicture(strFileName, False, True, Selection.Range)
ILS.AlternativeText = strFileName
End Sub

hal hostetler - mvp-outlook say:

An .OST file is a carbon copy of your email account on the server.  It is considered volatile and expendable; if an .OST file is damaged or erased, Outlook will rebuild the file with the data on the server.  The only exception to this involves IMAP accounts where, due to limitations in the protocol, things like Contacts, Calendars, To-Dos, and the like are stored locally and are subject to damage or deletion.  The email and folder structure are preserved on the server and a rebuild .OST file will have all that data restored to it.

A .PST (Personal STorage) file, by contrast, is a permanent offline storage folder.  These can be added and removed from Outlook at will and are suitable as long-term/permanent backup files.
