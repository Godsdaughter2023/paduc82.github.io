 The parentheses around the arguments aren't appropriate unless you either
assign the return value of the function or use the Call statement to call
it;  e.g., "x = Shell(foo)" or "Call Shell(foo)".  To invoke it and discard
the return value, just write "Shell foo".  Also, I think you'll have to
embed quotes around the path, because it contains a space.  Try this:
    Shell """C:\Program Files\Windows NT\Accessories\Wordpad.exe""",
vbMaximizedFocus
 If you want to call it an pass a file name on the command line, you might do
it like this:
    Shell """C:\Program Files\Windows NT\Accessories\Wordpad.exe""
MyFile.rtf", vbMaximizedFocus
 If you need to specify the path to the .rtf file as well, and that path
contains spaces, you'll need to enclose it, too, in its own set of quotes:
    Shell """C:\Program Files\Windows NT\Accessories\Wordpad.exe"" ""C:\My
Path\To\MyFile.rtf""", vbMaximizedFocus

they don't end the whole string literal.
 
To simplify the building of such strings, I usually have a public constant Q
defined in my databases:
   Public Const Q As String = """"
 
That defines just one double-quote.  Then I use that in string expressions
that need to embed a quote, like this:
'
   Dim strCommandLine As String
'
   strCommandLine = _
       Q & "C:\Program Files\Windows NT\Accessories\Wordpad.exe" & Q & _
       " " & _
       Q & "C:\My Path\To\MyFile.rtf" & Q
'
   Shell strCommandLine, vbMaximizedFocus
'
That makes it more readable, makes me less likely to mess up the quotes.

thomas albert adams say:

Turns out, the problem was "USER ERROR". I just finished installing WIndows 10 on the 14 systems in the computer lab here in the Senior Center. I discovered that I hadn't properly set the adjustments in the Network and Sharing Center. Here's what you need to do:

1. Open Network and Sharing Center

2. Click Change advanced sharing settings (in the left column)

3. There are 3 entries to change (Private, Guest or Public, All Networks.

4. Private: Click Turn on network discovery, Turn on file and printer sharing and Allow Windows to manage homegroup connections (recommended)

5. Guest or Public: Click Turn on network discovery, Turn on file and printer sharing

6. All Networks: Click Turn on sharing so anyone with network access can read and write files in the Public folders, Use 128-bit encryption to help0 protect file sharing connections (recommended), Turn off password protected sharing.

7. Click Save changes and close the window.

Some of these changes may require you to Log Out and then back in. Do that and you should then be able to "see" other systems in your network that have shared partitions, folders, etc.

Hope this helps, (it helped me)
