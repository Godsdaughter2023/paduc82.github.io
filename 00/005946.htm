Is it possible to create an access web app offline, and load it to - Microsoft Community

oscar2110 say:

Is it possible to create an access web app offline, and load it to sharepoint once it is complete?

Using Office 2013 Pro Plus, is it possible to create and test an Access web app offline, and load it to a SharePoint server when is it ready to be used? Part of this is personal work style, but there are times when I need to work and the internet is not available.
albert d. kallal say:

As a general rule when working with external data sources during development, you need the ability to utilize those data sources. The solution in most cases is thus to have a local web server during development, and then right before deployment you "switch" the connection to the live production data. However, obviously if you don't have use of the internet, then your data during development would have to come from somewhere.

 

With Access web, in 2013 you will need use of the web system during the development process. You can choose to create a 2010 web application, and that choice does allow local development without the web server until such time you have a connection, and then you sync the changes (so, Access 2010 is quite unique in this way that some server system is not required – but this is the exception for most web development system, not the rule).

 

If you choose a 2010 web format, then your tables will not reside in SQL server tables, but will reside in SharePoint tables – there are performance and design considerations for this trade off.

I am not 100% from 2013 how to from scratch create a 2010 format – you can grab any 2010 web template, and use one of those, (just delete what parts you don't want).

 

So, with 2013 web applications, you need a connection to the web server, but 2010 web apps do have the ability to work off line (which as noted is not an option for most web development systems ). And 2013 does allow one to work on + use 2010 web applications - you just need to start out with a 2010 web format (say a template).

 

So as a general rule, to do web development, then some kind of web server system (local or hosted) is required.

The reason it took so long for someone to answer is that there's nothing wrong with any of those things, they're all normal Windows 7 functions or operations.

You're not hacked, you're simply turning on settings like hidden files and looking around at things the typical consumer doesn't understand and should never look at, precisely since they are confusing.

All current version of Windows remove certain permissions from the user in order to protect them from themselves.  The OWNER permission exists to allow the original creator of a file to make changes, while not allowing others until they first change those permissions, which only an administrator is allowed to do.  The $Window-WS folder exists to store Windows update files during the early potions of an update process, when the files are being staged before installation.

There's nothing wrong here at all, you are simply confused by things you don't understand. Leave them alone.
