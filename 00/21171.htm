Duration fails to compute correctly when resources are changed - Microsoft Community

boringusername say:

Duration fails to compute correctly when resources are changed (Fixed Work task)

I believe I've run into a nasty bug in Project.  In short, I have Fixed Work tasks, and the Duration calculation appears to be erroneous as I modify the resources available to a given task.  If this is user error (i.e., not a bug), I'd appreciate guidance.

Here are the steps required to produce the apparent discrepancy:

Create a new project (blank)

At the bottom, set new tasks to Auto Scheduled.

File --> Options --> Schedule -->
  Work is entered in = Days
  Default task type = Fixed Work
Ok

Insert the column showing "Work" between the Duration and Start columns.

Switch to the Resource Sheet
Create two resources, "Bill" and "Clippy" accepting default options (including Type=work, Max=100%, costs=0.00)

Back to Gantt chart view
Create a task "Paint House" accepting defaults.

Select "Paint House", and bring up information (shift-F2).
Under Resources tab, add Bill and Clippy, each at 50%.
Ok

Now, in the Work column, specify 5 days of work for the task.  Since there's a total of 100% of a person working on the task, the duration is correctly calculated as 5 days.

With that task selected, bring up task information again (shift-F2).
Under Resources tab, increase Clippy's "Units" to 100% and hit Ok.

We would now expect the Duration for "Paint House" to be 5.0/1.5 = 3.33.  However, the Duration is unchanged.  This is discrepant, right?

At this point, the Duration calculation appears unpredictable.  For example, I now set Bill to 25% and Clippy to 75%, and the Duration is showing 10 Days rather than the expected 5.

And, if you keep playing with combinations of those two resource levels, you'll find that Project may decide to alter the Work - even though this is a FIXED WORK task.  Only the Duration should change when I alter the resource allocations.  For example, try Bill=100% and Clippy=0% - the WORK changes to 2.5 days!  (though on the plus side the Duration does match at 2.5...)  Is this a bug too? 

I have run Microsoft Update, but it does not indicate any available updates to Office and/or Project.  I'm not able to find on the MS web site what the latest version/patch may be; I am running Version 14.0.4760.1000 (32-bit).

Thank you for any help you can provide.

julie sheets say:

As you have discovered, if you want project to divide the work up based upon assignment units, you need to assign each resource separately.  You are correct, project calculates resource work only upon assignment.  After a resource is assigned then Project will recalculate one of variables of work. duration, or assignment units depending upon task type.  Project will not redistribute work if you change assignment units with multiple resources being assigned.  You can manually change work, and project will usually recalculate duration unless as in the circumstance you note, there are multiple resources assigned and one resource is still working on the task. 

To your point about changing work on a Fixed Work task -- it doesn't unless you change a resource's assignment units to zero.  If you don't want the resource to work on the task, remove them -- and as long as the task is effort driven (it is if the task is Fixed Work) the total work will stay the same and will be redistributed to the remaining resources based upon their assignment units.

I hope this helps.

Julie
