graham mandeno say:

It is a very good idea to use late binding to avoid problems like this with references.  It works like this:

1. Declare all referenced object variables As Object instead of using the object model type (e.g. As Excel.Application)

2. Declare any constants you use with their correct values (e.g. Const xlExcel8 = 56). You can find these values in the Object Browser while you have a reference set)

3. Always use CreateObject or GetObject (not the New keyword) to create your base object (usually an Application)

For example, the following procedure creates a new Excel workbook and saves it in the old (Excel 97 - 2003) formal:

Public Sub CreateXLS()
Dim objXLApp  As Excel.Application
Dim objXLBook As Excel.Workbook
    Set objXLApp = New Excel.Application
    Set objXLBook = objXLApp.Workbooks.Add
    '...
    objXLBook.SaveAs "C:\MyExcel.xls", xlExcel8
    objXLBook.Close
    objXLApp.Quit
End Sub

Using late binding, the same procedure looks like this:

Public Sub CreateXLS()
Const xlExcel8 = 56
Dim objXLApp  As Object
Dim objXLBook As Object
    Set objXLApp = CreateObject("Excel.Application")
    Set objXLBook = objXLApp.Workbooks.Add
    '...
    objXLBook.SaveAs "C:\MyExcel.xls", xlExcel8
    objXLBook.Close
    objXLApp.Quit
End Sub

The disadvantage of late binding is that you lose features such as intellisense and compile-time checking which don't matter at run-time but are very useful for development.  By using conditional compilation you can have the best of both worlds:

#Const EarlyBinding = 1

Public Sub CreateXLS()
#If EarlyBinding Then
Debug.Assert xlExcel8 = 56
Dim objXLApp  As Excel.Application
Dim objXLBook As Excel.Workbook
#Else
Const xlExcel8 = 56
Dim objXLApp  As Object
Dim objXLBook As Object
#End If
    Set objXLApp = CreateObject("Excel.Application")
    Set objXLBook = objXLApp.Workbooks.Add
    '...
    objXLBook.SaveAs "C:\MyExcel.xls", xlExcel8
    objXLBook.Close
    objXLApp.Quit
End Sub

When you are developing you add the reference to the Excel library and set the EarlyBinding constant to 1, and when you deploy your application to your client you remove the reference and set EarlyBinding to 0.

Note that the Debug.Assert line ensures that, when you are early binding, the value you are going to use later for the constant is the correct one.

Good luck!
Graham

This is the expected behavior... AFAICT all versions of Word through 2010 for the PC & 2011 for Mac work the same way.

richardnjeri say:

how can i adjust my phone to stop downloading applications via wi-fi if i have enough mobile bundles?

My phone is scheduling itself to download applications only via wi-fi whereas I have enough mobile bundles. How can I re-adjust my settings??
timoteo iv ofr say:

Hello Richard,

If you're using WiFi and cellular data at the same time, WiFi will be used as  default to connect to the internet. You can turn off your WiFi in order for you to use your cellular data.

You can also follow the steps below to use cellular data when downloading app updates:

  • For Windows 10 Mobile: go to Store >  > Settings > App updates > Update apps automatically when I'm on WiFi to Off.
  • For Windows Phone 8.1: go to Settings > Applications > Store > App updates > Only get updates over WiFi.

Let us know how it goes.
