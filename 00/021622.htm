Resizing Slide Causes Color Palette Change - Microsoft Community

chaching say:

Resizing Slide Causes Color Palette Change

I have PowerPoint 2013 and when I try to resize my slide presentation from Widescreen to Letter Paper, it changes the color palette.  It looks like it's choosing the default color palette for the theme.  Why can't I just resize it without it changing all my colors?
steve rindsberg say:

This is a known issue.  The explanation is a bit complex, so grab yourself a beer and get comfortable.

Many (all?) of the newer themes from Microsoft are actually what are called SuperThemes.  They not only have color variants (you see these in the Variants section of the design tab and when you start a new file), but also size variants; each color variant is duplicated inside the theme, one for 4:3 and one for 16:9 formats.

This is a good thing when you're changing slide sizes because it prevents the theme graphics from getting distorted.

But in order to do that, PPT applies the appropriate sized theme when you change sizes, and in doing so, applies the color theme as well.  If you've modified the color scheme then change sizes ... well ... you've seen what happens.

I suspect you can work around the problem by saving your custom color scheme then re-applying it after you've changed sizes.

chip pearson say:

Are you using the date in a formula? If so, you can use the IFERROR function to evaluate the formula and if an error occurs, display the text of you choice. For example,

 

=IFERROR(your_formula_here,"Bad Date")

 

If your_formula_here does not raise an error, the result of the formula is displayed. If your_formula_here does raise an error, "Bad Date" will be displayed.

 

Note that IFERROR was added in Excel 2007 and is not available in earlier versions. If you are using 2003 or earlier, you can use something like

 

=IF(ISERROR(your_formula_here),"Bad Date",your_formula_here)

This has the disadvantage that under normal circumstances (when the date is valid), your_formula_here is caclulated twice: once within ISERROR and again to get the result. This can cause performance problems if you have lots of formulas written this way.
