frederik long say:

You can easily see what's going on by adding a logging capability to your backup command, e.g. like so:

@echo off
if not exist c:\Logs md c:\Logs
xcopy  .. .. .. 1>c:\Logs\backup.log 2>&1

You will then see either or both these error messages:
- Network drive not found, or
- Insufficient permissions

Why? Because drives that are mapped in a foreground session do not exist in a background session. If they did then you would have a big security issue.

If you map the drive in the background session or use UNC coding (preferred!) then you must make sure that the account used for the scheduled job has at least read-access to the mapped drive.

how do i get a security code on my wifi as the neighbours are tapping - Microsoft Community

beccmcburnie say:

how do i get a security code on my wifi as the neighbours are tapping into my usage

the neighbours are tappin ginto my wifi how do i put security setting on my computer to stop this.

kosh vorlon say:

Hi Becc,

 

The procedure is essentially to configure the router to specific security settings appropriate for your router and your ISP and your computers and intranet.  You don't block your computers, you encode the router transmissions so others can't use it without the proper username and password.  Then you configure your computers (or other connected devices) so they know the proper codes and can communicate with the router.

 

When I ran into this myself, I discovered that the router manufacturer charged a fee; however, I discovered that my ISP supported that particular router and they helped me both with the router and my computers for free as part of their service - so I personally would start with your ISP to see if they'll help.

 

The following article gives basic instructions for the router (and some different ones), but it only covers the router (and not your connected devices) and it suggests options to use but "the best" sometimes isn't appropriate or available with your ISP or router or computers (meaning it's OK to learn about it in a general sense, but you need to contact people to get the correct settings).  I for example would like to use WPA2 but my ISP will only support WPA so that's what I use (and if I tried the other it wouldn't work and I'd have had no clue what I was doing wrong and be stuck and confused). http://www.ehow.com/how_6942409_set-up-wireless-router-security.html.  So think of this as merely an overview guide but use what follows instead.

 

Briefly, to access the router, go to any connected computer and go to Start / All Programs / Accessories / right click on Command Prompt and click run as Administrator.  Then type IPCONFIG /all.  For your primary connection, you should see a group of settings like Default gateway (and possible other entries) referring to a TCP/IP address (normally 192.168.0.1 or something similar to that).  That's the address of your router.

 

Now open IE (or your browser) and in the address field where you might sometimes type www.microsoft.com, instead type the router address (e.g., 192.168.0.1) and enter.  This should connect you to the router where it will ask for a username and password (almost certainly).  If you know it, great.  If not the ISP or router manufacturer can help.  Many use common default ones (which you should change once you get it but be sure to remember).  Some common username/password combinations include admin/admin, admin/password, administrator/password, administrator/admin, or just administrator or admin with nothing for the password.

 

Once in there should be a section for security.  There you will need to most likely provide a Security Type (WPA-personal, WPA2-personal,...), an Encryption Type (e.g., TKIP, AES,...) and a Network Security key (think password).  The first two depend too much on your ISP and setup for me to help there (and you may have more or different options too) - so they need to tell you what to use.  The router manufacturer probably can't help with this - but the ISP will know and can tell you.  Use a strong key as you want it hard to break.  Here's an article on creating strong passwords - for this and especially since you know you're being used, I'd max out and go even a bit further than they suggest: http://www.microsoft.com/security/online-privacy/passwords-create.aspx.  I'd choose one with 12 digits that is a random set of lower case letters, upper case letters, numbers, and symbols - but also one you won't forget but that nobody can guess (as you really don't want to have to reset it).

 

Once set, then even though not recommended by the router or ISP people because if you forget your passwords even they won't be able to help and the router becomes a paperweight (but it does prevent someone from resetting the router and using the same tricks I suggested above to find one that works - if they did work and you didn't need help to identify it), I'd consider changing the reset default username and password so nobody can use that method to bypass everything and get in (even though you'd notice that and end up being locked out yourself - and they'd pretty much have to have physical access to the router to reset it as I don't think it can be done remotely).  I did that with mine.  Others choose not to bother since they assume nobody is going to go to such extremes and it would make things simpler if you needed help later and forgot the password.  That's your choice.  I did it because I'm a bit paranoid and know I won't forget it - but for most I suggest they leave it alone as that may be going a bit overboard.

 

Next go to every device connected wirelessly or otherwise to the router and setup the security codes to allow that device (your computers and perhaps a printer or some other devices) to access the router and not be blocked just like your neighbors now will be.  Go to Network Connections and Sharing and then to the list of wireless connections and select your router connection and right click and click on Properties.  Click on Security and the three options mentioned above should appear.  Change them to match what you used for the router and save the settings.  Now close the connection to the router (if it's working which it shouldn't be) and re-establish a connection and you should be in allowed in.  Do the same for all devices (printers and other devices probably have special methods to change those settings and you may need help from those vendors to figure out how to revise the security settings to connect to the router - but generally it's really not that difficult and you can probably work it out yourself or by using self-help on that device's website or the manual if you still have it).

 

That's it.  You're now done and you're secure.  Again, you will need help from the ISP with the settings (and they may also help you with everything else too if you ask and need it), but with that exception, for the most part I think you can mostly do it on your own.  Go to the router manufacturer only if absolutely necessary and the ISP won't or can't help and you can't figure it out yourself since many of them charge for this type of thing (unless it's brand new and some even then) and you need info from the ISP anyway as the router manufacturer probably doesn't know the proper settings for every ISP (though they do for many, so it's not safe to assume they won't know - just less likely than with the ISP).

 

I hope this helps.  Assuming this hasn't already been moved to the Windows Networking Forum, and if this didn't do the trick and you need further assistance, please repost here: http://answers.microsoft.com/en-us/windows/forum/windows_other-networking?page=1&tab=all&tm=1300700638376.

 

Good luck!

Hi Garth,

The failed Windows Update could've triggered a corrupted/incorrect driver causing your device to display an improper resolution. To revert your display to the appropriate resolution, we suggest that you re-install your driver through safe mode. To do this, follow these steps:

  1. Start Windows in Safe Mode. To do this, follow the steps provided on this link.
  2. Once you're in the safe mode environment, right-click on the Start menu or the Windows icon.
  3. Select Device Manager.
  4. Click the drop-down arrow for the display driver you want to uninstall.
  5. Right-click on the display driver and select Properties.
  6. Under the Driver tab, select Uninstall.
  7. Select OK.
  8. Restart your device to reinstall a default or native driver.
  9. Check if the display issue persists. If not, proceed in performing a clean boot to troubleshoot the failed Windows Update. To do this, follow the steps provided on this link.
    Note: Click the drop-down arrow for Windows 10 to view the steps.
  10. Once you're in the clean boot environment, download and run the Windows Update troubleshooter found on this link.
  11. After the troubleshooter is done, run the update again.

Let us know if you encounter any challenges along the process.
