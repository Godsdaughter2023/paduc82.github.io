hairyfool say:

The key bit in your two states is the IPv4 Address.


When you are not connected it begins with 169. , when you are connected it begins with 192. .


Your computer is configured to request an address (think of it like a postal code), this is quite normal and the service which provides it is DHCP (Dynamic Host Configuration Protocol) and the server for this is your router. There are other settings included but this is the key.


When a PC cannot find a DHCP server there is a failover system called APIPA where all computers give themselves an address which is the 169 address. With that 2 (or more) PCs on a network missing a DHCP server can talk to each other, but only each other they cannot exit the network (no internet).


In your 1st 2 pictures DHCP has failed so the failover has come to effect, in the 2nd set DHCP has worked. So whatever you have done between the 2 has fixed the problem. It could be the Ethernet port on the router itself (try a different port as it should have 4)


You indicate that what you have done is reset the router, this will have restarted the DHCP server service in the router and it then starts issuing addresses. Presumably after a period of time the DHCP server hangs and stops doing its job. It might just be possible to be the Ethernet port of the PC but seems unlikely.


If the router is supplied by the ISP you need to see if you can get a replacement . If it is your own there are a couple of things to try. Reset the router back to factory default (the ISP may also ask you to do this before replacing it) although you will then need to reconfigure it. Alternative go to the manufacturers site and see if there is a firmware update. This will reflash the router and may clear out any suspect code even if the firmware version is the same.


Otherwise you will need to get another router. You were pretty close with the point on no default gateway, this is one of the other settings provided by the router and is the name for the way out of the network but the clue is the APIPA address.


As a workaround you could take the setting from when the connection is working and apply them to the network card. From your last picture click on the properties button and look for the IPv4 option. In their it will be set for Automatic... so change it to manual. Then enter these values:


IP Address:                192.168.1.127

Subnet Mask            255.255.255.0

Default Gateway       192.168.1.254


DNS Server                8.8.8.8

Alternate                   8.8.4.4


With those settings if the router is still working at a basic level you should be able to connect to the internet. If not reset the router and try again, if that does work it indicates the Ethernet side is failing altogether.

Codec

meghmala say:

Hi jimwalk,

 

If you get a message that says that your computer is missing a codec, you are probably trying to play, burn, or sync a file that was compressed by using a codec that Windows or the Player does not include by default.

A codec is software that is used to compress or decompress a digital media file, such as a song or video. Windows Media Player and other programs use codec to play and create digital media files.
You may refer to the below link for further guidance on the issue-

http://windows.microsoft.com/en-US/windows7/Codecs-frequently-asked-questions

You may use the below provided link and check if you get the required codec to download as per the file type you are trying to play -
http://www.microsoft.com/downloads/en/default.aspx

You may also use your favorite search engine to download and install a codec as per the file type you need to use.
