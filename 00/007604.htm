Usually this problem occurs if the Office installation goes corrupt, due to multiple versions of office installed in the machine.
Step 1: Remove Multiple/trial Versions of Office Suites:

1. Click Start, click Run, type control appwiz.cpl, and then press ENTER to open the Add or Remove Programs window.
2. Check for multiple/trial versions of Office & uninstall them.
 
If the problem still persists, go to the next steps.

Step2: Add the registry key
Registry disclaimer:
Modifying REGISTRY settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of REGISTRY settings can be solved. Modifications of these settings are at your own risk.

1. Click on Start>Run
2. Copy and paste the below switch

reg add HKCU\Software\Microsoft\Office\12.0\Word\Options /v NoReReg /t REG_DWORD /d 1
reg add HKCU\Software\Microsoft\Office\12.0\excel\Options /v NoReReg /t REG_DWORD /d 1
reg add HKCU\Software\Microsoft\Office\12.0\Outlook\Options /v NoReReg /t REG_DWORD /d 1
reg add HKCU\Software\Microsoft\Office\12.0\Power Point\Options /v NoReReg /t REG_DWORD /d 1
reg add HKCU\Software\Microsoft\Office\12.0\Access\Options /v NoReReg /t REG_DWORD /d 1
reg add HKCU\Software\Microsoft\Office\12.0\Publisher\Options /v NoReReg /t REG_DWORD /d 1
reg add HKCU\Software\Microsoft\Office\12.0\Infopath\Options /v NoReReg /t REG_DWORD /d 1

Restart the computer and check for the issue
 
If the above steps fail, reinstall Office. For more information on how to remove office check the below website:
http://support.microsoft.com/kb/971179

Access Subqueries - Microsoft Community

rob_morph say:

Access Subqueries

Hi,

I am trying to use subqueries in Access to return more than 1 field from the subquery, but I'm not clear how to do this.

My scenario: I have a table of Customers and a table of their Payments. For a particular customer, I want to know the latest payment date but also the type of that payment and the payment amount. I know I can use a subquery to get one of these fields, but how do I get all of them?

Here is my current SQL:

SELECT Customer.CustomerNr,

  (SELECT TOP 1 Payments.PaymentDate

   FROM Payments

   WHERE Payments.CustomerNr = Customer.CustomerNr

    ORDER BY Payments.PaymentDate DESC)

   AS LatestPayDate

FROM Customer

WHERE Customer.CustomerNr = '000123'
WHERE Customer.CustomerNr = '000123'

Any help would be much appreciated!
Thanks



hansv mvp say:

Try


SELECT Payments.*
FROM Payments
WHERE CustomerNr='000123' AND PaymentDate =
   (SELECT Max(PaymentDate)
    FROM Payments
    WHERE CustomerNr='000123')


or, if you want to include other fields from the Customer table:


SELECT Customer.LastName, Customer.FirstName, Payments.*
FROM Customer INNER JOIN Payments
ON Customer.CustomerNr = Payments.CustomerNr
WHERE Payments.CustomerNr = '000123' AND Payments.PaymentDate =
   (SELECT Max(PaymentDate)
    FROM Payments
    WHERE CustomerNr = '000123')

ken sheridan say: Try the following. This should return the latest payments per customer.  You can further restrict the outer query on Customer.CustomerNr by a parameter if you wish:

SELECT Customer.*,
P1.Paymentdate, P1.PaymentType,
P1.PaymentAmount AS LatestPayment
FROM Customer INNER JOIN Payments AS P1
ON P1.CustomerNr = Customer.CustomerNr
WHERE P1.PaymentDate =
  (SELECT MAX(PaymentDate)
   FROM Payments AS P2
  WHERE P2.CustomerNr = P1.CustomerNr);

scottgem say:
I wanted to try to explain something here. You DO need a subquery, but not for selecting the fields, only for selecting the records.

To get all the fields you want, you need to join the 2 tables on the key field fields and select the columns you want in your query. The purpose of the subquery is to restrict the records returned according to criteria. So if you want to restrict the query to the latest payment, that's what you use a subquery for.

ken sheridan say:
The only issue would be if the timestamp on 2 payments was exactly the same in which case both would be picked up.

You can use the table's primary key as an arbitrary tie-breaker in the event of simultaneous transactions.  This is done in the queries in my demo file RowNumbering.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

One of the queries numbers transactions sequentially per customer as follows, using the TransactionID as the tie-breaker:

SELECT COUNT(*) AS RowNumber, T1.CustomerID, T1.TransactionDate, T1.TransactionAmount
FROM Transactions AS T1 INNER JOIN Transactions AS T2
ON (T2.TransactionID<=T1.TransactionID OR T2.TransactionDate<>T1.TransactionDate)
AND (T2.TransactionDate<=T1.TransactionDate) AND (T2.CustomerID=T1.CustomerID)
GROUP BY T1.CustomerID, T1.TransactionDate, T1.TransactionAmount, T1.TransactionID;

Adding a subquery to this:

SELECT T1.CustomerID, T1.TransactionDate, T1.TransactionAmount
FROM Transactions AS T1 INNER JOIN Transactions AS T2
ON (T2.TransactionID<=T1.TransactionID OR T2.TransactionDate<>T1.TransactionDate)
AND (T2.TransactionDate<=T1.TransactionDate) AND (T2.CustomerID=T1.CustomerID)
GROUP BY T1.CustomerID, T1.TransactionDate, T1.TransactionAmount, T1.TransactionID
HAVING COUNT (*) =
    (SELECT COUNT(*)
     FROM Transactions AS T3
     WHERE T3.CustomerID = T1.CustomerID);

would return the latest transaction per customer, returning only one row per customer if their two or more latest transactions were simultaneous.

The file Balances.zip in the same SkyDrive folder also uses the primary key to differentiate between simultaneous transactions when computing balances.
