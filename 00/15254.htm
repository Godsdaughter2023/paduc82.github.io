    Filtering in Outlook 2016 Task Views - On or Before Today - Microsoft Community

    dennisad say:

    Filtering in Outlook 2016 Task Views - On or Before Today (also: On or After Today)

    I am posting this because I could not find an article that told me how to do this in Outlook. Now that I've figured it out, I thought I'd post the result for others who might find it useful.

    One of the task views I like to use shows me tasks that are NOT completed yet, but whose start dates are anytime today or earlier. I've seen descriptions of how to do this in SQL Server and Dynamics, but nothing for Outlook's Filter or Advanced Find settings.

    Outlook tries to be clever and, if you specify multiple conditions for the same field, it OR's them together. Conditions across multiple fields are AND'ed together. You can manually edit this in the SQL tab (when available), but then all other Filter tabs are disabled. Playing with the raw Outlook SQL, while it can be useful, is beyond the scope of this article/discussion.

    Using my example of a Task's Start Date, to find all tasks with a Start Date on or Before Today, you would enter two conditions in the Outlook Filter (or Advanced Find) "Advanced Tab":

    1. Field = Start Date
      Condition = Today
    2. Field = Start Date
      Condition = on or before
      Value = #Today#

    It is vital that you enter "Today" surrounded by hash signs as listed, otherwise it is treated as a character literal string. The reason you need both entries is that #Today# returns today's date at 12:00am. Without the Start Date Today entry, you'd not see any tasks that have a start time later than midnight today.

    This same method can be used for dates on or after Today (just subsititute on or after for on or before as the Condition for #2.

    I've tested this only under Office 2016, but it may work in earlier versions as well.
