john - project say:
LG_776,
Will it work on a network drive? Maybe. The problem with a network is the stability. Networks are subject to dropouts or other glitches that a user doesn't normally experience on a personal workstation. The other problem is the discipline I mentioned. When multiple users have write access to files in a linked structure, the probability that one of them may do something they shouldn't at some point is reasonably high. Training can alleviate some of these issues but there is always a chance of a mishap.

Using Save As on a linked structure effectively duplicates the link structure and that dramatically increases the probability of corruption. The best way to do backups is to create a static master, that is, uncheck the "link to project" option near the lower right corner of the Insert Project window. Be sure to select "save all" when closing so no link fragments are left.

Is a single person in charge of your project files? If not, I strongly suggest you set up that responsibility and insure that person understands best practices with linked structures in Project. Since you started in your initial post that you had multiple issues with crashes, it also sounds like a training program needs to be put into place for your users. As a point of reference, several years ago (and several version of Project ago) the program I worked on had a linked structure. It was network based. We had a formal training program for users. There was one person on the program that was responsible for managing the overall schedule and file structure. File corruption was not a problem.

But again, before you can go forward, you need to back up, clear corruption for your existing files and then rebuild. There is no fix or shortcut.
