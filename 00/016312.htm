scottgem say:

Ok, there are a few things here.

First, Access is a database development platform. (see my blog for a full explanation of what Access is). Part of Access is the ACE database engine. You do NOT need MSAccess.EXE to use data stored in Access tables. All you need are the ODBC drivers to link to the tables. 

So, if MSAccess.exe was installed, then it is probably the runtime version because the application was designed using Access. However, if it really is Version 8, that is ancient. Version would be Access XP I think. 

Second, if they are using an app developed with Access XP, under Windows 7 that could be causing some issues. 

But the bottom line here is that you are using a third party app. So the problem is with that app and you should be going back to the developer of that app for support.
