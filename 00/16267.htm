daniel g. grau say:
Are you using Insert > Symbol > Advanced Symbol > Shortcut? I don't think it can be done like that, because Word only shows a reduced character set (256 ASCII combinations minus 32 control characters = 224 characters). But there is another way:

  • Go to System Preferences
  • In the first row, select Language & Text
  • Click on the Input Sources tab at the top
  • Enable Show input menu in menu bar, at the very bottom

This will add a new menu choice next to the clock, at the top right of the menu bar. Click on it and select Show Character Viewer. You will see many Unicode characters. You can search for them using the Loupe tool at the bottom. Once you've found your character, double click on it to insert it into Word. Although you cannot assign it a key combination, you can define it as an AutoCorrect entry. There are already some such entries, predefined by Word. To see how they work:

  • Select Word > Preferences > AutoCorrect
  • Check Automatically correct... at the very top
  • Click on the Math AutoCorrect tab
  • Enable the two checkmarks
  • Click OK

Now, type \alpha or \beta. Word "autocorrects" them into α and β. That's how it would work in your case.

Say you've added a circled A (Ⓐ) to your document using Apple's Character Viewer. Set the character's size and color, select it and choose Tools > AutoCorrect. In the Replace field, type a unique sequence (like ca\ or \ca, i.e., something that you would not normally type). From then on, just type that to get the circled A.

Note that the AutoCorrect feature is language-dependent—what you define for English texts will no longer work if you switch the language to Spanish.

You could also use a keyboard layout editor like Ukelele to create a custom keyboard (which you would enable through System Preferences and would swap through Command Space), but I've tried it and it's too much work.

I created a form letter in which I inserted text fields.  I want to protect the form so that the only changes that can be made are restricted to the text fields.  I went to the developer tab and selected restrict editing.  Under Edit Restrictions I placed a check mark in the box labeled "Allow only this type of editing in the document" and then chose "Filling in Forms" from the combo drop down box.  I noticed that the "Yes start enforcing protection" button is not highlighted and therefore I can not enforce my restrictions.  My software is activated with Microsoft so I don't understand why this feature does not work.  Any suggestions on how to resolve this issue.
stefan blom say:

If the Design Mode button on the Developer tab is activated, click to deselect it. That should enable the Yes, Start Enforcing Protection button again.

You should also make sure that you didn't add any ActiveX controls to the document. "Filling in forms" protection is intended for the Legacy Forms, not for ActiveX controls.
