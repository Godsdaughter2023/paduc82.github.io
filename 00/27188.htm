On-Line:  First, click on Links:

bernie deitrick, excel mvp 2000-2010 say:

If you have three variables, you could set up a table of all the various combinations of values, with a step size that is reasonable. For example,  you could easily set up a table of 1000 different combinations with coarse differences to cover the entire range of allowed values, and calculate the IRR for all those combinations. Find the value that gives the max for that step size, and then change to values in the table to have finer differences around that one combination. If your relationship is well behaved, you should be able to find the max value relatively easily.

jeffhubb_ say:

I found the regkeys!  They're documented in the Group Policy settings reference for Microsoft Office 2010 (this is an Excel workbook).  You can set these under the "Policies" key or use group policy tools.  They should also work under the user's non-policy key: "HKCU\Software\Microsoft\Office\14.0\OneNote\Options\Versions\"  These are all REG_DWORDs:

  • PruneVersionsOverTime: Set this to zero (0) to disable pruning versions altogether (i.e. keep every version, forever).  This key overrides all the others listed below.  Note that your files may get pretty big over time if you set this (you can still manually delete versions within OneNote).
  • DaysOfHourlyVersions: The number of days to keep hourly versions.  Afterward, they collapse to one version per day.  (So I was wrong above when I said every half-hour.  And maybe different users don't affect version creation after all?)
  • MaxUniqueDayVersions: Number of daily versions to keep (per page).  Once a page has this number of daily versions, OneNote will start deleting old versions.
  • DeleteAllVersionsAfter: Number of days to keep versions.  Versions older thanx number of days will be deleted.  I believe the default is something like 6 months.

Note that these are per-machine settings, so everyone using your notebook will need these set (or else they're fall back to default behavior).  So these make good Group Policy settings.  Backups still aren't a bad idea as these settings won't stop people from manually deleting versions (or just ask them not to!)

You can instruct Access to run a specific Macro by using a command line switch:

"full path to msaccess.exe" "full path to your db" /x MacroName

In your case, the "Program/Script" would be the path to Access along with the path to your database. The "Arguments" would be the "x YourMacroName".
