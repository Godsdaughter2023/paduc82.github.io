sharath srinivasa say:

Hello,

As you may be aware, the current build of Technical Preview contains two Windows Store apps – the old Windows 8.1 Store (simply called “Store”), and the new Windows 10 Store, called “Store (Beta)”.

 

Throughout the forums, you can find troubleshooting methods for repairing the Store app – for example, this Wiki article provides an excellent compilation of these methods.  Unfortunately, these techniques will generally only repair the older version of the Store in Technical Preview.

 

If you are having problems launching the new “Store (Beta)”, we recommend running the following PowerShell script to re-register the new Store:

 

  1. In the Search box, type CMD

  2. Right click on Command Prompt and choose Run as administrator

  3. In the command prompt, enter the following command:

PowerShell -ExecutionPolicy Unrestricted -Command "& {$manifest = (Get-AppxPackage Microsoft.WindowsStore).InstallLocation + '\AppxManifest.xml' ; Add-AppxPackage -DisableDevelopmentMode -Register $manifest}"

jimstavropoulos say:

I found a workaround to this issue.

I have a Mac with El Capitan (not relevant) and Office 2016. When I open a Word attachment from Outlook, it prompts to Duplicate which requires that I save it first in a non-temporary folder area before I can edit it.

My workaround is

1. open the email in Outlook.

2. You will see the attachment with a Word icon. Next to it is the PREVIEW button which uses the Mac's built-in Preview application. Click on the preview button to open it in the preview application.

3. In the preview window in the upper right corner click on the OPEN IN WORD icon. It opens it in Word in edit mode without requiring a save.

I did not find the problem or resolution to be related to illegal characters.It's a deliberate sandboxing issue which makes sense from a security and UI point of view.
