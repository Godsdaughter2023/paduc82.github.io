james sanders say: I've experienced a similar problem with my Asus P8Z77-V motherboard and an Nvidia GTX 670. I'm getting video out to my monitor, but according to the Nvidia control panel there's no compatible hardware installed, so my games run like treacle.

In my case, I did find a workaround which might work for you too. I noticed in Device Manager that the System Device for my PCI Express port had an issue marker against it, so it wasn't working properly. For me, the device was labelled "Xeon(R) processor E3-1200 v2/3rd Gen Core processor PCI Express Root Port - 0151". A right-click on that to select Disable, and then Enable again caused my card to be detected and everything worked properly again.

One snag with the above workaround, it doesn't persist through a restart so every time I boot my machine I have to do this. Not a perfect solution, but the best I can come up with at the moment.

Hope this helps you out,
James

Hi,
This can be achieved by forcing Windows to ask for the username and password on the login screen. This can be done by:

Open Local Security Policy (secpol.msc in Run if you prefer) > Local Policies > Security Options > "Interactive Logon: Don't display last signed-in".

That should prompt you for a username and password during each logon. It is worth noting that this worked for me in build 1803 however the Policy name has changed with different versions of Windows 10. So your policy may be called something different if you try to set it. For example I believe the release build 1503 called it "Interactive Logon: Do not display last user name".
