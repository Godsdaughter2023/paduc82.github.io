davidspencer2012 say:

Changing the Excel default workbook and spreadsheet zoom to 125%

I have Office 2011 for Mac. How do you change the default characteristics of the Excel workbook (and spreadsheet when I add more sheets) so that it opens with a zoom set to 125% (not the usual 100%)? Thank you for your help. David
j.e. mcgimpsey say: By default, XL uses an internally generated default workbook, but you can substitute your own.

In your Startup folder (by default:

    HD:Applications:Microsoft Office 2011:Office:Startup:Excel:

but you can change it in Preferences) create a new workbook with all your default specifications (e.g., zoom, # of sheets, fonts/styles, etc). Save the workbook AS A TEMPLATE (.xltm) with the name "Workbook" (no quotes). In the Finder, navigate to that folder, select the file and choose Get Info from the right-/Ctrl-click popup menu, and delete the extension (.xltm) on the file. XL will now use that workbook as a default.

Create a one-worksheet workbook template in the same way and save it with the name "Sheet". Delete the extension in FInder, and that workbook's worksheet will be used for inserted worksheets.

john korchok say:

I've set up lots of templates similar to this. Here's a link to a sample document: Fixed Table

Please download it to try, Word Online doesn't display it correctly. This works well if all users are on Windows and at least Word 2007. If you have Mac or Word 2003 users, you'll want to use Legacy Form Fields or Doug's idea of a userform.

Tables are the way to create a Date/To/From block, with headings in a left column and fillin data on the right.

You'll see the the top line of the document and the table near the bottom is not editable, though you can pick a date or enter a name. The table can't be moved or resized. The rest of the document is editable. Here are the steps to create:

  1. In Word, open your template.
  2. Click at the top of the page, just to the left of the first return.
  3. Use Insert>Table and create a table. 
  4. Click inside the table, then select Table Tools>Layout>Properties.
  5. In the Text wrapping section, click on Around, then click on the Positioning button.
  6. For both Horizontal and Vertical, change Relative to: to Page. Set the Horizontal and Vertical positions to place it where you want on the page. OK out.
  7. In the left column of the table, enter the headings
  8. In the right column, place Content Controls from the Developer toolbar. It sounds like you already know how to do this.
  9. With the Restrict Editing pane open, check Allow only this type of editing in the document: and select No changes (Read only) from the dropdown.
  10. Select the body of the document except for the first carriage return, then check Everyone under Exceptions (optional).
  11. In turn, select the fillin area of each Content Control, then check Everyone.
  12. Finally, click on Yes, Start Enforcing Protection.

When you create the table in front of the first carriage return, it becomes anchored to that return. When the first carriage return becomes ineditable, so does the table anchored to it., even though it is moved to another position on the page.

Even when you don't restrict editing, floating tables are handy because a user can't accidentally click the border and drag one, like you can with a text box. They can only be positioned numerically through Table Properties or with the small handle that appears in the upper left corner of the table.
