If you don't have any of the specific examples, maybe you have found a "new" one. 

In Win 7 and 8 simply being logged in in an Admin userID no longer automatically gives you all access rights.  You MUST use "Run As Admin" to get additional install access rights.  That is why you are getting the prompt about allowing the app to make updates to your system.

When you get the program on Win 7 started using "Run as Admin", disable ALL addins (not just COM addins) using the following steps then shut down the application and try to start it normally

  1. Open the Options dialog from the File tab.
  2. Select the Add-Ins tab.
  3. Select COM add-ins in the Manage dropdown and then click Go.  (NOTE: Repeat for all types of addins)
  4. Check the check box next to the add-in(s) that you want to enable/disable in the COM Add-Ins menu.
  5. Click OK.

If after turning off all addins you can start the application 'normally' (not using "Run as Admin") then the problem is almost certainly one (or more) of the addins.  Slowly turn them on to find the culprit.
