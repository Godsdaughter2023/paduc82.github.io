Same issue on E72 and XM 5630 here. Other people experienced it also with N97.

you can read this info in http://discussion.forum.nokia.com/forum/showthread.php?t=192607

 

VoIP3.x registers a user, it writes a randomly translated username in the contact URI, i.e. aweofj293j2c0239jdwiofje0@nataddress instead of username@nataddress 

Ougoing calls are not an issue (when the client succeeds to register) howevver for incoming calls the server has to put this translatedusername in the request-URI of his INVITE. Some Registars do that, but yours and mine don't.

 

This interpretation of how SIP should work by Nokia was not implemented in prior versions of Nokia VoIP settings (v 2. and v 1 used on FP1 models such as E61, N95, etc) thus enabling incoming calls to work for all.

 

Now however it seems this is the new standard Nokia has chosen to go for with all new models thus leaving us with a choice to either :

1- have our service providers patch their registrars to make them work with this new Nokia implementation or

2- use alternative third party VoIP apps for our devices instead of the native/integrated Nokia SIP client. or

3- change SIP service provider to use ones that run Registrars compatible with this new implementation...

If you attach your new normal template to the documents, with the updatestyles on open option set, it should achieve what you require.

The following should do the job

    With ActiveDocument
        .UpdateStylesOnOpen = True
        .AttachedTemplate = "Normal"
        .UpdateStylesOnOpen = False
    End With

http://www.gmayor.com/installing_macro.htm   

If this achieves what you require, with a little modification you can use this as a custom process with http://www.gmayor.com/document_batch_processes.htm i.e.

Function ApplyTemplate(oDoc As Document) As Boolean
    On Error GoTo Err_Handler
    With oDoc
        .UpdateStylesOnOpen = True
        .AttachedTemplate = "Normal"
        .UpdateStylesOnOpen = False
    End With
    ApplyTemplate = True
lbl_Exit:
    Exit Function
Err_Handler:
    ApplyTemplate = False
    Resume lbl_Exit
End Function

john korchok say:

To extract, modify and replace sound files:

  1. Make a copy of the presentation. A separate folder is best for this. Add .zip as the file ending. Windows will ask if you want to change the file ending, choose Yes.
  2. Right-click on the file and expand it.
  3. You'll see a variety of folders that are the contents of the presentation. Open ppt>media.
  4. Narration files will be called media1.wav. Additional files will have the number incremented i.e. media2.wav, etc.
  5. Open the file in a sound editor. You can download the free Audacity sound editor from this link: Audacity Download
  6. Assuming you are using Audacity, choose Effect>Amplify. Adjust the Amplification by sliding the slider to the right to make it louder. Use the Preview button to test if it's loud enough. You may have to check the Allow clipping option. When you have a good result, click on OK.
  7. Choose File>Export Audio and choose the WAV format. Keep the file name as it was originally i.e. media1.wav.
  8. Close Audacity.
  9. Ensure the the new sound file is in the same folder as the original and has the same name as the original.
  10. In Windows, select the _rels, docProps and ppt folder, plus the [Content_Types].xml file and rezip, changing the file ending back to the original (probably .pptx)
  11. Open the file in PowerPoint and test.
