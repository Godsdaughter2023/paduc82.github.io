bob jones aka: cybertaz say:

Office Proofing Tools are separate & distinct from Custom Dictionaries -- that's the point :-)

A Custom Dictionary can be designated as being any of the Languages listed there. What that means is that if you add words to a Custom Dictionary named, for example, MyGreekWords which is designated as Greek Language, Word will check text marked as Greek in the document against that dictionary list (assuming that the Custom Dictionary is active).

For example:

If you use the term moy in an English document it will be flagged as misspelled because no such word exists in the English language. However, if you select the word & use Tools> Language to tag it as being Greek the word will be ignored regardless of how it is spelled because Office doesn't have Greek proofing tools -- IOW, Mac Office doesn't speak Greek so it doesn't know whether the word is spelled incorrectly. [If the language is one for which Office does have the necessary proofing tools, that lexicon of the main dictionary will be used automatically -- IOW, the spell check recognizes tagged language on the fly & uses the appropriate proofing tools IF they are available.]

OTOH, let's say you have moy in your Greek Custom Dictionary and mistype it in a document as mey (which I'm assuming is not a valid Greek word and/or isn't in your Custom Greek Dictionary). If the word is tagged as Greek your active Greek Custom Dictionary will cause the spell check to flag it as incorrect & offer moy as a replacement.

a. user say:

Hello,

Thanks for posting your query to Microsoft forum. I will certainly help you with this.

By default Internet Explorer disables running unsigned ActiveX controls.

While this is an important security measure on the Internet Explorer browser, if you are sure that the website is a trusted one then you can temporarily disable this security feature to get the website run the required ActiveX and later on re-enable the security.

To disable this feature:

a.    Open internet explorer, click Tools – Internet Options.

b.    Click the Security tab – Custom Level button.

c.    Under ActiveX ensure the following are set to enabled

i.   Run ActiveX and plug-ins – Enabled

ii.   Download Signed ActiveX Control – Enabled

iii.    Script ActiveX controls marked as safe for scripting – Enabled

d.    Set the following to “Prompt ”.

i.   Download unsigned ActiveX Control – Prompt

ii.   Initialize and script ActiveX controls not mark as safe – Prompt

e.    Click OK and Apply and OK.

f.    Restart the browser & check if the issue persists.

Refer:

Protect yourself when you use ActiveX controls

Private Sub UserForm_Activate()
Me.Image1.Picture = LoadPicture("S:\Shops\Test Track - Eng\Weather data\10MinAvgWindSpeed.gif")
Me.Image2.Picture = LoadPicture("S:\Shops\Test Track - Eng\Weather data\Barometer.gif")
Me.Image3.Picture = LoadPicture("S:\Shops\Test Track - Eng\Weather data\OutsideTemp.gif")
Me.Image4.Picture = LoadPicture("S:\Shops\Test Track - Eng\Weather data\WindDirection.gif")
Me.Image5.Picture = LoadPicture("S:\Shops\Test Track - Eng\Weather data\WindSpeed.gif")

End Sub

andreas killer say:
Copy that sub in a regular module
Rename it as ReloadPictures, and make it public
Replace "Me" with the name of your userform (e.g "Userform1")
Add the line
  On Error Goto ExitPoint
at the top and
  Application.OnTime Now+TimeSerial(0,0,30), "ReloadPictures"
  ExitPoint:
at the bottom. Use this sub in your form

Private Sub UserForm_Activate()
  ReloadPictures
End Sub
