Method 1: Use a trusted location, or create an exempt location
  • In PowerPoint 2007, if you trust the file that you want to open, you can move the file to a trusted location to override the registry policy settings.

    For more information about how to create, to remove, or to change a trusted location for files, visit the following Microsoft Web site:
    http://office.microsoft.com/en-us/help/HA100319991033.aspx (http://office.microsoft.com/en-us/help/HA100319991033.aspx)
  • In PowerPoint 2003, there are no trusted locations. You can create an exempt location to override the registry policy settings. To create an exempt location, follow these steps:
    1. Exit PowerPoint 2003 or PowerPoint 2007.
    2. Click Start, click Run, type regedit in the Open box, and then click OK.
    3. Locate and then click one of the following registry subkeys:
      HKEY_CURRENT_USER\Software\Microsoft\Office\11.0\Common
      HKEY_CURRENT_USER\Software\Policies\Microsoft\Office\11.0\Common
    4. On the Edit menu, point to New, and then click Key.
    5. Type OICEExemptions for the name for the key.
    6. On the Edit menu, point to New, and then click String Value.
    7. Type a string name, and then press ENTER. For example, type ExemptDirectory.
    8. Right-click the string name that you typed in step 7, and then click Modify.
    9. In the Value data box, type the path of the directory that contains the file, and then click OK. For example, if your document is in the C:\My Documents folder, type C:\My Documents in the Value data box.

      Note You must create the folder. Any subfolders are not automatically exempted. For any additional folders that you would like to make exempt, repeat steps 6 to 9 by creating string values such as "ExemptDirectory1" and "ExemptDirectory2."
    10. On the File menu, click Exit to exit Registry Editor.
Method 2: Disable the restriction on opening documents of certain types in PowerPoint 2003

Disable the restriction by changing the value of the FileOpenBlock subkey.

Important This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. But if you can still unlock your account using a security code, then you still have a chance. Here's what I can suggest to you :

1. Rename your primary alias.

- When you rename your primary alias, you won't be using the usual email address to login. The hacker will get a message that the account does not exist, but you will still be able to login to it, since you just renamed it. Take note that renaming the account will affect other services, because they will still look for the old email address.

2. Replace all your security info

- By replacing this, you will be stopping the hacker from getting a security code, preventing them access to your Account Settings, and stop you from locking you out of it.

3. Turn on Two-Step Verification

- When you turn this security feature on, the hacker will be asked for a code when they try to sign in to your account. Even if they have the password, they won't be able to simply login without a code, so it's best that you do #2 first before trying #3, or it will be ineffective.

But if you don't actually have a way to get a code, then you won't be able to do any of those above, and it's best that you just Contact Support.
