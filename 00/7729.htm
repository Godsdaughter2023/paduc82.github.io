Column A contains the "Smith, Bob" names.

Column B contains the "Bob Smith" names.

Let's say that column C contains the formulas that convert "Smith, Bob" to "Bob Smith".

In D1, enter the following formula: =MATCH(B1,$C$1:$C$100,0)

Fill down to D100 (or as far as needed).

D1 will either contain the row number of the cell in column A where the equivalent of the name in B1 can be found, or #N/A if there is no match.

hansv mvp say:

You could enter the following formula in a third column:

=MID(A1,FIND(",",A1)+2,100)&" "&LEFT(A1,FIND(",",A1)-1)

and fill it down as far as needed. This will reverse "Smith, Bob" to "Bob Smith". You can use this to match with column B using MATCH or VLOOKUP.

ruwim.b say:

The .json file is constructed to show everything what is saved on the Cloud server, but in a form that is not easy readable. The data saved on the Cloud for a particular conversation may e.g. contain no chats because they are too old or have been deleted. In this case, the viewer (index.html) will not show this conversation at all.

 

If you open the messages.json file in a text editor (e.g. Notepad), you will see that what is available for a particular conversation follows after the "MessageList":[ tag.If nothing is available for this particular conversation, you will instead see "MessageList":[].

 

The messages.json file contains also info on all rejected/blocked contact invitations, on members added/removed from a group chat, the status change of a group chat and so on. This kind of info is ignored by the index.html viewer.

 

In general, without knowing the exact structure of the .json file, it is not possible to extract the readable information. This is the job of a viewer/parser. The index.html viewer provides some basic parsing, but is still showing a lot of data that are not relevant for the end-user. Perhaps in the feature, a better parser/viewer will be available.

7-Zip is a 3rd-party application and so doesn't belong to Microsoft, thus it's not the responsibility of "windows" to maintain it, it's yours.  Since it's installed as the older executable type of program and not a new Universal Windows app from the Windows Store, Windows won't maintain it.

If you don't wish to be responsible for maintaining a 3rd-party app, then don't install it in the first place.  If you don't believe you did, then either some "friend" may have done it or possibly another app that carried it along as either an add-on or with malicious intent, such as a Potentially Unwanted Program (PUP).

If you truly don't want to maintain it, then uninstall it, since unless you actually use its added features the built-in compressed file capabilities in Windows are typically sufficient for most PC users.
