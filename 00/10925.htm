barb way say:

There is a Page settings that will cause all shapes to move away from a dropped/copied/pasted shape.  Look on the Page settings (right-click on the page, choose Page Setup), under the 'Layout & Routing' tab.  The setting is called "Move other shapes away on drop".  This is a page-level setting, so you may need to change it on multiple pages in your drawings.

This setting can be pre-set in a template, so that if you have more than one drawing that is behaving this way, you might want to check the template or drawing that was used to create them.  As far as I know, none of the Visio built-in templates have this setting on by default.

By the way - this is not a new setting, it has been around since at least Visio 2000.

dale howard [mvp] say:

Donald --

The Remove Task feature does NOT work the way you assume.  This feature is used by a team member to REJECT a task assigned to him/her, and the rejection must be approved by the project manager.  If the rejection is approved, the team member is removed from the task in the Microsoft Project schedule.

The Help article you reference is missing one important fact about the Remove Task feature. The system will not allow a team member to remove a task that is already in progress or completed.  So, you cannot and should not attempt to use this feature to remove completed tasks from the Timesheet page.  In fact, when a task is completed, the Timesheet page will no longer display the task, and the Tasks page will move the task to the Completed planning window section at the bottom of the page.

Hope this helps.
