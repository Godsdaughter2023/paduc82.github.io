daniel pineault say:

You don't edit/work with an mdw file directly.  It is used to secure a database file.  So you need to open the database (calling the mdw so security is applied) and through the database then you'll be able to open the security forms to add users and edit permissions, etc.

Using command line switches you'd open the db using a command like:

"C:\Program Files\Microsoft Office\OfficeXX\MSACCESS.EXE" "C:\...\...\YourDb.mdb" /WRKGRP "C:\...\...\YourMDW.mdw"

Also note, Admin rights on the network will not work on a secured db.  They are unrelated.  You will need the username and password of an Admin database user account otherwise you will not be able to make any changes.  That said, if your boss did it previously, then I'm assuming his login credentials (for the database, not network) have the necessary permissions to make the changes.

scottgem say:

As Daniel pointed out, you have to open the Access app using command line switches. Unless you are using a single System.mdw file for all apps, ULS secured apps are generally opened via shortcut that has the command line switches in the Target.

Once opened you can then run the Security wizard (I'm not sure where it is in 2013). to change permissions.

rob koch say:

Though the antimalware application used has some importance to protection from PUPs (termed Potentially Unwanted Software, or PUS by Microsoft), in general the browser security itself is far more important.  The key issue here is the browser's ability to protect from socially engineered malware (SEM).

A quick review of the first two documents at the following NSS Labs URL titled; "Browser Security Comparative Analysis Report - Socially Engineered Malware" and "Evolutions in Browser Security" both show quite clearly that the Microsoft Internet Explorer browser and it's associated SmartScreen Filter security component are far more effective than that of any other browser.

https://www.nsslabs.com/reports/categories/test-reports/browser-security

Though the Chrome browser appears to be relatively more effective at SEM protection than the almost completely ineffective Firefox and Safari (Apple) browsers, it's also made clear that the protection technology used by Chrome for SEM is highly prone to false positive detections, reducing its true effectiveness even further.

So since you appear to be having difficulties remaining safe from such SEM attacks, the most important step you could probably take is to stop using these highly insecure 3rd-party browsers.
