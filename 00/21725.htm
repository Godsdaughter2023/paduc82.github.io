How to FindRecord in 2 fields rather than 1 or all - Microsoft Community

phillip huy say:

How to FindRecord in 2 fields rather than 1 or all

I have a form that references a member data query.  The query contains both the field "LASTNAME" and the field
"SPOUSELASTNAME". 

On my form, I have an unbound textbox named "Lname" which is referenced in the function below.  The function below is referenced in the click and getfocus properties of my 'Find' button.  Currently, my function only searches the "LASTNAME" field of each record but I want to make it search both and ONLY the "LASTNAME" and "SPOUSELASTNAME" fields in the query.  I understand that the 'Onlycurrentfield' parameter of the findrecord method can be set to either acCurrent or acAll but I wish to use neither. 

Is there anyway to specify which fields to search in or is my only option to create another query and concatenate both fields into one?  Can I set a variable to  "LASTNAME" and "SPOUSELASTNAME" and use that in my argument?  If anyone has any idea of how to change my code to search both fields, please show me.  If not, would my concatenation idea work?


Function Findname_LName()
On Error GoTo Findname_LName_Err

    If (Eval("[LName] Is Null")) Then
        Exit Function
    End If
    DoCmd.GoToControl "[LASTNAME]"
    DoCmd.FindRecord Forms!Owners!LName, acAnywhere, False, , False, acCurrent, True
    DoCmd.GoToControl "[LName]"


Findname_LName_Exit:
    Exit Function

Findname_LName_Err:
    MsgBox Error$
    Resume Findname_LName_Exit

End Function





ken sheridan say: All I can say is that I've tested the code and it works as I described.  What it does is firstly try to find the next matching record in the form's recordsetclone.   If there is a match it goes to that record by synchronizing the form's bookmark with that of the recordsetclone. If there is no match, which could mean the only matches are before the recordsetclone's current record or there are no matches at all, it will try to find the first matching record in the recordsetclone.  If there is a match it goes to that record, if not it displays the message box.  This process is repeated each time the button is clicked while the value is still present in the LName control, so it continues to cycle through all the matches for however many times the button is clicked.

If you are experiencing different behaviour it suggests there is some other mechanism which is interfering with the process.  I notice that you mention putting code in the button's GotFocus event procedure.  Don't.  Put it only in the Click event procedure.

pamcaswell say: When you select  the numbering on Heading 7, does a grey background show for all Heading 7 "numbers"? If not, then you have more than one list at that level.  You can sometimes fix the problem by clicking on each heading 7 number (in order of occurrence), right clicking on it then selecting continue from previous. Another quick method is to select all Heading 7s and reapply the style. The last multilevel list (or the last List Style) linked to the heading is in the heading settings and will be applied to all instances, in so doing make one multilevel list.

How could this have happened?   You say you set the the start at value for numbering level 7 to A.  If the cursor is on the first occurrence of a heading level in a document (or, preferably, on the first number level 1 paragraph such as Heading 1), the value shown in the dialog would be 1 (or A in this case).  Any other number means the cursor in in a later occurrence of the style or that a start value other than 1 has been set.    Changing the start at number in the middle of a list splits the list.


By the way, if you used a calculation within the page field to correct the  appendix label "number" on the page, the TOC can't see or display calculated page numbers.
