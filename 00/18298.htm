I agree that what is called Native Boot -- running your same physical PC, but using the VHDX file as an optional Windows OS (dual-boot) could be a better test for you than Hyper-V, as all of your physical hardware will remain in use. The only issue with a Native Boot setup is that you cannot upgrade it via Windows Update as each new build is released. You would have to create install media to upgrade with.

You could also set aside a small partition on your physical hard disk and apply your image to that partition before upgrading it to the Insider's build. This would allow for subsequent build updates via Windows Update. So the main advantage of Native Boot is that you can have a dynamically expanding VHDX file, which only uses as little space as necessary, rather than dedicating specific amount of disk space to an Insiders partition.
