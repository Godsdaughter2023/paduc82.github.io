  1. On the phone, click on “ActiveSync” under Start menu or under Programs.

  5. Enter m.hotmail.com in the Server address field. Tap Next
  6. Select the data to sync. Hotmail supports all 4 data types via ActiveSync.
  7. The device will connect to Hotmail ActiveSync and perform an initial sync for the selected data.

Windows 10 Screws Up Publisher 2010

davidf2 say:

I believe what you are describing has do with a change in the fonts and has been documented since Win8 was introduced and continues with Win10. The 'new and improved' versions of some fonts that come with Win8 and Win10 render a slightly different size in Pub 2010 and newer which can look like the line spacing is off. Meanwhile to make life even more interesting Pub 2007 and older do not have the same problem. The only solution we have found thus far is to replace the problem fonts with older versions from a Win7 or older machine...or just run Pub 2007 on the Win8 and 10 machines.

Here is one thread that discusses the issue:

Reference: Fonts in Publisher 2010: http://answers.microsoft.com/en-us/office/forum/office_2010-publish/fonts-in-publisher-2010/9874dea3-4d6f-40f2-8772-2995f0bc6e5a - User discusses how text rendering problem is also tied to specific fonts and that you can fix the problem by installing older versions of the fonts.

If it were me and I wanted to experiment with replacing the Win10 version of a problem font with a Win7 (or older), then I would add .old to the Win10 font and not delete it from the machine. Then I would install the old version of the font. If the experiment did not work you can remove the newly installed Win7 font and remove the .old extension from the Win10 version of the font and go back to where you were.

If you don't have access to a Win7 or older machine from which you can get the old versions of the fonts you might want to read this reference:

Reference: How I restored my Windows 7 default fonts: http://www.withsteps.com/404/how-i-restored-my-windows-7-default-fonts.html - Gives instructions on how to install Windows 7 fonts and links to a folder containing the fonts. I have NOT tested this and would only install the fonts that you are using in the Publisher file that is not rendering correctly. In fact I again would reccomend renaming the Win8 or Win10 font file with ',old' extension and not delete it so you can go back to it in case installing the Win7 font causes a problem.

It is always best to develop a workbook in the earliest version with which it will be used.

That way you avoid such backward compatibility problems.

 

If for some reason you can't develop it in 2003, you can use VBA to set the .Interior.ColorIndex property of the cells to a number between 1 and 56 and you will get 2003-compatible colours.

 

The nice way to do it would be to have a userform with 56 boxes in the appropriate colours.
