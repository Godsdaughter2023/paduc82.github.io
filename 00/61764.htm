SOLVED

I solved this problem for myself and was able to install Windows 10.  

To cut to the chase, I had to boot to a CMD prompt then run Chkdsk on the System Reserved partition.  This had developed one bad sector affecting a file there called bootstat.dat.  When the Windows 10 installer tried to make a copy of that file it failed a cyclic redundancy check causing the instalation to quit.  But Chkdsk fixed bootstat.dat.  The error is in the middle of the section of the log I quoted previously  

\\?\F:\Boot\BOOTSTAT.DAT dir copy to \\?\C:\$WINDOWS.~BT\Sources\Rollback\Boot\BOOTSTAT.DAT: Data error (cyclic redundancy check). [0x80070017]

If you are getting the same error message, here are some tips.

My installation failed just before Windows was about to restart for the first time.

I found a short file called BlueBox.log in the \Windows\Logs\MoSetup folder that had the error code (80004005) at the end.

I then searched in the (very large) Setupact.log folder for the same error code.  This file was somewhere in a (temporary) root folder called Panther but I forget the exact location.

The real error was in this log, a little way before the 80004005 error code appeared.
