vaughnhathaway say:

I actually completed a successful install of Access 2019 with Volume Licensing today.   Below are guidelines to help you if you are encountering issues.

The sequence for installing Access 2019 (this is similar for all MS Office products that use Click-to-Run for Volume Licensing):

1.    Create xml file that is used to download and install the software.  For our purposes here, we’ll call in InstallAccess2019.xml

2.    Run “setup /download InstallAccess2019.xml” at command level

3.    Run “setup /configure InstallAccess2019.xml”

For my purposes, I installed the 32 bit version of Access 2019.   The xml script I used was the following:

<Configuration>
   <Add OfficeClientEdition="32" Channel="PerpetualVL2019">
      <Product ID="Access2019Volume" 

               PIDKEY="xxxxx-xxxxx-xxxxx-xxxxx-xxxxx" >
         <Language ID="en-us" />
      </Product>

   </Add>  
  <Display Level="full"

           CompletionNotice=”yes”

           SuppressModal=”no”/>  

</Configuration>

 

Note that the Display Level  part is probably unnecessary because I think the settings I list are the defaults.   Using this Display level shows you the progress of the install.  In the earlier example that had Display Level set to “none”, it would just silently end with no error messages to help you identify what is wrong.

 

Please see https://docs.microsoft.com/en-us/previous-versions/office/office-2013-resource-kit/cc179195%28v%3doffice.15%29 for details about the config xml.

 

Please see https://support.microsoft.com/en-us/help/2842297/product-ids-that-are-supported-by-the-office-deployment-tool-for-click for a list of product IDs that are supported by the Office Deployment Tool.

 

Please do a search for Windows Command Prompt for a description of how to invoke it, and see https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/windows-commands for a description of what you can do in it.

how do I change footnote-reference font without changing footnote font

When I did a "select-all" inside the footer containing footnotes, it selected all footnotes plus all of the footnote reference numbers. Thus when I modified the font for footnotes, it also changed the footnote-reference which I did not want to do. How can I modify just footnote-reference numbers without effecting footnote fonts?
doug robbins - mvp office apps & services (word) say:

Try running a macro containing the following code:

 

Dim afn As Footnote
With ActiveDocument
    For Each afn In .Footnotes
        afn.Reference.Style = "Footnote Reference"
    Next afn
End With

doug robbins - mvp office apps & services (word) say:

The font for the Footnote References is normally dictated by the Footnote Reference Style whereas that for the text of the footnotes is dictated by the Footnote Text style.

 

What you have done is over-ridden those styles by the direct formatting that you have applied.

If you use select all to select the footnotes and their references, you can then use Ctrl+Space to remove that direct formatting and what you should then do is modify the Footnote Text style so that it is defined with the formatting that you have previously applied.
