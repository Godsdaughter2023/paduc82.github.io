karl timmermans (outlook mvp 2012-2018) say:
Quick overview of OST and PST files

#1 - PST files only used by POP3 Accts
#2 - OST files used for Imap and Outlook.com/Exchange accounts

Answer to your question

You would need to configure all your accounts as POP3 and use a single PST file.

The issue you've encountered with Office 2016 is that you used auto-discover (entering your email address/password) to configure your email accounts which defaults to them being IMAP accounts.

What you want to do is create a new Outlook profile using the Control Panel <Mail> app and manually configuring the accounts as POP3. Make sure that you configure each POP3 account to use the same PST file - the first of which would be the one your copying on an on-going basis.

In short, there's no reason why you can't continue working the way you have in the past - just comes down to how the email accounts are configured.

ken sheridan say: You can base a form on a multi-table query provided that the data being entered is into the referencing table in the relationships, e.g. a query based on an Invoices table joined to a Customers table, which in turn is joined to a Cities table, which is in turn joined to a States table would be legitimate.  This form would be for inserting data into the Invoices table, but the query includes the referenced Customers, Cities, and States tables in order to return data from non-key columns in those tables.  In the form data would be entered into the CustomerID foreign key column in Invoices via a combo box.  The values from the non-key columns in the referenced Customers, Cities, and States tables would then show automatically in bound text box controls in the form.

When creating an invoice via this form it would also be necessary to insert data into an InvoiceDetails tables, each line in which would represent one line item in the invoice.  For this the InvoiceDetails table would not be included in the query.  Instead a subform based on the InvoiceDetails table would be embedded in the parent form, linked to it on InvoiceID.  This subform would be in continuous forms view, and each line item would be inserted as a separate row (record) in the subform, i.e. the subform is reflecting the one-to-many relationship type between the Invoices and InvoiceDetails tables.

jay freedman say:

On computer 'X', open the macro editor (Alt+F11). In the Project pane on the left, expand the icon for the template that contains the macros (is that normal.dot?) and expand the Modules icon under that.

Right-click a module and choose Export File. In the dialog that opens, go to a folder where you can save the exported file (and where you'll be able to find it easily later). Then click the Save button. The file by default has the same name as the module, with the ".bas" extension added.

If there are more than one module in the Modules folder, repeat the export for each of them.

If the template also contains one or more userforms, expand the Forms icon and export each form. Each form is represented by two files, one with a ".frm" extension and one with a ".frx" extension.

If the template contains any class modules, expand the Class Modules icon and export them. The files this time will have a ".cls" extension.

Transfer all of the exported files to computer 'Y' and store them in a convenient folder. Open the macro editor. Right-click the icon of the template where you want to add the macros, and choose Import File. In the dialog that opens, go to the folder that contains the transferred files, select one, and click the Open button. Repeat for the other files. (If you import a .frm file, its companion .frx file is automatically imported at the same time.)

charles kenyon say:

Both Jay and Graham have far more experience with macros than do I.

Although documents can hold macros, the usual place to keep them is in templates.

Not overwriting the existing normal template is a good thing. If the normal template on the receiving computer already has recorded macros in it and you are transferring recorded macros you will have to rename the NewMacros module in one of your templates to avoid overwriting the existing recorded macros. Both export/import and the Organizer copy entire modules.

Other templates can hold macros to be used by all files. Such a template is called a Global Template. You can create a new template to hold your existing macros. Once you have the macros in that template, place it in your Word Startup Folder and have the macros available for use in all files.

Note that while macros in a global template other than normal can be used in other documents, they cannot be edited without actually opening the template. Macros in the normal template can be edited without directly opening the normal template.

You may want to take this opportunity to organize your macros into different modules named by purpose. The storage place for any recorded macro will be the module NewMacros. If you have many macros, this gets crowded and the name is not especially helpful.

If you are sharing your macros you may even want to have different sets in different templates. I don't know of any limit on the number of global templates you can have active. I generally have between 6 and 12 running on my system. Note though that there is some extra delay in starting Word when you have multiple global templates being loaded. (I am talking seconds, not minutes, but not milliseconds, either.)

Finally, if you have keyboard shortcuts assigned to your macros, those will not transfer using either method. There are options available if this is the case.

rich michaels - mvp (greatcirclelearning.com) say:
Is the whole document in landscape orientation or did you insert a section break and change the one section to landscape?  

I ask because to properly view this image, you will need to adjust the left, right and top margins, otherwise you will have to shrink it down considerably and you might then lose detail.  If this one image is part of a larger document, you'll probably not want the margins for the main body of the document as small as will be required for this single image.

Your description of what is occurring also sounds like you have inserted the image as a "floating" picture (a.k.a with wrapping text) and would account for it moving to a second page.  Assuming that you set the page margins properly, I think you'd be better served by placing the picture in the document as an inline image ... no wrapping or overlaid text.

Hi Tim,

You can block the domain of the email addresses from which you are receiving these emails which in your case is "pharma.ca" & "birdpublications.com".

To add the domain to blocked senders list follow the steps below.

Login to https://outlook.live.com/ through the web browser.

If you see Outlook Mail on the upper left corner, then follow the steps below: 

1. Click on the Gear (settings) icon and select Options.

2. On the left pane, expand Mail > Junk Mail and select Blocked Senders.

3. Now add the domain of the sender you wish to block.

If you see Outlook.com on the upper left corner, then follow the steps below to block email accounts. 

1. Click on the Gear (settings) icon and select Options.

2. Under Preventing Junk email, click on Safe and blocked senders.

4. Type the domain and select  Add to list >>

marco_mi. say:

Hi fabio_10,

 

Let me help you with your concern regarding spam emails. Please do the following steps to minimize or to stop receiving unwanted emails.  

 

A. Scheduled Sweep - automatically lets you keep only the latest mail from a sender or have their emails be moved to deleted folder after 30, 60, or 90 days.

1. Open the email you would like to filter.
2. Click Scheduled Clean-up beside sender's name
3. On the pop-up window, choose your preferred option
4. Click OK.

B. Rules - we can create a rule that can block a specific word from the subject of the spam email.

1. Sign in to your Microsoft account. Go to Inbox.
2. Click on Gear icon then More mail options... at the upper right corner of the page.
3. Under the Customizing Outlook section, click on Rules for sorting new messages.
4. Click on the New button.
5. Under Step 1: Which messages do you want this rule to apply to, 
    a. First drop-down menu - choose Subject
    b. Second drop-down menu - choose contains
    c. Third field - input the word you would like to block
6. Under Step 2: What action do you want to apply, click on Delete these messages
7. Click Save button.

For more information, please click here.

C. Sweep - This will allow all received or even future emails coming from the specific sender be deleted.

1. Sign in to your Microsoft account. Go to Inbox.
2. Check the specific sender you would like to filter.
3. On the upper part, click Sweep
4. Choose Delete all from
5. On the pop-up box, just click Delete all.

Note: You can also block future messages from that sender by checking the box beside Also block future messages.

 

 

We also recommend that you follow these guidelines to help reduce junk e-mail:

 

* Don't reply to junk e-mail, even to ask to be removed from the sender's mailing list. The sender may simply use that response to confirm that your e-mail address is valid, and continue to send you junk e-mail. To report the message as junk e-mail, select the message, click Junk.

 

* To report the e-mail message as a suspected phishing scam (Falsely claiming to be a legitimate business to send you to a fake website or scam you into giving out private information.), click Mark as - Phishing scam.

 

* Remove your e-mail address from any web page, newsgroup, or bulletin board available on the Internet.

 

* To remove your e-mail address from newsletters or mailing lists that are on your safe senders list, you can click Unsubscribe. Microsoft notifies the people who created the newsletter to stop sending it to you. 

 

Note: If you receive junk e-mail that looks as if it was sent from your Outlook.com account, someone may have spoofed (Sending messages with fake or stolen account information) your account information.
