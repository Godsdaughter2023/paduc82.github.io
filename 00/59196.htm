j w stuart say:

It could be a sub-process or application that's running in the background
and taking all the CPU resources.

To find and display what could be the problem try Process Explorer:
http://technet.microsoft.com/en-us/sysinternals/bb896653.aspx
SysInternals Forums: http://forum.sysinternals.com/

Worth Reading: The Case of the System Process CPU Spikes
http://blogs.technet.com/markrussinovich/archive/2008/04/07/3031251.aspx

Once you have Process Explorer installed and running:
In the taskbar select View and check:
'Show Process Tree' and the 'Show Lower Pane' options.
Move your mouse cursor over any column in the right hand pane and
right click and check the following boxes:
'Command Line' and 'Version'.
Then expand the process named 'Explorer' (click on the + sign)
In the column on the left named 'CPU', look for any high CPU usage.
Next click on the CPU column to sort the processes by %CPU usage
(Highest to Lowest).

You can also mouse over and check any spike or high/low point in the CPU graph.

Move the mouse cursor over any process,
you should see a popup with some detailed info.
Then mouse over the process that's using most or all the CPU %.
Then click on that process to highlight it,
Now that  it's highlighted, right click and from the options listed select:
'Search Online'.
This should display what out there on the web about that process.
You can also double click on any process to open up a more detailed
'Properties' window.
Note: some entries like Explorer, System/Services, and Svchost
may need to be expanded to show the detail (sub processes),
in this case click on the + located to the left of the entry.

An alternate method when using Process Explorer
is to double click on the Graph just below the Menu bar.
This will open the 'System Information' window, which has a larger display
of all three graphs. Move your mouse over any spike in the
CPU Usage graph to see what process/application or service is the cause
of the spike.

Another tool available is: RAMMap
http://technet.microsoft.com/en-us/sysinternals/ff700229.aspx

Hi
Thanks for contacting Microsoft Community.

Normally, C:\ partition is supposed to have at least 20 GB or 10% free space for system operations like creating restore points and for paging.

Microsoft Visual C++ 2005 Redistributable KB2467175 is a security update for Microsoft Visual C++ 2005. Microsoft Visual C++ 2005 is a runtime library that many programs depend on. If you are planning to remove the installation files (deleting from temporary location), it is safe. But, if you are planning to remove it through Programs and Features, programs depending on this library will fail to run after removal.

Shrink the volume through command prompt

===========================

1. Click the Start button, then in the "Start Search" box, type CMD .

2. In the list above, under "Programs" right-click on CMD that appears above and choose "Run as Administrator" and click "Continue".

3. In the open Window, type the following commands one by one and press Enter at the end of each command:

 

C:

CD %windir%\system32

diskpart

list volume

 

Note: After running the "list volume" command, it will show all partition.

 

select volume <volume number>

 

Note: Please replace the "<volume number>" to the volume number that you want to shrink . For example,  Suppose If you want to shrink C and if the volume number of drive C is 2, we need run the command "select volume 2"

 

shrink desired=<N>

 

Note: Please replace the "<N>" to the disk space you would like to shrink. After run the command, it will shrink <N> MB space on drive C. (1GB=1024MB)

 

After the process finish the command, please restart the computer and check whether we have shrunk the drive successfully.


If problem persist Shrink the volume in Safe Mode

======================

1. Restart your computer and start pressing the F8 key on your keyboard. On a computer that is configured for booting to multiple operating systems, you can press the F8 key when you see the boot menu.

2. When the Windows Advanced Options menu appears, select Safe Mode, and then press Enter.

3. Log onto Windows by using the Administrator account or any user account with the Administrator privileges.

 After booting the system into Safe Mode, please try to shrink the volume again in Safe Mode.

Step 1: Disk Defrag

===================

1. Click the Start Button, click Computer.

2. Right-click on the C: drive and choose Properties.

3. Switch to the Tool tab and click on the "Defragment Now" button. Click Continue if necessary.

4. Click on Defragment Now.

 

Note: Please note that it may take some time to complete the disk defragment. Please wait until the action finishes.

 

After that, please try to shrink the C: drive again.


Step 2 : Transfer virtual memory

=================================

1. Click the Start Button, right-click "Computer", and select "Properties".

2. On the left pane, click "Advanced system settings".

3. Switch to the "Advanced" tab, and under "Performance", click "Settings".

4. On the pop-up "Performance Options", switch to "Advanced" tab, and under Virtual memory, click "Change... "

5. Clear the "Automatically manage paging file size for all drives” check box.

6. Under "Drive [Volume Label] ", select C: drive, click "No paging file", click "Set" ( Assuming you are trying to shrink C , If not select appropriate drive ).

Note: When prompted, please click Yes to continue.

7. If you have other partition like D follow these steps

Select D: drive, click “System managed size”, and click “Set”.
 
8. Click “OK” to apply the settings.

9. Please restart the computer to shrink C: drive again via Computer Management to check the issue.



Step 3 : Disable system restore and Hibernation

======================================

Please disable the System Restore feature to clear the storage folder:

  1.Click Start. In the search bar, type in "sysdm.cpl" (without the quotation marks) and press Enter.

2.Choose the "System Protection" tab and uncheck all items in “Create restore point automatically on the selected disks"
3.Click "OK".
