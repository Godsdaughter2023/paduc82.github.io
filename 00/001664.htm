lance cai msft say:

Hi DDPIACP,

I'd like to confirm if you have deployed an Microsoft Exchange server environment and all the meeting request is accepted automatically. 

If this is the case, it is a default behavior. 

It occurs because AddOrganizerToSubject and DeleteSubject are set to True.

To view these value for the Resource mailbox, run the one of the following cmdlets:

  • For Exchange Server 2016, Exchange Server 2013 or Exchange Server 2010 
    Get-CalendarProcessing -Identity <RESOURCEMAILBOX> | FL
  • For Exchange Server 2007 
    Get-MailboxCalendarSettings -identity <RESOURCEMAILBOX> |FL


To resolve this issue, follow these steps:
  1. Open the Exchange Management Shell.
  2. Run the one of following cmdlets:
  • For Exchange Server 2016, Exchange Server 2013 or Exchange Server 2010 
    Set-CalendarProcessing -Identity <RESOURCEMAILBOX> -DeleteSubject $False -AddOrganizerToSubject $False
  • For Exchange Server 2007 
    Set-MailboxCalendarSettings -Identity <RESOURCEMAILBOX> -AutomateProcessing AutoAccept -AddOrganizerToSubject $False -DeleteSubject $False

This all depends on the setting of your "Wrapping" property in Format>Picture>Layout.

Read as much as you have time for here, so you understand the concepts:
http://word.mvps.org/faqs/DrawingGraphics.htm

That was all written around Word 97/2000, so it's a fairly good fit for Word 2008, things haven't changed much there.

Two rules:

1)  Set all graphics "Inline with text" if you can.  That way, they cannot wander around, they become part of the paragraph you place them in, and you can move them/space them by adjusting the paragraph formatting.

2)  If you must use floating graphics, you need to understand that every floating graphic is anchored to a paragraph on the page that it appears on.  If you make changes that move that paragraph to a different page, the graphic will go too, and odd things may result.

Come back with any further questions you have.

nathan li msft say:

Hi Science Student,

Whether your future lab members could access your files depends on your Office 365 admin. If your Office 365 admin would delete your account after you leave the lab, your admin would have 30 days by default to access and download your files in your OneDrive for Business. After the this retention period, your OneDrive for Business will be sent to site collection recycle bin, where it is kept for 93 days.

My suggestion is that you may store your notebooks in a document library of a SharePoint site, thus the notebooks won't be affected even when your OneDrive for Business is deleted.

Nathan
