jay freedman say:

When you click "Insert Index", you see the Index dialog box. The print preview isn't showing you an actual index, just a representation of the format. For example, by default the index will be in two columns, and the page numbers will appear after the entry text, with a comma separator. If you prefer the numbers to appear at the right margin of the column, with or without a line of dots (a "leader"), set the options below the preview. If you want one column or three or more, use the options on the right.

Finally, click the OK button to insert the actual index in the document. However, if you've already inserted \f switches into all of the XE fields in the document, the "index" will initially appear as an error message that says "No index entries found." (because the index is looking only for XE fields that don't contain any \f switch).

Now you can insert the \f switch as Charles mentioned: Right-click anywhere in the index (or its error message) and click Toggle Field Codes. You'll see the field code, which looks something like this:

{ INDEX \c "2" \z "1033" }

Click just to the left of the closing brace and insert the \f switch and its identifier (for example, \f"a") with a space before the backslash. Then right-click the field code and click Update Field. If you still see the field code, press Alt+F9.

Function ParagraphHierachy(ByVal P As Paragraph) As String
  Dim This As String
  ParagraphHierachy = Trim(Replace(P.Range.Text, vbCr, ""))
  Do While P.Format.OutlineLevel <> wdOutlineLevelBodyText
    Set P = P.Previous
    This = Trim(Replace(P.Range.Text, vbCr, ""))
    ParagraphHierachy = This & "\" & ParagraphHierachy
  Loop
  This = P.Parent.Name
  ParagraphHierachy = This & "\" & ParagraphHierachy
End Function

john - project say:
Leo,
There are various ways to handle level-of-effort (LOE) tasks.

One is to use a hammock task. For details on how to set that up, see FAQ 19 - Hammock tasks, on the MVP website at, http://project.mvps.org/faqs.htm. However since hammock tasks require the use of paste links, which can be subject to corruption, there are probably other ways to handle LOE.

Another method is to set up periodic tasks, say a month long each, that represent LOE for each month. This can either be done with the recurring task option or manually. Monthly effort makes it easy to track and change as necessary.

An extension of the monthly method is a single (or multiple) tasks manually entered with a duration roughly equal to the total project duration.
