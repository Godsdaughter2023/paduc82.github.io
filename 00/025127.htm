bernie deitrick say:

There is no such thing as saving an individual worksheet.

BUT - You can track when there are changes to a specific worksheet - for example, this code:

1) Copy the code.
2) Right-Click the sheet tab of interest (perhaps, "HoganPplSoft_Users").
3) Select "View Code"
4) Paste the code into the window that appears.
5) Make changes as the the cell address as needed

Private Sub Worksheet_Change(ByVal Target As Range)
     Application.EnableEvents = False
     Range("A4").Value = Date
     Application.EnableEvents = True
End Sub

Note that this will update the date in cell A4 of the sheet that you right-clicked, and only when a change occurs on that one sheet: if the workbook is not saved afterwards, then the date change will not be saved.  You can also write code that will perform the same action on ANY worksheet - if you always want to record the last change date into cell A4 of every worksheet, then you could use this workbook level event (goes into the ThisWorkbook codemodule):

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
     Application.EnableEvents = False
     Sh.Range("A4").Value = Date
     Application.EnableEvents = True
End Sub

bernie deitrick say:

Your code 1) is the way to go. That code, though, needs to go into the codemodule of the ThisWorkbook object - find your workbook in the project explorer window, double-click the ThisWorkbook object, and paste that code into the window that appears.  And you need to save the workbook as a macro-enabled .xlsm file.

That kind of code  - event code, specifically workbook level event code - cannot be run like a regular macro, because it is tied to the workbook being saved. If you want to test it, use this code in a regular codemodule:

Sub TestMacro()
    Sheets("HoganPplSoft_Users").Range("A4").Value = Date
End Sub

And if you want better resolution of when the file was modified, use Now instead of Date, and format the cell for date and time.
