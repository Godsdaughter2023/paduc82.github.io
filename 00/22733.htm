To add text to a shape:

  • The orderly fashion: Control-Click (or Right-Click) and choose Add text, then start typing (if you are using the Clipboard, just Paste)
  • The immediate fashion: Click on the shape and start typing (or simply Paste)

Drag-and-drop will only work once you already have some text in the shape. Go figure why.

Shapes will repel each other if even one of them does not allow overlap. To control this:

  • Select the shape
  • In the main menu bar, select Format > Shape
  • In the list, select Layout
  • Click on the Advanced button
  • Select/Deselect "Allow overlap" as required

If a given shape does not allow overlap, no other shape will be able to come too close to it.

palcouk say:

If you use a single data file for multiple accounts, you would probably want to create subfolders off the Inbox/Sent folders in order to categorise/sort mail, which would require multiple rules or manually moving

A separate data file for each mail account ensures that there are separations and no rules required

diane poremsky mvp (slipstick.com) say:

 

If you use imap, outlook.com, or exchange server you will have multiple pst files. You can't avoid it - they are server based accounts and sync to outlook. Only POP3 can use one data file.

 

If you use multiple data files it keeps the personal and mail separate -you won't accidently reply or forward the message using the wrong account. If you need to remove an account, you can easily remove the mail by removing the data file. The data files are smaller and won't need to be archived. You may find you don't need to use rules to stay organized.

 

If you deliver all to one data file, everything is thrown together in one folder and you are more likely to need rules and custom views to keep things sorted.

 

As far as which is better, it really depends on your needs. Some people like separate folder sets, some don't. If you don't get much email, it makes more sense to use one data file.
