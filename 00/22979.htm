Use batch migration to migrate legacy public folders to Office 365 and Exchange Online

While working in Word on my new SurfacetPro3, I hit something that put me in Overstrike mode... that's when I discovered there's no "Insert" key.  Nor can I find any combinations for "Insert" or "Overstrike". How do I stop this???
jay freedman say:

Here are several alternatives. Pick your preference.

  • Go to File > Options > Advanced. In the Editing Options group, uncheck the "Use overtype mode" box.
  • Right-click the status bar at the bottom of the Word window. In the popup menu, click the Overtype entry to put a check mark next to it. That causes either "INSERT" or "OVERTYPE" to appear on the status bar, and clicking it will toggle from one to the other.
  • There is no built-in shortcut key (other than the Insert key), but you can define one. Go to File > Options > Customize Ribbon, and click the Customize button in the bottom left of the dialog. In the Customize Keyboard dialog, scroll down the Categories list and click All Commands near the bottom. Then scroll down the Commands list and click Overtype. Click in the "Press new shortcut key" box and type the key combination that you want; then click the Assign button, and finally close both dialogs.

fab067 say:

MS Project 2010 Conversion to MS Project 2000

We've recently converted from Project 2000 to Project 2010.  The Integrated Master Schedules that my group has produced have already been saved to the 2010 formats.  Corporate would like us to go back to 2000 for unknown reasons.

john - project say:

fab067,

Yup, that sounds like something corporate would do, and don't you just love it?

You mention Integrated Master Schedules which implies more than one. An IMS can be defined different ways but normally it means that the Cost Account Manger files (detail working schedules) are inserted into a top level master (via Insert/Project). That fact that you have multiple master files complicates the issue, which I am willing to bet is related to corruption due to all the jumping back and forth between versions.

The best scenario might be to revert back to the last saved files from Project 2000 and then go forward from there. I assume you did periodic backups, right?

Between Project 2000 and Project 2010 major changes were made to project file structure, not just added features but actual file structure. In that sequence of versions Project 2000 and Project 2003 are the most compatible (i.e. added features but no major file structure changes, at least as far as I know). You might have the most success if you can get all the subproject files opened cleanly in Project 2003 format. Once that is accomplished, then you can try taking the step back to Project 2000. I wouldn't try to make the conversion at the master level - work only at the individual file level. If some or all of the files still have problems, try clearing corruption by going to our MVP website at, http://www.mvps.org/project/faqs.htm and using one or more of the methods described in FAQ 43 - Clearing project file corruption and/or bloat. Once all the individual files are up and running, then create new master files as required.

John
