Hello Rob,

 

I did some quick checking and I'm not seeing a difference in how the packaging wizard handles the mdw file. I have found the following links explaining a possible method.

Using the Default Start Menu Items

http://msdn.microsoft.com/en-us/library/aa139982(v=office.10).aspx

http://social.msdn.microsoft.com/Forums/en/sbappdev/thread/7a7d69ea-c067-438b-9539-295215faf804

 

Secondly, the packaging wizard should create the trusted location registry keys for you, but there is a bug for Access 2010 that's still under investigation where it creates the trusted location under the 12.0 hive instead of 14.0. Until that's corrected, you can specify the registry key in the additional keys section. I took the following info from another forum post.

Root: Current User

Key: Software\Microsoft\Office\14.0\Access\Security\Trusted Locations\[Name]

Name: Path

Value: C:\Users\[UserName]\AppData\Local\[Folder]\

Substitute a name for [Name], the user’s name as it appears under the C:\Users or C:\Documents and Settings folder for [UserName], and the correct folder name for [Folder].

This will work for the specified user, but not for multiple users since the key being modified resides in each individual’s HKEY_CURRENT_USER location. One work around is to add the desired trusted location(s) to the users’ registry through a group policy at logon.
