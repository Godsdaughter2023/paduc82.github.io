dale howard [mvp] say:

Sneha --

To get an accurate picture of the work load for any resource, you really do need to schedule the tasks in some manner.  Rather than manually typing Start dates for the tasks, if the tasks have a dependency relationship with one another, then I would recommend you specify task dependencies (such as Finish-to-Start) on the four tasks.  This would then allow you to see if the resource is working at capacity or if the resource is overallocated.  Hope this helps.

trevor rabey say:

I don't know what you mean by "a milestone of 10 days" since a milestone is usually understood to be an event with zero duration.

If you have 4 tasks which have a cumulative duration of 13 days, there are many ways that 4 numbers can add up to 13, eg 1,1,1,10 or 1,1,2,9 or 1,1,3,8 and so on and so on. You would need to be specific about the duration for each task.

If you have 4 tasks and they all have the same resource assigned to them, and if all of the tasks are not related to each other as predecessors and successors, and are scheduled to start asap, then your resource will be over-allocated, but where and by how much will depend on the durations of the 4 tasks.

In any event, resource leveling will delay tasks until the over-allocation is resolved (ie disappears) and MSP does this very well. Go to Resource, Leveling Options and go from there.

john w. vinson say:

I'm not sure this is the only circumstance when this happens, but in my experience you'll get prompted for each table when the database has links to more than one backend location. That is, if you have 39 tables in one SQL instance, and two in another instance or in a backend Access database, you'll get this prompt.

One (drastic) solution is to DELETE all the table links in the frontend, and use External Data... Link to create fresh links to the backend. This lets you select all the tables and link them as a block. Another thing that might help is to select only those tables that are in the same backend in the Linked Table Manager.

scottgem say:
If you do a SELECT ALL in Linked Table Manager and there is more than one source location, then you will be prompted for each table. However, if you only select the linked tables from ONE source at a time, you only have to select the source once.

At least that's been my experience.

Saving Word document bar on bottem of page changed from green to blank - Microsoft Community

honeybludger say:

Saving Word document bar on bottem of page changed from green to blank with tiny red cross at the end.

I have a new pc, with Windows 7. On it installed a Free Trail of Office 2007. I purched my own Office 2007, installed it. Worked on a Word document.  Suddenly the automatic save function that is indicated by a green bar on bottom of the document, changed to a blank bar with a tiny red cross at the end.   I am not sure if my documents are saved corretly.

beth melton say:

Thanks for the description of the green bar - it helped us better understand what you're referring to. :-)

What you're seeing is normal. The red cross (or little circle with a white x) indicates you have Background Saves enabled. If this option isn't enabled then you won't see the cross at the end of the progress bar. If all you are seeing is the cross and you don't see any green appear in the progress bar then that's okay too. It just means your save is occurring so fast the green color doesn't have a chance to display.

If you want to turn off background saves:

1.  Click the Office Button

2.  Click Word Options (below the recent file list)

3.  Click Advanced

4.  Locate the section titled Save (near the bottom)

5.  Deselect Allow background saves.

However, depending on how fast your changes are saved if you disable background saves then you may not see anything in the status bar when you save your document. If you're using the status bar to help verify your document is indeed being saved then you might want to leave background saves enabled since you'll at least be able to see the red cross for verification. Again, don't worry if you don't see any green, it just means it's not taking very long to save your document.

~Beth Melton Microsoft Office MVP
