entegy say:
Accented characters are a tricky situation. The reason why the SMS limit is halved is because to use an accented character, the text ends up being sent in Unicode, which take up two bytes per character and not plain text, which is one byte per character. Accented characters are outside the normal character set and it seems older phones either had an option to turn off accented characters (which would then map the accented character to it's "normal" letter: ã becomes a) or just hid this fact.

hal hostetler - mvp-outlook say:

With regards to the crop tool, this is designed to be used with an older Outlook 97-2002 ANSI .PST file. It is NOT to be used an a native Outlook 2003-2010 Unicode .PST file. By the fact that your original file size was 3.5 Gb and it was functioning correctly, I have STRONG reason to believe you have the newer Unicode type file which has an upper limit of 20 Gb. Please verify which type .PST file you have and DO NOT RUN the crop tool if it is the newer Unicode variety. A Unicode .PST file size of 3.5 Gb is WELL within the size limits for this type file, compaction is not required by either Outlook 2003 or the file.  If you must compact it due to overall disk space considerations, have a look at this thread:

'Compact Now' is not reducing after clearning the 'delete folder’
