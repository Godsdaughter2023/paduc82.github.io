bill.p. say: Hi Shawn, I'm Graeme, an independent adviser here to help.

One way to disable the trackpad would be to use Device Manager. Here are the steps you can follow to do this:

shawn potter say:

Thanks for the tip! I tried that but when I rebooted the computer the touchpad device had been re-installed.

However, after a lot of investigation I've found a workaround in case other people read this thread.

It turns out my laptop (MSI) runs a synaptic driver for the touchpad (a very common driver it seems). When Windows updated, it installed a newer version of the driver (19.5.31.11 [2018-08-17]) that no longer seems to show the "Device Settings" tab for the mouse properties in the Control Panel. This means the "Disable internal pointing device when external USB pointing device is attached" setting was no longer available.

However, I was able to install the previous version of that driver (19.3.4.184 [2017-05-09]) which does have the setting mentioned above. I was able to do this by going to the device properties for "Synaptics SMBus TouchPad" in the device manager. On the driver tab of the properties dialog box, I chose "Update Driver", then "Browse my computer for driver software", and finally "Let me pick from a list of available drivers on my computer". Fortunately the old driver was in the list. After rebooting, I once again see the setting to disable the touchpad.

Also, for those that don't want to install the older driver, and who run an MSI laptop, you can temporarily disable it by pressing the FN key + F3. This will also remain in effect until you reboot, so for me the older driver is a better solution.

Hope this helps anyone else with this problem!

Do I need to use File History as well? What is the difference - Microsoft Community

cw00 say:

Do I need to use File History as well? What is the difference between Backup and Restore and File History?

I backup my files and system image once a week. Do I need to use File History as well? What is the difference between Backup and Restore and File History?

Original Title : File History

rdrewd2 say:

If you're already using another program for backups (either Backup & Restore Win7 or third party), then no, you don't need to add File History to your backup routine.  This is just opinion, but if you've got a backup solution that works and you're comfortable using, rely on that; run File History if you want to as a redundant backup, but not as your only backup.

In addition to a file backup, Backup & Restore (Win7) allows you to create a system image and repair disk, File History doesn't.  B&R will save to a separate partition or external drive, File History requires saving to an external drive only.  In my experience, File History also seems to adhere to any backup schedule rather loosely (I've had it miss backups on numerous occasions), while B&R seems to stick to a set schedule much more reliably.

The only advantages I've found with File History are that:  1) File History backups can be browsed through via file explorer, while B&R backups can't 2) File History is easily accessed from either Control Panel or the Settings menu, and 3) File History will easily allow for multiple chronological versions of the same file to be saved, i.e. you can very simply track changes you've made to a given document over time.  But personally, I don't think File History has any real advantage over using a decent third-party program, or even Backup & Restore (Windows 7).
