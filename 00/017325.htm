winsol say:

I created a table and when I copy and insert it on a page with smaller margins than on which I created the table, the complete table does not fit on the page. The table is too large. How can I reduce the size of the table so that it fits on the page with a smaller margin? Thank you.
suzanne s. barnhill say:
The setting is "Allow table to extend into margins." If this is checked, the table can extend into the margins. But if you have the table width set to 100%, it should be confined to the margins (assuming that the text wrapping is set to None). Make sure that none of the paragraphs in the cells has a negative right indent.

suzanne s. barnhill say:
You need to turn off auto resizing (Table Properties | Table | Options) so that  you can control the width of the table. There are also Compatibility Options (Office Button | Word Options | Advanced: Compatibility) that prevent a table from extending into the margins.

scottgem say:

Makes sense? :S

No! It doesn't make sense.

While there are good reasons to use unbound forms, unless you are dealing with those reasons, then I would not recommend that you use unbound forms. Clearly you are struggling with making them work, so why bother?

The difference between a bound form and an unbound form is that, with bound forms Access handles all the I/O between the form and the table. You don't have to do it. With unbound forms, you, as the developer, need to write code to handle it. And, unless you are very comfortable with writing such code (which you don't appear to be), you are creating headaches for yourself. When you are more comfortable with VBA code, you can play with Unbound forms, but for now I wouldn't.

Create a form with the customer table as its Recordsource. Click the Add New Record button on the Navigation bar, and you get a blank form to enter a new record. Close the form or move to a different record and the new record is saved to your table. Open the form, find the customer you want to edit and change whatever data you need to. close the form or move off the record and the changes are saved. NO CODE is necessary.

And, again you didn't answer a question I asked. I'm guessing here that you are using an autonumber for your ClientID field. If so, you can't update an autonumber so that could be why the SQL statement is failing.

For the error message "There was a problem sending the command to the program", use Windows Explorer to access the C:\Program Files\Microsoft Office\Office15 folder and right click on WINWORD.EXE then on Properties, go to the Compatibility tab and uncheck all the boxes.  It has also been reported that the problem can be overcome by editing the registry at HKEY_CLASSES_ROOT\Word.Document.12\shell\Open and deleting the ddeexec key.

The ddeexec can also be deleted from

HKEY_CLASSES_ROOT\Word.DocumentMacroEnabled.12\shell\Open.

That fix is described at ...
