Hi Mark,

 

Thanks for posting on Microsoft Community. We regret the inconvenience caused and will assist you in resolving the issue.

 

Before assisting further, I would like to ask you a few questions:

  1. Are you able to login into the user profile?
  2. Are you using Microsoft account or Local account to login into the user profile?

 

The above issue may occur if the fingerprint scanner driver is not updated, corrupted or incompatible on the computer. The above issue may also occur if the feature is not set appropriately on the computer.

 

Method 1: If you are able to login into the user profile, I suggest you to update the fingerprint scanner drivers from the device manager by following the steps below and check if the issue is resolved.

 

  1. Press Windows logo + X keys on the keyboard and select Device manager from the context menu.
  2. Expand Human Interface Device option, right click on the fingerprint scanner driver and select Update this software from the context menu.
  3. Follow the onscreen instructions and wait till the update is complete.
  4. Close the window and check if the changes are effective.

 

Method 2: Remove the Fingerprint Data saved on your PC by following the steps below and check if the issue is resolved.

 

  1. Press Windows logo key on the keyboard, type Settings and select the top most search result.
  2. Select Accounts and click on Sign-in options from the left side panel of the window.
  3. Under Windows Hello option, click on Remove.
  4. Follow the steps from a to c, only this time click on Set up under Windows Hello and check if you are able to add the fingerprint. You will need to scan your finger a few times to set up the Windows Hello feature on your PC.
  5. After the procedure is complete, close the window, lock the PC (press Windows logo + L keys on the keyboard) and check if the fingerprint is accepted to login into the user account.

 

Write to us with the answers to the above questions and the status of the issue on the same post for further assistance. Your reply is most important for us to ensure we assist you accordingly.

A Code 43 error is generated when Device Manager stops a hardware device because the hardware reported to Windows that it's having some kind of unspecified problem. 

First, I would suggest you to follow these steps and run the Hardware and devices troubleshooter:

  1. Type “Troubleshoot” without quotes in search and click on Troubleshooting in the results displayed on the left side of the screen.

  3. In the list of options, click on Hardware and Devices.

  4. Click Next and follow the on-screen instructions.

Yes, you can use it, but I strongly advise you have a secondary backup plan.

Using the Windows 7 64 bit disk, install the Windows 7 Windows Easy Transfer Utility from the DVD (To find it, just navigate to x:\support\migwiz, 'x:' represents the name of your disk drive. Once you have done that, simply do a clean install of Windows 7 64 bit disk. In Windows 7 64 bit disk, launch Windows Easy Transfer, follow the on screen wizard, point to where you stored the the WET file (external hard disk), and it will restore your personal files and settings. Hit Enter

Boot from the Windows 7 64 bit DVD
Click Install Now
Accept License Agreement
