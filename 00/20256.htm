Yes, I've discovered the version I'm running, but can you tell me how to reinstall it?


First you've got to uninstall it.  To do that, go to Control Panel/Programs and features/view installed updates and then find Internet Explorer 11 from the list.  Right-click on Internet Explorer 11 and select uninstall.  


Reboot (important).  Then, I believe, you need to go to Control Panel/Windows Update, and begin a Check for updates.  The Updater ought to notify you that Internet Explorer 11 is available for download. (*note: Somewhere I read something about waiting 15-20 minutes before performing a reinstallation after an uninstallation.)


The reason I say that "I believe" is because I ran into a similar situation over here ( http://answers.microsoft.com/en-us/ie/forum/ie11-windows_7/tried-uninstalling-ie-11-but-result-produced-no-ie/d7816f97-b7d5-43e7-ae69-c583424e7fbc?page=1 ) but I did not perform the Control Panel/Windows Update/Check for updates.  Rather I became confused by the inability to get online to download. 


Subsequent to all that now, I have read over here ( http://technet.microsoft.com/en-us/library/dn321445.aspx ) what I believe to be the proper outlined procedure.


But before you go to all this trouble, why bother?  You've got the latest version and update.  Also, when you uninstall, first look into your "Windows turn features on and off" and see if the checked listing is something other than Internet Explorer 11.  IE 11 is very buggy. 


IE 10 is slightly less, but  I believe you will be better served with IE 9.

If the sheet of interest was hidden from the VBA editor or by code, it is possible that the sheet will not be available from Excel's user interface and, in such case, the unhide option would be unavailable.

Try as follows:

 

Alt-F11 to open the VBA editor

Alt-IM to insert a new code module

In the new module, paste the following code

 

'=========>>
Option Explicit

'--------->>
Public Sub Tester()
    Dim WB As Workbook
    Dim SH As Worksheet

    Set WB = ThisWorkbook
    
    For Each SH In WB.Worksheets
        SH.Visible = xlSheetVisible
    Next SH
End Sub
'<<=========

 

Alt-Q to close the VBA editor

Alt-F8 to open the macro window

Select Tester | Run

chatmaker say:

I assume you want Form2 to be mode-less

From From1 use

Form2.Show False

While Form2.Visible: Doevents: Wend


From From2 use this when finished:

Me.Hide

That works just by the fact of not unloading Form2, but what if I want to unload the form? Would I have to unload from the caller every time? And what happens when I don't have a caller?

chatmaker say:

It's okay to not unload it. If you show it again it doesn't use any more memory, it just reappears. And if you need to, you can retrieve it's properties. But if you want to remove it from memory, with our without a caller just set Form2 to nothing from anywhere in your app (Form2 is a global object). The next time you show Form2 it will go back into memory with no problems.

Alternatively, we can instantiate an instance of Form2 from a calling routine. And when the calling routine ends, the instance will go out of scope and VBA will remove it from memory.

Dim myForm as Form2

Set myForm = New From2

myForm.Show False

johnsorc say:

Type mismatch in Access 2010

I've created a database to track defects on our product due to vendor issues.  I've then created an set of three queries for each vendor to give me all defects, defects unique to a product code, and quantity purchased information.  I recently changed a lot of manual entry data to formulas, and today noticed a big issue.  For example, I have dates and then I wanted them to display Month YYYY for plotting in Excel.  To do this I used the expression:
MonthName(Month([Defect Found])) & " " & Year([Defect Found]) for the date in the Defect Found field

and

MonthName(Month([Exact Pour])) & " " & Year([Exact Pour]) for the date in the Exact Pour field


In my query for unique data, the Exact Pour expression (which goes into the Charting field) has started giving me a Type mismatch error.  What I can't figure out is what is different between the two expressions.  I know the issue is with the Charting field because removing this from the query allows the query to run just fine.  Unfortunately, this is one of the key bits of information, and the data is worthless without it.

SELECT First([Casting Defect Log].Quantity) AS FirstOfQuantity, First([Casting Defect Log].[Added to Log]) AS [FirstOfDate Added to Log], First([Casting Defect Log].[Defect Found]) AS [FirstOfDate Defect Found], First([Casting Defect Log].[RT Month/Year]) AS [FirstOfRT Month/Year], First([Casting Defect Log].RMC) AS FirstOfRMC, First([Casting Defect Log].[General Part]) AS [FirstOfGeneral Part], [Casting Defect Log].[Ht#], First([Casting Defect Log].Charting) AS [FirstOfPour Date for Charting], First([Casting Defect Log].[Pattern Number]) AS [FirstOfPattern Number], First([Casting Defect Log].Size) AS FirstOfSize, First([Casting Defect Log].Descrip) AS FirstOfDescrip, First([Casting Defect Log].Issue) AS FirstOfIssue
FROM [Casting Defect Log]
GROUP BY [Casting Defect Log].[Ht#]
HAVING (((First([Casting Defect Log].Supplier))="AFG"));

I appreciate any help given, and can provide more info if needed.  Still fairly new to Access.

hansv mvp say:

If there are records for which Exact Pour is blank, MonthName(Month([Exact Pour])) will generate an error.

Try the following instead of MonthName(Month([Exact Pour])) & " " & Year([Exact Pour]):

 

Format([Exact Pour],"mmmm yyyy")

 

This will return a blank if Exact Pour is blank, and the month name followed by the year otherwise.
