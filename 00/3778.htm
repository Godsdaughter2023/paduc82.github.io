Hi SM,

Actually, the cmdlet “Set-Mailbox -Identity "user" -RetainDeletedItemsFor” controls the duration of deleted items in the Recoverable Items folder.

If the deleted items retention means the duration of deleted items in the Recoverable Items folder, which are deleted or deleted from Deleted Items Folder. You can use the Shell to configure deleted item retention for a mailbox as below:
Set-Mailboxplan -Identity ""ExchangeOnlineEnterprise"" -RetainDeletedItemsFor 30.

This operation can change the default retention for deleted items in Recoverable Items folder from 14 days to 30 days. However , this only apply to the newly created users. The existed users still need to be modified separately by running the command you mentioned in the thread.
Moreover, the cmdlet may not be deployed on all Office 365 tenants. If the cmdlet doesn't work on your tenant, this may indicate your requisite may not be realized now.


If the deleted items retention means the duration of items in Deleted Items Folder. You can change the default value by following the steps below:
1. Go to Exchange Admin Center.
2. Go to compliance management.
3. Create a new retention tag which is applied automatically to a deleted items folder. Then set the retention period as 30 days.
4. Add the retention tag to the default MRM retention policy.

zack barresse say:

I think you'd just need to add the value of the maintenance costs over the original duration, in which case it would be something like this.

=B1/(B2+INT(B1/B2/12)*B3)

Where:

B1 = Capex

B2 = Annual save

B3 = Annual maintenance cost

So an ROV of your above values calculates to 107.143, which is 8 years and change, which the above formula computes as...

600.000 / (5.600 + 8.000) = 44.12

The following code adds another worksheet and uses the raw data as input and the new worksheet as output. That way your raw data is left as is.

Note the comments in case you need to edit the raw data worksheet name and also maybe if you don't like the worksheet name I have used for the output.

The code creates the output worksheet if it does not exist and if it does exist then clears all data from it so ensure that you don't already have a worksheet by the same name that you don't want cleared.

Backup your workbook before running the code.

Also the variables are all dimensioned at the top of the module so they are available to all subs within the module. I found it easier to code and debug this way.

Just copy all of the code below into a standard module. If you want to run it from an ActiveX command button then just call the sub RptFormatting from it.

Dim wsRawData As Worksheet
Dim wsReport As Worksheet
Dim rngActiv As Range
Dim rngSubActiv As Range
Dim rngCel As Range
Dim strRptName As String
Dim bolActiv As Boolean
Dim bolSubActiv As Boolean
Dim bolMultiSubActiv As Boolean
Dim lngRow As Long
Dim strActiv As String
Dim strSubActiv As String
Dim rngCelSave As Range
Dim dblSubActivTotHrs As Double
Dim dblActivTotHrs As Double
Dim dblGrandTotHrs As Double
Dim dblSubActivTotAmt As Double
Dim dblActivTotAmt As Double
Dim dblGrandTotAmt As Double

Sub RptFormatting()
'Edit "Sheet1" to your Raw Data Sheet Name
Set wsRawData = Sheets("Sheet1")

'Edit "ShtReport" to your preferred name for report worksheet
strRptName = "ShtReport"

On Error Resume Next
Set wsReport = Sheets(strRptName)
On Error GoTo 0

If wsReport Is Nothing Then
  Sheets.Add After:=Sheets(Sheets.Count)
  ActiveSheet.Name = strRptName
  Set wsReport = Sheets(strRptName)
End If

wsReport.Cells.Clear

With wsRawData
  'Edit the 2 in .Cells(2, "A") to your first row _
   of raw data after the column headers.
  Set rngActiv = .Range(.Cells(2, "A"), _
    .Cells(.Rows.Count, "A").End(xlUp))

  .Range(.Cells(1, 1), .Cells(1, .Columns.Count) _
    .End(xlToLeft)).Copy Destination:=wsReport.Cells(1, 1)
End With

With wsReport
  .Range(.Cells(1, 1), .Cells(1, .Columns.Count) _
    .End(xlToLeft)).Font.Bold = True
End With

bolActiv = False
bolSubActiv = True
bolMultiSubActiv = False
lngRow = rngActiv.Cells(1, 1).Row

dblSubActivTotHrs = 0
dblActivTotHrs = 0
dblGrandTotHrs = 0
dblSubActivTotAmt = 0
dblActivTotAmt = 0
dblGrandTotAmt = 0

For Each rngCel In rngActiv
  'Following If/Then/End If only on first pass
  If Not bolActiv Then
    strActiv = rngCel.Value
    strSubActiv = rngCel.Offset(0, 1).Value
    Call ActivityHeader
    bolActiv = True
  End If
 
  With wsReport
     
      If rngCel.Value = strActiv And _
            rngCel.Offset(0, 1) <> strSubActiv Then
        Call SubActivityTotal
        lngRow = lngRow + 2
        bolMultiSubActiv = True
      End If
     
      If rngCel.Value <> strActiv Then
        If bolMultiSubActiv = True Then
          Call SubActivityTotal
          lngRow = lngRow + 1
        End If
        Call ActivityTotal
        lngRow = lngRow + 2
        strActiv = rngCel.Value
        Call ActivityHeader
        bolMultiSubActiv = False
      End If
 
      .Rows(lngRow) = rngCel.EntireRow.Value
      dblSubActivTotHrs = dblSubActivTotHrs + .Cells(lngRow, 4)
      dblActivTotHrs = dblActivTotHrs + .Cells(lngRow, 4)
      dblGrandTotHrs = dblGrandTotHrs + .Cells(lngRow, 4)
     
      dblSubActivTotAmt = dblSubActivTotAmt + .Cells(lngRow, 6)
      dblActivTotAmt = dblActivTotAmt + .Cells(lngRow, 6)
      dblGrandTotAmt = dblGrandTotAmt + .Cells(lngRow, 6)
      lngRow = lngRow + 1
  End With
 
  Set rngCelSave = rngCel
 
Next rngCel
 
Set rngCel = rngCelSave

Set rngCel = rngCel.Offset(1, 0)

If bolMultiSubActiv = True Then
  Call SubActivityTotal
  lngRow = lngRow + 1
End If

Call ActivityTotal
lngRow = lngRow + 2

With wsReport
  .Cells(lngRow, rngCel.Column).Offset(0, 2) = "GRAND TOTAL"
  .Cells(lngRow, rngCel.Column).Offset(0, 3) = dblGrandTotHrs
  .Cells(lngRow, rngCel.Column).Offset(0, 5) = dblGrandTotAmt
 
  .Cells(lngRow, rngCel.Column).Offset(0, 2).Font.Bold = True
  .Cells(lngRow, rngCel.Column).Offset(0, 3).Font.Bold = True
  .Cells(lngRow, rngCel.Column).Offset(0, 5).Font.Bold = True
 
 
  .Columns("A:F").ColumnWidth = 20
  .Columns("A:F").Columns.AutoFit
  .Rows.AutoFit
End With

Set wsReport = Nothing
Set wsRawData = Nothing
End Sub

Sub ActivityHeader()

With wsReport
  .Cells(lngRow, rngCel.Column).Offset(0, 2) = strActiv
  .Cells(lngRow, rngCel.Column).Offset(0, 2).Font.Bold = True
  .Cells(lngRow, rngCel.Column).Offset(0, 2).Font.Underline = xlUnderlineStyleSingle
  lngRow = lngRow + 1
  dblActivTotHrs = 0
  dblSubActivTotHrs = 0
  dblActivTotAmt = 0
  dblSubActivTotAmt = 0
End With

End Sub

Sub ActivityTotal()

With wsReport
  .Cells(lngRow, rngCel.Column).Offset(0, 2) = strActiv & " Total"
  .Cells(lngRow, rngCel.Column).Offset(0, 3) = dblActivTotHrs
  .Cells(lngRow, rngCel.Column).Offset(0, 5) = dblActivTotAmt
 
  .Cells(lngRow, rngCel.Column).Offset(0, 2).Font.Bold = True
  .Cells(lngRow, rngCel.Column).Offset(0, 3).Font.Bold = True
  .Cells(lngRow, rngCel.Column).Offset(0, 5).Font.Bold = True
 
  dblActivTotHrs = 0
  dblActivTotAmt = 0
End With

strActiv = rngCel.Value
strSubActiv = rngCel.Offset(0, 1).Value

End Sub

Sub SubActivityTotal()

With wsReport
  .Cells(lngRow, rngCel.Column).Offset(0, 2) = strSubActiv & " Total"
  .Cells(lngRow, rngCel.Column).Offset(0, 3) = dblSubActivTotHrs
  .Cells(lngRow, rngCel.Column).Offset(0, 5) = dblSubActivTotAmt
 
  .Cells(lngRow, rngCel.Column).Offset(0, 2).Font.Bold = True
  .Cells(lngRow, rngCel.Column).Offset(0, 3).Font.Bold = True
  .Cells(lngRow, rngCel.Column).Offset(0, 5).Font.Bold = True
 
 
  dblSubActivTotHrs = 0
  dblSubActivTotAmt = 0
 
End With
strSubActiv = rngCel.Offset(0, 1).Value
End Sub

ken sheridan say: The task would certainly be possible, provided that you have a good understanding of the database relational model, and how to represent it in an Access application.  The first thing is to decide on the logical model, which is the basis of the physical model, i.e. the set of normalized tables and relationships between them which represent the real world entity types and the real world relationship types between them.  Each entity type is represented by a table, and the columns in the table represent the attributes of the entity type.  Its important to understand that a relationship type is really just a special kind of entity type, so can be modelled by a table whose columns represent the attributes of the relationship type.  A many-to-many relationship type is always modelled in this way, but a one-to-many relationship type, while it can and sometimes is modelled by a table, is most commonly modelled by the primary key of one table being referenced by a foreign key in the other table.

A well designed database is unlikely to have any tables which come anywhere near the maximum permitted number of columns.  On the contrary, a well designed table will in most cases have a relatively small number of columns.  An excessive number of columns will often indicate that data are being modelled as column headings.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.

As regards the interface in most cases a subform is used within a parent form where the parent form's table and the subform's table are in a one-to-many relationship type.  Often the subform will be modelling a many-to-many relationship type, as this is done by resolving the relationship type into two or more one-to-many relationship types.  A common example is that of an OrderDetails subform which models a many-to-many relationship type between Orders and Products.  The OrderDetails table has OrderID and ProductID foreign keys, and is linked to the parent Orders form on OrderID.  A parent form will often contain multiple subforms, and these might be correlated, so that as the user moves from row to row in one subform, another subform's recordset is reloaded to show rows which reference the currently selected row in the first subform.  You'll find an example in CorrelatedSubs.zip in the same OneDrive folder to which I gave you a link earlier.  This uses data from Northwind as its example and represents this in both correlated subforms and conventional nested subforms.

As to whether you should convert to an Access database or stick with Excel, it really depends on how comfortable you feel in the development environment which Access gives you.  If you have a good grasp of the relational model, along with concepts like normalization and functional dependence, and have sufficient experience with VBA and SQL, then there is no reason why you should not be able to develop a robust and efficient database to answer your requirements.  You said in you original post that Access is easy to use.  Well, it should be easy for a user to work with a well developed Access database, but I think most experience Access developers would question whether developing an Access database is easy for someone with little or no previous experience of doing so.  The learning curve is not a shallow one, and to think that it is achievable without a not inconsiderable investment of time and effort would be foolish.  So, it's for you to decide whether you wish to put in that time and effort.  The quote from Richard Feynman below my signature is there for a reason.

Phone screen -- Troubleshooting

We are excited to provide an early preview into the newest feature for the Your Phone app – We are excited to provide an early preview into the newest feature for the Your Phone app – phone screen. You can now mirror your Android phone’s screen directly on your PC without having to dig for your phone. 

What devices are supported? 

Select here to learn what devices are supported.

How do I pair over Bluetooth? 

Select here to learn how to connect with and troubleshoot Bluetooth.

How do I switch keyboard language or layout? 

When you type using your PC’s keyboard the layout and language is controlled by a setting on your Android phone for physical keyboards. It won’t be affected by the language of the keyboard you’re using on your PC.  

You can click on the keyboard language button when you are using the phone screen feature. This will launch the keyboard language settings where you can choose the language of your preference.

You can also manually change this at anytime by:  

1. On your Android, open the Settings app.  

2. Tap System Languages & input.  

3. Under keyboards, tap Physical keyboard.  

4. Select the keyboard that you’re using on your Phone (E.g. SwiftKey or Samsung Keyboard).  

5. Pick a language or layout.  

Note: You’ll need to change this setting whenever you want to type using another layout or language. 

How do I change the mouse pointer speed? 

  1. Open your device's Settings app.  
  2.  Tap General management > Language and Input. 
  3. Under Mouse/Trackpad, change the pointer speed to the middle position. 

Same network troubleshooting 

The phone screen feature only works when your phone and PC can communicate with each other. They’ll both need to be connected to the same network (for example, your home’s Wi-Fi). Also, some network configurations block devices from communicating with each other—even if both devices are connected to the internet.  

Here are a few things to check for: 

  • Is your phone is connected to mobile data instead of Wi-Fi? Connect it to the same Wi-Fi your PC is connected to. Metered networks aren’t currently supported for the phone screen feature. 
  • Are your phone and PC connected to similar networks but in a different way? For example, one might be on a 5G network and the other might be on 2G. Or one might be wireless, and the other might be connected through a cable. You might also find that the networks do not share the same subnet or are not bridged properly. Try connecting both devices to the same network in the same way and, if necessary, follow up with your network administrator or consult your modem/router manuals on how to change this. 
  • Do you have a firewall or VPN on your phone or PC? It could be blocking communication between the devices, so try disabling them to see if that fixes the problem. 
  • Does your network have multiple access points? Each device might be connected to a different access point, preventing the devices from sharing a subnet or bridging properly. Follow up with your network administrator or consult your modem/router manuals on how to change this. 

General troubleshooting 

If you see an error message indicating your PC couldn't connect to your phone it can happen for a variety of reasons. Here are few things to try:  

  • Does your PC have internet connectivity? 
  • Is your phone turned on?
  • Is Battery saver mode turned off on both your phone and your PC?
  • Is airplane mode turned off on your phone? 
  • Is Bluetooth on in both your PC and your phone?  
  • Are your phone and your PC connected to the same network or Wi-Fi? 
  • Is your phone within Bluetooth reach from your PC?  
  • Are you signed into the Your Phone Companion app with the same Microsoft account as you're using on your PC?  
  • Are your system accounts working correctly? You can check. In Windows, go to Settings > System> Shared experiences. If you see a Fix now link in the Accounts section, select the link and follow the instructions.

How do I submit feedback?  

If you have Windows 10, Feedback Hub is built right in. To find it, select the Start  button, then select Feedback Hub. 

Once you have the Feedback Hub open, select Report a problem.  In the second step, select Applications > Your Phone and specify that you’re having issues with Bluetooth. Recreating your problem in the third step goes a long way in improving the Your Phone app.
