john w. vinson say:

Pretty easy actually!

Create a database with just a link to this ODBC connection and to the backend containing the static table. In the database create a macro named AutoExec which runs a query to delete all records from the static table, and then an append query to populate it from the ODBC linked table; the last step of the macro would just be a Quit command to exit and close the database.

jackie l say:

 

If you put your date fields on a form instead of having the parameter boxes pop up, your query will have the both parameters at the same time.  Put unbound text boxes on a form then refer to them in your query criteria.  The wizard is trying to execute after entering on the first parameter.

To avoid this, and the export wizard, put code similar to the following on a button on your form:

DoCmd.TransferText  acExportDelim, "Export Specification Name", "qryQueryName", "c:\ExportFileName.csv", True

Create your Export Specifications if wanted by stepping through the export wizard and chosing Save Export Steps.  The "True" will put field headers in your export.  Put False if no headers wanted.

Hope this helps
