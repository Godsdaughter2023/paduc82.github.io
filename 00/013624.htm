pj98rx say:

Good to know that it works now. One thing that could have been the reason for the problem is that the NAT mappings have expired in your operator network which would prevent the packets from properly reaching your mobile device. By default the SIP negotiation is done with the connectionless UDP packets and the NAT mapping is not necessarily maintained very long in the operator network for these UDP packets. By the time when the SIP server sends a response back, a new port may already have been assigned for your mobile and the SIP server packet is dropped by the operator network. It could therefore be dependent on timing conditions whether the SIP negotiation goes through OK.

 

If you have any more problems, you can try using the TCP protocol instead of UDP in your SIP settings. You can select this in the Transport type setting in the Proxy and Registrar settings of your SIP profile (default is AUTO which usually ends up using UDP) . With TCP protocol the NAT mappings are maintained longer in the intermediate routers and the packets won't get dropped as easily by the NAT devices as with UDP.

MS Word List Indent Options - Microsoft Community

joshhandyside say:

MS Word List Indent Options

Hi Guys,

Can you explain to me the difference between the number position and text indent when adjusting the indent options for a list?

Both seem to perform the same as the other.

Thanks in advance.

stefan blom say:
"Aligned at" is the position of the list symbol (bullet or number). "Text indent at" is where the second and subsequent text lines in the list paragraph starts. Assuming that "Follow number with" is set to "Tab character," "Add tab stop at" defines where the text of the first line in the list paragraph starts. Note that all of these values are measured from the left margin (of the page, table cell, text box, etc.). For more, see http://shaunakelly.com/word/numbering/numbering20072010.html.

joshhandyside say:

Can you also help me understand why, when I try and adjust the indentation of lists sometimes, that it doesn't indent at all.

Or, that when I set the text indent it moves the bullet point instead of indenting the text.

stefan blom say:

You must always change indentation via the multilevel list dialog box or, for single-level lists, via Adjust List Indents. It won't work reliably if you try using the indent indicators on the ruler.
