How can I batch correct the same error in multiple documents - Microsoft Community

lauriefree say:

How can I batch correct the same error in multiple documents

I have created several documents, and now I find they all have the same mistakes in them. Is there a way to do a search/replace in multiple documents at once, or must I do this one at a time?
macropod say:

The following macro allows you to browse to a folder containing the documents you want to process, then replace a given string in all documents in that folder automatically.

 

For installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm


Sub UpdateDocuments()
Application.ScreenUpdating = False
Dim strFolder As String, strFile As String, wdDoc as Document
Dim StrFnd As String, StrRep As String
strFolder = GetFolder
If strFolder = "" Then Exit Sub
strFile = Dir(strFolder & "\*.doc", vbNormal)
StrFnd = InputBox("What is the Text to Find")
StrRep = InputBox("What is the Replacement Text")
If StrFnd = "" Then Exit Sub
While strFile <> ""
  Set wdDoc = Documents.Open(FileName:=strFolder & "\" & strFile, AddToRecentFiles:=False, Visible:=False)
  With wdDoc.Range.Find
    .ClearFormatting
    .Text = StrFnd
    .Replacement.Text = StrRep
    .Forward = True
    .Wrap = wdFindContinue
    .Format = False
    .MatchCase = True
    .MatchWholeWord = True
    .MatchWildcards = False
    .MatchSoundsLike = False
    .MatchAllWordForms = False
    .Execute Replace:=wdReplaceAll
  End With
  wdDoc.Close SaveChanges:=True
  strFile = Dir()
Wend
Set wdDoc = Nothing
Application.ScreenUpdating = True
End Sub

Function GetFolder() As String
Dim oFolder As Object
GetFolder = ""
Set oFolder = CreateObject("Shell.Application").BrowseForFolder(0, "Choose a folder", 0)
If (Not oFolder Is Nothing) Then GetFolder = oFolder.Items.Item.Path
Set oFolder = Nothing
End Function

john w. vinson say:

This is in fact the most common way that a combo box works. A combo has several intersecting properties:

  • the RowSource is usually a Query including all the fields included in the combo
  • The Control Source is the name of the field in the form's recordsource into which the selection will be stored (the ID number in this case)
  • The Bound Column specifies which field in the Rowsource query will get stored into the Control Source
  • The ColumnWidths property controls which columns will be displayed where

What the user sees will be the first nonzero width column. So you'll want a RowSource query like

SELECT ObserverID, LastName & ", " & FirstName FROM Observers ORDER BY LastName, FirstName;

The ColumnCount would be 2 (you want to refer to both the ID and the name); the Bound Column would be 1 to store the ID; the ColumnWidths property would be

0";1.25"

to conceal the ID (with the zero width), and allow an inch and a quarter for the calculated name.
