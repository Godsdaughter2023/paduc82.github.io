Do you know how to add and run a macro? If you need instructions on that, just ask.

If you add this macro below to a button on the QAT, it will take you to each move.

Some caveats: It is quick and dirty--I wrote it now for this problem, so to get it to work properly, make sure that you start with the I-beam at the start of the document (Ctrl+Home).

Second, it works forward only. So it scans from where you are towards the end, and stops when it has found a move. When you have finished with that, then you run it again to go to the next move. Third, if you follow a move, that may mean that miss a different move between the original and new locations, so just be aware of that. Last, it will stop at both the original and new locations of each move, so essentially it will stop twice for one move, although I do think that that is what I would have wanted it to do, anyway.

Here's the macro:

Sub SelectNextMove()
On Error Resume Next
Dim r As Revision
Dim rng As Range
Set rng = Selection.Range
For Each r In ThisDocument.Revisions
    If r.Range.Start > rng.Start Then
        Debug.Print r.MovedRange
        If Err.Number > 0 Then
            Err.Clear
        Else
            r.Range.Select
            Exit Sub
        End If
    End If
Next r
MsgBox "All revisions have been examined.", vbInformation + vbOKOnly, "Revisions scanning completed"
End Sub

I had a run at this and couldn't get the columns to cycle through array processing no matter what manner of trickery I threw at it. The closest I cam up with was,

=SUMPRODUCT((A1:D1)/SUM(INDEX(A:D,,COLUMN(A:D))))

When I reached my final level of exasperation, I resorted to a User Defined Formula (aka UDF). Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module (e.g. Alt+I, M). Paste the following into the new pane titled something like Book1 - Module1 (Code), 

Function udf_Sum_Ratios(rNum As Range, rDenom As Range) As Double
    Dim dTtl As Double, c As Range
    For Each c In rNum
        dTtl = dTtl + CDbl(c / Application.Sum(Intersect(c.EntireColumn, rDenom)))
    Next c
    udf_Sum_Ratios = dTtl
End Function

Tap Alt+Q to return to your worksheet once that is pasted. The syntax is,

=udf_Sum_Ratios(<range of numerators>, <range of denomonators>)

For your sample data this would be,

=udf_Sum_Ratios(A1:D1,A:D)

That simple UDF appears to accomplish what native Excel functions seemingly cannot.

For the issue you have mentioned, I found another thread where

http://answers.microsoft.com/en-us/windows/forum/windows_tp-winipp/when-i-open-the-feedback-app-it-says-to-use/235c3457-891d-43c7-8e0b-cd3431963f4f

To convert the account, you can do the following:

  1. Sign in to your Microsoft account on Windows 10 Technical Preview.
  2. Click Start and then PC settings.
  3. Click Users and accounts and under Your Profile click Disconnect on the right of the screen.
  4. Enter your Microsoft account's password, and click on Next.
  5. Type in a Username, Password, and Password Hint that you want set for your Local account and click Next. Click on Sign out and Finish button.

In order to switch back, follow these steps:

  1. On the newly converted Local account, click Start and then PC Settings.
  2. Click Users and accounts and under Your Profile click Connect to a Microsoft account.
  3. Enter the Microsoft account ID and Password and click Next.
  4. Choose if you want OneDrive as your default file saving location, click Next.
  5. Click Switch.

Now check if the Windows Feedback app gives you the same error and reply with the result.
