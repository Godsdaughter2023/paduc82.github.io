Sub PromptReplace()
Dim SearchArray As Variant, ReplaceArray As Variant
Dim RngSel As Range, RngFnd As Range, i As Long
SearchArray = Array("the", "quick", "brown", "fox")
ReplaceArray = Array("A", "Slow", "Green", "Grub")
Set RngFnd = Selection.Range
For i = LBound(SearchArray) To UBound(SearchArray)

    .MatchWholeWord = True
    While .Execute
      With RngSel

            .Collapse wdCollapseEnd
          End If
        End If
      End With
    Wend
  End With
Next

Weighted target for variable number of differing tasks - Microsoft Community

russ c dawson say:

Weighted target for variable number of differing tasks

I'm trying to come up with a formula that will give a productivity figure for work done on three different tasks each given a different measure of time taken to fulfil the task.

Task 1 - 40 to be completed in an hour

Task 2 - 7 to be completed in an hour

Task 3 - 4 to be completed in an hour

The work is undertaken by workers who at the moment only record how many they do in an hour but not what task they have been doing.

The target based on the figures above is 51 per hour.  

However, sometimes Task 2 may be ignored because of the number of tasks building up on Task 3 so resource would be concentrated on Task 3 and vice versa. 
 
Management, are unable to grasp the fact that productivity will be effected and still insist on the target of 51.  They then ask we we have not met the target!

I need to be able to produce a formula that allows for movement of resource and adjusts depending on the tasks undertaken.  

So for instance, if Task 2 is not worked and Task 3 is, then you would expect the target to reduce accordingly to 48.  The reason for that is that (in simplistic terms) the worker for Task 2 has moved to 3 and therefore does 4, as has the original worker for Task 3.  

Sadly it is not so simple because of the variance of staff levels and the time taken doing each task.  They do record hourly how many they do. I anticipate adding exactly which task they did.

Your help would be much appreciated.

Translate number of tasks done to "points earned" where each type of task represents a different number of points.

Task 1: 1/40 = 0.025 points per task done

Task 2: 1/7 = 0.143 points per task done

Task 3: 1/4 = 0.25 points per task done

Total points earned, P = 1/40 * N1 + 1/7 * N2 + 1/4 * N3, where N1, N2, and N3 are the number of tasks done of type 1, 2, and 3 respectively.

The target, T,  can then be set to 1 point per worker, e.g. 3 points if there are 3 workers.

Example 1:
For 3 workers the target may be achieved with N1 = 40, N2 = 0, N3 = 8 giving P = 1/40 * 40 + 1/7 * 0 + 1/4 * 8 = 3
(In this case the total number of tasks, N1 + N2 + N3 = 48 according to your expectations.

Example 2:
For 3 workers the target may also be achieved with N1 = 0, N2 = 0, N3 = 12 giving P  = 3
In this case the total number of tasks = 12.

Hope this helps / Lars-Åke

This is the code I'm using... Is there someplace I can insert code to pass on which column and row are selected so I can use that to change the combobox properties, such as the width?

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
Dim str As String
Dim cboTemp As OLEObject
Dim ws As Worksheet
Set ws = ActiveSheet
On Error GoTo errHandler

If Target.Count > 1 Then GoTo exitHandler

Set cboTemp = ws.OLEObjects("TempCombo")
  On Error Resume Next
If cboTemp.Visible = True Then
  With cboTemp
    .Top = 10
    .Left = 10
    .ListFillRange = ""
    .LinkedCell = ""
    .Visible = False
    .Value = ""
  End With
End If

  On Error GoTo errHandler
  If Target.Validation.Type = 3 Then
    'if the cell contains a data validation list
    Application.EnableEvents = False
    'get the data validation formula
    str = Target.Validation.Formula1
    str = Right(str, Len(str) - 1)
    With cboTemp
      'show the combobox with the list
      .Visible = True
      .Left = Target.Left
      .Top = Target.Top
      .Width = Target.Width + 15
      .Height = Target.Height + 5
      .ListFillRange = ws.Range(str).Address
      .LinkedCell = Target.Address
    End With
    cboTemp.Activate
  End If

exitHandler:
  Application.ScreenUpdating = True
  Application.EnableEvents = True
  Exit Sub
errHandler:
  Resume exitHandler

End Sub
'====================================
'Move to next cell if Tab or Enter are pressed
Private Sub TempCombo_KeyDown(ByVal _
        KeyCode As MSForms.ReturnInteger, _
        ByVal Shift As Integer)
    Select Case KeyCode
        Case 9 'Tab
            ActiveCell.Offset(0, 1).Activate
        Case 13 'Enter
            ActiveCell.Offset(1, 0).Activate
        Case Else
            'do nothing
    End Select
End Sub
