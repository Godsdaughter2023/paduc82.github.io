There’s no need to remove anything from the Quarantine. Quarantined items could be safely left it in the Quarantine indefinitely, but will in fact be automatically removed after 90 days; or after the number of days specified with the -QuarantinePurgeItemsAfterDelay parameter.

-QuarantinePurgeItemsAfterDelay

Specifies the number of days to keep items in the Quarantine folder. If you specify a value of zero or do not specify a value for this parameter, items stay in the Quarantine folder indefinitely.

https://docs.microsoft.com/en-us/powershell/module/defender/set-mppreference?view=win10-ps

The default value of this parameter is actually 90 days; but you can set your own preference with the PowerShell Set-MpPreference cmdlet.

Set-MpPreference -ScanPurgeItemsAfterDelay [integer specifying the number of days]

The bottom line is that Project won't do what you want. The basic principle of Project's network is forward scheduling. When a given task (ID 2 in your example), is dependent on multiple predecessor tasks (ID 6, 17, 26 & 35), by definition ID 2 cannot start until ALL the other tasks start. If you need a particular task to start with the earliest of multiple predecessor tasks starts, then you will either have to do that manually or with some VBA.

Again, not the answer you want, but that's the way it works.

john - project say:

Ivan,

You're welcome and thanks for the feedback. Here's a pseudo-code outline of how I might approach it.

1. Use a flag field to identify each task that you want to make an "earliest" predecessor for each milestone. For example in your sample file above set Flag1 for tasks 2, 6,17,26 & 35. I assume you have other milestones, so set another flag for each of those milestone groups

2. Then cycle thorough all tasks in the file and look for tasks with each specific flag set, or, set up a filter to find the desired flag group and cycle through the ActiveSelection.Tasks set. Reset the filter to go through each milestone group.

3. In the loop look for the earliest start date of the predecessor tasks for each milestone group, save that unique ID

4. When all tasks in the selected group have been examined, set the UniqueIDPredecessors of the current milestone to the unique ID saved in step 3.

5. Repeat for each milestone group.

There are two ways to initiate the macro. The simplest is to manually run it after any edits that may affect the milestone start date. The more complex approach is to set it up with a Task Change Event so it will automatically run anytime a change is made. But keep in mind that the latter will fire on every single change and to keep things from bogging down, some type of screening code would need to be used to immediately exit for all changes that are not relevant.

update: This is a first cut at an actual macro. It is set up to handle 5 milestone groups - tweak as necessary. It should at least give you a running start

Option Compare Text
Sub EarliestLink()
Dim Mile(5) As String
Dim ES(5) As Date
Dim t As Task
Dim flg As Long
Dim NewPred As String
For i = 1 To 5
    ES(i) = ActiveProject.ProjectFinish
    Select Case i
        Case 1
            flg = pjTaskFlag1
        Case 2
            flg = pjTaskFlag2
        Case 3
            flg = pjTaskFlag3
        Case 4
            flg = pjTaskFlag4
        Case 5
            flg = pjTaskFlag5
    End Select
    For Each t In ActiveProject.Tasks
        If t.Summary = False And t.Milestone Then Mile(i) = t.UniqueID
        If t.Milestone = False And t.GetField(flg) = "yes" And (t.ScheduledStart < ES(i)) Then
            ES(i) = t.ScheduledStart
            NewPred = t.UniqueID
        End If
    Next t
    ActiveProject.Tasks.UniqueID(Mile(i)).UniqueIDPredecessors = NewPred & "ss"

Next i
End Sub
