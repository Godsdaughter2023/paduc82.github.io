There are several settings that affect the the way the message list is sorted.

  • View > Message list > One-line view.
    This gives the greatest flexibility in sorting.
    • Right-click on a column heading to add and change columns;
    • Left-click to sort by the selected field;
    • Click again to reverse the sort order;
    • Sorting by a non-date field (e.g. From) will show messages with the newest uppermost regardless of the previously-selected date sort order and regardless of the order of columns.

  • View > Conversations.
    • This command is only available when the message list is sorted by a date field;
    • With Conversations on, messages with the same subject (or referring to the same subject) are grouped together.

      Note: This is a departure from the traditional system of grouping messages on the basis of the In-Reply-To: or References: headers. In the event of a conflict, messages are sorted by subject regardless of the reference headers. This can give strange results: a new message with a common subject (e.g. Hi! or Re: Hi! ) will sort with older messages with the same subject, whereas a message in a thread where the sender has changed the subject will be divorced from the thread it belongs to.
    • Each conversation is sorted with the oldest message uppermost regardless of the date sort order;
    • Conversations are sorted within the whole list by the date of the newest message in the conversation.

      Note: This is a departure from the traditional sort order, where conversations ('threads') sorted within the message list in order of the original message in the conversation.

      This also means that with conversations collapsed (Options > Read > Automatically expand grouped messages is not selected), the date columns will appear to be out of order, because the date shown for a collapsed conversation is that of the oldest message in it, whereas it is sorted by the date of the newest message in it (press Ctrl-Shift-O for Options).
