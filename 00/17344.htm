jeeped say:

To copy B5 (second column, fifth row) from Sheet1 to Sheet2's E6 (fifth column, sixth row) this code would do.

Sub copy_with_formats()
    Sheets("Sheet1").Cells(5, 2).Copy Destination:=Sheets("Sheet2").Cells(6, 5)

End Sub

The single line of operative code could be copied, pasted and modified into additional lines for additional copy/paste operations. If adjoining cells were to be copied across, they could be grouped into a single line of code using .Resize(x, y).

ronrosenfeld say:

Another way of looking at the issue is that it is not possible to differentially format different words in a cell if those words are the result of a formula.  In other words, if

A1:  Hello world

you can make one word bold.

But if A2:  =A1

then there is no way of making just one of the words shown in A2 bold.

What a macro can do is replace that formula with the actual text, and then differentially format those words in the string.  But you lose the formula.

A macro could also automatically fill that cell on the occurrence of some event, such as when a cell in a certain column on some worksheet is changed.

But the macro cannot "leave" the formula in the cell (and differentially format the words)

Continue with these registry subkeys:
  1. Locate the following registry subkey:

    32-bit: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall
    64-bit: HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Uninstall

  2. On the File menu, click Export, type UninstallKey01, and then click Save.
  3. Under the Uninstall subkey that you located in step 1, click each subkey, and then determine whether the subkey has the following value assigned to it:
    • Name: UninstallString
    • Data: file_name path\Office Setup Controller\Setup.exe path

      Note In this example, file_name is a placeholder for the name of an installation program, and path is a placeholder for the file path.
  4. If the subkey contains the name and the data that are described in step 3, click Delete on the Edit menu. Otherwise, go to step 5.
  5. Repeat steps 3 and 4 until you locate and then delete every subkey that matches the name and the data that are described in step 3.
  6. Close Registry Editor.

anoop k m say:

In Registry Editor, go the locations mentioned in Step 1. Go to the first location if you’re using Windows 32-bit. Go to the second location if you’re using Windows 64-bit.

 

Step 2 explains you to take a backup of the Uninstall registry key before you make changes to it. To do this, select the Uninstall key and use the File > Export option as suggested.

 

Now expand the Uninstall key registry key and select any subkey under it. On the right hand pane, you will find three columns - Name, Type and Data. Delete the subkeys that contains the Name and Data information that is given in Step 3. Note that the ‘file_name’ is a substitute for the name of the installation program and ‘path’ is a substitute for the actual file path.

 

If you do not find any subkeys that match the given criteria, you may skip this step and close the Registry Editor.

That’s exactly the problem. You’re not supposed to look for that ‘specific string’.

As indicated, instead of “file_name”, you’ll find a real file name.

And instead of “path”, you’ll find a real file path.

 

To avoid confusion, this is what you have to do:

Under Uninstall key, delete all the subkeys that ends with 0FF1CE. That’s it.

 

And it’s not necessary that you’ll find these registry keys. Sometimes it’ll be already removed due to the steps done earlier. If you’re not able to find it, then it means that its not there.
