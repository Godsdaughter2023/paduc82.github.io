ken sheridan say: It's possible, and relatively easy with the right code behind the button, to generate a new row (record) in a table which duplicates non-key values from an existing row.  This can either be done by setting the DefaultValue property of each of the relevant controls to the current row's values, or by executing an INSERT INTO statement in code to insert a new row into the table.

Where this is legitimate the number of columns (fields) in which the data is duplicated is usually few.  Generating a new row in which most of the data is duplicated more often than not points to an incorrect table design in which data is redundantly duplicated, leaving the database open to the risk of update anomalies.  The redundancy in such cases is eliminated by decomposing the table into a set of correctly normalized related tables.  Hence Tom's questions.

Also, in situations where this sort of thing is legitimate, it is usually done in a single bound form which navigates to a new record in which the values are already present either as default values in an uncommitted record, or as values in a row already created by an INSERT INTO statement.  Where one or more of the columns in which values are to be added or amended manually is the primary key, or any candidate key, the former is necessary as to attempt to insert a row with duplicate or Null keys would raise an error.

To advise you correctly we really need to know what you are attempting in terms of the reality modelled by the database, rather than how you are attempting to do it.

How do phones <--> cloud sync work?

It seems that you have three places where your accounts and contacts live.

Windows Phone 7

Windows Phone 8.1

Outlook.com (aka Cloud)

The way this works is that if you add Outlook.com account to WP7 and WP8.1 the contacts from Outlook.com will be downloaded to those two devices. After all your contacts are downloaded to both devices and you decide to make a change on WP7, that change will then be updated on Outlook.com and then in turn WP8.1 will update based on that new change in the cloud the next time it syncs.

If you delete a contact on WP8.1, the change will be updated on outlook.com and then WP7 will update based on that change the next time it syncs or looks for changes.

However if you delete or modify something on Outlook.com, both WP7 and WP8.1 will update the contacts based on that change the next time it syncs.

You do not need Outlook 2013, unless you want to sync the data in your outlook.com account to your desktop computer.

MSN.com = Hotmail.com = Live.com = outlook.com ---- In terms of the online account that stores email, contacts, and calendars and that you set a phone up to sync this data to, these are all the same thing nowadays.  Microsoft rebranded all these services under the Outlook.com name a year or so ago.

Click the OneDrive icon and the http://onedrive.live.com website opens up

arkengel say:

To answer  your last sync question:

If your WP8.1 phone should explode or stop working you won't lost any contacts.

For the phone to delete the contacts on Outlook.com you'd have to create an action "delete" on the phone for the specific contact. The phone when connected to the web would then relay that message to Outlook.com and delete it from there.

However if your phone stops working, the contacts that were saved on the phone and the cloud should just be lost on the phone. But restoring the phone would bring them back to any new device you installed the new account on.

Backing up contacts,

Contacts, calendar and email are automatically synced or will sync every hour (depending on how often you set  your account to sync) You can see what settings here: 

settings > email + acounts > Microsoft Account > Currently syncing

options range from:

- As items arrive

- based on my usage

- every 15 minutes

- every 30 minutes

- hourly

- daily

If  you delete the contact on outlook.com/people.live when the phone contacts the server it will check for changes and delete the respective contact (or add a new one if you added through outlook.com/people.live.)

Using export feature on Outlook.com/people.live creates an excel sheet which can be read by Open Office Calc or Microsoft Excel. You wouldn't use this feature to bring them back on the phone. When you add your account to the phone it will try to sync contacts, email and calendar automatically from outlook.com not requiring this step. 

However backup can be important to turn on if you want settings like "Apps", "Background colors, pictures", "Text messages", "pictures" to be saved and retrievable in case you change or get another phone. 

Hope this helps.
