hansv mvp say:

There is no way to add extra rows to an Excel worksheet.

You could split the data across multiple worksheets, but this makes performing calculations across all data more complicated.

 

An alternative would be to store the data in an Access database, or even better, in a SQL Server database with an Access database as frontend. The size of an Access table is only limited by the maximum size of an Access database: 2 GB. A SQL Server table can contain a virtually limitless number of records.

jllatham say:

I will cast my vote with the others: set it up in some relational database that is not limited so much with it's boundaries.

 

You can actually import your current Excel data into a table in Access through commands available in Access - and if you already have data spread across multiple sheets in Excel, Access will let you create the table from your first sheet and then append the data in the other sheets to that table.

 

Normally, in Access, you set up data entry/editing forms that are associated with tables or queries to work with the data, BUT...  some people actually like to work directly in the tables which can be opened and when they are opened they pretty much look just like an Excel spreadsheet minus the row numbers and column letters.

 

The problem with Access is that you can't play "what if" so easily as you can in Excel and of course there are no easy (relatively speaking) ways of setting up formulas to get the results from math calculations, although you can use queries and even macros and VBA to do things like that for you.

 

One thing you can do is create a few Select queries to "filter" the data you need to work with in such a fashion and have that filtered data sent to a table which can then be exported as an Excel worksheet, and then you can work with that subset of your

 

But I'm going to also recommend that you consider the recommendation made by HansV:  put it into an SQL server database, and if you want a "front end" for it, use Access.  Because the sheer size of the data (1 million plus records) is going to put a strain on the Access 'engine' - remember it was originally designed to be a "personal" relational database and just doesn't have the power that the big kids on the block, SQL Server, Oracle, etc, have to manipulate that much data.

 

You can get a FREE copy of SQL Server 2008 Express which might fit the bill from this Microsoft page:

http://www.microsoft.com/en-us/download/details.aspx?id=25052

Regarding that missing article/troubleshooter:

Not very likely to help, but run troubleshooters from Control Panel > View by: small icons > Troubleshooting > run the Network and Internet troubleshooters.

Outlook.com and the long running script: If there's a Stop script button use it. The script problem is probably caused by ads. Setting up and using an ad blocking program (Adblock Plus or another) may help.

Another option: Other browsers like Edge, Chrome, Opera and Firefox are more forgiving of these script problems. Consider using another browser on affected sites

Some preventive maintenance tips:

1. About twice each week...

    Internet Options> General tab> Browsing History> Delete> uncheck "Preserve Favorites website data" then delete Temporary Internet Files. No reason to delete cookies, history,  passwords, etc.

2. Internet Options> General tab> Browsing History> Settings>

Set cache size (Disk space to use) at 200-250 MB. A larger cache won't speed up browsing.

3. When this slowness, sluggishness occurs, close all iexplore processes from Task Manager. IE can use a lot of memory, especially after viewing JavaScript heavy pages. This will get IE's memory usage back to an acceptable level.

4. Shutdown at night or restart at least once per day. Some users use sleep or hibernate, but Windows still, after all these years, likes an occasional restart.
