This is an upgrade problem.

There are many causes for upgrade failures.

They can be troubleshooted by viewing the Panther logs.

By default they may be hidden.

If an upgrade fails often the operating system returns to the prior working version and generates the logs.

It takes time to troubleshoot these logs.

Many of the entries in the logs are error codes that need to be decoded and deciphered.

Some of the problems could be:  problems with drivers, AV, insufficient disk space in partitions, etc.

The error codes are important as this would be the method to start the troubleshooting.

Open file explorer > search for >

C:\$windows.~bt\sources\panther

Within this folder you may find many files.

Look for:

setuperr.log

setupact.log

setupmem.dmp

bluebox.log

rollback

It may be easier to find these logs using everything search:  http://www.voidtools.com/

You should see immediate results as you enter characters and the more characters you enter the narrower the results.

Use one drive or drop box to post the logs that you find.

It depends on how Windows 10 was originally configured on your computer.
