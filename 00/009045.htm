How do I delete bug splat - Microsoft Community

dgm_47 say:

How do I delete bug splat

I have no games on my laptop using Windows 10 and want to delete bug splat. I did not download it nor do I know how it got here.
don varnau say:

Bug Splat is an error reporting tool being used by some app or program that you've installed.  It's not malicious.

https://www.bugsplat.com/

If bug splat is appearing at startup, you should try to disable or uninstall it or determine which program is causing it to run.

Start> Settings> Apps> Apps & features, if Bug Splat is listed uninstall it.

Plan B... Allow Bug Splat to send its report and you should be able to determine which program is involved.

When the message appears, right-click the Taskbar> Task Manager> Startup tab. If Bug Splat is listed in the startup list, disable it.

If those steps don't disable Bug Splat look at Task Manager> Startup again. What non-Microsoft programs are loading at startup that don't need to be, possibly a game.

If you have SlimwareDriver Update installed, uninstalling it should make Bug Splat disappear.

Don

Recommend you thoroughly review Microsoft Security Essentials – Installation Checklist and Frequently Asked Questions

 

MSE is concerned with prevention and detection of malware.  Remember that tracking cookies are not malware and MSE does not identify cookies like some programs do.

 

Since MSE is not identifying any "threats" it sounds like you are practicing good surfing habits and are limiting your exposure to malware. Keep up the good work.

 

If you want to test MSE go to http://www.rexswain.com/eicar.html and test it on an Eicar file.

 

FWIW I recommend your default actions for MSE be set to quarantine suspected malware.

http://answers.microsoft.com/en-us/protect/forum/protect_scanning/whats-the-difference-between-remove-and-quarantine/0b8ba9f6-5682-4880-b6c9-62940c2a150e


On this computer that I'm using MSE is not set to do any scheduled scans...I prefer an "occasional" manual scan.  I really think routine scanning is a bit overrated but others don't share my view.

 

And here is an excerpt from a previous post by MSE Forum Moderator Stephen Boots reference scans that sums it up quite well:

 

1. A periodic quick scan will check the common locations for the presence of malware.

2. A full scan will scan all local drives for any traces of malware

Typically, you do not need to perform a full scan.

Real time protection is monitoring activity and file access on your PC, so if you attempt to open an infected file, it should detect the infection and stop you.

A full scan is very time consuming and disk intensive. If you wish to perform a clean scan periodically, go for it. I rarely run one.

Your best defense is not scanning, but preventing infection in the first place:

In addition to MSE or other up to date antivirus software:

Make sure that the Windows Firewall is enabled.

Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

Make sure Internet Explorer is at version 8 or 9 and updated with all patches.

In Internet Explorer, use the SmartScreen Filter.

Make sure that IE Internet Security settings are at least set to medium-high (default).

Enable the pop-up blocker in IE.

On Vista and Windows 7 make sure that User Account Control (UAC) ON  and not running with elevated privileges.

Make sure that Windows Automatic Updates are set to at least notify, but the preferred setting is to download and install automatically. If you update manually, be sure to update as soon as possible after being notified of available updates.

Make sure that installed applications, especially Adobe Acrobat, Adobe Flash, and Java are at their latest versions. Many vendors are regularly updating and patching for security holes.

Never click through links from unknown sources and use caution even if they are from a "trusted" source.

Never open unsolicited email attachments.

Practice safe web browsing.

jay freedman say:

When you right-click the Word document, is the Open command in the menu shown in bold? (Whichever command is bold is the default used by the double-click.)

In the Windows registry, the subkeys under HKEY_CLASSES_ROOT\Word.Document.12\shell determine what happens when a .docx file is  worked on from Windows Explorer. There are separate subkeys for the Open command and the Edit command, among others. I can imagine a situation where the value of the entry in the Open/command subkey is missing or damaged, causing Windows to substitute Wordpad as a secondary program for .docx; and at the same time the Edit/command subkey is valid for Word and has somehow been marked as the default to run on a double-click.

Granted, this scenario is rather unlikely, but there might be others -- weird things do happen.

To rewrite the registry entries with the ones created by the Office installation, open the Run dialog from the Windows Start menu, enter the path to the Winword.exe file followed by a space and the switch /r, and click OK. The command line should look something like this:

"C:\Program Files (x86)\Microsoft Office\root\Office16\WINWORD.EXE" /r

Nothing visible will happen on the screen, but the registry will be updated.
