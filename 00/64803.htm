tryx3 say:

MAB,

Good.  You can use that to create the new account and copy across the existing files into the new account -

1  You can create a new user with a local user account on the computer without them having a Microsoft account.

You do this through

  • Start button,
  • Settings,
  • Accounts,
  • Family & other users,
  • [Other users section],
  • Add someone else to this PC,
  • I don't have this person's sign-in information,
  • Add a user without a Microsoft account.
  • Then you just enter a username, password [which you should write down straightaway] & password hint,
  • Now look back at the list in Family & other users, [Other users section],
  • Click on the entry for the new account,
  • Click on the Change account type button that appears,
  • Select Admin.

2  Then log in to the new account to test that it is alright [you are really just testing that you wrote the password down correctly as there is no reason to suppose that anything else could be wrong].

3  You could then make a Password reset disk through Control panel, User accounts but it is just as easy to write the password down somewhere secure.

4  Then you can create a new Standard account for your day-to-day use unless like me you use and Admin account for that - in which case create an additional Admin account now so that you have a backup in case your routine accounts get corrupted.

5  Then, whilst signed in to an Admin account, copy all the documents, music etc from C:\Users\OldUserName\Documents etc to C:\Users\NewUserName\Documents etc.

6  Your Windows settings in the new account will be at default values.  I do not suggest that you try to copy them over into the new account because you already know that the old account had glitches somewhere and you don't want to copy across any settings that are potentially already corrupt.  I tried to do this in Windows 7 [various procedures were suggested] but I always had umpteen errors so I stopped trying.  You could search the forum for this topic but I have not seen anything that I think would help you and, as I said, I think it might not be advisable.

    Well, I will throw out some things.  I cannot, however, keep up with your activities in a thread like this. 

    A Custom Refresh Image is not recommended to be used as a Reset image.  If you want a Custom Refresh Image, it doesn't go in the Recovery partition, but in a folder on the drive.  I would not think you would want to waste that space.

    A Reset image, according to what I have read, needs to be designated as install.wim or install.swm, but is required to be designated as install.  If you would have set the reagentc command to look at the directory where the install.wim file you mention is, the process probably would work.  I would have not way of knowing if a full Factory reset would use something other than the install image to rebuild your system.  My Dell has an install.swm and a base.wim which I believe are both used in the OEM process.

    a .wim file may have more than one image in the file.  That is what the index is for, to tell the process which image it needs.  On my system it uses the 3rd image, index 3, for the factory restore image.  The other two are for factory or diagnostic type uses.  It is too bad you don't have a copy of the original reagentc /info listing, or access to an original reagent.xml file.  Either one would confirm the information required.

    I have not tried using the path designator for the Reagentc command you used, but I will test it soon to see if that designation is acceptable.

    When a recovery drive is created, the factory image, if necessary, is split into .swm files and it takes more than one to complete the image, install.swm, install2.swm, etc.

    Removing a fingerprint:

    1.  Open the Settings app, and select Accounts.
    2. Click on Sign-in options on the left side, and click on Remove for Fingerprint under Windows Hello.

    Setting up a fingerprint:

    1. Open the Settings app, and select Accounts.
    2. Click on Sign-in options on the left side, and click on Set up for Fingerprint under Windows Hello.
    3. Click on Get started.
    4. Enter your PIN to verify it's you.
    5. Scan your finger on the fingerprint reader. You will need to scan your finger a few times to set up with Windows Hello.
    6. When all set, click on Close.
