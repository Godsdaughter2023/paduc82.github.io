greginmich say:

This is a very common issue for people using third-party AV apps. Quite frequently Windows Defender doesn’t get properly switched off in its registry key, and this causes a group of symptoms that includes a failure to start the service with Error 0x8007139f (in the UI); and a failure to update Windows Defender definitions with Error 0x80070643. The technical aspects of this error state were reproduced, illustrated, and described here:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-defender-error-code-0x8007139f/93096739-16ce-471b-8716-1776f0c69f89?page=2

Usually this problem will be resolved if you uninstall the third-party app, run its cleanup utility, and then reinstall it, because this should update the relevant Windows Defender registry entries. But if this doesn't work, or if you just don’t want to bother with recycling the installation, then you can use Group Policy to completely turn off Windows Defender. In the Professional edition of Windows 10, you can do this with the Group Policy Editor; and in the Home edition, you can use the Ten Forums Turn_Off_Windows_Defender.reg file. Everything you need is right here:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

And there’s also an even quicker method for Norton and Avast users that essentially mimics a reinstall by disabling the third-party AV app; starting Windows Defender; and then enabling the third-party app while Defender is still turned on:

http://answers.microsoft.com/en-us/windows/forum/windows_10-update/definition-update-for-windows-defender-kb2267602/d4edd314-a438-4ac2-b7a9-dc1c45a1ef62

This easy-as-1-2-3 method might also work with other AV products, but we don’t have any confirmation for that yet.
