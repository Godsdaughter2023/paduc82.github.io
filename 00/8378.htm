Don't ask me how to do this because I expect you're using Windows and I don't know the first thing about that operating system, but the basic concepts you'll need are as follows:

1) Your computer must be set to do IPv4 packet forwarding so that it can relay information between the Internet and your phone.

2) Your phone must be set to use the PC's IP address as its default gateway (this can be achieved by doing manual, advanced settings on the phone).

3) You must find out the IP addresses of the nameservers that your ISP puts at your disposal and also enter these into the access point advanced settings on your phone.

4) Both your computer's and your phone's WiFi adapters must be in Ad-hoc mode (this is a 1-to-1 connection, not a shared access point).

saataja say: ICS is effectively the same thing as "IPv4 packet forwarding". Enabling ICS configures your PC to act as an IP router between the internet connection (=PPPoE connection in your case) and another "local" network connection (=USB WLAN adapter in Adhoc mode).

 

I don't think you need to specifically disable IPv6 on your PC as long as the IPv4 is enabled and you have set up the ICS correctly.

 

Depending on how your PPPoE connection has been setup and your operating system version (e.g. ISP provided PPPoE client or XP/Vista built-in PPPoE client) you might have share either the Local Area Network connection or the PPPoE/ADSL connection. According to following link, PPPoE / ADSL is most likely the correct interface to share using ICS since you mentioned setting this up on Vista.

http://www.annoyances.org/exec/show/article04-105

 

Also take a look at "How to share PPPoE connection with ICS" instructions (although for XP):

http://support.microsoft.com/kb/316276

 

Note that you should select your WLAN Adhoc connection to be used as "Home networking connection" instead of LAN connection that is mentioned in step number 6. of the above mentioned kb316276 article.

 

There should not be any need to enter IP address, DNS servers and gateway details manually to you N85 since ICS should automatically enable your PC to provide IP addresses (192.168.0.x) via DHCP to client devices connecting to to your PC's ICS WLAN Adhoc network.

 

According to above mentioned links defining your WLAN USB adapter to become a ICS "Home connection" should have automatically configured this interface with static IP address 192.168.0.1 (netmask 255.255.255.0). If you choose to configure your N85 with static IP address settings (instead of leaving it to Automatic = DHCP) then "Default Gateway" setting on the phone should be your USB WLAN adapter's IP address instead of your PC's PPPoE interface IP address or the default gateway that your PC itself uses to access ISP / internet.

 

 

On the phone side you probably have already created new "Connection Method" (i.e. Internet Access Point) to your phone e.g. under "Internet" Destination with matching WLAN network name (SSID) and WEP key settings that your PC's WLAN adapter is configured with. To ensure correct WLAN operating mode on the phone verify that the "WLAN Network Mode" is set to Ad-hoc (instead of Infrastructure) on the given WLAN connection method you going to use when connecting to your ICS setup.

 

Note that you might end up with "No gateway reply" situation even if ICS has been correctly set up but you have e.g. mistakenly entered the WEP key incorrectly into the phone's or PC's WLAN settings.


If phone fails to connect to internet even after you are confident that ICS aspects have been correctly configured (and your PC itself can still access internet) you might perhaps carefully consider removing the WEP encryption momentarily from the PC's WLAN Ad-hoc connection and create another Ad-hoc mode WLAN connection method with matching settings (WLAN security mode = Open ) on your N85 and give it a try outruling the possibility that incorrectly entered WEP encryption key is causing connection to fail.

 

You should however avoid leaving your WLAN ad-hoc network permanently open and unprotected since even if security of the WEP encryption is almost obsolete it is still better than nothing and at least it keeps neighbours from using your network by mistake.
