IMAP goes to outlook.office365.com, using TLS on port 993 (indicated by the "imaps" URL schema), authenticating by user and password.

For completeness, that's the respective piece from the Mutt manual, explaining the smtp_url variable:

3.263. smtp_url

Type: string
Default: (empty)

Defines the SMTP smarthost where sent messages should relayed for delivery.
This should take the form of an SMTP URL, e.g.:

smtp[s]://[user[:pass]@]host[:port]

where “[...]” denotes an optional part. Setting this variable overrides the
value of the $sendmail variable.

Again, everything works, I use it daily. Just resending an existing mail doesn't work. Note that "resending" (in terms of RFC2822) is quite a different thing than "forwarding", even though both are somewhat related. When forwarding a message, you create a completely new message, with new headers, new addressees, possible more editing applied. When resending, the only changes that are supposed to be done to the original message is to add the (required or allowable) Resent-* header fields, and then feeding the message into SMTP again.

jerryduan_msft8021 say:

Hi Boyan,

OneDrive for Business sync client is not integrated into Windows 10 Enterprise. To get OneDrive for Business sync client, you have two choices:
1. Upgrade your OneDrive client to the Next Gen OneDrive for Business sync client by configuring file synchronization to their machines using their Office 365 business accounts. For the details: Deploying the OneDrive for Business Next Generation Sync Client in an enterprise environment

2. Install the OneDrive for Business client from the software page or via this KB article: https://support.microsoft.com/en-us/kb/2903984

However, with the first option, users are only able to sync their own OneDrive for Business library. The Next Gen OneDrive for Business sync client does not support syncing other SharePoint libraries currently. If you have to sync other SharePoint document libraries such as Team Sites or on-premises SharePoint environments, you should go with the option 2. In the KB article below you can view some other limitations of the Next Gen OneDrive sync client:
https://support.microsoft.com/en-us/kb/3125202

If you have any concerns about integrating OneDrive for Busienss client into Windows, we'd suggest you submit feedback via our UserVoice site: https://onedrive.uservoice.com/forums/262982-onedrive

Regards,
Jerry
