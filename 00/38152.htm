al jarvi say:

That's the split tunnel issue. In general the VPN tunnel can be configured on either the client side (like the native Windows PPTP VPN client for example) or the server side (like OpenVPN for example) to force all traffic through the VPN tunnel or not. If all traffic is being forced through the VPN tunnel, which is what your describing, then internet access is controlled by the VPN server. That is a security precaution to isolated the server side network from your local network.

I would check with your network/VPN admins for help simply because you maybe running into a server side config that may or may not be changed depending on their network security protocols.

How to only have a local account on the computer - Microsoft Community

jymwythawhy say:

How to only have a local account on the computer

This computer (running windows 10) is for church clerk use and thus does not need to have a windows account log in. a. user say:

You should actually have two user accounts but I'll get to that in a minute.

An account with Administrator permissions can delete any other user account. Log in to your computer with an account that has Administrator permissions and delete the undesired user account.

A computer should have at least two user accounts: one with Administrator permissions and another without Administrator permissions. The latter is called a Standard account.

You use the Administrator account only for tasks that require those higher level ('elevated') Administrator permissions, like installing software. You use the Standard account for your everyday computing, and that is where you store all your personal files.

This setup is just one layer in a multi-layer approach to security. If your Standard account gets hacked, at least the hacker won't have elevated Administrator permissions that could do a lot more damage.

You should have a Microsoft account but you don't need to actually use it. You'll need your Microsoft Account user name and password to get certain things done on your computer, like downloading apps from Windows Store, but you don't actually have to have a Microsoft account on your computer.

Don't ever let one of those amateurs from Microsoft technical support log into your computer.

a. user say:

By default, the first user account created has Administrator permissions. If you upgraded to Windows 10, then your previous version of Windows had an Administrator account. That would be Clerk, if you didn't create any other user accounts in your previous version of Windows.

If you did a clean install of Windows 10, you had to create a user account. That would be Clerk, and it would have Administrator permissions if you did not create any other user accounts.

Either way, Clerk should have Administrater permissions and you should be able to delete the other user from Clerk.

If Clerk is not an Administrator account, and the only other user account is the one created by the other person, then you've got a problem with your Windows 10 installation.

a. user say:

I hate to say this, but Windows did not make all those decisions on its own. Windows does not have the ability to do all that purely on its own. No one would design an operating system with a huge security hole like that.

If you're in the position where Clerk is no longer an Administrator account then you have one last way out. Windows has a built in Administrator account - it's a system account, not a user account. The built in Administrator is your lifeline when you can't access a user account with Administrator permissions.

The built in Administrator is normally disabled because it's not supposed to be used like a regular user account. I'll show you how to enable it and then you can either promote Clerk to be a user account with Administrator permissions or create a brand new user account with Administrator permissions and leave Clerk as a Standard account.
