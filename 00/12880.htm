Perhaps something like this...

With these values
A1: (blank)
A2: 1
A3: 2
A4: 3
A5: (blank)
A6: 10
A7: 20
A8: 30
A9: 40
A10: (blank)
A11: 100
A12: (blank)
A13: 1000
A14: 2000
A15: (blank)

This regular formula, when copied down, returns the sum of the cells between 2 blank cells...eg Sum of A2:A4, Sum of A5:A8, Sum of A11, Sum of A13:A14
B1: =IFERROR(IF(ISBLANK(A2),SUMPRODUCT(INDEX(A:A,LOOKUP(2,1/ISBLANK(A$1:A1),ROW(A$1:A1))):A1),""),"")
Copy that formula down through B15.

In the above example, the formulas return these values:
B4: 5
B9: 100
B11: 100
B14: 3000

This little demo file illustrates a number of queries for numbering, ranking, or partitioning rows in a query's result table.  Examples for updatable and non-updatable queries are given, the latter being more efficient if there is no need to return an updatable result table.  One thing to note is how the primary key of the table is brought into play as a tie-breaker where the column by which the query is ordered contains duplicate values.  This can be seen in the following example for instance, where the query is ordered by TransactionDate, which will contain duplicate values when two or more transactions occur on the same day:

SELECT COUNT(*) AS RowNumber, T1.TransactionDate, T1.CustomerID, T1.TransactionAmount
FROM Transactions AS T1 INNER JOIN Transactions AS T2
ON (T2.TransactionID<=T1.TransactionID OR T2.TransactionDate<>T1.TransactionDate)
AND (T2.TransactionDate<=T1.TransactionDate)
GROUP BY T1.TransactionDate, T1.TransactionID, T1.TransactionAmount, T1.CustomerID;
