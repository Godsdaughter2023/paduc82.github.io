Have you performed this below steps? If not please try with this steps hope this will help you

Step 1:

Open the Command Prompt (CMD) as an Administrator.

(Press Windows key + X + A)

Step 2:

Type in the following command to ensure that your network interface supports Hosted Network feature:

netsh wlan show drivers

If Hosted network supported shows “Yes”, that’s great.

If it says “No”, you can try to update your wireless driver. If the updated driver still does not help, then you need to upgrade your hardware.

Step 3:

Now, we need to configure Ad Hoc Wireless connection. Type in CMD this command:

netsh wlan set hostednetwork mode=allow ssid=<your desired network name> key=<your password>

Step 4:

Next, you need to start it. Type in CMD the command below:

netsh wlan start hostednetwork

Make sure your wireless network connection is enabled.

Step 5:

Navigate to Control Panel > Network and Sharing Center. Click on “Change adapter settings”

Right-click on Wi-Fi adapter > select Properties

Head to “Sharing” tab > check the box Allow other network users to connect through this computer's Internet connection > finally click on OK button.

Now you can connect your Wi-Fi devices such as iPhone, Android smartphone, iPad, laptop … to this ad hoc wireless network and enjoy internet connection sharing.

gerry c j cornell say:

When you run chkdsk, the larger the disk the longer it takes. This is both logical and what you must accept as the most likely outcome. The time taken can be significantly longer if there are bad sectors to repair. Usually it is best to let the process run to completion. Stopping the process can be just wasting the time already taken to fix a problem. Unless you intend to run another utility, like Seatools, I would let it continue. I would expect chkdsk to report failure if it encounters an insurmountable problem.

Something else you may wish to consider. Unless you have a very good reason for retaining a 1.5tb partition you can sub-divide the drive into more manageable partitions. This makes housekeeping i.e. cleaning and defragmenting a much more manageable task. Running chkdsk when required would take a lot less time. You would not need to restart the computer to run chkdsk on second and further partitions on the drive. Another advantage accrues if you get the situation that chkdsk automatically runs because the system detects a problem as the system boots.

 

Hope this helps, Gerry Cornell
