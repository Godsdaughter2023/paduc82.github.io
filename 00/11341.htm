The following describes how you can fool the printer into accepting data from a document that transgresses the non-print area boundary. Do be aware, though, that this is liable to result in content not being printed if you push things too far.

1. Set the page margins to at least the minimum supported.

2. Insert a single-cell table into the document with the width you actually want to print.

3. Cut your content from the body of the document & paste it into the table

4. Reposition the table to get the positioning you want. You can do that via Table Tools|Layout|Properties. I'd suggest you start b y simply centring the table on the page, then working from there.

What are all those FeatureOnDemand updates and why am I getting one, - Microsoft Community

cbarnhorst say:

What are all those FeatureOnDemand updates and why am I getting one, two, or even none?

Some of you are seeing not only build 16299.15 downloads but some others called by various names, all of which start with FeatureOnDemand.  What the heck are these and do you need them?  Note: a few of you are getting a Feature Update instead of a Build 16299.15 update.  They are the same thing, don't worry.  A Feature Update is not what I am talking about here.

FeatureOnDemand is something new.  Eventually, you will install only the core of a new version of Windows plus features that your hardware and software configuration supports and needs in order for all features of the OS compatible with your system can function.  KB4046355 is an omnibus package containing many payloads.  You may need some of them or not, but so far I have not seen reports of anybody needed all of them.  To understand what is going on think of that game you once installed that required a particular version of the .NET framework and you saw a message about that and had to OK the installation of the module.  This appears to be the same idea.  They are called pre-requisites.

Do a Bing Search on "Andromeda OS."  I think we are seeing the first fruits of this project to modularize Windows further.  The idea is that you need everything in the core OS but do not need (or cannot take advantage of) some of the many other features offered.  So you only get stuff you need.  The "on demand" wording refers to what your system needs, not what you want.  In the future, if you attach a new VR device, for example, to your system your system might download a FeatureOnDemand for it.

Andromeda OS is designed to make it easier to maintain Windows and reduce the Windows footprint on a system by system basis.  There is no point in installing portions of the OS that you cannot use so why do it?  Another advantage is to OEMs who can tailor the Windows they pre-install, aligning the installed code more closely to the configuration they are selling you.

I may have this all screwed up so take this post with a grain of salt, but it sure looks like modularization is what FeatureOnDemand is all about.  If you have further information, please jump in.
