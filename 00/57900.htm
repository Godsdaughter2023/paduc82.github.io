hairyfool say:

There is little point in copying the entire drive as a backup as a significant part of it would be unusable to restore to a rebuilt computer. This includes most directories apart from the User folders so if backing up by a simple copy the backup should be limited to the specific files and folders needed. On the new computer all programs would have to be reinstalled anyway. Be a bit more selective.

The only way to back up the whole drive is by using the System Image tool but this requires that the entire drive is restored.

Windows protects certain files and folders as being something even the Administrator should not modify in the normal course of events so require a somewhat convoluted process to break that protection.

The philosophy of backup is somewhat different with Windows 10. Logging on using the Microsoft Account stores key security information on the OneDrive profile. If you have to rebuild/replace the computer at some time in the future logging back in using that same account allows your security profile to be restored automatically.

If you then store your user files (documents, photos etc.) in your OneDrive space then again by logging in using the same account they become available immediately.

If you keep files locally on the PC then look at the File History tool. This copies all files in your user profile to another location (your 128GB key?) but a little bit more. Having copied the original file set it scans those folders in the background at set intervals for any changed files and makes new copies of only the new files. The "history" aspect is because even when it finds a modified file it does not delete the original but keeps it. This means you can restore a file that you regularly modify at any point in its history you choose.

hairyfool say:

I am not sure where "test" comes in. If there is a folder C:\Users\Test then a user called test has been created at some time, its not a Windows created folder. You should really only have two main user directories if yours is the only account ever created \Users\your_name and \Users\Public.

So 1st off check that no other users exist, if you find Test remove it but copy any files that you need from its folders. If the folder does not exist it cannot be written to so if the User does not exist delete the folder.

Remember File History will create one master backup of the selected folders. From then on it ONLY backs up modified files so in practice hourly, daily or weekly for the average home user will not make a significant difference. Where it comes in is if you make changes to the same file all day every day. Then you would get a file copy for each backup session so if for example you are writing a book then that file will have a sequence of backups but for many writers it could be useful to go to a specific draft at any time they choose. Alternatively if you tend to modify or create different files each time then you only get the new file once. So for example 10 files per day would produce 10 backups each day, 70 for the week. A weekly backup will backup 70 files, exactly the same.

So the effect of not being able to set it weekly will depend on how you use your computer. It will also have a bearing oh how fast the backup drive is filled because 100GB of documents will a lot of files, on my system 8500 photos are only 38GB.
