rob koch say:

These "feature" controls were added in MSE 2.0 and then removed during the beta for MSE 4.0 as I recall, so they've been gone for nearly 2 years now.

 

The purpose originally was to deal with some issues with the early version of MSE where particular modules could sometimes cause issues, either due to conflicts with other drivers or applications on a system.  This allowed both some more granular trouble-shooting and tuning of MSE under certain specific conditions.

 

However, the more common use that many less knowledgeable users made of these was to disable portions of the security in order to deal with performance issues in general, often due to either the presence of malware or simply underpowered hardware.  Unfortunately this opened the door for even more malware, so the inability of many to use them properly made the controls dangerous for precisely the users who needed MSE most.

 

So during the 4.0 beta as I mentioned these and some other controls were removed and though a small number of vocal users complained, their arguments were generally non-sensical and showed exactly why they had become a problem.  So their value in trouble-shooting was far less than the risk they created for the very non-technical users who most needed a simple to use anti-malware, which was the original purpose behind MSE.

 

Regarding the question about the meaning of the various controls, the 'Scan all downloads' specifically selected to enable or disable the monitoring of downloads via more recent versions of Internet Explorer, Outlook and some others that many anti-malware gain access to via the IOAttachmentExecute interface mentioned in the following KB article.

http://support.microsoft.com/kb/914922/en-us

 

Note that both Windows Defender and MSE are mentioned in the 'Applies To' section.  This is also how virtually all current major anti-malware products gain access to these same download and other data streams within Windows, since these API interfaces have existed and improved since the release of Windows XP with SP2 and Windows Server 2003.  Actually the IOfficeAntiVirus interface existed before this in versions back to Office 97 as I recall, but weren't integrated into Windows itself until later.  This is where the 'Office' portion of its name comes from and harkens back to the early days of Macro viruses and protection.

 

The Network Inspection system on the other hand operates via the Windows Filtering Platform which was first added into Windows Vista/Server 2008 and provides the network interface for firewalls and other network monitoring and control applications.  This is why NIS won't operate on Windows XP, since the WFP didn't exist on that platform.

http://msdn.microsoft.com/en-us/library/windows/desktop/aa366510(v=vs.85).aspx
