Hello Odin99

#1 - There is no automated process to transfer your Outlook "settings" to a new computer. You will have to configure each email account again.

,#2 - Using export/import to copy Outlook data from one computer to another is generally ill-advised expect for the one case as noted below. What, if anything, needs to be copied from one computer to another depends on the type (POP3, Imap, Exchange) account configured on the old computer along with the version of Outlook that is in use on the old computer.

To determine what (and how) email accounts you have configured go to File --> Account Settings --> Email tab (this assumes your using Outlook '2010 or later on this computer)

To see what data files you have configured in your profile, go to File --> Account Settings --> Data FIles tab. (Take note of of the folder/file name of any PST file in the profile along with which data file is set as the "default Data file" which is where the default contact/calendar folders are located.

For any <Exchange> account - nothing needs to be copied for the account. Everything is sync'd with the server

For any POP3 account, you will need to copy the PST file from the old computer (with Outlook closed) to the new computer. A POP3 account can either have its own PST file or share one with other POP3 accounts. TO detemine which PST file is being used, go to the Email tab, click on the POP3 account and look down to the bottom of that window for the PST file listed beside the <Change Folder> button.

If you are using Outlook '2013 or later on the old computer and IMAP accounts are involved, you will need to export any folders to a PST file that contain the words "This Computer Only" in the folder name AND which contain data that you want to retain, There is no need to export any email folders since those are sync'd with the server.

If you are using Outlook '2010 or earlier with IMAP accounts, you will need to copy the default data file for the profile which contains the default contact/calendar fodlers. OL'2010 or earlier use 2 PST files for an IMAP account - the first for the email folders which doesn't need to be copied and the 2nd for the contactcalendar folders

Beyond that you will need to copy any PST files configured that are not associated with a specific email account

john w. vinson say:

As I suspected! This is just one example of why a lot of us really dislike Lookup Fields. See http://www.mvps.org/access/lookupfields.htm for a critique.

Your Append query is trying to insert the text string "Production" into the numeric Status field. The Status field appears to contain "Production" or some other Status value, but it doesn't. What's actually in the field is a concealed numeric ID, and it's that ID which you need to insert into the table using your Append query: that is, insert [Status List].[ID] rather than the text field [Status]. The same may be happening with other fields if you have more Lookups.
