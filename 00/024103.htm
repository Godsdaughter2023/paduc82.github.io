You observed that absolutely right. All you can do in OneNote is paint onto an image (of a former PDF) and not even hold those two graphic elements together.

Yes, that's right. What you are drawing on isn't even a PDF. It's just a picture file, created the same way as if you would print out a PDF to paper and then put it on a scanner, inserting the resulting bitmap file into your note. It's even called "PDF printout" in OneNote. It's not a PDF anymore and has lost everything what may have been in the original file (annotations, form fields, watermarks, searchable text...).

What you need is a PDF editor with annotation capabilities. OneNote is not a PDF editor, it's a note taking program in which you also may store images (e.g. of PDFs).

All you can do (if you are using Windows desktop OneNote 2010, 2013 or 2016) is to move the "PDF" image to the background (right click for the corresponding menu command), thus preventing it to get accidentally moved. You could still move your annotations though.

Yes, Outlook can be automated, just as you can automate Word from Excel. The details, of course, depend on what specific information you're looking for, from which appointments. Start by using Tools | References to add a reference to the Microsoft Outlook library. Then add code to get the running Outlook instance or start Outlook if it is not already running. Here's one way to do that: 

Function OpenOL(Optional ProfileName) As Outlook.Application

    Dim objOL As Outlook.Application

    On Error Resume Next

    Set objOL = GetObject(, "Outlook.Application")

    If objOL Is Nothing Then

        Set objOL = CreateObject("Outlook.Application")

        objOL.GetNamespace("MAPI").Logon ProfileName, , False, True

    End If

    Set OpenOL = objOL

    Set objOL = Nothing

End Function

    How to chain animations across multiple slides vs. stacking multiple - Microsoft Community

    clintwoon say:

    How to chain animations across multiple slides vs. stacking multiple animations on a single build slide?

    Is there a way to chain animations from one slide to another, without having to manually advance to the next slide? I don't want it doing it on a timer, i just want the last animation to play to trigger the transition to the next slide.

     

    The cleanest, easiest to work with slides are those with only a few animations where, for the most part, the slide is readable without running through the animations.

    The best looking animation build sequences, are slides that are literally LITTERED with overlapping objects that you have to use the selection pane to sort through.  Looking at the slide standing still, it's a total mess.

     

     

    steve rindsberg say:

    You can take advantage of the fact that PPT won't trigger a timed slide advance until all the animations on the slide have finished.  So for example:

    Set Slide 1 to advance automatically after 1 second (or less if need be)

    Add whatever animations you like to the slide.

     

    When you play the show, it'll advance automatically from Slide 1 to Slide 2, but only after all of your animations have played. 

     

    If you need a bit of delay after the last animation but before the advance, add a dummy animation offscreen.
