andrewphysick say:

The size of the offline cash can't be disabled unfortunately (only reduced to 2% of the total primary disk space), even for stationary computers when the file history backup location is on the primary disk too.

This 'cash' feature has been specifically designed for when external storage devices are used for the main file history backup (because you are supposed to backup your data to a separate hard drive in case of disk failures etc), so that you always have the latest backup in-case the backup drive fails or in-case you are going to be disconnected from the backup drive (if using a laptop on the go etc).

I only have a small SDD too, so if you are backing up to your primary disk drive and are having issues with storage space, you can set the cash to 2%, and set the saved versions to only 1 month, and exclude large folders that contain music and films etc (if you only want work to be backed up etc).

Hope it helps!

frederik long say: There is a huge difference between Safe Mode and Normal Mode. In Safe Mode, only native Windows stuff gets loaded, plus essential drivers. In Normal Mode you get lots of extra stuff. If some of the extra stuff is poorly written then you have a problem.

If the above drive X: sequence of commands applies to any share on any machine and if it fails consistently then I would to this:

- Check if you can get it to fail in Safe Mode with Networking.
- Physically disconnect the machine from the Internet.
- Boot into Normal Mode.
- Run msconfig.exe and untick every item under the Startup tab.
- Reboot into Normal Mode and check if the sequence still fails.
- Restore the tick marks in pairs until the problem comes back.

It would help if you could perform these test soon rather than waiting two weeks so that the issue remains fresh in respondents' minds.
