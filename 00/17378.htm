elainewachtel say: The old account was pop3, but Comcast wants you to use imap, so I changed it.  What can I do?

karl timmermans [outlook mvp 2012-2018] say:

The old account was pop3, but Comcast wants you to use imap, so I changed it.  What can I do?

#1 - You can simply delete the POP3 email account - that has no impact on any PST files that were or are currently associated with the email account itself. Outlook doesn't delete .pst files.

#2 - Just for clarity - what you see in the Outlook NAV pane are the data files associated with your email account(s) and not the email accounts themselves (i.e. if you had 5 POP3 accounts using the same PST file - there would only be one folder tree for all 5 accounts). There will <always> be a separate folder tree in the Nav pane for every IMAP/Exchange ActiveSync/Exchange email account.

#3 - Now to your "missing data". The only reason/way that would happen is if you set the POP3 account to use a new Outlook data file instead of pay attention to the options on the right hand side which would have allowed you to select an existing PST file. You're going to have to locate your original PST file --> add it back to your Outlook profile --> and remove the new PST file that was created. Your old PST file will also likely have been the default data file which means that's where you're going to find your contacts/calendar data.

#4 - One thing about IMAP Accounts using OL'2013/OL'2016 is that you really want to pay attention to which data file is set as the "default" for the Outlook profile. If the IMAP account was/is <either> the first account configured or is the <only> account configured (which by definition means it was also the first) - contacts will be stored in the IMAP .ost file and will have the words "This Computer Only" in the folder description. In a nutshell, if you're OST file ever gets separated from the Imap account that created it - you will lose all the data in any folder marked with "This Computer Only".

Going to stop here - you first need to locate your old PST file and can then go through your various options once that's done.

jay freedman say:

The first question you need to ask yourself is whether the form contains legacy form fields, which is the only reason to restrict text "for filling in forms". If there are no legacy form fields -- for example, if the form uses only content controls to accept input -- then don't use that type of restriction. Instead, you can use one of two other kinds:

  • Use the "No changes (Read only)" type of restriction. Then select the areas (such as content controls and the places where the hyperlinks will be inserted) and check the Everyone box in the Exceptions section of the pane.
  • If you are using content controls, insert a Rich Text control at the place where the hyperlink will be added. Then select the whole document and click Developer > Group > Group. That protects all the text outside the content controls to prevent editing and formatting.

If you must use legacy form fields (because some users will have Word 2003 or earlier, or Word for Mac), you can insert a continuous section break before the hyperlink entry point and another break after the hyperlink. (This will force the hyperlink entry point to be in its own paragraph.) In the Restrict pane, after selecting "Filling in Forms", click the "Select sections" link that appears below the dropdown. In the dialog that opens, uncheck the section between the two section breaks, and click OK. Start enforcing protection. The user will be able to insert a hyperlink in the space between the section breaks.

Jump Lists - Microsoft Community

ronst. peter say:

Jump Lists

After upgrading to Office 2016, the Jump Lists for Excel and Access are not working. I am usin Windows 10

a. user say:

Hello Ron,

Thank you for posting your query in Microsoft Community.

Try the below method and check if this fixes the issue:

     1. Open Windows Explorer.

     2. Type or copy and paste the following into the address bar:

         %AppData%\Microsoft\Windows\Recent\AutomaticDestinations

      3. Identify files that are comparatively larger in size than the rest of the file, such as files that have file size of 1 MB or more.

     4. Delete the abnormal file.

The functionality on Jump Lists will be restored. But, it doesn't restore the previous pinned items. So, you have to pin the new items again.

Hope this information helps. Feel free to contact us again, if you have any other queries.

Thank you.
