Page Numbering in Word 2003. - Microsoft Community

renglert30 say:

Page Numbering in Word 2003.

I have a document that requires a page number in the footer 1, 2, 3, etc.  I also require a page number in the header that continues from a previous document in the set, 247, 248, 249, etc.  I've tried the AutoNum and Seq fields but they can only be used in the main document an every time I try and put in a Page field in the header it defaults to the footer and overwrites the 1,2,3, numbers.  How can I accomplish this?

 

Richard

jay freedman say:
You'll need a "nested field", that is, one field inside the code of another field.

Click View > Header and Footer to open the header pane. Type this text there:

   = 246 + PAGE

(I'm assuming from your post that 246 is the last page number in the previous document.)

Select just the word PAGE in that text, and press Ctrl+F9. That adds a pair of field braces around the word, like this:

   = 246 + { PAGE }

Now select everything from the equal sign to the closing field brace, and press Ctrl+F9 again. The result should look like this:

   { = 246 + { PAGE } }

Finally, press F9 to update the field, which will show the value 247. Close the header area, and you'll find the next page shows 247, and so on.

Sum Horizontally - Microsoft Community

excelhelps say:

Sum Horizontally

Hello all,

 

I have an Excel file with values that are displayed horizontally, and I would like to sum those values per category.  Because of the volume of data that I have in the file, I am not able to transpose them vertically so that I can sum them using the formula "SumIf". 

 

Below please find an example of my data:

 

Cat           Dog        Lion            Tiger

1               1             1                  2

3               1             4                  5

7               0             2                  6

 

Using the above example, I would like to be able to write a formula to sum the number of each animal using the animal reference.  If the above data are displayed vertically, I would be able to use the formula "Sumif(range, "Cat", [Sum Range])", and I would get a total of 11 for Cat. 

 

Could you please help me with the formula?

 

Thanks.

rik_uk say:

This will work based on the folowing assumptions

Row 1 is the top of the table with labels

Data starts in Column A with no gaps/blanks inbetween columns of data

Data starts in Row 2 and is filled down, with no blanks between rows of data

All columns of data are the same length...

Put the formula in G2, and type the item to sum in F2 (eg Cat)

 

=SUM(OFFSET(A1,1,MATCH(F2,$A$1:$D$1,0)-1,COUNTA(A:A)-1,1))

 

If the table is in a different area you can modify the equation to suit. This will automatically dimension for any lenght of table.

g north say:

Hi

A solution to your problem could be the following formula:

=SUM(OFFSET(A2,0,MATCH("Cat",$A$1:$D$1,0)-1,3,1))

 

Where A2 is the 1 below Cat, A1:D1 the range Cat to Tiger and 3 the number of rows to sum.

jeeped say:
The text string False is not the same thing as a true boolean FALSE value. If you are looking to compare to the text string False then your formula (e.g. =$O2="False") should be correct. However, if you are comparing to a true boolean FALSE value, then the formula should be =NOT($O2).

The other thing that concerns me is that you've mentioned that the Applies to: is =$N:$N but your formula starts computing at row 2 (e.g. $O2). I believe you will get more consistent results if the formula is =$O1 and Applies to: =$N:$N. The formula should refer to the row in the top-left corner of the Applies to: range; in this case that would be row 1 in the full column reference of $N:$N.

One or boh of those things should improve the validity and consistency of your conditional formatting.

Word 2007: Mail Merge and Getting Rid of Spaces - Microsoft Community

oceanmist say:

Word 2007: Mail Merge and Getting Rid of Spaces

I am preparing labels via a mailmerge from an Excel spreadsheet. The first line of the label is similar to:

Mr. John Doe

Some names do not have a courtesy title (Mr.) and would appear as "George Smith"; however, there is an extraneous space before the first name when there is no courtesy title. It is actually the space between the courtesy title and the first name. How do I get rid of it in the labels?

doug robbins - mvp office apps & services (word) say:

Use an If...then...Else construction

{ IF { MERGEFIELD Title } <> "" "{ MERGEFIELD Title } { MERGEFIELD firstname }" "{ MERGEFIELD firstname }" }

You must use Ctrl+F9 to insert each pair of field delimiter { } or you can insert the various mergefields and text as follows

IF «Title» <> "" "«Title» «firstname»" "«firstname»"

and then select that and press Ctrl+F9

and then Alt+F9 to toggle off the display of the field codes.

Hope this helps.
