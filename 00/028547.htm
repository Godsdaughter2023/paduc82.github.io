jay freedman say:

For one thing, the 5160 and 8160 product numbers are in Word in Office 365, as they have been for more than a decade. The "trick" is that the list of numbers is in "alphabetic" order rather than "numeric" order -- all the numbers that start with the same digit are grouped together regardless of how many digits there are.

With either Word's 8160 label or the Avery template you downloaded, you need to change the way you're inserting address blocks. Don't manually add another address block in the next cell after the first one. Just insert the address block in the first cell, and then click the Update Labels button:

That will add the «AddressBlock» merge field to all the other cells. If you want to see what the merged labels will look like before starting the actual merge, click the Preview Results button. Be sure to turn the preview OFF before you click the Finish & Merge button.

doug robbins - mvp office apps & services (word) say:

When you select Labels from the Start Mail Merge dropdown and the Label Options dialog appears, make sure that the Page Printers radio button is selected and that you have Avery US Letter selected from the Label vendors dropdown.  Then you WILL find the 5160 Address Labels

and also the 8160 if you scroll further down the list

While it does not matter which one you use, it will be easier to use either of them than the template that you obtained from Avery as it will not be setup with the necessary «Next Record» fields in all of the labels on the sheet EXCEPT for the first label.  

If on the other hand, you want to continue with the document that you have, you will need to insert the «Next Record» fields from the Rules dropdown in the Write & Insert Fields section of the Mailings tab of the ribbon.

Note, once you have set up the necessary merge fields in the first label on the sheet, you can use the Update Labels button to replicate that setup to the other labels on the sheet.

Some common reasons why Outlook will not shut down completely when you click either the X or Exit:

1.  PDA synchronization software with your PDA in the cradle. (activesync,
iTunes, etc.)
2.  WinFax Pro (especially 10.02 in Outlook 2003) - contact their support
for an alleged fix.
3.  COM Add-ins.
4.  Mail reminder add-ins that keep a stub of Outlook open in the background
to check for new mail and fire reminders.
5.  Anti-virus/firewall programs that are set to scan incoming and outgoing mail (always disable email scanning).
6.  Skype
7.  Infuzer
8.  Google calendar synch.

Disable all COM addins and remove the integration with your firewall and AV.

Open Outlook in Safe mode and start/stop Outlook every time after adding
back in your add-ins one by one.  The first one you have enabled and the
error message occurs again, you know what to do.

joeu2004 say: Stephen wrote:

I have been using the '=COUNTIFS' function on excel, and the formula seems ok but it's not giving me the right answer, I know there is two rows with the details I asked, but it came back with 0 for some reason.   The formula is as follows
=COUNTIFS(Quarter,"Q1",FinancialYear,"2012/13",AA5:AA19,
"'Data for D.D.Menus for Page 1'!$A$25",AC5:AC19,
"'Data for D.D.Menus for Page 1'!$B$33,'Data for D.D.Menus for Page 1'!$B$37,'Data for D.D.Menus for Page 1'!$B$38")


First, you have too many double-quotes.  If want to count when each of AA5:AA19 matches the value in $A$25 of worksheet 'Data for D.D.Menus for Page 1', do not put double-quotes around the latter cell reference.  The correct syntax is:

=COUNTIFS(Quarter,"Q1", FinancialYear,"2012/13",
AA5:AA19,'Data for D.D.Menus for Page 1'!$A$25, ...)

where "..." is the rest of the COUNTIFS parameter list.

Second, something is missing or garbled after that.  After you remove the incorrect use of double-quotes, the following syntax is
=COUNTIFS(..., AC5:AC19,
'Data for D.D.Menus for Page 1'!$B$33,'Data for D.D.Menus for Page 1'!$B$37,'Data for D.D.Menus for Page 1'!$B$38)

It is unclear what your intention is.  I suspect you want to count when each of AC5:AC19 matches
If that is the case, you cannot use one COUNTIFS expression to do that.  Instead, you might use the following SUMPRODUCT expression:

=SUMPRODUCT(--(Quarter="Q1"), --(FinancialYear="2012/13"),
--(AA5:AA19='Data for D.D.Menus for Page 1'!$A$25),
--((AC5:AC19='Data for D.D.Menus for Page 1'!$B$33)
+(AC5:AC19='Data for D.D.Menus for Page 1'!$B$37)
+(AC5:AC19='Data for D.D.Menus for Page 1'!$B$38)>0))

Caveat:  Beware of minor syntax errors on my part.  I have not actually entered that expression into Excel.

Note:  The double negative converts TRUE and FALSE to 1 and 0 respectively, which SUMPRODUCT requires in order for the expression to work as intended.

any arithmetic operation converts TRUE and FALSE to 1 and 0, which is why (AC5:AC19=...$B$33)+(AC5:AC19=...$B$37)+(AC5:AC19=...$B$38) works as intended, returning 1 or more if one or more conditions are true.
