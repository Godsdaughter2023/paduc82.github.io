MicroSoft Access 2019 Multiple entries for same ID entry

Hi!

I'm trying to build a simple database that will enable people to catalogue research findings.

So, in other words, say someone is looking up "Dogs with pink toenails" and there are two websites that have this.  (There could be 5, there could be 10 , there could be one.) the point is that I want someone to be able to enter "Dogs with Pink Toenails" into a record, and then for that same record be able to enter in as many URL's (or descriptions, or images, or notes) as needed. I thought I remember doing this in an older version but can't remember what I did.

Thanks for the help!

Mike

scott mcdaniel say:

You'd need a "Header" table and a "Detail" table. Your Header table would look something like this:

HeaderID - AutoNumber

HeaderValue - ShortText

In that table you'd enter the "Dogs with Pink Toenails" entry, and Access would assign that an Autonumber value

Your Detail table would look something like this:

DetailID - Autonumber

HeaderID - Long Integer, foreign key to your header table record

DetailValue

Your Detail value table would contain all the distinct instances of websites that contain the Header value, and each of those records would store the HeaderID value from the Header table.

You'd represent this in Access using a Form-Subform setup. Your Form would be based on the Header table, and your Subform would be based on the Detail table. Create each of those forms independently, then drag the subform into the main form, and set the master-child link fields correctly, and you should be good to go.

tml48324 say:

It looks like I've solved the problem.  

Hopefully, this will help someone else in the future.  (Assuming they have the same "problem")

I did a bunch of testing with Outlook (desktop), Outlook Online (Office 365), Tasks (desktop and online), etc.  

Every time I cleared a flag, it would come right back.  It eventually occurred to me that SOMETHING was fighting with Outlook in some capacity because there was a short lag between clearing a flag and it popping back up as flagged again (or completed) in my inbox.  

In the end, I determined that it was Microsoft's new To-Do app that was causing the "problem."

I had tried the program out when it was first introduced and was unaware that it was still "working" in the background.  

So even though I was clearing flags in Outlook (both in my inbox and tasks), To-Do was persistently adding the flags back into the system.  

Once I deleted the task/follow-up in Microsoft To-Do, the flag finally cleared.  

For anyone else facing this issue, or just working with To-Do in general, I WAS able to confirm that deleting the Task/Follow-Up in To-Do (not the email) did NOT delete the flagged email itself. (something I was concerned about).     It simply removes the task and then the email goes back into my Outlook inbox based upon the date the email was originally received.  

As an aside, the "glitch" is still persistent in that To-Do and Outlook seem to communicate with each other just fine with the limited exception that Outlook "clear flag" commands do not flow over to To-Do.   As such, I still have to go over to To-Do and then delete the task/follow-up so that it will not appear as a green checked "completed" item in my Outlook inbox.  

Maybe there is a setting to make that work better, but I have not discovered it yet.  If not, this is something that should go on Microsoft's radar to fix.  

Thank you to all who responded in helping me to resolve this issue.
