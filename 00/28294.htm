nvmaxine say:

Sorry for the delay. I took a deep breath and looked at it from another angle.

I created a hidden field Text1 on the Subform ProdInfo and gave it a value. Then I was able to transfer that value to the field Text1 on the Subform AddlInfo in the OnEnter of the Subform AddlInfo from the main form CertInput. It is now working AOK.

Private Sub Lot_No_Exit(Cancel As Integer)

If Forms![CertInput]![NewText] = "Yes" Or Me.NewRecord Then
    Me![Heat No] = DLookup("[Heat No]", "Lot Nos", "[Lot No]= Forms![CertInput]![Subform ProdInfo]![Lot No]")
    If Not IsNull(Me![Heat No]) Then
        Me![HeatDFARS] = DLookup("DFARS", "Heat Nos", "[Heat No]=Forms![CertInput]![Subform ProdInfo]![Heat No]")
        If Me!HeatDFARS = True Then
            Me!Text1 = DLookup("Text1", "AddtlInfoItems", "[Info]=""DFARS""")
        Else
            Me!Text1 = Null
        End If
    End If
End If

End Sub

Private Sub Subform_AddlInfo_Enter()

If Forms![CertInput]![NewText] = "Yes" Or Me.NewRecord Then
        Forms!CertInput![Subform AddlInfo]!Text1 = Forms!CertInput![Subform ProdInfo]!Text1
End If

End Sub

daniel pineault say:

There is nothing to fix.  You may have only created 30 tables, but there are other system and hidden tables that Access uses.  The TableDefs.Count property is a read-only property which simply return the object count related to the number of tables within the queried db.

You could try a query such as:

SELECT MSysObjects.Name
FROM MSysObjects
WHERE ((MSysObjects.Type=1) OR (MSysObjects.Type=4) OR (MSysObjects.Type=6))
ORDER BY MSysObjects.Name;

To get a listing of all the tables within the current db.

What is also a little odd is that CurrentDb.TableDefs.Count does not appear to return a proper count as it counts some system tables, yet omits others.  For instance, in my tests (Access 2013) CurrentDb.TableDefs.Count does not include:

f_72DC15AD7C104BFE9ABC43CF6F539822_Data

MSysComplexType_Attachment

MSysComplexType_Decimal

MSysComplexType_GUID

MSysComplexType_IEEEDouble

MSysComplexType_IEEESingle

MSysComplexType_Long

MSysComplexType_Short

MSysComplexType_Text

MSysComplexType_UnsignedByte

but the above suggested query SQL statement returns a listing of all the tables.

Furthermore, if you want to see where CurrentDb.TableDefs.Count is getting a count of 52 from you could use a function such as:

Function ListTbls()
    Dim i                     As Long
    For i = 0 To CurrentDb.TableDefs.Count - 1
        Debug.Print CurrentDb.TableDefs(i).Name
    Next i
End Function

hbill1 say:

Formula bar shows content that does not appear in the cell

I have a column of cells with dates but the formula bar indicates that there is a time stamp among the content. For example:

a cell with "07/11/2017" shows "7/11/2017  4:54:46 PM" in the formula bar. I've tried truncating the content, but when I paste the values, the time stamp is back. 

vijay a. verma say:
It is showing only date as cell is formatted as date. Hence, it not showing you the time part in cell.

If I interpret correctly, you want to see only date part in both cell as well as formula bar. In this, case you will need to retrieve date part in another column and paste this as values back into original column.

Put this kind of formula and drag down

=DATE(YEAR(A2),MONTH(A2),DAY(A2))

Now paste this result column as values into original column to retain the date only.

hbill1 say: Thank you! I also figured out that if you select the cells, go to the Data tab, and select Text to Columns, you can separate the delimited content by space. That will make separate columns for the date, time and "PM."
