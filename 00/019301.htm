ken sheridan say:
If you only count whole years, so that 364 days is still not a year of service, then DateDiff("yyyy", [StartDate], Date()) would give you that.

Not necessarily, Dirk.  It depends on the relative positions of the dates within the calendar year.  It's the same as returning someone's current age in years, for which one of the methods is that in the following function:

Public Function YearsElapsed(varDateFrom As Variant, Optional varDateTo As Variant) As Variant

    If IsMissing(varDateTo) Then varDateTo = VBA.Date
   
    If Not IsNull(varDateFrom) Then
        YearsElapsed = DateDiff("yyyy", varDateFrom, varDateTo) _
            - IIf(Format(varDateFrom, "mmdd") > Format(varDateTo, "mmdd"), 1, 0)
    End If
   
End Function

Edited to allow for Nulls


john w. vinson say:

You don't, within a table. This kind of calculated field should not be stored in any Table at all, since the years of service will change over time.

Instead, calculate it on the fly, in a Query or in the Control Source of a form or report textbox, using the function that Ken posted.

Tables are intended for data storage - not for data presentation.
