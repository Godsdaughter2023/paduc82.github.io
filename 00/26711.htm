briancarter_uk say:

Editted 27th Feb 2013 with a better macro (keep cursor left/right-ness during single page moves).

 

This annoys me a huge amount too.  So I wrote a macro to fix it (tested in Word 2010 professional only).


1. View -> Macros
2. Create a new macro named 'CursorUp' in 'Normal.dotm (global template)'
3. In the body of the macro, paste the following:

<code>
Sub CursorUp()
'
' CursorUp Macro
'
'
startPageNumber = Selection.Information(wdActiveEndPageNumber)
Selection.MoveUp
nextPageNumber = Selection.Information(wdActiveEndPageNumber)
If startPageNumber - nextPageNumber > 1 Then 

    Selection.MoveDown
    Selection.HomeKey
    Selection.MoveLeft
End If

End Sub
Sub CursorDown()
'
' CursorDown Macro
'
'
startPageNumber = Selection.Information(wdActiveEndPageNumber)
Selection.MoveDown
nextPageNumber = Selection.Information(wdActiveEndPageNumber)
If nextPageNumber - startPageNumber > 1 Then

    Selection.MoveUp
    Selection.EndKey
    Selection.MoveRight
End If

End Sub
</code>

4. Assign these two macros (CursorUp() and CursorDown()) to your up and down keys (previously they were assigned to LineUp and LineDown.

Do this via:

 a. File->Options->Customize Ribbon->(Keyboard Shortcuts:Customize):

 b. Categories: Macros (near the bottom - not alphabetical)

 c. Commands: CursorUp()

 d. Press New Shortcut Key  <Cursor Up>

 e. Assign

 f. Repeat (with appropriate changes) for Down.

briancarter_uk say:
It works with one catch.  It won't go from page two to page one. And it won't from from the second-to-last page to the last page. It works for all the pages in the middle.
Replace the existing macros with these.  This fixes the problem in my set up (I hadn't tested that scenario...maybe I should work for Microsoft :)). The column you end up in is a bit irritating (if enough people ask for a fix I'll look into it more).

Sub CursorUp()
'
' CursorUp Macro
'
'
startPageNumber = Selection.Information(wdActiveEndPageNumber)
startLineNumber = Selection.Information(wdFirstCharacterLineNumber)
Selection.MoveUp
nextPageNumber = Selection.Information(wdActiveEndPageNumber)
nextLineNumber = Selection.Information(wdFirstCharacterLineNumber)
If startPageNumber - nextPageNumber > 1 Then
    Selection.MoveDown
    Selection.HomeKey
    Selection.MoveLeft
ElseIf nextPageNumber = startPageNumber And startLineNumber = nextLineNumber Then
    Selection.HomeKey
    Selection.MoveLeft
    Selection.HomeKey
End If

End Sub
Sub CursorDown()
'
' CursorDown Macro
'
'
startPageNumber = Selection.Information(wdActiveEndPageNumber)
startLineNumber = Selection.Information(wdFirstCharacterLineNumber)
Selection.MoveDown
nextPageNumber = Selection.Information(wdActiveEndPageNumber)
nextLineNumber = Selection.Information(wdFirstCharacterLineNumber)
If nextPageNumber - startPageNumber > 1 Then
    Selection.MoveUp
    Selection.EndKey
    Selection.MoveRight
ElseIf nextPageNumber = startPageNumber And startLineNumber = nextLineNumber Then
    Selection.EndKey
    Selection.MoveRight
End If

2) The section 2 header showed the text of the Section 1 header, copied: "Header -- Section 1."

3) Using Backspace or Delete, she got rid of the part that was wrong: "Header -- Section [end of text]"

4) Then she typed in the `2': "Header -- Section 2"

5) Then she clicked on "Exit editing headers and footers" --

 

only to discover that all her other sections' headers were ruined. I was doing something similar with my last section's odd-pages header, and getting the same frustrating result.

greginmich say:

The MSRT won’t normally make any legitimate detections on a PC that’s running a supported OS and is protected by an up-to-date AV app – unless maybe the user hasn’t run a manual (or scheduled) scan for quite a while. In that case, it’s possible that your AV app might have missed some archived malware, since only a few AV apps provide the option for scanning archived files with real-time protection. But you should search for hidden (archived) malware by just running occasional Full Scans with Windows Defender Antivirus or with your third-party AV app. This will unpack and scan the contents of archive files just like the MSRT does – but it will also scan your files with the full Microsoft definition set (or the full third-party definition set) instead of just the small subset of definitions that are included with the MSRT. The MSRT really just isn’t up to the task of finding hidden malware, because it's blind to more threats than it can see. And when this archaic tool starts running forever and detecting threats that aren’t really there – well then maybe it’s time to move on.

If you’re serious about catching malware, then you’ll need to perform your manual scans with your installed AV app – or better yet, with Windows Defender Offline, which scans with the full Microsoft definition set, and which also runs outside of the Windows environment, which means that it doesn’t give threats the opportunity to conceal their presence by manipulating the Windows internals. Of course none of this would be necessary if we just had a reliable scheduled scan that would run our on-demand scans for us. Unfortunately, if you’re using Windows Defender Antivirus, then you probably don’t have a reliable scheduled scan (that’s a long and tiresome tale). But if you’d like to totally eliminate the need for manual scans, then you can set up a reliable scheduled scan with one of the methods here:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/windows-defender-how-to-schedule-a-scan/d4ba70ce-a70c-4d82-bb26-18902b4e8f9e

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/scans-using-defender/f9bc87fb-7c80-4dd6-9ed4-c6ffcbcaffff

While running your manual (or scheduled) scans with the installed AV app is a step in the right direction, you have to keep in mind that running your on-demand scans with the same AV app that’s providing your real-time protection will apply exactly the same set of definitions; and this means that the malware that a manual scan finds will generally be limited to inactive and archived malware, or maybe some fragments left over from a previous cleanup. So if you want to be able to find threats that aren’t being detected by your primary AV app, then you’ll need to run periodic scans with some third-party malware-removal apps. If you're using Windows Defender, then these tools will frequently include signatures that are missing in Microsoft’s “comprehensive” signature set – which actually doesn’t even comes close to covering the entire spectrum of online threats. And these third-party apps will consequently have the potential for detecting active malware that’s running on the system just because Defender wasn’t able to detect it. And on the other hand, if you’re using a third-party AV solution for your real-time protection, then exactly the same principle applies – you should be using Windows Defender’s Limited Periodic Scanning mode; and also running scans with these third-party on-demand AV apps, in order to see what’s being missed:
