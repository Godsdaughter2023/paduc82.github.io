      andyheaton say:

      Can I activate File Compression on a File History HDD?

      Hi, I have a 1.5TB back-up (File History) dedicated external hard drive.
      Recently, with more and more data, the drive is fast filling up and is now past half way capacity.
      I really don't want to have less backups (i.e. reducing the number of historic file copies).

      So, the question:

      Can I turn on file compression on the backup HDD without affecting the backup files & backup process.

      a. user say:

      Hi Andy,

       

      You can compress the data on your External hard drive to have more space, however you may have performance issue while trying to access those files as the computer will decompress them first, read them and then decompress them. It might cause data corruption as well. But as the hard disk space is more, you can go ahead.

       

      Let us know if you have any further queries related to the issue. We would be happy to help you.

      fierydragonlord say: File History simply stores copies of the files directly on the disk, tracked by a few configuration files. Compression won't interfere with File History, and modern computers have plenty of processing power so it shouldn't negatively impact performance. In fact, since the disk itself is probably the performance bottleneck, compression may actually increase performance.

      N.B. After you apply NTFS compression to the entire drive, be sure to defragment it (use the Optimize Drives tool built into Windows, or a third-party program such as Piriform Defraggler [note: no affiliation, just my personal preference]). In my experience, compression tends to cause a lot of file fragmentation on the drive, so defragmentation should increase performance when accessing files from the history.
