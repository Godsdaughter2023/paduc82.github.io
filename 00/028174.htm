I'm unclear exactly what you're asking about...but here are some things to understand.

 

The XBOX hub on Windows Phone is tied directly to the Microsoft Account used to set up your Windows Phone.  If you do NOT have an XBOX Live account, then no connection is made.  If your Microsoft Account is part of a Family account, then the messages you are receiving indicate that you are either underage to do the activity you are trying to do, or, that activity is not part of the allowable action.  For example Family accounts and XBOX Music Pass do not allow members of the Family account access.   This you will have to figure out with your family. 

 

If you are simply trying to play games using the XBOX Hub, consider this.  The XBOX Hub is simply a Games Hub catering to XBOX.  If you do not have a XBOX Live Account, then you cannot take part in XBOX Live specific things, like gamer profiles, game invitations, etc.  But you can still download games and play them from the Games Hub. 

 

If you wish to become part of XBOX Live, then you should consider purchasing a XBOX Live account.  Ideally you would do this using the Microsoft Account name you used to setup your Windows Phone.  This would tie all activities together under a single location.

 

Another thought. 

 

When you set up your Windows Phone, did you use your Microsoft Account?  If you did not, then all associations between accounts would not exist.  This could produce a variety of messages.

doug robbins - mvp office apps & services (word) say:

On the Margins tab of the Page setup dialog, set the left margin to 20mm and the Right one to 6mm, leaving the Multiple pages: setting at Normal.

Then on the Layout tab of the Page setup dialog, check the Different odd and even box under Headers and Footers and then anchor a text box in the Even Page Header, with the Text Wrapping of the text box set to Tight and with the text box extending right down the right hand side of the page, and with its width set so that the text wraps at the 14mm from the right edge of the paper position.  Remove any border from the text box so that it will not appear when the document is printed.

Add attachments to email generated by Access 2007

I have Access generate an Excel file daily ( 01Jan12 Data.elsx).  This file gets sent to other users ever day. I'm trying to automate this process and generate the email and add the current file as an attachment.  I have been able to generate an email using "DoCmd.SendObject" which works for .pdf made by Access, but it will not attach a previously generated file.

hansv mvp say:

Here is a function you can use

 

Function SendMail( _
    Recipient As String, _
    CC As String, _
    BCC As String, _
    Subject As String, _
    Message As String, _
    Attachment As String, _
    Optional ShowMail As Boolean) As Boolean

    Dim objOL As Object
    Dim objMI As Object
    Dim blnNotActive As Boolean
    Dim Recip As Object
    Dim Counter As Integer
    Dim arr

    SysCmd acSysCmdSetStatus, "A moment please. Your e-mail message is being sent."
    DoCmd.Hourglass True

    ' Check whether Outlook is active
    On Error Resume Next
    Set objOL = GetObject(, "Outlook.Application")
    blnNotActive = (Err <> 0)

    If blnNotActive Then
        ' If not, we start Outlook
        Err.Clear
        Set objOL = CreateObject("Outlook.Application")
    End If

    On Error GoTo Err_Mail

    ' Create e-mail message
    Set objMI = objOL.CreateItem(0) ' 0 = olMailItem
    With objMI
        arr = Split(Recipient, ";")
        For Counter = 0 To UBound(arr)
            Set Recip = objMI.Recipients.Add(arr(Counter))
        Next Counter
        arr = Split(CC, ";")
        For Counter = 0 To UBound(arr)
            Set Recip = objMI.Recipients.Add(arr(Counter))
            Recip.Type = 2 ' 2 = olCC
        Next Counter
        arr = Split(BCC, ";")
        For Counter = 0 To UBound(arr)
            Set Recip = objMI.Recipients.Add(arr(Counter))
            Recip.Type = 3 ' 3 = olBCC
        Next Counter
        .Subject = Subject
        .Body = Message
        .Attachments.Add Attachment, 1 ' 1 = olByValue
        If ShowMail = True Then
            .Display
        Else
            .Send
        End If
    End With
    SendMail = True

Exit_Mail:
    ' Release object memory
    On Error Resume Next
    If Not (objMI Is Nothing) And Not ShowMail Then objMI.Close 1 ' 1 = olDiscard
    Set objMI = Nothing
    If blnNotActive And Not (objOL Is Nothing) And Not ShowMail Then objOL.Quit
    Set objOL = Nothing
    SysCmd acSysCmdClearStatus
    DoCmd.Hourglass False
    Exit Function

Err_Mail:
    SendMail = False
    Resume Exit_Mail
End Function

 

You can specify multiple Recipient, CC and BCC addresses separated by semi-colons.

Example of usage:

 

Sub TestMail()
    If SendMail(Recipient:="firstrecipient;secondrecipient", _
            CC:="otherrecipient", BCC:="", _
            Subject:="Test", Message:="Hello World", _
            Attachment:="C:\Code.txt", ShowMail:=True) = False Then
        MsgBox "Something went wrong!", vbExclamation
    End If
End Sub

 

where firstrecipient etc. are e-mail addresses (it is not possible to post real-looking e-mail addresses)
