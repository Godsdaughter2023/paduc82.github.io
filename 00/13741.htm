jay freedman say:

Suzanne's suggestion is correct, but it needs some additional details.

First, a discussion of layers in Word:

  • Floating objects, such as your pattern graphic, can be in layers in front of the text of the body of the document, or in layers behind that text.
  • If the cursor is in a header or footer when you insert a floating object (which anchors the graphic in the header/footer), that object can be in a layer in front of the header/footer text or behind it.
  • The fact that's missing here is that every layer in the header/footer is behind any layer in the body text. So even though you made your pattern "behind text" in the body text, it's still in front of the text in the header. That's what you have to fix, by cutting the graphic from the body, clicking in the header, and pasting.

However, that involves another characteristic of Word documents, that the header (or footer) in each section of a document defaults to being linked to the one in the previous section.

If you simply paste a graphic (or text or anything else) in the header of one page, it will appear on all the pages unless you first turn that page into its own section and break the links to the preceding and following sections.

To do that, click at the end of the page before one that has a pattern graphic, and click Page Layout > Breaks > Next Page. Then go to the end of the page that has the graphic, and again click Page Layout > Breaks > Next Page. Next, click Insert > Header > Edit Header to open the Header & Footer Tools tab on the ribbon, and click the Link to Previous button to turn it off. Now you can paste the graphic into the header, and it will affect only the single page.

The pattern graphic will appear dimmed whenever the cursor is in the body text, but it will print as normal.

Macro and Dcount - Microsoft Community

cyberclive say:

Macro and Dcount

Hi

I have a database, and have created a macro to count the number of active tickets but get a Data Type Mismatch in criteria expression.

The macro I created is a DCount if Field One a technicians name and Field two the status of the ticket match then count, I am sure it is something simple. (Below I have changed name to user1)

=DCount("[Assigned To]+[Status]","Tickets","'User1'"+"'Active'")

any help would be appreciated

Clive

ken sheridan say: You should express the two criteria separately within a Boolean AND operation.  The asterisk character can be used as the first argument to return the number of rows which match the criteria:

=DCount("*","Tickets", "[Assigned To]='User1' And [Status] ='Active'")

Bear in mind that personal names can contain apostrophes.  My name in its original non-anglicized form for instance is Cináed O'Siridean.  To cater for this each literal quotes character can be represented as a contiguous pair of quotes characters like this:

=DCount("*","Tickets", "[Assigned To]=""User1"" And [Status] =""Active""")
