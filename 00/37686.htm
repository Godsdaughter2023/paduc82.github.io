wlmail.exe continues to run after the program is closed in order to conduct its internal housekeeping. Any transactions that have not been written to the database while the program was in use are written after the user interface has disappeared. If you interrupt this process, for example by killing the wlmail.exe process or closing down Windows, you run the risk of corrupting the database. The database will often be able to recover from such a corruption, but repeated violations will undoubtedly make it more difficult.

Last question first :

Junction Points :
excerpt :
2/2007 - A Junction Point is a physical location on a local hard disk that points to another location on that disk or another storage device. It is essentially a shortcut, thus the shortcut icon on Junction Point folders in Windows Vista.

For the most part, you can ignore Junction Points for everyday use of Windows Vista. The Junction Points are in Vista forbackward compatibility of pre-Vista applications. Pre-Vista applications (applications written for XP and 2000) look for folders that are no longer used in Vista. Folders with the “My” prefix such as My Documents, My Pictures, and so on. There are also folders in your user profile that have been changed in Vista, and pre-Vista apps might be looking for the old folder structure during installation. If youdon’t set the option to Show Hidden Files in Windows Explorer, you will rarely even see the Junction Point folders.

source : http://www.svrops.com/svrops/articles/jpoints.htm

How to hide protected operaing system :
Start button > in the search box, type folder options > View tab > scroll down to find " Hide protected operating system files ( Recommended ). check it > OK
