ken sheridan say: To filter the report by means of the WhereCondition argument of the OpenReportMethod you'll need to open it transparently in print preview , select the report object and print the first page with the PrintOut method of the DoCmd object:

On Error GoTo Err_Handler

     Const DOCNAME = "Labels College Contact"
     Dim strWhere As String

     strWhere = "[ID]=" & Me!ID
    
    Application.Echo False
    DoCmd.OpenReport DOCNAME , View:=acViewPreview, WhereCondition:=strWhere
    DoCmd.SelectObject acReport, DOCNAME
    DoCmd.PrintOut acPages, 1, 1
    DoCmd.Close acReport, DOCNAME

Exit_Here:
    Application.Echo True
    Exit Sub
    
Err_Handler:
    MsgBox Err.Description
    Resume Exit_Here

Note that it's important to turn screen painting back on after resuming from any error, or it would remain turned off after the procedure executes.

If a report is restricted by means of the WHERE clause of its underlying query referencing a control on a form as a parameter, rather than by
    DoCmd.SelectObject acReport, DOCNAME, True
    DoCmd.PrintOut acPages, 1, 1

Hello jaygourley

Short explanation

Win10 Mail App and Outlook Desktop client (part of Office suite - any version) are two <email clients> which are completely separate from each other. Each <email client> connects to one or more <email accounts>. There would be absolutely no reason as to why you would want to use both - leastwise not from anything in your question/description.

>>
Is there any danger it will erase old items from my Exchange mailbox?
<<

Any <email account> that is configured with the same Exchange <email account> will be working on the same data. Since the Exchange protocol is a bi-directional sync between server and <client> - what is done on one side is reflected on the other. In other words, if you delete an item using Outlook desktop, it gets deleted from the server (and vice versa). When you access the same account via the WIn10 Mail app - that same deletion gets reflected there.

>>
- Should I disable it altogether?
- How do I do that?
<<

If you haven't configured the account - it's not "enabled" so either remove the account from Outlook Desktop or from Win10 Mail (personal suggestion would be to keep Outlook desktop).

Remove the account from Win10 Mail by opening it, click on <Accounts>, click on the email account in question, select remove from this device.

>>
Is there a way to make Windows share requests link to Outlook instead of to the Windows mail app?
<<

Not at all sure about what you're asking in terms of "....make Windows share requests to Outlook............." - specifically as it relates to "Windows share requests" - what share requests are you referring to or do you just mean adding/deleting things like email, contact, calendar items?

regsvr32 wab32.dll
