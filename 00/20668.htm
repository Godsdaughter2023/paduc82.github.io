brian tillman [outlook mvp 2007-2019] say:
It's typical for mail providers to say, "It's not our fault" and just because your phone can connect, that doesn't really say a lot about the desktop being at fault. What the message means is that Outlook contacted the SMTP server, the server answered, Outlook sent an SMTP command to the server and the server never acknowledged it. Outlook gave up after its timer expired. Alternatively, Outlook could have made its initial connection attempt to the server but the server never responded.

Typically, this is an intermittent condition, but if you have a third-party Internet Security suite installed, that's often the cause of this type of error. Just for testing purposes, disable any AV or internet security suite and try again. That will, perhaps, allow you to narrow down the issue.
