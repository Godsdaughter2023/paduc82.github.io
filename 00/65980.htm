The Code 10 error code is generated when Device Manager has insufficient information or the error that is bubbled up by the device drivers is not recognized by Device Manager. vectorcities say:

Xbox app doesn&#39;t detect Xbox One controller

I use an Xbox One controller (the old model) connected by micro USB with a lot of PC games, but when streaming from my Xbox One using the Xbox app I always get a message saying there's no controller connected to the PC. I checked in the Device Manager and sure enough it's connected and working fine. The Xbox app and the Xbox Accessories app just won't detect it. Anybody have any ideas?

fallibleodin70 say:

1) Are you on Windows 10? I may be mistaken, but I think the Xbox Accessories universal app is only intended to work on Windows 10 PCs and Xbox One. If the app doesn't work on Win8.1 and it's still available to install on Win8.1 via the Store, it may just be an oversight by MS.

2) If you check the controller's entry in device manager, what is the date for the driver currently in use? Perhaps it's not pulled the latest driver from 10/29 yet and the XA app won't see the controller until that's up to date? If it's not this latest driver, just try to update the driver via Device manager, it should pull any never drivers from Windows Update.

3) Is the controller plugged into a USB 2.0 or 3.0 port on your computer? If plugged into a 3.0 port, try a 2.0 port if available. I highly doubt this would cause any issues, but I know it potentially can. (For example, I do a lot of modding on Android devices, and this was often a source of pain when using adb) Just something to consider as a last-ditch effort.

Lastly, what in particular do you want to do with the controller with the XA app on PC? Controller firmware updates can be done on your Xbox One, and I *think* most of the customization the app touts is exclusive to the Elite controller.

cy43rguru say:

In Device Manager navigate to the controller, select it then select View->Devices by Connection. You should see an expandable tree with: Xbox Controller->USB Input Device->XINPUT compatible HID device.

My suspicion is that if it only shows Xbox Controller with no children you may have a corrupt/bad device stack. Look for any errors in C:\windows\inf\setupapi.dev.log, searching from the bottom of the file up, or search for the last instance of dc1-controller.sys. If you can see an issue or error, please paste that information here.
