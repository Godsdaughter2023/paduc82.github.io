shyamal debnath - msft say:
Hi Leo,

When a DLP policy looks for a sensitive information type such as a credit card number, it doesn't simply look for a 16-digit number. Each sensitive information type is defined and detected by using a combination of:

Keywords
Internal functions to validate checksums or composition
Evaluation of regular expressions to find pattern matches
Other content examination

This helps DLP detection achieve a high degree of accuracy while reducing the number of false positives that can interrupt peoples’ work.

For example, the sensitive information type named Credit Card Number is defined by two patterns:

• A pattern with 65% confidence that requires:

A number in the format of a credit card number.
A number that passes the checksum.

• A pattern with 85% confidence that requires:

A number in the format of a credit card number.
A number that passes the checksum.
A keyword or an expiration date in the right format.

You can use these confidence levels (or match accuracy) in your rules. Typically, you use less restrictive actions, such as sending user notifications, in a rule with lower match accuracy. And you use more restrictive actions, such as restricting access to content without allowing user overrides, in a rule with higher match accuracy.

For detailed information, please refer to the article: Overview of data loss prevention policies

For your reference, what the sensitive information types look for see under Credit Card Number

rod gill say: I think we have a conflict of jargon here. As far as Project Pro is concerned a Deliverable is a task published as a Deliverable in Project Server by Project Pro. You cannot therefore insert a Deliverable in Project Pro without Project Server.

Without Project Server, a Deliverable is typically a milestone with a task description that matches the name of a deliverable in the Business Case. You may also flag the milestone as a deliverable using a custom flag field such as Flag1.

john - project say:
Rosie,
A "deliverable" has nothing to do with Project Server. If your deliverable is not represented by an "action", then it must be a milestone. For example, let's say you are writing a document - a book, a speech, a screenplay, etc. You may have multiple performance tasks described using action verbs that have a duration that will be dependent on the number or people working that task and the total effort content. When all the performance tasks are complete, you can deliver the end product - the document. All those performance tasks drive a milestone, which is a zero duration task with no resource assigned and therefore represents the "deliverable".

Hope this helps.

Hi,

 

After you concatenate the string constants you need to change the syntax of the saveAs line

 

 

Dim Wk As Workbook
  Dim Path As String
  Dim Stringconstant4 As String
  Dim Stringconstant As String
  Dim Stringconstant3 As String
  Dim Stringconstant2 As String
  Stringconstant = "Invoice Date"
  Stringconstant3 = " 04-2012 TN"
  Stringconstant2 = " (999) 999-8345"
 
 
  Stringconstant4 = Stringconstant & Stringconstant3 & Stringconstant2
  Path = "C:\Users\Public\Documents\AB\A 2012\Employee Email Workbooks\"
     Set Wk = Workbooks.Add
     Wk.SaveAs Filename:=Path & Stringconstant4 & ".xlsx"
