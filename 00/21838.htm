Hide Main Close Button - Microsoft Community

self untaught say:

Hide Main Close Button

Is there a way when you open a form to hide the main close application button? I want to force them to close the form, never close the app.
ken sheridan say: Another way you can control how a user closes an application is to open a hidden form at start-up, with the following in its Unload event procedure:

    If Not blnAllowExit Then
        Cancel = True
    End If

where blnAllowExit is a Boolean variable declared as Public in any standard module in the database.  You will then need to include a means for the user to close the application legitimately with the following code:

    blnAllowExit = True
    DoCmd.Quit

This will not hide the application's close button, but if the user clicks on it nothing will happen.

If you only want this behaviour to apply to one form only then, rather than opening the hidden form at start-up, open it in the relevant form's Open event procedure:

    blnAllowExit =False
    DoCmd.Openform "frmHidden",WindowMode:=acHidden

and in the same form's Close event procedure put:

    blnAllowExit = True
    DoCmd.Close acForm,"frmHidden"

Below are some ways you might tackle the problem until MS get a proper fix in place:

People page won't open... People page just "hangs" forever... can't access my Contacts list... address box does not populate when typing a new e-mail... etc.  This all seems to be the same very common current problem (bug?) with the failure of MS OUTLOOK's contact database to load correctly on the PC site.  Most browsers seem to be pretty equally affected (perhaps Chrome less so - if you have it - try it).

1.  You may be able to at least access your Contacts list by using OUTLOOK's Mobile site login page (https://profile.live.com/home/contacts?bfv=um).  Best to try this when you are fully logged out of OUTLOOK.  Sadly this is of limited use as the Mobile page "footer" link back to the PC site People page will just hang forever as well.  However it will prove your Contacts list still exists and in extremis you could copy/paste or write out e-mail addresses!

3.  Try a different browser - Chrome seems to be the least affected, others have had success trying Firefox, and some have succeeded by resetting whatever browser they were currently using.

4.  If you have access to other machines try them... especially if you have an older machine still running XP/SP3 - several users report that their old XP PC is unaffected by this problem.

Hope the above might help - you are not alone - there's a shed load of similar topics, information and other frustrated OUTLOOK users on the forum.  All in all it's a right pigs ear and hardly MS' finest hour?

I'd say go with cutover. It's easy to manage, but you will have to disable dirsync and only re-enable it after the migration is done.
