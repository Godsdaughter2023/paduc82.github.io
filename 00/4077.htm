bob jones aka: cybertaz say:

I agree with John - You most likely are clicking on the Fonts tool (just beneath the Colors tool) in the Background group on the Slide Master tab. Instead, once you've selected the 2 Placeholders on the large Master Slide thumbnail go to the Home tab to use the font list there or go to Format> Font in the main menu.

However, that will apply only to the current presentation. Unlike Word, PowerPoint does not have a provision for specifying a default font so you still need to specify it in each new presentation file you create. John's suggestion to create a custom Theme Fonts XML file is a good one but I believe it presents the same consideration - It can't be specified as the default for new presentation files (at least not without more extensive hacking :-)).

If you want a somewhat different approach than what John suggested, just save the modified file as a Template then base any new presentations on that template. If you do go the custom Theme Fonts route you still can apply it along with a choice of Theme Colors, save that as a Custom Theme & designate it as the Default Theme.

john korchok say:

If it's a PostScript font, Office no longer supports that format.

If it's an .OTF, .TTF or .TTC font, please open Applications>Font Book, select the Avenir you have installed, then choose File>Validate Font to see if there are any problems with it. Since you aren't using the Avenir that comes with macOS, you should disable the Apple version to prevent having 2 copies of the font activated at the same time.
