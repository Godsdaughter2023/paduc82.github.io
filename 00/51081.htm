a. user say:

Hi Tristan,

First, we must understand how an Android emulator works. Specifically, how it uses graphic acceleration.

Definitively, a graphic accelerator is a chipset that a computer program—in this case, the Android emulator—uses to offload the process of sending and refreshing images on a display monitor.

When creating an emulator, however, the developer can choose between hardware and software graphic acceleration.

Now, this could be the cause of the crashes. Despite meeting the hardware requirements, the emulators you're using could be running on software GPU emulation.

Furthermore, if the GPU hardware is incompatible with the emulator, it will be forced to use software GPU which furthers support our speculation.

In this case, it would be best to check with the developers of the emulator regarding the compatibility of the emulator and your PC's specifications.
