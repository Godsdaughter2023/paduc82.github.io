dsmartinez say:

I found a command-line solution by using Cipher Security Tool provided by Microsoft. As far as I understand, there is no (graphically) user-friendly way to add user access to a group of files; it must be done using Cipher via command-line.

For anyone else scratching their head as to how to do it:

1) Open up a CMD window in the folder in which the the files you would like to add user access are located (File->Open command prompt). (Alternatively, you can also simply open up a CMD window and specify a root directory or folder using /s:[dir:] command at the end); this will include all files and subdirectories within the specified directory.)

2) Type cipher /adduser /certhash:[enter the hash*, or thumbprint, of the EFS certificate you would like to add user access to]

* [ ] = Fill accordingly

* Replace [dir:] with appropriate directory, e.g. "c:" or "c:\Music"

* The certhash, or thumbprint, of a user's EFS certificate can be found by opening Certmgr.msc in Windows. In Certmgr.msc, go to Personal->Certificates, then double-click on the certificate you want the thumbprint of. Select the Details tab and scroll down to Thumbprint. Copy and paste it in Word and delete the spaces inbetween the numbers -- and there's the hash you enter after certhash: .

From a related thread... better than anything the gurus are posting: 

I had similar problem. Windows 10 logon didn't have option to use PIN instead of password, PIN set up window didn't have "I Forgot Pin"-option or "Remove PIN"-option. It's like PIN hadn't been even set.

I tried fix this issue with different try-outs mentioned here in Microsoft answer-forums.

(To mention some just for in case:

- Delete C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\NGC folder after taking it's ownership first,

- Change password

- Change to use local account instead of Microsoft account in Windows and then back

- Change some services to start from manual to automatic instead)

Nothing worked, but then I begin reading events from Events Log and I realized that everytime when I try setup PIN there's new event entry available:

Event Id: 7023

Source: Service Control Manager

Event details: Credential Manager was stopped because of an error: Access Denied

After doing some googling I managed to find instructions how to fix that Credential Manager service not been able to start (and that was actually the cause of an issue in PIN set up)

1. Goto C:\ProgramData\Microsoft

2. Open properties from folder named "Vault"

3. Make open Security-tab and check that there are full permissions for SYSTEM and Administrators

- If there is no permissions or there's only permissions for one user account (yourself), add missing permissions (Full Access for SYSTEM and for Administrators)

4. Goto Vault-folder that now has right permissions and check that every folder inside it (C:\ProgramData\Microsoft\Vault) has right permissions:

SYSTEM: Full

Administrators: Full

Users: Read, Execute and List contents

TrustedInstaller: Full

After permissions are set, there is no need to even reboot computer.

Just open PIN set up and go and set it finally up!

Note: Clearly this does not fix the issue if the issue isn't caused because of Credential Manager service fails to start, so I recommend to take a quick look for Event Log to see if there are issue entries.

You can also try start to start it manually by pressing [Win-Logo]+[X],

and open Elevated Command Prompt,

Type in: net start VaultSvc

If you get Access Denied-error or service didn't start for some other reason, these instructions may help you too.
