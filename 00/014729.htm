In answer to your 1st question, yes, it is a perpetual license, not a subscription. You can continue to use it 'forever' without incurring any further cost.

Keep in mind, of course, that it eventually may not be compatible with future advancements of the operating system & new hardware [E.g., Office 2004 is quite functional as long as it's used on versions of OS X prior to 10.7 (Lion) which further dictates that it must be on a Mac supported by that version of OS X.] IOW, you will have to continue using it on a Mac & OS version that supports it. Additionally, all product has a Life Cycle... a point in time where support will be discontinued & no further updates will be supplied. For Microsoft products, that is usually at least 2 years beyond the release of the next version. It sometimes -- but not always -- extends beyond that.

Additionally, you'd need to pay for the next version you might want to upgrade to, whereas with an Office 365 subscription you continue to be upgraded automatically as new versions are released (as per current policy).

Addressing your 2nd question, yes, Office 2011 (any edition) is supported on OS X 10.10.x [Yosemite].

Further to your 3rd question, you do not have to be a student to purchase the Home & Student Edition... the operative word being Home :-) The primary restriction of that license is that it is not to be used for business/professional purposes. However, it is eligible for a discount if you are a student, parent, or faculty member. You must be able to provide proof of eligibility to receive that additional discount (which I believe amounts to 10%). It also is available from sources other than the Microsoft Store at various price points.

myall_blues say:

Here's a method I used to do something similar to what you're talking about. It works by writing the values entered onto the form by the user  into a sheet in the workbook. I made that sheet hidden so that the users need not see it. There are several elements:

In the code for the form:

  • Code to write entered values onto the hidden sheet
  • Code to read previously entered values into the form
  • Code to abandon the entries if the user clicks cancel on the form

In a main module:

  • Code to check that the hidden sheet exists
  • code to read the values from the hidden sheet so that they can be used in subsequent processing.

On the form I just named all the entry fields the same with a sequential number on the end (e.g. txtcolumn1, txtcolumn2 etc). You don't have to do it this way but it makes it easier as you can read/write the values using a loop.

Form code:

1. Read existing values into the form

Private Sub UserForm_Initialize()
    '
    Dim r As Integer
    '
    ' Check that the sheet containing config data exists and
    ' populate form fields with current values if true
    '
    If CheckSheet("MacroData_Hidden") Then
        For r = 1 To 4
            frmGetFields.Controls("txtColumn" & r).Value = ActiveWorkbook.Worksheets("MacroData_Hidden").Cells(r + 1, "B").Value
        Next r
    End If
End Sub

2. Write entered values to the worksheet if user clicks OK on the form

Private Sub cmdGetFieldsOK_Click()

    '
    Dim r As Integer
    Dim wksNewSheet As Excel.Worksheet
    '
    ' Check that the sheet containing config data exists and create if not
    '
    If Not CheckSheet("MacroData_Hidden") Then
        Set wksNewSheet = Worksheets.Add
        With wksNewSheet
           .Name = "MacroData_Hidden"
           .Visible = xlSheetVeryHidden
        End With
    End If
    '
    ' Write values from form to config sheet for later reference
    '
    With ActiveWorkbook
        For r = 1 To 4
            .Worksheets("MacroData_Hidden").Cells(r + 1, 1).Value = frmGetFields.Controls("lblVar" & r).Caption
            .Worksheets("MacroData_Hidden").Cells(r + 1, 2).Value = frmGetFields.Controls("txtColumn" & r).Value
        Next r
    End With
    Unload Me
End Sub

3. Do nothing if user clicks Cancel on the form

Private Sub cmdGetFieldsCancel_Click()
    '
    ' Cancel button - unload form
    '
    Unload Me
End Sub

Main module code:

1. Read stored values into an array so they can be used

Option Explicit
Option Base 1
Public txtCol(4) As String

Sub ReadColumnVals()
    '
    ' Read columns and paths to be used from spreadsheet into relevant arrays
    '
    Dim r As Integer
    '
    For r = 1 To 4
        txtCol(r) = Worksheets("MacroData_Hidden").Cells(r + 1, "B").Value
    Next
End Sub

2. Check that the sheet containing stored values exists

Public Function CheckSheet(ByVal sSheetName As String) As Boolean
    '
    ' Check required worksheet exists
    '
    Dim bReturn As Boolean
    Dim oSheet As Object
    For Each oSheet In ActiveWorkbook.Sheets
        If TypeName(oSheet) = "Worksheet" Then
            If oSheet.Name = sSheetName Then
                bReturn = True
                Exit For
            End If
        End If
    Next oSheet
    CheckSheet = bReturn
End Function
