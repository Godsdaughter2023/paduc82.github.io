Solving the voluminous Certificate warnings in the Application Log - Microsoft Community

rseiler say:

Solving the voluminous Certificate warnings in the Application Log

Windows Insider machines seem to retain the last year's worth of a certain obscure certificate. The vast majority of them are expired, which is why the Application Log warns you (CertificateServicesClient-AutoEnrollment) about them on each boot and at other times throughout the day. I had nearly 50 (here are a few):

But there was no trace of them in certmgr.msc.

Fortunately, there's a way to find certificates by thumbprint in Powershell.

The first line of the output when searching on one of the mystery thumbprints is PSPath: Microsoft.PowerShell.Security\Certificate::LocalMachine

Whereas searching on the thumbprint of a known cert returns:  Microsoft.PowerShell.Security\Certificate::CurrentUser.

Then I noticed what the very top of certmgr.msc says: CURRENT USER, which means there's a whole other set of certs that it's not showing in that view.

Here's how to get the machine view:
-Start/Run/MMC
-File/Add-Remove SnapIn
-Certificates, Add, Computer account, Next, Local computer, OK
-Expand Personal

You should see something like this, unless you have done a recent fresh install of Windows (this screenshot only represents a small portion):

Now, on a normal non-Insider machine, you'll probably see one or so there. It may even be expired, but there's not 40+. I think there are that many for us as a byproduct of all of the successive Insider builds.

I simply deleted all the ones that have expired.

andreas killer say:

I'm trying to list all the pivot fields in a pivot table as children to a pivot table node, here is what I have

If you want to create a structured tree, it is easier to use the last added node to build the relationships between the nodes.

The following code is untested.

Andreas.

Private Sub UserForm_Initialize()
  Dim WS, TablaDin, Campo
  Dim RootNode As MSComctlLib.Node
  Dim LastNode As MSComctlLib.Node
 
  For Each WS In ActiveWorkbook.Worksheets
    If WS.PivotTables.Count > 0 Then
      'Add root node
      Set RootNode = Me.TreeView1.Nodes.Add(Text:=WS.Name)
      'Make child visible
      RootNode.Expanded = True
      For TablaDin = 1 To WS.PivotTables.Count
        'Add first child to last root node
        Set LastNode = Me.TreeView1.Nodes.Add(RootNode, tvwChild, Text:=WS.PivotTables(TablaDin).Name)
        'Add all children to last child
        For Campo = 1 To WS.PivotTables(TablaDin).DataFields.Count
          Me.TreeView1.Nodes.Add LastNode, tvwChild, Text:=WS.PivotTables(TablaDin).PivotFields(Campo)
        Next
      Next
    End If
  Next
End Sub
