Almost every document includes information such as; Hidden text, Object info, Personal Information (identifying information: author name), and other invisible info stored as metadata. To get rid of it, Word 2010 includes a Document Inspector feature which lets user inspect for these concealed elements and to remove them at once. It primarily stands for inspecting & removing; Personnel Information, Comments, Annotations, Meta info, XML data, Hidden text, and more such informations.

To start out with, launch Word 2010 document which you want to thoroughly inspect for hidden elements. Click File to bring up Backstage View. Under Prepare for Sharing and from Check for Issues options, click Inspect Document.

This will bring up Document Inspector dialog, listing multitude of options for inspecting; Comments, Revisions, Version, Document Properties, Personal Info, XML Data, Headers, Footers, Watermarks, Invisible Content, and Hidden Text, etc.

Enable desired options available in the list and click Inspect.

It will start inspecting for specified elements. Once inspection is completed , you can remove the inspected elements from the document by clicking Remove All with each option you’ve selected.

On clicking Remove All, it will instantly remove all the specified elements from the document.

Now your document is safe, and you can start preparing it for generic distribution.

You may also be interested in previously reviewed guides on How to create User Entry Forms in Word 2010 and Using new Stylistic Font Sets & Ligatures in Word 2010.

graham mayor say:

If your client wants to adopt an old normal template then that old template must go in the user's User templates folder (with Word closed), having first removed or renamed the original normal template. If it is a normal.dotm template then it must be opened for editing and Save As with the same name and location.

If it is a Normal.dot template then Word will create a new normal.dotm template based on the old one. The template will only be created on exit from Word and to this end it is better to have the option set to prompt to save the normal template. You will not be able to edit the template until it has been created.

When there is no normal template present, Word will start with default settings built into the program. That default state does not have compatibility mode as an option. Hold the CTRL key whiule starting Word to confirm. This will give you a safe mode prompt. Safe mode uses the default settings.

The implication of compatibility mode is that you do have an old normal template in the User Templates folder.

See also http://www.gmayor.com/what_to_do_when_word_crashes.htm   Orphaned lock files can really screw up the system.

stefan blom say:

To add to what Graham said, if Office was installed on top of an older version, Normal.dotm may be created as a copy of a renamed Normal.dot file; the name of such a file may be Normal1.dot. To ensure that Word creates a completely new Normal template with factory default settings, you have to move all files called Normal.dot, Normal.dotm, Normal1.dot, etc. out of the User Templates folder.

Sometimes, you won't actually see a (new) file called Normal.dotm until you have made changes that forces Word to create/save it, such as choosing a new default font.

Hi JohnKrikorian,

 

Thank you for posting your query in Microsoft Community Forums. We’re sorry for the delayed response.

 

As what I've understood on your issue, you have a new Outlook.com account and a Yahoo account registered as a Microsoft account and you want to transfer your Yahoo account's contacts to your new Outlook.com account.

 

Regarding this statement "But if I go into my new Outlook.com account settings and say Add an Alias, and enter the Yahoo address, it says It is Already taken, please try a new one (I know it's already taken, it's taken by me)", please be informed that you're not able to add an existing account as an alias and even though you're able to add your account as alias, its contacts will not be moved to that account.

 

If you want to import your Yahoo.com account's contacts to your Outlook.com account, follow these steps:

 

1. Login to your Yahoo.com account on https://people.live.com.

2. Click Manage and there will be a dropdown list.

3. Choose Export.

 

Note: You will receive a file in .CSV format.

 

To import contacts, please follow the steps below.
1. Sign in your Outlook.com account here: https://people.live.com
2. Click Import from file. (if you cannot see the Import from file, Click Manage and then click Add people.)
3. Under Step 2, Choose Another Outlook.com or Hotmail account
4. Click Choose File.
5. Select the specific file (the Microsoft Excel file you obtained when you exported your contacts).
6. Click Import contacts.

 

If you have any additional questions, feel free to post back.
