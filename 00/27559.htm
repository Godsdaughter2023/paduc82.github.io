Sub Example_HowToCreateAFormulaWithVBA()
  'Copy the formula from Excel as is
  '  =INDEX(C1:C2,MATCH($A$1,B1:B2,0))&"Whatever"
  'double each " sign inside:
  '  =INDEX(C1:C2,MATCH($A$1,B1:B2,0))&""Whatever""
  'surround the whole formula with " signs
  '  "=INDEX(C1:C2,MATCH($A$1,B1:B2,0))&""Whatever"""
  'Replace the cell references with a "placeholder" (any unique string that you like and doesn't exists elsewhere)
  '  "=INDEX(#C#,MATCH(#A#,#B#,0))&""Whatever"""
  'and assign to a string:
  Dim MyDefFormula As String
  MyDefFormula = "=INDEX(#C#,MATCH(#A#,#B#,0))&""Whatever"""
    
  'Now create the Range objects and refer to the real cells
  Dim a As Range, b As Range, c As Range
  Set a = Range("A1")                                    'A single cell
  Set b = Range("B2:B50")                                'Multiple cells
  Set c = Range("C2", Range("C" & Rows.Count).End(xlUp)) 'From C2 to last used cell in column C
  'Note:
  '  You can refer to cells in an other sheet (note the dot in front of Range!):
  'With Worksheets("That")
  '  Set C = .Range("C2", .Range("C" & Rows.Count).End(xlUp))
  'End With
  '  Or other already opened files:
  'With Workbooks("This.xlsx").Worksheets("That")
  '  Set C = .Range("C2", .Range("C" & Rows.Count).End(xlUp))
  'End With
  
  'Make a copy of your default formula into a work string
  Dim MyFormula As String
  MyFormula = MyDefFormula
  'Replace the placeholders with the cell references
  MyFormula = Replace(MyFormula, "#A#", a.Address)
  'Note:
  '  The first arguments of Range.Address controls where the $ appears in the formula
  MyFormula = Replace(MyFormula, "#B#", b.Address(0, 0))
  'Note:
  '  The part 'External:=True' is necessary if you refer to other sheets or files
  '  But it works also if the reference is in the active sheet
  MyFormula = Replace(MyFormula, "#C#", c.Address(External:=True))
  
  'Write the formula into the cell
  Range("A2").Formula = MyFormula
  
  'Note:
  '  If your locale language is not English you can use the FormulaLocal property instead
  '  But in this case the code works only on PCs with your locale settings
  'Range("A2").FormulaLocal = MyFormula
  
  'Note:
  '  If you want to create array formulas use the FormulaArray property
  '  But in this case use must use English formulas, a FormulaArrayLocale doesn't exists
  'Range("A2:A10").FormulaArray = MyFormula
End Sub

andreas killer say:
Well, I can give you a sample for the simple SUMPRODUCT formula, that gives you an idea what you're talking about.

SUMPRODUCT processes arrays in a way that is no easy to accomplish with pure VBA. The code below is just the basic, no error handling etc. if you want to include all that what SUMPRODUCT does for free, you need around 10 times more code then just that.

If you want to deal with MATCH / INDEX / SUMPRODUCT means you have to find a value in an array, then copy a part of the array into another, then continue as in the code below and make sure that all array sizes match... and there is nothing in VBA that helps you to make the task easier.

It's much easier to create the formula as I've shown above, then you can write it into the sheet and convert the result into a value (or use Application.Evaluate to get the result directly and write it into the sheet). All that is also not easy, but much easier as to write a pure VBA code.

Andreas. 

Sub Test()
  'SUMPRODUCT((J8:J30495<=EN1)*(J8:J30495>=EO1)*(G8:G30495="Sales")*CF8:CF30495)
  Dim J As Range, G As Range, CF As Range
  Dim Low As Date, High As Date
  Dim DataJ, DataG, DataCF
  Dim i As Long
  Dim Sum As Double
  
  'Get the dates
  Low = Range("EO1").Value
  High = Range("EN1").Value
  'Refer to the used cells in column J
  Set J = Range("J8", Range("J" & Rows.Count).End(xlUp))
  'Same size in column G
  Set G = Intersect(Columns("G"), J.EntireRow)
  'Same size in column CF
  Set CF = Intersect(Columns("CF"), J.EntireRow)
  
  'Read in all data
  DataJ = J.Value
  DataG = G.Value
  DataCF = CF.Value
  
  'Process the SUMPRODUCT
  For i = 1 To UBound(DataJ)
    If DataJ(i, 1) >= Low And DataJ(i, 1) <= High And DataG(i, 1) = "Sales" Then
      Sum = Sum + DataCF(i, 1)
    End If
  Next
  MsgBox Format(Sum, "0.000")
End Sub

jay freedman say:
It's possible that the Draw Table tool created an invalid chunk of HTML in the document's file -- Word seems to have some trouble with that sort of thing. The inability to select or delete the row indicates that the table's coding is probably corrupted. You might be able to use the Convert Table to Text button (near the right end of the Table Tools | Layout tab) and then Convert Text to Table (at the bottom of the Table button's gallery on the Insert tab) to clean it up.

If you have to rebuild the table, or if you run into this situation again, the Draw Table tool is definitely the wrong tool for the job. Instead, select the row(s) of column headers at the beginning of the table, go to the Table Tools | Layout tab, and click the Repeat Header Rows button. Every time the table extends across a page break, the same row(s) will automatically be repeated. If you add or delete rows in the middle of the table, you won't have to move the header row that's now in the wrong place; and changing the text in the first header row will change all of them. (You can't select or edit the repetitions, because they aren't really there.)

For adding rows that aren't header rows: At the end of the table, the simplest way is just to put the cursor in the last cell and press the Tab key. Anywhere else in the table, hold the mouse cursor in the left margin near the row where you want to add one, and click the blue plus sign that appears between rows there. Similarly, you can add a column by moving the cursor to the top of the table and clicking the plus sign between columns. To add more than one row or column, select as many existing rows as you want to add, then click the plus sign.

Another way is to use the buttons in the Rows & Columns group on the Table Tools | Layout tab.
