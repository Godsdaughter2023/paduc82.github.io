ken sheridan say:

My ultimate goal is to do the same with phone numbers and any other fields that may have multiple records per contact.

A query of the type I posted will only work if there are a maximum of two rows in the referencing table per row in the referenced  table.  In many situations this is unlikely to be the case.  The usual solution would be to create a cross tab query.  In the case of phone numbers the Type would be the 'column heading' and the contact names would be the 'row heading'.  If each contact has only one phone number of each type, then any aggregation operator can be used for the value at the intersection of the row/column, e.g. MAX.

If, on the other hand, a contact could have multiple numbers of the same type, then the above would not work of course.  To use a crosstab query, it would be necessary to firstly concatenate the phone numbers per type per contact into a single string expression, in which the separator would best be a carriage return/line feed so that the numbers appear vertically over several lines.  A crosstab query as described above could then be based on the query, the value at the intersection being in this case the concatenated string expression.

You'll find an example of a function for concatenating values from a related table in Concat.zip in my public databases folder at:

First create a ranking query [qrnkContactEmails]:

SELECT EMail.ID, EMail.Email, Count(EMail.ID) AS CountOfID
FROM EMail INNER JOIN EMail AS EMail_1 ON EMail.ID = EMail_1.ID
WHERE (((EMail.Email)>=[Email_1].[Email]))
GROUP BY EMail.ID, EMail.Email;

This will result in something like:

TRANSFORM Min(qrnkContactEmails.Email) AS MinOfEmail
