john w. vinson say:

It sounds like you are trying to do the calculation in a Table. Wrong place! Calculated fields in Tables are VERY limited (they were added mainly to support web applications). 

Instead, use this expression as the Control Source of a textbox on a Form or Report and show that to the user; or (if need be) add a calculated field to a Query by typing

IsItLate: IIF([datefield] < Date, "LATE", "")

in a vacant Field cell.

scottgem say:

So don't use a calculated column! 

What you should be doing is using the On Current event of your form:

If Me.datecontrol < Date() then

     Me.textcontrol = "Late"

Else

    Me.textControl = ""

End If

if you want to display Late in a query then you can use the expression:

Alias: IIF(datefield<Date(),"Late","")

scottgem say:
Access is an event driven platform. Almost all automation occurs from an Event. Almost all objects have events, the number and type depending on the object. A Form object has an On Current event which is triggered whenever the form's focus changes from one record to another.

Do this in a query based on a table by putting the following in the 'field' row of an empty column in design view:

LATE:[DateField]<Date()

You can format the column as Yes/No in the column's properties sheet in query design view, and in a form or report based on the query you can bind a checkbox to the LATE column if you wish.
