Windows 10 desktop compositor and high-PPI rendering

This is just a short story on a “1st world problem” with Windows 10 (1151 and earlier) regarding messed-up desktop rendering.

When using a high-PPI display (4K, 5K…) you need to use system UI scaling. It’s really a simple choice for a 5K 27" display with a physical resolution of 5120x2880 pixels — by default you should be using it at 200% scaling to get dense rendering of usable 2560x1440 with 4 physical pixels per 1 viewport (actually measurable/usable) pixel.

If you’re using a single high-PPI display or multiple identical (this is important) high-PPI displays exclusively, there is hardly any problem lately, since Windows 10 handles high-PPI displays and scaling just fine.

However, if it happens you try to use different displays and apply a different scaling factor for each, there is one huge and annoying problem that surfaces:

Windows 10 system is unable to properly render multiple displays at different scaling factors.
Windows 10 system is unable to properly render multiple displays at different scaling factors.

(unlike Mac OS X or Linux desktop managers which render all kinds of mixed scaling factors just fine).

To make it all a more realistic example and easy to understand, let’s take a real life situation: you have a regular, “normal” display like Dell U2711H at the resolution of 2560x1440 pixels and a modern Dell UP2715K display which is a double-density 5120x2880 panel.

To get the 5K display to match your older one in effective (viewport) resolution of 2560x1440 and to just gain a great crisp image, you’d set your 5K Dell to 200% scaling and leave your normal display as it is, at 100% scaling (we can ignore the 3rd display here).

Dell UP2715K set up as the primary display at 200% scaling.

In theory, this setting would get you your regular picture at the old display and a super-crisp picture on your 5K.

So, what is the problem here?

Well, the problem is that Windows 10 don’t handle it well. Windows 10 desktop compositor will render correctly only the screen set up as the primary display in your multi-monitor setup. All the other displays will not be granted proper rendering, but would get rendered in an ugly way that blurs the content.

Dell U2711H is being set at 100% scaling

So, if your primary display is the new 5K, it would get rendered perfectly crisp, but the old U2711H would get rendered even worse than usual: it will be a blurry mess.

On the other hand, if your primary display is the old U2711H and not the 5K, the old one would be rendered as usual (correctly), but the new 5K would look much worse as we can see in the following screenshot.

Let’s take a look at the brand new Dell UP2715K and different rendering situations:
Left: Dell 5K at 200% while not the primary monitor — Right: Dell 5K at 200% being the primary display

For a full resolution image, please visit: http://imgur.com/ebTZCwu

To the right is a Dell 5K display set at 200% scaling, rendered when set To the right is a Dell 5K display set at 200% scaling, rendered when set as a primary screen. Notice the crisp font you would expect. Nothing wrong.

But, to the left is the very same Dell 5K display, set to the very same 200% scaling factor, but the primary monitor selected is the old U2711H instead.

Notice awful jagged edges and blur?

The same thing happens with the old monitor, just in a different way: when 5K is the primary one, it’s been rendered fine, but U2711H becomes blurred.

If set up both monitors to the same scaling factor of 200% you get great rendering, but loose 50% of the effective resolution on U2711H. If you set both to 100% you can’t use 5Ks awesome crisp image and you can’t use the display at all since everything in the UI is so tiny.

And that is the problem that never happens on, for example, Mac OS X which renders different screens and scaling factors correctly.

Hopefully, Microsoft will be able to rework their desktop compositor to be able to handle mixed display scaling factors properly.

Reduce WinSXS Folder Size If You Installed Service Pack 1

If you have installed Windows 7 SP1 the following can reduce the size of your WinSXS folder. Note you will not be able to uninstall SP1 if you do this.

1. Click Start --> All Programs --> Accessories

2. Right click CMD.exe and click Run as Admin

3. Type: DISM /online /cleanup-Image /spsuperseded

4. Press Enter and exit command prompt






Free Up Space - Increase Hard Drive Space 

The following procedure cleans up files associated with your user account. You can also use Disk Cleanup to clean up all the files on your computer.

1. Click the Start button, in the search box, type Disk Cleanup, then click Disk Cleanup.

2. In the Drives list, click the hard disk drive that you want to clean up, and then click OK.

3. On the Disk Cleanup tab, select the file types you want to delete.

Note: To Remove all System Restore Points Except the Most Recent - Click "Clean up System Files" and you will get another tab called More Options. Now click More Options, then click "Clean up" under "System Restore and Shadow Copies" then confirm the deletion.

4. Click OK on the bottom of the Disk Cleanup window and in the message that appears, click Delete files.

Reduce the amount of space allocated for System Restore. 

1. Start-->Control Panel-->System and Security-->System

2. Next click on the 'System protection' link located in the upper left.

3. In the 'System Properties? window click on the 'System Protection' tab.

4. Locate the 'Protection Settings' box and select/click on the drive letter (usually C:)

5. Now click on the 'Configure' button to change the 'Disk Space Usage' Restore Settings.

6. Move the slider to reduce the amount of disk space System Protection can use (min. 3GB).

7. Click the 'Apply' and then the 'OK' button after you have adjusted the disk space.

Uninstall Programs You Don’t Need

1. ﻿Click Start, then click Control Panel

2. Click ﻿Add/Remove Programs and uninstall programs you don’t want

Reduce Pagefile Size

1. Open control panel and choose system and security.

2. Click on system

3. Click on Advanced system settings.

4. Click on advanced tab.

5. In the performance section click on settings.

6. Click on Advanced and choose Change in the Virtual Memory section.

7. Uncheck Automatically Manage Pagefile size

8. Select Custom and enter new size (don't go under 512MB) in both boxes and press Set.

9. Click OK and restart your computer

Turn of the Hibernation setting

If you don't use hibernation you can turn it off and free up disk space:

1. Start --> All Programs --> Accessories

2. Right click CMD.exe and click "run as admin"

3. type: powercfg -H off

4. Press Enter

5. Exit and restart your computer
