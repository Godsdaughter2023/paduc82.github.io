superfranky say:

hi ashok,

yes, to make ip address range safe in office 365, you can create a transport rule.
here are the steps:
1. sign in to the exchange admin center.
2. go to mail flow > rules.
3. click + to create a new rule.
4. click more options in the bottom of the pop-up window.
5. choose the condition the sender > ip address is in any of these ranges or exactly matches, and add the ip address or range.
6. choose the action modify the message properties > set the spam confidence level (scl), and select bypass spam filtering.
7. save the rule.

for the message that the internal users are not received, you can run a message trace to check if the messages are delivered to office 365.

karl timmermans [outlook mvp 2012-2018] say:
Sounds like you configured the email account in '2016 as an IMAP account by entering only your email address and password instead of manually configuring the account as a POP3 account.

By importing your emails - those would be sync'd back to the server.

The correct process would have been to configure the email account manually as a POP3 account and using the same PST file that was used with OL'2010 instead of letting Outlook '2016 create a new one.

Going to assume that you have contact/calendar data in the PST file used with OL'2010 that you want to retain so would suggest the following procedure given the current situation

#1 - Remove the Imap account from the Outlook profile
#2 - Manually add the account back as a POP3 account

How to manually configure an email account in Outlook 2016
https://www.contactgenie.info/how-to-manually-c...

If you do not want to keep any of your email on the server, be sure to go into Advanced Settings and uncheck the box for "Leave Messages on server....."

For this particular situation, be sure to let Outlook create a new PST file since all your emails are on the server. Once the email account configuration is completed, Outlook will download all the email to the newly created PST file

Go to File --> Account Settings --> Data FIles tab --> set the newly created PST file as the Default data file for the profile. This is where your default contact/calendar data will be stored

When done, add the PST file used to import from OL'2010 in '2016 by going to Outlook menu --> File --> Open & Export --> Open Outlook Data file

When the old PST file is added to to the profile, set the contact and calendar folders to a <list> view (i.e. Phone). For each folder

- select the specific folder you want to copy from (i.e. old contact folder)
- press CTRL+A to select all the items in the folder
- right-click the highlighted items --> select <Move> --> select <Copy to Folder> --> select the respective folder in the PST file created by Outlook when the account was configured.
*** reason for selecting <Copy to folder> instead of selecting just <Other Folder> is that the <Copy> process will not delete the source items should something go sideways

When you have completed copying all the desired items, highlight the root of the OLD PST file in the Outlook Nav Pan --> select <Close>

hansv mvp say:

Select the cells with the times.

In the following, I'll assume that A1 is the first cell within the selection. Change as needed.

On the Home tab of the ribbon, click Conditional Formatting > New Rule...

Select 'Use a formula to determine which cells to format".

Enter the formula

=OR(A1<TIME(5,0,0),A1>TIME(23,0,0))

Click Format...

Specify the desired formatting.

Click OK, then OK again.
