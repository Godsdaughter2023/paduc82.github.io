Hey Geoff, here's what Luke (the new owner of this) has for you, I hope this helps!

The DMR does not show up in the Action Center’s Connect menu intentionally.  Devices there are those which can be used for system-level connections such as Miracast for a cloned/extended desktop or a Bluetooth audio device which can be used as a default audio endpoint.  The DMR does show up in app-level connections such as the Cast To Device context menu and the new media element default transport controls.  From the app-level context, a piece of media or specific app scenario is sent to the remote device rather than the entire desktop experience.

 

The primary reason for not including DLNA devices in the system-level context is latency.  The DLNA protocol is designed more around the idea of streaming a file than as a live interactive stream.  While it is technically feasible, many devices are unable to provide an experience that would be responsive enough to work at the system level.

 

Eventually the new music app should have support for casting to such devices, the video preview app should already have this support enabled.

Windows Defender (on Windows 7, Vista, and XP) provides spyware protection only and is NOT an anti-virus product.

In Windows 8, 8.1 and 10 Windows Defender is a real-time anti-virus product.

When you install another 3rd party anti-virus product, Windows Defender should  in general be automatically turned off. In this case you will not be able to turn on Windows Defender again until the 3rd party AV program has been completely uninstalled.

So, since you say you have another AV(360 Total Security) installed, Windows Defender is turned off and that's how it should be.
