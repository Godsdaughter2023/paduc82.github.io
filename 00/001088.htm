bob jones aka: cybertaz say:

Well, I'm guessing that you won't like the explanation but here goes :-)

Object Linking & Embedding [OLE] is a Windows technology which never has been fully implemented on Macs. One of the shortcomings is that when you insert/paste content from an Excel file as an Excel Worksheet Object the entire file is embedded in the document. To make matters worse, a copy of the complet file is embedded for each Object inserted from it... i.e., 4 Objects = 4 embedded copies of the Excel Workbook file.

The only way I know of to resolve it is to UnZip the .docx file, delete the extra copies of the workbook, then restore the.docx file extension to the unzipped document file. If you aren't familiar with the process let us know, but frankly it may be better to insert the Excel material as something other than as an Excel Worksheet Object, especially if the reviewers don't need to edit the data using Excel's features.

1.  Created the following macro:

Private Sub Document_Open()
ThisDocument.CustomDocumentProperties("Path") = ThisDocument.path
End Sub

 

2.  Inserted the following field into the main document:

 

{ INCLUDETEXT "{ DOCPROPERTY Path}Chapter1.doc

"}

 

It returns an error.  Is anyone able to assist me as to where I might be going wrong? Thanks.

macropod say:

Hi mpennicott,

ThisDocument.path doesn't insert a "/" at the end, so you either need to add that to your 'Path' Custom Document Property or to the INCLUDTETEXT field code.

FWIW, you can achieve the same result without a macro:

{INCLUDETEXT "{FILENAME \p}/../Chapter1.doc"}
