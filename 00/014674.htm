Re (1): Just to clarify the terminology to avoid confusion - the word "list" should be replaced with the word "folder". May sound minor but it's always easier when everything is referenced correctly since different people use the word "list" to mean different things.

The Outlook AddressBook is a specialized "view" of the contact folders that have been set to be a member of the AB. This occurs in two ways, one, the "default" contact folder for the profile is automatically set to be a member of the AB and second, manually setting the folder to be a member which is done by right-clicking on the contact folder --> properties --> select Outlook AddressBook --> there's a checkbox for "Show this folder as an email address book". Checking/unchecking this box governs what folders are included in the AB

Re (2): Right-clicking an email address to save it as a contact results it being saved to the "default contact folder" for the profile. The default contact folder is stored in the "default data file" set for the profile. To determine what that is, go to File --> Account Settings --> Data Files tab --> check to see what data file is set as the default.

If you only have one data file but have two contact folders then you have created a separate folder in addition to the default "Contact" folder.

Re: (3) - You cannot remove the default contact folder for the profile from the Outlook AB.

The simplest solution to the problem as described would be to move the contacts to the "second" contact folder to the default contact folder.

>> - Part 1
How did I get 2 folders where both say: Contacts (This computer only) ?
<<

First and foremost, just to make sure that we're on the same page for the balance of this (and any additional) response(s) - there are going to be two locations when referencing "Contact folders" - the first and most important are those that appear in the Outlook Navigation Pane (the left hand pane) and the Outlook AddressBook.

So in terms of your contact folders - how many appear in the Outlook Nav Pane and if more then one, do each appear in a separate folder tree or are both in the same folder tree. The best way to determine this, while in the Outlook UI (not the AddressBook) - press CTRL+6 to display the full folder list in the Outlook Nav Pane.

There are only two ways to get two "Contacts (This Computer Only)" folders in your profile

First, is that you have two IMAP email accounts and the data file for the second one was set as the default for the profile after the account was configured.

Second, if only one email account is configured (one folder tree) - then the second contact folder was manually created.

There are no if, ands or buts about the above two scenarios. How things work doesn't change with Imap accounts doesn't change from system to system.

>> - Part 2
How did I get 2 folders where both say: Contacts (This computer only) ?
<<

The most important thing is that you really don't want to maintain your contact data in any "This Computer Only" folder or to be even more specific, any contact folder in an IMAP folder tree of folders. Contact/calendar folders do not sync with the server and if the email account is removed the underlying data file will be automatically deleted at which point you will lose any data in those folders unless you have explicitly backed up (exported) those folders to a PST file. Also, if Outlook can't find the .OST file, it will recreate the file and the "This Computer Only" folders will be empty. Also backing up an OST file on its own is a waste since an OST file can only be opened/used by the original Outlook profile/email account used to create it. To otherwise access an orphaned (stand-alone) OST file
requires the use of a 3rd party OST2PST conversion tool - none of which are especially inexpensive.

With the above said, what you really want to do is
1- to create a new PST file. From the Outlook Ribbon --> select <New Items> --> <More Items> --> <Outlook Data File>

2- go to File --> Account Settings --> data Files tab --> set the newly created PST file as the default for the profile - this will now be where your default contact/calendar folders will be located

3- back to Outlook general UI, select the contact folder that contains your contacts

4- press CTRL+A to select all the contacts in the folder

5- right-click on the selected contacts --> select <Copy to Folder> browse to the folder in the newly created PST file
*** reason I suggest "Copy to Folder" instead of Move --> Other Folder is purely as a precaution. The "copy" process won't delete the source items in case something goes wrong during the process whereas the "move" process does. Once everything is done and checked, the source contacts can be deleted

6- repeat for the 2nd "This Computer Only" contact folder if necessary

7- right-click on the contact folder in the newly created PST file --> Properties --> Outlook AddressBook tab --> give the contact folder a different name for the purposes of the AB for easier identification so not everything shows as "Contacts" etc - give it whatever name you like that makes sense to you

8- open the AB from the Outlook ribbon --> Tools --> Options --> Set contact folder by selecting <Custom> and moving the contact folder name from # 7 to the top of the list

This may all sound like a lot but really is simple and only takes a a few minutes. Long term, you are going to very happy you went thru the exercise

In most firms with more than a couple of computers the best solution is actually Microsoft’s Volume License program.  Their Per Device licensing has the same advantages of the Retail license – the license CAN be reassigned to a different device later if you choose to replace your computer and you can install a copy on a portable device that is used by the same primary user as the desktop.

The pricing tends to be favorable on Volume Licensing compared to the retail boxes you get at Best Buy, though perhaps not as favorable as you might expect.

The other big advantage is in management of the licenses.  With Retail Licenses every box comes with a product key – a series of numbers and letters that looks like this:

A1B2C-D3E4F-G5H6I-7J8K9-0L1M2
(obviously that one’s fake)

The thing is you have to keep track of which product key goes to which machine.  And when you install the software on multiple machines you have to enter the product key for each machine manually.  OEM versions these tends tend to print the product key on a sticker, attached to the system.  Makes sense, the license goes with the system so the key might as well too.

With volume licensing, however, you get ONE product key, which you’re allowed to install on as many devices as you bought licenses for (plus additional portables if applicable; see above).

“But I don’t have hundreds of machines!” you may say.  You don’t have to.  Volume licensing agreements start at just 5 (yes FIVE) licenses.  And they don’t have to be 5 licenses of the same thing.  If you only have 3 machines you can buy 3 licenses of Microsoft Office Professional 2010 and 2 licenses of something else…maybe Microsoft Streets and Trips or “Microsoft Math 3.0”.  Something cheap.  Ask your software dealer they can probably find you something.

Adding additional licenses later is easy – once you have the initial five licenses you have a Volume License Agreement and you can purchase additional licenses as needed.  You add another member of staff?  Buy one more license of Office 2010 for them.  No problem.

Most firms with 3 or more computers should be using Volume Licensing in my opinion, if for no other reason than the simplicity of management.  No need to track a bunch of product keys – just use one and your licenses are NOT handcuffed to the physical computer.

What is Software Assurance?

Regardless of where you are in your software lifecycle, Software Assurance provides benefits to help you upgrade to new software, get more productivity out of existing software, and support your current environment. When you are ready to upgrade to a new version of software, SA provides new version rights at no additional cost, deployment planning services and end-user readiness benefits such as online training and home use licenses. To keep your IT department up to date and to help your end-users be more productive on existing software, SA provides classroom, technical training and online training for end-users. How to activate Office 2010? KMS vs. MAK - http://4sysops.com/archives/how-to-activate-office-2010-kms-vs-mak/

By Michael Pietroforte | 2 Comments | Permalink

Both options to activate Office 2010, KMS and MAK, have their up- and downsides. This article discusses a few things you have to consider when you decide for the activation method in your organization.

Now that you have an overview of the Office 2010 KMS activation and MAK activation, I can address the question of whether KMS or MAK is the better option to activate Office 2010. Most of the arguments also apply for Windows Vista and Windows 7 activation.

Microsoft recommendations

You need at least 5 Office 2010 installations if you want to activate Office 2010 with KMS; however, Microsoft recommends using MAK activation if your network has fewer than 25 computers. If you have more than 50 Office 2010 machines, Microsoft recommends going for KMS.

 

Office 2010 KMS activation essentials - http://4sysops.com/archives/office-2010-kms-activation-essentials/

KMS involves running an internal company license server. Client computers will renew their license against the server. Each renewal will reset the expiry to 180 days, and it will try renewing every 7 days. Office will try contacting the server more frequently if expired or close to expiring. KMS is the preferred method of volume activation for businesses and incidentally KMS based solutions are the main source of activation “solutions”.

 

Office 2010 MAK activation essentials - http://4sysops.com/archives/office-2010-mak-activation-essentials/

MAK activate similar to a retail key. It is a one-time activation over the internet against Microsoft’s server. Unlike a normal retail key, each key can be used for a minimum of 500 activations to thousands of activations. Like a retail key you will have to reactivate if there’s a drastic hardware change or if you change your installation to a new computer.

 

Office Volume Activation - http://technet.microsoft.com/en-us/office/ee691939.aspx

Key Management Service (KMS). KMS uses a host computer to establish an activation service on your local company network. With a KMS host you can activate thousands of computers at the same time when they regularly connect to the company network. If you use a KMS host to activate Windows, you can use the same host to activate Office 2010.

Multiple Activation Key (MAK). With MAK, each computer activates Office 2010 with the Microsoft hosted activation servers over the Internet or by phone. MAK is recommended when you have fewer than 50 computers to activate, and for computers that are not regularly connected to the company network.
