As the error message clearly states, this issue may occur if more than 500 fonts are installed on the computer.

Note: This issue does not occur when you try to open files in Works Word Processor, in Works Calendar, or in Works Portfolio.

To work around this issue, delete fonts from the computer until you reduce the number of fonts to less than 500.

To delete fonts, follow these steps:

1. Click Start, point to Settings, and then click Control Panel.
2. In Control Panel, double-click Fonts.
3. Click the fonts that you want to delete from your computer.
Note: To select more than one font, press and hold down the CTRL key while you click each font.
4. On the File menu, click Delete.
5. When the Windows Fonts Installer dialog box appears with the following prompt, click Yes:
Are you sure you want to delete those fonts?
6. On the File menu, click Close.

Determine the number of fonts that are installed on the computer

Note: Because there are several versions of Microsoft Windows, the following steps may be different on your computer. If they are, see your product documentation to complete these steps.

1. Click Start, point to Settings, and then click Control Panel.
2. In Control Panel, double-click Fonts.

The status bar in the Fonts dialog box indicates the number of fonts that are installed on your computer.

Solution to 8007003 when using Get Windows 10 and Something Happened - Microsoft Community

chawoobie say:

Solution to 8007003 when using Get Windows 10 and Something Happened hangs at 24% to install Windows 10 upgrade

LIke everyone else, I have spent countless hours since July 29th trying to upgrade my ASUS desktop to Windows 10 from Windows 8.1.  After trying everything here and other sites, I did a chat with Microsoft Tier 1 tech several days ago, with same results.  He was going to bump it up to Tier 2, but I have yet to hear from anyone.

I decided to keep trying.  When I went to MSCONFIG to try clean boot, I noticed that when I clicked on the boot tab, Windows 8.1 was listed - but so was the Windows 10 Technical preview that I had been using prior to Windows 10 release.  I deleted that and thought I should also double check in Windows 8.1 boot manager.  When I tried to use Windows 8.1 System Recovery to get into the Troubleshooting menu I noticed two things.  First, in the bottom right corner was Evaluation build: 10222 - even though I thought I had deleted everything prior to trying to upgrade to Windows 10.  Second, I didn't have the Troubleshooting option!  I had Continue, USB/DVD (or some such thing), and Turn off your PC!!  Not good.

Both of those convinced me that I had some problems with the boot record that needed attention, but first I needed to get back the Troubleshooting option in order to run CMD to fix the master boot record.  I also noticed that when I ran Disk Management, I had two partitions on my hard drive that were labeled Healthy (OEM partition).  (In my case Partitions 1 and 5).

I found this article that explained that these needed to be changed back to Healthy (Recovery partition) in order to get Troubleshooting back:

Saltgrass replied on 

It looks like your first and last partitions have been changed to OEM.  So you need to use Diskpart to change them back, so first try listing the partitions on that drive so you will be able to select the correct partition. 

In Diskpart, you must make sure you are working on the correct disk and partition.  An asterisk on the left of the listing will normally indicate which item is selected.

To use Diskpart, open an Administrative Command prompt and type Diskpart.

Then use these commands to list the partitions on the drive, assuming your disk is disk 0.  These commands are given one at a time and can be shortened to 3 letters in most cases.

Diskpart

list disk

select disk 0

list partition

This will give you a list of partitions on that drive and you must select the correct one.  In you case it will probably be partitions 1 and 6, but make sure.

You then need to select the correct partition and change the partition ID back to a recovery partition.  You may want to copy the current Partition Type ID and save it in a Notepad file.  You can do that in a Command Prompt window by right clicking, highlighting, and then pasting it.

select partition 6

detail partition

Then set the partition Type ID with the following command.

set id=DE94BBA4-06D1-4D40-A16A-BFD50179D6AC

For some folks this is all it takes to change the partition back to a Recovery partition.  Check it to see if it works and then continue to make sure the Attributes are set correctly on the partition.  Listing the partitions again may now show Recovery instead of OEM.   Again, make sure you are using the correct partition.

gpt attributes=0x8000000000000001  which should be shown in the detail partition listing.

If it does not work, look through the thread, but during my tests, the partition was already set to Recovery and I had to change it to a normal partition, reboot, and then change it back.  Hopefully you will not need to do that.

If you have problems, do what you can of the procedure and post the results.   You will need to do this for both partitions.  Command can be reproduced and then modified in the Command Window by using the up or down arrow.

Once I had the partition relabeled as Healthy (Recovery partition), I was able to boot to System Recovery and select the Troubleshooting option which then allowed me to go into a CMD prompt to address the next issue - rebuilding my Master Boot Record (MBR).

Thanks to this link:   https://www.winhelp.us/repair-your-computer-in-windows-8.html, I went to Step 1.3 and rebuilt the MBR.

Once that was done, I rebooted my computer, inserted the Media Creation Tool DVD into my DVD player, and Windows 10 installed like a charm!!!

Hope this will help others, and thanks to the folks who provided the links that enabled me to not only ID the problem, but correct it as well.

Good luck!!
