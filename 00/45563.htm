In all my years with computers, I've never heard of a folder changing into a file - not ever, not even once.  Thank you for a new experience.

First, what is the filetype of this new file (the three digit extension following the filename)?  If you can't see it, go to Start / Control Panel / Folder Options / View and scroll to and uncheck the box next to hide extensions of known file types.  While you're at it, also unchick the box to hide system files and check the button to show hidden files and folders (those might come in handy later for you).

Next, what was in the folder?  I mean what types of files and about how many?  Where was this folder or where is this file (what is the full pathname (e.g., C:\Users\<username>\Documents\Budget\Quicken backups or whatever))?

What happens when you double-click to open the file?  Nothing?  An error message?  Does it actually open the folder (in which case, we merely need to create a new folder and transfer the contents to the new folder and then delete the old "file" to resolve the problem)?

Right click on the file and click on properties.  In the General Tab, tell me the type of file, the create date, and the modified date. In the Detail Tab, tell me the Authors and Last Saved by information and tell me if there is anything in the Description Secion.  In the Custom Tab, tell me anything it says.  If there is a Previous Versions Tab (it depends on your version of Vista), tell me if there are any previous versions listed and if they show as files or if they show as folders.

If this is a program and not data, we may be able to resolve this with System restore (though I'm not sure if it will work because it's an external drive and I'm not sure if it was attached and running when you created the restore point, but if it was and if it's a program it very well might - make sure the drive is plugged in, turned on, and recognized before you start). Do you know when this problem began?  Try a System Restore to a point in timeBEFORE the problem began.  Here's the procedure: http://www.howtogeek.com/howto/windows-vista/using-windows-vista-system-restore/.  Be sure to check the box to show more than 5 days of restore points.  If the first attempt fails, then try an earlier point or two.  NOTE: You will have to re-install any software and updates you installed between now and the restore point, but you can use Windows Update for the updates.  Use the recovery disk if the system prompt doesn’t work. The recovery disk works a bit different from the above procedures but if you follow the prompts from the System Restore menu option with the above information you should be able to restore with no problems.

Vista has no program to convert a file into a folder (or vice versa). I know of no 3rd party software that does this either.  Before your post, I would have said it can't happen.  If you have a backup, restore the old folder from the backup to replace this "file" and then delete the file.  If it is part of a program, then I suggest you uninstall the program and then re-install the program and that may correct the problem (though if the folder contained data or changed information, it will be restored to the original settings and that data will be lost - but if we've reached this point and you've replied and nothing was revealed that helps, that's already the case).

I hope this helps or that whatever you provide in your reply helps me, but to be honest, If the above doesn't work, I don't think we can restore the file to a folder (if that really did happen).

Good luck!

Hi,
 
1. Is the issue pertinent to this particular URL?
2. Do you remember making any changes to the system?
3. Where is the file located?
 
If you encountered this error by clicking a link on a Web page, the link might not be valid.
 
Follow the steps mentioned below and check if they help.
 
Step 1:
 
If you encountered this error by typing a URL in the Open URL dialog box, try opening the file by using a different transport protocol (for example, "http:" or "rtsp:"). The following list shows the protocols that are supported by the Player and examples of the correct URL syntax:
• mms://server/filename (for example, a file with a .wma, .wmv, .asf, or .mp3 extension)
• http://server/filename
• rtsp://server/filename
• mms://server/sami.asf?SAMI=http://server/sami/sami_demo.smi (SAMI file)
 
Step 2:
 
If the Player supports the protocol that you used in the URL and you still can't play the content, verify that the Player is configured to use all of its supported protocols.
 
To configure protocol settings
 
a.       In Windows Media Player, on the Tools menu, click Options.
b.      In the Options dialog box, on the Network tab, in the Streaming protocols area, select all the protocol check boxes (Multicast, UDP, TCP, and HTTP), so that a check mark is displayed in each.
 
Note that the supported protocols can be used only to stream digital media files. Windows Media Player cannot be used as a browser.
 
Step 3:
 
If the above fixes don't work, you can rename the networking information folder for the Player so that it can rebuild its networking information files after you restart it.
 
To rename the Player networking information folder
 
a.       Close Windows Media Player. (To do this, on the File menu, click Exit.)
b.      Click Start, and then click Run.
c.       In the Run dialog box, type the following (exactly as shown):
%UserProfile%\Local Settings\Application Data\Microsoft\Windows Media
d.      This step opens the following folder on your computer:
SystemDrive:\Documents and Settings\UserProfileName\Local Settings\Application Data\Microsoft\Windows Media
e.      In this folder, rename the folder 10.0 to 10.0.bak. For more information about renaming folders, see To change the name of a file or folder.
Note: the folder name will be different for Windows media player 11.
f.        Start the Player and try to stream the content again.
