If you wish to do so using VBA code, you can try the following homemade method.

Public Sub fullyRenameWorkbook()
  'Temporary save the current workbook name with path
  Dim oldName As String
  oldName = ActiveWorkbook.FullName
 
  'Pick up the new name from an Input box
  Dim newName As String
  newName = Trim(InputBox("Requested new name:", "User input", "[Type here the new workbook name]"))
 
  'If the new name has been filled in, rename the workbook (by saving it under the new name and deleting the old workbook)
  If newName <> "" Then
    ' Add backslash if needed
    newName = IIf(Left(newName, 1) <> "\", "\", "") & newName
   
    'Fully replace the workbook
    ActiveWorkbook.SaveAs ActiveWorkbook.Path & newName
    Kill oldName
  End If
End Sub

john mcghie say:
First, I need to understand what you mean by "master page".  Technically, Word does not HAVE a "master page", except in Publishing Layout View, which is a horrible kludge that should not normally be used in a Template because it is not compatible with other versions of Word.

Image degradation is often a result of down-sampling to 72 dpi.  This is more likely to happen in .doc/.dot format in Word 2008.  Working in .docx/.dotx will give you a better result (but only sometimes).  Word 2011 would be a better choice for this kind of thing.

EMF is a great format to use on a PC: on a Mac it depends on the end users having an external converter installed.  EPS would be a better choice, but is not available in Word 2008.  Both EMF and EPS are likely to result in a huge file, depending on what is in the picture.

Use of PDF for graphics requires a knowledge of how PDFs actually work.  Basically, "PDF" is the container: the  question is: what is in the PDF?  If the PDF contains a bitmap image, Word will unpack it to retrieve the original image from it, and that will be subject to the same constraints as any image.

Word 2008 usually saves the original picture into the Word file, then creates a 72 dpi "placeable header" that looks really rough on-screen.  All going well, it should print OK, but it looks a bit nasty on the client's machine.

If you put an EPS image into the PDF, that will display and print with unlimited resolution in Word 2011; in Word 2008, you will get the down-sampling for screen display that is your current problem.

There is no 100% guaranteed-to-work way around this; but you will get a lot closer in Word 2011.

Hope this helps

Here's a possible way to deliver it in Excel ...
 
Assume your 14 colsource data is in sheet: x,
cols A to N, data in row2 down,
with the key col = col F (Category)

List in P1 across the various categories, eg: Admin, etc
Put in P2:  =IF(OR($F2="",P$1=""),"",IF($F2=P$1,ROW(),""))
Copy P2 across/fill down to cover the max expected extent of source data, eg down to row 1500?

Click Insert > Name > Define **
Put under "Names in workbook:":   WSN
Put in the "Refers to:" box:
=MID(CELL("Filename",INDIRECT("A1")),FIND("]",CELL("Filename",INDIRECT("A1")))+1,32)
Click OK
**In Excel 2007, it should be Formulas > Defined Name

The above defines WSN as a name we can use to refer to the sheetname in formulas
It will auto-extract the sheetname implicitly (Technique from a post by Harlan) 
Note: The workbook is presumed saved beforehand

Then, in a new sheet named after one of the Categories, eg:  Admin
With the same col headers pasted into A1:N1
Put in A2:
=IF(ROWS($1:1)>COUNT(OFFSET(x!$O:$O,,MATCH(WSN,x!$P$1:$IV$1,0))),"", INDEX(x!A:A,MATCH(SMALL(OFFSET(x!$O:$O,,MATCH(WSN,x!$P$1:$IV$1,0)),ROWS($1:1)),OFFSET(x!$O:$O,,MATCH(WSN,x!$P$1:$IV$1,0)),0)))
Copy A2 across to N2, fill down to say N200, to cover the max expected extent for any Category. Cols A to N will return only the lines for the clategory Admin from the source sheet: x, with all lines neatly bunched at the top. Now, just dress the sheet up as desired, then make copies of it. Rename each copy as the other Categories (a one time job) and you'd get the corresponding results for each of those Categories. When you log into http://www.outlook.com do you see Outlook.com or Outlook Mail in the top left navigation?  If it shows Outlook Mail then your account has been migrated to Exchange.

For Outlook 2013 and Outlook 2016 the best solution is to remove the account in Outlook and add it back and let Outlook automatically configure it as an Exchange Account.

For Outlook 2007 and Outlook 2010 to add the account as Exchange it is necessary to update the clients to a supported version and remove the Outlook Hotmail Connector. 

Outlook 2010, https://support.office.com/en-us/article/Add-your-Outlook-com-account-to-Outlook-2010-6636e88f-3d21-413b-9fd1-38a99dfcb199?ui=en-US&rs=en-US&ad=US.

In Outlook when you click File, Account Settings, Account Settings, does your account show as Type ExchangeActiveSync or Microsoft Exchange or any other type?  It likely shows as ExchangeActiveSync.  If so you will need to do the following to fix it:

In order to fix the issue we need the account to be configured as Exchange.  Lets do this.  We will add your account in Outlook again as a new account and it will configure as Exchange.  Then we will remove the one listed as ExchangeActiveSync.  Follow these steps:
1.  Click File, Account Settings, Account Settings
2.  Click New
3.  Enter your Name, your Outlook.com email address, and password and click next to have Outlook add the account.
4.  Restart Outlook

Your account will now show up twice in Outlook and the new one you added will be Exchange.  It should work fine and your mail should be in the right folders.
We still need to remove the original account that is still configured as ExchangeActiveSync.  Follow these steps to do that:
1.  Click File, Account Settings, Account Settings
2.  Locate your Outlook.com account that has ExchangeActiveSync next to it and click on it and click Remove.
3.  Restart Outlook.
