hairyfool say:

When in standby mode the computer is not off so the BIOS does not reload its hardware, memory state is maintained. The only way it could get "restart to BIOS" is if the system has crashed out completely and forced the BIOS to run Power On Self Test, the last phase of this is enumerating the hardware. At that point BIOS has failed to find the SSD, which in practice is an allowable condition as most BIOS support some hardware changes on the fly. The fact that it has stopped at the BIOS setup screen would indicate a change has occurred that the BIOS at least needs verifying.

Windows itself could crash as a part of the recovery from standby so the BIOS reinitialising could be down to software but part of the POST is to issue a reset to all Bus connections which should reinitialise the SSD if Windows had left it in an unstable state. This would preclude Windows preventing the SSD being detected following POST.

hairyfool say:

I don't want to rubbish hardware testing but always treat the results with caution.

When we used to build PCs every system went through full hardware test before loading the operating system. Roughly 3% would fail to load with the vast majority resolved by replacing memory, that memory that had passed testing through 8 different modes. Windows is about the best hardware test program going.

You might get further clues from the system event log but I suspect all you will find is "the previous shutdown was unexpected" as far as the standby recovery is concerned

Step 1 is again before Windows loads so A) Windows cannot prevent it starting and B) means that no event logs are generated. The failure to find the drive after multiple reboots HAS to be hardware.

Easy to say when you have a repair workshop behind you but about the only option is swapping out suspect devices.

Drive manufacturers can have diagnostic utilities on their web site, preferably as a bootable CD/USB image and this may identify problems with the drive but SSD diagnostics are not as definitive as HDD testing. S.M.A.R.T. logs will be held on the drive and the diagnostics will normally read them.
