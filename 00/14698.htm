jeanette cunningham say: To create a new record in an existing table.
Assume that the form with the combo box is not based on the table where you want to create the new record.

Use the method of opening a recordset, adding the new record, grab the PK for the new record, then open the form at the new record.

Here is sample code from one of my apps.

'---------1. create the new record
Public Function NewContactKey() As Variant
On Error GoTo Error_Handler
   
    'This function creates a new Contact record and returns the key.
   
    Dim db As DAO.Database
    Dim rec As DAO.Recordset
   
    NewContactKey = 0   'Default no record created
   
    Set db = CurrentDb
    Set rec = db.OpenRecordset("tblContact", dbOpenDynaset)
   
    'Add the record
    With rec
        .AddNew
        !ContactName = "Microsoft"
        .Update
        .Bookmark = .LastModified
       
        NewContactKey = rec!ContactKey
    End With
   
Exit_Procedure:
    On Error Resume Next
    rec.Close
    db.Close
    Set rec = Nothing
    Set db = Nothing
    Exit Function
Error_Handler:
    Select Case Err.Number
        Case Else
            MsgBox Err.Number & ", " & Err.Description
            Resume Exit_Procedure
            Resume
    End Select
End Function
'---- new record created

'---2. write code to open the form at the PK for the new record - NewContactKey

Notes: in the above code, ContactName is the name for the selection from your combo. The user has selected 'Microsoft' as the Contact name to use for the new record.

greginmich say:

Third-party AV Apps are supposed to turn Windows Defender off by registering with the Security Center – which sets the DisableAntiSpyware and DisableAntiVirus DWORDs in the Windows Defender registry key to the value of 1 (disabled true). When that’s done properly, the Windows Defender screen should still load when you try to start Defender, and it will provide the option for turning on Limited Periodic Scanning by clicking on the big “Turn On” button. Turning on Limited Periodic Scanning resets the respective DWORDs in the Windows Defender key to the value of 0, and this activates the options for manual scanning, and the entire Windows Defender Command Line Interface – which allows for direct command line scanning and updating, as well as for scheduling scans and updates with Task Scheduler:

https://www.tenforums.com/tutorials/51514-turn-off-limited-periodic-scanning-windows-10-a.html

It also activates the Windows Defender Automatic Maintenance tasks, including the Windows Defender Scheduled Scan task – which, as you might know, isn’t working properly at this point. You can, however, presently run manual scans; use the Startup Shortcut methods; or schedule your own Windows Defender scans/updates once Limited Periodic Scanning has been enabled. Since Limited Periodic Scanning is really a new feature for Windows 10, no third-party AV app should be disrespecting this protocol by adding a DisableAntiSpyware DWORD to the Polices key (the Group Policy key) and setting its value to 1 been isolated by breaking its internet connection – because this setting actually does turn off Windows Defender irrespective of whether or not a third-party AV app is installed. The Policies key, unlike the Windows Defender key, is accessible via the elevated (Administrator) command prompt:

Determine the current status of the Group Policy registry switch:

REG QUERY "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntispyware

Turn Off Windows Defender:

REG ADD "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware /t REG_DWORD /d 1 /f

Turn On Windows Defender:

REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware
