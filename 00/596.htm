If you want to see the details or edit it, you need to have the correct permission. Here are the 2 methods that you can choose:

word - parallel pages - Microsoft Community

a. user say:

word - parallel pages

Hi!

I would like to create a Word document for further printing it in a book-like version. I would like to have on the left page a table (with 2 columns, for translating a text, so having the original and translated text beside each other), and on the right page I have to give the explanation why and how I made the translating process. Thus, one table (with the translation) would be flowing through pages on one side (like on page 1,3,5...etc.) while beside on the other page you can read the collaborating explanation (like on page 2,4,6...etc.). How can I do this?

suzanne s. barnhill say:

Word doesn't make it easy to have this kind of setup. It's easy enough to set up the parallel table columns for original and translation, but, in order to have different content on the facing page, you would have to end the table, insert regular text, complete it, then insert another table.

This is actually not a totally bad idea. With such a parallel translation, it's a good idea to start a new row for each paragraph, to keep the original and translation in sync, and Word also becomes uncomfortable (which is to say, unstable) with single-row tables that extend beyond one page. Unless paragraphs are very long, you should be able to put one or more on a page. That should also make it possible to confine the discussion to a single facing page. You probably want to defer pagination until editing is complete.

You could approach this task using text boxes, but Word limits the number of linked text boxes to 32, which would give you a total of 64 pages (32 for each linked series). If you need that type of setup, then you're better off using page layout software. If you have a version of Office that includes Publisher, you could use that, though Publisher is less capable than Word of more complex book features such as headers/footers, footnotes/endnotes, TOC, etc. For high-end publishing purposes, you really need something like Adobe InDesign (see http://www.adobe.com/products/indesign/free-trial-download.html for a free trial offer).

At least 2 things to consider:

If you use many (any?) 3rd party add-ins, they may not work in 64-bit version of Office/Excel.

 

The key to the error message is just 2 words "available resources" - and that does not necessarily refer to your RAM.  Certain areas in Excel are predefined with a size limit, and once you hit that limit, all the RAM in the world won't help.  I've got an i7 system with 12GB RAM on it and just recently I had a file that had so many empty!! columns and such on a sheet that I was completely unable to simply delete them -- not enough available resources.  Sometimes it'll give you an option to continue but without Undo capability, but it didn't even make that offer.  I basically had to scrap the workbook, luckily it was just a little piece of some development I was working on.

 

One last thing: you can't mix 32/64 bit versions of Office.  So if you plan on perhaps keeping an older version on the system when you upgrade later, you're kind of "stuck" with the bit version you choose initially.
