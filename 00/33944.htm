This has nothing to do with the new bug. This behavior only started recently, and it's affecting a lot of people. Steps to reproduce:

1. Navigate to a folder with images in it.

2. Double-click on an image, or drag and drop it into an application.

3. After closing the application or viewer, try to delete the image.

The images can't be deleted on affected systems. This is new behavior, has nothing to do with a preview pane or anything else. The file is not open in Windows explorer, and for the past 20 years, there was no problem deleting a file after using it - now, Windows is leaving a handle attached to the file, and there's no way to delete the file short of killing the primary explorer.exe process.
