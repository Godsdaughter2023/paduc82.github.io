scottgem say:

The problem here is more in your structure. Instead of having what sounds like 10 fields (5 checkboxes and 5 amounts) you should have a child table like so:

tblPricingOptions
PricingOptionID (PK Autonumber)
ParentID (FK)
OptionCode
OptionAmount

Then you add records to that table only for the options being used.

You can't suppress the display of column in a query based on another value, but you can do this on a report. But again, the issue is tyour database design which is not normalized.

I do not understand how you are using the query, but hiding columns is not an option in a query.  You can hide columns in a datasheet form by setting its control's ColumnWidth to 0 or by setting its CilumnHidden property to Yes (or True in VBA).
