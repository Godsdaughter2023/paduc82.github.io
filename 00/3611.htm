The second part returns corresponding records when the user does enter a value.  The problem is that 'Like "*"' does not return null values.  So if the source table is missing data for [SearchCriteria], the record in question won't be returned.  I need to be able to query all values, including null.

The search criteria 'Like "*" Or Is Null' does return any value, including null.  However, I can't figure out how to integrate the 'Or Is Null' part into my if statement in a way that works.  Can someone help me out with syntax here?  I've tried every permutation I can think of, and nothing seems to work.

john w. vinson say:

If (and I may be misreading) you want an exact match if the user enters a criterion, and all records if they don't, then I would recommend not using the LIKE operator at all. It's really designed for partial matches.

A criterion of

(FieldName = [Forms]![frmCaseSearch]![SearchCriteria] OR [Forms]![frmCaseSearch]![SearchCriteria] IS NULL)

will retrieve just that: either an exact match if there's a criterion, or all records (that is, all records which match any OTHER criteria specified) if there isn't.

john w. vinson say:

Hope you have a backup of the database, or at least of the data that went into it.. :-{(

A Query is just a VIEW of the data in your table. When you copy and paste data in a Query, it is actually updating - overwriting and permanently destroying - the data in the table.

Query datasheets are not particularly good for data presentation (this experience is just one reason, they are MUCH too vulnerable to data-destroying mistakes!). You should instead create a Report or a Form based on the query, and do your layout work there instead of in the query.
