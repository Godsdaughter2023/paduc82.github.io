ken sheridan say: Don't even think about creating multiple contact columns etc in the table.  That would be very bad design and create a lot of headaches for you in querying the data.  As you could have a situation where the same person is an owner of more than one property, and each property can be owned by multiple people you have a many-to-many relationship type between properties and owners, so you'd model this with three tables along these lines:

Properties
...PropertyID (PK)
...AddressLine1
...AddressLine2
...etc

Owners
...OwnerID (PK)
...FirstName
...LastName
...etc

and to model the relationship type:

PropertyOwners
...PropertyID (FK)
...OwnerID (PK)

toronto glen say:

This is a common error that many people make with excel; what you see is not always what you get.

Although you see Hours:Minutes:Second excel is actually storing the information as Days.

Each day value (24hours) is = 1; hence 3days time $22 = $66

In order to change days into hours, you need only multiply it by 24. Add the *24 and then divide it into the number processed.

SUM(C4*$22.00)*24

Remember, what you see on the screen is not how the numbers are being stored. An easy way to see the actual number in a cell is to set the format to General.

Hope this helps!
