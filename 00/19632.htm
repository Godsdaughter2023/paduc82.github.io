daniel pineault say:

Not sure I understand the question.  Forgive me if I am misunderstanding the question, but open the table in design view and change the property and save the changes.

 

Alternately, if you are looking for an automation approach then you could repurpose a functin like:

 

 

Function FixZLS()
    Dim db As DAO.Database
    Dim tdf As DAO.TableDef
    Dim fld As DAO.Field
    Dim prp As DAO.Property
    Const conPropName = "AllowZeroLength"
    Const conPropValue = False
   
    Set db = CurrentDb()
    For Each tdf In db.TableDefs
        If (tdf.Attributes And dbSystemObject) = 0 Then
            If tdf.Name <> "Switchboard Items" Then
                For Each fld In tdf.Fields
                    If fld.Properties(conPropName) Then
                        Debug.Print tdf.Name & "." & fld.Name
                        fld.Properties(conPropName) = conPropValue
                    End If
                Next
            End If
        End If
    Next
       
    Set prp = Nothing
    Set fld = Nothing
    Set tdf = Nothing
    Set db = Nothing
End Function

 

 

by changing Const conPropValue = False to Const conPropValue = True

suzanne s. barnhill say:
Pressing Alt while dragging the table borders should allow you to align the outside edges more precisely. If you have text boundaries displayed, you may see some discrepancy that would be helpful (such as a difference in the cell margin). If it's just the right edges you need to align, you could align both tables Right. It could be that the header paragraph has an indent (positive or negative) that would affect the table position. There are also many Compatibility Options that affect table formatting, including settings that determine whether tables are allowed to extend into the margins.

anula d'souza say:
You have to manually adjust the table width:
If you want all the columns of same width, select the complete table, click on Layout< Properties under Table group< Column< adjust the preferred width appropriately.
If you want to adjust only the last column so that all the tables are aligned properly you can highlight the last column lick on Layout< Properties under Table group< Column< adjust the preferred width appropriately.

There's nothing to do.  Theoretically installing the same product key on the exact same hardware should generate the exact same signature sent to the activation server and shouldn't "burn" an activation.
