kyle82185 say:

The Outlook 2003/2007 default NK2 file location is the same in Windows 7 x64 and x86 -- C:\Users\{username}\AppData\Roaming\Microsoft\Outlook\zzzzzzz.NK2 (z = Outlook profile name). This also applies to Vista and Server 2008.

The Outlook 2010 NK2 file is actually a DAT file and is located, on the above operating systems, at C:\Users\{username}\AppData\Local\Microsoft\Outlook\RoamCache\Stream_Autocomplete_y_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.dat (y = file index, x = random 16-byte key in hex form).

 

If you haven't unchecked 'Hide extensions for known file types' under Tools > Folder Options > View, then the file will just be called "Outlook" or "Default" (whatever your Outlook profile is named), instead of Outlook.NK2 or Default.NK2. Also, Windows 7 classifies it as an "Office Data File" in the 'Type' column.

If this file is not present, Outlook will create a new one when you open the program. Hope this helps!

don cuthbert say:

For 64-bit Windows 10 on most PCs it should be this file:
http://download.windowsupdate.com/d/msdownload/update/software/secu/2017/12/windows10.0-kb4054517-x64_f1c0e8eae96b357bc13e5668c5874904502106df.msu

To try an offline install of the downloaded update, create a folder with a short name, that is easy to find, such as C:\UPDATE and move the downloaded .msu file to it. Also create a folder called C:\SCRATCH to use for temp files during the install.  Then use SHIFT-RESTART to get a boot menu that includes a Command Prompt option (under Troubleshoot - Advanced Options).

Once you have logged-in to your Command Prompt, use a DISM command to install the update to your offline Windows. First confirm that Windows is still found on C:. If it is on D: or another letter in this mode, adjust the command line to match:

DISM /Image:C:\ /Add-Package /PackagePath:C:\UPDATE /IgnoreCheck /ScratchDir:C:\SCRATCH

Windows will now show version 16299.125, under Settings - System - About.
