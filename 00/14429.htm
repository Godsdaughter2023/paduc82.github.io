vanessa ouyang msft say:

Hi Rich,

Email attachments should be in "message/RFC822" attachment format generally. RFC 822 is a mime sub type. Section 5.2 of RFC 2046 addresses the "Message Media Type", and section 5.2.1 addresses the "RFC 822 sub type". The full internet headers and body of the message should be retained exactly as the message was received and forwarded intact as an attachment.
To ensure you can use normally, just drag&drop the old email into your new mail. If you open it then it looks bad (TextEdit ...) but don't care about it. You better not open it. Just send your new email. As soon as the email has been sent you see, if you open it, the embedded email as an envelope icon. If you open this icon the embedded email should now look correct and it should not be in the TextEdit format.

I would have your code make a copy of the Excel file, edit it to remove the two columns, save it and then do the import.  Then you can have it delete that Excel file.

 

Dim strPathAndFile As String

Dim strNewPathAndFile As String

Dim objXL As Object

 

strPathAndFile = "C:\MyFolderNameEtc\FileName.xlsx"

strNewpathAndFile = Left(strPathAndFile, Len(strPathAndFile)-4) & "_copy.xlsx"

 

FileCopy strPathAndFile strNewPathAndFile

 

Set objXL = CreateObject("Excel.Application")

 

With objXL

   .Workbooks.Open strNewPathAndFile

   .Workbooks(1).Worksheet("WorksheetName").Range("F:G").EntireColumn.Delete

   .Workbooks(1).Close True

End With

 

' then do your import using strNewPathAndFile
