plankwing say:

The solution for W10 to see the full extent of 4tb HDs is neither short nor sweet. Core of the problem is W8/W10 cannot see full extent of a 4tb HD without specific drivers (depending on which version of the opsys is

If one's computer is already set up with UEFI and the correct drivers for opsys prior to upgrading to W10, Microsoft has not included the ability to see that setup so installs its default driver controller thus hosing the ability for W10 to see the full extent of the 4tb HD. Not cool.

Before installing the opsys, go to the mobo web site and download the correct drivers for the planned version of W, in my case, W10 64 Pro. Instructions with the download usually require the correct drivers to be copied to the root directory of a floppy, CD, or USB drive and placed into the appropriate port. THEN the process of installing the opsys includes/requires a step where the install routine has to grab the drivers from the floppy, CD, or USB.

The not-so-short and not-so-sweet part is, that if the opsys is already installed, one will have to "roll back" and reinstall using UEFI process. Note that the board must support UEFI and that be specified in the BIOS for the drive/s.

Having to "roll back" and reinstall with the correct drivers is not a short and sweet task. That does not take into account it took me many phone calls, emails, and forum searches over two days to find the fix after the W10 update hosed the drivers. Worse, no one at Microsoft, Seagate (affected drive), or Partition Magic (recovery software) had a clue about what caused the problem or of the solution.

Note to Microsoft: When W10 update is installed onto existing W8 opsys that already has the correct "extended" drivers and UEFI for larger HDs, these drivers should be seen so the correct drivers will be found and installed so the full extent of the HD capacity is seen so not "hosed" by the default Microsoft drivers as is happening with a number of "us" out here.
