The short answer is that some AV products that disable Defender do so through changes to Group Policy.  I believe the method used may vary depending on your operating system and how things were originally configured.  It can most certainly be done that way rather than disabling the service.  Mine doesn't use Group Policy to do it; however, I have heard of cases where others have said the same thing as you.

 

Regarding Windows Defender the most likely reason it is deactivated at all is because you installed an AV program or Security Suite that did so automatically during its installation (and it was probably necessary for the new software to work properly without causing any problems).   This is certainly true if you installed MSE.

 

This is the most common method to enable or disable Defender: http://www.groovypost.com/howto/microsoft/vista/disable-windows-defender-service-in-windows-server-2008-or-vista/ (and pretty much works the same in Windows 7).  In XP, Defender must be uninstalled if you are using MSE (thus no Group Policy is required as it won't exist).  This is how it occurs on my system (but the method employed is not as important as the fact that it is accomplished when needed by whatever method - as appears to be the case here so you are fine - in Windows, there are often many ways to do the same thing and often it's simply a matter of choice or how things are already configured).

Windows Defender full scan and quick scan - Microsoft Community

alasdaironline say:

Windows Defender full scan and quick scan

when does window defender choose to run a full scan over a quick scan? As far as I can tell since I updated to Win 10 approx  2 months ago and  it never has ran a full scan.

[Original Title: windows defender]

greginmich say:

when does window defender choose to run a full scan over a quick scan? As far as I can tell since I updated to Win 10 approx  2 months ago and  it never has ran a full scan.

Windows Defender never runs an automatic Full Scan – by design. The MSE scheduled scan is allowed to run when the PC is in use, so it can accommodate long-running scans. But the Automatic Maintenance scan in Windows 8/8.1/10 is a horse of an entirely different color – it's only allowed to run when the PC isn’t in use (during system idle states), and it has to start over from scratch anytime it gets interrupted by user activity. Furthermore, it gets automatically terminated at the end of a 1-hour maintenance window – so Automatic Maintenance just wasn’t deigned for running Full Scans:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender/588b2978-dc88-432b-bc8e-066acf6b9258

You can either schedule a Full Scan manually, by following the steps in the above thread, or you can create the task automatically with SchTasks.exe, and then make any desired changes after the task is created: Right-click on the Start button and select Command Prompt (Admin). Then copy, paste, and run the appropriate command line:  

Windows Defender Quick Scan:

SchTasks /create /sc DAILY /tn “Defender Quick Scan” /st 19:00  /ru SYSTEM /rl HIGHEST /tr "'C:\Program Files\Windows Defender\MpCmdRun.exe' -Scan -ScanType 1"

Windows Defender Logon Scan (Quick Scan) with 2 minute delay:

 

SchTasks /create /sc ONLOGON /delay 0002:00 /tn “Defender Delayed Logon Scan” /ru SYSTEM /rl HIGHEST /tr "'C:\Program Files\Windows Defender\MpCmdRun.exe' -Scan -ScanType 1"

Windows Defender Full Scan:

SchTasks /create /sc WEEKLY /tn “Defender Full Scan” /st 19:00  /ru SYSTEM /rl HIGHEST /tr "'C:\Program Files\Windows Defender\MpCmdRun.exe' -Scan -ScanType 2"

You can modify the scheduled start time (/st 19:00) either before running the command line, or in the Task Scheduler Library after the task is created. This scan should be set to start at a time when the PC will normally be in use, since it won’t run when the PC is sleeping; and won't run on restart when a scheduled start is missed. If you want a more reliable scan, then you’ll have to add these features. You would check the Wake the computer to run this task box (Conditions tab), and the Run task as soon as possible after a scheduled start is missed box (Settings tab).

Note that you might have to turn off Require a password on wakeup in Power Options and/or enable wake timers in Power Options > Change when the computer sleeps > Change advanced power settings.

Also be advised that if you want your scheduled scans to run at full speed, then you’ll have to run this command at the elevated PowerShell prompt:

Set-MpPreference -ScanAvgCPULoadFactor 100

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/processor-speed-100/7306888a-9b1d-451c-b7c2-7e6e5e79b245
