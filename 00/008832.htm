Apparently the file I quoted is not the live building blocks file but an original file installed with the application. Apologies for that.

The active file is stored at
C:\Users\<UserName>\AppData\Roaming\Microsoft\Document Building Blocks\1033\14

where 1033 is the language code. - http://msdn.microsoft.com/en-us/goglobal/bb964664

When Word starts it will read whatever is stored in that folder. The templates stored there loaded and the building blocks they contain are added to the building blocks list. While the file is open in Word a normally hidden lock file is created in the same folder. If you set Windows Explorer to display hidden files, you should be able to see it.

If the building blocks template is missing, Word will create a new one (probably from the file I mentioned earlier). This mechanism can be impaired by the presence of an orphaned lock file.

I think the next step would be to delete everything from the folder
C:\Users\<UserName>\AppData\Roaming\Microsoft\Document Building Blocks\1033\14
with Word closed.

Rename the normal template from normal.dotm to normal.old and delete any lock files in the user templates folder (which by default is - C:\Users\<UserName>\AppData\Roaming\Microsoft\Templates)

See http://www.gmayor.com/what_to_do_when_word_crashes.htm for more information on lock files.

Now start Word. Word should create a new building blocks template in the folder at the top of this message with default entries. Now check Insert > Quick Parts > Building Blocks Organiser and establish that the entries are present.
