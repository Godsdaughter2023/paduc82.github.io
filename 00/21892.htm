That's hard to do using a formula - I'm not going to try.

 

I would use a macro that would go something like this
(I'm assuming each of sheets A to F has a row of headings in row 1 and contiguous data below the heading)

 

Sub GetHighScoreNames()
  Dim vSheet, vSheets
  Dim R As Range
  Dim WS As Worksheet

  Dim lRow As Long
  Set R = Worksheets.Add.Range("A1")
  vSheets=Array("A","B","C","D","E","F")
  For Each vSheet In vSheets
    Set WS=Worksheets(vSheet)
    With WS.Range("A1").CurrentRegion
      For lRow = 2 To .Rows.Count
        If .Cells(lRow, 2)>=0.8 Then
          R.Value = .Cells(lRow,1).Value
          Set R = R.Offset(1)
        End If
      Next

    End With
  Next

End Sub

Can't open database in Access 2007 - Microsoft Community

dmkg say:

Can't open database in Access 2007

A database originally in 2003 format was converted to the latest .accdb format in Access 2010 and works fine in 2010 but it won't open in Access 2007 because of "unrecognised database format".  Are databases created in Access 2010 not compatible with Access 2007 or do I need to save it in some other format?
nathan o. - microsoft support say:

Hi dmkg

 

After converting the database, did you make any design changes to any of the objects while in Access 2010, create any new objects, or toggle any of the objects to layout view?  If you do edit or create new objects in Access 2010 and use any of the new features, toggle to layout view, and use certain wizards this could prevent you from opening the file in Access 2007, especially if any of these new objects have code behind them. 

 

The reason I mention layout view and / or the wizards is because when these features are used, they place empty cell controls on your forms and reports, which are used to help preserve the layout of the controls on these objects (especially in web databases). Once the object has any of these empty cell controls on them and has VBA code behind the object, you most likely will see the error message you are seeing when trying to open the database in Access 2007, because unfortunately these empty cell controls are new Access 2010 controls. 

 

In order to open the database in Access 2007, the best workaround I have seen is to go into design view of all of your objects and remove these controls. You will then need to create a new database in Access 2010 and import all of the objects into it. At that point, you should be able to open the new file in Access 2007. Unfortunately, just importing into a new database will not work. You have to remove the empty cell controls first.

The marked answer in this thread discusses this -http://social.msdn.microsoft.com/Forums/en-US/accessdev/thread/99e2eedf-0a36-4d38-9713-f3c82018c804
