daniel g. grau say:
Make sure you have selected the U.S. keyboard in System Preferences. Here, if I select the German keyboard, page numbers in new citations are inserted with the "S." If I then select the U.S. keyboard, page numbers in new citations are inserted as "p." for single pages and "pp." for page ranges.

Citation fields are locked and the language specified when the citation was initially inserted cannot be changed.  (Use toggle control Opt F9 to see the languages: "\l 1031" is German and "\l 1033" is US English).

To make Word use the English "p." and "pp." abbreviations, you need to add a citation anew while using the proper keyboard layout.

If you have many citations that you need to correct, use this macro:

'--------------------------------------------

Sub ChangeCitationLanguage()

'This macro will change the language code from "1031"

'(German, appearing in two places) to "1033" (English).


Dim CurrentField As Field

 

ActiveWindow.View.ShowFieldCodes = True

FieldCount = Application.ActiveDocument.Fields.Count

For FieldPointer = 1 To FieldCount

    Set CurrentField = Application.ActiveDocument.Fields(FieldPointer)

    If CurrentField.Type = 96 Then

        CurrentField.Select

        CitationText = Selection.Text

        CitationEnd = Len(CitationText) - 2

        CitationText = Mid(CitationText, 2, Len(CitationText) - 2)

        If InStr(CitationText, "1031") > 0 Then

            Selection.Delete

            CitationText = Replace(CitationText, "1031", "1033")

            Selection.Fields.Add Range:=Selection.Range, Type:=wdFieldEmpty, _

                Text:=CitationText, PreserveFormatting:=False

        End If

    End If

Next

ActiveWindow.View.ShowFieldCodes = False

 

End Sub

'--------------------------------------------

If you don't know how to run a macro, see Install a macro in MacWord.

If you add content to a Publisher page, it does not 'push down' the content below like in Word. If you add more text than can be displayed in a text box you have to either enlarge the text box or link it to another text box. And if you lengthen a text box then you have to manually move everything below it to make room including a 'footer grahphic', or in other words move the 'picture box' down to make room for the additional text.

One option you could consider is to insert your footer image 'inline' in a text box. It will then move with text within that text box much as it would in Word.

Web pages templates are by default very long because when you convert them to html Publisher will automatically truncate the page after the last design element creating a custom length page. So if you are using the email template and want to convert to pdf you would want to change the length of the page after you have finished the design and page layout and before you converted it to pdf.

Bottom line is if you want everything to be automatically pushed down on a page then you should probably stick with Word. Publisher is better at page layout but you have to move the design elements around manually.

Spotlight can be a bit moody on occasions, but that hen things get complicated.
 
If you can't find anything in Outlook or through the main Spotlight
menu, it could be either because the spotlight index is corrupted or
because one of the mdimporters (the descriptors that tell Spotlight
what to index and how) is interfering or that it makes Spotlight crash
before it's done indexing.
 
To delete and recreate the whole Spotlight index, you need to use:
sudo mdutil -E /
in the Terminal to delete the index and force Spotlight to reindex
everything (which will take hours).
 
You can also check ~/Library/Logs/DiagnosticReports and
/Library/Logs/DiagnosticReports to see if you have any crash log in
there for anything with md in its name (mdimproter, mdls…).
 
mdimport -L will list all the registered md importers. You should  find
the one for Outlook in the list.
the entry should look lile that:
   "/Applications/Microsoft Office 2011/Microsoft
