You may use either of below as an array formula (CTRL+SHIFT+ENTER):

=MATCH(9.99999999999999E+307,IF($AH$3:$BH$3>0,$AH$3:$BH$3))+COLUMN($AH$3)-1 

or

=MATCH(9.99999999999999E+307,IF(3:3>0,3:3))

Here's a couple of things about scheduled scans. To run, the PC must be on and connected to the internet (it cannot be shutdown, in sleep mode, in hybrid sleep mode, or in hibernation). The power saving settings must be such that the PC does not shut down or go asleep during the scan (or it will be as if it never happened). Scans do not begin precisely when scheduled - some can start an hour or so earlier or later than the officially set time (depending on Microsoft's servers and the loads at that time).

john - project say:

Chris,

As I re-read your post I'm a little confused by your opening statement. You indicate it is a formula in the Text2 field which means it is a custom field formula. Yet the second sentence says, "it appears to run once a day when the file is opened...". A custom field formula will "run" any time the expression of the formula changes, not just when a file is opened. Are you again confusing custom field formulas with macros? An Open Event macro will run when the file is opened, and only when the file is opened.

So I have to wonder, what exactly are you doing. What is the formula for Text2? When does the macro, which is supposed to react to Text2, do?

If indeed you have a custom field formula in Text2 and you want to initiate some other action (e.g. macro) when Text2 changes, you will need a Change Event macro. If you can wait to run the macro until the file is closed or opened the next time, then you could use an Close or Open Event macro. The latter two are easy to implement and do not require a class module whereas the Change Event macro requires a class module. Also, a problem with a Change Event macro is that it will "fire" on every change, not just changes to Text2 so the code needs to be thought out carefully to only take action when Text2 changes, and that's not always so easy.

That's the general answer. Hope this helps. You can get more information on Event macros from the VBA help files. And since you have been asking questions about Project formulas and macros I suggest you consider getting a copy of Rod Gill's book on Project VBA. You can find out more at:

http://www.project-systems.co.nz/project-vba-book/index.html
