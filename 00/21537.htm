This is a perennial problem to which, AFAIK, there is no real solution except to print those pages separately and duplex them manually. A rule to look for "BlueHat Editor" in the message header might work

or you could run a script from a rule on all incoming messages e.g.

Sub MoveBlueHat(Item As Outlook.MailItem)
    With Item
        If InStr(1, .SenderName, "BlueHat Editor") > 0 Then
                    Item.Move Session.GetDefaultFolder(olFolderInbox).folders("FolderName")
        End If
        .UnRead = True
    End With
End Sub

Where FolderName is the sub folder of inbox into which you wish to move the message.
