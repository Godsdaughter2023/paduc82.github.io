Legend labels not displaying in chart with dynamic row source - Microsoft Community

jacobpage say:

Legend labels not displaying in chart with dynamic row source

I have a form containing a combo box (to select a type of item) and a stacked column chart showing number of item deliveries per agency per day. Because there doesn't seem to be a way to pass query parameters to a chart, I've had to create a query without parameters that contains the data I need and set that at design time so I could set up the chart, then have a change event handler for the combo box to create the dynamic SQL for the chart on change.

This sort of works. I am able to see everything correctly at design time. The chart has a legend that shows the name of each agency with a color code, and the stacked columns are colored based on the agency. However, once I populate the row source dynamically in the VBA, the legend no-longer shows the agency names, just the colors.

How can I get the legend to be populated correctly when dynamically changing the row source?

Here's what I've tried so far:

  • Using the link master/child properties - this does not work because the form is unbound. I'm not sure in this case how a bound form even makes sense, so this is a dead end.
  • Having the design-time query reference a form field. When attempting this, I get an error "The Microsoft Access database engine does not recognize '[Forms]![Item Delivery History]![itemCombo]' as a valid field name or expression."
  • Populating "Transformed Row Source" in the VBA. I'm told this is read-only.

If it helps, the query looks as follows (the value of the combo box is appended to the string, the query doesn't actually reference the control because of that error mentioned earlier):

SELECT

  it.name AS item_name,

  DateValue(o.date_order_filled) AS delivery_date,

  a.name AS agency_name,

  oi.quantity_requested AS quantity
FROM ((dbo_item_types AS it

  INNER JOIN dbo_order_items AS oi ON it.item_type_id = oi.item_type_id)

  INNER JOIN dbo_orders AS o ON oi.order_id = o.order_id)

  INNER JOIN dbo_agencies AS a ON o.agency_id = a.agency_id
WHERE

  oi.item_type_id = [Forms]![Item Delivery History]![itemCombo]

  AND o.date_order_filled Is Not Null
ORDER BY DateValue(o.date_order_filled), a.name;

The X axis is delivery_date, legend is agency_name, and Y axis is quantity sum. Any idea how I can get the legend to show up properly at runtime after dynamically changing the row source.

dhookom say: You don't need bound forms to link master/child. Just type in the control/field names rather than using a builder or wizard.

The Row Source of the chart is typically a crosstab query that requires providing the data type of the parameters in the query. This.allows referencing the form controls.

I'm not sure if this will fix the labels issue.

jacobpage say:

Thanks, dhookom. That's annoying that the button for establishing the link shows an error message leading me to think linking doesn't work, but you're right that typing in the control name worked.

Sadly, this did not completely fix the legend labels issue. However, some of the legend labels now show up, but some do not. I'm not sure what differentiates those that do and those that don't. It's not the length of their titles, because some really long ones show up where others do not.

jacobpage say:

OK, I have at least found a workaround in my form load event:

Private Sub Form_Load()
    For Each series In chart.ChartSeriesCollection
        series.DisplayDataLabel = True
        series.DisplayName = series.name
    Next
End Sub

This populates the display name for everything and sets the data labels. Sad that this is necessary and that they can't be universally populated, but this seems to work.
