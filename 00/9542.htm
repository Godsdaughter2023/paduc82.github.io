Master Page Editable Regions "Pastable"? - Microsoft Community

jimmiea say:

Master Page Editable Regions "Pastable"?

Hello,

We've created detailed text formatting in the master page area of a PowerPoint presenation.  We know that some users will be happy with the number of text areas given to them to use, but some will at some point want to copy & paste these text areas to create more boxes on some slides. 

However, when the box is copied & pasted in the main body of the slide, the cool formatting does not carry over with the copied box.  Is there a way to make a text area from the master page "pastable" if a user wanted to create multiple text areas on a slide?  Or is the only real solution to have multiple master slides with a number of text boxes to use?

Thanks!

rich007a say:

You need to be clear about Slide Masters and Custom Layouts.

If you go to the Slide Master View in ppt2007, you will see a kind of tree view on the left.  The top of the tree (slightly larger thumbnails) are slide masters.  Each slide master (there may be only 1) can have 1 or more Custom Layouts.

To get round your problem, you should create other custom layouts that can be applied with increasing numbers of Placeholders.  You can right-click on a Custom Layout and duplicated it.  You can then use Insert Placeholder on the Slide Master tab to add additional placeholders.  These will inherit the settings of the placeholder you put on the Slide Master.   Final step is to right-click on the Custom Layout thumbnail and rename them something sensible.

Then you just need to teach your users to change the layout to one with the necessary number of Placeholders when they initially create the slide.

 

If you really need to add a placeholder (with formatting) to an already created slide, where the layout has been modified a lot on the slide, the work around is to do the following:

Insert a new slide using the Custom Layout that contains the formatted placeholder.

Enter the text and set bullet levels as required.

Copy that placeholder from the newly inserted slide and paste it in your slide to be modified.  It still wont be editable, but providing you have finished entering the text and setting bullet levels, etc., that should not be an issue.

 

I know, painful isn't it.

 

Hope that helps.

Sub Worksheet_Change(ByVal Target As Range)
'Exit Sub
Dim SaveArray() As Variant
Dim Cell As Range
Dim Counter As Long

Application.EnableEvents = False

ReDim SaveArray(1 To Target.Cells.Count, 1 To 3)
If Target.Count > 1 Then
    With Worksheets("Sheet1")
        For Each Cell In Target.Cells
            Counter = Counter + 1
            If Left(Cell.Value, 1) = "P" Then
                SaveArray(Counter, 3) = Cell.Offset(0, -1) * 0.01
            End If
        Next
        Worksheets("Sheet1").Range("D65536").End(xlUp).Offset(1).Resize(Counter, 3).Value = SaveArray
    End With
End If
Application.EnableEvents = True

   
End Sub

 

andreas killer say:

If you want to manage multiple cells in the Change event, cut off all unused cells and use a FOR EACH loop to get each cell from the Target.

Andreas.


Sub Worksheet_Change(ByVal Target As Range)
  Dim R As Range
  'Act only in the used range of this sheet
  Set Target = Intersect(Target, Target.Parent.UsedRange)
  'Finish if out of this range
  If Target Is Nothing Then Exit Sub
  Application.EnableEvents = False
  'Visit each cell
  For Each R In Target
    Select Case R.Column
      Case 3
        If R.Value < 1 Then
          R.NumberFormat = "0.00%"
          R.Offset(0, 1) = "Percentage"
        Else
          R.NumberFormat = "0.00"
          R.Offset(0, 1) = "Dollar"
        End If
      Case 4
        If R.Offset(0, -1) = 0 Then
          R = "Percent"
        Else
          If Left(R.Value, 1) = "D" Then
            If R.Offset(0, -1) < 1 Then R.Offset(0, -1) = R.Offset(0, -1) * 100
            R.Offset(0, -1).NumberFormat = "0.00"
          Else
            If R.Offset(0, -1) >= 1 Then R.Offset(0, -1) = R.Offset(0, -1) * 0.01
            R.Offset(0, -1).NumberFormat = "0.00%"
          End If
        End If
    End Select
  Next
  Application.EnableEvents = True
End Sub

Hi Peter,

It looks like you will have to use AppleScript to change the ActivePrinter. Here is your code once again. You'll change where is says "EPSON00E*EF (IP)" to your printer's name. To make this work it appears that you have to first change the default printer for OS X and then change the ActivePrinter for Word. The script captures the prior setting of the default printer so if you want to reset it afterward you'll have to run another AppleScript that sets it back.

The final thing I noticed is there's a time delay from the point where the script changes the settings and when both OS X and Word recognize the switch. You might have to enter a delay factor to compensate.

Good luck

Sub PrintToSpecificTrays()
        Dim sScript As String, result As String, oldPrinter As String
        sScript = _
        "set applescript's text item delimiters to "","" " & vbNewLine & _
            "property newPrinter : ""EPSON00E8EF (IP)""" & vbNewLine & _
            "tell application ""Printer Setup Utility"" " & vbNewLine & _
            "set oldname to name of current printer" & vbNewLine & _
            "set current printer to first printer whose name is newPrinter" & vbNewLine & _
                "tell application ""Microsoft Word"" " & vbNewLine & _
                    "Activate" & vbNewLine & _
                    "active printer is newPrinter" & vbNewLine & _
                "end tell" & vbNewLine & _
           "end tell" & vbNewLine & _
        " set applescript's text item delimiters to """" " & vbNewLine & _
        "return oldname"
    
        result = MacScript(sScript)
        oldPrinter = result
