sharonmary say:

Hi Karen,

There's no indication of which Office version you are using but the instructions below should work - or point you in the right direction. I would recommend them in the order listed.

Check unavailable mapped drives

 

If Word crashes you or hangs when you save a document, the first thing to check is that you don't have any unavailable mapped drives. To disconnect a mapped drive:

 

1. Click on the Start button

 

2. Go to MyComputer

 

3. Click on Map Network Drive

 

4. Select the unavailable drive

 

5. DeselectReconnect at logon

 

6. Repeat steps 5 and 6 for each unavailable drive.

 

 

Open Word in Safe Mode

 

Check if you can save files in Safe Mode. Here's how:

 

1. Press the <Ctrl> key while you start Word

 

2. Click Yes at the prompt to open in Safe Mode.

 

3. Can you save the file? If yes, start Word normally.

 

4. Click on theMicrosoft Office Button

 

5. Click on theWord Options button

 

6. Click onAdd-ins and check the active Application Add-ins listed. It's quite likely that one of the Add-ins is interfering.

 

*Some fixes I have read recommend deleting the Normal template. You could always try this ... on the understanding that you may lose some personalization as Word will create a new default template next time you open it. 

 

 

Run the Office Repair tool

 

Have you tried running the Office Repair tool? If not, have a look at this knowledge base article:http://support.microsoft.com/kb/924611

Scroll down to the heading "How to repair problems in the installed 2007 Office programs and features"

 

I would suggest Method 1 first if you can as it's directly from an Office application. Otherwise, Method 2 works just fine. 

 

Method 1: Run Office Diagnostics from a 2007 Office program
Method 2: Run the Detect and Repair tool from Add or Remove Programs
 

 

Delete the Word Data registry key

 

If the Office Repair tool doesn't do the trick, try the fix in "How to troubleshoot problems that occur when you start or use Word 2010, Word 2007, Word 2003, or Word 2002": http://support.microsoft.com/default.aspx/kb/921541?p=1

 

I was able to resolve a similar problem recently by following the instructions to Delete the Word Data registry key under Manual steps to troubleshoot problems that occur when you start or use Word.If you follow this method do this be sure to follow the steps in Microsoft Knowledge Base:322756  "How to back up and restore the registry in Windows" (nowhere near as scary as it sounds!)

 

I hope that helps. If any of the above solutions work do let us know for the benefit of future visitors.

davidf2 say:

You can import or insert a Word file by first creating a text box > Insert tab > Text group > Insert File and browsing to where your Word file is stored. Publisher will automagically create as many pages and text boxes as necessary and flow the Word doc.

You can also add the 'Edit Story in Microsoft Word' icon/function to the menu. Right click the menu/ribbon where you want it > customize ribbon and find the command in the list of available commands. This will allow you to open the Word doc/story in Word for editing.

You can adjust character spacing by selecting the text > right click > change text > character spacing. You can adjust line spacing under Paragraph settings.

With that said if you cannot format/typeset your manuscript using the tools in Word, a word processor, you will probably not be happy with the more limited word processing tools in Publisher, a DTP. Stay with Word for word processing.

mel77084 say:

Microsoft assigns a number to each build to identify it, and since a particular build is made available to different ones at different times, you leave us in the dark when you describe a build as "the latest Windows Insider release". With less ink, you could say build _ _ _ _ _, and we would know what build you're referring to.

In case you're talking about having upgraded to build 14251, it has a known issue that if you try the "reset this pc",  your pc will become unstable, and you will need to reinstall Windows. 

Also, when someone says "I had a grey screen and nothing else", we don't get much of an idea of the length of time that the condition occurred, so it's difficult to be of help. We can explain at length that a new installation will show a blank screen for a while before the rolling BBs show for a wait period during a process, only to have the user reply, "yes, but I left it on all night, and it didn't change".

So, please, try to quantify such conditions in terms of approximate time lapse, to help us give more meaningful replies as we try to help you.

Now, in answer to your question, "is there another way to roll back?", yes, there is, but it requires preparing for failure in advance, and some are not willing to bother with it, preferring to just hope for no failures.

In our testing lab, a disc image of the OS partition of every computer is made as soon as each new build is installed and cleaned up. It's saved on an external drive, in addition to the last build before it, so that we always have convenient capability to restore either of them in a few minutes, in case the next build doesn't turn out well, or in case of hard drive failure. We don't want to rely on a bad build to do the restore, so we use a disc imaging program that must run in the OS to make the image, but also makes a bootable CD with the program on it, to perform the restore of the OS from the external hard drive.

We keep new hard drives on hand, so, a hard failure means a few minutes to replace the drive, and a few minutes to restore the previous build on it.
