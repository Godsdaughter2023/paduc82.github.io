    john - project say:
    DVM_CC,
    Wow, it just keeps on getting better. My previous import testing was done using Excel 2010 with Project 2010 and Excel 2007 with Project 2010. As I indicated, the % complete will only import if I set Project to manually scheduled. It doesn't matter is it is % complete or % work complete.

    This morning I tried Excel 2003 with Project 2007. No problem importing % complete. This configuration also seems to handle dependencies better, if they are properly structured in Excel. I do not have a configuration with Excel 2003 and Project 2010, which apparently is what you are using. However I suspect the failure to import % complete is a combination of data format in Excel and compatibility with Project 2010. I just haven't found the magic "key" yet.

    After a whole lot of testing with different configurations this is what I suggest (I tried this procedure with both Excel 2003 with Project 2007 and Excel 2010 with Project 2010
    1. In Excel set the data type for the % complete values to Text and the predecessor values to Text
    2. Configure the % complete values as whole numbers (i.e. 100=100%, 50=50%, etc.)
    3. For each task in Excel that has a predecessor, set its start date to be the same as the task with the earliest start date. For example, let's say you have a start milestone for your project and it's date is 3/1/12. Then set the start date of all tasks that have at least one predecessor to be 3/1/12 (This may sound counterintuitive but it will facilitate the non-schedule Excel data when it's imported into a schedule based Project application)
    4. Open a new blank Project file and set the Project start date to the earliest date. In my example, that would be 3/1/12
    5. Import using the Append option
    6. In the import map, dump the % complete data into Text1 and the predecessor data into Text2. Do NOT import those fields to their normal Project fields
    7. Run the import
    8. In Project first copy the Text2 data into the Predecessor field. This will configure the basic schedule and preserve existing durations and align start dates
    9. Then copy the Text1 data into the % Complete field

    That should do it.
