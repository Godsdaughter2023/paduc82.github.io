Automatic Maintenance combines all the background processes, including the third part software and apps to include themselves in the process.

It’s not feasible to add users to a Microsoft Teams by using a CSV file. However, you can add the users to a Distribution Group and type the group name in the Microsoft Teams Add member’s interface. If you are interested in adding users in Microsoft Teams using a CSV file, please share your idea with us via this link:

Import-Csv “D:\list.csv” | foreach{Add-DistributionGroupMember -Identity “GroupName” -Member $_.alias}

Please replace “GroupName” with your actual distribution group email address.

jay freedman say:
You're currently adding the signature as an InlineShape (using Selection.InlineShapes.AddPicture). Inline pictures cannot be behind text; they're in the text layer, and that's all they can do.

Only a Shape object (floating in the drawing layer) can be sent behind text or in front of text.

You can either insert the signature as a Shape to begin with (using ActiveDocument.Shapes.AddPicture) or convert the InlineShape object to a Shape object that you can then manipulate. Here's some very simplified code for that conversion:

Sub x()
    Dim ils As InlineShape
    Dim shp As Shape
    Set ils = ActiveDocument.InlineShapes(1)
    Set shp = ils.ConvertToShape
    With shp
        .WrapFormat.Type = wdWrapBehind
        .RelativeVerticalPosition = wdRelativeVerticalPositionMargin
        .RelativeHorizontalPosition = wdRelativeHorizontalPositionMargin
        .Left = InchesToPoints(1)
        .Top = InchesToPoints(1)
    End With
End Sub

I can't be more explicit than that without knowing the position on the page where the top left corner of the signature's bounding box should be.

Also, are there other InlineShapes in the document besides the signature? If so, your code would resize all of them; if not, then why are you using a For Each loop?

jay freedman say:
Inserting a Shape is similar to inserting an InlineShape, but there are a couple of additional things to do. The position of an InlineShape is determined entirely by the location within the text where it's inserted -- it behaves like a single character. A Shape's position is determined by three pieces of information:

1. Its "anchor", the range in the text to which the Shape is attached. Every Shape must be attached to some piece of text, which must always be on the same page as the Shape. If later editing moves the anchor to a different page, the Shape will also move to that page.

2 and 3. The Shape's .Left and .Top properties, which measure the distance (in points, where 1 inch = 72 points) from the top left corner of the anchor to the top left corner of the Shape.

So code for inserting, positioning, and sizing a Shape could look like this:

Sub y()
    Dim shp As Shape
    Set shp = ActiveDocument.Shapes.AddPicture( _
        FileName:="G:\Templates\E-Signatures\Sig.jpg", _
        SaveWithDocument:=True, _
        Anchor:=ActiveDocument.Paragraphs(1).Range)
    With shp
        .Left = InchesToPoints(3)
        .Top = InchesToPoints(5.5)
        .Width = InchesToPoints(2.5)
        .Height = InchesToPoints(0.9)
    End With
End Sub

If you prefer, the position and size properties (or any subset of them) can be included in the AddPicture command, although I find it makes the code a bit harder to read. This is almost equivalent to the above*:

Sub y()
    Dim shp As Shape
    Set shp = ActiveDocument.Shapes.AddPicture( _
        FileName:="C:\Users\Jay\Pictures\congarats.bmp", _
        SaveWithDocument:=True, _
        Anchor:=ActiveDocument.Paragraphs(1).Range, _
        Left:=InchesToPoints(3), _
        Top:=InchesToPoints(5.5), _
        Width:=InchesToPoints(2.5), _
        Height:=InchesToPoints(0.9))
End Sub

*The undocumented difference is that in the first version, the setting of the .Height and .Width properties obeys the Shape's .LockAspectRatio property, which defaults to True; while in the second version, the specified Height and Width measurements are used without regard for the original aspect ratio of the file being inserted.

hansv mvp say:

You can do this as follows. I'll use DateField as name for the merge field; replace this with the actual name of the merge field.

 

Press Ctrl+F9 to insert field brackets { } (do not type them yourself - that won't work).

Type the following between the brackets:

 

MERGEFIELD DateField \@ d \* Ordinal

 

Press F9 to hide the field code.

Place the insertion point after the merge field. Type a space, then the word of followed by another space.

 

Press Ctrl+F9 to insert a second pair of field brackets { }.

Type the following between the brackets:

 

MERGEFIELD DateField \@ "MMMM yyyy"

 

Press F9 to hide this field code too.
