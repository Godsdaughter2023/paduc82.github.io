melchizedek qui say:

Hello Gregory,

ReadyBoost uses external USB flash drives as a hard disk cache to improve disk read performance. Supported external storage types include USB thumb drives, SD cards, and CF cards. Since ReadyBoost will not provide a performance gain when the primary disk is an SSD, the system disables ReadyBoost when reading from an SSD drive.

ReadyBoost provides the most significant performance improvement under the following circumstances:

  • The computer has a slow hard disk drive. Computers with a primary hard disk Windows Experience Index (WEI) subscore lower than 4.0 will see the most significant improvements.
  • The flash storage provides fast, random, non-sequential reads. Sequential read speed is less important.
  • The flash storage is connected by a fast bus. Typically, USB memory card readers are not sufficiently fast. However, connecting flash memory to an internal memory card reader might provide sufficient performance.

In summary, ReadbyBoost may still be able to improve your device's performance that has a small amount of RAM and using a fast USB drive. It is also somehow subjective as it will depend on what kind of applications you're going to run/use on your computer. Adding additional RAM will surely improved performance much more than using ReadyBoost.

You can check this article to learn more on how to use ReadyBoots with an SD card or a USB flash drive. The steps also applies to Windows 10.

Windows 10 Installation Guidelines - Microsoft Community

theimmersion say:

Windows 10 Installation Guidelines

First off, this is not in any way official and thus many things can go wrong.
I will not be responsible if any damage (software or hardware) arise. USE AT YOUR OWN RISK.
Before we begin, it is recommended to read the entire article before doing anything.
Or at least, read and consolidate the entire step number and by those recommended sub sections completely before any actions.

===============================================
WINDOWS CLEAN INSTALL CHECK LIST AND GUIDELINES
===============================================
After installing the OS:

1. LAST CHECK AND SETUP OF OS INSTALL AND PERSONAL DATA STRUCTURES
Install any OS updates. Since Windows 10 updates are rather finicky, you can continue the steps and let it update when it requests. Microsoft™ ¯\_(ツ)_/¯
Move user folders and other data to its original state. (if changed, see RECOMMENDATIONS - USER DATA)
This means that before installing or launching any apps it is recommended to move the user folders -
especially those that are frequently used by apps like "Documents" and "Roaming" to their respective place.
Install preferred browser for latest drivers and other apps that either use -
online installers or need to be downloaded from the internet.

2. Install Drivers
Graphics Driver - It is recommended to install those drivers first for easier work.
Audio Driver - It is recommended to install those right after so you have audiable signals as well as visual ones of what you are doing.
Install any other drivers.
NOTE: I havent found any issues so far with installing all the drivers without restarting the PC.
I restart my system after all the drivers are installed that might request a restart.

3. Install OS Customization Tools And Apply
RocketDock, Rainmeter etc.

4. Install Applications and Programs
Discord, Photoshop, Blender, UE4, Games etc.

5. Install And/Or Setup Core Addons
Codecs, Plugins, Format Extensions etc.

6. DEFRAGMENT DRIVE
NOTE:
Most often than not, after a complete reinstall of the OS and all Apps that one might use, a drive defrag is not needed.

OPTIONAL:
Defragment any drives that are used in the setup process.
Meaning, if only one drive like "C" was being used for all the setups -
(OS, Apps and Programs) than only that drive needs defragmentation.
If "C" drive is used for OS and maybe some other core programs and -
"D" drive was used for other applications and programs, both need defragmentation.
Its not always absolutely necessary but its a good practice and a guarantee -
that fragmentation is not going to be an issue from that point on.
Regular defragmentations are very much still recommended. Based on the amount of data being added and removed,
one to two months defrag schedule is a good practice.

PORTABLE APPS
I recommend to store any portable apps in a separate folder from "Program Files".
For instance, i store all my portable apps in a folder "Programs" that are besides "Program Files" and "Program Files (x86)".
That way i know which ones are which at any given moment.

If you have or want to setup Notepad++ as a portable variant,

keep in mind that things like Edit with Notepad++ right-click context menu will be missing.
This requires Registry editing either manually or via a .reg file.
For manual registry editing there is enough tutorials out there to help you out.
I recommend .reg files for easy access.
Create both the reg file that adds and the one that removes the entries as well or you can create all three.

To add notepad++ to context menu, create a new text document with any text editor and copy/paste and edit the following:
(Edit with Notepad++ entry with icon. Default behavior if it were to be installed.)

Windows Registry Editor Version 5.00
[HKEY_CLASSES_ROOT\*\shell\Edit with Notepad++]
"Icon"="D:\\Programs\\Notepad++\\notepad++.exe"
BlankLine
[HKEY_CLASSES_ROOT\*\shell\Edit with Notepad++\command]
@="D:\\Programs\\Notepad++\\notepad++.exe %1"
(Edit with Notepad++ entry without icon. In case you dont want the icon.)

Windows Registry Editor Version 5.00
[HKEY_CLASSES_ROOT\*\shell\Edit with Notepad++\command]
@="D:\\Programs\\Notepad++\\notepad++.exe %1"

Note the underlined address, edit that address to point to the execute file. (where your notepad++.exe is currently located)

Yours might be D:\\Portable\\Notepad++\\notepad++.exe
Change accordingly.
TIP: Hold left shift and right click any files or folders, like notepad++.exe and in the context menu select Copy as path
and simply replace the current address. Note it might copy with punctuation's, remove any duplicates.

Save the file by selecting Save as type: to All Files

Name the file what ever you wish, i suggest something descriptive and add .reg in the name.
Example: Edit with Notepad++ Context Menu ADD with icon.reg for the entry with icon.
And/or: Edit with Notepad++ Context Menu ADD.reg for the entry without icon.

This will associate the file as a .reg file which we want. Now all you need to do to apply it is to double-click the file or right-click merge in the context menu.

To remove notepad++ from context menu, create a new text document with any text editor and copy/paste the following:

Windows Registry Editor Version 5.00
[-HKEY_CLASSES_ROOT\*\shell\Edit with Notepad++]

No editing needed for this just save the file by selecting Save as type: to All Files

Name the file what ever you wish, i suggest something descriptive but add DEFAULT in the name as well

so you know which one to use to revert to original settings and add .reg in the name.
Example: Edit with Notepad++ Context Menu REMOVE (Default).reg
This will associate the file as a .reg file which we want. Now all you need to do to apply it is to double-click the file or right-click merge in the context menu.

DOCKS
Using a portable version of RocketDock or any dock where the most important and frequently used apps and programs shortcuts are placed
is a good way to have a reference of which apps are installed and which are still missing.
An example of my usual PC environment is that i have all my apps pinned on RocketDock and when ever i do a OS clean install -
i simply run RocketDock and i see which apps are still missing (usualy indicated with a "?" mark).
Even if i dont plan on installing an app right away, i still keep the shortcut on the dock for future reference.
The only guidline here is to install apps on the same location when the shortcuts are being placed on the dock.
I usually always install all apps the same way they would be by their defaults but on a different drive.
For instance, if an apps default installation location is "C:\Program Files (x86)\Some App\App" -
i would simply set it to the drive letter i normally use for my apps which would be "D:\Program Files (x86)\Some App\App".
Notice that everything is the same except the drive letter.
Now if i ever wanted to install an app on the system drive or a different drive altogether,
all i had to do is set any shortcuts on the dock to the right drive letter and thats it.

Some apps like Rainmeter wont have the option for start when windows starts and the way to do this is by simply adding a shortcut to the startup folder.

Easiest way is to either locate the folder, by using Windows Key & R (run command) and entering Shell:startup

or making a batch file to keep in the Rainmeters folder (which i use and recommend).To make a batch file that opens the user startup folder, simply make a new text document with any text editor.
Copy/paste the following inside:
start Shell:startup

Save the document as All Files in the Save as type: and name it something descriptive like Open User Startup Folder.bat
Note to add .bat at the end of the name so the file gets associated as a batch file and your done.
Double click and it should open the right startup folder.

You can keep that file anywhere you want or keep it with Rainmeter for easy access.

How do I avoid using a password to access Windows 8.1 at startup

Just updated Windows 8 to Windows 8.1,But now to access my operating system it asks for a password,How can I avoid using a password or can I?
cmdrkeene say:

If your PC account is connected to a Microsoft Account online, you cannot remove the password requirement.  This protects your Microsoft Account, which has confidential and private information (such as your payment details).


You still have options though:

  1. Disconnect.
    You could disconnect your Microsoft Account from your PC Account (which means your PC login is a "local account").  However this will lose many benefits like settings backup and sync, plus block your ability to use apps, games, skydrive, and even the Mail app.

  2. Change the type of password.
    You can change the type of password used so that it's a simple 4-digit PIN number (like cell phones) or even a picture password (draw on a picture to unlock your computer).

  3. Use the sleep option.
    Sleep mode is very efficient, and you can tell Windows not to ask for a password when resuming from sleep.  Then the only time you need to type a password is during full system reboots, such as after updates are installed.  This happens monthly, so now instead of typing your password every day, you only have to type it once a month.
