Saving Excel 2007 file in ms-dos.csv format - Microsoft Community

maryphill say:

Saving Excel 2007 file in ms-dos.csv format

I need to save an Excel 2007 file in ms-dos.csv format to allow me to upload it to a website.  I need to save an Excel 2007 file in ms-dos.csv format to allow me to upload it to a website.  Each time I select this format, the file appears to be saved as such, but when I view it again after saving, it has been saved as comma-delimtied.csv.  This causes the file to reject at the website.  Has there been an service-pack update that caused the file format to revert to the comma-delimited.csv format?  Am I doing something wrong.  I have been uploading to this sight for a few years now and haven't had this problem before.
joeu2004 say: Mary wrote:
I need to save an Excel 2007 file in ms-dos.csv format to allow me to upload it to a website.  Each time I select this format, the file appears to be saved as such, but when I view it again after saving, it has been saved as comma-delimtied.csv.  This causes the file to reject at the website.  Has there been an service-pack update that caused the file format to revert to the comma-delimited.csv format?  Am I doing something wrong.

I don't think so; at least not what you think.

There is no external difference between file saved as a normal csv file and a file saved as an ms-dos csv file.  Both have the ".csv" extension.  And looking at the binary contents of the files, there is no internal difference that would allow software to distinguish one from another.

The only difference is how characters are "encoded" (written in binary) [1].  But that is merely an interpretation of the binary values 0 through 255, which is what software that reads the file sees.

I suspect there is something in the file contents that the website does not like.  And it is simply misinterpreting that to mean that the file was saved with the wrong CSV type.  In other words, the error message is misleading.

So look at the content of your original worksheet.

If you are certain you do not use any special characters (e.g. non-English characters), I suspect that the data contains non-breaking spaces (HTML &nbsp; ASCII 160).  This is quite common if you copy-and-pasted data from a webpage.

There are many ways to look for and correct the occurrences of NBSPs as well as other ASCII values that do not occur in normal natural languages.  But in order to advise you, we need more information about the form of the content of the Excel file.

Would you be amenable to uploading the Excel file to a file-sharing website?
