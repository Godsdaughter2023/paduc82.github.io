kosh vorlon say:

Hi Fred,

 

Here's a couple of things about scheduled scans. To run, the PC must be on and connected to the internet (it cannot be shutdown, in sleep mode, in hybrid sleep mode, or in hibernation). The power saving settings must be such that the PC does not shut down or go asleep during the scan (or it will be as if it never happened). Scans do not begin precisely when scheduled - some can start an hour or so earlier or later than the officially set time (depending on Microsoft's servers and the loads at that time).

 

Also, go to settings and check scheduled scans and uncheck the box that prevents them from starting unless the system is on and idle (if you're using the system when it's scheduled or it otherwise isn't idle - perhaps downloading Windows Updates or doing something else - then the scan will not start and will be missed). If a scan is missed it does not start at the next available time, but waits for the next scheduled scan.

 

Go to Start / Control Panel / Administrative Tools / Services / scroll to Task Scheduler and make sure it is started and automatic. If not set it to automatic and restart it. If it won't start, click on the dependencies tab and do the same thing for each of the ones listed. If any of those won't start, continue with it's dependencies and do the same thing. When you get to the end, reverse course and Start the ones you couldn't because of the dependencies as you go until you get to Task Scheduler and hopefully it will now go to automatic and start. If you changed nothing read on. If you changed anything, reboot and see if that helps.

It is common for a computer to have all three of these protocols installed at the same time. Installing one or more different protocols does not cause issues with a computer. Open Control panel.

  1. Double-clicking "Network and Dial-up connections."
  2. Double-click on the "Local Area Connection."
  3. Click the Properties button.
  4. If the protocol is installed, the protocol should be listed by the installed network adapter. Note: do not confuse the network adapter with the dial-up adapter.   5. If the protocol is not listed, click the Add or Install button.
  6. Select the protocol and click ok. If this is a corporate network and you are unfamiliar with the network you are connecting to, attempt first to install the protocol.   7. Once the protocol has completed the installation, unless you want to install additional protocols, reboot the computer.
