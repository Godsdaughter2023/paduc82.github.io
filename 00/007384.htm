Office 365 PlusPro: Cannot create custom IMAP account - Solved - Microsoft Community

zucker gerhard say:

Office 365 PlusPro: Cannot create custom IMAP account - Solved

I solved this one on the way of writing the question, still I thought it is worth sharing.

Before the upgrade to Office 365 I could just add my private IMAP account, but this was no longer possible with Office 365. Here is why:

I assume that the reason is that my IMAP account "*** Email address is removed for privacy ***" needs a username, which is "john" and not "*** Email address is removed for privacy ***"

The standard screen (see whole procedure below) only asks for a password, but not for a username.

The solution is as follows:

To add a custom IMAP acocunt,  go to File/Accounts and select "New"
Select Extended Options and "I want manually configure my account". 
Enter email address, say "*** Email address is removed for privacy ***". Select "Connect". 
After some time select the account type as "IMAP".
Enter IMAP server e.g. "mail.myemail.com" and SMTP server e.g. "mail.myemail.com" 
The next screen asks for the password
DO NOT ENTER THE PASSWORD, as it will then not ask your for the user name. Instead, leave password empty and press Enter.
Then a new dialog appears that asks for username and password.

a) Subnet Mask

b) Default Gateway

c) DNS Servers

Select your WLAN access point and go to Advanced settings:

IPv4 settings:

For phone IP address: Start with the Default Gateway IP and change the last number to something else.
For Subnet mask and Default gateway use the same as on PC
For DNS address: Change to user defined and enter same settings as on PC

3. Check for firmware updates for your router and install if there is one.

Workarounds beyond recapturing the audio/audio network stream are:
