grandpajoejoe say:

For people reading this thread I offer the following information.  I fixed the problem by noticing that the problem may have been caused when the system image was being created.  So I did a backup without creating the system image and the backup completed without error. 

 

Then I used the backup program to just create a system image.  This time error code (0x8007045D) was displayed.   The error report said "The operation failed due to a device error encountered with either the source or the destination. If the source or destination volume is on a disk, run CHKDSK /R on the source or destination volume, and then retry the opertion."

 

So I ran chkdsk /r and this program found a bad sector on my computer hard drive and fixed the error.  It took about an hour to run the program.  Now the system image and the data file backup work again.

 

So I believe the root cause of error (0x8078012D)  was I/O dervie error (0x8007045D).  So if you get the first error code you may want to look up fixes for the second error code to fix the root cause.

The error indicates that the shared resource might have been turned off, or the permissions might have changed. To troubleshoot this issue, we suggest that you perform the following:

  • Turn off any firewall software. If the firewall is the problem, you'll have to configure it to allow access to trusted zone addresses.
  • Try to PING your NAS drive.
  • Check if the following Services are started on your PC: Workstation, DHCP Client, DNS Client, Server, TCP/IP NetBIOS Helper, and Computer Browser.
  • All computers and devices should be in the same workgroup. File & Print Sharing has to be enabled on any computer.
  • Make sure that NetBIOS over TCP/IP is enabled in the network adapter properties.
