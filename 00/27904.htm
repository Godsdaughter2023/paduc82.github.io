Set the right size of the detail section of an Access Report, using image__

In an image control, I'm showing an image located in a folder. I use VBA code that reads the width and height of the image and rezises the image control (executed in detail_format event). However the reports shows a large empty area under each image (please see an example in www.solfin.net/report.jpg ).

I appreciate all your suggestions. Thanks in advance.

marshall barton say:

Since the image or the text may be the determining factor in the needed height, I think that can best be done using a text box with the expression =Null.  Make the text box and the detail section as tall as the detail image and text can possibly get.  The text box can be very narrow so it doesn't get in the way or overlap any of the other controls.  Then set both the text box's and the detail section's CanShrink properties to Yes.  The text box's expression will assure that it always shrinks bringing the bottom of the detail section up to the bottom of the other stuff in the detail section.

luis medina r say:

Thanks Marshal. Very interesting advice that I have to try. But a new issue appears: some items in the detail comes without any related image. So it's difficult to set a fixed height of the detail area. How can I solve this problem?

Luis

marshall barton say:

The original design height can be really big (max 22 inches).  It's not a fixed size because CanShrink will reduce it to the minimum needed for the actual stuff in the detail section.  The trick is that you need a shrinking text box that extends to the bottom of the detail's design because empty white space won't shrink.  Position the top of the tall skinny text box slightly below the item description text box. 

Because the height of the image is unknown at design time and I suspect that the description text next to the image is a CanGrow text box, there is no reasonable way to calculate the final height of the detail section in the Format event.  Besides, the calculated image height must be less than the detail design height so the shrinking text box trick is needed to reclaim the unused space when an image needs less than the maximum image height.
