No, there's no system setting that does this. That's why I recommended using DoEvents.

MacOS/MacXL and Windows/WinXL differ on how they use interrupts to update windows. There are no settings.

AFAIK, there's no specification in XL/VBA for the frequency of host application window updates, so the way WinXL works is an artifact, and not guaranteed to work in future Win versions (though there's no particular reason to think it will change). However, if you want to be sure your window updates, use DoEvents.

Most issues with display are actually go the other way - people want their macros to run faster by not interrupting to update the screen. In WinXL you'd use 

     Application.ScreenUpdating = False

to accomplish this (which doesn't affect MacXL currently).

ken sheridan say: I sounds like you are trying to insert a row into an OrderDetails table or similar, rather than an Orders table.  The basic model for an ordering database is generally like this:

Customers---<Orders---<OrderDetails>---Products

The OrderDetails table models the many-to-many relationship type between Orders and Products.  Data entry into the table is usually via a subform within an orders form, linked on OrderID.  Consequently, when inserting a row into OrderDetails via the subform, a record must have been first entered into Orders via the parent form.  The linking mechanism then automatically inserts the OrderID value from the parent form into the subform.
