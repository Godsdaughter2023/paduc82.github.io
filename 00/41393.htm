tryx3 say:

This thread is about Windows 10 logging off not MSN.  The considerations involved are not the same.

I suggest that you search the Outlook.com section of this forum for guidance [it includes issues concerning all of Microsoft's online service provision such as MS Accounts & MSN].  You'll probably also find some relevant threads in the Windows 10 Security, privacy & accounts section.

My only observations are so general that they are probably not useful. 

The intended use of user accounts within Windows 10 is that each individual using the computer has a separate user account on that computer. 

If the user choses to link that user account to an MS Account as you have done then it would not cause any problems as all other users of the computer would sign on to their own user accounts on the computer and, if relevant, to their own MS Accounts. 

You can log off from one user account on the computer and then log on to another user account on the computer quite quickly. 

If you cannot use this separation of accounts on your machine then you are likely to get recurrent privacy problems.  All I can suggest is that you:

  • Change your account on the computer to a "local user account" instead of an "MS Account" [this is done within Settings and you should be able to find a procedure for this in Windows Help if you are not sure about it].

- Which brand and model graphics processor ? Or more to the point,  does ACER provide a Win10 driver for your machines

graphics processor ( I will guess - No! ) ?  If not, I don't find it surprising you would have issues with games on Win10 .

It's not so much that old drivers are not compatible with newer versions of OpenGL or DX11/12. These newer versions of OpenGL and DX

don't affect games that use older versions.

The older versions would still be in Windows, and a newer drivers will still be able to use them because the games that use them can

use only the old version.

It's 's more likely a case that drivers which are not developed for WIn10 are not fully compatible with Win10.

Nvidia, AMD and Intel all have disclaimers regarding possible loss of stability and performance if using Win10 with their older products

that they have not released a Win10 drivers for.

The way I see it is, data pipelines have to flow smoothly for optimal performance. Use an outdated instruction set (driver)

and the flow may be generally slower, or it may hit snags. This could reduce performance at best, or crash the game at worst.

But I may be wrong... Never had a laptop, and my oldest integrated graphics is Intel HD4000, for which the mfg of my motherboard

supplied Win10 drivers.

- The Web Installer is automatic. It either installs any missing files, or pops up a message to the effect your Windows has them all.

- If using the DX Redist (June 2010) installer package it will first unpack the files into a folder location you choose (desktop is good)

You then need to open this folder and run the DXSETUP.exe to complete the reinstall of DX (delete the folder on the desktop after).

- The following is a list of the Direct X .dll's you will have when the files are up to date (OpenGL should be in there too).
Go to the Windows / System 32 folder, (and SysWOW64 if you have 64bit).  
They are in alphabetical order and will start with d3dx9 - 24 > 43 . Then d3dx10 - 33 > 43 & finally d3dx11 - 42 > 43.
There should also be - d3d9, d3d10 and d3d11 (among others).

These come before the D3Dcompilers-33>43+47.
- Win10 currently has only two d3d_12 files. One in System 32,and SysWOW64 .
- There are more DX files, but these are most of the graphics related Direct X  .dll's
