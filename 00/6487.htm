Each of those fields can be counted using a query calculated field like:
   Count(IIf(thefield, 1, Null))
or
   Sum(IIf(thefield, 1, 0))
or
   -Sum(thefield)

choose the one easiest for you to understand.

This is probably not the time to tell you that your table is a spreadsheet, not a properly normalized relational database structure, but keep this side comment in mind when you run into other difficulties with that table/spreadsheet way of doing things.

Features and updates:

Phones supplied by network operators are network-branded. This means that they alter the features of the phone to suit their marketing needs and basically turn it into a semi-functional device used to advertise to you. Features that they don't want you using are removed, their logo is plastered all over the place and links to their services are stuffed into the menus.

In some cases, they add a few features that are required to take full advantage of their subscriber services.

Given that the firmware in these phones is something based on Nokia's original firmware but subsequently modified by the network operator, any updates that the operator is planning on releasing will become available to your branded phone after they've been made available to generic, SIM-free phones. You will have to wait for your network operator to modify the firmware and release it, and this can often take a very long time, if indeed the operator bothers doing it at all. You may well be left with buggy firmware while users of SIM-free phones have already updated theirs and thus eliminated the bugs. Not to mention bugs introduced by the networks themselves during the branding process.

When updating a network-branded phone, you are not "debranding" the phone, ie. turning it into a generic, SIM-free phone. You are installing a newer version that has been branded by the network operator, hence the wait for the operator-approved version

The issue might be caused by the size of the file that you are trying to sync on OneDrive or how you are trying to sync it. You can try the following steps below to help resolve the issue when Notebook cannot be sync.

If the notebook that won’t sync is stored on OneDrive or on newer versions of SharePoint, try opening the notebook in OneNote Online (OneNote.com). This can help determine if the problem is caused by the server or by OneNote.

  1. In OneNote, click File > Info.

  2. Right-click the URL shown below the name of the affected notebook, and click Copy.

  3. In your web browser, click inside the address box, and press Ctrl+V to paste the copied URL.

  4. Press Enter to load the notebook in OneNote Online.

If you're able to successfully open the notebook in OneNote Online and the changes appear, the problem may be caused by the desktop version of OneNote. Closing and re-opening the notebook usually clears and successfully resets the connection.

  1. In OneNote, click File > Info.

  2. Next to the name of the affected notebook, click Settings, and then click Close.

  3. Switch back to the notebook in OneNote Online in your browser, and then click Open in OneNote on the ribbon.

Important: Depending on the notebook size and type of Internet connection you're using, it may take a while to sync all the notebook content over to the desktop version of OneNote.

If you're unable to open and view the notebook in OneNote Online, the source of the sync problem is likely with your server. If your notebook is stored on OneDrive, check the Microsoft Services status page for any known interruptions or outages. If your notebook is stored on SharePoint or Office 365, you’ll need to contact your administrator to report and troubleshoot the problem.

If you're able to open and view the notebook in OneNote Online but can't see the changes, the information hasn't synced back to OneNote Online. Check the sync status dialog for specific sync errors.

Warning: Don't sign in or sign out or remove and reinstall OneNote as this could result in data loss.

  1. In OneNote, click File > Info.

  2. Click the View Sync Status button.

  3. Check for errors in the dialog box, note the error code and message and check the list of error codes in the article.

Note: You may see a Get Help button next to an error. If you do, click it as it should open a troubleshooting article specific to the error.

ronrosenfeld say:

Here is a method using the FileSystemObject in VBA.

You must set a reference (see Tools/References in the top menu bar of the VB Editor) to Microsoft Scripting Runtime, for this to work as written.

The macro will first ask you for the word or phrase you wish to find.  It will then open a dialog box where you can select a folder.  It will then go through every file in that folder, selecting only the text files (those whose full names end with .txt), read them line by line and check to see if the search text is present.

If the search text is present, it will save the relevant data in a Collection of User Defined Objects (cLines) which includes the folder path, file name, line number and line text.

Finally, it will output the data onto a worksheet, and also highlight the search text.

Try it and see if it serves your purposes

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/ Class Module and
paste the Class Module code below into the window that opens.

Then, you must rename this Class Module cLines.

To rename the Class Module, show the Properties window (hit F4).  The first line has the Name property.

Then, again from the top menu, select Insert/Module and paste the Regular Module code into that window.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.o dialog box. Select the macro by name, and <RUN>.

Class Module

Option Explicit
Private pLineNum As Long
Private pLineText As String
Private pFolderPath As String
Private pFileName As String

    LineNum = pLineNum
End Property
Public Property Let LineNum(Value As Long)
    pLineNum = Value
End Property

    LineText = pLineText
End Property
Public Property Let LineText(Value As String)
    pLineText = Value
End Property

    FolderPath = pFolderPath
End Property
Public Property Let FolderPath(Value As String)
    pFolderPath = Value
End Property

    FileName = pFileName
End Property
Public Property Let FileName(Value As String)
    pFileName = Value
End Property

Regular Module

Option Explicit
'NOTE:  Set reference to Microsoft Scripting Runtime
Sub FindInFile()
    Dim sBaseFolder As String, sFindText As String
    Dim FD As FileDialog
    Dim FSO As FileSystemObject, FIs As Files, FI As File, FO As Folder
    Dim TS As TextStream
    Dim colL As Collection, cL As cLines
    Dim S As String
    Dim I As Long
    Dim R As Range
    Dim wsRes As Worksheet, rRes As Range, vRes() As Variant
   
'Set results worksheet and range
Set wsRes = Worksheets("Sheet1")
Set rRes = wsRes.Cells(1, 1)
   
sFindText = Application.InputBox( _
    prompt:="Enter Text to Find: ", _
    Title:="FindText", _
    Left:=0, Top:=0, _
    Type:=2)
   
'Select the folder
Set FD = Application.FileDialog(msoFileDialogFolderPicker)
With FD
    .AllowMultiSelect = False
    .Show
    sBaseFolder = .SelectedItems(1)
End With
Set FD = Nothing

Set FSO = New FileSystemObject
Set FO = FSO.GetFolder(sBaseFolder)
Set FIs = FO.Files

For Each FI In FIs
With FI
    If .Name Like "*.txt" Then
        I = 0
        Set TS = FSO.OpenTextFile(sBaseFolder & "\" & .Name, ForReading)
        Do Until TS.AtEndOfStream
            S = TS.ReadLine
            I = I + 1
            If InStr(1, S, sFindText, vbTextCompare) > 0 Then
                Set cL = New cLines
                With cL
                    .FileName = FI.Name
                    .FolderPath = FI.ParentFolder
                    .LineNum = I
                    .LineText = S
                End With
                colL.Add cL
            End If
        Loop
    End If
End With
Next FI

ReDim vRes(0 To colL.Count, 1 To 4)

vRes(0, 1) = "Folder"
vRes(0, 2) = "File"
vRes(0, 3) = "Line" & vbLf & "Number"
vRes(0, 4) = "Text"

With colL(I)
    vRes(I, 1) = .FolderPath
    vRes(I, 2) = .FileName
    vRes(I, 3) = .LineNum
    vRes(I, 4) = .LineText
End With
Next I

Application.ScreenUpdating = False
Set rRes = rRes.Resize(UBound(vRes, 1) + 1, UBound(vRes, 2))
With rRes
    .EntireColumn.Clear
    .Value = vRes
    With .Rows(1)
        .Font.Bold = True
        .HorizontalAlignment = xlCenter
    End With
    .EntireColumn.ColumnWidth = 45
    With .EntireRow
        .WrapText = True
        .VerticalAlignment = xlCenter
        .AutoFit
    End With
    .EntireColumn.AutoFit
   
    'Highlight the FindWord
    For Each R In rRes.Offset(1).Resize(rRes.Rows.Count - 1).Columns(4).Cells
        I = 1
        Do
            I = InStr(I, R.Text, sFindText, vbTextCompare)
            With R.Characters(I, Len(sFindText))
