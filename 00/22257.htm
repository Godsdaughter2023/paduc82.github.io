OK, I'll try to explain as best I can -- but Themes are still a 'work-in-progress' in Mac Office so there are some constraints that complicate things a little...

The component of a Theme that you're dealing with here is the Color Scheme. Each of the Themes has a set of colors [Color Scheme] associated with it. Those are the colors you see in the top row of the color palette in your last screen shot. If a different Theme or a different Color Scheme were in use, the colors in that row would not be the same ones. The shades in the columns directly below each of those 8 colors are variant shades of those same colors.

 If you make no direct changes to the colors used in the document the document colors will update to what is specified by the Color Scheme of the Theme you apply. That's why choosing Office, Capital, or any of the other Themes causes the colors applied to the cells to change. Each color & its variants are assigned to specific elements in a document, such as font colors used in Styles, Drawing Shapes, Chart components & the components of SmartArt Objects.

You may not have noticed, but a Theme also specifies a Font Scheme, so when you chose those other Themes the fonts used in the document changed as well -- again, provided you had not applied different fonts yourself.

The key point in the above paragraphs is what I have italicized. When you select a font that isn't a part of the current Font Scheme or pick a color that isn't a part of the current Color Scheme, it severs the reformatted content's connection to the Theme Font/Color. That explains what you describe in your EDIT about the color not continuing to change when selecting a different Theme.

This results in at least 3 confusing points with this particular template:
  1. The Theme used in the template (as well as its Color Scheme) is not a standard Mac Office Theme, &
  2. Even if it were, you cannot customize a Font Scheme or a Color Scheme in Word. You can only pick from the ones provided by the built-in Themes.
  3. Trying to effectively use Themes after breaking the connection is an exercise in futility.
I can bore you with this for another 20 paragraphs, but long story short IMO you shouldn't worry about Themes for this particular document.   1. Identify what shade is used in the cell above the one you need to recolor
  2. Identify the shade of the cell below the one you need to recolor
  3. Click in the cell to be changed
  4. Use the Shading button on the Tables tab or go into the Shading dialog
  5. Select a shade whose identity falls between the other two
[E.g., If one is Accent 5 & the other is Accent 4, pick Accent 1 Darker 40% for the one in between.]

Occasionally you'll get an error message telling you there's a problem with a website's security certificate. A site's certificate allows Internet Explorer to establish a secure connection with the site. Certificate errors occur when there's a problem with a certificate or a web server's use of the certificate. Internet Explorer helps keep your information more secure by warning about certificate errors.

What do certificate errors mean?

Error message

This website's security certificate has been revoked
What it means

You shouldn't trust this website. This often means that the security certificate was obtained or used fraudulently by the website.
Error message

This website's address doesn't match the address in the security certificate
What it means

A website is using a certificate that was issued to a different web address. This can occur if a company owns several websites and uses the same certificate for multiple websites.
Error message

This website's security certificate is out of date
What it means

The current date is either before or after the time period during which the certificate is valid. Websites must renew their certificates with a certification authority to stay current. Outdated certificates can be a security risk.
Error message

This website's security certificate isn't from a trusted source
What it means

The certificate has been issued by a certification authority that isn't recognized by Internet Explorer. Phishing sites often use fake certificates that trigger this error.

Error message

Internet Explorer has found a problem with this website's security certificate

What it means

Internet Explorer found a problem with a certificate that doesn't match any other errors. It could be because a certificate has been damaged, tampered with, written in an unknown format, or is unreadable. You shouldn't trust the identity of the site if a certificate has this error.

Is it ever safe to ignore a certificate error?

A websites's certificate provides identification of the web server. If the certificate has an error, it might indicate that your connection has been intercepted or that the web server is misrepresenting its identity. If you're absolutely positive of the website's identity, you know that your connection hasn't been compromised, and you understand the risks, you can choose to go to the website. However, we recommend that you don't ignore a certificate warning.

Can I go to a site even if I get a certificate error?

Yes. While this isn't recommended, you can click Continue to this website (not recommended) on the certificate error warning page to go to the website. If you ignore the warning page and go to a site that's presented a certificate containing an error, Internet Explorer will remember the certificate while you have your browser open. You can return to the site without receiving another warning for that certificate until Internet Explorer is restarted.

Can I turn off certificate errors?

No, you can't turn off certificate checking in Internet Explorer. If you receive certificate errors, it means the website you're visiting is having certificate problems and it doesn't indicate a problem with Internet Explorer.

You may want to try turning on/off add-ons and check the status.

dirk goldgar say:

This is not normal, and is a sign either of a corrupt database or a damaged installation of Access or Office.  For the latter issue, you can try repairing your installation of Access (or Office).  For the former issue, you have to get rid of the existing corruption and also examine your development practices to see if anything in them is contributing to the likelihood of corruption.

To get rid of the existing corruption, I suggest you create a new database and import everything from the old database into the new one.  You may also have to set some of the "Current Database" options in the new database to match those of the old.  Then, after importing, decompile your database according to the instructions here:

    http://datagnostics.com/dtips/decompile.html

Then compact the database, compile it, and compact it again.  That should get rid of most types of corruption.

Next look at your development practices for things that can contribute to corruption. 

  • If you are doing development work on your database over a network -- or worse yet, a WAN or internet connection -- stop doing that.  Work on a copy of the database on your local hard drive.
  • If you are in the habit of making code changes at run time, stop doing that.  For example, if you are testing the code and an error is raised, don't modify the code to fix the error while in debug mode.  Collect the information necessary to fix the code while you're in debug mode, but then stop the process, close all open objects that you don't intend to work on, put any form or report that you intend to work on into design view, and only then make the code changes.
