IramMCP wrote on 8/24/2010 at 5:59 p.m. CEST:

This works but people have to un-filter the form so that they can move around to different dates.

Right. Forget about that. We don't need it, because I think I have it now. Let's go back to where we started: Move "frm_FormDailyStats" back to the start-up, so that it is loaded when the second database is started manually. Also restore your original code:

Private Sub Form_Load()
    DoCmd.Maximize
    Me.Recordset.FindFirst "StatsDate=Date()"
End Sub

In the other database, we need not actually open the form via VBA. But I kept the code anyway as a comment. Important, however, is MoveFirst because after opening the form the first matching record has already been found. So we go back to first record and then execute FindFirst again. It works for me.

Private Sub Command453_Click()

      .OpenCurrentDatabase "\\Dcssresource\databases$\CMT\CMT_FE\Split_Databases\DailyStats_FE.mdb"
     '.DoCmd.OpenForm "frm_FormDailyStats", acNormal
      .Forms("frm_FormDailyStats").Recordset.MoveFirst
      .Forms("frm_FormDailyStats").Recordset.FindFirst "StatsDate=Date()"
      .Visible = True
   End With
End Sub

chip pearson say:

You might be thinking of Join and Split, but these are VBA functions, not Excel worksheet functions. You can use the concatenation operator & to combine the contents of two or more cells: =A1&B1

There is no worksheet equivalent to Split, but you could easily write such a function in VBA and call it from a worksheet cell.

Function ThisSplit(InString As String, _
        Delimiter As String) As String()
    ThisSplit = Split(InString, Delimiter)
End Function

Call it from a worksheet with:

=ThisSplit(A1," ")

This would be entered as an array into multiple cells, so you would press CTRL SHIFT ENTER rather than just ENTER. This will work if the range into which the function is entered is on one row spanning several columns. If you want the results in single column spanning several rows, use TRANSPOSE to rotate the array:

=TRANSPOSE(ThisSplit(A1," "))

wasim akram (v-wasak) say:

Hi

 

Thank you for raising this issue up to our concern.

 

In Yammer users get mapped to a Yammer network which has the same domain as per their email address domain.

 

For example: abc@xyz.com will mapped to xyz.com Yammer network.

 

So if your email address has the same domain as per your network domain , then I would suggest you use the email address which has the same domain on your desire network that you want to get connected to.

 

If it's an enterprise yammer network then it seems that you have multiple yammer network on your office365 tenant, and Microsoft does not recommend to have multiple yammer network on the same  tenant. So you will have to perform network migration.

 

Below are the example and process of network migration ….

 

  • Let's assume that you have two yammer network in one tenant.
  • Abc.com
  • Xyz.com

 

  • You your main yammer network is on abc.com domain. So you want all the users in office365 should login to abc.com yammer network.
  • However you notice that some users are not showing in abc.com yammer network, they are redirected to xyz.com yammer network.
  • So you want to perform merge and make it one yammer network in office365 tenant.
  • per the new update on the office365 messages center message ID: MC133153 , you will have  to perform network merge before October 16th of 2018 if you have multiple yammer network. Else Microsoft will do it automatically after  October 26th of 2018.
  • To perform merge you will have to login to abc.com with an verified admin rights > then you will have to go to network migration  page in yammer admin center > then you will see at the bottom xyz.com domain to merge > then click next.
  • You will get the information on the next page that xyz.com yammer network has let's say 10 users, 15 messages and 2 external network.
  • If you select merge then the data which is present on the xyz.com yammer network will be completely lost, only active users and the external network will be merge. And it is an irreversible    process.
  • If you think that the data on xyz.com is important and you wants to keep it.
  • Then login to this xyz.com yammer network with a verified admin rights >> then you will have to go to network migration  page in yammer admin center > then you will see at the bottom xyz.com domain to merge > then click next.
  • If you don't have  admin access to xyz.com yammer network.
  • Then create a test user in office365 and give the Global admin rights while creating the user, while creating the user you will get the option under license  to create the user without any license at the bottom.
  • Then login to office365 with this new user on an in-private browser.
  • And then go to office365 admin center and select yammer, you should be able to access the xyz.com yammer network with an admin rights.
  • Now let's assume if you have abc.com domain as the default domain in office365, after the network merge the yammer network URL and primary domain will also change to abc.com.
  • If you wants to change the URL and the primary  domain of your yammer network you will have to set that domain as the default domain in office365.
  • If you are not able to set the domain as the default domain in office365 because it’s a federated domain then you will have to send us a request to change the yammer URL and the primary domain
  • Also we recommend to enable enforce office365 identity in yammer. So that your yammer network will be fully committed by office365.
