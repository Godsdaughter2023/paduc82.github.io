julie fka profjulie say:

Unfortunately, there is no way to convert pocket word (and pocket excel) files to full verson word and excel files through Vista & Windows Mobile Device Center.  Furthermore, if you are running Office 2007 on your computer, Word 2007 and Excel 2007 cannot read these pocket version files.

 

You need to abandon the pocket versions altogether.  When you create Word or Excel files on your pocket pc, you'll need to select the SAVE DOCUMENT AS option and save the files as Word 97/2000 or Excel 97/2000 documents (tap the down arrow to the right of the Type box).  Your Pocket PC can read full version Word and Excel files so abandoning the pocket versions should not present a big problem for you.  The pocket versions are smaller files, but saving the files to a storage card makes the size a moot point.

 

If you are using Office 2007 on your Vista computer, the pocket pc will not be able to work with docx or xlsx files, so if you plan to work with these files on your pocket pc, you'll need to be sure to save the files as doc or xls files (Office 2003 files).

jw12thfan say:

Adding a new domain with 2 way trust to ADFS environment to allow SSO for O365

The business I work for recently acquired another company.  We are in the midst of trying to figure out how to get our existing ADFS environment in to support the new domain.   We have since setup a 2-way trust between the two domains.  From my research I have gathered that the following steps need to be taken.  Please let me know if there is anything missing or anything I may not need to do.  We also plan to use ADFS not just for O365, but also for Cisco Jabber and Cisco Webex which is already implemented on the existing domain.  So far this is just for O365 integration.  I figure it will be a little easier to get Jabber and Webex integrated once the O365 piece is figured out.

I am fairly green on ADFS and SSO integration.  Any input would be greatly appreciated.


DomainA.pri = Existing company

DomainB.com = Acquired company

  1. Verify support.microsoft.com/.../2681584 is installed on ADFS and ADFS Proxy server.
  2. On DC in DomainA.pri. Open AD Domains and Trusts.
  3. Open Properties and add UPN Suffix for DomainB.COM.
  4. Verify the UPN Suffix has been added by opening an existing User account in Active Directory. On the Account tab the logon domain dropdown should include the new UPN Suffix.
  5. Add the new domain to the tenant in O365 Portal. Login to Admin console and go to domains to add the new Domain.
  6. Login to CORPADFS and open ADFS Management Console. Go to Relying Party Trusts. Delete the Microsoft Office 355 Identity Platform trust.
  7. Open Microsoft Online services Module for Windows PowerShell.

Type:

Connect-MsolService

Update-MsolFederatedDomain –DomainName DomainA.pri –SupportMultipledomain

Convert-MsolDomainToFederated –DomainName DomainB.com –SupportMultipledomain

Update-MsolFederatedDomain –DomainName DomainA.pri –SupportMultipleDomain

8. Open Security Settings for ADFS Services AD account in DomainA.

9. Check the box for Allowed to Authenticate for Authenticated users.
