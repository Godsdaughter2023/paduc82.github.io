tobytu_msft say:

Hi Chris,

At what point would I need to consider using ADFS?

If you want to be able to access all of the applications and resources that you need to do business, by signing in only once using a single user account. Once signed in, you can access all of the applications you need without being required to authenticate (e.g. type a password) a second time.

In brief, the main function is for SSO (Single Sign-On) in Office 365. To achieve SSO, we need to deploy ADFS server.

Apart from Single Sign-on, what other advantages would I gain with ADFS?

Regarding this question, please refer to “Major Benefits” of this article.

What happens if my on-premise ADFS servers are not available?

Regarding “My understanding of ADFS is that all authentication requests are re-directed to the on premise ADFS server whereupon an authentication token is generated and used to sign on to cloud services.” Yeah, your understanding is correct.

ADFS works in Office 365 authentication flow. ADFS server is an essential component in SSO deployment, If the server is down, Office 365 can’t get any user’s credential from local AD. Therefore, it can’t authenticate any user. Then we can only convert our domain to be non-federated on our ADFS server. And the ADFS server can only be configured in on premise server instead of the Windows Azure.

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Target, Sheets("Master").Range("A4:G180")) Is Nothing Then
        Dim shts As Variant
        shts = Split("Jan;Feb;Mar;Apr;Jun;Jul;Aug;Sep;Oct;Nov'Dec", ";")
        Sheets("Data").Range(Target.Address).Value = Target.Value
        For sht = LBound(shts) To UBound(shts)
            Sheets(shts(sht)).Range(Target.Address).Offset(75, 0).Value = Target.Value
        Next sht
        MsgBox "Data updation in each month sheet complete. Click here to continue"
    End If
End Sub

Press ALT+Q to close the VBE and go back to Excel.
