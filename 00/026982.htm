Instead of expanding the output range, I suggest you select the appropriately sized output range before entering the function. Here's what works for me using Excel for Mac version 16.13.1:

grschinon say: This is a fairly classic problem with pop-port devices such as your 6021. Moisture and/or corrosion become lodged in the pop-port connector at the base of your phone and short out two pins that are normally shorted out by a headset. Therefore, your phone thinks that a headset is connected, hence the icon on your screen and the absence of sound coming from the built-in earpiece.

Take your handset to your nearest Nokia Care Point (you can find it here: http://www.nokia.si/najdi-prodajno-mesto) and ask them to clean the connector.

Sorry, this simply is not possible with Outlook for Mac.

First, Outlook for Mac has no VBA support so any VBA solution will not work.

Second, AppleScripts can be triggered either manually or using rules. However, an Outgoing rule will only work on a message after it is sent not before.

As for a custom header and footer, that is a function of integration with Microsoft Word for Windows. Outlook for Windows can use Word as its editor but Outlook for Mac cannot.

In this little demo file the option to 'Open form to go to a record' opens a bound form in which, in addition to the bound controls, there is an unbound combo box by which the user can navigate to an existing record in the table, or can select <New Contact> to navigate to an empty new record to insert a new contact.  This is done by using a UNION operation in the combo box's RowSource property:

SELECT ContactID, FirstName & " " & LastName,
1 As SortColumn, LastName,Firstname  
FROM Contacts  
UNION
SELECT
0, "<New Contact>", 0,"",""  
FROM Contacts
ORDER BY SortColumn, LastName, FirstName;

The following code in the combo box's AfterUpdate event procedure moves the form to the selected record:

    Const MESSAGETEXT = "No matching record"
    Dim ctrl As Control
    
    Set ctrl = Me.ActiveControl
    
    If Not IsNull(ctrl) Then
        If ctrl = 0 Then
            ' go to new record and move focus to FirstName control
            DoCmd.GoToRecord acForm, Me.Name, acNewRec
            Me.FirstName.SetFocus
        Else
            With Me.RecordsetClone
                .FindFirst "ContactID = " & ctrl
                If Not .NoMatch Then
                    ' go to record by synchronizing bookmarks
                    Me.Bookmark = .Bookmark
                Else
                    MsgBox MESSAGETEXT, vbInformation, "Warning"
                End If
            End With
        End If
    End If

The following code in the form's Current event procedure synchronizes the unbound combo box wit the current record if the user moves to a record other than by means of the combo box:

    ' synchronize go to contact combo box
    ' with current record
     Me.cboGotoContact = Me.ContactID

The following code in the form's AfterUpdate and AfterDelConfirm event procedures requeries the combo box to reflect the addition or deletion of a record, or any changes to the data ain a record:

     ' requery go to contact combo box
     ' to reflect changes to data
     Me.cboGotoContact.Requery
    ' synchronize go to contact combo box
    ' with current record
     Me.cboGotoContact = Me.ContactID
