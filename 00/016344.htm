john w. vinson say:

Scott's right in that such an address can be generated on the fly; but that doesn't allow for the likelihood that you might have two different people getting the same email (Jim Roberts and June Roberts = jroberts <at> domain.com).

 

I'd suggest putting some code in the Form used to populate the contact table, for example:

 

Private Sub Form_BeforeUpdate(Cancel as Integer)

Dim strEmail As String

<do any validation checks first, such as ensuring that FirstName and LastName both have values>

strEmail = Left(Me![FirstName],1) & Me![LastName]) & "@domain.com")

If IsNull(DLookUp("ContactID", "Contacts", "EMail = '" & strEmail & "'") Then

   Me!Email = strEmail

Else

   Cancel = True

   MsgBox("The email address " & strEmail & " is already in use, please change it", vbOKOnly)

   Me![Email].SetFocus

End If

End Sub
