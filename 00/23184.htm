  1. Download latest service pack.
  2. Extract msp files from service pack
  3. Copy msp files into installation source
  4. Install to fresh test machine
  5. Run Windows Update to look for new updates
  6. Step through list of updates one-by-one, searching the Update Catalog for each and every one
  7. Download all updates
  8. Extract msp files from each update one-by-one
  9. Copy msp files one-by-one into installation source (dealing with duplicate file names while I'm at it)
  10. Install again to fresh test machine
  11. Run Windows Update again to see if any additional updates are necessary
  12. Repeat if necessary

greg maxey say:

Automatically? No.  Formfields do not have a change event.  You can do it with a macro before A)saving or printing or B) do it with each individual field when you exit the field.

 

A macro for method A is provided below:
Sub ScratchMacro()
'A basic Word macro coded by Greg Maxey
Dim oFld As Word.FormField
For Each oFld In ActiveDocument.FormFields
  If IsNumeric(oFld.Result) Then
    If oFld.Result > 10# Then
      oFld.Range.Font.Bold = True
    Else
      oFld.Range.Font.Bold = False
    End If
  End If
Next oFld
End Sub

jeeped say:

I would imagine that the behavior is 'by design' and TBH I was a bit confused that the page with the continuous section break didn't adopt the new page number but I guess that is determined at the page break that created the page (at the top) whether it is a forced page break or an automatic one.

To my way of thinking, the behavior makes perfect sense to me. You created a new section with a continuous section break and it is at that point in the document that the page numbering must make the leap to the first number in the new sequence. Happily, the page number for that page (whether displayed in the header or footer) was determined at the top of the page so it remains in the old sequence. However, the leap to the new page numbering sequence takes place on that page so any subsequent pages would be numbered as that page +1 per subsequent page break.

FWIW, this behavior is in Word 2007 as well.

christhomas_180 say:

Thanks for taking the time to reply Jeeped.  

I understand this may make sense but it still doesn't seem right to me.  Surely if this logic applies then the page numbers on the subsequent page would increase based on the number of continuous section breaks (csb) on the previous page.  It only jumps by 1 regardless of how many csb's are applied to the previous page.

jeeped say:
It's jumping by 1 due to the automated page break, not any of the section breaks on the previous page. The section break just registers the point where the page numbering is reset. Add one page break (albeit automated) to that and if the section was reset to start at 7 then the next page would be page 8.
