Hi,

This is quite simple. If you want the precision of 1 hour, when entering the dates also enter the time of day (like 5/2/11 9:00) then Project will know the difference between actual Start and finish; when you just enter a start date, Project completes it with the default start time fram the Calendar options, and when you neter a finish date, Project completes it with the default finish time from the Calendar options. So it is you who tells Project the duration is the difference between default start and default end times. Project always, inevitably, absolutely, works with date/time in minuites, even if (as you will have defined in Option, View, Date Format) ask it to only show the day.

Or (which I prefer) enter Actual Start and actual duration (1 hr).

It is always bad (sometimes very bad) to enter BOTH a start and a finish date/time in Project, whether for planned or actual dates. You may expecty curious behaviour sometimes.

Why is it taking so long to receive the Windows Phone 7.8 update? - Microsoft Community

a. user say:

Why is it taking so long to receive the Windows Phone 7.8 update? (Updated 3/19/13)

We have been seeing a number of questions about where the update is from many of you who haven't received the 7.8 update yet. I wanted to take the opportunity to write up  post for all of you with an explanation of how the update process works.

 

 

a. user say:

The Windows Phone 7.8 update is being pushed out to devices gradually. There are a few reasons for doing the update process this way. One reason is so Microsoft can monitor the updates for any issues that need to be corrected before the update is pushed out to all devices. Another reason for staggering the update is to cut down on the user traffic trying to download the update to ensure that the update process is faster.

 

When updates are released from Microsoft they are first given to mobile operators and equipment manufacturers so that they may add device and operator specific applications and services to the update and test the update. The updates are then pushed out by the mobile operator as a staggered release. If you have not received the update yet then the update has not yet been pushed out to  your phone.

 

Some patience is needed while waiting for the updates. I haven't received any updates for the devices I use for testing either and I understand that waiting for the update can be a pretty frustrating process when you see that others have received it.

 

******Edit - 2/25/2013******

Please also keep in mind that it is up to your mobile operator to release the update. If they do not plan to release it it will not be available. You may want to check with your mobile operator about the availability of the 7.8 update.

 

******Edit 2 - 3/5/2013******

We have identified a minor issue in the Windows Phone 7.8 update and have temporarily paused delivery until it’s addressed.  We’ll have more information as it becomes available.

 

*****Edit 3 - 3/19/2013*****

The update has begun rolling out again. Some patience is needed as the update rolls out in phases again. As with other updates; your phone will alert you when the update is ready.

 

Regards,

Nate W.H.
