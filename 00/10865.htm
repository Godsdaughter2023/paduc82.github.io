I still want to keep the ASK field there so the user can edit the value if they want to.

graham mayor say:

I agreee with Doug and frankly I wouldn't use ASK and REF fields for this. I would use a mapped content control copied and pasted to each of the REF locations. https://www.gmayor.com/insert_content_control_addin.htm would make the insertion of mapped content controls a simpler operation.

Whatever you enter in any of the fields will be repeated in all the others. You could fill the initial control from VBA if required, but adding macros to forms requires that the user allows the macro to run, and you certainly cannot guarantee that, so for forms that you are distributing for completion, they are better avoided where possible.

If your form uses legacy form fields, it would be better to replace them with content controls also as forms with content controls are easier and faster to process than forms with legacy form fields.

doug robbins - mvp office apps & services (word) say:

There's probably better things to use than an Ask field, but you can use the following to set the result of such a field

Dim i As Long, rngfield As Range

With ActiveDocument

    For i = 1 To .Fields.Count

        If .Fields(i).Type = wdFieldAsk Then

            Set rngfield = .Fields(i).Result

            rngfield.Text = "Required Text"

        End If

    Next i

End With

jllatham say:

Private Sub TextBox1_Change()
  If Len(TextBox1) = 4 Then
    'simulate hitting the [Enter] key
    'by having it move to the next
    'control in sequence on the form
    'or you could move to another control
    Me.TextBox2.SetFocus
   
    'or move to button and click it!
    CommandButton1.SetFocus
    CommandButton1_Click
  End If
End Sub

Private Sub CommandButton1_Click()
  MsgBox "You entered 4 characters"
End Sub

 

The _Change() event fires each time you type into the text box, so once you've got the 4th character (in my example) in it, it goes off and does other stuff.  You could validate the entry at this point also.  Thing to remember here is you've kind of got to get that last character right!  You can edit and backspace and such until you do get 4 characters (or other number you choose), but as soon as that last one goes in, then the code will take over.

 

BTW: Didn't have to move to the CommandButton before calling its _Click() event, but it was a way to get out of the TextBox1 control itself.
