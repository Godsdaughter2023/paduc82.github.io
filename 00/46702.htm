I had the same problem and eventually cracked it through random trial and error.  I:

  • went to Organize -> Manage Libraries -> Music and added a new library, just an empty folder I created off the desktop.  Then I removed my one (default) library.  It's not very big, maybe 8GB.
  • tried a rip which was written to the new folder off the desktop, it was still pausing between tracks. 
  • went back to Organize -> Manage Libraries -> Music, added the real location of my music and deleted the desktop folder as a location.  I was asked if I wanted to share the "real location" folder or not and I said yes.  A random choice really.  The "share" option had an icon like that for elevated administrator privelege next to it.

I can't explain why that solved it, but it did, maybe something to do with the "sharing"?  Theory:  After the rip, MP is trying to write somewhere non-critical it doesn't have access to: tries, waits and gives up, hence the pause.  By saying I'd share the folder maybe I've relaxed some security and the write no longer failes.

  • delete from library (not deleting the file): fast
  • delete from library with file delete: slow (but file is deleted quickly as seen in Explorer, MP just hangs for some unknown reason afterwards)
  • add existing file to library: fast
  • use windows explorer to delete file: fast
  • other user on the same PC wil files on the same drive, no problems at all
