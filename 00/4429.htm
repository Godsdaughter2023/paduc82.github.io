prosenjitdey say:

IIF LOGIC FUNC IN MS ACCESS 10

i want to prepare a formula in expression builder of a query is that :  if value of a field (field 1) is > 3620001 then A, if value is >655001 but less than 3620000 then B and if value >25001 but less than 655000 then C1, and if value is >1 but less than 25000 then C2.

 I prepare this logic in MS Excel with the help of If AND formula but could not use AND in access.

Pls help.

how can i do this.

scottgem say:

Alias: IIF([fieldname]>3620001, "A",IIF([fieldname]>655001,"B",IIF([fieldname]>25001,"C1","C2")))

This is called nested IFs. Each condition is tested and if true the true result is used. If not it goes to the False condition. That's why you start with the highest value.

I have had success connecting to a UNC file share from Windows Mobile 6 by doing the following:

1. In Internet Explorer, enter the file share UNC name, e.g. \\myfileshare

A popup window should appear for your userid, password, and domain

2. Enter your userid and password

The File Explorer window should now open, showing the top-level share folders

 

One thing I have noted is that you can only navigate down in File Explorer on the share - if you try to navigate up, you get connection not found.  But, you can always navigate back to the top level by jumping over the Internet Explorer to re-enter the UNC file share name.

daniel g. grau say:
When placing footnotes "Beneath text," you initially end with blank space under the footnotes, which are all grouped together. Nothing can go between those footnotes and the end of the page, except more footnotes.

If you insert text before the footnotes (or type a lot of Returns), the footnotes will move down, and the space after them will shrink, but won't disappear.

See if doing Insert > Break > Page Break gets you where you want. Otherwise, provide a sample of how you want your text to look and I'll try to help you do it. It will help if you can visualize the nonprinting characters (namely, the end-of-paragraph mark, ¶). To do this, click on the ¶ button in the Standard toolbar (if you don't see it, select View > Toolbars > Standard).
