chip pearson say:

There is no need to put it in the ThisWorkbook code module. All that does is require that you prefix the variable name with ThisWorkbook (ThisWorkbook.MyVariable). An equally bad way would be to declare it as Public in any worksheet and prefix the variable with that worksheet name when referencing it (e.g., Sheet1.MyVariable). My advice is that you should only declare variables in object modules only if they are used exclusively within that object module, and to scope them as Private. You shouldn't declare Public variables in object modules.  A variable that is to be accessed from any other module be placed on a regular code module.

When VBA is running, it doesn't really matter where a variable is declared as long as it is properly scoped. Where to declare the variables is a matter of organization and personal style of the programmer. The compiler doesn't care.
