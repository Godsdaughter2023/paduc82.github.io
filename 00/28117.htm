"Under the hood," the actions are probably identical.  If you use Recordset.Delete, the form still has to Requery, but there's the added benefit of no "flash" of #Deleted, and the form will probably be left on the "next" record rather than the first one.  But you have to be careful that you're deleting the correct record.  I would do:

Private Sub txtQuantity_DblClick(Cancel As Integer)

Dim rst As DAO.Recordset

' Get a copy of the form's recordset

    Set rst = Me.Recordset

    Set rst = Nothing    Exit Sub

bob jones aka: cybertaz say:

If your version info is accurate, that's the most likely cause of the problem... Both Macs need to have their Office 2011 installations updated. Version 14.4.1 is the current update level. See the suggested procedure on this web page: Update Office 2011

On the 14.0 Mac you'll need to apply 14.1.0 first, then run Disk Utility to repair disk permissions. On the 14.1 system you can go directly to the 14.4.1 update. Repair permissions on each Mac & Restart once 14.4.1 is applied.
