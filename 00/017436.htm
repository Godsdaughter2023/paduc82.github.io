hansv mvp say:

You could keep FrmLogin open all the time (hidden if you prefer), and create a series of queries that select records that belong to the system specified in TxtSystem, by entering

 

[Forms]]![FrmLogin]![TxtSystem]

 

in the Criteria line for the System field. BTW, TxtSystem should be a combo box with 2 columns: the hidden first column for SystemID, and the displayed second column for SystemName. That way, you can pass the SystemID to the queries.

 

These queries can be used as the row source for combo boxes and as the record source for forms and reports.

hansv mvp say:
Yes, indeed. And you can also use queries as row source for combo boxes on forms. For example, on the FrmAddOutage form, you'll want to use a combo box to let the user select an application. By using a query as described in my previous reply, you limit the applications listed in the combo box to those belonging to the selected system.
