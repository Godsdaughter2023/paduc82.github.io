wwtan say:

Insert query result into field

***Post moved by the moderator to the appropriate forum category.***

hansv mvp say:

Perhaps you can use the DLookup function:

=DLookup("FieldName", "QueryName")

or

=DLookup("FieldName", "QueryName", "WhereCondition")

where FieldName is the name of the field whose value you want to return, QueryName the name of the query, and optionally WhereCondition a string specifying which record you want to look at.

scottgem say:

Since a query can return multiple columns and multiple rows you really can't insert the results into a textbox. Even if the query is designed to return a single column and row, Access won't know that.

But, as Hans stated, that's where a DLookup comes in. You might even be able to forego the query and use the DLookup to get the value you want.

ken sheridan say: If the value can be returned at any time by the query on the basis of existing data, then to insert the value into a column in a row in a base table would introduce redundancy.  Consequently, to do so would not only be unnecessary, but also illegitimate as it would leave the table open to the risk of update anomalies.

If, on the other hand, the value can be returned at the present time by the query on the basis of existing data, but should remain fixed at that value in the event of the data from which it is derived subsequently changing, then it is valid to do so.  The value could be assigned to a bound control in a form by calling the DLookup function, without any need to call the OpenQuery method of the DoCmd object, or the query could be changed to an UPDATE query to update the value in the a relevant column in the relevant row of the table.  The query could then be executed in code, but it would of course be necessary to save the row before doing so if it is currently being edited or inserted in the form.
