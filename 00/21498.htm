1.

>>... but I am thinking that since these comment fields, which are long text or memo fields are in the project table they are being loaded every time you open the main form and slow everything down.<<

That is not correct, AFAIK.  You can certainly base your Form on a Query selecting other Fields from your Table [Projects] excluding the Comments Field and in this case, your Form loading will not be affected by the Comments Field.

However, if you want to separate out the Comments Field to another Table, there shouldn't be any problem and in fact, I think a fair proportion of developers prefer the separation.

 

2. This leads to another question: Are you planning a One-to-One Relationship between the Table [Projects] and Table [M_Summary] or a One-to-Many relationship (where it is possible to have many Comments linked to a particular Project)?

It sounds to me that you intended to use the One-to-One relationship.  In this case, perhaps, you should provide details of this Table (Field names, PrimaryKey and possibly, ForeignKey?). 

 

3. To answer your question directly, we need the Table detials mentioned in point 2 as well as the RecordSource of the Form [popupProposal].  Normally, if your Form is bound and RecordSource is editable, Access should open the Form with the NewRecord as the CurrentRecord buffer (i.e. this is "automatic") if the filtering returns an empty Recordset so you should not need the Else part in your code (and consequently, you do not need the If construct, either).  However, [popupProposal] Form is an independent Form (so it doesn't know the value of the DocumentID on the calling Form) so you need to supply the appropriate value of DocumentID to allow the bound [popupProposal] to save the Record correctly with the correct linking value in Table [M_Summary].

Does my PC have to be on in order to run a scheduled scan on my - Microsoft Community

mikethomas-204 say:

Does my PC have to be on in order to run a scheduled scan on my PC?

Does my PC have to be on in order to run a scheduled scan on my PC?
greginmich say:

This is where I’ve explained how to get the MSE scheduled scan up and running:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/mse-scan-schedule-off-by-days/f0e68e51-02d6-4fce-bf0b-999b1f0268c8

The MSE scheduled scan doesn’t have the option to run the scan the next time the PC is started if a scheduled start is missed, or to wake the computer to run the scan – your PC has to be powered up and fully awake in order for the scan to run. So in order to get the scan to run you either have to reschedule it for a time when your PC will normally be in use, or you need to turn off the Windows power-saving feature (sleep) and leave your PC powered up all night long. I’d like to think that rescheduling was the option that the developers had in mind, and we do know that the CPU-capping feature was included just so that the scheduled scan could run in the background while the PC is in use, so it’s clear that rescheduling was an implicit assumption for the developers. Maybe the people who promote MSE just never turned this assumption into a recommendation because they were afraid that this would make them look like they were “taking sides” on a political issue, and coming down on the side of environmental protection.

[EDIT]

 

Task Scheduler uses two specific options to make sure that scheduled tasks run with perfect reliably:

1.) The “Wake the computer to run this task” option assures that the scheduled task will run whenever the PC is powered on at the scheduled time.

2.) The “Run task as soon as possible after a scheduled start is missed” option assures that the scheduled task will run even if the PC is powered off at the scheduled time.

 

So the question of whether or not the PC needs to be powered on at the scheduled time in order for the MSE scheduled scan to run isn’t nearly as frivolous as it might sound. The MSE scheduled scan won’t run at all if the PC isn’t powered on (and awake) at the scheduled time. But a scheduled scan that has the “Run task as soon as possible after a scheduled start is missed” option enabled will run the scan the next time the PC is powered up if the PC is powered down at the scheduled time. So the fact that the MSE scheduled task will never run the scan if the PC is powered down at the scheduled time is actually what makes it inherently unreliable:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_scanning/ms-security-essentials-unreliable-schedule-running/fc048489-a767-4881-8e06-2c5ab949bb9a

 

[End EDIT]
