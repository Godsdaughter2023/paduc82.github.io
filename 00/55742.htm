alanfreeman say: Hi and thanks
the problem seems to have 'solved itself' and I will describe below the steps I actually took.
But first, just to clarify, this is not (I think) a skydrive problem, unless the policies and caching procedures are the same as for networked files (which I'd be interested to hear about). I'm not using skydrive. I just have a homegroup in which Big Laptop has two 500GB drives and Little Laptop has a 250GB drive. Big Laptop hosts all the files for the house and Little Laptop accesses them via a homegroup. Both machines run Windows 8. I upgraded in December to 8.1 and then did a complete re-install of Windows 8, so they are both fairly clean.

'Access all files offline' is disabled on both machines, and all the shares are set to allow network users to decide which files they want to store online (I think this is the default)

Just to complete the picture there is only one user on each machine, who has Admin Privileges on that machine.

When away from home, Little Laptop can keep on using the shared files because whilst logged into Little Laptop, I mark them as 'always available offline'. Then when I get home, I just Sync (with the network shares, not with SkyDrive). However, I do this selectively, because there isn't enough room for all of them. So I just keep offline copies of the files I will work on whilst away, about 34Gb.

This solution is fine for me, I don't yet really have a use for Sky Drive as yet.

The problem is that while I can switch 'keep an offline copy' on selectively, I couldn't switch it off selectively. So one day I mistakenly set a huge folder to be readable offline and then discovered the offline files had gobbled up nearly all Little Laptop's disc space. But I couldn't switch this folder back to remove the offline files from the cache. The relevant menu options were all greyed out.

Globally switching off or disabling 'offline files' didn't help, because as soon as I re-enabled them, Little Laptop 'remembered' what it was supposed to be caching. I guess I could have reduced the cache space to something much smaller, but then all I would have done is remove the functionality that I needed because Sync would fall over complaining there wasn't enough room.

When I searched the web, I found that it was possible in Windows 7 to reset the whole cache, by adding an extra key in the registry (see  eg http://support.microsoft.com/kb/230738/en-us).

But this doesn't seem to work in Windows 8 (see http://answers.microsoft.com/en-us/windows/forum/windows_8-winapps/how-to-re-initialize-the-offline-files-cache-and/9e785cc9-d8ad-42fc-b772-36ac01db18e4). So I wanted to know how to do the same trick in Windows 8

What worked (don't quite know why) are the following steps:

(1) on Big Laptop, move all the files that used to be in the huge folder into a temporary store
(2) on Big Laptop, use the offline share tab on the properties of the once-huge folder, to deny users the right to keep an online cache.
(3) make the temporary store shareable
(3) shut down Big Laptop and restart it
(4) shut down Little Laptop and restart it
(5) Sync Little Laptop again, being careful not to request an offline copy of the temporary store

The greyed-out links seemed to 'wake up' and I was able to remove everything from the offline cache except the files I wanted. But I have no idea why.

Hope this helps somebody, and I think it would be useful to post, somewhere, a 'how-to' article saying how to reset the offline files cache.

Red-faced apologies if the skydrive fixes also work for networked offline files.

jetta48 say:

If its your turn and you can make no legal move whatsoever with any piece, but your King is not in check then it is stalemate. Stalemate counts as a draw in standard chess rules. For examples please see:

http://en.wikipedia.org/wiki/Stalemate .

If your King is in check and you cannot immediatley make a legal move to get out of check then it is checkmate. For the person whose King is in check this counts as a loss. For examples please see:

http://en.wikipedia.org/wiki/Checkmate .

In both of the above scenarios, the same can apply vice-versa (it can apply to your opponent too).

If the above explains what you are seeing in Chess Titans then the program is correctly following standard rules and it is not a program bug. If it doesn't explain what you are seeing then please explain more precisely the scenario.
