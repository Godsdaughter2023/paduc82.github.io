What you should really be doing is setting up Styles with the required paragraph formatting, which where appropriate would include the setting of the Keep with Next attribute and then using the appropriate styles as you enter the text into the document.

charles kenyon say:

First, learn to use Styles effectively. It will save you a lot of time if you are dealing with large documents.

Understanding Styles in Microsoft Word

That article is lengthy. You will still be saving time if you read it.

Second, you probably really want to be applying that formatting to your headings rather than to your entire document. Doing that is best done through modification of relevant heading styles.

Finally, you can select the entire document and apply "keep with next" to everything in the body of the document. I doubt that any experienced and knowledgeable Word user would recommend doing so. If you do, do it with a copy of your document. Applying "keep with next" to all text in a lengthy document is going to give you rather bizarre pagination, especially if coupled with "keep lines together."

jay freedman say:

The "set as default" button in any dialog may fail to save changes to the Normal template for several reasons, including interference from an add-in or an incorrect permissions setting on the Normal.dotm file. When that happens, you can open the Normal template itself "as a document", make the changes there, and save it.

To open the Normal.dotm template in the easiest way:

  • Press Alt+F11 to open the macro editor.
  • Press Ctrl+G (or click View > Immediate Window) to open the Immediate panel at the bottom right of the editor.
  • Type or copy/paste the following line into the Immediate panel and press Enter:

NormalTemplate.OpenAsDocument

  • Close the macro editor.

In your case, set the desired print tray in the Page Setup dialog and click OK (don't bother with the Set As Default button). In the body of the template, insert a space and backspace to remove it, to "dirty" the template so Word will know there's something to save. Then save and close the template.

suzanne s. barnhill say:

Make sure to change the print setting in the mail merge main document. If this is a letter merge, each letter is a separate section, so you would need to change the print tray setting in every single section/letter, but if you set it in the main document before merging, the setting will be replicated in the merged document.

Note that changing the defaults affects only new document based on the Normal template, so even if your mail merge main document was based on Normal, the change in defaults would not affect it.

greginmich say:

Only malware detections are reported in the UI History. Windows Defender’s operational history is recorded in Event Viewer, where specific types of events can be conveniently filtered by ID for analysis:

Right-click on the Start button and select Event Viewer > Applications and Services Logs > Microsoft > Windows > Windows Defender > Operational

https://technet.microsoft.com/en-us/library/mt622090(v=vs.85).aspx

The Windows Defender operational log includes entries for the automatic daily Quick Scans that you’ve already noticed. These are run by Automatic Maintenance – and the trigger-time for Automatic Maintenance can be configured in the Control Panel:

Right-click on the Start button > Control Panel > Change maintenance settings

But note that the run-time for the automatic scan can be much later than the trigger-time, depending on your work habits (see below). Also note that Windows Defender scans of all types can still be programmed with the Create Basic Task Wizard in Task Scheduler:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-set-windows-defender-to-a-custom-auto/ec19aab4-10ce-4f41-9990-343056fd104f

Full Scans and Custom Scans are not automated, so these still need to be performed manually unless you schedule them with the Create Basic Task Wizard. Also, the automatic Quick Scans only run when the PC isn’t in use (when the system is idle) – they get shut down anytime you start using your PC and then have to wait for inactivity before being restarted from scratch. So people who stay busy at their PCs might well want to run a manual Quick Scan as opposed to interrupting their work and waiting for the automatic scan to finish.

GreginMich

greginmich say:

You actually are "fully protected" without running any manual scans at all in the new Windows Defender, Jane. That’s because Windows Defender for Windows 8/8.1/10 comes with authentic on-access scanning, which means that file system activity is continuously monitored and files are automatically scanned whenever they’re read (opened) or written to disk. In addition, Automatic Maintenance runs a daily Quick Scan (an on-demand scan) automatically – without any configuration or scheduling. So scanning is now fully automated, and of course the new Defender also has automatic signature updates and automatic remediation for detected threats.

I’m pretty sure that it’s this totally automated protection that led the developers to believe that accessibility was no longer an important consideration. My personal thought is that accessibility is always an important consideration, and at the very least I would recommend pinning the Defender shortcut to the task bar:

Click on the Start button > All apps > Windows System > Windows Defender. Then right-click on the Windows Defender entry and “Pin to taskbar”.

I've also developed this relatively simple startup shortcut method for putting the Defender icon in the notification area every time you power up your PC:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/is-windows-defender-icon-supposed-to-be-in-system/6cfa9d1d-3db8-49a9-add7-280a284ccfdc
