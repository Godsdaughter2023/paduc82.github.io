amit tandon say:

Place this macro in a standard module:

Sub CopyRow()

Dim rngCopy As Range, wsCopy As Worksheet, wsPaste As Worksheet
Dim lCols As Long, lColsMod As Long, lColsSplit As Long, lResizeCols As Long, i As Long, n As Long

Set wsCopy = Sheets("Sheet1")
Set wsPaste = Sheets("Sheet2")
Set rngCopy = wsCopy.Range("A2:BU2")
lColsSplit = 3  ' number of columns to split
lCols = rngCopy.Columns.count
lColsMod = lCols Mod lColsSplit
wsPaste.UsedRange.Clear

n = 1
For i = 1 To lCols Step lColsSplit
    wsCopy.Activate
    If i > lCols - lColsMod Then
        lResizeCols = lColsMod
        wsCopy.Range(Cells(2, i), Cells(2, i)).Resize(, lResizeCols).Select
    Else
        lResizeCols = lColsSplit
         wsCopy.Range(Cells(2, i), Cells(2, i)).Resize(, lResizeCols).Select
    End If
    Selection.Copy
    wsPaste.Activate
    n = n + 1
    wsPaste.Range(Cells(n, 1), Cells(n, 1)).Resize(, lResizeCols).PasteSpecial Paste:=xlPasteValues
    wsPaste.Range(Cells(n, 1), Cells(n, 1)).Resize(, lResizeCols).PasteSpecial Paste:=xlPasteFormats
    Application.CutCopyMode = False
Next i

wsPaste.UsedRange.EntireColumn.AutoFit
wsCopy.Activate
wsCopy.Range("A1").Select
ThisWorkbook.Save

End Sub

karenkelly_968 say:

Why is internet explorer lock up and is very slow or stop working I have windows 10 on pc It was working ok a few month ago but not now why?

Original title: Internet explorer

ian easson say:

Under certain conditions, IE11 on Windows 10 does the following:

- Becomes very slow (pictures and videos especially take a long time to load)

- Consumes an inordinate amount of memory which grows uncontrollably (memory leak!) until IE11 crashes, or Windows notices this, and logs you out

- Occasionally says it cannot find a web site, although a refresh usually fixes this.

There are many discussions about the memory problem on this forum.  The usual answers are:

- Disable add-ons

- Check for malware

- Reinstall IE11 (by removing it through the control panel, rebooting, adding it back in via the control panel, and rebooting.

However, I did all this (as many have done), and found it didn't help.

However, I found a fix today which worked for me, and to my knowledge, it has not been mentioned in this form.

The fix?  Reboot your internet router!  Try it.  It might work for you (no guarantees).

Here is how I found it:

- I thought I would try Edge to see if it had the same memory problem.  It didn't.

- But, it still had the "web site not found problem.  Hmm ...  it isn't my PC that finds web sites, it's my router in conjunction with my ISPs DNS server.  Simple test?  Reboot the router.  Hey, Edge works now (and a lot faster, and without "web site not found)..)

- Let's try IE11 now.  It's really snappy.  And there are no "web site not found" messages.  And, after stress testing IE11 with multiple web sites open at the same time, the memory does not grow uncontrollably.  It does not crash any more.

My conclusion is that IE11 has a serious memory leak that only manifests itself when it's talking to a router in a "crazy" state (e.g., caused when the electrical power at some point flickered).  Edge does not have this memory leak.

jay freedman say:

How about a simpler plan?

Does each set of pages (4 in your example) end with a manual page break, and are there no other manual page breaks in the entire document? (To examine a sample document for these conditions, first turn on nonprinting characters by clicking the ¶ button on the Home ribbon; then open the Find dialog, enter the code  ^m  in the Find What box, and click the Find Next button repeatedly to see where the cursor stops.)

If that's how the document appears as it comes from the 3rd part application, then

  1. Open the Page Setup dialog (click the little arrow in the bottom right corner of the Page Setup group on the Layout tab) and click the Paper tab of the dialog. In the First Page list on the left, click on the name of the letterhead tray. In the Other Pages list, the tray for the rest of the pages should already be highlighted, but click it if something else was selected. Set the Apply To dropdown to "Whole document".
  2. Install the following macro in your Normal.dotm template and assign it to a keyboard shortcut or a Quick Access Toolbar button. Run it to replace all the manual page breaks with section breaks. Each new section will inherit the Page Setup settings, so the first page of each section will come from the letterhead tray. (That's also how a mail merge handles multiple trays.)
  3. Save the document and print it.

Sub CreateSections()
    Dim rg As Range
    
    Set rg = ActiveDocument.Range
    
    With rg.Find
        .Text = "^m" ' manual page break
        .Wrap = wdFindStop
        While .Execute
            ActiveDocument.Sections.Add Range:=rg, Start:=wdSectionBreakOddPage
            rg.Move unit:=wdCharacter, Count:=1
        Wend
    End With

    Set rg = ActiveDocument.Range
    
    With rg.Find
        .Text = "^m" ' manual page break
        .Replacement.Text = ""
        .Wrap = wdFindStop
        .Execute Replace:=wdReplaceAll
    End With
End Sub
