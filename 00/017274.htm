ken sheridan say: When inserting a new row into tblEquipList  you would select an equipment type from a combo box set up in the way Duane and Scott have described.  However, you might wish to assign the equipment to a new equipment type not currently represented in the tblEqipType list table, which will require a new row to be entered into that table before the row in tblEquipList can be saved.

To do this you can make use of the NotInList event procedure of the EqType combo box in the form.  This enables you to type the new eqTypeDescription into the combo box, which will cause the NotInList event procedure to execute.  In your case you will need the code to pop up a dialogue form, bound to  tblEqipType, at an empty new record, passing the eqTypeDescription value you entered into the combo box to the form. This will enable you to then insert values into the EqTypeVerInterval and EqTypeEVP columns.  When the dialogue form is closed, and focus is returned to the calling form, the new row will appear in the EqType combo box's list and the EqTypeVerInterval and EqTypeEVP will appear in the unbound controls.

You'll find an example of this in NotInList.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates the use of the event procedure in a number of contexts.  The City combo box in the frmContacts form would be the model to follow in your case.  It's NotInList procedure is as follows:

Private Sub City_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control
    Dim strMessage As String
    
    Set ctrl = Me.ActiveControl
    strMessage = "Add " & NewData & " to list?"
    
    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        DoCmd.OpenForm "frmCities", _
            DataMode:=acFormAdd, _
            Windowmode:=acDialog, _
            OpenArgs:=NewData
        ' ensure frmCities closed
        DoCmd.Close acForm, "frmCities"
        ' ensure city has been added
        If Not IsNull(DLookup("CityID", "Cities", "City = """ & _
            NewData & """")) Then
            Response = acDataErrAdded
        Else
            strMessage = NewData & " was not added to Cities table."
            MsgBox strMessage, vbInformation, "Warning"
            Response = acDataErrContinue
            ctrl.Undo
        End If
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If
    
End Sub

This code calls the frmCities form, in which the region in which the city is located can be entered/selected.  The code in the form's Open event procedure assigns the value, i.e. the city name passed to the form via its OpenArgs property to the city text box control in the form as its DefaultValue property:


Private Sub Form_Open(Cancel As Integer)

    If Not IsNull(Me.OpenArgs) Then
        Me.City.DefaultValue = """" & Me.OpenArgs & """"
    End If
   
End Sub

The above example is for demo purposes only, as in real life city names can be duplicated.  A more realistic scenario is to be found in DatabaseBasics.zip in the same OneDrive folder.  In this the section on 'entering data via a form/subforms' includes a contacts form in which correlated combo boxes are used to select a country, then a region and finally a city.  The first two are unbound, while the city combo box is bound to the CityID column in the table.  The NotInList event procedure of each combo box allows a new country, and/or region and/or city to be entered into the control, but in this case no dialogue form is opened as all the necessary data is entered into the referenced tables in the event procedures:

Private Sub cboCountry_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control
    Dim strSQL As String, strMessage As String
    
    Set ctrl = Me.ActiveControl
    
    ' subject to user confirmation insert a new row into Countries table,
    ' inserting NewData value into Country column
    strMessage = "Add " & NewData & " to list?"
    
    strSQL = "INSERT INTO Countries(Country) VALUES(""" & _
            NewData & """)"
   
    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        CurrentDb.Execute strSQL, dbFailOnError
        Response = acDataErrAdded
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If
    
End Sub
