Photo email feature broken - Microsoft Community

¡firedog say:

Photo email feature broken

Update 14 July 2016
        

The fault that caused this error for a short period in 2015 was corrected. Users now having trouble using the photo email feature are facing a different problem. The most likely reason is that the Microsoft Account being used has been migrated to the Outlook Mail platform; users will often see an error message including error code 0x8007007A.

Users seeing this error code may not be able to send any messages at all until the offending message has been removed from the outbox (below the account folders in the folder pane). Some alternative methods of sharing pictures are explained in this article: Avoid 'photo email' and send photos by email

Stop press 8 July 2015 13:00 UTC

This bug appears to have been fixed. Once the error has been cleared, it should now be possible to send photo email again. The 'slideshow' view may not work at OneDrive, but the photos can be viewed and downloaded.

In the past two days, a number of users have complained that the photo email feature is no longer working. After creating a photo email and clicking send, the process starts and then stalls.

At this stage, the following is recorded in the program log:

[13:04:26.65] 17d8                 Mail:            Zone_MailChk ERROR: (containx.cpp:3808), failed with 0x800CCE05 
[13:04:26.65] 17d8                 Mail:            Zone_MailChk ERROR: (containx.cpp:3808), failed with 0x800CCE05

Photomail failed to complete publish: System.Runtime.InteropServices.COMException (0x80010105):

   at WindowsLive.Writer.PostEditor.PhotoAlbums.PhotoMail.IPublishCallback.PublishComplete(Int32 hr, IHTMLDocument2 newEmail, IPhotomailAlbum photoAlbum)
   at WindowsLive.Writer.PostEditor.PhotoAlbums.PhotoMail.PhotoMailPublisher.Publish(IPublishCallback callback, IHTMLDocument2 document2, String language, String wpost)

To clear the error, the user must:

1.   Disconnect from the Internet (e.g. switch off Wi-Fi or unplug the Ethernet cable).
      This will stop the program from trying to send.

2.   Remove the message from the outbox (below the account folders in the folder pane).

3.   Click Stop in the send/receive progress window.
      An error message with code 0x80004004 (ABORT) is displayed.

4.   Close the progress window.

Instead of using the photo email feature, users could:

A.     Start a new email message from within Windows Live Mail (Ctrl-N), then use either

Attach files to send images as attachments to the message, or

Insert > Single photo to embed images into the message and apply some formatting.
The images will still be sent as attachments, but they will usually be visible to the recipient
in the body of the message.

B.     Upload the images to OneDrive, then use the sharing options there to send a link to addressees by email.

Will support staff please confirm this fault and inform users about when they may expect the broken feature to be mended?

There is a suggestion on the OneDrive UserVoice page asking the OneDrive team please to reverse the change that caused this fault. The more votes the suggestion gets, the more likely it is that the team will look at the problem.

PS A number of users have posted solutions involving forcefully closing down Windows Live Mail, e.g. by using Task Manager to end the wlmail.exe process or by re-starting the computer. These methods can't be recommended except in the last resort, because Windows Live Mail often conducts internal housekeeping after the program interface has disappeared. If this process is interrupted, damage to the program's databases may result.

The program will normally stop trying to send the message soon after the Internet connection has been broken; clicking once on Working online on the status bar may also speed up the release of the program's handle on the unsendable message so it can be removed from the outbox.

An option to have the Search utility show paths for all its results, would be useful.

 

Hi Ray

In Windows Explorer set the View for the search folder to Details, using the View Menu at the right side of the Toolbar.

Right click the Name column header and select the Folder item.

If you do not see the Folder item, click More, scroll down and select the Folder item. This will add the Folder column.

Use the left mouse button to drag the folder column to the left, next to the Name column.

 

Now when you search, the Folder column will show the path to each found file.
