    hansv mvp say:

    You could use

     

    ActiveWorkbook.SaveCopyAs "SomeOtherName"

     

    to save the active workbook under another name (without changing the format). This will leave the active workbook unchanged. You can then open the copy and save it (or one of its sheets) in another format. Finally, close the new workbook and delete it.

    Hi,

    I'm Jesper, one of the developers on Skype for Windows 10. The reason we're not showing strike-through correctly is that on the first Windows build we released in (OS build 14393), the Rich Text control we're using didn't support strike through text. So as a work around, we render it in grey with a dash before & after.

    Windows did add support for strike through in build 15063, but we didn't yet get around to pick that up. Partly becase the app still needs to run on older Windows build, and partly because, well, we didn't get any feedback on this that I know until you raised it now.

    We still have the work item open to support strike-through and it's pretty high up on the list at this point. I'll add a link to this post as well to give it a little extra weight.
