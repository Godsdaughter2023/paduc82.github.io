gizmogary say:

OK so I finally managed to figure it out for myself as none of the fixes worked as there is nothing wrong with Windows Updates etc.

I searched the registry for any mention of the Windows Apps folder location on the raid drive which was M:\WindowsApps... I found this key...

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Appx\PackageVolumes\2

"Flags"=dword:0000000e
"MediaId"="{20202020-5720-2D44-3561-2FBCD496DFEA}"
"MountPoint"="E:"
"Name"="\\\\?\\Volume{b2b40d7c-3e9f-11e4-8e09-806e6f6e6963}"
"SisPath"="M:\\WindowsApps"

As you can see, it was using the correct path but the mount point was incorrect, it is referencing E: which no longer exists... There was also another key which referenced the mount point correctly as M:. I removed the incorrect key and now the download is working...

I have no idea why it created this but at least it is working... and hopefully this info will help someone else..
