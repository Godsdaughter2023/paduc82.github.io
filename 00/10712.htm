For those who might not be aware of it, the new beta command line utility (MpCmdRun.exe) is capable of scanning individual files from the command prompt. Here’s the syntax provided:

 

 

-Scan [-ScanType #] [-File <path> [-DisableRemediation]] Scans for malicious software

 

-Scan [-ScanType value]

       0  Default, according to your configuration

       1  Quick scan

       2  Full system scan

       3  Single file custom scan

 

          [-File <path>]

               Indicates the file path to be scanned, only valid for custom scan

          

          [-DisableRemediation]

               This option is valid only for custom scan.

                 - File exclusions are ignored.

                 - Archive files are scanned.

                 - Actions are not applied after detection.

                 - Event log entries are not written after detection.

                 - Detections from the custom scan are not displayed in the user

pa bear - ms mvp say:

In order:

1. Close all IE windows on the second monitor (screen).

2. On the first monitor (screen), close all but one (1) of these smaller windows. "Grab" the corners of the remaining window with your cursor and "stretch" it to the desired size but do NOT maximize it!  Now close the resized window and do NOT open any new ones yet.

3. On the second monitor (screen) only, open new IE windows until you get one of the smaller ones. Now close all but the one (1) smaller window.  "Grab" the corners of the remaining window with your cursor and "stretch" it to the desired size but do NOT maximize it!  Now close the resized window.

4. Test.

NOTE: The resized window MUST be the last window closed in order for its size to be remembered.

Also note that some windows are hard-coded to open in a smaller size - not much you can do about that, other than installing IE New Window Maximizer: http://www.jiisoft.com/iemaximizer

I do something very similar, and have found and modified code to 'import' the list of file names into a specific table, then from there, another process them imports the actual file.  I've tuned it for my specific purposes, you should be able to also.

 

I break it into steps (easier for me to manage and visualize).

   * Import file names from given folder    (see code below #1)

   * Count number of files for looping process   (See code below #2)

   * Loop

      * Import files and process

      * Archive file

      * Delete file

 

Some assumptions: 

* The path where the text files are saved is constant.

* Once I have the file names, then there is the import process which loops through and imports each file, then copies the file to an archive folder.

Here's the import file name process (Thanks to Joe Fallon)

************************************************

#1

'Joe Fallon  -  Access MVP

'Create a table named tblDirectory with 2 fields:
'FileName  (Text 250)
'FileDate (Date / Time)

'SelectYN field  (Yes/No)   - I added

'Call the code below by pressing Ctrl-G to open the debug window and type:
'GetFiles ("c:\windows\")

'Paste this code into a regular module:

 

Sub GetFiles()
    Dim strPath As String
    Dim rs As Recordset
    Dim strFile As String
    Dim strDate As Date

   

strPath = Forms![frm-Menu]!txtFilepath

 

   'clear out existing data
        CurrentDb.Execute "Delete * From tbl_ImportFiles", dbFailOnError

 

   'open a recordset
        Set rs = CurrentDb.OpenRecordset("tbl_ImportFiles", dbOpenDynaset)

 

   'get the first filename
        strFile = Dir(strPath, vbNormal)

   'Loop through the balance of files
        Do
      'check to see if you have a filename
        If strFile = "" Then
            oTo ExitHere
        End If
       
        strDate = FileDateTime(strPath & strFile)
     
        rs.AddNew

      'to save the full path using strPath & strFile
      'save only the filename
        rs!FileName = strFile
        rs!FileDate = strDate
        rs.Update

 

      'try for next filename
        strFile = Dir()

   Loop

 

ExitHere:
   Set rs = Nothing
   MsgBox ("All files have been imported - ready for the next step.")
End Sub

*************************************

 

Next is to actually import the file  (this fits my needs, you'll need to adjust)

I use a YesNo field to keep track which file I'm processing ---->  SelectYN = -1 is the selected file

 

**********************************************

#2

Private Sub Command5_Click()
On Error GoTo Command5_Error
    Dim filepath As String
    Dim FileName As String
    Dim archivePath As String
    Dim file As String
    Dim test As String
    Dim countFiles As Integer
    Dim i As Integer
           
    filepath = Me.txtFilepath
   
    countFiles = DCount("*", "FolderNames")
   
'Check for any files available
    If countFiles = 0 Then
        MsgBox "No files to import", vbOKOnly, "Import Error"
        Exit Sub
    End If

 

'Check for selected files
    countFiles = DCount("*", "FolderNames", "SelectYN = -1")

    If countFiles = 0 Then
        MsgBox "No files selected to import", vbOKOnly, "Import Error"
        Exit Sub
    End If
    
    DoCmd.SetWarnings False
   
'Begin the transfer of the text files and Delete the Prior data from the "New_Data" Table
    DoCmd.OpenQuery "Q_Del_New_Data", acNormal, acEdit

 

    For i = 1 To countFiles
        FileName = DLookup("Names", "FolderNames", "SelectYN = -1")
        file = filepath & FileName
        Me.remitName = FileName

'Make sure all checkboxes a false

        DoCmd.OpenQuery "qry-x-updateCheckBox"
       
'Import the flat file
    DoCmd.TransferText acImportFixed, "Import Specification", "Raw_Data", file, False, ""
   
'Process data
    DoCmd.OpenQuery "Q_1NewData", acNormal, acEdit
    DoCmd.OpenQuery "Q_2CashAccounts", acNormal, acEdit
    DoCmd.OpenQuery "Q_3Cash1011", acNormal, acEdit
    DoCmd.OpenQuery "Q_4DoneYN", acNormal, acEdit

 

'Copy file to archive    
    FileCopy file, "F:\FMS_ADI Templates\ADI Files\Import\Archive\" & FileName


'Delete file

    Kill (file)

 

Next

 


Exit Command5:    
    DoCmd.SetWarnings True
    Exit Sub
   
Command5_Error:
    resume Exit_Command5

 

End Sub
