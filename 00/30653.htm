Some obvious things wrong with existing Skype 8 features - Microsoft Community

rseiler say:

Some obvious things wrong with existing Skype 8 features

I invite everyone to add their own to this starter list.

What you see below are what I consider to be Microsoft's half measures, or unforced errors, when it comes to Skype 8. Features implemented in such a way that you can only shake your head and wonder why they didn't finish the job.

A whole other thread could be started collecting features missing outright (compact UI, custom sounds, etc). These are just the ones done partially or poorly or weirdly--in most cases, issues not present (where applicable) in Skype 7. Also, it should be noted that the situation may vary slightly for a couple of these if using the Store app, which I'm not.

Messaging:

  1. "This message has been edited." But WHEN?
  2. URLs are spell-checked (no one, under any circumstances, benefits from this).
  3. Dividing line separating typing area from history can't be adjusted up or down, making for a painfully contracted space even after the very limited automatic expansion.
  4. When pasting something into the upper portions of a lengthy message, you'll be scrolled to the bottom of the message, losing your place.
  5. "React" emojis appear on every incoming message. :(
  6. You can change text size, but not specify the point size or font.
  7. The "draft" feature is nice, but it only applies if you properly exit Skype. In other words, it's not an autosave protecting you from crashes, when it easily could be.
  8. Skype often adds this demarcation line upon receiving messages: "-----Unread messages-----", and leaves it there until the recipient sends a reply. Even the act of typing a reply and not sending it isn't enough to convince Skype that you've read the message. It's more of an "Unreplied messages" notice than an "Unread" one, which is kind of senseless. Note: It will disappear if you switch to another chat and then back.
  9. [ADDED: 8/24/2019] "Enable split view mode" remembers neither a window's size nor position.

Notifications:

  1. No outward notification received upon getting online for a message sent to you when you were offline.
  2. No persistent notification (i.e. taskbar icon changes color) for a new message unless you happen to have the given chat window already open. If you don't, it's easy to miss messages if you step out even for a minute, since the only notification will be the 5-second one in Windows.
  3. "Contact comes online notification" setting works in limited circumstances and never with sound, limiting its usefulness almost completely.
  4. If you have another program completely full-screen (no visible desktop taskbar), then Skype's notification of an incoming message will not sound or appear, remaining unknown to you until you return to the desktop.

History:

  1. Cloud history: only 2 years (Microsoft runs Azure, and text compresses to almost nothing).
  2. Cloud export is sloppy, littering the output with HTML tags in some places, impeding readability (unlike local export).
  3. Ctrl-F search history: 6 months and/or 40 results, though it's so slow that you may not care.

Contacts:

  1. Every number called is left behind as a "conversation," cluttering up the main window (and they need to be deleted from web.skype.com in order to be permanently deleted).

Other:

  1. Skype's window position isn't remembered on startup (it's centered instead).
  2. [ADDED: 5/29/2019] The command /showplaces now exists in Skype 8 (it was a longtime command present in older versions of Skype until removed without explanation a year or two ago), but its companion command /remotelogout does not exist.
  3. [ADDED: 8/15/2019] The tray icon, despite turning yellow when status is manually set to 'Away,' will not change to yellow automatically with system inactivity (after X minutes). This handy visual cue existed in versions of Skype before v8.

a. user say:

By default, user accounts do not have access to each other's files. You can change that by altering the access control lists, but you're playing with fire. Do lots of research and be very careful before changing access permissions.

Track Changes in Word is intended for shared files on a common drive or, in the olden days, files that get passed around by email. If you store your work on OneDrive (for example) and allow other users to access those files, Track Changes can work the way you want it to work.
