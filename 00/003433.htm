hansv mvp say:

To be on the safe side, make a copy of the workbook (or of the worksheet) before you begin.

Open the workbook and activate the worksheet.

Press Alt+F11 to activate the Visual Basic Editor.

Select Insert | Module to create a new code module.

Copy the following code into the module:

 

Sub DuplicateRows()
    Dim r As Long
    Dim m As Long
    Dim Qty As Long
    Const FirstDataRow = 2 ' change as needed
    Const QuantityCol = "B" ' change as needed
    Application.ScreenUpdating = False
    m = Range(QuantityCol & Rows.Count).End(xlUp).Row
    For r = m To FirstDataRow Step -1
        Qty = Range(QuantityCol & r)
        If Qty > 1 Then
            Range(r & ":" & r).Copy
            Range((r + 1) & ":" & (r + Qty - 1)).Insert
        End If
    Next r
    Application.CutCopyMode = False
    Application.ScreenUpdating = True
End Sub

 

Change the value of QuantityCol to the column containing the Qty values.

Change the value of FirstDataRow to the row number of the first row that you want to process.

Switch back to Excel.

Press Alt+F8 to display the Macros dialog.

Select DuplicateRows, then click Run.

I would do this in a "non destructive" fashion by putting the result in a separate worksheet.  So, suppose the number of labels you want for each row is in the first column and the data start with row 2 (row 1 being a header).

Use the code below to delete an old worksheet named "Rows for Mailmerge", create a new worksheet by the same name, and put the repeated rows in this worksheet.

Put the code below in a standard module and run the duplicateRows subroutine.

 

Option Explicit

Const cTempWSName As String = "Rows for Mailmerge"
Sub deleteTempWS(WB As Workbook)
    On Error Resume Next
    Application.DisplayAlerts = False
    WB.Worksheets(cTempWSName).Delete
    Application.DisplayAlerts = True
    On Error GoTo 0
    End Sub
Sub duplicateRows()
    Dim Rng As Range: Set Rng = Selection.CurrentRegion
    deleteTempWS ActiveWorkbook
    Dim NewWS As Worksheet: Set NewWS = ActiveWorkbook.Worksheets.Add()
    NewWS.Name = cTempWSName
    Dim FreeRow As Long: FreeRow = 1
    Rng.Rows(1).Copy NewWS.Cells(FreeRow, Rng.Column)
    FreeRow = 2
    Dim I As Long
    For I = 2 To Rng.Rows.Count
        Rng.Rows(I).Copy NewWS.Cells(FreeRow, Rng.Column).Resize(Application.Intersect(Rng.Rows(I), Rng.Columns(1)).Value)
        FreeRow = FreeRow + Rng.Rows(I).Columns("A").Value
        Next I
    End Sub

The actual result came from a Nokia engineer, which solved the problem completely.

 

http://msdn.microsoft.com/en-us/library/ee942815(v=VS.85)

 

  1. On the NPS server start regedit.exe
  2. goto HKLM>System>CurrentControlSet>Services>Rasman>PPP>EAP>25

  5. Restart the NPS Server Service
