Now to deal with this error  in MSE:

 

0x80070005 is an Access is Denied error this is because either an application or executable does not have the necessary elevation to invoke a specific action. One possibility is you are trying to do this or did the update or installation as a standard user instead of as an Administrator - especially if you are using XP. Login to an Admininstrator account to proceed. This is not always required and I'm not sure about your circumstances or configuration or operation system, but given that error we should err on the side of cautiion and use the Administrator account here.

 

Let's start with some possible troubleshooting ideas and solutions from others who also experienced this error code:

 

Please see if the answer in the following thread helps: http://answers.microsoft.com/en-us/protect/forum/protect_start/error-code-0x80070005/1e5cc042-2fcc-4ca1-a489-b44b1343e224.

 

If not try the answer and the other suggestion in this thread: http://answers.microsoft.com/en-us/protect/forum/protect_start/0x80070005/563455d4-5ec5-4e8f-a8d3-b6a4005b189d. It is similar to the above, but includes another link which may apply depending on your version of Windows.

 

At this point if the above did not work, let's do a complete troubleshooting process to try to identify the cause and the solution.

bill manville say:

I don't know why the macros don't work for you (assuming the worksheet containing all the hyperlinks is active when you run the macro).  Do you get an error message?  If so hit Debug and tell us what the message was and what the Address and TextToDisplay properties of the hyperlink were.

As to why the change happens, I would guess that one of the following happened:

a) You received a copy of the spreadsheet as an email attachment, opened it in Outlook and saved it to its usual folder

b) You opened a "Recovered" version of the spreadsheet after an Excel crash and saved it back to its usual folder.

Hyperlinks can be either absolute or relative.  By default if the destination of the hyperlink is in the same folder as the workbook containing the hyperlink, or a sub-folder thereof, the hyperlink will be created as relative.  You can then move the set of files as a group (or access them from a different drive letter or path) and the hyperlinks will still work.  But if you Save As to a different folder the hyperlinks become absolute to the folder you opened the workbook from.  In scenarios (a) and (b) the file you opened was in an AppData folder when you opened it.

If your hyperlinks had all been absolute this would not have been a problem - though you would lose the ability to move all the files as a set and have them still work.

You can force all hyperlinks to be absolute by setting the master workbook's Hyperlink Base property.  File > Info > Show All Properties and it's about the 10th line down on the right hand side.  Set it to \\NoServer\NoFolder (for example).

Or you can build your hyperlinks using the HYPERLINK worksheet function:

If the path to the files is in a cell named Path =HYPERLINK(Path & "\ThisFile.xls","ThisFile")

You can then change the expected location of the files by changing the Path cell's contents, and the problem of the links changing by themselves will not occur.
