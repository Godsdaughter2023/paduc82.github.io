If you want it in the first line you must declare OurValue as a constant:
  Const OurValue = "action done"

But if you do that, you can't change the value at runtime. This can only be done with variables:
  OurValue = "action done"

BTW, the typ of the variable must be String or Variant. If a variable has no given typ, then the typ is Variant:

  Dim OurValue As String
or
  Dim OurValue As Variant
or
  Dim OurValue

And if you want to compare string's it's often a good choise to place a
  Option Compare Text
at the beginning of the module to make no distinction in the case. For example:
  If "Andreas" = "ANDREAS" Then
    MsgBox "Option Compare Text is set"
  Else
    MsgBox "Option Compare Binary is set"
  End If

Strings can also compared with the StrComp-Method, have a look at the OLH for this.

Another option when comparing strings is the LIKE-Operator, because you can use Wildcards (and more). For Example:
  If "Andreas" Like "Andr*" Then
    MsgBox "Match found"
  End If

And if you want to compare only a part of a cell you can use LEFT, RIGHT or MID to get a part of it.

Many people get into trouble when comparing string, because they have one and more blanks in a cell. In this case it's better to remove blanks by using the REPLACE-Method:

"action done" isn't "action  done", because there are 2 blanks in the second string and only one in the first, but
Replace("action done", " ", "") is Replace("action  done", " ", ""), because both strings became "actiondone".

A last word to formatted cells: Many people wonder why they did not find the month-name in a date, although it is displayed. If you want to do that, you have to use the .TEXT-Property of the cell, because a date is only a number.

Okay, finished with the confusion, don't worry, take your choise.  :-)

Use Paste Special in Word and paste as formatted text. That will turn both Char(10) and Char(13) into paragraph marks.

I think Marshall is mistaken, for once, about what you're asking.  I think you want to return those records from tblWeatherData where the TempOut = the maximum value of TempOut in the table.  This is the problem of returning data related to an aggregated result, as discussed in this article on The Access Web:

 

    http://access.mvps.org/access/queries/qry0020.htm
    Queries: Getting a related field from a GroupBy (total) query
 

If I'm right, I think you want something like this:

 

    SELECT
        TempOut, Format(tblWeatherData.[datetime],"m/d/yyyy") AS EventDate
    FROM tblWeatherData
    WHERE TempOut = 
        (SELECT Max(T.TempOut) FROM tblWeatherData T)
