ashraf a say:

After failing to receive any appropriate response from Microsoft, I continued to trouble shoot the issue with the help of a computer professional and got to the bottom of it (we think)

Windows 10 uses a different file permission system than previous versions. The easy fix is to start the software permanently in Administrator mode:

  • right click on outlook
  • Go to properties
  • Go to advanced and set it to run as administrator

In addition to the above, you should be aware that if you sync your contacts to a phone (in my case a Samsung S4 using Smart Switch), then the sync software also needs to start in Administrator mode otherwise it will not be able to access the contact database...

Mailboxes - Changing Exchange Attributes

Technical Level : Intermediate

Summary

  • A change to an attribute on-premises is not reflecting in the cloud
  • A change to an account in AzureAD is not reflecting in EXO
  • A change to an account in EXO is not reflecting in AzureAD


Details

Data Collection

See Collect Recipient Info from Powershell

Solution Analysis

Changes flowing to EXO (On-Premises to AzureAD):

  1. Make sure the account is properly linked between on-premises and AzureAD: follow Sync Anchor Values workflow

  1. Does AzureAD reflect the change?
    1. If not, two possibilities:
      • The attribute is not supposed to sync - see List of attributes that are synced by the Azure Active Directory Sync tool
      • DirSync is failing to export the change: refer to the DirSync Issues workflow

AzureAD to On-Prem:

  1. Make sure the account is properly linked between AzureAD and On-Premises: follow Sync Anchor Values workflow
  2. Does On-Premises reflect the change?
    1. If not, two possibilities
