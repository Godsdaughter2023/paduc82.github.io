gregorylockett say:

How can move items in print queue from 1 printer to another

title is the question,.... There are 2 printers set up, only need one,  the throwaway has items in queue and I'd like to move them.
lemp say:
The jobs that are already in the print queue are files that have been produced by the printer driver and therefore include printer commands that are specific to that printer.  Directing the print queue to a second printer will only work if the second printer can properly interpret the printer commands intended for the first printer.

If the second printer is the same type as the first, the easiest method is to temporarily redirect the port assigned to the second printer.

  1. Go to Printers and Faxes
  2. Right-click on the icon for the "good" printer and select Properties, then click on the Ports tab
  3. Scroll down the list of ports and make a note of the port to which the "good" printer is directed (the one with a check) Click Cancel to close the Properties dialog
  4. Right-click on the icon for the "throwaway" printer, select Properties, then click on the Ports tab
  5. Make a note of the port that is checked, uncheck it, and check the port that you noted in step 3.
  6. Click OK

waynecmartin say:

How to change "CRLF" to "LF" in Windows XP?

How do I cahnge the "CRLF" to "LF" in Windows XP. The "CRLF" is messing up my Oracle compilations.
wunders say: Traditionally this is accomplished with a program called "DOS2UNIX".  Every version of Linux comes with this command-line program built-in.  There are several versions of this program written for Windows.  One is here:

  < http://sourceforge.net/projects/dos2unix/ >

A better way to handle this is to use a Text Editor that will work with either line ending.  Once your source file has the correct line ending, the editor will continue to use this EOL character each time you edit the file.  IMHO, one of the better freeware Windows text editors with this feature is Notepad++

  Notepad++:  < http://notepad-plus-plus.org/ >

Open the file with Notepad++, then select  Edit -> EOL Conversions -> UNIX Format
This will convert the file to the LF EOL character.

Hello Krypto,

Thank you for contacting Microsoft Community.

We understand the inconvenience you have experienced.

Are you getting the message "Press any key to boot from CD or DVD" during startup?

From your description, I suspect that the problem might be with the bootable USB device which you're trying to install. Suggest you to check using a different USB stick or if possible new one by making it a bootable, to see if it works correctly. Do check by connecting the device to different USB port.

If your PC had a CD/DVD drive support, then try making a bootable Windows 10 DVD and check if it allows your PC to install Windows 10. Also make sure that the Boot Priority Order is set correctly in BIOS.

  • During the restart process, read the screen for any instructions that explain how to interrupt normal startup and enter the basic input/output system (BIOS) setup utility. Most PCs use the F2, F10, ESC, or DEL key to begin the BIOS Setup.
  • Look for a tab in the BIOS Setup Utility that is labeled Boot Order, Boot Options, or Boot. Following the directions on the screen, use the arrow keys to go to the Boot Order, then press Enter.
  • Locate the CD, DVD, or USB flash drive (this might be called Removable Device) in the Boot list. Following the directions on the screen, use the arrow keys to move the drive up so that it appears first in the Boot list. Press Enter. The boot order sequence is now changed to boot from the CD, DVD, or USB flash drive.
  • Press F10 to save your changes and to exit the BIOS Setup Utility. Select Yes in the confirmation window. The PC will restart. Allow the PC to restart normally. The scan will take a few minutes and remove any malware that may be infecting your computer.

Warning:

  • Be careful when changing BIOS settings. The BIOS interface is designed for advanced users, and it's possible to change a setting that could prevent your computer from starting correctly.

  • Updating BIOS should only be done if necessary (to solve a compatibility problem, for example). It can be a complicated process, and if an error occurs, your computer could be rendered inoperable. Be sure to follow the manufacturer's instructions exactly.
