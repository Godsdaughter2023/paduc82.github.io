There are several ways to accomplish this.  Below is a procedure for directly querying for the ID value.  The normal precautions apply here for using name as the basis for a query.  For example, what happens if you have two women named Jenny Doe in your

Public Function ReturnAutoID(sLastName As String, sFirstName As String) As Long
    Dim sSQL As String
    Dim rs As DAO.Recordset
   
    sSQL = "SELECT [ID] as MyID FROM MyTableName WHERE LastName='" & sLastName & "' AND FirstName='" & sFirstName & "'"

    ReturnAutoID = Nz(rs("MyID"), -1)   'Return -1 if ID does not exist for some unknown reason
    
   
End Function

jay freedman say:

Your macro doesn't stop because the logic says to check the contents of every required formfield each time the macro runs. The logic you want is to check only the contents of the specific formfield that's being exited.

The following version will find the name of the formfield being exited, and use that to assign that formfield to a variable. Then a Case statement uses the name to find out whether that field is one of the required ones. If so, it checks the contents and displays the InputBox if necessary.

Sub MustFillIn()
    ' exit macro for each field (or at least the required ones)
    Dim fld As FormField
    Dim sInFld As String
    
    ' the exit macro fires before the cursor leaves the field,
    ' so Selection.Bookmarks(1) is the name of the field being exited
    Set fld = ActiveDocument.FormFields(Selection.Bookmarks(1))
    
    Select Case fld.Name
        Case "Name", "Date", "Start", "Stop", "Hours", "Location", _
            "DefName", "CaseNum"
            While Trim(fld.Result) = ""
                sInFld = InputBox(fld.Name & " field must be filled in, fill in below.")
                fld.Result = sInFld
            Wend
        Case Else
            ' nonrequired field, so do nothing
    End Select
End Sub
