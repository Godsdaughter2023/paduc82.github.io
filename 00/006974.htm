Windows wouldn't arbitrarily try to convert an MBR partition to GPT, so if that's what occurred to cause this I'd be looking for what was done that may have triggered this conversion.

The only point I was making is that these items you're describing belong on a system using modern GPT partitions, so trying to remove them will fail and only cause additional problems.

The use of 3rd-party partition managers is dangerous, which is why the GPT partitions now contain self-protection, while the old MBR partitions were often damaged by malware or rogue partition managers, which some computer manufacturers attempted to avoid using firmware protection in the MBR days.

I can't say what triggered either the conversion of the partitions or the addition of those items if they weren't already present, only that their presence is normal.  Use this information to try and understand what changed and why, since that's where your original issue came from.
