ken sheridan say: Opening a query will not open the dialogue form.  For a query to reference a control in a form as a parameter the form must already be open.  So if you want to open a query as a datasheet then you must open the form first, and include a button on the form to open the query.

If the query is the RecordSource property of a report then you can set up the report to open the dialogue form when the report is opened.  You'll find two methods illustrated in ReportDialogue.zip in my public databases folder at:

ken sheridan say: I think you may be confusing the use of a simple parameter in a query, e.g. [Enter start date:], with a parameter which is a reference to a control on a form, e.g. Forms!frmDateDlg!txtStartDate.  The former is a cheap and cheerful solution, but a crude one.  If the query is opened a system generated prompt will ask you to Enter start date:  Where the parameter is a reference to a control on a form, if the query is opened while the form is not open it will work in the same way, and ask you for the parameter, Forms!frmDateDlg!txtStartDate.  It will not open the form.  When you use a dialogue form you must open the form first, and normally this would be built into your application design along these lines.

1. Open the dialogue form from a button on a menu form or similar.
2. Having entered the parameter values into the dialogue form open the query, or better still a bound form or report based on the query, by means of a button on the dialogue form.

My MultiSelect demo to which I referred you earlier is an example of this.  In my demo a report is opened from a dialogue form, rather than simply opening a query in datasheet view, which is a rather crude solution, and not one I would ever use in an application.

The dialogue forms in my demo use different means to restrict the report's results.  In some cases the report's query uses parameters, in others the report itself is filtered by the code in the Click event of the relevant button on the dialogue form.

suzanne s. barnhill say:

The blue (or grey, depending on Word version) box is by design. In any check box, that condition (filled but not checked or clear) indicates mixed conditions, which in this case indicates that some text in the document is formatted as "Do not check."

 

If you experience the problem at the end of the document, try selecting just the last paragraph mark and make sure that it is not formatted as "Do not check."

 

For further discussion, see http://WordFAQs.mvps.org/MasterSpellCheck.htm.
