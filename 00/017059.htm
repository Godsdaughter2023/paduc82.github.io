mouran yan msft support say: Hi Cory,

From the description, it seems that the default room calendar permission is changed. To confirm it, please follow the below steps to run a PowerShell command to get the default room calendar permission:
1.    Connect to Exchange Online PowerShell.
2.    Run the command:
Get-MailboxFolderPermission -Identity *** Email address is removed for privacy ***:\Calendar
Check to see if the default room calendar permission is set to “Reviewer”. If yes, all users who are not granted a special room calendar permission can see the subject line and the user who booked the meeting. This is an expected behavior.

If you don’t want them to see the subject line and the user who booked the meeting, you can run the following PowerShell command to set the default room calendar permission to “AvailabilityOnly”. After that, the users who are not granted a special permission of the room mailbox calendar can only see the calendar event as “Free/Busy”.
1.    Connect to Exchange Online PowerShell.
2.    Run the command:
Set-MailboxFolderPermission -Identity *** Email address is removed for privacy ***:\Calendar -user default -AccessRights "AvailabilityOnly"

jay freedman say:

The 'Background Saves' option is a red herring. What that does, when checked, is allow you to continue editing the document while Word saves the file 'in the background'; when it's unchecked, Word stops allowing editing until the file is completely saved. This would make a difference only if you routinely edit and save large documents that would take a long time to save.

There are a couple of possible reasons for the behavior you see. The most likely is that there's an add-in that has substituted the Save As command for the Save command. To see whether that's true, start Word while holding down the Ctrl key, and answer Yes when prompted whether to start in Safe Mode. That blocks the loading of add-ins (among other things). If the all the variants of the Save command then work correctly, try to track down which add-in is responsible. Remove or disable each add-in, one at a time, restart Word, and test the Save command. If removing a specific add-in cures the problem, check its source for an update or decide to leave it uninstalled.

Another possibility is that something in the Office installation and configuration (including its registry entries) is awry. Try repairing Office.
