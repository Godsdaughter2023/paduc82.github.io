jay freedman say:
The Help topic is misleading and incomplete.

First, the good news: Continue doing what you did in Word 2003, telling the Print function to print section 1 by entering
Now, to clear up the misconceptions:
  • The envelope is a separate section. The first page of the document is the start of section 2, and there is a section break between the envelope and the document, just as there was in previous versions.
  • There are two different items you can display on the status bar, the "formatted page number" and the "page number". To see both of these, right-click a blank spot on the status bar and check both items in the menu.
  • The formatted page number shows the number that would be displayed by a {PAGE} field, while the "page number" shows the physical count of pages from the start of the document and the total number of pages. In a document that has an envelope added to it, the envelope has a formatted page number of 0 and the first page of the document has a formatted page number of 1; but the "page numbers" say Page 1 of 2 and Page 2 of 2. The Pages box on the Print dialog goes by the formatted page number, not the physical page number (which is where the Help topic went wrong).

daniel g. grau say:
Fonts are system-wide and are available to all applications. This means that you cannot force Word (or any other application for that matter) to display only a subset of the fonts, except by using a third-party application like Suitcase or other font managers, or by globally enabling and disabling fonts with Font Book (but again, the latter will have a system-wide effect).

Whenever Word updates it's font menu, it's because it thinks the list of available fonts has changed. If this is not the case (i.e., your font list is stable), the erroneous behavior could be caused by duplicate fonts. Try this simple font-weeding procedure from John McGhie:

• http://word.mvps.org/mac/fontweeding.html

However, since Microsoft-compatible fonts are all bunched up in a separate folder, Font Book and the various font menus in the applications let you see what these fonts are. In Word itself, they appear under Font > Font Collections > Windows Office Compatible.

bob jones aka: cybertaz say:
OS X enables fonts to be installed in a variety of locations. Which Fonts folder they are installed into determines which Users/Programs have access to them. Availability is on a hierarchical basis, so the higher up in the hierarchy the more universally available the fonts are. This Apple article explains the structure:

http://support.apple.com/kb/HT2435

AFAIK, there really isn't any way to restrict the fonts available to Word other than to:

1- Run the program in a separate User Account which has no other fonts installed in the "Local"* Fonts folder &

2- Use a separate User Account for your other programs with fonts installed in the "User" Fonts folder for that account.

* If the Microsoft fonts are installed in the "local" folder they will be available to all  User Accounts, if installed in the "User" folder they will be available to only that user. "System" fonts will still be available to all User Accounts.

The delay in launching of Word isn't due to the number of fonts, though. The Office fonts cache 'remembers' the fonts that were available when an Office app was last used. On launch the caches is compared to the fonts that are then available & the cache is updated accordingly. If there is no difference the launch is virtually instantaneous. However, if changes have taken place the cache has to be updated & that's what causes the delay.

You can also speed things up a little if you turn off WYSIWYG display of fonts in Word Preferences. It may also be [even more] helpful to run Font Book or a comparable utility to resolve duplicate fonts. Restarting your Mac several times weekly may also speed things up a bit.
