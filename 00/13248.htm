mel77084 say:

"Every time Windows 10 Insider Preview updates ..." Do you mean updates, such as definition updates for Windows Defender, or do you mean upgrades to new builds?

Most updates won't change your system restore configuration, which is usually set at 2-5% of partition size.

On the other hand, upgrading to a new build should delete the old restore points and set the the configuration to off, so that you'll always need to reconfigure the OS partition to on. It usually leaves the size setting as it was, but sometimes it will zero it.

Then, as restore points are automatically or manually added, when more space is needed, it doesn't increase the size that you have set, but pushes the oldest restore point out to make room for the new one.

The file history, if turned on, behaves more like your description, because it makes a copy of every file that you open, so in a couple of weeks, it can grow to several GB, especially if you're dealing with ISO files, and such.

If it's activated, the user can select a location for the files to be saved, and it might or might not be possible to set it to save the files in with the restore points. (I don't know that, because I've never wanted to use it). But, if that is possible, and you happen to have it set that way, that might allow it to continually increase your restore point size setting.

If that is the case, the setting will be carried forward into the next new build upgrade. If you were to do a clean install the setting would be at default, again, unless you choose to change it.

jllatham say:

Try it this way:

 

Sub pastevalues()
lastrow = Range("BZ6051").End(xlDown).Row
Application.ScreenUpdating = False
For i = 1 To lastrow
If Cells(i, 3) <> "Boat" Then
  Cells(i, 78).Formula = Cells(i, 78).Value
End If
Next ' actually faster without the 'i'
End Sub

 

To explain:  slowest thing happening is trying to update the presentation on the screen, so

Application.ScreenUpdating=False

stops that from happening while changes are being made.  It is automatically reset when you exit the sub.

 

Instead of going through the .Select .Copy and .PasteSpecial processes, we just directly change the formula in the cell to the value it was calculating.

 

The VBA engine actually keeps up with For...Next loop pointers more efficiently if you do not include the counter reference in the Next part.  Now that makes the code a little hard to read at times, especially if you have lots of For...Next loops, so I usually add a comment with the Next statement to tell which For statement it matches, like:

For i = 1 to lastrow

   ...do stuff

Next ' end i loop

 

By the way, if there's a chance someone has in any way mistyped "Boat", then this test will help make sure you catch all options of it:

If UCase(Trim(Cells(i, 3))) <> "BOAT" Then

This removes any accidentally typed in spaces before or after the word and converts it to all uppercase for testing only, so if they typed " boat " instead of "Boat", it recognizes it properly.

 

 

Hope this helps things move along better for you.

New Code below (including the the UDF) to find the last used cell. It simply uses Find and works backwards because of the SearchDirection:=xlPrevious. By default the first cell of the range to search is the first cell on the worksheet but Find never finds the default first cell of the range; it always finds the next so the next cell after the default first (A1) is the last cell on the worksheet and it works backwards from there to the last used cell because it is searching for a wildcard (any value).

 

Sub PasteValues()
    Dim ws As Worksheet
    Dim rngSelected As Range
    Dim lngColumn As Long
    Dim rngToProcess As Range
    Dim rngArea As Range
    Dim rngLastCell As Range
   
    Set ws = Worksheets("Sheet1")   'Edit "Sheet1" to your sheet name
    ws.Select
   
    On Error Resume Next
    Set rngSelected = Application.InputBox _
            (Prompt:="Select any cell in the column to be processed.", Type:=8)
    On Error GoTo 0
   
    If rngSelected Is Nothing Then
        MsgBox "No range selected. Processing terminated."
        Exit Sub
    End If
   
    lngColumn = rngSelected.Column
   
   
   
    With ws
        .AutoFilterMode = False 'Ensures AutoFilter is turned off
       
        'Calls UDF to Find last used cell on worksheet
        Set rngLastCell = .Cells(LastRowOrCol(ws, True), LastRowOrCol(ws, False))
       
        'Apply AutoFilter from cell A1 to Last Used Cell
        .Range(.Cells(1, "A"), rngLastCell).AutoFilter
       
        .AutoFilter.Range.AutoFilter Field:=3, Criteria1:="<>Boat"
       
        With .AutoFilter.Range
            Set rngToProcess = .Columns(lngColumn) _
                                .Offset(1, 0) _
                                .Resize(.Rows.Count - 1, 1) _
                                .SpecialCells(xlCellTypeVisible)
                       
            Application.ScreenUpdating = False
            For Each rngArea In rngToProcess.Areas
                rngArea = rngArea.Value     'Same as Copy and Paste Values
            Next rngArea
        End With
        .AutoFilterMode = False 'Turn off AutoFilter
    End With
   
    Application.ScreenUpdating = True

End Sub


Function LastRowOrCol(ws As Worksheet, bolRowCol As Boolean, Optional rng As Range) As Long
    'Finds the last used row or column in a worksheet
    'First parameter is Worksheet
    'Second parameter is True for Last Row or False for last Column
   
    Dim lngRowCol As Long
    Dim rngToFind As Range
   
    If rng Is Nothing Then
        Set rng = ws.Cells
    End If
   
    If bolRowCol Then
        lngRowCol = xlByRows
    Else
        lngRowCol = xlByColumns
    End If
   
    With ws
        Set rngToFind = rng.Find(What:="*", _
                LookIn:=xlFormulas, _
                LookAt:=xlPart, _
                SearchOrder:=lngRowCol, _
                SearchDirection:=xlPrevious, _
                MatchCase:=False)
    End With
   
    If Not rngToFind Is Nothing Then
        If bolRowCol Then
            LastRowOrCol = rngToFind.Row
        Else
            LastRowOrCol = rngToFind.Column
        End If
    End If
   
End Function
