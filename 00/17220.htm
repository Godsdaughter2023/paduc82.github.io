jeeped say:
Select the cell or cells that you want to apply the date format to and tap Ctrl+1. When the Format Cells dialog opens, go to the Number tab and select Date from the left hand Category: list. Select 03/14/01 from the Type: list on the right and click OK in the lower right.

Edit: it has occurred to me that your regional setting may default the date formats available from mm/dd/yy to dd/mm/yy, If this is the case, select Custom from the left hand Category: list and manually type in mm/dd/yy to the Type: text box. Click OK to accept the new custom number format.

You can also set the system wide defaults for regional date, currency, etc. through the Control Panel's Region and Language applet.

    I extracted the files I needed from the latest Media Features Pack and manually registered them.

    Step 1:

    Download the lastest pack if you haven't already from here: https://www.microsoft.com/en-us/software-download/mediafeaturepack

    Step 2:

    Use 7zip to extract Windows_MediaFeaturePack_x64_1803.msu to a directory.

    Step 3:

    Extract microsoft-windows-mediafeaturepack-oob-package~31bf3856ad364e35~amd64~~.cab to another directory.

    Step 4:

    Search that directory for these files:

    evr.dll

    mfplat.dll

    mfreadwrite.dll

    Note: there are two different sized files for each.  Pick the biggest.

    Step 5:

    Copy those files to %windir%\System32 (usually c:\windows\system32)

    Step 6:

    Open an elevated command prompt by right clicking the Start button and select Command Prompt (Admin)).  It should open up into the directory where you pasted the files.

    Step 7:

    Type this without the quotes "regsvr32 evr.dll" and press enter.  It should say "DllRegisterServer in evr.dll succeeded.".

    The other files can't be registered so close the Command Prompt and enjoy your media features being back (or at least your ability to play Matroska videos from TPB lol)  If whatever application you're trying to run reports other files being missing, search for those and also copy them to System32.  If it still doesn't work, try registering them like you did with evr.dll.  I didn't need any other files (yet) so I don't know.

    When the actual Media Features Pack for build 17127 or newer comes out and you decide to install because this hack doesn't do everything you need, it might fail to install with this work around, so open another elevated Command Prompt in system32 and run "regsvr32 /u evr.dll"  When it reports "DllUnRegisterServer in evr.dll succeeded.", delete the files you copied into system32 and try the update again.  If Windows won't let you delete them because they're in use or something, then rename them appending .old to .dll (i.e. evr.dll.old) and reboot.  Once you're back up, delete them and try the update again.

    john w. vinson say:

    The Link Child Fields and Link Master Fields are not properties of frmMain, nor are they properties of the Source Object (the form object within a Subform control). They are propertiesof the subform control itself - the "box" containing the subform.

    In addition, if you have a two-field link you need to include the two fields (or controls), separated by a semicolon, in the single Link property. As written your code would totally ignor EmplID, replacing it with DOR.

    Try

    Me.sbfrmState.LinkChildFields = "EmplID;DOR"

    Me.sbfrmState.LinkMasterFields = "EmplID;TextDOR"
