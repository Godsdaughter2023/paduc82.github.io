Windows 10 Fall Creators Update MSVCR80.dll error - Microsoft Community

andwhynot say:

Windows 10 Fall Creators Update MSVCR80.dll error

Discovered the cause of the problem on my device and was able to resolve the problem.   I was moving to a different device.  IBM iNavigator was already installed and working until I copied my desktop objects to the device. Then the problem of iNavigator not starting with the MSVCR80 error returned.  I deleted all the copied desktop objects and iNavigator worked again.  So, I starting copying objects over a few at a time by type and discovered the cause by process of elimination

.   

Some time back a e newsletter published instructions for creating a link to the "mega" control panel in Windows, which I had done and had sitting on the desktop.  Something in the 1709 update changed desktop characteristics.  I noted this super control panel icon no longer had a title on it.  When I deleted the icon from the desktop (this required Admin rights to accomplish) iNavigator started to function normally again.    

So having this icon, which likely requires elevated right to use, somehow affected the desktop and the iNavigator application to cause the MSVCR80.dll error.   

If you have this problem, my suggestion is look at your desktop icons and shortcuts - looking for an icon missing a title.  Delete the icon and doing this may resolve the problem.  If you don't find this, copy all the desktop icons to a folder, delete all the desktop objects at the profile owner level (public icons should remain regardless) and then systematically copy the objects back by type until you find the object causing the problem.  Then delete that object.
