It is possible to kill Installagent.exe process using either powercfg -requestsoverride process installagent.exe command or task manager's details tab. However, because its execution is scheduled even if Windows Store automatic update is disabled, you should disable the scheduled task in Windows 10's Task Scheduler.

  1. run the Task Scheduler (local)
  2. expand Task Scheduler Library > Microsoft > Windows > WindowsUpdate
  3. double click Automatic App Update
  4. on the triggers tab, disable all available triggers by double clicking each of them and unchecking the "Enabled" check in the bottom.

spry chipper say:

Broken search in Windows 7 Libraries

I have a machine with windows 7 64 bit installed and i had issues with search in the libraries , especially the ones with Pictures and videos . My issue was that the search was not working within these libraries. I could not find out why. I tried many options,

  • Move the files away from the library and re import them
  • Reset the search index
  • Add new folders to the library and try the search again

All the options provided the same result, the search was not working within the libraries. But now I have found out the cause.

The search within the libraries totally stops working if the folders in a library are removed from the search index . In my case the folders from my video and picture library were removed by me from the index.

The strange fact is that the search stops functioning totally when the folders in the libraries are removed from the index. The search works prefectly in windows explorer, irrespective of the folder's inclusion in the index or not. But the same does not happen within libraries when the locations are removed from the index.

This i feel is a bug because i have also tried this in another machine with windows 7 32 bit. Please post if any of you have had the same issues as above.
