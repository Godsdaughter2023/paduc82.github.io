The system firmware has changed the processor's memory type range - Microsoft Community

emoxam say:

The system firmware has changed the processor's memory type range registers (MTRRs) across a sleep state transition (S4). This can result in reduced resume performance.

I have it at my windows logs from time to time. What does this mean ?

eliyas yakub [msft] say:

This message is intended to notify about system firmware (BIOS) problem. You should contact your system manufacturer (OEM) to get an updated firmware.

This is the response I received from our kernel engineer on this issue:

MTRR registers determine what ranges of DRAM may be cached.  We found some nasty problems where firmware wasn't setting them the same across S3/S4 versus cold boot.  We fixed this (class of firmware) problem in Win8 by saving and restoring the boot setting.  The error log message was intended to surface the problem to OEMs hoping future systems wouldn't have the issue.

-Eliyas

An issue has been identified in Windows Explorer: Certain actions with the Quick Jump in the Start Menu can cause unexpected behavior. The estimated severity is minor, and is not persistent on a restart.

No specific configuration or setup is required. Start Menu and Start Screen have the same behavior regarding this issue, and the steps to reproduce are the same for both Start types.

This issue can be reproduced by following the following steps:

1. Open the Start Menu (or Start Screen) by any method.

2. Click on the “All Apps” button.

3. Click on any letter header in the menu to open the quick select.

THEN (Pick one method)

4. Close the Start Menu (Or Start Screen) without selecting a letter.

5. Open the Start Menu (Or Start Screen) with any method.

6. Click on the “All Apps” button.

OR

4. Click the back button

5. Click on the “All Apps” button.

The expected result is for the Start Menu or Start Screen to not change.

Actual Result:

The actual result is that the all apps list on the Start Menu or Start Screen will be misplaced, and some items are cut off. This effect will persist until the workaround is applied.

Prevention:

Do not use the quick jump, or make a selection before closing or leaving the quick jump

Workaround:

Use the quick jump and make a selection, or restart Windows Explorer to clear the unintended behavior. Re-sizing the Start Menu vertically will revert the intended behavior to normal. (Applies to Start Menu only.)

Remarks:

Re-sizing the start menu vertically will reset the start menu back to its proper position, while re-sizing horizontally does not.

Sub GetPath()
    With Application.FileDialog(4) ' msoFileDialogFolderPicker

            ' Set the variable
