the shocking result I found was:

In other words the 12/31/2017 automatically switched to 31/12/2017 while the other two remained to be the same!

Why???

ken sheridan say: In Access date literals must be in US short date format (mm/dd/yyyy) or an otherwise internationally unambiguous format, such as the ISO standard for date notation of YYYY-MM-DD.  If a date literal is entered in UK format (dd/mm/yyyy), if it can be interpreted as a valid date in US format, e.g. 04/07/2017 (4th July in UK format) it will be accepted, but interpreted as 7th April 2017.  If it cannot be interpreted as a valid date in US format, as with 31/12/2017, it will be automatically corrected.

Personally I always use the ISO standard when entering a date literal.  This does not mean that, in the UK or elsewhere which uses the format dd/mm/yyyy, you need to enter dates into a control in a form for instance in this or US format of course.  It is entered in the local format as set in Windows control panel.  Similarly, with functions like the CDate function, its argument is entered as a string expression in the local format.

Outlook Mobile Reply All - Microsoft Community

jimmy isbill say:

Outlook Mobile Reply All

Im using Windows Mobile 6 with Office Mobile. When I "Reply All" to an email from multiple email addresses, it also includes my email address. Does anyone know how to turn this off? Or is this just a bug in Outlook Mobile?
jeffrey min say: Are you syncing with a Microsoft Exchange?  If so, this is a bug that was addressed in WM6 and E12 (Exchange 2007).  It sounds like you have WM6 but may be syncing against an E11 (2003) server.

The bug stems from the fact that Exchange server slightly tweaks your email address when you send mail.  For example if your email address is "*** Email address is removed for privacy ***", it may be tweaked to something like "*** Email address is removed for privacy ***". 

When you reply-all to a message that has been so stamped, the email client does not recognize the tweaked email address as being your own.

The fix for this involved both server and client side changes to expose a list of all the different "own" email addresses.

In short, there is no workaround except to upgrade to E12 server.

How to automatically maximize a form when opening DB or loading form - Microsoft Community

terry carroll say:

How to automatically maximize a form when opening DB or loading form

I have a very basic question - What form property or method should I use to maximize a form upon opening the form or database? 
hansv mvp say:

You'd use the On Open or On Load event of the form. You can create a macro with the Maximize action, and call this from the event, or create an event procedure in Visual Basic:

Open the form in design view.

Activate the Event tab of the Property Sheet.

Click in the On Open event.

Select [Event Procedure] from the dropdown list in this event.

Click the builder buttons (the three dots) to the right of the dropdown arrow.

Make the code look like this:

 

Private Sub Form_Open(Cancel As Integer)
    DoCmd.Maximize
End Sub

 

Switch back to Access.
