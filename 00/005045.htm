jim gordon mvp say:

Hi

Outlook 2011 used the concept of an Identity to store all your Outlook stuff. When you used File > Import in Outlook 2019, you pointed to an identity and imported it. Identities in 2011 were comprehensive and iteroperable. For example, if you set a reminder in one identity, regardless of which identity you had open at the time, the reminder would fire.

Outlook 2019 has a less robust, but analogous concept of a Profile, which replaced the Identity concept. Profiles are discreet. If you are using one profile, reminders and events from a different profile will not fire.

In both 2011 and 2019, your Time Machine backup will keep track of everything in case you need to restore. I have seen very few problems with Profiles compared with Identities, as the new Profile concept is much less complex and therefore less troublesome.

Ah, yes, that's completely different. The triangles and the disappearing/reappearing text (Microsoft refers to it as collapsing and expanding) is a standard feature of the heading styles (Heading 1, Heading 2, etc.). It functions very much like the collapse and expand controls in the Outline view.

This is built into Word, and has nothing to do with converting from PDF or other formats -- except that when you open a PDF file in recent versions of Word, it automatically applies some of those styles.

Strictly speaking, the triangles will appear next to any paragraph whose Paragraph formatting is set to an outline level other than Body Text:

The only way to get rid of them permanently is to define a custom style that looks like the equivalent built-in style but has an outline level of Body Text. Then apply that style to the paragraphs to replace the built-in style.

Since the triangles show up only when you mouse over a collapsible paragraph, it's probably better just to learn to ignore them. (There are many benefits to using the built-in styles.)

Creating Styles for Inline Headings and Cross-References - Microsoft Community

e.h.h. say:

Creating Styles for Inline Headings and Cross-References

I would like to know if it is possible to create styles specifically for inline headings that will be recognized as headings in a TOC? Also, can cross-references set to format with a specific style?

Thanks for the help!

jay freedman say:

You can use custom styles for inline (run-in) headings, but your life will be much easier if you use only built-in Heading N styles because the default TOC will pick them up. With a custom style or a TC field, you will have to change the options in the TOC dialog. Any heading (or other text) can be made run-in by using a style separator, as described at http://wordfaqs.ssbarnhill.com/RunInSidehead.htm.

There is nothing built into Word to apply a style automatically to a cross-reference. You can manually apply a character style to each cross-reference immediately after you create it. If the cross-references already exist throughout a document, you could use a Replace operation or a macro to apply a character style to all REF fields. (That may not be exactly what you want, as some REF fields may repeat the contents of ordinary bookmarks rather than the references to tables, figures, etc.)
