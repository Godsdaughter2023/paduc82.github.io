Backup Strategies for 2008/2009:

Question

When I backup directly to my CD I get an error.

When I backup to my USB drive/stick/memory I receive an error.

I cannot backup to the floppy A:\.

I get an error about files are missing after I restore my backup.

I cannot find my backup on Vista.

I cannot restore my backup file from 2008/2009.

I cannot e-mail my backup to my accountant.

I cannot SAVE AS the data back to the server.
 

Answer

The 2008/2009 file structure is quite a bit different from previous versions of Simply. The main data file is the ibdata1.data found in the SAJ folder. The MySQL database information is found in the SAJ folder under MySQL and simply folders. Because of the new file structure many types of backup will not work.

You will no longer be able to backup to the FLOPPY A:\ even though the option to backup to the A drive exists in the backup and restore menu.

You will no longer be able to SAVE AS or SAVE A COPY the data back to a server or a backup drive located across a LAN. It is still possible to SAVE AS on the local workstation not across a LAN. Update: This has been rectified for 2009, although if your file is quite large and/or have a slow network a time out error message may still generate

Depending on the size of your data you will know longer be able to e-mail a backup This is related to the size of the backup. Any many cases the data will twice the size of the previous versions.

You will no longer be able to directly backup to the USB/Memory/Flash drive.

Note: The following will address alternatives to the above issues.

Instead of backing up to the FLOPPY A:\ drive backup the data to the LOCAL C:\ drive and then copy the data to a USB/Memory/Flash drive or burn to a CD. There is to many files and folders to successfully backup to FLOPPY A:\ drives.

Instead of SAVE AS or SAVE A COPY of the data across the LAN, copy the data using MY COMPUTER or MY NET WORK PLACES. Make sure to copy the SAJ folder as well as the SAI file.

For USB/Memory/Flash drives copy the files or the backup from the hard drive to the external drives.

Do not backup to the SAJ folder. You will be backing up the BACKUP.

bruce hagen ~ mvp say:

Error Number: 0x800C0133 = a corrupt Inbox.

Move any mail you wish to save to a local folder you create. Then delete the problem Inbox as follows.

Tools | Options | Maintenance | Store Folder will reveal the location of your Outlook Express files. Write the location down and navigate to it in Windows Explorer or, copy and paste it into Start | Run.

In WinXP, Win2K & Win2K3 the OE user files (DBX and WAB) are by default marked as hidden. To view these files in Windows Explorer, you must enable Show Hidden Files and Folders under Start | Control Panel | Folder Options | View.

With OE closed, find the Inbox.dbx and delete it. A new one will be created automatically when you open OE.

General precautions for Outlook Express:

Do not archive mail in the Inbox or Sent Items. Create your own user defined folders and move the messages you wish to save to them. Empty Deleted Items folder daily. Although dbx files have a theoretical capacity of 2GB, I recommend about a 300MB max for less chance of corruption.

After you are done, follow up by compacting your folders manually while working *offline* and do it often.

Click on Outlook Express at the top of the folder tree so no folders are open. Then: File | Work Offline (or double click Working Online in the Status Bar). File | Folder | Compact all folders. Don't touch anything until the compacting is completed.

Turn off e-mail scanning in your anti-virus program. It is a redundant layer of protection that eats up CPUs and causes a multitude of problems such as time-outs and account setting changes. Your up-to-date A/V program will continue to protect you sufficiently. For more, see:
http://www.oehelp.com/OETips.aspx#3

What is a .bak file?

This is due to the OE update, (KB923694).  Now when you compact, a copy of your dbx files are sent to the Recycle Bin in the event that something should go wrong and messages, or entire folders, are lost when you are compacting.

You can empty the Recycle Bin any time you want and the bak files will go away until you compact again.
