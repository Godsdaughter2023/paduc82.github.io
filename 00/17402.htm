ken sheridan say: The upper case Sigma icon (a Greek S, not an E) groups creates a GROUP BY clause in the query.  In design view 'Group By'  will be shown in the 'Total' row of the design grid for each column.  This will cause one instance of each row to be returned by the query where the values in all columns are the same.   
Some types of columns cannot be grouped, however.  If your query is returning more than one instance of a row where the values are identical across all columns returned, then this might be the reason.  Examine the query in design view and see if it has something other than 'Group By' in the 'Total' row of the design grid.

Where all columns are capable of being grouped, a better way of eliminating duplicates is to use the SELECT DISTINCT predicate.  In query design view this is achieved by setting the Unique Values property of the query to True (Yes) in its properties sheet.  If any of the columns are of a type which cannot be grouped, however, this cannot be done.

Note that when a query is grouped or the SELECT DISTINCT predicate is used, this applies only to the columns returned by the query.  To return distinct rows where the values are the same over all columns in the tables, the SELECT DISTINCTROW predicate can be used.  This is achieved in query design view by setting the query's Unique Records property to True (Yes).  This is only appropriate where a JOIN of multiple tables can return duplicates across all columns, not with a single table, as any table which has rows duplicated across all columns would have no candidate key, and thus would not be a valid table.

As John points out, however you do it in a SELECT query, this only returns distinct values in the result table.  It does not 'eliminate', i.e. DELETE, any rows in the base table.  For that a DELETE query can be used.
 
BTW, the reason the upper case Sigma is used for the 'totals' icon is that it is the mathematical symbol for summation.

john w. vinson say:

You may want to talk to your instructor...

His suggestion will not actually eliminate any duplicates; the Sigma button changes the query to a Totals Query, which can be used to group the records by one or more fields and Sum, Count, Average, etc. the records within each group. You will need to use "Group By" as the aggregate operator in the query grid on all of the fields which define a duplicate.

If you want to actually REMOVE - permanently delete - duplicate records you will need other techniques. One would be to create a copy of your table, empty; use the Indexes tool to create a unique Index on the fields which identify duplicates; and run an Append query to copy the data into this new table. You'll get a warning message that "xxx records were not appended" - these are the duplicates.
