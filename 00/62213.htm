Sorry for the late reply. If you are able to reach the Windows desktop briefly, you should be able to perform these steps to stop the Blue Screens. I also suffered from this issue and these are the steps I undertook to resolve the problem:

When you reach the Windows desktop, immediately right click on the Windows taskbar and open Task Manager. You have a very limited time before another Blue Screen, so quickly kill any AMD services because their GPU drivers seem to be causing the issue. There are two AMD services that you have to kill, and one Catalyst Control Center to kill also. Be careful though because these might pop back up again or only one of the three might show up first before the rest do. Essentially, keep a constant eye on Task Manager. I had to try this several times because I was too slow, so I recommend to keep trying. You should also go to the Startup tab and disable Catalyst Control Center from launching at boot too for good measure.

Once you have those services killed, open Device Manager by searching for it in the Windows search bar/Cortana and manually update your AMD GPU to the latest driver by finding it under Display Adapters and right clicking the item in Device Manager and selecting Update Driver.

Since I changed the GPU driver, I haven't had another Blue Screen. This worked for me after lots of frustration, so I would give it a try. It seems that the driver Windows 10 gave my GPU was not the correct one.

I hope that it works for you because I know the frustration you're experiencing. If you cannot reach the Windows desktop, can't perform these steps, or need further clarification, I'd be happy to try to help.

trekdozer say:

How to Grant Permissions in Windows 7

1. Locate the file or folder in which you want to take ownership in Windows Explorer.

2. Right click on file or folder and select Properties from the Menu

3. Click on the Security tab, then click the Edit button. Click OK to confirm UAC elevation request if needed.

4. Select user/group from permission window or click add to add other user or group (you may need to do this for multiple users - ex: administrator and your own user account).

5. Now under the Permission section check the rights which you want for the user i.e check "Full Control" under the "Allow" column to assign full access rights control permissions to the user.

6. Click OK for changes to take effect, and click OK again to exit from the Properties window.

vanessa ouyang msft say:

Hi Art,

 

Thanks for your detailed information. You are right. If we run

Set-CsConferencingPolicy global -AllowExternalUsersToSaveContent $false, we can change the “Global” policy. However, we cannot run

Set-ConferencingPolicy BposAllModality -AllowExternalUsersToSaveContent $false, because to set or remove host tag instances is not allowed.

 

In this situation, to achieve your goal, I suggest you create a new policy for this setting, and run:

New-CsConferencingPolicy -Identity test123 -AllowExternalUsersToSaveContent $false.

And then grant the new policy you created to the user:

Grant-CsConferencingPolicy -identity "Ken" -PolicyName test123

 

Here is an article for your reference: Set Skype for Business Online conferencing policies for your organization.

 

Additionally, please let us know more details about how you check “conference policy”.

Vanessa
