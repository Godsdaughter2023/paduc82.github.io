karl timmermans [outlook mvp 2012-2018] say:
Re: the messages get deleted from the server if I delete them on this machine. It does not happen on the other machine with the POP3 settings.

Something that really needs to be understood and that is accounts configured as IMAP or Exchange sync email folders which means that what occurs on one side will occur on the other (i.e. delete from server, gets removed from Outlook and vice versa)

POP3 accounts do not sync anything - the <Inbox> is a one-way one-time (per machine) download from the server to the local data store. Whatever is done in Outlook is not reflected on the server. It should also be noted that everything else in a POP3 account is "local only" (other email folders, contacts, calendar etc).

Your only option is to manually configure the account as POP3 instead of using <auto-discover> (email address/password). To do that, you'll need to remove the account and re-add it back as a POP3 account by clicking on "Advanced Options" immediatelly underneath where you enter the email address and then check the box for "Let me setup my account manually"

For background reference regarding email account protocols

We're going to use custom data validation, and enter a formula to test if the input date is equal to or greater than the date in column A.

Ah! Word 2013 :(

It works fine in 2010.

What is the point of this exercise? Frankly text boxes in Word documents are best avoided where possible. They are essentially graphical elements and not in the text layer of the document. I try and avoid Word 2013, but it seems as though the graphics engine has changed ... again ... which screws up the positioning. Wouldn't a table work as well? It would be easier to program, it would be in the text range and would look essentially the same

Sub Graham2()
Dim oTable As Table
Dim oRng As Range
Dim i As Long
    ActiveDocument.Range.InsertParagraphAfter
    Set oRng = ActiveDocument.Range
    oRng.Collapse wdCollapseEnd
    Set oTable = ActiveDocument.Tables.Add(oRng, 2, 5)
    oTable.Style = "Table Grid"
    For i = 1 To oTable.Range.Cells.Count
        With oTable.Range.Cells(i)
            .Height = CentimetersToPoints(1.76)
            .Width = CentimetersToPoints(3.53)
            .Range.Text = i
        End With
    Next i
End Sub
