Sweet factory reset works!! :) Video is restored!! (I was actually doing a reset because i was getting the "certificate expired" error). Killed two birds with one stone.

 

I have modified step 2 below - this is because backing up ALL phone content means that when you restore it whatever it is that breaks the RealPlayer will be done again. So i had to do the whole process again, but this time only backing up the most important stuff. I've excluded Settings and Phone Memory. By doing this i can start reinstalling stuff one by one, by process of elimination to see what takes out the Real Player. Could be a random setting or it could be an app. If i find out i'll let you know.

 

Here's the condensed instructions:

 

  1. Connect via USB in PC Suite mode. Run Ovi Suite.
  2. In Ovi Suite click Tools (the spanner) and select Content Copier (it would've been too easy to call it phone backup!).  Because i'm not sure what settings are breaking the Real Player only backup Contacts, Calendar, Bookmarks and Notes. This means Settings and Phone Memory stuff won't be saved.Your choice though.
  3. Disconnect USB. Press power button once - Select Remove E: Memory Card
  4. Remove the SD Memory Card from side (NB: NOT THE SIM CARD) - I found using a penny helps to push it in to remove it.
  5. In the dialler type: *#7370#
  6. Enter your lock code: 12345 and press OK     (or if you've changed your lock code add that instead)
  7. Phone will restart - On reboot select your country, date, time as normal.
  8. Switch off phone
  9. Reinsert SD card
  10. Switch back on
  11. Connect via USB and restore backup from Ovi Suite.
I did notice that the working version of Real Player had 50% contrast as default in settings. The broken version was stuck on 0%.

thedbguy say:

Hi. I think the main difference between Access and Excel is there's a concept about "data types" in Access, where I don't think there is such a thing in Excel. In Access, what you store in the table is based on a data type. What you display is merely a "formatting" of the data.

For example, in Access, dates and times are stored as a double-precision floating point number. What gets displayed is dependent on the machine's Regional Settings. So, if you switch the Regional Settings on your computer, you will notice a different format showing for the dates in the table.

There are other ways to change how the dates are displayed without messing with Regional Settings. You can select an item from the Format property or enter a custom format. You can also use the Format() function in queries to change how the date is displayed.

Bottom line, what you "see" displayes has no bearing on how Access "sees" the data in the table.

Just my 2 cents...
