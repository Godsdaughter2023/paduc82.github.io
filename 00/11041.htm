Presumably, you want the following syntax:

=IF(F5-F1 < F1*0.05, "Excellent", IF(F5-F1 < F1*0.10, "Good", "Poor"))

=IF(ABS(F5-F1) < F1*0.05, "Excellent", IF(ABS(F5-F1) < F1*0.10, "Good", "Poor"))

I'm replying to your thread because it most closely matched my experience... we bought a fleet of refurbished Lenovo Thinkpad T430 laptops.

After installing the Windows 10 Fall Creators Update (version 1709) and updating drivers on the Lenovo support site, we starting getting the same Explorer crash loop that you are getting.

After trying all the crap you can find on the internet, I checked the Event Viewer and came up with the same error as someone else:

Faulting application name: explorer.exe, version: 10.0.16299.125, time stamp: 0xfeba44fb
Faulting module name: MbaeApi.dll, version: 10.0.16299.15, time stamp: 0xecade6c7

Windows Explorer would die and restart, non-stop every 10 seconds.  Just typing was a pain.  After some digging, I found that the module MbaeApi.dll is for Mobile Broadband Account Experience API.

Using this, I determined that some of my laptops had the Thinkpad WAN module installed and some didn't, which is why I had only a few machines with this problem.

To fix it, even while it's spazing out, do the following:

1. hit Control-Alt-Delete

2. click to run the Task Manager

3. in Task Manager, click File, Run new task

4. check the box "Create this task with Administrator Privileges"

5. paste in "devmgmt.msc" (which is the Device Manager) and click OK to run it

6. you'll probably have to close Task Manager so you can get to the DM window

7. in DM, roll down to and expand the Network Adapters section

8. find and right click on anything similar to "Sierra Wireless" or "Gobi 3000" or "Gobi 4000"

9. choose Uninstall Device and BE SURE to click the box Delete the driver for this device

10. click Uninstall to do it!

Wackiness should immediately stop... it did for me anyways.
