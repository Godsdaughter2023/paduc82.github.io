graham mayor say:

You would need to set a reference to the Word object library in the vba editor tools references and start a Word application from your macro in order to run a Word macro in it - something like

Sub OpenWord()
    Dim wdApp As Word.Application
    Dim wdDoc As Word.Document
    Dim WordWasNotRunning As Boolean

    WordWasNotRunning = False
    On Error Resume Next
    Set wdApp = GetObject(, "Word.Application")
    If Err Then
        Set wdApp = CreateObject("Word.Application")
        WordWasNotRunning = True
    End If
    Set wdDoc = wdApp.Documents.Add

    wdApp.Visible = True
    wdApp.Activate
   ' do whatever else you want here e.g.
    wdApp.Run ("macroname")

    ' and when you're finished
    If WordWasNotRunning = True Then
        wdApp.Quit
    End If
    Set wdDoc = Nothing
    Set wdApp = Nothing
End Sub

thomas higgins say:

Internet Explorer: can browse FTP site, cannot download binary files

This is a more complex issue than it appears at first so please read the whole thing before responding.

We run Internet Explorer 8 in our enterprise environment, over 1000 users use it, we will not push out an alternative browser.

We recently ran into an issue when some of our users were accessing a specific ftp site, when the users tried to access it, they recieved "Internet Explorer cannot display this webpage". After much research I found this kb article: http://support.microsoft.com/kb/941896/en-gb . The ftp site that had the issue ran linux, and once you have logged on to the site, it tries to place you into the users home directory. IE8 doesnt support this and tries to load the root directory "/". Because the user does not have access to this directory, IE displays the "cannot display the webpage" message.

One of the workarounds in the KB article describes using Windows Explorer instead, well that would be fine if Windows Explorer understood symbolic links, but it doesnt. So when using Windows Explorer, it reports 0 files in the directory. 

Because of this I tried the other workaround, which was to apply the registry change (although the KB article says it only applies to IE7, it also applies to IE8 on Windows 7). This allowed IE8 to correctly connect to the ftp site (hooray).  

If you would like a more visual look at what I did, please refer to my blog here: http://ausnerd.blogspot.com/2010/04/windows-ftp-issues-with-nix-servers.html

I thought everything was dandy, but here is the kicker: Users can't download files binary now. Text files are fine, they load correctly in Internet Explorer, but Binary files, the download window appears for 30-40seconds or so and then displays:

Unable to download <file> from <server>
Unable to open this Internet site. The requested site is either unavailable or cannot be found. Please try again later.

I downloaded and installed wireshark to see what was happening in the background. It seems that the file is actually downloaded. But once the file is completed I then recieve the error.

When I reverted the registry change, binary ftp files download correctly, but *nix servers that redirect to the user home directory dont load correctly! So I'm out of luck here guys, there is no good GUI FTP solution built into Windows. Firefox works fine (but we dont want to support it). We could give them an FTP client (but that would confuse the users).

What can I do?

Hi John,

Thank you for posting your query on Microsoft Community.

Let's try creating a rule to delete all the emails from the sender.

1. Login to https://mail.live.com/ through the web browser.

2. Click on the Gear (Settings) icon on the upper right corner and select Manage Rules. Try setting up the following rule.

  • Under When an email matches dropdown menu, select Sender Contains 
  • Type "mediamarkt.nl" or just type "mediamarkt"
  •  Under Do the following drop down, select Delete

3. Now, click on Create Rule.

If you have been upgraded to Outlook Mail (Preview) interface, then follow the steps below.

1. Click on the Gear (Settings) icon on the upper right corner and then click on Options.

2. Expand Mail and then click on Inbox and Sweep rules.

3. Select "+" icon and try setting up the following rule.

  • Under Name type the name you wish to keep for the rule.
  • Click on the dropdown menu and select It includes these words > in the sender's address
  • Type "mediamarkt.nl" or just type "mediamarkt" (without quotes)
  • Under Do the following drop down menu select Move copy or Delete and then click on Delete the message.

4. Finally, click on Ok.

Hope this helps. Please respond if you have further related queries.
