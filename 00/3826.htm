greg maxey say:

Assuming you have:

Name.......................Jack

Addresss.................123 Maple Street, Anytown  USA

 

Where "Name" and "Address" each start a new paragraph and the "....." is repeated periods, you can use the following VBA procedure to replace the periods with a dot leader tab set at

1 inch:

Sub ScratchMacro()
'A basic Word macro coded by Greg Maxey
Dim oRng As Word.Range
Dim oPar As Word.Paragraph
For Each oPar In ActiveDocument.Range.Paragraphs
  Set oRng = oPar.Range
  With oRng.Find
    .Text = "<*.{1,}"
    .MatchWildcards = True
    If .Execute Then
      oRng.MoveStart wdWord, 1
      oRng.Text = vbTab
    End If
  End With
  With oPar.TabStops(1)
    .Position = InchesToPoints(1)
    .Leader = wdTabLeaderDots
  End With
Next
End Sub

See: http://gregmaxey.mvps.org/word_tip_pages/installing_employing_macros.html for instructions to employ the VBA code provided above.

john p spencer say:

As a guess FullName is a text field.  If so, you need to add text delimiters to the criteria expression.

Me.MgrID = DLookup("LevelID", "tblManagement", "[FullName]=""" & Me.MgrName & """ AND [Level]='" & Me.Label37.Caption & "'")

Or you can use single quotes (apostrophe) if none of your names contains an apostrophe (O'Bryan).

Me.MgrID = DLookup("LevelID", "tblManagement", "[FullName]='" & Me.MgrName & "' AND [Level]='" & Me.Label37.Caption & "'")

Or you can use Chr(34) to do so

Me.MgrID = DLookup("LevelID", "tblManagement", "[FullName]=" & Chr(34) &  Me.MgrName & Chr(34) & " AND [Level]='" & Me.Label37.Caption & "'")

dave peterson say:

I have a VBS file that I keep in a secure folder with a shortcut to it on my
desktop.

I didn't want several versions (one for each application), so I have one file
that unhides any running office application (well, MSWord, Excel, PPT, and
Access).

This is the .vbs script:

======
dim myXL
On Error Resume Next
Set myXL = GetObject(, "Excel.Application")
If Err.Number = 429 Then
    msgbox "Excel is not running"
else
    myxl.visible = true
end If
On Error GoTo 0
Set myxl = nothing

dim myWord
On Error Resume Next
Set myWord = GetObject(, "Word.Application")
If Err.Number = 429 Then
    msgbox "Word is not running"
else
    myWord.visible = true
end If
On Error GoTo 0
Set myWord = nothing

dim myOutlook
On Error Resume Next
Set myOutlook = GetObject(, "outlook.Application")
If Err.Number = 429 Then
    msgbox "Outlook is not running"
else
    myOutlook.visible = true
end If
On Error GoTo 0
Set myOutlook = nothing

dim myPPT
On Error Resume Next
Set myPPT = GetObject(, "Powerpoint.Application")
If Err.Number = 429 Then
    msgbox "Power Point is not running"
else
    myPPT.visible = true
end If
On Error GoTo 0
Set myPPT = nothing

dim myAcc
On Error Resume Next
Set myPPT = GetObject(, "Access.Application")
If Err.Number = 429 Then
    msgbox "Access is not running"
else
    myAcc.visible = true
end If
On Error GoTo 0
Set myAcc = nothing
===========

Copy the stuff between the "========" lines and paste into NotePad.  Then save
it as a nicely named .vbs file (UnhideApps.vbs ???).

The bad news is that if you have several hidden instances of any of these
programs, you may have to run it several times.  Then close the visible
instance, rerun, close, rerun.  Until all are closed.

You can pop up the task manager (ctrl-shift-Escape is one way), look at the
processes tab to determine how many winword.exe's are running (visible or
hidden).

Again, I wouldn't make this part of the user experience.  But it's good while
developing.
