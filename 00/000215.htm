Page Down should scroll down the equivalent of a screen (and, similarly, Page Up moves up a screen). To move from one page to the next, use Ctrl+Page Down.

Note that the actual action excecuted by pressing Ctrl+Page Down (or Ctrl+Page Up) depends on the context. For example, if you are performing a search, Ctrl+Page Down would find next.

scott mcdaniel say:

I assume it fails due to the user context in which those procedures run. From what I've read, they do not run with "unprivileged" permissions, which means they'd work fine to work with HKLM, but not HKCU.

Generally speaking, HKLM should be used to stored values related to the program itself, while HKCU should be used to store user-specific settings. As an example, when installing the program I might set the InstallRoot value in HKLM (since that would never change, supposedly), while I might set the UserTheme value in HKCU, since each user may have their own Theme preference. For the most part, you should assume your user does NOT have HKLM rights (since most of them don't), and instead configure your program to use the HKCU hive, or some other method, to store these sorts of settings.

If you need to write to HKLM when deploying updates, then use a Windows installer, and require the user to run the install with elevated permissions (i.e. while logged in as a member of the Admins group at some level).

A lot of problems with Office 2013 are caused by third party (non-Microsoft) applications.  This is my (expanding) list of things to try when Office 2013\365 won’t start or crashes:

http://answers.microsoft.com/en-us/office/forum/office_2013_release-word/microsoft-word-2013-has-stopped-working/67b81016-849f-481a-bce5-bf04797ac03b

Do you have Abbyy Finereader installed on the PC? A number of people have reported that removing it overcomes problems with 2013

Do you have “Fun Tools” add-in? A number of people have reported that removing it overcomes problems with 2013

Do you have Toshiba Book Reader installed on the PC?  It has been reported that uninstalling that brings Office back to working status.

Do you have the Acrobat PDFMaker add-in.  It has been reported that uninstalling that overcomes an issue with a message appearing that states "The document contains macros. Macro support for this application is disabled. Features requiring VBA are not available. Would you like to open this document as a read only?"

http://answers.microsoft.com/en-us/office/forum/office_2013_release-word/microsoft-word-has-stopped-working-when-i-close/8375b5b6-4f7e-46ed-a02b-90152ecad287#LastReply
