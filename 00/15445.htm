Importing Contacts with Multiple Categories Into Microsoft Outlook

One of the simplest ways to import a list of Contacts into Outlook is by putting the contacts and their associated information into an Excel spreadsheet. This allows for ease of editing contact information and setting up proper formatting to prepare for import. Be sure to match column headings to the corresponding field names in Outlook. This will save time and reduce errors when importing the file into Outlook contacts. For example, if the heading for work phone numbers in your contac list is "Work", change it to "Business Phone" to ensure that it will directly poplulate the "Business Phone" field in the Outlook contact. Otherwise an additional field mapping step will have to be completed during the import to Outlook contacts process.

 

And to address the topic of importing multiple Categories: be sure to have a column on your spreadsheet named "Categories" and then put each category in the cell corresponding to the contact and be sure to separate each category name with a semicolon. Respectfully disagree with one premise which is to skip the manual mapping process and depend on Outlook correctly assigning fields. History and experience has proven that far more problems occur by skipping this minor intermediate step then any time saved (takes but one typo for a field not to be auto-mapped.


People are going to be far better off checking that everything is mapped correctly (whether auto-mapping is used or not) then to blindly complete an import and then figure out what if any issues have occurred.


For those living in countries where the list separator character is NOT a comma (i.e. semi-colon) - also important to remember that saving an Excel worksheet to CSV in these cases will result in all fields being bunched into the first field since Outlook expects a comma as the field separator in a CSV file whereas Excel will generate the CSV file using the list separator character defined in the Regional Settings.
