ken sheridan say: I'd have thought a join to a subquery would be a pretty efficient solution:

SELECT Q1.Company, Q1.Industry, Q1.SumOfSales
FROM qrySumSales AS Q1 INNER JOIN
    (SELECT Company, MAX(SumOfSales) AS MaxSumOfSales
      FROM qrySumSales
      GROUP BY Company) AS Q2
ON Q1.Company = Q2.Company
AND Q1.SumOfSales = Q2.MaxSumOfSales;

This would also return ties, which to my mind would be a more valid result than the arbitrary selection of an industry in the event of a tie.

vanderghast say:

You can "easily" get all of the ties, if you wish so, from the total query too. Remove the LAST aggregate in the SELECT clause, and add a.Industry in the GROUP BY clause.

 

SELECT a.company, a.industry, a.sumOfSales
FROM table AS a INNER JOIN table AS b
                             ON a.company = b.company
GROUP BY a.company, a.industry, a.sumOfSales
HAVING a.sumOfSales = MAX(b.sumOfSales)

 

 

It always amaze me to see how a subquery can be translated into a join.
