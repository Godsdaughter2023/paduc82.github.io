¡firedog say:

Using message rules, I want to sort and keep them in their individual folders. Is this a good idea?

 

If that's what you want, then yes, it's a good idea. The only drawback I can see is that if you can't see the folder in the folder pane, you can't see if there are any unread messages in it. The count next to the account name only shows the number of unread messages in the inbox.

Or should I move them to the Storage folders?

 

The only material difference between folders under a POP account and folders in Storage folders is that if you Remove the account, all its folders and the messages they contain are permanently deleted. A warning message appears to remind you of this if ever you're tempted to remove a POP account. 

can message rules move them from the inbox folders to the storage folders?

 

Message rules act on incoming messages. If the idea is to move them to their proper folders, then you won't ever see them in the inbox.

When you create a rule, you do have the option to Apply now so that messages already in the inbox can be moved to their proper folder by the rule. However, this function is buggy and may not work as intended.

can they go directly to storage in the first place?


Yes, if that's what you want. The problem with that is that Storage folders remain stubbornly at the bottom of the folder pane. If you have several accounts, Storage folders may be out of view. I have lots of accounts, so I use folders under the topmost account for my filing system. Every so often, I move older messages to archive folders under Storage folders. This keeps scrolling to a minimum.

Read more about rules, including a description of the Apply now bug, here: Use Message Rules

frederik long say:

Psexec.exe is a tool to execute commands on a remote machine. It gives you no benefit when you run it locally.

Deleting keys from ControlSet001 has no effect. You should delete the keys from CurrentControlSet.

You batch file contains a number of superfluous lines that only confuse the issue. Here is what you really need:

@ECHO OFF
setacl.exe -on "HKEY_LOCAL_MACHINE\SYSTEM\CurrnetControlSet\Control\Pastel" -ot reg -actn setowner -ownr "n:Administrators"
REG DELETE HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Pastel /f
Pause

Since setacl.exe is not a native Windows command you need to check its requirements and syntax from the site where you got it from. The native command would be reg.exe. Either way you will need to execute the command under elevated privileges. You must also create a System Restore Point before going ahead.

frederik long say:

Try this:1. Set your BIOS so that it uses the DVD drive as its primary boot device.
2. Boot the machine with your Windows 7 Repair CD or with your Windows 7 installation DVD.
3. Press a key when the message indicating "Press any key to boot from CD or DVD …" appears.
4. Select a language, a time and currency, and a keyboard or input method, and then click Next.
5. Click Repair your computer.
6. In the System Recovery Options dialog box, choose the drive of your Windows installation and click Next.
7. In the System Recovery Options dialog box, click Startup Repair.

I have seen a number of posts in which people reported that they were unable to boot from the CD/DVD drive. Here are the most common reasons for such failures:
- The poster overlooked the prompt to press a key in order to boot from the CD/DVD drive.
- The boot CD/DVD was not a boot disk or was damaged. Using it on another machine confirmed this.
- Some BIOS setting was incorrect. Booting with some other boot CD/DVD (e.g. from here: http://pogostick.net/~pnh/ntpasswd/) confirmed this.
- The CD/DVD drive was damaged or unable to boot from a boot CD or DVD.
- The machine had a hardware problem (e.g. a motherboard problem) that prevented it from booting in any mode.

Here are the few things we suggest you do:

 
  • Clean the connector on your power supply. Unplug the power supply from the wall. Clean the pins on the connector with a pencil eraser. Rub the eraser back and forth across the pins. Wet a cotton swab in rubbing alcohol. Clean the pins on the long, narrow part of the charger that you plug into your Surface. Rub the cotton swab back and forth across the pins. After you are done cleaning the pins, make sure all the pins on the connector are dry, plug the power supply into the wall, and then plug the connector back into your Surface.
  • Remove and reinstall battery driver. From Device Manager, expand the Batteries category. Double-tap or double-click Microsoft Surface ACPI-Compliant Control Method Battery, select the Driver tab, and select Uninstall > OK. Important! Please don't put a check mark on the box. Tip! Restart the Surface after uninstalling the drivers.

To optimize the battery life of your Surface:

  • Install the latest update on your Surface: Go to taskbar, type and select Check for updates.
  • Turn on Battery Saver: Go to taskbar, type and choose Battery Saver. Select Turn battery saver on automatically if my battery falls bellow and move the slider further to the right for longer battery life. This might slow down performance of some apps since they won't be running on the background all the time. To turn on Battery Saver automatically, use the switch below and enabled it. Set the other options, to get the result that you want.
  • Run the Power Troubleshooter: In the taskbar, search and select Troubleshoot in System Settings and choose Power then Run the Troubleshooter and follow the steps to see more ways to save battery power.
