It appears that you did the VBA calculation with variables whose type is Single.  To demonstrate:

Function vbaDiv(x As Single, y As Single) As Single
vbaDiv = x / y
End Function

=vbaDiv(1465000,173) returns 8468.2080078125 when formatted to 11 decimal places.

Instead, use type Double.

wong robertmaris wrote:
1.HOW TO COMPARE THIS CALCULATE SO RESULT IN VISUAL BASIC AS SAME AS EXCEL WORKSHEET ( WHAT CAN I FILL IN MY MODULE AT VISUAL BASIC )

2.iF i WANT ROTATE THIS CALCULATE SO AT EXCEL WORKSHEET CALCULATE AS SAME AS VISUAL BASIC CALCULATE, WHAT CAN I DO AT SETTINGS OPTION IN EXCEL?

Nevertheless, differences can and do arise between Excel and VBA for the same type-Double calculations.

This is because Excel converts pairwise arithmetic results to 64-bit binary floating-point, whereas VBA sometimes retains the intermediate 80-bit binary floating-point form used internally in Intel-compatible CPUs.

In VBA, the surest way to ensure the same result is to mimic Excel and store each pairwise arithmetic operation into a type Double variable.  But that is painful.

Generally, the best way to minimize differences is to round "formulas" (the right-hand side of a VBA assignment statement) to a reasonable degree of precision, when that is your intent.  For example, for monetary calculations, it is often reasonable to round to 2 decimal places.

[EDIT] Caveat:  Use WorksheetFunction.Round instead of VBA Round.  VBA uses "banker's rounding" (round half to even), which is different from Excel rounding (round half up).

However, differences might still arise, for example if your "formula" involves truncation or rounding up.

robert aldwinckle on forums say:

 what file extensions could be seen in Internet Explorer

One way is to open a cmd window and enter:


ftype  |  find  /i  "iexplore"


Then to see if  .rtf  extensions should be opened in IE you could enter:


assoc  .rtf


and see if the file type shown is one of the ones that you found with the first command pipeline.


Since you have Office installed you probably have Word and will probably see that .rtf is associated with that program.


However, if the filetype being downloaded is being "sniffed" as ZIP that means it probably isn't really .rtf but  is one of the new compressed formats.  You should be able to at least see the file in the Downloads, e.g. by pressing Ctrl-j.   Also, you can try modifying the effect of MIME sniffing by toggling that option in Security settings, Miscellaneous section.
