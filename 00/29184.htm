a. user say:

Your system is performing a Microsoft  Diagnostics Performance check during startup which is why startup has slowed to a crawl. And it seems to be running multiple times.  I don't know why this is happening - nothing in what you posted gives a cause for the check.  Are there other errors that precede these?  If so, could you post them as well?

Go to start / Search box and type in msconfig and enter.  Double click on the program icon that appears.  Click on the startup tab and check to see if Microsoft Diagnostics is listed as a startup item - it so, uncheck the box. (I don't thiink this is the problem, but it's best to check to be sure.)

Try a System Restore to a point BEFORE this problem started occurring.  Here's how: http://www.howtogeek.com/howto/windows-vista/using-windows-vista-system-restore/.  Check the box to show more than 5 days of restore points.  If this doesn't work, try an earlier point or two.

If this doesn't work, do a Startup Repair by booting to the genuine Windows Vista Installation Disk or your Recovery Disk. Here's the procedure: http://www.bleepingcomputer.com/tutorials/tutorial148.html.  You may need to change the BIOS to make the CD-Drive first in the boot sequence.  To do that boot the PC and await the screen where it shows the F key to enter boot menu or boot setup.  Push it quickly. Make the change, saver your work, and exit.  Insert the CD in the drive and reboot.  When prompted, press any key to boot to the PC.

If you do not have either disk you can make a Recovery Disk at http://neosmart.net/blog/2008/windows-vista-recovery-disc-download/ using burning software like: http://www.snapfiles.com/get/active-isoburner.html and of course a blank CD.  This Recovery File is every bit as good as that which comes with Vista except it can't do a reinstall.  You can do a System Restore or a Startup Repair or even use the Command Prompt.  Booting to the CD may be your only option to keep the PC "awake" long enough to do the procedures - it that even works.

If this doesn't work go to Sart / All Aprograms / Accessories / Command prompt and right click on command prompt and click run as Administrator.

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no errors it could not repair (if such errors occur, then post back here).

Next, type in chkdsk /f /r and enter and let it run.  It will scan and try to fix bad sectors on your hard drive.

If  that doesn't work, post back and we'll work on some other solutions to the problem -we still have quite a few options we can try - but I think the first recovery option (System Restore) is likely to work the best if you know ehen this started and can find a restore point prior to that time.

Good luck and I hope this helps - but please post back no matter the case.  We're here to help you as best we can.

Please disconnect all the external devices, except Keyboard and Mouse and then check. Also, check if the Keyboard works on other apps on the computer.

Try to run Hardware Troubleshooter. It checks for common issues and makes sure that any new device or hardware attached to your computer was installed correctly.

a. Press Windows Logo Key+X then click on Search screen.

e. Click Next and follow the on-screen instructions.

  4. To re-install, either you can restart the system or install the driver from the manufacturer’s website (recommended as they will have updated drivers).

rmdeq say:

I sorted it out:

 

PROBLEM 1:

AudMig: No audio endpoint migration settings found 0x2

 

SOLUTION 1:

Assign a drive letter to the 100MB System Reserved paritition used by Bitlocker, and then run Windows 8 Install. After Windows 8 is running, I don't think it needs a drive letter.

 

PROBLEM 2:

"autochk program not found, skipping autocheck"  flashes on screen at boot, followed by BSOD

 

SOLUTION 2:

Undo the registry edits you did to move User directories off the System drive. If you are doing RegEdit in the recovery environment, remember to Load Hive and Unload Hive. You can re-apply the edits after the installation, with the usual caveats.

 

Hope this helps!
