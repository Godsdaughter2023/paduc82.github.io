bob jones aka: cybertaz say:

I don't believe information on formatting a date is the answer to your question.

A date of 2.20.2018 remains 2.20.2018 regardless of which date formatting applied. The formatting simply causes the date to be displayed as 2/20/2018, 20-Feb-18, Tuesday, February 20, 2018, etc.

It sounds like what you're looking for has to do with the 4 year+1 day difference that existed between Mac & PC date systems used in Excel prior to Excel 2011. If that's the issue, that setting is found in the Excel Preferences dialog, specifically:

Excel> Preferences> Calculation - Use 1904 date system

The 1904 date system is what used to be the default for Mac Excel, whereas the Windows version used the 1900 date system. However, starting with Office 2011 Mac Excel now uses the 1900 date system used on the PC. That still is the case in Excel 2016. IOW, both Excel for Mac & for Windows now use the same 1900 date system, so you should find that the check box is unchecked.

You give no indication of what prompted your inquiry, but it may be that you are working with files created by a Mac version earlier than Office 2011... Perhaps copying/pasting between files which are based on differing date systems. If so, you may need to visit this Preferences pane:

Excel> Preferences> Edit - Automatically convert date system

NOTE: Changing the date system Preferences will not convert the dates in existing workbook files. The date system is a property of the file, so dates are displayed correctly regardless of which system normally is used by the version of the program opening the file. The default date system affects only new workbooks created by that version of the program. Essentially, the date system used in a workbook is transparent except when working between workbooks that use differing date systems.
