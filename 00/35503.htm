¡firedog say:

Thanks for an exemplary problem description. The only thing missing is the version of Windows Live Mail: I'll assume it's the latest, 16.4.3528.0331.

A couple of general comments:

  • BT has been advocating the use of secured connections for a long time now. It may be that they will sooner or later insist upon it, meaning that any open connection request (i.e. with SSL:No) will be rejected. This will probably happen account by account, depending on which server the mailbox is hosted on.
  • BT still haven't managed to transfer all BT Yahoo accounts to their own BT Mail service. It's not possible to tell from the error messages which you're using. Yahoo-powered accounts are notoriously unreliable when accessed from a mail client, and they're sensitive to minor corruptions in messages on the server.
  • The fact that another account on the same computer is behaving properly indicates that the problem lies with your account, not with anything in Windows Live Mail.
  • The second error message you quote is clearly marked as a temporary system error at the server, so there is nothing specific you can do to mitigate it.

That said, there are a couple of things you can try to ease the situation. The first is to access your BT account and change the password. This will refresh your credentials there and should remove any obstacle to authentication. Once you've done so, back in Windows Live Mail, right-click on the account name in the folder pane and select

  • Ensure that the Email username is your full email address 
  • Clear the password box and enter the new one.
  • Ensure that Log on using clear text authentication is selected
  • Ensure that My server requires authentication is selected.

It may take a bit of time for the password change to take effect, so please wait an hour or two before deciding that it hasn't made any difference.

There may be a message in the mailbox at the server that Windows Live Mail can't or won't download for some reason - it's too big, it contains malicious code or it's just plain dodgy. items. Then create a new folder or folders and move all of the messages out of the inbox and into the new folder. When the inbox is empty, select the account from the dropdown list on the bottom half of the send/receive button in Windows Live Mail. No new email messages should eventually appear in the status bar. Does it?

Back in webmail, move half of the messages that haven't yet been downloaded by Windows Live Mail back into the inbox and again try send/receive for this account in Windows Live Mail. If the fault recurs, we now know which half of the messages the bad one is in. Repeat the process, taking half of the messages each time until the dodgy message can be identified and dealt with in webmail.

In future, don't use the webmail inbox as a storage folder. Move any messages for keeping on the server into archive folders, leaving only a few days' worth of messages in the inbox at any one time. Empty the spam and deleted folders regularly.

As always, please include any error message in your reply.
