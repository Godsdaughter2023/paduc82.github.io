Addendum: The multiple monitors can be re-enabled without doing something extreme (like uninstalling and reinstalling your display driver). Here are some steps:

1. Select the single monitor which appears in the center of the display properties control panel UI

2. Don't bother with the "Detect" link button - it's useless

3. With the single monitor selected, locate the drop down labelled "Multiple Displays"

4. From the drop down, select "Duplicate on 1 and 2"

5. Apply/Keep that change when prompted

6. Go back to the same drop down, and now select "Extend Desktop on 1 and 2" (verbiage may not be correct, but a choice with similar wording will exist in the drop down at that point)

7. Apply/Keep that change, now you'll have monitors 1 and 2 working

8. Rinse and repeat for Monitor 3 (for me, I had 3 - people with 2 will stop at step 7)

The conclusion is: The HAL and PnP sub systems do recognize the additional monitors - it's a bug in the code that interfaces between the control panel UI for Display properties and the lower API's.

This means that if you play games on a machine that you also use with multiple monitors and you disable all but one for the gaming (so you don't hammer your graphics card, using more power and increasing MTTF, something a person actually cares about), then you'll have a wonderful additional display change step for each monitor that you have, thus making what was once a two click, single apply process now into a completely unintuitive and irritating process.
