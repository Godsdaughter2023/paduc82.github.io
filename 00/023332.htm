If they can somehow activate the public Windows 10 for you, then just install that instead of Insider Preview.  You can create an ISO to install the public build.  However, I don't know of any way to activate it without the Windows 7 product key.

Switching out a motherboard usually will remove Windows activation, so this is not unusual.  If you Windows 7 was a stand-alone license it could be transferred to the new "device" or new motherboard, but still you would normally need to purchase a product key, unless Microsoft support choses to give you a product key.

If you had never done a clean install since the device was Windows 7, or if you have any backup done from back then, you might be able to read the old Windows 7 product key from the original install of Internet Explorer with a free product key reader of some type.  Often Internet Explorer would still have stored the original Windows product key if you were still running an upgrade of Windows.  But if you have done a clean install (which you likely did with the new motherboard) this is not too likely a possibility for recovering the Win 7 key.  If you had a restorable old backup from the old motherboard, you could maybe restore it to a spare hard disk and boot to it in safe mode to read the old key.  Success retrieving the old Windows key this way seems unlikely, but it's the only scenario I can think of that could possibly work.  It might seem easier to just purchase Windows 10.

Here is a different approach.

Based upon how many Statuses you have just add into the first query --

SELECT Action_ID, Department, Sum(IIF([Status] = "Open", 1, 0)) AS CountOfOpen, Sum(IIF([Status] = "Close", 1, 0)) AS CountOfClose

FROM tblTable;

ken sheridan say: You could write your own function:

Public Function ActionCount(strDepartment As String) As Integer

    Dim rst As DAO.Recordset
    Dim qdf As DAO.QueryDef

    Set qdf = CurrentDb.QueryDefs("Action_Query_1")
    qdf.Parameters(0) = strDepartment
    Set rst = qdf.OpenRecordset
    
    With rst
        .MoveLast
        ActionCount = .RecordCount
        .Close
    End With
    
    Set rst = Nothing
    
End Function

In the report add a text box with a ControlSource property of:

=ActionCount([Department])

fa-lw say:

Is there any way to write an expression for the input to the department parameter query as the criteria of the DCount expression?

 

Is there any thing I can put in place of parameterqueryresult in the following expression?

ken sheridan say: You cannot call the DCount function against your current query.  You can either:

1.  Use the function I posted, which passes the parameter to the querydef object.

2.  Change the parameter in your query to a reference to a control on an open form and call the DCount function without a criterion:

=DCount("*","[Action_Query_1]")

3.  Create a copy of the query under a new name and remove the parameter.  Then call the DCount function with a criterion:

=DCount("*","[Action_Query_1_Without_Param]","[Department]=""" & [Department] & """")
