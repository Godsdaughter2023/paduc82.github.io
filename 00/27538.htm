[Resubmitted with critical typos corrected]

The Soc Sec format takes an actual 10-digit value like 123457890 and

The important take-away is: numeric formats affect only the appearance of values, not the actual value.

If do not like the dashes, don't choose the Soc Sec format. To allow for possible leading zeros (not possible with SSNs?), use the Custom format 0000000000 (10 zeros).

On the other hand, if you typed (or read from a text file) 123-45-7890 with the dashes, Excel recognizes that as

In that case, either you need an event macro to change what is typed at data-entry time, or you need a formula to remove the dashes in a parallel column. Then you could copy-and-paste-value over the original data entry.  For example, in B1:

=--SUBSTITUTE(A1, "-", "")

formatted as Custom 0000000000 .

Angle of text - Microsoft Community

gabrielfortune say:

Angle of text

In MS Word there used to be a function - in 'format text' I think - where you could change the alignment of a piece of text, so that it was at an angle or vertical, (as on the spine of a book). I can't seem to find this anywhere in the new version. Any ideas, anyone?
suzanne s. barnhill say:

What you're looking for is "Text Direction." You can rotate text 90 degrees clockwise or counter-clockwise in tables, text boxes, and frames. We'll leave aside frames, which you're unlikely to be using (and which require adding a Text Direction button to the QAT).

  • In a table, find the Text Direction button in the Alignment group on the contextual Table Tools | Layout tab.
  • In a text box, find the Text Direction button in the Text group on the contextual Drawing Tools | Format tab.

If you decide to add a Text Direction button to the Quick Access Toolbar, there are three different ones, which you can experiment with to see which function you prefer.
