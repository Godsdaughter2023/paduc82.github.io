chip pearson say:

You cannot get a list of random numbers without duplicates with a formula. You need to use a VBA function.  See http://www.cpearson.com/excel/randomNumbers.aspx for details and example and downloadable code. That module contains a function named UniqueRandomLongs which returns an array of integers, without duplication, given an inclusive minimum and maximum. Select the cells you want the values in and type =UniqueRandomLongs(1,5,5) and then press CTRL SHIFT ENTER rather than just ENTER. This will return an array of 5 integers between 1 and 5 inclusive.

 

You can downlod the zipped module file at http://www.cpearson.com/Zips/modRandomNumbers.zip .

 

The UniqueRandomLongs function is designed to return an array of horizontal orientation. That is, a range of cells all in the same row, spanning several columns. If you want a vertical array, one that is in one column spanning several rows, use

 

=TRANSPOSE(UniqueRandomLongs(1,5,5))

 

as your array formula. See http://www.cpearson.com/Excel/ArrayFormulas.aspx for details about array formulas.

I remember this formula and have been waiting to see one of your posts to point out that there's a potential error in it.

 

As a stand alone formula it can never produce a 1. For example with A1 empty put that formula in A2 and continually tap F9 and it wont produce a 1. Put a number in A1; any number between 1 and 5, and it will produce a 1.

 

The problem can be overcome with this simple change, Note the  +1 at the end. It will now produce unique randoms between 1 and 5.

 

=LARGE(ROW($1:$5)*NOT(COUNTIF($A$1:A1, ROW($1:$5))), RANDBETWEEN(1,5-ROW(A1)+1))

macropod say:

This is possible - it just takes a fair bit of work to implement. Basically, every cell that requires the reformatting needs to contain a field coded along the lines of:

{IF{MERGEFIELD Col4field}= "Classroom" {QUOTE "Text to display" \* Charformat} {IF{MERGEFIELD Col4field}= "eLearning" {QUOTE "Text to display" \* Charformat} "Text to display"}}

where 'Col4field' is the name of whatever you field is for column 4 and "Text to display" is the text to be formatted.

The format of the 'Q' in 'QUOTE', combined with the \* Charformat switch controls the formatting.

If the text to be formatted is just the output of another mergefield, you can use:

{IF{MERGEFIELD Col4field}= "Classroom" {MERGEFIELD Somefield \* Charformat} {IF{MERGEFIELD Col4field}= "eLearning" {MERGEFIELD Somefield \* Charformat} {MERGEFIELD Somefield}}}

The format of the 'M' in the 2nd & 4th 'MERGEFIELD', combined with the \* Charformat switch controls the formatting.

Note: The field brace pairs (i.e. '{ }') for the above examples are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues.

No macros required.

talking moose say:
Sorry, no. This behavior is actually based on the Finder's behavior for moving or copying files.

If you drag a file or folder to a different location on the same volume (same disk) then the behavior is to move.

If you drag a file or folder to a different location on a different volume (different disk) then the behavior is to copy.

If you drag a file or folder to a different location on the same volume (same disk) while the Option key is held down then the behavior is to copy.

If you drag a file or folder to a different location on a different volume (different disk) while the Command key is held down then the behavior is to move (or rather copy and delete).

In Outlook, different accounts would be considered different volumes.

Hope this helps!
