To install the next build (10041) after 9926 be sure to - Microsoft Community

j w stuart say:

To install the next build (10041) after 9926 be sure to install all Windows Updates - Revised 3/19/15

Technical Level : Basic

Summary

In particular check that the following 2 updates are installed. If they are not installed you can manually download the files (choose either the x86 for 32Bit systems or X64 for 64bit installations of Windows 10).
Details

Update KB3040021: https://support.microsoft.com/kb/3040021

Manual download: http://catalog.update.microsoft.com/v7/site/Search.aspx?q=3040021

Update KB3038930: https://support.microsoft.com/kb/3038930

Manual download: http://catalog.update.microsoft.com/v7/site/Search.aspx?q=3038930

1) Select the 32Bit or 64Bit version based on which version of the Windows 10 Preview you currently have installed and click 'Add' to add it to your basket (upper right corner of the page).

2) Now click on the 'View Basket' link and then click the 'Download' button to save it to a folder of your choice (I recommend the 'Download' folder)

3) Install the update or updates by double clicking on the file or files you downloaded. The update is installed nearly instantaneously.

4) Now check to see if your PC is set to the 'Fast' Ring, change it from 'Slow' to 'Fast' if you want the current build now.

Reference: Speed up when new builds are installed

5) Then run Windows Update to install the build. This may take a while to download and install so leave yourself plenty of time.

Original Reference: Gabriel Aul @GabeAul 

2 more hotfixes going out to #WindowsInsiders running 9926 today to prep for next build:

per jessen say:

You can only have one instance of an event code in a workbook (or worksheet) module.

 

The way to solve this is to let your code detect which product (sheet) you have selected, and then run the corresponding piece of code.

 

Sub Workbook_SheetDeactivate

If activesheet.name="VAS" then

     macro1

elseif activesheet.name="Product2" then

     macro2

elseif activesheet.name="Product3"  then

    macro3

Endif

End sub

 

 If the four product sheets are set up the same way, you can just use an object variable as reference rather than the sheet name.

 

Sub Workbook_SheetDeactivate

Dim DestSh as Worksheet

Set DestSh=ActiveSheet

 

LastSheet.Range("A1").Copy

destsh.Range("B10").PasteSpecial xlPasteFormulas

 

'Etc...

End Sub

 

BTW: You should avoid using select statements in your code as they are not needed, but they will slow down your code...

 

Your previous macro can be reduced to something like this:

 

Sub PAT()
'
Dim DestSh As Worksheet
Dim TargetCell As Range
Set DestSh = Worksheets("VAS")
Set TargetCell = ActiveCell

TargetCell.Offset(0, -7) = "PAT"
TargetCell.Copy
DestSh.Range("H1000").PasteSpecial Paste:=xlPasteFormulas, Operation:=xlNone, _
    SkipBlanks:=False, Transpose:=False
TargetCell.Offset(0, -4).Copy ' Remember TargetCell is equal to the activecell when the macro is run
DestSh.Range("D1000").PasteSpecial Paste:=xlPasteFormulas, Operation:=xlNone, _
    SkipBlanks:=False, Transpose:=False

  
 
'----Cut ---
With DestSh
    .Range("B1000").Formula = Date

    .Range("N1000").Formula = "PATs"

    ActiveWindow.Panes(1).Activate

        SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal

        .SetRange Range("A1:N1000")
        .Header = xlYes
        .MatchCase = False

        .SortMethod = xlPinYin

rohn007 say:

No problem, just do a normal uninstall through control panel, then download and install 64 bit.  Your license allows you to run either version as you need.

Now before you do it, have you answered this question, "Why do you want 64 bit". Simply because you are running 64 bit Windows is not enough.  MS still recommends 32 bit unless you have explicit need for 64 bit. The most common reason is Excel. Either you have dowloaded large data sets that cause file size to approach 2 GB, or you have complex spreadsheets (lots of calculations) that approach 2 GB of RAM when open.

The main reason for sticking with 32 bit is that many companies have not created 64 bit versions of their addons. This includes MS!  So before switching to 64 bit, check that there are 64 bit versions of all of the Addons you absolutely need.

The option to get the 64 bit version is found in the same place as the offline installer and downgrade version:
