oliviawebb98 say:

Adding an option group (numerical value) to options in a drop down list from a form.

I have made a form with a drop down menu in it and I would like to add a numerical value to these options. 
Thank you in advance. 

scottgem say:

What is the RowSource of the Combobox? Is it a Value List or a Table? The easiest would be to make it a table and add a field with your numerical value.

It would help more to understand what the combobox is used for and what numercial value you want to add and for what purpose.

ken sheridan say: Are you talking about a combo box?  If so you can include a numerical value in a hidden column which is specified the BoundColumn of the control.  The value of the control will then be that of the hidden column when you select an item from the visible column.  The following is an example of how to set up a combo box control which lists employees by name but whose value is the hidden EmployeeID:

RowSource:     SELECT EmployeeID, FirstName & " " & LastName FROM Employees ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

Where the RowSource is a value list the hidden numbers and visible text values are paired in the string expression.  Otherwise the combo box is set up as above.  the following is the value list for the RowSource property of a combo box which lists the month of the year by name, but whose value is the numerical month number, which makes for easier processing of the value:

1;January;2;February;3;March;4;April;5;May;6;June;7;July;8;August;9;September;10;October;11;November;12;December
