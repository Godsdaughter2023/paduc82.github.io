Need Specific Fields from the Database Documenter - Microsoft Community

billbeshlian say:

Need Specific Fields from the Database Documenter

Is there a way to query your the back end of an Access database to return the Names, Data Types, Sizes and Field Description only? I don't need all of the detail from the Database Documenter when you select Names, Data Types, Sizes and Properties option. Thanks.

Bill Beshlian

daniel pineault say:

Here, something like:

' strDb  : Full path and filename of the External Database to Query

' strTableName : Name of the table to retrieve information about

Function TableInfo(strDb As String, strTableName As String)
    On Error GoTo TableInfoErr

    ' Source: http://allenbrowne.com/func-06.html

    ' Author: Allen Browne, Modofied by Daniel Pineault

    ' Purpose:   Display the field names, types, sizes and descriptions for a table.
    ' Argument:  Name of a table in the current database.
    Dim db                    As DAO.Database
    Dim tdf                   As DAO.TableDef
    Dim fld                   As DAO.Field

    Set db = OpenDatabase(strDb)
    Set tdf = db.TableDefs(strTableName)
    Debug.Print "FIELD NAME", "FIELD TYPE", "SIZE", "DESCRIPTION"
    Debug.Print "==========", "==========", "====", "==========="

    For Each fld In tdf.Fields
        Debug.Print fld.Name,
        Debug.Print FieldTypeName(fld),
        Debug.Print fld.Size,
        Debug.Print GetDescrip(fld)
    Next
    Debug.Print "==========", "==========", "====", "==========="

TableInfoExit:
    Set fld = Nothing
    Set tdf = Nothing
    db.Close
    Set db = Nothing
    Exit Function

TableInfoErr:
    Select Case Err
        Case 3265&    'Table name invalid
            MsgBox strTableName & " table doesn't exist"
        Case Else
            Debug.Print "TableInfo() Error " & Err & ": " & Error
    End Select
    Resume TableInfoExit
End Function


Function GetDescrip(obj As Object) As String
    On Error Resume Next
    GetDescrip = obj.Properties("Description")
End Function


Function FieldTypeName(fld As DAO.Field) As String
'Purpose: Converts the numeric results of DAO Field.Type to text.
    Dim strReturn             As String    'Name to return

    Select Case CLng(fld.Type)    'fld.Type is Integer, but constants are Long.
        Case dbBoolean: strReturn = "Yes/No"            ' 1
        Case dbByte: strReturn = "Byte"                 ' 2
        Case dbInteger: strReturn = "Integer"           ' 3
        Case dbLong                                     ' 4
            If (fld.Attributes And dbAutoIncrField) = 0& Then
                strReturn = "Long Integer"
            Else
                strReturn = "AutoNumber"
            End If
        Case dbCurrency: strReturn = "Currency"         ' 5
        Case dbSingle: strReturn = "Single"             ' 6
        Case dbDouble: strReturn = "Double"             ' 7
        Case dbDate: strReturn = "Date/Time"            ' 8
        Case dbBinary: strReturn = "Binary"             ' 9 (no interface)
        Case dbText                                     '10
            If (fld.Attributes And dbFixedField) = 0& Then
                strReturn = "Text"
            Else
                strReturn = "Text (fixed width)"        '(no interface)
            End If
        Case dbLongBinary: strReturn = "OLE Object"     '11
        Case dbMemo                                     '12
            If (fld.Attributes And dbHyperlinkField) = 0& Then
                strReturn = "Memo"
            Else
                strReturn = "Hyperlink"
            End If
        Case dbGUID: strReturn = "GUID"                 '15

            'Attached tables only: cannot create these in JET.
        Case dbBigInt: strReturn = "Big Integer"        '16
        Case dbVarBinary: strReturn = "VarBinary"       '17
        Case dbChar: strReturn = "Char"                 '18
        Case dbNumeric: strReturn = "Numeric"           '19
        Case dbDecimal: strReturn = "Decimal"           '20
        Case dbFloat: strReturn = "Float"               '21
        Case dbTime: strReturn = "Time"                 '22
        Case dbTimeStamp: strReturn = "Time Stamp"      '23

            'Constants for complex types don't work prior to Access 2007 and later.
        Case 101&: strReturn = "Attachment"         'dbAttachment
        Case 102&: strReturn = "Complex Byte"       'dbComplexByte
        Case 103&: strReturn = "Complex Integer"    'dbComplexInteger
        Case 104&: strReturn = "Complex Long"       'dbComplexLong
        Case 105&: strReturn = "Complex Single"     'dbComplexSingle
        Case 106&: strReturn = "Complex Double"     'dbComplexDouble
        Case 107&: strReturn = "Complex GUID"       'dbComplexGUID
        Case 108&: strReturn = "Complex Decimal"    'dbComplexDecimal
        Case 109&: strReturn = "Complex Text"       'dbComplexText
        Case Else: strReturn = "Field type " & fld.Type & " unknown"
    End Select

    FieldTypeName = strReturn
End Function

jeeped say:

It would appear you've run into the infamous Last Cell property of Excel. To verify the cell that Excel believes is the last relevant cell, you can use the GoTo function. Press F5, then ALT+S, ALT+S <enter>. This will take you to the cell that Excel believes to the the outer extent of information. The cell may not contain any information but the row and/or column it is on either has information or had information at some point.

To reset the LastCell of a worksheet, select a cell that is one down and one right to the one you want to be the absolute extents of your worksheet (e.g. all the way down and right). For example, if you want Z99 as the new Last Cell, select AA100. Next, walk through these steps to completely remove extraneous information,

  1. Press CTRL+Shift+<right arrow>. Release and press CTRL+<spacebar>. Release and press ALT+E then D or right click the highlighted columns and select Delete.
  2. Press Home, then press CTRL+Shift+<down arrow>. Release and pressShift+<spacebar>. Release and press ALT+E then D or right click the highlighted rows and select Delete.
  3. Press CTRL+Home to return to A1. Immediately Save or Save As if you don't trust your keystrokes.
  4. Press CTRL+End and you should be in the Last_Cell (e.g. Z99).

It might seem like a lot, but it takes a lot more time to type it out here than it does to actually do it.

Due diligence: There is the potential here to delete a lot of possibly relevant data and/or formulas. Make sure you have a backup copy!

My solution was to connect the phone by Bluetooth rather than by cable.

Mine is a Windows 8.1 phone whose usb port must be damaged.  It will charge, but when connected to a PC, Windows Explorer, or File Explorer in Windows 10, will not see the phone.  Yet, when I connect another phone of the same model, also Windows 8.1 Phone, the pc can see it.  Then, how to transfer files from the phone to the PC, or from the PC to the phone?

First, I installed Files, an app in the Store, published by Microsoft.  Caution, there is another app that must be purchased, published by a third party.  Once installed, I can see the location of all files in the phone, just as I can from a PC when the phone is properly connected by cable.  But, since I can't connect by cable, I connected the phone to the PC via Bluetooth, then I performed file transfers over that connection.  Unlike a cable connection, and unlike viewing from Windows Explorer, I cannot select multiple files to copy and paste.  Instead, I must transfer files one by one, which was better than nothing.

BTW, the need for doing this arose because I was using Microsoft's Contacts+Message Backup app to backup SMS's and MMS's.  (I haven't found a way to restore Windows 8.1 Phone cloud backups to the Windows 10 Phone.)  With the windows 8.1 phone not having an SD card slot, I had to save the backup files onto the phone itself.  Using Files, I discovered that the files ended up in a subfolder named backup+restore in the Documents folder of the phone.  Without Files installed, I don't know if I would have been able to see the Documents folder anyway.  However, it was very handy to have Files so I could easily find what I was looking for, even before attempting to do the file transfer.

* Edit, a minute later: If you have Driver Update installed, uninstalling it should make Bug Splat disappear.

Bug Splat is an error reporting tool being used by some app or program that you've installed.  It's not malicious.

https://www.bugsplat.com/

If you tell/allow Bug Splat to send the report, you should be able to see which program is using Bug Splay to report errors. You can then uninstall or update that program or seared the web for solutions.

E.G. https://www.google.com/search?q=sketchup+bug+splat

If bug splat is appearing at startup, you should try to disable or uninstall it or determine which program is  program that's causing it to run.

Start> Settings> Apps> Apps & features, if Bug Splat is listed uninstall it.

Plan B... When the message appears, right-click the Taskbar> Task Manager> Startup tab. If Bug Splat is listed in the startup list, disable it.

If those steps don't disable Bug Splat look at Task Manager> Startup again. What non-Microsoft programs are loading at startup that don't need to be, possibly a game.
