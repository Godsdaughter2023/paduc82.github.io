    Removing check mark from field codes box in Word - Microsoft Community

    bro05bill say:

    Removing check mark from field codes box in Word

    I have a problem where, even after changing the normal.dot to uncheck the show field codes box, the change does not take and in all of my word documents with a hyperlink, including new documents, I have to manually go to the options in every document and uncheck the box to use the hyperlink, every single time.
    beth melton say:

    The option to show field codes instead of their values is stored in the Registry, not your Normal template. :-)

    First try these steps: 

    1.  Exit Word and Outlook if using Word as your email editor. If use Outlook and aren't sure, exit Outlook anyway.

    2.  Start Word and make sure you're working with a single document. (It can be the new document that's created or an existing document. It's just important only a single document is open.)

    3.  Click Tools/Options/View and make sure Field Codes is not selected

    4.  Click OK to close the Options dialog box

    5.  Exit Word and see if the Field Codes option remains deselected.

    If it does then you’ve rectified the issue. The underlying cause is having multiple documents open when you change your options. Each open document window ‘holds’ a set of options for Word and the options in effect for the last document window you close is what Word uses to update the Registry.

    For example, if you have DocA and DocB open and both have the Field Codes option turned on, if you change the Field Codes option in DocA (and not DocB) and close DocA before DocB, then the change you made to the Field Codes option is discarded when you closed DocA, because DocB is still open and maintaining its own set of options.

    If after stating Word you see Field Codes turned back on then chances are you have a third-party add-in that is changing the option when Word starts. To rule out your add-ins:

    Go to Tools/Templates and Add-Ins and see if there are any global templates or add-ins listed. If you find any then chances are they are located in your Office\Startup folder or Word\Startup folder. The location of the Word\Startup folder can be found under Tools/Options/File Locations and the Office\Startup folder will be located in the installation path for Office. Exit Word and move any add-ins out of either folder to prevent them from loading when Word starts. If you have multiple add-ins then you might want to move them one at a time and try reproducing the issue in order to determine which one is causing the problem.

    Word also supports COM Add-ins. For COM Add-ins, which are added through the registry rather than the Startup folders. To check for COM add-ins, you can use the following steps to access the COM Add-Ins dialog box:

    1. On the Tools menu, click Customize.
    2. On the Commands tab, under Categories, click Tools.
    3. Under Commands, click COM Add-Ins and drag the COM Add-Ins command to a toolbar location of your choice
    4. Click the OK to close the Customize dialog box.
    5. Click the COM Add-ins button to run the utility and deselect the add-ins to prevent them from loading when Word starts. Like the global templates, if there is more than one, disable them one at a time and try reproducing the issue in order to determine which one is causing the problem.
    ~Beth Melton Microsoft Office MVP
