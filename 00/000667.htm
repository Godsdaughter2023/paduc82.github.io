david a matson say: Greetings Wally!   Welcome to the Windows Mobile Total Access forums!

I understand your situation, but the problem is that while the computer [B]was[/B] meant to allow multiple users, a Windows Mobile Device [B]was not[/B] designed with multiple users in mind.

As such, while all of you will be able to synchronize with the device on each of your different XP profiles, you will each have the exact same information through ActiveSync (contacts, notes, appointments, etc).  If your "shared application" is specialized in that it asks for the users XP ProfileName (e.g. "Joe Blow" etc) while using the Mobile Device, then it's conceivable that the data gathered during that session might very well be specific to that user's profile, in which case you would have separatation between data pools.

I hope this helps.

trevor rabey say:

I am not exactly sure what you mean by "highlighted" since it has several possible meanings.

First one, if you want to show the school holidays as a shaded vertical stripe on the Gantt Chart, make a calendar that has those days as nonworking days. Then right click on the chart area and select nonworking time from the menu. Then you can select that calendar for nonworking tiime. Make it a nice bright cheerful colour.

Second one, there is the possibility of creating a highlight filter. This is like a regular filter except that the tasks which meet the value test come up highlighted in a colour, and the ones which don't meet the test also are displayed rather than being filtered out. To do this I would go to view, filter, more filters, select the date range filter, copy it, give the copy a distinctive name starting with "AA...", and then click on the highlight button.

I hope one of these is the one you want.

john - project say:

Hans,

I assume by "highlight" you mean you want to show the school holidays on the Gantt Chart display, independent of the actual working calendar. To do that, create a custom calendar with the school holidays as exceptions.

Then use that calendar to display non-working time in the timescale (double click on the timescale header and select the Non-working time tab in the Timescale window). Note, this will only affect the Gantt display, it will not have an affect on the project calendar.

Another option is to create a Project schedule wherein the "tasks" are school holidays

However, if you are only trying to show school holidays for construction contractors, then I think a better approach is to create a school year calendar in Excel with school holidays annotated. A good example of this is shown at this link:
