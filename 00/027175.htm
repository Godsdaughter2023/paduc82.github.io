Replace Exchange 2010 with a new Exchanger server

Split from this thread.

Thanks for coming back.

Once we move an on-premise mailbox to Exchange online by using Exchange hybrid migration, the on-premise mailbox will be deleted. In Exchange 2010, a deleted mailbox will become a disconnected mailbox rather than be purged from the database. The disconnected mailbox will stay there until the retention policy expires, which is 30 days by default.

You can use the Remove-StoreMailbox cmdlet to manually purge disconnected mailboxes.

If you want to keep the Exchange hybrid deployment, both a CAS server and a mailbox server are needed. If you want to keep only one server, you can remove the CAS server and then add the CAS role in the current mailbox server. After that, you need to re-run Exchange Hybrid Configuration Wizard to rebuild the hybrid deployment.

If you plan to remove the hybrid deployment and use only Dirsync, we also suggest you keep the mailbox server as you can use the Exchange Management UI to manage users locally. Once you remove all Exchange servers, all Exchange related attributes will be hidden from local AD. Notice that using Active Directory Computers and User and ADSI Editor to manage users are not officially supported.

By the way, we can also configure Office 365 for a multifunction device or application to send emails out. Hi Paul,

 

Here’re the steps to create a new Exchange server with all roll in as a replacement of the two old Exchange 2010 servers:

 

1. Deploy a new Exchange 2016/2013 server with all roles. 

2. Move mailboxes from Exchange 2010 database to the new database in Exchange 2013 or 2016. You can refer to this official article: https://social.technet.microsoft.com/wiki/contents/articles/33372.mailbox-migration-from-exchange-2010-to-exchange-2016.aspx. 

Note: if you have nothing to move, then you can skip this step. 

3. Remove the two Exchange 2010 servers.

4. Run the HCW in the new Exchange server to reestablish the hybrid deployment. 

 

By the way, if you need assistance about #1, here’s an official called Exchange Hybrid Assistance that can help you deploy Exchange 2016&2013 coexistence with Exchange 2010.

Word can only edit text in a graphic that is made with the Office drawing tools. Any graphic created in another program is just a graphic for Word - it can't access the individual content inside. "HotDocs" must use a different format/protocol to create this graphic than what Word uses.
