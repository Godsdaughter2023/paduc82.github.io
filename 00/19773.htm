Change formatting on a Table of Contents - Microsoft Community

teddaniels say:

Change formatting on a Table of Contents

This is not possible in my copy of Word for Mac 2011. The "Modify" option as grayed out, apparently permanently. Please help correct this situation.
john mcghie say:
You need to set the Format of the Table of Contents to "From Template" (in Options>Formats).  That is the only setting that enables you to change the format of a TOC.

You then change the format of the TOC by changing the formatting of the Styles TOC 1 through TOC 9 that Word uses to format the TOC.

When you do this, a "template" of your TOC formatting is saved with the document, so every time the TOC updates, Word can replace it with one formatted the way you want it.

The Word Help topic "Create or edit a table of contents" will show you how top do this.

Hope this helps

shermanconservatories say:

This may be a crude method - but it's the only thing that worked for me after finding no help online.

Go to C > Program Files > Microsoft Office > Root > Office16

Here you will find all the program .exe files for all the Office programs. One of these is the Upload Center program. It is called MSOUC.exe (you'll recognise it by the icon (big white arrow over an orange circle). 

Simply delete the file - then Upload Center no longer runs. Simple!

magdy ghobrial say:

Getting Rid of MSOUC.EXE

If you delete MSOUC.EXE, it resolves nothing, it will be reinstalled right back, OneDrive connects to Microsoft Server for updating and fixing errors, so it will be reinstalled.

First and foremost you need to disable MSOUC.EXE in Windows Firewall from both incoming and outgoing rules before deleting it:

You list the complete path in the firewall as follow:

C:\Program Files\Microsoft Office\root\Office16\MSOUC.EXE

Not a good idea to list it as

%Program file\Microsoft Office\root\Office16\MSOUC.EXE

Now you may delete MSOUC.EXE.

At this point, Microsoft server won’t be able to reinstall it back; however I went one step further:

I deleted these two files which are used by MSOUC.EXE from this folder:

C:\Users\User Name\AppData\Local\Microsoft\OFFICE\OTele

{CBCDE49F-946D-4BD9-96C5-8598BBBDBDDC} (0) - 4732 -

&

{CBCDE49F-946D-4BD9-96C5-8598BBBDBDDC} (0) - 4732 -

Now you can be certain; MSOUC.EXE won’t re-infect your PC again when you activate OneDrive, its Setup will pop-up but terminated in a split second by the firewall rules, it won’t be able to communicate with Microsoft server and reinstall the MSOU.EXE back.

I tested it 12 times in the past 24 hours and Microsoft server failed to reinstall MSOUC.EXE in my PC.

If anyone knows of a better solution, let us know!

 

P.S. This last step is not required and I don’t recommend it, I deleted every entry in the registry pointing to MSOUC.EXE except the four belonging to Firewall New Rules, those stay put.
