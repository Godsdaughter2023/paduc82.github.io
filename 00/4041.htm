When adding a rule to apply to emails based on size, there is a spot to specify the size (for instance, to automatically move emails over a certain size to a specific folder).  This works correctly in Outlook for Windows.  The number you specify is in Kb.  So, if you want to apply a rule to emails larger than approximately 20Mb you can put in 20000 (k is automatic) and it works fine.  There is an equivalent place to do this in Outlook for Mac 2011.  But, anything larger than 999 gets truncated (remember, that would mean 999kb).  If you enter 20000, it turns into 20.  If you enter 1000, it turns into 1.  I found another question on this site where someone recommended as a solution that the user set up a rule for emails larger than 20Mb, and he responded that would be great, but it won't let him enter anything over 999 and that anything larger gets truncated, so I know I'm not the only one.  His response when never answered.

jguice say: Yeah, same problem here.  Can't find a solution anywhere.

I submitted the following via the feedback link under help:

There is a bug in Outlook 14.3.1 (130117) when creating a size-based rule.  Sizes greater than 999 cannot be entered.  The problem appears to be that a text input formatter automatically adds commas which (on save) causes the first comma and any digits following to be truncated.

Sizes like 1024 should be enterable (e.g. for 1024k or 1m).
