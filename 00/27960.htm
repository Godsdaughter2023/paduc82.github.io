if you are not familiar with macros,

take a look here..

step1

Save As, your Workbook with extension .xlsm (macros enabled)

step2

Sub Disable_AutoF_AdvF()

step3

select Disable_AutoF_AdvF from the list and click the run button.

bobgreenblatt say: You are generating an infinite loop that causes the stack space overflow. With the addressbook activated, you call the updateaddressbook sub. That sub activates addressbook 2. It then reactivates the addressbook sheet which fires the worksheetactivate event which reruns the call addressbook code., etc. etc. etc.

There are a bunch of solutions. Here are 2: 1) you could insert an application.enableevents=false to prevent the activate event from firing (Remenber to re enable events when the macro ends or the sheet activate macro will never run again.); or 2, a much better solution is: don't select any sheets.

It is almost NEVER necessary to activate a worksheet when a macro is running. Doing so slows down macro execution and leads to problems like you are having, and is generally just sloppy code. The macro recorder generates very inefficient code. you should always go through it and "clean" it up.
