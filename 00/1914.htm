Unfortunately there is no help file associated with this property of a Userform and changing the value has no effect - what gives?

 

"The new phonebooks are here!"
hansv mvp say:

The Left is the distance of the left edge of the userform from the left edge of the screen in points, where 72 points = 1 inch.

For example, you could place a command button on the userform with the following On Click code:

Private Sub CommandButton1_Click()
  Me.Left = Me.Left + 72
End Sub

Each time you click the button, the form will jump 1 inch to the right.

The Top property similarly is the distance of the top edge of the form from the top of the screen.

jllatham say:

Here's another approach to it:  You said that column B is sorted already, and this works off of that assumption. It looks from the top down for the 1st match, then looks from the bottom up for the first match in that direction.  Have to do a special one-liner to check if the first instance is in row 1, but that's easy.  Using .Find is only slightly faster than the "For Each c In MyRange" method that Mike H. took - actually both perform about the same for lists shorter than roughly 50,000 entries.  However, I think this one might be a bit faster because we don't have to do the Union() operation for subsequent matches, and if there are a lot, that's a player because Union() is (in my opinion) a slow operation.

The code:

Sub FindMatchingRows()
  'redefine these Const values to match up with
  'your workbook's sheet names and layouts
  'the sheet to be searched:
  Const srcSheetName = "Sheet1" ' source data sheet name
  Const srcCol = "B" ' search column B for matches
  Const srcFirstDataRow = 2 ' first row to be searched in the column
  Const srcFirstColToCopy = "A"
  Const srcLastColToCopy = "C"
  'now about the destination sheet
  Const destSheetName = "Sheet2"
  Const destCol = "A"
  Const destFirstDataRow = 2 ' lowest row to paste into ever
 
  'working variables
  Dim valueToFind As Variant
  Dim searchRange As Range
  Dim foundCell As Range
  Dim firstRow As Long ' will hold first row # of matched value
  Dim lastRow As Long ' will hold last row #of matched value
 
  'get the search for value from the user
  valueToFind = InputBox("Enter value to find: ", "Search For...", "")
  If valueToFind = "" Then
    'user didn't enter anything, exit
    Exit Sub
  End If
  'This assumes that the search column is sorted in
  'ascending order (least to most)
  'create a reference to all entries in the column
  ' the reference goes from row 1 (presumably with headings in it)
  ' down to 1 cell beyond the actual end of the list
  Set searchRange = Worksheets(srcSheetName).Range(srcCol & "1:" & _
   Worksheets(srcSheetName).Range(srcCol & Rows.Count).End(xlUp).Offset(1, 0).Address)
  'find the first entry in the list
  Set foundCell = searchRange.Find(valueToFind, after:=searchRange.Cells(1, 1), _
   LookIn:=xlValues, lookat:=xlWhole, searchdirection:=xlNext)
  'if there were no headers in row 1, we can miss it there, so we do a special
  'test just for row 1!
  If CStr(searchRange.Cells(1, 1)) = valueToFind Then
    Set foundCell = searchRange.Cells(1, 1)
  End If
  If foundCell Is Nothing Then
    MsgBox "Sorry, but " & valueToFind & " was not found in the list!", _
     vbOKOnly + vbInformation, "No Match Found"
    GoTo CleanupAndExit
  End If
  'save this row number!
  firstRow = foundCell.Row
  'if we get here, there was a match, now look from the bottom of the list
  'up to find the last entry, which could be the same as the first
  Set foundCell = searchRange.Find(valueToFind, _
   after:=searchRange.Cells(searchRange.Cells.Count, 1), _
   LookIn:=xlValues, lookat:=xlWhole, searchdirection:=xlPrevious)
  lastRow = foundCell.Row
  MsgBox "First Row = " & firstRow & vbCrLf & "Last Row = " & lastRow
 
  'now you could set up the copy something like this:
  Range("A" & firstRow & ":" & "C" & lastRow).Copy
  'do your copy action as needed
 
CleanupAndExit:
  'good housekeeping
  Set foundCell = Nothing
  Set searchRange = Nothing
End Sub
