The rich text content control field  is in the footer.

Please advise.

Thanking you in advance.

jay freedman say:

Using only the content control events provided by Microsoft, typically you would write code in the Document_ContentControlOnExit event handler (see http://gregmaxey.mvps.org/word_tip_pages/content_controls.html).

Since that event fires whenever the cursor exits any content control in the document, you have to start the procedure by using an If statement or a Select Case structure to determine whether the control that was exited (which Word supplies as the first argument of the procedure) is the dropdown that you're interested in.

If it is the correct dropdown, you can get the selected value as a String variable, for example

strResult = ContentControl.Range.Text

You can use that value in any way you need in order to "update" the rich text content control (you haven't said what you mean by that).

If you must have the update happen immediately whenever the user selects a different value in the dropdown, then you need a more complicated macro. Microsoft has not given us a Document_ContentControlOnChange event, but it's possible to create one as described at http://gregmaxey.mvps.org/word_tip_pages/content_control_custom_events.html.

There are also a couple of different ways to tell VBA which control is the one you want to update. If it's the only content control in the footer, you could use

Set RichTextCC =  _

ActiveDocument.Sections(1).Footers(wdHeaderFooterPrimary).Range.ContentControls(1)

Alternatively, if you have assigned a Title or a Tag to the rich text content control in its Properties dialog, you can use one of these methods:

Set RichTextCC = ActiveDocument.SelectContentControlsByTitle("CC Title")(1)

Set RichTextCC = ActiveDocument.SelectContentControlsByTag("CC Tag")(1)

zack barresse say:

Hi Michael,

There are several workarounds for this. The easiest way for this to happen is if you use data validation with 'Time' as the 'Allow' method. Then just set your time to between 0:00 and 23:59.

Another way is to have an intermediary cell calculate the value of the input for you. Let's assume you have three cells here:

  • Time Input
  • Time Format
  • Time Calculation

Your 'Time Input' cell can be the same cell you're using, where users are entering "4:00" or "4". The 'Time Format' cell can have an intermediary calculation in it looking at the 'Time Input' and adjusting for user error. This assumes the user will not be entering in time values greater than 24 hours, or one day. Here is an example of the formula you could use for your 'Time Format' cell:

    =IF(TimeInput>1,TimeInput/24,TimeInput)

Another way you could write it is:

    =TimeInput/IF(TimeInput>1,24,1)

Both give the same result.

Then, the 'Time Calculation' is your downstream calculation, where instead of pointing it to the original 'Time Input' cell, you'd point it to the 'Time Format' intermediary calculated cell.

And another way altogether is to use a UserForm with VBA. This is a bit more in-depth, but gives you much more manual control of user inputs and what to do with them.

In any case, I'd recommend the first (data validation) solution.
