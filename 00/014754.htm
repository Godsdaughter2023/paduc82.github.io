Rolling Working Time in Calendar in Microsoft Project - Microsoft Community

cjdavisuk say:

Rolling Working Time in Calendar in Microsoft Project

Can anyone suggest a solution to setting up a calendar in MS Project 2016 that allows for a rolling working pattern of 11 days working then 3 days off. This pattern always starts on a Monday.

john - project say:

CJDAVISUK,

Okay, here tis. You will need to first create a custom 7-day workweek calendar. The macro uses that as the basis for the custom 11 days on - 3 days off calendar.

Hope this helps and if it answers your question please consider marking this response as the answer.

John

Note: there are some redundant variable declarations which can be deleted. They are artifacts from other macros in the same module.

Option Explicit
Option Compare Text
Public ver As String, ResData As String, ResNam As String
Public i As Integer, j As Integer, ResID As Integer, cday As Integer
Public Seed As Variant, Seed2 As Date, Seed3 As Date, S1 As Date, S2 As Date, F1 As Date, F2 As Date
Public fd As Date, StEx As Date, FiEx As Date
Public ResCal As Calendar

Sub Dayson_Daysoff_calendar()
'This macro will create a 11 days on/3 days off sequence
'It is based on a copy of an existing 7 day workweek calendar with default work time (8am to 5pm with 1 hour lunch)
'The number of exceptions is arbitrarily set at 25. That will cover just short of 2 years
'Macro written by John-Project Dec 13,2018

'first create the new custom calendar
On Error Resume Next    'in case the custom calendar already exists
BaseCalendarCreate Name:="11 on-3 off", FromName:="7-day week"
If Err > 1 Then
    BaseCalendarDelete Name:="11 on-3 off" ' clear old calendar
    BaseCalendarCreate Name:="11 on-3 off", FromName:="7-day week"
End If
On Error GoTo 0
'Ask user for the date to start the 11on/3off sequence
'   if no date is entered assume Monday of current week
Seed = InputBox("Enter start date of first 11 day work period" & vbCr & _
    "as mm/dd/yy" & vbCr & vbCr & "If no entry is made, Monday of current" & vbCr & _
    "week will be used")
If Seed = "" Then
    cday = Weekday(ActiveProject.CurrentDate): j = 0
    If cday = 1 Then j = -1
    If cday > 2 Then j = cday - 2
    Seed = ActiveProject.CurrentDate - j
End If
'reset seed of 1st exception as 11 days later
Seed = DateAdd("h", 264, Seed)
'add 2 days + 9 hours to arrive at a finish of 5:00 pm
fd = DateAdd("h", 57, Seed) 'add 11 days to the start date
For i = 1 To 25
    ActiveProject.BaseCalendars("11 on-3 off").Exceptions.Add Type:=1, Start:=Seed, Finish:=fd, Name:="exception " & i
    Seed = DateAdd("d", 14, Seed)   'sets start of next group 14 days later at 8:00 am
    fd = DateAdd("h", 57, Seed)    'sets finish of next group 13 days later at 5:00 pm
Next i
End Sub

robert aldwinckle on forums say:

All I get is a blank screen with a 'Diagnose Connection Problems' button, but when you press that it comes up no fault found, or words to that effect.


That only means that the diagnostic tried to get addressability to the problem host and ping it.  Unfortunately, that doesn't prove that you have HTTP connectivity at that address.   So, if the address is bad, there could be a host there that can return a ping but not one that could respond to your HTTP request.

Also, when you close that diagnostic you should be waiting for an automatic refresh of the page you are trying to access.   What happens then?   Watch for changes in all of the elements you can:  Status bar, Title bar, Address bar, Tab label, as well as the window content.

In fact, before getting that refresh try finding out whether the diagnostic has assisted in caching the lookup.   Eg. in a cmd window enter:

     ipconfig  /displaydns

It will probably help if before attempting the test you start with

     ipconfig  /flushdns

and if your cmd console buffer is too small you may want to capture the results, e.g.

     ipconfig  /displaydns  >displaydns.txt

followed by

     notepad  displaydns.txt


To further refine your symptom description you could try running Fiddler2 and see if it gives any clearer picture of what is going on.   Running Fiddler actually changes the network configuration for your scenario (Fiddler is a proxy) so it might even act as a workaround.   Also, using Fiddler's Request Builder will generate a cleaner set of diagnostics for the dnscache.

karl timmermans [outlook mvp 2012-2018] say:

Custom field import is not supported in any version of Outlook via Outlook import (desktop or web) regardless of account type.


ContactGenie import products (QuickPort/DataPort/MultiPort) do support user-defined fields/custom forms  which may be of interest but note that UDFs are not visible via OWA. UDF fields will however be migrated to/from contact folders in Office 365 "E" accounts unlike Outlook.com accounts where UDF info does not migrate bi-directionally even though CG programs do import directly into an EAS contact folder without any intermediate steps required.
