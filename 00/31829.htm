First, I opened msconfig and wrote down (or took screenshots of) all Services and Startup items.  Then I disabled everything and successfully booted into Safe Mode for the first time in months.  After that, it was just a matter of re-enabling the one at a time (or 50% at a time using a binary search) to find the bad apple.  After that I re-enabled everything and confirmed that the presence of a single service could kill Safe Mode.

Keep in mind, in Safe Mode almost every service should be disabled automatically but Windows 7 screwed up, and allows some services to start in Safe Mode despite the fact that it's called "Safe Mode".  In this case, the service that was allowed through was called TTService, which is as a licensing service for Admiralty Digital Publications.  My guess is this service is dependent on a Windows service that is not available in Safe Mode, and the presence of TTService in Safe Mode triggers a failure (with nothing logged to any Windows events).

New Rule:  If you have the TTService installed and enabled by default, then you cannot boot into Windows 7 Safe mode

The service itself is safe and desired under normal operation, but if it's set to automatically start then your Windows system is in serious jeopardy if you ever desperately need safe mode.

tx200 say:

I think I may have found a solution but it involves going into the registry editor and changing the alias address for the one that you want displayed.

All the relevant risks associated with editing the registry apply here so do at your own risk!

Open an elevated command prompt and type regedit to get to the registry editor or do a search for regedit from the taskbar.

Do a search for your alias email address using the Edit/Find command. You should eventually find the alias email address in a registry key similar to below. Numbers will probably be different for each user so I haven't put them all in.

HKEY_USERS/S1-5-21-**********-**********-**********-**********-***/SOFTWARE/Microsoft/ActivSync/Partners/{There should be a group of numbers and letters here that when clicked should show your email account details}

You should now see a registry key called 'Email' that will have your alias email address in the data field. Change this address by double clicking on the 'Email' key and in the box that pops up input the correct email address that you want to use in the mail app. Click OK to save the changes and exit out of regedit.

Lastly, reboot the PC for the changes to take effect and the correct email address should now be shown in the mail app.

This has worked for me on two PC's now and it's too early to say if the fix is permanent and you do need to replicate the procedure on each PC that has the problem but so far so good.

I upgraded to Windows 10 pro from Windows 10 home in order to be able to use Bitlocker. Now I'm told I need a USB key. How do I get this?

jan del say:

Hi, Margaret.

BitLocker requires you to have a USB key in able to access the information. You can create a USB key by using a blank thumb drive or USB drive formatted to NTFS OR FAT32 and follow these steps:

  1. Click Start and go to Control Panel.
  2. Look for System and Security then click it.
  3. Select BitLocker Drive Encryption.
  4. Under BitLocker Drive Encryption, click Turn on BitLocker.
  5. Choose how you want to unlock your drive during startup: Insert a USB flash drive or Enter a Password and then click Next.
  6. On the How Do You Want To Store Your Recovery Key Page, click Save the Recovery Key To A File.
  7. In the Save BitLocker Recovery Key As dialog box, choose a save location, and then click Save.
  8. You can now print the recovery key if you want to. When you have finished, click Next.
  9. On the Are You Ready To Encrypt This Drive page, click Start Encrypting.

Note: Do not remove the USB flash drive until the encryption process is complete. How long the encryption process takes depends on the size of the drive and other factors.

Get back to us if you need further assistance.
