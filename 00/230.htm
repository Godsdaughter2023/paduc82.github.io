Formula for pulling data in previous cell - Microsoft Community

zombette say:

Formula for pulling data in previous cell

What I am looking for is a working formula for D21 and D22 to show the averages for the data provided in cells F6 through F17. However, D21 needs to show the most recent data entered, and D22 needs to show the average of all the data entered so far, minus the current entry.

For example, in the image below, let's pretend my employee has entered in his averages for January through April in column F. Let's do 1 in Jan, 2 in Feb. 3 in Mar. and 4 in Apr. D21 should display "4" as that is the current month and last entered (in this instance) and D22 should display "2" which is the average of all the entries, minus the entry in April.

I would like this to continue on as they fill out the sheet throughout the year and also be able to leave cells blank or enter "n/a" if they hadn't started working until mid year or so.

I have a feeling I'm explaining this in a very weird way, but I'm hoping someone out there understands me.

amit tandon say:

Refer below image:

Enter below formula in cell D21:

=IFERROR(INDEX($F$6:$F$17,MATCH(9.99999999999999E+307,$F$6:$F$17)),"n/a")

Enter below formula in cell D22:

=IFERROR(AVERAGE(OFFSET($F$6,,,MATCH(9.99999999999999E+307,$F$6:$F$17)-1)),"n/a")

Cell D21 returns the last value entered in range F6:F17;

Cell D2 returns the AVERAGE of values in range F6:F17 (from F6 till the second-last value);

If all cells in range F6:F17 are blank or non-numerical, D21 & D22 will display "n/a".

jllatham say:

You can't actually access that name, but you can use it.

 

A sheet has 2 names:

  The name that you give it that shows up on its name tab, and

  The Code Name - the internal name.  You can actually change this in the VBA Editor, but not from the worksheet nor with VBA code either.

 

Lets say you have a sheet that has a name like "Data Entry" on its tab, but you see in the VBA development environment that it has a code name of Sheet1.  In your VBA code you could refer to it simply as Sheet1 throughout the code and the code will continue to work even if a user changes its tab name from "Data Entry" to "Source Data" or anything else.  If you need to find out (in code) what that sheet's current name on the tab is you can use a statement like

 mySheetName = Sheet1.Name

http://www.jlathamsite.com/Teach/VBA/WritingBulletProofCode.pdf

It explains all of this and a little more in section 2.  It's pretty good info, or at least I thought it was when I wrote it :).
