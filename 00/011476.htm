Various approaches assuming that you are using Exchange (Global Address List)

 

Option #1

If you want to combine your contacts in Outlook into the Global Address List, you'll need to have your Exchange Admin add those contacts to the Active Directory (either as contacts or users as appropriate)

 

Option #2

#1 - If you want to have a single folder of contacts - alternative option would be to create an Exchange Public (contact) Folder ensuring that all that require access to it are given the appropriate permissions by your Exch admin

#2 - Import your GAL into the newly created public folder

#3 - Move/copy/export your Outlook contacts into the newly created public folder

#4 - Ensure that this new contact folder is also added to the AddressBook

         --> right click on it --> Properties --> Outlook Address Book -->
         --> check the box "Show this folder as an e-mail Address Book"

At the end of the process - you will have a single folder with all your contacts

larrybuss say:

Hi folks

I was able to resolve this issue.

VMware Fusion installs links in the /Applications directory to the Windows versions of the Office apps to support the Fusion Unity function. Unfortunately the links that VMware Fusion creates use the same names as the Office 2016 apps (i.e. Microsoft Word, etc). The Office MAC installer recognizes these links and installs the applications in sub directories (i.e. /Applications/Microsoft Word/...).

I removed the VMware Fusion Unity links, uninstalled and installed Office 2016 and all is good now.

Thanks to everyone who responded.
