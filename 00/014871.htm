windows phone 8 QR Code scanner bug - Microsoft Community

traviscogdill say:

windows phone 8 QR Code scanner bug

Was using my nokia 920 at work using the built in code reader to test some QR Codes I was creating.

 

Thought my code creating program was wrong... turns out the reader is wrong in some cases.

 

In particular, if you have a QR Code encoded in the Numeric character set mode ... the last couple characters (numerals) may be left off.

 

For example, a Version 1 code with Error Correction level H is capable of storing 17 numerals.  The built in windows phone scanner on my nokia 920 only shows the first 15 numerals of such a code.

 

Well since MS doesn't accept bug reports as near as I can tell, thought I'd put this out there for whoever... didn't see any other discussions on it.

 

FYI I the free "BeeTagg" app on windows phone marketplace correctly reads the code, so that's a nice work-around.

 

It appears alpha-numeric and 8-bit codes work properly ... but if the message (or even just a portion part of the message) is encoded with NUMERIC mode, you may lose some characters from the end of the message.  This happens in code versions other than just version 1.  Have not tested if it happens in error correction levels other than H.

 

Code was created with libqrencode for those interested ... also tested with other encoders with the same end result.
