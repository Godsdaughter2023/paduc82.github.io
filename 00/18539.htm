What this means based on your description is that you previously had the email account configured as a POP3 account (which uses a PST file) and now have it configured as an IMAP account. The .ost data file associated with the IMAP account is a "mirror" of what is contained on the server. When you imported your PST file, it got sync'd back up to the server.

What I would suggest is the following

#1 - Log into your email account via webmail (using your browser) and delete old emails from your email folders. The one thing you need to be aware of is that if you delete <everything> then you will also delete all new emails since you imported your PST file

#2 - When item #1 is done, open the Control Panel <Mail> app and create a new Outlook profile and configure your email account again. Make sure to set the newly created Outlook profile as the "default" for the next time Outlook is opened.

Now whether you want to configure your account again as an IMAP account (the default if you only enter your email address/password) or as a POP3 account is something that you'll need to decide. However, as mentioned, it really sounds like it was configured as a POP3 account previously so this would be your likely choice. You will need to configure the account manually by clicking on <Advanced Options> --> <let me set up my account manually> after entering your email address. tom van stiphout say:

That is by design.

It sounds like you have a "monolithic" database (meaning: all database objects in one file) in a shared location (e.g. on a network server).

User1 opens in A2016. This changes the references (Alt+F11 > Tools > References) to those for A2016.

User2 opens in A2010. It cannot run with those references, and they are NOT changed back.

There are two REALLY BAD ideas in this configuration:

1. A monolithic database is against best practices of splitting between front-end and back-end. This is a FAQ. Search online to find many pages explaining this.

2. Either the monolithic database or perhaps the front-end (FE) (if it was indeed split) is in a shared location rather than on each workstation. That is also against best practices and a FAQ.

To recover from this snafu:
Open the References dialog as indicated above.
Observe the ones listed as broken. Take note of them, and uncheck them.

Select similar references for Office 2010.

Db now works again.

Before it gets messed up again, fix the two really bad ideas.

  Dim SlideDict As Object 'Dictionary
  Dim SlideItems As Variant
  Dim i As Long
  Set SlideDict = CreateObject("Scripting.Dictionary")
  SlideDict.CompareMode = vbTextCompare
   'Get all slides from the first presentation
  For Each SlideItem In Presentations(1).Slides
    SlideDict.Add SlideItem.Name, SlideItem
  Next
  'Test if slides from second presentation exists in the first presentation
  For Each SlideItem In Presentations(2).Slides
    If Not SlideDict.Exists(SlideItem.Name) Then
      Debug.Print "Slide " & SlideItem.Name & " did not exists in " & Presentations(1).Name
    Else
      'Compare the slides
      CompareSlide SlideDict.Item(SlideItem.Name), SlideItem
      'Remove it
      SlideDict.Remove SlideItem.Name
    End If
  Next
  'Report slides not in second presentation
  SlideItems = SlideDict.Items
  For i = LBound(SlideItems) To UBound(SlideItems)
    Debug.Print "Slide " & SlideItems(i).Name & " did not exists in " & Presentations(2).Name
  Next
End Sub

e.j. gun say:

Two methods:

  1. Select the cells with the function in them, Copy, then select empty cells, select Paste/Paste Special.../Values - the empty cells will contain only the text.
  2. Change Hans' Function to a Sub (shown below).  Select all the cells to be converted (in Column B?).  Run the subroutine (Developer tab, Macros).  The cells to the right of the selected cells (in Column C?) will be filled with the desired text only.

Please mark Hans' response as the Answer - because it is.  I'm only plagiarizing his work!

Eric

Sub ParseValues()

    Dim theRange as Range, theCell as Range

    Dim arr as Variant

    Dim i as Long, n as Long

    Dim ParseValue as String

    Set theRange = Selection

    For Each theCell In theRange.Cells

        If (theCell.Text <> "") Then

            ParseValue = ""

            arr = Split(theCell.Text, ",")

            n = UBound(arr)

            If n Mod 2 = 0 Then

                ParseValue = "Error - odd number of elements"

            Else

                For i = 0 to n Step 2

                    ParseValue = ParseValue & vbLf & arr(i) & Replace(arr(i + 1), Chr(34), "")

                Next i

                If ParseValue <> "" Then

                    ParseValue = Mid(ParseValue, 2)

                End If

            End If

            theCell.Offset(0, 1) = ParseValue

        End If

    Next theCell

End Sub

Function ParseValue(s As String) As String
    Dim arr As Variant
    Dim i As Long, n As Long
    arr = Split(s, ",")
    n = UBound(arr)
    If n Mod 2 = 0 Then
        ParseValue = "Error - odd number of elements"
    Else
        For i = 0 To n Step 2
            ParseValue = ParseValue & vbLf & arr(i) & " = " & Replace(arr(i + 1), Chr(34), "")
        Next i
        If ParseValue <> "" Then
