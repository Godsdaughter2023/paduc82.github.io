joeu2004 say: Nimalan wrote:

I have some VB Codes that I use in Excel 2013. My intention is to Bring the Selected Image to Front for certain seconds and then to sent it back automatically. the Codes that I use is as follows.

Sub Bring_Front()
Selection.ShapeRange.ZOrder msoBringToFront
Application.Wait(Now + TimeValue("00:00:03"))
Selection.ShapeRange.ZOrder msoSendToBack
End Sub

But the Above Codes makes the macro to delay by 3 seconds and then it execute the Bring to Front and Send to Back command simultaneously.

Try the following:

Sub Bring_Front()
Selection.ShapeRange.ZOrder msoBringToFront

DoEvents

Application.Wait Now + TimeValue("00:00:03")
Selection.ShapeRange.ZOrder msoSendToBack
End Sub

The intent of DoEvents is to allow the Excel thread to run, which brings the image forward.

However, on a multi-CPU or multi-core computer, I'm not sure that guarantees the Excel thread does run then.

The following might be a more reliable design.

Sub Bring_Front()
Selection.ShapeRange.ZOrder msoBringToFront

Application.OnTime Now + TimeValue("00:00:03"), "Send_Back"
End Sub

Sub Send_Back()
Selection.ShapeRange.ZOrder msoSendToBack
End Sub

Also, Date + (3+Timer)/86400 might be more reliable than Now + TimeValue("00:00:03").

The latter might delay for as little as 2 seconds instead of 3 seconds because VBA Now is accurate to only a second.

Actually, the following is most reliable (albeit overkill, arguably):

Dim t As Double

t = Date + (3+Timer)/86400

If Int(t) <> Date Then t = Date + (3+Timer)/86400

It is needed only when Date is called the first time near the end of the last 15.625 millisec of the day (!), and Timer is called the first time just after the system clock advances to the next day (!).

The probability is extremely small; zero if you never execute Bring_Front near midnight.  But it can happen.
