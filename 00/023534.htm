MSFlexgrid is almost certainly not installed on the Win10 box, and you can't simply copy the .ocx from another machine and try to use it. It must be installed through some mechanism, and since you're obviously using this in a development environment, you'll need to be sure that install is compliant with your needs. That means you'd have to install something that includes that .ocx, and properly installs/registers all the necessary dependencies, etc. This would typically be something like VB 6, Visual C/C++ or something along those lines. If you don't have those, then you can't install the component (and you shouldn't be using it in the development environment if so, since you don't have a license to use it).

The installation of Access does NOT include that control since it's not compatible with Access (never has been, except for the handful of versions that were put out under the Access Developer stuff, which was stopped after 2003). Moving forward, if you simply must have the MSFlexGrid (or similar functionality), I'd suggest you either use one of the very, very few Access-compliant grids, or move to a different development environment (like one of the .NET languages).

Had Phillip bothered to test his 'theory' before posting he would have found that the suggestion will do no good. The display size of the Row numbers & Column headings is fixed in Mac Excel -- There is no setting to change them. If the Row Height is reduced the "crowding" will occur.
