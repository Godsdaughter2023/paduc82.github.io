To answer your question the process is the same regardless of the price of the application.  Free and paid applications must go through the same certification process with the same cost structure.  It makes sense given that regardless of whether or not an application is sold or given away (a) the author can still be making money off of it and (2) the same effort is consumed on certifying it.

For 99 USD registration you are able to submit up to 5 applications for certification.  Note that if your application fails certification that counts towards one of your 5 attempts. For each application over the 5th application you must pay 99 USD to have it certified.  If you choose to submit your application to additional markets then there is an additional 9.99 USD to get the application recertified for that market.  This assums that you have a single install that will work for all of those marketplaces.  IF you make a different version of your application for each marketplace (ex: one for English and another for æ¥æ¬èª) then those are two seperate certification attempts.

Each year you will need to renew your registration for the Marketplace.  If you do not remove your registration then your applications would be removed from the Marketplace.

want to write MM : YY format in access 10 - Microsoft Community

prosenjitdey say:

want to write MM : YY format in access 10

I have a database. in the table one field is date type. I want to put information as MM:YY format. pls help. 

john w. vinson say:

As in 09:14?

Just use a Format property of

"mm\:yy"

either as a field format in the table, or (better) as the Format property of the form or report textbox in which you're displaying the date.

For entering the data, you'll need to use an actual date - and a date must be a full date, to the day, September 2014 isn't a date; it's 30 dates plus a whole bunch of times, accurate to the second. If it's important to be able to enter the date in that form, you'll either need some VBA code or switch from a Date field to Text.

Note that the Format of a field does not affect how the data is stored, just how it's displayed. A Date/Time field is actually stored as a Double number, a count of days and fractions of a day (times) since midnight, December 30, 1899. You can display this number using any of a vast array of different formats, but it's just a number underneath.

Your question was vague, but John partially answered you. You can't do what you want without some trickery. As John said you HAVE to enter a valid date value to use a Date/Time field. You can DISPLAY that value using just month and year, but the stored value had to be a valid date,

If you want to query a table for a data range by only entering the month and year. Then I would use an unbound form with 6 controls on it. One to select the Start Month, another the year, the next to select the End month and then the year  and two hidden controls to translate that to valid dates.

Then set the hidden controls to :

=DateSerial([cboYear],[cboMonth],1)  for Start date and

=DateSerial([cboYear],[cboMonth]+1,0) for End Date

Then in your query set the criteria to:

BETWEEN Forms!foormname,txtStart AND Forms!formname!txtEnd

marshall barton say:

First, you should use syntax that conforms to the MS standard, including Access.  Colon (:) is used to separate time parts - hours, minutes and seconds and apostrophe (') is not meaningful in a date.

Second, most dates require a four digit year.  Dec 12 means the 12th of December in the current year, which will not work at all for you.  OTOH, Dec 2012 means the 1st of December in the year 2012, which will then work using   Between StartDate AND Serial(Year(EndDate), Month(EndDate)+1,0)

Finally, if you really want to use text strings instead of real dates, then there are only a couple of choices, one is what Scott posted and another, though NOT recommended,  is to use the strict data entry rule of entering

   YY-MM   E.g.  12-09

including the "date" field in the tables you want to search.  This way, you can search for dates in the 21st century using  Between StartDate And EndDate
