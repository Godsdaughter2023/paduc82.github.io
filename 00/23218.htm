I came across two seemingly identical numbers that yield a difference (see below). I wonder if you could give me a hint about possible causes?

0.75000000000000000000 0.75000000000000000000 0 '=ABS(A9-B9)
0.50000000000000000000 0.50000000000000000000 0 '=ABS(A10-B10)
0.20000000000000000000 0.20000000000000000000 5.55E-17 '=ABS(A11-B11)
0.00000000000000000000 0.00000000000000000000 0 '=ABS(A12-B12)
0.80000000000000000000 0.80000000000000000000 0 '=ABS(A13-B13)

The work-around is:  =ROUND(ABS(A11-B11),2), if you want the results to be accurate to 2 decimal places.

In general, when you expect calculations to be accurate to a specific number of decimal places, explicitly round the results to that number of decimal places.

The situation is not uncommon.  For example:

The constant 0.1 is subtracted from the constant 0.3 in A3.  The difference between A3 and the constant 0.2 is calculated in A5.  Note the infinitesimal difference, about 2.78E-17.

In part, the problem arises because most non-integer decimal fractions cannot be represented exactly in binary; consequently, they must be approximated.  The exact representations of the values in column A are shown in column C.

Also, the binary representation that Excel and most computer applications use -- 64-bit binary floating-point -- can represent numeric values with more precision than the 15 significant digits that Excel formats.  In fact, it requires 17 significant digits to reproduce the binary representation exactly.

Consequently, calculated values like A3 often have more precision than Excel displays.

Compounding the confusion, note that =A3=A4 returns TRUE in A6, but =A3-A4=0 returns FALSE in A7.  Mathematically, they should have the same result, of course.

The difference arises because in A7, the subtraction is calculated exactly, whereas in A6, Excel arbitrarily considers A3 and A4 to be "close enough".

Likewise, =A3-A4 results in exactly zero arbitrarily, whereas =A3-A4-0 reflects the true infinitesimal difference.
