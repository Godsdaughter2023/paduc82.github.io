From what I've read this can be a problem with windows looking at the wrong CRC (from a USB Drive) and not seeing a problem when it sees the correct one on boot up so it doesn't know what to do.

What I tried that fixed it; I went into Bios and went to the SATA controller and changed the type from RAID to IDE and did a reboot. I chose the repair option and it made it to the checking disk for errors screen this time. I let it finish its check and then continued, It made it to the login screen.  I did not log in, I used the on screen shut down button and shut down the PC, then I changed the Controller back to RAID.

Everything works fine now. I don't know if this will work with different RAID settings, mine sees each HDD as individual drives. 

I hope this helps someone coming across this post as I drove myself nuts trying to fix this. Only thing I can figure is it tricked windows into thinking you were loading a backup drive and forced it to correct itself.
