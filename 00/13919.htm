How to import data already has key into an Access table also has autonumber as key field

I wrote a new Access database to replace an old Orecle database. I downloaded data from Oracle into a separate Access. While examining the data, I notice the oracle main table has unique key which is similar to Access autonum key, and all other tables use this unique key to build the relationship.

However, in my new Access database I also created a key field (autonum).  If I simply import the old data into my table, my table will create another key automatically. So I need to know how to import the old tables into my new database and still maintain the referencial integrity for the relationship. I also need to have the autonum feature to create unique key for future data entry.

Thanks for all the helps.  

albert d. kallal say:

You have a few choices here. First, if the Access table is empty then do a compact + repair to re-set the auto number. This should then allow you to import the Oracle table without any key violations.

If you cannot do the above, then simply add an extra column called ExtNumber. You thus then import the older primary key number into this new column (so it is NOT a autonumber, but just a long number column). You will of course now have "different" numbers for the auto number key, but you ALSO have the older key values that extra column.

You thus can simply update the child table FK value to the "new" auto number key. You can then delete or discard the ExtRow column once done.

The sql query to update the child records to the NEW key is quite simple, and the query will look much like this:

UPDATE Invoices INNER JOIN Customers ON Invoices.CustomerId = Customers.CustomerID SET Invoices.CustomerId = [Customers].[ID];

And you can build this in the query builder. Drop in the child table first, then the parent table.

john mcghie say:
Hi Bjorn:

This is a real bug that has persisted for a few versions in Mac Word.  A PDF contains the version of the program that wrote it.  Mac Word takes that literally.  PC Word will send the PDF to any version equal to or higher than the one that wrote the file.  We have been hoping they would fix this for some years.

There are two ways to get those files open:

1)  Find someone with a PC

2)  Take the file to pieces and dig out the PDF.  Now: this is a little complex, but quite simple once you get used to it.

a)  Save a copy of the file

b)  Change the extension of the copy to .zip

c)  Double-click it: OS X will decompress it, and a folder will appear.  (DO NOT try this on the original: once you have done this, Word can't read the file, and there's no easy way to put it back together...)

d)  Open the folder: you will find a little website inside.

e)  Open the "Word" folder, and look in the "Media" folder.

Now, this is the interesting bit.  In there you will find a file for each of the "things" in the document apart from the text.  The names will be relatively meaningless: things like "Image" or whatever.  Doesn't matter: they're only pointers: Word knows where they belong (you can look in the document.xml file is you need to know which belongs where...)

The EXTENSION is what will enable you to find your PDFs.  Chances are, Word has mislabeled them, and I can't tell you for sure what it will have mislabeled them TO.  Here, it appears to be using ".emf" for PDF.

Either change the extension to .pdf or double-click the file and point it at Adobe Reader when it asks which program (you hopefully do not have a program that can open .emf on your computer -- if you do, you will need to change the extension because it won't ask).

If you have guessed correctly, the PDF attachment will open straight up.

If not, try one of the other files: it WILL be in the Media folder, somewhere :-)

And use Help>Send Feedback to tell Microsoft that after five years it's high time they fixed this bug :-)

Hope this helps.
