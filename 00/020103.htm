bill manville say:

What you hoped would happen is entirely reasonable, but - as you have discovered - not what does happen. 

If it is something you have to do often then I would suggest a pair of macros.

Dim rCopied As Range

Sub MyCopy()
  Set rCopied = Selection
End Sub

Sub MyPasteVisible()
  Dim rCell As Range
  Dim rVisible As Range
  Dim rRow As Range
  Dim rArea As Range
  Dim iCopiedRows As Integer
  Dim iRowToCopy As Integer
  iCopiedRows = rCopied.Rows.Count
  Set rVisible = Selection.SpecialCells(xlVisible)
  iRowToCopy = 1
  For Each rArea In rVisible.Areas
    For Each rRow In rArea.Rows
      rCopied.Rows(iRowToCopy).Copy
      rRow.PasteSpecial xlAll
      If iRowToCopy = iCopiedRows Then iRowToCopy = 1 Else iRowToCopy = iRowToCopy + 1
    Next
  Next
  Application.CutCopyMode = False
End Sub

To use, paste the above into a standard module.
Select the range you want to copy and Alt+F8 > MyCopy > Run
Select the range you want to paste to and Alt+F8 > MyPasteVisible > Run

How to copy the results in the Find and Replace Dialog Box? - Microsoft Community

eskk_homel say:

How to copy the results in the Find and Replace Dialog Box?

Does anyone know of a way to copy the results in the Find and Replace Dialog box? I need to document all the external links in a workbook, I used the find and replace  option in Excel and found all the sheets and cells that contain the external link.  I would like to be able to copy all the information appeared on the dialog box (sheet, cell, and formula) to either a text file or another worksheet.  Does anyone know of a code that can do this?  Your help is much appreciated.

Amy

ossiemac say:

Try the following code. I assume that you simply set the parameter to Workbook in the Find and Replace dialog but unfortunately there is no VBA code that I am aware of to perform that function and it is necessary to loop through the worksheets in the workbook.

The following code adds a worksheet for the output. I left the added worksheet with the default sheet name and did not rename it because that entails ensuring that the worksheet name does not already exist and if it exists then ask the user whether to clear the worksheet or delete it etc. However, if you want additional code to do this then let me know but it is easy enough to rename it manually after the code runs.

There is also a caveat. If the "[" character exists somewhere in a cell the workbook where it is not actually a link then it will also be found.

Sub Macro2()
    Dim ws As Worksheet
    Dim wsOutput As Worksheet
    Dim rngToFind As Range
    Dim strFirstAddr As String
   
    Set wsOutput = Sheets.Add(After:=Sheets(Sheets.Count))
    With wsOutput
        .Range("A1") = "Location of Link"
        .Range("B1") = "Formula"
    End With
   
    For Each ws In Worksheets
        If ws.Name <> wsOutput.Name Then
            With ws.Cells
                Set rngToFind = .Find(What:="[", _
                                LookIn:=xlFormulas, _
                                LookAt:=xlPart, _
                                SearchOrder:=xlByRows, _
                                SearchDirection:=xlNext, _
                                MatchCase:=False)
                If Not rngToFind Is Nothing Then
                    strFirstAddr = rngToFind.Address
                    Do
                        With wsOutput
                            .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0) _
                                        = rngToFind.Address(External:=True)
                                       
                            'Following line concatenates single leading quote _
                             to the formula so it is dislayed as text _
                             otherwise it evaluates the formula _
                             and the value appears in the cell.
                            .Cells(.Rows.Count, "A").End(xlUp).Offset(0, 1) _
                                         = "'" & rngToFind.Formula
                        End With
                        Set rngToFind = .FindNext(rngToFind)
                        If rngToFind Is Nothing Then Exit Do
                    Loop While rngToFind.Address <> strFirstAddr
                End If
            End With
        End If
    Next ws
   
    wsOutput.Columns("A:B").Columns.AutoFit
End Sub

john mcghie say:

Hi Jennifer:

The usual culprit is the use of open-source word processors.  Open office and its derivatives do sometimes change the font of the Normal style (the "default" font, if you like) and from that will flow all manner of carnage.

Themes are also an invention of the devil, sent to amuse those who consider the use of a machine-gun excessive subtlety... In 40 years of doing this, I have yet to find a single use for themes!  (They were originally a response to our request for "Cascading Style Sheets".  Themes were the response, a comprehensive "failure to launch" that has not improved since.)

So your first step needs to be to tell your students "Ensure your document is set to base theme, named "Office".  Other themes will render your formatting non-compliant and result in your papers being sent back for correction!"

Once you have set the document to the Office theme, you can then run through and set the fonts of your desired styles. First, set the font of Normal style to be TNR, and then BREAK the LINK for each of the other styles.

  1. So:  For the Body Text style, go to Format>Style and select Body Text
  2. Click Modify
  3. Click Add to Template
  4. Set "Style Based On" to NO STYLE (this breaks the link)
  5. Click Format and choose Font
  6. Select TNR and 12 points
  7. OK your way out and do the next style

The most important part is setting Style Based On.  That is the link that connects every style to its predecessor in the style chain.  In a default blank document, that link is set to "Normal" for every style in the document.

You need to break that link, otherwise any change to the theme or Normal style is going to instantly cascade through the whole document. Normal is actually "Style 0" in any document, the equivalent of saying "same as previous".

This all sounds like a lot of work.  The following macro will do it all for you in a single click:

Sub TNR()

' Sets all styles to Times New Roman
' Macro written 6 July 2014 by John McGhie

On Error GoTo ErrHandler

Dim aStyle As Style

With ActiveDocument

    With .Styles("Normal").Font
        .Name = "Times New Roman"
        .Size = 12
    End With
    
    For Each aStyle In .Styles
        With aStyle
            If aStyle.NameLocal <> "Normal" Then
                If .BaseStyle = "Normal" Then .BaseStyle = ""
                If .Font.Size <> 12 Then .Font.Size = 12
            End If
        End With
    Next ' aStyle
End With
End

ErrHandler:
If Err = 5847 Then
    Err = 0
    Resume Next
    Else
        MsgBox "Macro was unable to process Style """ & aStyle.NameLocal & """" & vbCrLf _
        & "Possibly because its ""Based On"" property is already set to ""(no style)"""
        End
End If

End Sub

Notes:  The macro will work in a template, but you have to open the template as a "Document" using File>Open from within Word.

This macro changes only styles whose "Based on" style IS set to "Normal".  That's because you would normally set your Heading styles to be something else, and we want to leave that alone.

If the macro encounters a condition it can't handle, it will try to tell you which style it is having trouble with.

If you are unsure how to install a macro, Daiya explains how, here:

http://word.mvps.org/Mac/InstallMacro.html

Any questions, get back to us...

Hope this helps
