cbarnhorst say:

Yes, but Patch Tuesday pertains only to public release software.  There is no Patch Tuesday for pre-release software.  Patch Tuesday is the domain of the servicing branch only.  It exists for IT Pros in enterprises who requested a regular schedule to facilitate their deployment planning, with the understanding that urgent updates might be released at any time.

Notice that 18312 is a Fast Ring build that has nothing to do with the current release (currently 17763).  The patches prepared for the current release are done in the servicing branch, which is not involved with the pre-release branch.  It is not one monolithic effort.  It involves different, otherwise unrelated branches.  The flighting of a CU to 18317 in the pre-release branch on a Patch Tuesday is just coincidence, as far as I know.

Why does the RP Ring even exist?  Keep in mind that the release preview paradigm dates from Windows 7 a decade ago.  It continues in Windows 7 and Windows 8/8.1 up to the present.  The preview rollups that are presented to Windows 7 and 8.1 users are presented as optional updates a week or two ahead of the first Patch Tuesday of each month.  Since they are optional updates, no special targeting is needed because a user can opt-in one month and not the next.  But Windows 10 doesn't have optional updates, making targeting necessary in order to prevent update previews from being distributed to everyone instead of just those interested in previews.  Targeting requires a Ring, hence the creation of the Release Preview Ring (the target).  MSFT also needed to assign the RP Ring administratively and the choice was the Insider Program.  

RP members won't get involved with 19H1, et seq. until it is signed off on and then they get the final build as a normal early release update to the new current release.

As John said, queries are not very interactive. 

There are a number of ways you can do this. One way requires no coding at all. Create a "Split Form". This can be created using the More Forms option on the Create ribbon. A split form shows a datasheet view of your table or query with a form view of the record currently selected in the datasheet. You can then use datasheet filtering to create ad hoc queries to find the record you want and it will be displayed in the form view portion. 

Another way would be to create a form in Continuous form view. In the Header of that form, put unbound controls to either filter the form or pass criteria to a query. You can then use the Form's Double Click event to launch a new form filtered for the selected record.

julie fka profjulie say: There's no need to modify the registry to change the target folder of received files.  There's an option within the Bluetooth Settings.  You did not mention the device you have, so:

For a WM6.1 device,  tap Start > Settings > Connections tab > Bluetooth icon > File Transfer tab > find & select the folder you'd like to use.

For an earlier windows mobile device, tap Start > Settings > Connections tab > Bluetooth icon > Service tab > scroll down and select the File Transfer services option > then tap the Advanced button > find & select the folder you'd like to use
