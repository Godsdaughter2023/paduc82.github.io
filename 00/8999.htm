The following code in a macro prints a letter size document at 75% of its actual size:

ActiveDocument.PrintOut _
    PrintZoomPaperWidth:=0.75 * (8.5 * 1440), _
    PrintZoomPaperHeight:=0.75 * (11 * 1440)
You should be able to do something similar to achieve what you are after

jay freedman say:

This macro will work in all versions* of Word from Word 2003 to (currently) Word 2013. Even though the Print dialog isn't available from the user interface in 2010 and 2013, it's still usable from a macro. The SendKeys "%z" sends the accelerator Alt+Z to open the Scaling dropdown. I think the list of sizes in the dropdown depends on what Word gets from the printer driver, so it might not be possible to select a specific size in the general case. If you know all the PCs are using the same driver, you could probably add the necessary characters to the SendKeys string.

Sub PrintScaling()

   SendKeys "%z":Dialogs(wdDialogFilePrint).Show

End Sub

If you always want automatic scaling to the default paper size for your region (as set in the Windows control panel), you can go to File > Options > Advanced, scroll to the Print section, and check the box for "Scale content for A4 or 8.5 x 11" paper sizes". That's a set-once-for-all-documents option. However, see this thread for a problem that requires the option to be turned off.

*Except Word 2010 Starter Edition, the Word Web App, and probably Word for Windows RT or Word on iPad.

gary's student say: This can be done with an event macro.  Lets assume that we start with the worksheet password-protected with the password "obvious", but all the cells in column A unlocked.

Enter the following macro in the worksheet code area:

Private Sub Worksheet_Change(ByVal Target As Range)
Dim A As Range
Set A = Range("A:A")
If Intersect(Target, A) Is Nothing Then Exit Sub
ActiveSheet.Unprotect Password:="obvious"
Target.Locked = True
ActiveSheet.Protect Password:="obvious"
End Sub

Once a value has been entered in a cell in column A, the macro:

  • un-protects the sheet
  • locks the cell
  • re-protects the sheet

Because it is worksheet code, it is very easy to install and automatic to use:

   1. right-click the tab name near the bottom of the Excel window
   2. select View Code - this brings up a VBE window
   3. paste the stuff in and close the VBE window

If you have any concerns, first try it on a trial worksheet.

If you save the workbook, the macro will be saved with it.
If you are using a version of Excel later then 2003, you must save
the file as .xlsm rather than .xlsx

To remove the macro:

   1. bring up the VBE windows as above
   2. clear the code out
   3. close the VBE window

To learn more about macros in general, see:

http://www.mvps.org/dmcritchie/excel/getstarted.htm

To learn more about Event Macros (worksheet code), see:

http://www.mvps.org/dmcritchie/excel/event.htm

Macros must be enabled for this to work!

jay freedman say:

Doug's suggestion is a good one. I'll just answer the question as asked.

This macro assumes there are always exactly 18 sampser SEQ fields in the document. If the number of fields could change from one iteration to the next, some code would be needed to count the sampser fields (using another Find in a loop) and use that number to replace the hard-coded 18.

Sub RestartSeq()

    Dim rg As Range

    Dim strCode As String

    Dim lgStart As Long

    

    Set rg = ActiveDocument.Range

    rg.TextRetrievalMode.IncludeFieldCodes = True

    With rg.Find

        ' use wildcards because you don't know

        ' how many digits follow the \r

        .MatchWildcards = True

        ' locate the sampser SEQ field with an \r switch

        .Text = "sampser \\r[0-9]{1,}"

        If .Execute Then

            ' grab the matched text

            strCode = rg.Text

            ' extract the numeric part

            lgStart = CLng(Right(strCode, Len(strCode) - 10))

            ' add 18 to the numeric part

            strCode = Left(strCode, 10) & CStr(lgStart + 18)

            ' alter the field code

            rg.Text = strCode

            ' update all fields in the document

            ActiveDocument.Fields.Update

        End If

    End With

End Sub

H 0-15: Abs(Nz([HG1],0) Between 0 And 15)

+ Abs(Nz([HG10],0) Between 0 And 15)

Your issue with repeating columns is worse than I expected. Ideally each there should be up to ten HG records that you would sum in a query with an expression in a totals query like:
Abs(Sum(Nz(HG,0) Between 0 AND 15))

This all depends on what you are storing in your tables. I also don't like hard-coding values like 0 and 15 since they might change. I would hate to maintain a solution with hard-coded values in expressions that might require updating when business changed.
