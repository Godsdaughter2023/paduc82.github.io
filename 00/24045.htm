Sub Test()
  Dim LastK As Range, ThisK As Range, InsRow As Range
  Dim i As Long
  Dim StartDate As Date, LastDate As Date, StopDate As Date
 
  'Set the limits, StartDate must be > then Stopdate
  StopDate = Date
  StartDate = StopDate + 9
  'Find last filled cell in column K
  Set LastK = Range("K" & Rows.Count).End(xlUp)
  'Find the start
  For Each ThisK In Range("K5", LastK)
    If ThisK.Value2 >= StartDate Then
      Set LastK = ThisK
      Exit For
    End If
  Next

  'Get the last date
  LastDate = LastK.Value
  'From bottom to top
  For i = LastK.Row - 1 To 5 Step -1
    'Refer to the cell in column K
    Set ThisK = Range("K" & i)
    'Break?
    If ThisK.Value <= StopDate Then Exit For
    'Different date?
    If ThisK.Value <> LastDate Then
      'Remember this date for the next round
      LastDate = ThisK.Value
      'Insert a row below
      ThisK.Offset(1).EntireRow.Insert
      'Refer to the inserted row
      Set InsRow = ThisK.Offset(1).EntireRow
      'Refer to this row
      With InsRow
        'Do some formattings
        .RowHeight = 24.75
        'Note:
        '  The dot in front of Range means it refers to InsRow
        '  So "C1" refers incremental from the top left cell of InsRow
        With .Range("C1")
          With .Font
            .Bold = True
            .Name = "Arial"
            .Size = 18
          End With
          .NumberFormat = "[$-F800]dddd, mmmm dd, yyyy"
          .HorizontalAlignment = xlLeft
          'Get the date from the row below
          .Value = ThisK.Offset(2).Value
        End With
        .Range("C1:H1").Merge
      End With
    End If
  Next
End Sub

Correct. The macro calculates 2 dates at the start and it formats only the dates between.

When the dates are not sequential it is logic that the macro interrupts.

You can set StopDate and/or StartDate to different values to format a wider range.

An other solution is to use a Counter instead of a StopDate, so you can start anywhere (specified by StartDate) and go X steps up. See the code below.

Andreas.

Sub Test2()
  Dim LastK As Range, ThisK As Range, InsRow As Range
  Dim i As Long
  Dim StartDate As Date, LastDate As Date
  Dim StopCounter As Integer

  Const Steps = 9

  'Find last filled cell in column K
  Set LastK = Range("K" & Rows.Count).End(xlUp)
  'Find the date to start
  StartDate = Date + Steps
  For Each ThisK In Range("K5", LastK)
    If ThisK.Value2 >= StartDate Then
      Set LastK = ThisK
      Exit For
    End If
  Next

  'Get the last date
  LastDate = LastK.Value
  'From bottom to top
  For i = LastK.Row - 1 To 5 Step -1
    'Refer to the cell in column K
    Set ThisK = Range("K" & i)
    'Different date?
    If ThisK.Value <> LastDate Then
      'Remember this date for the next round
      LastDate = ThisK.Value
      'Insert a row below
      ThisK.Offset(1).EntireRow.Insert
      'Refer to the inserted row
      Set InsRow = ThisK.Offset(1).EntireRow
      'Refer to this row
      With InsRow
        'Do some formattings
        .RowHeight = 24.75
        'Note:
        '  The dot in front of Range means it refers to InsRow
        '  So "C1" refers incremental from the top left cell of InsRow
        With .Range("C1")
          With .Font
            .Bold = True
            .Name = "Arial"
            .Size = 18
          End With
          .NumberFormat = "[$-F800]dddd, mmmm dd, yyyy"
          .HorizontalAlignment = xlLeft
          'Get the date from the row below
          .Value = ThisK.Offset(2).Value
        End With
        .Range("C1:H1").Merge
      End With
      'Break?
      StopCounter = StopCounter + 1
      If StopCounter = Steps Then Exit For
    End If
  Next
End Sub

What are the advantages of Advanced Membership over Basic? - Microsoft Community

johnbetson say:

What are the advantages of Advanced Membership over Basic?

What are the advantages of Advanced Membership over Basic, in choosing between the I have not noticed a price for Advanced, I have afew Errors and have not noticed any changes, Would Advanced Help and how Much?
kosh vorlon say:

Hi John,

 

There is no price for SpyNet - it's included and is completely free (advanced or basic).  Advanced sends more information about the possible threat so that a cause and potentially a solution can be more easily developed by Microsoft and added to MSE to protect you the next time (or as soon as they can address it).  Part of SpyNet is also to supplement the definitions downloaded to your computer (you actually have a large number, but it is still a subset of the total available to Microsoft - including them all would overwhelm your system and slow it down).  If it detects something suspicious it checks its database for something related and, if applicable, downloads it to your system to deal with that threat (and thus your protection is improved).  Without SpyNet, you've crippled one of MSE's greatest advantages over many of its competitors.  As with all things, the more it knows about the threat, the better it is able to determine if it has something that will help - and Advanced Membership provides more information so you "may" be slightly better protected (but I'm not 100% certain about that - however, I am certain that it will better help Microsoft improve MSE so it protects you better and better as time goes on).  I recommend setting it on Advanced. 

 

If you are getting errors, then membership level is probably not going to change that - depending on the errors.  Troubleshooting problems is not part of its function.  Please provide the entire and exact word-for-word error messages with any codes so we can see what they refer to.  When do these errors occur (randomly as you work, at startup, during updates, during scheduled or manual scans, or when) and how often do they occur and is it always the same error or does it change (if it changes, try to provide a sampling).  It it not likely to be SpyNet-related unless you lost your internet connection (and even then, I'm not sure it would generate an error message).

 

Once we see and know more about the errors, we'll be able to provide better advice on how to proceed.  In the meantime, I suggest using Advanced not to deal with the errors but because it is the most helpful and possibly useful thing you can do (and it's free) and you absolutely do not want to turn it off without crippling its ability to protect your system.

 

I hope this helps.

 

Good luck!
