john w. vinson say:

There are several flavors of the Number datatype. The default is Long Integer, which is by definition a whole number. But you also have the choice of Single (a floating point number with about seven digits of precision and a range up to 10^38) or Double (a bigger floating point number with some 14 digits precision and a range up to 10^300 or so). You can also use a Currency datatype, which despite its name is just a number; it's a huge scaled integer with a exactly four (no more, no fewer) decimals and a range into the trillions.

For scientific measurements I'd probably suggest Double, using the builtin CDbl() function to convert your text.

Just be aware that floating point numbers are approximations - they're stored as a binary fraction, and (just as the fraction 1/7 cannot be represented exactly as a decimal number) they have roundoff error. If you add 0.1 + 0.1 + 0.1 the result will NOT be exactly equal to 0.3! If you know the limitations and work with them, though, Doubles are a good choice.
