a. user say:

I have 7 copies of gdiplus.dll in different folders.

One is for the FTP program I use, one is for MS Office, one is for a screen capture program I use and the other 4 are in different Windows Side by Side (c:\windows\WinSxS...) folders.  All of the gdiplus.dll files are made by Microsoft, but they are different versions underneath the installation folder for those programs.

I don't have Nero installed so can't look under there.

Files in the WinSxS folders are created and populated when you install various C++ Runtime Redistributables and contain runtime files and DLLs that some third party programs you installed may need, so you could have more, less or no WinSxS folders.  WinSxS is the technology that lets you have multiple copies of different versions of files with the same name on your system at the same time.

Some of mine go back as far as 2001, but that is not the date they were installed, it is the date of the file creation.  Some of mine are different sizes, dates and versions, but all belong to Microsoft (look at the Properties, Company and Product Name).  The program(s) that need to use the gdiplus.dll file will pull the copy they need from the right place at the right time when they need it.

If you would say where you discovered the mysterious gdiplus file (what is the complete folder path to the file), then you would know what program installed it and put it there.  I don't think it should just be hanging around flapping in the breeze someplace...

If you don't like it, rename it to something like gdiplus.xxx and if something needs it to get between the program and your display or printer and can't find it, I image that program would malfunction and hopefully complain about it missing.

a. user say:

Here is part of what the MS TechNet has to say about it:

hardware directly. Instead, GDI+ interacts with device drivers on behalf of applications. GDI+ is also supported by Microsoft Win64.


Here is what I say about it:

You may have several copies of gdiplus.dll on your system depending on what applications you have installed and what version of the Microsoft C++ Redistributes you have installed.  It doesn't mean you have to have any C++ development stuff installed, but some applications need to use the DLL to work properly.

Search your system and see where then are, look at the sizes, and the Version in the file Properties if you are curious.

MS Offices uses one version and you may have one in various c:\windows\WinSxS folders on your system for each C++ Redistributable library you have installed. 

Windows Side by Side (WinSxS) technology allows you to have multiple copies of different versions of a file with the same name on your system at the same time (in separate folders of course).  WinSxS technology helps relieve your system from a condition known as "DLL H**L"  (HE..LL) when there is a need to have different versions on the same file name on your system

When some one of you applications needs the gdiplus.dll file, it will pull the one it needs from the correct folder and use it.

It is usually included in the various C++ Redistributables, but other third party applications may install their own copy that they need in their own installation folder.  That one may or not be the same version as others on your system.
