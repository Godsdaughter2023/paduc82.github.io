Although this post is dated, it seems that nobody was able to provide a solution. In fact I've found several posts describing the same error which remain unsolved. I found a simple solution that worked (so far). This is exactly what I did:

I initially had other issues with WSR which I was trying to figure out. One thing I tried was a Clean Boot as described elsewhere in these forums. It should be easy to find instructions. After the clean boot I tried starting WSR. This is the first time I got the error message shown above. I did a Google search and found this article on devicemag.com  which gave a remarkably simple and (so far) effective solution. If you want to read the whole article, here's the link:

http://www.devicemag.com/2011/05/12/troubleshooting-windows-7-speech-recognition/

Here is the Cliff's Notes version with all the pertinent info:

Copy the following lines.
c:
cd\
cd windows
cd speech
cd commons

apisvr =SpeechUX 

Open Notepad and paste the text into a new document. Make sure that the commands are each on separate lines just like they appear above. Save the document to your desktop. Name it speech.txt

Exit Notepad. (It will not work if you skip this step as I learned)

Open Windows Explorer. Go to Organize>Folder and Search Options>View. Clear the check box for "Hide extensions for known filetypes."

Right-click on the speech.txt file and rename it speech.cmd

Right click and "Run as Administrator". 

Start Speech Recognition.

That's it. I hope this helps.
