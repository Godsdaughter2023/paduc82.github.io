    Workbook Link errors - Microsoft Community

    jastagliano say:

    Workbook Link errors

    I have a set of worksheets that are externally linked and I am constantly getting a #ref in a linked cell requiring me to open up several spreadsheets to get to the original cell to clear it.  Is there a reason this continues to occur.  Is there a certain number of "through" links that I should avoid.
    jim thomlinson say:

    Linked workbooks will break. It is just a matter of when and not if. Deleting cells, sheets or workbooks will cause errors. Changing sheet names in source books will cause errors. Inserting or deleting rows in source books will cause errors. Moving or renaming workbooks will cause errors. There is no easy way to handle these errors. I personally avoid linking workbooks as I find them far too unreliable.

    If you must link workbooks then do not use the IsError function as IsError will handle the #ref errors that you are bound to get causing incorrect result. Limit the users who can access the source books. Protect the source workbooks and worksheets. If you can, set the folder permissions where you keep  your source workbooks such that the files can not be renamed or deleted. Add comments to cells in your source workbook defining that they are linked and where they link to.

    HTH... Jim Thomlinson
