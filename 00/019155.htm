greg gray say:

How can I paste multiple addresses all at once from a spreadsheet?

I'm trying to paste a group of email addresses into BCC from Excel and outlook will only paste one at a time.
jamesmarwick say: FOUND THIS ELSEWHERE - IT HELPS!

The problem is that excel puts a Line Feed (LF) character after each email address. The LF character causes the To: field in outlook.com to ignore everything after the first line feed.

 

While there are many ways to solve this problem, one of the simpler methods might be:

  1. Ensure each email address ends in a semi-colon (;)
  2. Select the contiguous email address cells in the column
  3. Press Ctrl-C or right-click Copy
  4. Click on an empty cell anywhere on the spreadsheet or create a new temporary spreadsheet
  5. Right-click and select "Paste special"
  6. Select "All" and select "Transpose" then click "OK"
  7. The email addresses are now all in that cell reading across (the LF characters are gone)
  8. With the new cell still selected press Ctrl-C or right-click Copy
  9. Go to outlook.com and press Ctrl-V or right-click Paste.

The To: field should now have all your email addresses.

You can test to see if the file exists before trying to open it.

Let us say that you have a variable myFileName that holds the complete path to the possible file, so it might hold something like:

C:\Users\wulovejanus\20140701A.xlsx

You can test to see if 20140701A.xlsx actually exists like this:

If Dir$(myFileName) <> "" Then

  'open your file here, it does exist

Else

  'do not open the file, it does not exist

End If

If you want the reverse logic, then

If Dir$(myFileName) = "" Then

  'do not open it, it does not exist

Else

  'open the file, it does exist

End If

Of course, if you don't need the "Else" portion, you can just leave it out of things.

andreas killer say:
Right-click on the sheet tab
Choose "View Code"
Within the menu click Insert \ Module
Paste this code: 

Function ExtractTextNumber(ByVal Where As Range) As String
  Dim i As Long, j As Long
  Dim S As String
  Dim Temp
  Dim HasNumber As Boolean, HasChar As Boolean
  Temp = Split(Where(1).Value)
  For i = 0 To UBound(Temp)
    HasNumber = False
    HasChar = False
    For j = 1 To Len(Temp(i))
      If Mid$(Temp(i), j, 1) Like "#" Then
        HasNumber = True
      Else
        HasChar = True
      End If
      If HasChar And HasNumber Then
        ExtractTextNumber = Temp(i)
        Exit Function
      End If
    Next
  Next
End Function


Close the VBA editor
Apply the formula:

A2: =ExtractTextNumber(A1)

Andreas.

I eventually managed a regex solution

Function NumString(strIn As String) As String
    Dim objRegex
    Set objRegex = CreateObject("vbscript.regexp")
    With objRegex
        .Global = True
        .Pattern = "[a-zA-Z]*[\d]+[a-zA-Z]*"
        If .test(strIn) Then
            NumString = .Execute(strIn)(0)
        Else
            NumString = "Not found"
        End If
    End With
End Function

A2: =NumString(A1)

andreas killer say:

Can I ask how does the (1) in Temp = Split(Where(1).Value) work?

Hi Murray,

the (1) after Where is just there to make sure that the Value property returns only one cell. The same in other syntax is: Where.Cells(1)

If someone calls the UDF like =ExtractTextNumber(A1:B1) the Value property of the Range object returns an array instead of the contents of the first cell (and we get an error).

Andreas.
