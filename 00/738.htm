steve rindsberg say:

>> But, I don't want to insert a slide within the stack since that will mess up VBA programming I am using.

Because your other code depends on going to slides based on SlideIndex?

There are several ways of getting around that, if so.

For example, you could use SlideIDs.  When PPT creates a new slide, it assigns a unique SlideID to it. The SlideID never changes, no matter where in the presentation you move the slide.

So starting with Dave's:

Application.SlideShowWindows(1).View.GotoSlide Index:=1

you'd change it to

Application.SlideShowWindows(1).View.GotoSlide _

  Index := ActivePresentation.Slides.FindBySlideID(x).SlideIndex

where x is the SlideID of the slide you want to jump to.

To learn the SlideID of a given slide while setting this up, select the slide then run this in the Immediate window of the IDE:

? Activewindow.Selection.SlideRange(1).SlideID

Sub deleteHPageBreaks()
   Dim i       As Integer
   Dim myWS    As String
   Dim oldView As Variant

   myWS = "Sheet1"

   'the easy way (resets VPageBreaks, too)
   'Worksheets(myWS).ResetAllPageBreaks: Exit Sub

   'the hard way
   oldView = ActiveWindow.View
   ActiveWindow.View = xlPageBreakPreview

   With Worksheets(myWS).HPageBreaks
      For i = .Count To 1 Step -1
         If .Item(i).Type = xlPageBreakManual Then .Item(i).Delete
      Next
   End With

   ActiveWindow.View = oldView
End Sub

doug robbins - mvp office apps & services (word) say:

The procedure for applying protection to a document via the SaveAs>Tools>General Options dialog has NOT changed from either Office 2007 to 2013 or from earlier versions of Windows to Windows 8.1

However, rather than distributing a document for your users to open and then use SaveAs, the document should be supplied to the users in the form of a template that they use by using File>New and then selecting the template as the basis of the document that they want to use.

jay freedman say:

To answer the question directly: The Save As dialog in Word 2013 has exactly the same path Tools > General Options > Password to modify. To get to that dialog, you can use the F12 key instead of the ridiculous File > SaveAs > Computer > Browse path.

None of the stuff in the Protect Document menu can apply that password.

BUT! You are misusing Word by providing a read-only  document as a "template". The design intention is for you to provide a template -- a *.dotx file -- from which users create new documents. In that arrangement, the template will not be changed when they edit the document, even though the template is not protected.

The danger in using a document as a fake template is that it's far too easy, even for expert users, to press Ctrl+S without thinking and overwrite the "template" with the changed document. Because Word isn't intended to be used that way, it doesn't provide good tools for protecting the "template" document.

Please read http://shaunakelly.com/word/templates/templaterelations.html to understand how this works.
