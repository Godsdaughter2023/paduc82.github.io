¡firedog say:
It all depends on what encoding the sender uses. You'll probably find the following settings fit most situations. Press Ctrl-Shift-O for Options. On the Read tab,

Click the Fonts button and select fonts for at least Unicode (encoding Unicode UTF-8), User defined and Western European (encoding Western European). Select Western European as default. 
Click the International settings button and deselect Use default encoding for all incoming messages.

With these settings, WLMail will use the encoding given in the headers of a received message. If there is no charset= header, WLMail will make a guess based on a scan of the first KB or so of the message. If it guesses wrong, you can change the encoding used to read the message using the button on the ribbon.

To find your program version, in WLMail press Alt-H A. If this doesn't show you the version and build number, press Escape twice, then Alt-F B.
