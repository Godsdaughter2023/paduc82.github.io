I'm talking about regenerating tasks, not recurring tasks. I mentioned both in my first sentence to differentiate the two.

As I previously mentioned, Outlook supports both recurring tasks and regenerating tasks, but synchronizes only the recurring tasks. Here's the difference between a recurring task and a regenerating task. Let's say we have two weekly tasks that are both due on Tuesday but one is recurring and the other is regenerating. Now, let's say we complete both the recurring task and the regenerating task on Friday. Outlook would then create a new recurring task on the next Tuesday, which is 7 days after the task due date. But Outlook would create a new regenerating task on the next Friday, which is 7 days after the task completion.

john visio mvp say:
Unfortunately, the world of Visio starts in the bottom left corner.
John... the original Visio MVP (since 1993) Visio.MVPs.org

barb way say:

John speaks truth, as usual. Visio considers the 0,0 point to be the lower left hand corner.  You can manually add a margin of your own at the bottom by extending the page downward and leaving the area blank, for instance.  Not a very intuitive work around, but can work.

Note that in Print Preview mode you should see it as it will print, not at the dummy view in the Page Setup screen shows it.

Barb

john visio mvp say:
Alternatively, you could put a shape in the bottom left corner to force the drawing to the top right. The shape can be placed on a layer that does not print, but is still visble to move around.

fredbloggsxiii say: I know it's an old thread but I was having this issue and the shape at the bottom 'fixed' it.  I made it all transparent so it doesn't show in the print.

fatsimon say: With the 365 version of Visio, you can choose to centre the drawing via "Print >page setup" (works like Excel)
