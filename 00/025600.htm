suzanne s. barnhill say:

You can, of course, use the styles anyway you like, but there are several groups of styles whose names indicate how they're intended to be used.

Some of them are used automatically. For example, the document header will be in the Header style and the footer in Footer style. If you display tracked changes in Balloons, they will be in Balloon Text style. Comment Text is used for comments. Envelope Return and Envelope Address are used for the return and delivery addresses on envelopes. The Caption style is used for captions and the TOC series of styles for TOC entries and the Index series for index entries. Hyperlink and FollowedHyperlink are also applied automatically, as are Footnote/Endnote Reference and Footnote/Endnote Text and Page Number.

Others are more discretionary:

There is a series of Body Text styles: Body Text, Body Text 2, and Body Text 3; Body Text First Indent, BTFI 2, Body Text Indent (with 2 and 3). Their default formatting is "interesting," shall we say. I would rarely find a use for more than Body Text (flush left to follow left-aligned headings), Body Text First Indent (for other text paragraphs), and Body Text Indent (for text with just a left indent).

Block Text is intended for block quotations ("extracts"). It is indented on both sides, though the indent is larger than needed in all versions of Word, and in Ribbon versions it has a border that you wouldn't want in a serious document.

There are several types of list styles, a numbered series (List Number), a bulleted series (List Bullet), and a series with a hanging indent (List) that's intended for lists where you will type the number manually. The List Continue series have left indents to match the hanging indents of the corresponding List, List Number, and List Bullet styles; they're used for unnumbered/unbulleted paragraphs in a list item. When you are applying numbering to headings, you can choose one of the options in the List Gallery that shows, for example, 1 Heading 1, 1.1 Heading 2, 1.1.1 Heading 3, and so on, but you can also have a hierarchy of body text styles if you link your numbering to say, List Number, List Number, 2, List Number 3, etc.

The built-in heading styles are especially useful. Many users avoid using them because they don't like the default formatting, but of course, like any style, they can be modified to the user's preference, and they have "magic" qualities that make them preferable to user-defined styles for headings (see https://shaunakelly.com/word/numbering/usebuiltinheadingstyles.html).

Styles like Title and Subtitle are pretty clearly named, but you don't have to use them for just that purpose. Date, Closing, and Signature styles are useful in letters (I've never understood why there's not a built-in Inside Address style; I always define one).

There's a whole series of HTML styles that presumably are for Web pages, and of course there are table styles (which aren't listed in the Styles pane but can be seen in the Style dropdown if you add it to the QAT).

Does this help?

This has been the case with all versions of Excel.

 

Chip Pearson has a couple of macros that you can use to unselect a cell or groups of cells.

 

Sub UnSelectActiveCell()
Dim rng As Range
Dim FullRange As Range
'Chip Pearson
If Selection.Cells.Count > 1 Then
    For Each rng In Selection.Cells
        If rng.Address <> ActiveCell.Address Then
            If FullRange Is Nothing Then
                Set FullRange = rng
            Else
                Set FullRange = Application.Union(FullRange, rng)
            End If
        End If
    Next rng

    If FullRange.Cells.Count > 0 Then
        FullRange.Select
    End If
End If

End Sub


Sub UnSelectActiveArea()
'Chip Pearson
Dim rng As Range
Dim FullRange As Range
Dim Ndx As Integer
If Selection.Areas.Count > 1 Then
    For Each rng In Selection.Areas
        If Application.Intersect(ActiveCell, rng) Is Nothing Then
            If FullRange Is Nothing Then
                Set FullRange = rng
            Else
                Set FullRange = Application.Union(FullRange, rng)
            End If
        End If
    Next rng
    FullRange.Select
End If

End Sub

Your Row Source has only 2 columns: ID and Field_Value, so Bound Column cannot be 3. Set it to 1 instead, so that the ID column specifies the value of the combo box.
