bob jones aka: cybertaz say:

Labels in Word are generated as a Table, so the Gridlines command you want is the one that pertains to the Table of labels.

Click in the Table, then go to the Layout contextual tab & click the View Gridlines button on the left end of the Ribbon. Optionally, select Gridlines from the Table menu.

The Gridlines referred to on the View tab are for drawing/alignment purposes. Text Boundaries also do not pertain specifically to Tables, although they do cause the Table Gridlines to be displayed if that preference is enabled.

scott mcdaniel say:

you can't use an embedded subform

You can use embedded subforms if they're all Datasheet subforms. I forget how many "levels" you can go, but I seem to recall going at least 2 deep.

In my opinion, you're much better off using a more traditional Form/Subform design. If you create standard Forms for all those tables, you could then embed them into their upstream parent, and you'd end up with a single form that had the parent data, and a subform that contained the Child data. The subform would itself have a Subform, and so forth. 

While datasheets are quite easy, they also lack the robust event model available with Forms. Eventually you'll run into something that can't be done easily (or at all) with datasheets and subdatasheets, and you'll have to switch to a different UI design. Best to do that from the start.
