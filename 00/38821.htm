It's my understanding that even though I am an end user, I should have no problem getting it registered/validated as long as I replace the motherboard with the same make/model that came in the system originally.  Thanks in advance for your replies.

hairyfool say:

Replacing a motherboard with like-for-like should have no issues either with software, drivers etc. although you will probably have to revalidate.

Problems would only arise if the new hardware is significantly different from the original, in particular with the board chipset. That being said however in many cases you should be able to boot and install any required drivers during the initial startup. The one exception is the drive interface as this can generate a blue screen INACCESIBLE_BOOT_DEVICE and the only way around that is to pre-install the new drivers on the old system before the change although if the reason for the change is a dead motherboard that does not help.

I have once replaced a mobo with a failed IDE port by plugging in a PCI one which allowed it to boot, install the drivers for the card, install the drivers for the new board and then replaced the board.

In terms of the licensing issues it is a moot point about how much you can replace before it ceases to be a repair so invalidates the EULA. I did see somebody claim that as he could not get a direct replacement mobo a generic one would not fit the branded case and so he "replaced" the case as well. In other word all he did was take out the HDD and put it in a new clone system. Not allowed under the EULA.
