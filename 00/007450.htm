palcouk say:

A mail account configured as pop by default downloads all mail from the server, unless in the account setup it was set to leave a copy on the server.

By default the mail setup wizard in outlook 2013 configures the mail account as IMAP, which is a copy of what is on the server

You should be very wary of setting up your mail account in a mobile/other device, as a pop account as it will download all mail from the server, unless you have remembered to tick the option to leave a copy on the server.

A mail account configured as pop will not be in sync on multiple devices, sent mail will only appear on the device on which it was sent.

Contacts/Calender/Tasks will only appear on the local device and not be in sync for any other device

Office 365 home is not licensed for business use and doesn't feature a full exchange mail connection

bob jones aka: cybertaz say:

In Excel 2016 for Mac, the default measurement for Column Width has been [IMO] dumbed down to comply with that used in the Windows version - Characters. If you to Format> Columns - Width & enter the value 25 the width of the column will be set to 25 characters. If you use the column separator to drag the column width, a tip box displays showing 2 values: The first value is the number of characters, the second (in parentheses) is the number of pixels.

I consider this as being dumbed down because 'characters' is an arbitrary, archaic system. It originated when Excel was introduced to the PC environment at a time when only fixed-with fonts were available & choice of font/size as well as formatting was severely limited on DOS/Windows PCs. Also, it's predicated on the Default font specification in Excel Preferences. In today's software you can, for example, set the Column Width to 25 characters based on the default font/size of 12 pt. Calibri. If you then change the font size to anything larger than 12 pt or choose a font which is based on wider characters or change the formatting of the cells in most any way there is no way the cell can continue to display 25 characters without increasing the column width. IOW, a column set to 25 characters may be able to display only 11 characters - and that can vary in the same column based on which characters are contained in the cell - a W being wider than an i, bold being wider than non-bold, etc.

Pixel is not a fixed value, either. It varies based on the resolution of the display on which the work is being done. For most users it is a virtually meaningless measurement.

Unfortunately, Microsoft decided to revert Mac Excel to the Dark Ages rather than redesign Win Excel based on contemporary standards. These arbitrary values contribute to the disparity that occurs when the same workbook is displayed on the 2 separate platforms in prior versions & continues to do so.

Previous versions of Excel for Mac measured Column Width in hundredths of an inch. Fortunately, the Inch system still can be used by entering the value as 1.5"
