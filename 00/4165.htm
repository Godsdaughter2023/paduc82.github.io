justin dunn say:

Bug Report: Show Libraries Function on Windows 8.1 Preview 64-bit, Windows 8.1 RTM 64-bit and Windows 10 TP 64-bit

I have copied my message from Windows 8.1 Preview Forum - Bug Report: Show Libraries Function on Windows 8.1 Preview 64-bit to this new topic as the bug still applies to the latest build of Windows 10 TP.

In the Windows 8.1 Preview 64-bit (along with Windows 8.1 RTM 64-bit and Windows 10 TP 64-bit) version the "Show Libraries" function setting is stored in:
For 64-bit Software:
Registry Key: HKEY_CURRENT_USER\Software\Classes\CLSID\{031E4825-7B94-4dc3-B131-E946B44C8DD5}
Value Name: System.IsPinnedToNameSpaceTree
For 32-bit Software:
Registry Key: HKEY_CURRENT_USER\Software\Classes\Wow6432Node\CLSID\{031E4825-7B94-4dc3-B131-E946B44C8DD5}
Value Name: System.IsPinnedToNameSpaceTree

When you use the "Show Libraries" function, the value used depends on weather the software is 64-bit.
For example, in File Explorer (a 64-bit program) you can turn on the "Show Libraries" function and you will see Libraries displayed in File Explorer and in the Open/Save Dialogs of other 64-bit programs.
In 32-bit software like Mozilla Firefox, when you use a function that shows a Shell Browse window (like "File -> Open File" and "File -> Save Page As") you will not see Libraries in the Navigation Pane because it's a 32-bit program and the setting was only set for 64-bit programs.
You can right click on the Navigation Pane and turn on the "Show Libraries" function in the 32-bit program but that setting will only apply to 32-bit programs.

It is confusing and doesn't make sense to have the "Show Libraries" function setting be dependent on weather you are using a 64-bit program or a 32-bit program.
The "Show Libraries" function should set both values for a consistent User Interface between 64-bit and 32-bit programs.

I found a workaround to this problem by using the regln-x64 utility (http://www.tenox.net/out/#regln).
Run the Command (for each user):
regln-x64.exe "HKEY_USERS\%USER_SID%_Classes\Wow6432Node\CLSID\{031E4825-7B94-4dc3-B131-E946B44C8DD5}" "HKEY_USERS\%USER_SID%_Classes\CLSID\{031E4825-7B94-4dc3-B131-E946B44C8DD5}"
Replacing %USER_SID% with the SID of the user.
This workaround also reveals another problem with the way this setting is implemented, which is that the CLASSES hive is local only so those using roaming profiles will need to change this setting on each computer they login into which is not a problem with other settings like "Show favorites" that are stored in the main registry hive.

garrington66 say:

- a chapter first page on the right side

- no header on the first page

- page numbering in the footer, starting on first page

But:

If I add page numbers, they only get added to the first page of each chapter.

If I untick the 'different first page' box, the header appears on the first page.

So, how do I set it up to get no header on first page, but numbering starting in the footer from that page, throughout the chapter?

bob jones aka: cybertaz say:

First of all Do Not create any new Section Breaks.

Your issue is this: Since you don't want a Header on the 1st page (of each Section?) you checked the box for Different First Page. That's fine, but it applies to the Footer as well. You then went into the First Page Footer, itself, to insert page numbers so that's what you got... numbers on the first page only.

From that point what you needed to do was go to the Footer of one of the other pages in each section that didn't display page numbers, then use Insert> Page Numbers being sure that the Show numbers on first page was not checked - ticking that check box would override the Different First Page property. Repeat for additional Sections.

IOW, when you use Different First Page you need to insert anything that is to be the same into both components of the Section.

gord dibben say:

You can use a user defined function which ignores blank cells so does not produce extra commas for missing data.

Function ConCatRange(CellBlock As Range) As String
Dim cell As Range
Dim sbuf As String
    For Each cell In CellBlock
    On Error GoTo fred
    If Len(cell.text) > 0 Then sbuf = sbuf & cell.text & ","

'if want comma/space then change  ","  to ", "

        Next
    ConCatRange = Left(sbuf, Len(sbuf) - 2)
fred:
End Function

Copy and paste to a General module in your workbook.

usage is  =ConCatRange(A1:A10)

for non-contiguous ranges use double parens   =ConCatRange((A1:A10,C2,E4,B2:B5))
