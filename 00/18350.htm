scottgem say:
As Butch alludes to, your question indicates that you are not familiar with good database design practices. Excel is a flat file system and Access is a relational database, so you need to change your thinking from Excel. 

To create a unique partnumber, use an Autonumber datatype. This will guarantee that you have a unique ID. However, this ID may or may not be visible within the application. If you want to have an abbreviated code to identify an item and want to use the combination of characteristics that you describe, that can be done, but not in the way you are thinking.

You start off with creating two lookup tables for materials and types. They would look like this:

tluMaterial
MaterialID (Primary Key (Autonumber)
Material
Abbreviation

tluType
TypeID (PK Autonumber)
Type
Abbreviation

Now you products table would look like this:

tblProduct
ProductID PK Autonumber)
MaterialID (Foreign Key)
TypeID (FK)
other info about the product. 

The Foreign Keys would be populated using comboboxes on a form that select from the lookup tables. 

Now, if you want to display a Product code you could do so in a query. The query would join the Products table with the 2 lookup tables and add a column like:

ProductCode: tluMaterial.Abbreviation & tluType.Abbreviation & Format (ProductID,"0000")

Try the following. No need to select the cells; just fully reference the cells/ranges.

Note that the space and underscore at the end of a line is a line break in an otherwise single line of code. (After AutoFill)

Dim lastrow As Long

    lastrow = Worksheets("Sheet1").Range("D3").End(xlDown).Row
   
    With Worksheets("Sheet1")
        .Cells(.Rows.Count, "C").End(xlUp).AutoFill _
           Destination:=Range(.Cells(.Rows.Count, "C").End(xlUp), .Cells(lastrow, "C"))
    End With

End Sub

    b. Second drop-down menu - choose contains
    c. Third field - input the word you would like to block
