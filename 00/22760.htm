rick rothstein, mvp - excel say:

Here is a solution where you can keep the reference in the same cell as the entry takes place. This is event code (see below on how to install it) and what it does is physically change any value that is not a digit 0 to 5 to the number 3 and then Custom Formats the cell to display the text that was entered. This way, the cell's value (for non-digits 0 to 5) is really 3 (the Formula Bar will show 3), but the cell will display the value entered by the user. The one thing you have to do with the code, though, is change the value being assigned to the MonitoredCells constant (the Const statement in the top part of the code) to the cell or cell addresses you want to have this functionality. In my code, I set the range of cells to be monitored to A1:A10, but you can change this to any valid cell range.

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim Cell As Range, Temp As String
  Const MonitoredCells = "A1:A10"
  For Each Cell In Intersect(Target, Range(MonitoredCells))
    Cell.NumberFormat = "General"
    If Len(Cell.Value) Then
      Temp = Cell.Value
      If Not Cell.Value Like "[0-5]" Then
        Application.EnableEvents = False
        Cell.Value = 3
        Application.EnableEvents = True
        Cell.NumberFormat = """" & Temp & """"
      End If
    End If
  Next
End Sub

One thing to note with this method... if the user types in a number that is not 0 to 5 (for example, 6, 12, 456, etc.), that number will display in the cell but any reference to that cell will see the cell value as 3... personally, I think that could be confusing, so I decided to point it out to you... just in case.

 

HOW TO INSTALL Event Code
------------------------------------
If you are new to event code procedures, they are easy to install. To install it, right-click the name tab at the bottom of the worksheet that is to have the functionality to be provided by the event code and select "View Code" from the popup menu that appears. This will open up the code window for that worksheet. Copy/Paste the event code into that code window. That's it... the code will now operate automatically when its particular event procedure is raised by an action you take on the worksheet itself.

rich michaels - mvp (greatcirclelearning.com) say:
Hi PW39,

First and foremost, VBA UserForms in Office 2011 STINK, that's a technical term, :-) and they are really not compatible with UserForms on Windows VBA. Maybe I should qualify that a bit more. There is a level of compatibility but it is not a "developer friendly" level and a bunch of formatting work must be done if you are moving UserForms between platforms and you want them to look right or "as right as they can be."

Now that I have that off my chest... :-)

Images on Office 2011 UserForms can only be BMP formats so you will need to re-save that JPG image into BMP. You can do that with any number of Paint or Graphic programs. If you use MS Paint on the PC just open your JPG there and re-save it as a BMP and it will then import fine onto your Mac UserForm.

Regarding timed displays of UserForms on Office 2011 you can use the following code. There may be other ways to do this, but it was the way I figured out and it works, so I use it.

Module Code to Launch the UserForm:

Option Explicit
Dim UserForm1 As UserForm1
Sub OpenUserForm()
    Set UserForm1 = New UserForm1
    UserForm1.Show
    Set UserForm1 = Nothing
End Sub

Forms Code (inserted in UserForm1) that displays the form and auto closes it after 5 seconds:

Option Explicit
Private Sub UserForm_Initialize()
    'must have this because it runs first and displays the form
End Sub
Private Sub UserForm_Activate()
    'this routine auto executes next following the form's initialization & display
    Application.Wait (Now + TimeValue("0:00:05"))
    Unload Me
End Sub

The trick is having both the Initialize and Activate routines in your user form code. Obviously the form will display automatically if you only have either one of the routines (or neither of them) in your form's code. However, you have to have both in your code if you want to display the form as a splash screen and auto close it after an amount of time has passed.

Hope this helps
