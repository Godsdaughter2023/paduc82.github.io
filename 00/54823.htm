lemp say:

It's my understanding that opening a "help" function when the F1 key is pressed is a convention, rather than a global fact of life in Windows.

That is, it's the responsibility of whatever application program is running to recognize that the F1 key has been pressed and then to do whatever the developers of that program have determined is appropriate.  This way, you get help that's appropriate to the particular program rather than generic Windows help (which you can get by pressing F1 together with the "Windows Logo" key).

When you say that you "have a program needing to be used in an XP Shell on a Windows 7," are you referring to using XP Mode on Windows 7 Pro or are you using something else as the "XP Shell"?

What does happen when you press F1 while running your application program in your "XP Shell"?

It is labeled MEMORY.DMP. The difference between Small Memory Dumps and Kernel-Dumps in the simplest definition is a Kernel-Dump contains much more information at the time of the crash, therefore allowing further debugging of your issue. If your upload speed permits it, and you aren't going against any strict bandwidth and/or usage caps, etc, the Kernel-Dump is the best choice. Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.
