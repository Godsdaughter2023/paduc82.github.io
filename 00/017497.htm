Understanding Connection Filtering - Microsoft Community

osamamansoor5807 say:

Understanding Connection Filtering

We have created a transport rule for blocking certain users to recv incoming & outgoing email. we have some applications which uses some real IP's to send email on our all users.so I just enter these IP's on my transport rule as exception one of my colleague told me that you just enter these IP's on connection filtering so these IP's are automatically part of inside organization as inside organization is allowed for that block users

alexsun_2012 say:

Hi Osama,

In Office 365, connection filter works at the front of the Exchange Online Protection (EOP) services. When someone sends a message to your Office 365 mailbox, the incoming message initially passes through connection filtering, which checks the sender’s reputation and inspects the message for malware. When you add the IPs as safe in connection filter, the email will bypass the connection filter.

After that, the message continue through your organization policy filtering like your custom transport rules. If you have set up any transport rules that blocks the message, it will not arrive to the destination mailbox.

Back to your situation, we suggest you to add the IPs in the connection filter and make sure no transport rules blocks the emails. If anything is unclear or you run into any problems, please let us know.

Thanks,

Alex

larryyu_msft3110 say:

Hi Osama,

I'd like to confirm if the IP address has been defined as “block” in the connection filter. If yes, then the emails won’t be received.

As Alex said above, if you add the IPs to allow list in the connection filter and the IPs are not in the block list in mail flow, users are able to receive emails. If you add the IPs to block list in the connection filter, emails from those IPs will be intercepted. No matter whether the IPs are in the block list in mail flow or not, users are unable to receive emails.

al edlund say:

bruce,

To start with it might be helpful to understand how a shape get's from the stencil to the page. First (if it is a new shape) it is copied to the 'document stencil' and from there a copy is put onto the page (there's a couple of reasons this flow happens). You get to the 'document stencil' from the 'more shapes'. The reason this is important is that if you make a change in the "document stencil" it is (almost always) copied into the shapes you have already dropped on the document. If you make a change to the 'root' stencil, they are not. If you're going to customize your stencil first make a copy of it (rename the original) and then work on the copy.
