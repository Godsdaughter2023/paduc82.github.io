ken sheridan say: To insert a set of rows with data from a referenced table into a table which models a many-to-many relationship type between a parent form's table and another table requires that the primary key of the parent form's current record, and the primary key of each row in the other table are inserted into columns in the table which models the relationship type.  Say for instance the parent form is based on a Respondents table and the subform is based on a Responses table which models the relationship type between the Respondents table and a Questions table, the Responses table would structured be along these lines:

Responses
….RepsondentID  (FK)
….QuestionID  (FK)
….AnswerID  (FK)

In the AfterInsert event procedure of the parent (respondents) form values would be inserted into the RepsondentID  and QuestionID columns of the above table as follows, leaving the respondent to select the answer to each question in a combo box in each row of the subform:

    Dim strSQL As String

    strSQL = "INSERT INTO Responses(RespondentID,QuestionID) " & _
        "SELECT " & Me.RespondentID & ",QuestionID " & _
        "FROM Questions"

    CurrentDb.Execute strSQL, dbFailOnError

    Me.sfcResponses.Requery

The subform would be housed in the subform control sfcResponses, linked to the parent form by setting its LinkMasterFields and LinkChildFields properties both to RespondentID.  Requerying bthe subform after executing the INSERT INTO statement will reload its recordset, thus showing the newly entered rows in the subform.

what are the options for keeping home and work "outlook info" - Microsoft Community

rmdeq say:

what are the options for keeping home and work "outlook info" synchronized and separate?

Let's define "outlook info" as Email, Calendar, Contacts, Notes and Tasks. A person may have "home outlook info" (ie, "home email", "home calendar", "home contacts", "home notes", and "home tasks") and may have "work outlook info" (ie, same as preceding list with "work" instead of "home").

 

Some (very few) folks may choose to have no outlook info on their phone. Some folks may only want "home outlook info" on their phone, or their work may forbid their device from having "work outlook info". Some folks may get their phone through work and want ONLY "work outlook info" on the phone. These three cases in the truth table seem easy to manage through different synchronization techniques (eg, none, outlook connector, Office365). The interesting case is the fourth case, where someone wants to have both "home" and "work" outlook info on their phone.

 

Email seems to have been designed to be easy to separate. Different accounts are really either "home" or "work". When you create a new email, you usually get to select the account it will come from.

 

Calendar seems more complicated than email. WP7 does not ask when a new calendar item is created "What calendar would you like to create this in?" (when someone has multiple [overlay] calendars. So, if you need to invite your boss to a meeting to discuss a confidential topic, there's a chance the meeting could be created in your Windows Live calendar instead of your Outlook (ie, exchange) hosted calendar. On WP7 at least, it seems that the calendar your appointments are created in is the first writeable calendar in your list of calendars. Conversely, if you manage to get "Outlook" listed above "Windows Live" somehow, there's a chance that your employer learns personal information about you when you create an appointment that is intended to be personal.

 

Contacts, Notes and Tasks seem generally similar to each other, and a little bit like Calendar items. Maybe the only difference is that being double scheduled can be a disaster, but having the same contact in work and personal - if they belong in your work and personal lives - is probably not too bad.

 

In my case, work has adopted Office365. All my home contacts sync via USB cable between my WM6.5 phone and Outlook 2010 on my home computer. It's time to move these home contacts to somewhere online, but I don't want to move them to the Office365 server because I don't want to lose everything if I'm cut from that server (which has happened before). So, I guess I have to sync them with Hotmail Connector between Outlook 2010 on my home computer and Windows Live (or Hotmail or outlook.com). If I dump the WM6.5 phone for a newer WP, can I get my home contacts there? When I create a new Contact on the WP7 or WP8 phone, will it go to Office365 or outlook.com? Do I get to choose?

 

Is there a way to choose which calendar to create an appointment in (Office365 or Outlook.com)? In the case of meeting invitations, can the calendar account be based on the email account I received the invitation on?

 

Sorry if this question is more of an ecosystem question than particularly WP. The general proposition is that people have home lives and work lives and sometimes information from one should be carefully managed to not leak into the other.

julie fka profjulie say:
  • Windows Phone does not sync outlook-related notes at all. Instead it syncs notes that are included in OneNote notebooks.
  • Windows Phone syncs outlook related data (appointments, contacts & tasks) from an Office 365 account, Windows Live and Outlook.com accounts
  • Outlook 2010 syncs directly to Office 365
  • Outlook 2010 does not sync directly to Windows Live or Outlook.com. You need to use the Outlook Hotmail Connector (an add-in to Outlook) to sync Outlook 2010 data to Windows Live or Outlook.com.
  • The Outlook Hotmail Connector only syncs appointments and contacts.  It does not handle tasks at all.
  • Outlook 2013 syncs contacts, appointments AND tasks directly to Windows Live and/or Outlook.com (the Outlook Hotmail Connector is not needed if you use Outlook 2013)


So given your personal scenario, you can continue using Outlook 2010 to manage your work and personal data by:

  • Syncing work contacts, appointments and tasks from your Outlook 365 account directly to a Windows Phone
  • Syncing personal contacts & appointments from Office 2010 via the Outlook Hotmail Connector to your personal windows live/outlook.com account.  Then set the phone up to sync to this windows live/outlook.com account.
  • If you also want to sync personal tasks to windows live/outlook.com, you'll need to upgrade to Outlook 2013
  • You'll need to move the Outlook related notes that you want to sync to windows phone to a OneNote Notebook

On Windows Phone, when you add new contacts, appointments or tasks, you will be prompted to choose the account you want to add this new data to.  You can choose between your Office 365 or Windows Live/Outlook.com account.
