ken sheridan say: Unless you are going to set up a separate LastNames table in advance with one row for every name you can think of, a very simple way of enabling the selection of a name in your form would be to build up the list as each name is added, so if you enter Brown then the list will show Brown; if you then enter Green in the next record the list will show Brown and Green and so on.  The list will always represent every name currently in the table, so if you need to enter Brown or Green again you can select it from the list; if you need to enter White, but have no Whites yet in the table, you type it in.

To do this set the RowSource of the combo boxes for both the member and spouse last name fields to:

SELECT MemberLastName
FROM Members
UNION
SELECT SpouseLastName
FROM Members
ORDER BY MemberLastName;

Make sure the combo boxes' LimitToList property is in each case False (No).

In the AfterUpdate event procedure of the form requery both combo boxes with:

    Me.[MemberLastName].Requery
    Me.[SpouseLastName].Requery

To carry the value entered in the main member's last name combo box to their spouse's last name combo box you'd need code like this:

    Me.[SpouseLastName] = Me.[MemberLastName]

Where you put this code is up to you.  It could be in the MemberLastName combo box's AfterUpdate event procedure (after the above code) for instance to automatically pass the value when the MemberLastName  is entered or selected, or it could be in the Click event procedure of a small button suitably positioned beside the SpouseLastName control for instance to allow the user to do it electively.

Note that I have used column names MemberLastName and SpouseLastName which are specific descriptors of the attributes being represented in each case, and which do not contain spaces.  Both are design points I'd recommend.
