scottgem say:
What do you need to edit? If this is a report, then output to RTF instead of PDF. Otherwise it doesn't change anything.

Dim db As Database
Dim rs As Recordset
Dim strFilename As String

Set db = CurrentDB()
Set rs = db.OpenRecordset("queryname")

rs.MoveFirst
Do While NOT rs.EOF
    strFilename =  rs.Fields("[BLOCK_TYPE]")&"=”& rs.Fields(["BANNER_CODE]") &”_”&rs.Fields("[BLOCK_DESC]")&”.rtf"
    Me.StudentID = rs.fields("StudentID")
    DoCmd.OutputTo acOutputReport, "reportname", acRTF, strFilename
    rs.MoveNext
Loop


You will also need to add a textbox on the form where you kick this off, and change the Recordsource of the report to a query that has criteria for the student ID of:
Forms!formname!studentID

So each pass through the loop outputs a report just for that student.

What is the switch for a subscript with the EQ field,  ie the ? in { EQ (x\?(1))} to create x with 1 as a subscript?
jeeped say:
What is the switch for a subscript with the EQ field,  ie the ? in { EQ (x\?(1))} to create x with 1 as a subscript?

The EQ field uses \s to note superscript or subscript and one of four additional switches for positioning. Example:

{ EQ (X\s\do2(1)) } 

More information at:
Field codes: EQ (Equation) function

I'm sure you have your reasons behind using the { EQ } function, but it may be of worth to note that there is superscript and subscript character formatting within the Font dialog. Unicode characters for common superscripts and subscripts are also available from Character Map (All Programs, Accessories, System Tools, Character Map). The characters are on the second to last and the last pages of characters and can be copied and pasted directly into a formula without additional formatting, e.g. (x₁) uses U+2081.

a. user say:

Hello Susan Mulgrew:

 

Thank you for posting to the forum regarding the issues you are having completing the verification process for your child's account.  From my experience this issue arises when you try signing into http://accounts.live.com with the child account.  If you are getting this somewhere else, detail where you are getting this prompt.

 

Typically this issue arises due to settings or problems with either the parental account or the child account.  The first thing I would look into is the regional settings of your accounts.  Try signing into the following site with both accounts, http://xbox.com.  Once you are signed in go to MY ACCOUNT > billing contact information.  Look to the country set for both accounts.  One thing I have seen is that customers find the account was created in a country other then the country that issued your card.  If you find this to be your issue there are two options; create a new account in the proper region or consider having the account migrated to the proper region, http://support.xbox.com/en-US/billing-and-subscriptions/account-management/change-region.  In addition to this, try signing into http://accounts.live.com and verifying the settings for your adult account.  You should be able to see the current settings by clicking the edit option in overview.

 

If this regional conflict is not at the heart of your issue the next thing to review and update is the child account information listed on http://accounts.live.com.  Sign into the adult account on this site and go to permissions > kids accounts > edit.  If you do not see a child account listed here move on to the next possible resolutions.

 

If the issue continues, there are just a couple additional things to consider:

  1. Contact Microsoft account support to further discuss what may be causing this and possible solutions, https://support.microsoft.com/getsupport/hostpage.aspx?oaspworkflow=start_1.0.0.0&wfname=msa&pesid=10123&locale=en-ie&wf=0&ccsid=635134699774807516.
  2. Create or use another adult account when attempting the verification process.
  3. Try creating another child account and attempting the process of setting this account up as you did the current account.  Options 2 and 3 should at least help us determine which account is causing the trouble you have encountered.

Please let us know if you have any other questions or concerns.
