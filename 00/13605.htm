This issue is caused by having entries in the Blocked senders list that cause your own email address to be considered a blocked sender.

Briefly, check your Blocked sender list for cases such as these:

  1. Check to see if your email address is added and remove it, if found.
  2. Check to see if your domain is added and remove it, if found. A domain entry example is: domain.com or @domain.com
  3. Check for any entry errors in the email address for entries which might have a comma instead of a period. An example of an entry error is:

     

    In the 14.3.2 update we again began processing the local blocked sender list if your Exchange version is less than 2013. That is why this issue became apparent after the 14.3.2 update. The above errors in the block list were present before the update, but the list was not being used.

     

     

    This isn’t my problem:

     

    If the above does not explain the problem please post further details on this thread.

     

    One key point is how your own email address ended up in the blocked sender list. Some have suggested that the 14.3.2 update added your own email address to the blocked sender list.

     

    We do not believe that the 14.3.2 update did so, only that we started to process the entries that were already there. If long ago you accidently marked a message from yourself as a blocked sender that would explain this situation.

     

    If you have previously cleared the Block senders list and know that it now contains addresses that you did not add we are interested in understanding how that occurred.  Did you use the “Block Sender” option after clearing the list?  Can you identify a particular email that, when blocked, adds an unexpected or badly formatted address to the blocked sender list?

     

     

    More details:

     

    The client side junk email filtering code in Mac Outlook has long been disabled when connecting to Exchange servers. The junk email filter on the exchange server is relied on instead. Processing of the local blocked sender list was also disabled under the same conditions.

     

    In the 14.3.2 update the following changes were made:

    1. When connecting to an Exchange server version LESS THEN Exchange 2013 the local Blocked Senders list is now processed again. If either of the above errors are present in your blocked senders list they will now cause problems until the list is corrected
    2. When connecting to an Exchange server version Exchange 2013 (or greater) the local blocked sender list is not used and is inaccessible in the UI when viewing an Exchange 2013 account. The menu options “Mark as Junk”, “Block Sender” etc now cause the Exchange server based blocked sender list to be updated. Since it is server based, the updated blocked list applies even when this copy of Mac Outlook is not running.

     

     

    How do I know what version my Exchange server is:

     

    Select one of the mail folders from your Exchange account then look at the Tools menu.

     

    If the Junk E-mail protection menu item is enabled then your Exchange server is less than 2013 and the local blocked sender list is being used.

     

     

    Changes to 14.3.4:

     

    The version of the next update will be 14.3.4 (14.3.3 was an update only affecting subscription licensing for some languages)

     

    We will exclude the Sent Items folder from Blocked Sender processing in the 14.3.4 update.

     

    That Blocked Sender list will still filter as Junk messages sent to the Inbox. You still need to check your local blocked sender list if this problem is affecting you and not wait for the next update.
