If memory serves the boundaries of text boxes and other objects are not visible by default with Pub 2013 (bad decision by MS in my view). This means you have to mouseover the object to see the boundaries and yet they disappear when you select the object.

This issue could occur if normal.dotm template is corrupted. Rename normal.dotm and verify the result.

1. Press Windows + X.

2. Select Command Prompt(Admin)

3. Type following command press Enter.

ren %userprofile%\AppData\Roaming\Microsoft\Templates\Normal.dotm OldNormal.dotm

Method 2: Start Outlook in safe mode following steps mentioned below and verify the result.

* Hold Windows key + R.
* Copy and paste, or type the following command in the Open box, and then press Enter

Outlook /safe

Note: There is a space between Outlook and '/'

If Outlook works as expected in safe mode, disable the add-ins by following steps mentioned below and determine the problem causing add-in:

a) Click on File menu, click Options, Add-ins, Go button besides Manage: Com-in Add.
b) Check if there are add-ins listed, clear the checkbox.
c) Close the Office program and restart it.
d) Enable each add-in one at a time, restart the Outlook, and repeat the above procedure. If the issue reappears again, it will help you to determine which add-in is causing this issue.
