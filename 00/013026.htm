el_loco---01 say: The actual option is under Tools->Call Mailbox

You can edit the number there.

BTW did you know you can use Pause and Wait characters to add passwords and other keys to you dialing sequence?

Just type the number if you want to insert a pause hit the * key twice, if you want to insert a wait hit the * key three times.

This allows you to access your voicemail and enter the password without having to type it every time.

12344p1234

IF you enter this number in a contact or anywhere it will dial the first part (12344) and start the call then wait for about 150 ms then dial 1234.

If you put a wait character:

12344w1234

It will dial the first part then a pop-up will come on waiting for you to hit Send (Left soft key) and it will send the next string.

This is also useful if you use calling cards. You can concatenate numbers like:

12344p1234w1w7

    Further to what Charles said, Word does not save the Normal.dotm every time you close a document. Any prompt to save at that time pertains only to saving the document being closed. Word updates Normal only when you Quit the program.

    Even if you use one of the Default buttons (such as for Font, Page Setup, etc.) the changes are not written to the template until you Quit the program. It may be a bit misleading because clicking the Default does prompt as to whether you actually meant to have Normal updated, but the actual updating doesn't occur until you Quit Word.

    All updating takes place at once on Quitting the program. If the box is checked you will be prompted again before the template is modified. If the box is not checked the updating takes place automatically. I also agree that having the prompt active is highly recommendable for most users. IOW, responding Yes/No applies to any & all Default settings changed during that session with the program.

    OneNote 2010 does not have the ability to limit read and write permissions from the program. However, OneNote tracks the changes that are made to the notebook and highlights all changes made. As a workaround you may try the following:
