sanscesse say:

After a lot of experimentation, I've found a workaround:

Go through "Installed updates" and find the update that is for "Windows Update Client for Windows 7 ...". This'll be listed as "Update for Microsoft Windows (KB...)" in the list of installed updates, but the KB number varies. KB3075851 and KB3065987 are two possibilities, but it might be something else. You just have to Google each KB number in the installed updates list to figure out which update is the right one. Uninstall this update. Restart. Now Windows Update will work, though you must disable automatic installation of important updates and be careful not to allow any future update for the Windows Update Client (maybe with a different KB number) or this'll just get installed again, which breaks it again.

I suspect that this will cause problems in the future when certain updates require an updated version of the Windows Update Client. For example, I don't think that upgrading to Windows 10 will be possible. Microsoft or some other Windows genius really needs to look into this and figure out how to delete the thing that's confusing newer versions of the Windows Update Client...

andynich say:

If anyone is still looking in here....

I had this problem with a machine and managed to persuade it with the following changes in the registry:

As documented in this article - https://support.microsoft.com/en-us/kb/3080351  I set the machine not to do the Windows upgrade (although it had already been started). I also removed the temporary installation files for the Win10 upgrade.

But this left the machine in the state described in this post - insisting that it needed to restart to finish the previous installation - by which I am pretty sure it means the Win10 upgrade.

I then modified the following keys in the registry which had the desired effect:

HKLM\Software\Microsoft\CurrentVersion\WindowsUpdate\AutoUpdate\OSUpgradePendingRestart changed value from 1 to 0

HKLM\Software\Microsoft\CurrentVersion\WindowsUpdate\OSUpgrade\AllowOSUpgrade changed value from 1 to 0

After a restart I visited Windows update and it reported the "Windows 10 installation has failed" (which is good). A check for updates listed some optional updates with a nice green shield.

Hopefully the situation will last but this seems to have confirmed to the update system that it is no longer in the middle of an OS upgrade.

I have been stuck on this issue for hours with nothing working.  Finally I was able to solve it by doing things in this order.

  1. Turn off Automatic Windows Updates.
  1. Open Control Panel > Uninstall a program > View installed updates and wait for it to finish loading.
  2. Search for KB3035583 and uninstall it.
  3. Restart the PC.
  4. Launch Disk cleanup.  Once it opens click the button to Clean up system files.
  5. There will be the Windows 10 files in there listed under either Temporary Installation Files or Windows Update, the file will most likely be over 4GBs.
  6. Place a checkmark next to it and click OK to delete the files.
  7. Restart.
  1. Launch Regedit (Backup first) and delete this Key: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update\RebootRequired.
  2. Check for Windows Updates.
  3. Hide Update KB3035583.
  4. Install Updates.
  5. Turn on Automatic Windows Updates.

Now search for the Update by number, right click on it select Uninstall. It may or may not show a Failed status.

Now restart your computer run Windows Update and install it afresh.

santhoshthomaswd say: Hi All,

I also had the same problem of Lifecam VX700 not getting detected. This are the steps I did to make it work. You could also try.

1. On the computer properties, go to hardware tab and select windows update.
2. Enable "If my device needs a driver, go to windows update without asking me"
3. Go back to your webcam properties from device manager and select install driver automatically.
4. This option will search the most appropriate drivers for all the devices on your system. 

Hope it works for you.
