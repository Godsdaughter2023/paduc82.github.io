sargon say:

Xbox One TV resolution vs. Windows 10 streaming resolution

I am planning to buy an Xbox One sometime in the next couple months and I have a question about output resolutions.


Will the Xbox One output the same resolution over Windows 10 streaming as it does to the connected TV, or can the two be set independently?


The reason I ask is that my gaming TV does not support 1080p, so I will have the console set to output 720p to the TV.  When I stream to Windows 10 though, I would still like to be able to stream 1080p video.  Is that possible?

sargon say:

I can answer my own question here that the Xbox One will stream at whatever resolution it is set to in the settings, so if I set the console to 720p it will stream at 720p.  It isn't possible to pass a 720p signal out the HDMI output, but use a 1080p output for streaming (unless you constantly change the settings back and forth).  

The PC will scale to full screen no matter what, but 1080p looks much better than 720p on a 1080p monitor.  The problem is if I set to 1080p when streaming to my PC and forget about it, then the next time when I try to use my TV I would get garbled graphics because it can't accept the 1080p output, so I would need to go back upstairs and stream to my PC to reset the settings to 720p before I can use the TV again.

How I solved this problem was to use an HDFury 4S scaler to connect to my TV and scale the Xbox One down from 1080p to 1080i, which is the maximum resolution of my gaming TV.  That allows me to keep the Xbox One console set on 1080p at all times.

Method 1:

You may refer the following link.

Troubleshoot audio-recording problems

http://windows.microsoft.com/en-us/windows-vista/Troubleshoot-audio-recording-problems#

 

Method 2:

You may follow the steps mentioned below.

Step 1: Testing for an audio signal

a) Click Start, and then Control Panel.

b) Click Hardware and Sounds, and then click Managing audio devices.

c) Click the Recording tab.

d) To test the microphone, speak into it. To test line-in, send a signal to the line-in port from the source audio device (press play from the audio player). Look at the gray bars next to the device name on the recording tab:

i) If the grey bars next to the device name change to green, this device is active and an audio signal is being passed through the microphone or line-in port. This step shows that problem is related to sound settings in the software that uses the microphone. If you have not already done so, set the selected device to default and adjust the levels. Once the device is set to default and the levels are set, use and adjust the sound recording or voice carrying software.

ii) If the grey bars remain gray and inactive, the device is not recognizing a signal. Leave the recording tab open and go to the next step.

Step 3: Check volume levels and mute

The speaker volume for the microphone and line-in may be muted or turned down. Use the following steps to check the speaker volume levels for line-in and microphone.

a) Click Start, and then click Control Panel.

b) Click Hardware and Sound.

c) Click Sound.

d) Select Speakers, and then click Properties.

e) Click the Levels tab.

f)  Click the Muted button next to Microphone and any other line-in that is muted.

g) Use the slider bar to raise the level for Microphone and each line-in to at least 75.

h) Click OK.

Step 4: Update the audio drivers

You may find that a front microphone port does not work, a recording device is missing from the recording tab, a microphone is not recognized when it is plugged in, or other problem. There may be updated audio drivers that can fix these types of issues. Use the following steps to make sure audio drivers are up to date.

a) Connect to the Internet.

b) Click Start, and enter device manager in the Start search field.

c) Click the plus sign, next to Sound, video and game controllers.

d) Right-click the name of the sound hardware and select Update Driver Software.

e) Click Search automatically for updated driver software and wait for Windows to update the driver software.

f)  Windows checks a remote Web site to see if updated driver software is available.

i) If an update is available, let Windows install the update.

ii) If an update is not available, skip to the next step.

g) Once the update is installed successfully, test the microphone or line-in as was done in step 1. If the problem persists, go to the next step.

Step 5: Configuring Windows recording settings

By default, Windows Vista automatically enables microphones and line-in ports. Use the following steps to set the microphone/line-in port to be the default port and to configure the recording levels.

If the Recording tab in the sound window is not selected, open the Sound window and click the Recording Tab and make sure that the status is set as below.

1 - Microphone port - working and set as default

2 - Line In - working

3 - Digital In (S/PDIF port) - working

4 -Stereo Mix (Software mixer) - Disabled

a) If multiple microphones or line-in ports are available, select the port that you want to use. Click Set Default to make this device the default audio input device.

b) If the device you want to use contains a small circle with an arrow pointing down, the device is disabled. To enable the device, right-click the device name and select Enable.

c) With the device selected, click Properties, and then click the Levels tab.

d) Adjust the slider until to 75% of its range.

e) If Mute is on, click the mute button to un-mute the device. The mute button should show as a blue speaker without any red circle.

f) If Microphone Boost is available, adjust to +20.0 dB.

g) Click OK to make the changes.

h) Test the microphone as was done in step 1 to determine if Windows is able to detect a signal for the microphone or line-in being used. If you have performed all the steps and the microphone or source device still does not work, the audio ports or sound hardware on the computer may be bad.
