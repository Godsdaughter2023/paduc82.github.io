albert d. kallal say:

As a heads up, I seen this issue crop up on computers and one is NOT dealing with any large table etc.


I see cases were a person is working on a single database on their desktop – they were simply attempting to slice and dice some data and were being absolute roasted and nagged by some silly record locking error messages, I have NEVER had to change this setting in 15 years of using Access.


Two things should be tested here:


First, turn off row locking, and second if you just a user trying to get some work done then open the database as exclusive.


As noted a 52 meg database is RATHER small and having to resort to registry edits and having to change OPERATING system locks for using what amounts to a database with a few simple tables makes zero sense.


The row locking feature I speak of is file->options->client settings.

Down in the "advanced" section you want to un-check the box
[x] Open databases by using record-=level locking.


The above  setting may not effect or fix your issue, but it will SIGNIFIANCTLY effect how fast the database bloats when doing updates to data.


Next (and this should fix your issue) is open the database Exclusive. In fact CHANGE the default mode from Shared to Exclusive.


As noted if this is just a typical database that is NOT multi-user then you should not be seeing such record lock issues or exceeding them.


Try the above settings - I suggest a compact + repair and you exit and then re-enter the database and try running your code or update(s) again.
