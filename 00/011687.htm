davidf2 say:

Publisher forms require FrontPage Server Extensions (FPSE) on your web host and that you use HTTP uploading protocol. If you use FTP uploading that can corrupt the extensions and your forms will not work. Unfortunately most webhosts no longer offer FPSE and HTTP uploading protocol is impossible without them and you must use some other form handler or forms program provided by your web host to 'power' your form. You can sometimes change the properties of your form to use the webhosts alternative forms handler. Look at your webhost FAQs for information about forms and forms handlers.

Here is an old article that explains some of the basics involved in using a form with a Publisher web page: https://blogs.msmvps.com/dbartosik/2006/11/28/publisher-web-publication-forms-101/

FWIW I never found the Publisher forms easy to use and develop even before FPSE were abandoned by most web hosts. I found it easier to incorporate a form from:

phpFormGenerator: http://phpformgen.sourceforge.net/

However as of Pub 2010 MSFT started deprecating the web building functionality of Publisher and as of Pub2013 they stripped the ability to insert a HTML code fragment which I found an essential to incorporate a form in a Publisher web.

You can probably still use the form generated by the phpformgenerator but you would have to manually edit your HTML code and most people using Publisher do not understand how to code web sites. I would suggest that you try generating a basic form and uploading it to your host as a separate page and link to it from your Publisher generated site. You might also consider abandoning the idea of a form and just have your contacts email you. However...

I would suggest that before you go much further that you consider moving to a different program than Publisher to develop your website. It just isn't the right tool to use if you want anything more than a simple basic fixed site. If you need dynamic features such as a form you should use something else. We were able to build a decent cross browser compatible website with Publisher before Publisher 2010 but MSFT has hobbled Publisher web functionality in the current versions. IMHO you should invest your time in learning how to use another program...you aren't going to be happy with the limitations of the current version of Publisher.

MSFT Web Expression is now free and is a full standards compliant web program but also has a huge learning curve. It is over kill form most people that have used Publisher to build their sites. You might also look at WordPress which is also free and very popular. Even your web host probably has a free copy and templates are available that look like traditional web pages or blogs. And look to your web host for a free web building module for basic sites. There are lots of options out there and you should probably start finding an option to Publisher.

That is probably more than you wanted to know...

jay freedman say:

Your description is at least partly consistent with the possibility that the first paragraph (and maybe others) in the text box are formatted with an "Exactly" line spacing. In particular, the fact that all but the bottom-most sliver of the picture is cropped matches that explanation. The part that I can't match up is when you add more empty paragraphs and can then insert the whole picture -- in that scenario, every paragraph should have "Exactly" line spacing, and you'd still see only one line's worth of the picture no matter which paragraph you inserted it into.

Anyway, after you insert a text box and before you put in a picture, right-click in the box and click Paragraph. If the Line Spacing says "Exactly", change it to "At least". The whole picture should then appear as it used to.

If that works, the next question is where the "Exactly" setting came from. Start another text box and check its line spacing. If it's "Exactly", press Shift+F1 to open the Reveal Formatting pane. At the bottom of the pane, check the box for "Distinguish style source". Look through the formatting to see where the "Exactly" line spacing is listed. If it's part of the style applied to the paragraph, you need to modify the style to change it to "At least" (or the Word 2013 default, which is Multiple 1.08 lines).

In the file that you want translated, click Review, in the Language group, click Translate.

  1. Click Choose Translation Language.

  2. Under Choose document translation languages click the Translate from and Translate to languages that you want, and then click OK.

  3. Click Review, in the Language group, click Translate.

  4. Click Translate Document. The From and To languages that you selected are listed.

  5. Click Send.
