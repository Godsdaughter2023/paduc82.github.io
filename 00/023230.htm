tom_bpc say:

Chris,

First a couple clarifications:

1. "Macro" and "VBA" are practically synonymous.  Neither has any direct relationship to “Custom Fields” or “Custom Field Formulas.”  What you have shown is a task custom field formula, not a macro.

2. There's no need to assign task calendars just to satisfy your formula.  The (Calendar) argument in the ProjDateDiff function is optional.  If you omit the argument, then Project will use the Task Calendar if one exists or the Project Calendar if not.  The function will default to a resource calendar only if used in a resource custom field or a resource assignment custom field (e.g. in the resource usage view).

VBA is not so restricted; it allows you to compute whatever you want using whatever calendars you need.  See this thread from last week.

Re. your side note - all Date fields in Project include full date-and-time precision.  The full time of day is only displayed if an appropriate Date format (in General Project Options) is selected.

Other:

To me, your work is answering the question, "what must %Complete be to make the task appear on track?"  Project already provides two methods to do it: 1) Select task(s) and "Mark on Track" in the Schedule group of the Task ribbon; or 2) Select task(s) and "Update work as complete through..." in the Update Project dialog from the Project ribbon.

Good luck, tom

Documents containing macros cannot be saved with them intact in the .docx format; only in the .doc or .docm formats. The switch from Hotmail.com to Outlook.com isn't supposed to change anything about your mailbox, just on how it looks on the screen.  Every folder and every message you had before is supposed to be there still.
