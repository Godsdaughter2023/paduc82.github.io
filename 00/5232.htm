carol gardens say:

How to write a macro to restyle a document and to delete selected styles - Word 2013

I am bringing text over from one Word 2013 document to another.  I want the styles to come over, but I want to change the styling on some paragraphs and to also delete some of the styles.  I recorded macros and sometimes they work, sometimes they don't.  If there are no paragraphs with the style I want to replace, I want the macro to continue.  Same thing when I later try to delete specific styles.  If they don't exist I want the macro to continue.  Betcha there is an If statement that can tell the macro to continue if it doesn't need to do that step.  Recording is one thing, writing VBA code is another - and I'd sure appreciate some help.
jay freedman say:

VBA and Word are not without their quirks when it comes to styles.

  • A large set of styles are "built-in" and can't be deleted from any document. Although you can do something like ActiveDocument.Styles("Heading 1").Delete, all it will do is remove any non-default settings from the style and leave it as it's defined in the underlying template.
  • As a corollary, every built-in style is always a member of the ActiveDocument.Styles collection, so referring to its properties won't raise an error, regardless of whether the style is actually applied anywhere in the document.
  • Styles that are not built-in can be deleted completely by the same syntax.
  • Non-built-in styles are members of the ActiveDocument.Styles collection only if they have been used in the document -- either currently or at some time in the document's history (even though those paragraphs might have been deleted or assigned a different style) and have not been deleted, or if they're in the document's base template.
  • If you refer to a non-built-in style that isn't a member of the collection, VBA raises an error 5941 ("The requested member of the collection does not exist").
succeeded, and any other number means that an error was raised. You could use a macro structure like this:

Sub x1()
    Dim strTest As String
    On Error Resume Next
    strTest = ActiveDocument.Styles("StyleToChange").NameLocal
    If Err.Number = 0 Then
        ' the style exists, so go ahead and modify it
        ActiveDocument.Styles("StyleToChange").ParagraphFormat.SpaceAfter = 0
        ' or
        ActiveDocument.Styles("StyleToChange").Delete
    End If
End Sub

If you're also dealing with built-in styles, you could use the same structure, but this error trap won't tell you whether the style is actually applied somewhere in the document. There is an InUse property, ActiveDocument.Styles("StyleToChange").InUse, that theoretically should do this, but a large subset of the built-in styles always say they're in use regardless of whether they really are applied in the document. The only reliable method is to use a Find to look for the first location where it is applied. The Range.Find.Execute method returns True if it finds such a location or False if there is no location. That code would look something like this:

Sub x2()
    ' to modify a style's properties only if it's in use
    Dim rg As Range
    Dim bUsed As Boolean
    Dim strStyleName As String
    Dim strTest As String
    strStyleName = "StyleToChange"
    On Error Resume Next
    strTest = ActiveDocument.Styles(strStyleName).NameLocal
    If Err.Number = 0 Then
        Set rg = ActiveDocument.Range
        With rg.Find
            .Format = True
            .Style = ActiveDocument.Styles(strStyleName)
            bUsed = .Execute
        End With
        ' bUsed will be true if the style appears at least once in the document
        If bUsed Then
            ' make desired changes, such as...
            ActiveDocument.Styles(strStyleName).ParagraphFormat.SpaceAfter = 10
        End If
    End If
End Sub
