How do I calculate a payback period in excel when I also have a CAGR - Microsoft Community

thesolarguy say:

How do I calculate a payback period in excel when I also have a CAGR of 5%? For example $2000 Plus 5% per year divided into $16,000 to find out how many years is the pay off?

How do I calculate a payback period in excel when I also have a CAGR of 5%?  For example $2000 Plus 5% per year divided into $16,000 to find out how many years is the pay off? I can't figure out the formula.

 

Maybe this will clarify.  I sell solar electric systems.  The cost of energy increases at 5% per year.  I am trying to calculate the payback period using the adjusted increase in energy costs.

 

Example:

Joes Solar Electric System Generates $2000 per year of electricity at todays rates.

The Department of Energy states that the value of electricty is going to increase by 5% a year

If Joe payed $14,000 out of pocket for his system how many years will Joe's payback period be based on the increasing value of the electricity that he is generating?  (In the form of a formula for an excel document.)

joeu2004 say: SolarGuy wrote:
Joes Solar Electric System Generates $2000 per year of electricity at todays rates.

The Department of Energy states that the value of electricty is going to increase by 5% a year

If Joe payed $14,000 out of pocket for his system how many years will Joe's payback period be based on the increasing value of the electricity that he is generating?  (In the form of a formula for an excel document.)


There are many ways to solve this problem.

Suppose that B1 contains the initial investment (14000), B2 contains the 1st-year cost (2000), and B3 contains the incremental cost rate (5%).

Method #1 (low tech)
----------------------------
Set up the following:

C6:  =B1
A7:  1
B7:  =B2
C7:  =C6-B7
A8:  =A7+1
B8:  =B7*(1+$B$3)
C8:  =C7-B8

Drag A8:C8 down until column C is zero or negative.  Column A is the payback period, in either the last row or the row above, depending on how you want to treat the last (probably) fractional period.

Method #2 (Solver)
-------------------------
Enter the following formulas:

B4:  =A6+1
B6:  =B2*((1+B3)^(A6+1)-1)/B3
B7:  =B1-B6

In Solver, set up the following parameters:

Target:          B7
Equal to:      Value of:  0
Changing:   A6
Constraint:  A6>=0

Run Solver to find the result in B4 (payback period).  Note that B7 will probably not be zero; but it should be close.

You could also use Goal Seek in a similar manner.

Method #3 (direct)
-------------------------
Put the following formula into B4 (payback period):

=LOG(1+B3*B1/B2)/LOG(1+B3)

There's a fair amount of ambiguity in the question but you can work this out from first principles.

The sources of ambiguity include, but are not limited to, the compounding interval and the payment schedule.

From first principles you can always solve these kind of problems with a table with columns like:

Event Date Amount Balance

Each row will be one event, however it is defined for your problem.

john mcghie say:
The purpose of the file extension is to tell the computer what kind of content the file contains, and so which application to send it to.  The extension .doc says the file contains a binary stream of code intended for obsolete versions of Word.  The extension .docx says the file contains compressed XML code intended for the current versions of Word.

Generally, if you change the file extension to something other than what is actually in the file, you should expect the computer to be unable to open the file at all.

However, there is a bug in Lion/Office 2011 that causes Lion to send .doc files to the wrong application, which then cannot open the file.

If you change the .doc to .docx on a file that actually contains .doc, you trigger an error handler in Word that says "This is NOT .docx, I wonder if .doc will work?".  It's a nasty clunky work-around that relies on an error-handler to get around a bug.

Keep applying the updates to Microsoft Office and Lion as they come out: this bug will be fixed soon.

And if you do what the other poster said, open the file using File>Open within Word, you won't get the problem.  This is a Launch Services bug in Lion that will be fixed soon.

Generally, the rule is "Never change the extension, otherwise the file won't open at all."
