Scott's three-query solution is probably the simplest to implement, but this can in fact be done with a single "SELF JOIN" query. It can't be built in the query grid but the SQL view would be something like this (untested, air code):

SELECT A.[Division], A.[MntDt], A.[TransDate], A.[ActualRevenue], A.[PlanRevenue], B.[ActualRevenue] AS LastYearActual

One thing to note, if you ever lose access to your phone number, you wont be able to receive the authentication code. So be sure if you are ever going to change your phone number, change it in your Microsoft account first.

barb way say:

One problem is that the Indexing Service is not supported with Visio 2010, only Windows Search is supported.  If you have the Indexing Service enabled, first step is to disable it.  Then make sure that Windows Search is working (look for the service status - restart if necessary).

Second piece is to ensure that you have the Filter Pack for 2010 installed.  It may have been installed automatically when you installed Office 2010 / Visio 2010, but to be certain, you can download it and install manually.  Try this link:

http://www.microsoft.com/downloads/en/details.aspx?FamilyID=5CD4DCD7-D3E6-4970-875E-ABA93459FBEE&displaylang=en

Once you have both of the components in place, check that the Visio Content folder is listed in the Indexing Options in the Control Panel. 

With all Office applications closed, kick off a complete rebuild of the index under the Advanced button of the Indexing options.  (Yes, you can still use the programs if you want, but it will slow down the indexing).

When the Indexing is completely finished, open Windows Explorer, and browse to the Visio Content folder, and search for something like "car" - you should find several stencil files with that keyword. 

Then open Visio, and try the same keyword search.  You should also find several shapes with "car" as the keyword - probably around 25-30.

If all of that works, then you should be properly configured.  If you used to see more shapes, there are two things to be aware of.

1) If you have Visio Standard, you will not see all of the shapes that are in the larger versions.

2) If you used to find shapes from the web search servers, these are no longer in operation: http://support.microsoft.com/kb/2019049

Hope that helps!

Visio uses the results of the Windows Desktop Search, which in turn uses the filter pack to understand how to parse the Office 2010 files to get metadata and file contents.  WIthout the Filter Pack, Windows search can only 'see' properties such as file level keywords. 2. Whether prompted or not, reboot the system.

3. Check in Windows Explorer first and search for 'car' (singular).  If you get something like 25-30 files, that would be reasonable, since you do not have the Metric stencils.

4. If Windows Explorer is returning the right number of files, test Visio.

Since the WDS index is used for all Office 2010 applications search functionality, we have to see WDS return the right information before anything else makes sense.

Long winded version:

For those of you looking to fix this once and for all, what you're missing is the message you originally surpressed WAAAAAAY back in the day before you even had this problem.  The dialog box reads: "Visio cannot provide fast search results because windows search is not running or is not configured properly for Visio."

There's a problem with the settings in your indexing.  Believe it or not, the objects have no names (that are related to the object at least).  They only have details.  These details are "contents" of the object itself.  your Indexing settings needs to be fixed. Most of you getting to this page turned this feature off a long time ago not realizing anything broke.  Then, because you barely need to use Visio, you couldn't associate that checked box of "don't show me this again" with your change of the Indexing Options settings.

Summary for the win:

Open Indexing Options > Advanced > Tab: File Types > At the bottom, select "Index Properties and File Contents" > Click OK.  Boom, Done!

You will notice that you "items indexed" will start over.  And this is because everything automatically gets rebuilt with contents included.  Mine just finished and it's the first freakin time I've had Visio search results in almost 2 years.... omg.  You won't have any results until Indexing is complete.  If you still have no results after it is complete, be sure to add "C:\Program Files\Microsoft Office\Office15\Visio Content" to the Included Locations folder in order to index the standard Visio stencils.  Not just the stencils you downloaded from the internet that are most likely in "my Shapes."

So now... when you search in Visio with that option selected, you'll notice it doesn't tell you there is a problem with search options, however any of you who are here right now suppressed that message a very long time ago not knowing it was actually legit.
