I'm now at the point where I am writing the code to make sure that the data is valid before allowing a commit to proceed. I'm stuck on how to check for empty textboxes or combo boxes. Here's what I've got:

good = True

If (IsEmpty)) Then

     good = False 'Bad if it is empty

ElseIf(IsNull)) Then

     good = False 'Bad if it is Null

ElseIf(trim(Not ((TextBox.Value & vbnullstring) = vbnullstring)) Then

     good = False 'Bad if it is blank

End If


Because VBA doesn't use lazy evaluation like C, I can't put all three predicates into a single If statement; before I can check for blank I have to check if it is Null, and before I can check if it is Null I need to check if it is empty (I'm not sure about empty and null, but I know if I check if a null value is blank I will throw an error). Even if I need just two nested If statements, that's still a lot. I have six text- and comboboxes that I need to check, so that is a lot of code to test what seems to be something very simple.

Is there an easier way to do this?

dirk goldgar say:

In the vast majority of cases you can use:

    If Len(TheTextbox & vbNullString) > 0 Then

        ' The text box is not Null or empty string

    End If

If it's possible that the text box contains only one or more spaces (which is hard to get it to do) then you can make double sure this way:

    If Len(Trim(TheTextbox & vbNullString)) > 0 Then

        ' The text box is not Null or empty string or blanks

    End If

You do not need to test for Null first; the concatenation takes care of that.

karl timmermans [outlook mvp 2012-2018] say:
Multi-part answer to cover everything

#1 - GMaill accounts can be <either> Imap or POP3. All depends on you configure the account on both sides. Server side (i.e. you have to set the account to be either POP3 or Imap) as well as desktop side

Remember that POP3 only downloads the <Inbox> folder so as long as you're not using anything other then the Inbox on that account - you're going to be all good

#2 - Ignoring the 3rd account not configured yet as POP3 - the way you configure your existing POP3 accounts is as follows

Summary:

You'll need to select one of the existing PST files, make sure it's the default data file for the profile

Details

a) go to Outlook --> File --> Account Settings --> email tab
b) highlight each of the POP3 accounts --> go to bottom of that screen to the <Change Folder> button --> take note of the folder/file name where the email for that account is being delivered
c) switch to the Account Settings --> Data Files tab --> pick the PST file you want to use for all of your accounts and make it the default data file for the profile. The default data file is also where your default contact/calendar folders are going to be stored
d) exit Acct Set screen and go to main Outlook window
e) copy all the data from the folders in the PST file you're NOT going to be using to the corresponding folders in the one you've chosen to be used for all your accounts
f) <IF> by any chance, your IMAP .ost data file was set as the default for the profile and the IMAP folder tree contains contact/calendar folders (will have the words "This computer only" in the folder name), be sure to copy the items from those folders as well since you will lose everything if you don't when you delete the IMAP account to set it up as a POP3 account. Your email data will be fine, no need to copy anything from the <Inbox> BUT if you have other email folders, you'll need to copy those to the PST file as well

g) When all the above is done and you're satisfied everything has been copied correctly, go to Account Settings again, remove the IMAP account and add it back as a POP3 account just like you did for the Yahoo accounts with 1 exception

<make absolutely sure> that during the manual set up process you select the option to use an existing PST file and select the one you decided on to be your main file

*** above assumes you've gone into Gmail and set the Imap account to be POP3

h) go to Acct Settings --> Data Files tab and remove the PST file you are no longer using

That in a nutshell will give you a Unified <Inbox> for all your POP3 accounts

It is easy enough to copy only the visible rows after using autofilter, however I'm not sure you can 'Cut' non-contiguous rows. Even on a blank workbook if you select row 2 and row 5 and select 'cut' you will get an error.

To 'Copy' only the visible rows we use special cells -

ActiveSheet.UsedRange.SpecialCells(xlCellTypeVisible).Copy

 

My suggestion would be to either copy the values you want to sheet 2,  then go back to sheet 1 and delete the rows you copied

Or

Apply 'Sort' so that the rows you cut to sheet two are contiguous, then you should be able to 'cut' to sheet 2

 

If it isn't really necessary to use autofilter, you could also loop through each row and cut  each row that fits your criteria to be pasted to sheet two.
