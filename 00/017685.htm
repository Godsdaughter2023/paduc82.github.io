    kplaxton say:

    MS Project - Fixed work subtasks for longer fixed duration summary task

    What I want to do is create several subtasks with fixed work - how many hours each subtask will take to complete. Then, in the summary task, I want to manually enter a duration that is not dependant on the subtasks. For example, over the next 30 days, three subtasks will be completed, one with 4 hrs of work, one with 20 hours of work, another with 5 hours of work.

    How do I do this properly so that Project will automatically respect the max units for each resource?

    dale howard [mvp] say:

    KPlaxton --

    I use a combination of Units and Work to allow Microsoft Project to calculate the Duration of the task.  If you know a resource will not be able to work full-time on a task, then set a Units value to correspond what portion of a working day they will realistically be able to commit to the task.  For example, if the task requires 40 hours of work, but the resource can realistically work only half-time on the task, set the Units value to 50% and then Microsoft Project will calculate a Duration of 10 days.  I think that this technique gives you better and more reliable scheduling.  Just a thought.  Perhaps the others will feel free to share their thoughts as well.  Hope this helps.

    john - project say:

    KPlaxton,

    I'll add two additional thoughts to Dale's excellent response.

    First, make sure you understand the difference between duration and work. In Project they are not the same. Duration is the time span during which a task will be performed. Work is the effort one or more resources will expend to accomplish the task. If a single resource works full time on a single task, then duration and work will be equal, but that is often not the case.

    Be cognizant of Project's work equation:

    Duration = Work/Units

    If for some reason, setting the resource assignment level (i.e. units) is not feasible, then you can make the task a fixed duration task. When you set the work value for the task, Project will linearly spread that work content over the fixed duration, and, if you set the work value and then assign the resource, Project will automatically set the units value to achieve that condition.

    John

    mikem132 say:

    First, stuff disappears and re-appears in the catalog based on label licensing. One thing to do is look for a slightly different version of the same song---sometimes it will still be there under a different filename (the labels do this, don't blame MS).

    Second, Music Cache is the stuff you downloaded locally. This is DRM music you can play on that computer without an internet connection (well, for about a month, then it has to connect briefly to update the license files).

    Third, if something is no longer in the catalog, you can't sync across devices. You are really only syncing the file info and each device will download it individually....so if it is no longer there to download, it naturally will not "sync".

    If you don't already know---there are a couple ways to add music to what Groove sees as your music "collection":

    1. you can "add to collection"---this does not download it, but it appears in your list and can only be streamed while connected to the internet. You get a kind of bullseye icon on these tracks.

    2. you can "download"---this downloads a file locally for offline use. There is no icon on these tracks in your list.

    3. import your own stuff that is in Groove playable format (mp3, wma, etc). There is normally no icon next to this. However, if you had Xbox Music and did the "cloud match" of your own songs, you get a music note icon if this is "matched".

    The "match" thing is dead now. If you want your own tracks available to sync across devices, you upload to OneDrive.
