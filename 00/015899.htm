    macropod say:

    If you use content controls for the inputs instead of formfields, you can use an unprotected mailmerge main document, then apply the protection post-merge. Unlike most formfields, all content controls are compatible with mailmerges. You can even insert mergefields into text content controls, so users can edit the merged data post-protection if that's what you want. If you're worried your general employees might edit the mailmerge main document, the protection can be extended to that.

    The following macro will work with an unprotected mailmerge main document:

    Sub MailMergeToDoc()
    ' Turn Off Screen Updating temporarily.
    Application.ScreenUpdating = False
    ' Insert your document's password between the double quotes on the next line
    Const Pwd As String = ""
    With ActiveDocument
      ' Do the mailmerge
      .MailMerge.Execute Pause:=False
      ' Protect the output document
      ActiveDocument.Protect wdAllowOnlyFormFields, NoReset:=True, Password:=Pwd
    End With
    ' Restore Screen Updating
    Application.ScreenUpdating = True
    End Sub

    Simply add the macro to the the mailmerge main document's 'ThisDocument' code module and save it in the docm format. Because the macro is named 'MailMergeToDoc', it will intercept the 'edit individual documents' command, making the process seamless.

    If you need to protect the mailmerge main document as well, to safeguard against your general employees editing that, post back.
