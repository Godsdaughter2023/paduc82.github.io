kkckkc09 say: OK, I narrowed down the problem to the Normal.dotm template file. For those who don't know, this template file is opened invisibly in the background each time Word is run.  It's a container for some global settings such as styles and in my case, also macros that I had written or recorded over the years to make my work easier. 

If anyone has a problem with the context driven Synonyms function going missing again there are two options.


  1. close Word;
  2. delete Normal.dotm from C:\Users\<YourUserName>\AppData\Roaming\Microsoft\Templates
  3. start using Word as normal.  You should have the Synonyms function back.

  1. moved Normal.dotm from C:\Users\<YourUserName>\AppData\Roaming\Microsoft\Templates to another folder.
  2. rename the moved file to NormalFB.dotm
  3. restarted word, create a new file write some blah blah and save the file.  These actions will force Word to create a new blank Normal.dotm in C:\Users\<YourUserName>\AppData\Roaming\Microsoft\Templates.
  4. Close and reopen Word.  If you try to access any of your old macros... you should not be able to.
  5. Open NormalFB.dotm
  6. Go into the Developer tab and then click Visual Basic. 
  7. This will open the VBA editor and here you can create a new Module in your new Normal.dotm and copy paste the old macros from NormalFB.dotm into Normal.dotm and save the new template file.
  8. Exit the VBA editor and Word 2010.
  9. You should now have all your macros and the Synonyms function back.  If not then check your VBA code for any thing that might be upsetting language settings etc
  10. Lastly... make a backup of the new Normal.dotm......!!!!
Good luck...!!

Can I filter for items that have the Font Effect "Strikethrough"? If so, How?

frank arendt-theilen say:

Hi,

you can use the old Excel4 macro function: GET.CELL

  • Open the dialogbox Define Name (Ctrl+F3).
  • Insert a name like: IsStrikeThrough.
  • Refers to: =GET.CELL(23 ,OFFSET(INDIRECT("RC",FALSE),0,-1 ))+0*NOW()
  • Close the dialog.

The above formula assumes the column "Equipment Number" on its left hand side. If your column "Equipment Number" is in column D than insert in column E the formula:

=IsStrikeThrough

and copy it down. Now you can sort column E descending and delete the rows which holds the number 1 in column E.

 

Explanation to the formula:

1)
The 23 is used to return information about a cell. In this case it returns TRUE if all characters or only the first character in the cell are struck through, otherweise FALSE (seehttp://www.mrexcel.com/forum/showthread.php?t=20611 )

2)
You can change the -1 to any other distance of your entered formula: If your column "Equipment Number" is in column D and you change -1 to 4 enter the formula =IsStrikeThrough  in column H.

3)
With the red marked part of the named formula the formula gets volatile and immediatly returns the correct value if you change the formatting Strikethrough . It's not a must to use this part of the formula.

 

Regards,

Frank

stephen boots say:

I can't tell you if the procedures below will work with Windows 8, but they should.

On the other hand, I'd consider contacting Norton as the issue appears to be a miscommunication between Norton 360 (assuming that it is compatible with Windows 8!) and Windows.

 

When running this in Vista7, you must run the procedures as an administrator, elevated.

 

Start>Run>services.msc [enter]

Scroll down to Windows Management Instrumentation and double-click it.

Now click on the "Pause" button. Leave that window open and double-click

My Computer. Navigate to %systemroot%\Windows\System32\wbem (where

%systemroot% is the drive where XP is installed). Delete the Repository

folder and *only* the Repository folder. Now go back to the WMI service

window you left open and restart the service.

This will rebuild the Repository and hopefully straighten out the

incorrect entries for all your duplicates.

In order to see the Windows files, you may need to unhide them:

Make sure you are able to see all hidden files and extensions (View tab

in Folder Options).

Check "Display the contents of system folders".

Check "Show hidden files and folders".

Uncheck "Hide protected operating system files" and click "OK" to the

dialog box.

 

Open a command window (Start/Run --> cmd.exe) and run the following commands:

 

net stop winmgmt

cd /d %windir%\system32\wbem

ren repository repository.old 

net start winmgmt

It may take a minute or so to complete while WMI rebuilds the database.

 

Alternatively, see this blog post: http://ashwinrayaprolu.wordpress.com/2011/05/13/repairing-fixing-wmi-repository/
