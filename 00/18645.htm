But, on the surface at least it sounds like a typical formatting issue between 2 machines.  You have a couple of excuses to pick from:

simonc2 say:

Extracting values from a list in order

I have a long series of numbers in a column, several of the values are repeated. I want a function which will extract the smallest value from the list, second smallest, third smallest etc and place these in another column. If a value is repeated in the initial list, needless to say, I only want it to appear once in the summary column.

I'm using Office 2000.

t. valko say:

I have a long series of numbers in a column, several of the values are repeated. I want a function which will extract the smallest value from the list, second smallest, third smallest etc and place these in another column. If a value is repeated in the initial list, needless to say, I only want it to appear once in the summary column.

Try this...

Let's assume your numbers are in the range A2:A20.

Enter this formula in F2:

=MIN(A2:A20)

Enter this array formula** in F3 and copy down until you get blanks:

=IF(ROWS(F$2:F3)>SUM(--(FREQUENCY(A$2:A$20,A$2:A$20)>0)),"",MIN(IF(A$2:A$20>F2,A$2:A$20)))

** array formulas need to be entered using the key combination of CTRL,SHIFT,ENTER (not just ENTER). Hold down both the CTRL key and the SHIFT key then hit ENTER.

t. valko say:

You could also use the advanced filter to extract the unique values to another location and then sort the extracted values in ascending order.

  • A1 = column header
  • A2:A20 = numbers

Use the filter...

  • Select the range A1:A20 (we have to account for/include the column header with this method)
  • Goto the menu Data>Filter>Advanced Filter
  • Select Copy to another location
  • the list range should already be entered
  • Copy to, enter the destination cell address like F1 (we have to account for/include the column header with this method)
  • Select Unique records only
  • OK
  • Then sort these numbers in ascending order

If you want the data extracted to a different sheet then start the process from that sheet.

I have a long series of numbers in a column, several of the values are repeated. I want a function which will extract the smallest value from the list, second smallest, third smallest etc and place these in another column. If a value is repeated in the initial list, needless to say, I only want it to appear once in the summary column.

I'm using Office 2000.

Grateful for advice.

Assume your numbers start in A1, and the list cell reference is
defined by Rng  (e.g. A1:A250)

D1:    =MIN(Rng)

This formula must be **array-entered**:

D2:    =SMALL(Rng,ROWS($1:1)+SUM(COUNTIF(Rng,$D$1:D1)))
----------------------------------------

To *array-enter* a formula, after entering
the formula into the cell or formula bar, hold down
<ctrl><shift> while hitting <enter>.  If you did this
correctly, Excel will place braces {...} around the formula.

Then fill down until you start to get #NUM! errors, or use the
following *array-entered* formula in D2 and fill down until you
start to get blanks:

=IF(ISERR(SMALL(Rng,ROWS($1:1)+SUM(COUNTIF(Rng,$D$1:D1)))),
"",SMALL(Rng,ROWS($1:1)+SUM(COUNTIF(Rng,$D$1:D1))))

t. valko say:
I haven't managed to get this to work yet. Is this a formula which will work with Office 2000? I don't understand the syntax of the SUM(--FRQUENCY etc bit or the if statement within the MIN. Grateful for any clarification. Thanks.


The SUM(--(FREQUENCY expression is counting how many unique numbers are in the range. We're using this as an error trap. We compare the number of unique values to the number of rows the formula is copied to. If the number of rows exceeds the number of unique values then the formula returns a blank.

As far as the IF statement within the MIN function...

Once we get a "seed" value with the first formula: =MIN(...), all we need to do is find the next higher value for each successive cell. That's what the MIN(IF is doing. It's looking for the next higher value that is greater than the value from the previous cell. For example, if F2 = 10 then the formula in F3 is looking for the next higher value that is greater than 10.

These formulas will work in any version of Excel.
