scottgem say:

First, This is one of the situations where the rule of relations databases against repeating data doesn't work. Price is a time sensitive value and you need to capture the price at the time of the order. So you need a unitcost field in your Order Items table. This can be filled in automatically by including the cost in the RowSource of the Item combo, then use the Column property to assign it to the Cost control. 

From there you can calculate the Extended cost by multiplying the quantity by the unitcost both on your form and in a query that feeds the invoice.

don cuthbert say:

Another way that I used on all of my family's computers is to create the matching Edition of the Windows 10 ISO.  There are 5 possibilities per language (and 32 or 64-bit versions):

  1. Windows 10 Pro
  2. Windows 10 Pro N
  3. Windows 10 Home
  4. Windows 10 Home N
  5. Windows 10 Single Language

If you know your current Windows 8.1 Edition (usually Home or Pro) you can also upgrade by mounting the created ISO file in Windows 8.1 and running setup.exe from the ISO.  Choose to keep Windows settings and files so that an upgrade is performed and not a clean install.  Do not enter any product key during or after the upgrade.  Skip these screens.

Installing from the ISO any other way will not activate.  Activation is done by detecting that a valid upgrade has been completed and does not use a product key to activate the Windows 10 free upgrade.  The device is then registered on Microsoft Activation servers as a licensed Windows 10 device to allow future clean installs without a product key.

There is a chance that your drive partitions are not setup in an optimal way for Windows 10.  This can happen on a UEFI-capable computer that is setup with Legacy BIOS mode and an MBR-initialized hard disk instead of with UEFI mode and a GPT initialized hard disk.  Windows 10 install seems not to like this Legacy setup when it detects that a more optimal way is available.
