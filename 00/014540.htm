charles kenyon say:

It is possible. It will not be simple.

At least two methods:

  1. The most comprehensive and elegant is to use a UserForm to gather the information and populate the document. This requires vba programming skills.
  2. The second is done in the document itself with IF fields. You would need to decide whether to use Content Controls or legacy formfields. (Unless there is a strong reason to use legacy form fields, you should use Content Controls.)

Both will require substantial study and work. If you want to undertake this, people here will try to walk you through it.

I would advise learning about the IF Field  and about AutoText.

Additional reading:

  • Create a Form Using Word Content Controls
  • Making Forms with Microsoft Word
  • Create a Simple Userform
  • Create & Employ a Userform
  • Mapped Content Controls
  • Repeating Data (Populating Multiple Like Fields)

The links will give you the vocabulary to be able to talk about what you want to do.

In my opinion, you are better off with a group of templates stored as Workgroup Templates , possibly using AutoText in a common global template to share language used in all of them. That global template could hold a custom ribbon tab with your templates shown. This will likely have a shorter learning curve.

caseyfritz say:

There's a lot of questions to be asked here:

  1. What filesystem is on the network drive? (ntfs, ext3, fat, zfs)
  2. How is it being shared? (smb, cifs, afp)
  3. What OS is managing the share? (Windows, Mac, Linux, BSD)
  4. What permissions is the share using? (POSIX, ACLs)
  5. How are your users closing the document/program?
  6. Has the new user refreshed the directory listing after the other user closed the file?

To me it sounds a lot like the windows locking features for example mentioned here. I'm not sure that is particularly applicable but it might be worth checking out.

It is also possible that the share server is preventing access as it knows someone else was just accessing or writing to that file.

Ultimately I doubt you will get an answer here.
