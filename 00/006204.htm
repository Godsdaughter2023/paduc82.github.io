jmarco0921 say:

If you are on insider build 17017, you may be in repair windows restart loop. A lot of insiders are experiencing this problem in different ways. Apparently, there is corrupt system file called volsnap.sys. in this build. Other insiders have been able to get their pc to Advanced Startup-command prompt. Then if they have an old windows folder on their local drive (with that volsnap.sys file) they use the command prompt to copy old sys file to current sys file. That change will get pc to restart correctly. This fix includes making changes to pc boot menu in order to use another Windows OS boot up like DVD or USB as repair source.

I didn't know about this fix on my ASUS laptop when it went into repair loop. The only way I got it fixed was to use my licensed Windows 8 DVD. I had to change boot menu to read DVD as first boot option. Then I clean installed public 16299. I then upgraded to insider build 17017. I did not copy over that sys file from the public build (old window folder) to build 17017 sys file. And sure enough, my laptop on build 17017 went into another repair restart loop again. Now to fix 2nd restart loop I clean installed build 17017 but I found post that said one way to stop your pc when restart loop is to make sure your PC's restore point is turned off. I did that before installing any new software (that need reboot) and each time I attempted new installation I re-checked restore point.

I am on build 17017 now with no restore.

Here is answer post that discusses the volsnap.sys problem

Complete fix credit goes to Mark-Johnson.

"1.  Get an install USB for win 10.   Boot it and go to 'repair my pc' option (bottom of install choice screen).
2. find the 'command line prompt' option and load that.
3. work out what drive letter your normal PC's boot drive is allocated.   It's usually a couple of letters above what the one for first prompt.   or it might still be c: drive ... depends on something i dont understand.   anyway, choose drive letters until you find it (input a: or b: or c: or e: or f: etc) and use "Dir" command to check you got the right one when you find it.    ( happy to be told a better way here ).
4.   First create a new directory - lets say its called volbak  ...  command "md volbak".
5.   then issue command "copy \windows\system32\drivers\volsnap.sys \volbak"
6 - that should copy one file (it creates a backup of the current in use volsnap.sys
7.  then revert to the old volsnap with following command "copy \windows.old\windows\system32\drivers\volsnap.sys \windows\system32\drivers"
8 - you should get prompted to replace volsnap.sys.    Say yes.
9 - thats it.   close the command window, and choose to reboot into windows 10"

a. user say:

Hello Minni,

When you delete a contact it's usually moved to your Deleted Items folder in your mailbox just like when you delete an email message. The Deleted Items folder would be the first place you'd like to check if you deleted contacts. If the contacts are still there, here is how to recover it:

  1. In Outlook, go to your email folder list, and then click Deleted Items.

  2. Use the Search Deleted Items box (or press CTRL + E) to find the item you want to recover. Or you can sort by the Icon column to group all the contacts in your Deleted Items folder.
  3. When you find the contact, right-click it, and then click Move > Other Folder.
  4. To move the contact back to your list of contacts, click Contacts, and then click OK.

If the contacts are not in the Deleted Items folder, the next place to look is the Recoverable Items folder. Here's how to recover contacts from the Recoverable Items folder:

  1. In Outlook, go to your email folder list, and then click Deleted Items.
    • Make sure Home is selected, and then click Recover Deleted Items From Server.
  2. To help you find a specific contact in the Recover Deleted Items window, click the Subject column header to sort items by that column. Look for the name of the contact. You can also sort by the From column and look for blank entries because contacts don't have a value in the From field.
  3. Select the contact you want to recover, click Restore Selected Items, and then click OK.

Important: If Recover Deleted Items From Server is grayed out or isn't there:

  • You might be working in offline mode. To work online, see Switch from offline to online mode.

  • You might not be using an Exchange account. To check, see How do I tell if I have an Exchange account in Outlook. If you don't have an Exchange account (for example, you're using Outlook to connect to an IMAP or POP account) you can't recover a contact that's been permanently deleted or deleted from the Deleted Items folder. If you are connected to an IMAP account, you might be able to recover contacts from the Trash folder. See how: Restore deleted messages when using an IMAP account.

Do get back to us should you need further assistance.
