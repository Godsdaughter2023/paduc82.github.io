matt and suelabbe say: I found a code that I was able to slightly modify.  I do not remember where I saw this or who came up with it, otherwise, I'd give them the credit!  So, thanks to whoever came up with this!...posting in case someone out there is trying to do something similar:

Sub BtlRecSheetCreator()
Dim srcSheet As Worksheet
Dim destSheet As Worksheet
Dim baseCell As Range
Dim lastRow As Long
Dim rOffset As Long

Set srcSheet = ActiveSheet  'Worksheets("MASTER") ' change
Set destSheet = Worksheets("BottlingRec") ' change

lastRow = srcSheet.Range("C" & Rows.Count).End(xlUp).Row
Set baseCell = srcSheet.Range("C5")
'initialize rOffset to look in row 2 first
rOffset = 1
Do While rOffset <= lastRow
 If Application.IsNumber(baseCell.Offset(rOffset, 0)) Then
   destSheet.Range("CD1") = baseCell.Offset(rOffset, -1) ' col B description
   destSheet.Range("CD2") = baseCell.Offset(rOffset, 0) ' col C item #
   destSheet.Range("CD3") = baseCell.Offset(rOffset, 1) ' col D WO #
   destSheet.Range("CD4") = baseCell.Offset(rOffset, 2) ' col E Tax
   destSheet.Range("CD5") = baseCell.Offset(rOffset, 3) ' col F Tank
   destSheet.Range("CD6") = baseCell.Offset(rOffset, 4) ' col G Wine WO #
   destSheet.Range("CD7") = baseCell.Offset(rOffset, 10) ' col M Line
   destSheet.Range("CD8") = baseCell.Offset(rOffset, 11) ' col N gal
   destSheet.Copy
 End If
 rOffset = rOffset + 1
Loop
MsgBox "Job completed"
End Sub

rohn007 say:
You have found the cure to a very common problem. MS developers are aware of it.  Literally hundreds of people have posted questions about this issue.

One person came up with the idea to create a batch file to speed up the process of restarting the service

***********

One person reported that they have to restart the service every time after booting the computer.

In that situation, it may be easiest to create a batch file to issue the commands.  Then you can put it on your desktop or task bar to have quick access to run it as required.  The brain dead simplest form of this .BAT file would be:

net stop “Microsoft Office Service”

net start “Microsoft Office Service”

but it is better if you do some error checking to make sure you know what is going on if there are any problems. This page has a detailed batch file you can use as a template:

http://forums.techguy.org/windows-xp/831463-batch-script-restart-service.html

net stop

if %errorlevel% == 2 echo Could not start service.

if %errorlevel% == 0 echo Service started successfully.

 

net start “Microsoft Office Service”

if %errorlevel% == 2 echo Could not start service.

if %errorlevel% == 0 echo Service started successfully.

 

 

This page describes the whole process of creating a simple batch file: http://lifehacker.com/5645770/create-service+stopping-batch-files-to-optimize-your-pc-for-specific-tasks

This link goes into more detail: http://stackoverflow.com/questions/133883/stop-and-start-a-service-via-batch-or-cmd-file

http://www.aspnet-answers.com/microsoft/WSH/31708197/net-start-return-codes.aspx
