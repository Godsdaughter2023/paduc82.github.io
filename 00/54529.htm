SFC /scannow wants to have the XP installation CD that has the same service pack as is running on the computer.  That's why I said that using an sp2 CD with an sp3 system won't work (at least without a lot of complaints from sfc).

There is a great deal of misinformation and misunderstanding about what sfc /scannow does and does not do.  In Windows XP, the Windows File Protection feature monitors key system files.  If someone (or some malicious program) attempts to modify or delete one of the monitored files, WFP silently replaces the "bad" file with a good copy.  It gets the good copy from a hidden system directory at C:\Windows\system32\dllcache.  In Windows XP, running sfc /scannow only repopulates the dllcache folder so that it has all of the monitored files.  For a much more complete discussion of this, see THIS and the follow-up post by the same author a bit further down in the same thread.

As it turns out, regtrace.exe is one of the 3498 system files monitored by Windows File Protection -- but a copy may or may not be in C:\Windows\system32\dllcache.  I suspect that dllcache includes a copy of this file ONLY if the file is installed in C:\Windows\system32.

Regtrace.exe is not, however, installed in C:\Windows\system32 for most Windows XP installations.  My guess -- I don't know for sure -- that regtrace.exe gets put in C:\Windows\system32 (and a copy in the dllcaceh folder) if you install the optional IIS component. 

Bottom line:  if chkdsk reports that your hard drives are OK, don't worry about the Event Viewer messages you cited.  On general principles, however, you might want to do the following:

If you haven't done so recently, download, install, update, and run these two free programs. 

MalwareBytes AntiMalware

SUPERAntiSpyware

Use the free versions, not the free trials of the "pro" versions.  Pay attention during installation to uncheck any prechecked boxes that would install some additional third-party program.

Don't run the scans simultaneously.  Each scan will take a long time.  Start one up and go do a non-computer chore.  When the scan complete, let the program deal with anything it finds.

When done, you may keep or uninstall either or both programs.  If you keep SAS, I suggest changing its default setting to NOT run automatically when Windows starts.
