1. Open the Start Menu. Click on Settings.

techfreak_ say:

IP Webcam software might work with the desktop version of Skype. That type of software is generally not compatible with UWP Apps, only desktop applications. A developer would have to specifically make a version compatible with UWP Apps, otherwise that virtual software will not be seen.

In regards to your issue: If you are using IP Webcam software and it functions in an older version of Skype but it is incompatible or unstable with the latest versions, some type of compatibility is lacking in that IP Webcam software. It is more than likely utilizing older, unsupported protocols/filters or is missing a newer protocol. You would need to contact the maker of that software to see if they could provide a more up to date and compatible software. You could try locating other software that might work, work daisy chained, or utilize software supporting Newtek NDI, and enable NDI support in Skype. NDI support is built into both Skype for Desktop and the Skype for Windows 10 UWP App for just this reason. It is the compatibility added to Skype to support such configurations. I do not know if there are free 3rd party implementations out there.

Hi,

 


If you receive Windows Update error 80072efd while checking for updates, the cause might be a connection interruption between your PC and the Windows Update service.

 


Try the following methods and check if it helps.

 


Method 1:

 


You may refer to the steps from the following Microsoft article and check if it helps.

 

 

Error 0x80072efd when you check for updates by using Windows Update

 


Temporary connection-related errors may occur when you use Windows Update or Microsoft Update

 

 

Method 2:

 


I would suggest you to run the following command and check if it helps.

 

a)    Press Windows Key + X and select the option Command Prompt (Admin).

b)    Type the following command, and then press ENTER:

netsh winsock reset press Enter

netsh interface ipv4 reset  press Enter

netsh interface ipv6 reset   press Enter

Restart the computer and check if it helps.


Method 3:


I suggest you perform the following steps and check if it helps.



a. Press Windows Key + R and type "NCPA.CPL" (without quotation marks) and press Enter.

b. Right click on the connection that you use for the local connection, and then click "Properties".

c. Click to select "Internet Protocol Version 4 (TCP/IPv4)", and then click "Properties".

d. In the Internet Protocol window, let's change the "Preferred DNS server" to 208.67.222.222

e. Click 'OK' twice to complete the modification.

  

 

Hope it helps.
