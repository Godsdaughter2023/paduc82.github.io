I suggest you to follow these methods and check if it helps:

Method 1: Disable Antivirus program temporarily.

Important Note: Antivirus software can help protect your computer against viruses and other security threats. In most cases, you shouldn't disable your antivirus software. Method 2: Disable proxy connection.

If you have setup a proxy connection, I would suggest you to disable the proxy and check. Refer these steps:

  1. Open Internet Explorer browser.
  2. Click the Gear box at the top right corner in Internet Explorer.
  3. Click the Tools button, and then click Internet Options.
  4. Click the Connections tab, and then click LAN settings.
  5. Uncheck the option “Use a Proxy Server for your LAN”.
  6. Click Apply and Ok to save changes.

jeeped say:
This might seem like a simple enough task, but in actuality it is fairly complex as Excel 2007 has no native functionality to determine whether a cell's background (aka Fill) or font color is blue (or any of the 20-30 numerically indexed color codes that could reasonably be described as blue).

The first thing you will require is a User Defined Function or UDF that determines whether the cell in column B has a blue fill and/or blue font. Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module or tap Alt+I, M. Paste the following into the new pane titled something like Book1 - Module1 (Code),

Public Function udf_HasColor(rIsIt As Range, rHasIt As Range) As Integer
    'returns: 1 - Fill color matches
    '         2 - Font color matches
    '         3 - Fill & Font color match
    Application.Volatile
    udf_HasColor = _
      (rIsIt.Interior.Color = rHasIt.Interior.Color) * (rHasIt.Interior.Color <> xlAutomatic) + _
      (rIsIt.Font.Color = rHasIt.Font.Color) * (rHasIt.Font.Color <> 0) * 2
End Function

Tap Alt+Q to return to your worksheet. Take any unused cell (e.g. Z1) and apply either a blue fill or a blue font or both. Test this in another unused cell (perhaps C1) by referencing B1 and Z1 like this,

=udf_HasColor(B1, $Z$1)

The cell with the formula (e.g. C1) will contain 0 if B1 has neither the same font color or fill color as Z1, 1 if only the Fill color is the same, 2 if only the font color is the same and 3 if both the Fill and font color match Z1. once this tests properly you can use it in your Conditional Formatting Rules.

Create three Conditional Formatting Rules for column A based upon the following formulas.

=AND(TODAY()-$B1>1, TODAY()-$B1<31, NOT(udf_HasColor($B1, $Z$1)))        ◄ pink 
=AND(TODAY()-$B1>30, TODAY()-$B1<61, NOT(udf_HasColor($B1, $Z$1)))      ◄ green 
=udf_HasColor($B1, $Z$1)=1       ◄ blue 

Note that the final formula is to match Fill only. Adjust to 2 or 3 depending upon what cell formatting you want to match. With the correct Fill color in Z1 you should receive results like the following.

            

Lastly, even with Application.Volatile, Excel does not trigger a recalculation event based upon a change in cell formatting. You will have to force one yourself or tap F2<enter> on any cell.

CCleaner (which I recommend to some advanced users) only "involves the registry" if you choose to use its Registry Integrity component.

Unlike most similar utilities, CCleaner prompts the user to create backups of the changes being made when the Registry Integrity component is used. Another fly-in-the-ointment that could cause hmmapi.dll.mui (or any other IE8-related file) to go missing would be failing to disable all realtime protections prior to installing (or uninstalling) an IE upgrade or a Windows Service Pack. Such realtime protections include those afforded by your anti-virus application, any anti-spyware applications (including Spybot Tea Timer & Ad-Aware's Ad-Watch but not Windows Defender), and any third-party firewall (e.g., Norton; McAfee; ZA; Comdo; etc.).

No Windows version to-date supports multiple IE versions nor does MS support hacks to install other IE versions in a given OS (vs. in a virtual environment).
