a.log into your gmail/google apps account and filter for the emails that have labels.
b.select all emails with labels.
c.click on the labels icon.
d.click in the boxes next to each label until the boxes for all labels have been cleared.

Hi Syedsafwan,

If you are using the Outlook 2013, here is a workaround for you.

• Select the folder for which you want to delete duplicate items.

• Right-click a column heading, and then click Field Chooser.
• From the list at the top of the Field Chooser, click to select the All <item name> fields.
• Drag the Modified field to the table heading. 
• Verify that the duplicate items have a unique date from the original set of items. If it is unique, click the Modified heading so that the items are sorted by this field. 
• Click the first item in the set that you want to delete, scroll to the last item in the set that you want to delete, and then click the last item while you hold down the SHIFT key.

tony chen chn say:

Method one:

First of all, you need a private key of the certificate in order to sign anything.

Then, the certificate must include certain flags in Key usage and Extended Key Usage extensions.

And the last (but probably the most important) is that the certificate or its CA certificate should be trusted by the application that verifies the signature.

To make the story shorter, purchasing a certificate from one of known CAs (eg. GlobalSign) is the best option.

 

About digital certificate, please refer to:

http://www.digitallycreated.net/Blog/38/using-makecert-to-create-certificates-for-development

 

Method two:

trying to sign and save a particular XL file with a personal certificate created with SelfCert.exe and found out that there

seems to be a length limitation for the VBA code lines when digitally signing the code.

I used to have a very long source code line (actually a comment) in a module (1023 chars): after splitting this comment line into 2

comments of approx. 500 chars the signing worked fine. The behaviour was systematically reproducible.

Hope this helps, take a while to find the cause...

diane poremsky mvp (slipstick.com) say:

Are you trying to create a search folder that displays mail sent to david[at]my-domain.com or sent to the display name "David Smith" ? 

If so, you need to add both the addresses and display names to the filter.

 

You can create the filter on the Advanced tab of the filter dialog and enter the email address and the display name. An example of what the filter will look like is at the link I posted. If you don't want to restrict it to only mail sent to the address (ie, sent to you and someone else), use 'contains' instead of 'exactly'.

Or you can enter the names and addresses separated by comma's in the Sent To field on the messages tab: david[at]my-domain.com, "David Smith"

 

As an FYI, you can't filter for mail sent to you as a BCC as your name won't be in the To or CC fields. (Any messages that do not contain your address in the To or CC field were BCC'd to you.)

 

An alternate method is to create a rule that checks the To or CC field for your address and add a category then search on the category. This eliminates the problems of adding variations in your display name to the filter dialog.
