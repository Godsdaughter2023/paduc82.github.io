wunders say: Generally, you must have administrator privilege on the remote computer in order to shut down that computer.  Since both computers have XP Pro, you are halfway there (Remote users always authenticate as "Guest" on XP Home).  The computer that you are trying to shut down must have "Simple File Sharing" disabled as otherwise you cannot authenticate on that computer as anything except the "Guest" user.  So try turning off Simple File Sharing, make sure that your username and passwords are the same on both computers, and make sure that you have admin privilege on the remote machine and then you can shut down that machine with the "shutdown" command.

"How to disable simple file sharing and how to set permissions on a shared folder in Windows XP"
  < http://support.microsoft.com/kb/307874 >

Actually it is easier to use Remote Desktop.  Even though you don't have a "Shutdown" option under the Start menu, you can always bring up a command prompt window (Start -> Run -> "cmd") and enter the command:
   shutdown -s -f -t 35
This will give you 35 seconds to logoff that computer before it shuts down (assuming you have admin privilege).

You can also look into the "psshutdown" command from the PSTools Suite
  PSTools:  < http://technet.microsoft.com/en-us/sysinternals/bb896649 >

Hi Nayjd71,

You may try the following steps on your vista machine.

Step 1

Check the connectivity in Safe mode with Networking:

If the machine is not connected to a wireless network, you may also try to start the computer in safe mode with networking and check if you experience the same connectivity loss issue.

To start the computer in safe mode with networking, you may follow the given steps:

1.       Reboot the computer and keep hitting the F8 key.

2.       Choose “Safe mode with Networking” from the advanced boot menu options and hit Enter.

3.       Check if you experience the intermittent connection loss.

a)      If no, you may try to configure the machine in clean boot state and check if you have the problem fixed in normal mode. You can follow the given article:

 http://support.microsoft.com/kb/331796

Step 2

Check if the DNS client, DHCP client, RPC, RPC locator, HTTP SSL, Computer Browser services are started and automatic.

 To do this, type services.msc in start search box and press enter. In the services window check if the above services are started and automatic.

http://windows.microsoft.com/en-US/windows-vista/Manage-services-in-Windows-Vista-from-Windows-Vista-Inside-Out

Step 3

Reset TCP/IP stack and then check if it's working.

To reset the TCP/IP stack go to this article and either click on "Fix it for me" or follow the instructions to fix it yourself:
http://support.microsoft.com/kb/299357
 
Disable IPv6:
Try uninstalling IPv6 on all interfaces, removing any IPv6 virtual adapters, and resetting the TCP/IP stack. To remove IPv6, go into the properties for each network adapter and either uncheck the box next to the protocol "Internet Protocol version 6 (TCP/IPv6), that will disable it, or select it and click uninstall, which will remove it off the computer. Then go into device manager and remove any 4to6 adapters, WUN miniport adapters, or tunnel adapters.
 
NOTE: You should do this for each network connection.

ken blake say:

If you're trying to boot from a CD and it fails, the CD driver is *not* the issue. Drivers are used when Windows is running, and Windows is not running when you boot from a CD.

Either the CD drive itself is bad and needs to be replaced, or the CD is defective.
