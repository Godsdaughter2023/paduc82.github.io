Protecting a table in word document - Microsoft Community

dr.satheeshraj say:

Protecting a table in word document

I have a word document which contains a table.
I need my collegues to fill up the fields in the table whereas they are not allowed to edit the format of the table. 
Also they should enter the values only in the fields allowed in the table and the remaining parts of the documents should remain unchanged. The users should not be allowed to edit the others areas in the document including the header and the footer.

Is it possible to do this in word 2007. I know that it would be possible to do it in Excel 2007. But word is it possible.?

Please help me in this regard,....
jay freedman say:
Yes, it is possible to do it in Word 2007 in either of two ways.

One way uses what Word calls "Legacy Form Tools" and requires you to "protect the document for forms". A good short explanation of the procedure is at http://cybertext.wordpress.com/2010/07/22/word-2007-setting-up-protected-form-fields/.

The other way uses "content controls", which are inserted by the other buttons in the Controls group on the Developer tab of the ribbon (the Aa buttons, etc.). Put a content control in each table cell that needs to be filled in. When they're all in place, select the entire document, click the Group button to the right of the control buttons, and choose Group from the dropdown menu. Unlike protected legacy forms, the users will be able to put the cursor in parts of the document outside the controls, but they won't be able to type or change any formatting there.

One last bit of advice: Don't mix the two methods in one document. Content controls often don't work properly when legacy forms protection is turned on; and legacy form fields don't work when forms protection is turned off. And don't use the ActiveX section of the Legacy Tools gallery -- they don't play nicely with either kind of form.

Hi,

Yes, I would use a VBA procedure (same as a macro) that would look a bit like this:

sub Milestonecomplete

Dim Job as task

Dim Pred as task

dim PCT as integer

for each job in activeproject.tasks

if not job is nothing then

if job.milestone then

pct=100

for each pred in job.predecessortasks

if not pred.percentcomplete=100 then

pct=0

exit for

end if

next pred

job.percentcomplete=pct

end if

end if

next job

end sub

rod gill say: I would only do this for milestones with predecessors as some schedules have milestones set by dates to indicate other types of events. So I use:

Sub UpdateMilestones()
'Called by Project_BeforeSave event. Sets Milestones to 100% complete if all predecessor tasks are complete
Dim Complete As Boolean
Dim Tsk As Task
Dim TskPred As Task
    For Each Tsk In pj.Tasks
        If Not Tsk Is Nothing Then
            If Tsk.Milestone And Tsk.PredecessorTasks.Count > 0 Then    'Only if milestone with links
                Complete = True
                For Each TskPred In Tsk.PredecessorTasks
                    If TskPred.PercentComplete < 100 Then
                        Complete = False
                        Exit For
                    End If
                Next TskPred
                If Complete Then
                    Tsk.PercentComplete = 100
                End If
            End If
        End If
    Next Tsk
End Sub
