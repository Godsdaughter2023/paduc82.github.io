yadirabautista say:

I noticed that someone else needs help with this. If you need the User Account screen for something more complicated, then I can't really help you, but, if you want to make a local user account, I did find a solution. If you're an administrator on your computer, then do the following steps:

  1. Search "cmd" on your start menu.
  2. The program that comes up should be Command Prompt. Right click on the program and select "Run as Administrator".
  3. In the Command Prompt, put the following text in:
  • net user "[user account name here]" [optional password here] /add
  • Ex. net user "Abigail" 1234 /add will add a user account with the name Abigail and the password 1234

If you want to log in with a Microsoft Account, just make a local user account, log into it, and then log into the Microsoft Account from there (assuming your login screens do work and it's only the add user screen that remains blank). Hope this helps!

I have an idea, in the absence of a proper solution.
create a scheduled task to force a check for updates.
1) open task scheduler, click on "create basic task" in the right hand pane
2) give it a name and add a description, click next
3) set the trigger to daily and click next
4) set the start time, I suggest a few minutes after 3am in order for the pc to finish booting. and click next
5) select "Start a Program" and click next

/updatenow
and click next
8) check the details, tick the " open properties for this task...." and click finish
9) in the properties dialogue, on the general tab, confirm that it is running as an administrator
10) on the conditions tab you might want to tick "Wake the computer to run the task" and check other conditions
11) on the Settings Tab, stop the task if it runs longer than 4 hours and if the running task does not stop when requested, force it to stop
12) click ok

If you try to install an old 16-bit Windows application on a computer running Windows 7 x64, you will typically get an error like this:

”Unsupported 16-bit Application…  Setup.exe cannot start or run due to incompatibility with 64-bit versions of Windows.  Please contact the software vendor to ask if a 64-bit Windows compatible version is available.”

But what if you really need to run that old 16-bit application on the computer because the app is critical for some aspect of your business?  In that case, you could install Windows Virtual PC on your computer and download the Windows XP Mode virtual machine from Microsoft.  Because Windows XP Mode uses 32-bit Windows XP Professional, you should be able to install your 16-bit app in the virtual machine and launch the app from your Windows 7 start menu.
