rich michaels - mvp (greatcirclelearning.com) say:

I think this has become more complex with the advent of Office 365 and online accounts for product registration.

In the past the originating "Author" name of document was taken from the user name setting in Word. The user name setting is a required field and in corporate or OEM installed versions, many times it was pre filled with Microsoft or HP, etc because without it, Word spit up all over itself. This used to happen regularly in pre 2007 versions. If the end user did not go into their Option settings and change it, the user name stayed as it was with the pre filled name.

Now what I see happening is on the initial creation of new documents the Author is recorded as the Account Holder name. This occurs when the default Normal template is used to begin the document. If on the other hand the document is started from a custom template (Microsoft or other 3rd party supplied) the Author is usually recorded as the user name that is set within Word itself. However, there can be exception to this if the template is macro driven.

This same thing I am seeing happen on the name of the person who last modified the document. If there is an online account registration record then the name comes from the registration account otherwise it is coming from the user name setting in Word.

So to answer your multiple choice question I would have to add a 5th option... "All of the above" :-)

jay freedman say:

As a further complication in Office 2013 (with or without Office 365), there is a check box in File > Options > General, labeled "Always use these values regardless of sign in to Office". If that is checked, then whatever is in the User Name box just above it will be entered as the Author at document creation time. The User Name box can be changed at any time. It can also be changed by inserting an Author content control from Insert > Quick Parts > Document Properties and changing the text in the control.

The Company property can be changed on the File > Info page simply by clicking on the field and typing a new value, or by inserting a Company content control from Insert > Quick Parts > Document Properties and changing the text in the control.

greg maxey say:

Adding to the previous replies, there are also various ways to manipulate these values with code.

The Author and Company properties are also stored and accessible in the document built-in customXMLparts.  Author is a core property named "Creator" while Company is an extended property named "Company."

Last Modified is actually the built-in property "Last Author."  Last Author is apparently read only and determined by Application.UserName.

Sub ScratchMacro()
'A basic Word macro coded by Greg Maxey
  With ActiveDocument
    'Author and Company built-in properties are interconnected with _
    the built-in customXMLnodes creator and company.
    Debug.Print .BuiltInDocumentProperties("Author").Value
    Debug.Print .BuiltInDocumentProperties("Company").Value
    .CustomXMLParts(1).SelectSingleNode("/ns1:coreProperties[1]/ns0:creator[1]").Text = "Somebody else"
    .CustomXMLParts(2).SelectSingleNode("/ns0:Properties[1]/ns0:Company[1]").Text = "Some other company"
    Debug.Print .BuiltInDocumentProperties("Author").Value
    Debug.Print .BuiltInDocumentProperties("Company").Value
    .BuiltInDocumentProperties("Author").Value = "Gregory K. Maxey"
    .BuiltInDocumentProperties("Company").Value = "The Anchorage"
    Debug.Print .CustomXMLParts(1).SelectSingleNode("/ns1:coreProperties[1]/ns0:creator[1]").Text
    Debug.Print .CustomXMLParts(2).SelectSingleNode("/ns0:Properties[1]/ns0:Company[1]").Text
    'Last Modified is accutually the built-in property "Last Author"
    Debug.Print .BuiltInDocumentProperties("Last Author").Value
    'Which is apparently a read only property
    .BuiltInDocumentProperties("Last Author").Value = "Somebody else"
    .Save
    'Notice no change
    Debug.Print .BuiltInDocumentProperties("Last Author").Value
    'Determined by Applicaton.UserName
    Application.UserName = "Somebody else"
    .Save
    'Notice the change.
    Debug.Print .BuiltInDocumentProperties("Last Author").Value
    Application.UserName = "Gregory K. Maxey"
    .Save
  End With
End Sub

So amplifying Rich's lesson.  Using these properties you can make it appear that anyone anywhere created or modified a document and there is really no way to tell actual who does either.

joeu2004 say: Carlie wrote:

SUMIF is the wrong function in this context.  The first and third parameters should be the same size and shape.  Otherwise, SUMIF is probably doing what you expect.  GIGO.

Simpler implementation:

=SUM(J21,J34,J35) + IF(J36="yes,SUM(J14:J20,J24:J33))

And you might replace the first SUM with J21+J34+J35.  You need SUM if any of J21, J34 or J35 might be text that you want to ignore.
