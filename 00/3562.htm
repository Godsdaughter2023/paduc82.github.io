taylor jansen say:

UndoRecord with subform

I have a main form with a subform. The form is used for input new record only.

I want to add a button to undo a record. I got the following from this forum:

Private Sub Cancel_Click()
  If Me.Dirty Then
         Select Case MsgBox("Confirm cancel?", vbQuestion + vbYesNoCancel)
             Case vbYes
                 Me.Undo
            Case vbNo
                Exit Sub
             Case vbCancel
                 Exit Sub
         End Select
     End If
     DoCmd.Close acForm, Me.Name, acSaveNo
End Sub

However I found that the record cannot be undo
1. if the cursor is in the subform
2. if there is data in the subform

I know I should delete the record in subform first before I can undo the record in main form. Can anyone suggest the code?
ken sheridan say: If focus has been moved to the subform then the parent form's record will have been updated, so cannot be undone.  You will need to delete the parent form's record, and any related rows in the related table to which subform is bound.

Firstly, however, you will need to ensure that in the relationship between the parent form's table and the subform's table referential integrity and cascade deletes are enforced.

So, if we assume for this example that the parent form is bound to a table Clients with a numeric primary key ClientID, the code would be along these lines:

Private Sub Cancel_Click()

On Error GoTo Err_Handler

    Const MESSAGETEXT = "Confirm deletion of current " & _
        "record and any related records?"
    
    Dim strSQL As String
    Dim strCriteria As String
    
    strCriteria = "ClientID = " & Nz(Me.ClientID, 0)
    strSQL = "DELETE * FROM Clients WHERE " & strCriteria
    
    If MsgBox(MESSAGETEXT, vbQuestion + vbOKCancel, "Confirm") = vbOK Then
        ' has record been saved to table?  If so delete it,
        ' otherwise undo current record
        If Not IsNull(DLookup("ClientID", "Clients", strCriteria)) Then
            CurrentDb.Execute strSQL, dbFailOnError
        Else
            Me.Undo
        End If
        DoCmd.Close acForm, Me.Name
    End If

Exit_Here:
    Exit Sub
    
Err_Handler:
    MsgBox Err.Description, vbExclamation, "Error"
    Resume Exit_Here

End Sub

BTW you don't need to include the acSaveNo argument when closing the form; that only applies when changing a form's definition, not editing data.
