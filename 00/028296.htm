Task Scheduler: Only start a task if its not already running - Microsoft Community

mprother say:

Task Scheduler: Only start a task if its not already running

I want to keep Outlook running while I am logged on to my computer however I tend to inadvertently close it.  So I have set up a task to run Outlook on logon and every hour thereafter.  But I don't want it to start up another instance if it is already running.  So under Properties\Settings for the Task I have set:

'If the task is already running then the following rule applies:' to 'Do not start a new instance'

However, every hour a new instance of Outlook starts up irrespective of whether one (or more!) are already running.

Is there something I'm missing?

mprother say:

Thanks for your reply, I posted on the Technet forum as well and Oberwald pointed out that in Task Scheduler the 'if the task is already running' option refers to the task scheduler task rather than the application that it starts (subtle difference). And of course every hour it checks and that task isn't running so it restarts another instance of Outlook.

He suggested that I should get Task Scheduler to start a simple batch file at logon (and every hour thereafter:

@echo off
tasklist | find /i "Outlook.exe" && goto :eof
start /b "Outlook" "C:\Program Files (x86)\Microsoft Office\Office11\Outlook.exe"

So this checks to see if Outlook is running and ends if it is or starts it if it isn't.  It now works perfectly.

The title page should be page 1. If you want a blank page preceding it (a flyleaf or half-title page), then you need to add two pages (making the title page page 3).
