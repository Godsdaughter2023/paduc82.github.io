If you add new entries (rows) in 'Source' Range,

it is logic that Pivot table will not update the new row (rows).

Update (refresh), is for any change which takes place in A2:K100 (in Source range)

because you will lose all  data from this column in Pivot Table)

jeppe_ita say:

Best practice: How to work with PowerPoint templates in different languages?

Hi Office experts!

I'm looking for at bit of advice about working with different languages in blank templates in PowerPoint.

Our company communicates with customers in 2-3 different languages. We're thinking that we should have a PowerPoint template for each language.

We're talking about blank presentations (with a few handfulls of slide variants) and not completed presentations.

However, this does not seem like that's easily managed in PowerPoint, buy I fear were trying to solve a non-existing problem ;-)

  • Is it even possible to create, say, a German and an English base template, or is it best pratice to just work with one master template in English, and have users set the desired languge of their presentation in PowerPoint, on a case-by-case basis, once a new presentation is created from the template?

(Obviously, completed presentations can be saved as individual language versions (German/English), after they've been translated. What i'm talkin about here is the 'blank' base template, containing just slide variants and no preset information - only corporate logo and colorscheme, etc.)

Thanks for any help you can give me.

steve rindsberg say:

I would create templates for each language, ideally using a computer and version of Office IN that language.

For one thing, if you start with the correct language, all of the "Click here to {whatever}" prompts on the masters/layouts will be in the correct language.

For another, it can be very difficult (impossible w/o writing code or using a commercial add-in, in fact) to change the language of every bit of text in a presentation, and if you don't change them all, users will be frustrated and annoyed when their text is underlined as a misspelling because some setting has been left in, say, English but they're typing German. 

If someone has started with a template in one language and needs to pass the presentation off to someone else who'll use a different language, the next user can either change newly added text to the correct language using PPTs own tools, use an add-in to change the whole presentation to the new language or just live with the red-squiggly "spelling error" underlines ... whichever's easiest/most appropriate to the situation.
