j.e. mcgimpsey say: There's no capability within XL to do this.

However, it could be done using VBA (i.e., macro). The macro probably wouldn't be worth writing for a single instance, but if you do this regularly, it would save you time.

A very quick and dirty macro might look something like

    Public Sub SortFontCharacteristics()
        Dim nLastCol As Long
        Dim rSortRange As Range
        Dim rCell As Range
        
        With ActiveSheet
            With .UsedRange
                nLastCol = .Columns.Count + 1
                With .Cells.Resize(, nLastCol)
                    Set rSortRange = .Columns(nLastCol)
                    For Each rCell In .Columns(1).Cells
                        With rCell
                            .Cells(1, nLastCol).Value = _
                                IIf(.Font.Bold, "b", "z") & _
                                IIf(.Font.Italic, "i", "z") & _
                                Format(.Font.Size, "000.0")
                        End With
                    Next rCell
                End With
            End With
            With .Sort
                With .SortFields
                    .Clear
                    .Add Key:=rSortRange, SortOn:=xlSortOnValues, _
                            Order:=xlAscending, DataOption:=xlSortNormal
                End With
                .SetRange .Parent.UsedRange.Cells
                .Header = xlGuess
                .MatchCase = False
                .Orientation = xlTopToBottom
                .Apply
            End With
            rSortRange.EntireColumn.Delete
        End With
    End Sub


Post back if you want to pursue a macro, and I'm sure you can get more guidance.

scottgem say:

Actually, what you need is a central transactions table. This table will record all movement of stock in and out. 

So your Orders and Purchase orders would use this transactions table as the detail table. Each transaction would have a transaction type that would identify whether the transaction was incoming or outgoing. 

Also Quantity on Hand is calculated not stored. You don't store the quantity and add/deduct from that, Instead you use a query to calculate on hand inventory by adding all the incoming transactions and subtracting the outgoing ones.

I must say, by the way, that tracking usage down to the recipe level is very ambitious. Frankly, I would suggest that you start off by tracking items in container. So assuming you buy milk in Liter containers, you would track when each new container is opened or empty.
