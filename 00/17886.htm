Excel VBA nested error handlers

Hi All,

I am facing a strange behavior with nested error handlers.

I have a userform with his own error handler and somewhere in the code I call a function to filter an ADODB recordset. (Excel has sometimes issues with mixed data fields.)

The function has its own error handler.

Basically the code is as:

Sub UF_cmdButton

on error goto ...

variant=function

End Sub

Function Filter

on error goto ...

test

End Function

I know for sure that the test on the current record will fail because I am using:

If CDbl(RecordSet.Fields(intWherePos)) < CDbl(varWhereVal) Then blnAddRecord = True

and that field contains only "yes" or "no".

The strange thing is that it jumps to the function error handler for the first record and to the error handler of the UF for the second record. I tried already a lot of things without success.

I have a work-around, that is not the issue, I was just wondering why the first record is handled correctly and the second bumps me out.

Thanks for reading.

jan karel pieterse say:
Since your UF does not appear to have a Resume statement in the error handler, the first error will bring VBA in error handling state, but it never returns out of the error handling state. And error handler MUST either be terminated with a Resume or Resume Next statement, or an Exit Sub or Exit Function. If not, the error handling state is not reset and the next error will be handled differently (cause a runtime error).

systemsadminhb say:

Outlook 2011 (14.1.2) not showing new mail, and other troubles.

We have two major issues with Outlook 2011 (14.1.2) right now.  The first is affecting everyone in the office with missed mail messages and late delivery issues.  Outlook will not show all new messages in the inbox, sometimes notifying of new messages with the dock icon badge, sometimes not.  Restarting Outlook *usually* fixes it, but sometimes it doesn't.  It happens both with Exchange and POP accounts - sometimes it seems to be a problem with the unified Inbox, sometimes it's confined to the individual account.  There is another post about this issue here and it is still unanswered:  http://answers.microsoft.com/en-us/mac/forum/macoffice2011-macoutlook/outlook-2011-1412-exchange-inbox-not-showing-all/e3479ee5-f3a6-4a2a-b2fe-55fc6c7ccb53?msgId=997946da-9d3f-4451-a7ef-786c87ae34d5 (it actually says 'answered' but I don't consider that one solved)

The second I just discovered today.  My iPhone has experienced several total loss of contacts and I finally tracked it down to the Sync Services within Outlook.  What happens is that when a contact is added or edited within Outlook, and you are syncing with the Mac Address Book, it replaces the entire contents of the Address Book with the record(s) being edited in Outlook.  I was able to repeat it several times (after turning off MobileMe syncing - reloading the phone repeatedly got tiresome) by adding a contact or changing a contact.  This is absurd.  I have turned off Sync Services for the time being and am laboriously hand-syncing my contacts.
