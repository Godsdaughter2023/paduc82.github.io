ken sheridan say: You might be interested in the following alternative method, which uses some maths originally put forward by James Fortune in the old MS Access newsgroup, and which I wrapped in the following function following some discussion with David Fenton in the newsgroup:

Public Function RoundToInterval(dblVal As Double, _
            dblTo As Double, _
            Optional blnUp As Boolean = True) As Double

   ' rounds up by default.
   ' to round down pass False into function as
   ' optional blnUp argument
   
    Dim intUpDown As Integer
    Dim lngTestValue As Long
    Dim dblTestValue As Double
    Dim dblDenominator As Double
    
    If blnUp Then
        intUpDown = -1
    Else
        intUpDown = 1
    End If
   
    dblDenominator = intUpDown * dblTo
    dblTestValue = dblVal / dblDenominator
    lngTestValue = Int(dblTestValue)
    RoundToInterval = intUpDown * lngTestValue * dblTo

End Function

Calling it in the debug window, using the same values as in the example at the above MS link gives:

? RoundToInterval(3.33, 0.1)
 3.4

? RoundToInterval(3.33, 0.1, False)
 3.3

For time rounding, I assume that in your example you are rounding up to the nearest 15 minute interval, so:

? Format(RoundToInterval(#1:53#,#0:15#),"hh:nn:ss")
02:00:00

The Format function is used here to present the returned value in hh:nn:ss format.  The underlying numeric value would otherwise be returned:

? RoundToInterval(#1:53#,#0:15#)
 8.33333333333333E-02
