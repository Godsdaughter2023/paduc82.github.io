don cuthbert say:

There are scripts written by users available online that convert the Windows Update downloaded and processed files into an Install.wim or ISO for the Fast Ring builds. I use one called uup-converter-wimlib that I find through the MDL Forums website. The use of such solutions is not supported nor recommended by Microsoft, but it does provide you with a matching Fast Ring Install.wim file to use with DISM RestoreHealth.

As far as I know, DISM RestoreHealth will still give an error that looks like it cannot find the Source files (the same happens with the public build) but it still successfully finds and uses the WIM file to repair the component store, as shown in the CBS.log. For some reason it looks for a FlightSigning package that does not exist in the WIM file, which produces the error. If you run DISM while Windows is offline (after booting from install media) it will complete 100% if there are no component store errors.

Also, if sfc /scannow completes 100% you are probably OK.

Another way to get Fast Ring Source files is to install the Fast Ring build in a Virtual Machine (or another computer) and then share out the VM's Windows folder to point to as the DISM repair source.
