doug robbins - mvp office apps & services (word) say:

The issue is that when a document is saved as Web Page, Filtered, the following message appears

And, as that message indicates “Some Office features may not be available when you reopen this page” and one of those features is footnotes\endnotes, which, by saving the document in that format,  are converted to hyperlinks.

It is for that reason that when the document was re-opened and another footnote\endnote is inserted, that footnote\endnote will be treated as the first one in the document.

The document should be kept in Word document (.docx) format until complete and only then should it be saved as a Web Page Filtered.

By separating the footnotes from the main document and running a macro to delete the hyperlink fields from that document and then saving it with the name of source.docx, and then running the following macro with the following code on the original document, it was possible to recreate the endnotes (as regular endnotes) in that document

Dim doctarget As Document
Dim docsource As Document
Dim rnge As Range
Dim i as Long, j as Long
Set doctarget = ActiveDocument
Set docsource = Documents.Open("L:\Broken Documents\source.docx")
doctarget.Activate
With doctarget.Range
    j = 1
    For i = 1 To .Fields.Count
        If .Fields(i).Type = wdFieldHyperlink And InStr(.Fields(i).Code, "ftn") > 0 Then
            Set Rnge = .Fields(i).Code
            Rnge.Collapse wdCollapseStart
            Rnge.MoveStart wdCharacter, -1
            Rnge.Collapse wdCollapseStart
            doctarget.Endnotes.Add Rnge, , docsource.Paragraphs(j).Range.Text
            j = j + 1
        End If
    Next i
End With
With ActiveDocument
    For i = .Fields.Count To 1 Step -1
        If .Fields(i).Type = wdFieldHyperlink And InStr(.Fields(i).Code, "ftn") > 0 Then
            .Fields(i).Delete
        End If
    Next i
End With
