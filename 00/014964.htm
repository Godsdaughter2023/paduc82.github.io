a. user say:

For my own interest, I created a small Word addin that *should* let you show a dialog box that should let you select a Notebook section.

At the moment it is quite crude, but if you want to try it out, you can get it from here:

http://goo.gl/qjzwkP

It is currently called GotoNotebookTabv0_9.dotm
To try it out, you would need to do the following. I wouldn't spend more than 5 or 10 minutes on it, especially if things do not work as I suggest.

  1. Start Word, and have a look at Word->Preferences...->File Locations and note where the Startup folder is (it may be blank)
  2. Quit Word
  3. Download the .dotm it to your Mac and virus-check it
  4. If you can, copy the .dotm into that Startup folder and start Word

If you cannot copy to the Startup folder,

  • start Word and make sure no documents are open
  • click the Tools->Templates and Addins... menu option
  • click Add..., then locate the .dotm, select it and click Open.
  • Word should add the template to the "Checked items are currently loaded" list and check the box next to it. If the item is listed but the box is not checked, check it.
  • Dismiss the Templates and Addins... menu.

Now open a Notebook layout document.

Press ctrl-shift-cmd-G. A dialog box with a list of sections should appear. (Unfortunately, at the moment it will probably cover up some of your Word window and you may need to move it)>

The list is currently in section sequence . The entry corresponding to the current tab should be selected. Word only actually stores tab names when you modify them - otherwise, in the English language version, it calles them "Section 1",  "Section 2" and so on. At the moment the dialog will show "Section n" regardless of your User Interface language.

Locate the tab name that you want, and either double-click the name, or select it and click the Goto button. Word should select the tab that you selected. But at this point you can choose again. 

When you have selected the correct tab, click Quit.

To remove the template, delete it from your Startup folder if you put it in there. Otherwise, go back into the Templates and Addins... dialog, select the template in the list and click Remove. (As a last resort, move the template to your trash).

If anyone finds this at all useful, I would be willing to put a bit more effort into it. However, I suspect the leeway for improvement is quite limited. In particular, on Mac I can't make the form modeless, so that it just sits there (if I could, there would be no need for the buttons on it as long as I could get the form to respond to addition/removal/renaming of tabs).

ken sheridan say: You don't need to look it up in the table at all.  Simply use:

Private Sub txtUserName_Click()

    Me.User = Forms![frmLogin]![txtLogin]

End Sub

You only need to do this if the User control is a bound control.  Normally you would use an unbound control with a ControlSource property of:

=Forms![frmLogin]![txtLogin]

ken sheridan say:
The expression for a text box's ControlSource property would be:

=DLookUp("UserName","Users","UserLogin = """ & [Forms]![frmLogin]![txtLogin] & """")

The login form would need to remain open and hidden of course.
