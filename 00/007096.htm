ken sheridan say: As you have discovered, copying a linked tabledef object creates another linked tabledef object.  To copy the table per se you would need to copy the table from the back end, not the current database.  Alternatively you could create a link to the table in the target file and execute a simple INSERT INTO statement in the current database.  To create a link you would call the TransferDatabase method.  The following is an example from my bill of materials demo.  The strTempDb variable is the path to the external
DoCmd.TransferDatabase acLink, "Microsoft Access", strTempDb, acTable, "BoM_Temp", "BoM_Temp"

Alternatively you can do as you are attempting and execute an INSERT INTO statement in which the path to the target database is specified.  You cannot use a variable directly in the SQL statement, however.  You will need to concatenate the value of the variable into a string expression, e.g.

Dim strTargetFile As String
Dim strSQL As String

strTargetFile = <get full path from somewhere>

strSQL = "INSERT INTO tblPersonnel " & _
    "IN """ & strTargetFile & """ " & _
    "SELECT * FROM tblPersonnel"

CurrentDb.Execute strSQL, dbFailOnError

Whether you create a link to the target table, or execute an INSERT INTO statement which includes the path, the target table will first have to be created of course.  If this has not already been done manually, then you can do so in code by executing a CREATE TABLE statement.  The following is an example from the same bill of materials database which I mentioned earlier:

    ' create BoM table in temporary database if doesn't exist
    strSQL = "CREATE TABLE BoM (MajorPartNum LONG NOT NULL, " & _
            "MinorPartNum LONG NOT NULL, Quantity LONG NOT NULL, " & _
            "CONSTRAINT MajorMinor PRIMARY KEY (MajorPartNum, MinorPartNum) )"
    
    dbsTemp.Execute (strSQL)
    
In this the dbsTemp object variable returns a reference to the external database.  In my demo the reference is returned by calling the CreateDatabase method as the code creates a new temporary database, but in your case you'd call the OpenDatabase method to return a reference to the existing database.

If you have created a link to the target table, once you've inserted the rows into the target table via the link, you can delete the linked tabledef object by executing a DROP TABLE statement in code.
