Here's how to find the codes on the active sheet:

Sub FindCodes()
Dim myStr As String
Dim myC As Range
For Each myC In ActiveSheet.UsedRange.SpecialCells(xlCellTypeConstants)
If myC.Text Like "L##.##.####" Then
MsgBox myC.Address & " has " & Right(myC.Text, 10)
End If
Next myC

End Sub

To copy just the digits and periods, you would need to write out Right(myC.Text, 10) to another cell, or copy myC and then after pasting remove the leading L

Hiding/unhiding content is trivial. It's also a very poor method of trying to control what a user sees or prints when using a form. That because the display and printing of hidden content is controlled by how the user has their Word installation configured. So you can format as much as you like as hidden text on your PC, so that you don't see or print it only to have it fully visible and/or printed on another PC that's configured differently (yes, they might even end up with text they can't see being printed!).

• If goods do not conform to contract at the time of sale, purchasers can request their money back "within a reasonable time". (This is not defined and will depend on circumstances)

• For up to six years after purchase (five years from discovery in Scotland) purchasers can demand damages (which a court would equate to the cost of a repair or replacement).

• A purchaser who is a consumer, i.e. is not buying in the course of a business, can alternatively request a repair or replacement.

• If repair and replacement are not possible or too costly, then the consumer can seek a partial refund, if they have had some benefit from the good, or a full refund if the fault/s have meant they have enjoyed no benefit.

ronrosenfeld say:

Read through and try to understand the macro.

The algorithm is something like:

  • Read the source data (from wsSrc) into an array  vSrc
    • After this is done, there are no further references to the wsSrc worksheet, so, yes, you can overwrite it.
  • Cycle through each row in the array until getting to the first line that contains the string "Record #" in the first  column Do Until vSrc(I, 1) = sColHeader
  • Start collecting that and each subsequent row into a collection, omitting lines where the first column contains Record #,  is blank or contains something that looks like a date.
    • Excluding the blanks also excludes subsequent repeats of Report Title since that content is in column 2.
  • Write the collection into a "results array"  vRes
  • Write the array back to a worksheet  wsRes
So to change where you want the results to go, all you need to do is change the Set statement that defines wsRes.  As you can tell by the algorithm, there is no reason why it could not overwrite.
