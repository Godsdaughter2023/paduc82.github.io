#taher(duda) say:

query

This error is usually associated with versioning differences between two or more installed versions of Access. If you have two versions of Access installed on your computer, this error can occur if:
The database was converted from an earlier version to a newer version. Attempts to open the converted database by using the earlier version of Access will cause an error.
An advertised or Install on first use install state was selected for the most current version of Access on the computer. If a database is double-clicked in Microsoft Windows Explorer, an attempt is made to open the database using the HKLM_CLASSES_ROOT registry settings for the last version of Access installed successfully. Since an advertised instance of Access is not installed yet (all the registry entries and executable files are not yet present) users may attempt to open a newer (converted) version of an Access database with an earlier version of Access. The user should run Access once by double-clicking the advertised icon to install the advertised version fully before attempting to open a converted database.
The version which was installed last starts and attempts to open any database file or database object double-clicked in Windows Explorer. The same is true for Office applications listing a shortcut or database object embedded within a document or worksheet.

The issue has been reported to MS since 2013 in multiple Community questions, and never got an answer as far as I know, let alone a "solution".

I would go with option A. This is purely because the warning you get if option 2 is chosen which reads something among theses lines 'Are you sure, this will discard all your changes'.

My understanding is that if you edit a document saved in OneDrive in offline mode (i.e. when you are not connected to internet), upon reconnecting to the internet the OneDrive server will detect the differences between two documents. It will not save the document and will give you options to 'Keep your version' (one that was edited offline) or the 'keep the server version' (one that did not register changes). Choosing 'Keep my version' means that it will be uploaded and will replace the older version.

I must say this is pretty bizarre system and Dropbox offers much better systems of 'revisions'.
