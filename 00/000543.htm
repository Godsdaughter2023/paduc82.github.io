No, file versions are not saved in the actual file.

jay freedman say:

Aaron,

It isn't clear whether you want the outside recipient to see the edits or not.

Stefan is correct that any history maintained by OneDrive will not go with the document file. However, any Tracked Changes that haven't been accepted or rejected, or comments that have not been deleted, will be in the file and will be visible.

If you don't want the outside recipient to see any edits, comments, or personally identifiable information, use the Document Inspector (File > Info > Check for Issues > Document Inspector) to remove some or all of that from a separate copy. Then send only that copy.

steve rindsberg say:

IIRC, relative links don't work in Mac PowerPoint, and your explanation suggests that perhaps it's that relative links either a) Don't work in Acrobat or b) don't work on the Mac, period.

As a test, start with your existing PDF and subfolders full of linked files.  Open it in Acrobat and re-create one of the links that's not working.  Test to make sure the new link works, and have a look at the link's properties to see whether Acrobat has recorded a relative location or a full path.

Then close Acrobat, move the files to a different folder (move, not copy ... you want to make sure that the original linked file isn't in the original location any longer).  Then open the PDF (from its new location) and check to see whether the link now works).

If not, then the onus is on Adobe or MacOS.

john korchok say:

OS X is a different operating system and uses different characters for the path separator. In Windows, you use the backslash (\) while OS X uses the forward slash (/), or in older versions the colon (:). In addition, OS X disk drives use names (Macintosh HD) instead of letters (C:).

You could try putting all your referenced files on a web server, then the standard internet protocols will apply and your links will work on both platforms.

steve rindsberg say:

What you've described in your test results corresponds pretty well to my experience with links to files Windows PPT (and most of the same info seems true of Acrobat and PDFs):

- If you link to a file in the same folder as the PPT, the link points to just the file name, (ie, myfile.ppt) rather than to the full path (ie c:\somefolder\subfolder\whatever\myfile.ppt).

- If you produce this type of "pathless" link, then so long as the PPT and the linked files stay together in the same folder, no matter where, PPT will be able to find the linked files, unless you open the file in PPT via the Recently Used menu.  So don't do that, 'k?

- If you link to a file in some other folder than the one where the PPT lives, the link will be fully pathed, and will only work if the files are never moved or if the same folder structure is reproduced on any other computer where you want to use the presentation.

- It's possible to convert fully pathed links to pathless ones (or even relative links) and have it work under Windows.

- Macs ... true to the "Think Different" ads ... think different.  Including about links.  IIRC, relative links don't work, or at least don't always work.  But your testing suggests that pathless links to files in the same folder as the PPT do, which is a very good thing to know.  Thanks for reporting your results!

Given the above, you'll want to save yourself trouble by putting all the files you INTEND to link into the same folder as the PPT file first, THEN do the links.  And be sure to save the PPT to that folder once BEFORE starting to make the links.
