BUG - Wrong DST period for Iran (Tehran) time zone

Hi,

I'm surprised that there is a not-yet-solved-old bug in Microsoft Windows which causes the OS to think that the Daylight Saving Time over Iran (UTC+3:30 - Tehran) starts on 16th of March and ends on 16th of September, while that's a wrong belief!

Actually every year, the DST period on Iran starts on 22 March, 00:00 Local Standard Time, and ends on 22 September, 00:00 Local Daylight Time.

You can check sources :

http://en.wikipedia.org/wiki/Daylight_saving_time_in_Iran

http://www.timeanddate.com/worldclock/city.html?n=246

http://wwp.greenwichmeantime.com/time-zone/asia/iran/time-iran/

http://www.timeanddate.com/time/zone/iran/tehran

http://www.timetemperature.com/middleeast/iran_time_zone.shtml

http://24timezones.com/world_directory/current_tehran_time.php

Please fix this annoying bug, which is causing mass scale problems in Iranian Community and Local IT infrastructures who are using popular Microsoft Windows OS.
