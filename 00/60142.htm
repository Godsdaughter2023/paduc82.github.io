hairyfool say:

The restriction preventing the operating system being replaced is called Secure Boot, is only available on UEFI BIOS and is only active with Windows 8/10 machines. If the computer had Windows XP on it it would not have Secure Boot enabled even if available.

Windows XP/Vista/7 OEM systems just had the key integrated into the installation image which checked for an SLIC tag in the BIOS to verify that the license was valid for that particular manufacturer and did not activate online or by phone.

Disk based installations must activate online/by phone so do not verify the license using the BIOS and ignores the tag built in even if it exists.

If your license key is retail it should either activate automatically or revert to the Phone Activation however the "Product Key Not Accepted" is not normal in such circumstance. At what point does this message come up? If during Windows setup the computer is not on line and it is failing the internal checks of the Product ID encrypted in the key, it is not being rejected by the activation servers.
