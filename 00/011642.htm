herb tyson say:

If you know what color shading was used, you can use a macro to set the formatting color by including a variation of the following line in the macro:

Selection.Find.Font.Shading.BackgroundPatternColor = wdColorBlack

This one, for example, searches for text that is shaded black. Once the macro has been run, you'll see Pattern: Clear (Black) under the search text in the Find & Replace dialog.

Sub FindBlackShading()
    Selection.Find.ClearFormatting
    Selection.Find.Font.Shading.BackgroundPatternColor = wdColorBlack
    With Selection.Find
        .Text = ""
        .Replacement.Text = ""
        .Forward = True
        .Wrap = wdFindContinue
        .Format = True
        .MatchCase = False
        .MatchWholeWord = False
        .MatchWildcards = False
        .MatchSoundsLike = False
        .MatchAllWordForms = False
    End With
    Selection.Find.Execute
End Sub

So, after the macro has been run once, you can simply use RepeatFind to find the next occurrence. All this assumes that you know what color has been used.

There *might* be a way to search for any shading, but nothing I tried does it. There are others who are much more adept at VBA than I am, however, who might know a way.

cbarnhorst say: There is no "supervisor" handing out drive designations.  Drive designations are not handed out by UEFI/BIOS.  Drive designations are determined when Windows is installed each time.  Designations in partition A were determined when Windows was installed on partition A.  If you partition that volume for partition B a new drive designation is added by Windows A when the Windows A first detects the new partition.  When Windows is installed in partition B, Setup compiles its own hardware list and assigns drive designations for itself.  It does not read what Windows in partition A did.  It neither knows nor cares.

Base on your description, I would guess that your problem is you are using curly/fancy double quotes in your code.  You probably copied it from an email or web page where they had been converted from normal double quotes.  I would change them to regular double quotes since the VBA interpreter does not recognize these curly double quotes as double quotes. 

Sub NextInvoice()
    Range("E5").Value = Range("E5").Value + 1
    Range("A20:E39").ClearContents
End Sub
