This is an extract from my answer to your other thread.

It is difficult to provide a definitive answer to this question, without knowing the type of keyboard your computer THINKS you are using - or the keyboard you are actually using.  However, the following workaround should provide a temporary solution.

Take a sheet of paper and a pen - and draw a diagram of your keyboard.

Boot your computer into Safe Mode - Command Prompt.

Then, press each key in turn.  So if you are using a QWERTY keyboard, you would type QWERTYUIOP{ }.

On your diagram, write down the upper case character that each key produces.  Repeat the process for the lower case characters - and also for each row of keys.

This will probably take a while but sooner or later, you will have a record of which key you need to press for every character.

I don't think "removing the joystick" is the best possible solution. I think "removing the joystick and thus noting that indeed it was the joystick keeping the system awake and thus talking with my joystick manufacturer about why they need to keep sending stay-awake messages" is a reasonable path to solution.

Bear in mind that in many circumstances if you are using a joystick you really don't want your system to sleep.  I'm sure there's a fun argument to be had there as regards sleep/don't sleep, but -- that's really a discussion to be had with people designing joysticks that want to keep the system awake.  From a game programming perspective and from a joystick perspective, it may make sense to some limited extent but perhaps that should be bounded.  You really really really want to talk to the people making those joystick drivers.  If the joystick is telling the system to HEY STAY AWAKE the system is generally obligated to listen to it.  The system should not be in the business of ignoring stay awake requests. :\
