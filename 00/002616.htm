Is there a common keyboard shortcut available to close toolbox windows - Microsoft Community

bettertoday say:

Is there a common keyboard shortcut available to close toolbox windows in Word 2013?

I'd like to close toolbox windows in Word 2013 using a keyboard shortcut. Is there a common shortcut available in Office 2013 to close toolbox windows?

Your helpful response is appreciated.

jay freedman say:

I think "toolbox windows" might mean task panes, such as the Navigation, Styles, and Research panes. If that's correct, then there are no keyboard shortcuts to close them or to do anything else with them.

If you want to close all open task panes at once, you can install the following macro and assign a keyboard shortcut to run the macro.

Sub CloseTaskPanes()
    Dim tp As TaskPane
    For Each tp In Application.TaskPanes
        If tp.Visible Then tp.Visible = False
    Next
End Sub

It would be possible to make individual macros, one to close each possible pane (or at least the ones you use most often) and assign each one a different shortcut. For example, this one closes just the Navigation pane:

Sub CloseNavPane()
    Application.TaskPanes(wdTaskPaneNav).Visible = False
End Sub

Move an ms access back-end to sharepoint online - Microsoft Community

ionysiqueira say:

Move an ms access back-end to sharepoint online

Hello!. How can I move an ms access back-end to sharepoint online?

albert d. kallal say:

Under database tools you can click on “SharePoint”.

It quite much a wizard and step by step approach.

Keep in mind a few things:

Any related table must be an auto number primary key, and the child table thus must use a standard long number to relate back.

Make sure for any combo boxes etc. that you base on the “id” and display values is ALSO setup as a relationship. The reason for this is during up-loading the auto numbers WILL change and thus any table NOT setup as related will up-load, but the standard “long” numbers (the foreign keys) in the child tables will not be updated correctly.

So no compound primary keys are allowed.

All primary keys must be auto number.

You also want to ensure you built the relationships using the “wizard” and not say the relationships window. While it is possible to use the relationships windows – it VERY easy to get the relationship settings wrong.

In this video I explain how to correctly setup related tables BEFORE you upload to SharePoint. And the video shows how you can link a front end to those existing tables on SharePoint.

https://youtu.be/3wdjYIby_b0?list=PL27E956A1537FE1C5

Keep in mind that I also recommend that initial uploaded tables each have less than 5000 rows. Tables over time can grow to say 10,000 rows but beyond that the performance will suffer.

Keep in mind the above uses "web publish", but you use the SharePoint option in the ribbon to achieve the same results - the key here is the steps outlined before you attempt to send data to SharePoint on-line. Note that the low cost small business essentials office 365 plan  at $6 per month supports SharePoint tables - so it is rather low cost.
