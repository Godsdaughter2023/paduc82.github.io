jay freedman say:
You are not going to be able to do this with legacy form fields, no matter how you try it. Protecting a document for filling forms absolutely prohibits the cursor/insertion point/Selection object from being placed anywhere within a protected section except inside a form field. If the insertion point is in a non-form-field location while the section is not protected, and then form protection is applied, the insertion point will be forced to the next unprotected location (either a form field or an unprotected section).

If you can replace all of the legacy form fields with content controls, then you can use the "No changes (Read only)" protection type, and mark the content controls as editable by Everyone. That type of protection doesn't affect the ability to move the selection to any location, but does prevent edits except where they're allowed.



However, the content control solution won't work if the form must be usable in Word 2003 and earlier. Also, if you use check box content controls in the form, they won't be usable in Word 2007.

$SPsiteCollection = Get-SPSite "http://SharePointSite.com"

# Loop through all Webs in the Site Collection

foreach($SPweb in $SPsiteCollection.AllWebs)

{

#loop through all lists in web

foreach ($SPlist in $SPweb.Lists)

{

if($SPlist.EnableVersioning -eq $true)

{

write-host "Setting versioning Limit for :" $SPlist.title

$SPlist.MajorVersionLimit = 20;

# To Disable Versioning use: $SPlist.EnableVersioning=$false

# To Enable Minorversion: $SPlist.EnableMinorVersions = $true;

}

sreejith_n say:

Hello,

Greetigs of the day !!

Thank you for posting,

It seems your Tenant is associated with multiple Yammer networks - one with .com and one with .com.br.

You can browse to Yammer Admin center > Network Migration to see the list of Yammer networks associated with your office 365 tenant. 

Multiple Yammer networks may get created if the domains that are added in office 365 have a past history of Yammer either a free Yammer network was created using this domains or domains might be connected with other tenant which was using Yammer enterprise.

 

Once you verify this domains in Office 365 the Yammer network associated with that domains gets connected to your office 365 tenant and you have multiple Yammer network scenario.

 

I would like to inform that having multiple Yammer networks in one office 365 tenant is not recommended by Microsoft, since in Multiple Yammer network scenario, we might get issues like users from same tenant redirected to different Yammer networks and not able to see the data / groups on other  Yammer network, user profile, profile pictures not syncing from office 365 to Yammer, users of same tenant getting redirected to different Yammer networks with separate URL.. Also the new feature  Yammer Integration with Office 365 groups Now Rolling out is not available in multiple Yammer networks scenario.

 

In order to fix this issue, we will have to perform Network Migration and consolidate multiple Yammer networks into one so all the domains that are verified in your office 365 admin centre gets synchronised to a single Yammer network

 

 

It is recommended to initiate a Network Migration from Yammer network whose Data is important and need to be preserved.
