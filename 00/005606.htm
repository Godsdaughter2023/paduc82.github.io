andre da costa say:

Microsoft has noted, Windows 10 Technical currently does not support limited storage devices and those with compressed operating systems. They are currently working on a solution:

We’re working on bringing upgrade to low capacity devices

The reason Windows 8.1 devices using WIMBOOT are not yet able to upgrade to Windows 10 is because many of the WIMBOOT devices have very limited system storage. That presents a challenge when we need to have the Windows 8.1 OS, the downloaded install image, and the Windows 10 OS available during the upgrade process. We do this because we need to be able to restore the machine back to Windows 8.1 if anything unexpected happens during the upgrade, such as power loss. In sum, WIMBOOT devices present a capacity challenge to the upgrade process and we are evaluating a couple of options for a safe and reliable upgrade path for those devices.

Source http://blogs.windows.com/bloggingwindows/2015/03/16/how-windows-10-achieves-its-compact-footprint/

So, you should skip installing Windows 10 on your device for now.

Orgbrat,
It is unfortunate that malware writers expend so much effort to get their wares past security software, but that's their jobs. The antimalware writers are fighting a never ending battle agains this junk.

To answer your specific question, no other real time protection should be installed with MSE.
 

You should never have more than one security product installed on the PC providing active protection/scanning. This can cause performance issues, system instability, and can hinder the effectiveness of both products at providing protection.

don cuthbert say:

Many others with this error resolved the issue (in regards to Build 10130) by ensuring that NX is enabled in their BIOS settings.  This goes by several names and descriptions in BIOS (NX, XD , XN are 3 common ones).  It has to do with Hardware Data Execution Prevention (DEP) and protection from viruses and other malware.  If you can find such a setting on a screen in your BIOS and Enable it, the Build 10130 install of Windows 10 may then work for you.  Setting your BIOS to Factory Defaults may also work even if there is no visible option to enable DEP.

UEFI Secure Boot requires DEP, so if you see that option, enable it.  On one netbook at my home I had no way to enable Secure Boot except to choose Factory Defaults in the UEFI-mode BIOS settings.  Build 10130 installed OK even though there was no specific setting to enable or disable NX in the BIOS.

There was another suggestion that the EFI partition might be on the wrong partition.  If you are able to completely wipe your hard disk of all partitions and allow Windows to initialize the disk and setup the disk partitions according to its defaults, that may also help.
