donaldgilligan say:

Duplicate emails in Outlook?

I am receiving duplicate emails in Office for Mac 365. How do I stop them?
talking moose say:

This is an inherent problem with the POP email protocol.

If you’ve chosen to leave messages on the server, the server may occasionally reset the unique message IDs that help Outlook keep track of what it has already downloaded.

When those message IDs get reset, Outlook considers those to be new messages and downloads them.

When configuring your live.com account in Outlook, allow it to automatically use the IMAP protocol. This will synchronize all messages to your computer. (That’s the nature of POP.) However, you should then file your old messages to folders and move them out of your Inbox. Or delete those old messages.

Optionally, you can log in to your live.com account using a web browser and create folders there. Move your old messages out of the Inbox and into those folders. This will allow you to still use POP.

If you continue to use POP, though, you’ll more than likely see this issue happen again in the future.

Hope this helps!

rohn007 say:

You are running into a conflict in how various ways of creating documents.

Legal requirements include line numbers.  On a typewriter they would either use paper with pre-printed line numbers and they just had to learn to align with those numbers.  Or they could manually add the line numbers. A little more difficult. But the key point is, the document was created line by line.

Early word processors worked the same way, they created TXT (text) files, also line by line.  So they could add a feature that inserted line numbers at the start of each new line.

MS Word on the other hand creates documents a paragraph at a time. Text is continuously flowed from line to line to the end of the paragraph indicated by the paragraph mark inserted with the enter key.   It is considered "bad practice" in word processors, like MS Word, to end eacg line within a paragraph with an enter. But that is what you are going to have to do. 

In MS Word there is a "line numbering" feature. But it works outside of the actual body text. It inserts the line numbers in a separate "layer" independent of the body text.  MS has not made an automatic provision for exporting those line numbers into a TXT file

This is where a second issue crops up.  Old typewriters, and early text editors used "monospaced" fonts. Each character, be it "i" or "W" was given the same amount of space on the line. So a typical document line was around 70 characters long.  "Modern" Word Processors, like MS Word, typically use "proportional" fonts. The letter uses only as much space on the line as it "needs". It makes the text easier to read.

So, to convert your "word processor" file into a line numbered text file that your PTX program can use you have to do several things.

#1. Change font from proportional to fixed space.  Most common fixed "pitch" font is "Courier".

#2. Define a hyphen / end-of-line margin, max line length

#3. split paragraphs into roughly 70 character lines, each ending with a paragraph mark

#4. insert a sequential line number at the start of each line by selecting the whole document and applying autonumbering

#5  use the word function to convert autonumbers to text:

    https://wordribbon.tips.net/T010248_Converting_Automatic_Numbering_to_Manual_Numbering.html

#6. From Word, SaveAS the file in TXT format

If the document is small enough, you can do it manually. Breaking into individual lines is the longest part of the process, otherwise you need to create a macro to do all of this.  You could to the font change manually.

The macro will have to count characters looking for a word break near the 70 char count (this is the logic I haven't worked out myself)

loop back to add the line numbers at the start of each line.

Hopefully someone else will be able to run with this starting point to create a macro.
