Work-around to resolve this issue (found and provided by MS) is as follows

Copy everything below between the lines (but exluding those lines) to a text file with a .reg extension:

-----------------------------------------------------------------------------

Windows Registry Editor Version 5.00

[HKEY_CLASSES_ROOT\htmlfile\shell\opennew\command]
@="\"C:\\Program Files\\Internet Explorer\\IEXPLORE.EXE\" %1"

----------------------------------------------------------------------------

Once the file is saved - click on it - allow it to make the requisite change and problem will be solved (leastwise it was on the W10 systems I checked after the registry edit was applied)

teylyn say:

Hello,

Excel 2016 does not offer the old web queries by default. In 2016 the default for queries is Get & Transform, which can pull data from the web.

Depending on your license (Office 365 versus perpetual), you can set up Excel to show the legacy wizards for web query. Go to File > Options > Data.  Here you can select which legacy data wizard you want to use.

Then you can click Data ribbon > Get Data > Legacy wizards and you will see the option there

From here on, the wizard will work as in previous versions. 

If you don't see the options to turn on the legacy wizards, you will either need to upgrade to Office 365, where these features are alredy included, or wait for the next release of Excel as a perpetual license, which is probably a few more months away.

The idea here is that Office 365 customers get new features as they are released, but perpetual licenses will only get new features when a new "version" is made available.

Does that make sense?

lenny33 say:

Word 2010 Checkbox Content Control - Change defaults

I have a requirement to make several checkboxes on my form defaulted to "Checked" rather than "Null".  I can find no setting under properties to do this, unlike legacy vb checkbox fields.

 

Is there somewhere else I can look?

 

Regards - Lenny33

jay freedman say:
In the template or document that you're preparing as the base for the form, just click the check boxes that you want checked, then save. When you base a document on the template, or reopen or copy the document, those boxes will be checked.

If you want overkill, you could write a Document_New macro in the template that will check any check box content control whose Tag property has the value "defaultchecked":

Private Sub Document_New()
    Dim cccoll As ContentControls
    Dim cc As ContentControl
   
    Set cccoll = SelectContentControlsByTag("defaultchecked")
    For Each cc In cccoll
        cc.Checked = True
        Debug.Print cc.Title
    Next
End Sub

Of course, the template will have to be saved as a macro-enabled (.dotm) file.
