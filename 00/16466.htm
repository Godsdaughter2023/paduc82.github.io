Hi,

I had this problem, and it turned out that Windows was using a Microsoft Standard Controller driver for one of my SATA Controllers... the one the drive was hooked to.

This was a Asmedia Controller that was on the motherboard to add extra SATA Ports...

The Intel SATA Controller was working properly...

I obtained Asmedia Drivers for that Controller and Windows 10, and everything is now working...

One strange side effect though... The Asmedia SATA Controller is no longer listed as a IDE ATA /Atapi Controller... It has been reclassified as a "Storage Controller"
