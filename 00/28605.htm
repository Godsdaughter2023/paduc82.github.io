How to Transfer or Update data from Excel to SharePoint list using VBA - Microsoft Community

derekchaplin say:

How to Transfer or Update data from Excel to SharePoint list using VBA

For some reason this simple solution to this frequently asked question doesn't seem to be found in MS's help or forums. A collegue found this code on VBA A2Z (http://vbaa2z.blogspot.com/2016/10/e05-add-new-data-to-sharepoint-using_23.html)

The only requirment is that you have to add the Microsoft Active X Data Objects Library reference under the Tools, References menu in the VBA editor.

Option Explicit
Sub AddNew_SP()
Dim cnt As ADODB.Connection
Dim rst As ADODB.Recordset
Dim mySQL As String

Set cnt = New ADODB.Connection
Set rst = New ADODB.Recordset

mySQL = "SELECT * FROM [DatabaseX];"

With cnt
    .ConnectionString = _
    "Provider=Microsoft.ACE.OLEDB.12.0;WSS;IMEX=0;RetrieveIds=Yes;DATABASE=http://lp-pc;LIST={ENTER LIST GUID HERE};"
    .Open
End With

rst.Open mySQL, cnt, adOpenDynamic, adLockOptimistic

rst.AddNew
    rst.Fields("Division") = [D2]
    rst.Fields("Department") = [D3]
    rst.Fields("Category") = [D4]
    rst.Fields("Item") = [D5]
    rst.Fields("Month") = [D6]
    rst.Fields("Budget") = [D7]
    rst.Fields("Actual") = [D8]
rst.Update

If CBool(rst.State And adStateOpen) = True Then rst.Close
Set rst = Nothing
If CBool(cnt.State And adStateOpen) = True Then cnt.Close
Set cnt = Nothing

End Sub

The path you were given is for Windows Vista, or Windows 7 or 8. The path is different in Windows XP -- instead of "c:\users\<user>\appdata\roaming", it's "c:\documents and settings\<user>\application data". In both cases these are hidden folders that normally aren't visible or searchable. Fortunately, there's an easy way to get there.

Open Windows Explorer (the My Computer icon on the desktop). In the Address box near the top, paste this in and press Enter:

 %appdata%\microsoft

That will go to the right place in any version of Windows, no matter what your user name is, and no matter whether you've selected the option to show hidden folders.

From there, click and open the Document Building Blocks folder and the 1033 folder under that. Now you should see a file named Building Blocks.dotx. Move it to some other folder such as c:\temp (make a new folder if necessary). Now restart Word and insert a page number or any other building block.

If you go back to the 1033 folder now, you should see that Word has created a new one (actually it just copies the file from the original installation, which is in another hidden folder at C:\Program Files\Microsoft Office\Office12\Document Parts\1033).

sharief k say:

Hi Harry,

 

A proxy server is computer that functions as an intermediary between a web browser (such as Internet Explorer) and the Internet. Proxy servers help improve web performance by storing a copy of frequently used webpages. When a browser requests a webpage stored in the proxy server's collection (its cache), it is provided by the proxy server, which is faster than going to the web. Proxy servers also help improve security by filtering out some web content and malicious software.

 

Proxy servers are used mostly by networks in organizations and companies. Typically, people connecting to the Internet from home will not use a proxy server.

 

If you place a check mark, Internet Explorer uses the proxy server while accessing internet. For more information, refer to the link.

http://windows.microsoft.com/en-in/windows7/change-proxy-server-settings-in-internet-explorer
