scott mcdaniel say:

If the code works, it should be okay like that. I'm not sure why you're automatically adding a new record in the main form's open event, but that's your call.

You can combine them in the main form's event like this:

Private Sub Form_Open(Cancel As Integer)

Me.NameOfYourSubformControl.Form.OrderBy= "PrSNo"

Me.NameOfYourSubformControl.Form.OrderByOn = True

Note that NameOfYourSubformControl is the name of the Subform Control on the main form, and may or may not be the same name as the Form you're using as a subform. This can be tricky, so be sure that you've got the right value for this.

You can also set them directly in the Design view of the form, if you'd prefer. Since this is not something that will change, it would make sense to set them in Design view, and modify them when needed by code.

I don't know that you'd have any performance issues with this code. If you need to filter, then that's the way you filter.

hansv mvp say:

The problem is that the underlying table All Records does have blank values in the date fields in some records. Even though those records aren't included in the query, Access tries to evaluate the function for those records when you try to filter. You can get around this by changing the function:

 

Public Function DeltaDays(StartDate As Variant, EndDate As Variant) As Variant
    'Get the number of workdays between the given dates
    Dim dbs As DAO.Database
    Dim rstHolidays As DAO.Recordset

    Dim MyDate As Date
    Dim NumDays As Long
    Dim strCriteria As String

    If Not IsDate(StartDate) Or Not IsDate(EndDate) Then
        DeltaDays = Null
        Exit Function
    End If

    Set dbs = CurrentDb
    Set rstHolidays = dbs.OpenRecordset("Holidays", dbOpenDynaset)

     For MyDate = StartDate To EndDate
        Select Case Weekday(MyDate)
            Case vbSaturday, vbSunday
                'Do Nothing, it is NOT a Workday
            Case Else
                'Thanks to "RoyVidar" 2/18/04
                strCriteria = "[HoliDate] = #" & Format(MyDate, "mm\/dd\/yyyy") & "#"
                rstHolidays.FindFirst strCriteria
                If rstHolidays.NoMatch Then
                    NumDays = NumDays + 1
                End If
        End Select
    Next MyDate

    DeltaDays = NumDays
    rstHolidays.Close
    Set rst = Nothing
    Set dbs = Nothing
End Function

 

By the way, your database has probably been created in Access 2 or Access 95. The Visual Basic project contains a reference to the Microsoft DAO 2.5/3.51 Compatibility Library. This will cause problems if you ever want to open the database in Access 2007 or 2010.

It's best to clear this reference and to set a reference to the Microsoft DAO 3.6 Object Library instead, in Tools | References... in the Visual Basic Editor.

    I think the simplest way is to make use of a helper column with a formula, and then sort by that helper column.

    So if your data is in column A, starting at A1, then in B1 enter the formula:

    =IF(LEN(A1)=3,LEFT(A1,2)&"_"&RIGHT(A1,1),A1)
