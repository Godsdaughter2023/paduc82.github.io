Compositor scaling for legacy applications on high-DPI screens ("retina")

Hello everyone,

I've got an important feature request here that I'd like to see implemented.

The software market for Windows is a rich one, there are many applications available, written over the span of many years.  Some of these applications and tools have not been updated to support scaling of UI objects for high-DPI screens ("retina" as some call them).  These screens are becoming more popular these days and Microsoft's Surface Pro is a good example.

Windows certainly offers scaling of text, but it often breaks the UI layout for the said legacy applications that were not written with the scaling support.  Instead, those can be rendered at their original DPI (96?), and scaled up using the compositor.  The compositor in Windows already supports scaling, like in window preview mode when you mouse over it in taskbar or flip through windows using the Windows key + Tab.

So since the compositor already has some scaling capability and the compositor is GPU accelerated it would not be too difficult to introduce this type of scaling for legacy applications.  I believe Apple's OS X already does this for non-"retina" applications, so this approach is certainly not foolish to ask for.

I understand that many users don't yet care about high-DPI monitors, but they will quickly discover the problem with legacy applications once they own one.  These displays will become more popular over the lifetime of Windows 10 so I would see it fitting that this relatively simple but effective solution be introduced and help convince people to buy new PCs (with high-DPI screens, like Surface Pro) with Windows 10.  Microsoft really wants us to use Windows 10, right?

What do you guys think?  Can this be pushed to Microsoft's priority feature list?

Excel - incrementing cell reference by 5 - Microsoft Community

harry stevens say:

Excel - incrementing cell reference by 5

Hi,

I have an issue regarding incrementing formulas, hope my explanation's clear.

I use two sheets - a main one for displaying data and another named "RAW" for the raw data that is constantly updated.

On the main sheet, I have a formula that calculates the average between two values. Simplifying it, it looks like this :

=SUM(RAW!K3+RAW!K6)/(RAW!C3+RAW!C6)

Now on the raw data sheet, I need to get the values from the formula above, incrementing it by 5 down.

So the next formula should contain the values from :

=SUM(RAW!K8+RAW!K11)/(!RAWC8+RAW!C11)

Simply dragging these two formulas doesn't work, it only increments the values by 2.

Of course, changing each value individually will work, but there are a LOT of raw entries that I need to use. I have been googling answers about this and found about OFFSET, but I'm struggling to figure out how it works.

Can someone explain to me how OFFSET works, as well as a proper formula on how to do what I want? That would be really helpful.

hansv mvp say:

OFFSET has several forms, but the simplest is OFFSET(mycell, number-of-rows, number-of-columns). This refers to the cell number-of-rows down and number-of-columns to the right of mycell.

(Remark: if number-of-rows is -5, it refers to 5 rows up from mycell. If number-of-columns is -3, it refers to 3 columns to the left of mycell).

Let's say the first formula is in row 2. You can then use

=(OFFSET(RAW!$K$3, 5*(ROW()-2), 0)+OFFSET(RAW!$K$6, 5*(ROW()-2), 0)) / (OFFSET(RAW!$C$3, 5*(ROW()-2), 0)+OFFSET(RAW!$C$6, 5*(ROW()2), 0))

ROW()-2 is 0 in row 2, 1 in row 3, 2 in row 4 etc.

5*(ROW()-2) is 0 in row 2, 5 in row 3, 10 in row 4 etc.

bernard liengme say:

My approach is similar to Hans

=(OFFSET(RAW!K$3,5*(ROW(A1)-1),0)+OFFSET(RAW!K$6,5*(ROW(A1)-1),0))/(OFFSET(RAW!C$3,5*(ROW(A1)-1),0)+OFFSET(RAW!C$6,5*(ROW(A1)-1),0))
