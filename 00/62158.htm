Next, the built-in Administrator account is automatically disabled by default during Out-of-Box-Experience (OOBE) setup because Microsoft has required that OEMs do this before delivering systems to their customers since Windows Vista.

Not sure what you mean by "Encrypted files loaded in with it", but since you mentioned a TPM  in your other thread, this would require the setup of various encryption facilities to allow for the security of the keys used to support and encrypt various other items which the TPM is responsible for.

The normal process of a Microsoft scripted installation sets up an X: virtual drive in RAM or from the mounted CD/DVD drive in order to allow that mounted device to be used as the installation source for operating system files and possibly other installation packages like drivers as well.  I'm not sure from your description where the folders dated 2001 might be located, but that's not really surprising if they're either on the virtual drive or possibly even the initial folders on the hard drive since these may default to some earlier date if the new operating system isn't fully running yet.

The only thing I hear that makes me concerned is the fact that you've tried changing physical networks and possibly other "tricks" to try and head off this elusive "malware" that you believe is infiltrating during this process.  Since many of the critical first steps performed automatically during this initial setup can cause damage to things like the TPM, BIOS firmware, disk partition information, or any number of other configuration or file installations on the disk drives, it's no surprise you've ended up with several damaged systems as a result.

When setting up a new PC, the best process is to always insert a physical Ethernet network cable and the power supply module before ever starting the laptop, then follow the rest of the installation process provided to the letter, with no other physical or system power changes unless explicitly told to do so.  Plugging the laptop into a UPS power protection unit is best if available, but insuring that power is at least stable with no likely interruption due to an impending storm or other issue is the minimum requirement.

Your fears are actually exactly the opposite of the truth here, with the false concern about highly technical issues you don't understand causing you to perform completely incorrect activities which are virtually guaranteed to destroy any new computer before it can successfully complete initial installation.
