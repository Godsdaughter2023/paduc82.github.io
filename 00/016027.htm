hansv mvp say:

Converting automatic numbering to fixed text is not difficult, but it is irreversible: there is no way to turn the fixed text back into automatic numbering.

If you'd like to try it:

  • Press Alt+F11 to activate the Visual Basic Editor.
  • Press Ctrl+G to activate the Immediate window.
  • Type or copy/paste the following line:

    ActiveDocument.ConvertNumbersToText
  • With the insertion point in the line of code, press Enter.
  • Switch back to Word.

    3. Password is your password
    4. Account name is youraddress without the @juno.com
    5. Send your messages with this name is your regular name
    6. Incoming email server is pop.juno.com
    7. Account type is POP3
    8. Outgoing (SMTP) email server is smtp.juno.com
    9. Keep all the checkmarks checked
    10. Tap "Sign in"

Buzzmag, the AppContainer network restrictions are sensitive to your network configuration. If your network as configured as a Private Network the Firewall considers your router not part of a public network and so it is unable to contact it and the IE under Metro doesn't works. To solve the issue you have to set manually the preferred and alternate DNS addresses (I used Google DNS addresses: preferred = 8.8.8.8, alternate 8.8.4.4).

Alternatively you can disable the network restriction by reconfiguring your network settings. First, use the sidebar's context menu to tell Windows to “forget” your network connection. Then, re-establish it as a “public” network. As a result the Firewall will consider your router part of a public network and thus AppContainers are freely able to contact your router address (192.168.1.1) and thus the privateNetworkClientServer capability is not required to contact local / non-routable addresses.

I hope that can help you to solve the issue.
