Hi,

 

I'm not sure what these restrictions you talk about mean in real terms but this is how I would do it.

 

Developer tab|Insert|Click the 'More Controls' button on the bottom right of ActiveX toolbox, select 'Calendar Control 12 from the dropdown. 

 

On the worksheet left click anywhere and hold down the left mouse button and drag to create a calendar the size you want.

 

Right click the worksheet tab and paste all the code below in on the right and then close VB editor.

 

Back on the worksheet ensure you 'Exit design mode'

 

Now if you click in F2:F100 you get a calendar popup. Click the calendar for a date in the active cell. Click outside F2:f100 and the calendar is hidden.

 

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    If Target.Cells.Count > 1 Then Exit Sub
    If Not Intersect(Target, Range("F2:F100")) Is Nothing Then
        Calendar1.Top = Target.Top + Target.Height
        Calendar1.Left = Target.Left + Target.Width / 2 - Calendar1.Width / 2
        Calendar1.Visible = True
        Calendar1.Value = Now
    ElseIf Calendar1.Visible Then Calendar1.Visible = False
    End If
End Sub
Private Sub Calendar1_Click()
    ActiveCell.Value = (Calendar1.Value)
    ActiveCell.NumberFormat = "dd mmm yy"
End Sub

julie fka profjulie say: Just my opinion, but I think you've got way too much action going on between devices and it probably confuses the syncing, resulting in duplicate records.  I think you need to decide which application is going to be your primary repository of information and set up the syncing relationships between the other devices to be subordinate to the primary device.  Let me explain how I sync between my winmo device, my computers (a laptop and tablet UMPC) and Google Calendar.

Outlook on my laptop computer is my primary repository of data.  I enter most all my appointments on my computer, but I also may enter appointments on my pocket pc.  I do not enter appointments directly into Google Calendar, but use it more as a way to view my calendar when I don't have my computer or winmo device readily available.  Google Calendar is also available to my husband to view so he knows what I'm doing. 

I use Google Calendar Sync to sync between Outlook and Google Calendar.  The sync is only one way, from my computer to Google Calendar. I do not key information directly into Google Calendar and so do not sync data from Google Calendar to my computer.

I use ActiveSync to sync between my winmo device and my computer.  The sync is two way, so changes or additions to either device is passed back and forth between the two devices.

I do not sync between my Winmo device and Google Calendar at all.

I sync between Google Calendar and my tablet UMPC (this is a second computer).  The sync is one way from Google Calendar to my tablet.  I don't enter new appointments or modify appointments on my tablet.

I've done the same thing with Tasks, only I use Toodledo for that.

This is how I have eliminated the problem of duplicate records when syncing to multiple devices.

To find the functions you can call using AppleScript, drop the PowerPoint icon from the Applications folder onto the AppleScript Editor icon in the Applications > Utilities folder. If you accidentally drop the whole application into the Utilities folder, please put PowerPoint back into the Applications folder so PowerPoint will run properly.

The Script Editor application will present a list of AppleScript commands available to you, and you will recognize many of them as VBA commands. I don't know what you're trying to do, so here are two samples and maybe they will assist.

In the AppleScript editor:

To open a file
ignoring application responses
    tell application "Microsoft PowerPoint"
        activate
        open alias "/Users/username/Desktop/Folder Name/sfilename.docx"
    end tell
end ignoring

To press the Return key
tell application "System Events"
    tell process "Microsoft PowerPoint"
        keystroke (return)
    end tell
end tell

The first example is more common. You can substitute commands of your choosing at the point where the example does Open Alias

If you come across a situation where the VBA command you want to use is not already in the AppleScript dictionary, you can use
do Visual Basic "type your VBA code here"

in the first example where the example has Open Alias.

marvin ma msft say:

Hi Jason,

To my knowledge, if the computer is in managed states, the programmatic access security settings is controlled by the admin. 

If it's managed by AD DS and the group policy has no effect, you can try to modify the registry to check if it helps. Before you change it, please backup the registry.

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Office\16.0\Outlook\Security

DWORD: ObjectModelGuard

Value: 2

You can also set the keys below. As always, if the keys don't exist in the registry, you'll need to create them.

HKEY_CURRENT_USER\Software\Policies\Microsoft\Office\16.0\outlook\security

DWORD: PromptOOMSend

Value: 2

DWORD: AdminSecurityMode

Value: 3

DWORD: promptoomaddressinformationaccess

Value: 2

DWORD: promptoomaddressbookaccess

Value: 2

And if the issue persists,

1. May I know your Office 365 environment, pure cloud, on-premises or hybrid?

2. Besides, does the issue only happen on the remote desktop server? 

3. What's the version of Outlook?

4. Did you install Office 365 in a Windows Server?

Regards,

Marvin
