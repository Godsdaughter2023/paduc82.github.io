If you actually copied OneNote folders (and the containing .ONE files) to your local sync folder, this is completely wrong and most likely the cause of your problems. OneNote is not using the OneDrive sync client to synchronize notebooks on a per file basis. Also placing ONE filed and notebook folders in your local OneDrive folder (or uploading them to OneDrive by yourself) won't help.
OneNote is using its very own sync mechanism to communicate with OneDrive. If you (what I assume) have local notebooks, i.e. in a subfolder of your document folder and want to place them on OneDrive instead, this is the (only) way to go:
First, move your complete notebook folders back from the OneDrive folder to their original place (usually they were iniside your documents folder unter "OneNote notebooks")
Open the local notebooks in OneNote 2016 and click on "File" menu. Then choose "Share" from the left menu and select your OneDrive account. If this is not in the list you would have add your account in OneNote first via "Add a place".
If done correctly, OneNote will report so and in your local OneDrive folder you would find a file with the name of your notebook, which is just a URL (link) file without any apparent content. This is ok and as it is supposed.

I just want to repeat: Never try to manually place OneNote files or folders into the local OneDrive folder or upload them to OneDrive other than from within OneNote and the Share menu.

It uses a macro to count words in citation fields. I was wondering if this or something similar applies to Word 2013 and if so exactly how one would go about using it.

suzanne s. barnhill say:

Here's a sort of kludgy workaround that will work:

  1. Create a character style that you can apply to inline citations. It should be based on Default Paragraph Font and have no other characteristics; that is, it will not apply any formatting and will be completely transparent. For illustration purposes, let's call the style Citation.
  2. When you want to get a word count of words in the document, redefine the Citation style to be Hidden. The word count in the status bar will omit the Hidden text.
  3. Once you've got the word count, Undo the redefinition of the Citation style. You can do this repeatedly if necessary.
  4. Note that you will not be able to use the NumWords field unless you insert it when editing is complete (and citations are hidden) and then unlink it with Ctrl+F11.

tina chen msft say:

Hi Brian,

We may encounter the error message "Read error on file "_catalogs/lt/.stp" if we are in the scenarios as the article mentioned.

This is the limitation in SharePoint online, and it is not feasible to get it change via Central Administration > Application Management >Manage Web Application > Your Web Application >General Settings > Resource Throttling in SharePoint online.

To work around this issue, save the template without selecting the option to Include Content. Or limit the number of Managed Metadata columns in a list or library that you want to create a template from to fewer than 10.
