mrhayman12 say:

Trouble booting? Check this out. [Preview Builds 16193+]

To the Moderators: I made this so people will not have the same problem finding help like I did. Please let me know if this post will cause issues and I will promptly take this down. Thanks!

Anyways, let's get to the point. You go to start up your PC (or laptop) to find it doesn't want to start. Safe Mode isn't working. You try Startup Repair. It thinks the system is fine. You try SFC, the system file checker, and Chkdsk, the system disk integrity checker. Both come out clean. You're probably thinking, "What the heck" and trying your best not to throw your computer out the window right now. Before you do that, I have one last thing you can try.

Try to bear with me, as I don't want to leave anything out. There's something called the Registry. It is the one thing that holds hundreds of "keys", which pretty much allow the system to "remember" what it needs to know in order to allow you, the user, to do things like sign in, open pictures and other files, and even just run your apps in general. If the registry is damaged, it can cause apps to stop working, cause users to be unable to sign in or even get to a cursor even. Now to the fix.

You'll need access to a second computer, or Windows 10 installation media. It does not have to be the Insider Preview version. There's a possibility your Admin account isn't working, so that's why we're working from installation media. So, have you got install media or a USB with at least 8GB? Let's start with if you don't have a disc but you do have a flash drive laying around.

Now, you're going to want to go to the insider.windows.com page to download an ISO of the latest Insider Preview. Make sure you know what Edition you're on. You will most likely be on either Pro or Home. After you have that downloaded, you're gonna want to download something called "Rufus". There's a portable version on the creator's homepage. We'll use this tool to create a bootable USB with the Windows installer on it. It's relatively straightforward, but be careful what drive you select. Anyways, now to the next step.

This is about to get a bit more technical than it was already.

Now you're going to want to go back to the affected computer and boot from the USB. If it boots, you're all set. If it doesn't, i'll make another post on this explaining what to do.

Here we go!

Choose your language, keyboard style, etc. and click Next. Now you're gonna want to click Repair My Computer. Click Troubleshoot, then Advanced Options. Click Command Prompt. If you have more than one installation of Windows, pick the copy of Windows 10. Otherwise, a command window will appear. Type the following and hit Enter:

notepad

Now press the Ctrl key and the letter O, and click My Computer, then your Windows drive. Hold shift and right-click. Click "Open Powershell window here" or "Open command prompt window here". DO NOT CLOSE ANY WINDOWS! Now you need to type the following, and hit enter after each line: 

cd ..

cd %SystemRoot%\System32\config\RegBack\

copy * %SystemRoot%\System32\config\

It should've completed with no errors. Go ahead and close all windows and hit Shut Down in the boot options menu that appears. You're almost done!

After you reboot and you get to your login screen, congrats! You're almost done. After you sign in, right-click on the Windows logo and click "Command Prompt (Admin)" or "Windows PowerShell (Admin)".

If you have  Command Prompt (Admin), click Yes on the UAC prompt and then type "powershell" into the box. If you have PowerShell open, prepare for the next step. Now, you need to type the following command. (You can copy/paste it. It might not work though.):

Get-AppXPackage | Foreach {Add-AppXPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

Now just hit enter and wait. A lot. After a bit, close the window and open the Store app. You should now have a now fully fixed Windows installation! :)

If J4 = "Sales Volume", then unhide rows 333:339, 342:345, 348

Private Sub Worksheet_Change(ByVal Target As Range)
   Const primeCellAddress = "$J$3"

rik_uk say:

With a little manipulation and by using the Range object instead of Rows, the code can be simplified... and answer your request. Using range you can enter multiple row references, but note that a single row in the range statement is represented by row:row still...

For Case statements it is possible to put them onto a single row using : to separate the statements (some people like, some don't). See if this does what you are expecting...

Private Sub Worksheet_Change(ByVal Target As Range)
Const altCellAddress1 = "$J$1"
Const altCellAddress2 = "$J$2"
Const primeCellAddress = "$J$3"
Const secondCellAddress = "$J$4"
On Error GoTo RecoverEventProcessing
    If Target.Address <> primeCellAddress Then Exit Sub
    Application.EnableEvents = False
    Application.ScreenUpdating = False
    ActiveSheet.Unprotect
    Rows("10:394").EntireRow.Hidden = True
    Select Case Target.Value & Range(secondCellAddress).Value
        Case "DepartmentSales Volume":  Range("10:52,355:361,364:364").EntireRow.Hidden = False
        Case "Department# Of Sales":    Range("10:52,355:359,362:364").EntireRow.Hidden = False
        Case "TypeSales Volume":        Range("55:60,365:371,374:374").EntireRow.Hidden = False
        Case "Type# Of Sales":          Range("55:60,365:369,372:374").EntireRow.Hidden = False
        Case "SalespersonSales Volume": Range("333:339,342:345,348:348").EntireRow.Hidden = False
        Case "Salesperson# Of Sales":   Range("333:337,340:343,346:348").EntireRow.Hidden = False
    End Select
    If Target.Value = "Salesperson" Then
        If Range(altCellAddress1) = "All Units" Then
            Range("152:255").EntireRow.Hidden = False
        ElseIf Range(altCellAddress2) = "All Segments" Then
            Range("152:153,258:309").EntireRow.Hidden = False
        Else
            Range("152:153,312:332").EntireRow.Hidden = False
        End If
    End If
RecoverEventProcessing:
    If Err <> 0 Then Err.Clear
    On Error GoTo 0
    ActiveSheet.Protect
    Application.EnableEvents = True
    Application.ScreenUpdating = True
End Sub
