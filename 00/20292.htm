john w. vinson say:

Be sure that there is no Format specified on the field (whether in the Table design, the query, or the textbox on the form); any such format will limit the field to 255 bytes. Similarly, make sure that you're not sorting by this field (in the table's order-by, in the query, or in the Form).

joehogan1 say:

Index - Match Function Across Multiple Worksheets

Hi all!

I feel like I'm very close with this, although perhaps should have left something so (potentially) complex until I had more familiarity with Index and Match functions.

So I have two sheets which both contain a lot of data about cars (Make/Model/Engine Size/Horse Power etc). One of the sheets contains a reference ID number per model of car, and the end goal is to get this reference number across to the other sheet if the data matches!

So what I'm currently working with is as follows...

The 'Data' Sheet - This is the sheet I want to bring the reference ID number in to

My attempted formula goes into A1 here to hopefully bring the RefID in from the second sheet...

The 'Cars' Sheet - A load of third party data but I need those reference ID's!

*Note that in both screenshots above I have hidden columns of data which have no bearing on the task*

Now in MOST cases i'm looking for an exact match in data, although there are certain range requirements to be met as well. These are:

  • Brand = Make
  • Model Name = Model
  • CC's = CC +/- 50
  • Fuel = Fuel
  • KW = KW
  • HP = BHP +/-1
  • Start Year >= RangeStartDate
  • End Year <= RangeEndDate

So the moment of truth... Where i've ended up is here:

={INDEX(Cars!A1:P87025,MATCH(1,(Cars!B:B=C2)*(Cars!C:C=D2)*((ABS(Cars!I:I-F2))<50)*(Cars!K:K=I2)*(Cars!F:F<=U2)*(Cars!G:G>=V2)*(Cars!N:N=H2)*(OR(Cars!J:J=J2,(ABS(Cars!J:J-K2)<=1))),0),1)}

Can someone point out where I'm going wrong here? I'm fairly confident that my matching criteria (in terms of the bullets above) are correct, but beyond that at a bit of a loss.

joehogan1 say:

To anyone who may come across this, I'VE FOUND THE SOLUTION!

Which is...

{=INDEX(Cars!$A$2:$A$87025,MATCH(1,(C2=Cars!$B$1:$B$87025)*(D2=Cars!$C$1:$C$87025)*((ABS(Cars!$I$1:$I$87025-F2))<50)*(I2=Cars!$K$1:$K$87025)*(Cars!$F$1:$F$87025<=U2)*(Cars!$G$1:$G$87025>=V2)*(H2=Cars!$N$1:$N$87025)*((J2=Cars!$J$1:$J$87025)+((ABS(Cars!$J$1:$J$87025-K2)<=1))),0))}

Compared to the previous formula I had in the original post there are a few changes:

  • In the logic used within the match function I have removed the OR in place of a Boolean equivalent (I don't THINK a match function can call the OR function)
  • Changed the sheet references to absolute values rather than just the whole column ($)
  • Changed the INDEX array reference to specifically the column holding the value I wanted to return rather than the whole sheet (this was causing a #REF! error)
  • Removed the optional column number for the INDEX function (the final 1 in the formula above) as this was simply in error

If anyone is looking for an explanation as to how the logic in this works, it's mainly down to the criteria within the MATCH function. A good explanation of this can be found here:

If you have an iOS, Android or Windows Mobile device, you can download the free apps for those devices and view PowerPoint presentations on them. Unless you are an Office 365 subscriber, you won't be able to edit or save them, but viewing works fine.
