Hi,


Which drive is it the utility continues to check?


If there has been an improper shutdown detected by Windows or a flash drive was removed improperly, Windows will scan the drive. 


You can press ESC which should stop it before it gets started.


To query the dirty bit, type the following in a command prompt window.  You'll receive a message Volume C:  is dirty or Volume C: is not dirty



Start > Type CMD

Type or copy/paste fsutil dirty query C:   (C being the drive it is checking.  If not C, type the correct Drive Letter.)

Press Enter


If the fsutil tool reports that the drive letter is not dirty, the disk checking process will not want to run for that drive


Next type CHKNTFS /X C

Press Enter

This tells your system not to check that drive when you reboot


If you've scheduled Chkdsk to run perform the following:


How to Cancel CHKDSK After It Has Been Scheduled

http://support.microsoft.com/kb/158675/en-us


a. user say:

If you never want XP to check the dirty bit on the volume again, the correct syntax is:

chkntfs  /x  c:

But, that will tell XP to never check the dirty bit on the volume again (until you undo it). 

To undo that one would enter this command:

chkntfs  /d

Since there might be a legitimate reason for XP to run chkdsk and a legitimate reason to run a chkdsk with error correction on a reboot, are you sure that is such a good idea to tell XP to never check it again?  What if it really does need to be checked?

This is not fixing the problem of the dirty bit being set and not being cleared.  Using chkntfs  /x would be fixing the symptom of the problem which is chkdsk running on every reboot and leave the original problem.  I would rather work on fixing the problem instead of the symptom, but that could just be me.

I would rather query the dirty bit and if the volume is dirty and if a chkdsk with error correction at startup will not clear it and the chkdsk log in the Event Viewer doesn't give you any clues, then run the chkdsk with error correction from the Recovery Console or from a Hiren's Boot CD so you can watch it and see what is going on.

It could be that the drive has a problem and the dirty bit will just never clear no matter how hard you try (I have one like that) so then one might consider telling XP to never check it, but I think it is going to fail eventually...  I would at least give the other ideas a shot to see if they will be able to clear it (if it is set).
