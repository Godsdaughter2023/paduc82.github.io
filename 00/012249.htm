@ Zeroom

The message implies that  Windows Explorer  (explorer.exe),  which runs your  Desktop and Taskbar  among other things, crashed.   There should be a clue about that crash in the Event Viewer's Windows Applications  log.

There is a standard recovery action for this event that you did not mention trying:  pressing Ctrl-Alt-Delete.   That would allow you to start Task Manager and using it start another instance of  explorer.exe  (although it may be of interest to know why it wasn't able to restart and recover itself.)   Then,  since it was probably just  explorer.exe  which was repeatedly crashing you could have used the same mechanism to start other tools to help you, without prematurely rebooting, etc.   Hence, the symptoms that you noted, being unable to click on anything on the screen, would be completely understandable.  They would be just artifacts then.   No program would still be running which was watching for events related to them, such as clicks on the Display area where the Start menu or Taskbar was drawn.

Since you note that this problem seems to be related to installing IE9 my guess would be that the symptom you are seeing is related to a deficiency in your video support.   IE9 tries to use your graphic card's  GPU  to offload much of its rendering overhead.  There is an option which can prevent this.  E.g. in Interent Options, Advanced tab, Accelerated graphics section (right at the top of the tab)   you could  check

Use software rendering instead of GPU rendering*

*Takes effects after you restart Internet Explorer

So, in fact, this could have been one of the actions that you could have taken via Task Manager, start the Internet Options program.   You might not have had access the Control Panel itself because of the problem with explorer.exe but the Internet Options dialog could have been opened at the tab in question by doing  File, New Task (Run)...  for

     control.exe  inetcpl.cpl,,6

kosh---01 say:

Hi john10e,

 

Thanks you for posting and welcome to the forum.

 

Windows Phone does not support playing music over the USB connection. You can use a Bluetooth connection, but the receiving device must support Bluetooth v2.1 with EDR (Enhanced Data Rate) and the A2DP (AdvancedAudioDistributionProfile) for stereo audio.

 

The USB connection is used for charging and for connection your Lumia to a PC or Mac to transfer media files and perform system updates using Zune (PC) or the Microsoft Windows Phone Connector (Mac)
