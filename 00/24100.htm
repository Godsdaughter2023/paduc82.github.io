lz. say:

Hi,

If you want only one extra column (Bonus) and the Grand Total per row/column to work in the Pivot enter the following array formula in C2 (calculates the Bonus only when the last occurrence of a name exists for a given date):


=IF( ROW()<>MAX( ($A$2:$A$36=$A2)*($B$2:$B$36=$B2)*ROW($A$2:$A$36) ), 0, CHOOSE( SUMPRODUCT( ($A$2:$A$36=$A2)*($B$2:$B$36=$B2) ), 0, 0, 50, 100, 100) )

and validate it with [Shift]+[Ctrl]+[Enter] then copy it down as necessary.

The above formula assumes Dates are in range A2:A36, Names in B2:B36 - adjust at your convenience.

a. user say:

I am indebted to the eventid.net website for help in understanding this issue.

The Event ID for this error (in Event Viewer) is 10005. The general format for this type of error is:

DCOM got error "<error description>" in order to run the server: {COM object GUID}

In my case, the full error description is: 

DCOM got error "1084" attempting to start the service tiledatamodelsvc with arguments "Unavailable" in order to run the server {4661626C-9F41-40A9-B3F5-5580E80CB347}.

To determine whether the error is caused by Windows or a third-party product I followed these steps:

1. Using Regedit navigate to the following registry value
HKLM\System\CurrentControlSet\Services\service_name where service_name is the name of the service displayed in the event description.
2. In the right pane double-click ImagePath.
3. The Edit String dialog box is displayed. Leave this dialog box open. If Svchost.exe is displayed in the Value data box then under the service name navigate to the Parameters folder and then double-click ServiceDll. 
4. Click Start > Computer.
5. Using the information displayed in the Value data box, navigate to the program.
6. Right-click the program name and then click Properties. The Properties dialog box for the program is displayed.
7. To determine who the vendor is for this service refer to the Version tab (you can also find the path to the executable file in this tool).

With these steps I was able to determine that the server in question is tileobjserver.dll which is described as the "Tile Data model server." The GUID for this server (4661626C etc.) is probably new to Windows 10, which is why I couldn't find any information about it with a web search.

I have only a superficial understanding of data models. But now I know that tiledatamodelsvc and tileobjserver.dll are components of Windows 10, and I have likely come across a bug that I will report.

nathan o. - microsoft support say:

Hi Billie,

 

It sounds like you are running into a bug within Access 2010. When you have an embedded macro on a form and use the option to “Convert Form’s Macros to Visual Basic”, it has been reported that Access 2010 will crash or you will see behavior similar to what you reported. If you click the button to convert the macro, Access 2010 will prompt to see if you want to add comments and error handling to the VBA. If you click OK on this, you will then receive the following message:

 

 

There was a problem opening the macro ‘[Embedded Macro]’. Do you want to continue? 

 

If you click Yes on this message, it will either once again crash or says “Conversion Finished!”, even though the macro is not converted. If you click No on this message, you receive an “Invalid procedure call or argument” message.

 

 

As mentioned above, this behavior has been filed as a bug and is being looked at by our development team. Unfortunately, I can’t guarantee if or when it will be fixed, but they are aware of the issue. At this time, the only workaround I have would be to create a normal macro, and then you should be able to open that macro up in design view and use the “Convert Macros to Visual Basic” option.

 

Hope this helps!
