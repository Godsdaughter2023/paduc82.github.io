bernie deitrick, excel mvp 2000-2010 say:

Use Conditional formatting for the cells, with the formul option, and a formula like

=WEEKDAY($B2,2)>5

Replace the B2 with the address of the cell on the active line with the date value entered.

Then set the formatting to gray background.

Try this...

Assume you have dates in the range A2:A32 of each sheet.

Select all the sheets that you want this to apply to.

not shaded. The formula will be relative to the active cell.

• Goto the menu Format>Conditional Formatting
• Select the Formula Is option

• =AND(A2<>"",WEEKDAY(A2,2)>5)
• Click the Format button
• Select the Patterns tab
• Select the desired shade of gray
• OK out

jay freedman say:

The triangles appear next to paragraphs that are assigned Heading styles, whenever you position the mouse pointer over one of those paragraphs. If you click the triangle, it collapses all the text that follows that heading, up to the next heading of the same level. If the heading is already collapsed, clicking it expands the text.

Technically, the collapse/expand feature applies to any paragraph that has been assigned an outline level (in the Paragraph dialog or in its style's definition) other than Body Text. If you don't mess around with unusual outline levels, though, only the Heading 1 through Heading 9 styles are collapsible.

jennsuz say:

Excel formula to multiply a percentage of a number, round answer, and subtract the answers to equal zero.

I'm looking for formula to multiply a percentage of a number, round the answers, and subtract the answers to equal zero.  See below.  Notice the sum is -0.01.   Any ideas? 

A1 is 169,172.24   Formulas used
A2 = -82,894.40            B2 is 49% Cell A2 formula is =ROUND(-$A$1*B2,2)
A3 = -60,902.01            B3 is 36% Cell A3 formula is =ROUND(-$A$1*B3,2)
A4 = -25,375.84            B4 is 15% Cell A4 formula is =ROUND(-$A$1*B4,2)
A5 = -0.01                     B5 = 100% Cell A5 formula is =SUM(A1:A4)  Cell B5 formula is =SUM(B2:B4)

joeu2004 say:

Whenever you round, there is likely to be "quantization" errors.  The simplest way to avoid that is to change the last formula.  In your example:

A2: =-ROUND($A$1*B2,2)

A3: =-ROUND($A$1*B3,2)

A4: =-ROUND(A1+SUM(A2:A3),2)

A5: =ROUND(SUM(A1:A4),2)

FYI, the use of ROUND in A4 and A5 is to avoid anomalies that arise with the binary arithmetic that Excel uses internally.

However, that puts all of the quantization error into the last formula (A4).  There are other methods that seek to "distribute" the quantization error more fairly.

jennsuz say: Thank you.  That is very helpful.  What other methods are there?  It would be nice if I could set up a standard form for distributions like this. 

joeu2004 say:

Some people suggest the use of the so-called "banker's rounding" method.  They're wrong, for so many reasons.  First, it makes a difference only when the rounded portion is exactly half; x.xx5, if you want to round to 2 decimal places.  Second, it helps only if you have a nice mix of "even" and "odd" numbers; that is, x.x05, x.x25, x.x45, x.x65, x.x85 v. x.x15, x.x35, x.x55, x.x75, x.x95.  "Banker's rounding" is easier to implement using VBA, since that is how the VBA Round function works, differently from the EXCEL ROUND function.  Anyway, let's put "banker's rounding" aside.  I don't recommend it.

Other than the simple method that I described already, I prefer a similar method that rounds the cumulative sum.  It works best if we have a large amount of data, say percentages that sum to 100% in B2:B100.  Then in your case:

A2: =-ROUND($A$1*B2,2)

A3: =-ROUND($A$1*SUM($B$2:B3)-SUM($A$2:A2),2)

Copy A3 into A4:A100

I "know" of three other algorithms, all of which claim to be more sound statistically.  But, sigh, I've forgotten their details, descriptions and pointers to explanations.

IMHO, it's not worth the trouble, usually.  99.99% of the time, I just use ROUND, and I add a footnote to the effect of:  "the sum of the numbers are a little off due to rounding".  Such a footnote is generally accepted for financial reports.

The only time that I make an effort to sum to the whole is when I'm creating examples.  And for that purpose, I usually fudge the data so that ROUND is sufficient.
