  • Disable (preferrably uninstall) your Antivirus utility before attempting the upgrade.
  • Disable General USB Devices (example - Smart Card Reader). 
  • Do a clean boot and try again.
  • If you are upgrading using the .ISO file, disconnect from the Internet during setup, if you are connected by LAN (Ethernet) or Wi-Fi, disable both and attempt setup again.

  • If you are connected to a domain, switch to a local account
  • If you have any external devices attached to the machine, disconnect them (example, USB keys, external hard disk, printers, non-essential devices).

Onenote export pdf, cut strokes at the end of pages - Microsoft Community

연상박 say:

Onenote export pdf, cut strokes at the end of pages

This is not a question but a suggestion.

You will probably most of the cases get cut texts at the end of pages when exporting as pdf.

That's because Onenote is based on single infinite scrolling page but pdf is split multiple pages.

Converting Onenote doc to pdf means it is inevitable to cut somewhere on the long long single page.

The first page is long enough, then Onenote cut no matter what text or stroke on there

and to deal with some issues, it duplicates the area between previous page and next page.

I tried to deal with it. The only solution is exact estimation of "where will be the end of the page?".

If you succeed to estimate end of the page, you can stop writing there and write again from the new estimated top of the next page.

I could get nice and no cut, no duplicated pages in this way.

But the problem is estimation. No one can estimate exact point of cut on long Onenote page.

So I suggest that Onenote should have a new page option that contains "page markers" 

which indicate page cut point when you export as pdf.

ken sheridan say: I can only assume this to be a student project or similar as it is so divorced from any real world workplace situation, viz no weekends, no holidays, no lunch breaks.

The solution to the problem as you state it is quite straightforward:

1.  Get the difference in days between the start and end dates.
2.  Add 1 to the value in days.
3.  Multiply that by the difference in minutes between the start and end times of the working day.
4.  Subtract the start time of the working day from the start time of the course on the first day in minutes and adjust the total value in minutes by that number.
5.  Subtract the end time of the course on the last day from the end time of the working day in minutes and adjust the total value in minutes by that number.
6.  Use integer division by 60 to get the hours and the Mod operator to get the remaining minutes to return a value as hours and minutes.

Wrap the above in a simple VBA function which accepts the start date/time and end/date time of the course as its arguments, and call it in a query.

However, if you want a real world solution which also takes account of the days of the week worked as standard, public and concessionary holidays and lunch breaks, I'd be happy to post the relevant functions.

ken sheridan say:

As far as I can see your query is only applicable to a one day course, Karl.  To cater for a course over one or more days something like this should work:

 

SELECT EmployeeID, CourseID,

DATEDIFF("d", [StartDateTime],[EndDateTime]+1) * 480

- DATEDIFF("n",#09:00:00#, TIMEVALUE([StartDateTime]))

- DATEDIFF("n", TIMEVALUE([StartDateTime],#17:00:00#)) AS TotalMinutes,

TotalMinutes\60 & ":" * FORMAT(TotalMinutes MOD 60,"00") As TotalHoursMinutes

FROM [EmployeeCourses];

 

The constants 480, #09:00:00# and #17:00:00# represent the number of minutes in an 8-hour day, and the start and end times of the standard day rsepectivbely.
