ken sheridan say: Is the database split into front and back ends?  If not then it should be; multiple users accessing a single file is an open invitation to corruption.  The built in database splitter wizard will do this for you, though it is a simple task to do so manually.  The back end should contain all of the tables, the front end should contain all other objects i.e. forms, reports queries, modules etc, along with links to the back end tables.

The back end should be installed in a shared location, to which each user has full permissions, on the system.  A separate copy of the front end should be installed on each user's local machine or in a 'personal' location on the system to which they only have access.  Ideally there should be a separate Access installation on each user's local machine, though a single Access installation on the system will work.

The front end can, if you wish, be distributed as a .accde file,  but if you do so be sure to retain a copy of the .accdb front end in a secure location for development purposes, and for creating a new .accde file if necessary in the event of upgrading to a later version of Access.
