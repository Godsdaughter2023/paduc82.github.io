greg carmack - mvp 2010-2020 say:

The EFI System partition on GPT format disk is typically 100mb just like System Reserved on an MBR drive.  An additional 450mb partition may be present which houses WinRE bootable repair tools that previously were embedded in C.  There is a known issue where certain backup programs write to the System partition to clog it preventing Upgrade or Imaging. 

An immediate solution is to borrow space from C or elsewhere to add to System partition.  This video demonstrates how space can be borrowed from any partition even if not adjacent:   blocks extending.  It is not needed.  Always back up fully before partition operations. 

If the problem System partition is 100mb then the issue is almost always file intrusion.  Assign a letter to browse into the partition to find and delete the intruder.  I'd also uninstall any backup program that might be doing this.   Enlarging the partition may fix the error but what's important too is to prevent further clogging of the partition.

Couldn't Update System Reserved Partition

You get this error if the System Reserved partition in Disk Management is too small. The system reserve partition needs to be at least 300 to 450 MBs.

Press Windows key + R

Type: diskmgmt.msc

Hit Enter

Click the system drive to view the size.
