john - project say:

JaneH,

Jan is right on the money if you use the Task Change Event. It basically will be "firing" with each and every change - not what you want. However there might be a less aggressive way. If you have a limited number for fields you wish to track for changes it is fairly easy to write some VBA code to date stamp those changes. To make it work, first capture the current data of desired fields in spare fields. For example if you want to know if Start and Finish fields change, capture the current dates in Start1 & Finish1. A similar approach would be used for other fields of interest. After all the editing is done, in a Close event macro compare the saved data with the current data and update the change date field (e.g. Date10) as desired.

If you really want to get fancy, the code could even use cell background or font color to highlight the changed fields so they would appear the next time the file is open.
