peter thornton say:

But isn't it unusual that color schemes for charts can replace color schemes in a worksheet?

Yes but there can be good reasons. The bottom two rows of the palette are typically termed the "chart colours" as these are the default colours used in chart Series. However any colour format in the workbook that can accept a palette colour can accept one of those colours just as readily as any other colour in the palette.

My guess, is that a previous well informed user has customized the bottom two rows of the palette while in 2007 to be same as the colour scheme of a 2007 workbook, or rather customized with same colours as used in the 2007 file. The objective then is that when the file is converted to xls and opened in say 2003, colour formats will 'map' to the closest matching colours in the 56 colour palette and hence look the same as they did in 2007.

If in the meantime someone applied an new colour scheme in 2007 would then need to recustomize the palette. But as I say that's only a guess.

However, nothing in all that should lead to what you have described, namely a customized palette 'infecting' other workbooks. It can't happen, unless manually or with code the palette is copied from one workbook to another.

With the information given I'm not sure where the changes are occurring.  I take it some of your users have Excel 2003 or older, or maybe something else like say OpenOffice?

If the workbook can be saved there's nothing you can do to prevent the old style 56 colour palette from being customized, which might be the reason for your chart colours changing. IOW workbook protection does not protect against the workbook's palette from being customized.

If that alone is the reason you can restore the original 56 colour palette in the workbook's save event (though it might surprise the user who's saving it). Post back if want to know how to do that.

Locking cells and protecting sheets does nothing to protect chart formatting (user changing the colours), unless the sheet is protected such that objects (in particular embedded charts) are protected. Simple way to tell is can you select the chart.

The formats of charts created in 2007 and "converted" to older xls format cannot be exactly recreated in 2003.  The thing to look for is

mySeries.Fill.Type
Normally this should return one of the msoFillType values, say 1 for solid. Normally none of these can be zero, except in a 2007 created chart later viewed in 2003.

Compare the palettes in original/changed charts

pal = activeworkbook.colors
for i = 1 to 56
activesheet.cells(i,1) = pal(i)
next

If it's not the palette will need to dig deeper, perhaps to prevent changes or restore them.
