omkara108 say:

Datediff - month - WithoutRound

Hello,

 

I want to calculate the nubber of month between two dates

 

so i use the datediff function -  datediff("m", 01/01/2005, 15/05/2005)

 

the problem is that the number of months that i get  is round without the digit after the decimal point.

 

what shuld i do in order to  get the number of month with digit after the decimal point.

 

thank you.

vanderghast say:

DateDiff does NOT ROUND, it computes the number of changes. It will return that there is ONE MONTH between the 31st of January and the 1st of February, but none between the 1st of January and the 31 of January. So the result has nothing to do with rounding.

 

Someone can use

 

DateDiff("m" , start, end ) + (Day(start) > Day(end))

 

to get an "age" in month (like we have an age in years, in a more common way). For an age in month(s) and day(s):

 

( DateDiff("m" , start, end ) + (Day(start) > Day(end)) )  & "  month(s) and "  & ( end - DateAdd( "m", DateDiff("m" , start, end ) + (Day(start) > Day(end)), start))  &  " days"

 

 

* someone can use >=  instead of >, if that is a better fit for the given case.

scottgem say:
Use "d" instead of "m" and divide by 30.

If you need a more precise calc, you would need to create a custom function that looked at the difference between the start date and end date and calculate the number of days for the remaining month. But that would be more complex code.
