    macropod say:

    If you're looking for code to automate the insertion and captioning of multiple pictures, see: http://www.msofficeforums.com/word-vba/16772-4-digital-images-1-page.html#post47919

    It's not clear from your post, though, what automation you're looking for when pictures are rotated. If you're wanting them to be re-scaled to fit the cells when rotated, you might employ a macro like:

    Sub RotatePic()
    Application.ScreenUpdating = False
    Dim r As Long, d As Long, sWdth As Single, sHght As Single, Shp As Shape, iShp As InlineShape
    With Selection
      If .Information(wdWithInTable) = True Then
        If .InlineShapes.Count = 1 Then
          With .InlineShapes(1)
            With .Range.Cells(1)

              If .HeightRule <> wdRowHeightExactly Then Exit Sub
              If .Column.PreferredWidthType <> wdPreferredWidthPoints Then Exit Sub

              .Range.Select
              sWdth = .Width
              sHght = .Height
            End With
            d = (CLng(InputBox("Rotate this pic 90º:" & vbCr & "1. Left, or" & vbCr & "2. Right")) - 1.5) * 180
            Set Shp = .ConvertToShape
          End With
          With Shp
            r = .Rotation Mod 180
            .Rotation = .Rotation + d
            Set iShp = .ConvertToInlineShape
          End With
          With iShp
            .LockAspectRatio = True
            If r = 0 Then
              .Height = sWdth
              If .Width > sHght Then .Width = sHght
            ElseIf r = 90 Then
              .Height = sHght
              If .Width > sWdth Then .Width = sWdth
            End If
            .Range.ParagraphFormat.Alignment = wdAlignParagraphCenter
          End With
        End If
      End If
    End With
    Application.ScreenUpdating = True
    End Sub

    tim liu msft say:

    Hi SirTyler1826,

     

    I’d like to explain that when you add custom domain in Office 365 portal, it will be synced to Azure AD. You may check in the DOMAINS section in your Azure portal. You don’t have to add it to Azure AD manually.

     

     

    If you change the default domain in Office 365, when you create a new user, it will use the default domain. This setting won’t influence the existing accounts. In Azure AD Domains section, you can also see the custom domain is now displayed as PRIMARY DOMAIN.

     

    The key point is adding alias with the new domain for users. In this step, if you don’t set the alias as primary, you just add another email address for that user. He still uses the former account to sign into Office portal and his Window 10 computer. If you want to let that user use the custom domain account to sign into Office portal and computer (joined to Azure AD), you need to set that alias as primary. After setting the alias as primary, user needs to use the new alias (custom domain account) for signing. You don’t have to configure other settings on his computer. The change for his account will also be synced to Azure AD (in the USERS section, you can see that user is displayed with the alias). User can directly use the new alias to sign in his Windows 10 computer.
