rob koch say:

You are fighting the wrong battle, which is why you'll never win.

Windows Defender since Windows 8 is included as part of the original operating system installation, so it's expected to be there by Windows Update and will be updated with other operating system components any time that might occur.

However, Defender is also specifically designed to be disabled by any properly designed and configured reputable 3rd-party security application via well documented parameters, remaining installed but silently waiting in the background on the system in case the primary security application should fail to operate for any reason.    At this point, if the primary security system fails to initialize, Windows Defender should be re-enabled as both a warning and a fallback to basic protection until the issues with the primary security application can be determined and resolved.

The problems most have with this situation is attempting to fight it, since that's exactly what malware might attempt and so destined to fail or at least cause flaky, inconsistent results that can't be guaranteed.

So the best course of action is to instead work with the Windows Defender protection as designed and after first insuring that it is properly installed and working, then install a Windows 8.1 certified security application that will thus also be designed to disable Defender properly.  If there are any problems with this occurring at that point, the issue can be researched and resolved as a supported configuration, which attempting to disable Windows Defender manually or especially trying to remove it completely has never been.

Again, you're fighting entirely the wrong battle.  Get a clue.
