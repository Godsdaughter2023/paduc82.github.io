Word for Mac does not properly handle keyboard layouts with different - Microsoft Community

daniel rosenberg say:

Word for Mac does not properly handle keyboard layouts with different maps for different modifiers

I am using a keyboard layout (Swedish Dvorak) which has different key maps depending on modifier. For unmodified, Shift and Option keys, the maps correspond to the Dvorak layout. For Cmd and Ctrl keys, the maps correspond to the QWERTY layout.

This is common with non-QWERTY layouts; it is a way to ensure that the user can still operate all the commonly used keyboard shortcuts one-handed, such as those for copy, cut, paste, undo, redo, bold, etc. It is also done because many non-QWERTY users still have QWERTY printed keys on their keyboards, and keyboard shortcuts often require visual orientation rather than pure tactile orientation.

Regardless of the reasons why this is done, it is something that is fully supported by the Mac OS X input model.

In all other Mac applications I have come across (except, coincidentally, for Xamarin and Visual Studio for Mac) this behaves as expected. E.g. in my case pressing the C key on a QWERTY-marked keyboard inputs the letter J as per the Dvorak layout, while pressing Cmd+C is still processed as Cmd+C and invokes the copy command.

However, in Word for Mac (and indeed also the other Office for Mac applications) this does not work as expected.

A few common keyboard shortcuts do work as expected, including clipboard operations and undo. Most others, like bold (Cmd+B) and italic (Cmd+I), do not work - for these, Word seems to completely disregard the modifier map and instead process shortcut key combinations against the letter each key happens to correspond to in the unmodified map.

This renders Word for Mac practically useless for me, and I imagine all others who happen to use this type of keyboard layout.

The expected behavior is for Word for Mac to always respect the modifier map of the current keyboard layout, and not hardcode key event handling against the QWERTY letter mapping.

ken sheridan say: I'd suggest that you add a DateTimeStamp column, of DateTime data type, to your table, with a DefaultValue property of Now().  You don't need to show this in the form, but if you base the form on a query which sorts the records in DateTimeStamp order you'd simply need to move to the last record in the form's recordset, which you can do in its Load event procedure.

If you wish to sort the form other than in DateTimeStamp order you'd need to use code to find the record with the MAX DateTimeStamp value in the form's RecordsetClone and navigate to it by synchronizing the form's Bookmark with that of the recordset.

ken sheridan say:

I am not sure in a macro but on the open event you could use

DoCmd.GoToRecord , , acLast

That would only work if the form's records are in order of date/time entered, which might not be the case.  The only really reliable way is to date/time stamp each row when it's inserted.  You can then navigate to the row with the latest date/time value.  It's easily done in the form's Load event procedure:

Private Sub Form_Load()

    Dim dtmLatestDateTime As Date
 
    dtmLatestDateTime = DMax("DateTimeStamp", "TableNameGoesHere"")
 
    With Me.RecordsetClone
        .FindFirst "DateTimeStamp = #" & _
            Format(dtmLatestDateTime, "yyyy-mm-dd hh:nn:ss") & "#"
        If Not .NoMatch Then
            Me.Bookmark = .Bookmark
        End If
    End With
 
End Sub

Office 2010 Starter re-install? - Microsoft Community

joy11xx say:

Office 2010 Starter re-install?

I had to do a re-set on my 4-year old Toshiba Satellite C670, 64-bit after "the black screen of death".   I am running Windows 10.  How do I get my Office 2010 Starter back?  I have all my files, but the system cannot read them, or they only open in "notepad" which is not sufficient.
dan at it associates say:

Your Tosh probably came with W7 and Office 2010 Starter Edition.  It was common for W7 computers to have a Trial of Office and an option to use Starter if you did not convert the Trial into fully functional Office.  When you upgraded to W10, it (Office 2010 Starter) came along for the ride.

When you re-set, you lost all installed programs, including Starter.

Unfortunately, there is no installer for Office 2010 Starter that will put Starter back onto a W10 installation.  The best you could do with Starter is to re-image your Tosh from the factory recovery discs, which will put the original W7 software load, which included Office 2010 Starter, back onto the computer.  Whereupon you would do another upgrade to W10, keeping Starter.

If that does not sound like a productive way to spend the better part of a week, then the alternative way to use a free edition of Office is to use the surprisingly competent online web apps, which you can access from www.office.com using your Microsoft Account.  Docs on your computer can be uploaded to OneDrive and then from there you can edit, print, and create new ones, with the option to save them to the computer or OneDrive.
