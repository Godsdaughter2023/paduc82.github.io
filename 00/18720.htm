1. Synchronization fails with message "There is a problem with the server. Please try again later.":

During setup for Windows Live for Windows Mobile or when synchronizing e-mail, synchronization may fail with the message "There is a problem with the server. Please try again later." If this issue persists, it is likely to be a known problem where at least one e-mail in the inbox prevents the completion of the synchronization. Microsoft has identified this issue and is currently working to correct this temporary problem.
Corrective action:
From a desktop computer using Windows Live/Hotmail, move e-mail from your inbox to another folder and then trying synchronizing e-mail on your device again. During setup for Windows Live for Windows Mobile or when you Switch Windows Live ID, syncing mail with a large inbox or more than 400 contacts with setup options enabled for "Store Windows Live contacts in your mobile phone's contact list" and "Merge duplicate contacts" can appear to fail with the message "Synchronization Fails". No contacts will be available in the contacts application, but the sync process is running in the background and will eventually complete. Until the sync process completes, the device may operate slowly and some functionality will appear to be disabled.
Corrective action:
Do not cancel setup and allow the process to complete. Once the process is complete, contacts will be available in the contacts application.

karl timmermans (outlook mvp 2012-2018) say:
>>
Can I link my Outlook on my PC with my Galaxy Note 9? I have Office 365 Home.
<<

Short answer: Yes BUT there are two ways to accomplish that

Background

Outlook desktop, by itself, does not directly connect/sync with any other device but what it does do is connect to an email account on the server. What information gets "sync'd" depends on how the email account is configured.

For POP3 - no information is "sync'd" - it is strictly a one-way, one-time download of new items in the <Inbox>.

For IMAP - email folders are sync'd with the server which means what is added/deleted/changed on the server via webmail gets sync'd with Outlook desktop (and vice versa).

For Outlook.com/Exchange accounts - all data (email, contacts, calendar etc) is sync'd bidirectionally.

So to answer your question to sync Outlook data with other devices with the above in mind............

Option #1

Configure the same email account on any device - it doesn't require the Outlook app which is one of many different <email clients> available

Option #2

To sync Outlook desktop data directly between the PC and another mobile (phone/tablet) device requires a 3rd party sync program.

The following article describes everything in more detail

Syncing Outlook with an Android smartphone
https://www.slipstick.com/outlook/sync-outlook/...

kevinkotowski say: I think I figured out my issue, and it might help others, too.

At about the same time I made this recent upgrade, I also was doing some maintenance on my inbox to delete something like 10k emails after archiving them to my local computer.

However, I forgot to permanently delete these emails from the "Deleted items" folder.

After I cleared out the "Deleted items" folder, it appears that the *runaway* CPU usage on Mac Outlook stopped. The CPU % on Outlook now is between 1-1.5% on my machine, which is much more reasonable!
