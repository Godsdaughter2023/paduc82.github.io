Yes, that is the way it is supposed to work.

What you want to do is "Import" the styles into the existing documents.  In theory you want to apply 'batch" updates to many documents. I haven't seen this specific change applied this way.

What you could do is create a macro and put it on the QAT, then you can teach your users to click on the button when they are using an existing document as a "template" (but it is not really one in the context of MS Word).

Here is an example of a quickie macro I recorded to copy 1 style from NORMAL.DOTM to the current document

Sub a_junk()
'
' a_junk Macro
'
'
    Application.OrganizerCopy Source:= _
        "F:\Public\Office\Word\Templates2010\Normal.dotm", Destination:= _
        "Document2", Name:="Comment Text", Object:=wdOrganizerObjectStyles
End Sub


You could have to customize it to handle the styles you want, and you would have to add code to find the current file name for use in the "Destination:=" parameter.

swischner say:
About #4: Correct, If you just copy the notebook files to OneDrive, OneNote won't even see them. While you still you could open them as local files (from your OneDrive sync folder), they would never sync.

About #2: If you could edit your notes before, they may already have been converted to the newer format. But "old format" is for sure not the problem here, because OneNote 2013 and 2016 can at least open older notebook files without the need to convert them first. See here: https://support.office.com/en-us/article/file-f...

This sounds to me more like a file corruption of some kind. The only attempt I can think of at the moment is: Separate a (or some) ONE file from its folder or remove the "open notebook.onetoc2 file". For if that one is invalid or corrupt, the notebook won't open. Every time you try to "start" a .ONE file using the Explorer, OneNote is looking for that .onetoc2 file that holds the notebook structure (e.g. all sections belonging to the notebook and their order). Unfortunately, OneNote files, although roughly based on XML, are in a proprietary binary format. There is no viewer, converter or other program that is able to open them.
