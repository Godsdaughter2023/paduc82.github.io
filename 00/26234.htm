Sub OnExitDeptList()
    With ActiveDocument

            Case "HR"

doug robbins - mvp office apps & services (word) say:

You could use a Combo Box Content Control, for which, if you assigned the Title "Dept" to it, and you put a { DOCVARIABLE Code } and a 

{ DOCVARIABLE Location } field in the document in place of your existing FormFields, you would use the following code in the This Document object in the Visual Basic Editor.

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
With ActiveDocument
    If ContentControl.Title = "Dept" Then
        Select Case ContentControl.Range
            Case "HR"
                .Variables("Code").Value = "001"
                .Variables("Location").Value = "Atlanta"
            Case "Finance"
                .Variables("Code").Value = "002"
                .Variables("Location").Value = "Chicago"
            Case Else
                .Variables("Code").Value = " "
                .Variables("Location").Value = " "
        End Select
    End If
    .Range.Fields.Update
End With
End Sub

However, for a document of the type that you mention, I would create a template in which you inserted a UserForm that contained a combobox that I would suggest you populate from an external source such as an Excel Workbook, which makes it easy to edit\amend the details associated with each entry.

For information on that, see the following pages of Greg Maxey's website:

http://gregmaxey.mvps.org/Populate_UserForm_ListBox.htm
