keratl say:

What Andreas is saying (if I may paraphrase) is that it is unusual behavior for Excel to extend the used range without any user behavior (based on your simplified use case of opening a new/blank file, saving it without doing anything else, and seeing the file grow).

Therefore, *something* is telling your copy of Excel that your files are using more rows than they actually are. You can absolutely use code to delete the extra rows, but until you identify the root cause, you will be doing that every time you save a file. It would be much more efficient to figure out why Excel is extending the "used range", and stop the problem at the source.

The most likely reason for this behavior is you have a poorly designed add-in installed, which is why Andeas suggested opening Excel in safe mode, and seeing if the problem still occurs when you create a new file from scratch (in safe mode), close it, re-open it, and close it again. If the file didn't grow, then Excel is not at fault, it is one of your add-ins.

Alternatively, you can deactivate all add-ins (both COM and Excel Add-ins) and maybe even any add-on packs. Then add them back in one at a time, testing between each one by creating a new file, saving it, closing, re-opening, re-saving. When you get to the point that the filesize starts growing again, you know which add-in to leave permanently disabled.
