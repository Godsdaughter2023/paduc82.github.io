bob jones aka: cybertaz say:
There is no need to "Import" anything -- Excel files are Excel files. Simply open them on your Mac in the same way as you would open them on a PC by double-clicking the file icon in a Finder window or using File> Open from within Excel.

You should find that there is no problem re Dates. Previously, Mac Excel used a Date System different from what was used by the Windows version, but Excel 2011 was designed to use the same one. If you Open the files as described above perhaps what you mentioned will be resolved. If not, please give specific details re how they differ... examples of what they should be & what they actually are.

In general, compatibility should not be an issue. However, Excel 2007 & 2011 do not share the exact same feature set. It depends on how the files were constructed & what features were used to create them, but those discrepancies should be minimal.

File management in OS X is not really much different than in Windows. Both operating systems use a hierarchical filing system represented by nested folders. A user is free to create their own branching structures to organize their files to suit themselves. The main distinction on a Mac is that files cannot be moved, copied, deleted or renamed from within the Open & Save As dialogs of your programs. The Windows OS permits doing so, but other than creating new folders 'on the fly' OS X demands that file management be done using Finder.
