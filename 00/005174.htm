john w. vinson say:

You have a classic Many to Many relationship here: each Leader can attend meny Meetings, each Meeting can be attended by many leaders. The proper way to handle this is with three tables: a table of Leaders (which it seems you already have); a table of Meetings, with the meeting date, purpose, location, any other information about the meeting itself as an event; and a table of Attendance, with the MeetingID (a link to the primary key of Meetings) and the leader's ID. You would get a tall-thin table with one record for each person at each meeting.

 

The report you describe can be generated from these tables using a Crosstab Query, with the meeting date as the column header and the troop leader's ID as the row header. Data entry can be done using a Form based on the meetings table (or the leader table, if you prefer) with a Subform based on the attendance table.

MATCH function bug? - Microsoft Community

yaguato say:

MATCH function bug?

Hi all;

I apologize if this has been discussed before, I couldn't find anything here in the quick search I did. 

I've been using successfully the combination of VLOOKUP + MATCH functions in the past without issues. However, today I encounter a problem I couldn't figure out why. After hours of trying, it turns out than the MATCH function does not work properly if you have a list of numbers spaced at any values below 0.25.

I would like people to replicate the error and tell me if the have the same issues or not. For example If create a column from A1 to A20

A1 = 1

A2 = A1 + 0.25

A3 = A2 + 0.25

...

A20 = A19 + 0.25

Then use the match formula as

D1 = 3

D2 = MATCH(D1,A1:A20,0) - this should give you the right answer, 9

Now repeat the formula in column A with 0.20, not 0.25. The value should be 11, but you'll get #N/A (see below).

I'm using Windows 10 Pro and Home and Office 365 and Office 2016 Pro (In case it matters). Both Office are x86


Can anyone confirm this?

Thanks in advance.

bernie deitrick say:

You are using the False (0) parameter in your MATCH function, and computers really suck at representing 0.1 multiples in binary and getting exact matches.  But that is what 0 wants MATCH to do.

So, you can use

=MATCH(E1,B:B,TRUE)

which will sorta work because your values are sorted (it may give 10 instead of 11),

or array-enter (enter using Ctrl-Shift-Enter)

=MATCH(E1,ROUND(B:B,2),FALSE)

But it would probably be better if you change the formula in B2 to

=ROUND(B1+0.2,1)  

or even

=ROUND(B1+0.2,15)  

and copy that down.

Also, if you put 1 in B1, and 1.2 in B2, then select both and drag the fill handle down, your MATCH will work.

You could also use the advanced option setting  "Set precision as displayed"  but that is generally a bad idea if you don't like intermediate rounding.
