Private Sub Worksheet_SelectionChange(ByVal Target As Range)
Dim LastRow As Long
Dim c As Range, MyRange As Range
Range("A:A").Interior.ColorIndex = xlNone
If Selection.Address = "$E:$E" Then
LastRow = Cells(Cells.Rows.Count, "E").End(xlUp).Row
Set MyRange = Range("E2:E" & LastRow)
For Each c In MyRange
    If c.Value <> "" Then
        c.Offset(, -4).Interior.ColorIndex = 6
    End If
Next
ElseIf Selection.Address = "$D:$D" Then
    LastRow = Cells(Cells.Rows.Count, "D").End(xlUp).Row
    Set MyRange = Range("D2:D" & LastRow)
    For Each c In MyRange
        If c.Value <> "" Then
        c.Offset(, -3).Interior.ColorIndex = 8
    End If
    Next
Else
Range("A:A").Interior.ColorIndex = xlNone
End If
End Sub

creating a label mail merge with variable quantities - Microsoft Community

perdeb say:

creating a label mail merge with variable quantities

i need a way to print labels and be able to varie the quantities for each label - i already know how to print labels and mail merge, thats not my issue, what i need is some way to print say 6 of one label then 4 of the next etc

i can create the database in Access or Excel

a. user say:

You can try this in Access (The principle is sound but I haven't re-checked this recently so in practice there may be problems).

You need a table called Multiplier. I suggest you

 a. keep the autonumbered ID field that Access creates

 b. make the second column a numeric field called multiplier

 c. suppose you can have up to 4 copies of each label. You then need records like this (Access will create the ID and it does not matter what it is):

ID multiplier

1 1

2 2

3 2

4 3

5 3

6 3

7 4

8 4

9 4

10 4

(In other words, if you want to allow up to 1000 labels, you'll end up with 1000 records where multiplier=1000, 999 records where it's 999, etc.)

Suppose your label data table is called addressdata. It should have a unique ID column (you can use Access's autonumber feature again), a column called quantity that specifies the label count, and whatever other data columns you need (not sure this will work if there are any memo fields, BTW)

Then create a query called multilables whose SQL is like this:

SELECT a.* FROM [addressdata] a

LEFT JOIN [multiplier] m

ON a.[quantity] = m.[multiplier] 

ORDER BY a.[ID]

If you execute this query, you the results should show the correct number of copies of each addressdata record. Use that query as the data source for your merge. 

Creating the multiplier table could obviously be a pain if the quantities can be large, but if you need help with that, post back and I'll try to create some suitable code to do it.

There may be a simpler way to do this in Access, but the above approach can also be used, with some modifications, in Excel. It's just simpler and probably more reliable to do it in Access.

Peter Jamieson
