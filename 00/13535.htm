marshall barton say:

You should have two work tables.  A utility table (named Numbers) with one field (named Num) and popupated with record values 1,2,,...,999.  This table can be useful in many situations and should probably be included id in every database.

The other table (named tblFreqs) is used for your frequency combo box and would have the unit of time, the number of units for the frequency and the number of records needed by the frequency.  I think it would look like:

FreqName Unit N  Dups

weekly        w   1   52
biweekly     w   2   26
monthly      m   1   12
quarterly     q    1    4
senuannual q   2    2
annual    yyyy   1    1

Then you can use an append query (named MakeDups) to add the records to the table:

INSERT INTO tblEvents (eventname, eventdate, eventfrequency)
SELECT tblEvemts.eventname,
               DateAdd(tblFreqs.Unit, Numbers.Num & tblFreqs.N, tblEvents.eventdate),
               eventfrequency
FROM Numbers, tblEvemts INNER JOIN tblFreqs
     ON tblEvemts.FreqName =  tblFreqs.FreqName
WHERE tblEvents.EventID = Forms!yourform.txtEventID
     And Numbers.Num < tblFreqs.Dups

Be sure to change the table and field names from my made up names to the names you actually are using.

With all that in place, you can then use a command button's Click event procedure (or some other action) to run a line of VBA code like:

CurrentDb.Execute "MakeDups", dbFailOnError

suzanne s. barnhill say:
If you have "Always create backup copy" checked at File | Options | Advanced: Save, then there will be a "Backup of Normal.wbk" file that is the previous version. With Word closed, you can delete or rename the current Normal.dotm and rename the backup to Normal.dotm. If you don't have that option enabled, this might be a good time to do so!

jay freedman say:

I'm not aware of any Microsoft update (so far) that changes or replaces Normal.dotm in any version. I'd suspect something else, such as an overzealous "cleaning" program.

In general, it's a good idea to bypass Word's attempt to save everything in Normal.dotm, precisely because it's prone to unwanted changes. Macros and keyboard shortcuts can be placed in global templates (those stored in %appdata%\Microsoft\Word\Startup). Styles should be stored in other templates that are specific to the types of documents you write. The New Document command that bases the document on Normal.dotm should be used only for print-and-throw-away documents.

In addition to relying on the "Always create backup copy" (which isn't really a backup...), you should be making regularly scheduled backups of the entire drive, or at least of your data files including the Templates folder. If you aren't doing so, now you know why it's important.

Your normal.dotm can become corrupted or replaced while you are using it.  For example, if you are using Microsoft Word with Endnotes and an email announcement states that a new version of Endnote is available.  You upgrade the Endnote and you get an error message during the installation saying that "Word is running".  So, you quit Word and continue installing Endnote, which installs a "Cite While You Write" add-in to Word.  The latest versions of that installer also replaces your normal.dotm with a new style called Endnote Bibliography.  

When you try to open up the document that you were working on when you upgraded Endnote, Word document no longer opens your Word files.  So, you quit Microsoft Word and the program prompts you to save your existing Normal.dotm template document.  If you say "No", Microsoft will not quit.  If you say "Yes", it may replace it with a blank Normal.dotm.  This just happened to me and resulted in a wasted 3 hours trying to figure out how to fix the problem.  I tried everything including copying and pasting on old version of Normal.dotm (from a backup disc) to the Library (located under "Users" >  [your username] > "Group Containers" > "UBF8T345G9.Office" > "User Content" > "Templates").  To find that folder, do a search for "Normal.dotm" under Finder while pressing the Option key.

The solution was simple but took while to discover.  If the above should happen to you and you install a new version of Endnote, just restart the computer before restarting Word.  The reason is because Endnote installed the CWYW add-on and the modified Normal.dotm file to your library files.  These library files are part of the system files that are loaded into RAM when you start up your computer.  You have to restart the computer in order for the altered library content to be loaded into your RAM.  Otherwise, your Microsoft Word does not have access to the changes of the library folder until the computer is restarted (rebooted).  I tried everything including reinstalling Microsoft Office and Endnote.  Nothing worked until I rebooted my Mac (a MacBook Pro late 2016) running MacOS Sierra version 10.12.5.
