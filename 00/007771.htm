john - project say:

Sally,

Is "advane!" the same as "advance"?

I'll give you a "yes" on both questions. The secret is to test for the shortest time difference first since formulas are evaluated left to right. If you test for the longest time difference first, all three differences will pass the test and you will end up with false results.

You don't specify whether the 30, 60 & 90 days are calendar days or working days but since they are values normally associated with standard calendar "accounting", I'll assume you mean calendar days. You also don't specify if the time is from the start of a task or the finish and whether the date is the scheduled date or the baseline date but I'll assume you intend the scheduled finish.

The following screen shot shows an example file with the associated custom field formula. You could include the graphical indicators as part of the Text1 customization. I've shown it as two separate fields (Text1 for the formula and Text2=Text1 with graphical indicators) so it is hopefully a little easier to understand and visualize.

If the formula is too hard to read in the screen shot, it is:

Text1=IIf(DateDiff("d",[Finish],[Project Finish])<=30,"30",IIf(datediff("d",[Finish],[Project Finish])<=60,"60",IIf(datediff("d",[Finish],[Project Finish])<=90,"90",0)))

If you want working days instead of calendar days, substitute ProjDateDiff([ date1, date2, calendar) where date1=[Finish] and date2={Project Finish}. The "Calendar" argument is only necessary if you need to use a task calendar that is different from the Project calendar.

Hope this helps.

John
