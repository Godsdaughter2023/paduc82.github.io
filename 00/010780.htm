mikek703 say:

It has been some years since I worked on a document with a table of contents but I do remember a few things that might lead you to the answer.

 

I think when you choose to run the TOC it will ask you to update page numbers if you have not made any changes to the text in the titles.  If you have made changes, if I remember correctly it will update the whole table.  That's probably when you lose the formatting.  The way to fix that is to make sure you update and save the styles that are used in each TOC level.  For example, if you have TOC 1 as bolded and italicized then you must update the TOC 1 style in order to keep the formatting as bolded and italics.  You can't just bold and italicize without updating and saving the styles (this goes for any styles) because when the TOC generates it will revert back to its original formatting.

Like I said, it's been a while, but I think this may help.

 

Mike

 

p.s. don't recall if "save" is an extra step; it probably it saves itself when you update it.

suzanne s. barnhill say:

All of the TOC styles, by default, are set to update automatically. In theory, all changes you make to any entry at a given level will be made to the style itself and propagated to all entries at that level. In practice, this applies primarily to changes in paragraph formatting; changes in font formatting usually require recourse to the Modify Style dialog (or "Update to Match Selection").

 

If "Automatically update" has been disabled for any of the TOC styles, then, yes, they'll have to be modified explicitly, but with the default settings, changes in paragraph formatting should be saved in the style, and updating the TOC should not remove indents. If any numbering has been applied to the styles, however, it is vital that the indents be set in the Define New Multilevel List dialog and not in the Paragraph dialog.

mikek703 say:

It's very difficult to explain something like this without seeing what your document.  But if you are applying the Heading 1, Heading 2, Heading 3, Heading 4 styles to the appropriate levels in the main body of text, then when you generate an automatic table of contents you will see something like this:

 

1.0  BLAH BLAH BLAH

   1.1  BLAH BLAH BLAH

   1.2 ETC ETC

        1.2.1 etc etc

 

EACH OF THE ABOVE LINES IS FORMATTED SOMEHOW. YOU CAN FORMAT EACH LINE THE WAY YOU WANT, FOR EXAMPLE:  1.0 AS BOLD, 1.1 AS BOLD ITALICS indented, 1.3 AS ITALICS indented.

 

Then you can Update each TOC style.  In Word 2010 you just "select" the TOC line and right click on it; then select "Styles," then "update TOC 1 to match selection."  Do that for each of your styles and your document's TOC will remain formatted this way each time you update it. 

 

Hope this makes some sense.

stefan blom say:

You can't apply styles to the table of contents field (if that is what you are trying to do). What you can do is customize the built-in TOC 1, TOC 2 etc. styles which Word applies to table of contents entries.

Note that you mustn't apply direct formatting to your headings, because such formatting will reflect in the TOC. Instead, customize the heading styles if you don't like the predefined formats.

Use the Modify Style dialog box when customizing a style. For example, press Ctrl+Shift+S to display the Apply Styles pane, type in the name of the style and then click Modify.
