Windows 10 - Day 10 impressions - Microsoft Community

jkotest say:

Windows 10 - Day 10 impressions

Here's my thoughts on the Preview so far.  I speak as someone who will be upgrading from Windows XP, having never extensively used Vista/7/8/8.1.

The big one, probably the showcase and at least the most important feature of this first build is the Start menu.  I like the layout - I like where my account name/options are, and the power button.  I like the section beneath it that I can pin any Metro app or executable to - my one complaint is that out of all the programs on my computer, for some reason the "Help and Support" app alone refuses to be pinned there; might want to figure that out.  The section below of recent programs seems fine.  I like that the All Apps section opens in a scrollable panel instead of splatting all over the desktop.  However, I would like to be able to create folders in the Start menu and sort programs into them the way I want - including Metro apps. 

The tile half of the Start menu seems fine; I like that people can remove all the tiles if they want.  I prefer to have some there.  I really appreciate that newly installed programs and apps aren't automatically pinned there by default, but only show up when I specifically chose to put them there.

I like the right-click context menu for the Start button; it's a nice way to quickly access some common administrative tools without having to clutter Start or the taskbar with them.

The Search bar in the Start menu and on the taskbar seems superfluous.  I suspect they were both added in order to see which one users tend to use most.  FWIW, I seem to prefer the one in the Start menu more.

As for the rest of the UI - I really happen to like the minimalist design.  A title bar and in some cases a footer bar have a function; side "bezels" are unnecessary screen clutter.  It is very clean, and feels very responsive without a bunch of distracting or gratuitous visual effects to take up time and resources.  But it needs to be made somewhat more consistent - for example, the This PC icon seems to reflect this minimalist design, while the Vista-era folders and Recycle Bin icon do not.  I do understand that people want the choice of retaining some of those older 3D-style design elements (they do look nice), so keeping them as an option wouldn't be a crime - but some new art consistent with the UI design might be nice too.  By release day, Windows 10 definitely needs to become visually its own OS.  At the same time, there's no reason to have to throw the baby out with the bathwater - the tile function is fine, and the fact that what is essentially an "alpha" build is so remarkably stable in most respects is a testament to the strength of the Windows 8 engine that it's being built off of.

I notice that whenever I choose one of the "high contrast" themes, I can change the colors of things like title bar text and inactive windows, but when using a default or downloaded theme I can only pick title bar color and nothing else.  That's problematic, especially when using very dark colors, as the black title bar text becomes impossible to read - bad news.  I don't see a good reason not to have the same UI coloring options that are available to the high contrast themes in all themes. 

To tell from the forums here, a lot of Windows 7 users really want that Aero design.  I guess the devs can add it if they want - I don't particularly care.  Another option would be to just allow the third-party mod community create an "Aero" add-on for those who want it, the way they seem to have with Windows 8.1.

That's all for now.  Can't wait for the first build update.

ken sheridan say: The following in the button's Click event procedure will toggle the form between filtered for the datanumber being 20 or less, and unfiltered to show all records:

If Me.FilterOn Then
    Me.FilterOn = False
Else
    Me.Filter = "datanumber  <=20"
    Me.FilterOn = True
End If

For a more flexible approach you could add two unbound text box's to the form, cboFrom and cboTo, each with a RowSource property of:

SELECT [datanumber] FROM [YourTableName] ORDER BY [datanumber];

The buttons code would then be:

If Me.FilterOn Then
    Me.FilterOn = False
Else
    Me.Filter = "datanumber  >= " & Me.cboFrom & _
        " And datanumber  <= " & Me.cboTo
    Me.FilterOn = True
End If

This would enable you to select the range from the two combo boxes and then click the button to filter the form.
Ken Sheridan, Stafford, England

david sanchez cuberos say: This problem is very easy to solve. You save the ActiveSync installer (setup.msi) to a location that will always be available to all users on the machine. I usually make a new folder in the root of the C drive, but I'm sure you could put it on the network. You then run the installer from there as an admin, and after that, ActiveSync works for all users (as long as the installer is accessible). Be warned though: if you install it directly from a CD, it doesn't work. You'd think that if you left the CD in the drive, it would work for other users, but it doesn't (don't ask me why), so always copy the msi to a local or network drive. And there's no going back if you've already installed it from a CD, or from Temporary Internet Files. I fiddled with the registry and all that, but the only way I found to fix it is to fully un-install it, and then re-install it from the correct location.

This is silly, because when you get ActiveSync on a CD with your PDA, if you follow the instructions, you install it directly from the CD. But because you need admin permissions to install it, and it only works for YOU if you install it from the CD, everyone would have to be an administrator and install it themselves for it to work. Go figure.
