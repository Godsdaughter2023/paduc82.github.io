dorky noob say:

Just a FYI

I also am an experienced tech and my best MS cert is an MCSE.

I work with many types of operating systems and have performed many upgrades over the years on many operating systems.

My SOP when upgrading and applies to all OS...

As part of the system prep, I uninstall all third party "enhancements," and I'd include your Stardock as a typical one I'd remove at least temporarily. If I want it later, I'd have the install files handy and re-install later.

I do use my judgement when considering what apps which make major changes to the OS to keep through the upgrade. An example of this would be Virtualbox or VMware which provide services and make kernel modifications. These are products with a high enough profile I assume they work with Microsoft to install and interface with Windows the best way possible to not only perform properly but also survive upgrades. Another app I commonly use is "MyDefrag" which although makes critical deep changes to how data is orgainized on disk, is done using the same disk API the standard Windows defrag utility uses.

In other words, I pick through the installed apps, and <know> how each is installed and what changes are made to the system including the degree and types of changes in the Registry (in general, not necessarily detailed). Although end users wouldn't be expected to study their machines to this degree, Sr SysAdmins have a duty to do this as part of their responsibilities.

The other thing I'd personally recommend (although haven't discussed with others) is to use a DVD to upgrade systems, don't rely on the online update. Besides being more bandwidth efficient, it standardizes how the upgrades are performed. And the DVD is supposed to support all scenarios, not custom tailored to a specific machine's configuration (which you hope is done correctly).

You may be doing some of these things already, these are principles that underlie whenever I support a number of machines....

standardize the process

minimize 3rd party influence

research what I don't know or suspect ahead of time

test on a sample before rolling out to all

HTH
