Store (Beta) App Update problem - try these workarounds - Microsoft Community

elise [msft] say:

Store (Beta) App Update problem - try these workarounds

Thank you again for your continued patience with the Store update issue. The information gathered so far has been tremendously helpful. As we’ve looked into the various reports and logs people have submitted, we’ve found that though symptoms are the same (or similar) there are a number of different underlying issues. We’ve resolved some of these issues, and are still working on others. 

So far, the causes of this issue fall into three categories :

  1. In many cases the required update will fail because the Store does not close itself during the update, and therefore the update fails due to the Store being in use. This is a bug, and we expect it to be fixed in the next OS flight. This appears to occur more frequently on machines with automatic updates turned off.
  2. Region/Language settings configured for languages not yet supported. Even if these settings are currently configured for a supported language (e.g. EN-US) in some circumstances it appears the system attempted to retrieve a new version of the Store in a language not supported yet, and never retried in a supported language.
  3. Things we don’t understand yet.

If you are currently seeing this issue, we ask that you try the following.  NOTE: Even if you’ve tried these before, please do so again in the following order, as some of the steps are incrementally different than earlier in the thread:

  • Turn automatic updates on. At least while you’re trying to fix the Store. Updates can be enabled by running the following from an elevated command prompt:
                reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsStore\WindowsUpdate" /v AutoDownload /t REG_DWORD /d 4 /f
  • Set your primary language and region to EN-US.
  • Clear existing Store checkpoints.
    • del %localappdata%\Microsoft\InstallAgent\Checkpoints\*.dat
    • Logoff/logon
  • Manually start the update task.
    IMPORTANT:  Make sure the Store app is closed while the task running.  Otherwise, you may run into issue #1 above.
    • Start | Schedule Tasks (to launch the Task Scheduler)
    • Browse to Task Scheduler Library | Microsoft | Windows | Windows Update 
    • Right click Automatic App Update
    • Choose Run
    • In the Scheduler, status will change from Ready to Running
    • Depending on how many updates are queued this can take from 5 – 30 minutes.

If the issue persists, please let us know, and we’ll likely reach out for more info. Thank you so much!
