the mazzter say:

I recommend dual booting as it allows you to easily switch back if you have problems or when the trial period expires.

I had no problems myself with dual booting (I used a VHD to store Windows 10) but here are some things to keep in mind.  More general notes with Windows dual booting than specific ones, unless noted:

- Recycle Bin formats may not be compatible between two different Windows versions; you may lose Recycle Bin contents when you switch.

- For the same reason, I recommend only turning on System Restore on for drives the other OS is not already configured to System Restore (usually for me the old OS does everything except the new OS's drive and the new OS handles its own drive).

- Feel free to clobber page files from one OS with the other's. They are only valid while a single OS is running anyway and they'll get recreated on boot if invalid. I guess. I never had a problem, at least.

- Drive letters should be switched to match both OSs as closely as possible, apart from the system C drive, to ensure maximum path portability. Unfortunately page files may prevent drive letter switching; you may have to turn off some page files and reboot before you can fix drive letters.

- When you select the time zone, the system clock will be assumed to already be on Redmond time (bleh) and adjusted. You will need to switch it back. Turn on Internet Time sync while you're at it.

- You can use NTFS junctions to map app data from one OS to the other so apps running in either will keep in sync automatically. So for example I will mount my Windows 7 drive as Y: and make a junction from C:\Users\Dan\AppData\Local\blah to Y:\Users\The MAZZTer\AppData\Local\blah. I try to avoid junctioning system folders but subfolders are fair game to me.  Some caveats follow:

- Google Chrome encrypts cookies, saved passwords, and installed extension lists using your user profile so if you switch OSs all of these disappear.  I recommend not linking a profile between two OSs and just letting Chrome Sync keep stuff in sync.

- Origin reacts very poorly to NTFS junctions of its app folder (many game installs will fail). I recommend copying the app data folder instead.

- Of course this won't help with some apps that store data in the registry, such as Google Play Music Manager.  You'll still need to set those up again.

- Steam will complain about its missing service. Go ahead and let it fix it.  It may take a VERY long time... it scans every file in every Steam game you have installed for some reason so if you have a 3TB drive filled it will take hours. Renaming the SteamApps folder before running will probably cut this time to nothing but I haven't tried it. You can probably also make the service by hand but I couldn't get that working.

- Windows 10 OneDrive will sync to your user profile by default, but you can change where it syncs to if you already had a folder set up under your old OS.

- Boot from VHD is only possible because Windows 10 preview is a preview of the Ultimate Edition or whatever. Make sure if you get a full version and intend to VHD boot that it will support it based on the Edition.

- Some minor stuff like performance score doesn't work when booting from VHD.
