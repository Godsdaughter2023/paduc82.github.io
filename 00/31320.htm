boyans say:

You cannot use EasyBCD to dual-boot Windows UEFI installations as EasyBCD cannot create UEFI/GPT loaders.

 

UEFI loader is file winload.efi. BCD entry must be set accordingly.

(winload.exe is loader for BIOS/MBR disk style)

 

Another problem is that only 64-bit versions of Windows 7/8 can be booted with UEFI.

 

32-bit versions can be booted only the "old" BIOS/MBR style way.

--------------------------------------------------------------------------------------------------------

 

I assume you have a GPT drive with Windows 8/64-bit installed and

a MBR drive with Windows 7/32-bit installed.

 

You can dual boot using the firmware device selection key during POST.

On my DELL computer F10 key allows selection of device to boot from.

 

See message on you computer during POST - there should be two keys, one for set-up and another

for device selection.

Your system is hosed. With the millions of updates and the service packs
(and the closure of the file version database
http://support.microsoft.com/dllhelp) it is not possible to know which
version of these files you need (or where to get them - your windows DVD
will most likely have very old versions).
 
Type the following line into a command prompt (it is one line). It checks if
the harddrive is predicting its own failure. It hopefully will return
False - the drive isn't failing.
 
wmic /namespace:\\root\wmi PATH MSStorageDriver_FailurePredictStatus get
predictfailure
 
I'd run a disk check in case disk corruption is the cause of the file
corruption.
 
Right click your drive in My Computer, choose Properties - Tools- and choose
Check Now. Tick both boxes and then Start. This will take hours. Or you can
type chkdsk c: /r in a command prompt.
 
While we are testing hardware I'd do a memory test.
Type memory in Start's search box and choose Memory Diagnostic. Or type
MdSched.exe in a command prompt. Run both Normal and Advanced.
 
Then I'd backup all files and do a factory reset. The mechanics differ as to
who made your machine. Read the manual or post back with your brand. A
factory reset sets the machine to how it was when you first got it. Any
changes (like installing software) will be lost. All your user files will be
lost, which is why you need to backup your files.

Hello Philip,

Thank you for posting your query in Microsoft Community.

The issue might due to the incorrect characters entered into the password field. This happens when the input keyboard language changes to some other. Let us troubleshoot the issue in the following way.

* Enter the password using either Normal Keyboard or using you cursor to type the password with On-Screen Keyboard.

* Before clicking the login, press the eye symbol (I.e. Reveal Password) option in the password field, to see the password is correct.

Let us know if problem persist, even you have entered a correct password.

* Press Windows key, type Settings into the search space and hit Enter key

* Click on Accounts and select Sign-in options

* Under PIN, click Add, Confirm with user profile password if it prompts, then configure the PIN.

Once done, sign-out from the User Profile and login using PIN option, to see if it works well.

Keep us posted how it went.
