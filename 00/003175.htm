gina whipp say:

Sounds like you are using Autonumber.  Autonumber is notorious for doing that and even, sometimes, assigning negative values.  The only thing that can be guaranteed with Autonumber is that it will be unique (and even that sometimes breaks if your database becomes corrupt).  That said, if you want sequential numbers then use...

DMax("YourIDField","YourTable") + 1

...in the Forms Before_Update event.  Gives you much more control over the numbers.  This does not mean you have to remove your Autonumber PK just add another field.

Side note, I have seen the numbers jump when there is a hiccup in the Network or someone tried to append some records but that is no guarantee that that is what happened.

ken sheridan say: If you don't need sequential numbers, but simply a distinct identifier per row, then the actual values are immaterial.  They should never be exposed to the user in the forms and reports with which the user interfaces with the data.

installation of the Explorer 11 fails with an error code 9C57

This is a sign that you have corruption in WinSxS.  Try using CheckSUR to see what it reports (and perhaps fixes).

Typically, it is usually also a sign that the OS is not up-to-date in terms of security patching.  In the case of its browser functionality you should have at least the level of modules implied by the January 2016 KB article (since that is when support for other versions of IE stopped).  If not, it would be a better idea to see about getting that fixed before worrying about changing the browser level.

A related tool that may yield some surprises or confirm discrepancies is  fsutil  hardlink list

OK - finally an answer for you - it's to do with blanks in the named range...

This is in the validation help file (about half way down page)

7. To specify how you want to handle blank (null) values, select or clear the Ignore blank check box.

  1.  Note   If your allowed values are based on a cell range that has a defined name and there is a blank cell anywhere in that range, selecting the Ignore blank check box allows any value to be entered in the validated cell. So if the range had been full of names this little quirk would not of come to light...
