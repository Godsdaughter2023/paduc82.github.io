pbezucha say:

A bubble chart might be what you want - it is a psuedo-scatter chart with a third dimension (the diameter of the bubble) telling you the magnitude of the value of a third parameter at each location.  Perhaps the bubble size could indicate the number of points at a location?  You could turn on data labels to show that value, probably.

 

If you set your data up something like this:

Data Pair Left Right Count
(6, 3) 6 3 1
(1, 3) 1 3 2
(1, 3) 1 3 2
(2, 5) 2 5 3
(2, 5) 2 5 3
(2, 5) 2 5 3

 

where the "Count" is a formula like:

=COUNTIF($A$2:$A$7,"("&B2&", "&C2&")")   <== adjust to suit your data.

 

You can then select columns B:D (Left, Right, Count) and create a bubble chart.  Turn on the data labels, and both the size of the buble and the label will tell you how many times that data pair showed up in your list.

 

 

HTH,

Eric

 

P.S. - select "Format Data Series", "Fill" and then "Vary colors by point" if you want each bubble to have a different color.


There is another method, in my opinion the most proper. Let you have your data treated exactly by Eric’s method, and positioned exactly in the same way, i.e. the counts of identical points in the column adjacent to y-values, and an xy-chart based on the (x,y) pairs. Then select the series on the chart and deploy the following macro:

 

Option Explicit

 

Sub MarkerSize()

'Macro sets the size of markers (in points) according to

'the numbers in the column adjacent to y-values column.

'The series must be preselected.

Dim SP As Points, I As Long, W As Range

'A linear relation is applied between numbers and point sizes,

'with constants:

Const MinSize As Long = 3, SizeStep As Long = 2

Set SP = Selection.Points

Set W = Range(Split(SP.Parent.Formula, ",")(2))

For I = 1 To SP.Count

  SP(I).MarkerSize = SizeStep * W.Cells(I).Offset(0, 1).Value _

    + MinSize

Next I

End Sub

 

You can readily see the arithmetic relation between the count of repeating points and the size of markers (in the case of circles let’s call them bubbles). The macro is not protected against any possible wrong calling.
