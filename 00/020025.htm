query: "qry BH Population - Dual Claims", which is a subset of "tbl STARS dataset" that includes only those records associated with a "dual" population (mental health and substance abuse)

 

"qry tbl STARS Dataset UPDATE BH Pop value DUAL", which is an update query -- based on "qry BH Population - Dual Claims" -- that updates the "BH Population" field in "tbl STARS dataset" with a value of "Dual".  Records in "tbl STARS dataset" are updated if they have the same ICN (claim control number) as found in "qry BH Population - Dual Claims".  SQL for this query follows:

 

UPDATE [qry BH Population - Dual Claims] INNER JOIN [tbl STARS Dataset] ON [qry BH Population - Dual Claims].ICN = [tbl STARS Dataset].ICN SET [tbl STARS Dataset].[BH Population] = "Dual"
WHERE ((([qry BH Population - Dual Claims].ICN) Is Not Null));

I am able to run "qry tbl STARS Dataset UPDATE BH Pop Value DUAL" in the Institutional Inpatient clients database without problems.  However, when I attempt to run the same query in the Institutional Outpatient clients database, the following error message appears: "Operation must use an updateable query". 

 

Any thoughts on why the error appears in one database, but not another?

 

Thanks,

 

Daniel Collison

 

 

 

 

 

john w. vinson say:

A query will be updateable only if it has a Join from a Primary Key (or a field which otherwise has a unique Index) to an indexed foreign key of the same datatype. If [qry BH Population - Dual Claims] is itself not updateable, then the UPDATE query using it won't be updateable either.

Check the joins and indexes in the Outpatient database!

 

I'd really strongly urge simplifying your object names, leaving out punctuation (- can be misinterpreted as an arithmatic operation) and blanks. If you have punctuation or blanks in the names, you must consistantly enclose them in [square brackets].

You can use the DATEDIFF function:

DateDiff Function

Returns a Variant (Long) specifying the number of time intervals between two specified dates.

Syntax

DateDiff(interval, date1, date2[, firstdayofweek[,firstweekofyear]])

The DateDiff function syntax has these named arguments:

Part Description
interval Required. String expression that is the interval of time you use to calculate the difference betweendate1 anddate2.
date1, date2 Required; Variant (Date). Two dates you want to use in the calculation.
firstdayofweek Optional. A constant that specifies the first day of the week. If not specified, Sunday is assumed.
firstweekofyear Optional. A constant that specifies the first week of the year. If not specified, the first week is assumed to be the week in which January 1 occurs.

Settings

The interval argument has these settings:

Setting Description
yyyy Year
q Quarter
m Month
y Day of year
d Day
w Weekday
ww Week
h Hour
n Minute
s Second

The firstdayofweek argument has these settings:

Constant Value Description
vbUseSystem 0 Use the NLS API setting.
vbSunday 1 Sunday (default)
vbMonday 2 Monday
vbTuesday 3 Tuesday
vbWednesday 4 Wednesday
vbThursday 5 Thursday
vbFriday 6 Friday
vbSaturday 7 Saturday
Constant Value Description
vbUseSystem 0 Use the NLS API setting.
vbFirstJan1 1 Start with week in which January 1 occurs (default).
vbFirstFourDays 2 Start with the first week that has at least four days in the new year.
vbFirstFullWeek 3 Start with first full week of the year.

Put the following in the BeforeUpdate event procedure of the control bound to the date of birth field in your data input form:

    Const MESSAGETEXT = "Employee's age is outside allowed age range 18 - 64."
    Dim ctrl As Control
    Dim varDoB As Variant
   
    Set ctrl = Me.ActiveControl
    varDoB = ctrl.Value
   
    If Not IsNull(varDoB) Then
        If DateDiff("yyyy", varDoB, Date) - IIf(Format(Date, "mmdd") < Format(varDoB, "mmdd"), 1, 0) < 18 Or _
            DateDiff("yyyy", varDoB, Date) - IIf(Format(Date, "mmdd") < Format(varDoB, "mmdd"), 1, 0) > 64 Then
           
            MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"
            Cancel = True
        End If
    End If
