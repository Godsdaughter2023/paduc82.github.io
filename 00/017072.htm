steve rindsberg say:

I do believe you've found a bug in PPT.  Thanks for describing it so well.

I have one workaround you can try. Let's use your second line as an example:

Select the "T" and press Ctrl+B to UNbold it (along with the number.)

Put the text cursor just to the left of the T then choose Insert | Symbols | Symbol from the ribbon.

In the Symbol dialog box, choose Font:  (normal text) 

Make sure that from: Unicode (hex) is selected

In the Character code: box, type:  034f

Verify that that selects the character named:  Combining Grapheme Joiner

Click Insert

You can now select just the T and bold it again.  The character you just inserted has no width, but fools PPT into thinking that there's an unbolded character before the T.

And yes.  It's ridiculous that this should be necessary!

it.neverends say:

Is it possible to copy the filtering criterias in excel 2010 (all checked and un-checked lines of the filter) to a new worksheet or perhaps view them in a more visual way?

ossiemac say:

OK. I guess I just did not imagine you would be working with that many criteria. I thought it would be more like a few for each filter column.

 

The line continuations is the space and underscore that forms a line break in an otherwise single line of code and VBA just cannot handle the number required.

 

Let’s go back to your original question of how to obtain a list of the criteria for each column so you can apply it to another workbook.

 

If you are only setting the filters by selecting multiple criteria at each drop down then you can use the following method to obtain a list of the criteria for each column. (Will not work with complex filters like greater than and less than; only with multiple selections from the dropdowns.)

 

Don’t know what your expertise is with Excel so will try to provide a full explanation just in case.

 

1. Need a blank worksheet. Insert one if you don’t have one.

With filters set:

2. Select the first filtered column of visible data from the column header to the bottom of the data; NOT to the bottom of the sheet.

3. Copy the data and Paste to Cell A1 in a blank worksheet. (Only the visible data copies and pastes.)

4. Select the pasted data from the column header to the bottom.

5. Select Advanced Filter. (Data ribbon in the Sort & Filter block)

6. Select copy to another location.

7. Click the button at the right end of the Copy to field then select cell B1 on the sheet to populate.

8. Click the button to right of displayed field to return to previous dialog box.

9. Check Unique records only.

10. Click OK.

11. Delete column A and you will be left with the unique list only.

 

You now have a list of the criteria that was set for the first filtered column. You can repeat the steps for the additional filtered columns and just place them in adjacent columns.

 

Copy the unique list to a blank sheet in your other workbook.

 

Now on the worksheet with the data, copy the column headers only and paste them immediately to the right of the existing column headers. You will have a duplicate set of column headers with no data under them.

 

In the first column of the additional column headers you can insert a formula similar to the following that looks up the first column of the data against the unique list to place a True or False under the duplicate header.

 

=IF(ISNA(VLOOKUP(A2,Sheet2!$A$2:$A$30,1,FALSE)),FALSE,TRUE)

 

You can repeat the formula across under the appropriate duplicate column headers but changing the Lookup Value cell and the Table Array to match the unique list for that particular column.

 

Copy the formulas down for the full length of your data.

 

Now all you have to do is apply AutoFilter and set the duplicate column header filters to True and the data in the other columns will be filtered with the same filtering as the original data.

 

Feel free to get back to me if you require any further explanation.

ken sheridan say: This is an example of the code for the last (and most efficient in my experience) method described by John, in this case where the first subform lists report categories and the second reports in the currently selected category in the first.  Firstly for the first subform control, sfcReportCategories, underlying form's module:

Option Compare Database
Option Explicit

Private Sub Form_AfterInsert()

    Me.PARENT.sfcReportList.Form.ReportCategory.DefaultValue = """" & Me.ReportCategory & """"
    
End Sub

Private Sub Form_Current()

    Me.PARENT.sfcReportList.Requery
    
End Sub

Secondly for the second subform control, sfcReportList, underlying form's module:

Option Compare Database
Option Explicit


Private Sub Form_Current()

     Me.ReportCategory.DefaultValue = """" & [Forms]![frmCorrelatedSubForms]![sfcReportCategories].[Form]![ReportCategory] & """"

End Sub

The second subform's RecordSource property is:

SELECT ReportList.*
FROM ReportList
WHERE ReportList.ReportCategory=[Forms]![frmCorrelatedSubForms]![sfcReportCategories].[Form]![ReportCategory];

dale howard [mvp] say:

ReajeevAsthana --

Yes, your understanding is correct.  If you enter progress using only % Complete, Microsoft Project assumes each task started and finished ON SCHEDULE.  The way to get better tracking is to include the Actual Start and Actual Finish fields as you enter progress on tasks.  For example, suppose that a task starts on time but finishes two days late, taking two days longer than planned.  In this scenario, Microsoft Project will show a positive Cost Variance, indicating the task is over budget on cost.  And then this would impact your EVM calculations.  Hope this helps.
