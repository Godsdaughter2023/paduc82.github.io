rohn007 say:

In general what you are asking about is known as "optimizing" your spreadsheet. There are  very many things you can check to make your spreadsheet work better.

Before buying a new computer I would spend some time applying optimizing techniques to your spreadsheet.  Very often using the right single technique can clear a "bottleneck" in the processing and speed up your worksheet by orders of magnitude.  From hours to minutes or even seconds.

*******************

The first (non) optimization technique I suggest is not generally considered one. That is switching from simple spreadsheets to "Excel Tables". Tables switch from row/column references (A1) to "structured References" (readable column names). This makes understanding the formulas easier to understand.  And tables automatically handle copying formulas to new rows and provide some simple formula automation for things like total rows and "slicers" / filters.

The next (non) optimization is to load your tables / data into "PowerQuery" / "Data Manager". PowerQuery compresses the data, making the overall size of your spreadsheet file smaller, making it faster to process calculations. PowerQuery can automate importing new data from outside resources and "data cleaning" of that data before using it for calculations.

The final (non) optimization is using PivotTables.  PivotTables give you automated formula generation. You click and drag, and pick options from ribbons, dialogs and context menus to generate the required formulas in the background. You don't have to worry about learning formula syntax, let Excel worry about it.

The wiki in the next link is a collection of links to free online webinars that demonstrate these 3 "new" tools and links to learning resources.

Download the replay recordings for future reference, and download the example workbooks.  If you are gung ho, you could add time references to the workbook back to the recorded session to make future cross reference easier to find specific features / instructions (I’ve been doing that).

These free sessions run roughly 1 hour each followed by a short shill for their related courses (which definitely sound worth the price).

****** New Computer Suggestions ************

There are 3 general considerations when buying a "more powerful" computer. The class computer chip: i3 / i5 / i7 / i9.  The higher the number very much in general the more powerful. 

Along with the "i" number are generation number like 5***, 6***, 7***, 8***. Again, in general the higher the number, the "newer" and more powerful the computer chip.

The second major consideration is RAM, the Random Access Memory available to the computer. RAM is "fast" memory that the programs and data are loaded into to make them available to the "chip" for processing.  RAM is MUCH MUCH faster than the "hard drive" where the program code and data is permanantly stored. These days most low end computers start at 4GB, and go up from there.  8GB is a common amount on mid range computers. 12-16GB is currently the adequate upper limit for most users. Any more than that without careful research is just a waste of money for the average user.

The 3rd major consideration is where you will "permanently" store your programs and data, the HD (Hard Drive). HDs store data on spinning magnetic disk There are 2 general attributes to consider, size and speed.  The absolute minimum size you should buy in a new computer is 64GB.  256GB is large enough to hold your programs and lots of user data. Disk prices have gone down so much that it common to see 1TB (1000GB) disk drives. Any bigger than that is an extravagance.   Disk speed is also important.  There are 3 disk speeds, 5400 / 7200 / 15000.  5400 is too slow and 15000 is simply too expensive (not enough performance improvement) for a personal computer. 

A new type of "hard drive" has been coming into the market. It is called a SSD. Recently the prices have come down to the point that it is possible to get a large enough SSD at a reasonable price that you don't need HD at all. As a compromise builders have been selling "dual drive" computers with a small SSD and a relatively large HD.  The minimum SSD size you should consider is 128GB (256GB is better) paired with 1TB HD.

Personally, if I was custom ordering a computer I would pay extra to get a larger SSD and eliminate the HD altogther. 

SSD are so much faster than HDs, that simply swapping out the HD in an old computer for a (smaller) SSD makes it perform like a new, faster computer.

Trying to buy the "newest" components costs a LOT more. You can get very good performance at a "reasonable" price by buying 1 or 2 levels from the newest versions of each category.

If I was spec'ing a new "high end" computer for myself today I would look at:

  • an i5 or i7 chip, i9's are too expensive.
  • 12-16GB of RAM, more is a waste
  • 256-512GB SSD, possibly a 1TB HD if they "threw it in"

Depending on the compromises, in general this sort of machine would cost between $800 and $2000. Spending more than that is wastefully extravagant.

Google for more specific suggestions and reviews of computers and hardware.

rubberduckvba say:

Application.Calculation/ScreenUpdating/EnableEvents will make Excel work less between your VBA statements, but won't make your VBA code any more efficient.

What you need to lookout for is where in the VBA code are you accessing the worksheet, why, and whether it's actually needed.

If the macro started out as macro-recorder generated code, it's likely riddled with Range.Select, Worksheet.Activate, Selection.DoSomething late-bound calls - for example if you enter "42" in cell A1, this code might get generated:

    Workbooks("Book1").Activate

    Sheets("Sheet1").Activate

    Range("A1").Select

    Selection.Value = 42

In reality, if "Sheet1" exists in ThisWorkbook at compile-time, it can be statically accessed by its code name, rather than dereferenced from the Sheets collection. This code would be exactly equivalent, vastly more efficient, and more importantly it behaves identically regardless of whether it's written in a standard procedural module or in the code-behind of Sheet3:

    Sheet1.Range("A1").Value = 42

In general, accessing a worksheet should be considered an expensive operation, and thus done as minimally as possible. Instead of iterating cells in a loop, copy the range values into a variant array, and iterate that in-memory array instead - if you heed Rohn007's advice and use tables instead of plain old ranges, you don't even need to work out where the "last row" is:

    Dim contents As Variant

    contents = Sheet1.ListObjects("Table1").DataBodyRange.Value

Depending on what your code is doing, working with a 2D variant array like this can make the difference between VBA code that completes in days, vs VBA code that completes in mere seconds; you can write the modified/computed data into the array, and when you're ready you can dump the array in a single operation onto the worksheet, overwriting the entire range in a split-second.

Don't just throw hardware at the problem, make sure your code is efficient first.
