A few things jump out at me:

 

1) Why the double-negative?  Just use:

 

        If FolderToCheck Is Nothing Then
            'I need to create it
            Set CreateArchiveSubFolder = obj.Parent.Folders.Add("Archive")
        End If
 

2) You ignored my comments about setting objects to nothing when looping.  It may be that this can't be an issue in your code, since I guess you can't have messages selected in two different folders, but for robustness, I would Set FolderToCheck = Nothing before you attempt to set it to obj.Parent.Folders("Archive"):

 

        Set FolderToCheck = Nothing
        On Error Resume Next
        Set FolderToCheck = obj.Parent.Folders("Archive")
        On Error GoTo 0
 

3) On an efficiency note, you have two different objects, FolderToCheck and CreateArchiveSubFolder, but then you don't use either later in the code.  So how about:

 

Sub ArchiveMessages()
    Dim obj As Object
    Dim msg As Outlook.MailItem
    Dim ArchiveSubFolder As Outlook.MAPIFolder
   
    For Each obj In ActiveExplorer.Selection
        If TypeName(obj) = "MailItem" Then
            Set msg = obj
           
            Set ArchiveSubFolder = Nothing
            On Error Resume Next
            Set ArchiveSubFolder = msg.Parent.Folders("Archive")
            On Error GoTo 0
           
            If ArchiveSubFolder Is Nothing Then 'create it:
                Set ArchiveSubFolder = msg.Parent.Folders.Add("Archive")
            End If
           
            msg.Move ArchiveSubFolder 'move the message
        End If
    Next obj
End Sub

 

But (based on that argument that you can't have messages selected in more than one folder), it may be more efficient to do something like this:

 

Sub ArchiveMessages2()
    Dim obj As Object
    Dim ArchiveSubFolder As Outlook.MAPIFolder
    Dim ActiveFolder As Outlook.MAPIFolder
   
    Set ActiveFolder = ActiveExplorer.CurrentFolder
    On Error Resume Next
    Set ArchiveSubFolder = ActiveFolder.Folders("Archive")
    On Error GoTo 0
   
    If ArchiveSubFolder Is Nothing Then 'create it:
        Set ArchiveSubFolder = ActiveFolder.Folders.Add("Archive")
    End If
   
    For Each obj In ActiveExplorer.Selection
        If TypeName(obj) = "MailItem" Then
            obj.Move ArchiveSubFolder 'move the message
        End If
    Next obj
End Sub

I'll report this as a bug because the reason for using the nonbreaking hyphen in this instance is to keep the hyphen with the word, not to keep the word and hyphen with what precedes it. A workaround is to use character 02D7, which is not exactly like a hyphen (but better than the minus sign, 2212, which can be used as a nonbreaking en dash), but it is nonbreaking and doesn't have the same unwanted effect on preceding spaces. How to suppress a header - Microsoft Community

robertallen1111 say:

How to suppress a header

How do I suppress a header?  Do not want headers to appear on every page.
shashirekha say:

If you want to use the field code then you can use the switch provided by “Graham – Word MVP” to remove headers from certain pages in a word document.

{ IF{ Page } <> 2 "{ IF{ Page } <> 5 "Put your header here" }" }

 

Or a calculated field

 

{ IF{ =({ IF{ PAGE } = 2 1 0 } + { IF{ PAGE } = 5 1 0 }) } = 0  "Put your header here”}

 

Use CTRL+F9 for each bracket pair. If you have lots of pages from which you wish to remove the page number, then second method is easier to use – simply add + {IF {PAGE} = n 1 0} for each page where n is the page number.

suzanne s. barnhill say:
Headers are a section property. You cannot suppress a header for a given page (well, you can do it with a field, but this requires you to be able to specify the page number); you must isolate the page into a section of its own, unlink its header from the preceding and following ones, and then empty it. But Word does offer a way to omit a header on the first page of a section; see http://wordfaqs.mvps.org/headerfooter.htm
