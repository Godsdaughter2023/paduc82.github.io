jay freedman say:
One way to find out which styles are built-in vs. user-defined is to try to delete them. If it actually goes away, it was user-defined; if it stays (although possibly reverts from some modified state to its original definition), it was/is built-in.

The only other way I'm aware of is in a macro. Every style is represented by a Style object, and each Style object has a property named BuiltIn that has the value True if it's a built-in style, or False otherwise. This macro will check every style in the current document, and then drop a list of the user-defined styles (if any) into a new blank document for you to refer to. See http://www.gmayor.com/installing_macro.htm if needed.

Sub ListUserDefinedStyles()
    Dim stl As Style
    Dim nameList As String
    Dim listDoc As Document
   
    For Each stl In ActiveDocument.Styles
        If Not stl.BuiltIn Then
            nameList = nameList & stl.NameLocal & vbCr
        End If
    Next
    If Len(nameList) > 0 Then
        Set listDoc = Documents.Add
        listDoc.Content.Text = "User-Defined Styles" & vbCr & _
            vbCr & nameList
    End If
End Sub

Regarding your PS: A new blank document contains definitions of all the styles present in your Normal.dotm template (which are all built-in unless you created user-defined styles and chose the "New documents based on this template" option in the New Style dialog). The Style pane can show all of them, or only those that are currently "in use" or "recommended".

To change what the pane shows, click the Options link at the bottom and change the setting in the "Select styles to show" dropdown. You can also change which styles are "recommended", by clicking the Manage Styles button (the third of the three similar buttons at the bottom of the Styles pane), clicking the Recommend tab of the dialog, and using the "Show", "Hide", and "Hide until used" buttons.

pamcaswell say: When you delete a style, you are deleting it from the document, but not from Word. And whether a style remains in all styles pane depends on whether it is a built-in style and on the view set in the Style Pane options.

Recommend view shows the recommended styles that are available for use whether they are in the document or not. So deleting a built-in style from a document indeed does delete it from the document, but the recommend view still shows the style as still available for use --but with its built-in settings. A deleted custom (user-defined) style will no longer appear in the list.

The All-styles view shows all the styles that are not marked hide or that are marked hide until used but have been used.  Again, deleting a built-in style from a document indeed does delete it from the document, but the all styles view still shows it as available for use --with its built-in settings. A deleted custom (user-defined) style will no longer appear in the list.

The In-current-document view shows the styles that are actually used in the document.  Any style that can be deleted from a document will disappear from the list.

The In-use view shows all the styles that are or have been used in the document and its template (at the time the document was created). Any style that can be deleted from a document will disappear from the list.

The  remove  from Quick Style Gallery command is different from delete from document.  I've not noticed any styles remaining after being removed.

It sounds like you're using Track Changes. If you don't want to track changes in your document then you'll need to turn off Track Changes (On the Review tab, in the Tracking group, click Track Changes) and then Accept all changes in the document, also found on the Review tab.

If you want to track changes but don't want to see formatting changes on the screen, then on the Review tab, in the Tracking group, click Show Markup and turn off Formatting. Or you can use the Final view to suppress them from view on your screen. The only way to prevent track changes from being created is to not use track changes.

subscripts in matrices - Microsoft Community

danielagorra say:

subscripts in matrices

How do I put subscripts in matrices of word 2010?
jay freedman say:
Method 1: Click in the desired entry box in the matrix. Type the characters before the subscript, then type an underscore, and then type the characters of the subscript. For example, x sub 12 would be typed as  x_12 . Then press the space bar, and the subscript will appear. This works both with and without a matrix. For a superscript, use a caret instead of an underscore, as in x^2 to write x squared.

Method 2: Click in the desired entry box in the matrix. On the Equation Tools tab of the ribbon, click the Script button and click the Subscript template (the second one in the top row). The entry box now shows a smaller box to the right, where you enter the subscript.
