a. user say:

The VSS Service should not be running unless you are doing some kind of a backup that uses it.

Are you using any kind of third party backup software that might require the VSS Service to be running all the time?

If defrag used to work, what do you think might have changed since the last time it worked properly?

I have my VSS Service set to "Manual" - that way, it does not start with XP on a reboot and run all day with nothing to do, but it will start by itself when I do a backup and stop itself when the backup is complete (it runs when it needs to).

To check you VSS Service, click Start, Run and in the box enter:

services.msc

Click OK to open the Services applet and find the Volume Shadow Copy Service.

Check the Status and Startup Type columns.

If you are not doing a backup at that moment, the Status column should be empty (not Started) and the Startup Type should be set to Manual.

If that is not the case, right click the VSS Service, Properties, Stop (to stop it), change the Startup Type to Manual and OK your way out (make sure it doesn't start up again by itself).  If it does start up again for no good reason, you need to figure out why and fix it.  Reboot to be sure the changes "stick".

See if your defrag works when the VSS Service is not running...

Ok I disabled hyper-v with command line and it seems that the flicker is now gone.  Working fine for the past 15 minutes. Here is the command

dism.exe /Online /Disable-Feature:Microsoft-Hyper-V
