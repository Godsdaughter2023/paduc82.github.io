Copy certificate from one pc to another (XP Pro to XP PRO) - Microsoft Community

mh118 say:

Copy certificate from one pc to another (XP Pro to XP PRO)

How should I locate and copy my certificate from one PC to another?  Thank you.
diana d say:

Hi Mh118,

Welcome to Microsoft Windows Vista Forums.

To import a certificate

1. Open Certificates.

 2. In the console tree, click the logical store where you want to import the certificate.

Certificates - Certificate holder > Logical store

 3. On the Action menu, point to All Tasks and then click Import to start the Certificate Import Wizard.

 4. Type the file name containing the certificate to be imported. (You can also click Browse and navigate to the file.)

 5.If it is a PKCS #12 file, do the following:

• Type the password used to encrypt the private key

 • (Optional) If you want to be able to use strong private key protection, select the Enable strong private key protection check box.

 • (Optional) If you want to back up or transport your keys at a later time, select the Mark key as exportable check box.

 6. Do one of the following:

• If the certificate should be automatically placed in a certificate store based on the type of certificate, select automatically select the certificate store based on the type of certificate.

 • if you want to specify where the certificate is stored, select Place all certificates in the following store, click Browse, and select the certificate store to use

 To export a certificate

1. Open Certificates.

2. Do one of the following:

• If you are in Logical Certificate Stores view mode, in the console tree, click Certificates.

Certificates - Certificate holder > Logical store  > Certificates

 • If you are in Certificate Purpose view mode, in the console tree, click Purpose.

Certificates - Certificate holder > Purpose

 3. In the details pane, click the certificate you want to export.

4. On the Action menu, point to All Tasks, and then click Export.

 5. In the Certificate Export Wizard, click No, do not export the private key. (This option will appear only if the private key is marked as exportable and you have access to the private key.)

 6. Provide the following information in the Certificate Export Wizard:

• Select the file format you want to use to store the exported certificate: a DER-encoded file, a Base64-encoded file, or a PKCS #7 file.

 • If you are exporting the certificate to a PKCS #7 file, you also have the option to include all certificates in the certification path

 Check the Link “Import and Exporting Certicate: http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/sag_cmimportexport.mspx?mfr=true

Export Certificate: http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/sag_cmprocsimport.mspx?mfr=true

Import Certificate: http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/sag_cmprocsimport.mspx?mfr=true

Diane,

I am having the same issues and have subscribed to this thread to follow any progress. On my own, I have been able to make some progress, but frankly it has led to other problems. In case my solution helps yours, try this:

  1. Remove the connected accounts from Outlook.com
  2. Once removed, log into the accounts you were trying to connect to Outlook.com and change those login passwords.
  3. Once changed (and tested outside of the Outlook.com environment), try adding back in one of the connected accounts to see if it works. If it does, you'll see a message in the Connected Accounts area something like "Update in Progress." Basically what Outlook.com is then doing is importing all of the messages that have been on your main mail server but which haven't yet downloaded into Outlook.com, so it can take a while.
  4. Once done, LOOK IN YOUR MAIN OUTLOOK.COM INBOX FOR THESE MESSAGES.  I had filtering rules set up to manage my connected account messages, but apparently those rules no longer work and everything was sent into my main Inbox.  It took me a while to also realize that Outlook.com was flagging them as either imported or unread by a pink "tag"/flag next to each message.  That's how you can distinguish them in the jumbled mess.

Now, the sad news is that when I try sending outbound email using the connected account's identity, I'm now getting a 505 error message that the message failed to send because of a duplicate date problem. Other people have reported this problem and it is still unresolved. So, in effect, my solution to fix the connected accounts to bring them into Outlook.com has been rendered moot by this new problem.
