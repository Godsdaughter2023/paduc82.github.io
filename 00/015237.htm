e-mule say:

If people are still having this issue, the problem may be a recent update to Edge and the status of Windows Firewall. 

For me, W10 continually failed to do updates because I use a third-party firewall and had the Windows Firewall disabled.  The updater is tied to Windows Firewall, and I had to disable my firewall and then start Windows Firewall through the Services menu (right click Start, then select Computer Management, Services and Applications, Services, find and right click Windows Firewall, select Start).  If Windows Firewall is disabled and not simply un-started, right click on Properties from the Windows Firewall services entry and change the drop down from Disabled to Manual, then right click on the Windows Firewall service entry and select Start.  Once Windows Firewall was started the system did the updates. 

However, if one of the things that gets updated is Edge, you will have to ensure Windows Firewall is running the first time you try to run Edge.  The reason is Edge does an initialization (offering to import things, etc) and if the Windows Firewall is not running, it will not start.

Once Edge has been run for the first time, or the first time after an update, you can stop Windows Firewall (or stop and disable it) and turn whatever firewall you are using back on.  Everything should work from that point on, until the next Edge update.

Hope this helps somebody.
