Is there anyway to "slave" the visible check box (the state of the box is tracked separately in another column) to a row regardles of whether the height of the row varies?

ossiemac say:

If you have used ActiveX Controls then turn on Design Mode and Right click the check box and select Format Control -> Properties tab and select Move and Size with cells.

If you have used Forms Controls then the Property above is not available but you can Right click the Checkbox and select Properties tab and select Move but Don't size with cells.

However, both the above are not overly reliable and personally I think that the following method in lieu of using Check Boxes is a better option.

  1. Remove the existing checkboxes.
  2. Select the cells in the the required column, Excluding the Column Header, and set the Font to Wing Dings.
  3. While range is still selected format to center both vertical and horizontally. (Use Center and Middle Align icons in the Alignment block of the Home Ribbon.)
  4. Right click the worksheet tab name and select View Code to open the VBA editor.
  5. Copy the code below and paste into the VBA editor.
  6. Edit the code with the column number for the check marks and the column Id for True or False linked cells.
  7. Close the VBA editor Alt/Q.
  8. Save the workbook as macro enabled workbook.
  9. Ensure macros are enabled. See Help for hoe to do this. (Option Disabled with notification should suffice.)

Now to insert a check mark double click a cell in the column and to remove the checkmark again double click it again.

You can even use AutoFilter with the Check Marks although you will see a different character in the Filter options but I am sure you will work that out.

bernie deitrick say: Copy the code below, then follow the instructions here:

Private Sub Workbook_Open()
Dim shtCurr As Worksheet
Set shtCurr = Worksheets(Format(DateSerial(Year(Date), Month(Date) - 1, 1), "Mmmm"))

shtCurr.Activate
End Sub

jllatham say:

Go back and uncheck that "Trust access to the VBA project object module" setting!  This is a security issue - the only time that should be checked is when someone has developed code that changes other code in the workbook.  That is extremely rare.  The bottom line is that if you leave it checked, it is then possible for you to open a workbook at a future time that has "malicious" macros in it that might alter your code or add code to your workbook that turns it into a virus or worm.

 

You should be able to set your Macro Security as shown in the graphic.  This is the setting I use - it requires one extra click to open a workbook with macros, but it provides a safety net should you get a workbook from someone else that has macros - you can ask yourself "why does this have macros?  does it need them?" before enabling them and if there's real doubt in your mind, you can always simply ask them "what's up with the macros?".

chris whithear say:

As long as your system is activated correctly after upgrading to Windows 10, you should be fine to complete a fresh install and the system will retain its activation status.

Before performing the upgrade, you can check the activation status in a couple of ways.

1. Command Prompt

Open a command prompt by hitting Win + R on your keyboard or right-clicking on the Start button and then Run from the popup menu.

Enter slmgr /xpr and a popup windows should appear stating you are permanently activated such as below.

2. Windows Settings app

Open the Windows Settings app and then click Updates and Security.

Then go to the Activation section in the options at the left of the settings app and if your system was activated correctly after upgrading to Windows 10, it should display that your system is activated with a digital entitlement.

If one or both of them say as per above, then you can safely go ahead and reinstall. When you do the install you will still be prompted for a installation key which you can skip using the (hard to miss) option at the bottom of the install page and you may be prompted more than once for the installation key.

Once the install is complete, your system should once again show it is activated using the steps above.
