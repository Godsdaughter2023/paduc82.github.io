john - project say:
Jhill6908,
I'm not sure if this will work but you can try it. This method basically emulates Project being on a server with users on individual PCs, (i.e. the way it was done before Project Server was invented).

First, start by copying your working Global (i.e. the current common copy you both are using). Assuming you are running Windows 7, it should be at, C:\users\[your name]\app data\roaming\microsoft\MS Project\14\1033. Put that copy in a common location that both of your computers can access (e.g. a LAN server or WAN server).

Second, take your current working global and move it to the recycle bin. Don't worry, if something goes awry, you can restore it from the recycle bin.

Third, right click on your Project 2010 icon that you use to start Project, (mine is a shortcut on my desktop), to open the Microsoft Project 2010 Properties window. Select the Shortcut tab. In the "start in" box, type the path to the common location where you have placed the global. Hit Apply.

If all goes well, when either of you open Project, it will open that common global file. The thing I don't know (because I haven't gone through this full process myself), is whether this will limit you to only one user (i.e. write access) at a time - it may. If so, and you can live with that restriction, you should be all set. If it doesn't work, or you can't live with a write access restriction, then this method won't work. In that case, restore you working global from your recycle bin and keep trading globals via e-mail.
