dirk goldgar say:

This should work, however I neglected to mention (Sorry!) I open the form as a acDialog form.  Can I get the calling forms location then set within the dialog form?

Sure.  You could pass the name of the form and, if desired, the name of the calling control via the OpenArgs argument, or -- possibly easier -- use the fact that in the Open event of the dialog form, the calling form is still the active form, and use Screen.Active form to get a reference to the form.  Here's an adaptation of the previous code that could be placed in the Open event of the dialog form to position that form next to the active control on the calling form:

'------ start of code ------

   
    Dim frmCaller As Access.Form
    Dim ctlTarget As Access.Control
   
    Set frmCaller = Screen.ActiveForm
    Set ctlTarget = frmCaller.ActiveControl
   
    Dim lngBorderHoriz As Long
    Dim lngBorderVert As Long

        lngBorderHoriz = (.WindowWidth - .InsideWidth) / 2
        lngBorderVert = (.WindowHeight - .InsideHeight) / 2

            (.WindowLeft - Me.WindowWidth) + lngBorderHoriz + ctlTarget.Left, _
            .WindowTop + lngBorderVert + ctlTarget.Top
           
    End With
   
End Sub

I just tested it, and it worked for me the way I expected.

albert d. kallal say:

·        I assuming you mean the new built in web browser control and not an ActiveX browser?

 

(It would REALLY help if you had mentioned the version of access and they you are using the new native web browser control).

 

Remember, this control is like ANY other access control. You can bind the control to an expression or EVEN a column in the underlying table for the form.

 

What this means you can go:

 

Me.WebBrowser0.ControlSource = "=(""www.bing.ca"")"

 

 

However, try the following neat trick:

 

In design mode, drop a text box on your form.

 

Now, make the web control data source an expression that includes that text box and some string work.

 

Try this:

 

=("www." & [text2] & ".com")

 

Now, just type in Google, or Bing or ebay or whatever into the text box, and you see the web control respond just like any other native control on your form.

 

So, you don't need nor use a navigate to. This works just like a text box. In fact, if you bind the web control to a column, then just go:

 

Me.WebBrowser0 = "www.bing.ca"

Or

Me.WebBrowser0.Value = "www.bing.ca"

 

So this works like any other bound control that access has.

 

HOWEVER if the control is un-bound, then you can NOT just assign it a value like the above. I can't really say this is a bug, but I do think the assignment should work just like a text box. However, when it is bound control, then just setting the value of the control does work.

 

As mentioned, you can bind that to any legitimate access expressions we have, including VBA functions.

 

This control is beyond useful and opens up the display of web photos, web pages with java and cool text, displaying part and catalog items. And even more cool is you can launch a form, let the user browse or search, and then grab the URL back into the form to be saved. marysully say:

Each recipient list has a file name with an “mdb” extension and is most likely stored in your “My Data Sources” folder on your PC . Go to that folder and delete the data source files you no longer need.

doug robbins - mvp office apps & services (word) say:
One the Mailings tab of the ribbon, expand the Start Mail Merge dropdown in the Start Mail Merge section of the ribbon and then select Normal Word Document and save the document.  That will remove the data source from the document.  If desired, before saving, select the type of mail merge main document that it was and then save the document.  It will then be saved as a mail merge main document, but without a data source attached to it.
