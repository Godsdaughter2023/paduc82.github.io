If the combo box is always on the form to be filtered, you don't need to open the form again ("DoCmd.OpenForm") because the form must already be open.  All you need to do is set and activate the form's Filter, using the correct reference.  Your code could look like this:

 

'------ start of code ------

Private Sub cbo_jump_to_AfterUpdate()

    On Error GoTo Err_cbo_jump_to_AfterUpdate

    Dim stLinkCriteria As String
   
    stLinkCriteria = "[pdID]=" & Me![cbo_jump_to]
   
    With Me
        .Filter = stLinkCriteria
        .FilterOn = True
    End With
   
Exit_cbo_jump_to_AfterUpdate:
    Exit Sub
   
Err_cbo_jump_to_AfterUpdate:
    MsgBox Err.description
    Resume Exit_cbo_jump_to_AfterUpdate
   
End Sub
'------ end of code ------


 

marshall barton say:

First, don't open a form that is already open.  That implies that using:

   DoCmd.OpenForm Me, , , stLinkCriteria

is unnecessary as well as incorrect.  (If it were useful, it would be Me.Name instead of just Me.)


That line is only needed if the form is not already open, in which case, setting the Filter property is redundant.


The other error I see is the With statement, it should just be

   With Me


Actually, I don't see much of a reason for so many lines of code in this simple situation.  I think I would write it this way:


Private Sub cbo_jump_to_AfterUpdate()
On Error GoTo Err_cbo_jump_to_AfterUpdate

With Me
     .Filter = "[pdID]=" & Me![cbo_jump_to]
     .FilterOn = True
End With

Exit_cbo_jump_to_AfterUpdate:
    Exit Sub

Err_cbo_jump_to_AfterUpdate:
    MsgBox Err.Description
    Resume Exit_cbo_jump_to_AfterUpdate
   
End Sub

If you don't know if the form is already open or not, and you want to put the procedure in a standard module so you can call it from any form, then you can use:

Public Sub JumpTo(FormName As String, WC As String)
On Error GoTo ErrHandler


If CurrentProject.AllForms(FormName).IsLoaded Then
   With Forms(FormName)
      .Filter = WC
      .FilterOn = True
   End With
Else
   DoCmd.OpenForm FormName, , , WC
End If

ExitHere:
    Exit Sub

ErrHandler:
    MsgBox Err.Description
    Resume ExitHere
End Sub

And then call that procedure from any form using a line like:
   Call JumpTo(PhoneDetails, "[pdID]=" & Me!cbo_jump_to)

jeeped say:

Lookup formulas do not carry the formatting with them; just the raw value. In some cases, this can be as raw as an integer (e.g. 41,857) when a date is retrieved (e.g. 08-Aug-2014). You need to pre-format the receiving cell with the formula. Conditional formatting could change the cell's specific formatting with the value(s) being retrieved.
