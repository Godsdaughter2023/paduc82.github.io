a. user say:

Hi Pantelis,

Thank you for posting your query in Microsoft Community.

I understand the inconvenience you have experienced, while creating Windows Back Up. Conflict arises when internal application are using the files which you're trying to Back Up or trying without admin rights. Follow below mentioned steps and check if it helps:

* Press Windows Key + X and select Command Prompt(Admin)

* Type or copy paste the below mentioned command line into command prompt

"net localgroup administrators useracc /add" (without quotes)

Note: 'useracc' in the command line should be replaced with your useraccount name. For example: If your account name is computer, then the command line is "net localgroup adminstrators computer /add"

Close the command prompt once it is executed.

* Press Windows Key, type Services in Search and hit Enter key. When the Services page is open, check for Volume Shadow Copy service.

* Right click on Volume Shadow Copy service (VSS) and Check if it is started. If it running Stop the service and Start it again.

Now try to Back Up the data and see if the problem still persist. In case problem still persist, suggest you to try creating New User account with admin rights and check if it helps.

Note: Creating new user account will impact the 'Go Back' option. Where this Go Backhelps you revert to previous OS from Windows 10.

Hope this will address your query.
