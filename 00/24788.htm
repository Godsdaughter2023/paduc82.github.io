teylyn say:

Your approach with the "Between" is as good as mine with the formula. By defining a numeric range, you're implicitly ignoring text. It's a matter of personal preference and none of these approaches should cause any issues.

teylyn say:

Hi again,

the formula in column S will return an empty string when the division of the other cells returns an error. A string is always greater than any number, therefore the conditional formatting with just the  "Cell Value greater than" is not sufficient. 

Instead of one of the pre-packaged rules for the green condition, use the option "Use a formula to determine which cells to format", then use this formula (with column S selected, starting with S5 as the active cell):

=AND(ISNUMBER(S5),S5>1.201)

Note that there are no $ signs in the reference to S5, which means the formula will always look at the current cell when evaluating the condition.

dustiiwolf say:

True, but reFS might have had additional years of internal testing, and it's not like NTFS was just now introduced; NTFS was released along with Windows 3.1. Around 90% of that 20+ years of testing and improvements included usage within "live operating environments": Real world scenarios such as usage within consumer applications and operating systems, (i.e. the file system for the main disk partition of Windows and for use with external storage).

Theres also the facts that:

  • ReFS is already available within Windows Server for formatting additional disk drives
  • ReFS is additionally available within the Windows Preinstallation Environment for formatting any drive (removable or otherwise, save for the install [C:] drive), and is recognized by Windows 8.1
  • To quote Steven Sinosfsky,

"With this in mind, we will implement ReFS in a staged evolution of the feature: first as a storage system for Windows Server, then as storage for clients, and then ultimately as a boot volume. This is the same approach we have used with new file systems in the past."

So far they have implemented ReFS as a storage system for Windows Server, and as of 8.1, as a supported file system for drives already formatted with ReFS, so depending on whether you could interpret ReFS' current implementation within Windows 8.1 as "storage for clients" or not, then the next step, according to Sinofsky, would be to implement it as a boot drive within a future version of Windows.

Even if it there's plans for full implementation in consumer versions of Windows before use as a boot drive, It's a sizeable possibility that Windows 10 will at least include the ability to format drives with the ReFS filesystem, in which case it's possible that recovery images and even Wimboot images can be stored on ReFS partitions.

Lets Say on Sheet 1 it is made up from "D15 : D" and from Sheet 2 "N18:N" and combine both in one data validation list. The ranges could be used from names ranged or dynamic ranges, but the problem how to combine both of them.

  • Right-click the sheet tab
  • Select the View Code option
  • Paste the following event code:

'=========>>
Option Explicit

'--------->>
Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Dim SH As Worksheet
    Dim RngA As Range, RngB As Range, rngDV As Range
    Dim aCell As Range, bCell As Range
    Dim arrA As Variant, arrB As Variant, arrDV() As Variant
    Dim aRow As Long, bRow As Long
    Dim i As Long, j As Long, k As Long
    Dim x As Long, y As Long
    Dim sStr As String
    Const ShName As String = "Sheet2"                                       '<<==== Change
    Const sDvCell As String = "D15"                                               '<<==== Change
    Const sDvCell2 As String = "N18"                                            '<<==== Change
    Const sDVaddress As String = ("A1:A10")                              '<<==== Change

    Set SH = ThisWorkbook.Sheets(ShName)

    With Me
        Set aCell = .Range(sDvCell)
        i = aCell.Row - 1
        aRow = LastRow(Me, aCell.Resize(.Rows.Count - i))
        Set RngA = aCell.Resize(aRow - i)
        Set rngDV = .Range(sDVaddress)
    End With

    With SH
        Set bCell = .Range(sDvCell2)
        j = bCell.Row - 1
        bRow = LastRow(Me, bCell.Resize(.Rows.Count - j))
        Set RngB = bCell.Resize(bRow - j)
    End With

    arrA = RngA.Value
    arrB = RngB.Value

    ReDim arrDV(1 To UBound(arrA, 1) + UBound(arrB, 1))
    For x = 1 To UBound(arrA, 1)
        k = k + 1
        arrDV(k) = arrA(x, 1)
    Next x

    For y = 1 To UBound(arrB, 1)
        k = k + 1
        arrDV(k) = arrB(y, 1)
    Next y

    sStr = Join(arrDV, ",")
    With rngDV.Validation
        .Delete
        .Add Type:=xlValidateList, _
             AlertStyle:=xlValidAlertStop, _
             Operator:=xlBetween, _
             Formula1:=sStr
        .IgnoreBlank = True
        .InCellDropdown = True
        .InputTitle = ""
        .ErrorTitle = ""
        .InputMessage = ""
        .ErrorMessage = ""
        .ShowInput = True
        .ShowError = True
    End With
End Sub

bob jones aka: cybertaz say:

No shortcut key assignments are required. What I understand you to be asking for is a standard part of Word's AutoFormat as You Type feature:

  • To start the bulleted list, type an asterisk (*), press either the Spacebar or the Tab key & then type your list item
  • When you then press return the line will indent, the * will change to a solid bullet & the new line will be at the same level
  • To Demote to a lower level, just press Tab after pressing return. (A different bullet character is assigned to each of 9 levels.)
  • To Promote one level use Shift+Tab or press return for each level up
  • To end the automatic bulleting press return twice after typing the last item. (Note that if you're ending the list on a demoted level you may need to press return more than twice depending on the last level used.

Word keyboard shortcuts

scottgem say:

First, using macros IS programming to some extent. It is just programming done through a GUI interface rather than writing lines of code. But the macro language is 2007 is VERY limiting. If you understand how macros work, you should be able to learn the VBA equivalents very easily.

Second I think you need to modify your database structure. Your table should look like this:

tblTaskTrack

TaskTrackID (Primary Key Autonumber)

TaskID (Foreign Key) more on that in amoment

DueDate

CompleteDate (Prefer using a Complete date rather than a Completed checkbox so you know when it was completed.

Now you will also need a new table:

tluTasks

TaskID

TaskName

Repeating the name for each task record is redundant. So you should have a combobox on your form to select the TaskID whihc will list the Tasknames to choose from. The combobox wizard will do that for you.

So, now to repeat your record the code is simple: Use the After Update event of the CompletedDate control. The code would be:

Dim strSQL As String

strSQL = "INSERT INTO tblTaskTrack (TaskID, DueDate) " & _

               "SELECT TaskID, #" & DateAdd("m",1,Me.Duedate) & "# AS DueDate " & _

               "FROM tblTaskTrack " & _

               "WHERE TaskTrackID = " & Me.TaskTrackID & ";"

CurrentDB.Execute strSQL, dbFailOnError

That's it. When you press the button the current record will be added with a new date.

You can make this more flexible, by adding a combobox to select period. Instead of incrementing by a month, you could select the increment period as days, weeks, months, quarters, years. You could even add a textbox to select the number you want to increment. If you would like more details on that let me know.

shirley wang msft say:
Hi NenLi,

We can copy a list or a library in a SharePoint site with all contents to another site via the Save document library/ list as template feature. Here are the detailed steps for your reference:

1. Go to the teamsite and access the library which we would like to copy with the classic UI.
2. Click LIBRARY > Library settings > Permissions and Management > Save document as template.
3. File name > Template name > Include Content > OK.
4. A message saying Operation Completed Successfully appears and click OK.
5. Go to the List Template Gallery in the site via Site Settings > Web Designer Galleries > List templates.
6. Download the template to our computer via clicking on the title.
7. Go to another site and click Settings > Site settings > Web Designer Galleries > list templates.
8. Upload the template saved on the computer to the page via clicking FILES > Upload Document > Save.
9. Click Settings > Add an App > select the library template > Name > Create.

If we can’t find Save document library / list as template in the site settings page, please enable Custom Script in SharePoint admin center > Settings > Custom Script as below. Then, wait 24 hours to take effect for it.


In addition, we can also use Open with Explorer to copy files to another site easily:

1. Go to the library in a teamsite with classic SharePoint UI.
2. Click LIBRARY > Open with Explorer. The library will open with file explorer.
3. Create a new library in another site and open it with files explorer as above.
4. Copy all files in the first explorer to the second one.
5. Check if the files can be copied to another site.

In these ways, when we access the files in another site, we can’t get know the original library and site where it comes from. The library is just copied to another site instead of syncing files between locations. We can’t find the correct source list.
