There's another factor not included in the KB article that Anoop mentioned. Because there are no other option button controls available for inserting in the body of a Word document, I believe that the option buttons you used are ActiveX controls. If that's true, they are the cause of both the large size of the document and its slow loading time. As long as you use those controls, there won't be anything you can do to speed it up.

A discussion of the properties and drawbacks of ActiveX controls may be found at http://msdn2.microsoft.com/en-us/library/aa140269(office.10).aspx.

An alternative that would reduce the document size and make it open as quickly as possible is to replace all 10 of the option buttons for each question with a single dropdown form field from the Legacy Tools button on the Developer tab (or, if all users have Word 2007 or later, use a dropdown content control for each question). If using form fields, you'll need to apply forms protection to the document to make them work.

In any case, whether with option buttons, form fields, or content controls, you must have some macro programming to store the results of filling out the form; but the programming is different for each one. If you need help with that, post back.

macropod say:

Suppose your datasource contains a field named 'Last' and that field contains an X against each last record. In that case, you might use a series of fields coded as:

{SET Last {MERGEFIELD Last}}{NEXTIF {REF Last}<> "X"}

Exactly how the article's author envisaged combining this with an IF field isn't apparent.

That said, what the 'Help' article is suggesting can be done better using Word's Catalogue/Directory Mailmerge facility (the terminology depends on the Word version). To see how to do so with any mailmerge data source supported by Word, check out my Microsoft Word Catalogue/Directory Mailmerge Tutorial at:
http://windowssecrets.com/forums/showthread.php/154370-Microsoft-Word-Catalogue-Directory-Mailmerge-Tutorial
or:
http://www.gmayor.com/Zips/Catalogue%20Mailmerge.zip
The tutorial covers everything from list creation to the insertion & calculation of values in multi-record tables in letters. Do read the tutorial before trying to use the mailmerge document included with it.
 
The field coding for this is complex. However, since the tutorial document includes working field codes for all of its examples, most of the hard work has already been done for you - you should be able to do little more than copy/paste the relevant field codes into your own mailmerge main document, substitute/insert  your own field names and adjust the formatting to get the results you desire. For some worked examples, see the attachments to the posts at:
http://www.msofficeforums.com/mail-merge/9180-mail-merge-duplicate-names-but-different-dollar.html#post23345

http://windowssecrets.com/forums/showthread.php/157725-Word-2010-Merge-from-excel-into-Table-Directory?p=928391&viewfull=1#post928391
 
Another option would be to use a DATABASE field in a normal ‘letter’ mailmerge main document and a macro to drive the process. An example of this approach can be found at: http://answers.microsoft.com/en-us/office/forum/office_2010-word/many-to-one-email-merge-using-tables/8bce1798-fbe8-41f9-a121-1996c14dca5d

All I am looking for is a simple VBA line of code to populate the Control with a string.  Since they do not have names, I am setting their Titles.  So I need to be able to set a specific Titiled control with a value, something like:

ActiveDocument.ContentControl("LastName").SetPlaceholderText Text:="Smith"

Of course this does not work.

Is the only solution to loop through all the controls?  Seems ridiculous!

Thanking you in advance for the help.

macropod say:

Formfields haven't been deprecated - they're still fully supported. As for working with content controls you could use:

'Change the placeholder text
ActiveDocument.SelectContentControlsByTitle("LastName")(1).SetPlaceholderText Text:="Smith"

or:

'Change the content
ActiveDocument.SelectContentControlsByTitle("LastName")(1).Range.Text = "Brown"

depending on whether you want to change the placeholder text or the content.
