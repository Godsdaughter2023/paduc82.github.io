texmexdragon say:

Ok.....solved my own question.  

And a big "duh!!!!" on me. 

I have been messing with the "format"....i.e. removing axis labels, etc and THOSE do not update.    So, for example, if your chart has Jan - June monthly data in it, and you go and omit the Feb and March axis label, the chart in Excel will reflect this of course.....but those are NOT changes that carry over to PowerPoint.     It is the DATA itself that only refreshes.    So, to use my example, if the Jan - June data changed....then that data does indeed update in PPT.     If one wanted to physically omit the Feb and March numbers from the PPT chart, then you can choose "select data" and it will open up the dialog box allowing you to deselect those months from the chart.   

I was "expecting" that Excel would also carry over the formatting changes on the refresh...but it only refreshes the data.

Mark this as solved!

Prefill Word formfields with data? - - Microsoft Community

dan guinan say:

Prefill Word formfields with data? -

Can I open a word document and prefill formfields with data? 

Kind of like I can open a PDF with FDF file and fill in most of the form before the user completes the form?

jay freedman say:

Just as programming is needed to fill a PDF form from FDF data, the same is true of a Word form. Programming Word can be done within Word in the Visual Basic for Applications language (also known as a macro), or outside of Word in any of the languages supported by Visual Studio.

There are a lot of factors to consider in planning this sort of work.

  • The macro or external program can know about all the formfields in advance -- for instance, if the form is based on a template, and that template also contains the macro. Or (with some more effort) the code can be programmed to examine the form and determine what fields it contains.
  • The data can be supplied by an external file (usually a text .txt file or a comma-separated .csv file, but also an Excel file or an Access table or XML file); in that case the macro or program has to be able to read and understand the file's structure. Or if the data is always the same, it can be incorporated directly in the code.
  • Depending on what version(s) of Word will be used to complete the form, you may use the older "legacy formfields" or the newer content controls.

If you describe more about the way you envision the form being used, we can probably narrow the choices and make some recommendations.
