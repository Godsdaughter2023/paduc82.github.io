Moved from:  Office / Word / Windows other / Office 2016

jay freedman say:

AutoText would work, as would any of the other categories of building blocks. They would, of course have to be stored in the same template on which the users are basing their documents, and the macros to make it all operate would also be in the same template (which would therefore have to be a *.dotm file instead of *.dotx).

Since you're restricting the editing, use legacy form field check boxes. The restriction would be for "filling in forms". (That type of restriction not only prevents any editing/formatting outside of form fields, but also disables a number of features such as spell checking and inserting graphics; if those are necessary, there are workarounds.)

Your template will need a bookmark for each possible AutoText insert. For this to work properly, each bookmark in the template must enclose one or more space characters.

Each check box's Properties dialog lets you specify Entry and Exit macros. For this project, only Exit macros are needed, one per check box. (With some fancier programming it would be possible to make one macro handle all the check boxes, but that's for a later lesson.)

Each exit macro should look like this, where only the numbers (1 in this macro, 2 in the next one, and so on) change:

 Sub Exit1()
    Dim rg As Range
    With ActiveDocument
        Set rg = .Bookmarks("BK1").Range
        .Unprotect Password:=""
        If .FormFields("Check1").CheckBox.Value = True Then
            .AttachedTemplate.AutoTextEntries("AT1").Insert Where:=rg
        Else
            rg.Text = " "
        End If
        .Bookmarks.Add Name:="BK1", Range:=rg
        .Protect Type:=wdAllowOnlyFormFields, NoReset:=True, Password:=""
    End With
End Sub

Here I've used the name "AT1" for the AutoTextEntry that corresponds to the check box "Check1" and the bookmark "BK1". You can use whatever naming convention you like for these. If your template's editing restriction has a password, insert it into the empty double quote marks in the .Unprotect and .Protect statements.

Add the macros to a module in your template, and then open the Properties dialog of each check box and set the corresponding Exit macro (their names won't be available in the dialog until the code exists). Finally, turn on the "filling forms" protection and save the template.

Some comments on things in the code that may not be obvious:

  • An AutoTextEntry can't be inserted into the document (outside of a form field) while the document is protected. The macro must unprotect the document, insert the AutoText, and reprotect the document. This happens quickly enough that users can't do any editing while the protection is turned off.
  • When the AutoTextEntry or the space character is inserted, Word deletes the bookmark. That requires the macro to reinsert the bookmark so it can be used again if the check box changes again.
  • If the bookmark doesn't enclose at least one character, the insertion will be placed to the right of (outside) the bookmark, and the macro won't work properly; hence the requirement for a space to start with.
  • When the document is reprotected, if the optional parameter NoReset is False or omitted, the form fields will return to their default values. Setting it to True prevents the current values from being wiped out.

From your description, the proxy address was previously used by a deleted user. Based on my knowledge, after you delete a user mailbox, the deleted user mailbox will turn into inactive mailbox. Remove-MsolUser -UserPrincipalName <user account> -RemoveFromRecycleBin true

At this point, there really is not a way to easily share templates across users.

1. Create the notebook with the sections and pages which you want everyone to use as templates.

Is it possible to have "hover" or "popup" text in MS - Microsoft Community

torontonian77 say:

Is it possible to have "hover" or "popup" text in MS Word?

Sorry if I'm not explaining this correctly. The effect that I'm trying to achieve is like the one at this site:

So when I hover over the link, the citation appears over top of it. Is it possible to do something similar in Word? If so, how? Thanks in advance for any assistance.

suzanne s. barnhill say:

Nothing quite like that, no. There are several actions that will generate a popup or ScreenTip in Word:

  1. A hyperlink. You can bookmark the current location and insert a hyperlink to that bookmark (so that following the hyperlink does not move the selection). In the Edit Hyperlink dialog, you can specify the text to be displayed in the popup. But the quantity of this text is limited (255 characters, I think), and no formatting is possible (no superscript numerals as you have there, no colored title bar, etc.). And there is no way to avoid having it include "CTRL + click to follow link."
  2. A comment. You can insert fairly long comments, and some formatting is available, but display of comments will depend on a user's local settings (and version of Word). If balloons are enabled, the comment may be displayed in a balloon, or display of the Reviewing Pane may be triggered by the presence of comments. Both of these will be in addition to the ScreenTip on mouseover. Not to mention that the comment will include information on who made the comment and when (some of this will remain even if you use the Document Inspector to Remove Personal Information from the file).
  3. An endnote. This would probably come closest to what you want. When you insert a footnote or endnote reference in the text, the entire text of the note is shown in a ScreenTip when you mouse over the reference number, so this might be the closest you could come to a solution. Notes can have almost the same range of formatting as ordinary document text, but unfortunately none of this is shown in the ScreenTip, so that's not very satisfactory. You'd need to leave the reference mark visible (not format it as Hidden, even though it would still work that way) in order to give the reader something to mouse over.

The bottom line is that none of these is really a perfect solution. What your screen shot shows is an HTML feature that is not available in Word.
