brandon2003 say:

My guess is that you're seeing hardware limitations, specifically hard drive read and write speeds. When performing a copy like this, the data gets copied to the target machines RAM first, before it is written to it's hard drive. I'm thinking that the 30 secs of high speed that you're seeing is the data going into RAM (which is really fast), but that RAM gets filled up after a bit and has to wait for the hard drive (which is slow) to write data before it can be fed any more. If the target hard drive is a regular spinning disk drive, then yes, this will be slow.

Hope this helps.
