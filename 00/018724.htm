Hello,

Windows authentication, which includes both NTLM and Kerberos v5 authentication, is best suited for an intranet environment for the following reasons:

  • Client computers and Web servers are in the same domain.
  • Administrators can make sure that every client browser is Internet Explorer 2.0 or later versions.
  • HTTP proxy connections, which are not supported by NTLM, are not required.
  • Kerberos v5 requires a connection to Active Directory, which is not feasible in an Internet environment.

ryan lv msft say:

Hi Dchap123,

I'd like to share the outcome of my research and test on Office channel update behaviors when configuring group policy "Update Channel" and related registry settings.

1.    Two Win 10 computers with a domain controller (Windows Server 2012 R2) configured to push group policy settings.
2.    Office Product “Office 365 Business” and “Office 365 ProPlus” installed via Office 365 portal on individual machines.

Available entries under registry path HKLM\Software\Policies\Microsoft\office\16.0\common\officeupdate\updatebranch are:
Current --- Current Channel
FirstReleaseCurrent --- Office Insider slow (First Release for Current Channel)
InsiderFast --- Office Insider Fast
Business --- Deferred Channel
Deferred --- Deferred Channel
FirstReleaseDeferred --- First Release for Deferred

Note:
1.    Use "FirstReleaseDeferred" instead of "Validation" for First Release for Deferred channel. See reference here.

2.    Same behavior happens for both “Office 365 Business” and “Office 365 ProPlus” when performing Update Now action after setting the registry/group policy.
3.    Setting up corresponding group policy value will perform the same effect.

Office client will not update when you want to switch between Current and First Release for Deferred channel by setting up the registry/group policy as above.

For example, if the installed channel is Current Channel (i.e. “Current”) and you set registry/group policy value to “FirstReleaseDeferred”, Office client regards installed version as up-to-date and channel won’t be changed.

hansv mvp say:

As you have found, you cannot use a formula in the cell linked to the check box. Instead, use a combination of formatting and conditional formatting.

Select the cell or cells linked to check boxes.

On the Home tab of the ribbon, click the arrow in the lower right corner of the Number group.

Select Custom as category.

In the Type box, enter three semi-colons:    ;;;

Click OK.

All cells now appear empty, but don't worry.

On the Home tab of the ribbon, select Conditional Formatting > New Rule...

Select 'Format only cells that contain'.

Leave the first dropdown set to the default 'Cell Value'.

Select 'equal to' from the second dropdown.

In the box next to it, enter TRUE

Click Format...

Activate the Number tab.

Select Custom as category.

In the Type box, enter three semi-colons followed by an X:    ;;;X

Click OK, then click OK again.

FALSE values should now be hidden, and TRUE values should display as X.
