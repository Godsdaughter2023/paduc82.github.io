amit tandon say:

My previous code covered rows inserted from 6 onwards. Below code covers rows inserted at any row number and the formula will be displayed in A1 style viz =SUM(A6:A20).

Sub AddRow3()

Dim Msg, Style, Title, Response
Dim rngAdd As Range, rngSum As Range, StartRow As Long, EndRow As Long
Set rngAdd = ActiveCell
Set rng = Range("A:A").Find("=SUM")
Set rngSum = Range(Replace(rng.Formula, "=SUM", ""))
StartRow = rngSum.Row
EndRow = rngSum.Row + rngSum.Rows.count - 1
Msg = "Do you want to insert a row?" 'Define message.
Style = vbYesNo + vbDefaultButton2 'Define buttons.
Title = "Insert a Row" 'Define MsgBox title.
Response = MsgBox(Msg, Style, Title)

If Response = vbYes Then 'User chooses Yes.
ActiveSheet.Unprotect
    rngAdd.Offset(0, 0).EntireRow.Insert   'Add a row "above" the current position
    rngAdd.EntireRow.Copy    'Copy the row you just "moved down"
    rngAdd.Offset(-1, 0).EntireRow.PasteSpecial xlPasteAll
    'Paste "move" the old data "up" into newly created row
     'This prevents the Run-time error '1004': No cells were found,
    'from appearing.
    On Error Resume Next
    Application.EnableCancelKey = xlErrorHandler
    rngAdd.EntireRow.SpecialCells(xlCellTypeConstants).ClearContents
    'Clear old row so that it "appears" to be a "new" row
    If rngAdd.Offset(-1, 0).Row >= StartRow And rngAdd.Offset(-1, 0).Row <= EndRow Then
        rng.Formula = "=SUM(A" & StartRow & ":A" & EndRow + 1 & ")"
    ElseIf rngAdd.Offset(-1, 0).Row = EndRow + 1 Then
        rng.Offset(-1, 0).Formula = "=SUM(A" & StartRow & ":A" & EndRow & ")"
        rng.Value = ""
    End If
ActiveSheet.Protect
End If
rngAdd.Offset(0, 0).Select

End Sub

I have an Access Database that has a Form and a Subform.  The Form is a list of Clients (Name, address, phone number, etc.) and the Subform contains all of the issues I log for that client (such as if they call me, the first record of the Subform is the phone call, if I receive an email from them with a question then I create a new record on the Subform so I can log all times I have helped the client.)

I'd like to be able to print a record I have open on the Subform so I can have a print-out of all open issues that I still have to work on.  I created a Print button with the Command Wizard (Print Record option) and I changed the Subform "Can Grow" option to No and it will only print the First record of the subform.  If I have say, Record 2 open in the Subform, and press the Print button, Access will still print the first record and will go to the first record even if I was on Record 2.

Chelle

boblarson say:

Set up a REPORT the way you want it to be.  Reports are meant for printing, forms are not optimized for printing so it is harder to get them the way you want them and your options are limited.  But a report you can call from your form with the PK of just that one in the subform like:

 

DoCmd.OpenReport "ReportNameHere", acViewPreview, WhereCondition:="[PKFieldNameHere]=" & Me.SubformControlName.Form.ControlNameHere

 

Where SubformControlName is the name of the control on the parent form which houses/displays the subform and not the subform name itself unless they share the exact same name.  The .Form. part stays as written as it tells Access you want to refer to a method or property of the subform and not the subform control.
