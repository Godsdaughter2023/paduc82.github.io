john w. vinson say:

Just note that a scanned image of a document is just that - an image, black pixels on white. It does not become data, usable for calculations, filtering, searching, sorting etc. until you use Optical Character Recognition, parse it into individual text strings, and load  it into a properly normalized set of Tables.

Sure, you can keep a copy of the scanned image for printing - but that's about all it's good for, without a pile of other work (minimally retyping all the data on the image into textboxes on a Form).

Good luck!

The MSRT normally runs and deletes itself automatically at completion, per this excerpt from the linked KB Article:

http://support.microsoft.com/?kbid=890830

How to remove the Malicious Software Removal Tool

The Malicious Software Removal Tool does not use an installer. Typically, when you run the Malicious Software Removal Tool, it creates a randomly named temporary directory on the root drive of the computer. This directory contains several files, and it includes the Mrtstub.exe file. Most of the time, this folder is automatically deleted after the tool finishes running or after the next time that you start the computer. However, this folder may not always be automatically deleted. In these cases, you can manually delete this folder, and this has no adverse effect on the computer.

bernie deitrick say:

It's not necessary - dates in Excel are really just numbers, the count of days since 1/1/1900. The display formatting doesn't change the underlying number, just how it looks in the cell. In the formula bar, you cannot control the display, which is taken from your Windows Date Settings.

So, if your user types 1/1 or 1/1/17 or 1/1/2017 or 1-1 or 1-1-17  etc.   then Excel will enter 42736 into that cell - or January 1, 2017.  In the formula bar you will see the default Windows format, and in the cell will appear the formatted date string that you have chosen for that cell.

Using strings for dates is a bad idea, but if you need to control how the date is used in a string, you would need to use a formula like

="This is the entered date: " & TEXT(A2,"dd/mm/yyyy")

where cell A2 has that date. 

In VBA, that would be done like

Msgbox "This is the entered date: " & Format(Range("A2").Value,"dd/mm/yyyy")

Msgbox "This is the entered date: " & Range("A2").Text    'Returns the formatted display for cell A2
