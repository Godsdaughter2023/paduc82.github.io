Issue with Excel Save & Send for CSV (Excel 2010) - Microsoft Community

ryan.echternacht say:

Issue with Excel Save & Send for CSV (Excel 2010)

I think I've discovered a small bug with Excel's Save & Send functionality when sending CSV files encoded as UTF-8.  When sent, the data in the file is transformed into tab-separated (instead of comma separated) and then unusable by the receiver.  This is using Excel 2010

Background:

I found this while exporting a report from Sql Server Reporting Services 2008 (as CSV).  I then opened it in Excel, and it looked find, so I did File > Save & Send > Send Using Email > Send As Attachment.  When my receiver looked at it, it was messed up (everything was smushed into the leftmost column).  I played around with it (a lot) and eventually discovered (using Notepad++) that Excel CSV's (and hand crafted CSV's using notepad) are encoded using "UTF-8 w/o BOM" (byte order marking); CSV's from Sql Server Reporting Services use plain" UTF-8".  When I changed the encoding using Notepad++ I could fix my SSRS report (or break my natural excel reports).  The file being sent via Save & Send was encoded as "UTF-8 w/o BOM".  

Reproduction steps:

1.  Create an CSV file in excel, with at least 2 columns of data

2.  Save this as a CSV

3.  File > Save & Send > Send Using Email > Send as Attachment > email it to yourself

4.  Open the file and see everything worked correctly.  

5.  Close the file, and open it using Notepad++

6.  Change the encoding to "UTF-8" (Menu Bar > Encoding > "UTF-8")

7.  Save and Close

8.  Open in Excel again

9.  File > Save & Send > Send Using Email > Send as Attachment > email it to yourself

10.  See that the columns are not recognized correctly.  

11.  Open the new file in notepad++ and see that the commas were replaced with tabs (and that the file has been re-encoded using "UTF-8 w/o BOM")

I tested this using Excel 2010, but I would bet this is prevalent in newer versions as well (I don't have access to them at this time).  Can anyone reproduce this?
