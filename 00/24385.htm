A USB is a physical device, a virtual drive is just a drive in memory only.

It takes the next letter drive you are not using.

(example drive C: - your hard drive, D: - CDROM ,E: is free - then it will mount on E:)

When you reboot the mounted drive is lost (you need to mount it again).

It acts like a USB - you insert the USB it mounts and you remove it it auto dismounts or you

can right-click and eject a USB.

A virtual drive you mount - but you either right-click and unmount or reboot your pc to remove it..

The reason it works for installing windows is that it uses the mounted drive of the window install DVD

to upgrade your system BUT it copies the files it needs after your reboot onto your actual PC's

hard drive in a hidden folder called C:\$Windows.~BT.

Windows 8 or 10 has the ability to mount ISO directly using windows explorer

without the need of third-party software (but lower versions of windows need it)

If you have windows 8 or windows 10 - NO you do not need any third-party app to

mount the ISO.

If you have windows 7 or lower then YES you need a third-party app to mount the ISO

(like daemon tools - http://www.daemon-tools.cc/downloads )

If you have already installed (in window 8 or 10) a third-party app or program to burn ISO then

your just need to right-click the ISO and choose "Open with - Windows explorer".

As long as the “Check for the latest virus & spyware definitions before running a scheduled scan” feature is enabled, a command line scan will also update the signatures. If you specify the ScanType argument without including the identifier, the scan will default to the type of scan specified in the scheduled scan settings. The correct syntax for a Quick Scan is:

 

MpCmdRun.exe -Scan -ScanType 1

 

And for a Full Scan:

 

MpCmdRun.exe -Scan -ScanType 2

 

The command line scan results are logged to the System Event log and also to C:\Documents and Settings\<Account Name>\Local Settings\Temp\MpCmdRun.log (for Windows XP).
