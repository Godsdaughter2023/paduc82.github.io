Sorry, Jimmy, but for once MS Support was right :-)

What your screen shot shows is simply the list of Languages that can be used to tag (i.e., format) the text to indicate what its Language is. That list includes quite a few Languages for which Mac Office does not provide Proofing Tools, such as Hindi, Hebrew, Galician, Basque, Arabic, Chinese, Slovanian, Welsh... etc. Applying one of those tags to the text provides 2 advantages:

  1. It determines which set of Proofing Tools is used to check that specific text &
  2. Text tagged in that way will not be marked incorrect if it isn't "correct" based on the Proofing Tools that are available

Both, however, depend on Proofing Tools being available for that Language, & Mac Office does not include any Greek Proofing Tools. erick chu msft say:

Hi Roy,

About certificates, the answer is no. Generally, when you run the Hybrid configuration wizard (HCW), HCW will automatically get these certificates and let you choose one. You just need to choose the related SSL certificate. In order to display the certificate correctly, you need to make sure the following 4 steps:

  1. The Certificate must be a third party trusted certificate.
  2. The proper names must be on the certificate, such as mail.consoto.com or *.contoso.com.
  3. The SMTP service (the sending and receiving servers) must be assigned to the certificate.
  4. The certificates must have a private key.

Here is a reference:

https://blogs.technet.microsoft.com/exchange/2015/09/04/introducing-the-microsoft-office-365-hybrid-configuration-wizard/

For the PAC file, we recommend you bypass the forward proxy if it is possible. If not, you can add the related IP ranges to your firewall. Please check the information in the link below:

https://docs.microsoft.com/en-us/microsoftteams/office-365-urls-ip-address-ranges

If you have any questions when doing that, please feel free to let us know.

Best Regards,

There is no need to set up a shared mailbox on the Outlook client because it will sync to the client if you grant the Full Access permission to users. So I’d like to confirm some information below to troubleshoot the issue:

To grant the Full Access permission to users:

Login to EAC(Exchange admin center)->recipients->shared->double click the mailbox->mailbox delegation->add users to the Full Access box.

In addition, you can try running the cmdlet below in Exchange Online PowerShell to check if the issue persists:

Add-MailboxPermission -Identity shared mailbox name -User 'user name' -AccessRight FullAccess -InheritanceType All -Automapping $ture

To connect to Exchange Online PowerShell:
