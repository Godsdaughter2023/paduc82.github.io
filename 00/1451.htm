ken sheridan say:

As far as the text boxes are concerned, check that in each case their Locked property is False (No) and their Enabled property is True (Yes).  Its hard to see why they should not be as those are the defaults, but The Locked property being True would explain why you can't enter any values.  Each text box's ControlSource property should be the name of a field in the tblProjects table of course.

The ControlSource property of the combo box should be CustomerID.  The way it works is that its RowSource property returns the CustomerID and the CustomerName values from the tblCustomers table.  By setting the BoundColumn property to 1 the first column, CustomerID is the one which supplies the value to the underlying CustomerID field in the tblProjects table (this field must be returned by the form's RecordSource query of course), but by setting the first dimension of the control's ColumnWidths property to zero you see the second column, CustomerName.  So when you select a name the CustomerID value for that name is the value which goes into the underlying field.  Consequently the value in the column identifies the customer for the project.  This is how a relational

BTW, on a slightly different tack, you do realise that by setting a form's DataEntry property to True it only allows the entry of new records?  The property name is a little confusing and we've found that sometimes people, not unreasonably, think it means that it also allows existing records to be edited.  For that the DataEntry property should be False and the AllowEdits and AllowAdditions properties True.

Again departing slightly from the immediate problem, you could simply use SELECT * FROM tblProjects; as the form's RecordSource query rather than including the individual fields.  That guarantees you are not missing any fields.  You could even just use the tblProjects table itself as the form's RecordSource property, though a query is better; for one thing it allows you to order the rows by one or more fields so they are returned in a logical order.

Sum a range of worksheets using Names (variables) for the worksheet names

=SUM('FMP:July 2015'!A1)

Now, I want to replace the static worksheet names with Names.  I have Names defines as follows:
FirstWS=FMP
LastWS=July 2015

This works: 
=SUM(INDIRECT("'"&FirstWS&"'!"&"A1")) - It returns just the value in 'FMP'!A1
=SUM(INDIRECT("'"&LastWS&"'!"&"A1")) - It returns just the value in 'July 2015'!A1

But when I try to sum the range of worksheets, I get a #Ref!:
=SUM(INDIRECT("'"&FirstWS&":"&LastWS&"'!"&"A1"))

Please help!

a. user say:

Re:  using names in formulas

How about an alternative?  Assign a formula to a name (allWS)...
  =SUM(Sheet1:Sheet4!$A$1)  (replace "Sheet1" and "Sheet4" with actual sheet names)
Using   = allWS   in a cell will then return the sum.

Changing the name on one or both of the sheet tabs changes the sheet names in the named formula.
