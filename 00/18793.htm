Sub ExportALL()
Dim WB, myWB As Workbook

N = N + 1
ReDim v(N)
v(N) = FilesPath
FilesPath = Dir()
If N = 0 Then Exit Sub
Set WB = Workbooks.Open(FolderPath & v(N))
For x = 1 To WB.Worksheets.Count
WB.Worksheets(x).Rows("2:" & r).Copy
LRow = myWS.Cells(Rows.Count, 1).End(xlUp).Row
myWS.Cells(LRow, 1).Offset(1).PasteSpecial xlPasteValues
Application.CutCopyMode = False
Next x
WB.Close False
Loop
On Error GoTo 0
Application.ScreenUpdating = True
End Sub

benishiryo say: hi saw40.  some formulas like LEFT has a optional & default value.  it has 2 arguments; text & [num_chars].  those in square brackets are optional.  its default value is 1, so you can use:
=LEFT(A1)

but LEFT, RIGHT & MID makes it a text & not a number.  so it depends on whether your table_array for the VLOOKUP are numbers or text.  if it's text, then the formula above is sufficient.  if they are numbers, you have to convert them like what Ashish did by multiplying by 1.  few other options:
=VALUE(LEFT(A1))
=LEFT(A1)+0
=--LEFT(A1)

compared to multiplication, +0 & double negatives, the 3rd one is supposedly faster in calculation.
