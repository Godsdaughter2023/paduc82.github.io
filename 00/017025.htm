teylyn say:
Hello, I can reproduce that, but I think it's by design. 

A source data table for testing has dates from 1-Jan-2013 to 25-Dec-2013. The pivot table has the date in the row. Grouping by days with 7 day increments results in a pivot table sorted by date value.


That is how the grouping works out of the box. As soon as you try to sort by the grouped column, though, Excel will interpret the column contents as text strings and sort them by 1, 2, 3, etc, instead of the date values.

So, if you want grouping by week, just do the grouping, then do not attempt any sorting. 

As an alternative, add a column to your source data and calculate the week number with a formula like

=WEEKNUM(A2)

copy down. You may want to adjust the formula to cater for your regional settings and what day constitutes day 1 of a week or what week is week 1 in your regional calendar. 

Then you can add that column to the pivot table and adjust the starting date for the first week manually:


Arrange the pivot table layout to be in tabular form, so the week numbers show on the same row as the week time frames (date). Now you can sort by the week number column (week) and the chronology will stay intact.

mike h.. say:

Hi,

The code I gave you won't create the tables on the new worksheets because you didn't tell us there where tables, try this version. The formatting should copy over but you may have to do some minor manual tweaking.

Sub Copy_Data()
 Dim r As Range, LastRow As Long, ws As Worksheet
 Dim LastRow1 As Long, nm As String
 Dim src As Worksheet
 Set src = Sheets("Sheet1")
 LastRow = src.Cells(Cells.Rows.Count, "F").End(xlUp).Row
 For Each r In src.Range("F2:F" & LastRow)
    
         On Error Resume Next
         Set ws = Sheets(CStr(r.Value))
         On Error GoTo 0
         If ws Is Nothing Then
             Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = CStr(r.Value)
             'Un comment this row to copy header
             src.Rows(1).Copy ActiveSheet.Range("A1")  '<=== added line to copy headers
             LastRow1 = Sheets(CStr(r.Value)).Cells(Cells.Rows.Count, "F").End(xlUp).Row
             src.Rows(r.Row).Copy Sheets(CStr(r.Value)).Cells(LastRow1 + 1, 1)
            
             Range("A1").CurrentRegion.Select
    ActiveSheet.ListObjects.Add(xlSrcRange, Range(Selection.Address), , xlYes).Name = _
        "Table1"
        Set ws = Nothing
         Else
            nm = ws.ListObjects(1).Name
            LastRow1 = Sheets(CStr(r.Value)).Cells(Cells.Rows.Count, "F").End(xlUp).Row
              ws.ListObjects(nm).ListRows.Add 'AlwaysInsert:=True
             src.Rows(r.Row).Copy Sheets(CStr(r.Value)).Cells(LastRow1 + 1, 1)
           
             Set ws = Nothing
         End If
 Next r
 End Sub

rena yu msft say:

Hi Gene,

 

You can drag the select window to select more cells. For a large range of cells, using the named ranges is the best way. However, the Excel for iOS can’t name ranges. But it supports the existing named ranges.

For example, you name the ranges in the Excel for Windows, then you open the workbook in the Excel for iOS. In this way, you can use formula with the existing named ranges.

In the meantime, we suggest you vote this UserVoice: Allow naming of ranges and tables if you hope to add the feature to Excel for iOS. This is the best way to make our products and services better for you and others.

 

Thanks,

Rena
