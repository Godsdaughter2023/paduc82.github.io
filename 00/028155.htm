a. user say:

Assuming you are using either the Jet or the ACE OLE DB provider to open the file, the problem is probably that the Windows registry specifies two lists of extensions, e.g. for ACE you might find them in

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Office\12.0\Access Connectivity Engine\Engines\Text

The relevant keys are "DisabledExtensions" and "Extensions"

AFAICS "DisabledExtensions starts with a "!" which means that the extensions listed are actually the ones which are not disabled, so you need to add "prn" to that list. But you don't actually seem to have to add it to the other list.

But altering the default ACE (or Jet) settings is not such a great idea, so you will probably be better off creating your own key(s) in the registry and specifying your key tree in the connection string in the "Jet OLEDB:Registry Path=" item. If you are not allowed to make the necessary changes in the registry, you'll probably have no option but to rename the .txt file before opening it.

It isn't something I've attempted for a long time so I don't have the full details to hand, but I hope that gives you some starting points.
