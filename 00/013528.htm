It would not be in templates but in C:\Users...\Roaming\Microsoft\Word\Startup
 
The file cited in the error message is the "owner file" for an add-in placed in Word's (or the Office) Startup folder by ABBYY FineReader 6 (for an explanation of owner files, see "Description of how Word creates temporary files" at http://support.microsoft.com/?kbid=211632)

Ordinarily this temporary file would be deleted when the Finereader6.sprint.dot file was closed, but I'm guessing that this add-in template is no longer in the Startup folder. Perhaps you uninstalled FineReader 6 while Word was open, and some vestiges of it were left.

In any case, you can delete this temporary file from the Startup folder and try again. If you no longer get an error message, then the problem is solved. If you get a different error message, post back with more information.

bob jones aka: cybertaz say:

If, say, you try to make every letter 'e' in to bold, you would Find: e and Replace: ^& and click Cmd-B for bold, right?

Well, no :-) There are several ways to accomplish what you describe wanting to do, but the steps as you've related them won't work in any version of the program [regardless of the ongoing discussion in this thread]. One main reason is that in the dialog window neither Command+B nor most keyboard shortcuts [other than cut/copy/paste] will work.

The simplest method does not involve Replace at all*. In the Adv. F&R dialog;

  1. Type the letter e in the Find what field
  2. Check the Highlight all items found in: box
  3. Confirm that the above refers to Main Document
  4. Click the Find All button
  5. Critical Step: Either close the dialog window or click the document title bar to shift focus back to the document
  6. Now use Command+B or click the Bold tool on the Ribbon

[*] However, you must use Adv. F&R, it cannot be done using the Search Pane of the Sidebar without moving from one found instance to the next.
