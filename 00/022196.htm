I am using Access 2007 (SP3) in Windows XP Professional (SP3) on a Dell Optiplex 960.  Everything is fully up-to-date via Windows Update.

Do you have a solution to fix this problem?

stuart bratesman say:

Problem solved.  It turns out that a Visual BASIC function referenced by one of the fields in the detail section of the report was generating an "#error" value on page 29 of the 30-page report, which cause the export to terminate, but without displaying a pop-up error message on the screen.  It just looked like it was done, but it never finished.

Now that I've fixed the error (the Visual BASIC function included a DLOOKUP() function that was looking for a table that didn't exist), everything is working fine.

bob jones aka: cybertaz say:
As Daniel pointed out, Microsoft Word is available for the Mac as a part of Office:mac suites but not as a standalone program. However, it is comparable but not identical in all respects to its Windows siblings... but then again they differ from one another as well :-) No matter what you choose there will be a learning curve involved.

There are at least a dozen word processing programs for Mac. Several are free, some are inexpensive [<$50 US], others cost more. Some are available as standalone programs, others only as components of a Suite that includes other apps, so cost can be deceptive.

IMO, nobody can tell you which is right for you unless they're completely objective, know exactly what your current/anticipated requirements are & have evaluated them all for those purposes. Most people have a tendency to recommend what they are comfortable with.

I'd suggest you do some investigating on the web. Narrow down the choices based on independent reviews as well as the developers' claims. Take advantage of free trial offers to see what you like best then select accordingly... and don't overlook the availability of support in the event you should need it :-)
