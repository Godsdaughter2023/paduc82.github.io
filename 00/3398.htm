an Excel sheet has over a million rows and you can't remove them. In Page Layout view it may look like you have 87,000 pages, but that is not so. You don't have to print the empty cells.

Check with the Print Preview that the correct content is being printed before you commit it to the printer.

ken sheridan say: You will need one or more additional columns representing attributes which differentiate each customer, and which do not accept Nulls (Required property = True in Access).  In most situations I would use address  data for this as it would be unusual to have two or more customers of the same name at the same address.  You can then include the CustomerName, and however many of the address columns are necessary to guarantee distinctness, in a single Unique index.

For instance, you might want to create a unique index on the CustomerName, AddressLine1 and CityID columns .  This is done in table design view by selecting Indexes on the Design ribbon.  In the dialogue which opens enter a suitable name for the index in the first empty row of the Index name column, DistinctCustomer, say.  In the Field Name column of the same row enter CustomerName, and with this row still selected, in the Index Properties below, select 'Yes' as the Unique property.  Then enter AddressLine1 in the Field Name column in the row immediately beneath CustomerID.  Don't enter anything in the Index name column for this row.  Then enter CityID in the Field Name column in the row immediately beneath AddressLine1.  Again, don't enter anything in the Index name column for this row.  Close the dialogue and save the table design.

For data entry, e.g. in a form bound to an Orders table or similar you would bind a combo box to the foreign key CustomerID column and in the combo box's drop down list return the customer names, along with their address data.  You'll find an example in NotInList.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the combo box to select a contact in the opening form is set up in this way by having the following as its RowSource property:

SELECT Contacts.ContactID, [FirstName] & " " & [LastName],
ConcatValues(", ",AddressLine1,AddressLine2,City,Region,PostCode)
FROM Regions INNER JOIN (Cities INNER JOIN Contacts
ON Cities.CityID = Contacts.CityID) ON Regions.RegionID = Cities.RegionID
ORDER BY Contacts.LastName, Contacts.FirstName;

This calls the following function to concatenate the address data into a single string expression:

Public Function ConcatValues(strSeparator As String, ParamArray arrVals())
 
    ' Pass this function a character or characters
    ' to be used as the separator followed by the values to be combined
    ' For example: strFullName =
    ' ConcatValues(" ",FirstName, MiddleName,LastName)
    
    Dim X As Integer, strLine As String
    
    For X = 0 To UBound(arrVals)
        If Not IsNull(arrVals(X)) And Trim(arrVals(X)) <> "" Then
          strLine = strLine & strSeparator & arrVals(X)
        End If
    Next
      
    ' remove leading separator character(s)
    ConcatValues = Mid(strLine, Len(strSeparator) + 1)
 
End Function

jay freedman say:

To do this manually, first turn on line numbering for the entire document (Layout > Line Numbers > Continuous). Then click once in each "Section Title" paragraph and click Layout > Line Numbers > Suppress for Current Paragraph.

If you've applied one style to all of the "Section Title" paragraphs and a different style to all the "line" paragraphs, you can use a macro to set up the line numbering all at once. Use the following macro (see http://www.gmayor.com/installing_macro.htm if needed), and change the style name in quotes to the name of the style you used for the "Section Title" paragraphs.

Sub SetLineNumbering()
   Dim rg As Range
   Set rg = ActiveDocument.Range
   With rg.PageSetup.LineNumbering
      .Active = True
      .StartingNumber = 1
      .RestartMode = wdRestartContinuous
   End With
   
   With rg.Find
      .Format = True
      .Style = ActiveDocument.Styles("Heading 2")  ' <== change this!!
      .Wrap = wdFindStop
      While .Execute
         rg.ParagraphFormat.NoLineNumber = -1
         rg.Collapse wdCollapseEnd
      Wend
   End With
End Sub
