ken sheridan say: Rather than storing the user name on demand, e.g. by means of a 'remember my user name' checkbox in the form, one approach would be to enter the current user's user name automatically by having a table which, in addition to the Access user name, stores the user's Windows login name, e.g.

Users
....UserName
....SystemLoginName

To get the system login name add the following module to the database:

' module basGetUser
Option Compare Database
Option Explicit

Declare Function GetUserName Lib "advapi32.dll" Alias "GetUserNameA" (ByVal _
   lpBuffer As String, nSize As Long) As Long

Public Function GetUser() As String

   Dim strBuffer As String
   Dim lngSize As Long, lngRetVal As Long
   
   lngSize = 199
   strBuffer = String$(200, 0)
   
   lngRetVal = GetUserName(strBuffer, lngSize)
   
   GetUser = Left$(strBuffer, lngSize - 1)

End Function
' module basGetUser ends

In your login form you can then call the GetUser function and use its return value to look up the current user's Access user name with the following as the form's Load event procedure

Private Sub Form_Load()

    Dim strCriteria As String
    
    strCriteria = "SystemLoginName = """ & GetUser() & """"
    Me.txtUserName = DLookup("UserName", "Users", strCriteria)
    
End Sub

where txtUserName is the Name property of the text box in the form.
