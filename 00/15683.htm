If they were created with Word 2003, then they are not Works documents. They will have a .doc file extension, and you should be able to open them in Word 2016 without issue.     The error 0x80042104 is a message that may mean Outlook is telling the server to leave messages in the server and the server is telling Outlook it can't do that. You did not include the full text of the error, but I'll bet it said something like "Your e-mail server does not have the features required to allow messages to be left on the server”. AT&T uses Yahoo for email and this shows up with Yahoo from time to time.

You'll see a box there labeled "Schedule an automatic send/receive very". Set the minutes value to 10 or higher, then click OK.

    Hello Paul,

    • I would like to inform you that there might be few possibilities that a user is not able to login through mobile as below-
    • if you use Yammer Single Sign-On with ADFS, there’s several mobile Yammer Apps that doesn’t actually use this for authentication
    • Even if the user is disabled from the office 365 portal or yammer than also through yammer mobile app will not able to login
    • For the security purpose the temp password works only for one-time login in mobile app.
    • When you create a temporary password it generates a cookie that’s stored on the device, and doesn’t actually hit your ADFS server for authentication.
    • After you log out and login again with the same password it will redirects to corporate ADFS authentication page.

     

    • Please follow below steps to generate temporary password:-
      1. Login into yammer home network
      2. Click on three ellipses on top Left > Click on Apps
      3. Select iOS app.
      4. There will be temporary password with email id.
      5. Please try to login with this temporary password in mobile app
