Hi Adriannelson,

Based on your description, there are two methods:

1. Deploy Lync Hybrid. The method can be a little complex and SSO is required. In a hybrid deployment, the DNS records are pointed to Lync on-premises. When a Lync account is moved to Lync Online using Lync Server Management Shell, the contacts and meetings will not be affected. There will be less down time. After all the users are moved to Office 365, you can refer the following steps to decommission Lync on-premises:
a. Make sure all the users are moved to Lync Online.
b. Point SRV records to Lync Online.
c. Decommission Lync server by preferring to the article: https://technet.microsoft.com/en-us/library/gg195815(v=ocs.14).aspx

2. Without deploying Lync hybrid, you need to disable the Lync accounts from Lync on-promises and enable them in Lync Online directly. In this situation, there will be some service down time. Here are the detailed steps for your reference:
a. Disable the Lync on-premises account. For example, you can run the command “Disable-csuser” using the Lync management shell. Normally, this command will also remove all Lync related attributes from local AD.
b. Run Dirsync, sync the user accounts to Office 365 or you can create accounts directly in cloud.
c. Assign the Lync Online licenses to synced user accounts.
d. Change the Lync related DNS records such as SRV records and CNAME records to Office 365.

If anything is unclear, please feel free to post back.
