Hi JFresh,
 
Malicious Software Removal Tool and Windows Updates are not responsible to turn firewall on again and again.
In this scenario we can set the Windows firewall to manual and its status as stop.
Here is how:
1. Hold the Windows key and type R, enter "services.msc" (without the quotes) and press Enter

3. In the dropdown box that says "Manual", set it to "Stop" in the service status and then click on "Apply"

    Meghann,

    1  The first reply is incorrect but it is always wise to have a spare [i.e. unused] password-protected local Admin user account in case your day-to-day user account ever gets corrupted - you'll then be able to log in using the spare Admin in order to rescue the situation.  I think you ought to make such an additional account now before doing anything else -

    • Start button,
    • Settings,
    • Accounts,
    • Family & other users,
    • [Other users section],
    • Add someone else to this PC,
    • I don't have this person's sign-in information,
    • Add a user without a Microsoft account,
    • Write down the username and password that you are about to use [somewhere readily accessible yet secure from loss],
    • Enter a username, password & password hint,
    • Back in the list of other accounts you can now see in Family & other users, [Other users section], 
    • Click on the new account,
    • Click on Change account type,
    • Select the Administrator account type,
    • OK.
    • [Just in case you have heard of Password reset disks {for local user accounts only}, note that they achieve nothing more than writing the password down achieves]

    2  With that done,

    2.1  If your existing account is a local one then you can change its name in Start menu, Windows system, Control panel, User accounts, Change your account name.  Use the improved procedure in my post starting To change the username of a local user account without leaving behind remnants of the OldName.

    2.2  If, as I suspect, your existing account is an MSAccount-linked one then you can only change its name online within the MSAccount concerned. 

    • Prepare in advance for any difficulties, download the MSAccount troubleshooter from How to Troubleshoot Microsoft Account Issues - TechNet [don't worry about their reference to Windows 8, it is the same troubleshooter for both Windows 8 & 10]
    • You might need to have several attempts at logging into your user account after changing your name [the login must be done whilst still connected to the internet so your computer can query the MSAccount servers] after changing its name for it to work.
    • [but my only experiment in doing this was very early in Windows 10 life so perhaps they have improved things now]
    • If it continues to refuse to let you log in then try running the downloaded troubleshooter.

    Congratulations & the best of luck,

    Error message: "The User Profile Service failed the logon. User profile cannot be loaded”, when logging on to Windows 7 or Windows Vista

    http://support.microsoft.com/kb/947215

    Note: When you are trying to rename a key  take the ownership first and then rename it.

    Follow the steps:

    a. Right click on the file or directory, and then select Properties on the right click menu.
    b. Click on Security tab.
    c. Click on Advanced button at the bottom.

    d. In “Advanced Security Settings” dialog window, click on Owner tab.
    e. Here you will be able to see current owner .To take ownership of the object, click on the Edit button. If UAC prompts for administrator’s password or permission to continue, enter the correct password or press Continue button.

    f. Additional “Advanced Security Settings” dialog will appear. Highlight the user name (for example, Administrators) in the Change owner to box that you want to assign as the owner for the object. Click OK to make the change.
    g. Back in original parent level “Advanced Security Settings” window, you will see the existing owner of the file or folder has changed to the user you just selected.
    h. Click OK button to exit this window.
    i. Click OK again to exit completely from the Properties window.
    j. The ownership is now belonged to user or user account that been selected. To assign necessary permissions to the user too, repeat step 1 to 3 to open the object’s Properties window again.
    k. In object’s Properties window, clicks on Edit button to change permissions. If UAC prompts for administrator’s password or permission to continue, enter the correct password or press Continue button.

    l. Highlight the Administrators or the user who wants the permissions on the object be changed in the “Group or user names” box.

    If the user ID or group that you want to manage the permissions for the object doesn’t exist, click on Add button, and type in the user name or group name desired into the Enter object names to select box, and finish off by clicking on OK.
    m. In the Permissions for Administrators box below (or any other user name or group name you chose), click on “Full Control” under the “Allow” column to assign full access rights control permissions to Administrators group.
    n. Click “OK” twice when done.

    Hope this helps.
