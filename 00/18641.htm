chip pearson say:

Yes, but what if the date in B1 is formatted differently than the dates in A1:A5? If B1 contains the date 8/18/2012 formatted as 18-Aug-2012, Find should find the date even with the different format.  If you use xlLookIn:=xlFormulas rather than xlValues, the formats of the dates don't matter.

chip pearson say:

For dates, you should use LookIn:=xlFormulas, not xlValues. You get the error you cite because the Find method returns Nothing if the date is not found, and your MsgBox attempts to use Dest, which is Nothing.

I need to merge/purge two lists but keep only most recent data from both

Hi Very Smart Excel People,

Goal:

Merge two similar lists of names and email addresses (one "NEW" and one "OLD") then purge only the duplicate names & emails from the OLD list.

The resulting list would include :

all the unique names and associated email addresses AND the duplicate names and email addresses from "new" list.

d-k-s say:

Hi Erica

By "identifier" i mean a tag that's used to identify which data is from the old spreadsheet and which is from the new.

I did come up with a relatively simple solution using existing Excel features

The key was when I learned that Excel deletes duplicates from the "bottom up" all I needed to do was to organize the data so that the old duplicate data was below the new duplicate data. 

Here's what I did:

1. Tag all the rows in the Old spreadsheet with the number 1 in an unused column - (col. "M" as an example)

2.  Tag all the rows in the NEW spreadsheet with the number 0 in the unused column ("M")

3.  "CUSTOM SORT" (ORDER A to Z) the combined spreadsheets using the fields you will select in REMOVING DUPLICATES,  and column M

This will bring all your duplicates together and order them so that the old data is below the new data. 

4.  Run REMOVE DUPLICATES  - this will remove only the old duplicate data from the OLD spreadsheet but keep ALL the data from the NEW spreadsheet.

Easy Peasy!
