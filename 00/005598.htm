Where do these items (words) get stored? Where do these items (words) get stored?

 

 

Thanks in advance for any replies. Thanks in advance for any replies.

 

 

jay freedman say:

AutoCorrect entries that are saved as "Formatted" are stored in the Normal.dotm template and are available only in Word.

Entries that are saved as "Plain Text" are stored in C:\Users\<username>\AppData\Roaming\Microsoft\Office\MSOxxxx,ACL, where the xxxx is a 4-digit Locale ID -- for example, 1033 for English (US). The entry is placed in the ACL file for the proofing language applied to the entry's content. These entries are available to all Office programs that use AutoCorrect.

john w. vinson say:

It sounds like you need another table or two. Typically one would have a Students table with one record for each student (and no course or tutoring information); a Courses table of all the courses offered; an Enrollment table linking them, with foreign key fields for the ID Number and CourseCode; and a Tutoring or Sessions table, with fields for the ID Number, CourseCode, SessionDate and perhaps other fields (such as the ID of the tutor or comments about the tutoring session itself). So if John Doe had five sessions in PSY 200, there would be five records in the Sessions table with his ID and that course code, each with its own date.

You could then create a report based on a Query joining Sessions to Students and to Courses; make it a totals query grouping by student name (hint: don't use the reserved word Name as the Name of a field, Access can get confused!) and by CourseCode, with appropriate criteria on SessionDate.

ken sheridan say: As it happens there is a demo file StudentLog.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

You might have to copy the text of the link into your browser's address bar (not the link location).  For some reason it doesn't always seem to work as a hyperlink.

The file is really intended to show how to build correlated subforms, but happens to be based on a student registration and attendance database, so it might prove a suitable model in your case.  To return the results you are looking for the following query would work in my file.  The start date in this case is when a student first attends a session of a course.  The DateValue function is used BTW because the CourseDate column has full date/time values, not just the date, so this returns only the date part:

SELECT [FirstName] & " " & [LastName] AS FullName, Students.StudentID, Courses.CourseName,
DATEVALUE(MIN(CourseSessions.CourseDate)) AS StartDate, COUNT(*) AS Attendances
FROM (Courses INNER JOIN CourseSessions
ON Courses.CourseID = CourseSessions.CourseID)
INNER JOIN ((Students INNER JOIN CourseRegistrations
ON Students.StudentID = CourseRegistrations.StudentID)
INNER JOIN CourseAttendances
ON (CourseRegistrations.StudentID = CourseAttendances.StudentID)
AND (CourseRegistrations.CourseID = CourseAttendances.CourseID))
ON (CourseSessions.CourseDate = CourseAttendances.CourseDate)
AND (CourseSessions.CourseID = CourseAttendances.CourseID)
GROUP BY Students.StudentID, [FirstName] & " " & [LastName], Courses.CourseName;

If you wanted to show the start date as the first session of a course, even if the student in question did not attend that session, then you'd need to use a subquery to return the date:

SELECT [FirstName] & " " & [LastName] AS FullName, Students.StudentID, Courses.CourseName,
    (SELECT DATEVALUE(MIN(CourseDate))
     FROM CourseSessions
     WHERE CourseSessions.CourseID = Courses.CourseID) AS StartDate,
COUNT(*) AS Attendances
FROM (Courses INNER JOIN CourseSessions
ON Courses.CourseID = CourseSessions.CourseID)
INNER JOIN ((Students INNER JOIN CourseRegistrations
ON Students.StudentID = CourseRegistrations.StudentID)
INNER JOIN CourseAttendances
ON (CourseRegistrations.StudentID = CourseAttendances.StudentID)
AND (CourseRegistrations.CourseID = CourseAttendances.CourseID))
ON (CourseSessions.CourseDate = CourseAttendances.CourseDate)
AND (CourseSessions.CourseID = CourseAttendances.CourseID)
GROUP BY Students.StudentID, [FirstName] & " " & [LastName],
Courses.CourseID, Courses.CourseName;

You could easily add parameters to either of the above to restrict the results to attendances within a specified date range for instance, e.g. in the case of the first query above:

PARAMETERS [Enter start date:] DATETIME, [Enter end date:] DATETIME;
SELECT [FirstName] & " " & [LastName] AS FullName, Students.StudentID, Courses.CourseName,
DATEVALUE(MIN(CourseSessions.CourseDate)) AS StartDate, COUNT(*) AS Attendances
FROM (Courses INNER JOIN CourseSessions
ON Courses.CourseID = CourseSessions.CourseID)
INNER JOIN ((Students INNER JOIN CourseRegistrations
ON Students.StudentID = CourseRegistrations.StudentID)
INNER JOIN CourseAttendances
ON (CourseRegistrations.StudentID = CourseAttendances.StudentID)
AND (CourseRegistrations.CourseID = CourseAttendances.CourseID))
ON (CourseSessions.CourseDate = CourseAttendances.CourseDate)
AND (CourseSessions.CourseID = CourseAttendances.CourseID)
WHERE CourseSessions.CourseDate >= [Enter start date:]
AND CourseSessions.CourseDate < [Enter end date:]+1
GROUP BY Students.StudentID, [FirstName] & " " & [LastName], Courses.CourseName;

This would prompt the user for the start and end dates of the range at runtime when the query or a report or form based on it is opened.

Is there an easy way to type on both sides of an inserted vertical - Microsoft Community

jcmu99 say:

Is there an easy way to type on both sides of an inserted vertical line?

Hello, trying to type on the left side of an inserted vertical line and then once I have filled up the entire left side I would like to type on the right side.  there an easy way to do that? I know that I can just keep typing across the line and allow it to spill over to the right side but I don't wanna do that. If you look at my screenshot the area in red is where I try to click and type but I cant--the only way I cant type on that right side is by starting on the left side. Is there any type of formatting or alignment I need to do to fix that?

jay freedman say:

There are, I think, at least three ways to do what you're asking. Which one is best for you depends on whether you want to align text on the right with related text on the left. Only the last one involves adding a vertical line separately from the other steps -- if you already have the line and you're using one of the other methods, first delete the line.

If you just want to fill the left side with text first and have any additional text continue at the top on the right:

  • On the Page Layout ribbon, click the Columns button and click More Columns at the bottom.
  • In the Columns dialog, click the picture for Two and check the box for "Line between".
  • Click OK.

If you want to be able to make paragraphs on the right align with related paragraphs on the left:

  • In the Insert ribbon, click the Table button and select the first two boxes in the top row. That will insert a one-row, two-column table.
  • If you don't want borders around all of the table cells, but just the vertical line between the columns:
    • Select the whole table by clicking in the margin to the left of the row.
    • On the Table Tools | Design ribbon, click the down arrow on the Borders button and click No Border. If the table seems to disappear, go to the Table Tools | Layout ribbon and click the View Gridlines button to show dashed (non-printable) outlines of the cells.
    • Click in the left cell. Then click the same Borders arrow and click Right Border.
  • Type text in the left cell. Then you can tab or click into the right cell to add text there.
  • Each time you want to align the paragraphs in the two columns, put the cursor at the end of the last cell in the right column and press the Tab key to start a new table row.

If you just need to put occasional text on the right:

  • Set the right margin of the page at the position where you want the left column to end.
  • Insert your vertical line at that right margin.
  • When you want some text to the right of the line, go to the Insert ribbon and click the Text Box button, then click Draw Text Box in the bottom of the menu.
  • With the special cursor (like a big plus sign), drag a rectangle where you want the text. Drag the sides or corners of the rectangle to the size you want. Type the text inside the box.
  • If you don't want a border around the text, click the Shape Outline button on the Drawing Tools | Format ribbon, and click No Outline in the menu.

For more information, see http://wordfaqs.ssbarnhill.com/UsingColumns.htm.
