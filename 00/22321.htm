david a matson say: Greetinsg JR.Watchet!   Welcome to the Windows Mobile Total Access forums!

It sounds like your Access Point or an upstream Router's DHCP service is not doing it's job by assigning IP addresses. 

1.) DHCP is the default connection method for any device that is not already setup with a static IP itself.   Based on your description of an "unusual" IP address, it sounds like your device decided on a non-routeable self-assigned IP Address of 169.*.*.*, indicating that your device assigned it's own IP address when it could not communicate with your LAN's DHCP server.

2.)  If you want a Static IP, then you need to set your device up as such.  You can use your Profile program to set this up (I presume), or you can go to Start > Settings > Connections Tab > Connections Icon, then select Configure Wireless Networks option (your WiFi should be enabled to see this button), then Network Adapters (tab).  Select the appropriate Driver for your Wireless connection, then input the static IP you wish to use.

3.) The "adapters" that you see in that list are drivers for your Hardware specific to your PDA, mixed in with some "generic" drivers that may allow your hardware to emulate one of those pieces of equipment.

I hope this helps clarify the situation!
