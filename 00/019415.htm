Check if value exists in other table: FAST ..... Check if value - Microsoft Community

paul domaskis say:

Check if value exists in other table: FAST ..... Check if value DOESN'T exists: SLOW

Why does it take mere seconds to check if a field value exists within another table (sysTree) but minutes to check if a field does *not* exist in another table?

-- This runs in seconds
--
UPDATE myTable new
SET [In sys tree ?] = "YES"
WHERE new.Node IN ( SELECT Node FROM sysTree )

-- This runs in minutes
--
UPDATE myTable new
SET [In sys tree ?] = "NO"
WHERE new.Node NOT IN ( SELECT Node FROM sysTree )

ken sheridan say: The NOT IN predicate requires a complete table scan.  It also will fail in the event of a NULL being encountered at the column position in any row.  I'd recommend using the EXISTS and NOT EXISTS predicates respectively.  In most cases it is significantly faster, though the relative sizes of the two sets is a factor in this.

UPDATE myTable
SET [In sys tree ?] = "YES"
WHERE EXISTS
    (SELECT *
     FROM sysTree
     WHERE sysTree.Node = myTable.Node);

UPDATE myTable
SET [In sys tree ?] = "NO"
WHERE NOT EXISTS
    (SELECT *
     FROM sysTree
     WHERE sysTree.Node = myTable.Node);

Be sure that the Node columns in each table are indexed.

a. user say:

Hi Rene,


Thank you for posting your question on Microsoft Community.

I suggest you to run DISM tool and check if it helps.

Please follow the below steps.

 

Run the DISM tool

1. Swipe in from the right edge of the screen, and then tap Search.
(If you're using a mouse, point to the lower-right corner of the screen, move the mouse pointer up, and then click Search.)

2. Enter command prompt in the search box.

3. In the list of results, press and hold or right-click Command Prompt, and then tap or click Run as administrator. You might be asked for an admin password or to confirm your choice.

4. In the Administrator: Command Prompt window, type the following commands. Press the Enter key after each command:

dism.exe /online /cleanup-image /scanhealth

dism.exe /online /cleanup-image /restorehealth

5. To close the Administrator: Command prompt window, type exit, and then press Enter.

6. Run Windows Update again.

 

You may also refer to the link “System file check (SFC) Scan and Repair System Files” Created by ZigZag3143 (MS - MVP), for more information.
