There are several sheets that have a link so it is difficult to know what the name of the sheet they would return to is, just that they should return to the previous sheet, whatever sheet that was. Any help would be greatly appreciated. Thanks.

jeeped say:

... I have a workbook where each sheet has a hyperlink to additional info. I'm looking for a macro that will return the user to the previous sheet (the one they were on when the clicked the link.) ...


I'll assume that the additional info you are referring to is another worksheet within the same workbook.

Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module. Paste the following into the new pane titled something like Book1 - Module1 (Code).

Public sLastSheet As Variant
Public sThisSheet As Variant

Public Sub mcrReturnToSheet()
    Sheets(sLastSheet).Activate
End Sub

Next, find ThisWorkbook in the upper-left Project - VBAProject pane and right-click it then select View Code. Paste the following into the new window titled something like Book1 - ThisWorkbook (Code).

Private Sub Workbook_Open()
    sLastSheet = ActiveSheet.Name
    sThisSheet = ActiveSheet.Name
End Sub

Private Sub Workbook_SheetActivate(ByVal Sh As Object)
    sLastSheet = sThisSheet
    sThisSheet = Sh.Name
End Sub

Tap Alt+Q to return to your worksheet(s). Tap Alt+F8 to open the Macros dialog and use Options to assign a Shortcut key: to the mcrReturnToSheet macro.
Save your workbook as a macro-enabled workbook and close it. Subsequently opening the workbook will allow the use of your shortcut key combination to return to the previously active sheet.

Can this be done in Access 2010?

hansv mvp say:

Open the table in design view.

If you don't see the Property Sheet, press F4 to display it.

Enter the following in the Validation Rule property:

[TextField] Is Null Or [DateField] Is Not Null

where TextField is the name of the text field and DateField is the name of - you guessed it - the date field.

In the Validation Message property, enter an appropriate warning message to be displayed if the user enters a value in the text field but not in the date field.

If necessary, the warning will be displayed when the user tries to leave the record, or to save it explicitly.

john w. vinson say:

If you're entering data into the Table directly (you shouldn't be, that's what Forms are for) you can open the table in design view and put in a Validation Rule such as:

[textfield] IS NULL OR ([textfield] IS NOT NULL AND [datefield] IS NOT NULL)

In the Validation Text field put some polite, user friendly* meaningful message such as

"Date field must be entered if Text field contains data"

Better would be to do your data entry with a Form; you could then use the AfterUpdate event of the text field to fill in today's date in the date field (if that's appropriate), or the Form's BeforeUpdate event to check for data in the two fields and require a date.

rohn007 say:
If as you say you have Office 365 Home (not Personal or Office 2016 Home and Student), then NO you do not have to deactivate it on the old machine.

365 Home allows you to install Office on up to 5 PC and/or Mac desktop/laptop type computers at the same time (as well as 5 keyboardless tablets AND 5 smartphones). As the link Edgar provided says, that total of 5 can also be shared with other email accounts, but that does not appear to be relevant to your question.

On the new machine, log in to the MyAccount site and click on Install. It will be active on both machines at the same time. After the old computer dies you can "deactivate" it from the MyAccount site (logged in on any machine) to free up that activation count.

davem121 say:
Hi Mark, you do not need to deactivate Office 365 Home - you can install that on up to 5 PC's at the same time,

On the new PC, gGo to http://www.office.com and if you're not already signed in, select Sign in.

If the system is dying, you can if you wish deactivate that installation, so that your number of installations is still one!

On the same page, you can simply select Deactivate Install
