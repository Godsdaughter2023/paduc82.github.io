Word Template Form Help

Thanks
jay freedman say:
When you're in the Save As dialog in Word 2010, you can save a template in any of three formats by selecting from the "Save as type" dropdown:

  • Word Template (*.dotx) is one of two newer formats, introduced with Word 2007. This one can contain macros and userrforms.
  • Word 97-2003 Template (*.dot) is the older format used in Word 97 through 2003. It can contain macros and userforms.


If you saved the template as either *.dot or *.dotm, your userform can be displayed by the Document_New() macro -- but only if you use the File > New command to make a document based on the template. If you use the File > Open command to open the template itself, then -- as obvious as this may seem -- you aren't making a new document, so Document_New() won't run. Instead, Document_Open will run.


One more bit: In Word 2010, there are also three different document formats: *.doc (the old format), *.docx (the no-macros format), and *.docm (the macro-enabled format). If you put your macro and userform into a document -- either .doc or .docm -- then each time you open the document, the Document_Open macro will run; but the Document_New macro will never run.

Changing default printing of document markup - Microsoft Community

vklrc say:

Changing default printing of document markup

I want Word to display markup on the screen when I'm viewing or editing a document. But I don't want it to print the markup when I print.

What I need is a way to turn the print option "Print Markup" off by default.

I'm guessing there is no way to do this except through an AutoOpen macro.

Thanks!

jay freedman say:

You can use a macro to display the old (Word 2007) FilePrint dialog, which has a separate dropdown for selecting what to print, and force that dropdown to show "Document" instead of "Document showing markup".

Install this macro in your Normal.dotm template and assign a Quick Access Toolbar button to run it. You can also assign a keyboard shortcut, even reassigning Ctrl+P to it.

Sub PrintNoMarkup()
    ' use the old FilePrint dialog
    ' force "Print What" box to "Document"
    SendKeys "%wd%a"
    Dialogs(wdDialogFilePrint).Show
End Sub

This sort of subterfuge is necessary because Microsoft has never provided any way for macros to control what settings are used in the Backstage (the pages displayed by the File tab). Fortunately, most of the old dialogs are still in the program, just hidden from the casual user.
