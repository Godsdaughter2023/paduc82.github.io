john w. vinson say:

Just use a Format (in the table field or a form or report textbox) and leave the Input Mask off altogether. For dates, they can be more hassle than benefit. Access will automatically translate an entry of 5/5 to #05/05/2011 00:00:00# which can be displayed any way you like, and will also reject 5/50/11 as an invalid date. (However, entering 5/50 will give May 1, 1950 in the table, so you may still want some date range validation).

 

scottgem say:
A Date/Time value is stored in the table as a double precision number where the Integer portion is the number of days since 12/30/1899 and the decimal portion a fraction of a day. So how you enter the number and how you display it is completely customizable within the object (form, query, report, table) you are in. You can have a report show June 5, 2011 and a form 6/5/11.

You can do search on Access Input masks for a complete explanation of how they work.
