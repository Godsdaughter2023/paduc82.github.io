hansv mvp say:

You can't convert an .MDE database to a .MDB database - when the .MDE was created, all source code of the VBA modules in the database was removed; there is no way to reconstruct it.

 

You can copy the tables from the .MDE database to a new (or existing) .MDB or .ACCDB database.

 

You'd have to get hold of the .MDB file that was originally used to create the .MDE database if you need to edit the design of the reports, forms and code modules.

daniel pineault say:

I think you do not understand the purpose of an mde/accde.

A developer creates/designs/develops an mdb/accdb and then generates an locked down copy (mde/accde) for distribution to users.  They purposely create the mde/accde because it prevents people from stealing their work and/or making unauthorized changes.  If and when changes are required, the developer simply makes those changes to the master mdb/accdb and then generates a new mde/accde for their users.  That is simply how Access development is done.

One never modifies an mde/accde.

If you loose the master mdb/accdb then you have a choice, start the entire database over or call upon the services of a few companies who have figured out how to reverse engineer the compilation process (they will, rightfully, require proof of ownership).

Microsoft will not publically divulge how to uncompile an mde/accde it would entirely defeat the entire purpose of creating such locked down files and would enable anyone to steal another developer's work (lawsuits would abound!).

As a developer, it is up to you to always retain a copy of the master mdb/accdb to be able to do further development.  ALWAYS!

If you think this doesn't make sense, feel free to create a suggestion on access.uservoice.com and send your feedback through the Access feedback option in the GUI.  but realistically, as explained above, there is a reason things are setup the way they are, there is simply no reason to ever need to edit an mde/accde when all you need to do is modify the master mdb/accdb reissue a new copy of the mde/accde.
