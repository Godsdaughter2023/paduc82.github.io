Hi BJL1960,

For Office 365 send on behalf of, the user has the permission of "send on behalf of" about another user and the user changes the sending address (From) while sending the email.     barb bowman say:

    I saw this one myself. Here's what fixed it:

    1. You will need to use your password to log in obviously.
    2. Open File Explorer and go to C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft.
    3. Right click the Ngc folder and then select Properties.
    4. Go to the Security tab and click on Advanced button.
    5. Locate the Owner section at the top and then click Change link.
    6. In Enter the object name to select enter Administrators, if you’re using administrator account, or your user name (what you use to log in to Windows) and click Check Names and then OK.
    7. Check Replace owner on subcontainers and objects and click Apply and OK.
    8. Exit all dialog windows.
    9.  Right click the Ngc folder and rename to Ngc.old (acknowledge any UAC prompts)
    10. Restart your Surface 3

    After logging in with your password, a new Ngc folder with proper permissions should have been created and you should be able to set up a PIN.
