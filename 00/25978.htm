E63 Alarm tone without Call tone - Microsoft Community

mrb314 say:

E63 Alarm tone without Call tone

I don't know if this is elswhere in the forum, but I couldn't find anything and had to work it out for myself.

 

If you set the Profile to the default Silent, the phone doesn't ring of course, but unlike my previous phone neither does the Alarm. There is no setting in Personalise to set the Alarm Tone, and Nokia's Help files are pretty useless.

 

So if you want to be woken by the Alarm but not by any calls do the following:

- record a few seconds of silence on the mike and save it (eg to "Silence")

- use Tools>Profiles>Silent>Personalise>Ringing_tone and set it to your new recording (eg "Silence")

- set  Tools>Profiles>Silent>Personalise>Ringing_type to Ascending (or probably anything except Silent)

- set  Tools>Profiles>Silent>Personalise>Ringing_volume to anything above zero ( which is one "blob")

(- leave all the other settings as defaults)

 

Phone calls will ring with "Silence" - so you don't hear them, but the Alarm will use it's normal tone.

 

Hope this is useful to someone.

a. user say:

Hi stuck,

 

Thank you for posting your concern here in Microsoft Community Forum. It's my pleasure on helping you to re-instate your alias account on your Microsoft account.

 

Please be informed that this warning usually occurs if the email came from a third party domain. If the sender’s domain does not have SPF record configured/set up in the domain’s registrar, Outlook.com detects the email as potential fraud since it was forwarded from 3rd party IPs.

 

What is SPF Record?

 

Sender Policy Framework (SPF) is an email validation system designed to prevent email spam by detecting email spoofing, a common vulnerability, by verifying sender IP addresses. It is a security standard that has been implemented to stop forgery of sender address.

SPF allows administrators to specify which hosts are allowed to send mail from a given domain by creating a specific SPF record (or TXT record) in the Domain Name System (DNS).

 

To resolve this, the sender’s domain administrator should add the SPF record in the domain registrar.

 

As workaround, you can add the 3rd party account to your safe senders list in Outlook.com address. To do so, please refer to the steps below:

 

1. Sign in to your account. Go to inbox.
2. Click the Gear icon located at the upper-right side of the page.
3. Select More mail settings.
4. Under Preventing junk email, click Safe and blocked senders.
5. Click Safe senders.
6. Enter the email address or domain on the box labeled Sender or domain to mark as safe: and click Add to list >>.

 

Please get back to us if issue still persists.

 

Thank you!
