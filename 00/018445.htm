On top of that, is there a way to, in the same cell, ALSO accept input already in hh:mm format and not apply any formatting?

Any insight anyone may have to lend here would be greatly appreciated. Thanks!

haytham amairah say:

Hi Dave,

Excel cannot read a whole number as time and convert it to time, even if the cell format is in time format because the origin of the time in Excel is a decimal number between 0 and 1.

You need to helper cell contained a formula to take this number and convert it to the time format.

Let's suppose that the number is in cell A1, put this formula in cell B1:

=TIME(0,A1,0)

And then customize the time format of the cell B1 in this format (hh:mm).

Basically, we must stay away from names that start with <letter><number>, <letter><letter><number> and <letter><letter><letter><number>, at least up to XFD [1].

The Excel parser tries to interpret them as cell references.

I would agree that it is an unfortunate design flaw.  But so be it.
