rick rothstein, mvp - excel say:

The macro below does not work for me:

Private Sub Worksheet_Change(ByVal Target As Range)

Selection.Font.ColorIndex = 8

End Sub

The above macro changes the font color in the NEXT cell I click in after I change a value in a certain cell.  Can someone send me a macro that will change the font color of the cell that I change the value of?

The reason you are changing the NEXT cell is because the Change event does not fire until a value has been changed. That means either you clicked into another cell, you Tabbed out of the cell or you hit the Enter key... assuming you have the option turned on to move the selection when the Enter key is hit (this is a default action unless you change it), then all three of those actions moves the selection (active cell) to some place other than cell you changed. That means when you use Selection (or if you had used ActiveCell) in your code, it is pointing to the currently selected (active) cell (the one you move to in order to complete the Change action). As Hans pointed out, you would use the Target argument that VB provides as part of this change event... it is a direct reference to the cell that was changed. So, as Hans pointed out, your line of code should be...

Target.Font.ColorIndex = 8

jay freedman say:
The SkyDrive sharing worked perfectly, no problems there.

The table pasted as a picture shows up in VBA as a Shape object instead of a field, with a .Type value of msoLinkedOLEObject (numeric value = 10). From that you can extract the information that would be in an equivalent LINK field:
  • The Shape object's .LinkFormat.SourceFullName contains the path and filename of the worksheet that was copied from.
  • The Shape object's .OLEFormat.ClassType contains the ClassName string of the field code, in this case "Excel.Sheet.8".
  • The Shape object's .OLEFormat.Label contains the PlaceReference string of the field code, such as "Sheet1!R4C4:R13C11".


The following code should be able to change the worksheet path in each linked picture in a document. The result will still be a picture that doesn't display a field code when you press Alt+F9.


Sub ShapesLink()
Dim shp As Shape, newshp As Shape
Dim src As String, labl As String, typ As String, anchr As Range
Dim cod As String
Dim ht As Single, wd As Single, lft As Single
Dim wrp As WdWrapType

For Each shp In ActiveDocument.Shapes
    If shp.Type = msoLinkedOLEObject Then
        With shp
            src = Replace(.LinkFormat.SourceFullName, "\", "\\")
            labl = .OLEFormat.Label
            typ = .OLEFormat.ClassType
            Set anchr = .Anchor
            wrp = .WrapFormat.Type
            ht = .Height
            wd = .Width
            lft = .Left
        End With
       

        ' Here, put in code to change the value of the string src to use the new path


        cod = typ & " """ & src & """ " & labl & " \a \p"
        ActiveDocument.Fields.Add Range:=anchr, _
            Type:=wdFieldLink, Text:=cod, PreserveFormatting:=False
        Set newshp = anchr.Paragraphs(1).Range.InlineShapes(1).ConvertToShape
        With newshp
            .WrapFormat.Type = wrp
            .Height = ht
            .Width = wd
            .Left = lft
        End With
    End If
Next shp
End Sub

ken sheridan say: A simple solution would be not to use a navigation form, but a form in single form view, with subforms for the various categories of condition, each linked on PatientID or similar, and each on a separate page of a tab control.  In the parent form you can include a navigational combo box to select the patient and move the form to that patient.  The subforms would then automatically show that patient's data.

You'll find an example of a form/subforms set up in this way in FormsDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file take a look at the 'Tabbed Forms' option.  In my case the subforms are in continuous forms view, but they could equally well be in single form view if preferred.  The first page of the tab control includes data from the parent form, so in your case the main patient identification data could be shown above the tab control, so that it is permanently visible, with subsidiary patient data on the first page of the tab control directly, not in a subform.

One caveat; using a patient's name only to navigate to their record is unsatisfactory as personal names can legitimately be duplicated.  I was once present at a clinic when two patients arrived within minutes of each other, both female, both the exactly the same first and last names and both with the same date of birth.  The navigational combo box would be better, therefore, if its list included other data by which patients of the same name can reliably be differentiated, e.g. by including address data.  You'll find an example in NotInList.zip in the same OneDrive folder.

In this little demo file the combo box to select a contact in the opening form is set up in this way by having the following as its RowSource property:

SELECT Contacts.ContactID, [FirstName] & " " & [LastName],
ConcatValues(", ",AddressLine1,AddressLine2,City,Region,PostCode)
FROM Regions INNER JOIN (Cities INNER JOIN Contacts
ON Cities.CityID = Contacts.CityID) ON Regions.RegionID = Cities.RegionID
ORDER BY Contacts.LastName, Contacts.FirstName;

This calls the following function to concatenate the address data into a single string expression:

Public Function ConcatValues(strSeparator As String, ParamArray arrVals())
 
    ' Pass this function a character or characters
    ' to be used as the separator followed by the values to be combined
    ' For example: strFullName =
    ' ConcatValues(" ",FirstName, MiddleName,LastName)
    
    Dim X As Integer, strLine As String
    
    For X = 0 To Ubound(arrVals)
        If Not IsNull(arrVals(X)) And Trim(arrVals(X)) <> "" Then
          strLine = strLine & strSeparator & arrVals(X)
        End If
    Next
      
    ' remove leading separator character(s)
    ConcatValues = Mid(strLine, Len(strSeparator) + 1)
 
End Function
