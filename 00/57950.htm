The board is shown as compatible with Windows XP on the ASUS site (they have the chipset drivers available for it), the latest nForce drivers may have a bug affecting Windows XP, newer drivers sometimes have bugs in which case it is best to stay with the old drivers.  New drivers rarely provide new functionality or increased performance to existing hardware, they usually correct bugs or are rolled out for the new generation of hardware while keeping backward compatibility.  Often times the bug fix are pretty specific and if you don't have a problem newer drivers may not make any difference.  For example, if a specific cad software program has problems with certain aspect of video rendering  new drivers may be released to address the issue, if you don't use the cad software there is nothing to gain by installing the new drivers and you may just end up with an unexpected glitch with your particular hardware.

If you got the 0x7b error when installing XP that is because the installation did not have the drive controller drivers for the array, you must supply these drivers to the Windows XP installation on a floppy diskette via the F6 method at the very beginning of the installation or slipstream them to the installation CD if you cannot use the F6 method.

My two cents on installing Windows (or any other operating system) on RAID-0 is that in 99% of cases it's just plainly a bad idea that is twice as prone to failure than a regular installation on a single disk.  RAID-0 is ideal for disk intensive applications like video editing where large amounts of data must be written to disk, writing 100MB of data across 2 disks in 50MB stripes is twice as fast as writing 100MB to one disk, use an array for this but keep the operating system on a disk of its own.  For more speed on the operating system disk if at all possible I would consider a SCSI disk or a 10,000 or 15,000 RPM drive like Raptor or Cheetah instead of a RAID-0 array.
