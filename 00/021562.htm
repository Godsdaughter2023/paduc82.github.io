How to: Pre-test if a Clean Install of Windows 10 will Activate - Microsoft Community

don cuthbert say:

How to: Pre-test if a Clean Install of Windows 10 will Activate before wiping your hard disk

How to:

Pre-test if a Clean Install of Windows 10 will Activate before wiping your hard disk

Note: A full clean-install of Windows does not backup or keep any programs or data.

Method 1

  1. Download the matching Edition of the Windows 10 ISO to your activated genuine Windows 10 upgrade.  There are 5 possible Editions per language, so be sure to create the right Edition of the Windows 10 ISO that your device is licensed for.  You can clean install either 32-bit or 64-bit Windows 10 if your device supports both.
  2. Create some un-partitioned space to test the clean install.  Shrink a partition using Disk Management.  20GB is enough for Windows 10 x64 for testing purposes.
  3. Create a bootable ISO on a USB flash drive with the Windows 10 ISO.  The Windows USB/DVD Download Tool can create this in NTFS format for Legacy BIOS mode devices and Rufus can be used to create a FAT32 formatted flash drive for UEFI BIOS mode devices.
  4. Boot your computer with the USB flash drive and choose Custom to install to the 20GB un-partitioned space.  This will create a multi-boot setup and the new clean install will be the default at first.  The default OS can be changed using msconfig later.
  5. Do not enter any product key during or after the clean install.
  6. Windows 10 should auto-activate after connecting to the Internet.
  7. If this install of Windows 10 auto-activates, you have proven that disk as designed.

Method 2

  1. If you want to do this without changing your partitions and to use the least drive space possible for the test, it is a bit more complex to do.  First create a VHDX file for this in Disk Management.  Select your disk partition for drive C: or any other internal drive volume you wish to use.  Under Action menu, create a dynamically expanding VHDX file of 40GB maximum size (e.g. D:\CLEAN.vhdx).  The actual initial file size will only be 4MB, but the file size will expand as needed to a maximum of 40GB.


     
  2. Follow the steps above to boot with the right ISO for Windows 10.
  3. When Windows install begins press Shift-F10 to get a command prompt.
  4. Type diskpart at the Command prompt.
  5. In diskpart, list volume to see your disk volumes.  The drive letters may be different than expected in this mode.  Find the drive letter for the partition where you created the VHDX file.  If that partition has no drive letter, select volume "x" (where "x" is the volume number needing a letter) and then type assign to give it a drive letter.  List volume to see the drive letter assigned.
  6. select vdisk file=D:\CLEAN.vhdx
  7. attach vdisk
  8. Exit diskpart (type exit)
  9. Exit the Command prompt (type exit)
  10. Now in Windows Setup choose Custom and look for the 40GB drive that represents your VHDX file and clean-install Windows 10 to that drive.  Refresh the list of partitions if needed.  It will only expand the VHDX file as needed to hold the test clean-install of Windows 10.
  11. Do not enter any product key. Skip these screens.
  12. This will become the default boot OS in a multi-boot setup.  It is called "native boot" as it uses all of your device's hardware.  Only the hard disk is virtual.  You can later change this default OS back to your main Windows install using msconfig.
  13. See if Windows 10 auto-activates after install.

________________________________________________________

Either of the above methods will pre-test your device's ability to activate a clean install of Windows 10 without first deleting all of your drive partitions.

a. user say:

Unable to open Visio documents from Sharepoint using Visio 2013 MSI and Microsoft Office 2016 Click-To-Run

Issue

When launching Visio documents from Sharepoint using Internet Explorer, Visio will launch with a major version of 16 instead of 15 and prompt for activation even if a valid volume license has been applied.

 

Steps to Reproduce

  1. Visio 2013 (x86) must be installed via MSI. Office 2016 (x86) must be installed via Click-To-Run.
  2. Open a SharePoint document library using Internet Explorer
  3. Click on a Visio document
  4. Visio will open and prompt for activation
  5. The version displayed by checking File -> Account -> About Visio will show a major version of 16. The rest of the version number will match current version of Office 2016.
  6. Closing Visio, downloading the Visio document and then opening from Explorer will allow Visio to open properly and major version shown under File -> Account -> About Visio will be 15.

 

More Details

The issue appears to be with the SharePointWorkspace.Application COM interface and usage of an Office 2016 DLL called Interceptor.dll as following the below steps resolves the issue:

  1. Open HKEY_CLASSES_ROOT\SharePointWorkspace.Application\CLSID
  2. Collect the GUID set as default in this key
  3. Open HKEY_CLASSES_ROOT\ Wow6432Node\CLSID\[Your GUID here]\InprocServer32
  4. Change the value of (default)
    1. from C:\Program Files (x86)\Microsoft Office\root\Office16\Interceptor.dll
    2. to C:\Program Files (x86)\Microsoft Office\root\Office16\MSOSYNC.exe
    3. Reattempt to open your Visio document directly from the SharePoint library in Internet Explorer and it should open right up.

*Note: I am not sure if changing the file path to MSOSYNC.exe resolved the issue or just breaking the connection to Interceptor.dll

 

Version Details

Office 2016 – C2R – x86 – 16.0.6366.2036

SharePoint Online

Internet Explorer 11 – 11.0.9600.18124

bernd p. say:

There are at least 3 ways:

#1

If you have access to Onedrive AND the notebook is synched AND your desktop client is working:

Use the Change Location feature of the notebook's properties page to download and store the notebook now locally.

#2

If you have access to Onedrive but no desktop client available:

Use the Web App to download the notebook's folder as a ZIP-file.

#3

If you have no (more) access to Onedrive AND the notebook is open in your desktop client:

Use the Export feature of Onenote to store your notebook locally in a .onepkg file (technically it's also a ZIP file but opens directly in Onenote after double click).

hansv mvp say:

Don't select ranges, but check whether the result of Find is Nothing:

 

Sub build_analysisList(forAnalysis As String)
    Dim PEws As Worksheet
    Dim sRow As Long, eRow As Long
    Dim rng As Range
    Application.DisplayAlerts = False
    Set PEws = Worksheets("Port Excel Lines")
    Set rng = PEws.Range("BE:BE").Find(What:=forAnalysis, _
        LookIn:=xlFormulas, LookAt:=xlPart, _
        SearchDirection:=xlNext, MatchCase:=False, _
        SearchFormat:=False)
    If rng Is Nothing Then
        Exit Sub
    End If
    sRow = findStartRow("BE", forAnalysis)
    eRow = findEndRow(sRow, "BE", forAnalysis)
    PEws.Range("B" & sRow & ":BD" & eRow).Copy
    Worksheets("Analysis List").Cells(2, 1).PasteSpecial xlValue
    Application.DisplayAlerts = True
End Sub
