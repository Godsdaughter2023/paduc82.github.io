dianeoforegon say: First, update to 14.2.2 then start over in a new Identity.

How to move your data to a new Identity

Verify on the server that there are no duplicates.

Clean up your inbox and Sent folder. Archive older messages to a subfolder at the root level by year. While Outlook for Windows seems to handle large Inboxes, Outlook for Mac works best with smaller Inboxes.


The Inbox syncs more frequently than any other and should be kept as clean as possible. (this is the number one cause of problems with sync). Do NOT make subfolders under the Inbox.

Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.

Access 2013 Calculate Sick Days over time period - Microsoft Community

tparky say:

Access 2013 Calculate Sick Days over time period

Hi Guys,

I have a question that I'm hoping for some answers and or suggestions too. We have an Access Database that currently holds data on staff Absences. We wont a query or a set of queries that will produce the following results for us so that we can better Manage our Staff Absence.

Background on the Tables

Staff Table: StaffID, First Name, Last Name

Staff Absence: ID, StaffID, SicknessStartDate, SicknessEndDate, DoctorsNote

RetrunToWork: AbsenceID, MeetingHeld, LineManager, Notes.

We want queries that will produce the following;

Total Number of days Absent over a varied period 4, 6, 12 and 18 months

Sickness without Doctors Notes

Sickness long than 10 Working Days

I currently have one Query that gets the total number for all staff absence but need it to do it for Individual staff Members it then needs to produce a report that can be emailed to the Line Manager of the Member of staff.

I don't have a clue about access so I am looking for help with this or some pointers.

scottgem say:

The easiest first:

SELECT * FROM [Staff Absence]

WHERE DoctorsNote Is Null;

Second easiest:

SELECT AbsenceID, StaffID, SicknessStartDate, SicknessEndDate, DateDiff("d",SicknessStartDate,SicknessEndDate) As Duration

FROM [Staff Absence]

WHERE Duration > 10;

If you need Duration to be only work days, search on NetWorkDays function and you should find functions to calculate just work days.

The last one is the most complex. The simplest way is to test just the start date:

SELECT AbsenceID, StaffID, SicknessStartDate, SicknessEndDate

WHERE SicknessStartDate BETWEEN earliestdate AND latestdate;

You can supply the earliest and latest date in a variety of ways.

dhookom say:

I'm not sure Scott's second query will work since [Duration] is an alias:

SELECT AbsenceID, StaffID, SicknessStartDate, SicknessEndDate,

DateDiff("d",SicknessStartDate,SicknessEndDate) As Duration

FROM [Staff Absence]

WHERE Duration > 10;

I would recommend:

SELECT AbsenceID, StaffID, SicknessStartDate, SicknessEndDate,

DateDiff("d",SicknessStartDate,SicknessEndDate) As Duration

FROM [Staff Absence]

WHERE DateDiff("d",SicknessStartDate,SicknessEndDate) > 10;

How do you expect to handle the number of days over a period if an absence begins before the start of the period and ends during the period?  This typically calls for a table of dates [tblDates] with a single field [TheDate] and storing every possible date ever required by your application:

[TheDate]

1/1/2000

1/2/2000

1/3/2000

...

7/23/2014

7/24/2014

...

12/28/2050

12/29/2050

12/30/2050

12/31/2050

You can then more easily query the absence days for a specific date range.

mimmo57 (d. laurenza) say:

hI,

for Total Number of days Absent over a varied period 4, 6, 12 and 18 months

I think that you need two queryes .

Query1

SELECT StaffAbsence.StaffId, IIf([SicknessStartDate]>DateAdd("m",-4,Date()),[SicknessStartDate],Null) AS SD4, IIf([SicknessEndDate]>DateAdd("m",-4,Date()),[SicknessEndDate],Null) AS ED4, IIf([SicknessStartDate]>DateAdd("m",-6,Date()),[SicknessStartDate],Null) AS SD6, IIf([SicknessEndDate]>DateAdd("m",-6,Date()),[SicknessEndDate],Null) AS ED6, IIf([SicknessStartDate]>DateAdd("m",-12,Date()),[SicknessStartDate],Null) AS SD12, IIf([SicknessEndDate]>DateAdd("m",-12,Date()),[SicknessEndDate],Null) AS ED12, IIf([SicknessStartDate]>DateAdd("m",-18,Date()),[SicknessStartDate],Null) AS SD18, IIf([SicknessEndDate]>DateAdd("m",-18,Date()),[SicknessEndDate],Null) AS ED18
FROM StaffAbsence;

Query2

SELECT Query1.StaffId, Sum(DateDiff("d",[SD4],[ED4])+1) AS G4, Sum(DateDiff("d",[SD6],[ED6])+1) AS G6, Sum(DateDiff("d",[SD12],[ED12])+1) AS G12, Sum(DateDiff("d",[SD18],[ED18])+1) AS G18
FROM Query1
GROUP BY Query1.StaffId;

joel mcbeth say: I have had the same experience that existing rules only run on new e-mail, and not existing.

As a workaround though, I have found that you can click in the search box and select advanced search.  While it doesn't give you the same options (contains, begins with, etc) it lets you generally search for things by keyword, subject, to, and from.

You can then click the select all button on the results, then click the "select everything" link if it is more than one page.  Although I would be careful to make sure that all of the search results are what you want to perform the bulk action on as I have no idea how it uses the words you search to find the mails.

Once you have them all selected you can use the buttons at the top to delete, move, archive, categories, etc. like you would in your inbox.  You can do most if not all of the things you can with rules using this method.
