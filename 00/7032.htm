tasos_k say: Hi,
try this.

ALT+F11 to open VB editor.

Insert > module and paste the code below on the right

Close VB editor.


Sub ExcelToText()

Dim myPath As String, myTxt As String

Dim r, c As Long

r = ActiveSheet.Cells.Find(What:="*", _

SearchOrder:=xlByRows, SearchDirection:=xlPrevious).Row

c = ActiveSheet.Cells.Find(What:="*", _

SearchOrder:=xlByColumns, SearchDirection:=xlPrevious).Column

myPath = "C:\"

myTxt = "TextFile.txt"

myFile = myPath & myTxt

For i = 1 To r

For j = 1 To c

Cells(i, c + 1) = Cells(i, c + 1) & " " & Cells(i, j).Value

Next j

Next i

Open myFile For Output As #1

For i = 1 To r

Print #1, ActiveSheet.Cells(i, c + 1).Value

Next i

Close #1

ActiveSheet.Columns(c + 1).ClearContents

Npad = Shell("C:\WINDOWS\notepad.exe " & myFile, 1)

End Sub


To run the macro, press ALT+F8, 

select <ExcelToText> from the list and click the run button.


note: your text file is saved on C  as TextFile.txt

You might also look into File-->Save As and the .CSV file format.  That is actually a text file, which if you will change the filename created from .csv to .txt, will be opened with notepad.  Most spreadsheet and database applications contain the native ability to import a .csv file without even changing the name to .txt. 

But if you have some type of custom application needing to read the information, the 'save as .csv, change name to .txt' operations usually make writing the code to import it at the other end easy since the rules for how the data is contained within the format are known.

ken sheridan say:

The short answer is  that you can't.  An INSERT INTO statement can only insert rows into a single table.  If you need to insert rows into multiple related (not linked, that's something entirely different) tables you would need to execute multiple INSERT INTO statements, which you can do in VBA by building and executing each INSERT INTO statement successively, ensuring that a row is first inserted into each referenced table before a row is inserted into a referencing table.  You might for instance need to insert a new row into a Cities table where there are no referenced rows in a Regions table and a Countries table, in which case a row would first be inserted into Countries, then into Regions and finally into Cities, so as not to violate referential integrity.

 

However, it is unclear from your question which tables you wish to insert rows into on the basis of the SQL statement you have cited, or whether what you are aiming at would be a legitimate operation.  It would help if could explain in detail what you are trying to do in terms of the real world entity types represented by the tables in question.

2. If you are using the new Outlook.com user interface, please click on the Cog icon before your name in the upper part of the page. Click on “Options” then select “More Mail Settings”. 3. From the “Hotmail Options” screen under the category “Preventing junk email”, click “Filters and reporting”.

4. Under "Choose a junk email filter" category, click on Standard.

5. Click Save.

 

You can also create a rule that will automatically put your Gmail senders to your inbox. Here are the steps:

 

1. Go to your Microsoft Hotmail or Outlook.com Inbox page.

2. If you are using the new Outlook.com user interface, please click on the Cog icon before your name in the upper part of the page. Click on “Options” then select “More Mail Settings”. If you are using the old Hotmail user interface, please click Options which is located on the upper right corner of the window and beside the question mark icon and select “More options”.

3. From the Hotmail Options screen under the category Customizing Hotmail, click Rules for sorting new messages.

4. Click New. On Step 1, select Sender's address, contains on the dropdown menu for the first and 2nd box respectively.
