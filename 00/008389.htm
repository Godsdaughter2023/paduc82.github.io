john korchok say:

There is no VBA command to do what you want. The simplest way to create one big paragraph from several and retain some formatting would be to do a find and replace on the paragraph marks, replacing them with spaces.

For all style-based attributes, all character and paragraph attributes will be defined by the paragraph mark at the end of the concatenated text. For character-based attributes that have been applied with local formatting (select text, then change setting in Font panel), the local formatting will be retained after the paragraph marks between paragraphs are removed.

If it's really important to save character formatting, you could use VBA to read the style characteristics of each paragraph, bookmark the text in each paragraph without including the paragraph mark, remove the paragraph marks, select each bookmark and reapply the font formatting that each section originally had.

paulcordts say: There are of course lots of ways to do this.  

When you paste the data, the entire range is then selected, so we can take advantage of that.

This should work for continuous data
Selection.End(xlDown).Offset(2, 0).Select

For data with empty rows use this (I would just use this in case the data ends up having empty rows)
Cells(selection.row+selection.rows.count+1,1)

The general format is Cells(R,C) so change the column number as needed

You would then have to go and copy the next range of data to do your paste.

If you get creative you don't even have to select the next data range to copy.

On a blank sheet put data with empty spaces in A1:a10 and B1:B10 and step through this.

Sub test()

Range("A1:A10").Copy
Range("C1").Select
ActiveSheet.Paste

Cells(Selection.Row + Selection.Rows.Count + 1, 3).Select
Range("B1:B10").Copy
ActiveSheet.Paste
End Sub

In this case Selection.Row =1 (the first row of the selection) and Selection.Rows.Count=10 (the total number of rows in the selection).
Since 10+1=11 you only need to add one to this total to leave one blank row between data sets.

Notice that I used both styles of cell reference Range("C1").select and Cells(R,C).select in the same piece of code.  This is not an issue.  I prefer the A1 style when using a workbook, but when you need to use formulas the RC style is the only way to go.

Since you are switching sheet to get data, you can select the data on the source sheets.  When you switch back to the destination sheet, the proper cell will still be selected,

paulcordts say: Assuming your data is in column A and is continuous (no spaces) you can use this to select the cell in column A 2 rows below the end of the data. Then paste your data.

Sheets("WorksheetC").Range("A1").End(xlDown).offset(2,0).select 

If your data has blanks you want to work from the bottom up

Sheets("WorksheetC").Range("A10000").End(xlup).offset(2,0).select 

Note:  If you have more than 10,000 lines of data use a larger number
