    Sub PointLabel()
      Dim mySrs As Series
      Dim iPts As Long
      If ActiveChart Is Nothing Then
        MsgBox "Select a chart and try again.", vbExclamation, "No Chart Selected"
      Else
        For Each mySrs In ActiveChart.SeriesCollection
          With mySrs
            For iPts = 1 To .Points.count Step 5

                mySrs.Points(iPts).ApplyDataLabels _
                    ShowSeriesName:=False, _
                    ShowCategoryName:=False, ShowValue:=True, _

        Next
      End If
    End Sub

Hello,

you can use a free tool created by Rob Bovey, called the XY Chart Labeler. With this tool you can assign a range of cells to be the labels for chart series, instead of the Excel defaults.

Using a formula, you can have a text show up in every nth cell and then use that range with the XY Chart Labeler to display as the series label.

If the full chart labels are in column A, starting in cell A1, then you can use this formula to create a range with only every fifth label in another column:

=IF(MOD(ROW(),5)=0,A1,"")

...  If the date (01/01/2012) is a default date ...


You have a couple of choices here and it depends upon whether the 1/1/2012 date is the default forever or the default for this year. In either event, you cannot just plug 1/1/2012 into the formula. You will have to construct a true date value using DATE() or DATEVALUE() like this,
=DATE(2012,1,1)
... or,
=DATE(YEAR(TODAY()),1,1)
... or,
=DATEVALUE("1/1/2012")
... or,
=DATEVALUE("1/1/"&YEAR(TODAY()))

So your formula becomes something like,
=DATEDIF(DATE(2012,1,1),EDATE(TODAY(),C1*12),"y")&" Y- "&DATEDIFDATE(2012,1,1),TODAY(),"ym")&" M - "&DATEDIF(DATE(2012,1,1) ,TODAY(),"md")&" D"

=DATEDIF(A2,EDATE(TODAY(),C2*12),"y")&" Y- "&DATEDIF(A2,TODAY(),"ym")&" M - "&DATEDIF(A2,TODAY(),"md")&" D"

I don't think you will find it necessary to adjust all three of the TODAY() values since you are only adjusting by an integer worth of years but some testing may be worthwhile around Feb 29 of a leap year.

john - project say:
Tyler,
Normally the assignments shown in the Task Usage view are listed in order of resource ID and the user can't manually rearrange them. However let's say you have certain "priorities" for resources. For example, let's say your resources on the Resource Sheet are listed as follows:
#1 Joe worker
#2 BIll worker
#3 Jim 2nd lead
#4 Bob lead

But, you would like to see all tasking listed with the lead guy first, the 2nd lead after that and the two workers after that. So on the Task Usage view you want to see the assignments lists in this order:
Bob
Jim
Joe
Bill

One way to do that is as follows. On the Resource Sheet, use the Resource Group field as your listing priority. Give your lead resources a "1", your second leads a "2", and other workers as whatever order you want. Then on the Task Usage view, use the sort function to first sort by ID (Task ID) and then by Resource Group. The end result will be resource assignments listed in the order of priority.

Hope this helps.

John
