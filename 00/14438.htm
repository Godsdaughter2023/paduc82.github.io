jeeped say:
I may be talking out of the top of my hat but here is my interpretation.

=350*2/12 is the native Excel formula syntax and the formula remains in its typed form and uses Excel's 15 digit floating point precision.

Using +350*2/12 requires that Excel use built-in translation capabilities to accommodate older (and foreign) spreadsheet input styles (... Lotus or Quattro perhaps?). During translation, repeating decimals are resolved to a finite number within the 15 digit floating point that Excel has as an upper limit of precision to avoid floating point errors.

On a related point, this may be an easy way around floating point errors in native Excel formulas.

Understanding Floating Point Precision, aka “Why does Excel Give Me Seemingly Wrong Answers?”

In any case, it would be wrong for Excel to change 2/12 into 0.166666666666667, much less 0.1666666667.  Contrary to what Jeeped says, it does not "avoid floating point errors"; instead it creates them.

If we enter =350*2/12 into A1 and =350*0.166666666666667 (not +350*2/12) into A2, then =(A1=A2) returns FALSE.  In fact, if we format A1 and A2 as Number with 13 decimal places, we see that A1 is

58.3333333333333, but A2 is 58.3333333333334.


So rounding 2/12 to 0.166666666666667 makes a significant difference [1].

As you might imagine, it would be even worse if Excel truly rounded it to 0.1666666667 [sic], as Vitali wrote.  Then =350*0.1666666667 displays 58.3333333450000.

[1] It is possible that +350*2/12 only looks like =350*0.166666666666667, but the initial value is truly 350*2/12.  Excel does that in some unusual circumstances.  So =(A1=A2) might return TRUE initially.  However, if the Formula Bar displays =350*0.166666666666667, the value will change to that result the next time the cell is edited (e.g. press F2, then Enter) and the next time a full recalculation is performed (e.g. ctrl+alt+F9).  That is why I instructed to enter =350*0.166666666666667 in A2 in order to demonstrate the problem.

stefan blom say:

In Word you can't set font size as a percentage.

To make the font sizes of headings match Normal, you will have to modify the styles one by one, make sure that the base style is Normal and then set the size equal to that of Normal. Later, if you change Normal, the headings will follow.

Alternatively, you can run a macro such as the following:

Sub MatchNormalStyleSize()
Dim i As Long
For i = 1 To 9
With ActiveDocument.Styles("Heading " & CStr(i))
.BaseStyle = wdStyleNormal
.Font.Size = ActiveDocument.Styles(wdStyleNormal).Font.Size
End With

Next i
End Sub
