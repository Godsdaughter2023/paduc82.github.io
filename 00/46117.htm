raduban say: Hi Brian,

Thanks for the reply!

I also thought it was a virus and spent the last few hours trying to find it. I used ProcessMonitor and looked at each file and registry access before program startup.

It turned out to be the video card driver. I have an Nvidia card and had played with the settings a while back. I think that aurora.dll was automatically inserted by the video driver to enable some special effects (Nvidia already uses a shim DLL file at program startup: nvinit.dll). However, something broke and therefore aurora.dll could not be found.

Long story short, if anyone has the same problem they should reinstall the Nvidia drivers and perform a clean installation (clear all previous settings).

from being locked on (not desirable if not using Nvidia 3D vision).
