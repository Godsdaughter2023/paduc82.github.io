a. user say:

You didn't say what type of data source it is, but MailMerge can't use more than 255 columns in Excel despite the fact that since 2007, Excel has been able to use far more columns. The limit exists no matter which method you use to connect (DDE, ODBC, OLEDB).

As far as I know, the 255 limit applies to all data sources that Word accesses via the Jet/ACE provider (e.g. Access, Excel, text files that Word accesses via OLE DB or ODBC)

However, Word should work with a delimited text file with more than 255 columns as long as delimiter characters in the data are correctly "wrapped up" - e.g. if your data contains a comma, the field has to be surrounded by quotes. So you may be able to export your Excel data as a .csv file and use that.

ken sheridan say: Firstly create a table named Counters with a single column named Counter of integer number data type, defined as the primary key, and filled with values from 1 to a number which exceeds the maximum number of a product you will ever  produce in a day.  The table can be extended later if necessary of course.

You also should have a products table with columns which include ProductCode, which should be the primary, and ProductName .  Even though you might have only two products this table is necessary to a well designed database as it ensures that all data are stored as explicit values at column positions in rows in tables in accordance with the Information Rule (Codd's Rule #1), and allows the integrity of the database to be protected by the enforcement of referential integrity.

Create a form named frmDailyProduction or similar, in continuous forms view, based on the following query:

PARAMETERS Forms!frmDailyProduction!txtProductionDate DATETIME;
SELECT *
FROM DailyProduction
WHERE ProductionDate = Forms!frmDailyProduction!txtProductionDate
ORDER BY ProductCode;

Add the following controls to the form header;

1.  A text box named txtProductionDate, formatted as Short Date.  In the text box's AfterUpdate event procedure put:

    Me.Requery

This will requery the form to show if rows have already been entered for the date entered.

2.  A combo box set up as follows:

Name:    cboProduct

RowSource:     SELECT ProductCode, ProductName FROM Products ORDER BY ProductName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

5.  A text box named txtQuantity.

4  A button with the following code in its Click event procedure:

Const MESSAGE_TEXT = "A date, product and quantity must be entered."
Dim strSQL As String

If Not IsNull(Me.txtProductionDate) _
    And Not IsNull(Me.cboProduct) _
    And Not IsNull(Me.txtQuantity) Then

    strSQL = "INSERT INTO DailyProduction (ProductionDate,ProductCode) " & _
                  "SELECT #" & Format(Me.txtProductionDate, "yyyy-mm-dd") & "#," & _
                  "ProductCode " & _
                  "FROM Counters, Products " & _
                  "WHERE Counter <= " & Me.Quantity " _
                  "AND ProductCode = " & Me.cboProduct
    
    CurrentDb.Execute strSQL, dbFailOnError
    Me.Requery
Else
    MsgBox MESSAGE_TEXT, vbExclamation, "Invalid operation"
End If

In the form enter the date, for which the date picker will pop up, select a product, enter a quantity, and click the button.  The number of rows corresponding to the quantity value will be inserted ready for you to enter data into controls bound to other columns in the table.  I assume that there are other columns as otherwise entering multiple identical rows would serve no real purpose.  Repeat for each product.

Your question is a bit vague but you can check with the WEEKDAY function on which weekday the holiday falls:

=WEEKDAY(<New Year's Day>, 2) >5 says if it is a Saturday or Sunday. In that case use WORKDAY to find then next working day. =WORKDAY(<date>,1,<additional holidays>)

zorvek say:

In the US, New Years Day is observed on the closest weekday--if it falls on a Saturday, then Friday is the observed holiday. This formula finds the closest weekday given a date:

=WORKDAY(A1,CHOOSE(WEEKDAY(A1),1,0,0,0,0,0,-1))

Can I use Office Home and Busines 2010 on 2 PCs with 2 users? - Microsoft Community

david ian hammond say:

Can I use Office Home and Busines 2010 on 2 PCs with 2 users?

Can I as a business use Office Home and Busines 2010 on 2 pcs with 1 disc with 2 users?
dan at it associates say:

Office 2010 H&B is licensed to the license holder for use one or two licensed devices.

 

The OEM and Product Key Card licenses allows the license holder the use of Ofice 2010 H&B on one device (aka computer), while the full retail and direct purchase and download from Microsoft licenses (both referred to as "FPP" licenses) allow the use of Office 2010 H&B on two devices.

 

The two device license permits the license holder to install and use the license on his primary device (computer) and allows a second use on the license holder's portable device (aka laptop).  A strict interpretation of the licensing terms would allow more than one user to use Office on the primary device but would not allow anyone other than the license holder to use Office on the portable device.

 

A usage scenario that has a business entity with two workers, each using one computer, installing and using one FPP license for Office 2010 H&B on both computers would be outside of the licensing terms.  For such a scenario, the most economical way to use Office 201 H&B in compliance with the licensing terms would be to purchase one Product Key Card license for each of the two computers.  If you foresee deploying new computers in the next 2-3 years, or foresee a need for one or both of the employees to need Office 201 H&B on a portable device in addition to the devices they are now using, then it would make sense to buy an FPP license for each person as the FPP license will allow the transfer of the license to new computers (the PKC license does not), and the FPP license will allow a second activation of Office in addition to the first on each person's portable device if such device is added in the future (the PKC license does not permit an additional activation).
