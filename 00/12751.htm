dave peterson say:

This:
Private Sub UserForm_MultiPage2_Initialize()
isn't a built in event.

If you're calling that procedure from another subroutine, then that's not the
problem.

But if you're thinking that this event will be called like userform_initialize
will, then that's the problem.

You could either merge your code into userform_initialize procedure (or some
other procedure???) or have that procedure call yours.

Private Sub UserForm_Initialize()
    Call UserForm_MultiPage2_Initialize
End Sub

======
Just a note...

If you're looking for those built-in procedures, you can use the dropdowns at
the top of the code window in the VBE.  The combobox at the top left will show
you the controls/objects/categories and the combobox on the right will show you
the options that are built into excel.  (And stuff you've created--so be
careful!)

Error = Cannot open this item, The necessary security algorithm is not supported


The signed emails are coming from a IOS (MAC) running Outlook!!!

 

There is a Hotfix for Outlook 2007 on a XP machine. http://support.microsoft.com/kb/952020

 

Will the Hotfix work on a Windows 2003 Server, which I believe is a XP based machine or is another Hotfix available?

 

 

Scenario:

 

 

On a Windows Vista-based computer that has Microsoft Office Outlook 2007 installed, you create an S/MIME message.

·         You send this message as a clear-text signed message.

·         The recipient tries to open the message in Outlook 2007 on a Windows XP-based computer.

   

Cause:

Windows Vista uses the Cryptography Next Generation (CNG) algorithm to sign an e-mail message. This algorithm is unavailable in Windows XP. When Outlook 2007 calls the Crypt32 function to decode a signed e-mail message, the Crypt32function fails immediately because it does not recognize the hash algorithm. Therefore, Outlook 2007 cannot decode the message.

Say your time is in A1, enter in B1: =A1-TIME(0,1,0)

joeu2004 say:

If the date/time is in A1, then in another cell:

=A1-TIME(0,1,0)

or

=A1-"0:1"

If you want to change A1 itself, enter 0:1 into another cell, copy that cell, then paste-special-subtract into A1.

Caveat:  Excel might automatically change the format of A1.  In that case, also change the format of A1 to what you had before.
