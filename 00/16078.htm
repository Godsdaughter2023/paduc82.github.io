Delete row when last rows are hidden - Microsoft Community

dan_1981 say:

Delete row when last rows are hidden

In an excel file I have only the area A1:T10 visible (meaning that the columns after T are hidden and the rows after 10 are hidden).

When I delete a column the visible columns are with one less and when I insert a column the visible columns are with one more.

Why when I delete a row the number of visible rows is the same after deletion? Whan can I do so that when I delete a row I get a row less visible?

 

This was originally asked here http://social.msdn.microsoft.com/Forums/en-US/exceldev/thread/3b83a417-653d-4270-9627-ae20da07d96d/

rich007a say:

Try this:

1) Select cells A1:T10 and define a named range "MyRng" via Formulas tab > Defined Names group > Define Name.  Make sure it has workbook scope.

2) Right-click on the sheet's tab at the bottom of Excel and View Code.

3) Paste the following, then close the VB editor:

 

Private Sub Worksheet_Change(ByVal Target As Range)
' First create a named range called "MyRng" covering the cells you initially want to
' remain visible.
'
    Dim strNmRngAdr As String, strNewNmRngAdr As String
   
     'Keep named range anchored at A1
    strNmRngAdr = ThisWorkbook.Names("MyRng").RefersTo
    strNewNmRngAdr = Me.Name & "!$A$1" & Right(strNmRngAdr, Len(strNmRngAdr) - InStr(strNmRngAdr, ":") + 1)
    ThisWorkbook.Names("MyRng").RefersTo = Range(strNewNmRngAdr)
   
     'Hide rows below and columns to the right of the named range:
    With Range("MyRng")
        .EntireColumn.Hidden = False
        .EntireRow.Hidden = False
        Range(.Cells(.Rows.Count + 1, 1), .Cells(Me.Rows.Count, 1)).EntireRow.Hidden = True
        Range(.Cells(1, .Columns.Count + 1), .Cells(1, Me.Columns.Count)).EntireColumn.Hidden = True
    End With
End Sub


That should fix it.  The code above does two things:
1) It anchors the top-left corner of the named range to A1, in case you select row 1 and insert a new row, or select column A and insert a new column.  If you insert any new rows or columns in any other row, the named range will expand automatically.

2) It hides all rows below the named range, and all columns to the right of the named range.

These two things happen automatically each time any changes are made to the sheet.

Hope that helps.
Cheers
Rich

When you are using POP3 as a protocol.........

 The re-sending of old emails maybe due to a bug on the server when the server assumes some of the emails as new emails that need to be sent. 

 

That is a false statement in two ways:

The server does not send emails to a pop3 email client.  The pop3 client connects to the server and downloads the emails from the server.

The pop3 client keeps a list of all emails downloaded from the server.  The server does not keep the list.


When using pop3, only the online Inbox emails are downloaded. 

Whenever you setup a new device or program to connect using pop3, all online Inbox emails will download to the device or program.  All emails that the device has not downloaded previously are new emails.

bill manville say:

I would modify the data so that there was just one complaint code per row.
A procedure like this might do it:

Sub SplitComplaintCodes
  Dim R As Range
  Dim stCodes As String
  Dim iCodes As Integer
  Dim iCode As Integer
  Dim vCodes
  Set R = Cells(Rows.Count,"A").End(xlUp) ' assumes no extraneous data below the table
  Do Until R.Row = 1  ' assumes there are headings in column 1
    stCodes = R.Cells(1,2)
    iCodes = Len(stCodes)-Len(Replace(stCodes,";",""))
    If iCodes >1 Then
      ' multiple codes
      R.Offset(1).Resize(iCodes-1).EntireRow.Insert  ' make new rows above the row below
      R.EntireRow.Resize(iCodes).FillDown  ' copies all fields down
      For iCode = 1 To iCodes  ' store one code per row
        R.Cells(iCode,2) = Left(stCodes,InStr(stCodes,";"))
        stCodes = Mid(stCodes, InStr(stCodes,";")+1)
      Next
    End If
    Set R = R.Offset(-1)  ' previous row
  Loop
End Sub
