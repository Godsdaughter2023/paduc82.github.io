What exactly happens when you change a computer's name, and it - Microsoft Community

bawrites say:

What exactly happens when you change a computer's name, and it asks you to reboot?

What exactly happens when you change a computer's name, and it asks you to reboot - relating to network identification and registration?  Specifically when a computer is not yet on a domain but is still within a workgroup.  Is there a broadcoast of some kind from the PC notifying the other computers that it's name has changed?
brad groux say:
Computers in work groups don't actively seek one another, so if you rename a computer it won't broadcast anything. The reboot is to clear the original name from the system's cache, it will think it is still the old name until the reboot takes place. Computers in work groups simply "shake hands" with one another whenever they need to communicate.

Computers that are part of a domain are created a "Computer Account" which is assigned a random password which changes every 30 days by default automatically. These computer accounts function nearly identical to an Active Directory user account, which means they aren't proactive they simply cache credentials with their permissions that they provide if/when requested by assets in the environment.

Active Directory user and computer accounts are stored in the NTDS.DIT database, which is the backbone of an AD environment. When a computer name is changed in a domain, the PC needs to rebooted simply for the PC. The name change in Active Directory will replicate throughout the environment based upon the replication settings of that environment. 

I'm oversimplifying much of this, but the overall concept is there.
