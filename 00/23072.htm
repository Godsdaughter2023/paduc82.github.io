mikelee101 say:

Scientific Notation driving you nuts too?

I find myself having to paste large numbers into Excel frequently; UPCs, FedEx or USPS tracking numbers, etc., and Excel always wants to convert these to scientific notation.  In the case of a number longer than 15 digits, it then truncates it on you and replaces the last digits with zeros, which is not good when you're trying to track a package.

After much searching and reading of posts and threads and not finding much of a solution, I figured I'd try a couple of things.  Below is a fairly simple routine that works for me, and I figured by posting it here maybe some folks that are way better at this than me could edit/grow/adapt it to fit more varied situations.

I'm always pasting into a column, hence the .end(xldown).  I'm usually pasting 20-50 at a time, and there's never data below the activecell.  The data I'm pasting usually comes to me from an email, so it's in HTML format.

I've assigned it to the ctrl + j keyboard shortcut, since windows doesn't seem to use that for anything, so now all I have to do is remember to hit that instead of ctrl + v when I'm pasting tracking or UPC data.

I know from reading other posts there are plenty of folks out there who are driven just as insane by this as I was, so I figured I'd put this out there in the hopes that it might help a few more of us.

Oh, and if anyone from Microsoft is reading this, how about an option or a registry key or anything so that we can just turn scientific notation off?  Please?

----------------------------

Sub PasteUPCs()
Dim Rng As Range

'use this for pasting UPCs and FedEx/USPS tracking numbers without having them go to scientific notation

'ctrl + j for keyboard shortcut

Set Rng = ActiveSheet.Range(ActiveCell, ActiveCell.End(xlDown))

Rng.Cells.NumberFormat = "@"


ActiveSheet.PasteSpecial Format:="Unicode Text", Link:=False, _
        DisplayAsIcon:=False, NoHTMLFormatting:=True

ActiveCell.Column.AutoFit


End Sub

------------------

mike middleton say:
I have a made a forecast trendline and I need values from it (the x-intercept). Is it possible to obtain these values?

If it's a linear trendline, you could use the INTERCEPT and SLOPE worksheet functions, where

Y = INTERCEPT + SLOPE * X

The value of the INTERCEPT function is the Y intercept, i.e., the value of Y when X = 0.

If you want the value of X when Y = 0, a bit of algebra shows that X = - INTERCEPT / SLOPE when Y = 0.

-  Mike Middleton, www.TreePlan.com, www.MikeMiddleton.com

dirk goldgar say:

I'm using 2010. I should have explained myself clearer. I want to populate another text box (unbound) after a selection has been made from the date picker.

 

Select date (from picker) and populate that date in another unbound text box.

 


Are there two text boxes, then:  one to receive the date directly from the date picker, and another to which that date should automatically be copied?  If so, then you can use the AfterUpdate event of the first text box to assign the value of that text box to the second text box, like this:

 

'------ start of code example #1 ------

Private Sub Text0_AfterUpdate()

 

    Me.Text2 = Me.Text0

 

End Sub

'------ end of code example #1 ------

 

Note, though, that the AfterUpdate event won't fire until the focus leaves the text box.  That is, you may use the date picker for Text0, select a date in the date picker, and have that date appear in Text0, but Text0's AfterUpdate event won't fire until you tab or click out of the text box, or otherwise shift the focus someplace else.

 

If you need to have the second text box show the selected date *immediately*, you can use the Change event of the first text box instead of its AfterUpdate event, like this:

 

'------ start of code example #2 ------

Private Sub Text0_Change()

 

    Me.Text2 = Me.Text0.Text

End Sub

'------ end of code example #2 ------

 

Notice that now you have to refer to the .Text property of Text0, because the date shown in the text box hasn't yet been committed, so its .Value property hasn't been changed.

 

Be aware that the trick of using the Change event and .Text property is best when the "source" text box is only modified by the date picker.  If you type in the text box, each keystroke will be reflected in the second text box, which may not be what you want.
