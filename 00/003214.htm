faraz a qureshi say:

Query for Last & Second Last entries of group together - Access 2013

Hi,

In an Access database table I have a data like:

  1. ItemID (String)
  2. ItemTag (String)
  3. TransferredOn (Date)
  4. TransferredTo (String)

I want a query to show the Last User of the item from whom the item was transferred.

A sort of:

ItemID; ItemTag; TransferredOn; TransferredTo; LastUser (TransferredFrom)

A new item with no former user may reflect the LastUser (TransferredFrom) as Null, None but necessary.

Thanks for your help.

ken sheridan say: The following will return all transfers per item in descending date order, with the user from whom the items was transferred:

SELECT IT1.*,
    (SELECT TransferredTo
     FROM ItemTransfers AS IT2
     WHERE IT2.ItemID = IT1.ItemID
     AND IT2.TransferredOn =
         (SELECT MAX(TransferredOn)
          FROM ItemTransfers AS IT3
          WHERE IT3.ItemID = IT1.ItemID
          AND IT3.TransferredOn < IT1.TransferredOn))
AS TransferredFrom
FROM ItemTransfers AS IT1
ORDER BY ItemID,TransferredOn DESC;

The following will do the same but restricted to the latest transfer, or Null transfer, per item only:

SELECT IT1.*,
    (SELECT TransferredTo
     FROM ItemTransfers AS IT2
     WHERE IT2.ItemID = IT1.ItemID
     AND IT2.TransferredOn =
         (SELECT MAX(TransferredOn)
          FROM ItemTransfers AS IT3
          WHERE IT3.ItemID = IT1.ItemID
          AND IT3.TransferredOn < IT1.TransferredOn))
AS TransferredFrom
FROM ItemTransfers AS IT1
WHERE IT1.TransferredOn =
    (SELECT MAX(TransferredOn)
     FROM ItemTransfers AS IT4
     WHERE IT4.ItemID = IT1.ItemID)
ORDER BY ItemID;

ken sheridan say:

I guess it's occurring only because of duplicate dates in TransferredOn?

That is correct, and it indicates a problem with the data.  If the column is to contain only date values we must assume that an item cannot be transferred more than once on the same day. This assumption is necessary as, if only the date values are stored in the column and an item could be transferred more than once in the same day, then it would be impossible to know which transfer preceded the other(s) in that day.  If we can't make this assumption, then to provide this information it would be necessary to include the time of day in the values to make them distinct per ItemID.

This draws attention to the importance of good indexing in tables. If we assume that an item cannot be transferred more than once on the same day, or the time of day is included in the values to allow for two or more transfers per ItemID, the ItemID and TransferredOn columns should be included in a unique index, which is most easily done by their being made the composite primary key of the table.
