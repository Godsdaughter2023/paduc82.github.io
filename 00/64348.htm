It's actually not displaying file history, but files in the same location. If you start typing "a..." for example, the Save dialog will 'helpfully' autofill with the first file name of the same type also starting with the same letter, and also display the other file names of the same type also starting with the same letter in the drop-down list.

There's no way to disable this. The only way around it is to move all the existing files (images in your case) out from the Downloads folder before saving new ones, or to save with a starting letter that no existing file name starts with.

davem121 say:

To stop this behavior in Windows 7 you used to be able to edit the registry, just open your registry editor . . . though this key does not seem to exist in Windows 10 . . .

HKEY_CURRENT_USER > Software > Microsoft > Windows > CurrentVersion > Explorer > AutoComplete

Inside that key are two values that need to be changed: "Append Completion" and "AutoSuggest".

Set both of those to "No"
