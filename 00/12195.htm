ken sheridan say: If we look at this in the context of my outline of the relational model in my earlier reply there are, it seems to me at first sight (and I must stress that this is only an initial analysis), entity types, and thus tables, with attributes, and thus columns as follows, though you'll doubtless have better names for them:

Patients
....PatientID
....Firstname
....LastName
...etc

This is self explanatory.

DataAreas
....Data Area

This will have rows with values Biochemistry, Hematology, Urinalysis.  As its single column, the column is of course the primary key.

DataParameters
....DataArea
....DataParameter

This holds the different parameter types of applicable to each data area, so there would be rows such as:

Biochemistry    Sodium
Biochemistry    Potassium
Biochemistry    Urea
Biochemistry    Creatinine
etc

Other rows would have the parameters for hematology and urinalysis.  The primary key of this table is a composite one of both columns.   

Visits
....PatientID
....VisitDate
....other attributes of the visit as a whole

Assuming a patient visits only once on any one day the primary key of this table is a composite one of PatientID and VisitDate

This is the basic framework of the model.  I haven't included Initials as an attribute anywhere as its unclear what this refers to, the patient? the physician? the phlebotomist? the medical biochemist?

What the entity type Tests models is the relationship between Visits and DataParameters, so the Tests table would be like this:

Tests
....PatientID
....VisitDate
....DataArea
....DataParameter
....TestTime
....TestResult

This table has two composite foreign keys, one comprising PatientID and VisitDate and referencing the composite primary key of Visits, the other comprising DataArea and DataParameter and referencing the composite primary key of DataParameters.  The primary key of the table is thus a composite one of all four of these columns.  TestTime and TestResult are non-key attributes.  If all tests for a visit are recorded as a single time, however, then there would be no need for a TestTime column as the date/time value would be stored in the VisitDate column and therefore entered only once per patient/visit.

Once the principal tables have been filled the insertion of rows into the Visits and Tests tables requires a minimum of data entry as this can be done by means of a Visits form with a Tests subform, the latter in continuous forms view.  A patient would be selected from a combo box's list in the main parent form and the visit date entered.  Each test is then entered by inserting a row for each in the subform, selecting the data area and data parameter from combo boxes, the latter correlated with the former so it lists only the parameters for the selected data area, and the visit time (if necessary) and test result entered in other controls in the subform.

Output of the type you wish would best be done in a report rather than in a form based on a query.  A report allows grouping and very flexible control over the layout to present the data in the most appropriate format.

Method 1: See whether pictures are available on another Web site

Sometimes the link to a picture on a Web site is broken, and that is why a picture is not being displayed. Visit another Web site to determine whether pictures are being displayed correctly on it. Method 2: Clear temporary Internet files

Clearing your temporary Internet files creates more space on your hard disk. If your hard disk is close to being full, this could affect your ability to view pictures on a Web site.

1. Start Internet Explorer 8.
2. On the Tools menu, click Internet Options.
3. On the General tab, click Delete under Browsing History.
4. In the Delete Browsing History dialog box, click to clear all of the check boxes except for the Temporary Internet Files check box, and then click Delete.
5. Click OK to close the Internet Options dialog box.

If step 2 doesn’t help, you may go ahead with method 3.

Method 3: Verify that the Show Pictures option has not been turned off

If the Show Pictures option in Internet Explorer is turned off, you cannot see pictures. To determine whether the Show Pictures option is turned off, follow these steps:

2. Click the Advanced tab, and then verify that the Show Pictures check box is selected under Multimedia.

It's important to distinguish between "paragraphs" and "Lines".

Word will arrange paragraphs in multiple columns.  It will try NOT to split "lines" in to columns.

I think it is important to turn on your "Show all nonprinting characters" button ¶ so you can see what you are doing.  You will then be able to see the paragraphs and section breaks concerned.

When you do that, you will discover Word puts a Section Break above the text and another below it.  The section breaks contain the column definitions: they are what "makes" the columns.

To do what you need, both section breaks need to be "Continuous".  There are multiple types of section break: the only kind that does not throw a new page is the "Continuous" break.  To find out, click AFTER the section break and then use Format>Document>Layout "Section starts on..." to see what kind of section break is above the cursor.

If you see anything except "Continuous", it's going to throw a new page.  Add a continuous section break below the bad one, then delete the bad one.  Don't delete the bad one forst or you will reformat the entire document.

Cheers.
