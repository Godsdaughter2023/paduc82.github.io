IRQL_NOT_LESS_OR_EQUAL is caused by kernel mode device drivers that used an improper address. It indicates that an attempt was made to access an invalid address while at a raised interrupt request level. This is either a bad memory or a pageability problem with the device driver code.

The error that generates this bug check usually occurs after an installation of a faulty device driver, system service, or BIOS. If you have recently added a hardware to your system, remove it and see if the error appears when you use your computer.

somebody who crashes windows say:

Of course, the volume (C:, I suppose) is in use, when the computer is turned on. So the command window offers you to do check disk on next restart. Type

Y

if you want to check disk (on next restart)

or

N

if you don't want to check disk at all

If you typed Y, you can then Restart your computer, and then, after shutting down and starting again, before allowing you to log in, it will start checking disk (white letters on blue screen). It can take two-four-eight hours, depending on the size of the disk. I usually do it at night.

P.S. In order not to forget about check disk it's better to restart you computer right after you typed in the command prompt chkdsk /f /r ENTER Y ENTER

Check disk doesn't have 100% guarantee of solving your problem, but at least it repairs as much corrupted files as possible (files are usually corrupted if they are used during power failure or abnormal shutdown or physical failure of computer on the floor or something like that)

kp20kp say:

Just to let everyone know, even after i factory reset my laptop the black screen problem was not fixed. What end up working to fix it permanently was for me to edit a power setting within the device manager for the "Intel Management Engine Interface." I noticed that Microsoft themselves recommended to downgrade this certain driver so it must have been this. Here's how i did it

1) Go to the device manager (I typed it in the cortana box)

2) Go to system devices

3) Scroll down to Intel Management Engine Interface

4) Open its properties

5) Press the power management tab

6) Uncheck the box that allows the computer to turn it off to save power.

7) Press OK and Restart the computer
