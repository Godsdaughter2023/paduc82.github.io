rick rothstein, mvp - excel say:

If you can use a worksheet formula and your text in the range is guaranteed not to contain three dots except when you have a substring shaped like you described...

=COUNTIF(C3:H9,"*.*.*.*")

If you really need this in VB code...

CountOfSubStrings = WorksheetFunction.CountIf(Range("C3:H9"), "*.*.*.*")

where you would replace my example range of C3:H9 with your actual range. If that text substring is always 3 digits followed be 3 digits followed by 4 digits followed by 3 digits, then we can tighten the above a little more by replacing "*.*.*.*" with "???.???.????.???" which might help avoid identifying incorrect entries if the text in the range is not well controlled.

bobgreenblatt say:

So, C15 contains the sum of C2:C14, representing the total amount for the month. And, You want C16 to contain the total amount paid to date, indicated by check marks in D2:D14. I am assuming that D12:D14 contain the word "Paid" or are blank. If so, one way is to put this formula in C16:

=sum((c2:c14)*(d2:D14="Paid"))

This formula must be array entered. Type the formula in the formula bar and then press control-shift-enter. If correct, Excel will surround the formula text with braces { }

You can also us, in C15 Sumif if you choose. Look in Excel help for the syntax.

Error while importing PST/CSV file in Outlook 2013 with Hotmail account:


This error is the result of trying to import into an Exchange ActiveSync (EAS) account. The EAS service provider (Outlook.com / Hotmail) does not support importing (or moving), only copying into the folder. If you need calendar or contacts imported, you’ll need to import them into a local PST file then copy the appointments or contacts into the EAS account’s calendar or contacts folders.


Use a list view, especially in a Calendar, then select all and right-click, drag and copy the items to the EAS account. If the items you copied are not synced up to the server after a few minutes, press F9 or the Send and Receive button.

andreas killer say:
Forget the reference, the reference is only related for the "vbext_ws_Maximize" constant.

You said: "the line is ignored", that means that your settings trust the VBA project is set properly, otherwise you'll get an RTE 6068.

Well, your description can only mean that the line is executed and works, but not in the way that you expected. .-)

Please execute this sub:

Sub Test()
  VBE.ActiveWindow.WindowState = 0
End Sub

As you see, the VBE is a multiple windows application, like Excel. With this property you can only size the window inside the VBE, not the VBE itself.

If you want to maximize the VBE itself, you must use the MainWindow object instead.

Sub Test()
  VBE.MainWindow.WindowState = vbext_ws_Maximize
End Sub
