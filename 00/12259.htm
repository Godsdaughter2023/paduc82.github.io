I want to delete Page Numbers from Table of Contents, but retain mouse-click interactivity

I want to eliminate the page numbers on the far right, but still allow the user to "click on" the Chapter and go to the page.

I have eliminated the page numbers 2-3 times, but then I cannot "mouse over & click"  the Chapter Title and move to the Chapter in the document??

Am I missing something simple?
john mcghie say:
No, you are not.  "Microsoft" is missing something simple.

In Office for Mac, the Table of Contents generator lacks the ability to create hyperlinks.  Every time it regenerates a TOC, the  PC version creates a hyperlink anchored to the whole TOC entry, the Mac version has this feature disabled.

On both the PC and the Mac, the page number itself (and only the page number) contains a cross-reference to the target page.  If you double-click the page number, you will get to the page, on either the PC or the Mac.

But if you remove the page number on the Mac, you also remove the link.

The work-around is to re-generate the TOC on a PC.  That will re-generate the hyperlinks.  If you then do not re-generate the TOC on the Mac, the Hyperlinks will keep working.

It's a silly little bug that has been with us on the Mac for ten years or so, but they show no signs of fixing it.

    The short answer is yes... why not?

    The important point of this question is not Excel nor the Userform, it is about the Solidworks object model.

    I know that object model a little and I can say that it is important that you use the correct data types, you can not use the automatically data type conversation done by VBA.

    Inside Excel you can mix up anything, e.g. write a string "0:12" into a double and it works. In Solidworks such things fails.

    If you have questions about the Solidworks object model I suggest you ask here:
    https://forum.solidworks.com/community/api

    In here are only experts for Excel.

    But as I know Solidworks too, why not use a usual Excel table inside Solidworks to build the different configurations? No code and a lot easier.
