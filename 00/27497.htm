I have a 200-page document on Word - I wish to email 50+ selected - Microsoft Community

ejd2 say:

I have a 200-page document on Word - I wish to email 50+ selected pages. How?

I wish to extract about 50 pages from a 200-page Word document and email them - doing them 1 page at a time appears to be somewhat time-wasting. Is there a speedier way to do this and how is it done?
jay freedman say:
Make a copy of the document, either by copying the file or by opening the document and using File > Save As to make a copy under another name. Then delete the parts of the copy that are not the parts you want to send.

To do this, put the cursor at the start of a part to be deleted. Then use the scroll bar (NOT the arrow keys or Pg Up/Pg Down) to locate the end of the part -- the object is to be able to see the end point WITHOUT moving the cursor. When you can see that spot, hold the Shift key and left-click at the end point to select everything in that part. When the selection is correct, press the Delete key. Save the result (Ctrl+S). Repeat this procedure for each part that you need to remove, until all that remains is the part you want to send.

If at any point you think you made a mistake, you can click the Undo button or press Ctrl+Z to restore what you just deleted. If you make a complete mess, delete the copy of the document and start over with a new copy from the original.

The main reason for doing it this way is that the formatting that applies to the whole document (margins, orientation, headers/footers, etc.) is stored in the last paragraph mark of the document. If you try to copy just the part of the document you want to send and paste it into a new blank document, that formatting would not be copied.

However, if you import the data formatted as General, then change the format of the cells or column to Text afterwards, anything that appears to be a number to Excel will continue to be interpreted as a number even though the cell format is Text.  In other words, TYPE(A1) returns 1 (numeric), not 2 (text).

The CRITICAL_PROCESS_DIED bug check has a value of 0x000000EF. This indicates that a critical system process died.

Bug Check 0xEF: CRITICAL_PROCESS_DIED:

https://msdn.microsoft.com/en-us/library/windows/hardware/ff560358(v=vs.85).aspx

Method 1: Boot into Safe Mode and check if it helps.

Safe Mode starts Windows with a limited set of files and drivers. Startup programs don't run in Safe Mode with Networking, and only the basic drivers needed to start Windows are installed. To enter the Safe Mode:

•Restart your PC.

•Press Shift + F8 before the Windows logo appears.

•Click See Advanced Repair Options.

•Click Troubleshoot.

•Click Advanced Options.

•Click Windows Startup Settings.

•Click Restart.

•You then see a list where you can choose Safe Mode.

You may also refer the below article to enter the Safe Mode:

http://windows.microsoft.com/en-us/windows-8/windows-startup-settings-safe-mode

Note: Applies to Windows 10 as well. 

If the issue does not persist in Safe Mode, then perform a clean boot to check if any third party application is causing the issue, as clean boot helps in eliminating software conflicts.

How to troubleshoot problems using clean boot in Windows: http://support.microsoft.com/kb/929135
