How to use VBA to copy text between two specified words? - Microsoft Community

anand yrh say:

How to use VBA to copy text between two specified words?

Hi,

I am trying to copy the text, tables which is in between two different words.

eg: in page one the start text is 'Summary' and in page two the end text is 'End of Summary', here in between these two words, I have some content and tables which need to be selected and copied to another document.

This process will be done for 100s of docs 

Regards,
Anand
hansv mvp say:

Try this as starting point. You may have to tweak it to suit your needs.

 

 

Sub CopySummary()
    Dim lngStart As Long
    Dim lngEnd As Long
    Dim docNew As Document
    Selection.HomeKey Unit:=wdStory
    With Selection.Find
        .ClearFormatting
        .Wrap = wdFindStop
        .MatchCase = False
        .Text = "Summary"
        If .Execute = False Then
            MsgBox "'Summary' not found.", vbExclamation
            Exit Sub
        End If
        Selection.Collapse Direction:=wdCollapseEnd
        lngStart = Selection.End
        .Text = "End of Summary"
        If .Execute = False Then
            MsgBox "'End of Summary' not found.", vbExclamation
            Exit Sub
        End If
        lngEnd = Selection.Start
    End With
    ActiveDocument.Range(lngStart, lngEnd).Copy
    Set docNew = Documents.Add
    docNew.Content.Paste
End Sub

armen stein say:

Hi Alex,

 

The problem is that the sproc is running in the connection to the Access session, so SQL is giving up when the connection is broken.  But you know that already.

Another approach is to disconnect it entirely by throwing a "request" record into a driver table in SQL Server.  Then a SQL agent job looks in the driver table periodically (every minute?  every hour?) and executes any updates it finds there, perhaps inside a transaction so it's all or nothing.  At the end of the update process, the agent job updates the request record as Complete.  Another advantage of this approach is that you can review the history of updates, debug problems and measure performance more easily.

Hope this helps,

You can use a formula and compare the strings to determine how many percent of A1 match A2 

B1:  =SUMPRODUCT(--(MID(A1,ROW(INDIRECT("1:"&LEN(A1))),1)=MID(A2,ROW(INDIRECT("1:"&LEN(A1))),1)))/LEN(A1)

andreas killer say:
You are right, Excel highlight that as duplicates, because within the default conditional formatting rule Excel converts text that looks like numbers as numbers.

Within Excel, the same thing happens as if you apply this formula:
B1: =VALUE(A1)
As you can see in the screenshot below, the numbers becomes the same. This is due to the max. possible precision.
https://support.microsoft.com/en-us/help/78113/...

You can avoid that if you add any non numerical char to the number (at front or end).
C1: ="#"&A1
Now Excel "sees" the text as text, does no conversation and compares the strings.
