albert d. kallal say:

There is no support in Word for multiple values.

However, there is a trick that does usually work.

Built a query in query builder.

 

Include all of the columns you want but NOT the multi-choice colums.
So

ID
FirstName
LastName
Address
Etc.

For the multi-value column, use this format in the query builder for the colum:

 

MyColors: DLookUp("Color","customers","id = " & [customers].[id])

 

In the above, we thus add a column called MyColors based on a multi-value colum Color.

 

So the multiple choice columns is Color, and our table the for is based on is customers.
The primary key (pk) is ID.

Now for your merge code button in that form, you use the query and NOT the forms data source.

You thus go:

‘ we using MergeAllWord so we have force form to save data:

Dim strSql       as string
If me.dirty = true then
   Me.dirty = false
End if

strSql = "select * from qryMultipleColors where id = " = me!id
MergeAllWord strSql

 

The above will thus merge the one record we are looking at, but use the QUERY IN place of the forms data source. We thus had to add the id = criteria as above to restrict the merge to the ONE record we are looking at. So we use MergeAllWord command in place of MergeSingleWord.

 

So if you replace the multi-value column with a dlookup() like above, then all of the choices will be placed in that column.

--------------------------Steps to Create Macro--------------------------
Open Outlook > Go to Tools > Choose Macro > Select Macros
Type in a macro name and click 'Create'
Paste the code below into the new Microsoft Visual Basic Project window.

   Sub MyMacro()
       Set newItem = Application.CreateItemFromTemplate("c:\Path\To\MyTemplate.oft")
       newItem.Display
       Set newItem = Nothing
   End Sub

Save & close the window ('Return to Microsoft Outlook' under the File menu)
------------------------------------------------------------------------------

--------------------------Steps to Add Macro to Standard Toolbar--------------------------
Right click on the toolbar area in Outlook and choose 'Customize'
Click the 'Commands' tab & scroll down until you find Macros under the Categories window.
Locate your Macro in the 'Commands' window
Drag your Macro to the appropriate location on the Standard Toolbar
Once in place, the 'Modify Selection' drop down menu under the 'Categories' window should be available.
Click 'Modify Selection' to rename it, apply a better looking icon etc.
Click close on the 'Customize' screen when finished.
--------------------------------------------------------------------------------------------------

---------------Steps to Add Macro to 'File > New' Menu & the 'New' Dropdown button on the Standard Toolbar---------------
Essentially you can do the same steps as above, and drop the macro under the either location but here's an alternate method.
Right click on the toolbar area in Outlook and choose 'Customize'
Click the 'Rearrange Commands' button
When the new window opens, choose either:

  • File | New from the 'Menu Bar' drop down
  • Standard | New from the 'Toolbar' drop down

Once you've made your decision the steps are the same.
Click Add which will open the 'Add Command' window that looks exactly like a portion of the 'Commands' tab you were in earlier.
Locate your Macro in the 'Commands' window
Click your Macro and click OK
With your new Macro menu item selected, click the 'Move Up' or 'Move Down' buttons to place it in the appropriate location.
With your new Macro menu item selected, click the 'Modify Selection' to rename it, apply a better looking icon etc.
Click close on the 'Rearrange Commands' screen when finished.
Click close on the 'Customize' screen when finished.

olivier terwindt say:

SUMIFS (inventory)

Good evening,


I've been having some trouble with the =SUMIFS function:

I would like to make it add a given amount (orders) to my stock, and subtract a given amount (in case of sales) from the same stock.

jllatham say:

Not a lot to go on here, but let me try to get you started.  Let us say that you are trying to determine stock based on dates?  Then you might have a SUMIFS() that tested a date column and totalled the orders, and then combine it with one that compares the same dates and totals the sales and you subtract one from the other to get remaining inventory.  Something like this in a formula:

=SUMIFS(to total orders) - SUMIFS(to total sales)

 

But if you are running the count from day one through now, you might not even need SUMIFS(), you could just have a formula similar to this:

=SUM(orders column) - SUM(sales column)

 

If you still need SUMIFS() it would be nice to know a little about the layout of your worksheet(s) and what criteria need to be met to determine if something is an order or something is a sale.
