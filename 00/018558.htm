mnaka_atl say:

we had exact same issue when we migrated from 2011 to 2016. We used the import 2011 Outlook identity  function in Outlook 2016. We found over 50% of time that Reminders would be unable to be dismissed no matter what we tried. Sometimes the Reminders/no dismiss issue would not show up until a week or more later after using the imported. The only reliable method we've found to avoid the issue is to create in the beginning ( or recreate the profile if the issue crops up ) in the following

  1. Remove the Exchange account from the Outlook Identity (2011) or Outlook Profile (2016) so you just have local items in the Identity/Profile
  2. Export the local items to a Outlook Archive .olm file some where on the drive
  3. Create a new, clean Outlook 2016 profile using the Outlook 2016 Profile Manager (right click on Outlook 2016, choose Show Package Contents> Contents>Shared Support> Outlook Profile Manager) and set it as the default Profile
  4. Reopen Outlook 2016 into the new, clean Profile
  5. Import the .olm in Outlook 2016 - this will restore local store messages, contacts etc.
  6. Re-add the Exchange account to Profile so it resync's with server
  7. Reopen any shared calendars the old Profile/Identity may have had open (screenshots prior to step 1 are helpful if there are many)

So yes, I agree this is a LONG AND TEDIOUS process, especially for an Import Outlook 2011 Identity feature in 2016 that is obviously ****. But it will get you functional reminders again and work for those users with locally stored items.
