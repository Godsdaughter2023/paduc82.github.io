Application Catalogs/Repositories - Info &amp; - Microsoft Community

ceroberts75 say:

Application Catalogs/Repositories - Info &amp; Installation

This is a post may be necessary as there are many questions all the time on setting up repositories (repos) on thier n900.

 

This post will/should be a good start for those that are not familiar with adding them manually, and if you were to open this post with your n900 browser, just click on the links and they will be automatically installed on your n900.

 

 

Repositories are catalogs on Internet servers containing software you can download for your device.

In order to be able to use a repository, you need to add it to the Application Manager of your Maemo phone. Launch Application Manager and select "Software catalogues" from the pull-down menu. You will see a list of already configured repositories and a "New" button to add a new one.

To add a new repository, you have to enter its name (whatever you like), URL (web address), distribution (should be "fremantle" in case of Maemo 5), and components (which vary depending on repository so you'll need to check it; it's usually "free non-free" but there are also custom ones).

Once you add the new repository, application list under the "Download" button of the App Manager will be refreshed and apps available in the newly added catalog will be listed. Now you can just tap them to install them in your device. You will also be notified about any updates available for installed applications.

 

 

 

The main ones are:

 

Maemo Extras is the main repository, meant to be used by end-users. It contains applications that have been tested and verified for compatibility, safety, proper functionality, "optification" (i.e. installation in the bigger /opt partition not to take space in limited rootfs), etc. If you are a person that does not like using applications which are still in "beta" stage, or you don't feel the need to test/have everything right as it comes, you may want to restrict yourself to only using this Extras repository.

 

EXTRAS 

 

 

 

Extras Testing repository contains freshly created applications that haven't yet been tested/verified. And this is what this repository is meant for: it is for those who would like to test new applications. Using this repository, you can get all new apps right after they have been released, and additionally you can help with testing them against any problems, "improper behaviour", or any other issues. Once an application contained in this repository has been tested and found safe, compatible and properly working, it will get "promoted" to the above mentioned Extras repository for general use.

 

If you decide to use this repository, please be aware of two possible "risks":

  • applications contained there may NOT be correctly "optified". This means that some of them may NOT obey the rule to install larger packages in the large 2 GB /opt partition rather than the small (70 MB or so of free space on a "fresh" device) rootfs. This may lead to the "/" (root) partition getting filled. Of course, once you notice this, you may just unistall those applications, so it is not "irreversible"
  • apps in Testing may NOT pay too much attention to proper power management. This means that installing some of the applications from Testing may result in shorter battery life. Again, as above, if you notice such an issue, you can uninstal the "problematic" software.

 

EXTRAS-TESTING

 

 

 

 Extras-Devel this is the "lowest-level" repository with applications that usually are on very early development stage, let's call them "alpha" versions. If you are not used to dealing with alpha/beta software, or if you want to be sure that software you install won't mess with your phone, do not use this repository. It is not meant for end-users, but only for people who know what they are doing.

 

EXTRAS-DEVEL

My-Maemo - The largest singular repository for tested on devices and safe to use applications from the extras/extras-testing/extras-devel repositories.

A "Self Join" query will be needed here. Do you have multiple agents? If so you will need another field to indicate WHICH agent is checking in or out!

If not, you could use:

SELECT ChkIn.EvlDateTime, ChkOut.EvlDateTime, Format([ChkOut].[EvlDateTime] - [ChkIn].[EvlDateTime]), "hh:nn:ss") AS TimeLoggedOn
FROM yourtable AS ChkIn, yourtable.AS ChkOut
WHERE ChkIn.AgentState = 0 AND ChkOut.AgentState = 1
AND ChkOut.EvlDateTime = (SELECT Max(X.EvlDateTime) FROM yourtable AS X WHERE X.EvlDateTime < ChkOut.EvlDateTime AND X.Status = 1);

john w. vinson say:

There are in fact TWO date/time fields that you need to compare. They happen to reside in different records, but in order to do a calculation comparing them you must use a Query pulling both of them into one "virtuall" query record (you don't need a new table and they don't need to be in the same table record, just in the same Query record). This is called a "SELF JOIN" query because you're joining the table to another instance of itself, to bring the two records together.

My query was an attempt (probably incomplete or unsuccessful) to do just that.

If you have a field (not previously mentioned) for the identity of the Agent you'll need to include that field in the self-join.

So it depends on how your webpage is created, but you might be able to do this:

 

Open the page in Internet Explorer then press F12 to open the Developer Tools.

You should then be able to (having already "got" an IE object) do something like:

 

    Dim IEdoc As HTMLDocument
    Dim ieElCol As IHTMLElementCollection

    Dim bPageLoadFail As Boolean
   

'    If IE.readyState = READYSTATE_COMPLETE Then.....

   

     'Populate fields:
    Set IEdoc = IE.Document
    
    Set ieElCol = IEdoc.getElementsByName("YOUR_ELEMENT_NAME")
    Set ieTypeEl = ieElCol.Item("YOUR_ELEMENT_NAME")
    If ieTypeEl Is Nothing Then
        bPageLoadFail = True
        GoTo ExitPoint
    End If
    ieTypeEl.Value = Range("A1").Value
   
ExitPoint:
    If bPageLoadFail Then MsgBox "Internet Explorer was unable to load page." & vbCr & vbCr _
                    & "Please check you have a working intranet connection.", vbCritical + vbOKOnly, "Title"
    Set IEdoc = Nothing
    Set IE = Nothing
    Exit Sub

End Sub

Try this code below.  No guarantees that this will work, as I'm still not certain about how your webpage is constructed....

 

Sub LoadPageInIE()
' Add references in the VBE via Tools > References to:
' - Microsoft Internet Controls
' - Microsoft HTML Object Library
'
    Dim IE As InternetExplorer
    Dim IEdoc As HTMLDocument
    Dim ieElCol As IHTMLElementCollection
    Dim ieElCustDocRef1 As HTMLInputElement
   
    Dim strURL As String
    Dim dbPauseSec As Double, dbStart As Double
    Dim bPageLoadFail As Boolean
   
    strURL = "http://www.google.com" '<<UPDATE ######################
   
    On Error GoTo ExitPoint
    
     'create a new instance of Internet Explorer:
    Set IE = New InternetExplorer 
    
    IE.Visible = True
    IE.navigate strURL 'navigate to the desired page
   
     'Wait for page to load:
    dbPauseSec = 10         'Set time to timeout in seconds
    dbStart = Timer     'Set start time
    Do
        DoEvents
    Loop While Timer < dbStart + dbPauseSec And IE.readyState <> READYSTATE_COMPLETE
    If IE.readyState <> READYSTATE_COMPLETE Then
        MsgBox "IE timed out"
        GoTo ExitPoint
    End If
    
     'FOR DEBUGGING ONLY:
    On Error GoTo 0
   
     'Populate fields in IE page:
    Set IEdoc = IE.Document
   
    Set ieElCol = IEdoc.getElementsByName("customDocRef1")
    Set ieElCustDocRef1 = ieElCol.Item("customDocRef1")
    If ieElCustDocRef1 Is Nothing Then
        bPageLoadFail = True
        GoTo ExitPoint
    End If
    ieElCustDocRef1.Value = "SOME TEST TEXT"
   
ExitPoint:
     If bPageLoadFail Then MsgBox "Internet Explorer was unable to load page." & vbCr & vbCr _
                     & "Please check you have a working intranet connection.", vbCritical + vbOKOnly, "Title"
     Set IEdoc = Nothing
     Set IE = Nothing
End Sub
