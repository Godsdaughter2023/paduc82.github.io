Listed below are solutions which have worked in the past for those getting a "No gateway" message when trying to connect to a wlan.

 

1. Check to see if MAC filtering is enabled on your router, if it is, you will need to add the MAC address of your phone to the "allowed" list. Use the code *#62209526# to get your phones MAC address.

2. On your pc/laptop, click on Start>Run and type ipconfig/all and note the settings for:

a) Subnet Mask
b) Default Gateway
c) DNS Servers

Now, on your Phone settings: Goto Menu>tools>settings>connection>access points. Select your WLAN access point and go to Advanced settings then Pv4 settings.

 

For phone IP address: Start with the Default Gateway IP and change the last number to something else. Do not use the same IP as Gateway of any of your pc's.
For Subnet mask and Default gateway use the same as on PC
For DNS address: Change to user defined and enter same settings as on PC

3. Check for firmware updates for your router and install if there is one.

 

4. Try different channel settings on your router.

 

5. Disable WirelessLAN Power saving mode. To do this, goto Tools>Settings>Connection>Wireless LAN>Options>Advanced settings>Automatic configuration>change>Power saving, set this to disabled.

6. Disabling QoS (Quality of Service) has been known to resolve this issue on some routers.

lz. say:

Hi

(replaces previous reply - deleted in the meantime). The concatenation of a range/array you expect requires the CONCAT function - not available in Excel 2013. If you search the web you will find User Defined Functions that does it.

A Get & Transform (aka Power Query for Excel 2010/2013) option:

Once Power Query installed:

  • Format A1:A6 as Table > Click somewhere in that table
  • Go to Power Query tab > From Table (this launches the Power Query editor)
  • From the Home tab > Advanced Editor > Select all > Paste the below query code > Done
  • File > Close & Load

Query code

    Source = Excel.CurrentWorkbook(){[Name="Table1"]}[Content],
    #"Replaced Value" = Table.ReplaceValue(Source,",","",Replacer.ReplaceText,Table.ColumnNames(Source)),
    #"Transposed Table" = Table.Transpose(#"Replaced Value"),
    ColumnNames = List.Generate(()=> 1,
                each _ <= Table.RowCount(Source),
                    each _ +1,
                        each Text.Combine({"Column", Text.From(_)}, "")
                                ),
    #"Merged Columns" = Table.CombineColumns(#"Transposed Table",ColumnNames,Combiner.CombineTextByDelimiter(", ", QuoteStyle.None),"Concatenated names")
in
    #"Merged Columns"

amit tandon say:

This can be done using the Formula TextJoin, which is available in Excel 2016:

Array formula (Ctrl+Shift+Enter) in cell B2:

=TEXTJOIN(", ",,MID(A2:A4,FIND(",",A2:A4)+2,LEN(A2:A4))&" "&LEFT(A2:A4,FIND(",",A2:A4)-1))

Add*

Another shorter option: Array formula (Ctrl+Shift+Enter) in cell B2:

=TEXTJOIN(", ",,MID(A2:A4&" "&A2:A4,FIND(",",A2:A4)+2,LEN(A2:A4)-1))

Presuming single space after comma, else use TRIM additionally.

A pivot table is not an object on top of a worksheet (as a chartobject is) but it is the content of cells.

You can clear the pivot table by e.g.
ActiveSheet.PivotTables(1).ClearTable
but then you would have to add fields to it again when you wanted it back

You could make it less visible by setting its font colour to white (assuming the background is also white), but if the user selects a cell in it he will be able to see the content.

Range("A1").PivotTable.TableRange2.Font.ColorIndex=2
