a. user say:

You misunderstand. What you are looking at are not programs - they're processes.

A process is the execution of a portion of application code. A single application may execute in many processes. That's the developer's design.

Dividing the execution of an application into processes makes it possible for the CPU to switch smoothly between tasks, which makes a computer run faster. It also makes the program more stable; if one process crashes it won't take down the entire application.

Google opens processes for more than just tabs. And they're not running 'in secret' either. If you want full details for each process, install Process Explorer from Microsoft.
