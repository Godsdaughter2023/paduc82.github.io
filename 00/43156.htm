Greetings Peter!

 

After reading through your description and checking the Windows Update log file that you have sent, I will try to explain what has happened in the computer. Please let me know if you need any clarification.

 

In the Windows Operating System, the services are interdependent on each other. If service 'A' depends on service 'B' and service 'B' has been stopped/disabled by a user or program, service 'A' will not function properly. And as a result, operations that depends on service 'A' will not produce desired results.

 

The Windows Update operation basically depends on three service in a computer - The Background Intelligent Transfer Service (or BITS), the Cryptographic Service and the Windows Update Service. BITS is the service that is responsible for downloading the files (needed to install an update) from the Microsoft servers. And this service depends on the COM+ services in order to properly function.

 

The Windows Update log from your computer suggest that there were some problems with the download of that update. And BITS is responsible for downloading the files.

 

In your computer, disabling the COM+ Event System service has led to stopping of BITS and has refreshed BITS. And after the COM+ Event System service was re-enabled, BITS started working and it downloaded the files necessary in a correct order. Actually, when Windows Update does not function properly, we usually suggest doing the steps that I have mentioned in my reply in this thread. Sometimes, if the error is caused by a transient download failure, simply restarting BITS or stopping BITS, renaming the 'SoftwareDistribution' folder and restarting BITS helps as well.

 

The other services that you have mentioned has no apparent dependencies or relation to the Windows Update operation. I do not think that they had anything to do with the Windows Update behavior as well.

 

If you come across any future issues with Windows Update installation in the computer and you want to troubleshoot it by checking the services, I would suggest you to make changes only to BITS, Cryptographic and/or Windows Update service as I described in the thread that I have referred to. That makes the diagnosis easy.

 

Another important point that you need to notice is that the first error code that you got with this update failure sometimes points to a problem with the Microsoft Office software installed in the computer. So, if you get the same error code (0x8007066F) with Office Updates in the computer, you may have to repair the Microsoft Office installation. But again, that is only if you get the same error repeatedly with updates for any of the Microsoft Office programs installed in the computer.

 

I hope this explanation helps you understand the scenario. Do let us know if there is any other issue that needs help, we will be glad to be of assistance.

Not only is the Compatibility Checker often wrong about display chips like Nvidia 6150 and 7300 not running fine in Windows 10 - since it requires gamer-level performance when most users don't game - but once one does unnecessarily change the card it will not reset.  This is a known bug we've reported but isn't fixed.
