Hi,

Thank you for posting in Microsoft Community. I understand your concern and I'll be glad to assist you.

Please follow through.

  1. In Windows Explorer, right-click the file or folder you want to work with.

  2. From the pop-up menu, select Properties, and then in the Properties dialog box click the Security tab.

  3. In the Name list box, select the user, contact, computer, or group whose permissions you want to view. If the permissions are dimmed, it means the permissions are inherited from a parent object.

 Turn off UAC (User Account Control)

Before you can do anything, you must turn off the UAC, or you will be locked out of the following steps.

1) Start -> Settings -> Control Panel -> User Accounts
2) Click "Change User Account Control Settings"
3) Move slider all the way down to "Never Notify"
4) Reboot

2) Take Ownership

Yes take ownership. Even though you are logged on as an Administrator, you can't change files that don't belong to you. The Program Files folder is set to the Trusted Installer group and the Administrator doesn't have the rights to change anything. So now we have to claim all the files and folders.

1) Open Windows Explorer
2) R-Click on Program Files -> Properties -> Security Tab
3) Click Advanced -> Owner
4) Click Edit
5) Select Administrators -> Put a checkmark in Replace owner on subcontainers & objects -> Apply
6) Wait a while.....
7) When it finishes, Click OK on all boxes to close everything

3) Fix Permissions

Now that you own the files, you have to give yourself permission to modify them

1) R-Click on Program Files -> Properties -> Security Tab
2) Click Advanced -> Change Permission
3) Select Administrators (any entry) -> Edit
4) Change the Apply To drop down box to This Folder, Subfolder & Files
5) Put check in Full Control under Allow column -> OK -> Apply
6) Wait some more.....
7) When it finishes, the dialog boxes may hide behind the Explorer window. Minimize it and click OK on all the the dialog boxes

8) Reboot PC.

Hope this post helps. Get back to us for further queries. We are happy to help.

Windows Installer folder - Microsoft Community

saras51 say:

Windows Installer folder

When I run norton virus I get a message that a file named C;\Windows\installer\b20d81.msi is corrupted. But I cannot find a directory called C:\Windows|installer to delete the file. Can any body help me please
boboneill say: Hello!

First, to do any editing in c:\windows\installer, you may need to disable User Account Control. User Account Control (aka UAC) is the protection mechanism that prevents unintentional or unauthorized folder, file, and system changes.

The "installer" folder resides in a protection area that is governed by UAC, as do all files/folders in the c:\windows directory...so you will want to temporarily disabled UAC to remove the corrupted MSI file.

The quickest way to disable UAC is to click Start, select Run..., then type msconfig. Select the Tools tab and scroll down until you find "Disable UAC". Click on this option one time and it will turn blue. Then click OK. This change will require you to reboot your PC, so reboot now.

After you log back in you will see a small popup box related to msconfig that you didn't see before...this box is simply telling you about changes made with msconfig before you rebooted; it is informational only and can be removed by clicking OK.

Now that UAC is disabled you can move to step 2...getting into the "Installer" folder.

The "installer" folder is a protected operating systems location.  This means that even if you tell Windows Vista to make hidden folders viewable, you still will not be able to see the "installer" folder in c:\windows without making a second, less secure, file system change.

While you can easily make this change to make the "installer" folder viewable, for a one-time navigation to that location, you may find this approach more efficient.

Click the start button, choose Run..., then type c:\windows\installer. At this point, an explorer window should appear that reveals the contents of the installer folder.

BE CAREFUL :-)  Now that you are in this folder, you will want to be very careful with what you do in there. The "delete" key is not your friend in most cases...and it is less so in a folder like c:\windows\installer.

The c:\windows\installer folder stores installation information for all programs that were installed onto your PC. If you remove something from this folder, you may get unpredictable results with one or more of your applications....this is especially frustrating if you delete an MSI file for a program you use less frequently...meaning, you delete the MSI file today, but dont see strange behavior until you try to access the application several days/weeks later.

Norton may have identified b20d81.msi as corrupt for any number of reasons; unless this file is specifically flagged as a virus or spyware, I suggest you isolate this file someplace for a couple of months...perhaps on a flash drive that doesnt contain any critical files...this way you will have a backup of it in case you begin to notice strange behavior with your applications.

When you are finished, dont forget to turn UAC back on...you can easily do this by running msconfig again (click start...then select run and type msconfig, choose "tools" and turn UAC back on, then reboot).

Good Luck!

thanks, bob

    'Basic Input/Output System', a firmware run by a PC at the time of boot-up,

    multiple operating systems and infect different kinds of motherboards.
    According to them, BIOS-based malicious software can disseminate not just on
    various OSs, but also by a number of hardware. These attacks are hard to
    identify and block.

    Earlier during March 2009 at the Vancouver CanSecWest security conference,
    researchers Anibal Sacco and Alfredo Ortega of Core Security Technologies
    Inc. performed a general BIOS attack that could push malware inside various
    BIOS types, as reported by search security on June 18, 2009.

    A hacker who hijacked the BIOS in the above manner could gain complete
    control over the basic firmware irrespective of the OS.

    Even if all browser applications and OS patches are put in place, it is
    still possible to fully compromise computers at a very low level without any
    vulnerability exploitation. Evidently, the BIOS malware has been effectively

    BIOS rootkit that might help to execute the attack.
