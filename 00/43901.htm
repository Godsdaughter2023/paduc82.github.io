When RAM is low, Windows reaches out to the page file on your hard drive, and starts caching programs there in addition to your RAM. When you use ReadyBoost, it acts as just another page file. dax1792 say:

ReadyBoost intercepts all reads and writes to local hard disks and copies any data being read or written to a cache on the Flash drive. The data is compressed and encrypted.

When ReadyBoost sees random reads that can be satisfied from the cache, it reads from the Flash Drive, but because hard disks have better sequential read access than flash memory, it lets reads that are part of a sequential access pattern go directly to the disk.

Whether this works obviously depends on the workload. I never noticed any improvement with ReadyBoost and I've never seen anyone ever report any improvement in real life. The general feeling is that it might work  when the amount of memory is small, probably less than 2GB. Wikipedia reports an improvement on 512K Vista system, which no one would consider running today.

When you get to 3GB, you are probably getting enough caching in Standby RAM for the effect to be negligible.

arminatorx say:

Just to be on the safe side: Backup your important documents to another drive temporarily and connect the drive to your Xbox console. 

Even if the console normally uses drives for Media use without formatting, it might be that the drive is already formatted in an unsupported file system (maybe you're using Linux or MacOS and have the drive formatted in a Linux file system).

Do NOT let the console format it for game storage! Use it for media use only!

Then, when you were able to record your gameplay, and your documents are still intact, you can simply use the drive normally.
