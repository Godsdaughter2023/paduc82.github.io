david-e say:

The "can't access this file" error message will show a path to the "pst" file which is the one you need to change.

An easy way to locate this file is to type "pst" (without the quotation marks) into the Search. That will list all your Outlook "pst" mail files.

Right click on the one you are getting the error message for, and then select "Open file location."

Then right click on the file again, and select "Properties" from the pop-up menu.

From the Properties dialogue box, select the "Security" tab.

In the "Group or user names" box you will probably see 3 listed: System, Administrators, and one that is the name you are using.

Then click on "Edit" to change permissions, and back in the "Group or user names names" box, click on your name to highlight it. This will show the permissions you have to access this file.

Make sure the "Full control" "Allow" box is checked. Then click the "OK" to save your changes. And "OK" to close the Properties dialogue.

That should fix the problem.

Word unilaterally deciding what your indents should be, even when you - Microsoft Community

ollybuxton say:

Word unilaterally deciding what your indents should be, even when you have told it otherwise

Has anyone else had this experience:

You're in a document. You select a bulleted list (or Word deduces you want one). Word also infers that you wish to indent your bulleted list inside the margin of the normal paragraph.

Word is wrong about this.

Being a pedant, you adjust the margin of the bulleted list to match the normal paragraph, using the ruler.

You then create a new style based your adjusted margin. Let's say you call this style "bulleted". (You could achieve the same thing another way by selecting an existing style called "bulleted" in the style window and select "update to match selection").

So far so good. Your bulleted style now has a hanging indent flush with the normal margin, just how you like it.

But then you take another paragraph change that to your new "bulleted" style. Suddenly, Word goes rogue on you. 

Here is what happens:

  • The new paragraph you have just selected adjusts obediently to the newly defined "bulleted" style. The hanging indent is nicely flush with the margin.
  • But all other instances of bulleted style revert to the original indented margins that Word originally selected for you, but that you told it you didn't like.

Repeat over, and over, and over again. (It has nothing to do with the "style for following paragraph" nor the "Automatically update document styles" box mentioned in other threads).  I think it has something to do with a conflict in the style sheet between the margin settings in the "paragraph" dialogue box and those in the "numbering" dialogue box.

I have managed to fix it in some style sheets, but can never remember what I did to fix it, and it continually recurs. This has happened over a decade of more in many different organisations, so I don't think it is anything to do with me or my installation of word.

Can anyone provide a simple explanation and an easy fix?

stefan blom say:

To correctly change the indentation for a single-level numbered or bulleted list, do the following: Click the first list symbol (bullet or number) to select all of those belonging to the same list. Right-click and choose Adjust List Indents from the context menu.

"Bullet position" (or "Number position" for a numbered list) determines the position of the list symbol. "Text indent" defines where the second and subsequent lines start. "Add tab stop at" determines where text start after the list symbol.  All measurements are from the left margin (of the page, table cell, text box, etc.).

If you want a hanging indent, "Text indent" and "Add tab stop at" should be the same value.

Applying same margin and other formats to text in multiple tables in a - Microsoft Community

abi8220 say:

Applying same margin and other formats to text in multiple tables in a Word document

I have a Word document with several tables throughout the document. Rather than checking each property individually, is there a(n) (easy) way to apply the same text margins, for example, from the first table to the rest of the tables?
jay freedman say:

Whether this will work as you want it to depends on which formatting other than cell margins you want to change.

Click in the first table. On the Table Tools | Design tab, click the lower down-arrow at the right end of the Table Styles gallery

and click New Table Style at the bottom of the dropdown. In the Create New Style From Formatting dialog, select the formatting you want -- most of it will be picked up from the table you clicked in, but some items such as borders may need to be reselected. Besides the formatting you see in that dialog, there's much more on the menu of the Format button in the bottom left corner.

Once you've created a custom table style, click in each of the other tables and apply the style.

If this is still too tedious, it's possible to write a macro that will "read" the first table of the document and apply its formatting to the other tables. It would be necessary to know which specific bits of formatting you want to copy.
