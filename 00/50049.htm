That's a server-side problem, meaning that the problem exists on the computer that is receiving your message. There isn't anything you can do about it, because it's not your problem.

To make things even worse, and more complicated, your message can pass through dozens of other computers before it finally arrives at its destination on the internet, and a problem with any one of those computers can result in the same message. cmdrkeene say:

Windows 7 had the big previews too, as long as the Aero engine was running (Aero was the engine that provided smooth graphics, transparency, and some other effects by using the graphics card to draw the desktop).


In Windows 8, most of the Aero effects are gone (transparency and some other effects), but the engine is still there because it's required to use hardware accelerated graphics for fast performance (otherwise, your powerful graphics card would go unused).


The taskbar team actually didn't change anything, the text display mode is still there.  It just doesn't get triggered usually because the Aero engine is running the graphics.




How to do it anyway!

Luckily, you can actually force this.  There's a registry value that is used to decide how many previews there have to be before it switches over to text mode.  This is so you don't end up with a billion tiny previews if you have a lot of windows open.

there can be a number of reasons for this error - let's start with the 'easy' fix.
 
What may have happened is that the Licensing Store has become corrupted - and is looking in the wrong place.
I would try rebuilding the Licensing Store first, -
 
Recreate the Licensing Store
1) Click Start button.
2) Type: CMD.exe into the 'Search programs and files' field
3) Right-Click on CMD.exe and select Run as Administrator
4) Type: net stop sppsvc   (It may ask you if you are sure, select yes)
Note: the Software Protection service may not be running, this is ok.
5) Type: cd %windir%\ServiceProfiles\NetworkService\AppData\Roaming\Microsoft\SoftwareProtectionPlatform
6) Type: rename tokens.dat tokens.bar
7) Type: cd %windir%\system32
8) Type: net start sppsvc
9) Type: slui.exe
10) After a couple of seconds Windows Activation dialog will appear. You may be asked to re-activate and/or re-enter your product key or Activation may occur automatically.
 
Run MGADiag again, and post the report
 
I that doesn't fix it, then we're in unknown waters, and it's a case of whether you're willing to stay the course while we both learn, or whether you'd rather try MS support direct, in which case you use the no-cost support offered by MS for WGA problems.
