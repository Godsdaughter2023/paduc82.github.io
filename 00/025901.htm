Based on your description, you have create a drop down list in a workbook, and you would like a name to disappear from the list with the name selected in Excel. I would like to suggest you refer to my example below to hide the name in the drop down list.
    1. Go to Formulas> Name Manager and define a name for the original name list, called NameRange.

    2. Add a column named Exclude name to record the name which will be hidden in the drop down list. You can add a drop down list to select which name you want to hide. Then define a name for the cell in Name Manager, called ExcludeName.


    3. Add a column and use this formula to differ the name which will be hidden in the drop down list from other names: =IF(COUNTIF(ExcludeName,NameRange),"",SUM(MAX(B$1:B1),1))

    4. Add a column named ID, a column named Left and use this formula to record the range excluding the hidden name: =IF(D2>MAX($B$2:$B$6),"",INDEX(NameRange,MATCH(D2,$B$2:$B$6,0))). Then define the range as LeftName in Name Manager.


    5. In this case, you add a drop down list excluding the hidden name.

john mcghie say:
Hi Mike:

There are a few "entertainment issues" running Word 2008 on the later versions of Apple OS X.  Upgrading your operating system to one that is four versions later than the one your software was designed for is "thrill-seeking", and you need to be prepared for some excitement.

The one that is likely to hit you immediately is that your autotexts are likely to all disappear!  OS X.8 keeps its preferences and support files in a different place to earlier versions, so when Word starts, it may not be able to find them.

To recover from this, Quite Word, then find your Normal.dotm template and move it to the folder where OS 10.8 keeps it.  Or use Word>Preferences>File Locations to set your "User Templates" setting to the existing location of Normal.dotm.  Either will fix it.

It's better to move the template, since if things get lost later, Word will automatically find it again.

In the later versions of OS X, it can be a fiddle to find the folder:
  1. On the Go menu, click Home.
  2. Open Library.
    Note The Library folder is hidden in Mac OS X Lion. To display this folder, hold down the OPTION key while you click the Go menu.

  3. Open Preferences, and then open Microsoft.
    Note If you have Service Pack 2 installed, open Application Support instead of Preferences.

  4. Move Normal.dotm to /Library/Application Support/Microsoft/Office/User Templates/My Templates
I would not bother going up to Word 2011 now, there's a new one due in a few months (if you do buy 2011 now, you will get a free upgrade to the new one when it comes, your call...)

Neither Microsoft nor Apple are going to do anything about issues with old software, they both want you to buy a new one :-)

You will probably get some issues with window layering: Word 2008 does not know about Mountain Lion's "Spaces".  And you may find some fonts won't work:  Office can load only TrueType fonts and OpenType fonts containing TrueType outlines.  The new OS will gobble a few extra keystrokes you may have assigned elsewhere.

But it will work, probably well enough until the new one comes out.

Hope this helps

Migrating From Office 365 To Office 365 tenant - Microsoft Community

sandeep-n say:

Migrating From Office 365 To Office 365 tenant

Hey guys,

So we are planning to do a office 365 to office 365 migration

Our old tenant is 1) abc.onmicrosoft.com

New tenant 2) xyz.onmicrosoft.com

Users are being synced to the new domain from the active directory.

old tenant has domain abc.com registered in it.

We want to migrate abc.com to xyz.onmicrosoft.com.

Currently users are using *** Email address is removed for privacy *** upn. 

So we would like to move all those users to xyz.onmicrosoft.com and keep using the same UPN 

and no disturbance in user experience.

Can be done? Any ideas?

barry xie msft say:

Hello Sandeep,

 

If you used the steps you mentioned, you may lost the email data. In Step 1, you want to sync users to the new target Tenant. But, as the users are in the old domain now, you may not be able to sync them to the new tenant. You need to remove abc.com in the old tenant and add it as a custom domain in the new xyz.onmicrosoft.com tenant. Also, if you don’t have Exchange Server deployed in your local environment, you will not be able to use IMAP migration. We highly recommend that you use PST to move the data.

 

If you only deployed Active Directory Domain Controller and Active Directory Federation Server in local,  the steps should be:

1. Change the primary email address of the users you want to move from abc.com to abc.onmicrosoft.com.

2. Stop the Azure AD connect. Then back up the data of these users using PST.

3. Remove the abc.com domain from the old tenant. The affected users’ suffix should be abc.onmicrosoft.com now.

4. Add abc.com domain to your new xyz.onmicrosoft.com tenant.

5. Configure Azure AD connect to sync users from Local AD to the new xyz.onmicrosoft.com tenant. The users will have new email addresses as abc.com. Give these accounts the required licenses.

6. Import the email data to the new email accounts.

7. Make sure no data is lost. Cancel the abc.onmicrosoft.com accounts in the old tenant.

 

Regards,

Barry
