suzanne s. barnhill say:
To add to Herb's suggestion, I'll just point out that the text box added to any AutoShape with the Add Text command is always rectangular; it is never the actual shape of the AutoShape unless the AutoShape is a rectangle.

herb tyson say:

If I were doing this, I would use two linked text boxes (grouping doesn't do it--linking does). Draw the first text box for the vertical part of the L, then draw the second text box for the horizontal portion. Put all of the text into the first text box, then click the Link tool in the Drawing Tools Format ribbon tab (in the Text section). This turns the mouse pointer into a bucket icon... click that bucket in the horizontal text box, "dumping" excess text from the first text box into the second.

If you format both text boxes so that there is no shape outline or so that the shape fill & outline are the same color, then you can snug them together and it will look like an L. (You can also group them so that the L can be moved as a unit.)

a. user say:

You're logging into the same user account when this happens, aren't you?


Your PC would have a number of user accounts created in it, each according to the specifications of respective users. However, owing to some grave issues, it wouldn’t be long before one of them gets corrupted.



Think of it this way...

User : essentially the credentials - username/password.

User Profile : Files, Folders and Registry settings that make up the user's experience/own 'world'.  Like Desktop, Documents, Favorites, and the way the screen is laid out, etc.  C:\USERS\<username> is the location of these things in Vista/7.

User Account : the combination of the two listed above.

So - you have a corrupt user account.  If you want to continue on the way you had before, you must get the files from the old user profile moved 'out of the way' and/or 'copyable' so you can take what is still good and use it in a new user profile.

The way it is now - when you attempt to logon, you get a temporary profile (C:\USERS\username.temp or C:\USERS\username.computername or C:\USERS\username.domainname or just C:\USERS\temp or something) that will be obliterated at logoff. 

If you only utilized one account on the system before and don't know much - you will likely use this to create a new (fully functional) user and logoff and then logon as the new user creating the user profile and completing the creation of the user account.  Then you can utilize that new user account to copy the old stuff (from the corrupt profile directory structure) to the new account (desktop, to the actual folders, etc.)

If you know a little - you could rename the user profile directory of the corrupt account and restart.  Then log in with the corrupt user credentials - and depending on HOW the account was corrupted - it shoud recreate the user profile directory, etc.  Then you could copy the stuff you need (not everything - since that profile is corrupt) to the new profile - finally deleting the older renamed profile.

There are several ways to resolve a corrupt account. ---Shenan Stanley, MVP---

Hi Lawrence,

You have two problems here.

The first is that the page number is appearing on the Title Page itself.
You can take care of that by, under the insert tab, editing your header or footer and checking the option for different first page. This activates the (blank) first-page headers and footers (without your page number)

The second problem is that it is appearing in your Table of Contents. Even if there is no page number displayed on your title page, it will continue to show up in your Table of Contents as page 0.

To fix that you must either change your Table of Contents field or the style for whatever is showing up on your TOC.. I suspect you used the Heading 1 style for your title. Try, instead, using the Title style. You can modify the style to appear the way you want.

This may be a weird question but, can you use Excel on your computer? When you copy and paste content from Excel and some other programs into Powerpoint, you're embedding an OLE object into Powerpoint. The OLE object has a "client" (the program that makes use of the object ... that'd be PowerPoint) and a "server" (the program that supplies the object to PowerPoint).

If the program that created the object in the first place isn't installed on your computer, you'll get a Server Not Found message when you doubleclick the object in order to edit it.

If you are getting this error then Power Point cannot connect or access Excel. Try repairing Office first. If the issue persist, you may need to reinstall Office.

cristin yan say:

Hi,

 

If you really want to use CTRL, the ListBox for ActiveX Control can meet your needs.

 

How to add a ListBox (ActiveX Control)?

Click the Developer tab >> Insert >> ListBox (ActiveX Control)

 

If you can not see the Developer tab, please do the following steps first:

Right-click the ribbon >> Customize the Ribbon >> check Developer

 

How to add a column data to the added ListBox?

e.g, there are four data rows in column A:

──

 A

──

 5

 8

 6

 3

──

 

Press [Alt+F11] to open the VBE Environment, and then paste the following code to a Module (Insert >> Module)：

 

Sub AddRangeData2ListBox()
    Dim arrList
   
    arrList = Range("A1:A4")
    ListBox1.List = arrList
End Sub

 

{5, 8, 6, 3} will add to the ListBox after you run the macro above.

 

How to set the selection type of the added ListBox?

Go back to Sheet1, click the Developer tab >> Design Mode >> Properties >> select the added ListBox >> MultiSelect >> 2 - fmMultiSelectExtended >> Design Mode

 

Note:

In this way, every time you open the Excel document, the list is empty, to keep the original data, you should add the Workbook_Open event in ThisWorkbook like below:

 

Private Sub Workbook_Open()
    AddRangeData2ListBox
End Sub
