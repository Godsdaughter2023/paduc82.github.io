dale howard [mvp] say:

Wiki Questioner --

Excellent questions, my friend!  To accomplish what you want to do, you will need to create a master project and insert each of the projects as subprojects of this master project.  If you are using resources in your projects, I would also strongly recommend you create a shared resource pool file if you are not using Project Server 2013.  Otherwise, you will see the names of your resources duplicated in the master project.

To create the cross-project links, open the master project and expand all of the subprojects.  Select a predecessor task in one project, press and hold the Control key, select the successor task in another project, and then click the Link Tasks button in your Task ribbon.  Repeat this for every set of cross-project links you want to set and then save and close the master project.

When you open one of the subprojects, you will see gray "ghost tasks" that represent the external tasks linked to tasks in the open project.  If you include both the Predecessors and Successors columns in any Gantt view, you will be able to see the name of the project for the external tasks.

Word of warning:  Keep the subprojects, the master project, and the shared resource pool file (if you elect to use one) in the SAME FOLDER and do not move them.  Also, tell your PMS not to do any File > Save As procedures with any of the subprojects as well.  Hope this helps.

ken sheridan say: Here's an example from one of my applications of generic error handling by branching:

Private Sub cboStrength_AfterUpdate()

    On Error GoTo Err_Handler

    ' assign selected unit to txtUnit control
    Me.txtUnit = Me.cboStrength.Column(1)
    ' requery cboDrugForm control to show forms
    ' for selected drug/strength/unit
    With Me.cboDrugForm
        .Requery
        .SetFocus
        .Dropdown
    End With

Exit_Here:
    Exit Sub

Err_Handler:
    MsgBox Err.Description & " (" & Err.Number & ") ", vbExclamation, "Error"
    Resume Exit_Here
    
End Sub

If you don't want to specify the error change the line which calls the MsgBox function to:

    MsgBox "Your error message goes here", vbExclamation, "Error"

Please show me how to insert some more lines in the code to make it stop without opening VBA windows in case of any errors, and then the message "Code error".

What is wrong with the examples you have already been given? The article Gina linked you to has a very good discussion of using the On Error command to handle when an error is thrown. If you use a MsgBox to display the error as shown, the Debug option will not be shown and the user can't enter into the VBE. 

If you want to handle specific errors, then the code snippet I gave you to use a Case statement will allow you to do different things on different errors. But you really should adjust your code to NOT error out instead. 

Finally, you might consider distributing your apps as an accde. Converting to accde compiles your code and then removes it, so a user can't enter the VBE no matter what.
