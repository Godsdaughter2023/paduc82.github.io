AppLicense

This works fine unless a field has no ticked check boxes - then the result for all fields is blank.

Is there any way around this? I wouldn't mind it If just the field with no ticked checked boxes came back blank, but I lose every thing. 

scottgem say:

You have a many to many relationship. One person can have multiple license and one license can apply to multiple people. So you need three tables:

tblPeople with PersonID and information about the people holding the license

tblLicenses with LicenseID and a field to identify what the License is for

tblPeopleLicense This is a junction table to model the M:M relationship. It has one table for each combination of person and license.

While a checkbox interface is easy for the user, its not proper table design. So you can use the checkbox interface, but with an unbound form that writes a record to tblPeopleLicense for each license. From there its an easy matter to use a Group By query to count the licenses.

barb reinhardt say:

I tend to define the chart names when I want to do something like this.   To do that, select the chart and click on the LAYOUT TAB.   You can change "Chart 1", etc. to the name you want

 

Option Explicit

Private Sub ChangeAxisScales()

Dim myWS As Excel.Worksheet
Dim myChtObj As Excel.ChartObject
Dim myCht As Excel.Chart
Dim myError As Long

Set myWS = Worksheets(1)

Debug.Print myWS.Name  'verifies you're using the worksheet you want

Set myChtObj = myWS.ChartObjects("Chart 1")

Set myCht = myChtObj.Chart

'assumes Worksheets(1) is also the active sheet

On Error Resume Next
myCht.Axes(xlValue).MinimumScale = myWS.Range("Y87").Value
myError = Err
On Error GoTo 0

If myError <> 0 Then
    MsgBox ("Error in change of Chart Minimum scale")
End If


End Sub
