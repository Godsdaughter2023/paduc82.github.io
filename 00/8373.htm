ossiemac say:

With multiple criteria in the same range it is necessary to sum each criteria separately and then add them together. Don't require SumIfs; just SumIf

Range("A4").Value = Application.WorksheetFunction.SumIf(CritRng, Crit1, SumRng) + _
                         Application.WorksheetFunction.SumIf(CritRng, Crit2, SumRng)

However, SumIfs will work but still need to treat each criteria separately and add them together.

Range("A4").Value = Application.WorksheetFunction.SumIfs(SumRng, CritRng, Crit1) + _
                        Application.WorksheetFunction.SumIfs(SumRng, CritRng, Crit2)

john w. vinson say:

You do indeed seem to need some more tables. Rather than just one big table for everything, you need to identify the "Entities" - real-life persons, things, or events - relevant to your application. Each kind of Entity should have its own Table, with fields for the "Attributes"  of that entity: discrete chunks of information which you need to know about each individual entity.

A Client is clearly an entity; he or she has a LastName, FirstName, contact information, other biographical data. A Session is a different kind of entity; each client will have zero, one, or many Sessions. Each Session will have a ClientID (as a link to the primary key of the Clients table), a SessionDateTime, and perhaps other information pertinant to this one particular session. And each Session would have zero, one, or many Notes, so you need a Notes table with fields for the SessionID (or possibly the ClientID if notes can be independent of sessions), a memo field for the note itself, and perhaps other fields such as the date and time that the note was recorded.
