  6. Click Finish. You can find the imported messages in the left pane, in a folder named Storage folders.

¡firedog say:

OK. Because both the old and new versions are the same build, you could save yourself a lot of work by replicating the old store folder and contacts databases on the new machine. The problems that people face when they end up in your situation usually arise from having different old and new versions.

I've not been able to test this, but it might work and it's worth trying.

 

If Windows Live Mail is up and running on the new machine, please export any mail messages you may have accumulated (sent, received, drafted) since setting installing the program. Similarly with any contacts you may have added, because we'll be trying to reproduce your data as they were at the time the old machine died.

  1. Close down all running programs and disconnect from the network.
  2. Navigate to the store folder and rename it to, say, OldWindows Live Mail
  3. Rename the Contacts folder at %localappdata%\Microsoft\Windows Live\ to OldContacts
  4. Copy those two folders from the old drive to take the place of the ones you renamed.
  5. Repair the installation: type wlarp into the start search box and press Enter. This should open the Uninstall/Repair dialogue for Windows Live Essentials. Select Repair...
  6. When it's finished, reboot the computer and launch Windows Live Mail.


With any luck, it should look more or less like it did on the old machine. There is one certain difference:  

for each mail account, right-click on its name in the folder pane and select Properties. On the Servers tab, remove any saved password and re-enter it (stored passwords are encrypted using a key derived from the user profile. This will be different on a different drive or a different machine).

 

The last four digits of the full version number represent the date the release was compiled, so seeing those Dates modified reflect this.

Incidentally, the version involved here is the latest available for Vista. For W7 and W8, the latest version is 16.4.3505.0912. I'd be surprised if you recently acquired a machine with Vista, so I wonder why you still have that version of WLMail.

You can create a stop and start script for Windows Update.

Create a Notepad file with each of the following and save them with the .bat extension.

stop-updates.bat:

sc config wuauserv start= disabled
net stop wuauserv

sc config bits start= disabled
net stop bits

sc config dosvc start= disabled
net stop dosvc

pause

start-updates.bat:

sc config wuauserv start= auto
net start wuauserv

sc config bits start= auto
net start bits

sc config dosvc start= auto
net start dosvc

pause

Now you can simply right click on stop-updates.bat and select 'Run as Administrator'

Source: http://sdbr.net/windows-10-update-failure/
