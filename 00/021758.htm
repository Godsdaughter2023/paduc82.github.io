saranbl say:

How to check a text file is open?

Hi,

I have a export VBA from a site. Don't remember where I got the code. I use this code to export to a text file. But if that text file is open this will not get saved. Is there any way to check if the file is open or not? If the file is open then check again till the file is closed and then save the data in the text file.

Public Sub ExportToTextFile(FName As String, _
    Sep As String, SelectionOnly As Boolean, _
    AppendData As Boolean)

Dim WholeLine As String
Dim FNum As Integer
Dim RowNdx As Long
Dim ColNdx As Integer
Dim StartRow As Long
Dim EndRow As Long
Dim StartCol As Integer
Dim EndCol As Integer
Dim CellValue As String


Application.ScreenUpdating = False
On Error GoTo EndMacro:
FNum = FreeFile

If SelectionOnly = True Then
    With Selection
        StartRow = .Cells(1).Row
        StartCol = .Cells(1).Column
        EndRow = .Cells(.Cells.Count).Row
        EndCol = .Cells(.Cells.Count).Column
    End With
Else
    With ActiveSheet.UsedRange
        StartRow = .Cells(1).Row
        StartCol = .Cells(1).Column
        EndRow = .Cells(.Cells.Count).Row
        EndCol = .Cells(.Cells.Count).Column
    End With
End If

If AppendData = True Then
    Open FName For Append Access Write As #FNum
Else
    Open FName For Output Access Write As #FNum
End If

For RowNdx = StartRow To EndRow
    WholeLine = ""
    For ColNdx = StartCol To EndCol
        If Cells(RowNdx, ColNdx).Value = "" Then
            CellValue = Chr(34) & Chr(34)
        Else
           CellValue = Cells(RowNdx, ColNdx).Value
        End If
        WholeLine = WholeLine & CellValue & Sep
    Next ColNdx
    WholeLine = Left(WholeLine, Len(WholeLine) - Len(Sep))
    Print #FNum, WholeLine
Next RowNdx

EndMacro:
On Error GoTo 0
Application.ScreenUpdating = True
Close #FNum

End Sub

FExnam = "Tracker"

    ExportToTextFile FName:="C:\Test\" & FExnam & ".txt", Sep:=";", _

appriver_dpetree say:

Giles

Your domain has a "MX" record that tells the world where to send email for that domain. You will have to log into your current DNS provider and change the MX record in your DNS to point to Office 365. This will mean that any email that is sent to any users on your domain will go to Office 365.

Once you do that, the users do need to configure outlook to connect to Office 365.

If you setup your CNAME in your DNS for the "Autodiscover" record, then when you go to setup the new profile in Outlook, the only thing it ask for is the users email address and password. Outlook will configure automatically.

So, to answer your question, Once you change your MX records, all users emails will go to Office 365. So all users need to connect outlook to Office 365

appriver_dpetree say:

P.S.

You do NOT need to change your Name severs. Where is your DNS hosted? You can leave it there if you are happy with it.

The only thing you need to do is just change the MX records for mail, and add the CNAME for autodiscover record.

Now if you are using Lync (Skype for business), you can add those DNS records too

But you do NOT need to change Name servers.

Why does the new meeting check box "All day event" change the - Microsoft Community

riversofbelief say:

Why does the new meeting check box "All day event" change the free/busy status from Busy to Free? Can that be disabled?

Why does the new meeting check box "All day event" change the free/busy status from Busy to Free?  Here's why it matters.

- Causes meeting room "resource" conflicts
- Causes blocked time on calendar not to be blocked at all, causing what seems like random Outlook scheduling issues

Open a new meeting.  The default state for your status is "Busy".  You go about setting up the meeting.  Date, message, attendees, got it.  Then you check the box for "All day event" because it is an offsite meeting or an all day course or a customer site visit, or whatever.  If you don't pay attention, you will miss the fack that as soon as you checked the all day event box, the Free/Busy status CHANGED to "Free".  You save the meeting thinking you have that time blocked out, but you don't.

Consequence?  People that look you up in Outlook schedule on top of that time because it is shown as Free.

That conference room you marked as "Resource" so it would be reserved.  It ISN'T, because the meeting says the time is Free.  Yes you marked it as a resource, but you also left the meeting as Free.  You just didn't know it because YOU didn't check that box, Outlook did it for you.  So you show up at your all day long customer meeting and walk in on another meeting in progress because when they scheduled their two hour meeting, theirs got scheduled with Busy. 

To the average person it is very frustrating to not know why the resource wasn't reserved, and the calendar wasn't blocked, as expected.  I paid attention and figured out what was happening, but it means that everyone has to pay attention to Free/Busy whenever they change the state of "All day long", and that means people will continue to have meeting and schedule conflicts, and to blame Microsoft, for what appears erratic behavior.

In fact, it is perfectly repeatable if you know what to look for, but it still looks like a Microsoft use case design error.

Is there a way to defeat this "help" so the check box doesn't automatically change Free/Busy, in a way that it corrects the issue for the entire organization at once?  If not, for an individual install at least?
daniel jackson say:

Hi,

This is the expected behavior in Outlook 2007, Outlook 2010 or Outlook 2013.
By default Outlook always creates an All Day Event as Free . 

When you check All day Event, meeting or appointment changes to an Event as it would last for 24 hours or longer. Events and annual events do not occupy blocks of time in your calendar; instead, they appear as banners. An all-day appointment displays your time as busy when viewed by other people, but an event or annual event displays your time as free.

You would probably have to create a custom form to change the default setting of Outlook to do this and may ask this in the Outlook Tech Net Forum:
http://social.technet.microsoft.com/Forums/en-US/outlook/threads

If you have any other questions related to Office products, reply and I’ll be glad to help you.
