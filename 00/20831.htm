Here is the fix to this.  Had the same thing happen, and after setting my DCOM settings, all issues were resolved.

Hello,

I finally found out what the issue is with this.  It actually isn't a "Protected view" issue, but a DCOM issue.

Somehow the DCOM permission got changed, so it wouldn't allow certain applications access to simple things like Preview mode on Outlook 2010 / 2007, or opening up attachments without it saying file corrupt.  I also was having an issue with Pop ups in IE8, where javascript messages about security tokens were not authenticated.

The solution is as follows, and then you can turn on your protected mode back on in word, excel, and your "Preview mode" in outlook will start working again.

Go to start, then run.  Type dcomcnfg and push o.k. Component Services should come up.

Expand Component Services, click on Computers, then "Right click" on My Computer and go to "properties".

Click on Default Properties tab, and make sure the following is set:

Default Authentication Level: Should say Connect, not "None"

And Default Impersonation Level: SHould say Identify.

When those are set properly, hit o.k and turn on your protected mode back on your Office applications.  Everything works as expected :)

What is the easiest way to merge a single row of data into a single - Microsoft Community

chris-k. say:

What is the easiest way to merge a single row of data into a single Word Form?

I have some long rows/records of client data in excel

I want to merge just one row of client data into one official document in Word format (several pages)

I prefer not to connect the spreadsheet and rather just copy/paste the single row of data somewhere else (blank back page maybe)

TIA

macropod say:
Unless you want to open the workbook, copy the data and paste wherever you want them to go, one cell at a time, the simplest way is to use Word's mailmerge tools to connect to the Excel workbook and place the mergefields wherever you want the corresponding data to appear. You can then use either filtering or a SKIPIF field to limit the merge output to the record of your choice. The SKIPIF field might be coded along the lines of:
{SKIPIF{MERGEFIELD Client}<> {FILLIN "What is the client name to use?" \o}}
where 'Client' is the name of the mergefield holding the client name/ID or whatever. When you Finish the merge and send the output to a new document, you'll have a new document with just the record you want and all the fields where you want them.

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues.

doug robbins - mvp office apps & services (word) say:

If you have a table with the first row containing names that could be used as the names for  DocumentVariables, and in that row, you inserted a Bookmark (say DataTable) to identify that table, and you were to paste the data to be displayed in those variables into the second row of the table, then if you run a macro containing the following code, it would populate { DOCVARIABLE [variablenamefromcellinfirstrowoftable } fields in the document with the relevant data from the second row:

Dim i As Long, j As Long
Dim varname As Range
Dim varvalue As Range
With ActiveDocument
    With .Bookmarks("DataTable").Range.Tables(1)
        For i = 1 To .Columns.count
            Set varname = .Cell(1, i).Range
            varname.End = varname.End - 1
            Set varvalue = .Cell(2, i).Range
            varvalue.End = varvalue.End - 1
            ActiveDocument.Variables(varname.Text).Value = varvalue.Text
        Next i
    End With
    .PrintPreview
    .ClosePrintPreview
End With

macropod say:

Is there no way to merge from a single row table within the same document?

Or is it possible to create hyperlinks to table cells so i can just paste a whole row of data to populate the form with new data?

Using a table in the same document as the datasource is not possible with mailmerge. In any event, the process I outlined even eliminates the need to copy the table from Excel.

Hyperlinking would be a lot of work, as you'd need to bookmark the contents of each cell in the table before being able to use it and would tie you to keeping the table in the document.

Presumably, though, your table has the same structure for all clients (e.g. ID, name, address, city, state, zip, etc.). In that case, if your document has bookmarks for each of these, you could use a macro like the following to populate those bookmarks and delete the table when done:

Sub PopulateDocument()
Application.ScreenUpdating = False
Dim i As Long, Rng As Range, ArrBkMk
ArrBkMk = Array("ID", "name", "address", "city", "state", "zip")
On Error Resume Next
With ActiveDocument
  For i = 0 To UBound(ArrBkMk)
    Set Rng = .Tables(1).Range.Cells(i + 1).Range
    With Rng
      .End = .End - 1
      Call UpdateBookmark(ArrBkMk(i), .Text)
    End With
  Next
  .Tables(1).Delete
  .Fields.Update
End With
Application.ScreenUpdating = True
End Sub

Sub UpdateBookmark(StrBkMk As String, StrTxt As String)
Dim BkMkRng As Range
With ActiveDocument
  If .Bookmarks.Exists(StrBkMk) Then
    Set BkMkRng = .Bookmarks(StrBkMk).Range
    BkMkRng.Text = StrTxt
    .Bookmarks.Add StrBkMk, BkMkRng
  End If
End With
Set BkMkRng = Nothing
End Sub

If the same data need to appear in multiple locations, simply use a bookmark for the first location and then cross-reference that for the rest.

Hi,

You said your code was a single line so you could modify it like this

 

Sub Somesub()
Activesheet.unprotect password:="xxx"

Selection.AutoFilter Field:=4, Criteria1:="TRUE"

Activesheet.protect password:="xxx"

end sub

Now when you call your code with CTRL+Y the worksheet is fleetingly unprotected, your code runs and the sheet is protected again. You have to change xxx to your worksheet password and note it is case sensitive.
