bill pytlovany say:

Many anti-malware programs make changes to this file assocation as a way to protect you from yourself.

 

The file types  .SCR and .REG were often abused and used to trick users into running what turned out to be malware.   Programs change your default program to notepad to protect you.

You'll notice most browsers will no longer run .reg files.  Instead they'll display the text of the script if it's a legitimate.  For a long time Outlook hasn't allowed you to download .SCR files.

 

WilPatrol will alert you to this change and allow you to change it back if you want but what ever malware program is making this change will keep trying to change it to notepad.

 

I usually recommend either using the Lock Filetypes feature in WinPatrol or just tell WinPatrol not to monitor .SCR and .REG and let your other security program handle it.

aaron - support engineer say: Hello Jrbisk,

I had noticed that in one of your earlier post after thge fact that I had already posted my comments.  Maximizing the throughput size for Network applications is not a bad idea.  By any chance, when you configured your registry with the MaxMptCt=255 reg key, did you install this hotfix http://support.microsoft.com/kb/937082?  It should be included in SP1. 

To configure the autodisconnect of inbound concurrent connections.  Simply, open the CMD (Run as Administrator)

type:  net config (server/workstation) autodisconnect:(Mins)

Example:  net config server autodisconnect:5

The command above means that after 5 mins of idle time, the server hosting the shared folder will disconnect the inbound connection freeing it up for another computer to possibly use that connection.  A Vista computer will remain connected to regardingless whether the connection is idle or not.  You would have to manually disconnect by typing

net session /delete

This will delete all of the inbound connections on the host computer.  This command will not affect any current document or interrupt the clients access to the share.  Only delete the inbound sessions on the server to ensure that 10 are available.  Below is some additonal information, I am sure will assist with resolution on this issue.


You must set the following registry entries on the server.
  • Set the value of MaxWorkItems to at least four times the value of MaxMpxCt. By default, MaxWorkItems is set to 64 in Windows Vista. You can set this entry to a maximum of 1,024. We recommend that you use this setting only on a computer that has sufficient memory. If you use the maximum setting, we recommend that you use at least 1 gigabyte (GB) of memory.
  • By default, the value of the Size entry is 1 in Windows Vista. Set the Size entry to at least 2. If you experience problems, set the Size entry to 3.
    • A value of 1 optimizes Windows Vista for local applications.
    • A value of 2 balances resource usage between local application and remote applications.
    • A value of 3 optimizes Windows Vista for network activity.
