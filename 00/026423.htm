jllatham say:

Eventually a variation of G_North's formula will work pretty nicely for you, and I'll get to that.  I want to talk about the lookup table(s) and also about data validation a little first.

People make mistakes, and for your data to be as correct as possible, we need to reduce the chance of human error in entering the Reasons and/or Events.  A typographic error during data entry could cause an error in the calculated End Date.  You can help prevent typographic errors in entering the Reasons and/or Events by using Data Validation in those cells.

 I would set up 2 distinct tables, 1 for Reasons and one for Events just as you have shown in your last example. These would be on a separate sheet from the 4-column group sheet(s).  This allows you to easily set up for the data validation in columns A and B in the 4-columns of information section.  What this is going to do is create a dropdown list with the names of Reasons from the Reasons table into cells in column A, and another dropdown list with the names of Events from the Events table into cells in column B.  Then your operators can select from those lists instead of typing if they want to, but still allow them to type entries and providing "spell checking" when they do that!

Go to the sheet with the tables on it and select all of the cells in the "Reasons" table with the description of the reasons in them and give that list a name.  You do this by typing a name into the "Name Box" and terminating it with the [Enter] key.  The Name Box is that area above the worksheet that normally shows you which cell is the selected/active cell.  If you select cell A1 on a sheet, it shows A1 in it.  If you select cells A2:A9, it shows A2.  With all of the reason descriptions selected, click in the Name Box and type in a name (no spaces).  Keep it simple and meaningful: Reasons would be a good name for this list.  Remember to end it with the [Enter] key.

Next, do the same thing for the cells in the Event table with the event descriptions; again, keep the name simple, and Events[Enter] would probably be a good name for them.

You have just created two "named ranges".  In Excel 2003 you can even manage these named ranges through the Excel menu bar with Insert --> Name --> Define.

(NOTE: we could even give the two tables with both columns in them separate names to make using the VLOOKUP() formula easier to understand -- but that's a lesson for another day).

OK, back on the sheet with the 4-column group now.  Select all cells that will hold a REASON entry and from the Excel menu choose  Data --> Validation and the data validation dialog will pop up.  Choose "List" from the dropdown below "Allow:" and down where it asks for Source: type in

=Reasons

and click the [OK] button.

Next choose all the cells in column B where you can have an Event entry and again use Data --> Validation and select "List" and this time for "Source:" type in:

=Events

What you have accomplished at this point is a method of reducing human data entry error and of assuring that your VLOOKUP() functions will work properly when needed!

Finally, we are ready to set up the formula to determine the End Date:  But this is independent of whether or not you chose to set up data validation as I just described.

I'm going to assume you set up 2 separate tables on Sheet2:
The "reasons" table goes from A2 down and over to B12 (labels on row 1)
The "events" table goes from A15 down and over to B25 (labels on row 14)

This formula assumes that if a Reason is provided, then the Event is not even a factor, but if no Reason is provided, then the Event is used to determine months of eligibility.

As I said, a variation of G_North's formula works very well, so in D2 you can put this formula:

=IF(AND(ISBLANK(A2),ISBLANK(B2)),"No Reason/Event Provided",IF(ISBLANK(A2),IF(ISNA(VLOOKUP(B2,Sheet2!$A$15:$B$25,2,FALSE)),"Error in Event Entry",DATE(YEAR(C2),MONTH(C2)+VLOOKUP(B2,Sheet2!$A$15:$B$25,2,FALSE),DAY(C2))),IF(ISNA(VLOOKUP(A2,Sheet2!$A$2:$B$12,2,FALSE)),"Error in Reason Entry",DATE(YEAR(C2),MONTH(C2)+VLOOKUP(A2,Sheet2!$A$2:$B$12,2,FALSE),DAY(C2)))))

where Sheet2!$A$2:$B$12 is the reference to the "reasons" table of data and  Sheet2!$A$15:$B$25 is the reference to the "events" table of data.

If you would prefer just to return the Begin Date as the End Date when neither a Reason or Event has been entered, you could change it to this:

=IF(AND(ISBLANK(A2),ISBLANK(B2)),C2,IF(ISBLANK(A2),IF(ISNA(VLOOKUP(B2,Sheet2!$A$15:$B$25,2,FALSE)),"Error in Event Entry",DATE(YEAR(C2),MONTH(C2)+VLOOKUP(B2,Sheet2!$A$15:$B$25,2,FALSE),DAY(C2))),IF(ISNA(VLOOKUP(A2,Sheet2!$A$2:$B$12,2,FALSE)),"Error in Reason Entry",DATE(YEAR(C2),MONTH(C2)+VLOOKUP(A2,Sheet2!$A$2:$B$12,2,FALSE),DAY(C2)))))

Format the cell as a Date.

The formula actually says something like this:

If both Reason and Event are blank, report that no reason/event has been entered (or just echo the Begin date), but if at least one of them is present, then check if the Reason entry is empty, and if there is no Reason provided, then use the Event entry to determine the End date, and finally, if a Reason was provided, use the Reason table to determine the End Date.  And while using either the reason or event table to try to get an end date, if there is no match in the table, tell the user that the entry is incorrect.

A simplified version of the formula may help you see this better:

=IF(AND(ISBLANK(A2),ISBLANK(B2)),0,IF(ISBLANK(A2),"Use Events","Use Reasons"))

The "Use Events" and "Use Reasons" sections get filled with the IF(ISNA()) complex formulas.

I hope this helps you with your effort.
