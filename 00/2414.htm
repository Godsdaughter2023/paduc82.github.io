Excel 2013 performance with the stronger algorithm is absolutely horrible.  With macro intensive workbooks the wait time for routines changed from "instantaneous" to "plenty of time for a coffee run".  

I understand that this change is "By Design", however, if you are going to do something by design which hinders performance so negatively, there must be other changes in design to mitigate the effects.  Microsoft has failed it's users and not even tried.  

Security is very important.  I will not argue that point.  It is good to see Microsoft trying to keep us safe.  I will give some suggestions on what could be done to make things easier for Excel Developers.

1. Before a protected cell in a protected worksheet can be changed, it must first be unprotected.  Macros are able to change protected cells without unprotecting worksheets by applying the "wSheet.Protect UserInterFaceOnly:=True" option.  A common practice is to apply this to all needed sheets at startup.  With the new more secure algorithm there is a delay added to each sheet this option is applied to.  If you are dealing with a large number of sheets the delay can be significant.  

My suggestion for this would be that the UserInterFaceOnly option STAYS APPLIED like most other options instead of being reset each time you close the workbook.   Why should be have to do this each time the workbook is opened?  

2. For most workbooks the password protecting each sheet is the Same.   Sheets should be able to be Grouped just like shapes.  This way if you have a workbook with 1000 sheets (Note: I do not have any workbooks with 1000 sheets - only making a point) they be grouped and share the Same password and properties.  Then you can just apply the UserInterFaceOnly:=True option to the group and incur only a single delay instead of 1000 which could take a few hours.  

3. In the case of a QueryTables.Add command UserInterFaceOnly does not work.  The sheet must be unprotected.  It seems like it could be an oversight.  I can't see any reason why this should be this way.  

These are my suggestions.  I hope that Microsoft will do a better job in the future with their releases.  In the 2010 release my only real complaint was their bad decision to remove the date picker.  This release was far worse. It is very disheartening when a release comes out which creates serious incompatibility issues.

7- Now set your target memory in your phone for messages.

8- Open Communication Centre and Select your desired folder then click on file > Import messages by selecting multiple messages *.vmg files. (You don’t have to import messages one by one but you can import all messages in a single folder at once)

9- And you have done it.

rich007a say:

Yep, I think that's a "feature".

 

The data labels are "fragile" and as soon as you do anything to their textframes (via VBA or via the GUI) their link to the data is broken.

 

There is a work around though...

 

If you click on a data label (you may have to click twice so that only one label is selected) then put the cursor in the formula bar and press = and then click on the cell which contains that value  (e.g.   =C3    ) then the links will not be broken when you run your code.  Or we can do that with code...  Try this:

 

Sub FixSeriesDataLabels()
    Dim i As Long, lnPntsCnt As Long
    Dim ser As Series
    Dim strFormula As String, strValAddr As String
    Dim vFormula As Variant
    Dim rngVals As Range
   
    Set ser = ActiveSheet.ChartObjects("Chart 3").Chart.SeriesCollection(1)
   
     'Get range of series values:
    strFormula = ser.Formula
    vFormula = Split(strFormula, ",")
    strValAddr = vFormula(2)
    Set rngVals = Range(strValAddr)
    
    lnPntsCnt = ser.Points.Count
    For i = 1 To lnPntsCnt
        With ser.Points(i).DataLabel
            .Formula = "=" & rngVals(i).Address(External:=True)
            If i Mod 4 = 0 Then
                .Format.TextFrame2.TextRange.Font.Bold = msoTrue
                .Format.TextFrame2.TextRange.Font.Fill.ForeColor.RGB = RGB(255, 0, 0)
            Else
                .Format.TextFrame2.TextRange.Font.Bold = msoFalse
                .Format.TextFrame2.TextRange.Font.Fill.ForeColor.RGB = RGB(0, 0, 0)
            End If
        End With
    Next i
End Sub

Hi Frans,

A few words of caution:

1) I wouldn't be comfortable applying formulas to a very large number of labels, I can imagine that slowing things down (presumably there is a reason Excel doesn't have this formula link by default!).  Although I would imagine that if you have "that many"

 

2) Be aware that my code above assumes there are no commas in both the sheet name and the series name.  The comma is used as the delimiter when splitting the series formula in to an array.  If you wanted to make this robust, I would add code to ensure there are never any commas in sheet names, and then to handle potential commas in the series name, you could look at the last-but-one element in the array:

 

=SERIES("My Series Name",Sheet1!$A$9:$A$11,Sheet1!$B$9:$B$11,1)    is split to:

=SERIES("My Series Name",
                                             Sheet1!$A$9:$A$11,
                                                                              Sheet1!$B$9:$B$11,
                                                                                                                1)

 

but:

=SERIES("My Series Name, with comma",Sheet1!$A$9:$A$11,Sheet1!$B$9:$B$11,1)    is split to:

=SERIES("My Series Name,

                                           With Comma",
                                                                    Sheet1!$A$9:$A$11,
                                                                                                     Sheet1!$B$9:$B$11,
                                                                                                                                       1)

 

In each case, the last-but-one element is the address of the series values.
