Immersive Control Panel items not shown in search (workaround - Microsoft Community

sveinan say:

Immersive Control Panel items not shown in search (workaround & behavior)

PS: All regular problemsolving actions like reindex, sfc.exe, Dism.exe, and more have been tried. Always open to other solutions, but please reproduce the specific problem described here. Then make sure the solution solves this specific case. Prove me wrong, but with material. Always happy to learn something new :) As of now, cannot find a good explanation for behavior below, except a bug in Windows 10.

Problem:

When searching for items in Windows 10 Settings app, or Start menu. Nothing, or only classic Control Panel items are listed. If 'nothing', then it is probably because indexing is turned off for relevant files. If only classic Control Panel items, then it looks to be an effect/feature/bug of turning indexing off/on for relevant files, after KB3105210 has been applied (as of 05nov2015). It is the last case, with only classic Control Panel items, this post is about.

Workaround:

- NOTE: Workaround is very short in its description of steps, you need to have certain Windows knowledge.

- NOTE: Uninstalling KB updates you do on your own risk. Usually goes ok, but can never be guaranteed.

- NOTE: KB3105210 is latest cumulative update (as of 05nov2015). It will be replaced/superseded in future.

- Disconnect LAN/WiFi/Internet connection (no unintended Windows Update)

- Uninstall KB3105210 update (View installed updates, classic Control Panel)

- Reboot

- Have 'Indexing Options' dialog open (Indexing Options, classic Control Panel)

- Turn indexing OFF for following location (folder/subfolders/files):

- C:\Users\<user>\AppData\Local\Packages\windows.immersivecontrolpanel_cw5n1h2txyewy\LocalState\Indexed\

- Wait for 'Indexing complete.' (in Indexing Options dialog box)

- Turn indexing ON for following location (folder/subfolders/files):

- C:\Users\<user>\AppData\Local\Packages\windows.immersivecontrolpanel_cw5n1h2txyewy\LocalState\Indexed\

- Wait for 'Indexing complete.' (in Indexing Options dialog box)

- Connect LAN/WiFi/Internet connection

- Run Windows Update to get KB3105210 update installed again

- Reboot

- Search in Windows 10 Settings app, or Start menu, should work

- NOTE: The important part here is to reindex relevant files without KB3105210 installed.

How to reproduce/provoke problem:

- NOTE: To keep this easy to reproduce with minimal variables, done in a VM with clean image

- NOTE: Media used was English Windows 10 x64 ISO build 10240 (i.e. 6846432 iso from MSDN)

- Ready a clean VM with Windows 10 USB/DVD/ISO media

- Install 'Windows 10 Pro' with defaults (i.e. English/US on all language options)

- Perform Windows Update until all updates installed (especially: KB3105210, as of 05nov2015)

- Search (Windows Settings app): "ease" (OK, 6 'new' items, 4 classic items)

- Have 'Indexing Options' dialog open (Indexing Options, classic Control Panel)

- Turn indexing OFF for following location (folder/subfolders/files):

- C:\Users\<user>\AppData\Local\Packages\windows.immersivecontrolpanel_cw5n1h2txyewy\LocalState\Indexed\

- Wait for 'Indexing complete.' (in Indexing Options dialog box)

- Search (Windows Settings app): "ease" (FAIL, no results at all, IS LOGICAL, no indexing)

- Turn indexing ON for following location (folder/subfolders/files):

- C:\Users\<user>\AppData\Local\Packages\windows.immersivecontrolpanel_cw5n1h2txyewy\LocalState\Indexed\

- Wait for 'Indexing complete.' (in Indexing Options dialog box)

- Search (Windows Settings app): "ease" (FAIL/BUG, only 4 classic items, NOT CORRECT)

- Disconnect LAN/WiFi/Internet connection (no unintended Windows Update)

- Uninstall KB3105210 update (View installed updates, classic Control Panel)

- Reboot

- Turn indexing OFF for following location (folder/subfolders/files):

- C:\Users\<user>\AppData\Local\Packages\windows.immersivecontrolpanel_cw5n1h2txyewy\LocalState\Indexed\

- Wait for 'Indexing complete.' (in Indexing Options dialog box)

- Turn indexing ON for following location (folder/subfolders/files):

- C:\Users\<user>\AppData\Local\Packages\windows.immersivecontrolpanel_cw5n1h2txyewy\LocalState\Indexed\

- Wait for 'Indexing complete.' (in Indexing Options dialog box)

- Search (Windows Settings app): "ease" (OK, 6 'new' items, 4 classic items)

- Connect LAN/WiFi/Internet connection

- Run Windows Update to get KB3105210 update installed again

- Reboot

- Search (Windows Settings app): "ease" (still OK, 6 'new' items, 4 classic items)

- NOTE: Temporary solution though. If indexing is redone OFF/ON, problem will be back.

- NOTE: Also, if new items/files added to .\Index\ area, will probably not show in Search.

Background:

First got these symptoms when installing Windows 10 Pro on a physical test PC I had. An old habit is to remove all file indexing C: disk (if still recommended, another discussion). Soon after I observed I was missing the 'immersive' Settings app entries when searching. Tried to go on the web. Lots of similar experiences, but no real solution. First thought it was connected to non-English regional settings (Norwegian) with English display language. Started testing with a clean VM install. Found behavior with standard English/US too, as I have described above.

Conclusion:

Can only wait to see if Microsoft acknowledges this problem. Maybe it's solved already in a future update. Or, by-design, reindex is not 'allowed' anymore ;) In any case. Hope this info is of use to some. At least I have my search results back, for now.

lemp say:
I'm not an expert, but that response seems a bit glib to me, especially given your observation that the process was only using 700 MB of memory.  In any case, with Windows managing virtual memory, the amount of virtual memory available (i.e., the page file size) will be automatically increased if necessary, as long as there is space available on the hard drive.

Take a look at the Performance tab of Task Manager.  The 3 numbers under "Commit Charge" show how the system is using memory.  "Total" is the amount of memory now being used.  "Limit" is the amount of memory (physical RAM plus page file) available.  With 4 GB of physical RAM, I would expect this to be 8 GB.  "Peak" is the maximum amount of memory (real plus virtual) used since the last system start.

How does "Peak" compare to "Limit" on your system?

The problem is probably related to the fact that a process running in a 32-bit operating system can only address 4 GB of memory.  Although a process can choke.  In that case, the only solution would be, as you were told, to simplify the model.

Or, of course, upgrade to Windows 7 or another 64-bit operating system.
