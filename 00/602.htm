john w. vinson say:

I'd (strongly!) suggest not using prompt parameters such as [Start Date]. Instead, create a little unbound form, frmCrit say, with textboxes like txtStart and txtEnd. You can then use

Parameters [Forms]![frmCrit]![txtStart] DateTime, [Forms]![frmCrit]![txtEnd] DateTime;
Transform.... <your crosstab query>

frmCrit could then have a command button to open the Report, or you can open frmCrit in the Report's own Open event; open it in dialog mode and put a command button on the form to set the form's Visible property to No to actually launch the report.

It's always permissible to use the Parameters declaration; with crosstab queries it is (for some reason) required.

Using a Form will prevent these multiple prompts, since the form is already open.

john w. vinson say: Base the report on the Query (using the form parameters), and open the Report. There's no need to open the Query at all; use it as the report's Recordsource and when you open the report, Access will use the query directly.

hansv mvp say:
Basically, it's creating a sheet of labels. See http://support.microsoft.com/kb/170130/en-us.

suzanne s. barnhill say:

If your version of Office includes Publisher, you'd be well advised to use it instead of Word, as Publisher allows you to easily create a single copy of your business card and then print it 10-up onto card sheets.

If that is not an option, then you need to go to Tools | Mailings | Envelopes and Labels. On the Labels tab, click Options... and then select the vendor and stock number of your business card sheets (such as Avery 5371) and click OK. If the card layout is simple, you can type it in the label window and choose "Full page of the same label." You can then either print a sheet of cards or use the New Document button to create a page that you can save.

If you need to do a little more fancy footwork with the card, use New Document button to start with. The sheet of cards is set up as a table, so make sure you have table gridlines displayed (Table | Show Gridlines) so that you can see the card boundaries. There are some limitations to what you can do in a table cell (another reason why Publisher is a better choice), but once you have your card the way you want it, copy the cell, select the entire table, and paste. This will paste the cell contents to all the cells.

If this doesn't give the desired results, there's a macro at the end of Graham Mayor's article at http://www.gmayor.com/graphics_on_labels.htm that will do the job.

tim liu msft say:

Hi Nalewis09,

Based on my test, if you are using work/school account (the account related to Office 365 for Business services, looks like name@domain.com or name@domain.onmicrosoft.com) and store notebooks in your OneDrive for Business library, after you share the notebook with others, they could open the notebook in OneNote Online. Click this button to open the shared notebook in their OneNote 2016 (users may also select OneNote for Windows 10 when they get the pop-up for opening app selection). If your notebooks are stored in Microsoft account (related to consumer OneDrive, looks like name@outlook.com or other email addresses), when user open the notebook with shared link in OneNote Online, there is no Open in OneNote button which can achieve your requirement.
