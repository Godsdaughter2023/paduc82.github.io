ken sheridan say:

Am I right in thinking that the code you posted in your original message outputs the report's to the PDF files correctly, but then falls over when it reaches the end of the recordset?  As a consequence you don't get the final message?  If so I can't see why the error cannot simply be trapped with On Error Resume Next before the line Me.Bookmark = .Bookmark.  This on its own should do the trick, though really you should examine the Err.Number value to take appropriate action if the error is other than the anticipated one.

As regards a 'more correct way' that's a matter of opinion, but in this sort of situation I don't think I'd step through the form by synchronizing bookmarks, but get the value of the Memo column in the recordset and pass this into a function as an argument.  The same function could be called by the command button on the form, in which case it would pass in the value of the Memo column in the form's current record.  The function would by and large be the same as the command button's current code, but would use the Memo value passed into it rather than referencing it with Me.Memo.

PS: The Description value would also be passed in to the function.
