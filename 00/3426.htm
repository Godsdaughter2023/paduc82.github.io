thedbguy say:

Hi,

First of all, let's try to walk through your logic. You want to use DLookup() to retrieve a "value" from a table. So far, so good. To do that, you ask the user to enter a Tail_Number that might be in the table. Okay. So, let's say the user entered the following number "163178." Now, you want to compare it to the table's data, so you try the following code:

TailNumberFromTable = DLookup("Tail_Number", "Platform_Status", "Tail_Number = TailNumber)

Okay, the first mistake is the syntax of the DLookup() statement. The syntax should be in the following form:

DLookup("FieldName', "TableName", "Criteria")

So, what's wrong with your syntax? First of all, the criteria portion is not enclosed in double quotes. To fix that, we might do this:

TailNumberFromTable = DLookup("Tail_Number", "Platform_Status", "Tail_Number = TailNumber")

That should take care of the syntax problem. However, the structure is now incorrect because we actually want to use the value "163178" instead of TailNumber. To fix that, we can change the expression to this:

TailNumberFromTable = DLookup("Tail_Number", "Platform_Status", "Tail_Number = '" & TailNumber & "'")

That's assuming TailNumber and Tail_Number are both Strings/Text data. So, after all that, Access will now be able to evaluate the code properly, and it might look something like this:

TailNumberFromTable = DLookup("Tail_Number", "Platform_Status", "Tail_Number = '163178'")

Great, now we're in business. But wait, what that expression says is retrieve the value of Tail_Number from the Platform_Status table "if" the Tail_Number is equal to "163178." So, if we already know the value we want to retrieve (by using the criteria "Tail_Number=163178"), then why do we need to use DLookup()? If the value 163178 exists in the table, then we get the same value back from DLookup(). If it doesn't exists, then DLookup() will return a Null value, which will result in an error because TailNumberFromTable was declared as a String, which cannot contain Nulls.

So, if the ultimate goal is to check if the value entered by the user exists in the table, then I recommend that you use DCount() instead. For example:

If DCount("*", "Platform_Status", "Tail_Number='" & TailNumber & "'") > 0 Then

'found it

Else

'not in table

End If

Hope that helps...

This is almost easy, ChrisHep!  However, the fact that you are putting this on two different sheets makes it more complicated.  I don't know of an elegant, few-click solution for putting the unique list on a separate sheet.  Here's how I would do it.  It is, admittedly, a bit brute force.

  1. Use an advanced filter to create a unique list of values on the current sheet.  Be sure to label your list with a heading at the top of the column. 
       * Select your data.
       * Then click Data on the menu bar, point to Filter->(over and down) Advanced Filter, and click.
       * In the Advanced Filter dialog box, click the radio button in front ofCopy to Another Location.
       * Click in the Copy to box, then another place on the current sheet in an empty column.  Try Column AA.
       * Be sure to click to check Unique records only.
       * http://support.microsoft.com/kb/262277
  2. Use the Copy and Paste Special functions to transpose.
       * Select the unique list that you just created.
       * Edit->Copy.
       * Click an empty cell where the first list item is to appear. You should also have sufficent empty cells to its right on the current row. 
       * Edit->Paste Special, click to checkTranspose at the bottom of the dialog box.
       * http://office.microsoft.com/en-us/excel-help/rearrange-transpose-data-from-columns-to-rows-or-vice-versa-HP010224502.aspx (works for 2003 as well).
  3. Duplicate the results on your second sheet.
       * Click on an empty cell on your second sheet.
       * Type the equals (=)sign and click on the first value in your newly created, unique list.
       * Press Enter.
       * Use the AutoFill handle to fill in the remaining values.
       * http://office.microsoft.com/en-us/excel-help/fill-data-automatically-in-worksheet-cells-HP010342512.aspx?CTT=1#BMaboutfillhandle

Good luck!  Please let me know how you work it out.

Gmail doesn't provide an Exchange server so you cannot use an Exchange account with gmail.  You need to use either a POP account or an IMAP account.  Use the Mail applet in Control Panel to remove the Exchange account from your mail profile and add either an IMAP or POP account.  Instructions are on gmail.

bj_701 say:

How do I fix or troubleshoot Search Companion which isn't saerching properly ?

My Search Companion isn't working properly. Even when I typed in search for 'Internet Explorer', nothing showed up. How do I fix or replace Search Companion ? Thanks.

a. user say:

Hi John,


Welcome to Microsoft community and thanks for posting the question.

 

To assist you better I would appreciate if you could answer the following questions:

 

1)     Have you made any recent hardware or software changes to your computer prior to the issue?

2)     Do you get any error code / message?

 

I would suggest you to perform the methods and check if it helps.

 

Method 1: Troubleshooting Search Companion in Windows XP

 

If the Search Companion dialog box is not displayed, press F3 to open it. If it still doesn’t display, then follow step 2.

 

Method 2: Test the issue in Safe mode

 

Check if you’re able to get the Search Companion in Safe mode. To do this, follow the steps below.

 

a.    You should print these instructions before continuing. They will not be available after you shut your computer down.

b.    Click Start and then click Shut Down.

c.    In the drop-down list of the Shutdown Windows dialog box, click Restart, and then click OK.

d.    As your computer restarts but before Windows launches, press F8. 

 On a computer that is configured for booting to multiple operating systems, you can press F8 when the boot menu appears.

e.    Use the arrow keys to highlight the appropriate safe mode option, and then press ENTER.

f.     If you have a dual-boot or multiple-boot system, choose the installation that you need to access using the arrow keys, and then press ENTER.

g.    Once you successfully boot into Safe mode, open Windows Explorer or My Computer and check if search companion works.

 

If it appears and works, then follow method 3.

 

Method 3: Perform a clean boot

 

A clean boot helps in eliminating software conflicts. Visit the link below to know how to perform a clean boot.

 

How to configure Windows XP to start in a "clean boot" state

 

http://support.microsoft.com/kb/310353

 

Note: Once the troubleshooting is over, reset the computer to start as usual.

 

Method 4: Reinstall the Search Companion Files

 

If the issue still persists beyond these steps, then re-install search companion files by following the steps mentioned below.

 

If you’re not able to get the search companion in any of the following windows listed below

 

·         Windows Explorer

 

·         My Computer

 

·         Microsoft Internet Explorer, then the files required to use Search companion is missing or corrupt.

 

I would suggest you to reinstall the search companion files.

 

a.    Log on to the computer by using an account with administrator permissions.

b.    Click Start, click Run, type %systemroot%\inf, and then click OK.

c.    Locate the Srchasst.inf file.

d.    Right-click the Srchasst.inf file, and then click Install. This reinstalls the files that Search Companion uses.

 

NOTE: You may need your Windows XP CD-ROM to finish installing Search Companion.

 

Additional information:

 

How to Configure the Search Companion in Windows XP

 

http://support.microsoft.com/kb/307980
