julie sheets say:

Pardon me for bumping in.  What you are seeing when assigning the task calendar to the task is common.  If there were no resources assigned, the task calendar would drive the scheduling of the task.  If, however, you have resources assigned to the task as well as attempt to add a task calendar, Project will schedule the task for working timein common between the task calendar and the resource calendar.  So if your task calendar is Sunday through Saturday (7 working days per week) and your resource calendar is Monday through Friday, the common working time is Monday through Friday.  If you wish to force the resource work work on the task calendar, select the option "Scheduling ignores resource calendar" on the Advanced tab of the Task Information dialog to the right of where you picked the calendar.

If you select that option, you should not need to alter the resource's calendar.

I hope this helps.

Julie

According to https://docs.microsoft.com/en-us/visualstudio/vsto/office-primary-interop-assemblies?view=vs-2019#separateassemblies, the interop PIAs are installed by Visual Studio, in a folder in the Program Files directory.
