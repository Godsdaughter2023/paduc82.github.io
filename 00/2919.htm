brian tillman [outlook mvp 2007-2019] say:

I don't know of any rule operation logging (although I agree there should be such a facility).  I'd disable each rule but the one I wish to test and see if the rules operate correctly when on its own by using a free web-based mail service like gmail to send myself test messages that include the conditions in the rule.  I'd then start adding more of the rules in iteration to see if there were unexpected rule interactions.  Eventually, you should see the unwanted effect and can determine why it's happening.

Do you use the "stop processing more rules" action on the rules that preceed the rules that perform the deletion?  If not, add that action.  Outlook applies all rules to each message and rules are evaluated in a top-down order.  If multiple rules apply to a message, the actions specified by those rules should all be executed.  So, if an earlier rule's action is to, say, move a meessage to a folder and there is no "stop processing" action and a later rule that deletes the message also applies, the message may wind up deleted.  Sometimes it's appropriate for multiple rules to ack on a message, but when you want exclusivity, you should include the "stop processing" action.

Brian Tillman [MVP-Outlook]
