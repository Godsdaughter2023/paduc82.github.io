jay freedman say:
That's correct. Word 2010 has two "Compatibility Modes", one for .docx documents created in Word 2007 (or based on templates created in 2007) and one for .doc documents from earlier versions. The main difference is that 2010 won't let you use features in those documents that aren't supported in the earlier versions. An example of that is the Text Effects button in the Font group of the Home tab in 2010, which didn't exist in 2007.

If you want to remove the Compatibility Mode, open the document in Word 2010 and then click File > Convert.

    It is really futile to try to explain the (mis)behaviors of Microsoft products.  It is what it is.

    We have learned over the years that when we change a cell from Text to any numeric format or vice versa, it does not alter the type of content.  That is ISTEXT(A1) is still true in the first case, and ISNUMBER(A1) is still true in the second case.

    No matter the reason, the point is:  when we do either of those format changes, we must "re-enter" the cell content.

    For an individual cell, the easiest way is:  select the cell (a single click will do), press F2, then press Enter in your case.  (Press ctrl+shift+Enter for a formula that must be array-entered.)

    Herb wrote:

    Next, Ctl-Tilde.  What this does is stretch all my cells out to an unmanageable width.  And still leaves me with a bogus entry in the target cell.


    Ctrl+` (back-quote, not tilde, although they share the same key on most keyboards) toggles Display Formula mode.

    If you had seen formulas in all cells, it might have been reasonable to think that you were in Display Formula mode inadvertently.  (It happens!)  So pressing ctrl+` would have toggled you
    Your observation confirms that you were not in Display Formula mode in the first place, since pressing ctrl+` put you into Display Formula mode.

    Hopefully you discovered that you only need to press ctrl+` again to return to normal mode.

    Herb wrote:

    I went back up to the formula bar, entered into it, swung all the way down to the end and hit {plain}Enter.  Now I have results.

    In another thread, someone demonstrated a method using SUMIF, actually published by someone.  I will need to study your situation more to see if and how it might be applied.  Or perhaps someone else will beat me to it.

    In the meantime, I would enter:

    =VLOOKUP(INDEX($D$6:$S$6,1,ROWS($V$6:V6)),'GL Trial Balance 6.30.12'!PT_Table,3,False)

    into V6, then copy (or drag) the formula down through V21 (16 in total).  Then use =SUM(V6:V21) where you had wanted to use =SUM(VLOOKUP(D6:S6,...)).

    Caveat:  If the VLOOKUP might fail and you do not require Excel 2003 compatibility, you might want to use the following to mask the error for SUM:

    =IFERROR(VLOOKUP(INDEX($D$6:$S$6,1,ROWS($V$6:V6)),'GL Trial Balance 6.30.12'!PT_Table,3,False),0)

    PS:  I use INDEX above instead of INDIRECT or OFFSET because even the latter two expressions might be easier to read and comprehend, those are "volatile" functions, whereas INDEX is not.  "Volatile" functions cause the formula to be recalculated every time any cell in any worksheet in the workbook is modified (edited or entered).  That can cause "inexplicable" delays in various operations.
