Please be informed that if an Alias is a Send only account, the alias is designed to attach an "On behalf of" text on the outgoing email since the account is using the servers of your primary account to send the message, which allows it to avoid being tagged as spam. To check this, please follow these steps:

  1. Go to your Inbox. 
  2. Click the gear icon next to your profile name at the top right corner of the page.
  3. Click More mail settings.
  4. Click Your email accounts under Managing your accounts.

If the alias that you are using is a third party domain, you need to configure it as a Send and receive account in your Outlook.com page so you can configure the settings and remove the "On behalf of" text on the email address.

 

If the address is a stand-alone Microsoft account like @live.com, @outlook.com, or @hotmail.com added as a Send only option, you will not be able to add it as a Send and receive option. The only way to remove the On behalf of text is to remove the account as an alias, delete its own record as a primary account, wait for the of 30 days recycle period to expire and so it will be available for registration or added as a new alias in your account.

burrwalnut say:

Somehow it doesn’t sound like a conventional hack (if there is such a thing as conventional hacking). Over the years, from various websites, I’ve compiled this list of some things that can be checked, I’ve assumed you’re using Windows 10):

1. First of all check if any rogue programs are running. Start Task Manager by pressing Ctrl+Shift+Esc then on the Startup Tab, right-click and Disable any entries that are unknown to you. Repeat this for the Services Tab, where you can check an entry online. If you disable anything restart the computer.

2. It’s probably not relevant these days but this only takes a few seconds to check. Open a Run window (Windows Logo key+R), type cmd and press Enter. Now type system.ini and press Enter. If under [drivers] there is an entry user=user.drv, you may have been hacked, so restart the computer and check again. An entry of timer=timer.drv is safe.

3. Now check the net statistics:

3A. Open a Run window (Windows Logo key+R), type cmd and press Enter. Now type netstat -ano and press Enter. If ‘Established’ is in the State column, make a note of the PID and the Addresses alongside it, as someone may be hacking you. If the IP Address begins with 192.168, you are safe as it’s part of your home network.

3B. To check if you are being hacked, open Task Manager again and go to the Details Tab.

3C. If the PID that you noted in 3A appears and it is not a name that you recognise, right-click it and End the task. You can click the word PID at the top to sequence the numbers to make it easier to find. Restart the computer and check again.

3D. If you didn’t find the PID, restart the computer and rerun the netstat –ano command. Open Google in a browser window and type the IP Address into the search box. If it’s a suspicious site, restart the computer and check again.

zorvek say:

Use the Application.OnKey method to run a specified procedure when a particular key or key combination is pressed. The syntax is listed below.

   Application.OnKey Key, [Procedure]

Key is a string indicating the key to be pressed and Procedure is a string indicating the name of the procedure to be run. The Key argument can specify any single key combined with ALT, CTRL, or SHIFT, or any combination of these keys. If Procedure is omitted then the default Excel function is invoked or nothing happens if Excel does not use that key or key combination - use this syntax to clear a previously defined OnKey setting. If Procedure is "" (empty text) then nothing happens when Key is pressed - use this syntax to prevent an Excel function from being invoked when that key or key combination is pressed.

To pass one or more parameters to the procedure include the parameters in the Procedure parameter as they would normally be entered in code. Separate the first from the procedure name with a single space, and separate each subsequent parameter from the previous with a comma and space. The entire Procedure parameter must start and end with a single quote when including parameters. For example, to set up an OnKey procedure call with two parameters - a number and a string, use the following syntax:

   Application.OnKey "{F1}", "'RoutineName 10, ""Text""'"

To call a routine in a worksheet module, preface the routine name with the name of the code name of the worksheet:

   Application.OnKey "{F1}", "SheetCodeName.RoutineName"

A common technique for setting up a keystroke intercept for a workbook is to do so in the Workbook_Activate event handler. The example code below, when added to the ThisWorkbook module, sets up the keystroke intercept when the workbook opens/activates and removes it when the workbook closes/deactiavtes. It is important to remove it or the keystoke will continue to be intercepted with other workbooks until the Excel application is quit and restarted.

Private Sub Workbook_Activate()
   Application.OnKey "{F1}", "RoutineName"
End Sub

Private Sub Workbook_Deactivate()
   Application.OnKey "{F1}"
End Sub

To specify combination keys use one or more of the following three special characters:

   SHIFT: + (plus sign)
   CTRL: ^ (caret)
   ALT: % (percent sign)

For example, to specify CTRL+a:

   Application.OnKey "^a", "Procedure"

To specify SHIFT+CTRL+RIGHT:

   Application.OnKey "+^{RIGHT}", "Procedure"

To assign a procedure to one of the special characters enclose the character in braces:

   Application.OnKey "^{+}", "Procedure"

To specify characters that are not displayed when the corresponding key is pressed (ENTER or TAB, for example), use the codes listed below. Each code represents one key on the keyboard.

   BACKSPACE: {BACKSPACE} or {BS}
   BREAK: {BREAK}
   CAPS LOCK: {CAPSLOCK}
   CLEAR: {CLEAR}
   DELETE or DEL: {DELETE} or {DEL}
   DOWN ARROW: {DOWN}
   END: {END}
   ENTER (numeric keypad): {ENTER}
   ENTER: ~ (tilde)
   ESC: {ESCAPE} or {ESC}
   HELP: {HELP}
   HOME: {HOME}
   INS: {INSERT}
   LEFT ARROW: {LEFT}
   NUM LOCK: {NUMLOCK}
   PAGE DOWN: {PGDN}
   PAGE UP: {PGUP}
   RETURN: {RETURN}
   RIGHT ARROW: {RIGHT}
   SCROLL LOCK: {SCROLLLOCK}
   TAB: {TAB}
   UP ARROW: {UP}
   F1 through F15: {F1} through {F15}

Note that all OnKey settings are ignored when a cell is being edited and when a user form is displayed and active. To intercept key strokes when a user form is active use the UserForm_KeyPress event procedure in that form's code.
