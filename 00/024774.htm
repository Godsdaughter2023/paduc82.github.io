How do you print a list of Worksheet Names? - Microsoft Community

cxc00 say:

How do you print a list of Worksheet Names?

I have a workbook with aprox 180 worksheets and I wish to print a list of all the worksheet names, is there a way to do this?
ron coderre say:

You'll need some VBA code...

• ALT+F11.....to open the Visual Basic Editor
• Right-click on your workbook name (listed in the "Project-VBA Project" window)
• Select: Insert Module
• Copy the below code and paste it into that module

Sub ListWkshtsOfWkbk()
Dim wks As Worksheet
Dim rngStart As Range
Dim intCtr As Integer
If MsgBox(Title:="PLEASE CONFIRM", _
      Prompt:="Ok to create the worksheet list beginning with the current cell?", _
      Buttons:=vbQuestion + vbYesNo) = vbNo Then
      Exit Sub
End If
Set rngStart = ActiveCell
intCtr = 0
For Each wks In ActiveWorkbook.Worksheets
   rngStart.Offset(RowOffset:=intCtr).Value = wks.Name
   intCtr = intCtr + 1
Next wks
End Sub

To run that macro...
• Select a cell with plenty of blank space below it in your worksheet
• ALT+F8....To show the Macros window
• Select: ListWkshtsOfWkbk
• Click: Run

You'll be prompted for permission to create the list...Click: Yes
The list will be created, beginning with the current cell.

Is that something you can use?

superservo15 say:

No worries, I just searched for "OneNote forum" and this is what came up.

Actually I stumbled upon the solution last night so I thought I would share.

 

IF for some reason the folder name has changed, if you use the OpenHierarchy to open the top level folder (that has changed) with the CreateFileType of ctfNone, then for any other section groups or files you are adding below it, the 0x80042006 is not thrown.

 

So the code I call before adding the section groups is:

 

onenoteApp.OpenHierarchy(change.CoursePath,

System.String.Empty, out xml, Microsoft.Office.Interop.OneNote.CreateFileType.cftNone);

 

Hope this might help anyone else :)
