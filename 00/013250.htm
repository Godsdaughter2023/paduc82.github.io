braintrove.com say: You will need to programmatically add the timezone information to the time field

XML stores Time in the following format:

hh:mm:ss

However, optionally (and not directly supported (i.e. cannot be directly added) by the InfoPath UI):

hh:mm:ssZ+hh:mm

where + can be either + or - and the final hh:mm is the timezone value you want to add.

For example:

10:03:24Z-07:00 for Mountain Time

You might be able to do  this with a lookup table, but it might be much easier by using code.




UPDATE / ADDITIONAL NOTE:

The InfoPath 2013 date/time picker will automatically format a timezone ONLY IF you do not manually set the time. HOWEVER, it does not include the "Z". I'm not sure if this has any effect. Perhaps it is not required (i.e. its assumed). An example XML value by selecting only a date in IP2013 is:

2012-12-05T00:00:00-07:00

Once I manually change the time, I get this:

2012-12-05T16:30:00

HOWEVER, if I modify the date again, I now get:

2012-12-05T16:30:00-07:00

So, it looks like Microsoft attempted to build this in, however, it appears to have an issue (perhaps intentionally) when you manually type in a time, where the timezone is dropped.

dianeoforegon say:
  • Open Outlook Preferences under Outlook in the Menu bar.
  • Select Sync Services
  • Select items you want to sync. Tasks is one of the options.

Outlook merges data when synced.


Sync means the data on one device or app is copied to another device or app so they are all the same regardless of where the data was added or deleted.

In a perfect world this would never result in duplicates or deletions. Sadly, sync is not perfect, especially when you mix platforms and products. Using all Apple applications or all Microsoft (using Exchange) are your best options. When you mix the two, is when you start seeing more issues.

For any data that is synced, you need to make frequent backups of your data in case you need to restore. Under File > Export select export as .olm file and uncheck all but contacts. Do the same for events. You might have a problem with contacts and not events so you need individual backups so you don't have to deal with duplicates.
