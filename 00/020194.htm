If an app or update that you recently installed caused problems with your PC, you can restore Windows back to an earlier point in time, called a restore point. System Restore doesn’t change your personal files, but it might remove recently installed apps and drivers.

This option takes your PC back to an earlier point in time, called a system restore point. Restore points are generated when you install a new app, driver, or Windows update, and when you create a restore point manually. Restoring won’t affect your personal files, but it will remove apps, drivers, and updates installed after the restore point was made.

  1. Right-click (or press and hold) the Start button, and then select Control Panel.
  2. Search Control Panel for Recovery.
  3. Select Recovery > Open System Restore > Next.
  4. Choose the restore point related to the problematic app, driver, or update, and then select Next > Finish.
Note:

ken sheridan say: You have incorrectly included the aggregation expressions in the GROUP BY Clause.  Also, you are using the column headings assigned to the aggregated columns in the computed column.  You can often get away with this, but I would recommend that you repeat the expressions.  Amend the query as follows:

SELECT [Date], [First Name],
SUM([Sum Of Eng Hrs]) AS [Sum Of Sum Of Eng Hrs],
AVG([Avg Of Hrs]) AS [Avg Of Avg Of Hrs],
SUM([Sum Of Eng Hrs])/AVG([Avg Of Hrs]) AS Expr1
FROM [Daily prod]
GROUP BY [Date], [First Name];

But, you should never use Date as a column name.  As the name of a built in function it is a 'reserved' word, and should not be used as an object name.  Use a term more specifically descriptive of the attribute, e.g. ProdDate.  I'd also suggest you use a more meaningful name for the computed column than the default Expr1 which Access assigns in design view.

suzanne s. barnhill say:

By default, headers use the Header paragraph style. If you apply direct font formatting to all or part of a header without changing the style, you may run into trouble if the template is set to "Automatically update document styles." And it's possible that even updating a field would cause a change. The \* MERGEFORMAT switch is supposed to protect the field from changes in formatting (that is, it purports to promise to retain the format you apply), but it's notoriously unreliable. The \*Charformat switch is usually more dependable; it picks up the format applied to the first character of the field, but of course if you've applied the same formatting to the entire field, then that's even better.

Note that you cannot expect a StyleRef field to pick up formatting from the text it references. Unfortunately (and maddeningly), the only font formatting a StyleRef field will honor is Hidden (the one format you might wish it would ignore).
