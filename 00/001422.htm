Try this macro as a worksheet macro on Sheet1:

Private Sub Worksheet_Change(ByVal Target As Range)
  If Not Intersect(Target, Range("A3:A7")) Is Nothing Then
    Worksheets("Sheet2").Range("A3:A5").EntireRow.Hidden = (Range("A3") = "No")
    Worksheets("Sheet2").Range("A8:A10").EntireRow.Hidden = (Range("A4") = "No")
    Worksheets("Sheet2").Range("A12:A15").EntireRow.Hidden = (Range("A5") = "No")
    Worksheets("Sheet2").Range("A17:A20").EntireRow.Hidden = (Range("A6") = "No")
    Worksheets("Sheet2").Range("A22:A25").EntireRow.Hidden = (Range("A7") = "No")
  End If
End Sub

eric_ruck say: Despite the Nokia phone support person being openly hostile and completely unhelpful, I figured out that the procedure they tell you to do doesn't install the drivers completely.

Despite the fact that I'm annoyed, here's how I fixed the problem:

Run the device manager (Start, Run..., type "devmgmt.msc" and Enter)

At the very bottom (or at least near the bottom) will be an item named "Wireless Communications Devices". Click the + and in the tree there should be something about "Nokia N95 USB Parent."

Double click to show the properties, and select the button "Update Driver...". Allow Windows to search the internet. This should take a couple of minutes and install all sorts of nodes for the different interfaces the phone supports.

When it's done try Nokia Software Updater again, it should work this time.

it_guy---01 say:

Connecting your USB cable to a motherboard USB slot cures many known device problems.

 

Avoid using Front or USB nested ports when updating.

 

It's to do with MAC addressing - firmware update software requires a valid MAC address to 'discover the device'  (should read "correctly and specifically identify the device") - a slave USB is not individually assigned a MAC address as such (it's done virtually in Windows).

 

The confusion comes because most software applications don't have this requirement so is able to manage your device and seems to work perfectly.
