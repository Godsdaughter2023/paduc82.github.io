abi99 (2) say:

Which server addresses do you use with Yahoo! Mail exactly? For example the server addresses imap.mail.yahoo.com and outgoing.yahoo.verizon.net do not support SSLv3 anymore. However, the E-mail client within Windows Phone 7.8 supports just SSLv3. Actually, this is a software bug created by Microsoft because at least TLSv1.0 should be supported as well. However since October 2014, Microsoft does not support Windows Phone 7 anymore. Therefore, to continue to use E-mail on a Windows Phone 7.8 you have to:

A) use an E-mail service provider, which still offers SSLv3, like FastMail (some collect E-mails from other servers), 

B) use an E-mail service provider, which offers Exchange ActiveSync (EAS; that is not broken in WP 7.8), or

C) access your E-mail via the Internet Explorer, because that supports TLSv1.0 as well.

You are able to test SSLv3 compatibility with the help of OpenSSL yourself:

openssl s_client -connect imap.mail.yahoo.com:993 -cipher DES-CBC3-SHA:EDH-DSS-DES-CBC3-SHA:RC4-SHA:RC4-MD5 -ssl3

If you do not use UNIX, download stunnel-installer-exe, which contains a command prompt to start OpenSSL on Windows.

Affected are Nokia Lumia 900, Nokia Lumia 800, Nokia Lumia 710, Nokia Lumia 610, Nokia Lumia 510, Samsung OMNIA 7 (Samsung i8700), LG OPTIMUS 7 (LG-E900) and many more…

2. Click Content type publishing > click the Url in hubs section

3. Click create > enter the name, choose the parent content type and group > OK.

4. Click on the newly created content type > Manage publishing for this content type > choose publish > OK

3. In the Email signature options page of OWA, paste the image in place.
