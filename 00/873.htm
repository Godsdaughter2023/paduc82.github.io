scottgem say:

well, using a combobox search will be quite a hassle because my main table (commandes2017) has a lot of fields and will have a lot of records. 

I want to make things easy for us by allowing to duplicate the records, and then just change the dates and the revision number if applicable.

The number of fields or records doesn't really matter here. Each record should be able to be uniquely identified by some value or combination of values. So that's what the user searches on. A combo is better for that.

If you are only changing dates and revision numbers then those should be in a child table. You should not be duplicating data. A primary principle of a relational database is that data should exist in one place and one place only. So you should have your master record as a sort of header record. Then if you need to add a new record with different dates and revisions, that should go in a child record with a Foreign Key to link back to the master.

The new Microsoft Skype for Windows 10 (Skype UWP) application does not have any option to delete chat history.

 

However, what you can do is to manually delete the main.db database file where the local chat history is saved.

 

Sign out of Skype UWP. This will shut the application down, so no processes are running in the background.

 

Navigate to the local Skype UWP profile folder.

 

https://answers.microsoft.com/en-us/skype/forum/skype_win10-skype_messms/skype-for-windows-10-chat-history-not-displaying/f09c65e4-d76e-4c30-8c4c-fba6c26a72d8
