Don't feel insulted. There was an explanation given for the change in direction. I don't have a direct link handy but if you search for video from the 15 February 2010 Mobile World Congress announcements by Steve Ballmer and Joe Belfiore (msp) you'll find an expanded explanation. Long story short Microsoft initially went in the direction of updating the UI of Windows Mobile and after burning some time on the project they decided it would never get them where they wanted to be. Windows Mobile did have a lot of flexibility. It was also the only mobile operating system that had certain certifications for security and other certifications of concern to businesses, enterprises, and power users. But power user's tend to be a smaller market than the general consumer. They decided to target an audience that was a bit more main stream; there are a lot of people being introduced to Smartphones that never had them before. So Microsoft decided to scrap the Windows Mobile 7 project, reorganize their resources, and start from scratch with 1.5 to 2 years left to create a new OS and get it on the Market in time for the 2010 holidays.

It doesn't have MMS, no custom ringtones, no in car navigation. What has happened? Even my PPC 2003 had mms, ringtones and sat nav. Bing maps is a feeble attempt to counter Google Maps, and Microsoft have taken away the IE features that Google Maps needs to run, so I don't see Google Maps ever coming to WP7.

Taken away doesn't properly explain what happened. With each new version of Windows Mobile it was possible to take code and functionality that had been created for the previous versions and include that functionality in a newer version. So the last version of Windows Mobile that we saw was the result of cumulative development over a period of about 10 years. Windows Phone isn't Windows Mobile. So the case for many of the features that are "e;missing" the reason isn't that they were removed, but that they have not yet been developed.

There was no way to develop the functionality that Windows Mobile had accumulated over the past decade in a relatively short amount of time. Instead Microsoft decided to define the minimum functionality that the phone needed for launch and plans to add additional functionality with future updates (ex: all the phones have a compass/magnometer, but the functionality won't be available to developers until a later update). Unlike Windows Mobile, Windows Phone is updatable without the carriers and OEMs getting in the way. So every one will get the updates.

mike h.. say:

Hi,

There's nothing wrong with the code but I don't recommend selecting sheets and ranges, you will be able to do what you want without selection.

I suspect it's going wrong because you have the code in the wrong place, it must go in a 'General' module.

Do it like this.

ALT+F11 to open VB editor. Right click 'ThisWorkbook' and insert module and paste your code in there and run it.

That's the whole point, you can't select another worksheet or range from code unless it's in a general module.
