Referencing a bookmark in more than one location

For example, the information the user inputs into textbox 1 is inserted where I have bookmark 1, but I want that information inserted into bookmark 1 on page 1 and also on page 2, 3, 4, 5, etc. I also need to do the same for the information in textbox 2, 3, 5, 8. etc. I am trying to make this template easier to fill out and I do not want the user to have to input the same information multiple times on a userform and Word is not allowing me to put the same bookmark in more than one spot.

jay freedman say:

There are multiple ways to repeat information throughout a document. A half dozen are described in Greg Maxey's article at http://gregmaxey.mvps.org/word_tip_pages/repeating_data.html.

I prefer to avoid bookmarks for this sort of document, because (a) it's too easy for bookmarks to be deleted or overwritten and (b) it's easy for a novice macro programmer to get the data inserted next to the bookmark instead of inside it, and then the REF fields don't see the data to repeat it.

Instead, I suggest the use of custom document properties and DOCPROPERTY fields, described in section 5 of Greg's article. Your userform puts the user's data into document properties -- e.g., one for "name", one for "age", one for each other piece of data. For example, if the Last Name box on the userform is called tbxLNm, then this code will create a custom document property called LastName:

    ActiveDocument.CustomDocumentProperties.Add _
        Name:="LastName", LinkToContent:=False, _
        Value:=tbxLNm.Text, _
        Type:=msoPropertyTypeString

Then your template can have multiple instances of the {DOCPROPERTY "LastName"} field in various places to display the value.

After the code sets the values of the document properties, it needs to call ActiveDocument.Fields.Update to make the fields show the values.

There's a similar method that uses document variables instead of document properties, and DOCVARIABLE fields to display them. The code to create a document variable is a bit simpler:

    ActiveDocument.Variables("LastName").Value = tbxLNm.Text

There are other things you can do with these. For example, if a user should be able to restart the userform to edit the values (maybe they made a mistake the first time...), the userform's Initialize procedure can put the existing values of the properties/variables into the text boxes, so all the user has to do is make the necessary changes instead of re-entering everything.

hansv mvp say:

You could run the following macro. It will take a long time to process 30K rows though.

 

Sub MoveDuplicates()
    Dim wshSrc As Worksheet
    Dim wshTrg As Worksheet
    Dim r As Long
    Dim m As Long
    Dim t As Long
    Application.ScreenUpdating = False
    Set wshSrc = ActiveSheet
    Set wshTrg = Worksheets.Add(After:=wshSrc)
    wshTrg.Name = "Duplicates"
    wshSrc.Range("A1").EntireRow.Copy Destination:=wshTrg.Range("A1")
    t = 1
    wshSrc.UsedRange.Sort Key1:=wshSrc.Range("A1"), Key2:=wshSrc.Range("B1"), Header:=xlYes
    m = wshSrc.Range("A" & wshSrc.Rows.Count).End(xlUp).Row
    For r = m To 3 Step -1
        If wshSrc.Range("A" & r) = wshSrc.Range("A" & (r - 1)) And _
                wshSrc.Range("B" & r) = wshSrc.Range("B" & (r - 1)) Then
            t = t + 1
            With wshSrc.Range("A" & r).EntireRow
                .Copy Destination:=wshTrg.Range("A" & t)
                .Delete
            End With
        End If
    Next r
    Application.ScreenUpdating = True
End Sub

migrate from Google Mail - Microsoft Community

mc heffner say:

migrate from Google Mail

How do I migrate emails, folders, etc from google mail/calendar into O365?

diane poremsky mvp say:
Are you migrating to Outlook.com or a business Office 365 account and using Outlook 2016? (or any windows Outlook desktop version)

If you have a lot of mail in google, add the account to outlook profile and export to a pst. Calendar and contacts needs to be exported from the gmail.com interface as CSV files.

Add the new account to Outlook and import everything. The imported imap folders may have the wrong view - change it in View menu, change view and a I have a macro to fix permanently.

If you don't have a lot of mail, you could add the new account to the profile and drag mail to the folders in the new mailbox. It's not really workable if you have a lot of folders though. And you'll still need to export calendar & contacts to CSV and import.

coldwell shao msft say:

Hi MC Heffner,

 

To migrate the data such as emails and calendar from your Gmail account to your Office 365 account in Outlook for Mac. Please try the following steps:

 

  1. Add your Gmail account in Outlook for Mac via Google Preview.
  2. Add your office 365 account in Outlook for Mac as well, then drag the emails under your Gmail account and then drop them into your Office 365 account. You can also drag the calendar events under your Gmail account and drop them into the calendar of your Office 365 account.

 

Best regards,

Coldwell
