drchucky say:

 I have lost the microphone option to have the phone type what is spoken.  Is there a fix?

Hello Bruce -  The "microphone option" (little mic above the keyboard letters) you refer to "have the phone type what is spoken" is know as "dictation", although some others may refer it in the forums as "voice text" or "talk and text".  I can confirm that dictation is only supported for W10M phones in English (United States) and that unless that is your active language you won't see a microphone button in the candidate pane.

My sister had the same problem when upgrading her Lumia 830 to W10M and could not figure out why this was so.  Turns out that when you upgrade from WP 8.1 if you have your region and language set to anything else other than USA & USA ENGLISH you will not see the "dictation icon" (Cortana might also be disabled depending on the region).  You need to change the region and language setting in WP8.1 to English (United States) prior to upgrading to W10M.  In your case, to fix this you would need to do either one of the following:

1) Roll back your phone to WP 8.1 using WDRT (Windows Device Recovery Tool) and make sure to set the language to English (United States) and then upgrade to W10M.

OR

2) Completely wipe your phone to W10M factory settings and choose your language as English (United States)  during set up.

Regardless, which route you take, make sure to BACKUP ALL OF YOUR PHONE'S INFORMATION before doing so.  

Good luck!!

Specifically, the last two version updates have made a quantum leap in Defender’s level of protection against ransomware and zero-day threats. Here’s a synopsis of the recent changes that have been made in Windows Defender:

Block at First Sight (Seen):

Windows Defender can now immediately block suspicious or unknown files; and then automatically analyze a sample and generate a signature within a matter of seconds

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-antivirus/configure-block-at-first-sight-windows-defender-antivirus

https://blogs.technet.microsoft.com/mmpc/2017/07/18/windows-defender-antivirus-cloud-protection-service-advanced-real-time-defense-against-never-before-seen-malware/  

We can also use PowerShell to upgrade the default settings for the Block at First Sight feature:

You can increase the default Cloud Block Level by running one of these commands at the elevated PowerShell Prompt:

Set-MpPreference -CloudBlockLevel High

Set-MpPreference -CloudBlockLevel HighPlus

Set-MpPreference -CloudBlockLevel ZeroTolerance

And you can also increase the allotted analysis time by running this command at the elevated PowerShell prompt:

Set-MpPreference -CloudExtendedTimeout 50

Windows Defender Exploit Guard:

The exploit protection features that were previously provided by EMET are now integrated into Windows 10.

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/exploit-protection-exploit-guard

https://blogs.technet.microsoft.com/srd/2017/08/09/moving-beyond-emet-ii-windows-defender-exploit-guard/

Attack Surface Reduction:

We also have the ability to add Attack Surface Reduction rules with PowerShell in Version 1709:

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/enable-attack-surface-reduction

https://blogs.technet.microsoft.com/mmpc/2017/10/23/windows-defender-exploit-guard-reduce-the-attack-surface-against-next-generation-malware/

For example, here’s the first rule that I set up immediately by running this line at the elevated PowerShell prompt

Rule: Block JavaScript or VBScript from launching downloaded executable content:

Set-MpPreference -AttackSurfaceReductionRules_Ids D3E037E1-3EB8-44C8-A917-57927947596D -AttackSurfaceReductionRules_Actions Enabled

Then to add additional rules, we use the Add-MpPreference command:

Rule: Block executable content from email client and webmail:

Add-MpPreference -AttackSurfaceReductionRules_Ids BE9BA2D9-53EA-4CDC-84E5-9B1EEEE46550 -AttackSurfaceReductionRules_Actions Enabled

Controlled Folder Access:

Windows document folders are protected by default, and we can add this ransomware protection to additional folders, as well as whitelist trusted applications in order to allow them access.

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/controlled-folders-exploit-guard

https://blogs.technet.microsoft.com/mmpc/2017/10/23/stopping-ransomware-where-it-counts-protecting-your-data-with-controlled-folder-access/?ocid=cx-blog-mmpc

PUA Protection:

Windows Defender has actually been able to detect and block Potentially Unwanted Applications for some time now, but many people still don’t know that this feature is disabled by default and needs to be enabled by running this command line at the elevated PowerShell Prompt:

Set-MpPreference -PUAProtection 1

Then confirm that PUA Protection was enabled by returning the current state for PUAProtection:

$Preferences = Get-MpPreference

$Preferences.PUAProtection

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-antivirus/detect-block-potentially-unwanted-apps-windows-defender-antivirus  

That's enough to convince me that better days are ahead for Windows Defender, and there isn't much doubt that these new features will mitigate the risks associated with ransomware and zero-day threats.
