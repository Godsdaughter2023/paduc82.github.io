gord dibben say:

Give this macro a try.

 

Sub Negsignleft()
Dim cell As Range
Dim rng As Range
    'move minus sign from right to left on entire worksheet
    On Error Resume Next
    Set rng = ActiveSheet.Cells. _
            SpecialCells(xlCellTypeConstants, xlTextValues)
    On Error GoTo 0
    For Each cell In rng
        If IsNumeric(cell.Value) Then
            cell.Value = CDbl(cell.Value) * 1
        End If
    Next cell
End Sub

stephenk101 say:

According to blog of Microsoft Japan Outlook support team, KB3085522 from the latest patch tuesday also causes the same problem Google translation. We tested this on one workstation and indeed, the problem also occurs when KB3085522 is installed and KB3055041 is not.

Right, both September (KB3085522) and August (KB3055041) patches for Outlook 2010 cause this issue.  We had installed both patches, so we decided to remove both.  Each patch has potentially more than one .msp file - outlook-x-none.msp, and outlookintl-xx-yy.msp where xx-yy is the 2-digit international language and country code for the local installation.  Removing the outlook-x-none.msp patch is (in my experience) enough to correct the issue.  Here is the command we used to remove outlook-x-none.msp from both September and August.

msiexec /I {90140000-0011-0000-0000-0000000FF1CE} MSIPATCHREMOVE={BA8586B9-BC29-4E59-BA98-2DF7002F256E} /qn /norestart

msiexec /I {90140000-0011-0000-0000-0000000FF1CE} MSIPATCHREMOVE={48133637-4DD6-4E69-BFB8-10B96B0F72EA} /qn /norestart

We've done this for about 3000 devices, and it only requires closing and reopening Outlook to correct the issue (although the updates will usually require a restart if Outlook is open).

When rolling these back, Outlook will return to the latest installed patch version before these patches (July KB3054976 if you installed it).  If you image new computers from a captured .wim file, you may want to prevent August & September from redeploying, and predeploy July KB3054976 again so that they don't roll back to an update from a long time past.

Microsoft unofficially expects to resolve this issue in their November patch-Tuesday cycle.

Error 0xD000000C when activating Office 2016 for Mac

Applies To: Excel 2016 for Mac Outlook 2016 for Mac PowerPoint 2016 for Mac More...

You might see this error when you try to activate Office 2016 for Mac:

The system requires that you sign on to a valid account. The error code is: 0xD000000C

If you've signed into your Mac using a Network account such as Open directory accounts, then this error occurs because Office 2016 for Mac can't write your credentials into the keychain as it is "read-only."

Troubleshoot error code 0xD000000C on Office 2016 for Mac

  1. Make sure the internet is working on the Mac you're trying to install on. You can do this by opening your internet browser and going to www.microsoft.com. If the page loads, your internet connection is working.

  2. Are you using a network or local user account on your macOS? If you are signed in with a network account, sign out and sign back in with your local user account. Try activation again.

  3. If you don't have a local user account, you must create one:

    1. From the Apple menu, select System Preferences > Users & Groups.

    2. Select Click the lock to make changes, and type your password. Select Unlock.

    3. Select +, and then fill in the new account information, and then select Create User.

    4. Log into your local user account and activate again.
