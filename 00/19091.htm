Sub CopyToBottom()
 Dim rngToCopy As Excel.Range
 Dim LastRow As Long

 Set rngToCopy = Selection.Cells

 rngToCopy.Copy Destination:=Worksheets(2).Cells(LastRow + 1, 1)
End Sub

Click a cell in a subform datasheet and open a form. - Microsoft Community

flopbot say:

Click a cell in a subform datasheet and open a form.

I know this can be done. I hope it’s not too advanced for me.

 

Let’s say I have a main form[Frm Exhibit Database]and a subform[Tbl Exhibit Work Log subform].  The subform is in datasheet view for ease of reading, but I’d like the user to be able to click on any cell in the datasheet and open a form related to that specific record. Then, somehow, they’d click back out to view the datasheet again.

 

The problem is that I don’t really know SQL, visual basic, or how to build a macro.

 

My question:  Can anyone point to me easy to use (step by step) instructions on how to do this … or other advice. Thank you in advance!!

ken sheridan say: You are going to have to handle two issues here;

1.  When the popup form is opened you'll need to pass the current problem ID value to it and set its problem ID control's DefaultValue property to that value.

2.  If you navigate to a different record in the subform form while the popup form is still open you'd need to refilter the popup form and change its DefaultValue property to the new problem ID value.

My recommendation would be to eliminate the second issue by opening the popup form in dialogue more, which forces the user to close it before returning to the original form.  That then leaves you with the first issue, which you can solve by changing the code to open the popup form to:

Me.Dirty = False

DoCmd.OpenForm "Test Work Log Pop Up", _
    WhereCondition:="[Problem ID] = " & Me.[Problem ID], _
    WindowMode:=acDialog, _
    OpenArgs:=(Me.[Problem ID])

Then in the popup form's Open event procedure put:

If Not IsNull(Me.OpenArgs) Then
    Me.[Problem ID].DefaultValue = """" & Me.OpenArgs & """"
End If

The first line simply ensures that the current record in the subform is saved before opening the popup form, which is necessary if you've just inserted the record in the subform.  The next bit of code will open the popup form and pass the current problem ID value to it as its OpenArgs property.  This is then used to set the problem ID control's DefaultValue property.  Note that this property is always a string expression, regardless of the data type of the underlying field, so is wrapped in literal quotes characters as above.  Note also that a you are setting the property of a control you must have a Problem ID control in the popup form, bound to the Problem ID field.  This can be hidden, however, by setting its Visible property to False.

I'm not sure what you mean by "the new record isn't saved to the table".  A new record will only be saved in the popup form's underlying table when you enter data other than the problem ID value in the form after moving to an empty new record with code such as:

DoCmd.GoToRecord, Record:=acNewRec

in the 'Add New Record' button's Click event procedure.  If you are doing this, but getting an error when you close the popup form it may well stem from the absence of a problem ID value in the new record, which setting the DefaultValue property as above should deal with.

Ken Sheridan, Stafford, England
