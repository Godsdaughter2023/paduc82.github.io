albert d. kallal say:

Read this article of mine:

http://www.kallal.ca/RunTime/Index.html

 

The runtime installs to the EXACT SAME path and location as the full edition. This ALSO includes the 1000+ registry changes etc. (again, they are 100% the same as the FULL edition).

 

If you already have Access 2007 (full edition) installed, then thankfully installing the runtime edition does NOTHING at all (since you would not want to over write your full edition).

 

Keep in mind that the location of the runtime install and even the dll';s and location of msaccess.exe file for runtime is IDENTICAL  in location to the full edition of Access.

You CAN NOT install both runtime and full edition on the same machine. (only one or the other is allowed).

 

So AFTER you install access (either full or runtime), then you can copy your mdb, accdb files to that computer and click on them. This approach works for the runtime and thus no different connection exists between runtime and full edition – when you double click on a file then the Access application is launched based on the file extension. There is no special "connection" between your applications and the runtime or full edition of Access.

 

So after you install the  runtime, then you just copy the mdb or accdb files to that computer and double click on them to launch them.

 

So how Access full or runtime installs on your computer is much the same.

 

The path name to full edition of Access (and thus also the same to runtime) is thus:

C:\Program Files (x86)\Microsoft Office\OFFICE11\msaccess.exe

(the above assumes a x64 bit computer)

Thanks, John.  I did that and it works great.  Appreciate all the help.

In case others are interested, I’ve documented below instructions for implementing the solution in Project Server:

Create 2 custom enterprise fields in Project Server:

1_week_out

Entity: Task

Type: Date

Formula: Int(Date())+7

 

Out-of-Date Tasks

Entity: Task

Type: Number

Formula: IIf([% Complete]=100,1,IIf(Int([Finish])=Int(Date()) And [% Complete]<100,2,IIf(Int([Finish])<Int(Date()) And [% Complete]<100,3,IIf(Int([Finish])<[1_week_out] And [% Complete]<80,2,1))))

Graphical Indicators: 1= Green, 2= Amber, 3=Red.

Then check-out the enterprise Global and update a view with the new enterprise field, ‘Out-of-Date Tasks’. 

 

As a recap, here’s what I was solving for:

1.        If Finish is < today and % complete is not 100%: RED icon. Tasks that meet this criterion are clearly out of compliance and require immediate attention. 

 

2.        If Finish is within 7 calendar days of today and task is not =>80% complete: AMBER icon.  Plan owner should review these tasks and determine if action needs to be taken now vs. waiting until the task falls out of compliance.

 

3.        If task is 100% cmplt: GREEN icon. No action required.

How do I find WordArt Style 8 from MSWord2007 in MSWord 2010? - Microsoft Community

jean970 say:

How do I find WordArt Style 8 from MSWord2007 in MSWord 2010?

I can't locate a style 8 in MSWord 2010. I am a stundent doing a project that must be in style 8. Please help!

Thanks

suzanne s. barnhill say:

It will not be easy to get the exact same effect because (as you've seen) WordArt in Word 2010 has been entirely revised. Moreover, the Text Effects feature has been added so that you can get similar effects with ordinary text.

But here's a description of Style 8 in Word 2007 WordArt:

Fill Color: RGB 51/102/153 (in the hexagon of More Fill Colors | Standard, this is the second hex from the left on the top row)

Line Color: None

Shadow Style: 5, which is most comparable to Offset Diagonal Bottom Right

Font: 36-pt Times New Roman Regular

If you then apply Text Effects | Transform | Warp | Square, that's going to be about as close as you can get, I think. It's not perfect, however, as the two types of WordArt just aren't comparable.
