Hi,

Please answer the following questions.

- Is this computer connected to a domain network?

Method 1: Please follow the below steps to reset Proxy settings

1. Please click on "Start" Button, click "All programs", and click "Accessories".
2. Right click on Command Prompt and select Run as Administrator.
3. Type "netsh winsock reset" (without quotes) at hit enter.
4. Then type “netsh winhttp reset proxy “at the command prompt, and press Enter.
   Type the below command one after the other and hit enter.
   Ipconfig /flushdns
   Ipconfig /release
   Ipconfig /renew
5. Once the above command is completed please restart the computer and check if the issue is resolved

Method 2: Disable proxy server

1. Launch Internet Explorer, access the "Tools" menu and click on "Internet Options" from the dropdown menu.
2. Select the "Connections" tab.
3. Click on "LAN Settings" (if you use broadband connection) or click on the "Settings" button within the "Dial-Up and Virtual Private Network settings" (if you use dial-up connection).
4. Disable (uncheck) the check box that corresponds to "Use a proxy server" option, then click "OK" to confirm.
 
Method 3:  Release and Renew IP address

1. Press the "Windows" key and the "R" key on your keyboard to open the Run dialog box.
2. Type "cmd" in the text box and press "Enter" to open the command prompt.
3. Type "ipconfig /release" (without quotation marks) and hit "Enter" to release the IP address.
4. Type "ipconfig /renew" (without quotation marks) and press "Enter" to renew the IP address.
 
Method 4: Obtain IP Address automatically

1. From the Start menu, select Control Panel, or Settings and then Control Panel.
2. Double-click Network and Sharing Center.
3. From the list of tasks on the left, click Change adapter settings (in Windows 7).
4. For a wired network connection, right-click Local Area Connection, and select Properties.
For a wireless network connection, right-click Wireless Network Connection, and select Properties.
5. From the Networking tab, click Internet Protocol Version 4 (TCP/IPv4), make sure it is checked, and then click Properties.
6. Click Obtain an IP Address Automatically, and then click OK.

Hi Heinz,

The error PC/Device needs to be repaired 0xc000000e could imply that Windows can't find the System Files used for booting, the boot configuration data or the system files is corrupted, or the Disk File System has a bad configuration. For us to sort this out, please look into your System Configuration and see if the boot options are still present. 

  1. In the Cortana search, type in msconfig, and hit Enter.
  2. Once the System Configuration window opens, click the Boot tab.
  3. Check if both boot options are available.

If it's not indicated there, then try to locate Where Windows is installed. Follow these steps:

  1. In the Cortana search, type in cmd, and hit Enter.
  2. In the Command prompt, type in C: and hit Enter.
  3. Type in DIR, then hit Enter.
  4. Once you're inside the Directory, check for:

Program Files
System.sav
Users
Windows

   5. If you can't locate Windows inside the C: drive directory, you would need to check the other drives.

Once you've located where Windows is installed, perform the MBR repair (step 5 from the first troubleshooting I've suggested to you). 
Note: Change the drive letter when entering the MBR repair to the appropriate drive where you located Windows.

Example:

Windows was stored in D: drive, from:
chkdsk c: /f
bootrec /fixmbr
bootrec /fixboot
bootrec /rebuildbcd
bcdboot c:\windows /s c: 

change to: 
chkdsk d: /f
bootrec /fixmbr
bootrec /fixboot
bootrec /rebuildbcd
bcdboot d:\windows /s d: 

Please let me know how it goes so I'll be able to assist you further.

Regards.

vanessa yar say:

Thank you for the detailed response, Heinz. I've further checked on this issue and it appears that your boot files may have been moved to your Z: drive that's why it became your default boot drive. To correct this, please perform the commands below:

bootrec /fixmbr
bootrec /fixboot
bootrec /rebuildbcd
bcdboot c:\windows /s c: 
bcdboot z:\windows /d

Note that bcdboot c:\windows /s c: will move your boot files back to the C: drive and will make it as the default drive, then the bcdboot z:\windows /d will copy your boot files on your secondary drive. For more detailed information, you may refer to the following links:

Repair the boot menu on a dual-boot PC

BCDBoot Command-Line Options
