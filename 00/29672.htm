saltgrass say:

In my experience, the compression of a backup image is around 55% of the original used space.  This may depend on the type of files being compressed, and if you have files that compressed more than normal, it might look the way it does.

The -allCritical switch normally includes all partitions necessary to restore you basic system, which are a Recovery, EFI System and OS partitions.  On Legacy installs it would be the System Reserved, OS, and possibly Recovery if you had an extra one of those.

I do not know if using -include:C: and -allCritical would make any difference.  The utility will list which partitions are being backed up, so you can check.  I use WBadmin when I want to include my Large Recovery Image partition in the backup.

net stop bits
net stop msiserver
ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

rmdir C:\Windows\SoftwareDistribution\Download

When complete, hit Enter, then restart, then try updating again.
