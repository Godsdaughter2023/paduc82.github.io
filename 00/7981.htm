geoffrey chaucer say:

Hi David,

You are absolutely right. The editor used to produce a website is of no consequence on the ranking of the site.

I have learned the hard way to follow Google's SEO guidelines.

For the benefit of others, here are a few basic tips to optimize a website irrespective of its generator.

In that order:

1- Appropriate file names, i.e: use text names, not page1, page2, etc . . .

2- Appropriate page titles accurately reflecting content of each page.

3- Clear and explicit content.

4- Meta tag with accurate description of content unique for each page. Do not use a generic description tag on every page.

5- Judiciously chosen internal links name.

6- Create with Notepad a separate sitemap document named Sitemap.xml which lists each and every htm / html document on the site, and upload it on your site. Do not create a link to the Sitemap.xml on your web pages as it is addressed only at robots.

Here is a sample:

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"  
  <url>
    <loc>http://www.YourDomain.com/index.htm</loc>
  </url>

  <url>
    <loc>http://www.YourDomain.com./bolts-nuts.htm</loc>
  </url>

  <url>
    <loc>http://www.YourDomain.com/screws-nails.htm</loc>
  </url>

  <url>
    <loc>http://www.YourDomain.com/hammers-saws.htm</loc>
  </url>

  <url>
    <loc>http://www.YourDomain.com/special-sale.pdf</loc>

  </url>

( add tags in this format to cover every document in your website )

</urlset>

URLs in the list above must be accurate. If the website was created with Publisher 2003 or 2007 using an index_files folder, the URL of documents in that folder must include /index_files/ i.e.:

<loc>http://www.YourDomain.com/index_files/screws-nails.htm</loc>

( see: http://static.googleusercontent.com/media/www.google.com/en//webmasters/docs/search-engine-optimization-starter-guide.pdf )

7- Meta tag with appropriate key words unique for each page.

Cheers, David.

Geoffrey

yaqub k say:

Hi Larry James,

Thank you for posting your query on Microsoft Communities.

You might experience this issue for a number of reasons. Some common issues that can cause these problems are: Corrupted or incompatible drivers, Missing updates, Network connection settings, Hardware or Software problems or TCP/IP could be damaged or corrupted.

Please assist me with the following information related to this issue:

  1. What is the make and model of your computer?
  2. Are you using a wired or a wireless Internet connection?
  3. Do you receive any error message when you try to connect to Internet?
  4. Do you see any error or exclamation mark on the network icon that is on the bottom left of your taskbar? Check if you see any exclamation in Device Manager under network adaptors. Follow the steps: 
  • Press Windows Key + X.
  • Select Device Manager.
  • Expand Network adaptors.
  • Check if there is any error or exclamation mark.

Please follow the methods mentioned below:

Method 1: We need to check the status of the Internet Connection on your computer. Please follow the steps and let us know the status.

Follow the steps:

Step 1: Internet Connection troubleshooter.

a. Press the Windows Key + W on the keyboard.

b. Type troubleshooting in the search box and then press Enter.

c. Select Troubleshooting. Click on view all and select Internet Connections.

d. Click next and follow the on-screen instructions.

Step 2: Perform Network adapter troubleshooter.

a. Press Windows Key + W on the keyboard.

b. Type troubleshooting in the search tab and press Enter.

c. Click View all on left panel.

d. Click Network Adapter, new window will popup.

e. Follow the on screen instructions.

Step 3: Run a ping test and check if any packets are lost. Refer the steps.

a. Press Windows Key + X.

b. Select Command Prompt (Administrator).

c. In Command prompt, type the following command

Ping www.msn.com

d. Check if there are any packets are lost.

Method 2:

Step 1: One of the components of the Internet connection on your computer is a built-in set of instructions called TCP/IP. TCP/IP can sometimes become damaged or corrupted. If you cannot connect to the Internet and you have tried all other methods to resolve the problem, TCP/IP might be causing it.

Let’s reset the winsock entries and TCP/IP stack and to troubleshoot the issue further.

Follow the steps:

a. Press Windows Key + X together and select Command Prompt (admin).

b. Type the following command and hit Enter after each:

netsh winsock reset catalog (reset winsock entries)

netsh int ip reset reset.log hit (reset TCP/IP stack)

c. Reboot the computer and check.

Step2: Flush your computer's DNS cache.

a. From Start screen, type command prompt.

b. Right-click on Command Prompt and choose Run as Administrator.

c. At the command prompt, type the following commands. Press Enter after each command.

ipconfig /flushdns
ipconfig /registerdns
ipconfig /release
ipconfig /renew

Please reply us on the status of the issue to assist you further.

reynold050 say: Every time I restarted, the internet would not connect. I had to shut down, remove the power cord and the ethernet cable, and wait 15 minutes. During this time something reset. I would boot up and the internet connected. With the internet connection down, the Windows troubleshooter said "the local area connection doesn't have a valid IP configuration." Then I tried IPCONFIG/RENEW in the Cmd window, it said "unable to contact your DHCP server." Not much help. Luckily I found the a description of the cause on an Intel discussion site. It is the replacement driver installed by Microsoft. I have an Intel network card. During the upgrade to Win10, the Intel driver was removed and replaced by a Microsoft driver. You need to install the manufacturer's driver, which will push out the MS driver. Use Device Manager to select your card, right-click on it and select Properties. Look at the manufacturer and model no. of the card. Go to the manufacturer's site for driver downloads. Hopefully there will be a driver for your card listed for Win10. I installed the Intel driver and restarted, the connection to the internet worked instantly and perfectly.
