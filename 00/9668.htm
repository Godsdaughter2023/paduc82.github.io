Also, newer versions of Access are less forgiving of coding anomalies than older ones. For example, this code works in 2003:

DoCmd.OpenReport "YourReportName", acViewPreview, "SomeField=SomeValue"

But it won't work in 2007 or higher, since the criteria code is entered in the "Filter" argument element, and not the "Where" argument.     kingpin007 say:

    You said you have installed about 300MB of sis files.

     

    Sis files are much smaller in size before it is installed. Some applications also carries a huge amount of database which the size is not shown on the used space.

     

    For example a dictionary application may state that the sis file is around 500KB but when you install it it will take up about 800MB and the word database can go up to 1GB.

     

    The 1GB will fall under the other files and it sometimes cant be found if you browse thru the App Mgr.

     

    Prehaps you can try to reformat your memory card and try reinstalling one application at a time to really see which one is taking up all the space.
