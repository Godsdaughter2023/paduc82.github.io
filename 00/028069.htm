It should be noted that a named range of the form OFFSET(A1,0,0,COUNTA(A:A),31) is probably the worst of all formulas that you can use.

The use of OFFSET will cause the formula to be recalculated every time you edit any cell in any worksheet of the workbook, as well as any other time a recalculation occurs.  And the full-column range A:A causes Excel to look at 1+ million rows each time OFFSET is recalcuated.  Finally, every cell where the named reference is used is also recalculated.

That can add up to serious editing and recalculation delay.

It would be better to use a formula to calculate the last-row number (probably using MATCH) in a cell (e.g X1; or a named reference?), then use an INDEX expression of the form $A$1:INDEX($A:$A,$X$1).

(Don't be misled by the reference to A:A in the first INDEX parameter.  Excel does not need to search all 1+ million rows; it just indexes into them.)

As for the exact form of the formula to calculate the last-row number:  that depends on a lot of details that you are not telling us.  To begin with, how is the Totals row identified and exactly where?  And what else is in the same column; in particular, any blank-appearing cells?
