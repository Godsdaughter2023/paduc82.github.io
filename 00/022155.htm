FORECAST.ETS.CONFINT - Wrong interpretation of a confidence interval?

What exactly is the function FORECAST.ETS.CONFINT calculating?

A confidence interval is the range in which we are confident that the true population parameter lies. For example, a 95% confidence interval is the range of values such that we are 95% sure the true population mean is included in that range.

A prediction interval represents the range where a future observation could be. For example, a 95% prediction interval is the range of values such 95% of possible future values will be included in that  range.

For reference, see Dr. Rob Hyndman's blogpost explaining this distinction. https://robjhyndman.com/hyndsight/intervals/

In Microsoft's explanation of the FORECAST.ETS.CONFINT function, they say that they are calculating a confidence interval, but that, for example, a 90% confidence interval will include 90% of possible future data.

VBA to find top 7 consecutive days - Microsoft Community

dttodgg say:

VBA to find top 7 consecutive days

Hello,

I have only dabbled with VBA, but this seems to be a perfect situation to use vba.

I have a spreadsheet with tons of data.
Column A has machine.id
Column B has date
Column C has revenue

I'm looking for which machine had the most revenue in a consecutive 7 day period.

Even if machine #3 only had 4 BIG days and 3 days with little or no revenue, it might "win" because the other machines had cumulatively less for any 7 consecutive days. The 7 days to not have to match.

I guess I'm looking for each machine's best 7 consecutive days - when they were (beginning date - ending date, and amount (total for 7 days)).
From that I can list the top x winners and when they occurred.

Thanks in advance!
lars-Åke aspelin say:
Some clarification about the date field and general file layout.

Are the dates in the column sorted, or may the order be arbitrary?
- The file will be sorted by Column A then Column B

Are all dates present? That means that the problem is to find the 7 consecutive cells in the column C with the highest sum.
- Some dates for a particular machine.id might be missing

Or may there be "gaps" in the dates in column B? That means that the number of cells in column C representing a 7 day period may be less than 7, maybe as low as just 1 cell if the adjacent dates are both more than 7 days away.
- YES, there can be gaps in the dates for a particular machine.id

Can the same date occur on more than one row? That means that the number of cells in column C representing a 7 day period may be more than 7.
- YES and NO. There is only one date associated with each machine.id, but since there are many machine.ids, there are many of the same date.

This file will come from a pivot table.
Column A will have the machine.id
Column B will have the date, but could be missing days
Column C will have revenue, but could be positive, negative or zero

Machine A   3/1/2012   $3.00
Machine A   3/2/2012   $2.00
Machine A   3/4/2012   $4.00
Machine A   3/5/2012  -$0.50
Machine A   3/6/2012   $0.00
Machine B   3/1/2012   $9.50
Machine B   3/2/2012   $2.00
Machine B   3/10/2012   $4.00
Machine B   3/11/2012   $2.00
Machine B   3/12/2012   $3.00
Machine B   3/13/2012   $1.00
Machine B   3/14/2012   $2.00
Machine B   3/15/2012  -$1.00
Machine B   3/16/2012   $3.00
etc...

Thanks again!



With you machine IDs in column A, dates in column B, and revenues in column C, on rows 1 to 14, and the machine ID for which you want the most revenues for a period of 7 consecutive dates in cell G1, try the following formulas:


For the maximum revenue:

=MAX(SUMIFS($C:$C,$A:$A,G1,$B:$B,"<"&($B$1:$B$14)+7,$B:$B,">="&($B$1:$B$14)))

Note: This is an array formula that has to be confirmed with CTRL+SHIFT+ENTER rather than just ENTER.

For the first date in the period with max revenu:

=INDEX($B$1:$B$14,MATCH(MAX(SUMIFS($C:$C,$A:$A,G1,$B:$B,"<"&($B$1:$B$14)+7,$B:$B,">="&($B$1:$B$14))),SUMIFS($C:$C,$A:$A,G1,$B:$B,"<"&($B$1:$B$14)+7,$B:$B,">="&($B$1:$B$14)),0))

Note: This is an array formula that has to be confirmed with CTRL+SHIFT+ENTER rather than just ENTER.

Change the $B$1:$B$14 in all places to suit the size of your actual data.

These formulas can be copied either down (if you have other Machine ID's in cells G2, G3, etc) or across (if you have other Machine ID's in cells H1, I1, etc)

Hope this helps / Lars-Åke
