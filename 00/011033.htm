Run-Time Error 3077 on Find a Record - Microsoft Community

steve459 say:

Run-Time Error 3077 on Find a Record

I have a drop down box to find records with the following code:

 

Sub Combo4_AfterUpdate()
    ' Find the record that matches the control.
    Me.RecordsetClone.FindFirst "[Product Code]=" & Me![Combo4]
    Me.Bookmark = Me.RecordsetClone.Bookmark
End Sub

 

All is great until I search for a record begining with "Y", "Z" or "WIZ".  Immediately get a runtime error 3077.  Totally stumped; can anyone help please.

 

 

ken sheridan say: As you are not wrapping the value of the combo box in quotes characters when building the string expression, Product Code is presumably a number data type, and the text value is in the second (visible) column of the combo box.  Can you guarantee that the form's recordset contains rows with all the values listed in the combo box?  If not, you should amend the code to account for this by testing for a match before synchronizing the bookmarks.  You should also allow for the possibility of the combo box being updated to Null:

    Const MESSAGETEXT = "No matching record found."

    If Not IsNull(Me.[Combo4]) Then
        ' Find the record that matches the control.
        With Me.RecordsetClone
             .FindFirst "[Product Code]=" & Me.[Combo4]
             If Not .NoMatch Then
                 Me.Bookmark = .Bookmark
             Else
                 MsgBox MESSAGETEXT, vbInformation, "Warning"
             End If
         End With
    End If
