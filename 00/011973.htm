daniel pineault say:

AfterUpdate is too late, the update has already been performed.  If you want to validate the data, you need to use the BeforeUpdate event.

Typically you'd do something along the lines of:

Private Sub Field2_BeforeUpdate(Cancel As Integer)
    If "*" & Me.Field2 & "*" Like "*" & Me.Field1 & "*" Then
        MsgBox "The data is invalid", vbCritical, "Error"
        Cancel = True
'        Me.Field2 = Null
'        Me.Field2.SetFocus
    End If
End Sub

Sub abc_01()
Dim WS As Worksheet, newWS As Worksheet
Dim X As String
Application.ScreenUpdating = False
Set WS = Sheets("Sheet1")

jllatham say:

Here is another possible solution.  The "limits/assumptions" are provided in the code as comments.  It automatically figures out how long the list of names is without user input (and how many columns are used), and will deal with duplicate names (such as two Brown or two Johnson entries in the list), example: you have two Brown entries, first sheet would be named [Brown], the second one for the second Brown would be named [Brown-2] and if there were 3 Browns, last one would be [Brown-3], etc.

Instructions for inserting either TasosK's or this code into a workbook can be found here:

http://www.contextures.com/xlvba01.html#Regular

The code:

Sub CreateRepSheets()
  'assumptions:
  '   Row 1 contains Labels: 1 for each column used
  '   List of names is in column A
  '    can change above by changing the
  '    Const values declared below.
  '   The sheet with list will be selected when
  '    this macro is run.
  '   List of names ends with empty cell after the
  '   last name in it and before the TOTAL entry row.
  '
  Const nameCol = "A"
  Const labelRow = 1
  Dim listWS As Worksheet
  Dim repWS As Worksheet
  Dim lastColumn As Long
  Dim lastRow As Long
  Dim nameColNum As Long
  Dim topLabelRange As Range
  Dim bottomLabelRange As Range
  Dim avgRange As Range
  Dim repDataRange As Range
  Dim rowPtr As Long
  Dim newWSName As String
  Dim dupeNumber As Long
 
  Set listWS = ActiveSheet
  lastColumn = listWS.Cells(labelRow, _
   Columns.Count).End(xlToLeft).Column
  'so we can reference name column by # later
  nameColNum = Range(nameCol & labelRow).Column
  'base last row on end of names list
  'will point to last row w/name in it
  lastRow = listWS.Range(nameCol & labelRow).End(xlDown).Row
  'set up some references to things we will have to copy
  'several times outside of the loop for performance.
  Set topLabelRange = _
   listWS.Range(listWS.Cells(labelRow, nameColNum), _
   listWS.Cells(labelRow, lastColumn))
  Set bottomLabelRange = _
   listWS.Range(listWS.Cells(lastRow + 1, nameColNum + 1), _
   listWS.Cells(lastRow + 1, lastColumn))
  Set avgRange = _
   listWS.Range(listWS.Cells(lastRow + 3, nameColNum), _
   listWS.Cells(lastRow + 3, lastColumn))
  'ready to begin copying data
  Application.ScreenUpdating = False ' improve performance
  For rowPtr = labelRow + 1 To lastRow
    'set reference to rep data to copy
    Set repDataRange = listWS.Range(listWS.Cells(rowPtr, nameColNum), _
     listWS.Cells(rowPtr, lastColumn))
    newWSName = Trim(listWS.Cells(rowPtr, nameColNum))
    'test if a sheet with this name already exists
    'if it does, we will add -# where # increments
    'for each duplicate name match.
    dupeNumber = 1 ' reset
    On Error Resume Next
    Set repWS = ThisWorkbook.Worksheets(newWSName)
    Do While Err = 0
      DoEvents
      'already have a sheet with the name!
      'try another name
      dupeNumber = dupeNumber + 1
      newWSName = Trim(listWS.Cells(rowPtr, nameColNum)) _
       & "-" & CStr(dupeNumber)
      Set repWS = ThisWorkbook.Worksheets(newWSName)
    Loop
    Err.Clear
    On Error GoTo 0
    ThisWorkbook.Sheets.Add _
     after:=ThisWorkbook.Sheets(ThisWorkbook.Sheets.Count)
    Set repWS = ActiveSheet
    repWS.Name = newWSName
    topLabelRange.Copy ' copy to clipboard
    repWS.Range("A1").PasteSpecial Paste:=xlPasteAll, _
     Operation:=xlNone, SkipBlanks:=False, Transpose:=False
    repDataRange.Copy ' on to clipboard
    repWS.Range("A2").PasteSpecial Paste:=xlPasteAll, _
     Operation:=xlNone, SkipBlanks:=False, Transpose:=False
    bottomLabelRange.Copy ' on to clipboard
    repWS.Range("B3").PasteSpecial Paste:=xlPasteAll, _
     Operation:=xlNone, SkipBlanks:=False, Transpose:=False
    'for the averages, we will copy the Values and
    'then copy the formatting
    avgRange.Copy
    repWS.Range("A4").PasteSpecial Paste:=xlPasteValues, _
     Operation:=xlNone, SkipBlanks:=False, Transpose:=False
    repWS.Range("A4").PasteSpecial Paste:=xlPasteFormats, _
     Operation:=xlNone, SkipBlanks:=False, Transpose:=False
    repWS.Range("A5").Activate ' an appearance thing.
  Next ' end rowPtr loop
  Application.CutCopyMode = False
  listWS.Activate ' back to list worksheet
  'cleanup
  Set avgRange = Nothing
  Set topLabelRange = Nothing
  Set bottomLabelRange = Nothing
  Set repDataRange = Nothing
  Set listWS = Nothing
  Set repWS = Nothing
End Sub

You can only map the range to an other sheet by the usual way: Pass the address to the range object.

Andreas.

Private Sub RefEdit1_Exit(ByVal Cancel As MSForms.ReturnBoolean)
  Dim Address As String
  'Get the address without any sheet name
  Address = Range(RefEdit1.Value).Address(0, 0)
  'Map to an other sheet
  RefEdit1.Value = RangeAddr(ActiveSheet.Range(Address))
End Sub

Function RangeAddr(r As Range) As String
  'Return a userfriendly address from R (usefull for hyperlinks)
  If InStr(r.Parent.Name, " ") > 0 Then
    RangeAddr = "'" & r.Parent.Name & "'!" & r.Address(0, 0)
  Else
    RangeAddr = r.Parent.Name & "!" & r.Address(0, 0)
  End If
End Function

All of the sudden today my Lexmark Pinnacle printer has started printing envelopes in red ink. I have made no changes from last week!
suzanne s. barnhill say:

To elaborate on what Charles has said, some inkjet printers, for reasons their manufacturers consider sufficient, use composite black (made up of the colors in the color cartridge) instead of true black (from the black cartridge) for envelopes and other smaller paper sizes (possibly because this is what is best for photo printing, and small formats may be assumed to be photos). When your "black" starts coming out red (or blue or green), it's usually a sign that the balance of inks in the color cartridge is no longer even, and you will soon need to replace it.

When printers are designed this way, there is no way to force them to use the black ink cartridge for smaller paper sizes other than to define the paper size as Letter or A4 and make up the difference with larger margins. Even in that case, the difference in feed method for smaller paper sizes may prevent you from deceiving the printer.

charles kenyon say:

bob jones aka: cybertaz say:

It depends greatly on exactly what is being copied & the source from which it was copied. It usually involves far more than just the visible content.

For example, if the source has 1" margins & the entire body of content is copied [including the ending ¶] the layout specs are copied as well. When you paste into an otherwise blank document, the margins of the destination document will be reset according to those specifications... Just think of the new document as being a lump of clay waiting for someone to give form to it. Especially if a Word document is the source, avoid copying the ending ¶ in a Section or at the very end of the document.

Those markers carry more information than just the location of a paragraph break. If you aren't in the habit of doing so, I strongly recommend having the nonprinting characters displayed when doing any editing.

daniel g. grau say:

You cannot delete files from within Word. You need to be in the Finder. Once there, select the file(s) or folder(s) to delete and use either of the following:

  • File > Move to Trash in the Finder's main menu
  • Press ⌘⌫ (Command Backspace)
  • Drag the files to the Trash can

In the Finder's window you can do multiple selections by:

  • Clicking and dragging to select a group of items
  • Pressing ⌘ (Command) and single-clicking on an item to add it to or subtract it from a selection
  • Clicking on the first item of a list, then ⇧ click (Shift-click) on another item to select a whole span
