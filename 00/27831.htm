marshall barton say:

Thar's good info and now I see that the form's bookmark is not changed when you go to a new record.  So that approach won't work and rather than mess with it further, let's go back to your original idea of using the DefaultValue property.  I think that will work if you add code to the form's Load event procedure:

   If Me.NewRecord Then
      Me.curCurrentBalance.DefaultValue = 0
   Else
      With Me.RecordsetClone
         .MoveLast
         Me.curCurrentBalanceDefaultValue = !curNewBalance
      End With
   End If

And in the curNewBalance text box's AfterUpdate event:

   Me.curCurrentBalanceDefaultValue = Me.curNewBalance

But, I have an uncomfortable feeling that there is something about what you are trying to do that just isn't right.  A new balance seems like it should be calculated by adding an amount to a previous balance.  That would make the current balance a calculated value that shoukd not be stored in a table.  And then both balances would be running totals that should also be calculated.  That is very easy to do in a report, but messy to do in a form.  OTOH, I think a tranactions form should only have data about a transaction and not have information about balances.  IOW, I think using a spreadsheet as a model is not a good idea.

This issue is due to the fact that Gmail also has a special 'All Mail' and 'Starred' folder in addition to your Inbox folder. When you connect to Gmail via an IMAP account, you can either unsubscribe from these folders or apply a filter to your To-Do List view.

Unsubscribe from the ‘All Mail’ and

  2. In the IMAP Folders dialog click on the Query button.
  3. From the folder list that is being retrieved select; [Gmail]/All Mail
  4. Press the Unsubscribe button.
  5. Press OK to close the IMAP Folders dialog.

Step 2: Apply a filter to the To-Do List

If you’d like to keep those folders in Outlook, then you can also apply a filter to the To-Do List to hide the duplicate items. The filters that you need to add are;

  • Field: In Folder
  • Condition: doesn’t contain
  • Value: All Mail

  • Condition: doesn’t contain §Value: Starred.
