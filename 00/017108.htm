there are two possible approaches to this: formula or formatting.

A formula like this will work: =IF(COUNT(W3:W4)=2,W4-W3,"")

Or with formatting, just use the formula =W4-W3 and format it with custom format

0;;;

Replace the 0 with 0.00 or any other number format you prefer. The formatting approach will hide all negative and zero values. If your data can produce negative values as the result of the formula, the formula approach may be better.

The correctness or problem depends on what might in column W and how robust (flexible) you want your formula to be.

Notice how your formula above returns the null string ("") under some conditions.  Since that is text, you could not reference that cell (X4?) directly in an arithmetic expression, e.g. =X4+Y4.  That might result in the #VALUE error if the value of X4 is the null string.

Likewise, if the value of W4 or W3 might contain a formula that returns the null string (or other text), your modified formula would return the #VALUE when evaluating the condition W4-W3<=0.

The COUNT formulas that Teylyn and I provided avoid the #VALUE error in that case.  The formula using OR(W3="",W4="") avoids the #VALUE if W3 and W4 only might be empty (no constant and no formula) or their value is the null string.

dan v [msft] say:

FIXED 1/15/16: Excel, Word or Outlook problems caused by a Windows 10 DisplayLink driver

Hi all,

Some of you have reported the following errors, which were caused by a bug in a DisplayLink driver update that shipped in an earlier Windows 10 update.

  • In Excel, you might see "Cannot quit Excel" and then... "The file is corrupt and cannot be opened."
  • In Word, when you open a legacy file, the file open in protected view and then crashes with error: "Microsoft Word (protected view) has stopped working."
  • In Outlook, when previewing attachments, you would see "The file cannot be previewed because there is no previewer installed for it."

The latest driver, which fixes the issue, is now available via Windows Update for Windows 10. To get the driver, do the following:

In Windows 10, select the Start button, select Settings > Update & security > Windows Update and click Check for updates.

Alternatively, you can download the driver directly from the DisplayLink site.

If you disabled hardware acceleration as a temporary workaround, you should re-enable it so that you don't experience performance issues with Office 2016:

  1. Open the Office 2016 app where you disabled hardware acceleration.
  2. Click File > Options > Advanced.
  3. Under Display, clear the Disabled hardware graphics check box.

Many thanks for your patience. If you're still experiencing issues, please start a new thread and let us know.
