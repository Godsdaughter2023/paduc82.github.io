jaap van ekris say:

Hi Kristof,

 

If you activate WiFi on a device it will automatically assume that you prefer it with respect to GPRS. So activating a WiFi connection will basically put the GPRS on hold in a low-power mode and all trafic will be rerouted through WiFi.

 

Having said that, there is an issue in practical applications which might be blocking for your specific application. When users are using automatic synchronisation options (like push-email or an interval based polling), the device will continue doing this even when it is asleep. When the device is asleep, it will put WiFi in a sleeping mode as well, but GPRS will remain in low power mode. Problem is that the WiFi connection basically is dead in the water, and GPRS is "only" in low power mode, with the side-effect that this kind of sync will be performed through GPRS.

 

If people perform the sync manually, then there is no problem because the device is in an interactive state at that moment.

wmai9999 say:

Office 365 activation failing on Windows 10

I'm getting an error message saying "we couldn't connect to the server. Please try again in a few minutes. 0x80072F8F" when trying to activate anyone's install of Office 365 2016 Business. I've tried every fix I could find online for this specific message. I've tried to activate on a different network, disabled the network firewall, spent hours on the phone with a Microsoft tech who tried a multitude of things.

We got it to activate on an old Windows 8.1 machine just fine. There has to be some conflict with Windows 10 and Office 2016. This is destroying my deployment rate of new Virtual Machines and the OS upgrade of our laptops. Please help!

Update 03/01/2018: 

So we changed a registry setting which was located in the: HKEY_LOCAL_MACHINE > SOFTWARE > MICROSOFT > WINDOWS > CURRENT VERSION > INTERNET SETTINGS > WINHTTP > DefaultSecureProtocol

 

We changed this value to 800. So it read like this: DefaultSecureProtocol 0x00000800 (2048)

This resolved the issue on one laptop but not the virtual machines. I will have to rebuild the ones that have stopped working. Now we have to activate Office before we join the new machines to the domain because sifting through dozens of Group Policy registry edits isn't on our list of things we can do right now.
