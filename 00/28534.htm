hansv mvp say:

If the customer has inserted a PDF file by clicking Object on the Insert tab of the ribbon:

  • Select the object (i.e. the icon).
  • Note the name of the object in the left hand part of the formula bar.
  • Press Alt+F11 then Ctrl+G to activate the Immediate window in the Visual Basic Editor.
  • Type

    ActiveSheet.Shapes("Object1").Visible = False

    where Object1 is the name of the object.
  • Press Enter.
  • Switch back to Excel.
  • If the user wants to unhide the object, do the same, but with

    ActiveSheet.Shapes("Object1").Visible = True

    using exactly the same name

jim gordon mvp say:

If Word, Excel or PowerPoint experiences a crash, on the next startup the app looks to find the autorecover file and tries to pick up working where it left off. When the file that the app is trying to open is corrupt and causes a crash, then an endless loop is created where the app crashes trying to open the autorecover file.

Holding the Shift key down says "don't try to recover" and that breaks the endless circle. 

I think once you do the Shift thing that cancels the app from attempting to load that particular damaged file in the future. There are lots of questions "where does Excel store my autorecover file" here in answers. Search for that topic if you want to manually find the damaged file and delete it.

Symbols replacing punctuation in body of email - Outlook 2011

In upgrading my exchange server, Outlook 2011, and Lion operating system two weeks ago, I am now getting strange symbols when I send emails from my Outlook (2011).  The periods and commas are being replaced by these symbols: � and Š.  Has anyone else experienced this problem?  Please advise how to fix this from happening.  The emails are fine when I compose them - and the symbols are showing up when they are received or in the body of the reply email.

talking moose say:
This is more than likely due to encoding issues.

I've found that Mac OS X will turn three periods in a row into an ellipsis. In other words, it's replacing the three single periods with a true single ellipsis character. When that is sent in the subject line through our Exchange 2007 server then it is getting converted to the Š character.

Checking the source of the message I can see the encoding is changed to a "Windows" encoding of some sort. This is an issue for either the sending or receiving Exchange server administrator to address.

If you turn off AutoCorrect for these characters then they will not be converted. Do this for Outlook under Outlook menu -->  Preferences... --> AutoCorrect  --> AutoFormat tab. To stop Mac OS X's replacement you can UNcheck Edit menu --> Substitutions --> Text Replacement while your cursor is blinking in the subject line or body of a message.

Hope this helps!
