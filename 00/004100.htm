Trouble Shoot Word

 

A General Starting point when Word does something unexpected

<snip>

As a general rule, when something seems to be going wrong, or at least not the way you want it to go, in Word “menu’ go to Tools and then Options, in Word “ribbon” go to File then Options, and study all of the various options, both turned on and off.

 

You may even get lucky and find some other settings that you want to try that have nothing to do with the current problem.

When I was teaching Word and other Microsoft Office programs at a private business college, I had three rules:

  1. Always check Tools Options
  2. When in doubt, right-click on something
  3. If you’re wondering how to set up a special kind of material in a document, consider whether Word tables are suitable. The placement officer of the school came to me several times and asked how I thought she should set something up. Each time the answer was, “I’d use a Word table.” By the fifth time she had stopped asking--she got the point.

</snip>

NOTE: part of this procedure is disabling addins. If you want to take a shortcut, check to see if you have a BlueTooth device driver or addin, ie “send to Bluetooth”, installed. Disabling BlueTooth is rapidly moving to the top of my Office trouble shooting process. Blue Tooth drivers have been identified as the cause of problems with many different “random” symptoms. If you have BlueTooth, disable it in control panel, device manager and possibly as an addin in Office apps.  It’s not that I think there is anything inherently bad with Blue Tooth. The problem is with the people writing the drivers.

marshall barton say:

Right!  You have some code in the Current event that is setting the value of a text box or other bound control.  This is not an uncommon mistake that forces Access to save the "New" record that is essentially blank.

If you are doing that by accident, then remove those lines of code.  If you are doing it to set a default value just in case a user wants to create a new record and you want to save them the trouble of having to enter a value that must be there, the change the code to set the control's DefaultValue property instead:

   Me.somecontrol.DefaultValue = """" & somevalue & """"

That is the proper way to set a default value because it will have NO EFFECT, until a user actually types a character into a bound control in a NEW RECORD.

If a user inadvertently enters a character into the new record, they can hit the Esc key to undo that entry and clear the record's Dirty flag (hit Esc twice if you already set the value of a different control).

If you have trouble locating the offending code, post a Copy/Paste back here and we'll see if we can spot it.

ken sheridan say: Further to Marshall's last response, be sure to always include the literal quotes characters around the value as Marshall has when setting the DefaultValue property of a control, regardless of the data type of the column to which its bound.  Surprisingly, it is especially important to do so with dates, as a date in short date format can otherwise be misinterpreted as an arithmetical expression and give the wrong result.

Table of Contents Page Number formatting to match document page number - Microsoft Community

ozgur (oz) balamir say:

Table of Contents Page Number formatting to match document page number format / style

Is it possible to match the page number format with the page number format in the document. i have two page number formats divided by a section breaks between. Contents, introduction, etc is in (i, ii, iii....) style and the main text page numbering is in (1-1, 1-2,...,3-4,...,4-5) style. I coudlnt get the table of contents to show this page style. It counts page numbers in (1,2,3,...) style
suzanne s. barnhill say:

What is the syntax of the page number (PAGE) field? Press Alt+F9 to display the field code. If it contains a format switch such as \*roman, the number will be displayed as roman numerals, but the underlying number will still be Arabic, and that's what will be displayed in the TOC. The only way to format page numbers to appear correctly in the TOC is to choose i, ii, iii in the  Page Number Format dialog.

In order to add the section number to the page number so that it will appear in the TOC, you have to use "Include chapter number," and the "chapter number" must be the paragraph number of a paragraph in one of Word's built-in heading styles to which automatic numbering has been applied.

suzanne s. barnhill say:

To avoid this problem, in your front matter, don't use the heading that provides the chapter number. Instead, use a user-defined heading identical to (but not based on) the built-in heading. So, for example, if you are using Heading 1 to provide the "chapter number," don't use Heading 1 in the front matter. Instead, define a Front Matter Heading style that has the same formatting as Heading 1 except for the numbering. You won't be able to include that in your page numbering, but I don't imagine you want to, anyway.

When assigning numbering to the built-in heading styles, it's easiest if you use one of the provided multilevel numbering schemes that are already linked to those headings, such as 1 Heading 1, 1.1 Heading 2, 1.1.1 Heading 3. For more on heading numbering, see https://shaunakelly.com/word/numbering/numbering20072010.html
