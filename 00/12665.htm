First, there is no need to use SUM.  Simply use the "+" operator.  That should eliminate the 30-argument error.

(I presume you are editing a ".xls" file in compatiblity mode, since your "applies to" line says you are using Excel 2007, which normally does not have a 30-argument limitation.)

Second, there is no need to test ISNUMBER($F73).  The PRODUCT function ignores non-numeric parameters.

So at a minimum, your example above could be written simply as:

=PRODUCT($F$73,$J$73)/100 + PRODUCT($G$74,$J$74)

Note that I moved "/100" outside the PRODUCT function.  It is arithmetically equivalent.

However, you might still encounter a "formula too long" error if your fully-loaded formula exceeds one of several applicable limitations, including a limit on the number of characters in a formula.

You can shorten your example formula as follows:

=N($F$73)*N($J$73)/100 + N($G$74)*N($J$74)

That assumes that F73, J73, G74 and J74 are never the logic value TRUE.

However, if you show us more of the arguments (and preferably all of them), we might be able to see a pattern that could lead to a much simpler formula.

(I wonder if you meant to write PRODUCT($G$74,$J$74/100), similar to PRODUCT($F$73,$J$73)/100.  Then ostensibly, it appears that you might be multiplying pairs along a diagonal.  But F*J/100+G*J/100+H*J/100+... does not allow for 30 similar arguments unless the terms can become ...+J*J/100+K*J/100+L*J/100+....  And you might avoid the repetitive J/100 simply by dividing the values in column J by 100.  For example, if column J contains percentages, calculate them as true percentages -- decimal fractions formatted as Percentage.)
