Hi,

It could be possible that the drive may have some errors that it is not saving.

  • Have you tried saving the data from any other Windows 7 machine (if available) to ensure it is working?

Windows Backup won't back up the following items:

  • Files stored on hard disks that are formatted using the FAT file system.

  • Files that are in the Recycle Bin.

  • Temporary files on drives smaller than 1 GB.

  • Program files (files that define themselves as part of a program in the registry when the program is installed).

You can try the following steps and check.

Step 1: At first, you may want to check the external drive for any error. 

 

a. Right-click on (My) Computer and select Manage.

b. Select Disk Management.

c. In the bottom window, right-click on the partition box, which is found to the right of the box containing the Disk number (ie, Disk 1).

d. In the dropdown menu, select Properties.

e. In the Properties menu, select the Tools tab.

f. Select Check now...

Important: While performing chkdsk on the hard drive if any bad sectors are found on the hard drive when chkdsk tries to repair that sector if any data available on that might be lost.

Step 2: Please refer to the suggestions provided in the article below to backup your files

Backing up your files

a. user say:

Just some Google searches on that message gets quite a few results and what seems to be the solution most of the time is to run a chkdsk with error correction on both the source and destination volumes.  Even if your drives are brand new.

Why don't you perform a Google search and read some of the results too so you can see how other people have solved the problem?

The bad part about that is it will probably take a long, long time depending on the size of the volumes, the amount of data on the volumes, the speed of your system and what the chkdsk finds to do...  and if it reports any issues that have been fixed I would run it again until it runs clean with no reported issues.

Sometimes if you can't figure out what the problem is, you need to figure out what the problem is not.

So you really need to be sure some data integrity issue is NOT the problem.  Your Google search on the error will reveal that a data integrity issue WAS the problem in the majority of cases and solved with chkdsk with error correction.

Something like this for each volume:

And you will have to wait while it checks your system drive where Windows is installed so be ready for that.

MS Backup is probably the worst choice you can make for keeping your data safe.

There are companies out there that are in the backup and restore business and that is all they do all day long and they usually have good technical support.

One popular alternative is Marcium Reflect...  After your chkdsk(s) are clean why not install the free version and see how that works.  If you don't like it, you can just uninstall it.

If you like it and you want to enable more features that are simply impossible with Windows backup you can buy it.
