daniel g. grau say:
It's not that the feature doesn't exist, you just did not find it. Do as follows:
  1. Select the paragraphs (triple-click on the first one and drag down)
  2. In the main menu bar, select Format > Bullets and Numbering
  3. Click on one of the numbering schemes
  4. Click OK

After that, renumbering will be automatic.

  • To change the list's indentation, select the numbered paragraphs and use the ruler or Format > Paragraph.
  • For a permanent change to that particular numbering format, click on the Customize button before doing Step 4.
  • To restart from 1, click on the paragraph that you need to be #1, then Right-click (or Control-click) and select "Restart Numbering" in the contextual menu.
  • If you repent, click on "Continue Numbering."

bob jones aka: cybertaz say:

One important point to bear in mind is that what you set as the Default Language pertains only to new documents you create... It has no bearing on existing documents. They will continue to open in whatever language formats are applied to the text they contain.

Try this;

  1. In a new blank document go to Tools> Language
  2. Select what you prefer
  3. Click the Default button & confirm updating the Normal template
  4. Quit/Launch the program, then go to Tools> Language -- What is indicated?

In an existing document you need to Select All, then apply the preferred Language setting. That setting should be retained, assuming you save the document after doing so. Are you saying that isn't the case?

Based on your description there is another factor that could be involved. In an existing document that contains Languages that differ from your default you must be sure to click OK rather than Close if you use the Default button. Otherwise, your Default will be updated & any existing 'normal' text based on your previous language default will be changed, but preexisting content that had been formatted to deviate from the default will not be reformatted. Example:

  • My default Language is English US
  • I have a document which is English US except for a passage formatted as French
  • I select all, go to Tools> Language & choose Bulgarian
  • I click Default & confirm
    • If I click OK my Default is changed to Bulgarian & all content is reformatted accordingly, but...
    • If I click Close the English US text is reformatted as Bulgarian but the French text remains French

Another point relevant to "while I am working"... Does 'working' include copy/paste from other sources? If so, that would explain the cause. The content you copy often carries its own language formatting as applied in the source. Not only is that formatting retained, but if you continue to type immediately following the pasted content the new text will inherit the same Language attribute as what was pasted. The issue can be avoided by using Edit> Paste Special - Unformatted Text.

Private Sub cmdDuplicateRecord_Click()

    Dim lngNewEmployeeID As Long
    Dim strSQL As String

        "(EmployeeID,FirstName,LastName,Department,City,DateAppointed) " & _

        "FROM Employees " & _
        "WHERE EmployeeID = " & Me.EmployeeID

In Word 2007, wavy blue underlines are used for two distinct purposes:

  • They may show where inconsistent formatting is applied.  This usage was introduced in Word 2003
    (http://office.microsoft.com/en-us/word-help/what-do-the-underlines-in-my-document-mean-HP005270413.aspx).
  • They may show "contextual" spelling errors, words that are spelled correctly but not appropriate for the context where they appear
    (http://office.microsoft.com/en-us/word-help/get-to-know-word-2007-i-create-your-first-document-RZ010227511.aspx?section=5).

In both cases they're the best guess of an inanimate program that doesn't "understand" what you wrote but is just following rules. If you're sure of your correctness, you can ignore the underlines. To turn off the features, open the Word Options dialog and:

  • On the Proofing tab, uncheck "Use contextual spelling".
  • On the Advanced tab, uncheck "Mark formatting inconsistencies".
