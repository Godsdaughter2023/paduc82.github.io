langware say:

Sylartsy,

The feature you are referring to is called "autocomplete".

It is set (by Microsoft) in the HTML code for the Sign-on page.

Previously, Microsoft had set the "autocomplete" attribute (in the old Sign on page) to "on". So when you typed the first character of your user name into the Sign on page, a drop down list was displayed showing remembered user names that matched that character typed into the input box.

Microsoft has released a new version of their sign on page ... and in the new version .... the "autocomplete" attribute is set to "off". Thus, no user names are being remembered.

Fixing this issue is simple for Microsoft  ... all they have to do is set the "autocomplete" attribute to "on". They have not done this, and they have refused to even explain why they have disabled the "autocomplete" attribute (i.e., they have not explained what problem was solved by disabling the "autocomplete" feature.

No one from Microsoft (with the authority to fix this problem) reads these forums. I suggest you use the following steps to report this to someone in Microsoft ...

1. Go to this page:  http://windows.microsoft.com/en-us/windows-live/id-support

2. Click on "I am having trouble signing in"

3. A page will open where you can describe the problem signing in (i.e., auto-complete not working when typing the first few characters of your ID into the new Microsoft Account sign in page).

4. Click the Submit button.

You will get email with a case number. Then you may get email with a scripted reply telling you to enable auto-complete in your browser. Verify that auto-complete is working in your browser by signing on to other sites you use (such as netflix, gmail, facebook, etc and typing the first few characters of your ID). Then respond to the email you received and tell them that auto-complete works on other sites ... it just does not work on the new Microsoft Account sign in page. Be persistent and eventually your case will get escalated to a higher level.

Since Microsoft has refused to set the "autocomplete" attribute to "on", I have written a script that will run when the Sign On page is opened, and will set the "autocomplete" attribute to "on".

 

If you use Firefox as your browser, and have installed the free Greasemonkey add-on (that allows user scripts to be run in Firefox), then install the following script which will fix the problem:

// ==UserScript==
// @author         langware

// @date           May 9, 2016
// @run-at         document-idle
// @grant          none
// ==/UserScript==


var elmLink = document.getElementById("i0281");

if (elmLink != null)
    {
    elmLink.removeAttribute("autocomplete");
    }
