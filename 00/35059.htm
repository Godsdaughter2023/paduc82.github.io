andre.ziegler say: Bug Check 0x101: CLOCK_WATCHDOG_TIMEOUT

The CLOCK_WATCHDOG_TIMEOUT bug check has a value of 0x00000101. This indicates that an expected clock interrupt on a secondary processor, in a multi-processor system, was not received within the allocated interval.

This can be a CPU issue or a chipset/BIOS issue. The problem is, that MSFT doesn't document those newly added Bugcheck codes inside the WinDbg documenation. So you are not able to see why this Bugcheck was raised.

Have you tried BIOS Update?

Hey r1975,

Build 1803 came out rather recently, since April 30th. This probably didn't give Microsoft enough time to push out these flash drives with 1803 straight out of manufacturing.

You can use the Media Creation Tool with the same license and everything would be alright, the license or product key is not build-specific.

As long as you use the same computer all the time with no significant hardware changes, significant being if you were to do a motherboard swap, or if you swap the hard drive with another computer, and use that copy of Windows 10 on that computer, if it turns out to be, if you don't do any of that, you'll find that any of your activation attempts would go smoothly and without any qualms. That's the digital license in a nutshell, it's a digital entitlement to Windows 10 that is unique to your hardware.

That product key should be in your e-mail (hopefully), and from there, if you were to reinstall Windows in the future on the same exact hardware using the Media Creation Tool, you should be fine using that.

1. Is your computer still in a clean boot (msconfig) state?
2. Have you set the computer on incremental backup?

Check for related errors in event viewer.

Your computer seems to be in a clean boot state, I'd have you enable half the services and startup items to determine which program is conflicting and causing the issue to occur. Once you determine the culprit, please uninstall the program or contact the program manufacturer and restart your computer in normal mode.

With regards to Windows backup running, I'd have you disable and enable the Windows Backup service and check if the issue persists.

a. Click ‘Start’, type ‘services.msc’ and hit Enter.
b. Locate “Windows Backup service”.
c. First stop and then start the service.
