marshall barton say:

I think you are confusing DSum and Sum.  Sum really only makes sense in a form's header or footer section or in a report's Report header/footer and in a group's header/footer.  OTOH, DSum can be used anywhere.  The syntax of two is very different so be sure to keep them straight in you head.

DSum is a short way of running a query and returning a value.  The returnd value is based on the field/expression specified in the first argument.  The third argument is what the query's Where clause would be, but without the word "Where".  One thing you seem to be messing up is that you should not use a table name in this third argument (the table/query name is specified once in the second argument).  Also the conditions in this abbreviated Where clause must have the field names from the table (specified in the second argument) inside the quotes and you concatenate the values from the form outside the quotes.  Looking at what you tried, you put Patient!ID inside the quotes and Patient_ID outside the the quotes, which is backwards.  Try doing it this way in a text box on the Patient form:

   =DSum("Amount","Transaction","Class='Co-Pay' And PATIENT_ID=" & Me.ID)

a. user say:

I haven't done much testing, but something like this should give you a starting point:

Sub putFilenameFieldInFooters()
Dim bPresent As Boolean
Dim doc As Word.Document
Dim fld As Word.Field
Dim hf As Word.HeaderFooter
Dim i As Integer
Dim rng As Word.Range
Dim sc As Word.Section
Set doc = ActiveDocument
For Each sc In doc.Sections
  For Each hf In sc.Footers
    ' Although the VBA/Word objects may exist the footers may not
    If hf.Exists Then
      ' We are moving forward so have already processed any linked footers
      If sc.Index = 1 Or Not hf.LinkToPrevious Then
        bPresent = False
        ' crude check for filename field in the last para
        For Each fld In hf.Range.Paragraphs(hf.Range.Paragraphs.Count).Range.Fields
          If fld.Type = wdFieldFileName Then
            bPresent = True
            Exit For
          End If
        Next
        If Not bPresent Then
          Set rng = hf.Range.Paragraphs(hf.Range.Paragraphs.Count).Range
          ' Only add a paragraph if the last para is non-blank
          ' (You could do more to eliminate trailing blank paras. if you wanted)
          If rng.Text <> vbCr Then
            rng.InsertParagraphAfter
            Set rng = hf.Range.Paragraphs(hf.Range.Paragraphs.Count).Range
          End If
          ' change "FILENAME" to "FILENAME \p" if you want the path as well
          rng.Fields.Add _
            Range:=rng, _
            Type:=WdFieldType.wdFieldEmpty, _
            Text:="FILENAME", _
            Preserveformatting:=False
          rng.Fields.Update
          Set rng = Nothing
        End If
      End If
    End If
  Next
Next
Set doc = Nothing
End Sub

charles kenyon say:

See FileName and Path Shortcut Key Add-In which adds a macro to insert this field at the insertion point. The macro is attached the keyboard shortcut to Alt+Ctrl+Shift+F.

The key to understanding headers and footers in the Ribbon versions is that when you insert a header or footer using a building block, you replace the existing one. You need to get so that you are able to type in (edit) the footer to add something without replacing.

Here is the macro.

'
' FileNamePathInsert Macro
' Macro written 12/10/2011 by Charles K. Kenyon
' "&chr(10)&"Inserts filename and path field at insertion point.
' Attached to Keyboard Shortcut: Ctrl+Shift+Alt+F
'
    Selection.Collapse
    Selection.Fields.Add Range:=Selection.Range, Type:=wdFieldEmpty, Text:= _
        "FILENAME  \* Caps \p ", PreserveFormatting:=False
End Sub

Again, this puts the field at the insertion point.

knowledge---01 say: GSM by nature protects itself from such things. You do not need a fancy handset.

In GSM there is three numbers that are sometimes referred to as the GSM Trinity.

MSISDN - Your cellular number
IMSI - Your SIM card serial number
IMEI - Your handset serial number

You can have any of these blacklisted - which means that it will no longer be accepted onto the network or any other networks that shares blacklisting information.

For example, if someone steals your phone and you ask your operator to blacklist it and give you a new handset with your old number they will do the following :

See what the IMSI (SIM card serial) and IMEI (handset serial) was that you used with your MSIDN (cell number) before it was stolen. This is retrieved from any activity you had on the network like Voice calls/SMS/MMS/GPRS. They will then blacklist the IMEI and IMSI (so the old SIM card will not work anymore and the old handset will also not work on the network).

They will then give you a new SIM card and bind your old MSISDN (cell number) to the new IMSI (SIM). You are then ready to use it in a new handset (IMEI).

These things was thought out right at the beginning of GSM back in the 80's to cater for all sorts of scenarios involving crime and also user flexibility.

For example, did you know that you can have two SIM cards that are bound to the SAME number? This is handy if you have a built in GSM car phone. Your network provider can give the IMSI in the car phone preference over the one in the handset (phone in your pocket).

When the user gets into the car and starts it and the car phone logs into the GSM network the network will kick the handset in the users pocket from the network (which will continually try to log back on). Once the car is switched off the handset may log onto the network again.

These days with Bluetooth and newer technologies things like that are no longer used that much, but back in the 80's and 90's it was quite convenient.

GSM is a very clever system. I have worked with it a lot and you get to know it and respect it very quickly.
