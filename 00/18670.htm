ken sheridan say: Each TableDef object has a Connect property which includes the path to the back end file.  Assuming that you have a single back end file and this is a native Access .mdb file you can loop through the current front end database's TableDefs collection and, where the Connect property is not a zero-length string, refresh the links to the file as a UNC path.  At its simplest the code would be along the lines of the following function, which you would call, passing the UNC path into it as the strPath argument:

Public Function RefreshTableLinks(strPath As String)

    Dim dbs As DAO.Database
    Dim tdf As DAO.TableDef
   
    Set dbs = CurrentDb()
   
    ' Loop through TableDefs collection and refresh link to each table
    For Each tdf In dbs.TableDefs
        If tdf.Connect <> "" Then
            tdf.Connect = ";DATABASE=" & strPath
            tdf.RefreshLink
        End If
    Next tdf

End Function
