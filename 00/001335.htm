On that basis, assuming that the account(s) was/were configured as POP on the old computer, the first thing is to get a copy of the PST file(s) that were associated with the profile. The default locations for pst files used by Outlook are

windrive:\Users\%username%\AppData\Local\Microsoft\Outlook\

hpholm say:

Are there any Over The Air updates available for the phone? - Please check manually from App list > Settings > phone update = check for updates.

This check must either complete in ~30 seconds with 'Your phone is up to date ...' or inform there's an update available. Connect to a stable WiFi to complete the update(s), if any.

With different Microsoft accounts in play, please notice a factory reset is required in order to change the primary Microsoft account on the phone.
From Settings > About = reset your phone (backup first ...)

Correct setting of date, time, year, location and timezone is vital for the phone to establish a secure connection to the Microsoft account and the app store.

HTC 8X AT&T phones are stuck at WP8.0 and an expired certificate is the culprit for the account issues on those phones since November 2015.
It could still be worth a try applying the recovery tool, if the symptoms on your phone match the description at the top of the KB article.

- In Settings > About > More info - what is the OS version?
- What is the error messages and/or error codes, word for word?

You decide how to proceed - and how much time to spend on the project.
WP 8.0 is more than 36 months old and the support for WP8.1 ends July 2017.

hpholm say:

The activation code request is a different issue. The instructions to call Microsoft, as seen on the error screen on the phone, no longer works around the world (it may still work in the USA) - Everybody and everything Microsoft are outsourced since 2015.

https://www.microsoft.com/en-us/mobile/support/contact/

and start a chat from the options on the page is probably the best official option now.

EDIT: Make a note of the phone's IMEI number, dial *#06# - they need that number and information regarding your location and mobile operator.

If this works, make sure to keep a copy of the activation code. It's the 'license key' for the phone, and may be required if the phone is hard reset later.

I would also try backup the phone, then perform a hard reset. With a SIM card that works for phone calls and texts inserted during the first guided set up, registration of the phone should happen automatically.

If both the support chat and the hard reset fails, then I would recycle the phone or give it to somebody who is happy to get a simple phone without all the cloud stuff.

In a contract, I need to make sure that certain text always appears on the same page as the signature block. I have these paragaphs set as "Keep with next". This works fine in some cases, but if the sig block is two columns, which requires (as far as I can tell) a Continuous Section Break before and after, then Word ignores the "Keep with next" and will separate the sig block on the next page if it doesn't naturally fit on the same page. That is, it will leave them on the same page if the natural flow of the document supports this, but it ignores my "Keep with next" setting and only bumps the sig block, and not the text above that should be linked via the "Keep with next" setting. Is that clear?

 

Is what I want to do possible, or is does a Continuous section break always disable "Keep with next"?

 

Thanks for any help,

Colin

stefan blom say:
A section break always interrupts the text flow, so "Keep with next" won't work across the break, as you have noticed.

granitestatecolin say:
A section break always interrupts the text flow, so "Keep with next" won't work across the break, as you have noticed.
OK, I can understand and accept that. Is there a work-around that isn't worse than the original problem? I suppose I could stop using columns, which is what creates the section break, but they provide exactly the right layout effect. Any other suggestions?

Colin


stefan blom say:
You could fake newspaper columns with a two-column table. Of course, this is likely to require manual intervention, and therefore it may not help at all.

doug robbins - mvp office apps & services (word) say:

As Stefan suggests, you could use a table rather than newspaper style columns, which are not really the best thing for this purpose.

 

In all of my contracts, I mostly use a three column table with the centre column being narrow and there for the purpose of providing some horizontal separation between the two signature blocks.

 

You will want to set the rows of the table so that they cannot break across a page.
