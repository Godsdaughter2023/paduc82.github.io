hi brian,

i can totally understand your situation that you want to prevent the end users from changing or removing their profile photo.

to achieve the goal, please following the steps below:

1. create a new outlook web app policies in exchange admin center (eac). log in to eac > permissions > outlook web app policies > click “+” > input policy name in the pop up window > click save to close it.
 

2. connect to exchange online using remote powershell.

3. run the command “set-owamailboxpolicy -identity <your new owa policy name> -setphotoenabled $false” to set the parameter of “setphotoenabled” as “false”. and then you can check it via running command “get-owamailboxpolicy -identity <your owa policy name>”, like the screenshots in my test shows:
 
 

4. after the user’s profile photo being uploaded, please grant the new policy to the users that you want to disallow them changing or removing the photo. please go to eac > recipients > mailboxes > double-click the user > mailbox features > scroll down to click view details in the “email connectivity” option > click browse to select the new owa policy > click ok to save.
 

after finishing the steps above, the options that the user change or remove the profile photo will be gone.

Off Topic - Background

Office was designed from day to to be uninstalled and re-installed as the primary trouble shooting tool.  Along with that design choice they assumed that the "normal" user would only have one version of Office installed. When the new version came out, the normal user would automatically upgrade to the next version (even back in the 1980's they assumed everyone would "drink the coolaid").  So when an application starts, it checks to see which Office version is currently "installed". If the application is not in the installed version, then the configuration process is triggered.

However, MS developers working on the new version quickly got tired of waiting for that process to complete, so the created the NoReReg registry key to stop it.
