The memory card password is not there to protect your data within the phone, it will only be requested if it is detected that the card has been inserted to a different device (or if your phone is reformated or reflashed).

Let's say the data are in rows 2 to 100.

Select these rows.

techfreak_ say:

PCI Compliance is a specialty generally requiring a pre-designed all-in-one solution that meets every requirement or several solutions to form compliance. Standard end-user services like messaging technologies might meet one or more facets of PCI Compliance but your own policies and procedures ensure that full compliance is met. As far as I know, Skype does not claim any sort of compliance. If you are concerned whether or not it meets your encryption requirements, you can view more details in this FAQ and the linked security and legal pages. If one of your requirements is whether or not the data will exist beyond the chat session, you have the option of deleting your own instant message transmitting that data. That would be an action you would have to do yourself. This is an example where your own policies and procedures would help you meet a level/aspect of compliance.

rich michaels - mvp (greatcirclelearning.com) say:
As Doug has said, increasing the duration between auto recovery saves can reduce the inconvenience and I've also seen comments saying if you have it set to less than 10 minutes, that has caused issues. I've experienced that myself so I go with the default 10 minutes… if I have it running at all.

Keep in mind this is an AutoRecovery file and not necessarily, as I understand, a fully assembled version of your file. I know there is an option you can check for it to save the last "autosaved" file if you don't save when closing but that only happens at document closing event. It is far better IMHO to turn on the "Always create backup copy" switch located in Word > Options > Advanced > Save and whenever you stop to take a breath or think, train yourself to press Ctrl+S. Before writing over your current file the "Always create backup copy" makes a copy of the former version and adds "Back of" to the file name.

With that option turned on you not only have the version of the file in memory, a last saved version, and the one before that. If you are really concerned about this, as many of my customers are, you can also get a copy of my zAPPs-GotUrBack app and it will keep not only the last saved, but the prior two as well… and it does this for Word, PowerPoint, and Excel. For more info go to the website listed in my signature. Some have told me with this running and by changing their saving habits, they've turned AutoRecovery off. I'm not advocating that… I'm just saying.

The last two things I want to suggest is for you to take a look at where your AutoRecovery files are being stored. Maybe if it's going to a mapped network drive location and changing it to something local or just a different location might help. Finally, do you have any other auto backup running? Slowdowns can occur when auto backups hangup because temporary files are coming and going dynamically. AutoRecovery works by trashing the old recovery file and assembling and renaming current temporary files to the new saved auto recovery file. In other words… traffic jam time.
