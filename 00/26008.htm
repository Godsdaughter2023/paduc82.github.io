You can't use / or \ in file names.

The following information is based on a Forum note or Microsoft  Document on the subject. This defect is an old problem that started when Mac switched from using PowerPC to Intel and the same issue applied to Dos and windows first.

Some of the Forbidden Characters are:

Numbers before names, Superscript/Subscripts.And these Characters:

+ * ” ‘ \ | / ; [[ ]] << >>

Numbers after Names are okay but before extensions.

You cannot use these characters in the name of:

A) Hard Drive.

B) Directory (Folder) Names.

C) Any File Names.

D) Any Device Names.

E) And on servers

For Servers: no names can have the name File or Fileserver in any path names.

You can use _ (underscores).

NOTE: I haven’t included dashes/hyphens because I haven’t found any information.

Maps Improvement:

 

1. This data will not be sent over cellular (GPRS/3G) connection. Data is sent when Ovi Maps service is used over WiFi connection with a mobile device. Data is also sent with PC applications related to Maps service such as map loader or Ovi Suite. This is done in order to prevent/minimise cost to the user. Amount of data is comparably low; as a result less than 20 kbytes is generated in one hour (while driving). If the data is not sent with WiFi immediately after collecting it will be stored in the memory card for later sending. As a maximum of 1MB will be used so it will not fill your memory card.

2. Collected data is used in two ways; improving the map content and the application itself. Changes to the map content, such as new roads, changed street directions, speed limit changes etc. can be identified from the data. Navteq will use this data for updating the map and these updates are free of charge to the maps application users. Application will be improved based on statistical models. The more data Nokia receives the better the map data and Ovi Maps application experience will be for the users.

3. Data is collected in both of these modes but the device needs to move at car speeds.

Also note that there are absolutely no personal identifiers in the data. Furthermore the collection happens in cycles; collecting data, pause, collecting data, pause... This is done for the sake of user privacy as these short "trails" will further increase the anonymity when inserted into a database with data from different users.
