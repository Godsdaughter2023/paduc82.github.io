trevor rabey say:

I disagree entirely with your description of "those practices", and would be happy to discuss why in greater detail and offer much better alternative approaches.

Your question "Why shouldn't I use manually scheduled tasks?" is a good one which requires a good answer which is probably long and detailed. However, I can provide a short answer which is the one that I provide to my trainees and colleagues, "because I say so".

Manually scheduled tasks were only introduced in MSP2010. Before that, they did not exist and users were forced to come to grips with auto scheduling and how to make it work. This imposed a need to understand how the Critical Path Method works and to appreciate that it is built into the MSP scheduling engine, which is a good thing. Not difficult, but essential, and requires a bit of patience and practice, and perhaps some study.

This makes it unpopular.

Manually scheduled tasks allows a user to avoid patience, practice and study, and simply draw a picture with horizontal bars and many people are happy to settle for that. You don't need software for that. You can just use a crayon. But if you have MSP and just want to draw a picture with horizontal bars on it, then manually scheduled tasks can be used for that. Maybe they have a limited usefulness in the roughing out stage or to knock up a quick cartoon of the plan. But why not just do the project planning properly from the get go instead?

Manually scheduled tasks is similar to just typing in the dates and not using any predecessors or successors. If you already know the dates, you don't need MSP. But why buy a dog and then do the barkin'?

You need a project plan which is in a form which let's MSP do what it is designed to do. You need a critical path. You need a plan which is maintainable, so that you can monitor and record progress during the execution of the project, and re-schedule as required.

There are numerous things which MSP will allow you to do, which you should not do if you want or need any of the above. An example would be linking the summaries as predecessor or successors. It is allowed, but just because it is in the software does not mean it is a good idea.

Your description suggests the document has multiple Sections and/or a page layout using a different first page setup - and possible even a different odd/even page setup.

What you need to understand is that each Section in a Word document contains three headers and three footers and that, in a multi-Section document, each of these headers and footers may or may not be configured to continue from the previous Section.

If you open the document and press Ctrl-A, then Alt-P-S-P to open the Page Layout dialogue and choose the Layout tab and either of the 'Different odd and even' or 'Different first page' boxes is checked or shaded, that's probably part of your issue. Unless you need to have different page headers in one or more Sections, try clicking on the checked or shaded option(s) to uncheck them, then ensure the footer table appears on the first page.

If you press Ctrl-End, to go to the end of the document, Word's Status bar should tell you how many Sections the document has. If the number is more than 1, that's probably another part of your issue. Again, unless you need to have Sections with different page headers (which you might do if you have mixed page orientations, for example), you could delete all the section breaks via Find/Replace, where:

Find = ^b

Replace = nothing

If you need to have Sections with different page headers and/or different page headers in one or more Sections, you could use the following macro to ensure all footers display the footer table:

Sub ReplicateFooters()
Dim Sctn As Section, HdFt As HeaderFooter, i As Long
With ActiveDocument
  For Each Sctn In .Sections
    For Each HdFt In Sctn.Footers
      HdFt.LinkToPrevious = True
    Next
  Next
  With .Sections(1)
    For i = 2 To 3
      If .Footers(i).Exists Then
        .Footers(i).Range.FormattedText = .Footers(1).Range.FormattedText
      End If
    Next
  End With
End With
End Sub

The only things you'll need to be aware of regarding the above macro are that:

a) the footer orientation may be 'wrong' on some pages in documents with mixed page orientations; and

b) any updates to the owner of the document, the date it was last edited will have to be done for the 'Different odd and even' and/or 'Different first page' footers.
