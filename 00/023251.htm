    *   %appdata%\Microsoft\teams\databases

 e.  From within ‘GPUCache’, delete all files.

    *   %appdata%\Microsoft\teams\GPUcache

 f.  From within ‘IndexedDB’, delete the .db file.

    *   %appdata%\Microsoft\teams\IndexedDB

 g.  From within ‘Local Storage’, delete all files.

    *   %appdata%\Microsoft\teams\Local Storage

 h.  Lastly, from within ‘tmp’, delete any file.

*   %appdata%\Microsoft\teams\tmp

====================================================

Once the cleanup is finally completed, you can now restart Teams and check the results.

christopher miller say:
Hi Ambili,

When you send a file over Bluetooth to Windows Phone, there must be an app that has registered itself as a receiver for that file type.  Media files to the standard Windows Phone media locations, other files go to the location associated with the app that can accept the files.

If you have a PDF viewer app on your phone (I use Microsoft's PDF Reader app), the app will mange the .PDF files in it's own library.  When you open the PDF app, it should list all of the PDF files in it's library.  You and read and delete the files from within the app.

You only have to click on tap view if you want to view the file after it was sent over.  It's a good check to make sure the right file was sent over.

If you delete the file from with the app, the it's gone from your phone.  If you delete the app, all of the PDF files it had in it's own library will be deleted too.

Files that do not have an app registered to receive them will go to temporary storage and will get flushed out Windows Phone needs the space.

If you can't find a missing PDF file, just try resending it.

FWIW, I rarely send a file via Bluetooth, I just put in a SkyDrive/OneDrive folder so I can access it from anywhere.
