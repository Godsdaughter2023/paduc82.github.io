Based on your description, you would like to only give user access to certain services (e.g. Sway) in an Office 365 licensing plan (e.g. Enterprise E3 plan).

 $LicenseOptions = New-MsolLicenseOptions -AccountSkuId <AccountSkuId> -DisabledPlans “TEAMS1”,”FLOW_O365_P2”,”POWERAPPS_O365_P2”,“OFFICE_FORMS_PLAN_2”,”SWAY”

How to break ties looking at several criteria - Microsoft Community

mshaw200 say:

How to break ties looking at several criteria

I am looking for assistance on breaking ties.  I compete in water ski competitions where each competitor skis 2 preliminary rounds on Saturday with the top 16 moving on to the finals.  

The top 8 places are picked from the skiers that have the single round best score (from either round).  Places 9-16 are taking from the remaining skiers based on the best 2 round averages.  

For the top 8 places, ties are broken in the following way: 
     (1) by the skier with the highest back up score
     (2) by the skier with the highest preliminary seeding

For places 9-16, ties are broken in the following way:
     (1) by the skier with the highest single round score
     (2) by the skier with the highest preliminary seeding

I currently have the following columns:

Skier Round 1 Round 2 Best Round Backup Average

Best Round Ranking Adj Avg Cumm Ranking Skier

The column just past the grey bar has this formula in it:    =(D2/MAX(ABS($D$2:$D$51)))+(E2/(10^(MAX(LEN(D$2:D$51)+1))))
Best Round Ranking has this formula in it:  =RANK(H2;$H$2:$H$51;0)+COUNTIF($H$2:H2;H2)-1
Adj avg column basically removes all of the skiers that rank 1-8 in the best round ranking column then they get ranked in the cumm ranking column.
bernie deitrick, excel mvp 2000-2010 say:

The basic idea is to create a composite number. You want to come up with numbers that have the form

XX.XXYYYYZZZZ

Where XX.XX is determined by your primary score, YYYY is determined by your first tie breaker, and cannot change any of the XX.XXX digits, and ZZZZ is for the second tie breaker and cannot change any of the other digits. The ideal result would be

XX.XX0YYY0ZZZ

so that you don't have to worry about impacting higher level tie breakers.

So, convert your first tie breaker to a value between, say 1 and 10000, and then divide by 1,000,000 and add it to your score. Convert the second tie breaker to a value between 1 and 10000, and divide by 10,000,000,000

Then I would use two columns of formulas - one to determine scores based on the top 8 criteria, and one for scores on the 9-16 criteria.

For the Top 8, I would use something like this:

=MAX(Score1, Score2) + MIN(Score1, Score2)/1000000 + (100-Seeding)/100000000000

Then for the 9-16 scoring, use

=AVERAGE(Score1,Score2) + MAX(Score1, Score2)/10000000 +  (100-Seeding)/100000000000

Then use formulas to pick the top eight from the first column, and exclude those from consideration in the choice of 9-16.

The numbers that you divide by and the constant that you use for the seeding adjustment depends on the number or competitors, and you need to take into account how many decimal places your scores have, etc.

joan pin say:

Hi,

Word also allows you to create index sub-entries. These are index entries that are subordinate to other index entries, and generally appear indented under the main index entry. To insert an index sub-entry, follow these steps:

  1. Select the text you wish to appear as the sub-entry.
  2. Press Alt + Shift + X keys. Word will displays the Mark Index Entry dialog box. Notice that the text you selected is shown in the Main Entry text box.
  3. Press Ctrl + C to copy the selected Main Entry text to the Clipboard.
  4. Position the insertion point in the Sub-entry text box.
  5. Press Ctrl + V to paste the text into the Sub-entry text box.
  6. Change the information in the Main Entry text box to indicate the main index entry to which this sub-entry should be subordinate.
  7. Click on Mark.
  8. Click on Close to dismiss the Mark Index Entry dialog box.

Feel free to respond if the provided steps work for you.

jay freedman say:

The purpose of the Mark Index Entry dialog box is to create an XE field in the document. The syntax of the XE field is explained at Field codes: XE (Index Entry), which may help you understand what the dialog box does.

As mentioned in the "Text" section of the article, a colon indicates a subentry. Although it doesn't mention this, a second colon creates a third-level subentry, and so on. As an example, if you index the name of scientist George Washington Carver, these results are possible...

  • If the Main Entry box in the dialog contains Carver:George Washington, or if the Main Entry box contains Carver and the Subentry box contains George Washington, the result in both cases is the field {XE Carver:George Washington}. The Index field will display two levels.
  • If the Main Entry box contains Carver:George:Washington, or if the Main Entry box contains Carver and the Subentry box contains George:Washington, the result in both cases is the field {XE Carver:George:Washington}. The Index field will display three levels.

The other items in the Mark Index Entry dialog box are adequately explained in this page, which you can open by clicking the question mark in the dialog box's title bar.
