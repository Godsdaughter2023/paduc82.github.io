john korchok say:

After adding the field to a document, you could save that document as a template. Then use the File>New command to create new documents from the template and they'll all include the date field.

To save the field as AutoText, select the field, then choose Insert>AutoText>AutoText. The date will appear in the field where you want to type the name for the AutoText entry. Replace it with a name like Bob suggested, then click on the Add button.

In use, whenever you want to insert the date, start typing the entry name. As soon as the date appears in a light yellow floating tag, hit the Return key and the date will be entered.

bob jones aka: cybertaz say:

Unfortunately, this is a shortcoming of Word 2008 which never was corrected before deprecation of that edition. AFAIK, there is no "fix" or means of modifying the AutoComplete feature to add the format you prefer.

What you can do is use Insert> Field> Date & Time - CreateDate. Use the Options button in that dialog to add this custom format [as shown in bold]:  CREATEDATE  \@ "MMMM d, yyyy"

The field can be saved as an AutoText Item with a name such as *date which will function in a fashion similar to the one normally used to complete dates by simply typing the first 4 characters then pressing return when the tip displays.

grschinon say: If the phone is requesting a PUK code then it's the SIM that's locked, not the phone. In other words, if you put another SIM in your phone it'll work fine, and if you put your current SIM in another phone, that other phone will also ask you for a PUK code.

Your operator is the only organisation that can give you the PUK code because the SIM is theirs, not Nokia's.

Assuming you're in the UK and you've registered with "My T-Mobile", you can log into your account and the PUK will be given there. It's also printed on the plastic envelope in which your SIM and small booklet with useful numbers came. These methods of obtaining the PUK code are usually free while asking customer service for it may be charged for.
