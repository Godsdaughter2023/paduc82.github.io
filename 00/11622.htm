jay freedman say:
The picture is not "cropped dynamically" -- in fact, it isn't cropped at all (by Microsoft's definition of "cropped"). The entire picture is still present after you drag it to a larger size, as you can see by the dotted outline.



However, the picture is contained inside a cell of a table that is set to an exact row height (2.3") and fixed column width (2.44"). That's the only part of the picture that will be visible, no matter how much larger you make it. The row height is set on the Row tab of the Table Properties dialog (change "At Least" to "Exactly"), and the fixed column width is set by clicking AutoFit on the Table Tools | Layout tab and choosing that option.

If you really want to crop the picture, as in discarding the parts that aren't visible, you must use the Crop tool on the Drawing Tools tab (which is available when the picture is selected).

In that case you use that code inside an event routine and after your event routine ends the code of the form itself is executed and sets the focus to the next Textbox. (Or an other event routine of your code is executed and calls SetFocus...)

Private Sub tb_Bkr_Exit(ByVal Cancel As MSForms.ReturnBoolean)
  If tb_Bkr.Value <> "" And tb_Bkr > 10 Then

Exact subject [FW: AL TAYYEB ELECTRONICS - 1978  - FAX MACHINE]

However if I remove the hyphens, and searching again the email is found.

Is there some kind of restriction on the use of hyphens?

Outlook version: 2013 (15.0.481.1000)

Thanks,

Aa

keratl say:

the search function interprets the hyphen as a "minus", which eliminates the following word from the search. Any matches to the main search term (before the hyphen) will be shown, but any containing the word after the hyphen will be suppressed from the results.

One option is to enclose the search term in double quotes, which tells the search algorithm that the hyphen is part of the string you are looking for;

"FW: AL TAYYEB ELECTRONICS - 1978  - FAX MACHINE"
