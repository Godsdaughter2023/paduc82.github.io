This issue occurs when the processor does not support Windows 8. The processor must support Physical Address Extension (PAE), NX, and SSE2. Most CPUs have support for these features, so if you receive this error, it is likely because the NX feature is not enabled on your system.

 

 

Method 1-

You can try booting to computer using the System Recovery options assuming that you are unable to boot to the desktop.

Refer the steps-

a)      Boot with the Windows 8 DVD.

b)      On the “Startup options menu” select “Troubleshoot”.

c)      Now, click on “Advanced boot options”.

d)     Try the various recovery options and check.

Note: Performing a System Restore will undo any changes made on your computer earlier. You may also lose unsaved information as well. Proceed at your own risk.

Make sure you back the data on the computer before proceeding further.

 

Method 2–

To resolve this error, follow the manufacturer guidelines to enable NX (“No eXecute bit”), or the equivalent XD (“eXecute Disabled”) feature, within the BIOS security settings.

If the BIOS settings for the NX support option are not available on your system, you may need to contact the manufacturer for updating the BIOS. Note that some older processors may not contain these features and will be incompatible with Windows 8.

Note: Updating and changing the configuration of the BIOS is at your own risk.

 

Method 3-

Assuming that you have changed the BIOS settings and now, the CPU is capable of supporting

Windows 8, I would have you perform a Clean install after formatting the hard drive.

User accounts - Recommended Setup (Vista andWindows 7 )

You absolutely do not want to have only one user account. Like XP and all other modern operating systems, Vista and Windows 7 are multi-user operating systems with built-in system accounts such as Administrator, Default, and Guest. These accounts should be left alone as they are part of the operating system structure.

You particularly don't want only one user account with administrative privileges on Vista and Windows 7 because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult. It isn't impossible to activate the built-in Administrator to rescue things, but it may require more work than you want to do. Best not to get into a bad situation to begin with.

The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. Running as a Standard user is best practice for security purposes and will help protect your computer from infection. After you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

If you want to go directly to the Desktop and skip the Welcome Screen with the icons of user accounts, you can do this:

File Explorer often becomes unresponsive and freezes for several seconds.

Fresh installs of windows 10 are also affected. A system with many hard disk volumes and many files are more likely to be affected. File Explorer is constantly checking for file and folder changes and for registry changes so that it may update the list of files and the navigation tree.

lemp say:
If the adapter worked to "download 142 Windows Updates" and then no longer works, chances are that you downloaded a driver for it that isn't working properly.  Because you have something to compare, do the following on your other Win 7 64-bit system and on the problem Win 7 64-bit system:

Open Device Manager (type Device Manager in the Search box and press Enter)
Expand the Network adapter section by clicking the arrow to its left
Double-click the entry for your wireless adapter
Click the Driver tab
Are the Driver Provider, Driver Date, and Driver Version the same on both machines?

On the bad machine, click the General tab.  Please provide the complete text from the box under "Device status."  Do this as soon as you see the message that "This device cannot start (Code 10)."


Possible solutions:

1.  Uninstall the device from Device Manager; disconnect the adapter from the computer; reboot the computer; reinstall the device using the CD that came with the adapter.  If you no longer have the CD, download the CD image from http://www.buffalotech.com/ and burn it to a writable CD.

You may not see the device if it's not connected.  If you don't see it under "Network adapters," look in the "Universal Serial Bus Controllers" category.

You can also temporarily view disconnected devices in Device Manager as follows:
Open an Elevated Command Prompt (ECP) window (type cmd in the Search box, then right-click on Cmd.exe and select "Run as administrator" and respond to the UAC prompts)
In the ECP window, type the following and press Enter after each line
    set devmgr_show_nonpresent_devices=1
    devmgmt.msc
When Device Manager opens, click the View menu and select "Show hidden devices"

1a.  You may also be able to uninstall the drivers using the CD that came with the adapter:
  • Insert your AirNavigator CD. When you insert CD, AirNavigator Setup Wizard will launch automatically. To launch it manually, double click [My Computer], then your CD-ROM, then [AirNavi.exe].
  • Click [Options].
  • Click [Remove Drivers].
  • Follow the instructions on the screen to remove drivers.

2.  Perhaps Microsoft's Diagnose and fix Windows USB problems automatically troubleshooter will help.

Other notes:

  • You should not have seen "Device is Enabled" in Device Manager -- that box is in Device Manager in XP but not in Windows 7.  Where did you see this message?
  • You may see advice on some web sites to "uninstall every device under the Universal Serial Bus controllers hardware category in Device Manager as part of the driver reinstall."  Unless you have a keyboard with a PS/2 (round) connector, DO NOT DO THIS. 
  • I suggest that you not use -- at least initially -- the Buffalo Client Manager software.  If you have that installed, uninstall it.  The CD has an option to do this.

lemp say:
I don't have much more to add.

The usual steps for "unable to connect" to a wireless network are

(1) go into the router's configuration utility (using a computer connected to the router with an Ethernet cable), and (temporarily) disable ALL security (usually, this just means configuring "no encryption," but some folks also have added additional things such as MAC filters, time of day filters, IP filters, and the like; those should all be disabled)

(2)  delete the "profile" for the wireless connection on your computer.  If Windows 7 is in control (and in your case it may or may not be; you may have to deal with the Buffalo Client Manager), go to Manage Wireless Networks and delete the profile (see image below)

(3) View available wireless networks and attempt to connect to your now unsecured network

If that succeeds, then you can go back into the router's configuration utility and reconfigure your wireless encryption.  Don't add any other "security" features (they are all easily spoofed, anyway).
