a. user say:

Copying data into an existing table is a pain. It's generally much easier to insert the data then format the table how you want.

If you're trying to copy/paste the info. from the program where you are executing the query, what is that program?

e.g. if it is something like SQL Query Analyzer, you should be able to select the resulting grid and copy/paste into Word. You then get a sequence of tab-delimited paragraphs. As long as you don't have any tabs in the data fields themselves, and as long as you have fewer than around 64 columns, you can

 a. select those paragraphs

 b. use Table->Convert->Text to table to convert the text to a table

 c. formt the table how you want.

FWIW if I were doing a lot of these and knew the queries were reasonably simple and short, I would probably use a different approach, roughly as follows:

 d. use Word View->Toolbars to display the Database toolbar

 e. use Insert->Database to insert some data from a table in the database that you are trying to use. To do that, you'll need to create a suitable .odc file that contains the necessary connection information. The information should be inserted as a Word field. The result will be a table - there are some predefined formatting options that may be enough for your purposes.

 f. Use Alt-F9 to display the field code. You should see something like

{ DATABASE  \d "C:\\Documents and Settings\\username\\My Documents\\My Data Sources\\myodcfile.odc" \c "Provider=SQLOLEDB.1;Integrated Security=SSPI;Persist Security Info=True;Initial Catalog=mycatalog;Data Source=myserver;Use Procedure for Prepare=1;Auto Translate=True;Packet Size=4096;Workstation ID=mywsname;Use Encryption for Data=False;Tag with column collation when possible=False" \s "SELECT * FROM \"mytable\"" \h }

 g. You can generally cut that down to something more like

{ DATABASE  \d "C:\\Documents and Settings\\username\\My Documents\\My Data Sources\\myodcfile.odc" \c "Provider=SQLOLEDB.1;Integrated Security=SSPI;Persist Security Info=True;Initial Catalog=mycatalog;Data Source=myserver;" \s "SELECT * FROM \"mytable\"" \h }

 h. When you've got that to work, save the field into another document or as an autotext for re-use.

 i. Then insert the query text you really want to use, select the DATABASE field, and press F9 to re-execute the field. Use Alt-F9 to toggle to/from "field code" and "field results" view.

 j. When you're done, you can either leave the field in your text, or select it and use ctrl-shift-F9 to "unlink" it, i.e. replace the field by the results.

 k. That can take a bit of getting used to (and you'll probably find that Word is a bit picky about the SQL you use - for example, you may find that you need to alias table names more than you should need to) but it can be quite a handy way to work.

wsm93 say:

Ms Access - Filter record on subform based on multiple selection in List box

Hello,

I have a form that have 4 listbox and subform.I want to filter subform based on multiple section in list box.For one selection it work but not for multiple selection using Master Child Link.Two list box have string and two have numeric value. i want to select multiple values from list box and that filter record below based on selection. If there is a way to achieve this by properties that would be best otherwise VBA code would also be good to go.

Any help would be appreciated.
ken sheridan say: You cannot use the LinkMasterFields and LinkChildFields properties if a multi-select list box is used, so leave both properties empty.  Instead you can filter the subform  whenever an item is selected in one of the list boxes.  To do this add the following function to the frmListBox form's module:

Private Function FilterSubform()

    Dim varItem As Variant
    Dim strFilter As String
    Dim strItemList As String
    Dim ctrl As Control
    
    Set ctrl = Me.Country
    
    ' if at least one country is selected
    ' loop through the control's ItemsSelected
    ' collection and build a value list of
    ' selected country names
    If ctrl.ItemsSelected.Count > 0 Then
        strFilter = " And "

        For Each varItem In ctrl.ItemsSelected
            strItemList = strItemList & ",""" & ctrl.ItemData(varItem) & """"
        Next varItem
        
        ' remove redundant leading comma
        strItemList = Mid(strItemList, 2)
        ' assign filter to strFilter variable using IN operator
        ' against the value list
        strFilter = strFilter & "Country In(" & strItemList & ")"
    End If
    
    
    Set ctrl = Me.List5
    strItemList = ""
    
    ' do the same for years list box
    If ctrl.ItemsSelected.Count > 0 Then
        strFilter = strFilter & " And "

        For Each varItem In ctrl.ItemsSelected
            strItemList = strItemList & "," & ctrl.ItemData(varItem)
        Next varItem
        
        strItemList = Mid(strItemList, 2)
        strFilter = strFilter & "Years In(" & strItemList & ")"
    End If
    
    Set ctrl = Me.List11
    
    ' if a product is selected add to strFilter variable
    If Not IsNull(ctrl) Then
        strFilter = strFilter & " And Product = """ & ctrl & """"
    End If
    
    ' remove redundant leading ' And ' from strFilter variable
    strFilter = Mid(strFilter, 5)

    ' if a string expression for the filter has been built
    ' set Filter property of the subform to string expression
    ' and turn on filter by setting subform's FilterOn property to True
    If Len(strFilter) > 0 Then
        Me.[qryRecent subform].Form.Filter = strFilter
        Me.[qryRecent subform].Form.FilterOn = True
    ' otherwise turn off filter by setting subform's FilterOn property to False
    Else
        Me.[qryRecent subform].Form.FilterOn = False
    End If
         
End Function

In the properties sheet of each of the list boxes enter the following as the After Update event property:

=FilterSubform()

The above code assumes that you want to filter the subform to rows which match any of the selected countries AND any of the selected years AND  the selected product.  If, on the other hand you wanted rows which match any of the selected countries OR any of the selected years OR  the selected product, then the function would be:

I've tried using "=eomonth(cell,0)" but it only gives the offical last day of any month and not the last day of the month on that list.  For an example of the problems, the last work day of a month may be on a Friday but the offical last day of the month is on the weekend and so the list ends with the date for that Friday (the date I wish to refer to and obtain the data next to it) yet using "=eomonth" would give (31/x/200x).  Sorry if I've just confused everybody but thanks in advance for your answers.
ronrosenfeld say: If the dates are in Column A, then the following formula will return the last date of the month in the adjacent Column A cell, in that list.

This formula must be **array-entered**:

=MAX((MONTH($A:$A)=MONTH($A2))*(YEAR($A:$A)=YEAR($A2))*$A:$A)

Then fill down as far as required.
It will operate faster if you replace $A:$A with a range that will encompass all the dates, if it is smaller.   e.g:  $A$1:$A$1000

And merely insert the correct references for the different columns
----------------------------------------

the formula into the cell or formula bar, hold down
<ctrl><shift> while hitting <enter>.  If you did this
correctly, Excel will place braces {...} around the formula.
