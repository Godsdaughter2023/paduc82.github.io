vic in utrecht say: Thanks for your response, HansV!

I tried your replacement, but got the same messages. I then stepped through the code and found out that the problem was when executing the statement Recipients.resolveALL.

When I made the statement inacitive, the code shown below worked again.I then reverted to my original code.

For your information: On a Windows 7 system, Outlook has to be inactive. On an XP system Outlook has to be active. This anomally started after going to Access 2010. The issue with Recipients.resolveALL probably came after the mentioned Outlook update.

The issue with the Outlook being already running was because the error routine did not close the Outlookapplication.

Here is the code I am using now:
:
Sub ap_CreateOLMailItem(strRecipient As String, _
                        strCC As String, strSubject As String, strMessage As String, strDisposition As String)
On Error GoTo Error_ap_CreateMailItem
    Dim objMailItem As Outlook.MailItem
   Set olkApp = New Outlook.Application
   
    Set olkNameSpace = olkApp.GetNamespace("MAPI")
    Set objMailItem = olkApp.CreateItem(olMailItem)
   
    With objMailItem
        .To = strRecipient
        .CC = strCC
'        .Recipients.ResolveAll
        .Subject = strSubject
        .BodyFormat = olFormatHTML
        .HTMLBody = strMessage
        If strDisposition = "Send" Then
            .Send
        Else
            .Display
        End If
    End With
   
    Set objMailItem = Nothing
    Set olkNameSpace = Nothing
    Set olkApp = Nothing
    Exit Sub
Error_ap_CreateMailItem:

    If Err.Number = 287 Then
        MsgBox "This command needs Outlook to be active." _
            & vbCrLf & "Please start Outlook and click on ""OK"" when Outlook is running"
        Resume
    Else
        If Err.Number = 429 Then
            MsgBox "This command needs Outlook to be INactive." _
                & vbCrLf & "Please close Outlook and click on ""OK"" when Outlook is actually not running anymore"
            Resume
        Else
            MsgBox Err.Number, Err.Description
        End If
    End If
End Sub
