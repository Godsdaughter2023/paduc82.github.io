ossiemac say:

Try the code below.

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Dim rngCols As Range
    Dim lngColFirst As Long
    Dim lngColLast As Long
    Set rngCols = Columns("D:O")    'Set a variable to the required range of columns
    With rngCols
        lngColFirst = .Columns(1).Column
        lngColLast = .Columns(.Columns.Count).Column
    End With
    If Not Intersect(Target, rngCols) Is Nothing Then   'Not Nothing then is something
        On Error GoTo ReEnableEvents
        Application.EnableEvents = False
        Range(Cells(Target.Row, lngColFirst), Cells(Target.Row, lngColLast)).Select
        Target.Activate 'Ensures Initial selected cell will be the clear cell
    End If
ReEnableEvents:
        If Err.Number <> 0 Then
            MsgBox "Error occurred in Private Sub Worksheet_SelectionChange"
        End If
    Application.EnableEvents = True
End Sub

I'll try to explain some of the code and if there is still anything else you do not understand then feel free to get back to me.

When a range is assigned to a variable (As in Set rngCols = Columns("D:O") the variable is rather like a mini worksheet and the first row of the range is 1 and first column of the range is 1. Therefore in the With/End With nest  .Columns(1) references the first column of rngCols and then Column refers to the column number of the worksheet so the code assigns to lngColFirst the worksheet column number equivalent to the first column of rngCols. Similarly lngColLast = .Columns(.Columns.Count).Column  assigns to lngColLast  the worksheet column number equivalent to last column of rngCols

The reason for using the With/End With nest to assign the column numbers is so that the only the range assigned to the range variable needs to be changed in the code for it to work in a different range. Could have simply used the following line of code for the select but to use for a different range then both the range for the range variable and the following line of code needs to be edited.

Range(Cells(Target.Row, "D"), Cells(Target.Row, "O")).Select

Target.Activate    When any range is selected, the first cell selected is not highlighted; it remains white. This line makes the Target cell that cell.

Application.EnableEvents = False   suppresses further calls to this event otherwise when the code selects the required range there would be a recursive call to the same event sub.

On Error GoTo ReEnableEvents   is there because if an error occurs somewhere in the code then Events remain turned off until they are either turned back on with code or Excel is restarted. This is included in the production code but usually commented out until testing is finished otherwise the developer does not know which line the error occurred. If events are turned off during testing due to an error to turn events back on manually Alt G will open the Immediate window in the VBA editor and then enter the following code line and press Enter.

Application.EnableEvents = True

The MsgBox incorporated in  If Err.Number <> 0 Then is purely so the user knows an error has occurred and can report it to the developer.
