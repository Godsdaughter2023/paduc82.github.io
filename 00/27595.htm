Hi

Your installation of Office is right up-to-date, so we can rule that out as a problem.

SAFE BOOT

 

I think the next thing to do is relatively simple procedure: Perform a Safe Boot by holding the SHIFT key down as your Mac starts up from a restart or shut down.

 

It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot Mac OS fixes troubles with your Mac's directory and file system. The font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. This allows you to test your app(s) by running them with a minimal set of fonts, an empty font cache, and without potentially conflicting apps running.

 

Once you reach the desktop in Safe Mode try running the app having a problem. Is the problem still there?

 

If the problem is gone: Performing a Safe Boot may have fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart.

 

If the problem is still evident in Safe mode, you may still have a bad font. Use Font Book to restore system fonts, then test again in Safe Mode.

 

If the problem was gone in Safe Mode but returns during a regular boot: I think the first thing to do would be to use a font utility to validate ALL your fonts. Mac OS comes with a free font utility called Font Book, which you can use for this purpose. Get rid of any font that doesn't pass as clean and green. Then, restart your Mac and immediately empty the trash. If you've gotten this far and still have a problem, then there are more things to explore. But if you get this far you've probably gotten rid of the crash. Either way, please reply with your results.

Hi Michael,

 

After reading this, I first have to say it doesn’t sound as if you had the user-level security securely setup since the users could still work with the database using the default security file. Secondly, I should advise you that user-level security is being deprecated and is no longer available starting with the 2007-2010 file format (accdb).

 

Onto your current issue, if your network drive was down, the users would have received a message “Microsoft Access couldn’t find the file ‘xxx.mdw’. This file is required for startup.” or “You do not have the necessary permissions to use the ‘xxx.mdb’ object. Have your system administrator or the person who created this object establish the appropriate permissions for you.” As you can see, Access does not present a method for joining to the default security file and it doesn’t do it automatically.

 

There are some possible scenarios which may cause this type of change.

1.     If Access was joined to your security file, a repair, reinstall or upgrade of Access may cause Access to revert to the default security file.

2.     If you were using a shortcut that provided the path to the security file, perhaps the shortcut was modified or the users tried opening the database file directly.

matthewlersch say:

nested if(or) function

I have an estimating workbook that utilizes a number of dropdowns and pricing tabs that are referenced depending on what is selected in the dropdowns and other cells. I want to write a formula that is based on numerous conditions to look for pricing on the correct tab.

Currently I have columns labeled interior and exterior. If there is an "X" in the exterior cell then the door price cell will refer to the extprice tab for pricing otherwise it looks at the intprice tab. I use a Vlookup based on the door size value in a different cell. This works fine. 

Now I want to use another column labeled door material which uses a dropdown for Hm (hollow metal) or SCWD (solid core wood). I want to write a formula in the door price cell (previously mentioned) that will look at pricing based on Hm or SCWD. If Hm is selected then the pricing has to refer to the extprice or intprice formula mentioned above.

Make sense?
jllatham say:

Without going into a lot of detail and worksheet layout, let us say that you now have 2 VLOOKUP() formulas that work, one for interior doors and one for exterior doors,

 

You could set up something like this:

 

=IF(typeDoorMaterial="HM",VLOOKUPforHollowCores,VLOOKUPforSolidMetal)

Assuming typeDoorMaterial dropdown is in B6, then it might look more like

=IF(B6="HM",VLOOKUPforHollowCores,VLOOKUPforSolidMetal)

and of course the VLOOKUP() formulas are going to be a bit more complex.  As I recall from a previous question, you're also dealing with interior/exterior in those formulas at this time.

 

Sometimes a good way to build up these somewhat complex formulas is to build each piece separately and get each piece to work 'stand-alone' and then combine them into one larger formula.

 

Example, maybe you set up a VLOOKUP() to deal with interior/exterior door pricing based on width (that previous question of yours) in cell A1 and A2.  You get that working.

Then you build up a VLOOKUP() for hollow metal or solid core in A3 and A4 using references back to A1 and/or A2 where appropriate

Then you build up an IF() statement in A5 to choose results from earlier built up formulas and referring to them by their cell addresses.

Then finally you take that last formula and replace individual cell references in it with the actual formulas that appear in those cells, and voila! you have your complex formula.

manta---01 say: If you are using some other Bluetooth stack than Microsoft's own (like Widcomm), you need to manually set the phone to be the Bluetooth modem. Insturstions are in PC Suite help, but here also:
To install the modem drivers:
Activate the Dial-Up Networking profile from your Bluetooth software to install the standard Bluetooth modem to your computer. After the Dial-Up Networking Service has detected your phone and installed the standard Bluetooth modem, it starts creating a connection. Cancel this so that the Nokia modem can be installed using the same COM port (in step 6 below).

Connect a compatible phone to the PC using Bluetooth wireless technology. Note: If you are using Microsoft Windows XP Bluetooth, you can pair your phone with the PC in PC Suite’s Get Connected wizard. When using other Bluetooth stacks, you have to use the stack’s own pairing wizard.

In the Windows Control Panel, select Phone and Modem Options.

In the Phone and Modem Options, select Modems > Add.

The next dialog asks if you want Windows to detect your modem. Select Don't detect my modem; I will select it from a list. Click Next.

The next dialog asks you to select the modem you want to install. Select Nokia from the manufacturer list and your Nokia modem from the list on the right. Click Next.

In the next dialog, choose to use only Selected Ports. Select the same serial port that was created in step 1 to be used in Dial-Up Networking Service. Click Next. Depending on the PC’s settings, a Digital Signature warning may be shown during the installation. Press Yes to continue the modem installation.

Click Finish when the installation is complete. Your modem is now visible in the modem list. Click OK to close the Phone and Modem Options window.
