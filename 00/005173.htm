ken sheridan say:

Calling the OpenForm method while a form is open will filter the open form to whatever is the WhereCondition criterion.  So if you want to stick with the separate form rather than using a subform you can put the following in the Click event procedure of each of the combo boxes:

    Dim ctrl As Control

    Set ctrl = Me.ActiveControl

    If Not IsNull(ctrl) Then
        DoCmd.OpenForm "frm_Street_Names_Offs", _
             WhereCondition:="[StreetNameID]=" & ctrl.Column(0)
    End if

This will open the form if it's not already open, or filter it to the value in the current control's first column if it is open.  BTW, if the combo box's BoundColumn property is 1 you don't need to reference the column as the control's value will be that of the first column, so the following will be sufficient:

    WhereCondition:="[StreetNameID]=" & ctrl

 

Ken Sheridan, Stafford, England

marshall barton say:

You need the hidden text box so you can specify it as the LinkMaster.

It's difficult for me to imagine a form where you can not park a small, hidden text box, but if that's really an important issue, an alternative is to use the subform's Filter property in each combo's Click event:

With Me.subformcontrol.Form
   .Filter = "StreetNameID = " & LeaveBy1_ID_Combo.Column(0)
   .FilterOn = True
End With

The reason I am strongly biased to using LinkMaster/Child is because prior to A2007, the Filter property was too buggy to use with a subform.  Now that the bugs have been fixed, it seems like you might be willing to trade a text box for the additional lines of code needed to use the Filter property.

See Ken's comment about dropping the .Column(0)
