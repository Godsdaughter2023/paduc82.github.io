SELECT T1.[CustomerID], T1.TransactionDate, T1.Credit, T1.Debit,
SUM(T2.Credit-T2.Debit) - T1.Credit + T1.Debit AS OpeningBalance,
SUM(T2.Credit-T2.Debit) AS ClosingBalance
FROM TransactionsCD AS T1 INNER JOIN TransactionsCD AS T2
ON (T2.TransactionDate<=T1.TransactionDate)
AND (T2.TransactionID<=T1.TransactionID
OR T2.TransactionDate<>T1.TransactionDate)
GROUP BY T1.[CustomerID], T1.TransactionDate, T1.TransactionID, T1.Credit, T1.Debit
ORDER BY T1.TransactionDate DESC , T1.TransactionID DESC;

By returning the balances in computed columns in a query you are guaranteed that they will always reflect the values in the Credit and Debit columns.  Storing the balances in a column in the base table introduces redundancy and leaves the table open to the risk of update anomalies.

in preferences."  My preferences are already set that way, yet it keeps asking me to change it. What is the problem here?

daniel g. grau say:
Not all fonts are embedded in PDF files (I understand this depends on each font, for copyright reasons). If the font is not embedded, and if it is not be available in the target computer, it will be substituted. Therefore:

  • Use a common font, not a fancy one, or
  • Do Print > PDF > Save As PDF-X (this format requires font embedding), or
  • Open the resulting PDF in the Preview application, start to Print but click on the Preview button. This will take you back to the Preview application with a new file, where you should do a Save As PDF.
