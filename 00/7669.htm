If you use exactly the same parameter in several columns, Access will prompt only once for it.

scottgem say:
Well first, you have what is called a Repeating group, Using numbers to identify fields violates normalization rules. A better structure would be a sub table for reviewers.

tblReviewer
ReviewerID (PK Autonumber)
ParentID (FK)
PersonID (FK)
ReviewerRole (Primary/Secondary)

With this structure you can easily return all records that a person was a reviewer for.

As for prompting. I almost never use Parameter prompt queries. Instead I place a control on a form to enter the value I want to use to filter. This works because you can reference a value in any control on an open form using the syntax:

=Forms!formname!controlname

So you enter the value once and everytime its needed it pulls it from the form.

Updating Named labels referencing external ranges used in UserForm control RowSource property

I have a number of workbooks created by exporting individual tables from a SQLite database. [The SQLite export utility generates files with a '.csv' extension. I am developing a host workbook which reads and prepares the csv files for processing and directs subsequent operations based on user selection.  In each workbook, I define Named ranges for:

  • the last cell, 'LastC';
  • each label in the headline, the left-most label always being '_id';
  • the database expressed relatively via 'OFFSET(LastC,0,0,ROW(_id)-ROW(LastC),-COLUMN(LastC))';
  • and a new/existing record work area named 'NwRcrd' located above the headline again defined relatively via 'OFFSET(_id,-1,0,1,COLUMN(LastC))'.

In the host workbook, I create Names for each of the local definitions using external reference syntax; i.e. the Name 'object.LastC' is defined as 'object.csv!LastC' and 'object.database' as 'object.csv!database'. I also create definitions for each data column in the database; i.e. 'object.dbC._id' is defined as 'OFFSET(object.database,0,COLUMN(object._id)-1, ROWS(object.database),1)'.

The host workbook displays a userform which prompts the user to make one or more selections before proceeding to a lower level operation. The 1st step would be to make a selection from the combobox displaying the unique identifiers - i.e. Name - in the 'object.csv' table. This selection is facilitated by the RowSource specification for the combobox given as 'object.dbC.Name', which is correctly populated following the initial loading and preparation of the family of csv files.

The above selection then defines an '_id' which is used as a Foreign Key value, labelled 'column.ObjectID', when referencing the 'Name' field in the 'column.csv' table. As such, the 'object.dbC.Name' selection event causes the examination of the 'column.dbC.ObjectID' Named range to identify a sorted subset within the 'column.dbC.Name' range - named 'column.dbW.Name' - which then becomes the RowSource for the second-level combobox on the userform.

All the above works except for the final step: the second-level combobox does not get populated by its defined RowSource range. Upon termination of the macro, all ranges have been correctly defined and addressability from the host workbook can be demonstrated. Moreover, the RowSource property of the second-level combobox is displayed when the combobox is exercised within the IDE.

I have instituted Application.Calculate and userform REPAINT functions but to no effect.

Any ideas?

I discovered that, following the creation of the Named range, if I clear the RowSource property and then re-specify it, the combobox drop list is populated with the correct items.

Hi Insiders,

This is Patrick again from the Office Collaboration team. Ever have a document shared with you and then had trouble finding it again? Yeah, us too! To help with this issue, we’ve created a new Shared with Me feature. You can get an early look at the feature on Word, PowerPoint and Excel in the latest Office Insider Fast update for Office 2016 Version 1610 (Build 7329.1000 or greater.)

Here’s how it works: When someone shares a document, presentation, or spreadsheet with you, that file will automatically appear when you click Open > Shared with Me.

For the file to show up in Shared with Me , the shared document must be stored on OneDrive, OneDrive for Business, or SharePoint Online.

We’d love for you to take this feature through its paces, paying particular attention to the following:

  1. How long does it take for documents shared with you to show up in the list?
  2. Is the list accurate?

As an Office Insider Fast participant, you're first in line to see and use the Shared with Me feature. Don't hesitate to let us know how its working for you and where we might improve!

Thanks!
