a. user say:

Hi,


Thank you for posting your query on Microsoft Community.


I'd be glad to help you.

"How do I get to know why windows requires a reboot?"

Windows can’t update or remove files that are in-use. When Windows Update downloads new updates, it can’t apply them to Windows system files right away. Those files are in-use and locked against any changes. To actually install these updates, Windows has to shut down most of the operating system. Windows can then restart the system and load the updated files when it boots.

Reboots can also be necessary when updating or removing certain types of software. For example, if you’re using some sort of antivirus program or a hardware driver that hooks deep into the system, its files will be loaded into memory and protected against modification. When updating or removing in-use files, Windows may need to reboot the computer and modify the files before the system fully starts up.

Hi,

Thank you for writing to Microsoft Community Forums.

The Windows Precision Touchpad is a new class of input devices that provide high precision pointer input and gesture functionality. Microsoft’s precision drivers are made for Windows 10 and customized for ideal trackpad experience, however not all devices can run the precision touchpad drivers, though Microsoft has made precision touchpads a requirement on new hardware with newer versions of Windows 10.

You can check if your device already has the precision drivers installed.

Navigate to Settings->Devices->Touchpad.

If you see the Your PC has a precision touchpad text, it means the driver is already installed.

If the driver is not installed, you can follow the steps given below and see if you can get it to work.

  • Extract the downloaded drivers to a temporary folder.

  • Right click on Start menu (Windows logo) and select Device Manager.

  • Expand Mice and other pointing devices.

  • Right-click on Elan device and choose Update driver.

  • Select Browse my computer for driver software.

  • On next prompt, select Let me pick from a list of available drivers on my computer.

  • On the next page, click the Have Disk button at the bottom right corner of the window.

  • In the Install From Disk window, click the Browse button and navigate to the folder where you extracted the touchpad driver.

  • Select the Autorun.inf file in that folder, and then click the Open button.

  • Select ELAN Pointing Device option in the list of models, and then click the Next button.

  • You will see a warning saying that Windows can’t verify that it is compatible with your hardware.

  • Click on Yes to proceed with installation.

  • Reboot the computer once the installation completes.

If the touchpad doesn’t work after system reboot, roll back the driver to the previous version.

  • Extract the downloaded drivers to a temporary folder.

  • Right click on Start menu (Windows logo) and select Device Manager.

  • Expand Mice and other pointing devices.

  • Right-click on Elan and select Properties.

  • Click on Roll back driver to reinstall its previous version.

An unfortunately common fault with repair services is that they don't properly flash the default BIOS with the proper SLIC table - this results in a non-genuine status, as the SLIC table is required to be able to self-activate the OEM_SLP Key which most computers ship with.

 

The problem is that your system is unable to properly read the BIOS SLIC table that should be present in the system to be able to self-activate the OS.  

 

 You need to change the Product Key to the one on the COA sticker on the machine's case (or inside the battery compartment),  using the Change Product Key link at the bottom of the System properties page.

 

You may need to activate by phone.

zathjd say:

THIS FIX DOES NOT REQUIRE PREVENTING WINDOWS UPDATES It solves the problem by renaming the alpham driver associated with the zboard so windows no longer blocks it. It is a permanent solution and leaves your system in its original condition security-wise.


There are people who are posting the driver they generated with this process for download. Please recognize that it would be very easy to put malicious code into a driver installed in this manner. I strongly recommend just generating your own as described below. Download someone else's driver at your own risk.


0) If you don't already have ZEngine installed, you can get it here all the way at the bottom under ZEngine.

2) In the Kernel2K dir, rename every file that starts with Alpham or alpham to something else (ex. Alpham1.sys => Merc1.sys). 

3) Open the renamed .inf  file in a text editor (Notepad++, etc)  and do a similar find and replace on "Alpham" replacing with "Merc" (or whatever you used), replace all. (some people have reported needing to run their text editor as an administrator)

NOTE: Some people have reported being able to skip steps 4-7. You can try jumping from here directly to step 8. If it doesn't work for you, come back and start at step 4 again.

4) Delete merc12.cat (the security catalog file)

5) Install Windows Driver Kit ( it's going to give you a number of warnings during install that you won't be able to develop drivers since you don't have an IDE installed. Just OK through them... you're not developing a driver)

6) Add the inf2cat.exe installation directory to your PATH variables (Type path in search and select "Edit environment variables for your account" then select Path in the top window and click edit. Click new then add the directory of your inf2cat.exe file. Mine was in D:\Program Files (x86)\Windows Kits\10\bin\x86. Hit enter, then OK all open windows.)

7) Generate a new .cat file (open a Command Prompt and type: inf2cat/driver:"path" /os:10_X64,10_X86 where "path" is the directory of merc.inf. So, for me it was inf2cat /driver:"D:\Program Files\Ideazon\ZEngine\Kernel2k" /os:10_X64,10_X86)

9) In Device Manager locate the entries "Ideazon Merc MM USB Human Interface Device" and "Ideazon Merc USB Human Interface Device" (most likely under Human Interface Devices)


11) Enjoy a working zMerc again!
