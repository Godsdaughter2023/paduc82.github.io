BCCode: D1     0x000000D1

Cause

A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high.

This bug check is usually caused by drivers that have used improper addresses.

If the first parameter has the same value as the fourth parameter, and the third parameter indicates an execute operation, this bug check was likely caused by a driver that was trying to execute code when the code itself was paged out. Possible causes for the page fault include the following:

  • The function was marked as pageable and was running at an elevated IRQL (which includes obtaining a lock).
  • The function call was made to a function in another driver, and that driver was unloaded.
  • The function was called by using a function pointer that was an invalid pointer.

slightlytwisted say:

Just ran into this issue as well: an HP laptop would connect fine while connected to a power source, but would lose WiFi connectivity when running on battery. The issue started after upgrading to Windows 10.

After some digging around, it turned out the issue was related to this knowledge base: https://support.microsoft.com/en-us/kb/928152 (You may experience connectivity issues or performance issues when you connect a mobile PC that is running Windows to a wireless access point).

Essentially, the issue was that on battery power, Windows would put the laptop's wireless adapter into 802.11 power save mode and this mode which is not supported correctly by all wireless routers, causing connectivity to be lost. Going into the advanced power settings for battery mode and setting the adapter to "maximum performance" and reconnecting wifi as described in the KB fixed the issue.
