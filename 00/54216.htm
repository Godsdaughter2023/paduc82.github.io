holtto say:

Windows 10 upgrade crashes (error 0xC1900101 - 0x2000C)

This is my Windows 10 update story. I hope this will help someone else.

I have a Samsung 535U3C laptop.

It all started when i was stuck in tech preview (build 10074). When Windows 10 came out, i couldn't update to it. Windows Update didn't offer me the update and installing through MediaCreationTool it crashed.

At first, i backed up and formatted my drive C and installed Windows 10 in a clean drive. But then (of course) i faced problems with product key activation. So after some googling I quickly realized that i would have to go back to win 8.1 and update to Windows 10.

I downloaded Win 8.1 iso file from Microsoft website and installed it. That went in smooth and without errors. So I thought Win 10 would now install nicely when i had a clean Win 8.1 installed - YEAH RIGHT.

First I tried to update right from the first boot - error 0xC1900101 - 0x2000C.
At that point I started Googling again. I found many many many things people have tried to fix this 0xC1900101 - 0x2000C error (disk clean up, messing up with BIOS, installing with upgrade tool or iso file, disabling USB devices, increasing pagefile size and so on).
I think i tried them all. For real, I tried to install Windows 10 update like 30-40 times. Every single time it crashed quickly after first boot in copying files (~ 15%) phase. My laptop just went off. Booted up and boom, back to win 8.1.

But finally Christmas came! Before starting the installation, from iso file in my drive C, I disabled my Qualcomm Atheros AR946x Wireless Network Adapter and Qualcomm Bluetooth AR3012 Bluetooth device from Windows device manager. Of course i can't be 100% that disabling was the thing, but i think it's highly likely it was.

After installation i had to remove my WLAN and LAN adapters from device manager before my internet connection started to work.

But I won this fight. Windows 10 up and running smooth. Sorry for my bad english. :)
