ken sheridan say: Try changing the criterion so that it looks for a row with a an ExamNumber value of 0 if the control in the form is Null.  You'd also omit the If statement of course, so the code would become:

   Dim strCriteria As String

   strCriteria = "[ExamNumber] = " & Nz(Me.[ExamNumber],0)

    DoCmd.OpenForm "Exams", WhereCondition:=strCriteria, WindowMode:=acDialog, OpenArgs:=Me.Name
    
Assuming there cannot be an exam number of zero, the form will open filtered to no records, so will go to straight to an empty new record.  By passing the name of the calling form to the second form as its OpenArgs property, this will enable code in the second form's AfterInsert event procedure to requery the calling form and navigate to the newly inserted record.  This code would be:

    Dim frm As Form
    Dim lngExamNumber As Long

    If Not IsNull(Me.OpenArgs) Then
        Set frm = Forms(Me.OpenArgs)
        lngExamNumber = Me.ExamNumber
        frm.Requery
        With frm.RecordSetClone
            .FindFirst "ExamNumber = " & lngExamNumber
            If Not .NoMatch Then
                frm.Bookmark = .Bookmark
            End If
        End With
    End If
