When built-in functions like Date() and Format() stop working it is because of a broken reference in 99 out of 100 times.

Press Ctrl+G to go to the code window. At the top, click on Tools>References. See if any are marked MISSING. If so, uncheck them and find the latest version of those libraries below.

andreas killer say:

I am working on writing formulas with Absolute cell References and i AM NOT UNDERSTANDING CAN SOME ONE PLEASE HELP ME i NEED THIS TODAY THE QUESTION IS

Make a new file and write the formula =A1 into cell B1.

Select cell B1 and drag it one cell down. (Or copy B1 to B2)

Select cell B1 and drag it one cell to the right. (Or copy B1 to C1)

The result in B2 is the formula =A2 means the row has changed.

The result in C1 is the formula =B1 means the column has changed.

That is the behavior with a relative formula.

----------------

Make a new file and write the formula =A$1 into cell B1.
Select cell B1 and drag it one cell down. (Or copy B1 to B2)

Select cell B1 and drag it one cell to the right. (Or copy B1 to C1)

The result in B2 is the formula =A$1 means the row is as before.

The result in C1 is the formula =B$1 means the column has changed.

That is the behavior with a column relative and row absolute formula.

----------------

Make a new file and write the formula =$A1 into cell B1.

Select cell B1 and drag it one cell down. (Or copy B1 to B2)

Select cell B1 and drag it one cell to the right. (Or copy B1 to C1)

The result in B2 is the formula =$A2 means the row has changed.

The result in C1 is the formula =$A1 means the column is as before.

That is the behavior with a column absolute and row relative formula.

----------------

Make a new file and write the formula =$A$1 into cell B1.

Select cell B1 and drag it one cell down. (Or copy B1 to B2)

Select cell B1 and drag it one cell to the right. (Or copy B1 to C1)

The result in B2 is the formula =$A$1 means the row is as before.

The result in C1 is the formula =$A$1 means the column is as before.

That is the behavior with an absolute formula.

----------------

In short: The $ sign tells Excel: Do not change.

security essentials will not complete due to sleep mode issues - Microsoft Community

dougharrison say:

security essentials will not complete due to sleep mode issues

I have WIN7 with security essentials, that will not complete a full scan due to sleep mode issues. I am looking for a permanent solution other than a MAC
greginmich say:

In situations where a system task will be interrupted by sleep transitions, many people will probably try to avoid having to wait for the task to finish by increasing the “Sleep after” time interval, or by just turning off the sleep feature altogether. And I’m afraid that many of those people won’t comply with the suggestion to return the sleep setting to its original state after each task, as Steve and others here have conscientiously recommended. So I have some reservations about this revamping of the sleep logic, and feel that it could actually end up working contrary to the goals defined by the Energy Star program. This concern is reinforced by the fact that the Microsoft documentation doesn’t make any mention of toggling the sleep setting (How can I prevent my computer from automatically sleeping or hibernating?):

 

http://windows.microsoft.com/en-us/windows7/Sleep-and-hibernation-frequently-asked-questions

 

http://www.energystar.gov/index.cfm?c=power_mgt.pr_power_mgt_manual_act_win7

 

While there isn’t any answer for this idling-to-sleep issue that doesn’t involve a little work, there is a simple alternative that solves the idling-to-sleep issue without any modification of the sleep setting, and can thus help keep things compliant with the Energy Star guidelines. The alternative solution derives from the fact that the limitations that we see in the MP Scheduled Scan task (the inability to wake the computer or to keep the computer awake) simply don’t exist in user-defined Task Scheduler scan tasks. So in order to “reprogram” the MP Scheduled Scan task with these “missing” options, all you really have to do is create your own Task Scheduler scan task, and then enable the “Wake the computer” option. This will keep the computer awake for the duration of the scan so that you don’t have to change your power settings or fiddle around on your computer to keep the scan running. The “stay awake” mode might have been designed primarily to keep unattended tasks running, but it’s something that you can also use to keep the computer awake when you schedule your scan for the working hours in order to help conserve energy, as I suggested here:

 

http://answers.microsoft.com/en-us/protect/forum/protect_start/scheduled-scans-not-working/fd769043-f67e-4990-8805-a02427f22dff

 

When you set up a scheduled scan task with the default settings, the scans will run at full speed; which is fine for overnight scans, or for working-hours scans on powerful computers. But this might be a problem for working-hours scans on computers with marginal resources, because the scan task will tend to consume system resources to the point where the system’s responsiveness is affected. So if you need to throttle down your scans in order to run them during a work session, you can do this by invoking the NT Authority\System account: Simply click on the “Change User or Group” button and enter “System”. This will cause your scan task to run through the Security Essentials GUI (main interface), and it will then obey the CPU-usage setting there, while still retaining the ability to wake the computer and keep it awake for the duration of the scan. This configuration thus gives you the advantages of the MP Scheduled Scan task without the limitations, and allows you to take full control of your scheduled scans.

 

Now, while you can’t directly “reprogram” the Security Essentials “Scan now” scans, notice that once your task has been set up in Task Scheduler, you do have a “functional equivalent”, because the task can be run “on-demand” by right-clicking on it and selecting “Run”. Or better yet, you can set up a desktop shortcut: Right-click on the desktop; select “New”, and then “Shortcut”; and then enter this path for the “location of the item”:

 

C:\Windows\System32\schtasks.exe /run /tn "TaskName"

 

Put the name of your scan task in quotes. Then right-click on the shortcut and select “Properties”; click on the “Advanced” button; and check “Run as administrator”.

 

Basic Tasks are extremely easy to set up; just try it and see: Task Scheduler > Action menu > Create Basic Task

Browse to: C:\Program Files\Microsoft Security Client\Antimalware\MpCmdRun.exe

Then enter these “arguments” for a Full scan:   -Scan -ScanType 2

Or these “arguments” for a Quick scan:   -Scan -ScanType 1

 

http://windows.microsoft.com/en-US/windows7/Schedule-a-task

prevent spaces being entered into a cell - Microsoft Community

everynameseemstobetaken say:

prevent spaces being entered into a cell

Hi,

Is there a way i can prevent spaces being entered into a cell by a user.  There is bit of a twist to this one in that the first three characters will always be "MMP"

I'm getting MMP 123 rather than MMP123 and its playing havoc with my Lookup!!

 

Thanks

John

 

jllatham say:

One way to do it would be with Data Validation.  Choose the cell and then Data --> Data Validation.  Choose a Custom rule and enter this for the formula (assumes the cell in question is B2):

=B2=SUBSTITUTE(B2," ","")

Add a custom error message that tells the user to make the entry without any space characters.  This would take care of the problem at the "front end" at the expense of some annoyance to the user, but eventually they'll get trained not to enter spaces.

If you want to not "bother" the end user up front, you could alter your lookup formula in much the same fashion, like:

=VLOOKUP(SUBSTITUTE(B2," ",""),R1:X99,3,False)

I actually prefer the first one because it keeps the entries looking like they should in the workbook and any added formulas using those values later on don't need to know the secret handshake of using the SUBSTITUTE() function to get them to work.
