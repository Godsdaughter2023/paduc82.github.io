williamwhitlow say:

Delete item in Word Index

In Word 2016, I have created an index in a document. I have never done this before, so I am struggling. Assume the word “cheese” is one of the entries in the index. This word appears multiple times in the document. I have decided I want to remove this word from the index. What is the procedure for deleting a word from the index?

jay freedman say:

The entries in the index are pulled from XE fields in the body of the document, which you created by using the Mark Entry dialog box. The XE fields are all marked with Hidden font formatting, so normally they're invisible. To remove a particular entry from the index, you have to delete every XE field that mentions that entry, and then update the index.

Click the ¶ button on the Home tab to display nonprinting and hidden characters, so you can see the XE fields. Open the Replace dialog. In the Find What box, enter

^19 XE "cheese"

The code ^19 matches the opening field brace, the XE is the field code, and the word or phrase for the entry, in quotes -- for your example it's "cheese" but substitute whatever term you want to delete from the index.

Leave the Replace With box empty, and click the Replace All button. On completion, Word will tell you how many instances it removed.

Finally, click the ¶ button again to turn off the invisible characters (this is important so the visible XE fields don't mess up the page numbering). Right-click in the index and choose Update Field in the menu.

This is the best I could do in Word's equation editor:

It's built as three separate equations in three paragraphs, the top and bottom lines and the π labels in the middle. The small labels are in the superscript portion of a "script" template rather than in an arrow operator. The arrows are drawn with the Insert > Shapes tool.

DarrenW_310,

Do you have cross-project links (i.e. links between tasks in different subprojects)? If so then you will see "repeated" tasks in the hierarchy of the timeline task selection window. That's because cross-project links create "ghost" tasks in each subproject and although you won't see the ghost tasks in the Gantt display at master level, (since the real task is already displayed), they do appear in the hierarchy.

What I do see is duplication of tasks. If you select a task via the hierarchy window and then also add the same task by right clicking, you will see the task duplicated on the timeline.

It won’t overwrite the entire values stored in the ProxyAddresses attribute.
