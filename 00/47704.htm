songoku'sson say:
Hello Ashu! Here's a few solutions.
1. In Task Manager select the 'Server Busy' program, right-click and select 'Go to Process'. This should tell you what program is causing the error. Either edit or uninstall the program.

2. Run 'msconfig' and untick any unecessary programs BUT don't remove something unless you know what it is. A quick google search should provide you with information on any oddly named processes.

3. Create a new user account. With a fresh user account the process that is starting will not run anymore. Simply migrate over any documents/music/etc. to the new account.

4. Uninstall possible suspect programs, particularly ones that have been recently installed or undergone a significant update.

A repair upgrade is the simple process of installing a new build over the existing installation of Windows 7, using your installation DVD. What this does is replace broken operating system files while preserving your personal files, settings and installed applications.

 

Doing the Repair Update

The repair upgrade is a straightforward process. Simply boot to your Windows 7 desktop. Insert your Windows 7 installation DVD, then click Install Now.

When the setup menu appears, click Upgrade.

Setup will begin copying files. This will also replace any broken operating system files that might be preventing Windows from functioning properly.

There you have it: your personal files and installed applications are preserved. I found this especially handy for fixing corrupt operating system features. [quote user="Gen Reaqz"]und was ist ipv6?[/quote]

IPv6 ist der Nachfolger von IPv4. Das ist das Protokoll mit dem sich Geräte im Internet gegenseitig finden. Du kannst entweder lange Wikipediaartikel darüber lesen, oder wenn es Dir nicht so genau um die Details geht, hier die Kurzzusammenfassung mit nicht 100%ig faktischer Genauigkeit:

IPv4 beschreibt die IP Adressen, wie Du sie von "Zuhause" vielleicht schon kennst. Jedes Netzwerkgerät in einem IP Netzwerk (sei es nun das Weltweite Internet oder Dein LAN bei Dir zuhause) muss eine einzigartige IP Adresse besitzen, um mit anderen Geräten Daten austauschen zu können.

Das sind die Adressen die meistens so aussehen: 123.45.67.89

Leider haben damals die Erfinder von dem IP Protokoll nicht mit dem Erfolg des Internet gerechnet, und nun gibt es mehr Internetgeräte als es freie IPv4 Adressen gibt.

Die letzten freien IP Adressen wurden 2011 an Internet Provider verkauft.

Um den Adressmangel von "nur" knapp 4 Milliarden IPv4 Adressen zu beheben, wird nun IPv6 so langsam eingeführt. In dem Adressbereich haben dann 360 Sextillionen Geräte eine eigene Adresse. Um einen Begriff von dieser riesigen Zahl zu haben: Jedes Sandkorn auf der Erde könnte seine eigene IPv6 Adresse bekommen, und man hätte immer noch Adressen übrig.

Jetzt ist aber das Problem, dass IPv6 ein klein wenig anders seine Geräte addressiert als IPv4. Damit also auch alte Geräte, die kein IPv6 können noch im Internet "mitreden" können, gibt es verschiedene Wege dies zu bewerkstelligen.

Ideal wäre natürlich der Dual Stack Betrieb, bei dem jedes Gerät neben der IPv6 Adresse auch eine IPv4 Adresse bekommt.

Nun ist es bei UnityMedia/KabelBW aber so, dass sie salopp gesagt nun mehr Kunden als freie IPv4 Adressen haben. Daher müssen die das anders machen. Unity Media hat sich glaube ich für DS-Lite und Carrier Grade NAT entschieden.

Wenn Du Dir meinen NAT Post mal durchliest und kapiert hast, was der Router bei Dir zuhause macht, musst Du Dir das nur "größer" vorstellen. Der Router sitzt nun im Rechenzentrum von UnityMedia, hat ein paar wenige öffentliche IP Adressen, und "versteckt" dahinter viel mehr Kunden-Anschlüsse.

Und da kommen auch die Probleme mit Xbox Live zustande.

Von daher, da diese Art wie UnityMedia/KabelBW Dich ins Internet Verbindet nur für Websurfen und E-Mail so richtig funktioniert, bei Serveranwendungen wie Multiplayerspielen und Party-Sessions auf Xbox Live aber enorme Probleme macht, solltest Du schnellstmöglich versuchen, denen klar zu machen, dass UnityMedia / KabelBW ihren Teil des Vertrags nicht so erfüllen können, wie Du es beim Vertragsabschluss erwartet hättest.

Dummer Autovergleich:

Du bist in einem Autogeschäft, und der Verkäufer empfiehlt Dir einen super schnellen Sportwagen mit niedrigem Sportfahrwerk und superben Fahreigenschaften. Du bist begeistert und geblendet von der rasenden Geschwindigkeit. Du kaufst also den Wagen, und stellst fest, dass Du mit dem tiefergelegten Fahrwerk nicht über den Bordstein in Deine Garageneinfahrt fahren kannst.

UnityMedia wird Dir also sagen, dass mit dem Auto alles in Ordnung ist, weil Du kannst ja auf Autobahnen und normalen Straßen prima fahren. Also ist an dem Auto nichts kaputt oder defekt.

Da Du aber über einen Bordstein fahren musst, oder auch mal über einen Feldweg fahren willst, war Dir das beim Kauf nicht bewusst dass das Auto das nicht kann, und daher Du den Vertrag rückabwickeln willst, weil das Auto nicht zu Deinen Anforderungen passt.

UnityMedia/KabelBW ist für "normale" 08/15 User die einfach nur Surfen und E-Mailen wollen eine prima Sache. Schnelle Kabelnetze und all das sind für 90% der Leute genau das was sie wollen und brauchen.

Aber die anderen 10% die eben einen Server auf ihrem Rechner laufen haben wollen oder Xbox Live nutzen wollen, fallen hier eben auf die Nase, weil UM/KabelBW eben so "komisch" den Übergang auf IPv6 machen.



TL;DR


Du hast also eigentlich nur zwei Möglichkeiten wie Du das lösen kannst:

Verlange eine Vertragsänderung und eine ECHTE IPv4 Anbindung. In Ausnahmefällen (und wenn noch Adressen frei sind) sollte das mit viel Drohgebaren und ständigem Rumnerven beim Support noch drin sein. Drohe viel mit Vertragskündigung.

Oder kündige den Vertrag komplett und wechsle zu einem Provider der ECHTEN Dual-Stack Betrieb anbietet.

Du selber kannst zuhause GARNICHTS ändern oder einstellen. Du MUSST UnityMedia/KabelBW dazu überreden Dir einen funktionierenden Anschluss zu liefern, oder Du musst eben kündigen. Einen anderen Weg gibt es nicht.

If you don't know where DMP files are located, here's how to get to them:

 1.    Navigate to the %systemroot%\Minidump folder.
 2.    Copy any and all DMP files in the Minidump folder to your Desktop and then zip up these files.
 3.    Upload the zip containing the DMP files to Skydrive or a hosting site of your choice and paste in your reply.

If you are going to use Skydrive but don't know how to upload to it, please visit the following:

http://www.wikihow.com/Use-SkyDrive

Please note that any "cleaner" programs such as TuneUp Utilities, CCleaner, etc, by default will delete DMP files upon use.

If your computer is not generating DMP files, please do the following:

1. Start > type %systemroot% which should show the Windows folder, click on it. Once inside that folder, ensure there is a Minidump folder created. If not, CTRL-SHIFT-N to make a New Folder and name it Minidump.

2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

4. Double check that the WERS is ENABLED:

Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. Also ensure that under Startup Type it is set to Automatic rather than Manual. You can do this by right clicking it, selecting properties, and under General selecting startup type to 'Automatic', and then click Apply.

Patrick
