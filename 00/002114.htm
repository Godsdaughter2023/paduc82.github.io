iry chen msft say:

Hi Emily,

According to your description and capture, it seems the full access permission of the user has been removed, to check it, please run the PowerShell command as below:

  1. Connect to Exchange Online PowerShell.
  2. Run the command: Get-MailboxPermission -Identity <Mailbox ID1> -User "user name"

If you get the result for example: AccessRights is {FullAccess}, it means the permission hasn’t been removed, if you get nothing or others, it means you have removed the permission.

If it still hasn’t been removed, please try to disable the automapping as below steps:

   1. Connect to Exchange Online PowerShell.

   2. Run the command (in order to add back the full access permission and disable the automapping):

        Add-MailboxPermission -Identity <Mailbox ID1> -User <Mailbox ID2>-AccessRights FullAccess -AutoMapping:$false  

   3. Run the following command to remove the user’s full access permission again:

        Remove-MailboxPermission -Identity <Mailbox ID1> -User <Mailbox ID2> -AccessRights FullAccess

(Please note that the Mailbox ID1 is the shared mailbox and the Mailbox ID2 is the user’s mailbox)

Reference link: https://technet.microsoft.com/en-us/library/jj984289(v=exchg.160).aspx

                         https://technet.microsoft.com/en-us/library/aa998218(v=exchg.160).aspx

                         https://support.microsoft.com/en-sg/help/2646504/how-to-remove-automapping-for-a-shared-mailbox-in-office-365

Please contact us if you have any concern or update.

Thanks,

Iry

andre for directly say:
Press Windows key + R
Type each of the following commands then hit Enter to open the respective application.

winword
excel
powerpnt
msaccess
outlook
mspub

This should open Word, Excel, PowerPoint, Access, Outlook and Publisher.

You can then right click and pin each to the Taskbar if you wish.

If that does not work..

Use the fix it tool to completely uninstall Office:
https://support.office.com/en-us/article/uninst...

Proceed to reinstall Office 2019 from the ISO again.

If that does not work...

Could you confirm if this also happens in the default Administrator account?

Open Start
Type: cmd
Right click CMD
Click Run as administrator

At the command prompt, type:
net user administrator /active:yes

Hit Enter

Close command prompt, restart, then try signing into the Administrator account.

Disable your Antivirus

Try running the file again.
