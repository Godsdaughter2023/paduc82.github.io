Make a list of addresses in word 2016 - Microsoft Community

xazos say:

Make a list of addresses in word 2016

Hi, so I already have created a list of names and addresses in Excel.  Now I would like to print the list using word.  I would like to print them in 3 columns alphabetically going down the column such as in a phone book not across.  How do I do that?

doug robbins - mvp office apps & services (word) say:
  1. Set up your document with the three columns using the Columns facility in the Page Setup section of the Layout tab. 
  2. Then on the Mailings tab, expand the Start Mail Merge dropdown and select Directory.
  3. Use the Select Recipients item on the Mailings tab and select Use and existing list and browse to and select the Excel workbook in which you have the list of names and addresses.  (It is assumed that the first row of the worksheet will have headings such as Name, Address, City, State, etc)
  4. Using the Insert Merge Field button on the mailings tab, insert one set of the field names in the configuration that you want them.
  5. Save the document
  6. Expand the Finish & Merge facility in the Finish section of the Mailings tab and then select Edit Individual Documents as the destination to which the merge should be exectuted.

It is probably best to use Shift+Enter when you want to start a new line when inserting the fields, so that all of the address will be in the one paragraph and format that paragraph so that the lines are kept together and so that it has a space of (say) 12 pts after it.

Then when you execute the merge, each entry will be separated by that space and you will not have entries split across two columns.

You could achieve that end without needing to make any changes to the data source, by using a field in your mailmerge main document coded as:

{QUOTE{IF{MERGESEQ}= 1 {SET Alpha ""}}{SET NameRef {MERGEFIELD Last_Name}}
{IF{REF NameRef }= "A*" "{SET Code "A"}"}
{IF{REF NameRef }= "B*" "{SET Code "B"}"}
{IF{REF NameRef }= "C*" "{SET Code "C"}"}
{IF{REF NameRef }= "D*" "{SET Code "D"}"}
{IF{REF NameRef }= "E*" "{SET Code "E"}"}
{IF{REF NameRef }= "F*" "{SET Code "F"}"}
{IF{REF NameRef }= "G*" "{SET Code "G"}"}
{IF{REF NameRef }= "H*" "{SET Code "H"}"}
{IF{REF NameRef }= "I*" "{SET Code "I"}"}
{IF{REF NameRef }= "J*" "{SET Code "J"}"}
{IF{REF NameRef }= "K*" "{SET Code "K"}"}
{IF{REF NameRef }= "L*" "{SET Code "L"}"}
{IF{REF NameRef }= "M*" "{SET Code "M"}"}
{IF{REF NameRef }= "N*" "{SET Code "N"}"}
{IF{REF NameRef }= "O*" "{SET Code "O"}"}
{IF{REF NameRef }= "P*" "{SET Code "P"}"}
{IF{REF NameRef }= "Q*" "{SET Code "Q"}"}
{IF{REF NameRef }= "R*" "{SET Code "R"}"}
{IF{REF NameRef }= "S*" "{SET Code "S"}"}
{IF{REF NameRef }= "T*" "{SET Code "T"}"}
{IF{REF NameRef }= "U*" "{SET Code "U"}"}
{IF{REF NameRef }= "V*" "{SET Code "V"}"}
{IF{REF NameRef }= "W*" "{SET Code "W"}"}
{IF{REF NameRef }= "X*" "{SET Code "X"}"}
{IF{REF NameRef }= "Y*" "{SET Code "Y"}"}
{IF{REF NameRef }= "Z*" "{SET Code "Z"}"}
{IF{REF Code}<>{REF Alpha} "{SET Alpha {REF Code}}{REF Alpha \* Charformat}"}}

As you'll see, except for the first and last lines, the above is all just a series of fields coded as:

{IF{MERGEFIELD Last_Name}= "A*" "{SET Code "A"}"}

with just two letters being changed. The formatting applied to the R in the final REF controls the output display format. The line breaks aren't really needed, either - they just make it easier to understand the field construction.

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues.
