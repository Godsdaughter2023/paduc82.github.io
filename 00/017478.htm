john korchok say:

Sorry to bear bad news, but master documents are not a reliable form of Word document over the long term. Eventually, file corruption will claim your work. The best use for master documents is to create them for printing a collection of child documents, then deleting the master. It's not hard to write a macro to automate this process.

To answer your question, you can create a hyperlink to a heading or a bookmark in your child document. Then follow these steps:

  1. With your cursor in the master document, choose Insert>Hyperlink from the menu at the top of your screen, or Insert>Link on the Ribbon.
  2. Choose This Document.
  3. Expand Headings or Bookmarks (in your case Bookmarks) by clicking on the arrowhead to the left.
  4. Select a Heading or Bookmark that is in the child document, then click on OK.

If the data is on some other sheet, you can easily move that sheet so that it is the first sheet in the workbook by clicking on the sheet tab and dragging it to the left.

  3. The field names must be in the first row of the worksheet and there must be a field name in all of the cells in that row that are within the range of columns that contain the data.

  4. The field names must contain only alphanumeric characters (No @,#,$,%,&,(,), etc) and the field names must not start with a numeric character (0-9).

  5. The data must start in the second row of the worksheet and there should be no empty rows within the range of data that is to be processed.

You may also want to download:

  1. the Merging with Attachments document that is also on that page which explains how the system is used.  It is not actually necessary to have separate attachments as the facility can be used to send just the documents created by the merge itself as attachments, either as the body of the message itself or in the form of Word files or .pdf files.

  2. the Mail Merging with Charts document that is also on that page.  That document explains how you must set up the Excel Data Source and the Mail Merge Main document to be able to execute a merge with a Chart that is unique to each record in the data source.

  3. the Using the Many to One Facility document that describes how to use that facility.

I have data for captains and first officers throughout a month. I wish to display this data in a chart. Hours worked and Hours off. I do not want to stack the data, I would like to overlap. I need to do this for captains and first officers seperatly but on the same chart. tom-s say:

David_t87,

Suggested layout which I think is something like what you're after:

Set out data as follows - A3 down is dates; B2 column header FO on duty, with B3 down the data; C2 column header FO off duty, with C3 down the data; D2 column header Cpt on duty, with D3 down the data; E2 column header Cpt off duty, with E3 down the data.

Make a column chart with columns B:E forming 4 separate series, and the dates along the X horizontal) axis.

On the chart right click one of the Cpt on duty bars and select Format Data Series from the menu which appears; then click the Axis tab of the Format Data Series dialogue box, and select Plot series on Secondary axis, then click OK.

Do the same thing for the Cpt off duty series but before clicking OK click the Options tab and select -100 for the Overlap value, then click OK.

Then right click either of the FO series, select Format Data Series, click the Options tab and select -50 for the Overlap value, then click OK.

When that's all done you should find you have 2 sets of overlapping bars for each date, with the Cpt on duty & Cpt off duty bars in the foreground.
