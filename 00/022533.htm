Forget the idea.  If you use Excel, the spreadsheet will be downloaded, unless you use the spreadsheet as a database, and then any interactivity will be whatever you can provide in PHP, asp or asp.NET.  This is NOT trivial. And SQL Server , mySQL or Access will provide a much better (and easier to use) database than Excel.

Plan B is also doomed to failure. The spreadsheet will always be downloaded to the temporary Internet Files folder from wherever it is stored, unless it is stored on a shared drive. In this (shared drive) scenario, a website is superfluous, and access to the spreadsheet is restricted to those using your LAN, or who have access to your LAN via a VPN connection.

If you need a website, then the database solution is the answer, with interactivity provided by PHP, asp or asp.NET.  Visual Studio Web Express (VSWE) is a better tool for SQL Server and asp.NET, and Expression Web for PHP - FrontPage is a non starter for the required extensive server side programming.  I think (never used it) Dreamweaver has more support for "classic" asp than FrontPage VSWE or Expression Web (which have no support).

Cleaning up a messy excel spreadsheet. - Microsoft Community

justinhadden say:

Cleaning up a messy excel spreadsheet.

I have been given a very messy spreadsheet that I would like to clean up.

Example:

FirstName PartnerFirstName LastName Address1 Address2 City State Zip
Gary Man
123 Overthere St.
Map KT, 44557
Pop Guy & Jesse Guy
456 Overhere Dr.
Jasper GA, 76492

And it goes on like this for about another 6000 rows. Is there a workaround I can use to make cleaning up easier? Or am I going to be stuck manually cleaning this up for the next few days?

 

jllatham say:

Well, perhaps at least not as much cleanup - or maybe a worse mess than you have.  Try this and see.

The code below is rather primitive (doesn't even attempt to split 2-line street addresses) and may or may not give you something you can work with some more.  But it does not destroy the original data, so at least you'd have that left if it doesn't do what you want.

Other potential problems:  last names like "St. James" or "du Pont" they'll get split and end up as "James" and "Pont".  Same for Jr, Sr, II, III etc.

To put the code into your workbook, open it up, press [Alt]+[F11] to get into the visual basic editor.  In there, choose Insert --> Module from the menu and copy the code below and paste it into the module presented to you.  Make any changes needed to the Const values that are bold below.  Close the VB editor by pressing [Alt]+[Q].  IMPORTANT: Save the workbook as a macro enabled workbook, type .xlsm or .xlsb.

Try it out by going to the [View] tab (or [Developer] tab if you have that shown) and running the BasicCleanup macro.  You should end up with a new sheet named CleanedUp and that's where the revised data ends up at.

Here's the code:

Sub BasicCleanup()
'Note this is a rough attempt, vagaries of data entry can disrupt results
'it will create a 2nd worksheet named CleanedUp and move the data it
'thinks it has found onto that sheet, leaving the original data untouched
' Assumptions:
'     a group of data consists of 3 rows of information:
'       name
'       street address (may be difficult to split 2-line addresses)
'       city, st zip
'*************************************************************************
'change these Const values as needed to align with your worksheet layout
  Const dataSheetName = "Sheet1" ' change as needed
  Const firstDataRow = 2 ' first row with the very first name entry
  Const dataCol = "A" ' column the data to work with is in
  Const firstNameCol = "A" ' column we will put first name only into
  Const partnerNameCol = "B"
  Const lastNameCol = "C"
  Const add1Col = "D"
  Const add2Col = "E"
  Const cityCol = "F"
  Const stateCol = "G"
  Const zipCol = "H"
'end of user redefinable values
  Const RevisedDataSheetName = "CleanedUp"
  Dim dataWS As Worksheet
  Dim newWS As Worksheet
  Dim dataRange As Range
  Dim anyDataCell As Range
  Dim itemCount As Integer ' 1=names, 2=streets, 3=city, st zip
  Dim rawData As String
  Dim nextRow As Long
  '
  Dim theLastName As String
  Dim theFirstName As String
  Dim thePartnerName As String
  Dim tempText As String
  '
  Dim street1Text As String
  Dim street2Text As String
  '
  Dim theZip As String
  Dim theState As String
  Dim theCity As String
 
  Set dataWS = ThisWorkbook.Worksheets(dataSheetName)
  Set dataRange = dataWS.Range(dataCol & firstDataRow & ":" _
   & dataWS.Range(dataCol & Rows.Count).End(xlUp).Address)
 
  On Error Resume Next
  Set newWS = ThisWorkbook.Worksheets(RevisedDataSheetName)
  If Err <> 0 Then
    'sheet does not exist, create it
    Err.Clear
    On Error GoTo 0
    ThisWorkbook.Worksheets.Add After:=dataWS
    Set newWS = ActiveSheet
    newWS.Name = RevisedDataSheetName
  End If
  On Error GoTo 0
  newWS.Cells.Clear
  'set up headers on new worksheet
  With newWS
    .Range(firstNameCol & 1) = "First Name"
    .Range(partnerNameCol & 1) = "Partner Name"
    .Range(lastNameCol & 1) = "Last Name"
    .Range(add1Col & 1) = "Address 1"
    .Range(add2Col & 1) = "Address 1"
    .Range(cityCol & 1) = "City"
    .Range(stateCol & 1) = "State"
    .Range(zipCol & 1) = "Zip Code"
  End With
  dataWS.Activate
  Application.ScreenUpdating = False ' prevent flicker, make faster
 
  itemCount = 0 ' initialize
  For Each anyDataCell In dataRange
    itemCount = itemCount + 1
    If IsEmpty(anyDataCell) Or itemCount > 3 Then
      itemCount = 0 ' start new group
    End If
    If Not IsEmpty(anyDataCell) Then
      rawData = Trim(anyDataCell)
     
      Select Case itemCount
        Case Is = 1
          nextRow = newWS.Cells.Find(What:="*", _
           After:=newWS.Cells.Cells(1), _
           Lookat:=xlPart, _
           LookIn:=xlFormulas, _
           SearchOrder:=xlByRows, _
           SearchDirection:=xlPrevious, _
           MatchCase:=False).Row + 1
         
          'parse names entry
          'if it has & or " and " in it, assume 2 names
          If InStr(UCase(rawData), " AND ") > 0 _
           Or InStr(rawData, " & ") > 0 Then
            'retest for specific
            If InStr(rawData, " & ") > 0 Then
              theFirstName = Trim(Left(rawData, InStr(rawData, " & ") - 1))
              tempText = Trim(Right(rawData, Len(rawData) - (InStrRev(rawData, " & ") + 2)))
              theLastName = Trim(Right(theFirstName, Len(theFirstName) - (InStrRev(theFirstName, " "))))
              theFirstName = Trim(Left(theFirstName, Len(theFirstName) - Len(theLastName)))
              thePartnerName = Trim(Left(tempText, Len(tempText) - Len(theLastName)))
            Else
              theFirstName = Trim(Left(rawData, InStr(UCase(rawData), " AND ") - 1))
              tempText = Trim(Right(rawData, Len(rawData) - (InStrRev(UCase(rawData), " AND ") + 2)))
              theLastName = Trim(Right(theFirstName, Len(theFirstName) - (InStrRev(theFirstName, " "))))
              theFirstName = Trim(Left(theFirstName, Len(theFirstName) - Len(theLastName)))
              thePartnerName = Trim(Left(tempText, Len(tempText) - Len(theLastName)))
            End If
          Else
            'assume single name
            theLastName = Trim(Right(rawData, Len(rawData) - InStrRev(rawData, " ")))
            theFirstName = Trim(Left(rawData, Len(rawData) - (Len(theLastName) + 1)))
            thePartnerName = ""
          End If
          With newWS
            .Range(firstNameCol & nextRow) = theFirstName
            .Range(partnerNameCol & nextRow) = thePartnerName
            .Range(lastNameCol & nextRow) = theLastName
          End With
         
        Case Is = 2
          'parse street addresses
          'until more info is obtained, just treat this as a lone line address
          street1Text = Trim(rawData)
          With newWS
            .Range(add1Col & nextRow) = street1Text
          End With
         
        Case Is = 3
          'parse city, state zip
          theZip = Right(rawData, Len(rawData) - InStrRev(rawData, " "))
          rawData = Trim(Left(rawData, Len(rawData) - Len(theZip)))
          theState = Right(rawData, Len(rawData) - InStrRev(rawData, " "))
          rawData = Trim(Left(rawData, Len(rawData) - Len(theState)))
          'more processing of theState - get rid of comma
          theState = Trim(theState)
          If Right(theState, 1) = "," Then
            theState = Left(theState, Len(theState) - 1)
          End If
          theCity = Trim(rawData)
          rawData = ""
          With newWS
            .Range(cityCol & nextRow) = theCity
            .Range(stateCol & nextRow) = theState
            .Range(zipCol & nextRow) = theZip
          End With
        Case Else
          'do nothing
      End Select
    End If
  Next ' end anyDataCell loop
 
End Sub
