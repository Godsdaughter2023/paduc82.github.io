a. user say:

If nothing prevents you from using VBA, then I suggest you attempt the following, which relies on the DATABASE technique. If that isn't possible for you, can you at least try to confirm the data structures you are dealing with, as you will then really have to revert to the approach you started with, and it's much easier to advise if people know exactly what the field names are, and so on.

Let's say that 

1. your Excel workbook is called c:\wbs\mywb.xlsx

2. your sheet in Excel is called mysheet

3. the columns in Excel are labelled as follows:

Case_Number 
Doing_Business_As 
Address_1_Street_2 
Address_1_State_Province
Postal_Code 
Device_Cost
Device_Serial
Language
Email

4.Language can be "E" or "F"

5. You have multiple rows for the same Case_Number, for for a given Case_Number, only the Device_Cost, Device_Serial vary (i.e. you are using your Language Filter to decide which Case_Numbers to report in which language). 

6. You are emailing the results.

Then your mail merge main document needs to connect to a data source that has one row for each Case_Number for one of the languages, containing the following two columns:

Case_Number
Email

You can do that by opening your Word Mail Merge Main Document, starting up the VBE, copying the following code into a module, and running it:

Sub myconnect()
ActiveDocument.MailMerge.OpenDataSource _
  Name:="c:\wbs\mywb.xlsx", _

  SQLStatement:= _

    " SELECT DISTINCT" & _

    "   Case_Number," & _

    "   Doing_Business_As," & _

    "   Address_1_Street_2," & _
    "   Address_1_State_Province," & _
    "   Postal_Code," & _

    "   "Email " & _

    " FROM [mysheet$]" & _

    " WHERE Language = 'E'" & _

    " ORDER BY 1"
End Sub

6a. (For the French language document, you would need the appropriate Language test).

7. Then you should be able to insert the merge fields for Case_Number, Doing_Business_As, Address_1_Street_2, Address_1_State_Province and Postal_Code in the usual way.

8. Where you need the list of items/prices, insert the following nested field (to begin with)

{ DATABASE \d "c:\\wbs\\mywb.xlsx" \s "SELECT m.Device_Serial, m.Device_Cost FROM [mysheet$] m" }

Please notice that in the field code, the \ characters are doubled up.

9. Execute that field code and ensure that it works. It should retrieve all the records. Whatever you do, do not try to execute the merge until the DATABASE field is working correctly

10. Then modify the field by inserting a WHERE condition so that the whole thing looks like 

{ DATABASE \d "c:\\wbs\\mywb.xlsx" \s "SELECT m.Device_Serial, m.Device_Cost FROM [mysheet$] m WHERE m.Case_Number = { MERGEFIELD Case_Number }" }

The { } around the MERGEFIELD field must be the special field code braces that you can insert using ctrl-F9.

If the Case_Number is numeric, that should beOK, but if it is a non-numeric string, you will need to put single quotation marks around it, like this:

{ DATABASE \d "c:\\wbs\\mywb.xlsx" \s "SELECT m.Device_Serial, m.Device_Cost FROM [mysheet$] m WHERE m.Case_Number = '{ MERGEFIELD Case_Number }'" }

Update all the fields and see if that is returning the records that it should for the current Case Number, and that the layout is good enough for your purposes.

If so, I would try using the Mailmerge preview facilities to move through a few records, execute the field codes manually as rquired by using F9. Then try merging a few records. If all is well, I would do a complete merge to a new document before proceeding to whatever merge you are actually trying to produce.

There are ways to apply some formatting to the DATABASE field, but they tend to break down if the table created by the field spills onto additional pages.

ismet kocaman say: IMHO, as an alternative, you can try the below code and then filter Text1 field for "Elapsed":

Sub CheckElapsed()

    Dim T As Task
    Dim DurationString As String

    For Each T In ActiveProject.Tasks
           DurationString = T.GetField(pjTaskDuration)
           If Mid(DurationString, InStr(1, DurationString, " ") + 1, 1) = "e" Then
              T.Text1 = "Elapsed"
           Else
              T.Text1 = "Not Elapsed"
           End If
    Next T

End Sub

PS.
- Just make sure "Add space before label" is checked in Tools | Options | Edit, before testing the macro.
- Test it on backup copies of mpp files.
- Modify "e" for your language if it is not English.
- The last parameter in InStr function is a space character enclosed in double quotes (i.e. Chr(32) ).
