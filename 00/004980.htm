Combining two text fields into one text field using Access 2010 - Microsoft Community

juliezumas say:

Combining two text fields into one text field using Access 2010

Hi,

I am trying to combine two text fields into one field in a query.  I have created a table where there are multiple "Comments" fields because multiple comments can be made on one record.  But when I run the query, I want to see one "Comments" column, not "Comment1", "Comment2", since the comments are all about the same thing.  Unless I could've set this up differently, but I wanted the comments to be associated with each particular record.  I tried Expression Builder where I created a new variable "Comments" but I used the "+" to combine the two fields.  What ends up happening is the text from both of the comment fields end up mushed together.  I would still like for the comments to be separate records and link to other identifying information (the person's name and title).  Is this possible?  And if so, how would I go about doing that?  Any help would be wonderful!

Julie

dhookom say: When concatenating strings, you should use & rather than + unless you have a unique reason to use +. I'm not sure why you don't use a single memo field for comments.

ken sheridan say: If by 'I would still like for the comments to be separate records' you mean they should each be in separate rows in a related table, then you are absolutely right to want to do this; that would be the correct design in a relational database if you don't use a memo field (now known as 'long text').  You would simply include a foreign key column in the Comments table which references the primary key of the table in which you currently (and incorrectly) have the multiple comment columns.

For data entry you would then use a subform in a form based on the current (now comment-free) table, basing the subform the new Comments table and linking the parent form and subform on the keys.  There is a subform wizard which will set this up for you, but it's a trivial task to do so yourself.

If you then want to concatenate the comments into a single string expression in a query, there are a number of concatenation functions available if you google around a bit.  There is one in Concat.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes a number of concatenation functions, including one for concatenating values from a related table, using the highly efficient GetString method of the ADO recordset object.  Most others you'll find do so by looping through a DAO recordset to build a string.

scottgem say:

Unless I could've set this up differently, but I wanted the comments to be associated with each particular record.  ... I would still like for the comments to be separate records and link to other identifying information (the person's name and title).  Is this possible?  And if so, how would I go about doing that?  Any help would be wonderful! 

Julie

Yes you should have set this up differently. Whenever you have numbered fields (like Comments1, Comments2, etc.) You have what is called a Repeating Group which is not good design. You should have a table like this:

tblComment

CommentID (PK Autonumber)

ParentID (FK)

CommentDate

Comment

The ParentID would be the Primary Key of the record where you currently have all these fields. That what will associate the comments with a particular record. 

If you want all the comments to be in one column of your query you can use one of the concatenation functions Ken mentions, to concatenate the comments from each record into one column.

dhookom say:

If you can't change your current table structure, you might consider using a normalizing union query with SQL like:

SELECT [PrimaryKeyFieldName], 1 as CommentNumber, Comments1 as Comments

The actual SQL will depend on the names of your table and fields as well as the number of comment fields.

Watermarks under inserted images - Microsoft Community

reeper5333 say:

Watermarks under inserted images

I imported a Print screen image into my document, but is stayed in front of the watermark blocking part of it out.  What do I need to do to the image to let the watermark show through it? 
jay freedman say:

If your inserted image has a single color (such as white) as a background, you can make that color transparent. Select the picture, go to the Picture Tools tab of the ribbon, click the Color button, and click Set Transparent Color at the bottom of the gallery. With the special cursor, click somewhere on the background of the picture.

If that won't work because there is no single color in the background of the picture, you might be able to use the Remove Background tool, but that's more work -- you may have to add pieces that the automatic tool missed, or remove pieces that it identified in error.

Some technical detail: Word places things in several "layers," with things in the closer layers obscuring ones in the farther layers. The watermark is always in the farthest layer. Next are one or more layers of pictures, text boxes, etc. that are "behind text," then the text layer, and then things that are "in front of text." Anything that's transparent, or shapes that are set to "no fill," will allow items in farther layers to show through.
