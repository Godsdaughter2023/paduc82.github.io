converting time to number - Microsoft Community

carlm42 say:

converting time to number

I have a question of how to get time to number.

I have in cell A1 total items = 127
In cell B1 is total time formated hh:mm 6:50 total time is 410 minutes (formatted as [m])
in cell c1 I need to have formula of total items/total time in minutes 127/410

correct answer is actually 0.309756

when I try to convert the time to decimal I get very different answers 

What is the best way to convert the total time of 6 hours and 50 mins to get the answer of 0.309756?

all the discussion and help that I can search have not found a method to convert the 6 hours and 50 mins to a usable decimal for the correct answer.
jeeped say:
There are 1440 minutes in a day. If you have 06:50 (e.g. TIME(6, 50, 0)) and want 410 then you would multiple by 1440.

=TIME(6, 50, 0)*1440     ◄ equals 410

If you have 410 and want to find the hours and minutes in a time format then,

=410/1440                     ◄ equals 06:50:00 or TIME(6, 50, 0)

For the "total items/total time in minutes" this becomes,

=A1/(B1*1440)                 ◄ 127 ÷ (TIME(6, 50, 0) × 1440) = 0.309756

Addendum: brackets around the (B1*1440) may be necessary in this case.

hal hostetler - mvp-outlook say:
I have a Cox account and it supports both IMAP (which supports multiple device syncing for email) and POP (which does not support syncing).  It sounds to me like you have all your devices except the Windows 7 computer either setup as IMAP or setup as POP with the "Leave a copy of mail on the server" option set.  The Win7 machine, I can just about guarantee, is setup as POP WITHOUT the "Leave a copy" option set, so it downloads whatever email is present and deletes it from the server.  The best solution is to set all devices to use IMAP so email and folder structure will be the same on all of them.  Alternatively, check the Win7 machine an make sure the "Leave a copy of mail on the server" option is enabled.

brian tillman [outlook mvp 2007-2019] say:
If you configre the account as POP, leaving messages on the server, then yes.  If you configure as IMAP, then deleting it on any machine using IMAP will delete it on all machines using IMAP for that mailbox.  It won't stay on the server, since IMAP is a two-way sync.

hal hostetler - mvp-outlook say:
If all clients are setup to use the IMAP service, deleting an email on one of them deletes it from the server which, in turn, is reflected to all devices - the email is deleted from all.  If all accounts are setup as POP, all must be set to leave a copy of email on the server and ONE of them (makes no difference which one) must be set to "Remove email from the serve after (xx) days" so you don't run out of room.  With IMAP, moving an email to a folder is also reflected on all clients, they will all have the same folder structure.  With POP, the email must be moved manually on each client to whatever destination folder you choose..

    echo s say:

    Yeah, the Help I turned up by searching on Mirror Text is wrong. If you'll tell me what keywords you were searching on, I'll report this to the Help folks for correction.

     

    That said, you still need a solution! What you have to do is apply a warp text effect to the text or wordart before it will actually mirror when you flip it.

    So, Insert WordArt or Insert Shape and choose a textbox. Type in the text. Then select the textbox and go to Drawing Tools Format tab | Text Effects | Transform. Choose one of the transforms there. Square is pretty close, but your text may seem "tall" (for lack of a better word). If that's the case, just resize the textbox so it's shorter, and the "text" will follow suit. Basically, applying a transform makes the text work more like an image, so when you resize the textbox, the text itself resizes, too -- which it doesn't do if it's still "regular" text, you know?

    Once it's the size you want it to be, go to the Drawing Tools Format tab and click Rotate | Flip Horizontal and then Flip Vertical.
