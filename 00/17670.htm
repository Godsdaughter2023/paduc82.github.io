joseph goss say:

Adding all mailboxes inot a security and compliance case.

I have need to put all mailboxes in my organization onto an eDiscovery hold.  I have created a case in eDiscovery in the Security & Compliance center but there is no way for me to add multiple emails into the case unless I search one by one.   Does anyone know if there is a Powershell command or another way to put all of my mail boxes on an eDiscovery hold?
barry xie msft say:

Hi Joseph,

 

Yes, there is a PowerShell command to place all mailboxes on hold. Here are the steps:

1. Connect to Exchange Online PowerShell with an administrator account.

2. Run this command to place all mailboxes on Litigation Hold:

Get-Mailbox -ResultSize Unlimited -Filter {RecipientTypeDetails -eq "UserMailbox"} | Set-Mailbox -LitigationHoldEnabled $true -LitigationHoldDuration 2555

 

Or run this command to create an In-Place Hold:

New-MailboxSearch "Hold-CaseId012" -SourceMailboxes "joe@contoso.com" -InPlaceHoldEnabled $true

Note: Change joe@contoso.com to your own email address.

 

Regards,

Barry

scottgem say:
Access is an event driven development platform. This means that things happen when an event is triggered. So, if you want to record the time something happens, you can use an event that will be triggered. For example; if you want to record when someone fills in the Notification sent value, then you can use the After Update event of the control bound to the field. Just have another control bound to the timestamp field and set its Visible property to No. Then in the After Update event use the Code Builder to enter one line of code:

Me.timestampcontrolname = Now()
