I think your solution above, which basically 'manually' split the database would have worked.

The "BAD filename or Number" error is a general MS Office error (error 52) that, in this case, refers to a bad reference to the location of the default database.

In this regard, I had noticed that with MS Word and Excel I was getting a bad file location to the default directory whenever I tried to SAVE a new document.
