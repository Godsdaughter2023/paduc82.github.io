HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer

    Enforce referential integrity - MS Access - Microsoft Community

    rob nash say:

    Enforce referential integrity - MS Access

    Hi

    Please can you help

    I am trying to 'enforce referential integrity' between two records, based upon a date. When I try to set the 'enforce referential integrity' ii fails as it states that effectively a child record can not be linked to a parent.

    I have run a query against the two records in question and it produces a match in all cases. Is there another way that I can find the offending child record(s) in question?

    Many thanks

    ken sheridan say: Firstly, the column in the referenced table must be either the primary key of the referenced table, or defined as a candidate key by its inclusion in a unique index (no duplicates).  If Access won't let you do this then you have duplicate values in the column in the referenced table.  If the column in question can legitimately allow duplicates, e.g. an OrderDate column in an Orders table, then you cannot relate this to another table in which a corresponding date column also allows duplicate values.

    To find illegitimate duplicate values in a column in the referenced table open the following query:

    SELECT *
    FROM YourReferencedTable AS T1
    WHERE EXISTS
        (SELECT *
         FROM YourReferencedTable AS T2
         WHERE T2.YourDateColumn = T1.YourDateColumn
         HAVING COUNT(*) > 1);

    To find rows in the referencing table without a match in the referenced table open the following query:

    SELECT *
    FROM YourReferencingTable AS T1
    WHERE NOT EXISTS
        (SELECT *
         FROM YourReferencedTable AS T2
         WHERE T2.YourDateColumn = T1.YourDateColumn);
