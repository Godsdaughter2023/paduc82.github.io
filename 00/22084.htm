hi christophe,

emails sent from office 365 use the transport neutral encapsulation format (tnef) by default. if the recipient message system cannot process the format, the recipient will receive a winmail.dat attachment.

since you have set the default format to html, i’d like to clarify that the format setting of a particular address can override the setting of the outlook client. besides, even if the setting of the particular address is changed, outlook may use the setting of autocomplete version if you select the address from autocomplete list.
 

in this situation, i suggest you disable tnef encoding for all messages sent to all domains or a specific one outside your organization by using windows powershell. for more detailed information, please refer to the following article: https://support.microsoft.com/en-us/kb/2487954
