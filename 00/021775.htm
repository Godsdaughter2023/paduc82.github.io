tony chen chn say:

Hi,

 

When we open an XML source, Excel checks if the file refers to an XML schema. If the XML file doesn't refer to schema, we will receive the following message: The specified XML source does not refer to a schema. Excel will create a schema based on the XML source data. 

And then Excel imports the data in the XML source. If the XML data contains duplicate fields inside an element, Excel spreads the data over multiple rows. Each row contains one unique set of data. This can cause a small set of data in XML to be displayed as a larger number of rows in Excel.

The cause seems to be Excel2010 doesn't recognize your XML schema.

rich michaels - mvp (greatcirclelearning.com) say:

The paths you are searching are not Mac Office 2011 paths. You must have loaded Mac Office 2016 or Office 365 at some point, because those are the paths for those version of Mac Office.

The Mac Office 2011 path to the ACL file is this:

/Users/<Home Folder>/Library/Application Support/Microsoft/Office/Preferences/Office 2011

You don't mention it, but hopefully the ACL file from the Office is also for a Mac. The Windows and Mac versions of the ACL file are not compatible.

Hi Miguel,

I recognize the situation, being Dutch. The store is comming. To us it is comming in about 6 months.

About a seperate LiveID: apperantly it is against the EULA. So from a legal point of view, it is certainly not reccomended.

Also from a technical perspective, it might hurt you a lot more than you realize. Even when you create, for example a French or Spanish, LiveID you still can only access Freeware. To get this to work you also need to have all your regional settings on your phone set to the region of your LiveID as well (except the language, but local search does not work as expected). In order to be able to buy apps you also need a european creditcard (for example Mastercard) but also have to have a Spanish or French billing adress (they never send any bills, so the reason for this requirement is completely lost on me...). I was lucky because all my ID's were already tied to the UK because I worked a lot in our UK offices when I created them.

Problem is that you can't migrate LiveID's back, and all your purchases are tied to your LiveID. So when things do go local in Portugal you might still run into trouble because you might not be able to access the local market for portugal, because such a LiveID is for the wrong market. You can't change a region of a LiveID, so you then end up with the choice: throw away a LiveID with all the apps bought, or have no access to portuges apps.

So trickery with LiveID is not a reccomended path both from a legal and technical perspective...
