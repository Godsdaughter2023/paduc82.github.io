Hi,

 

These certificate error messages usually occur when the computer's clock is set so that the date/time is later than the expiry date of the web server's SSL certificate. You may also get the error message when website is using a certificate that was issued to a different web address or if a company owns several websites and uses the same certificate for multiple websites.

 

If you receive certificate errors, it means the website you're visiting is having certificate problems and it doesn't indicate a problem with Internet Explorer.

 

You can follow some methods to try and resolve this issue:

Method 1:

Check if the date and time on the computer are set properly. If not, you can refer to the link to sync the date and time on the computer with Internet Time Server:

 

Follow the section “To set the clock”

http://windows.microsoft.com/en-in/windows-vista/set-the-clock

 

Method 2:

You can also follow a workaround to try and resolve the issue. Follow the steps and check if it helps:

a)      Click on the message “Continue to this website (not recommended)” on the browser.

b)      Now, a red Address Bar and a certificate warning appear. Click the Certificate Error button to open the information window.

c)       Click View Certificates, and then click Install Certificate.

d)      The warning message that appears, click Yes to install the certificate.

 

For reference, you can refer to the link:

“"There is a problem with this website's security certificate" when you try to visit a secured website in Internet Explorer”

Refer the section “Workaround”

http://support.microsoft.com/kb/931850

=((1+XIRR(B1:B13, A1:A13))^(1/12)-1) * 12
Nominal monthly compounded rate = 35.42%

The XIRR function compounds DAILY and returns an Effective rate of return.  The true IRR of any cash flow (evenly spaced periods (days, months, quarters, years) is the Nominal rate of return so you should convert the Effective rate of return to a Nominal rate before showing the IRR of the relevant cash flow stream.

I don't believe I would do this in the query but rather an expression in the report. However you might be able to use a DCount():

SELECT *, IIF(DCount("*", "table2","IsNull([DateComplete]) AND TicketNum = " & TicketNum ) >0, "Not Complete","Complete") As Complete
