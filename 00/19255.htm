greg maxey say:

In line with Doug's "easier" assessment and if you wish to using content controls:

Add a CC titled "Date Picked" and CCs titled Date+1M, Date+2M etc.:

Run the following in the ThisDocument module:

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
  Select Case ContentControl.Title
    Case "Date Picked"
      If IsDate(ContentControl.Range.Text) Then
        ActiveDocument.SelectContentControlsByTitle("Date+1M").Item(1).Range.Text = _
                       Format(DateAdd("M", 1, ContentControl.Range.Text), "MMMM dd, yyyy") 'Formatted date.
        ActiveDocument.SelectContentControlsByTitle("Date+2M").Item(1).Range.Text = _
                       DateAdd("M", 2, ContentControl.Range.Text) 'Default format.
      Else
        ActiveDocument.SelectContentControlsByTitle("Date+1M").Item(1).Range.Text = vbNullString
        ActiveDocument.SelectContentControlsByTitle("Date+2M").Item(1).Range.Text = vbNullString
      End If
  End Select
lbl_Exit:
  Exit Sub
End Sub

See: http://gregmaxey.mvps.org/word_tip_pages/installing_employing_macros.html for instructions to employ the VBA code provided above.

Meldung "keine Rückmeldung" - sowohl imBetriebssystem als - Microsoft Community

uweilsen say:

Meldung "keine Rückmeldung" - sowohl imBetriebssystem als auch in Office-Anwendungen

Die Meldung tritt meistens in Office-Anwendungen auf. Wie kann man das Problem beheben ?

andreas killer say:

Naja, wenn die Anwedung wirklich hängt, dann gar nichts.

Wenn die Anwendung einfach nur zu beschäftigt ist um Windows die Rückmeldung zu geben das sie noch arbeitet... nich hinkucken, Kaffee trinken gehen. Irgendwann verschwindent die "keine Rückmeldung" von alleine.

Man kann den Timer verlängern ab dem Windows die Meldung bringt:

HKEY_CURRENT_USER\Control Panel\Desktop\HungAppTimeout

Der Wert ist in Millisekunden und sollte auf 3000 bis 5000 stehen, also 3 bis 5 Sekunden.

Andreas.

uweilsen say:

Hallo Andreas, vielen Dank für Deine Antwort aber  wirklich hilfreich ist sie nicht.

Die Wartezeit ist aber zu lang, mach malmehr als 20 Stunden oder der Rechner stürzt total ab.

Wenn ich einen neuen PC mit Betriebssystem kaufen würde, ist diese Meldung wirklich verschwunden oder wie kann ich diese Meldung vermeiden ?

andreas killer say:

Die Wartezeit ist aber zu lang, mach malmehr als 20 Stunden oder der Rechner stürzt total ab.

Das liegt dann aber nicht an der Meldung, sondern an dem Programm.

Die Details um welches Programm es dann geht und was es gerade macht müsste man sich dann genauer anschauen.

Der eigentliche Sinn und Zweck dieser Meldung ist das man erkennen kann ob sich ein Task aufgehängt hat oder nicht...

Das funktioniert nach einer einfachen Idee, aber dazu muss ich ein bißchen ausholen.

Wenn man Eingaben in ein Programm macht, dann geschieht dieses (für das Programm selber gesehen) nicht über z.B. die Tastatur oder mit der Maus, sondern jedes Programm hat eine Schnittstelle mit der es Nachrichten vom Betriebssystem empfängt.

D.h. wenn Du eine Taste auf der Tastatur drückst, dann geht diese von der Hardware an einen Windowstreiber, der gibt sie an den Kernel weiter, dieser wertet das aus und ermittelt für welches Fenster sie bestimmt ist und schickt letztendlich eine Nachricht an das Fenster, sprich das Programm. Dort läuft sie zuerst in einen Puffer und das Programm kann sie dort abholen und dann damit machen was es will.

Zusätzlich zu den normalen Eingaben verschickt Windows an jedes Fenster eine Art Ping und das Fenster pingt zurück. Auch diese Nachrichten laufen in den gleichen Puffer.

Läuft dieser über, bzw. kommt das Ping nicht zurück, dann bringt Windows nach ablauft der HungAppTimeout Zeit die Meldung "keine Rückmeldung".

Das sagt aber eben nur aus das das Fenster / Programm im Moment nicht auf die Windowsnachrichten reagieren kann, weil es z.B. gerade eine sehr intensive Operation ausführt... das ist ganz normal und kommt selbst bei den besten Programmen vor.

Bei z.B. schlecht programmierten Excel-Makros ist das normal, sobald das Makro fertig ist verschwindet die Meldung von ganz alleine, da Excel dann wieder Zeit hat den Nachrichtenpuffer zu verarbeiten.

Verhindern kann man das nicht, es sei denn man wechselt das Betriebssystem... aber bei anderen sieht es auch nicht viel anders aus.

Andreas.

I want to create a document that is linked to this list, but that retains a standard format.  To explain, this list changes somewhat frequently, and I want the numbers of the questions to adjust automatically whenever a new question is added.  Currently, I have to go into the list's word document and manually adjust the numbers.  This is becoming quite tedious, as the list grows longer and longer.

At the same time, I want to general format of the list to remain the same.  That is, it is divided into two columns on the page, and spills onto 4 or 5 pages at its present state.

doug robbins - mvp office apps & services (word) say:

I had assumed that the wanted the questions in the left column and the answers in the right column so that you would have inserted the merge field that contained the question into the left hand cell and that which contained the answer into the right hand cell.  Not the same merge field into both columns.

 

To get the layout that you describe, via the Page Setup section of the Page Layout tab of the Ribbon, format your document so that it has two columns and then insert a one cell table into the left column and insert the merge field into that cell.
