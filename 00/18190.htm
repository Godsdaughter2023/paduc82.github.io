First, the cell value only appears to be 3548.22 (corrected) because of the way you formatted the cell; presumably Number with 2 decimal places.

If you increase the number of formatted decimal places to 13 decimal places, you will see 3528.2189980539600.

We see 3528.2189980539600 instead of 3528.2189980539647 because Excel formats only the first 15 significant decimal digits.

However, Excel actually requires 17 significant decimal digits in order to reproduce the exact internal binary representation, in accordance with IEEE 754, the binary floating-point industry standard.

Excel uses XML (compressed) to represent "xlsx" and "xlsm" files.  That is why the XML representation is infinitesimally more precise.

@saadn:

 

Updates are usually only released via FOTA OR via NSU, not both. Also, one system is not aware of the updates released via the other system. Just because FOTA is telling you that there are no updates does not necessarily mean that there are none using NSU. You should try and fix the issue preventing you from using NSU and then attempt an update that way.

 

@noshinhaque:

 

Regardless of the method you try and use to update your phone, you will not be able to do the update until '3' release it.

 

If you want to know when that will happen you have to contact '3' because they, and they alone, are responsible for the release of their branded firmware, not Nokia.
