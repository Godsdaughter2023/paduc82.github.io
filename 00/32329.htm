Please read these instructions carefully and completely before you begin installing Windows 7: There are important steps that you need to follow to upgrade from your PC from Windows XP to Windows 7 in order to preserve your files and settings.

Before you begin, you'll need to back up all your data to an external storage device, and you'll need all of the installation discs for programs you want to keep. Please make sure you have an external hard drive (see details below).

1. Run Windows Upgrade Advisor (windows.microsoft.com/upgradeadvisor) to see if there are any known issues that might affect the installation and whether you should install the 32 or 64 bit version of Windows 7.

2. Save your files and settings on an external hard drive using Windows Easy Transfer (windows.microsoft.com/windows-easy-transfer). Hi CSholmire,

Thank you for letting us know your concern.

I would suggest you the below methods. Try performing these and check for the issue after each method:

Method 1:

  1. Open a Command Prompt as the user that the drives are not being remembered. Do not use the "Run as Administrator". In this case we don't want to do that. At the Command Prompt type: net use <enter>
  2. Does it say something like this: C:\>net use. New connections will not be remembered.
  3. If it says they "will not be remembered", then you need to change that. Type: net use /p:yes

That will change the system setting for the user to make the drives remembered all the time and will be the default action.

If the above is not the case then map the drive through the Command Prompt and see if the mapping is remembered.

net use /p:yes

net use r: \\servername\sharename

You may change the drive letter R: above to any free letter you wish. However remember that if you use a letter that is too close to the beginning of the alphabet you may overlap with USB devices and then one or the other will be missing. You can replace the R: with * and it will start with the next free drive letter starting at the bottom of the alphabet. That should make the drive be remembered through log off and reboot.

Method 2:

The long-term fix is a registry edit that will make mapped drives visible in both standard and administrative sessions.

  1. Press Windows +R and type in : regedit
  2. In Regedit, navigate to HKLM//SOFTWARE//Microsoft//Windows //Current Version//Policies //System.
  3. Create a new REG_DWORD entry named "EnableLinkedConnections".
  4. Set its value to 1.
  5. Reboot after adding the registry entry and programs will be able to see mapped drives forever.

It might be prudent to wait until after you get your fresh install of Windows updated to Service Pack 1 and have all the critical MS updates installed before troubleshooting...  you could be trying to track down issues and problems that may have been resolved in some of those updates (i.e. wasting your time).

The error 1068 when starting a Service means that one of the dependency services is not configured or running properly so for Windows Audio that would be these things that need to be running:

All of those Services should already be set to start automatically and running all the time (if you want your sound to work) but you can check them if it you desire to be extra sure from the Services applet in Control Panel, Administrative Tools.

The error 1053 indicates a timeout waiting for the Service to start and the default timeout is 30000 milliseconds (30 seconds) so that should be plenty of time.  Some advice will say to edit the registry and increase the time to 60 seconds or even higher but that does not make much sense (and never fixes the real underlying problem) - 30 seconds should be plenty of time.

After a fresh Windows installation it is important to update the drivers for your motherboard with those offered by the motherboard manufacturer - starting with what is on their WWW site.  That would be things like chipset drivers, audio drivers, video drivers and network drivers since what comes on the Windows installation CD are going to be generic drivers and you need the drivers that are specific for your system.

Some systems come with a driver installation CD which might have the correct drivers (or at least better than what comes on the Windows CD) but those can also be out of date.  So the best thing to do is visit the motherboard manufacturers WWW site and see that they have and then even the manufacture of the hardware devices themselves (so visit the Realtek WWW site for audio drivers).

Generally it is not a good idea to ever accept any hardware device drivers offered by Windows update - Microsoft does not know what the best drivers are for your system so if you see them offered you should decline them (or uninstall them) and then hide those updates so you never see them again.  Always get hardware drivers from the device manufacturers WWW page - they know what is best for your system.

Since the MS Answers forums implementation does not prompt for any fundamental system information when a new question is asked we know almost nothing about your system.  This inherent design flaw makes troubleshooting and problem solving take longer than it should but that is the way they run things here.
