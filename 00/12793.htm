swischner say:
Oh my. This is not going to be easy (to explain at least), but there is a chance we can restore your OneNote data. A lot of text to follow, but this is necessary.
Let me start with some important general explanations:

1. OneNote “files” on OneDrive are not the same as Word or Excel documents. They aren’t single files which can be moved and copied around freely. OneNote is storing its notes in a special hidden format on OneDrive. Usually you see a “file” with the OneNote logo and the name of your notebook, but that one is only a placeholder, does not contain any notebook data and cannot be downloaded at all.

2. OneNote is NOT using the local OneDrive sync client for Windows but his own sync mechanism, which is working directly with the Online files. So it does not help to manually copy or move files, that seem to be notebook files or folders, to your local OneDrive directory or manually upload them to OneDrive using the browser. Both will result in some files on your OneDrive that look like OneNote notebooks but are of no use to the different OneNote clients. If you have done that, please move all of them (only the OneNote related files, not Word or Excel documents, they are fine) back to your (any) local temporary folder.

3. There are two completely different and separate OneNote applications for Windows: OneNote for Windows 10 is being installed with the operating system itself and not dependent on or related to MS Office in any way (no matter what the marketing department of Microsoft is saying about that, but that is another long story). You can easily identify that version: Its icon is labeled “OneDrive” and it has no “File” menu. You can not use that version to restore your notes as it’s not able to open any files, but only work with notebook content, that is stored on OneDrive in the proper format (which is not the case by now). The second version is called (and labeled) “OneNote 2016”. This is installed with Office 2016 or 365 (there is also a free version of this, but this won’t help us for the upcoming procedure for technical reasons. For one of the two solutions I am going to show you, you will absolutely NEED OneNote 2016 from MS Office).

Now, this seems to be the actual situation:

The OneNote notebooks that you had on OneDrive, are destroyed in their original form. You need another copy/instance of your note content, to build new notebooks on OneDrive in the correct format.
If all of your notes are still visible in the OneNote application (just not synchronizable, which is normal for this situation), your best option would be:
- Don’t close the notebooks under any circumstances and don’t reset or reinstall OneNote!

- From within the app create new notebooks with different names (we can rename them later) on OneDrive. One for each “old” notebook you want to recover.

- Now (still from inside the OneNote app!) copy (don’t move for security reasons!) all content from old notebooks to it’s corresponding new version section by section.
In OneNote 2016 this can be done by opening, expanding and pinning the notebook list to the left and dragging all sections, holding CTRL (for copy) from old to new notebook. Or use the context menu (right click) and the copy or move command. For this rescue method, you can also use the OneNote app from Windows 10, just that the dragging method won’t work here and you have to use the context menu.

- After you have done that (with one or all notebooks), check if the new versions are syncing now. You may also want to check on OneNote Online, if they are containing your data now.

- When you are sure they are ok, you may close the old versions on your OneNote and rename the new notebooks using the browser and OneDrive online at will.

The second rescue method would require, that your attempt to copy all content from OneDrive to your local folder, really included your OneNote data as well. As I explained before, that is not supposed to happen, because the notebook data is hidden. But there is a glitch in that “hiding”, that may lead to the actual notebook files getting downloaded. This is the case if your notebooks (the “placeholder” files) weren’t in the main directory on OneDrive, but in a subdirectory. In that case, the ZIP file that was created on the download, also contains the notebook files in the following format:
Notebooks are folders with the corresponding names.
Section groups (if exist) would be subfolders in the notebook folders
Sections are .one files in the folders (the page data is stored inside them)
Every notebook folder contains an open notebook.onetoc2 file

If you have this files and structure, you are lucky – at least if you have OneNote 2016 (from MS Office, see #3 in the beginning) as well:
If not already done, unpack the folder structures as explained above, inside a temp directory on your PC.
Using the explorer, double-click on a .onetoc2 file (or on any .one file, if the .onetoc2 is in the same directory). This should open the notebook in OneNote 2016 as a local notebook (only the Office version of OneNote 2016 can do this!).
In OneNote 2016 click on File – Share, select your OneDrive account from the left list and click on “Move notebook”. This will transfer the notebook to OneDrive and create the correct format there.
Repeat this for any notebook. Now you should be good.

This second method is a little faster and more convenient than the first one, but it needs you (I know I am repeating myself here) to use OneNote 2016 from MS Office and a complete version of your notebook files in your downloaded ZIP.
I hope I could help save your notebooks with this necessarily lengthy explanation.

suzanne s. barnhill say:

The AZWizardModul is part of Avery's template software add-in. Since Word natively provides built-in "label" definitions for most Avery products, it is quite unnecessary, and the version you have likely doesn't work with Word 2010, so you'd need to get an updated version in any case.

On the File tab, select Options, then Add-ins. In the Add-ins window, look for the AZWizard add-in. When you select it, you can see the full path in the panel below. Navigate to this location and remove it.
