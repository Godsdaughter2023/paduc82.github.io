nopext say:

How do I set a default font for Charts?

[Splitting this question from this PowerPoint question because this new question is about Excel chart defaults]

bob jones aka: cybertaz say:
The default Colors & Fonts for Charts are based on Themes. Each Theme comprises 2 fonts in its Font Scheme -- a Theme Headings font & a Theme Body font. In some Font Schemes the same font is used for both, in some the Heading & Body Font are different.

By default, new PPT presentations use the Office Theme which calls for Calibri to be used as both. However, any Theme can be set as the default for new presentations by selecting the preferred Theme & using the Set Current Theme as Default option from the Save Theme dropdown in the Themes group on the Themes tab of the ribbon. Charts use the Theme Body font, which in the Office Theme is Calibri.

Regardless of which Theme is used, one of the other Font Schemes can be selected from the Fonts dropdown in that same area of the Ribbon. As in other elements of the presentation, the font used in Charts will be changed automatically to whatever the Theme Body font of the selected Font Scheme happens to be. E.g., selecting the Apothecary Theme or Font Scheme would cause the font used in Charts to change from Calibri to Century Gothic, selecting the Newsprint Theme or Font Scheme would change the Chart font to Times New Roman.

The one thing that will interfere is formatting a Chart element -- such as the Title -- using a font that deviates from what the Theme/Font Scheme calls for. Doing so severs the relationship for that element of that Chart, so any further change of Theme or Font Scheme will apply to the other elements but not those that have been directly formatted.

andreas killer say:

Paste the code below into the code module of the sheet.

Andreas.

Option Explicit

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim NewSheetName As String
  Dim Sh As Object
  'Only if one cell is changed
  If Target.Count > 1 Then Exit Sub
  'Only in column A
  If Target.Column > 1 Then Exit Sub
  'Get a valid sheet name from the input
  NewSheetName = ValidSheetName(Target.Value)
  'Already there?
  If SheetExists(NewSheetName) Then Exit Sub
  'Check all sheets
  For Each Sh In Sheets
    'Skip out sheet
    If Sh.Name <> Me.Name Then
      'Done if the right place is found
      If Sh.Name > NewSheetName Then Exit For
    End If
  Next
  'Found?
  If Sh Is Nothing Then
    'No, add to the end
    Set Sh = Sheets.Add(After:=Sheets(Sheets.Count))
  Else
    'Insert anywhere in the middle
    Set Sh = Sheets.Add(Before:=Sh)
  End If
  'Rename it
  Sh.Name = NewSheetName
  'Copy the formats
  Me.UsedRange.Copy
  Sh.Cells(1, 1).PasteSpecial xlPasteFormats
  Application.CutCopyMode = False
End Sub

Function SheetExists(ByVal SheetNameOrIndex As Variant, _
    Optional ByVal Wb As Workbook = Nothing) As Boolean
  'True if sheet SheetNameOrIndex exists
  On Error Resume Next
  If Wb Is Nothing Then Set Wb = ActiveWorkbook
  SheetExists = Not Wb.Sheets(SheetNameOrIndex) Is Nothing
End Function

Function ValidSheetName(ByVal SheetName As String) As String
  'Removes invalid chars from Sheetname
  Const InvalidChars = ":\/?*[]"
  Dim i As Integer
  For i = 1 To Len(InvalidChars)
    SheetName = Replace(SheetName, Mid(InvalidChars, i, 1), "")
  Next
  ValidSheetName = Mid(SheetName, 1, 31)
End Function

larry yu msft say:

Hi Doug,

 

It’s not feasible to have one domain federated with two or more Office 365 tenants because one custom domain can only be added to one Office 365 tenant at the same time. We can have multiple domains under one tenant, but this cannot resolve the problem. So, it’s not feasible to have two Office 365 organization using one login (domain).

 

To allow external users join Microsoft Teams is only a potential solution for the problem because external access isn’t available in it. The related team is working on several ways to expand the value of Microsoft Teams to more diverse types of organizations and teams, including the ability for guests outside the team or organization to participate.

graham mayor say:

The following should work for you.

Sub Send_As_Mail_Attachment()
'Needs a reference to the Outlook object library
' send the document as an attachment _
in an Outlook Email message
Dim bStarted As Boolean
Dim oOutlookApp As Outlook.Application
Dim oItem As Outlook.MailItem
On Error Resume Next
'Save the document
ActiveDocument.Save
'Print the document
ActiveDocument.PrintOut
'Get Outlook if it's running
Set oOutlookApp = GetObject(, "Outlook.Application")
'Outlook wasn't running, start it from code
If Err <> 0 Then
   Set oOutlookApp = CreateObject("Outlook.Application")
   bStarted = True
End If
'Create a new mailitem
Set oItem = oOutlookApp.CreateItem(olMailItem)
    With oItem
        .to = "*** Email address is removed for privacy ***"
    'Set the recipient for a copy
        '.BCC = "*** Email address is removed for privacy ***"
        .Subject = "This is the subject"
        .Body = "this is the message body"
    'Add the document as an attachment, you can use the _
    .displayname property
    'to set the description that's used in the message
        .Attachments.Add Source:=ActiveDocument.FullName, _
        Type:=olByValue, DisplayName:="Document as attachment"
        .Display
        '.Send
        'Use .Send instead of '.Display to send to the Outlook outbox (or to send
        'directly if you have Outlook so configured ... but not until you are happy
        'with the results
     End With
'Clean up
If bStarted = True Then
    oOutlookApp.Quit
End If
Set oItem = Nothing
Set oOutlookApp = Nothing
End Sub

a. user say:

Windows 10 for the x86 PC will be just as vulnerable to viruses and other forms of malware as desktop Windows has always been, and Windows Defender/Security Essentials are the same thing, and are widely considered only "basic" protection that works well for those who are capable of practicing safe computing habits - basically people who are not likely to be exposed to malware in the first place.

The Metro store applications run in a different space, but overall the vulnerabilities in Windows are still there, whether it be via Java, Flash, Silverlight, Internet Explorer, whatever you use that could be vulnerable to exploit.

For others, who share computers, or who tend to go to questionable websites, or download files without really knowing the source, respond to fake popups saying to update something, etc., Windows Defender is not enough for proactive protection, so I would recommend third party software in this scenario.

Additional safeguards could include using Firefox with the NoScript extension, Malwarebytes Anti-Exploit, etc.

If you search for the word "antivirus" in this forum, there are numerous threads with people mentioning what works or doesn't work in the latest build of Windows 10.  The one I use is 360 Total Security 6.8.0.6012(For Windows 10 only) 

Be aware that this particular software package comes with default settings for low powered computers with minimal RAM, like netbooks, and if you want the full protection you need to manually change the protection mode from "Balanced" to "Security", and make sure that the Avira and BitDefender engines are downloaded and active.
