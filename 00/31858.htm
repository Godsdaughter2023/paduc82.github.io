WBAdmin and WHS backup fails after Windows 8 to 8.1 upgrade due to system recovery partition free space.

I experienced issues with VSS after upgrading from Windows 8 Pro MCE to Windows 8.1 Pro MCE through the Store on 10/17 on both a laptop and a desktop.  The laptop would not properly backup using wbadmin or WHS 2011 anymore due to an insufficient space problem.  Using wbadmin (wbadmin start backup -backuptarget:x: -allCritical -vssFull -quiet), it provided a useful description of the problem: the minimum free space is 50MB for a <500MB volume, i.e. the System Recovery Partition (350MB).  On the laptop, there was 27MB free and on the desktop, there was 48MB free.  I have no idea why there is a difference between the two systems which had the same version of windows. I was able to further isolate the problem by testing a backup of the C drive without the system recovery partition by omitting the -allCritical option (wbadmin start backup -backuptarget:x: -include:c: -vssFull -quiet).  I have also posted this message in a related Windows 7 thread in case somebody finds a similar behavior in Windows 7.


The spoiler is that I copied the WinRE.wim from the desktop to the laptop system recovery partition (\recovery\winre\winre.wim) since it was about 20MB smaller than the

laptop's copy.  This left 48MB free and provided enough space to perform the backup.  I also performed other steps before this as noted below, so they may have helped, but

I'm not sure that they did much at all when compared to copying the WinRE.wim file.
