timliu_msft1261 say:

Hi Techadmin2,
 
I'd like to answer your questions as below:
1.They are not two types of external users. We can say these are two sections for external user management. The #EXT# user is new Guest Platform feature from 2015. After external user accept the invitation, they will appear in the Active Users section with the format account#EXT#@yourdomian.com. Before, external user didn't display in this section.
 
Meanwhile, the External Users section displays the date when external users accept the invitation and which accounts are used to log on. We can say it’s another section for manage external users (give you other information which isn’t displayed in Active Users section). The external accounts in these two sections are the same accounts.
 
For some reasons, the users who display in External Users section will not display in the Active Users section. For example, the external account is a deleted work/school account or the external user hasn't been added after the new Guest Platform rolled out for some unexpected reasons. For these users, you can share items to them again if they cannot access them. After they accept the invitation, there should be #EXT# accounts for them in Active Users section.
 
2.There is no global way to see a list of all the sites to which an external user has access. There is also no global way to see a list of all documents that have been shared externally. As a work around, we can use the Compliance Search feature in the Compliance Center, by using the ViewableByExternalUsers property in a search query. For more information, see Searching for site content shared with external users. (Sign into Office portal>Admin tile>ADMIN>Compliance>Search>+>create a search with the query in the article, e.g. ViewableByExternalUsers:true AND ContentType:Document NOT FileExtension:aspx. This query can be edited as you want.)
 
Moreover, we understand it’s better if admins can check the access of a specific external user. It is suggested you submit the requirement via Uservoice. Your good ideas will help us improve our products.
 
Thanks for your understanding.
 
Tim

Word 2008 and British spelling - Microsoft Community

johnstephenhewitt say:

Word 2008 and British spelling

Why does spell check on Word 2008 not accept British spellings such as 'colour' even when the UK English custom dictionary has been selected? Is there anything I can do to avoid having my correct spelling automatically and annoyingly questioned?
bob jones aka: cybertaz say:
Settings in Word do not determine what vocabulary is used to check the spelling of text -- It's the language formatting of the text that tells Word what dictionary to use.

Select the text, go to Tools> Language, choose the correct language & make sure the Do Not Check...check box is cleared.

Other points involved are:

  • The Default Language you choose pertains only to documents you create from then on. It does nothing to change the language formatting in existing documents.
  • Pasting content from other sources often carries language format with it. If so, not only will that text not be checked as you expect, but anything you type from that point on may be formatted as the same language.
  • Custom Dictionaries are best set to (none) as their language in order for them to function universally. Also, they only augment the built-in dictionaries based on words added by the user.

For more detail & a better understanding visit this web page:

Spelling, Language, Dictionary
