macropod say:

Try enclosing both sides of the SKIPIF test, i.e.:

{SKIPIF "{MERGEFIELD Class}" = "9B"}

or:

{SKIPIF "«Class»" = "9B"}

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. Likewise, the chevrons (i.e. '« »') are part of the actual mergefields - which you can insert from the 'Insert Merge Field' dropdown (i.e. you can't type or copy & paste them from this message, either). The spaces represented in the field constructions are all required.

An alternative approach is to use the mailmerge filters to exclude records from the Class field  'equal to' 9B. See under mailings|Select Recipients>Filter.

Is there a way that I can click/drag "copy format" (hope that's the correct description) in my calendar worksheet so that it'll follow the columned source for my data?

Appreciate any help you all can give.

jeeped say:
You can copy the data from the source and Paste Special, Transpose into the destination. This will convert your rows of information into columns.

If you are seeking a 'live' update, put this into your second worksheet's (Sheet2?) F4 cell,

=OFFSET(Sheet1!$A$2,COLUMN(A:A)-1,0)

You can then fill right until you reach the end of your one year period. If the data in Sheet1!A2:A366 changes, the data in Sheet2!F4:NF4 will update.

Note that OFFSET() is considered a volatile function and will recalculate on any calculation cycle, not just the ones that involve cells referenced by it.

OFFSET function Returns a reference offset from a given reference
COLUMN function Returns the column number of a reference
