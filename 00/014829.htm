jay freedman say:
In Word 2013, click File, click Options, and click the Save tab on the left side of the Options dialog. In the "Default personal templates location" box, type the path to the folder that contains your custom templates. Click OK. The File > New page will now show the Featured and Personal links.

There are two other situations like this, involving the Workgroup Templates folder (which is intended for templates shared with other users on a network server, but can also be used on an individual computer as just another template folder). You set this by going to the Advanced page of the Options dialog, clicking the File Locations button near the bottom, and entering a value for the Workgroup Templates item in the list.
  • If a Workgroup Templates location is defined but a Personal Templates location is not defined, the File > New page shows Featured and Shared links.
  • If both a Personal Templates location and a Workgroup Templates location are defined, the File > New page shows Featured and Custom links. The Custom link goes to an intermediate page that displays the two folders, and you click one to get to the templates it contains.

jay freedman say:
Yes, you can make Personal the default by adding an entry in the registry.

First, you must have the correct path in the Default personal templates location" box of the Options > Save dialog.

If you don't want to edit the registry directly, you can download this file, save it to disk, and double-click it. Answer the prompts and then restart Word.

To do this manually, in the regedit.exe program, locate the same key that I mentioned in my second reply to this thread:

HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Word\Options

Right-click the key and choose New > DWORD (32-bit) Value. Enter the name

OfficeStartDefaultTab

and press Enter. When that name appears in the list on the right side of the window, double-click it. In the box that opens, type the number 1 and press Enter.

Personally, I prefer to turn off the option (in File > Options > General) for "Show the Start screen when this application starts" so I never see that screen at all. jay freedman say:

The underlying problem here is that you aren't declaring variables, so VBA is unable to verify their data types. If you include the line Option Explicit at the top of each module, VBA will catch this kind of error and force you to declare your variables. (To make this automatic, click Tools > Options in the macro editor's menu bar and check the box for "Require variable declaration".)

The trap you fell into is that the .Font.Color property has a data type of Long (a signed integer), but OldColor and NewColor have a data type of Variant/String because of the quote marks you put around the RGB function calls. That means that a paragraph's .Font.Color will never equal OldColor, and the body of the If...Then will never execute.

The quick fix is just to delete the quotes around the two RGB calls (leave them around the font name).

The proper fix is to declare the variables:

Dim OldFont As String

Dim NewFont As String

Dim OldColor As Long

Dim NewColor As Long

Dim ParagCount As Long

Dim J As Long

and any other variables you use throughout the macro. Then, if you tried to assign a quoted string to a variable whose data type is Long, VBA would give you a compile-time error message.

The code is inefficient in other ways:

  • If OldFont and NewFont will always have the same value, there is no point in including NewFont or setting the .Font.Name to it -- the paragraph's font will just remain what it was.
  • If the macro will be applied to very long documents, it will slow down noticeably as it gets toward the end of the document (because every reference to .Paragraphs(J) has to count up from 1 to the current value of J). Instead, use a For Each structure. To do that, you'll need to declare a variable 'Dim Parag As Paragraph' instead of ParagCount, and you won't need the variable J at all. The code can be this:
        

Option Explicit

Sub x()
    Dim OldFont As String
    Dim OldColor As Long
    Dim NewColor As Long
    Dim Parag As Paragraph

    OldFont = "Calibri"
    OldColor = RGB(15, 36, 62)
    NewColor = RGB(0, 150, 0)

    For Each Parag In ActiveDocument.Paragraphs
        If (Parag.Range.Font.Name = OldFont) And _
              (Parag.Range.Font.Color = OldColor) Then
            Parag.Range.Font.Color = NewColor
        End If
    Next Parag
End Sub

One more suggestion: If you use styles in the document to apply the font and color, you don't need a macro at all -- you can just modify the style, and every paragraph that has that style will immediately change to the new color.

john w. vinson say:

you're looking in the wrong place. SetWarnings is a feature of Macros (which can also be used in VBA). It's not part of Query design.

To run an update query without warnings you can use the RunSql action from a Macro, just include a line in the macro to SetWarnings False prior to running the query.

Better, however, is to execute the query from VBA code using the Execute method; this not only suppresses warning messages, it lets you trap errors as well:

Dim db As DAO.Databse

Dim qd As DAO.Querydef

On Error GoTo Proc_Error

Set qd = db.Querydefs("NameOfMyQuery")

qd.Execute dbFailOnError

Proc_Exit:

   Exit Sub

Proc_Error:

   MsgBox "Error " & Err.Number & " executing NameOfMyQuery:" & vbCrLf & Err.Description

  Resume Proc_Exit

End Sub
