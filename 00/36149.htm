andrea g. b. say:

Hey I had the same exact problem and this is how I fixed it, even if it doesn't work everytime! (but it is repeatable with an high rate of success)
My keyboard is a CASIO CTK - 2400 and I run win10 on a Laptop/Tablet configuration.
This is how I managed to make it work and what I suggest you try:
-Connect the keyboard to a USB 3.0 port and turn it on (this is how I did it so I suggest you try this before trying it another way)

-Go to Settings>Devices>Connected devices and click on Device manager

-Check for the connection (my PC did recognize the keyboard as an unknown USB device)

-Press right click on Sound, video and game controllers and select "Scan for hardware changes"

-When I did it the PC finally recognized the keyboard showing a CASIO driver

after that the keyboard wouldn't still send MIDI signals but I knew it was recognized so I insisted to find a way with the DAW Fruity Loops and this is how I make it work every time!

-turn off the keyboard

-open the DAW and turn on the keyboard

-refresh the MIDI device list

-when it's connected to the USB 3.0 port it works only briefly (couple seconds)

-when it's connected to the USB 2.0 port it works perfectly (just a really tiny lag but it may be because of the keyboard anyway nothing too bad)

-sometimes it stops working even when plugged in the USB 2.0 port my suggestion is to reset the DAW and then the PC and see if it fixes it

This is not ideal for live perfomances... but good enough for studio sessions.
I hope this was of help, God bless!
