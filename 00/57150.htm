sainath irp_mj_create say:

Windows 8.1 Group Policy Objects Overview

Summary
Group Policy is still the complex Management tool to play with Windows 8.1 Operating System. The below article explains about the Group Policy, Processing of Group Policies and different settings which got introduced in Windows 8.1
Details

Group Policy Overview:

One of the key Management tool to manage and maintain Windows Desktop Operating Systems is Group Policy. Administrators can use in-built policies or create custom Group Policy Preferences to manage Windows 8.1 Enterprise Desktop Operating Systems. Group Policies can be configured to manage User or Computer settings. Group Policies can be configured local to an Operating System or from a central management location using Active Directory Domain Controller. Local management of group policies are useful for the Windows 8.1 Enterprise clients configured under Kiosk mode or the Client Operating Systems configured under Workgroup mode which are placed under Isolated environment. Domain managed Client Operating Systems get an advantage of being centrally managed by Domain Controller and doesnât require manual intervention on the Client Operating Systems.

 

Local Group Policy can be managed by Group Policy editor (Gpedit.msc) which can be launched by an Administrator account to configure User / Computer settings. Domain managed client Operating Systems can install Remote Server Administration Tool [RSAT] tool on Windows 8.1 Enterprise / Professional Operating Systems to launch Group Policy Management Console and configure the Group Policies for a specific OU / Child OUâs respectively. Windows 8 can take advantage of PowerShell to configuring and linking Group Policies for Domain Joined Computers.

 

 

 

 

 

Group Policy Processing in Windows 8.1:

It is always a Group Policy Architect Design decision, whether to configure Synchronous or Asynchronous processing of Group Policies. Windows supports both Synchronous and Asynchronous processing of Group Policies, with Asynchronous, Windows 8.1 will finish the boot process and provides users with logon screen, while Group Policies gets loaded eventually. With Synchronous processing , User is not presented with Logon screen until the processing of Group Policy, the issue is time consuming over slow networks. Windows 8.1 has new ability to process the Group Policies with Synchronization set, when on slow networks, Windows 8.1 will stop processing the Group Policies set to Synchronize and process them when Windows 8.1 switched to faster networks.

 

 

 

 

 

Group Policy Settings In Windows 8.1 Enterprise:

There were several Group Policy changes which got introduced in Windows Vista and Windows 7 which provided Administrators to configure complex settings and helped Architects to provide complex solutions to customers which were not possible during Windows XP age. Some of the changes and Group policy settings includes

          a)      ADMX Templates which used XML standard for the settings rather conventional ADM templates.

          b)      APPLocker Policies which gave Administrators to configure Application related

                   settings with advance capabilities. APPLocker was a replacement to Software Restrictions Policy.

          c)      Advance Audit Policy Configuration which included more than 50 Group Policy Audit Configuration settings.

          d)      Group Policy Preferences which extended the ability to configure range of settings which were not

                   possible with the default Group Policies.

andre da costa say:
Windows 10 Digital License
When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate. You will prompted to enter a product key a couple times through the installation, click I don't have a key and Do this later.

There is no need to know or get a new key, but if you have issues with Digital Licensing or the Activation Troubleshooter, you can utilize an existing Windows 7 or Windows 8/8.1 product key you already own to reactivate if necessary. or use the reset function in Windows 10.

Learn more:
http://answers.microsoft.com/en-us/insider/wiki...
