hansv mvp say:

The third argument of DLookup is basically the WHERE part of the SQL of a query, without the word WHERE itself.

It is a text string, so the fixed text should be enclosed in quotes: "...".

Variable parts should be outside the quotes, concatenated with the fixed parts using &.

Number and yes/no values can be concatenated without needing extra characters:

..., "Amount=" & curMyAmount)

Date values should be enclosed in # characters, and they should be formatted as mm/dd/yyyy or as yyyy-mm-dd, not as dd/mm/yyyy:

..., "OrderDate=#" & dtmMyDate & "#")

String values should be enclosed in quotes. Since we're already using double quotes for the entire argument, we can use single quotes (aka apostrophes) for this purpose:

..., "LastName='" & strMyName & "'")

See http://www.eileenslounge.com/viewtopic.php?f=29&t=2672&p=21115#p21115 for more info on the latter, with some alternatives.

Now to your question: you can combine conditions with AND:

    txtPdADM = DLookup("PdEnrolADM", "tblLunchData", "CurtMonth = '" & _

        CurrentMonth & "' AND CurtYear = '" & CurrentYear & "'")

Hi dbcity,

The reason for using field switch to specify the date format is to ensure the output from your mergefield matches the format of the output from the date field. suzanne s. barnhill say:

It sounds like what you're saying is not so much that the style is not being applied as that you're not seeing the results (Spacing Before/After) that you expect when you apply it. This could be caused by "HTML paragraph auto spacing," an option that causes Word to honor only the larger of two spacing settings between paragraphs. For example, if you have a paragraph with 6 points Spacing After followed by one with 18 points Spacing Before, you expect to get 24 points; what you in fact get is 18 points.

Unfortunately, this "option" was made permanent in Word 2013. You can still disable it if you revert the document to Word 2010 format (which is actually pretty simple), but that may not be the best way forward for you.

As for submitting a sample, it doesn't have to be the actual text you're working with; you could make a copy of the text and substitute other content (nonsense words, lorem ipsum text, etc.).

suzanne s. barnhill say:

Ah, this is a different issue. Spacing Before is suppressed at the top of a page (and Spacing After at the bottom). This is true in all Word versions. What has changed in Word 2013 and above is that it is suppressed even after a manual page break or column break, and there is no longer a Compatibility Option to change that (Spacing Before is retained after a section break).

There are several possible workarounds:

  1. You can revert the document to Word 2010 format (if you're interested, I can tell you how).
  2. You can insert a Next Page section break (I don't recommend this unless a section break is needed for any other reason; every section break adds complexity to the document, especially in dealing with headers and footers).
  3. You can add another paragraph before the heading. Although I don't recommend this approach, if you choose it, you could format it as "Page break before," with 1-point font size and line spacing, or you could change the Spacing Before of the heading to compensate for the space of this extra paragraph.
  4. You can put the heading in a single-cell borderless table; in a table cell, the Spacing Before will be retained. The downside to this is that headings in tables are not displayed in the Navigation pane.
  5. You can put the heading in a text box or frame. The text box, however, has the same drawback as the table cell: even if it has In Line with Text wrapping, it won't show up in the Navigation pane; moreover, it will not show up under Headings in the Cross-reference dialog. In a frame, however, even if the wrapping on the frame is set to Around, the Navigation pane and Cross-reference dialog can "see" it.

If you don't use the Navigation pane or cross-references, then the drawbacks of tables and text boxes are not an issue; the headings will still all appear in a TOC. But if you are interested in using a frame instead, find instructions for using frames in the "Insert and format a frame" section of http://wordfaqs.ssbarnhill.com/MarginalText.htm#TrueMarginalText. As noted in the article, the advantage of a frame is that it can be included in the definition of the style.

hi ron,

by saying “a sharepoint sites inbox”, do you mean the site mailbox in office 365?

if yes, here are the steps to sync the site mailbox to the outlook client:

1. make sure the user has been added to either the site owners group or the site members group as an individual user, rather than as part of a security group.

2. sign in to outlook web app (https://outlook.office365.com/owa) with the user account. navigate to settings (the gear icon on the upper right corner of the page)—options—other. click go to the earlier version and select site mailbox.

3. double-click the site mailbox, click sync status and start sync.

4. wait for some time, you will be able to check emails in the site mailbox.
