rob koch say:

The reason almost none of these fake popup messages are detected by security software is that they are advertising driven, displayed via script contained within one of the ads displayed by whatever ad supported pages (in your case MSN News) you happen to be viewing at the time.

All of the major news websites and other popular ad supported websites have had varying numbers of such popups delivered through one or more of the ad networks which sell advertising slots on their platforms.  There are literally hundreds of these displaying thousands of ads, so finding and killing off the offenders is a needle-in-the-haystack game of whack-a-mole.

The most popular method of stopping these is thus by using ad blockers, though the proper use of privacy controls like turning Off the Windows 8/10 Privacy setting "Let apps use my advertising ID...", enabling the browser setting "Send Do Not Track requests to sites you visit...", as well as disabling 3rd-Party (e.g. Tracking) Cookies can also be effective.

The reason for this is that these advertisements including the popups are being delivered to your PC based on tracking information the websites provide to the advertising networks which have identified your browsing habits as someone who may be more likely to respond to the popup by doing whatever action it requests.  In your case, installing a [likely malware] fake update.

In case it's not clear, the website based, advertising network supported model is collapsing, while the in app display model which is typically better controlled has been growing in parallel.  Though a few apps may still allow display items that can be abused to inject scripts, these will likely eventually disappear as this avenue for malicious popup advertising becomes better understood.

titaniumtroop say:

The problem re-appeared after a restart, and this time the right-click trick didn't work. I contacted support in the manner suggested by Mr. Xu. The recommendations were as follows:

(1) Reset Preferences

Download and run the "OutlookResetPreferences" app located at https://blogs.technet.microsoft.com/office_for_mac_support_team_blog/2017/08/10/mac-outlook-support-utilities/.

(2) Delete Registration Info

Please Quit the app, then copy paste this command "~/Library/Group Containers/" in your Spotlight search. And then move this file "~/Library/Group Containers/UBF8T346G9.Office/MicrosoftRegistrationDB.reg " to the desktop, then launch Outlook application and try again.

1. Quit Outlook for Mac.
2. From the menu bar click on Go > Go to Folder and paste the below.
~/Library/Containers/com.microsoft.Outlook/Data/Library/Preferences
3. Move the com.microsoft.Outlook.plist file to the desktop.
4. Re-open Outlook, check the behavior. 
Let us know how it goes.

(4) Create New Profile

Let's try creating a new profile and add the email account, it will take some time to re-download emails. To create a new profile please follow the following steps: 

1. Quit Outlook 
2. Go to Applications folder in Finder 
3. Right-click on Microsoft Outlook.app and click on "Show Package Contents" 
4. Navigate to Contents --> Shared Support --> Outlook Profile Manager 
5. Double click and launch Outlook Profile Manager 
6. It will open profile manager and show all the profiles you have (usually you will have only one profile called "Main Profile", but there could be more). It doesn't matter how many profiles you have. 
7. We will now create a new profile 
8. To do so click on the "+" button on bottom left 
9. A new profile would be created which you can rename 
10. Click on the new profile and click on the gear icon at bottom and select "Set as Default" 
11. Now the new profile you created would become the default profile 
12. Go ahead and launch Outlook. It will launch with the new profile. 
13. When you launch with the new profile Outlook would be blank with no data and no account added. Your old profile is still available, so don't be alarmed about it. 
14. Now you need to add all your accounts again 
15. Navigate to Outlook menu bar> and click on Tools> Hold the option key on the keyboard and click on accounts 
16. Click on "+" and choose your account type from the dropdown list.
17. Enter your email address, password and check 'Configure automatically'.
Fill in the necessary information in the boxes and Then click on 'Add Account'.

This will sync everything again from the server. Give it some time to sync everything 
You may also refer to the link: https://support.office.com/en-us/article/Manage-profiles-in-Outlook-2016-for-Mac-fed2a955-74df-4a24-bef6-78a426958c4c
Note that your old profile is not lost, so in case you want to switch back then quit Outlook, open Profile manager again and make that profile default and Outlook with launch with your original profile after that.

Conclusion

Option #4 worked. I found this technique last week before making the original post, albeit without Steps 1–3. It worked for a bit, then the folder list disappeared from the Sidebar again. I will post a follow-up if the bug reappears.

Note that setting up a new profile may cause you to lose other settings such as signatures. Based upon my efforts last week, it appears to be a non-trivial task to get a signature transferred between profiles. The solution offered by the tech was to find a sent message and copy-and-paste the signature. I use signatures for common responses to several types of questions (e.g., instructions, required documentation, etc.), but all these are filed away. Again, this is a non-trivial solution.
