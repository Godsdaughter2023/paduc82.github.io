jay freedman say:
Although this project is fairly simple in concept, there are many possible choices for ways to do it -- each with advantages and disadvantages -- so it takes a number of decisions to make it work. I'll try to guide you to the setup that I think is the most robust (resistant to errors and easy to update) rather than the simplest.

One thing is clear: no matter how you try to do it, the project is going to require macro programming to select the correct data for each item once the user selects an office. Word doesn't offer any code-free way to handle it.

The next decision is whether to use a "protected form" with a dropdown field in the body of the template, a dropdown list box content control, or a "userform" (a custom dialog box) containing a list box. Any one of these would work, but they differ in when they execute the necessary macro code. I prefer the userform, because you can launch it automatically from the AutoNew macro when the template is used to create a new document, and the rest of the code is part of the userform.

Another decision is how to store the data so the macro can retrieve it when it's needed. You could write the data directly in the macro code, but that means you'd have to edit the macro if an office's information changes or your company opens a new office. A better choice is to store it in a table in a separate Word document, in an Excel worksheet, or even in a plain-text file so that it's easy to edit; the drawback is that the data document's location must be known (and the macro has to handle gracefully the possibility that the file can't be found).

The last decision is how the macro code inserts the data at the correct locations. The most robust method for this is to have the macro create a document variable for each piece of data, and to place DocVariable fields in the body of the template to display the values of those variables. For example, once the macro has read the data document and found that the phone number for the New York office is 212-555-1234, the code sets the value of the document variable:

ActiveDocument.Variables("Phone") = "212-555-1234"

and update the active document's fields, and the field {DocVariable Phone} in the body of the document will show the phone number. You can put two or more of the same DocVariable field into the template to repeat the same data in multiple places.

If you need step-by-step instructions, post back.

jay freedman say:
Just to correct a minor misunderstanding, although it shouldn't change the decision:

"Protected forms" are not related at all to Microsoft Rights Management. They use the form fields that were provided in Word 2003 and earlier, now renamed as "legacy form fields". To make them operate, you then "protect" or "lock" the form by clicking the Protect Document button on the Developer tab, setting the editing restriction type to "Filling in forms", and clicking Start Enforcing Protection. Working with these form fields in VBA is reasonably straightforward, but not as convenient as setting document variables.

You'll find the add-in from http://gregmaxey.mvps.org/word_tip_pages/cc_var_bm_doc_prop_tools_addin.html to be very useful in managing document variables.

littlelionkidd say:

For anyone else having this problem, I was able to fix it by doing the following:

Step 1:

Browse to the following location:

· C:\Program Files\Microsoft Office\Office 14 (Windows 32 bit)

· C:\Program Files x86\Microsoft Office\Office 14 (Windows 64 bit)

Locate the file "ppcore.dll"

If "ppcore.dll" is missing, then you need to download it and add it to the Office 14 folder or Powerpoint will not run.

Step 2:

Google "ppcore.dll download" and use a trusted site to download the file.

I got mine from DLL2U.com (http://dll2u.com/dll/ppcore.dll/2169/).

Make sure that the .dll file is the correct version (the link above is for Office 2010).

Step 3:

Save the file to the C:\Program Files x86\Microsoft Office\Office 14 folder.

If you get an error message saying that you must be an administrator in order to do this, save the file to your desktop and then drop it into the Office 14 folder after it downloads.

Step 4:

Just to be safe, in the Office 14 folder, locate the file "POWERPNT.EXE".
Right click the file and choose "Run as administrator".
Powerpoint should open.
