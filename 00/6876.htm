john - project say:
sohaibnasir,
Well yes you can do it using recurring tasks but an automated approach will probably take about as long to implement as doing the whole thing manually. Of course you could also automate it with VBA, but again, unless you have a multi-year project, writing the code could take longer than entering the tasks manually.

To use Project's recurring task feature you will have to separately set up the main tasks (i.e. Task A 1, A 2, etc.) and subtasks (i.e. Task B 1, Task C 1 and Task B 2, Task C 2, etc.). First set up the main tasks with a weekly recurring pattern and duration of 5d for as many occurrences as necessary to span the whole project, assuming the default 5d workweek. Then under each of those, insert a pair of recurring tasks with a weekly recurring pattern for 2 occurrences. You will then need to outdent, delete redundant main tasks and indent the subtasks. The whole thing is a rather laborious process and it might be just as easy to manually set up the task and subtasks and then do a copy and paste multiple times. Of course you would have to manually go back and change the task names as necessary.

Again, if you have a long project, this could be implemented in VBA.

John

braintrove.com say:
Typical things to consider when evaluating performance issues:
  1. Is this a Browser form or a Filler form? Browser forms will typically be your problem point for performance.
  2. What types of controls are in the view?
  3. Are all 90 controls in a single view?
  4. What types of data connections do you have?
  5. Do the data connections run on load or after an event occurs?
  6. How much data is being returned from each data connection?
  7. Are your data connections to in-house or external data sources?

Your source of pain is most likely your data connections, secondly repeating controls. Could be a combination of both.

The easiest way to pinpoint your problem is to start eliminating the possibilities. First, set all your data connections to not query on load. Then turn them back on, one-by-one, and see when your form slows down.

If it is not the data connections, then try deleting controls from your view until performance picks up (typically repeating controls: tables, sections, lists, etc.).
