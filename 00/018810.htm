dale howard [mvp] say:

Christine --

First of all, if the resource has been set to Inactive status, you will need to get your application administrator to set the resource back to Active status temporarily.  Then you can use the Delegation feature to create a delegation session where you are logged into PWA as the user.  In that delegation session, you can navigate to the Timesheet page in PWA and submit the timesheet for approval.  Then you can exit the delegation session and approve the timesheet.

To create the delegation session, navigate to the PWA Settings page in Project Web App and click the Manage Delegations link.  Create the delegation of you working on behalf of the resource who left.  After creating and saving the delegation, click the Act as a Delegate link and activate the delegation session.  Once you activate the session, PWA treats you as if you are the user who left.  Hope this helps.

Hi Gill,

You are correct.  It was the case that when the focus was in a PivotTable, adding a new measure would not only add the measure to the data model, it would also add the measure to the values area of the PivotTable.

This changed recently.  We had a large number of bug fixes that were being made to the code where you add or edit measures, and as part of that work, a change was made so that the measure would not automatically be added to the PivotTable.

Since that change was made, we have received feedback from enough people who want the measure added automatically, that we have made yet another change to restore the behavior that adds new measures to PivotTables automatically.  Bottom line is that you will get an update that restores the original behavior, and newly added measures will be automatically added to PivotTables after you get that update. 

I can't tell you exactly when the most recent update will reach you, but you may get this change as part of the August update (but maybe sooner or later depending on your channel.)

Sorry about the inconvenience and confusion caused by our changes.  The good news is that measure authoring should be much more robust than in the past based on a number of bug fixes we've made in Power Pivot in recent months.

Howie
