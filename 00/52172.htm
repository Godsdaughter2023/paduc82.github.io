  1. Do you get any error code/message when you give any print job? If so, what is the exact error message that you get?

There are many instances that suggest the print spooler service malfunctions due to some specific third party anti-virus programs installed in the system. Other reasons include corruption with the dependency services, other third party software conflicts etc.

In order to diagnose and troubleshoot the issue, I would suggest you to try the following methods and check if the issue persists.

Method 1:

Let's start the all the dependencies service for the printer spooler service. Follow the steps provided.

  1. Press Windows key + R and type services.msc in the search field and press ENTER.
  2. Scroll down toPrint spooler.
  3. Right click and click properties, click the tab “Dependencies”.
  4. Note all of the dependent services listed, and then close the Print Spooler Properties window. 

If any of the dependent services are missing or cannot be started, try uninstalling all installed printer drivers and then reinstalling them.

If the issue still exists, try the next method.

Method 2:

  1. Press Windows key + R and type services.msc in the search field and press enter.
  2. In the Services listed stop the Print Spooler service. 
  3. In Windows Explorer, navigate to the folder below, and open the PRINTERS.
            C:\Windows\system32\spool\PRINTERS
  4. Click on Continue or Yes (if prompted by UAC) to approve opening the PRINTERS folder.
  5. Delete all files in the PRINTERS folder until it is empty, and then close Windows Explorer. 
    Note: DO NOT DELETE the PRINTERS folder itself. Only delete its contents.
  6. Open Services, and start the Print Spooler service.

If the issue exists, try the next method.

Method 3:

I would suggest you to refer to the steps posted by gagua replied on October 28, 2013 in the following thread and check if it helps.

Print Spooler does not start at Windows Restart

a. user say:

When trying to enable Windows Defender, you may get the following error: ‘This app is turned off by group policy’ and this is caused by two main reasons:

- If your PC is part of a domain and the domain controller assigned some policies, Windows Defender may get blocked.

- If you’ve installed a 3rd party antivirus, it is likely to block Defender and the block may remain active even if you’ve uninstalled the application. In such a condition, when you try to enable defender, you’ll get the error.

In both cases, it has nothing to do with your PC still running Windows 8 that is no longer supported since 2016.

However,  this error is largely fixed by asking your system administrator to enable Windows Defender via Group Policy or you can also fix this by turning it on with the Local Group Policy Editor or applying a registry tweak.

Since support for Windows 8 has ended, fixing this issue is more or less counter-productive and waste of time because you'll still be faced with the OS inability to receive and install updates.

So, you're encouraged to check if your PC hardware meets the minimum requirements, back up your data, purchase Windows 10 Home or Pro from Microsoft Store and perform a clean install.
