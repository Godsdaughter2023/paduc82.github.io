Macro to create a list of linked images in word document? - Microsoft Community

mikestarr say:

Macro to create a list of linked images in word document?

I've got a number of Word documents with many linked images in each document. Is there a macro to generate a list of linked images within a document?

 

Thanks,

 

Mike Starr

greg maxey say:

The error means it has hit a picture that isn't linked. I you select all and toggle field codes (ALT+F9) do "all" your pictures change to the approprite field code? If not then the one or ones that don't are not linked pictures. I've added a condition to skip those.  This code picks up linked pictures in tables here:

 

Option Explicit
Private oRng As Word.Range
Sub PictureIndexII()
'Declare variables
Dim oILS As InlineShape
Dim oPic As Object
Dim oBM As Bookmark
Dim strCaption As String
Dim strBM As String
Dim oRng As Word.Range
Dim i As Long
'Clear existing photo captions
ResetPhotoCaptions
'Delete existing index if exists
If ActiveDocument.Bookmarks.Exists("PhotoIndexRange") Then
  ActiveDocument.Bookmarks("PhotoIndexRange").Range.Delete
End If
'Move to end of document
ActiveDocument.Bookmarks("\endofdoc").Select
With Selection
  'Insert a formatted Index Heading
  .TypeText vbCr
  .Style = ActiveDocument.Styles("Index Heading")
  .ParagraphFormat.SpaceBefore = 12
  .ParagraphFormat.SpaceAfter = 6
  Set oRng = Selection.Range
  oRng.Text = "Index of Document Pictures" & vbCr
  'Mark the start of the index text range
  ActiveDocument.Bookmarks.Add "PhotoIndexRange", oRng
  oRng.Collapse wdCollapseEnd
  oRng.Select
  'Setup index list and tabs
  .Style = ActiveDocument.Styles("Index 1")
  .ParagraphFormat.TabStops.Add Position:=InchesToPoints(6.5), _
     Alignment:=wdAlignTabRight, Leader:=wdTabLeaderDots
  'Process each picture type InLine Shape and list in the index
  For Each oILS In ActiveDocument.InlineShapes
  
    i = i + 1
    If oILS.Type = wdInlineShapePicture Or wdInlineShapeLinkedPicture Then
    
      If oILS.Range.Fields.Count > 0 Then
        Set oPic = oILS
        strCaption = oPic.LinkFormat.SourceFullName
        strBM = strCaption
        'Make string suitable for bookmark name (eliminate spaces)
        strBM = "BM_" & i
        'Wrap oPic in bookmark
        Set oBM = ActiveDocument.Bookmarks.Add(strBM, oPic.Range)
        On Error GoTo Err_MissingLabel
        oPic.Range.InsertCaption Label:="Photo", TitleAutoText:="", Title:=": " & strCaption, _
        Position:=wdCaptionPositionBelow, ExcludeLabel:=0
        ActiveDocument.Bookmarks("\endofdoc").Select
        'Create the index entry
        .TypeText strCaption & vbTab
        'Hyperlink to the bookmark
        ActiveDocument.Hyperlinks.Add Anchor:=Selection.Range, _
           Address:=ActiveDocument.Name, SubAddress:=oBM.Name, _
           TextToDisplay:=oPic.Range.Information(wdActiveEndAdjustedPageNumber)
        .TypeText vbCr
      End If
   End If
  Next oILS
   'Remove empty paragraph at end of document
   .Paragraphs(1).Range.Delete
End With
Set oRng = ActiveDocument.Range
oRng.Start = ActiveDocument.Bookmarks("PhotoIndexRange").Range.Start
ActiveDocument.Bookmarks.Add "PhotoIndexRange", oRng
Exit Sub
Err_MissingLabel:
If Err.Number = 4198 Then
  CaptionLabels.Add Name:="Photo"
  Resume
Else
  MsgBox Err.Number & " " & Err.Description
End If
End Sub
Sub ResetPhotoCaptions()
Set oRng = ActiveDocument.Range
With oRng.Find
  .ClearFormatting
  .Format = True
  .Forward = True
  .Style = ActiveDocument.Styles("Caption")
  .Replacement.Text = ""
  .Wrap = wdFindStop
  While .Execute
    If oRng = ActiveDocument.Range Or oRng = "" Then Exit Sub
    If InStr(oRng, "Photo") > 0 Then
      oRng.Paragraphs(1).Range.Delete
      oRng.Collapse wdCollapseEnd
    End If
   
  Wend
End With
End Sub
Sub AddCaptionLabel()
CaptionLabels.Add Name:="Photo"
End Sub

I am trying to calculate on a date field in an Access form.  I have two fields, the first is a date/time field called Last PM Date, the second field is a numeric field called Date PM Calc.  I want to add the entered number of days from Date PM Calc to the Last PM Date field and enter the result into another field on the form called Next PM Date.  I'm getting a result, but it's a weird date result.  I'm sure it has to do with one field being date/time and the other numeric, but I don't know how to get around this. Any suggestions would be greatly appreciated.  Thanks!
ken sheridan say: The first question to ask is whether the Next PM Date is immutably determined by the values of the Last PM Date and Date PM Calc fields.  If it is then you should not store it in a separate field in the table, but always compute it when required.  Otherwise you are introducing redundancy and the risk of update anomalies.  You simply set the ControlSource property of the Next PM Date control in the form to:

=DateAdd("d",[Date PM Calc],[Last PM Date])

If, on the other hand, the Next PM Date is not immutably determined by the values of the Last PM Date and Date PM Calc fields, but can legitimately be changed to a date which is not Last PM Date + Date PM Calc, then you should have a column in the table for it, and you can assign a value to it in by putting the following code in the AfterUpdate event procedures of both the Last PM Date and Date PM Calc controls:

If Not IsNull(Me.[Date PM Calc]) And Not IsNull(Me.[Last PM Date]) Then
    Me.[Next PM Date]  =DateAdd("d",Me.[Date PM Calc],Me.[Last PM Date])
Else
    Me.[Next PM Date] = Null
End If

The reason you are currently getting the strange values BTW is that the value of the Last PM Date control is being treated as an arithmetical expression, not a date.  Because 30 December 1899 is day-zero in Access's date time implementation , and 4/20/2012 as an arithmetical  expression evaluates to 0.000099403578528827, this value represents the date time value of 30 December 1899 00:00:09.  You can see this yourself by entering

? Format(4/20/2012,"dd mmmm yyyy hh:nn:ss")

in the debug window.

This indicates that the times in column A that look the same are in fact slightly different. You can get around this by inserting the following line below For i = 1 To UBound(arr, 1):
