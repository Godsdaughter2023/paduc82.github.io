If you want to connect 100 shapes to one shape then you can use the below VB code.


Sub Connectshapes()


Dim shapeids As Integer


For shapeids = 2 To Visio.ActivePage.Shapes.Count


Visio.ActivePage.Shapes(shapeids).AutoConnect Visio.ActivePage.Shapes(1), visAutoConnectDirRight


Next


End Sub


If you want to connect all shapes to each other then you can use the connectshapes command in Visio. This command is hidden by default. You can go to Options->Customize ribbon-->Choose commands from all commands--> Add Connectshapes to the customribbon.

dirk goldgar say:

I hardly know anything about Access so I'm not exactly sure how to describe what I want...

 

I've been building a database to track on-site training contracts.  It opens to a form which acts somewhat like a "switchboard" with various buttons to other places.  But the main thing is a subform on that form which is a datasheet listing all the contracts.  I did this because it's easy to sort/filter the contracts in a datasheet.  A click on any part of any record in that subform opens the Contracts form at that specific record.  All this works great.

However, once on the Contracts form, the navigation buttons are pointless because of the filter that was applied in getting to this particular record.  If I remove the filter, the form switches to the first record...

How can I click on one of the contracts listed on the "switchboard" and go to that record in the Contracts form and then, once there, be able to navigate the records in that form?

THANKS in advance!


The trick for this would be, not to *flter* the form when you open it, but to position it to the record you want immediately after you open it.  Here's example code:

'------ start of example code ------
Function OpenContract(vContractID As Variant)

    ' Open the contract form to the contract passed as ContractID.

    DoCmd.OpenForm "frmContracts"

    If Len(vContractID & vbNullString) > 0 Then
        Forms!frmContracts.Recordset.FindFirst "ContractID = " & vContractID
     End If

End Function
'------ end of example code ------

That assumes that the ContractID (or primary key field) is numeric, not text.

You would call this from whatever event(s) you want to use, passing the ContractID of the current record when you do.
