ossiemac say:

Try the following. Basically it does the following:

  • Hides the blank lines in the original data and copies the visible data to a temporary worksheet.
  • Sets up the printing parameters as if the worksheet is to be printed because that is actually how it is sent to the pdf file.
  • Print parameters set for one page wide by however many long.
  • Print parameter set for rows to repeat at top of sheet.
  • Returns to the original data worksheet (Original ActiveSheet)
  • Deletes the temporary worksheet.

If you have any questions about the code then feel free to get back to me.

Sub create_and_email_pdf()
   
    Dim wsActive As Worksheet
    Dim EmailSubject As String
    Dim EmailSignature As String
    Dim CurrentMonth As String
    Dim DestFolder As String
    Dim PDFFile As String
    Dim Email_To As String
    Dim Email_CC As String
    Dim Email_BCC As String
    Dim OpenPDFAfterCreating As Boolean
    Dim AlwaysOverwritePDF As Boolean
    Dim DisplayEmail As Boolean
    Dim OverwritePDF As VbMsgBoxResult
    Dim OutlookApp As Object
    Dim OutlookMail As Object
    Dim rngRow As Range
    Dim rngToCopy As Range
    Dim wsTemp As Worksheet
   
   
    Set wsActive = ActiveSheet
    CurrentMonth = ""
   
    EmailSubject = "Family totals report for " & Range("k10").Value
    OpenPDFAfterCreating = False
    AlwaysOverwritePDF = False
    DisplayEmail = False
   
    With Application.FileDialog(msoFileDialogFolderPicker)
        If .Show = True Then
            DestFolder = .SelectedItems(1)
        Else
            MsgBox "You must specify a folder to save the PDF into." & vbCrLf & vbCrLf & "Press OK to exit this macro.", vbCritical, "Must Specify Destination Folder"
            Exit Sub
        End If
    End With
   
    'CurrentMonth = Mid(wsactive.Range("H6").Value, InStr(1, wsactive.Range("H6").Value, " ") + 1)
    'Following line used by OssieMac in lieu of previous line for testing purposes
    CurrentMonth = Format(wsActive.Range("H6").Value, "mmmm")   'Extracts month from an actual date
   
   
    PDFFile = DestFolder & Application.PathSeparator & wsActive.Name _
        & "_" & CurrentMonth & ".pdf"
   
   
    If Len(Dir(PDFFile)) > 0 Then
        If AlwaysOverwritePDF = False Then
            OverwritePDF = MsgBox(PDFFile & " already exists." & vbCrLf & vbCrLf & "Do you want to overwrite it?", vbYesNo + vbQuestion, "File Exists")
            On Error Resume Next
            If OverwritePDF = vbYes Then
                Kill PDFFile
            Else
                MsgBox "OK then, if you don't overwrite the existing PDF, I can't continue." _
                    & vbCrLf & vbCrLf & "Press OK to exit this macro.", vbCritical, "Exiting Macro"
                Exit Sub
            End If
        Else
            On Error Resume Next
            Kill PDFFile
        End If
       
       
        If Err.Number <> 0 Then
            MsgBox "Unable to delete existing file.  Please make sure the file is not open or write protected." _
                    & vbCrLf & vbCrLf & "Press OK to exit this macro.", vbCritical, "Unable to Delete File"
            Exit Sub
        End If
    End If
   
    With wsActive
        For Each rngRow In .UsedRange.Rows
            If WorksheetFunction.CountA(rngRow) = 0 Then
                    rngRow.Hidden = True
            End If
        Next rngRow
       
        Set rngToCopy = .UsedRange _
                   .SpecialCells(xlCellTypeVisible)
    End With
              
    Set wsTemp = Sheets.Add(After:=Worksheets(Sheets.Count))
   
    rngToCopy.Copy Destination:=wsTemp.Cells(1, 1)
   
    With wsTemp.PageSetup
        .PrintArea = wsTemp.Cells(1, 1).CurrentRegion.Address(1, 1)
        .PrintGridlines = True
        .PrintTitleRows = "$1:$1"       'Rows to repeat at top if multiple pages
        .Orientation = xlLandscape
        .Zoom = False                   'Zoom must be off to next 2 lines to work
        .FitToPagesWide = 1             'Ensure will fit across 1 page
        .FitToPagesTall = False         'Set to 1 if all to be on one page
    End With
   
    wsTemp.ExportAsFixedFormat Type:=xlTypePDF, _
                                Filename:=PDFFile, _
                                Quality:=xlQualityStandard, _
                                IncludeDocProperties:=True, _
                                IgnorePrintAreas:=False, _
                                OpenAfterPublish:=OpenPDFAfterCreating
   
    With wsActive
        .Activate                   'Return to original data sheet
        .Rows.Hidden = False        'Unhide the hidden blank rows on main data sheet.
        .Cells(1, 1).Select         'Goto first cell on sheet
    End With
   
    Application.DisplayAlerts = False   'Suppress message for next line
    wsTemp.Delete       'Delete the temporary sheet
    Application.DisplayAlerts = True    'Turn alerts on again
   
   
    Set OutlookApp = CreateObject("Outlook.Application")

        .Display

        '.CC = Email_CC
        '.BCC = Email_BCC
        .Subject = EmailSubject & CurrentMonth
        .Attachments.Add PDFFile
            
        If DisplayEmail = False Then
         
        .Send
         
        End If
   
    End With

Set a filter on a Form based on search field - Microsoft Community

kil kim say:

Set a filter on a Form based on search field

I'm trying to create Web DB with 2010 in a SharePoint site. I'm try to build a web compatible macro to run it in a SharePoint Site.

 

I have a navigation form and there are three tabs.  One tap has a form and I want to set a filter on the form based on a user types search word in a search box (Non-bound text box) that I created.

 

The form is based on a query.  I added a field "Searchable" in the query and I used concatenated several fields so that I can search on that field.  The "searchable" field is hidden field in the Form.

 

What is best way to handle it and how can I do it?

albert d. kallal say:

I cannot say that using concatenated columns for searching ever was a very good idea in client or web based versions of Access.

 

If you want to search by say Lastname OR company name, I would just include both criteria in the filter command and not bother with all that messy concatenation stuff anyway. One good reason here is that such columns cannot use indexing and they are rather slow in terms of performance. And of course it becomes quite hard to match such a column made up of several columns and this quite much suggests you have to use wild card searching (again even slower).

 

However, the basic code you need will look like this:

SetLocalVar
    Name   strSearch
   Expression  = "*" & txtSearchBox & "*"

SetFilter
Where Condition = [FullName] Like [LocalVars]![strSearch]

 

So the above will do a wild card search on a concatenated column called FullName. Note how we surrounded the input search with an * to allow wild card matching.

However, the above is less than ideal if you going to have more than say say about 600 records. I would thus not concatenate the two columns you need to search on. By searching on each column then you ensure that indexing can be used.

Let's assume that the above full name was to search for Last Name or company. And we want to type in the first few charactors. I would suggest the following code will not only elimonate the concatenated column, but is far better and faster:

SetLocalVar
    Name   strSearch
Expression  = txtSearchBox & "*"
SetFilter
Where Condition = [LastName] Like [LocalVars]![strSearch] or [Company] like [localVars]![strSearch]

 

Note in the second example we remove the first "*" for the wild card. And then we search directly on the high speed indexed columns. While the result is much the same, the second example will run 100 if not a 1000 times faster.

Edmonton, Alberta Canada
