What wrong is the date format? - Microsoft Community

oemsoft say:

What wrong is the date format?

In cell T1, there is a given date 12-Jan-2012, which is d-mmm-yyyy format, and in the regional setting for Date under control panel, the short date format is set to be dd-mmm-yy.

 

When I run following macro DMY, the date is changed from 12-Jan-2012 to 1-Dec-2012,

 

Does anyone have any suggestions on what wrong it is? and what is controlling the date setting format?

 

Thanks in advance for any suggestions

 

Sub DMY()

    Columns("T:T").Select
    Selection.TextToColumns Destination:=Range("T1"), DataType:=xlDelimited, _
        TextQualifier:=xlDoubleQuote, ConsecutiveDelimiter:=False, Tab:=True, _
        Semicolon:=False, Comma:=False, Space:=False, Other:=False, FieldInfo _
        :=Array(1, 4), TrailingMinusNumbers:=True
End Sub 

hansv mvp say:

TextToColumns is intended to convert text values, as its name indicates.

 

If cell T1 contains 12-Jan-2012 or 12/01/2012 as a text value (for example because the cell is formatted as Text, or because the value has been prefixed with an apostrophe ' when it was entered, the macro will change it to a real date value corresponding to 12 January 2012.

 

But if the value in cell T1 is already a real date value, the actual value of the cell is 40920 (the number of days from 31 December 1899 to 12 January 2012), regardless of the regional settings and of the number formatting of the cell. You can check this by setting the number format of the cell to General.

VBA uses US date format mm/dd/yyyy internally, so when it represents the value as text (since TextToColumns converts text), it 'sees' the date as "01/12/2012". Because you specified DMY, the code then faithfully changes this to 01-Dec-2012.

 

Confusing, isn't it?

 

The moral of this story: use TextToColumns only to convert text values, not to convert values that are already stored as dates.
