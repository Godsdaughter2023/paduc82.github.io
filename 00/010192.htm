bill manville say:

"Link" in Excel means a dynamic link between cells such that when the source is updated the destination also changes. 


What you have is a QueryTable in one workbook whose source happens to be a table in another workbook.  All that you will ever get by refreshing the query is the values that are now in the source workbook.  Formulas and formatting of the source are irrelevant (the mechanism is designed for retrieving data from databases which have neither formulas nor formatting).

 

To preserve formatting applied to the querytable when refreshing:

With the cursor in that querytable, Table Tools: Design > Properties > ensure Preserve cell formatting is checked.

bill manville say:

If your source sheet (say Sheet1 in Book1.xls) contains a formula, e.g. =A1+A2 and you copy the cell containing that formula and paste it into your destination sheet the formula will become

 

=[Book1.xls]Sheet1!A1+[Book1.xls]Sheet1!A2

 

If you change the value in A1 in Sheet1 in Book1.xls the formulas in both workbooks will update (if both are open).

 

So I am guessing that what you want to do is to copy a range from the source and paste it into the destination, but if that doesn't meet your needs you will have to explain what you are trying to achieve in more detail, preferably with a link to a copy of the workbook on a file-sharing website, or a picture here of the relevant parts of your workbooks.

bill manville say:

The issue that is causing us problems is your desire to have the formulas that exist in the source replicated in your copy.

 

If there really is a requirement to copy aribitrary formulas and have them referencing cells in the source workbook then you will need to copy and paste.

 

If, on the other hand there are formula columns in the source data that reference other cells in the source data - for example in each cell in column E there is a formula that divides the value in column D by the value in column C on the same row - then you could set up corresponding formulas in your workbook and just pull the values of the constant (non-formula) columns in from the source workbook.

Using a query would be a perfectly acceptable approach in that case.

 

Finally, if you really don't need the formulas and just the results of the formulas are required then again

using a query as you were doing would be fine.
