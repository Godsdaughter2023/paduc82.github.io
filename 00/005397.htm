Hello Ann

Answer to your question is: "It's neither"

Uninstalling your versions of Office/Outlook doesn't remove the Outlook profiles. When you installed '365, it copied the '2010 profile which contains the email account and data file information.which in this case, the PST file was the default data file for the profile and is where the default contact/calendar folders are stored.

Now, if you configured your account as an IMAP account, it would have retained the PST file as the default data file which is actually a good thing.

Some background info.............

An IMAP account in Outlook '2013 and later, an IMAP account uses an .ost file to store it's email data. If it's the first account configured (first as in, nothing was configured previously, that .ost file would also contain contact/calendar folders with the added words of "This Computer Only" in the folder description and it's not really recommended to store your contact/calendar data in those folders for the following two reasons

#1 - If you ever remove the IMAP account, the .ost file would also be removed meaning that the contact/calendar data will be lost
#2 - If Outlook can't find/open the .ost file for the account, it will create a new one with empty contact/calendar folders - ergo, your contact/calendar data will be lost.

An .ost file can only be opened by the original Outlook profile/email account used to create it. If you have an "orphaned" .ost file, you would need to use a 3rd party OST2PST utility (not free) to retrieve any data from the .ost file.

Getting back to your current scenario................continue to use the existing PST file as the default data file for the profile which prevents loss of contact/calendar data as described above. If everything is as described, you will have one OST file and one PST file configured in your current Outlook profile (assuming only one email account). The PST file will be set as the default data file. To verify this, from within Outlook, go to File --> Account Settings --> Data FIles tab OR you can open the Control Panel <Mail> app and get to the same place.

If things are different from what I've just described then there is something missing from your description but to also answer your direct question of how to "start over" , the process is simply to open the Control Panel <Mail> app (with Outlook closed), <ADD> a new profile, configure your IMAP account and set the newly created profile as the default for the next time Outlook is opened. Your contact/calendar data would still be in the PST file which you would either have to add to the profile (and set it as the default data file) or import the contact/calendar folders into the "This Computer Only" folders (not recommended).

Entourage expects the Microsoft User Data folder to be in your Documents
folder. For OS X users, that's the Documents folder in your User's folder.
When Entourage opens in a new blank Identity, it's usually User error.

deleted the file. If you have deleted the trash, it's gone. Sorry!

Select Switch Identity (under Entourage in the Menu bar) and see if your old
identity is still listed.

To find your old database, search for "Microsoft User Data" (without the
quotes).

If you have not used a new identity in Entourage, then copy the entire
Microsoft User Data folder to the correct location.

Fix if you have more than one Microsoft User Data Folder...

Move the entire identity folder you want to use to the Office Identities
folder that Entourage is recognizing. Your identity is named "Main" by
default. Be sure to rename one so you do not overwrite. Your identity folder
can contain more than one identity. Switch Identities to view your old mail.
If you have not used the new identity, delete it. If you have used it and
want to combine the new mail with your original database, drag each folder
to the desktop from one identity. This creates an MBOX file. Switch
identities. Drag the MBOX files into the folder window of Entourage.

You can delete the old folder after you are satisfied with the transfer.
