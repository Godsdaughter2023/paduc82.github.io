How to add unbound text data and bound data on a form to a table - Microsoft Community

fcin say:

How to add unbound text data and bound data on a form to a table

I have a form that writes to a table. In the form I have bound fields.. I have a text box, unbound that has an if then statement which updates based on entries on another bound field. I would like to have the information that is entered in the bound fields as well as the unbound text field added to the table.
ken sheridan say:

The form only needs to join the tables in a query if the comouted value should not be saved to the table.

Now that I have a clearer picture of your logical model, there is no need for a subform at all.  A subform is used to represent a one-to-many relationship type between the parent form's underlying table and the subform's table.  In your case the relationship type is the reverse, many-to-one.  All you need do is base the parent form on a query which joins its current table to the referenced table on the Item name Type column, and return the value from the SLA Days column in the referenced table in a bound control in the parent form.  Set the control's Locked property to True (Yes) and its Enabled property to False (No) to make it read-only.

If the expected close date value is time independently determined by the key of the referenced table An unbound control in the parent form can then compute the expected close date very easily with:

=WorkDaysAdd([Date Opened],[SlaDays])

As the SlaDays column can apparently only have values of 3 or less then there is no need for an IIF function call.

If, on the other hand, the expected close date value is not time independently determined by the key of the referenced table, but can either be manually changed in the parent form's record, or should remain static in the parent form's record even if the SlaDays value in the referenced table is subsequently changed then there should be an SlaDays column in the parent form's table and the text box in the form should be bound to it, not to the SlaDays column in the referenced table.  The referenced table should not be included in the form's underlying query.  The  Item Name Type combo box should be set up as follows:

ControlSource:  [Item Name Type]

RowSource:     SELECT  [Item Name Type], [SLA Days] FROM [YourReferencedTableName GoesHere] ORDER BY  [Item Name Type];

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  8cm;0cm

If your units of measurement are imperial rather than metric Access will automatically convert the values to inches.

The value should then be assigned in the AfterUpdate event procedure of the Item Name Type control in the parent form with:

With Me.ActiveControl
    Me.[[SLA Days] = .Column(1)
End With

The Column property is zero-based, so Column(1) is the hidden second  column.  The expected close date control should be an unbound text box with the same expression as before as its ControlSource property.

This second scenario is directly analogous to the very common one of a unit price for a product looked up from a referenced products table and assigned to a UnitPrice column in an OrderDetails table.  Northwind does this in its order details subform for instance.
