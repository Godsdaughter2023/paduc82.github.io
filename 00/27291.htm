graham mayor say:
If you want the numbering sequence to remain the same in the first column, then you are going to have to use automatic numbering to maintain that number, otherwise the numbers will follow the sorted items in the second column.

If the numbering starts at 1, then you can use SEQ fields to set the numbering. If it starts at a different number, you will need to insert a calculated field in each cell of column 1 e.g.

{ ={ SEQ Col1 } + { DocVariable varStart } -1 }

You can then use a macro to provide the start number, sort the table on column 2 and update the SEQ fields e.g.

Dim iStart As Long
Dim oVars As Variables
Set oVars = ActiveDocument.Variables
On Error Resume Next
iStart = oVars("varStart").Value
If oVars("varStart") = "" Then
    iStart = InputBox("Enter start number", , 1)
End If
ActiveDocument.Tables(1).Sort _
        ExcludeHeader:=False, _
        FieldNumber:="Column 2", _
        SortFieldType:=wdSortFieldAlphanumeric, _
        SortOrder:=wdSortOrderAscending
oVars("varStart").Value = iStart
ActiveDocument.Fields.Update

There is no way for the results to update dynamically, but what may work for you is to run the macro as you tab out of the last cell in the table. This will create a new empty row, but is an action you can trap e.g.

Sub NextCell()
Dim iStart As Long
Dim oVars As Variables
Dim oTable As Table
Dim iRow As Long, iCol As Long
Set oTable = Selection.Tables(1)
Set oVars = ActiveDocument.Variables
iRow = oTable.Rows.Count
iCol = oTable.Rows(iRow).Cells.Count
If Selection.Range.InRange(oTable.Cell(iRow, iCol).Range) Then
    On Error Resume Next
    iStart = oVars("varStart").Value
    If oVars("varStart") = "" Then
        iStart = InputBox("Enter start number", , 1)
    End If
    ActiveDocument.Tables(1).Sort _
            ExcludeHeader:=False, _
            FieldNumber:="Column 2", _
            SortFieldType:=wdSortFieldAlphanumeric, _
            SortOrder:=wdSortOrderAscending
    oVars("varStart").Value = iStart
    ActiveDocument.Fields.Update
End If
Selection.MoveRight Unit:=wdCell
End Sub

Solution to the problem:

I figured out the solution to my Windows 8.1 Nokia phone showing the wrong time.  I went to Settings -> Date-Time and set the time to Automatic.  The time zone field was empty, however when I selected the drop down my correct time zone showed in the list--as if the correct time zone was selected even though it wasn't showing in the time zone field.  I changed the time zone to another time zone, allowing it to populate the time zone field.  I then changed the time zone back to my time zone, which populated the time zone field correctly and gave me the correct time.  This is obviously a glitch in Windows 8.1, but at least a fixable one.

Just to reiterate: If your Automatic time is displaying the wrong time and your time zone field is empty (even though the time zone list shows it as selected), change to another time zone, allowing it to populate the time zone field, and then change back to your actual time zone, allowing it to populate the time zone field.

jim gordon mvp say:

Since your workbook has a lot of sheets, you might try using a process of elimination.

First, unhide any hidden or very hidden worksheets.

Next, select half the sheet tabs (Command-click the tabs), then right click on one of the selected sheet tabs and choose Move or Copy. Click the checkbox for Copy and choose To book: (new workbook).

Save the new workbook and see how big it is.

If it's big, you know the problem is one of the sheets you copied. If it is small, those sheets are OK.

Repeat for the remaining worksheets.  Keep copying until you isolate the problem worksheet(s). That way you only have to repair the broken worksheet(s).
