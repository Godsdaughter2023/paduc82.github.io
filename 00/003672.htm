    How to reenable IE to ask for password and save it if you previously - Microsoft Community

    asölkhn åsia nåsi gsadg say:

    How to reenable IE to ask for password and save it if you previously have answered no to that question for that website

    Hi,

    I haven't found a posted solution for this problem, but I managed to figure it out,

    at least for Windows 10.

    The key is the command vaultcmd that you use to manage credentials.

    1. First list your vaults and save the name of the vault where your web credentials likely are stored

    The names seems to be localized.

    > vaultcmd /list

    In my case the name was Webbautentiseringsuppgifter

    2. List the credentials in this vault 

    > vaultcmd /listcreds:name_of_vault

    3. Find the site for which you want to reenable password and save the Credential schema name,

    resource  and identity 

    For me the schema name was Windows Web Password Credential

    4. List the schemas and find the GUID for the schema saved in 3  

    > vaultcmd /listschema

    5. Delete the credential. identity_to_remove and url_to_remove should be typed just as the are written in listcreds,

    no brackets or quoutes.

    > vaultcmd /deletecreds:"name_of_vault" /credtype:{guid_of_schema} /identity:identity_to_remove /resource:url_to_remove

    Credentials deleted successfully.

    * Original title: How to force IE11 ask for password / reenable password

    how do i get my document pane to save as an outline when saving as a - Microsoft Community

    daniellehoffman say:

    how do i get my document pane to save as an outline when saving as a pdf

    I have just updated my MAC office to 2011.  I can no longer save the heading styles as bookmarks or an outline when saving my file as a pdf.  Anyone know how to do this other than inserting a bookmark at each heading?
    john mcghie say:
    Hi Danielle:

    The Table of Contents Generator in Word for Mac cannot generate hyperlinks.

    If you have a document shose TOC was last generated on a PC, and you included the \h switch, the hyperlinks will be in the TOC and they will work.  The money you re-generate the TOC in Word for Mac, the hyperlinks will be removed, because the Mac TOC generator can't produce them.

    If you then print the document to PDF, using either the Adobe PDF writer or the Apple one that was made by Adobe, the hyperlinks will not be transported into the PDF, whether they are in the document or not.  As Bob says, we're not sure whether this is Apple's Fault, Adobe's Fault, or Microsoft's Fault.  But none of them seem to want to fix it.

    If you generate the PDF on a PC, depending on which software you use, the hyperlinks will be transported into the PDF.  Both the Microsoft and the Adobe PDF generators will pass hyperlinks through.

    If your text contains strings that "look" like hyperlinks (http://www.smh.com.au) then the Adobe PDF Reader will "guess" that these are hyperlinks, and enable them.  Other PDF readers may not.  If the text does not look like a hyperlink ("Click here to read the Sydney Morning Herald") then it won't work in any PDF reader.

    The Mac TOC Generator will insert bookmarks at each heading for you automatically, and REF fields at each page number in the TOC (as the PC one does).  So if you click the page number in a TOC on Mac Word, you will jump to the appropriate heading.  But those REF bookmarks and their destination anchors will not be carried into the PDF.

    All this is a long-winded way of agreeing with Bob: this is not working because what you are missing is the Hyperlinks function.  Office for Mac has never had it, and still doesn't.

    There's no work-around: you either send a Word document, or you add the hyperlinks manually and save as a web-page, or you can't send hyperlinks.

    Sorry
