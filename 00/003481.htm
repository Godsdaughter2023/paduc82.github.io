First you have to set the workfunction reference, and assign the result to a variable. Next the first and third argument is required to be a range reference, where argument1 is the table, argument3 is the criteria. Finally argument2 is column label or the number of the column in the table.

Try something like below:

myInt = 12

For i = 1 To myInt
     'do something
     If i = myInt Then
          myvar = myInt - 1
          Range("G2") = myvar
          MyResult = WorksheetFunction.DCount(Range("MyTable"), 3, Range("G1:G2"))
     End If
Next

I am trying to redirect my e-mails using an old .pop account via outlook 2010 (on PC 1) to an @outlook.com account using outlook 2013 (on PC 2)

You can do all of it with one computer (PC2) using O'2013 by doing the following:

#1 - Add your POP account to your Outlook '2013 profile. If you want these same items to appear on PC1 - just make sure you set the POP account to "Leave messages on the server" on PC2

#2 - <ADD> your Outlook.com account to your O'2013 profile but configure it as an Imap account (this will result in your Outlook.com account being configured twice in the same profile - once as an EAS account assuming that's how you're using it now as well as an Imap account)

#3 - Create a rule moving or copying new Inbox items for the POP account to the Inbox associated with the Outlook.com Imap account. Once the Imap account syncs - you'll see the same messages in your Outlook.com EAS account <Inbox> folder.

Powerpoint and HD

steve rindsberg say:

The print size will be whatever size you print it at.  The notion that physical sizes are built into digital images is a (sometimes helpful) myth.  All you can be sure of is pixel count.

But yes, if you set the slide up to be 20" wide, then when you save as an image, it'll likely be 1920 pixels wide (96 X 20") though that 96 factor may vary from system to system depending on your Windows settings.  Export one image as a test, do the math and you'll know the magic number though.

Steve Rindsberg PowerPoint MVP
PowerPoint FAQ
PPTools Add-ins for PowerPoint

steve rindsberg say:

>> So, it is only images that have to be sized for the right resoultion... All other graphics are vector based and will be automatically/dynamically adjusted, is that correct?

Exactly so.

>> If I want to export the slides as images, then I guess that I'll need to change the resolution manually... True?

When you export as (actually Save As) images, you get one choice of resolution, and PowerPoint makes it for you.  The result depends on several factors but as a rule it's slide size (in inches) times 96, though different video settings can affect that.  In other words, for the usual 10" wide slide, you'll get images that are 960 pixels wide.

In case you need more flexibility (and higher quality images) I have a commercial add-in that fills in some of the useful bits that PPT leaves out. http://www.pptools.com/imageexport/  The free demo is fully functional, in case you'd like to try it out.
