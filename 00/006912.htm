What Microsoft did RIGHT in the past - Microsoft Community

big bad bombastic bob say:

What Microsoft did RIGHT in the past

I have decided to switch directions a little bit to POINT OUT the *kinds* of things that Microsoft has done CORRECTLY in the past, what made them successful.  This, of course, is in OPPOSITION to the direction that is driving Windows 10 into becoming an INFERIOR product.

1.  Windows for the masses - Windows 3.0!

Instead of doing what IBM wanted, which tied OS/2 PM 1.2 [very 3.0-like] to the PS/2 computers, and was VERY hard to get for "regular" computers [I tried], they re-vamped their Windows product into the SKEUMORPHIC success that was Windows 3.0!

2.  32-bit windows and pre-emptive threading for EVERYONE

Microsoft YET AGAIN did the right thing when they spent a LOT of effort on Windows '95, which had pre-emptive multi-tasking for 32-bit applications, and kept enough of the 16-bit world around to make 16-bit applications COMPATIBLE.  Not only that, they added the 'Start Menu' button, which made accessing a hierarchical list of applications and files a LOT easier than before.

3.  OLE and Automation using COM objects

The COM object idea was not a new one (CORBA and other unix implementations were using a very similar construct), but it was a GOOD one.  The idea that an object could be allocated and queried for supporting interfaces made it a LOT easier (than things like DDE) for applications to share data and even call each other to edit and present data.  It was now possible for some 3rd party application to use Word or Excel to edit and/or present Word and Excel files.  It had its limitations, but it was a good start at integrating the desktop between applications.

4.  high performance graphics+sound support

Although I would prefer OpenGL compatibility for cross-platform, DirectX was a very good idea, and made PC games in windows possible without writing special hardware-specific drivers to do it.

5.  The release of Windows XP - pure 32-bit and 64-bit support

When Windows XP released, it was for general consumption.  No longer did we have 16-bit "legacy" drivers slowing down the system.  Segment loads are 'expensive' (16 cpu cycles each time) and limit access to memory to 64k chunks.  By going pure 32-bit you could see performance improvements in many areas, from disk access on large files to video graphics.  And, of course, the beginnings of 64-bit computing.

OK these were things Microsoft "did right".  Maybe there are more.  These 5 can be backed up by MARKETING DATA, of course, seeing increased sales, relative popularity, and so forth.  You COULD add Windows 7 to the list, with the 'Aero' and widgets, but these were present in Vista also and Vista did POORLY in the market, but not because of any innovations like Aero and widgets.  It did poorly because it was HARDER TO USE and "finding familiar things" was DIFFICULT at best.

Anyway, here's my list.  You'll see that Windows 8.x and 10 are *DEMONSTRABLY* missing from this list.  I don't see them as things done RIGHT, and market data on 8.x confirms this.  If I am trying to accomplish something here, it is to REMIND MICROSOFT of the things they did RIGHT, and to POINT OUT what they are currently doing WRONG, and to ENCOURAGE them to go 'RIGHT' again.

jay freedman say:

When you create a custom style, or modify a built-in style, the bottom of the Modify Style dialog box has two option buttons: 

If the "Only in this document" option is selected, the style is stored in the current document (or, in the special case where a template has been opened for editing, the style is stored in that template). If the "New documents based on this template" option is selected, the style is stored in the template on which the current document is based -- your Normal.dotm template unless you chose a different template in the File > New dialog.

The styles will also be stored in any document based on that original template or copied from that document.

The details are explained in the section "Where do styles come from? How does Word decide what styles to put into a new document?" of https://shaunakelly.com/word/styles/stylesms.html.

Once you've located the template or document that contains the styles you want, you can copy the styles to a new template or document by using the Organizer as described at http://www.addbalance.com/usersguide/styles.htm#Organizer.

Will a Shared file still be available to the Sharee if the - Microsoft Community

blakepeters say:

Will a Shared file still be available to the Sharee if the Sharer's account is deleted?

I am using the business version of O365 and I wanted to know if I share a document with someone via OneDrive and I leave the company (i.e., account will be deleted) will the files I shared with my colleagues still be available to them or will the files be gone when my account is deleted?
tina chen msft say:

Hi Blake,

It depends on OneDrive for Business retention and deletion policy set by your SharePoint admin or OneDrive admin.

After an account is deleted, the account deletion is synchronized to SharePoint Online. The OneDrive Clean Up Job runs, and the user profile is marked for deletion. The profile will be preserved in the database in a deleted state. The default retention period is 30 days. During the 30 days, the shared files can still be accessed by your colleagues. After 30 days, the files can’t be accessed any more.

However, SharePoint admin can change the retention time to a maximum time: 10 years. It means even if your account is deleted, your shared files can still be accessed during the retention time you set. Here are two methods:

1. SharePoint admin can refer to OneDrive for Business retention and deletion to use PowerShell to change the retention time of the OneDrive for Business site.

2. OneDrive admin can go to OneDrive admin center -> storage -> days to retain files in OneDrive after a user account is marked for deletion.

Regards,

Tina

stuck in word say:

Hyperlinks not showing in blue or underlined

Hello. Can anyone help with making hyperlinks display blue, underlined in Word? It doesn't show the field codes, and when I hover over it i see the link but I need to have it in blue and underlined to stand out. Thanks in advance!
jay freedman say:
If there's a hyperlink, it should be formatted with the character style named Hyperlink (or, if the link has been used, the style named FollowedHyperlink, which is usually purple).

Put the cursor on the hyperlink, and open the Styles pane (click the little arrow in the bottom right corner of the Styles group on the Home ribbon, or press Ctrl+Alt+Shift+S). Does it say that the Hyperlink style is applied to the text there? If not, then select the whole hyperlink in the text and apply the Hyperlink style (you might have to click the Options link at the bottom of the Styles pane and choose to show all styles).

If the Hyperlink style is applied already, hover the mouse over the Hyperlink style's name there and look at the definition -- the default says "Font: Underline, Font color: Hyperlink". The actual color used by "Font color: Hyperlink" is defined by the current theme selected on the Page Layout ribbon. If the style definition is correct, you may need to choose a built-in theme or fix the color selections if it's a custom theme.
