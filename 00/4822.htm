To fetch data from another workbook where any component of the path is variable would involve the usage of the INDIRECT() function.  The problem with the INDIRECT() function is that it will not work if the source workbook is closed.  As one opens the source workbook, the value in the destination workbook will work fine.  Since it is not practical to have the source workbook open at all times that the destination workbook is also open, the INDIRECT() function solution will not be a viable one.

Here's my take:

1. Type the path (C/Fiscal Year/Month/Report.xlsx) of the source workbook in a certain cell of the destination workbook

2. In the destination workbook, use the INDIRECT.EXT() function to fetch data from a closed workbook.  This function will work even if the source workbook is closed.  The INDIRECT.EXT() function is not a built in function - it part of the Morefunc add-in (third party add-in) which you can download from this link of my website.

3. As you change the path in step 1 above, the formula will fetch data from the workbook at that location even if the source workbook is closed.

If there is only one row in the column to the left, then .End(xlDown) will go to the bottom of the sheet.  Do you want it to fill to the bottom of the sheet?  The code below will NOT fill down if there is only one row of data in the column to the left of the selection - and that data row is the same as the row number of the top of the selection:

Sub Code1()
    Dim LastRow As Long
    If Selection.Cells(1, 1).Column > 1 Then
        LastRow = Cells(Selection.Row, Selection.Cells(1, 1).Column - 1).End(xlDown).Row
        If LastRow <> Cells.Rows.Count Then
            Selection.Resize(LastRow - ActiveCell.Row + 1).FillDown
        End If
    End If
End Sub

Likewise, this will NOT fill down if there is only one row of data in the column to the right of the selection - and that data row is the same as the row number of top of the selection:

Sub Code2()
    Dim LastRow As Long
    Dim ColToRight As Long

    LastRow = Cells(Selection.Row, ColToRight).End(xlDown).Row

Templates vs. Themes? - Microsoft Community

design123 say:

Templates vs. Themes?

I've been using PowerPoint for Mac for many years and I still don't know the precise difference between Templates and Themes. Can someone please give me an answer with specific clarity regarding the differences?

Thanks!

bob jones aka: cybertaz say:
Don't feel bad... there's still a lot of cloud enveloping the relatively new concept of Themes as well as some 'missing pieces' to the puzzle:-) I'll try to keep it as short -- but as informative -- as I can... And keep in mind that we aren't even getting into customizing Themes.

The most simple distinction is:
  • A template is a defining file [cookie cutter] on which new working files are based
  • A Theme is a collection of coordinated formatting specifications

Another fundamental difference is that a template is specific to a particular program [i.e., you can't create a PPT presentation using a Word template or an Excel file from a PPT template] whereas Themes are universally available to all apps in the Office suite. That's the core purpose of Themes -- to provide identical appearance (heading & body Fonts, Color scheme & graphic Effects) in any Office document regardless of what type of document it is or which program created it.

Themes are a bit easier to understand in terms of Word documents & Excel spreadsheets, but PPT throws a monkey wrench into the works because it also includes the graphic design elements for the underlying master slides of a presentation. The best correlation I've found is to think of a Theme in PPT as the equivalent of what were formerly called Design Templates. You decide what graphic design & layouts you want using the Master Slides & select a Theme that specifies the default Fonts, default color scheme & default graphic effects for Charts, SmartArt, WordArt, Shapes, etc. [Not to confuse things more, but Effects currently are one of the missing pieces in the Mac version.]

That's all saved in your template (or just specified in a presentation), but the appearance of the presentation will coordinate with any Word or Excel documents that use the same Theme. If you switch to a different Theme in the document, the fonts, colors & effects automatically update throughout based on what the newly applied Theme calls for. It should be noted that the updating occurs only for those elements which have not been directly formatted to deviate from what the previously applied Theme specified. [E.g., If you directly change the font formatting of the title on one slide it severs the connection between that title & the Theme, so if you switch Themes all other titles will update but that one won't.]

The problem is that as I stated there isn't a true whitelist, so even if Microsoft chooses to respond by adjusting their current detection(s) affecting your specific file to stop the current false positive, this typically doesn't stop the problem completely.  That's because the newer versions of Windows Defender SmartScreen operate using cloud based detections rather than simply signatures, so it's relatively common for these to mistakenly trigger when unsigned files are involved.

Microsoft knows this is an issue for smaller developers, so they've done what they can to improve the situation for those in a small business which create their own applications for internal use.  However, the reality is that just like the valid small developer, the malware creator typically doesn't wish to spend money for a certificate, especially since they might potentially be identified by acquiring it and will assuredly lose that money once their application is identified as malware.

This situation creates the problem that the smaller developer must make the decision how widely they will distribute their program, since it's no longer easily possible to do this without purchasing a certificate.  Unfortunately, that's a necessary evil of the current situation, where malware developers commonly use the nature of "free" software to distribute their malicious apps.

So rather than drop you directly into the submission page you found, I'd direct you instead to the following page which includes information about certificates and a FAQ explaining that no file whitelist exists.

Resources for software developers
