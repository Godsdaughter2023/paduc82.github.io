"Valid" to us humans may mean something different than "valid" to PowerPoint. 

I'd try installing a known-valid-to-POWERPOINT driver as a test.  Instructions here:

 

How to install a TEST printer driver
http://www.pptfaq.com/FAQ00952_How_to_install_a_TEST_printer_driver.htm

 

But first, verify that the fonts used in the presentation are actually present on the 2003 PC that's having trouble displaying the presentation.

Open the presentation on the 2003 PC, choose Format | Replace Fonts

Look at each font in the upper of the two dropdown list boxes.

If any fonts have a ? icon next to them, it means that the presentation needs them but the font's not installed on the computer.  In that case, PPT is substituting a different font which might have different character widths, so linebreaks change.

Another possibility:  if someone has applied different tab/ruler settings to multiple paragraphs in the text boxes, that won't work in PPT 2003, where each text box has only one set of tabs that apply to all paragraphs within the text box.

Steps are common for Windows Vista and Windows 7.

Internet Explorer 32 bit

Internet Explorer (64-bit)

If you want to change the default internet explorer from 32 bit to 64 bit you may perform the following steps.
You can change the program windows uses by default to open with different file extensions.

a. Click on Start and go to Default Programs.
b. Click on Associate a file type or protocol always open in a specific program.
c. Browse to all the extensions where internet explorer is set as default and then click onchange.
d. Now Click on browse and then go to location C:\Program Files\Internet Explorer\iexplore.exeand select it.
e. Click Ok and then Close.
f. Click Save.
