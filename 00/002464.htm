Each patient has a unique record number, but each patient makes multiple visits.  At every visit, there are lab tests.  At some visits there are special lab tests.  Table 1 is a list of patients (no duplicates).  Table 2 contains visits with the associated lab tests.  The tables are linked by the patient record number, with One-to-Many (Patients are 'one', lab visits are 'many').

Query-A and Report-A list every test done, (One page for each patient) in order of the visits.  (Too busy for us to add notes to)

First attempt was to create a query with these columns:  Patient number, Name, Visit Date, Special Test #1.  The "SORT" row for Visit Date is set for 'Group By', and the "CRITERIA" for "Special Test #1" column is '>0'.  But it gives all of the results for each patient's "Special Test #1" (not just the most recent result).  Once we have a query that lists just the single latest result for Special Test #1 for every patient, we will link to Report-B, bringing down the encounter date and the test result.  Will have to make similar queries for each of the half-dozen Special Tests.

djsteele, mvp say:

Unless you can guarantee that there's an appropriate ORDER BY clause, DLast is essentially meaningless.

You haven't given the actual table layouts, so take the following as a template, not actual SQL!

The following should give you the latest special test date:

SELECT PatientID, Max([Visit Date]) AS LastTest
FROM Table2
WHERE Len([Special Test #1]) > 0
GROUP BY PatientID

You can then join that to the table itself:

SELECT Table2.PatientID, Table2.Name, Table2.[Visit Date], Table2,[Special Test #1]
FROM Table2
INNER JOIN
(SELECT PatientID, Max([Visit Date]) AS LastTest
FROM Table2
WHERE Len([Special Test #1]) > 0
GROUP BY PatientID) AS Subq
ON Table2.PatientID = Subq.PatientID
AND Table2.[Visit Date] = Subq.LastTest
