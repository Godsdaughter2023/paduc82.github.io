Movie Maker was retired last year (January 2017) and was last updated in 2012. I don't know of any places you can have the file fixed for you online.

The most common reason these days that a file is considered corrupted is that one or more VideoClips in your project has an outTime that is 0 or otherwise smaller than the inTime.

Make a copy of your .wlmp file, open the copy in NotePad and search for all occurrences of outTime. Make sure that the value is bigger than the inTime in all cases.

Once you fix them all and save the .wlmp, the copy should now open.

Remember: Always work in a copy in case you need to resort back to the original.

cmdrkeene say:

It actually is a scheduled job.  I'm not on my Windows computer at the moment, but if you look through Task Scheduler, there is an Indexing or Windows Search index task that is set to run with "idle time" as a trigger.  Basically any time you're not using your computer for x minutes, the task is triggered and begins its duties.  If you touch the computer, the task ends instantly.  The indexer also uses NTFS's journally to determine what needs indexed, by basically querying and finding what indexable files have been modified since the last job.

After the first time index build, it's all incremental, so it should quiet down.

It may be possible to manually modify that scheduled job yourself, though.  I have not tried but it seems logical.
