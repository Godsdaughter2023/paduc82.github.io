pbezucha say:

By better reading your request, I see now your problem opposite: keep all the data in chart, but reduced in the table, OK? In that case, the solution is to hide each but n-th row on the datasheet and ensure (in chart Options) that your chart will depict all the data.

The macro below does the hiding.

PB

 

 

 

Option Explicit

 

Sub HideButNth()

Dim R As Long, RF As Long, RL As Long

Static N As Long

RF = Selection.Row

RL = Selection.End(xlDown).Row

If N = 0 Then N = 2

N = Application.InputBox("Select the first data cell!!!" _

  & vbCrLf & "and input N for unhidden each Nth", "Hiding data", N, , , , , vbLong)

Application.ScreenUpdating = False

Range(Rows(RF), Rows(RL)).Hidden = False

If N <= 1 Then Exit Sub

R = RL

Do

  If R Mod N <> 0 Then Rows(R).Hidden = True

  R = R - 1

Loop Until R <= RF

Application.ScreenUpdating = True

End Sub

Unruly list paragraph - Microsoft Community

denniswilson_1273 say:

Unruly list paragraph

When I modify List Paragraph to change indents and tabs, then do Reset to 1 (or Reset to 'a'), the style reverts to the old indent/tab settings. Whats the solution?
stefan blom say:

The List Paragraph style, or any paragraph style for that matter, can't be used to control the indentation of a numbered list. You will have to modify the list format itself. Once you have set up the indentation correctly, it won't change as you restart the numbering.

For a single-level list, you can click the first list symbol, so that Word selects all list items; then right-click and choose Adjust List Indents from the context menu.

"Number position" defines the position of the list number. "Add tab stop at" determines where the text of the first line (following the list number) starts. "Text indent" is the location for second and subsequent text lines of the numbered paragraph. All measurements are from the left margin, which means that if you want a hanging indent, you should set "Add tab stop" equal to "Text indent" (and both values must be larger than "Number position").

Make the desired changes for indentation in the dialog box and click OK when you are done.

If you are using numbering with a particular paragraph style, you can update that style to match the selection. That way, you can easily reuse the list format.

For a multilevel list, you can set up indents in the Multilevel List dialog box. Click in the first list item and click Home | Multilevel List | Define New Multilevel List. See also http://www.shaunakelly.com/word/numbering/numbering20072010.html (which applies even if you are not setting up numbered headings).

It sounds like you're mixing levels. An Access Database file (.mdb or .accdb extension, for example) is not a table; it's a container which may contain multiple Tables, Forms, Queries and other objects.

Excel 2013 PowerPivot wouldn't come up -- here's what solved - Microsoft Community

w-phs say:

Excel 2013 PowerPivot wouldn't come up -- here's what solved it for me, maybe this will help solve the problem for others

I installed Office 2013 (typical install - no customization and no errors during it) earlier this week and could not get PowerPivot to come up in Excel. After Office had been installed, I went to the Add-Ins under the Excel options on the start page and clicked on the PowerPivot COM add-in, then selected OK. As it said it would come up after I restarted Excel I did just that, only to see that the PowerPivot tab wasn't theree. Next, I went to the Developer tab and selected it from the COM add-ins drop down. I saved the worksheet (even though I knew it wouldn't make a difference but I just had to do it), exited Excel and restarted it. Again, no PowerPivot, and when I looked at the COM add-ins dropdown on the Developer tab, it was no longer selected.  I checked an Office 2013 Excel help article, which included the steps I had just tried and it also suggested that I make sure the add-in wasn't disabled; it wasn't. I then spent 2 days trying every suggestion that I could find on the internet, I repaired Office 2013, I uninstalled and reinstalled it, and I also messed with every Trust Center setting combination possible - all without success. So this morning I decided to go back to basics, which is when I finally came up with what needed to be done to get this to work - and it's something that shouldn't have even been necessary. You'll understand why as you keep reading.   

 

I thought the typical, non customized install would take care of what it needed for the COM add-ins to work, but that was incorrect apparently. For some incomprehensible reason the path to the PowerPivot add-in wasn't considered a "trusted location" and had to be manually added as one in Excel's Trust Center. 

 

I added this path on that tab:   C:\Program Files\Microsoft Office\Officel5\ADDINS\PowerPivot Excel Add-in\PowerPivotExcelClientAddIn.dlI 

 

To do this, go to Excel Options -> Trust Center -> Trust Center Settings and select the Trusted Locations tab, which is where you add it. While on this tab you'll probably notice a couple of things:

  1. There are already several of the Excel locations for it's extension in here as a default. (Not sure why this one wasn't included as a default too.)
  2. You may notice on the bottom left of this tab there is an option to select "Allow Trusted Locations on my Network" but it says "not recommended". (So does this mean we have to manually add all of the paths to the Trusted Center that weren't added during the install of the Microsoft product?)

After I added the path, I wanted to make sure all possible "trust issues" would be addressed, so I checked the Trusted Publishers tab to see if there was a row with the  Microsoft Corporation (found in the "Issued to" column) and Microsoft Code Signing PCA (found in the "Issued By" column). It was - luckily during my numerous attempts to make this work it got added when I had selected an option presented in the trust center to make the publisher of this add-in a trusted one.     

 

After I did these things, voila, I now have PowerPivot up and running. But man, this took far too much effort and was very frustrating, which is why I've added this post - hopefully this helps reduce the amount of frustration for someone else with this same issue.

 

 

 

Note to Microsoft:  Seriously? This is your software and add-in, so why in the world isn't the add-in path trusted? And why isn't Microsoft already included in Trusted Publishers? Please take care of these kinds of things before releasing software to help us use our time more efficiently. It will be greatly appreciated!

Hi Nmber_4,

For Match function, if match_type is 0 and before the character.

Since you use asterisks (*)in Column A(*15) and Column B(*015), an asterisk matches any sequence of characters. If we find column A(*15) in Column B(*015), it ignores the "0" before 015(*1=*01) and the columns will match, the value return to 1. If we find Column B(*015) in Column A(*15), the "0" can't be matched in Column A(*15), it will return #N/A.
