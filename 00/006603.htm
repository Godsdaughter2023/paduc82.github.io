reinob say:

Do you have a single partition in your PC? :)

 

At least in the Unix/Linux world it is common to have *at least* two partitions. One for the system (/), and one for home (/home). That would be the two 4GB partititions on your N9.

 

Because most Windows PC cannot even see the ext2/3/4 partitions both the N900 and N9 include a big fat FAT32 partition to put your "data". This partition is then visible from your PC, so you can use to share data.

 

The system (/) and home (/home) partitions cannot be FAT32 because then not only a lot of space would be wasted, but there would be no support for permissions (and hence no security) and the whole system would be much slower. Plus you would likely end up with a corrupted system if your phone reboots or is turned off abruptly.

 

It might make sense to combine the / and /home partitions (e.g. both ext4), but then you would lose the possibility of easily making a backup (to backup your settings, just backup the whole /home partition, etc.)

 

In short: it makes sense the way it is.

 

In principle you could make the /home partition smaller (e.g. 2GB, thus leaving you with 10GB for MyDocs), but for that you'd need to re-flash it with a patched firmware. This is easily done on the N900, but (still?) not clear if/how it works with the N9.

Preservation Hold library - Microsoft Community

brad abbott say:

Preservation Hold library

I am working with retention policies applied via labels in SharePoint online. According to the info on the site once a retention policy is applied, if a document is deleted by a user prior to the set retention period, a copy will be created in a Preservation Hold library. I however cannot find any information that tells me how to find and view this  Preservation Hold library and its contents. Can anyone help?
sheen valentine ali msft say:

Hi Brad,

 

After checking from the backend, I can see that the issue is resolved. I would like to post a summary of it for other members to easily see the information.

 

Issue description

After applying a retention policy to a site, a Preservation Hold Library should be created. In this case, we couldn't find the library even after retention policy is applied.

 

Root Cause

This is by design.

 

Resolution

When you turn on the hold, the Preservation Hold Library will not be created, it will be created only when the first items need to be copied to the preservation library. So it is expected that after turning on the hold, you do not see preservation library , only when you delete the first items.

 

When updating an item that exists before you turn on hold , it will create the preservation hold library as well, so the library is created only when needed.

 

Regards,

Sheen
