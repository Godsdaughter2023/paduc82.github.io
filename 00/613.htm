If it be convenient, please contact admin to check if your mailbox also enabled litigation hold, temporarily remove all holds(including litigation hold and in-place hold) from your account, then clear Recoverable items folders, and then you can try to delete items to see if it works. If there’s any third-party add-ins in Outlook and Outlook Web App, temporarily disable them before test.

 

Besides, from the screenshot, we can see that in Outlook the recoverable items folder is empty, but it also have some subfolders that are unseen for user. To further confirm all folders aren’t full, you may contact admin and let his check recoverable items folder quota and current folder size:

 

1. Connect to Exchange Online PowerShell.

2. Run the following command to get recoverable folder quota:

        Get-Mailbox -Identity “User Name” | FL recoverableitems*

3. Run the following command to get recoverable folder size:

        Get-MailboxFolderStatistics “User Name” -FolderScope RecoverableItems | FL Name,FolderAndSubfolderSize,ItemsInFolderAndSubfolders

4. Run the following command to check mailbox hold status:

       Get-Mailbox “User Name” |FL Name, LitigationHoldEnabled ,InPlaceHolds

 

If the folder size displayed in PowerShell output shows it’s reaching the quota, the suggested way is to let admin temporarily remove all holds applied on your mailbox (since holds will prevent user from deleting items in mailbox)->totally cleanup recoverable items folder->delete items in Outlook Web App.

 

If admin has tried to temporarily remove holds, clear recoverable items folders, but issue still exists. Then it may be related with server side, may I confirm if other user in organization have the same issue?
