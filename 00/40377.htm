I've gone to HP Support and downloaded the Hardware Wizard, the Windows 8 Print Fixer and the latest Driver versions for the printer for Windows 8, but no difference.

 

My other Windows XP desktop and Windows Vista laptop both still print fine to the same printer.

 

I used the XP desktop to share it's print queue, and configured Windows 8 to print to that queue, and you can see it takes ages for it to spool to the XP queue, before the XP then prints it fine.

So overall same amount of time to print, the delay is just in the transfer from Windows 8 to Windows XP queue.

 

I hooked up an old Canon printer to the network, and same issue for Windows 8, it prints one line at a time really slowly.

 

Windows 8 Help Desk say contact HP (even though I told them same issue with Canon).

 

When I turn on the Task Manager Monitor you can see the Network traffic go up and spike/fall for the amount of time the Windows 8 is trying to communicate to the HP printer over the network, or to the XP queue over the network.

 

All other network activities are fine, browsing the web, downloading all the driver updates, copying my data files from the Vista laptop to Windows 8 etc.. all were super fast and no issues.

 

I then got a USB cable and connected the printer directly to the Windows 8 and it prints fine direct.

 

So looks like there's a problem somewhere between the Print spool or drivers on Windows 8 and the network stack or communicating externally.

 

Any ideas?

Thanks,

Dave,

skypolodb say:

New twist to the plot.


So I watched the Print queue and the print job was a PDF file 3.5MB file.  And then watching my Firewall and Task Manager Performance Ethernet tab it showed it was transmitting at 200Kbps - so approx 28,000Kb so it takes about 140 seconds... or a little over 2 mins...


But like I said earlier all Internet browsing and downloading drivers and copying files from my Laptop are super fast.


So then I took a 3.5MB file from the Windows 8 and tried to copy it back across the network to my Laptop... and low and behold it took almost 3 mins... processing under 200Kbps.


So this is not a printing issue... it's an issue with Data out my Network Interface... Data In is fine.


This may still be a Windows 8 issue, but could be a Network Card, Cable or Hub issue... so time to troubleshoot each of them... 


Then I'll get back to you with my findings.

skypolodb say: Problem Solved.

This was an Avast Internet Security issue.

Once I noticed it wasn't just a sending print jobs issue, and was an outbound network transfer issue, I looked into Device Manager at my Network Settings.  There were over 60 "Avast NDIS Miniport" drivers attempting to bind, all with Failures.

I uninstalled Avast Internet Security.

Then after a reboot I had to manually Uninstall each of the 60 miniport drivers.

Then after another reboot, problem solved.  Printing fine.  Outbound data transfer fine.  And now my Wireless Interface can also find and connect to wireless networks (which it couldn't do before either).

I'll stick to Windows Defender for now.

malkeleah say: You don't know for sure that this isn't a monitor issue unless you have connected the computer to a different monitor and the same problem occurs. Random failures are most often caused by failing/bad hardware. The usual troubleshooting steps for this kind of problem are:

1. Connect the computer to a different monitor. If the problem doesn't exist, it's time to replace the original monitor.

2. If the problem continues when the computer is connected to a different, known-working monitor then the culprit is your video card. Uninstall it and install a known-working video card. If that solves the problem, discard the original video card.

Before you go through the steps above, make sure to first check the cable going from the computer to the monitor. Make sure the connection is snug without being overly tightened and that there are no bent pins on the cable. Obviously you need to check the pins when both the computer and the monitor are turned OFF. You can also try swapping out the monitor cable with a known-working one.

Testing hardware failures often involves swapping out suspected parts with known-good parts. If you can't do the testing yourself and/or are uncomfortable opening your computer, take the machine to a professional computer repair shop (not your local equivalent of BigComputerStore/GeekSquad). If possible, have all your data backed up before you take the machine into a shop.
