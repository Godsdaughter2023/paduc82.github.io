a. user say:

Hi ashleywb. You are correct that the sync limit for OneDrive for Business is 20,000 items and 5,000 items for team site libraries. However, you can store hundreds of thousands if not millions of items in a single document library. The limits for SharePoint 2013 are in the millions of items. So for all practical purposes the amount of files and folders you can store in Office 365 is virtually unlimited.

The problem is if you breach the sync limit the sync tool will not work. And if you breach the list view threshold (another limit) then some functions of libraries are disabled such as breaking permissions inheritance.

If you don't use the sync tool then the sync limit is not so much of an issue.

Myles Jeffery | Thinkscape Limited

Try mapping OneDrive for Business and team site libraries to network drives with Zee Drive, its easy!

iris he msft say:

Hi ashleywb,

There is no strict limit for how many items can be stored in OneDrive for Business library and SharePoint Online site libraries. The number of items depends on the storage of the site and the size of the files you stored.

The storage of OneDrive for library is 1TB. The storage for SharePoint Online sites depends on the tenant storage you have. It is 10 GB + 500 MB per subscribed user + additional storage purchased. See the Storage base per tenant section in this article to learn more: SharePoint Online and OneDrive for Business: software boundaries and limits.

The 5,000 threshold limit is for a view. That means, if we uploaded 10,000 items to a OneDrive for Business library, only 5,000 items can be showed up in a view, remained 5,000 items are hidden. To let all these 10,000 items show up in a library, we need to create a new view to filter 5,000 out. This is complicated. So, for this scenario, we normally suggest customers store items in several libraries.

For 20,000 items limit, you are correct, it is only for sync, it is not related to how many items you can uploaded to the library.

Please let me know if there is anything unclear.

Thanks,
Iris

Sorry, you can't "uninstall" or disable that message. Your choices are to assign the Alt-key shortcut to something, or press Esc when you make a mistake and trigger the message.

As your research may have revealed, autotext in Word 2007 and 2010 is now part of the Building Blocks feature -- it's just one of many "galleries". The default set of building blocks (which doesn't include any autotext entries) is in the file
    %appdata%\Microsoft\Document Building Blocks\1033\14\Built-In Building Blocks.dotx
(the '1033' is for US English, and you may have other folders for other languages; %appdata% is an environment variable that always points to your C:\Users\<your name>\AppData\Roaming folder).

However, Word will automatically load building blocks stored in any .dotx file that appears in the folder
    %appdata%\Microsoft\Document Building Blocks

So the equivalent process for creating your template is:

  • Open a blank document and save it as a "Word Template (*.dotx)" file in %appdata%\Microsoft\Document Building Blocks.
  • Create at least one autotext entry in the body of the template itself. (This is necessary to get the template to appear in the list of places to save entries. After the first entry is there, you could close the template an create entries from text in any document.) While it's selected, press Alt+F3. This opens the Create New Building Block dialog.
  • Enter the name of the entry. Leave the gallery set to AutoText. Choose or create a category (more about this below). Make sure the Save In box contains the name of the template. Click OK.
  • Repeat for other entries. As I said above, you can create all the entries from text in the body of the template (and delete that text when you're done with it) or you can open another document and use its text to create entries; as long as the Save In box of the dialog is set to the template, that's where the entries will be stored.


The categories within the AutoText gallery replace the former reliance on styles to separate the entries. Of course, there is no longer any Insert > AutoText menu; the replacement is the Insert > Quick Parts > AutoText gallery on the ribbon. The category names serve as separators between groups of entries. You can create as many categories as you like. Hint: The categories appear in alphabetic order, and ones that start with an underscore character get sorted to the top.

If users would rather avoid the mouse, the keyboard shortcut of typing the unique first part (at least 4 characters) of an entry's name and pressing F3 works the same as in previous versions. There has been some back-and-forth about the tooltips that should appear when an entry's name is recognized; Word 2007 doesn't show them at all (except for dates) while Word 2010 recognizes at least those stored in Normal.dotm and maybe others.

To distribute the template to staff, you should get it into the %appdata%\Microsoft\Document Building Blocks folder on each computer. You can do this with a network login script that calls xcopy.exe -- talk to your IT staff about the information in http://www.word.mvps.org/FAQs/MacrosVBA/DistributeMacros.htm. It's possible to get Word to load the entries from a template on a server, but you'd need to use the Document Template button on the Developer tab (the equivalent of Tools > Templates & Add-Ins) to attach the template on every computer. In addition, when users attach directly to the network-stored template, you can't open it to edit it or replace it because it's "in use"; local copying avoids that problem.
