    • From the Start Menu go to the Search Bar and type devmgmt.msc, ifDevice Manager opens then skip to step 2.  
    • From the Start Menu, on the right hand side will either be Administrative Tools orComputer (Computer should always be there). From Administrative Tools, if listed, scroll down toComputer Management and open.     • Finally, you can use Control Panel. Go to the Start Menu, and on the right hand side should be a listing forControl Panel, open, then go to Hardware and Sound. At the top ofHardware and Sound you should see Device and Printers and four blue links under that heading, from here you'll want to selectDevice Manager, if Device Manager opens then skip tostep 2.

Go ahead and expand this entry, then look for a subtree named PocketPC USB Sync. If you see a PocketPC USB Sync entry, thenRight Click > Select Disable. Once the PocketPC USB Sync entry has been disabled thenRight Click > Enable.

 

WMDC should now automatically load and sync with your device. You should also be able to use Windows Explore to browse the device. Please let me know if this workaround does not solve your issue.

louloizou say:

Trial version import pst files

I have replaced the hardrive and re-installed office Home and Bus (3 User)  I enter the activation key and it still says I have trial version with 28days to go? I can't open the pst folder does it need to be full version to open folder or import?

a. user say:

You don't import pst files.

(Courtesy of Russ Valentine [MVP-Outlook])

Importing an entire PST may well corrupt your profile and may create a ghost
PST that you can't close. Importing PST's will lose:
1.  Custom Forms
2.  Custom Views
3.  Connections between contacts and activities
4.  Received dates on mail
5.  Birthdays and anniversaries in calendar
6.  Journal connections
7.  Distribution Lists

Opening a PST file will preserve all of these. That is why we do not advise
people to import a native file into Outlook.

 

Copy the pst file(s) to your Documents folder (do NOT overwrite any existing pst files, or paste them in the default Outlook pst location).

Then in Outlook do File-Open-Open Outlook Data File and navigate to where you put them.

  7. Click the Previous button to go back to "Header -Section2-".

jay freedman say:
> Any thoughts on how the page numbers (including the new 3a, 3b, 3c can be picked up in a TOC?

They can't be picked up automatically. The TOC will show the letters a, b, or c, but not the 3 that you typed.

If you create the TOC as the last thing before distributing the document, you can put the cursor in the TOC and press Ctrl+Shift+F9 to change it from a field to plain, non-updating text. Then you can type the 3 where it's needed. If the document is updated again afterward, you'll have to rebuild the TOC from scratch.

How to divide 2 document property quick parts in one document by - Microsoft Community

jmac7993 say:

How to divide 2 document property quick parts in one document by section

I have 2 status metadata quick parts in a document that need to contain different values. However, when I change one, it changes the other to the same value. How can I split these up so that I can put different values into the same type of quick part on within the same document?
jay freedman say:

That is not possible. When you use one of the "document property" quick parts, you're setting the value in the XML hidden within the document file. All other instances of the same quick part are also attached to the same XML value. Changing any of the instances updates the single XML value, which changes the display in all the other instances of the same quick part.

The solution is to repurpose one or more of the unused quick parts, each of which has its own XML value. You can change the titles of the inserted content controls that display the values, so they all have the same title -- that won't change which XML value goes with each control.

Another alternative is to define custom XML values, and insert content controls attached to them rather than to the built-in XML.

To make this job easier, get the add-in from http://gregmaxey.mvps.org/word_tip_pages/cc_var_bm_doc_prop_tools_addin.html.
