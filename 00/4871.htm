stephen boots say:

If the PC is part of a domain served by WSUS, you can get MSE updates from WSUS.

If the PC is simply not connected to the Internet and has only local access, you will need to download the updates manually (or create a scheduled task to do this) to a computer that has access and then you'd need to create a scheduled task to retrieve the downloaded update file and execute it to install the updates.

Note that the manual update download from the Portal is the full engine and definitions database and is over 60 megs in size.

You can manually download the (large file with engine and full definitions) latest definitions from the portal here: https://www.microsoft.com/security/portal/default.aspx

date format changes on mail merge

Can someone help me with this problem.  The date format changes from dd/mm/yy to mm/dd/yy when I mail merge between word and excel.  The date is set on my system for dd/mm/yy and the cells in excel are formatted as a date dd/mm/yy

hansv mvp say:

Right-click the date merge field (in the Word document with the merge fields).

Select Toggle Field Codes from the context menu.

You should see something like

{ MERGEFIELD MyDate \* MERGEFORMAT }

where MyDate is the name of the merge field.

Change it to

{ MERGEFIELD MyDate \@ "dd/MM/yy" }

Press F9 to hide the field code and to update the result.
