loandtee say:

motherboard for emachine et1641-02w

If you have an equipment failure do you have to purchase the rights to a new operating system or can you use the original one on say if a motherboard is replaced??   Steve
john_barnett say:

Hello loandtee,

When the operating system is pre-installed by the manufacturer the operating system is tied to that machine. If the motherboard dies then you normally have to get a replacement from the PC manufacturer (it has to be the same type/model number etc) for the opewrating system to work.

If you purchase a new copy of the operating system then a 'retail' copy is preferable (obviously more expensive too) rather than an OEM copy (OEM copy is what the PC manufacturer would use.) A retail copy allows you to move the operating system (providing the original machine has died) to a new machine. You are therefore not tied to the same machine. For example, although I use Windows 7 I also install (on a virtual machine) windows vista for troubleshooting purposes. Since Vista was released (my copy is retail) I have moved it three times to different new PC's. I couldn't do this with an OEM copy. Also beacuse |I was selling the old machines I re-installed the original OEM copies onto the machines I was selling. This is so I do not infringe the licencing agreement.

 

This forum post is my own opinion and does not necessarily reflect the opinion or view of Microsoft, its employees, or other MVPs.

Hi,

As well as John_Barnett's very good Information, here is what Microsoft has to say about replacing Motherboards in preinstalled OEM Computers (eMachines, Gateway, Dell, etc):

 

http://oem.microsoft.com/script/contentpage.aspx?pageid=552846#faq1

Q. Can a PC with an OEM Windows operating system have its motherboard upgraded and keep the same license? What if it was replaced because it was defective?

A. Generally, an end user can upgrade or replace all of the hardware components on a computer—except the motherboard—and still retain the license for the original Microsoft OEM operating system software. If the motherboard is upgraded or replaced for reasons other than a defect, then a new computer has been created. Microsoft OEM operating system software cannot be transferred to the new computer, and the license of new operating system software is required. If the motherboard is replaced because it is defective, you do not need to acquire a new operating system license for the PC as long as the replacement motherboard is the same make/model or the same manufacturer's replacement/equivalent, as defined by the manufacturer's warranty.

 

Yes; if you do not follow the advice given by Microsoft, you will void your License.

jesebel ova say:

Hi Larry,

This issue may occur if the service is started by the Local System account instead of by the Local Service account (NT AUTHORITY\LocalService). The Local System account may have insufficient permissions to start the service. For the Windows Audio and Windows Audio Endpoint Builder services, follow the steps below:

  1. Set the service and any dependent services to run under the NT AUTHORITY\LocalService account.
  2. Click Start, type "Services.msc” in the Search programs and files box, and then press ENTER.
  3. Locate and double click the Windows Audio service.
  4. Click the Log On tab.
  5. In the This account text box, type “NT AUTHORITY\LocalService”.
  6. Set both Password fields blank.
  7. Click Apply and then OK.
  8. Repeat these steps for the Windows Audio Endpoint Builder service.
  9. Restart both services.

Let us know how it turns out.
