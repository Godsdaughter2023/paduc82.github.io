crutcher say:

I just ran into this problem this weekend and the post from BossDweeb is spot on.  The specific folder locations are:

c:\windows\system32

c:\windows\sysWOW64

In my case I had two games that weren't working.  Astroneer and Rocket League.  This was significant because it turns out Astroneer is a 64bit game and Rocket League is 32 bit.

Here's what I did step by step:

In steam, right click on the game and select Properties

Click the "Local Files" tab

Click "Brose Local Files..."

This will pull up a windows file explorer window.

The game should have a "binaries" folder.   Click (or double click) that folder.

There should be a Win32 or Win64 folder.  Go into that folder.

You should be able to find the game's EXE file (e.g. RocketLeague.exe)

Launch it.  If you get an error box saying "*** is either not designed to run on windows or it contains an error***", look for the file referenced in the c:\windows\system32 and c:\windows\syswow64 folder.

I used a second windows file explorer window to find the file(s).

Keep doing this until you don't see that error any more.  You will see an error that the you just deleted DLL isn't available any more, but that's what you want to see.

Now run the direct X setup program.  If it tells you to extract files, you have to extract the files and then go into the folder you extracted them into and run the dx setup program from there.

In my case it was important to delete the files in both folders because they were broken in both places.  I fixed Rocket League and thought, "great, Astroneer should just work!" but then found it was not launching.  So I followed these steps and discovered Astroneer had a Win64 folder instead of Win32, so it's a 64bit program and I had to delete the files from System32 (which I didn't do when I fixed Rocket).
