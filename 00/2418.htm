If this is just a forward, then it is really easy to fake it as you can write in the quoted original message header since that is just text in the email.

It gets a little bit more difficult when the original message is forwarded as an msg-attachments, but even then it is still not impossible or all too hard to fake when you know what you are doing.

re. the ODBC thing, there are two different issues.

 1. If the Word document you are using has an existing connection to the database, that *might* explain why you are seeing an "ODBC" message. Since around Word 2002 or 2003, by default Word has made connections to Jet/ACE using OLE DB, and Microsoft's code for opening databases via ODBC basically gets confused by ",accdb" extensions. You should probably try to open your Word document and disconnect it from any existing data source.

 2. Neither ODBC nor OLE DB can see the stuff that is going on in the Access program. All they can see is the underlying data that is stored in the .mdb or .accdb file. They can't use the "!" notation to retrieve information from forms that happen to be open in Access.

There is a whole bunch of other stuff about ODBC v. OLEDB that I will not attempt to address in detail here.

If you have margin settings identified as Left and Right, then you do not have mirror margins. When you select "Mirror margins" in the "Multiple pages" dropdown on the Margins tab of Page Setup, the side margins change to Inside and Outside.
