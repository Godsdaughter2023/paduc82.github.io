brian tillman [outlook mvp 2007-2019] say:
If you use a Reply All instead of a Forward and, instead of sending the message, you deselect the encryption and save it, it will contain all of the original addressees so you will be able to communicate with them at a later point.  If there are embedded images, they'll be removed, but as Pollardie says, you have to compromise.  Forward/save will keep the original content if there are images.  Reply will retain the addressees but lose any attachments/embedded images.  You could also use Forward, copy the addressees to the proper fields from the original, then save the forward instead of sending to have both the addressees and any attachments/embedded images.

john - project say:

shellfish,

Project does not have a "move" function for columns, so nobody "turned it off" in the template you are using. Project fields (e.g. Task Name, Duration, Start, etc.) are added as columns in a view by inserting where desired. The Hide function simply hides that field from view since fields in Project can not be deleted.

Another method for setting up the view display is to edit the view table. That is done through the View/Data group/Tables function.
