Based on my research and test, it’s true that we cannot see the private items in shared mailboxes in the Outlook client even if the full access permission is granted. And the root cause is that the Delegate can see my private items option is not ticked for the shared mailbox. Generally, if that’s a regular user mailbox, we can set the option like mentioned in the article below:
Allow someone else to manage your mail and calendar

As it’s a shared mailbox, we cannot configure it separately in the mail profile. So you may check if the following workaround (which works fine from my side) works for you:
1. Convert the shared mailbox to a user mailbox.
2. Assign an Exchange online license and reset its password.
3. Configure it in the Outlook client and add the users who need to see private emails as delegates with Delegate can see my private items selected.
 
4. Remove the Exchange license and convert the user mailbox back to a shared mailbox.
5. Create a new profile and set up your account in it so the shared mailbox can get re-downloaded automatically again via Auto-mapping.
6. Check if it works.
NOTE: We recommend you back up all the data in the shared mailbox by exporting to a .pst file in the Outlook client in case of losing data during the process.

beth melton say:

Is there a reason your data isn't combined in the same worksheet? By separating them you tend to lose a lot of Excel's capabilites. For example, if they are all combined in a single worksheet you can use the AutoFilter feature to display all tasks for one specific project, those that are marked as high priority, etc. You could also use a PivotTable to further analyze your tasks, such as the count of high priority tasks for each project, etc.

If you do opt to combine them then I'd also utilize the Table feature (found on the Home tab, in the Styles group) which gives you even more functionality, such as a total row, ease in sorting, the AutoFilter automatically displays, along with various formatting options such as banded rows and/or columns.

If you want to try out what I'm referring to, first you'll need to have the project identified for each task. If you don't already have this, insert a new column, type the project description/number in the first cell of your task list and select the cell, place your mouse pointer in the lower right corner of the cell selector on the AutoFill handle and double-click to fill the data through the remaining tasks. If you're using a project number and it increments sequentally, click the AutoFill icon at the bottom of the filled range and select Copy. Then copy the data from a few worksheets onto a new worksheet.

Make sure the first row of your task data contains your field headings and that there are no empty rows or columns. Click somewhere in your data and on the Home tab, in the Styles group, click Format as Table and make any selection. After you confirm your table range you'll see a filter display at the top of each of your columns (fields). Click the filter for your Project field and make a selection. You'll find additional table options on the Table Tools tab, such as the ability to turn on a Total row so you can quickly show the count of your project tasks, options for formatting your table, etc.

~Beth Melton Microsoft Office MVP

andreas killer say:
Below is the customized macro, copy the code below the existing macro and save the file as XSLM. So you have an example if you run into a similar issue, or if you want to apply a different formatting.

Close the VBA editor
Select the chart
Press Alt-F8
Choose the macro
Click Run

Andreas. 

Sub Macro1_All_Series()
  ' Customized Macro1 to format all series
  ''
  'Remarks:
  'As you see I've removed the first two lines(where you have selected the chart and the series) and
  '  the last 2 lines with "ChDir" (I don't know where they came from, superfluous).
  '
  'The important (but simple) part is to replace "Selection" with a variable and
  '  surround the code with a FOR EACH loop.
  Dim S As Series
  For Each S In ActiveChart.SeriesCollection
    With S.Format.Line
      .Visible = msoTrue
      .ForeColor.ObjectThemeColor = msoThemeColorBackground1
      .ForeColor.TintAndShade = 0
      .ForeColor.Brightness = 0
    End With
    With S.Format.Line
      .Visible = msoTrue
      .Weight = 2.5
    End With
    With S.Format.Line
      .Visible = msoTrue
      .ForeColor.ObjectThemeColor = msoThemeColorBackground1
      .ForeColor.TintAndShade = 0
      .ForeColor.Brightness = 0
      .Transparency = 0
    End With
  Next
End Sub
