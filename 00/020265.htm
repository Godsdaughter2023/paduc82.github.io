albert d. kallal say:

I suggest making a “general” routine you call in the before update event.

So your code becomes:

Private Sub Form_BeforeUpdate(Cancel As Integer)

 

   Call Audit(me.Name, me)

 

End Sub

 

Now in a standard code module “outside” of the current form,  you have this (in standard code module, not a form).

 

 

Sub MyGAudit(strTName as string, f As Form)

 

 '/ INVOKE AUDIT LOG

     If f.NewRecord Then

         DoCmd.SetWarnings False

         Call Audit_Log_Entry(strTName, "APPEND", [Full Name], Null, Null, Null)

     Else

         DoCmd.SetWarnings True

         '/ loop for all AUDIT-tagged controls ---------------------------------

         DoCmd.SetWarnings False

         For Each Control In f.Controls

             If InStr(Control.Tag, "AUDIT") Then

                 If Nz(Control.Value) <> Nz(Control.OldValue) Then

                     FIELD_NAME = Control.ControlSource

                     OLD_VALUE = Control.OldValue

                     NEW_VALUE = Control.Value

                     If InStr(Control.Tag, "Memo") Then

                         Call Audit_Log_Entry(strTName, "MODIFY", [Full Name], FIELD_NAME, "", "")

                     Else

                         Call Audit_Log_Entry(strTName, "MODIFY", [Full Name], FIELD_NAME, OLD_VALUE, NEW_VALUE)

                     End If

                 End If

             End If

         Next Control

         DoCmd.SetWarnings True

    End If

  

End Sub

 

So just pass the form (object) to the ONE routine. Now you just place a call to audit in each sub form beore update event. I defaulted to the forms name, but you could pass any string, or name of table or whaterver you please.

 

Keep in mind that your main form before update code does NOT run if they ONLY edit data in a sub form anyway. So you would miss logging if users only edit data in a sub form.

 

This idea is only one extra line of code in the sub form (before update event), and would allow multiple sub forms, or even nested ones.

 

So make a “general” AuditLog routine you call and thus any form can make that call. And "bonus" is you can now add logging to any form - and not have to write a new routine, or place that code inside of the forms code module.

 

There is a “way” to build a “list” of all sub forms, and even nested sub forms (in the current main form), but it likely better to add a single line and code call in each sub forms before update event.

 

And my spider sense thinks the call should occur in the after update, not the before. (perhaps “oldvalues” are lost, but if they are retained, I vote for after update event.

Function MissingNumbers(Rng As Range) As String

    Dim lngRngCol As Long           'Column of Rng parameter

    Dim rowLast As Long             '5 rows after the UDF function row
    Dim rngToProcess As Range       'From rowFirst to rowLast

    Dim i As Long                   'Used in loop to process Min to Max row
    Dim Nums() As Variant           'Array for the missing numbers

        rowFirst = lngCallerRow - 5
    Else

Function MissingNumbers(Where As Range) As String
  Dim Lo, Hi
  Dim i As Long
  Dim Numbers
  'Find the limits
  Lo = WorksheetFunction.Min(Where)
  Hi = WorksheetFunction.Max(Where)
  'Prepare
  Numbers = Array()
  On Error GoTo Missed
  'Search for each number inside the limit
  For i = Lo + 1 To Hi - 1
    WorksheetFunction.Match i, Where, 0
NextNumber:
  Next
  'Combine and return the result if any
  If UBound(Numbers) >= 0 Then MissingNumbers = "Range " & Where.Address(0, 0) & " - " & Join(Numbers, ", ")
  Exit Function
 
Missed:
  'Add the number to the result array
  ReDim Preserve Numbers(0 To UBound(Numbers) + 1)
  Numbers(UBound(Numbers)) = i
  Resume NextNumber
End Function
