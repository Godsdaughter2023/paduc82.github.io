jay freedman say:

The first part of the setup is to save your document as a macro-enabled template (*.dotm) file -- otherwise you can't save macro code or AutoText entries in the file.

The next step is to create an AutoText entry of the entire initial content of the form.

  • If you want each repetition of the form to start a new page, right-click the first paragraph and choose Paragraph in the menu. In the dialog, go to the Line & Page Breaks tab and check the box for "Page break before".
  • Select everything (Ctrl+A), and press Alt+F3 to open the New Building Block dialog. Enter NewForm in the Name box (the macro below uses this name); make sure the template's name is in the Save In box; leave everything else as is, and click OK.

Finally, set up the macro that will be called by the Add Form button.

  • On the Developer tab, click the Design Mode button.
  • Right-click your Add Form button and choose View Code from the menu.
  • In the macro editor, delete the default two lines of code on the right, and then copy and paste the code from below.

Private Sub Macro1_Click()
    Dim rg As Range
    Dim ate As AutoTextEntry
    Set ate = ActiveDocument.AttachedTemplate.AutoTextEntries("NewForm")
    Set rg = ActiveDocument.Range
    rg.Collapse wdCollapseEnd
    ate.Insert Where:=rg, RichText:=True
End Sub

Now close the macro editor. IMPORTANT: Click the Design Mode button again to turn it off. Save the template.

The proper method for using a template is to double-click it in the File Explorer, which opens a new document based on the template. Initially the document will look just like the template, and the code and the AutoText will remain only in the template. Clicking the button to add another form will run the code from the template, and it will get a copy of the AutoText.

If you're sending this to other users by email or sharing a cloud copy, tell them that the template should be stored in their Templates folder (%appdata%\Microsoft\Templates) because otherwise the macro will probably be blocked by the standard security in Word.

john mcghie say:

What Bob is telling you is that the Spelling Checker is ALREADY installed.  It's part of Word: Word can't run without it (as you have discovered).

What is happening is one of two things:

1)  Word can't reach the spelling checker because of disk access permissions.  That's what Bob is suggesting (and it's the most likely one).

2)  The text you are trying to check is marked with a language you do not have in Mac Word.

To find out, select some text where the error appears and go to Tools>Language... and see which language is highlighted.

If it's not a language that you recognise (maybe text that has come in from the Internet) select the whole document and set its language to the one you want to use.

For example: This error could appear if you had some text set to English New Zealand if you have a copy of Word set to English USA.  Change the text to English USA and it will come right.

You will have to delete entire message, you can't remove a portion of that.

1. Individual delete is possible only for alerts sent by you where you can right click > Remove

2. If you want to delete the received alerts - It will not be possible to delete received alerts individually through Skype application. But you can delete the entire conversation by right clicking on left pane > Delete conversation.

This will delete the chats only from the device not from the cloud. Hence, if you have other devices where you are using Skype say mobile, tablet and another computer, they will be available on those devices. If you want to remove from other devices also, you will need to delete on each device separately.

Also, if you delete from your device doesn't mean that the history will be deleted on the side of the person who you chatted with. They will remain intact on their side.
