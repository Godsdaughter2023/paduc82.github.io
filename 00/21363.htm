stevecaponigri say:

Divide Table of Contents Pages

We use equations in page numbers to store two documents in one. (When printing, we either print the whole thing, or print just the even pages), so in the file, the page numbers that show in the footer are:

1, 1, 2, 2, 3, 3, 4, 4, etc.

Until now, we have made Table of Contents manually, which I want to stop doing.

When creating a table of contents, the true page numbers are shown, which don't match up with what the users see on the printed pages. I can't see a place to be able to add an equation to the page numbers in the ToC, but it has to be possible, right?

How do you add equations to the pages in the Table of Contents?

jay freedman say:

> I can't see a place to be able to add an equation to the page numbers in the ToC, but it has to be possible, right?


No, it does not have to be possible.

The automatic table of contents in Word is created by a single {TOC} field, and all its capabilities are controlled by the switches that it recognizes (https://support.office.com/en-us/article/field-codes-toc-table-of-contents-field). None of them allow the possibility of using anything other than the "formatted page number" generated by Word. That page number can be altered by inserting a section break and changing the starting page number for the section, but that's not what you're doing.

The one thing you can do is create your automatic TOC without any page numbers (use the Custom Table of Contents command, and clear the check box for "Show page numbers" in the dialog; this inserts a \n switch in the resulting TOC field code). Then click in the TOC and press Ctrl+Shift+F9 to convert the field result to plain text. Then you can manually insert a tab character and the correct page number after each heading. Of course, this should be done only after all other editing has been completed, so you minimize the chance that the TOC will have to be regenerated.
