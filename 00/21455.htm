Hello Diaz

The RATE function in Excel is a multipurpose financial function that is capable of finding interest rates not just for single sum investment but also for investments that have periodic payments. Due to this generality in RATE function, the underlying TVM equation does not permit solving for the RATE using algebraic manipulation.

To explain further let us look at two different TVM equations

1) PV FVIF(i%, n) = FV
2) PMT FVIFA(i%, n) = FVA
3) PMT FVIFAD(i%, n) = FVAD

1)

Let us look deeply in to each of these equations, the first one is used to find the future value of an investment of single amount

FVIF(i%,n) refers to future value interest factor or future value of a $1. It is expanded to the following form

FVIF (i%, n) = (1+i)^n

If we bring in compounding frequency in to consideration the FVIF turns in to following

FVIF (i%, n) = (1+i/c)^(n*c)

TO solve for interest rate in this situation we have do the following

PV (1+i/c)^(n*c) = FV
(1+i/c)^(n*c) = FV/PV
(1+i/c) = (FV/PV)^(1/(n*c))
i/c = (FV/PV)^(1/(n*c)) - 1
i = [ (FV/PV)^(1/(n*c)) - 1 ] * c

In this arrangement the signs of PV and FV will be both positive


2) Now let us look at the second case where we have a series of periodic payments and we are tasked to find the interest rate that sets equal the future value of payments to the present value of initial investment

PMT FVIFA(i%, n) = FVA

FVIFA(i%, n) is the future value interest factor of an ordinary annuity and its expanded form looks like this

FVIFA(i%, n) = [ (1+i)^n - 1 ] / i

Now replacing this value in 2) we get

PMT [ (1+i)^n - 1 ] / i = FV

From here it is very clear that we are not able to solve for interest rate i using algebraic manipulation. This will leave us with options of making use of iterative methods or what are called numerical methods to solve for the interest rate i.

The equation [ (1+i)^n - 1 ] / i is a shortened form of geometric series up to n terms

The acutal series looks as follows

(1+i)^1 +  (1+i)^2 +  (1+i)^3 + ....  (1+i)^N

It is actually an infinite series and the sum of first N terms is equal to

[ (1+i)^n - 1 ] / i


3)

PMT FVIFAD(i%, n) = FVAD is an altered form of 2) for an annuity due here we have an extra interest factor

PMT FVIFAD(i%, n) = PMT FVIFA(i%, n) (1+i)

Let us now come back to the original question of RATE function being unable to find the RATE for your single sum investment

The TVM equaion that is used by the RATE function in Excel and other financial calculators is the combination of our equation 1, 2, and 3 as follows

PV (1+RATE)^NPER + PMT (1+RATE*type) [(1+RATE)^NPER - 1]/RATE + FV = 0

Now that we have put together our equations 1) 2) and 3) in a single place we have to seek a general solution to solve for the interest rate.

However it could have been programmed that when PMT = 0 the equation which reduces to the following

PV (1+RATE)^NPER + FV = 0

could have been programmed to solve for RATE using algebraic manipulation

But since RATE has a task for finding RATE when there may be a periodic payment PMT thus it was programmed with Newton Raphson to find the RATE no matter if is for single amount investment or for one with periodic payments

The iterative method called Newton Raphson uses a seed value as an initial entry point in to the iterative calculations that are checked for convergence within 20 iterations

If the values converge within 20 iterations using a default seed value of 0.10 or 10% the RATE function returns a value

If not, it returns a #NUM error and you are now tasked with using a new seed value or guess rate to ignite the iterative process of finding the RATE

As you noted RATE returned a value for NPER of 84 and above yet below this it returned a #NUM error

This is so since the value for rate did not converge using a default guess rate of 10%

Now that it looks obvious that the actual rate is above 20% for such an NPER we would give the RATE function a guess value that is closer to the actual interest rate to RATE to return the result

formula to calculate budget vs actual percentage - Microsoft Community

happy_girl3 say:

formula to calculate budget vs actual percentage

Hi, I am trying to work out the % difference between budgeted payments against actual payments

In column A - we have our budged figures In column B - we have our actual figures

Can you please advise how to this this, and would it be the same formula to calculate actual vs budget.

Thanks in advance

ashish mathur say:
You are welcome.  If my reply helped, please mark it as Answer.

ejione say:

I do lots of this, in Excel 2007.  I will write these for a blank workbook, so as to avoid “Error” or “Divide by 0” messages. If you want to display the difference of budget and actual, in another Col, or Col C, =if(A2=0,0,b2-a2).  

This displays a 0 if A2 is 0 or Blank.  In place of 0, you can use “”, double quote, to display a blank cell.

A2 being the divisor must never divide by 0. 

This is difference, of Actual to Budget.  If the Actual is less than budget, you will have a negative difference, indicating that actual is less than budget. If Actual is more than budget it will be positive.

With that difference in Col C, in Col D or other, you can Calc the percentage of difference.  =If(A2=0,0,C2/A2).    If the difference is more, % will be Positive. If difference is less.%  Negative.

This results in the % of change over/under Budget. 

IF you do not want to display the actual difference, then in Col C/other, =if(A2=0,0,(b2-a2)/A2)

This allows you to enter the formula/s into blank cells with no data, Yet.

This also assumes that there will be NO data in Actual if NO data exists in Budget. If that should happen your % will be distorted/inaccurate

A Blank Template can be saved with all formulas intact.

Hope this is your need.
