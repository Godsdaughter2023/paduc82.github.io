Email Address Policies - Standalone Exchange Online - Microsoft Community

james.doutsis say:

Email Address Policies - Standalone Exchange Online

A few months ago I posted this: https://community.office365.com/en-us/f/148/t/435122

In short, the Email Address Policies do not work in Exchange Online Standalone.  Since then the Email Address Policies tab has been removed from EAC.  This situation is a huge pain for Standalone subscribers with multiple domains.

You can vote for the feature to be added here: https://office365.uservoice.com/forums/273493-office-365-admin/suggestions/8426817-implement-email-address-policy

Here are some scripts that will update most mail enabled resources.  Usual stuff, use at your own peril.  I am not responsible for what it may do.  In saying that, it worked for me.

Mailboxes except Public Folders:
$users = Get-Mailbox -RecipientTypeDetails 'RoomMailbox', 'EquipmentMailbox', 'SchedulingMailbox', 'LegacyMailbox', 'LinkedMailbox', 'LinkedRoomMailbox', 'UserMailbox', 'DiscoveryMailbox', 'TeamMailbox', 'SharedMailbox', 'GroupMailbox'
foreach ($a in $users) {$a.emailaddresses.Add("$($a.alias)@DOMAIN.NAME")}
$users | %{Set-Mailbox $_.Identity -EmailAddresses $_.EmailAddresses}

Public Folders:
$users = Get-Mailbox -PublicFolder
foreach ($a in $users) {$a.emailaddresses.Add("$($a.alias)@DOMAIN.NAME")}
$users | %{Set-Mailbox $_.Identity -EmailAddresses $_.EmailAddresses}

This does not update Distribution Lists.  RecipientTypeDetails does not seem to allow the types that list DLs as outlined here msdn.microsoft.com/.../microsoft.exchange.data.directory.recipient.recipienttypedetails(v=exchg.150).aspx

If someone with better scripting abilities could weigh in with a way to tackle DLs that would be fantastic.

For a short rant.  It is difficult to believe this isn't included.  There would never be a "customer want" to not include this.  It only introduces an administrative overhead and possible nightmare for Office 365 admins.  Recipient Policies have been around in some fashion in Exchange 5.5 (site addressing maybe? So long ago) but definitely from Exchange 2000 onwards but removed in Office 365 Exchange Online Standalone

james.doutsis say:

Here you go.  Distribution Lists:

$users = Get-DistributionGroup

foreach ($a in $users) {$a.emailaddresses.Add("$($a.alias)@DOMAIN.NAME")}

$users | %{Set-DistributionGroup $_.Identity -EmailAddresses $_.EmailAddresses}

Note - this applies the DL's Alias as the secondary SMTP address.  Some manual intervention may be required.

Data type mismatch in criteria expression - Microsoft Community

emilyftw say:

Data type mismatch in criteria expression

Hi,

I have the following query (see below), which pulls from a prior query wsq3 and a Staff table (via multiple joins).  The wsq3 query is mostly fully populated, but has some entries (for dates for which there are no milestones) for which the only data is (1) the date for that day in calDate and (2) FirstLine = 'None'.  Other values for those rows are Null.

wsq3 is limited (via a parameter) to run only over a specified date range (two weeks of weekdays beginning on a specified Monday).  If I run the below query and specify a Monday for which all of the days over the next two weeks have milestones, then the query runs fine.  However, if I run it for a date range where there is at least one day where there are no milestones (and so only calDate = respective date and FirstLine='None'), then I get a "Data type mismatch in criteria expression error."  Any insight why?

I'm not confident that this is the issue, but the only thing I can think of is that, because some of the entries in the FirstLine = 'None' rows have Null values in the other fields, maybe that is causing problems somewhere in one or more of my expressions.  But I don't think I'm doing anything that you can't do with a Null.  I think you can concatenate Nulls.  (In fact, other queries process these Null fields in much the same way I am here, and they don't give me any errors, unless there is some relevant difference I'm not thinking of.)

Also, the error message says "in criteria expression," so I have to think it is tripping on my (wsq3.FirstLine = 'None') statement, but I can't figure out why.  If I remove that statement, it works fine.

I also tried changing to (wsq3.FirstLine & '' = 'None'), thinking that adding the zero length string might force a string-to-string comparison rather than a Null-to-string comparison, but I get the same error under that approach.  I've also tried IsNull(wsq3.SecondLine) instead of (wsq3.FirstLine = 'None'), since any time FirstLine = 'None' then SecondLine must be Null, but also same error under that approach.

Really appreciate any assistance!
Emily

Query wsq4:
SELECT wsq3.calDate, wsq3.Milestone & wsq3.Applicant & wsq3.CaseName AS SortString, wsq3.FirstLine, wsq3.SecondLine, sL.Division AS LeadDivision, sM.Division AS MemberDivision, sM2.Division AS Member2Division, sR.Division AS ReviewerDivision
FROM (((WeeklyScheduleQueryStep3Detail AS wsq3
LEFT OUTER JOIN Staff AS sL
ON wsq3.OEPILead = sL.First & ' ' & sL.Last)
LEFT OUTER JOIN Staff AS sM
ON wsq3.OEPIMember = sM.First & ' ' & sM.Last)
LEFT OUTER JOIN Staff AS sM2
ON wsq3.OEPIMember2 = sM2.First & ' ' & sM2.Last)
LEFT OUTER JOIN Staff AS sR
ON wsq3.OEPIReviewer = sR.First & ' ' & sR.Last

WHERE (sL.Division='DETA') OR (sM.Division='DETA') OR (sM2.Division='DETA') OR (sR.Division='DETA') OR (wsq3.FirstLine = 'None');

jack stockton say:

You cannot set criteria on outer joins where there is no data.

 

Break each of the outer joins into sub-queries with their own criteria.  Then create the master query with all the sub-queries.

emilyftw say: How do you define "no data"?  There weren't any rows of any subqueries that had actually no data.  They all had at least two fields populated.

In the end I ended up completely re-designing my series of queries to be more efficient.  In a previous string I had followed some other advice to add the "None" row for the dates with no milestones early in the process.  In my re-design, I did all of my filtering and expression building in a few steps in separate queries, and then add the "None" rows at the end, and that worked better.  Not sure why though.

Anyway, thanks much!
Emily

doug robbins - mvp office apps & services (word) say:

If you have an Excel Workbook, with the text for each footnote in column 1, with the text for the first footnote in cell A2, the text for the second footnote in A3, the text for the third footnote in A4 and so on.  Running a macro with the following code will set the text of each footnote to the data that is in the relevant cell

Dim xlapp As Object
Dim xlbook As Object
Dim xlsheet As Object
Dim i As Long
On Error Resume Next
Set xlapp = GetObject(, "Excel.Application")
If Err Then
    bstartApp = True
    Set xlapp = CreateObject("Excel.Application")
End If
On Error GoTo 0

'  Change the path\filename in the following command to match that of the Excel file containing the text for the footnotes.

Set xlbook = xlapp.Workbooks.Open("C:\Users\dkr\Downloads\Broken Documents\footnotes.xlsx")
Set xlsheet = xlbook.Worksheets(1)
With ActiveDocument
    For i = 1 To .Footnotes.Count
        .Footnotes(i).Range.Text = xlsheet.Range("A1").Offset(i, 0)
    Next i
End With
