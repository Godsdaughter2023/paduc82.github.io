Slipstreaming worked when Windows Setup used a file copy method to install the software.  Both Vista and Windows 7 use a image file so slipstreaming is not applicable, at least not in the sense it was with XP.   Instead you have open the image file, edit, and then close the image again.  An entirely different procedure.  There is plenty about editing an image for deployment on TechNet.

There is a tool called VLite for editing hotfixes and such into a Windows 7 image but the problem is that there is a lot more to a service pack than just hotfixes and patches.  It also does rewrites of core files that are not available in any hotfix or patch and can only be gotten by running the service pack installer.  That was what autostreamer was able to do with XP service packs.

What we all really need is .iso files for Windows 7 with SP1.  In other words, the images that went to manufacturing when SP1 released and will be in retail boxes shortly if not already.  It should also be the image available for download from the Microsoft Store.  TechNet and MSDN subscribers will shortly have these images as well.

I don't want to get your hopes up too much since you may be missing more than that one file, but if just wininet.dll is missing (or maybe one or two more files), there should be copies in other places on your system and you might be able to replace it/them.

Start with easy stuff first:

If you press Ctrl-Alt-Del you should be able to get into Task Manager.

From TM, you can click File, New Task and in the Open box enter:

%SystemRoot%\system32\restore\rstrui.exe

Click OK and the System Restore Dialogue box should open and you can see if you can get a Restore Point to work and restore your system back to a date prior to the incident.

We know XP keeps backup copies of important file here (assuming XP is installed on your C drive):

c:\windows\system32\dllcache

If you installed Service Pack 3 from a download, there should also be copies of most "good" files here:

c:\windows\ServicePackFiles\i386

So from TM, click File, hold down the Ctrl key, click New Task and a Command Prompt Window should open.

copy c:\windows\system32\dllcache\wininet.dll   c:\windows\system32

or:

copy c:\windows\ServicePackFiles\i386\wininet.dll   c:\windows\system32

Hopefully, one of those will result in a One File(s) copied message.  Respond in the affirmative if asked to overwrite an existing file.

Enter 'exit' to close the Command Prompt window and from TM, click Shutdown, Restart and see how things look then.

If other files are missing, make a note of them and repeat the copy process from TM and see if you can replace any missing files XP complains about during startup.

If for some reason the copy commands fail or if the backup copies of the files are missing, we can look at another plan but try the easy stuff first.
