I've learned how to turn the filtering off in the macro so that all lines are visible before it copies old data from and then pastes new data in the spreadsheet , but I want to re-set my column headings with the filtering down-arrows so that the user doesn't have to re-set the Filtering.  Unfortunately with this worksheet, some of the column headings are actually 2 cells in multiple rows that have been merged to create one cell and other column headings are just one row high.  For example, the column headings will be:

 

A4

B4

C4

D3:D4

E4

F3:F4

 

The only way I know to auto filter with the column headings like this is to unmerge the cells D3:D4 and E3:E4, apply filtering, then re-merge the cells.  I need to know how to avoid this manual re-filtering and have it happen within the macro so that it's seamless to the user.

 

Can anyone guide me?

 

Thanks!

ossiemac say:

Your example indicates D3:D4 and F3:F4 are merged and then you indicate unmerging D3:D4 and E3:E4. The code example below assumes F3:F4 as per the example.

 

The reason for ensuring that AutoFilter is turned off is because AutoFilter is a toggleing command and if it just happens to be On then it gets turned off so Setting AutoFilterMdoe to False ensures it is turnd off before the remaining code is executed.

 

 

Sub SetAutoFilter()
    With Sheets("Sheet1")           'Edit "Sheet1" to your sheet name
        .AutoFilterMode = False     'Ensure AutoFilter is turned off
        .Range("D3:D4", "F3:F4").UnMerge
        'Next line references row 4 from column A to last used cell in row 4
        .Range(.Cells(4, "A"), .Cells(4, .Columns.Count).End(xlToLeft)).AutoFilter
        .Range("D3:D4").Merge
        .Range("F3:F4").Merge
    End With
End Sub

ossiemac say:

Actually you do not need to unmerge the cells first. If Selecting the range to apply AutoFilter in the interactive mod then both rows get selected but with VBA the Row 4 can be referenced and directly apply the AutoFilter without Unmerging.

 

Sub SetAutoFilter_2()
    With Sheets("Sheet1")           'Edit "Sheet1" to your sheet name
        .AutoFilterMode = False     'Ensure AutoFilter is turned off
        'Next line references row 4 from column A to last used cell in row 4
        .Range(.Cells(4, "A"), .Cells(4, .Columns.Count).End(xlToLeft)).AutoFilter
    End With
End Sub

Cannot Print Attachments From Office 365 In Google Chrome

We use Office 365 at our company and can view and download attachments, but when we try to print them in-window, a new page opens and nothing loads.

It appears to work fine if you go in-incognito in chrome but we can't do that every single time. Has anyone else come across this issue?

a. user say:

After speaking with a Microsoft Representative, we've determined it was a browser issue. To remedy it, we performed the following steps.

  1. Choose the vertical ... in the top right corner of chrome
  2. Choose Settings
  3. Scroll down and open "Advanced"
  4. Find "Reset and cleanup"
  5. Chose "Restore settings to their original defaults"
  6. Close Chrome and opened

This setting restores defaults and turns off all extensions in your browser and deletes your cache but does not remove your bookmarks ETC. After restarting chrome and signing back in it works just fine.
