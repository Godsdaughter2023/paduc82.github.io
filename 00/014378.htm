daniel g. grau say:
UK Troll, based on Peter's code, I wrote a macro that will show the current cell's coordinates as a popup menu. This is what it looks like:

 

Pretty neat. The simple version has six lines of code:

Sub ShowCellCoordinates()
Set myBar = CommandBars.Add(Name:="Custom1", Position:=msoBarPopup, Temporary:=True)
myBar.Controls.Add Type:=msoControlButton
myBar.Controls(1).Caption = ChrW(AscW("A") + Selection.Cells(1).ColumnIndex - 1) & CStr(Selection.Cells(1).RowIndex)
myBar.ShowPopup
End Sub

You need to follow these instructions to paste the above code as a macro into the Visual Basic Editor:

  • Create A Macro

Then you should assign it an unused key shortcut (like "Control C") by following these instructions:

  • Assign command or macro to hot key

Once you're done, click on a cell and press its shorcut. It works nicely, except that Word's "Help" menu choice is always present. And you need to get used to clicking outside the menu in order to dismiss it.

With the simple code above, if the cursor is not within a table, you will get an error message. The message can be elegantly hidden by adding more code to channel the error. The 11 lines of code follow:
 
Sub ShowCellCoordinates()
Set myBar = CommandBars.Add(Name:="Custom1", Position:=msoBarPopup, Temporary:=True)
myBar.Controls.Add Type:=msoControlButton
On Error GoTo NotATable
    myBar.Controls(1).Caption = ChrW(AscW("A") + Selection.Cells(1).ColumnIndex - 1) & CStr(Selection.Cells(1).RowIndex)
    myBar.ShowPopup
Exit Sub
NotATable:
    myBar.Controls(1).Caption = "The cursor must be within a cell"
    myBar.ShowPopup
End Sub



jim gordon mvp say:
The grid labels in a Word table are laid out the same as in Excel.

The table cell in the upper left corner is A1. Columns are letters A, B. C... and rows are numbers. The only difference is that you have to count the rows and columns yourself if you want to use row and column numbers to reference the table cells.

You could always use the first row as the letters of the columns, then hide them when you don't want to see them (use the table options on the Table Layout tab of the Ribbon or adjust the row height to zero). Remember your first row of data will be 2 instead of 1 if you use this trick.

An alternative is to insert a text field (from the Developer tab on the Ribbon) into each cell you wish to use in the calculation. Double-click the text field. In the Field Settings you will see BookMark1, 2, etc. which you can use in your formulas, or you can define a name for each field bookmark by typing a new name into the dialog and then refer to the bookmark name in the formula.
