maor bracha say:


Hi Curt,

The reason it works for migrated mailboxes is that Exchange remote move migrations "translate" migrated mailboxes to "remote mailbox" on-prem representation, holding required attributes such as email addresses and target address for routing.

Creating an AD based user, means managing user attributes locally as mentioned.
While licensing such a user would provision a mailbox, once a sync cycle occurs certain configurations will be "reset to default" as the user attributes are set to null (ProxyAddresses for an email address management, for instance)

To mail enable an AD user in a Hybrid environment, you would have to create a remote user mailbox representation, or at least populate required attributes. 
Creating a remote mailbox would assign email addresses according to your policy. 

Further reading available here: 
https://technet.microsoft.com/en-us/library/ff625227(v=exchg.141).aspx
If you would follow this guide, take a look at your on-prem EMC, you will notice your migrated mailboxes are represented as remote mailboxes.

If you already created an AD user and would like to enable it for Exchange online in a Hybrid, use the command "Enable-RemoteMailbox" 
https://technet.microsoft.com/en-us/library/ff607313(v=exchg.160).aspx

This will create a local Exchange representation by populating required attributes and would allow you to easily manage Exchange online mailboxes.

tobytu_msft say:

Hi John,

This is an expected behavior.

Since the shared mailbox is different from our normal user mailbox, we need to assign send as or send on behalf right to users, the sender is ourselves instead of the shared mailbox. So the sent item does not appear in the shared mailbox.

Now we can manually enable this feature via PowerShell, so that we can control Sent Items when using shared mailboxes.

Connect to Exchange Online using remote PowerShell

Run the following cmdlets, it can be enabled by setting feature enable flag to TRUE:

For emails Sent As the shared mailbox:

set-mailbox <Shared mailbox name> -MessageCopyForSentAsEnabled $True

For emails Sent On Behalf of the shared mailbox:

set-mailbox < Shared mailbox name> -MessageCopyForSendOnBehalfEnabled $True

If you then want to disable the feature again, you can do it as follows:

For emails Sent As the shared mailbox:

set-mailbox < Shared mailbox name> -MessageCopyForSentAsEnabled $False

For emails Sent On Behalf of the shared mailbox:

set-mailbox < Shared mailbox name> -MessageCopyForSendOnBehalfEnabled $False

Please let me know if you run into any errors.

Regards,

Toby
