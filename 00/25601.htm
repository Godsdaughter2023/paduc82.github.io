1. Autonumbers can't be edited.

3. If you are using an autonumber for a field which you care what it is then you are using it incorrectly.  An autonumber should only be used for something which you need a UNIQUE number and normally that should not even be seen by users.  It should be used to maintain referential integrity of your data between tables.

hansv mvp say:

SQL is by default case-insensitive: it does not distinguish between "sql", "SQL" and "Sql".

You can use the StrComp function to perform case-sensitive comparisons:

 

 

strSQL = "SELECT * FROM [tablename] WHERE StrComp([fieldname], '" & ChkVal & "', 0) = 0"

That's typically a LaunchService issue.

Id' select the file in the Finder, then press command-i to get the Info panel for the file.

From there, you should be able to make sure that Excel is indeed setup to open these files. If it's not, you can select it and make sure that the same action is set for all files of the same file type.

In some cases, that still doesn't work. If that happens to you, I'd suggest using a free utility like Onyx to reset the LaunchService database.
