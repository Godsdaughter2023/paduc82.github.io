ken blake - microsoft mvp say:

On Sun, 14 Nov 2010 20:52:50 +0000, irishgo888 wrote:

I have reduced the number of startup programs and services on windows 7

You should never have reducing the number as an objective. You should
be concerned about what the startup programs are. Read the
following:

First, note that you should be concerned with all programs that
start automatically, not just with those that go into the system tray.
Not all auto-starting programs manifest themselves by an icon in the
tray.

On each program you don't want to start automatically, check its
Options to see if it has the choice not to start (make sure you
actually choose the option not to run it, not just a "don't show icon"
option). Many can easily and best be stopped that way. If that doesn't
work, run MSCONFIG from the Start | Run line, and on the Startup tab,
uncheck the programs you don't want to start automatically.

However, if I were you, I wouldn't do this just for the purpose of
running the minimum number of programs. Despite what many people tell
you, you should be concerned, not with how many of these programs
you run, but which. Some of them can hurt performance severely, but
others have no effect on performance.

Don't just stop programs from running willy-nilly. What you should do
is determine what each program is, what its value is to you, and what
the cost in performance is of its running all the time. You can try
internet searches and ask about specifics here.

Once you have that information, you can make an intelligent informed
decision about what you want to keep and what you want to get rid of.

but my start time is still very slow (it takes over 20 minutes for windows to start up after I put in my password).  How can I further speed up my startup?

Two points:

1. You may have reduced the number but you may not have stopped the
important ones. Read the above again, and do as I suggest with regard
to which you should stop.

2. More likely than your problem being startup programs is that you
are infected with malware. What anti-virus and anti-spyware programs
do you run? Are they kept up to date?

Also, what services can I disable? I have disabled quite a few optional services but I don't know if there are still other services that I could disable.

Disabling services is unlikely to help you with your problem and is
almost always a bad idea.

Tinkering with Services is a recipe for creating a computer where things do not work! Leave them alone.

Look at Applications which load at Start Up. Decide whether they need to load at "Start Up" or whether they are better used on an "On Demand" basis.

Develop a housekeeping routine and stick to it.

Determine where bottlenecks on performance exist by observing which operations are slow and how they impact on memory and CPU usage. Keep your computer free from error by keeping an eye on errors in Event Viewer and deal with them as they arise.

frederik long say:

It is not possible to reset a password while in Repair Mode.

Here are two easy and one painful method to reset a Windows password:

  • Log on using the password key file on a USB flash disk (if you ever made one)
  • Log on under the alternative admin account (if you ever created one and made a note of its details)
  • Use the process below. The process is complex and requires some careful typing.
     1. Set your BIOS to use the DVD drive as the primary boot device.
     2. Boot with a Windows Repair CD.
     3. Press a key when the message indicating "Press any key to boot from CD or DVD …" appears.
     4. Select a language, a time and currency, and a keyboard or input method, and then click Next. You are now in Repair Mode.
     5. Select an Operating System, then click Next.
     6. Select the Command Prompt.
     7. Identify the correct drive letter for your Windows installation. It could be a drive other than C: but it won't be X:. You do it by typing the commands
        dir C:\Win* {Enter}
        dir D:\Win* {Enter}
        dir E:\Win* {Enter}
        etc.
        until you find the Windows folder.
        I will call that drive "Q:".
     8. Type these commands:
        cd /d Q:\windows\System32 {Enter}
        move  sethc.exe  \  {Enter}
        copy  cmd.exe  sethc.exe  {Enter}
     9. Reboot normally.
    10. When challenged for a password, tap the Shift key five times. You will see a small black screen.
    11. Type these commands:
        net user  {Enter}
        (To see a list of all accounts that exist on your machine)
        net user Administrator {Enter}
        (To see if the Administrator account is active)
    12. Type these commands:
        net user "xxx" ppp  {Enter}
        (Replace xxx with the account for which you wish to set a new password, and ppp with the new password)
        net user Administrator /active:yes {Enter}
        (To activate the Administrator account if desired)
    13. Log on under this account, using the new password.
    14. Reboot the machine into Repair Mode as per Steps 1..6 above.
    15. Type these commands:
        cd /d Q:\windows\System32 {Enter}
        copy \sethc.exe {Enter}
    16. Reboot into Windows.
    Instead using a Windows Repair CD in Step 2, you can use your Windows 7 Installation DVD. The steps are similar to the ones above.
When the pain has subsided, suggest to your friend that a small amount of planning ahead will save her the pain next time round.

There are numerous threads regarding networking problems which occur after upgrading to Windows 10, or from updates to Windows 10 systems. The upgrades and updates corrupt networking on Windows 10. Unfortunately, the consequences of the corruptions are a wide variety of networking problems - many of which appear unrelated. As a result, many threads speak of computers not appearing in the network, "0x80070035 network path not found" errors, Windows Sockets errors, protocol error messages, and other strange errors. These errors are frequently due to the network settings being corrupted by the Windows 10 upgrades and updates.

 

I have a local area network (LAN) consisting of several computers connected via ethernet cable and WiFi to a Comcast modem/router. My goal is in Windows File Explorer in the "Network" item to see both my computer and the other computers in my LAN. And to share drives between these computers via "Use user accounts and passwords to connect to other computers". Previously this worked in Windows 7. It did not work after upgrading all the computers to Windows 10 even though all my network settings were identical.

 

The following process corrected my situation:

Reset your modem/router to factory settings.

Start | right mouse | Command Prompt (Admin)

   Run the following two commands, then reboot.

      netsh winsock reset

      netsh int ip reset resettcpip.txt

Resetting the modem/router to factory settings makes sure the router is not the issue.

Running the two netsh commands resets network component settings.

If you have a local home network,

in "Open Network and Sharing Center",

make sure it is "private".

In order to see your local home networked computers using advanced sharing to their drives,

in "Change advanced sharing settings",

in Private (current profile), enable the following:

   Turn on network discovery

   Turn on file and printer sharing

   Use user accounts and passwords to connect to other computers

In order to see your local home networked computers using advanced sharing to their drives, both your PC and the other local network PC require an account on both computers with the same name.

In addition, for each computer, you must go into their

"Credential Manager" | "Windows Credentials" tab

and give the Password for the common account to the other computer.
