When installing operating systems forget about drive letters, they mean nothing.  At the setup stage of the installation, disk are not identified by letters they are identified by numbers (starting at 0), the BIOS will present the disks in its own order to the Windows setup program and this will affect the way Windows enumerates them and assign drive letters.

The Windows Mount Manager assigns drive letters and stores them in its database, the database is stored in the registry at:  HKEY_LOCAL_MACHINE\SYSTEM\MountedDevices, keeping the Mount Manager's database in the registry ensures drive letter persistence.  You can change any drive letter after Windows is installed with the built-in disk management tool (diskmgmt.msc) and MountedDevices database will be updated to reflect the changes.  Due to drive letter persistence and the preference accorded to the boot volume at boot time Windows would not under normal circumstances change the drive letter of the boot volume after Windows is installed as this would cause problems with the installation.  My guess is that when you reintroduced the old disk to the system Windows booted the old installation rather than the new one.
