zigzag3143 (ms -mvp) say:
[quote]
"It's not a true crash, in the sense that the bluescreen was initiated only because the combination of video driver and video hardware was being unresponsive, and not because of any synchronous processing exception".

Since Vista, the "Timeout Detection and Recovery" (TDR) components of the OS video subsystem have been capable of doing some truly impressive things to try to recover from issues which would have caused earlier OS's like XP to crash.

As a last resort, the TDR subsystem sends the video driver a "please restart yourself now!" command and waits a few seconds.

If there's no response, the OS concludes that the video driver/hardware combo has truly collapsed in a heap, and it fires off that stop 0x116 BSOD.

If playing with video driver versions hasn't helped, make sure the box is not overheating.

Try removing a side panel and aiming a big mains fan straight at the motherboard and GPU.

Run it like that for a few hours or days - long enough to ascertain whether cooler temperatures make a difference.

If so, it might be as simple as dust buildup and subsequently inadequate cooling.

I would download cpu-z and gpu-z (both free) and keep an eye on the video temps

Note: This all centers around the fact that you create an Image Backup of both your Windows XP and Windows 7 partitions to an external DVD drive or in a pinch a second hard drive. Less catastrophic method of removing Windows XP:
This method deletes all the Windows XP files but leaves the BCD Store (C:\BOOT) and the bootmgr file intact.

The first thing I tried from within Windows 7 was to delete the Windows XP C:\Windows folders and all files located at the root level. The only folder I left in place was C:\Boot as this is where the BCD store is located, the bootmgr file and the System Volume Information folder (which could not be deleted).

I then checked to see if I could still boot to Windows 7. Since the BCD store was not removed, Windows 7 was still able to boot without any issues. Windows XP was gone so even though the boot menu still displayed Windows XP as one of the boot menu options choosing to boot XP will only result in the Windows Boot Manager window with the message “Windows failed to start. A recent hardware or software change might be the cause. To fix the problem:” followed by three choices.

Of course you can use EasyBCD to remove the Windows XP entry in the boot menu. The end result is that the first partition on the hard drive is now empty except for the \BOOT folder and the bootmgr file. As long as you don’t delete these two files you can use the partition for any purpose. Not the best arrangement but it’s functional and no you can’t use the extend feature in Windows 7 to expand the Windows 7 partition as the now largely empty XP partition is physically located in front of the Windows 7 partition.
