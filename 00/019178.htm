Sort Data in Multiple Columns - Microsoft Community

burtlake say:

Sort Data in Multiple Columns

I have a data table that is in three columns, each about 100 entries long.  I would like to sort all three columns (which are adjacent to each other) into a continuous alphabetical order.  In other words, the first column would begin with names that start with "A", and the last column would end with names that start with "Z".  Everything in between would be in order.

 

I know I can temporarily move all the data into one column, then do the sort.  But this is time conuming.  I there a way I can do the sort with the data still in the 3 columns?

lars-Åke aspelin say:

I have a data table that is in three columns, each about 100 entries long.  I would like to sort all three columns (which are adjacent to each other) into a continuous alphabetical order.  In other words, the first column would begin with names that start with "A", and the last column would end with names that start with "Z".  Everything in between would be in order.

 

I know I can temporarily move all the data into one column, then do the sort.  But this is time conuming.  I there a way I can do the sort with the data still in the 3 columns?


Try this macro:

Sub sort_selection()
  Dim nrows, ncols As Integer
  Dim mydata() As String
  Dim temp As String
  Dim i, j As Integer
  nrows = Selection.Rows.Count
  ncols = Selection.Columns.Count
  ReDim mydata(nrows * ncols)
  For i = 1 To ncols
    For j = 1 To nrows
      mydata((i - 1) * nrows + j - 1) = Selection(j, i).Value
      If mydata((i - 1) * nrows + j - 1) = "" Then mydata((i - 1) * nrows + j - 1) = Chr(255)
    Next j
  Next i
  For i = 0 To nrows * ncols - 2
    For j = i + 1 To nrows * ncols - 1
      If mydata(j) < mydata(i) Then
        temp = mydata(i): mydata(i) = mydata(j): mydata(j) = temp
      End If
    Next j
  Next i
  For i = 1 To ncols
    For j = 1 To nrows
      Selection(j, i) = mydata((i - 1) * nrows + j - 1)
      If Selection(j, i) = Chr(255) Then Selection(j, i) = ""
    Next j
  Next i
End Sub

The two lines with Chr(255) is there just to make sure than any blank cells turn up at the end instead of in the beginning.
If you don't have any blank cells, or don't mind them coming in the beginning, these lines can just be removed.

Select your data to be sorted, e.g. cells A1:C100, and run the macro.

Note that all formulas in the selected range will be removed. Just the values will survive this macro.

Hope this helps / Lars-Åke

VBA disable cut and drag in excel 2010

mjcorkery (mct regional lead) say:

Hi,

 

The Application object has two properties that you will need to set, Application.CutCopyMode
and Application.CellDragAndDrop.

 

You will want to set the value of both of these properties to fals for the workbook. Remember that you need to set these back to true, otherwise cut and drag will not work in other workbooks.

 

Here is some sample code. I chose to use the activate and deactivate events of the workbook. When you click in the workbook, the Activate event fires and the . When you click to another workbook the Deactivate event fires and the value of the two properties is set to true again.

 

To add this code, open the VB editor. Locate the ThisWorkbook code page for the workbook. Add this code.

 

Private Sub Workbook_Activate()
  Application.CutCopyMode = False
  Application.CellDragAndDrop = False
End Sub

 

Private Sub Workbook_Deactivate()
  Application.CutCopyMode = True
  Application.CellDragAndDrop = True
End Sub
