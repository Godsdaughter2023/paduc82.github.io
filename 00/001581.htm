a. user say:

Hi Wendy,

Thanks for the requested information in PM.

I have checked the result of Outlook Connectivity Test. It seems that the test runs successfully. But I noticed that you have create an Autodiscover record which is pointed to Office 365 for your custom domain. It may cause this issue. In Exchange 2003, the Autodiscover service is not supported. So the Outlook Connectivity test passes successfully after you manually specify the server settings. But if you create an Autodiscover record which is pointed to Office 365, it may connect to Office 365 when you create a migration endpoint using this Autodiscover record. Please remove this Autodiscover record (you need to create an Autodiscover record in post-migration stage.

Moreover, if Exchange Online cannot connect to the source server automatically when creating a migration endpoint, please provide values for the following boxes:
1. Exchange server   Type the fully qualified domain name (FQDN) for the on-premises Exchange server. This would be the host name for your Mailbox server; for example, EXCH-SRV-01.corp.contoso.com.
2. RPC proxy server   Type the FQDN for the RPC proxy server for Outlook Anywhere. Typically, this will be the same as your Outlook Web App URL; for example, mail.contoso.com. This is also the URL for the proxy server that Outlook uses to connect to an Exchange server.

If you encounter any problem, please feel free to let me know.

Hi Michael,

Thank you for posting the query in Microsoft Community Forums.

1. Does the other parent account is registered as administrator account?

2. Does this account gets activity report from the child?

3. Have you recently changed any settings for this particular account?

The parent’s account must be an administrator account, you may change the account type to administrator if using a standard account.

Check if this account is receiving activity report from the child's account.

If this isn't working for you, it may be that you used this Microsoft Account to set up a different machine.  If that's the case, you'll have two separate families with two sets of parental controls settings.  In that case, I think you'll have to turn off Family Safety on one machine and then turn it back on using the same Microsoft Account as the other machine.

If the issue still persists, you may remove all the account and then reconfigure child's account again. You may also refer the section 'What type of account should I use with Family Safety?' to get more information on how to create 2 parents without any conflicts.
