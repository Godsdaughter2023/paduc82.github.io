malkeleah say: First of all, putting the programs on a separate partition isn't useful. If you have to reinstall Windows, you'll need to reinstall all the programs anyway since none of the necessary Registry entries will be there after a clean install. Putting the data on the other partition is fine. Another issue I see is that you named your Partition A:\ and A:\ and B:\ are generally reserved for floppy drives, even if you don't have any. Better to call it something else.

Now, to get out of this pickle try going to the diagnostic menu and booting to Last Known Good Configuration. Do this by repeatedly tapping the F8 key at computer startup.

If LKGC doesn't fix the issue, then I'm afraid you will need to reinstall Windows. You should delete all partitions and create new ones (if desired) during this process. I don't know what you mean by "I can't reinstall Windows 7 because the .exe file won't open". Unless you are using an upgrade version of Win7, you would boot with the Win7 DVD and not start the installation from within Windows itself. Please clarify what type of computer you have (make/model) and how you got Win7 if you need more help.

If you must reinstall Windows and haven't backed up your data, you can either put the drive into an external USB drive enclosure and attach it to a working computer to copy off the data or boot the machine with a Linux Live CD such as Knoppix and copy the data to an external hard drive. Let me know if you need help with any of that.

A great way to back up your system for future disaster recovery is to purchase an imaging program such as Acronis True Image and an external hard drive. Store the image of your system you make with TI on the external drive and you can restore to a working machine in relatively little time. TI can also do incremental imaging backups so you don't need to restore back too far.
