inbox messages - Microsoft Community

robertstusak say:

inbox messages

How do I purge deleted messages?

jay sun msft say:

Hi Robert,

Items that you delete are moved to the Deleted Items folder. In this way, Outlook client and Outlook Web App can be configured to empty the Deleted Items folder automatically, or you can manually empty the folder at any time.

If you use Office 365 for Business account, you can configure emptying Deleted Items folder automatically in Outlook Web App by following steps:

1. Login to Outlook Web App.
2. Click the gear icon on the upper right > choose Mail > select Automatic processing under Mail > select Message options.
3. Select Empty the Deleted Items folder when I sign out check box.
4. Click Save.

If you use Outlook 2016 client, you can set the automatic emptying Deleted Items folder by following steps:

1. Go to File > Options.
2. Click Advanced > select Empty Deleted Items folders when exiting Outlook check box.
3. If you want to be notified before the Deleted Items folder is emptied automatically, you can select Prompt for confirmation before permanently deleting items check box under Other on the Advanced tab of this Options page.

Meanwhile, you can manually empty the Deleted Items folder by right-clicking the Deleted Items folder and clicking Empty Folder.

Besides, after emptying Deleted Items folder, if you don’t want deleted items come back, you can erase them in Recover Deleted Items by following steps:

1. Select the Deleted Items folder.
2. Click Home on the ribbon > select Recover Deleted Items from Server. Or click Folder on the ribbon > select Recover Deleted Items.
3. Choose the items you want to erase or click Select All.
4. Select Purge Selected Items > click OK.
5. It should be noted that completing steps above will prevent you from being able to recover the deleted items you erased.

For Outlook Web App, you can select Deleted Items folder > click Recover deleted items > select Deleted items you want to erase > click Purge.

Regards,
Jay

How do I store entries from a Combo box as text rather than a number in Microsoft Access?

ken sheridan say:
The combo box needs to be set up to hide the bound column by setting its ColumnWidths dimension to zero.  For a combo box t select an employee by name for instance:

ControlSource:   EmployeeID (for a bound control; leave blank if the combo box is an unbound 'navigational' control)

RowSource:     SELECT EmployeeID, FirstName & " " & LastName FROM Employees ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

Alternatively, you can concatenate the names so that the last name is first, which would be better with a large list of names as it allows the user to enter the initial characters of the last name and progressively go to the first match as each character is entered:

RowSource:     SELECT EmployeeID, LastName & ", " & FirstName FROM Employees ORDER BY LastName, FirstName;

scottgem say:

Please see my blog article on displaying Data from Related tables (blog address in sig). this articles discusses why you are seeing what you are seeing and techniques for displaying the related data.

In a relational database data exists in ONE place and one place ONLY. So if you have a lookup table of animals. You don't want to repeat the animal name for each record. You want to use a code (usually numeric) to represent the selected animal. Then you use joins in queries or other methods to pull the data from the related table.

Which is exactly what it will do if set up in the way I described.  The value of the combo box, and thus the column to which it is bound is a numeric foreign key referencing the numeric primary key, usually an autonumber, of the referenced table.  Bt returning both the key and the text column in the combo box's RowSource query, and hiding the former by setting its width to zero, the latter is shown.
