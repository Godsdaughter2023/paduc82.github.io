Do not display label on report if field is null for a record - Microsoft Community

marylynnowen say:

Do not display label on report if field is null for a record

I have created a report which prints a separate page for each record in the database.  I would like the report to not print the label for any field which does not contain data, but to print it for each field that does contain data for that particular record.  What is the best way to accomplish this?

john w. vinson say:

One (rather ugly) way to do this is to append the label to the textfield content in the Query upon which the form is based. E.g. if there's a field EMail which might or might not exist, put

ShowEmail: "EMail: " + [EMail]

to concatenate a label EMail: to the content of the field. The + operator (unlike the & operator) propagates NULL so this calculated field will be NULL if the [Email] field is NULL.

You can try uninstalling the Logitech driver to see if the mouse will work better with the generic Windows mouse driver:

  1. In Windows, click on the Start orb and choose Control Panel.
  2. Click on Hardware and Sound, then under the Devices and Printers heading click on Device Manager.
  3. Look for the Logitech mouse under Mice and other pointing devices.
  4. Right-click on the Logitech mouse and choose Uninstall. Click on OK.
  5. Restart your computer. You should see a notice from Windows that it has found new hardware. It should assign the generic driver to your mouse automatically. Then see if it works any better in PowerPoint.
