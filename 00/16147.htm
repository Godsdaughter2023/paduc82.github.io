ossiemac say:

I don't know why your particular line of code is failing. I have created a dummy workbook from the info you have provided and it works for me.

It is possible that you have a non visible character in there somewhere so try deleting the entire line (not just the code in the line but the entire line) and then re-type the line of code.

You can also try inserting the parameter "Destination:="  for the part that is to be pasted although it should work without it. 

Also a few tips for your VBA coding.

Try to use variables that will be meaningful so that when you have code with hundreds of lines it is easy to identify what the variable is used for. I would code something like the following where the worksheet variables and range variables are easy to identify to what they represent. The benefit of using the prefix ws for worksheet and rng for range is a debateable issue but it does ensure that reserved words are not used and I always use them. However, the rest of the variable should be identifiable as to what it represents and in the following example I have abbreviated the item they represent.

Sub PrntForm()

    Dim wsOrd As Worksheet, wsFrm As Worksheet, rngCust As Range, rngCell As Range
   
    Set wsOrd = ThisWorkbook.Sheets("Order")
    Set wsFrm = ThisWorkbook.Sheets("Form")
   
    Set rngCust = wsOrd.Range("Customer")   'Defined Range Sheet Form F16:F27
   
     For Each rngCell In rngCust
         If rngCell.Value = "A" Then
             rngCell.Copy Destination:=wsFrm.Range("E18")    'Get the customer name
             rngCell.Offset(0, -4).Copy Destination:=wsFrm.Range("E29")  'Get the Date
             wsFrm.PrintOut From:=1, To:=rngCell.Offset(0, -2).Value, Copies:=2, Collate:=False
         End If
    Next rngCell

End Sub

jay freedman say:
After you read the articles that Suzanne listed, you may understand better what a Userform is and how it works. (On the other hand, you may just be more confused. <grin>)

Basically, a Userform is a custom-built dialog box. It can have places for you to enter text, and/or checkboxes, lists to choose from, a calendar, and some other bits & bobs. It should also have the usual buttons for OK, Cancel, maybe Help -- whatever the job at hand calls for. The other part, sort of hidden behind the visible dialog box, is macro code that makes the whole thing operate. The same macro code can also do things to the document that was open when you displayed the Userform (and even to other documents, but that's a complication).

So what I'm suggesting is that you set up one template to contain all of these pieces:
  • A group of separate building blocks, one for each unique kind of report. Each one contains the bold headings (Procedure, etc.), each heading followed by a content control where you can type the details of the current report. (There are some additional details to know about; for example, you can tell a content control that its contents should be formatted with a specific style, which could specify numbering, indents, etc.) Each building block will also include markers of some sort to be replaced by the surgeon's name, the patient's name, or other data; the information needed to replace those markers will be entered in boxes on the Userform, and transferred to the document by the Userform's code.
  • At least one Userform. (Another complication: you could have a separate Userform corresponding to each kind of report, or there could be just one Userform that reconfigures itself as needed, depending on which kind of report you select from a list.)
  • An AutoNew macro, an AutoOpen macro, and probably some other macros to handle special tasks (no idea yet what those might be).
  • A button on the Quick Access Toolbar that runs a macro, which in turn displays the Userform(s). Whether there is one button, or one for each kind of report, depends on how you prefer to work. There can be a keyboard shortcut in addition to/instead of the button(s).
  • The body of the template can be completely blank, and each document based on that template will also start out blank. Alternatively, if you have a header containing the institution name, address, logo, etc., that could be in the template so it appears in each new document. Everything for each report, both the body and the footer, will be dropped into the document by the Userform's code.


The workflow would run something like this:

  1. At the start of the day, use File > New (or maybe a custom macro in your Normal.dotm template) to create a new document based on the report template. An AutoNew macro in the report template immediately displays the Userform to get started on the first report.
  2. On the surface of the Userform, you fill in any information for which you've provided entry boxes, including surgeon's name, patient's name, case ID, whatever else is useful (especially things that are repeated within the report or its footer). Then click the OK button.
  3. The code of the Userform prepares the basic report: (a) If this is not the first report in the document, insert a Next Page section break at the end of the document, break the link to the footer of the previous section, and insert the footer for the new section. If it is the first report of the day, it just has to insert the footer. (b) Insert the appropriate building block for this report type. (c) Replace each marker with the right data from the Userform entries. (d) Put the cursor in the first content control of the new section, then stop.
  4. You type in the contents of each part of the report, and delete any parts that aren't needed.
  5. You save the document. The first time each day, the document is unnamed, so you'll be shown the Save As dialog to choose the folder and enter the name. A macro in the template can intercept the Save command and suggest the correct folder and a filename based on the current date, which you can accept or modify.
  6. For the next report, if the document is still open, you click the button on the Quick Access Toolbar or press the shortcut to display the Userform again. If the document was closed, and you open the same document again, an AutoOpen macro in the template will run and display the Userform.


I hope that's starting to make sense. Again, if you want more extensive help with design and/or implementation, contact me by email through http://jay-freedman.info/contact/contact.htm.
