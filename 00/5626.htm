You will have alter the key bindings so when you press the enter key it will no longer do what it is suposed to do but what you want it to do instead.

Option Explicit
Sub TurnThisOn()
  CustomizationContext = ActiveDocument.AttachedTemplate

Sub AutoOpen()

End Sub
Sub AutoNew()

End Sub
Sub AutoClose()
  TurnThisOff
End Sub

ken sheridan say: The reason for creating a .accde file in the first place is to prevent users making design changes.  The original developer of the database should have split the file into separate front and back ends, and retained a version of the front end file in its original .accdb format.  If he or she did not do the latter, you will be unable to amend the report design, but if they did you should amend the design in the .accdb file and then create a new .accde front end from it, and then distribute copies of the amended front end to all users.

If the original developer did not split the file, which from your screen shot I suspect to be the case, then any original .accdb file's tables will not contain the current data of course.  In which case you should split the original .accdb file into front and back ends, for which you can use the built in wizard.  Then delete all tables from the back end which the wizard has created, and import all tables from your .accde file into the empty back end.  Place this back end file in a shared location on the system, to which all users have full access.  In the .accdb front end refresh the links to the back end with the built in Linked table Manager.  You can then make design changes to objects in the front end, before you create a .accde version of it for distribution to all users.

hal hostetler - mvp-outlook say:

What kind of file you get, .OST or .PST, depends on the type of email account that created it.  POP accounts create and use .PST files, IMAP, Outlook.com/Live mail/Hotmail, and Exchange Server accounts create .OST files.  It would appear that your email account was setup as a POP account prior to the crash, and is setup as an IMAP account now.  Your old .PST file cannot be used as primary storage with an IMAP account (you can't make it a delivery location).

That leaves two choices:

1. Re-create the account as a POP account, so you can re-use the old .PST file.

2. Keep the account setup IMAP, open the old .PST file using File > Open and Export > Open Outlook Data File, then copying any/all data you need from it into your IMAP storage file.
