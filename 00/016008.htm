In Windows 8/8.1/10, Windows Defender’s protection is fully automated. Defender’s real-time protection now continuously monitors file system activity and automatically scans files on-access. And Automatic Maintenance replaces the old scheduled scan by performing Quick Scans on a daily basis. There is an option to reschedule Automatic Maintenance: Right-click on the Start button > Control Panel > Security & Maintenance >Maintenance> Automatic Maintenance > Change maintenance settings.

But if you reschedule Automatic Maintenance for a specific time during the day, with the idea of actually running the Quick Scan at that time – then you’ll also need to take your coffee break at that time – because this scan gets cancelled whenever you start using your PC – and it only gets restarted when the system returns to idle. The prime directive for the Automatic Maintenance scan is strictly to avoid interfering with your work, and it completely sacrifices punctuality to that end.

Since “scheduling” the automatic scan only schedules the trigger-time, and not the actual run-time, it really doesn’t work very well for a scheduled scan. But if you just leave the scheduled time at its default setting, and turn your PC off at the end of the day, then Automatic Maintenance will be triggered shortly after startup every day; and it will be started whenever the system reaches idle. If you leave the PC on for a good part of the day, then Automatic Maintenance should be able to find a time slot where the system remains idle long enough to get the maintenance chores done.

On the other hand, if you're in the habit of turning off your PC when you’re not working, or if you'd simply prefer an old-fashioned kind of scan that starts on time and runs without interruptions, then you can still get that by setting up an independent scheduled task with the Create Basic Task Wizard:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-set-windows-defender-to-a-custom-auto/ec19aab4-10ce-4f41-9990-343056fd104f

robert aldwinckle on forums say:

IT person said that an add-on was probably interfering. 

Note the word "probably".  That is the usual case.  So usually a quick test using No Add-ons mode is enough to put you on the right track to diagnosing the cause of the symptom.   However, there are other installed programs, notably security and privacy related packages which are not installed as add-ons but do have just as much if not more interfering effect than incompatible add-ons.  After all, interfering is what they are designed for and capable of doing.  So, unfortunately, the next step in diagnosis is called clean-boot troubleshooting and again there is a quick test to help check if you would be on the right track but a more complete procedure will be required to either clearly identify a suspected cause or follow-up more intensively if the first test is inconclusive.  E.g. often the interfering effects of some packages persist even when they have been supposedly disabled or in some cases even after they have been supposedly removed (evidently incompletely).  In the latter case you can note some posters referring to (usually) downloadable cleanup procedures which seem needed to supplement deficient uninstall procedures.
