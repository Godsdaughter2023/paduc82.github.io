abdul raqeeb kola say:

Hi,

 

Code 39

Windows cannot load the device driver for this hardware. The driver may be corrupted or missing. (Code 39)

Cause

There are several causes of Code 39 errors:

  • A required device driver is missing.
  • A required binary file is corrupted.
  • There is a problem with the file I/O process.
  • A driver that is referencing an entry point in another binary file is unable to load.

 

Method 1:
I would suggest you to run the Hardware and Devices troubleshooter for Windows 8 and check for the issue.

Follow the steps to run the troubleshooter:

a. Press Windows key + C on your keyboard to show the Charms bar.

b. Type Troubleshooting and click Troubleshooting under Settings.

c. Now type Hardware Troubleshooter in the search option.

d. Click on Hardware troubleshooter to run the troubleshooter.

Note: Hardware and Device Troubleshooter checks for any issue on the hardware and devices connected to the computer.
Please refer to the link below:
Follow the section “Run the Hardware and Devices Troubleshooter”.
http://windows.microsoft.com/en-US/windows-8/what-device-isnt-installed-properly
 
Method 2:

If the above step does not help us to diagnose the issue, uninstall the drivers for USB [Universal Serial bus] host controller and restart the computer. Let Windows install the generic drivers and then see if it helps you to fix the issue.

 

a)     Press Windows key + R, type Devmgmt.msc.‌

 

b)     If you are prompted for an administrator password or confirmation, type the password or provide confirmation.

 

c)      Look for the device under USB [Universal Serial bus] host controller, right click on it and select Uninstall.

 

d)     Reboot the computer.

 

e)     On reboot, Windows 8 should re-install the drivers automatically.

 
Now try to install the drivers for your USB NIC.
 
Please let us know the results.
