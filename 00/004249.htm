Function Distance(Origin As String, Destination As String) As Double

' Requires a reference to Microsoft XML, v6.0

Dim myRequest As XMLHTTP60
Dim myDomDoc As DOMDocument60
Dim distanceNode As IXMLDOMNode
    Distance = 0
    ' Check and clean inputs
    On Error GoTo exitRoute
    Origin = Replace(Origin, " ", "%20")
    Destination = Replace(Destination, " ", "%20")
    ' Read the XML data from the Google Maps API

    Set myDomDoc = New DOMDocument60

stefan blom say:

billschuette say:

Yes, but that did not solve problem of not printing in Color.
After reading many other similar questions I found a method to solve it.

First "Close MS Word" and cancel any documents in printer queue.

1) Open Windows Taskbar Settings
2) Query for to find "Services (local)" page and Select 3) Scroll down to and highlight Name: "Print Spooler"
4) Check the "Status" column (mine said "Running")

5) Check "Startup Type" column (mine said Automatic)

6) Locate and left click on "Stop the service"
7) Check that "Status" now shows a blank space in place of "Running"
8) Locate and left click on "Restart the Service"

9) Check to ensure "Status" now says "Running" and "Startup Type" says "Automatic"

10) Close Taskbar and open MS Word again and try to print a color document

This worked for me and maybe it will work for others.

Mac Word can't do that.

  1. You need to compile your EndNotes at the end of the document,
  2. Then convert the EndNotes list to ordinary text (Command + Shift + F9)
  3. Then cut-and-paste to move it where you want it.

Do this last before submitting for publication, because once you convert the end-notes list it will no longer update.
