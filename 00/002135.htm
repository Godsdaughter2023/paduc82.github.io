MINIF, MAXIF and MODEIF with Array Formulas - Microsoft Community

jeeped say:

MINIF, MAXIF and MODEIF with Array Formulas

Summary
Technical Level: Intermediate Applies to: Excel, all versions This article will deal with adding conditions to some standard statistical functions for pseudo MINIF(), MAXIF() and MODEIF() formulas through combining each function with a nested IF() in an array formula.
Details
Array formulas are written into the cell or formula bar just as any standard formula but must be finalized with Ctrl+Shift+Enter rather than simply Enter. When entered correctly, Excel will wrap the formula with braces (e.g. { and }). You do not type these braces in yourself.

Entering a formula as an array formula produces cyclic processing. Some degree of care should be exercised in limiting referenced ranges to actual cells containing relevant data instead of full column references. Due to their cyclic processing, larger cell ranges in array formulas will increase calculation load exponentially.

 

My examples will use the following data matrix. Due to space limitations, a limited number of values will be used but statistical functions work better on larger data sets. MODE() for example will return an #N/A error if looking at a set of unique (no duplicate) numbers since by definition, it is looking for number duplication.

 

For the sake of these examples, we will use Apples as the criteria and I've applied some conditional formatting to highlight the rows that hold the Apple data.

 

For the MINIF() equivalent, we want to exclude any non-Apple quantities from consideration. An IF() function is nested into the MIN() calculation that will discard all non-Apple quantities.
       =MIN(IF($B$2:$B$99=$J3,$C$2:$C$99))
       =MIN(IF($B$2:$B$99=$J3,$C$2:$C$99))
After typing the formula in as above and applying Ctrl+Shift+Enter the formula will look like this in the formula bar:
       {=MIN(IF($B$2:$B$99=$J3,$C$2:$C$99))}
       {=MIN(IF($B$2:$B$99=$J3,$C$2:$C$99))}

This formula looks at the fruits in column B and compares them to the fruit listed in J3. If there is a match, the associated quantity in column C is considered in the MIN() calculation. Any quantity that is not associated with Apples is discarded as FALSE. Note that I have extended the cell ranges below the 15th row to cover future additions to the fruit/quantity data but not excessively.

 

For the MAXIF() and MODEIF() formula equivalents, we will duplicate the process to exclude any non-Apple quantities from consideration.
       =MAX(IF($B$2:$B$99=$J3,$C$2:$C$99))
       =MAX(IF($B$2:$B$99=$J3,$C$2:$C$99))
       =MODE(IF($B$2:$B$99=$J3,$C$2:$C$99))
       =MODE(IF($B$2:$B$99=$J3,$C$2:$C$99))
These formulas each require Ctrl+Shift+Enter.

 

When entered as an array formula the nested IF() function can quickly provide one or more conditions to base the parent function’s calculations upon. If more than a single condition is required, you can nest another IF() function. AND() and OR() functions do not work well within array formulas. For example, with the sample data above an extra condition could be quickly added to only consider the Apple quantities for the month of June.
       =MIN(IF($B$2:$B$99=$J3,IF($A$2:$A$99<DATE(2013,7,1),$C$2:$C$99)))
       =MIN(IF($B$2:$B$99=$J3,IF($A$2:$A$99<DATE(2013,7,1),$C$2:$C$99)))
In this new formula, the result would be 6.

 

Due to the cyclic nature of array formula calculations, each range must be the same size.

Generic detections are usually a Heuristics uses non-specific detection methods to find new or unknown malware which allows the anti-virus to detect and stop if before doing any harm to your system. Heuristic scanning methods vary depending on the vendor. Some claim to allow emulation of the file's activities in a virtual sandbox. Others scan the file more intensively, searching line by line inspecting the code in a file to see if it contains virus-like characteristics. If the number of these characteristics exceeds a pre-defined threshold, the file is flagged as possible malware. The disadvantage to using heuristics is that it is not as reliable as signature-based detection (blacklisting) and can potentially increase the chances that a non-malicious program is flagged as malicious. With heuristics, there is always a potential risk for a "false positive" when the heuristic analysis flags a file as suspicious or infected that contains no malware.

suzanne s. barnhill say:
To add to what Doug has said, if there has ever been text in the alternative headers, it will remain there even when the header is not in use. You may need to enable both Different First Page and Different Odd and Even in order to see them. Note that the "primary header" (the one used if you don't have either of the above options enabled) becomes the Odd Page Header when you enable Different Odd and Even, so there are really only three headers to deal with. If you have more than one section, you may well have three different headers in each. And, if a section contains fewer than three pages, you will need to temporarily add pages in order to display all the headers.

doug robbins - mvp office apps & services (word) say:

If by Header, you mean Header as in Header and Footer, it is probable that there are multiple Headers in the document.

The Headers can be different in each Section of the document and within a single section, you can have 

First Page Header

Primary Header

Even Page Header

Odd Page Header

You probably only deleted one of them.

If the above is the situation that you have, referring to them as Parent/Child headers is not correct.
