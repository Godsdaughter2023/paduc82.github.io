macropod say:

A paragraph break in Word is defined by a Cr-Lf pair (ASCII 13, 10), but VBA Range testing will find and treat them as a single character with a test for either VbCrLf or VbCr, for example. Using Word's Find function (with or without VBA), you can also find them via ^p or ^13.

Another character you might encounter is ASCII 11, which provides a manual line break within a paragraph. Using Word's Find function (with or without VBA), you can also find these via ^l or ^11.

In a similar vein, tabs exist as ASCII 09 or, using Find, via ^t or ^09. Do note that you can't use Find to locate table end-of-cell/row markers in tables.

Visually, tabs, line breaks, paragraph breaks ,and end-of-cell/row markers are represented in Word as  →, ↵, ¶, and ¤ symbols, respectively.

Option Explicit

Private Sub Workbook_BeforeClose(Cancel As Boolean)
    Dim FileToSave As Variant, Resp As VbMsgBoxResult
   
    If Not Me.Saved Then
        Resp = MsgBox("Do you want to save changes to '" & Me.Name & "'?", _
            vbYesNoCancel + vbExclamation, "Microsoft Office Excel")
       
        If Resp = vbCancel Then
            Cancel = True 'Abort the close
       
        ElseIf Resp = vbYes Then
            FileToSave = Application.GetSaveAsFilename( _
                FileFilter:="Excel Macro-enabled Workbook (*.xlsm),*.xlsm", _
                Title:="Save As")
           
            If FileToSave <> False Then
                Application.EnableEvents = False

                ActiveWorkbook.SaveAs Filename:=FileToSave, _
                    FileFormat:=52 'xlOpenXMLWorkbookMacroEnabled

                Application.EnableEvents = True
            
            Else
                Cancel = True 'User pressed Cancel in SaveAs dialog, abort the close
           
            End If
        
        Else
            Me.Saved = True 'User said No so close without saving
        End If
    End If
End Sub

What versions of Publisher and what OS did you create your old pub files with? If you created your old Publisher files on Win7 or older machines and are now using a Win10 machine you will find that test is rendered at a different size. This became an issue as of Win8 and the only workaround was to install Win7 fonts on the Win10 machine. You would notice your formatting is skewed when you open an older Pub file that was created on one of the older OS. This probably only accounts for some of your problems, but if it is the main one then one solution is to reinstall Pub 2007 if you have a copy and use it instead of Pub 2016. Users have reported that Pub 2007 and older are not subject to the same text rendering problem.

Just do a custom install and opt out of overwriting other versions. MSFT made it impossible to run any Office 2013 programs with Office 2016 so don't waste your time with that. FWIW I prefer the old style menu of Pub 2007 anyway. Plus MSFT started downgrading a number of things as of Pub 2010. I have Pub 2013 and 2010 but use Pub 2007 and 2000 for most of my production work.

While it would be great if you could copy and paste entire pages created from multiple versions of Publisher on multiple versions of Windows, there is no easy way. If you copy and paste just text then you will get the option of retaining the original formatting or change to the new formatting as per the new text box. You also can opt to use the options under Paste Special to retain formatting. However I expect you want all 85 pages to look the same which means you will need to tweak the formatting.

If it were me I think I would create a text box that has the correct font, font size, line spacing, etc and then right click and save as a Building Block. Now in any current or new document you can go to Insert > Page Parts and insert that newly created default text box into your document. Copy text from your old document > select you new doc and text box >Home tab > click the down arrow below Paste and choose Paste Special > Unformatted text and you will get the text formatted as per the new text box. You can also right click and under Paste Option choose A (keep text only) which is the same as unformatted text.

Also look for the Ctrl box attached to the text box after pasting. That will give you the option to retain source formatting or not.

Bottom line there is no easy way to do what you want...sorry. However if you have three publications that you want to merge into one PDF document with all formatting intact then download and install primopdf (free version) and it will allow you to append a pdf file. This means you would print your first publication to the PDF driver to your desktop or somewhere else you can find that file. Then open the next Pub file and print it to the same location and choose the same file name. Primopdf will ask you if you want to append that PDF or overwrite it. You can use primo for anything you can print from and I tend to use it more than the Office pdf tools.

Again sorry there is no easy way to workaround the issues you are having but hopefully I have given you a few ways to make it a bit easier.
