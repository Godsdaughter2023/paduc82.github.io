Disable Password Changing Option in OWA - Office 365 - Microsoft Community

swaminathaudiyapatham say:

Disable Password Changing Option in OWA - Office 365

Please find the knowledge sharing – Disable Password Changing Option for users.

 

Knowledge Sharing:

 

Customer wanted to disable the password changing option for the users at office 365 portal. According to the case customer wanted the users to have OWA access, however they shouldn’t have the option to change the password in OWA. Normally we recommend password never expire option, still the users can login to the portal and they can change  the password up to their preferences.

 

In order to disable that option we can follow the below steps using power shell – (Azure Active Directory Power shell)

 

First create a test user or test it one user at the customer environment.

 

Please run the below command to add a new Management role called “AllButPasswordChange”

 

New-ManagementRole -Name AllButPasswordChange -Parent MyBaseOptions

 

Now we will run the below command to see if the management Role is added to Management role Entry

 

Get-ManagementRoleEntry AllButPasswordChange\Set-Mailbox

 

Using the below command we will list out the different parameter categorized under this role

 

Get-ManagementRoleEntry AllButPasswordChange\Set-Mailbox | Select -Expand parameters

 

Once you run this command you will get the following output:

 

AcceptMessagesOnlyFrom

AcceptMessagesOnlyFromDLMembers

AcceptMessagesOnlyFromSendersOrMem

DeliverToMailboxAndForward

ErrorAction

ErrorVariable

ExternalOofOptions

ForwardingAddress

ForwardingSmtpAddress

GrantSendOnBehalfTo

Identity

Languages

MailTip

MailTipTranslations

MessageCopyForSendOnBehalfEnabled

MessageCopyForSentAsEnabled

OutBuffer

OutVariable

Password

RejectMessagesFrom

RejectMessagesFromDLMembers

RejectMessagesFromSendersOrMembers

RequireSenderAuthenticationEnabled

UserCertificate

UserSMimeCertificate

WarningAction

WarningVariable

 

Now we will go ahead and remove the highlighted password parameter from the list. In order to remove that specific parameter. Please do run the below command.

 

Set-ManagementRoleEntry AllButPasswordChange\Set-Mailbox -Parameters Password –RemoveParameter

 

Since we have created a new role assignment entry and we have removed the password parameter, everything is set to go, new we will go ahead and create a new role assignment policy in the name “AllButPasswordChange” using the below command.

 

New-RoleAssignmentPolicy AllButPasswordChange

 

Since we have to test it with one user we have to create new Management Role Assignment and we have to assign the AllButPasswordChange policy to that scope. So that it can be assigned to the user for testing. Please find the below command to assign the New policy to the new management role.

 

New-ManagementRoleAssignment -Role AllButPasswordChange -Policy AllButPasswordChange

 

Once you run this command you can find a new user roles created in portal. Please login to the portal go to Exchange Control Panel. Then check under permissions, you will find an option user roles. Once you click on user roles to find the new user role created for the users.

 

Now try to assign the same for one user with the below command.

 

Get-Mailbox "Venkat IT" | Set-Mailbox -RoleAssignmentPolicy AllButPasswordChange

 

Now please login to that user and you will see the change password options and if you click on it will take you to page cannot be displayed page and users won’t be able to change the password.
