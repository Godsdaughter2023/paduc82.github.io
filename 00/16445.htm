As for sequential songs, the easiest way would be to time out where you need the song to start playing and add it to that slide (for example, if the first song stops playing 3 seconds into slide 31, you should have the next song start 3 seconds into slide 31.

There is a catch to this method, in that CPU speed can have an impact on the timing of the slideshow in relation to your sound file. For example, on a slower computer your song might end on slide 29, but on a faster one it could end on slide 31, resulting in some dead time in your presentation. From my own experience, it seems that theactual duration of slide transitions depends on what types of transitions you are using, how memory heavy your slide is, and the overall speed of the computer you are using. If you plan on using a different computer for the actual presentation than you are for the it would be a good idea to test it out in advance so you can make any adjustments that need to be made.

It is almost never necessary to select the ranges in VBA; just reference the range as per the following code. Note that I have added optional code to unhide if cell is not equal to zero

Sub Hide_Col_Zero()

jeeped say:

... My question is how to pick and colour the lowest value from B1,D1,F1,H1 cells (alternate cells in a row) for entire 50 items ...  


The best method I can come up with is to create a Conditional Format rule using a formula.

First select the alternating columns. Press F5 to get the GoTo dialog and paste the following into the Reference: text box,
H1:H99,F1:F99,D1:D99,B1:B99
... then click OK. Your worksheet should look something like this:

                        

Note that I have hidden row 6 through row 44 in order to show the extents of the data range.

Now open Format, Conditional Formatting. When the Conditional Formatting dialog appears, change the first drop down in Condition 1 from Cell Value Is to Formula Is. Next type or paste in the following to the text box on the right,
=AND(ISNUMBER(B1),B1=MIN($B1,$D1,$F1,$H1))
Now you just need to complete the conditional format rule by selecting some formatting to distinguish the lowest values for each row. Click the Format button and select a new format. I selected a yellow background fill for my sample.
When you have completed your formatting selection and clicked OK, you should be back at the Conditional Formatting dialog.

            

The above image is a pretty close example of what your own should look like. Click OK to save your new rule. When you are back at your worksheet, select A1 to clear the multiple column selection. Your new rule should be in effect.

                     

Your own results should be similar to the above sample worksheet. Note that there is the possibility of two values being highlighted if they are both equal and the lowest values in their respective rows.

Yes, it sure is a bug.  Whoever spec'd / coded that functionality multiplied the current value of the 'axis crosses at' by the 'display unit' *each* time the file is opened (or at some point in the save-close-reoopen cycle).  The result is that each save-reopen multiplies the value of 'axis crosses at' by the display unit.

As you concluded, you cannot use the 'display unit' setting.  But...

You can do what I (and I imagine many others) did before Microsoft introduced the 'display units' setting.  Create a second column that contains actual-value / 1000 and plot this new column.  Now, you can get the desired de-congestion and the 'axis crosses at' will work as expected.
