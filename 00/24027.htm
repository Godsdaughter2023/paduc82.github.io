ken sheridan say: If you are inserting or deleting rows by some means other than in the subform itself, you must call the Requery method to include/remove the inserted/deleted rows.  The Refresh method only reflects changes to data in the subform's current recordset, whereas the Requery method reloads the recordset.  If you are inserting or deleting rows in code in the parent form's module for instance you'd follow this code with:

    Me.[NameOfSubformControl].Requery

where NameOfSubformControl is the name of the control in the parent form's Controls collection which 'houses' the subform.  The name might or might not be the same as the subform control's source form object.
