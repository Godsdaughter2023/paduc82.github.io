Update KB3081448 - Microsoft Community

ralph adelman say:

Update KB3081448

It appears that KB3081448 did not install when the last updates were available.  When I check for any updates, I get an error message 0x80096004.  Why won't this update install, since KB3081449 and the other update available with these two did?

a. user say:

Hi Ralph,

Welcome to Microsoft Community.

I understand the inconvenience you faced. I will certainly help you with this.

The reason behind this problem is a built-in service which is called "Windows Update". If this service is not running in background and if the service is set to disabled in Windows operating system, you'll always get this different error message while downloading ,updateing any software or upgrading your operating system.

I would suggest you to follow the below methods to fix issue.

Method 1. You need to re-enable "Windows Update" service and your problem will be fixed. Simply follow these simple steps to fix the problem:

 

1. Press Windows + R keys together to launch RUN dialog box. Type Services.msc in the textbox and press Enter. It'll open Services Manager.

2. Scroll down to bottom and you'll see a service "Windows Update" in the list. Double-click on it and set its Startup type to Automatic.

Method 2. Try to run windows update troubleshooter.

Follow the steps.

1. Go to the Control Panel then type in search box "Troubleshooting".

2. Then click on windows update.

3. Follow the on screen instructions.

Method 3.

I suggest you to manually reset the Windows Updates Components. Refer to the steps below to reset the Windows Updates Components:

  1. Windows Key + X on the keyboard and then select “Command Prompt (Admin)”      from the menu.

  2. BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                               net stop wuauserv

                               net stop cryptSvc

                               net stop bits

                               net stop msiserver

    SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                              ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                              ren C:\Windows\System32\catroot2 Catroot2.old

    BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                            net start wuauserv

                            net start cryptSvc

                            net start bits

                            net start msiserver

    Exit in the Command Prompt to close it.

I hope this helps.
Let us know the status of the issue so that we can assist you better.
 
Thanks.
