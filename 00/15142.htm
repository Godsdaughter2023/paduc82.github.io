john - project say:
JJ Donovan,
You won't be able to get your format with an built-in report or even with grouping, filters or modification of a view or report, but you still can get the report you want by using other methods.

You say you want the report by outline level but I think you mean by outline number. Assuming that's the case the report you want could be generated by collecting the resource assignment level for each resource at a given summary outline code, summing the values and then writing that result into the Notes field using VBA. I suggest the Notes field because resource names can sometimes be lengthy and attempting to use an extra text field (e.g. Text1), might cause the result to exceed the 255 character limit. The Notes field does not have that limit.

It is also possible to use a set of customized fields with formulas that parse the Resource Names data for each resource assignment. Those values can then be summed up to the parent summary line. The Notes field at summary level can then concatenate the summed up custom fields data to produce the format you want. A little work to set up but doable.

Another option is to export the Outline Number and Resource Names fields to Excel using the Export Wizard. You can then manipulate the Excel data to produce your report format.

Personally I'd chose the first option (i.e. VBA). It is the most versatile.

Hope this helps.

John

Update: I couldn't help myself. This code should do what I think you want. If not, at least I had fun.

Option Explicit
Sub ResourceReport()
Dim RAU() As Single 'array for res assign units at each summary level
Dim t As Task
Dim st As Task
Dim a As Assignment
Dim Rcnt As Integer
Dim Rptflag As Boolean
Dim i As Integer
Dim Nam As String

'set up array dimension
Rcnt = ActiveProject.Resources.Count
ReDim RAU(Rcnt)
'loop through all tasks looking for summary tasks that have
'   performance subtasks
For Each t In ActiveProject.Tasks
    If Not t Is Nothing Then
        'clear notes and loop through performance tasks to
        '   gather assignment unit levels for array elements
        '   for this parent task
        t.Notes = ""
        For Each st In t.OutlineChildren
            For Each a In st.Assignments
                Rptflag = True
                RAU(a.ResourceID) = a.Units + RAU(a.ResourceID)
            Next a
        Next st
    End If
    'if this summary task has performance subtasks, write concatenated
    '   summed up assignment level data from array into summary level
    '   Notes field
    If Rptflag = True Then
        'search array for non-zero values for concatenation
        For i = 1 To Rcnt
            If RAU(i) > 0 Then
                Nam = ActiveProject.Resources(i).Name
                t.Notes = t.Notes & Nam & "(" & RAU(i) * 100 & "%),"
                'once data is added to notes string, clear this array
                '   element value so it's ready for next summary level
                RAU(i) = 0
            End If
        Next i
        'remove trailing comma from string
        t.Notes = Mid(t.Notes, 1, Len(t.Notes) - 1)
        Rptflag = False
    End If
Next t
End Sub

It appears difficult to set up Office Outlook to achieve this.
I have set up each 'alias' as a POP3 account (using the 'alias' email name but with the main acoount login and password) but found that the inbox just pulls back the main Hotmail inbox and not the  'alias' folder. On the plus side  I can send messages from the 'alias' account.
I then tried the Outlook Connector. This pulls back the main inbox and the 'alias' folders, however it appears that replies are all sent through one email address - as defined on the website.
The only solution seems to be to use Outlook Connector for fetching the inboxes, but set up POP3 accounts for the send messages.This is very untidy and a bit unsatisfactory as just replying to a  message will use the default address NOT the alias address without manually changing it..

Nope, you've correctly evaluated your options. Configuring 1 "main" account via the Hotmail Connector and additional "send only" POP3 accounts for your aliases is indeed the way to deal with it.

 

It is indeed a bit untidy but Microsoft sees aliases more or less for receiving only. This is the same for Exchange accounts. They are aware of this feature suggestion though but no change has been announced yet.

 

You could register it as a feature suggestion yourself as well by calling Microsoft Support. You'll need to supply credit card information to register your call but it of course won't be charged for a feature suggestion.
