HTML <video> tag won't load a second video - Microsoft Community

steveh01 say:

HTML <video> tag won't load a second video

I have copied (basically from MS site) the following simple HTML code below. It runs fine on Windows 8 and Windows 10, but on Windows 7 (with IE11) the "Load Video" option - which attempts to replace the presently-loaded video file1 with file2 - gives an "Invalid Source" error in the player control and does not load the new video (file2). (It also runs correctly with Firefox.)

Now I know file2 is valid, because if I swap file1/file2 in the code below, file2 plays but the file1 fails with the same error when the Load Video button is pressed.

I have tried F12/Emulation and changed "Edge" to IE10/IE9, etc. but the same error still occurs. The console view indicates an error: MEDIA12899: AUDIO/VIDEO: Unknown MIME type. How can that be even though the FIRST mp4 file plays correctly?

Any ideas how to fix this? It would be greatly appreciated! Thanks.

Found the reason this was occurring: the .htaccess file did not include the MIME for .mp4 files After adding that on the server, the new video loaded correctly.

<!DOCTYPE html>

<html>
<body>

<button onclick="playVid()" type="button">Play Video</button>
<button onclick="pauseVid()" type="button">Pause Video</button>
<button onclick="loadVid()" type="button">Load Video</button><br>

<video id="myVideo" width="480" height="270" controls>
  <source src="<path-to-file1.mp4" type="video/mp4">
  Your browser does not support HTML5 video.
</video>

<script>
var vid = document.getElementById("myVideo");

function playVid() {
    vid.play();
}

function pauseVid() {
    vid.pause();
}
function loadVid() {
    vid.pause();
    vid.src="";
    vid.src = "<path-to-file2>.mp4";
    vid.load(); vid.play();
}
</script>

</body>
</html>
