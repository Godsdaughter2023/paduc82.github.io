365 Photos - Microsoft Community

philwarren2495 say:

365 Photos

I have added photos of staff to our Active Directory and have made sure they all meet the requirements of photos in 365. We have Directory Sync running but photos seem to be very hit and miss. Is there any way of forcing through the ad thumbnails to 365. Also i thought i had run a powershell script to disable users from placing there own photos into there profiles but it appears that some are still able to set there own photos.
alex sun msft say: Hi Phil,

The photo you upload to local AD cannot be greater than 96×96 pixels in resolution and 100 KB in size. In Office 365, the user photos are stored in Exchange Online. So please make sure your user has got an Exchange Online mailboxes. Meanwhile, you can check the directory synchronization client for any error logs. You can check if the thumbnailsPhoto attribute has got the binary value successfully in the Metaverse Search.

To disable end user's ability to change their photo, you can run the Exchange Online PowerShell command Get-OwaMailboxPolicy | Set-OwaMailboxPolicy -SetPhotoEnabled $false to modify the OWA policy. Therefore, they will not have the Change button under their photos.

However, at this moment, they are still able to change the photos from OWA Options->General->My Account. The user role MyBaseOptions grants users the ability to access and manage the mailbox related configurations including change user photo. So we need to remove the permissions from the role using the PowerShell command.

To avoid any potential issue, I suggest you create a new user role as the same as MyBaseOptions role, and remove the permissions by PowerShell. Here are the detailed steps you can refer to:

1.Connect PowerShell to Exchange Online, run New-ManagementRole -Parent MyBaseOptions -Name TestRoles //Create a test role
2.Get-ManagementRoleEntry "TestRoles\*" //View all the role entries
3.Remove-ManagementRoleEntry "TestRoles\Set-UserPhoto"
  Remove-ManagementRoleEntry "TestRoles\Get-UserPhoto"
  Remove-ManagementRoleEntry "TestRoles\Get-User" //Remove the entries from the test role.
4.Create a test user role assignment policy in Exchange admin center->Permissions->User role and add the test role to the policy (Please don't add the MyBaseOptions role).
5.Assign the test user role assignment policy to your user.

You can test the steps with a test user account. If they can work fine in your test, you can implement the policy on all the users. If you run into any problems, please let us know.

Thanks,
Alex

keith o connor say:

Admin access to all mailboxes

Hi, as an administrator I need access to all of the mailboxes in our organisation to be able to review certain emails etc. I know how to grant myself access via powershell and how to grant myself access to all - however when I do this it also add's those mailboxes into outlook which means I end up with over 800 mailboxes attached, and obviously that isn't usable.

Is there a way to grant myself permissions to access these mailboxes via OWA without having them appear in my desktop client?

Thanks!

alexsun_2012 say:

Hi Keith,

When we assign the Full Access permission to a user, Auto-Mapping is enabled as well. Auto-Mapping allows Outlook clients to automatically map to any mailbox to which a user has Full Access permissions. So to prevent your Outlook client openning so many user mailboxes, you can disable Auto-Mapping using the cmdlet Add-MailboxPermission -Identity TargetMailbox -User 'YourMailbox' -AccessRight FullAccess -InheritanceType All -Automapping $false after you connect PowerShell to Exchange Online. After you disable the Auto-Mapping, you are still able to access the user mailboxes in Outlook Web App.

If you need further assistance, please let us know.
