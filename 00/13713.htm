Multi-part question

#1 - Backing up your data

There are a variety of ways to go about this and nothing has changed except IMAP email accounts now use an .ost file for email data in OL'2013/'2016 whereas OL'2010 and earlier used a PST file

a) close Outlook and simply copy any relevant PST file (if applicable or use the Outlook export function for any email folders contained in an .ost file

b) use either the free or enterprise version of SafePSTBackup (enterprise version allows scheduling and converts .ost files to .pst files for Imap, Outlook.com and Exchange accounts among other things)

#2 - Reading PST files without Outlook

A PST file is proprietary file specific to Outlook so either the PST file you want to read has to be added to Outlook OR you will need a PST previewer (which doesn't require Outlook) of which there are many - free and paid. Just do a search using the phrase "Outlook PST Viewer" (or anything similar) and there should be no shortage of results

How to: Switch from 32 bit versions of Windows to Windows 10 64 bit

Created by Andre Da Costa
Technical Level : Intermediate Summary
As with previous releases, Windows 10 will be available in 32 and 64 bit architectures. Windows 10 64 bit comes in handy when you need to address at least 4 GBs or more of RAM. In fact, Windows 10 64 bit supports up to 512 GBs of RAM, while Windows 10 32 bit can utilize up to 3.2 GBs of RAM. Because the memory address space is much larger for 64 bit Windows, that means, you need twice as much memory than 32 bit Windows to accomplish some of the same task, but you are able to do so much more, you can have more applications open, do things like run an Antivirus scan in the background without it affecting your system performance. Windows 10 64 bit is more secure too, malicious code cannot easily infiltrate it, drivers are more reliable since they must be signed before they can work with 64 bit Windows 10.

Compatibility is a critical consideration if you decide to switch to 64 bit Windows 10, you will need 64 bit device drivers for any hardware devices you might have. Also, there is no 16 bit subsystem in Windows 10 64 bit, which means, your applications must be 32 bit or 64 bit only, no 16 bit installer or uninstallers. Also, if you decide to move to Windows 10 64 bit in the future, there is no upgrade path from 32 bit Windows, clean install only.

Is my computer 64 bit capable?

Before you can start the process to migrate to 64 bit Windows, you need to find out if your system is 64 bit capable. If you are already running a 32 bit version of Windows 8 or later on your system, you can find out by doing the following:

Press Windows key + X on your keyboard

Click System

Look under System > System type:

There it will list whether the processor is x64 based processor or not. In my case I have 64 bit capable processor. Once you have confirmed this, you can move on to the next step.

For Windows 7 and earlier releases:

You can determine if your processor is 64 bit capable by using a Instructions field, look for the instruction set called AMD64 or EM64T. If you see them, then your processor is 64 bit capable. If you are running a 32 bit version of Windows, whether Windows 8 or Windows 7, you need to consider carefully the steps to migrate from it to 64 bit Windows 8. Regardless of the ubiquity of 64 bit computing, key factors such as hardware and application compatibility remain a issue, especially for legacy systems.

Please note, some older 64 bit processors might not be capable of running 64 bit Windows 10.

To install Windows 10 on your PC, the processor (CPU) must support the following features: Physical Address Extension (PAE), NX, and SSE2. Most CPUs have support for these features, so if you receive this error, it is likely because the NX feature is not enabled on your system.

To resolve this error, follow manufacturer guidelines to enable NX (“No eXecute bit”), or the equivalent XD (“eXecute Disabled”), feature within the BIOS settings. This feature is typically found in the Advanced or Security tabs within the BIOS settings, and can be referred to by a variety of names, including but not limited to:

  • No Execute Memory Protect
  • Execute Disabled Memory Protection
  • EDB (Execute Disabled Bit)
  • EVP (Enhanced Virus Protection)

If the BIOS setting for the NX (XD, EDB, or EVP) support option is not available on your system, you may need to contact the manufacturer to update the BIOS. Note that some very old processors may not contain these features and will be incompatible with Windows 10 64 bit. The solution is to use 32 bit Windows 10 even if your computer is currently running a 64 bit version of Windows or stick with that version or purchase a new device with Windows 10 64 bit.

jay freedman say:

It certainly is possible to create a template like that. At the simplest, you'll need these parts:

  • In the body of the template, insert DOCVARIABLE fields as placeholders for the information that has to change for each client. For example, you'll want fields like {DOCVARIABLE "FirstName"}, {DOCVARIABLE "LastName"}, and {DOCVARIABLE "StreetAddress"}. If the same information should be in two or more places, just repeat the same field. Fields can also go in headers and/or footers. When you've finished, apply forms protection to prevent editing. Save the template as a macro-enabled (*.dotm) file.
  • In the macro editor, insert a userform and a module. Their default names will be UserForm1 and Module1, but it's good practice to change those names.
  • In the graphic view of the userform, add labels, text boxes, buttons, and whatever else you need to create a form for entering the necessary data. There should be one place to enter each of the pieces of data you named in the DOCVARIABLE fields. There should also be an OK button and a Cancel button.
  • In the code view of the userform, write code in the OK button's Click procedure to take the contents of the text boxes and other data entry controls and store them in document variables (that is, members of the document's .Variables collection) with the same names used in the DOCVARIABLE fields. At the end of the procedure, you'll need to unprotect the document, update its fields, and restore the protection. A very simple example (tbxFirstName is a text box for entering the first name, and tbxLastName is a text box for the last name):

Private Sub cmdOK_Click()
    With ActiveDocument
        .Variables("FirstName").Value = tbxFirstName.Text
        .Variables("LastName").Value = tbxLastName.Text
        If .ProtectionType <> wdNoProtection Then .Unprotect
        .Fields.Update
        .Protect Type:=wdAllowOnlyFormFields, NoReset:=True
    End With
    Unload Me
End Sub

  • In the module, write a macro named AutoNew -- that's a special name that tells Word to run it each time you create a new document based on this template. The job of that macro is to display the userform and run its code. If you've changed the name of the userform from UserForm1 to something else, you also have to change that name in this macro.

Sub AutoNew()
    Dim dlg As UserForm1
    Set dlg = New UserForm1
    dlg.Show
    Set dlg = Nothing
End Sub

The userform code can become much more complicated, depending on what you want it to do. For example, it can check for data that hasn't been entered or that may be incorrect; it can pull data from an Excel worksheet or other source; and it can save the document with a specific name in a specific folder. If you want to pursue that, look at http://gregmaxey.mvps.org/word_tip_pages/create_employ_userform.html.

One more thing: If this seems like too much complication, you can stick with using mail merge. Just save the merge main document and reuse it when making a new document -- there's no reason you can't run a merge with only one record.
