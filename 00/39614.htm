vijin v say:

Hi Paul,

 

The SQL Server “version” is often called the “patch level” of SQL Server. In each major or minor release of SQL Server, there is an increment to the version number to differentiate it from earlier versions. This change to the version is used for many purposes. This includes displaying version information in the user interface, controlling how files are replaced during upgrade, applying service packs, and also as a mechanism for functional differentiation between the successive versions.

 

As well as few programs are integrated on a specific SQL version.

 

When you install different versions of SQL Server side-by-side on the same machine, these components are automatically upgraded to the latest version. Such components are usually uninstalled automatically when the last instance of SQL Server is uninstalled.
