greginmich say:

Start by excluding the executable file that loads the 7-Zip program with a process exclusion. The “process exclusion” was specifically designed to work around this type of issue by preventing MsMpEng.exe from doing its normal real-time scanning of the files accessed (read or written) by a process (program). Although this should theoretically solve the problem, there have been quite a few reports of run-away real-time scanning, as Steve notes, so the issue might be more complex.

 

You will also probably want to exclude both the uncompressed and compressed versions of this large file with file exclusions, so that on-demand and scheduled scans don’t get bogged down. Do the following:

In this example we sum a range of A1:An where the row number for n is in b3

rblack40 say:

I did not name anything so the result in this should have come out B3 and not 2.888. This is incorrect in Excel. B3 has 2.888 but it also should be a named range per cell. Once the cell is named then you would type the INDIRECT function in cell A1 and use the appropriate name to get the circular reference and the mixed result of both B3 and 2.888.

 

andrea jones - all about resources say:

You do not need to use range names with the INDIRECT() function. The whole point of INDIRECT() is that it can take a string (or cell reference containing another cell reference) and give the value of that cell or range (as in Mike's SUM() function example above). I don't have the book you are referring to but I looked at that section on Google books and it was demonstrating exactly what I described above (although I don't think it tells you which cell to
put the INDIRECT() function in).

The process my example above goes through is:

=INDIRECT(c6) is entered in cell

Therefore Excel looks in cell C6 and sees the contents B3. This is treated as a cell reference so the function in cell A1 becomes

=INDIRECT("B3")

This makes Excel look at cell B3, the contents of which are 2.888 so the function returns the result 2.888 which is correct.

Mike's example is talking about a different situation but also uses the cell reference B3 (don't confuse this with the use of B3 in my example). In Mike's example a user can enter a finish row number in cell B3 to sum a range. You can pick up this value and concatenate it on to the string "A1:A" (assuming the range started on row 1), e.g. if the value in cell B3 were 10 then the concatenation "A1:A"&B3 would result in "A1:A10". This then passes the range to the SUM() function giving the desired result.

If you want to use a named range in an INDIRECT() function then you enclose it in quotes e.g. =INDIRECT("MyNamedCell") would give the value 2.888 if cell B3 had the range name MyNamedCell.  I think you are confusing yourself, talking about range names, circular references, etc.  Have another look at my example and then progress to Mike's to see how INDIRECT() can be useful when you need to construct a cell reference from other values.

mike h.. say:
Explain the AREAS function.


Hi,

 

An area is a range of contiguous cells so the formula

 

=AREAS(A1:A10)

 

returns 1 because A1:A10 are a single block of cells

 

The formula

 

=AREAS((A1:A10,C1,F1))

 

returns 3 because of the 3 non-contiguous cell. Note also that in this syntax we need an additional set of parenthesis

A1:A10 is a single block of contiguous cells i.e. no gaps so that's 1 AREA.
