Footnotes size - Microsoft Community

dianacarolinareyes say:

Footnotes size

Hello, 

I need to change my footnotes from superscript to regular number but JUST in the footnote NOT in the text. Is there a way to do that ?

jay freedman say:

Select any one of the footnote numbers in the footnotes. Press Ctrl+Shift+S to open the Apply Style box; the style name shown there will be Footnote Reference. Click the Modify button.

In the Modify Style dialog, click the Format button in the bottom left, and click Font in the menu that pops up.

In the Font dialog, clear the check box for Superscript and click OK.

Back in the Modify Style dialog, if you want this change to apply to all documents that you create in the future, click the option button for "New documents based on this template". This won't affect any existing documents other than the one that's currently open.

Click OK. All the footnote numbers in the document will immediately change from superscript to normal size.

stefan blom say:

Of course, if you modify the Footnote Reference style, that modification will be applied to reference numbers in the footnote area AND in the main body of the document.

To remove superscript in the footnote area only, you can do the following: On the View tab, click Draft, and then click the Show Notes button on the References tab. Word will display all footnotes in a separate pane at the bottom of the document window. Click in that pane and press Ctrl+A to select all the text. Press Ctrl+SpaceBar to clear all direct formatting from text. This works as long as you don't have a lot of direct formatting (such as bold or italic) to text in the footnote pane.

jay freedman say:

Thanks Stefan. My mistake.

If the text of the footnotes frequently contains directly formatted text or text to which a character style such as Subtle Emphasis or Strong was applied, then Ctrl+A, Ctrl+spacebar will remove all of that formatting. If you can't use a macro, you're stuck with selecting each footnote number in the panel -- individually, or all the ones visible by holding down Ctrl and selecting others with the mouse -- and pressing Ctrl+spacebar.

Sub UnsubscriptFootnoteNumbers()
    Dim rg As Range
    
    Set rg = ActiveDocument.StoryRanges(wdFootnotesStory)
    With rg.Find
        .Style = ActiveDocument.Styles("Footnote Reference")
        .Wrap = wdFindStop
        While .Execute
            rg.Font.Reset
            rg.Collapse wdCollapseEnd
        Wend
    End With
End Sub

greginmich say:

The only way to prevent unsuspecting people from being victimized by these Tech Support Scams is to minimize their chances of encountering them. This would ideally be done with the Windows Defender SmartScreen filter on Microsoft browsers (and potentially with the new Windows Defender Network Protection feature on non-Microsoft browsers) – and also by keeping high-profile websites free of malvertising. But the scammers are easily able to outpace the SmartScreen filter by simply registering new domains at a lightning pace – and we currently have unresolved malvertising issues on some very high-traffic sites that are piping in these Tech Support Scam redirects.

At the same time, the scammers have pretty well managed to bypass Edge’s Dialog Loop Protection mechanism, and have forced us to revert to using Task Manager and the indirect restart trick to remove Tech Support Scam pages – and this leaves Edge users who haven't learned this trick susceptible to the scam. So I think that now would be a good time for shifting the focus from escaping these Tech Support Scam pages, to preventing them.

The current malvertising campaign on the MSN news portal has given me the opportunity to see firsthand how a good ad-blocker extension, like uBlock Origin, will selectively prevent compromised advertising domains from connecting to the host site – and will thus prevent these Tech Support Scam redirects from being piped in.

Installing an ad-blocker extension in Edge is safe and easy, because the Settings menu will take you directly to the Microsoft Store, where you’ll find a good selection of ad blockers to choose from. I’m very impressed with the effectiveness of uBlock Origin in mitigating this issue, and also with the way it gives us such a clear window into the extensive domain connection activity that’s going on behind the scenes at all of the high-profile sites. But others have reported success with AdBlocker Ultimate, which is also very well-reviewed. And of course, these extensions are all free, so if you’re unhappy with the first one you pick, you can just switch over to another one:

For Microsoft Edge, find an ad-blocker extension in the Microsoft Store:

Settings and more > Extensions > Get Extensions from the Store

The documentation for uBlock Origin (and the instructions for installing it on other browsers) is available here:

https://github.com/gorhill/uBlock

Microsoft Logon should be non-administrator ONLY - Microsoft Community

big bad bombastic bob say:

Microsoft Logon should be non-administrator ONLY

Although I'm not a fan of the 'Microsoft Logon', I can see how a LOT of people might want this to simply device data synchronization and access the "Windows Store".  OK, I get it, there are some people who WANT this.

But regardless of how any of us want or do NOT want a 'microsoft logon' every time we use our computer, the fact remains that the VERY FIRST LOGIN YOU CREATE has ADMINISTRATOR PRIVILEGES ON YOUR MACHINE!

This is SO bad for SO many reasons.

A few years ago I had to clean some malware off of a Vista machine that had been infected with some kind of 'scareware' application through a 0-day vulnerability, apparently in a graphic that had been uploaded to a popular web site (I reported it, it was taken down shortly after).  One thing I had sort of *insisted* upon for the owner of this machine (who agreed with me) is that "all web surfing is done from a non-admin login".  As a result, the 0-day malware did NOT take over the entire computer.  A power down/reboot to the administrator login allowed me to EASILY remove the malware (after going to a web site to figure out what files were involved), and after a couple of manual registry repairs to the user's registry, it was fixed.  Not much hassle, really.  It could have been a *LOT* worse.  A *LOT* worse.  If an admin-level user had been infected, the ENTIRE COMPUTER would have been compromised.  Repair would have been a HORRIBLE MESS.

THAT being said, I believe THIS security model is the BEST for ANY windows machine, for the above example, and because of my 'non windows' experience, where running as 'root' is DEFINITELY frowned upon.

The problem in the PAST, and XP was a good example, is that THE SOFTWARE you run does not necessarily behave well if you don't have administrator privileges, so UNFORTUNATELY a LOT of end-users have gotten into the *BAD* habit of running as ADMINISTRATOR, even though Vista, 7, and later have made it VERY practical to NEVER run as an administrator.

matthewkmiller say:

Amazon has their SPF records that list a pretty large number of IP addresses their emails can come from. This allows email administrators to block emails from Amazon that come from other IP addresses since they are not legitimate messages sent from Amazon.

Somewhere in the forwarding you have set up, the email is gaining a new source IP address. Then, when outlook.com does an SPF lookup, it sees the list of IPs that emails from Amazon are allowed from and see that the source IP address of this email is now the mailspool.net IP address.

The problem is definitely a disagreement between mailspool.net's IP address and your SPF record. They are giving the message a new source IP address that is not in Amazon's SPF record. This is something that mailspool.net should fix as they are the ones giving the email a new source IP address.
