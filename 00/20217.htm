This isn't perfect, but it might do. Type 235U92 into the cell. Turn the cell's "Wrap text" on. Put the text cursor between the 5 and U and press ALT+Enter to put them on two line. Select the 235 and make it Subscript. Select the 92 and make it Subscript also. Select the U and make it Superscript and also increase its Font Size a lot (maybe double what the cell's normal Font Size is). Now, go back and put the text cursor in front of the 235 and type several spaces. You will have to play with the number of leading spaces (which will be dependent on the Font Size you choose for the letter U) and the size of the U, but eventually you should be able to get a decent looking layout.

scottgem say:

My problem is that I need to account for all 4 fields (once combined) to be the primary key 

 


No you don't! The only purpose of a primary key is to uniquely identify a record. I think what you want to say is that you need the combination of the four fields to be unique in your table. This is a different thing. I do not recommend composite keys. Instead you should use an autopnumber as your primary key and create a multi-field unique index on the combination of the four fields. This way makes it easier to join your table to other tables, yet still maintain the uniqueness of the combination of fields.
