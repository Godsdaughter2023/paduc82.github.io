Enter the following in a normal module
======================================
Sub callUF()
    UserForm1.Show vbModeless

Enter this code in the ThisWorkbook module
====================================================
Private Sub Workbook_SheetActivate(ByVal Sh As Object)
    UpdateUF Sh
End Sub


Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
    UpdateUF Sh
End Sub

    UserForm1.Label1.Caption = WorksheetFunction.Sum(Sh.Range("A:A"))

====================================
Private Sub UserForm_Initialize()
    UserForm1.Label1.Caption = WorksheetFunction.Sum(ActiveSheet.Range("A:A"))

christopher miller say:

Microsoft has an SDK, with libraries for Windows Phone, Android, and iOS.  You would write an app that would subscribe to the heart rate sensor.  Your app would receive notifications when the heart rate changed.  Right now, the heart rate sensor is the only sensor that requires the user to consent to having the data collected.


For more information about the SDK and example code, please see http://developer.microsoftband.com/.  For developer support, you'll want to use the microsoft-band tag on Stackoverflow,

hallmtn say: Thanks Christopher.  Currently I am prototyping the decision support tool in Excel for Mac.  I use a web API with HTTP POST or GET methods to fetch live data in Excel.  Can I access the Band SDK from Excel for Mac?  Is there a Band web API?

christopher miller say:

You can't access the Band SDK from the Mac.  The SDK is for apps running on Windows Phone, Android, and iOS.  You would need to write an app for the phone that listened to Band sensor notifications and the phone would push it to your web api

There is a Web API of sorts, but it only lets you pull information down to the Band, you can't push information (like heart rate) back up.  It's called Web Tiles and is in public preview.

antonbe say:

Ok, I have managed to fix the problem. 

What did not work for me (but may work for others):

  • Rebuilding the launch services database / clearing icon cache 
  • Repairing disk permissions (using disk utilities)
  • Repairing disk (using disk utilities)
  • Reinstall office 
    This got it even worse, the icons of all other office applications broke as well

What did work for me:

  1. Download LiteIcon and open it
  2. Download an icon, such as this office icon or any other png file. In the end it doesn't matter. 
  3. Within LiteIcon, under system icons click "Others"
  4. Drag the icon you have downloaded in step 2 onto the Document icon
  5. Click Apply Changes 
  6. The program will close and you will be signed out, sign in again and all office icons should work as normal. Funny thing is, it does not use the icon I assigned to it in step 4. Instead it just uses the correct office icons. I assume LiteIcon just somehow fixes the icon cache on my mac.
