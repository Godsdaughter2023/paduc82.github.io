allen305 say:

As I mentioned in my previous post, the System Event Errors are occurring on 3 Windows 10 Pro PCs ( 1 32 Bit and 2 64 Bit). Sorry, but I find it hard to believe that I have 3 corrupted user profiles. This reminds me of the Windows 10 Pro v1511 update where networked PCs disappeared  and regedit failed.  Microsoft Support had persons like me jumping through hoops for problems created by the Windows Update.

After many web searches, I found a suggested solution as follows:

Open regedit,and navigate to the following key:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\SubSystems\Windows

Find the following value: "Windows SharedSection=X,Y,Z".

Where X,Y and Z are the values you'll found there.

The Desktop Heap Is the the "Y" value.

Double this value, and then reboot the box.

It turns out that too many interactive services are running on the machine, since this services allocate some from the Desktop heap, when it becomes depleted the Win32K will starve for memory even with plenty of memory and disk space.

I have tried the above changes on one of my 64 Bit Windows 10 Pro PCs. The System Event Errors, Event Id 10010, are no longer in the Event Log. If I do not see the Event Id 10010 on the aforementioned PC, I will apply the changes to the remaining computers.  Unfortunately, I do not know who to credit for this solution. Thanks to whomever :-)
