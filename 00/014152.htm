ken sheridan say: If you wish to enter one or more instructions which relate to the current record in the form, then you can include a field of 'long text' data type (formerly known as a 'memo' field)  in the table to which the form is bound and bind the text box to that field.  By default the EnterKeyBehavior property of a text box control bound to this type of field is set to start a new line in the field, i.e. it enters a carriage return/line feed.

However, this is not a good design in a relational database as the field is being used inappropriately as a data structure.  The correct way would be to have a separate Instructions table in which there is a foreign key column which references the primary key of whatever table the form is based on.  It would also include a text column to hold the instructions.  Instead of a text box in the form you'd then use a subform based on the Instructions table and linked to the parent form on the primary/foreign key fields.  The subform would be in continuous forms view and include only one control, a text box bound to the text field in which the instructions are entered.  as few or as many instructions per parent record can then be entered simply by inserting a record in the subform for each.

ken sheridan say:

I'd agree with Karl that the way to handle this is to date/time stamp an employee when called.  I think the process can be simplified, however, removing the need for an additional table.  You don't say how your list of available employees is built so I'll assume a table Employees which includes columns EmployeeID, FirstName, Lastname, PhoneNumber and OTAvailable, the last being a Boolean (Yes/No) column which indicates an employee's availability for overtime.  Add a LastCalled column of date/time data type to this table and fill it initially with an artificially early date/time value (1 January 2000 say) with the following update query:

 

UPDATE Employees

SET LastCalled = #2000-01-01#;

 

The RowSource for a combo box from which to select an employee would be:

 

SELECT EmployeeID, PhoneNumber, FirstName & " " & LastName

FROM Employees

WHERE OTAvailable

ORDER BY LastCalled, LastName, FirstName;

 

Its other properties would be:

 

Name:  cboEmployee

BoundColumn:   1

ColumnCount:   3

ColumnWidths:  0cm;0cm;8cm

 

If your units of measurement are imperial rather than metric Access will automatically convert them.  The important thing is that the first two dimensions of the ColumnWidths property are zero to hide the first two columns.

 

Add a text box to the form with a ControlSource property of:

 

=cboEmployee.Column(1)

 

The Column property is zero-based so this will show the phone number of the selected employee.

 

When the currently selected employee is called the current date/time can be assigned and the employee moved to the bottom of the list by executing the following code, e.g. from a button on the form:

 

Const MESSAGETEXT = "No employee selected."

Dim strSQL As String

 

strSQL = "UPDATE Employees " & _

    "SET LastCalled = #" & FORMAT(NOW(),"yyyy-mm-dd hh:nn:ss") & _

    "# WHERE EmployeeID = " & Me.cboEmployee

 

If Not IsNull(Me.cboEmployee) Then

    CurrentDb.Execute strSQL, dbFailOnError

    Me.cboEmployee = Null

    Me.cbEmployee.Requery

Else

    MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"

End If
