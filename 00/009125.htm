bernie deitrick say:

There are a lot of specifics that you don't address (like ranges, names of sheets and folders, etc.) but the macro is fairly straight forward. I have assumed:

1) The name of the subfolder you mentioned can be determined through date logic - you need to replace DateLogicForFolderNaming with the actual logic, and "yyyy-mm-dd" with the date format.

2) The sheet with the data in each file is named "Time", and the data starts in cell A1, and column A is completely filled.

3) The folders are stored in H:\TimeSheets

4) The employee name is part of the file name - which is what I am storing in column E to identify the employee

5) You want values and not formulas, and the formatting in each column is already in place.

Sub ConsolidateTimeSheets()
    Dim strDate As String
    Dim strPath As String
    Dim strFileName As String
    Dim wkbkSummary As Workbook
    Dim wkbkTemp As Workbook
    Dim strSName As String
    
    strSName = "Time"
    
    With Application
        .DisplayAlerts = False
        .EnableEvents = False
        .ScreenUpdating = False
    End With
    
    
    strDate = Format(DateLogicForFolderNaming, "yyyy-mm-dd")
    
    strPath = "H:\TimeSheets\" & strDate & "\"
    strFileName = Dir(strPath & "*.xlsm")
    Set wkbkSummary = Workbooks.Open(strPath & strFileName)
    
    With wkbkSummary.Worksheets(strSName)
        .Range("E2:E" & .UsedRange.Rows.Count).Value = strFileName
    End With
    
    strFileName = Dir()
    
    Do While strFileName <> ""
        Set wkbkTemp = Workbooks.Open(strPath & strFileName)
        
        With wkbkTemp.Worksheets(strSName)
            Intersect(.Range("A:D"), .Range("A1").CurrentRegion).Copy
        End With
        
        wkbkSummary.Worksheets(strSName).Cells(Rows.Count, "A").End(xlUp).Offset(1, 0).PasteSpecial xlPasteValues
        
        With wkbkSummary.Worksheets(strSName)
            .Range(.Cells(.Rows.Count, "E").End(xlUp)(2), .Range("E" & .UsedRange.Rows.Count)).Value = strFileName
        End With
        
        wkbkTemp.Close
        strFileName = Dir()
    Loop
    
    wkbkSummary.SaveAs strPath & "Consolidated time file.xlsm"
    
    With Application
        .DisplayAlerts = True
        .EnableEvents = True
        .ScreenUpdating = True
    End With
End Sub

Installing from the command line often works for me where double-clicking the installer won't. It may sound intimidating but it's really quite simple.
  1. Open the Terminal application found in /Applications/Utilities.
  2. Enter the following into the window: "sudo installer -pkg ". Note the trailing space.
  3. Drag the Office Installer.mpkg file into the Terminal window. This will fill in the location of the file.
  4. Finish the command with: " -target /".
The whole line will look something like "sudo installer -pkg /path/to/Office Installer.mpkg -target /".

Hit the return key and you'll be prompted for an administrator name and password. After you enter your credentials you'll see the Terminal work away. Hopefully, the last line will read something like "installed successfully".

Hope this helps!

marshall barton say:

You are supposed to use the ORDER BY clause to specify a tie breaker field.  If you really don't care which record wins the tie, use a unique field (e.g. the primary key) or maybe a date/time field.  Your TOP 10 query does not have enough fields to do that so you may be stuck mucking about in the report such as Hans described.

BUT, your base query might generate incorrect results.  This query does not use any aggregate functions (Count, Sum, etc), so you should not have a Group By clause.  You should turn off the Totals option and maybe(?) set the Unique Values property to Yes.

The Having clause should only have calculated fields that use an aggregate function.  If you continue to use the Totals option (adds the Group By clause) even when you do not have any fields like that, then, in spite of what the stupid query designer does with criteria, you should switch to SQL view and change HAVING to WHERE to correct the query.

hansv mvp say:

If there are ties for the last place, a "Top n" query will return all ties - it has no way to decide which one(s) should be included and which one(s) should be excluded.

You can force your report to include only 10 records even if the query returns more:

Place a text box in the Detail section.

Set its Visible property to No.

Name it txtCounter.

Set its Control Source to =1.

Set its Running Sum property to Over All.

Create an event procedure for the On Format event of the Detail section of the report:

Private Sub Detail_Format(Cancel As Integer, FormatCount As Integer)
  Cancel = (me.txtCounter > 10)
End Sub
