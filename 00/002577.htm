suzanne s. barnhill say:

Unfortunately, it is not possible to do this in Word, which is word processing rather than page layout software. Any graphic that is "wrapped" (that is, text wraps around it), must be anchored to a text paragraph on the same page. You can insert a graphic and anchor it to a caption, but that caption must still be part of the flow of text. That being the case, it makes more sense to make both graphic and caption part of the flow of text. Here's how to do it:

Don't even try to insert (or place) graphics until editing is complete and you know that the text and pagination will not change. You can insert a placeholder ("Figure 1 goes about here") or insert the graphic itself in a temporary location, but don't bother with final formatting.

Once you are ready to place all the graphics, insert a page break after the last line of text on the page that will precede the illustration. Insert the graphic and caption and another page break.

If the page before the illustration conveniently ends with the end of a paragraph, or if your paragraphs are not justified and don't have a first-line indent, then you're done. In most cases, however, a little more work will be required to make the text appear continuous from the page preceding the illustration to the one following it.

At the end of the last line before the first page break, insert a line break (Shift+Enter). This will justify the line. It will also likely push the paragraph break to the next page. Select just the paragraph mark (for this you need to have nonprinting characters displayed; Ctrl+*) and format it as 1 point. If that does not suffice to keep the blank line on the same page, you will need to format it as Hidden (Ctrl+Shift+H); when you now hide nonprinting characters, the extra line should disappear.

On the page following the illustration, if your paragraph style begins with a first-line indent, you will need to remove it. This is best done by applying an unindented style. I use Body Text (unindented) and Body Text First Indent for these purposes.

rob koch say:

You don't recommend changing the registry, but what do I do about all the broken registry items on my computer? Don't they cause errors? I have several registry items on my computer for programs that I removed a long time ago. I thought that having all these broken items can cause errors and slow down your computer. 

Only if they're already creating errors that typically display messages.  Most other items are simply "leftovers" and have no effect on computer performance.

This is due to the fact that the registry is a database and not a sequential file, so its size doesn't really matter and even a few hundred leftovers are tiny compared to the normal size of a modern registry.

The issue with cleaning the registry is that unless you understand what you are doing, you're far more likely to cause additional damage than fix any that already exists.  That's because there are valid reasons for applications to leave things behind or in what appears to the typical person to be a "broken" state.  In some cases applications leave behind remnants on purpose that deal with certain indeterminate states that might exist when other programs that depended upon these try to access them after they are uninstalled.

In fact, the Microsoft Office applications were known for leaving portions of the registry apparently "unfinished" when there were parts of Office apps that hadn't been fully installed.  This allowed the operating system to request the disk for these uninstalled portions on "First Run", at which point a popup would display requesting the CD that contained the portions needing to be installed.

Though this type of First Run situation is less common today with everything downloaded from the Internet, there are still similar situations existing that few other than developers might understand.  So unless the operating system is operating horribly, it's typically best to leave the registry alone or in extreme cases, to perform either a repair or a full reinstallation which is guaranteed to fix such problems.

suzanne s. barnhill say:

The Edit tab functions the same as the old Styles dialog, providing a complete list of styles so that you can easily select one to modify (even if it isn't currently listed in the Styles pane). By default the styles are listed in "As Recommended" order (more on that with the next tab), but if you change it to "Alphabetical," you can easily find a given style and then click Modify... to open the Modify Style dialog for that style.

The Recommend tab allows you to set the priority of styles, which determines the order in which they will appear in the Quick Styles Gallery and in the Styles pane when it is set to display styles in "As Recommended" order. If there are styles you never use, you can choose "Make Last" and "Hide" so you won't see them. If there are styles you use frequently, you can assign a higher priority (lower number) so they will be on the top row of the Quick Styles Gallery. Styles with the same level will be sorted alphabetically. Note that not all styles appear in the Quick Styles Gallery, regardless of recommendation level, and there is no way to set that from the Manage Styles dialog (a flaw, in my opinion). Instead, you have to right-click a style in the Styles pane and choose "Add to Style Gallery" or "Remove from Style Gallery" as desired. It can require a lot of trial and error to get the priority order just right for your purposes, so it may take several trips to the dialog to tweak your settings.

The Restrict tab is important only if you are using document protection to prevent formatting changes. You can select which styles are visible and which ones' use is permissible, whether the user can switch style sets, and so on.

The Set Defaults tab is where you can set, all at one time, a default document font and the default paragraph settings for the Normal style. You can also do this through the Font and Paragraph dialogs by clicking Set As Default. In those dialogs and in the Manage Styles dialog, any change you make can be applied to the current document only or to the attached template, so that it affects new documents based on that template.
