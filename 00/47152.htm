Hi Xr6Lover  ,  

Thank you for writing to Microsoft Answers Community Forums.

Most possible cause here could be one of the third party programs you have installed.It could be one of the plug-in  As the first move try the steps below we will try to disable DEP

Disabling Data Execution Prevention (DEP)

Data Execution Prevention is a security feature that can be a bit of a pain at times. Often things will simply appear not to run at all; hopefully you will notice a bubble from the taskbar letting you know that DEP is the cause. If Vista sees that a process is being spawned that "could" be unwanted, DEP shuts it down. This is especially common in some application installations: if a Windows Installer setup (MSI) calls an executable in Vista, DEP could very well put a stop to it. If you are trying to run an installation or other executable being stopped by DEP, it could save you some trouble so turn it off while you attempt to give it another shot.

Run a command prompt as administrator. From the start menu, select "All Programs", then "Accessories" and right-click on the "Command Prompt" shortcut and choose "Run As Administrator". From here you can use the following command to disable Data Execution Prevention (DEP) with the following command:

bcdedit.exe /set {current} nx AlwaysOff

Keeping your command prompt open, run your setup or other process being stopped by DEP. Then, to turn it back on again, do the same and run the following:

bcdedit.exe /set {current} nx AlwaysOn

http://windowshelp.microsoft.com/Windows/en-US/help/1d9bb9b4-f6ba-466d-ac2b-7b8c4f8361611033.mspx#E1B

See if the above Microsoft article helps about DEP.It also shows how to turn on/off DEP for a particular Program.

You can try the following steps as well

Make sure that the DVD Maker is not running.

Open Windows Movie Maker (they are related) and select Tools/Options/Compatibility Tab.

Remove the check mark from any third party filters in that list.

If this works, look for updates to any third party CD/DVD burning programs.

If the above step fails you might have to try a clean boot to find out which third party program is causing the DVD maker to crash. Follow the steps below for clean boot procedures.

Click Start, type msconfig in the Start Search box, and then press ENTER. (If you are prompted for an administrator password or for a confirmation, type the password, or click Continue.)

 

On the General tab, click Selective Startup.

Under Selective Startup, click to clear the Load Startup Items check box.

Click the Services tab, click to select the Hide All Microsoft Services check box, and then click Disable All.

Click OK.

When you are prompted, click Restart.

Log back into your Account.

Test DVD maker

 If the problem does not return while your computer is in clean boot mode, a background program is likely causing interference. You might have to update the program or uninstall the program for this program to work. You can try to determine which background program is causing the problem while the computer is in clean boot mode.

How to perform a clean boot procedure to determine whether background programs are interfering with a game or a program that you currently use

Click Start, type msconfig in the Start Search box, and then press ENTER. (If you are prompted for an administrator password or for a confirmation, type the password, or click Continue.)

When you are prompted, click Restart.

You can also try doing a system restore to a point when the DVD maker was working fine.

To restore the operating system to an earlier point in time, follow these steps:

Click Start, type system restore in the Start Search box, and then click System Restore in the Programs list.If you are prompted for an administrator password or confirmation, type your password or click Continue.

In the System Restore dialog box, click Choose a different restore point, and then click Next.

In the list of restore points, click a restore point that was created before you began to experience the issue, and then click Next.

Click Finish.

The computer restarts, and the system files and settings are returned to the state that they were in at the time that the restore point was created.

    mikebar say:

    How do I establish a journal in Windows 10 storage spaces?

    Performance in my parity storage space is really slow and I'm trying to improve the speed. I've read about journals, but don't see an option to create journals in Windows 10 Pro, and need some help setting it up. I have 4, 10 TB drives in a parity (with resiliency) storage space. Can I use HDD's for the journals or do they have to be SDD's? Can I use any size drives? Do I need mirrored drives or can I use a single drive for the journal? And what's the command to establish the journal? I don't see an option in the GUI, so I'm guessing I need to use the command line in PowerShell.

    Any help is appreciated.

    Thanks,

    Mike

    adrianne buc say:

    Hi Mike,

    Can I use HDD's for the journals or do they have to be SDD's?

    • Yes you can. However, it will be slower. What is really recommended for journaling/caching is SSD or NVMe.

    Can I use any size drives?

    • Yes you can.

    Do I need mirrored drives or can I use a single drive for the journal?

    • Either will do.

    And what's the command to establish the journal?

    • PS C:\>Set-PhysicalDisk -FriendlyName <FriendlyName> -Usage <Usage>
    • Wherein FriendlyName specifies the friendly name of the physical disk on which to set attributes.
      Use command PS C:\> Get-PhysicalDisk to get physical disk FriendlyName.
    • Wherein Usage specifies the allocation method (usage) for the disk.
    • Example: PS C:\>Set-PhysicalDisk -FriendlyName PhysicalDisk2 -Usage Journal

    We hope this helps.

    swapnil rustagi say:

    IF you are referring to the Microsoft Update Catalog which lists all updates that were released and is used to manually search for an update, then it's possible to install the ActiveX locally without using Internet Explorer at all.

    Just for information, an ActiveX control is simply a registered .dll or .ocx file.

    Please download the .dll file from my File Storage at My Opera.

    NOTE: Since file uploading websites do not allow direct uploading of .dll .bat .exe and such files; I have compressed the MicrosoftUpdateCatalogWebControl.dll in a ZIP file. Please extract the ZIP file.

    After you have the DLL file with you.

    1. Copy MicrosoftUpdateCatalogWebControl.dll to C:\Windows\System32.

    2. Click the Start ORB (formerly called Start button) and in the search field (where it says Search programs and files with a blinking cursor) type 'cmd' (without quotes). From the list of search results, right-click cmd.exe and select Run as Administrator.

    3. Once in the command prompt, type the following command and press Enter-

    regsvr32 MicrosoftUpdateCatalogWebControl.dll 

    3. You should see a message informing that the DllRegisterServer succeeded.

    Close the Command Prompt.

    You should now be able to use Microsoft Update Catalog in Internet Explorer.
