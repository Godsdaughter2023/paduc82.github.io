Remember 3-2-1 to have good backups

o

  • at least 3 copies of any file,
  • at least 2 different storage mediums (say on your hard drive plus at least one of the following: thumb drive, DVD or the cloud),
  • at least 1 copy offsite -- in case of fire etc!  (ie the “cloud” meets this requirement, so does using a thumb drive moved to a different building, or a server in a different building)

Detect Language Automatically Does Not Work Properly For The Bilingual - Microsoft Community

yunus emreteke say:

Detect Language Automatically Does Not Work Properly For The Bilingual Users

Dears,

I am investigating for weeks to overcome the issue of Detecting Language automatically and using proofing tools while using Word and Outlook in more than one language.

To be specific, I am frequently composing texts and emails in Turkish, English and Italian and I want to use Detect Language Automatically feature both in Outlook and Work. Following a huge investigation, I am convinced that it is not properly built in. 

To be helpful to the bilingual users of this community I would like to share the two methods that I used to workaround.

It is assumed that you have installed the language packs and proofing tools.

Solution-1- Creating a style with the desired language detection.

1- On the style TAB, click the down arrow and "Create a Style"

2- Name the style, for instance, name it "Turkish"

3- Click on "Modify"

4- Bottom of the page, click "format"

5-Choose "Language" and select Turkish.

6- Tick "New Documents based on this template" and Click ok.

When you open a new file, if your default language is set to ENG (you can check the bottom right of the screen on the system tray) when you choose the created Turkish style, you will see that it is set to Turkish and your text will be checked according to Turkish.

Solution-2- Changing The Default Language of The System

If you use more than one language, presumably the relevant languages are introduced in Windows. For instance, when I click the language part on system tray on the bottom right of the screen, I can see "Turkish", "English" and "Italian".

If you would like to write a Turkish email or text, as soon as you change the Language type Microsoft Word and Outlook automatically starts to check your text in that language. I am aware it is a bit frustrating to go there and changing the language. However, by using shortcuts the process could be fastened.

ALT + Shift (Press press both at the same time)  = Change the Language

CTRL+Shift (Press press both at the same time)  = Change the Keyboard Type (in case you have installed different keyboard for the language)

Consequently, I accepted and I am convinced that detecting language automatically is not properly built in. But in particular, with solution-2, I can work rapidly by a shortcut.

Needless to say, if someone has achieved to use "detect language automatically" for different languages, his/her feedbacks are more than welcomed.

The only thing left in the now useless Status bar is the Zoom indicator which is completely redundant in at least two other menus, e.g. Tools (Alt-x), and View (Alt-V).

By design the functionality which used to be available down there has been relocated to be less conspicuous albeit also less useful.  For example, instead of the handy icons we could doubleclick on to get into various Options tabs we now need to go wherever we want conventionally.  However, the Security zone indicator has sort of been preserved in the File, Properties display.
