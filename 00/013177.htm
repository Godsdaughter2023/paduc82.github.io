richard a.hesslein jr. say:

Microsoft Security Essentials scan

rob koch say:

Though Smeed is correct that a scan can be performed without an Internet connection, this removes a significant portion of the extended cloud based protection that Microsoft Security Essentials is capable of providing.

The following is a portion of item 8 from the MSE Consolidated FAQ Smeed referenced.

"If an event occurs that requires immediate attention, the MSE update system will force an update to occur.  Also, MSE employs a “Dynamic Signature Service” (DSS).  Whenever MSE encounters something it does not know or recognize, it will send information to “Microsoft Active Protection Service” (MAPS) - unless you have explicitly chosen to not join MAPS.  Depending on what MAPS determines from inspecting the information, it will cause an automatic download of definition updates to handle the malware. It is recommended that you select at least the Basic MAPS participation level."

What all of this really means, is that the automatic or dynamic updating of MSE definitions via the "Microsoft Active Protection Service” (MAPS) only works if the Internet is available.

So though you can scan manually or MSE itself will still perform its daily automatic configured scan even without an Internet connection available, it will of course only "know" about those specific malware threats contained in its current definitions database.  So it's always best to be connected to the Internet if possible when scanning in order to have full access to the extended capabilities that MSE updates and the MAPS cloud based protections provide.

xinxin liu say:

There are various reasons that scrolling is slow on Excel for Mac 2011. Please first try a few troubleshooting steps:

 

0. Make sure both MacOS and Mac Office are patched with the latest updates.
1. Run "disk utility" (a system utility) to repair the disk permission
2. Run "font book"  (a system utility) to resolve duplicated fonts
3. With all Office applications shut down,
    a. Delete com.microsoft.Excel.plist from ~/Library/Preferences/
    b. Delete ~/Library/Preferences/Microsoft/ (the whole folder)
4. If step 1~3 doesn’t fix the problem, create a new user account and see whether Excel works on the new user account.

 

If these steps don't improve the performance, you can end me (xinxin_at_microsoft.com, replacing _at_ with @). I should be able to find the root cause and provide some advice.
