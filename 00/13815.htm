a) Press “Windows Logo” + “W” keys from the keyboard.

marshall barton say:

With a spreadsheet like table structure such as that, the IsNull(f1) + ... is about the best you can do.

 

Instead of that you should have one Tools table and a Types table linked on toolID.  The Tools table would have one record per tool with a few fields that relate to all tools.   The Types table would have one record per type with a few fields that describe a type.

 

Then you only need on more table, call it ToolTypesInfo, to connect the tools to the types.  This table would probably only need three fields one for the toolID, one for the typeID and the third for the tool type info.  You can use a compound primary key on the toolID and typeID fields.

 

The user interface for this arrangement would normally be a main form based on the tools table and a sub form based on the ToolTypesInfo table.  This sub form only need three controls, one field.  The toolID text box can be invisible and the typeID combo box would have a row source query for the typeID (while it displays the description of the type).  This sub form could be displayed as a datasheet, but most people use a continuous form.

 

With this arrangement you can query the ToolTypesInfo table to count the number of rows grouped by toolID to easily see how many types each tool has filled in.

   SELECT toolID, Count(*) As numTypes
   FROM ToolTypesInfo

   GROUP BY toolID

 

And for the description of types that were not used, you can use a query like:

 

   SELECT tooldescription, typedescription
   FROM Tools, Types

   WHERE typeID Not IN(SELECT typeID

                                       FROM ToolTypesInfo

                                       WHERE Tools.toolID = ToolTypesInfo.toolID)
