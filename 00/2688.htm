Hi Ashley789,

Based on my test, if you just upload the same file from your local computer to SharePoint Online to replace the online one, the shared link may be retained.

However, from your description, it seems that you see the error message and then go to the read mode when you open the file in both Excel Online and Excel 2016. If yes, I suggest you first try the following if you are using Office 365 ProPlus to see the result.

1. Back up all files online and all your local computer.

2. If you are using the OneDrive sync client to sync the files, please right-click on the OneDrive sync client’s icon>More>Settings>Account>Unlink this PC and remove the local sync folder.

3. Clear IE cache via Tools>Safety>deleting browser history, reset IE (do the similar steps in other browsers) and close the browser.

4. Close all Office 2016 applications >sign out of all accounts in the Office 2016 applications>navigate to Control Panel>User Accounts>Manage Windows Credentials under Credential Manager>remove all MicrosoftOffice,16_Data:ADAL:xxx.

5. Follow the steps and refer to the screenshots in the article to run the commands with the administrator right on the computer.

Note: For the 32-bit Office 2016 applications, run C:\program files <x86>\Microsoft office\office16>cscript ospp.vbs /dstatus.

For the 64-bit Office 2016 applications, run C:\program files\Microsoft office\office16>cscript ospp.vbs /dstatus.

6. Restart the computer.

7. Sign into the Office 365 portal (portal.office.com) with your work and school account and navigate to the site to open the file.

8. Click on the file>click on Open in Excel.

Meanwhile, I’d like you to confirm if you can edit the file after you download the file and open it in Excel 2016.

Moreover, I suggest you add all shared users to an Office 365 group and then share the file as a convenient way if some issues happens.
