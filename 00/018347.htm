asd1234 say:

  So I solved my problem with the connectivity issue.

  My particular problem ended up being an expired certificate between my Exchange 2013 and my Domain Controller.  It was coming up with a Source: MSExchange OAuth for Event ID: 2004

  The Certificate did not come up on any normal Exchange cert queries. I was able to run:

Get-AuthConfig

  This showed the offending certificate

  Once I ran:

1.  New-ExchangeCertificate -KeySize 2048 -PrivateKeyExportable $true -SubjectName "cn= Microsoft Exchange Server Auth Certificate" -DomainName "*.mydomain.com" -FriendlyName "Microsoft Exchange Server Auth Certificate" -Services smtp

2. DO NOT accept the replace SMTP certificate when prompted

3. copy the thumbprint of the new certificate. This is an example: G239541FH5F58D4821967DD8F894B27410F7S072

4. $a=get-date

5. Set-AuthConfig -NewCertificateThumbprint G239541FH5F58D4821967DD8F894B27410F7S072 –NewCertificateEffectiveDate $a

6.  Accept to continue despite the fact that the certificate effective date is not 48 hours into the future

7. Set-AuthConfig –PublishCertificate

8. Remove any reference to the previous certificate (it may not be there anymore) by doing Set-AuthConfig -ClearPreviousCertificate.

This fixed a bunch of errors in my Event Viewer.
