ken sheridan say: Splitting the database into separate front and back end files would almost certainly be the most appropriate in your situation, the tables only going in the back end file and the form's reports queries etc in the front end file.  The built in database splitter wizard can do this for you.

The back end file should be installed in a shared location on your system, to which all users have full permissions.  A separate copy of the front end should be installed on each user's local machine or in their 'personal' folder on the system if you adopt that strategy, as my organisation did.  The built in Linked Table Manager can be used to refresh the links to the back end if you create the front ends before moving the back end to its shared location.

Ideally Access itself should be installed on each user's local machine, but in my organisation a single copy was installed in a shared location, and this appeared to work fine.  The speed of the network will obviously have a bearing on this of course.

You should keep a 'master copy' of the front end as a .accbd file for development purposes, but distribute the front ends as .accde files.  This will prevent users from modifying objects, though they'll be able to enter and update data, open reports etc as normal.  When you wish to update the front end it is simply a question of making a new .accde file from your 'master copy' and distributing copies of this to users to replace their current copies.

And be sure to back up the back end very regularly!
