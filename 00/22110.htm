neo yu msft say:

Hi LarsFreiberger,

In Exchange Online (with no DirSync deployed), we can use the following steps (as described in: https://support.microsoft.com/en-us/kb/2723654) to receive the NDR bounced back to the DG (distribution group):

  1. Add the sender as an owner in the DG.
  2. Connect PowerShell to Office 365 by referring to: http://technet.microsoft.com/en-us/library/jj151815.aspx.
  3. Type the following cmdlet, and then press Enter:

Set-DistributionGroup   "<DistributionGroupName>"   -ReportToManagerEnabled  $true

Since you have DirSync deployed, all the changes made to the DG need to be done in the on-premises. Due to that the attribute exists when we have on-premises Exchange server deployed, if you want to achieve the goal, I suggest you use the following steps:

  1. Create the DG directly in Exchange Online, then set the attributes as showed above.
  2. Or you can create shared mailbox in Exchange Online for these users, then grant send as/full access permissions. In this way, the NDR will be returned to the inbox in the shared mailbox.

Or if you want to keep the DG in the on-premises, to check if any workarounds that we can made to the attributes in the on-premises AD, it’s recommended that you turn to our Windows Server Forum for expert assistance, your understanding is appreciated.

-----

First, you wrote your formula backwards.  Your formula is, essentially:

IF(INT(D21)<>0, "good", "bad")

It displays "good" when INT(D21) is __not__ zero, not when it __is__ zero.

So your first example seems right to you ("good") only because you thought (0.0000) would truncate to exact zero, when in fact it truncates to -1.

Likewise, the second example truncates to -1. It looks wrong to you ("good" instead of "bad") only because your formula is incorrect.  It should be, essentially:

=IF(INT(D21)=0, "good", "bad")

-----

Second, looks can be deceiving due to formatting.

I suspect that you are misinterpreting "$ (0.0000)" as exact zero.

But in the Accounting format, it is a negative value that rounds to zero when displaying 4 decimal places.

(It should round to -0.0001. But that is an unrelated issue.)

In the Accounting format, "$ -" (dollar dash) is exact zero.

So both examples show a non-zero negative value in D21.

And you should expect INT(D21) to be the same value in both examples.

Ergo, you should expect the same result from the IF() expression.

If you want to round or truncate toward zero for negative as well as positive values, you should use ROUND(D21,0) or TRUNC(D21).

-----

Third, it is remarkable that you expect D21 to be exact zero in the first example, if indeed you do, as your follow-up comment suggests, IMHO.

Note that formatting only affects the __appearance__ of a cell value. It does not affect that __actual__ cell value.

(Unless you set the option "Precision as displayed", which I do __not__ recommend.)

Note the SUM(D11:D20) is 7.00075 when formatted to 5 decimal places, the same as column E.

So you should expect a difference of -0.00005 for SUM(D10:D20) in the first example, and a difference of -0.00015 in the second example.

Public Function MyRGBColor(Rng As Range) As Long
    If Rng.Cells.Count <> 3 Then
        MyRGBColor = CVErr(xlErrNA)
        Exit Function
    End If

    With Rng
        MyRGBColor = RGB(.Cells(1).Value, .Cells(2).Value, .Cells(3).Value)
    End With
End Function
'<<=========

This UDF may be used in the worksheet like a native Excel function. Therefore, assuming that your data is in columns A:C and starts in A2, in D2 enter the formula 

      =MyRGBColor(A2:C2)

and drag the formula down as far as required.

#1 - Contacts are maintained in one or more "folders" where one is defined as the "default contact folder" for the profile. The default contact folder is contained in the data file set as the default for the profile

go to File --> Account Settings --> Data files tab to see which data file is set as the default 

#2 - The Outlook AddressBook is a specialized "view" of the contact "folders" set to be members of the AB. That's done by right-clicking on the contact folder --> properties --> select Outlook Address Book tab --> make sure that the box for "Show this folder as an email address book" is checked

#3 - Assuming that you're referring to <People> as it relates to the OL Nav pane, that's simply a list of all contact folders that exist in your profile. Some shortcuts for the Nav Pane = CTRL+6 = show all folders, CTRL+3 = show only contact folders, CTRL+ 1 show only email folders
