k.c.s. say:

I just tried this with the instructions at the http://support.microsoft.com/kb/321379/en-us link you provided and it worked fine.

Given the error you are getting here are my guesses at what the problem may be.  Please try these out and reply back:

- Because it is complaining about, "file does not have a program associated.." I would first make sure that when you created the .bat file with Notepad that you didn't actually create a file called "Printdir.bat.txt" or "Printdir.bat.some_bad_extension".  If your Windows Explorer settings are set to hide file extensions when you look at a file name then perhaps the REAL file extension you gave the batch file is being hidden from you and Windows doesn't know how to open a file with the hidden extension type.  Make sure you can see all file extensions and ensure the batch file is named with the .bat extension.

- If you can see all file extensions just fine in Windows explorer and your file is really named Printdir.bat, make sure you have saved it in your Windows root directory.  If you navigate to %windir% do you see Printdir.bat inside of there?  If not, put it there.  If you do, when you double-click the bat file does Windows run it without error and print the dir?  If it works, you know the batch file is working correctly.

- There is a typo in the instructions for Vista/Windows 7, ensure your Registry entries look like this:

HKEY_CLASSES_ROOT\Directory\shell\Print Directory Listing\command

This shows the new Key under shell called "Print Directory Listing" and inside of that Key/folder another Key/folder named "command"

Inside of the "command" key/folder there should only be one entry with a Name of (Default).  Double-click (Default) and make sure its Value Data is set to:

Printdir.bat %1

- You shouldn't  need the "associate" step in Windows 7.

- You may want to add, "Add Print Directory Listing to Windows Explorer shortcut menu" to your question title so others can find it.

robert aldwinckle on forums say:

a black screen and cursor.

If you see anything on it the screen is blank, not black.  Also "cursor" is ambiguous.  Is that describing a mouse pointer or an underline character, which would probably be at the top of the screen?

In fact Ctrl-Alt-Delete might actually be doing something but only doing it off screen.  So one approach to dealing with that possibility is to imagine what would be showing and what keystrokes you would need to use to deal with it if you couldn't use your mouse.  E.g. in that case you would probably have to press End and then Enter just to get the Task Manager window opened.  But then it might still be off screen (e.g. on another monitor) and you would have to do more keystrokes to move it.  E.g. if it was actually on another monitor you could press Win-Shift-CursorRight (depending on the suspected location of the other monitor); otherwise you could press Alt-Space M, wave your mouse pointer to see if it had changed its pointer icon, then press CursorRight and move the window with the mouse; alternatively you could continue pressing an arrow key (hoping that you had guessed the necessary direction) until you saw enough of the window appear and low enough that you could grab its title bar with your mouse.

Finally others have noticed that Narrator may be immediately available to help guide them.  Press Win-Enter (with your speakers turned on) to check for that possibility.

 

Good luck

Robert Aldwinckle
