The problem may actually be with your monitor's settings.  When a window opens off screen, it is because your computer mistakenly thinks you are using two monitors side-by-side.  It displays the second window on a (nonexistent) monitor.  To check this out........

 

Right-click on a blank area of the Desktop and select "personalize".

Select "Display" (bottom left corner)

Select" Change Display Settings"

Click "Detect" (button). 

     

If there is a second monitor detected........

Click on the monitor (#2)

Select "Multiple Displays" from the dropdown menu.

Select "Show Desktop on # 1 Only"

Select APPLY

Select OK

 

The window should now appear solely on your primary monitor.  If Windows tries to display on a different screen, it will just replicate the main screen.

General question about cleaning up recorded macros - Microsoft Community

jamesatkinsonqc say:

General question about cleaning up recorded macros

I have been working with and learning Excel and VBA on my job for a number of years and have found the RECORD MACRO feature to be invaluable in achieving things that I could have never programmed on my own.  But the results seem a bit bloated.

To all the Masters...

Are there any rules of thumb or general tips you can share with us about how to condense and simplify the wordy and extraneous code that we get from recording macros?

Maybe a resource on line or a helpful guide to what's necessary and what's not.

Thanks for your help!

James

hansv mvp say:

While recording a macro, each time you select a cell or a range of cells, a line is added to the macro.

In many situations, selecting cells is not really necessary during the execution of the macro, so check carefully whether you can eliminate lines with Range(...).Select.

For example: copy cell E1 and paste it to cell E2.

Code in the recorded macro:

    Range("E1").Select  ' Select the source cell E1
    Selection.Copy      ' Copy it
    Range("E2").Select  ' Select the target cell E2
    ActiveSheet.Paste   ' Paste

We can condense the first two lines to

    Range("E1").Copy    ' Copy the source cell

By avoiding selecting the cell, the code becomes shorter and more efficient.

We can go further: if you look up the Copy method of the Range object in the Object Browser (press F2) or in the online help, you'll see that you can specify the target range:

    Range("E1").Copy Destination:=Range("E2")

We have now reduced the original four lines to a single one, and avoided selecting cells altogether.
