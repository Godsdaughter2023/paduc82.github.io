Calculating weighted averages - Microsoft Community

munchkin130 say:

Calculating weighted averages

This is for an evaluation form that has 11 sections (A-K).  Each section has multiple radio button questions, each weighted, which provides a section score.  The each section score is then weighted to show an overall score.  Hope I've explained properly.

 

The formula below calculates each section score by the weighted percentage shown in the coorilating cell (AA20, AB20, etc).  It works fine when a score is present for section A-K, but sometime a certain section does not apply.  In those cases the calculation that shows the section score displays the words "Section does not apply"" instead of a score.  The absense of a number makes the formula below show "#Value!" for the score.  How can I get this formula to ignore the sections that do not apply? 

 

=(AA20*ScoreA)+(AB20*ScoreB)+(AC20*ScoreC)+(AD20*ScoreD)+(AE20*ScoreE)+(AF20*ScoreF)+(AG20*ScoreG)+(AH20*ScoreH)+(AI20*ScoreI)+(AJ20*ScoreJ)+(AK20*ScoreK)

 

 

 

 

bernie deitrick say: Create another range where ScoreA....ScoreK are brought together using links (if they are not already contiguous), then use

=SUMPRODUCT(AA20:AK20,RangewithScores)
like
=SUMPRODUCT(AA20:AK20,AL20:AV20)
or with a named range
=SUMPRODUCT(AA20:AK20,Scores)


joeu2004 say: Munchkin wrote:

The formula below calculates each section score by the weighted percentage shown in the coorilating cell (AA20, AB20, etc).  It works fine when a score is present for section A-K, but sometime a certain section does not apply.  In those cases the calculation that shows the section score displays the words "Section does not apply"" instead of a score.  The absense of a number makes the formula below show "#Value!" for the score.  How can I get this formula to ignore the sections that do not apply? 

=(AA20*ScoreA)+(AB20*ScoreB)+(AC20*ScoreC)+(AD20*ScoreD)+(AE20*ScoreE)+(AF20*ScoreF)+(AG20*ScoreG)+(AH20*ScoreH)+(AI20*ScoreI)+(AJ20*ScoreJ)+(AK20*ScoreK)


Although Bernie's solution using SUMPRODUCT is preferred, you could get away with doing the following:

=AA20*N(ScoreA)+AB20*N(ScoreB)+AC20*N(ScoreC)+AD20*N(ScoreD)+AE20*N(ScoreE)
+AF20*N(ScoreF)+AG20*N(ScoreG)+AH20*N(ScoreH)+AI20*N(ScoreI)+AJ20*N(ScoreJ)
+AK20*N(ScoreK)

The N() function returns zero when its argument is text.

In either case, the assumption is:  the sum of the weights is 1.

That is probably the case since you say AA20:AK20 contains "weighted percentages".  But not everyone sets that up correctly.

In the unlikely event that the sum of the weights is not 1, the formulas should be:

=(AA20*N(ScoreA)+AB20*N(ScoreB)+AC20*N(ScoreC)+AD20*N(ScoreD)+AE20*N(ScoreE)
+AF20*N(ScoreF)+AG20*N(ScoreG)+AH20*N(ScoreH)+AI20*N(ScoreI)+AJ20*N(ScoreJ)
+AK20*N(ScoreK)) / SUM(AA20:AK20)

or

=SUMPRODUCT(AA20:AK20,A20:K20) / SUM(AA20:AK20)

where A20:K20 represents ScoreA throught ScoreK.

joeu2004 say: PS....  I wrote:
Munchkin wrote:

sometime a certain section does not apply.  In those cases the calculation that shows the section score displays the words "Section does not apply"" instead of a score.  [....] How can I get this formula to ignore the sections that do not apply?


What do you mean by "ignore"?

In the formulas Bernie and I provided, the weighted average treats those sections as if the score is zero.  That pulls down the weighted average of the other sections.

Is that what you want?

A numerical example is always best to clarify your intent.

Suppose the 20 scores are 100, 110, 120, etc through 290 in A20:K20 (increment by 10).

And the 20 corresponding weights are 2.60%, 2.85%, 3.10%, etc through 7.10% and 7.85% in AA20:AK20 (increment by 0.25% except 7.85%, so the sum is 100%).

The weighted average is 212.10, SUMPRODUCT(A20:K20,AA20:AK20).

Now suppose the last score does not apply.  That is, we have "does not apply" instead 290.

Should the weighted average be 189.3350, using 0*7.85% for the last score?

Or should the weighted average be about 205.4639, SUMPRODUCT(A20:J20,AA20:AJ20)/SUM(AA20:AJ20), averaging all except the last score?
