Here is a procedure you could use:

Sub SearchFolder(ObjectName As String, ObjectType As String)
    ' Change the value but keep the \ at the end
    Const strFolder = "C:\Archive\"
    Dim strFile As String
    Dim dbs As dao.Database
    Dim ctr As dao.Container
    Dim doc As dao.Document
    strFile = Dir(strFolder & "*.*")
    Do While strFile <> ""
        strFile = Dir
        If LCase(strFile) Like "*.mdb" Or LCase(strFile) Like "*.accdb" Then
            Set dbs = Nothing
            On Error Resume Next
            Set dbs = OpenDatabase(strFolder & strFile)
            On Error GoTo 0
            If dbs Is Nothing Then
                Debug.Print "Cannot open database: " & strFile
            Else
                Set ctr = dbs.Containers(ObjectType)
                Set doc = Nothing
                On Error Resume Next
                Set doc = ctr.Documents(ObjectName)
                On Error GoTo 0
                If Not doc Is Nothing Then
                    Debug.Print ObjectName & " found in: " & strFile
                End If
                dbs.Close
            End If
        End If
    Loop
End Sub

For ObjectType, you can use:

Tables (this includes Queries)

Forms

Reports

Scripts (this means macros)

Modules

The output will be in the Immediate window.

Examples:

SearchFolder "qryProducts", "Tables"

SearchFolder "frmSchedule", "Forms"

Sub SearchFolder(FolderPath As String, ObjectName As String, ObjectType As String)
    Dim fso As Object
    Dim fld As Object
    Set fso = CreateObject("Scripting.FileSystemObject")
    Set fld = fso.GetFolder(FolderPath)
    Call ProcessFolder(fld, ObjectName, ObjectType)
End Sub
 
Sub ProcessFolder(fld As Object, ObjectName As String, ObjectType As String)
    Dim sfl As Object
    Dim fil As Object
    Dim dbs As DAO.Database
    Dim ctr As DAO.Container
    Dim doc As DAO.Document
    For Each fil In fld.Files
        If LCase(fil.Name) Like "*.mdb" Or LCase(fil.Name) Like "*.accdb" Then
            Set dbs = Nothing
            On Error Resume Next
            Set dbs = OpenDatabase(Name:=fil.Path, ReadOnly:=True)
            On Error GoTo 0
            If dbs Is Nothing Then
                Debug.Print "Cannot open database: " & fil.Path
            Else
                Set ctr = dbs.Containers(ObjectType)
                Set doc = Nothing
                On Error Resume Next
                Set doc = ctr.Documents(ObjectName)
                On Error GoTo 0
                If Not doc Is Nothing Then
                    Debug.Print ObjectName & " found in: " & fil.Path
                End If
                dbs.Close
            End If
        End If
    Next fil
    For Each sfl In fld.SubFolders
        Call ProcessFolder(sfl, ObjectName, ObjectType)
    Next sfl
End Sub

Previous Versions of Word 2008 Document - Microsoft Community

teraihicik say:

Previous Versions of Word 2008 Document

My kids erased a bunch of my work on a Word 2008 document...how do I recover a previous version?
bob jones aka: cybertaz say:
Undo can usually be used to recover backwards in a document until the document is closed. Once the document is closed its Undo stack is purged.

Word does not automatically save 'versions' of your documents. When a document is revised & saved the additions are added to the file in which they're made. Also, any content that has been deleted or replaced by new content is removed.

The only ways to revert to prior versions are:
  • Use Save As to create a new version at each editing session,
  • Use the Always create backup copy preference in the Save options,
  • Constantly use Track Changes [which is ill-advised], or
  • Rely on a third-party backup system such as the OS X Time Machine.

To help avoid such occurrences in the future it's recommended that a separate User Account be created for each individual who uses the Mac... At least have a User Accounts for the parents &  a separate User Account for the kids. That way the user's log in does not provide access to files belonging to other users.
