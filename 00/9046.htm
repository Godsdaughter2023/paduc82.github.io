A macro containing the following commands will enclose the selected text in quotes.

 

With Selection.Range
    .Text = Chr(34) & .Text & Chr(34)

    ken sheridan say: The Access application properties include a default template form and report name, which in Access 2003 are stored in the system workgroup file.  You then need to create an unbound form and report of these names designed as the default.  When you place a copy of the default form and report in a database new form or report objects will use the default form or report as the paradigm.

    Do not make the common mistake of thinking that it is the form or report which is stored in the system workgroup file.  It is only the names of the objects.  You must place copies in the database file itself.  This does of course mean that you are not limited to a single default.  You can keep a set of differently designed template forms and reports in a database and copy whichever you want to use into your current database, changing the name of the copy to that of the default form or report as set in the application properties.

    A database which does not include a copy of the default form or report template will use the standard Normal template.

    I'm afraid I can't remember exactly where these properties are to be found in Access 2003, but they shouldn't be too difficult to track down from the main database menu bar.

    How to fix "double data" in Vlookup Function - Microsoft Community

    hafizal rosyid say:

    How to fix "double data" in Vlookup Function

    I have a question about vlookup, I have included vlookup function and the result there are double data because the array table that I call is not complete, how would the solution to avoid double data appear and i want double data became "0" ?

    see this picture :

    teylyn say:

    Hello,

    you are using the Vlookup function without the 4th parameter. This defaults that parameter to "TRUE" and results in an approximate lookup, where Vlookup returns the next smallest value if an exact match is not found.

    Your lookup table does not have the value 8 or 10, so the next smallest value, 6, is used instead and the text "Buffet" is returned in both cases. This is wrong for both, of course.

    If you want to show a zero or some other text if a value is not found then you must use the 4th parameter and set it to False. In your case,

    =vlookup(E4,$A$2:$C$7,2,FALSE)

    This will return "#N/A" for any value that it cannot find. You can wrap the function in an Iferror statement to turn the "#N/A" into a zero, a space or some other text, like this

    =iferror(vlookup(E4,$A$2:$C$7,2,FALSE),0)

    =iferror(vlookup(E4,$A$2:$C$7,2,FALSE),"")

    =iferror(vlookup(E4,$A$2:$C$7,2,FALSE),"value not found")

    Let me know if that helps.
