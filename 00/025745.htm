I don't know what you mean by a function that "returns in cells B4 through E4 and enter it in E6".  And I don't know how you calculate "pretax margin".  For simplicity, I'll assume you sum B4:E4 [1].

If the pretax margin cannot be negative, enter the following into E6:

=LOOKUP(SUM(B4:E4), {0, 0.05, 0.07}, {"BELOW TARGET", "MEETS TARGET", "EXCEEDS TARGET"})

Note that the opposite of <5% is >=5%, not just >5%.

For educational purposes, this is how you would write a nested IF expression:

=IF(SUM(B4:E4) < 5%, "BELOW TARGET", IF(SUM(B4:E4) < 7%, "MEETS TARGET", "EXCEEDS TARGET"))

That is less efficient because we might need to calculate SUM(B4:E4) twice.

joeu2004 say:

(Resubmitting with major corrections.)

Enter the following into B6:

=LOOKUP(B4, {-1E300, 0.05, 0.07}, {"BELOW TARGET", "MEETS TARGET", "EXCEEDS TARGET"})

Then copy B6 and paste into C6:E6.

-1E300 is the number -1 times 10 to the power of 300.  It is just a "large" negative number.

PS....  If you are uncomfortable with -1E300 and you know that the pretax margin will never be less than some number, use that number.  For example, if the pretax margin will never be less than -10,000% (!), use -100 instead of -1E300.

joeu2004 say:

The LOOKUP formula is best, IMHO.  But an alternative that you might understand better is:

=IF(B4<5%, "BELOW TARGET", IF(B4<7%, "MEETS TARGET", "EXCEEDS TARGET"))

As for the LOOKUP formula, Google "excel lookup function" without quotes and look at the support.office.com help page. In a nutshell....

LOOKUP finds the highest value in the array constant {-1E300, 0.05, 0.07} that is less than or equal to B4, and it returns the corresponding value in the array constant {"BELOW TARGET", "MEETS TARGET", "EXCEEDS TARGET"}.

So if B4 is greater than or equal to 0.07, the correponding value is "EXCEEDS TARGET".  If not, and if B4 is greater than or equal to 0.05, the corresponding value is "MEETS TARGET". Otherwise, assuming that B4 is less than 0.05 and greater than or equal to -1E300 (nearly all negative values), the corresponding value is "BELOW TARGET"

0.05 and 0.07 are alternate ways of writing 5% and 7%.  We cannot use "%" in array constants.

    arnimdontes say:

    Here is solution!

    Found a solution that worked for me:

    1. Start regedit.exe
    2. Navigate to "Computer\HKEY_CURRENT_USER\Software\Microsoft\MediaPlayer\TunerConfig"
    3. Look for setting PREFERREDMETADATAPROVIDER -- it should have a value "pmpMusicMatch"
    4. Edit the setting PREFERREDMETADATAPROVIDER, and clear the value so you have a blank value
    5. Save / exit regedit.exe
    6. Restart Windows Media Player, you should be restored back to fai.music.metaservices.microsoft.com
