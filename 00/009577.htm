bob jones aka: cybertaz say:

My take is a bit different from John's :-) I'm guessing that you're referring to what happens if you dbl-click a file in a Finder Window.

If so, the process is the opposite of what you might think... OS X launches the parent application of the file involved based on file type association. Typically, it will launch the newest available version of the program regardless of which version was used to create the file unless a different version is already running. If a specific version is already in use the file will open in that version. IOW, the file, itself, doesn't determine which version of the creating program is used -- when dbl-clicking any Word document file OS X will launch the Office Preview version unless Word 2011 is currently in use.

In earlier versions of OS X this could be changed only on a file-by-file basis. Fortunately, it appears that Apple has modified this somewhere along the way. To alter it...

  • Quit all Office programs
  • Right-click any Word document in a Finder window & select Get Info
  • From the Open With dropdown select the preferred version
  • Click the Change All... button
  • Confirm the change
  • Repeat for each Word document file type [.doc, .docx, etc.]

From then on, documents of that type will open with the version you specify unless a different version is already running. If both Preview & 2011 are running the document will open in the specified version as well.

As John indicated, you can avoid having to make the change altogether by using File> Open from within the version in use. However, there are 2 drawbacks to consider;

  1. When you no longer are using the Preview you'll most likely have to reset the Open With specification, &
  2. Moreover, keep in mind that Preview is just a fancy word for BETA -- it's a work in progress meant to be used for testing purposes. It is not intended for routine work.

jay freedman say:
This behavior isn't available in the file handling dialogs (Open, Save As, etc.) in Office, so I assume you're asking about the Windows File Explorer.

On the View ribbon of File Explorer, in the Current View group, click the Group By button and choose None. The separators appear when the Group By is set to Name.

Note that the Group By setting is independent of the Sort By setting that includes the same list of items. Sort By controls the order in which files are listed.

v_agarwal say:

What do you mean by "resort my data" ?

Relative reference is moving cell reference in the same order as you copy your formula.

Assuming your value is in cell D5,   now in E5 you have given a formula as =D5 then if you copy E5 to E6 then it will move original reference to cell as =D6

In case you prefix $ sign it makes row or column or both absolute.
