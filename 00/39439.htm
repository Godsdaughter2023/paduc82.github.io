I'll assume, for the sake of argument, that Windows will always be smart enough to know which applications.

Yeah this is really the UAC system. Basically if a program tries to send any data to a protected location like /windows or /programfiles (or registry locations), it will get blocked. This went a long way to preventing viruses from compromising a whole system.

This is also why some programs (usually lighter program that can just "run" without being installed but also programs like Google Chrome) can be installed on a user-per-user basis. Since it doesn't need to push any data to a protected location, it never gets blocked. Employees can use that to their advantage because they don't need IT anymore to install a program. And if the employee does install something that causes a problem, it only hurts their own profile folder and you can fix it by just deleting it and letting the profile be built again upon next login.

    Hi Irham,

    Normally, we get this error code: “0xC004C008”, when we install a copy of Windows Operating System on multiple PCs. But, in this case, the error is due to the recent hardware changes on the computer.

    When we use the Windows product key to activate a copy of Windows Operating System, the computer generates a unique identifying code based on its hardware configuration. This code provides no identifying information to Microsoft; it is merely a summary of the hardware, you have installed on the computer. Windows sends this code to Microsoft’s servers along with the product key, linking the two in the database.

    Thus, when we reactivate Windows after any important hardware changes on the computer, there would be a mismatch between the hardware combination and the product key and due to which the activation process fails with this error code.

    Adding "every setup and IP address" nullifies the whole purpose of having a firewall in the first place. Please don't take this in the wrong way: If you can't properly configure a third party firewall, you shouldn't use one. Setting up a user configurable firewall is very difficult and requires a lot of knowledge about networking.

    If you want to make your life a lot easier, I suggest you uninstall the McAffee Firewall and let the Windows built-in firewall do it's job instead, for the following reasons:

    You say you connect through your PC. So I assume you are bridging the Xbox via your PCs WLAN to a WLAN router. The router itself is a hardware firewall, as all incoming data packets, for which no specific port forwarding rule has been configured, will be dropped. Which means: Hackers from outside your network will never reach the PC and it's firewall in the first place, unless you specifically told your router to forward incoming requests to your PC.

    So the only thing the Firewall in your PC is protecting from is from users from inside your own network. So you won't need a complicated and costly firewall to protect your PC, unless your little brother or your mom have a secret identity and are really malicious hackers themselves.

    By using the Windows built in Firewall, the Internet Connection Sharing should automatically open the appropriate ports for you.
