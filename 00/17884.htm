tom van stiphout say:

The ribbon requires you to setup a callback function. From there you can call your OpenForm function.

In the ribbon definition:

<button id ="cmdAbout"
       label="About Application"
       size="large"
       imageMso="BevelTextGallery"
       onAction="cmdAbout_OnAction"/>

In a standard module write the callback function with this exact signature:

Public Sub cmdAbout_OnAction(ByVal control As IRibbonControl)
    OpenForm "frmAbout"
End Sub

Note how the callback function has a "control" argument. This is so you can re-use the same function for several buttons, inspecting the control to see which one was clicked (hint, hint).

Hi Ian,

The malware that you experienced, may have hijacked your WD by setting

a policy that disables it.

You can determine if this is true by observing the following Registry key.

Hkey_Local_Machine\Software\Policies\Microsoft (select) Windows Defender.

The only key that this folder should contain is "Default".  If it contains any

others it has been altered, possibly by malware.

You can clear this condition by exporting the Windows Defender folder to

your Desktop (as a precaution) and then delete the Windows Defender folder

in the key. It will reconstruct on Restart with  only a "Default" key in it.

If you have W10 Pro you can achieve these same results, using your Local

Group Policy Editor. Selecting "Windows Defender" and observing its policies.

"Real-Time Protection" in particular.

doug robbins - mvp office apps & services (word) say:

By default, the Normal.dotm template is loaded as a Global Template and you cannot, and do not need to add it in the way that you were trying to.

The exclamation mark merely indicates that the file is macro enabled which is normal for the Normal.dotm template.

If you go to the Developer tab of the ribbon and click on Macros in the Code section of the ribbon, do you see your macros in the list of Macros?

In the absence of your saying that you had created them in a template other than the Normal.dotm template, if they do not appear in that dialog, then it is possible that your Normal.dotm template has been replaced by a new one.

You have not said what version of Windows you are using, if you have upgraded to Windows 10, there is a possibility that, as part of the upgrade process, your Normal.dotm template may have been renamed.  Look in your User Templates folder for a file with Normal in its name such as PreNormal, OldNormal or NormalOld.

Of course, if you are working in a corporate environment, there is a possibility that a misguided network administrator has deleted your Normal.dotm template.
