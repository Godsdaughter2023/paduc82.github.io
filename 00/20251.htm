auto back up excel with date and time stamp - Microsoft Community

deej18 say:

auto back up excel with date and time stamp

Hello,

We have a shared excel spreadsheet that we would like to backup daily three times a day.  Having the date and time stamp on the backup files is important because we don't want the files to be overwritten.  The best solution would have run in the background and specific times.       

Any suggestions?

andreas killer say:
3 times a day is difficult, a simple way is to make a copy before (or after) the file is saved.

But "shared" means the file is on a server, that would mean a user must have admin permissions to the server directory, otherwise he can not save a copy of the file in the same directory.

And the macros on the users PC must be enabled!

Make a sample file on your local PC and save as XLSM
Copy the code below into the code module "ThisWorkbook"
Save it a few times and look into the directory afterwards, you should see files with a date/time stamp.

Then copy the file on the server, share it and try if it works.

Andreas. 

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
  'Saves a backup of this file with date/time in the same directory
  Dim BackupName As String, i As Long
  If SaveAsUI Then Exit Sub
  With ThisWorkbook
    i = InStrRev(.FullName, ".")
    BackupName = Left(.FullName, i - 1) & Format(Now, " yyyy-mm-dd hh-nn-ss") & Mid(.FullName, i)
    On Error Resume Next
    Me.SaveCopyAs BackupName
  End With
End Sub

jiaxing bian msft say: Hi AGomes#,

This is an expected behavior.

Here is the reason:
The Modified date in the SharePoint library is when the list entry last got modified. On the other hand, the modified date in the webdav folder reflects when the actual file was last modified.

Given the situation, we suggest you try Lambert’s suggestion to use the “All Documents” view.

Jiaxing

Known issues with Cortana reminders - Microsoft Community

anannya podder say:

Known issues with Cortana reminders

Some of the features that have been demonstrated with Cortana may not work until a later build. We are aware of some issues with reminders, such as:

  • Unable to edit a reminder – once a reminder is created, you cannot change it.

  • "Sorry, I couldn't create that. Please try again later." – you may get this error if you try to create complex reminders, like reminders with a specific location.

  • If you turn Cortana off and then on again, reminders won’t work – the can be solved by signing out and signing in again, or restarting your device.

  • When a reminder completes, it does not show up in Cortana’s History Notebook.
