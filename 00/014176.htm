    A Report based on Crosstab Query asks for parameters twice, why? - Microsoft Community

    dlm101 say:

    A Report based on Crosstab Query asks for parameters twice, why?

     Using the query wizard, I generated a Crosstab-Query that uses two parameters, [StartDate] and [EndDate].  Running the Crosstab-Query I am prompted for the parameters (once time for each) and then the query generates.

     

     I later used the report wizard to create a Report from the Crosstab-Query. When I run the Report I am prompted for the [StartDate] and [EndDate] parameters twice:

    • The first set of prompts must have dates entered or it will fault immediately.  Any valid dates entered for the first set of parameter prompts is disregarded.
    • The second set of prompts is where the parameters get the data to generate the Report.  Entering valid dates for the second set of prompts lets the report generate.

    Functionally it works, however it is a cumbersome to have to enter dates twice.

    Two questions:

    1. How do I edit this Report so it's down to asking for parameters once?

    2. Why does this happen?

     

     

    tom van stiphout say:

    Access sometimes needs to requery the data source. It may also happen if after previewing the report you want to print it: another set of prompts.

     

    I would recommend you create a Criteria form with two date fields and a Preview button, and modify your query to "look back" to that form for its criteria.

     

    scottgem say:

    Two questions:

    1. How do I edit this Report so it's down to asking for parameters once?

    2. Why does this happen?

     

     

    1. You don't (can't) edit the report to eliminate multiple prompting
    2. It happens because, when using a parameter prompt query, every time Access needs to requery the data source it has to prompt for the values. 

    So the answer is to not use parameter prompt queries. I almost never use them.

    Instead create a form to enter the criteria as Tom suggests. This works because you can reference a value in any control on an open form using the syntax:
    =Forms!formname!controlname

    Some added advantages to this are:
    a) you can use a date picker to enter the dates
    b) you can default the date range to include all records or based on circumstances (i.e default to the previous month)
    c) you can verify that valid data values are entered (i.e. the start date is earlier than the end date)
    d) you can reference these values to show the date range on the report

    And these advantages are just for a date range. You can use list controls to supply other criteria to ensure that valid values are entered.
