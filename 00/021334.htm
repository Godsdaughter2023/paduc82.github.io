charles kenyon say:

What, precisely, does the Add Text button do in References > Table of Contents > Add Text?

I just noticed that the Add Text button on the References tab does not seem to add a TC field but does do something that puts selected text into a Table of Contents.

What is this button doing?

jay freedman say:

All it appears to do is to apply the selected Heading 1, 2, or 3 style to text that you type after inserting a paragraph between "real" TOC entries. These are the Heading styles, not the corresponding TOC 1, 2, or 3 styles -- they would look very odd within a TOC, both in font formatting and in lack of indentation (other than any defined in the heading style).

Deleting such an entry is the same as deleting any text -- select it and press Delete.

All in all, it strikes me as a completely useless button.

charles kenyon say:

Hi Jay,

If you select text, in the document body, and click on this button, you create something that shows up in a TOC. You can assign a level (1-3) to it. In some ways, it is similar to a TC field, but it is not a field. AFAIK, the standard TOC for style levels picks it up.

I think I have discovered the answer to my question, and it probably means that Suzanne will want to update her TOC Tips and Tricks page. In a document  where heading 1 is not a linked style (probably rare) it simply applies the level as direct paragraph formatting. In a separate document, where Heading 1 is a linked style, it applies that style to the paragraph.

It is paragraph level formatting and it can be changed/removed in the paragraph formatting dialog. One more thing to keep track of when trying to figure out what is happening!

I agree that it appears to be not only a useless button but one that is likely to be a source of confusion.

stefan blom say:

If I remember correctly, using the Add Text command in older versions of Word would set the "Outline level" property instead of applying a heading style. I'm guessing that this is consistent with your observation that the style must be "paragraph" rather than "linked" for Add Text to behave that way.

For what it is worth, if you want to prevent paragraph with direct "Outline level" formatting from being included in a table of contents, you can deléte the \u switch from the TOC field code.

That often happens to me when I get frustrated while trying many different things to make something work.

Certainly, that formula always returns zero if A1<>"mystring" and you just press Enter inadvertently.

So, when you look at the formula in the Formula Bar, is it surrounded by curly braces, e.g. {=QUARTILE(...)}?

It should be.

-----

Another possible typo that you might have overlooked:  an errant comma in the IF expression, to wit:

=Quartile.Inc(If(Data!A1:A100="MyString",Data!K1:K100,),1)

Note the comma after K1:K100 and before the parenthesis.  With the errant comma, a mismatch returns zero instead of FALSE.  Of course, QUARTILE does not ignore zero, but it does ignore FALSE.

-----

Finally, if some, but not all of your data in K1:K100 are inadvertently text, QUARTILE.INC will return an unexpected value.  Zero is possible under some circumstances.

Fill L1:L100 (or some parallel column) with formulas of the form (L1 shown):

=ISTEXT(K1)

Be sure that they are all FALSE.

QUARTILE et al do not require the data to be sorted.  But certainly removing 0s and "blanks" (whatever you mean by that) from the QUARTILE range will eliminate the problem with your original formula.

On the other hand, it might be easier in the future to array-enter one of the following (press ctrl+shift+Enter instead of just Enter; see the assumptions below)

=QUARTILE.INC(IF(Data!A1:A100="MyString",IF(Data!K1:K100<>0,Data!K1:K100)),1)

or

=QUARTILE.INC(IF(Data!A1:A100="MyString",
IF(Data!K1:K100<>0,IF(Data!K1:K100<>"",Data!K1:K100))),1)

The simpler first formula should work if by "blank", you mean empty (no constant; no formula).  The second formula is necessary if "blank" also includes cells whose value is the null string (""), typically due to an IF expression.
