doug robbins - mvp office apps & services (word) say:

If you use a macro containing the following code, you will not be asked the question

Dim toc As TableOfContents
Dim tof As TableOfFigures
With ActiveDocument
    .Range.Fields.Update
    For Each toc In .TablesOfContents
        toc.Update
    Next toc
    For Each tof In .TablesOfFigures
        tof.Update
    Next tof
End With

charles kenyon say:

Installing Macros

There is no setting to switch to update entire table.

You could try the following macro. I've made some tweaks, so don't blame Greg if it doesn't work. Thank him if it does. It runs on my computer. This should update all fields in a document, including Table of Contents fields, without generating a query.

Sub UpdateAllFields() ' From Greg Maxey 2013-09-30
' http://answers.microsoft.com/en-us/office/forum/office_2007-word/how-to-update-all-fields-in-a-document-at-once/54b43181-0ea7-4abc-bfb9-e0555125dfe5?page=3
'
    Dim rngStory As Word.Range, lngJunk As Long, oShp As Shape
    Dim oToc As TableOfContents, oTof As TableOfFigures, oToa As TableOfAuthorities
    '
    lngJunk = ActiveDocument.Sections(1).Headers(1).Range.StoryType
    For Each rngStory In ActiveDocument.StoryRanges
      'Iterate through all linked stories
      Do
        On Error Resume Next
        rngStory.Fields.Update
        Select Case rngStory.StoryType
          Case 6, 7, 8, 9, 10, 11
            If rngStory.ShapeRange.Count > 0 Then
              For Each oShp In rngStory.ShapeRange
                If oShp.TextFrame.HasText Then
                   oShp.TextFrame.TextRange.Fields.Update
                End If
              Next
            End If
          Case Else
            'Do Nothing
        End Select
        On Error GoTo 0
        'Get next linked story (if any)
        Set rngStory = rngStory.NextStoryRange
      Loop Until rngStory Is Nothing
      For Each oToc In ActiveDocument.TablesOfContents
        oToc.Update
      Next oToc
      For Each oTof In ActiveDocument.TablesOfFigures ' Added by CKK
        oTof.Update
      Next oTof
    '  For Each oToa In ActiveDocument.TablesOfAuthorities ' Added by CKK
    '    oToa.Update
    '  Next oToa
    Next
End Sub

drifter_77 say:

Outlook (O365) OST File Size Limit Issue

Hello,

Our company uses Office 365 with an E3 license.  This gives us Mailboxes with size limits of 100GB, but we also have Cached Exchanged Mode turned on.  We are also still using Windows 7 32bit for the most part (in the middle of a Win10 implementation). With mailboxes that can get up to 100GB, what will happen once the OST file comes close to its size limit?  Will it split?  What is the default size limit of an OST, is it still 50GB?

We have some users with extremely large mailboxes and i can foresee this issue coming up soon.

Thank you,

roady [mvp] say:

The default limit is still at 50GB but this can be adjusted. When the ost-file gets close to the limit, an error will appear that it can no longer write new data to the ost-file.

Note however that streams of data within an ost-file are compressed so the size of the mailbox isn't the same as the size of the ost-file. Also, you can adjust the Sync Slider for your account to adjust the time-frame for data that gets synched to the local client which will reduce the footprint of the ost-file as well.

Also note that depending on the storage that you are using, you might run into performance issues long before running into the storage limitation issue. With an ost-file of that size, the use of a relatively modern SSD is recommended.
