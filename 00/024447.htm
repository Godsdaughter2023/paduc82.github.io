swischner say:

Hi Randy,

I guess this should be rather easy to solve. First, you were right about the commandline switch. It used to be OneNote.exe /safeboot and bring up a dialog in which you could choose to delete the local cache files. That switch has been removed for no apparent reason. The mentioned dialog only comes up if OneNote has crashed 3 times. But we probably do not need to remove the cache.

You're also right about it not being a good idea to put local OneNote files (e.g. .ONE) in a sync folder of a cloud service like Dropbox, Box, Google Drive or even OneDrive(!). This seems to work at first look but will almost certainly lead to sync errors and duplicate folders if you access those files from different machines. To have notebooks accessible from different devices you would need to move the notebooks to OneDrive using the options offered inside of OneNote 2010/2013/2016.

If you open OneNote 2016 it will try to access the (local) notebooks and can't find them where they are expected. What happens? Does OneNote crash or just keep open with an error message? In the latter case you may try to CLOSE all open notebooks first. This should as well remove the cache files for this notebook.

After that you may either use the FILE - OPEN commands to open the notebooks from their actual storage location or

Open the notebook folders via Windows Explorer and double click the file with the extension .onetoc2 (as far as I remember its named open notebook.onetoc2 or similar). That should open it correctly in OneNote. Repeat for other notebooks.

In OneNotei you can set the storage location (Options, Save) for notebooks. This is not needed to locate notebooks which are stored elswhere. This setting is just to tell OneNote where to create new notebooks (if you do not specify another location upon creation).

I hope these hints gave you access to your notebooks back.
