suzanne s. barnhill say:

There are two possible issues here:

  1. Although you have indicated you have Word 2003, the description of your first problem sounds a lot more like what happens in Word 2007/2010. Although you apply an outline (multilevel list) format in those applications, the button that lights up in the Paragraph group on the Home tab is the one for simple numbering (if you apply outline bullets, the Bullets button lights up). If you really have Word 2003, then I don't know what's going on.
  2. If you have applied one of the outline formats linked to headings, then you're running into the "next style" issue. The "Style for following paragraph" defined for all of Word's built-in headings styles is Normal, so when you press Enter at the end of a heading, the next paragraph is in the Normal style.

There are several possible approaches to deal with #2.

  • If you're creating the outline as the framework for a document and will later have body text following the headings, then you can work in Outline view; in that view, when you press Enter at the end of a paragraph,  you get another paragraph in the same style, and you can use the buttons on the Outlining toolbar or tab to promote and demote heading levels.
  • If the entire document is to be an outline, you can change the following style for the heading styles to be the same style instead of Normal. If you do this, however, be sure at the same time to disable the "Keep with next" property for all but the first couple of heading levels; otherwise, all your paragraphs will clump together, each trying to stay with the following one, and make the document unstable.
  • A better approach if the entire document is to be an outline is to use outline numbering linked to something other than heading styles; I recommend the List Number series.

For more on using outline/multilevel list numbering, see these articles:

http://www.shaunakelly.com/word/numbering/OutlineNumbering.html and
http://www.shaunakelly.com/word/numbering/numbering2003.html or
http://www.shaunakelly.com/word/numbering/numbering20072010.html

teylyn say:

Hello,

The text uses a different date format than your regional settings and pasting them into Excel will cause half of them to be turned into wrong dates, and the other half to be not recognized as dates at all. Save the original text in a text file, open it in Excel and use the Text import Wizard to set the correct date format in Step 3. 

Details:

Excel will try to interpret the text as dates according to your regional settings. If you regional setting for dates is DMY order, then a text like "10/05/2014" will be interpreted as May-10-2014, but if your regional setting is MDY order then it will be Oct-5-2014.

Therefore, it is important to keep the text as text and use a special technique in Excel to make the text into valid dates that don't get their month and day switched around.  You may want to store the text in a plain text file that you can edit with Notepad. Make sure that you understand what the correct order of the text is. Is it meant to be DMY or MDY?

If the original text goes like this:

06/06/2014
10/06/2014
10/06/2014
11/06/2014
11/06/2014
12/06/2014
12/06/2014
12/06/2014
13/06/2014
13/06/2014
13/06/2014
16/06/2014
16/06/2014
16/06/2014
17/06/2014
17/06/2014
17/06/2014
17/06/2014

then it is DMY, because the dates after the row starting with 13 cannot be in any other format.   If your computer uses MDY instead, the first few rows will be turned into dates, but the month and day will be switched around. 12/06/2014 is MEANT to be June-12, but a computer with MDY will see it as Dec-06.

If, however, the original data looks like this:

06/05/2014

06/10/2014

then it is clearly MDY format, since otherwise the data in the last rows would not be dates. Place it in a text file.  Now open Excel and use the File > Open command to open that text file.  If you do not see the file, change the setting in the dialog to show All files, not just Excel files. Hi Bryyan,

Thanks for the information and I would like to make a summary here to clarify the issue. You set up your Office 365 email account and other accounts in a single Outlook profile and move the folders out from your Office 365 by drag/drop. Then the folders and its emails are lost due to the previous client crash.

As is known, your Office 365 emails and folders in Outlook Exchange connection mode are at real-time sync with Outlook Web App. Since you moved the folders away from Office 365 account, they will not be preserved in the online side. alexsun_2012 say:

hi katherine,

unfortunately, there is no such permission that allows users to create or edit but not delete. the editor permission allows to create, edit as well as delete. moreover, in office 365, only the owner can share the calendar to other in outlook web app with different permissions. as a global admin, you can share the calendar using the following cmdlet after connecting powershell to exchange online:

add-mailboxfolderpermission -identity *** Email address is removed for privacy ***:\calendar -user *** Email address is removed for privacy *** -accessrights editor

after you hide the shared mailbox from the gal using the set-mailbox cmdlet, you can still set the permissions via powershell.

for your last query, yes, shared calendar can be viewed in owa for android app. currently, outlook app for ios and android is not officially supported in our forum. for your concern about outlook app for ios and android, it is suggested that you contact the in-app support directly for assistance. thanks for your time and understanding.

best regards,
alex

alexsun_2012 say:

Hi Katherine,

The users can add the shared calendar in Outlook Web App. After that, the shared calendar will be synced to the OWA app automatically from the Outlook Web App.

For a self-created calendar, we can share it with others with the classified permissions like reviewer, editor or owner from Outlook Web App or Outlook client directly. However, for a shared mailbox calendar, we cannot do it in Outlook Web App or Outlook desktop. Admins have to use PowerShell to assign the permissions, which may be a little more complex (See the cmdlet above).

However, as I mentioned above, there is no such a permission available for users to edit, but disable the ability to delete. Deleting comes together with the creating/editing within the Owner or Editor permission.

Thanks for your understanding.

Alex
