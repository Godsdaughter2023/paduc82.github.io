It has always been possible to collapse and expand heading levels in the Navigation pane. And if you choose to collapse to, say, level 1, that setting still won't be preserved if you save, close and reopen the document.

Word 2013 has added a feature that you may find useful, though. 'In following line edit ComboBox1 to your ComboBox name
Private Sub ComboBox1_Change()

    Dim rngToSearch As Range
    Dim rngToFind As Range
    Dim valToFind As Variant
    Dim arrClearList()
   
    valToFind = ComboBox1.Value 'Edit ComboBox1 to your ComboBox name
   
    With Worksheets("Agent")

    End With

    Set rngToFind = rngToSearch.Find(What:=valToFind, _
            LookIn:=xlFormulas, _

joeu2004 say:

Ril wrote:

So, what cell J3 do is it should pull formula from another worksheet and use it in a background to multiply two different cells which in this case is I and B.

If J3 contains the formula as text -- created by using the FORMULATEXT function? -- I am not aware of any way to convince Excel to directly execute the function and use its result as the cell value.

(On the other hand, I am not familiar with any new features of Excel 2016.)

Two alternatives that you might consider.

1. Create a VBA function of the form:

Function myeval(s)

myeval = Evaluate(s)

End Function

That is a bare-bones implementation that assumes "s" returns formula as a string with or without "=" on the left. Certainly, you can spend a lot of time to make the function more robust.

2. Create a named reference in Name Manager, entering a formula of the following form into the Refers To field:

=Evaluate("=I638*B638")

Unfortunately, we cannot use the EVALUATE function directly in the cell; at least, not in Excel 2010.

But honestly, I don't see the point in doing either. In a related thread, you talked about returning the formula as text from VLOOKUP. But I don't believe either of the alternatives above adjusts the cell references, like copy-and-pasting a formula might do.

3. Redesign your VLOOKUP table, if indeed that is how you find the formula.  For example, if the VLOOKUP returns just a row number, your formula in J3 might be something of the form:

=INDEX(I:I, VLOOKUP(....)) & INDEX(B:B, VLOOKUP(...))

[EDIT] Even better: Design the table lookups so they return the values that you require. Ideally, use VLOOKUP. But if you cannot because the lookup column is not the leftmost column of the lookup range, use a combination of INDEX(...,MATCH(...)). Thus, your formula in J3 might be:

=VLOOKUP(...,9,0) * VLOOKUP(...,2,0)

where relative columns 2 and 3 correspond to the values now in column B and column I.
