jay freedman say:

Characters from the Wingdings font are not active controls; the box you used is just a character, like the round bullets.

Are you using boxes and round bullets at the same outline level in the same list? That won't work -- to have different kinds of bullets in the same document, either they must be assigned to different outline levels or they must be in separate lists.

That isn't likely for Wingdings, which is a standard Windows font. When a document is attached to an email, there can be problems caused by the various parts of the email system, but again that's fairly rare.

Disable all add-ins ( Office Button > Excel Options > Add-Ins ). 

Move any files from your XLSTART folders.

---

Macro to list XLSTART files:

 

There are 3 possible XLSTART folders to look in.
You can check them all by opening Excel, Alt+F11 to the visual basic editor, Insert> Module, paste in the following code and run the ShowXLSTARTFiles macro (put the cursor in the macro and press F5):

 

Sub ShowXLSTARTFiles()
  ShowFiles Application.StartupPath
  If Application.AltStartupPath <> "" Then ShowFiles Application.AltStartupPath
  ShowFiles Application.Path & "\XLSTART"
  Debug.Print "Done"
End Sub

Sub ShowFiles(Path As String)
  Dim stFile As String
  stFile = Dir(Path & "\*.*")
  Do Until stFile = ""
    Debug.Print Path & "\" & stFile
    stFile = Dir()
  Loop
End Sub

In the immediate window (Ctrl+G if it isn't visible) you will see listed any files in any of the 3 folders.If there are none it will just show "Done"

I have found you can correct the error by deleting the WinRT app data, and letting the "start menu app" rebuild the database. Here are instructions to wipe out the data. For good measure, I suggest you also re-register all the WinRT apps (basically the same as reinstalling them). Note, however, that I did not delve too deep into the specific problem, so these are general instructions that will

1) Log in as local Administrator. (You cannot delete your own files while you are logged in. You could also create a temporary local account that is an Administrator, but you avoid some NTFS permission issues if you simply use local Administrator account.)

2) Open C:\Users\YouUserName\AppData\Local either by navigating there in Explorer or typing it in the run dialog (Win+R). The AppData folder is hidden, so you can either enable hidden files or navigate to C:\Users\YourUserName in Explorer and then click the location bar and manually add \AppData to the path and press enter. 

3) Locate two folders: Packages and TileDataLayer

4) Either delete both folders, or move them to another location if you wish to keep a copy of the data.

5) Log back in with your user account.

6) Press Windows Key+X, release, then press A to open an elevated command prompt.

7) Type powershell then press enter.

8) Run this command to re-register all your apps:

Get-AppxPackage | % { Add-AppxPackage -DisableDevelopmentMode -ForceApplicationShutdown -Register ($_.InstallLocation + “\AppxManifest.xml”) }

9) Finally, run this command to re-register your system apps:

Get-ChildItem $env:windir\SystemApps\ -Name “AppxManifest.xml” -Recurse | % { Add-AppxPackage -DisableDevelopmentMode -ForceApplicationShutdown -Register ($env:windir + “\SystemApps\” + $_) }

    van dinh say:

    Are you sure you use Memo Field in your Access database?

     

    Memo Field can store up to 2 GB characters per Field value!

     

    But note that the max size of an Access database is also 2 GB.  Thus, if you have a Memo Field (close to) that long, you can't store anything else in the database.

     

    I suspect that you use Text Field in your Access database. A Text Field max length is only 255 characters...

     

     

    tom van stiphout say:

    How exactly are you "downloading" data into your Access database?

    If I had to create reports, I would use ODBC to create linked tables to SQL Server, then build quiries in Access based on those tables, then use those queries for my reports.

    Downloading to me implies copying data from over there to over here. That is not necessary for reports. Leave the data where it is.

    If there is a lot of data in the tables (say > 100,000 rows), I would definitely consider creating views or stored procedures in SQL Server to gather the data, and then link (if views) or call those objects from Access.

    Another option is to use SQL Server Reporting Services (SSRS) for your reports.

    Memo field size is unlikely to be your problem, but how you use those memo (text or ntext in SQL Server) fields is.

    I have the same problem. The way I solved this issue (so far is working)

    is to use  ADODB

     

     

    Public Function job_import(the_job As String)
    Dim conn As Object
    Dim rs As Object

    Set conn = New ADODB.Connection
    Set rs = New ADODB.Recordset

    conn.CursorLocation = adUseClient
    conn.Open "Driver={SQL Server};Server=INTRANET;Database=M2MDATA02;Trusted_Connection=yes"

    'Clear table
    CurrentDb.Execute "DELETE FROM local_jobs"
    CurrentDb.Execute "DELETE FROM local_routing"

    'Jobs
    var_sql = "SELECT fjobno, fpartno, fpartrev, fsono, fstatus, fcompany, fddue_date, fjob_name, fopen_dt, fprodcl FROM Jomast WHERE fjobno='" & the_job & "'"
    rs.Open var_sql, conn, adOpenStatic
    Do While Not rs.EOF
        var_sql = "INSERT INTO local_jobs (fjobno, fpartno, fpartrev, fsono, fstatus, fcompany, fddue_date, fjob_name, fopen_dt, fprodcl, sys_type) VALUES ('" & Trim(rs("fjobno")) & "','" & Trim(rs("fpartno")) & "','" & Trim(rs("fpartrev")) & "','" & Trim(rs("fsono")) & "','" & Trim(rs("fstatus")) & "','" & Trim(rs("fcompany")) & "','" & rs("fddue_date") & "','" & Trim(rs("fjob_name")) & "','" & rs("fopen_dt") & "','" & Trim(rs("fprodcl")) & "',-1)"
        CurrentDb.Execute var_sql
        rs.MoveNext
    Loop
    rs.Close

    'Routing
    var_sql = "SELECT foperno, fjobno, fpro_id, fopermemo FROM JODRTG WHERE fjobno='" & the_job & "' ORDER BY foperno"
    rs.Open var_sql, conn, adOpenStatic
    Do While Not rs.EOF
        var_sql = "INSERT INTO local_routing (foperno, fjobno, fpro_id, fopermemo) VALUES ('" & Trim(rs("foperno")) & "','" & Trim(rs("fjobno")) & "','" & Trim(rs("fpro_id")) & "','" & Trim(rs("fopermemo")) & "')"
        CurrentDb.Execute var_sql
        rs.MoveNext
    Loop
    rs.Close

    'End
    conn.Close
    Set conn = Nothing
    Set rs = Nothing

    End Function
