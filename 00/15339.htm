djsteele, mvp say:

SELECT FEE_ID, ML, Max(SYS_DATE)
FROM MyTable
GROUP BY FEE_ID, ML

If there are other fields in MyTable that you want to display, you'll need to join the table to that subquery:

SELECT FEE_ID, ML, SYS_DATE, Field1, Field2, Field3
FROM MyTable INNER JOIN
(SELECT FEE_ID, ML, Max(SYS_DATE) AS MaxDate
FROM MyTable
GROUP BY FEE_ID, ML) AS Subq
ON MyTable.FEE_ID = Subq.FEE_ID
AND MyTable.ML = Subq.ML
AND MyTable.SYS_DATE = Subq.MaxDate

bob jones aka: cybertaz say:

Well, I'm coming from a different direction :-) I believe the others thought you were referring to the language of the User Interface [UI] such as Menus, labeling & dialog windows. However, the Czech language is not one of the 16 languages supported by the O2016 UI. That suggests to me that you must be referring to the Language attribute of the text in your documents, which is a completely different matter.

The Default Language set by going to Tools> Language pertains only to new documents you create from that point on. If the problem you're having is with existing documents, changing the Default will do no good whatsoever. Language is an attribute of the text, itself, similar to Bold, Italic, etc.

In existing documents you need to select the text, then specify the appropriate language. You can do so in either of several ways;

  • Using Tools> Language (but not changing the Default)
  • Modifying the Style used to format the text to include the correct Language attribute
  • Creating new Styles that include the correct Language attribute & apply them accordingly

For a comprehensive explanation please see: Spelling, Language, Dictionary
