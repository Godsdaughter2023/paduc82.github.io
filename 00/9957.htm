I see several mistakes.

-----

1. LookAt:=xlPart should be LookAt:=xlWhole if you want to avoid the Find mistake.

This is the primary source of your problem.

The problem is:  1.1 matches the left part of 1.19.  Since 1.19 appears first in the range, Find(LookAt:=xlPart) will correctly match 1.19 before it sees 1.1.

Of course, if 1.1 appeared first in the range, then the Find method would seem to work.  But that is really a coincidence.

Does LookAt:=xlWhole work against your intent?  That is, what are you trying to accomplish with LookAt:=xlPart, if anything in particular at all?

-----

2. Dim [...] iMin As String should probably be iMin As Double.

As String, iMax = Application.Min(oRg) appears to assign the value found by Min() as a string formatted to up to 15 significant digits.  That might not be the same as the actual value found by Min().

Consider the following experiment.  Replace 1.14 with the formula =1.1-1E-15.

That value displays as 1.1 even if we format to 14 decimal places (the maximum Excel will format usefully).  But the actual value is infinitesimally less than 1.1.  Therefore, it is now the true min in the range.

Thus, in Excel, =MATCH(MIN(A2:A17),A2:A17,0) returns the index of the previous 1.14 value (13) -- now 1.1-1E-15 -- instead of the index of the truly 1.1 value (12).

But oRg.Find(What:=iMin) finds the truly 1.1 value instead of the truly min value 1.1-1E-15.

-----

3. In conjunction with #2, replace the oRg.Find usage with the following (or an equivalent statement):

    oRg.Cells(WorksheetFunction.Match(iMin, oRg, 0),1).Select

Even when iMin is type Double, oRg.Find matches the truly 1.1 value instead of the truly min value 1.1-1E-15.

I suspect that even when oRg.Find(What) is numeric, oRg.Find matches against strings, namely the values formatted to up to 15 significant digits.

WorksheetFunction.Match behaves like the equivalent Excel formula, if that is your intent.

-----

Unrelated comment....

I don't know what you think you might accomplish with the Application.Wait (Now+TimeValue(...)) statements (which are commented out in your posting).  But I suspect they do not do what you expect.

If you actually executed Application.Wait (Now+TimeValue(...)), I believe it blocks (stops execution of) the entire Excel process, not just the VBA thread.  At least, that is the literal interpretation of the Application.Wait help page, to wit:  "The Wait method suspends all Microsoft Excel activity".

If your intent is to give the Excel thread time to complete an operation, I do not believe Application.Wait does that.  I would call DoEvents instead.

However, DoEvents does not really guarantee that the Excel thread completes before resuming VBA execution, even on a single-CPU single-core computer.  Moreover, I don't know the semantics of DoEvents on multi-CPU and/or multi-core computer.  It might not block the VBA thread at all (since Excel can run on another core/CPU).

In order to ensure that only the VBA thread is blocked for a fixed period of time, I would use Application.OnTime.  But that will complicate your "macro" design significantly.  You actually would need to break into several macros.

However, I believe that Excel usually completes its operation before returning control back to VBA.  There are exceptions, I've discovered empirically.  But I don't believe the operations in your macro are among them.

Sub Test()

    Dim oRg As Range, oRgFound As Range
    Dim dbMin As Double, dbMax As Double

   
    If ActiveCell.Row < 16 Then MsgBox "Can't run from above row 16": Exit Sub

    If ActiveCell.Column < 7 Then MsgBox "Can't run from left of col 7 (G)": Exit Sub
   

    Set oRg = ActiveCell

    Do While oRg.Value <> "" And IsNumeric(oRg.Value)

       

        If oRg.Value < -2 Then

            Set oRg = Range(oRg.Offset(-15, -6), oRg.Offset(20, -6))

           

             'Find the minimum value:

            dbMin = Application.Min(oRg)

           

             'Get cell containing the min value:

            Set oRgFound = oRg.Cells(WorksheetFunction.Match(dbMin, oRg, 0), 1)

            

             'Copy min value to cell one col to the right:

            oRgFound.Offset(0, 1).Value = oRgFound.Value

           

            Set oRg = oRgFound.Offset(15, 5)

        End If

       

        If oRg.Value > 2 Then

            Set oRg = Range(oRg.Offset(-20, -6), oRg.Offset(15, -6))

            

             'Find the maximum value:

            dbMax = Application.Max(oRg)

            Set oRg = oRgFound.Offset(15, 4)

        End If

       

        oRg.Select

    Loop
