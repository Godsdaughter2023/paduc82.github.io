Hi, I'm wondering if there is any way whatsoever I can change my default slide size from the default 4:3 or 16:9 to A4 size (210 × 297mm). I usually make a lot of Powerpoints for school and they need to be A4 so that they fit in my A3 sketchbook and it would be nice if I could open a Powerpoint and start right away.

john korchok say:

Here's how:

  1. In PowerPoint, select the Design tab, then click on Slide Size and choose Custom Slide Size...
  2. In the Slides sized for: dropdown, choose A4 Paper (210x297 mm), then click on OK. 
  3. Format the deck for whatever default formatting you want to appear on each presentations.
  4. Still on the Design tab, click on the More button in the bottom right corner of the Themes Gallery (looks like a horizontal bar over a down-pointing triangle).
  5. Choose Save Current Theme, give it a memorable name and click on Save.
  6. Click again on the More button of the Themes Gallery, find your new theme in the Custom group, right-click on it and choose Set as Default Theme.

Now all new presentations will be in A4 format.

1.  First create a table with just one column and one row.  The table and column can be called whatever you like; the important thing is that it has just one row, in which the data can be anything whatsoever.  The table is just to get the report to print one label, and the contents of the table are immaterial.  Use this table as the report's RecordSource.  You can then base the query on this table, but instead of returning data from the table reference the control's on your form, e.g.

SELECT Forms![YourForm]![Control1] AS Line1,
Forms![YourForm]![Control2] AS Line2,
Forms![YourForm]![Control3] AS Line3,
Forms![YourForm]![Control4] AS Line4
FROM [YourTable];

The report would contain four text box controls, each of which would be bound to the Line1 etc columns returned by the query.

2.  The other method would be to create a table which includes the four columns, one for each line, and base both the form and the report on this.  You can then enter values in bound controls in the form.  To ensure that the controls are empty when the form is loaded you'd out code like this in the form's Load event procedure;

Me.[Control1] = Null
Me.[Control2] = Null
Me.[Control3] = Null
Me.[Control4] = Null

With this approach you need to save the record before opening the report, so the code in the button's Click event procedure would be along these lines:

' ensure record is saved
Me.Dirty = False
' print report
DoCmd.OpenReport "YourReport"
