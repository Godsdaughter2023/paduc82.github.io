Read Stefan's response. I expand on that response.

3. I assume that you mean that some commands become hidden in menus when you make the window smaller. This is by design to still allow access to all commands. No, you cannot stop this from happening. Short answer: No ... <sigh>

Office 2007 introduced the "new and improved" user interface, the "Ribbon".  It was improved by taking all control/customization away the users to make life easier for support staff.

#1 The ribbon design is set in stone. All you can do to make it smaller is minimize it using the <CTL><F1> shortcut.

#2 no. The button and/or text display is entirely controlled by program, by the design parameters written into the ribbon code.

#3a ??? I don't understand the question. The ribbon isn't affect by the case of the characters.

#3b Sorry no. That is a built in "feature" of the ribbon. It resizes buttons, and text automagically to make "best" (as defined by MS!) use of space.

#4 Not really. There is nothing directly in the user interface. You can use the File menu > Options command > Customize Ribbon option to define your own "custom" tabs and groups using the existing command. But that will still leave button size, images and text under the control of Office.  You can take more control, by defining custom macros for each command. That allows you to define images, but only from a very small set.

The only way you can take full control of the ribbon design is by writing custom RibbonXML. The link to the article that Charles provided is a good one. If you do decide to go that route I can provide you with links to several more articles.  Frankly, it's more work than it is worth.

But I agree with his suggestion, give the ribbon a chance. It's like athletes foot, it grows on you ... <g>

Hyperlinked TOC entries do not display the formatting which will be applied to "standard" hyperlinks.

suzanne s. barnhill say:
By design, links in the TOC and TOF, as well as hyperlinked cross-references, are formatted with the Hyperlink character style, but it is suppressed. If you view the document in Web Layout view, you will see the hyperlinks (and no page numbers), but the Print Layout view is intended (as the name implies) for print, where that formatting would not be desirable. Ironically, if you unlink a TOC, the hyperlinks no longer function, but the Hyperlink style is displayed.

ActiveDocument.Protect - Microsoft Community

kevin_n_pa say:

ActiveDocument.Protect

I've developed a form with tables and form fields. Protection is controlled with the following code.

ActiveDocument.Protect Password:=MyPassword, NoReset:=True, Type:=wdAllowOnlyFormFields

Problem is, form fields can be replicated/duplicated through the use of the Enter key. That is, with a form field selected, pressing the Enter key adds a new line where it adds a duplicate form field.

Is it possible to prevent this functionality and still permit form field editing?

Thanks.

jay freedman say:

The Enter key isn't really creating a separate new field -- it's just that the Enter key creates a paragraph break within the field, and the new paragraph is separated from the previous one by the Space After setting of the current style.

There are two issues here. One is that you don't want to allow new paragraphs inside a form field, and the other is that you probably don't want entries in a field to expand the depth of the table row and throw off the alignment of the form.

The first issue can be handled with some macros, which are given at http://support.microsoft.com/kb/211219. Although the article says it's applicable to Word 2000 and 2002, it's actually still valid all the way up to Word 2013 (currently).

The second issue is a bit touchier: If someone types in a long text, even without pressing Enter, the field will expand and push the rest of the table down. You can prevent the misalignment by going into the Table Properties dialog and setting the row height(s) to an Exact size. The user can still type more than will fit in the table cell, but the overflow will disappear below the bottom of the cell.

If the filled forms will be processed by macros, you may have to deal with overfilled fields, but that's another issue.

Save a Powerpoint Presentation with commentary as a video / film - Microsoft Community

julia_hi say:

Save a Powerpoint Presentation with commentary as a video / film

Hello!

I'm desperately trying to convert my powerpoint presentation into either a film / video or a "self-running-presentation". I use powerpoint 2011 for mac. I have recorded an audio commentary on each slide (which contain text or videos) and now I need to string it all together into something that runs on it's own. If I convert it into a film, it doesn't play the audio. If I click "record presentation" I can afterwards click on "play presentation" and it does so nicely, saving the times for each slide and the audio commentary i recorded, BUT it still shows the loudspeaker icon on each slide (which isn't very nice) and it doesn't start automatically, as soon as I open the file. Please help! 

john korchok say:

There are 2 ways to get your narration into a video. The first is to record the show playback with QuickTime Player. See my answer on this page for the details.

The second method is to extract your narration, then add it back in again:

  1. Make a copy of your presentation and change the file ending to .zip. 
  2. Expand the Zip file, then open the ppt folder, then the media folder.
  3. Copy the media1.wav file. If you have more than one .wav file, you'll want to edit them together into one big sound file using a sound editor like the free Audacity.
  4. Open the original presentation and choose File>Save as Movie.
  5. Click on the Movie Options button.
  6. Beside Background soundtrack, click on the dropdown and choose Select soundtrack...
  7. Pick your audio file that you extracted. Click on OK to export the movie with your narration as the sound track.

To make the audio play automatically, click on the speaker icon. Then select the Format Audio tab of the Ribbon and change the Start dropdown to Automatically.

Unless you need to start and stop audio during a presentation, you can move the speaker icon off the slide to the area beside it and it will still play.
