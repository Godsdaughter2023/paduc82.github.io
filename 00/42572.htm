the real keith ballard say:

The "LBD" (full name LBD.SYS) driver is part of Lavasoft's Ad-Aware anti-malware software. Unfortunately, at some time in the past, Lavasoft seem to have configured their installer/uninstaller incorrectly, and when you uninstall Ad-Aware, it leaves behind references to the "LBD" service driver, including one which tries to start the service during boot-up.

The quickest way to fix this is use REGEDIT, and search down to:

HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Services

In the list of services, you will see "Lbd" listed. It will probably have a number of sub-keys listed below it.

Simply place your highlight over the main "Lbd" key, and delete it, along with the sub-keys. This should fix the problem.

If you're not comfortable editing the registry directly, I suggest you have a search around for utilities which show which software is run when you start your computer. Unfortunately, utilities like MSCONFIG don't seem to show the "Lbd" service, even though it should list all startup programs. You may have to keep trying different tools until you find one which shows the "Lbd" entry.

    authroot.stl invalid key usage - Microsoft Community

    joe-90 say:

    authroot.stl invalid key usage

    In the Applications Event Log I get the following CAPI2 error being logged:

    Failed extract of third-party root list from auto update cab at: <http://www.download.windowsupdate.com/msdownload/update/v3/static/trustedr/en/authrootstl.cab> with error: A required certificate is not within its validity period when verifying against the current system clock or the timestamp in the signed file.

    Investigating this further  authrootstl.cab contains the following file: authroot.stl.

    On investigating authroot.stl, I found that the field: Key Usage - has an invalid Digital Signature (80) value.

    Microsoft have had this problem with their release of authroot.stl now for many months, and even after installing the latest released Windows 7 64-bit Service Pack 1, and having all the latest windows updates installed Microsoft continue to release authroot.stl with an invalid Key Usage field value of 80.

    When will Microsoft release an authroot.stl Signed Trust List with a valid key usage field?

    I'm surprised that this has been going on for many months now, and most people have been incorrectly thinking the fault is due to a certificate not within a valid time period, due to the erroneous and misleading error description giving for the CAPI2 error in the Application Event log.

    I look forward to Microsoft acknowledging this flawed authroot.stl file and will release a proper signed authroot.stl file soon.

    Joe-90
