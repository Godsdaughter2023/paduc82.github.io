armen stein say:

If you migrate the DB into SharePoint, then you have two ways to update the data using an Access 2010 front-end:

1.  An Access client application, which stores the data in local Access tables and automatically syncronizes them with the SharePoint back-end.  The client apps can use client forms and VBA like a normal Access app.  It can also tolerate slower connections over a WAN.

2.  An Access web application, which runs in a browser and doesn't need Access installed on the PC.  These are limited to web-safe forms built with Layout view, and can't use VBA, only macros.  There are other limitations too - you would want to research this new method before you jump into it.

Another option is to use a SQL Server back-end database, hosted by a provider or SQL Azure.  Then connect an Access app to it over the WAN.  The Access app needs to be well optimized using client-server techniques.   I've written a PowerPoint presentation on that.  It's called "Best of Both Worlds" at our free J Street Downloads Page:

http://www.JStreetTech.com/downloads

If you include the chapter number in your caption, numbering will restart with each chapter.

Without the chapter number, numbering is continuous.

Insert a Caption - Complex Documents

It is possible to include the chapter number without restarting, but not just using the dialog.

The numbering is actually set in a SEQ field. You can edit the field to remove the restart switch.

You would want to remove the \s 1 switch at the end of the field. This assumes your chapters are numbered in level 1.

You can toggle display of field codes using Alt+F9.

Here is the code for a Table caption which restarts numbering at each chapter.

Here is the same caption with the restart switch removed.

Then toggle back. Captions with chapter numbering but without resetting the numbering cannot be automatically inserted. The coding for each caption though is identical. Once you change one, you can copy and paste it to replace others. Otherwise, you would need to modify each caption's fields.

Remember to press Alt+F9 a second time when you are done editing the fields to restore the display of field results.

Documentation: Sequence Field

Note that it is not a new Word section that triggers the restart, but the presence of a new paragraph in the outline level set in the Caption Numbering dialog. In the above examples, this is the level 1.

chadh360 say:

An update for this for anyone who has a similar issue.

The job is now complete, and everything is in place within the necessary time period.

These are the steps involved:

-Create a migration batch for the mailboxes

-Wait for that to be fully synchronised

-Delete the migration batch

-Carry out a public folder migration using the steps found here:

https://technet.microsoft.com/en-GB/library/dn874017(v=exchg.150).aspx

-When you're ready to 'cutover', complete the public folder migration, and delete it when done.

-Set up a new migration batch for the user mailboxes

-This will carry on delta syncing, so should just take a short while dependent on how long it's been since the original batch migration was done

-Once that's finished, you'll have everything over.

Not as seamless as the serial public folder migration, but it still does the job.

The only issues I really noticed after completion were as follows:

1) Any user emails that were received before the end of the first migration, but completely deleted (ie removed from the deleted items and possibly the dumpster as well)  between deleting the first migration and setting up the second would reappear - I guess the migration lost track of them and left them in the Office 365 mailbox. This hardly affected anybody, and only resulted in a few additional copies of email.

2) Out of Office data isn't updated during the migration - the settings at the time of the creation of the first batch stick on the Office 365 mailbox. I'm pretty sure that this is a problem on migrations in general, though. Just something to watch out for.

Hope that helps somebody for the future.
