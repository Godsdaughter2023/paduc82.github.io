hansv mvp say:

There is a Before Update event at two different levels: that of individual controls and that of the form as a whole.

 

The Before Update even of a control fires before the value of the control is updated. It doesn't fire before the record as a whole is updated.

 

The Before Update event of a form fires before the current record is updated. It doesn't fire before individual controls on the form are updated.

 

As you have found, undoing all changes in the Before Update event is not ideal. I'd use the On Unload event of the form and cancel the unload if not all required data have been filled in. That would allow the user to visit the subforms and return to the main form without clearing the record.

 

 

Private Sub Form_Unload(Cancel As Integer)
    ' Test whether required information has been filled in
    If ... Then
        MsgBox "Not all required information has been provided."
        Cancel = True
    End If
End Sub

If you install two different Word versions on the same platform (e.g. I currently have 2003, 2007, 2010, 2013 and 2016 installed for support purposes)  then they will conflict due to shared resources. Word 2016 is better in this respect than its predecessors, but you will still have to provide separate Word startup and user templates folders for the two versions if you wish to keep both; and make the registry changes as shown in the web page.

Create four folders e.g.

C:\Word 2010 Templates

C:\Word 2016 Templates

C:\Word 2010 Startup

C:\Word 2016 Startup

Locate the original startup folder If you have not changed the preferred startup folder it can be located (in English language versions of Windows) by typing
%appdata%\Microsoft\Word\Startup
in the Windows File Explorer Address bar and pressing Enter.

Copy the files it contains to the two startup folders.

Repeat for the User Templates (%appdata%\Microsoft\Templates) and copy the files to the two Templates folders.

You can then install Office 2016.

Having done so ...

Open Word 2010 and from File > Options > Advanced > file locations, change the user templates folder to C:\Word 2010 Templates and the Startup folder to C:\Word 2010 Startup.

Then from File > Options > Trust Center > Trust Center Settings > Trusted Locations, make the folders

C:\Word 2010 Templates

C:\Word 2010 Startup

Trusted locations.

OK out of the dialogs and close Word. Restart Word and it should now be using the new locations.

Repeat the folder changes for Word 2016.

Note that having done so Word 2016 will show Compatibility Mode in the title bar as it will be using Woird 2010 templates. Open each of the Word templates in both 2016 folders in turn and SaveAs with the same names and locations to update them to 2016 format.
