chip pearson say:

I think you need to use a VBA user defined function. For example,

 

'>>>>>>>>> BEGIN CODE
Function AverageNoZero(FirstSheet As String, _
        LastSheet As String, _
        RR As Range) As Variant
Dim WSStart As Long
Dim WSEnd As Long
Dim WSNdx As Long
Dim Addr As String
Dim R As Range
Dim N As Long
Dim V As Double
Dim DataRange As Range

With ThisWorkbook.Worksheets
On Error Resume Next
Err.Clear
WSStart = .Item(FirstSheet).Index
If Err.Number <> 0 Then
    AverageNoZero = CVErr(xlErrRef)
    Exit Function
End If
WSEnd = .Item(LastSheet).Index
If Err.Number <> 0 Then
    AverageNoZero = CVErr(xlErrRef)
    Exit Function
End If
If WSStart > WSEnd Then
    AverageNoZero = CVErr(xlErrRef)
    Exit Function
End If

If RR Is Nothing Then
    AverageNoZero = CVErr(xlErrRef)
    Exit Function
End If

Addr = RR.Address
For WSNdx = WSStart To WSEnd
    Set DataRange = .Item(WSNdx).Range(Addr)
    For Each R In DataRange
        If IsNumeric(R.Text) Then
            If R.Value <> 0 Then
                N = N + 1
                V = V + R.Value
            End If
        End If
    Next R
Next WSNdx
End With
If N = 0 Then
    AverageNoZero = CVErr(xlErrDiv0)
    Exit Function
End If
AverageNoZero = V / N
End Function
'>>>>>>>>> END CODE

 

Here, FirstSheet is the first sheet to examine. LastSheet is the last sheet to examine. RR is the data range to average. This range may reside on any of the worksheets -- it doesn't matter since the code gets the address of the range as a string and applies it to each sheet.

 

The function will return a #REF error if either FirstSheet or LastSheet does not exist, if LastSheet is to the left of FirstSheet, or if RR is Nothing. It will return #DIV/0 if no non-zero values were found.

 

You can call the function from a cell with something like

 

=AverageNoZero("Sheet1","Sheet4",A1:A10)

 

This will return the average of the non-zero values in range A1:A10 on all worksheets from Sheet1 to Sheet4 inclusive.

Settings\<user name>\Application Data\Microsoft\Excel\XLSTART
folder.

  1. Click Save.

swischner say:
It's always hard to tell, where particular sync problems are coming from. If only one page or section is affected, most of the times some content or just an internal reference ID to the page got corrupted. Sometimes it's a glitch on Microsofts servers and all one can do is hope, that it will sync at a later time. But I would not rely on that.

In general, unsynced data lives only in the local cache (independent of the OneNote version used) and that is a dangerous situation.

What I do in these (thankfully rare) situations:

Create a new, empty section (always better than just a page, because sections are complete, separate files in OneNote's internal data structure) with a different name and then copy (not move!) all pages from the "old" to the new section. Ther is a command in the context menu of the section for that. Now see if the new section syncs (the sync error for the old section will most likely remain). Cross check the synced content in OneNote online. If all is good, you may delete the old section and rename the new one to the original name.




Bit off topic and a bit know-it-all: (:))
There is no such application as OneNote 365. But Microsoft themselves are struggling with the naming of their OneNote (and Office) versions.

The Desktop version of OneNote for Windows is named like the corresponding MS Office, e.g. OneNote 2016
The OneNote app that comes with Windows 10 is just called OneNote (or "Store version" or "OneNote UWP". "UWP" stands for "Universal Windows Platform" and means all new, tablet optimized apps, which were formerly called "Metro apps"). Best refer to it as "OneNote Windows 10 app".
OneNote Online is the version that runs inside any web browser.

For a date in a1

 

First workday of the month

 

=DATE(YEAR(A1),MONTH(A1),1)+CHOOSE(WEEKDAY(DATE(YEAR(A1),MONTH(A1),1)),1,0,0,0,0,0,2)

 

and the last

 

=DATE(YEAR(A1),MONTH(A1)+1,0)-(MAX(0,WEEKDAY(DATE(YEAR(A1),MONTH(A1)+1,0),2)-5))
