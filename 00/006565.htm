Might be worth making sure your code explicitly destroys any object variables you create, by setting them to nothing in inside-out sequence (Range before sheet before book etc)


I don't know how much difference that may make, but it is actually a practice I try to follow in my own coding, and I always explicitly Set object = Nothing at the end of my code even though, in theory, Excel is supposed to destroy the objects when the Sub/Function terminates.

Typically I'll wait until the end to do all of that housekeeping/cleanup, but I definitely don't see it hurting to do it immediately after you are finished using an object.

In this case, where there are so many iterations and such a large quantity of data it might even be worth trying to ensure the object resource is released back to the system during operation.  A trivial example:

Where we might normally depend on the system to properly reallocate resources, we might write:

For X = 1 to 100
   Set anyWS = Sheets(X)
   '... do more stuff with the anyWS object

Next

we might try being very explicit/controlling:

For X = 1 to 100
   set anyWS = Nothing  ' or could be moved to the end of the loop, but still inside of it
   Set anyWS = Sheets(X)
   '... do more stuff with the anyWS object

Next

john w. vinson say:

The Access relationships window is frustratingly limited and buggy. Tables vanishing offscreen is a common problem, as is the limited space available and annoying navigation limitations.

You can (sometimes) find and delete offscreen tables by selecting a table which is onscreen, removing your hand from the mouse, and using the TAB key to jump to another table; press the DELETE key to remove it from the diagram, and re-add it and position it where you want. A desperate extreme solution is to clear the diagram and make a new one by adding all tables; the tables will be in a square grid with lines every which way, necessitating tedious hours of rearrangement. Or you can add a "central" table and choose the option to add related tables, crawling the tree. 

I really wish MS had made the Relationships Diagram feature of SQL/Server Management Studio available for Access; you can have multiple diagrams (say for different logical groupings of tables) and other benefits. I once upsized a big Access app to SQL PURELY to get the diagrams for documentation.

fride360 say:

Wheel of Fortune game

Right now I have basically everything set up correctly.  The only issue I have right now is if a word is too long to fit on the top row, it continues on the second row instead of dropping the whole word down to the second row.  I tried to find a similar game online but didn't find any.  Does anybody have any idea how I can do this?  It might be hard for someone to help me if they don't know how I have it set up.

Thanks

tom ogilvy say:

Easiest would probably be to use a scatch area or another sheet where you have formulas to break down a phase in to the rows of your "board"

so it appear you have your string in G5.  Assume it is "This is a sample phrase"

It further appears you have 12 spaces in the top row, so we need to find where to break the string

A formula like this will show us where

=SUBSTITUTE(G5," ","#",13-LEN(SUBSTITUTE(LEFT(G5,13)," ","")))

that produces

"This is a#sample phrase"

I have used the # character to mark where to break the string.  I can then use the Find function against that result to get the break point

=FIND("#",SUBSTITUTE(G5," ","#",13-LEN(SUBSTITUTE(LEFT(G5,13)," ",""))))-1

returns 9.  So my first row will be Left(G5,9)

and I carry down  Mid(G5,11,255)

then I can repeat the process on that next row taking into account the number of spaces.

I would use as many cells as I need to make this a simple formula.  You can start combining formulas as you become more familiar with what you are doing.

Once you all the rows broken down, then you can use another section to center the words the rows of your board.  You can do this by padding spaces on the left, right or left and right as necessary.

Then the grid that populates the board can reference this second area to pick up the words. 

While excel has the ability to wrap text - there is no way to pick up the decisions it has made and utilize them in populating your board.  You have to design that intelligence yourself using formulas like shown above and others.    For example, to center you subtract the length of the line to be centered from the number of spaces available and put half that number of spaces in front and half that number of spaces after - then trim it to the total length.
