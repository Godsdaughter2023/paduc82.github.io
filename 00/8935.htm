a. user say:

Browsing history (under Tools / Internet Options / General / Browsing History)  is separate and distinct from autocomplete (Tools / Options / Content / Autocomplete / Settings). Autocomplete controls what appears in the address bar dropdowns.   If you want both of them deleted, you need to delete them both (you can choose in Autocomplete only to stop saving the browsing history aspect and not lose the forms completion and password and other aspects).

Setting the browsing history to delete on exit will not only delete on exit from that point forward but will also delete all your current and past history that is stored).  If you remove the autocomplete option to save the browsing history in the address bars then they will no longer be saved and ther'll be nothing to delete.

I hope this helps.

try this code,

Sub All_CSV_in_Separate_Shts()
'Mar. 07, 2018
Dim myWB As Workbook, wb As Workbook
 Set myWB = ThisWorkbook
 Dim fPath As String, fName As String
 '## delete old sheets named: sht_1/2/3/4....
 Application.DisplayAlerts = False
 For Each sh In myWB.Sheets
 If sh.Name Like "*sht_*" Then sh.Delete
 Next
Application.DisplayAlerts = True
'##
 Dim t As Long
 On Error Resume Next
 With Application.FileDialog(msoFileDialogFolderPicker)
 .Title = "please, select a folder"
 .AllowMultiSelect = False
 If .Show = True Then
 fPath = .SelectedItems(1) & "\"
 fName = Dir(fPath & "*.csv")
 If fName <> "" Then
 Application.ScreenUpdating = False
 t = 1
 Do Until fName = ""
 Workbooks.OpenText Filename:=fPath & fName, DataType:=xlDelimited, Local:=True
 Set wb = ActiveWorkbook
 wb.Sheets(1).Copy after:=myWB.Sheets(myWB.Sheets.Count)
With ActiveSheet
.Name = "sht_" & t
 .UsedRange.EntireColumn.AutoFit
 End With
 wb.Close False
 t = t + 1
 fName = Dir()
 Loop
 Application.ScreenUpdating = True
 Else
 MsgBox "no files"
 End If
 Else
 MsgBox "cancel"
 End If
 End With
End Sub

VLOOKUP isn't case sensitive, North Bay will be matched if the lookup value is nORth BaY or any other combination of upper/lower case.

 

FALSE in vlookup makes it find an exact match but NOT case sensitive. TRUE allows for an approximate match. If you use the TRUE swith then column 1 of the lookup vector must be sorted ascending or you could get incorrect results.

bob jones aka: cybertaz say:
There is no direct method built into Excel or Outlook for Mac.

The Excel sheet first needs to be saved as a Text file. Any of the .txt file types can be used but Tab Delimited Text (.txt) is probably best.

You can the go into Outlook's File> Import wizard to import the data. To make things easiest on yourself;
  • Only the records to be imported should be contained on the sheet
  • The columns should have field names starting in cell A1
  • The field names should match those used by Outlook [in order to minimize the amount of mapping required when importing]
  • The records should be consecutively listed beginning in row 2 -- no empty columns in the data range, no empty rows separating records
If this is something that needs to be done repeatedly you may be able to automate the process [e.g., with an Automator Workflow, VBA procedure]  or perhaps find an available example on the web. Unfortunately, the solutions I've seen offered for the Windows platform will not work on a Mac.
