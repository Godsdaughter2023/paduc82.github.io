ronrosenfeld say: Well, the "best" solution depends on a number of unknowns.

For example, if you only need to set this up once, you could just use formulas on the summary sheet, Once you have set up all 28*4 columns.

For example, and I used line 13 because it returns valid results for the four parameters:

B13:  =COUNTIFS(Master!$A$2:$A$725,A13,Master!$B$2:$B$725,"<>")
C13:  =AVERAGEIFS(Master!$B$2:$B$725,Master!$A$2:$A$725,A13)

And these formulas below must be **array-entered**:

D13:  =STDEV(IF(A13=Master!$A$2:$A$725,IF(Master!$B$2:$B$725<>"",Master!$B$2:$B$725)))
E13:  =GEOMEAN(IF(A13=Master!$A$2:$A$725,IF(Master!$B$2:$B$725<>"",Master!$B$2:$B$725)))
----------------------------------------
To **array-enter** a formula, after entering
the formula into the cell or formula bar, hold down
<ctrl><shift> while hitting <enter>.  If you did this
correctly, Excel will place braces {...} around the formula.

In addition, the formulas should be "wrapped" in an IFERROR function to return "" in the event of an error.  Furthermore, you may want to custom format the cells so as to not display 0's when that is the result.

Obviously, you will need to set up the header row manually, and change the "B" column to subsequent columns every fourth column.

If this sheet is going to be constantly updated, formulas may be better unless this plethora of array formulas causes performance to be slow.

If the Summary is better off being dynamically generated after each data collection, or if you run into trouble setting up the worksheet for all of the columns, we can certainly do that in VBA.  If performance is to slow, VBA "may" be helpful also.
