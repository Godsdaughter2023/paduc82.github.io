Is there a way to count how many images are in a document? - Microsoft Community

carolejeanx say:

Is there a way to count how many images are in a document?

I'm not finding an answer in all other online searches.  Does anyone know if this can be done?  Thanks.

jay freedman say:
Try this macro:

Sub CountImages()
    Dim inlines As Long
    Dim floaters As Long
    Dim sh As Shape
    Dim tbxs As Long
    Dim msg As String
   
    With ActiveDocument
        ' count text boxes, which are Shapes but not images
        For Each sh In .Shapes
            If sh.Type = msoTextBox Then tbxs = tbxs + 1
        Next
       
        inlines = .InlineShapes.Count
        floaters = .Shapes.Count - tbxs
    End With
   
    msg = "Inline shapes" & vbTab & inlines & vbCr
    msg = msg & "Floating shapes" & vbTab & floaters & vbCr
    msg = msg & vbTab & "Total" & vbTab & (inlines + floaters) & vbCr
    msg = msg & "Counts do not include headers/footers, etc."
   
    MsgBox Prompt:=msg, Title:="Count of Images"
End Sub

Assumes phone numbers are all in column A, starting with A1:
 
Formula Method (adapted from contribution by Harlan Grove):
 
First, create a Named Formula
Names in Workbook: Seq
Refers to: =ROW(INDEX($1:$65536,1,1):INDEX($1:$65536,255,1))
 
This ARRAY FORMULA
(committed with CTRL+SHIFT+ENTER, instead of just ENTER)
removes ALL non-numerics from the string and formats the result as you
desire:
 
 
=TEXT(SUM(IF(ISNUMBER(1/(MID(A1,Seq,1)+1)),
MID(A1,Seq,1)*10^MMULT(-(Seq<TRANSPOSE(Seq)),
-ISNUMBER(1/(MID(A1,Seq,1)+1))))),"000-000-0000")
 
Or, you can use a macro. The macro solution allows checking that you
have a valid phone number. In this instance, I "defined" a valid
phone number as being one which consists of exactly 10 digits; but you
can certainly vary this.
 
To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.
 
You may need to change the method of selecting the column of phone
numbers to process.
 
To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select
the macro by name, and <RUN>.
 
==================================
Option Explicit
Sub FormatPhoneNum()
Dim rg As Range, c As Range
Dim re As Object
Dim s As String
'Many ways to set up the range to check
Set rg = Range("A1", Cells(Cells.Rows.Count, "A").End(xlUp))
 
Set re = CreateObject("vbscript.regexp")
re.Pattern = "\D+"
re.Global = True
For Each c In rg
s = re.Replace(c.Text, "")
 
'Check for valid phone number entry
If Len(s) = 10 Then
c(1, 2).Value = Format(s, "000-000-0000")
Else
c(1, 2).ClearContents
End If
Next c
End Sub

US (and some other North American) phone numbers, when an area code is
included, will always be ten digits.
 
The formal definition is here:
http://en.wikipedia.org/wiki/North_American_Numbering_Plan
 
Given that a human may enter this in a variety of formats, and may
sometimes omit the area code, it is difficult to devise something
comprehensive.
 
If we restrict the entry to the NANP, and assume that separators may
be absent; or may be one (or more) in the set of
<hyphen>,<space>,<dot>, <right parenthesis>; and also that the
individual may enter 7 digits, 10 digits (or possibly 11 digits with a
leading 1), then we can use this regex to extract the first valid
phone number in the string:
 
"(([2-9][0-8]\d)[-.)\s]*)?([2-9]\d{2})[-.\)\s]?(\d{4})"
 
However, to reformat it as the OP wishes, if we allow seven digit
phone numbers, I don't think we can use the Replace method because I
don't know how to conditionally add that initial hyphen.
 
So, I would suggest (and hopefully the indents will post properly this
time):
 
==================================
Sub FormatPhoneNums()
  Dim rg As Range, c As Range
  Dim re As Object, mc As Object
  Dim sResult As String
  Const sPat As String = _
    "(([2-9][0-8]\d)[-.)\s]*)?([2-9]\d{2})[-.\)\s]?(\d{4})"
 
Set rg = Range("A1", Cells(Rows.Count, "A").End(xlUp))
Set re = CreateObject("vbscript.regexp")
    re.Pattern = sPat
 
For Each c In rg
    If re.test(c.Value) = False Then
        c.Interior.Color = vbRed
        c(1, 2).ClearContents
    Else
        Set mc = re.Execute(c.Value)
        sResult = mc(0).submatches(2) & "-" & mc(0).submatches(3)
        If Len(mc(0).submatches(1)) = 3 Then
            sResult = mc(0).submatches(1) & "-" & sResult
            c.Interior.ColorIndex = xlNone
        Else
            c.Interior.Color = vbGreen
        End If
        c(1, 2).Value = sResult
    End If
Next c
End Sub
=====================================
 
With a few caveats:
 
1.  When doing things like this, where the conversion might be
incorrect, I make it a practice to put the results in the adjacent
column.
 
2.  The sub will highlight any entries that it thinks do not include a
valid phone number.
 
3.  The sub will pick up the first combination of numbers, with valid
(or no) separators, that comprise a valid seven or ten digit number.
If the area code is invalid, then it will not be included.   It should
be relatively easy for the OP to pick out entries with no or invalid
area codes, but I've highlighted them also, in a different color,
assuming that the OP will want to manually add in the proper area
codes.
 
4.  As before, the sub selects column A values, but this could be
changed.
