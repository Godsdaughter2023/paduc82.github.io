joseph ban say:

To assist you further regarding your concern, we suggest that you manage the HomeGroup setup. To do this, you may follow these methods:

Method 1: Make sure that all your computers are connected to the same Home network


Verify each basic network information and setup connection under Control Panel. To do this, you may follow these steps:

  1. Click Start, type Control Panel and press Enter.
  2. On the Control Panel search box, type Network and Sharing Center and click it from the search results.
  3. Check if your computers are connected to the same network under the View your active networks section.
  4. Your devices should be connected to the same Home network.

Method 2: Check your Internet connection


All your computers should be connected to the same Internet connection. Check it under Network & Internet from the Settings. You may click Start > Settings > Network & Internet. Depending on your connection, check the Wi-Fi or Ethernet.

Restarting your device, trying to create or join to a HomeGroup again will solve this conflict.

Method 3: Make sure that the computers have the same correct time

Go to Settings > Time & Language and adjust time as needed.

Method 4: Make sure that the Private network connection under Network and Sharing Center from the Control Panel are configured properly


To manage the Private network connection, you may follow the steps below:

  1. Follow the steps 1 and 2 on Method 1 to open Network and Sharing from the Control Panel.
  2. Click Change advanced sharing settings and expand the Private section.
  3. Select the turn on option from Network discovery and File and printer sharing.
  4. Under HomeGroup connections, select Allow Windows to manage homegroup connections (recommended).
  5. Once you’re done, click Save changes and check if the issue persists.




Keep us updated about your concern if you need additional assistance.

Note:
Event logs are special files that record significant events on your computer, such as when a user
logs on to the computer or when a program encounters an error. Whenever these types of events occur,
Windows records the event in an event log that you can read by using Event Viewer. Advanced users might
find the details in event logs helpful when troubleshooting problems with Windows and other programs.
Event Viewer tracks information in several different logs. Windows Logs include:


• Application (program) events. Events are classified as error, warning, or information, depending on
 the severity of the event. An error is a significant problem, such as loss of data. A warning is an
 event that isn't necessarily significant, but might indicate a possible future problem. An information
 event describes the successful operation of a program, driver, or service.


•Security-related events. These events are called audits and are described as successful or failed
 depending on the event, such as whether a user trying to log on to Windows was successful.


•Setup events. Computers that are configured as domain controllers will have additional logs displayed
 here.


•System events. System events are logged by Windows and Windows system services, and are classified as
 error, warning, or information.


•Forwarded events. These events are forwarded to this log by other computers.

 

Applications and Services Logs vary. They include separate logs about the programs that run on your
computer, as well as more detailed logs that pertain to specific Windows services.
