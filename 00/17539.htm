One of the most common behavior of Windows Defender is that if it detects a third-party antivirus software, then Windows Defender will shut itself off. Windows is designed to detect the presence of security software installed, and we often we get to see the notification in the Action Center. One of the common issues we have seen in the past, is when there are multiple security software installed. It often creates a conflict between them and in the end, in creates more trouble than doing any good. Frequent freezing, crashing to applications are some of the common issues which occur when you install more than one antivirus software.

So to avoid this situation Windows Defender will turn itself off when it detects another security program. Likewise if this security application has a Firewall then Windows Firewall will also stop working. So in case you want Windows Defender to work, then you will have to uninstall your third-party security software and reboot the system.

Often users maybe unaware if a security software is indeed installed or not because when you buy a new Laptop mostly it comes with a trial version of security software like Norton, McAfee etc. Then when you install applications like Flash or java it might prompt you to uninstall free security scanner when most of them won’t notice. Trailing average calculation - Microsoft Community

jaygourley say:

Trailing average calculation

How do I define a function to sum cells in column based on time proximity?  I have a spreadsheet on which rows contain data timestamped in a specific column.  I'd like to calculate a trailing or leading sum of entries within some defined  time span.

joeu2004 say: Jay wrote:
How do I define a function to sum cells in column based on time proximity?  I have a spreadsheet on which rows contain data timestamped in a specific column.  I'd like to calculate a trailing or leading sum of entries within some defined  time span.

If you do not require Excel 2003 compatibility, you could write:

=AVERAGEIFS(B2:B1000,A2:A1000,">="&X1,A2:A1000,"<="&X2)

where X1 is the start date/time and X2 is the end date/time, A2:A1000 are the timestamps, and B2:B1000 are the data to be averaged.

If you require Excel 2003 compatibility, you would write could write:

=SUMPRODUCT(B2:B1000,(A2:A1000>=X1)*(A2:A1000<=X2))
/ SUMPRODUCT(ISNUMBER(B2:B1000)*(A2:A1000>=X1)*(A2:A1000<=X2))

Note:  If you are certain that B2:B1000 contains only numeric data (no empty cells or text), you could eliminate the ISNUMBER(B2:B1000) test.

jaygourley say:

Thanks for the suggestion.  But I need to define the time frame relative to the timestamp. 

 

I would like a running number so that each line shows the sum of values within one week of that line.   Every line is identical in format, but dates don't have a fixed number of lines.  One day may include three or four lines, each with its own timestamp.  Another day may not have any entries. 

 

So I'm looking for a way to tell Excel: "sum those values in column five if the timestamp in column two of that value's row is within one week of the timestamp in this row."

 

joeu2004 say: Jay wrote:

Thanks for the suggestion.  But I need to define the time frame relative to the timestamp.  I would like a running number so that each line shows the sum of values within one week of that line.   Every line is identical in format, but dates don't have a fixed number of lines.  One day may include three or four lines, each with its own timestamp.  Another day may not have any entries.  So I'm looking for a way to tell Excel: "sum those values in column five if the timestamp in column two of that value's row is within one week of the timestamp in this row."


If you do not require Excel 2003 compatibility (it would be nice if you told us if you do), you could write:

=IFERROR(AVERAGEIFS($E$2:$E$1000,$A$2:$A$1000,">"&A2-7,$A$2:$A$1000,"<="&A2),0)

where A2:A1000 contains the timestamps and E2:E1000 contains the data ("column five").


=IFERROR(AVERAGEIFS($E$2:$E$1000,$A$2:$A$1000,">"&A2-7,$A$2:$A$1000,"<"&A2+7),0)

Change ">" and "<" to ">=" and "<=" if you want to include the date 7 days before and after the current date.

i_miller say:

The Dell tech support guy found the problem- and it is the Dell printer driver!  It turns out that they have several drivers available for this printer (an H625).  The PCL6 driver has the bug that misleads Word about its duplex capability.  However, the PS driver works fine.  I switched to it, and the problem is resolved.
