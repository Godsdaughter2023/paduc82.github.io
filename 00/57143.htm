One of the reasons for not using Windows 8's File Recovery, it still has the same problems as Windows 7's Backup and Restore.

 

===============================================

 

How to troubleshoot Windows Backup and Restore issues when a reparse point folder or its subfolder is added to a user library in Windows 7
http://support.microsoft.com/kb/973455/en-us


Backup issues
Issue 1: Error code 0x80070001
Issue 2: Error code 0x81000037
Restore issues
Issue 3: You cannot locate the files in the File Restore Wizard
Issue 4: Error code 0x80070003


Reparse Points
http://msdn.microsoft.com/en-us/library/aa365503(VS.85).aspx


You can create a text file with all the entries as follows:


1) Click on Start and type CMD in search bar


2) Right click on the cmd.exe entry in the "Programs (1)" list and select "Run as administrator"


3) At the command prompt which will be: C:\Windows\system32>    enter the commands listed below.
3a) C:\Windows\system32> CD C:\users   and press the enter key
3b) Next enter:  dir   and press enter
one of the entries in the list will be your account name, example: My Name
3c) now enter: CD "My name"   and press enter


4) The command line prompt should now be:  C:\Users\My Name:   where 'My Name' represents your account name.


5) Now enter the following at the prompt:  dir  /al  /s  >  C:\Results.txt


6) Open Windows Explorer and navigate to C:\    you should see the "Results.txt" file you just created.


7) use the content of this file to review all the junction points. None of which should point to files located on the backup drive. Note: Trying to backup files on the USB drive to the USB drive can cause the error 0x81000037.


Also see what one user did to solve this problem: http://answers.microsoft.com/en-us/windows/forum/windows_7-system/backup-error-81000037/59cb6fa8-2c8a-4dde-a0e8-9e4aa7bf5629

The correct answer is that there are no full versions of Windows 8 except OEM and System builder versions.  If you bought Win 8 retail, you got an upgrade version.  Because of this MS requires you to install a previous OS before installing Win 8.

That is the correct answer.  Now, if you are like me, and your previous OS is XP then you are in a bit of a pickle.  My assumption is that I can no longer install and activate Windows XP as support as expired, giving me no way to actually install a previous OS to upgrade.

In that spirit, there is a workaround.

1. Press Windows Key + R combination, type Regedt32.exe in Run dialog box and hit Enter to open the Registry Editor.

2. Navigate to the following location: HKEY_LOCAL_MACHINE/Software/Microsoft/Windows/CurrentVersion/Setup/OOBE

3. In the right pane of this location, you’ll see MediaBootInstall named DWORD, which has Value data of 1. Double click on this DWORD to modify it.

4. Change the Value data to 0 from 1. Click OK. Close the Registry Editor.

5. Now open administrative Command Prompt (Windows Key + X, select Command Prompt(Admin)) , type following command: slmgr /rearm and hit enter.

6. Click OK to the dialog box and reboot.

This should now allow you to activate, or if not, it will at least give you the link to activate by phone that was missing before.  Mine didn't like the fact that I was re-installing and it spit out another error that the key was in use by another computer (the one I was rebuilding), but it did allow me to activate by phone.

Good luck!
