Cutting the Corners off Text Boxes - Microsoft Community

goober4473 say:

Cutting the Corners off Text Boxes

I'd like to cut two of the corners off of some text boxes, but I'm having two problems:

1. The default shape that looks like this has too large of cuts. I can modify this, but there's no way to perfectly mirror my edits to the points of the shape.

2. The cuts change in size and aspect ratio along with the text box. What I'd like is to give all of the text boxes the same sized cuts, no matter their overall size.

Is there a way to solve either of these problems?

jay freedman say:

After some poking about in an area of the object model that I've never previously bothered with, I've come up with a tiny macro that will set the corners of all your boxes the same.

In the macro language, an AutoShape (which is what your boxes are, and technically not text boxes despite the fact that they can contain text) has a list of "Adjustments" that represent the various handles for changing the shape's proportions. For your boxes, which go by the name msoShapeSnip2DiagRectangle, the second of these Adjustments is the one that controls the top right and bottom left corners' size.

The gotcha here is that the value of that particular Adjustment is expressed as a fraction of the width of the shape, rather than as an absolute measurement. That requires doing a little arithmetic: the value you want to assign to the Adjustment is the desired absolute size (the distance from the top right corner to the yellow handle) divided by the shape's width.

The following macro will set the size of the corners in all of the "text boxes" in the whole document to the same value, which you can edit in the macro code where noted. (See http://www.gmayor.com/installing_macro.htm if needed.) 

Sub AdjustCorners()
    Dim oShp As Shape
    For Each oShp In ActiveDocument.Shapes
        If oShp.AutoShapeType = msoShapeSnip2DiagRectangle Then
            With oShp
                .Adjustments(2) = InchesToPoints(0.25) / .Width  ' <== change 0.25 to desired size in inches
            End With
        End If
    Next oShp
End Sub

You can run the macro multiple times on the same document -- shapes that already have the right corner size will remain that way, and any that haven't been set properly will be fixed.

=IF(COUNTIF(Sheet2!A:A,A1)>0,"",A1)

=OFFSET(Sheet1!$C$1,0,0,COUNTA(Sheet1!$C1:$C78),1)

 

In Sheet2 column A, data validation:

 

List, Source =MyNames

Each name you select in data validation results in an empty string in sheet2 column B.

graham mayor say:

If your document is already split into sections i.e using section breaks, then you can use the macro Sub Splitter() near the end ofhttp://www.gmayor.com/individual_merge_letters.htm to split the document by section into separate documents. Frankly it is time I updated that macro, so I would offer the following version. The macro splits the document at the section breaks and names the documents in the format "Chapter n.doc". The original document is not affected, though I would recommend using it on a copy of the document.

http://www.gmayor.com/installing_macro.htm

Sub Splitter()
Dim Letters As Integer
Dim Counter As Long
Dim DocName As String
Dim oDoc As Document
Dim oNewDoc As Document
Set oDoc = ActiveDocument
oDoc.Save
Selection.EndKey Unit:=wdStory
Letters = Selection.Information(wdActiveEndSectionNumber)
Selection.HomeKey Unit:=wdStory
Counter = 1
While Counter < Letters
     DocName = "D:\My Documents\Test\Merge\Chapter " & Counter & ".doc"
     oDoc.Sections.First.Range.Cut
     Set oNewDoc = Documents.Add
     'Documents are based on the Normal template
     With Selection
         .Paste
         .EndKey Unit:=wdStory
         .MoveLeft Unit:=wdCharacter, Count:=1
         .Delete Unit:=wdCharacter, Count:=1
     End With
     'Do what you want with the new doc before it is saved here
     oNewDoc.SaveAs FileName:=DocName, _
     FileFormat:=wdFormatDocument, _
     AddToRecentFiles:=False
     ActiveWindow.Close
     Counter = Counter + 1
Wend
oDoc.Close wdDoNotSaveChanges
End Sub

If your document does not have section breaks it wouldm't be that difficult to include the code to add them before splitting.
