c: In the Internet Options dialog box, click the Security tab.

d: Click Default Level.

e: Click OK.

Do not perform Method 2, if Method 1 resolves the issue. You can try Method 2 if Method 1 does not work.

Method 2:

Every time that you open a browser to view a webpage, your computer stores a local copy of that webpage in a temporary file. If the size of the temporary Internet files folder becomes too large, some display problems may occur when you open webpages. Periodically clearing the folder might help to resolve the problem.

don cuthbert say:

I had trouble seeing network shares from 17083 yesterday. What worked was to use the shared device's IP Address, instead of the network name.  In my case I had to look for \\172.16.1.66\shared in File Explorer.  Then I entered my MSA user name and password and saved those credentials to File Explorer.

I had also tried this command at a Command Prompt, which sometimes helps:

net use \\172.16.1.66\shared

I entered my user ID and password when prompted (using my MSA account info).

Of course, your MSA needs to have the required permissions on the network shared folder to access it.
