Fix for “Office is Busy” and Office repair doesn’t work.

I had this issue on several computers and then found this fix.  It made Office run quicker and the annoying “Office is Busy” messages stopped.

Open any of the Office programs (Word, Excel, etc.).  Go to the File Tab, then find “Account” and click on this.  On the right side of the screen, your should see “Office Updates” shown with a box to the left of the title.  If you click on “Update Options” select “Turn off Updates”.

No more “Office is Busy” messages and no more slow responding Office windows.

 

<snip> I have been having this problem, as well.  While performing an installation repair operation has worked for me, it has taken a lot of time, and requires closing and re-opening all of my Office applications.  A post on another thread mentioned a better solution that has worked for me: Simply stop or restart the “Microsoft Office Service”. 

  1.    You can do this by using the Control Panel >  Services applet.  Find the service name in the list
  2.    Right click on the service name
  3.    Select Properties
  4.    Click on the stop button, wait to see a message that the service has been stopped
  5.    Click on the start button

 It seems to have resolved the problem within seconds, and it is much less disruptive (though not nearly as nice as a real fix for the problem).  I hope this helps. </snip>

In that situation, it may be easiest to create a batch file to issue the commands.  Then you can put it on your desktop or task bar to have quick access to run it as required.  The brain dead simplest form of this .BAT file would be:

net stop “Microsoft Office Service”

net start “Microsoft Office Service”

but it is better if you do some error checking to make sure you know what is going on if there are any problems. This page has a detailed batch file you can use as a template:

http://forums.techguy.org/windows-xp/831463-batch-script-restart-service.html

net stop

if %errorlevel% == 2 echo Could not start service.

if %errorlevel% == 0 echo Service started successfully.

 

net start “Microsoft Office Service”

if %errorlevel% == 2 echo Could not start service.

if %errorlevel% == 0 echo Service started successfully.

linking or import, which one is better ? - Microsoft Community

tobemaster say:

linking or import, which one is better ?

Hi,

I,m learning Access.

I want to make a table from Excel spreadsheets but I am not sure whether I use link or import excel.

 

Everyday employees submit their labor sheets in the morning and I have to make some reports based on these labor time sheets.

I have to collect these labor sheets and then put these in the excel.

After that I want to make a table based on this excel.

 

Which one is better to make forms and queries and reports?

 

Thanks

 

ken sheridan say: If it is merely a question of using the Excel data as a basis for your Access reports then it would be better to link to the Excel worksheet.  Any changes to the data made in Excel will be reflected in the linked Access table.  You will also be able to create forms in Access based on the linked table, but these will be read-only.  You only really need to import the data if you want to make changes to it after importing.

tobemaster say:

" You will also be able to create forms in Access based on the linked table, but these will be read-only."

 

 

I have a tblEmployee table and in this table , I got EmployeeID.

In the linked EmployeeTimeSheet table has employeeID, too but duplicated.

 

I'd like to connect both table so I can get data from tblEmployee.

How can I do that?

EmployeeTimeSheet table doesn't have primary key.

Should I make a new table for EmployeeTimeSheet?

Can I use Action query to do that?

 

 

ken sheridan say:
Should I make a new table for EmployeeTimeSheet?

Only if you need to change the timesheet data in Access.  If so design an EmployeeTimeSheet  table in Access so that its columns exactly match those in the Excel worksheet.  You can then link to the Excel worksheet and use an 'append' query to insert rows from the linked table into the Access EmployeeTimeSheet table.  This can be joined to the tblEmployee table on EmployeeID to create a query for use as the RecordSource of a form and/or report.

If you don't need to change the timesheet data in Access, just join the tblEmployee table to the linked table.

tobemaster say:

Thank you

I got a problem with my linked table, tblEmployeeTimeSheets.

In this tblEmployeeTimeSheets, there are StartTime, EndTime.,and WorkHours(calculated field: EndTime-StartTime).

 

Format for these columns in Excel was like 12:00 AM.

But after I linked this excel into Access, datatype in tblEmployeeTimeSheets is like 12/30/1899 12:00:00 AM.

I tried to fix but changing format doesn't work in table design but in frmEmployeeTimeSheets, I can drop '12/30/1899' because I can format to Medium Time.

Is this ok?

 

 

ken sheridan say: The reason you see the time like that is because Access implements the date/time data type as a 64 bit floating point number in which the integer part represents the days and the fractional part the time of day.  It starts from 30 December 1899 00:00:00, which is represented by zero.  So there is really no such thing as a time value per se, only a number representing a date/time value.  This does not matter as you should never interface with the data via a table's raw datasheet view, but only via forms or reports, in which the value can be formatted however you wish.

Where it can make a difference is if the date/time data type is being used to store durations of time of less than 24 hours.  There is nothing wrong with doing this as all that's being stored in reality is the duration in days, which can be formatted as normal time format.  If you sum a set of such values, however, you can't then format them as time, as the result will always show as less than 24 hours even if the times sum to greater than that.  This can be overcome by writing functions to return the result in a correct format, and you'll find examples in the demo file Timesheet.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.
