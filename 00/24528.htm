nayana s m say:

Hello Brad,

As I understand you would like to change the ‘Search Engine’ when right click on any content within any Office application.

In order to change that you need to make changes to the Office registry keys.

Follow the steps provided below on how to make changes to the registry key:

Disclaimer: Back up the registry key before you modify or delete any key by following the steps below.

For more information about how to back up and restore the registry, follow these steps to back up the registry:

  • Press the Windows + R keys to open the Run dialog box.
  • Type in the following text, and then press Enter.
  •          regedit
  • If the User Account Control window appears, click Continue.
  • On the File menu, click Export.
  • In the File name box, type a name that you will remember, such as Registry Backup.
  • Select a location where you want to save the Registration Entries (.reg) file.
  • Click Save.

1. Click Start->Run (Windows key + R key) and type regedit in the ‘Run’ command window ->click ok.

2. Locate the registry key HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Common\General

3. In the right pane of ‘General’ > Create a ‘New String key’ by right clicking on the anywhere in the pane and click on ‘String value’ and name it as ‘SearchProviderName’. Right click on it and click on ‘Modify’. Mention the search provider name’ as Google in ‘Value’.

4. Create another string key and name it as SearchProviderURL. Right click on it and click on ‘Modify’. Enter the value as http://www.google.com/search .

5. Once done, close the registry window and check if this as enabled the ‘Search with Google’ option.

Whenever I start Outlook 2013 (from Office 365), my two Exchange ActiveSync (EAS) accounts on Live.Com always remain "Disconnected".  The best bandaid so far that I've found came from a different thread.  Go to the "Send/Receive" tab, click "Work Offline" to go into "Offline" mode, then click "Work Offline" again to go back "online" and the disconnected accounts with connect (at least for me).  This bandaid has proven very consistent for me.  From then on the accounts will remain "connected" until I shut down Outlook and start it up again.  At that point they'll be disconnected again, and I'll have to repeat the bandaid.

It is definitely a bug that Microsoft needs to fix, as confirmed by many people.  It needs to be given some priority since it can be detrimental to a person's business uses of Outlook.  If the account is left unknowingly disconnected, the user will not realize that s/he is not receiving their emails.

cbarnhorst say:

When you roll back to a previous build the abandoned build is recorded in the registry so that Windows Update won't offer it again.  The reasoning is that it wasn't working for you so you don't want it to show up again.  The build number is recorded under the Applicability Key as "Recovered from 14915" (or whatever the number was of the abandoned build).  Changing the build number to "0" or deleting the Recovered From key will allow WU to again offer it.  This is what the Applicability key looks like, but I don't have a rollback recorded so there is no Recovered From key in the snip.

If you are comfortable with regedit you can reset your registry so you can get 14915 again.  My advice is simply wait for the next new build to flight and go from there.  It will probably be a week.  Only use the above method if you know how to export and import registry keys (a backup precaution).
