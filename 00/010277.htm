bernie deitrick say: Use code like this, storing the random numbers in column A of the worksheet: "Storage" (which you need to add to your workbook) - you may want to store other information along with the number, in case you ever need to retrieve a user's specific information

Private Sub cmdGenerateID_Click()
    Dim Low As Long
    Dim High As Long
    Dim r As Long

    Low = 100000
    High = 999999
getNum:
    r = Int((High - Low + 1) * Rnd() + Low)
    If IsNumeric(Application.Match(r, Worksheets("Storage").Range("A:A"), False)) Then GoTo getNum
    Worksheets("Storage").Cells(Rows.Count, "A").End(xlUp)(2).Value = r
    Me.Arec5.Value = r

End Sub

bernie deitrick say: Run this macro once, with an new blank sheet named "Number List" active (you can use whatever name you want):

Sub GenerateList()
Range("A1").Value = "Numbers Available"
Range("B1").Value = "Randomize"
Range("c1").Value = "Used Numbers"
Range("A2:A900000").Formula = "=ROW()+99998"
Range("A2:A900000").Value = Range("A2:A900000").Value
Range("B2:B900000").Formula = "=RAND()"
Range("A:B").Sort key1:=Range("B1"), Order1:=xlAscending, header:=xlYes
Range("B:B").EntireColumn.Delete
End Sub

If you will never use all 900000 numbers, you can shorten the list (and make the workbook smaller) by code like this - say you only need 10,000 numbers:
Range("A10001:A900000").Clear
just before the End Sub line.

Then when you want to use a number:

Sub UseNumber()
Dim v As Variant
With Worksheets("Number List")  'change this to the actual sheet name
v = .Cells(.Rows.Count, "A").End(xlUp).Value
.Cells(.Rows.Count, "A").End(xlUp).Cut .Cells(.Rows.Count, "B").End(xlUp)(2)
End With
MsgBox v
'use v to place the value wherever you need it
End Sub

It is not easy defeating all of the automation built into Word.

You can control the resizing by clicking inside of the table, then on the Table Tools > Layout contextual tab > Alignment group > Cell margins drop down > and uncheck the option to "Automatically resize to fit contents"

This controls the cell width. But it does not stop text from wrapping to a new line and pushing the table down.

If you want to control row height, you have to explicitly define the row height. Select the row, or rows, or whole table. Right click. Select Table Properties. In the Row tab click in the check box to turn on Specify Height. Set the Height you want and MAKE SURE that "Row Height is" drop down is set to "Exactly"

The problem with this is that it does not limit the number of characters entered in the cell. So text can wrap down, out of sight. Automation ...

FITTING YOUR TEXT IN A CELL

This tip (11572) applies to MS Word versions: 2007 | 2010

jay freedman say:

I can think of two scenarios that match the behavior you see. Neither of them is quite probable.

If you save AutoTexts (or styles or macros or some other things) in Normal.dotm, you should see a prompt when you shut down Word, asking whether to save changes in Normal.dotm. If you click Do Not Save or press the N key, the things you think you saved are discarded. The next time you start Word, those things will not be there. This could be the result of not reading the prompt, or (because it often immediately follows a prompt asking about changes in the document being closed) it could be the result of accidentally clicking twice in the same place.

The other scenario is a decision by corporate IT to overwrite each user's Normal.dotm with an approved file at each login. That probably is not the case in your location, but it's worth checking whether it has recently changed.

There is one piece of information that would be useful: After saving an AutoText in Normal.dotm and closing Word, being careful to click Save in the prompt, use Windows Explorer to go to the Templates folder (%appdata%\Microsoft\Templates, unless it was modified during setup) and check the Modified Date of the Normal.dotm file. Is it the current date and time, or something older?
