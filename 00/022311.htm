ronrosenfeld say: Thanks for stimulating me to look in this direction.

I have done some investigation and I believe I have uncovered the interpretation of the numeric array argument when setting up autofilter for the Date field.

Looking at both the behavior on the worksheet, as well as the contents of the xml file that is the worksheet (open the Excel file as a zip file, and examine sheet1.xml, it is apparent that those numbers refer to the level that is checked in the dropdown for the date.

A relevant line in the xml file might look like:

<dateGroupItem dateTimeGrouping="month" month="12" year="2012"/>

The interpretation of that numeric argument seems to be:

0  year
1  month
2  day
3  hour
4  minute
5  second

and the date/time string will expand in the above argument so as to show all the relevant components.
