{IF { MERGEFIELD What_Color_Apples } = "Red" "The apple is not yellow" "" }{IF { MERGEFIELD What_Color_Apples } = "Green" "The apple is not yellow" "" }{IF { MERGEFIELD What_Color_Apples } = "Yellow" "The apple is yellow" "" }

macropod say:

Certainly, if there are 120 different text descriptions, 120 IF tests may be required, though for your second example they could be reduced to:

The apple is {IF{MERGEFIELD What_Color_Apples}<> "Yellow" "not"} yellow.

After all, it's either yellow or it isn't - you don't need to test all 120 possibilities to establish that!

Where numeric data are involved, it's often possible to use a few formulae to simplify processing.

thomas-4543 say:

That wouldn't work for me. There are about 120 different options that would produce about 30 different answers.

macropod say:

Unless you can provide more details about what your options actually consist of, it's impossible to advise on how the process might be streamlined. Also, depending on your data source, you may be able to add a field that outputs either the answer (thus eliminating the IF tests in Word altogether) or at least a code that Word can apply an IF test to in order to generate that answer (reducing the number required to no more than 30).

There are also other approaches that can eliminate the IF tests altogether if an answer code is output, via the use of INCLUDETEXT fields.
