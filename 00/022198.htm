herb tyson say:
At this point, the homework assignment has passed, but it occurs to me that this could become a useful thread, and we might take advantage of its being here by creating a list of recommended practices, even if I personally hate the term "best practices". Here are a few things I recommend:
  1. Use the styles Heading 1, Heading 2, etc. to organize reports and other longish documents. This not only helps the reader understand how your document is organized, but it also lets you easily view the document's overall structure in outlining mode or using the Navigation pane. Moreover, it lets you easily reorganize if/when you need to.
  2. Use Word's caption and cross-referencing features for tables, figures and other numbered items. Not only does the built-in captioning feature automatically number your tables and figures, but it lets you move, delete, and insert items without having to manually renumber everything. And, when you're done, it lets you automatically create a list of tables/figures for front matter.
  3. Commit Ctrl+Z and Ctrl+Y to your fingers' memory. They are Undo and Redo, respectively. Using them, you can step backwards and forwards through the current session's history of writing/editing a document.
  4. Forget what your 1960s typing teachers taught you about double-spacing between sentences. When using proportionally spaced fonts (i.e., almost always), the contemporary standard is single spacing between sentences. In addition to following contemporary publishing standards, single spacing after sentences saves bits and reduces file sizes.
  5. In Word 2007 and later, to see more of your document on screen, Ctrl+F1 toggles the ribbon on and off. Clicking any tab brings it back momentarily when you need it, and (of course) Ctrl+F1 brings it back period.
  6. Get into the habit of pressing Ctrl+S (Save) frequently when you're writing.
  7. When using Word 2007 and later versions, put your frequently-needed tools onto the quick access toolbar. You can do this by right-clicking the tool in the ribbon, and choosing Add to Quick Access Toolbar.
  8. Move the Quick Access Toolbar (QAT) to below the ribbon. Do this by right-clicking the QAT and choosing Show Quick Access Toolbar Below the Ribbon.
(Please feel free to add to the list!)

suzanne s. barnhill say:

I'd second all of those and add that using styles generally (not just the heading styles) is a great timesaver. If you apply styles based on function, you can forget about formatting while you're writing. For example, in a book-type document, you will typically have Body Text (with no first-line indent) following headings and Body Text First Indent for the remainder of the paragraphs. If Body Text (instead of Normal) is assigned as the "Style for following paragraph" for all the heading styles and Body Text First Indent as the following style for Body Text, this is automatic. Use Block Text for block quotes and other paragraph styles as required, based entirely on function, not formatting.

When you have finished editing or at some other point when you have enough text to get an idea of the results, you can work on modifying the styles to look the way you want--changing the font, line spacing, indents, etc. Because you have applied styles to the paragraphs, you need make these changes only once, in the styles themselves, rather than in every paragraph. This allows you to concentrate on writing or editing without thinking about formatting and vice versa.

Another instance when procrastination is in order is in setting tab stops. If you're going to have several paragraphs in which you want the same tab stops, but the text between them varies in length (in a form, for example), type all the lines with just a single tab character between elements. It will be a jumbled mess, but after you've typed all the paragraphs so that you can get an idea of where the tab stops need to be, you can select them all and insert the tab stops as required, dragging them around until you get just the right arrangement.

Ditto for tables: Don't bother to adjust column widths until you get a feel for the amount of text that will be going into each one. If auto resizing is enabled (which it is by default), Word will "help" while you're inserting text, but be sure to turn auto resizing off when you get down to the task of establishing the column widths you want.

Another important concept, related to styles: Whenever you need to create the same type of document repeatedly, save yourself work by creating a template. This can store not only your styles, formatted as required for that type of document, but also fields and boilerplate text to save you time.

Target in Hyperlink in Word? - Microsoft Community

gulfbreezey say:

Target in Hyperlink in Word?

Can you put a target statement in a hyperlink you create with Word?  Since it isn't really letting me put in HTML I don't quite know if I can do this but it sure would be nice to have my word documents no open a web page over top of themselves :>)

a. user say:

There are two different things:

  1. the location in the hyperlinked document to open, or to move to
  2. which window in a browser you want to open the hyperlinked document in.

Using bookmarks is how you do (1), as you describe.

The original question was about (2) - if for example you save a Word document with hyperlinks as an HTML file, open that in a browser, then follow one of the hyperlinks, does the hyperlinked document replace the existing one (i.e. does it open in the same tab or window) or does it open in a new tab.

Word's Hyperlink field has a \t option that lets you specify, for example, \t "_blank" to open the link in a new window or tab.

Although you can insert this option in a HYPERLINK field in a Mac Word document, Mac Word ignores it, and does not generate the appropriate HTML.

In Windows Word, Word will generate the HTML for a link with one of the standard targets, e.g. if you have a HYPERLINK field such as

{ HYPERLINK "answers.microsoft.com \t "_blank" }

then Word will generate HTML like this:

<a href="answers.microsoft.com" target="_blank">

jim gordon mvp say:
There's no way to do this in Word's interface.

However, the file format Word uses is the international standard: Office Open XML. A Word file is a zip document, which you can open and edit by changing the file extension from (.docx) in Finder to (.zip).  Then double-click the zip file to see its contents. If you know HTML you should be able to find the hyperlink, as everything is saved in XML (plain text). The add your target code, save the XML file, and then zip the folder. Change the file extension back to (.docx) and everything ought to work.

connor s say:

If your numbered list starts in A1 (that has a number you've input for the start, i.e. 1), then in cell A2 the formula would be: =MAX($A$1:A1)+1

Then you can copy that cell to any place else in column A to continue the numbering.

Sheet2 would work the same way but instead of putting a number in cell A1 you can put: =MAX(Sheet1!A:A)+1
