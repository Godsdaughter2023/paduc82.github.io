albert d. kallal say:

You quite much have to "store" the formatted text.

In the past, even with VBA, I often written a bit of code, and thus we store a text (string) in place of an actual number.
 
And adding a wee bit of code makes data entry real easy.
 

So, in the control after update event, you can place this code:
 

If len([FaxNumber]) = 7 Then
   SetProperty (FaxNumber, Value, ="(780) " & left([FaxNumber],3) & "-" & mid([FaxNumber]4,6))
End if

If len([FaxNumber]) = 10 then
   SetProperty (FaxNumber, Value,="(" & Left([FaxNumber],3) & ") " & Mid([FaxNumber],4,3) &

     "-" & Mid([FaxNumber],7,4))
End if

 

So, the above makes data entry quite easy. If a user types in

1234567
 
Then the text becomes
 
(780) 123-4567
 
So, the code would insert the default area code and nice () – makes for nice quick data entery.
 
If the user types in the area code, say this:
 
4031234567
 
Then the text becomes this:
 
(403) 123-4567

Excel sorting is not in ASCII order - Microsoft Community

usermcw say:

Excel sorting is not in ASCII order

Does Excel sort according to some other standard? 

 

jeeped say:

... Maybe Microsoft had some reason for doing it that way ...


The soft order that Excel uses has to do with language. If a strict ASCII code ordinal was used, a lower case a would come after the upper case Z and several special characters. When Excel sorts alphabetic strings, all of the following characters come before a B,
A a ª Á á À à Â â Ä ä Ã ã Å å Æ æ
... which makes more sense when sorting alphabetic strings in a multilingual product. Someone has probably written this down in a standard, possibly even an ISO standard, but I'm not specifically aware of it.

Use CHAR() and CODE() together with ROW() and Paste Special, Values to create a list of characters that you can sort by for an example list you can refer to. You can find one I made a while back here.

OneDrive and Security Groups - Microsoft Community

bazz_365 say:

OneDrive and Security Groups

I'm trying to implement Security Groups in conjunction with OneDrive for Business share access and I'm having some dramas. I have the groups set up and members added. I share a link with one of the Security Groups, and members are able to access the shared content. Problem is, because the link is restricted to internal staff, it seems to be overriding the invited group by allowing all staff in the domain to access the share (if they get their hands on the link), even if they're not members of the Security Group.

I want to restrict the share to only the members of a Security Group. Is it possible to do this?

The only product registered against O365 is OneDrive, so my options are limited - I can only create Security Groups.

Any help to resolve this is greatly appreciated.

tina zhou msft say:

Hi Bazz_365,

Thanks for the query and glad to know you have worked it out.

I have added screenshot for Links under Sharing in OneDrive admin center. Once you set it, for files and folders sharing link, it will work as you expect if you share with specific people (e.g. user, mail-enabled security group. Office 365 group).

If you would like to share with security group, please click ... near Send Link > Manage Access > Grant Access to share.

Regards,

Tina

[Updated by Tina Zhou MSFT,  05:41 AM, September 26 , 2018 (UTC)]

bazz_365 say:

Never mind, I've found a way around it.

Set the default OneDrive share type to "Specific People" under the Sharing menu in the OneDrive admin area.

Back in OneDrive online, right-click the folder, click Share. Click the more button (three dots) in the Send Link header. Click Manage Access.

Click Grant Access, then add the security group. The access will apply only to that group.

Then right-click the folder, copy the link as normal, and send to recipients. I tested with a few accounts it seems to work as desired.
