When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate.

So, there is no need to know or get a product key, if you need to reinstall Windows 10, you can use your Windows 7 or Windows 8 product key or use the reset function in Windows 10.

If you are prompted to enter a product key, click 'I don't have a key' and 'Do this later'

Also, earlier today, I connected my laptop to my TV (also over HDMI) but I only had it extend my display, and that (of course) worked fine.

UPDATE: After dealing with this issue for about over a week, I have found out some interesting details. This issue happens on any monitor, TV or screen I hook onto my laptop after the April Update. However, this issue seems to fix itself after a certain amount of time. I haven't been able to test it extensively, but it seems that after about 15 minutes, I can use the "Second screen only" setting and have it work as intended. Physically switching monitors (for example, from my external monitor to my TV) seems to reset and reintroduce this issue again, making me have to wait ~15 minutes before I can use the "Second screen only" setting again... Strangely enough, rapidly unplugging the HDMI cable from the monitor and replugging it again doesn't seem to reintroduce the problem. It might only be reintroduced upon physically switching monitors.

I'll do more testing in the coming days to see exactly how long I'll have to wait for the problem to fix itself, if it fixes itself on every monitor I have available, and what exactly causes the issue to reintroduce itself anyway. For now, does anyone have any idea what's up?

UPDATE 2: After some more testing, it seems this issue is always introduced when I hook my laptop up to my TV. The TV is an LG Smart TV, though the exact brand I'm not entirely certain of at the moment.

Either way, like I said, the issue is introduced when the laptop is connected to the TV through HDMI. No matter how long I wait or what I do, the issue doesn't get solved. If I then unhook the laptop and hook it back onto my external monitor I usually use, the issue is still there, but it'll go away after a seemingly random amount of time. Sometimes it's almost instantly, and sometimes it can take an hour, it seems.

This feels like the issue is with the TV. (Or rather, the issue is probably still with Windows or the drivers; it's just that the TV triggers this issue.) The TV introduces the issue, which makes it persist for a while, even after unplugging. The external monitor never introduced this issue on its own. It was always the TV.

I should say that I definitely didn't have this issue before. I've always used that TV in the past and never had this issue. It's only since the April Update that I started having this issue.

UPDATE 3: I finally found a way to reproduce this, at least on my laptop. It isn't random at all. It just appeared so! (I mean, of course it wasn't random.)

It seems that if I am operating on battery power, the "Second screen only" mode doesn't work properly. But if I plug in my power adapter, then it will work just fine! It apparently seems completely independent of which screen I use. I don't know exactly why it's like this, and it absolutely isn't an issue I had before the April Update, though I can't say for sure if it is the actual April Update that introduced this issue. At least I know how to prevent this issue now, though it is still strange and should be looked into!

UPDATE 4: I did some more testing and found out that this only happens when on battery saver. For some reason, battery saver breaks projecting to the second screen only.
