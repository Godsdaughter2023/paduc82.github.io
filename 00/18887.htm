jay freedman say:

I hope the end result of this is to print the document on paper, rather than to use it on-screen.

I think the only way to accomplish this without going mad is to create two separate documents, with the left-hand pages in one document and the right-hand pages in the other. That way, the pages will be consecutive in each document, and the page numbers in the footers and in the table of contents can be generated in the usual manner. For editing purposes, you can display the documents side-by-side with synchronous scrolling.

To generate the printed document, first print the right-hand pages single-sided. Turn the pile of sheets over and put them back into the printer feed tray (without the last page), and print the left-hand pages in reverse order on the backs of the sheets.

If you have an absolute requirement to have the whole thing in one document that's viewable on-screen, the only method that generates the correct page numbers in the TOC is to put a section break at the end of each left-hand page, and restart the page number in the next page/section at the "correct" value, i.e., one lower than Word would assign. This would be extremely tedious to set up in a large document, and a nightmare to maintain if changes are needed.

A slightly better alternative is to use SEQ fields in the footers instead of PAGE fields, with the \c switch in the field on the right-hand pages to repeat the value in the left-hand footer. Be sure to update the fields with the F9 key. When you generate the TOC, the page numbers will all be "wrong" (i.e., the real page numbers); to fix that, select the TOC and press Ctrl+Shift+F9 to convert the TOC from its field to plain text, and then type in the "correct" (fake) page numbers.

There could be a small problem with your Window installation. The Check disk command cleans up small sections of the disk that may have been “orphaned” / corrupted when Windows is not shut down normally. The SFC command will check and repair Windows system files, then the DISM commands will repair any other minor problems with Windows.
In start menu type “PowerShell”
Right Click on the result and select “Run as Administrator”

run next command to schedule a check and fix the disk the next time the computer is restarted. At the end of the run it will display a status report.
CHKDSK C: /F

You will have to restart the computer for that check to work.
********DISM - DISM - Deployment Image Servicing and Management ************
After ChkDsk is done go back to the admin prompt and run following commands, one at a time to Repair The Installed Windows Image
this command checks whether the image has been flagged as corrupted by a failed process and whether the corruption can be repaired:
DISM /ONLINE /CLEANUP-IMAGE /CHECKHEALTH

this command scans the installed windows for component store corruption:
DISM /ONLINE /CLEANUP-IMAGE /SCANHEALTH

this command performs Window repair operations automatically:
DISM /ONLINE /CLEANUP-IMAGE /RESTOREHEALTH

DISM creates a log file (%windir%/Logs/CBS/CBS.log) that captures any issues that the tool found or fixed. %windir% is the folder in which Windows is installed. For example, the %windir% folder is C:\Windows.
https://docs.microsoft.com/en-us/windows-hardwa...

https://support.microsoft.com/en-us/help/929833... - Process shown with pictures

Right Click on the result and select “Run as Administrator”

Type (or paste) the following command and press <enter> to Scan the integrity of all protected system files and replace incorrect versions with correct Microsoft versions.
SFC /SCANNOW
