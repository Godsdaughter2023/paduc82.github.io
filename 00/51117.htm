Your problem is unique. I've never come across anything similar in eight years of working with this program. Admittedly, there are probably not many people who have made use of the forwarding capability of message rules ...  The only very slightly analogous situations I have come across have always been traceable to interference from third-party software. Two which I remember are SpamFIGHTER and BullGuard, both perfectly fine programs that clearly hadn't been tested adequately against Windows Live Mail when a new version was released. The program developers were always quick to put things right by making an update available.

Before declaring myself 100% baffled, I wonder whether you'd be willing to disable F-Prot for just long enough to go through a receive/forward-by-message-rule cycle, simply to eliminate that as the cause. Although F-Prot doesn't scan email, I think that Windows Live Mail creates a temporary file for the message to be forwarded. If this is the case, then that file has to be written to disk - which means that F-Prot will intercept it and scan it before releasing it back to the program's messaging function. This is a potential scenario for the corruption that's apparently happening.

It would in fact be best if you could uninstall F-Prot completely (they probably have a utility on their website to remove the most deeply-embedded bits), in which case I'd recommend installing Microsoft Security Essentials meanwhile. This is a free, lightweight but very efficient anti-virus program that should protect you just as well as F-Prot. We know (from my own experiments) that the corruption you're seeing doesn't happen where MSE is the anti-malware application in use. What say you?
