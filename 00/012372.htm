Win 8 Backup and Restore: Refresh, Recimg and Reset

Win 8 has refined your backup and restore options. Before Win 8 you could create an Image Copy backup of the full drive (or selected directories), but it was hard to separate backups of the OS, Applications and personal Data unless you created a separate data partition.  I still think it is a good option.

Out of the box, MS has been advertising the “Refresh” and “Reset” options. Personally, I think they missed a valuable selling point by not mentioning RecImg.EXE in conjunction with Refresh. It removes my main concern/objection to Refresh.

Reset is equivalent to using the “Recovery Partition” on previous versions of Windows. The factory would create a “Recovery Partition” that you could use to “reset the computer to factory fresh”. It would typically include all of the factory “crapware” and it would wipe out all of your data files and installed applications.  A less than desirable outcome but an appropriate reaction to a bad malware infection.  The “recovery partition” type of restore typically had to be accessed at boot time while Reset can be accessed from within Windows 8.

Refresh is roughly equivalent to an image copy backup.  A Reset will restore a backup of the OS and Metro applications. It will wipe out “legacy” / x86 desktop type applications BUT it will not affect your personal data files, which is much better than a Reset.  But there is still that issue of wiping out your installed desktop applications. I do not have documented confirmation, but I’m guessing that Metro apps are treated like personal data and those directories/folders are simply not reset back to the old condition.

RecImg.EXE is a backup tool that ships with Windows 8. It creates the backup “image” that is used by the Refresh tool. This image that was created at the factory is why using Refresh will wipe out installed desktop programs. They haven’t been backed up. However, you can create new images that will include recently installed desktop applications.

RecImg Manager is a third party tool the provides a GUI to access the RecImg.EXE command line tool and backup files.

Totals Queries - Access 2010 - Microsoft Community

dudley james say:

Totals Queries - Access 2010

I have a group of records from a select query. The query results contain, multiple records for the same ID. Can I use some form of a Totals Query to select just one record of the records that have multiple ID's? In other words I need in the case of multiple records, just to select one of the multiple records while retaining of the single records. I will then use the results of this query in another query that summarizes results. Thank you.
ken sheridan say: Yes, you can GROUP BY the ID column and use either the FIRST or LAST aggregation operator on all of the other columns.  These operators don't really do what their names suggest, but they do guarantee that the values returned will all be from the same row.

There are less arbitrary ways of returning just one row per ID value, e.g. those with the highest or lowest value in some other column, but if which of the rows per ID is returned is immaterial the above method is a simple solution.

qa guy commelec say:

A Totals query will rollup records that are exactly the same but if any field has different data then both will be display.  You must somehow determine which of the multiple records you want in the results. 

Do you have a DateTime or Autonumber field that can be used to pick the latest or oldest?

If so, then use the ID and that field in a totals query.  Then join the totals query with the table on ID and DateTime so as to only list the one record.
