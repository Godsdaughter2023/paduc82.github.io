rick rothstein, mvp - excel say:

As Jim said, you can't run a macro while in edit mode, but there may be an alternative way to achieve your end result without using a button... insert meta-characters into your string and let VB's event code translate them into the desired formatting. Here is what I am thinking. Pick two characters to act as delimiters, one for subscript and the other for superscript) and encase the characters you want to subscript and superscript with the appropriate delimiter (you would use them like you use quote marks, one on each side of the text to be delimited). In the code below, I chose the ` symbol (backward apostrophe located on the same key as the tilde [~] character) for the superscript meta-character and the | symbol (pipe symbol located on the same key as the back-slash [\] character) for the subscript meta-character. Let's consider an example to see how this would work. Our test phrase will be...

This is my test case.

and let's say we wanted to superscript the word "is" and subscipte the word "test" in that phrase. So, to use them, you would enter this text constant in the cell...

This `is` my |test| case.

and it should get appropriately. Okay, so below is the code that will automatically process your text constant entries. To implement it, go into the VB editor (press Alt+F11 to do this) and find the ThisWorkbook entry in the list of Microsoft Excel Object in the Project window (usually located on the top, left side of the screen) and double click it. This will open up a code window... copy/paste the code below into that code window. That's it... any text entry with the above encoding will be processed automatically. Note that if you need to edit the text in the cell, you will have to reapply the meta-characters as the code will remove them from the cell text after processing them. Also note that you cannot embed either meta-character inside a pair of the other meta-character. Finally, if you want to change the meta-characters being used, you would do that in the two Const statements in my code (which is which should be obvious by their names), although you should pick characters that will never appear in your text constant strings (the code will not be able to tell an actual character from its being used as a meta-character). Okay, here is the code...

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
  Dim X As Long, Position As Long, TextArray() As String
  Const SubScript As String = "|"
  Const SuperScript As String = "`"
  If Target.Count = 1 Then
    If (Not Target.HasFormula) And _
       (Target.Value Like "*" & SubScript & "*[!" & SuperScript & "]*" & SubScript & "*") And _
       (Target.Value Like "*" & SuperScript & "*[!" & SubScript & "]*" & SuperScript & "*") Then
      TextArray = Split(Target.Value, SubScript)
      Target.Characters.Font.SubScript = False
      Target.Characters.Font.SuperScript = False
      For X = 1 To UBound(TextArray) Step 2
        Position = InStr(Target.Value, SubScript & TextArray(X) & SubScript)
        Target.Characters(1 + Position, Len(TextArray(X))).Font.SubScript = True
        Target.Characters(Position, 1).Delete
        Target.Characters(Position + Len(TextArray(X)), 1).Delete
      Next
      TextArray = Split(Target.Value, SuperScript)
      For X = 1 To UBound(TextArray) Step 2
        Position = InStr(Target.Value, SuperScript & TextArray(X) & SuperScript)
        Target.Characters(1 + Position, Len(TextArray(X))).Font.SuperScript = True
        Target.Characters(Position, 1).Delete
        Target.Characters(Position + Len(TextArray(X)), 1).Delete
      Next
    End If
  End If
End Sub

faysal farooqui say:

Use the below ExcelVBA code to do this:

Sub HideRow1()
    If Range("B6").Value = "Yes" Then
       Rows("53:64").Select
    Selection.EntireRow.Hidden = True
    Else: Rows("53:64").Select
    Selection.EntireRow.Hidden = False
    End If
End Sub

' In this ExcelVBA code "B6" is the current workbooks, current worksheets cell. Before run this code, active the worksheet where B6 cell value meets or not meets the condition.

Auto-populate fields in Word - Microsoft Community

brucehoffort say:

Auto-populate fields in Word

Does anyone know how to auto-populate fields in Word?  For example, I want to use someone's name and job title several times in a word document and don't want to have to type it in each time, for every new name and document.  Thank you.
rohn007 say:

You want to use the cross reference feature:

  • Select the text you want to copy
  • Insert tab > Links group > click on Bookmark
  • enter a descriptive name for the bookmark, ie Customer_Name, Job_title
  • Click on the Add button

To use the bookmarked text:

  • Insert tab > Links group > click on Cross_reference
  • Select "Bookmark" from Reference Type drop down
  • make sure Insert Reference to says "Bookmark Text"
  • select the required bookmark name from the list
  • click on Insert button
  • click on close button

In File menu > Options command >Advanced option > Document Content section:

(Optional) Make sure that "Show Bookmarks" is turned on

(Optional) make sure that Field shading is set to "Always".

With those 2 options you can more easily identify where bookmarks are cross references exist ie;

"Show bookmarks" puts the braces around the words (marked with yellow highlight).

"Field shading always" makes the cross referenced text display in the grey box (marked with green arrow) <geeky grin for green arrow>

suzanne s. barnhill say:

A better approach would be to define a character style that includes both the font color and the strikethrough property (see http://wordfaqs.mvps.org/StyleRef.htm for instructions to create a character style). You can assign a keyboard shortcut to this style, add it to the Quick Style gallery, or select it from the Styles pane or floating Apply Styles dialog. What you unfortunately can't do in Ribbon versions of Word (though you can in Word 2003 and earlier) is add a style to the toolbar. You could write a macro to apply the style and put that on the QAT, but that seems like overkill compared to a keyboard shortcut.

Also, depending on why you're doing this, you should perhaps be using Track Changes instead.
