sandrameltzer say:

Contacts.

karl timmermans (outlook mvp 2012-2018) say:
>>
There is a folder called Contacts1(this computer only) in Office 365 and a it is empty. There is a folder called Contacts that has all my contacts
<<

#1 - The "This Compute Only" folder indicates that this belongs to an IMAP account.

#2 - This sounds like you did an in place upgrade from an older version of Outlook ('2010 or earlier) and your contacts are contained in a PST file which would have been the case for <either> a PST or Imap account. In this scenario, the contact folder you're referring to would be in a separate folder tree,

To confirm this - go to File --> Account Settings --> Data File tab

*** the one variation would be is if you have your contacts contained in a folder that's part of an Exchange account that you added to your profile after configuring the IMAP account

>>
How do I get my contacts to the proper place so that it is used in Outlook and Word
<<

You wouldn't want your contacts stored in a "This Computer Folder" since if the account is ever deleted, the OST file gets deleted and the data in the "This Computer Folders" (contacts/calendar etc) will be lost. Similarly, if Outlook cannot find the ost file for whatever reason, the OST file will get recreated and the email folders re-synced. However, the "This Computer Folders" will be empty.

What is likely the issue is that the folder where your contacts reside is not stored in the data file set as the "default data file" for the profile. All that needs to be done is to set the data file accordingly in the Account Settings data file tab

hansv mvp say:

To calculate the difference between a start time in A1 and an end time in B1, regardless of whether B1 is before or after midnight, you can use =MOD(B1-A1,1) just as in the desktop version.

Of course, the function name depends on your local language, and if you use comma as decimal separator, use ; instead of , to separate the arguments in the formula.

Excel Online does not currently support custom number formats, so there is no way to display hours over 24.

You can display the sum as decimal hours: use a formula such as =24*SUM(C1:C10) and format the cell with this formula as General or as Number with the desired number of decimal places.

bob jones aka: cybertaz say:

It's not a bug :-) It's a common misconception that Language is either a Program setting or a document setting... Neither is true except for the Default used when a new document is created. Language is an attribute of the text (actually, of the individual characters, similar to Bold, Italic, etc.), so if you do nothing to change it the entire document will continue to be in whatever the Default Language is set to.

However, when content is copied from another source it carries its own Language attribute with it. The Language of the pasted text very well may differ from that of the text already in the document. Further, continuing to type from the spot where the pasting was done will cause the new content to carry the same language attribute as the pasted text. For more details you're welcome to use the Search feature on the upper left of this page - There literally are dozens of conversations of conversations on subject. Also, this LINK is to one of the most definitive articles on the subject.

One way to avoid the issue is to use Edit> Paste Special- Unformatted Text or use Edit> Paste & Match Formatting rather than simply dumping in copied content. However, that also may remove wanted formatting carried by the pasted text. If that's the case you'll either have to reformat the pasted content or select it & apply the preferred language

bob jones aka: cybertaz say:

Like so many other aspects of personal computing, the same terminology can have differing implications... this is the case with "automatic language detection". The term has 2 different meanings:

  1. Word uses what is referred to as automatic language detection for the purpose of Spelling & Grammar checking. In this context the term means that Word detects the language attribute of the text being checked in order to use the appropriate Proofing Tools for that body of text. Both Word for Mac & Word for Windows work the same way in this regard.
  2. The same term often is used to describe how Word for Windows detects language contextually as you type & endeavors to apply the corresponding Language attribute to that text. This behavior, though, is found only in Word for Windows. Word for Mac never has incorporated this functionality.

When you select all & choose a language that language attribute is applied to the selected text. However, if you then paste content that caries a different language attribute, the language is retained by the pasted text. It does not change the language of any other text in the document. You can verify this by clicking in or selecting any of the preexisting text, then going to Tools> Language to see what is indicated. [It's more readily apparent if you have Language enabled on the Status Bar.]

IOW, if the default language for text typed into a new document is specified as English UK the text you type will carry that attribute. If you then paste text which carries the French attribute it will be pasted as French, but the existing English text will not be affected. However, if you then continue to enter text within or at the end of the Pasted French text the new content you type will continue to be marked as being French. As mentioned before, it's similar to formatting attributes such as Bold... If you paste Bold text then start typing the new content will be Bold as well.
