ossiemac say:

Hello Keith,

 

Not a perfect answer but what you can do is create a TextBox and a DTPickercontrol.

 

Reduce the width of the DTPicker so you can only see the dropdown arrow without the field.

 

Position the visible dropdown at the right end of the TextBox. (You can sum the TextBox Left position and TextBox width and set the DTPicker left to the summed value.)

 

Each time the DTPicker is changed the TextBox is updated with the value. When the DTPicker Dropdown is clicked then the DTPicker Enter event fires and sets the DTPicker date to today.

 

Apply the code below. The line "Me.TextBox1.SetFocus" moves off the DTPicker so that if the dropdown is clicked again before the user moves to another control then the Enter event will fire and reset the default date to today.

 

Drawback is that if you want to select today's date then there is no change to the DTPicker value and hence the change event does not fire. To get around this I have included an Exit event so if today's date is selected then the user will need to move off the control to update the TextBox.
 

Private Sub DTPicker1_Enter()
    Me.DTPicker1.Value = Date   'Sets DTPicker default to today's date. (TextBox does not change.)
End Sub

 

Private Sub DTPicker1_Change()
    'Edit format in next line to your desired format to display the date.
    Me.TextBox1 = Format(DTPicker1.Value, "dd mmm yyyy")
    Me.TextBox1.SetFocus
End Sub

 

Private Sub DTPicker1_Exit(ByVal Cancel As MSForms.ReturnBoolean)
    'Edit format in next line to your desired format
    Me.TextBox1 = Format(DTPicker1.Value, "dd mmm yyyy")
End Sub

 

Note that if you want to use the date in the TextBox then it is in Text Format and it is necessary to use DateValue function to convert it to a date like the following example. (I always like to use a date format in the TextBox that includes the alpha month and 4 digit year like "dd mmm yyyy" because DateValue never gets confused with it like it can with all numeric like dd/mm/yy and mm/dd/yy.)

 

Private Sub CommandButton1_Click()
    Dim dteTxtBox1 As Date
    dteTxtBox1 = DateValue(Me.TextBox1) + 7    'Adding 7 days confirms datevalue working properly.
    MsgBox dteTxtBox1
End Sub

rich michaels - mvp (greatcirclelearning.com) say:
Hi nutyga,

I'll take you up on your offer... "Any help will be appreciated" :-)

Here's my "any help". You took a bad turn when you headed toward the Publisher layout. It's probably fine for creating Kiwanis Club Newsletters at a local club level but it's definitely not right for a "paid by client" gig and the "Publisher layout" of Word for the Mac should never be considered on par with Indesign or Quark.

However, Word straight-up with no gimmicks (aka Publisher layout) can accomplish what you've been asked to do... you just have to clear your mind and not get caught-up in jargon. Or probably better said, forget what you think you know due to Indesign and Quark jargon and learn what this tool's jargon means.

I don't know the Publisher view of Word for the Mac... I have no use for it. It seems to place everything in floating text boxes which will become a nightmare when your client opens the document in Page Layout view and tries to start adding text. The phone calls to you are going to be incredible. :-)

Anyway to my "any help."

Organize your project around a "collection" of complimentary "parts." By collection I mean a look and feel which include document layout (margins & orientation), style sets, colors, logos, etc. And the parts of the collection are individual documents that include the cover page, end page, TOC page, content pages, etc.

For each collection's "look'n feel" stuff create a unique Word "dot" template. The template defines, document layout, fonts, styles, etc.

Then instruct your user(s) on how to open the collection template and to use Word's built-in function of Insert > File... for inserting the collection's cover page, inside cover page, TOC page, content page(s) and end page. Alternately, you could have each collection's template insert a shell that includes all of the aforementioned parts.

Doing it this way your "master" is the collection's template and it defines the "look'n feel" of the parts. Parts are parts... they are not masters.

Hope this provides a level of "any help."
