There isn’t an official article about giving uses some folders’ permission in a shared mailbox. If you want to archive this feature, please do the following steps:

Add-MailboxFolderPermission -Identity sharedmailbox@domain.com -User user@domain.com -AccessRights FolderVisible

Note: Please refer to this article and change the permission from Reviewer to any other role you want to assign to the users.

Add-MailboxFolderPermission -Identity sharedmailbox@domain.com:\folder1 -User user@domain.com -AccessRights Reviewer

Note: If you have already given a user a permission on a specific folder and want to change the user’s permission, please refer to the command below:

Set-MailboxFolderPermission -Identity sharedmailbox@domain.com:\folder1 -User user@domain.com -AccessRights Editor

4. Navigate to File -> Info -> Account Settings -> Account Settings… -> E-mail -> Choose an account and Click Change… -> Click Advanced in More settings -> Type the shared mailbox’s address in Add…

5. Finally, you can see the shared mailbox with specific folders in Outlook 2013.

Whether a user displays hidden objects or not is a personal. application-wide preference. It is not stored with the database, and it is considered impolite for third-party applications to change this preference. But if you really want to, you can execute the line


Application.SetOption "Show Hidden Objects", 0


when the database is opened, for example in the On Load event of the startup form.

To display hidden objects, you'd use


Application.SetOption "Show Hidden Objects", 1
