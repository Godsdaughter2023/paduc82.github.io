john corbridge say:

It would appear that whilst the device driver from Makerbot supports the Replicator 2X, the printer driver only supports the Replicator 2. This means that the complete driver package is installed for both the 2 and 2X, but the printer driver is only compatible with the 2 model. 

The Replicator 2X has a different device hardware ID, so identifies itself differently to the plug and play system. This still triggers the installation of the Makerbot plug and play drivers, but leads to issues when actually trying to print.

This explains why the printer shows up in Devices and Printers as a "Replicator 2", rather than a 2X.

Any application that interacts with the printer through the print queue, such as the 3D Builder application, will fail to print, as the expected device ID does not match the actual one.


Any application that interacts with the printer at the device level, such as MakerWare or ReplicatorG, will work fine, as it is just using the device driver component.

Hi Kimjai26,

Beeping sounds during post (initial start-up) are typically trouble codes generated by the motherboard firmware's self check process.

(A beep/long pause between beeps is typically a single digit code. A beep/short pause beep (or more than one with a short pause in between) then long pause would be a double digit code)

Find/download your system/motherboard manual for that documentation/trouble code list, if you need help with this let us know.
