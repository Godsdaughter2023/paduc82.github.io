jlknicinc say:

trojan.FakeAV

JLKNICINC
rob koch say: It depends what you consider an 'infection', since there are two stages to any of these fake anti-malware products and the first stage is rarely detected by any antivirus product, since they simply change too fast.  The second stage is the actual installation of the true malware and is more often detected by anti-malware, though this still assumes an anti-malware program is installed and up to date.

The first stage is actually typically a simple web page, accessed via a number of means such as false search results (SEO), injected pop-ups using scriplets on otherwise valid web pages or links in fake email messages for Microsoft or other updates.

http://www.microsoft.com/security/antivirus/rogue.aspx

Unfortunately, since these change in both location and operation so quickly, the only thing between the PC and their execution is often the person in front of the PC.  Microsoft and some other browser developers have included things like the the SmartScreen Filter which detects and blocks or displays an alert for phishing pages and even some malware.  Since these programs are web based and receive their information from the 'cloud' they are more likely to be able to detect these quickly changing pages.

http://www.microsoft.com/security/filters/smartscreen.aspx

The problem is that all of these systems are reactive, so someone must detect the pages first for them to be registered as 'bad'.  It's also required that Internet Explorer 8 be installed to have SmartScreen Filter and at least IE 7 for the Phising filter.  Since many users don't ever get around to installing these updates, the only remaining protection is the anti-malware program if they even have one.  So you can see that in many cases the typical PC doesn't have sufficent protection in place, especially older version of Windows such as XP, since these updates often aren't set to automatic.  And in most cases, some number of even protected PCs will be infected before a 'new' fake antivirus is identified and added to either level of protection.

Rob

Is there a way to fix this terribly annoying Preset permanently???  Is this an HP driver issue or a Microsoft issue?

john mcghie say:

Mike:

Michael is correct: it can be difficult to make this stick.

Basically, Word asks MacOS for the printer settings (paper size, impression margins, DPI and capabilities). MacOS queries the printer driver, which queries the printer itself.

The information is stored so things don't have to ask every time.  To make this stick, you may need to DELETE the printer from MacOS, then restart the computer. That should cause MacOS to autodetect the printer and put it in again.

Before printing anything, go into System Preferences>Printers, right-click the printer and see if it will allow you to set the defaults there.

If it won't, start Word and go to File>Print...  Open the Default settings and make the change there.

Hope this helps
