zigzag3143 (ms -mvp) say:

"It's not a true crash, in the sense that the bluescreen was initiated only because the combination of video driver and video hardware was being unresponsive, and not because of any synchronous processing exception".

Since Vista, the "Timeout Detection and Recovery" (TDR) components of the OS video subsystem have been capable of doing some truly impressive things to try to recover from issues which would have caused earlier OSs like XP to crash.

As a last resort, the TDR subsystem sends the video driver a "please restart yourself now!" command and waits a few seconds.

If there's no response, the OS concludes that the video driver/hardware combo has truly collapsed in a heap, and it fires off that stop 0x116 BSOD.

If playing with video driver versions hasn't helped, make sure the box is not overheating.

Try removing a side panel and aiming a big mains fan straight at the motherboard and GPU.

Run it like that for a few hours or days - long enough to ascertain whether cooler temperatures make a difference.

If so, it might be as simple as dust buildup and subsequently inadequate cooling.

I would download cpu-z and gpu-z (both free) and keep an eye on the video temps

therealteggy say:

I had a scenario where I already had an .mig file stored on a external drive, but I wanted to use it for starting up my windows 10 machine. I was severely disappointed to find out that Windows Easy Transfer was not installed, and there was no option.

However, I did find a work around. I used a Windows 7 64 bit machine, and start easy transfer on that machine. I then selected “An external hard drive”, then select “This is my new computer”.

Then it will prompt you whether or not you have done your backup yet, respond no. Then you will be prompted “Do you need to install Windows Easy Transfer on your old computer”, select “I need to install it now”. You will be then prompted on where you want to install it, select “Shared network folder” and select a destination on this pc. (previous steps can be referenced (http://www.howtogeek.com/howto/3179/migrate-xp-to-windows-7-with-easy-transfer-and-a-usb-drive/)up until  the copying of Easy Transfer files)

Go to the folder where the Windows Easy Transfer was created, and zip the folder up.

Now move the zipped folder over to the new Windows 10 machine. Unzip the folder and select the amd64 folder for 64 bit Windows 10, or x86 folder for 32 bit Windows 10. Then select MigSetup.exe.

Select “Next”, select from “External Drive”, “This is my New Computer” , Answer “Yes” and select the .mig file you want to import and select open. The process should continue and finish successfully.

Hopefully this has allowed you to utilize your existing windows Easy Transfer files.
