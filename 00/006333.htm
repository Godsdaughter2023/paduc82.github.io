Assuming that Manufacturer is a text field, you have to enclose the value in quotes:

    DoCmd.RunSQL "UPDATE tblSupplies SET Manufacturer = " & _

ken sheridan say: Without being able to debug your form I can't offer any explanation for the behaviour which you are encountering.

A form/subform in which the parent form is based on the referenced table in a one-to-many relationship, and the subform is based on the referencing table in the same relationship, which appears to be what you have, should be easily set up.  The only mechanism affecting the link between the parent form and subform is that of the LinkMasterFields and LinkChildFields properties of the subform control.  These should be the names of the primary key of the parent form's table, and the corresponding foreign key in the subform's table.  In most cases neither of these will represented by bound controls in either the parent form or subform, as the former will usually be an autonumber, whose value is inserted automatically, and the latter will have its value inserted automatically by virtue of the linking mechanism.

You might like to take a look at FormsDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates a number of ways of interfacing with related tables like this, using subforms or linked forms, in the latter case either in dialogue mode or synchronized mode.  In the demo, two subforms are used, each on separate pages of a tab control in the parent form to save space.  Unlike linked forms, subforms require no code whatsoever to effect the link, everything is achieved by the LinkMasterFields and LinkChildFields properties, so unless you have some unnecessary code or similar, it's hard to see why you should be experiencing the untoward behaviour.
