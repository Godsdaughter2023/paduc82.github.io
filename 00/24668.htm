braintrove.com say: Okay--so one problem is the condition. You don't need conditions (this results in the if/then model). Instead we use filters. A single rule to set the Role Name for every given selection in Role Description

To try this out, I suggest adding another temporary set of controls to this or a different view. Then once you understand what it is doing, move it into your working view. You should be able to delete all those conditional rules and replace them with a single one.

The following assumes you've set up a sample repeating table with the drop-down and text box fields.

1. Set the drop-down to populate from the roleData/RoleDescription in the XML.
2. Select the drop-down, and then in the Home tab, click Add Rule | This Field Changes | Set A Field's Value.
3. In the Rule Details dialog box, for the Field select T1WorkPlaceRoleName (the sibling element to the drop-down that is changing values).
4. Click the "fx" button for the Value.
5. In the Insert Formula dialog box, click Insert Field Or Group.
6. Select roleData/RoleName from the XML file, and then click Filter Data.
7. In the Filter Data dialog box, click Add, and then chose Role Description, Equals, and Use Formula.
8. Type "current()" (without the quotes--this means the current field your rule is running on... i.e. the drop-down with the changing value), and then press OK six times.

Now you can preview and test to see that for each row you insert, you can select a description in the drop-down and get the associated value placed into the text box via this single rule that uses filtering.

There's no auto answer for the device itself unless u plugged in related accessories. Try this ..
