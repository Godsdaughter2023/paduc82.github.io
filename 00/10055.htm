ken sheridan say: I think you've partly answered your own question, and the answer is not to use the form's OrderBy property at all but to change the ORDER BY clause of its RecordSource property  in code.  To add a sort order for a column would be a matter of inserting the column name, preceded by ORDER BY, and followed by ASC or DESC as appropriate if there is currently no ORDER BY clause, or preceded by a comma if there is already an ORDER BY clause.  To remove a column would be a matter of replacing the column name (by means of the Replace function), along with ORDER BY and ASC or DESC if there is only one column in the ORDER BY clause, or along with the surplus comma if there are two or more columns currently in the ORDER BY clause.  The code to do this would be a little tedious to write, but not difficult.

If you want to stick with the OrderBy property, then the methodology would be similar.  In each case it boils down to parsing a string expression.

Here's some rough and ready code off the top of my head which should give you the general idea:

    Dim strSQL As String
    Dim strSQL_Current As String
    Dim intStart As Integer
    Dim intEnd As Integer
    
    strSQL = "SELECT * FROM Addresses"
    Debug.Print strSQL & ";"
    
    ' add a sort by state ascending
    If InStr(strSQL, "ORDER BY") = 0 Then
        strSQL = strSQL & " ORDER BY " & "State ASC"
    Else
        strSQL = strSQL & ",State ASC"
    End If
    Debug.Print strSQL & ";"
    
    ' add a sort by county ascending
    If InStr(strSQL, "ORDER BY") = 0 Then
        strSQL = strSQL & " ORDER BY " & "County ASC"
    Else
        strSQL = strSQL & ",County ASC"
    End If
    Debug.Print strSQL & ";"
 
    ' add a sort by city ascending
    If InStr(strSQL, "ORDER BY") = 0 Then
        strSQL = strSQL & " ORDER BY " & "City ASC"
    Else
        strSQL = strSQL & ",City ASC"
    End If
    Debug.Print strSQL & ";"
    
    ' remove sort by County
    strSQL_Current = strSQL
    intStart = InStr(strSQL, "County")
    intEnd = InStr(intStart, strSQL, ",")
    strSQL = Replace(strSQL, Mid(strSQL, intStart, intEnd - intStart), "")
    strSQL = Replace(strSQL, ",,", ",")
    If strSQL <> strSQL_Current Then
        Debug.Print strSQL & ";"
    End If
    
     ' remove sort by City
    strSQL_Current = strSQL
    intStart = InStr(strSQL, "City")
    intEnd = InStr(intStart, strSQL, ",")
    If intEnd <> 0 Then
        strSQL = Replace(strSQL, Mid(strSQL, intStart, intEnd - intStart), "")
        strSQL = Replace(strSQL, ",,", ",")
    Else
        strSQL = Left(strSQL, intStart - 1)
        If Right(strSQL, 1) = "," Then
            strSQL = Left(strSQL, Len(strSQL) - 1)
        End If
    End If
    
    If strSQL <> strSQL_Current Then
        Debug.Print strSQL & ";"
    End If
    
     ' remove sort by State
    strSQL_Current = strSQL
    intStart = InStr(strSQL, "State")
    intEnd = InStr(intStart, strSQL, ",")
    If intEnd <> 0 Then
        strSQL = Replace(strSQL, Mid(strSQL, intStart, intEnd - intStart), "")
        strSQL = Replace(strSQL, ",,", ",")
    Else
        strSQL = Left(strSQL, intStart - 1)
        If Right(strSQL, 1) = "," Then
            strSQL = Left(strSQL, Len(strSQL) - 1)
        End If
    End If
       
    ' remove ORDER BY clause if empty
    If Right(strSQL, 9) = "ORDER BY " Then
        strSQL = Trim(Left(strSQL, Len(strSQL) - 9))
    End If
    
    Debug.Print strSQL & ";"

brian tillman [outlook mvp 2007-2019] say:

Of course they won't show up.  You downloaded them and deleted them from the server.  That's the way POP works, unless you tell it otherwise.  However, with AOL, all is not lost.  AOL's main method of access is usually IMAP.  IMAP is a two-way protocol so if you create another IMAP account to reference the same AOL mailbox, you'll then be able to simply drag all the downloaded messages from your current Inbox into the IMAP account's Inbox and they'll all be copied back to the server just like they were originally.  You can then delete the POP account.  You'll still have the default folders and now a set if IMAP folders.  This can't be changed, but just designate the IMAP account as the default account.
