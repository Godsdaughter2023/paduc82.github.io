How to assign E1/K1 and EOA license together using Powershell?

How to assign E1/K1 and EOA license together using Powershell?

# Script to assign everything location and all licenses

$powerUser = "*** Email address is removed for privacy ***" #Your global admin account

$powerPass = "xxxxxx" #Change it to your password

$password = ConvertTo-SecureString $powerPass -AsPlainText -Force

$adminCredential = New-Object -TypeName System.Management.Automation.PSCredential -argumentlist $powerUser,$password

Connect-MSOLService -Credential $adminCredential

$AccountSkuId = "domain:ENTERPRISEPACK"  #Change it to your AccountSkuId

$UsageLocation = "US"

$LicenseOptions = New-MsolLicenseOptions -AccountSkuId $AccountSkuId

$Users = Import-Csv c:\scripts\Users.csv #Change it to the path of your csv file

$Users | ForEach-Object {

Set-MsolUser -UserPrincipalName $_.UserPrincipalName -UsageLocation $UsageLocation

Set-MsolUserLicense -UserPrincipalName $_.UserPrincipalName -RemoveLicenses $AccountSkuId

Set-MsolUserLicense -UserPrincipalName $_.UserPrincipalName -AddLicenses $AccountSkuId -LicenseOptions $LicenseOptions

}

Hi Parvinder,

The error message shows that the Exchange Online service plan included in E1/K1 has conflict with EOA license. Please first confirm if you want to exclude Exchange Online (Plan 1) and include EOA. If so, you can modify the script as below.

#Change all the codes after line "Connect-MsolService"
$AccountSkuId1 = "domain:ENTERPRISEPACK"  #Change it to your domain and change ENTERPRISEPACK to required SKU IDs.

$AccountSkuId2 = “domain:EXCHANGEARCHIVE” #Change it to you domain.

$LicenseOptions = New-MsolLicenseOptions -AccountSkuId $AccountSkuId -DisabledPlans EXCHANGE_S_STANDARD
#this line excludes the conflicting Exchange license in E1/K1 license

$Users | ForEach-Object {
Set-MsolUser -UserPrincipalName $_.UserPrincipalName -UsageLocation $UsageLocation

Set-MsolUserLicense -UserPrincipalName $_.UserPrincipalName -AddLicenses $AccountSkuId1 -LicenseOptions $LicenseOptions
#use this line to add E1/K1 license without Exchange Online (Plan 1)

Set-MsolUserLicense -UserPrincipalName $_.UserPrincipalName -AddLicenses $AccountSkuId2
#use this line to add EOA license

}

Note: If you decide to use Exchange Online (Plan 1) license and exclude EOA license, you can replace two “Set-MsolUserLicense” lines of codes with the codes below:

Set-MsolUserLicense -UserPrincipalName $_.UserPrincipalName -AddLicenses $AccountSkuId1

Windows prompts with this message when unknown programs (unsigned) try to write data to protected system folders or registry settings, and UAC (User Account Control) is seeking your permission.  This type of activity could be dangerous sometime, so it prompts every time.

If the program has an unknown publisher, it doesn't have a valid digital signature from its publisher. When we execute a normal program, UAC first checks if that program has keywords like

UAC also checks the programs manifest file for the "requireAdministrator" tag added in its manifest config, if that is found, UAC will then go to the next step. When UAC has confirmed the first step, UAC will check if that program is blocked by policy or if the programs publisher is blocked, when UAC successfully confirms that it goes to the next step.

How to activate Office 2010? KMS vs. MAK

http://4sysops.com/archives/how-to-activate-office-2010-kms-vs-mak/

By Michael Pietroforte | 2 Comments | Permalink

Both options to activate Office 2010, KMS and MAK, have their up- and downsides. This article discusses a few things you have to consider when you decide for the activation method in your organization.

Now that you have an overview of the Office 2010 KMS activation and MAK activation, I can address the question of whether KMS or MAK is the better option to activate Office 2010. Most of the arguments also apply for Windows Vista and Win7 activation.

Microsoft recommendations

You need at least 5 Office 2010 installations if you want to activate Office 2010 with KMS; however, Microsoft recommends using MAK activation if your network has fewer than 25 computers. If you have more than 50 Office 2010 machines, Microsoft recommends going for KMS.

 

MS 2010 Volume License Activation Options - MAK (1 shot) vs KMS (reusable)

http://4sysops.com/archives/series/office-2010-activation/

 

Office 2010 MAK activation essentials

http://4sysops.com/archives/office-2010-mak-activation-essentials/

MAK activate similar to a retail key. It is a one-time activation over the internet against Microsoft’s server. Unlike a normal retail key, each key can be used for a minimum of 500 activations to thousands of activations. Like a retail key you will have to reactivate if there’s a drastic hardware change or if you change your installation to a new computer.

 

Office 2010 KMS activation essentials

http://4sysops.com/archives/office-2010-kms-activation-essentials/

KMS involves running an internal company license server. Client computers will renew their license against the server. Each renewal will reset the expiry to 180 days, and it will try renewing every 7 days. Office will try contacting the server more frequently if expired or close to expiring. KMS is the preferred method of volume activation for businesses and incidentally KMS based solutions are the main source of activation “solutions”.

Office Volume Activation

http://technet.microsoft.com/en-us/office/ee691939.aspx

Key Management Service (KMS). KMS uses a host computer to establish an activation service on your local company network. With a KMS host you can activate thousands of computers at the same time when they regularly connect to the company network. If you use a KMS host to activate Windows, you can use the same host to activate Office 2010.

Multiple Activation Key (MAK). With MAK, each computer activates Office 2010 with the Microsoft hosted activation servers over the Internet or by phone. MAK is recommended when you have fewer than 50 computers to activate, and for computers that are not regularly connected to the company network.
