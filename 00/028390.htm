In MS Word 2010, I am editing a saved html file that has web hyperlinks to other parts of the same document.  I can manually edit the hyperlinks and "Place in this Document" for each link.  Is there way to do this for all the links in the document without having to edit each one individually?

doug robbins - mvp office apps & services (word) say:

Maybe this old code will do it for you:

 

' Macro created 26/10/01 by Doug Robbins to update links in a document
'
Dim alink As Field, linktype As Range, linkfile As Range
Dim linklocation As Range, i As Integer, j As Integer, linkcode As Range
Dim Message, Title, Default, Newfile
Dim counter As Integer

counter = 0
For Each alink In ActiveDocument.Fields
    If alink.Type = wdFieldLink Then 

        Set linkcode = alink.Code
        i = InStr(linkcode, Chr(34))
        Set linktype = alink.Code
        linktype.End = linktype.Start + i
        j = InStr(Mid(linkcode, i + 1), Chr(34))
        Set linklocation = alink.Code
        linklocation.Start = linklocation.Start + i + j - 1
        If counter = 0 Then
            Set linkfile = alink.Code
            linkfile.End = linkfile.Start + i + j - 1
            linkfile.Start = linkfile.Start + i
            Message = "Enter the modified path and filename following this Format " & linkfile
            Title = "Update Link"
            Default = linkfile
            Newfile = InputBox(Message, Title, Default)
        End If
        linkcode.Text = linktype & Newfile & linklocation
        counter = counter + 1
    End If
Next alink

graham mayor say:

Assuming that speakers' names are always one 'word' long, this is easy enough to achieve with a macro e.g. as shown below. It gets rather more complicated if speakers names are of variable numbers of words, unless the name always starts a new paragraph.

Option Explicit
Sub Macro1()
Dim orng As Range
Dim i As Long
    'Process colons
    Set orng = ActiveDocument.Range
    With orng.Find
        Do While .Execute(FindText:=Chr(58) & " [a-zA-Z]", MatchWildcards:=True)
            orng.MoveStartWhile "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ", wdBackward
            orng.Case = wdTitleWord
            For i = 1 To 2
                orng.Words(i).Font.Bold = True
            Next i
            orng.Collapse 0
        Loop
    End With
    'Process periods

    With orng.Find
        Do While .Execute(FindText:=Chr(46) & " [a-z]", MatchWildcards:=True)
            orng.Case = wdUpperCase
            orng.Collapse 0
        Loop
    End With
    'Process bracketed comments
    Set orng = ActiveDocument.Range
    With orng.Find
        Do While .Execute(FindText:=Chr(91), MatchWildcards:=False)
            orng.MoveEndUntil Chr(93)
            orng.End = orng.End + 1
            orng.Font.Bold = True
            orng.Collapse 0
        Loop
    End With
lbl_Exit:
    Exit Sub
End Sub

joeu2004 say: IeoCheng wrote:

I have a very simple vlookup which came across to be the numbers that are causing the problem.

My formula :
A1 cell type 100
B1 cell type =A1*10
[....]

B2 cell type =vlookup(A2,B1:C1,2,false)
[....]
Replace the number in A1 cell with 183.96
Replace the number in A2 cell with 1839.6
B2 cell returns #N/A 

Change the formula in B1 to:

=ROUND(A1*10,1)

Change 1 to whatever accuracy you expect (maybe 2).

The root cause of the problem is that most non-integers cannot be represented exactly in 64-bit binary floating-point, which is how Excel represents numbers and performs arithmetic.  For overwhelming details, see http://support.microsoft.com/kb/78113 .

In your example, the constant 1839.6 is actually 1839.59999999999,990905052982270717620849609375.  But 10*183.96 is 1839.60000000000,0136424205265939235687255859375 because 183.96 is actually 183.960000000000,007958078640513122081756591796875.

As you can see, the representation of 1839.6 is not exactly the same 10*183.96.

Now, IF(A2=B1,TRUE) returns TRUE only because of a dubious heuristic poorly described under the misleading title "Example When a Value Reaches Zero" at http://support.microsoft.com/kb/78113 .

In fact, IF(A2-B1=0,TRUE) returns FALSE for the reason explained above.

IeoCheng wrote:
I know if you copy and special paste value of B1 works.

Although it might work in this case (I'll take your word for it; I did not try it myself), it does not always work.

For example, suppose A1 is =12345.67+2^-39 and A2 is =12345.678+2^-39.  Copy A1:A2 and paste-special-value into B1:B2.  Then, MATCH(A1,B1,0) returns a #N/A error (no match), but MATCH(A2,B2,0) returns 1 (exact match).
