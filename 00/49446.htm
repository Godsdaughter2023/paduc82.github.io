How do I delete a 'system restore point' - Microsoft Community

sepbj say:

How do I delete a 'system restore point'

I created a 'system restore point'  which won't allow me to go further back.  How do I delete this 'point' (date) and return to 'normal' system restore?
lemp say:
It's difficult to be certain without actually seeing your computer and testing its behavior, but it seems to me that there are a few things combining to confuse you, including the coincidence that today happens to be the 18th July.

The best System Restore resource I know of is http://bertk.mvps.org.  Much of the following is based on information from that web site.

In general, it's not a good idea to try to restore to a long-ago Restore Point.  The potential for problems increases the longer the time between now and the creation of the Restore Point.  See http://bertk.mvps.org/html/whentouse.html Thus, it was probably not a good idea in the first place to try to restore to a month-old Restore Point (a much better way to "go back" to various points in time is to create a full system image using software designed for that purpose).

On the other hand, in normal operation, System Restore creates a system checkpoint approximately every 24 hours

System checkpoints: System Restore creates restore points on a regular basis even if you have not made any changes to the system. System Restore automatically creates these restore points every 24 hours of calendar time, or every 24 hours your computer is turned on. If your computer is turned off for more than 24 hours, System Restore creates a restore point the next time you start the computer. The computer must be idle for a few minutes before System Restore creates a scheduled restore point.  (http://bertk.mvps.org/html/tips.html#13A)


Thus, when you looked "after about a month," you should have seen Restore Points between the time you looked and the one from May 18.  If the May 18 restore pont was the only Restore Point showing, System Restore is not working properly.  There are two general possibilities:  System Restore is not creating automatic restore points or restore points are being deleted.  Both possibilities are discussed here http://bertk.mvps.org/html/srauto.html and here http://bertk.mvps.org/html/missingrps.html

I particularly suggest that you test System Restore as described above. http://bertk.mvps.org/html/tips.html#3

As far as not being able to go back "beyond" your initial month-old restore point, restore points are only kept for 90 days or until the allocated space fills up: 

When the allotted disk space is reached, the oldest restore point will be purged on a first in first out (FIFO) basis. Otherwise, restore points over 90 days are purged automatically by default.  (http://bertk.mvps.org/html/tips.html#7)


Finally, your "moved" restore point may just be a coincidence.  When you turned off System Restore as Wunders originally suggested, all of the existing restore points should have been deleted.  A new automatic system checkpont was probably made today.  If you want to confirm that this is a coincidence, wait until tomorrow (19th July), turn System Restore off and then back on.  Your July 18 restore point should be gone.

This behavior occurs when drivers do not handle power state transition requests properly. The error message most often occurs during one of the following actions:

 

•Shutting down

•Suspending or resuming from Standby mode

•Suspending or resuming from Hibernate mode

 

To prevent the error message from occurring, update or remove the offending driver. This behavior is not limited to device drivers; it can also occur with file system filter drivers (for example, drivers installed by an anti-virus, Remote Control, or backup program). To isolate the driver that is causing the problem, use the following steps:

 

1. Check for driver updates that may be available for your hardware.

2. Update software that uses filter drivers (for example, anti-virus, remote control, and backup software).
