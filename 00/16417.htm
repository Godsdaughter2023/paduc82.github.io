Hi.  I can't tell what this does, but here are a few observations.

You are assigning a value to a cell.

Hence, you can't assign a value  to a vba variable at the same time (ie cd1=""CAS"")

Another is that you are evaluating a string.

"SUMPRODUCT(SUBTOTAL(9,OFFSET(d2,n-MIN(ROW(d2)),,1)),-- ..."

This string includes text like "d2" and "n"

These are just strings.  Evaluate has no way to know that these are vba variables.

You have to replace the string "d2" with the actual data.

Evaluate doesn't work with vba variables anyway.

An arrow IS a line.  It just has line ends that aren't null.

MSGraph is a different application with different defaults etc.  Convert To Excel converts your MSGraph objects to Excel 2007 objects, which do look very different.  It might be better not to convert them if this is an issue.

>> Why can I not specify my paste preference, e.g. "Keep source formatting", as a default in Options?

A LOT of people want to know the answer to THAT one.

Basically, tables just aren't fully cooked. 

>> Why do I need to select two or more shapes before being able to select "Align to Shape"?

Where do you see Align to Shape?  Align Selected Objects, maybe?  The idea there is that PPT will align one or more other objects to another object. You can't align one object to another unless you have two of them selected.  What are you trying to do that you're having trouble with?

It's not all that sophisticated ... it only stores one set of formatting attributes.  It'd sure be nice if it could remember more than one.

It's a presentation-level setting rather than part of the print dialog, and all you can store is stuff that you can choose on th print dialog.  But I agree.  A) it'd be far better if you could choose High Quality on the print dialog rather than having to go diving for it.  And then it could logically be one of the saved print settings for a document.

> Why does a Final document still show placeholders, they can never be used?

Anyone can mark the document as not final again.  Marking it as final isn't a form of protection, other than warning users that you've asked them not to mess with it.  In any case, the placeholders won't appear when viewing the presentation in slide show or printing it.  Fill in the rest:  why's this a problem for you?  We can take that to MS.

jay freedman say:

If you turn on nonprinting characters, you'll see that the character inserted by the Enter key inside a plain text content control really is a line break and not a paragraph mark. Unfortunately, there is nothing you can do to change that.

The only alternative I see is to replace the plain text control with a rich text control, which does allow inserting a genuine paragraph break with the Enter key. If that's unacceptable for other reasons, I think you're stuck with the current behavior.
