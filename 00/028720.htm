ken sheridan say: You don't need an SQL statement at all.  You can return a reference to a clone of the form's recordset by means of the RecordsetClone property.

You'll find an example in DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In the section on 'retrieving data from the database' in this little demo file you'll find a bound form in which a contact can be selected in an unbound combo box.  The code in the control's AfterUpdate event procedure is as follows:

    Dim ctrl As Control
    
    Set ctrl = Me.ActiveControl
    
    If ctrl = 0 Then
        DoCmd.GoToRecord record:=acNewRec
    Else
        ' find matching row in form's RecordsetClone
        With Me.RecordsetClone
            .FindFirst "ContactID = " & ctrl
            If Not .NoMatch Then
                'move form to selected record
                ' by synchronizing bookmarks
                Me.Bookmark = .Bookmark
            End If
        End With
    End If

The control's RowSource property is:

SELECT 0 AS ContactID,0 AS SortColumn,NULL AS LastName,
NULL AS FirstName,"<New Contact>"
FROM Contacts
UNION
SELECT ContactID,1,LastName,FirstName, Lastname & ", " & FirstName
FROM Contacts
ORDER BY SortColumn, Lastname,FirstName;

The first part of the UNION operation adds a <New Contact> item at the top of the combo box's list, to navigate to an empty new record.  Note that the combo box's BoundColumn property references the hidden ContactID column.  The concatenated Lastname & ", " & FirstName column is the visible column by virtue of the ColumnWidths property being set to 0cm;0cm;0cm;0cm;8cm to hide the first four columns.

ken sheridan say:

............I get a Run Time 7951 error stating that "You entered an expression that has an invalid reference to the RecordsetClone property."

The error which you are experiencing strongly suggests that the form is unbound.  Is that the case?

You said in your original post 'I have a combo box on a form in which I want to find a record'.  You cannot find a record in an unbound form, as it has no recordset in which you can navigate to a row.  Your last reply to Scott suggests that what you might actually want to do is assign values to other controls in an unbound form on the basis of a selection in a combo box.  If a single value is to be assigned to each control, which your reply to Scott implies, then the other controls should be text boxes, not combo boxes.  The values can be assigned in a number of ways, e.g. by referencing the Column property of the first combo box.

Combo boxes would only be used if you want to restrict the rows returned by the combo boxes' RowSource properties to a subset of rows based on the selection in the first combo box.  These are known as 'correlated' or 'cascading' combo boxes.

Combo boxes can be correlated in two ways:

1.  The RowSource of the combo boxes, other than the one in which the first selection is made can reference the first combo box as a parameter.  The other combo boxes are requeried in the AfterUpdate event procedure of the first combo box.

2.  An SQL statement can be built in code, concatenating the value selected in the first combo box into the string expression, which is then assigned to the RowSource property of the other combo box.

I think you need to express what you are attempting to do in real life terms in the context of the database, rather than how you are attempting to do it.  We'd then be in a better position to advise you on the best methodology.

A static master is simply a new single file that is created by unchecking the "link to project" option in the lower right corner of the Insert Subproject window. karl timmermans (outlook mvp 2012-2018) say:
>>
I then noticed that the older emails were slowly repopulating and became up to date. How do I get my contacts back?
<<

This is not going to be the answer you want to hear but nonetheless.

From the description, your account is configured as an IMAP account and more then likely, your contact/calendar data was stored in folders with the added words "This Computer Only". It sounds like your OST file was recreated by Outlook which then re-syncs with the server for the email folders. Unfortunately, the data contained in "This Computer Only" folders would be lost unless you explicitly/manually backed up (exported) those folders to a PST file. If an old .ost file exists for this account by any chance, then the only way to retrieve any info from within the file is to use a 3rd party OST2PST conversion utility.

About IMAP/Exchanged based OST files

#1 - An OST file can only be opened by the Outlook profile/email account originally used to create the file

#2 - Backing up an .ost file separately will require a 3rd party OST2PST conversion utility

#3 - If Outlook can't find/open an OST file, it will recreate the file. <IF> the original .ost file was set as the default data file for the profile, it will contain "Contact/Calendar" folders with the added words "This Computer Only". The default data file stores the "default" contact/calendar folders for the profile.

#4 - To mitigate the loss of data in "This Computer Only" folders, suggest the following:
a) add a new PST file to the profile (from Home ribbon, select <new items> --> <more items> --> <outlook data file>
b) Set the newly created PST file as the default data file for the profile
c) Copy all contact/calendar data from the OST folders to those in the PST file
