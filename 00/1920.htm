That setting is "sticky" -- when you check the box, it should stay checked through multiple sessions and reboots. There is indeed a registry entry to store the setting, but don't mess with it.

If you're trying to find a way to check the box for other users who open your document, don't do that. Settings like this are part of each user's personal preferences and should not be changed without their permission. Also, if your document were to change it without the users' knowledge, they would be rightfully angry at the "out of nowhere" change in behavior that appears to be permanent and affecting all their documents. Of course, they'd be angry at Microsoft rather than you, which is hardly fair.

Doug's suggestion, since it would require a user to be aware of the add-in, seems reasonable; but because the setting is sticky, the add-in isn't necessary. Just ask the user to check the box in Options once.

graham mayor say:

The Word data key for Word 2003 has an entirely different construction from that of Word 2010. The Word 2010 recent documents are not stored in the equivalent of that key and any attempt to use the 2003 data key with 2010 will cause Word 2010 to stop working. It is not possible to import the recent documents list from Word 2003 into Word 2010. The simplest task is to take a note of the recent files, and open them in Word 2010, which will build the list.

bernd p. say: I'm afraid we're talking about a mistake made by opening an arbitraryly selected Windows folder as a notebook.
A mistake often discussed here.

To understand what happens by opening any Windows folder in Onenote you have to know the Onenote concept of implementing its storage:

  1. A Onenote notebook is implemented as a folder (not a file !) within the Windows file system (so any Windows folder can accidentally be interpreted as a Onenote notebook)
  2. A Onenote notebook normally contains one ore more sections. These are implemented as files from type *.one.
  3. Each section contains one or more page(s). These are just part of of an *.one file.
That's a "normal" notebook.

But you may have more substructures, called section groups. These are implemented just as folders within a notebook's folder. And so on. Nested. Without limit.

Onenote doesn't know whether a file system folder is really a notebook. It believes you know what you open ...

You got the picture ?

You could just open your C: drive as a notebook ...
