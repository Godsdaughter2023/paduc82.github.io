ewgssi say:

Matt, this is a philosophical answer which may not be a lot of use as far as existing worksheets are concerned, but after more than 25 years working with Excel I have found the following practices to be essential to solid and simplified coding:

  1. Make liberal use of the '[Formulas]Define Name' command to refer to individual cells or cell ranges. If nothing else, your formulae will be more readable.
  2. A lot of array formulae can be avoided by defining a label such as 'thisC' to refer to the active cell: select cell $A$1 and choose '[Formulas]Define Name' command; change the definition to be "=A1" - no $ signs! [This has the effect of labeling the active cell. And, wherever you reference the label 'thisC' - either directly or indirectly - you are referencing that cell. You can then reference other cells on the sheet using the 'OFFSET' formula.]
  3. Don't be afraid to define independent calculable elements which can then be referenced as 'callable' items in superior formulae. This is the easiest way of reducing formula length.
