Steps to perform SFC scan:
1. Click the Start button
2. From the Start Menu, Click All programs followed by Accessories
3. In the Accessories menu, Right Click on the Command Prompt option
4. From the drop down menu that appears, Click on the 'Run as administrator' option

8. Be patient because the scan may take some time
9. If any files require replacing SFC will replace them. You may be asked to insert your Vista DVD for this process to continue
10. If everything is okay you should, after the scan, see the following message "Windows resource protection did not find any integrity violations"
11. After the scan has completed, Close the command prompt window restart the computer and check.
The scan may take some time, so be patient. Windows will repair any corrupted or missing files that it finds. If information from the installation CD is needed to repair the problem, you may be prompted to insert your Windows Vista installation disk.

They came to me when their machine couldn't successfully start Windows due to a corrupt registry.  I took a snapshot of the machine and rebuilt the box with the OEM media to reinstall Windows 8.1 plus apps.

A couple of questions

  1. What files does Live mail store its data in (eg OLK standalone mode stores in PST's) and where are they.
  2. Where is calendar data stored.  I have read a number of MS forum posts that all refer to Microsoft Accounts, however the user is adamant they don't have an MS account.  So I am assuming the data is stored elsewhere (Bigpond's servers perhaps?)
  3. In the forum and support posts for Windows Live mail, posts referring to transferring data from one machine to another assumes you have access to a working system with Live Mail running, something this family no longer had.  How do I transfer data that I can recover from the backup to the newly rebuilt machine.

Any assistance is greatly appreciated

ron sommer mvp 2009-2017 say:

C:\Users\username\AppData\Local\Microsoft\Windows Live Mail\Calendars\

If they have never logged in with a Microsoft account, there will only be one folder in the Calendars folder called DBStore.

If there are also folders with email address names, then they have logged in with a Microsoft account.

Note: the latest upgrade of Microsoft accounts makes a Microsoft account calendar stop working.

Only the default calendar will work.

¡firedog say:

... their Bigpond.com mailbox.  

... the user is adamant they don't have an MS account.  

Some Bigpond mail accounts were and probably still are hosted on the Outlook.com platform, which makes them automatically Microsoft Accounts. This may be significant. A Microsoft Account can use any valid email address, not just @hotmail etc. What matters is whether the user was customarily signed in in Windows Live Mail or not.

What files does Live mail store its data in

Mail account settings, mail folders and messages, calendars and a few other bits and pieces are stored in ESENT databases in the Store folder. Ron gave you the default location; generically, it's

%localappdata%\Microsoft\Windows Live Mail

Contacts are stored separately in one or more databases in the folder called

%localappdata%\Microsoft\Windows Live\Contacts

How do I transfer data that I can recover from the backup to the newly rebuilt machine.

The best way may be simply to copy the two folders \Windows Live and \Windows Live Mail from the old user account to the new one. If the program has already been installed, these folders will already exist; rename them before inserting the copied folders in their place. If the program hasn't yet been installed, the installer should find the copied folders and adopt them.

Because sensitive data (e.g. passwords) are encrypted using a key based on the user profile, they will not survive this exercise, so they will have to be re-entered.

If the user didn't sign in in the old installation, it's important that he doesn't sign in in the new one; if he does, he risks losing his calendar data because of changes in calendar synchronization with calendar.live.com (basically, they broke it).
