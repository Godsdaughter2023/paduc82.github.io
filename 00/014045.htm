greginmich say:

I have another AV app installed (Norton, McAfee, Avast, etc.) – so why are Windows Defender definition updates downloading and then failing to install?

We’ve already identified the most likely cause of this issue – Windows Defender quite often doesn’t get properly turned off in its registry key when a third-party AV app is installed.  

But Error 0x0070643 is a generic error code, so how can I be sure that this registry issue is really the problem in my case?

The formal diagnostic procedure for this issue is to run this command at the command prompt (copy, paste, and enter this line at the command prompt):

reg query "hklm\software\microsoft\windows defender" /v DisableAntispyware

If the return value is 0x0, then Windows defender hasn’t been properly disabled in its registry key.

But I’ve been warned to stay away from the registry; and I have a severe case of Command Line Phobia – isn’t there some other way to see if this is actually my problem?

Of course there is. The informal diagnostic procedure is to simply type “Defender” in the Cortana search bar and then hit the enter key once the “Windows Defender” entry is selected. If you see this screen, then Windows Defender hasn’t been turned off in its registry key. It’s still turned on; but the service can’t be started – so Windows Defender is in a state where it can’t install the definition update:

OK, it looks like that’s my problem; but do I really need to fix it?

No, you don’t – there haven’t been any serious consequences resulting from this issue. Other updates seem to be processed without any trouble, and the associated Error 577 (on attempting to start the Windows Defender Service in services.msc) is a false alarm. There’s really nothing wrong with the Windows Defender Service executable (MsMpEng.exe).

This stuck update really bothers me; so how can I fix it?

Repeating the installation of the third-party AV app after uninstalling it and running its clean-up utility should switch Windows Defender off in its registry key. However, this has been reported to fail repeatedly with Norton products; and redoing the installation might be considered too time-consuming in other cases – so we have a shortcut that uses Group Policy to completely turn off Windows Defender. This is done with the Group Policy Editor in the Professional edition of Windows 10, and with the Ten Forums Turn_Off_Windows_Defender.reg file in the Home edition. Everything you need is right here:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

We also have a prospective solution that appears to directly update the relevant registry entries when the third-party AV app is disabled; Windows Defender is started; and the third-party app is then turned back on while Defender is still turned on. This seems to work because it mimics the state changes that occur in a reinstallation:

http://answers.microsoft.com/en-us/windows/forum/windows_10-update/definition-update-for-windows-defender-kb2267602/d4edd314-a438-4ac2-b7a9-dc1c45a1ef62

Important Notice & Disclaimer:

Forum moderator Stephen Boots recently published this Wiki article on the subject of Limited Periodic Scanning:

http://answers.microsoft.com/en-us/protect/wiki/protect_defender-protect_scanning/limited-periodic-scanning-in-windows-10-to-provide/3dcfc8dd-36b4-43f2-9a0a-732e43191cf3

So we see that Microsoft has opted for a new feature that could either add chaos to confusion for this definition update issue, or potentially resolve it – but which in either case will probably break our current ideas about the proper operational state for Windows Defender when a third-party AV app is installed. But on the bright side, it sure is good to hear from Steve.  

GreginMich

Outlook Mail (Preview) and Windows Live Mail - Microsoft Community

max stewart say:

Outlook Mail (Preview) and Windows Live Mail

What follows was originally posted as a reply to another post on the outlook.com board. But it was suggested I open it here as a discussion topic.

In the wonderful world of Microsoft, there were two and only two desktop products that supported contact groups (contact categories)  that would also synchronize with a central (cloud) copy : Office Outlook (chargeable) and Windows Live Mail (WLM - free; WLM is the distant successor to Outlook Express that that was the standard PC-based mail program as long ago as Windows 3.1 or thereabouts). And you need the central copy in order to allow your PC to synchronize contact updates and contact groups updates with other devices such as a phone.

The central copy could also be accessed via a browser using originally Hotmail, then outlook.com. Note that the central copy was in principle independent what product (e.g. outlook.com from a browser; Outlook from a desktop; WLM) was used to access it.

Last year, Microsoft started a process of converting the central copy itself from a special-purpose mail service to full Exchange Server (the server product used by businesses). This is in line with their approach elsewhere of simplifying and standardizing their other services such as putting Window 10 everywhere (PC, tablet, phone,...) and putting the pressure on everyone to convert.

Many outlook.com users have already been converted. The new Exchange Server email service comes with a new browser-based website front-end: Outlook Mail (a.k.a. outlook.com 'preview version' and a variety of other names - not all polite...!). Outlook Mail replaces outlook.com for those who have been converted.

Outlook Mail supports contact groups, as of course does the Exchange Server service that it uses. And Outlook Mail comes with a collection of other products (some you have to pay for), one of which is People (Contacts). And Microsoft appears undecided whether a contact GROUP is 'something special' or merely a specific type of CATEGORY, in which case they can use the existing category mechanism of clients such as Outlook. Expect to see both terms used somewhat interchangeably (if confusingly).

Now, a slight digression...

WLM uses a special protocol called Deltsync to synchronize PC contacts and contact groups with the central outlook.com copy (this was provided to desktop Outlook by an add-on called the 'hotmail connector'). 

But Microsoft has at least nine other protocols up their corporate sleeve used by businesses to synchronize local and central copies of contacts and email (and other things)....   among which are Exchange Active Synch (EAS - a 'lite' version which uses relatively low bandwidth and is widely used by them and third parties for linking smartphones and tablets to the central service) and a couple of 'full' Exchange Server protocols (Outlook Anywhere - a.k.a. RPC over HTTP, which is now being superseded by MAPI over HTTP).

When an outlook.com user has been converted to Outlook Mail, they will find that WLM cannot use Deltasync to connect to the new central version - because Exchange Server (at least in the form Microsoft offers it) does not support Deltasync.  Instead, they have been induced to move to Windows 10 and use the Mail app. The Mail app uses EAS to synchronize with the central version. AND EAS DOES NOT SUPPORT CONTACT GROUPS. The Mail app is also functionally deficient in many areas: it is a triumph of glitz over substance. 

Users of Outlook versions up to Outlook 2010 could add the Hotmail connector and have full access to contacts and contact groups from central outlook.com. By some miscalculation on Microsoft's part that defies belief, the Hotmail connector was withdrawn for Outlook 2013 which was released over three years ago.  Instead, EAS was provided, and this meant that users of a flagship Microsoft product (Outlook) has no access to contact groups from another key (if free) Microsoft service (outlook.com). Only now are outlook.com accounts being converted to Outlook Mail en masse - and many have yet to be converted. THREE YEARS; 400M USERS (to be fair, most will not be Outlook users).

WLM users who remain on Windows 7 and 8 have been offered an update to the latest WLM (2012) that supports EAS, but to date this has been beset with a host of problems. AND WHEN WHEN IT WORKS IT USES EAS - HENCE NO CONTACT GROUPS.

You are left with two Microsoft-supplied options: work online via a browser and Outlook Mail, or buy or subscribe to chargeable desktop Outlook.

I apologize for the length and tediousness of the foregoing. If the new Windows 10 Mail app had been functionally equivalent to WLM but with an additional touchscreen interface, Microsoft would have saved themselves a host of problems and unhappy users. But this would have meant in effect supporting Exchange's 'MAPI over HTTP'* instead of Deltasync* in order to allow the move to an Exchange Server service. On the desktop, this is now the exclusive province of Outlook - which comes as part of an Office 365 subscription.

This is my view: I would welcome others!

* the synchronization mechanism is a teeeeny bit more complex..... : I have deliberately blurred the difference between identifying what has changed on each side and the protocol to send and receive the changes. In Exchange, synchronization is handled by a component called Message Store
