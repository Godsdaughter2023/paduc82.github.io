nette s say:

If it's the whole document that you've set up as a two column document it should spill into second column only when the first one is full.

If you have a document where only part of it is two column you will have a continuous section break after the columns. Place your cursor in the section AFTER the columns and change the section break to a new page section break (Layout tab, click on the little arrow in the bottom left corner of the Page Setup group, select the Layout tab. doug robbins - mvp office apps & services (word) say:

Run a macro containing the following code:

ActiveDocument.Compatibility(wdNoColumnBalance) = True

Or, to toggle the setting, use:

With ActiveDocument
    If .Compatibility(wdNoColumnBalance) = True Then
        .Compatibility(wdNoColumnBalance) = False
    Else
        .Compatibility(wdNoColumnBalance) = True
    End If
End With

How do I protect a header in a worksheet? - Microsoft Community

gtrpkr say:

How do I protect a header in a worksheet?

I have been able to protect a worksheet by locking specific cells while allowing other cells to be accessed. I have not been able to find out how to protect my header from being changed.
j.e. mcgimpsey say: How much work are you willing to do?

There's no built-in way to protect headers and footers. I've done this a couple of ways with VBA:

  1. use the Workbook_Open(), _BeforeClose(),  _Activate(), and _Deactivate() events to disable the Headers&Footers menu item when the workbook is active. This kept most people out of the headers until Page Layout view..., so the code would need to set Normal view, and disable changing views as well.
  2. Use the Workbook_BeforePrint() event to reset the headers to your desired content before any printing occurs.
Both of these suffer from requiring the user to allow macros to run. If they don't, neither protection will work. To combat THAT, you'd need to do something like make your first sheet a splash screen that told the user to enable macros, then make the rest of the sheets hidden with the xlVeryHidden property. That in turn means that once macros are turned on, you'd need to include code in the Workbook_BeforeSave event that rehides those sheets (or the user will simply be able to open the workbook once with macros enabled, save the file, then close and reopen it without macros).

And none of these methods is really secure (any more that it's about 30 seconds work to bypass worksheet protection).

Read-only Compatibility mode - Microsoft Community

josee & stephenfrosst say:

Read-only Compatibility mode

I bumped into my computer a few minutes ago, and now my Word 2013 files all come up with Read-only compatibility Mode.  This does not affect the editing of the files.

How can I get rid of this?

rohn007 say:
Bumping the computer should not trigger this.  Did you turn off the power with that bump?

The "Compatibility mode" flag is displayed when you are opening a file created in an older version of Office. You can get rid of the flag by "converting" the file format.

With the file open, go into File menu > Info command > Convert button > Compatibility Mode text > OK (on informational message).

The "read only" part can happen 2 ways. In the File Open dialog, click on the Open button drop down arrow, select Open read only.

The other way is crashing Word when a file is open. That leaves behind a small "flag" file that indicates someone else has the file open and you can only open it as read only.
