graham mayor say:

Generally speaking it is preferable not to include macros in protected forms as you cannot force users to run them and so the object may be defeated. However, if you wish to save the form, that is not too difficult to arrange provided you know where you wish to save the form and how you wish to name it.

The following example will set the folder as the User's documents folder and name the document with the content of the formfield 'FieldName' and display the SaveAs dialog so the user can make changes as required.

Sub SaveForm()
Dim strPath As String
Dim strName As String
    strPath = Environ("USERPROFILE") & "\Documents\"
    strName = ActiveDocument.FormFields("FieldName").Result
    If strName = "" Then
        MsgBox "Complete the 'fieldname' field!"
        ActiveDocument.FormFields("FieldName").Select
        Exit Sub
    End If
    With Dialogs(wdDialogFileSaveAs)
        .Name = strPath & strName
        .Show
    End With
End Sub

Array Index Out of Bounds (again) - Microsoft Community

bill j. say:

Array Index Out of Bounds (again)

I have a routine in Microsoft Access that produces an email.  I had to modify the code for Office 2016 where .Display needed to be brought to the top of the stack:

The following works with Windows 7 and MS Access 2013

Public Function ProcessEList(stTo As String, _
                            Optional stCC As String, _
                            Optional stSub As String, _
                            Optional stRpt1 As String, _
                            Optional stRpt2 As String, _
                            Optional stMsg As String, _
                            Optional stRpt3 As String) As Boolean

    Dim objOutlook
    Dim objMail

On Error Resume Next
    Set objOutlook = GetObject(, "Outlook.Application")
    ProcessEList = False

    If Err.Number <> 0 Then
        Set objOutlook = CreateObject("Outlook.Application")
    End If

    If IsNull(stCC) Then
        stCC = ""
    End If

    Set objMail = objOutlook.CreateItem(cnMailItem)

    With objMail
        .To = stTo
        .CC = stCC
        .Subject = stSub
        .HTMLBody = stMsg
        .BodyFormat = cnHTMLBodyFormat
        .Attachments.Add stRpt1
        .Attachments.Add stRpt2
        .Attachments.Add stRpt3
        .Display
            AppActivate stSub, True
    End With

    ProcessEList = True
