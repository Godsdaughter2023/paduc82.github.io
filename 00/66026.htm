¡firedog say:
  1. 'Attach' usually refers to the process of sending a file along with a mail message. If you want the recipient to see an image in the body of a mail message, the usual term is 'embed'. This covers both the case where the image to be embedded is stored on a web page and retrieved when the recipient reads the message and the case where the image is transmitted together with the mail message as an attachment.
  2. You can insert a photo into the body of a mail message by using the Single photo option on the Insert tab. This will not require you to sign in if you don't want to.
  3. The photo email feature that uses a photo album doesn't insert images into the body of a message. It arranges for the image files to be uploaded to your OneDrive, while sending thumbnail images to the recipient so he can view them and save copies if he wants to. This process requires you to be signed in with a Microsoft Account.
  4. When you changed your password, you probably enabled two-step verification. If you did, you must generate a unique app password to use in dumb mail clients like Windows Live Mail that don't have the ability to request and submit security codes.

Good work!  I'm glad chkdsk option included in my oiriginal post solved the primary problem (if only temporarily).  It does sound like your hard drive may be dying (or getting very close to dead).  We should do a complete diagnostic test of it to see what it shows.

To test your hard drive, check the manufacturer and then get the diagnostic utility from:http://www.techsupportforum.com/hardware-support/hard-drive-support/302602-hard-drive-diagnostic-utilities.html.  If it fails the test, replace it.  If it passes the test, then there's probably nothing wrong with it that can't hold off for a while (but the corruption on the hard disk is a sign that there is something going bad with the hard disk even if it passes this test).

The csrss.exe crash almost certainly did not cause the corruption problems on your hard drive - that's much more likely a hardware problem (but the above will see) - and it's more likely that the corruption caused the crash.

I can see no harm in installing the hotfix - though I'm not sure it will solve the problem (especially if you're not really using console applications or it's not occurring only when you close them).  Yes, it seems to apply to your situation in some ways and may resolve that issue, but I really think the underlying issue is the hard drive itself.

I hope this helps.

Good luck!

  1. Close any running programs and open Windows so you are back at the desktop.
  2. Click on the Start () button.
  3. In the Start Menu Search Box type msconfig
  4. Then press enter on your keyboard.
  5. The System Configuration utility will open
  6. Make sure Startup Selection is set for Normal Startup.
  7. Click on the Boot tab.
  8. Make sure  Safe boot is unchecked
  9. Press the Apply button and then press the OK button.
  10. Click on the Restart button to reboot your computer.
