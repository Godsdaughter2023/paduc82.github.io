Laurie,

My pleasure.  Learned a few new things in looking this up.

First off, FixLinks won't help in this situation.  I think you can pretty well trust my opinion on this; I wrote it.  ;-)

OTOH, it might help if you need to bulletproof links to individual PDF files outside a portfolio. First, though, I'd put all the PDFs in the same folder as the saved PPT/PPTX, and only then create the links to them.  In some cases, that creates a "pathless" link which usually works on other systems, so long as the PPT and linked files remain in the same folder.  In other cases, PPT insists on including the full path to the linked file.  On your PC.  So it breaks when you move it somewhere else.  That's the kind of thing FixLinks can help with.

steve rindsberg say:
Portfolio creates one uber-file that contains other subsidiary PDFs.  If the links in PPT point to the original (ie, subsidiary) PDFs, then I'd *expect* them to break if you only send the portfolio PDF plus the PPT.

If the linked-to file isn't there (as an independent file on the hard drive, not as part of a zip or portfolio), the link will break. 

I've done a bit of research but haven't found a way of linking from an external document to a specific file within a portfolio, but you can apparently create links within the portfolio.  So you could conceivably link from the PPT to the portfolio PDF and have its opening page contain a menu of links to the other PDF documents within the portfolio.

Or you could just distribute the PPT along with all the linked PDFs.
