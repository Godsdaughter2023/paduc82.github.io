br1836 say:

Address Book Won't Load in OutLook Express

Running from a limited account Outlook Express fails opening the Address Book with
the following messages:
   1). The Address Book failed to load. Outlook Express s incorrectly
       configured, please reinstall.
   2). Unable to open the Address Book. The Address Book may not be
       installed properly.

However, if the limited account is changed to an administrator account there
is no problem.

There are two accounts on the PC; HP_Administrator and User1. User1 is the
account having the problem. HP_administrator has administrator privileges
and if User1 is given administrator privileges then I noticed that both accounts
use the same address book. That is, from Outlook Express
"Addresses -> Help -> About Address Book" in both accounts show

 C:\Documents and Settings\Administrator\Application Data\Microsoft\Address Book\Administrator.wab

as the Address Book being used.

This seems incorrect. I would expect Administrator to be replaced by HP_Administrator
or User1 in the above path depending on which account Outlook Express was
run from. It appears that the OS is configured to use the Address Book shown above,
regardless of which account is running Outlook Express. If User1 is a limited account
its not possible to view the location of the Address Book due to the the error that
is occurring, but I suspect that Outlook Express is trying to access the Address Book
shown above and a permissions problem occurs due to the limited account User1 trying
to access the Address Book in the Administrator folder.


Windows XP with service pack 2 is the OS and its Outlook Express 6.

What needs to be done so that Outlook Express can successfully run in the
limited account User1.
br1836 say: The problem has been resolved. I've written the solution for people
who, like myself, have no experience with the Windows Operating sytem.

The sites at these links should be read. For Outlook express
   http://www.insideoe.com,

specifically

    http://www.insideoe.com/files/wab.htm.


For information on the registry and how to edit it the following two sites
give necessary information and should be read carefully.

 1). http://www.techsupportalert.com/content/learn-how-use-windows-registry-editor-regedit-one-easy-lesson.htm

 2). http://pcsupport.about.com/od/termsr/p/registrywindows.htm
     (You will find other useful links on this site)

One note; There are different sets of "HKEY_CURRENT_USER" keys, 1 set for each
account. To change a "HKEY_CuRRENT_USER" key for the account User you must be
logged on as User.  



In the Registry, the key "HKEY_CURRENT_USER\Software\Microsoft\WAB\WAB4\Wab File Name"
contains the pathname of the address book being used by Outlook Express, for
the account you are logged into.


When logged in as User and examining the Registry with regedit

  "HKEY_CURRENT_USER\Software\Microsoft\WAB/WAB4\Wab File Name"

 was set to

  C:\Documents and Settings\Administrator\Application Data\Microsoft\Address Book\Administrator.wab.

Since User is a limited account, User lacks the privilege to access the address book at this
location, which resulted in the errors given in the original post. The solution is to
modify this pathname. Using regedit the pathname was changed to

  C:\Documents and Settings\User\Application Data\Microsoft\Address Book\User.wab.


Since C:\Documents and Settings\User is the root folder for the account User, User has
the privilege to access the address book at the new location. Outlook express was
then able to access the address book.




On reflection, it seems to me that the above solution just treats a symptom, rather than the cause.
The original question actually should be:

  How is the system configured so that when a new account is created the new account
  will use an address book at

  C:\Documents and Settings\New Account\Application Data\Microsoft\Address Book\New Account.wab,

rather than

  C:\Documents and Settings\Administrator\Application Data\Microsoft\Address Book\Administrator.wab,


It shouldn't be necessary to edit the Registry for each new account.

ahamedsajad say:

increase gaming performance

Hi all,

Can I increase gaming performance by Windows ReadyBoost technology?
bossdweeb say: I don't think ready boost will  increase gaming performance, but no harm trying.
One way to find out if you need more RAM is to run a monitor on the RAM while
playing a game.

Gaming performance is mostly reliant on a good graphics card, and to a some degree on cpu speed. Some games are more cpu heavy than others
( MS flight sim is one).
Having 4GB system RAM is desirable as well. I have 8GB, but even with my most
demanding game I have never used more than 45+% of it.
Ready boost may help if there is only 2GB system RAM, but I'm not really sure as games rely more on VRAM (fast dedicated video RAM) and may allocate the system RAM in such a way that Ready boost is of little benefit, or just not use it at all.

If you are using a laptop with Integrated Graphics as so many people are doing these days there is nothing you can do to improve performance.


sanmartin say:

ReadyBoost provides an additional cache which allows to read data from your harddisk faster than the harddisk itself. It's not an "additional RAM" but it can serve to unburden your system RAM from similar caching techniques if the available RAM amount is low. On the other hand, operations using an external USB device are definitely slower than operations within the actual system memory, so any memory issues concerning gaming performance should be addressed by increasing the system RAM.

Two personal observations: I've tried ReadyBoost with 1.5 GB system RAM and didn't notice any improvements (maybe even some decreased performance due to the additional load/unload operations on the USB flash device). With 8 GB RAM, the most memory-hungry games are rarely consuming more than 3 GB, so there's no need for "tricks".

What BossDweebe wrote, gives another aspect: "Memory" for games includes two big and widely independent parts, video memory and system memory. The system memory provides the data by reading from / writing to the harddisk, the video memory (on the GPU / video card) helps to process them by making the GPU calculations faster. You can't speed up the VRAM by increasing the system RAM, and vice versa.
