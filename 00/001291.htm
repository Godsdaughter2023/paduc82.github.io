If we click the SharePoint title in App Launcher, we will get the SharePoint home page. It is not feasible to launch team site directly via clicking SharePoint. carol's hubby say:

I'm a bit confused because this was a question for me back in December. However, I saved the answer from Microsoft Answers. Here it is. My original problem was that I didn't want the system (for example) to automatically start a 2 when I finished a 1 element on a list. Quoting, from Microsoft Answers:

Disable Multilevel Lists                  From M’soft Answers December 10, 2011

It sounds like you want to type the numbers yourself (static text) and not have Word continue the numbering or indent sublevels. If that's the case, you're not being troubled by multilevel lists. It's autoformatting that you need to turn off.

Go to Word options > Proofing > AutoCorrect Options > As You Type tab. Clear check boxes before

1. Automatic bulleted list,

2. Automatic numbered lists,

3. Format beginning of list item like the one before it, and

4. Set left and first indent with tabs and backspaces. You may have to play around with these settings a bit to decide what you want.

Note that multilevel lists are flexible and customizable enough to cover most numbering requirements. You can choose the number stye (1,2,3, A,B,C, etc), alignment, and indention (or lack of it) for each level when you set up the list.

    Asp.Net app to edit documents using office web apps server - Microsoft Community

    leelamanas say:

    Asp.Net app to edit documents using office web apps server

    Hello,

    I am trying to build an Asp.Net application which contains a document store (showing a list of docx files). Once you click on a document it should open the right Office Web Apps Server viewer/editor. 
    Is this possible to implement?

    Thanks,
    Manas
    rob rolnick say:

    Hi leelamanas,

     

    Yes, it is possible.  However, there are a number of considerations you'd need to make before doing this.  From a technical point of view, the Office Web Apps Server is designed to support your scenario.  However, there are several documented protocols you'd need to implement on the host (your Asp.Net application containing the document store.)  For example, viewing requires the WOPI protocol, and editing would require the FSSHTTPB protocol (aka Cobalt protocol) as well.

     

    There is also licensing considerations.  The Office Web Apps Server license is free for viewing, but paid for editing. 

     

    Hope that helps,

    Rob Rolnick
