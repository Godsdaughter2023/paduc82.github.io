Hi Vyonnah:

OK, start Word, on the file menu choose Open Recent...

If the document is still where you saved it, it will be in that list.

If not, open Finder and type some words from the file name of your document into the Spotlight Search field at the top right of the Finder.

scottgem say:
I'm not a SharePoint expert, but you should be able to upload your data table to a Sharepoint List, then link the Access FE to that list.

But I agree with that article that using SQL Server back end would be better. I disagree with that article about protection. Let me explain.

A lot depends on what you are trying to protect from But if you create your front end as an accde. Disable the bypass keys. Disable menus. Set the form as the default and set the Data Entry property to Yes, you have gone a long way towards protecting the data. This essentially hides any code or settings in which the user can tell where the back end is. They can't see any previously entered data or get at the table.

So my point is you can protect the FE to not allow access to where the back end is. Moving the data to a SharePoint list, SQL Server database or Azure database just adds some more levels of security.

Yes, you could have no local tables and the form would be bound to the linked Sharepoint list. What I was saying, is that you can go a long way to protect whatever back end you use.
