Hello Kenneth

Contact sort sequence is not a "sync option" but rather is governed by whatever email client is in use whether that is Outlook or any other (or webmail configuration as the case may be).

There are two different places that people commonly refer in term of 'sort order"

#1 - The Outlook AddressBook
#2 - A specific contact folder

The two items above are separate and distinct.

To set the Outlook AB to appear in First/Last sequence, you need to

a) go to File --> Account Settings --> select the "Address Books" tab
b) highlight the AddressBook then --> select <Change?
c) choose the option to show the AB in First/Last order

To set a given contact folder in First/Last sequence

d) open the contact folder in question from the Outlook Nav Pane
e) select the <View> ribbon
f) click on <View Settings>
g) click on the <Sort> button and configure it to use the desired sort order which in this case would be <FirstName> as the first field followed by <LastName> as the second field

Your existing macro will delete the |, not simply superscript the character following it. To reverse the process you could use something like:

With ActiveDocument.Content.Find
  .ClearFormatting
  .Replacement.ClearFormatting
  .Font.Superscript = True
  .Format = True
  .Text = ""
  .Replacement.Text = "|^&"
  .MatchWildcards = True
  .Execute Replace:=wdReplaceAll
End With

Do note that the | will also be superscripted. You could either use a second F/R to 'correct' that or a much slower single F/R with a loop to process each found item individually.

rich michaels - mvp (greatcirclelearning.com) say:

After a little research and thinking about this more I came up with the following macro, which does, what I think you are after... select a sentence, paragraph, etc that contains edits and then invoke the macro.

What I am not sure about is the exact placement of the vertical bars per your specs. The odd page placement as an offset to the right margin in particular confused me, your documents must have some pretty wide margins.

Anyway, try the macro out and hopefully it does the job for you.

Sub InsertEditMarkers()
    'Written by Richard V. Michaels, Microsoft Word MVP
    'February 2018
    'http://www.greatcirclelearning.com
    
    Dim shp As Word.Shape
    Dim Top As Single, Bottom As Single
    Dim Left As Single, Right As Single, TextArea As Single
    Dim wght As Single, Even As Boolean
    Dim SelStart As Word.Range, SelEnd As Word.Range
    
    With ActiveDocument.PageSetup
        TextArea = .PageWidth - .LeftMargin - .RightMargin
        Left = .LeftMargin - 36 '(36 = 72 dpi / 2)
        Right = .LeftMargin + TextArea + 36
    End With
    'weight of vertical line
    wght = 2.25
    
    
    Set SelStart = Selection.Range
    SelStart.SetRange SelStart.Start, SelStart.Start
    Top = SelStart.Information(wdVerticalPositionRelativeToPage)
    Set SelStart = Selection.Range
    If SelStart.Paragraphs.First.Range.InRange(Selection.Range) Then
        With SelStart.Paragraphs.First
            Top = Top + .SpaceBefore + .SpaceBeforeAuto
        End With
    End If
    
    Set SelEnd = Selection.Range
    SelEnd.SetRange SelEnd.End, SelEnd.End
    Bottom = SelEnd.Information(wdVerticalPositionRelativeToPage)
    Set SelEnd = Selection.Range
    If SelEnd.Paragraphs.Last.Range.InRange(Selection.Range) Then
        With SelEnd.Paragraphs.Last
            Bottom = Bottom - .SpaceAfter - .SpaceAfterAuto
        End With
    End If
       
    Even = (Selection.Information(wdActiveEndPageNumber) Mod 2) - 1
    If Even Then
        Set shp = ActiveDocument.Shapes.AddLine(Right, Top, Right, Bottom)
    Else
        Set shp = ActiveDocument.Shapes.AddLine(Left, Top, Left, Bottom)
    End If
    With shp.Line
        .ForeColor.RGB = RGB(0, 0, 0)
        .Weight = wght
    End With
End Sub
