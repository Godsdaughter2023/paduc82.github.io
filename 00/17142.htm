2. For Windows sleeping even when there are background tasks, you can try changing the Sleep settings in Windows to never sleep. Windows will normally let the computer Sleep if it is idle for a period of time (this is not affected by the background tasks).

3. Regarding the speed, heavy network load may play a part in OneDrive not reaching higher speeds, as you mentioned, at night the speed can reach up to 10MB/s. This might be because less people are using the network at night.

I completely understand your situation; in most cases, a company would have a lot of libraries to sync and setting it up for the first time in multiple computers will take time as OneDrive needs some time to run the first time syncing process.

As Linda has mentioned, it is not feasible to sync multiple libraries at once using an automated process.

How to calculate time spent between working days

 Everyone

I want to calculate how much working spent for order to finish between working days

For example I have A2 where start date and time "20-11-17 11:21 AM" and at B2 end date "05-12-17 5:20 PM" 

Noting that working hours are from 9:00 AM to 5:00 PM

I want to calculate how much working spent between the two times excluding days off and holidays 

Thanks, 

hansv mvp say:

Let's say you have a range that lists holidays and named it Holidays

=NETWORKDAYS(A2,B2,Holidays)-1+MOD(B2,1)-MOD(A2,1)

formatted as General will return the time in days, in your example 11.249.

If you want it formatted as time, apply the custom format [h]:mm. This will result in 269:59.

mike h.. say:

Hi,

See the snip below. The times for the working day are in D1 and d2 but you can put them anywhere and edit the formula. The formula in F1 is this:-

=((NETWORKDAYS(A1,B1,Holidays)-1)*(D2-D1)+IF(NETWORKDAYS(B1,B1,Holidays),MEDIAN(MOD(B1,1),D2,D1),D2)-MEDIAN(NETWORKDAYS(A1,A1)*MOD(A1,1),D2,D1))

Note that in the formula (in bold) is the named range of Holidays Create this named range and add any holiday dates.

EDIT,,, I forgot to metion you format the formula call with the custom format of [hh]:mm
