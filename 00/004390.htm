Option Explicit
'Main Function
Function SpellNumber(ByVal MyNumber)
    Dim Dollars, Cents, Temp
    Dim DecimalPlace, Count
    ReDim Place(9) As String
    Place(2) = " Thousand "
    Place(3) = " Million "
    Place(4) = " Billion "
    Place(5) = " Trillion "
    ' String representation of amount.
    MyNumber = Trim(Str(MyNumber))
    ' Position of decimal place 0 if none.
    DecimalPlace = InStr(MyNumber, ".")
    ' Convert cents and set MyNumber to dollar amount.
    If DecimalPlace > 0 Then
        Cents = GetTens(Left(Mid(MyNumber, DecimalPlace + 1) & _
                  "00", 2))
        MyNumber = Trim(Left(MyNumber, DecimalPlace - 1))
    End If
    Count = 1
    Do While MyNumber <> ""
        Temp = GetHundreds(Right(MyNumber, 3))
        If Temp <> "" Then Dollars = Temp & Place(Count) & Dollars
        If Len(MyNumber) > 3 Then
            MyNumber = Left(MyNumber, Len(MyNumber) - 3)
        Else
            MyNumber = ""
        End If
        Count = Count + 1
    Loop
    Select Case Dollars
        Case ""
            Dollars = "No Dollars"
        Case "One"
            Dollars = "One Dollar"
         Case Else
            Dollars = Dollars & " Dollars"
    End Select
    Select Case Cents
        Case ""
            Cents = " and No Cents"
        Case "One"
            Cents = " and One Cent"
              Case Else
            Cents = " and " & Cents & " Cents"
    End Select
    SpellNumber = Dollars & Cents
End Function
      
' Converts a number from 100-999 into text
Function GetHundreds(ByVal MyNumber)
    Dim Result As String
    If Val(MyNumber) = 0 Then Exit Function
    MyNumber = Right("000" & MyNumber, 3)
    ' Convert the hundreds place.
    If Mid(MyNumber, 1, 1) <> "0" Then
        Result = GetDigit(Mid(MyNumber, 1, 1)) & " Hundred "
    End If
    ' Convert the tens and ones place.
    If Mid(MyNumber, 2, 1) <> "0" Then
        Result = Result & GetTens(Mid(MyNumber, 2))
    Else
        Result = Result & GetDigit(Mid(MyNumber, 3))
    End If
    GetHundreds = Result
End Function
      
' Converts a number from 10 to 99 into text.
Function GetTens(TensText)
    Dim Result As String
    Result = ""           ' Null out the temporary function value.
    If Val(Left(TensText, 1)) = 1 Then   ' If value between 10-19...
        Select Case Val(TensText)
            Case 10: Result = "Ten"
            Case 11: Result = "Eleven"
            Case 12: Result = "Twelve"
            Case 13: Result = "Thirteen"
            Case 14: Result = "Fourteen"
            Case 15: Result = "Fifteen"
            Case 16: Result = "Sixteen"
            Case 17: Result = "Seventeen"
            Case 18: Result = "Eighteen"
            Case 19: Result = "Nineteen"
            Case Else
        End Select
    Else                                 ' If value between 20-99...
        Select Case Val(Left(TensText, 1))
            Case 2: Result = "Twenty "
            Case 3: Result = "Thirty "
            Case 4: Result = "Forty "
            Case 5: Result = "Fifty "
            Case 6: Result = "Sixty "
            Case 7: Result = "Seventy "
            Case 8: Result = "Eighty "
            Case 9: Result = "Ninety "
            Case Else
        End Select
        Result = Result & GetDigit _
            (Right(TensText, 1))  ' Retrieve ones place.
    End If
    GetTens = Result
End Function
     
' Converts a number from 1 to 9 into text.
Function GetDigit(Digit)
    Select Case Val(Digit)
        Case 1: GetDigit = "One"
        Case 2: GetDigit = "Two"
        Case 3: GetDigit = "Three"
        Case 4: GetDigit = "Four"
        Case 5: GetDigit = "Five"
        Case 6: GetDigit = "Six"
        Case 7: GetDigit = "Seven"
        Case 8: GetDigit = "Eight"
        Case 9: GetDigit = "Nine"
        Case Else: GetDigit = ""
    End Select
End Function

“Old” Tips – “Hey dummy, why are you giving me tips to older versions of Office or Windows” <grin>

Yes, many of the tips are "old".

But, inspite of the new "revolutionary" metro gooey shell pasted on top of the "traditional desktop", much (most?) of the underlying windows code is "evolutionary" and remains essentially the same (with some real improvements, like the new Copy processing).  Which means many of the "old" techniques still work, if you can figure out where the access point to the dialog boxes have been "improved to" (aka hidden, <grin> ) .  That often includes tips going back to XP and EARLIER! 

I still use "DOS" batch command files in Windows 8.  And "DOS" command line startup parameters to modify Windows 8 shortcuts / tiles.

The same concept applies to Office. Like the Menu, the Ribbon is just a shell used to access the underlying commands that do the real work. The commands haven’t changed, just the way you access them has changed.  Many of the “old” macros work in the new er version os of Office without any change.

Rule to add text in Subject Line - Microsoft Community

ryan luthman say:

Rule to add text in Subject Line

Is there a way to always include certain text in an email Subject Line for emails sent to specific Contacts?  Outlook 2016.

Thanks!

graham mayor say:

You cannot do it with a rule, but you can with a macro. The following code in the ThisOutlookSession module will interrogate each message you send and if the address shown is a recipient it will add the text to the end of the subject. (You could just as easily add to the start of the subject or replace the subject).

Private Sub Application_ItemSend(ByVal Item As Object, Cancel As Boolean)
Dim oRecip As Recipient
    For Each oRecip In Item.Recipients
        If oRecip.Address = "*** Email address is removed for privacy ***" Then
            Item.Subject = Item.Subject & " - Add this text after the subject"
            Item.Save
            Exit For
        End If
    Next oRecip
End Sub

Because of Outlook security you may need to self certify the macro project - see

http://www.gmayor.com/create_and_employ_a_digital_cert.htm
