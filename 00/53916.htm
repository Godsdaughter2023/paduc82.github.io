malkeleah say: You have Kodak EasyShare (a problematic program at its best) set to start with Windows. Either you uninstalled it or it is damaged and it can't start; hence the error messages.

If you didn't uninstall Kodak EasyShare and don't use it, go to Control Panel>Add/Remove Programs and uninstall it.

If you did uninstall Kodak EasyShare then a reference was left to it in Startup. Remove the reference:

Start>Run>msconfig [enter]

This brings up the System Configuration Utility. Look on the Startup tab and find the probable culprit. Uncheck the box next to its name, Apply and OK out. You don't need to restart immediately, but the next time you do you'll get a dialog saying you've used the Utility. Just tick the box that says in effect, "don't bother me about this again".

Important - Do not use the System Configuration Utility to stop processes. Instead, use Start>Run>services.msc [enter] and do not stop any services unless you really, really know what you're doing.

How to Troubleshoot By Using the Msconfig Utility in Windows XP - http://support.microsoft.com/?id=310560
The free Autoruns program is very useful for managing your Startup - http://www.microsoft.com/technet/sysinternals/default.mspx

If you do use Kodak EasyShare, uninstall it first and then reinstall the latest version. There is still no reason to have it start with Windows.

When you reinstall Windows, you also need to reinstall all of the drivers that are appropriate for your computer.  Although Windows XP has generic drivers for a lot of things, graphics cards are particularly fussy.

Unfortunately, I couldn't find anything on the Chaintech site that looked remotely like your card.  I did findthis site , that says that the Chaintech D33058 uses an nVidia GeForce FX5700 (or FX 5700LE) chipset.  The latest nVidia driver for that chipset ishere .

mgnscar say: Thanks that's very helpful. How will I know if I need to install a driver for the motherboard? since I should install that first? Also, it's not from a computer store. It was built by an ex boyfriend so I definitely won't be contacting him for support. :)

lemp say:
Thanks that's very helpful. How will I know if I need to install a driver for the motherboard? since I should install that first? Also, it's not from a computer store. It was built by an ex boyfriend so I definitely won't be contacting him for support. :)

If the computer was built from parts by your ex, you have a 2-step process.  First you have to find out what hardware you have and then you have to go to the web site of each mfr and see what's available to download for each specific hardware item.

For step 1, there are a number of system information utilities available.  Try SIW (System Information for Windows) available freehere . The main problem with SIW is that it displays so much information that it may be hard to pick out what's relevant to the task of determining what drivers you need.

Once you have SIW downloaded and installed, scroll down to the Hardware section.  If you click on "Motherboard," you'll find the manufacturer and model.  Also check Network Adapters, Video (which may tell you a bit more about your graphics card), and Sound Device.   There is much more detail about each of these items under the "Devices" entry, but you probably don't need that level of detail.  If you have an internal modem (i.e., mounted inside your computer), look for its information under Devices. That should be about it, unless you notice something else behaving peculiarly.

I would make a System Restore point before installing any of the drivers.  If things are worse after you install a driver, use System Restore.  Also note that for the video board, sound board, and network adapter, you may be able to use Device Manager to "roll back driver" using the button with that name on the Drivers tab of the Properties for each of those items as shown in Device Manager.
