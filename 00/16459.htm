Hi everybody,

I encountered a strange behavior in PowerPoint (2016) I want to share with you. I wanted to create a toggle button that sets the zoom level of the presentation (in design mode) between 'zoom to fit' and zoomlevel = 80%, but I found out PowerPoint sometimes (and that are the worst bugs) returns an error if I set the zoomlevel to 80% with VBA:

        ActiveWindow.View.ZoomToFit = False
        ActiveWindow.View.Zoom = 80

The error is the non-descriptive 'View (unknown member) : Failed.', err.number: -2147467259 

It happens suddenly when you select another slide and run the code. I have no clue why the error raises but I found out that if you change the ViewType to ppViewSlideSorter and back to ppViewNormal the error does not occur anymore:

        ActiveWindow.View.ZoomToFit = False
        ActiveWindow.ViewType = ppViewSlideSorter
        ActiveWindow.ViewType = ppViewNormal

        ActiveWindow.View.Zoom = 80

ugly, but functional

Cheers

Martin

john sr wilson say:

This sometimes repros here too so it looks like a bug.

This seems to work and for the user is visually less intrusive I think.

Sub To_80()
Dim osld As Slide
Dim oshp As Shape
On Error Resume Next
Set osld = ActiveWindow.View.Slide
Set oshp = osld.Shapes.AddShape(msoShape5pointStar, 1, 1, 1, 1)
ActiveWindow.View.ZoomToFit = False
ActiveWindow.View.Zoom = 80
oshp.Delete
End Sub

There does have to be a slide selected with this though. Your original would solve this but you could combine the two only switching view if there was an error maybe.
