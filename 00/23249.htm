Access Query that uses a variable in its WHERE clause

SELECT tblArtifacts.ArtifactNumber, tblArtifacts.ArtifactName, tblArtifacts.ArtifactStatus, tblArtifactUses.UserArtifactNumber
FROM tblArtifacts LEFT JOIN tblArtifactUses ON tblArtifacts.ArtifactNumber = tblArtifactUses.ArtifactNumber
WHERE tblArtifacts.ArtifactNumber = 10;

What I would like to do is make the WHERE clause at the end variable so that when I display this query in subform, I can set the value of the what tblArtifacts.ArtifactNumber should equal when the form is first loaded. Right now, it is hard-coded as 10 just to test that the query works, but I would like to make the 10 a variable. Something like this:

SELECT tblArtifacts.ArtifactNumber, tblArtifacts.ArtifactName, tblArtifacts.ArtifactStatus, tblArtifactUses.UserArtifactNumber
FROM tblArtifacts LEFT JOIN tblArtifactUses ON tblArtifacts.ArtifactNumber = tblArtifactUses.ArtifactNumber
WHERE tblArtifacts.ArtifactNumber = desiredArtifactNumber;

I'll be opening the form via VBA, so I have all of those tools at my disposal. How do I go about setting up the query to accept a variable (or its equivalent), and how do I pass that to the query when I load the form that will be displaying it as a subform?

Thanks,

-Chris

hansv mvp say:

Create a form (or use an existing one), and create a text box txtArtifactNumber on it. Set its Format property to General Number.

Let's say the form is named frmMyForm.

Change the WHERE-clause to

WHERE tblArtifacts.ArtifactNumber = Nz(Forms!frmMyForm!txtArtifactNumber,0)

Open the form/subform from frmMyForm. You can check whether txtArtifactNumber has been filled in when you do so.

ken sheridan say: As the query is a subform's RecordSource you can omit the WHERE clause completely, and, in the parent form, include an unbound text box or combo box in which the artefact number can be selected from a list.  Set the subform control's LinkMasterFields property to the name of the unbound control in the parent form, and the LinkChildFields property to ArtifactNumber.  You should also remove the redundant tblArtifactUses.UserArtifactNumber column from the query's SELECT clause to avoid an ambiguous reference.

If the subform is already linked on another column then the above two values should be added to the LinkMasterFields and LinkChildFields properties, separated in each case from the existing value by a semi-colon.

When a value is entered/selected in the unbound control in the parent form, the subform's recordset will be reloaded to include only the matching rows.

gord dibben say:

If you think you may be interested in VBA to achieve uppercase. . . . .

Place the event code in the worksheet.

Private Sub Worksheet_Change(ByVal Target As Excel.Range)
    If Target.Column > 1 Then Exit Sub
    On Error GoTo ErrHandler
    Application.EnableEvents = False
    Target.Formula = UCase(Target.Formula)
ErrHandler:
Application.EnableEvents = True
End Sub

fprefect say: Resolved it!

1. Go to the 'Windows Phone' App in Facebook - enter it in the search box. Note - Go to the app, not the group.
2. Refresh several times - Ctrl+F5
3. Go to Facebook Home
4. Windows Phone should be there under 'Apps'
5. Click on the pencil icon next to it and select  'Edit Settings'
6. Set up your wall post viewing rights the way you want them.
7. Save the settings

It should work now - at least it did so in my case.
