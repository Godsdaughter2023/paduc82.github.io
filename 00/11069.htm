Only by selecting the 'between' option and entering the start and end dates for the range.  You can also select the Advanced Filter/Sort item from the ribbon and build a 'filter'.  However, the real solution is to build your own interface for filtering the results.  You could do this either by means of a separate dialogue form, and use an option group in the dialogue form for "this quarter", "this month", etc.  and include code in the Click event procedure of a button to open a  form or report based on the query filtered to the range in question by means of the WhereCondition argument of the OpenForm or OpenReport method, or you could include the option group in the header of a form bound to the query and put code in its AfterUpdate event procedure to filter the form.

It's important to recognize the fact that MS Access is a development environment, not an out-of-the-box solution.  If you want to go beyond the limited functionality of the built in interface, or what's available as pre-developed  templates, then you have to undertake the development to achieve your objectives.

Hi,

The ISNUMBER formula coming back as TRUE is a good thing, nothing more to do there but =E10=O2 coming back as FALSE is the problem. O2 is a poor choice of cells picked up from your first post and the O is the letter 0 NOT zero.

What the formula does is SUM row 11 where the corresponding cell in row 10 is the same as O2 and from your first post it looked as if you had ST in that cell. Note that it doesn't need to be O2 it can be any cell but somewhere we need a cell that contains the text value ST.

We could do the formula like this but it's much easier to maintain if you use a cell reference.

=SUMIFS(E11:P11,$E$10:$P$10,"ST")

Here the lookup value ST is built into the formula
