printer says there is no connection to my computer, is it MSE? - Microsoft Community

suzyqbear say:

printer says there is no connection to my computer, is it MSE?

My printer says there is no connection to my computer but everything is hooked up correctly and the settings are correct and one screen tells me that the printer is working. What would cause this to happen, could itt be MSE?

Printer problems since I installed Windows 7 or maybe when I downloaded MSE. Any advice?

kosh vorlon say:

Hi suzyqbear,

 

I'm more inclined to suspect the printer and/or network is not properly setup in Windows 7 and/or there's an issue with the printer drivers which may need to be updated (in which case, see the last paragraph of this post for the link you need), but MSE can possibly be the cause so since you're here, let's address that.

 

I don't believe it's an issue of file sharing but it could be certain printer files being blocked by MSE - I've seen this before and what's suggested below worked to solve that problem (though I don't have enough information - and even if I did - to give you any type of guarantee but it's worth a shot).

 

If you're certain MSE is the cause, then we can exclude the files and/or processes needed by the printer but being blocked from scans. If that works, then it was MSE and the exclusions will solve the problem (but open a tiny hole in your security regarding the exclusion, so please don't get into a habit of excluding things).

 

To do it, open MSE and go to the Settings tab and click on excluded files/folder and exclude the files by their full path. Then click on excluded processes and exclude the files there as well. That should do the trick. If not, either some other file or process is being blocked by MSE or MSE is not to blame. Go to the History tab and try to see what it detected. If it shows nothing, then chances are it may not be MSE but something else (the printer setup, printer drivers, or many other things) stopping the process. How do you know it is MSE doing this? Do you get a message identified as coming from MSE (or from the printer) saying the action was blocked or some other way? What exactly happens when you press the button to print - everything that happens or doesn't happen in the order it occurs? Does it identify the specific files being blocked?

 

To check this, close anything using the internet (email and anything else) except the one browser window with the one tab where you request printing of the coupons to minimize risks while you are unprotected. Then go to the Settings Tab and click on Real-time protection and uncheck the box at the top to turn it off, save the changes and MSE should go from a green border to a red border and the Home Tab should indicate you are not protected and there should be a big Start button. Now with MSE disabled, try printing. If it works, then MSE is likely the cause and we need to find something else or something different to exclude. If it still doesn't work, then MSE is not the cause. In either case, check the box for real-time protection and save the changes and if necessary hit the start button on the home page of MSE to get protection up and running again and everything back to green.

 

If MSE is proven not to be the cause, then remove the exclusions you added earlier and we may need to refer you elsewhere to seek out the problem. If MSE is the problem, then open Task Manager and go to processes and see if you can spot the process being used when you try to print in case there is a different file than the program file or those reported by the troubleshooter. If not, then go to Device Manager and exclude the Printer's device driver files in MSE (assuming you haven't already done so).

 

If this doesn't help, then please post back with results of what you tried and answers to any questions asked, including the version of Windows you are using, the manufacturer, make and model of the printer, how the printer is connected (by LPT cable, USB cable, wireless, through a network to a stand-alone device or to a device connected to another computer, or what).   But truthfully, if the above ruled out MSE, then it would be better to repost with that information to the Hardware section of the Windows Forum for your version of Windows http://answers.microsoft.com/en-us/windows/forum/windows_7-hardware?page=1&tab=all&tm=1300700638376 or to go to technical support for the printer you are using.

 

I hope this helps.

 

Thanks and good luck!

Formula in cell H4:

=(H3>EOMONTH(E4,-1))*(H3<=EOMONTH(F4,0))*G4

This will consider the start Month / Year (wef 01 May 18) in cell E4 and the End Month / Year in F4 (till 31 May 2020). If H3 date is within these, as above, the formula returns G4, else the formula returns zero.

Cells E4 / F4 / H3 are valid dates, though you may format them to appear as May-18 or any other.

ken sheridan say: It was explained to you in this thread in the MSDN Access forum that to do what you are attempting would involve adding multiple subforms to the parent form, giving each a separate RecordSource property which returns a single record.  This would mean that the number of records shown would be finite, and you would need to write a query which would return the first, second etc record from the subset of records in the referencing table which relate to the parent form's current record.

As you say there could be up to 5 referencing rows, adding 5 subforms to the parent form would presumably be adequate.  As regards retuning the separate records there are SQL techniques which can number rows distinctly, so you could then restrict each subform's query to where the computed number = 1, 2, 3 etc.

You'll find examples of row numbering queries in RowNumbering.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In my demo the qryNumberRowsByGroup_Join query numbers rows per customer as a non-updatable result table:

SELECT COUNT(*) AS RowNumber, T1.CustomerID, T1.TransactionDate, T1.TransactionAmount
FROM Transactions AS T1 INNER JOIN Transactions AS T2
ON (T2.TransactionID<=T1.TransactionID OR T2.TransactionDate<>T1.TransactionDate)
AND (T2.TransactionDate<=T1.TransactionDate) AND (T2.CustomerID=T1.CustomerID)
GROUP BY T1.CustomerID, T1.TransactionDate, T1.TransactionAmount, T1.TransactionID;

So, if I were to create a Customers parent form I could embed five subforms within it, each identical apart from their RecordSource property.  The RecordSource property for the first would be:

SELECT *
FROM qryNumberRowsByGroup_Join
WHERE RowNumber =1;

That for the second would be:

SELECT *
FROM qryNumberRowsByGroup_Join
WHERE RowNumber =3;

That for the third:

SELECT *
FROM qryNumberRowsByGroup_Join
WHERE RowNumber =3;

and so on.  Each subform would be linked to the parent form on CustomerID.

As I said, the above query returns a non-updatable result table (as a result of using the COUNT aggregation operator).  The demo also contains alternative queries which call the VBA DCount function to return an updatable result table, so if updatable subforms are needed then the following qryNumberRowsByGroup_Updatable query could be used rather than the qryNumberRowsByGroup_Join query:

SELECT DCOUNT("*","Transactions","CustomerID = " & CustomerID & "
AND TransactionDate  <=  #" & Format(TransactionDate,"yyyy-mm-dd") & "#
AND (TransactionID <= " & TransactionID & "
OR TransactionDate <> #" & Format(TransactionDate,"yyyy-mm-dd") & "#)") AS RowNumber,
CustomerID, TransactionDate, TransactionAmount
FROM Transactions
ORDER BY CustomerID, TransactionDate, TransactionID;

It should not be difficult for you to replicate one or the other of these queries with your own table.  In the above queries the rows are returned in transaction date order per customer, with the primary key TransactionID used as the tie-breaker in the event of two or more transactions by the same customer taking place on the same date.   If the order in which the rows would be returned per customer were unimportant, however, then the criteria could be simplified to:

SELECT DCOUNT("*","Transactions","CustomerID = " & CustomerID & "
AND  TransactionID <= " & TransactionID) AS RowNumber,
CustomerID, TransactionDate, TransactionAmount
FROM Transactions
ORDER BY CustomerID, TransactionID;

The same would be true of the JOIN criteria in the qryNumberRowsByGroup_Join query.  If the subforms are not to be updatable a JOIN is much more efficient than calling the DCount function.
