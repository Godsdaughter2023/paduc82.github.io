doug robbins - mvp office apps & services (word) say:
It is most likely that with a French Language installation, you will need to replace the commas in the TOC field with semi-colons, as in the WIndows Control Panel>Regional Settings, the decimal and thousands separators are different from those on the machine on which the document was created.

jay freedman say:

The problem is that the names of the built-in Heading styles in French Word don't use the word "Heading" -- that's only in English.

The workaround for this is to assign aliases (language-independent alternative names) to the style names. To do that, right-click the Heading 1 style in the Style gallery and click Modify; in the dialog, type a comma and an alias after the built-in name. It should look like this if you choose the alias H1:

Select the "New documents based on this template" option and click OK.

Repeat this for the Heading 6 style and for any other style that will be included in the TOC.

Now you can change the TOC field code to this:

{ TOC \h \z \t "H1,2,H6,1" }

and update the field.

jay freedman say:

Ask the people who still have the issue to open the document and then open the Styles pane (Ctrl+Alt+Shift+S, or click the launcher arrow in the bottom right corner of the Styles gallery). If they look at the Heading styles in the pane, do they see both the built-in names and the aliases? If the aliases are missing, something is wrong with the way the document is being transferred from one computer to another. Perhaps those people are opening a copy of the document that hasn't been modified? I've tested to confirm that normally the aliases are preserved in the document file and are present when opened on another computer.

The apparent issue of the setting going back to "Only in this document" is just the way Word works. When you select the "New documents" setting and click OK, that causes the modified style to be saved in the attached template as well as the open document. If you open the Modify dialog again, the "Only in this document" option is again selected, but the previous modifications are still in the template. The style in the template affects only future new documents based on the same template (http://shaunakelly.com/word/templates/templaterelations.html).

ACCESS 2010 - The message error to open an object (.mdb) - Microsoft Community

haniac say:

ACCESS 2010 - The message error to open an object (.mdb) is: I don't have the necessary permission to open it.

I'm converting my object, which was developed in Access 2003, to Access 2010. But I can't open it. In Access 2003 I used an .mdw for the workgroup administrator. It says that I don't have the necessary permission to open it. What I should do?? How to use an workgroup administrator in Access 2010? Thanks
albert d. kallal say:

There is a number of ways to do this.  If you open a accDB file, then you will NOT see the workgroup options. If you open a non secured mdb file, then access will display the workgroup options, including even the wizard to secure the database.

 

However, in your case you already have a secured database. So, the most simple approach here is to create a shortcut that tells access what workgroup to use.

 

The following shortcut works for me:

 

"C:\Program Files\Microsoft Office\Office14\MSACCESS.EXE"

"c:\RidesDev\SkiRides\RidesXP\RidesXP.mdb"

/wrkgrp "c:\RidesDev\SkiRides\RidesXP\Rides.mdw"

/User "RidesAdmin" /pwd "mypassword"

 

So, create a shortcut that sets the path to the mdb and also includes the workgroup file. The users and password above is NOT needed and if you leave it out then access will prompt you for the user and password during startup.

 

Note that all of the above shortcut is on one line (with a space between each). So, you can right click on your desktop, and paste in the above for the shortcut (change the file name to yours).

Hi Off2work. 2GB is the largest file size you can upload to a team site document library or to your personal OneDrive for Business.
