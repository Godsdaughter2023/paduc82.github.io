Is Insider build 10240 the same as the Global Release? *Answer Inside*

This question has been asked many times and I wanted to share some insight into the answer.

Short answer:

Yes!  They're the same.

Longer answer:

Insiders received 10240.  After receiving it, Microsoft was still fixing bugs, making small changes, etc.  Insiders received all of these fixes in the small daily updates over the week prior to our official release on 7/29.   So if you have installed each of the small updates from Windows Update (WU), then yes, you are on the same build and version as we officially released.

Restated ->  (Insider build 10240 + few small updates) = Same version of the OS released globally.

You don't need any different license keys, revised builds, and you don't need to clean-flash your PC to be on the same OS that was delivered.  You're already on it!

bill mosca say:

Phil

Your vendor query is all wrong. You are getting the counts for each type of vendor regardless of the year. That's why it comes up the same for each year. Nothing shows for 2014 because that year is not in the table.

Here is what you should be using as qryVendorTypeTotals:

SELECT DISTINCT SUM(IIf([VendorType]='Food',1,0)) AS Food
              , SUM(IIf([VendorType]='NP Food',1,0)) AS NPFood
              , SUM(IIf([VendorType]='Arts & Crafts',1,0)) AS ArtsCrafts
              , SUM(IIf([VendorType]='NP / Sponsor',1,0)) AS NPSponsor
              , SUM(IIf([VendorType]='Kids',1,0)) AS Kids
              , tblPaymentInfo.FestivalYear
FROM            tblVendorInfo
                INNER JOIN tblPaymentInfo
                ON              tblVendorInfo.VendorID = tblPaymentInfo.VendorID
GROUP BY        tblPaymentInfo.FestivalYear
HAVING          tblPaymentInfo.FestivalYear =[Forms]![frmReporting]![txtFestivalYear]
               
                ;

 

Using a GROUP BY query to add up each record if it matches a certain criteria is a very common way to break up types.

bill mosca say: I think I found it. qryBalanceDue2 excludes 0 balances. It returns nothing for 2013. Because it has no records, your query for the sub report returns no records. I took out the <>0 criteria in qryBalanceDue2 and the grand totals shows with its figures.
