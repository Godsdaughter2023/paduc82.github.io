jeeped say:

.. Is the above process the only way to inspect the XML tags or is there a faster way to do it?


As you know, a DOCX document is actually a .ZIP file that contains many internal components. There is an internal folder called word which will always contain a document.xml file. This file is the basis of the document's layout and content.

Assuming that the DOCX archive structure has not been corrupted, it can be opened in an archive utility. I use WinRARfor this with a simple right-click Open with... from Explorer so I don't have to rename the file extension to .ZIP. Once opened in WinRAR, you can drill down into the word folder and see the document.xml file. I use Notepad++ as a text editing tool and have the .xml file extension associated with this program so I can simply double-click document.xml in WinRAR to open an editing session. Notepad++ has cursor positioning in the right-hand side of the status bar, syntax highlighting/pairing and a GoTo facility that accepts a file offset.

I should mention that after making any change to word/document.xml you need to save it in Notepad++ then go to WinRAR and acknowledge that you want to update the file in the DOCX archive before using it for another purpose.

Hi,

I don't fully follow this but the formula below returns the second match, it's an ARRAY formula so see below for how to enter it.

The 2 at the end makes the formula return the second match so change it to a 3 for the 3rd etc.

=INDEX(Export!D1:D20000,LARGE((Export!A1:A20000=B7)*ROW(A1:A20000),COUNTIF(Export!A1:A20000,B7)+1-2))

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

john korchok say:

The simplest way to accomplish this is to unembed the spreadsheet and instead link it to the Word document.

  1. In Excel, select the spreadsheet area and copy it.
  2. In Word, choose Home>Clipboard>Paste, clicking on the down-pointing arrowhead on the bottom of the Paste button.
  3. Click on Paste Special.
  4. Change Paste to Paste link.
  5. Set As to Microsoft Excel Worksheet Object.
  6. Click on OK.

Then repeat the same steps for the chart in your spreadsheet., but set the Paste Special As to Microsoft Excel Chart Object.

Now you can double-click on the spreadsheet, edit it, and the chart will update both in Excel and in Word.

steve schapel say:

Telesphore,

Looks like you are trying to use the IIf() function, in which case note that it has two Is - IIf not If.

However, you would need to "nest" them, as follows:
=IIf([NameOfSubreport].[CumulativeMark]="A+";"Summa cum laude";IIf([NameOfSubreport].[CumulativeMark]="A";"Magna cum laude";IIf(... ...)))
But this function has a limit to the number of nested levels permitted - I think it's 7, so that may be a problem for you.

Simpler therefore may be the Switch function:
=Switch([NameOfSubreport].[CumulativeMark]="A+";"Summa cum laude";[NameOfSubreport].[CumulativeMark]="A";"Magna cum laude";...  ...)

Better still, though, would probably be to make a table (let's name it MarksDescription), with 2 fields (let's say they are called Mark and Description), where you enter all possible marks and their corresponding descriptions.  Then, add this table to the query that the subreport is based on, with the MarksDescription.Mark field joined to the CumulativeMark field in the existing query.  Then you can have a hidden control for the Description corresponding with the subreport's CumulativeMark, right on the subreport itself.  And then, the Control Source of the textbox on the main report can simply refer to that:
=[NameOfSubreport].[Description]
Hope you understand what I mean.

Steve Schapel, Microsoft Access MVP

steve schapel say:

Telesphore,

No, it is not confusing.  That is exactly what I was trying to ask you about.

A subreport is a control on the main report.  That subreport control has a name, as shown in its Nom property.  The subreport is used to display another report within the main report.  That "another report" also has a name.  The name of that other report is what you see in the Objet Source (Source Object) property of the subreport.  The name of the subreport control, and the name of the report displayed by the subreport control, may be the same, or maybe not.  In either case, you must use the name of the subreport, and not the name of the subreport's source report, in your calculated expressions.

Anyway, the main thing is that I am happy to know that this is working for you now.

Steve Schapel, Microsoft Access MVP
