brian tillman [outlook mvp 2007-2019] say:

Aliases are constructs of the mailbox - i.e., server-side entities.  Mail clients don't know anything about aliases.

However, if you create an alias for the mailbox (on the server), you can then set up a send-only account in Outlook 2013 that can specify the alias as the sender address.  For all mail servers I've used that are not Outlook.com/Hotmail, this works great.  The Outlook.com server, however, will automatically change any outgoing alias sending address to your primary alias, so it won't work with Outlook.com.  Even though the message Outlook 2013 sends will have the correct sender address, the recipient will see only the primary address as the sender.

ken sheridan say:
I 'am' using a db and not a spreadsheet. 

Yes, but what Karl meant by 'doing spreadsheet' is that you have designed a relational database table as if it were a worksheet in a spreadsheet application like Excel. 'Doing spreadsheet' is a colloquial term for what in the language of the database relational model is known as 'encoding data as column headings'.  A fundamental principle of the model is the Information Principle.  This requires that all data be stored as values at column positions in rows in tables, and in no other way.

The correct way to store data of this nature in a relational database table is to have a table with one column in which each value is the month in question, and another column a value such as 'Paid'.  As  a general rule a well designed database table should be tall and skinny, not short and fat.  Data can be output in the sort of format you are using by means of a crosstab query or by using multi-column subreports in a parent report.  You'll find a simple example of the latter type of solution as Payments.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

In this little demo file the column headings (dates per week) are supplied by one subreport, the values of the payments per customer by another subreport.
