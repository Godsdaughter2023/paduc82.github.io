hansv mvp say:

I'm not ScottGem but perhaps I can help.

On the Create tab of the ribbon, click Query Design.

Access will display a new query and pop up the Show Table dialog.

Select the Annual Learning Plan table and click Add.

Also select the Employees table and click Add.

Click Close to discard the Show Table dialog.

Access may automatically have joined the tables on Employee ID (i.e. drawn a line from Employee ID in one table to Employee ID in the other table).

If not, drag the Employee ID field from the Annual Learning Plan table to the Employees table and drop it on the Employee ID field there. This will create a line joining the tables.

In all cases, double-click the join line.

Select the option "Select ALL record from 'Annual Learning Plan' and only those records from 'Employees' where the joined fields are equal".

Click OK. The join line should now have an arrow pointing towards the Employees table.

Add the fields from the Annual Learning Plan table to the query grid below; you can also drag * from the field list to the query grid.

Add only the fields from the Employees table that you want to display on your form, such as Unit, Last Name etc.; do not include the Employee ID field from the Employees table.

Close and save the query.

Set the Record Source of your form to the newly created query.

You can now add the fields from the Employees table to the form. When an Employee ID is entered or selected, the other fields will be populated automatically.

Please read my response. While Hans' approach can be made to work its not the approach I would use. partially because of the issues you are experiencing. I would leave the Recordsource of your form as the table (though I would rename your objects so there are no spaces and shorten them). But I would remove any field int he Learning table that is also in the Employee table, except for EmployeeID. 

One of the principles of relational databases is that data should exist in ONE place only. So you do not want redundant data. For example, lets say an employee changes division. With your structure you would have to change their record in both the employee table AND the Learning table. If you just have that info in the Employee table, then it will always be current. 

From there you use the techniques illustrated in my blog to DISPLAY the related data.

Another option is to use a main form bound to your Employee table and a subform bound to the Learning table and linked on EmployeeID. Then the main form will display the employee info and the subform just the learning info.
