scottgem say:

You can also look at my blog on Sequential Numbering.

The reason why this isn't working is because you are not committing the record immediately after generating the number. Until you commit the record, the table hasn't been updated so the DMax can pull the same number. 

You can shorten your code to the following:

If IsNull(Me.counter) Then

    Me.Counter = Nz(DMax("[Counter]", "file_details", "GRP='" & [Forms]![umara]![GRP] & "'"),0) + 1

    Me.Dirty = False

End If

You don't need the requery or the GoTo Last. 

The Nz function returns a 0 if there are no matching records, The Me.Dirty = False commits the record.

Ken's demo, has additional code that further helps ensure that the counter isn't duplicated. If you have an active database where records are being created constantly, that may be more appropriate.

One last point. Autonumber is a specific datatype in Access which generates a unique ID. What you are doing is sequential numbering not a custom autonumber.

john korchok say:
If the pictures are in-line with text (the default), you cannot group them, since Word is treating them as text. Right-click on each picture and choose Size and Position. Click on the Text Wrapping tab and make each one In front of text (or Behind text, that works too). Now select by holding down the Shift key and clicking on each one, then Group.
