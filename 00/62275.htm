58cherrysunburst say:

Just a quick note. I don't think it's really critical to the Q&A here, but the example given for the default gateway is potentially misleading in this answer.

So, while the router's interface acting as the default gateway could be any IP address in the subnet, it does HAVE TO BE in the same subnet. So since this example is a 255.255.255.0 mask, then the default gateway would have to be 192.168.0.XXX where XXX is any valid host IP address NOT currently being used. Note that the word "valid host" in this case implies that it is NOT 0 and NOT 255.

Specifically, given this mask and the example IPs, the default gateway cannot have a third octet value of 1.

Hi everyone,

I have solved the problem with a quick fix. Hope this helps someone out...

In summary or TLDR; 

  • Do all the regular update driver stuff, set power settings stuff....
  • Use Regedit to go here HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\GraphicsDrivers
  • add/set TdrDelay to decimal 15
  • ...if crashes persist you should see more error messages in the event log now. If driver errors then try this:
  • add/set TdrLevel to decimal value 0
  • Check your logs again and clear out any other errors that appear

...that got me over the problem.

In detail:

The main problem here is the lack of notes in the event log. Even when using furmark and gpuz to monitor the results you'll get no warning or log of a black screen. I suspect the crash happens before the event is logged.

...and that's the key. TDR...TimeoutDetectionandRecovery. A cheesy fix used to stop a BSOD before it happens.

Step 1 Try this and see if you get an error in the event log

  • Go to search and type 'regedit'. when regedit.exe comes up right click it and select run as administrator.
  • Navigate to HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\GraphicsDrivers
  • find or add new (to add right click and select new then DWORD) REG_DWORD TdrDelay
  • Double-click to edit TdrDelay and set it to 15 decimal

Now TDR is going to wait 15 seconds before restarting the driver. That should give it enough time to log an error.

Step 2 See if you have a driver error, especially this one

  • Run your machine until it does the black screen thing.
  • When it black screens, wait 1 minute before the hardboot.
  • Go to search and type 'event viewer'. When the desktop app comes up right click it and select run as administrator.
  • Navigate to Windows Logs, then System
  • look for driver errors or warnings. If you see this little guy then you've got the same annoying problem as me
    • Display driver nvlddmkm stopped responding and has successfully recovered.
    • 4101 error that was not showing up before.
...ok. now that we have a smoking gun we can try a few things to kill it. Step 3 Nuke TDR

  • Go to search and type 'regedit'. when regedit.exe comes up right click it and select run as administrator.
  • Navigate to HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\GraphicsDrivers
  • find or add new (to add right click and select new then DWORD) REG_DWORD TdrLevel
  • Double-click to edit TdrLevel and set it to 0 decimal.

That disables TDR. Reboot and try to make it blackout. In my case the blackscreen stopped and hasn't been back for over a week.

Step 4 (advanced) Check in periodically and try some video card tuning.

  • Once a day check the logs and see if new errors show themselves
  • Check out OC_Guru, or GPUZ and get some video card monitoring going
  • Underclock as needed to get additional stability.
