Elevation has to do with permissions.  Is this happening for one user or all users?  One machine or all machines?  Can the users start Excel manually?  Can they open Excel by double-clicking on an Excel file?

You could try and use the same technique I created to deal with an Outlook issue, see: http://www.devhut.net/2014/10/31/createobjectoutlook-application-does-not-work-now-what/ 

I ran a quick test, you still need the 2 other functions from the link, but this modified version to launch Excel works fine, give it a try, hopefully it will solve your current headache.

'---------------------------------------------------------------------------------------
' Procedure : StartExcel
' Author    : Daniel Pineault, CARDA Consultants Inc.
' Website   : http://www.cardaconsultants.com
' Purpose   : Demonstration of how one can start Excel if it isn't already started
'               and bind to the instance to perform automation with it
' Copyright : The following is release as Attribution-ShareAlike 4.0 International
'             (CC BY-SA 4.0) - https://creativecommons.org/licenses/by-sa/4.0/
' Req'd Refs: None required, Req GetShortPathName Declaration
'
' Usage:
' ~~~~~~
'
'
' Revision History:
' Rev       Date(yyyy/mm/dd)        Description
' **************************************************************************************
' 1         2018-05-21              Initial Release to help in a forum question
'---------------------------------------------------------------------------------------
Function StartExcel()
    On Error GoTo Error_Handler
    Dim oExcel                As Object
    Dim sAPPPath              As String

    If IsAppRunning("Excel.Application") = True Then    'Outlook was already running
        Set oExcel = GetObject(, "Excel.Application")    'Bind to existing instance of Outlook
    Else    'Could not get instance of Outlook, so create a new one
        sAPPPath = GetAppExePath("excel.exe")    'determine outlook's installation path
        Shell (sAPPPath)    'start outlook
        Do While Not IsAppRunning("Excel.Application")
            DoEvents
        Loop
        Set oExcel = GetObject(, "Excel.Application")    'Bind to existing instance of Outlook
    End If

    oExcel.Visible = True

Error_Handler_Exit:
    On Error Resume Next
    Set oExcel = Nothing
    Exit Function

Error_Handler:
    MsgBox "The following error has occured" & vbCrLf & vbCrLf & _
           "Error Number: " & Err.Number & vbCrLf & _
           "Error Source: StartExcel" & vbCrLf & _
           "Error Description: " & Err.Description _
           , vbOKOnly + vbCritical, "An Error has Occured!"
    Resume Error_Handler_Exit
End Function
