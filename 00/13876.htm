Dealing with #N/A Error in an IF statement that uses Index and - Microsoft Community

linus2012 say:

Dealing with #N/A Error in an IF statement that uses Index and Match

I have a formula that's an IF statement that uses Index / Match of Member ID to compare the logical test.  If the test is True, then it would use Index / Match to look for an Average number in an array and multiplies the Average by a Factor from another array. If the test is False, then it just uses Average number from the array.  See below for the formula - applied bold, italic for easier reference.

 

=IF($B2=INDEX(Member_ID,MATCH($B2,Member_ID,0)),INDEX(Factor,MATCH($B2,Member_ID,0))*INDEX(Average_Num,MATCH($I2&$K2,PlaceofService&Condition,0)),INDEX(Average_Num,MATCH($I2&$K2,PlaceofService&Condition,0)))

 

This formula works (peforms the True part of IF statement) if the initial Member ID during the logical test is in the array.  However, if the Member ID is not in the array, I get an #N/A error rather then performing the False part of the IF statement.  Is there an easy way to address this?  I wanted to use an IFERROR or IFNA but neither of these produce the result that I wanted.  I understand why I'm getting the #N/A error but need a way to address it.  Any assistance will be helpful.  Thanks.

 

 

 
paulcordts say: Use the ISERROR function.  You also need to reverse your conditions

=IF(ISERROR($B2=INDEX(Member_ID,MATCH($B2,Member_ID,0))),INDEX(Average_Num,
MATCH($I2&$K2,PlaceofService&Condition,0)),INDEX(Factor,MATCH($B2,Member_ID,0))
*INDEX(Average_Num,MATCH($I2&$K2,PlaceofService&Condition,0)))

This becomes your logical test
ISERROR($B2=INDEX(Member_ID,MATCH($B2,Member_ID,0)))
This becomes your value if true (The member ID is not found)
INDEX(Average_Num,MATCH($I2&$K2,PlaceofService&Condition,0))
and this your value if false (The member ID is found)
INDEX(Factor,MATCH($B2,Member_ID,0))*INDEX(Average_Num,MATCH($I2&$K2,PlaceofService&Condition,0))

Hi Sarsha:

Bingo!  If you continue to run software while the updater is trying to update it, you "SHOULD" get a message saying "Please close the following applications and try again..."

But if you don't, then you can get a "mixed salad".

Recovering from this can be tricky.

First, download the following two files:

Microsoft Office for Mac 2011 Service Pack 3 (14.3.0)

Microsoft Office for Mac 2011 14.5.3 Update

Then re-start your computer (we need to ensure that NOTHING else is running, this time).

First run the first file, the service pack.  Hopefully, it should tell you that it has already been installed.  If it does, that's good.  Go on to the next file.  If it doesn't, let it complete installing, then re-start again.

Then run the second file (the 14.5.3) update.

That should install quite happily, and by then, you will have a complete matching set of components and you should have solved your problem.

If not, then we're staring a complete remove and replace in the face, and you're going to have to find a disk (and product key) from somewhere...

Unless Jim has a better idea...
