By any chance have you tried copying your data into a new workbook and then sorting in it (after closing the original)?  Maybe something corrupt in the workbook?

I set up a test sheet filling the same range you have (A6:P558096) with random values from 55-10000 and random single alpha characters form A-Z in column A.

Then I sorted using one numeric column, the alpha column (reversed) and another numeric column.  Bad news for you - it sorted extremely quickly - just a couple of seconds.  No errors.

System is i7 3.33GHz, 12GB RAM, Windows 7 Ultimate 64-bit, Office 2010 Pro 32-bit version.

Here's the code I used to fill the random data (took much longer to fill with data than it did to sort it)

Sub FillTestRange()
  Const firstRow = 6
  Const lastRow = 558096
  Const firstCol = 1 ' A
  Const lastCol = 16 ' P
  Dim rowPtr As Long
  Dim colPtr As Integer
 
  Sheet1.Select
  Application.ScreenUpdating = False
  For rowPtr = firstRow To lastRow
    For colPtr = firstCol To lastCol
      If colPtr = 1 Then
        Cells(rowPtr, colPtr) = Chr(Int((90 - 65 + 1) * Rnd + 65))
      Else
        Cells(rowPtr, colPtr) = Int((10000 - 55 + 1) * Rnd + 55)
      End If
    Next
  Next
  MsgBox "Done at last!"
End Sub
