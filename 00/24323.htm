In your original code you have several sections like this:

    startrow = Selection.Row
    Selection.End(xlDown).Select
    endrow = Selection.Row

The 'problem' at that point is that Selection is several thousand rows.  startrow=Selection.Row works fine, it returns the first row number of the selection.  The problem was in getting the endrow value.  Excel starts looking at the last row of the selection and then down to find the end.  That was taking you all the way down to the last possible row on the sheet which created the size match problem during the attempted .PasteSpecial action.

I changed all of those sections to look like this (I simply commented out the problem lines just to keep them for historical purposes):

    startrow = Selection.Row
    'Selection.End(xlDown).Select ' at this point all under Selection are empty rows!
    'so let's look from A6 down to find the real end?
    endrow = Range("A6").End(xlDown).Row

    'if empties in the middle are to be ignored
    ' endrow = Range("A" & Rows.Count).End(xlUp).Row
    If endrow = Rows.Count Then
      'all were empty below A6, so limit it to row 6
      endrow = 6
    End If
    'endrow = Selection.Row

    startrow = Selection.Row
