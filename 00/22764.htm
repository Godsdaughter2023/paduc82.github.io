tom ogilvy say:

Possibly this:

Sub MakeXLSX()
Dim sPath As String, sName As String
Dim wkbk As Workbook, sName1 As String
sPath = "C:\Data\Test\"     '<==  change to your path
On Error GoTo Errhandler
If Right(sPath, 1) <> "\" Then sPath = sPath & "\"
sName = Dir(sPath & "*.xlsm")
Application.EnableEvents = False
Do While sName <> ""
  sName1 = Replace(LCase(sName), ".xlsm", "")
  Application.DisplayAlerts = False
  Set wkbk = Workbooks.Open(sPath & sName)
  wkbk.SaveAs Filename:=sPath & sName1 & ".xlsx", FileFormat:=xlOpenXMLWorkbook
  wkbk.Close SaveChanges:=False
  Application.DisplayAlerts = True
  sName = Dir()
Loop
Kill sPath & "*.xlsm"
Errhandler:
Application.EnableEvents = True
End Sub

You don't  have to change the base name because the different extension will allow both files to exist.

Note that the Kill statement will delete all the workbooks left behind with an .xlsm extension (the original workbooks).  If you don't want that to happen, comment out that line.    

Depending on your security settings, you may have to answer a prompt to enable macros for each file as it is opened. 

It would be a good idea to copy a few files to a dummy directory and test the macro on these copies before you commit to processing your original directory

charles kenyon say:
Yes. You need to set the TOC options.

If you look at your References tab you will see there are buttons for a Table of Contents and a Table of Figures. The Table of Figures, though, looks for Figure captions, not Heading 4.

When inserting your first Table of Contents, just go ahead an insert a TOC. The default is to pick up headings 1-3.

For your Table of Figures, put your insertion point outside your first TOC and use the Table of Contents button again but this time click on the choice to "Insert Table of Contents..." This brings up the dialog. Tell it you want headings 1-4. Then click on the Options button.
Clear the level settings for everything except Heading 4 and set the TOC level of Heading 4 to 1 or 2.

OK your way out. When asked if you want to replace the existing TOC, say NO.

    macropod say:

    First, get rid of the \* MERGEFORMAT switches; they can mess with the formatting of your output, especially when its length varies. You can use Find/Replace to delete them.

    Second, if you want to ensure consistency across the documents, make sure they're using exactly the same Style definitions and that those Styles are used consistently.

    The only reason for any content to be 'dropped' in the destination document is that, contrary to your expectations, it's not within the source document bookmarks you're referencing. I can't comment on your macros as you haven't posted them.

    PS: You're not using 'merge codes'; you're using INCLUDETEXT fields.
