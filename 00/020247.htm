It appears that you have tried about everything there is to fix your problem.

For any antivirus that you have uninstalled, you should also obtain the

removal tool from its Web site, and run it too. Program remnants will

cause problems with Windows Defender as well.

I did not see whether you have examined the Registry for the presence of a

"Policy" that might have Windows Defender stymied though.

If not you should go into the Registry and navigate the following branch.

Hkey_Local_Machine\Software\Policies\Microsoft and select the folder,

Windows Defender

The only key that should be present is "Default".

Any other key present is one that has been set by Malware, a Third Party

Antivirus, or you.  You will most likely find

DisableAntivirus. Both of which you can disable by changing their DWord

value to 0.  If you find any other besides "Default", that you have not set,

consider deleting the complete folder. As a precaution, you should export

it first, to your desktop.     The folder will reconstruct itself.

Unless they have changed with v1903, the services you will be interested

in are "Security Center" and "Windows Defender Antivirus". All the

rest of the Windows Defender services as well.

CD\     <enter>

rohn007 say:
I haven't seen a defined best practice for it. I can throw out some random thoughts on it.

Office 365 is a "click to run" installation. That means that Office is not installed in the traditional meaning. Instead you install the CTR Virtual computer environment, which downloads the parts of the programs you use from the internet.  The advantage of this approach is that it totally isolates the program code from everything else on your computer.

The only point of contention between the 2 versions is in Windows itself. Which version do you want Windows to consider the default when you double click on a file in Explorer. CTR can be made default, but it does not follow the old pattern.

Since Office 2007, 2010 and 2013 are part of an evolutionary (de-volutionary?) chain, there is very little learning curve going from one to the other, other than the really bland user interface. 

One advantage of installing 2013 along side the previous versions is that people can fall back to the "comfortable" version if they encounter problems.  If you read through the forum, since 365/2013 was released Feb 1 there have been a lot of problems installing and keeping it working.
