To stop the timer event from being called, you need to use:

   Me.TimerInterval = 0

Then, to start the timer again use:

   Me.TimerInterval = 1000   'one second is 1000 milliseconds

If I understand what you are trying to do, I think your pause button's code might look like:

   If sTimerOn Then

      Me.TimerInterval = 0
   Else

     Me.TimerInterval = 1000

michael bauer (vboffice.net) say:

Outlook doesn't share categories, neither the names nor the colors.

 

When a category is assigned to an item, Outlook reads that category name and looks for it in a master category list and if found, it shows the correspondent color. Which master list is searched depends on where the item is stored: If it's your mailbox or a public folder, your master list is used. If it's stored in a shared mailbox, that mailbox' master list is used.

 

In order to see the master list of a shared mailbox, you need appropriate permissions (I always forget which ones...). Try it with full permissions, if I remember correctly, SentAs permission is needed.

michael bauer (vboffice.net) say:

Back to what colors?

 

Let's repeat it, please: The category name is stored with the appointment item that you see in the calendar. The category color is stored in a different place. If the user has the necessary permissions to get the color from your mailbox, then it shows that one. If the permission is missing, Outlook defaults to the color stored in the user's own mailbox.

 

Category Manager just gets category names and colors from a central file and stores them in each user's mailbox. This way you can ensure that each user is using the same category list. When that's given, Outlook should always display the same color for a certain category name without any permissions: Due to the missing permission Outlook should get the color from the user's mailbox, not yours. And as the user has same category list that you have, he/she should also see the same color.

 

I recommend to try it with just one category as mentioned before until you really know that everything's working the way you need it. Don't forget to test a restart of Outlook.
