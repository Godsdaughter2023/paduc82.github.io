To specify these settings, follow these steps:
Start Word, and then open the document you want to change.
In Microsoft Office Word 2003 and in earlier versions of Word, click the page or the section that you want to print, and then click Page Setup on the File menu.

In Microsoft Office Word 2007 and Word 2010, click the page or the section that you want to print, and then click the Page Layout tab. In the Page Setup group, click Page Setup to open the Page Setup dialog box.
Click the Paper tab.
Under Paper size, click the paper size that you want.
In the Paper source area, perform one of the following steps:
If the printer can automatically select the correct paper tray to use, click Default tray in the First page list, and then click Default tray in the
Other pages list.

Default tray is the default setting in both of these lists.
If the printer cannot automatically select the correct paper tray to use, you may have to select the paper tray that contains the paper size that you selected in step 4. To do this, click the appropriate paper tray in the First page list and in the Other pages list.
If you want to change the document's page orientation or its page margins, click the Margins tab.
Select the options that you want.
Perform one of the following steps in the Apply to box:
To use the paper size that you specified in step 4 for only the current section of the document, click Selected text.
To use the paper size that you specified in step 4 for the current section of the document and for all remaining sections in the document, click This point forward.
To use the paper size that you specified in step 4 for the whole document, click Whole document.
Click OK.
Repeat steps 2 to 9 for each section of your document for which you want to specify settings.

ken sheridan say: Because you are using the LIKE operator in the query this will not return rows with a Null at the column position because comparing Null with anything always evaluates to NULL neither TRUE nor FALSE.  You need to test for OR <parameter> IS NULL.  In query design view, if you wish to return rows where the value entered into the control as the parameter is a substring within the column in the table, you'd enter the following (as a single line) in the criteria row of the column in query design view:

LIKE "*" & [Forms]![frmSetTextFilter]![txtDescriptionShort] & "*" OR [Forms]![frmSetTextFilter]![txtDescriptionShort] IS NULL

If you want to return rows where the value at the column position exactly matches that entered into the text box as the parameter, use the following:

[Forms]![frmSetTextFilter]![txtDescriptionShort] OR [Forms]![frmSetTextFilter]![txtDescriptionShort] IS NULL

The way it works is, if there is a value entered as the parameter the first part of the OR operation will evaluate to TRUE for those rows where there is a match, so only those rows will be returned; if there is no entered as the parameter, i.e. it is NULL,  the second part of the OR operation will evaluate to TRUE for all rows regardless of the value, or lack of value, at the column position, so all rows will be returned.

One thing you'll notice is that if you save the query and then reopen it in design view Access will have moved things around.  Don't worry, the underlying logic will be the same.  Where multiple parameters are treated in this way, however, there is the possibility that the query might become too complex to open.  Even if not the logic will be obscured when you look at it in design view, and amendments will be more difficult.  With multiple premasters it's best to write and save the query in SQL view.

When you use parameters in the report's query like this you simply need to open the report from the form either by means of a macro or by means of code.  The button wizard can easily set this up for you.
