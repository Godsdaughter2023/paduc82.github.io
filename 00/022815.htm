suzanne s. barnhill say:

Although your numbered styles have a numbering level, they have not been assigned an outline level other than Body Text. In order to be included in a TOC, styles must be either included explicitly (through the Table of Contents Options dialog) or by having the outline level of the style set to something other than Body Text. You can modify AG Article 1 to have an outline level of Level 1 as follows:

  1. Display the Styles pane.
  2. Right-click on the AG Article 1 style and choose Modify...
  3. In the Modify Style dialog, click Format and choose Paragraph.
  4. In the Paragraph dialog, set the outline level to Level 1.
  5. Click OK twice to exit.

Since you have referred to 2010 and the Quick Style list, I'm assuming you're using Word 2010 and have modified your "applies to" data accordingly.

pamcaswell say: In addition to Suzanne's advice, if a built-in TOC format is used, to see the headings, they must have paragraph outline levels 1 to 3. 

suzanne s. barnhill say:
Unless, of course, you select more than three levels to show. The "automatic" tables don't give you this option without editing the TOC field. In order to have more control over the TOC (not just the number of levels but numerous other formatting options), you must use References | Table of Contents | Insert Table of Contents to get the dialog.

butch pedorcik say:

Okay, I assume that your pivot table is in the same Excel file that your source data resides.  One way that you can accomplish what you want is to create your pivot table in a separate Excel file and link to your source data where it currently is.  You would then have some additional freedoms in a connection string that you don't have when the pivot table is in the same workbook.

 

This is how you can do that.  In a new Excel document:

  1. From the Data tab, select the From Other Sources button and pick From Data Connection Wizard
  2. Select ODBC DSN and click Next
  3. Select Excel Files and click Next
  4. Now navigate to your source file and click OK
  5. My source file has one sheet of source data so I simply hit Next at the Select Datbase and Table window
  6. Again I left default data on the Save Data Connection File and Finish window and clicked Finish
  7. On the Import Data window select PivotTable Report and click OK

The resulting Pivot Table created by the wizard in my case has a Command Text of:

 

SELECT * FROM `C:\DOCUMENTS AND SETTINGS\BPEDORCIK.MYDOMAIN\Desktop\Source Data.xlsx`.`Sheet1$MyData`

 

(The Command Text can be found by selecting the Options tab of the Pivot Table Tools and selecting the Change Data Source drop-down.  Then select Connection Properties and select the Definition tab in the Properties window.) 

 

I modified mine to be:

 

SELECT Master Name, Master Name as Vendor, add any other fields you want included
FROM `C:\DOCUMENTS AND SETTINGS\BPEDORCIK.MYDOMAIN\Desktop\Source Data.xlsx`.`Sheet1$`

 

You'll notice the change of discretely selecting individual fields instead of * thus allowing a dynamically created field.  I also removed the MyData portion to have the pivot table look at all of Sheet1 in the event that I add any records that may end up outside of the MyData range.

 

If you have a problem at step 5 or 6 like data source contains no visible tables, you will just need to define a name for some range of data in your source file.  This can be done in your source data by right-clicking and selecting Define Name...  In my example I defined a range named MyData that I eventually removed from my command text to expand to the full sheet.
