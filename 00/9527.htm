asifmohammadullah say:

Does Excel VLOOKUP work without the 'range_lookup' parameter?

ossiemac say:

Yes it works without the Range Lookup.

However, if you insert the comma as per your example #1 then it defaults to False.

Without the comma as per your example #2 then it defaults to True.

Reason for False if comma is used with no value after it is my speculation. It looks after the comma for a value and does not find a value and therefore it becomes zero which is False. If the comma is not used then it automatically defaults to True.

See the following link for more info. (Google will provide you with lots of hits but I have not found any documentation for the use of the comma; I just know that is what occurs.)

https://support.microsoft.com/en-nz/help/181213/how-to-use-vlookup-or-hlookup-to-find-an-exact-match

asifmohammadullah say:

Thanks OssieMac...

I got your point, but isn't it a 'bug' that has revealed to be beneficial? Or the formula itself supports this syntax format?

ossiemac say:

Thanks OssieMac...

I got your point, but isn't it a 'bug' that has revealed to be beneficial? Or the formula itself supports this syntax format?

I don't see it as a bug. Certainly not one to worry about. It is just how code works. If you enter a formula in a cell like the following and the referenced cell (A2) is blank then it also returns zero.

    =A2

 If you use the comma in your Vlookup formula and don't insert a value then it is blank and returns zero.

Personally I avoid defaults like the plague. I was taught this back in the mid 1970's and have always adhered to it because one does not finish up with indeterminate results. How hard is it to enter the True of False and ensure that you get the returned result that you are expecting?

Hi Gary,

If you are using work resources you can change the assignment contour in the Task Usage view.  Changing the assignment contour (double click on the assignment row) will change the cost distribution.
