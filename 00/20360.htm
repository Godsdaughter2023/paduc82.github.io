john w. vinson say:

It's purely a guess, but try changing the name of the fieldname [Month (only)]. It's possible that Access is misinterpreting the parentheses as a function call: Month(datefield) is a function which returns a month number.

I'll usually avoid using ANY punctuation other than underscores in fieldnames; parentheses, operators such as - or + or *, even blanks can cause problems. Users need never see fieldnames in any case - table datasheets should be kept "under the hood", and on a Form you can have a label displaying anything you like instead of the fieldname.

john w. vinson say:

I'm not sure I can dig through all this mess and find the cause of the error message, but...

it appear that you're using fieldnames for formatting report purposes: vertical bars, parentheses in the fieldname, reserved words like Sum and Month. I would STRONGLY recommend that you not do so!

The other issue is that you're creating an alias Month_only and then reusing that alias in a further expression (the IIF). Access often will not allow you to reuse newly defined names; you need to recapitulate the expression rather than naming it here, and using that new name over there.

Is the Month field a Text field containing "201501", or is it a Date/Time field? If the latter, then the Right() function won't have the desired result. The Format of a date doesn't change the fact that it's stored as a Double Float number, but only the appearance.
