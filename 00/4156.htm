File specific actions for improving Excel performance - Microsoft Community

erik davidsson (msft) say:

File specific actions for improving Excel performance

Description:
This article goes through some actions that can be taken with any typical Excel file, and are more or less guaranteed to speed it up. For a shorter version of the below, please refer to this article. The below are ordered based on their relevance, with the most powerful and simple ideas first.



1. XLSB format
If your file size mainly comes from a big amount of formulas and text in cells (and not pictures etc.), this format can compress your file heavily. With such a text based file, saving it in the .XLSB format could result in a file size of less than 50%, opening/saving time less than 50%, and heavily reduced memory usage during opening and handling. This comes with almost no tradeoff and thus, is why it is put at the top of this list.

To try this, just click File – Save as, and choose the Excel Binary .XLSB file format instead. Then compare opening time and file size with the original file.



2. Excessive amounts of Shapes
A simple way to see what objects exist inside a sheet is to go to the Home tab, and to the furthest right you have “Find & Select”, and “Selection Pane”:

Then, you can click on the first sheet, hold down the Ctrl key, and press PgDn or PgUp repeatedly to quickly scroll through all the sheets.

(It is also possible, but not that likely, that you have Hidden (or even Very Hidden) sheets with excessive Shapes. Pressing Alt+F11 and pressing F4 will reveal what the “Visible” property is set to, for the sheet that you click on in the dropdown list to the left)

If you see a sheet with several hundred shapes or a lot more shapes than you would expect, it makes sense to reduce the number of shapes to check if it makes any difference for the file performance. To delete all shapes, clicking “Go To Special…” in the screenshot below, choosing “Objects”, and pressing Delete on the keyboard:


(If there are so many shapes to be found in the Selection Pane that you cannot select and delete them, one possible workaround is always to first close Excel, then rename the file from .XLSX to .ZIP in Windows, open the .ZIP file, open the subfolder XL, open the folder Drawings, and delete the big files in there. This will cause Excel to repair the file. Please note that this will also delete other shapes, and may have the undesired side effects of data loss as you may lose valid objects too, or other content, so use it with caution.)

3. Remove Styles that addins have added, that you don’t need
If you click the “Home” tab and go to the Styles group, you can click the small button “More”.

The default for new Excel files is just 47 styles. Therefore, if you in this dropdown can see a big mess with what looks like a huge amount of duplicates, probably some add-in must have added them. This can cause performance issues. It could be a good idea to clean them up in your file, resetting them to only use the Default Ones, plus the ones that you actually are using at the moment.

To do this, download this addin. This will add a button called “Remove Styles” next to the above mentioned “Styles” window:

The button shows how many Styles there are in the file at the moment. As seen above, there were 55429 Styles. After clicking that button, then saving and reopening the file (important), we now only see the Microsoft Built-in Styles, which are 47 in total, plus the amount you were actually using, which usually will not go above 15.

For experts: You can also run ?activeworkbook.styles.count to quickly see the amount of styles.


4. VLOOKUPs and other formulas referring to full columns
Check your VLOOKUPs and similar formulas, e.g. by pressing Ctrl+F to search in Excel, and write an equal sign =. (You may also want to choose "Options" and "Look in: Workbook" to look at the whole workbook at the same time.) Since Excel formulas (usually) start with an equal sign, this will find you all formulas. If you can see that they are referring to a full column, they may be slower than needed. Instead of simply referring to something like “A:B”, you can refer to something like “A1:B5000” instead. This may sound like a small change, but this 2nd reference is actually 200 times smaller than the 1st (since Excel has 1 million rows) and should be faster to calculate. You can read more in the section “Formula” from this article, or here.



5. Used Ranges
Make sure that your workbook is only using the cells that you actually need. See this example below from a workbook that had set a table to be =$1:$20

You might think that you just asked it to use 20 rows, and you indeed did, but you also asked it to use every column of those rows, and in e.g. XLSX format, that means 16384 columns. See above how a seemingly simple table stretches all the way to the right to the XFD column, the last one in Excel.

If you instead of the above use this table definition…
=$A1:$Z20
…your table will be 8000 times smaller, which is a great thing from a performance perspective.


Even if you don't have a table stretching all the way down or to the right, the Last Cells in your sheets may be much further to the right and/or down than you would expect. This can be checked by pressing Ctrl+End in each sheet. Read more about how to reset that under Step 6 here.



6. Conditional Formats
If your file has a huge amount of CFs listed under “Home” tab – “Conditional Formatting” – “Manage Rules”, it may be very slow to work in. You can use the roll bar at the top with the arrow keys to go through your sheets. In the picture below, we can see a scrollbar to the right, indicating a very long list:

 
If you notice any delay whatsoever when moving between different sheets in the "Show formatting rules for:" dialogue in the screenshot above, this is an indication that you should clean them up. Redesigning these to have much fewer rules, and each of them referring to a full range instead of individual cells, can provide a very big performance difference. It may be some work to redesign this, so you may want to test “Home” tab – “Conditional Formats” – “Clear Rules” – “Clear Rules from Entire Sheet”, just to test that it really makes a performance gain, before you start redesigning them.


(Wait, how did my file get thousands of Conditional Formats? This can happen if you Copy and Paste excessively. Most people don't consider that when they copy paste information, they may also copy CFs and other formatting. If you have experienced this issue in a file and want to minimize the chance of it happening again going forward, see this: “When you copy and paste cells that have conditional formatting set in Excel 2016, the conditional formatting rules are duplicated even though the rules already exist in the cells” with a reg key called ReplaceCFOnPaste mentioned in this article that you may want to set. This will prevent most of these issues in new files, but it will not clean up CFs in existing files)




7. Controlling calculation mode, and volatile functions:
-Sometimes, a single “=TODAY()” formula at the top of a sheet that is used in the calculation chain can render a complete workbook volatile, causing each data entry to perform a calculation. More info:
https://msdn.microsoft.com/en-us/library/office/bb687891.aspx#sectionSection2
http://www.decisionmodels.com/calcsecretsi.htm
-Changing the calculation mode from Automatic to Manual under Formula - Calculation Options, and then trigger it manually with e.g. F9 only when needed in this workbook, can also provide the breathing room that your file needs for becoming acceptable to work with.
-Keeping historical data as formulas can both risk the data accuracy, AND slow down the file. All data that should never be changed can be copied, and then “Paste Special –  values only”, to convert your old formulas into static values.


8. Broken links

Usually, broken links don't cause issues with file performance, but it can definitely happen and you may still want to clean them up to improve the file health. To clean them up, you can check your “Data” tab - ”Edit Links” dialogue, and check for links that point to e.g. the C:\ drive:


As seen in "Location", what is selected is a link pointing to a local C:\ drive. This may be intentional if you are the only one working on this file, but usually it is not intended and when the file is sent to another computer, the linked file will not exist. If other users might open these files, you would probably want to correct those links before that, or remove them.


(You can also click “Startup Prompt” in the Screenshot above, and make a choice in the menu that pops up, if you don’t want to see a prompt to update links whenever you open this file.

You can also break any links that are obviously wrong or not needed with “Break Link”. If you want to do this but find that nothing happens when you try, the link is most likely inside a Defined Name, see the section below)


9. Defined Names

Defined Names usually don't cause problems with performance, but you may still want to clean them up to improve the file health. You can find the Name Manager in “Formulas” tab – “Name Manager”. Here is an example from a file:

As seen, there are very many names that have “#REF!”, a reference error, in the “Refers to” column. These are very likely not in use any more, and should be deleted/corrected. Even ones that only have “#REF!” as part of the “Refers To” column should be deleted/corrected. There may also be ones that only have “#REF!” in the “Value” column, those can probably also be removed, but be more careful with removing those.


Defined Names can also be hidden. If you suspect there are, you can remove them with the File – Info – Check for issues – Inspect - Document Inspector, or code like this.

Automatic disabling of DEP depends on what configuration has been set in Boot.ini file, and it also depends on the application.

 

Please check what Configuration value has been set in the “Boot.ini” file (boot configuration file).There are actually 4 global configurations for DEP. The ‘Boot.ini’ file switches are as follows:

 

  • /noexecute=Configuration_Value

There are four options to this switch:

 

  1. OptIn:       

This setting is the default configuration. On systems with processors that can implement hardware-enforced DEP, DEP is enabled by default for limited system binaries and programs that "opt-in." With this option, only Windows system binaries are covered by DEP by default.

 

  1. OptOut:

DEP is enabled by default for all processes. You can manually create a list of specific programs that do not have DEP applied by using the System dialog box in Control Panel. Information technology (IT) professionals can use the Application Compatibility Toolkit to "opt-out" one or more programs from DEP protection. System compatibility fixes, or shims, for DEP do take effect.

 

  1. AlwaysOn:

 

This setting provides full DEP coverage for the whole system. All processes always run with DEP applied. The exceptions list to exempt specific programs from DEP protection is not available. System compatibility fixes for DEP do not take effect. Programs that have been opted-out by using the Application Compatibility Toolkit run with DEP applied.

 

  1. AlwaysOff:

 

This setting does not provide any DEP coverage for any part of the system, regardless of hardware DEP support. The processor does not run in PAE mode unless the /PAE option is present in the Boot.ini file.

 

To modify the DEP Value and ‘TurnON DEP’ globally, we must edit the ‘Boot.ini’ file.
