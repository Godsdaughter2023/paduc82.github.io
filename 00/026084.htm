I believe (in my case) that I have found the cause.

After looking at different computers in the organisation that I work for, I found that some of them worked properly in regards to the Window Insider Program (+ Link an Account) window, while others didn't.

Those that didn't, were the ones that I had changed a Local Security Policy setting (something that I had overlooked and failed to mention in my reply to Jesrael Bor)

This setting: [Local Policies\Security Options\User Account Control: Run all administrators in Admin Approval Mode] had been changed from 'Enable' to 'Disabled' in order for me to get easier access to the root folder in C: drive.

Unfortunately, this seemed to have had the side effect of not showing any options when you try to Link an account for the Windows Insider Program setup.

Having changed this setting the way I did, I believe, should not have had the above effect.

Despite this, by typing "secpol.msc" without the quotes in the Cortana search bar and then right clicking the entry that has the text: "Microsoft Common Console Document", and selecting Run as administrator, navigate to the following:

Local Policies\Security Options\ Run all administrators in Admin Approval Mode

If it has been set to Disabled, then set to Enable.

Exit and Reboot the system.

For me this worked.

When a phone updates to a new version, there are chances when all the updates do not jive well with the existing settings of the phone.
