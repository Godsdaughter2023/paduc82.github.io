Word's preference is to use the "Title" document property as the document name, truncating at certain characters. If that property has not been set, it goes for the first line (outside of the header) in the document, again, truncating at certain characters such as commas. This has been the default at least since Word 97.

I don't know of any way to change this default behavior short of vba.

If your document's template has a title document property, that property is inherited by the new document and forms the basis for the default document name the first time the document is named. Open that template and check the advanced property. File > Info > Properties (on right side) > Advanced Properties > Summary (tab).

el_loco---01 say: AAC is a variation of the MPEG-4 audio encoder.

MP3 is MPEG-1 Layer 3 audio codec.

AAC is therefor a much more advance codec. So (like some people mentioned), AAC files tend to be smaller.

As a rule of thumb (audio quality):

64 kbps AAC=128 kbps MP3
128 kbps AAC=192 kbps MP3
192 kbps AAC=256 kbps MP3

Higher bitrates it doesn't matter, the audio quality is so good that the limiting factor is the source audio quality.

What's cool is that you can encode songs with AAC at 128 kbps and the resulting file is smaller than an MP3 at 128 and the sound quality (of AAC) is better.

There is also an even more condensed format eAAC+ that allows for even smaller files without loss to quality. But they are not so supported (i.e. in PC only Nokia Music Manager will play these songs), and there have been some issues with the meta data.

eAAC+ files at 64 kbps are like MP3 at 160 kbps (about) in terms of audio quality and the file size is less than half.
