However I get very different results, conditional formatting returns 13 items in the top 10 % and the pivot table only returns 10 (and I double checked that it was top 10 % not top 10 items). 

I understand the logic behind the conditional formatting results however I cannot wrap my head around how 10 items can make up 10% of 134 items.

bernie deitrick say:

The logic is that a Pivot Table in Excel will return the largest items that add up to at least 10% of the total, not 10% of the items.  If you had 99 items with the value 1 and one item with the value 11, you would only get that 11 shown.  CF will highlight 10% of the population.  If you want the same from the Pivot Table, you could add a column to your data with a formula like 

=B2>LARGE(B:B,10%*COUNTA(B:B))

And use that column as a page filter, set to TRUE

marshall barton say:

The first thing you need to do after adding a field to a back end table is compact the front end.  Then see what issues remain. 

Compacting the front end, flushes all query plans so the next time a query is used, it will rebuild its plan, picking up any new fields and any fields that are in a different position in the table's field list.  There may be(?) internal queries that you are not aware of where compacting the front end will clean up those too.
