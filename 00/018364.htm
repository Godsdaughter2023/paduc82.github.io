Using =INDIRECT() - Microsoft Community

gary's student say:

Using =INDIRECT()

If I put:

A1:A4

in cell B1, the formula =SUM(INDIRECT(B1)) will give the same result as =SUM(A1:A4).

 

If, however, I put

Sheet1:Sheet3!B9

in cell B1. the formula =SUM(INDIRECT(B1) gives an error while =SUM(Sheet1:Sheet3!B9) works just fine.

 

Why is this?

ronrosenfeld say: AFAIK, INDIRECT will not accept a 3D reference, but if you construct an array, where each element refers to a single sheet, you can use it.

For example, this array formula will work -- note the N function and the fact that the ADDRESS function is returning an array with each element referring to a single sheet.

=SUM(N(INDIRECT(ADDRESS(9,2,,,"Sheet"&{1,2,3}))))

There are other ways to construct 3D arrays, but each element of the array can only refer to a single sheet; and the N function is required to convert that array into something Excel will SUM (for reasons unknown but apparently first discovered by Laurent Longre).

Here's another way.

B1:  B9
B2:  1:3

=SUM(N(INDIRECT("sheet"&ROW(INDIRECT(B2))&"!B9")))


jllatham say:

INDIRECT() doesn't seem to like the reference to multiple sheets.

If you put Sheet1 into E1 and Sheet2 into F1 then

=SUM(INDIRECT(E1 & ":" & F1 & "!B9")

also returns a #REF! error, as does putting Sheet1:Sheet2 into C1 and then building formula:

=SUM(INDIRECT(C1 & "!B9")

I begin to suspect that the evaluation of an INDIRECT() statement triggers the error when it encounters the colon.  Can't prove it, and someone may come along and provide an exception to that suspicion.  It may be by design if the evaluation of INDIRECT() specifically triggers on the presence of the colon in the reference to be evaluated.
