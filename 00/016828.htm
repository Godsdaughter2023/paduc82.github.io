bernie deitrick say:

When a workbook is open, it loses the path in its name:

Workbooks("C:\Program Files\Affiliates\Score_History.xlsx").

should be

Workbooks("Score_History.xlsx").

 

And there is no need to activate the workbook prior to doing the VLookup - since you have fully qualified the range being used.

That type mismatch error usually means that the value that you are looking for is not found in the first column of the range given. Usually handled like:
