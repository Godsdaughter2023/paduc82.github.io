Paste link an entire row based on the value of a single cell - Microsoft Community

haywoodoaks say:

Paste link an entire row based on the value of a single cell

Hi, within my workbook (Excel 2007) I have a main summary sheet where data is edited.  This has columns that show field names, numbers, areas etc and the crop that will be grown in them.  Data is entered here. The other worksheets are for individual crops eg barley, wheat, carrots.  I would like the workbook to paste link data that is entered in the summary sheet to the relevant crop sheet so, for example, if I click on the barley worksheet it will display only the fields that barley is being grown in. So I need to paste link entire rows based on a single cell value.

Thank you.

 

jllatham say:

This solution is a bit complex, so I'll explain in a lot of detail.

Let's say that your main summary sheet is named Sheet1 and that the column with the crop name is in is column E on that sheet.  Lets say also that you don't think you'll ever have more than 9,999 rows of entries on that sheet.  And finally, lets say you want to return information from columns A through F of Sheet1 over to the individual crop sheets.

For the sheet that you want only 'barley' entries in (entries from Sheet1 with Barley [case is not important] in column E), then the first row on the individual crop sheet where you want to return data from column A of Sheet1 barley entries, enter this formula:

=IF(ISERR(INDEX(Sheet1!$E$1:$E$9999,SMALL(IF(Sheet1!$E$1:$E$9999="Barley",ROW(Sheet1!$E$1:$E$9999),10000),ROW()-ROW($E$1)+1)-ROW($E$1)+1)),"",INDEX(Sheet1!A$1:A$9999,SMALL(IF(Sheet1!$E$1:$E$9999="Barley",ROW(Sheet1!$E$1:$E$9999),10000),ROW()-ROW($E$1)+1)-ROW($E$1)+1))

Now, this needs to be an array formula, to make it an array formula instead of committing it with the [Enter] key, you commit it using the [Ctrl]+[Shift]+[Enter] keys pressed together.  Then the formula should appear like this in the formula bar:

{=IF(ISERR(INDEX(Sheet1!$E$1:$E$9999,SMALL(IF(Sheet1!$E$1:$E$9999="Barley",ROW(Sheet1!$E$1:$E$9999),10000),ROW()-ROW($E$1)+1)-ROW($E$1)+1)),"",INDEX(Sheet1!A$1:A$9999,SMALL(IF(Sheet1!$E$1:$E$9999="Barley",ROW(Sheet1!$E$1:$E$9999),10000),ROW()-ROW($E$1)+1)-ROW($E$1)+1))}

Any time you edit the formula, you'll have to commit it again with the 3-key combination.

Once you've got that basic formula into a cell, you can select the cell, move your cursor until it's over the lower left corner of that cell and becomes a bold + symbol instead of the normal fat cross that Excel displays.  Left click and drag it to the right 6 columns.  It will automatically adjust to pick up the proper information from columns A, B, C, D, E and F from Sheet1.

Finally, select all the cells with the formulas in them and again move to the lower-left corner of the leftmost cell until you get that bold + symbol again and drag down for some distance on the sheet so you can pick up lots of rows of data from Sheet1.  Now all of your Barley information will show up on that sheet.

You can then copy the formula from the very first cell you entered it into on the Barley sheet and paste it into another sheet, say the one for Wheat, and simply change "Barley" in the formula to "Wheat" in the two places in the formula where "Barley" was at, and then repeat the fill-right and fill-down operations for Wheat.  Continue for as many sheets as you need.

A possibly easier way would be to set up one sheet for Barley (or another crop) all filled in and working and copy the entire sheet and then use Edit Find and Replace to replace Barley with the name of the next crop.

You'll notice that the formula is an IF() formula, with the 3 parts of an IF formula in it:
IF( test condition, action if true, action if false)

The first part of the formula [the test condition] is testing for an error condition and if the test does create an error, then an empty string ( "" ) is put into the cell, but if the test found some data, then pretty much the same formula is executed again, but it actually returns the data from a column.  Here's the basic test condition without the ISERR() portion, followed by the action-if-false part of the formula:

INDEX(Sheet1!$E$1:$E$9999,SMALL(IF(Sheet1!$E$1:$E$9999="Barley",ROW(Sheet1!$E$1:$E$9999),10000),ROW()-ROW($E$1)+1)-ROW($E$1)+1)
INDEX(Sheet1!A$1:A$9999,SMALL(IF(Sheet1!$E$1:$E$9999="Barley",ROW(Sheet1!$E$1:$E$9999),10000),ROW()-ROW($E$1)+1)-ROW($E$1)+1)

Notice that the only difference between these two formula parts is that the very first range that the test condition examines is Sheet1!$E$1:$E$9999 but the range the second half returns changes that part to Sheet!A$1:A$9999 so that the data will come from column A.  If you examine the other formulas in the first row formulas you created with the right-fill operation, you'll see that only that one piece of the formula changes from column to column, the rest of all the formulas will be exactly the same.

I hope this helps you get something set up that works for you.
