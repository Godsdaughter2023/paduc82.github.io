steve rindsberg say:

There are a couple ways you could do this, all involving VBA (ie macros).

Because of the way PPT deals with macros, the simplest thing would be to set up a macro to run in slide show view.

Add this macro to the presentation then assign it as the on-click Run Macro action setting for some shape in the presentation ... maybe a textbox that holds something like "Click here to enter customer name".  They wouldn't have to do this in front of the customer, just run it once beforehand to set the text as desired, then make sure NOT to click the "magic box" during the presentation.  Worst case, if they click it by accident, hit enter w/o filling anything in when the input box comes up and nothing more happens.

Sub GetCustomerName()

Dim sCustName As String

sCustName = InputBox("Customer name:")

' Don't do anything with blanks
If Len(sCustName) = 0 Then
    Exit Sub
End If

' Populate the presentation
Call FillInCustomerName(sCustName)

End Sub

Next, you need to write the FillInCustomerName subroutine that actually does the work.

Sub FillInCustomerName(sName As String)
' This assumes that you've tagged the shapes
' that need to be filled in

Dim oSl As Slide
Dim oSh As Shape

For Each oSl In ActivePresentation.Slides
  For Each oSh In oSl.Shapes
    ' if the shape has a tag named CustName
    If Len(oSh.Tags("CustName")) > 0 Then
      oSh.TextFrame.TextRange.Text = sName
    End If
  Next
Next

End Sub

Now ... about Tags:

Working with Tags (and a bit about Functions)
http://www.pptfaq.com/FAQ00815.htm

manojganapathi say:

I have encountered the same issue as well and here is my suggestion:

Alert on Yammer Embedded Webpart:
As of May 1 2015, Yammer no longer supports Internet Explorer 9 and will no longer work reliably in this browser. To continue using Yammer, you must upgrade your browser. For supported versions, please go to Office 365 system requirements. You can also access Yammer by downloading our mobile apps.

Above alert is shown since few weeks on our On-Prim SharePoint site for the Embedded Yammer. This occurs on IE browsers with older versions i.e. IE 8, IE 9, IE 10. As you specified your browser is still IE 11 and able to see the issue, the most possible reason might be your site is running on document mode of either IE 8 or IE 9 or IE 10, which was the same case for me. Check your IE document mode, your site might be customized to a lower IE document mode to support some other functionalities.

There is no way to fix in this case as of now, because "Using Yammer together with Internet Explorer 8 or Internet Explorer 9 is no longer supported.". Check this article on Microsoft Support:

https://support.microsoft.com/en-us/help/2967362/yammer-life-cycle-support-information-for-internet-explorer-8-internet

I suggest to upgrade IE document mode to latest version i.e. Internet Explorer 11 (Edge) Document mode to fix this issue.

Thanks,

Manoj
