daniel pineault say:

So ensure your function is in a standard module (not a form or report module and not a class module):

Public Function appTitle(frm As Access.Form)
    CurrentDb.Properties("AppTitle") = frm.Name
    Application.RefreshTitleBar
End Function

Then in the load event you would do something like:

Private Sub Form_Load()
    Call appTitle(Me)
End Sub

Ensure your code compiles without any errors.

You should also be sure to add proper error handling throughout.

Also, the code above, will set the application title to match the form's name, not it's caption.  If you wish to use the form's caption, you'd need to use something more along the lines of (once again it needs error handling and exception for no property existing):

Public Function appTitle(frm As Access.Form)
    If Len(frm.Caption) = 0 Then
        CurrentDb.Properties("AppTitle") = " "
    Else
        CurrentDb.Properties("AppTitle") = frm.Caption
    End If
    Application.RefreshTitleBar
End Function

Lastly, the code above assumes that the database AppTitle property already exists and will err if run on a database that has not already had a title assigned to it.  You should technically add specific error handling to manage that specific error so it creates the property if it does not exist to make your code more robust.

I suspect it wouldn't work for you because of the above.  If you go into your database properties for the current database and set an initial title, then the code will most probably run without issue for you.  That all said, adding the above suggested error handling would resolve the issue.

Hi gatineau_geek,

 

Thanks for post back.

 

Well, it’s likely that those two users have the same name in the Global Address List (GAL). And the “from” of any email you received is displayed according to the GAL prior to your contacts so even if you distinguish them in the contacts there’s no effect.

My suggestion is to contact your admins and let them edit the Global Address List so that the two users could be distinguished explicitly.

 

As a workaround, you may set up inbox rules and move email from a specific sender to corresponding folder to distinguish them. For instance, you may add a rule that when your mailbox receives an email sent by Smith, John (employee), the email would be moved into a folder which is prepared for email sent by John, the employee. And repeat this process for Smith, John (manager). Then you could tell who send the email according to the folder the email belongs to.

techfreak_ say:

Skype has no mobile-only subscriptions. Only mobile and landline or landline only subscriptions. Skype provides no way to specify a subscription preference. The smallest subscription package will always be consumed first. After that subscription is consumed, it will move onto the next longest subscription (your unlimited subscription). The only way to ensure the first 100 minutes is only consumed by mobiles, is to place only mobile calls for your first 100 minutes of the month. If you place any landline calls in that first 100 minutes, they will deduct from your 100 minute subscription, like any mobile would.

The only way to control which subscription is consumed is to create a new account. With that secondary account you can sign up for your 100 minute subscription and keep the unlimited subscription on your primary account. This way you can place only mobile calls on your secondary account and landline calls on your primary account.

If you reference your usage history, you will only see consumption. If you calculate the first 100 minutes of that usage (after rounding up), you can determine when the 100 minute subscription was fully consumed. If there was any call in which you were disconnected at that period of time (due to the subscription being consumed), that would be the time when the 100 minutes was fully consumed. If you redialed the number and it was a landline, the unlimited subscription would cover the cost of the call, from that point on. If the number was a mobile, you would have been charged credit for the callback (if you had a credit balance) and that would be displayed in your usage history.

The Europe unlimited mobiles and landlines subscription is the only remaining subscription that covers both. Unfortunately it is twice the cost of your original subscription that was discontinued.
