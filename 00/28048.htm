joeu2004 say:

zhenguolei wrote:

when i write some marcos in excel, when i set some number in currency format to cell, only 2 digi are keeped. I can solve it with rng.value=csng(price).  But, is this a bug?
[....]
Dim price As Currency
[....]

price = 12.34454

[,,,,]

rng.Value = price

First, use CDbl instead of CSng.  In general, never use type Single.  We lose too much precision.  Moreover, Excel effectively uses type Double for values stored in cells.

Second, use Value2 instead of Value.  As you discovered, property Value treats certain data types in special ways.  Property Value2 avoids such special interpretations.

Third, I think you should declare price As Double, in the first place, not Currency.  Look at the Currency help page.  Type Currrency always stores values with just 4 decimal places.  So x.34454 would be changed to x.3445, in the first place.  Is that really your intention?

Finally, yes, it appears that when we store type Currency into a cell with property Value, Excel rounds(?) to 2 decimal places.  I don't know if MSFT documents that anywhere; I don't have time to search.  But I do believe that others have said as much when the problem has been noted previously.  MSFT probably considers it a "feature", not a bug.

-----

[EDIT] Vaguely documented in the Range.Value2 help page, to wit:

``The only difference between this property and the Value property is that the Value2 property doesn’t use the Currency and Date data types. You can return values formatted with these data types as floating-point numbers by using the Double data type.``

But of course, you have to know what to look for first. :-(

When combining POP and Exchange accounts, you can have a single Inbox if you set the delivery location of the POP account to the Exchange Inbox. Otherwise, you will have separate folder sets for each account. That's required because each account must reflect that's on the server for that account and so the servers control what you see. Server X won't have the same contents server Y has.

You can always copy the contents of the PST to the folders of the Exchange account so those messages are online as well. Once you copy the data you can ditch the PST.
