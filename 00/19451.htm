chris_msft say:

Hi mho,

I'm really sorry to hear about the time you spent recreating your itinerary.

I think I understand what you were trying to accomplish, but please correct me if I'm wrong. But basically you had data on your MacBook Pro and then with your new iPhone, you restored the data and only part of the data was restored.

As for OneNote, you don't need to do any backup's, since everything is sync'd to the cloud using OneDrive. I will give a basic explanation of how OneNote sync works, but a very good explanation of OneNote sync is located here: http://omeratay.com/onetastic/?i=onenote-sync

For your purposes and Mac OneNote in general, the first half of the article is relevant.

As for OneNote sync, the way it basically works is that when you make changes with either your Mac or iPhone, it first stores it locally in a cache file. You can not open these files from OneNote, but OneNote uses them as temporary storage. Then when you have an internet connection, OneNote will sync the changes up to OneDrive. That is where the master copy is stored.

After the server has the changes, it will pass those changes down to all devices connected to that notebook.

As for your case, it sounds like you restored the cache file which confused the server as to which were the latest edits. In general if you get a new device, all you need to do is make sure that all the data is sync'd from the old device up to the server. Then log in from the new device and it will sync everything down.

If you can restore the data from 'Time Machine' and wifi off, I'd suggest you do that and copy the data out of OneNote. Then let OneNote sync with the server and once that is done, copy and paste the data back into the notebook. This will then get sync'd up to OneDrive and then sync'd to all other devices connected to the notebook.

Let us know if you have further questions on this.

jmh3143 say:
The error "Microsoft Account does not exist" usually means:

The person is logging in using an Alias.
If the user renamed the affected email address into a new account, the original account will become an Alias.
Thus logging in using the original account (Alias) will trigger the system to prompt the user that it does not exist. The solution is to log in using the new account.

The person mistyped the email address.
Retyped the correct email address.

The person did not access the account (WITH A BROWSER) within 365 days.
The account will be deleted permanently in the system due to inactivity.
You may recreate the account as a new account if it is still available.

macropod say:

My question is why does Word 2010 return the character "c" when I type "? selection.text" even though no characters are selected.

Because that's how it's designed. If you want to test what the selection encompasses, use code like:

Dim StrRpt As String
With Selection
  Select Case .Type
    Case wdSelectionIP: StrRpt = "Insertion Point Selection"
    Case wdNoSelection: StrRpt = "No Selection"
    Case wdSelectionNormal: StrRpt = "Normal Selection: " & .Text
    Case wdSelectionBlock: StrRpt = "Block Selection"
    Case wdSelectionColumn: StrRpt = "Column Selection"
    Case wdSelectionRow: StrRpt = "Row Selection"
    Case wdSelectionInlineShape: StrRpt = "InlineShape Selection"
    Case wdSelectionShape: StrRpt = "Shape Selection"
    Case wdSelectionFrame: StrRpt = "Frame Selection"
  End Select
End With
MsgBox StrRpt

  selection.Range.Start = 2
