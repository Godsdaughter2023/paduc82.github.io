Hi Steven,

As John Korchok has told you that dialog has not been available in the Word object model for VBA since Word 2003 for the PC and I believe it was available also for Word 2004 for the Mac... but nothing since.

As a workaround you can use this code:

Sub RunCommandBarControl()
    Dim ctl As CommandBarControl
    For Each ctl In Application.CommandBars("Format").Controls
        If ctl.Caption = "Picture..." Then
            ctl.Execute
        End If
    Next
End Sub

I hope this helps.

rich michaels - mvp (greatcirclelearning.com) say:

Hi Steven,

The code to open the Macros dialog looks like this:

Sub RunCommandBarControl()
    Dim ctl As CommandBarControl
    For Each ctl In Application.CommandBars("Macro").Controls
        If ctl.Caption = "&Macros..." Then
            ctl.Execute
        End If
    Next
End Sub

When working with CommandBars, a Menu control can always be addressed at the highest level, which is the application itself. In other words, even though the Tools menus contains a sub-menu with the label of "Macro" you access the Macro menu as I have shown above in the sample VBA code. 

Regarding your questions concerning Excel... they are best asked on the Excel forum because that way the most experts with that application will see them.

With Excel the way to have a "global" template available with custom macros to access is to create an Excel template workbook, include your custom macros in a Module of the Excel template, and then place this Excel template in Excel's Startup folder, where it will become enabled when you start Excel.

/Applications/Microsoft Office 2011/Office/Startup/Excel

When you get ready to Print a document:

  1. Choose Print from menu file
  2.  Locate the checkbox for 2-sided (or Duplex) printing and check the Box.

scottgem say:

The product total automatically updates on the first item but won't on the second or third.

Can you please advise how i can get around this.

Yes, fix your design! What it sounds like you are doing is storing the current quantity on hand based on what is ordered. But this is a CALCULATED values and should not be stored. QOH should be calculated in a query by adding the Incoming transactions and subtracting the outgoing transcations. A query like:

SELECT PRODUCTID, SUM(Incoming) AS Received, SUM(Outgoing) AS Sold, Sum(Incoming)-Sum(Outgoing) AS QOH

FROM Transaction

GROUP BY ProductID

ORDER BY ProductID;

Will give you the actual QOH at any time.

How to Place Gray edges of pages at margins in Word 2008?

john korchok say:

Keep in mind that since this is at the page edge, printing to most desktop printers will create a white stripe because it can't print to the edge. If you're creating a PDF instead, this won't be an issue.

With your document open in Word, double-click in the header area (at the top of the page) to open it. Draw an rectangle the height of the and the width you want, then color it light grey. Use the cursor keys of numerical positioning to get ti to the page edge or slightly over (I usually create a 1/100" overlap off the edge).

Objects in the header and footer automatically repeat on each page, but you may have to copy the rectangle to other page headers if you have section breaks, a different first page, or odd and even headers.

bob jones aka: cybertaz say:

Before anything else, let me emphasize that Word is a word processing program, not a Page Layout program. What you want to do can be done, but it's a somewhat involved process. There are several aspects of the program that need to be understood in order to accomplish it;

  • It requires the use of Word's Section Breaks to isolate the series of pages on which the border is to appear. That implies that other factors [such as Headers & Footers] need to be considered.
  • The graphic object needs to be placed in the Header in order for it to repeat on each of the pages in that Section.
  • To render on paper (or PDF) flush to the edge of the pages you have to use Format> Document - Page Setup to specify whatever Paper Size your printer offers for borderless printing.
