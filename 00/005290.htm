joeu2004 say:

For no good reason (defect, IMHO), XIRR does not like zero for the first cash flow.  In that case, the function always returns +/-2.98E-9, which should be interpreted like the #NUM and #DIV/0 error returns.

As you noted, entering a small negative value in B4 (I use -1E-6) remedies the problem, with little impact on the derived IRR.

I use a negative value because that is the sign of your first real cash flow (-80 in C4).  I would use +1E-6 if the first real cash flow were positive.

The derived IRR is 82.6079308986664% (in B6).

In this case, we can verify the derived IRR by noting that =XNPV(B6,B4:E4,B1:E1) returns about -5.14E-09, which is relatively close to zero.

However, if the derived IRR were negative, XNPV would not work because it does not accept negative discount rates, for no good reason (defect, IMHO).

More generally, we can calculate the NPV with the formula =SUMPRODUCT(B4:E4/(1+B6)^((B1:E1-B1)/365)) .

joeu2004 say:

@Joe....  In the future, please do not piggyback one discussion with unrelated questions ("unrelated" = completely different facts).

I don't know where you are finding those "remedies", but they are misleading, if not incorrect.

1. The format of the cells is irrelevant.  In fact, the data can even be text, as long as Excel can interpret the text as numbers.

2. Certainly, "reducing" the number of iterations cannot improve the XIRR outcome.  But I don't believe XIRR is affected by the Max Iterations configuration, in any case.  (It was not in past versions of Excel.)

3. It does not matter whether the first cash flow is an "outflow" or an "inflow", as long as you use the sign of all cash flows consistently.  For example, XIRR returns the same result for {-100,50,60,70,80} and {100,-50,-60,-70,-80}.

The thing to experiment with is the "guess" parameter (3rd parameter).  It defaults to 10% (0.1).

Sometimes, -10% is a better "guess", even if the expected result is positive.

But a more methodical way to determine a reasonable "guess" is to look at the NPV curve.

In one column (D1:D21), enter the discount rates -99%, -90%, -80%,..., -10%, 0%, 10%, 20%,..., 100%.  You might need to go beyond 100%.

In a parallel column (E1:E21), ostensibly we want to enter formulas of the form =XNPV(D1,$A$1:$A$10,$B$1:$B$10), where A1:A10 contains the cash flows and B1:B10 contains the corresponding dates.

But as I noted previously, XNPV does not allow negative discount rates, for no good reason (defect, IMHO).

So we must use formulas of the form =SUMPRODUCT($A$1:$A$10/(1+D1)^(($B$1:$B$10-$B$1)/365)).

In would be prudent to format E1:E21 as Scientific, since the NPV might "blow up" (become unduly large) for some discount rates.

Hopefully, you will see one sign change in the NPV values.  For a "guess", use the average of the two corresponding discount rates.

[EDIT] PS.... Suppose the sign change is between 10% and 20%.  Use a "guess" of 15%.  If that still is not a good enough "guess" for Excel XIRR, you might try looking at the NPV curve between the two rates.  Create a table of NPVs for the discount rates 10%, 11%,..., 19%, 20%.  If the sign change is between 12% and 13%, use a "guess" of 12.5%.  If necessary, repeat with the discount rates 12%, 12.1%,..., 12.9%, 13%.  And so forth.

Caveats....

1. Sometimes, XIRR cannot determine an IRR, even with a good "guess".

2. Sometimes, there truly is no IRR for the cash flow model.  Often, that is because the cash flow model is incorrect or unrealistic.

3. Sometimes, there truly are multiple IRRs.  In that case, you might see multiple sign changes in E1:E21.  You might need to coerce XIRR into finding the "correct" IRR -- the one that makes the most sense for the cash flow model.  But sometimes XIRR cannot coerced.

4. Beware of XIRR returning what appears to be zero.  If you format the cell as General or Scientific, it is often +/-2.98E-9.  That is really an error result, similar to #NUM and #DIV/0.  Often, it is because the cash flow model does not fit the requirements of XIRR, implicit as well as explicit.  See my first response in this thread.

If you still have no success with XIRR, start a new thread, and post the values and dates that you are using.

"=SUM(R2C:R[-1]C)" is a string (bound by quotes), so if you want to replace part of this with a variable, just break out the string into multiple parts, and insert your variable, e.g.:

"=SUM(R" & "2" & "C:R[-1]C)"  breaks the original into three strings, but the & brings it all back together so it is functionally the same
