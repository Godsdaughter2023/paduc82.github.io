cortig say:

winmail.dat issues are a pain!

Outlook for Windows uses that format (if configured to use it) for rich text emails. The problem is that most email clients (Including Outlook for Mac) can’t read the format and the rich text portion of the email appears as the winmail.dat attachment. That also happens sometimes with inline images in messages (encapsulated in the .dat attachment).

In any case, that explains why it only happens for messages in rich text.

There are a number of utilities on Mac that will extract the content of the .dat but that’s not a practical solution at all.

The best option is to make sure that Outlook for Windows is NOT configured to send emails in that format.

You can google-up how to do that and a number of sites will give you the details.

Here’s one for instance:

https://www.lifewire.com/prevent-sending-winmail-dat-attachments-1173717

(the first answer I found)

To add insult to injury, some email servers also perform the automatic conversion of the rich text content and use the “evil” winmail.dat format. n that case, the server administrator has to be contacted to request to have the feature turned off.

Corentin
