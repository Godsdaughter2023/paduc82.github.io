royjeffries say:

Word 2010 Save and Send, Create PDF/XPS Doc

Word 2010, Save and Send, Create PDF/XPS Doc, Default format is PDF/A.  How can I change that permanently so ISO 19005-1 compliant (PDF/A) is not checked under Options?  Is there a registry settings?  I could not find that setting to turn it off in the OCT.  TIA
graham mayor say:
I am not aware that it is an available option to set in the registry. It is probably buried in the binary format of the Word Data key. depending on what you are doing, the answer maybe to create a macro to SaveAs PDF associated with you application, so that when you create a PDF from it it has the required settings. e.g. the following saved in the normal template will intercept the command you referred to (though not SaveAs) and force the settings you want. Make whatever changes you need to the list of settings at the end of the macro. The difference with this function from the original is that it saves the document as a document first.

Sub DocExport()
Dim strDocName As String
Dim strPath As String
Dim intPos As Integer
Start:
'Find position of extension in filename
strDocName = ActiveDocument.Name
strPath = ActiveDocument.Path & "\"
intPos = InStrRev(strDocName, ".")
If intPos = 0 Then        'Document is not named
    ActiveDocument.Save        'so save it
    GoTo Start:
End If
strDocName = Left(strDocName, intPos - 1)
strDocName = strDocName & ".pdf"
'Save file with new extension
ActiveDocument.ExportAsFixedFormat _
        OutputFileName:=strDocName, _
        ExportFormat:=wdExportFormatPDF, _
        OpenAfterExport:=False, _
        OptimizeFor:=wdExportOptimizeForPrint, _
        Range:=wdExportAllDocument, _
        From:=1, _
        To:=1, _
        Item:=wdExportDocumentContent, _
        IncludeDocProps:=True, _
        KeepIRM:=True, _
        CreateBookmarks:=wdExportCreateNoBookmarks, _
        DocStructureTags:=True, _
        BitmapMissingFonts:=True, _
        UseISO19005_1:=False
End Sub

Hi MarkDart,

The short answer to your question is yes.
The default MRM policy there archives emails every 2 years automatically by default on the root folder for all mailboxes.
Then the policy will keep emails in the archive mailbox until you delete them from the archive mailbox. The emails will be moved to the same folders as previously in the archive mailbox.

For the detailed information, see Enable an Archive Mailbox.
