I'm not sure what you mean. The default for building a formula is relative reference.

If you were to record a macro to add A1+B1 it will record this and the formula will be =A1+B1 using relative references:

ActiveCell.FormulaR1C1 = "=R[-2]C+R[-2]C[1]"

If you want to turn the relative reference into an absolute reference you have to get rid of the braces manually. This example changes A1 to $A$1 in default reference style.

ActiveCell.FormulaR1C1 = "=R1C1+R[-6]C[1]"

rick rothstein, mvp - excel say:

That will not work... the argument to an individual Case statement with a Select Case block is not a String... it is a series of Strings... the commas are part of the Case statement's syntax, they are not part of the argument. In what you are doing, the sheet name would need to match the entire contents of MyString to execute its Case statement's code... match a part of it would not work. Here is a quick demonstration of that...

Sub Test()
    Dim MyString As String, N As String
    MyString = """Name1"",""Name2"",""Name3"""
    N = "Name2"
    Select Case N
        Case MyString
            MsgBox "Matches"
        Case Else
            MsgBox "NOT A MATCH"
    End Select
End Sub

Now as to what you are attempting to do, I would take a slightly simpler approach than the one Mike suggested to you...

Sub SomeSubToo()
    Dim MyString As String, ws1 As Worksheet
    MyString = """" & Join(Application.Transpose(Range("MyRange").Value), ",") & """"
    For Each ws1 In ThisWorkbook.Worksheets
        If InStr(1, MyString, """" & ws1.Name & """", vbTextCompare) Then
            ' The worksheet name is in the string. do something
        Else
            ' The worksheet name isn't theere Do something else
        End Select
    Next
End Sub

pete_uk say:

If you don't want users to see your confidential information then you should not include it within the file.

If you must, though, then you could <move> that column to a new worksheet, so that the formulae using the data in it will still work, and then you can use a bit of VBA to make that new sheet very hidden. This means that it will not be visible as a hidden sheet (i.e. won't appear on the list of hidden sheets and so Unhide from within Excel will not work on it) and cannot be deleted, but of course any determined user could uncover it with the reverse VBA.

Anyway, here's what to do to make "new_sheet" very hidden:

Alt-F11 will bring up the VB Editor, and CTRL-G will put you in the Immediate window, and you can type the following command:

Sheets("new_sheet").Visible = xlveryhidden

(or copy it from here). Obviously, put your own sheet name between the quotes. To make it visible again, just type the following:

Sheets("new_sheet").Visible = xlSheetVisible

You can also apply a password to protect the VBA project, and I understand that these are a bit harder to break than the sheet-level protection.

Hope this helps.

Pete
