Usually this error message occurs if a transport rule or spam filtering is set up with the Defer the message if rule processing doesn’t complete option enabled, or the scanning process shows time out when inspecting the attachment during text extraction.

I suggest you let the sender try the following steps to see if the issue can be fixed:

  1. Password-protect the message before s/he sends it.
  2. Disable the Defer the message if rule processing doesn't complete option in the transport rule. Be aware that this option may be set on any rule in the rules list. To modify transport rules, use an admin account to log into Exchange Admin Center> Mail Flow> Rule.
  3. Check whether any a third-party service somewhere between the sender and the recipient breaking the content.
