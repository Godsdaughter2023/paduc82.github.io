allenfei_msft say:

Hi Udayreddy,

Thanks for your detailed information.

Actually, what you are experiencing is an expected behavior. For users who have read or less than read permission to a file, they will not be allowed to share it to external users, they can share it to internal users but this invitation requires approval by site admins to take effect.

For users who have contribute or more than contribute permission to a file, they will be able to share it with both external and internal users.

a. user say:

Although Word has the facilities to do this, a way to do it that I believe is less reliant on Word's sometimes tortuous language marking/LTR/RTL script support (and perhaps, on how the recipients of your document have their copy of Word set up) is to use VBA to create a whole bunch of "Document Variables" then use a slightly more complex field to insert your page numbers.

e.g.,

 a. use the following VBA to set up Document Variables called "persian1", "persian2",..."persian9999" that contain the string you need for each number

 b. instead of using a page field like { PAGE } to insert the page numbers, use a nested field

{ DOCVARIABLE "persian{ PAGE }" }

where each pair of {} is inserted using ctrl-F9 and eberthing else is plain text entered from the keyboard. It's probably easier to create when you are editing LTR using e.g. English.

A crude bit of VBA to create the Variables is as follows. Once you have set them up for a document, they will stay there until you modify them. Graham Mayor's page may help you install and run the VBA macro:

http://www.gmayor.com/installing_macro.htm

 

 

Sub setpersianvars()

Dim i As Long

Dim n As Long

Dim s As String

Dim p As String

For n = 1 To 9999 ' increase this if your document is really long!

  s = CStr(n)

  p = ""

  For i = 1 To Len(s)

    p = p & ChrW(&H660 + Asc(Mid(s, i, 1)) - 48)

  Next

  ActiveDocument.Variables("persian" & CStr(n)).Value = p

Next

End Sub
