hansv mvp say:

For what it's worth, try this:

 

Sub tableformat()
    Dim i As Long
    Dim oRow As Row
    Dim oCol As Column
    On Error GoTo ErrHandler
    For i = 1 To Selection.Tables.Count
        For Each oRow In Selection.Tables(i).Rows
        Next oRow
NextStep:
        For Each oCol In Selection.Tables(i).Columns
        Next oCol
NextTable:
    Next i
    Exit Sub
ErrHandler:
    Select Case Err
        Case 5991
            MsgBox "Table #" & i & " has vertically merged cells"
            Resume NextStep
        Case Else
            MsgBox "Error " & Err.Number & ": " & _
                Err.Description & " in table #" & i
            Resume NextTable
    End Select
End Sub

hansv mvp say:
Once two (or more) cells have been merged, they form one cell. There is no property that specifies that the cell is the result of merging. So there is no really good way to do what you want. You might count the number of cells per row and per column, annd compare the width and height of cells, but that's tedious and not 100% conclusive.

what is SIM card used for? - Microsoft Community

stephennielsen say:

what is SIM card used for?

Can it be used to hold music and other content items?
adistianto yuwono say:

SIM cards store network-specific information used to authenticate and identify subscribers on the network, such as IMSI (your phone number), Authentication Key (contain the key needed by your phone to connect to your wireless carrier), LAI (Location Area Identity, to store your location using operator's noetwork) and Operator-Specific Emergency Number.

The SIM also stores other carrier-specific data such as the SMSC (Short Message Service Center) number, Service Provider Name (SPN), Service Dialing Numbers (SDN), Advice-Of-Charge parameters and Value Added Service (VAS) applications.

Most SIM cards can store a number of SMS messages and phone book contacts. The contacts are stored in simple "name and number" pairs. The number of contacts and messages stored depends on the SIM; early models would store as few as five messages and 20 contacts while modern SIM cards can usually store over 250 contacts.

SIM cards can't be used to store your personal data, such as music and other content. It would be the internal storage or the removable storage (if present) in your device. For Windows Phone, the choice of removable storage that can be used on certain devices is only microSD cards.

SUGGESTION: Outlook Appointments/Meetings - Selecting Which - Microsoft Community

richh67 say:

SUGGESTION: Outlook Appointments/Meetings - Selecting Which Calendar

I'm posting here in the hopes that someone will know the right place to send feedback for Outlook 2013 for desktop (Windows). It seems that the other Office applications have the smiley to send feedback, but I don't see it in Outlook.

I have several email accounts in my Outlook application. But when I want to create an appointment or a meeting, I want all of them in the same calendar. The way it works now it is far too easy to forget to check which calendar it is going to be saved to, and appointments end up in the wrong calendar.

First, it is not obvious which calendar you are saving to. There is just a little thing in the status bar that says "In Folder Calendar" with a calendar icon. Not which calendar. Not only that, but when you're creating a calendar item, everything you are entering is at the top of the form, and this is at the bottom where your attention isn't.

Second, It takes 3 clicks to see which calendar you're saving to (File | Move to Folder | Other Folder) and two more clicks to select a different calendar (the calendar you wish to select, and OK). This is far more complicated than it needs to be, and it is not an obvious process.

How do you solve this?

  1. Put a combobox above or to the left of the Subject field, a location where it is obvious
  2. In that combobox, include not only the word "calendar" (which should be obvious since it is a calendar item), but also the name of the calendar that is selected. This way the user doesn't have to click it to know which calendar the item is going to be saved to.

By making this simple change, the user has to click zero times to know which calendar the item will be saved to, and only has to click twice to change the calendar if the selected one isn't the intended one.

Also, it would be great to select which one of these in the list is default, since currently it seems that there is no default, and if you are creating a calendar item from the email pane by using the New Items selection, it selects the calendar associated with whichever email account was last viewed.

If someone knows a better place to send this feedback, I ask that you please let me know what it is.

Thank you,

Rich
