A "redistributable" is code included by the developer of an application program.

We have no way of knowing which programs you installed in the past requires any particular one of these items.

If you uninstall a C++ redistributable, the application that requires it will no longer work. 

Based only on the stated sizes in your screenshot (which are likely not very accurate), these items total about 160 MB, which today is really not that much disk space.  If you are running low on disk space, there are a number of steps you can take, but the real solution is to get a larger (and/or second) disk.

There's a reasonably good discussion of this issue here:  https://askleo.com/do_i_need_these_microsoft_visual_c_redistributables/

bill smithers say:

Also read this information about them:

Why Are There So Many “Microsoft Visual C++ Redistributables ...

Any additional versions of the Visual C++ Redistributable you see on your system were installed along with some program that required it. When a developer codes in a specific version of Visual C++, the code libraries for that version must also be present on the user’s system for the application to run. That means that, for example, if a developer used Visual C++ 2005 (or Visual Studio 2005) to create a program you’re installing, you can expect to see the Visual C++ 2005 Redistributable installed on your system along with the program.
