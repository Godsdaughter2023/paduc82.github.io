Private Sub Worksheet_SelectionChange(ByVal Target As Range)

a. user say:

Hi Jennyn,

The most common configuration issues which lead to e-mail duplication are as follows:

◾Rules

◾Multiple accounts

◾Profile recreation

◾Synchronization tools

◾Virus scanner/firewall

◾Tools to clean up the mess

To prevent them from happening again, you will have to troubleshoot things one by one.

I suggest you to check if the rules you applied prior to the issue are causing error.

Rules, and more specific “Move to folder” rules, is the easiest one to get yourself some duplicates. This is because for each e-mail received, Outlook will fire all configured rules against it. The following example will show how this can lead to duplicates.

In this example there are two rules configured;

  1. Apply this rule after the message arrives
    from Robert Sparnaaij
    move it to the Outlook folder

  2. Apply this rule after the message arrives
    with Outlook in the subject
    move it to the Outlook folder

Now imagine that you receive an e-mail from me where I have the word Outlook in the subject; both rules with apply! So it will get moved out of the Inbox two times and thus create a duplicate.

The way to prevent this, is to add the action “stop processing more rules” to the rule. If we would add that action to the first rule and the rule applies the action “stop processing more rules” will prevent that the second rule will be executed as well.

Keep in mind that if you have even more rules configured that should always trigger – for instance that assigns messages to a category or marks it for follow up – you’ll need to sort your rules in the correct order as well.

You can bookmark the value in the datepicker content control e.g. with the name 'BarDate' and then in the footer use the following field construction to display the date without punctuation. This you can format with your barcode font.

{ IF { BarDate} = "Click*" "" "{ BarDate \@ "ddMMyyyy }" }

The only snag is that Cross References do not update automatically when the supplied data changes, so you need to force that update. You can do this from a macro using the exit event from the content control e.g. in the ThisDocument module of the form add the following code.

The code will run each time you leave a content control. As it appears you only have two such controls, there is no need to limit the code to run on exit from the date picker.

Option Explicit

Private Sub Document_ContentControlOnExit(ByVal ContentControl _
                                          As ContentControl, _
                                          Cancel As Boolean)
    Call UpdateAllFields
lbl_Exit:
    Exit Sub
End Sub

Add a VBA module to the form and in it insert the following code

Option Explicit

Sub UpdateAllFields()
Dim oStory As Range
    For Each oStory In ActiveDocument.StoryRanges
        oStory.Fields.Update
        If oStory.StoryType <> wdMainTextStory Then
            While Not (oStory.NextStoryRange Is Nothing)
                Set oStory = oStory.NextStoryRange
                oStory.Fields.Update
            Wend
        End If
    Next oStory
    Set oStory = Nothing
lbl_Exit:
    Exit Sub
End Sub

Save the form as a macro enabled document.
