Tonya,
I curious as to what you mean when you say, "the projects will not have individual project plans". If your subprojects aren't individual plans how can they be inserted into a master? I must be missing something.

Okay here's an idea. Instead of saying you will have 100 projects, 50 for boats, 25 for planes and 25 for cars, why not have 3 subprojects. One for boats, one for planes and one for cars. In each vehicle plan organize your summary lines by phase or activity type. Under each summary line show tasks for each vehicle. For example let's say your boat plan is to refurbish the cabins for 50 customers. And for simplicity let's say the "phases" are deconstruct, restructure and rebuild. Under the deconstruct summary list each boat with a performance task such as, "Boat 1 - remove doors and drawers", Boat 2 - remove doors and drawers, etc. Then "Boat 1 - disassemble interior structure", Boat 2 - disassemble interior structure, etc., followed by the remaining performance tasks under the deconstruct summary. When all those tasks are in place, do the same for the next phase (i.e. restructure) for all 50 boats.

Repeat the above structure for each of the other vehicles.

Using this method it is very easy to update by adding/removing phases as necessary. You can also easily add or remove a particular item (i.e. add more boats or delete some boats or planes or cars) if you structure the file to have identifiers for each customer (e.g. use Number1 to store the customer number, or Text1 if you want to identify them by name).

Now when you build the master, you only have three subprojects. Much less chance for corruption that can sometimes occur with dynamic master files.

Hope this helps.

John

For reference yes, you could do a spreadsheet and then use an import map, (I don't see a particular need for a macro), to bring the generic plan into Project. However, you can insert the same subproject multiple times, but not dynamically. Let me splain.

The default mode for inserting subprojects into a master is dynamically. With a dynamic master the subprojects do not actually become part of the master, rather, the master only contains pointers to each subproject. The advantage of a dynamic master is that changes made at master level are reflected in the subprojects and vice versa. The main disadvantage of a dynamic master is that creating and maintaining the structure requires a great deal of file discipline or corruption will result. However, if you unckeck the "link to project" in the Insert Project window, you will create a static master. In a static master Project basically copies the entire contents of each subproject into the master and there is no dynamic link connection. You can therefore insert a given subproject as many times as you want into a static master. Furthermore, you indicated in your first post that you have no intention of working at the subproject level, you are going to do everything at master level.

So the bottom line is if you want to create a template file for each change type, you can do that and then insert them into your static master as needed.

ossiemac say:

Currently I have it so it will save the name of the file automatically the same as the workbook. 

What can I do so it will allow you to change the name of the PDF file prior to saving?

The code sample below provides you a little extra. You can select the required worksheet and then enter the required name for the file.

When you get the first InputBox, select the required worksheet and then click on any cell in the required worksheet. The reason it is necessary to click on a cell in the required worksheet is that the code is actually expecting a range for the input and it then identifies the parent of that range for the required worksheet.

Next you will get an InputBox to enter the required file name. Just type in the name you require. Ensure that you only use valid filename characters because I have not included validation to ensure there are no illegal characters.

If you don't want the worksheet selection part then just delete all of the code between the asterisk lines.

Sub SaveAs_ActiveSheet_PDF_2()
    Dim strTitle As String
    Dim strPrompt As String
    Dim wb As Workbook
    Dim newName As Variant  'Must be a variant so that Cancel can be used with InputBox
    Dim ws As Worksheet
   
    Set wb = ThisWorkbook
   
    '*****************************************************************************************
    strTitle = "Worksheet selection"
    strPrompt = "Select required worksheet plus any cell on the worksheet." _
                & vbCrLf & "If Active sheet required just select a cell." _
                & vbCrLf & vbCrLf
   
    Do
        On Error Resume Next
        Set ws = Application.InputBox(Prompt:=strPrompt, _
                                    Title:=strTitle, _
                                    Type:=8).Parent
       
        If Err.Number = 424 Then
            MsgBox "User cancelled. Processing terminated."
            Exit Sub
        End If
      
        On Error GoTo 0
       
        If Not ws Is Nothing Then   'Not nothing then is something so is valid.
            ws.Select
            Exit Do
        Else
            MsgBox "Invalid selection. Select the worksheet and any cell on the worksheet."
        End If
    Loop
    '**************************************************************************************
   
    strTitle = "File name entry"
    strPrompt = "Enter the required file name."
   
    Do
        newName = Application.InputBox(Prompt:=strPrompt, _
                                    Title:=strTitle, _
                                    Type:=2)
       
        If newName = False Or newName = "" Then
            MsgBox "User cancelled or did not enter a name. Processing terminated."
            Exit Sub
        Else
            Exit Do
        End If
    Loop
    
    ActiveSheet.ExportAsFixedFormat xlTypePDF, "..\desktop\" &  _
                newName & ".pdf", OpenAfterPublish:=True
   
End Sub
