jim gordon mvp say:

a) recovering lost work

Your description about how Excel works is accurate. The idea is that every so often Excel saves a temporary file in the background as you work. If Excel crashes, when Excel opens it grabs that temporary file and displays it and says: "Oops, I crashed. I found this temp file. Maybe you can pick up from here." 

Unfortunately, some crashes will wipe out the file you are working on completely and your saves are lost, too. If you are picky about versions you can use Save As and give each save a unique file name (perhaps based on date and time).  You could write an Excel macro that does this for you.

b) prevent this from happening again

It looks like you are diligent about keeping your software and Mac OS up to date. That is good. The next time you get a crash, copy the the crash log and paste it in as a reply. If you know how to use Console app, grab the most recent log and use that. 

The most common problem causing crashes is bad fonts.  A simple troubleshooting step to perform is to do a Safe Boot. It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot the font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. 

Once you reach the desktop in Safe Mode try running Excel. Is the crash still there? If not, then maybe simply performing a Safe Boot fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart. Try Excel again. Is the crash still there?

If the crash was gone in Safe Mode but returns during a regular boot then there are more things to try. The first would be to use a font utility to validate ALL your fonts. Get rid of any font that doesn't pass as clean and green, restart your Mac and then empty the trash. If you've gotten this far and still have a crash, then there are more things to explore. But if you get this far you've probably gotten rid of the crash. Either way, please reply with your results.

Cover Pages are one of the types of building blocks. The default ones are stored in the file Built-In Building Blocks.dotx; the working copy of that file is stored in the user's profile at %appdata%\Microsoft\Document Building Blocks\1033\16 (assuming you're using English US, otherwise the 1033 is replaced with another Locale ID number). When you say you removed the default cover pages, that implies to me that you did so only for your computer. The other users will still have their unchanged Built-In Building Blocks files.

I'm not so sure about why your custom cover pages don't appear, but I suspect it's because they weren't actually saved in your template. graham mayor say:

The code in question has nothing to do with creating a COM add-in, and you don't need a COM add-in (or that code) to use your own ribbon tabs.

You can create an add-in template and modify the ribbon of that template to add to or replace the tabs on the standard ribbon. If you position your tab before the Home tab, it will be the default tab rather than the Home tab. You can choose whether or not to remove the Home tab.

What you need is a primer on ribbon editing - see http://gregmaxey.mvps.org/word_tip_pages/customize_ribbon_main.html  

If you want this personal tab to apply to all documents then put the add-in template in the Word startup folder If you have not changed the preferred startup folder it can be located (in English language versions of Windows) by typing
%appdata%\Microsoft\Word\Startup
in the Windows File Explorer Address bar and pressing Enter.

If you want it only to apply to a particular template, then modify the ribbon of that template.

rohn007 say:
That "... isn't working properly ..." pops up at the most inopportune times.  You can try doing an Office Repair to see if that stops the problem.

Another thing you can try is starting Word in "Safe Mode". If that stops the problem from happening, then the cause is one of the addins. You can disable them one at a time to find the culprit.

Shortcut to Start Word in Safe Mode or Excel in Safe Mode, Hold <CTL> key when clicking on icon or using the Start menu search window and entering WINWORD.EXE /a  or  WINWORD.EXE /safe
When you have this sort of failure, there is no way to recover all of your changes. You can modify how much is lost by making the "autosave" interval shorter. But if the document is long shorter intervals can be disruptive, blocking editing while it is doing the autosave.

The autorecovery 'autosave' only creates a temp backup that is invoked when there is a program crash.

Go into File menu > Options command > Save options. Make sure there is a check mark beside "Save AutoRecover Information every" ?? minutes.  The default is 10 minutes, make it smaller, how small depends on your preferences.
