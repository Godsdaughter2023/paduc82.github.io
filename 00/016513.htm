The first error relates to system files or WU corruption so open PowerShell (Admin) and run

DISM /Online /Cleanup-Image /RestoreHealth and sfc /scannow in that specific order.

If that doesn't help, try these other common troubleshooting steps to fix WU issues:

  • Run Windows Update troubleshooter or wu10.diagcab
  • Make sure date & time are correct
  • Uninstall any 3rd party antivirus program
  • Delete the "Download" folder in the Windows/SoftwareDistribution folder
  • Rename the "SoftwareDistribution" folder by running RenameWU.cmd with admin rights
  • Set Feedback & Diagnostics to FULL
  • Verify MS account
  • Unplug any external USB devices
  • Reset Windows Update (use the auto-script)
  • Switch Insider Ring to FAST
  • Remove or disable Bitlocker
  • Disconnect from the internet once the download is complete
  • Run Microsoft's Software Repair Tool

jan karel pieterse say:

Hi,

I advise against sharing your workbook. Excel's sharing feature is buggy and error-prone.

If you are just appending records to a list, I would do it like this:

Use your existing solution for data entry, but in the macro behind the button, have the macro open a separate workbook which will hold the table. This second workbook does not need to be shared, nor does the first one. WHat the macro has to do is check whether or not the second workbook is "in use".

Some example code:

Option Explicit

Sub Demo()
    OpenWorkbook "C:\Users\You\DataWorkbook.xlsx"
End Sub

Sub OpenWorkbook(sFileName As String)
    Dim dTime As Date
    dTime = Timer
    Do
        DoEvents
    Loop Until FileAlreadyOpen(sFileName) = False Or Timer - dTime > 30
    If FileAlreadyOpen(sFileName) Then
        MsgBox "Opening workbook in read-write mode failed, please try again later"
    Else
        Workbooks.Open sFileName
        'Now copy the records and save and close the file
    End If
End Sub

Private Function FileAlreadyOpen(sFileName As String)
' returns True if sfilename is currently in use by another process
' example: If FileAlreadyOpen("C:\FolderName\FileName.xls") Then...
    Dim lFile As Long
    lFile = FreeFile
    On Error Resume Next
    Open sFileName For Binary Access Read Write Lock Read Write As #lFile
    Close #lFile
    ' If an error occurs, the document is currently open.
    If Err.Number <> 0 Then
        FileAlreadyOpen = True
        Err.Clear
        'MsgBox "Error #" & Str(Err.Number) & " - " & Err.Description
    Else
        FileAlreadyOpen = False
    End If
    On Error GoTo 0
End Function
