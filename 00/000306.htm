Just moving to a new record in the subform will display it at the top of the window, hiding the existing records above it. To display 2 existintg records plus a new record, I'd create an event procedure for the On Current event of the main form:

 

Private Sub Form_Current()
    On Error Resume Next
    DoCmd.Echo False
    Me.sfCaseNotes.SetFocus
    With Me.sfCaseNotes.Form
        .SelTop = .SelTop - 2
    End With
    Me.SomeControl.SetFocus
    DoCmd.Echo True
End Sub

 

where SomeControl is a control on the main form.

Downside is that the focus in the subform will not be on the new record, but on the next to last existing record.

    greg wilson 2 say:

    Hi Owen,

    Be advised that this was hastily written and tested minimally.

    Errors in Code:

    1)  As E.J. Gun pointed out, you cannot declare a variable as Public in a class module.

    2)  The word "Save" is a reserved word in VBA. You need to rename the routine. I suggest "SaveBackup".

    3)  Most glaring is the statement "Application.Run ("Auto_Save")" in the Save() routine. If it actually worked then you would have a non-terminating recursive loop. In other words, the Auto_Save() routine calls the Save() routine, which in turn calls the Auto_Save() routine, and so on, forever...

    4)  I never use the Application.Run method. As far as I know, it is used to call routines in other workbooks. In your case, I fail to see how recursive code would ever be needed. If for some reason it is needed then you only need to state the routine or function name, optionally with the "Call" statement preceeding, e.g. "Auto_Save" or "Call Auto_Save".

    5)  The Save() routine is actually declared as "Function Save() As String". It does not return a value, it only executes actions. If I renamed it to "SaveBackup" in order to avoid using a reserved word, then It should be changed to "Sub SaveBackup()" instead.

    5)  You use the variable FormatDate in the SaveCopyAs call, but FormatDate is not declared nor given a value.

    6)  I don't like the name "Auto_save". Prior to xl97 the "Auto_" prefix was used to execute some routines on workbook open instead of using "Workbook_Open". This is obsolete. Using it is confusing if nothing else.

    7)  I don't see a reason to have the code in the ThisWorkbook class module. I'd put it in a standard module unless there is a reason not given.

    Suggested Code:

    Const backupfolder As String = "C:\excelbackup"

    Sub AutoSaveBackup()
    If Dir(backupfolder, vbDirectory) = "" Then MkDir backupfolder
    Call SaveBackup
    End Sub

    Sub SaveBackup()
    Dim FormatDate As String, Formattime As String
    Dim msg As String, wbnm As String

    FormatDate = Format(Date, "dd-mm-yyyy")
    Formattime = Format(Time, "hh-mm-ss AMPM")
    Application.DisplayAlerts = False

    With ActiveWorkbook
        wbnm = FormatDate & " " & Formattime & " " & .Name
        .SaveCopyAs Filename:=backupfolder & "\" & wbnm
        .Save
    End With
    Application.DisplayAlerts = True
    msg = "Backup of workbook successful." & vbCr & vbCr & _
    "Workbook saved to directory:" & vbCr & backupfolder & _
    vbCr & vbCr & "Workbook saved as:" & vbCr & wbnm
    MsgBox msg, vbInformation, "Workbook Backup"
    End Sub
