graham mayor say:

As others have said this is not entirely straightforward, but it helps to start with a new document based on the original document. I suspect the following macro is as close as you can get without knowing what is on the 'page'.

Note that Word is not a page layout application. There are no 'pages' in a Word document, just flowed text and a variety of disparate elements to make up the whole.

http://www.gmayor.com/installing_macro.htm   

Sub CopyPageToNewDoc()
Dim oSource As Document
Dim oTarget As Document
Dim oRng As Range
    ActiveDocument.Save
    If Len(ActiveDocument.Path) > 0 Then
        Set oSource = ActiveDocument
        Set oRng = oSource.Bookmarks("\page").Range
        WordBasic.DisableAutoMacros 1
        Set oTarget = Documents.Add(ActiveDocument.FullName)
        oTarget.Range.FormattedText = oRng.FormattedText
        oTarget.PageSetup.DifferentFirstPageHeaderFooter = False
        oTarget.Sections(1).Headers(wdHeaderFooterPrimary).Range.Text = ""
        oTarget.Sections(1).Footers(wdHeaderFooterPrimary).Range.Text = ""
        WordBasic.DisableAutoMacros 0
    End If
lbl_Exit:
    Set oSource = Nothing
    Set oTarget = Nothing
    Set oRng = Nothing
    Exit Sub
End Sub

As far as the white space on some of the edges of a pdf, the issue was that a custom page size (10.9" x 8") was being used and the print to pdf facilities, such as Microsoft Print to PDF and PrimoPDF, do not, as installed, provide for the setting of custom page sizes, in-spite of, in the case of the PrimoPDF printer, it  displaying a "Postscript Custom Page Size" in the list of page sizes, but there is no way, via the user interface to specify the dimensions of that page size.

While it is not for the faint-hearted, there are instructions at:

for modifications that can be made to the Microsoft Print to PDF driver to allow the definition of a custom page size.

Note: Once you get over the hump of modifying the Microsoft Print to PDF Driver, so that it will accept new form sizes defined via the Control Panel>Devices and Printers dialog, that is probably a simpler method of dealing with the issue.  The "hump" is that you need to change some security settings for the folder that contains the Printer Driver to be able to save a modified copy of it into that folder.
