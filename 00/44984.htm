wunders say:

Windows preserves case in file and folder names, but when accessing files and folders, it is case insensitive (ie Upper/Lower Case does not matter).

 

Profile folders changing from a straight user name to a user name followed by a dot and a domain name, on the other hand, happens as the result of a corrupted user profile.  Corrupted user profiles usually occur as a result of (or is accompanied by) file system corruption.  File system corruption can cause the capitalization issues you notice.  To recover from a corrupt user profile, choose your method from the following:

 

"How to recover the damaged user profile in Windows XP"

  < http://support.microsoft.com/kb/555473 >

 

"Windows XP Problems if your Profile is Damaged"

  < http://support.microsoft.com/kb/326688 >

 

"How to copy data from a corrupted user profile to a new profile in Windows XP"

  < http://support.microsoft.com/kb/811151 >

 

Since file system corruption is highly indicated, you should also perform a Disk Check (chkdsk) operation with the "/R" or "Repair" option to ensure your file system is in good shape:

 

"How to perform Disk Error Checking in Windows XP"

  < http://support.microsoft.com/kb/315265 >

slance310 say:

Hi,


This what I had to do to get my X5340 to work....


1) Printer must be On for it to show up in Devices and Printers after the Upgrade - Just this one time.

    Otherwise, only the Lexmark Fax shows up and the Printer can't be REMOVED.


2) Now UNHOOK the Printer


3) REMOVE Lexmark Printer and Lexmark Fax in Devices and Printers.


4) STOP the PRINT SPOOLER.


5) Uninstall software - TWO Firewall Security Messages Pop Up about "ALLOWING' Printer Communication..


Select "ALLOW".... if you Allow both Shared and Private on both windows, you end up with two printers in Printer Preferences, Properties, etc. I unchecked a "Shared" on the first window.


6) After REBOOT, STOP Print Spooler AGAIN.


7) Reinstall Software\Drivers ... Hook Printer back up when Prompted.


8) Unhook Printer again and REBOOT ... This step isn't necessary if Bidirectional Support is already ENABLED.


Skip to step 10 if Enabled.


9) Enable Bidirectional Support and Reboot...

To do this, go to Devices and Printers and select "Printer Properties" and then the "PORTS" tab... The check box is near the bottom.


10) Set up Printer Preferences if desired, and it's ready to go.
