Cause

A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high.

This bug check is usually caused by drivers that have used improper addresses.

If the first parameter has the same value as the fourth parameter, and the third parameter indicates an execute operation, this bug check was likely caused by a driver that was trying to execute code when the code itself was paged out. Possible causes for the page fault include the following:

  • The function was marked as pageable and was running at an elevated IRQL (which includes obtaining a lock).
  • The function call was made to a function in another driver, and that driver was unloaded.
  • The function was called by using a function pointer that was an invalid pointer.

- You cannot stop recovery from booting to the X: drive.  Recovery is designed to boot to X: because the hard drive isn't working from some reason.  You can use the Command Prompt to switch drives by typing the drive letter followed by a colon (:) and pressing Enter.  Also, this won't be possible unless you've loaded the RAID drivers and have a RAID array that's visible to those drivers (and the OS that you've booted into).

Then how can I reset the Msconfig options for windows 7 (Boot tab) without entering windows?
- You have to be able to:  access the hard drive, then mount the registry hives using the installation that you've booted into.  Then edit the registry settings that apply to MSCONFIG.  Once that's done, then you have to merge the modified registry hives into the registry that's on the hard drive.
