RDP connection fails from Microsoft account to Local Admin account

Issue:  

  • Windows 10 Pro desktop cannot connect to another Windows 10 Pro desktop on the same wired network.

Details:

  • When connection is attempted, an error states the credentials are invalid.
  • The credentials entered are the local admin account on the target computer.
    • Computer attempting the connection is logged into the Microsoft account.
  • When both computers were running Windows 7 Pro - there were no RDP issues.

Troubleshooting:

  • Confirmed credentials are correct and CAPS is turned off.
  • Firewall rule for Remote Desktop is enabled and option is enabled in the System Properties.
    • Connection to target computer is successful using a Samsung Galaxy Tab S that is connected to the WLAN.
  • Downloaded Remote Desktop app from the Windows Store.
    • Encounter the same "Invalid Credentials" error as the Remote Desktop Connection software.
  • Updated firmware for router (Netgear R6300).
  • Power cycled both desktops.

Resolution:

  • Logging into the Microsoft account on the remote computer resolved the issue.

Conclusion:

  • When you're trying to establish a Remote Desktop Connection from a computer that is logged into your Microsoft account to a computer that is logged in to the Local Admin account, the connection will fail stating the credentials are invalid.

zigzag3143 (ms -mvp) say:
I do not recommend reverting as I have seen far too many  go wrong.

Reverting can only be done if you installed within the last 30 days (if you created a new user it must be removed before you revert)

To revert go to all settings>update & security>recovery>roll back to a previous OS

This will only work if the windows.old, and 2 hidden folders ($Windows.~WS & $Windows.~BT) are still present.  If it is not you will have to re-install.

You may simply be better off fixing the win 10 issues.
