stevemcc2.99 say:

Hello Again Frederik.  

As a result of your offer to look at the corrupted file, I stumbled upon the root cause.  Thanks! 

The Short Story: 

When there's a comma somewhere in the name of a folder in the path to the target destination, Microsoft Print to PDF will create the user-specified filename at the specified target location but the file size will be 0KB.

The Long Story:

When I attempted to attach the corrupted files to the email that I was about to send to you per your offer, GMail refused to attach the files because they were "0KB".   So I figured you want more test data.  I anticipated you might ask about how many nested folders I had in my Documents file structure to get to the target folder.  My troublesome target folder was 3 levels under Documents (where Documents was in the root of "This PC".  I set up four nested folders under Documents with simple test names.  Result:  Microsoft Print to PDF worked for every level.  That led me to scrutinize my original troublesome target folder.  I saw an obvious difference from my simply-named nested test folders:  my original target folder included a comma in the name.  When I removed the comma, Microsoft Print to PDF worked!!!

This explains why printing to the desktop always worked:  there wasn't a component in the path that had a comma in the name.  My issue was with the folder name, but I also confirmed that a comma in the actual filename caused the same problem.

So thanks for your offer to help out.  Without your simple request to send files, I may never have discovered (or investigated) the file size characteristic.  One thing led to another to discover the comma difference.  With this "discovery", I added the word "comma" to my prior-unsuccessful Google searches about this issue.  I found others had already found the root cause, e.g. as seen here. 

Best Regards and thanks again for the motivation.

DMZ hat rein GARNICHTS mit Streamingdiensten oder Bandbreitennutzung zu tun. Da die Konsole bei einem Videostream aktiv "vom Inneren" Deines Netzwerks "hinaus" ins Internet die Daten des Videos anfordert, weiß der Router von welchem Gerät die Daten kommen, und kann die ankommenden Daten korrekt weiterleiten.

Lediglich bei Peer2Peer Verbindungen wie Multiplayer oder Chats, bei denen plötzlich "fremde" Konsolen von "Außen" zu Deiner Konsole verbinden wollen ohne dass die Konsole dem Router explizit gesagt hat, dass sie eine Verbindung mit der fremden Konsole haben will, bräuchstest Du eine Portweiterleitung oder DMZ.

Portweiterleitungen und DMZ verbessern aber nicht den Ping oder die Bandbreitennutzung!

[quote user="Dennis Lab"]

...Ich kann die nur sagen, dass alle meine Geschwindigkeitstests via LAN am Notebook bei ca. 106 Mbit/s landen. Ping 16-26

Nur beim Xbox Verbindungstest gibt es Probleme (paketverlust 2%, Latenzzeit manchmal 200)

[/quote]

Das liegt daran, dass Du bei Speedtests wie z.B. Speedtest.net weltweit Deinen nächsten Server suchen kannst. Und manchmal wird der Speedtest Server auch von Deinem Internetprovider direkt gehostet, so dass Du eigentlich nicht ins "fremde" Internet musst, sondern quasi im "LAN" Deines Internet-Providers diese "Ludicrous Speed

" erreichen kannst.

Die Xbox scheint aber für ihre Speedtests in den Netzwerkeinstellungen nur US Server zu verwenden, mit entsprechenden Laufzeiten, Pings und Paketverlusten.
