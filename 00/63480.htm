Not sure if you have already resolved the issue; but wanted to share my solution anyway which I found somewhere and lost the link.

I had an issue when the WiFi would connect but not work. Airplane mode would be ON, and when I attempt to turn it off it would toggle back to on. THis is on a desktop system (not a laptop; custom built, with windows 10)

In my case the culprit was the UPS connected to the machine with the default driver.

To verify - disconnect UPS and reboot - the whole Airplane mode will be gone and wifi will work properly.

To fix - install UPS driver from the UPS manufacturer - in my case I had APC Smart UPS 750 connected, I've installed PowerShute business edition agent. Reconnected UPS and rebooted - no airplane mode and all still works fine.

Speculation:
Apparently when the system sees UPS without the third party driver attached it think's it's a laptop and optimizes and configures itself in a mobile mode. Installing third party driver causes the ups port to be claimed by the third party service which prevents windows from making assumptions.

andre da costa say:
If you add a secondary language to Windows 10 version 1803 that shares the same base language as the Windows display language, neither language can be removed.

For example: If English (United Kingdom) is the current Windows display language, and you then add English (India) to the Preferred languages list, afterwards neither English (United Kingdom) nor English (India) can be removed from the Preferred languages list.

Note: The most notable effect of having multiple related languages in the Preferred languages list is additional entries in the text input selector. You can remove unwanted language list entries using these steps:

1. From the Start menu, search for powershell.

2. Right-click Windows Powershell in the search results and choose Run as administrator.

3. Type or copy/paste the following command at the command prompt, and then press Enter.

Get-WinUserLanguageList

After typing this command, make note of the “LanguageTag” listed for the language you wish to remove, as you will need it in the next step.

4. Type or copy/paste each of the following commands at the command prompt, and then press Enter. In the following commands, instead of typing <languagecode>, replace it with the language tag you wish to remove, such as en-in for English (India).

$LangList = Get-WinUserLanguageList
$MarkedLang = $LangList | where LanguageTag -eq "<languagecode>"
$LangList.Remove($MarkedLang)

Set-WinUserLanguageList $LangList -Force

A clean install of Windows 10 on fully supported hardware always works correctly the first time. Anything other than that is a gamble.

If you upgrade your operating system, you may or may not be successful, and even if you seem to be successful at first, your operating system may suffer later. ('You' here doesn't refer only to you personally.) This is true regardless of what operating system is installed and regardless of how technically experienced you are.

That's why people who really understand technology always prefer a clean install. However, a clean install demands a great deal of technical knowledge because you are starting from scratch. Most people have no reason to possess that kind of technical knowledge.

An upgrade to Windows 10 on fully supported hardware only works correctly if your previous operating system and all installed software are setup and maintained only in ways that are supported by the software developer. That's not the case for a great many people, who do all kinds of unsupported things to their computers. Upgrading a computer like that is totally unpredictable.

In your case, the best thing to do is to show your computer to a professional. If that person is able to diagnose what's going wrong and fix it, great. If they can't accurately diagnose the problems, or if there are too many problems to fix, then the best thing to do is to backup your personal data and do a clean install.
