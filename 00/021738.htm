bealers say:
Welcome to OneNote, it's awesome.

Your own workflow is of course very personal and it's hard to comment on the 'right' way to do things.

It's ok to create a page with just a few lines of text, disk space is cheap and OneNote indexing is blazingly fast. If you wanted to create a section group per month, say, and a section per day with pages per thought/idea/screen grab/whatever then this  of course totally fine and would certainly help you if you want the search results to provide you with just that bit of text.

You've already mentioned tagging. If you have not already then I'd definitely suggest creating your own custom tags and tagging key bits with your own contexts then using the "Find Tags" pane to quickly access things.

If you want to datestamp everything that you write then these three short-cuts may help:

Insert today's date. ALT+SHIFT+D
Insert the current date and time. ALT+SHIFT+F
Insert the current time. ALT+SHIFT+T

There are loads of other useful shortcuts but my favourites are:

  • Win S (Win + Shift + S in WIn 8.1) to screen grab. I've set my Onenote up to not ask me what to do with the screen grab, just immediately file it in my Inbox
  • Ctrl + . (bullet)
  • Ctrl + - (strikethrough)
  • Ctrl + Enter (in a table, add a new row at the cursor)
  • Shift + Alt + then the arrow keys. (move a block of text around, including re-orderng it in a list, table and changing the indent amount)
The best end user group that I've found for OneNote is the Linkedin one. it's not high traffic but there's quite a few people on there, mostly professional users.

Finally, and maybe more pertinent to your question, my Notebook layout is briefly described below, this is what works for me but as I say, it's a very personal thing.

Notebook: Me (I have this open always, it's the first thing I do as I start my day)
Section: Inbox
Page: Process (random thoughts go here)
Page: This week
Page: Monday
thru to 
Page: Sunday

Note: this section is also set-up in OneNote settings as the default location to send *anything*, so screen grabs go here as does anything else. I then regularly move these 'for filing' pages elsewhere.

Section: Someday
*Not yet projects, but might be one day

Section: Lists
* Books to read, films to watch, articles to write, etc.

Section: Agendas
* Meeting stuff, mostly upcoming. Post meeting they get filed elsewhere.

Section Group: Projects
*Personal projects, one per section. Everything from house maintenance to what to buy my wife for Christmas.

Notebook: A really Big Project
* something that is taking up a lot of my time and needs lots of information storing

Notebook: Work: Private
* work projects, one per section (or section group if big) this is not shared

Notebook: Work: Shared
* Things I share with my team. Project documentation and the like.

Notebook: Reference
* An A-Z index of sections (or section groups) of anything and everything that needs filing.

Notebook: Archived
* I drag my daily journal into here organised by Year (section group) and month (section). Some days the page(s) may only have 1 piece of text, other days it might be lots.

tom robinson in vancouver say:

First of all, I'm assuming you are referring to VBA code, and not to "classic" macros, which I do not use.

Access files sometimes get corrupted, so that any attempt to open the VBA editor causes Access to crash. This happens to me every few months. Internally, there are actually two versions of all your VBA code: Source code and Compiled code. (in simple terms!) When the Compiled version gets corrupted, recompiling from Source will fix it. Except when Access crashes before you get the chance.

That's where the /decompile switch will help. Use it like this: (adjusting for your Access version)

"C:\Program Files (x86)\Microsoft Office\Office14\MSACCESS.EXE" /decompile "c:\myproject\This one crashes.accdb"

This will load your project into Access, discarding all the compiled VBA code. Then in the code editor, do Debug/Compile then Save. This works for me, 80% of the time. 

I've tried Bill Mosca's suggestion of importing into a new database. Sometimes that works, and sometimes Access still chokes on the modules which contain corrupted code. It will certainly let you salvage all the Access objects which aren't corrupted.

...Tom

rohn007 say:

From the Keyboard you can use <CTL><SHF><8> to toggle on display non-printing characters On and off. Or you can mouse to the Home tab / Paragraph group / the "backward" P icon.

Justified means that Word forces left and right margins to be flush, straight line. Unfortunately, Word is all thumbs when it comes to doing this. It always has been.

There are 2 compatibility options you should also check in File tab / Options command / Advanced option / Compatibilty section:

WordPerfect (WP) has "always" done full justification the way that old style manual printers did, by "intelligently" expanding and contracting spaces between letters AND word. Word takes the simple, brute force approach of only adding full spaces between words. This makes the word appear too far apart. This can be controlled by turning on the following option in the document:

Do full justification like WordPerfect 6.x for Windows: To achieve full justification, WordPerfect compresses the spaces between words while Word expands them. This often results in different line breaks and leads to different page breaks.

When Justification is turned on, Word will grossly expand short lines that end early using th <SHF><ENTER> soft return character. Turning on the next option will prevent this

Don't expand character spaces on the line ending Shift-Return: When a "justified" line of text ends with SHIFT+ENTER, Word prevents extra spacing between words and makes the line appear as "Align Left".

In File tab / Options command / Display option: In the "Always show these formatting marks on screen" pick the nonprinting characters you want to see. Personally, I go with "Show all formatting marks". 

To tell if a para is "justified" you want to check the paragraph alignment. The visual way is to look a the Home tab / Paragraph group to see which of the alignment icons is highlighted (orange on my machine) to indicate that it is turned on. Another way is with the <SHF><F1> shortcut. This toggles display of the "Reveal Formatting" pane. Look in the Paragraph section, under "Alignment".

When you find a para with the justified option turned on, check to see if it has the same, expected, style applied. Justification may have been manually applied to override the style based justification.  You can try using <CTL><Q> shortcut to remove manual formatting applied to the paragraph (like you can use <CTL><SHF><Spacebar> to remove manual character formatting).

jack cook - aka help_line say:

Set up My Family on the Windows Phone website

 

My Family on the Windows Phone website is the way to turn on app and game downloads for a child's phone. If the child has a Windows Phone 8, you'll also be able to manage their app and game download settings, including game ratings, on the My Family page.

To set up My Family, you'll need to sign in with both the child's and the parent's Microsoft account email address and password. Then you can accept the Windows Phone Store Terms of Use so the child can download apps.

If you want the child to be able to download Xbox games as well, you'll need to go to the Xbox website and accept the Xbox Terms of Use on their behalf. This is separate from the Windows Phone Store Terms of Use.

Running into issues? See My Family FAQ for answers to some common setup questions.

Note

My Family is only available for Windows Phone 8.

To set up My Family so a child can download apps and games

  1. On your computer, go to My Family on the Windows Phone website.

  2. Sign in using the parent's Microsoft account email address and password. If the parent doesn't have a Windows Phone, that's fine—you can still set up My Family. If the parent doesn't have a Microsoft account, you'll have a chance to create one.

  3. On the My Family setup page, click Get started.

  4. To add the child to My Family, click Go. On the sign-in page, enter the child's Microsoft account email address and password. If the child doesn't have a Microsoft account, click Sign up to create one.
