So going to Start / Control Panel / Folder Options / Views / and checking show hidden and system files does not resolve the problem?  That's strange.  And these hidden files are only located on your D-Drive, not your C:\ drive?  Is your D:\ Drive your system recovery partition drive (as provided by your manufacturer)?  If so, this may explain the problem as that drive is NOT to be used in any way except to do a restore to factory settings if that ever becomes necessary.

What kind of data does this folder contain (doccuments, programs, music, videos,...) or does it contain a variety of different types of files?  What are the extenstions on these files (what file types are they) - are they from some special program (if so, what program and what version) or are they "normal" extensions?  Please list some of them here so we can understand better.  Does this happen with ALL folders stored on this drive or only certain ones?

Sometimes certain file types don't show up in Windows Explorer for some reason (I still haven't figured it out entirely but it seems to be from older programs or file types that do not have associations).  This might explain some of your problem but not if the entire drive is not visibile in Windows Explorer (but the folders can be viewed with a Command Prompt).

Here's the general procedure to show hidden files in Vista from: http://www.bleepingcomputer.com/tutorials/tutorial62.html.

Windows Vista

To enable the viewing of Hidden files follow these steps:

  1. Close all programs so that you are at your desktop.
  2. Click on the Start button. This is the small round button with the Windows flag in the lower left corner.
  3. Click on the Control Panel menu option.
  4. When the control panel opens you can either be in Classic View or Control Panel Home view:

    If you are in the Classic View do the following:
    1. Double-click on the Folder Options icon.
    2. Click on the View tab.
    3. Go to step 5.

    If you are in the Control Panel Home view do the following:
    1. Click on the Appearance and Personalization link .
    2. Click on Show Hidden Files or Folders.
    3. Go to step 5.

  5. Under the Hidden files and folders section select the radio button labeled Show hidden files and folders.
  6. Remove the checkmark from the checkbox labeled Hide extensions for known file types.
  7. Remove the checkmark from the checkbox labeled Hide protected operating system files.
  8. Press the Apply button and then the OK button and shutdown My Computer.
  9. Now Windows Vista is configured to show all hidden files.

I know I haven't given you very much assistance here (unless the problem is simple which it probably isn't).  This could also be caused by problems with your system files.  Let's check them:

Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator (you can skip this step if using the disk).

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.  Try to post any corrupted files here so we can see if they can be repaired with good copies from the installation disk (unless there are too many).

While in Command Prompt, type chkdsk /f /r and enter and let it run.  It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a potential cause.

If that doesn't work, then please post any repeating error messages from the Event Viewer concerning Windows Explorer (Start / Control Panel / Administrative Tools / Event Viewer – or typing eventvwr from the command prompt).  Here's how to use Event Viewer: http://www.bleepingcomputer.com/forums/topic40108.html.

If that doesn't work, we'll need to do a system repair/upgrade using the genuine Windows Vista Installation Disk (one you own or one you can borrow from ANYONE).  Here's the procedure: http://www.vistax64.com/tutorials/88236-repair-install-vista.html.  Although this will not affect your data, settings, or programs, you should still backup your data before starting just to be on the safe side. If the other procedures don't work, THIS one is almost certain to work.  You may have a lot of updates to re-install (including any service packs you had to remove).  If the version on the system came with SP1 or SP2 pre-installed and the disk is an earlier version, then you'll need to make a slipstream disk as follows: http://www.vistax64.com/tutorials/151606-vista-sp1-slipstream-installation-dvd.html.  

Hopefully one of these procedures will resolve your problem.  If not, post back and we'll try something else.  Post back in any case so we'll know how it turned out.

Good luck!

elstaci say:

Seems like a bug in windows 10 concerning DISM. I have the same issue. took me hours and days to figure a round about way to repair corrupt files. 

First: go to windows folder and subfolder LOGS. Open DISM folder and delete the dism log.

second: Run dism /online /cleanup-image /scanhealth. 

Third: go back to windows folder and subfolder LOGS and open the DISM log. Go to the bottom. There it will show you what files are corrupted. copy all the files names if more than one.

Fourth: Unhide hidden system files on C: drive. Go to ESD folder /sources and locate install.wim or boot.wim.  Use 7zip to open the archives. I couldn't open Install.wim but I could open Boot.wim. Inside boot.wim I found the file that was corrupted on my main drive. Copied the file to documents or desktop.

fifth: Went back to my Windows folder. Located the corrupted file. Changed the Permissions on it so that I could replace the corrupted file with the new uncorrupted file from Documents or Desktop. Replaced it.

sixth: Ran SFC /scannow now - came back all good Ran DISM Restorehealth, came back all good. 

It isn't easy but I was able to replace the corrupted file which was .ttf Font file in Seguoi (WinSxS Folder).

Now I get good SFC and DISM repairs. Until Microsoft fixes the DISM problem this is the way I will manually repair my computer in the future when hopefully DISM will be working a
