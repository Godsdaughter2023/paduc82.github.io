Hi Richard,

Thank you for posting your query on Microsoft Community forum.

I appreciate your efforts taken to resolve the issue.

The issue may occur due to one or more of the following reasons:

  • This issue can occur if you have the Delete browsing history on exit box checked in your Internet options.
  • If your hard disk is getting full, you may need to free up disk space.
  • The number of days that Internet Explorer keeps a history of the webpages you have visited may be set to 0 or a low number. After that period of time, the history is automatically deleted.
  • Your history may be corrupt or locked by another process.
  • Registry keys may have become corrupt or modified.
  • Your user profile may be corrupt.

In order to troubleshoot this issue, I would suggest you to try the following methods and check if they are helpful (Ignore the methods, you have already tried).

Method 1: Check to make sure the Delete browsing history on exit box in your Internet Options is not checked.

  1. Open Internet Explorer.
  2. Click the Tools button , and then click Internet Options.
  3. Under the General tab, and then under Browsing History, make sure the Delete browsing history on exit box in not checked.

If the issue still exists, try the next method.

Method 2: Check to see if there is enough free space on your hard disk

  1. Open Internet Explorer.
  2. Click the Tools button , and then click Internet Options.
  3. Under the General tab, and then under Browsing History, click Settings. Under Disk space to use, make sure the disk space is set to 250MB, and under History, make sure the Days to keep pages in history is set to 20.

If the issue still exists, try the next method.

Method 3: Delete your temporary Internet files
It’s a good idea to delete temporary Internet files periodically to free drive space.

  1. Open Internet Explorer.
  2. Click the Tools button , and then click Internet Options.
  3. Under the General tab, and then under Browsing History, click Delete.
  4. In the Delete Browsing History dialog box, select Temporary Internet files, Cookies, and History, click Delete, and then click OK.  

    Your changes will take effect the next time you open Internet Explorer.

If the issue still exists, try the next method.

Method 4: Reset Internet Explorer settings

Try to reset the Internet Explorer settings and check if the issue persists.

Change or reset Internet Explorer settings

Other possible workarounds include:

 1. Export your Excel sheet to .csv format and use that as a data source

 2. Select your data, Edit->Copy Edit->Paste into a new Word document, save, close, and use that as the data source.

The basic problem is that for a very long time there has been no single, simple, reliable way of using an Excel data source as a Mail Merge Data Source, and the problem has apparently never been high enough on anyone's list to do anything about it.

The basic concept here is if NOTHING is done then ZERO code and validation occurs.  This issues much holds true in MOST of the computer industry and has been this way for 20+ years.


In other words if you tab through a text box then MOST of the events in regards to data do NOT fire. I mean if you did not change anything in a text box then MOST developers would not expect the before change event to fire since you did not change anything!!! So this makes much sense.


So you DO SOMETHING WHEN SOMTHING has changed!!! This is how computers work in general!

I mean if you are looking at a form and don’t do anything then events don't fire!  And if you don't change anything then again events in general don't fire.


Of course when the data is about to be saved then the forms BEFORE change event does fire and additional validation can be placed in that part of the form to give "friendly" messages to the user to this fact.


Keep in mind that a user might UNDO and NOT commit their changes in a form. A user might edit, and THEN un-do.  This means that NONE of those validation events needed occur because you DO NOT KNOW that the user going to save the record!


As pointed out then for REQUIRED columns simply set that column as required in the table (you thus not have to write code for these cases).

This has the VERY interesting effect that if I edit a control that is required, tab to next control and THEN go back to previous control and EMPTY the control NOW the control has changed and the events that CHECK for empty can now fire!  (so if you *empty* a control then some message can occur - but not tabbing thought the control without any change made).
