grschinon say: I think you'll find it's a hardware fault with the ambient light detector.

There are a few things you can do to find out if the problem is hardware or software-related.

1) Try a simple restore factory default settings. This won't delete any data of yours, you'll juat have to redo your profile, theme, wallpaper etc. settings again. Just "dial" *#7780# from standby and supply the security code when prompted. The security code will be 12345 unless you (or the previous owner!) have changed it.

2) Next try a soft reset. This will reformat your phone's C: drive so you'll want to back up any contacts, calendar events, messages etc. that are stored there beforehand. Either do a backup to the memory card or use PC Suite to back data up onto your computer. Enter *#7370# from standby and provide the security code when prompted.

3) If the soft reset doesn't work or if you don't have the security code then you can always try a hard reset (which will also delete your data so back up first):
i) switch the phone off.
ii) press and hold the '3', '*' and green 'call' keys
iii) with these keys still pressed down, power the phone up
iv) release the 3 keys only once you see the phone show signs of life such as the "handshake" logo or the language selection screen

4) If none of the above work then try to update the phone's firmware. That will clear everything out including the phone's flash ROM and you'll be starting with a clean slate.

The above operations, if unsuccessful, will rule out a software issue. If the phone still has the problem then it's a hardware issue that only your nearest Nokia Care Point can deal with.

ken sheridan say: It would be simpler to forget the subform and reference the underlying data itself via a recordset object, but doing it your way you could open a form or report based on the unrestricted final query and filter it on the basis of the string expression.  However, the expression doesn't appear to include the column name.  It would be simpler to build a delimited value list:

If the value is a number data type:

  For pkIDL = 1 To numRecs
        selICList = selICList  & "," & Me!MatlSelectedSubform.Form.ITEMCODE
        recSet.Bookmark = Me!MatlSelectedSubform.Form.Bookmark
        recSet.MoveNext
        If (Not (recSet.EOF)) Then
            Me!MatlSelectedSubform.Form.Bookmark = recSet.Bookmark
        End If
    Next pkIDL

    ' remove leading comma
    selICList  = Mid(selICList ,2)

    DoCmd.OpenForm "SomeForm", WhereCondition:="ItemCode In(" & selICList & ")"       

If it's a text data type amend it as follows:

    selICList = selICList  & ",""" & Me!MatlSelectedSubform.Form.ITEMCODE & """"
