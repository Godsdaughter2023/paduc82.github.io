You will have to use VBA for this purpose. This can not be accomplished through native Excel functionality. The only limitation of below approach is that you will have to give a button to call this macro for printing, this can not be utilized through Excel's Print Button. ( Edit - Note - The subsequent reply gives the functionality through Print Button also)

1. Make a backup of your workbook. 
2. Open your workbook and ALT+F11
3. Locate your Workbook name in Project Explorer Window
4. Right click on your workbook name > Insert > Module 
5. Copy paste the Macro code given
7. Save your file as .xlsm

'*** Macro Starts. Run it through a button ****

Sub PrintDiffFoot()
    ActiveSheet.PrintOut From:=1, To:=2
    ActiveSheet.PageSetup.CenterFooter = "&P-2 of " & ActiveSheet.PageSetup.Pages.Count - 2
    ActiveSheet.PrintOut From:=3
    ActiveSheet.PageSetup.CenterFooter = ""
End Sub

roger carlson access mvp say:

You need to be aware of what actually happens when you Compact.  Access creates a new database called Database.mdb (in earlier versions it was db1.mdb), copies the table structures, then copies the data over (removing extra space as it does).  After the transfer is complete, it deletes the old database and renames Database.mdb to the original name.  Depending on when the compacting process is interupted, you may be left with just the Database.mdb file.

One solution is to use VBA to compact your back-end from the Front-end.  That way, you can control the process.  When I do this, instead of deleting the original file, I'll rename it to something else (generally appending the date and time to the file name) so I save the original.

On my website is a small sample database which illustrates this: CompactBackEndDB.mdb.
