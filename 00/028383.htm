tushar mehta say:

You will have to experiment to check if the Application.DisplayAlerts property helps you.

That said, if you set links to automatic update, the use of .UpdateLinks should be redundant.  So, my guess is you have found a bug.  Essentially,

* If links are set to automatic update, UpdateLinks is not needed.

* If links are set to manual update, UpdateLinks should cause links to be updated.  If it doesn't it serves no purpose.

You may want to use the following *untested* workaround.  I adapted the code from the PPT VBA help for the LinkFormat.AutoUpdate property.

For Each sld In ActivePresentation.Slides For Each sh In sld.Shapes If sh.Type = msoLinkedOLEObject Then 'If sh.OLEFormat.ProgID = "Excel.Sheet" Then 'sh.LinkFormat.AutoUpdate = ppUpdateOptionManual 'End If sh.LinkFormat.Update End If Next Next

ken sheridan say: What is the data type of the DuesYear column, and in what format is the data stored?  If the year is defined as the calendar year in which the accounting year ends, I would expect an integer number data type with values in the format 2018 for the current accounting year.  If, on the other hand the accounting year is entered as 2017-18, which is the convention generally used here, the data type would have to be text.  In either case the parameter should be declared, e.g. if the year is stored as 2018:

PARAMETERS [Enter accounting year:] SHORT;
SELECT *
WHERE PaidForLife = TRUE OR (Paid = TRUE
AND  DuesYear = [Enter accounting year:]);

Or if the year is stored as 2017-18

PARAMETERS [Enter accounting year:] TEXT(7);
SELECT *
FROM MyTable
WHERE PaidForLife = TRUE OR (Paid = TRUE
AND  DuesYear = [Enter accounting year:]);

Note that the parentheses around the AND operation are important to force it to evaluate independently of the OR operation

The other way in which accounting years are sometimes defined is by the start date of the accounting year as a date/Time data type.  The UK tax year for instance starts on 5th April, so the current tax year would be defined as year starting #2017-04-05#.  To restrict a query by accounting year the following function could be used to return the accounting year in the format 2017-18:

Public Function AcctYear(DateVal As Date, MonthStart As Integer, DayStart As Integer) As String

    Dim dtmYearStart As Date
    
    If MonthStart = 1 And DayStart = 1 Then
        ' accounting year is calendar year, so return single year value
        AcctYear = Year(DateVal)
    Else
        ' get start of accounting year in year of date value
        dtmYearStart = DateSerial(Year(DateVal), MonthStart, DayStart)
        
        ' if date value is before start of accounting year
        ' accounting year starts year previous to date's year,
        ' otherwise it starts with date's year
        If DateVal < dtmYearStart Then
            AcctYear = Year(DateVal) - 1 & Format(Year(DateVal) Mod 100, "-00")
        Else
            AcctYear = Year(DateVal) & Format((Year(DateVal) + 1) Mod 100, "-00")
        End If
    End If

End Function

This could be called in a query like this:

PARAMETERS [Enter accounting year:] TEXT(7);
SELECT *
FROM MyTable
WHERE PaidForLife = TRUE OR (Paid = TRUE
AND  AcctYear(DuesYearStartDate, 7,1) = [Enter accounting year:]);

Where DuesYearStartDate is a column of date/time data type.  The accounting year should be entered in the parameter prompt in the format 2017-18.

ken sheridan say: If you are trying to count all rows from the query, you can call the COUNT operator, and add a GROUP BY clause to the query, including all other columns in the clause:

PARAMETERS [Enter Fiscal year for Dues:] SHORT;
SELECT Member_Info. PaidForLife, Dues.Paid, Dues.DuesYear,
[Enter Reporting Month] AS [Report Month],
[Enter Reporting Year] AS [Report Year],
DCOUNT("MemberID","Member_Info") AS [Total Members],
COUNT(*) AS Paid
FROM Member_Info INNER JOIN Dues
ON Member_Info.MemberID = Dues.MemberID
WHERE Member_Info.PaidForLife = TRUE OR (Dues.Paid = TRUE
AND Dues.DuesYear = [Enter Fiscal year for Dues:])
GROUP BY Member_Info. PaidForLife, Dues.Paid, Dues.DuesYear;

In an aggregating query like this you cannot use the asterisk character in the SELECT clause to return all columns, however, so if you want to return any other columns from either of the tables you'll need to include them in the SELECT clause and in the GROUP BY clause.   Note BTW how I've used carriage returns to make the SQL more readable.
