In ref to items #1 & 2 - understanding is correct in both cases.

Now as for where your contacts are stored.....................actually all depends on your configuration but will start with the .ost part of your question. 

An OST file is an "offline" (mirror) store of the info that resides on your server the last time Outlook sync'd with the email account to which the .ost file belongs <so.........> if your contacts are on your server, those contacts will be in the contact folder located in the .ost file which brings us to the following questions

#1 - How many contact folders do you have in your profile? (by either looking at the <People> Nav Pane or the Full Folder Nav Pane list (use CTRL+6) to show the full folder list)

#2 - If you have more than one <folder tree> in the <Nav Pane> - means you have one or more additional data files in which case, which of those data files shows as the "default data file" for the profile (that's the data file that contains the default contact/calendar folders). To check that, from OL menu, goto File --> Account Settings --> Data Files tab (this will show all data files for the profile and indicate which one is the default).

#3 - When you open the AddressBook (via the OL ribbon) --> from AB menu --> Tools --> Options 

how many contact folders appear in the box under the "Custom" radio button? (that will be the same list that would appear when you open the AB and click on drop down list under "Address Book". If you have more than one and the default is opening up "empty" - select another folder to find your contacts. 

Only the contacts in the Outlook.com folder tree will sync with the Outlook.com account on the server.

#1 - If I delete 1 of them, they will all be deleted.

Just to get some basics out of the way in case it applies

#1 - Any contact created which has multiple valid email addresses and/or fax numbers will appear once in the AB for each email address and fax number (normal behavior)

#2 - Have a problem with this part of the description (contact may or may not appear in "People") - the Outlook AB is nothing more then a specialized "view" of the contact folders designated to be a member of the AB - so begs the question - how many contact folders do you have? Have never encountered a scenario where a contact exists in the AB but not as a contact item in a contact folder within the Outlook profile.

As an aside, contacts really should be created within the contact folder where you want it to reside versus via the AB dialog process

1. open up regedit (start --> run --> regedit)
2. navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Installer\Products
3. Search for "Microsoft Security Client" which should be a subbranch of the above directory

4. Right Click on that random string and select Export to back it up in case something breaks
5. Right Click on that random string and select Delete
6. Rerun the installer

This worked for me, and it will also work for any program that gives an error like this, not just Microsoft Security Essentials

It also happens when I open a new MS Word document and begin typing, the text starts off being crossed out. Does anyone know how I can stop this?
pesach shelnitz say:

You may have added strikethrough to your default Normal.dot template. If this is the cause, you can fix it by performing the following steps.

  1.  Open a new document.
  2.  On the Format menu, click Styles and Formatting, and then in theStyles and Formatting task pane, point to the Normal style, click the arrow to open the drop-down menu, and clickModify.
  3. Click the Format button, and on the drop-down menu, click Font.
  4. In the Font dialog box, clear the Strikethrough checkbox.
  5. Click OK.
  6. If you want to prevent something like this happening again, clear the Automatically Update checkbox.
  7. Click OK.
Hope this helps,
