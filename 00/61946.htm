¡firedog say:

Here's what I've learned so far...

This doesn't all quite tally with what I've learnt ...

When you install KB3093594, you're actually doing a Repair of Windows Live Mail. The process also upgrades WLMail from v16.4.3538.0331 to v16.4.3563.1216.

If KB3093594 was downloaded and installed before 17 December, it brought Windows Live Mail 2012 up to build 16.4.3563.0918. If it was installed later, then it brought the program up to build 16.4.3564.1216

If you somehow manage to uninstall KB3093594, doing so does NOT restore WLMail 16.4.3538.0331.

Depends how you uninstall it. I don't know what effect uninstalling KB3093594 in Windows Update has, but uninstalling patch d-Mail.msp (using msiexec) does restore 16.4.3528.0331.

"Installing" KB3093594 effectively 'kills' DeltaSync access on your end 

Not so. The update adds support for EAS. It does nothing to DeltaSync, IMAP, POP3 or NNTP. However, it does have two consequences for DeltaSync accounts: the drafts folder will no longer synchronize (because the version of EAS in use doesn't support it), and the display of Sent items in the message list is incorrect, probably because sent messages are now all sent by SMTP from the server end and subsequently downloaded to the client. This clearly wasn't accounted for in the new code, while the old code includes different display settings for incoming and outgoing messages. Sent messages thus seem to be treated as incoming for display purposes. The message list display can be corrected for messages already in the folder by dragging them to a different one and then back again.

Calendar sync doesn't work well for Outlook.com accounts, with empty calendars being added to the calendar list in circumstances yet to be nailed down. Performance is better for Outlook Mail accounts.

Contacts sync works as before for Outlook.com accounts. Categories do not sync with Outlook Mail accounts, but (intriguingly) do sync between instances of Windows Live Mail.

Existing DeltaSync accounts have to be removed and added back again after the update. The program will work out whether to use DeltaSync or EAS for the re-added account.

Now all we need is a new update since the latest one was withdrawn on 1 February. Affected users (i.e. those whose account has been upgraded to Outlook Mail) will find instructions for using IMAP to synchronize the mail account here: Windows Live Mail and Outlook Mail

Microsoft Visual C++ (often abbreviated as MSVC or VC++) is a commercial (free version available), integrated development environment (IDE) product from Microsoft for the C, C++, and C++/CLI programming languages. It has tools for developing and debugging C++ code, especially code written for the Microsoft Windows API, the DirectX API, and the Microsoft .NET Framework.

The Visual C++ Redistributable Packages install runtime components that are required to run C++ applications built with Visual Studio 2012.

Different program uses different Visual C++, if you remove one, and a software package explicitly requires it, that software may stop working. Since many of these are x86/x64 pairs, you may very well need to keep those installed if you want all of your software to function properly.
