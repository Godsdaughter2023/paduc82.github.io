Last day of (ISO)week - Microsoft Community

hans knudsen say:

Last day of (ISO)week

I have year in A1 and week number i A3.

The following two formulas seem to work fine, that is finding the last day of the (ISO)week:  (for me Monday is 1 and Sunday is 7)

=DATE(A1,1,7*A3+4-WEEKDAY(DATE(A1,,),11))   

DATE(A1,1,4)-MOD(DATE(A1;1;2);7)+7*A3-1

Will someone help me by explaining the logic behind these formulas?

jp ronse say:

Hi Hans,

See the snip.

ISO WN 1 is defined as the first week (Mon-Sun) with a Thursday or IOW at least four days (Thu, Fri, Sat, Sun).

As you can see in the snip, the ISO WN for the first day of the year can be 1, 52 and even 53. By adding 11 days, we are sure that we are somewhere in ISO Week 2. (The first formula.) To get the last day of the previous week we have to subtract the weekday of the day we are on (01/11/yyyy). What is nice in the formula is the use of

WEEKDAY(DATE(A1,,),11)

DATE(A1,,) returns the last day of November. The DATE formula allows to use negative arguments like =DATE(2015,-1,1)

meaning that we want the date of the first of January 2015 but 2 months ago (zero based) and we get 11/01/2014.

With month and day equal to zero means, for the month =>previous month and for the day => last day of previous month. As now the 11th day is exactly n weeks away of the last day of November, we can use this short method to know the weekday we are on.

Note that the second formula has some typo's: mixt of "," & ";" as separator.

MOD(DATE(A2,1,2),7) is another method to calculate the weekday.

" If I remove the control source all the MainCategory fields are filled with the same selection as I make in the first one."

This sounds like you are using a continuous form. In a continuous form, its ONE set of controls made to display multiple times. So, if you have an unbound control, all rows will display the same selection. 

Second, why are you using a SELECT Case with a single statement? The way a cascading combo works, is the row source of the second combo is filtered by the selection in the first combo. So your SubCategory table should have a field for MainCategory. Then, in the After update event of the first combo, you set the RowSource of the second to a SQL statement with criteria being the value of the first.

ken sheridan say: It's quite correct for the main category combo box to have an empty ControlSource property because there should be no column (field) in the form's table for this.  Because the main category is determined by the sub category, you only need a column for the latter in the table; to have both would introduce redundancy and the risk of update anomalies.

The problem of using correlated combo boxes like this in continuous forms or datasheet view can be overcome in the former, but not the latter, by using 'hybrid' controls, in which a text box is superimposed on the combo box to give the appearance of a single combo box control.  the form is based on a query which joins the table to the referenced sub category and main category tables.  The text boxes are bound to the columns from the referenced tables, while one combo box is bound to the subcategory column in the form's referencing table, and the main category combo box is unbound.

You'll find an example as ComboDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Take a look at the form for a simple 2-level hierarchy.  In this you'll see that that d
the txtDistrict and txtParish text boxes are bound to the text columns from the Districts and parishes table, the cboDistict combo box is unbound and the cboParish combo box is bound to the ParishID column in the referencing Locations table.

You probably just need to change my location text box to a combo box whose ControlSource is the name of the foreign key column in the form's table which references the primary key of the Basketball_Championship table (I'd call the foreign column something like Basketball_ChampionshipID or whatever is appropriate, however – you don't need to stick with my column names).

Basketball_Championship

Then the Combo box would be set up like this:

 

ControlSource:    Basketball_ChampionshipID
