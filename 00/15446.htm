I need a customer field (e.g. Text1) to apply to both Task and - Microsoft Community

davidd_27 say:

I need a customer field (e.g. Text1) to apply to both Task and Resource is there any way i can do this. The Project option is not available.

I need a customer field (e.g. Text1) to apply to both Task and Resource is there any way i can do this.  The Project option is not available.

andrew simpson uk say:

There are 2 seperate text1 fields: one for resources (resource.text1), and one for tasks (task.text1).

 

There are then two more text1 fields for assignments: one for assignments accessed via the tasks collection (tasks.assignment.text1), and another for assignment accessed via the resources collection (resources.assignment.text1)

 

If you've added the text1 field to the Gantt chart view, what you're seeing is the Task.Text1 values. If you add the text1 column to the Resource sheet view, then you're seeing the Resource.Text1 values.

 

If you add the text1 column to the Task Usage view, then you'll see the task.text1 values against the task rows. If you expand a task row to see the assignments beneath it, you'll then see the Tasks.assignment.text1 value for that assignment.

 

If you do the same with the Resource usage view, then you'll see the resource.text1 values against the resources, and if you then expand a resource to see the assignments for that resource, then you'll see the resources.assignment.text1 values.

 

Without customisation, these 4 text1 values all act independently. It is possible to replicate say the task.text1 value to the tasks.assignment.text1 and resources.assignment.text1 fields, as each assignment only ever relates to one task.

 

However, it wouldn't be possible to replicate the task.text1 value to the resource.text1 field, as a resource can be assigned to many tasks, and conversely, a task can have many resources.

 

I hope this helps,

Andrew

a. user say:

Hi Aamir,

There is something I need to explain.
For the domain users accessing Office 365 services from the Extranet, there are two ways:
1. You can set up an AD FS proxy server in DMZ as Allen mentioned.
2. Build up VPN connection (You don't need to deploy an AD FS proxy server because you can access the AD FS server after making a VPN connection to your organizational private network).

For non-domain users accessing Office 365 services from public network, I'd like to confirm whether non-domain users are the users who have their domain accounts, but use the non-domain-joined computers or they never have their domain accounts in your organization. If the former, you don't need to do additional configuration if you already have an ADFS server deployed. When these users access Office 365 services from public network through VPN, they will be prompted to authenticate using their domain accounts. If the latter, they cannot access Office 365 using SSO.

Here is an article about 'configure extranet access for AD FS on Windows Sever 2012 R2' for your reference, but there is no best practice about it.
msdn.microsoft.com/.../dn528859.aspx
Note: AD FS proxy server in AD FS 2.0 is known as Web Application Proxy in AD FS 3.0.

Edward
