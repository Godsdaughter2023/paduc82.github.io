Don't want driver updates from MS? Do this. - Microsoft Community

a1jungle say:

Don't want driver updates from MS? Do this.

If you do not want MS to automatically download and install drivers, you can stop almost all of them by doing the following. You might have to do this with each build, but it is worth it.

Navigate to "System" (right click on start and select "system"), Advanced system settings, Hardware tab, Device installation settings, select "No", Then select "Never", and save your changes.

Now you are responsible for updating drivers.

You may still see drivers download. If you do, just don't install them. Go to your device OEM website and download the driver and install it. When you update windows again, that driver will no longer be listed in your updates. At least this has been my experience so far with multiple builds using this method.

I hope this helps some of you.

Jon

Sub AddCheckBoxesRange()

Dim c As Range
Dim myCBX As CheckBox
Dim wks As Worksheet
Dim rngCB As Range
Dim strCap As String

Set wks = ActiveSheet
Set rngCB = wks.Range("B3:Z3")
'Set rngCB = Selection
strCap = "ChkBox"

For Each c In rngCB
  With c
    Set myCBX = wks.CheckBoxes.Add _
      (Top:=.Top, Width:=.Width, _
       Height:=.Height, Left:=.Left)
  End With
  With myCBX
    .Name = "cbx_" & c.Address(0, 0)
    .LinkedCell = c.Offset(3, 0) _
        .Address(external:=True)

john w [msft] say:

KNOWN ISSUE: Interruption When Minimizing Certain Windows Store Audio and Communications Apps in Build 9879

Hello Insiders!

Build 9879 of the Windows 10 Technical Preview exhibits a bug with certain Windows Store apps that are capable of background audio. Many music apps, like Xbox Music, fall into this category as do certain communication apps like the Windows Store version of Skype (the desktop version of Skype is not affected). The problem occurs when Windows is notified that an affected app is “minimized”. This notification happens in a few different circumstances:

  • When an affected Windows Store app is actually minimized (the obvious case)
  • When switching between desktops (if you are using multiple desktops)
  • When your machine goes into “connected standby”, i.e. the screen turns off but audio is expected to continue playing

In the case of a music app like Xbox Music, the playback of your music will be paused. In the case of a communication app like Skype, your call will be disconnected. We have fixed this bug, unfortunately we completed the work just a little too late to include the fix in Build 9879. The next build will include this fix. The temporary workaround is to avoid minimizing the affected apps.

Thanks for your continued testing of Windows 10, and please let us know if you see other issues.

John W [MSFT]
