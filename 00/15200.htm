Hi Mayank10Windows,

Yes, it is possible. We can perform the following steps to install Office.

Here are the steps:
a. Download Office Deployment Tool on your PC: 
https://www.microsoft.com/en-us/download/details.aspx?id=49117
b. Double-click OfficeDeploymentTool.exe, and then extract the following files to the local path on your PC (e.g. c:\ODT ).
       •setup.exe

       •configuration-Office365-x64.xml

       •configuration-Office365-x86.xml

c. Open any Configuration file with Notepad, and then paste the following text to it and rename it as the configuration.xml file.
 
<Configuration>
    <Add OfficeClientEdition="32" Channel="Monthly">
        <Product ID="O365ProPlusRetail">

            <Language ID="en-us"/>

            <ExcludeApp ID="Access" />
            <ExcludeApp ID="Groove" />
            <ExcludeApp ID="OneNote" />
            <ExcludeApp ID="Lync" />
            <ExcludeApp ID="Excel" />
            <ExcludeApp ID="OneDrive" />
            <ExcludeApp ID="Outlook" />
            <ExcludeApp ID="Publisher" />
            <ExcludeApp ID="Word" />
        </Product>
    </Add>
</Configuration>
 
Note that the parameter OfficeClientEdition changes depending on your client edition.
d. Open a command prompt as an administrator and input the following command. To do this:
Click Start > In the Start Search box, input cmd > Right-click the Command Prompt, select Run as administrator > Input cd c:\ODT and press Enter > Input setup.exe /download configuration.xml and press Enter.
e. Install Office.

Input setup.exe /configure configuration.xml

For more information, please refer to the articles 

Overview of the Office 2016 Deployment Tool and Configuration options for the Office 2016 Deployment Tool.

bob jones aka: cybertaz say:

I don't mean to be picky, Jim :-) [Ya know I love ya buddy] but I don't think your suggestion provides the solution Saber87 wants for at least 2 reasons:

  1. When using the Filter buttons, each Sort overrides the previous - IOW, you can sort on only 1 field at a time, so your Step 6 would nullify Step 5 & Step 7 would nullify Step 6. For a multilevel sort you'd need to use the Data> Sort dialog to specify the 3 sort keys. Unfortunately, there's no provision for saving the multilevel sort other than as a macro, but even so...
  2. Saber87 thinks s/he wants this to run automatically "... so i don't have to click or press any buttons". I'm not the VBA guy, but if that can be done it seems it would require an Event Macro which would take more than simple recording, right?

I stressed the word thinks because instantaneous sorting may seem desirable if only a single record is added once in a while. However, if successively entering several records having the list repeatedly sorted on entry of each one would drive the user crazy, even with well-written code :-) & I really don't see any advantage gained by doing so. IMO it's better to enter the new records without regard for order, then sort the list once when done.
