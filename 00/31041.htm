a. user say:

If you have the free version of OneDrive, it only provides 5 GB of storage, which won't be enough space to hold a system image.

If you have the paid version of OneDrive (50 GB for $1.99/month) you'll have enough disk space, but Microsoft limits file sizes to 10 GB. That won't be large enough for an image.

Even if you use another cloud storage provider that allows larger file sizes, like Google Drive or Dropbox, your upload speed is sharply limited, which means it will take many hours to upload your system image. It could take half a day. And while the upload is happening, you'll be limited in your ability to use other applications that need significant upload bandwidth, like Skype.

Even if you're willing to put up with the inconvenience of reduced upload bandwidth, once your image finally lands in the cloud, there's no way to check whether it uploaded correctly, because none of the major cloud storage providers have a method for calculating hashes in the cloud. And for a file as large as a system image over a packet-switched network, you want to be sure.

The bottom line: Don't do it.

harshpoddar say:

In order to keep the efficiency of CPU utilization, Windows always have some "Services" running in the background. This does help keep the CPU indulged but most of the time also lingers the operation and causing overhead or a complete deadlock if their are frequent potential processes which use disks I/O operations, reading extensive memory locations, communicating with peripherals, etc.

Most of the time, the software we install also starts some services in the background which may necessarily be not important for Windows to keep up. Also, at times most of the software like Google Chrome has a background host program which inundates memory. They initialize and capture enormous amount of available disk space from the very time the OS boots up, thus causing consistent deadlocks or slow operations.

To avoid this, you can disable these services at the startup so that other important programs doesn't gets chocked up. For this,

Open Windows Task Manager (Right click on taskbar -> Start Task Manager) -> Select Start Up tab

Here you can see list of all programs which starts coherently. Looking carefully on the list you can easily figure out which program is running unnecessarily and disable. This will not only free up disk utilization but also help Windows boot faster next time.
