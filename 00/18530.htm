jeeped say:

Unfortunately, text values cannot be formatted with the same versatility as true numbers with a custom number format, e.g. splitting off individual characters and separating them with hyphens,  slashes or spaces like you can with digits. There is simply no way that I am aware of to use a custom number format on that value to achieve what you are looking for as long as Excel still considers it a text value and not some form of true number value. There can be numerically conditional digits with the # symbol and forced leading zeroes with multiple 0's but the @ symbol is text and there is absolutely no room to modify that from its whole.

I did manage to get rid of the RIGHT() function but introduced another as well as the double unary.

=LEFT(A2,4)&TEXT(--MID(A2,5,99),"_(00000 0")

LEFT function Returns the leftmost characters from a text value
RIGHT function Returns the rightmost characters from a text value
MID function Returns a specific number of characters from a text string starting at the position you specify
TEXT function Formats a number and converts it to text
Double Unary Converts a textual representation of a number to a true number. (see VALUE)
VALUE function Converts a text argument to a number

ramesh.kumar say:

Hi,

Thank you for posting your query on Microsoft Community.

As per the issue description you are getting error message publisher is not verified while downloading files from Internet Explorer.

Windows prompts with this message when unknown programs (unsigned) try to write data to protected system folders or registry settings, and UAC (User Account Control) is seeking your permission.  This type of activity could be dangerous sometime, so it prompts every time.

If the program has an unknown publisher, it doesn't have a valid digital signature from its publisher. This doesn't necessarily mean the program is harmful—many older, legitimate programs lack signatures. However, you should use extra caution and allow a program to run only if you obtained it from a trusted source, such as the original CD or a publisher's website. If you’re unsure, look up the name of the program on the Internet to determine if it’s a known program or malicious software.

When we execute a normal program, UAC first checks if that program has keywords like
"setup", "install", "update" etc, using the UAC Installer Detection Technology (IDT) (If is a setup.exe then elevate). 

UAC also checks the programs manifest file for the "requireAdministrator" tag added in its manifest config, if that is found, UAC will then go to the next step. When UAC has confirmed the first step, UAC will check if that program is blocked by policy or if the programs publisher is blocked, when UAC successfully confirms that it goes to the next step. 

When UAC has confirmed the first two steps, it then checks if the publisher of that program is "Windows®" it's self, but in our case it isn't so UAC goes to the next step, and checks for a regular publisher. If the publisher is verified (this means also that the program has a valid digital signature) then UAC will prompt with a gray background and gold shield icon: "The application is Authenticode signed and trusted by the local computer".

If you are sure about these programs requesting for permission and wish to disable this prompt, you may follow the steps in the article below.

I have a range H6:P49 that has text entries in each.  Upon double click in a cell, I would
like to have that cell content to be copied into the same row column E.

I would also like to have the header of the column (row 5) of the cell double clicked to be copied
into that same row column D.

How to?

ossiemac say:

Try the following.

Right click the worksheet tab name and select view code and then copy the code below and paste into the VBA editor.

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)

    If Not Intersect(Target, Range("H6:P49")) Is Nothing Then
        Cancel = True   'Cancels edit mode that is noramally invoked with double click
        Target.Copy Destination:=Cells(Target.Row, "E")
        Cells(1, Target.Column).Copy Destination:=Cells(Target.Row, "D")
    End If
   
End Sub
