      mouneshwar r say:

      Hi Mary,

       

      This issue happens if Microsoft Diagnostic Tool (MSDT) Console is configured to run as a NT (New Technology) service. This also occurs if MSDT is trying to start as a console application without being properly setup for that.

       

      I would suggest you to try the steps mentioned below.

       

      Method 1: Test the issue in Safe mode

      Check if you receive this pop-up error by booting your computer in Safe mode.

      http://windows.microsoft.com/en-US/windows-vista/Start-your-computer-in-safe-mode

       

      Method 2: Perform a clean boot

      A clean boot helps in eliminating software conflicts. Visit the link below to know how to perform clean boot in Windows Vista

      http://support.microsoft.com/kb/929135

      Note: Once the troubleshooting is over, reset the computer to start as usual

       

      Method 3: Perform a system file checker

      http://support.microsoft.com/kb/929833

       

      Method 4: Re-configure MSDTC

      If the issue still persists beyond these steps, then try the steps mentioned below.

      a.    Close all the programs.

      b.    Click on Start; in the start search type cmd.

      c.    Right click on cmd.exe and choose to “Run as administrator”

      d.    Hit Enter after typing each command.

      Msdtc –uninstall

      Msdtc –install

      Sc query msdtc

      e.    Now MSDTC should be in a stop state. Do start the service to check if MSDTC is working correctly. Type the below command for the same and hit enter.

      Net start msdtc

      f.     Close the command prompt.

       

      Now restart the computer to see if the issue still occurs or not.

       

      Additional information:

       

      Check the event viewer for the exact problem details

      You’ve to look in event viewer and see if there are any MSDTC related events that are logged right around when you see this error pop up.

       

      What information appears in Event Logs

      http://windows.microsoft.com/en-US/windows-vista/What-information-appears-in-event-logs-Event-Viewer

      Provide the problem signature for any further troubleshooting.

       

      Hope this information is helpful and let me know if you need any further assistance.

      Port Forwarding - Microsoft Community

      xbr poseidon say:

      Port Forwarding

      SMC7004VBR

      Login Information

      • Gateway - 192.168.2.1
      • Username - none
      • Password - none

       

      Port Forwarding

      1. Open a web browser.
      2. Log into the router by entering the default gateway (192.168.2.1) into the address bar of the internet browser.
      3. When prompted to login leave the password space blank.
      4. Select Advanced Setup.
      5. From the menu on the left select NAT.
      6. From the NAT submenu select Virtual Server.
      7. First entry, in the Private IP column enter the last digits of the console’s IP address.
      8. In both the Private and Public Port columns enter 3074.
      9. Select TCP from the Type column.
      10. Second entry, in the Private IP column enter the last digits of the console’s IP address.
      11. In both the Private and Public Port columns enter 3074.
      12. Select UDP from the Type column.
      13. Click the Apply.

       

      Disable Port Forwarding

      1. Open a web browser.
      2. Log into the router by entering the default gateway (192.168.2.1) into the address bar of the internet browser.
      3. When prompted to login leave the password space blank.
      4. Select Advanced Setup.
      5. From the menu on the left select NAT.
      6. From the NAT submenu select Virtual Server.
      7. Find the ports you want to remove and put a “0” in for the Private IP.
      8. Select Apply.

      c. Right-click on the Adapter and selectProperties.

      d. In the General tab, under Device Status, check if the device is working properly.
      e. You may also check if there is any error message or yellow exclamationmark.

      Please follow the methods mentioned below.

      Method 1:

      I would suggest you to check if Bluetooth Service is started and set to automatic.

      a. On the desktop, press Windows key + R to open theRun Dialog.

      b. Type Services.msc and press Enter.

      c. Check if the Bluetooth Support Service isstarted and set to Automatic.

      d. If not, right-click on Bluetooth Support Service and selectProperties.

      e. From the drop-down, select Automatic and click onStart.
