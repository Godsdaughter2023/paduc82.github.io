You should run chkdsk.

Select Start, Run, type cmd and press ENTER. Type "chkdsk c: /f /r" without the quotes and hit ENTER. Make sure you include the spaces indicated. Enter Y when asked whether you wish to run chkdsk on restarting the computer. Exit and restart the computer.

Marking off bad sectors on a hard drive takes time so be patient. Marking off does not repair a bad sector. It places pointers on the drive telling the system not to read or write to those sectors which have been damaged.

If the number of bad sectors continues to increase after you have run the procedure above several times then you should replace the drive. If an important system file is written to a bad sector you can corrupt registry hives and lose the whole contents of the drive.

On the other hand if having run chkdsk you see no more new bad sectors then the drive can work for you for years.

Since I have responded to these kinds of things so many times, I offer this copy/paste explanation (so I will never have to type it in again).:



Most of the time (in recent memory) when there is a problem after installing Microsoft updates, the problem is because the computer was somehow afflicted before/when the updates were installed and the updates merely bring to the surface an existing problem that was not obvious before.   This is especially an issue when security updates come out.

If the last thing folks remember doing is installing MS updates, it is a typical reaction to somehow implicate the updates for post update problems when things don't work correctly afterwards and especially on the next reboot.  Since the last thing folks remember is installing updates, Microsoft takes the heat.

Another reaction is to try to remember all the updates you installed and somehow uninstall all the updates one by one from the XP Recovery Console until you find the one that is "responsible".  Unless you wrote all the update numbers down, that would be more challenging, but not impossible.  It is not going to be helpful advice in most scenarios, but you can sure try it.

Security updates sometimes patch holes in XP and when the security holes are patched (and XP is "tighter"), the affliction can no longer do whatever is has been doing, things that used to be allowed to happen are no longer allowed to happen as a result of the tighter security, the system now has some new problem (especially when it no longer boots), and it appears that installing the updates broke the system when in fact, it is just a coincidence.

The problem is always going to be a preexisting condition on the system that you just did not know about.

Another thing that can happen is that the Microsoft updates will change or update some critical XP system files.  Depending on what your antivirus or antimalware software is, those programs may interpret the new or updated files as being a threat or somehow infected and quarantine (effectively delete) the files it thinks are suspicious.

That doesn't mean the updates are bad - it means your antivirus/antimalware software is bad (probably out of date).

The next time you reboot your system, you will have some kind of a problem or you won't boot at all if the files are missing or mismatched.  Once again, the problem is not with the updates, it is some other condition on your system with your antivirus or antimalware programs you are using (they could not be aware of what the new files from Microsoft look like).
