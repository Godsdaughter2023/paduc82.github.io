mel77084 say:

The awkwardness here is caused by the fact that the devs didn't intend for 3rd party anti-virus/anti-malware programs to be installed....the Windows Defender program that they built into the system is really (in my opinion) quite good, and, although some 3rd party party security programs will at least seem work in the Win 10 preview, some may cause problems because they have only been developed and tested for earlier Windows versions, not this one.

Eventually, all of those 3rd party security programs will be developed and tested to support Windows 10, but that's for then, not now.

If you install one of them now, you'll need to uninstall it each time you go to upgrade to a new build of the preview, and that can be an extra, unneeded hassle; then, when you reinstall it in the new build, it may appear to work, while, in fact, causing problems that you may believe to be bugs on the OS.

The Windows Defender program that's built into the preview is set so that is not easily turned off, other than by installing a 3rd party security program; and then, when the 3rd party program is uninstalled, it does not usually turn Windows Defender back on, which can be a problem if you decide not to reinstall the 3rd party progam in the new build.

In such a situation, you can turn win def back on by running "sc start WinDefend Enable" in the Command Prompt (run as Administrator). It will then continue to run, as the devs intended. In the system services, where you can normally select various settings, the devs have "manual" locked in, so that you can't change it, but Defender will continue, once have restarted it.
