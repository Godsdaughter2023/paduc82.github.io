pamcaswell say: I helped someone with this problem  on another forum some months ago.  In that problem document, when the second and third number levels were first set up, a user had made them bold.  Several versions later headings 2 and 3 were not bold, but their numbers were.  Someone fixed this by clicking the numbers and then Ctrl+b, which turned off the bolding in the text. That fixed the text, but because  the numbers in the text had direct/manual formatting (the bold used to unbold), the numbers in the TOC were bold.  If you think your problem has a similar cause, try this:

In the  multilevel list dialog, click the font button  and make sure the Font style box is empty.  Do it for every level that's showing up wrong in the TOC, or to be safe for every number. Let the paragraph style determine the  "font style".

In the text, select each  number level (not the paragraph.  Actually click one of the numbers), and press Ctrl+spacebar to clear any direct formatting. 

If the text now looks the way you want it to, update the TOC.  If the unwanted bolding is gone, you're done.

If  this fix doesn't work, please explain more about the problem.

HTH,
Pam



suzanne s. barnhill say:

By default, TOC styles all  have the same formatting (which does not include bold). TOC entries do not reflect the formatting of the headings they represent unless that formatting has been applied as direct formatting. If the heading style includes the bold formatting, it will not be picked up by the TOC.

Assuming that you are using Heading1 for the paragraphs numbered 1.0, 2.0, etc., and Heading 2 for paragraphs 1.1, 1.2, 2.1, 2.2, etc., then the TOC entries, by default, will be:

1.0   Introduction

         1.1 Purpose

         1.2 Scope

If you have changed the indent of TOC 2 to be the same as that of TOC 1, then they will be aligned as you show, but there will be no change in font formatting. You must include the bold formatting as part of the definition of the Heading 1 style.
