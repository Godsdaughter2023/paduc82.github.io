Nokia E72 IMAP mail problems - Microsoft Community

bsmojver say:

Nokia E72 IMAP mail problems

Cross posted from here: http://discussion.forum.nokia.com/forum/showthread.php?p=680673#post680673

 

Yeah, E72 e-mail client needs real work if it wants to be a competitor with iPhone 3GS (two of which I own) and Blackberry.

So far, I found these problems with it:

1. The e-mail client should be able to display all IMAP folders. Currently there is no option to do that. This is a major functionality problem. In fact, displaying all folders available on the IMAP server should be the default. Instead, the client offers the folders that don't exist at all - silly choice indeed (for instance, I get a choice to see "Sent Items", which doesn't exist on any of my IMAP servers).

2. If large number of IMAP folders is selected, e-mail client won't work any more (becomes frozen for lengthy periods of time) and may even corrupt mailbox setup (in my case, it renamed two of my mailboxes to the same name - it just called them Email). Note, it didn't corrupt my mail - it corrupted mailbox setup on the phone. Essentially, mail doesn't work after large number of folders have been subscribed to. The client keeps connecting/disconnecting and eventually gives up.

3. There is no selection for sent folder. This is a critical feature for any IMAP client. It would be nice if drafts could be selected too, but that's not critical.

techfreak_ say:
Skype only stores history ~30 days of your history in the cloud.  Skype updates do not delete your local profile.   If you are logged into the same account, that data would be left intact.  Your history is stored in a main.db file.  It should be a rather decent size.  If you have a main.corrupt file, that means your previous main.db file was corrupted and could not be used.  It was renamed and a new one was created.  You could try reading the file with 3rd party software like SkypeLogViewer.  If that file is not present or you have even more data missing, this generally means more general hard drive file corruption occurred on your hard drive.  You can try to perform a check disk and see if any of that missing data comes back.

swischner say:

Although I am not sure if this is related to your recent problem (but it might well be)....

Wo has told you to store OneNote-"files" in Dropbox? This is total nonsense. It might appear to be working but there are several serious disadvantages to this.

OneDrive (the cloud service which is supposed to be used by OneNote and NOT by copying ONE files in the local sync folder like you do with Dropbox) is using the Cobalt (MS-FSHTTP) protocol for syncing, similar to SharePoint. This prevents from every little change in a note uploading and downloading COMPLETE section files as Dropbox has to do. This not only creates a huge amount of data and bandwith usage but also -- more important -- prevents concurrent changes to a section. You should already have experienced serious sync problems and especially autmomatically created copies of sections.

As deleted pages are landing in a special section (the recycle bin is nothing else) this might well be the cause of your problems. Different users are deleting content, every one of them is accessing the same section (recycle bin), creating concurrent and conflicting versions of it when syncing back to Dropbox. This all is prevented by the Cobalt protocol of OneDrive.

(Of course you would not be able to access notebooks stored on Dropbox or any other cloud service than OneDrive / OneDrive for Business with any OneNote version other than the windows desktop program. So no iPad, Android, Mac... But you pobably don't need this anyway, otherwise you would long have noticed)
