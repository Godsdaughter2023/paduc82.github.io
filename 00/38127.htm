hairyfool say:

When the BIOS finishes testing and setting up the computer it refers to its boot order and looks for a boot signature from each device in the list. Once one drive responds that it is a bootable device it passes control to that device.

 

The error message is indicating that the device has accepted the boot task but is then not able to continue due to a fault. I have to say that the situation is not good.

 

You could try booting from a Windows DVD to see what is detectable but your options are limited. A better choice would be to download the diagnostic disk image from the drive manufacturer and test the drive.

Note : you might want to start in safe mode with networking or with command prompt, depending what you are planning to do.

If you manage to get in, do a system restore. Pick the date when you did not have this problem as your restore point.

Start button > search box, type system restore >press Enter key > uac prompt > click Choose a different restore point > Next > select the date as your restore point, click it > Next > Finish
Sit back and wait. The machine will re-start when it is done.
