e.j. gun say:

This is correct.  The Worksheet_Change event will not fire based on a change from an external link (either another workbook, or a data query from the web or some other external data source).

I suggest you take a look at the Worksheet_Calculate event.  It will trigger when the workbook updates an external link.  All you have to do is add logic that tests whether the change is in the range of interest, and then go from there.  HOWEVER, there is no "Target" range in that routine, so you will have to be clever in trying to figure out if the calculation impacted the range of interest.

jay freedman say:

Editing the Normal template's Track Changes options will affect only new documents that you base on that template; there is no effect on existing documents.

To turn off the tracking of format changes in existing documents, install this macro in a module in the Normal template:

Sub AutoOpen()
    With ActiveDocument
        .TrackFormatting = False
        
        ' to remove all previously tracked formatting,
        ' uncomment the following lines (delete the apostrophes)
'        Dim rev As Revision
'        For Each rev In .Revisions
'            If rev.Type = wdRevisionProperty Then
'                rev.Accept
'            End If
'        Next
    End With
End Sub

If you want to accept any formatting changes that are already in the document before you received it, delete the six apostrophes that appear in the first column in the macro, changing those lines from comments to code. (Don't remove the apostrophes from the two lines before that, unless you delete the two entire lines, lest you get an error.) That code will accept revisions that consist of changes in direct formatting such as bold, italic, underlines, or colors. Revisions that consist of changing which style is applied will not be affected.

Note that the ".TrackFormatting = False" command turns off the check mark in the Advanced Track Changes Options dialog but not the one in the smaller Track Changes Options box. However, that's all that's necessary to prevent the tracking of formatting in the document.

steven mulkerrins say:

Hi Midge,

1. Unfortunately I don't think you can get more fine grained than that. User management administrators can manage all aspects of users in your Office 365 tenancy. You can delegate out group ownership, which will allow group owners to manage the membership of their groups, but they will not be able to add new users to Office 365.

There is an option that may help here, if all of your users will be setup in active directory first, then you can federate Active Directory with Office 365. This means that you won't need to manually create users in Office 365, and then you can let group owners manage the membership of their groups.

2. As above, this is not possible at this time. 

3. You can't currently undo a deleted group, at least not yourself anyway. It may be possible to recover one from a backup by raising a support request (groups are basically a mailbox and file store so both should be covered under backups), but I am not 100% sure about that.

As an alternative to groups, you could consider SharePoint Online team sites. These offer a lot more functionality than groups, but may require more planning and training. They will give you more fine grained control over what you can delegate out to end users and site owners, although you still have to consider how users get created in Office 365 in the first place. And specifically, they have recycle bins that you can use to recover deleted files or sites. They are also definitely covered by Office 365 backups and can be restored by raising a support ticket.

4. This is obviously going to be different for everyone, but typically once it is in a steady state as you describe it requires very little administration. You may have to handle adding new users if you are not going to federate with active directory.

I do recommend that administrators review a few things periodically such as the Office 365 roadmap to be aware of upcoming developments and the service health dashboard to be aware of any ongoing issues. There are some good useful links for administrators on the Office 365 Fasttrack hub: http://fasttrack.microsoft.com/office/manage
