scottgem say:

This all starts with your table design. Unless your invoices only include one item per invoice you need a details table. The Invoice (actually Orders) table should include only minimal info; InvoiceID, CustomerID, order date, event date, event address. The Details table should have the info about each item that is part of the order InvoiceID, ProductID, unit cost, quantity, etc.

From there you create a query that joins those two tables on InvoiceID. You will probalby also need to add the Customers and Products tables to get needed info from those tables. Once you have a query that displays all the data you want on the invoice, you can use that to build your report.

The key thing to note about the InvoiceDetails table is that it contains UnitPrice and TaxRate columns even those already exist in the Products table.  This is because prices and tax rates (VAT in the demo) change over time, but the values in each invoice need to remain static as those when the invoice was raised.  In the invoice details subform in the main invoices form when a product is selected in the combo box, its AfterUpdate event procedure is executed, with the code assigning the current price and rate from the products table to the two columns in the InvoiceDetails table, on which the subform is based.  The current price and tax rate are in hidden columns in the combo box's RowSource, and are referenced by means of the control's Column property, which is zero-based, so Column(0) references the first column, Column(1) the second column, and so on.

gord dibben say:

You could use Worksheet event code with a macro.

Probably easier ways in newer versions of Excel but I haven't kept up.

Here is some event code which runs a macro to color the cells containing a string found in A25

Event code goes in sheet module.

Private Sub Worksheet_Change(ByVal Target As Excel.Range)
    Dim ocell As Range
    For Each ocell In ActiveSheet.Range("A1:A24")
        If ocell.Value = Range("A25").Value Then
           With Application
                .ScreenUpdating = False
                .EnableEvents = False
           End With
            Highlight_Word   'run macro
        End If
    Next
          With Application
               .EnableEvents = True
               .ScreenUpdating = False
          End With
End Sub

The event code above runs a coloring macro which you store in General module.

Sub Highlight_Word()
'colors cells containing search string entered in A25
    Dim rng As Range
    Dim ocell As Range
    Dim myword As String
    On Error GoTo endit
    Set rng = ActiveSheet.Range("A1:A24")
    rng.Cells.Interior.ColorIndex = xlNone
    myword = ActiveSheet.Range("A25").Value
    If myword = "" Then Exit Sub
    For Each ocell In rng
        If InStr(ocell, myword) Then
            ocell.Interior.ColorIndex = 8
        End If
    Next ocell
    ActiveSheet.Range("A25").Interior.ColorIndex = xlNone
endit:
End Sub
