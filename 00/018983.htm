I think I understand what your code means at this point.  Take a look at the following procedure and give it a try.

Function GetNextAuthCode(Optional Segment1 As String) As String

    Dim strReturn As String
    Dim strSeg1 As String
    Dim strSeg2 As String
    Dim strSeg3 As String
    Dim intSeg4 As Integer
    Dim varMax As Variant
    Dim strCriteria As String
   
    strSeg1 = "BOH"                '1st segment rule is unknown and thus can be optionally input
    strSeg2 = Right(Year(Date), 2) '2nd segment is the last 2 digits of the current year
    strSeg3 = "A"                  '3rd segment is an alpha character incremented when 4th segment exceeds 999
    intSeg4 = 1                    '4th segment is an index between 1 and 999
   
    If (Segment1 <> "") Then
        strSeg1 = Segment1
    End If
   
    strCriteria = strSeg1 & strSeg2
   
    varMax = DMax("Right([AUTH],4)", "OrderAuth", "Left([Auth],Instr(1,[Auth],'-')-1)='" & strCriteria & "'")
   
    If (Not IsNull(varMax)) Then
        strSeg3 = Left(varMax, 1)
        intSeg4 = Val(Mid(varMax, 2)) + 1
        If (intSeg4 > 999) Then
            'If 4th segment is greater than 999, increment the preceding
            'alpha character in 3rd segment and reset 4th segment to 1
            strSeg3 = Chr(Asc(strSeg3) + 1)
            intSeg4 = 1
        End If
    End If
   
    strReturn = strSeg1 & strSeg2 & "-" & strSeg3 & Format(intSeg4, "000")
   
    GetNextAuthCode = strReturn
   
End Function

jim gordon mvp say:

There are three things to discuss here. Liking vs embedding, file formats, and Mac vs PowerPoint versions.


Linked video: 

If a video is linked, the PowerPoint presentation file and the video file are discreet, independent files. PowerPoint saves the path to the video content as a link. The video content is not copied into the presentation. Links to file content can be "absolute" where the complete file path is saved, or "relative" where only the name of the file is saved. There is a long thread in the MS Answers forum that goes into great detail about exactly when PowerPoint saves a relative link and when it saves an absolute link. Getting links to work when moving a presentation from one computer to another is tricky. The general rule to create a relative link is to make a folder, put all the video and audio content into the folder along with the presentation file. Then insert the links to the content. To share the presentation, you share the folder that contains the presentation file and all the linked content.

Embedded video:

When a video is embedded, PowerPoint makes a copy of the video file and incorporates the video within the PowerPoint file itself so that there is no link to break when sharing the presentation file. Although this makes the presentation files larger, you don't have to distribute the video file along with the presentation. Embedding makes this foolproof and much much simpler. You don't have to worry about links at all when content is embedded instead of linked.


OOXML file formats vs old file formats


OOXML

OOXML stands for Office Open XML. This is the current international standards body open file format and is now the default format for Microsoft Office applications. OOXML supports embedding of audio and video files. By default, in current versions of PowerPoint for Mac and PC the default is OOXML called PowerPoint presentation (.pptx). You should always use one of the current file formats unless you plan to play your presentation on PowerPoint v.x (Mac) or PowerPoint earlier than 97 for Windows. Here is a list of the current file formats for PowerPoint 2011:

http://www.dummies.com/how-to/content/office-2011-for-mac-file-formats-for-saving-powerp.html


Old Binary formats

There are plenty of old presentations and templates that were saved in the old binary formats (.ppt .pps .ppa .pot). These formats were widely criticized by the open source community as being non-human readable (as if you would actually want to sit around reading the code all day). In response to the criticism, Microsoft was obliged to switch to the open standards format files OOXML. The old binary formats do not support embedded video. They support only linked video, which is why you should avoid them whenever possible. 


Mac vs PC PowerPoint

Mac PowerPoint

PowerPoint on the Mac has supported QuickTime file formats for many years. To handle movies in Windows Media file formats, Mac users can install the free Flip4Mac codecs that give QuickTime the capability to hanlde wmv. This capability is not related to linking or embedding, but is directly related to the file type of the video content.

Windows PowerPoint

PowerPoint on Windows has supported Windows Media formats for many years. You can play QuickTime content (.mp4, mp3, etc) in PowerPoint 2010 for Windows, but no other version supports modern file formats other than wmv. To play QuickTime movies in PowerPoint 2010 for Windows, the free Apple QuickTime application must be installed on the Windows computer either by itself or as part of iTunes.


The best cross-platform scenario is to save your Presentation in PowerPoint 2011 on your Mac and play it in PowerPoint 2010 in Windows (where QuickTime is installed). 


Since you left out the versions of PowerPoint and the movies you are working with, I can't tell you exactly where you are losing the movie content, but you should now be able to diagnose this yourself with the above information.
