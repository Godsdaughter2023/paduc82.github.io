Find and Replace in Word: error replacing the ^13 - Microsoft Community

michaeleschuler say:

Find and Replace in Word: error replacing the ^13 (paragraph break) code

I've created the following "find" string to find multiple spaces that appear at the end of a paragraph: ( {1,})(^013)

and use the following "replace" expression to delete the space: \2

The problem is that if a replacement is made on the last line of the document, the formatting of that line is reset to normal and an additional paragraph character is inserted. In searching for an answer I came across only the following warning (on http://word.mvps.org/faqs/general/usingwildcards.htm):

"If you use ^13 in a replace string, invalid characters, that look like paragraph marks but aren't, will be inserted – so beware!"

The source page is around five years old. Does anyone have more information on this issue that can be shared?

hansv mvp say:

You can avoid the formatting from being reset by using ^p in the 'Replace with' box.

Or clear the check box 'Use wildcards', enter ^w^p in the 'Find what' box and ^p in the 'Replace with' box. ^w is the non-wildcard code for white space (any number of spaces and/or tabs).

In both cases, an extra paragraph mark will be added though. You'll have to remove that manually (or through a macro).

For what it is worth, MVP Graham Mayor has noted in his article at http://www.gmayor.com/replace_using_wildcards.htm that "^p DOES NOT WORK in wildcard search strings! It must however be used in replacement strings, but when searching, you must look for the substitute code ^13."

graham mayor say:

Better would be to convert to a T/F table, where you have a column for every player and enter TRUE if they played on that date, and FALSE if they did not. 

With your example table 1 on a sheet named "Record", in row 1 starting in Z enter your complete roster of players for the entire season(s).

=ISNUMBER(MATCH(Z$1,$A2:$Y2,FALSE))

I never specified a sheet name in the code so by default it will work on the active sheet i.e. the one that's visible.

 

All the code does is loop from the bottom of column A to the top and every time it finds the word ''Race' in a cell it will insert a row above that word.

 

If that question mark after race is really in the cell try this variant.

 

Sub Race_To_Insert()
Dim lastrow As Long, x As Long
lastrow = Cells(Cells.Rows.Count, "A").End(xlUp).Row
For x = lastrow To 1 Step -1
    If UCase(Left(Cells(x, 1), 4)) = "RACE" Then
        Rows(x).Insert
    End If
Next
End Sub
