stanimal21 say:

Yes it is however OneNote notebooks store version history on a per-page level, not the entire notebook. Try this:

  1. Browse to OneDrive.com and open the notebook in question in OneNote Oneline
  2. Click the "View" tab and then click "Recycle bin"; restore any deleted pages from there

If you see pages that were never deleted but the contents are gone, try this:

  1. Browse to OneDrive.com and open the notebook in question in OneNote Online
  2. Right-click on any page and choose "Version history"

That's not a good sign. Have you checked the OneDrive recycle bin? That might have something there you can recover. Also, have you ever opened this notebook on a regular computer or laptop recently? OneNote automatically keeps backups of each page/section in this directory by default:

In that directory are version folders (mine has 15.0 and 16.0 for the different versions of OneNote I've installed before) and then different notebooks are backed up there.

When an upgrade to a new build fails, these steps may enable it to succeed:

  • Completely uninstall any non-Microsoft anti-virus software.
  • Perform a clean boot before upgrading. https://support.microsoft.com/en-us/kb/929135 
  • Disconnect any un-needed peripherals, such as devices connected via USB and SD cards.
  • Disable all network adapters after the build has downloaded but before it Restarts, or before running setup.exe, if you are upgrading via a mounted ISO or Windows 10 install media.

Sometimes disconnecting the Internet, such as unplugging the Ethernet cable, will enable a stalled upgrade to complete.
