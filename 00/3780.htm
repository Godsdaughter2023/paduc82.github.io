jeeped say:
... I get an error msg:  "The directory name is not valid."
 ···
RD “C:\\Users\\rmiller\\Libraries\\Documents\\My Documents\\ProjectDocs\\PICSInfo.docx” ...

The RD field code syntax is correct but it appears that you have a little confusion between libraries and folders in the actual path. You've noted a Windows 7 operating system, If that is the case, that path should be closer to,

{ RD “C:\\Users\\rmiller\\Documents\\ProjectDocs\\PICSInfo.docx” } 

If you look at the folder in Explorer, you will see the interpreted library path in the upper left.

               

Click to the right of the path to convert it to an actual path.

               

Copy this and paste it into the RD field and then edit to double up the back-slashes.

BTW, the RD field has an /F parameter switch to locate the references document relative to the host document so full paths should not be necessary.

Field codes: RD (Referenced Document) field
