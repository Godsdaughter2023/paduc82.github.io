Calculations between rows - Microsoft Community

ordnance1 say:

Calculations between rows

I have a table (currently called Test1) with 1300 records:

 

Date        Route   Start      End        Location    Miles

2/1/2012   A         07:12    07:12      1                99.1

2/1/2012   A         07:12    07:12      1                99.2

2/1/2012   A         07:20    07:21      2                101

2/1/2012   A         07:23    07:24      3                102.5

2/1/2012   B         09:03    09:04      3                742.6

2/1/2012   B         09:17    09:17      4                744.1

 

I need something that will calculate the difference between miles. Ideally not between routes (if possible). Is this something that can be done in a query? Or any other way?

 

ken sheridan say: The problem is that some rows are not ordered in any way e.g.  

2/1/2012    A    07:12    07:12     1                99.1
2/1/2012    A    07:12    07:12     1                99.1

It's important to understand that a table is a set.  A set has no intrinsic order, so concepts like before or after are meaningless.  A sort order can be imposed upon rows in a table, but only on the basis of values at column positions in the rows.  In this case the logical basis for the sort order may be the Start column, but in the above rows the values in this column are the same, so neither row is before or after the other.  In your later posts the Miles value in these rows were also the same, but in your original they differed by 0.1.  It is impossible on the basis of the values in the other columns to say whether this represent a positive or negative difference.  For that you would need some other column which reliably allows the rows to be sorted distinctly on a logical basis.  If this logical basis is the order in which the rows were inserted into the table the best option for this would be a DateTimeStamp column, inserting the current date/time into it automatically when a row is inserted into the table.  It should then be a relatively simple task to return the 'difference'.  Using subqueries, which while less efficient, may be easier for you to implement than a JOIN:

SELECT T1. *,
T1.Miles -
    (SELECT Miles
     FROM Test1 As T2
     WHERE T2.Route = T1.Route
     AND T2.DateTimeStamp =
         (SELECT MIN(DateTimeStamp)
          FROM Test1 AS T3
          WHERE T3.Route = T1.Route
          AND T3.DateTimeStamp > T1.DateTimeStamp))
AS Difference
FROM Test1 AS T1
ORDER BY T1.DateTimeStamp;

In the absence of a DateTimeStamp, or other column which enables the rows to be reliably sorted distinctly, I don't see any basis by which you can do this however.  If the table has an autonumber primary key column, then you might be able to use that, but an autonumber is intended to insert distinct vales only.  Usually the values are incrementing, however, so it might do the trick.   But as with a DateTimeStamp this again assumes that the appropriate sort order is that in which the rows were inserted. Is that the case?
