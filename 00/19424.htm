"Logged in as an administrator" is not the same thing as logged in as the Administrator.  You need to use a different administrator's account to delete another user's TIF even if that other account has administrator authority.   Even in XP that would be the preferred way to do it.  If you are dual-booting you could do the equivalent thing from a different partition.

scott mcdaniel say:

Does the corruption occur during development, or during daily use?

If during development:

-- Work on a local copy of the application only. Don't open it from a network resource

-- Don't modify code in Break mode. Stop the code, open the object in Design view, and make the changes.

-- Debug - Compile frequently

-- Compact frequently

If during daily use:

-- Run a split configuration. Create a Backend with nothing but Tables, and a Frontend with everything else. Install a COPY of the FE on each local machine, and link all of those copies to the same BE.

-- Check your network. Faulty networks are a common cause of corruption.

Here's a link to a few resources that might help:

http://allenbrowne.com/ser-25.html

http://www.techrepublic.com/blog/10-things/10-ways-to-prevent-access-database-corruption/https://www.access-programmers.com/corruption-issues.aspx

You might also perform a Decompile. To do that, create a shortcut with a Target like this:

"full path to msaccess.exe" /decompile

Run that shortcut, then open your database. Access will decompile it, and you can then use Debug - Compile to recompile.

Running head, and hanging paragraphs - Microsoft Community

thelioness77 say:

Running head, and hanging paragraphs

How do I format a running head on my title page, and a hanging papgraph on my reference page?
somasundaram pandiaraj say:

Running Header

The header appears at the top left of every page. It includes a short title (a shortened version of the running head) and page number, with five spaces between the short title and the number. Word 2007 allows you to insert this header automatically; you do not have to type it at the top of every page.

1)      Insert the page header using the Header tool in the Header & Footer group on the Insert tab. The header appears at the top left of every page. It includes a short title (a shortened version of the running head) and page number,

2)      Select the Blank format. This will add a header section to the top of the document page. You will need to be sure the justification is set to left hand alignment (Home tab, Paragraph group).

3)      Click on the Type text box and then just start typing your running head.

4)      After the last word of the title, insert a page number (Insert tab, Header & Footer group, Page # tool, Current Position, Plain Number) and then use tabs and spaces to align the page number with the left margin.

 

Hanging Paragraph

Word can hang the indentation of every paragraph you type. A hanging indent is an indentation where the first line sticks out to the left and the rest of the paragraph is indented. To have Word hang and indent a paragraph, you need to complete five simple steps.

1. Position the insertion pointer in the paragraph you want to hang and indent. A hanging indent is a good way to present paragraph lists.

2. Click the Page Layout tab in the Ribbon. You will find Indents and paragraph spacing in the Paragraph group.

3. Open the Paragraph dialog box by clicking the Dialog Box Launcher button in the lower-right corner of the Paragraph group. Many of the paragraph-formatting commands require you to enter values, so the finer controls are in the Paragraph dialog box.

4. In the Indentation area, select the Hanging option from the Special drop-down list.

The By box automatically reads 0.5", which means that Word automatically hangs and indents the paragraph a half-inch. Enter another value to make your indent more or less outrageous.

5. Click OK.

The selected block of text, or the current paragraph and the remaining paragraphs you type, all have a hanging indent.

Actually, he appears to be using Word 2010, so there is no Office Button. What's probably confusing to a new ribbon users is that in Word 2003, there would have been a Print to File checkbox when you choose File - Print. In Word 2010, Print to File is at the bottom of the list of "Printers" when you choose File - Print. If you use the old FilePrint command to trigger the legacy Print dialog box, then you'll get the Print to File checkbox, and Print to File will not be in the list of printers (at least not in the legacy dialog).
