pere fouras say:

Outlook 2019 Imap folders not syncing

I met this problem on Outlook 2019. After investigation, the problem appeared to be at the server side: an old message, of which the header could be seen, had not been fully downloaded, and could not anymore be fully downloaded: each attempt lead to an error message. Deleting this old message appeared to instantly unblock the syncing.


Further details below, hoping it might help someone


I recently installed Microsoft Office 2019 on one of my laptops. 


Everything looked cool (easy installation, new interface...) until I realized my inbox was not syncing completely; only messages before a certain date came in (May 31st 2018), plus the most recent ones (January 2019).


I read a number of forums describing this or similar problems, but none ot the hints gave a result and I was starting to regret having moved to Office 2019.


Then I decided to move a number of older messages from my Imap server to a .pst file, doing this from another laptop with Office 2010 (on which I had no problem synchronizing). In the process, I saw that one given message could not be copied.


After investigation, it appeared that only the header of this ancient messages had been dowloaded, but not the full message; and when trying to download the message I got an error saying it was impossible to get the message. Connecting to my Imap server through my webmail, same thing: the header of the message could be seen, but it was impossible to see the body of the message.


This problem seemed thus to be a problem at the server side and not in my inbox. And I observed that this old message was just the one following the last synced message.


Seeing that I probably will never recover this old message, I deleted it; instantly, I could see the the synchronzation process to resume in Outlook 2019 on my other laptop. 


My conclusion is that it was this message of which only the header, but not the body, was available on the server that was blocking the syncing process
