a. user say:

This is an old thread, but it's one of the top search results on Google when searching for this error code. This is why I'm adding my findings here for others to find, when I solved this particular error.

The problem was caused by the malware called Total Anti Malware Protection, which mimics MS Security Essentials. It introduces changes to the registry in order to launch itself instead of Security Essentials, and these changes prevents Security Essentials from being reinstalled with the above error code.

To fix this issue:

  1. Press windows key+R and type regedit
  2. Go to HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\Current Version\Image File Execution Options
  3. Look at each subkey, they have the names of exe files. If the key contains the value Debugger, it means the particular .exe file is being redirected
  4. Delete the entire key for the .exe

You may expect to find msconfig.exe (or just msconfig) in the list, in addition to .exe files associated with Security Essentials.

hansv mvp say:

DateDiff with the "w" argument counts the number of weeks (7-day periods) between Time In and Time Out.

Access doesn't have workday arithmetic built in. See http://access.mvps.org/access/datetime/date0006.htm for two functions you can copy into a module and use in queries and expressions. WorkingDays excludes Saturdays and Sundays, and WorkingDays2 excludes public holidays too (using a table tblHolidays).

hansv mvp say:

Press Alt+F11 to activate the Visual Basic Editor.

Select Insert > Module to create a new module.

Copy the WorkingDays2 function from the link above to the module. That is, the code from

 

Public Function WorkingDays2(StartDate As Date, EndDate As Date) As Integer

 

up to and including the

 

End Function

 

lower down. You can use the function in a query based on the table that contains Time In and Time Out. In the first empty column, enter the following in the Field: row

 

Days to Process: WorkingDays2([Time In], [Time Out])

 

When you use the query as record source for a form or report, you can add a text box bound to the Days to Process field.

 

Alternatively, you can perform the calculation in the form or report itself: create a text box with Control Source

 

=WorkingDays2([Time In], [Time Out])
