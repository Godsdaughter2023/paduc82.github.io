Word 2003 creating index using captions - Microsoft Community

ppf say:

Word 2003 creating index using captions

I have a Word document which includes some captions under photographs and maps. When I create my table of contents, the captioned items display. Now, I have changed the document a bit insofar as I have included the caption in a text box - which Word 2003 allowed me to do. However, now those captioned items do not show in the index. If I am able to place a caption in a text box, why won't it appear in the TOC?  Many thanks.
suzanne s. barnhill say:

You can create a Table of Figures based on any specific caption; you must choose the caption you want. But note that user-defined caption labels are stored in the Normal template, so they do not travel with the document.

A real drawback to Word 2003 and earlier is that if you select a wrapped graphic of any kind and use Insert | Reference | Caption, the caption is automatically created in a text box, and Word cannot "see" text in a text box for the purpose of creating a Table of Figures or cross-references. This problem was "solved" in Word 2007 by making text boxes visible for these purposes, but unfortunately the "solution" is not backward compatible.

By and large, the best solution is always to leave graphics In Line With Text whenever possible and create the caption in a separate text paragraph. Applying "Keep with next" formatting to whichever paragraph comes first will keep the graphic and caption from becoming separated by Word, though it doesn't "group" them so that the user can handle them as a single object. If is is necessary to wrap text around a graphic and its caption, then it's still best to create them inline and then insert them both together into a frame. Frames have become increasingly difficult to access in recent versions, but they still work as well as they ever did and are often the best solution.

suzanne s. barnhill say:
Word 2003 cannot see captions in a text box; later versions can. But you can use a frame instead of a text box, and Word 2003 can see that text.

Every date has a year in it. You may not see the year depending on how it is foramtted but a year number is there... When you convert the date to number format you get the number of days that have elapsed since jan 1 1900 (Jan 0 to be exact). The decimal portion of a date equates to time which is probably not too important in this case. So the number you fave posted equate to the following dates (including time)...

 
Jan 5, 1900 6:57 AM
Jan 7, 1900 3:21 AM
Jan 5, 2009 12:00 AM
Jan 6, 2009 12:00 AM
Jan 7, 2009 12:00 AM

 Note that the first 2 dates are in the year 1900 so they come first...
