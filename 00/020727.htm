dale howard [mvp] say:

Paul --

Welcome to a confusing new feature (not a bug) in both Microsoft Project 2010 and 2013.  To see what is really going on, do the following:

  1. Apply the Task Usage view.
  2. Temporarily insert the Assignment Units column and the Peak column.
  3. Locate the task and assignment in question.

Notice that the Assignment Units value for the resource assigned to the task shows the OLD Units value.  Notice that the Peak column contains the NEW Units value.  When you examine the Work values in the timephased grid, notice that the Work values are correct, as you might expect.

You are seeing the results of Microsoft fixing a bug that was in the software for a long time.  The bug was actually fixed in Microsoft Project 2010 for the first time, which leaves us with the confusing behavior you are seeing.  When you change the Duration on a Fixed Work task, Microsoft Project will always recalculate the Units, but it will show the new value in the Peak field.  The Units field will continue to retain the old value, but the Peak value actually drives the assignment.

So, you are not going crazy.  You are just one of many users, both novices and power users, who are confused by this bizarre behavior in Microsoft Project.  Hope this helps.
