ken sheridan say: I'd strongly endorse what Gina says about printing a report which includes subreports rather than printing the form.  The latter approach, while cheap and cheerful, is a crude solution, whereas a report allows you far more scope in layout and the ability to tailor the presentation of the data to a printed format.

There is another way you can restrict the report to the current client, which is to base it on a query which references the client's ID control in the current form as a parameter.  If you are not showing the ID number in the form a control bound to it can be added and its Visible property set to False (No) to hide it.  An advantage of restricting the report in this way rather than by the WhereCondition argument of the OpenReport method (or macro action) is that it makes it possible to easily output the report in other formats, e.g. a PDF file.  You'll find an example as InvoicePDF.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

If a report is based on a query which references a control in a form as a parameter, provided that the form is open, you can open the report in any way you wish, and it will be restricted to the current client.

karl timmermans (outlook mvp 2012-2018) say:
>>
Is Gmail IMAP or POP ? They have both in the google email settings.
<<

It is <either/or> depending on what setting you select server side. The Gmail server must be told how you plan on accessing the account (POP3 or IMAP). Once that's done, you need to configure it accordingly in Outlook which for your purposes would be POP3 so that more then one account gets delivered to the same <Inbox>

If you're doing this for the first time in Outlook, the first GMail account configured should be set to create a new PST file whereas the 2nd and any subsequent POP3 accounts would need to use the same PST file (same <Inbox> for the delivery of email.

In terms of whether or not any of this can be changed? Answer: Yes, at any time but you need to follow two primary steps

#1 - Set the Gmail server setting accordingly (i.e. from POP3 to IMAP or vice versa)

#2 - You will need to configure the account again in Outlook using the correct protocol but you must first remove the original account configuration since you can't configure the same email account using 2 different protocols at the same time

Add a Gmail account to Outlook
https://support.office.com/en-us/article/add-a-...

*** to configure GMail as a POP3 account in Outlook you MUST do it via "manual configuration" since if you only enter the email address/password, it will default to using IMAP for the account.

karl timmermans (outlook mvp 2012-2018) say:
>>
is it more advantageous to use IMAP over POP or the other way around?
<<

Contrary to what many others might say on various websites, there is no single right or wrong / better or worse answer since it depends entirely on your particular situation and what you need/want.

In a nutshell

POP3 Accounts - only downloads the <Inbox> from the server in a one-way one-time download. Nothing gets sync'd back to the server

IMAP Accounts - email data is sync'd with the server or in other words, what changes on the server is reflected on the desktop and vice-versa. Allows for multiple devices to always see the same data for the account. Contact/calendar data is not sync'd with the server
