joel ivory johnson say: I replied to this question in the development forum but I wanted to come back here and post the cause to the problem just to close this thread.

The first problem is the executable was not made for a mobile device.  It was made for a desktop.  The public key token is what indicates this.  "B77A5C561934E089" is the public key token for all of the assemblies/DLLs for the desktop version of the framework (I can't remember the token for the compact framework at this time).

Even if the program were compiled for the compact framework it would not be able to do anything on a mobile device.  It's not logging some one into a network per se.  It is mapping paths on the network to drive letters.  Since Windows Mobile devices don't use drive letters this is a task that doesn't translate into windows mobile devices at all.

In my reply to this in the developer forum I list all of the paths that are being mapped to drives.

I will tell you that simply using Excel does not in and of itself make something "open source", any more than implementing code in any computer language makes that source code "open source" or that writing a book in any natural language makes the book a public document not subjectable to copyright restrictions.

joeu2004 say: Jerry wrote:
But it's hard to say what parts of it are enforceable:  It says "no part of this .... maybe copied, published, reproduced..."

If I publish the formula =A2+B2  and that formula happens to be somewhere in that workbook, am I guilty of copyright infringment?


The application of a particular copyright to a "derived work" -- that is, misuse of of some part covered by a copyright -- depends on the context and on the extent of the amount "derived".  Here, "derived work" is a technical legal term which a court determines.

If you write a book that contains the phrase "tomorrow is another day", taken out of context, there would be no copyright infringement.

But if you describe a scene with a southern lady standing on a hillside overlooking a burned-out mansion, and you write, "My home!  I'll go home.  And I'll think of some way to get him back.  After all, tomorrow is another day", you can pretty well bet that the current owners of the various copyrights to "Gone With the Wind" would have a pretty solid case of copyright infringement.

The formula =A2+B2 taken out of context would not likely be viewed as a "derived work" and a copyright infringement.  But in the broader context of some algorithm, it certainly might be.

Perhaps a better example would be some of the very complex array formulas that people like yourself have posted here as solutions to particular problems.  I cannot think of one, off hand.  Obviously, if you post them here for others to use, there is no copyright.  But if you use them in your own copyrighted producted and someone leveraged just that one formula for their own purpose without permission, you might be entitled to copyright protection of even that individual formula.  It could be argued that it is an algorithm by itself.

[EDIT] Disclaimer:  My comments are about copyrights as they are interpreted in the US.  I believe they also apply to international copyright law.  But of course, each country might set their own standards for enforceability.
