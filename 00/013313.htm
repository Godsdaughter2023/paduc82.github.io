jay freedman say:

Replace the line

Documents(ActiveWindow.Caption).Close

with the line

ActiveDocument.Close

I think the problem is that you're using older documents that display "[Compatibility Mode]" in the title bar. That would be included in the ActiveWindow.Caption value, and would make an invalid file name. I think it's odd that the original macro used that instead of just ActiveDocument, which would have worked in Word 2000 as well.

An alternative is to use Graham Mayor's add-in from http://www.gmayor.com/SaveInTwoPlacesAddIn.htm.

Whether the connection attempts to refresh or not should be a property of that connection in that workbook.  so you should be able to control that by changing the property to what you want and saving the workbook.  Also, you should be able to change those settings with code. 

Refresh links is not a security setting, but

Security settings are something separate.  Security settings are the choice of the user and should not be able to be manipulated by code.

this article, while written for DDE, gives some insight into the various settings involved:

http://support.microsoft.com/kb/930479
DDE links are not automatically updated in Excel 2007

>Maybe, even prompts and warnings come before the code, can we write a code to close the prompts or warnings? 

I am talking about opening prompts and warnings.  And the answer is no because everything stops until the user responds - so again, code has not begun running at this point.

>if the users enable it, the file will try to retrieve the data as long as the file is opened. 

If you mean by responding to the prompt, the prompt won't be given unless settings are there for an update to occur.  (it may be mixed in the workbook - some do and some don't).  But if the user agrees, it shouldn't change the setting for those set not to update.  No if the user edits the properties of the query table, then that is something that would be difficult to stop.  You can instantiate the beforerefresh event and cancel the refresh - but that requires a lot of work and I am not sure you need to get all that involved.  Also note that any code based solutions in a workbook can be easily defeated by not enabling macros.  Where security is set to high and no certification is made, this is done silently - so the user wouldn't even no you were trying to prevent something.

All that said, I may not be directly addressing your question - I don't know - there are a lot of parts and pieces involved here.

greginmich say:

In Windows 10, how do I add Windows Defender as a small icon on my toolbar (the one at the bottom of my screen, next to time and date)?

The Windows Defender icon appears in the notification area when the Windows Defender user interface is started. So what you have to do to get the icon is start the user interface at startup. You can do that by adding a startup line to a registry Run key, either manually, or with a reg file, but you don’t need to modify the registry in order to add a startup line, because Windows has always included a Startup folder so users can run programs at startup without editing the registry. I’ve already shown how to add a Windows Defender icon at startup with a Startup shortcut method here:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/is-windows-defender-icon-supposed-to-be-in-system/6cfa9d1d-3db8-49a9-add7-280a284ccfdc

 

But I’ve just recently developed a much quicker way to do this by using the specific line for the startup option that you want – icon only; icon with Quick Scan; icon with update; or icon with update and Quick Scan:

 

"C:\Program Files\Windows Defender\MSASCui.exe" -hide

"C:\Program Files\Windows Defender\MSASCui.exe" -QuickScan -hide

"C:\Program Files\Windows Defender\MSASCui.exe" -Update -hide

"C:\Program Files\Windows Defender\MSASCui.exe" -UpdateandQuickScan -hide

 

1.) Select and copy the line for the option you want.

2.) Right-click on the desktop and select New > Shortcut and then paste the run line as the location (target), and then name the shortcut.

3.) Open File Explorer: Start Menu > File Explorer; and type (or copy and paste) shell:startup (user Startup folder) or shell:common startup (common Startup folder) into the Quick access box to open the Startup folder.

4.) Then drag and drop the shortcut into the Startup folder. All done!

 

 

GreginMich
