ken sheridan say:

In your original post you said:

'EmployeeName (this is a drop down field that is connected to another table I created called tblEmployee so I can add employees from the table and they will update in the dropdown)'

From this I assumed that the EmployeeName column in the time tracking table is a long integer number data type referencing the EmployeeID primary key (probably an autonumber) of the employees table.  In a combo box bound to this column the value of the combo box would be the numeric value from a hidden first column of the control's RowSource and you'd see the corresponding name.  This is how the combo box I described in my first reply is set up, and would be the case if you (albeit unwisely) used the lookup field wizard when designing the time tracking table.

If, on the other hand, EmployeeName in the time tracking table is a text column with actual name of the employee, then that would explain the error, but more importantly the time tracking table would be incorrectly designed as the column would not be referencing the primary key of the employees table. Personal names cannot be used as keys as they can legitimately be duplicated.

As an illustration of this sort of model, the image below is from my Timesheet demo in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

You'll see how the autonumber primary, EmployeeID, of Employees is referenced in the three referencing tables by similarly named foreign key columns, each of which is a long integer number data type.  Each employee's names appear only in the Employees table, thus avoiding redundancy in the referencing tables.

ken sheridan say: Sorry, I inadvertently omitted the literal quotes characters:

If Not IsNull(Me.OpenArgs) Then
    Me.EmployeeName.DefaultValue = """" & Me.OpenArgs & """"
End If

The DefaultValue property is a string expression regardless of the data type of the column in question, so needs to be wrapped in quotes characters as above.  Note that the above wraps the value in double quotes characters, represented by a contiguous pair of quotes characters within each literal string.  People often use single quote characters in contexts like this, but in the case of personal names, this can fail as names can include apostrophes.  My own name in its original unanglicized form is a case in point, Cináed O'Siridean.
