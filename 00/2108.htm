with many controls comboboxes and textboxes in main form I think could achieve your task with two classes, managing with events controls on them.

Routines check missing filled compilation controls the new records and new existing ones, changing backcolor of textbox and combBoxes.

on load event of subForm you called the classes, and them do the work for you.

code:

load events:

Option Explicit
Private cFrm As clsFrm

    Set cFrm = New clsFrm
    cFrm.Start Me
End Sub

     cFrm.unload
     Set cFrm = Nothing
End Sub

clsfrm class :

Option Compare Database
Option Explicit

'

Public Sub Start(ByVal frm As Access.Form)
Dim itmCtl  As clsCtl
Dim ctl     As Access.Control

        If TypeName(ctl) = "textBox" Or TypeName(ctl) = "combobox" Then
          Set itmCtl = New clsCtl
          itmCtl.Start ctl, Me
          myCtls.Add itmCtl, ctl.Name
        End If
      Next
     
      Set itmCtl = Nothing
      Set ctl = Nothing
End Sub

       Set myCtls = New VBA.Collection
End Sub

Dim i As Long
     
      For i = myCtls.Count To 1 Step -1
        myCtls.Remove i
      Next
     
      Set myCtls = Nothing
     
End Sub

Dim bool As Boolean

  
    If TypeName(ctl) = "textBox" Or TypeName(ctl) = "combobox" Then
       If IsNull(ctl.Value) Or Len((ctl.Value) & vbNullString) = 0 Then
           ctl.BackColor = vbRed
           MsgBox "this control is not filled:" & vbCrLf & ctl.Name _
                  , vbInformation, "information"
           ctl.BackColor = vbWhite
           ctl.Undo
           bool = True
       End If
    End If

Set ctl = Nothing

Private WithEvents myCbo    As Access.ComboBox

    Set frm = ctl.Parent
    Set myParent = objFrm
    
    Select Case TypeName(ctl)
    Case "textBox"
         Set myTxt = ctl
         myTxt.OnDirty = "[Event Procedure]"
    Case "combobox"
         Set myCbo = ctl
         myCbo.OnDirty = "[Event Procedure]"
    End Select

     
      Set myTxt = Nothing
      Set myCbo = Nothing
      Set frm = Nothing
      Set myParent = Nothing
End Sub

     If myParent.checkFRMcontrols(frm.Parent) Then Cancel = True

End Sub

beth melton say:

What you're seeing isn't a bug. The way tables align to surrounding text was changed in Word 2013. As you have shown, in Word 2010 and below, text in tables aligned to the surrounding text. As of Word 2013, the table borders align to surrounding text.

This is actually a good thing. It may not seem like it, but it is. We now have more flexibility. If ,my tables have formatted borders, then I want the borders to align to surrounding text. If they don't, then I want the text to align. If you want the text to align, then go to the Table Properties and in the Options, set the Left Indent to 0". For a single column, set the Right Indent to 0" as well. But for multiple columns, leave the Right Indent to accommodate needed space between the columns. 

Now, this does have the drawback of when documents are opened in Word 2010 and below as they will maintain the old table rules. This was a bigger issue for the Word 2013 release, but 8 years later, it's not as major as it once was.

jonathansmith4 say:

My table left indent is set to 0 but is still indented as per my screenshot above. 

suzanne s. barnhill say:

What you're probably referring to is the "Indent from left" setting on the Table tab of Table Properties. If you click the Options... button to open the Table Options, you will see the "Default cell margins." The top and bottom margin are set to 0" and the left and right to 0.08" (giving 0.15" between cells). You can set the left margin to 0" as Beth suggested.

This will affect all cells, however, so a better option might be to select just the left column, then open Table Properties, select the Cell tab, click Options..., clear the "Same as the whole table" box, and set the left margin for the cells in that column to 0". If text in the table is justified, then you will want to do the same then for the rightmost column, setting the right margin to 0".

beth melton say:

If Suzanne's reply didn't resolve what you're seeing, if you're referring to the appearance of the left table border appearing indented compared to Word 2010, that's the new table behavior.

In Word 2010 and below, tables were actually outdented so the text would align with surrounding text. If you want the table border to extend into the margin, then you'll need to set negative left indent.

How can I encrypt critical fields, something like  *** - so that the recipient cannot make out, and at the same time, It can be uploaded in their system for testing.

I have Office 2003

thanks..
Richard 
jllatham say:

You have a two-pronged problem: protecting your customer's personal information and providing good failure condition files for testing.  Quite often it is the actual "type of", "number of" or "nature of" the data in an application that is at least a contributing factor in the failure.

As teylyn pointed out, there really is no hiding information in Excel.  The tools to undo any type of protection that was mentioned are many, mostly free and efficient.

Here is a small macro that you could use to 'hide' the data in a copy of the workbook by way of simply making all numeric characters 9s and all alpha characters Zs.

Sub ObfuscateData()
  'this converts all numerics in a group of selected cells
  'to the digit 9
  'and all alpha characters to "Z"
  '
  Dim anyChar As String
  Dim CLC As Integer
 
  'deal with digits 0-9
  For CLC = Asc("0") To Asc("9")
    anyChar = Chr$(CLC)
    Selection.Replace What:=anyChar, Replacement:="9", LookAt:=xlPart, _
        SearchOrder:=xlByRows, MatchCase:=False
  Next
  'deal with alpha characters
  For CLC = Asc("a") To Asc("z")
    anyChar = Chr$(CLC)
    Selection.Replace What:=anyChar, Replacement:="Z", LookAt:=xlPart, _
        SearchOrder:=xlByRows, MatchCase:=False
  Next
End Sub

Please help me!

I desperately need all of my chat history and image history back for an upcoming court case.

I need my chat history from 2017 and earlier. Can someone please help me restore this? 

EDIT:

I would like to add that i have already downloaded my chat history export package along with the viewer for it. When i tried viewing history with a user, on the left side it would show that i have 20 messages with them last being from 2016. But when i click to view messages it would be blank. Why is this? Why is it showing me how many messages i have with them and the last chat date but not the messages itself?!?!

techfreak_ say:

History older than 2 years is not available from the cloud. Any previous "forever" history from classic Skype (7.x), would be solely reliant on the existence and non-corruption of the main.db file on the computer. If you are still using the same computer or retained a copy of an uncorrupted main.db file, you can retrieve your history using the export tool in Skype for Desktop or third party software (like SkypeLogViewer [nirsoft.com], Skyperious [suurjaak.github.io] or a SQLite Database browser [sqlitebrowser.org]) .

If that main.db file exists, it would be present in your %appdata%\Skype folder in a sub folder consisting of your Skype name. When you use the 7.x export tool, available in Skype for Desktop, it exports this data to a folder that allows you to view your older chat history in your web browser. Other 3rd party software allows you to either view the contents of the file, or export the information into other formats. 

If you have no access to that main.db file, you would not have access to older chat history.

If any older data appears to exist in the 2 year old cloud data, it would be nothing more than a place holder for the contact. It may contain an old cached message count, mood message or a partial old message preview. This does not mean any other data is retrievable.

Images and other data sent as file attachments are only retained for 30 days. In-line images sent in conversations are only stored in the cloud for 2 years. Main.db files only reference these images when you view older history. Any older images would not be retrievable unless they already exist in your %appdata%\Skype sub folders.

Can you create sections in MS Word just like in Excel? - Microsoft Community

pianxg say:

Can you create sections in MS Word just like in Excel?

I was wondering if I could create extra sheets in MS Word in the same way that one can do that in MS Excel?

Thanks in advance.

hansv mvp say:

Word does not have the concept of worksheets.

 

You can start a new page in a document by pressing Ctrl+Enter, or by clicking Breaks > Page on the Page Layout tab of the ribbon.

 

Word does have the concept of sections, but they have a different meaning. A section is a part of a document that can have its own page setup (margins, page orientation, headers and footers). So you can have a document in which the first section has portrait orientation and the second section has landscape orientation.

You can start a new section by clicking Breaks on the Page Layout tab of the ribbon and selecting one of the options under Section Breaks.

stefan blom say:
To add to what Hans has said, Word sections store any settings related to Page Setup (= any option in the Page Setup dialog box, for example line numbering) and page numbering format and restart options.

For more on Word sections, take a look at http://word.mvps.org/faqs/formatting/workwithsections.htm.
