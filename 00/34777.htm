a_user_here say:

Well you have an i5, which is definitely 64-bit capable.  In fact most processors put out in the last 10 years or so have been 64-bit capable.

Please note a few things.

1) 64-bit doesn't run any faster than 32-bit unless a program is starved for memory.

2) 64-bit in general needs 1.25 to 1.5 more memory than 32-bit.

3) If you want to switch from 32-bit to 64-bit, it has to be a clean install, so you will have to make sure that you backup you personal files before doing it, and restore them after you have Windows 10 64-bit installed.

Here is a link on the procedure:

http://answers.microsoft.com/en-us/windows/wiki/windows_10-win_upgrade/how-to-switch-from-32-bit-versions-of-windows-to/40d90c4e-8845-4bcb-bb2f-aeab312caf74

(Note as a developer I disagree with the statement in this article that you need twice as much memory.  Yes the addresses go from 32-bits to 64-bits, which would seem like you need twice as much space, but in practice not everything is an "address/pointer to something".  As in there is also data, and it can be as small as a byte.  Not to mention that a lot of programs in Windows 10 64-bit are still running as 32-bit programs.)

    lemp say:

    Well, I can't say that I know the details of how Windows does printing, but given that all print jobs are in the same queue, if the driver for any of your printers is corrupted and puts a corrupted print job in the queue, all the other jobs in the queue will be stuck behind the bad one.

    The problem is that assuming that you do have a corrupted printer driver, you don't know which one it is unless the problem always happens right after you try to print with one particular printer.  You may end up having to remove and reinstall all of the printers.  See description of "Full" mode "Fixit" routine below.

    Microsoft used to have two handy automated "FixIt" routines for clearing out the printing subsystem, but in its zeal to move everyone to Windows 10, Microsoft "retired" those routines and they are no longer downloadable.  Fortunately, however, the PowerShell script that was the "guts" of those routines is still available.  You can read about these routines here:  https://blogs.technet.microsoft.com/askperf/2012/02/24/microsoft-fixit-for-printing/

    You can accomplish the same thing as running either the "Print Reset Light Mode" or "Print Reset Full Mode" FixIts by following the directions below.  If removing and reinstalling the printer driver(s) as described in my earlier post doesn't help, try the "Light" mode first; it won't hurt and might help.  If you have to use the "Full" mode, be aware that it will remove all printers other than Microsoft Fax and the Microsoft XPS Document Writer.  You will have to reboot after running "Full" mode and then reinstall any printers that you want to use.

    Print Reset Tool PowerShell script instructions.

    1. Disconnect the printer from the computer
    2. Click Start, type Programs and Features into the Search box, and press Enter.
    3. Uninstall any software relating to your Canon printer and restart the computer
    4. Go to https://github.com/jdickson289/Print-Reset-Tool/ and download print-reset.ps1 and make a note of where you save the file
    5. Click Start, type PowerShell into the Search box, right-click on the link to Windows PowerShell, select "Run as administrator" and respond as needed to the UAC prompt 
    6. Type the following command into the PowerShell window and press Enter
      Set-ExecutionPolicy Unrestricted
    7. In response to the "security risks" question, type Y and press Enter
    8. Type the following command into the PowerShell window and press Enter.  In typing the command, use the full path of the location where you saved the print-reset.ps1 file.  In the example shown below, print-reset.ps1 was saved in C:\Temp
        C:\Temp\print-reset.ps1 -full
    9. Type R to run the script
    10. After the script completes, restart the computer
    11. Run the printer driver installation package(s) you downloaded from the printer manufacturer's website
    12. If the software doesn't prompt you to connect the printer, connect it when the software installation is complete. 
    13. The printer should install as soon as it is connected and turned on.
    14. Go to Devices and Printers and confirm (a) that the printer is installed and set as the Default Printer and (b) right-click on the icon, select Printer properties, and click "Print Test Page"

    This is what the PowerShell window should look like as you proceed through steps 6 through 9 (I didn't enter R because I didn't want to reset my own printing system).  Don't forget to use the actual location where you saved the print-reset.ps1 file rather than C:\Temp as shown below.

    Note that in step 8 if you omit the -full switch and run the script with no switches, you'll get a complete description of what the script does.  See the screenshot below.
