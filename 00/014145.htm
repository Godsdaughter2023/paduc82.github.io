viprocon say:

I'm an IT professional that focuses on business consulting and happened across this unaddressed thread and figured I'd chime in.  As I do business consulting, I don't beat around the bush with stuff like this so I hope the following will provide closure and basic guidance: 

PST files are not designed to be actively used in the cloud, nor used/accessed over the network your computer is attached to.  PST files are I suggest you ignore them.  

As explanation:  a PST file is a database file so it and Outlook are in constant communication with each other.  By design, and for this to work, they need to both be on the same computer together.  There's a lot of technical reasoning for this but that's the way it is, period.  

Every file of any type has both the data you see, and some system data you don't see.  So while you may not be sending or receiving emails at a given time, Outlook will still be doing things with the file in the background.  This causes the file to "change", even if only to record some obscure piece of system information like the fact that it last talked to Outlook 15 milliseconds ago.  But cloud sync services have no idea why a file changed, all they know is that it did, and as such by design a cloud sync service will re-sync that entire file to make sure the cloud copy is up to date.  Which sucks if it's a large file that "changes" frequently as PST files do.  

In the old days, for environments where people wanted to have the ability to log into any computer and still access their email., Microsoft created Exchange Server which cost a lot of money and was only for businesses, so the design there was that all people's mailboxes are kept in a central location on a server and not on each individual computer in the form of  PST file.  .  

But as technology evolved and mobile devices became ubiquitous, demand increased for the ability to access email (plus calendar, etc.), from anywhere and from any device.  

So various solutions have been introduced and then have evolved over the past 10-15 years.  Speaking just on Microsoft, since we're talking about PST files and such, Office 365 is the way to go.  Hotmail, newly rebranded as Outlook.com, is now just a feature-reduced free version of Office 365.  So whether you have free Hotmail, free Outlook.com, or are paying a subscription fee for one of the various Office 365 editions, it's all the same design:  your data is stored on Microsoft servers "in the cloud", and you can configure any device you own to access that data over the Internet using software or apps installed on your device.  And because it's stored in the cloud, it is therefore not stored on your device, your device is just used to gain access to that data.  

So with that, you would need to sign up for one of these services (if not already), and then put all the contents of your PST file into that cloud service using whatever technique/method that particular service you signed up for requires.  As it varies widely, the best option to handle that is to engage an IT professional (or just a friend who knows enough to get this task done).  

As long as you understand the concept that the goal is not not store your email on a file on your computer and try to find ways to get to it from many devices, but instead, put your email into "the cloud", and then access it from there.  

Sorry, much more long-winded than I expected to be but that's the gist of it all.

[jjjsheridan wrote:]
In Excel 2010, how do I find the circular references?

It can be difficult sometimes.  The Circ Ref operation that Rich mentions sometimes does not work.  But recently, someone suggested a methodology that seems to work well.

Usually, you can find one cell that Excel says has a circular reference.  The status (lower left) points to that cell, or Excel produces a circ ref warning when you edit it, or the value is zero "inexplicably".

Select that cell, and click on Formula, then Trace Precedents.  Continue click on Trace Precedents until you see a red arrow.  (Sometimes difficult to see in large complicated worksheet.)

Any red arrow points to the cell with a circ ref.  Remember that if that cell contains a formula that refers to one or more ranges, the circ ref can be in any (or all) of the cells in any of those ranges.  So it might still be difficult to uncover the problem.
