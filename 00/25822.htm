ken sheridan say:

I think you need to examine your logical model.  At present I can make little sense of it, and there are some glaring inconsistencies.  The dbo_TblClasses table for instance has a composite key of ClassID and AcademicQuarter.  Why?  ClassID appears to be the table's key as it is not referencing the key of any other table, and  AcademicQuarter appears to be a foreign key referencing the key of dbo_TblQuarters.  It follows from this that there is no need of an AcademicQuarter column in any table which references dbo_TblClasses.  The same applies as regards InstructorID.

I also note that there does not appear to be a single enforced relationship in the database.  An unenforced relationship serves no purpose whatsoever, and the lack of them means that the database's integrity is prejudiced.

I think that the best I can do is refer you to my file StudentLog.zip in my public databases folder at:

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file is intended to illustrate the use of correlated subforms, but happens to be based on a hypothetical student registration and attendance database, whose model is as below:



This will not provide a ready-made template for your model of course, but it serves to illustrate some basic principles.  Note how all relationships are enforced, and there is a logical linear progression from CourseAttendances to Students in one direction, and to CourseNames in the other.  In the lower part of the window StudentYears models the relationship type between Students and AcademicYears.

The interface principally uses combo boxes in correlated subforms for registration and attendance recording.  The key thing to appreciate here is that the interface is really just a representation of the underlying logical model, so it's getting the model right in the first place which is the key to an efficient and reliable database.

As regards your point about the need to show columns from multiple tables, there is no reason why this cannot be done in a combo or list box's list.  Columns from referenced tables can be included by including those tables in the control's RowSource query.  The important thing is that the control's BoundColumn must contain distinct values.

mcrider_tx say:

SOLVED:

I trouble shot to no avail and finally reached out to Microsoft Tech Support.  MS did not explain why this was happening, but I share the steps below that resolved the issue.  My machine is now entering sleep mode properly with Outlook open.

I did confirm that Outlook is designed to allow the computer to sleep/hibernate even if connected to a network and regularly polling for mail.  It is also quite common and normal to leave Outlook open when not in use.  So there is no need to close Outlook just to have your computer enter sleep mode.  In my case Outlook ’07 is connected to two IMAP accounts, an Open-Exchange account and a Windows Live account. It polls for mail every 15 minutes.  Windows Live serve stays connected and Open-Exchange polls for updates anytime I change folders.

Here are the steps the Tech took.  I have no Idea which solved the problem.

1)      Under >Control Panel >User Accounts and Family Safety >User Accounts, Select “Change User Account Control settings”.  Then move the slider to Never notify.  Restart your computer.

2)      Select Start and enter CMD in the search window.  Open the CMD application and at the command prompt type sfc /scannow.  In my case, no issues where discovered by the scan.

3)      Select Start and enter msconfig in the search window.  Open msconfig.  Select the Services tab, then enter a check mark in “Hide all Microsoft services”.  Then select “Disable all”.

4)      Next select the Startup tab and select “Disable all”.

5)      Restart your computer.

6)      Restore all the setting changes made above.

7)      Restart your computer.

After the steps above, my computer returned to normal operation;  entering sleep mode as designed with Outlook open.

If AutoSave is Off, that means you saved your file somewhere other than OneDrive or SharePoint. AutoSave is active only when the document you are working with was saved to OneDrive or SharePoint.
