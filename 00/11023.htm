"Object is locked skipped", "File locked", "Locked file. Not tested", "file cannot be accessed, "Access Denied", "Some files could not be scanned", "file could not be opened", "Error Opening", "Password Protected" or "Encrypted" notations in an anti-virus/anti-malware scan are not uncommon. Some files and services are locked by the operating system or running programs during use for protection, so scanners cannot access them. Other legitimate files, especially those used by security programs, may be obfuscated, encrypted or password protected in order to conceal itself so they do not allow access as a protective measure. When the scanner finds such an object, it makes a note and then just skips to the next one. That explains why it may show with such notations but no action taken in certain anti-virus or anti-malware log scan reports. These are normal when using many security scanning programs so there is seldom a need for concern.

    saltgrass say:

    "All drives are formatted GPT and I am using UEFI boot on Win 8 and Win 10, but need Legacy boot enabled in in BIOS (as well) if I want to boot the Windows 7 system."

    This statement cannot be correct since you cannot boot Windows 7 as Legacy on a GPT drive.  Maybe including a picture of your Disk Management picture would clear this up.

    Even systems with Legacy installs would always install on the drive listed first in the Bios priority.  In order for the boot files to be placed on another drive, it had to be set first in the Bios, but the easiest way was to remove other drives.

    Multiple UEFI installs are a little different in that the boot files for a second UEFI install will be placed in the EFI System partition even if that drive is not set up as the primary drive.  Because of this, you can accept using the one boot partition and setting up Boot Menus or remove the drive with the EFI System partition and set up another one on the new drive.  You would then choose which OS to boot from the Boot Device Menu.

    But when you say a drive is corrupted when installing with all drives present, it is hard to conclude what you are actually seeing.  It is important to realize a UEFI bios may not hold a "Priority" boot device when changes are made and it needs to be reset in the bios.

    Booting a System as Legacy and UEFI can be a pain.  Windows 7 will install fine as UEFI, if you ever decide to do that, but make sure you are using Windows 7 SP1 install media.
