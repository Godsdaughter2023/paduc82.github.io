bob jones aka: cybertaz say:

Without specifics re the Custom Size you're using (also the message isn't shown in your posting) I can't provide a definitive answer, but here is some general information which may put you on the right track...

There are several factors to take into consideration:

  1. Word allows specifying a document size of virtually any dimensions (although the maximum effective printable area is 22"x22"). That doesn't mean that the selected printer is capable of handling paper of that size. If the dimensions exceed the printer's capacity printing will be based on the Default Paper Size for that printer & automatically will be scaled to fit by the print driver.
  2. If the document dimensions fall within the printers capacity the specified document dimensions will be used. That can be confirmed by going to File> Print, selecting Paper Handling from the Copies & Pages dropdown to see the Destination Paper Size. Also, custom sizes which the printer can handle will be listed in the Destination Paper Size dropdown, but any custom sizes that are outside the capacity of the printer will not even be listed.
  3. The Paper Size as well as Scaling can be changed on the Paper Handling pane of the Print dialog.
  4. Custom Paper Sizes do not appear in the Size tool on the Layout tab of the Ribbon. That device lists only the specific standard paper sizes supplied by the print driver for the default printer, it isn't a listing of document dimensions.

tasos_k say:

Hi,

try something more 'slow'

(select the desired range)

Sub Selection_Text_to_Number()
'Dec 08, 2015
Dim r As Range, x As Integer, i As Integer, s As String
For Each r In Selection.SpecialCells(xlCellTypeConstants)
x = Len(r.Value)
s = ""
For i = 1 To x
If IsNumeric(Mid(r.Value, i, 1)) Or Mid(r.Value, i, 1) = "." Then s = s & Mid(r.Value, i, 1)
Next
With r
.Value = s
.NumberFormat = "General" '<< or  .NumberFormat = "0.00"
.Value = .Value * 1
End With
Next
End Sub

Note

with: "," comma separator, depend on regional settings (decimal places)

re: msgbox width

The Demo sub works for me, as written, and displays the single character "a" 

To enable a wider msgbox in all code in a single project (not all projects), something like the following should work.
Note that that the constant "MBcaption" is padded with 15 spaces and

is declared at the top of a standard module (not a worksheet module).

MBcaption is used for the caption in all  message boxes.
If you need different text in other message boxes, you can declare additional constants.

Personally, I just pad the text or the caption with spaces if I want additional width in a msgbox.
'---
Option Explicit
Public Const MBcaption As String = "ThisHour is:-               "

Sub ThisIsWorkingForMe()
Dim ThisHour As Integer

ThisHour = 9
MsgBox ThisHour, vbOKOnly, MBcaption
End Sub
'---
