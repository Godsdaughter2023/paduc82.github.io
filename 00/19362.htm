vss_711 say:

Frustrating, ain't it?  Try going to Windows 10 Settings.  Select Time & Language.  Select Language.  This problem is the result of a funky little landmine in Windows 10, that when you are using two languages, you can be misled by the indication that "English is the Windows display language".  You can also set English within Windows as your "default" language, which also may not solve the problem.  So that's the bad news.  The good news is that there is a way to get it to work (at least it worked for me).

In Settings, in this same section, under the heading "Languages", you will see a paragraph that says "You can type in any language you add to the list.  Windows, apps and websites, will appear in the first language in the list that they support".  So the key is to change the order of your list, so that English appears first in the list.  

The way I got English to appear first in the list was to click instead on "Additional date, time, & regional settings" under the Related Settings heading.  From there, click on the Language heading, and you will see your list of languages.  If English is not first on the list, click once on it to highlight English.  There is a menu bar above the list of languages.  Click on the "Move Up" button until English is at the top of your list.  Save and reboot.

As I said, very funky.  Low marks for user friendliness.  Hope this is of help.

The copy of the workbook makes all the difference when I can see exactly what is in the worksheet.

I have modified the default print area to find the last row with formulas in columns G:L.

To test for the rows containing data it is necessary to commence the test from row 12 (after the column headers).

Below is a UDF (User Defined Function) that you copy into a standard module (It is a generic UDF that I use to find the last row or column of a given range or on a worksheet). The other code can also go in a standard module or you can leave it in the worksheet's module as per the example workbook but if you move it to a standard module then you will need to re-assign the macro to the button.

Feel free to get back to me again if any problems.

Copy following UDF into a standard module.

Function LastRowOrCol(ws As Worksheet, bolRowCol As Boolean, Optional rng As Range) As Long
    'Finds the last used row or column in a worksheet
    'First parameter is Worksheet
    'Second parameter is True for Last Row or False for last Column
    'Third parameter is optional. Use to find the last row or column in a specific range
   
    Dim lngRowCol As Long
    Dim rngToFind As Range
   
    If rng Is Nothing Then
        Set rng = ws.Cells
    End If
   
    If bolRowCol Then
        lngRowCol = xlByRows
    Else
        lngRowCol = xlByColumns
    End If
   
    With ws
        Set rngToFind = rng.Find(What:="*", _
                LookIn:=xlFormulas, _
                LookAt:=xlPart, _
                SearchOrder:=lngRowCol, _
                SearchDirection:=xlPrevious, _
                MatchCase:=False)
    End With
   
    If Not rngToFind Is Nothing Then
        If bolRowCol Then
            LastRowOrCol = rngToFind.Row
        Else
            LastRowOrCol = rngToFind.Column
        End If
    End If
   
End Function

Sub MyPrintMacro_2()
    Dim rngPrintArea As Range
    Dim rngRow As Range
    Dim lngLastRow As Long
   
    'Next line calls UDF to find the last used row in columns G:L
    lngLastRow = LastRowOrCol(ActiveSheet, True, Columns("G:L"))
   
    With ActiveSheet
        'Next line sets a default print area from G9:Last row with formulas in columns G:L
        Set rngPrintArea = .Range(.Cells(9, "G"), .Cells(lngLastRow, "L"))
    End With
   
    For Each rngRow In rngPrintArea.Rows
        If rngRow.Row > 11 Then 'Starts after column headers in row 11
            If WorksheetFunction.Sum(rngRow.Value) = 0 Then Exit For
        End If
    Next rngRow
   
    Set rngRow = rngRow.Offset(-1, 0)   'Set to one row before sum of row equals zero
   
    Set rngPrintArea = ActiveSheet.Range(Cells(9, "G"), rngRow)
  
    'Assign the address of the range containing data to PrintArea
    ActiveSheet.PageSetup.PrintArea = rngPrintArea.Address
   
    ActiveSheet.PrintOut Copies:=1  'Print the worksheet
   
End Sub
