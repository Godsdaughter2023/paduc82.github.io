1.  (Answer) I personally don't use the Personal workbook. I know it has its uses but I prefer to put the VBA code into separate workbooks because it often needs to be edited for specific purposes and then does not work in other situations.

 

2. (Answer) I wonder if you are looking in the correct area. Do you have an area with white space at the far left (at the left of the Insert function icon) in line with the Formula bar. I guess that we can all learn something new and there may be conditions where the name of a selected object is not displayed in that area but I am not aware of it.

 

3. (Answer) The recorded code can be edited to remove the Activation (Selection) of objects. Actually professionally written code almost never uses Activate and Select. However, if you are relying on recorded code then that is how it records. In your case if there is only ever one chart on a worksheet then the name of the chart can be edited in the code to replace it with the index number 1. This is the index number of the chart on the worksheet. Example following:-

 

ActiveSheet.ChartObjects(1).Activate

 

Using this method alleviates the need to know the chart name but will only work where there is only one chart because the charts are numbered in sequence starting from 1. The index number is NOT related to the number associated with the automatic naming of the chart such as Chart 1 or Chart 2. It is related to the number of existing charts on the worksheet. Example if you create 3 charts being Chart 1, Chart 2 and Chart 3 then their index numbers will be 1, 2 and 3. However, if you delete Chart 1 and Chart 2 then Chart 3 becomes index 1.

 

Hope this helps.

The Solution:
