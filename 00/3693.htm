If its a single cell array formula the value corresponding to the cell $B$5 is returned. If its a multi-cell array formula, it returns a 2x2 matrix of #VALUE!

The above returns a 2x2 matrix with results as follows:

SUM(B5:D6)  |   SUM(D5:F6)

SUM(B7:D8)  |   SUM(D7:F8)

3) {=SUM(SUM(OFFSET($A$5,{0;2},{1,3},2,3)))} entered as an array formula somewhere in the worksheet.

Also, I'll be very thankful if someone can explain to me the breakdown of calculation logic used to generate these results.

xor lx say:

Like I said, unfortunately you cannot use the syntax with ":" to generate an array of range references. INDEX simply doesn't operate like that.

In fact, you cannot use INDEX to return an array of range references, unless those references are to single cells. Hence, to generate the sum of those ranges using INDEX would require indexing separately each of the required cells within those ranges before passing to SUM, e.g. to sum B5:D6:

=SUM(INDEX($A$5:$G$14,N(IF(1,{1;2})),N(IF(1,{2,3,4}))))

or, as you wish, to sum B5:D6, B7:D8, D5:F6 and D7:F8:

=SUM(INDEX($A$5:$G$14,N(IF(1,{1;2;3;4})),N(IF(1,{2,3,4,4,5,6}))))

though obviously this approach is not quite as flexible as that using SUBTOTAL(OFFSET, nor is its syntax so obvious.

One advantage, however, of this approach is that, unlike OFFSET, the reference passed to INDEX need not be a reference to an actual range in the worksheet. Hence, in more complex constructions we can take advantage of this property to manipulate arrays which are e.g. derived from other functions.
