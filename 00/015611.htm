kathy s - microsoft support say:

Hi,

There is no direct way for Access to talk to Project. However, if Project data can be exported to txt, xls or csv file format (for example), Access has the ability to import or link to that data.

There are a couple of different approaches you could take depending on what you need.

1. You could import your project data contained in the text/excel file into an Access database and then build queries and reports off of this data. Using this approach will create static data in your Access tables and each time you wanted to update the data you would need to re-export your project data to text/excel and then reimport your updated project data back into your Access database.

2. You could link to your project data contained in the text/excel file from your Access database and then build queries and reports off of this data. Using this approach, your data would still be pretty much static within the Access database. The difference with this approach is you would only need to replace the text/excel files if you wanted to update the data (provided they are named and saved the same as the original files). The linking would remain in place and you wouldn't need to make any changes directly in your Access database.

Here are a couple articles that you might find helpful:

Import or link to data in a text file
http://office.microsoft.com/en-us/access/HA012322271033.aspx

Import or link to data in an Excel workbook
http://office.microsoft.com/en-us/access/HA012194191033.aspx

http://office.microsoft.com/training/training.aspx?AssetID=RC102722321033

Hope this helps,
Kathy

Referencing some of the help I received thusfar, and the sample code from various sources:

        

Private Sub lbl_ChangeBGColorThemeAllForms_Click()
    ChangeThemeAllForms
End Sub

Public Sub ChangeThemeAllForms()
'Purpose  : Changes all forms' color theme.
'DateTime : 1/04/2011 10:35
'Author   : Bill Mosca
'Requires :
    Dim obj As AccessObject
    Dim frm As Form

    For Each obj In CurrentProject.AllForms
        DoCmd.OpenForm obj.Name, acDesign
        Set frm = Forms(obj.Name)
        Call ChangeTheme(frm)
        DoCmd.Close acForm, frm.Name, acSaveYes
        Set frm = Nothing
    Next

End Sub
'--------------------------------------------------------------------------------
Function Fn_HexRGBtoLong(HexString _
        As String) As Long
'Purpose  : Converts HexRGB color values of A2K7
'           property sheet to long color values.
'DateTime : 1/05/2011 08:57
'Author   : A.D.Tejpal
'Note     : Hex string version of RGB as depicted for color in property
'           sheet in Access 2007 does not represent a straight hexadecimal number. It is a 6
'           character string where first two characters represent R, middle two G and last
'           two B (RGB components of color value).
    Dim Txt As String
    Txt = IIf(Left(HexString, 1) = "#", _
            Mid(HexString, 2), HexString)
    Txt = Right("000000" & Txt, 6)
    Fn_HexRGBtoLong = _
            RGB(CLng("&H" & Left(Txt, 2)), _
            CLng("&H" & Mid(Txt, 3, 2)), _
            CLng("&H" & Right(Txt, 2)))
End Function
'--------------------------------------------------------------------------------
Public Sub ChangeTheme(frm As Form)
'Purpose  : Changes color theme based on template colors.
'DateTime : 1/04/2011 11:15
'Author   : Bill Mosca
    Dim ctrl As Control

    On Error Resume Next

    'If using 2007 or higher pass the Hex value using Fn_HexRGBtoLong function.
    'If using 2003 or earlier pass the Long value without the function

    With frm
        .Section(acDetail).BackColor = Fn_HexRGBtoLong("#B13F9A")
        'Access 2003
        '.Section(acDetail).BackColor = 10108849
        .Section(acHeader).BackColor = Fn_HexRGBtoLong("#B13F9A")
        .Section(acFooter).BackColor = Fn_HexRGBtoLong("#B13F9A")
    End With
End Sub

Here is something simple to start with.

 

Function NewEDIPassword(sID As String) As String
    'sID is used only so function will produce a different password for every record is used with a query.

    Randomize
    Dim x$, O1$, O2$, b, n
   
    x$ = "abcdefghjkmnpqrstuvwxyz" 'Do not use i, l or o
    O1$ = ""
    O2$ = ""
   
    For b = 1 To 6 'First 6 character as alpha
        n = Int(Rnd * Len(x$)) + 1
        If O1$ = "" Then
            'Upper case first character
            O1$ = UCase(Mid$(x$, n, 1))
        Else
            O1$ = O1$ + Mid$(x$, n, 1)
        End If
    Next
   
    x$ = "23456789" ' Do not use 0 or 1
    For b = 1 To 2 'Last 2 characters as numeric
        n = Int(Rnd * Len(x$)) + 1
        O2$ = O2$ + Mid$(x$, n, 1)
    Next
   
    NewEDIPassword = O1$ & O2$
End Function

Excel 2013 grouping symbols gone (SOLVED) - Microsoft Community

ppdm2800 say:

Excel 2013 grouping symbols gone (SOLVED)

Following up on this thread: Excel 2013 grouping symbols gone

I had the same problem: +/- grouping symbols had vanished into thin air, the only workaround was the useful tip given by VV.936: select all cells and then click on 'AutoFit Row Height' (Home tab, Cells group, under Format).

To find a solution, I converted my xlsb workbook to xlsx, made a copy, applied VV.936's workaround, and compared both xlsx archives (xlsx files being in fact zip archives in disguise). There, in the xml file of my sheet, there was a setting called 'defaultRowHeight' that was set to 0. Resetting it to 12.75 corrected the issue.

Here is how to fix the issue in a xlsx file, step by step:

1. Open the xlsx file/archive with an archive manager (eg. 7-Zip)

2. Browse to the \xl\worksheets\ folder

3. Open sheetn.xml (n being the number of your worksheet)

4. Find the <sheetFormatPr ...> tag, near the top of the file

5. If you have the same problem I had, the 'defaultRowHeight' argument should read defaultRowHeight="0". Replace it by defaultRowHeight="12.75"
6. Save the xml file, and make sure the change is reflected in the xlsx file/archive

Hope this helps!

trevor rabey say:

That was a slip up on my part.

MSP cannot tell you directly how many people you need, because its not managing the project, and it does not know how you want to balance time against cost, or how many people you can get

What it can do, however, is tell you how long the project will take, given the tasks as you have modeled them with their respective durations and predecessors/successors.

It does this, if necessary, with resource leveling, which delays tasks which have over-allocated resources assigned, and comes in various flavours depending on the check boxes selected in leveling options, and the priority of each task.

You have as many people as you have defined as resources.

You need enough people so that none of them are over-allocated, but over-allocation can be resolved in a number of ways, and getting more people is only one of them.

If by "on time" you mean that you don't want to delay tasks and choose not to perform resource leveling in any flavour, and if you have over-allocated resources, you still have a method which is this. In the resource sheet, increase the max units of the over-allocated resources until they are no longer over-allocated. if it is 1500% then you need 15 of those guys, or that guy and 14 other like just him.
