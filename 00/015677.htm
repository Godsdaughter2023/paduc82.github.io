ken sheridan say: Data entry should always be by means of a bound form, never in a table's datasheet view.  In the form you would use a combo box bound to the foreign key PatientID column, set up like along these lines:

RowSource:     SELECT PatientID, FirstName & " " & LastName FROM Patients ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

Alternatively you can set up the RowSource property as follows to show the names in the format LastName, FirstName:

RowSource:     SELECT PatientID, LastName & ", " & FirstName FROM Patients ORDER BY LastName, FirstName;

This is better if you regularly make use of the autoexpand mechanism to select a value from the list with the keyboard.

If you wish to show other values for the selected patient in the form you can do so by basing the form on a query which joins the table to the Patients table on PatientID and return the relevant columns, e.g. address data, form the Patients table in the query.  Control in the form can then be bound to those columns, setting their Locked property to True (Yes) and their Enabled property to False (No) to make them read only.

Another approach, would be to embed the form as a subform in a form bound to the Patients table, linking the subform to the parent form on PatientID.  The PatientID value would be automatically inserted into the subform's recordset by the linking mechanism when a new row is inserted in the subform.

doug robbins - mvp office apps & services (word) say:

Your document had precisely the configuration that I questioned – a Continuous Section Break, with the first paragraph on the next page being on that page because it was formatted to have a page break before it.

 

You can get away with that provided there is nothing else on the page containing the Continuous Section Break after that Section Break.  But in your case, there was an empty paragraph. 

 

As a result, the text after the Continuous Section Break actually starts on Page IV of the document and as that page is numbered as a result of it being in the previous Section, the page containing the start of the new Section (the empty paragraph) cannot be given the page number 1.

 

If that empty paragraph is deleted, so that the Section does start on a new page, the page numbering is corrected

 

However, in my opinion, if you want a Section to start on a new page, you should use a Next Page Section Break.
