hey you! say:

SOLVED: I've had this problem literally for YEARS.  I've searched forums with the simpleminded (and wrong) suggestions of checking if the right-button key is stuck, and updating the touchpad drivers.

The solution is disabling the touchpad two-finger tap!  THIS gesture brings up the context menu.  You don't need it; there's a right-button for that.

Go to HKEY_LOCAL_MACHINE\SOFTWARE in the Registry (you need to be on an Administrator account).  Locate the touchpad manufacturer's entry under this, then search for a Registry entry like "2FingerTapAction" (it may be different for different manufacturers).  Change the value to zero.  PROBLEM SOLVED!

Method 1: Try resetting the TCP/IP stack, release and renew your IP lease and flush and register DNS.

To reset the TCP/IP stack:

Open a command prompt as administrator and execute the following command:

netsh winsock reset

netsh int ip reset

ipconfig /release

ipconfig /renew

ipconfig /flushdns

ipconfig /registerdns

For more details look to this article and either click on "Fix it for me" or follow the instructions to fix it yourself:

Method 2: Use the automatic Windows Network troubleshooter in Windows 10

Open the network control panel (Run ncpa.cpl)

Right click on the adapter and hit diagnose.

For more details look and screen shoots look at this older article:

http://windows.microsoft.com/en-US/windows7/Using-the-Network-troubleshooter-in-Windows-7

If the troubleshooter is unable to fix the issue, let us know what the error was. If it is unable to identify an issue try the following methods.

Method 3: Try turning off the firewall temporarily to see if it's causing the problem.

Open a command prompt as administrator and execute the following command:

netsh advfirewall set allprofiles state off

For more details look at this older article:

http://windows.microsoft.com/en-us/windows/turn-windows-firewall-on-off#turn-windows-firewall-on-off=windows-7

Method 4: Try disabling third party antivirus or malware services.

Run > "msconfig"

On the Serivces tab look for third party antivirus or malware software (ie. McAfee, BitDefender, Norton, etc.) and disable (uncheck) them to see if that is causing the issue. If it is please contact the third party software development team for further assistance.

If you have VPN software installed let us know and we will provide additional guidance or you can refer to this threads:

http://answers.microsoft.com/en-us/windows/forum/windows_10-networking/after-upgrading-from-windows-81-to-windows-10-no/42b1852e-4faa-47a2-85ca-dc8e190c7c27

http://answers.microsoft.com/en-us/windows/forum/windows_10-networking/windows-10-wifi-limited-connection-only/51e822d3-35e9-4e22-81cf-5a7177ee64b5

You can also check and make sure the DHCP client, DNS client, IP Helper and TCP/IP NetBIOS helper services are running by running services.msc however instead it would be easier to try rebooting the device to make sure the device  is not in a bad state due to an unexpected event.

If any of these methods fixes your problem please update the thread specifying what method was ultimately useful as your feedback could help other users in the community. If none of this work for you let us know.
