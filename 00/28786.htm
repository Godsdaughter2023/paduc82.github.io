Filtering for top or bottom values

Follow these steps to filter for the top or bottom numbers in a table:

1.      Click the filter arrow for the numeric column by which you want to filter data.

The filter drop-down list appears.

2.      Point to Number Filters in the drop-down list.

3.      Choose Top 10 in the resulting menu.

The Top 10 AutoFilter dialog box appears.


4.      From the first option, select whether you want the Top (highest) or Bottom (lowest) values.

5.      In the second option, select the number of items you want to see (from 1 to 500).

This spin box displays a default value of 10.

6.      In the third option, select whether you want to filter the items by their names or by their percentiles.

For example, choose to list the top 10 customers per their sales dollars, or list the top 10% of your customer base.

7.      Click OK.

Excel displays the records that match your criteria.



Extract the Top Five Records

F1: Criteria Heading -- Leave blank
F2: Criteria:   =D2>=LARGE($D$2:$D$28,5)

The LARGE function returns the fifth largest value in the range D2:D28.
An absolute reference is used for the range -- $D$2:$D$28.


Cell D2, the first cell with data in the Total column, is compared to this value. A relative reference is used for this -- D2. Rows with a Total value that is greater than or equal to the fifth largest number, pass through the Excel advanced filter. (Note: In the case of ties, more than five records may be returned)



Introducing Small And Large

SMALL and LARGE are two functions that find more than just a single minimum or maximum value. They're useful when you want to find the top or bottom five values, for example. To do this, the SMALL and LARGE functions require two arguments: the cell range and the position relative to 1.

For example:

SMALL(A1:A10, 1) is the same as MIN(A1:A10). It finds the minimum value in the range.

SMALL(A1:A10, 2) finds the second smallest value in the range of A1 through A10. In this example (where cells A1 through A10 contain ten even numbers beginning with 2), the formula returns a value of 4.

There are two important differences between the terms SMALL/LARGE and MIN/MAX.

SMALL and LARGE accept only one range in the function arguments. Thus, SMALL(A1:A3, B1:B3, 1) doesn't work. MIN and MAX can search through 30 cell ranges.

When using SMALL or LARGE, if the range contains no numeric values, the function returns #NUM!. MIN and MAX return a value of 0.

Putting Large (And Small) To Work

While you can always sort an array to find the top value(s), I've found it more useful to use the LARGE function, especially if I just need the top two or three values in a very long list which I don't want to rearrange. (A SORT will reorder the rows, and I don't want to change the order of the original data.)

The longer the list, the more useful the LARGE and SMALL functions. To illustrate their use, however, I'll use an artificially small example.

For example, imagine the following data in columns A and B of rows 1 through 5 (sales person and number of sales for the month):

Adams     10

Baker       15

Carson    20

Denton       8

Everson   12

To find the first- and second-highest sales for the month, use cells D1 and D2 (respectively) for the formulas:

=LARGE(B1:B5,1)

=LARGE(B1:B5,2)

You could, of course, use the MAX function in cell D1's formula instead, but I'm using the LARGE function in both cells for consistency.)

The largest value is 20, so D1 will evaluate to 20. The next largest value is 15, which is what D2 will contain. But who are the corresponding salespeople with these sales? Just knowing "20" and "15" usually isn't enough.

To get the answer, combine the INDEX and MATCH functions with the LARGE function. In cell E1. enter:

=INDEX(A1:A5,MATCH(LARGE(B1:B5,1),B1:B5,0))

This evaluates to Carson -- the person with the highest number of sales.

In cell E2 enter:

=INDEX(A1:A5,MATCH(LARGE(B1:B5,2),B1:B5,0))

Excel returns Baker -- the salesperson with the second highest number of sales.

Dissecting The Formula

Let's dissect how the formula in cell E2 works. (Cell E1 is much the same.)

Start by examining all portions of the formula, looking for the part that is executed first. With complex formulas such as this, it's usually easiest to look for the portion of the formula surrounded by the most parentheses on both sides -- the innermost part of the formula, if you will. In the formula in E2, the LARGE function and its arguments comprise the innermost part of the formula, and thus you would evaluate it first.

LARGE(B1:B5,2) finds the second largest value in the range B1 through B5. That value is 15. So with that portion of the formula evaluated, you can now chip away at more of the formula, which is now simplified to this:

=INDEX(A1:A5,MATCH(15,B1:B5,0))

Again, look for the portion of the remaining formula that is surrounded by the most parentheses and you've probably found what needs to be evaluated next.

MATCH tells you the position of the cell in the range -- first, second, third, and so on -- matching the value you're looking for (the first argument).

Therefore, Excel matches the first argument (15) with values in the cells of the second argument (cell range B1 through B5), looking for an exact match (the third argument in the MATCH function -- 0 -- means "find the first exact match"). That is, Excel looks for an exact match for 15 in B1 through B5.

Excel makes the match, finding a value of 15 in the second cell in the range B1:B5 -- so the MATCH portion of our formula returns 2. Thus, at this point you have evaluated the formula down to:

=INDEX(A1:A5,2)

The INDEX function is rather straightforward because you're using it with a one-dimensional cell range -- a single column. INDEX returns the value of the cell in the range (indicated by the first argument) in the position of that range indicated by the second argument. In this case, Excel evaluates the formula as the second cell in the range A1:A5 -- cell number 2 in the range -- which is Baker.

The Problem With "Top" And "Bottom" Lists

Everything works fine with the formulas in cells E1 and E2 as long as the number of sales is unique for each salesperson. However, you run into trouble when the top value repeats. For example, suppose the sales figures are as follows:

Adams      40

Baker      40

Carson     20

Denton     21

Everson    22

In cell E1, the formula

=INDEX(A1:A5,MATCH(LARGE(B1:B5,1),B1:B5,0))

returns Adams, since the highest sales were made by Adams. So far, so good.

However, cell E2's formula

=INDEX(A1:A5,MATCH(LARGE(B1:B5,2),B1:B5,0))

which should return the value of the second-highest salesperson, also returns Adams.

The problem occurs because the MATCH function finds the first value in the list. Though the second-highest sales figure is also 40, when Excel goes looking through column B, it matches "40" with "Adams". MATCH stops looking when it finds the first match. VLOOKUP and HLOOKUP have the same problem -- they assume unique values and stop when the first match is made.
