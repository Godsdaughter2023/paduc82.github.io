Re-installing usually does nothing on a Mac, because the files that get damaged are not on the CD to be reinstalled.  There's a chance you will make the problem worse.  So don't do that!

You can install as many times as you like on that licence, provided that you install to the same machine.  But let's try the simple things first...

Perform Steps 1 to 4 (Only!) as described in the following article:

http://support.microsoft.com/kb/2026222/

That should be sufficient to get the programs able to start.

Then turn Automatic Updates back on, and they should do the rest for you.

Updates are absolutely critical to any program on the Mac: about 80 per cent of the computer activity in any program is actually in the operating system: if the program is not updated, problems like yours are inevitable.

If this doesn't solve the issue for you, come back: there's several other things we can try, but that's the simplest and quickest :-)

    AVERAGEIF date has passed

    I have a yearly accounting file that I update for each month. I try to get the average at the bottom to keep updated through the year, as it is now it will only be correct at the end of the year.

    EXAMPLE:

    IN OUT BALANCE
    01.01.14 500 -200 =B2+C2
    01.02.14 600 -400 =D2+B3+C2
    01.08.14 =D3+B4+C4
    TOTAL =B2:B4 =C2:C4 =B5:C5
    AVERAGE =AVERAGE(B2:B4) =AVERAGE(C2:C4) =AVERAGE(D2:D4)

    I have tried to find a solution with AVERAGEIF and <=TODAY as a criteria to get a system that ignore future months, but can´t get it to work.

    Kind Regards

    Andy

    bob jones aka: cybertaz say:

    The most effective approach IMO is to not add the record to the list until there is data for that record :-) Especially if you use Excel's Tables feature your formatting & formulas will carry down automatically as new records are added. See Excel Help on the topics related to Excel table

    However, the Functions you're using will give you what you want as long as the incomplete records don't contain Zero values... truly empty cells (as well as cells containing Text or logical values) are not a factor in the operation of the functions. for example, the Average for column B will add B2 & B3 but will ignore B4 then divide by the actual number of values found in the specified range [which is 2]. The result in B6 (based on your example) literally should be the equivalent of: (500+600+0) divided by 2 = 550.

    The only column that presents a problem is the one for Balance -- because of the calculations in that column for the months with no data. They are effectively yielding a Zero value as their result, so one option would be to not have the calculations in place for future months.

    One option to resolve that would be to use the AVERAGEIFS() function, so assuming you do list all 12 months the function wound be written as:

    =AVERAGEIFS(D2:D13,B2:B13,">0")

    There are other possibilities as well but which will work depends greatly on how the sheet is actually set up.

    Make sure the Windows Installer Service is running

    • Click on Start
    • type ' services.msc' without quotes in Search Window
    • Right click
    • select Run As Administrator
    • now look for 'Windows Installer' service and double click on it
    • change the startup type to 'Automatic' if it's not set to automatic
    • make sure that the service status reads 'Started', if not click on start to start the 'Windows installer service' > Apply > OK.

    Verify the Userid has “Full Control” access rights on the following key:

    • Click on Start
    • Type' regedit' without quotes in the Search Window
    • Right click
    • select Run As Administrator
    • navigate to the following registry entry
      HKEY_LOCAL_MACHINE \Software\Microsoft\Windows\CurrentVersion\Installer >
    • right click on installer
    • Permissions
    • select the user profile name and verify there is full control , if not check the box full control to the corresponding user
    • click on Apply > OK
    • Reboot the computer

    How to Turn AutoSave Off - Microsoft Community

    suzy_g say:

    How to Turn AutoSave Off

    You can turn off AutoSave for each file that you want to only save changes manually using the AutoSave toggle at the top left of the application. Your preference will be remembered next time you open the file again.

    If you accidentally make changes to a document, you can easily revert to an earlier version of the document using Version History, which you can find via the Document Titlebar.

    Additionally, if you want to turn AutoSave off by default for all cloud files, you can do so through File > Options > Save in each app where you want it disabled. For example, in Word, simply uncheck “AutoSave OneDrive and SharePoint Online files by default on Word” and then restart for the changes to take effect.

    When it’s turned off, you can manually turn on AutoSave by using the AutoSave toggle at the top left of the application window or the Alt + 1 accelerator key.

    Please note that AutoSave refers to the new feature for Office 365 subscribers that enables a more seamless experience, including simultaneous real-time editing when working with files stored on the cloud. For files that don’t have AutoSave, such as files on your local hard drive, or for non-subscribers, there is still AutoRecover, which can help recover changes if you forget to save manually.

    To learn more, please see https://support.office.com/en-us/article/What-is-AutoSave-6d6bd723-ebfd-4e40-b5f6-ae6e8088f7a5
