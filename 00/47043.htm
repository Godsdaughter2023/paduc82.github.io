Additional Potential Fix for Windows 10 V1903 Update failed with error - Microsoft Community

stone free say:

Additional Potential Fix for Windows 10 V1903 Update failed with error code 0x80070005

Hi,

I've had terrible trouble trying to get both the previous 1809 update, and the current 1903 Update. I finally got the 1903 update installed which finished at midnight last night.

I had attempted to install it first via Windows updates, and then with the update assistant and with the Media Creation tool.  Last time I managed to get a clue to what was going wrong because the Media Creation tool gave a more complete error.  However this time they've ruined the Media Creation tool because it now disappears at the end of a failed install, just leaving a standard message box saying Update Failed, and no longer gives you any error information.

The fix both this time and last time are related to the use of WSL (Windows Subsystem for Linux) Last time it was enough to just revert the change of going into Settings\Update & Security\For Developers and changing from "Developer Mode" to "Sideload Apps" and then removing the Developer mode package.

Last time that was enough and resolved the problem however this time that too failed, now there is an additional step.

Start a command prompt then run:

wslconfig /list

This will provide you with a list of any WSL installations, unfortunately you will need to remove all of these to get the 1903 update to work

For each installed WSL run the command

wslconfig /unregister DistroName

I hope this helps someone, even if it just happens to be me when attempting the next big update
