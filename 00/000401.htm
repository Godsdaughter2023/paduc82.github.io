joeu2004 say: Mary Lou wrote:

I have a column with 13 values of funds to be paid over the next 13 years.

[....] 

How can i calculate the present value for each row and then a total present value for all payments? 

You can do this without having to know the mathematical calculation.  That is why Excel (Lotus, Visicalc) created functions.  (Klunk!)  Consider the following:



A2:A14 and B2:B14 are the data you provided.

The individual PVs are calculated in B2:B14.  The formula in B2 is simply =A2.  The other formulas are of the form (B3 for example):
=PV($D$2,A3-$A$2,0,-B3)

Note that in order to calculate PV, we must have a "discount rate".  Often, that is chosen based on business requirements.  I have chosen 10% in D2 arbitrarily.

Also note that -B3 is used in the PV expression.  Thus, the sign of the PV is the same as the sign of the cash flow, which is the convention used when calculating NPV (the sum of the PVs).

The sum of the cash flows in B15 is simply:  =SUM(B2:B14),

The sum of the PVs in C15 is simply:  =SUM(C2:C14).

As a double-check, note that C15 should be the same as the NPV calculated in C16 as follows:
=NPV(D2,B3:B14)+B2

Note carefully the construction of that formula.  In particular, the range is B3:B14, not B2:B14; and B2 is added after the function.

This is peculiar to Excel (and Lotus?).  It is because Excel NPV discounts the first value in the range.  Usually, the first cash flow is considered the "present time", so it is not discounted.

gord dibben say:
Thanks, I'm not familiar enough with VB to know modify that for a range of row/columns though, such as a 32 by 32 or 16 by 22 grid


If you want to give it a try here is the code and a couple of macros to choose row heights and column widths.

 

The first two macros convert cell dimensions to millimeters.  You trigger them from the other two macros below.

 

Sub SetColumnWidthMM(ColNo As Long, mmWidth As Integer)
' changes the column width to mmWidth
    Dim w      As Single
    If ColNo < 1 Or ColNo > 256 Then Exit Sub
    Application.ScreenUpdating = False
    w = Application.CentimetersToPoints(mmWidth / 10)
    While Columns(ColNo + 1).Left - Columns(ColNo).Left - 0.1 > w
        Columns(ColNo).ColumnWidth = Columns(ColNo).ColumnWidth - 0.1
    Wend
    While Columns(ColNo + 1).Left - Columns(ColNo).Left + 0.1 < w
        Columns(ColNo).ColumnWidth = Columns(ColNo).ColumnWidth + 0.1
    Wend
End Sub

 

Sub SetRowHeightMM(RowNo As Long, mmHeight As Integer)
' changes the row height to mmHeight
    If RowNo < 1 Or RowNo > 65536 Then Exit Sub
    Rows(RowNo).RowHeight = Application.CentimetersToPoints(mmHeight / 10)
End Sub

 

These next two are run to set the dimensions.

In this case I set for a 32 x 32 grid of 8mm squares.

 

Sub col_width()
    Dim i As Long
    For i = 1 To 32
        SetColumnWidthMM i,  8
    Next i
End Sub

 

Sub row_height()
    Dim i As Long
    For i = 1 To 32
        SetRowHeightMM i,  8
    Next i
End Sub
