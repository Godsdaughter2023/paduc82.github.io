john w. vinson say:

If your companies belong to two different categories which require different handling, you really should consider adding a Category field to the Company table to distinguish them. Any time you need to put a 50-branch IF in your code... you need a Query instead!!!!

If you had a Yes/No field in your table, SpecialRequirements, that was YES for "PSE - BAKER GENERATION" and all the other companies of its ilk, and NO for the other companies, then you'ld need only one IF, or (better) just a criterion selecting the desired subset of companies by checking that one field.

 

boblarson say: One thing to keep in mind as you work through your database.  Always try to come up with a way where you do not have to HARD CODE values into code or macros.  Keep it generic and reusable.  It makes maintenance a breeze and makes it less likely that you will have to work on code or macros once they are in place.

Two alternatives for your boss:

1. Have a Yes/No field in the table which can be added when you add a company and changed at any time, and have your query report very simple.

2. Leave that field out and have a skilled programmer go into the VBA code and change a 50-branch complicated IF block every time a company is added, or its name is changed, or it changes from one category to another.

I know which I'd consider preferable.

If Company IN ("PSE - BAKER GENERATION", "XYZ - ALPHA GENERATION", "ACME TOOL AND DIE", "SUPREME BLIVETS INC.", <etc. through all 50 companies>)

don cuthbert say:

If you are running build 14376 and trying to update to the latest build 14383, you may have to switch between Fast and Slow Rings and then back to Fast Ring and Restart.  Then leave the computer sit for several hours or overnight and the latest Fast Ring may then be offered via Windows Update.  Hopefully this will get your Windows Insider device back up to date.  If nothing seems to work, you might be able to "fetch" the update via a Hyper-V virtual machine, as I tested yesterday.

If you have Hyper-V, a method I tested yesterday was to create a VM in Hyper-V by installing Windows 10 build 10586 from the ISO.  I then signed into Windows on that VM using my Windows Insider Microsoft Account (Windows will not be Activated, however).  I then joined the Insider Program on that VM and the Fast Ring, let it get the various public updates and left it sit for several hours.  I re-checked Windows Update and build 14383 was offered and began to download.

Once build 14383 has been downloaded, Windows Update on the VM will create an ESD file in the hidden folder at C:\$Windows.~BT\Sources\install.esd.  This file can be used to create a Windows 10 build 14383 ISO by using a script called ESD Decrypter 4, as described in this thread:

http://answers.microsoft.com/en-us/insider/forum/insider_wintp-insider_install/instructions-create-a-bootable-iso-file-using-esd/d2768b51-221a-436e-af4e-d3b9118a8864?auth=1

The way I do this is to copy the ESD file from the VM to a shared folder on my physical PC.  Then place it in the ESD Decrypter folder to create an ISO from it.
