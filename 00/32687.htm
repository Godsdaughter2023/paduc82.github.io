don cuthbert say:

I don't think that Windows 10 normally uses the PBR method, although it is still possible to setup. PBR uses a WimBoot compressed image for restoring Windows. That is why I was wondering if it was a leftover from Windows 8.1. Here is an article explaining the change in approach that was done in the original development of Windows 10.

https://blogs.windows.com/windowsexperience/2015/03/16/how-windows-10-achieves-its-compact-footprint/#RDx4ATbvscBGQS1X.97

WimBoot and Push Button Reset was an approach used for devices with low storage capacity, but Windows 10 now uses CompactOS instead, although it is still possible to setup a WimBoot image.

I run Windows 10 from a 128GB SSD also. I just finished totally wiping it and doing a clean install of Windows 10 build 16299.15, the Fall Creators Update. My SSD used to fill up and get short of space, but Windows 10 now allows the choice of where to store new Apps, and where to store the user folders, right in the Settings App. I store on my large hard disk, instead of on the SSD.

Because I use the HD for storage, it becomes quite easy to completely wipe the SSD and reinstall Windows at any time. So I choose to do this with each new version of Windows 10. However, I do not have a lot of software to reinstall each time, which makes it easier to start over.

In your case, each new Windows 10 version may be replacing the PBR partition contents, continuing with the original method that was designed for Windows 8.1.

It sounds like your partition is still used in the original WimBoot approach. So a switch in method might require deleting all partitions from the SSD and reinstalling Windows 10. After an upgrade to the Fall Creators Update might be a good time to consider such a change. Because you use another HD already, it might not be that hard, as you can use the Settings App to change storage location for new Apps (it creates a matching username folder at the root of the HD you choose). I also individually move each user folder from C: to my D: drive, by right-clicking and using the Location tab and Move option.

To resolve error code 0xc000000f, we recommend to backup your files. You may follow the steps below:
