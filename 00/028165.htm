Option Explicit

 

Private Const RIBBONFILE As String = "%F%"

Private Const RIBBONPICTUREFORMAT As String = "%JP%"

 

Private Sub Workbook_SheetActivate(ByVal Sh As Object)
    Select Case UCase(TypeName(Sh))
        Case "CHART"        ' Force Ribbon to "Home" tab
            Application.SendKeys RIBBONHOME
        Case "WORKSHEET"    ' Do whatever you desire...
            Application.SendKeys RIBBONACROBAT
        Case Else           ' Should not be any other kind of sheet, but...
            Application.SendKeys RIBBONHOME
    End Select
End Sub

jay freedman say:
Unfortunately, while you can apply shading to individual table cells, Word doesn't support filling a cell with a picture or image. The workaround is this:

  • Click in a cell that you want to have a background.
  • Click Insert > Text Box > Draw Text Box. (If you need to do this often, add the Draw Text Box button to the Quick Access Toolbar.)
  • Draw a text box so its edges match the edges of the table cell. (If the box jumps out of place, don't worry about it; you'll fix that in a later step.)
  • Right-click the edge of the text box and click Format Text Box.
  • On the Colors and Lines tab of the dialog, click the Fill Effects button.
  • In the Fill Effects dialog, click the Picture tab and click the Select Picture button. Select the image file. Check the "Lock picture aspect ratio" box and click OK.
  • Back in the Format Text Box dialog, set the transparency to 50%. (You may have to set this higher or lower to make the text readable against the background.)
  • Click the Layout tab of the Format Text Box dialog, click Behind Text, and click the Advanced button.
  • On the Position tab of the Layout dialog, set the Horizontal Absolute Position to 0 to the right of Column, and the Vertical Absolute Position to 0 below Paragraph. Click OK.


To add this background to other cells, copy the text box (you may have to click Home > Select > Select Objects to be able to select the box instead of the text) and paste it into the other cells.

I saw the archive file size has increased, but the original Outlook data PST file is still 2GB.

Any idea why? thnks.

nayana s m say: Issue regarding vcard, it is by design in Outlook 2010. You cannot disable it.

brian tillman [outlook mvp 2007-2019] say:

It is a design feature of PSTs to keep their size for a while even when data has been removed.  This is called white space.  Asking the file system to allocate more space to a file is "expensive" - it takes more computer or i/o resources.  So, when data is deleted from a PST, the space it occupied is marked for reuse.  Reusing already allocated space is much faster than getting more.

Over time, as the white space reaches a particular percentage of the PST, Outlook will compact the PST and release the unused allocation back to the free space on disk.  You can also force a compaction by right-clicking the root of the folder tree, choosing Properties, clicking Advanced, and then clicking Compact Now.
