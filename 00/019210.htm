john mcghie say:
Since no-one else has been silly enough to try to explain what is going on, I will stick my neck out and contribute my wild-ar$ed-guess as to what is happening.

Word stores a vector image in two or three "layers".  One layer is the original vector, the middle layer is a "screen display version" and the remaining layer is the "placeable header".  (This is not a scientifically-accurate description, I am using a metaphor).

If Word "looses" the vector information, it will go looking for the intermediate, and if that doesn't exist, it will show the placeable header.  Which is created at 72 dpi and looks really pixelated. The displayable image will exist only if Word needs it (according to complex rules, its purpose is to speed up display of the document).

So what we have here is a document in which Word has "lost" the link to the vector information.

That can be caused by a huge variety of things, but my first suspicions fall on Track Changes, Math Equations, Footnotes, Floating pictures.

If there are any tracked changes in the document, make a copy, accept all changes in the document, then re-insert the bad pictures. This time they should stay crisp.

If you think it's not that, email the file to me at john (at) mcghie.name and I will see if I can figure it out for you.  Don't forget to say in your email who you are and what the problem is: I get a lot of email :-)  Oh:  Best to send the email in Plain Text: HTML email often gets dumped by the spam filter.

andreas killer say:

Don't do it that way, a much better and easier way is to use the cell format feature.

Eg. write the date to start into A1 and format the cell as you like e.g. use
dd. - dddd

After that apply the formula

=A1+1

in A2.

If we drag that down, we are done.
If you want to stop the dates at the end of a month, you have to test if the month changes and return "" instead.

=IF(MONTH(A1)=MONTH( A1+1),A1+1,"")
