jay freedman say:

Yes, you will need VBA for anything that complex. ("Not complex" is pretty much limited to repeating the same information in two or more places, with cross-references or STYLEREF fields.)

A crucial point in designing a document (or, better, a template) to be automated with VBA is to be able to refer to the changeable locations by name. That can be done by inserting bookmarks, form fields, or content controls at those locations. The alternative, using the Find function to search the document for specific text or custom styles, tends to become unwieldy when there are more than a few changes to be made.

Another point comes up when you plan to delete parts of a document that were inserted with building blocks: When you insert a building block, it becomes just part of the document and isn't distinguishable as specific building block. That is, your code can't simply say "delete building block X" because there's no way to recognize it as X. The closest you can come to that is to use a 'building block gallery content control' to insert the proper building block for X, and then you can assign the title and/or tag of that content control. (This type of content control is discussed in the last half of http://gregmaxey.mvps.org/word_tip_pages/build_deploy_custom_bb_galleries.html.)

laura lct say: Hi Jay - I appreciate all the info, links and quick response time.  I'd like to know how far logic can be added without VBA  using the building block control, bookmark and an IF field. jay freedman say:

I can make that IF field work, but only if the building block that you want to use as the "trigger" is text with a usable length. That's because the IF condition has to compare the value of the bookmark's contents to the value you want.

As an example, insert a building block gallery content control and set its Properties dialog to use the AutoText gallery. Now, most AutoText entries are text, but they can contain anything that can be in a document, including graphics, tables, embedded Excel ranges... So there's no guarantee that the building block chosen in the gallery control will be text.

Insert a bookmark around the content control; let's give it the name BB1.

Next, insert your IF field with this code (note that the pair of braces around BB1 and around the entire field are created by pressing Ctrl+F9, not by typing braces from the keyboard):

{ IF { BB1 } = "ATTN:" "Yes" "No" }

Now you can test this. First click the tab of the gallery content control and choose the built-in building block that consists of the text ATTN: . Click on the IF field and update it with F9 -- it should say Yes.

Click the gallery content control again and choose any other building block. When you update the IF field, it will say No.

As an additional test, insert a picture in the document, select it and press Alt+F3, give it a name, and make sure the Gallery box says AutoText; click OK. You can delete the picture from the document; it has been stored in the Normal template as an AutoText entry. Select that entry as the value of the content control, and update the IF field -- it will still say No because any picture cannot compare as equal to the text "ATTN:". However, it won't be possible to test for the condition "BB1 equals this particular picture" because Word has no way to compare pictures.

john korchok say:

When I open the file in Word for Windows, I notice it's set up for book fold printing, which is why it's trying to fit 2 pages on 1 sheet. Unfortunately, that feature is not available in Word 2011, so there's no way to turn it off in that version.

Then it has a non-standard page size. You could still print the booklet spreads to PDF, but you would have to create a custom page size:

  1. With the file open in Word 2011, choose File>Page Setup
  2. Change the Paper Size dropdown to Manage Custom Sizes
  3. Click on the Plus sign below the list of custom page sizes. 
  4. Double-click on Untitled and give it a name like 9x15
  5. In the Paper Size fields, enter 9 in Width and 15 in Height. (Width is always the smaller measurement in page sizes)
  6. Make all the margins 0. Click on the OK button, but then on the Page Layout dialog, click Cancel. (This page size is only for printing, you don't want to apply it to the document).
  7. Back in the document, choose File>Print.
  8. In the Print dialog, click on Page Setup. 
  9. Select the 9x15 page size and make sure it's landscape. Click on OK
  10. Back in the Print dialog, click on PDF and choose Save as PDF. Name the PDF and click on Save. The file is printed 2 pages per sheet.

As an alternative for a quick print for proofing purposes, you can also not bother with all the above. Just go to File>Print, change Copies & Pages to Paper Handling, check Scale to fit paper size, choose a standard page size. Choose PDF>Save as PDF, name it and click on Save. The pairs of pages will be scaled to fit the chosen paper.

hansv mvp say:

If all cells in column F are blank, selectRange will be an empty range, and you cannot copy an empty range.

So insert the following lines above the offending line:

    If selectRange Is Nothing Then
        MsgBox "There are no cells to copy", vbInformation
        Exit Sub
    End If

By the way, if you are working with an Excel 2007-2013 workbook (.xlsx, .xlsb or .xlsm), each column will have 1,048,576 rows, so looping through all cells of column F is not a good idea. I'd use

    For Each cell In Range("F1:F" & Range("F" & Rows.Count).End(xlUp).Row)
