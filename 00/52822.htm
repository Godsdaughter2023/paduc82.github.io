Depending on your set-up, we cannot determine the path where you have saved the mails within your hard drive. what we can only provide are steps in case you've backed up your emails thru the web.

 

 Please check the following information below on how to recover your lost folders:

  • In case you're using POP account, check if you're leaving a copy of the email on the server. (To check, click on mail account in WL Mail>Properties>Server>Delivery). - In Explorer, %USERPROFILE%\Local Settings\Application Data\Microsoft\Windows Live Mail\<Email account name>\Deleted Items

  • If email is no longer available on server itself, there is a low chance of these mails will be recovered.
  • 'Storage folders' Deleted Items only contains items deleted from Windows Live Mail’s user created folders. If the program is functioning properly and the deleted messages are not present in the email account's ‘Deleted Items’ folder or not archived elsewhere in Live Mail or the user's hard drive, unfortunately, these files cannot be retrieved.

cmdrkeene say:
The behavior is dependent on the file system used.  Most external drives use FAT32 -- a system from the early 1990's that was replaced with systems that can hold more data and are more reliable.

To work around it, consider arranging by the "picture date taken" instead of the "file date modified".  You can do this in details view by right-clicking the column header and choosing "more...".  Or, if you are in your pictures library (and have it set to show contents from your external drive), you can even stack-up the pictures by their taken date.

r-p-h say:

Hi,

 

Thanks for the reply.

 

Unfortunately folders cannot be arranged by date taken. As I said it is strange that my folders date modified changes however my pictures does not.

 

This doesn't happen when I copy folders from my PC to my HDD on my vista PC however it occurs when I copy folders between HDD's on my windows 7 pc. Not sure then if it is because of copying between 2 HDD's ?

cmdrkeene say:
Possibly.  I know that moving on the same drive does not change the modified date, because nothing is actually moved at all (the bits on the drive stay in the same spot, only the library record of where to show the folder is actually changed).  But if you move it to a new drive, the file system on the new drive sees that as an entirely new file.
