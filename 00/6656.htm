andreas killer say:
Don't worry about that a DropButtonClick event doesn't fire, this is not possible.

http://msdn.microsoft.com/en-us/library/ee440968%28v=office.12%29.aspx?ppud=4

I've tried an other way without using this state and used a loop until KeyPressed or MousePressed, but this causes big troubles in some situations.

The biggest problem is not that an event doesn't fire, the problem is that more events fires from event routines if your code contains "DoEvents". Okay, that sounds complicated, and well, it is so.

We can make it a little bit safer with this TimerFunc:

Private Function DropDownTimerFunc(ByVal hwnd As Long, ByVal wMsg As Long, _
    ByVal nEvent As Long, ByVal nSecs As Long) As Long
  'Calls the DropDown method until the dropdown is opened or a key or mousebutton is pressed
  On Error GoTo ExitPoint
  Do
    DoEvents
    DropDownForm.ActiveControl.DropDown
  Loop Until DropDownVisible
ExitPoint:
  KillTimer hwnd, nIDE
End Function

And now change the code in the form as follows for a test:
- Comment out the "DoEvents" in the ComboBox1_Exit event routine
- Comment out any DropButtonClick event routines

[jokes on]

I hear you talking: "This guy is crazy, why should I do this, the code can not work with this changes and will hang, because DropDownVisible will never get True!"

But stop talking and stop thinking for a moment, save the file and run the form.

After a few tries I hear you shout: "Hey, that's much better! I don't need this DropDownVisible boolean! Let's get rid this out of the code!"

[jokes off]

The code works, because "DropDownForm.ActiveControl.DropDown" causes an error if the control changes during the loop. I know it sounds impossible that this could be possible, but it is so.

The DoEvents inside the loop let other events run, this is important so that Excel can open the dropdown. But it let also Excel / VBA / Windows continue other codes!

And now the crash (close any other task, save any file, this is critical!):

Enable the "DoEvents" in the ComboBox1_Exit event routine and try again.

Use the Taskmanager to kill Excel.

I'm not sure why this is an overkill and the only way I know to prevent it is to use the state of the dropdown and exit the loop if it opens. I've tried a few situations and found that it's safe.

Certainly we can make it a little more comfortable by integrating a class module, which takes over the DropButtonClick events. (Unfortunately we can not take over the Enter event, see help to RTE 459) But I think so it's easier to understand.

Andreas.

greginmich say:

How do you place a short cut for Windows Defender for Windows 10?

If you’re just looking for a way to make Windows Defender a little more accessible, then this information might prove useful:

To unhide the Windows Defender icon:

The notification area icon was returned in Version 1607 of Windows 10, but it’s hidden by default – so in order to unhide it you have to either drag it out of the hidden icons box and drop it in the notification area proper – or else – right-click on the taskbar and choose Settings > Notification area > Select which icons appear in the taskbar, and then turn on the Windows Defender notification icon.

To put a shortcut on the taskbar itself,

Click on the Start button and scroll down to Windows System > Windows Defender, and then right-click and select More > Pin to taskbar.

To add a desktop shortcut  that performs a specific action:

Starts Windows Defender:

"C:\Program Files\Windows Defender\MSASCui.exe"

Runs a Quick Scan:

"C:\Program Files\Windows Defender\MSASCui.exe" -QuickScan

Runs a Full Scan:

"C:\Program Files\Windows Defender\MSASCui.exe" -FullScan

Updates the signatures for Windows Defender:

"C:\Program Files\Windows Defender\MSASCui.exe" -Update

Updates the signatures and runs a Quick Scan:

"C:\Program Files\Windows Defender\MSASCui.exe" -UpdateandQuickScan

1.) Select and copy the run line for the option that you want.

2.) Right-click on the desktop and select New > Shortcut and then paste the run line as the location (target). Then click Next, name the shortcut, and click Finish.

To automatically run a Quick Scan, signature update, or update and Quick Scan every time you start the PC:

1.) Open File Explorer: Start Menu > File Explorer; and then type (or copy and paste) shell:startup (user Startup folder) or shell:common startup (common Startup folder) into the Quick access box and hit the Enter key to open the Startup folder:

2.) Then drag and drop the desktop shortcut into the Startup folder.

To run an unattended Full Scan:

Follow the steps in this thread to create a batch file that manages the power configuration in order to keep the PC awake, and also (optionally) shuts down the PC when the scan is finished:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-can-i-run-windows-defender-walk-away-and-have/7b57798e-b2da-40a6-9c74-afd9ff2dff11

For example - I am tracking health insurance coverage and payments.  My master form would be at the claimant level, there would be 2 tabs, one for a Medical coverage subform and one for dental coverage subform.  Would this be 2 levels (Master/Subforms) or 3 (Master/Medical tab/Dental tab)? 

 

Within the medical and dental subforms would be provider subforms, nested within those would be premium payment subforms.  The problem is figuring out how many levels this would add up to 4 (Master/Coverage Type/Provider/Premiums) or 7 (Master/Medical Coverage Type/Medical Provider/Medical Premiums/Dental Coverage Type/Dental Provider/Dental Premiums)?

 

For ease of design, my plan is to build each subform individually and then gradually combine them until I have the master form but if this presents problems I will resconsider the approach.

john w. vinson say:

The nesting refers to subforms WITHIN subforms. A Tab Control is nothing but a tool to manage screen space - two subforms on different tabs are (logically) treated exactly as if they were two subforms side by side on the main form. The only limits on the number of subforms arranged in this way are the 728 total form objects limit and practical considerations of performance; an excessively "heavy" form with dozens of subforms can be very slow to load and to move from record to record, since each subform must be populated before the current record can appear on the mainform.

One solution is to have the subforms on the Tab Control stored with empty Recordsource properties; you can set the Recordsource for the subform(s) on a tab page in the tab control's Change event, so that only one pages' subforms need to be active at any one time.

 

hansv mvp say:

Your description would have 3 levels: main form - subform - subform of subform, i.e. 2 levels down from the main form. You can have as many of these subform - subform of subform combo's as you like (within practical limits)
