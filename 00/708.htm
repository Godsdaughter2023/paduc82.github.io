Run a macro containing the following code when the selection is in the table:
 
Dim i As Long, j As Long, k As Long, l As Long, n As Long
Dim temp As String
Dim vartemp, strTemp() As String
Dim crange As Range
temp = ""
With Selection.Tables(1)
    i = .Rows.Count
    j = .Columns.Count
    For k = 1 To i
        For l = 1 To j
            Set crange = .Cell(k, l).Range
            crange.End = crange.End - 1
            temp = temp & "," & crange.Text
        Next l
    Next k
    temp = Mid(temp, 2)
    vartemp = Split(temp, ",")
    strTemp = vartemp
    WordBasic.SortArray strTemp
    n = 1
    For k = 1 To i
        For l = 1 To j
            .Cell(k, l).Range.Text = strTemp(n - 1)
            n = n + 1
        Next l
    Next k
End With

bob jones aka: cybertaz say:
No argument with your geometry, but I don't see the concern being the Area (H x W) occupied by the bounding box containing the line. He's concerned about the size of the Shape contained within the box, i.e., the length & arc of the Line.

In this screen shot Object B is a duplicate of Object A scaled to 500% of the original size. The starting/ending points of B are 5x the distance apart & the distance from 'base' to apex is 5x that of A. Ergo, Line B itself is approx. 5x the length of Line A.

It's not at all accurate to do so, but Deleting all points of each path, B winds up being a straight line roughly 5x the length of straight line A.

At any rate, it's been pointed out several times that precision is not a hallmark of the drawing tool accessories in Office apps. The OP will have to experiment & adjust on his own in virtually every case. Again I emphasize that if accuracy & proportion are concerns other software should be used, but it now sounds as if approximation is sufficient.
