ken sheridan say: A bound combo box will normally have as its ControlSource property the name of a foreign key column (field) in the table to which the form is bound.  The RowSource property will normally be a Query which returns rows from the table which the foreign key references.  For example a combo box might be bound to a CityID foreign key column in a table of contact addresses, in which case the RowSource property would be along theses lines:

SELECT CityID, City
FROM Cities
ORDER BY City;

The only situation in which the combo box needs to be requeried is if a new row is inserted into the referenced table, Cities in the above example.  This would normally be done by code in the control's NotInList event procedure which includes a Response argument whose value is set to acDataErrAdded to requery the control and add the new item to is list.

From your description it seems you are referencing the same table in the control's RowSource query rather than a separate referenced table.  If so, the table needs to be decomposed into two related tables, so that the combo box can be bound to the foreign key column in the referencing table, while the combo box draws its list from the referenced table.  The one-to-many relationship between the two tables should include the enforcement of referential integrity.

ken sheridan say:

It's not as simple as that I'm afraid.  acDataErrAdded is a built in constant whose value is assigned to the Response argument of the NotInList event procedure in code.  Where the referenced table has only the one column in which data is inserted the code can do so transparently, as in this example for inserting a new region via a region combo box in a form based on a Cities table in which the combo box's query returns rows from a Regions table:

Private Sub RegionID_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control
    Dim strSQL As String, strMessage As String

    Set ctrl = Me.ActiveControl
    strMessage = "Add " & NewData & " to list?"

    strSQL = "INSERT INTO Regions(Region) VALUES(""" & _
            NewData & """)"

    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        CurrentDb.Execute strSQL, dbFailOnError
        Response = acDataErrAdded
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If

End Sub

In a situation where there are other columns in the refered table, as in the following example for a city combo box in a form based on a Contacts table in which the combo box's RowSource returns rows from a Cities table, the code in the NotInList event procedure opens a form, in dialogue mode, based on the Cities table and passes the new city name to it as the OpenArgs argument of the OpenForm method:

Private Sub City_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control
    Dim strMessage As String

    Set ctrl = Me.ActiveControl
    strMessage = "Add " & NewData & " to list?"

    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        DoCmd.OpenForm "frmCities", _
            DataMode:=acFormAdd, _
            Windowmode:=acDialog, _
            OpenArgs:=NewData
        ' ensure frmCities closed
        DoCmd.Close acForm, "frmCities"
        ' ensure city has been added
        If Not IsNull(DLookup("CityID", "Cities", "City = """ & _
            NewData & """")) Then
            Response = acDataErrAdded
        Else
            strMessage = NewData & " was not added to Cities table."
            MsgBox strMessage, vbInformation, "Warning"
            Response = acDataErrContinue
            ctrl.Undo
        End If
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If

End Sub

In the cities form's Open event procedure the value passed to the form is assigned to the DefaultValue property of the City text box control, ready for the user to select or enter the region in which the city is based:

Private Sub Form_Open(Cancel As Integer)

    If Not IsNull(Me.OpenArgs) Then
        Me.City.DefaultValue = """" & Me.OpenArgs & """"
    End If

End Sub

It is possible to avoid using any code at all.  In the case of the contacts form, for instance, setting the ListItemsEditProperty of the city combo box to the name of a form bound to the Cities table would cause the cities from to open when a new city name, not currently in the list, is entered into the combo box.  Unlike the above code the new name is not passed to the form, however, so you would then have to type the name again into the City text box in the form.

heart:       2665

spade:      0170

club:         0167

heart:       0169

diamond: 0168

Probably the best way, once you've set it up, is to use the Insert > Symbol > More Symbols dialog box.

  • In that box, at the top, choose your preferred font (common ones that include the symbols are Arial, Courier New, Times New Roman, Lucida Sans Unicode, and Segoe UI Symbol).
  • In the character code box near the bottom, type one of the numbers from the first list above, and click the Insert button.
  • Repeat for the other three symbols, and then close the dialog box.
  • Once you've done this, the four symbols will appear in the dropdown from the Insert > Symbol button on the ribbon, and you can just click one to insert it into your document.

Windows Defender can now immediately block suspicious or unknown files; and then automatically upload and analyze a sample, and generate a signature; all within a matter of seconds

Controlled Folder Access:

Controlled Folder Access is turned off by default; so you’ll need to turn it on in the Windows Defender Security Center app > Virus & threat protection > Virus & threat protection settings. Once Controlled Folder Access is turned on, standard document folders will be protected by default, and you’ll also have the option to add ransomware protection to additional folders, as well as whitelist trusted applications in order to allow them access to your protected folders. If you have trouble whitelisting a friendly app, then you can set this feature to run in Audit Mode, where it will identify intrusion attempts, but won’t block them. Run this command at the Administrator PowerShell Prompt to set Audit Mode:

Set-MpPreference -EnableControlledFolderAccess AuditMode

$Preferences = Get-MpPreference
