halima s say:

Hi ILUVHAWAII,

Welcome to Microsoft Answers Forums.

By default, your computer goes to sleep after a short period of inactivity to conserve energy. To keep the computer awake longer, increase the amount of time of the Sleep after setting.

To keep your computer awake longer

1. Open Power Options by clicking the Start button, clicking Control Panel, clicking System and Maintenance, and then clicking Power Options.
2. In the left pane, click Change when the computer sleeps.
3. On the Change settings for the plan page, in the Put the computer to sleep list, click the amount of time under On battery or Plugged in (or both), and then click Save changes.

How is "input language" different from a "keyboard layout"?

In the "Text Services and Input Languages" dialog box, there are options for setting "Hot keys for input languages". What puzzles me is the "Change Key Sequence" dialog box. It has two sections: "Switch Input Language" and "Switch Keyboard Layout". Exactly how is an "input language" different from a "keyboard layout"? What exactly is an "input language"?

I'm pretty sure I know what a keyboard layout is... or at least I thought I knew up until now. I know about Qwerty, Azerty, Dvorak and Qwertz. These are the layout models that most keyboard layouts are based on. If I had to guess I would say that an "input language" (in Microsoft terminology), is any language-specific, functional keyboard layout, which may or may not match up with the visual layout of the computer keyboard. So in a nutshell, it's a damn keyboard layout?! So why is Microsoft trying so hard to make a distinction between "input language" and "keyboard layout"? Now go ahead, try to explain that to me, please. Any Microsoft MVP's willing to explain this?

My problem was with the Ctrl + Shift hotkey. I use one display language and three "input methods". I often unintentionally triggered the hotkey while typing. I have managed to disable it now and all is well. But I would really like someone to explain the difference between "input language" and "keyboard layout" (and also "input method"). The hotkey Alt + Shift is supposedly used to "switch input language". But in my case, it's not doing a damn thing?! The Windows 8.1 hotkey for switching keyboard layout or "input method" is Win + Space.

My "display language" is English. Adding a new "language" in Windows 8.1 is basically about selecting a language from a long list of languages. A "language" in this sense is like a package (not to be confused with language pack) consisting of one "input method" and one "display language". So you can't have one without the other. If you add "Swedish" language, Windows both adds Swedish keyboard layout, but it also changes the display language (if I remember correctly). It also effects the language of some Windows 8 apps (the ones that don't have language options). So if you want English to be the display language, not only in the Desktop (app) environment, but also in the Windows 8 apps, you have to make sure you have only English display language, and English input method (keyboard layout), and then click Options and add additional input methods, like one for Swedish. (This is in contrast to adding a "language".) Then you have to go to advanced settings and set "override for default input method". This makes sure that "input method" (keyboard layout) doesn't fall back to English on English display language (but stays put at Swedish on English display language).

Setting language preferences is a bit easier in Windows 8 than in previous versions. The best thing is that MUI language packs are now available to all "editions" of Windows. It doesn't matter if you have Windows 8 or Windows 8 Pro. But this terminology is very confusing. Input method? How about "typing on a keyboard"? Input language? Keyboards and computers are not aware of a "language", they use scancodes. I also wish that language preferences worked more seamlessly (with no need for overrides) throughout the entire system (including the brave new world of "modern apps"). But all in all, Windows 8 is a good step forward in terms of language preference settings. Being able to use your own primary language as the display language, without having to pay for premium versions of Windows like the Windows 7 Ultimate makes sense.
