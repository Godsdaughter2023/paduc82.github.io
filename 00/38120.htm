malletkatman say:

I found out what happened.  When copying the desktop icon for the program to the target folder, a copy of the actual program file, rather than a shortcut to it, was copied to the folder, making (apparenyly) two copies of the program on the system.  (Probably the desktop icon/item was the file and not a shorcut link to it somewhere else.)  Realizing this I continued theDelete File operation in the folder, the folder entry went away, and the desktop icon/program remained and continues to run when selected.  So when moving or copying an item, one should check the item's properties first.
