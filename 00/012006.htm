Exporting XML - Microsoft Community

rothrock42 say:

Exporting XML

I have a spreadsheet in Excel 2010 that I want to turn into an XML file.

 

I've figured out how to turn the the Developer section on in the Ribbon.

 

All the tutorials I've found require the use of an XSD file for mapping the data to export. I tried a simple one for part of my data and it seemed like an awful lot of work for a very simple return--at least on this project.

 

I want to export every row of every column and I want the node to be named by the heading of the column.

 

Is there a simpler (faster) way to do this?

 

chip pearson say:

You can roll your own XML writer, bypassing all the XML hoops in Excel. The code below will write XML out to a text file. Change the range RR, marked with '<<<, to the range of data, including the header row, that is to be written out to the XML file. The root element is named <elements> and each row of the data range is named <element>. The values within <element> are named from the header row and have a value from the data range. This code creates quick and dirty XML; no namespaces, no support for attributes, etc.,, just well-formed XML with data from the worksheet.

 

Sub CreateXML()
    Dim HeaderR As Range
    Dim R As Long
    Dim C As Long
    Dim FNum As Integer
    Dim FName As Variant
    Dim RR As Range
    Dim S As String
    Dim N As Long
   
    FName = Application.GetSaveAsFilename( _
        FileFilter:="XML Files (*.xml),*.xml")
    If FName = False Then
        Exit Sub
    End If
   
    Set RR = Range("A1:C4") '<<< CHANGE. INCLUDE HEADER ROW.
    Set HeaderR = RR(1, 1)
    FNum = FreeFile
    Open FName For Output Access Write As #FNum
    Print #FNum, "<?xml version=""1.0"" encoding=""utf-8""?>"
    Print #FNum, "<elements>"
    For R = RR(1, 1).Row + 1 To RR.Cells(RR.Cells.Count).Row
        N = 0
        S = Space(4) & "<element>" & vbNewLine
        For C = RR(1, 1).Column To RR.Cells(RR.Cells.Count).Column
            N = N + 1
            S = S & Space(8) & "<" & HeaderR(1, N) & ">" & _
                CStr(RR.Worksheet.Cells(R, C).Text) & _
                "</" & HeaderR(1, N) & ">" & vbNewLine
        Next C
        S = S & Space(4) & "</element>"
        Print #FNum, S
    Next R
    Print #FNum, "</elements>"
    Close #FNum
End Sub

butterflyone say:

Word perfect to MS Word. I'm trying to get my word perfect quickwords into MS word's auto correct. Please help.

I have a lot of "quickwords" on Word Perfect, but I am changing over to MS Word, and I'm looking for a way to get these into MS Word auto correct.  Please Help
butterflyone say: First, I want to especially thank Suzanne for hanging in there with me.  Without you, I would not have been able to finally get this to work.  Thank you, thank you, thank you!!!  And thanks to everyone else who tried.   Jay, I didn't know before I went through the steps I did that running the AutoCorrect macro didn't replace but only added to AutoCorrect.  I was afraid I would lose what was in AutoCorrect and that it would be replaced by the info on the document I was using.  I guess I could have saved some time if I had read that before I did what I did.  Anyway, below I'm putting the steps that finally worked in getting the Word Perfect quickwords into MS Word AutoCorrect for those who may still be looking for a way.  I hope my struggles in finding this out will help many out there who seem to be looking for a way and can't find an answer.  Again, Suzanne, thanks sooooooooooo much for your help.  

Okay, here it is; the steps that I used, which worked, to convert Word Perfect QuickCorrect to MS Word AutoCorrect.  Please note that I have MS Word 10 and Word Perfect 8.  Also, I made copies of each document below and renamed them and then used the copies to work with in case I messed up.  Note that these macros can be found online using a Google search.  Before running the AutoCorrect.dotm make sure you have your macros enabled through the trust center, but be sure to go back to your trust center and fix it back afterward.

1.      Use Macro “QWManage.wcm” in Word Perfect to make a document  of the quick words.

2.     Use Macro “AutoCorrect.dotm” in MS Word to make a document of the AutoCorrect words.

3.     I put the copy I made of the Word Perfect QuickWord document on my desktop and then opened it in Word using “all files (*)” on the drop down box.

4.     “Ctrl+A” to select all and then change font to Times New Roman 12 (since I had a different font I was using in Word Perfect, and Word’s document was in this font)

5.     Take off the heading that shows on the top of the document

6.     Ctrl+Shift+8 (which actually is Ctrl+*) to show all the codes.

7.     Ctrl+A to select all.

8.     In “editing” choose “replace” and in the find type ^p and in the “replace with” leave it blank.  Click “replace all”.  This will take out the paragraph markings, or so it would seem (more on that later).  Don’t worry about the other little markings that look like a circle with hash marks on four sides.

9.     Now, here’s where it gets a little tricky.  Click on the little box that appears when you scroll to the left of the table; this will highlight the table, BUT – be careful, because as I found out, for some reason I had more than one table.  Anyway, you’re going to have to do the next step for each table that comes up.  I ended up with about 4 different tables; I don’t know why.

10.       Okay, with the table highlighted, choose “layout” on the table tools and then convert to text (separate by tabs, which should be the default), and click OK.  You will then see the first abbreviation then a little arrow (tab symbol) and then the full word followed by paragraph symbol, which we thought was supposed to be gone.  But that’s okay, leave it.

11.          Now, if you have more than one table, which I did, you have to go down that list, which should still be highlighted, and find at the end where it says “section break (continuous)” in the middle of a grayed line that follows the last quick word that converted to text.  This line is why you have more than one table.  Take that line out, not the quick word, just the section break line that follows it. 

12.        Now, follow the preceding steps and do the same to the next table and so on.

13.         When you have converted  all the tables you have to text and have removed the section breaks, then “Ctrl+A” to select all.

14.          Press Ctrl + H to open the Replace Dialog.   In the “find what” type ^p (which represents a paragraph mark), and in the “replace with” type ^tFalse^p  (a tab character, the word “false”, and a paragraph mark).  Choose “replace all”  This should add a tab character and the word “false” to the end of every paragraph, in this case the end of each line.

15.        Now, press Ctrl+ A to select all. 

16.        On the “Insert Tab”, in the Tables group, click on the Table button and choose “Convert text to table.”.  Again, tabs should be the default choice for “separate text at,” and 3 columns should be indicated.

17.        Click OK to make the conversion.  This should give you a three-column table that you can copy and “merge” with the table in the AutoCorrect Word table.  Again, please use the copy you made of the AutoCorrect Backup document to work with just in case.

18.         Now, copy the table you just made from the quick words, and when you go to paste it at the bottom of the AutoCorrect Word table, use the “merge” tables option.

19.       Now, on mine, what I added came out all looking underlined, but I found out later that it didn’t matter in the end.  Next, I sorted the table using the “Table Tools” layout tab and then “AZ Sort”.  This then gave me a complete table alphabetized from A to Z.

20.       Another tricky part next.  Now you have to use the “AutoCorrect.dotm” again and choose “replace” to put your new table into AutoCorrect, BUT it will most likely challenge you for different reasons.  What I found out (in order to save someone else time) was that it replaced up until it came up with the error, so after searching for what the codes meant and what I could do to fix it, I finally just went through the document and checked the abbreviations sporadically (the ones I added were still underlined so I could tell which ones they were) to see if they worked, and when they didn’t, I fine-tuned it in order to find where the hold- up was.  Once I determined where it stopped working, I took out that row by clicking on “Table Tools”, Layout, and then clicked on the table with the red X on it (on the left side of the ribbon) and deleted that row. 

21.        Run the macro again and see if you get another error and repeat as necessary.  I had to take out about five rows in the end.  Once again, I don’t know why, but I just did what I had to do.  Each time I ran the macro, though, it captured more of the table until it finally gave me a completed table.  Hip Hip Hurray!!!!!  IT WORKS!!!!

22.        One thing though, if you have formatted things in your MS Word Auto Correct already, like things with columns and such, make sure to check they’re still formatted correctly.  I had to redo a few of mine.  Once again, I don’t know why since these were already in MS Word Auto Correct, but it was what it was, so to speak, so I just corrected them.
