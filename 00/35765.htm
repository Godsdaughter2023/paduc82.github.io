a. user say:

"...  finding out that my tower PC is no longer supported by Windows 10."

Wrong.

If anything, it's the manufacturer of your computer who does not support it on Windows 10. That means the manufacturer has not tested your model on Windows 10, has not attempted to keep it current with the changes in Windows 10, and won't help you if your computer has a problem with Windows 10.

So if you want to complain ... complain to the manufacturer. Ask them why they won't support a 3-year old computer on the newest version of Windows. Tell them that you'll never buy that brand again.

If it was up to Microsoft, every computer in the world would be supported on Windows 10. As far as Microsoft is concerned, the more computers that can run on Windows 10, the better.

Now here's a tip for you: Just because the manufacturer isn't supporting your computer any more, that doesn't necessarily mean it won't run well on Windows 10. Dell doesn't support my (9-year old) computer on Windows 10, but it runs very well. Dell didn't support Windows 8.1 on my computer, and that ran well. They also didn't support Windows 8 and that ran well too.

Lack of support means you have to do the work that your manufacturer didn't do: You have to do the research and the testing, you have to replace the components that won't work on Windows 10 with components that will work (if that's even possible,) you have to search for updated drivers (if they exist) and you have to find supported versions of your applications, or replace them with applications that are supported on Windows 10.

I did all that work and my reward is a dinosaur of a computer that runs very well on Windows 10, so far. I had to replace the OEM video card and my original WiFi adapter; I had to find Windows 10 drivers for all my hardware, and I had to toss a few of my cherished applications. My inkjet multi-function printer works sort of well.

The best place to do your research is the manufacturer's own community forum, where you can correspond with people who are using the same model of computer that you're using. Ask them if they got their computers to run on Windows 10 and how they managed to do it. That's what I did.

All this is a lot of work, but that's what you have to do because your manufacturer isn't supporting your computer on Windows 10. Of course it's much easier to blame everything on big, bad Microsoft, but not only will that not give you a working computer, you're showing everyone how much you know about computers.

mike hall... always say:

I remember that program. It shipped with Corel WordPerfect 7 Office Suite.

 

The 'not enough memory' doesn't apply to RAM installed. You get the message because the memory addresses used by the program are in conflict with what Windows is using.

 

The problem with many of the older programs is that they were written badly. In an attempt to make programs run better, there was a tendency to use memory addresses that shouldn't have been used. The premis was that the memory addresses would most probably not be used by the majority of users because the stuff normally using the addresses would be beyond the average user.

 

Such was computer life in the days of Windows 9x. As Microsoft tighten up on OS integrity, and they have been since the introduction of XP, the once 'virtually free' memory addresses have become inaccessible to whatever 3rd party software authors were unofficially using.   

 

So now you have a choice. Do you keep the program to which you have become attached over the years, or accept the Windows updates which at the end of the day keep your computer and OS more secure?

 

To find out which update caused the issue, you will have to uninstall the latest updates one by one until the 'error' corrects itself, then re-install all of the other updates.

I apologize for not addressing you in a timely manner. Let me help you in resolving the issue.

Failure of Remote Procedure Call and Remote Procedure Call Locater services may also result in the this error. So you should firstly confirm that these services are functioning properly.

I would suggest you to check the status of Remote Procedure Call locator service. Make sure the Remote Procedure Call status is Started and set to Automatic. RPC Locator should be set to Manual. Follow the below steps for the same:

a. Click Start and type services.msc in the search box

b. Search for the Remote Procedure Call locator service

Make sure the start- up type is manual. Steps to set the services to automatic

c. Search for the service

d. Right click on the service

e. If the service is stopped then start the service

f. Select Properties

g. And in the Start-up type select Manual

Keep us posted on Windows related queries and we will be happy to assist you further.
