The reason your are not seeing the full 4Gb of ram is because this is by design, some of that space is used by you different hardware components and is not available to the operating system and applications. The amount you can use varies, depending on what hardware you have installed, but can range from as little as 2GB to as much as 3.5GB. It's usually around 3.1GB, and you are very close to that.

The reduction in available system memory depends on the configuration of the following

  • The devices that are installed in the computer and the memory that is reserved by those devices.
  • The ability of the motherboard to handle memory.
  • The System BIOS version and settings.
  • The version of Windows 10 that is being used.
  • Other system settings.

Note The amount of usable memory in the examples are not exact amounts. Usable memory is a calculated amount of the total physical memory minus "hardware reserved" memory.

volunteer j say: FWIW...Windows Live Photo Gallery has a database file
that can become corrupted. (uninstalling / reinstalling WLPG
does not affect this file)

If you rename the file...it will force Photo Gallery to create a
new database file the next time it's launched and the gallery
will be as it was the first time it was launched.

I'm not positive that this will solve the problem you are dealing
with but if you would like to give it a try thefilename is...Pictures.pd6.

Just rename it to OLDPictures.pd6. This may cause you to
lose some tags and metadata that were not written back to
the photo files.

The path to the file is :

Windows\Users\%Username%\AppData\Local\Microsoft\
Windows Live Photo Gallery\Pictures.pd6

Pictures.pd6 is a hidden file...the following article explains
how to show hidden files:

Windows 7 - Show hidden files
http://windows.microsoft.com/en-US/windows7/Show-hidden-files

If this does not improve the problem...you can delete the
new Pictures.pd6 file and rename OLDPictures.p6 back to
Pictures.pd6 and the gallery should be as it was before.
