Make COLORFUL colour scheme the default for SmartArt in PowerPoint

Is there a way to set the colour scheme for a SmartArt Graphic to default to Colorful instead of the primary colour scheme (even if this means creating a macro)?

Thanks!

steve rindsberg say:

SmartArt inherits its colors from the color theme of the presentation.

A bit of vba can take the SmartArt colors "off-theme" by assigning specific RGB values to the individual shapes:

    Dim oSmartArt As SmartArt
    Dim x As Long
    
    Set oSmartArt = ActiveWindow.Selection.ShapeRange(1).SmartArt
    With oSmartArt
        For x = 1 To .Nodes.Count
            With .Nodes(x)
                .Shapes.Fill.ForeColor.RGB = RGB(255, 0, 0)
            End With
        Next
    End With

rob candee say:

How can I prevent a macro's workbook from taking focus when I invoke the macro?

I invoke macros from a custom tab in Excel.  The macros I invoke are in separate workbooks.  When I did this in Excel 2010, the macro would run, and the original workbook would remain the active workbook (the workbook which contained the macro would remain minimized).  However, in Excel 2013, after the macro runs, the workbook which contains the macro takes focus, and I have to minimize it to get it out of the way.  How can I prevent this from happening?

(I don't want to close the macro workbook, as there are additional routines in the workbook that I invoke with keyboard shortcuts as needed.  The last one of those closes the macro workbook.)

Thanks!

ossiemac say:

Try something like the following workaround. You might want to place each part in its own sub and call it from the subs that you call from the original active workbook. However, this will entail declaring wbActive and winStatus variables as public variables.

    'This code goes at the start of the called code (or in its own sub and call it from start of called sub)
    Dim wbActive As Workbook
    Dim winStatus As Long
   
    Set wbActive = ActiveWorkbook   'Workbook that calls the code
    winStatus = ActiveWindow.WindowState    'Save window status of workbook calling the code.

    'Main processing code here in lieu of the MsgBox
    MsgBox "Workbook called = " & ThisWorkbook.Name

    'This code goes at the end of the called code (or in its own sub and call it from end of called code)
    ThisWorkbook.Activate   'Workbook containing VBA code
    ActiveWindow.WindowState = xlMinimized  'Workbook containing the code
    wbActive.Activate   'The original workbook that called the code
    ActiveWindow.WindowState = winStatus    'Re-set window state of original workbook that called the code.
