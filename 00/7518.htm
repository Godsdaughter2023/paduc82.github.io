Hello Kenny

Short answer to your primary question is: No. There is no unified <Inbox> capability for IMAP accounts in any version of Outlook. Each IMAP account will have its own data file <Inbox>.

Also, if you re-configure all your POP accounts to IMAP, you also need to be aware of the impact it will have on contact/calendar folders. Would strongly suggest retaining the current PST file in the Outlook profile and set that data file as the "default data file" to continue using that data file for the storage of your default contact/calendar folders.

As a backgrounder, if you create a new Outlook profile to configure your IMAP accounts, the first account will have contact/calendar folders with the added words "This Computer Only" in the folder description and it is ill-advised to store your contact/calendar data in those folders. If you ever remove the account from the profile, the OST file associated with account will also be deleted and data in any "This Computer Only" folders will be lost.

Hi JanaM_286,

To answer your question:

  1. Is it possible to set the Library up so nobody can edit the document when it is in review/approval? The originator and people who have the permission to manage library has the permission to edit the document.
  2. Can we set up a folder/library or whole site so users cannot see documents which were submitted but are not yet approved? Yes, you can. If someone submits an item to a list or library that requires approvals, it is in Pending status until someone approves it. Usually, when an item is in Pending status, only the item originator and the people who have permissions to manage lists and libraries can see it. After its status changes from Pending to Approved, it becomes visible to anyone who has permission to view the list or library. Rejected articles stay in the list or library until the originator—or someone else who has the necessary permissions—deletes them.
  3. Can we add a description column - a field which would have to be filled in when submitting a document- in which a contributor would add a description of the document which would then help with search for the particular document on our site? You can create a new column in Library settings, let’s say “Originator Comments” and make it and say Yes to the Require that this column contains information field. When the originator will submit a new document, he/she will require to fill this column.

To set a library require approval, you can do it in Library Settings> Versioning settings, set Require content approval for submitted items? to Yes. In the Draft Item Security section, you can determine which users can see draft items in the document library.

Note: If security in the site library is configured to allow any user to read items, then anyone who has Read permissions to the site can see all items in the list or library, whether the items are in Approved status.

For your reference: Require approval of items in a site list or library
