A few things to try.

First, your last parameter doesn't look right.  If you're trying to send in the person's name in the report's OpenArgs, they just specify it.  You don't need the "=me.OpenArgs".

It looks like you're missing a [ before the first stxtPersonName.

Since you're providing a WhereCondition (the person's name), you should remove the parameter for PersonName from your query.  The only two parameters should be the start and end dates.  The WhereCondition for the report will take care of selecting just the one person.

scottgem say:

First, I rarely use Parameter prompt queries. They are very limiting. (see dhookum's reply here: http://answers.microsoft.com/en-us/office/forum/office_2013_release-access/parameter-value/bc96ba78-a3a8-4a48-b8cb-d1c3757c4e6d)  Instead I use a form to enter the criteria for my reports and reference those values using the syntax:

Forms!formname!controlname

Any value on an open form can be referenced using that syntax. 

There are at lease three problems with your code

1) its not clear what the field and control names are. If they are the same that the names are correct.

2) I'm assuming Person name is a text string but you are not treating it as such it should be:

"[stxtPersonName] = '" & Me.stxtPersonname & "'"

3) Names may not be unique so you should have a primary key for people table and that's what you should be comparing (i.e. "[PersonID] = " & Me.PersonID

As an aside, People names should be broken out into their conponents, not stored as a single field. Also, If you have names with apostrophes (I.E. O'Hara) then you will need to account for that in your concatenation.

I need to remove the two lines that appear under my words. How do I do this?
suzanne s. barnhill say:

Three possibilities:

  1. The text at the insertion point (or the default empty paragraph in a new Word document has acquired double underline formatting. If this is the case, then selecting the text and pressing Ctrl+Spacebar (ResetFont) will clear it.
  2. Your Normal style has acquired double underline formatting. You will need to modify the style. Display the Styles task pane (Alt+Ctrl+Shift+S or click the dialog launcher in the bottom right corner of the Styles group on the Ribbon). Select Normal, right-click and choose Modify... Remove the underlining from the font formatting (you can probably do this by pressing the U button twice; if not, use Format | Font), and select the radio button for "New documents based on this template."  When you quit Word, if you are asked whether you want the Normal template to be saved, say yes.
  3. If the two lines appear just below each paragraph rather than below each word, then they are likely a border; see http://word.mvps.org/FAQs/Formatting/CantGetRidOfLine.htm.
