alexsun_2012 say:

Hi Chris,

Yes, if the affected users have mailboxes on both on-premises Exchange and Exchange Online, it could lead to the issue. When you completed Exchange Hybrid configuration, a user's mailbox locates on your on-premises Exchange server and the user is synced to the cloud as a mail user by Dirsync or Azure AD connector tool. After you run the mailbox move wizard, the mailbox is moved to the cloud and converted to mail user in your on-premises.So the user should have only one mailbox in Exchange Hybrid.

From the error message, the mailbox migration failed because there is an existing cloud mailbox already for the user. So please go to Office 365 Exchange admin center https://outlook.office365.com/ecp to check if the affected users are mail users. You can also run the Get-User cmdlet after you connect your PowerShell to Exchange Online.

If the affected users are not mail users, it is considered as the root cause. In this situation, it is suggested you delete the users and reprovision the mail users by Dirsync. You can refer to the steps below.

1.Filter the affected users in Dirsync. You can move the user accounts to a filtered OU so that they won't be synced to Office 365. https://blogs.technet.microsoft.com/praveenkumar/2014/04/11/how-to-do-ou-based-filtering-in-office-365/
2.The filtered users will be moved to Office 365 Recyclebin. You can delete the user accounts including the cloud mailboxes from Recyclebin using the Remove-MsolUser -RemoveFromRecycleBin -Force.
3.Move the users back to the previous OU and force Dirsync to sync the users back. The user will be synced back to Office 365 as mail users.

After these steps, please re-run the migration wizard again. If you run into any problems, please let us known. If there is any error message, please provide it here.

bob jones aka: cybertaz say:

Simply for clarification, AutoCorrect & Check Spelling as you Type are 2 separate features.

The former actually makes other changes in addition to automatically correcting certain - but not all - spelling errors. The spelling corrections can be linked to the main Spelling & Grammar Check dictionaries based on a Preference setting. Also, it works even if the latter is turned off.

The latter uses the default & user-defined dictionaries to proof what is typed but does not change any text in the document. It merely marks what it doesn't recognize as being correctly spelled. It also works even when AutoCorrect is disabled.

john korchok say:

For the red wavy lines:

  • With your cursor blinking in a document that doesn't display red wavy lines, click on Tools>Language... Is a language set? If not, choose a language and click on the Default button to set that language as the default for all new documents.
    If a language is set, make sure that Do not check spelling and grammar is unchecked.
  • Open Tools>AutoCorrect... and make sure that Automatically correct spelling and formatting as you type is checked.

  • Still in Tools>AutoCorrect..., ensure that Capitalize first letter of sentences is checked.

lynda_m say:

vlookup

I am having a serious brain dead period at the moment. I used to use vlookup at work on a regular basis but for some reason I can't get it to work for me at the moment.

I have 17 spread sheets 16 of which hold the data I want transferred to sheet 17.

On spreadsheet 17 I have 17 columns. It  needs to find the persons name on both spreadsheets and then pick up the number of points they scored and put them  into the total sheet in the appropriate column.

Thank you in advance.

jllatham say:

Let's go over the basics of a VLOOKUP() and maybe you'll see the error of your ways.  In any examples below I'll assume we are setting up a formula for row 2 on the 17th sheet.

VLOOKUP(what to find, where to find it, relative column data to return, ordered list)

#1 - what to find: that's going to be a reference to the person's name - I'm guessing that it is in column A on your 17th sheet.  While the row will probably need to change, the column should not, so you probably want an address like $A2 for that parameter.

#2 - where to find it: this can be a single column, or a table of several columns.  The first column referenced in it must contain the same type of data you use for the 1st parameter.  The address must include the starting column and at least enough columns to include the information you want to return.  Assuming that on sheets 1-16 the names are in column C and the points are in column G and goes from row 2 to row 1044, you could reference it this way (change sheet name as needed)

Sheet1!$C$2:$G$1044

In versions of Excel after 2003 you can cheat and leave off the row numbers at the expense of some efficiency (speed), as
Sheet1!$C:$G

#3 - this is a relative column number and in a table that goes from C to G, G is the 5th column, so you would have a 5 for this value.

#4 - ordered list.  That's not what Help calls it, but it might as well be,  By setting this to FALSE, you look for an exact match regardless of how the table is sorted, or not sorted.

So for row 2 to retrieve data from Sheet1 we might have this formula:

=VLOOKUP($A2,Sheet1!$C:$G,5,False)

that will fill nicely into the following columns on the 17th sheet, although you'll have to change the sheet name in each column.  Once you get all of row 2 set up, you can select them all and fill down the sheet to the end of your list of names.

Now, for those sheets where a name doesn't exist and where you want to hide the #N/A! alert, you could write it this way (but I'd wait until I got it working to make the change)

=IFERROR(VLOOKUP($A2,Sheet1!$C:$G,5,False),"")

or if it might be used in Excel 2003 or earlier:

=IF(ISNA(VLOOKUP($A2,Sheet1!$C:$G,5,False)),"",VLOOKUP($A2,Sheet1!$C:$G,5,False))
