shanealexander say:

Although this is an old post, and refers to Windows Insider / pre RTM versions of W10, it may help others that google Unisurf tablets and want to upgrade to W10 Home from W8.1 with Bing.

 (as I have a Unisurf tablet, I found this post).

Correct that Unisurf (or other) tablets with only 16gb HDD's and a large recovery partition will find it difficult to A) run normally, B) upgrade to W10, due to a lack of disk space.

I found my W8.1 Unisurf tablet also almost impossible to use unless I freed up space.

Definitely advised to create a Recovery USB stick, the process below will do this and potentially even remove the hidden recovery partition   http://www.intowindows.com/how-to-delete-recovery-partition-in-windows-8

Below can be completed on a W8 system to clear up space (pre W10 upgrade, or even just for normal use).

- Stop the Background Intelligent Transfer Service
- Stop the Windows Update service
- Delete folder C:\Windows\SoftwareDistribution
(above deletes previously downloaded Windows Update patches)

- Open an admin command prompt
- Run  cleanmgr /sageset:12
- Select everything
- Then at same command prompt, run  cleanmgr /sagerun:12
(above deletes as much as Windows can)

Then to upgrade to W10 Home (from W8.1 with Bing), by downloading and putting W10 on a USB stick, and run setup.exe from the USB when in W8.1.

 (not covered here, google it  :-)   )

Note though ... IMO it is best to completely wipe W8.1 and install W10 from scratch, with a "custom" install booting direct to the W10 USB key created earlier, where you delete all partitions on the drive.

For Unisurf tablets though, you will need to copy off all the drivers first from your W8.1 system, e.g. copy the whole C:\Windows\System32\DriverStore\FileRepository  folder to a USB key.

This is because W10 doesn't have all drivers for the Unisurf tablets.

And then once W10 is built, upgrade the drivers on all devices, from Device Manager, and point to the "FileRepository" copied off earlier.

The unisurf upgraded successfully. What happened was:

  • Then a download screen popped up indicating that there was insufficient space on drive c: ... and a drop list for drives...

  • There seemed to be a successful download of some 2.7GB ... but ... battery was almost flat ..
  • The unisurf has a single, micro USB port that is used for USB or charging; not both simultaneously.
  • Installation was halted while device was put on charge.
  • Install did not resume after charge
  • The USB stick had zero content - even though the download progress had indicated 100% completion. That may have been a roll-back or that the drive was formatted FAT32. Don't know.

Do it again:

  • Leave the charging lead plugged into the micro USB port.
  • When the Select Drive for Download screen appeared: I selected the microSDHC card - a 16GB card formatted to NTFS that had 10GB free space
  • Windows Update completed the download to the micro SDHC  card, then continued with the need to Restart screen... etc ... etc...

The Upgrade to Windows 10 went almost perfectly.

  • The Recovery partition had been deleted
  • Windows 10 then had a 13.8GB partition to install in.
  • All previously installed apps and desktop applications, and user data and settings were transferred
  • Bluetooth keyboard and mouse settings and bindings were brought into Windows 10 from Win 8.1
  • Using Recovery > Create USB drive works... takes a long time, but a valid Windows 10 USB drive image was created... looks good; but not tested

Problems:

  • Touch Screen not working - the Bluetooth keyboard and mouse were working. Uninstalled the Touch driver, restart and the driver was re-installed correctly and Touch works again. This would have been a nuisance if the Bluetooth devices had not been bound in Windows 8.1 and carried into Win 10
  • microSDHC Card is not recognised for OneDrive location... this is a major pain with Windows 10 and is the subject of a forum thread. Workaround is to the mount the card as a Volume.

Summary:

  • On Windows 8.1 there was no way to delete the Recovery Partition and reclaim much needed disk space.
  • Manually deleting the Windows 8.1 Recovery Partition causes a boot error - apparently there is a partition check somewhere in the boot routine
  • Windows 8.1 had bloated so much that trying to keep just 1GB of free space had become a challenge
  • Windows 10, so far, seems to manage Windows Updates more efficiently and OS growth is not noticeable. Fluctuating between 3.47GB up to 3.90GB free space - much more usable than Windows 8.1 had become.
  • The unisurf is much more usable on Windows 10 than it was on Windows 8.1

  • Reserve your upgrade then monitor Windows Update. DO NOT ATTEMPT A MANUAL INSTALL UNLESS YOU ARE EXPERIENCED. I did a manual install of an early Win 10 Preview and it took an external, powered USB hub, USB drive / DVD ISO, USB keyboard and mouse plus a lot of driver re-installation from a backed up DriverStore
  • Create a Windows 8.1 Recovery USB drive BEFORE doing the Upgrade to Windows 10. The OEM image will be lost.
  • When prompted for a drive to download to - select the mciroSDHC Card - it is used for download and temporary working space - suggest a 16GB card... it may be OK on 8GB... but why bother?
  • Leave the device plugged into the charger - it may take 5 hours to complete - the battery is only good for 4 hours
  • Have a USB keyboard (and mouse) with USB hub handy, just in case the Touch screen driver fails to install correctly.

don cuthbert say:

The Keyboard language screen is the same as what you get when booting from a Recoverydrive USB.  You don't have one of those in a USB port, do you?  Also, your UEFI boot settings need to be set to boot from the correct location, if there are other options.

Maybe your BCD is not setup properly.  You need to have Windows 10 as the default OS in the boot menu during an upgrade to a new build or the reboots will get confused.  Do you have a dual or multiple boot setup?
