ken sheridan say:

.................both allowed type-ahead feature based on previous records in a field. 

It would be easy to do that with a combo box; you'd simply return distinct values of the column to which it is bound as the RowSource property, ordered by the column, and requery the combo box in the control's AfterUpdate event procedure.  However, to do so would indicate a badly designed table as it would mean that multiple instances of the same value would be included in the column without it referencing a related table in which there are single instances of each value as a key.

Where there is occasion to repeat a value in a column the column should in most cases be a foreign key referencing the primary key of a referenced table, and a combo box should be used as the control for the foreign key column in the referencing table. An example would be a Product column in an OrderDetails table, which would be foreign key referencing the primary key of a Products table.  An example of an exception would be a LastName column in an Employees table or similar.  Names can be legitimately be duplicated of course, but it would not normally be thought necessary for the LastName column to reference the key of a LastNames table.  I have known it done, but I see no cogent reason for doing so.

Where a combo box is used in situations such as the Product example I gave above, the AutoExpand property allows what you have termed 'type-ahead', and the NotInList event allows a new row to be inserted into the referenced table where the value is not currently represented in the combo box's list.

A foreign key might not contain the text value of course, but a numeric foreign key referencing a 'surrogate' numeric key, usually an autonumber, of the referenced table.  Rather than the product name it would be a numeric ProductID.  In situations where the text values are not distinct this is necessary.  A city name for instance would be an unsuitable key as city names can legitimately be duplicated.  A distinct CityID must be used to distinguish between two or more cities of the same name.  In this scenario the AutoExpand property of a combo box can still be used, but it would go to the first instance of the city name, so it might be necessary to then select another instance below it if the first is not the city of that name which is wanted (other columns in the combo box's list would allow the correct one to be identified).  The NotInList event procedure of the control can only be used to add the first city of each name, however.  Other means of adding further cities of the same name would be required.

mattg_1969 say:

VLookUp and duplicated values

Hi,

 

I have a worksheet called Projects.  In Column A is the due date and in Column D is the Project Name.  These exist in rows 2 through 500 (but this could expand).  I have another sheet called Daily Schedule.  On it I want to look up any project project with a due date of today and return the Project Name to Column A so the user will have an instant view of all projects due today. There will be multiple projects with the same due date. 

 

Here is my formula:

 

=VLOOKUP(A$1,Projects!A:D,4,FALSE) ****CELL A1 has today's date****

 

The problem with this is that it only returns the first value from Column D in the Projects sheet repeatedly even when I copy the formula down 20 rows.

 

I also tried this:

 

=VLOOKUP(A$1,Projects!$A2:$D500,4,FALSE)

 

Copying this down I did not have dups but ended up with a lot of #N/A vaules where there was no match

 

What I need to do is have the formula reference all of Column D on the Projects sheet and return all unique values that have the current day's date to the Daily Schedule sheet Column A.

 

Any help is greatly appreciated.

mike h.. say:

Hi,

 

Put this ARRAY formula in a cell; see below on how to enter it, and when you have array entered it drag down as far as required. It will start returning blanks when it runs out of matches for a1.

 

 

=IFERROR(INDEX(Projects!$D$1:$D$2000,SMALL(IF(Projects!$A$1:$A$2000=A$1,ROW($A$1:$A$2000)-ROW($A$1)+1),ROWS(A$1:A1))),"")

 

 

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.
