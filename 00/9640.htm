Does anyone effectively solved this problem?

tom van stiphout say:
You cannot mix data macros and VBA. That is by design.

albert d. kallal say:

 

You most certainly can do this.

 

Assuming for this discussion the database is not split, then any VBA public function can be called from a data macro like this:

 

SetLocalVar (MacroDummyVar, MyUpdate())

 

 

So in above, the data macro will call a VBA function called MyUPdate(). Note that you can also pass values to the function. And if the function does need to return a value, then in above MacroDummyVar will take the VBA value from the function.

 

So even table triggers etc. can thus call VBA code. That VBA code can in turn run SQL updates, or quite much do whatever you want.
