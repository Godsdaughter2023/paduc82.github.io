If the cursor is on the first page when you insert the watermark, then the watermark will appear only on that first page (because the watermark is "anchored" in the first-page header).

graham mayor say:
You can do this with a simple macro http://www.gmayor.com/installing_macro.htm  

The following will remove the formatting from any paragraph that is completely formatted as Italic and apply the named style (shown in bold - which you should change to the style you wish to apply). It does not affect paragraphs that are part formatted in Italic.

Sub ItalicParas()
Dim oPara As Paragraph
For Each oPara In ActiveDocument.Paragraphs
    If oPara.Range.Font.Italic = True Then
        With oPara.Range
            .Font.Reset
            .ParagraphFormat.Reset
            .Style = "MyItalicStyle"
        End With
    End If
Next oPara
End Sub

If you want to limit the search to only paragraphs formatted with Normal style then add the condition below

Sub ItalicParas()
Dim oPara As Paragraph
    For Each oPara In ActiveDocument.Paragraphs
        If oPara.Range.Font.Italic = True Then
            If oPara.Style = "Normal" Then
                With oPara.Range
                    .Font.Reset
                    .ParagraphFormat.Reset
                    .Style = "MyItalicStyle"
                End With
            End If
        End If
    Next oPara
End Sub

andreas killer say:
Well, the problem with the page breaks is that they are related to the current used printer and its settings.

If I have a different paper size than you, then a fixed page break does not work properly. And even if we have the same paper size, my printer driver could be set to have a larger margin.

To only way out of this dilemma is to use this settings in the sheet

  With ActiveSheet.PageSetup
    .FitToPagesWide = 1
    .FitToPagesTall = 3
  End With

so Excel has a chance to adjust the output. A more safer way is to generate sheets for only on page and print them with .FitToPagesTall = 1

kevinfletcherd say: Check the SATA mode in the BIOS, you might also check you boot mode.  

jerkson say:
Check the SATA mode in the BIOS, you might also check you boot mode.  

Why are you telling him to check his SATA mode (IDE/AHCI) if he's running an 80Gb IDE drive. Try reading the full post before replying since the poster was kind enough to include a lot of details about the system.

That said, I will second Kevin's comment on the boot mode. Some BIOSes call this something different like "Legacy Boot (on/off)" Legacy Boot OFF = UEFI boot mode. You want UEFI if your PC supports it. I had issues installing with UEFI turned off on my test system (was running win7 previously) and once I set legacy boot mode to OFF I was in business.

I had this problem when trying to install 10. I believe it is because Pentium 4 processors are no longer supported by Windows ~8.1 and later.

Unlike Windows Vista and 7, Windows 8 (as from build 8400) is not compatible with all models of the Pentium 4 processor. 

Pentium 4 are compatible with SSE2 and EAP: the problem may come from the NX bit.
