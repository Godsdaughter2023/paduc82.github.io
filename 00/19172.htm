rangao_msft4227 say:

Hi Stephen,

Your assumption is right. You just need to update the first and third records in the new DNS hosting provider.

When we change the DNS hosting provider from A to B, we need to change the Name Server records of the domain to point to B. Therefore, there will be two main scenarios regarding the downtime of the email service:

Scenario 1: If the new DNS hosting provider B allows us to create DNS records for Office 365 Exchange Online before we point the Name Server records to it, we can firstly create all the necessary DNS records at the new DNS hosting provider, and then change the Name Server records to point to it. In this scenario, the downtime of the email service depends on how long it takes for the Name Server records and the DNS records for Exchange Online to take effect. Ideally, the downtime of the email service might only last from a few seconds to several minutes.

Scenario 2: If the new DNS hosting provider B doesn’t allow us to create DNS records for Office 365 Exchange Online before we point the Name Server records to it, we need to create the necessary DNS records after changing the Name Server records. Then the downtime will last for a while until we complete the DNS records set-up process. Therefore, the downtime depends on how long it takes for the Name Server records to take effect, how long it takes us to complete the DNS records set-up process, and how long it takes for the newly created DNS records for Exchange Online to take effect. Typically, it will take more time in this scenario.
