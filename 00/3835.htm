amit tandon say:

Your query does not seem to be clear ... it seems you want to extract all values in the range B63:L274 ie values in all rows where A63:A274 = C4, and then calculate their average.

In case you mean this, then use below formula for Average (excluding Blank Cells):

=SUMPRODUCT((C4=Sheet1!A63:A274)*(Sheet1!B63:L274))/SUMPRODUCT((C4=Sheet1!A63:A274)*(Sheet1!B63:L274<>""))

Though you may use below array formula (Ctrl+Shift+Enter) also but there may be a variance on account of blank cells:

=AVERAGE(IF(Sheet1!A63:A274=C4,Sheet1!B63:L274))

And in this case you may update the AVERAGE formula - array formula (Ctrl+Shift+Enter) - to below to exclude blank cells:

=AVERAGE(IF(Sheet1!A63:A274=C4,IF(Sheet1!B63:L274<>"",Sheet1!B63:L274)))

tecknode say:

"Outlook view" vs Outlook "Exchange view"

OK, can someone point to anything that COMPARES Outlook view and Exchange view in Office Outlook?

 

I'm looking for specifics, NOT how-to.

 

NOTE: The "Office Version" list below does not have "All versions" but I want an answer that can apply to ANY version that allows Exchange view

 

roady [mvp] say:
If I understand what you're saying, If you run Office Outlook in the Exchange view you are looking at the eMails ON the Exchange server? What about all the other things like Contacts, Notes, etc?

That implies that Office Outlook using the Outlook view you are looking at eMails (after downloaded) on your local desktop?


To clarify what I'm asking, Office Outlook:

If you are NOT using an Exchange Server (just a normal POP) EVERYTHING is on your desktop system, including your downloaded eMails. They're in PST files.

Now, if you ARE using an Exchange Server, where is all the Outlook files kept if you use the Exchange view? Are there still PST files on your local desktop?


No, that is not what I'm saying. Views are how you are displaying your list of emails, contacts, calendar, etc...; how it is sorted, how it is filtered, which columns are displaying, etc... You can customize a default view but you can also define your own.

Unlike current version of Exchange and Outlook, various view settings could not or were not stored within the mailbox or Public Folder itself. This means that shared folders looked different for the person you shared it with. For instance, it showed in the default view.

The option "Automatically generate Microsoft Exchange views" created a version of your custom view and stored that within the mailbox or Public Folder so that it was also available to the person you shared that folder with or when you accessed your mailbox on a different client.

Modern versions of Outlook no longer store their view settings in a separate views.dat file but within the mailbox (either an Exchange mailbox or a pst-file) itself. When using Exchange, these view are thus also available to the persons your share the folder with or when accessing your mailbox on a different client. As views are now stored within the mailbox and Exchange is capable of storing complex Outlook views, the option to generate separate Exchange view is now no longer needed and also no longers exists in modern versions of Outlook.

 

As for your other question;
Yes, when using a POP3 account, everything is stored in a pst-file. You have the option to create additional pst-files. For instance to create an archive or to store mails, contacts, notes, etc... for each account in a separate pst-file

When you are using an Exchange account, then everything is stored in the database of the Exchange server. You have the option to configure Outlook for Offline usage (in modern Outlook versions this is called Cached Exchange Mode) so that you can continue to work with your data when you are not connected to the Exchange server. In that case, your data is kept in an ost-file. When connected again, Outlook will upload the changes that you made while offline to the Exchange server and download any new items from the Exchange server.

For an overview of files and locations which Outlook uses see;
http://www.howto-outlook.com/howto/filesandsettings.htm

It would seem that someone accidentally used Ctrl+A, Ctrl+F9 and wrapped the entire document in a set of field braces. Consider this simpler example (with Alt+F9 i.e. field codes displayed).

{ "I want to keep this { page } number." } 

If you simple place the insertion point (i.e. cursor) anywhere inside the field and tap Ctrl+Shift+F9, then everything will be stripped since Ctrl+Shift+F9 strips field codes to their resulting text and there is none. "I want to keep this { page } number."
