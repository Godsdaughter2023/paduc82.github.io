2 - Implementation

To avoid an array formula validation (Ctrl+Shift+Enter) the solution involves the AGGREGATE array function (Excel Windows >/= 2010, Excel Mac >/= 2011). The same thing can be achieved with previous versions of Excel with a more complex array formula (not covered here).

NB: Although formulas are validated with Enter only they still involve array calculations. For performance reasons it is highly recommended to adjust the data set range to the minimum required size. In other words one should avoid referencing a full column (ie. A:A) or a full row (ie. 3:3) if not required

AGGREGATE function              That array function is extremely powerful for a number of things…

Corresponding workbook with same explanations and formulas available here

2.1 - Challenge

The possible duplicates and empty cell is one thing. The other one is we don't know if we will find - in that example - 5 unique Top/Bottom values to report. Of course we can use the IFERROR function to return "" (or something else) when there's no more value to report (so less than 5).

It's personal but I don't like this approach as it might hide data quality issues. For that reason IFERROR isn't used here. Instead, we calculate in a "helper column" (# Uniq.) - that can be hidden and/or moved somewhere else in the sheet - the number of unique values per range. And the main formula will first check if that max number of unique values has been reached. The benefit of that approach is obvious: if the max value has been reached the key part of the formula doesn't need to be computed
