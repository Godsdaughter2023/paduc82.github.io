Hello itm2

You can get to get to the settings you're looking for vi the following steps

#1 - Press CTRL+ALT+S which brings up the Send/Receive settings window
#2 - Select the Group you want to change (most people only have one) --> Click <Edit>
#3 - Select the email account --> click on <Account Properties

Alternatively, with Outlook closed, open the Control Panel <Mail> app

You definitely do NOT want to remove the IMAP account since that will result in losing all data contained in any folders (contact/calendar) that have the words "This Computer Only" in the folder description.

As a side note, if you do have contact/calendar folders with "This Computer Only" in the folder description, then would strongly suggest the following to avoid the potential loss of data

a) create a new PST file in the profile
(from the Outlook ribbon --> click on <New Items> --> <More Items> --> Outlook data file
b) go to File --> Account Settings --> Data Files tab --> select the newly created PST file --> set it as the default data file for the profile
*** the "default data file" is where the default contact/calendar folders are stored
c) copy all the contact/calendar data from the "This Computer Only" folders to the corresponding folders in the newly created PST file

Why do you want to do the above?

An IMAP folder has an .ost file associated with it and if it is set as the default data file, contains the default contact/calendar folders marked as "This Computer Only". If you ever remove the IMAP account, the OST file also gets deleted which means that data in the "This Computer Only" folder will be lost. Similarly,, if Outlook can't find the OST file, it will create a new one, sync thhe email folders with the server but the contact/calendar folders will be empty (if there is no other data file set as the default data file for the profile.

#1 - An .ost file is nothing more then a "mirror" of the data that resides on the server. If Outlook can't find it when it opens , it recreates a new one and re-syncs with the account so if you are encountering an issue right away after this - it's either something coming down from the server OR something on your system - see next point

#2 - If you have the Norton A/V Outlook addin installed - strongly recommend dispensing with it permanently as long as you have the Norton system level product running. Outlook A/V addins serve no useful purpose with an active up-to-date system level A/V installed "in general" cause more aggravation/problems then anything claimed to resolve with the Norton add-in frequently coming up in conversations (posts in this forum). If you didn't explicitly decide "NOT" to install the addin, likely that it's there. Personally don't use Norton anymore so have no idea about the install process is for any of their products.

Go to File --> Options --> Addins
