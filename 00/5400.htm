a. user say:

Whether or not "Contains" appears for an Excel data source also depends on how Word makes the connection to the Excel file. 

By default, Word will connect using a thing called OLE DB. In that case you should see the "Contains".

However, in some circumstances (don't ask me to remember what they are, although if for example you have applied a password to you Excel file, Word can only use the DDE connection method) Word may connect using the older ODBC, and in that case you do not see the "Contains". As a last resort, it might connect using DDE, and in that case you do not see the "Contains" either.

However, I think you would normally know about it if either of those two things happened. A key difference is that you would typically see different dialog boxes - for ODBC, you typically see something with an Options... button and a dropdown list of data source names. Usually you have to click the Options button and check the boxes in the resulting dialog in order to see any tables you can select from. For DDE, you get a dialog box which lets you enter a "Named or Cell Range" that defaults to "Entire Spreadsheet"

If you think that might be happening, I suggest you go into Word->File->Options->Advanced->General (near the bottom of "Advanced" and check "Confirm file format conversion on open". Then, when you try to attach the data source, you get a choice (if it's a .xls, three choices are presented to you. It it's a .xlsx, you have to check Show All to see anything other than the OLE DB option.

Microsoft has a bewildering number of licensing and marketing schemes for Office in particular, so without knowing a lot of details, here's a down-n-dirty guide:

If you purchased it from a school bookstore, directly from Microsoft http://www.microsoft.com/student/office/en-us/default.aspx, or other retail or online seller at the average academic selling price, then almost certainly the answer is yes.
