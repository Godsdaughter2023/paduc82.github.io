sho nut say:

Thanks for taking the time to respond.

I suspected as much with Personal vs. Business, but it didn't hurt to ask.

Based on your response, I tried the individual file version history, (glad to see it's now implemented for all file types now and not just Office documents) but it's blank.  In my case, each file was encrypted and renamed <filename>.<extension>.david, so I checked for version history the file directly, and then renamed it back to <filename>.<extension>, but the version history was still blank.  I suspect this is because OneDrive probably sees it as a totally new file and not a modified version of the original, and thus there is no history.

As above, thanks for your time.

For anyone else that runs into an issue with their OneDrive being encrypted, this is what did help (naturally these are very time-sensitive with a ransomware attack, and are specific to the David Ransomware with the .david extension):

Find a machine other than the affected one that is still syncing the same OneDrive and stop the sync.  On that machine, there may still be files that may be renamed but not yet be encrypted, or possibly not yet even renamed.  It appears that the process is rename, then encryption, writing of the encrypted file, then deletion of the original renamed file.  For files synced locally (I don't think you can change the extensions online), these renamed files can be renamed back (simply remove the .david) - and they are good to go.  It's possible to do a mass rename in a folder by typing CMD in the address bar in Windows Explorer and typing [ren *.david *.] in the Command Prompt window that pops up (remove the square brackets).

Then check the recycle bin.  Files in the recycle bin with the .david extension were generally not encrypted, just renamed.  After moving the encrypted originals to another location, restore these files from the recycle bin and remove the .david extension as above.

Try an undelete utility such as Recuva to recover anything deleted and try the steps above.

Good luck, and remember to always have your OneDrive Personal backed up on a machine that would never be directly exposed to ransomware, with an incremental backup on that machine.  Having it backed up on the machine possibly exposed to ransomware won't help, that backup drive will get encrypted too if it's attached at the time - don't ask me how I know this.
