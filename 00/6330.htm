In all versions of Excel both for Mac and Windows including Excel 2011 for Mac, you can use Edit Query to launch Microsoft Query when a Table that is a query table (not a pivottable) is active. Using Microsoft Query you can edit the query and return a new result set to the query table.

This means that you have a fairly easy job of making your own button that launches Microsoft Query for a PivotTable using a VBA macro in Excel 2011.

To set things up, you would make a query table on a workhsheet (Data > Get External Data > New Database Query). Then you would use the PivotTable wizard to use the resulting query table as the source of the PivotTable.

You can record a Macro that selects the query table, and then more VBA code to using the Execute command to programmatically click the Edit Query button (hint: use the Edit Query CommandID to make a temporary button on a toolbar) to launch Microsoft Query. When it's working the way you want, you can assign the macro to a toolbar button, menu and/or contextual menu, or when you click on an object or perform an action that you specify.

As far as I can tell, Excel has never offered the Edit Query command for PivotTables in any version of Excel, either Mac or Windows. So this is not a bug. You can use Help > Send Feedback to Microsoft and suggest it as a new feature for a future release of Excel. But given the relative simplicity of making your own macro and assigning it to a Toolbar Button, menu, or contextual menu you don't have to sit around and wait. You can enable this functionality on your own using VBA. VBA is in Excel precisely for the purpose of adding custom functionality such as this.

The Workbook_Open subroutine is shown below.

Private Sub Workbook_Open()

   Worksheets("Sheet1").Activate

   LastRow = Cells(Rows.Count, "A").End(xlUp).Row
   For k = 1 To LastRow
    mymonth = DatePart("m", Cells(k, "B"))
    thismonth = DatePart("m", Date)
    myday = DatePart("d", Cells(k, "B"))
    thisday = DatePart("d", Date)
    If mymonth = thismonth And myday = thisday Then
       mymessage = "Happy birthday, " & Cells(k, "A")
    End If
   Next k
   If Len(mymessage) <> 0 Then
    MsgBox mymessage
   Else
    myrand = Int((LastRow - 1 + 1) * Rnd + 1)
    mymessage = Cells(myrand, "C")
   End If
   MsgBox mymessage
End Sub

suzanne s. barnhill say:

Pleading paper, as provided by stationers, has a rule (line) down one side. You can replicate this in Word, but most printers won't print it clear to the edge, so you're already starting at a deficit.

If you're just referring to the line numbers in the margin to the left of this line, you can do that, too, by anchoring a text box to the header. The Pleading template provided in Word 2003 will still work in Word 2010, but you can replicate it by adding a similar text box. The line numbers aren't "real" line numbers (you get those with the Line Numbers command in the Page Setup dialog), but they will match the lines of your pleading provided you use the same spacing for the numbers in the text box that you use for the paragraphs in the pleading.

If you want to have a specific number of lines on each page, you will have to disable "Widow/orphan control" to prevent Word from moving a two- or three-line paragraph to the next page if it won't all fit.

Again, you have posted this as a discussion. If you want specific instructions for creating a pleading template, you will need to post a question.

vaseem ahmed say:

Hi GUZZLR,

Internet Explorer imposes a time-out limit for the server to return data. By default, the time-out limit is as follows:

Internet Explorer 4.0 and Internet Explorer 4.01 5 minutes
Internet Explorer 5.x and Internet Explorer 6.x 60 minutes
Internet Explorer 7 and Internet Explorer 8 60 minutes
 
Internet Explorer does not wait endlessly for the server to come back with data when the server has a problem.
If a page does not return within a few minutes, design your server processes to return data within 5 minutes so that you don’t have to wait for a long time.

You can usually break down long processes into smaller pieces. Or, the server can return status data to update users about the process. In addition, you can create a long server process with a messages-based or asynchronous approach so that it returns immediately to the client after the job is submitted and notifies users when the long process is finished.

Important: This article explain you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. For more information about how to back up and restore the registry, click the following article number to view the article in the Microsoft Knowledge Base:
http://support.microsoft.com/default.aspx/kb/322756

Although Microsoft does not encourage this approach, you can change the default time-out setting for Internet Explorer. To do this in Internet Explorer 4.01 Service Pack 1 or a later version of Internet Explorer, follow these steps:

1. Add a ReceiveTimeout DWORD entry with a value of (<number of seconds>)*1000 in the following registry subkey:
HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings
For example, if you want the time-out duration to be 8 minutes, set the value of the ReceiveTimeout entry to 480000 (<480>*1000).
2. Restart the computer.

Note : This setting does not work on versions of Internet Explorer that are earlier than Internet Explorer 4.0 SP1.

For more information check out the KB article given below:
http://support.microsoft.com/kb/181050
