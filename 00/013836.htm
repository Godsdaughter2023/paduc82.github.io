reinob say:

Just to complete on what was said above.

 

The Z: drive is a (EEP-)ROM-drive containing your firmware. You can have a look at what's in there, using some file manager, but it's read-only.

 

The D: is your RAM. Apparently part of it is used (or shown) as a RAM-disk.

 

The C: drive is the internal memory.

 

The E: drive is your memory card.

 

If you install an application on the memory card, its files will most likely go somewhere in the E:/system folder. Same for the C: drive (C:/system).

 

There you have a number of subfolders, such as Apps, Data and DMGR.

 

Application-related files are "spread" amongst those folders, which may not be what you are used to in e.g. Windows, but actually makes sense (think of Unix/Linux with /usr/bin, /usr/lib, etc.)

 

Please note that even if you use a third-party file manager (like Y-Browser), there are files and folders that you simply cannot even see, as access to those folders requires capabilities that are not granted to normal applications.

 

I know this is sad for a "neat-freak" (which I also am), but that's how it works, and there are good reasons why it works that way.

macropod say:

Your field code can be reduced to:

{={MERGEFIELD Quote_Product_Price_Per_Unit}*{MERGEFIELD Quote_Product_Quantity}-{MERGEFIELD Quote_Product_Manual_Discount}}

or:

{=«Quote_Product_Price_Per_Unit»*«Quote_Product_Quantity»-«Quote_Product_Manual_Discount»}

where the field brace pairs (i.e. '{ }') are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac) and the chevrons (i.e. '« »') are part of the actual mergefields inserted from the 'Insert Merge Field' dropdown.

Do note, too, that merge previews are not a reliable indicator of the mailmerge output, so you should merge to a new document for testing purposes at least. You should also be aware that your mailmerge output will still contain the underlying formulae. You can eliminate those by wrapping the above fields in a QUOTE field, thus:

{QUOTE{={MERGEFIELD Quote_Product_Price_Per_Unit}*{MERGEFIELD Quote_Product_Quantity}-{MERGEFIELD Quote_Product_Manual_Discount}}}

or:

{QUOTE{=«Quote_Product_Price_Per_Unit»*«Quote_Product_Quantity»-«Quote_Product_Manual_Discount»}}
