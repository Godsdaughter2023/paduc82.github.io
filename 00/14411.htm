anthony o'b say:

This appears to be a type mismatch issue between Outlook and SharePoint/Project.

Outlook tasks do not have a time component (ie you can only specify whole days).  If you have SharePoint Due Dates set as DateTime type, syncing adds a time to your due date (eg 17:00).  When this syncs back to Outlook it seems to be rounded up and the date is seen as +1 day.  If SharePoint Due Dates are set to Date type you don't get the issue, but opening your tasks in Project does the same thing (sets a time of eg 17:00), which flows back through to Outlook.

So changing the field type in SharePoint may help, but I have not solved the Project side yet.

As one helpful MS VP commented elsewhere, Tasks in Outlook "have no time" and went on to suggest a workaround was to use appointments instead.  Hoping we can find a more constructive outcome :-)

My Problem:  The form can be filled,  and a signer can sign, but when the signer tries to save the form with their signature so they can forward a signed version of the form to the next signer, they get a message that the form owner has restricted editing which prompts a pop-up box asking if they want to edit anyway.  Essentially, the only option is to "edit anyway" and that's when the signature is voided and I can't accomplish what I wanted - basically a fillable form that could be signed and routed to the next approver for signature.

Is there a way to restrict editing on certain parts of fillable forms and allow edits (in this case signatures) at the bottom of the form?

Thank you for sharing any options/tips!

 

jay freedman say:

Word does not properly support multiple electronic signatures in a single document. For example, read the thread at https://social.technet.microsoft.com/Forums/office/microsoft-word-and-electronic-signatures-multiple-signatures-in-a-single-document. The reason for this is that Word uses the entire document file to create a cryptographic "signature", and editing the document in any way invalidates that signature because the file no longer matches it.

If you just want the signers to type their names without the cryptographic feature, you could use legacy form fields and the Filling In Forms protection by inserting a continuous section break before the signatures. Then in the Restrict Editing pane, click the Select sections... link and uncheck the box for the final section.

Alternatively, if everyone who will edit the document has Word 2007 or later, do not use "legacy" form fields and do not use the Restrict Editing pane. Instead, use only content controls. To provide the equivalent protection so that edits can be made only in the content controls, select the some or all of the document and click Group > Group in the Developer ribbon. If you want a portion of the document to be unprotected for the use of the Signatures feature, don't include that portion in the grouping.
