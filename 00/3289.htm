ken sheridan say: The DLookup function can only return a value from a single row, so you would have to extend its criteria argument to include another column which, in conjunction with the SectionID column would constitute a candidate key of the  1aFunction table.  You would then need a further control to reference as the value for this second column.  This could result in a rather clumsy interface and there is a more efficient solution.

Rather than a text box in the parent form, use a subform, in single form view whose RecordSource is a query which references the cboSection control.  Within this subform include the other subforms, which can be in single or continuous forms view as desired, linking them on the Function columns by setting this as the LinkMasterFields and LinkChildFields properties of the subform controls.  You'd then simply need a means of navigating within the first subform, which can be by Next, Previous, etc command buttons if you don't want to use the built in navigation buttons.

You'll find an example of nested subforms like this in CorrelatedSubs.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file, which uses Northwind data, there are examples for both correlated and nested subforms.  The former correlates two continuous subforms, but the latter will probably suit your requirements better.  In this the first subform includes all orders for the parent form's current customer, showing one order at a time, and navigation can be either via the built in navigation buttons, or by means of an unbound combo box in which an order is selected by date.

The nested subform is in continuous forms view and shows all order detail rows for the selected order.  By virtue of the standard linking mechanism the nested subform is automatically requeried once the first subform is moved to a different order record.
