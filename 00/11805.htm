That "gentleman" would be JP Ronse, I presume.  Readers might want look at the various threads that Ivana has started on related subjects.  Also see the uploaded workbook that she references there.

As I noted in another thread, Excel flags a circular reference only when it encounters it during calculation.  Consequently, circular references in conditional functions like IF and CHOOSE might go unnoticed until the conditional expression causes Excel to evaluate the faulty expression.  For example, put the following formula into A1 in a new workbook:

=IF(FALSE,A1+B1,C1)

Excel does not flag an error.  But change FALSE to TRUE, and Excel flags the circular reference.

I suspect the situation that you encountered -- a maze of circular references so complex that Excel does not have the resources or time to track them down -- arose because:  (a) many of the circular references stem from IF expressions which might not have been evaluated at the time you entered the formulas; and (b) many formulas were not (and still are not!) correctly array-entered as they should have been at the time of initial entry.  #B is a problem because the normal-entered "array" formulas were not fully evaluated as intended until you array-entered them.

So to answer your broad question:  you can trust Excel's circular reference detection feature only to the extent of its limitation.  A circular reference can go undetected due to temporary circumstances.  By the time circumstances change and it is detected, the situation might be too complex.

albert d. kallal say:

Just keep in mind that you don’t really need any VBA code at all.

 

If you use the linked table manager and link using a file DSN, then that connection is DSN less.

 

At that point you can simply distribute the front end to each workstation, and no further setup is required.

 

What the above means is you really don’t need any VBA code at all to create DSN-less connections.

 

So if you JUST link the tables using the linked table manager the result is in fact a DSN-less connection. (Just make sure you choose “file” DSN when you create the linked tables).

 

Access in ALL CASES creates by default DSN-less connections for you if you link using a file DSN.

 

No VBA code is required at all here.

 

Once you link the tables, then Access ignores the actual DSN you created. You could even delete the DSN, and access will continue to just run fine.

 

So just link you tables to the SQL server – at that point you are free to distribute the Access front end to each workstation – no DSN or re-linking will be required on each computer and a DSN is NOT required to be created or setup on each workstation.

3.  It lets you choose the option to use the Free ad-supported edition of Office called Office  2010 Starter Edition.  Starter has only Word 2010 and Excel 2010, and the Starter editions of these programs have some limitations that most home users will never encounter.  If you decide that Starter is not enough Office for you, you can always use the built in links to buy a full version of the Office 2010 edition that's right for you.

dianeoforegon say: First check your "This is Me" Contact.

Open Contacts
Click on "Organize" tab in the Ribbon
Select icon "Me". This will bring up your "This is Me" contact. If this is the wrong contact. Select new one.

Next check info in Word:
Open Word
Select Word>> Preferences>> User information
Edit your User Information and select OK
Log out of Word then reopen to see if the proper changes have been implemented
