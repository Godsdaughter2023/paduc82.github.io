cody c. say:

Denmarfl,

It sounds like you're a bit confused on how IP addresses work. Here's a general explanation:

One of the primary tasks of a router is to hand out IP addresses. The router is configured with an IP address scheme and DHCP range. The scheme is the192.168.2.X part of the IP address. In home networks, the first 3 numbers never change. The last number in the IP address differs per network device. You seem to have a good understanding between Dynamic and Static IP address so I'll skip that. The IP address DHCP range is the range of IP addresses the router is allowed to hand out dynamically. Everything outside that range is a safe IP address to set as static.

For example...

My router's IP address scheme is 192.168.2.X. I've configured my router's IP address DHCP range to be between 50 and 100. So I know anything between 192.168.2.50 and 192.168.2.100 should be considered taken because at anytime the router could hand out one of those IP addresses and cause a conflict. So when I'm configuring the static on my printer, I can choose anything between 1-49 and 101-254 since those are all open IP addresses.

All this to say, you need to log into your router and find out what the IP address range is. Once you know that, you can pick a static IP address for your printer that you know won't cause a conflict later down the road. 192.168.2.0 is (in most cases) not a valid IP address. You need to configure the printer's IP address and then create the New Port as suggested by HP.

Hope this helps explain things!

spiritx say:
Hi,

Well you sort of answered your own question. If the driver or device is not
Windows 10 compatible that is the issue.

Try this :

In Device Manager - Double Click the device - Drivers tab - update driver
(will probably do nothing) - when that completes Right Click on the device
and uninstall it- then Reboot.

If needed try re-installing the driver in compatibility mode :

Download the latest available drivers from the maker's website. Right Click
the setup.exe (installable file  if a ZIP unpack first) - click properties then
the Compatibility tab. Check Run in Compatibility mode - select the OS
same as your driver.

The compatibility center indicates the Samsung HS3000 is Windows 10
compatible.

I can't find any information on the compatibility of the Samsung 350V5C
though it seems to be since the site discusses installing Windows 10.

And check with Samsung Support :

Samsung - Support - click the appropriate link
http://www.samsung.com/us/support/main/supportMain.do

Samsung - Visit Your Country Site
http://www.samsung.com/us/common/visitcountrysite.html

Be sure you have the latest BIOS/Firmware, chipset, and other major drivers.

Hope this helps.

Hi,

Since you have a 32-bit Windows 10, it is advisable that you download apps and drivers that are only 32-bit. You may download the 64-bit apps and drivers, however, it may cause compatibility issues. Converting a 64-bit app or driver to 32-bit is restricted. You will need to download a new one that is compatible to your system, which in this case is 32-bit.
