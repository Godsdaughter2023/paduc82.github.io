Disable printing of an picture with picture content controls - Microsoft Community

computermaniacal say:

Disable printing of an picture with picture content controls (switching the picture)

Hello,

in a letter with an logo I want to disable the printing of that logo sometimes.
The only way I have found so far is using picture content controls. One with the logo and an empty one. What is the simplest way to switch between the two overlaid content controls?

Should be simple for the user (not just "send to background/foreground" via the context menu) - e. g. a button. Any hints are highly appreciated.

Many thanks and greetings
computermaniacal

jay freedman say:

Use just one picture content control, not two. Follow these steps to set it up.

  • In Windows Explorer, single-click on the file that contains the logo and make a note of its exact dimensions in pixels as shown in the status bar.
  • In MS Paint or some other graphics editor, create a picture that's all white, with the same pixel dimensions as the logo. Save it (preferably in the same folder as the logo) with the name "empty.jpg" (or whatever file type you like). The macro below is going to swap the two pictures within the single picture content control, and you don't want the control changing its size when that happens.
  • Open the picture content control's Properties dialog and enter Logo as its title. It would be a good idea to check the box for "Content control cannot be deleted".
  • Open the macro editor.
  • Click View > Immediate Window. In the Immediate window, type the following line and press Enter:

ActiveDocument.Variables("Logo").Value=0

  • Insert a module in the document's project and paste the following code into the code window:
    

Sub ToggleLogo()
    Dim CC As ContentControl
    Dim strLogoName As String
    Dim strEmptyName As String

    strLogoName = "D:\Pictures\cat_peek 2.jpg" ' <== change this to actual path\name
    strEmptyName = "D:\Pictures\empty.jpg"     ' <== change this to actual path\name
    Set CC = ActiveDocument.SelectContentControlsByTitle("Logo")(1)
    If ActiveDocument.Variables("Logo").Value = 0 Then ' currently blank
        If CC.Range.InlineShapes.Count Then CC.Range.InlineShapes(1).Delete
        ActiveDocument.InlineShapes.AddPicture _
            FileName:=strLogoName, Range:=CC.Range
        ActiveDocument.Variables("Logo").Value = 1
    Else  ' currently visible logo
        If CC.Range.InlineShapes.Count Then CC.Range.InlineShapes(1).Delete
        ActiveDocument.InlineShapes.AddPicture _
            FileName:=strEmptyName, Range:=CC.Range
        ActiveDocument.Variables("Logo").Value = 0
    End If
End Sub

  • Change the paths and filenames where indicated to the real locations of the two files.
  • Save the document as a macro-enabled document (*.docm) or template (*.dotm).
  • Create a button on the Quick Access Toolbar to run the ToggleLogo macro. Be sure to set the dropdown at the top of the right-hand column in the Customize dialog so the button is saved in this document or template, not in Normal.dotm.

Each time you click the button, the picture content control will switch from the logo to the empty picture or vice versa.

Excel formula and funtion language - should be language independent - Microsoft Community

a. user say:

Excel formula and funtion language - should be language independent

When working with Excel, cell formulas and functions are typed with local names.

E.g. SQRT is called KVROD in danish.

Any Excel sheet with danish cell formulas and functions will subsequently fail on calculating formulas when Excel language is not Danish, since KVROD is unknown in all other languages :-(

1. Excel should work with unique English formula and function names "behind the scenes" - so only Danish, German, Spanish, Russian, Portuguese etc. are display-names.

2. It should be optional to show either the unique English or the local display names, independent of the users/computers regional settings.

This has been an issue and a pain for extremely many years now. It is impossible to share calculations between Excel users using different languages.

Please fix this, before designing unimportant new icons.....

/Berthelsen
