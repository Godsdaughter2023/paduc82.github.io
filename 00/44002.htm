mr.fusion say:

I think this has to do with the virtual folder layout mediatomb creates by default. One music file is duplicated in "Music / Albums", "Music / Artists", etc. One quick fix is to build your own import script.

change

<virtual-layout type="builtin">

to

<virtual-layout type="js">

and customize your import script (requires that mediatomb ist linked against spidermonkey/libjs), mostly found here:

/usr/share/mediatomb/js/import.js

I've deleted the generation of all virtual folders except one (in my case: "Artist / Album").

If now you see each file twice, you also have to hide the "PC directory":

<pc-directory upnp-hide="yes"/>

see http://mediatomb.cc/pages/documentation and http://mediatomb.cc/pages/scripting

btw: I don't know if it's WMP12 or mediatomb's fault.

mckaysmalley say: I had the same problem in MediaTomb as well. After investigation, I found that MediatTomb builds its own virtual directory by default whenever you add any files to the directories that MediaTomb has access to. My solution was pretty simple:

You have to change the type attribute of the virtual layout tag:
     <virtual-layout type="builtin">

to:
     <virtual-layout type="disabled">

This did the trick for me. All I have to do add files to the server is add files to that directory, and I don't have to worry about duplicates.
