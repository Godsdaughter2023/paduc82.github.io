kevin_kk say:

Windows 10 Fall Creator Update bug report

"Uninstall" option wrongly applies to portable application.

How to reproduce:

1. Go to the unzip/untar/... directory of any portable software and locate its executable (denoted as "x.exe")

2. Create a shortcut for "x.exe", i.e. "x.exe.lnk", and copy it to "%ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs" (this step requires administrator privilege)

3. Press "Win" button on keyboard and type "x.exe"; a list containing "x.exe" should appear within the start menu

4. Right-click the "x.exe" and there is an "uninstall" option -- but this is a portable application!

If one goes directly to Settings/Control Panel, "x.exe" cannot be found in their installed software lists, though.

I was having this issue as well, but managed to force her to change my name. Do the following:

  • Open Cortana's window and navigate to the Notebook panel.
  • Within the Notebook tab, navigate to the About Me section.
  • Select Change my name.
  • Type in the name you wish Cortana to call you.
  • When prompted to, choose That's wrong at the bottom.
  • You will be asked to "tap" the microphone to change the way Cortana pronounces your name. Do so.
  • Select This one when she has the updated pronunciation.
  • To test that she is referring to you correctly, ask "What is my name?"

It seems Cortana defaults to the pronunciation of the name, not the name itself. Theoretically, you could type your name to be Derek, but tell her to pronounce it as "Your Majesty" and she will adopt that pronunciation.

Code 10 error messages are produced by hardware devices that “cannot start”. This failure to start is in direct relation to a bad or faulty device driver. Code 10 errors are reported by the Windows Device Manager and are a clear indicator that the driver for the malfunctioning hardware device is either corrupted or generating errors of its own. In either case, the hardware device driver has gone bad during its operation and should be replaced.

elizabeth23 say:

QUOTE

Files are encrypted through the use of algorithms that essentially rearrange, scramble, and encode the data. A key pair is randomly generated when you encrypt your first file. This key pair is made up of a private and a public key. The key pair is used to encode and decode the encrypted files.

If the key pair is lost or damaged and you have not designated a recovery agent, and then there is no way to recover the data.

Why you must back up your certificates

Because there is no way to recover data that has been encrypted with a corrupted or missing certificate, it is critical that you back up the certificates and store them in a secure location. You can also specify a recovery agent. This agent can restore the How to back up your certificate

To back up your certificates, follow these steps:

  1. Start Microsoft Internet Explorer.
  2. On the Tools menu, click Internet Options.
  3. On the Content tab, in the Certificates section, click Certificates.
  4. Click the Personal tab.

    Note There may be several certificates present, depending on whether you have installed certificates for other purpose.
  5. Select one certificate at a time until the Certificate Intended Purposes field shows Encrypting File System. This is the certificate that was generated when you encrypted your first folder.
  6. Click Export to start the Certificate Export Wizard, and then click Next.
  7. Click Yes, export the private key to export the private key, and then click Next.
  8. Click Enable Strong protection, and then click Next.
  9. Type your password. (You must have a password to protect the private key.)
  10. Specify the path where you want to save the key. You can save the key to a floppy disk, another location on the hard disk, or a CD. If the hard disk fails or is reformatted, the key and the backup will be lost. (If you back up the key to a floppy disk or CD, you must store that disk or CD in a secure location.)
  11. Specify the destination, and then click Next.

cmdrkeene say:

They are supposed to only show up if either of these happen:

  1. You point to the very far right-hand corner of the screen, then drag the pointer up the side of the screen.

    or

  2. You swipe your finger in from the right edge of the touch screen or touch pad.

 

Many times when typing, I see that a palm touching the touchpad causes it to detect this swiping motion and show the charms and clock unintentionally.  You can disable these gestures from the touchpad settings.

 

Unfortunately, every company that builds touchpads has a slightly different way of showing these options.  Synaptics brand touchpads are easy to find from the Mouse control panel.  Toshiba computers require you to download a special "Toshiba control center" program to allow you to set these options.  Dell uses the "Dell QuickSet" tool they ship with their computers.

 

If you look around your control panel and don't see any touchpad settings, please let us know what brand and model of computer you have and we'll try to find the manual for you and look it up.

 

Or if you prefer, you can contact the manufacturer directly -- they would know more about their systems than we would.

 

 

If you're using a regular old mouse, just don't move it to the corners, you can just ignore it.  Even if you do move the pointer over to the corner and those icons appear (or the Start thing at the bottom-left) you can just ignore it and it will disappear immediately as soon as you press any key.

gokul t say:

Hi mab80,

Firstly, I’d suggest you to uninstall the sound drivers from Device Manager.

To uninstall sound drivers,

1. Click ‘Start’, type ‘Device Manager’ in the Start search box and hit Enter.
2. Locate ‘Sound, video and game controllers’ and expand the same.
3. Select the sound drivers.
4. Right click and select ‘Uninstall’.

Follow the onscreen instructions. Restart the computer, if required.

After uninstalling the sound drivers, I’d suggest you to download the Windows Vista drivers from the Toshiba site and install the same in Vista Compatibility mode and check if that fixes the issue.

Here’s the link from where you may download the drivers for Toshiba Satellite M115-S1061

http://www.csd.toshiba.com/cgi-bin/tais/support/jsp/modelContent.jsp?ct=DL&os=&category=&moid=1468779&rpn=PSMB6U&modelFilter=M115-S1061&selCategory=3&selFamily=1073768663

To install the drivers in Vista Compatibility mode,

1. Right click the installation file and select Properties.
2. Click the Compatibility tab.
3. Under Compatibilty mode, check the “Run this program in compatibility mode for:” option.
4. Select Windows Vista (Service Pack 1) or Windows Vista (Service Pack 2) from the available options.
5. Click Apply and OK.

Refer the link below for more information on compatibility mode:

Make older programs run in this version of Windows
http://windows.microsoft.com/en-US/windows7/Make-older-programs-run-in-this-version-of-Windows

Proceed with the installtion.

If the above steps do not help in resolving the issue, you would have to wait for Windows 7 drivers for your model from Toshiba.

You may contact Toshiba for more support:

http://www.csd.toshiba.com/cgi-bin/tais/support/jsp/home.jsp

Hope this helps!
