Word "print layout" view:

Word "print preview" view:

suzanne s. barnhill say:

The only way I've found to get the numbering to appear is to have a printable character before the separator in place prior to the separator being added.

This is what I was suggesting. If you are using a numbered style, and you press Enter to get a new paragraph, you get a new numbered paragraph, but if you then press Enter again without adding text after the number, the number is removed. But if you type a single space after the number, then press Enter, you get an empty numbered paragraph. That's basically what you're dealing with here. Make no mistake about it: when you insert a style separator, you still have two separate paragraphs even though they appear to be one. But if you don't have at least a space in the first paragraph, then it's effectively nonexistent and consequently can't be numbered.

If you put a space in your numbered heading before adding the style separator, you'll get the result you want. If you try to add a space after inserting the style separator, then yes, the space will be formatted as Hidden, but you can select it and unhide it with Ctrl+Shift+H. This does work; I have tried it in Word 2013.

If you don't want to use a space (because it throws the alignment off), you can use a No-Width Optional Break or No-Width Non Break (both available from the Special Characters tab of the Symbol dialog, or use 200C or 200D  followed by Alt+X).

I'd say that displaying the number in Print Layout view when it will not print (and accordingly is not displayed in Print Preview) is a bug, however.

Okay, I stand corrected. Incredibly, if you use one of the Automatic Table building blocks (which is inserted in a content control), the links are much harder to implement (in fact, my first impression was that the links couldn't be used at all).
