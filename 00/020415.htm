Charles, this is an old problem with a non-obvious answer. Graham Mayor explained it in his 8/17/2009 post at https://groups.google.com/forum/#!topic/microsoft.public.word.vba.general/Bk85E1tO9vI: If you run the userform (or the macro that calls it) from the VBA editor, the SendKeys will see the editor as the active window and send the keystroke there. You must call the macro from the document -- use a MacroButton field, a Quick Access Toolbar button, or the Alt+F8 macros dialog.

Also, you need to hide the userform before you call SendKeys, to prevent the keystroke being sent to the userform. I rearranged your original Click procedure this way:

Private Sub cmdOK_Click()
    Dim iParagraph As Integer, iSentence As Integer, iChoice As Integer
    Dim strText As String
    iParagraph = txtParagraphs.Value
    iSentence = txtSent.Value
    iChoice = 1
    If oBtnHelp.Value = True Then iChoice = 1 'Help File text
    If oBtnLorem.Value = True Then iChoice = 2 'Lorem Ipsit
    If oBtnQBF.Value = True Then iChoice = 3 'Quick brown fox
'    MsgBox (iParagraph)
'    MsgBox (iSentence)
'    MsgBox (iChoice)
    '
    Me.Hide

    ' Random "Help File" text
    If iChoice = 1 Then
        strText = "=Rand(" & iParagraph & "," & iSentence & ")"
        GoTo lbl_Exit
    End If
    If iChoice = 2 Then
        strText = "=Lorem(" & iParagraph & "," & iSentence & ")"
        GoTo lbl_Exit
    End If
      
lbl_Exit:
    Selection.TypeText strText
   
    ' Press Enter
    SendKeys "~", True
End Sub

It works as long as it's called from the document. (Technically, I'd prefer to use a Select Case instead of the If..Thens, and remove the GoTos.)

First, I have to disagree with Scott McD on one point. the main advantage of Split Forms is the ability to use Datasheet filtering to find the record(s) you want to view.

ken sheridan say: The Programs(s) column appears to be a multi-valued field.  This represents a many-to-many relationship type, but hides the underlying relationship.  In a conventional model by means of a 'junction' table the columns in the table which models the relationship type would normally each be of long integer number data type.  This is reflected in a multi-valued field which references another table similarly, so its underlying value is really a set of long integer numbers.  Consequently you cannot restrict a form's recordset on such a column in the way you are doing  by pattern matching using the LIKE operator.  To reference a multi-valued field in a query's WHERE clause you must do so on its Value property and the value referenced must be the long integer number which maps to the row in the referenced table with the text value in the relevant non-key column.
