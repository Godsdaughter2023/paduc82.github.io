a. user say:

I haven't heard that about Windows7, but it is certainly possible as I don't use the product and have no training in it (but I hope it isn't true).

The answer depends on what email program and version you are using - it's impossible to answer without knowing that.  Each stores the emails and contacts in different ways in different files and sometimes in different locations.  In general, you need to find these files and then add them to your backup process (or move them to a directory that is already in the backup process).  I use Outlook 2007 and I have my outlook.pst file and archive.pst file (the files with all my emails) and the personal address book moved from the Outlook folders to the Documents folder in a special Outlook email folder I created so that normal backups will include the files (though I frequently do complete image backups so it doesn't make any differernce where the files are located).  Here's the procedure in case that's your program: http://support.microsoft.com/default.aspx/kb/287070.  Some email programs will allow this movement and others won't - then you simply need to add the files manually wherever they are located.  There may be more than one file that you need to move or include for all the information in your email program (emails - current and archived, contacts, tasks,...) - but again that depends on the program.

If you provide the name and version, we'll either try to give you an answer or refer you to experts in the specific program who will know much better than us exactly how to do it (we have most of the references for most of the commonly used email programs).  There you will get a better response (but we do need to know the program and version).

a. user say:

Hello,

Thank you for posting the query on Microsoft Community. I am glad to assist you on this.

The Code Integrity component of Windows enforces the requirement that kernel-mode drivers be signed in order to load. Windows always generate Code Integrity operational events and optionally will generate additional system audit events and verbose diagnostic events that provide information about the status of driver signing.

The Code Integrity operational log includes warning events that indicate that a kernel-mode driver failed to load because the driver signature could not be verified. Signature verification can fail for the following reasons:

  • An administrator preinstalled an unsigned driver, but Code Integrity subsequently blocked loading the unsigned driver.
  • The driver is signed, but the signature is invalid because the driver file has been altered.
  • The system disk device might have device errors when reading the file for the driver from bad disk sectors.

For more information, I suggest you to refer the below Microsoft article and check if it helps.

https://msdn.microsoft.com/en-us/library/windows/hardware/ff539911(v=vs.85).aspx

You can try the below steps and check if it helps.

  1. Open the Task manager. Here’s a tip: Press CTRL+Shift+ESC.

  2. Click File > Run New Task.

  3. Make sure you have a check mark beside “Create this task with administrative privileges”.

  4. Type CMD.

  5. Type the following 4 commands at the CMD prompt:

    dism /online /cleanup-image /restorehealth

    sfc /scannow

    powershell

    Get-AppXPackage -AllUsers |Where-Object {$_.InstallLocation -like "*SystemApps*"} | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

  6. Close the CMD window.

Media Center suddenly unable to watch live TV, channel not available - Microsoft Community

bob_cherylwhiting say:

Media Center suddenly unable to watch live TV, channel not available until recorded.

New computer with Windows 7, Office 2010 and no other apps.  WinTV dual tuner.  Primary use is as a media center.   All has worked well for weeks.  Suddenly, channel 11 shows channel not available but will appear if record is selected.  Other channels display as expected.  We frequently watch channel 11 as well as schedule recording on channel 11 with no problems.   Tried rescanning all channels both from the media center and from the WinTV apps. WinTV apps can display two channels including channel 11.   Media Center scheduled recording on channel  11 still works.  There was a post similar to this back in 2009 but with no resolution.  Has anyone solved this problem?
bob_cherylwhiting say:

Problem resolved:

Select the WMC TV schedule/directory and select the channel (11) on the far left.  It brings up the specific channel schedule with an “edit channel” button on the left.  This button brings up a menu on the left that includes “Edit Sources.”  Edit sources for the channel (11.1) showed three entries.  I only have two tuners.  If the pointer hovered over the checkboxes, The first showed tuner #2 the second showed tuner #1 and the third showed tuner #1 again. I unchecked the third duplicate tuner #1 entry.  WMC now tunes channel 11 OK.  I have no idea how this setting occurred.  Problem resolved
