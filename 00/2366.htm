jay freedman say:

That method works for table captions and figure captions, because the table TOC and figure TOC stop collecting text at the first (hidden) paragraph mark after the caption's label. It does not work for headings because the main TOC collects any text with the specified styles. Any text that you want to have omitted from the TOC must have a style other than one of the heading styles. Your method applies the same style to both parts of the heading, and the style separator does nothing to change that.

There is no option setting that will make the method work for headings.

Please read the "Hidden paragraph mark" and "Lead-in emphasis and style separator" sections of http://wordfaqs.ssbarnhill.com/RunInSidehead.htm. Note especially the paragraph

Format the first paragraph as Heading 3 (or whatever style you’re using for this heading level) and the second paragraph as Body Text (or whatever text you’re using for the text paragraphs).

In your case, you should create a custom style that has the same font formatting as the Heading X style (for X = 1, 2, etc.), but based on the Normal style rather than on Heading X. Because the Normal style has an outline level of "Body Text" and the custom style inherits that, the TOC will not include any text to which the custom style is applied. Use the custom style for the second paragraph, and otherwise follow the instructions.

suzanne s. barnhill say:
To add to what Charles and Jay have said, note that you do not need to format the paragraph mark as Hidden when using a style separator, but you do have to have two separate paragraphs with two different styles. When you have different styles, you can place the insertion point anywhere in the first paragraph and insert the style separator, which has the same effect as making the paragraph Hidden except that the two paragraphs appear combined even when you have Hidden text (and other nonprinting characters) displayed.
