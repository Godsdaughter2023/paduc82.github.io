Step 1: Try to clear the DNS cache.

  1. Click Start, and then type cmd in the Start Search box.

4.    Reboot your computer.

peter_cuthbert say:

Hi Folks

For those of you still suffering this problem, I have come across a fix that works in two out of three cases (i.e. it works for two of our machines but not the third.)

The fix is to disable 'Fast Boot' which presumably (I'm guessing!) allows Win 10 time to collect the IP address from the router.  That explanation does not really explain why our third machine on the wired network still has the issue.  Anyway, Here is what you do:

Right click the start button and go to Control Panel.

Click on Power Options (May need to adjust the view to Categories)

Select 'Choose what the Power Buttons do' and then near the top of the screen choose 'Change settings that are available'.

This will un-grey the tick boxes at the bottom of the screen.

Untick 'Turn on Fast Startup' then Save the Changes.

Exit from Control Panel.

Next time your PC is switched on go off and make a coffee while it boots as start up is a whole lot slower, but if it picks up the IP address you will not have to do a re-boot (or two) so time will be saved overall.

Hope this helps.

Pete

netsh winsock reset catalog (reset winsock entries)

netsh int ip reset reset.log hit (reset TCP/IP stack)

c. Reboot the computer and check.

Step2: Flush your computer's DNS cache.

a. From Start screen, type command prompt.

b. Right-click on Command Prompt and choose Run as Administrator.

c. At the command prompt, type the following commands. Press Enter after each command.

ipconfig /flushdns
ipconfig /registerdns
ipconfig /release
ipconfig /renew
