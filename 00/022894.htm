That's exactly what a template is supposed to do: spawn a new document without opening the file itself.

If you save a document as a template and styles or other items disappear, then those items weren't actually in the document to begin with. They were in the template to which the document was linked, or some other file.

AutoCorrect entries are stored in an ACL file, not in a template. Each language that you use in Word has a separate ACL. For Word 2016 for Mac, the ACL files are stored in your hidden user Library folder.

The missing styles were likely stored in the older template on which your document was based. Most often, this is Normal.dotm. If you create a new blank document, do you see your styles?

You have to open a document to do a Save As to create your template. The original document should still be on your computer. A spotlight search should help to find it.

shane devenshire say:

To answer your question of what's wrong with COUNTA consider:

=COUNTA(IF(A1:A4="Bob";A1:A4))

A1:A4="Bob" evaluates to {TRUE;FALSE;FALSE;TRUE}

IF(A1:A4="Bob";A1:A4)  evaluete to {"Bob";"";"";"Bob"}

COUNTA counts all nonblank entries and "" is not treated as blank here. 

If the range A2:A3 were blank the above results would still be 4!  However, in that case COUNTA(A1:A4) would return 2.  And if fact if all the cells were blank the array formula would return 4!?

So one would be justified in saying your logic is correct but Excel work's differently.
