But the point is that the list is probably populated by Word looking at the registry for OLE objects that conform to a particular specification. I am pretty sure that Word does not create a separate list of object types that you could edit. 

So if you want an object to appear in the list and be useful, the object needs to be of the correct type and properly registered. 

If you want to ensure that things that are no longer on your system (WordPad etc.) do not appear in the list, you would need to find the relevant registry entries and either delete them, or modify them in such a way that Word will not list them. 

If you want to exclude other things (that are on your system, but you do not want to see listed), then I suspect that you could do that by altering the properties of the registry entries, but that would probably be a bad idea since it might have unexpected side effects. 


If you want to delete entries from the registry, I would
 a. ensure your system is fully backed up and can be recovered
 b. be certain that you have identified the correct registry entries
 c. export the relevant part of the registry and name the file carefully so that you know what you exported
 d. delete the tree
 e. check that your action had the desired effect. If not, re-import the tree straight away. If you can't do that, recover the system.

I'm not that knowledgeable aboutt he details: if I were doing this, I would look for each "Equation" key (not just the version-specific keys such as Equation.3", and look at how they are interrelated, then start removing subtrees and check what happens in Word. You willtypically find entries under HKEY_CLASSES_ROOT. I would make the assumption that everything installed on the machine (or at least, everything made available to all users on the machine) would be in HKEY_LOCAL_MACHINE, and that each user might also have HKEY_CURRENT_USER keys, which would also be mirrored per-user in HKEY_USERS and in HKEY_CURRENT_CONFIG.

So if all I needed to so was remove the entries related to a single user (me), I'd probably start with HKEY_CLASSES_ROOT and HKEY_CURRENT_USER (HKEY_USERS and HKEY_CURRENT_CONFIG should automatically reflect what you do in HKEY_CURRENT_USER.

Another approach, if you can backup your system and be confident that you can restore it fairly quickly and easily, might be to run the Office setup program and see if you can find the options to remove the components that you do not want. It is always possible that the Setup will fail if the components have already been partially removed, but I would hope that Setup would succeed and remove any remaining traces.
