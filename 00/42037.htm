gerry c j cornell say:

However, if i go inside c drive and check properties of all folders including hidden files and protected operating system files, windows shows the total as 29.1 GB.

My interpretation of your statement is that you are getting these figures from Windows Explorer. The reporting from Windows Explorer has never been 100% accurate.

As I mentioned earlier you need to take into account the gain from file compression. When Disk CleanUp is run and it is configured to compress, it reduces the size of older files. Thus a 100 kb file may be compressed to a smaller size so that it occupies less space on the disk. Compressed files by default are coloured blue in Windows Explorer. If you look at the file properties of a compressed file you will see reported the original file size and the size on disk. The sum of these gains can be significant if the backup files, which can be used to reverse the updating of windows files, have been compressed. The overall effect of disk compression will be to make the discrepancy you think you have even bigger.

When reconciling disk usage you need to make sure you have included the System Vulume Information folder and it's contents. Although you can see the folder in Windows Explorer the space occupied is not reported and it occupies a a lot of disk space. Treesize reports the figure.

Another misleading feature of the reporting by Windows Explorer of disk usage is that, even with “Show Hidden Files” enabled, some files still remain hidden. These “super” hidden files have always been difficult to detect as you cannot see them in Windows Explorer. With Windows XP it was relatively easy to detect one of these files because, if fragmented, they would be listed as a fragmented file in Disk Defragmenter. With Vista and Windows 7 the listing of fragmented files in Disk Defragmenter was dropped so you have to rely on third party tools like Treesize to reveal the existence of these files.

I do not have a comprehensive list of files that remain hidden in Windows Explorer but the most notable in terms of size relate to protected storage (backup files to protect against the consequences of accidental file deletion). Norton started the practice with Protected Storage and Retrospect was another providing this type of software. Another unreported file is the MFT file (Master File Table) which is an index maintained by Windows so that it knows where each file is located on the disk. However the MFT files is not so large so it does not create an obvious discrepancy but it is nevertheless part of the discrepancy.
