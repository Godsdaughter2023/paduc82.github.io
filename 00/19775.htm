Description of problem
In Gmail, go to Settings/"Forwarding and POP/IMAP"/POP Download

  1. Outlook.com assigns the date 11/02/06 to all mails that arrived before 11/02/05. Probably, this bug occurs for messages that are not POP-enabled.
  2. All other mails will get the correct date.

Possible solution (haven't tried it yet):
In Gmail, select the status: "Enable POP for all mail (even mail that's already been downloaded)".

This should make all mails POP-enabled and the Outlook.com import-engine gets an easier job.
