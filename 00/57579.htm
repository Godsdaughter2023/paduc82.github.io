Why can't Windows Live Mail 2011 purge deleted messages from IMAP - Microsoft Community

n4aof say:

Why can't Windows Live Mail 2011 purge deleted messages from IMAP accounts?

Unlike earlier versions of Windows Live Mail, the 2011 version does not appear to have any setting to automatically purge deleted IMAP messages and it also does not have any way to manually delete IMAP messages.

 

With a POP3 account, "deleted" messages are moved to the trash folder, and WLM2011 provides an easy way to empty the Trash folder.

 

With an IMAP account, WLM2011 displays the Trash folder and provides the same easy way to empty that Trash folder; BUT deleted messages are not moved to the Trash folder.  Instead, deleted messages are marked as deleted and left in the INBOX

 

By default WML2011 hides these messages, but under Accounts - IMAP settings, it has a choice to display deleted messages. 

 

Regardless of whether the "deleted" messages are displayed or not, they remain in the INBOX of the IMAP account. 

 

The function to actually remove the deleted messages from an IMAP account has always been called "Purge" deleted messages, but that function apparently doesn't exist in WLM2011

 

So far the only way to purge deleted IMAP messages that I have found is by using a browser to visit the IMAP account webmail to perform this function.

 

 

 

n4aof say:

After reading a lot of related questions and mostly a lot of flailing around hunting of options in strange places, I found that the difficulty is based on two things in WLM2011:

1. The defaults for handling IMAP accounts are terrible

and

2. The settings for handling IMAP accounts are scattered in several different places (most of which are different from earlier versions, making them even harder to find)

 

The critical setting towards solving my problem is under OPTIONS - MAIL - ADVANCED, where you need to put a check in the box for "Use the 'Deleted Items' folder for IMAP accounts." Once you do this, WLM2011 will move "deleted" IMAP messages into the Trash folder, THEN you can use the function to empty the trash folder as the way to purge the deleted messages.

 

I cannot understand why anyone chose to have WLM2011 default to hiding the "deleted" IMAP messages _AND_ chose to have it default to not using the trash folder.  That combination leaves users thinking that deleted messages are immediately gone forever (resulting in some questions here) while being unable to actually remove the "deleted" messages (resulting in some issues with overflowing inbox problems).

Format a storage device

Important When you format a storage device, it erases all of the information saved on that device, which can’t be recovered. Make sure that you don’t want the information stored on that device, or that you’ve backed up the information to another device before you continue.

To format a storage device

  1. Press the Guide button  on your controller, go to Settings, and select System Settings.
  2. Select Storage or Memory.
  3. Select the storage device that you want to format, and then press Y  on your controller.
  4. Select Format. You will see the following message:
    This will delete all content on this device. Do you wish to continue?
  5. Select Yes to format the storage device.
  6. If prompted to enter your console serial number, do so, and then select Done. This step helps guard against accidental formatting.
