To get the desktop with the (Win 7) style start menu:

1) Go to the desktop.

2) Place the mouse pointer onto the middle of the taskbar and right click once.

3) Left click on PROPERTIES.

4) Left click on the START MENU tab.

5) Put a tick in the USE THE START MENU INSTEAD OF THE START SCREEN option.

6) Click APPLY.

7) Log out and then log in again.

Doing this sets the desktop as the default instead of the start screen (Metro UI) but you cannot have access to the start screen (Metro UI) after this is done unless you reverse the above steps.

Joe, I suggest you report it regardless, even if you believe the problem is unique.  There are millions upon millions of unique configurations and yours might be the magic bullet that hit on a bug that might hit a small segment of the user base once the code goes into production.

As for digital entitlement, there is nothing in Windows to support digital entitlement so the build wouldn't have anything in it that changed DE.  Digital entitlement refers to the status of the computer, not the software.  Activation tells you that Windows is activated with a digital entitlement, not by digital entitlement.  Digital entitlement is a handshake between the computer's unique identifiers and Microsoft's servers and is requested by the licensing service.

My point is that digital entitlement is based on the computer's registration as a Windows 10 device.  Any copy of the same edition of Windows 10 on the computer is therefore digitally entitled to activation based on the computer's status.  Since DE is based on the computer's registration status I doubt that changes in the software would impact it.  Nothing concerning DE except its confirmation is written to the software and nothing is written to the computer's firmware so there is nothing regarding DE anything in the build could change.  That is why it is an effective anti-piracy technique.

DE doesn't look for anything because it is only a status.  The licensing service is what confirms DE when it submits the computer's unique identifiers to Microsoft every few hours.  There is nothing on a hard drive that affects DE.  DE is the result of a "handshake" between the computer's identifiers and Microsoft's registration database.  Hard drives are not part of the computer's unique ID for Windows's 10 purposes and only the system drive ever was under the old activation scheme.  

Activation should return in a few hours after moving the hard drive back to the original computer but if not I think if you did an upgrade-in-place with the OS on the problematical hard drives you would restore activation.  It is a good experiment in any case.

bob jones aka: cybertaz say:

Pursuant to what Stefan wrote: There are many items listed in the dialog for customizing the Ribbon & QAT which currently do not work. Some are carryovers from previous versions which no longer are functional but haven't been removed, some represent features in the Windows software which have not [yet] been implemented in the Mac version. It appears that the Show Comments command which you added falls into one of those categories... probably the latter.
