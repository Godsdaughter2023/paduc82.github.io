Password protection of individual files is a function of the application used to create the files. Yes it does.  It takes a look at your files and copies any files that got changed since the last time the backup ran.  The File History feature even keeps each changed version so that later you can go back in time and, for example, see what a file looked like before the last change.

cmdrkeene say:
It should be fairly quick after the initial massive pull that grabs the full set.  After that the most time-consuming part is running through every file on the system and computing a hash of it, then comparing that hash to the backup to tell if the file that exists is newer/changed since the old backup.  After all those comparisons are done (takes a while), it copies anything changed to the destination.

cmdrkeene say:

First, please don't try to explore your backup.  Like other backup systems, File History performs a compression on the backup sets and you can't just click open the folders to see the backups.  There is also the matter that it backs up any file multiple times, once for every time the file changed, so you can "go back in time" and see whta the file looked like yesterday, last week, last month, or last year.  Since there's multiple copies of the same file, but files can't have the same name, the software performs functions to make this work.

So, rather that just clicking open the folders, you need to use the program that can read the compressed versioned copies.  Luckily you do that from the same place you setup the back up in the first place. Just go back there and choose "restore personal files" at the top left of the screen:

Now, if any file wasn't backed up, it will tell you.  The reason why is usually because the file is already on the backup drive (you can't backup a backup of a backup).  But just let me know what the reason says and we'll learn why.
