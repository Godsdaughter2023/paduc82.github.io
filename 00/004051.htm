bob jones aka: cybertaz say:
There are a few things you can control, but there are others which you can't :-)

The issue with column widths can usually be avoided by using the same versions of the same fonts [preferably those that ship with Office] on both the Macs & the PCs. Font metrics have a lot to do with spacing, so when font substitution occurs it can have a dramatic effect. Also, avoid the common temptation to 'tighten' everything -- leave some breathing room even if it means using a slightly smaller font size or wider column. Take advantage of the Scaling feature available on both PC & Mac for fitting to the required paper size when printing.

The printers/drivers also contribute to the cause. Not all printers handle things identically, so when a file is displayed on any other system that uses a different printer there's likely going to be some degree of adjustment made. This is a matter over which you often have little control but if you follow the suggestions above it should happen less often. Also, it may help to use the Page Setup - Options to select Format for: Any Printer rather than for the specific default printer.

Other issues such as monitor dot pitch, display resolution & OS distinctions are usually out of your control. Fortunately, they may result only in display issues that don't carry over into the actual printed work.

bobgreenblatt say: The worksheets will be remarkably close if you use fonts that are the same on the PC version. Stick to Ariel or other common fonts. Also make sure you and your clients have the latest updates for their print drivers.

jllatham say:

Based on your last post, I believe this code/macro would do the job for you quite well:

Sub CopyToList()
  'The sheet the work is to be done to must be selected
  'before you start this macro
  '
  'the address of the first cell to copy & repeat
  Const SourceCell_1 = "F1"
  'the address of the second cell to copy & repeat
  Const SourceCell_2 = "G1"
  'the 'key column' - one that goes on down the sheet
  Const keyColumn = "A"
  'column to repeat SourceCell_1 data into
  Const destCol_1 = "B"
  'column to repeat SourceCell_2 data into
  Const destCol_2 = "C"
  'now some variables we need
  Dim lastRow As Long ' will be last row in keyColumn
  Dim nextRow As Long ' will be next row for destCol_1 & _2
 
  lastRow = Range(keyColumn & Rows.Count).End(xlUp).Row
  nextRow = Range(destCol_1 & Rows.Count).End(xlUp).Row + 1
  If nextRow >= lastRow Then
    'everything already filled up!
    MsgBox "No 'room' to add entries.", vbOKOnly, "Quitting"
    Exit Sub ' and end
  End If
  'actually, we can fill blocks of cells very quickly!!
  'fill the empty destCol_1 cells
  Range(destCol_1 & nextRow & ":" & destCol_1 & lastRow).Value _
   = Range(SourceCell_1).Value
  Range(destCol_2 & nextRow & ":" & destCol_2 & lastRow).Value _
   = Range(SourceCell_2).Value
  MsgBox "Done", vbOKOnly + vbInformation, "Task Finished"
End Sub

vijay a. verma say:

You may use below macro. Set the sheet name appropriately which is in Bold.

'**** Macro Starts *****

Sub CopyCells()
    Dim Ws As Worksheet
    Dim FirstRow As Long, LastRow As Long
    Set Ws = Worksheets("Sheet4")
    If Ws.Range("A1") = "" Then
        FirstRow = Ws.Range("A1").End(xlDown).Row
        Else
        FirstRow = 1
    End If
    LastRow = Ws.Range("A" & Rows.Count).End(xlUp).Row
    Ws.Range("F1:G1").Copy Ws.Range("B" & FirstRow & ":" & "B" & LastRow)
End Sub
