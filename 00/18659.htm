5. the formula now becomes =vlookup(lookup value, named range name, 2 , false)

 

it will then find the lookup value in the first column and return the corresponding value from the second.

Get faster logons, after an OS update or upgrade. Windows Update installs OS updates and upgrades, which requires system reboots. Until now, following a reboot you needed to logon manually, for the system to complete the upgrade process. However, with our new feature, Microsoft Account and Local Account Users can make the OS save user’s credentials on disk temporarily, between Windows Update initiated reboots, so that the user is automatically logged in and the system is locked to keep the user safe. Note that the feature is available to MSA and Local accounts that do not have work policies.

  1. Open up an admin command prompt.
    1. Run Start-> All Apps->Windows System->Command Prompt
    2. Right Click->More->Run as administrator
  2. Run the following commands in the command prompt

REG ADD "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v

ARSOUserConsent /t REG_DWORD /d 0x00000001 /f

max, singapore say:

> .. Is there a way to pull the data for each row into a report based on someone entering item #'s in a table

Here's a simple formulas way
Assume your source data is in Sheet1, data in row2 down, key col = col A (item#)
In another sheet,
Assume the input for item# will be made in A2
Put in B2: =IF(Sheet1!A2=$A$2,ROW(),"")
Copy B2 down to cover the max expected extent of source data, say down to row 30000?
Minimize/hide col B
Put in C2: =IF(ROWS($1:1)>COUNT($B:$B),"",INDEX(Sheet1!A:A,SMALL($B:$B,ROWS($1:1))))
Copy C2 across by as many cols as there are within the source, then fill down by the smallest range large enough to cover the max expected extent of results for any particular item#, say down to row 50? Format the cols as required to be consistent with the source. And that should yield the desired result lines, all neatly packed at the top


max, singapore say: > .. Do you know what a modification to this would be to be able to select more than one item # 
> .. Let's say that i needed to be able to select 15 different items numbers, would I just insert more lines at the top and imbed 'IF' statements?

The required change to handle multiple item#s at one go is quite straightforward ..
Assume the inputs for all item# of interest will be made in A1 down
Put instead in B2: =IF(COUNTIF(A:A,Sheet1!A2),ROW(),"")
Copy down all the way. That's it. No change required to the extract formulas, and the desired results will appear as before
