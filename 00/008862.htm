Linking a successor task's Start date to its predecessor - Microsoft Community

emerald77 say:

Linking a successor task's Start date to its predecessor task's Finish date

I have a very simple project plan consisting of just 8 tasks.  The 8th task's Start date needs to begin the day after the 7th task's Finish date.  The 7th's task's Start date needs to begin the day after the 6th task's Finish date, and so forth.  The only date that is unmovable is the 8th task's Finish date.

In other words, I need to set up the Predecessors, Successors, and Constraint Type fields in such a way that the Start and Finish dates for Tasks 1 through 8 (except for Task 8's Finish date) are derived by Task 8's Finish date and the Durations for each task.

Being relatively new to Microsoft Project, I would greatly appreciate any guidance/assistance.  Thanks.

dale howard [mvp] say:

Bob --

No, you did not miss a step, but you are definitely misunderstanding how Microsoft Project works.  First, you should not manually type the Start and Finish dates of any tasks.  Let the tool calculate dates using the Duration values you enter for each task, along with your task dependencies.  Manually typing Start and Finish dates will cause Microsoft Project to automatically set a constraint on the task, and the software does this without asking you.

Secondly, Microsoft is set by default to calculate the Finish date of your project based on your Start date.  You enter the Start date for the project by clicking Project > Project Information and then entering the Start Date value.  If you need to schedule the project backwards, from a Finish date and letting the software calculate the Start date of the task, do the following:

  1. Click Project > Project Information.
  2. Click the Schedule From pick list and select the Project Finish Date value.
  3. In the Finish Date field, select the date your project must finish.

Keep in mind that you should always be scheduling your project from a Start date, letting the software calculate the Finish date.  This is the only easy way to know when your project is currently scheduled to finish, especially if the project schedule slips.

Your questions are good ones, Bob, but you have much to learn, too much for you to learn by asking questions in this forum.  I would strongly recommend you purchase a book about Microsoft Project, or take a course from someone who knows how to use the tool.  Either approach would help you to become a better Microsoft Project user.  You might also consider joining MPUG, as this organization holds many useful webinars and has lots of information in its web site.  Just a thought.  Hope this helps.

dale howard [mvp] say:

emerald77 --

Select all eight tasks and then click the 'Link the Selected Tasks' button in the Schedule section of the Task ribbon in Microsoft Project.  This will establish a Finish-to-Start dependency on all of the selected tasks, calculating the Start date and Finish date of each Successor task automatically.  You DO NOT need constraints other than As Soon as Possible on each task as the dependencies will take care of the scheduling for you.  Hope this helps.

I think I begin to see your problem - it is actually over on your [IF Data] sheet, right?

What you need to know for columns C and D is whether or not a particular Spec item is listed on the [Sep-Dec 2014] sheet or not.  Let's look at a couple of rows on the [If Data] sheet and set up a formula that will pretty much do that for you.

In cell C17, put this formula:

=IF(ISNUMBER(MATCH(B17,'Sep-Dec 2014'!$E$6:$XEZ$6,0)),355,0)

Put this one into D17

=IF(ISNUMBER(MATCH(B17,'Sep-Dec 2014'!$E$6:$XEZ$6,0)),339,0)

You should see 355 and 339 in the two cells

On down to next row, 18:

In C18:

=IF(ISNUMBER(MATCH(B18,'Sep-Dec 2014'!$E$6:$XEZ$6,0)),281,0)

and in D18

=IF(ISNUMBER(MATCH(B18,'Sep-Dec 2014'!$E$6:$XEZ$6,0)),249,0)

in these cases you should see 281 and 249 in the cells

Let's see one where the number in column B is not on the [Sep-Dec...] sheet.  On down to row 19,

In C19:

=IF(ISNUMBER(MATCH(B19,'Sep-Dec 2014'!$E$6:$XEZ$6,0)),355,0)

and in D19

=IF(ISNUMBER(MATCH(B19,'Sep-Dec 2014'!$E$6:$XEZ$6,0)),339,0)

These should remain zero.

The ISNUMBER(MATCH(B#,'Sep-Dec 2014'!$E$6:$XEZ$6,0)),  portion of the formula simply returns a true or false indication as to whether or not the entry in column B appears in row 6 of the [Sep-Dec] sheet or not, and if it does, it puts up the designated non-zero value (or zero as in the situation at row 16).

Now, you could replace those hard coded values such as  ,281,  ,249, ,355, and ,339,  with a VLOOKUP() formula into your table you have set up over on the [VLOOKUP Data] sheet.  Lets say in the 'Case to Case' area, you need to pull the values from the columns you have labeled Spec 3 and Spec 4 from that table.

So the formulas at row 17 could become:

In C17:

=IF(ISNUMBER(MATCH($B17,'Sep-Dec 2014'!$E$6:$XEZ$6,0)),VLOOKUP($B17,'VLOOKUP Data'!$A$1:$M$8,4,FALSE),0)

The advantage here, is that you don't have to tailor each formula on each row on that [IF Data] sheet.

Share Point disable download feature - Microsoft Community

vikas awasthi- [o365] say:

Share Point disable download feature

Hi,

Is it possible to prevent users from downloading a file in SPO?

if yes, how?

fetaa say:

hi vikas,

 

yes, we can only grant users the view only permission for the site and uncheck the following two permissions in the view only permission level. after that, the users who have the view only permission can't download files from the libraries of the site.

open items - view the source of documents with server-side file handlers.

view versions - view past versions of a list item or document.

 

if the users are in the site members group, you may remove the users' permissions by removing everyone except external users from the team site members group, and then only grant the users view only permission. to do that:
1.remove the users from the site collection:
go to sharepoint online sites>click the gear on the upper right corner>site settings>under users and permissions, click people and groups> check everyone except external users >click actions and select remove users from the group.

2.remove the open items and view versions permissions from the view only permission level and assign the users the view only permission:

a.go to site settings>under users and permissions, click site permissions>under permissions, click permission levels>find and click view only>under list permissions, uncheck open items and view versions permissions >click submit at the bottom of the page.

b.go back to the site permissions>click grant permissions>input the users' email address and select view only permission>share.

 

 

if there is any update, please let us know.

 

thanks,
felix
