Get More Details On the Error

 

First convert the number 2147942403 from decimal to hexidecimal.

 

Start Calculator (Start - All Programs - Accessories - Calculator) and choose View menu - Scientific, then View menu - Decimal. Enter 2147942403. Then View menu -  Hex. It will say 0x80070003.

 

To decode 0x80070003. The 0x means it's a hexadecimal number, the 8 means error, the first 7 means it a windows error, and the rest of the number, 3, is the actual Windows error.
 
To look up the error we need it in decimal format. Start Calculator (Start - All Programs - Accessories - Calculator) and choose View menu - Scientific, then View menu - Hex. Enter 3. Then View menu -  Decimal. It will say 3.

net helpmsg 3

davidmcandy say:

mui files contain language strings. Files in en-us sub folder will contain english files for the dll in the parent folder. The files know about each other and operations on one are automatically directed to the right file.

 

Windows hides the version number of mui files because they are different programming teams. To get the version of the mui file type this

 

ntsd /z "C:\Windows\System32\en-US\cscsvc.dll.mui"

 

then type

 

lmv

 

at the prompt.

 

Type q to exit.
