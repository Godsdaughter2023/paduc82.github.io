ramesh srinivasan say:

This happens if you've configured Adobe to check for and set itself as default. When Adobe tries to set itself as the default by writing to the registry (which is not supported in Windows 10), Windows 10 simply resets the "UserChoice\.pdf" registry setting, due to hashing algorithm in place.

So, the ultimate fix is to prevent Adobe from resetting itself as the default. You should be able to do that in options. Also, make sure you have the latest version of Adobe Reader.

The "NoOpenWith" fix is valid in situations where the reset happens despite the above precaution taken, in some cases that have been reported here before. If "NoOpenWith" is applied, then the UserChoice\.pdf is removed (null) when a wrong hash is detected, but Edge won't be set as default. Then the file association settings in HKEY_CLASSES_ROOT or the per-user setting under HKCU\Software\Classes is used.

brian souder say:

I this the same for IE11 and Edge?

ramesh srinivasan say:

This is for all file types. For instance a program tries to set itself as the default for .htm files in an incorrect way, then it would be reset to Edge automatically.

But Internet Exporer doesn't corrupt the file association settings anyway. When you click Set associations in Internet Options, it just tells the user to goto Default Apps to configure the defaults... just exactly how Microsoft wants 3rd party programs to behave.

Then what's resetting the default browser? May be another browser, or your html editing program. For example, Google Chrome has an option to chec/reset itself as the default at every startup. This option has to be unchecked. Same for other programs.
