stefan blom say:

If Word is changing the language at the insertion point, you can prevent that from happening by clearing the "Detect language auomatically" option at Review | Language | Set Proofing Language.

Note that after you have changed the default proofing language (at Review | Language | Set Proofing Language) you may still encounter problems in existing documents. Modifying the language of the Normal style should help, but sometimes you must resort so selecting all the text and manually applying the language of your choice to the selection.

darkkoala say:

Mail Merge using headers in the first column?

I know it's traditional to use the headers from the first row of an excel document in a word document mail merge, but I worked for a company who actually had their table rotated so the headers were in the first column. I'm trying to duplicate that now and failing miserably. The excel document we use has 28 headers to fill in and it would be great to be able to see it all on one page rather than having to scroll back and forth. Reading headers from the first column rather than the first row would enable that. Is there a way to make that happen?

Thanks!

jay freedman say:

Excel can do this without a macro.

  • Select and copy (NOT cut) the entire range of data.
  • Select an empty sheet in the workbook.
  • Right-click in cell A1 of the new sheet.
  • In the Paste options on the right-click menu, click the Transpose button (4th button from left).

Use the new sheet as the data source for the mail merge.
