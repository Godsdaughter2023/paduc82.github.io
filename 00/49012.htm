robert aldwinckle on forums say:

                    see the lock screen, and am able to log in. Then I just get a black screen. 

How should I proceed?

Two suggestions for the Login screen:

  1. Start Narrator.  Win-Enter if you're lucky; otherwise use the Accessibility menu.
  2. Use the Project dialog (Win-P) to see what you have and try something else (such as Duplicate instead of whatever you are getting).  Note that usually when I make this suggestion my assumption would be that you would not have visibility for that dialog and therefore suggest you just press CursorUp and Enter to try something else.  (Narrator should help in that case by at least naming the button you would navigate to in case you wanted to try doing a different navigation before pressing Enter.)

It's interesting that you can actually do those things with Task Manager but it's not clear what the state of explorer.exe is then.  Should we assume that explorer.exe is crashing?  Then Event Viewer (probably Applications log) should have evidence of that.  Then the question would be "why?"  And the usual answer to that is interference from incompatible system modifications such as a third-party replacement to a Start button.  Etc.  (Do you have such things?)

So, what all else have you tried doing with your Task Manager workaround and have you tried it in a safe mode boot?  One possibility which would be fun to try is starting ProcMon first instead and then seeing in it what is happening to explorer.exe when you start it.

For a simpler alternative test try giving your explore.exe command an operand, say a drive letter.  Then instead of seeing the Desktop, Taskbar, etc. you would only expect to see a File Explorer window open at that directory (and hopefully not crashing until you had at least seen its window open).

 

HTH

Robert Aldwinckle
