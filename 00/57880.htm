HOW TO WRITE , COMPILE AND RUN JAVA PROGRAM ON "WINDOWS 8 X64 BIT" IN COMMAND PROMPT

How to Write , compile and Run java Program via Command Prompt(CMD) in 64 bit Windows 8/7
=======================================================================================
In the previous version of windows writing java programs is done via "edit " command. Bur now in the modern 64bit version of windows don't have the classic MS-Editor(edit.exe) though 32bit versions still have it.. In 64bit Windows the only option to write any text via CMD is "notepad"........All the procedure is as follows

1. Download latest JDK 64bit (jdk1.7.0_25 x64) & install it.

2. Right click on Computer icon(desktop) and click properties
    click Advance System Settings
    click Environment Variables
    In the system variables box click on Path ----Click Edit
    In the variable value just ADD ";C:\Program Files\Java\jdk1.7.0_25\bin" without quotes at last. i.e. Don't delete a single letter of the previous value, only add the above text after it
    save every thing you've done by clicking OK.

3. Now open CMD(windows key +R and type cmd & Enter)

4. Type "notepad" without quotes

5. Write your java program and save it in your Home Directory. i.e. C:/Users/usrername(currently you have logged in)

6. Now you can compile your java program by   javac<space><filename.java> and by typing    java<space>filename you can run your java program.

**This process is also applicable for 32bit windows(7&8). Change is that you have to download latest jdk 32bit ( till now latest version is 1.7.0_25) version and you can use "edit" or "notepad" any command to write your program. All the rest is same

slance310 say:

Hi,

Normally the EFI partition is created on a UEFI based machine when Windows is installed.... You can live without the Recovery Partition... I delete all of mine...

About the only inconvenient thing is that some of your REFRESH features may not work fully... It's only around 450mb anyway, so it's not going to hold a complete installation.

I think that the reason it wasn't created is the way Windows was reinstalled...

I'm pretty sure it is only created when you reinstall with a specific type of flash drive...

UEFI Bios requires that Windows be installed from a flash drive...  Unfortunately none of Microsoft's tools to create a installation Flash Drive will work

What you might try is to obtain an image of Win 8.1, or possibly use a DVD, along with a tool called RUFUS... RUFUS may or may not be able to use a DVD in place of a ISO Image...

The big sticking point is that the Microsoft tools create a NTFS file system Flash Drive.

This works fine for non UEFI machines, or if UEFI is disabled...

UEFI Bios needs the Flash Drive to be of FAT32 file type.... RUFUS is one of the few tools that can do this...

If you're not going to lose a lot of programs and stuff, you might try installing directly to the SSD... I'm not sure if you can Clone just the EFI partition to the SSD separately, but maybe..

I do know that you can make an IMAGE of the EFI partition and then Restore it to the SSD after you have resized the OS to make room at the end of the OS partition....

Without knowing more of your method, I don't know if your cloning program will also do an image or not... Some will and some won't.

slance310 say:

Hi,

Something I just thought of that I should have asked at the very start...

Did you set your SSD as the Boot Device in your BIOS ?

Usually when you switch boot devices your BIOS may need to be set to boot from the new one. Sometimes it guesses correctly, but a lot of times not.

slance310 say:

Hi,

All of the different manufacturers are a bit different, but in addition to selecting the boot devices and the order... like 1) USB Flash 2) CD\DVD Rom 3) HDD...

There should be another selection to select WHICH HDD is the primary HDD boot drive.

They should both show up... If you have already reformatted your HDD, you might try just unhooking it and use the SSD until you can get it to boot...

My BIOS shows the model number of the HDD\SDD so you can tell them apart...

The thing of it is that some UEFI machines don't boot to the Windows Partition...

They boot to the EFI partition, then switch to Windows... I'm not sure about your machine.

It might boot to Windows and use the EFI partition as a reference...

I'm not sure what type of Recovery drive you have... I am also not sure that just creating a EFI partition is enough... There may be files that are required...

To be honest, I don't have a lot of practice with UEFI Bios stuff...

Most of my machines are of the older type.. My laptop uses Hybrid UEFI, but I haven't had many occasions when I needed to mess with it.

BTW... Another possibility for selecting your boot device would be a to hit the proper key at the manufacturers screen when you power on...

Something like ESC or F9, F11... This will let you pick a boot device that works for just the one boot...
