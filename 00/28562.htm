lenoreandken say:

How do I generate tests from random selections of questions in an Access question database that I have built?

Hi there.  I have a database of test questions.  I would like to take random samples of this database to build tests (in the form of a report based on my table of questions). It seems like this should involve the RND() function but I'm not sure how to link it to my report.  Does anyone have any ideas?  Thank you.
john w. vinson say:

You can use the Top Values property of a query, with help
from a little VBA. Put this little function into a Module:

 

Public Function RndNum(vIgnore As Variant) As Double
Static bRnd As Boolean
If Not bRnd Then
  'Initialize the random number generator once only
   bRnd = True
   Randomize
End If
RndNum = Rnd()
End Function

 

Then add a calculated field to your Query by typing

Shuffle: RndNum([fieldname])

in a vacant Field cell, where [fieldname] is any field in
your table - this forces Access to give a different random
number for each record.

Sort the query by Shuffle, and set its Top Values property
to the number of records you want to see.

a. user say:

Hey Satyam,

If you want to transfer the SOA to Local AD for a user object who's status shows up as "In Cloud" in Office365 Admin center --> Users --> Active Users, which has a different UPN than Local AD.

I understand that this user has 'UPN' set as "*** Email address is removed for privacy ***" in LOCAL AD and 'UPN' and 'Primary Email' in WAAD are same as '*** Email address is removed for privacy ***'.

As per this situation, if you want to integrate LOCAL AD, then the UPN in Local AD should match UPN in WAAD. Primary SMTP Address could be different. (You can keep the same Primary Email address '*** Email address is removed for privacy *** )

1. Change the UPN in WAAD for this user from '*** Email address is removed for privacy ***' to '*** Email address is removed for privacy ***' Use "Windows Azure Module for Windows Powershell" to achieve this.

    Set-MsolUserPrincipalName -UserPrincipalName "'*** Email address is removed for privacy ***" -NewUserPrincipalName '*** Email address is removed for privacy ***'

2. Collect the ObjectGUID for the On Premise User Object .

3. Convert ObjectGUID to ImmutableID and stamp that to "In Cloud" User Object.

   Get-MsolUser -UserPrincipalName "*** Email address is removed for privacy ***"  -ImmutableId "Converted ObjectID"

4. Set up "AAD Connect" with the "Anchor Tag" as "ObjectGuid" and run the "Full Import, Full Sync on the LOCAL AD Connector and 'Export Cycle' on the WAAD connector.

Once the changes would be complete, you would see that the status of the user would convert to "Synced With Active Directory"

The process mentioned above would change the "Primary SMTP Address" for the user as well. So now you would need to change it back....

1. Open adsiedit.msc.

2. Navigate to the OU where this user object is lying.

3. Right-click on the user object '*** Email address is removed for privacy ***', and then click Properties.

4. Select proxyAddresses, and then click Edit.

5. Type SMTP:'*** Email address is removed for privacy ***, and then click Add.

6. Click OK, and then click OK again.

       7. Close ADSI Edit.

Run the DIRSYCN cycle again. Now you would see UPN for the user as '*** Email address is removed for privacy ***' and Primary E-Mail Address" as ''*** Email address is removed for privacy ***' and the status would be "Synced with Active Directory".

Please click 'Yes' if it worked.

You need to make sure the import specifications exists in your database, using the following query to check it.

 

SELECT MSysIMEXSpecs.SpecName, MSysIMEXColumns.FieldName, MSysIMEXColumns.Start, MSysIMEXColumns.Width, MSysIMEXColumns.SkipColumn

ORDER BY MSysIMEXSpecs.SpecName, MSysIMEXColumns.Start, MSysIMEXColumns.Width;

1.Import once manually to a new table

stefan blom say:

Make sure that both Spacing Before and Spacing After are set to zero (and not to "Auto" or something else) and that Line Spacing is set to "Single." That should do the trick.

Note, however, that with some fonts and especially at some (large) point sizes, you can experiment with Line Spacing set to "Multiple," specifying a value smaller than 1. Or you can use "Exactly." Most likely, a bit of trial and error will be required to find the optimal value.

Of course, you should also make sure that there are no blank paragraphs between text paragraphs. Click the ¶ button on the Home tab to show nonprinting marks.
