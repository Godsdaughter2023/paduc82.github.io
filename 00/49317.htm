Windows file search stopped working, causing freezes - Microsoft Community

sre.bmi say:

Windows file search stopped working, causing freezes

This weekend, Windows search just stopped working.

 

If I go to the start page (Windows 8) and start typing, I'll get results for Apps and Settings but not for Files. It can't find any files, on any drive. If I click on the "Files" suboption while running the search (or even after the search ends), I get some spinning circles and the start menu becomes non-responsive. I have to either reboot or log out to get control of my computer back.

 

Again, this feature worked perfectly until this weekend. The only thing that I can think of that I did was maybe install some Windows Updates.

 

I've tried a bunch of stuff to fix this and nothing has worked:

 

  • Disabled and reenabled the search service
  • Check my indexing options and locations - all looks good
  • Rebuilt my index - done this at least 5 times now
  • Enable and disable from the registry
  • Reset the start page in the registry
  • Ran sfc /scannow - nothing found
  • I ran the indexing troubleshooter wizard
  • I ran the MS automatic search fix app from their website

File search is completely broken from both the start page and the charms.

 

I did try opening My Documents in Exlporer and did a search from there and it *did* successfully find the file. Of course that's just doing a file system scan and not using the search service.

 

What else can I do to fix this? I really need to get this feature working because it's something I use often!

 

Any help would be greatly appreciated.

 

sre.bmi say:

Well, 4 days of troubleshooting and I've got it figured out and fixed. So if anyone else runs into this problem, here's the info below.

 

The Symptom:

 

Doing a file search from the Search Charm or Start Page in Windows 8 return 0 results. Clicking on Files search sub-option at any time causes explorer to freeze up.

 

The Cause:

 

Last week, WHS v1 was reporting that my System disk was unhealthy. Drive info showed that they were not failing, however. I told WHS to repair the drive and it did and the drive was healthy again. However, in doing so it appears to have messed up or obsoleted the Windows file index used by Windows Search.

Since Power Pack 2, WHS has had Windows Search 4.0. The thing about Windows Search 4.0 is that it works with the clients connected to the WHS machine and essentially shares the index.

So basically, I run a file search on my Windows 8 machine (the client) and it goes and asks WHS if it has that file too, WHS either returns jibberish or some error, and Search on Windows 8 aborts and return 0 results.

 

The Solution:

 

The fix was easy. I told the WHS machine to rebuild its index file. For good measure, I told the Win 8 machine to rebuild it's index too, but I don't believe that that part was necessary.

 

TLDR:

 

4 days of troubleshooting and all it took was 1 button press (rebuild index on WHS) and a 30 minute wait to fix it.

When this happens, do the following:

Boot from the Windows 10 install media

a) At the Install Now screen

b) Click Repair your computer

c) Click Advanced Options

d) Click Troubleshoot

Click the Command Prompt

type the following commands the hit Enter after each:

  • DISKPART
  • list volume

Next you will select the volume where you would like to install Windows 7

  • Type: select disk #

# represents the number the volume

  • Type: clean
  • Type: convert mbr
  • create partition primary
  • Type: active
  • Type: format quick fs=ntfs
  • Type: exit
  • Type: exit

Close the command prompt, restart your computer then restart setup again then proceed with the installation.
