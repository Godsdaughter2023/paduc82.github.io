You can't do that at the FIELD level but you can at the TABLE level.  To get to the table level validation rule area open the table in design view, and do NOT click on a field.  Then in the PROPERTIES dialog look for VALIDATION RULE (the tab for the properties should say GENERAL) and put in just the field names:

 

[SellPrice] > = [CostPrice]

marshall barton say:

If the validation rule is in the table's validation rule, not a field's rule, try it without the table name.

 

A field's validation rule can only refer to that one field and should not include the field name (e.g.  >100) so this won't do what you want.

 

If the rule is in a form control's ValidationRule property, it's like in a field's rule.  So, move it to the table's rule or, alternatively, use the form's BeforeUpdate event procedure instead of a validation rule (set the Cancel argument to True if the validation fails).

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

c) Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

d) Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

e) Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and check if the above steps resolve the issue.
