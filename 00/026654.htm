over 15 digit number is not showing properly in excel - Microsoft Community

sanjoy kr paul say:

over 15 digit number is not showing properly in excel

1234567891234560000
joeu2004 say: Sanjoy Kr Paul wrote:
Subject: over 15 digit number is not showing properly in excel
1234567891234560000

Correct.  This is a limitation of Excel.  On data entry, Excel will interpret only up to the first 15 significiant digits, replacing any more to the right with zeros, effectively truncating the number.  Also, Excel will display only up to the first 15 significant digits, rounding the 16th digit if necessary.

If the 19-digit "number" is really a numeric id -- for example, an account number -- you should enter it as text.  How you do that depends on the source of the data entry.  If you are entering it manually, you can prefix the id with an apostrophe (single quote); for example, '1234567891234560000.

If the 19-digit number is truly intended to be used as a number, you are SOL.

However, your example contains only 15 significant digits.

If your concern is only the appearance -- 1.23457E+18 by default -- that is easy to remedy.  Simply change the format from General to some other numeric format, for example Number with 0 decimal places.  Select the cell(s) or column, right-click and click on Format Cells, then the Number tab, and click on the Number and enter 0 (zero) in the Decimal Places field.

OK, I figured out what was causing the problem.  I downloaded the file and noticed that in a few of the cells there was a small square among the text.  When the browser would go to load those cells for preview or editing, it would cause the error.  So, if the cell with the square was far enough down on the list that it didn't load immediately, the page would load just fine.  Once you scrolled down or over and that cell was loaded, the error would occur.

a. user say:

Hi Eric,

Thank you for your interest in Windows 10 Technical preview.

This issue occurs if the following scenarios are true:

  • Windows Update or Microsoft Update tries to install a driver a second time for an additional piece of identical hardware such as a graphics adapter.
  • Windows Update or Microsoft Update tries to update a currently installed driver to an unsuitable version of that driver.

I suggest you to follow the methods below and check if it helps.

Method 1: Run Windows Update Troubleshooter

I suggest you to run the Windows Update Troubleshooter. Follow these steps to run Windows Update troubleshooter:

  1. Press Windows key + W and type troubleshooting.
  2. Select Troubleshooting.
  3. Select View all on the top left corner.
  4. Click Windows Update troubleshooter.
  5. Follow the on-screen instructions to run the troubleshooter.
  6. Check if issue persists.

Method 2

I suggest you to check if the services responsible for Windows update is running. Refer to these steps:

  1. Press Windows key +R, type “services.msc”.
  2. Locate the BITS, Cryptographic, MSI Installer and the Windows Update Services and check if it is running.
  3. If not, right click on the service and click start to start the service.

Method 3

Place the computer in clean boot and then try installing updates.

How to perform a clean boot in Windows (Applies to Windows 10 Technical preview)
http://support.microsoft.com/kb/929135/en-us

Note: After clean boot troubleshooting steps, refer to the section "How to reset the computer to start as usual after troubleshooting with clean boot" in the provided link to return your computer to Normal startup mode.

Hi,

I suggest you to manually reset the Windows Updates Components. Refer to the below steps to reset the Windows Updates Components:

  1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.
  2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

   3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

   4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

   5. Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and check if the above steps resolve the issue.

For further reference, you can refer to the link:

How do I reset Windows Update components?
