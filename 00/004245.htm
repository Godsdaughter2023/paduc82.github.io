bob jones aka: cybertaz say:
Assuming that this statement is accurate: "...lost the file as it had never been saved." I'm afraid there is nothing to be recovered or "get back". Your description suggests that Excel was used independently to enter data into a workbook for the purpose of creating the charts, but that the workbook was not saved. Until the work is saved [written to disk] there actually is no
When you copy a chart, the data on which it is based does not get copied along with it. The source is identified as a part of the Linking operation, but if that source no longer exists it can't located so it can't be retrieved. That's why you're getting the message when you attempt to access the data. The PPT file was able to recover the charts - unsaved changes made in the existing PPT file - because of the AutoRecover feature. But because the Excel workbook containing the data was never saved even AutoRecover can provide no help.

Regrettably, all I can do is offer some advice for the future :-)

  • Use the Insert> Chart feature within PPT. Excel will still be involved for entering the data but the worksheet is stored within the PPT file rather than in a separate Excel workbook. This is probably the better method if it's just a matter of a single chart based on that specific data.
  • If you need to generate several charts based on the same data or use the same data to create charts in a variety of other documents, enter the data directly into an Excel Workbook but be sure to for each chart, so it usually isn't the better way if there are several charts to be displayed in the presentation.
