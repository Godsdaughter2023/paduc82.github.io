Yes.   The point is that that changes the User-Agent string used by the requests you make to the site to be more similar to ones which would be made by earlier browsers.   In fact, by using the IE9 Developer Tools (F12) there are ways that you could test other UAS, not just IE ones.

Also, it may help to go to this test site and see what it says about your User-Agent string, in both modes:

http://www.fiddlertool.com/ua.aspx

 

More generally, the messages that you are seeing are often misleading; so it may help to think of them more as "The site I am trying to connect to doesn't like the way that I am making a request..." rather than what the message is saying to you explicitly.   ;  )


Another IE9 Developer tool you could try to help you figure out what is wrong is its Network Capture facility.   To use it effectively it would probably be best for you to have two otherwise identical traces to compare, e.g. one of the request working and one of it failing.   That way you don't really have to understand what you are looking at; you can just compare them for their essential differences.

Typically  2501 is when the user cancelled out of an action and you didn't code for that.

marshall barton say:

The usual reason is because something (the form?) canceled its Open event.

The first thing to check is if the OpenForm line is the one that generates the error.

Assuming it is, then look at the form's Open/Load event procedures to see if there is any code/macro that prevents the form from opening.

The use of breakpoints on the line before and the OpenForm line as well as in the form's Open and Load event procedures usually helps identify where things went wrong.

marshall barton say:

If that fixed it, then I can think of only two possible reasons for it not working before.

One would be that somehow an invisible character in some strange way was entered when you typed the line.  This doesn't happen often, but I have seen it happen on rare occasions.

The other reason is your form, or more likely its VBA module was corrupted.  There's no explaining how it happens, but the first thing you need to do when it seems like something might be corrupted is to make a backup copy of the database in case it gets worse.  Because no one can recognize corruption in all of its manifestations, making backup copies frequently is a very good habit to have.
