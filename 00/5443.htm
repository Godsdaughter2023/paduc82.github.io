When you try to open an embedded file, Word simply hands off to the Operating System, which sends the call to whichever application is registered as being able to handle that file.

0 people found this helpful

hansv mvp say:

This is not a bug; it is a consequence of the way Excel handles numbers. Numbers are only accurate to 15 significant digits.

POWER(10,17) = 100000000000000000

POWER(10,2) = 100

Strictly mathematically, their sum is 100000000000000100

The second 1 is the 16th digit, so it is outside the accuracy that Excel offers. The 1 is ignored.

If you calculate POWER(10,17)+6*POWER(10,2), the mathematical result is 100000000000000600. The 6 is rounded up by Excel so it returns 100000000000001000. The second 1 is the 15th digit, so it falls within Excel's accuracy, if only barely.

bob jones aka: cybertaz say:

I doubt that the files, themselves, are hidden :-) If the files are opening automatically [When you launch Excel?] the 3 most likely reasons are;

  • The files are stored in the ~:UserName:Library:Application Support:Microsoft:Office:Excel folder. The Library folder for your User Account may be hidden if you're using OS X 10.7 or later. See this Apple KB on how to access it. Quit Excel, then move the files to a different location, or
  • The files are stored in ~:Applications:Microsoft Office 2011:Office:Startup:Excel which is not a hidden folder. Again, Quit Excel & move the files to another location, or

It's also possible that your version of OS X is responsible for opening files automatically based on your settings in System Preferences> General.

Note, however, that the Personal Macro Workbook is intended to open on launch in order for the macros it contains to be available. That's why it is stored in the location indicated in the first bullet point above.

First of all, is Project fully updated? It should be version 14.0.7197.5000 (April 2018 update). If not, run Windows Update.

Hope this helps.
