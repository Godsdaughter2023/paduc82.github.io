suzanne s. barnhill say:

In Word 2010 (and 2013) the shortcut Ctrl+F is assigned by default to NavPaneSearch. You can reassign it to EditFind, as follows:

  1. Right-click on the Quick Access Toolbar or Ribbon and select Customize the Ribbon.
  2. In the Customize the Ribbon and keyboard shortcuts dialog, beside "Keyboard shortcuts" (lower left corner), click Customize...
  3. In the Customize Keyboard dialog, select All Commands under Categories.
  4. Scroll down to NavPaneSearch and select it.
  5. Select Ctrl+F in the Current keys box and click Remove.
  6. Now scroll to EditFind and select it.
  7. In the Press new shortcut key box, press Ctrl+F (dont' type "Ctrl," just press the Ctrl key).
  8. Click Assign, then Close.

I would make two suggestions:

  1. You may want to assign a different keyboard shortcut to either NavPaneSearch or NavPane because otherwise opening the pane requires selecting the View tab and checking the box for it, which is cumbersome.
  2. Instead of reassigning Ctrl+F, you may want to take the alternative route of just using Ctrl+H instead. This opens the Replace dialog, in which you can then select the Find tab.

daniel g. grau say:
It looks like macro virus code that did not fully complete execution due to code incompatibility, often a reference to a Windows disk "C:\". The code has most probably auto-propagated to the Normal template (that's the first thing they do). The second thing they do is disable macro protection, but you still had that on, so again, virus execution did not complete. Try this:
  
  • Quit Word, press Shift and launch it through the dock, not by double-clicking on a document. This will temporarily bypass the Normal template (as well as your Preferences) and will make Word ignore any macros stored there.
  • If no messages are shown, it means the regular Normal.dotm had macro code. Quit Word and move your contaminated Normal.dotm to the desktop. To find it:
    • Go to the Finder
    • Select Go > Go to Folder…
    • Type ~/Library (don't forget the twiddle, top-left key)
    • Navigate to Application Support/Microsoft/Office/User Templates
  • Launch Word, again from the dock, but this time don't press Shift. Since Word won't find any Normal.dotm file, it will offer to create a new one upon quitting. When asked, do so.
  • Make sure you leave macro protection active (Word > Preferences > Security  > Warn before opening...). From that point on, never enable macros for a document you open unless you know they should be there.

If you have any customizations in the old Normal.dotm template (like your own toolbars, menu options you've added, repeating headers and footers for new documents, language settings, keyboard shortcuts and the like), you can later copy them through the Organizer. (Of course, you'll be required to open the old Normal.dotm, and you'll get a macro warning. If you need to do that, post back.)


To clean up your contaminated files (that would be every Word file you opened since the problem started), you can do this manually by Saving As. This is viable if you only have to disinfect a couple of documents:

  • Open your documents (any way you want), but refuse the macros through the Disable macros button (macro proteccion is still enabled, of course). If you misclick, you'll have to start the whole thing over.
  • For each document, do a Save As. If the document's current extension is .doc (or .dot), save as "Word 97-2004 Document (.doc)." In all other cases, save it as "Word Document (.docx)." You will get a warning that all macros will be lost, which is what you want.

RTF files are immune because they cannot store macros, so don't bother with those.

There is another place macros can lurk, and that is in templates stored in /Applications/Microsoft Office 2011/Office/Startup/Word. Make sure the folder does not contain anything you did not place there. (In order to confuse the user, virus macros may change the default extension in templates to .doc or .docx or .txt.)

If you have tens of files all over your disk, download this free antivirus program:
  • Sophos

Your best protection from now on is to heed the macro warning.

swischner say:
.onebin files belong to the OneNote local cache (the copy of your notes that OneNote is actually applying edits to before they get synchronized to the "real" version on your drive or in the cloud).

Only OneNote itself can make use of .onebin files, You can't open them directly and copying, moving or backing them up is of no use. They are in a very special split up, binary format.

The file format OneNote is using for notebook data is .ONE (these are actual sections) and ONEPKG (a ZIP-like archive file for backup and transports).

Note: Only OneNote for Windows that comes with MS Office (OneNote 2010, 2013, 2016, but not the free standalone version) can open files (like .ONE) directly at all. All other OneNote clients (Win 10 app, Android, iOS, Mac..) need the notebooks to be stored in a special hidden format in the cloud and cannot handle files directly at all.

To show both the horizontal and vertical rulers, do the following: Make sure that the document is showing in Print Layout view (View tab | Print Layout). On the View tab, check the box for "Ruler."
