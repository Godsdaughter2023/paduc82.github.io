albert d. kallal say:

Perhaps you have access in tabbed mode as opposed to overlapping windows? However, if this is the case, then you would have never been able to size forms except for dialog ones. Remember, in the context of tabbed mode, then your forms ALWAYS take on the size of the access application window.

 

So, check if your running in tabbed mode. Office button->access options (lower right), then current database, you see a choice for tabbed, or overlapping windows.

 

Remember, in tabed mode your forms always take on the size of the whole application window. So, the form sizing is rather loose in this regards. Some however like this tabbed mode since the bonus is then the gray background behind forms can NEVER been seen. In fact, you can use this to hide the access UI. You will thus JUST see your form only when you do this.

 

However, when you use tabbed mode (and it is the default for 2007/2010), then you always see the form sized to the same as the application (Access) window size in view mode, and you don't have a chance/choice to size the window.

 

You realistically can't size a form in design mode without seeing and ascertaining how much of the borders etc and the rest of the form will look when you put in in normal mode, this has always been the case, even for quite older versions of access.

 

However, I suspecting you are using tabbed document mode?

 

So, check the "mode", and then also ensure that border is set to sizeable (at least until you done sizing, and then flip it back to thin if you desire to remove sizing ability from your users AND assuming you are using overlapping windows mode).

rob koch say:

As long as the Startup Type selection is "Manual", the Status of the WdNisSvc service should currently be "running" and can't be disabled, as I believe this service is self-protecting as is the WinDefend service.

The way that WdNisSvc works is that it always runs during system startup to determine whether there are any known new network vulnerabilities which have not yet been patched on this particular system.  If so, the service will automatically load the specific network monitoring and protection modules which apply for the particular vulnerability and the WdNisSvc will remain operating.  If instead the WdNisSvc initialization routines determine that there are no currently known vulnerabilities in its definitions which remain unpatched they will typically shutdown the service since it's not needed.

Since at the present time there is a new zero-day vulnerability within the version of Adobe Flash which is embedded in the current version of Internet Explorer 11, it is likely that this is why the WdNisSvc remains running.

Though a patch for Internet Explorer 11 Flash was already released late last week, this was for another zero-day vulnerability that was also discovered last week.  Once the new patch for Adobe Flash in IE11 is released, expected later this week, the WdNisSvc should once again be disabled shortly after system startup is completed.

Rob

Function ConcatenateA(ByVal Delimiter As String, ParamArray Args()) As String
  'Returns a string with each item in Args seperated by Delimiter
  '  Each item can be a cell, a range, a value or an array
  '  Each item is trimmed, empty Args are ignored
  '  Example: =ConcatenateA("; ",A1,B1:C3,"x",{1,2})
  Dim All As New Collection
  Dim Item, Data, Result() As String
  Dim i As Long
  Dim R As Range
  For i = 0 To UBound(Args)
    If IsObject(Args(i)) Then
      Set R = Intersect(Args(i), Args(i).Parent.UsedRange)
      If Not R Is Nothing Then
        For Each Item In R
          Item = Trim$(Item.Text)
          If Len(Item) > 0 Then All.Add Item
        Next
      End If
    Else
      Data = Args(i)
      If IsArray(Data) Then
        For Each Item In Data
          Item = Trim$(Item)
          If Len(Item) > 0 Then All.Add Item
        Next
      Else
        Data = Trim$(Data)
        If Len(Data) > 0 Then All.Add Data
      End If
    End If
  Next
  If All.Count = 0 Then Exit Function
  ReDim Result(1 To All.Count) As String
  i = 0
  For Each Item In All
    i = i + 1
    Result(i) = Item
  Next
  ConcatenateA = Join(Result, Delimiter)
End Function
