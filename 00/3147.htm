Without any JOIN, this sets up what's called a Cartesian product of table1 * table2 records. Are you really, really sure that's what you intended to do? All Cartesian products are read-only by design.

dhookom say:

SELECT F.*, F.Revenues_Gov+F.Revenues_OSR AS TotalRevenue, 
DLookup("CommunityRevenueCategory", "CommunityRevenueSize", "Revenues_Gov+Revenues_OSR BETWEEN " & LowerValue & " AND " & UpperValue)
FROM FirstNational_Community_Info F

ken sheridan say:

You might be able to return an updatable result set by using an INNER JOIN:

SELECT FirstNational_Community_Info.*, [FirstNational_Community_Info]![Revenues_Gov]+[FirstNational_Community_Info].[Revenues_OSR] AS TotalRevenue, CommunityRevenueSize.CommunityRevenueCategory
FROM CommunityRevenueSize  INNER JOIN FirstNational_Community_Info ON (([FirstNational_Community_Info].[Revenues_Gov]+[FirstNational_Community_Info].[Revenues_OSR])
BETWEEN [CommunityRevenueSize].[LowerValue] AND [CommunityRevenueSize].[UpperValue])
ORDER BY FirstNational_Community_Info.FirstNationalID;

Parenthesizing a BETWEEN....AND operation in a JOIN clause will normally work.  Your query is unusual in that the first operand of the operation is the result of the addition of two values, so it is probably prudent to parenthesize the addition operation also.  I'm not sure if this will work, though, but it's worth a try.

Note the use of the dot operator rather than the exclamation mark when qualifying a column with the table name in SQL.
