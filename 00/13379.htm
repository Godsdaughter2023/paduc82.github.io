ken sheridan say: Apparently the value in the control bound to the date column is being overwritten, probably with a Null, when a new record is initiated, so you need to identify the cause of this.  I have to take issue with Tom on this point, however:

'it is normal that a default value shows up only after a new record is created (typically by entering the first character in the first field), and the form_beforeinsert event runs'

The DefaultValue property of a control inserts a value in to the control as soon as the form is moved to an empty new record and it shows up immediately.  It does not initiate a new record, however; that only happens when a value is assigned to a control or to a column in the form's recordset, whether by the user manually entering data or by code in the form's module, e.g. in its Current event procedure, or elsewhere.

Another thing you need to be aware of with the regard to the DefaultValue property is that it is always a string expression, regardless of the data type of the column to which a control is bound.  Consequently when assigning a value to the property the value should be wrapped in literal quotes characters.  This is particularly important with dates, which can otherwise be interpreted as an arithmetical expression.  This regularly confuses people when they try and carry forward a date from one record to the next new one and find they get a default date/time value somewhere around the start of the 20th century (this results from Access's use of 30 December 1899 00:00:00 as the origin for its implementation of the date/time data type).  In your case, by referencing the control in the header you should not have this problem, but if you were to assign the value to the property in the AfterUpdate event procedure of the control (my normal approach) with:

Me.YourDateControl.DefaultValue = Me.defaultdatestat

you would almost certainly encounter this behaviour, depending on the format in which the date is entered in the control.  The correct syntax would be:

Me.YourDateControl.DefaultValue = """" & Me.defaultdatestat & """"

BTW do not be tempted to use the date delimiter:

Me.YourDateControl.DefaultValue = "#" & Me.defaultdatestat & "#"

This will fail on systems such as mine which do not use the US short date format.

Function Getlnkpath(ByVal Lnk As String)
    On Error Resume Next

doug robbins - mvp office apps & services (word) say:

Use a macro containing the following code:

 

Dim i As Long
With Selection.Tables(1)
    For i = 1 To .Rows.Count
        With .Rows(i).Shading
            If .BackgroundPatternColor <> wdColorAutomatic Then
                .BackgroundPatternColor = wdColorAutomatic
            End If
        End With
    Next i
End With
