brian tillman [outlook mvp 2007-2019] say:

Technically, if you read the email specification, you'd see that what comes in front of the @ in an address is case-sensitive. ABC at domain and abc at domain are supposed to be different mailboxes. However, in practice, most mail systems treat upper and lower case as the same. So, it boils down to whether or not you can control it. I don't believe that's possible because any mail system in the delivery path can do what it wants and you don't have any control over other mail systems.

So, my advice is to "fuggeddabotit". It's not worth the worry.

a. user say:

AFAIK this is because

 a. Word takes your filter conditions and generates an SQL query from the

 b. someone at MS has decided that whenever you have a <>, the SQL that is generated from the filtering dialog box has also to check for "is null."

 c. so your filter conditions end up like this:

SELECT * FROM `Sheet1$`  WHERE `Invoice` > 0 And (`Paid` <> 'paid' OR `Paid` IS NULL ) And (`Paid` <> 'canx' OR `Paid` IS NULL ) And (`Agreed` <> 'agreed' OR `Agreed` IS NULL ) And (`Multi` <> 'yes' OR `Multi` IS NULL )

 d. and when you reopen your filter dialog box, that lot is translated back, but there is only space for 6 conditions so most of your query disappears.

AFAIK there are two main possible workarounds, both rather nasty:

 e. connect using DDE rather than OLE DB. (In Word 2010, check FIle->Options->Advanced->General->Confirm file format conversion on open, go through the connection process again, check the "Show all" box in the additional dialog box if you can't see a DDE option in there. But it doesn't always work.

 f. always specify your query directly in Word VBA, and never change the criteria using the filter dialog box. You can use e.g.

Sub Conect2XL()

Dim strWorkBookFullName As String

Dim strWorkSheetName As String

' set to the full pathname of your Excel workbook, e.g.

strWorkBookFullName = "c:\mywbs\mywb.xlsx"

' set to the name of your Worksheet, followed by a $,  e.g.

strWorkSheetName = "Sheet1$"


  ActiveDocument.MailMerge.OpenDataSource _

    Name:=strWorkBookFullName, _

    SQLStatement:= _

      " SELECT * FROM [" & strWorkSheetName & "]" & _

      " WHERE [Invoice] > 0" & _

      " AND [Paid] <> 'paid'" & _

      " AND [Paid] <> 'canx'" & _

      " AND [Paid] <> 'paid'" & _

      " AND [Agreed] <> 'Agreed'" & _
      " AND [Multi] <> 'yes'" 

 

End Sub

NB, that SQL statement may need some tweaking.

If you aren't familiar with installing/running VBA macros etc., Graham Mayor's site has a useful page at:

http://www.gmayor.com/installing_macro.htm
