doug robbins - mvp office apps & services (word) say:

You would need to run a macro on exit from the formfield in which you enter the start date that uses the DateAdd() command to increment the date that is the .Result of that formfield and use the result of the command to set the .Result of the other formfields

Say there are 6 formfields that you want to populate and that the formfields have bookmark names of Date1, Date2, Date3, etc and you enter the start date into the Date1 formfield, if you run macro containing the following code, it will populate the Date2, Date3, etc. formfields with the relevant dates when you exit from the Date1 formfield

Dim i As Long
With ActiveDocument
    For i = 2 To 7
        .FormFields("Date" & i).Result = DateAdd("d", i - 1, .FormFields("Date1").Result)
    Next i
End With

There's no reason to ask this specifically about SCEP, since though it's been improved over time, the operation of MAPS isn't any different on that product from any of the other Microsoft security products which have contained this since they acquired the technology from Giant AntiSpyware (then called SpyNet) back in late 2004.

This has always been covered fairly completely by the Microsoft Privacy Policy relating to its security products, specifically Windows Defender in this case.

MAPS in the cloud How can it help your enterprise - Microsoft Secure

"The data gathered is treated with confidentiality. See the Microsoft Privacy Statement for details. To help protect your privacy, reports are sent to Microsoft over an encrypted connection. Relevant data is analyzed."

Within the huge Privacy Statement, look under the Product-specific details section on the left for Windows, Security and Safety features, which will take you to a section including Windows Defender, from which I've included the excerpt below.  Bolding is mine to highlight the important information.

"

Windows Defender Antivirus. Windows Defender Antivirus looks for malware and other unwanted software on your device. Windows Defender Antivirus is automatically turned on to help protect your device if no other antimalware software is actively protecting your device. If Windows Defender Antivirus is turned on, it will monitor the security status of your device. When Windows Defender Antivirus is turned on, or is running because Limited Periodic Scanning is enabled, it will automatically send reports to Microsoft that contain data about suspected malware and other unwanted software, and it may also send files that could contain malware. If a report is likely to contain personal data, the report is not sent automatically, and you'll be prompted before it is sent. You can configure Windows Defender Antivirus not to send reports and suspected malware to Microsoft.

As for how that data is handled, MAPS and the Cloud protection have always been completely automated, which performs most of the analysis, collation and review of submitted, no matter where they come from.

What this means is that the "team" that Cyber mentions rarely includes an actual human, since the artificial intelligence systems that Microsoft has developed can do most of this far faster and better than even hundreds of humans working together.  So it's only in extremely rare cases that a human might actually view even a single file, typically only when some significantly new, unknown malware is suspected by the machines and automated tools that these are not certain how to handle.

To better understand how the above works, see the following document, especially the section titled; Machine learning, behavioral analysis, and other evolutions.

Antivirus evolved - Microsoft Secure

What all of this should show clearly is that even in the unlikely event that Defender's Cloud protection might actually request a file that contains personal data, the file is not only promised by Microsoft to be safely handled, but it will simply be buried in massive automated systems along with millions of others, which in most cases no human will ever examine or even be allowed to access.

Realize that malware is an executable, so even in those few cases where data files like those for Office or an Adobe PDF might contain macros, the chances that one of these might become modified while in your personal possession to contain a malicious macro or other code is so tiny as to be infinitesimal.

In other words, you're worrying about a situation so unlikely that it's a pointless concern.
