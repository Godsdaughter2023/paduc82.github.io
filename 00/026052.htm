The calendar view is limited as far as both how many tasks can appear as well as formatting.  If you are interested in focusing on a particular service - you could use a filter to show only those tasks for the particular service.

Regarding your first question:  Not directly in the calendar view - however, if you are using the Professional Version of Project 2010, the Team planner view shows the resources as a column on the side with their tasks on a Gantt-like view.

Again, the calendar view can be filtered to show only a particular resource's tasks.

Automatic scheduling allows you to create relationships (predecessors and successors) in the project file.  If Task B cannot start until Task A is complete, you can create a Finish to Start relationship between A and B.  If Task A takes less time than planned  and the finish of A is changed, then Task B's start is automatically adjusted earlier.

Regarding color coding - I don't know whether you wish to code the bars or the name of the task.  If you'd like the name of the task - add the Marked field and set one of the services to "yes". Creating an MD5 or SHA1 in Excel or other office applications (This is the best method)

After looking around on the web at various solutions to creating a checksum/hash in excel that is either

I have made this so simple using the WEBSERVICE and CONCATENATE functions in excel to produce the result I was after.

Firstly I created an API that anyone is welcome to use it is at https://hashing.labs.coop or http://hashing.labs.coop

Hi,

I have discovered another situation that causes queries not to show up.

I had already embedded a Union type query within a Select query to hide it when trying to link into Excel. I still couldn't see it. When I went via Data > From Other Sources > From MS Query I got an Error "Undefined function NZ() in expression".

It turns out (in this case at least) that the embedded use of NZ is the cause of the problem as it cannot be referred to in an ODBC connection. I had NZ() in multiple sub-queries which eventually were UNIONED and then hidden in a simple Select query.

By substituting iif([Column_name] is null, "",[Column_name]) wherever I had used NZ() in all of the sub-queries, I was able to see my original query via the more normal Data > From Access.

myall_blues say:

Hi

Not necessarily an answer for you, but perhaps some info to get you started. As I read it you need 3 teams a day (of 4 players) over 4 days = 12 teams total, and I assume you don't need to match handicaps and so forth.

This is a combinations problem - you have 12 players and need teams of 4. There are then 12!/(8! * 4!) = (12*11*10*9)/(4*3*2*1) = 495 possible ways to make teams of 4 from 12.

(You can get this number in excel, by the way, using the COMBIN function; =COMBIN(12,4))

So as a minimum you could generate these 495 possible team combinations and then pick out 12 that suit you. Generating the 495 is really a job for a macro - you can find code at this link http://excel.tips.net/T006766_Listing_Combinations.html

Use the first code example that uses numbers - the second example using letters appears not to work. If you change the 10 to 12 in this part of the code 

Sub TestCNR()
    Cnr 10, 4
End Sub

and run it, it will generate the 495 sets for you.

If you need help on how to install and run a macro, post back.
