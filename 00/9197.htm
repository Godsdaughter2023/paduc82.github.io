Select your country from the drop-down list.

If you entered the code correctly, you will see a confirmation message, and your Caller ID will be activated in 24 hours.

To set up caller identification using your Skype Number:

Sign in to your account.
https://secure.skype.com/account

Click Save Settings. Your caller identification is now set and ready to go.

How long does it take for caller identification to be activated?

Caller identification is activated immediately for Skype Numbers.

It may take up to 24 hours to activate caller identification for mobile numbers. Before the activation, you will receive the SMS text message confirming that caller identification is about to be activated.

scottgem say:
I suspect splitting is only partially the problem. Its not the backend being split that is the problem, its the front end and, more importantly, its the move from a shared FE to individual ones. As John, splitting is a necessity on a multi-user app. Given that many people leave the app open I'm surprised corruption hasn't been more rampant. 

From what you said, because you have been sharing a single copy of the app, when someone creates an adhoc query and then saving it, that query now becomes available to all users. When you split it and installed local copies, then any queries that one user made would not be available to any other user and, if the FE is updated, their stored queries will be erased. 

So the first solution I see is to split the database, but share the FE among all users. With one FE, queries made will be available to all users.

But that's not an optimal solution, but it might be a temporary one to appease the revolting mid-managers.

The optimal solution, IMHO, is to build an adhoc query functionality. You need to analyze the type of queries they need. Its possible that canned queries with forms to supply criteria will be sufficient. Or you can create a more complex facility where you store the fields the user wants and the criteria in a table. then use that table to build a SQL statement that can then be used to display the data they want.
