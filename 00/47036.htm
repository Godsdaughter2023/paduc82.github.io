Note

If the computer is connected to a network, network policy settings may prevent you from following these steps. Logically, one may think that if a logging service is stopped you'll miss only the log information not the entire feature (and this without any error message, and without any troubleshooter finding anything wrong). The same reasoning applied also to the teams that developed all the other Windows apps/features, as they work very well without this service running. However, if enabled, they will use the service of course, logging their specific messages, and this is what any user would expect (this behavior being for a very long time, since it's inception more than 10 years ago). Not anymore, so it seems. People disable it but only in their home computers, as the information that some app crashed at address 0xfff01ab3 doesn't help very much. 

Windows Search existed in the XP world, but back then it was recommended to be disabled it as it was a CPU hog as the Indexing, and even today they are better off disabled to help SSDs last longer. Now, why would  someone disable such useful features as Windows Search and Indexing and not disable a less useful feature for a home computer as the Event Log?
