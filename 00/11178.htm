graham mayor say:
This undoubtedly relates to port allocations.

SMTP (outging mail server) by default uses port 25. Many internet service providers will restrict the use of port 25 to their own SMTP servers, so you need to ascertain whether the service provider of your 'work' e-mail address has an alternative port available. For example two of the providers I use offer port 587 and a third uses port 2525.. Almost invariable the server will require authentication,

If you tell us who the e-mail service providers are we may be able to provide more information.

graham mayor say:
That's the problem. Your account uses port 25, and that is fine while you are working from home, but when you are working away, you are not connected to your home service provider and port 25 on the host server to which you are connected is reserved for SMTP servers associated with the host server and your 'foreign' server on port 25 is rejected.

Change the SMTP port allocation to 587 as you have been instructed and check the box that indicates the outgoing server requires authentication on the outgoing server tab, and apply the authentication that your supplier says is required (usually the same settings as the incoming server).

As for the service providers, what is the domain name after the @ in your e-mail addresses.

adehad say:

I had a similar issue, but I got a BSOD after about an hour of work. This happened about 5 times,  each time windows recovered faster,  but still a pain.

My workaround was:

I ran it using Windows 8 Compatibility setting (and as Admin).

  1. You will need to find the application under "C:\Program Files (x86)\Microsoft Office\Office16"
  2. Find the office program you want to change (Word is WINWORD)
  3. Right Click - Troubleshoot compatibility
  4. Go through it and choose Windows 8 when they ask for when the program last worked properly
  5. Click Test Program
  6. Go back to the compatibility dialogue box and click next
  7. Ask to save those settings for next time program opened

I have not experiences any crashes as of yet. (Using for about 8 hours straight now with regular saving (and auto saving), while running the same programs that caused the crash before [Skype call, around 20 active tabs])
