keith a. miller say:

I'm afraid this behavior is, as they say, "by design".

It seems that for any file system folder that is also inserted into the Shell namespace (UserName folder on Desktop, User folders under This PC, etc.), the namespace path takes precedence over the file system path.

If you need a folder's file system path, use <Shift>+Right-click => Copy as path.

If you frequents access C:\Users, pin that to Quick Access.

Keith

keith a. miller say:

The namespace structure is also responsible for the folder view discrepancy you are seeing. The index for saved views (HKCU\Software\Classes\Local Settings\Software\Microsoft\Windows\Shell\BagMRU) is a tree structure rooted at the (virtual) Desktop. So it has two views saved. One for:

Desktop\This PC\Documents

and another for:

Desktop\This PC\C:\Users\<UserName>\Documents

as well as all the subfolders. Making use of:

Folder Options => View tab => Apply to Folders

to set your preferred view for each folder type, as well as:

Folder Properties => Customize tab => Also apply this template to all subfolders

will minimize but not eliminate discrepancies.

Keith
