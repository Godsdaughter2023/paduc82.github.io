Can I be given malware or viruses through my ip? - Microsoft Community

a. user say:

Can I be given malware or viruses through my ip?

I recently used a wiki and learned that changes made displays my public ip address. I wasn't aware of this and I don't know much about this so can other than knowing the general city that I live in can somebody give me a virus or infection using my ip address?
rob koch say:

This question can't be answered that simply, so you'll only receive mostly confusing replies of both yes and no depending upon who answers.

This is because everything that any Internet attached device ever does must pass through the IP address number that your Internet Service Provider (ISP) provides to identify the external interface of your router, modem or individual PC to both send and receive whatever data you transfer between your devices and the Internet.

This is what the various security and false security advertising attempts to use to scare you into buying either the security products you do need or those that are instead simply trying to fool you into giving them money for nothing.  In truth, the only difference between these is how reputable the company is and so how valuable whatever they are selling is to your true security.

What you should truly be asking is how to prevent malware of any sort from simply walking right in through either your Internet router, modem and/or PC/device network interface without being monitored, blocked or otherwise challenged in any way.

In this case, the typical first line of defense in a home network is the router, which also usually includes a simple firewall today, whether the router was originally purchased by you or supplied by your ISP.  If this router device is quite old, there have been many significant security problems found during the last dozen years or so that affected many of these, so either investigating these or even just replacing one that's several years old or more might be worth the money and effort.

However, there's no simple way to confirm this without investigation, so if this isn't handled by your ISP you'll probably want to involve someone with more technical knowledge.  Here's an article that discusses many of these potential router security issues, though it's a bit more paranoid than others, but it provides some insight into the concerns regarding configuration and maintenance as well.

Your Router's Security Stinks: Here's How to Fix It

Along with your external router though, you also need to be certain the firewall built into your own PC device is secured as well.  Based upon your selections above, I'm assuming your PC is using Windows 8.1, so the following article discusses several ways that this can be properly configured for the best security.  This assumes, though, that you are not using a 3rd-party firewall, so make sure you're certain of this first.

Network Location - Set to Private or Public in Windows 8

These are the most obvious and important issues regarding security related to your IP and network(s), while the address itself is inherently not secret, but also not something you need to advertise to anyone either.  The fact that any information request that your PC makes will exit through your PC or router's Internet Protocol (IP) address, means that the response from any web or other server will automatically be returned to your IP address as well.

So clearly the computer on the other end of this conversation must know your IP address, but securing and otherwise hiding that fact (commonly called "stealth") from any other computers you haven't directly requested information from is the most typical and best security configuration that can exist.

Word doc's I send as email attach cannot be opened by recipients - - Microsoft Community

a. user say:

Word doc's I send as email attach cannot be opened by recipients - any ideas?

Any help would be appreciated re the above question. I have changed settings to 97-2004 x tried dragging from desktop, still unable to open files.
john mcghie say:
The MOST important thing you can do is "Don't tell them you are on a Mac!".

There is no difference between the file sent from Word 2008 and the file sent from Word 2007 on the PC.  None at all: exactly the same inside.

The reason they can't open them is because they have their email set up wrong at the other end.

You should tell them that, and ask them to contact their systems person.  Otherwise you will never survive this...

There are things you can do to assist the process.  The first is to send .docx files, with the file name extension correct.  Windows does not need extensions so much these days, but your Mac does, as do all the mail servers in between.  The extension tells the receiving computer what's in the file.  A badly-set up computer doesn't know what to do next if it receives a file without an extension.

The next thing you can do is to Zip the file before emailing it.  This SHOULDN'T make any difference, but with a really bad mail client, it fools it long enough for the user at the other end to get the attachment, save it to their desktop, and open it.

Tell the other users NOT to try to open files by double-clicking in their email.  As Windows users they shouldn't be doing that anyway, because it practically guarantees they will get viruses.  However, if they SAVE the attachment to their computer, then use File>Open from inside Word, the file will almost always open, whether it has an extension or not.  Once Word gets hold of it, it will figure it out: if the extension is missing, the file never gets sent to Word, which is why it doesn't work.

Also:  Give up on web-mail.  Various kinds of web mail do strange things to attachments: used on a Mac, some of the free webmail clients prevent attachments going out at all.

If you have Word, you have Mail and Entourage on the computer: set one of them up to handle your email and things will be a lot more reliable for you.

Hope this helps
