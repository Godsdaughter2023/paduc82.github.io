How to view members of a group in Outlook - Microsoft Community

christinebot say:

How to view members of a group in Outlook

How do I view the people in each group ? I need to make sure I've deleted / added the right people to the right groups and not missed anything.

Thank you for your replies to my question about moving people from a group but not from my contact list. Took me a while  but I got there in the end !

Thank You :-)
kathleencollins say:

Please call it Outlook.com, not Outlook.

It is because, for years, there is Microsoft Office Outlook, an email program. We call it Outlook.

So make a distinction between the 2 to avoid confusion or someone gives you wrong answer or ask you " which one ? ".

How to go to People window :

Open outlook.com > place your cursor over the Outlook logo > a down arrow will appear > click it > select People > you are now in contact list window.

 

How to access a group :
In the People window, near top left, click at the word " All v " > a drop down box will appear with a list of your groups > click the one you want > all the contacts within that selected group are now on the left side > scroll to see the whole list.

 

How to add contacts to the group you have created :

Still in People window > click a contact you want to be added to the group > if you have multiple contacts to add, click the box at top left of the contact > continue clicking the top left box of each contact you want to add to the group until you are done> at the toolbar, click Groups > check the box in front of the group name > click Apply > all the contacts you have selected are now in the group.

Note : make sure you click squarely at the check box. If you click outside the box, all previous boxes you have checked will be nullified, and you’ll have to start all over.

How to delete a contact from a group but keep it in the main contact list:

Go to People > click the contact you want to remove from a group > toolbar, click Groups > un-check the box in front of the group in question > click Apply when done.

How to Edit contacts within a group

On the People window, top left, click the “ All v “ > you group is on the drop down list > click the group name > now you have all the contacts within that group listed > click one to view > click Edit at the toolbar to do editing......
Note : Edit option only appears at the toolbar after you have selected a contact.

issygirl say:

ACCESS 2007 Renumbering Deleted Records

I need to know how to renumber records after deleting duplicate records.  I want sequenced numbers and not sure if you can do it in ACCESS like you do it in EXCEL. Thanks! I need them in order again after deleting some records. 

ken sheridan say: Create and execute an 'update' query like this:

UPDATE [YourTable]
SET [YourTable].[YourNumber] =
DCOUNT("*","[YourTable]","[YourNumber] <=" & [YourTable].[YourNumber]);

However, as the number per row will change whenever you do this, this does mean that the number can have no real meaning as an attribute of the entity type which the table represents, so why not just return the row number each time as a computed column in a query?  The following query, for example, numbers the rows returned by a query on a Transactions table, sorted by transaction date, with an autonumber column being used to arbitrarily any two or more transactions on a single date:

SELECT COUNT(*) AS RowCounter, T1.TransactionDate, T1.TransactionAmount
FROM Transactions AS T1 INNER JOIN Transactions AS T2
ON (T2.TransactionID<=T1.TransactionID OR T2.TransactionDate<>T1.TransactionDate)
AND (T2.TransactionDate<=T1.TransactionDate)
GROUP BY T1.TransactionDate, T1.TransactionAmount, T1.TransactionID;

Alternatively for an updatable query (which the above isn't)

SELECT DCOUNT("*","Transactions","TransactionDate  <=  #" &
Format(TransactionDate,"yyyy-mm-dd") & "#
AND (TransactionID <= " & TransactionID & "
OR TransactionDate <> #" & Format(TransactionDate,"yyyy-mm-dd") & "#)")
AS RowCounter, TransactionDate, TransactionAmount
FROM Transactions
ORDER BY TransactionDate, TransactionID; 
Ken Sheridan, Stafford, England

Can you explain why you want them renumbered? Usually records can be sorted by some value within the record.  So using an arbitrary number really doesn't mean anything. If you want to be able to sort records based on when they were entered add a timestamp field to your table. Set the default value to =Now(). Then you can always sort on that field to re-order them.

As Ken correctly pointed out, a numbering like you are talking about and he suggested is meaningless.  If you want sequential numbering then you can never delete records.

ken sheridan say: To update the values:

1.  BACKUP THE DATABASE FIRST!

2.  Open the query designer.

3.  Switch to SQL view.

4.  Paste in the first SQL statement.

5.  Edit it to your real table and column name.

6. Run the query.

But why?  You then have a completely meaningless set of sequential values.  If you want to be able to return rows in the order entered date/timestamp a column as Scott describes.  You can delete rows until the cows come home, but the ability to order the rows will still be retained.  If you really need to number them you can do so with a computed column in a query, or an unbound control in a report with a ControlSource of =1 and a RunningSum property of 'Over All'.
