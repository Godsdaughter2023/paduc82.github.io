ken sheridan say:

  1. If total Sum Debits and Total Sum Credits are not equal then , data shall not be saved

As there will always be times during data entry when the rows don't balance, the only way you could do this with a bound form would be to bind the form to a daybook table, posting the rows from the daybook to a ledger table only when they balance, i.e. HAVING SUM(Debit) = SUM(Credit).  The daybook table's entries can be temporary, emptied by means of a DELETE statement being executed once the rows have been posted to the ledger by means of an INSERT INTO statement.

The alternative would be to use an unbound form and post the rows to the ledger by means of a set of INSERT INTO statements within a Transaction, only committing the transaction when the rows balance. The balance following the insertion of each row could be held in a variable declared as Currency simply by incrementing the value of the variable with Nz(Credit,0)-Nz(Debit,0).  The value of the variable would need to be zero for the rows to balance and the Transaction committed.

ken sheridan say: That is only possible if you execute all the inserts within a single Transaction.  A Transaction has CommitTrans and Rollback methods, so you'd call the former if the rows balance, the latter if they don't.

Using a bound form it is simply not possible.  Think about it.  If a credit of 100 GBP is entered into an empty account and the row saved the account is immediately not in balance, so if a constraint were imposed on the table to do what you want, it would never be possible to insert a row into the table.

You correctly identified the accounting methodology when you refer to that adopted by 'normal accounting software', i.e. each atomic transaction is posted to a daybook.  When a set of atomic transactions are then posted to a ledger they will either balance or not, so only at this stage can your requirement ne met unless you do execute all the inserts within a single Transaction.

Thanks for your post. According to your description, you have the owner permission of the shared calendar, but you are unable to share them. It should be noted that although you have the Owner permission of one shared calendar, the exact owner of this calendar is still the person who shared it to you. When the calendar owners share their calendars to other users, they are only granting these other users permission to view the calendar. The calendar owner can increase these users’ permission, such as Owner, so that these users can do more than just view the shared calendar. If you have the Owner permission of one shared calendar, you can Create, read, modify, and delete all items and files, and create subfolders. As the folder owner, you can change the permission levels that other people have for the folder (But does not apply to delegates). You can refer to the share calendar permission level part of this article.

scoobyman---01 say: Hi indup

If you connect phone to your PC as a Windows USB "mass storage device" you should get a green icon in your taskbar. You need to right click on this and stop it so phone can be safely removed.

thechosenone22 say: You don't specify which phone you have, but I presume it's the N95. With the newer phones (i.e. NOT phones like the 6230) have the function of acting like a hard disk when you connect them by USB cable to your PC - this is what the mass driver mode is (or whatever the exact name is called - I can never remember !)

This is in addition to acting "normally" by being connected to PC Suite in order to do a backup/synchronise and so on. However the phone can NOT be in both states at the same time - it can only be in 1 or the other (I think there is one other state called image printing).

When your phone is connected to your PC in mass driver mode, if you unplug the USB cable there is a potential for all the data NOT to have yet been written to the memory card (as Windows doesn't always write data as soon as it is created to the memory card - instead it might wait until it has 1Mb worth is created - for example - and then write it all in one go which would reduce the overall amount of time taken in order for data to be written to the memory card - this is called caching) - this is applicable to all USB based storage devices including pen drives and iPods.

So what your phone is essentially telling you is that if you have your phone connected to your PC in mass driver mode, before pulling the cable out, you should eject the hardware in the Windows approved manner. You can do this easily by right clicking on an icon near your clock which has a green arrow pointing left above what looks like a grey box and choosing "Safely remove hardware". Note - If you have other USB devices, you may see more than one device in the list which then appears.

You CAN disconnect your phone from your PC if it is in PC Suite mode without having to disconnect it in the above manner (in fact the option to safely remove hardware won't be displayed) - this is of course also assuming PC Suite isn't currently transferring
Hope this explanation helps.
