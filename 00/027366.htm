    Here is the code:

    Private Sub Workbook_Open()
        If Application.Visible = False Then
            Exit Sub
        Else
            SomeOtherMacroRuns
        End If
    End Sub

    bill manville say:

    I don't have 2010 with me to test this today, but it might be worth trying this;

    Private Sub Workbook_Open()
      Application.OnTime Now, "Continue_Open"
    End Sub

    with the following in a standard module:

    Sub Continue_Open()
        If Application.Visible = False Then
            Exit Sub
        Else
            SomeOtherMacroRuns
        End If
    End If

    That gives Excel the chance to finish its startup before your code runs.
    If that workaround works, please let me know.  It seems to me that you have found a bug if the behaviour differs from previous versions, and it should be reported to Microsoft.
