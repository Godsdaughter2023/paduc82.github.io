I have to concur with John.  It seems to me you are introducing unnecessary complications, which can easily be eliminated by using bound forms in the conventional way.  All you need in a bound images form, whether it is a parent form or subform is immaterial, is a combo box bound to the CaptionID foreign key column (or whatever it's named; I hope you are not using the semantically meaningless ID as column names), set up in the usual way to hide the bound first column and show the text caption value from the Captions table.  It's then a simple task to execute some simple code, which is standardised, in the combo box's NotInList event procedure to enable the user to type in a new caption not currently represented in the Captions table.

The only issue you might encounter would be if the current image record is locked by virtue of currently being edited in another form in your file, or by another user in a multi-user environment.  This would only be the case if an 'optimistic' locking strategy is being employed, in which case the built in dialogue for resoving conflicts would kick in.  If a 'pessimistic' locking strategy is being employed it would not be possible to even begin to edit a currently locked record.

Is it possible to bind a text box to a field in the query of a combo box on the same form?

If the table in question is both in the form's Recordsource, and in the combo box's RowSource query.  You'll find a simple example in FindRecord.zip in my public databases folder at:

 

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

 

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form definition accordingly. 

 

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

 

In this little demo file a form's RecordSource property is:

 

SELECT *

FROM Contacts

ORDER BY LastName, FirstName;

 

and the RowSource property of an unbound navigational combo box in the form is:

 

SELECT ContactID, FirstName & " " & LastName, 1 As SortColumn,

LastName,Firstname

UNION

SELECT 0, "<New Contact>", 0,"","" 

FROM Contacts

ORDER BY SortColumn, LastName, FirstName;
