frederik long say:
To update, I got an error when entering the string into cmd, the actual string entered was net user rich administrator /active:yes

The command
net user rich administrator /active:yes
is incorrect. There are three variants of it that would be correct:
net user rich administrator
(to set the password of the "rich" account to "administrator")
net user rich /active:yes
(to activate the "rich" account)
net user administrator /active:yes
(to activate the administrator account)

It is highly likely that you used some further variant of this command to set a password for the "administrator" account. To undo the damage, do this:
  1. Use your Windows 7 Repair CD to boot the machine into Repair Mode.
  2. Use System Restore to set Windows back to a point before you started playing with the Administrator account.
  3. Create, test and document a spare admin account, same as you have a spare house key.
  4. Do whatever you wish to do with the Administrator account.
Note also that the Administrator account has the same privileges as any other admin account.
