No version of Excel previous to 2016 has allowed setting print titles on grouped sheets.

Macro is required for that.

Sub Print_Row()
For Each oSheet In oSheets
       If oSheet.Type = xlWorksheet Then
            oSheet.PageSetup.PrintTitleRows = "$1:$1"
       End If
Next
End Sub

ken sheridan say: No, but you can create a dummy check box as follows:

1.  Add the real check box to the form and set its Visible property to False (No).

2.  Add a text box control to the form and size it to whatever size you want the dummy check box to be.

3.  Set the text box's FontName property to Wingdings; its Enabled property to False and its Locked property to True.

4.  Set the ControlSource property of the check box to:

     =IIf([TheRealCheckBox],Chr(251),"")

5.  Add a command button to the form and size it to fit over the dummy check box.

6.  Set the command button's Transparent property to True.

7.  In the Click event procedure of the transparent command button put:

    Me.[TheRealCheckBox] = Not Me.[TheRealCheckBox]

Using this approach you can have pretty much whatever character you like in the dummy check box.  You can also change the size and colour to whatever you wish.

Ken Sheridan, Stafford, England

To do what you want the only thing which I can think of immediately would be to include a square text box bound to a Boolean (yes/no) column in the table, positioned behind the check box and with a Locked property of True and an Enabled property of False.  Set both the ForeColor and BackColor properties of the check box to whatever the BackColor property of the detail section is by default.  Put code in the check box's GotFocus and LostFocus event procedures to set the value of the bound text box to True when it receives focus and False when it loses focus.  In the form's Current event procedure ensure that some other control has focus and set the value of the bound text box to False.  Use conditional formatting to change the value of the bound text box to a suitable colour when its value is True.

    larry yu msft say:

    Hi Bruce,

     

    The billing notifications are sent to global admins in our Office 365 organizations. It’s not feasible to make the notifications only be sent to a shared mailbox. However, we can follow the steps below to make the notifications be sent to the default global admin and a shared mailbox:

    1. Log in to Office 365 admin center.

    2. Navigate to Users > Active Users.

    3. Select the global admin account and then click Edit in the Roles section.

    4. Select Customized administrator > check all administrator roles except for Billing administrator. (Please perform Step 3 and 4 for all other global administrators)

    5. Navigate to Billing > Billing notifications.

    6. Select the default global administrator and then add the shared mailbox to the Alternative email address box.

    7. Click Save.

     

    If you want the notification only be sent to a shared mailbox, we suggest you submit your feedback to the related team via clicking the Vote button in the link below:

    https://office365.uservoice.com/forums/273493-office-365-admin/suggestions/10780035-billing-notifications-vs-global-admins

    Providing your feedback is the best way to perfect our products and services.
