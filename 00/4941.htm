Moving From Fast Ring To Slow Ring - Microsoft Community

airbus a-330-200 say:

Moving From Fast Ring To Slow Ring

I need to find out. If I move from the fast ring to the slow ring, will I have to reset or do another clean install? 

I was also considering moving to skip ahead but I saw where it said it was closed and if I selected it my settings would be reverted. I'm not sure what to do now tbh. Does anyone have any suggestion as to how I go about it? Thanks

Moved from: [Windows Insider Program / Windows Insider Preview / Windows Update / PC]

cbarnhorst say:

If you are on Fast Ring nothing will happen if you try to move from Active Development to Skip Ahead and are rejected.  If you are on Slow Ring and try the same thing with the same result nothing will happen.  However any time you try to move backwards from Fast Ring to Slow Ring a reset has to happen to proactively make such a move.  If you allow your build to remain as-is until Slow Ring catches up to Fast Ring (usually at the time the new version of Windows is released to the public) you will avoid a Reset.

The one thing that will always force a reset is to leave Fast Ring Skip Ahead for Slow Ring since there is never a time when the build numbers of the two ever coincide. 

The rule of thumb is that moving directly from a higher-numbered build to a lower one always requires a reset.  That is why the "catch-up" approach is advisable.

jeeped say:
Actually the SEARCH(...) for separation or aliquot is wrapped in ISNUMBER() which equates this to a boolean TRUE or FALSE which equates to a 1 or 0 when used in a mathematical equation. 

This means that if B15 contains separation and/or aliquot it resolves as TRUE which is 1 times the number in D15. If B16 does not contain separation or aliquot it resolves as FALSE which is 0 times the number in D16. This goes down row by row to B49 and D49 adding either either the value from column D or 0.
 
                    

The most common reason behind the #VALUE! error in that type of equation is a text value in column D. Double-check the values in D15:D49 for text that looks like a number.

                   

In an unused column to the right, put in this formula into row 15,

=LOOKUP(TYPE(D15),{1,2,4,16,64},{"Number","Text","Boolean","Error","Array"})

... and fill down to D49. The text values should be immediately evident.

Due to the disparate sizes and placement of the tables, I believe that your best course of action is to a) use a modification of the SUMPRODUCT() supplied earlier for each table in an unused column to the right and thensum the 7 subtotals in B5 or b) stack 7 different SUMPRODUCT() formulas, adding each together.
