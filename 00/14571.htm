joeu2004 say:

That said....

Re: ``I want to know how the IRR function is calculated within Excel, can I look this up somewhere? Since my manual calculation doesn't match the IRR function result. For example IRR gets 14,33 % Where NPV=0 but I get 11,07 % when NPV=0``

First, it is possible to have multiple IRRs.  So you might find one; and Excel might find the other.  It is unusual, but not impossible, to have two IRRs so close.  But it depends on the shape of the NPV curve for varying IRRs.

Second, the details of the internal interative method are not clearly defined anywhere.  An out-of-date KB indicates that the IRR function uses a Newton-Raphson iteration.  But I infer from details and experiments that the implementation uses an approximation of the derivative (perhaps the Newton quotient) instead of the exact derivative, which is computable.  If that is the case, the step value is undocumented.

Another important unknown is how the IRR function calculates the NPV for each iteration.  Based on experiments, I believe it uses the same algorithm that the NPV function does. If that is correct, there is an issue that might explain the difference that you see.

The NPV function calculates NPV = CF1/(1+IRR) + CF2/(1+IRR)^2 + ... + CFn/(1+IRR)^n.  Note that the first cashflow is discounted.

The more common definition is NPV = CF0 + CF1/(1+IRR) + CF2/(1+IRR)^2 + ... + CFn(1+IRR)^n.  Besides the subtle relabeling of the cashflows (CF1 is "CF0"), the important difference is:  the first cashflow is not discounted.

The difference does not matter if NPV is exactly zero for some IRR.  By "exactly zero", I mean that the NPV displays 0.00E+00 when formatted as Scientific.

However, it can make a difference if NPV is slightly different from zero, e.g. 1E-4.

Yet another reason why we need to see an example Excel file.  It will help us to see your formulas and IRR model, not just "your values".

bob jones aka: cybertaz say:

You don't specify whether you mean centered Horizontally or centered Vertically.

Horizontal centering can be directly applied using the Center Alignment tool on the Home Tab in the Paragraph group, but it is better to include that attribute in the Style(s) being used in the document.

Vertical centering is a Section attribute which can be applied by going to Format> Document - Layout to select it from the Vertical alignment dropdown menu. If the document comprises multiple Sections select Whole Document from the Apply to dropdown menu.

Where does text format Plus: Pattern: Clear (White) - Microsoft Community

catbill say:

Where does text format Plus: Pattern: Clear (White) come from?

I like to work with colored pages.  Sometimes when I paste web text into Word, there is some sort of fill around the text.

I realize that  I can clear the formatting but I want to understand what this is.

It is some sort of text level formatting but I can’t figure out how to control it.    I tried changing “Text Fill” to “No Fill.”

But this is the result.

So, what is the formatting and how can I control it (other than erasing all text formatting)?

jay freedman say:

It's shading applied to text.

If the "Plus: Pattern: Clear (White)" is the only text formatting in the selection, of course you can just click the Clear Character Formatting button in the Style Inspector or press Ctrl+spacebar.

If the shading isn't the only text formatting, then to remove it without affecting any other formatting:

  • Select the affected text (or all text).
  • In the Paragraph group on the Home tab, click the down arrow on the Border button and click Borders and Shading at the bottom of the menu.
  • Click the Shading tab of the Borders and Shading dialog box.
  • Set the Apply To dropdown set to "Text".
  • Set the Fill dropdown to any color except white or "No Color", and then change it to "No Color". (If you don't do this little dance, the dialog won't do anything!)
  • Leave the Style dropdown set to "Clear".
  • Click OK.
