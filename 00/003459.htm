john - project say:
E.J.Gun,
I think the answer may be pretty simple. Some fields in Project are calculated by Project and can not be edited by the user. the Early Finish field is one of those.

Using the Format statement does nothing for data entry, it is used to format data output. If you are writing a date into a user definable date field in Project (e.g. Start), Project will basically take anything that can be interpreted as a date but using a string expression is probably the easiest (e.g. "5/10/12").

When using the SetField method in conjunction with the FieldNameToFieldConstant method, the value will be interpreted in the current format of that field. For example, if the option is set to enter duration in days a value of 10 for the SetField value will be taken as 10d. If the option is set for hours, the value will be taken as hours.

Just for reference, Project is a scheduling application and giving it took much data defeats it's ability to calculate a schedule. For example, attempting to give a start date, a duration and a finish date forces Project to set constraints and will most likely not result in the true schedule of the task.

Hope this helps.

John
