Justification in Word - Microsoft Community

rohn007 say:

Justification in Word

Technical Level : Basic

Summary
Word support 4 types of text justification on the ribbon, plus a hidden 5th. There are also a few Word "Options" that affect justification. This wiki will briefly review them.
Details

The Bing dictionary defines Justification as:


alignment of margins: adjustment of the lengths of spaces between and within words in text in order to make both left and right margins align.

I would alter that a little. Justification controls how words and letters are arranged on a line in relation to the left and right margins and each other.

The text in this wiki is "Left aligned", as is hand written text. All of the lines start in the same place on the left margin, while the right margin is "ragged".

Some languages are hand written "Right aligned". The lines start on the right margin while the text ends leaving the left margin "ragged". As well, in an effort to keep digits aligned and decimal places in currency lined up, most numbers and monetary values are often right-aligned if all entries have the same number of decimal places, or "decimal aligned" on a "decimal tab".
 

When printing was invented, some printers preferred the way their page looked when both the left and right margins lined up in a straight vertical line.  They did this by varying the space between letters and words to force the alignment. They did this in a couple of ways, either by creating multiple sets of letters that had wider or narrower spacing between letters and by changing the size of the space character, and number of space characters, between words.

Originally, Word implemented justification in a very simple way, by only adding extra spaces between words. It met the definition of justification, but not the intent of "old school" print setters by creating unnaturally large spaces between words.  When the conversion to Word for Windows was done, MS added an option to "do justification like WordPerfect". This option allowed word to do justification like traditional type setters did, by adding spaces between LETTERS as well as words. Word 2013 made that the default.

Word uses a non visible "hyphenation" zone to determine which word will wrap to the next line. Therefore, short lines, ie the last one in a paragraph, do not normally use the paragraph justification.

Below is a screen capture illustrating the different types of justification. I have turned on the option to display text margins to show the end of line word relationship to the margins



There a 2 more "Alignment" types in Word ("Decimal" and "Bar"), but they relate to TABS as they were used on manual typewriters.

When writing numbers, they are generally "Right aligned", or actually "Decimal Aligned" so that the "Ones", "Tens", "Hundreds" etc values are vertically aligned so they can be easily added.  This is seen in programs like Excel, that default numbers to right aligned.

In Word, you can use Right Aligned justification if you don't have a decimal place. But it you do have a decimal place, you don't want to add zero's to pad numbers for alignment because that implies additional precision that may not be present in the measurement.   Rather than using right justification, in Word it is better if you use a "Decimal Tab" to force numbers to align on the decimal point. There are 2 ways of defining tabs, either in the tab dialog or in the tab box on the ruler.


To use a decimal tab:

  1. you define the Decimal tab location,
  2. then to use it you first type a <TAB> char,
  3. type the number which will be aligned just to the right of the decimal tab,
  4. type the decimal which will be aligned on the decimal tab
  5. type the decimal values



The "Bar Tab" inserts a vertical "bar" character. It was originally used to manually create tables.

greginmich say:

Thank you for the concern, I have tried uninstalling Malwarebytes and retarting my computer and nothing happened. Is there any way I can start the service manually as the error description suggests?

You can try manually starting the Windows Defender Antivirus Service with these steps:

  • Press Win Key + R.

  • Type “services.msc” and hit OK.

  • Right-click on the Windows Defender Antivirus Service and click Start.

Or you can try setting the Group Policy to enable Windows Defender:

To add the Group Policy setting that enables Windows Defender: Type “cmd” in the search box; and then right-click on Command Prompt and select Run as administrator.

And then copy, paste, and enter this command:

REG ADD "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware /t REG_DWORD /d 0 /f

Then click the Restart button (it might be necessary to restart the PC first).

If this doesn’t work, then you should remove this setting: Type “cmd” in the search box; and then right-click on Command Prompt and select Run as administrator.

And then copy, paste, and enter this command:

REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware 

Note that this REG command is just the safe and easy way to perform the registry edit that was described in the answer to this thread:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/windows-defender-the-threat-service-has-stopped/5724ac44-80ff-4a2f-a3a5-46686a92edd9

It sets Windows Defender's operational state to “always on” [DisableAntiSpyware = 0 (logical “no/never”)], whereas [DisableAntiSpyware = 1 (logical “yes/always”)] sets Defender’s operational state to “always off”; and where removing the DisableAntiSpyware registry entry simply returns Defender to its default operational state – where Defender will be automatically disabled by the installation of any third-party AV app, and automatically enabled when a third-party AV app is uninstalled.
