Use REGEDIT to remove the Office keys from the registry and then delete the Office folder under %ProgramFiles%..  The keys are under HKEY_CURRENT_USER\Software\Microsoft and HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft.  There are HKEY_CLASSES_ROOT keys as well for the various Office file types, but those should be rewritten when you install the newer version.

hansv mvp say:

The options that you mention are only available for column charts, bar charts, line charts and area charts - when you have a date axis there, you can specify the min and max as a date, and the major and minor units as days/months/years.

But for an XY Scatter chart, the x-axis is strictly numeric, even though you can format the numbers as dates. The min, max, major unit and minor unit are always expressed as a number of days, So you'll have perform your own conversions.

It has always been like that in previous versions too.

hansv mvp say:

There is no way to set the major unit exactly to a decade, for the reason you mention. You can set the major unit to 4 years = 1461 days, or to a multiple of that.

Similarly, there is no way to set the major unit exactly to 1 month, since months vary in length.

The concept of our calendar, with its units of varying length, clashes with that of a scatter chart, where the axis is strictly numeric.

cortig say:

POP accounts by nature do not sync. When you connect, your computer downloads all emails from the server, but never syncs back the status of the emails (read, unread, replied, moved to a folder…), or any addition you would make on your Mac.

Depending on your settings, the messages remain on the server, or get automatically deleted.

You need to connect through a protocol designed for syncing like IMAP or Exchange.

If you are lucky, the server you already use supports IMAP and all you have to do is to re-add the account as an IMAP account.

If not, you need to get a new email address on a server that supports syncing (even a free account like Gmail for instance).

Disable your anti-virus application, all Norton Add-ons (if any), any real-time protection provided by an anti-spyware application (except for Windows Defender; includes Spybot Tea Timer, SpySweeper Guards, AVG AS guard.exe, etc.), disable any third-party firewall AND THEN enable the Windows Firewall.

NB: You're "working without a net" now: Don't to ANYTHING else online (e.g., browsing; reading email; chatting) except #4 below until your anti-virus application has been re-enabled!  If you're running ZA Free firewall, you may need to uninstall it instead of disabling it.

2. Uninstall "Security Update for Windows Internet Explorer 7 [or 8] (KB976325)" or "Security Update for  Windows XP (KB976325)" via Add/Remove Programs (make sure Show Updates box is checked at top right) and reboot.

3. Re-do #1 above.

4. Go to http://windowsupdate.microsoft.com | Select CUSTOM and scan | Install any Critical Security Updates offered (e.g., KB976325).  If a Root Certificates update is listed in the Optional Software Updates category on the left, install it to take full advantage of IE's enhanced security.

5. Reboot & make sure everything in #1 above is enabled again and, if applicable, disable the Windows Firewall and enable (or reinstall) your third-party firewall.
