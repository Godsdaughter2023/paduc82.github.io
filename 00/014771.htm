steve rindsberg say:

Katrina's reply leaves a lot to be desired.

The FILE format of the two versions is identical, but if you choose a font when creating the computer and the font isn't available on another computer where you open the presentation, a different font will be substituted, which often throws off the size of text, line breaks and that kind of thing.  This is true regardless of Mac/PC. 

The fix is to choose fonts that are known to be on all computers where you'll open the presentation.   Google "powerpoint safe fonts" (without the quote marks) to find a number of articles that will help you choose fonts.

As to colors, changing, we can lay that at the Mac's feet.  The Mac does color management automatically.  This can be useful in some circumstances, but it's completely counter-productive for PowerPoint's purposes.

Windows PCs don't meddle with the color, so what you see on screen will differ between platforms.

How you disable color management on the Mac seems to vary by Mac OS version and as a mainly-Windows-kinda-guy, I'm not really up to speed on that.  Mention your MacOS version and perhaps someone who knows will jump in with details.

MS Word changes my caption field codes when I insert a new caption - can I stop it?

I have a document with a heading style for the main body (the standard Heading 1, Heading 2, etc., in a numeric multilevel list, e.g. section 1.2.3.4), and a heading style for the appendices (named AppHeading 1, AppHeading 2, etc., in a multilevel list with the first style a letter and the rest numbers, e.g. section A.2.3.4).

When I caption a figure in the main body, and include the chapter number, I get automatic styleref and seq fields,

Figure { STYLEREF 1 \s }-{ SEQ Figure \* ARABIC \s 1 }

In the styleref field, Word finds me the name of the most recent style 1. Apparently Word has it hard coded in that 1 means "Heading 1". The \s in the styleref field strips out any non-incremental text from the chapter name (i.e. if the name of the section were "Section 2.", I'd get just "2").

In the seq field, Word returns the next number in the sequence named Figure, formats it as an Arabic numeral (1, 2, 3, etc, as opposed to a Roman numeral or an ordinal number like 1st, 2nd), and resets the numbering at heading level 1.

So for the third figure in section 2, I get Figure 2-3. Which is exactly what I want.

But in the appendix, my "chapter number" and the sequence reset should be from "AppHeading 1". That's easy enough to change in the styleref field. There's no apparent way to do that in the seq field, though, so I'm forced to insert a Heading 9 style on the title page of my appendix and use
