karl timmermans (outlook mvp 2012-2018) say:
>>
I moved recently to Office 365. The removal of categories in Mail, Contacts and Calendar dismays me. It was one of the most useful features. When will it return?
<<

Hello Roger

What you're describing would be applicable to IMAP accounts.

If the above is the case, highly recommended to move the data in "This Computer Only" folders (contact/calendar) to a PST file - via the following steps

1) create a new PST file in the Outlook profile (Home Ribbon --> New Items --> More Items --> Outlook Data File
2) Go to File --> Account Settings --> Data files tab --> select the newly created PST file --> Set it as the default
3) Move the data in the "This Computer Only" folders to the corresponding folders in the PST file
4) Emails that want/need to categorize should similarly be moved to folders in this (or any) PST file

The lack of categorization for IMAP accounts is due to a limitation of the IMAP protocol - not an Outlook deficiency.

karl timmermans (outlook mvp 2012-2018) say:
>>
because I had categories with IMAP before migrating to Office 365 desktop
<<

Would I be correct in assuming that you migrated to O'365 from Office '2010 or earlier? (Imap accounts in those versions used PST files - one for email folders and one for the default contact/calendar folders)

Re: Sync'ing across devices

#1 - Not applicable to contact/calendar data since that doesn't sync with the server - in any version of Outlook and regardless whether or not Outlook is used since that's defined by the IMAP protocol used by all <email clients>

#2 - In terms of Email - yes, you would lose sync functionality if the emails were moved to a PST file

#3 - As originally mentioned, really suggest that you seriously consider moving your contact/calendar data as described since if (1) the IMAP account is ever removed from the profile, the associated OST file will also be deleted at which point all data in "This Computer Only" folders will be lost or (2) Outlook cannot find/open the underlying .ost file, it will create a new one, sync the email folders but the "This Computer Only" folders will be empty (ergo - original data will be lost)

An .ost file can only be opened/read by the original Outlook profile/email account that created it. If you have a copy of an "orphaned" ost file, you ould need to use (purchase) a 3rd party OST2PST conversion tool to access any data within it.

karl timmermans (outlook mvp 2012-2018) say:
One other thing to add into the mix of things to think about....

If you did have the desire to sync your contact/calendar data across all devices, that can be easily be accomplished by creating a free Outlook.com account (if you already don't have one) which would be created as an Exchange account by default.

Add the Outlook.com account to one of your computers, set the .ost file for the Outlook.com account as the "default data file" instead of going the PST file route, copy the contact/calendar data to the corresponding Outlook.com folders and then add the Outook.com account to all other devices.

Everything (email, contacts, calendar) gets sync'd with the server with an Exchange account and there is no requirement to use the Outlook.com account for email purposes.

cbarnhorst say:

Your license is tied to the unique IDs of the motherboard (digital licensing).  When you change the motherboard you also change the unique IDs, hence loss of the old license.  Your new motherboard needs to be licensed.  Go to Settings/Update and Security/Activation and click the link that takes you to the Store so you can purchase a license.

As Softspra points out, a replacement with the device manufacturer's recommended replacement part is OK but replacement with a retail upgrade part is not as far as retaining the old license is concerned.  

cheri.maloch say:

Last time I had to upgrade (granted it's been a good long while) the motherboard on a PC, the licensing was locked like this, but I was able to contact Microsoft support and they unlocked the activation key so I could use the same one. I didn't have to buy a new license. So yes, while it does lock the activation out for the key, that license is still *yours*, and the PC that is was running on is no longer being used. It's tantamount to uninstalling office off one machine and installing it on a new one. It's still one license. You are just transferring it to a different piece of hardware. That being said, you can't expect to use an OEM product key for a Dell and expect it to work on an HP (for example), but a retail copy with a retail license key should transfer just fine if you can get in touch with Microsoft support to reset the activation.
