macropod say:

In TBender's case, a macro wasn't needed to solve the problem. First, though, you should understand why the message occurs. It occurs because the  printer's driver was telling Word that the margins for the document are encroaching on the pirnter's non-printing area - a zone reserved for paper handling. That could result in some content not being printed and/or creating smudges when ink on an inkjet printer gets transferred to the paper-handling mechanism. As per your post, a macro can be used to switch off the warning, but the cure might be worse than getting the warning so you can fix the problem.

Given that TBender's document was being used for a label merge, there was a very simple fix:

1. Change the wrapping for the table used to create the labels to 'around', with a centred alignment; then

2. Change the page margins to whatever minimum (or more) your printer requires.

Make these changes in the mailmerge main document. Thereafter, you can merge & print without the need for a macro and without getting the errors.

Note: The mere fact others might be able to generate Avery 8160 labels, for example, without getting the warning isn't relevant unless they're also using the same printer driver.
