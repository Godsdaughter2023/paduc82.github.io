talking moose say:
Mehmet is correct.

Because Outlook now stores messages as individual files you're not nearly as likely to run into corruption as with Entourage, which used a single monolithic file.

However, the time you'll spend maintaining your identity will be proportional to its size. In other words, you'll spend more time rebuilding your identity if it gets corrupt, spend more time copying your identity to a new machine or spend more time migrating your data to different software.

Lots of folks like to use email as their personal information database, which was never the intention of email applications like Outlook (Windows or Mac). Take advantage of multiple archives and take old data "offline". Mail from 10 years ago isn't important enough to keep in an active identity. You can export information to .olm files to divide your data into smaller chunks that you can retrieve as needed or take advantage of a third party application like OEAX to keep your current active data separate from your historical data.

Hope this helps!

jllatham say:

Ok, first you need to understand that all cells on a worksheet are "Locked" by default.  When a cell is Locked and the sheet is made protected (with or without a password), then the user cannot enter data into Locked cells.

So before you assign the password to lock the sheet, first select the cells your users will need to use to enter/edit information in and use

Format --> Format Cells --> and go to the [Protection] tab in that dialog and remove the check in the box next to "Locked".

After that you can hide the rows/columns that you want to and then Protect the sheet - quick way to do that is to right-click on the sheet's name tab and choose "Protect Sheet..." from the list and go from there.

That should cover the basics of it all.

toasty_springs say:

Macro - Cut and Paste Row

I am trying to get a macro that will cut and paste certain rows from one sheet (TM) if the value in column G is changed to "Closed" it will then cut and paste the data to the second sheet (TMC).

Please could someone point me in the right direction?

Many Thanks

cimjet say:

HI

This is an event macro, it will run every time you type Closed in column G.

On your Worksheet Tab, right click and select "View Code". Or press Alt+F11, it will do the same.
That will take you to the VB Editor.
Now paste the macro on that page.
Press Alt+Q to return to your worksheet.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim sh2    As Worksheet
    Dim sh1    As Worksheet
    Dim finalrow As Long
    Dim lastrow As Long
    Dim i      As Long
    Set sh1 = Sheets("TM")
    Set sh2 = Sheets("TMC")
    Application.EnableEvents = False
    finalrow = sh1.Cells(Rows.Count, 1).End(xlUp).Row
    If Not Intersect(Target, Range("G:G")) Is Nothing Then
        If UCase(Target.Value) = "CLOSED" Then
            lastrow = sh2.Cells(Cells.Rows.Count, 1).End(xlUp).Row
            Cells(Target.Row, 1).EntireRow.Copy Destination:=sh2.Cells(lastrow + 1, 1)
        End If
   End If
   Application.EnableEvents = True
End Sub

cimjet say:

Hi

My other post will copy and transfer to the other sheet.

This one will cut and paste to the other sheet also delete the empty rows.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim sh2    As Worksheet
    Dim sh1    As Worksheet
    Dim finalrow As Long
    Dim lastrow As Long
    Dim i      As Long, val as long
    Set sh1 = Sheets("TM")
    Set sh2 = Sheets("TMC")
    Application.EnableEvents = False
    finalrow = sh1.Cells(Rows.Count, 1).End(xlUp).Row
    If Not Intersect(Target, Range("G:G")) Is Nothing Then
        If UCase(Target.Value) = "CLOSED" Then
        val = Target.Row
            lastrow = sh2.Cells(Cells.Rows.Count, 1).End(xlUp).Row
            sh1.Cells(Target.Row, 1).EntireRow.Cut Destination:=sh2.Cells(lastrow + 1, 1)
            sh1.Cells(val, 1).EntireRow.Delete
        End If
   End If
   Application.EnableEvents = True
End Sub
