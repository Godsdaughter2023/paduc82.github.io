ronrosenfeld say:

This is a common problem, also seen in Tax Tables, and it is easily solved and maintained with a lookup table.

Create a table with three columns:  Volume, BaseCost, and Cost per Unit

Volume Base Unit
0 $0.00 $1.00
10 $10.00 $1.50
25 $32.50 $1.75
50 $76.25 $1.80

The logic for the Base cost is that it represents the cost for items up to and including the Volume on that line.  So with the CostTable in D1:F5, you can use this formula to fill out the Base Column in the table:

E2:  $0.00

And select E3 and fill down as needed

Then, with your Volume in A1, the following formula will return the costs:

=VLOOKUP(A2,CostTable,2)+(A2-VLOOKUP(A2,CostTable,1))*VLOOKUP(A2,CostTable,3)

With the table, you can easily change the unit costs, add or subtract tiers, etc.

The definition of a split Access database is that the tables are stored separately from the front end. Doesn't matter whether the tables are in an Access file or some other database store. You can find it in the advanced tab listed under security. Everytime I uncheck it, I go back and its rechecked. I need it to stay unchecked for my itunes to let me sign in. I have windows vista.
afzal taher say:

Hi Cristal D,

Welcome to  Microsoft Answers!

Certificate revocation list contains all the serial numbers of the digital certificates, which have been revoked. The server verification requires it for checking
but they are not trusted due to several possibilities like authorized person, certificate expiration date validity, matching of server name with the name on the certificate.

Disable any security softwares and try to uncheck “server certificate revocation” and see if it works.

If this doesn’t work, then follow the steps mentioned below to uncheck the option by modifying the registry:

1. Click the Start button, and in the search box, type in 'regedit'.

2. Next, go to [HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\] and right click on the DWORD value 'Certificate
      Revocation' and select 'Modify'.

 3. Set the value data as '0’ and click 'OK'.

 4. Exit from the registry and restart the computer once and check.

 Let us know if it helps.
