rich michaels - mvp (greatcirclelearning.com) say:

Hi AJ,

It's not a setting of any particular background color that automatically changes the foreground text color, it is Table Styles setting that is applied to the whole table that is causing this ... and I guess Office 2007 must not pickup these overriding style settings. In Office 2016 I know that a copied table with special table style settings gets pasted properly into Excel.

If you are using VBA you can look at the Table Style that is applied and determine what you need to do. "ActiveDocument.Tables(1).Style" If it returns "Table Grid" then no special table style has been applied. But if it returns  "Grid Table 5 Dark" then that is one where a text effect has been applied to the header text and first column text. To use this method, I think you'd have to manually designate all potential built-in table styles that apply a special text fill effect.

Another VBA approach would be to look for Text Fill Effects. For example, ActiveDocument.Tables(1).rows(1).Range.Font.Fill.ForeColor.RGB will return 16777215 if a White fill effect has been applied to the text of the row or it returns 0 if no fill effect is applied.

I hope this helps

Here is a macro that deletes the first comment in a document. It does not know which section the comment is in. 

Sub CommentDeleteFirstComment()
'   Charles Kenyon  2018/12/26

'
    On Error GoTo NoComment
    ActiveDocument.Comments(1).DeleteRecursively
    On Error GoTo 0
    Exit Sub
NoComment:
    On Error GoTo 0
    MsgBox Prompt:="There are no comments in this document.", Title:="Oops! No Comments Here."
End Sub

Sub RemoveWarningStyleText()
'
'   Charles Kenyon 
'   Deletes first instance of text formatted in Warning Syle in document

jay freedman say:

Word does not do OCR.

If the PDF was created by storing a picture of a page -- that is, if no OCR was done before the PDF was created -- then all Word sees is a picture. In your screen shot, the presence of the hole-punch circles indicates that it's a picture, not editable text.

If the PDF was created from an electronic text document (Word, WordPad, Notepad, etc.) or from a scan that has already been through OCR, then Word can import that text and apply any formatting that's stored with it.

Given the PDF that you have now, you'll have to use some other program to do the OCR. If you don't have a third-party program, you can use OneNote.

graham mayor say:

The illustration, as Jay has already said, is an image of the PDF in a Word document. That's why you only see the first page. Objects inserted into Word cannot span pages.

Word 2013 is the first version of Word to offer PDF conversion, but it is a rudimentary tool and can only extract text from a PDF where the PDF actually contains text. Where it's a scanned page i.e. an image of a page (as appears to be the case here) there is no text in the document and Word cannot convert it.

Word has no PDF functionality so does not offer the facility to search graphical objects for 'text'.

One Note does offer rudimentary OCR software capabilities, but it is never going to be as good as a dedicated OCR tool, unless you are very lucky with the formatting. If you want more accurate OCR, then you should look at ABBYY Finereader, which is arguably the best around. Even that isn't perfect!
