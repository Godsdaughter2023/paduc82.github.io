You could use a custom VBA function:

Function GetValue(s As String, rng As Range) As String
    Dim cel As Range
    For Each cel In rng
        If s Like "*" & cel.Value & "*" Then
            GetValue = cel.Offset(ColumnOffset:=1).Value
            Exit Function
        End If
    Next cel
End Function

In your example, enter the following formula in F1 on the first sheet:

=GetValue(B1,'Sheet2'!$B$1:$B$4)

douglas horton say:

Bear with me. The answers I see so far do not cover all possibilities, so I am giving you a more thorough answer. It's a fairly straight-forward process once you get the basics down.

  

This would be an easy problem if Table 1, column B had only colors. Then "Yellow" would match "Yellow". You might want to build your table like that in the future. There are ways to write a complicated formula using INDEX(), MATCH() and LEN(), but I doubt if your data is clean. For example, you might have "Red and white shirt" or "Blouse, black". I'll give you a way to use the menus and mention cleaning the data as part of the process.

Here's the  solution: Use Text to Columns to split column B apart.

Keep your data safe. First always make a copy of your spreadsheet and work in that copy.

1. Insert a column before column C, so that what is column C becomes column D. Select any cell in column C, then on the ribbon, HOME > Cells > Insert... >  Insert Sheet Columns. 

Note that in your example, everything is only two words. If there are items with more than two words, you may want to either add more columns or, to be safe, copy this column to a new sheet altogether.

2. Select you text in column B, then on the ribbon, DATA > Text to Columns. The wizard will pop up. On the first page, select "Delimited" and then Next. On the 2nd page, make the delimiters the "Space", and select "Finish". The next column is blank because you just created it, so if you get a warning about data in the next column, you can ignore it and click through it.

Note what I said earlier though about items that have more than two words. They will have more than one space and take up more columns. To prevent destroying data, you may want to be working on a new sheet with column B copied to it.

You should now have a column with just your colors. I rarely see clean data like that, though, so you might want check it for any odd entries. For example "Red and white shirt" will become "Red", "and", "white", "shirt" or "Blouse, black" will become "Blouse," and "black". I would suggest using Autofilter to quickly look through the color column to see if anything besides colors are there. When you find something, do a little manual editing to get the correct words in the correct column.

Now all of that was to get a clean column of colors data! It is easiest to START with properly ordered data. Now, here is the formula that does the work: =INDEX(Table2!C:C,MATCH(Table1!B1,Table2!B:B,0))

I have assumed these tables are on different sheets, so they are written here as different worksheets designated by the "!" after them.

Let me explain beginning inside the formula. 
MATCH(Table1!B1,Table2!B:B,0) brings back the row number of whatever color is in cell B1. It gets the name from Table1 B1, looks at Table2 column B, counts thenumber of rows down to an exact match (the "0" tells it to be exact.)

Now that Excel knows the Table2 row number, INDEX(Table2!C:C,rownumber) tells it to look in Table2 column C and count down and return watever it finds in that cell. In this example, that would be "Navy".

I hope this helps!

Typically you would have the client data in a Table, displayed on a Form; and use an unbound "Combo Box" on the form to find and display that client's record when you select the name. You wouldn't want to retype the name every time - just pick it from an alphabetical list, using the combo's Autocomplete feature, which will jump to the first instance of the name when you start typing.

The form would display the phone number (or numbers!), address, etc.

Tom's point is well taken - the combo will jump to the FIRST Jim Smith, and then you can (if need be) scroll down until you're sure you have the right person.

If you're trying to reenter, or even autofill, the client's data anew for each record - well, don't; that's not how databases work! The client info should be stored once, and once only, in the Clients table, with a unique ClientID field; other tables (sales, meeting schedules, etc.) would store only the ClientID which will link to the client's data.

ken sheridan say: Before deciding on this you need to consider the question of the functional dependency of the phone number.  

If a customer changes their phone number, and you wish that number to be reflected everywhere in the database, including previous orders for instance, then the phone number is  functionally dependent on the primary key of the Customers table, so appears only once in the database as a column in the Customers table.  To show the phone number in orders a form or report can be based on a query which joins the Orders and Customers tables on CustomerID.  A text box control in the form or report can then be bound to the PhoneNumber column from Customers.

If, on the other hand, you wish the phone number for each order to be that which was current at the time of the order was made, regardless of any later changes, then the current phone number is functionally dependent on the key of Customers, but the phone number at the time of an order is functionally dependent on the key of Orders, and there would be PhoneNumber columns in both tables.  When an order is inserted via a form, the form can again be based on a query which joins the Orders and Customers tables, but a text box control in the form would be bound to the PhoneNumber column from the Orders table.  A value would be assigned to it when a customer is selected, by means of code in the AfterUpdate event peocedure of the control bound to the foreign key column from Orders which assigns the value of the PhoneNumber column from Customers to the text box bound to the foreign key.

You'll find an example of the second scenario above in InvoicePDF.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the AfterUpdate event procedure of the customer combo box in the invoice form assigns the customer's current address data to controls bound to columns in the Invoices table as follows:

    ' insert current address data as invoice address data
    Me.AddressLine1 = Me.CurrentAddressLine1
    Me.AddressLine2 = Me.CurrentAddressLine2
    Me.cboCity = Me.CurrentCityID
    Me.PostCode = Me.CurrentPostCode
    Me.Email = Me.CurrentEmail

You'll find an example of the first scenario in the Inventory.zip demo in the same OneDrive folder.  In this the Orders form is again based on a query which includes the Orders , Customers, Cites etc tables, and the address data is shown in controls bound to columns from the relevant table, so, if a customers address changes their new address data will be shown for all orders, past and present.

Option Explicit

Sub WrongOBS()
  Dim C As Range

  'Clear "OBS"
  Range("K3", Range("K" & Rows.Count).End(xlUp)).ClearContents

  For Each C In Range("Q3", Range("Q" & Rows.Count).End(xlUp))
    '1) For each line, check effectivity ("TO" , column Q) and if date is < TODAY(), _
      mark line on Column K with "OBS". We'll call this an "OBS" parent.
    If C < Date Then Range("K" & C.Row) = "OBS"

    '2) For each line under an OBS line, if "LEV#" is larger than "OBS" parent, then _
      also mark with "OBS" until the next line in which "LEV#" is equal or lesser than _
      the"OBS' parent in question.
    If Range("K" & C.Row - 1) = "OBS" Then
      If Range("J" & C.Row) > Range("J" & C.Row - 1) Then Range("K" & C.Row) = "OBS"
    End If
  Next
End Sub

Sub RightOBS()
  Dim C As Range
  Dim Dict As Object 'Dictionary

  Set Dict = CreateObject("Scripting.Dictionary")
  Dict.CompareMode = vbTextCompare

  'Find the lowest level for all items, use the description as key.
  'This are the "OBS" parents.
  For Each C In Range("L3", Range("L" & Rows.Count).End(xlUp))
    If Dict.Exists(C.Value) Then
      If Range("J" & C.Row) < Dict(C.Value) Then Dict(C.Value) = Range("J" & C.Row)
    Else
      Dict.Add C.Value, Range("J" & C.Row)
    End If
  Next

  'Clear "OBS"
  Range("K3", Range("K" & Rows.Count).End(xlUp)).ClearContents
  'Clear "Purchase Lvl?"
  Range("R3", Range("R" & Rows.Count).End(xlUp)).ClearContents

  For Each C In Range("L3", Range("L" & Rows.Count).End(xlUp))
    'Effectivity cleanup function
    '1) For each line, check effectivity ("TO" , column Q) and if date is < TODAY(), _
      mark line on Column K with "OBS".
    If Range("Q" & C.Row) < Date Then Range("K" & C.Row) = "OBS"

    '2) For each line under an OBS line, if "LEV#" is larger than "OBS" parent, then _
      also mark with "OBS" until the next line in which "LEV#" is equal or lesser than _
