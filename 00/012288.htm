Getting error code 'Enter Parameter Value' on WHERE clause - Microsoft Community

jim1919 say:

Getting error code 'Enter Parameter Value' on WHERE clause

I've tried various syntaxes in the WHERE clause, and now show lots of brackets because Design View wants to show them. Seems I shouldn't need brackets because my field names are strictly alphanumeric. Anyway, all combos seem to come to same result: the "Enter Parameter Value" message. Here's the SQL:

INSERT INTO Test ( AccountNumber, Admit, FCid, FCGroup, Balance )
SELECT Placement2.AccountNumber2, Placement2.Admit2, Placement2.FCid2, Placement2.FCGroup2, Placement2.Balance2
FROM Placement2
WHERE (([Placement2].[AccountNumber2])<>([Test].[AccountNumber]));

john w. vinson say:

The parameter prompt is arising because you are only inserting into the [Test] table - not selecting from it - so no fields from [Test] are available as criteria. If the intent is to exclude those records where AccountNumber already exists in Test use a "frustrated outer join" query instead:

INSERT INTO Test ( AccountNumber, Admit, FCid, FCGroup, Balance )
SELECT Placement2.AccountNumber2, Placement2.Admit2, Placement2.FCid2, Placement2.FCGroup2, Placement2.Balance2
FROM Placement2 LEFT JOIN Test
ON [Placement2].[AccountNumber2]=[Test].[AccountNumber]
WHERE [Test].[AccountNumber] IS NULL;

The LEFT JOIN will find all records in Placement2, and if there is a record in TEST it will find that match; the criterion of IS NULL will include only those records which  do not have a match.

John W. Vinson/MVP

sylvain lafontaine say:

Access ask you for a parameter value for [Test].[AccountNumber] because it doesn't recognise it and thinks that you want to use a dynamic parameter. If you remove the INSERT part and keep only the SELECT part, you'll get a better idea:


SELECT Placement2.AccountNumber2, Placement2.Admit2, Placement2.FCid2,
Placement2.FCGroup2, Placement2.Balance2
FROM Placement2
WHERE (([Placement2].[AccountNumber2])<>([Test].[AccountNumber]));

There is not [Test] table in this query: therefore the above query is invalid, hence the confusion for Access. You need to add this table to the Select part of the query by using either a LEFT JOIN as indicated by John or an NOT EXISTS() statement:


INSERT INTO Test ( AccountNumber, Admit, FCid, FCGroup, Balance )

SELECT Placement2.AccountNumber2, Placement2.Admit2, Placement2.FCid2,
Placement2.FCGroup2, Placement2.Balance2
FROM Placement2
WHERE Not Exists (Select * from Test Where Test.AccountNumber =
Placement2.AccountNumber2);

For your INSERT query to be valid, you must be able to run the SELECT query by itself without the INSERT part.  You can take the above query, remove the INSERT part and still be able to execute the remaining as a valid Select query.

bernie deitrick say:

1) Copy this code.
2) Right-Click the sheet tab of interest.
3) Select "View Code"
4) Paste the code into the window that appears.
5) Save the file as a macro-enabled .xlsm file.
6) Make changes as needed

Private Sub Worksheet_Change(ByVal Target As Range)

    'If only one cell at a time is being changed, un-comment the next line
    'If Target.Cells.Count > 1 Then Exit Sub

    'To apply to a specific range on the sheet,
    'change the range address to the range of interest
    If Intersect(Target, Range("A2:B2")) Is Nothing Then Exit Sub 'A2:B2 is the specific range

    'Turn off events to keep out of loops
    Application.EnableEvents = False

    'Get rid of the changes
    Application.Undo

    'Turn events back on to get ready for the next change
    Application.EnableEvents = True
End Sub

In Outlook Web App, it's not feasible to filter calendar items by categories. To work around it, you can use the term "Category:Events" in the search bar to find all items which are assigned with the Events category.

darcie [msft] say:

Hi, Onkar--

 

Which user guide are you referring to? Using Continuum with your 950 and laptop is a wireless scenario, in which you need to do the following:

 

On your laptop, go to Settings ->System->Projecting to this PC

Set the first setting to Available everywhere or Available above lock screen.


If you don't see an error in Settings saying your laptop doesn't support Miracast (You can ignore the one that says "This device might have trouble displaying your content because its hardware wasn't specifically optimized for Miracast”), launch the Connect app on the laptop (can search for it in Cortana). It should say "Ready for you to wirelessly connect to <your laptop name>."

 

Then, on your phone, expand the settings and tap the Connect quick action. Your laptop should appear on the list of devices to connect to. Once you choose it, you should see the Continuum projection on your laptop screen.

 

Hope this helps!

-Darcie
