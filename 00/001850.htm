How to Increase OneDrive for Business Storage Limit to 5TB - Microsoft Community

prathaprabhu say:

How to Increase OneDrive for Business Storage Limit to 5TB

Dear Support,

I have heard that Office 365 users can increase their storage upto 5TB. But it seems like it is only applicable for the new user added to the admin account. Can you please tell me how to increase the storage size of already existing user. 

Thanks

Prabhu

prathaprabhu say:

Dear Sheen,

Here is the step by step tutorial on how to increase OneDrive for Business storage account to 5TB.

Users with Office 365 Enterprise E3, E4 and E5, or with equivalent Government and Education plans or with just SharePoint Online Plan 2 or OneDrive for Business Plan 2 and at least 5 licensed users are eligible for free OneDrive for Business quota increase from 1TB to 5 TB. Upgrade should be released automatically, but if you don’t want to wait, you can increase the quota manually using SharePoint Online management shell.

First you need SharePoint Online management shell installed on your computer. And for this you need PowerShell 3.0, which is a part of Windows 10 and for Windows 7 or older Windows Servers you can download it from

Windows Management Framework 3.0.
https://www.microsoft.com/en-us/download/details.aspx?id=34595

After PowerShell 3.0 installation you can install SharePoint Online management shell.
https://www.microsoft.com/en-us/download/details.aspx?id=35588

Connect-SPOService -Url https://contoso-admin.sharepoint.com -credential *** Email address is removed for privacy ***

where replace contoso by a name of your tenant and add admin credentials.

Using  Get-SPOTenant | Select OneDriveStorageQuota we can check current quota (in megabytes)

Now we set new default quota to 5 TB which is applied automatically to all new users

Set-SPOTenant -OneDriveStorageQuota 5242880

If we now again check the setting  PS C:\windows\System32> Get-SPOTenant | Select OneDriveStorageQuota, we should see 5TB quota

Now all new users will have 5TB quota, but current users are not affected by this change. Now we set the quota to current users

Set-SPOSite -Identity [siteId] -StorageQuota 5242880

where  -Identity is an address of OneDrive for Business, so in my example https://lu******an-my.sharepoint.com/personal/lu******z .

If we now check the setting  Get-SPOSite -Identity https://lu*****an-my.sharepoint.com/personal/lu*******cz, we should see new 5TB quota

Source: https://www.lukasberan.com/2016/04/onedrive-business-quota-increase/

Thanks

Prabhu

ken sheridan say: With a three-table hierarchy like this, to create a form in which data can be entered into all three tables you'd need a projects parent form, in single form view, and two subforms, one for tasks, the other for task dates.

The simplest way to do this would be for the tasks form to also be in single form view, and to embed this in the projects form as a subform.  The task dates subform can be embedded in the tasks form as a 'nested' subform and can be in continuous forms view.

The other option is for both subforms to be in continuous forms view, embedding both in the projects parent form.  The task dates subform can then be correlated with the tasks subform by making its RecordSource a query which references the Task_ID foreign key of the tasks subform's recordset as a parameter.

You'll find examples of both these approaches, using data from Northwind, in CorrletedSubs.zip in my public databases folder at:
