<<

The POP protocol only downloads items for the <Inbox> and deletes those items from the server if the <Leave Messages on server> option is unchecked. There is no configuration to sync any other folder with the server.

What would probably be a better scenario in this instance would be to configure the account in Outlook as an Exchange account and then "archive" the folders you want which would give you the data in a PST file the way you want.

martinleung say:

How to display a filename of another workbook in a cell without using marco?

I'm using CELL("filename", A1) formula.

But the reference cell is from another workbook. it worked initially, but result become #NA if the reference file is closed.

How can i solve this without the use of VB?

 

Many thanks

jeeped say:
If you put the =CELL("filename", A1) formula into the other workbook and reference that cell from the first, you should be able to overcome the fact that the remote reference is closed. For example,

in Book2's Sheet1!A1 put =CELL("filename", A1) and close it.
In Book1's Sheet1!A1 put ='[Book2.xlsx]Sheet1'!$A$1

That should overcome the #N/A errors that you are receiving.

All that does is let you install FROM the memory card. It does NOT let you install TO the memory card. So you won't save any space, unless your intent is to leave the xap files on the card, install when you want to run a program, then uninstall when done.


The xap from memory card is designed so you don't have to waste wireless data to download an app from the store. You download to a PC, presumably on a 'free' connection, then transfer to the phone SD card. It still installs in main memory.


That said, it has taken some time to show up in the store for xap files on my SD card. Sometimes a power down reset has helped. Sounds like your card is fine, considering all the media stuff seems to work. Xap is in the root of the card, right?
