Excel

Is it possible to put some form of protection in place in an excel workbook whereby once a text entry is made (or a value selected from a drop down list), the cell is automatically locked and can't be changed?  Grateful for any feedback.

hansv mvp say:

This requires VBA code.

Step 1: unlock cells

Select the cells that the user will be allowed to change. If the user should be able to change all cells initially, select the entire sheet.

Press Ctrl+1 to activate the Format Cells dialog.

Activate the Protection tab.

Clear the check box labeled Locked.

Click OK.

Step 2: protect the sheet

On the Review tab of the ribbon, click Protect sheet.

Tick/clear check boxes to specify what the user will be allowed to do.

At the very least, leave the check box 'Select unlocked cells' ticked. Otherwise, the user won't be able to do anything.

If you want to set a password that will be needed to unprotect the sheet, enter it. Do not forget this password!

Click OK. If you specified a password, you will have to enter it again as confirmation.

Step 3: write the code

Right-click the sheet tab.

Select View Code from the context menu.

Copy the following code into the worksheet module.

Private Sub Worksheet_Change(ByVal Target As Range)
    Me.Unprotect Password:="secret"
    Target.Locked = True
    Me.Protect Password:="secret"
End Sub

Replace secret with the password that you used. If you didn't set a password, use "".

If you want to lock only part of the worksheet this way, for example the range B2:D50, use this:

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Range("B2:D100"), Target) Is Nothing Then
        Me.Unprotect Password:="secret"
        Intersect(Range("B2:D100"), Target).Locked = True
        Me.Protect Password:="secret"
    End If
End Sub

Step 4: save the workbook

Switch back to Excel.

Save the workbook in a macro-enabled format: .xlsm, .xlsb or .xls, but not .xlsx.

Sub SplitOutArialNineElevenAndTwelve()
  Dim X As Long, A As Long, B As Long, C As Long, NextSpace As Long
  Dim Result As Variant, Txt As String, Words() As String
  Txt = Replace(Range("A1").Value, Chr(160), " ")

    End Select
  Next
  Range("A5").Resize(UBound(Result), 3) = Result
End Sub

john w. vinson say:

No, not with an autonumber; and that's intentional and that's a good thing.

 

An Autonumber has one purpose and one purpose ONLY: to provide a meaningless unique identifier for a record. What would you want to happen if you had 112,852 records and had to delete #2? Ok, Access could renumber 112,850 records in the table; if you had this table related to sixteen other tables, it could renumber the foreign keys in all of them; but it couldn't renumber the notes on Post-It notes on twelve people's desks, or the memory "Oh I've got to fix that error in ID 112,714"  in someone's mind...

 

If you want a count of records, don't use an autonumber; run a totals query and count the records. If you want sequential numbers for a report, generate them dynamically on the report or in a query. If you REALLY REALLY need self adjusting gapless numbers in a table... please explain why and what are the tables involved.

ken sheridan say: If you do need to number rows sequentially and to adjust the numbering as rows are inserted into or deleted from the table, then this should be done as a computed column in a query.  Take a look at RowNumbering.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little file illustrates a number of queries for numbering or ranking rows in various contexts.  Bear in mind that a table, as a set, has no intrinsic sort order, so you need a basis in the data for ordering the rows in the query's result table.  If you want to do so by order of insertion into the table, then the easiest method is to add a DateTimeStamp column to the table and set its DefaultValue property to Now().  You can then use this column as the basis for ordering the rows.

Rarely there may be cases where it is appropriate to store sequential numbers in a column in a table.  I do this for instance in a table of regular financial transactions, in which the order in which the transactions are executed is indicated by an index number.  When an intermediate new row is inserted, or an existing row deleted, or its position in the sequence amended, the following code is executed to reset the values:

    Dim rst As DAO.Recordset
    Dim strSQL As String
    Dim strRS As String
    Dim n As Integer
    
    Me.Dirty = False
    
    ' get current RecordSource property
    strRS = Me.RecordSource
    
    ' set form's RecordSource to a zero-length string
    ' to allow change to table definition
    Me.RecordSource = ""
    
    ' drop index
    strSQL = "DROP INDEX PaymentIndex ON RegularPayments"
    CurrentDb.Execute strSQL, dbFailOnError
    
    strSQL = "SELECT PaymentIndex FROM RegularPayments ORDER BY PaymentIndex DESC"
    Set rst = CurrentDb.OpenRecordset(strSQL)
    With rst
        .MoveLast
        n = .RecordCount
        .MoveFirst
        Do While Not .EOF
            .Edit
            .Fields("PaymentIndex") = n * 10
            .Update
            n = n - 1
            .MoveNext
        Loop
    End With
    Set rst = Nothing
    
    ' reinstate index
    strSQL = "CREATE UNIQUE INDEX PaymentIndex ON RegularPayments(PaymentIndex)"
    CurrentDb.Execute strSQL, dbFailOnError

    ' reinstate form's RecordSource
    Me.RecordSource = strRS
                    
    Me.Requery

Note that the numbers are at intervals of 10 rather than 1.  This allows intermediate numbers to be inserted before the above code is executed if it is necessary to insert a transaction between two existing ones.
