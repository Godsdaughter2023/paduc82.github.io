ken sheridan say: You'll need two group levels, one which groups by a fixed number of rows, the other by the values in the group's data.  At present I assume you have the latter.  Within this you should add the second group level.  You can use integer division to return a different value per n number of rows, but before doing this you'll need to number each row sequentially within the existing group, which you can do in the report's underlying query.  The following is an example which numbers transactions sequentially starting at one for each Status value and groups each 10 rows per status:

SELECT COUNT(*) AS RowCounter, (COUNT(*)-1)\10 AS TenGroup,
T1.Status, T1.TransactionDate, T1.TransactionAmount
FROM Transactions AS T1 INNER JOIN Transactions AS T2 ON (T2.Status=T1.Status)
AND (T2.TransactionDate<=T1.TransactionDate) AND (T2.TransactionID<=T1.TransactionID
OR T2.TransactionDate<>T1.TransactionDate)
GROUP BY T1.Status, T1.TransactionDate, T1.TransactionAmount, T1.TransactionID;

You can then give the new TenGroup inner group a header and footer section with the fixed spaces for user input on the printed report rather than including these in the outer group.  Force a new page after the inner group's group footer to start each group of ten (or whatever) on a new page.  The RowCounter column in the above isn't actually necessary for the report of course, but is included so that the way in which the rows are grouped is more apparent in the result set.
