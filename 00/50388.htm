a. user say:

Hi Steven,

I would suggest you to run SFC from Advance options which is in offline mode.

 

Follow the steps:

a)      Press Windows and C key together and select Settings.

b)      Click on Change PC settings.

c)       Click on Update and Recovery.

d)      Click on Recovery.

e)      Under Advance Startup, select Restart now.

f)       Click on Troubleshoot.

g)      Click on Advance options.

h)      Click on Command prompt.

i)        Type the following commands and press Enter from the keyboard:

Assuming the offline Windows drive is drive F:

Set the variable for the log file, so SFC doesn't attempt to store to log to the read-only media:
set WINDOWS_TRACING_LOGFILE=F:\TEMP\CBS.log

Run SFC with offline parameters:
sfc /scannow /offbootdir=F: /offwindir=F:\Windows

Once that command has finished, it's output should be logged to F:\TEMP\CBS.log, or wherever else you specified.
So, to pull the errors out of the file, run:
findstr /c:"[SR]" F:\TEMP\CBS.log

This will output all error lines to the command prompt, if you want that output to go to a file instead, run:
findstr /c:"[SR]" F:\TEMP\CBS.log > F:\TEMP\sfcdetails.txt

Now, your details regarding the failed repairs should be contained in F:\TEMP\sfcdetails.txt

Hi Leon,

We noticed that it's been more than 24 hours since your last post on this thread, so we just want to follow up on you and check if the DNS settings have been fixed and if you're now able to connect to the Internet properly. Otherwise, we're here to help as best we can. We understand that receiving no assistance when you really need it is frustrating, so in order to provide better assistance and narrow down the cause of the issue, we just have a few questions for you:

  • Did you make any changes to your computer before the issue started six months ago, like installing a program?
  • Do you see any specific error messages when you connect to the Internet? If yes, please post a screenshot.
  • Are you using a wired connection, or a wireless connection?

As an initial recommendation, we suggest using the built-in Network troubleshooter. You can do it by following these steps:

  1. Click Start.
  2. Type network problems and select Identify and repair network problems from the results. Follow the steps on the screen.

An outdated network adapter driver can cause connection problems. Check if there's an updated driver by following these steps:

  1. Click Start.
  2. Type Device Manager and select it from the list of results.
  3. Under Network adapters, right-click your adapter and select Properties.
  4. Go to the Driver tab and click Update Driver.
  5. Click Search automatically for updated driver software. If Windows can't find anything, check your device manufacturer's website (using a different device that can connect to the Internet) and download the latest network adapter driver. Save it to a USB flash drive so you can install it on your computer.

If the problem still isn't fixed, use networking commands by following these steps:

  1. Click Start.
  2. Type cmd, right-click Command Prompt from the listed results, select Run as Administrator, and click Yes.
  3. Type netsh winsock reset and press Enter.
  4. Type netsh int ip reset and press Enter.
  5. Type ipconfig /release and press Enter.
  6. Type ipconfig /renew and press Enter.
  7. Type ipconfig /flushdns and press Enter.

After flushing the DNS, try setting the DNS server again using these steps:

  1. Right-click the network status icon in your taskbar.
  2. Click Open Network and Sharing Center.
  3. Click Change adapter settings.
  4. Right-click the connection that you're having problems with and select Properties.
  5. Select Internet Protocol Version 4 (TCP / IPv4) and select Properties.
  6. Select Use the following DNS server addresses.
  7. For Preferred DNS server, use 8.8.8.8.
  8. For alternate DNS server, use 8.8.4.4.
  9. Click OK twice.

We'll wait for your response and the results of the troubleshooting.
