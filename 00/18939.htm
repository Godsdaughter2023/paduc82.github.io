I agree with Rich: templates is the way to go for this use.

It is certainly worth mentioning that this needs to work with Windows Word as well: in fact that's critical.  It takes some options off the table (not good options, so no loss...)

I would create a separate template for each layout, i think.  My experience is users will find it easier to "choose the two-column portrait" template than to fiddle around trying to remove from their document layouts they do not intend to use. Typically users run with their control characters and non-printing characters turned off, which means they can't see what they are doing.  This produces gruesome destruction if you ask them to edit complex structures.

Make sure that your finished artefacts are ".dotx" files.  Word has several file types: ".docx" is not a template, ".dotx" is a template with text and graphics, ".dotm" also stores macros (active content).  Thee are a few others we're not interested in here.  Dotx is the one you want: it operates as a template: when a user double-clicks one, they get a new document, rather than edit the existing file.

Please also throw out your DTP/InDesign methods for this project.  Word is a text flow application, a word-processor.  Any attempt to fix things on a page and have text flow around them is going to end in tears because the design of a word-processor is intended to prevent this.

Instead, anchor your floating objects to paragraphs and accept that they will move up and down the document with the text.

While you "can" do complex layouts with spans and straddles in Word, if you give these to end users, think bull in a china shop.  So don't  :-)

Let it flow and Word will do it all for you.

Hope this helps

bob jones aka: cybertaz say:
There are at least 3 points which may force you to rethink your approach :-)
  1. Master Pages are an element of Publishing Layout View in Mac Word only -- Publishing Layout View does not exist in any Windows version of Word or versions of Mac Word earlier than Office 2008,
  2. What are called Master Pages in Mac Word are structurally & functionally different from what you know as Master Pages in a true page layout program such as InDesign. [E.g., items on a Master Page can only be modified on the Master Page, not on the Content Page to which they pertain.]
  3. Many of Word's features (Footnotes/Endnotes, Captions, Cross-references) cannot be used in a Pub Layout document

Under the hood, Publishing Layout documents are just like any other Word document. The main difference is that the features of the 'mode' make it easy for users to "do things" without having any idea what "things" it is they're "doing" :-) IOW, Master Pages are basically the equivalent of Sections thereby allowing for differing Header & Footer content from one 'set' of pages to another. Therefore, a Master can't really exist unless there is actually at least 1 Content Page to which it applies.

macropod say:

Another way, without the If/Then construct:

{QUOTE{SET Val {MERGEFIELD Value}}{={={Val}<40}-{={Val}<=10}*2 \# "'{Val}';'{Val \* Charformat }';'{Val \* Charformat}'"}}

where the 'V' in the first 'Val \* Charformat' expression is coloured blue and the second one is coloured yellow.

 

Note that the above caters for the exact specification (e.g. 39.5, 10.5), whereas yours assumes integers.

 

A 'correct'  If/Then construct would be:
{IF{MERGEFIELD Number} > 10 {IF{MERGEFIELD Number} < 40 {MERGEFIELD Number} {MERGEFIELD Number \* Charformat}} {MERGEFIELD Number \* Charformat}}

where the 'M' in the first 'MERGEFIELD Number \* Charformat' expression is coloured yellow and the second one is coloured blue.

doug robbins - mvp office apps & services (word) say:

If F7 is the field that contains the numbers, you would use F7 in place of Number in the If...then...Else field construction.

 

As your requirement necessitates the use of a nested If...then...Else field construction, it's really beyond the capabilities of the Rule utility.

 

One way of constructing the field would be to enter 5 instances of the «F7» with a space between them at the location where you want the formatted number to appear

 

«F7» «F7» «F7» «F7» «F7»

 

Then press Alt+F9 and in place of the above, you will see

 

{ MERGEFIELD F7 } { MERGEFIELD F7 } { MERGEFIELD F7 } { MERGEFIELD F7 } { MERGEFIELD F7 }

then select the last 3 and press CTRL+F9, which will give you

 

{ MERGEFIELD F7 } { MERGEFIELD F7 } { { MERGEFIELD F7 } { MERGEFIELD F7 } { MERGEFIELD F7 } }

 

then select everything and press CTRL+F9 again so that you have

 

{ { MERGEFIELD F7 } { MERGEFIELD F7 } { { MERGEFIELD F7 } { MERGEFIELD F7 } { MERGEFIELD F7 } } }

 

Then enter the balance of the field construction, inserting a space either side of the < and the > and after the 11 and the 39.

 

If the numbers are not integers, then use either of the constructions posted by macropod.

Hi Gene,

To better assist you, please specify if you have a Windows 10 Home or Pro.  Group Policy Editor is not available in any Home or Starter edition of Windows, be it Windows XP, Windows 7, Windows 8.1 or Windows 10.

In case you have a Windows 10 Pro edition, you may follow the steps below on how to reset the all Group Policy objects using Local Group Policy Editor.

  1. Use the Windows key + R keyboard shortcut to open the Run command.
  2. Enter gpedit.msc and click OK.
  3. Browse the following path to view all the settings:

    Local Computer Policy > Computer Configuration > Administrative Templates > All Settings

  4. On the right side, make sure to sort the settings by State to view only the ones that are Enabled and Disabled.
  5. Double-click each policy that is Enabled and Disabled and change its state to Not configured.
  6. Click OK and repeat the steps for each policy.

    If you have also changed settings under User Configuration, you need to go through the same steps mentioned above, but to get to the All Settings use the following path:

    Local Computer Policy > User Configuration > Administrative Templates > All Settings
