ronrosenfeld say:

Using your data, it seems to work fine here.  Even using the structured reference.  Note that with the formula in the same row, you may only need to refer to the column header.  And with the formula within the table, you can use an unqualified (no table name) reference.

Note the formula in the formula bar, next to fx.

As to why you are seeing a formula and not a result, be  certain the cell was formatted as General BEFORE you enter the formula.  If that is not the problem, I would have to examine the workbook.

Actually, the formula in Column F, as written, will return an ARRAY of all the values in the Asset BarCode column.  You are only seeing the one that is on the same row since it displays the first, and there is only one row in your sample table.  To "force" the formula to return ONLY the one value, and not an array, you need to add the ThisRow parameter, so the formula might look like:

=RIGHT(Table1[[#This Row],[Asset Bar Code]],7)

If UDF work on other Computer as Well ? - Microsoft Community

rajatupadhyay say:

If UDF work on other Computer as Well ?

I have created a UDF in VB which works on the computer I created it. But the same function don't work on other computer if I share the sheet to other team members. Is there any way ??

hansv mvp say:

If you store the UDF in a module in the workbook where it will be used, it should in principle work on other computers. But users will have to enable macros. The easiest way to do that is to make the folder containing the workbook a trusted location for Excel.

See Create, remove, or change a trusted location for your files.

rajatupadhyay say:

Thanks HansV, Easiear way not possible in my case as I always need to share the same file out of the shared network.

I also tried to put UDF in a module on workbook (With Macro Enabled). But the workbook is when opened on other computer not able to access the UDF(Shows the parent computer's directory there in cell rather then the functions result)

hansv mvp say:

If you want to place the UDF in a shared file, you can create an Excel add-in (.xlam) and save the add-in to a shared network folder. Each user can then install the add-in through Office button > Excel Options > Add-Ins > Go... > Browse...

Users will have to make the folder containing the add-in a trusted location; for this they must allow trusted locations on the network.

 

If the users see a formula in the cell instead of the result, make sure that the cell has not been formatted as Text, and that Excel is not set up to display formulas. Ctrl+` toggles between displaying formulas and displaying results.
