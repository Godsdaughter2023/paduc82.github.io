mluck94 say:

Can I delete individual files in a Windows 7 backup?

Thanks,
Mike
kosh vorlon say:

Hi Mike,

 

First, it can normally be done, but it isn't simple (and far more complicated/difficult {to the point where some will say "no, it can't be done"} if it's a complete image backup rather than a full backup; and even with a full backup, more complicated but just more effort if indivdual differential backups are also involved since you'd potentially need to check each of those also).

 

That said, the answer is also going to depend on what you used as the backup destination.  If it was and external HDD or a separate internal HDD or a network location, then you'll probably be OK.  If you used a set of DVDs and/or CDs, then it will be either extraordinarily difficult or, if the discs are not RW types (but R types), impossible (since you cannot make changes to them once they are written).

 

See this thread to see if it helps:  http://answers.microsoft.com/en-us/windows/forum/windows_7-performance/delete-individual-files-from-a-backup/5dbbe219-e062-4a11-9d87-a8ec26d97174.  Remember to use browse for files since you won't see the individual files with browse for folders.

 

If that doesn't help, then I recommend searching and/or posting for assistance in the TechNet Forum dedicated specifically to backup issues here:  http://social.technet.microsoft.com/Forums/en-US/windowsbackup/.

 

In my own case, since I do both periodic complete image backups as well as weekly full backups (with daily differential backups), AND maintain multiple copies as "backup backups" so to speak, it would be even more complicated.  If I felt the need to do this (but I can't see a good reason it would be NEEDED that desparately and would just let time heal the problem), I'd probably delete the file from the system and simply run another set of backups which while time-consuming, may in the end be simpler.

 

Good luck!

 

Kosh

  4. Restart your PC and Windows will attempt to reinstall the driver.
