WHERE are you executing the code?  It is unclear from what you posted.  I can only guess that you are still trying to do this in the change event of the control instead of in the after update event.

I would also recommend that you look at Marshall Barton's response, especially his recommendation on how to execute the queries.  I've taken the liberty of requoting his code and making a few changes to the code.

Private Sub EMP_STATUS_AfterUpdate()
Dim db As DAO.Database
Dim Answer As Integer

       On Error GoTo Err_MovePartTime

If Me.EMP_STATUS = "ACTIVE-PartTime" Then
   Answer = MsgBox("Emp_Status changed to Active-Part-Time.  It will be moved to the" & _
           "Part Time Table.  Are You Sure?", vbQuestion + vbYesNo, "EMPLOYEE STATUS CHANGE")
   If Answer = vbNo Then
       Me.EMP_STATUS.Undo
   Else
       Me.Dirty = False     'Save record
       Set db = CurrentDb()
       db.Execute "qry1_Append_PartTime-from_Mstr-to-PartTime", dbFailOnError
       db.Execute "qry2_Delete_PartTime_frm_Master", dbFailOnError
       Me.Requery
   End If
End If


Exit_MovePartTime:
    Exit Sub

    MsgBox Err.Number & " - " & Err.Description
    Resume Exit_MovePartTime
End If
End Sub

norman david jones say:

Hi Peter,

The fact that the last row does not respond to filtering suggests that the filter range does not include this row. This is confirmed by the fact that the application of the filter does not turn the row number indicator blue.

This may be because the filter range was inadvertently defined incorrectly, because there is a hidden blank row before the last row or because the problematic row was added after the filter range was defined .

In each case, try deleting the filter and then restoring and applying the filter.

If your videos and photos are stored in the microSD memory card of your Lumia 950, we recommend that you perform a ScanDisk maintenance on it.
