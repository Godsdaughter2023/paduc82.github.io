a. user say:

Hello Shivam,

This problem occurs in situations where Shutdown.exe cannot access the UserDomain environment variable. The shutdown task that is scheduled by using the at command runs under the System account. When the task is run under the System account, the UserDomain environment variable cannot be accessed because it does not exist. But the issue mostly occurs on server domains.

We would suggest you to try the following steps:

1.  Boot your system into safe mode.

2.  Press Windows key + R.

3.  Type sysdm.cpl and click OK.

4.  Click on "Advanced" tab

5.  In "Startup and Recovery", click on "Environment variables"

6.  Under "System variables" Click on "New"

7.  For "Variable Name", type in windir

8.  For "Variable value", type in C:\Windows

9.  Click OK on the all the dialog boxes that pop-up.

10. Reboot your system in normal mode.

If, however, your system is connected to normal mode, please refer to the following support article.

Computer Does Not Shut Down and You Receive a "System Could Not Find the Environment Option That Was Entered" Error Message When You Schedule a Shutdown Task by Using the AT Command

Let us know what happens.

Hello,

You may try to perform check disk in Windows Recovery Environment (WinRE) and see if that helps. You may follow the suggestions mentioned below to do the same.

1. Insert the Windows 7 installation disc into the disc drive, and then start the computer.


2. Press a key when the message indicating "Press any key to boot from CD or DVD …” Appears.
