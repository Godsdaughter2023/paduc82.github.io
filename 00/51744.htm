The best tested drivers for the computer's configuration are always from the computer or motherboard manufacturer.

The next best source is the component manufacturer.

On computers the information that is available about drivers are their version and dates.

There is no displayed information on file corruption.

A new version of Windows can become corrupt.

And in a similar fashion drivers whether new or old can become corrupt.

So drivers when they produce BSOD are often uninstalled and re-installed.

It would be best to install the driver displayed on the computer or motherboard manufacturers website.

Next best is the component manufacturer.

Perform Windows updates after installing the drivers as Windows will automatically update both Microsoft and non-Microsoft drivers.

Start your System, then just after the Manufacturers logo disappears and Windows 10 tries to boot, press and hold the power button down for 5 - 10 seconds to perform a hard shut down

Do this Twice

On the third boot attempt, Windows will boot into the Windows 10 Repair Environment, from there you can access Startup Recovery, Safe Mode, Command Prompt . . . etc.

Go to Troubleshoot - Advanced Options - System Restore, see if you have a recent restore point you can revert to. Restoring will hopefully repair any errors in your Registry . . .

Failing that go to Troubleshoot - Advanced Options - Command Prompt and run these commands one at a time:

bootrec /FixMbr
bootrec /FixBoot
bootrec /ScanOs
bootrec /RebuildBcd

Close Command Prompt, exit the WinRE and see if your System now boots into Windows

If you do not have a recent restore point, go to Troubleshoot - Advanced Options - Startup Settings, click restart.
Upon restart press 4 to boot into Safe Mode, if you can get into Safe Mode, backup your data, reboot and see if your system boots normally

If all this fails go to Troubleshoot - Advanced Options - Reset this PC, choose to keep all your files and Apps
This will re-install Windows 10 whilst keeping your data and applications

nick dich say:

Thanks for clarifying this one. It might be that your computer is on Fast Startup. Fast Startup combines the elements of a cold shutdown and the hibernate (sleep) feature. When this is enabled, Windows closes all applications and logs off all users, just like what a normal shutdown do. But rather than dumping everything in RAM when turned off, Windows will save an image of your loaded kernel and drivers, making it look like it is on hibernate mode. This way, when you turn ON the computer, it simply reloads the file, making it boot times much faster compared to what we use to have before.

If you do not want this to happen, there is a way to turn Fast Startup off. Kindly do the steps provided below:

  1. Right click on Start. and select Run.
  2. Type Control Panel.
  3. Search for Power Options and look for Change Settings that are currently unavailable.
  4. Kindly uncheck Turn on fast startup (recommended).
  5. Make sure to Save the changes.

Please let us know if these steps helped you with your concern.
