Hi Cathy,
 
First, the password you are being prompted for is the Access ULS (User
Level Security) password being requested by your MDW.  It is not
related to the network password, it's separate.  Do the other users
get prompted for a password?  They should if they're using the same
MDW, but they might not be since you're pointing to an MDW on the
local PC, not a shared one on the network.
 
If you don't think this app should be using Access ULS, and other
users don't enter a password, then try dropping the WRKGRP command
line switch to default back to the default System.mdw.
 
Regarding your control error, this might not be related to the
security issue.  It could be that the application is using a control
that isn't properly registered on the PC.  First check references to
see if anything is marked as Missing.  You might also have to open up
forms in design view to determine if there are any custom controls
being used - they'll probably show up as a white box if they aren't
installed properly.
