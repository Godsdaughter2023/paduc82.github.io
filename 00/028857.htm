ericqc say:

I assume you already realize that if your main ToC will include the figures AND they also need to be shown in their own ToC, you can't use the \b switch to isolate them via bookmarks.

If your figures use unique styles, you can just add a 2nd TOC field code (either entered within field code braces via Ctrl-F9 or via the References > Table of Contents > Custom Table of Contents... dialog.

For example, if your figures were consistently set with a unique style — say, Heading 4 — you could just add a second ToC for them alone. If you had used Heading 4, they would have an outline level of 4 (the default for Heading 4). This would map into a TOC 4 in a default table of contents. If you'd used a different style, you could set Outline level to Level 4 in the Format Paragraph dialog in the style to achieve the same thing.

To build your main ToC, the default settings would be sufficient, and the resulting field code would look something like this: { TOC \o "1-4" }. (Press Alt-F9 to toggle between the field codes and their results.)

You could then build a second ToC by specifying to only include the Heading 4s: { TOC \o "4-4" }

If you'd used a different style for the figures — say for example, a custom one named "Figure" — your TOC field codes will need to reference it with a \t switch. This can be done via the Table of Contents dialog by choosing Options... and setting the TOC level for the custom style. Whichever way you choose to do it, the TOC field code will look something like this: { TOC \t "Figure,4" }. This will build a ToC consisting of just the Figure-styled paragraphs, and use the TOC 4 style. If you wanted to set this ToC so the figures used a different level, you could change the level indicator (in this case, 4).

Word can have multiple TOC field codes, and various switches are available to manage what gets included. ToCs can be made from a mix of levels, styles and TC field codes. Refer to this Office support page for information about the various TOC field code switches.

macropod say:

Normally, you would number your figures using the Insert Caption tool, and then use the List of Figures tool to create that list.

And you would either mark your TOC entries by using appropriate heading styles (which I prefer) or by using TC fields. And then you would the Table of Contents tool to create that list.

Word's 'List of Figures' and 'Table of Contents' tools impose a severe field update performance hit on documents with large numbers of entries. You can generate essentially the same output - without the performance hit - via Insert|Quick Parts>Field>TOC>'Table of Contents', then choosing whichever of the 'Table of Contents', 'Table of Figures', etc. you want to work with. It's generally also best to uncheck the 'preserve formatting' option before clicking on 'TOC'.

Set up your table like this:

Column =>

Row 

 A B C D E F G
1 Unique ID Role Email Unique ID Role A Role B
2 123456 Role A Email 1 123456 Email 1 Email 2, Email 3
3 123456 Role B Email 2 22334455 Email 4, Email 5  
4 123456 Role B Email 3
5 22334455 Role A Email 4
6 22334455 Role A Email 5

With your list of non-duplicated IDs in column E starting in E2, and your list of unique roles in row 1 starting in F1.

Then, in F2, array-enter (enter using Ctrl-Shift-Enter) this formula:

=IFERROR(INDEX($C$1:$C$1000,SMALL(IF($A$2:$A$1000=$E2,IF($B$2:$B$1000=F$1,ROW($A$2:$A$1000))),1)),"")
& IFERROR(", " &INDEX($C$1:$C$1000,SMALL(IF($A$2:$A$1000=$E2,IF($B$2:$B$1000=F$1,ROW($A$2:$A$1000))),2)),"")
& IFERROR(", " &INDEX($C$1:$C$1000,SMALL(IF($A$2:$A$1000=$E2,IF($B$2:$B$1000=F$1,ROW($A$2:$A$1000))),3)),"")

Then copy that down and across the create your summary table.

If you have more than 3 possible duplicated roles for any ID, then you need to add an additional IFERROR block, with 4, 5, 6, etc. as the index (I hope you can see the pattern). If you have more than 1000 rows of data, then increase the 1000s to suit. Don't use entire columns (though it is tempting) because that will really slow down your workbook.
