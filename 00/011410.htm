DirectMusic in Windows 10 - Microsoft Community

themelodymaker say:

DirectMusic in Windows 10

Hello, I've installed the Windows 10 (64-bit) technical preview in a virtual machine via VirtualBox using Windows 8 settings by way of testing out the computer game I made to make sure that it's going to work when Windows 10 is officially released.  It currently works as is on every version of Windows from 95 up to 8.1.

I'm happy to say that in a Windows 10 virtual machine, the game runs near perfectly, with the exception that it won't play music.  After some diagnostics and further testing, I found out that not only the game but also any other application I try that attempts to load GM.DLS (in \Windows\System32\drivers\) fails.  Those applications referring directly to it simply presents an error message saying the file was not found (even though it's there) -- but others which simply attempt to load the default DLS without specifying that particular file raise no error, but no music is heard.

Does anyone else have this problem?  Do I need to try installing the Windows 10 preview as an actual host operating system before it will work (as opposed to a virtual machine)?  Is there something else that needs to be done?

themelodymaker say:

I did some more tinkering today, and I finally figured out what was wrong.  Keeping in mind that I've been trying the 64-bit version, I happened to notice that unlike in Windows 7 & 8 64-bit, the file GM.DLS was missing from the "\Windows\SysWOW64\drivers" folder -- so I copied it over to there from the "\Windows\System32\drivers\" folder.

That was all it took to fix my programs.  They now work perfectly, music and all.

However, I strongly recommend that you look into making sure that the file is in that folder as well by the final Windows 10 64-bit release, so other users don't have the same problem I did at first.

gina whipp say:

The maximum size of an Access database is 2 GB, yours is 1.6 GB.  As you use the database it needs *space* to move around.  Baring that in mind as you hit that 2 GB mark *things* start to happen like what you are experiencing.  So, since SQL Server is out...

Try splitting your database and if the portion holding the Tables is still large... split the Backend and use a single Front end to linking to multiple Back ends.  You might also try Compact & Repair and see if that gets you to a smaller size but that will only last as long as you don't *do too much* in the database.
