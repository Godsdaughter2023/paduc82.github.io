john - project say:
Dooniem,
Wow, what a difference a little understanding can make! Now your request is much clearer to me. Just for reference, in Project, "columns" are fields and row cells are called task fields (or resource fields or assignment fields, depending on the view).

Yes you can do a custom field with a formula. Here is the formula that works in the Flag1 field. Note, this assumes axis numbers are entered into the Text2 field sequentially ( i.e. 1,2,3, etc. and not 3,4,1,2). Note also that my default delimiter is a comma whereas yours is a semicolon so you will have to adjust the formula accordingly.

IIf(len([Text2])>1 And instr(mid([Text2],1,1),"1")>0 And instr(mid([Text2],2,1),","),"yes",IIf([Text2]="1","yes","no"))

The formula for the Flag1 and Flag2 fields must be more complex by necessity since a simple Instr function will give the same result for "1", "11" and "12". Whereas the formula for flag fields 3 through 12 can be much simpler. This is the formula for the Flag12 field.

IIf(instr([Text2],"12")>0,"yes","no")

You could avoid any custom field formulas altogether if you simply enter as many "yes" or "No" (for flag fields) or a "0" or "1" (for number fields) values into the appropriate fields. Granted, it's may seem like a little more effort but it is pretty easy to select a number field for example and hit "0" or "1". Just a thought.

Now that you have the tiers identified for each task, you can use filtering or grouping to organize the tasks in many useful ways.

Hope this helps.

Most common reason for 'Outlook can not connect to SMTP server' error is the wrong Outlook settings. This is especially true for newly configured email accounts.

A badly typed mail server name for a wrong port setting in Outlook can cause connectivity problems to the mail server and end up giving an error while trying to send or receive mail.

By default, the port settings for mail-related services are these, and a connection problem with these ports can lead to the error in Outlook:

SMTP - port 25/587
POP3 - port 110
IMAP - port 143
Secure SMTP (SSMTP) - port 465
IMAP4 over SSL (IMAPS) - port 993
Secure POP3 (SSL-POP) - port 995

There are also chances that some web masters will have duplicate profiles in Outlook that will have an old mail server configured and therefore would be trying to connect to the wrong mail server.

Session merging and the nomerge option

Exclusively in IE there is a -nomerge option which means that new windows do not share the same session cookie.

This has important benefits:

  • It's a key feature for enterprise cloud because it allows people to login multiple times to a cloud service.

  • It's a privacy and security feature because when you open a new Window it does not inherit your bank's session cookie. A side effect is that Facebook can't track you either.

Spartan doesn't seem to support this.

We have successfully defended IE in a number of installations because IE is the only browser that supports this.

daniel g. grau say:

Oh, that!

Don't worry about it. It does not mean that Word will be pasting HTML code (you can only get HTML code if you work on a .html file). It's actually what Word thinks is contained in the Clipboard, and is therefore recommending you should paste it by keeping the attributes supported by the HTML markup language. But I have yet to see different results between pasting as "HTML Text" and "Formatted Text (RTF)." However, "Styled Text" will paste the contents minus the hyperlinks and paragraph formatting.

Opinions on Clipboard contents are not uniform. If you place in the Clipboard some text with special "Text Effects" (like a large colored shadow or a reflection), Word will also recommend the "HTML Format" Paste Special option, even though HTML does not support such effects, while OS X will simultaneously flag that as "Formatted Text (RTF)." (To see this, place such text in the Clipboard, then go to the Finder and select Edit > Show Clipboard.)

In fact, no one seems to exactly know what the various paste options do, not even the "missing guides" appearing in Google books. Just search the web for varying opinions (and scant explanations). Some tutors even act surprised

at the results.

As for your second problem, when pasting from the web Word will always apply the "English (US)" language attribute to the pasted text. As mentioned in my first post, the usual solution is to select the text in question (with Select All or Cmd A, for instance) and then use Tools > Language to apply the proper language. Note that language is not a global preference setting, but more like a character attribute such as boldface or point size. I always thought it should appear under Format, since that is what it is.

robert aldwinckle on forums say:
 
What could this be?
 
Slow or deficient DNS.  
 
FWIW what I do in such circumstances is first check that my connection is still there e.g. with  ipconfig /all  and  nslookup.   Then flush the dns cache, e.g.   ipconfig  /flushdns.   Then  ping -n 1  the host name and all related names that the ping reverse lookup and ipconfig /displaydns exposes (essentially ping -n 1 all the names that an  nslookup  exposes).   Finally check with ipconfig /displaydns to see what your  Time to Live  for those cached lookups would be for IE.  Sometimes they can be mere seconds so if IE is to get any benefit from them you would have to have another window all ready to go with its request or even have all this, including the iexplore.exe request, written in a cmd script.  
 
A reason why IE might be so sensitive to this timing would be an artifact of an old IE4 feature called AutoScan which was also related to AutoSearch which still exists if search from the Address bar is enabled and if the user does not use a protocol prefix with the request.
 
Here is an example of what I mean by the above procedure using  www.microsoft.com
 
nslookup  www.microsoft.com 
 
Non-authoritative answer:
Name:    lb1.www.ms.akadns.net
Addresses:  65.55.12.249
          207.46.170.123
          207.46.170.10
Aliases:  www.microsoft.com
          toggle.www.ms.akadns.net
          g.www.ms.akadns.net
ipconfig  /flushdns
ping -n 1  -w 1  www.microsoft.com
ping -n 1  -w 1  toggle.www.ms.akadns.net
ping -n 1  -w 1  g.www.ms.akadns.net
ping -n 1  -w 1  lb1.www.ms.akadns.net
ipconfig  /displaydns
 
Note:  I added the  -w 1  switch because all of those hosts ignore  ICMP  and we don't want to wait for the default timeout.
 
Only when all of those ping are done would I expect that the cache could be used by  IE.
 
In fact, something unexpected is happening.   I can't get them all cached at the same time.  Even with the minimal timeout and even when I have them already entered and using cmd window's F7 to expedite re-entry of them.   YMMV.

If the Time To Live for the others was the same as these why aren't they still in the cache?  Having just the last two entries in the chain still cached won't help IE with its lookup.
 
This is actually a bad example of the technique because with just one alias and its canonical name it usually works beautifully.   OTOH it may be a good example to show you what could be happening in your case, so then assuming your initial lookups are slow too it could explain why IE doesn't get connected (assuming, of course, that DNS lookup is the cause of your symptom but we have to start there with no other details but the error message and symptoms you are citing).
 
BTW if these lookups are the problem and it is chronic an alternative is to hardcode problem lookups in your HOSTS file for the hostnames you want to use.  But then you would have to remember that you had done that in case the IP address you chose went out of service some time.   So if you think that this is the cause of your problem symptom you should probably notify your ISP (or simply wait until their server's cache makes the real lookup more to IE's liking.  <w>)

How do you half a page in columns in Word doc. and the other half of - Microsoft Community

kimgreenwood say:

How do you half a page in columns in Word doc. and the other half of doc. not with columns?

How do you half a page in columns in Word doc. and the other half of doc. not with columns?
suzanne s. barnhill say:

There are basically two approaches, both requiring section breaks because columns are a section property:

  1. Below the section with columns, insert a Continuous section break (Page Layout | Page Setup | Breaks | Continuous. In the new section below the break, change the number of columns.
  2. Select text that you want to have a different number of columns from the rest of the document. Select the desired number of columns (Page Layout | Page Setup | Columns). Word will insert the necessary section breaks for you.

Any portion of the document that has a different number of columns from the rest must be in a separate section, which means that it has to have a section break at the beginning and another at the end. With method (1), you can insert the breaks as you go. With method (2), you must have text (at least one empty paragraph) following (and omitted from) the selected portion, so that Word will insert both before and after section breaks. The breaks that Word inserts will be Continuous by default; in some cases it may be preferable to have a Next Page break.
