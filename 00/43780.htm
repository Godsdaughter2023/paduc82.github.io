ricardo bohner say:
Basically in Windows Explorer there is the logical (numeric) and the Binary classification mode for filename classification:

Since Windows XP windows uses the logical mode as default.

For example in logical mode "a44" comes after "a5" because "44" is bigger than "5" and the logical mode analyses the filename as a whole while binary mode analyses each letter individually and thus "a44" would be listed before "a5" because 4 is less than 5 (respecting the alphabetical order).

If you want to change from logical mode (default) to Binary mode you can use Group Policy Editor or Registry:

Group Policy Editor: (Type gpedit.msc in the run command to open)

Computer Settings -> Administrative Templates -> Windows Components -> File Explorer (or Windows Explorer depending on win ver) -> Disable numeric classification for File Explorer (Windows Explorer) = Enabled

* If you have a home version of Windows you may not have Group Policy Editor available

Registry Editor: (type regedit.exe in the run command)

To make the change for current user only:

Navigate to:

Hkey_Current_User\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer 

If the Explorer key doesn't exist create it. Inside explorer key create a Dword value called: "NoStrCmpLogical" and set it's data to "1"

To make the change for all users:

Navigate to:

HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer 

If the Explorer key doesn't exist create it. Inside explorer key create a Dword value called: "NoStrCmpLogical" and set it's data to "1"

* You may have to restart your computer or finish the explorer process and restart it for the changes to take effect.

Let say you have a file called: banana.txt and you want to put it in the end of a listing in a folder. You could change your clock to 2030, copy banana.txt to another drive, delete the original and copy banana.txt back.

That way banana.txt would have a date of 2030.

You go to sort -> date -> Oldest first. That way banana.txt would be in the end of the list. If you have a file called apple.txt and want to put it right before banana.txt you go and set its date to 2029 and so on....

You can also set the classification mode to binary and rename your files like this:

"021 wurstsalat.txt'

"022 apple.txt"

"023 banana.txt"

This way you can keep the order of files that you want. Also on the desktop you can have your files the way you drag them. I think in XP you could also drag your files and folders the way you wanted inside a normal folder but they canceled this in Windows Vista or 7.....

No.

Windows is built on the specific architecture defined for a PC and has to have a Intel x86( compatible ) processor as that is the instruction set it uses.

Android devices require ARM RISC processors and have a completely different architecture.

Apart from that Windows requires 1GB (32bit) or 2GB (64bit) memory
