larry yu msft say:

Hi Nicklnglis,

 

It’s not feasible to share a calendar to a DG (Distribution group).

 

However, we can share a calendar to a SG (Security group). Please see the steps below:

1. Create a SG in EAC (Exchange admin center) > recipients > groups > click the add icon (+) > Security group.

Note: Add all the staffs to the SG when creating it.

2. Open the Office 365 calendar in OWA (Outlook Web App).

3. Click Share > type the SG’s email address into the Share with box > Send.

Then all staffs in the SG should receive the share invitation.

 

Feel free to let us know if you encounter any problem during the procedure, we will assist you further.

 

Regards,

Larry

Hi Tore Olav,

 

Generally, this is because the current object that AAD Connect trying to sync has duplicate UPN values already assigned to another object. For example, Bob Tyler has bob@contoso value in its ProxyAddress, and he is already synced to Azure AD. Then an new account for Bob Smith is created with also *** Email address is removed for privacy *** in its ProxyAddress, thereby during the sync process in AAD Connect the error is thrown.

 

The key to deal with this issue is to find the duplicate UPN and ProxyAddress. You can perform this either by using this article and this following KB: https://support.microsoft.com/en-us/help/2647098/duplicate-or-invalid-attributes-prevent-directory-synchronization-in-o
