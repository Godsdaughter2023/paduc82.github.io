stefan blom say:

As you have noticed, hyperlinked cross-references are not formatted as hyperlinks. This is by design.

However, when a cross-reference repeats the formatting of the cross-referenced item, the cause is that the cross-referenced item has direct font formatting applied to it. The easiest fix is to select the paragraph containing the cross-referenced item and press Ctrl+SpaceBar to remove direct font formatting. If that brings back undesired formatting, modify the underlying paragraph style instead.

When you are done, update the cross-reference by selecting it and pressing F9.

stefan blom say:

At a guess, the idea is that cross-references should display as normal text in printouts. Obviously, this made more sense in the past, when most Word documents were read in a printed version.

You can force a cross-reference to make use of the formatting applied to the field code itself if you add the \* CHARFORMAT switch to the field.

Press Alt+F9 to reveal field codes. You'll see something similar to, but not identical to, the following:

{ REF _Ref2800825 \h }

Add \* CHARFORMAT as follows:

{ REF _Ref2800825 \h  \* CHARFORMAT }

Press F9 to update the field and press Alt+F9 again to hide field codes.

ken sheridan say: Enter the following in the debug window (aka 'immediate' window)

? CurrentDb.QueryDefs("YourQueryNameGoesHere").SQL

This should print the query's SQL statement to the debug window.  You can then copy and paste it into a new query in SQL view to edit and test the SQL statement.
