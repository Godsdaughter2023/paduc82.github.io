Once the header area is open, the main text gets faded, and you can left-click or right-click the watermark picture. When the picture is selected, the Picture Tools ribbon tab will appear automatically.

To close the header area when you're done with the watermark, double-click in the text area (or, the long way, go to the Header & Footer Tools tab of the ribbon and click the Close Header and Footer button).

Unwanted field in an Access table. - Microsoft Community

nelson fields say:

Unwanted field in an Access table.

I've created a form in Access from a table. Somehow, in the form a field has been created (Expr 1048) that duplicates an existing field. I can't delete the field. It doesn't show up in the table. Every time I insert the desired field, it defaults to the unwanted field.  How do I delete this field?
scottgem say:

Several issues here. First, I'm not sure why you "systematically added the fields and added descriptors for each field". It is much easier to use the Form Wizard to generate the form, then modify the layout in Design mode,

Second, you don't have "fields" on a form, you have controls that may or may not be bound to fields in a table. This is a subtle but important distinction.

Third, It looks like your table is not properly normalized. You are violating the first rule of normalization by using field names to define your data. Data should be defined by values inn rows and columns. To use an example you have 2 fields 1/2 Girth and Full Girth. What you should have is three tables. One table to identify the person, one table to list all the possible measurements and a 3rd table to capture the measurements. something like this:

tblPersonMeasurements

PersonMeasurementID (PK Autonumber)

PersonID (FK)

MeasurementID (FK)

MeasureAmount

Fourth, spaces and special characters in object names is not a good idea I suggest using camel notation like HalfGirth or underscores like Half_Girth.

I know these don't address your question, but we try to offer help for your database beyond just answering your questions.

What I suspect happened is you dragged the same field onto your form twice. When you drag a field onto a form from the Available list, Access creates a control for the field and names it the same name as the field. Since control names have to be unique on the form Access couldn't use the fieldname again so it used a default name. To delete it open the form's Property dialog and select the control from the pull down list on the top. This will select the control and you can then press the delete key to get rid of it.

The other possibility is that the table is not the Recordsource, but the Recordsource is a query which lists the same field twice. Again, Access will create a different name for the second instance. In that case, you can delete the field from the Recordsource. If you did not create the control then that will eliminate the entry in the field list.
