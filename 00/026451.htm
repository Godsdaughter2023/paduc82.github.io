Finding the 2nd most occurring Value (word) in Excel 2016 - Microsoft Community

richardyusa say:

Finding the 2nd most occurring Value (word) in Excel 2016

I used the formula:

 =INDEX(Table1[State],MODE(MATCH(Table1[State],Table1[State],0)))

to return the MOST occurring value, but I can't figure out how to display the 2nd most occurring value.

amit tandon say:

If you want to return the second most occurring value in a column, you may use below formula - enter as an array formula (CTRL+SHIFT+ENTER):

=INDEX($B$2:$B$100,MATCH(LARGE(FREQUENCY(IF($B$2:$B$100<>"",MATCH($B$2:$B$100,$B$2:$B$100,0)),ROW($B$2:$B$100)-ROW($B$2)+1),2),FREQUENCY(IF($B$2:$B$100<>"",MATCH($B$2:$B$100,$B$2:$B$100,0)),ROW($B$2:$B$100)-ROW($B$2)+1),0))

This returns the 2nd most occurring word in column B.

Microsoft Access Query Help - Microsoft Community

kai hanson say:

Microsoft Access Query Help

I conducted a survey that I stored the information in an access database. Instead of leaving the missing questions blank I put a "9" instead. Now it is time to do some calculations with the surveys yet I now want to change the 9's to blank values that are not used in the calculations. I did a replace function to change the 9's to blank values in a query but I cannot do any calculations with this. Can you tell me how to fix this?

scottgem say:
What is the structure of your table? If you want to do calcs on the field it needs to be a valid numeric value. 

kai hanson say:

There are many fields in the table all numeric values, for example: A person can put for a particular field numbers 1,2,3, or 4 (surveys were scale ranges) or they could place the number 9 that represents that the question was left blank (rather than leaving it blank and not knowing if it was supposed to be left blank or someone just forgot to put something in). Now it's time for the analysis part in which I want to get rid of all the 9's and put them as blank so I can do calculations. In a query  report this is what I did to get rid of the 9's

Expr1: Replace([DP8a],"9","")

However once I have created this new field in the query I am getting a mismatch error when I try to do sums, average, etc. Any thoughts about how I can do this?

scottgem say:
This doesn't tell us the structure of your tables. 

But what you are doing is storing a string value as Expr1. So you will always get a data mismatch. Again, you have to have valid numeric values. A Null or " " are not valid numeric values. 

kai hanson say:

I guess I am not quite following when you say the structure of my tables. It's just a simple survey that we are putting in access. A survey ID, name, and, date to start the table. Then there are fields representing a question. For example this is an example of a typical field:

Field Name: DP4

Type: Numeric

Validation Rule: Is Null Or In (1,2,3,4,9)

 

Now what I would like to do is find the average of this field yet I want all the 9's to just go away (they are only there to achnowledge that the question was left blank) So I manually went and deleted all the 9's for a particular field such that there were " " and I successfully got an average. I was curious if there is a query that will do this for me? Or I had a hunch that Expr1 above was being stored as a string, is there a way to store that as a numeric value?

scottgem say:
That's the structure what are the fields in your table. Your problem is your structure is incorrect. You are using a horizontal layout when you should be using a vertical layout. 

A survey database requires a specific structure with 4 tables.

tblQuestions, one record for each question in the survey
tblRespondent: Identifies who is completing the survey
tblResponse: Identifies who is completing the survey and when
tblAnswer: One record for each answer identifying the response and question

From there it would be easy to do a query that ignores all 9s and averages the rest.


kai hanson say:

Alright, I will adjust as follows. Would you now be able to provide me with step by step directions on how to create a query that idnores all 9s and then can do calculations on the rest?

Thanks

Kai

john w. vinson say:

You can keep your present (incorrect, Scott's advice is correct!) structure and still do your calculations. However, to do so you cannot change the 9 to a text string "" - you can't do math with text strings!

 

Instead, use an expression

 

IIF([fieldname] = 9, Null, [fieldname])

 

in your calculation.
