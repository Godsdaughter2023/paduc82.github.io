andreas killer say:

Copy the code below into the code module of the sheet.

I assume that there is no other code.

Andreas.

Option Explicit
Option Compare Text

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim A As Range
  'Which cell is changed?
  Select Case Target.Address(0, 0)
    Case "G1"
      'Do nothing if no Autofilter is present
      If Not Me.AutoFilterMode Then Exit Sub
      'Refer to the Autofilter range
      Set A = Me.AutoFilter.Range
      With A
        'Which value has the changed cell?
        Select Case Target
          Case "", "NONE"
            'Remove the autofilter in column CG
            .AutoFilter .Column - Range("CG1").Column + 1
          Case Else
            'Set the autofilter in column CG
            .AutoFilter .Column - Range("CG1").Column + 1, "=" & Target
        End Select
      End With
  End Select
End Sub

In fact, just a simple non-specific ping works for me but just to be clear I would try:


ping  -4  -n 1  -w 1  google.co.uk

ipconfig  /displaydns


and find (right-click, Find Up if necessary) your  google.co.uk  item in there as an  A(Host)  record.   The Time To Live on that gives you the number of seconds that  IE  would have to try to use that address.

russellbowden say: Most commonly webserved platforms use a Spherical Mercator projection, a version of WGS84. 

The most reliable coordinate base to use on such services is WGS84 Latitude and Longitude Decimal Degrees for spherical mercator, or regular WGS84, both will display correctly.

Your EPSG reference for this is EPSG:4326 for true WGS84 and will work fine, or EPSG:3857 for spherical mercator. You can search for these EPSG values on the Web to find suitable conversions from the projection you are currently using.

ken sheridan say: If you change this line:

  .FindFirst "Person = " & Me.combo

to:

  .FindFirst "Person = """ & Me.ActiveControl & """"

then it should work, as the ActiveControl property will return a reference to the combo box, regardless of its name, and as the Person column is a text data type its value needs to wrapped in literal quotes characters, each represented by a contiguous pair of quotes characters in the expression.

The dot operator (period) at the start of the line means that what follows is a property of the form's RecordsetClone.  As the line is contained within a With Me.RecordsetClone…..End With construct there is no need to precede the dot operator with anything.

You'll find examples of the use of unbound 'navigational' combo boxes in various contexts in FindRecord.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

You'll see in this demo that, where a specific person is being identified in the code, this is done by referencing the numeric ContactID column, which is the first (hidden) column of the combo box, and thus supplies its value when a person is selected.  People should never be identified by name, always by a distinct numeric identifier.  Personal names can legitimately be duplicated (I was once present at a clinic when two patients arrive within minutes of each other, both female, both with exactly the same names, and both with the same data of birth!). Consequently, when listing people in a combo box, it is as well to include some supplementary data in a second column to differentiate between two or more of the same name.  In my same OneDrive folder my NotInList demo does this.  While this demo is primarily intended to illustrate the use of the NotInList event, its opening form includes a combo box which lists people with their address in a second column, concatenating the address from a number of columns into a single string expression.
