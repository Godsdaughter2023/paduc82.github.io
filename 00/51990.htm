    mark bu say:

    The Microsoft Account and Yahoo accounts are two separate accounts.  Both accounts may use the same login ID (the Yahoo email address), and may be setup with the same password. For added security it’s recommended that you use two separate passwords.  

     

    To log into the Mail, the Microsoft account password is needed. To access the Yahoo email from the Mail app, the Yahoo password needs to be properly set for that accounts settings, Charms -> Settings -> Accounts -> <Yahoo Account>.

     

    It is possible to remove the Microsoft Account; however the steps to remove the account will depend on whether or not the Microsoft Account is linked to the Windows User account.

     

    Note that removing the Microsoft account from Mail will remove all accounts from Mail, Calendar, Messaging, and People.

     

    Mark

    frederik long say:

    Let's assume that you kept your file in c:\Users\Hossam\Documents\Holidays.

    1. Use Windows Exporer to navigate to c:\Users\Hossam\Documents.
    2. Save the whole Holidays folder in a safe place.
    3. Right-click Holidays, then left-click Restore previous version.
    4. Follow the prompts.

    Note also these points:

    • Emptying the Recycle Bin makes no sense at all. Doing so does not speed up the machine but it deprives you of the option of restoring a deleted file.
    • You do not appear to have a robust backup process. This violates the basic rule that important files must at all times be stored on two independent media.

    For the benefit of others monitoring this thread, the problem solved itself. 

     

    After leaving my laptop connected to the Internet through a LAN cable (not WiFi) for a couple of days, I noticed one morning that it was transferring data in at a high byte rate per second.  (You can monitor this by clicking on the internet connection in the system tray, choosing Network and Sharing Center, and then clicking a small link that says "Local Area Connection)." 

     

    The high bit rate transfer lasted for a minute or two, then stopped.  I checked back a couple of times later, and it still was not doing anything.  But I left it hooked to the Internet all night, and when I checked the next day, the Windows 10 Update had downloaded. 

     

    I'm speculating here, but it seems that the server at Microsoft may sometimes send the Windows 10 download in spurts, suggesting that you need to leave the computer hooked to the Internet, and be patient (not in terms of minutes, but days).

     

    IMPORTANT:  If after you have the download, and you want to wait before you install it, go immediately to Windows Update  (you can find that in the control panel), and change the settings for Important Updates to set for installation of updates, the next time it is that time, it will start the update to Windows 10 by itself.

     

    And don't bother unchecking it, because when you come back again, it will have re-checked itself.

    Hi C, this method works . . .

    Click your Start Button, type services and hit Enter

    Scroll down to find Windows Update Service

    If it is Started, double click it and stop that service

    CLose the services App

    Click your Start Button, type cmd, right click Command Prompt and choose 'Run as Administrator'

    Run this command and hit Enter

    sc delete wuauserv

    Close Command Prompt

    Restart your PC


    If you also have the Update Assistant installed on your system:

    The Update Assistant in Windows 10 cannot be uninstalled, but you can get rid of it

    Click your Start Button, type Task Scheduler and hit enter
    In the left Pane, navigate to: Windows - Update Orchestrater
    Right Pane double click 'Update Assistant'
    Click on Triggers Tab
    On each of the triggers ( At login . . . etc.) double click and uncheck 'Enabled'

    Under 'Update Assistant, there is another event 'Update Assistant CalendarRun'
    Do the same for all triggers on that event

    Restart your system, that's It!

    When finished and after a system restart, open the services app and make sure the Windows Update Service is now gone . . .
