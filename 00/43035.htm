Hi Paul, since Microsoft removed Homegroup in version 1803 of Windows 10, networking has become more difficult in Windows 10, this has worked for most users:


On each Windows 10 PC:

Click your Start Button, type Services and hit Enter

When the Services app opens, scroll down to find the following services, then double click each one in turn, set their startup type to Automatic and start the service if necessary

1. Function Discovery Provider Host

2. Function Discovery Resource Publication

3. SSDP Discovery

4. UPnP Device Host

Save that and close the Services app

That's It!

Open Control Panel

Go to Network and Sharing Center

Select Change advanced sharing settings in the left pane

Expand the network you want to disable password protected sharing on by clicking the down arrow on the right of the profile

Select 'Turn off password protected sharing' and then click Save changes.

I have seen and read a lot of posts within this area of the forum but none of them seems to clearly explain why the Mail App in Windows 8.1 fails to pull all mail received into the Inbox of an Outlook.COM/Hotmail account.

I have three email accounts,

1. A  hotmail account

2. An Eircom Mail Account

3. A Gmail Account.

On the Outlook.COM Web Portal, my mail is setup to receive not only my hotmail email but it also setup to allow me receive mail from my Eircom Account. Now for the kicker.... When I receive an email from any of the three accounts I can see all the mail from these accounts in both the Outlook.COM web portal and in the mail app on my Window Phone 8 device. However, in the Windows 8.1 Mail App I can only see email from the Hotmail and Eircom accounts.

Have I been mistaken? Is not the function of the Windows 8.1 Mail App to pull all mail from the specified email account, especially if you log into your machine with an Outlook.COM/Hotmail account? I understand that I can add the GMAIL account to the Windows 8.1 Mail App, but deleting a mail from the GMAIL inbox from within the App does not remove it from within the Outlook.COM Inbox or the Windows Phone 8 Inbox, and to me that defeats the purpose of the Mail App in Windows 8.1 as does the option of installing another mail client.

Thank you for posting on Microsoft Community. I understand the inconvenience you faced. I will certainly help you with this.

If there no updated drivers available on the manufacturer’s website, then try installing it in compatibility mode. Compatibility mode runs the program using settings from a previous version of Windows. Follow these steps to install the drivers in compatibility mode and check if that helps.

  • Download the drivers that are available from the manufacturer’s website.
  • Right-click on the driver setup file and click on ‘Properties’.
  • Click on the ‘Compatibility’ tab and check the box ‘Run this program in compatibility mode for’ and select Windows 7/8/8.1 operating system from the drop down menu and proceed with the installation. Once this is done, restart the computer and check if the issue persists.

danida_u say:

Hi

Skype Name is different from a display name. A Skype Name is the unique username that was created when users first joined Skype while a display name is the full name on the user profile and can be seen by the user and your contacts when logged into Skype.

If you created your Skype account using an email address then what you have is a Microsoft account, not a Skype Name. You can find additional details on this FAQ. What's my Skype Name?
