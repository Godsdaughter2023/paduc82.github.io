stephane zwel say:

First of all, converting the XLS file to XLSX will not make it faster to open, because XLSX adds XML overhead and zip compression.

If you get an error or warning message, it may be because indeed the file is corrupt, but it may be because Excel is super extra sensitive to everything it reads.

My suggestion would be to try open the file using LibreOffice or OpenOffice. You can resave as XLS from there, or even XLSX.

If your file does not contain sensitive information, you can always share it for further inspection.

A trusted source is a different concept than whether or not the file being open is corrupt or not.

When Excel opens a file, there are two phases : the first one is the reading of sheets. You can see it happen with the progress bar at the bottom of Excel. If it hangs somewhere in the middle, it's indeed some "corrupt" problem in one of the sheets. After sheets are read, Excel figures out whether it needs a recalc, which is arbitrarily lengthy. And then it runs VBA macros that are programmed to run when the sheet opens.

Hello

Thank you for the quick response,

What for your opinion it can refer to?

stephane zwel say:

If OpenOffice shows no data in cells, it probably thinks it is corrupt in the internal headers, at the beginning of the file (which happen to drive the reading of sheet bodies), so it skips everything. With or without an error message.

You can try with LibreOffice (which is more advanced and will perhaps fix the problem while opening it).

Is that truly an XLS file by the way? Sometimes, this is just a text or xml file whose filename ends with xls, and it therefore confuses Excel and co. If you want to know, open this file with Notepad, and see if it looks like text or giberish. XLS is binary so it should look like giberish.

Other than that, share this file somewhere for further inspection.
