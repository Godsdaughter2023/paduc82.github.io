kosh vorlon say:

Hi mcgarym,

 

This is the primary Windows executable file - a nasty one to have a problem with.

 

Let's start by opening MSE and going to Settings and Excluded Files and Folders and add this file there temporarily. Now run the scan and see if it works and finishes. If it was a quick scan, do a full scan also just to be sure it's working and that this file is the culprit. If so, MSE is working but there's a problem of some sort with that file. We'll come back to that shortly.

 

If it stalls on a different file instead after you excluded that one, then there is a bigger problem to resolve.

 

In either case, we should try to determine the cause since exclusions are holes in your security and the fewer the better - preferably none - and there's no good reason I know for this file to be doing that and many for not excluding it:

 

**********************************************************

 

Let's check the hard drive for corruption. Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator.Type chkdsk /f /r and enter and let it run. It will want to schedule itself to run at the next restart.Answer yes and then reboot to run the program. It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a potential cause. Now remove the exclusion and see if it still freezes. If not, problem solved. If so, read on.

 

Let's check the system files for corruption. Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator.Type SFC /scannow and enter and let it run. will scan and try to fix some of your system files. Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228).

 

Now try scanning and see if that resolved things. If so, problem solved. If not, keep the exclusion removed temporarily (for security reasons) and manually add it before each manual or scheduled scan (and be sure to remember or it will freeze up on the scan). If more than one file is still causing it to freeze, then we need to resolve the problem before you can scan again.
