Method 3

Reset the content of the Catroot2 folder/ Rename the software distribution folder

 

Step 1

Reset the content of the Catroot2 folder

a.     Type the following commands, and press Enter after each command:

·         net stop cryptsvc

·         md %systemroot%\system32\catroot2.old

·         xcopy %systemroot%\system32\catroot2 %systemroot%\system32\catroot2.old /s

b.     Delete all contents of the catroot2 folder, but do not delete the catroot2 folder.

c.     Type the following command, and then press Enter:

           net start cryptsvc

d.     Exit the Command Prompt window

 

Step 2

Rename the SoftwareDistribution folder

a. Open an administrative Command Prompt window.

b. Run the following commands, and press Enter after each command:

·         Net stop wuauserv

·         cd %systemroot%

·         Ren SoftwareDistribution SoftwareDistribution.old

·         Net start wuauserv

Try to install updates again.

ken blake say:

Assuming that you are running a 32-bit version of Windows, read the following:

All 32-bit client versions of Windows (not just XP/Vista/7/8/10) have a  4GB address space (64-bit versions can use much more). That's the theoretical upper limit beyond which you can not go.
But you can't use the entire  address space. Even though you have a  4GB address space, most people can only use *around* 3.1GB of RAM. That's because some of that space is used by hardware and is not available to the operating system and applications. The amount you can use varies, depending on what hardware you have installed, but can range from as little as 2GB to as much as 3.5GB. It's usually around 3.1GB.

Note that the hardware is using the address *space*, not the actual RAM itself. If you have a greater amount of RAM, the rest of the RAM goes unused because there is no address space to map it to.
