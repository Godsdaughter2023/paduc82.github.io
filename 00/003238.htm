ken sheridan say: You can parse out the substring from the position of the first letter, if any, onwards with a simple little function such as:

Public Function GetAlpha(strVal As String) As String

    Dim n As Integer
    Dim strChr As String
    
    For n = 1 To Len(strVal)
        strChr = Mid(strVal, n, 1)
        Select Case strChr
            Case "a" To "z"
            GetAlpha = Mid(strVal, n)
            Exit For
        End Select
    Next n
    
End Function

Note that the string comparison is case insensitive.  You can use this for the second sort level in the way Karl suggested, e.g.

SELECT *
FROM [YourTable]
ORDER BY Val([YourField], GetAlpha([YourField]);

dale howard [mvp] say:

Stormy Girl --

No, you do not need to have all of the sharer projects open in order for the shared resource pool file to work correctly.  However, you do need to initially open each project and connect them to the shared resource pool file.  After doing this, you will need to save each project and then save the shared resource pool file as well.  After this, you can close all of the files.  The next time you opens a sharer project, the system will prompt you to also open the shared resource pool file in Read-Only mode.  When you make changes to the project and save the changes, the system will also prompt you to save the changes to the shared resource pool file, which you should do.  The system will actually close the shared resource pool file temporarily, reopen it Read/Write, save the changes to the shared resource pool file and close it, and then reopen it Read-Only again.  It's a pretty amazing process.  Hope this helps.
