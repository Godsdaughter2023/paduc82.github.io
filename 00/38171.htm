¡firedog say:

*** I now want to recover my contacts from the backup. ***

In cases like this, it's always best to place the user data files first, before installing the program. The installer will then with any luck find the data files and take them into use, if the versions are at all compatible.

There is no provision for restoring contacts from a file backup. The two methods provided by Windows Live Mail are (1) synchronizing with the contacts set at contacts.live.com, and (2) exporting the contacts set to either CSV or VCF. Since you chose not to use either of these options, it will not be easy to recover your contacts.

There was a major change in contacts database management between build 3538 and 3555. Which build did you have on the crashed disk? You can see it in the file or program version in the properties of wlmail.exe, if program files were backed up. If not, you may be able to see the program version and build numbers in this text file:


C:\Users\{user}\AppData\Local\Microsoft\Windows Live\Installer\Catalog\short.xml

The contacts database for Windows Live Mail 2011 is, if I recall correctly, 

C:\Users\{user}\AppData\Local\Microsoft\Windows Live\Contacts\Default\15.4\DBStore 

This will, I think, only be of use if you also install Windows Live Mail 2011 on the new drive. To do so, you should first uninstall the 2012 version.

The files in C:\Users\{user}\Contacts are not used by Windows Live Mail.

Do you always use the same three paragraphs, or sometimes one or two?

If all three, you can simply type in the fresh text in-between the paragraphs in the template.

a. user say:

Is there a process I have to do in order to transfer my SSD with my current OS (Windows 10) on to the new PC?

Yes! But it requires third-party software.

Your Windows installation currently has drivers for your current computer's hardware. But you're going to move it to a computer that has different hardware, so it needs different drivers. And it's not only about the drivers - the hardware abstraction layer in your existing copy of Windows won't match the hardware in the new computer.

You can solve this in two ways:

1- reinstall Windows on the new computer, which also means reinstalling all your application software; or

2- make an image backup of your existing hard drive and do a hardware independent restore on the new SSD.

'Hardware independent restore' is a feature of most third-party backup image applications, although different applications call it by different names. The way this works is that the backup image application takes an image of your hard drive and substitutes generic drivers and a generic hardware abstraction layer. This makes it easy to restore the image to different hardware.

After you restore the image to your new computer, the new computer will work right from the get go, although you'll want to install proper drivers as soon as you can.

A hardware independent restore is beautiful for two reasons:

1- It's quick. It only takes minutes to restore an image of your C: drive.

2- Your computer will be usable immediately after the restore.

Activation is another issue: Whether or not Windows 10 will be activated on the new computer depends on what kind of license you used to activate Windows 10 in the first place.
