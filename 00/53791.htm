a. user say:

Regardless of how many recovery partitions there actually are, there should only be two: one for the OEM's factory reset procedure and a second for Windows 10's own reset procedure.

The OEM's factory reset is invoked by pressing a function key during restart. Windows' reset can be invoked from within Windows or by booting the computer from a Windows recovery disk or flash drive.

In real life there will sometimes be more recovery partitions, because people try to recover and get it wrong, and they also play around with the Disk Management snap-in.

In my opinion - which you are welcome to disregard and we can still be friends - a computer doesn't need any recovery partitions. All it needs is a backup image. If a user makes a daily backup image there is no real need for recovery, which makes you start over from the beginning. The same is true for the OEM recovery partition: After the computer is powered on for the first time and has completed all its setup routines, make an image of the result.

Regardless of whether it's the OEM's reset or Windows' reset, that's a nuclear option that is supposed to be a last resort.

marill say: Thanks, but I managed to fix it before I could reply to the topic. (I've been unable to login here for some reason to answer, strange)

Here's how I did it:

Start > Programs > ACE Mega codecs pack > ACE Mega codecs pack Dashboard

In the codecs list I had to look for the ASF Support and/or Windows Media Audio decoder codec in the Microsoft section, tick it on, and save the changes.

Then I dragged a new wma file into Movie Maker so it'd find the "new" codec. Now in my compatibility list there's one called WM ASF reader. Voila, it works again.

I do find it strange that a Windows program even needs a codec to run it's own file formats, when it used to be able to do it before. Other movie makers I've looked at needs no codecs for wma or wmv's, as it should be. What's happened, I don't know :(
