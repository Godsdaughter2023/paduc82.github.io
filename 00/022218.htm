Drawing objects in Headers or Footers frequently can be problematic - even more so than in the body of a document :-)

I'd suggest that you remove them altogether. Then, in the Header paragraph containing the text go to Format> Borders & Shading to apply a Bottom Border to the paragraph. Adjust the Right Tab in the Header if you want the border to extend beyond the text.

davidf2 say:

I believe the key information to your problem is the fact that you used the Win7 machine to change the font. Starting with Win8 and continuing with Win10 text is rendered differently in a Publisher text box than with Win7 and older OS machines. You will get about 5% more text in each box although font size, spacing, etc are all the same as before. Of course this means your layout will be skewed if you move back and forth between a Win10 machine and a Win7 or older OS machine.

As your primary machines are Win10 you need to avoid editing with a Win7 or older machine. The fix for this one file is less clear. If you had a lot of old Pub files that were produced with Win7 or older OS then you could copy the Win7 versions of the fonts to the Win10 machine. But since you are going to be using only Win10 in the future this does not make sense. You might have to simply rebuild this particular file.

You might be able to get away with just opening the file on a Win10 machine, changing the font to a Win10 font, tweaking the format so the text box is aligned correctly and then sending the file to the other person with the Win10 machine and have them view the file to see if the formatting is fixed.

You could also try copying all the text and pasting it into a NotePad file which will strip all formatting. Then create a new text box on the Publisher file with a Win10 font and paste the cleaned text back to the new text box. Adjust and format the new text box the way you want it and delete the old text box.

Here are a few references as to this different size text that started with Win8 fonts in case you are interested:

May 2016 -  Reference: Windows 10 Screws Up Publisher 2010: http://answers.microsoft.com/en-us/office/forum/office_2010-publish/windows-10-screws-up-publisher-2010/e2638788-7176-4573-8776-b1a7bcbeca99 - Another example of where replacing the new version of fonts fixes the problem.
 
October 2015 - Reference: Fonts in Publisher 2010: http://answers.microsoft.com/en-us/office/forum/office_2010-publish/fonts-in-publisher-2010/9874dea3-4d6f-40f2-8772-2995f0bc6e5a - User discusses how text rendering problem is also tied to specific fonts and that you can fix the problem by installing older versions of the fonts.
 

July 2015 - Reference: Publisher 2007 and Publisher 2013 paginates differently: http://answers.microsoft.com/en-us/office/forum/office_2013_release-publish/publisher-2007-and-publisher-2013-paginates/2ff3812a-5874-46dd-8c57-87aa3f86a321 - Well tested and documented thread showing the text rendering issue continues in Win10, but is limited to Pub 2013 and probably 2010. If you install Pub 2007 or 2003 in Win10, Win8, Win7, WinVista or WinXP then the files still render text the same and are cross OS compatible.
