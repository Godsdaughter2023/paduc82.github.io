Forget my previous reply. I was thinking something completely different. You could create a function that returns a Worksheet based on its codename:

Function WorksheetFromCodeName(CodeName As String, _
        Optional WBook As Workbook = Nothing) As Worksheet
    Dim WB As Workbook
    Dim WS As Worksheet
    If WBook Is Nothing Then
        Set WB = ThisWorkbook
    Else
        Set WB = WBook
    End If
    For Each WS In WB.Worksheets
        If StrComp(WS.CodeName, CodeName, vbTextCompare) = 0 Then
            Set WorksheetFromCodeName = WS
            Exit Function
        End If
    Next WS
End Function

Then, call this with code like:

Sub AAA()
    Dim N As Long
    Dim WS As Worksheet
    For N = 1 To 3
        Set WS = WorksheetFromCodeName("Sheet" & CStr(N), ThisWorkbook)
        If Not WS Is Nothing Then
            ' do something with WS

        End If
    Next N
End Sub

mike h.. say:

>Dumb question: what are codenames?

Hi In VB code there are three ways to refer to a worksheet:-

1. Sheet name. This is what you see on the sheet tab.
2. Sheet index. This the sheet position in the workbook. Sheets(1) being the first sheet.
3. Sheet codename.

Now if we take the first 2 it's easy for a user to rename or reorder sheets so any code using these can fall over very quickly.

The last is codename and initially in a workbook it's the same as the sheet name. If you open VB editor you will see something like

Sheet1 (sheet1)
Sheet2 (sheet2)
etc

Now the first name for the sheet (codename) is on the left, the second name is the name (on the right) you see on the sheet tab.

So if we refer to the sheet codename in our code it's much less likely the code will fall over because changing the codename can be done but is much less likely to be done by the average user.

If this post answers your question, please mark it as the Answer.

Public Sub SetCodeNames()
  Dim X As Long

Public Function CN(CodeNameIndex As Long) As Worksheet

Outlook will reply using the account the message was sent to. New messages are sent using the default account. If you have multiple accounts and you wish to reply using a different account from that to which the message was posted, then you will need a macro to make the change (or you can change it manually from the message itself). Maybe something like the following with the account name replacing the bold face text.

Sub ReplyUsingAccount()
Dim oAccount As Outlook.Account
Dim objItem As Outlook.MailItem
Dim oMail As Outlook.MailItem
Dim strAcc As String
Dim i As Long
Set objItem = ActiveExplorer.Selection.item(1)
strAcc = "someone at somewhere.com"
For Each oAccount In Application.Session.Accounts
    If oAccount.DisplayName = strAcc Then
        Set oMail = Application.CreateItem(olMailItem)
        With oMail
            .SendUsingAccount = oAccount
            .To = objItem.SenderEmailAddress
            .Subject = "Re: " & objItem.Subject
                       .HTMLBody = "<br><br><br>" & _
                        "<hr width=""50%"" size=""2"" noshade />" & _
                        "<font color=""#6699ff"">" & _
                        objItem.HTMLBody & "</font>"
            .Display
        End With
    End If
Next oAccount
Set oAccount = Nothing
Set objItem = Nothing
Set oMail = Nothing
End Sub
