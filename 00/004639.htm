suzanne s. barnhill say:

While some fonts do include circled numbers, they usually aren't in the position where Word expects numbers (Unicode 0030 to 0039), so they won't work for numbered lists, and in most cases only the digits 0-9 are included. There are specialty fonts with characters that allow you to combine left and right halves to get two-digit numbers, but, again, these are not suitable for auto numbering.

If you are willing to add the numbers manually, using the List style, which allows you to type a number at the left margin, then tab to the hanging indent position, then you will find two styles of circled numbers (black on white and white on black, both sans serif) in the Wingdings font and two different (serif) sets in Wingdings 2.

If you don't need them to be entirely circled, you can add parentheses on both sides. In the Define New Multilevel List dialog, choose the style followed by a closing parenthesis and then, in the "Enter formatting for number" box, type an opening parenthesis before the number field.

Hi Phil,

For the current situation, please run the following CSOM scripts, which will help recreate the four term sets under the existing Search Dictionaires group.

Note: please change the Admin URL and Icid based on your own situation. In addition, you may consider using Windows PowerShell ISE instead if you still get the error when using SharePoint Online management shell.

#Load SharePoint CSOM Assemblies

Add-Type -Path

Add-Type -Path

Add-Type -Path

#Variables for Processing

$AdminURL = "https://yourdomain-admin.sharepoint.com/"

$TermGroupName = "search dictionaries"

$lcid = “1033”  #locale ID

Try {

    #Get Credentials to connect

    $Cred = Get-Credential

    $Credentials = New-Object Microsoft.SharePoint.Client.SharePointOnlineCredentials($Cred.Username,

    #Setup the context

    $clientcontext = New-Object Microsoft.SharePoint.Client.ClientContext($AdminURL)

    $clientcontext.Credentials

    #Get the term store

    $TaxonomySession=[Microsoft.SharePoint.Client.Taxonomy.TaxonomySession]::GetTaxonomySession($clientcontext)

    $TermStore =$TaxonomySession.GetDefaultSiteCollectionTermStore()

    $clientcontext.Load($TaxonomySession)

    $clientcontext.Load($TermStore)

    $clientcontext.ExecuteQuery()

    #Get the term group   

    $group=$TermStore.Groups.GetByName($TermGroupName)

    #Add term sets

    $termset1 = $group.createtermset("company inclusions",

    $clientcontext.load($termset1)

    $clientcontext.executequery()

    $termset1.setcustomproperty("iscustomdictionary",

    $termset1.owner

    $clientcontext.executequery()

    $termset2 = $group.createtermset("company exclusions",

    $clientcontext.load($termset2)

    $clientcontext.executequery()

    $termset2.setcustomproperty("iscustomdictionary",

    $termset2.owner

    $clientcontext.executequery()

    $termset3 = $group.createtermset("query spelling inclusions",

    $clientcontext.load($termset3)

    $clientcontext.executequery()

    $termset3.setcustomproperty("iscustomdictionary",

    $termset3.owner

    $clientcontext.executequery()

    $termset4 = $group.createtermset("query spelling exclusions",

    $clientcontext.load($termset4)

    $clientcontext.executequery()

    $termset4.setcustomproperty("iscustomdictionary",

    $termset4.owner

    $clientcontext.executequery()

    Write-host "Term Sets Created Successfully!" -ForegroundColor

}

Catch {

    write-host -f Red "Error Creating Term Sets!"

}

For other users who come across this thread, please note this script is userd to create the four term sets under the existing Search Dictionaries group which has no value in it.

karl timmermans (outlook mvp 2012-2018) say:
What it means is that you have the email account configured using the IMAP protocol which only syncs email folders with the server. "This Computer Only" folders means exactly that - the data is local to the computer. The downside to having data in "This Computer Only" folders is that if you delete the IMAP account from the Outlook profile, it will also delete the OST file associated with the account which is where the folders are kept. Similarly, if Outlook cannot find the OST file, it will automatically recreate it and resync the email folders but all data in "This Computer Only" folders will be lost.

You have two choices to sync your contact/calendar data with another computer

#1 - Use a 3rd party program
or
#2 - Create a free Outlook.com account and configure it on each device where you want to share the contact/calendar data

macropod say:

Hi Tom,

You can clear out the old data by saving the file using File|Save As. Of course, if you're creating templates, it's best to save the documents as templates (.dot extension) in your templates folder. That way, you'll get a fresh document based on the template whenever you need one.
