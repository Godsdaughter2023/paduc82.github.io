Managed to solve my problem today, 03/nov/17.

Here is what I did.

1) Open regedit

2) Navigate to HKEY_CLASSES_ROOT\Excel.Sheet.12\shell\Open

3) Create key ddeexec. Set its default value to [open("%1")]

4) Go inside new key "ddeexec"

5) Create key application. Set its default value to Excel

6) Create key topic. Set its default value to system

7) Navigate to HKEY_CLASSES_ROOT\Excel.Sheet.12\shell\Open\command

8) Make sure default value ends with -> Excel.exe" /dde

If there is anything else, remove it

This solves the problem to .xlsx files. Did the same on "Excel.Sheet.8" key to solve the problem for .xls files.

Hello nicecap

Outlook desktop connects and sync's with <email accounts> - it does not directly sync with other apps (which includes the Win10 apps) or directly with any device.

In the case of Outlook desktop, sync'ing contact/calendar data requires an Exchange based account (i.e. Outlook.com). You cannot sync contact/calendar data from any other service (i.e. Google) without using a 3rd party addin or sync program. The Win10 app does enable you to add accounts from other services like Google etc.

So, if you want to sync contact/calendar data across all your devices (email clients), then the easiest approach would be to create an Outlook.com account (if you don't already have one) and configure it on each device and add the account to Outlook desktop program.

Please note, am simply explaining how things work. Not trying to defend anything so please don't shoot the messenger if this isn't the answer you're looking for which I'm pretty sure is the case BUT it is the way things work and that can't be changed at this point in time.

  4. Select the account you're having problems with and click 'Account Properties...'
  5. Click the Advanced tab
  6. Enter a 'Root folder path'; probably to be; 'Inbox' as below
  7. And then Ok, Ok, Close
