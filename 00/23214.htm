Give this formula at B2 and copy it down.

=IF(COUNTIF($A$2:A2,A2)=1,SUMPRODUCT(($A$2:A2<>"")/COUNTIF($A$2:A2,$A$2:A2&"")),VLOOKUP(A2,$A$1:B1,2,0))

Windows Defender on client systems since Windows 8 have included both the antivirus and spyware engines and definitions.  I'm not certain at what point the server versions of the operating system may have also included these, but since Windows Server 2008 was built from the same base as Windows 7 client, these would have been similar and contained only antispyware.

Note there is no mention on viruses or other malware, since these weren't included in the detections for older versions of Defender.  Officially Microsoft recommended using 3rd-party anti-virus products or their own current managed products [read paid] with Windows Server up until more recent versions where antivirus is included, which from a quick search seems to have first begun with Windows Server 2016.

Making a bilingual manual, languages on opposite pages

I am making a bilingual manual to be printed 2-sided. English goes on one page and the French version of the same text goes on the opposite page. I will need different headers and footers, but the same page numbering. The left side is page 1 of the English manual and the right side is page 1 of the French manual. I will also need two table of contents. Can this be done in Word 2007?
suzanne s. barnhill say:

This is virtually impossible to do in a single document. Much the simplest approach is to create separate documents for the two languages and then interleave the pages for printing. If you have Adobe Acrobat or the like, you can create a PDF into which you insert the pages alternately.

Also, consider that most multiilingual manuals are not constructed this way; usually they have the two (or more) languages in separate sections, often with one starting at each end, so that users flip the book to choose one or the other.

suzanne s. barnhill say:

Be aware that there is a limit to the number of linked text boxes you can have. I think it's 32.

Page numbering is doable to an extent, using calculated page numbers; numbering will be correct on the pages but won't be picked up by the TOC, so it's rather pointless.
