rich michaels - mvp (greatcirclelearning.com) say:
I hate that too... I create something that does not work and I have no one to blame or ask for help. :-)

So it sounds like the added header isn't really a header that is attached to the table. When the other rows push over onto the next page is there a non-editable copy of your header displaying on the second page? 

If not, then what you are thinking is a header row is not looked upon by Word as the actual header of that table. Word is designating these two objects as two different tables and that is why you are getting this behavior. Since this header row was added later it might be because that row is a slightly different size, have a different number of cells, or some other weird reason why Word thinks it is a separate table. 

What you can do is place your cursor at the end of row marker of the first row (non-header) and press enter. This will create an identically formatted blank row. Now copy the row 1 (non-header) to the new blank row and then clear the content from row 1. Then use this blank row 1 as the header row. Place all of your header content in it and use Table > Layout > Properties > Row to mark Row 1 as the header row. Delete your other former header row and test.

The behavior should be when a row is pushed to the second page, a non-editable header row appears at the top of the page.

Now regarding adding blank paragraph marks to expand row size... you don't need to do that. In the Table > Layout > Properties > Row tab there is a setting for Row Height. You can set this individually by row or select all rows and change them all at once.

Hope this helps

The medialess package says "For use only with a new Microsoft Office Ready PC", but I'm not sure if there is a technical or legal rationale, since I have the official backup discs.

dan at it associates say:

Hello jelliott3,

You asked, "Can I install a 2nd instance of this suite on my existing desktop...?

A:  No.

The Medialess Licensing Kit (MLK) license for Office 2007 is valid only on the computer onto which the license is first installed and the license is not permitted to be moved to any other computer.

See paragraph 2 of your Software Licensing Terms, licensed devices.  http://www.microsoft.com/downloads/en/details.aspx?FamilyID=4285d6f7-dfdd-44a6-a21d-8e9899082b15

Full retail (as opposed to upgrade retail) licenses for Office 2007 suites are permitted to be simultaneously installed on the license holder's primary computer plus the license holder's portable (aka laptop) computer.  There is one exception, Office Home and Student 2007, whose retail license permits the product to be installed simultaneously on up to three computers in the same houselhold when such computers are not used in revenue generating activity.
