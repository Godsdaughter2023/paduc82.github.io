I believe I have resolved this.

It appears that the requirements for this command actually include .net framework 4.5 or above

Just for what it's worth (and for the lurkers) you are NOT obliged to use datasheet view for a Subform. That's what the wizard gives you, but you can build the subform from scratch (in either Single or Continuous view), or change the DefaultView property of the subform that the wizard builds.

I'd try to avoid basing data entry/editing forms on a query joining a one-to-many relationship, because it can get really confusing when you're adding records to both tables in the same operation.

Go to C:\Users\<username>\AppData\Local\Packages\Microsoft.ZuneMusic_[random string/hash]\LocalState\Database

I only had one folder listed, and the name was random.  Likely another hash of something user-specific.

Rename (or delete) the folder, and the database will be rebuilt on the next program run.
