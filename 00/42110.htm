Scheduled task running twice with two Event Triggers? - Microsoft Community

himselfv say:

Scheduled task running twice with two Event Triggers?

Does anyone else has this problem? If you have a task in Task Scheduler which has two Event triggers and one fires, and the task is not configured as "only one instance", Task Scheduler spawns two instances of the task instead of one.

Disable the second (non-firing) trigger, and only one instance is spawned.

Disable the first (firing) trigger instead and none is spawned.

Example task:

    <?xml version="1.0" encoding="UTF-16"?>
    <Task version="1.2" xmlns="http://schemas.microsoft.com/windows/2004/02/mit/task">
      <RegistrationInfo>
        <Date>2016-06-13T20:06:03.7929126</Date>
        <Author>whatever</Author>
        <URI>Task name</URI>
      </RegistrationInfo>
      <Triggers>
        <EventTrigger>
          <Enabled>true</Enabled>
          <Subscription>&lt;QueryList&gt;&lt;Query Id="0" Path="Microsoft-Windows-TaskScheduler/Operational"&gt;&lt;Select Path="Microsoft-Windows-TaskScheduler/Operational"&gt;*[System[Provider[@Name='Microsoft-Windows-TaskScheduler'] and EventID=103]]&lt;/Select&gt;&lt;/Query&gt;&lt;/QueryList&gt;</Subscription>
        </EventTrigger>
        <EventTrigger>
          <Enabled>true</Enabled>
          <Subscription>&lt;QueryList&gt;&lt;Query Id="0" Path="Microsoft-Windows-TaskScheduler/Operational"&gt;&lt;Select Path="Microsoft-Windows-TaskScheduler/Operational"&gt;*[System[EventID=201]]&lt;/Select&gt;&lt;Suppress Path="Microsoft-Windows-TaskScheduler/Operational"&gt;*[EventData[Data[@Name='ResultCode'] and (Data="0")]]&lt;/Suppress&gt;&lt;/Query&gt;&lt;/QueryList&gt;</Subscription>
        </EventTrigger>
      </Triggers>
    [...]
      <Settings>
        <MultipleInstancesPolicy>Parallel</MultipleInstancesPolicy>
    [...]
      </Settings>
      <Actions Context="Author">
        <Exec>
          <Command>cmd.exe</Command>
        </Exec>
      </Actions>
    </Task>

This should run each time any task finishes with non-zero error code. The first EventTrigger is a catch-all net for another event which should not fire. Disable it and run any failing task, this task starts once. Enable it, run any failing task, this task starts twice, even though the conditions for the first EventTrigger are not met.
