Hi CraigSharp3484,

From my side, I can reproduce the issue. The original primary SMTP address always comes back after removing it.

After further investigation, we found it’s an expected behavior. If you consider the feature great important, we welcome and encourage you to submit feedback to our relevant team. Actually, many features come directly from your constructive comments. We appreciate your time and efforts.

As a workaround, we suggest you delete the shared mailbox and then it’s possible to shift the SMTP address to a DG (Distribution group).

Before deleting the shared mailbox, it is recommended that you use the Outlook for Windows to export the shared mailbox’s data to a pst file. Note: you need to grant a shared mailbox’s Full Access permission to another normal user mailbox so that when the normal user mailbox is set up in Outlook for Windows, the shared mailbox also shows up.

Some do it that way but very few.  Most have a code based on your user name.  So in your device, under Owner Information, you provide that name and they generate a code for you.  It works on another device as long as the name is the same.
