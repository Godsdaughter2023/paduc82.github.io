bernie deitrick say:

    1) Copy the code below.
    2) Right-Click the sheet tab of interest.
    3) Select "View Code"
    4) Paste the code into the window that appears.
    5) Save the file as a macro-enabled .xlsm file.

Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Cells.Count > 1 Then Exit Sub
    If Target.Value = "" Then Exit Sub
    If Intersect(Target, Range("A:A")) Is Nothing Then Exit Sub

    Application.EnableEvents = False
    With Cells(Target.Row, "B")
        .Value = .Value + Target.Value
    End With
    Application.EnableEvents = True
End Sub

kirk b. - microsoft support say:  

Hi pklocke,

 

I think I follow what you're looking to do, but from the little I found in playing around with the process, it may not be worth it to try to do this programmatically rather than using the built-in functionality that allows you to transfer your tables to SharePoint.  I was able to get the following code to work for me . . . sorta:

 

Dim objTable As Object

 

For Each objTable In CurrentDb.TableDefs

   If Left(objTable.Name, 4) <> "Msys" Then

       DoCmd.TransferDatabase acExport, "WSS", Me.txtSitePath, acTable, objTable.Name, objTable.Name

       DoCmd.DeleteObject acTable, objTable.Name

       DoCmd.TransferSharePointList acLinkSharePointList, Me.txtSitePath, objTable.Name, , objTable.Name

   End If

Next objTable

 

You have to use TransferDatabase to export as there is no export TransferType argument for the TransferSharePointList method. 

 

Note that in the code I had to test for whether or not the table I'm exporting is a system table (that's the check to see if it starts with Msys), and that's just the tip of the iceberg of potential problems that might crop up with this method.  Other issues I encountered that I'm didn’t address in the sample code here were relationships keeping a table from being deleted and linked tables being exported to SharePoint. There of course could be others that I'm not thinking of or didn't see as well. 

 

I should note that the code above is running on the click event of a button I created on a form and that form also includes a text box control called txtSitePath where the user would enter his or her site name. 

 

These links might be handy as you work on this issue:

 

http://msdn.microsoft.com/en-us/library/aa662942(office.11).aspx - Covers programmatically exporting to a SharePoint list

http://msdn.microsoft.com/en-us/library/bb225970(v=office.12).aspx - Talks about the TransferSharePointList method

 

I hope that helps at least get you started on this. Good luck!

 

Kirk Beller

karl timmermans [outlook mvp 2012-2018] say:
Does this happen to involve an IMAP account? If yes, and if your contact/calendar data was stored in folders with the added label of "This Computer Only" in the folder description, then the data in those folders is permanently lost unless you manually backed up (exported) those folders to a PST file.

If you have a copy of the original .ost file containing those folders, you would need a 3rd party OST2PST file conversion program to recover those folders.

To prevent the loss of contact/calendar folder in the future, some articles that may be of interest
