john w. vinson say:

Sure; use the DateDiff() function:

LOS: DateDiff("d", [EntryDate], Date())

will return the number of days (well, to be exact the number of midnights that have passed) since the EntryDate.

You can use "m" instead of "d" to calculate the number of months, but this may not give the desired result - it will calculate the number of calendar months, so an EntryDate of July 1 and an ExitDate of July 30 will return 0, while an EntryDate of June 30 and an ExitDate of July 1 will be shown as a full month. Calculating the number of days and dividing by 30 would be good if all months were 30 days long but of course they're not; would that be close enough though, for reporting purposes?

scottgem say:

When you add a column to a query the format is:

alias: expression

Alias is the label you want to give the result to refer to or display. So in Graham's example LOS is the alias. The colon (:) indicates the end of the alias and the start of the expression. The syntax for the DateDiff() function is type,first date, second date. Type is a letter code indicating the period you want to return. For example d = days, m = months etc. You can find the full list in Access VBA Help. The second and third arguments are the dates, usually referencing names of fields that contain the date values.

john w. vinson say:

Create a new query based on your table.

Select whichever fields from the table you want to display on the form or on the report.

Go to the first vacant Field cell, after the last field you've added; or select a column and use Insert... Column to create a new blank field.

In the vacant cell in the Field row (not the criteria row) type:

LOS: DateDiff("d", [Arrival Date], NZ([Exit Date], Date()))

The Date() function returns today's date from the system clock; the NZ() function will give you the Exit Date if there is one, and today's date if there isn't; the DateDiff() function will calculate the number of days between the Arrival Date and whatever date the NZ() function returns.

Your confusion seems to be that you're trying to use the Criteria. The purpose of the Criteria row is to let you apply a filter, or conditions to decide whether or not to retrieve a record. That's not what you want to do here! Instead you want to create a new, calculated field that doesn't exist in your table; to do so you need to enter an expression in the Fields row of the query grid.

 

John W. Vinson/MVP
