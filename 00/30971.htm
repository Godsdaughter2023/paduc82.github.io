shinmila h say:

Hi CussingNChattanooga,

 

The issue could be due to various factors such as faulty or incompatible component from a software provider other than Microsoft.

 

Here are few things you can try to fix the problem.

 

Method 1: Uninstall third party codec or plug-in

 

Do you have any third party audio or video codec, Directshow filters or plug-in installed?

 

Incompatibilities are known to exist with some of the components in these codec packs that can cause serious playback issues in Windows Media Player and other multimedia programs.

 

In case you have installed any third party codec or plug-in for Windows Media Player, please uninstall them to check if they’re causing the problem.

 

Method 2: Run Windows Media Player Library troubleshooter

 

The troubleshooter makes sure Windows Media Player is correctly installed on your computer, and checks to see whether your Player Library is corrupted.

 

For details, see: Open the Windows Media Player Library troubleshooter

 

Method 3: Run Windows Media Player Settings troubleshooter

 

This troubleshooter looks for issues related to playing audio and video files in the player by checking its configuration and network settings.

 

Refer: Open the Windows Media Player Settings troubleshooter

 

Method 4: Uninstall and reinstall WMP (Windows Media Player).

 

Step 1: Uninstalling Windows Media Player:

 

a)   Go to Start and in the search type  Turn Windows features On or Off.

b)   Click on  "Turn Windows features On or Off".

c)    Browse to the Media Features and clear the checkbox for Windows Media Player.

d)    Restart the computer.

 

Note: This will return the player to its installation default. As such you’ll lose all the settings for the player.

 

Step 2: Reinstalling the Windows Media Player:

 

a)   Repeat the steps a) and b) from Method 4 Step 1.

b)   Browse to the Media Features and place a check mark in front of Windows Media Player.

c)    Restart the Computer.

 

That should be it.

 

Regards,

Shinmila H - Microsoft Support

poppy yang msft say:

Hi Mina,

 

Please let your administrator check if this option is disabled by following the steps below:

1. Connect to the skype for business powershell. (you need to install the skype for business powershell module and windows powershell 3.0).

2. Run the following command to check your conference policy.

Get-CsOnlineUser -Identity user@domain.onmicrosoft.com | select ConferencingPolicy 

3. Then check if the attribute enableappdesktopsharing of the conference policy is set to none.

get-csconferencingpolicy -Identity BposSAllModality

note: please replace the BposSAllModality of your own policy.

 

To enable desktop sharing for specific users using windows powershell, please involve your admin follow the steps below:

  1. run the following command, you can get some built-in policies in which the attribute enableappdesktopsharing is set to Desktop. 
    get-csconferencingpolicy | where-object {$_.enableappdesktopsharing -eq "Desktop"}
  2. you can choose one policy which meets your request and run the command to grant the policy to one user. please replace the policy name and the user identity when running the command.
    grant-csconferencingpolicy -policyname BposSAllModality -identity user@domain.com
