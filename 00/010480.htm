don cuthbert say:

You began with a non-standard partitioning layout for Windows 10. Normally, there is a Recovery partition at the beginning of the disk, which is also the System partition. During an OS upgrade, a new Recovery partition is created by first shrinking C:. This becomes the one in use, but only until the option to Go back to the previous version expires.  The original Recovery partition still has the previous OS build winre.wim file. Once the rollback option is done, Windows deletes the temporary Recovery partition, leaving unallocated disk space. The contents are relocated to the normal Recovery partition location.

Since your disk does not have this standard partition, winre.wim is located at C:\Recovery\WindowsRE\ instead. If you run ReAgentc /info it will show this as the active location of Windows Recovery Environment. Windows RE runs as a RAM disk from winre.wim at this location if Windows cannot boot, or if you restart with SHIFT-Restart. Having Windows RE in it's own partition is a more reliable setup, in case something on C: partition becomes corrupted. Then you can still boot to the Repair Environment, which remains very stable and is rarely written to.

john sr wilson say:
This should make any footer invisible and remove textboxes in the bottom inch or so.

Sub zapem()
Dim osld As Slide
Dim L As Long
Dim sngMargin As Single
sngMargin = ActivePresentation.PageSetup.SlideHeight - 72
For Each osld In ActivePresentation.Slides
osld.HeadersFooters.Footer.Visible = False
For L = osld.Shapes.Count To 1 Step -1
If osld.Shapes(L).Type = msoTextBox And osld.Shapes(L).TextFrame.HasText Then
If osld.Shapes(L).TextFrame.TextRange.BoundTop > sngMargin Then _
osld.Shapes(L).Delete
End If
Next L
Next osld
End Sub
