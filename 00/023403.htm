Changed Left Margin and Table Of Contents Outside Margins - Microsoft Community

nessa1124 say:

Changed Left Margin and Table Of Contents Outside Margins

I changed margin in finished document to 1.5 inch for left and 1 inch for all others. With the change my Table of Contents now falls outside of the right margin....how do I adjust?  I have to keep the 1.5 margin for submittal.  Thnx in advance.
suzanne s. barnhill say:
When you first insert a TOC, Word sets a right-aligned tab stop for the page numbers based on the current margin settings. If you later change the margins, Word doesn't adjust the tab stop, so you have two choices:
  1. Delete and reinsert the TOC (easier if the TOC is not complex).
  2. Manually move the tab stops. This is most easily done using the ruler. Just drag the tab stop from out in the margin back to where the margin/indent marker is. Since the TOC styles are set to update automatically, you need do this for only one entry at each level, and it will be applied to all.
One caveat with (2): If you are using the default 1.25" margins (in Word 2003 and earlier), Word sets the tab stop, not at 6" as you might expect, but at 5.99" (or some slightly smaller amount--I don't remember exactly). I'm not sure what the reason for this is, but I haven't experienced any difficulties with setting it to 6" or 6.5" or whatever the actual margin width is.

dr. strangelove say: You should be able to uninstall from the “Add/Remove Programs”, or “Programs and Features”, depending on your OS.  If that fails, you can use this procedure provided by forum contributor Aiscer.

1.     Download the MSE installer file from the website and save the file on the Desktop.  This will allow you to find it easily after the download is done.

2.     Now press the Windows key (or Start key) and R both at the same time. This will open the RUN dialog box.

3.     Next, delete/clear any entries inside RUN, then drag the MSE installer file into the RUN dialog box.

4.     Depending on your Operating System, after dragging the file into RUN, it should appear as one of the three below:

·         C:\Documents and Settings\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-xp.exe

·         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-vista-win7.exe

·         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-amd64fre-en-us-vista-win7.exe

5.     Now put a SPACE at the end of the line, followed by a  /U  as seen below:

·         C:\Documents and Settings\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-xp.exe /U

·         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-vista-win7.exe /U

·         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-amd64fre-en-us-vista-win7.exe /U

6.     Click OK.

7.     Restart your system after the uninstall.

don varnau say:

> "When using outlook 11"

- - - - -

Does this mean that you only see the message when accessing email at outlook.com / https://outlook.live.com/owa/ ?

If you see the message on other sites...

If this is happening only on one (or a few) website(s) there really is (by Internet Explorer standards) a long running script on the page and IE is doing you a favor by providing the "Stop script" option. Click on the Stop script button.

Again, if only a site or two are affected...

It might break a desirable feature on the site, but placing a site in Restricted Sites will disable JavaScript and the script messages will stop.

Internet Options> Security> Restricted Sites...

If you're seeing the message on all or nearly all sites, a browser add-on might be causing it.

Note: The Google Toolbar has caused script problems for some users.

Start Internet Explorer without add-ons from Start> Programs> Accessories> System tools> Internet Explorer (no add-ons)
or

from Start> Run (or Windows logo key + r) type in iexplore.exe -extoff

(iexplore.exe<space>-extoff) then hit Enter.

If the problem goes away, an add-on is causing it.

To determine which add-on is causing the problem see this old (but still relevant) article:

IEBlog  Troubleshooting and Internet Explorer’s (No Add-ons) Mode:
http://blogs.msdn.com/ie/archive/2006/07/25/678113.aspx

Another option: Other browsers like Edge, Chrome and Firefox are more forgiving of these script problems. Consider using another browser on affected sites.
