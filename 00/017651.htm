Automatic page number x of y appears as only 1 page upon opening file - Microsoft Community

satch dobrey say:

Automatic page number x of y appears as only 1 page upon opening file

I have 4 pages and use the automatic page numbering Page x of y in the header.  Upon opening the file it always  says Page 1 of 1 in the header.  If I scroll down and back up it will change to the correct Page 1 of 4.  This is very frustrating  as I will be passing this on to others and they will print before scrolling up and down and then they will think I made a mistake and throw it away. This is just one example but it happens every time I have multiple pages.  Why is this happening?  It appears to be a fatal flaw. If it is fixed in a later version I should receive that version free.  Any help greatly appreciated,
bob jones aka: cybertaz say:

That's due to the fact that the Page X of Y AutoText item inserts a NUMPAGES Field] which, when placed in Headers & Footers, do not automatically update on opening a document... Regular page numbering (Insert> Page Numbers) inserts only a PAGE Field.

It isn't noticed because that field starts with a specified number, whereas NUMPAGES depends on a count which is done when the document repaginates. Some type of action needs to occur, such as scrolling or pressing F9 to Update Fields as well as any number of other actions. All versions of Word -- PC or Mac [at least through Office 2010/2011] -- work the same way. I'm not sure about Office 2013 as I've never used it, but I'd be surprised if it were different.

Although I believe an auto_open macro can be used in other versions of the program it won't do any good in Word 2008 because there is no support for VBA. Whether anything could be done with Apple Script I don't know, but that would be useless if any potential recipients are using a different operating system.

FWIW:

  • The page numbering will automatically update when the document is printed,
  • The page number & total number of pages are both displayed in the Status Bar
  • If these are 'finalized' documents, type the total # of pages rather than using the Field
  • If the recipients are to do nothing but print the documents it may help to introduce them to the concept of Desktop Printing or send them PDFs rather than Word documents.
