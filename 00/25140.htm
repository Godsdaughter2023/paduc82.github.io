If you have ticked the check box "Always create backup copy" in the Save section of Office button > Word Options > Advanced, Word will create a backup copy with extension .wbk each time you save a document. If you're sure you won't need the backup copies, you can safely delete them.

graham mayor say:

I agree with Suzanne - you should always run with Word creating backups. It will be worth the trouble of removing them when you find you need the content of one of them ... and you will need the content of one of them sooner or later.

Note that the backups are not true backups, but the previously saved versions. If you want true backups you need a macro -http://www.gmayor.com/automatically_backup.htm .

As for removing them. The following macro will remove all the backup files in a chosen folder that do not have matching document files.http://www.gmayor.com/installing_macro.htm

Option Explicit
Public Function MyFileExists(Path As String) As Boolean
' Requires a reference to the Microsoft Scripting _
  Runtime (scrrun.dll) object library
Dim objFSO As Object
Set objFSO = CreateObject("Scripting.FileSystemObject")
Select Case objFSO.FileExists(Path)
    Case True
        MyFileExists = True
    Case False
        MyFileExists = False
End Select
Set objFSO = Nothing
End Function

Sub DeleteBackups()
Dim strFilename As String
Dim strPath As String
Dim strDoc As String
Dim strDel As String
Dim vExt As Variant
Dim i As Long
Dim fDialog As FileDialog
Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)
With fDialog
    .Title = "Select folder and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
               "List Folder Contents"
        Exit Sub
    End If
    strPath = fDialog.SelectedItems.Item(1)
    If Right(strPath, 1) <> "\" _
       Then strPath = strPath + "\"
End With
If Left(strPath, 1) = Chr(34) Then
    strPath = Mid(strPath, 2, Len(strPath) - 2)
End If
strFilename = Dir$(strPath & "*.wbk")
While Len(strFilename) <> 0
    vExt = Array(".doc", ".docx", ".docm", ".dot", ".dotx", ".dotm")
    For i = LBound(vExt) To UBound(vExt)
        strDoc = Mid(strFilename, 11)
        strDoc = Left(strDoc, Len(strDoc) - 4) & vExt(i)
        If MyFileExists(strPath & strDoc) Then
            strDel = MsgBox("Delete backup file for " & vbCr & _
                            strDoc, vbYesNoCancel, _
                            "Delete unwanted Word backup files")
            If strDel = vbCancel Then Exit Sub
            If strDel = vbYes Then Kill strFilename
        End If
    Next i
    strFilename = Dir$()
Wend
End Sub

 I usually use a sum function, but the default it count.  Can I change the default so that my values with start with a sum function?

j.e. mcgimpsey say: I don't know of any way to change the default - XL makes its best guess based on the type of data. Usually numeric fields will get Sum, non-numeric fields will usually get Count. If you have numeric data that is by default getting Count, you might check to make sure that there aren't values stored as Text.

However, if you just want to speed up the change, here's one way using a (very) quick and dirty macro. You could customize a toolbar and put the macro buttons on it, or give them keyboard shortcuts to speed things up even more:

    Public Sub PivotCountToSum()
        Dim ptTable As PivotTable
        Dim pfField As PivotField
        
        On Error Resume Next
        Set ptTable = ActiveSheet.PivotTables(1)
        On Error GoTo 0
        If Not ptTable Is Nothing Then
            With ptTable.PivotFields
                For i = 1 To .Count
                    On Error Resume Next
                    Set pfField = .Item("Count of " & .Item(i).Caption)
                    On Error GoTo 0
                    If Not pfField Is Nothing Then
                        pfField.Function = xlSum
                        Set pfField = Nothing
                    End If
                Next i
            End With
        End If
    End Sub

and to switch it/them back to Count:

    Public Sub PivotSumToCount()
        Dim ptTable As PivotTable
        Dim pfField As PivotField
        
        On Error Resume Next
        Set ptTable = ActiveSheet.PivotTables(1)
        On Error GoTo 0
        If Not ptTable Is Nothing Then
            With ptTable.PivotFields
                For i = 1 To .Count
                    On Error Resume Next
                    Set pfField = .Item("Sum of " & .Item(i).Caption)
                    On Error GoTo 0
                    If Not pfField Is Nothing Then
                        pfField.Function = xlCount
                        Set pfField = Nothing
                    End If
                Next i
            End With
        End If
    End Sub
