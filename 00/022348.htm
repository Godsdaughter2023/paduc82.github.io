jeeped say:

What you are describing is the precise definition of a circular reference. If you reference a range of cells with a function like AVERAGE() that may include blanks, filling those blanks in after-the-fact with something that references the AVERAGE formula is circular.

Larry wrote:

You must use an intermediate column to record the scores of participating members, with empty cells for absent members.

[EDIT] Unless you enable Iterative Calculation mode, which I do not recommend.  If you insist on avoiding intermediate columns, I would use a VBA macro to replace the missing data.

If you want to average on a week-by-week basis, put

If you want literally what you wrote -- the value for the absent members (the same value for all) is 1 less than the average of the participating members -- put the following formula into C2 and copy down and across through T33:

=IF(CC2<>"", CC22, AVERAGE(CC$2:CC$33)-1)

Be careful with the "$":  that is CC$2, not $CC$2, for example.

But note that the value for the absent members will not be AVERAGE(C2:C33)-1.  Is that okay?

On the other hand, do you mean:  when you are done substituting a value (the same value) for the absent members, the value for the absent members average for all members (participants plus absentees)?

That is, the value of the absent members will be AVERAGE(C2:C33)-1 when you are done substituting.

If that is what you want, put the following formula into C2 and copy down and across through T33:

=IF(CC2<>"", CC2, ( SUM(CC$2:CC$33)-COUNTBLANK(CC$2:CC$33) ) / COUNT(CC$2:CC$33) - 1)

Again, be careful with the "$":  that is CC$2, not $CC$2, for example.

-----

If you want to average over all weeks, not week-by-week, the formulas above become:

=IF(CC2<>"", CC22, AVERAGE($CC$2:$TT$33)-1)

and

=IF(CC2<>"", CC2, ( SUM($CC$2:$TT$33)-COUNTBLANK($CC$2:$TT$33) ) / COUNT($CC$2:$TT$33) - 1)

-----

If you are interested, the derivation of the latter formula is as follows....

We want the following, where p=#participants, a=#absentees:

( Sigma(x[i], i=1,...,p) + a*(avg-1) ) / (p+a) = avg

Sigma(x[i], i=1,...,p) + a*avg - a = (p+a)*avg

Sigma(x[i], i=1,...,p) - a = (p+a)*avg - a*avg = avg*(p+a-a) = avg*p

( Sigma(x[i], i=1,...,p) - a ) / p = avg

where Sigma(...) = SUM(range), a = COUNTBLANK(range), and p = COUNT(range).

sylvie liang msft say:

Hi David DawesYJ,

 

To create a Skype for Business meeting on Outlook, both the clients of Skype for Business and Outlook should be signed in with the same account, which means the account should have licenses. But a shared mailbox doesn’t have any licenses. Therefore, a shared mailbox can’t be signed into the Outlook client or Skype for Business client. So, you can’t create Skype for Business meetings from shared mailbox calendar.

 

As a workaround you can assign a license to the shared mailbox just like normal users. Go to Office 365 admin center and the shared mailbox is listed in the active users, you can edit the licenses.

 

After assigning the license, based on my tests, sign into the Skype for Business client with the shared mailbox account, then the user should be able to set up Skype for Business meetings via Outlook Web App (OWA).

 

I understand this workaround may require an extra license. Other community members are welcome to share any experience if you have the similar situation. Meantime, we will be very grateful if you vote through the link below to help us improve our products and features:
