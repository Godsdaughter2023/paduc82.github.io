pc-learner say:

The correct way to do this is:

1. Let the current Windows version update to Windows 10 x86 32bit.

2. Now you are running on Windows 10, verify that your Windows 10 is activated.  If you are using a Microsoft account to log in, also go online and verify the Windows 10 license has been associated with your account.

3. Back up your PC and gather your applications and their keys - you will need to reinstall your data and programs.

5. Now you are ready to do a clean install of Windows 10 x64 64 bit.  Download the media and create a bootable USB or DVD, and run a Custom install keeping nothing.  When the install asks for a product key, skip it - the install will be activated automatically later using the MS stored key.

6. Reinstall everything.

You cannot do an upgrade install from x86 to x64 on any version of Windows, and you need to do an upgrade from your current eligible version of Windows to Windows 10 in order to receive the free, permanent, license.  Only after that will you have a Windows 10 license which will be used to  activate any subsequent clean install or reinstall.

To clarify the last sentence of pc-learner's very clear instructions, you will not have a Windows 10 product key to use for the x64 clean install.  You won't need one.

How do I write a divide code for Access 2007? - Microsoft Community

jayjoe7424 say:

How do I write a divide code for Access 2007?

I want to divide two fields, then multiply by 100 and have them appear in another field. I would like to multiply by different numbers depending on the values in field 1 and field 2. Say 3,000 divided by 3,000,000 or 30,000 divided by 30,000,000 then I would like to multiply by 100, but then if its 3,000 divided by 30,000,000 I would like to multiply by 1,000, etc (each depends on the amount of zeros being divided). How do I write a normal divide code for field 1 and field 2 to appear in field 3? However, not all the records are filled in, so what would be the code under the condition that field 1 is blank but field 2 is not? I would like it to appear as 0. Also vice versa. Lastly, how would I write the code where there is a 0 in field 2? This is undefined, I would like this to appear as 0 also.

john w. vinson say:

You shouldn't do this in a Table but it's not hard in a Query. I don't understand your logic of how to determine the multiplier, but to do the division you can use the IIF() and NZ() function to check for NULL and convert NULL to zero respectively. In a Query, type\

IIF(NZ([Field2]) = 0, 0, NZ([Field1]) / [Field2]))*100

The IIF function takes three arguments - a true/false expression, something to do if it's TRUE, something to do if it's FALSE. So if Field2 is NULL the NZ function will return 0; or if it is 0 already, the IIF will return the second argument - 0. If Field2 is not null or zero, NZ([Field1]) will return 0 if Field1 is NULL; you'll then divide Field1 by Field2 (or 0 by Field2 if Field1 is NULL). This calculation will then be multiplied by 100.

John W. Vinson/MVP
