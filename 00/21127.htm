cortig say:
Uninstalling and reinstalling won't do a thing for this problem.
I would:
- Rename the old/restored identity by adding some sort of a tag to the
identity folder name (e.g.: Main Identity (Old))
- place it in the same ~/Documents/Microsoft User Data/Office 2011
Identities folder as the active, working one
- Launch Outlook pressing the option key down to get the the Database Utility
- Select the old one to load it in Outlook.
- Let Outlook rebuild the index.
- if the messages are there, export them to a .old archive
- launch Outlook pressing the Option key down to reselect the active,
good identity
- import the .olm
