xentr say:

Fix for MTP driver problems when connecting a device. (code 31)

1) Connect your device.

2) Open Device Manager.
    ( Press Windows key +R and type "devmgmt.msc" and press enter.)

4) Go to Other devices -> Unknows MTP device -> Properties -> Driver -> Details

5) All the files displayed should have a digital sign by Microsoft.

Note: This path is an example: C:\Windows\System32\DRIVERS\UMDF\WpdMtpDr.dll

6) Navigate to the following path: "C:\Windows\System32\DRIVERS\UMDF"

7) Claim the ownership of the following file: "WpdMtpDr.dll"
-> Select properties
-> Click the security tab
-> Click advanced
-> Click the owner tab
-> Click edit
-> Select your username and click ok
-> Again click ok
-> Click the security tab -> Select your username and click edit
-> Check all the options for your username and click ok

Note: If you are prompted for an administrator password, enter the password and click ok.

8) Download the official .dll file from this link: "http://www.dllme.com/dll/files/wpdmtpdr_dll.html" and save it to your desktop.

9) Drag and drop the file you downloaded into the map UMDF and select "Replace all".

10) Unplug your device, close the map, plugin your device and it should work .

This issue can occur if the Security Center service is not configured to run under the Local Service account but the dependent services are configured to run under the Local Service account.

To resolve this issue, make sure the Security Center Service and all its dependencies are configured to run under the Local Service account. Try these steps:
Step 1: Set the security center and its dependencies services to run under Local Service account.
a. Click Start, type services.msc and hit enter.
b. Double click on Security Center, click Log On tab.
c. Select "This account" click Browse button, type Local Service and click ok.
d. Type the password and confirm the password and click ok.

Note: Perform the same steps for Windows Management Instrumentation service.
