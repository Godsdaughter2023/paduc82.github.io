As I mentioned above, two accounts are all independent, so, it won't influence each other.

davidchiao say:

I found the solution.

It turned out if you need to connect SharePoint to Outlook calendar, you need to

  • Configure your SharePoint server (2013) as kerberos instead of NTLM
  • Configure your SharePoint server (2013) to enable anonymous access
  • Configure your SharePoint 2013 site (and/or list, document library) to allow anonymous access

Here is a reference link I found on Googling:

http://www.sharepoint-journey.com/how-to-enable-anonymous-access-in-sharepoint-2013.html

After I did this, IT WORKED!!!! I was so happy to find this solution because this issue impact my entire organization.

Hope it is useful for you as well.

Hi,

 

Let’s start the Office application in safe mode and verify whether issue re-occurs. Follow the below mentioned steps to start Office application in safe mode:

* Hold Windows key + R.
* Copy and paste, or type the following command in the Open box, and then press Enter.
Outlook /safe

Note: There is space between Outlook and '/'. Replace Outlook with WinWord to start Word in safe mode.
