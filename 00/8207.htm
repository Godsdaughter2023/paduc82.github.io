sheenali_msft say:

hi simranjit,

end users in this context means site owners, or users that has the appropriate permission to add an app to the site.. members of a site do not have this permission by default.

the specific permission needed for a user to be able to add an app is the ‘manage web site’ permission.



by default, only users with the full control permission level has this permission.

if you would like to give other permission levels this specific permission, you can do so by going to:
1.    site settings
2.    under users and permissions > site permissions
3.    on the permissions ribbon, click permission levels.
4.    choose the permission level that you would like to add this permission to.
5.    tick the ‘manage web site’ box, and submit.

additionally, you mentioned that you would like your users to be able to request an app. if you would like your users to request an app, you will need to set the option 'should end users be able to get apps from the marketplace?' to no.

if it's set to yes, users will be able to directly add an app to a site without first requesting it.

jllatham say:

I've tested this with several different ways of creating the Sources:= string and it appears to me that your code should work, UNLESS you've misspelled something in the path/workbookname/worksheet name portion of it in your code.  One thing you didn't mention is the specific error message you are getting when you try to run the code.  If I introduce an error to the filename or worksheet name, I get a "Cannot open consolidation source file: [filename.xls]!SheetName" error.

IMPORTANT QUESTION: But I'm a little confused about the workbooks in use.  Are you consolidating data in the same workbook, or is the data to be consolidated really in another workbook?  You're working on a worksheet named 'Mat by Cust' and trying to get your mycount value from that sheet, then you are referencing a worksheet of the same name in a workbook (same or different?) and setting up a range of R19C3:RmycountC14.  R19C3 is cell C19!  and that's where you're trying to place the results ( Range("C19").select).  So if the data to be consolidated isn't actually in another workbook, you may be setting up something resembling a circular reference.

Also, why the Selection.ClearContents immediately after the .Consolidate statement?  You'd be "erasing" the first value returned by the .Consolidate statement.

BTW, earlier I said you need to make sure mycount was at least 15, I should have said 19 since mycount is actually referring to a row number and not a column number.  I believe I'd rewrite your code like this:

Sub consolidateMyData()
'
Dim mycount As String
Dim lastRow As Long
'  
'count number of lines
    Sheets("Mat by Cust").Select
