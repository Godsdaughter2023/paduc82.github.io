shelisaj say:

Data, Import pipe delimited text

I am working on a task that calls for importing a pipe delimited text file into Excel and saving it as a CSV file.
Unfortunately, when importing the file into Excel and saving it as a CSV file, the "pipes" are converted to commas after the import.

I think this is more of a bug than a question.
ron coderre say: You're right. When Excel imports the pipe-delimited file, the pipes are converted to "cell breaks".
Excel does not have a native method to create pipe-delimited files.

To create one after you're done editing,
try this:

• File.Save_As...Save As Type: Text (tab delimited)
...Set the file name and the destination
...Click: Save

Next, replace all TAB characters with pipe characters:
• From Windows Explorer...Open the edited text file
• Use your mouse to select one of the TAB characters
• Right-click the character...Select: Copy
• Edit.Replace
...Find what: (Right-click in this field...select: Paste)
...Replace with: |
...Click: Replace All
• Save the file

Is that something you can work with?
