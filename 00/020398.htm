    Another case of "fix me" with 0x80310052 - how I broke it and - Microsoft Community

    c_mendoza say:

    Another case of "fix me" with 0x80310052 - how I broke it and fixed it.

    My Win10 boot partition was getting a little low on space after updating to build 10525. There was an unused Linux partition next door, so I decided to wipe that and expand C: into the filled space.

    Because of the old dual boot, this disk had been using grub for its bootloader, but that had to go.

    As a quick fix (this was the mistake), I made a new boot setup with EasyBCD. This allowed Windows to boot and run just fine on the reconfigured drive, but that pesky 0x80310052  kept popping up for the insider update checks.

    There are lots of solutions suggested on this forum. Removing the PC from the Windows site, switching over to a local account, re-joining insider, yadda yadda, seemed to work, but didn't persist across a reboot. The option to restore to an earlier version was unavailable, so I seemed to be stuck.

    Apparently, the problem was ultimately that new boot loader I installed with EasyBCD, some option didn't agree with the insider updates. (This particular error code can point to BCD issues when Bitlocker throws it, and since I had just messed with exactly that, it was a useful lead in this case.)

    The steps I took that ultimately cured the error were as follows. Some may be superfluous but it worked.

    First, I opened an admin cmd session and ran

    bcdboot C:\Windows

    This in itself did not solve the problem, so I made up a Windows 10 installer USB stick, booted from that, hit shift-F10 to get a command prompt and ran:

    bootrec /fixmbr

    bootrec /fixboot

    bootrec /scanos

    ...and then closed out the installer and rebooted.

    On returning to regular Windows, the "fix me" error is gone, back to receiving insider updates.

    Applications from the Windows Marketplace for Mobile may have have licenses, but won't have registration codes. That statement is seemingly ambiguous unless the meaning of "license" and "registration code" is  made clear. A license in this context is something that enables some one to run a software title. A registration code is a specific type of license that usually has to be entered by the user.  For Windows Phone applications that use licenses the license will automatically be entered into the phone during the installation process.  You as a user are never have to see them.

    There are some applications that are available both inside of the MArketplace and directly from the publisher of the software.  If you acquire the software directly from the publisher then you may need to enter registration information. But the same title from the marketplace won't require registration information.

    Hi Hasitha,

    Per this article Restrictions and limitations when you sync files and folders, 20000 items is not the sync limitation of the new OneDrive sync client.

    Although SharePoint Online can store 30 million documents per library, the sync performance of OneDrive may start to decline when you store more than 100,000 files in a single OneDrive for Business site or team site library.

    About this issue, please follow these steps and let me know the result.

    1. Make sure you have installed the latest Windows Updates.

    2. Open administrative Command Prompt (press Win + S to search for and right click Command Prompt, choose "Run as administrator") and run regsvr32 KERNELBASE.dll command to re-register this DLL file.   

    3. Reset OneDrive (Be aware that resetting OneDrive re-syncs all your OneDrive files, which could take some time depending on how many files you’re syncing.)

     a. Press the Windows key + R.
     

     b. In the Run window, enter:
    %localappdata%\Microsoft\OneDrive\onedrive.exe /reset
     

     c. Click OK.
     

     d. Make sure the OneDrive icon in the notification area of the taskbar disappears and then re-appears after a minute or two. Here’s how to check: Right-click the white OneDrive cloud icon in the notification area, at the far right of the taskbar.
    (You might need to click the "Show hidden icons" arrow next to the notification area to see the OneDrive icon.)
     

     e. If the OneDrive icon doesn’t re-appear after a few minutes, open the Run window again and enter: %localappdata%\Microsoft\OneDrive\onedrive.exe
