Vista/Windows 7 –

C:\Windows.OLD\ServiceProfiles\NetworkService\AppData\Roaming\Microsoft\SoftwareLicensing\Tokens.data

C:\Windows.Old\ProgramData\Microsoft\OfficeSoftwareProtectionPlatform\Tokens.DAT (if you did an in-place Windows Upgrade)

marshall barton say:

Why, oh why, is it so hard to make this work?   Why is VBA not smart enough to simply run a query that work just fine in the GUI?

Because VBA is NOT Access.  VBA is a programming language that can be used in any Office program, but can only utilize the objects provided by the libraries you selected.  OTOH, Access has lots of stuff built into it that can and does run queries, including resolving any query parameter references.

When you use VBA to open a recordset, you are using the database engine Jet/Ace library, DAO, not Access.  The database engine does not know anything about higher level programs that may or may not have prompting or form reference resolution capabilities.  Therefore, you have to use VBA code to resolve the form references before opening the query object's recordset.  The usual code utilizes Eval to retrieve the value of the form references:

Dim db As DAO.Database

Dim qdf As DAO.QueryDef

Dim prm As DAO.Parameter

Dim rs As DAO.Recordset

Set db = CurrentDb()

Set qdf = db.QueryDefs("name of a saved query")

For Each prm In qdf.Parameters

    prm = Eval(prm.Name)

Next prm

Set rs = qdf.OpenRecordset(dbopenDynaset)

ken sheridan say: To establish a recordset from a query with parameters you should firstly return a reference to the querydef object, evaluate the querydef object's parameters, and then call the OpenRecordset method of the querydef object, e.g.

    Dim rst As DAO.Recordset
    Dim qdf As DAO.QueryDef
    Dim prm As DAO.Parameter

    Set qdf = CurrentDb.QueryDefs("NameOfQuery")
    
    For Each prm In qdf.Parameters
        prm = Eval(prm.Name)
    Next prm
    
    Set rst = qdf.OpenRecordset

Parameters in a query called by another query are members of the latter query's Parameters collection, so the above will work in this context.

Using:

Outlook 2013 (2 accounts in Outlook, 1x IMAP/SMTP and 1x Microsoft Exchange)

Word 2013

Excel 2013

I used the Step-by-step Mail Merge Wizard with a Word file and an Excel list with emails and names. 

In the last step..

...when I selected Mail Format "Plain Text" - emails were sent from my default IMAP/SMTP Account, however they were NOT saved in the sent folder.

...when I selected Mail Format "HTML" - emails were sent from my NON-default Microsoft Exchange Account, and they WERE  saved in the sent folder of the Microsoft Exchange Account.

Solution: 

Outlook - Options - Mails - Send messages - CHECK "Always use the default account when composing new messages"

Now Emails sent both in Plain Text and HTML format are sent from the default IMAP/SMTP Account and are saved in the sent folder :)

Self-taught... started learning in 1981 though, so I have been at it for awhile now.<g>

 

However, you need to replace the code I posted with the following, otherwise you will lose control of your workbook for an extended amount of time if you clear all the cells on the monitored sheets... it will eventually come back to life, but it takes a loooong time to do so... the code below should avoid the problem.

 

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
   Dim Cell As Range
   If Sh.Name = "Sheet1" Then
     If Not Intersect(Target, Columns("N:R")) Is Nothing And _
        Not Intersect(Sh.UsedRange, Columns("N:R")) Is Nothing Then
       For Each Cell In Intersect(Target, Intersect(Sh.UsedRange, Columns("N:R")))
         If Len(Cell.Value) < 7 And IsNumeric(Cell.Value) Then
           Cell.NumberFormat = String(Len(Int(Cell.Value)), "0") & _
                               Left(".", -(Cell.Value < 100000)) & _
                               String(6 - Len(Int(Cell.Value)), "0")
         Else
           Cell.NumberFormat = "General"
         End If
       Next
     End If
   ElseIf Sh.Name = "Sheet2" Then
     If Not Intersect(Target, Range("B10:B14,E10:E14,O13:R13")) Is Nothing And _
        Not Intersect(Sh.UsedRange, Range("B10:B14,E10:E14,O13:R13")) Is Nothing Then
       For Each Cell In Intersect(Target, Range("B10:B14,E10:E14,O13:R13"))
         If Len(Cell.Value) < 7 And IsNumeric(Cell.Value) Then
           Cell.NumberFormat = String(Len(Int(Cell.Value)), "0") & _
                               Left(".", -(Cell.Value < 100000)) & _
                               String(6 - Len(Int(Cell.Value)), "0")
         Else
           Cell.NumberFormat = "General"
         End If
       Next
     End If
   End If
 End Sub
