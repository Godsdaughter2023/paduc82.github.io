Sub Send_Outlook_Mail()

Dim OutApp As Object
    Dim OutMail As Object
    Dim strbody As String

    If ActiveWorkbook.Path <> "" Then
        Set OutApp = CreateObject("Outlook.Application")
        Set OutMail = OutApp.CreateItem(0)

        strbody = "<font size=""3"" face=""Calibri"">" & _
                  "Todd,<br><br>" & _
                  "Please see the link below for the TL Daily CHEF Report. Let me know if you have any questions about the information and/or arrangement.<br><B>" & _

ken sheridan say: This is a very common mistake.  It's what's known in the jargon of the database relational model as 'encoding data as column headings'.  As Doug and Duane have explained data should only be stored in one way, as explicit values at column positions in rows in tables.  This is known as the 'Information Principle' and was Codd's Rule #1 when he introduced the relational model for databases back in 1970.

As this issue crops up so frequently I've put together a little demo file which illustrates how to correct this type of design fault.  You'll find it as UnencodeColumns.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

While my file only uses four Boolean (Yes/No) columns as an example, and could be corrected by means of a UNION query, the solution it illustrates executes a series of SQL statements in a loop in a VBA procedure called by a button on a form.  This avoids the problem Doug drew attention to of there being too many columns for a single UNION query to deal with.  My demo makes certain assumptions about the naming of the columns, but even if these are not met it would not be difficult to amend the code accordingly.
