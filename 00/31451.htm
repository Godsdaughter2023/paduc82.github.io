Windows 10 Hibernate problems since Creators update - Microsoft Community

davex4 say:

Windows 10 Hibernate problems since Creators update

Hi. I have an Acer Aspire 5560 laptop that has been upgraded from Windows 7 to Windows 10 Pro V10.0.15063. (32 bit.) When I used Windows 7, hibernate worked fine. Similarly, when I upgraded to Windows 10 hibernate worked OK. However, since installing the Creators update, hibernate does not work all of the time.

What is somewhat puzzling is that 60% of the time I hibernate, and when I resume working, everything is as should be. All the files I left open and web pages are there - i.e. hibernate has worked correctly. The other 40% of the time when I resume from hibernate my laptop boots up from scratch as though I have previously shut down which I did not. Obviously I have then lost everything I left open. Also when this happens my wifi comes up as disabled which I have to reset through the laptop's keyboard, and my default browser is minimized. 

I have searched through the community and the web and notice that many people have had a similar problem, but not where hibernate sometimes works and other times it does not. Nothing has changed on my laptop insomuch I have not installed any new software/hardware (or updates), just Windows updates.

I have tried unchecking fast start up, checked that all drivers are up to date, ran power troubleshooter, changed all hibernate settings to 'do nothing' shutdown, reboot and changed do nothing back to hibernate, but I still the problems persists. 

I will be grateful if someone can help me and restore my faith in Microsoft. 

 

davex4 say:

Well, no one from Microsoft seems to be interested in this problem anymore, but I have found a solution.

If you read my original post, I stated that when Hibernation failed and my laptop rebooted instead, then my wifi would be disabled and I had to reset it. So I went to Device Manager / Opened  Network Adapters / right click on Wireless Driver / Properties / Power Management / and unchecked 'allow the computer to turn off this device to save power'. 

I did the same to the Ethernet Driver.

This improved the situation, however hibernate would still fail at times, but nowhere near as bad as before. I then searched for updated drivers, but the maker of my laptop only had old drivers available. So, I went to the manufacturer of the drivers site, in my case Broadcom, and downloaded their Wireless Lan and Ethernet drivers.

Once downloaded, instead of executing the software, I right clicked on the download and selected 'Troubleshoot compatibility'. I let Windows do it thing and then clicked on apply fixes. Made sure 'allow the computer to turn of this device to save power' was unchecked on the new drivers and hibernation now works correctly.

Hi,

 

There may be a need for setting drive jumper depending on the type of IDE drive. To use it externally you may have to change its jumper setting.

 

Remove the jumper(s) completely from the IDE drive and check if that makes any difference.

 

Also if feasible, try and connect the IDE drive to another computer running Windows 7 via USB port and check if the issue is specific to your computer or Windows 7 in general.

 

With respect to the phantom drive issue,

 

-      Have you installed any CD/DVD burning software lately?

-      How long have you had this issue?

 

Some third-party CD/DVD burning software install virtual drives to mount image files.

 

Phantom drives also seem to appear if you have Microsoft Office 2010 beta installed on your computer.

 

See Phantom drive appeared once Office 2010 Beta Installed for information.

rajesh govind say:

 

Hi Jose,

 

This could be due to the incompatible drivers installed for the Intel(R) PRO/Wireless 3945ABG network card on the computer. I would suggest you to update the drivers and check if that help.

 

You may run the Intel Driver Update Utility Tool to find the latest driver updates for your computer and then install all the driver updates and check if that helps. You may run this tool from this link:

 

Intel Driver Update Utility Tool

 

If that does help, then follow instructions given below:

 

Uninstall and Reinstall the Intel(R) PRO/Wireless 3945ABG Network Card Drivers

 

Follow these steps to uninstall and reinstall the driver:

 

Step 1: Uninstall the Intel(R) PRO/Wireless 3945ABG Network Card Drivers

 

a)       Press “Windows Logo” + “X” keys on the keyboard.

b)       Click on “Device Manager” from that list.

c)       Search for the network card device from the device list, right click on it and then select “Uninstall”. On the un-installation window, if you have an option: “Delete the driver software for this device” you may select that and then remove the corrupted drivers from the computer.

d)       Follow the on-screen instructions to complete it and the restart the computer if prompted.

 

After the restart, you may again open the Device Manager and then click on the “Scan for hardware changes” button at the top of the Device Manager window and check if Windows 8.1 detects the network card device and installs an appropriate driver for it automatically. If that happens, then check if it resolves the issue. If the issue persists, then go with the next step.

 

Step 2: Check the Windows updates for any latest driver updates.

 

Check in Windows updates if any driver updates are available. If so, install the same and check if it helps.

 

a)       Press “Windows Logo” + “W” keys from the keyboard.

b)       Type “Updates” in the search bar and click on the icon “Windows Update” from the search result.

c)       On the “Windows Update” window, click on the link: “Check for updates” from the left hand side of the window to receive a fresh list of updates.

d)       If the updates are available, then install all the updates listed there.

 

If Windows Update does not offer the latest Windows 8.1 driver, then try the next step.

 

Step 3: Update Driver Software from the Manufacturer’s website:

 

You can download the drivers from this link:

 

Intel® PROSet/Wireless Software for Windows 8.1

 

Hope this is helpful. Please get back to us with the results.
