joeu2004 say:

Generally, cell formats determine how Excel displays a numeric value, not how it

Excel's interpretation of dates depends on the date formats specified in the Regional and Languages control panel.

So if the R&L short date format is m/d/yyyy, you must enter dates in that form (although Excel also permits m/d/yy as well as other variations).  Then it is displayed as dd/mm/yyyy, according to your Custom format.

For another example, if we format a cell to display time as [m]:ss (min and secs), then enter 1:23, Excel displays 83:00 because 1:23 is interpreted as h:m.

-----

PS....  Excel does sometimes change cell formats despite our explicit settings.  Sometimes it is a defect (e.g. when re-opening a file).  Sometimes it is just Excel's eccentric nature (changing Currency to Accounting when re-entering a formula using a financial function).

But in your case, I suspect the cell format per se is not changed (select the cell, right-click and click Format Cells > Number (tab)).  Instead, I suspect your dd is larger than 12, the max mm.  You can confirm the data type by using the ISTEXT function.

hpholm say:

Abi99 in Nokia Discussions figured out 059V992 is a SWAP product code, caused by an unofficial or incompetent repair of the phone. These product codes should never appear on a customer phone, since they do not receive further OS or firmware updates. An old post, different device, still valid:
http://discussions.nokia.com/t5/Software-Updates/Nokia-Belle-availability-information/m-p/1299655#M110031

The official answer is to take the phone to a Nokia/Microsoft Care Point or an authorized workshop, and have them change the product code to a proper, local product code and refurbish the phone.
Fees will apply.

The unofficial answer is to refurbish the phone yourself with the firmware for a local product code, using internal Care tools not intended for customer use. Then the phone will behave normally and will receive future updates Over The Air like everybody else with that product code.
The product code embedded in the phone is not changed during this, so the user (and a future owner) is in trouble if he decides to use the recovery tools officially available for customers - these tools will load the firmware for the real product code of the phone again.
