How to copy a table style - Microsoft Community

nislander14 say:

How to copy a table style

Does anyone know how you apply the style of one table to another table? This should be very easy to do, but I've never found out how to do it in 30 years of using Word for Mac!
john korchok say:

If your table is styled using one of the default Microsoft styles, then float your mouse over the table style that is highlighted in the Table Styles gallery. The style name will pop up. Make a note of that, then select the other table and apply the same style from the style gallery.

If you have modified a default style and want to apply that to other tables, then you have more work to do. Word is very poor at creating a new table style from a sample. Select the source table, then choose New Table Style... from the Table Style Gallery dropdown. Give the style a name and click on OK. A new Custom row appears  at the top of the Table Style Gallery with your new style in it.

Select that style and apply it to a new table to see what formatting Word missed in creating the style. Then select Modify Table Style... from the bottom of the Table Style Gallery dropdown. 

Initially, this dialog has Apply formatting to: set to Whole table. Concentrate on the Fill and Border settings, since most of the font formatting just doesn't work here.

Now change the Apply formatting to: dropdown to additional features your table style may have. If you have an option to turn a header row on, select Header row from the dropdown and format the fill and border for that.

Repeat these formatting steps for First Column and/or banded rows, if your style includes then, observing the effect on the table style preview. When you're done, click on OK. The table style is resaved and the style on the new table updated. Check that clicking the Table Design options turns the Header Row and other features on and off. It may take a few rounds of modifying the style to get it working as you want. Then apply the style to other tables.

brian tillman [outlook mvp 2007-2019] say:

While most people replace the existing Office version when they install a new one, it's not mandatory that you uninstall Office 2003 before installing Office 2007.  The installation process should detect the older version and ask if you want to install the new version instead of or along side the old version.  Personally, I don't see a reason to keep the older version if you intend to install the newer.  If you intend to use Outlook, however, that's different.  Only one version of Outlook can be installed at any one time and it's best to uninstall the older Outlook before installing the new one, even though the installation procedure should find the older one and remove it first.
