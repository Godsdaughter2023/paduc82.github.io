Calculate length of record that contains NULL field values - Microsoft Community

pat dools say:

Calculate length of record that contains NULL field values

Good morning,

How do you calculate the length of a record when any number of fields within that record can contain NULL values? The 'Len' function returns NULL for the record length if ANY of the 50+ fields that make up my record are NULL.

I found this piece of code that looks like it might be helpful, but it does not run in a standard Access query window:

 

RecLength: Len(Trim(Nz(.Fields("Link"), "")))

 

I am not a VBA developer, and I am hoping there is a quick way to get this done within my query.  What sort of statement can I write that will yield the value of the length of a record when some of the fields in the record will have NULL values?

john p spencer say:

Instead of the plus sign as the concatenate operator, use the ampersand.

Len(Field1 + Field2) will return null if either field is null

Len(Field1 & Field2) will return a value if either field has characters.  If both are null, the expression will return Null.  If you want to ensure zero when both fields are null, then concatenate in a zero-length string.

LEN("" & Field1 & Field2)  will return zero if both fields are null.

Note that the Len function converts all values to a string.  It does not give you the length of a record, it does give you the length of the values.  (For instance, numbers, dates, Yes/No fields all have a defined size and take up a specific amount of bits in a record).

snapperrapper say:

Catch FileDialog save event

Hi

My code:

Public Function saveDoc(docName As String)

  With fd
     .InitialFileName = "S:\" & docName
    .Show
    .Execute
  End With

End Function

Thanks in advance.

/SnapperRapper

   
jay freedman say:
.Display method that the FileDialog object lacks. That's important for what you want to do.

The three methods involved in using Dialog objects are

  • The .Show method displays the dialog, and if the user clicks the OK button, it immediately executes the dialog's action (in this case, the save).
  • The .Execute method does not display the dialog, but just executes the action. It assumes that all the other properties of the dialog object have already been given appropriate values.
  • The .Display method displays the dialog box, but does nothing else. When the user clicks OK or Cancel, control returns to the macro, which can examine the properties in the object and do whatever work is needed before calling the .Execute method. That's what you want to do.

            

Public Function saveDoc(docName As String)
  Dim fd As Dialog
  Dim initName As String

  initName = "S:\" & docName

  Set fd = Dialogs(wdDialogFileSaveAs)
  With fd
    .Name = initName
    .Display

    If .Name <> initName Then
        MsgBox "You changed the file name."
        ' do whatever you need here,  e.g.
        If Right(.Name, 4) <> "docx" Then
            ' modify the dialog object to change
            ' what will happen when .Execute runs
            .Name = "S:\error.docx"
        End If
    End If

    ' finally, do the actual save with whatever
    ' is in the memory-resident properties of
    ' the dialog object
    .Execute
    MsgBox "Saved as " & ActiveDocument.FullName
  End With

End Function
