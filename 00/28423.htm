tom van stiphout say:

I'm afraid you have a bad database design. There is no need to store this data in the parent form (or its table); it is already in the subform (and its table).

This is a classic many-to-many situation: Each contractor bids on many jobs, and each job is bid on by many contractors. The table in the middle of this relation might be called tblBids, with a primary key over ContractorID + JobID, and some additional fields including BidAmount and BidSelected.

LowestBid would NOT be in this table: It is a calculated field and as such does not belong in any table, but is calculated in a query (select Min(BidAmount) ...).

Get the database design right, and many other parts will fall into place. Do this incorrectly, and the app will never really work.

-Tom. Microsoft Access MVP

Present the same footnote on footers of several pages - Microsoft Community

antonpuz say:

Present the same footnote on footers of several pages

Hi,

The thing is I want the explanation at the bottom of the page to appear on all pages where I use a footnote.

charles kenyon say:

Hi,

I am using footnotes as part of my Word 2016 (on mac, version 16.18 (181014)) document, I use cross-references to reuse the footnotes on several pages.

The thing is I want the explanation at the bottom of the page to appear on all pages where I use a footnote.

Example:

footnote #4 is created on pages #12

on the bottom of page #12 there is an explanation for it: 4 this is a super important component

I used cross-reference to footnote #4 on page #18

I want the explanation: "4 this is a super important component" to appear on the bottom of page #18 also.

Thanks,

Anton.

The article you reference shows how to use the same footnote number to refer to the same footnote on multiple pages and have it be semi-dynamic. It does not put the same footnote as a footnote on multiple pages, which is what I think you want.

I do not believe there is any way to do exactly what you want in the Mac version and only a work-around in the Windows version. If the document is created or edited in the Windows version you could use a mapped content control for your footnote text and then repeat that content control at the bottom of page 18 as a new footnote. The advantage of the mapped content control is that a change made in either place is made in all places using that content control.

I believe you can do mapped content controls on the Mac, but do not have a Mac on which to verify this. The thing is, in the Windows version there are fairly easy ways of doing this that do not involve you writing XML coding. I do not know that those tools exist on the Mac.

If you have access to a Windows version of Word (version 2007 or later) you can use the Document Property content controls or one of several Add-Ins that let you make custom mapped content controls. The following are currently only for Windows versions.

  • Repeating Data Using Document Properties Content Controls and Other Mapped Content Controls
  • Insert Content Control Add-In
  • Content Control Tools

There are other methods of repeating data in multiple places in Word, many of which work on the Mac. They are outlined in this other article by Greg Maxey: Repeating Data (Populating Multiple Like Fields)

For the most part, those are one-way. That is, there is one spot where you type the information and it is repeated automatically in other places. They require more maintenance than do the mapped controls. Again, this would be a new footnote, with a new footnote number but with repeated content.

How to create RMA # from year and serial number

Using Excel 2003 on XP, I want to create a 5-digit number for a Return Material Authorization (RMA) that will take the form YY-SSS where YY are the last two digits of the year (example 13) and SSS is a number starting 001 and advancing by 1 for each RMA issued. Whenever the year changes (for example to 2014) I want the SSS part of the number to reset to 001. So if, by December 31, 2013 we have assigned up to RMA 13-099, and RMA assigned on January 1, 2014 would be assigned the number 14-001.

 

I have played around with function RIGHT to generate the YY part, although the  conflicts between cell formats of Date and Text are challenging; but the SSS has me stymied. I tried starting with cell D1=001 (but had to format is as text to get the leading zeroes to display, then for D2 I tried using IF. With YY in Column C I entered the function into D2 as IF(c2=c1,D1+1,001), but D2 then displayed the formula rather than the result.

 

Any suggestions as to how to do this?

 

Thanks.

 

Mike

hansv mvp say:

Let's say you have dates in B1 and down.

Format column C with the custom number format 00.

In C1, enter the formula =MOD(YEAR(B1),100) and fill down.

Format column D with the custom number format 000.

Enter 1 in D1, and =IF(C2=C1,D1+1,1) in D2, then fill down.

In E1, enter =TEXT(C1,"00")&"-"&TEXT(D1,"000") and fill down.
