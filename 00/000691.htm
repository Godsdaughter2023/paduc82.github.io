Outlook often rebuilds main identity database - why? I have - Microsoft Community

rob white63 say:

Outlook often rebuilds main identity database - why? I have several versions of main ID - how many do I need?

Lately, I keep getting prompted that the Main Identity database needs to be rebuilt.  When I look at the identities folders I have several folders dating back to Office 2004, 2008, and now 2011.  The main identity folder is in the 2004 folder, all others are backups.  I even found an old Entourage file (main identity.rge), which I moved to the trash.  What can I do to stop the rebuilds, and can I delete the 04 & 08 folders?
dianeoforegon say: 1)  If you are using Outlook 2011, you don't need any of the other Identity folders. You can delete Office 2004 Identities and Office 2008 Identities folders in the Microsoft User Data folder unless you need to import any previous Identities into Outlook. Drag to trash to delete.

 ~/Documents/Microsoft User Data/

 Also delete Microsoft Office 2004, Microsoft Office 2008, Microsoft AutoUpdate in Applications. Drag to trash and delete. Next delete all Microsoft cache files in this location:



Delete the Fonts Disabled folder in your User's Library folder. You Office 2004 fonts were moved here when you installed Office 2008.


2)  If rebuilding does not fix after attempting, you need to export your data and import into a new Identity. Further attempts could just make it worse.

You don't mention if you have upgraded to 14.2.1. If not please upgrade AFTER you do the following and BEFORE you create the new Identity.


Make a duplicate first of your Identity in the Finder.

  • If you have Exchange and IMAP accounts, delete the accounts so they delete the cache files in Outlook.
  • Under File > Export select the default action to export all items as .olm file. Save to Desktop.
  • Rename your Microsoft User Data folder to Microsoft User Data OLD in Documents.
  • If you have not upgraded to 14.2.1, do this now. Be sure to restart after upgrading.
  • Open Login Items in System Preferences > Accounts. Delete any Microsoft Database daemons.
  • After upgrading, and restarting open the Database Utility.
    /Applications/Microsoft Office 2011/Office/Microsoft Database Utility.app
  • Create your new Identity and set it as default. How to create a new Identity or switch to a new Identity in Outlook 2011
  • Add your Exchange and IMAP accounts. See notes below about Inbox.
  • Import your .olm file. Under File > Import select to import the .olm file **
  • Note the Microsoft User Data folder will look different. Some folders and files have been moved to the Library/Application Support folder.

** Your Imported .olm file will show up under an imported folder in the folder list. You'll need to select the messages in the default folders (Inbox, Drafts, Sent, Deleted Items) and move to the local folders. You can drag any custom folder to it's correct location. You can delete the imported folder structure after you move all the messages.

You'll also need to move the contacts, events, tasks, notes to their local folders.

The new Outlook Identity will not know you have downloaded the old messages and will download all again. Before you enter your POP account in Outlook, log into your account with your browser. Move all the older mail out of the Inbox. Outlook only downloads from the Inbox. Leave only those new messages you have not downloaded.

Prune your Inbox. It's best to keep these folders as clean as possible. Archive messages after reading to another folder. Any folder with more than 2GB of data is going to be difficult to rebuild and can cause database rebuild failure.

Do not make subfolders under the Inbox for Exchange accounts. Also not advised for IMAP.

The more messages you have in the Inbox the more likely you will have sync issues.


The Inbox syncs more frequently than any other and should be kept as clean as possible. (this is the number one cause of problems with sync). Do NOT make subfolders under the Inbox.

Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.
