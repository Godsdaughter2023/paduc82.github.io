Page numbers that show up on every page or multiple pages are always in the header or footer (even if they look like they are elsewhere on the page).

Page numbers are fields. They do not have to be inside shapes or frames. Despite what I just said, they do not have to be in a header or footer; it is putting them in the header or footer that gets them showing up on multiple pages. Otherwise, they just show up on the page where you insert the field.

Page Numbers in Word 2007-2016

So, to really get an idea about what is going on with page numbers, you will need to get at least some handle on headers and footers.

Bottom line: consider just entering the field without the shape. Then it will stay put.

Excel remove password multiple files on network - Microsoft Community

syrkrasi say:

Excel remove password multiple files on network

I need to remove the workbook open password (which is the same) to multiple files on our network.  I found a code dated back in 2014 by Graham Mayor; however, I am unable to hide or set automatic update all links.  I tried many UpdateLink= or := options to no prevail.

Please help

Thanks 

Here is the code G. Mayor posted

Sub RemovePasswords()
Dim xlBook As Workbook
Dim strFilename As String
Const fPath As String = "C:\Path\"        'The folder to process, must end with "\"
Const strPassword As String = "openpassword"        'case sensitive
Const strEditPassword As String = "editpassword" 'If no password use ""
    strFilename = Dir$(fPath & "*.xls") 'will open xls & xlsx etc
    While Len(strFilename) <> 0
        Application.DisplayAlerts = False
        Set xlBook = Workbooks.Open(Filename:=fPath & strFilename, _
                                    Password:=strPassword, _
                                    WriteResPassword:=strEditPassword)
        xlBook.SaveAs Filename:=fPath & strFilename, _
                      Password:="", _
                      WriteResPassword:="", _
                      CreateBackup:=True
        xlBook.Close 0
        Application.DisplayAlerts = True
        strFilename = Dir$()
    Wend
End Sub

prashantkumar96 say:

First of all, disabling Windows Update is not a great idea. Windows also require updates as much as you antivirus do. So my suggestion would be to change settings to 'Check and let me choose' rather than disabling it and install updates when you have free time, but do install them.

About your question, I assume the method you are following is to create a BAT file with this command and run it as scheduled task. This will always create a popup. I haven't tried it but you can try this:

Instead of putting this command in task "C:\WD_update.bat" (where WD_update.bat is the file in which this command is saved), type this command:

"C:\Program Files\Microsoft Security Client\msseces.exe" -update -hide

This will not show a popup. But a problem I detected with it is that after running it, MSE UI will not start(means MSE will work in background but it will not show its window). To counter this, you can edit the trigger to end this task after 5 min or so.

Like I said, I haven't tried this so I can't ensure success. I will try it and share the results. You also try it.
