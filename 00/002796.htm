hansv mvp say:

Excel expects that the delimiter character in the csv file is the same as the list separator character that you can set in the Region control panel.

On systems that use a point as decimal separator (numbers look like 3.14, for example), the comma is generally used as list separator.

On systems that use a comma as decimal separator (numbers look like 3,14), the semi-colon ; is generally used as list separator.

If you open a csv file that uses comma as delimiter on a system that uses semi-colon as list separator, Excel will not recognize the comma as delimiter because it expects semi-colons. As a result, all data will be in the first column, separated by commas.

There are two workarounds:

  • Set the list separator to the character used as delimiter in the csv file. This is done in the Region control panel, or:
  • Leave the system settings unchanged. After opening the csv file, select the first column. On the Data tab of the ribbon, click Text to Columns. Select Delimited, then click Next. Select the delimiter, then click Finish.
