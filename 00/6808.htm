a. user say:

This technique really just uses the { PAGE } number as a sequence number, which is why it's only easy to make it work in simple situations (e.g. where you don't need to create a Table of Contents that references page numbers)

That said, 

You can get i, ii, iii by changing the fields to be as follows:

{ ={ PAGE }*2-1 \*roman }
and
{ ={ PAGE }*2 \*roman }

(for I, II, III you would need

{ ={ PAGE }*2-1 \*Roman }
and
{ ={ PAGE }*2 \*Roman }
)

You can restart the page number at any section break via, e.g., Insert->Page Numbers...->Format, but depending on exactly where you need the break, you might also have to change the fields a bit in each section.

pollardie say:

You can navigate to one such notebook and hit Shift-F9 - which will force a sync and sync should work from now on.

Also you can create a new notebook and move your sections to it.

To avoid the problem:

1) Use a short name for the notebook

2) Avoid using sections groups

3) Use short names for sections

4) Do not have too many sections

It is not the "size" of the notebook that causes problem, it is the total size of the filenames stored on the server.
