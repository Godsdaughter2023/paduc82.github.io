Windows 8.1 Update: How to uninstall broken update packages? - Microsoft Community

gborn say:

Windows 8.1 Update: How to uninstall broken update packages?

Technical Level : Intermediate

Summary

Some users are facing problems duringWindows 8.1 Update install. Especially  update KB2919355 refuses to install correctly. In some cases third party internet security suites are a reason, why the update won't install successful. Other show stoppers are damaged system files, a corrupt component store or missing start menu folders. Also a system-reserved partition, running out of free space, may be a show stopper. 

Damaged system files shall be fixed, executing system file checker. A corrupt component store may be fixed using DISM (see this article). But in some cases, both methods are not successful - damaged Windows files or component store files could not be fixed.

Analyzing several cases (also one component store corruption, I provoked on one of my test systems) indicates, that an unsuccessful Windows 8.1 Update install can broke the system (WinSxS). Partly installed update packages leaves Windows component store (WinSxS) in a corrupt state - which can't be fixed using sfc and dism. 

Further attempts to install update packes (using .msu files) fails with error 0×80070003. Some users are able to use system restore and reset Windows 8.1 to a state before installing Windows 8.1 Update. In most cases the system corruption is also fixed and the system is good for a new update install attempt. 

But in case when no restore point is available or system restore won't work, another approach is required. Uninstalling update packages for Windows 8.1 Update may be an option to fix corrupt system files or a broken component store. Below is a short description, how to uninstall update packes. 


Details

Remove broken update packages 

Open an elevated e command prompt window (see)

or press Windows-key + X - then click on Command Prompt (Admin).

Click Yes to the User Account Control messagebox that appears.

In the elevated command prompt window execute the following commands:

dism /online /remove-package /packagename:[package-name]

dism /online /cleanup-image /startcomponentcleanup

Below are the package-name for KB2919355 - be aware, that there a two distinct file names for 32 and 64 bit Windows. 

Package_for_KB2919355~31bf3856ad364e35~x86~~6.3.1.14  (32 Bit)

Package_for_KB2919355~31bf3856ad364e35~amd64~~6.3.1.14 (64 Bit)

So the commands to remove KB2919355 from a 64 bit Windows are written as:

dism /online /remove-package /packagename:Package_for_KB2919355~31bf3856ad364e35~amd64~~6.3.1.14

dism /online /cleanup-image /startcomponentcleanup

After executing these commands, a Windows restart is required.

How to detect the package name of an installed update?

The biggest challange will be "how to detect, if a package is installed and how it is named". To check for a package and ask for the package name, use the following command in elevated command prompt window:

DISM /online /get-packages /format:table | findstr KB2919355

where KB2919355 is a placeholder (name) for the requested package. The command given above delivers either the name of KB2919355 update package - or a blank line, if that update isn't installed. In a 2nd step the package name may be used to uninstall the package using the commands given above.

Just repeat these steps for all update packes used in Windows 8.1 Update failure. Here are the package KB numbers for all updates used in Windows 8.1 Update


KB2919442
KB2919355
KB2932046
KB2937592
KB2938439

After uninstalling a package, a restart is required. If everything went well, ssystem file checker and checking component store using DISM (see this article) should report a proper system. If that's the case, try a new attempt to install Windows 8.1 Update again.

Hope this helps to avoid resetting or refreshing Windows 8.1. A more detailled discussion of the issue raised here may be found in the following article (unfortunately only in German).
