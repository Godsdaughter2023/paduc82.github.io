dirk goldgar say:

What is the nature of this "Public Drive"?  Is it a just a shared folder on your local network, or is it provided by a file-sharing service like Dropbox or OneDrive?  You should never open an Access database directly from Dropbox or OneDrive;  they aren't designed for the low level of file I/O that Access requires, and which makes this setup prone to corruption.

If you are sharing the database on your local network, could it possibly be opened by different versions of Access?  That can lead to a down-level user being unable to open the database.  The proper arrangement in that case is a "split" architecture, where there's a separate database "back-end" file on the network containing only the tables, while each user has his own copy of the "front-end" database, which contains all of the other database objects, and linked table to the shared back-end.

Close all office applications and change your default printer through Start > Devices and Printers to another printer. It may be some physical one or One Note or XPS or PDF or anything else. If this is the problem, then you will need to update your printer driver for default printer.

8. Many third party applications also cause the problem like Tuneup Utilities, Abby Finereader, Kaspersky Anti Virus  etc. See if you have any of them installed. Close them one by one and see whether it helps.

9. Sometimes, add-ins can cause problems. To determine, if add-in is a problem start Excel in safe mode and see if problem has gone away. If yes, then start Excel normally and disable add-ins one by one and start Excel again every time you disable an add-in to determine the culprit add-in.
