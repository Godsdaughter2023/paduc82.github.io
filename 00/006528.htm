It is as it should be. Publisher prints the first and last pages on one side of the sheet of paper, the second and next-to-last pages on the other side of the sheet, and so on. The first and last page must be single pages. Look at your print preview to see how it will work.

davidf2 say:

You didn't specify what PDF tool you are using so assuming you are using the built-in Office PDF tool try this:

If your publication is setup as a booklet 5.5 x 8.5. go to Page Design tab > down arrow beside Page Setup >Type 5.5 width, 8.5 height & change Layout type from Booklet to One page per sheet > OK. Go to File tab > Create PDF.

If your newsletter is printed on tabloid (11 x 17) stock, the above is the same except in Page setup you would select letter.

You could also try this: Go to File > Save As > Change file type to PDF > Options > Print Options > confirm or change to One Page per sheet & Page size is Publication Page. You could also go to File > Export as a PDF (or whatever it is called in Pub 2016) and again change the Print Options.

$proxyaddr = "$AdminURI/_vti_bin/UserProfileService.asmx?wsdl"
$UserProfileService= New-WebServiceProxy -Uri $proxyaddr -UseDefaultCredential False
$UserProfileService.Credentials = $creds

# Set variables for authentication cookies
$strAuthCookie = $creds.GetAuthenticationCookie($AdminURI)
$uri = New-Object System.Uri($AdminURI)
$container = New-Object System.Net.CookieContainer
$container.SetCookies($uri, $strAuthCookie)
$UserProfileService.CookieContainer = $container

# Sets the first User profile, at index -1
$UserProfileResult = $UserProfileService.GetUserProfileByIndex(-1)
Write-Host "Starting- This could take a while."
$NumProfiles = $UserProfileService.GetUserProfileCount()
$i = 1

Connect-SPOService -Url $AdminURI -Credential $UserCredential

# As long as the next User profile is NOT the one we started with (at -1)...
While ($UserProfileResult.NextValue -ne -1) 
{
Write-Host "Examining profile $i of $NumProfiles"
# Look for the Personal Space object in the User Profile and retrieve it 
# (PersonalSpace is the name of the path to a user's OneDrive for Business site. Users who have not yet created a  
# OneDrive for Business site might not have this property set.)
$Prop = $UserProfileResult.UserProfile | Where-Object { $_.Name -eq "PersonalSpace" } 
$Url= $Prop.Values[0].Value

# If OneDrive is activated for the user, then set the secondary admin
if ($Url) {
$sitename = $siteURI + $Url
$temp = Set-SPOUser -Site $sitename -LoginName $secondaryadmin -IsSiteCollectionAdmin $false -ErrorAction SilentlyContinue
Write-Host "Added secondary admin to the site $($sitename)" 
}

# And now we check the next profile the same way...
$UserProfileResult = $UserProfileService.GetUserProfileByIndex($UserProfileResult.NextValue)
$i++
}

Experience:
- It seems there is a bit of mis-allocation of resources going on.... As a user, having one single large source of applications to download would tremendously increase the usability of my phone. Podcasts aren't simply songs that you can listen to multiple times, starting over and enjoy each time... they are more akin to a saved radio program. To that extent, when you pause one, it should remember where that particular file was paused, and begin at that location the next time the file is opened... not at the beginning. However, if for whatever reason I do a manual update when I get home, it resets my update schedule. My reoccurring updates should happen everyday at the time I set, and not be effected by manual updates.

Profiles:
- Profiles seems somewhat unclear as to what it effects. By this, I mean that each profile has a series of preferences within the profiles application that will change and are bound to each profile. That is clear enough... but then, changing profiles also changes my preferences for certain other applications... such as bluetooth. This would make sense if profiles was some sort of universal settings exchange framework... but at this point, I'm not sure exactly which applications/settings change with a profile, and which not. It would be quite cool to make it possible to have all apps change.
- Probably should come with an "Airplane mode" as a lot of people seem to ask about this.

That's all I can think of after about 5 days of using this phone, perhaps I'll stop by with more later. Hope this was useful
