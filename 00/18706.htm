jay freedman say:
It is not true that all Word options were stored in Normal.dot, at least as far back as Word 97. A large number of items in the Options dialog were, and are still, stored in the registry. You'll find lots of information in the subkeys under HKEY_CURRENT_USER\Software\Microsoft\Office\12.0. A good reference for some of this is http://word.mvps.org/faqs/customization/DataKeySettings.htm. Starting with Word 2007, as you noted, several other groups of customizations have been moved to separate files.

Also, as you mentioned, some features are shared among the Office programs. These include proofing (spelling & grammar), AutoCorrect (but only unformatted entries), and color scheme.

kim stomboli say:

Just as you replied we figured it out.  It was exactly where you said. 

 

HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Word\Data

 

There were three keys in there.  It was the one called Settings.  We copied it from a user that the template was working properly to a user that the template wasn't working.  Once we did that... the template worked fine.  We are now pushing that setting out to everyone.

ossiemac say:

You can create a delay with the OnTime so that the code waits a specified amunt of time before it starts to run. Where you have "MACRO CODE HERE" in you example, you need to place all of that code in another procedure which I have called "Sub Process()"

See comment against line to adjust the delay to suit.

Sub Refresh_all()
    Dim dteStartTime As Date         'In VBA Date variables also used for Time
    ActiveWorkbook.RefreshAll
    DoEvents
   
    dteStartTime = Now + TimeSerial(0, 0, 5)  'Currently 5 second delay. Adjust to suit
   
    Application.OnTime dteStartTime, _
       Procedure:="Process"    'Process is the name of the sub with code to be processed

End Sub

Sub Process()
   
    'Insert code to be processed in lieu of the MsgBox
    MsgBox "This process started 5 seconds after the OnTime call"

End Sub
