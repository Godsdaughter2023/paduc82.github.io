a. user say:

Hi Richard,

Thank you for posting your query in Microsoft Community.

Rollback is dependent on the availability of the Windows.old folder & other folders, if you delete that, then you may not be offered the option to rollback.
If you receive the error, "We’re sorry, but you can’t go back. The files we need to take you back were removed", then it could mean two things:


a. You are carrying out the rollback operation after a month of upgrading to Windows 10.
b. You have deleted the $Windows.~BT and $Windows.~WS folders required by the system to perform the rollback operation.

But if you rename these folders within 30 days, you may be able to rollback Windows 10 after 30 days limit.

As of now, if you receive this message, you can check if you have any old system restore points available and try to restore your Windows 10 to the oldest point. To perform system restore, refer to the following steps:

a. Press Windows key + R.

b. Type rstrui.exe in the Run dialog box.

c. Follow the onscreen instructions to perform System Restore.

Disclaimer: When you use System Restore to restore the computer to a previous state, programs and updates that you installed are removed.

As is confirmed in the help (I did not make that up) the default 'Outlook' account in Mail cannot be removed.

If yours can then it isn't the default Windows login account and you should see two accounts with the same name (Outlook) in Accounts (on the right pane). One of them must be the Windows Login account aka Microsoft account.

The Windows Login account (or Microsoft account) is an account to receive mail from Microsoft only, which is probably the main reason why it can't be removed. That account can be used to send mail but it will send it on behalf of  'myname@mydomain' (account name), the reply address. That is the email address that you provided when you set up Windows for the first time. From what I gathered this is also known as the Primary Alias if you don't change anything.

So what you see under Outlook (the Microsoft account) is the Primary Alias. Perhaps that changes if you change the account into a 'send and receive' account, I don't know.

This issue could occur if Aero Peek feature is disabled or due to outdated display card drivers.

I would suggest you to try following methods and check if it works.

Method 1:

Many users who have upgraded from Windows 7 to Windows 8, may want to use Aero Peek to quickly preview the desktop. Follow the given below instructions to enable the Aero Peek feature in Windows 8.

Here is how to enable Aero Peek feature in Windows 8:

  1. Switch to desktop, right-click on the taskbar and then click Properties to open Taskbar Properties.
  2. Under Taskbar tab, tick Use Peek to preview the desktop when you move your mouse to the Show Desktop button at the end of the taskbar checkbox to enable the feature.
  3. Click Apply button. You are done!

Note that the Show Desktop button may not appear on the right of the taskbar even after enabling the feature but the feature will work. Just move the mouse cursor to the extreme right of the taskbar to peek the desktop. Also note that the Aero Peek keyboard shortcut (Windows + Spacebar) doesn’t work in Windows 8.

Method 2:

If the above method doesn’t help, I would suggest you to update the display card drivers and check. Follow the steps:

Step 1: Update the Display driver.

Please follow the steps given below to update the display driver.

a. Press Windows key + X when you are at desktop.
b. Select Device Manager.
c. Expand Display adapters.
d. Right-click the display driver and click Update Driver Software.
e. Select Search Automatically for Updated Driver Software.

Step 2: Check if there are any updates for display drivers available in Optional Updates; this might fix the issue. Refer to the following steps.


a. At the Start screen, type Windows Update.
b. Click or tap Settings on the right side of the screen, and then select Install optional updates from the left results.
c. When the Windows Update control panel opens, click on “Check for Updates” on the top left corner to receive a fresh list of updates.

Method 3:

If the issue persists, I would suggest you to uninstall and reinstall the Display driver.

Please follow the steps mentioned below to uninstall the display driver.


a. Press Windows key + X when you are at desktop.
b. Select Device Manager.
c. Expand Display adapters.

d. Right-click the display card driver and click Uninstall.
e. Windows will prompt you to confirm the device’s removal. Click OK to remove the driver.
f. After the un-installation is complete, reboot your computer as soon as possible.

Computer will load the generic display drivers. Now, please go to the manufacturer’s website, download and install the compatible display card drivers for your computer and check.

Also see: Download and install drivers

http://windows.microsoft.com/en-ca/windows-8/all-drivers

I hope this information is helpful.

Please do let us know if you need any further assistance, we will be glad to assist you.

cmdrkeene say:

That's probably an issue with Excel's limitations as an MDU program (Multi-Document User-Interface).  Up until version 2013, opening multiple Excel files would open them into one Excel window container, and the documents themselves were all trapped inside that container.

In other words, Excel was stuck like this from birth until version 2013, which finally changed the interface model.

In 2010, Excel took the first small step by finally being able to show individual document previews in the taskbar, even though it was really just one running "program".  It did this by drawing extra off-screen windows (literally off your screen, they were just drawn with negative coordinates so that you couldn't see them on your pixel plane).  Then the taskbar would see them and show previews.

But it's been buggy and still is even in version 2013.  This new version of Excel makes every single document appear inside it's own container window, so it "feels" more like how Word or Notepad runs.  But Excel is just faking it, it's still using the MDI model underneath (the VBA object model for macros would break otherwise).

So anyway, I doubt the problem you're seeing is truly a Windows 8 bug, or if it is, it's probably because of the shell tricks Excel is using to draw the previews in the first place.
