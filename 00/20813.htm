Is there a way to register the OLEDB driver as 64 bit even though it isnt?

nathan o. - microsoft support say:

Hi Colin,

 

What kind of application are you using and what steps did you go through to ensure it was compiled as a 32 bit application? 

 

To see if the OLEDB provider is working properly you could create a 32 bit UDL file and test it? To do this, you can do the following:

A.      Create a test udl file that we can use by navigating to the root of your c drive.  Within here create a new text document.  You can do this by right clicking within the c drive in windows explorer and choosing new -> text document.  Rename the document it creates so that it has a name of text and a file extension of .udl (ex. test.udl).

B.      Next go to a run command run the following command:  C:\Windows\syswow64\rundll32.exe "C:\Program Files (x86)\Common Files\System\Ole DB\oledb32.dll",OpenDSLFile C:\test.udl

C.      This should open a Data Link Properties dialog.  Within this dialog, click on the provider tab and select “Microsoft Office 12. Access Database Engine OLE DB Provider” from the list and click the Next button.  This should bring you to the Connection tab.

D.     On the connection tab click into the Data Source text box and type in the path to your database (including the database name).  You shouldn’t need to enclose this in double quotes.  (Ex: C:\Temp Databases\Database.accdb).

E.      The username should default to Admin with the blank password option checked.  If this is not the case, you should set it to this.

F.       Click the Test Connection button and see if it tests successfully.

 

If this tests correctly, then the provider should be installed correctly and the problem is most likely related to the connection string within your application and / or it isn’t running as a 32 bit application.

 

Best Regards,

Nathan O.
