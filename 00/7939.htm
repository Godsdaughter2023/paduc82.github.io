Validate Hyperlinks in a word document - Macro Code - Microsoft Community

bluedolphin9 say:

Validate Hyperlinks in a word document - Macro Code

I am seeking help to write a macro code for MS word which will validate all the hyperlinks in the word document and automatically add a comment to the ones which fail. The comment text should say "Link doesn't work" .
jay freedman say:

The following code works. I tested it on a document containing 800 assorted URLs, of which 45 were found to be invalid. The CheckURL function was copied from https://www.datanumen.com/blogs/batch-validate-hyperlinks-word-document-via-vba.

              

Function CheckURL(strURL As String) As Boolean
  Dim objDemand As Object
  Dim varResult As Variant

  On Error GoTo ErrorHandler
  Set objDemand = CreateObject("WinHttp.WinHttpRequest.5.1")

  With objDemand
    .Open "GET", strURL, False
    .Send
    varResult = .StatusText
  End With

  Set objDemand = Nothing

  If varResult = "OK" Then
    CheckURL = True
  Else
    CheckURL = False
  End If

ErrorHandler:
End Function

Sub CommentInvalidLinks()
  Dim objDoc As Document
  Dim objLink As Hyperlink
  Dim strLinkAddress As String
  Dim nLinks As Long
  Dim nChecked As Long

  Set objDoc = ActiveDocument

  With objDoc
    nLinks = .Hyperlinks.Count
    For Each objLink In .Hyperlinks
      nChecked = nChecked + 1
      StatusBar = Format(nChecked / nLinks, "Percent") & " checked"
      strLinkAddress = objLink.Address

      If Not CheckURL(strLinkAddress) Then
        objDoc.Comments.Add Range:=objLink.Range, Text:="Link doesn't work"
      End If

      DoEvents
    Next objLink
  End With
End Sub

The macro could be expanded to do other things, such as putting up a message box to show objDoc.Comments.Count at the end of the processing.

It depends on how you bought Office in the first place.

If you bought it installed directly at the factory, an OEM license, or an online download of a "1-user" license then NOPE. You will have to make M$ happy by buying a new copy.

karl timmermans (outlook mvp 2012-2018) say:
Just to clarify something, what appears in the Outlook Nav Pane (Outlook left pane) is NOT the email account but rather the data files that are configured for your profile.

From the description, this sounds like a PST file that was associated with a POP3 account that was deleted from the profile. You can remove this file by going into Account Settings --> Data Files tab. The exception would be if the PST file has been set to be the "default data file" for the profile which would be where the "default" contact/calendar folders are stored. You would have to set another data file as the default before the file can be removed.

If the other accounts are configured in this profile are IMAP accounts, I would not set an OST file to be the default. Will provide more info on this if this describes your environment.
