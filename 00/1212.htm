albert d. kallal say:

Access 97 to my knowledge was a 32 bit program.


However we are talking about a VERY old bit of software here (on the order of 18 years!!!).


However I have run and installed access 97 on a new windows 7 x64 bit box.


The tips are for such old software:

Turn off any virus software. Those "old" .exe files look and act like virus software!


Turn off windows UAC for the install. This protection will want to MANY times pop up during the install and the VERY older installer has no idea that modifications are being prevented by windows 7.


So you REALLY want to turn off UAC during the install.


Next up is do NOT let the a97 CD auto run.


Open up the CD and right click on the setup.exe and choose "run as administer". In fact in the old days all software did install as a "admin" in sense of things.


Do a custom install - don't install any of the office tools or "extra" options from office. In fact don't install any other part of office 97 for that matter.


So with UAC turned off, virus software turned off, and running the install as an admin I found access 97 installed without issues onto a clean windows 7 box running x64 bit version of windows.


Considering that a machine back then had 16 megs, or 32 megs of ram, and today we have a min of 4 or 8 gigs. The speed at which 97 runs on a modern machine is nothing short of astounding – simply astounding. It boggles the mind how many features Access 97 had back then too!


Going back to that time frame is truly like a episode of "back to the future".


A true gold star belongs to the folks in Redmond that SUCH older software can and still does run today on a brand new box. The  long term ability to run such older software is a rare thing in our industry with phones and tablets that have relative short life spans of only a few years.


Best regards,
Albert D. Kallal (Access MVP)
Edmonton, Alberta Canada
