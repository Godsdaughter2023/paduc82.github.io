marshall barton say:

How you would do that depends on what information you have about each story and how the information is organized.  A properly noramalized data structure would have a table for stories with fields for date of publication, publisher, etc.  Then a second table would be a table of chraracters with fields for the character's characteristics such as height, hair and eye color, personality traits etc.  Then a third table would be used to connect those other two tables by using the primary key fields from the related story and character records.  This third table eould also have fields for the character's role in the story along with anything else that is character/story specific such as age at time of story, function in the story, etc.  With this arrangement your dropdown list would be based on a query that joins the three table to get the desired information you want to display in the list.  While this may sound complicated to the uninitiated, it is the most flexible approach and can be used to deal with almost any kind of question about a story's characters.

OTOH, if your story table includes the character information in each story record, almost anything you want to do beyond just displaying a story's records will require jumping through all kinds of hoops and cause you no end of pain.

So, after that little lecture, what data do you have and how is it structured?

First, I'm going to post code that I think will do what you want it to do and then I will discuss things a little.  The code goes into the worksheet code module just like your code.  I would delete all of your code in it since both codes would run and your's is turning everything red.

The difference between _SelectionChange and _Change is that _SelectionChange 'fires' or 'triggers' each time you select any cell or group of cells on a sheet, while _Change only fires when a value change takes place through user or macro action in a cell.  So with _Change we will only look at the 7 cells of interest and only look at them when a value in them changes.

The code is between the '=========== entries

'======================

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim qtrCell As Range
  Dim crazyCells As Range
  Dim anyCell As Range
  Const myRed = 3
  Const myGreen = 4
  Const myYellow = 6
 
  Set qtrCell = Range("F3")
  Set crazyCells = Range("D14,D36,D96,D108,D115,D126")
  If Not Application.Intersect(Target, crazyCells) Is Nothing Or _
   Not Application.Intersect(Target, qtrCell) Is Nothing Then
    'at least one of the cells of interest changed
    'start by setting all 'crazy' cells red, but hide it from view
    Application.ScreenUpdating = False
    crazyCells.Interior.ColorIndex = myRed
    'determine the Quarter based on F3 value
    'the default property for a Range is .Value
    'so we don't have to specify it like qtrCell.Value
    Select Case qtrCell
      Case Is = 1
        For Each anyCell In crazyCells
          'using Select Case instead of IF ELSE THEN
          Select Case anyCell
            Case Is = 0
              'do nothing, leave red.
            Case Is < 0.25
              anyCell.Interior.ColorIndex = myYellow
            Case Else
              'presumed to be >= .25
              anyCell.Interior.ColorIndex = myGreen
          End Select
        Next ' end anyCell loop
      Case Is = 2
        For Each anyCell In crazyCells
          'use IF THEN ELSE this time
          If anyCell < 0.25 Then
            'do nothing, leave red
          ElseIf anyCell < 0.5 Then
            anyCell.Interior.ColorIndex = myYellow
          Else
            'presumed to be positive and >= .5
            anyCell.Interior.ColorIndex = myGreen
          End If
        Next ' end anyCell loop
     
      Case Is = 3
        For Each anyCell In crazyCells
          Select Case anyCell
            Case Is < 0.5
              'do nothing, leave red
            Case Is < 0.75
              anyCell.Interior.ColorIndex = myYellow
            Case Else
              'assumed to be >=.75
              anyCell.Interior.ColorIndex = myGreen
          End Select
        Next ' end anyCell loop

      Case Is = 4
        For Each anyCell In crazyCells
          Select Case anyCell
            Case Is < 0.75
              'do nothing, leave Red
            Case Is < 1
              anyCell.Interior.ColorIndex = myYellow
            Case Else
              'assumed to be >= 100%
              anyCell.Interior.ColorIndex = myGreen
          End Select
         
        Next ' end anyCell loop

      Case Else
        'not a valid quarter, do nothing
    End Select
  End If ' end of test for change in cell of interest
  'housekeeping and cleanup
  'releases used resources back to the system
  Set qtrCell = Nothing
  Set crazyCells = Nothing
  Set anyCell = Nothing

More discussion:

Please don't be put off or discouraged by any of this - we all start out at a learning stage with misconceptions and misunderstandings about how things work in Excel and VBA.  But with experimenting and coaching and help we all gradually build up a better understanding of these things.

First - within VBA 'Selection' refers to the currently selected cells on a worksheet, so it really probably didn't have much to do with what you wanted to examine and test when the code ran except occassionally in your code.

I find it easier to see what is going on in multiple IF THEN situations to use Select Case, so in 3 out of 4 quarters I used that construct to evaluate the "crazy cells" rather than IF ELSEIF ELSE constructs.  But for the 2nd quarter I did use IF ELSEIF to show you how it would look. 

I did make a one assumption about the percentage entries: that they would not be negative.

And about percentages: even though they might appear as 25.3% when formatted, they are actually decimal values in Excel.  So when you see 25.3% displayed, what is really in the cell is .253 (or some longer decimal like .2532731446).  That is why my tests are against values of .25, .5, .75 and not against 25 50 and 75.

Now to talk about that mysterious Application.Intersect() function.  And I will admit that at times I absolutely fight with this to get the results I need.  Sometimes it just seems difficult to work with.  But read up on it a little, and you will find that basically it is a test to see if two or more ranges of cells 'overlap'.  The way I've used it here is to see if when the value of a cell on the sheet changed if that change took place in one of the 7 cells we need to work with and test.  So the actual testing code only runs when a change in value takes place in either the Quarter cell or one of the 6 percentage cells.

Finally, a little more about the multiple cell reference set up in crazyCells. crazyCells refers to all 6 percentage cells, so you can affect them all at once like I did when I set all of them to red-fill with crazyCells.Interior.ColorIndex = myRed.  But we need to examine each one individually to see what color to change them to later on.  That is done by settin up the For ... Next loops that look like:
For Each anyCell In crazyCells ... Next

The Excel engine is smart enough to know that because we declared anyCell as a range that it is to "pull apart" crazyCells into individual ranges that are contained in it.  So as the loop is executed, anyCell represents each of the many cells that are represented by crazyCells, one at a time.  So within those loops, anyCell represents cells D14, D36, D96, D108, D115 and D126 as the loop progresses.  Hope that makes sense to you.

OH, one more finally: the example that Gary's Student also picked up on:

If Range("F3").Value = 3 and Crazy.Value >=50 <75 Then

You can't combine the test for >=50 <75 like that.  That actually evaluates to
CrazyValue >= 50

which is evaluated to True or False and then that result is compared to < 75.

I think what you wanted at those points was:

If Range("F3").Value 3 And Crazy.Value >=50 And Crazy.Value < 75 Then

but like Gary's student said, Crazy.Value would be looking at the value of all 6 cells represented by Crazy and not an individual cell that was part of the makeup of Crazy.

I do hope these comments help you understand why I wrote the code the way I did and in general with the way things work in VBA when referencing cells (ranges) on worksheets.  Yours was a valiant effort, and I think that with time and continued visits to this forum and other similar ones, you're going to make Excel absolutely dance for you!
