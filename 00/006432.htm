Hello Paulri,

Thank you for choosing Microsoft Community and posting the question.

From your description, I understand that want to know how to skip the notification when you try opening an already opened Word file.

This behavior may occur if Word determines that the owner file for the document already exists and this may also occur if one or more of the following are true:

1) Word previously quit improperly and, therefore, did not delete the owner file.

2) A second instance of Word is running in the background with the document already open.

3) The document is shared over a network, and another user has it open and this is irrelevant to the version if Word.

To resolve this issue, ensure that another user on the network does not have the document open. If the document is in use, open the document as read-only.

If the file is not in use, quit all instances of Word, and then remove the owner file and verify the result.

To do so, follow the below mentioned steps:

1. Check in ‘Task Manager’ whether WINWORD.EXE process is running in the background.

*Open Task Manager by right-clicking the taskbar, and then clicking Start Task Manager.
Tip: You can also open Task Manager by pressing Ctrl+Shift+Esc.
*Click on ‘Processes’ tab
*Look for WINWORD.EXE. If found, right click on the same and then click on End Process and End Process.

2. Start Windows Explorer, and then browse to the folder that contains the document file that you tried to open when you received the error message.

3. Delete the owner file.


The owner file is located in the same folder as the document that you tried to open. The owner file name uses the following convention: It begins with a tilde (~), followed by a dollar sign ($), followed by the remainder of the document file name. The file name extension is .doc. For example, the owner file for Document.doc is named ~$cument.doc.


4. Start Word. If Word asks you whether you want to load the changes that were made to the Global or Normal template, click No.


5. Open your document.

I hope the above suggestions helps to resolve your issue. Take your time to try the suggestion and let me know the results at your earliest convenience.

Thank You

graham mayor say:
Assuming that you have not renamed the multipage from its default name, you need

Private Sub cmdBack_Click()
Me.MultiPage1.Value = 0
End Sub


The pages are numbered sequentially from 0, so page 1 is '0'

You could always name the tab and then click the tab to access it e.g.

Private Sub Userform_Initialize()
With Me.MultiPage1
    .Pages(0).Caption = "Employees"
    .Pages(1).Caption = "Caption for page 2 tab"
End With
End Sub
