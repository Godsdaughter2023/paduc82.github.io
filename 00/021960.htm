tom ogilvy say:

BillBeshlian,

in your evaluate pictures, you show you are trying to match the string "221134950" to your column X.  If column X holds the number   221134950  then there will be no match.  For illustration,

put 123 in D2

then test

=Match("123",D2,0)

and

=Match(123,D2,0)

and you will see the string "123" returns #N/A

so this is a possible source of problems in your file. 

You can convert a string to a number (if the value is numerical) by multiplying by 1

=Match("123"*1,D2,0)

You know how your data is organized - I am just pointing out why you might be getting the #N/A error if you believe the target value is in the lookup range/column.
