sheldrick say:

Hey Kareem701, Welcome to the Nokia Forums!

 

With some WLAN access points there can be power save related problems when WMM (Wi-Fi Multimedia) and U-APSD (Unscheduled Automatic Power Save Delivery) are enabled in the access point.

 

This may cause the data transfer to phone stop every time the phone goes to sleep mode.

 

As a result, when using e.g. the web browser, the user might have to push the link several times until the link opens.

 

Incoming VoIP calls or push e-mail may not work properly because the phone does not receive the data packets when in power save mode.

 

This is a problem in the access point. Some of the WLAN access point devices are not properly indicating in the beacon packet that there are packets buffered for the mobile device when the device is in sleep mode. The problem has been observed at least with the following WLAN access point devices:

  • D-Link DAP-1353 (Firmware 3.00)
  • Netgear WNR2000 (Firmware V1.2.3.7)
  • D-Link DIR-635 (Firmware 2.32EU)
  • Linksys WAP4410 (Firmware 2.0.0.5)

The following workaround solutions could be used to solve the problem:

  1. Check if there is an update available for the WLAN access point firmware. The issue has been reported to access point manufacturers and they might have released a fix for the problem.
  2. Disable WMM power save in the WLAN access point.
  3. As a last resort, if anything else is not helping, disable power save in the mobile device WLAN settings (Settings -> Connectivity -> WLAN -> Options -> Settings -> Options > Advanced settings > Power saving). The drawback of this is that without power save the phone is consuming more energy and this will drain the battery much faster.

I hope this solves your problem, please get back to me if it solves your problem or if you require more assistance!

 

Sheldrick
