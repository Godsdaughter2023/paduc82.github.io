rmjo say:

Good news, bad news, and more ...

(Many!) Findings to Date

  • The problem is definitely associated with Word-to-PDF conversions, and may be due to:
    - Converting to a PDF and then saving the Word file.
    - Interrupting a PDF conversion
  • The problem may or may not also be due to a virus.
  • Not only does it add extra characters to StyleRef fields in Word, it is also known to spit the characters into the PDF; to affect renderings with other applications; and to disable / lock Update Field.
  • Adobe has known about the problem since at least December 2006, and is reported for Versions 8 and 9.
  • Since no fix has been forthcoming, the community consensus is that Adobe either can't fix it (can't figure out the problem) or won't fix it (in favor of pursuing something else or because of an Adobe/Word standoff. Conversely, Microsoft could potentially do something about it but can't or won't.
  • The problem affects headings, headers, footers, and anything associated with StyleRef.
  • There are many, many hit-and-miss workarounds; what works for one person won't work for another.

Suggested Workarounds that Don't Involve Messing with Styles

  • Always save Word before saving or printing to PDF. After creating the PDF, quit (do not save) your Word file.
  • Get rid of PDFMaker; don't use it.
  • Don't use any Adobe-PDF products. Find an alternative such as NitroPDF (which I've never heard of until now).

Macro to Clean Up Messed-Up Word Files

God bless Alan McConville, whoever you are, for posting in the Adobe Forums a macro that cleans up the document so you don't need to step through every instance.

=== Begin Quoted Posting ====

Once the characters are in a document it's hard to remove them without affecting links. With the ZWAdobeF font embedded in the document other systems cannot render the Word file and installing the font elsewhere would cause licensing issues. I've used the following macro to replace instances of the font with something more portable using a character which does not appear onscreen/in print.

Sub killAdobeZWQuick()
Selection.GoTo What:=wdGoToPage, Which:=wdGoToNext, Name:="1"
Selection.HomeKey Unit:=wdStory
Selection.Find.ClearFormatting
With Selection.Find
.Font.Name = "ZWAdobeF"
.text = "^?"
.Replacement.text = ""
.Forward = True
.Wrap = wdFindContinue
.Format = True
.MatchCase = False
.MatchWholeWord = False
.MatchWildcards = False
.MatchSoundsLike = False
.MatchAllWordForms = False
End With
Dim i As Long: i = 0
Do Until 1 <> 1
' Find the font
Selection.Find.Execute
If Selection.Find.found = True Then
i = i + 1
Selection.Font.Name = "Arial"
Selection.Font.Size = 1
Selection.text = ChrW(Val("&H206F"))
Else
Exit Do
End If
Loop
MsgBox ("Fixed " & CStr(i) & " Characters")
End Sub

The code has no warranty with it or a license, it's an example of what can be done. If you use it you do so at your own risk. It will probably not work for non "western" font pages.

=== End Quoted Posting ===

 

To create the macro for Word 2007, you may need to save your document as a .docm (macro-enabled) file. You can then run the macro in the docm file or in the docx file.

Thought I should mention that anyone who converts from Word-to-PDF is probably at risk for the ZWAdobeF font problem. And you won't know it until you discover that "Update Field" is locked and/or that all your StyleRefs suddenly have extra characters.

Should you be motivated to inspect your PDF'd Word files , then I suggest trying the following:

  • Right-click in random StyleRef fields to see if Update Field is enabled or disabled (unlocked/locked).
  • Use Edit/Find to search out the ZWAdobeF font.
  • Use the Styles Pane / Manage Styles -- and its various tabs, as well as Import/Export for the styles organizer -- to see if the ZWAdobeF font is lurking anywhere.
Keypoint : Fields that were created after a PDF corrupted a Word file will not be locked and/or will not be pre-pended. Consequently, the fact you can update (for example) a ToC without problems does not mean there are no problems.

Concordance file for indexing using text within quotation marks - Microsoft Community

jelehe111 say:

Concordance file for indexing using text within quotation marks

We are using Word 2003.  We have a very large document that needs to be indexed.  We created a concordance file for the index as we would like to avoid manual marking.  The problem is that we only want to mark occurrences of text within quotation marks.  A term is defined once, using quotes, and then is used again and again without the quotes.

We only want to index that first usage within quotes.  We tried creating the concordance file with the text within quotes in the first column, but we find that text within quotes is ignored when we do the automark.  Does anyone know a workaround? 

FYI:  There is no pattern as to where in the document a term may be defined.

Thanks in advance.

jay freedman say:
Word's concordance mechanism can't do what you want without the aid of a macro.

I hope you still have the concordance file you created, with the quotation marks in the first column. Install the following macro in your Normal.dot template (or a global template) according to http://www.gmayor.com/installing_macro.htm, and then...

Open the document to be indexed.
Press Alt+F8 and run the macro.
Enter the full path to the concordance file and click OK.
When the macro finishes, it will pop up a message to say it's complete.
To see the index entries, you'll have to turn on nonprinting characters with the ¶ button.

'---- start code ------

Sub MarkTermsWithQuotes()
    Dim indexedFile As Document
    Dim concordanceFileName As String
    Dim concordanceFile As Document
    Dim concTbl As Table
    Dim searchTerm As String
    Dim indexTerm As String
    Dim rw As Row
    Dim rg1 As Range, rg2 As Range
   
    Set indexedFile = ActiveDocument
    concordanceFileName = InputBox("Enter the full path of the concordance file:")
    Set concordanceFile = Documents.Open(concordanceFileName)
    Set concTbl = concordanceFile.Tables(1)
   
    For Each rw In concTbl.Rows
        Set rg1 = rw.Cells(1).Range
        rg1.MoveEnd wdCharacter, -1
        searchTerm = rg1.Text
       
        Set rg1 = rw.Cells(2).Range
        rg1.MoveEnd wdCharacter, -1
        indexTerm = rg1.Text
       
        Set rg2 = indexedFile.Range
        With rg2.Find
            .Text = searchTerm
            .Wrap = wdFindStop
            If .Execute Then
                rg2.Collapse wdCollapseEnd
                rg2.Fields.Add Range:=rg2, Type:=wdFieldIndexEntry, Text:=indexTerm, _
                         PreserveFormatting:=False
            End If
        End With
    Next
   
    concordanceFile.Close savechanges:=wdDoNotSaveChanges
    MsgBox "Complete."
End Sub

' ---- end code ----
