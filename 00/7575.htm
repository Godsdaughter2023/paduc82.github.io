Option Explicit

 

Public Enum olSaveAsTypeEnum

  olSaveAsTxt = 0

  olSaveAsRTF = 1

  olSaveAsMsg = 3

End Enum

 

Private WithEvents Items As Outlook.Items

 

Private Const MAIL_PATH As String = "\\10.80.1.25\Corporate\C_A_D\BCSB System\Validation Error email from CBO"

 

Private Sub Application_Startup()

  Dim Ns As Outlook.NameSpace

 

  Set Ns = Application.GetNamespace("MAPI")

  Set Items = Ns.GetDefaultFolder(olFolderInbox).Items

End Sub

 

Private Sub CopyWhatever(ByVal Item As MailItem)

  If TypeOf Item Is Outlook.MailItem Then

    SaveMailAsFile Item, olSaveAsMsg, MAIL_PATH

  End If

End Sub

 

Private Sub SaveMailAsFile(oMail As Outlook.MailItem, _

  eType As olSaveAsTypeEnum, _

  sPath As String _

)

  Dim dtDate As Date

  Dim sName As String

  Dim sFile As String

  Dim sExt As String

 

  Select Case eType

    Case olSaveAsTxt: sExt = ".txt"

    Case olSaveAsMsg: sExt = ".msg"

    Case olSaveAsRTF: sExt = ".rtf"

    Case Else: Exit Sub

  End Select

 

  sName = oMail.Subject

  ReplaceCharsForFileName sName, "_"

 

  dtDate = oMail.ReceivedTime

  sName = Format(dtDate, "yyyymmdd", vbUseSystemDayOfWeek, _

    vbUseSystem) & Format(dtDate, "-hhnnss", _

    vbUseSystemDayOfWeek, vbUseSystem) & "-" & sName & sExt

 

  oMail.SaveAs sPath & sName, eType

End Sub

 

Private Sub ReplaceCharsForFileName(sName As String, _

  sChr As String _

)

  sName = Replace(sName, "/", sChr)

  sName = Replace(sName, "\", sChr)

  sName = Replace(sName, ":", sChr)

  sName = Replace(sName, "?", sChr)

  sName = Replace(sName, Chr(34), sChr)

  sName = Replace(sName, "<", sChr)

  sName = Replace(sName, ">", sChr)

  sName = Replace(sName, "|", sChr)

End Sub

cassandranelson- [o365] say:

Exchange Online and Onpremise DirSync

Get-DistributionGroup -Id "Everyone_group" | select HiddenFromAddressListsEnabled:$true

How do I change it on the on-premises?

ranmo_msft say:

Hi Cassandra,

What you encountered was expected. When DirSync is deployed, the source of authority in on-premises AD. You need to manage objects such as users and groups from your on-premises AD instead of Azure AD. As this is an Exchange attribute (HiddenFromAddressListsEnabled), you should have at least one Exchange server in your On-premises AD so that you will have a supported way to manage it and then sync it to Office 365. Without Exchange servers in on-premises AD, it will put you in an unsupported scenario. Using ADSI edit tool or other third party tools to edit the attribute in AD and then sync to Office 365 is not supported by Microsoft. That’s why we strongly recommend customers keep at least one Exchange server when using DirSync to manage objects after mailbox migration to Office 365.

However, based on my personal experience, you can use ADSI edit tool in DC and go to msExchangeHideFromAddressLists attribute and set it to true. Then run DirSync, it should reflect in Office 365. But please note that this is an unsupported method for those customers who don’t have Exchange servers in on-premises AD and if you encounter any issues or problems, Microsoft might not be able to provide you support for it.

If anything is unclear, please feel free to let us know and we are glad to assist you.

Thanks for your time on this.
Ran

About the only way I know is to format the pictures with text flow (NOT "inline"). Those won't display at all in the Outline view (or the Draft view).
