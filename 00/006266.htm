I am trying to search for specific data in the bound column of a listbox (to eventually determine the listindex of a row). Herefore I want to use ADO. To do this , I think, you must use the listbox data as ADO recordset. How do you do this (code) ?

dirk goldgar say:

I am trying to search for specific data in the bound column of a listbox (to eventually determine the listindex of a row). Herefore I want to use ADO. To do this , I think, you must use the listbox data as ADO recordset. How do you do this (code) ?


A list box has a Recordset property, but in an .accdb or .mdb file it's a DAO recordset, not an ADO recordset.  So you *could* search its recordset using DAO methods. However, if your purpose is to find the ListIndex of the matching row, I don't think that's a very good way to do it.  Other people working with the Recordset property of a list box have found it to be rather quirky.

Why not just loop through the ItemData collection of the list box directly?  That way you will certainly end up with the appropriate ListIndex.  Code might look something like this:

 

'------ start of example "air code" ------

Dim ix As Long

With me.lstMyListBox

    For ix = 0 To (.ListCount - 1)

        If .ItemData(ix) = "What I'm Looking For" Then
            Exit For
        End If

    Next ix

    If ix >= .ListCount Then
        MsgBox "Found it: ListIndex = " & ix
    Else
        MsgBox "Didn't find it."
    End If

End With

'------ end of example "air code" ------
