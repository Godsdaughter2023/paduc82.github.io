robert aldwinckle on forums say:

getting occasional problems when trying to browse certain sites


If they are publicly accessible and you need help post a URL.

One possible explanation would be that you are connecting to a server specifying HTTP 1.0 level protocol but using HTTP 1.1  headers in your request.   I think both protocols use a common verb that IE would be using for you  GET  but it may be including other things that the server is not prepared to recognize.  

Often it is the other way around, e.g. when I am using telnet 80 to test a host's connectivity and using  GET /   which has an implied  HTTP/1.1  associated with it and the server waits for more, apparently assuming we are doing HTTP 1.1, so I give it a  Host:  header and then it is happy.

In IE9 you can use the Developer Tools, Network Capture to get a better idea of what is going on in terms of protocol.

A Left Join will show all records in [New Cards Table] whether or not they have matching used card records. It may not be essential, but I'd use the NZ() - Null To Zero - function to be sure that you get a zero sum even if there are no records:

 

SELECT [New Cards Table].[Company Name], [New Cards Table].[Card Number], [New Cards Table].[Card Amount], [Cards Used Table].[Card Name Used], [Cards Used Table].[Card Number Used], NZ(Sum([Cards Used Table].[Amount Used])) AS [SumOfAmount Used], [Card Amount]-NZ((Sum([Amount Used]))) AS Expr1
FROM [New Cards Table] LEFT JOIN [Cards Used Table] ON [New Cards Table].[Card Number] = [Cards Used Table].[Card Number Used]
GROUP BY [New Cards Table].[Company Name], [New Cards Table].[Card Number], [New Cards Table].[Card Amount], [Cards Used Table].[Card Name Used], [Cards Used Table].[Card Number Used];

To get a left join in the query design grid, select the join line between the table icons and view the Join Properties. Choose option 2 (or maybe 3, depending on how you added the tables) - "Show all records from <parent table> and matching records from <child table>".

Access, by default, is a multi-user platform. So this functionality is built in. However, to insure data integrity and not cause corruption, a multi-user database should be split between the back end (the tables) and a front end (everything else). The back end will be stored on a network shared drive. The front ends will be stored locally on each user's PC and linked to the shared back end.

As users enter data, new records will be created in the linked tables. All data is stored in tables. You can then create reports or charts to display the data or use forms to browse the data.

find a range of numbers - Microsoft Community

david o. bryan say:

find a range of numbers

I have a financial spreadsheet.

If i have a bill for around $670, I would like to find this number on this sheet.

Since I am not sure that the value is exactly $670, is there a function that I can use to

find the cells on this spreadsheet that have values between 570 and 770?

I prefer NOT to data sort, or filter.

chip pearson say:

Suppose your data is in column A, A1;A10. In C1, enter the exact value you are intested in, such as 670. Then, in C2, enter the width of the acceptable range, such as 100. Then select A1:A10, enter Conditional Formatting. Select "Use A Formula ..." and enter

 

=ABS(--$A$1:$A$10-$C$1>=$C$2)

 

Click the Format button to select the format you want applied if a value lies outside the data range. If you want to highlight cell within the  limit, change the comparison:

=ABS(--$A$1:$A$10-$C$1<=$C$2)
