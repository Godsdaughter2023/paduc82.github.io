matthewjakachira say:

Count up timer for PP?

Can anyone help me in building a count up timer for PowerPoint 2010. IT needs to be setup so that the number will continue to count up on the slide it will be positioned. It will be used for a days without injury slide if that helps at al

e.j. gun say:

Here is an example for you to try.  Build it, confirm that it works, and then try to apply it to your specific needs.

 

Steps:

  1. Open a blank PowerPoint presentation with a single, blank slide
  2. Do a "Save As" and save it as a ".pptm" (macro enabled) file
  3. Insert a single TextBox.  Type in "00:00:00".  Format the font, font size, etc, as desired.
  4. Rename the TextBox "TimerTextBox" by selecting it, clicking on the Home tab, selecting "Select/Selection Pane..." and then typing a new name in for "TextBox 1"
  5. Insert a single, "Custom" action button: Insert/Shapes/Action Buttons/Custom
  6. Size the Action Button to exactly cover the text box.  Make the Action Button at least partially transparent to show the Text Box below it.
  7. Click on the Developer tab, and select "Visual Basic"
  8. In the Visual Basic Editor, select "Insert/Module" to insert a new code module.  Add the code below to that module.
  9. Run the slideshow
  10. Click on the action button to start the macro.
  11. The count up should begin

Will this work for you?  It can easily be adapted to other needs.

 

Eric

 

'===== BEGIN CODE =====

Option Explicit

Private Const TIMERLIMIT As Single = 30# ' Number of seconds to count before stopping
Private Const UPDATEDELTA As Single = 1# ' Update display every (pick a value) seconds

Sub UpdateTimerTextBox()
    Dim tStr As String
    Dim time1 As Single, time2 As Single, time3 As Single
    Dim updateTime As Single
'
    time1 = Timer()
    time2 = Timer()
    time3 = Timer()
'
    ActivePresentation.Slides(1).Shapes("TimerTextBox").TextFrame.TextRange.Characters = "00:00:00"
    DoEvents
'
    While (time3 - time1 < TIMERLIMIT And Application.SlideShowWindows.Count > 0)
        time3 = Timer()
        If (time3 - time2 > UPDATEDELTA) Then
            tStr = Format((time3 - time1) / (24# * 60 * 60), "hh:mm:ss")
            ActivePresentation.Slides(1).Shapes("TimerTextBox").TextFrame.TextRange.Characters = tStr
            DoEvents
            time2 = time3
        End If
    Wend
End Sub

'====== END CODE =====

Outlook.com POP settings for Mail on OSX and iOS - Microsoft Community

lukemoores say:

Outlook.com POP settings for Mail on OSX and iOS

Okay so after lots of experimentation I've finally worked it out, it would have been easier if Microsoft published this as they should have done. Here's how to do it

1) Add account in preferences and enter e-mail, name and password as usual

2) Open the outlook.com tab of Accounts in the preferences window

3) Incoming mail server should already be set to pop3.live.com

4) User name is *** Email address is removed for privacy *** (as opposed to just *youraddress*)

5) Click for the drop down menu of Outgoing Mail server and click edit SMTP server list

6) Click on outlook.com and enter the server name smtp.live.com

7) Click on the Advanced tab and select use custom port and enter 587 (though it should work anyway as it is a default port)

8) Set Authentication to Password and enter your username (remember the @outlook.com after your address) and password then click OK

9) Set TLS Certificate to None

10) Click on the Advanced tab and make sure the "Include when automatically checking for new messages" box is checked

11) Make sure the Port is set to 995 and the Use SSL box is checked

12) If it isn't already set Authentication to Password

Voila, That was for OSX but the same principles apply to iOS, with outlook.com the settings Mail automatically sets tend to be incorrect so make sure you do the above.

Hope that Helped,

Luke

Smokie9,

Your formula works fine although it could use a little tweaking.

First, since you want the text data to appear at summary level you need to make sure have you checked the option to "use formula" for "Calculation for task and group summary rows" when setting up the custom field.

Second, as written the formula will produce an error for any task row, summary or otherwise, if there is no deadline (i.e. value is "NA"). To resolve that you first want to check for this condition and then test for your deadline dates. This formula is one way to do that:

IIf([Deadline]>100000,"Deadline not set",IIf(datevalue([Deadline])=datevalue("6/29/18"),"2018.Q2",IIf(datevalue([Deadline])=datevalue("9/28/18"),"2018.Q3",IIf(datevalue([Deadline])=datevalue("12/28/18"),"2018.Q4","Deadline not set"))))

You might not need the last nested IIF if there is no chance a user inadvertently enters the wrong deadline date. For example, if the user entered "12/27/18" instead of "12/28/18", the formula will produce a "Deadline not set" even though there is a deadline.

Hope this helps.

John

pbezucha say:

Don’t be misled: your sorting conditions are far from being basic simple and frequent.

Now, it depends on whether your task is several-shots or repeating. In the second case, the study of “Computer Science” will really pay off. For an illustration, the following macro will do what you want. Before launch, you must select the range to be sorted. No protections are built in, neither next adjacent columns are sorted alongside.

Option Explicit

Sub SortWithMidpoint()

Dim R1 As Range, R2 As Range, N As Long, I As Long, J As Long, _

  RT As String, Chr As String, R1Addr As String, WSH As Worksheet, WS As Worksheet

Const Midpoint As String = "."

Set R1 = Selection

N = R1.Count

R1Addr = R1.Address

Application.ScreenUpdating = False

Set WS = ActiveWorkbook.ActiveSheet

'open an auxiliary worksheet in the same workbook

Set WSH = ActiveWorkbook.Worksheets.Add

R1.Copy Destination:=WSH.Range(R1Addr)

Set R2 = WSH.Range(R1Addr)

Set R2 = Range(R2(1), R2(N).Offset(, 1))

For I = 1 To N

  RT = R2(I, 1).Text

  'skip, if omitting non-numerics is not necessary...

  For J = Len(RT) To 1 Step -1

    Chr = Mid$(RT, J, 1)

    If Not (IsNumeric(Chr) Or Chr = Midpoint) Then RT = Replace(RT, Chr, "")

  Next J

  '... down to here

  J = InStr(1, RT, Midpoint)

'conversion into integer

  R2(I, 2) = 1000 * CLng(Mid$(RT, 1, J - 1)) + 10 * CLng(Mid$(RT, J + 1, Len(RT) - J))

  'with ranking "E"-suffix:

  If Right(R2(I, 1).Text, 1) = "E" Then R2(I, 2) = R2(I, 2) + 1

Next I

R2.Sort Key1:=R2(1, 2), Order1:=xlAscending

WS.Range(R1Addr).Value = WSH.Range(R1Addr).Value

' delete the temporary sheet without alerts

Application.DisplayAlerts = False

WSH.Delete

Application.DisplayAlerts = True

Application.ScreenUpdating = True

End Sub

You need to use webbugs - not perfect, but more reliable than read receipts.  A web bug is an image embedded from a web server - when the person opens the message, they need to get the image from the server. It can be a 1x1 transparent image or something like a logo. Most bulk mail software includes tracking capability and includes the email address and other information. You'd need access to the server logs - so it's better to use software that supports it.
