scottgem say:

First, you do not need, nor should you include a field list in A DELETE query. It should simple be DELETE * FROM Table

Second, you have a French bracket around the table name in the FROM clause. Don't know if that is a typo or what.

Third, You don't need both the INNER JOIN and the WHERE clause. They are redundant. 

Fourth, While you can sometimes delete via a Join, I find that sometimes it doesn't work and a subquery works better:

DELETE * 

FROM [SELECTION QUALIFICATIONS without duplicates]

WHERE person_id IN(SELECT Person_id FROM [CADET SELECTION]);

talking moose say:
Your Exchange administrator needs to configure both an internal Autodiscover address and an external Autodiscover address for your users.

When users launch Outlook for Mac it will look at the user's email address to resolve the domain (bob @ example.com). It will then attempt to contact autodiscover.example.com and this should resolve to the correct server depending on your location to the network (inside or outside your company network).

To disable autodiscover in your Outlook for Mac clients you can run the following AppleScript:

tell application "Microsoft Outlook"
set background autodiscover of exchange account 1 to false
end tell

This must be run while Outlook is open for the user. It cannot be done en masse for all users on all machines.

You either already have #VALUE! error(s) in one or more of the ranges or there is TEXT in the range Opex Report September 2008 $F$14:$F$289.

=SUMPRODUCT(--('Opex Report September 2008'!$M$14:$M$289=$A6),--('Opex Report September 2008'!$O$14:$O$289=C$4),'Opex Report September 2008'!$F$14:$F$289)
