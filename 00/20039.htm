bare paddington say:

It may have been mentioned, but there is a solution to this problem. (Don't get excited - it's not a proper one).

There seem to be two flavours. Some seem to get this problem on upgrading to Windows 10, and others (like me) on Windows 8.1 accepting the upgrade to Office 2016.

For the second group, the answer is to uninstall the upgrade and reinstall Office 2013. I know! Groans all round, but I've done it and it worked.

There is one caveat that I'm aware of: if you use VBA, as I do in some Access databases I've developed, then you may find that your code has 'compile errors' after the reversion back to 2013. The solutions to this problem elsewhere talk about late-binding and other such complex topics, but it is actually far simpler: The 'references' in the relevant Office program need to be reverted to how they were before you did the original upgrade. When you find the list of references, untick the references to the 2016 versions of, say Excel, and tick the old one that you would have had selected before. I had to untick one, but tick two (the objects as well, so experiment.)

Hope this helps people.

jack cook - aka help_line say:

There is NOW an app for that called contacts+message backup

Contacts + Message Backup app is a Settings app that enables you to make backup of all your contacts (in VCF format) and messages (SMS & MMS including attachments) to SD card (in XML format) for later restore to same or another Windows Phone.

Backup is stored in 'backup+restore' folder on the SD card.

Known Issue: Restoring a backup of messages from a dual SIM Windows Phone restores all messages to message store for the first SIM/slot.

Note: To use this app, please go to Settings on your Windows Phone.  This app will NOT appear in your list of installed apps.
