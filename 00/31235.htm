wsrmarkoe say:

When Speech Recognition doesn't understand what I am saying, it goes through a correction process that eventually asks me to  spell the word. However, when it doesn't understand the letters as I spell them, I have no way to identify the word I am trying to record.

There are two methods for making sure WSR, Windows Speech Recognition, understands the letters you are spelling.

  1. Use the universal military alphabet. For example Alpha for letter a, Bravo for b, Charlie for c, etc. If you do a browser search you will find the rest of the military alphabet letters.
  2. You can also say for example, "Ess as in Sam, Pee as in people, ee as in example, ee as in elephant, cee as in California, h as in happy, etc. In other words the letter and any common word starting with that letter.

Best way to make any correction is to select not only the incorrect word but the words around (one or two on each side) and say correct. Very often the cause of the context of the surrounding words the correct group of words will show up in the correction box.

Marty Markoe, MyMSSpeech.com

alan morris papercut software say:

Make sure the LPD service is running in the 740.  There is a config page from the printer that will give you the IP address and hostname and the available network services supported by the network card.  Make sure the computer can ping the network card on the printer (ping HOSTNAME).  If you cannot ping the device you will not be able to print to the device

Add a Standard TCP/IP Port for the device  (Printui /s /t3)  Then enter the HOSTNAME (or IP), when the SNMP protocol is running on the printer network card, the port will get configured using the infomation in the tcpmon.ini file

PROTOCOL1       = "LPR"
QUEUE1          = "AUTO"
SNMP1           = "?"
LPRDspool1      = 1
