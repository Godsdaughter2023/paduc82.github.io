lead3 say:

There is a file in

c:\Users\*Your User Name*\Appdata\local\Microsoft\Windows\History\Low\History.ie5

called index.dat that controls the amount of Internet Explore history that is stored.

Once this file reaches slightly more than 16MB's, History stops recording (I'm guessing a bug since doing this makes no real sense).

If you delete this file before it reaches that size, history will be retained. Even if you delete it after, the previous history is saved.

Index.dat will be recreated and you will not lose the current history.

This is a special hidden file and to see it or delete it you must open a Command Prompt window at elevated privileges.

Right click on the command prompt icon and select Run as administrator.

Then type

cd c:\Users\*your user name*\Appdata\local\Microsoft\Windows\History\Low\History.ie5

Substituting your real user name

Then type

dir /as

to see the files in that directory. See if index.dat is up to the limit.

Make sure to Close IE 9 if open, then

Type

del index.dat /as

to delete the file.

Edit: You must do this periodically to keep history continuously.

Troubleshoot Windows Installer

2438651 How to troubleshoot problems that may occur when you install, uninstall, or upgrade a program on a Windows-based computer

http://support.microsoft.com/kb/2438651

 

Alternate:

Unregister Windows Installer, and then reregister Windows Installer. To do this, follow these steps:

1. On the "Start" menu, click "Run:.

2. In the "Open" box, type "msiexec /unreg", and then press ENTER.

3. On the "Start" menu, click "Run".

4. In the "Open" box, type "msiexec /regserver", and then press ENTER.

If this does not help the I would suggest to install the Windows Installer Cleanup utility and run that to remove the leftover Java installer files that may be causing issues with your current installation.

For further information, visit the below mentioned link:

http://support.microsoft.com/kb/290301
