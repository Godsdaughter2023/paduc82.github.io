Sub Get_Unique()
Dim lastrow As Long, x As Long, c As Range
Dim ws As Worksheet, dst As Worksheet
Set ws = Sheets("Sheet 1")
Set dst = Sheets("report")
dst.Range("AQ1").EntireColumn.ClearContents
lastrow = ws.Cells(Cells.Rows.Count, "AC").End(xlUp).Row

    If IsDate(c) Then

andrew tischaefer [msft] say:

Ok, I've been looking into this and while it is difficult to pinpoint exactly what may be happening on your computer, I was able to reproduce some variation of this problem myself and we were able to investigate.

At the risk of getting too technical, I will provide some technical details here.  Office and OneDrive have to talk to each other in order to correctly map the local path on your drive (c:\username\OneDrive\filename.docx) to the version on the OneDrive service (http://onedrive.com/bunchofurlstuff/filename.docx) so that we can properly keep both versions in sync with each other.  In your case, that "talking" is being interfered with or is broken, and thus Word only sees that the file is local (c:\username\OneDrive\filename.docx) and doesn't know that it also exists online in the OneDrive service. Hence the confusing experience of you opening the file from your local OneDrive folder and getting told to save it to OneDrive to enable sharing.  This is also why, if you open the file directly from the OneDrive service (navigate to the web site and open there) Office knows that the file is on the service and all the sharing stuff works - because in that case, we already know the file is on OneDrive.

There are a few things that we know can cause the relationship between Office and OneDrive to get temporarily messed up.

First, if the OneDrive application is not running, then Office can't query the OneDrive application to map the local file to the web copy and thus we treat it as a local file.  This would manifest itself in the behavior that you describe above, where we don't believe that the file is stored in OneDrive.  The fix for this would be to manually start the OneDrive application.

Second, we know of a few cases where on or during upgrade, OneDrive and Office can get out of whack.  Restarting your computer, or restarting OneDrive, should resolve that.  We found one unexpected case of this in our own testing over the last day and are engaging with the OneDrive team to understand it, but even in that case, a restart fixed everything up.

If neither of these solutions work to resolve the problem for you, please reply back to this thread so that we can get more information about your specific configuration and continue to troubleshoot.

Again, sorry for the inconvenience.  I hope this helps and if not, we will continue to work on it until it does.

I have a table with data in columns A thru E.  It extends to almost 2000 rows.  How is the best way to make a macro that will automatically remove all rows that have a value of less than $1 in column "C"?  If I do this manually to create a macro it will be different the next time when I am using different data.  For this reason, I need a macro that can look for these values each time and automatically delete those rows.

Thanks,

Doug

mike h.. say:

Hi,

Try this code.

Sub delete_Me()
Dim x As Long, LastRow As Long
LastRow = Cells(Cells.Rows.Count, "C").End(xlUp).Row
For x = LastRow To 2 Step -1
    If Cells(x, "C").Value <= 1 Then
        Rows(x).Delete
    End If
Next
End Sub
