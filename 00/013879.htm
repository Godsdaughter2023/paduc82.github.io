Setting up the database as you describe will give you a bad start.

For example, you should not store the components of a project in the project table. If you have a fixed set ofpossible components (01 is always Marketing in any project), you could have three tables:

tblProject (ProjectID, ProjectName)

tblComponent (ComponentID, ComponentName)

TblProjectComponents(ProjectID, ComponentID)

This will give you a flexible structure not limited by the numebr of components in a project.

In a similar way you can create a table with standard Tasks an a a table for the taks per ProjectComponent:

tblTask (TaskID, TaskName)

tblProjectComponentTasks (ProjectID, ComponentID, TaksID, HoursSpend, DateSpend)

ken sheridan say: I think a ComponentTasks table is also needed, with key ComponentID, TaskID.  It would be possible to extend TblProjectComponents to ProjectComponentTasks, with key ProjectID, ComponentID, TaskID (so not the same as your tblProjectComponentTasks which would reference it), which would be in BCNF, but it would not be in 4NF and would be subject to potential update anomalies.  Its decomposition into ProjectComponents and ComponentTasks means that both are in 4NF and the possible update anomalies are eliminated.  The two tables are referenced by tblProjectComponentTasks on ProjectID, ComponentID, and ComponentID, TaskID respectively, so its integrity is now protected by the enforced relationships.

ken sheridan say:

Relational databases are not spreadsheets; they are models of an underlying reality in terms of its entity types and the relationship types between them.  As far as I can see the model outlined by Peter, along with the extension proposed by me to plug the gap which seemed to me to exist in the model, reflects the underlying reality of your business model very closely if the Employees entity type (and thus table) is added to the model.  The key element is Peter's tblProjectComponentTaskstable, which models the relationship type between projects, components and tasks.  To this needs to be added an EmployeeID column to reference the key of Employees.  That would, I believe, give you a basic complete model.

The 'tables' which you refer to in 4,5 and 6 are not base tables, but the result sets of queries on the base tables.  Yes, Access can do it, but it does require some knowledge of creating queries, and in all probability the ability to do so in SQL, as I doubt you'd be able to do so via the visual query design interface.  It also requires a sound understanding of the theoretical database relational model.  I mean no disrespect when I say that some of the statements in your posts in this thread do suggest that you might not be so fully conversant with the principles of the relational model as would be necessary to achieve what you wish.  This is not intended as a criticism; we have all had to negotiate the learning curve in our time, and we would be happy to guide you, of course, but you will need to get into a relational mindset first.

On the basis that the best way to learn is to do it, I'd suggest that you set up a database along the lines we've described and throw some dummy data at it to test that it does adequately model the underlying reality.  Be sure to ask questions of the model as you go along.  Try to break it by adding data which represents the most extreme situations you can think of, even if they are unrealistic ones.  If the model won't handle a particular scenario, then it will need amending.

Once you are satisfied that the model is a robust one, then the question of how to build queries to extract the sort of information you require from the data can be examined, along with the appropriate user interface for both inserting data and extracting information from it.

Ken Sheridan, Stafford, England

The best solution may be to use another browser, but here's my standard response.

If this is happening only on one (or a few) website(s) there really is (by Internet Explorer standards) a long running script on the page and IE is doing you a favor by providing the "Stop script" option. Click on the Stop script button.

- -

Again, if only a site or two are affected...

It might break a desirable feature on the site, but placing a site in Restricted Sites will disable JavaScript and the script messages will stop.

Internet Options> Security> Restricted Sites...

Internet Options> Advanced, unchecking Use smooth scrolling might help.

The script may be coming from an ad. Running an ad blocker like Adblock Plus may help on some sites.

https://adblockplus.org/

- -

If you're seeing the message on all or nearly all sites, a browser add-on might be causing it.

Note: The Google Toolbar has caused script problems for some users. See if disabling that toolbar stops the messages.

Start Internet Explorer without add-ons from Start> Programs> Accessories> System tools> Internet Explorer (no add-ons)
or

from Start> Run (or Windows logo key + r) type in iexplore.exe -extoff

(iexplore.exe<space>-extoff) then hit Enter.

IEBlog  Troubleshooting and Internet Explorer’s (No Add-ons) Mode:
http://blogs.msdn.com/ie/archive/2006/07/25/678113.aspx

Memory  usage: Internet Explorer will allow memory usage to increase to unreasonable levels Closing all iexplore processes from Task Manager and starting with a new process may be inconvenient but will get memory usage back to a reasonable level. This might help with script problems.
