Use this formula for the date...

 

=1*TEXT(LEFT(A1,8),"0000-00-00")

 

and this formula for the time...

 

=1*TEXT(RIGHT(A1,4),"00\:00")

 

You will need to use the Cell Format of your choice to change the displayed numbers into Date and Time displayed values. You should apply the Cell Format to the formula cells before you copy the formulas down, or across, if that is what you plan to do.

graham mayor say:
Word mail merge is a one to one merge format i.e. One record per 'document'. In order to put more than one record on a  Word page then you need either to create a label merge e.g. you appear to want four 'documents' per page, so use a four cell table as your merge document http://www.gmayor.com/merge_labels_with_word_2007.htm

Beyond that it gets altogether more complicated - see http://www.gmayor.com/ManyToOne.htm or Paul Edstein's merge tutorial which you can download from my web site.

john mcghie say:
PNG, EMF and EPS would be my choices.

Don;t forget that Microsoft Office can handle only RGB colour: 24-bits.  If you go over that, it will down-convert with unpredictable results.

Also:  The Word header is dimmed on-screen, to indicate that it's not part of the editable text.  There's nothing you can do about that: Word will always dim the running header and footer.

You could, of course, place your letterhead in the TEXT, not the running header.  If you do, it will look the same in both print and on-screen.  But it then is part of the editable text, and so may be moved or deleted by careless users.

Such a strategy is fine for the first page: but if you want to control replication to the subsequent pages you will have to use the running header and section break.

Word 2008 is not the weapon to be using for this: it's graphics abilities were not "high".  It does tend to produce a 72 dpi placable header for a vector illustration.  That's one reason Phillip is suggesting PNG: it's Word's native format.  It's highly compressed, but it's still a bit large for this purpose.

I wouldn't use JPEG, but neither would you :-)

maor bracha say:

1.  Why i could not send mail to external mail after add autodiscover and spf1 records?

SPF records identify the authorized originating servers(/services), allowed to send email messages as your domain name.

If a message is sent from a host that is not specified in the SPF record - the recipient (/receiving server) will suspect it as spoofing (spam) and therefor would block it.

You should also notice the "-all" at the very end of it. This means "Hard Fail" = most recipients will definitely block your message. If you use "~all" this means "Soft Fail" = more loose configuration, and it is then up to the recipient, some will block and some will not. 

Since you have another mailing server (I assume an Exchange on-premise) that is not included in the record, together with the "-all" hard fail > results in messages being blocked.

This has nothing to do with Autodiscover.

 2. Are you sure if i add spf1 as your comment, i can send mail to external mail?

It is up to you to make sure that all host names and IP addresses are included in the SPF records to allow all of them to send messages.

Using the "MX" switch in the SPF will probably be acceptable in most basic scenarios, as it include the hosts in your MX record and allowed senders as well.

I will elaborate a bit more about host names and IP addresses in the next question.

3. "other DNS hostnames or known IP addresses"?  What is it? Where i get them? 

In host names and IP addresses I mean other mailing solutions you might have deployed:

  • Exchange on-premise.
  • External mass email service (for marketing).
  • Other mailing services that might use your domain name.

Each of these services have an IP address represented to the world when going out to the internet (sometimes referred to as NAT), and at times there are other DNS records to group such services together.In your current SPF record- spf.protection.outlook.com is actually a DNS name that groups together a few Exchange online IP addresses.

So, if you know any other services that uses your domain name, you should include them as well.

For a basic configuration, including MX is usually enough,


4. what is TTL?

TTL is a DNS record's Time To Live .

When DNS servers query a record they cache it locally for a certain amount of time, based on that value set by the DNS zone owner. 

For example, if your TTL is set to 24 hours, and you make a change to a record - it may take up to 24 hours for external organizations to see that change.

Hope this helps,

Maor Bracha
