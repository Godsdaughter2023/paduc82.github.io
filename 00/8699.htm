The solution is relatively simple to correct.  If someone selects a different email address as their default.  Insert that email address into the reply to: header line in the smtp header.  It eliminates the problem.  This is why if you use the desktop outlook connector and fill in the reply to: field provided in the connector you don't have this issue.  The problem is that most phone mail clients do not have this field for "reply to" and you also don't have it on the website.

Database is stored in a network folder.  Could that be causing a problem?  I have never had this happen before.

bill mosca say:

Your problem is that the db is on the network. Rebooting your machine will have no effect on the locking file. If you look at the folder you will see either an LACCDB or LDB file with the same name as your database. That is the locking file. Occasionally, that file will not get destroyed if a user exits the database improperly such as rebooting before closing the database. Your network guys can delete that file and that should fix the problem.

In the future, you should split the database so the back end (database with only the tables) is on the network and each user has a front end (with all the forms, reports, queries, macros and code) on his local drive. That will prevent the hanging lock file that you have now.
