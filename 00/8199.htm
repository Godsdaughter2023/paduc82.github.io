john w. vinson say:

The DateSerial() function can do this very nicely. It takes three arguments, a year, a month number, and a day number, and returns a date time value. Try a criterion on AbsenceDate of

>= DateSerial(Year(DateAdd("m", 3, Date())), 9, 1) AND < DateSerial(Year(DateAdd("m", 3, Date()))+1, 9, 1) 

tom van stiphout say:
There is a school of thought that says you should have a "Dates table": fill a table with the next 20 years of dates, and additional subject-matter specific date related fields, such as DayName, Weeknumber, AcademicYear, and others as-needed. Then you can join with this table and pick up the necessary values. This is very high performing, whereas some date math functions could become a bit tedious.

ken sheridan say: Academic years are analogous to accounting years, so you could use the following function:

Public Function AcctYear(DateVal As Date, MonthStart As Integer, DayStart As Integer) As String

    Dim dtmYearStart As Date
    
    If MonthStart = 1 And DayStart = 1 Then
        ' accounting year is calendar year, so return single year value
        AcctYear = Year(DateVal)
    Else
        ' get start of accounting year in year of date value
        dtmYearStart = DateSerial(Year(DateVal), MonthStart, DayStart)
        
        ' if date value is before start of accounting year
        ' accounting year starts year previous to date's year,
        ' otherwise it starts with date's year
        If DateVal < dtmYearStart Then
            AcctYear = Year(DateVal) - 1 & Format(Year(DateVal) Mod 100, "-00")
        Else
            AcctYear = Year(DateVal) & Format((Year(DateVal) + 1) Mod 100, "-00")
        End If
    End If

End Function

Call it in the query with:

WHERE AcctYear(AbsenceDate,9,1) = AcctYear(Date(),9,1)

kergga say:

Moving a lot of data will take at least nominal time, no matter how you do it. The biggest factor may be your approach (e.g. your code) for the copy.

The slowest method would probably be looping through columns and rows to copy the data cell by cell. Depending on the details of the project, I would probably set "range B" equal to "range A", or I would just use code to copy the whole Sheet1 and insert it into workbook B (and change the sheetname if needed).

The latter approach will work fine if you are primarily working in VBA, but if you have a lot of formulas that point to the Workbook B Sheet2 then you need to be aware that deleting the old sheet and renaming the new sheet with the original data sheet name can break your formulas. One workaround is to use named ranges, and use code to change the named ranges to point to the new worksheet.

If you provide the copy/paste code you are using, and a little more information about how the data is used in your destination workbook, it would be easier to help you pick an approach (community members might throw some sample code your way along with their recommendations) :)

Using a lookup or index to return a range - Microsoft Community

tripleavery say:

Using a lookup or index to return a range

Hello,

I'm trying to lookup a value and return the range of cells that value represents so I can nest it in a formula.  My data looks like this

111           222          333          444

7                  9           1               3

19                22           14            16

4                  12            16           13

If I have the number set "333" in E1, I would like F1 to return C2:C4. 

The way I would usually do this is: =IF(E1=A1,A2:A4,if(e1=b1,b2:b4... etc)).  Two problems arise with that: 1) I have 78 columns.  2) When I use quotations it views the answer as text rather than a range, or when I don't use quotations it returns the value in the top cell only. 

Each header is unique, so HLOOKUP could conceivably work.  I'm thinking some sort of INDEX function, but I'm really stuck.  I tried to simplify this as much as possible, but the end result is it will be nested and used to pull a random number from the range identified.  Thanks!

Anthony

 

lars-Åke aspelin say:

Try this formula:

=INDEX(A2:D4,,MATCH(E1,A1:D1,0))

Note: This is an array formula that has to be confirmed with CTRL+SHIFT+ENTER rather than just ENTER.

Example:

If you have 333 in cell E1, select the range E2:E4 and array enter the formula you will get a copy of C2:C4 in E2:E4.

Hope this helps / Lars-Åke
