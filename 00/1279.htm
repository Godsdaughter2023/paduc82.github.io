Not entirely clear as to what all has transpired but just to clarify a few things 

#1 - Has originally stated, contact/calendar info is NOT "sync'd" for any POP3 or IMAP account - nothing to do with it being a Yahoo Account or Outlook 

#2 - When your account was configured as POP3 - your contact info was stored in the PST file associated with that account. Removing a POP3 account from a profile does NOT delete the associated PST file so it's just a matter of locating the original PST file for the account. Default location for PST files using OL'2013/'2016 is:
"C:\Users\UserID\Documents\Outlook Files"

#3 - If you're account is currently configured using IMAP - again, just a simple matter of adding the original PST file as per item #2 back to your Outlook profile and setting that PST file as the "default data file" for the Outlook profile.

(the default data file contains the default contact/calendar folders for the Outlook profile)

Step#1 - To add the PST file back 

- from OL menu --> File --> Open & Export --> Open Outlook Data File

Step#2 - To set PST file as the "default data file"

- From OL menu --> File --> Account Settings --> Data file tab --> highlight the PST file you just added --> select "Set as default"

*** as long as you opened the right PST file, your contacts will be in the contact folder under the PST file folder tree in the Nav pane

#4 - If you're going to configure the account again as a POP3, then it's a matter of selecting the original PST file when configuring the account

As for the issues related to not downloading email etc - that "might" have been as simple as running ScanPST on the PST file to correct any potential issues but emphasis is on the word "might". Impossible to know for sure.

If all else fails and you want to get all your Yahoo contacts back into Outlook, just a matter of exporting the contacts from Yahoo to a CSV file and importing them into Outlook. If you're account is configured as IMAP with a contact folder having the words "This Computer Only" in the description, personally would not import into that folder. Instead would create a new PST file, set it as the default for the profile and import the contacts into the contact folder contained in the newly created PST file.

OneDrive Files On-Demand: "Free up space" greyed out - Microsoft Community

ia_dev say:

OneDrive Files On-Demand: "Free up space" greyed out

With the "File On-Demand" settings activated, the option "Free up space" from the Windows Explorer contextual menu is grayed out on parent directories.

However, it is available for individual files.

This happens if the "Free up space" has been already clicked once on a given directory. After adding files (locally available), the option is grayed out on this directory, although some children files are available locally (thus space not "freed up").

Steps to reproduce the error:

1/ Create a directory, e.g. OneDrive/test

2/ In this directory, create a file, e.g. OneDrive/test/file1.txt

3/ Wait for the OneDrive sync to end.

4/ The "file1.txt" will be locally available

5/ Right click on the "test" parent directory, and select "Free up space"

6/ Wait for the OneDrive sync to end.

7/ The "file1.txt" will switch status from "local" to "online"

8/ Add a new file "file2.txt" in the "test" directory: OneDrive/test/file2.txt

9/ Wait for the OneDrive sync to end.

10/ The "file1.txt" will be locally available

11/ Right click on the "test" parent directory, the option "Free up space" is now grayed out

Windows 10 x64, Version 10.0.17134 Build 17134

OneDrive for Windows, Version 2018 (Build 18.222.1104.0007)
