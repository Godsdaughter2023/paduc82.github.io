frederik long say:

It's a bit late now, seeing as how you've deleted the profile, but why wouldn't you simply use the SET command to restore %USERPROFILE% to what it's supposed to be?

Restoring %UserProfile% to its correct value with the SET command is futile because it changes this variable only in the current child process, not in the the registry. As soon as the OP closes this child process the change is lost.

This location holds the value for all NEW user profiles:

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList/ProfilesDirectory

This location holds the value for the current user's profile folder:

HKEY_CURRENT_USER\Volatile Environment\UserProfile

Whether resetting it to the correct value would solve the OP's problem I do not know.
