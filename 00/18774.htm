edm_stl say: Hello Everyone,

After many hours of troubleshooting we have resolved this issue in my domain.  For me, this was not a Microsoft problem, we caused this problem ourselves at some point.

Long story short; Even though the emails that were sent did not acknowledge the actual senders (as it does in traditional 'send as') my problem was that all of my users were granted 'send as' permissions in Active Directory.  

At some point or another one of the security groups in my Active Directory had been mistakenly granted 'send as' and that group had been added to an Organizational Unit that all of our users were contained in.

The resolution was to remove the 'send as' permission from everyone's account which I mentioned in my previous post:

  • Add-ADPermission -Identity "Eric STL" -User hramis -Accessrights ExtendedRight -ExtendedRights "send as" -Deny
  • Within 'Active Directory Users and Computers' open the properties of the account > Select the Security Tab > Advanced > Add > Select-other-users > Check the Deny checkbox on the 'Send As' option
  • http://technet.microsoft.com/en-us/library/bb124403.aspx
Thank you to everyone that helped troubleshoot this.

RULE BASICS

There are 3 different kind of rules:

Mailing List Manager (MLM)
Junk Mail Filter
Rules (custom rules you create)

They run in the order listed above. The custom rules also run in the order they are listed. (top runs first)

1) POP Rules act on a message ONLY when it first arrives in the Inbox (On My Computer), NOT when it arrives in any other folder. Once it's moved to another folder it won't be seen by rules.

2) IMAP Rules are meant to act on a message ONLY when it first arrives in the IMAP INBOX, not when moved to another IMAP folder.

3) Exchange Rules act on a message ONLY when it first arrives in the Exchange Inbox.

Each type of account has it's own rules. Click on the correct tab to check your rules. POP, IMAP, Exchange, Outgoing

el_loco---01 say:

You don't get it.

 

AVI files are NOT necessarily smaller than MP4 of the same quality as it depends on what codec is used inside these containers.

 

But anyways.

 

WMV is not a good alternative for you since the support for WMV is not very good in the 5800. But in general WMV9 (the version 9 of the codec) using Variable bitrate can be similar size than a same bitrate DIVX or XVID file in an AVI container. It depends also on the audio codec used and the bitrate, framerate and resolution.

 

I've done many tests and IMHO AVC (H264) using VBR will produce similar images than DIVX at the same bitrate.

 

 Once you reach a certain bitrate treshold the difference in quality of all codecs is minimal.

 

Just like in audio.

 

At 256 kbps or above the difference in audio quality is imperceptible between MPEG-1 Layer3, AAC, EAAC+, WMA, etc.

 

But an audio file encoded at 32 kbps will show the difference.

 

Codecs are rated by their ability to compress data. By using very high bitrates you are not compressing the data so even old codecs give good quality, it's when you use very low bitrates that you can see the benefit of a newer/better codec.
