kevin hau say: This error is usually caused by external Windows related problems, rather than issues with Microsoft Security Essentials itself. 

You can follow the steps here to start Windows in a Clean Boot, just ensure that "Microsoft Security Essentials" is the only thing enabled on the startup tab in MSCONFIG before you reboot: <http://support.microsoft.com/kb/310353>  This will remove all other startup items other than Microsoft Security Essentials to make sure it's not one of these other programs causing the issue.

The clean boot is usually the best first step to fix this type of issue.  If you notice that the problem goes away in this clean boot mode the problem is caused by one of the other startup programs on your computer.  You can go through the list of startup items in MSCONFIG and just enable them one at a time until you find the one that is causing the problem.

I hope this helps!

charlie spencer say:

Try this instead.

In File, Options, Mail, Arrival, disable 'Display an alert'.

In your existing rules, enable the action 'Stop processing more rules'.

Add a new rule at the end with actions to display an alert and / or make a sound.  Since you stopped processing rules for the previous exceptions, the only incoming messages that will trigger this rule are the ones you haven't already processed.

jim.jamz say: What if you receive new mail that isn't filtered through a rule?  Will there still be a notification for it? i.e. random unsorted mail that goes directly to your inbox.

charlie spencer say:

Specify the 'Alert' rule to apply to all incoming messages so there will be no unfiltered messages.  By stopping the processing on messages covered by previous rules, the only messages that will trigger the alert are those that haven't tripped all previous rules.  That's why you put the 'Alert' rule last.

This isn't untested; I've been doing it for years.
