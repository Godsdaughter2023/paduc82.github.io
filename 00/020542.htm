3> Unzip, go to r35 folder,
and install flashplayer8r35_winax.exe (remember ending with ax i.e. with active control, don't use the other one)
 
4> Insert your CD and use autorun/setup.exe.

After completeling all installation of programs from cd.

5> Reinstall flashplayer v9 from abode website.

john mcghie say:
Hii James:

Oh, I experience this from time to time :-)  I am a Technical Writer, long and complex documents are my stock in trade.  I doubt if "Joe" has ever seen what you and I would refer to as a "long" document (something around 2,000 pages or upward).

The only "cure" for the slowness is Word 2010, preferably running native on at least a quad-core system with 16 GB or thereabouts of RAM.  A serious workstation will speed things up dramatically when doing this stuff.  Although you would be surprised at how much faster Word 2010 is running on even a cheap office PC, when doing this sort of thing.

The real problem is that Word 2011 has the "old" algorithm, and it simply bogs down and struggles once the document gets complex.  Word 2007 and 2010 have a new algorithm that is substantially more efficient.

Of course you are aware that you can leave the cross reference window open?  and that if you do, you will not get that frustrating grinding while it parses the entire document? You can simply leave the x-ref window open and skip around the document.  If you do, save frequently: you are stressing Word quite a lot doing this, and it will crash from time to time (but it will do that anyway...)

Sadly, the cross-reference window is a fixed size; not the greatest example of intelligent UI design.

A technique I sometimes use when really struggling is to create the cross-reference on the page where the target is (so I can see everything).  Having created the cross-reference, I then cut it and skip back to the place where I wanted it, and paste.  To help with the skipping, I sometimes use Shift + F5 (jumps between the previous 3 cursor locations).  Sometime I use a bookmark ("Here") macro to enable me to instantly skip back to where I came from.

But sadly, what you are experiencing is the reality. Anything you can do to improve machine performance will help. So things like a reboot and quitting other applications will help.

Cheers

john mcghie say:
Hmmm...  That doesn't happen here.  The Insert Cross Reference dialog is designed to stay open for exactly this purpose.

What I do is:

1)  Click where I want the cross-reference

2)  Insert>Cross Reference

3)  Choose the object I want to cross-reference.

4)  Click Insert

5)  Click back in the document

6)  Scroll to the next place I want a cross reference

7)  From the dialog, choose the next object.

8)  Click Insert

Rinse and repeat.  The dialog never closes.  So Word never has to recompile the list of document objects.  

I don't understand what you mean by "It's also a shame that page references have to be anchored by the closest footnote to that page" -- that's not true :-)

What are you trying to do?  Say "See page 138"?  You can do that with a bookmark on the offending page: then cross-reference to the bookmark. If you don't want to futz around with bookmarks, you can cross-reference to a "Numbered Item", which may be hidden.  Just so long as Word generates the number, it will be referenceable.

You will find that Word 2010 running in Parallels on that iMac will be more than ten times faster than Word 2011 for this sort of thing.  Boot the thing to Windows 7 native and it will be 100 times faster.

And "Yes" the software engineers are keenly aware of the issue:  they were deeply concerned about it in the run-up to the product shipment.  I told them not to worry about it: very few people would ever notice.  I was right...  It's you and I, so far :-)  They would have had to hold up the product on-sale date for another six months to fix it.

Yeah, we'll have a look at the next version.

chip pearson say:

GetSaveAsFileName merely returns the name of a new or existing file. It does not actually save the file. The code below prompts the user for a SaveAs file name and saves the file as the specified name in the appropriate file format:

 

Sub AAA()
    Dim Ext As String
    Dim FormatType As XlFileFormat
    Dim N As Long
    Dim FileName As Variant
    FileName = Application.GetSaveAsFilename(filefilter:= _
        "Excel 97-2003 (*.xls),*.xls,Excel 2007 Macro (*.xlsm)," & _
            "*.xlsm,Excel 2007 No Macro (*.xlsx),*.xlsx")
    If FileName = False Then
        Exit Sub ' user cancelled
    End If
    N = InStrRev(FileName, ".")
    Ext = Mid(FileName, N + 1)
    Select Case LCase(Ext)
        Case "xls"
            FormatType = XlFileFormat.xlExcel8
'        Case "xlsx"
'            FormatType = XlFileFormat.xlOpenXMLWorkbook
        Case "xlsm"
            FormatType = XlFileFormat.xlOpenXMLWorkbookMacroEnabled
    End Select
    ThisWorkbook.SaveAs FileName:=FileName, FileFormat:=FormatType
End Sub
