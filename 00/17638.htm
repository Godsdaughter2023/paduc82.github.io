carrie m. say:

Pivot: Multiple groups based on value, not on current data

I'm combining data from 11 individual scores to 4 groups.  I can do this by highlighting those columns in the pivot and using Group, but I don't find it reliable over time.  Questions are answered on an 11 point scale (0-10) and my groups are 0-4, 5-7, 8-9, and 10.  However, it's possible that at Q.4 nobody answered "1" this quarter, so my pivot table does not show a column for it.  Next quarter, someone does say "1", and now my pivot is incorrect, because the 0-4 group does not actually contain 1.

Is there a way to create multiple groups where you get to input the start and end of the range?  It seems that when looking at the Group Fields dialogue, I can only make one group.

adam vero say:

You don't need to group by value, you can just select the ones you want to put together and group them by hand if that is easier. The trick to stop them falling out again is this:

In the Pivot Table  right click on the field you are grouping and go to field settings (or do this from the field list task pane).

On the Layout & Print tab, tick the box for "show itmes with no data".

Now, if a particular score is not used one month, it will still show in the group rather than dropping out, which in turn explicitly reminds you that no-one answered that score this month.

If you want to keep them in the groups but NOT display them when empty, use a value filter for that field: drop down the filter list, choose value filter > greater than, [choose your score field if you have more than one value field in the PT] put 0 in the box. This type of filter is dynamic, so now when you refresh, items will appear and disappear from view, but still be kept in the PT cache so they remember their groups.

 

Hope this helps

Adam Vero MCT, MMI, MBMSS:CRM 4, MCSA:Messaging Meteor IT - Training and Consulting Services

hansv mvp say:

Let's say the scores are in column C, starting in C2.

Add a new column to your source data, with column heading (for example) Score_Group.

In the second row of this column, enter the formula

=1+(C2>4)+(C2>7)+(C2>9)

and fill down. This column will contain 1 for 0-4, 2 for 5-7, 3 for 8-9 and 4 for 10.

Update the source range of the pivot table to include the new column, and add it to the pivot table instead of the score itself.

You are more apt to get an answer in the Insider Forum since it's an Insider version. In fact, I will move this thread to Insider Forum. 

However I ' try to explain before doing so about AutoRecover, make Backups. 

Technically, there is no such thing as autosave. There is Feature called Auto-Recover. The way it supposed to work is  Your working a document, and like a good user is expected to do every so often, you save the document. Then all of a sudden Word shows The spinning Pizza wheel a message comes up, and the program quits. Now if you have done everything as you should have you immediately open Word Backup If AutoRecover saved what it supposed to, It will ask do you wish recover document. You answer yes. What you will get is what was save the last time you hit save before the crash. If you didn't do any saves your toast. If set for less than ten minutes odds are Word will not have time between saving attempts to retain anything.

Okay now there another Feature and this works differently, In Word Preferences → Save, you have checked Make Backups. What this does AS you work you save your document how often depends on how much you trust Word not to crash or corrupt your file. In this case, you are working, and you decide the version you had before your latest change, is what you need. So you save this the go to your backups and pick the desired version and open it. And it as if you hit rewind on a tape recorder.

In both cases, it depends on "you" periodically saving the document. Unless you regularly save, neither will work.

suzanne s. barnhill say:
Well, there is no reason to have both of them wrapped in the first place. Set the wrapping on the picture to In Line With Text. When you insert a text box, it will be anchored to the inline picture. You can display object anchors using File | Options | Display, and you can drag the anchor (if necessary) to the appropriate paragraph. Once it's where you want it, in the Layout Options for the wrapped text box, you can lock the anchor so that it won't move from the correct paragraph.
