niyamath u khan say: Hi NIranjan,

This can happen for several reasons, but the one i have seen most often is when there are lots of objects, such as arrows, text boxes, call outs, etc in the XLSX file.  Excel 2007's XLSX format handles these objects much differently than the earlier XLS format.  Many times, these objects may be invisible because they have mistakenly had a zero height or zero width applied to them when the underlying cells, rows, or columns are deleted.  Clean up needless objects and look for invisible objects in your worksheets to get your XLS file format size more manageable.  If you suspect you may have invisible objects, you can hover your cursor around and watch for the style of the cursor to change an object selector style even though it does not appear to be over an object.  In extreme cases, you can select all the cells of a suspect worksheet and perform a PASTE.SPECIAL into a clean worksheet using the FORMULAS and NUMBER FORMATS options and then perform another PASTE.SPECIAL using the FORMATS option.

michelle-excel say:

How do I know/change the 'toggleState' of a custom ribbon Button

I have an add-in with custom ribbon tab, one of the buttons is a toggle button and I want to determine its state (pressed or not-pressed)

M
tushar mehta say:
I have an add-in with custom ribbon tab, one of the buttons is a toggle button and I want to determine its state (pressed or not-pressed)
I also want to be able to change its state according to what the user does.

I can't work out how to either see or change its state.

Thanks

M


If the add-in is under your control, use the callback functions associated with OnAction and other attributes such as getVisible and getPressed.  You can also change the label or the image through the callback functions for getLabel.and getImage respectively.

Here's one example.  There's a toggle button that the consumer uses to show / hide a set of other buttons.  The code below manages the state of the toggle button as well as the other buttons.

First, store the result of the user action in a global variable.

 

Option Explicit

Dim Rib As IRibbonUI, _
    BtnsHidden As Boolean
Sub RibbonLoaded(ribbon As IRibbonUI)
    Set Rib = ribbon
    End Sub

Now, when the user clicks on the button, use

 

Sub ShowHide(control As IRibbonControl, pressed As Boolean)
    BtnsHidden = pressed
    Rib.Invalidate
    End Sub

When the ribbon is rebuilt (after the Rib.Invalidate), the following function decides whether a button is shown or not.

 

Sub getVisible(control As IRibbonControl, ByRef returnedVal)
    returnedVal = Not BtnsHidden
    End Sub

The following 2 functions adjust the label and the image of the toggle button.

 

Sub ShowHideLabel(control As IRibbonControl, ByRef returnedVal)
    If BtnsHidden Then
        returnedVal = "Show All"
    Else
        returnedVal = "Hide All"
        End If
    End Sub

Sub getImage(control As IRibbonControl, ByRef returnedVal)
    Set returnedVal = Application.CommandBars.GetImageMso( _
        IIf(BtnsHidden, _
            "SmartArtDemote", "SmartArtPromote"), _
        64, 64)
    End Sub

andreas killer say:

"Run-time error '1004': This operation is not allowed. The operation is attempting to shift cells in a table on your worksheet."

The reason for this error is that the table below has 3 columns and the top table has 2 columns.

If you add a row to a table using ListRows.Add means only 2 cells are inserted into the sheet in this case, which would destroy the table below.

The solution is to insert an entire row and resize the table.

Andreas.

Option Explicit

Sub Main()
  AddTableRow ActiveSheet.ListObjects(1)
  AddTableRow ActiveSheet.ListObjects(2)
End Sub

Sub AddTableRow(ByVal Table As ListObject)
  Dim R As Range
  'Get all cells in this table
  Set R = Table.Range
  'Get the bottom right cell
  Set R = R.Resize(1, 1).Offset(R.Rows.Count - 1, R.Columns.Count - 1)
  'Insert a row below
  R.Offset(1).EntireRow.Insert
  'Resize the table
  Table.Resize Range(Table.Range, R.Offset(1))
End Sub
