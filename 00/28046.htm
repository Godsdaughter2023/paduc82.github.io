Avoiding emoticons - Microsoft Community

phillro say:

Avoiding emoticons

Microsoft has followed the trend in introducing emoticons into type-ahead.  So, for instance, if one types (s) in the Live Mail editor, one gets the emoticon "".  If, like me, you find the English lexicon rich and subtle enough, so that you, like me, have no desire at all to use crude cartoon jokes to supplement good English, then this is a minor annoyance.  No more annoying than other type-ahead artifacts that are there because they are useful to other people.

However, one becomes extremely, and unnecessarily, annoyed when it comes to deleting the emoticon,

Consider what happens with (s) transmogrified into a "".  Naturally, one hits Backspace to delete the emoticon and restore what was actually typed.  The Live Mail editor almost does that, but not quite.  Backspace restores the (s) before the emoticon, so that, in order to remove the emoticon, one has to move the cursor back over what one actually typed, back to the emoticon, delete that, and then move the cursor forward again.

In designing the edit sequence this way, the assumption must be that the writer, hitting Backspace, is most likely to want to delete the (s), which is placed at the cursor.  That is exactly the opposite of what is going to be true in 99% of cases.  It the writer actually wanted the emoticon, he/she would be happy with the transmogrification, and not be hitting the Backspace key at all.  If the writer is hitting the Backspace key, the strong likelihood is that he/.she does not want the alien intrusion, and wants to get back to the common use of English (s), meaning, an optional plural.

The designers of the Live Mail editor (and many others) have managed to annoy those, like me, who just want type English, and at the same time have not served the other users who want to augment Shakespeare's language with silly pictures.

Can this be corrected?  Is it an occasion for an International Standard Emoticon Erasure Procedure initiated by Microsoft, to be rolled out to all other editing software?

stefan blom say:

Since you are using Word 2007, AutoComplete does not work for AutoText (Building Block) entries.

To create an AutoCorrect entry instead, do the following: Click the Office button, and then click Word Options. In the Proofing category, click the AutoCorrect Options button. On the AutoCorrect tab, make sure that "Replace text as you type" is selected. Then type in thimc for "Replace" and enter To Whom It May Concern in the "With" box. Click Add. Then click OK twice. Each time you type in thimc it will be converted to To Whom It May Concern.

Of course, "thimc" is just an example; you can use whatever you find appropriate.

graham mayor say:
Word 2003 had an autocorrect entry with "To Whom It May Concern." pre-configured. Word 2007 and 2010 do not.

The autotext function did not work so well in 2007, but has been fixed for 2010. You can add the entry to the available building blocks which may be inserted from QuickParts on the Insert Tab of the ribbon.

Type "To Whom It May Concern." in a document. Select it and press ALT+F3. You can then save it as an autotext entry.

You could also save it as an autocorrect entry. Autocorrect works in much the same way as autotext, but requires a trigger key sequence to activate it. Again select the text then File > Options > Proofing > Autocorrect options and you will find the text pre-selected in the dialog. Add a key command eg #to then type #to to enter it into the document.

If you are going to make much use of autocorrect, it would be a plan to add the autocorrect options command to the QAT (Quick Access Toolbar) . You will find it in the all commands group.

Autocorrect works equally well in Word 2007.
