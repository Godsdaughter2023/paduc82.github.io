Assuming that you mean clear cells:

Sub ProcessData()
    Dim m As Long
    Application.ScreenUpdating = False
    m = Range("A:B").Find(What:="*", SearchOrder:=xlByRows, SearchDirection:=xlPrevious).Row
    With Range("A1:B" & m)
        .AutoFilter Field:=1, Criteria1:=True
        .ClearContents
        .AutoFilter

        .ClearContents
    End With
    ActiveSheet.AutoFilterMode = False
    Application.ScreenUpdating = True
End Sub

ossiemac say:

The user will always need to check the box in Macro Security Settings "Trust access to the VBA project object model" or the references cannot be added.

Following will add the reference requested.

Sub RefToLibrary()
    ' Create a reference to the VBA Extensibility library.
    ' On Error in case the reference already exits or
    ' Trust Access to the VBA Project is not set in Options.
   
    On Error Resume Next
    ThisWorkbook.VBProject.References _
                  .AddFromGuid "{0002E157-0000-0000-C000-000000000046}", Major:=5, Minor:=0
    If Err.Number = 1004 Then
        MsgBox "User must set ""Trust access to the VBA Project object model"" in Macro Security"
        Exit Sub
    End If
    On Error GoTo 0
   
End Sub

For your second question to add any reference, the only method I know to identify the reference GUID is to manually select the required references first and then run the following code and print them to the Immediate window and identify the GUID you want from the description.

Sub List_Available_References()
    Dim i As Long
   
    For i = 1 To ThisWorkbook.VBProject.References.Count
        With ThisWorkbook.VBProject.References(i)
            Debug.Print .Name, .Description, .GUID, .Major, .Minor      'Edited since initial posting (Major and Minor Params)
        End With
    Next i
End Sub

ossiemac say:

Hello again Dan_5707,

Even though I have provided an answer to your query, IMHO I believe that you should resist the temptation to add references with VBA code. The User must permit access to the VBA project and that makes their computer vulnerable to accepting malicious code.

Rather than use the Ref "Microsoft Visual Basic for Applications Extensibility 5.3", it is better to initially create the required objects and use the Visible property to display or hide the objects when required.

If the Users have different versions of office then it is best to initially create the code with the Reference and using Early binding for ease of using the intellisense during development and then change it to Late binding and dimension the objects as Objects. Then the user does not need the reference and you will not get errors associated with the references due to different versions of office.
