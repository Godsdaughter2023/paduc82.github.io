Helpful sounds to provide feedback

Hi, I’m Kori, a Program Manager on the Core Experiences team. Starting in Version 1707 (Build 8326.2033) in PowerPoint 2016, Word 2016, Excel 2016, and Outlook 2016 for Windows Desktop, we’ve updated the Office Sounds add-in to be a permanent setting in Office and created a new home for such accessibility settings. We’d love for our Insiders to have the first look (or listen) of this feature update.

Office Sounds can enhance your productivity in Microsoft Office by providing audio cues. For example, a sound cue might notify you when options on the screen change. Or, a sound can confirm that an action completed (such as send a mail, delete something, or paste content).

If you’ve never tried Office Sounds, simply visit File > Options > Ease of Access and enable the Provide feedback with sound setting. If you’ve already been using the Office Sounds add-in, then you won’t notice any change, the sounds will keep playing as they always have. However, you now have the option of using a modern, updated sound set. In the Ease of Access options, you will see a Sound Scheme setting, where you can choose between Modern and Classic sound options. Whatever sound settings you choose will affect all Office programs that support these options.

You’ll notice that the Office Sound settings now live in the new Ease of Access options, along with other accessibility related settings, such as Auto-Alt Text. We plan to include future accessibility settings in this location and hope this will make setting up your apps easier.

Some events that trigger sounds:

• Opening or closing a pane
• Autocorrect
• Undo and redo
• Paste
• Saving a document

We’ll continue improving our feedback story for accessibility, so if you notice you need sound feedback somewhere, please let us know. If you’d like to provide feedback, just reply in the comments for this post, or if you’re in an app, click File > Feedback.

Thanks,

Kori,
The Core Experiences Team

jeeped say:
The Excel 2010 Limits and Specifications states that the number of Sheets in a workbook is only Limited by available memory (default is 3 sheets), but like you, I've found that Excel doesn't like to give back memory it has allocated; no matter whether that is due to VBA or worksheet formulas. Using ReDim in the manner you are describing probably doesn't relinquish the memory allocated but does allocate more memory when ReDim is subsequently used to expand the array again. You are probably better off to dimension the array to the largest extents expected to be used and reuse the static size, zeroing or setting each to vbnullstring before reuse, Setting calculation to manual, ScreenUpdating and EnableEvents to false never hurts. Limit any formatting that isn't General as much as you can. Long columns of special formatting (complicated custom is the worse) can eat memory. Limit formatting to the extent of the data range as well.
