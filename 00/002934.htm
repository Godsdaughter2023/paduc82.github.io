john mcghie say:
First, I need to try to persuade you NOT to attempt to fix things by "re-installing" on a Mac.  The best that can happen is "nothing".  If a file already exists on disk, the installer will refuse to overwrite it, no change will take place.  If a file DOES get replaced, there is a high risk of mixing updated versions of components with un-updated versions.  That buys you just about all the trouble there is.

The files that tend to have problems on a Mac are not actually on the Office disk to be re-installed.  These files are "generated" while the software is running, and are 'personal' to your machine.  In your case, the file that is the source of the trouble is probably an Apple file that Microsoft Office barely knows about (the system-level Font Cache).

Now: let's get to your problem.  The first thing I would do is carefully perform this procedure:

http://word.mvps.org/Mac/fontweeding.html

This should solve the issue for you.

You should be aware that some fonts "won't" either show up or work in Microsoft Office.  Office is looking for Unicode fonts of type TrueType or OpenType.  Old (very old!) versions of fonts won't work because they are not Unicode.  And "Type 1" or "PostScript" fonts won't work in Mac Word, although they will work in PC Word.

Hope this helps

john - project say:

NxDavidC,

You can unlink a dynamic master file but it's better to create a static master to begin with and avoid potential corruption.

To unlink suprojects in an existing dynamic master, do the following:

1. Select the insertion point summary line of the first project in the master

2. Go to Task/Properties group/Information icon/Advanced tab

3. Unchedk the "Link to project" option and hit "OK"

4. Repeat steps 1 through 3 for each subproject

Once the dynamic master is unlinked save the master and each subproject. That will insure the edits to the link structure are properly captured without leaving link fragments that will cause corruption.

As far as being able to update your master, it is easy to automate the process of creating a new static master each time. Just record a macro of you going through the steps and then run that macro whenever you need a new updated static master.

As far as the resolution of the data, first make sure you have selected "months" as the "level of usage data to include in the report" in the Visual Reports window. If you have done that then expand the data in Excel as Trevor explained.

John
