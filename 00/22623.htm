Sub InsertSlideDialog()
'
' InsertSlideDialog Macro
' This inserts a properly formatted slide dialog box.
'
    ActiveDocument.Tables.Add Range:=Selection.Range, NumRows:=2, NumColumns:= _
        1, DefaultTableBehavior:=wdWord9TableBehavior, AutoFitBehavior:= _
        wdAutoFitFixed
    With Selection.Tables(1)
        If .Style <> "Table Grid" Then

    Hi,

     

    I work for Microsoft on the Excel Web App team.  The behavior you are seeing might be a bug so I'll explain a bit about how Excel Web App works with respect to saving but unfortunately need to ask you for some additional information.

     

    First, Excel Web App autosaves in Edit mode about every 2 minutes. I say "about" because there's a set of operations that trigger saving more often (e.g. going from Edit in browser and clicking Edit in client).  Also, this save occurs on our SP server and is independent of the browser window. So you can close the browser and if you go back really quick the save may not be done. At first, I thought you were experiencing this behavior so I was going to respond what you are seeing is "expected behavior".

    However, you post above suggests that your are in Edit in browser mode, making changes, that eventually get thrown away.  I'd appreciate you answering a few questions:

     

    1) Go into Edit in browser mode. Make a change. Then click View ribbon and Reading View. You should see a short delay because this is a case  where we trigger save.  If you don't see your change then there's a problem.

     

    2) Go into Edit in browser mode and make another change.  Open a new browser window and navigate to the same file (this time in View mode) and refresh a few times. Within 2 minutes you should see your change. 

     

     Aside: the oddness you mention above about not seeing the change in view mode is because you are seeing last saved copy... then when you edit again you're actually "joining" the already existing editing session (that hasn't finished saving).

     

    Thanks

    Matt Gauthier
