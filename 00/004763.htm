        .MatchWholeWord = False

jay freedman say:

Change .MatchWildcards = True to .MatchWildcards = False.

The @ symbol is a special character in wildcard searches, which is why the macro isn't finding actual @ characters in the document. You could make it work as a wildcard search by changing the .Text string to "\@", but there's no reason to use wildcards here.

rob s t say:

I resolved this issue by modifying the registry:

  1. Exited Outlook
  2. Started registery editor
  3. Navigated to HKCU-SOFTWARE-Microsoft-Office-16.0-MailSettings 
  4. Changed Names of the following Keys:
    1. ComposeFontComplex -> ComposeFontComplex1
    2. ReplyFontComplex -> ReplyFontComplex1
  5. Start Outlook - Keys are recreated on starting Outlook
  6. Deleted renamed keys (ComposeFontComplex1 and ReplyFontComplex1)

I assume that you could delete the Keys initially without renaming them.
