rohn007 say:
I think your question is making sense. The problem is you still don't quite understand how indexes are generated.

You have to create a separate "index field" for each word to appear in the index. It will be slow if you manually mark each word to index. That is why I suggested a concordance. In the process of creating this special case of the concordance file I am showing you how to take advantage of the features in other applications, word for creating documents, excel for sorting and cleaning up duplicates.

Here are more detailed instructions, expanding on the high level suggestion I made earlier:

create copy of the file
Find/replace to replace spaces with paragraph mark after each word
select all
Copy

Switch to Excel, blank new spreadsheet
Paste into blank Excel spreadhsheet (you now have 1 column of words)
Data tab > Sort & Filter group > Sort command (now have sorted list of words)
Data tab > Data tools group > Remove duplicates command
Select the column
Copy the column
Paste it beside the first column (now have 2 sorted lists of unique words)
Select the 2 columns
Copy the 2 columns

switch to Word, sorted file
paste into word (replace the sorted list)
you now have a 2 column table (which is required by the concordance feature)
Save this version of the file

In Word Swap to the original file to be indexed
References tab > Index group > Insert Index button (NOT! "mark entry" dropdown!)
Click on "automark" button at bottom of dialog
find your concordance file, open it (this will insert all of the index field codes for you, Boom!)

References tab > Index group > Insert Index button (yes, same button again)
change any settings you want, ie number of columns
click OK
this will use the index fields marked by the concordance file to generate your new index

Much faster than counting words, once you figure out how to do it.

Does this make more sense?



When you create the copy of the original I suggest adding the word "Concordance" to the end of the file name, or simply use the same dummy/work file for ALL of your documents (replacing the text each time you need it). That way when you go to use the concordance you can easily relate it to the original file. This is a temp working file. If you edit the original document, potentially adding new words you will delete the current concordance table, copy the contents of the updated source file and go through the process of recreating the concordance. Then you can use the concordance to update the index

jay freedman say:
Opening a document directly from Outlook does open them in Protected View (not that I ever do that except just now to test it), and Word doesn't get "stuck" or display any message other than the Protected View banner across the top. So yes, it must be possible.

When things in Word "go bump in the night," the culprit is often an add-in. The troubleshooting method is to disable all add-ins and check whether the correct behavior returns. If so, then re-enable one add-in at a time and check again, until you find the one that causes the problem to start again.

Start by going to File > Options > Add-Ins and looking at the list under "Active Application Add-Ins". If any of them show "COM Add-in" in the Type column, set the dropdown at the bottom of the dialog to "COM Add-ins" and click the Go button. In the next dialog, uncheck any checked boxes and click OK. (If you get a message that the add-in must be removed by an administrator, try restarting Word "As Administrator" from the right-click menu.)

If there are any add-ins in the list that show "Template" in the Type column, close the Options dialog and shut down Word. In Windows Explorer, type or paste this into the address bar and press Enter:

%appdata%\Microsoft\Word\Startup

Move any templates (.dot, .dotx, or .dotm files) out of that folder to some other folder for temporary storage.

Now try opening a document from Outlook. If it doesn't get stuck, then start re-enabling the add-ins by moving one template at a time back into the Startup folder, or by going into the COM Add-ins dialog and checking a box. After each one is enabled, try again to open a document from Outlook.

If disabling all add-ins doesn't fix the problem, then it might work to run a Repair of Office from Programs & Features in the Control Panel.

 10. Problem fixed.
