kevthu say:

Appending multiple tables and maintaining key relationships

I have 5 tables that contain info that is related by key/foreign key relationships.  I want to duplicate the set of data in these 5 tables related to a given record in the main table. ( I'm allowing the user to have a "sandbox environment" where they can duplicate data sets and extensively manipulate them for experimental purposes without affecting the original data.)

For clarity, I'll just indicate the table relationships and keys (for simplicity, also assume I'm appending back into the same tables but am adding a marker to distinguish these appended records from any original records):

Table1 = Main Table with Table1ID as primary key
Table2 = holds foreign key of Table1ID; has primary key of Table2ID
Table3 = holds foreign key of Table1ID; has primary key of Table3ID
Table4 = holds foreign key of Table1ID; has primary key of Table4ID
Table5 = holds foreign keys of both Table3ID and Table4ID; has primary key of Table5ID

My plan so far is:
1 - Store the value of the original record's Table1ID for referencing the original records in Tables 1 thru 4
2 - Append the Table1 record
3 - Discover the max of the Table1ID after this append (ID is autonumber)
4 - Append tables 2, 3 and 4 using the max Table1ID discovered in previous step as the foreign key

However, I stumble with how to handle table 5.  I need to keep the relationships to tables 3 and 4 intact, but am not sure how best to accomplish this.  These are also one to many relationships, so there can be quite a few records involved.  Thanks for your suggestions.






tom van stiphout say:

If table5 has FKs of table3 and table4, is there a M:M between table3 and table4?

 

If yes, you should have table5's PK over the two FK fields.

kevthu say: Thanks for your reply Tom.

Table3 holds records describing entities.
Table4 holds records containing numerical data that needs to be associated to Table3 records in quantities that are not auto-calculable and are not the same as what Table4 stores.
Table5 holds numerical data derived from Table4 but re-configured into multiple records that are directly applicable to Table3 records.  The data in Table5 cannot be determined until the data in Tables 3 and 4 exist.

One record in Table3 can be related to many Table5 records
One record in Table4 can be related to many Table5 records

kevthu say: I've found a solution.  I am storing the old ID's in a new field with the records I when I append them.  This allows me to build the last append query for Table5 with joins to these original ID's and match up the records correctly.

victor l. havin say:

Mobile Device Center doesn't work with Outllok 2010. MAPI Configuration is completely wrong!

Important: Before delegating me to the Windows Mobile support, please read the message to the end. I already contacted the Windows Mobile support and they have no idea.

System: Windows 7 Ultimate 64 bit.

Email Client: MS Office Home and Business Outlook 2010. Version: 14.0.5113.5000 (32 bit). Trial version.

Mobile client: Windows Mobile 6.5. Standard. CE OS 5.2.21745

The problem:

When I connect my phone to the computer and start the Mobile Device Center, I am getting this message:

“There is no email program associated to perform the reauested action…”

If I click "OK" to continue, then all synchronization options for the mail, contacts and calendar are disabled. So I can not synchronize my mobile device with Outlook 2010.

Out of curiosity I did a little research and this is what I found:

1. The message above comes from the MAPI32.DLL. MAPI32 shows this message if it can't find DLLPathEx entry in the HKLM\Software\Microsoft Outlook key and there is no file named MAPI32X.DLL in the PATH.

2. The MAPI32.DLL is loaded by the WmdHost.exe, which is a 32 bit MAPI hosting application for mobile devices. So in this case both MS Outlook and the MAPI host are 32 bit applications and there should be no problem with the bitness.

3. MAP32.DLL is looking for the WmdHost.exe entry inside HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Windows Messaging Subsystem\MSMapiApps. In my case, this entry is set to "Microsoft Outlook"

4. MAPI32.DLL expects to find the appropriate entry in the HKEY_LOCAL_MACHINE\SOFTWARE\Clients\Mail. On my machine there is no entry named "Microsoft Outlook" in this key. There is an entry named "Microsoft Outlook Virtual" instead.

This leads to the MAPI initialization failure and the message described above.

First, I suspected Outlook configuration problem on 64 bit systems. However, if I replace "Microsoft Outlook" with "Microsoft Outlook Virtual" in the WmdHost MAPI configuration, I keep getting the same message. This time because MAPI stub fails to locate the DllPathEx entry. If I add this entry and set it's value to the outlook MAPI  DLL, the MAPI initialization is still failing. This problem is less obvious, as it is the result of a CLR exception. I suspect some required components are missing in the Outlook MAPI server.

So it looks like either Outlook 2010  was not designed to work with Windows Mobile or the implementation is seriously flawed.

I guess my questions are:

1. Did somebody try to sync outlook 2010 with a Windows Phone on a 64 bit system? Did it work? I will be very surprised if it did.

2. Does Microsoft test the MAPI implementation on their own platform? In this case Windows 7 64 bit.

3. Is there any solution for this problem or any future plans to fix it?

Thank you in advance,

stevelockwood---01 say:

This took me a while to work out and needed some coding.  I exported a CSV from Google Contacts (but you could adapt the method to work with any CSV extract).

 

The description below is quite full and detailed but it actually boils down to a fairly simple procedure which should not take too long to do.

 

In summary it's like this:

 - 1) Export your contacts from your source into a CSV file

 - 2) Open the CSV file in Excel and run the macro (below) - this will output a file called BACKUP.DAT

 - 3) Copy BACKUP.DAT to your phone's memory card, eg using the USB transfer

 - 4) Run the "restore contacts" process on your phone

 

 

1) Export contacts from Google contacts into CSV format (I used the CSV for Outlook and other programs format)

 

2) Open the CSV file in Excel and run an Excel macro against it which outputs a file called BACKUP.DAT.   The code for the macro is at the end of this post (I couldn't attach the actual spreadsheet, sorry).

 

 

(

This is a simplified VCARD format.  you could even create it with a text editor.  It looks like this:

 

BEGIN:VCARD
VERSION:2.1
N;ENCODING=QUOTED-PRINTABLE;CHARSET=UTF-8:;=
Fred=20Bloggs;;;
TEL;VOICE;CELL:+44000000000
END:VCARD

BEGIN:VCARD
VERSION:2.1
N;ENCODING=QUOTED-PRINTABLE;CHARSET=UTF-8:;=
Freda=20Bloggs;;;
TEL;VOICE;CELL:+440000000000
END:VCARD

 

etc

 

Notes:

 - spaces in the name are replaced by =20

 - there is one number per contact

 - to do multiple numbers you have to set up multiple contacts, eg: Fred(home), Fred(mobile) etc

)

 

3) Now copy the resulting file, BACKUP.DAT to your Nokia 220.  It will need to have a memory card inserted.  Copy the file to the Backup folder on the memory card.

 

(

If you are not sure where to put the file, first run a backup on your phone:

 Settings

 Backup

 Create Backup

 Contacts

 Done

 

Then connect your phone with the USB cable and select Mass storage.

 

Browse the files on the phone and you will find a file called BACKUP.DAT in the Backup folder.  For safety you can rename this file to, for example, BACKUPSAVED.DAT

 

Replace the BACKUP.DAT on the phone with the new file you have created with your macro.

)

 

4) Now you simply restore the backup file.  WARNING this will completely erase all contacts in your phone and replace them with the ones you exported in your CSV file.  It will not do any sort of merge.  That's fine if you are setting up a new phone but be careful if you already have contacts in your phone.

 

To do the restore:

 - Settings

 - Backup

 - Restore backup

 - Contacts

 - Done

 

That's it.

 

 

Now for the macro

 

Option Explicit

Sub OutputContacts()
Dim ws As Worksheet
Dim colCount As Integer
Dim rowCount As Integer
Dim s As String
Dim r As Integer
Dim c As Integer

Dim fn As Integer
Dim mn As Integer
Dim ln As Integer
Dim h As Integer
Dim h2 As Integer
Dim m As Integer
Dim b As Integer
Dim b2 As Integer
Dim o As Integer

Dim name As String

    ' set up
    Set ws = ActiveSheet
    colCount = ws.UsedRange.Columns.Count
    rowCount = ws.UsedRange.Rows.Count
    Open ws.Parent.Path & "\backup.dat" For Output As #1
    
    ' read the headings
    r = 1
    For c = 1 To colCount
        s = ws.UsedRange.Cells(r, c).Value
        If s = "First Name" Then fn = c
        If s = "Middle Name" Then mn = c
        If s = "Last Name" Then ln = c
        If s = "Home Phone" Then h = c
        If s = "Home Phone 2" Then h2 = c
        If s = "Mobile Phone" Then m = c
        If s = "Business Phone" Then b = c
        If s = "Business Phone 2" Then b2 = c
        If s = "Other Phone" Then o = c
    Next
    
    ' read the rows
    For r = 2 To rowCount
        name = ws.UsedRange.Cells(r, fn)
        If ws.UsedRange.Cells(r, mn) <> "" Then name = name & " " & ws.UsedRange.Cells(r, mn)
        If ws.UsedRange.Cells(r, ln) <> "" Then name = name & " " & ws.UsedRange.Cells(r, ln)
        name = Trim(name)
        If name <> "" Then
            s = ws.UsedRange.Cells(r, h).Value: If s <> "" Then vcard name, "h", s
            s = ws.UsedRange.Cells(r, h2).Value: If s <> "" Then vcard name, "h2", s
            s = ws.UsedRange.Cells(r, m).Value: If s <> "" Then vcard name, "m", s
            s = ws.UsedRange.Cells(r, b).Value: If s <> "" Then vcard name, "b", s
            s = ws.UsedRange.Cells(r, b2).Value: If s <> "" Then vcard name, "b2", s
            s = ws.UsedRange.Cells(r, o).Value: If s <> "" Then vcard name, "o", s
        End If
    Next
    
    Close #1
    
    MsgBox "Done"
    
End Sub

Sub vcard(name, t, number)
    Print #1, "BEGIN:VCARD"
    Print #1, "VERSION:2.1"
    Print #1, "N;ENCODING=QUOTED-PRINTABLE;CHARSET=UTF-8:;="
    Print #1, Replace(name, " ", "=20") & "(" & t & ");;;"
    Print #1, "TEL;VOICE;CELL:" & Replace(Replace(number, " ", ""), "-", "")
    Print #1, "END:VCARD"
    Print #1,

End Sub

Option Explicit

Dim LastRange As Range

Private Sub Worksheet_FollowHyperlink(ByVal Target As Hyperlink)
  Dim This As Range
  'Unhide previous cells if any
  If Not LastRange Is Nothing Then
    With LastRange
      If .Rows.Count = Me.Rows.Count Then
        .EntireColumn.Hidden = False
      ElseIf .Columns.Count = Me.Columns.Count Then
        .EntireRow.Hidden = False
      End If
      Set LastRange = Nothing
    End With
  End If
  'Refers the link to cells?
  If Target.Type = msoHyperlinkRange Then
    'Access the destination
    Set This = Application.Range(Target.SubAddress)
    With This
      'Entire column or row?
      If .Rows.Count = Me.Rows.Count Then
        .EntireColumn.Hidden = True
        Set LastRange = This
      ElseIf .Columns.Count = Me.Columns.Count Then
        .EntireRow.Hidden = True
        Set LastRange = This
      End If
      'Back to the cells with the link
      Target.Range.Activate
    End With
  End If
End Sub
