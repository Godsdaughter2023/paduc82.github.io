ken sheridan say: If you are hoping to use a single form as the SourceObject property for each subform control in the parent form, but make amendments individually to the design of each, then I'm afraid that won't work.  Once you amend one the same amendments will be applied to the design of all in each subform control.  To do what you want, in the navigation pane you should copy and paste the source form object of the first subform control and rename the copy.  You can then amend the copy independently of the first.  The second subform control's SourceObject property should then be changed to the name of the new copy.  Repeat this process for each subform control, using a distinct name for the new source form object in each case.

However I'm curious about your statement '……….and they all need to create new records to the same table simultaneously when the main form is updated'.  Updating a record in the parent form will only create a new row in a referencing table if code or a macro is executed to do so.  Normally data is entered in a subform by the user.

Some years ago I did create a small companies registration database for an accountancy firm, in which a parent companies form included three subforms for the directors, officers and shareholders of the company.  These all drew their data from a single Personnel table, but in each case by means of a query which restricted the subform's recordset to those of the relevant 'capacity'.  Often the same people appeared in all three of course.  It may be that you have something broadly similar in mind, but this does not explain your statement quoted above.

You are currently telling us how you are attempting to do something, but not what it is which you are attempting in terms of the real world entity types and the relationships between them which the database models.  If you can explain the what rather than the how, we will be in a better position to advise you on the best means of achieving your aims.

Send to > Bluetooth device in the right-click contect menu is not an IE-specific option.

 

<QP>

To restore "Bluetooth device" to the "Send to" menu, add a shortcut to fsquirt.exe under the SendTo folder by following these steps:

  1. Go to the Desktop, and open up a File Explorer window. Or, from the Start screen, search for "File Explorer"
  2. Click on the address bar at the top of the File Explorer window (under the ribbon), and paste in the following location:

    %userprofile%\AppData\Roaming\Microsoft\Windows\SendTo

  3. Hit Enter, and File Explorer will move to the SendTo folder. Right click in the main pane of the window, and go to New > Shortcut.
  4. The Create Shortcut wizard will appear, and will ask you to provide the location of the item you want to create a shortcut to. Paste in the following location, and then hit Next:

    %windir%\system32\fsquirt.exe

  5. You will be prompted to provide a name for the shortcut. Type "Bluetooth device" as the name, and hit Finish.


There [should] now be a "Bluetooth device" shortcut present in the SendTo folder. If you right-click on a file, "Bluetooth device" will be present in the "Send to" menu, and you will be able to send files via Bluetooth.

</QP>
