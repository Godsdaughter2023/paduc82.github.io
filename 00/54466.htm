As I explained in the other thread, VirtualBox and some other VPN solutions hook
into the network stack and stop Miracast functions from working. There are other
Miracast adapters on the market (netgear, belkin, action tec, etc.) that all
have the same functionality and which also prevent Miracast from working. None
of these vendors document these limitations. Following is a list of what *I*
know to interfere with Miracast in general, not limited to specifically the
Microsoft WDA:
 
1. VPN software (Netgear ReadyCloud software also adds a hidden VPN device)
 
2. group policy settings if you are on a domain
 
3. firewall rules on a domain (you should not normally have to change the
Windows firewall if you are a consumer not on a domain)
 
4. third party anti virus and/or firewalls - recommend completely uninstalling
these, restarting, and then checking to see if you can add a wireless display.
 
5. Virtualization software like VirtualBox
