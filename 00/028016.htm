  • Remove data you no longer need.

To maximize free spaces to your phone memory, you can remove or move the following:

 

  • Messages from Inbox, Drafts and Sent folders in Messaging
  • Retrieved e-mail messages from the phone memory
  • Saved Web pages. Clear the web browser cache to free some phone memory (e.g. select Options > Clear privacy data > Cache).
  • Saved images, videos, or sound files (in Gallery, Photos, Videos or Music player)
  • Contact information (in Contacts)
  • Calendar notes (in Calendar)
  • Downloaded application packages
  • If there are applications which are not needed any more, uninstall them with Application Manager
  • Any other data that you no longer need

If you are deleting multiple items and any of the following notes are shown: Not enough memory to perform operation. Delete some data first or Memory low. Delete some data, try deleting items one by one (starting from the smallest item). After deleting data, restart (power off/on) your phone.

suzanne s. barnhill say:

Well, that just doesn't make sense. In a Letter-type merge, every merge field you insert should return some data provided there is anything in the associated data field in the data source. I'm wondering if perhaps you've established a range in the Excel sheet (and have named that range as your data source) that excludes the CC column. In that case, however, I'd expect an error message.

 

The bottom line is that Word should not care what or where your merge fields are; if you have a column called Fubar in your data source, and you want to insert it anywhere in the document, then a { MERGEFIELD Fubar } should do the trick. How are you inserting the merge field? And what is the name of the column in the data source? It's conceivable that you are using some reserved name that Word is choking on; if you changed the column heading (and then the merge field), that might make a difference.
