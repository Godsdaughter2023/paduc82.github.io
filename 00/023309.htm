pptx too large to email - Microsoft Community

richue say:

pptx too large to email

Not a question, but a possible solution.

I had a 40-slide, 50MB PowerPoint presentation that I wanted to email and so had to reduce considerably in size. Looked at various options, including Compress all pictures, which knocked off a few MB. Eventually, I divided it into 10 x 4-slide presentations to see where the bulk of the filesize was, and isolated it to a single slide containing an image built from eight hi-res images. I copied and pasted the image into Paint and saved it as a JPG and a BMP, and inserted each onto their own slide. Then I printed each one and compared with the original. The BMP was slightly better but I went for the JPG and inserted that image in place of the original. The total file size reduced from 45 MB to 6 MB, just by changing one picture!

rich007a say:

Here's a good tip for finding that pesky oversize picture.

Make a copy of your file (must be .pptx or .pptm) and rename it to .zip

Open the file in WinZip or Windows Explorer and browse to the ppt\media folder.  This folder contains all the images in your presentation, and if you have the right view (details) in Windows Explorer you can easily see (and sort by) the file size.  Double click on the offending file to see which picture it is and hopefully you'll recognise it and be able to find it quickly in the presentation.

 

As for managing images in a presentation, using paste special > as JPEG will usually work well for photos.  For anything "computer generated", e.g. a chart or CAD model images or a diagram with text, you may find pasting as one of the two Metafile Picture formats (Windows or Enhanced) works best.

Then in theory, using the compress pictures utility should take care of any scaled pictures (i.e. sized to <100% of original size) and cropped areas to leave a lean mean presentation file.

 

Hope that helps.

Rich

The whole idea of "push" is that the client (phone) needs a permanent connection in order to be notified when something happens (e.g. new message).

 

If the phone is not connected, it cannot automagically know when to connect!

 

I'm sure your iPhone also keeps the connection (whether you see it or not) in order to be able to handle push notifications.

 

The only possible alternative would be if the phone received some special message in the form of an SMS (as a phone is always "ready" to receive an SMS), which would in turn trigger a connection.

 

This is probably how the Blackberry does it, but it requires a proprietary server for that so it's not a general solution.
