ken sheridan say: The DateDiff function is not subtractive in terms of the dates per se; rather it subtracts the first date's month from the last date's month, so the difference between any date in March this year and September this year will always be 6, regardless of the day of the month in each date.  The same is true of years.  My function works on the basis that the month intervals are the periods from the day of the month in the earlier date to the same day of the month in successive months, so if the day of the month value in March is greater than the day of the month value in September the difference would be 5.  Again the same is true of years.

To use my function you merely have to paste it into a standard module in your database, making sure that the module is saved under a name which differs from that of the function and any other objects in the database.  In my case the module is named basDateStuff.

You can then call it in a query like this:

Age:GetAgeMonthsDays([DoB],TRUE)

You don't need to include the Now function as my function defaults to the current date if the optional final argument is omitted.


To get someone's age in years you can use the following function:

Public Function GetAge(varDoB As Variant, Optional varAgeAt As Variant) As Variant

    If IsMissing(varAgeAt) Then varAgeAt = VBA.Date
    
    GetAge = DateDiff("yyyy", varDoB, varAgeAt) - _
        IIf(Format(varAgeAt, "mmdd") < Format(varDoB, "mmdd"), 1, 0)

End Function

In a query:

Age:GetAge([DoB])


To get someone's age in months:

Public Function GetAgeMonths(varDoB As Variant, Optional varAgeAt As Variant) As Variant

    Dim intMonths As Integer

    If Not IsNull(varDoB) Then
        If IsMissing(varAgeAt) Or IsNull(varAgeAt) Then varAgeAt = Now()
        
        intMonths = DateDiff("m", varDoB, varAgeAt)
        
        If Day(varDoB) > Day(varAgeAt) Then
            intMonths = intMonths - 1
        End If
        
        GetAgeMonths = intMonths
    End If
    
End Function
