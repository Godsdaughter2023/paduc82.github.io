I don't know of a way to do that.

Initially I though that setting the "prefix" Dalet characters so that they were not spell-checked might work, and that did seem to work, when I first applied the language formatting, but something went wrong after that (or it was difficult to see an easy way to do that).

I hoped that it might be possible to work around the problem by using a Dalet followed by something like a "no width joiner" character when you need this prefix. Although that works when you insert the no width joiner manually, to be practical there has to be an easier way to do it, e.g. using a keyboard shortcut associated with an Autotext, or Autocorrect. However, none of the obvious ways of doing that seemed to work - inserting the relevant character, or any of the other obvious alternatives, as part of an Autotext, or using VBA, just does not seem to work the same way. 

Perhaps someone else can see a way to do it.

Otherwise, the only way I found that seems to have some chance of working is to set up a bookmark using field codes as follows. This is unwieldy, especially if you are unused to using fields. (in fact, without using it for a while, I know I have no idea how unwieldy it might turn out to be :-( ). But it does seem to fix the spelling problem.

Insert the following nested fields at the beginning of your document:
{ SET D { QUOTE 0x05D3 0x200D } }

then insert 
{ D }

whenever you need the special Dalet. (0x05D3 is the Dalet character, and 0x200D is a "Zero width Joiner")

Every pair of { } characters is a pair of the special field code braces that you can insert using ctrl-F9.

You can insert the { D } by selecting it, setting it up as an Autotext then associating a keyboard shortcut with that.

When "field codes" are set to display, you should see
{ D }
When "field results" are set to display, you should see a Dalet followed (on the left) by a sort of blocky character, which should disappear when you toggle the Show/Hide button.

You can toggle between "codes" and "results" using alt-F9.

There are also various possible ways to remove the special characters, e.g. if you need to export the text, but it would probably be better for you to have a look at this and any other suggestions and see whether any of them is a useful approach, or whether it would be easier to deal with each "error" flagged by the spell checker.

3 Points On How To Make Voice Command (SIND) Work More - Microsoft Community

friedbrains say:

3 Points On How To Make Voice Command (SIND) Work More Effectively

Technical Level : Intermediate

Summary

Details
Both Nokia S40 and S60 devices uses what Nokia calls Speaker Independent Name Dialing or SIND for Voice Activated Dialing. What it does, is that instead of recording your own voice tag for each or some of your contacts (older S40 was limited to 10 voice tag only), the phone now assigns or generates the "voice tags" automatically. There is no need for you to record any voice tag, and you can not do it either even if you want it! In effect, now all of your Contacts has a "Voice Tag" generated by your phone, and this voice tag is exactly as how is your contact was written. Sort of err, "Verbatim", if I may use that for lack of better words.

Many people are a bit confused about this and others find SIND ineffective. Base on my experience there are some key things that you can use to make SIND work more effectively for you. By the way, this will apply to both S40 and S60 phones.

1) LET THE PHONE TEACH YOU
Listen as to how your phone's voice is pronouncing the built-in voice tags and try to mimic it. Try to do this with the contacts the you always call, say the top 10 or so. Your family members for sure, and so forth. I would say, depending on how much free time you have, listen as how your phone pronounce the names, the more the better.

2) TEACH THE PHONE
Do the voice dial as much as you can. The more that you do it, the more that your phone will also adapt to your voice. Yes, your phone will also try to adapt to your voice.

3) BE CAREFUL HOW YOU WRITE THE NAMES
If you have not noticed it, if you type the names in all capital letters, like "ABN-AMRO" your phone will pronounce this contacts by letter, as in "A", "B", and so forth. Your phone will pronounce it "A-B-N A-M-R-O" and it will not pronounce it as A-B-N- Amro. Hence, you can actually experiment on how you are writing the names but still the key is how the phone says it, and you must mimic it as must as you can.

Hope this helps, SIND might not be perfect, but once you got the hang of it, it will work! Like in Sports, the key is practice.

Hi Jason,

You can set alert for files or folder if anything is changed by going to Classic view in OneDrive for Business. To achieve this, please follow the steps below:

1. Open OneDrive for Business on your browser
2. Click "Return to classic OneDrive" at the lower-left corner

3. Click the gear icon on the upper-right corner and click "Ribbon" to turn it on.

4. Select the file and go to FILES > Alert Me > Set alert on this document

5. To set an alert for a folder, open the folder then go to LIBRARY > Alert Me > Set alert on this library

6. In the New Alert window, you can customize your alert like enter the names of the users you wish to send the alert and what type of changes you want to be alerted to.
