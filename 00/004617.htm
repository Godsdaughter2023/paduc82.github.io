marshall barton say:

You can not compare anything to Null, not even another Null.  Null represents a unknown value and doing something with null results in another Null, which is neither True or False.  To check for a Null value in VBA, use the IsNull function:

   If IsNull(Me.cboPertainingTo) Then

 

The same comparison in a query in a query should use the SQL operator  Is Null.  E.g.
   WHERE somefield Is Null

and Null is an intrinsic name sp don't put it in quotes:

   VALUES (" & intDocID & ", " & intIncidentID & ", Null)"

But, because a new record inserted by INSERT INTO will leave any unspecified field as Null, it could be omitted as Bob said.

 

OTOH, if you changed intPertainingTo to a string variable, you could set it to "Null" instead of 0.  Then you would not have to use If to chack for 0 and you would only need one strSQL= line.

  

If IsNull(Me.cboPertainingTo) Then
intPertainingTo = "Null"
Else
intPertainingTo = Me.cboPertainingTo.Column(0)
End If
strSQL = "INSERT INTO tblIncidentDocLink (DocID, IncidentID, PertainingTo) VALUES (" & intDocID & ", " & intIncidentID & ", " & intPertainingTo & ")"

z_a_s say:

Wildcards with DCount

Hello.

I feel like I may be working to hard, because this should be an easy fix. Im having difficulties finding it online.

Basically, I have a form with a text field. The ControlSource of this field is something like:

=DCount"[MinOfActivity]","qryQR","[MinOfActivity] = LIKE '*Apple*'")

I want to be able to use a wildcard, hence the two * around "Apple." I need the wildcards because my data could be classified as "Apple-Alpha" "Apple-Beta" "Apple-Gama" etc.

Any ideas?

Thanks

john w. vinson say:

You can use the = operator for exact match, or the LIKE operator for wildcards - but it's one or the other, not both!

If Apple is always the first word you only need one wildcard:

=DCount("[MinOfActivity]","qryQR","[MinOfActivity] LIKE 'Apple*'")

This will count the number of records with non-null MinOfActivity. I'd probably just count records:

=DCount("*","qryQR","[MinOfActivity] LIKE 'Apple*'")

Without knowing what qryQR is I can't be sure just what you want, but this should help your syntax issue!
