I am a developer trying to solve a problem with our product (an IT software deployment solution). I need help from someone with deep knowledge of Windows Installer and/or (probably) how it interacts with Windows UAC.

Our product is often used by users to deploy and install .msi's remotely. Our client software is installed and running with local system rights (basically as a service) so it is fully capable of launching Windows Installer. Sometimes users wish to install an msi on a client workstation when the logged-on user doesn't have full administrative rights to the computer. In such situations, the msi can be installed locally by our client software using an administrative logon account. In other words, our client first logs on as the admin user, then launches msiexec (using the CreateProcessAsUser API) to execute and install the msi. The advantage of using an admin logon account (as opposed to launching msiexec as local system -- which our client can also do) is the .msi can then reside on the network, rather than on the target workstation.

shenan stanley say:

Inside Internet Explorer...

Press the ALT+A keys (same time) then release both then the next key you press (by itself) should be "O".  That should bring up the "Organize Favorites" window where you can do *more* than just erase.
