cmdrkeene say:

Creating a new profile on the computer is one easy way out, and it won't require you to re-install anything like Office or Adobe.  Just create a new local user account (by declining to put in or create an email address when creating a new user).  If that new account works, you can open the C:\Users folder, drag your personal files to the new account, and then (later, when you want) delete the broken profile.

System Restore is another good option if the problem just began within the last week or so. You can roll back to a restore point during a time before the problem started, and it also won't impact any of your settings, programs, or files.

PC Reset is the last and more extreme option.  It keeps your settings and files (pptx, docx, etc.), but any programs that didn't come pre-installed from the factory will be removed and you'll need to install again yourself. If Office/Adobe came pre-installed, then they'll be kept, it's only non-preinstalled things that get lost during a reset.

davidmcandy say:
Folders can't be read only. It goes against the rules of the file system. As folders are specially formatted files, and the storage space for the read only attribute exists, you can set/unset the read only bit but it has no effect. The file system ignores it except it resists deletion.
 
As the read only attribute is unused for folders Windows Explorer uses it (or the system attribute) to mark a folder as special. When opening a folder for viewing it uses the presence of the read only attribute to tell it to use a different viewer from the standard folder viewer. Like the fonts folder - instead of showing files it shows installed fonts. This use is specific to Windows Explorer and there is no support to other programs for it.
 
As Windows Explorer doesn't want you changing the flag it uses it doesn't show if a folder is marked read only or not. It shows indeterminate. If you click the box it will change between ticked, unticked, and indeterminate. Clicking the box and changing it, it will apply to files in the folder not the folder.
 
Attrib shows the attributes of a folder
 
attrib "%userprofile%\documents"
 
And you can see even though read only is set you can save files to your Documents folder.
 
Attrib can only change an attribute if the file doesn't have hidden or system attribute also set. To change it on these you must change all three to off.
 
attrib -r -s -h "%userprofile%\documents"
 
For Help on attrib type
 
attrib /?
