Few things you can check:

Back story.

My windows 7 computer had different drive letters than my windows 10 upgrade.  In my original configuration the SSD drive was much smaller so that I had to use my storage array to hold the file index information.  I upgrade the SSD drive to a large size and cloned the OS back on it.  I also upgrade my storage array after confirming the SSD upgrade as well.  When I did that I changed my drive letters.  About 5 weeks later I upgraded windows 7 to windows 10.  Didnt think any of these changes would make a difference to the OS.

I was getting desperate searching around on the forums and found mention to problems with drive letters and the indexing.  If the index was stored in an old system under a different drive letter than in the current system their could be problems.  Who knows why?

So to fix the problem on windows 10 .....

1) Plug as USB thumb drive into my systems.

2) Opened the Disk Management form the control panel

3) Selected the USB drive from the list of Volumes

4) Right clicked it

5) Selected change drive letter and paths from the menu

6) Clicked change in the next window

7) Selectd R (my original letter for my RAID array) from the list of drive letters

8) Hit OK.

The indexing started working again right away.
