marshall barton say:

The way I like to do that sort of thing is to write a sub procedure in a standard module to disable or make invisible any disjoint group of controls.  The way I specify a group of controls is to include a group identifier in the control's Tag property.  For example, a control might have its Tag set to Acc,HR,Sales to include it in three groups.  Then the procedure would look like:

Public Sub HideControls(frm As Form, Grp As String)
Dim Ctl As Control
   For Each ctl In frm
      If ctl Like "*" & Grp & "*" Then
         Ctl.Visible = False
      Else
         Ctl.Visible = True    'might not want to do this line?
      End If
   Next ctl
End Sub

Then the logic in you form would call the procedure when the group has been determined:
   HideControls Me, "HR"

There are many variations on this theme depending on your needs.  For example. you may want to add another argument to specify if the group of controls should be made visible or invisible. Sometimes it might be handy if the procedure remembers (in a Static variable) the previous group and restore the state of the same group when the Grp argument is not specified.

This method does not make easy modifications to the logic, since you have to go back to all controls and manipulate their tags, individually.  But it is a nice solution to the VBA side of things.   And changes to a static set of controls does not require changes to the code, once the ShowHideControls call is established.
