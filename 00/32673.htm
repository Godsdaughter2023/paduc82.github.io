... check the outgoing mail server if it has the correct port number/s.

Which ports are to be used depends entirely on how the mail server is configured. We can't know this until we know at least which server is involved.

Go to Windows Live Mail, and open Accounts tab > Properties > Advanced tab.

The properties dialogue for an account will only be available if one of its folders is selected in the folder pane. There is no Advanced tab for an HTTP account.

On the box next to Incoming mail, enter 465, and make sure that the checkbox is ticked.

465 is the standard SMTP port for secured, authenticated outgoing mail. No mail server will deliver incoming mail over port 465.

On the box next to Outgoing mail, enter 995,

995 is the standard POP3 port for secured incoming mail. No SMTP server will accept a mail submission over port 995.

... go back to Outgoing Mail, and use port 993.

993 is the standard port for secured IMAP. No SMTP server will accept a mail submission over port 993.

Before we can make sensible suggestions to remedy a failure to send, we have to know what CANNOT SEND MAIL means. If there is an error message, it will tell us (a) the type of account, (b) the name of the mail server concerned, (c) the protocol being used, and in many cases (d) the port being used, (e) a server error code and response, (f) a program error code and (g) the security setting. Only when we have the enquirer's explanation of what happens and the full text of the error message can we diagnose the problem and suggest remedies. If you are really desperate to recover those contacts, here's something to try. There are no guarantees that it will work, and none that it won't mess up your installation of Windows Live Essentials requiring a clean uninstall and reinstall.

This assumes that you updated your installation of Windows Live Essentials 2011. I have not been able to test the procedure, so you're on your own.

  1. At Control Panel > Folder options > View, select Show hidden files, folders and drives and deselect Hide protected operating system files.
  2. Close down all Windows Live programs and services. To ensure they're all stopped, after shutting down programs, open Task Manager, ensure you can see processes from all users on the Processes tab, find any Windows Live processes (they will all start with wl and mention Windows Live in the description) and end them.
  3. Start Windows Explorer (Winkey + E) and navigate to
    C:\Users\<user>\AppData\Local\Microsoft\Windows Live\Contacts\Default
    If you see two subfolders called 15.4 and 15.5, proceed. If you don't, stop here.
  4. Expand the folder called 15.5 and its sub-folder DBStore. This should contain four or five files and two folders.
    Rename all the files by adding   old   to the extension, like this:
    contacts.edb ->contacts.edbold
  5. Do the same with the files in the LogFiles folder except the two .jrs files.
  6. In the Backup folder, delete the New sub-folder.
  7. Expand the folder called 15.4 and its sub-folder DBStore. Copy the four or five files and paste them into the folder 15.5\DBStore.
  8. Do the same with the files in the LogFiles folder, except the two .jrs files.
  9. In the Backup folder, delete the New sub-folder.

You have now substituted the old database files for the new, empty ones. To see if this has helped,

  1. Type   wlarp   into the start search box and press Enter. This should open a new window; select Repair...
  2. When it's finished, reboot your computer, launch Windows Live Mail and test.
