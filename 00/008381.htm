george thalassinos say: Ok, I managed to make it work. Still I cannot consider this a proper solution since it relies on a trick. And there is a catch to it which I feel I should share.

So, after adding the last ListFillRange Table line during the refresh procedure I add a special string value (i.e. "...") in a NEW Table column (i.e. called EndOfFile) which remains blank for every line but the last. The actual ComboBox value is the data line & EndOfFile. And I do not have to remove this suffix - it's no big thing to have it on the last list element.

Hence in the CB1_Change procedure I test for the existence of the "..." ending and decide on whether to execute the actual code.

Still I found a catch running the code.

1) If the currently selected item on the list BEFORE the refresh was NOT THE LAST one, the ComboBox would display all ListFillRange values but the last (until I reopened the workbook). This was corrected with a Me.CB1.ListFillRange="R_ListValues" (the name of the ListFillRange) in the CB1_Change procedure, as the CB1_Change procedure was firing up AFTER the refresh procedure re-added to the list the currently selected item.
2) If the currently selected item on the list BEFORE the refresh was THE LAST one, the ComboBox would display all ListFillRange values but the last (as in 1) BUT the CB1_Change procedure WOULD NOT fire up. Therefore a second Me.CB1.ListFillRange="R_ListValues" had to be added inside the refresh routine.

All this is very comforting and rewarding after the hours of trial and error I spent, but I really don't like it.

Perhaps there is an easier way.
If you know it please share.
Thanks for reading this through.

****************************** FINAL CODE **********************************

Private Sub CB1_Change()

Dim vnRows As Integer
Dim vsEndOfLine As String
Dim tbl1 As ListObject

Set tbl1 = ThisWorkbook.Worksheets("Report").ListObjects("T_LISTVALUES") ' Table with CB values in first column
vnRows = tbl1.DataBodyRange.Rows.Count
vsEndOfLine = Right(ThisWorkbook.Worksheets("Report").Range("R_ListValues").Rows(vnRows).Offset(0, 0).Value, 3) ' R_ListValues range is the first column of the T_LISTVALUES Table and is the ListFillRange for CB1

If vsEndOfLine = "..." Then

Me.CB1.ListFillRange = "R_ListValues"

With ThisWorkbook.Worksheets("Report")

     .Range("C_LabelsCTotal").Value = ""
     .Range("C_LabelsWTotal").Value = ""
     .Range("R_LabelsCList").ClearContents
     .Range("R_LabelsWList").ClearContents

End With

Call ThisWorkbook.DisplayReport

End If

End Sub

scottgem say:
Again, you don't need to store this data. You use the expression or queries to do calculations. 

You can create a query named qryOutOfService like:

SELECT [VEHICLE NO], DOA, DOD, DateDiff("d",[DOA],[DOD]-1) AS OutOfService
FROM [Off Road];

Then another query:

SELECT [Vehicle No], Sum([OutOfService])
FROM qryOutOfServce
Group By [Vehicle NO];

That gives you a total downtime per vehicle.

j.e. mcgimpsey say: All pictures inserted in a worksheet will be stored on the drawing layer ("above" the cells). You can copy/paste, use Insert/Picture, etc.. Ctrl- or right-click on the picture to add a hyperlink (which can be to a file, a cell, an internet address, etc.)

Of course you can insert a picture into a comment with MacXL...

   - Create a comment and clear any text
   - Select the comment border (so the size handles are showing)
   - Choose Format/Comment from the menu
   - In the Format Comment dialog, select the Colors and Lines pane
   - In the Fill:Color drop down, choose Fill Effects
   - In the Fill Effects dialog, select the Picture pane
   - Click Select Picture, navigate to your picture file, and click Insert
   - Click OK to dismiss the Fill Effects dialog
   - Format the Line surround, or choose No Line
   - Choose other formatting as needed (e.g., Size pane/Lock Aspect Ratio)
   - Click OK to dismiss the Format Comment dialog

There are 3 different ways to open a delimited text file (.csv or .txt) in Excel.

1. Double-click the file in Finder. Excel will apply rules that it thinks might work. Sometimes you get a usable result. Data types are not applied. General format is applied to all cells.

2. In Excel, use File > Open and then navigate to the file. You may have to change the type of file to open from Excel to Text (.csv) or All Files, depending on the version of Excel you have. This should fire up the import wizard you are familiar with.

3. In Excel, use the Data tab of the Ribbon and choose From Text. The same thing is using the Data menu and choosing Get External Data > Import Text File. Navigate to the text file. This should fire up the import wizard you are familiar with.
