a. user say:

The only copies of afd.sys that "matter" are here:

c:\windows\system32\drivers
That would be the one that XP "uses".

c:\windows\system32\dllcache
This is where XP keeps backup copies of important files - like afd.sys

Any other afd.sys files you see, like in your location:

C:\WINDOWS\$HF_MIG$\KB2592799

Things that look like that are backup copies of files saved when you install a Windows Update (in that case, Windows Update KB2592799). 

When you install an update, XP makes a copy of the original file so that if you ever find a need, desire or whim to uninstall the update, the original file can be restored.

When you see stuff like that, it tells you that KB2592799 included a new version of afd.sys.

As a matter of fact, if you look at that Microsoft Update, it will tell you that it includes a new copy of afd.sys:

http://support.microsoft.com/kb/2592799

More details about that Ancillary Function Driver (afd.sys) and the vulnerability fixed by that update can be found here:

http://technet.microsoft.com/en-us/security/bulletin/ms11-080

You could have lots up updates that involve new copies of afd.sys (and lots of other files) and generally, these backup copies of files take up very little room on your HDD. 

Some folks are keen to delete all these backup copies of the many files that get updated with Windows Updates either because it just bothers them or they think they are somehow going to reclaim a significant amount of HDD space (and they won't), but that is up to you to decide.  I just leave 'em.

They do not "come into play" while XP is running.
