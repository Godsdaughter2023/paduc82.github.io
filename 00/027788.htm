MS Word 2016 Page Count

Why does it then show 1 of 26 pages as my page count? I want it to read 1 of 23.

suzanne s. barnhill say:

The NUMPAGES field always returns the total number of pages in the document, regardless of the page number formatting (see https://wordmvp.com/FAQs/Numbering/PageNumbering.htm for more on how Word handles page numbering).

It will help you to visualize this if you have both "Formatted Page Number" and "Page Number" on your status bar. In this situation, "Page Number" will show the actual page count and the current position within it. The "Formatted Page Number" will show the result of the PAGE field for that page.

In the example below, I have a seven-page document in which I have inserted a Next Page section break on the third page and restarted numbering in Section 2. With the insertion point on the next-to-last page, Page Number shows "Page 6 of 7," which is the true count, while Formatted Page Number shows "Page: 3," which is the number that would be printed on that page if I inserted a PAGE field.

If you are wanting to insert a Page X of Y field (that is, "Page { PAGE } of { NUMPAGES } in your document that shows the count of numbered pages, you can either use SECTIONPAGES instead of NUMPAGES, if all the pages are in a single section, or use a calculated NUMPAGES field:

{ = { NUMPAGES } -2 }

Either of these will return 24, which is the correct number if you really started numbering on page 3 of 26 pages.

With Wireshark, yes. I just double-checked for you:

 

Short answer

You are bitten by SSL POODLE. Because of that, Symbian Anna is not compatible with Foursquare anymore, and you have to upgrade to Nokia Belle Refresh (somehow).

 

Long answer

On Symbian Anna, Foursquare for Symbian connects to the server foursquare.com via SSLv3 connection only. this is a software-bug in the underlying QtNetwork library because SSLv3 was deprecated more than 14 years ago. On Tuesday, a SSL vulnerability was revealed by Google called SSL POODLE. For a website administrator, the easiest way to avoid a SSL POODLE attack, is to disable SSLv3, and rely on it successors like TLSv1.0. And Foursquare disabled SSLv3. Now, you report issues longer than that – Foursquare changed the root authority to DigiCert High Assurance EV Root CA (and forgot to include the legacy intermediate certificate). Furthermore beginning this month, Foursquare changed from SHA-1 to SHA-256. Symbian Anna still works with Qt 4.7.4 (double-checked with QtInfo).
     I am able to reproduce your error message on my Symbian Anna. All this works out-of-the-box with Nokia Belle Refresh and its Qt framework in version 4.8 because on Nokia Belle Refresh, the handshakes completes (TLSv1.0), and I get: ‘Please make sure the entered username and password are correct.’ However, I do not have a Foursquare Account to check the rest.
