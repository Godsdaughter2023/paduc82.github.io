You can just restart your phone or you can rebuild the library as it is corrupted.

- Ensure you can see hidden files in Windows Explorer:

Windows Explorer .. Tools .. Folder Options .. View

Select "Show hidden files and folders"
- Connect the phone to your PC using the USB cable in "Data transfer" mode
- Find and rename (or delete if you're really brave) files with names like:

E:\private\101FFC31\mpxv1.mpd
E:\private\101FFC31\pcv5.mpd
E:\private\101ffca9\harvesterdb.dat
(assuming the phone's mass memory / card has been connected as E)

This will take longer than normal as it's a complete refresh, but it should now work.

Date format in mail merge - Microsoft Community

jonathanharries say:

Date format in mail merge

 I am using Access to create a data source including a date field. Date is in the UK format.

Using word to create labels but data field is being changed to USA format. How do I stop this happening please.

jay freedman say:

Dates are stored in Access as numbers (the number of whole days and fractions of days since a "zero" date), and those numbers are passed to Word during a merge. Word knows nothing about any display formatting you applied in Access, so you have to supply formatting information in the merge field for Word to apply.

Look at the "Mail Merge and Date Fields" section of http://www.gmayor.com/formatting_word_fields.htm, as well as the "Date-Time Picture (\@) field switch" section that lists the possible formatting characters.
