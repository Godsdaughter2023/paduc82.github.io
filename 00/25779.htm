huni shen msft say:

Hi ClaireB44,

 

This usually happens when you are using a picture other than 96dpi.

 

When inserting a picture, Outlook will rescale the image as if it was a 96dpi image, upon sending, Outlook will convert and compress (re-render) the images to 96dpi with the new dimensions permanently.

 

To work around this, you can re-render the image to 96dpi yourself while maintaining the dimensions of the picture. Or you can try the following:

 

  1. In Outlook > File > option > Mail > Ctrl + click “Signatures”
  2. there is a file called signature-file-name.htm. Open this file in notepad
  3. Search for '<img' without the quotes(Ctrl + F). Then change the dimension that match your actual horizontal and vertical size in pixels.

 

Thanks,

Huni

bernie deitrick say:

This may not work if you have other values on the three sheets, but try just this, not using the INDIRECT at all.  If the sheet names are changed, the formula will update.

=MAX('Case 1'!1:1048576)+MAX('Case 2'!1:1048576)+MAX('Case 3'!1:1048576)

This relies on the fact that the grand total (sum) of subtotals will always be the maximum value on the sheet, if that is all that is on the sheet.
