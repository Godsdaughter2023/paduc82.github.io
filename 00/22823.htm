brian tillman [outlook mvp 2007-2019] say:

The POP protocol knows nothing about folders. The only transfer it knows how to make is from the server-side Inbox to the client-side Inbox. The only way to download messages currently in the Sent folder on the server is to log into the server using a web interface, then copy or move the contents of the Sent folder to the Inbox folder. Then the POP client may be able to see the messages and download them.

Whether or not it can see them is up to the server. If the server doesn't think they're new messages, it won't tell the client about them and the client won't ask for them.
