You can do this:

Public Function PrintWordDoc(Path As String)
Dim oApp As Object 'Word.Application
Dim oDoc As Object 'Word.Document
Set oApp = CreateObject("Word.Application")
oApp.Visible = False
Set woDoc = oApp.Documents.Open(Path)
oDoc.PrintOut Copies:=1
oApp.Quit
Set oDoc = Nothing
Set oApp = Nothing
End Function

You'd then call it as needed:

PrintWordDoc("The full path to your word document.doc")

Because each patient record contains the ability to enter up to nine diagnostic codes, I created nine pivot tables to progressively filter out the records and avoid double counting.  I am then totaling details of each pivot tables and breaking them down by county.  The resulting reports are then copied and pasted into Word-based reports for each county. For each county's data, I am having to visit the filters for each of the nine pivot tables to deselect the previous county and select the next county. Its very much possible through 'SLICERS' option in pivot table which is new functionality in office 2010. Create a slicer for county and assign this slicer to all the 9 pivot tables.
So whenever you will make a change in slicer, it will effect all the pivot tables.
