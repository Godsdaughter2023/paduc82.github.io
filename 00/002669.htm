john mcghie say:
No, that was not a mis-type :-)

Windows normally does NOT require filename extensions.  It reads the "file header" which is the first 500 bytes of the stored file.  Windows prefers to do that because the information there tells it not only what kind of content is in the file, but at which version.

Mac OS X, on the other hand, very definitely needs the file extensions.  This is a UNIX (or Unix, I can never remember which way round it is supposed to be...) system, and they very definitely need extensions to look up their Launch Services Database to see which application to send the file to, and with which commands.

There's three common ways this can go wrong:

1)  You send without an extension.  That will work fine if the file gets to Windows, but if it's being opened inside the email program, Windows may never see it and strange things happen.

2)  The extension is wrong.  Again, you will usually get away with this if the file is on the Windows desktop, but definitely not if it's double-clicked from inside the email program.

3)  for a while, some of the Freemail services (and I think gmail was one) were actually stripping the file extension and ADDING the old type code/creator code, then replacing the .docx extension with .doc.  All beautifully internally consistent: sadly, "wrong"!  That was a few years ago, they should have fixed that one by now.

4)  Windows can be set to automatically unzip compressed files it receives, just like Mac OS X can.  Sadly, if you unzip a .docx, it's unreadable.

You should also check that you got the Mavericks patch for Apple Mail 7: there were some issues that were fixed with the patch last month-ish.

Hope this helps

bob jones aka: cybertaz say:
A footnote to John's reply to your question if I may :-)... Windows 3.0 actually was an application that ran on the DOS Operating System. It provided a GUI for that OS but was not an OS in itself. DOS was, indeed, insistent on file name extensions.

As the Windows OS evolved it initially had to maintain parity for DOS-based programs so the extension requirement continued to be enforced. IIRC, that requirement was discarded with the introduction of Windows XP.
