Hey everyone,

 

There isn’t a way to disable to the history from dropping down, but you can remove the search provider box using the registry. (this is essentially what the group policy does, except through the registry instead) You will still be able to type your search into the Address Bar and it will display the results in the current tab/window.

 

Modifying REGISTRY settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of REGISTRY settings can be solved. Modifications of these settings are at your own risk

 

·         Click Start

·         Click All Programs

·         Click Accessories

·         Click Notepad

·         Copy the below:

 

Windows Registry Editor Version 5.00

 

[HKEY_CURRENT_USER\Software\Policies\Microsoft\Internet Explorer\InfoDelivery\Restrictions]

"REG_DWORD"="NoSearchBox"

"NoSearchBox"=dword:00000001

 

·         Paste the above into Notepad

·         Click File

·         Click Save

·         Choose the location you would like to save the .reg file

·         Type RemoveSearchBox.reg as the file name

·         In the Save as type: box, select All Files

·         Click Save

 

To use this script: (or the one you will create below)

·         Go to the location you saved the RemoveSearchBox.reg file

·         Right click the RemoveSearchBox.reg file

·         Click Merge

·         Click Yes on the User Account Control prompt (if prompted)

·         Click Yes

·         Click OK

 

It would be a good idea to go ahead and create the script that removes the registry key you created using the script above in case you ever want the search box back. You can do this by following the same instructions from above except with this paste instead of the one listed:

 

Windows Registry Editor Version 5.00

 

[-HKEY_CURRENT_USER\Software\Policies\Microsoft\Internet Explorer\InfoDelivery\Restrictions]

"REG_DWORD"="NoSearchBox"

"NoSearchBox"=dword:00000001

 

Hope that helps, look forward to hearing back from you.

ken sheridan say:
Any ideas how to do this using a combobox?


Assuming the back end is a native Access database file, the following is a stripped down version of some code from one of my own applications.  Name your combo box cboBackEnd and set it up so that it lists the paths to the various back end files, or these could be in a hidden bound first column with a plain English description of each in the second visible column.  Paste the code into the combo box's AfterUpdate event procedure;

    Dim dbs As DAO.Database
    Dim tdf As DAO.TableDef
   
    Dim strMessage As String, strBackEnd As String
   
    Set dbs = CurrentDb()
   
    If IsNull(Me.cboBackEnd) Then
        strMessage = "Path to source database must be " & _
            "selected before updating links."
        MsgBox strMessage, vbInformation, "Warning"
        Exit Sub
    Else
       strBackEnd = Me.cboBackEnd
    End If
   
    strMessage = "Are you sure you wish to link to " & _
        strBackEnd & " as the source database?"
   
    If MsgBox(strMessage, vbOKCancel + vbQuestion, "Leave Planner") = vbCancel Then
        Exit Sub
    End If
   
    If Dir(strBackEnd) = "" Then
        strMessage = "The database you have specified as the " & _
            "source was not found."
        MsgBox strMessage , vbInformation, "Warning"
        Exit Sub
    End If
   
    For Each tdf In dbs.TableDefs
        If tdf.Connect <> "" Then
            tdf.Connect = ";DATABASE=" & strBackEnd
            tdf.RefreshLink
        End If
    Next tdf
   
    strMessage = "Linking completed."
    MsgBox strMessage, vbInformation, "Confirmation"

ken sheridan say:
How do I set up the combobox, so that it lists the paths to the various back end files

One way would be to create a little table, BackEnds, with two text columns (fields) BackEndPath and BackEndDescription.  In the first column enter the full path to each back end file and in the second a more meaningful short description of each.  Then set up the combo box on an unbound dialogue form like this:

RowSource:     SELECT BackendPath, BackEndDescription FROM BackEnds ORDER BY BackEndDescription;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the dimension is zero to hide the first column.  You'll now see the plain English descriptions listed, but when you select one the value of the control will be the hidden path, and this will be used by the code to refresh the links.

Tip: to find the path to the current back end enter the following in the debug window (aka immediate window - press Ctrl-G to open it)

? CurrentDb.TableDefs("SomeTableName").Connect

where SomeTableName is the name of any linked table.  This will give you the table's Connect string, which  starts with ';DATABASE=',  followed by the path.
