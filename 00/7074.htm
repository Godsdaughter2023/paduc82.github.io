a. user say:

Joan,

Outlook on the web shows the mobile devices that you use to connect to your mailbox. If a device isn't syncing properly, you may delete it from this list. Doing so removes the partnership between the mobile device and your mailbox, but doesn't delete the email account from your phone. Removing this partnership often fixes sync issues. Here's how:

  1. Sign in to Outlook.com
  2. Choose Settings, then click Options.
  3. In the left navigation bar, under General, choose Mobile Devices.
  4. Select your phone in the list, and then select Remove to delete the device partnership.
  5. Select Yes in the confirmation box.
  6. Go to your mobile device and power it off completely.
  7. Restart your mobile device after a few seconds, and then sync your account again.

Hello arejfour

One important piece of information is whether or not you plan to upgrade to '365 on the same machine where '2010 is currently installed (an in-plae upgrade) OR this involves a new machine with Office 365

If it's the same machine, then there is nothing you need to do - all the same settings/files etc will be used by Outlook '365. The only difference is going to be that the PST file for any IMAP account used for the email data will now be an .ost file.

Now, if this is a new installation of Outlook................

#1 - There is no automated process that is going to copy over the current email account settings. You're going to have to configure each account again.

#2 - For POP accounts, you'll need to copy the PST file associated with the account to the new machine and ensure that you use it when you configure the account instead of letting Outlook create a new one.

#3 - You also should take note of which PST file in '2010 is set as the "default data file" which is where the default contact/calendar folders are stored. This could well be a PST file associated with a POP account or it can be a stand-alone PST file if an IMAP account was the first account configured in '2010

#4 - Now, since you indicate that you have multiple accounts, the sequence in which you configure these accounts makes a big difference. The data file associated with the first email account will be the one set as the default data file for the profile. If you have both POP and IMAP accounts, then I would strongly suggest that you configure the POP account first and IMAP accounts afterwards. Note, that the setting for "default email account" and "default data file" are two mutually exclusive and unrelated things.

If this involves a separate PST file as noted in item #3, you will need to add this file to Outlook separately and set it as the default data file

Am sure you're likely to have questions so rather then extending this response any more and make it longer then necessary,, will be able to provide more specific details once more info is provided about your configuration.

In case you need this, to see the list of email accounts (along with type) in '2010, go to File --> Account Settings --> Email tab

To determine all the data files configured in the profile, go to
File --> Account Settings --> Data Files tab
