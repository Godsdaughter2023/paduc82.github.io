Creating Copy Numbers to Control Documents - Microsoft Community

matthewflanagan say:

Creating Copy Numbers to Control Documents

I would like to be able to control my documents more efficiently than I am.  If I were to create a document in MS Word, and needed 8 copies, is there a way to have MS Word print "copy 1 of 8", "copy 2 of 8", etc.? 
doug robbins - mvp office apps & services (word) say:

See the article "Sequentially numbering multiple copies of single document using a macro” at:

http://www.word.mvps.org/FAQs/MacrosVBA/NumberCopiesOf1Doc.htm

Today I would do it somewhat differently:

In your document where you want that information to appear, insert

Copy { DOCVARIABLE copynum } of { DOCVARIABLE numcopies }

using Ctrl+F9 to insert each pair of { } and entering the text between them.  Then use Alt+F9 to toggle of the display of the field codes.

Then create the following macro

Sub PrintNumberedCopies()
Dim i As Long
Dim numcopies As Long
numcopies = InputBox("How many copies do you want to print?", "Print numbered copies")
With ActiveDocument
    .Variables("numcopies").Value = numcopies
    For i = 1 To numcopies
        .Variables("copynum").Value = i
        .PrintOut False
    Next i
End With
End Sub

How can I dynamically rank information that will change columns on a weekly basis. ie rank column E (week 1) rank column H (week 2) etc. I wold like to keep the ranking in column A but have it change each week as new data is entered into the next % Lost column without having to change the RANK formula each time.

tom ogilvy say:

 

*3 was because your sample data was 3 columns wide for each block of data.  Countif counted the number of blocks of data and multiplying by the width of a block (3 columns) it determines how many columns are occupied by data.  Then your working/preliminary columns were A and B.  A and B represent the 2 in the formula.  Count the number of 3 column Blocks and Add 2 gives the location of the column to be ranked.

based on your description the multiplier would be

countif()*4+4  is the blank column.  then subtract 1 to get the % Lost.  For the offset, subtract 2:  countif()*4+2

so given your data, the formula would be:

=RANK(OFFSET($A$1,ROW($A3)-1,COUNTIF($2:$2,"% Lost")*4+2,1,1),OFFSET($A$3,0,COUNTIF($2:$2,"% Lost")*4+2,100,1),1)

This assumed the % Lost value was in row 2 (the second header row - adjust to match your situation )  The above formula is set up to be entered in A3. 

You say it is set up for 12 weeks (I set it up for 4 columns each week and the  6th % Lost was in column AA, so I am not sure about the 12 weeks you speak of.)

Nonetheless, saying it is set up concerns me.  My formula assumes the header will only contain the value % Lost out as far as you have populated data - so the last entry with % Lost marks the column to  rank.   If you have pre-entered 12 weeks (12 instances of % Lost), then the formula won't be able to tell which is the last column.   If that is the case, you could count numbers in the third row

=(count($E$3:$IV$3)/3)   gives the number of weeks with numbers

=(count($E$3:$IV$3)/3)*4+4   give the last column in the week and subtract 1  to bet the next to last column  =(count($E$3:$IV$3)/3)*4+3

subtract 2 for offset   =(count($E$3:$IV$3)/3)*4+2
