lene fredborg say:

I have tried to make some systematic testing. The problem with a bottom border being applied to one or more extra cells to the left of the selected cell seems to be related to column widths that are not uniform and/or to horizontally merged or split cells.

Below I have included the conclusions from my tests plus a suggested solution to fix the problem when it occurs. The Office team has received the entire test report.

The test results in Word 2013 and Word 2016 were identical and I was able to provoke the bottom border error in quite a lot of situations. I could also provoke the problem in Word 2007 and Word 2010 but in fewer situations than in later versions.

In Word 2013 and Word 2016, the problem seems to occur most often under the following conditions in case no cell borders have been applied beforehand

  • RULE 1: The selected cell is not in the first or last table row and the selected cell is not the first cell in a row (since the problem always results in extra bottom borders in cells to the left of the selected cell) AND

  • RULE 2: The left and/or right edge of the selected cell is not aligned to the left and/or right edge of a cell above.

RULE 1 seems to apply always.

RULE 2 does not apply always.

The problem does not seem to be influenced by settings like cell margins or Automatically resize to fit contents.

The problem seems only to apply to bottom borders – not top borders.

When checking a cell that has been applied an undesired bottom border, the Borders and Shading dialog box shows no bottom border!

Workaround – suggested solution to fix the problem

You cannot remove the bottoms border directly from the extra cells since such cells do not react as if they have any bottom border.

After testing a number of ways to fix the problem, including the ones mentioned by other users in this thread, I have found that the following workaround works each time and is the easiest to use:

If you apply a bottom border to a selected cell and notice that extra cell(s) to the left get a border too:

  1. Undo at once to remove the applied border.
  2. Select the cell that must have a bottom border + the cell to the left of it.
  3. Apply a bottom border to the two cells.
  4. Select the leftmost of the cells only and remove the bottom border.

The following has been added 23-Sep-2016:
It appears that Step 4 may not remove the bottom border from the leftmost cell if it is the first cell in a row. Then try to select the cell below the leftmost cell and remove the top border from that cell instead (since the problem seems to never occur in the last row, there should be a cell below the problem cell). This solved the problem in my tests. Even if the right edge of the cell below is not aligned with the right edge of the cell above, the result was correct in my tests and the original problem cell got a correct bottom border.


If this is done using e.g. the Home tab > Paragraph Group > Bottom Border icon or the icon on the Mini Toolbar, it can be done rather quickly.

john w. vinson say:

Well... yes. If you open an empty table it will have "no current record" - there is no record IN the table to be the current record!

 

And your code has several problems. You're opening the table object in Add mode, and THEN opening the same table again as a recordset and going into the new record (doing the same thing twice)... not even counting the unmatched quote.

 

It's simply not that hard!

 

Try:

 

Dim RstInstallData As DAO.Recordset

Dim db As DAO.Database

Set db = CurrentDb

Set RstInstallData = db.OpenRecordset("tblInstallData")

RstInstallData.AddNew

   RstInstallData!InstallID = 4301

   <also give values to any required fields>

   RstInstallData.Update

rstInstallData.Close

Set rstInstallData = Nothing

 

Or, probably better, run an Append query without opening any recordsets at all, or open a Form based on the table and add the new record from there.

1. Close Outlook.

2. Right click on Start Menu and select Control Panel.

3. Choose User Accounts -> Mail > Show Profiles.

4. Select Add, and in the Profile Name box, type a profile name, then choose OK.

5. Enter an email address and a password for the primary email account you use in Outlook and then walk through the steps to create the profile.

6. Once the profile is created, on the Show Profiles dialog, choose Prompt for a profile to be used.

7. Start Outlook, choose your new profile and check if the issue persists.

For more troubleshooting steps, you can refer on this link. Note: It also applies to Outlook 2016.
