I don't know if you got this solved but I had exactly the same scenario and I eventually worked it out.  It is all to do with the remembered user that your Outlook cache is keeping.

When you start typing someone name in an email or in a meeting invite it auto-fills in the contact from a previous time (in the old days this was called the NK2 cache).

What you need to do is this...

start typing the persons name and when auto-fill displays the remembered name click the delete icon next to it (or using your keyboard cursor keys highlight the remembered contact name and just press Delete on the keyboard).

Then start typing in the name / email address of the person you want the meeting with, then press ALT+K (or click the Check Names button).  This will attempt to resolve the name into one it has found from the exchange address book / outlook address book. 

Now when you go to the Scheduling Assistant it will display the free / busy information for that person.

[SOLVED}

Since it seemed as if the problem may have been that a cookie is not being stored, I checked the IE11 Internet Options settings. Under General Settings in the Browser History settings, I had the Delete Browsing History on exit box checked. Apparently, when that box is checked, the browser is also looking at the Delete Options that are selected (a separate screen that comes up when the  Delete button next to the check box is hit) and it follows whatever things are checked under those options as what it does when closing the browser. I mistakenly assume it was only deleting the browser history, but apparently it will also do whatever option has check marks. I mistakenly left the clear cookie button checked. So each time the browser was closing, the cookie associated with the NO THANKS selection, was being removed. After unchecking the clear cookies button, the NO THANKS option is now permanently recognized and the annoying Google Recommends Using Chrome option is gone.
