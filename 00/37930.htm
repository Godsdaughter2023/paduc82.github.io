gerry c j cornell say:

Your last post contains an important clue. Turn off System Protection on partition E and F! You should only be monitoring your C partition. That is where all system files and settings are located. If your external drive was connected when the restore point was created then when you disconnect the drive you break the restore point. You will have to stop monitoring all drives, restart your computer and then check the box before your C partition to stop monitoring E and F.


I do not keep old restore points. It's a waste of disk space. If you restore using a restore point more than 14 days old you can create problems. Restore points are most useful when you need to undo a recent change to your system. You should not use system restore to remove Windows Updates. These should be uninstalled. The exception is if using system restore enables you to boot the computer when other methods do not work.


System restore points are located in the System Volume Information file. The restore point once created cannot defragmented. If they are multi fragmented they stay so until they are removed. They cause free space fragmentation so any files written to a conventional drive are fragmented as soon as they are created. It helps if your remove redundant files and defragment before you create a restore point. I always defragment before I down load monthly updates and large programme installs.

If the 3rd party software is uninstalled or disables, make sure that Windows native Firewall is On .
