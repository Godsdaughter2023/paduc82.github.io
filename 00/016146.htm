Based on my personal experience, if the recipient server had set corresponding reverse DNS settings, the issue may be related to your existing SPF record. doug robbins - mvp office apps & services (word) say:
When in the situation where you have the file open with an opened word document, which I assume is actually a mail merge main document, if you change the type of that document to a Directory type mail merge main document and in it, you insert a one row table with as many columns as you have fields in the mailing list and into the cells of that table, you insert the merge fields and then you execute the merge by selecting Edit Individual Documents from the Finish & Merge dropdown in the Finish section of the ribbon, the document that is produced will contain a table with one row of data for each record in the data source.  You can then copy and paste that table into Excel and then insert a row at the top of the work sheet into which you insert a name for each field.

a. user say:

If you have Excel, you should be able to import the .mdb in Excel, then save the worksheet in .csv format.

More detailed steps for Excel 2016

 a. Open Excel

 b. Open Other Workbooks (or any other sequence where you end up at the File-Open panel

 c. Use the Browse option - in the dialog box, select the file type dropdown near the bottom right of the box and select All Files (*.*), otherwise .mdb files will not be listed.

 d. locate and open your .mdb. Excel should display a dialog box change the list of file types Browse to the .mdb location

 e. Get through any security dialogs

 f. Excel should then display the Import dialog box, with the "Table" option selected, "Existing Worksheet" selected, and =$A$1 as the address. Click OK

 g. Click File->Save As, browse to a suitable location, change the Save As Typeto a suitable .csv option (the UTF-8 format version is usually a good choice but it does depend on what the program you want to use it with accepts) and save the file.

There is only one table and one query in a normal address .mdb created by Word (Word calls these things "Office Address Lists"), which is why you do not actually have to select which table to Import.

1. Click "uninstall Ad-Aware" in the lavasoft folder of your start menu

2. If it does not work, open your control panel, click add/remove programs and locate Ad-Aware SE

3. If it still does not work, locate and run (double-click) the file "unwise.exe" (the .exe might not be shown). It should be found in C:\program files\lavasoft\Ad-Aware SE [Personal/Plus/Professional]\

If that does not work, or if you get a message saying something like "Can't find Install.log," perform a manual uninstall, which is not a very difficult procedure. Do the following:

3. Delete the entire ad-aware folder

4. Delete your Lavasoft Ad-Aware link in your start menu (if present)

5. Delete the Ad-Aware icon(s) from your desktop(if present)

6. Navigate to C:\Documents and Settings\[User Name]\Application Data\Lavasoft and delete the entire Lavasoft folder.

7. Empty your trash bin
