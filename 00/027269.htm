Does this do what you want?

Sub FillMeDown()
    On Error Resume Next
    Dim Limit As Long
    Dim r As Range, r2 as Range
    Set r = Selection
    Limit = ActiveSheet.UsedRange.SpecialCells(xlCellTypeLastCell).Row
    Set r2 = Range(r, Cells(Limit, r.Column))
    r.AutoFill Destination:=r2    ' copy formula down
    r2 = r2.Value                          ' replace formulas with values
End Sub

Caveat:  That assumes that the values returned by the formula are not strings that start with "=".  If they might be, replace r2=r2.Value with the following:



Note the change in computing Limit.  Consider a worksheet where the cell used is A3 and the last cell used is Z9.  In that case:

Limit = ActiveSheet.UsedRange.Rows.Count

sets Limit to 7.  But I think you intend Cells(Limit,r.Column) to be a cell in row 9.

Also note the difference between ActiveSheet.UsedRange.SpecialCells(xlCellTypeLastCell).Row and ActiveCell.SpecialCells(xlCellTypeLastCell).Row.

Often they are the same.

But consider the example above in a worksheet where we had entered a value into D12, then deleted it.  In that case, ActiveCell.SpecialCells(xlCellTypeLastCell).Row is 12.

ActiveSheet.UsedRange returns the actual current range of non-empty cells, namely Range("A3:Z9").

But I believe you want the row number of the last cell in ActiveSheet.UsedRange.

There are many ways to do that.  But I discovered that ActiveSheet.UsedRange also resets ActiveCell.SpecialCells(xlCellTypeLastCell) to Range("Z9") in my example.

So ActiveSheet.UsedRange.SpecialCells(xlCellTypeLastCell).Row returns the row number of the last used cell in worksheet.

PS:  P-p-please replace

Range("P3").Value = "=VLOOKUP(MenuData!A3,working!K:L,2,FALSE)"

with

Range("P3").Formula = "=VLOOKUP(MenuData!A3,working!K:L,2,FALSE)"

Frankly, I was taken aback by the behavior of the .Value assignment, given that there is a .Formula property.  With the .Value assignment, initially I expected Excel to store the string as text, not interpret the formula.

I finally rationalized Excel's behavior.  But assigning a formula to .Value is like assigning the value 3 to a variable called Two.  IMHO, it is "bad programming practice" because it might miscommunicate your intention.  As an author, I think you would like to avoid that.
