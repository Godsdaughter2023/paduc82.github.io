Hello Ashish,

Acknowledging the details that you have provided.

In most cases, this error is transitory or short-lived issues caused by very slight interruptions in communications between the computer being updated and the Windows update servers.

To assist you with this, let us try running the Network troubleshooter and see if it would do the trick. To do this:

  1. In the search box on the taskbar, type Network troubleshooter, and then select Identify and repair network problems from the list of results.
  2. Follow the steps in the troubleshooter and see if that fixes the problem.

If that doesn’t fix your connection problem, try these things:

  • Reset the TCP/IP stack
  • Release the IP address
  • Renew the IP address
  • Flush and reset the DNS client resolver cache
To run these networking commands in a command prompt window
  1. In the search box on the taskbar, type Command prompt, press and hold (or right-click) Command prompt, and then select Run as administrator > Yes.
  2. At the command prompt, run the following commands in the listed order, and then check to see if that fixes your connection problem:
    • Type netsh winsock reset and press Enter.
    • Type netsh int ip reset and press Enter.
    • Type ipconfig /release and press Enter.
    • Type ipconfig /renew and press Enter.
    • Type ipconfig /flushdns and press Enter.

Since you recently updated Windows, it's possible that your network adapter driver needs to be updated as well. While it's likely you already have the most recent driver, if you're having trouble with a device, it would be best to have the latest update installed on your PC. To do this:

  1. In the search box on the taskbar, type Device Manager, and then select Device Manager from the list of results.
  2. In Device Manager, select Network adapters > the network adapter name.
  3. Press and hold (or right-click) the network adapter, and then select Update driver > Search automatically for updated driver software. Follow the steps, then select Close.
  4. After installing the updated driver, select the Start button > Power  > Restart if you're asked to restart, and see if that fixes the connection issue.

If Windows can’t find a new driver for your network adapter, visit the PC manufacturer’s website and download the latest network adapter driver from there. If your PC can't connect to the Internet, you'll need to download a driver on a different PC and save it to a USB flash drive, so you can manually install the driver on your PC. You’ll need to know the PC manufacturer and model name or number.

To manually install the network adapter driver

Do one of the following, depending on what kind of file you downloaded from the PC manufacturer’s website:

  • If you downloaded an executable (.exe) file, just double-click the file to run it and install the drivers. That should be all you need to do.
  • If you downloaded individual files, and at least one file has an .inf file name extension and another has a .sys extension, do the following:
     
    1. In the search box on the taskbar, type Device Manager, and then select Device Manager from the list of results.
    2. In Device Manager, select Network adapters > the network adapter name. (If it’s not listed there, check in Other devices.)
    3. Press and hold (or right-click) the network adapter, and then select Update driver > Browse my computer for driver software.
    4. Select Browse > select the location where the driver files are stored > OK.
    5. Select Next > follow the steps to install the driver > Close.
    6. After you've updated the driver, select the Start button > Power  > Restart if you're asked to restart, and see if that fixes the connection issue.

Check your hotspot and see if it changes anything.

ken sheridan say: Your expression won't work as the Count operator counts the instances over a set of values.  What you appear to be trying to do is count those distinct months which have a value in the Supplemental column.  You can do this with a query which uses the DISTINCT predicate in the SELECT clause of a subquery, e.g.

SELECT COUNT(*) AS DistinctCount
FROM (SELECT DISTINCT [Service Month]
            FROM [YourtablenameGoesHere]
            WHERE NZ(Supplemental, "") <> "") AS DistinctMonths;

This will count the distinct months with a value in Supplemental over the whole table.  To restrict the count to a subset of rows you would need to include relevant criteria in the subquery's WHERE clause, e.g. to restrict it to the current year:

SELECT COUNT(*) AS DistinctCount
FROM (SELECT DISTINCT [Service Month]
            FROM [YourtablenameGoesHere]
            WHERE NZ(Supplemental, "") <> ""
            AND RIGHT([Service Month],4) = YEAR(DATE())) AS DistinctMonths;

More likely the extra criteria in the subquery's WHERE clause would reference controls in a dialogue form from which the report is opened , where the controls are parameters to restrict the report's coverage.

You can then reference the query in the report wih:

=DLookup("DistinctCount","YourQueryNameGoesHere")

jomel_santos say:

Hi,

This could be an issue with the compatibility of the application itself since the Insider build was just released. If you have downloaded the build through the Fast ring, issues can still occur. For us to effectively assist you, kindly confirm if you have downloaded the build through the Fast ring. Also, tell us what is the exact error message if you're receiving any.

Meanwhile, we suggest resetting the mail app by going to  Settings > Apps > Apps and Features > Click on the Mail and Calendar App > Advanced Options > Reset.

If that doesn't work, try to re-register the Mail and Calendar app again:

Re-register the mail and calendar app:

1. Right click on the start menu and right click on Powershell. Select Run as Administrator.
2. Copy and paste the following command:

get-appxpackage *microsoft.windowscommunicationsapps* | remove-appxpackage

3. Go to the Windows Store and download the Mail and Calendar app again.

Do get back to us with the results.

Dax,

I had this problem as well.

The solution appears to be to do the 'fast reset'.

I think that some account settings became scrambled during one of the updates.

Doing the fast reset - keep files, do not erase discs - resets your account settings.

Once this has been done you should be able to add/remove email accounts.

This is bit of a painful option - it may remove some apps  (visual studio,  MacAfee anti-virus, Linq Pad, etc,) and you may have to re-install some of the apps that have been removed.

I've just done this myself and 'Mail and Calendar' is back to normal.
