jeeped say:

I believe you should base your conditional formatting rule on a formula and while a fairly complex formula using native Excel function could be constructed, it would make life infinitely easier to add a User Defined Function (aka UDF) that will single-handedly resolve the upper and lower tolerance limits and the two values from a single cell.

Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module (e.g. Alt+I, M). Paste the following into the new pane titled something like Book1 = Module1 (Code),

Public Function udf_Tolerance(c As Range, t As Range) As Long
    Dim c1 As Double, c2 As Double, tl As Double, tu As Double
    On Error GoTo ErrorOut

    ' split on <space>±<space> and adjust upper and lower
    tl = CDbl(Split(t.Value, " ± ")(0)) - CDbl(Split(t.Value, " ± ")(1))
    tu = CDbl(Split(t.Value, " ± ")(0)) + CDbl(Split(t.Value, " ± ")(1))

    ' split on hyphen
    c1 = CDbl(Split(c.Value, Chr(45))(0))
    c2 = CDbl(Split(c.Value, Chr(45))(1))

    udf_Tolerance = Abs(CBool(c1 >= tl And c1 <= tu And c2 >= tl And c2 <= tu)) + 1
    Exit Function
ErrorOut:
    udf_Tolerance = 0
End Function

Tap Alt+Q to return to your worksheet. Use this function just as you would any native Excel function. The syntax is,

=udf_Tolerance(<cell with two values>, <cell with tolerance>)

This will return a 1 for outside tolerance, a 2 for within tolerance and a 0 on error (e.g. blank cell, improper format in cell, etc) if it cannot resolve the values supplied.

You can use this with two conditional formatting rules based upon a formula. For an Applies to: range of =$E$8:$E$35,$I$8:$J$35 and within tolerance for green, this would be,

=udf_Tolerance(E8,E$6)=2

A second rule for the same Applies to: range of =$E$8:$E$35,$I$8:$J$35 and outside tolerance for for, this would be,

=udf_Tolerance(E8,E$6)=1

Note that this UDF cannot be used on columns G:H as the column header showing the tolerance is in a different format than the others.

rohn007 say:
Try this:
<snip>
Macro to Always open doc with Nav Pane displayed

If you just want the action assigned to this document (and not to the Normal template) you might try adding a macro called Document_Open() to the object called ThisDocument.

Note that there is one of these objects attached to the Normal template so make sure you edit the version of ThisDocument that is attached to Project(name of your file) under Microsoft Word Objects. Use the code that Stefan supplied in his DisplayNavPane() macro, so:

Sub  Document_Open() 

   Application.TaskPanes(wdTaskPaneNav).Visible = True

End Sub

 

When you save the document the macro will stay with it.

I'm assuming you know how to get into viewing/editing macros.

Every time this document is opened, no matter which machine opens it, your Navigation Pane should appear, whether the particular user's Normal template is set to display it or not. Open it with the shift key pressed and the macro will be bypassed, if you need for some reason to temporarily prevent it from running. Disable it completely but keep it present by simply renaming it.
