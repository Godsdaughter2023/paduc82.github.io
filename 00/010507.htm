jcmu99 say:

How do you set a margin for one page only Word 2010?

Please see my screenshot below --I see how to set a margin for whole document and this point forward but is there a way to apply it to a single page or selected pages?

suzanne s. barnhill say:

Margins are a section property. In order to have different margins, the page will have to be a separate section. When you select "This point forward," Word inserts a section break for you.

Often you don't actually need different margins (and hence a separate section) for a single page. If it's the side margins you need to change, you can accomplish this with paragraph indents (positive or negative). To increase top and bottom margins, you can add empty paragraphs. But of course if you need to make the top and/or bottom margin smaller, then you will need a separate section.

PS commands are different in different version of AzureAD module. Please download and install the AzureADPreview 2.0.0.114  and then apply the required template on your AzureAD settings and then customize this template, apply it to the specific group.

All the command here：

Save-Module -Name AzureADPreview -Path c:\new -requiredversion 2.0.0.114

Install-module -name azureadpreview -requiredversion 2.0.0.114

connect-azuread

$temp =Get-AzureADDirectorySettingTemplate |Where-Object {$_.displayname -eq "Group.unified"}

$setting =$temp.createDirectorySetting()

new-azureaddirectorySetting -directorySetting $setting

get-azureaddirectorySetting |foreach values


$group = get-azureadgroup | where-object {$_.displayname -eq "xxxgroup"}

$settings = get-azureadgroup | where-object {$_.displayname -eq "Group.unified"}
$settings["AllowToAddGuest"] = "False"

Set-azureaddirectorysetting -objectid $settings.id -directorysetting $settings

Chesspin1: my 8X was bricked too from the Windows Insider WP10TP botched install, however a Recovery File was found by WDRT some weeks ago - that appeared to be carrier/model-specific. I know you are not alone in not receiving a Recovery File via WDRT, however I have 2 suggestions:

  1. I have seen some users report that WDRT kept saying no firmware was found on the server, but strange as it sounds, it was only after they changed USB ports and/or USB cables that one was found. If you've been using a USB3 port, try a USB2 port (and vice-versa, though it seems USB2 is more likely to work). Change cables, if possible, and try the different ports again. Plug directly into computer, don't use a USB hub.
  2. There was an official acknowledgement on the Insider forums some time ago that Microsoft was working with mobile operators to get all the recovery files (that they had some already, and were working on getting others). I have seen lots of people post separately words to the effect of "I'm still waiting", but what I have not yet seen is one thread where anyone (whatever their phone) who is still waiting for their Recovery File can speak up, complete with specific details about their phone that might help Microsoft determine which recovery files they still need to obtain from the manufacturers.

    So my suggestion is to do that, to start a new thread in the Insider Program->Preview->Update->Phone forum titled something like "Post here if you're still waiting for a WP8.1 Recovery File", and state in it (and request others do the same) the following info:

    - Phone name and original carrier, ex: "HTC 8X, originally on Rogers"
    - Model ID, ex: "PM2330002"
    .. this info is found on the 1st line on the bootloader screen (reached via holding VOL-UP AND VOL-DOWN during boot up)
    - CID (Carrier ID), ex: "ROGER001"
    .. the CID identifies what the phone's original carrier-branding was, and it seems the Recovery File your phone requires is specific to the CID.

Good luck!
