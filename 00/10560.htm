It is possible to set up VBA event code to sort the data as you enter each item.

I do not recommend this practice.

A "watermark" is simply a graphic with the contrast and brightness adjusted to achieve the desired result.  Therefore, you need to make those adjustments to the graphics in the document in which you have the graphic in the individual cells.

suzanne s. barnhill say:

To add to what Doug has said, watermarks are formatted as 85% brightness and 15% contrast. In earlier versions, this is easily applied by selecting Washout from the Color dropdown on the Picture toolbar that appears when you select a graphic. In Word 2007, it's a bit more difficult to find but still present. On the contextual Picture Tools | Format tab, in the Adjust group, click Recolor. On the dropdown, under Color Modes, the third from the left is Washout.

 

If you then select Picture Corrections Options from the Brightness dropdown, you'll see Word has formatted the picture as 70% Brightness and -70% Contrast, as the scale has been redefined in Word 2007 (instead of 0 to 100%, with normal being 50%, it's now -50% to 50% with normal being 0).

rob koch say:

The URL filtering performed by Microsoft is all handled via the core SmartScreen Filter technology mentioned in the following article.  Though it specifically mentions Vista, the same basic methods still apply, it was simply first published at that time.

http://windows.microsoft.com/en-US/windows-vista/SmartScreen-Filter-Information-for-administrators-and-website-owners

If your website or a specific URL found on it becomes blocked by Microsoft, it will display a red screen indicating that fact and offering additional selections to both manually bypass the block and also to submit a request to have the site removed from blocking.  Though I don't recall how specific the information displayed is about the particular violation, I had successfully gotten a site removed from blocking after making changes and submitting a couple years ago.

The components of URL blocking that are embedded within MSE and the other Microsoft security products are all linked on the back end into the SmartScreen Filter system, so making the request there will eventually update all of those other security products as well.

Rob

diane poremsky mvp (slipstick.com) say:

You cannot control delivery receipts - it's up to the recipient's email administrator whether to enable it.

Delivery receipts are generated by the recipient's server and many companies turn this off - gmail, yahoo, outlook.com, and godaddy do not send delivery receipts (those are the servers I have accounts on, along with office 365 exchange online, which does return receipts, at least from other exchange online domains). You may receive a message similar to this when the server doesn't generate a return receipt: Delivery to these recipients or groups is complete, but no delivery notification was sent by the destination server, but that is the best you'll do.

If you need to know they received the message, email is probably not the best way to communicate.

roady [mvp] say:

This is not an issues caused by Outlook.

Most mail servers are configured not to send out delivery receipts to prevent harvesting of existing email addresses.

A spammer could then send out lots of emails to random addresses and see for which they get a delivery receipt which would confirm the existence of the address and be included in future spam rounds as well.
