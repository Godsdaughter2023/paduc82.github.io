In this screenshot, the column labelled "t" is used as a data series instead of the X axis value.

Edit the data source for the chart and remove the "t" data series. 

Then edit the "y" data series and make sure that the X and the Y  values in the series dialog refer to the correct cells.

In some versions of Excel, the first row in a table will be interpreted as a series name and creating an XY chart will result in two series, one for the X value and one for the Y value, plotted on a numbered category X axis.

In Excel 2013, though, using a data set similar to your data layout will produce a correct XY chart. See this screenshot with some fantasy data:


I wonder which steps you took in Excel 2013 (the version you flagged this question with) to arrive at the chart you show in your screenshot. If you can retrace these steps and show them here, I'm happy to help troubleshoot the approach.

jeeped say:
You can SUM() alternating columns using SUMPRODUCT() with a criteria like ISEVEN() or MOD() on the column index numbers, but you cannot have text in between the valid columns as these will processed (although subsequently discarded) and an error will be thrown when the text values are used mathematically.

This method will quickly self-destruct as soon as you insert columns that break the sequence that the SUM this, not that rules are based upon. Seeing as you only have 12 columns and the ability to insert or delete columns between the months seems important, you are probably best off to simply write the twelve months longhand. Although this makes for a lengthy formula, the cell addresses will survive the insertion or deletion of columns.
