rob koch say:

Adding to Le Boule's recommendation, here's an older thread on the TechNet website discussing SmartScreen Application Reputation with links to other Microsoft Blog articles as well.

SmartScreen and application reputation

Since your question really relates to application development, the MSDN Forums would be the most likely place to find others with similar issues, but either that or TechNet would be better than here as already mentioned, since these forums are meant for consumer PC users.

Note that based on my understanding of application reputation, the more downloads a file receives the better the reputation.  So those clicking 'Run anyway' will add to this download count while those aborting the download in any way will simply not increase the counter.

The only way that I'm aware that reputation can be decreased is if an executable file contains malware, which can not only decrease the reputation for that specific file (reset it to 0), but also the certificate used to sign the file if one exists, which affects all of the other executable files signed using the same certificate from that specific publisher.

See the Best Practices section of the following article for more details about how to best establish reputation.  The entire article also discusses the newer EV certificates that are the only way to eventually build enough reputation that the SmartScreen warnings will never appear for newly distributed executables.

Microsoft SmartScreen & Extended Validation (EV) Code Signing Certificates
