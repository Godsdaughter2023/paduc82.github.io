Assuming you have two ranges, one for the dates, and one for the times, and that those two ranges are named "Dates" and "Times", and given a certain date that you want to compare against in cell D1:

=COUNTIFS(Dates,D1,Times,"<17:00")+COUNTIFS(Dates,D1-1,Times,">=17:00")

I am also assuming that from 17h00 is considered the next day, not up to 17h00 is considered the current day (in which case it would be):

If the dates and times are stored together in a single column, then just simplify this formula accordingly.

An Access 2003 database running on XP OS has a Personnel table (tbl_personnel) containing the fields PM_ID (a unique personnel number assigned on joining the company) PK; LastName; GivenNames; BirthDate and Gender.  From time to time, Head Office releases a downloaded MS Access table (tbl_pmdownload) containing updated data (PM_ID PK) for tbl_personnel. Is it better to use an append query or an update query (or combination) to achieve the desired updating of the local table? I have tried an append query by itself but am getting the error message:

 

"Microsoft Office Access can't append all the records in the append query.

Microsoft Office Access set 0 field(s) to Null due to a type conversion failure, and it didn't add 236 record(s) to the table due to key violations, 0 record(s) due to lock violations, and 0 record(s) due to validation rule violations."

 

This is the code I have come up with at present...

 

INSERT INTO Tbl_Personnel ( Gender, LastName, GivenNames, BirthDate, PM_ID )

SELECT Tbl_pmdownload.Gender, Tbl_pmdownload.LastName, Tbl_pmdownload.GivenNames, Tbl_pmdownload.BirthDate, Tbl_Personnel.PM_ID

FROM Tbl_pmdownload LEFT JOIN Tbl_Personnel ON Tbl_pmdownload.PM_ID = Tbl_Personnel.PM_ID

ken sheridan say: You need to do this in two stages:

1.  Delete all rows from Tbl_Personnel which have matches in Tbl_pmdownload with:

DELETE *
FROM Personnel
WHERE EXISTS
    (SELECT *
     FROM Tbl_pmdownload
     WHERE Tbl_pmdownload.PM_ID = Personnel.PM_ID);

Any existing rows not matched in the imported data will be retained.

2.  Insert all rows from Tbl_pmdownload  into Tbl_Personnel with:

INSERT INTO Tbl_Personnel
SELECT * FROM Tbl_pmdownload;

This will replace the rows deleted in stage 1 with the current imported personnel data, and will add any rows for new employees not previously represented in Tbl_Personnel.

Before undertaking any set operations like this it is of course imperative that Tbl_Personnel be backed up.
