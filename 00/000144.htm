Weighted Standard Deviation - Microsoft Community

scott93063 say:

Weighted Standard Deviation

I want to calculate how balanced a series of three numbers are.  Calculating the standard deviation only works when the basis is the same.  For example:

     A    B    C     Average     Std Dev.P

1   6    5    4         5                .8

2   8    4    3         5               2.1               obviously, row 2 is not as close to balanced as row 1 and has a higher standard deviation

3   30  25  20        25             4.1               in this case row 3 is closer to being balanced than row 2, but the standard deviation is higher

4   35  23  17        25             7.5               row 4 has the same average as row 3, is not as balanced and has a higher standard deviation

5  255 250 245     250            4.1               in this case, row 5 has the same standard deviation as row 3, but is closer to being balanced

These numbers come from 3-phase power measurements (in Amps).  Balancing them improves the overall utilization.   I've tried other excel functions like estimated standard deviation, variance, average deviation, and deviation squares, but none give the answers that I want to rate the rows from most to least balanced.

One method that seems to work is to weight the results by dividing the standard deviation by the average.  This gives the right info:

               std dev.P / avg

Row 1              .16

       2              .43          <- this series is the least balanced

       3              .16

       4              .30

       5              .02          <- this series is the most balanced

But, I don't know if this is the right way to calculate the weighted deviation.   Is this correct ?  Is there an Excel formula I've missed ?

joeu2004 say: Scott wrote;

One method that seems to work is to weight the results by dividing the standard deviation by the average.  [....] But, I don't know if this is the right way to calculate the weighted deviation.   Is this correct ?

What you describe is the relative standard deviation, not the weighted standard deviation.  It is a percentage measurement; that is, the standard deviation is a percentage of the mean.

And yes, relative standard deviation is a useful statistic for comparing the standard deviations of sets of samples.

In contrast, the weighted standard deviation is the square root of the weighted variance.


mike middleton say:

I suggest that the "correct" answer depends on what you want to define as "balanced."

Your relative standard deviation, StDevP/Average, may be appropriate for your purposes.

You would obtain similar results (for your five examples) if you express each of the three values as a percent of average and then compute the standard deviation of those three percent-of-average values.

Note that in either case, when you use standard deviation, that each deviation-from-average is squared (as a step in the computations), so an extreme deviation contributes much more to the standard deviation result.

If you want to avoid that "magnification" effect, you could use Excel formulas to compute the mean of the absolute percent deviations (or some other formula that avoids squaring the deviations).

- Mike Middleton, www.MikeMiddleton.com

How to set a word limit in a text box? - Microsoft Community

tbidadmin say:

How to set a word limit in a text box?

Hi there! 

Does anyone know if there is a capability to set a word limit in a text box? For example, if submitting an application of sorts you can cut it off at 250 words?

Thanks!

stefan blom say:
No, you can't set a character limit. However, text boxes inserted via Insert tab | Text Box | Draw Text Box have a fixed height. Typing "excessive" into such a text box would mean that the excessive text disappears. While that isn't exactly what you are trying to accomplish, it is rather useful. 

john korchok say:

For a standard text box, Stefan's answer is correct. If you are referring to an ActiveX Text Box, as found on the Developer tab, then you can set a maximum length by right-clicking on the text box, choosing Properties and setting the MaxLength property to 250.

A similar control is the Text Form Field in Legacy  Forms Controls. Right-click on it, choose Properties and set Maximum length to 250.

stefan blom say:
Good point. Note that for a legacy text form field, setting the maximum length still doesn't prevent a user from pressing Enter "excessively." Putting the text form field inside a table cell with fixed dimensions works around that.

Using Word for Mac v. 15.23 (with OS El Capitan 10.11.6) and suddenly the program changes the language mid-document so basic English words show as errors. When I click on the word, the language changes to Italian or French or something else. I've tried every fix in both the OS settings and the Word settings. I've reformatted the Style and it will not keep English as the default.

bob jones aka: cybertaz say:

Language is a property of the text. If a language attribute has been applied directly it overrides the Style. Select all [Command+A] then change the language attribute by going to Tools> Language or by using the Language button on the Review tab.

Another option is to select the errant text then use the Clear Formatting item on the Styles Pane to restore the underlying Normal Style. The potential problems with that are doing so removes all formatting & you're likely to miss something due to the Language property being 'invisible'.

Tip: The type of scenario you described often is the result of content having been copied from another source, especially web pages. Unless you're certain about what you're copying after you paste use the Paste Options button to select Match Destination Formatting or Keep Text Only, or use Edit> Paste and Match formatting [Shift+Option+Command+V] to paste.

Macro settings are user-level settings, not workbook-level settings. If the user has selected "Enable all macros", macros will be allowed without a popup. If they have selected "Disable all macros without notification", macros will be disabled without a popup. There is nothing you can do in your workbook to change this.
