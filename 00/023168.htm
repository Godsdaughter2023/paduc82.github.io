john mcghie say:
Hi John:

You're making perfect sense.  The "Document Inspector" exists only in PC versions, and enables you to find comments or change tracking that the author may otherwise have hidden.

Mac Word does it a little differently: on opening the file, all tracked changes or comments in the document will be forced visible, whether the last user hid them or not.  So if they are there, you will see them.

If you "Accept All", then the changes will be deleted during the next save.  Some people forget to save the file, then wonder why the changes are still tracked :-)  Same with Comments: the deletions do not leave the file until you save and close.

You can turn on Word>Preferences>Security>Privacy Options>"Warn before printing, saving, or sending..."

You might like to suggest to your people that they sharply reduce their use of tracked changes.  It leads to lots of busted documents.  We are still working on Microsoft to fix these bugs.  But so far, they have decided that fixing this is too hard, so I would try not to use the function in Mac Word.

This is a known problem. Office applications use # to separate the parts of a hyperlink: the display text, the address and the subaddress. So an actual # in a URL causes confusion.

Some people report that using another browser as default works. It doesn't work for some others, though...

# is considered to be an unsafe character that should be avoided in URLs...
