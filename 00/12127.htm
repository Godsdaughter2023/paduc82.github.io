How do I get the TOC to include a letter I am adding in the footer - Microsoft Community

dc_lady say:

How do I get the TOC to include a letter I am adding in the footer before each page number?

In one section I am using number like "D-1" and I want that full page number to show in the TOC.
suzanne s. barnhill say:

The page number that Word will include in the TOC is the number that is displayed on the page by the PAGE field (or the number that would be displayed if the page had a printed number). You can add anything you like before or after the PAGE field, but the TOC will not pick it up. The only way to add this type of prefix is to use the "Include chapter number" feature in the Page Number Format dialog. And the only thing Word will accept as the "chapter number" is the number (it can be a letter, of course) of an autonumbered paragraph in one of Word's built-in heading styles.

What this means for practical purposes is that if you have appendixes numbered A, B, C, D, and so on, you can add A, B, C numbering to the style you are using for the appendix numbers or titles, and, provided that style is one of the heading styles Heading 1-9, then it can be included in the page number.

For more on numbering appendixes (if that's your situation), see http://www.ShaunaKelly.com/word/numbering/NumberingAppendixes.html. For more on including chapter numbering, see http://word.mvps.org/FAQs/Numbering/ChapterNumber.htm.

The above describes the usual case. There is one other rather devious method, described at http://sbarnhill.mvps.org/WordFAQs/TOCTips.htm#ChapterNumbering, which may or may not work for you if the letters you want to use are not sequential.

Hello,

I have solved this problem.  Further investigation revealed that the File Permissions for the relevant "Local App Data"

folder (OneDrive folder in Local App Data) was set to No Permissions.  I therefore completed the following:

1) Edited the Folder Permissions to match my configuration for OneNote  and Outlook (nearby listings in the Local App Data Folder)

2) Deleted the existing OneDrive Credentials

3) Rebooted

4) OneDrive started by itself, and I signed in without errors.
