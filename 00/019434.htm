dj_senseless69 say: Managed to fix the problem!

I've been reading through a few posts in the forum. Couldn't find anything specifically for the problem I was having however I noticed a lot of posts mention the same processes. I've listed them below with a few tips:

1. Uninstall all Nokia Software from Add/Remove programs (inluding PC Connectivity Solution, which doesn't actually mention that it's a Nokia program until you click to uninstall it)

2. Delete all Application Data (click run on the start menu and type %appdata%. Delete anything related to Nokia i.e. all folders called Nokia, PC Suite or NSeries)

3. Delete Nokia registry entries (click run on the start menu and type Regedit. Search through the registry and delete entries related to Nokia, PC Suite and NSeries. I only deleted folders called the above names, not single entries in other folders called something else. I also didn'd delete anything that looked like it was related to a USB connection)

4. Download & Run Nokia PCSuite Cleaner (Lots of people mentioned this but didn't say where to get it from so after 15 mins of searching I found it. Here is a link to the Nokia download page: https://www.nokiausa.com/A4410030. When i tried clicking download it didn;t work so here's another link: http://download.chip.eu/en/Nokia-PC-Suite-Cleaner-3.6_977782.html. When running this I selected the option called remove files and registry entries. I got an error message after telling me it couldn't find certain things and to restart but still seemed to work)

5. Reinstall PCSuite (I installed the latest version again, this time onto my C:. Not sure if that really made any difference but I wanted to install it to a different location just in case)

That has solved the problem. I can now view my messages and contacts and browse all files (Phone Memory & Mass Memory).

Hopefully this will help when others get the same problem, and I wouldn't be surprised if it fixed other problems too.

nathan o. - microsoft support say:

When you created this key in your local machine hive, what was the exact path of the key you were trying to set?  If you just modified the current user path and pointed it to a local machine path (basically substituting HKEY_LOCAL_MACHINE or HKEY_CURRENT_USER), this most likely won’t work if you are on a 64 bit OS and using the 32 bit version of Access 2010.  If you are using a 64 bit OS and the 32 bit version of Access, you will need to use the key within the Wow6432Node.  For example:

 

HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Office\14.0\Access\Security\Trusted Locations\Location9

 

So if you have a working key within the current user hive, you can export it and modify this .reg file, but when doing so just make sure the key begins with: HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node

 

If you are on a 32 bit OS or you are using the 64 bit version of Office, then just modifying substituting HKEY_LOCAL_MACHINE or HKEY_CURRENT_USER should work.

 

 

Best Regards,

Nathan O.

Microsoft Online Community Support

Nothing in an index is linked. Bear in mind, too, that TOC entries are linked to a specific heading; an index entry wouldn't have anything other than an INDEX field for a potential 'link'.
