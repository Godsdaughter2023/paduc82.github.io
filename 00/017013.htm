It would be possible to have the calling code wait while the form is loaded and visible.  A procedure for that might look like this:

'------ start of code ------
Sub WaitWhileVisible(FormName As String)

    Dim blnExit As Boolean
   
    Do
   
        DoEvents
       
        If CurrentProject.AllForms(FormName).IsLoaded Then
            If Forms(FormName).Visible = False Then
                blnExit = True
            End If
        Else
            blnExit = True
        End If
   
    Loop Until blnExit

End Sub
'------ end of code ------

Ignore the YEAR when comparing dates in a formula

I am keeping track of upcoming birthdays.  I'm looking for a conditional format that will highlight the cell if a birth day and month (ignore the year) are within the next 10 days.  Currently if I compare a birthday, 04/6/59 to today's date + the date 10 days forward, nothing happens because 04/6/59 does NOT fall between 04/03/12 and 04/13/12.  I want it to ignore the year in the formula and only evaluate whether 04/06 falls between 04/03 and 04/13.  Thanks in advance, Judy
hansv mvp say:

Let's say the birth dates are in B2:B50.

Select this range, then select Format | Conditional Formatting...

Select Formula Is from the first dropdown, then enter the following formula in the box next to it:

 

=AND(DATE(YEAR(TODAY()),MONTH(B2),DAY(B2))>TODAY(),DATE(YEAR(TODAY()),MONTH(B2),DAY(B2))<=TODAY()+10)

 

Here, B2 is the active cell within the selected range.

Click Format... and specify the desired formatting.

Click OK twice.

andrew simpson uk say:

I've done something similar in VBA. I've implemented a solution whereby the user can generate a report titled the "Forecast Worksheet" from the schedule in MS Project , which is in MS Excel. This worksheet is sent round to various non-MSP users, who can then update the ETCs and % complete, and the Planner can then import these updated values back into MSP.

 

I can't share the code, but some high level pointers if you decided to develop something similar yourself:

  • You need to ensure you've got a reliable foreign key linking the schedule and the excel data. I achieved this by generating the worksheet from the MSP schedule which meant the task number in Excel matched exactly what was in MSP - you don't have to do it this way, but you'll need to be accurate when entering the unique identifiers/foreign keys that link both data sets.
  • Ideallly, you'd want to add a button on the MSP toolbar to start the import - but there are a series of checks you'll want to do before it launches in, i.e. check that there actually is a project (application.projects.count>0), that the activeproject contains tasks (activeproject.tasks.count>0), possible also check that the status date has been set.
  • The button would launch a browse for file window - from here you'd locate the Excel version of the schedule and click Open.
  • This would kick off the import routine, looping through each line in the excel file, and for each excel entry looping through the tasks in MSP looking for a match on the task number (whichever field you decide to use). When the routine finds a match, you then update the MSP values you want changed.
  • The routine will need to build an error log as it goes along, identifying Excel entries that can't be resolved to MSP entries (or MSP entries that can't be resolved to Excel depending on your validation rules), duplicate entries, and any other kind of errors that you may encounter. On completion, if any error were encountered we should present this log to the user allowing them to review the issues identified.

It's quite a lot of work to develop the solution - starting from scratch it took me about 5 days, so you need to be sure that it will save you/your team at least that amount of time throughout its life.

 

Hope that helps,

SuperFetch attempts to predict which applications you will launch next and preloads all of the necessary data into memory. Its job is to maintain and improve system performance over time.
