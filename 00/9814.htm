Convert Salary Wage to Hourly Wage - Microsoft Community

kenthortin say:

Convert Salary Wage to Hourly Wage

I need a formula that would convert a salary to an hourly wage based upon 50 hours.

In other words if a person makes $42,640 a year on salary based upon 50 hours.  I need to convert that to an hourly wage so that if they still work 50 hours they will get paid the same.  40 hours would be a an hourly rate and the last 10 hours would be at time and a half.

joeu2004 say:

KentHortin wrote:

if a person makes $42,640 a year on salary based upon 50 hours.  I need to convert that to an hourly wage so that if they still work 50 hours they will get paid the same.  40 hours would be a an hourly rate and the last 10 hours would be at time and a half.

If A1 is the annual wage, the hour rate is:

=ROUND(A1/52/(40+10*1.5), 2)

Of course, you can simplify 40+10.15 to 55.  I wrote it that way:  (a) so you would understand the derivation of 55; and (b) so you could change the number of regular and time-and-a-half hours.  (40 and 10 could be in B1 and C1, for example.)

Caveat:  You cannot ensure that "they will get paid the same" if they work for a year, due to the necessary real-world rounding.  In this example, the hour rate is $14.91.  Note that 14.91*(40+10*1.5)*52 = $42,642.60.

[EDIT] PS....  I think the above is the correct calculation for payroll.  But arguably, there are 365/7 weeks in a year.  In that case, the hourly rate formula is:

=ROUND(A1*7/365/(40+10*1.5),2)

which is  $14.87 for the example.  And the full-year pay would be:

=ROUND(14.87*(40+10*1.5)*365/7,2)

which is $42,645.04.

Sub UnassignShortcut()
    Application.OnKey "%."

macropod say:

One would ordinarily expect the bottom border for the first column to be the same as for the others - either all or have a bottom border or none do. However, the programming in Word doesn’t understand the conditional formatting used by Table Styles, so it gets confused by the first column. What typically happens is that Word adds a “bottom border” to the table border for the first column on the page boundary (so the table doesn’t just get cut off in the middle). The following macro works around the problem by initially re-applying the Table Style, thus erasing any underlining that might have shifted position as a result of rows changing position, then finds whatever rows are at the bottom of a page and applies the same border formatting to their row's first cells as the last row's first cell has.

Sub CorrectTblBorders()
Application.ScreenUpdating = False
Dim Tbl As Table, iRw As Long, iPg As Long
With Selection
  If .Information(wdWithInTable) = False Then Exit Sub
  Set Tbl = .Tables(1)
End With
With Tbl
  .Style = .Style
  iPg = .Rows(1).Range.Information(wdActiveEndPageNumber)
  If .Rows(.Rows.Count).Range.Information(wdActiveEndPageNumber) = iPg Then GoTo Done
  For iRw = 2 To .Rows.Count
    If .Rows(iRw).Range.Information(wdActiveEndPageNumber) = (iPg + 1) Then
      .Rows(iRw - 1).Cells(1).Borders = .Rows(.Rows.Count).Cells(1).Borders
      iPg = iPg + 1
    End If
  Next iRw
End With
Done:
Set Tbl = Nothing
Application.ScreenUpdating = True
End Sub
