hansv mvp say:

Let's say that the form is named frmApplication, and that you have a control on this form bound to the IDNumber field; the name of this control is also IDNumber.

 

Open the query in design view.

Enter the following in the Criteria: row of the IDNumber column:

 

[Forms]![frmApplication]![IDNumber]

 

Close & save the query.

 

Open the form in design view.

Activate the Event tab of the Property Sheet.

Click in the On Current box.

Select [Event Procedure] from the dropdown.

Click the builder dots ... to the right of the dropdown arrow.

Make the code look like this:

 

Private Sub Form_Current()
    Me.[ContactsListBox].Requery

where ContactsListBox is the name of the list box that displays the query results.

ken sheridan say: The fact that the data is imported from Excel is not really relevant to the report.  You would base the report on a query which joins the tables like this:

SELECT TBLDelivery.*, TBLTransport.Transport
FROM TBLDelivery INNER JOIN TBLTransport
ON TBLDelivery.Sent =Transport.Sent;

The query will return some columns from TBLDelivery which you won't want in the report, but that's no problem; you simply bind controls in the report to only those columns which you do want to show, including the Transport column.

A form can similarly be based on the query, but need not be; it can be based solely on TBLDelivery.  There are a number of ways you can then show the Transport value in the form, e.g. you can add an unbound text box control with a ControlSource property of:

=DLookup("Transport","TBLTransport","Sent = """ & [Sent] & """")

This will return the value at the Transport column position in TBLTransport where the value at the Sent column position matches the Sent value in the current row in TBLDelivery.
