When Microsoft starts preparing for a new release, it goes through a process called refactoring. It seems this is introducing some significant code changes that breaks certain protocols and API's required by older apps and drivers. Considering the Windows Driver Model has not changed much in its near 30 year history not mention the application programming interfaces such as Win32, this might be culprit.

Try simplifying your configuration by keeping only your mouse and keyboard attached, that means:

Disconnecting multiple display monitors.
Remove discrete video cards, use the on-board graphics if your motherboard supports it.
Swap out memory modules - a RAM module might be corrupt - you can also ensure RAM sticks are installed slots 1 and 3. Your RAM modules might be corrupt, try scanning them for errors.
If you have more than one hard disk installed, disconnect all except the target disk where Windows 10 will be installed.
Reset your BIOS Defaults.
Disconnect all external devices, since connected devices can sometimes confuse setup.

  1. I turned VT-d off in the BIOS, after reading that there may be some problems with that.  I'm going to turn it back on here in a moment (when my RAID gets done rebuilding) to see if that really affected anything.  It didn't initially, but it may have contributed to the whole.

  3. Running the tool, I downloaded the ISO image for Windows 10 1804 x64, build 17134.

  4. lemp say:

Rather than use the "update driver" button in Device Manager, you generally should get driver updates -- if at all -- from the computer manufacturer.

First, you need to determine which specific edition of Windows 7 you have and whether it is 32-bit or 64-bit.  Click the Start Orb, right-click "Computer" and select "Properties."  Make a note of the Windows edition (at the top of the page) and the "System type" (in the middle of the page).

Now go to http://h20564.www2.hp.com/hpsc/swd/public/readIndex?sp4ts.oid=5229466&swLangOid=8&swEnvOid=4052

Click the + next to "Driver - Graphics" to see what is available for your system. 

For example, if you have a 32-bit version of Windows 7 Home Premium, the graphics driver for you is the "AMD Video Driver and Control Panel (International) version 8.947.1.1-120708A-142653C"  This driver package will actually work for all versions of Windows 7, whether 32-bit or 64-bit, and I suggest that you select this one even if you have Windows 7 Pro 64-bit.

On the other hand, if you have the 64-bit version of Windows 7 Professional, you have a choice between that driver and the "AMD Video Driver and Control Panel (American, International) version 15.201.1301 Rev.F" (and I have no idea of the differences).

Click the link that is the name of the driver package you select (rather than the "Download" button) to see more details, including installation instructions.

Before you download the driver package, which is quite large, open Device Manager, double-click on the entry for your graphics adapter, and select the "Driver" tab. 

  • What is shown for Driver Provider, Driver Date, and Driver Version?
  • Compare these to the driver from the HP web page (date: 20 Jul 2012) (version: 8.947.1.1-120708A-142653C).  The version shown on your computer may only have the first section (8.947.1.1).

If the driver already installed is from AMD or HP (rather than Microsoft) and is newer than the driver from the HP website, it probably will not be worthwhile download and installing the driver from the website.  In this case, post back for further instructions, and be sure to include both whether you a have 32- bit or 64- bit system AND the name of your graphics adapter as shown on the "General" tab of the Device Manager "Properties" for your device.

Tell them the version of Photoshop you have and the error message you received. 

Then explain that you have an HP ProBook4545s with Windows 7 Pro 64-bit and that your laptop has an AMD Radeon HD 7520G with driver version 8.947.1.1000. date: 7/7/2012. 

Ask them if your graphics card will fully support your version of Photoshop with a newer driver or if it is the hardware itself that is lacking in some capability.  See if they know the first version of the driver that should "fully support" Photoshop.  You could also ask them exactly what "features may not work correctly."  Perhaps those are features that you won't care about missing.

If Adobe says that your hardware should be capable of fully supporting Photoshop, then ...

Option 2: 

Use the "AMD Video Driver and Control Panel (American, International) version 15.201.1301 Rev.F" driver from the HP page I linked in my earlier reply.

Note:  Options 3 and 4 will try to get the latest AMD driver for your graphics card.  Although this will probably work, computer manufacturers, particularly for notebook computers, may have customized the hardware.  In this case, the generic driver from the manufacturer of the graphics card (AMD in this case) may or may not enable some custom feature. 

Option 3: 

Go here: http://support.amd.com/en-us/download

In the section headed "Automatically Detect and Install Your Driver" click the "Download now" button and then run the AMD auto-detect utility, which is supposed to determine and install the latest driver.  Instructions here: http://support.amd.com/en-us/kb-articles/Pages/GPU131ObtainGPUDriversAuto.aspx

Option 4:

Go here: http://support.amd.com/en-us/download

Use the "Manually Select Your Driver" section to find and download the driver for your graphics adapter.  This will not be completely straightforward because your particular model isn't listed (see the note above Option 3). 

Read the articles linked under "Driver Install How-to Articles" for advice, but I believe that you will end up at this web page:  http://support.amd.com/en-us/download/desktop/legacy?product=legacy3&os=Windows+7+-+64

The "Windows Certified" (WHQL) driver on this page appears to be slightly newer than the HP driver in Option 1 above, even though it is dated a few months earlier.  This seems hardly likely to make a difference with respect to Photoshop "features."

The second driver (16.2.1) is obviously newer, but it is a "beta" product.  See the note on the release notes (http://support.amd.com/en-us/kb-articles/Pages/AMD-Radeon-Software-Crimson-Edition-16.2.1-for-Non-GCN-Products-Release-Notes.aspx):  "AMD Radeon Software Crimson Edition 16.2.1 Beta Driver for non GCN products is a notebook reference graphics driver with limited support for system vendor specific features."

Firmware Update

  1. Plug the computer into a LAN port via an ethernet cable.
  2. Open a web browser.
  3. In the address bar at the top, type in 192.168.2.1
  4. Click Login in the upper right.
  5. Input the password and click Login (by default, the password should be left blank).
  6. Select Firmware Update from the menu on the left under the Utilities section.
  7. The firmware version should be listed at the top of this page.
  8. Open another tab in the web browser.
  9. Go to www.belkin.com.
  10. Scroll down and click on Support.
  11. Enter the model number of your device in the search box.
  12. Scroll down to the "Downloads" section under the search results.
  13. Click the link with the model number of your device labeled Firmware Update.
  14. Locate the version of the router you have and click where is says download to download the latest version of the firmware.
  15. Save the file to somewhere that is easy to locate such as the desktop.
  16. Go back to the original tab that you opened the router's GUI page with.
  17. Click on browse.
  18. Find the firmware bin file and click open.
  19. Click Update and let the software install without interruption.
