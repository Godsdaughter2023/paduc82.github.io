kb.lee say:

Windows process triggering SP4 fan

SP4 i7/8GB/256GB with current Windows updates + 2/17/2016 firmware

My SP4 was idling on the table without any foreground or background user apps. An excellent state for Windows to enter into low-power mode. To my surprise, something triggered the cooling fan. It sounded like it was spinning at full speed.

I fired up Task Manager. A background process called “Windows Modules Installer Worker” was consuming > 35% CPU while the rest of the processes were sleeping more or less. This process is apparently part of Windows Update and it was supposedly performing update related task in the background. My device was fully updated at that point in time. There was no reason why it was hogging the CPU like a runaway process and causing the fan to come on for extended period of time. If I could put this process in check then I would be eliminating one more source of unnecessary power drain.

I disabled Automatic Windows Update. Keep in mind this is something frowned upon by Microsoft when enforcing Automatic Windows Update is the order of the day to ensure users get timely updates and fixes. I believe users who have the habit to trigger Windows Update regularly should be given the flexibility to opt for Manual Update. However, with Windows 8 and above, users can longer disable Automatic Windows Update conveniently. Nonetheless, You can still turn off Automatic Windows Update using Global Policy Editor. So far so good. I'll see how this impacts battery life... positively I hope.
