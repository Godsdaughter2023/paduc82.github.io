chip pearson say:

The recorded code is using a VERY confusing and poor way to reference cells. The code generated by the recorder is (in part)

Key:=ActiveCell.Offset(0, 1).Range("A1:A11")

Here, the range A1:A11 is relative to ActiveCell.Offset(0,1). If the active cell is, for example, I7, ActiveCell.Offset(0,1) is J7, and the range A1:A11 is RELATIVE TO THAT CELL. So, you are NOT dealing with the actual range A1:A11, you are dealing with the range where the relative A1 is J7, relative A2 is J8, and so on to the relative range A1:A11 being the actual range J7:J17.  This code specifies the key of the sort, setting it to column J. It is further confusing that the key is a range of multiple cells (J7:J17). It is more clear to use a single cell as the key.

The other line you remark upon is

 .SetRange ActiveCell.Offset(-1, 0).Range("A1:B12")  

This is the same confusing reference style as described above. The range A1:B12 does NOT refer to the real range A1:B12. Rather, it specifies a range A1:B12 RELATIVE TO THE ACTIVE CELL. Again, this is EXTREMELY bad programming.

As I say, this is a VERY confusing way to reference cells. I would never use such a method. Why MS decided to record this type of code is beyond me.

 

Cordially, Chip Pearson Microsoft MVP, Excel Pearson Software Consulting, LLC www.cpearson.com
