Alright, then we got it.

The DLL that you use is a 64bit DLL, that's why the RecordCount returns a LongLong. WinXP can only handle 32bit DLLs, that DLL returns a Long datatype.

A LongLong is a 8byte integer which supports larger numbers.

The type-declaration character for LongLong is the caret sign "^", I guess that's why the debug tooltip shows it.

There are several factors here, all of which make Jay's response really the only correct one:

  1. In order to open a document in Draft view at all, you have to have enabled the option to "Allow opening document in Draft view" at File | Options | Advanced: General. If the recipient hasn't enabled this option, a document saved in Draft view will not open in Draft view.

The band failed miserably. The Surge is what the band should have been. The band looks decent but the Surge gets the job done, albeit not being as pretty as the band.
