joeu2004 say:

Apparently, the value in C25 is text, and the value in D25 is numeric.

I am referring to the type of the value, not the format of the cell. In other words, ISTEXT(C25) is TRUE, and ISNUMBER(D25) is TRUE.

In Excel, all text is "greater than" all numbers.

You should correct the data. I suggest that columns B and C contain numeric date and time.

To convert text to numbers, Data > Text To Columns might work.

Almost certainly for column C, except for the unusual regions (?) that separator hours and minutes with something other than colon.

But column B might be problemmatic if the form (apparently m/d/y) differs from your region.

-----

In any case, AND(C25>=13, D25<20) is nonsensical when comparing time. Your intent might be:

AND(HOUR(C25)>=13, HOUR(D25)<20)

That presumes numeric time.

Also, AND(C12>="13:00", D12<"20:00") will not work with numeric time. Instead, write:

AND(C12>=TIME(13,0,0), D12<TIME(20,0,0))

or

AND(C12>=--"13:00", D12<--"20:00")

The double negate (and any idempotent arithmetic) effectively converts a text representation of a number to an actual numeric value. Again, that presumes that Excel recognizes the text as a representation of a number.

ken sheridan say:

Can you do that from a report?  I tried that as well and was unsuccessful.

Let's assume as a simple example that you have tables Customers and Orders, related on a numeric CustomerID as the primary key of the former and a foreign key in the latter.  You might then have a customer's form, in single form view, within which is an orders subform in continuous forms view.

To create a report you have two options:

1.  Base the report on a query which joins the two tables on CustomerID, and in report design view group the report by CustomerID.  Put the customer data in the group header and the orders data in the detail section.  Each customer's data will print once, with their orders data listed below.

2.  Create a report based solely on the Customers table, putting the customer data in the detail section.  Create another report based on the Orders table, again putting the orders data in the detail section.  Then embed the orders report as a subreport in the detail section of the customers report.  The end result will be much the same as 1 above, but a subreport gives you greater flexibility over the layout.  Link the subreport to the parent report on Customer, as you would do with a subform

Whichever way you do it, add a button to the parent customers form and put the following code in its Click event procedure:

    Const REPORT_NAME = "rptCustomerOrders"
    Dim strCriteria As String

    strCriteria = "CustomerID = " & Me.CustomerID

    ' ensure current record is saved
    Me.Dirty = False

    ' print the report, filtered to the current customer
    DoCmd.OpenReport REPORT_NAME, WhereCondition:=strCriteria

where rptCustomerOrders is the name as which you saved the report when you created it.
