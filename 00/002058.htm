ken sheridan say: That is correct ranking by [ETA Date] within each subset of [HA30 MOSC] values, which is what you asked for, "to rank the data in my query first by MOSC and then by the ETA Date".  What you appear to in fact want is sequential numbering of each subset of [HA30 MOSC] values ordered by [ETA Date], [VIN] where [VIN] values are distinct per [ETA Date] value.  You'll find examples of various row numbering and ranking solutions in RowNumbering.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

Translating the appropriate query from my demo to your query's result table should be something like this:

SELECT Q1.[HA30 MOSC], Q1.[VIN], Q1.[ETA Date],COUNT(*) AS Rank
FROM Qry_Open_Vins AS Q1 INNER JOIN Qry_Open_Vins AS Q2
ON (Q2.[VIN]<=Q1.[VIN] OR Q2.[ETA Date]<>Q1.[ETA Date])
AND (Q2.[ETA Date]<=Q1.[ETA Date]) AND (Q2.[HA30 MOSC]=Q1.[HA30 MOSC])
GROUP BY Q1.[HA30 MOSC], Q1.[ETA Date],Q1.[VIN];

rob koch say:

That How-To Geek article does a fairly good job of explaining the key differences between Linux and Windows that make viruses more likely with Windows, though in trying to be diplomatic it isn't completely clear.

Windows has always been an entirely commercial product, so there's significant money involved in both the consumer and commercial sides of the software business and always has been.  Since each Windows system is developed from the base of the version(s) that came before it, many of the malware created for one version can also operate across other versions as well.

In the case of Linux which is Unix-like, the code was written from complete scratch so that the system would act very much like a Unix system, but wouldnâ€™t contain any Unix code.  This means that the often freeware Linux distributions can't be directly infected by malware written for the more commercial Unix systems that preceded it.  This results in a very fragmented application structure which tends to require compilation and even outright re-development on different Linux or Unix platforms, reducing the ability for executable malware to work across large numbers of machines.

So the result of these differences are that Microsoft products tend to make money, with large amounts of common code and so vulnerabilities making development of malware targeting Windows both easier and more lucrative than the more fragmented Linux, who's users are not only generally more technical, but also less wealthy geeks as well.  So any malware attacking Linux would find a much harder time making money, which since that's why most malware is now created means that Linux malware is also less commercially viable than Windows as well.

Finally, your last question about certified applications wouldn't need to be asked if you had even Windows 8, since starting with that version and continuing with Windows 10 Microsoft has added the Windows Store, which supplies applications that are both vetted and contain certificates to verify developer identity and application integrity.  This didn't exist with previous versions such as the Windows 7 you are using and is one of the major security improvements behind Windows 10 and why Microsoft offered the free upgrade to consumers as well.

So there are many reasons both technical and monetary for the behind the popularity of Windows by malware purveyors, the most obvious of which are the far greater numbers.  Since all of the older general purpose computing systems were originally designed for developers, with the smaller, less costly and more mobile devices like phones and tablets becoming more popular with users, the end for these older platforms is coming.  Malware is only one of the reasons, though for many consumers it's likely one of the more important reasons, since it means they don't need to be computer geeks to protect themselves.
