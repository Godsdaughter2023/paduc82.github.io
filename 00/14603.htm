Sub DemoPoly()
    Dim a, b, c
    Dim z, x1, x2

It's a little tricky.

For example, suppose you were looking at historical data from 1985.  The correct date for a specific y value may be in 1986.  So, if the function did any comparisons to today's date, the returned value would be incorrect.  Again,  you will have to decide which value to return based on your data.

D1 is the cell that holds the y value to find.

 

=FindFutureDate(D1,x_Dates,Y_Values)

 

Function FindFutureDate(y, xs As Range, ys As Range)
    Application.Volatile ' Make function Recalculate
    Dim a, b, c 'Coefficients
    Dim z, k    'Temp values
    Dim x1, x2  '2 calculated dates

 

    z = "INDEX(LINEST(#y,#x^{1,2}),1)"
    z = Replace(z, "#x", xs.Address)
    a = Evaluate(Replace(z, "#y", ys.Address))

 

    z = "INDEX(LINEST(#y,#x^{1,2}),1,2)"
    z = Replace(z, "#x", xs.Address)
    b = Evaluate(Replace(z, "#y", ys.Address))

 

    z = "INDEX(LINEST(#y,#x^{1,2}),1,3)"
    z = Replace(z, "#x", xs.Address)
    c = Evaluate(Replace(z, "#y", ys.Address))
   
    k = b ^ 2 + 4 * a * (y - c)
    If k >= 0 Then ' No Complex Values
        x1 = (-b + Sqr(k)) / (2 * a)
        x2 = (-b - Sqr(k)) / (2 * a)
    Else
        x1 = 0: x2 = 0
    End If
   
    'Look at these two dates if interested
'    Debug.Print (FormatDateTime(x1, vbLongDate) & "  &  " & _
                 FormatDateTime(x2, vbLongDate))

    FindFutureDate = FormatDateTime(WorksheetFunction.Max(x1, x2), vbLongDate)
End Function

One description is having data that needs to be processed to obtain output in some other format or perhaps a report based on the source data.

 

The idea is rather that change or rearrange the Source data, the Source data is left as is in a worksheet (or workbook) and the Output data is in another worksheet (or workbook). The big advantage of this is if there is an error in obtaining the desired end result, the Source data is not changed. Also the developer can compare the Source data to the Output data to establish if the Output data is correct.

 

Another area where it is used is Data In is what the user enters and Data Out is what reports etc are obtained from the input data. There has been a saying for a long time now that Garbage In; Garbage Out which means if there is an error entering data then reports etc obtained form the data is not going to be correct.

greg maxey say:

You could use a content control dropdown, its OnExit event, and a couple of bookmarks.

 

Lets say your text a is regular text and text b refers to text defined by the bookmark RefText.  Add another bookmark where the text should appear.  Name it "Target"

 

Add the CC and title it "Toggle" add two list entries "A" and "B"

 

Private Sub Document_ContentControlOnExit(ByVal CC As ContentControl, Cancel As Boolean)
Dim oRng As Range
Dim oFld As Word.Field
Select Case CC.Title
  Case "Toggle"
    Set oRng = ActiveDocument.Bookmarks("Target").Range
    oRng.Delete
    Select Case CC.Range.Text
      Case "A"
        oRng.Text = "Some defined text"
        ActiveDocument.Bookmarks.Add "Target", oRng
      Case "B"
        Set oFld = ActiveDocument.Fields.Add(oRng, wdFieldEmpty, "REF RefText", False)
        Set oRng = ActiveDocument.Range(Start:=oFld.Code.Start, End:=oFld.Result.End)
        ActiveDocument.Bookmarks.Add "Target", oRng
        oFld.ShowCodes = False
        oFld.Update
      Case Else
        oRng.Text = ""
        ActiveDocument.Bookmarks.Add "Target", oRng
  End Select
End Select
End Sub

Outlook (both for the desktop and Outlook.com online) has a folder named "Deleted Items" that it uses to store items you delete.
