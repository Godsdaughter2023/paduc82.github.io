When you need to protect the privacy of an email message, encrypt it. Encrypting an email message in Outlook means it's converted from readable plain text into scrambled cipher text. Office 365 Message Encryption (Information Rights Management) - To use Office 365 Message Encryption, the sender must have Office 365 Message Encryption, which is included in the O365 E3 license.

Function RangetoHTML(rng As Range)


    Dim fso As Object
    Dim ts As Object
    Dim TempFile As String
    Dim TempWB As Workbook
 
    TempFile = Environ$("temp") & "/" & Format(Now, "dd-mm-yy h-mm-ss") & ".htm"
 
    'Copy the range and create a new workbook to past the data in
    rng.Copy
    Set TempWB = Workbooks.Add(1)
    With TempWB.Sheets(1)
        .Cells(1).PasteSpecial Paste:=8
        .Cells(1).PasteSpecial xlPasteValues, , False, False
        .Cells(1).PasteSpecial xlPasteFormats, , False, False
        .Cells(1).Select
        .UsedRange.Columns.AutoFit
        .UsedRange.HorizontalAlignment = xlCenter
        Application.CutCopyMode = False
        On Error Resume Next
        .DrawingObjects.Visible = True
        .DrawingObjects.Delete
        On Error GoTo 0
    End With
 
    'Publish the sheet to a htm file
    With TempWB.PublishObjects.Add( _
         SourceType:=xlSourceRange, _
         Filename:=TempFile, _
         Sheet:=TempWB.Sheets(1).Name, _
         Source:=TempWB.Sheets(1).UsedRange.Address, _
         HtmlType:=xlHtmlStatic)
        .Publish (True)
    End With
 
    'Read all data from the htm file into RangetoHTML
    Set fso = CreateObject("Scripting.FileSystemObject")
    Set ts = fso.GetFile(TempFile).OpenAsTextStream(1, -2)
    RangetoHTML = ts.ReadAll
    ts.Close
    RangetoHTML = Replace(RangetoHTML, "align=center x:publishsource=", _
                          "align=left x:publishsource=")
 
    'Close TempWB
    TempWB.Close savechanges:=False
 
    'Delete the htm file we used in this function
    Kill TempFile
 
    Set ts = Nothing
    Set fso = Nothing
    Set TempWB = Nothing
End Function

foreach($i in get-RemoteMailbox -Onpremisesorganizationalunit "ou=OUName,dc=domain,dc=com" -ResultSize Unlimited) { $i.EmailAddresses | ?{$_.ProxyAddressString -like 'SIP*'} | %{Write-Host $i  $_}}

Remove existing SIP addresses O365 & Onprem:

foreach($i in get-RemoteMailbox -Onpremisesorganizationalunit "ou=OUName,dc=domain,dc=com" -ResultSize Unlimited) { $i.EmailAddresses | ?{$_.ProxyAddressString -like 'SIP*'} | %{Set-RemoteMailbox $i -EmailAddresses @{remove=$_} }}

foreach($i in get-Mailbox -organizationalunit ""ou=OUName,dc=domain,dc=com"" -ResultSize Unlimited) { $i.EmailAddresses | ?{$_.ProxyAddressString -like 'SIP*'} | %{Write-Host $i & Set-Mailbox $i -EmailAddresses @{remove=$_} }}

Add new email address:

set-mailbox -identity username-emailaddresses @{Add='SIP:*** Email address is removed for privacy ***'}

Alternatively one could add SIP addresses to the address policy in Exchange, but since removals do not sync with Azure and it is difficult to get support for that, I figured it's safer for now to identify and adjust existing conflicts only rather than give everyone a new address that cannot be removed.

scottgem say:
First Date is a reserved word in Access and shouldn't be used for Object names.  So add two unbound controls to your form:

txtStart and txtEnd

Set the default values to DMin("[datefield]","tablename") and Dmax(["[datefield]","tablename"). This way the two controls will, by default display the earliest and latest dates. 

Then set the criteria in your query to:

BETWEEN [Forms]![Searchform]![txtStart] AND [Forms]![Searchform]![txtEnd] 

You can set the After Update event event of txtStart to:

Me.txtEnd = Me.txtStart, so it defaults to a 1 day range.

ken sheridan say:

I would like to edit this to allow me to search a range (6/4/2011 - 6/5/2012), etc.. but also still being able to search a specific date.

So if I type in just 6/4/2011 it only returns the results for that day but if I type 6/4/2011 - 6/5/2012 it will show all that fall within that range


A query for that would be like this:

PARAMETERS [Forms]![Searchform]![txtStartDate] DATETIME,
[Forms]![Searchform]![txtEndDate] DATETIME;
SELECT *
FROM [YourTable]
WHERE ([YourDateField] >= [Forms]![Searchform]![txtStartDate]
    AND [YourDateField < NZ([Forms]![Searchform]![txtEndDate],
    [Forms]![Searchform]![txtStartDate])+1);

Note how the date range is defined as on or after the start date and before the day following the end date.  This allows for dates with a non-zero time of day to be returned.  If a BETWEEN....AND operation were to be used any such rows on the final day of the range would not be returned.

You cannot absolutely guarantee that all dates will have a zero time of day unless you have made specific provision in the table definition to disallow such values, so the above method of defining a date range is more bulletproof.

ken sheridan say: Firstly, I assume when you say 'I type 6/4/2011 - 6/5/2012'  you do not in fact mean you are entering it as a single expression, but are independently entering the two date values in the two controls txtStartDOL and txtEndDOL.

With all those unnecessary parentheses which Access throws it it's near impossible to follow the logic of such a Byzantine WHERE clause, but start with just the restriction on the date range, and see if this gives the expected results.  If so build up the WHERE clause from there.

WHERE [DateofIncident] >= Forms![Search Form w/ State]!txtStartDOL AND [DateofIncident] < NZ(Forms![Search Form w/ State]!txtEndDOL, Forms![Search Form w/ State]!txtStartDOL)+1

hansv mvp say:

It is a Windows setting.

Open Windows Explorer.

Click Organize > Folder and Search Options... (in Windows XP: Tools > Folder Options...)

Activate the View tab.

If the check box "Hide extensions for known file types" is ticked, Windows will not display extensions such as .txt, .doc(x), .xls(x), .jpg etc.

Refreshing a OneDrive shared with Anyone - Microsoft Community

pip cartwright say:

Refreshing a OneDrive shared with Anyone

I have a Word file on OneDrive which I have shared with Anyone and have put the link on a website. However the link has a lifetime max of 3 months. I would like to retain the unlimited access for the world without updating the URL of the link. Is this possible, please?
alex wang msft say:

Hi Pip,

If you have already uploaded the link and it has an expiration date, the link will be invalid after that. You will need a new link to replace it. 

If you haven't uploaded the link, it is possible if you change the link you get when you share the file.

The link you get should be like this:

https://<company name>-my.sharepoint.com/:x:/g/personal/<Your account>/Eb56gXXXAxlDteXXXXXtCsgBQ_-Sn7c4QX3E30h5NLoYvg?e=m7fxYW.

Remove the part start from ?e= and all letters behind it.  Then share the link. This way you do not need to update the link or replace it. But the link will still be expired after the expiration date. You just need to share it again and set the new expiration date or don't use the expiration date at all. It will work until the next expiration date. You could repeat sharing the file when it about to expired and don't need to update the URL.

The expiration date is showed in OneDrive Online, Select the file and click the information button(an " i " in a circle) in the upper right of the page. There is a link button.

Click it and you will see the link and it's expiration date. If you want to disable the link, you could stop sharing too.

Alex
