Hi & welcome too,

Unless you have a back up of your important email data to access, retrieval will most likely be non existent.

Backing up your important Outlook.com email data

The possible reasons why e-mails suddenly disappear.
1. Messages were manually deleted either due to accidental deletion or were moved or deleted due to your account filter settings.

2. Messages were deleted due to account compromise issues where a hacker might have gained access to your account and deleted the emails.

3. You are using an e-mail client or a third party spam filtering software to access your account.
This often happens when using the Pop3 protocol to retrieve your mail and the option “Always leave a copy of the message on the server” is not selected.

4. The account was marked as “Inactive." Accounts become inactive if you don’t sign in via web browser at least once a year.

5. If the feature Conversation Threading feature is turned on.

6. If you use Sweep to schedule a cleanup.

7. If you rename your account to @Outlook.com a prompt would have asked you – ‘Where do you want to save your emails?’ You will have the option to create a new folder for your old account or save the emails in the Inbox of your renamed account.

graham mayor say:
The following uses late binding so you don't need the object library reference to Word. The macro finds the words listed in Cell A15 of each sheet in the named document and shows a message box to indicate what has been found.


Option Explicit

Sub MyCount()
Dim cnt As Long
Dim xlsheet As Worksheet
Dim appWd As Object
Dim wdDoc As Object
Dim oRng As Object
Dim bStarted As Boolean
Dim strFindText As String
    On Error Resume Next
    Set appWd = GetObject(, "Word.Application")
    If Err Then
        Set appWd = CreateObject("Word.Application")
        bStarted = True
    End If
    appWd.Visible = True
    Set wdDoc = appWd.Documents.Open(FileName:="P:\xxx.docx")
    For cnt = 1 To ActiveWorkbook.Sheets.count
        Set xlsheet = ActiveWorkbook.Sheets(cnt)
        strFindText = xlsheet.Range("A15")
        Set oRng = wdDoc.Range
        With oRng.Find
            .MatchWholeWord = True
            Do While .Execute(strFindText)
                oRng.Select
                MsgBox strFindText & " found!"
                oRng.collapse 0
            Loop
        End With
    Next cnt
    wdDoc.Save
    wdDoc.Close 0
    If bStarted = True Then
        appWd.Quit
    End If
    Set oRng = Nothing
    Set xlsheet = Nothing
    Set wdDoc = Nothing
    Set appWd = Nothing
End Sub

The following works

Option Explicit

Sub myfind()

Dim lastsheet As Worksheet
Dim appWd As Object
Dim wdDoc As Object
Dim wordf As Object
Dim bStarted As Boolean
Dim strFindText As String
Dim strReplaceText As String
Dim fillcolumnrange As Range
Dim i As Long
    On Error Resume Next
    Set appWd = GetObject(, "Word.Application")
    If Err Then
        Set appWd = CreateObject("Word.Application")
        bStarted = True
    End If
    appWd.Visible = True
    Set wdDoc = appWd.Documents.Open(FileName:="P:\Construction Projects\Subcontract - Template.docx")
    Set lastsheet = ActiveWorkbook.Sheets(ActiveWorkbook.Sheets.Count)
    Set fillcolumnrange = lastsheet.Range("A1:A12")
    For i = 1 To fillcolumnrange.Rows.Count
        strReplaceText = lastsheet.Range("B" & i)
        If Not fillcolumnrange.Row = "" Then
            strFindText = lastsheet.Range("A" & i)
            Set wordf = wdDoc.Range
            With wordf.Find
                .MatchWholeWord = True
                Do While .Execute(strFindText)
                    wordf.Text = strReplaceText
                    wordf.collapse 0
                Loop
            End With
        End If
    Next i

    'word document saving and closing after changes
    wdDoc.SaveAs2 FileName:="P:\path\docname.docx", FileFormat:=16
    'close word document
    wdDoc.Close 0
    If bStarted = True Then
        appWd.Quit
    End If
    Set wordf = Nothing
    Set lastsheet = Nothing
    Set wdDoc = Nothing
    Set appWd = Nothing
End Sub

bob jones aka: cybertaz say:

Just for clarification, please note that there is a significant difference between a hard return (indicated by the ¶ symbol) & a Page Break. Deleting the ¶s doesn't necessarily remove a Page Break. Since the OP of this thread didn't return to state otherwise we have no idea whether the suggestion worked for her. Typically, though, posters return for more help if what's suggested doesn't work. If it does work they often go their merry way without returning to say so. At any rate, just because it "does not work" for you means only that your cause is not the same & should have been posted separately.

Off the top of my head there are at least 5 reasons why an unwanted final page would appear in a document. How to remove it is determined by what it is that's causing it. Inappropriate use or the return/Enter key resulting is extra ¶ symbols is but one of those possibilities & is the simplest to recognize & eliminate. The other possibilities are;

  1. The use of a manual [forced] Page Break
  2. The use of a Section Break (Next Page)
  3. The use of Page Break Before paragraph formatting (Line & Page Breaks)
  4. A Table at the end of the document which comes too close to the bottom margin... Tables in Word always are followed by an empty ¶ which cannot be deleted.

Item #4 should be apparent if that's the issue. With the nonprinting characters displayed you should be able to determine which of the others may be at fault. Sometimes, though, #1 & #2 can be difficult to spot, especially if inserted at the end of a full line of text. Once you identify the cause, let us know if you have any question on how to remove it.
