A couple of thoughts:

1) If you have two tables that you need to populate, but have hit a limitation of the Input Form utility, why not create a third table that can contain all the information you need to gather from the user, then have the two "nice looking" tables reference cells in the "ugly" master table?

2) On a VBA UserForm, if you have a userform with a control that can contain a value, you use the .Value or .Text properties (read help file and experiment with each with various controls, e.g. textbox, combobox, etc.).  So if you have a textbox called TextBox1, you could say something like

ThisWorkbook.WorkSheets("Sheet1").Range("A1").Value = Me.TextBox1.Value

You would normally put a line like this in the Click event of a Button (like an OK button) that closes the form.

Be sure to make use of InteliSense (the drop-downs that appear when you type in VBA).

Since I'm not good at VB, I would use Word for that. I'll be taking the easier steps below. (Work on a copy of the Excel file, just in case).

Assuming that all the English+Arabic texts are in column A,
- Select and copy column A.
- To get rid of the table format, open Notepad and paste the column to it. (Warning: With Notepad you're going to lose all text formatting. If you want to keep it, paste to Word instead and use the "Convert table to text" feature and skip the next step).
- Copy the text from Notepad and paste it to Word.
- In Word, open Find and Replace with CTRL+H. Click More>> and check Use wildcards.
- In Find what, paste the following:
([a-zA-Z]) @([ء-ي])
In Replace with, paste the following:
\1^t\2
(These codes suppose that there is one or more spaces between the English text and the Arabic one)
- Click Replace all.
- Now copy the whole text in Word and return to Excel. If column B already contains texts that you don't want to lose, insert a new column before it.
