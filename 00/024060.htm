Hi Charlie,

Importing contact into Outlook for Mac can be done in two steps. Here is a guide on how you can import your contacts into Outlook for Mac:

Step 1: Export Contacts

  1. From your email account, choose Contacts.
  2. Select More and click on Export.
  3. Choose the contact group you want to export.
  4. Choose the export format Outlook CSV (for importing into Outlook or another application).
  5. Choose Export.
  6. When prompted, choose Save As, and browse to a location to save the file.

Step 2: Import Contacts

  1. Import the CSV file in Mac contact. (system contact)
  2. Select the contacts and drag them to desktop. (VCF file).
  3. Right click the VCF file, then choose “open it with Outlook”.
  4. The contacts should have been imported in Outlook for Mac. Click contact to check it.

daniel g. grau say:

This has been covered in two previous posts: here and here.

In addition to the proposed solution of a patch over the cell, here is another tedious alternative. In a few words, we'll create a single big cell, paste the original cells there and set the big cell's margins to zero.

This is the document you provided as a sample, where the text in the second gray cell (Job History) disappears as soon as you apply a gray shading to it:

1) First, click on the table, go to Table >Table Properties and in the Text Wrapping section select "None." This will facilitate copying and pasting that we'll do next. You can always restore that setting once the table is formatted. After doing that, you may need to reposition the table.

2) Since the problem surfaces when there are merged cells, we need to eliminate that factor. Select the cells and copy them to the clipboard. Then, merge all cells to the right of "Job History" and delete the extra text to leave it blank.


3) This creates one big cell. Select the blank cell, then choose Edit > Nest Table to paste the clipboard cells there. Note there appear gaps because of the internal cell margins. However, "Job History" becomes visible.


4) Select the nested table and take out all lines in the periphery to prevent outside border lines from combining and becoming thicker. While the cells are still selected, choose Table > Autofit and Distribute > AutoFit to Window.

5) Place the cursor inside the big table by clicking on its bottom right (you can barely see the circled cursor above) or by pressing Down Arrow. Go to Table > Table Properties > Cell > Options. Take out the checkmark in "Same as the whole table" and set all margins to zero. This is the result:

ron sommer mvp 2009-2017 say:

An alias created for a Microsoft account has always been just the address.  The profile name is used for the alias addresses.

 

An alias is an additional email address that uses the same inbox, contact list, and account settings as the primary alias.  Account settings would include profile name and signature.

 

The purpose of having an alias address is if the alias address starts to receive spam, you can delete the alias and still have your account.

 

If the main address (primary alias) starts receiving spam, you cannot delete the address without deleting the account.

andreas killer say:

My knowledge of Basic is fairly good so I can build a list, prompt, create loops etc. 

Okay, then it should be enough to show you a "basic code" that works in a simple scenario:

Make a new Word document, create a table and place the cursor into any cell.

Make a new Excel file, write the numbers from 1 to 5 into column A, select a cell with a number.

Copy the code below into a regular module and run it.

Study the comments and debug the code line by line, then you should be able to understand how it works.

Any questions?

Andreas.

Option Explicit

Public DocName As String

#Const Direction = 0

Sub Test()
  Dim xlRange As Excel.Range
  Dim xlCell As Excel.Range

  Dim wdApp As Object 'Word.Application
  Dim wdDoc As Word.Document
  Dim wdCell As Object 'Word.Cell
  Dim wdTable As Object 'Word.Table
  Dim wdColumn As Object 'Word.Column
  Dim wdRow As Object 'Word.Row

  'Install an error handler (there are many points where we can fail)
  On Error GoTo ErrorHandler

  'Get all data in Excel at the current location
  Set xlRange = ActiveCell.CurrentRegion

  'Identify Word process
  Set wdApp = GetObject(, "Word.Application")
  ' Build list of word docs
  For Each wdDoc In wdApp.Documents
    Debug.Print wdDoc
  Next

  'Use a UserForm for this:
  '  !* Through all word processes
  '  add doc name to list
  '  Display List
  '  Prompt for required doc

  'I assume that your form returns the document name
  'So this line is just to make this "pseudo code" work:
  DocName = wdApp.Documents(1).Name

  '!Create handle to required doc
  Set wdDoc = wdApp.Documents(DocName)
  If wdApp.Selection.Tables.Count = 0 Then
    MsgBox "Cursor is not in a table"
    Exit Sub
  End If
  'This is the 1st selected cell
  wdDoc.Activate
  Set wdCell = wdApp.Selection.Cells(1)

  'This is the whole table, current row and column
  'That might be helpful to get the next cell
  Set wdTable = wdApp.Selection.Tables(1)
  Set wdRow = wdCell.Row
  Set wdColumn = wdCell.Column

  'The rest of your description is vage, and there is no need to "move" to a cell.

  For Each xlCell In xlRange
    If wdCell Is Nothing Then
      MsgBox "To much data"
      Exit For
    End If

    'Transfer cell data
    wdCell.Range.InsertAfter xlCell.Value

    'Where is the "next cell"? Left? Right? Up? Down? Here are some choices:

#If Direction = 0 Then
    'From left to right, then down one row
    Set wdCell = wdCell.Next
#End If
#If Direction = 1 Then
    'Only down
    If wdCell.RowIndex = wdTable.Rows.Count Then
      Set wdCell = Nothing
    Else
      Set wdCell = wdColumn.Cells(wdCell.RowIndex + 1)
    End If
#End If
#If Direction = 2 Then
    'Only right
    If wdCell.ColumnIndex = wdTable.Columns.Count Then
      Set wdCell = Nothing
    Else
      Set wdCell = wdRow.Cells(wdCell.ColumnIndex + 1)
    End If
#End If
  Next

ErrorHandler:
  Select Case Err.Number
    Case 0
      'Success
    Case Else
      If Err.Source = "" Then Err.Source = Application.Name
      Debug.Print "Source     : " & Err.Source
      Debug.Print "Error      : " & Err.Number
      Debug.Print "Description: " & Err.Description
      If MsgBox("Error " & Err.Number & ": " & vbNewLine & vbNewLine & _
          Err.Description & vbNewLine & vbNewLine & _
          "Enter debug mode?", vbOKCancel + vbDefaultButton2, Err.Source) = vbOK Then
        Stop 'Press F8 twice
        Resume
      End If
  End Select
End Sub
