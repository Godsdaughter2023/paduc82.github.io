OK - I think I have the picture. The following Outlook script should do the job. You will need to insert the path and name of your local data file where indicated and ensure that the sheet names match. Here the local file is "Sheet1" and the incoming sheet is "DATA" to match your sample. Also to mach your sample I have only imported to column S. If there are likely to be more columns, change the two references from S to the new column limit.

Create an Outlook rule that will identify the incoming messages containing the attachments and run the script from the rule. Each message is then processed to add the data to the local file as it arrives.

Sub CopyAttachmentToExcel(olitem As Outlook.MailItem)
Dim xlApp As Object
Dim xlWB As Object
Dim xlTempWB As Object
Dim xlSheet As Object
Dim xlTempSheet As Object
Dim lngTempLast As Integer
Dim lngLast As Integer
Dim strFname As String
Dim strTempPath As String
Dim bXLStarted As Boolean

Const strPath As String = "D:\My Documents\Collate.xlsx"        'the path and name of the local workbook
strTempPath = Left(strPath, InStrRev(strPath, "\")) 'The path of the temporary file

On Error Resume Next
Set xlApp = GetObject(, "Excel.Application")
If Err <> 0 Then
    Set xlApp = CreateObject("Excel.Application")
    bXLStarted = True
End If
xlApp.Visible = True

On Error GoTo 0
'Open the workbook to input the data
Set xlWB = xlApp.Workbooks.Open(strPath)
Set xlSheet = xlWB.Sheets("Sheet1") 'The sheet in the local workbook

'Process the message attachment
With olitem.Attachments.item(1)
    If Right(.DisplayName, 4) = "xlsx" Then
        lngLast = xlSheet.Range("A" & xlSheet.Rows.Count).End(-4162).Row
        strFname = strTempPath & .DisplayName
        .SaveAsFile strFname
        Set xlTempWB = xlApp.Workbooks.Open(strFname, editable:=True)
        Set xlTempSheet = xlTempWB.Sheets("DATA")
        lngTempLast = xlTempSheet.Range("B" & xlTempSheet.Rows.Count).End(-4162).Row
        xlSheet.Range("A" & lngLast + 1, "S" & lngLast + lngTempLast - 1).Value = xlTempSheet.Range("A2", "S" & lngTempLast).Value
        xlWB.Save
    End If

End With
xlWB.Close SaveChanges:=True
xlTempWB.Close SaveChanges:=False
If bXLStarted Then
    xlApp.Quit
End If
Set xlApp = Nothing
Set xlWB = Nothing
Set xlSheet = Nothing
Set xlTempWB = Nothing
Set xlTempSheet = Nothing
Set olitem = Nothing
End Sub

langware say:

I use a free browser add-on called Ad Block Plus.

With Ad Block Plus, it is possible to hide the "Upgrade to Premium" button. If you are familiar with Ad Block Plus, here's how to hide the "Upgrade to Premium" button:

NOTE:  You will also need to install the "Element Hiding Helper for Adblock Plus" in order to select an element (i.e., the "Upgrade to Premium" button) to be blocked.

1. With your browser open to your outlook.com inbox page (and Ad Block Plus already installed), left click  the Ad Block Plus icon ... or click Tools in the browser's menu and click Ad Block Plus in the dropdown menu. An Ad Block Plus menu will open.

2. In the Ad Block Plus menu, click "Select an element to hide"

3. Hover your mouse over the "Upgrade to Premium" button. A red box will be drawn around the button. If the red box is not around the button, adjust your mouse until the red box is over the element you wish to hide. Then press the left mouse button.

4. A "Compose element hiding rule" window will open. Click the "Add element hiding rule" button at the bottom of that window.

5. The window will close and your inbox should be displayed .... with the "Upgrade to Premium" button gone (you might have to refresh the page or sign out of outlook.com and sign back in).

6. If you make a mistake or want to undo the Ad Block Plus rule you just made, open the Ad Block Plus menu (as described in step 1) and click "Filter preferences".  When the Ad Block Plus filter preferences" window opens, click the "Custom filters" tab. Then look at list of filters on right side (under the heading "filter rule"). The rule to block the "Upgrade to Premium" button will look something like this: outlook.live.com##._n_15     To premanently remove the rule: Right click that rule and click "delete" in the dropdown menu.

In addition to Adblock Plus, a free browser add-on called uBlock Origin can also be used to block a specific element on a web page (ie block the "Upgrade to Premium" button).

EDIT (July 10, 2017) to clarify how to remove/disable the rule that blocks the "Upgrade to Premium" button.

EDIT (Aug 6, 2017). Microsoft must have changed the HTML for the inbox page. Today, I noticed that my folder list (left side of inbox) was missing, and the "Upgrade to Premium" button was visible.

outlook.live.com##._n_v3.o365button.o365buttonOutlined.ms-font-m.ms-fwt-sb.ms-fcl-w.ms-bgc-tp.ms-bcl-tp.ms-bgc-td-f.ms-bcl-tdr-f

My folder list is now visible, and the pesky "Upgrade to Premium" button is again made invisible.

jeeped say:

... I'm trying to maintain a document that has multiple sections and each section has to be saved separately ...


I believe you are on the right track with the { INCLUDETEXT } field for the purposes you stated.
You are not going to see the { INCLUDETEXT } field and the document text at the same time. You will either see the field result or the field code. The keyboard shortcut to switch between the two modes of display is Alt+F9.
If you make changes to one of the independent section documents, you simply need to update fields in your master document. A quick method is to tap Ctrl+A (e.g. Select All), then tap F9 (e.g. Update Fields).
The field braces (e.g. { and }) are special characters and cannot be simply typed in. If you want to manually add a field, tap Ctrl+F9 (e.g. Insert Field) to open a set of field braces and type the field code and any parameters within them. A field can be copied to a new location and modified. Remember to Ctrl+A then F9 when you are done.
If you use Insert Object or Insert File, the content will be static and not updated when the source has been updated.
The LINK command is typically used to bring a section of an Excel worksheet into your document with live updating. I do not think its use is appropriate here.
