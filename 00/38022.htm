tim kotoff say:

I too have the Rosewill RHTS-8206, and had a problem with it after upgrading to Windows 10.  However, mine was still recognized in Device Manager as a USB Sound Device.  It sounds to me like your USB controller is having trouble reading the device, which is most likely a driver issue.  I would suggest you follow Ruth's suggestion under "Method 2", except I would download the new driver first.  (And I'm not sure why she keeps mentioning a Logitech headset.)

Once you resolve that issue, though, you will likely run into the same issue I had, where Windows went back to using a generic driver for the headphones and I lost the use of its custom control applet.  I wouldn't mind so much, except for how it handles the volume; roughly 0-10% is the entire range of volume, and 11-100% is effectively full blast.

There is a newer driver on the website as of 5/13/2015, however it's for Windows 8/8.1.  Many drivers for Windows 8 will still work with 10, but the setup program does an OS check and fails on 10.  So what I did was:

  1. Download and extract the newer driver
  2. Open Device Manager
  3. Right-click the device, choose "Update Driver Software..."
  4. Select "Browse my computer for driver software"
  5. Then browse to the extracted folder

Windows 10 accepted the driver.  I rebooted, my control applet was back running in the Notification Area, and volume returned to normal.
