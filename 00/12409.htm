e.j. gun say:

What Doug is saying is that you do not need to "Trust access to the VBA Project Object Model" in order to develop macros in Office.  The only time it is important to have that box checked is when you are writing code that can either a) modify itself (i.e. access its own code module and change the code while running) or b) create additional code/classes/forms on the fly and add them to the VBA Project.

If you are simply writing VBA code that does "normal" stuff, you don't need that option.  And BTW, that option is very dangerous and is one way the malicious code can be introduced into your computer, so it's best to avoid it if possible.
