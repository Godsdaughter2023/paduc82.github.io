ken sheridan say: In the Format event procedures of both the Consignee group header (and footer if it has one) and the detail section put:

    Cancel = (TotRuns < 10)

ken sheridan say: You would presumably only want to cancel the groups higher in the hierarchy if the TotRuns value is <10 in all instances of the Consignee group within the higher level group.  For that I think you'd have to examine the data in the underlying tables directly.  In each group header's format event you could open a recordset which returns the TotRuns values per consignee for the current shipper or city, and set the return value of the Cancel argument to true if none of the rows in the recordset has a value of 10 or more, i.e. exit the procedure once a row with aTotRuns value of 10 or more is encountered, but if the end of the recordset is reached without encountereing any rows with a TotRuns value of 10 or more the set the return value of the cancel argument to true.

As your query includes parameters you'd need to restrict the recordset on the values of the parameters by concatenating them into the SQL statement.  For this the parameters would really have to be references to controls in a dialogue form rather than the simple system generated parameters you are using at present.

michael brando say:

Got answer from Microsoft, the empty "View sync problems" is a known issue on the Mac but the following advice allowed me to find the problem and correct it.

Solution:

  1. Go to System Preferences > Extensions.
  2. Then click on "Finder" on the left half of the window
    You should see a "One Drive Finder Integration" option. Make sure it's checked.
  3. If is checked already, uncheck and recheck again.
  4. You should now see green check marks next to files that have been synced with One Drive.
