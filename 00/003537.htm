dirk goldgar say: the SQL for that would look like this:

    UPDATE LocalFields SET LocalFields.Number1 = (SELECT Max(ID) FROM ClientInfo)

However, Access won't let you do that, because it will think you are running a totals query when you aren't.  So instead you can write this:

    UPDATE LocalFields SET LocalFields.Number1 = DMax("ID", "ClientInfo")

It isn't clear to me, though, why you would store that information in another table, when you can look it up with DMax() any time you want it.
