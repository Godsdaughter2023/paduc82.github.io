karhubear say:

E72 calendar sync

knight_templar say:

Reboot your phone
- Remove the battery while your phone is turned on, then try the syncing again.
Verify that your client version is 2.07.22 or newer
- Recommended version for Tube (5800 Express Music) users is 2.0.73, which includes only some Tube specific settings.
- From your phone's Application manager select Mail for Exchange->Options->View details
- If older, then download the newest client for Mail for Exchange.
Test syncing without calendar (Menu -> Mfe -> Mail for Exchange -> Options -> Edit Profile -> Sync content -> Synchronize calendar -> No)
- If it works, (re)enable your calendar sync and try to do a 'full re sync' and see if the calendar starts syncing after that (Menu -> MfE -> Mail for Exchange -> Options -> Full resync).
- If the problem persists/returns even after a Full resync, please check the steps below.
Check for conflicting items
- Open Outlook calendar and change the view to 'By Category' (View->Arrange by->Current View->By Category).
- Remove from Outlook all conflicting items.
- Also change the Outlook view to 'Folder List' :
- Sync Issues folder is visible now, check from it for possible conflicts (Sync Issues -> Conflicts folder) and if any, remove them all.
- After the check/remove, do a full resync and test if the sync goes through.


Cleaning the Outlook Calendar
- Change calendar to category view (View->Arrange by->Current View->By Category):
- Delete or archive to a pst file following calendar items;
A) All items that are old and are not needed anymore

B) Events that are larger than 1MB (View -> Arrange by -> Current view -> Customize current view -> Fields -> All Appointment fields -> add Size field)

C) Events that are sent to hundreds of recipients

D) Items without end date (add the missing end date or remove the item)

- After the cleanup, do a fullresync and test if the sync goes through.
Still having problems?

- Try installing the latest Mail for Exchange client.  Version 2.9 fixes a major calendar sync issue.  The issue exists if there is a "sync died (3)" error message in the Admin_log(x).txt.  This file can be found under the C:\Data\MailforExchange folder.
- Before starting the installation:

No need to uninstall the old client, install the Mail for Exchange client right over the old version.
In case of upgrade installation, it's important to run a full resync after completing the installation (Menu->MfE->Mail for Exchange->Options->Full resync).  Otherwise the client might not work properly.
If you however decide to uninstall it or install the client to any other "clean" device, note that calendar/contacts/tasks synchronization is disabled by default.

graham mayor say:

You can set the value of the first SEQ field to the value of a docvariable (or docproperty) with a nested field e.g.

{ SEQ Counter \r { docvariable varCounter } }
{SEQ \r {DocProperty Counter}}

You may be interested in http://www.gmayor.com/BookmarkandVariableEditor.htm which provides a simple means of manipulating variables, doc properties etc.
