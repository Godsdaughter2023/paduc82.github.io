doug robbins - mvp office apps & services (word) say:

Use a macro containing the following code:

Sub MultiReplace()

Dim StrFind As String, StrRepl As String

Dim i As Long

StrFind = "∂,∆,π,lee "

StrRepl = "de,trang,P,loo"

Set RngTxt = Selection.Range

For i = 0 To UBound(Split(StrFind, ","))

Selection.HomeKey wdStory

Selection.Find.ClearFormatting

With Selection.Find

.ClearFormatting

.Replacement.ClearFormatting

.Text = Split(StrFind, ",")(i)

.Replacement.Text = Split(StrRepl, ",")(i)

.Format = False

.MatchWholeWord = True

.MatchAllWordForms = False

.MatchWildcards = False

.Execute Replace:=wdReplaceAll

End With

Next i

End Sub

macropod say:

That's because the macro uses:

.MatchWholeWord = True

The reason it would be coded that way is so that, when you're looking for words it doesn't replace strings that are only parts of words. To that end, you should either use two different macros, one with:

.MatchWholeWord = True

for words and one with:

.MatchWholeWord = False

for letters, or combine the two in the one macro. For example:

Sub MultiReplace()
Dim StrFind As String, StrRepl As String, i As Long
StrFind = "the,quick,brown,fox"
StrRepl = "over,the,lazy,dog"
Set RngTxt = Selection.Range
Selection.HomeKey wdStory
Selection.Find.ClearFormatting
With ActiveDocument.Content.Find
  .ClearFormatting
  .Replacement.ClearFormatting
  .Format = False
  .MatchWholeWord = True
  .MatchAllWordForms = False
  .MatchWildcards = False
  .Wrap = wdFindContinue
  .Forward = True
  For i = 0 To UBound(Split(StrFind, ","))
    .Text = Split(StrFind, ",")(i)
    .Replacement.Text = Split(StrRepl, ",")(i)
    .Execute Replace:=wdReplaceAll
  Next i
  .MatchWholeWord = False
  StrFind = "a,b,c,d  "
  StrRepl = "e,f,g,h"
  For i = 0 To UBound(Split(StrFind, ","))
    .Text = Split(StrFind, ",")(i)
    .Replacement.Text = Split(StrRepl, ",")(i)
    .Execute Replace:=wdReplaceAll
  Next i
End With
End Sub

millerto say:

MS Word 2007 inserting blocks of text best method

I have a word document that I want to send to others and I would like them to be able to click a button and have a block of text inserted.  What is the best method to acheive this?  I thought one of these three might do the trick: macros, autotext or building blocks?

any help will be greatly appreciated.

gerry knight say:

As Graham mentions, you can not store AutoText/BuildingBlocks in a document.  However, you can store plain text blocks in variables, and use those.  Suppose you have:

The quick brown fox jumps over the lazy dog. The quick brown fox jumps over the lazy dog. The quick brown fox jumps over the lazy dog. The quick brown fox jumps over the lazy dog. The quick brown fox jumps over the lazy dog. The quick brown fox jumps over the lazy dog. The quick brown fox jumps over the lazy dog. The quick brown fox jumps over the lazy dog. The quick brown fox jumps over the lazy dog.

You could select that and put it into a variable ("Fox").  Variables can take values from anywhere.

Sub UsingVar()
ActiveDocument.Variables("Fox").Value = Selection.Text
End Sub

Now you can put it anywhere,using the Value of the variable.

Sub MacroName()
Selection.TypeText ActiveDocument.Variables("Fox").Value
End Sub

That being said, you need to work out fully the strategy you want to use.  How are your users going to access your "blocks"?  Do you need formatting options?  Text put of a variable is "plain text", so it will take on the formatting of the paragraph style.

Another possible option is to use an INCLUDETEXT field.

Sub MacroName()
   Selection.Fields.Add Range:=Selection.Range, Type:=wdFieldEmpty, Text:= _
      "INCLUDETEXT  ""c:\\Yadda\\DescriptionSource.doc"" D1", _
      PreserveFormatting:=True
End Sub

This would add, as an INCLUDETEXT field, the contents of the bookmark D1, from the file DescriptionSource.  Yes, the file with the contents MUST be available (and thus is not much different from sending a template file along with your document).  But it is an option.  You could send it with the document, with instructions to put them both in the same folder.

Gerry Word MVP

You cannot store autotext/building blocks in a document. If you are sending this as a document you will have to use a macro, though you cannot force users to run it.

will type the text at the cursor.

Hi kari,

It’s not possible to download responses from multiple forms into a single workbook. However, here’s a workaround to use data connection feature to refresh data connecting with multiple sheets. Please follow the steps below:

  1. Download responses to multiple workbooks.
  2. Create a new sheet in a new created workbook that you want to collect all the data information.
  3. Data Tab>New query>From File>From Workbook>Select the workbook that you want to connect the data from>Import>Select the sheet that you want to import data from.
  4. When you want to refresh the data, please go to Data Tab>Refresh All, then all the data connecting with multiple sheets will be updated.
