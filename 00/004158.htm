scottgem say:

First, Comboboxes have a RowSource, not a Recordsource. The Rowsource is what dictates what is displayed in the combobox. 

It sounds like you are trying to do what is called Cascading comboboxes. This is a standard technique and it is well documented. Basically it involves setting the Rowsource of one combo to be filtered for the selection in the previous combo.

My question here is why are you using a subform. Is the mainform bound to anything? It doesn't sound like it is. 

If the main form is not bound, you don't really need a subform. You can put your filtering combos in the Header of the form, then apply the filter after the selections are made.

scottgem say:

You don't link them. you use them to filter. You can do this one of two ways. You can either reference the values selected in the combos from the query using the syntax:

=Forms!formname!controlname

I would only use this if values must be selected in all three combos. Then have a button that requeries the form. 

The other option would be to build a filter. You would add a button to the form header with code like:

Dim strCriteria as String 

If Not IsNull(Me.combo1) Then

     strCriteria = "field1 = '" & Me.combo1 &"' "

Else

     strCriteria = ""

End if

 

If Not IsNull(Me.combo2) Then

     If Len(strCriteria)>0 

         strCriteria = strCriteria & "AND field2 = '" & Me.combo2 &"' "

      Else

         strCriteria = "field2 = '" & Me.combo2 & "' "

      end if

Else

     strCriteria = strCriteria

End if

If Not IsNull(Me.combo3) Then

     If Len(strCriteria)>0 

         strCriteria = strCriteria & "AND field3 = '" & Me.combo3 &"' "

      Else

         strCriteria = "field3 = '" & Me.combo3 & "' "

      end if

Else

     strCriteria = strCriteria

End if

Me.Filter = strCriteria

DoCmd.RunCommand acCmdApplyFilterSort

So that code build a filter statement from the 3 combos then applies the filter to the Recordsource.
