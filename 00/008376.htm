pesach shelnitz say:

Hi Faraz,

The following macro is a skeleton macro for opening an Excel file from Word and copying data from Excel cells into a Word document. You can add code to interactively obtain the range of cells that you want to copy from Excel, create a table in Word, insert the data from Excel, and sort the data in the Word table. Write back if this macro is a suitable start and you need help to implement the additional actions that I described.

Sub GetExcelFromWord()
    Const Error_FileNotFound = 1004
    Const Error_NotRunning = 429
    Const Error_NotInCollection = 9
    Dim fileName As String
    Dim wkbName As String
    Dim fso As Object
    Dim xlApp As Object
    Dim wkBook As Object
    Dim newInstance As Boolean
    Dim fileOpened As Boolean
    Dim x As Long
    Dim i As Long
    Dim j As Long
    Dim k As Integer

    With Application.FileDialog(msoFileDialogFilePicker)
        If .Show Then
            fileName = .SelectedItems(1)
        Else
            MsgBox "You didn't select an Excel file to open."
            GoTo ErrorHandler
        End If
    End With
  
  ' Split fileName into path and workbook file name.
    Set fso = CreateObject("Scripting.FileSystemObject")
    wkbName = fso.GetFile(fileName).name
    If wkbName = "" Then
        MsgBox "A suitable file was not selected."
        GoTo ErrorHandler
    End If
   
    On Error Resume Next
    Set xlApp = GetObject(, "Excel.Application")
    If Err.Number = Error_NotRunning Then
        Set xlApp = CreateObject("Excel.Application")
        newInstance = True
    Else
        newInstance = False
    End If
    On Error GoTo 0
    xlApp.Visible = True
    
    On Error Resume Next
    Set wkBook = xlApp.Workbooks(wkbName)
    If Err.Number = Error_NotInCollection Then
        Err.Clear
        Set wkBook = xlApp.Workbooks.Open(fileName)
        If Err.Number = Error_FileNotFound Then
            MsgBox "The file specified could not be opened.", _
                 vbCritical Or vbOKOnly, "File Not Opened"
            GoTo ErrorHandler
        Else
            fileOpened = True
        End If
    End If
    wkBook.Activate
    On Error GoTo 0
     i = 1
     j = 1
    For x = 1 To wkBook.WorkSheets.Count
        With wkBook.WorkSheets(x)
            For i = 1 To .UsedRange.Rows.Count
                For j = 1 To .UsedRange.Columns.Count
                    With .Cells(i, j)
                        XlData = .Value
                        ' Add code to copy the contents
                        ' to somewhere in Word, for example
                        Selection.InsertAfter i & ", " & j & ":" & XlData & vbCrLf
                    End With
                Next
            Next
        End With
    Next
ErrorHandler:
    If fileOpened = True Then
        wkBook.Close
    End If
    If newInstance = True Then
        xlApp.Quit
    End If
    Set fso = Nothing
    Set wkBook = Nothing
    Set xlApp = Nothing
End Sub

a. user say:

You will probably actually need a macro that copies cell-by-cell, but it is just possible that in simple cases, something like the following will do what you need:

 

Sub InsertFromExcel()

Dim strFile As String

Dim strSQL As String

' Set these as appropriate

strFile = "c:\mywb\mywb.xlsx"

' If the sheet is called mysheet and the range is A1:D20, use

strSQL = "SELECT * FROM [mysheet$A1:D20] ORDER BY 1,2,3,4"

 

' If there are more columns, you will need a longer ORDER BY,

' e.g. if you have A1:E20, use

 

strSQL = "SELECT * FROM [mysheet$A1:E20] ORDER BY 1,2,3,4,5"

' However, Jet will impose a maximum number of columns

Selection.Range.InsertDatabase _

 

 

  LinkToSource:=False, _

  SQLStatement:=strSQL, _

  DataSource:=strFile, _

  From:=-1, _

  To:=-1, _

  IncludeFields:=True

End Sub


The main problems with this are:
 a. there may be problems if InsertDatabase wants to use the first row in the specified range as column names, because it may not recognise the values as valid column names
 b. the OLE DB provider that it uses imposes data types on each column, and that will cause problems if you have mixed data types in a column

Try this ARRAY formula. See below on how to enter it. the lookup value is in C1

=MIN(IF(A1:A10=C1,IF(B1:B10<>"",B1:B10)))

dirk goldgar say:

I've looked at your database, and the problem occurs because the SalesPersonCommissionForm is set as Modal;  that is, its Modal property is set to Yes.  If you change that to No, then the report has the focus after the form is closed.

Note that, if you make that form non-modal, it won't prevent the user from interacting with other forms and objects, the way having it modal does.  There are probably ways around this, if you think it necessary.

dirk goldgar say:

Although I have used macros in my database; however, I don't prefer to use it. I used macros only to open and close forms and reports in my database. Instead, I prefer to use VBA because it is easy to understand & more flexible than macros.

I am not expert in VBA programming; however, somehow I've learnt VBA through internet searching and you may have seen that I've used VBA on some places in my database .

Could you please convert macro into VBA code you have used in the Open event of the report?

I wish I'd known you preferred VBA;  I do, too, and I had to translate the original VBA I conceived of into a macro.

Here's the event procedure for the report's Open event, in VBA:

'------ start of code ------

    

Private Sub Report_Open(Cancel As Integer)

    On Error GoTo Report_Open_Err

    ' Open parameter form in dialog mode.
    DoCmd.OpenForm "SalesPersonCommissionForm", WindowMode:=acDialog

    ' If the form is not still open, then the user has cancelled the report.
    If Not CurrentProject.AllForms("SalesPersonCommissionForm").IsLoaded Then
        Cancel = True
    End If


Report_Open_Exit:
    Exit Sub

Report_Open_Err:
    MsgBox Err.Description, vbExclamation, "ERROR " & Err.Number
    Resume Report_Open_Exit

End Sub
'------ end of code ------
