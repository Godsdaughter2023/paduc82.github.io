Using standard time of day values when you really are using them as a duration is not all that good an idea.  Generally, durations should be stored as a total of the smallest unit of time that you are concerned with (in your case seconds).  OTOH, if your users are entering those values because that's how they think of them, you can convert them at various points between data entry and formatting them for display.

westyfield2 say:

Configuring Homepage and Search settings

Hi,

We all know about changing the home page and search settings for users via GPO.  The GPOs themselves of course are just changing the relevant registry values in Software\Policies\Microsoft\Internet Explorer\Main and Software\Policies\Microsoft\Internet Explorer\SearchScopes and if you can't use GPO for whatever reason you can just write the required REG ADD scripts yourself.

As per this various other locked thread, users will get a warning that an unknown process (their system admin!) is changing their home page.  https://answers.microsoft.com/en-us/ie/forum/ie11-windows_7/choose-your-homepage-and-search-settings/2966bcb3-3ddb-4a26-8c51-07b5aa40e4ef?page=3&auth=1 and https://social.technet.microsoft.com/Forums/ie/en-US/eacc3b33-c045-4335-815e-5603a218cabb/set-homepage-and-default-search-scope-in-an-environment-with-dynamic-local-user-profiles?forum=ieitprocurrentver&prof=required

As discovered in the linked threads, the IE warning message can be prevented by the additional registry change

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Internet Explorer\Main\FeatureControl\FEATURE_EUPP_GLOBAL_FORCE_DISABLE]
"iexplore.exe"=dword:00000001

With that additionally entry made, the user's homepage gets changed and the user doesn't get the warning notification.

Similarly if you do the SearchScopes to set the user's default search engine, users will get a warning that an unknown process (their system admin!) is changing their search engine.  Unfortunately however the FEATURE_EUPP_GLOBAL_FORCE_DISABLE entry doesn't fix that, and the user still gets a warning that an unknown process is changing their search engine.  Of course the user can click on the yellow box to accept the change, although users may be reluctant given the fact that IE calls it an unknown process rather than even whatever your calling login.bat was named.

So, does anyone know how to write the SearchScopes in to the registry and get IE to accept them without warning the user?  I was hoping to find an equivalent for SearchScopes for what FEATURE_EUPP_GLOBAL_FORCE_DISABLE is to the Start Page.

bons1 say:

I FINALLY got the answer that works!  I searched again and found that a genius posted the following reply back in 2010----don't know why i didn't find it when i searched yesterday....but i never give up!!  :)  So anyway.....i followed the directions below and i now have my recent documents showing on my task bar just like they used to! I thought I'd post it here in case someone else needs it.

Check the recent document list within word application. If its fine then there might some corruption in the recent items stored in ‘AutomaticDestinations’ folder.

 

To resolve and fix the issue, follow these steps:

 

1)   Browse to the location mentioned : %AppData%\Microsoft\Windows\Recent\AutomaticDestinations

 

Note: the folder ‘AutomaticDestinations’ is hidden

 

2)   Delete all the files whose size is 1MB or more.

 

Note: If you cannot find any file sized 1MB or more then delete all the files. This will eventually remove the Jump Lists’ recent items for all the applications. The recent items on the Jump Lists will be re-added as the program is been used. If you delete all the files then ensure that the applications are pinned again to the taskbar.

MS-DOS
También descarga el programa ISO de hdat2  descomprimes la iso en cualquier carpeta
y copias en el pendrive de arranque la carpeta  hdat2   (esta en la carpeta
descomprimida iso)

Entra en tu pendrive y abre el archivo autoexec.bat  
> abrir con > bloc de notas   >editar
y  al final  escribes :

CD hdat2
hdat2.exe /W    

Al poner el pendrive se debe poder arrancar desde el pendrive
si no se puedes arrancar, se debe entrar en BIOS y poner en la
pestaña BOOT que inicie en primer lugar con el USB
