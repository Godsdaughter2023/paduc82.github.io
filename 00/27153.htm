        ken sheridan say: Are you able to insert the data correctly via your forms?  I can't comment on the use of a navigation form I'm afraid, this not being something I would ever use myself as I'd build in the functionality myself via unbound controls with appropriate code in their relevant event procedures.

        Assuming you can enter the data into the tables via your forms the first step to building a report is to create a query which joins the relevant tables.  The tables will be joined on the appropriate primary and foreign keys, and can be done entirely in query design view by dragging from the primary key of the table on the 'one' side of each relationship to the corresponding foreign key of the table on the 'many' side.  Where you have composite keys made up of multiple columns it's just a question of dragging several times from one table to the other on the relevant column each time.

        Having joined the tables like this you can then add the columns (fields) you want in your report to the design grid in query design view by dragging each from the table to a an empty column in the design grid below.  Having done this, enter >2 in the 'criteria' row of the ConditionCode column in the design grid.  This will restrict the query's results to those rows with a code of 3 or greater.  You can open the query in datasheet view to test it, and you should find it returns multiple rows per apartment and per room within each apartment.

        Once you are satisfied that the query is returning the correct rows you can create a report based on the query.  You can do this from scratch in report design view, but you'll probably want to use the report wizard as this cuts out a lot of the drudgery, and you can always amend the resulting design to fine tune it to your requirements.  However you do it you'll want to group by apartment first, and then by room so that the same data is not repeated unnecessarily in the report, but is shown in group headers, with the details per room listed in the report's detail section.
