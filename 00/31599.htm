al jarvi say:

The native PPTP VPN server on the office Vista machine only needs TCP Port 1723 forwarded. Close the other ports. With that said it also needs GRE Protocol 47 traffic to be passed by any firewall/router the VPN server is behind. Some firewalls/routers call that PPTP Pass Through or VPN Pass Through or like the Windows Firewall pass that automatically if TCP Port 1723 is opened. This page has troubleshooting at the end.

http://theillustratednetwork.mvps.org/Vista/PPTP/PPTPVPN.html

If any firewall/router does not pass GRE Protocol 47 traffic you need to either replace the device or upgrade to newer firmware to address the issue.

Setting the VPN server in the DMZ exposes it to the public internet. Normally I would not do that because of security risks.

An alternative maybe to use the TeamViewer VPN function. Note TeamViewer is NOT free if used for business use.
