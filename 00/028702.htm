doug robbins - mvp office apps & services (word) say:

To deal with the column to the right of the column in which you have the selection, use:

Dim crange As Range

Set crange = Selection.Cells(1).Range

crange.MoveEnd wdColumn, 1

crange.Select

With Selection.Font

    .Name = "Algerian"

    .Size = 16

End With

Selection.Collapse wdCollapseStart

To deal with the column to the left, use

Dim crange As Range

Selection.MoveLeft wdCell, 1

Set crange = Selection.Cells(1).Range

crange.MoveEnd wdColumn, 1

crange.Select

With Selection.Font

    .Name = "Algerian"

    .Size = 16

End With

Selection.Collapse wdCollapseStart

Selection.MoveRight wdCell, 1

andreas killer say:
There is no need to sort nor to move the rows on top. If you create a Range object that contains all cells in column G, you can expand that to the entire rows and copy that rows to a different sheet.

Very very simple example: 
Sub Test()
  Dim All As Range
  Set All = Sheets(1).Range("G3,G5")
  All.EntireRow.Copy Sheets(2).Range("A1")
End Sub

hansv mvp say:

Let's say that you want to move the rows that don't contain "Data".

The following macro assumes that row 1 is a header row.

Sub MoveData()
    Dim wshS As Worksheet
    Dim wshT As Worksheet
    Dim m As Long
    Application.ScreenUpdating = False
    Set wshS = ActiveSheet
    Set wshT = Worksheets.Add(After:=wshS)
    m = wshS.Range("G" & wshS.Rows.Count).End(xlUp).Row
    wshS.Range("G:G").AutoFilter Field:=1, Criteria1:="<>Data"
    With wshS.Range("G2:G" & m).SpecialCells(xlCellTypeVisible).EntireRow
        .Copy Destination:=wshT.Range("A1")
        .Delete
    End With
    wshS.Range("G:G").AutoFilter
    wshS.Select
    Application.ScreenUpdating = True
End Sub
