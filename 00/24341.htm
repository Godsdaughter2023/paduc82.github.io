When you click cancel do you see the url change to https?

It basically means that the secure (https) web page that you are trying to open contains images and other elements that are hosted on a non-https location. They call it mixed content.

If you trust the website, add it to your trusted sites list.

Mozilla Forum -https://support.mozilla.org/en-US/questions/748605

You can't suppress the "Although this page is encrypted, the information you have entered is to be sent over an unencrypted connection and could easily be read by a third party" warning for security reasons.

greg wilson 2 say:

Try Rob Bovey's VBA CodeCleaner:  http://www.appspro.com/Utilities/CodeCleaner.htm

The subject of what actually happens under the hood of VBA code modules is above my pay grade. But the general concept is that not all that exists in a code module is visible. And VBA does not always completely clean up after itself during the code development process, especially after crashes. Some versions of Excel may be more sensitive to VBA "garbage" than others. This may explain the version difference.

What CodeCleaner essentially does is programmatically export all VBA code to text files, delete the code modules, recreate them, and import back the exported code from the text files. Text files are too rudimentary to support anything other than the visible text, and so all garbage gets lost in the process.

Even if this doesn't solve the problem, having CodeCleaner is a must in my opinion. So it won't be a wasted effort.

Hope it works !!!
