Hi Gereth,

I have the workaround for this. You are right, WSUS is the problem. MS has been told along time ago but wont fix it, god forbid an enterprise user wants to use Cortana....

The fix is simple: simply make your computer forget its wsus connection for awhile. In a normal wsus setup, group policy will add the settings back later, or you can just put the wsus settings back afterwards yourself.

-------- Steps ---------

--- Simple ---

Delete the "HKLM\Software\Policies\Microsoft\Windows\WindowsUpdate" key

Run the following 2 commands as an admin:

net stop wuauserv
net start wuauserv

Force a windows update check

Download language voice and handwriting packs.

--- Detailed ---

-- Delete wsus info --

Open regedit and delete (or if your feeling concerned export then delete) the HKLM\Software\Policies\Microsoft\Windows\WindowsUpdate key.

-- Restart the update service --

In a administrative command prompt run the following commands:

net stop wuauserv
net start wuauserv

-- Check for windows updates --

Settings -> update & security -> windows update -> check for updates

It might not pick up anything new, but you sometimes need to force the update check to make windows re-search for the language packs to help with the next step.

-- Install the handwriting and voice language packs --

- Install the voice and handwriting packs -

Settings -> Time & Language -> Region & Language -> (language you want) -> options

You should see the page say "searching windows update for language features". You wont see anything change until it does.

-- tidy up --

If your org has group policy settings, then the wsus settings will come back on their own, otherwise just reimport the reg key.
