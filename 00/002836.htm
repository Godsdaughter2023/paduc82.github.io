Saving to "Office XML" format reveals the characters are represented with directives similar to:

<w:r>
     <w:rPr>
          <w:noProof/>
     </w:rPr>
     <w:sym w:font="WP TypographicSymbols" w:char="0041"/>
</w:r>

This was for an opening curly quote, i.e. "A" in the WP Typographic Symbols character set.

So that does provide one potential elaborate automation, by exporting to XML and fixing up these instances with the directives needed for normal smart quotes and other characters.

Further ideas are still welcome for how to identify and replace the characters in-place, since this approach risks subjecting the documents to other conversion format issues or losses by forcing everything to be converted to and then back from XML and native .DOC or .DOCX format.

If you have the ability to use a programming language such as C#, you can use the Microsoft.Office.Interop.Word API to open documents and save them as XML files with WDSaveFormat.

Then, use the System.Xml API to open the XmlDocument and parse the document using XmlNodeList, changing w:font="WP TypographicSymbols" to "YOURFONT" and correcting the hex code for w:char="0041".

meleemeleelee say:

To add,

If you are using C# to tackle this problem, you will need to first convert the files to 2003 Word Documents (.doc), then save as .xml with the compatibility mode set to wdCurrent. This way all the .xml is updated to the latest schema and you can parse the xml all the same.

Enter below formula as an array formula (Ctrl+Shift+Enter) in cell D2 and copy down till D11 - this returns the Largest 10 values from column B:

=IFERROR(LARGE(IF(FREQUENCY(IF(B$2:B$20<>"",MATCH(B$2:B$20,B$2:B$20,0)),ROW(B$2:B$20)-ROW(B$2)+1), B$2:B$20), ROW(B1)),"")

bernie deitrick say: Move the three files to another folder, and rename them something like Personal1, 2, and 3. Then start Excel and record a macro to your Personal library and Excel will create a new Personal file. Open the other files and copy the code from each into your new file - just drag and drop the modules into your new Personal - Debug to make sure that there are no duplicate macros - then close Excel, which should prompt you with "Do you want to save changes...?" 

Restart Excel to make sure your new Personal file is OK (and has all the code you need), then you can delete 1, 2, and 3.
