ExecuteMso "PictureFitCrop" Equivalent in PowerPoint 2007? - Microsoft Community

yo-eman say:

ExecuteMso "PictureFitCrop" Equivalent in PowerPoint 2007?

I have some VBA code that uses Application.CommandBars.ExecuteMso "PictureFitCrop" to coerce a picture to fit within a shape, show the entire picture, and maintain its aspect ratio. This works fine in PPT 2010, but fails in 2007, because, I believe, the "PictureFitCrop" does not exist in 2007.  Is there an equivalent method or ExecuteMSO parameter that will accomplish the same thing? If not, is there some code I can borrow without having to roll my own?  Thanks!
yo-eman say: I think I have it figured out.  The rescaling method that my previous routine used only took into account one dimension.  I needed to see if the re-scaled imaged exceeded either dimension.  Here is the revised code:

Sub RescaleImage()

    Dim picOriginalHeight As Single
    Dim picOriginalWidth As Single
    Dim shpHeight As Single
    Dim shpLeft As Single
    Dim shpTop As Single
    Dim shpWidth As Single
    Set shp = Application.ActivePresentation.Slides(1).Shapes.AddPicture("C:\WashingtonMonumentLandscape.jpg", False, True, 20, 20, 594, 402)
''    Set shp = Application.ActivePresentation.Slides(1).Shapes.AddPicture("C:\WashingtonMonumentPortrait.jpg", False, True, 20, 20, 594, 402)
    'save target shape's dimensions
    shpLeft = shp.Left
    shpTop = shp.Top
    shpWidth = shp.Width
    shpHeight = shp.Height
    shp.Select
    'Re-scale shape to image's original dimensions
    shp.LockAspectRatio = True
    shp.ScaleHeight 1, True
    picOriginalHeight = shp.Height
    picOriginalWidth = shp.Width
       
    ScaleWidthRatio = shpWidth / picOriginalWidth
   
End Sub

How to create a calculated field that calls the same calculated field - Microsoft Community

puchakombucha say:

How to create a calculated field that calls the same calculated field to create a running sum in MS ACCESS

I am creating a Access database query from an excel spreedsheet where the row "TotalCapitalCalls" is calculated by itself and "CurrentCapitalCall", which is static. In essence it's a running sum with two fields, "CurrentCapitalCall" and "TotalCapitalCalls" together.

The data from the spreedsheet:

                                 A          B           C           D

1 CurrentCapitalCall                     1,000,000     240,000       2,000,000       1,960,000

2 TotalCapitalCalls                       1,000,000   1,240,000      3,240,000       5,200,000

The calculation is as follows:

                                 A          B           C           D

1 CurrentCapitalCall                 1,000,000      240,000        2,000,000      1,960,000

2 TotalCapitalCalls                   1,000,000     =+B1+A2       =+C1+B2      =+D1+C2


So the formula is repeated as data moves to the next TotalCapitalCall field, summing the total to the next 'TotalCapitalCalls' field.

I am trying to create a calculated field in a query using the same calculated field name in the calculation. I need the calculated field "Total CapitalCalls" to sum itself with the field CurrentCapitalCall.

For example:

SELECT LlamadoDeCapital.TotalCapitalCommitted, 
    LlamadoDeCapital.CapitalCallRequestDate, 
    LlamadoDeCapital.CapitalCallDate, 
    LlamadoDeCapital.CurrentCapitalCall, 
    CurrentCapitalCall + TotalCapitalCalls 
AS TotalCapitalCalls
FROM LlamadoDeCapital;

After running the query, I get this message:

Circular reference caused by alias 'Total Capital Calls' in query definition's SELECT list

Is it possible to call the calculated field in it's own expression??

or

Is there better solution to summing the calculated field with itself and the other field?

I have also tried to use the DSum function but when the query is run, nothing is summed in the "TotalCapitalCalls" column.

SELECT LlamadoDeCapital.TotalCapitalCommitted, 
    LlamadoDeCapital.CapitalCallRequestDate, 
    LlamadoDeCapital.CapitalCallDate, 
    LlamadoDeCapital.CurrentCapitalCall, 
    DSum("CurrentCapitalCall","LlamadoDeCapital","CapitalCallDate <=" & [CapitalCallDate]) AS TotalCapitalCalls
FROM LlamadoDeCapital;

Sincerely appreciate any help any one can offer! 


karl donaubauer say:

Hi,

Do you have the Excel sheet as a linked table or did you import the data into a local Access table?

Maybe there's a problem with the date format in the DSum expression. Try to convert it like this:

DSum("CurrentCapitalCall","LlamadoDeCapital","CapitalCallDate <=" & clng([CapitalCallDate])) AS TotalCapitalCalls

puchakombucha say: That worked!!!! You're amazing, Mark! Thank you!!! But I don't understand why that worked? Would you mind explaining?? 

karl donaubauer say:

Hi,

JET/ACE SQL needs dates in American or ISO format to understand/handle them. So you could also use e.g.

"CapitalCallDate <=" & Format(CapitalCallDate, "\#yyyy\-mm\-dd\#")

Internally dates are treated as numbers starting with 1899-12-30 as day 0 --> today is day no 42249). So as a kind of shortcut you can also deliver a date as Long which is what CLng() does.
