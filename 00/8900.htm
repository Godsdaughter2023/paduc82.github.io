Any suggestions?  

jay freedman say:
You may have a third-party add-in that's interfering with the feature. A quick way to test that theory is to run Word in safe mode to see whether the problem stops. There are a couple of ways to do this:
  • Hold down the Ctrl key while you click the Word program's icon. When a message box asks whether to start Word in safe mode, click Yes.
  • Press Windows key + R to open the Run box, and type winword.exe /safe, then press Enter.

Then open a document that has tracked changes, or make some new tracked changes. Try the mouse hover.


In safe mode, Word doesn't load any add-ins, it uses the built-in defaults rather than anything saved in the Normal.dotm template, and it ignores most options saved in the registry.


If the problem stops, try disabling or uninstalling one add-in at a time and re-testing the mouse hover. If taking out one particular add-in makes the problem stop, you can either do without that add-in or check for an updated version.


FYI: Uninstalling and reinstalling Word almost never solves anything, because it doesn't touch the add-ins or the Normal.dotm template (on the theory that you will reinstall and want to keep all your customizations). It's quite difficult to get a completely clean uninstall.

rohn007 say:

Is this only happening in one document, or in all documents?

If it is only one (or very few selected documents), that suggests that the files have become slightly corrupt.  Not bad enough that you can't open the file, but enough to break some functionality.

There are a couple of things you can try to fix the problem.  Work with a copy of the file so you always can fall back to a "known good" (even if it is bad, not as bad as something you have "messed up" experimenting with fixes!) version.

Try to repair the document in Word

  1. In Word, use the "File > Open" dialog
  2. navigate to the file location
  3. SINGLE click to select the file
  4. Click on the drop down arrow beside the "Open Button"
  5. select "Open and Repair"
  6. check to see if the problem is fixed.
  1. Use the File menu > Save As command
  2. Select an alternate format, ie "RTF"
  3. Save to the new format
  4. Open the new format file
  5. Save As back to DOCX
  6. check to see if the problem has been fixed

The process of saving to the alternate file format may cause some formatting or unsupported features to be stripped out. I have always seen suggestions to use RTF or DOC formats, but you could also give ODT (OpenOffice) format a try ...

  1. Open the problem file
  2. use <CTL><N> to create a new blank file
  3. In the problem file, turn on the "Hide/Show" option in Home tab, to show all of the non printing characters
  4. Copy the body of the problem file. Make sure to NOT select the very last para mark in the document (it is one place that "corruption" can hide
  5. Paste into the new blank file
  6. Save the new file with a new name
  7. check to see if the problem is fixed

"Historically" LibreOffice / OpenOffice has been better at dealing with subtle file corruption issues.  Many times people have reported that simply opening a MS document in LibreOffice then saving it has been enough to correct some simple corruption.  Since LibreOffice is free, it is worth trying.

Post an Example file for us to play with

You can also give us access to an example file so we can play with it.  If the document contains sensitive/secret information strip out the secret information.  Cut the file size down to a few pages that show the problem.  Add notes to the file explaining what the problem is.  Describe the steps you are taking to cause the problem so we can try to recreate the problem. Tell us exactly what to look for.

Then upload the example file to your OneDrive account. Give us an Share / Edit link to the example file.
