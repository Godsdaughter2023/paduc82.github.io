bernie deitrick say: Event code is not all that hard to do. For example, if you wanted the date in the cell to the right of any single cell that changes on a sheet when it is changed, copy this code, right-click the sheet tab, select "View Code" and paste the code into the window that appears.

Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Cells.Count > 1 Then Exit Sub
    Application.EnableEvents = False
    With Target(1, 2)
        'Choose any date/time format you like
        .NumberFormat = "mm/dd/yyyy hh:mm AM/PM"
        .Value = Now
    End With
    Application.EnableEvents = True
End Sub

hi

let me repeat Tom's last statement

Let's think hard about these (and others) database design issues before we rush into building forms.

You are trying to resolve a classic Many to Many relationship propblem.

( this is what Scot's subform would also resolve)

Assuming that the basic data for students etc is already in place - the data entry of behaviors needs to be a fast and painless as possible.

I assume this form would be only a small part of a much more involved DB.

I would use an unbound form. ( lets call it "frmBADDIES" )  with . .

1 . . a Combo ( or list) box to select a class (teacher should already be associated with a class)

2 . . A Listbox with list of students - this list is restricted to the class selected in combo box (1)

3 . . a listbox of behaviours . . this should be a table so - As Tom mentioned -  you can easily add new behaviour items.

4 . .  a listbox of behaviours associated with each student. This is where you resolve the M-M issue.

It is a table that links several bits of data. So it needs the following fields

Link_Key . .  a unique id  (key) the identifies the "record" (autonumber is fine)

Student_Key . .  foregn key link to a student

Behaviour_key . .  links to the behaviour record

Class Key . . links to the class in which it happened.

Date . .  what date did the behaviour take place.

 . .  maybe extra fields if you need to store more details .

the data entry process then is . .

open form

select class - code in afterupdate event requeries the student list

select student - code in afterupdete requeries the student behaviour list

select a behaviour - this triggers the afterupdate event - you could immediately run some code to create a new student behaviour record - or pop up a msgbox - or make a button visible to add the new record

then requery the student behaviour listbox.

NOTE . . this is a mouse only driven data entry method . .

this structure may sound tedious - but it is really simple to construct and use . . just takes a bit of practice.

hope this has been good food for thought.

a. user say:

Hi,

 

Thank you for posting on Microsoft Community. We regret the inconvenience caused and will assist you in resolving the issue.

 

The above issue may occur if the Remote Procedure Call (RPC) services are not set to Automatic and Running under Startup type and Status from the Services window.

 

Remote Procedure Call (RPC) services is the Service Control Manager for COM and DCOM servers. It performs object activations requests, object exporter resolutions and distributed garbage collection for COM and DCOM servers. If this service is stopped or disabled, programs using COM or DCOM will not function properly. It is strongly recommended that you have the RPCSS service running.

 

Step 1: Follow the steps below and check if Remote Procedure Call (RPC) service is set to Automatic and Running under Startup type and Status:

 

  1. Press Windows logo key on the keyboard, type Services and select the top most search result.
  2. Locate Remote Procedure Call (RPC)services and check if Status shows Running and the Startup Type is set to Automatic.
  3. If not set, double click on the service and select from the drop down bar Automatic under Startup type and Service status to Running.
  4. Click on Apply and OK.
  5. Check if the changes are effective after closing the window.

 

Step 2: Perform DISM Scan after following Step 1 and check if you are still receiving the above error code and error message on the PC.

 

Windows corruption errors may prevent Windows updates and service packs from installing. For example, an update might not install if a system file is damaged. The DISM scan or System Update Readiness tool may help you to fix some Windows corruption errors.
