dirk goldgar say:

First: Is the form bound to an ADODB recordset?  That would normally only be the case if the form was in an ADP (Access Data Project(), though I suppose you might have assigned an ADODB recordset to the form.  If the form's recordset is a DAO recordset, then no ADODB recordset's bookmark would be compatible with it.

 

If the form's recordset is an ADODB recordset, then code could look like this:

 

    With Me.Recordset.Clone

        .MoveFirst

        .Find "[Dept_ID]='" & Me.cbx_Depts.Value & "'"

        If Not .EOF Then

            Me.Bookmark = .Bookmark

        End If

    End With

 

By the way, in your DAO code, you should not be calling the RecordsetClone's .Close method.

Unable to enter dates in Excel - Microsoft Community

omymyohellyes say:

Unable to enter dates in Excel

Hi, all.

I find that I am unable to enter dates as dates in Excel.  Excel is treating every date I enter as text.  I am NOT talking about a CSV text import - I am talking about manually entered dates.

Example:  Cell entry:  8/14/2016

8/14/2016 is exactly what the cell contents are.  No leading apostrophe or space - nothing.  If I format the cell as a date  DD-MMM-YY it does NOT show 14-Aug-16.  It retains the text as 8/14/2016.

I can format the cell as date before entering it, and it shows as text.

The DATEVALUE function does not convert to date value.  Copying and paste special values returns the text string, not a date.  I am simply unable to enter a date.

Is there a universal entry rule that I am unaware of that enters all dates as text that I may have toggled??

hansv mvp say:

8/14/2016 will only be recognized as a valid date if your system date format has been set to a US date format such as m/d/yyyy or mm/dd/yyyy in the Region control panel.

If your system date format is dd/mm/yyyy as in many European countries, or the ISO date format yyyy-mm-dd, 8/14/2016 will not be recognized as a date, hence it will be seen as text.

tom ogilvy say:

Also, this is somewhat symptomatic of a setting in Excel options.

If you go to file=>Options, then go to advanced and scroll all the way down under "Lotus Compatibility Settings for . . .", if you have a check in "Transition formula evaluation", you will get this behavior.  You can uncheck this box if it is checked.
