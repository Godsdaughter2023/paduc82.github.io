Microsoft Security Essentials and Windows Defender - Microsoft Community

lechwe3322 say:

Microsoft Security Essentials and Windows Defender

I have Vista & use Microsoft Security Essentials (MSE).  MSE has disabled Windows Defender.  In Vista, how do I use Software Explorer to change my startup programs since I can no longer open Windows Defender to access it?
kosh vorlon say:

Hi Lechwe,

 

Windows Defender is part of Windows Vista and Widows 7 (and must be disabled if using MSE). It cannot be uninstalled from Vista or Windows 7 - you didn't install it (it's part of the operating system which is why it cannot be removed and why you'll see files for it). It's not in Add/Remove programs because it cannot be removed. If using Windows XP, then Defender must be uninstalled if using MSE. Normally the installation of MSE does these things automatically.

 

Windows Defender's protection is fully covered by MSE and is not needed. In fact, if using MSE do NOT even try to turn on or install Defender or you will cause problems, crashes, freezes and all sorts of other problems. Defender is NOT needed if you have MSE.

 

See the following for additional information: MSE and Windows Defender.

 

Now MSE does not have an equivalent for Software Explorer, but there are several alternatives, some of which are better than Defender was anyway.

 

There are at least three ways to manage start-up programs (possibly more, but these are enough). I use all three depending on the circumstances.

 

1. Go to Start / All Programs / Accessories / Run and type msconfig.exe and enter. Go to the Startup Tab and you can check or uncheck the programs you want to run at startup. The two drawbacks are that you get little information about the programs and you can't add programs that aren't already there to the listing. And this one is not as easy on the eyes as Defender either.

 

2. Download the free WinPatrol: http://www.winpatrol.com/download.html and you will not only have control over those programs, but you can also add new ones AND, what I like best, you can allow it to run in the background and it will alert you if anything tries to add itself to the startup menu without your knowledge (as sometimes happens with some downloads and installations). You don't have to use that feature if you don't want, but I find it convenient. It also provides more information than msconfig. I find it better than Defender and would use for startup-program control it even if it were possible to run Defender.

 

3. Download the free Autoruns: http://technet.microsoft.com/en-us/sysinternals/bb963902.aspx. This is the gold standard for controlling startup programs and everything else at startup with loads of details and options and different views. The danger here is that it is so good and powerful, that if you're not careful, you can remove something you really don't want to remove and hose your system (but if you pay attention and show use caution and don't change anything unless you know EXACTLY what you are doing and the ramifications of doing it, you'll be fine).

 

I rarely use msconfig. I mostly use WinPatrol. When I'm really into cleaning things up and need to know exactly what is happening a boot and startup, then I go to Autoruns to get all the details I need.

 

This will cover managing start-up and boot programs and processes and everything related much better. And if you choose to run WinPatrol only on-demand and not with real-time scanning (as I do), then none of them will take up any resources unless you are using them.

 

I hope this helps.

 

Good luck!

 

Kosh
