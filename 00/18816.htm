The work-around is to use the connection string.

While you can reset the command string using a string like so:

ActiveWorkbook.Connections("qry_data").ODBCConnection.CommandText = CommStr

You cannot do the same thing with the connection string. You must define it as an array and not as a string.

[code]

Dim ConnStr() As String
Dim CommStr As String

' Remake Connection String
ConnStr = Split(ActiveWorkbook.Connections("qry_Data").ODBCConnection.Connection, ";")
ConnStr(2) = "DBQ=" & Range("Def_Dir") & "\" & Range("Data_File")
ConnStr(3) = "DefaultDir=" & Range("Def_Dir")
ActiveWorkbook.Connections("qry_Data").ODBCConnection.Connection = Join(ConnStr, ";")

' Remake Command String
CommStr = "Select * FROM `<PATH>`.`blad1$` `blad1$` WHERE (`blad1$`.`Vert#`='CC')"
CommStr = Replace(CommStr, "<PATH>", Range("Def_Dir") & "\" & Range("Data_File"))
ActiveWorkbook.Connections("qry_data").ODBCConnection.CommandText = CommStr

' Refresh the query
ActiveWorkbook.Connections("qry_data").Refresh

[/code]

Here is a working workbook!

http://www.aswinvanwoudenberg.com/index.php/2009/01/12/countdown-clock-in-excel/

In the first paragraph there is a little hidden link under the word "here" that allows you to download the example file.

Its relatively simple, using a very short timed event and worksheet formulas to display the countdown.  Looks like it was written back pre 2010, because you'll notice that his "countdown to" date is Jan 1, 2010 :).  Just change that to a date in the future and you'll see it in action.

He has the countdown to date/time in cell B1, with other formulas in cells B5, C5, D5 and F5 to display the calculated days, hours, minutes and seconds remaining in the countdown.

Now, I would have done a few things differently, and I'll show you here what they are.

First, I'd change the ThisWorkbook code to this:

Private Sub Workbook_BeforeClose(Cancel As Boolean)
  On Error Resume Next
  Application.OnTime nextTimeToRun, "OnTimerFunc", , False
  If Err <> 0 Then
    Err.Clear
  End If
  On Error GoTo 0
  ThisWorkbook.Saved = True
End Sub

Private Sub Workbook_Open()
    OnTimerFunc ' this starts the timed event
End Sub

 

Then I would change the code in the modFunc code module to this:

 

Option Explicit
Public nextTimeToRun As Date

Public Sub OnTimerFunc()
    nextTimeToRun = Now + TimeValue("00:00:01")
    Application.OnTime nextTimeToRun, "OnTimerFunc"
    shtCountdown.Calculate
    'Application.Caption = _
     CStr(shtCountdown.Range("B5").Value) & " days, " _
     & CStr(shtCountdown.Range("C5").Value) & " hours, " & _
     CStr(shtCountdown.Range("D5").Value) & " minutes, " & _
     CStr(shtCountdown.Range("F5").Value) & " seconds"
    DoEvents
End Sub

jllatham say:

Here's a "cheap" way to use that CountDown workbook without changing much at all!!

Add a new sheet to your workbook to hold the countdown results as you wish to display them (might even be able to copy your current sheet using C1 and C2 into it).  Let us, for this example say that it is named "ShowRemainingTime".

On the current [Countdown] sheet, change cell B1 to contain this formula:

=ShowRemainingTime!C1

Now, if you want to you can even hide the [Countdown] sheet.

On the ShowRemainingTime sheet, enter the date/time to count down to into cell C1, and enter this formula into cell C2:

=Countdown!B5 & " Days " & Countdown!C5 & " Hours " & Countdown!D5 & " Minutes " & Countdown!F5 & " Seconds Remaining"

The only change to the code is to also recalculate this new sheet when the other is recalculated, so the code for the OnTimerFunc() becomes (and note I removed the code that changes the Excel window Caption.

 

Public Sub OnTimerFunc()
    Application.OnTime Now + TimeValue("00:00:01"), "OnTimerFunc"
    shtCountdown.Calculate
    Worksheets("ShowRemainingTime").Calculate
    DoEvents
End Sub

Hi Deveshovi,

 

1. If you get "Certificate error" during installation of Nokia Store application:

This error generally occurs due to the certificate is out of sync. User can update the device firmware to get the latest certificates. Set the device date and time to the current then download and install the application.

 

If the issue still persists, check the details (certificate name, start and expire date) of certificate from the error message and change the current date to accommodate in between the start and expire date of certificate.

 

2. If you get "Unable to install" or "Certificate may yet not be valid":
If you get such an error message, please set the device date to before 12.6.2007 (or for some applications maybe even earlier), attempt to reinstall .sis file, installation continues normally. After installation you may set date back to current date.

 

All Nokia Content Signing Certificates have a validity period. During the time when a certificate is valid any signature that is associated with it can be validated. When a certificate expires, the signature is not valid anymore. Nokia Content Signing Certificates used between 13.6.2002 and spring 2006 for Symbian installation system packages (.sis files) was valid until 12-Jun-2007.

 

Nokia is currently working on updating its Content Signing Certificates.

 

If the steps above do not help, please call Nokia Care Points with the below details:

 

- Application name

- Device product code, firmware version, IMEI number

- Details of error (certificate name, start date, expire date)

- Nokia account ID
