Specify a parent/child relationship with Parent Field = - Microsoft Community

hallralph say:

Specify a parent/child relationship with Parent Field = OR( child field1, child field2)

Access 2010 Subform Wizard

Form has a field (Contestant ID), Table for subform has two fields for players in a match( Contestant ID1, Contestant ID 2). I want the subform to show all rows where the contestant of the was in a match whether the contestant was in slot 1, or 2.

 

The wizzard allows only AND conditions between up to 3 match fields between Parent and Child.

 

parent Id = 12345

Child contestant 1=12345, contestent 2 = 67890

Child contestant 1 = 95124, contestant 2 = 12345

 

 

 

ken sheridan say: The root of the problem is that your table design is flawed.  By having separate columns for slot 1 and 2 contestants you are doing what is known as 'encoding data as column headings'.  A fundamental principle of the database relational model is the Information Principle.  This requires that all data be stored as values at column positions in rows in tables, and in no other way.

What you have here is a many-to-many relationship type between matches and contestants.  A many-to-many relationship type is always modelled by a table which resolves the relationship type into two one-to-many relationship types like so:

Matches----<MatchContestants>----Contestants

So, the tables would along these lines:

Matches
....MatchID
....MatchDate
....Venue
....etc

Contestants
....ContestantID
....FirstName
....LastName
....etc

and to model the many-to-many relationship type between them:

MatchContestants
....MatchID
....ContestantID
....Slot

The primary key of the last table above is a composite one made up of the MatchID and ContestantID columns.  In this table these must be of straightforward long integer number data type, not autonumbers as in Contestants and matches.  The Slot column should be given a ValidationRule property restricting its values to 1 or 2.

You can create a parent form, in single form view,  based on Contestants and within it a subform, with ContestantID as the LinkMasterFields and LinkChildFields properties, based on a query such as this:

SELECT MatchContestants.MatchID, Slot, MatchDate, Venue  
FROM MatchContestants INNER JOIN Matches
ON MatchContestants.MatchID = Matches.MatchID
ORDER BY MatchDate;

The subform can have a combo box bound to the MatchID column, and  text boxes bound to the Venue and Slot columns The combo box would be set up as follows:

RowSource:     SELECT MatchID, MatchDate FROM Employees ORDER BY MatchDate;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column and that the second is at least as wide as the combo box.

This arrangement would allow you to select a match by date when entering data in the subform to assign a contestant to a match.  The Slot value (1 or 2) would be entered in its text box.  The Venue text box would show the venue for the selected match automatically, so to make this control read-only set its Enabled property to False (No) and its Locked property to True (Yes).

You can of course arrange the controls differently, e.g. to select a venue in the combo box and show the date automatically, or you may wish to select a match by some other attribute.  Whatever the case the basic principle is that the combo box bound to the Match ID column needs show some value in its visible (second) column by which you can identify the match, i.e. it should be a column with distinct values, or if no single column can do this, a combination of columns.

a. user say:

Hi Surya,

As per the information, I would like to inform that you cannot disable only the Open option when you download a certain file. However you can disable the entire pop-up for certain files that you wish not to view the window for. To do so, follow the steps below:

  • In Internet Explorer 10 or 11 you will first have to click on a file of the type you want to open automatically next time.
  • This will give you the Open/Save prompt.
  • Select the Save option.
  • Then in Internet Explorer click Tools
  • Under Tools click the View Downloads option
  • You will see a popup containing the file that you just downloaded.
  • Right-click on the file
  • In the options presented, make sure to un-check the option labeled: Always ask before opening this type of file

Hope this helps
