After much experimentation I have finally found a simple (ish) working solution to this problem, and am posting it in the hope it will help somebody else.

The Problem:

Using the EZDok camera addon for Microsoft Flight Simulator X under windows 8.1, the mouse-look feature (enabled by holding the middle mouse button to pan around the cockpit) does not work with the newest version of the software (EZDok v1.7). It has something to do with file permissions (as I outlined in my original post), but beyond that I have no clue of the exact cause of the problem.

The Solution:

I have taken many steps to get this add on to work, and I don't have the patience to undo / redo all of these to work out which ones are actually needed, so this is what I suspect to be the order of importance, however some steps may not be necessary. Do the first step only, then try FSX. If it didn't work, try the next step, etc.

  1. Every time, before starting FSX, run the installer for the original v1.5 EZCA (yes, even if you have v1.7 installed as I do). The version stays as v1.7 and I have not noticed any ill effects in use for doing this. At the end of the installation it brings up the EZCA Config, just click Configure FSX, then start FSX. This whole process take around 10 seconds if you have a shortcut to the installer on your desktop, and this is the only thing I have to do every time before I start FSX.
  2. a. If the above alone doesn't work try these steps one at a time (checking to see if it works after each step) - change the compatibility settings of the EZCA installer to 'Run as Administrator' (right click the file, go to 'Properties', go to the 'Compatibility' tab). Then, run the EZCA v1.5 installer again.

    b. Change the compatibility settings of the EZCA installer to 'Windows XP (Service Pack 3)". Then, run the EZCA v1.5 installer again.

    c. Change the compatibility settings of FSX.exe AND EZCA.exe to 'Run as Administrator'. (EZCA.exe is located in C:/Program Files (x86)/EZCA/ by default). Then, run the EZCA v1.5 installer again.

    d. Change the compatibility settings of FSX.exe AND EZCA.exe to 'Windows XP (Service Pack 3)". Then, run the EZCA v1.5 installer again.

  3. Disable UAC (User Account Control). Go to Control Panel > System and Security > Change User Account Control Settings, and set to 'Never Notify'. Restart your computer after this step and run the EZCA v1.5 installer again before attempting to start FSX.

  4. If all above has not worked, try what I suggested in my original post about setting folder permissions for the 'Users' group to 'Full Control'. Restart your computer after this step and run the EZCA v1.5 installer again before attempting to start FSX.

This is what works for me - I hope it does for you too, as I'd almost given up on Windows 8 before I came across this. Good luck!
