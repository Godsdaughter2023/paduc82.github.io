Thanks in advance!

Terry....

myall_blues say:

Hi Terry

What you ask can really only be done with a macro, but I would really advise against setting up a spreadsheet where you add a value in a single cell that then progressively updates a total. If you make a data entry mistake or need to subsequently change a day's value you end up in a world of pain getting the total correct, if you know that it is wrong at all.

A better solution is to set up a simple page (perhaps one for each month) in which you enter the date in one column, and then the corresponding production data in other columns.

Then you can calculate the running monthly totals or whatever using simple formulas (=SUM for example). If that total has to be on another worksheet page no problem, and you could for example create a tab called 'Monthly totals' in which you put each month name in one column and then add the formulas that look at the individual month pages.

Note that the formula can span a bigger range than that for which you have data to allow for the future days in the month, so for example =SUM(B2:B32) will account for all eventualities of month size and you don't need to change the formula every day.

Hope this is helpful

Murray (#SA :-)

WM6 auto-configures @gmail.com addresses to use POP3, since the auto-configure feature was baked before Gmail offered IMAP access.

To reconfigure your account as IMAP you must use a workaround:
1) Delete your existing POP3 account

5) Finish setup but do not attempt to sync
6) Edit the new account

Going by your description of the exported data, the issue is simply that the wrong e-mail field got exported or your importing the wrong e-mail field.


More specifically, the email value you indicate occurs if you export the <E-mail> field (or E-mail 2, E-Mail 3) which is comprised of 3 different fields <E-mail Address>, <E-Mail Type>, <E-mail Display Name>. (these fields are the ones that appear when you click on the + sign beside the root e-mail group and are also the fields that get exported when using the default field map)


The solution to the problem is to ensure that you are only importing the E-Mail Address field exclusive of everything else.


Assuming that you have other existing contcts and only want to delete only the contacts that were involved in the last import with the incorrect email addresses, create a customized "list view" (i.e. phone book view) and add the creation or modified date to the view and sort the contats on that field. This will result in all contacts created from the import to be together and delete all applicable contact items. Then re-run the import ensuring that the correct email address field is mapped.
