  • net start cryptSvc
  • net start bits

rob koch say:

It's unlikely this problem is caused by malware and much more likely that either something is shorting these keys together or the keyboard encoder circuitry is failing.

You didn't specify what kind of computer or keyboard, so if a desktop PC then swapping with another keyboard is worth a try.  If it's a laptop, then plugging an external keyboard into a USB port or trying the on screen keyboard in accessibility mode using a mouse to type will confirm whether software is intercepting and changing the keys or the problem is hardware.

The clue here is that each keypress results in 2 codes, typically for keys either right next to each other or at least in 2 small groups in the top row of keys.  This indicates some sort of physical short in the matrix between these 2 groups or a logical failure in the encoder chip translating them.

After testing to confirm if possible, I'd contact your technical support for the computer and ask them what options for replacement might be.  If it's a basic PC keyboard then any local computer or big box seller of them should have replacements available if it's out of warranty.

Using Picture Tools in Word 2010 - Microsoft Community

clairmatin say:

Using Picture Tools in Word 2010

Using Word 2010, I might choose to make changes to a picture using Picture Tools, such as set color saturation to zero.  This makes the picture appear in grayscale.

Do the changes I made get applied to the stored picture, or are they applied only when rendering the picture when opened?  I ask this because I pick up the picture later from inside the zip/docx.


jay freedman say:
It's easy enough to do a little experiment...

In a blank document, I inserted one full-color JPG picture, and set its saturation to zero (as in your example). I saved the document, changed its extension to .zip, and opened the word\media folder in the zip.

The folder contains two files: image1.jpeg is the grayscale version, and hdphoto1.wdp is the full-color version of the same picture. Evidently, the .wdp signifies a .hdp (high-resolution lossless-compression) file created by Word; it can be opened by Irfanview and Windows Photo Viewer, among others, but not by MS Paint. It also can't be inserted as a picture into a Word file (you get an "object" icon that opens the picture in another program), but Irfanview can convert it to .jpg format and then you can use that.

This makes sense: if you change the extension back to .docx and reopen the document, you can select the picture and click the Reset button on the Picture Tools tab. You get back the original picture. The only way Word could do that is by storing the original in the file. For performance reasons, though, the grayscale copy also has to be in the file; if the program had to reapply all the changes to every picture in a substantial document, it would take a very long time (at least in computer terms).

Interestingly, after resetting the picture and saving again, the word\media folder in the zip file now contains the file image1.png -- not a .jpeg like the grayscale version.
