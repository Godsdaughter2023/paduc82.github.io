Nokia PC Suite – how to import contacts from a csv text file - Microsoft Community

cdl_1 say:

Nokia PC Suite – how to import contacts from a csv text file into your mobile

Hi all,

I thought I’d share my “findings” on how to import a list of contacts from a text file into your mobile using Nokia PC Suite.

Exporting the current contacts to a text file is the best way to understand Nokia’s format. You can do it easily with PC Suite.
You’ll notice that the file is separated by commas, and that all fields (even the blank ones) are enclosed by "". This is probably the trickiest part: since I know of no way to enclose blank fields in "", I came up with another solution: it might not be elegant but it works!

1) Prepare your list of contacts in a spreadsheet (e.g. Excel or an open-source equivalent) in the Nokia format (e.g. with columns in the same order as you see in the file exported by PC Suite from your mobile)

2) Create another tab in the spreadsheet

3) Enter this formula in cell A1 of the new tab:
="|" & MyTab!A1 & "|"
where MyTab is the tab with the contacts data.

4) Drag the formula all the way down. This ensures all cells are now enclosed by pipes, even the blank ones. In other words, 1234 becomes |1234| . You’re free to replace the pipe with any character you like, as long as of course this character never occurs in Mytab

5) Export the newly created tab into a csv file

6) Open the csv file with a text editor (notepad, notepad++, etc)

7) Replace every occurrence of | with "

You now have a file compliant with Nokia format, and ready to be imported into your mobile using Nokia PC Suite.

I tried to input the quotation symbols " directly into Mytab, but wasn’t successful, because that’s the character Excel uses to identify and enclose strings.

If anyone knows about a more straightforward, less cumbersome process, I’d be glad to hear it!

As an alternative, you could import your text file into Outlook and sync it with your Nokia. However, I don’t like Outlook, I think following my process is faster than mapping every field to the Outlook fields, and, when I tried using Outlook, I had plenty of problems because Nokia doesn’t bother documenting which outlook fields are synced and which are not, so some of your precious data may simply not be synced at all.

Also, I’d like to complain to Nokia for requiring such a cumbersome format, and most of all for not documenting this anywhere! Come on guys, it wouldn’t have cost you much to add a couple of comments on the help document of PC Suite…

Hope this helps! Bye!

lars-Åke aspelin say:
I am trying to count a range of cells that contain names. I would like to be able to use the strikethrough font on individual cells so that those cells are not included in the count when the font is used within that range of cells.

I think that you will need macros for this. Try this User Defined Function:

Function count_not_strikethrough(r As Range)
   Application.Volatile
   For Each c In r
      If Not c.Font.Strikethrough Then
        n = n + 1
      End If
   Next c
   count_not_strikethrough = n
End Function

Hope this helps / Lars-Åke

What happens if you push F9 after changing the font? The formula should now be recalculated to show the correct number of  not strikedthrough cells.

To get a recalculation done, try this macro to toggle the strikethrough status of the selected cells:

Sub toggle_strike_through()
  For Each c In Selection
     c.Font.Strikethrough = Not c.Font.Strikethrough
  Next c

  Application.Calculate

End Sub

You can assign this macro to a key combination, e.g. CTRL+Q
