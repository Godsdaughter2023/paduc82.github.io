When you use quotes like you did you create a literal. So Access interprets that you want to convert the string [customer] not the contents of the field. Since the symbols [ ] can't be converted they are left out.

Customer:StrConv([YourTableName].[Customer],3)

jay freedman say:
You can add an AutoText gallery to either or both of the ribbon and the Quick Access Toolbar.

On the ribbon:
  • Right-click anywhere on the ribbon and choose Customize the Ribbon.
  • In the dialog, click the tab in the right-hand list to which you want to add the gallery, and click the Custom Group button below the list. (You can't modify any of the built-in groups, so you need to add a custom group.) Click the Rename button and change the name to 'Auto' or some such.
  • On the left side, set the dropdown at the top to All Commands. Scroll down the list and click on AutoText. Click the Add button between the lists, and click OK.


On the Quick Access Toolbar:

  • Right-click anywhere on the ribbon and choose Customize Quick Access Toolbar.
  • On the left side, set the dropdown at the top to All Commands. Scroll down the list and click on AutoText. Click the Add button between the lists. If you want to, click the AutoText item that's now on the right-hand list and click the up or down arrows in the right margin to move the button in the list.
  • Click OK.


Compatibility mode should have no effect on this. If you want to remove compatibility mode (so you can use the features that are new to Word 2010), here's how: Use the File > Open command, navigate to the Templates folder (usually at %appdata%\Microsoft\Templates), and open the Normal.dotm template. Click File > Info and click the Convert button. Save the template and close Word, then reopen it. Compatibility mode will be gone, unless you open an old document that hasn't been converted yet (and you can use the same treatment on it).
