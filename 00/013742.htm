HI,

From your description it appears you have mistakenly placed some websites in your IE Trusted sites list.... website security in the Trusted zone (Medium) is actually LESS secure than the Internet Zone (Medium High)... If you are a HOME user then you should not make changes to your IE Security zone settings... they are designed to work with public access websites without making changes to the default security settings.

Google like other large websites use an Authentication service on another domain (account.google.com) so when your homepage (google.com) opens in your browser it tries to contact account.google.com to verify your google account credentials... sending your IE browser into a loop, since google.com and account.google.com are in different IE security zones...(IE has a security zone setting: "Prevent navigation into a zone of lower integrity"... other web browsers do not use Security Zones.

Take any public access websites out of your IE Trusted Sites list....

Tools>Internet Options>Security tab.

1. click "Reset all zones to default"

2. then select the Trusted Sites icon, then the 'Sites' button and remove any websites you have placed in the list.

Save changes. Then close and reopen IE.

google.com should open now and log you in automatically to your google account... (you will see your account avatar in the top rhs of their webpage.

If you share your computer with other family members, make sure you have created separate windows accounts for each user, otherwise (if you share the same windows login details) other members will be able to access the website(s) accounts... and see your surfing history... or login automatically to sensitive websites like your internet banking websites.

Note also: you must be connected to the Internet when you start IE in order to navigate to the google landing page.... by default IE will show "You are not connected to the internet" message or a blank page.

john mcghie say:

Hi Ferdinand:

Have a quick check to ensure that Tracked Changes has not been left turned on.

The RTF version of a file is effectively the UNCOMPRESSED version of the binary version of the file.  The binary version of a file should be around four times larger than the .docx.  The uncompressed version of that should be between 10 and 20 times the size of the compressed version.  So your RTF size is about right.

Daniel's suggestion to round-trip the file to RTF and back has the effect of removing "stranded RTF" from the file. Phillip has explained that a "Save" tends to create a file which is effectively the original plus a series of change logs that describe how the current version differs from the original.  As he says, this enables the "undo" function.

However, it is possible under some circumstances to make a change, then partially undo it, leaving a large swatch of text or graphics "stranded" -- no longer referred to in the file, although Word cannot detect this.  When you get large "bloating" of a file, this usually means graphics are involved.

If you save this mess out to RTF, Word writes out the whole file as RTF, and you see 8 MB file size.  However, when you read that file back in, the RTF filter ignores any code it does not use.  It does not pass it back in to the file.

Hence Daniel's method is one way to clean up a file in which you suspect a large lump of code has become "stranded".

Hope this helps
