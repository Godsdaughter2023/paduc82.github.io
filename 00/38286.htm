Hi TitanOne,

Interesting thing you're trying to do, hopefully I can help. You have the right idea, the batch file should give way to an elevated Command Prompt, although the runas command you've mentioned will not work for your intents and purposes, since it will prompt you for a password, which is not automatic as you would expect.

This may not be as practical a solution as you anticipated, but maybe you could disable UAC? Go into Start, then search for Change User Account Control settings, then click it. In the new window, set it to Never notify. UAC should now be disabled.

Now we have to make sure that your batch file has elevated privileges. Where your batch file is, right-click it and navigate to Send to > Desktop (Create shortcut).

With your new shortcut, right-click that, and go into Properties. In the Shortcut tab, click the Advanced... button. Mark "Run as administrator". Because you've disabled User Account Control, this will not prompt you.

Now map that binding to your new shortcut.

ronniewarren say:

Can a email address be traced to origin accurately

Can the origin of an email address be traced and if so how.

brian tillman [outlook mvp 2007-2019] say:
The simple answer is, "no."  Nearly everything in the metadata for a message can be faked.  About the only thing you can mostly rely on is which network forwarded the message to your mail server.  I'd tend to trust most of the "Received" headers, but with a grain of salt.

ronniewarren say: How can a network be traced. For example, would the network identify to us whether it is in North America, Europe etc. If the sender says that they are in Seattle would the network info support that.
