rickhenderson77340 say:

different normal.dotm for english and spanish documents

I would like to use a different normal.dotm version for spanish and english documents; something like normalSp.dotm and normalEng.dotm. Is there a way to accomplish like this?
jay freedman say:

I'd question whether it's necessary to have different files, but that's a different discussion.

There's no way to do this directly in Word. During its startup, the program loads the file with the specific name Normal.dotm from the User Templates folder. It won't look for any file with a name that's different by even one character. If it doesn't find that file, it will create one with that exact name (if you do anything during the session that requires saving to the Normal template).

To get this effect, you'll have to work outside of Word, with a pair of batch files. Each file would copy one of your two templates to the Normal.dotm name, overwriting the existing template. Then the batch file starts Word, which uses the Normal.dotm file.

The other thing you require is a way to save any changes made to the template during the session, getting them into the correct file. That can be done with a macro named AutoExit, which runs automatically when Word is about to shut down. It would determine which of your two templates was used to create the current Normal.dotm, and save a copy of Normal.dotm to that file.

The programming specifics of the batch files and the AutoExit macro depend on where you plan to store the templates.

Clearly a very old thread now, but I've recently hit this issue and identified that the reason for hit an miss display of addresses is simply down to the cumulative length of the various fields selected. So, if you have the usual Name, Company, Phone No, email, etc. with some blank lines and then have a SHORT(ish)  address it will display happily. But, if when added the address and above information comprise more lines than available on the Business card, it won't display. It doesn't just show the available number of lines-worth of text from the address as you'd expect; it simply refuses to show anything from that data field. So the solution is any permutation of compacting the way the address is written together with removing any unnecessary blank lines and/or other fields from the business card. Then the address will display perfectly OK every time.

Updating from what?   This sounds like you started from before the November update where the W10 networking problems really started.  In order to work around that I had to reconfigure my system to make sure that my ISP's DHCP was not being used.  Reason:  it does not support local IPv6 and that update broke ISATAP somehow.  So your symptom (especially the "10 minutes" observation) reminds me of trying to use the Homegroup and the HG troubleshooter while there was no local IPv6 connectivity.  They both stupidly continued pretending it might work without checking for that fact first!

To check if you have IPv6 LocalNetwork connectivity use PowerShell  Get-NetConnectionProfile.

Related to this but I don't know if it matters, I disabled Teredo and enabled ISATAP explicitly (instead of just letting it try to work by accident).

PS>netsh in isatap sh st
ISATAP State           : enabled

But I am sure that my real fix was stopping using my ISP's DHCP and switching to my Dlink.  Previously a configuration with the Dlink either bridged from the modem or as a separate subnet had been working but the November update broke both of those cases.
