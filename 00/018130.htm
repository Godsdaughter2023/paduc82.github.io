ken sheridan say:

The report is to help the user to compare results between specific years like 2016 to current. I only want the user to type the year "2016" and end year "2019"  as recommended to use a start and end date.  Will the query above require me to type out the whole date on the form like "1/1/2016" and "12/31/2019".

For that you would need StartYear and End Year controls in which the years (not dates) are entered or selected in the dialogue form, and the query would be along these lines:

SELECT *
FROM Sales
WHERE (Agent = Forms!f_NetSales!Agent
    OR Forms!f_NetSales!Agent IS NULL)
AND (YEAR(SaleDate) >= Forms!f_NetSales!StartYear
    OR Forms!f_NetSales!StartYear IS NULL)
AND (YEAR(SaleDate) <= Forms!f_NetSales!EndYear
    OR Forms!f_NetSales!EndYear IS NULL);

You'll find an example of something similar to this in MultiSelect.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

The option to 'Restrict Report's Underlying Query on One or More Fields' in this little demo opens a dialogue form in which multiple controls can be used to restrict a report, including two text box controls in which a range can be defined by start and end dates.
