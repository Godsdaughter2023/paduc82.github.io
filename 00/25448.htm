mike h.. say:

Hi,

Instead of trying to guess the colour you've used it's far simpler to compare the colour to a reference cell.

Alt +F11 to open VB editor. Right click 'ThisWorkbook' and insert module and paste the code in. On the worksheet call with

=CntColor(A1:A17,B1)

where A1:A17 is the range to count and B1 contains a character of the same colour as those you want to count.

Function CntColor(rng As Range, Fclr As Range) As Long
For Each c In rng
If c.Font.Color = Fclr.Font.Color Then
    CntColor = CntColor + 1
End If
Next
End Function

rohn007 say:
Microsoft counts current activations.

The number of concurrent activations (and other rights) depends on the type of license you buy

OEM (factory installation) and PKC (Product Key Card) can be activated on only one machine. Period.

FPP (Retail) licenses bought as online download or Boxed CD may be activated on 2 machines at the same time, your "primary" device and a portable device. Home and Student may be activated on up to 3 machines in your home at the same time.  As well, FPP installations may be transferred to new machines as required.


For all types of license, you can do some (unspecified amount of) upgrading of the computer, such as replacing the HD and still activate on the same machine.

There are "volume", non-retail, licenses that work under other terms.

Laura,

To do this you need to create and use a custom namespace in your RibbonX code and then use the "Q" attribute when identifying the controls.  The following is a small sample of code that might serve as the "Master" global AddIn template:

<

 

customUI
xmlns=http://schemas.microsoft.com/office/2006/01/customui
xmlns:nsMaster="MasterRibbonAddin">
<ribbon>
  <tabs>
    <tabidQ="nsMaster:masterTab"label="Master">
      <groupidQ="nsMaster:masterGrp1"label="Master Group 1">
        <buttonid="btn1"label="Button 1"imageMso="HappyFace" />
        <buttonid="btn2"label="Button 2"imageMso="HappyFace" />
      </group>
      <groupidQ="nsMaster:masterGrp2"label="Master Group 2">
        <buttonid="btn3"label="Button 1"imageMso="HappyFace" />
        <buttonid="btn4"label="Button 2"imageMso="HappyFace" />
      </group>
    </tab>
  </tabs>
</ribbon>
</customUI>

And the following might be used in a simple "Slave" template to insert a new group between the two master groups:

<

 

customUIxmlns="http://schemas.microsoft.com/office/2006/01/customui"

xmlns:

 

nsMaster="MasterRibbonAddin">
<ribbon>
  <tabs>
    <tabidQ="nsMaster:masterTab">
      <groupid="slave1"label="Slave Group"insertBeforeQ="nsMaster:masterGrp2">
     
      </group>
  </tab>
</tabs>
</ribbon>
</customUI>

bernie deitrick say:

Use this macro:

Sub TwoColDBtoCrossTab()
    Dim rngC As Range
    Dim rngTable As Range
    Dim shtCT As Worksheet
    
    Set rngTable = Range("A1").CurrentRegion
    
    Set shtCT = Worksheets.Add
    shtCT.Name = "Cross Tab"
    
    Set rngTable = rngTable.Offset(1, 0).Resize _
    (rngTable.Rows.Count - 1, rngTable.Columns.Count)

    
    For Each rngC In rngTable.Columns(1).Cells
        If IsError(Application.Match(rngC.Value, shtCT.Range("1:1"), False)) Then
            shtCT.Cells(1, shtCT.Columns.Count).End(xlToLeft)(1, 2).Value = rngC.Value
        End If
    Next rngC
    
    For Each rngC In rngTable.Columns(2).Cells
        shtCT.Cells(shtCT.Rows.Count, Application.Match(rngC(1, 0).Value, shtCT.Range("1:1"), False)).End(xlUp)(2).Value = rngC.Value
    Next rngC
    
    shtCT.UsedRange.EntireColumn.AutoFit
    shtCT.Range("A:A").Delete
End Sub

2)  Click in the Footer again, then go to Format>Font.  You will see the same dialog, but this is NOT the same setting.  The setting in the Style controls every part of every footer; the setting you see here controls only the selected characters.  Again, it should match the setting in the Style: "Automatic".  Correct it if necessary and OK your way out.

3)  Now go to Tools>Track Changes>Highlight Changes.  The top option, "Track changes while editing" should be OFF.  If it isn't, correct it and OK your way out.  If Track Changes while editing was ON, now go to the Review tab, and make sure that "Tracking" is set to "Final showing markup".  If it is not, it is possible that what you type will be hidden.

4)  It is possible that what you type is off the page below the printing area.  Go to Format>Document>Margins.  Make sure the bottom margin of the document is set to 2.5 cm, and that the "From edge" setting is 1.25 cm.  They can be more, but if they are less, it is possible that the footer is being hidden.

5)  If none of those fixed it, it's probably the font itself that's bad.  Try this:
http://word.mvps.org/Mac/fontweeding.html

Hope this helps
