arminatorx say:

NAT and port forwarding have nothing to do with the console being connected via WiFi or network cable. 

So in order to troubleshoot that issue, try to connect your console via a network cable (just temporarily for testing purposes). 

If the console still disconnects, you need to check with your internet provider if they are doing CG-NAT. 

If the console works properly with the cable, but disconnects via WiFi, you probably have interference that disconnects your console "inside your home". 

Either try a different WiFi channel (setting in your WiFi router), or if no suitable interference-free channel can be found, connect the console with an ethernet cable. If running cable through your house is not possible, try using a home plug adapter like those from Devolo or similar makers.

As you know, there are no Downgrade Rights from Windows 8.1 > Windows 7 Pro.

You need Windows 8.1 Pro for Downgrade rights.

Bad sectors usually indicate a hard drive that is beginning to fail. Usually, lost or bad clusters are the result of improper program terminations, like those due to power loss or shutting off a computer without closing the applications first.

You may try repairing the Windows system file using SFC and DISM command but this could be temporary fix.

Once the hard disk gets bad sectors their might be chances of increasing Bad cluster which would lead to hard disk failure. In such cases, subsequent scans can reveal a larger number of bad clusters. This means the hard drive will completely fail at some point in the near future.

You can format the hard disk and run the check disk before installing Operating System but after some time you may face performance issue with Windows due to hard disk failure.

As soon as you see any bad sectors show up on a disk that contains data, make sure you back up the disk immediately, or you risk possible loss of your data. I would avoid making any significant changes to the software setup on the drive, or adding any new data to it.

Hi Muni,

 

Due to the high volume of enquiries related to Windows 10, we could not respond to you on time. I suggest you to follow the suggestions below and check if that works for you:

  1. Open the Task manager.  Here’s a tip:  Press CTRL+Shift+ESC.
  2. Click File > Run New Task.
  3. Make sure you have a check mark beside “Create this task with administrative privileges”
  4. Type CMD.
  5. Type the following 4 commands at the CMD prompt:
    dism /online /cleanup-image /restorehealth
    sfc /scannow
    powershell
    Get-AppXPackage -AllUsers |Where-Object {$_.InstallLocation -like "*SystemApps*"} | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}
  6. Close the CMD window

Check to see if the issue is resolved. If not, restart the PC.
