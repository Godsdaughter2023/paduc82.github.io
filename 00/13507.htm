F1 plan was formerly known as a Kiosk Plan. This plan is for light users who would access email occasionally, typically through kiosks, hence the name.

Some example of users who would be good for F1 licensing includes pilots, air stewardess, nurses, drivers..etc.

The F1 license allows email access via outlook web and pop3 only. It does not allow access via Outlook desktop client via MAPI.

Your options are as follows -

1. Upgrade those users who require Outlook Desktop access to Business Essential, E1,E3,E5 or Exchange Online Plan 1

2. Configure Outlook desktop to access F1 plan using pop3. However, you will not get a rich user experience such as calendars, global address lists...etc

If you do not need any of the other features you can consider Exchange Online Plan 1 at USD 4 per user per month

If you have less than 300 users, the Business Essential plan costs USD 5 per user per month will work as well.

If you have more than 300 users, than E1 at USD 8 per user per month may be required.

You can also mix and match, meaning you can mix F1 + Exchange Online Plan1 + Business Essentials + E1 all in the same tenant in different combinations and number of users.

You could do literally as you suggest by using the SendKeys function to simulate typing Alt+F (to activate the Fields list within the dialog) and then typing "Ref". This should work, although SendKeys has a reputation for being flaky.

Sub x()
    With Dialogs(wdDialogInsertField)
        SendKeys "%fRef"
        .Show
    End With
End Sub

The alternative is to write a userform (http://gregmaxey.mvps.org/Create_and_employ_a_UserForm.htm) containing a listbox, which is loaded with the names of all the bookmarks in the document, and that inserts a REF field when the OK button is clicked. yangzheng_msft6300 say:

Hi Philip,

If you have an existed distribution group with the students in it, you can only check each name of the members and remove them. If you just want to create a group without all the students, as a work around, a dynamic distribution group can achieve that in Office 365. A dynamic distribution group is a group with dynamic members. Those users with the attribute satisfying the rule of dynamic distribution group will be included in the group. So you can create a dynamic distribution group and set up an employee type attribute to staff. Then the student users will not be included in this group.

Regards,
Yang
