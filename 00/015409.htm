seansykes2019 say:

Having an Auto_Generated Number start from a certain number

Thanks

Sean

scottgem say:

Assuming that your auto generated number is a an AutoNumber datatype yes there is a way to do this. But it is probably wise not to mess with it. Let me explain.

An autonumber datatype has one purpose and one purpose only. It is used to provide a unique value to identify a record. there are many people who subscribe to the theory that an autonumber PK should be only used internally and not visible to the users. I am not one of them however. I do use Autonumbers as a visible identifier, but if you do, you need to understand that it will not be sequential. That there may be gaps. If the client is OK with that then the trick is to make a copy of the structure of the table and change the Autonumber to a Number>Long Integer. Then enter one record with 6000 in the ID field. Next, use an append query to Append that record to the master table. New records should then start with the number 6001. 


If you already have records that you want to renumber. Use a new blank table as new master. And after you append the 6000 record, then append the existing records without the ID field.

If the client wants a sequential number, then check out my blog on Sequential Numbering.

Shortcuts for switching desktops and moving windows across them. - Microsoft Community

brenns10 say:

Shortcuts for switching desktops and moving windows across them.

It's exciting to see Windows moving in the direction of virtual desktops (workspaces).  As a Linux user, I've missed this feature constantly in Windows.  So, I just want to share my opinion to Microsoft on how they can improve virtual desktops, specifically the keyboard shortcuts.

I'm used to a really simple scheme for navigating and moving windows around (completely via keyboard shortcuts).  The shortcuts are:

  • Control + Alt + Arrow Key = move one workspace/desktop in the direction of the arrow you pressed
  • Control + Shift + Alt + Arrow Key = drag the currently focused window with you as you move

I know that you're using the Windows key instead of the alt key for moving between workspaces, and that's fine.  I would highly recommend using the Shift variant for moving windows across workspaces.
