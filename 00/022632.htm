Function Distance(lat1, lon1, lat2, lon2, Unit)
  Dim theta, dist
  theta = lon1 - lon2
  dist = Sin(deg2rad(lat1)) * Sin(deg2rad(lat2)) + Cos(deg2rad(lat1)) * Cos(deg2rad(lat2)) * Cos(deg2rad(theta))
  dist = acos(dist)

  Select Case UCase(Unit)

  End Select
End Function

Function acos(Rad)
  If Abs(Rad) <> 1 Then
    acos = PI / 2 - Atn(Rad / Sqr(1 - Rad * Rad))
  ElseIf Rad = -1 Then
    acos = PI
  End If
End Function

'  This function converts decimal degrees to radians
Function deg2rad(Deg)
    deg2rad = CDbl(Deg * PI / 180)
End Function

Function rad2deg(Rad)

tom ogilvy say:

toptom2001

You can write range references as relative, absolute or a mixture.   If I want to reference A1 and then have it change to A2 as I drag the reference down the column, and have the column letter change if I drag it right across a row, then I just use   A1.   Both the column reference and the row reference are relative. 

If I want to fix the column reference, then I would use  $A1

If I want to fix just the row reference, then I would use  A$1

If I want to fix both references then I would write  $A$1

for your formula/situation, you need to make the lookup range absolute

=VLOOKUP($F3;Sheet2!$A$1:$B$72;2;FALSE)

I also made column F absolute in the first argument but it would not be affected either way if you just copied down a column.

drgdc say:
As others have noted the Windows Phone 8007000E error is because of an error with Windows Phone not being able to handle Gmail contacts. Contact photos have specifically been reported to trigger the error. International characters may cause problems as well.

I was able to resolve my instance of the error by taking a rather drastic action. I exported all of my Gmail contacts to a file, deleted all contacts from Gmail, then imported the contacts file. The exported file, being a CSV file, did not include the offending contact photo(s). A less drastic action would be to identify large contact photos and delete/replace the photos.

If you choose this option be sure you select all of your contact upon export and verify you have all contacts in the exported file before deleting Gmail contacts.

My export included all of my Labels and after the import Gmail continued to use linked contact photos (e.g. contact's Google profile photos).

Exporting Gmail contacts, https://support.google.com/mail/answer/24911?hl=en
Import contacts as a CSV file, https://support.google.com/mail/answer/14024?hl=en
