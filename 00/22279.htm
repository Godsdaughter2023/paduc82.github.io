sheen valentine ali msft say:

Hi DellT,

You can consider using the query below to search:
ManagedPropertyName:"searchParameter or Alias:"searchParameter"

This will provide you with only the exact match of the search parameter.

The Alias is optional, but if your managed property name long and/or is hard to remember, you can give  it an Alias to remember and search it more easily. To add the Alias, you can go to the SharePoint Admin Center > Search > Manage Search Schema > search for your managed property > Edit > Alias > enter the Alias.

Note: You can enter the same Alias as your column name.

Here's an example; I have created two documents, one with the name Document A, one with the name important document.

I've given the managed metadata column named "Alias" an Alias "alternateName". Document A is given the alternate name of important document, while the important document is given the alternate name of document A1.

When searching "important document" without the query above, I got two results, the document named important document, and the document named Document A with the alternate name important document.

When searching with the query "alternateName:"important document"", I got only one result, the document named Document A that has the alternate name important document.

Regards,

Sheen
