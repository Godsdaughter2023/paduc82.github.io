    davidio143 say:

    Calculate sum in new column, when value in another column changes

    I have a large worksheet of data I'm using as a lookup table, and that I will want to copy/paste data into from another worksheet.  On the separate worksheet, I have the following data that I need to calculate a total on in a new column.  Example:

    A (Land Identifier)         B (Crop)           C (Allotment)         D (new column w/ sum)

    11                                     Corn                 14.2                         34.2

    11                                     Soybeans        20.0                         34.2

    22                                     Corn                 75.0                         75.0

    31                                     Corn                 50.0                         70.0

    31                                     Wheat              20.0                         70.0

     

    I need to compute the sum of the allotment(s) with each change in land identifier in Column A.  Some identifiers in Column A will have only 1 allotment, some will have 2 or more.  I will use a vlookup to transfer the data from this separate worksheet to my main lookup table.  It will serve my purposes if the sum is repeated in Column D for each identifier in Column A, as I eliminate duplicates as part of a macro after copying to a separate worksheet.  Is there a formula that will be able to identify when to sum the values in Column C when the value in Column A changes?  I hope this makes sense - thanks for any help that can be offered!

    jllatham say:

    Put this formula into D2

    =SUMIFS(C:C,A:A,A2)

    and fill it down to the end of your list.  It will adjust each time you change a value in column A on the sheet.

    This formula will not work in Excel 2003 and earlier versions of Excel.

    An equivalent for all versions of Excel would be:

    =SUMPRODUCT(--($A$2:$A$1000=A2),--($C$2:$C$1000))

    Excel 2003 and earlier don't allow you to reference the entire column, so your formula has to include row numbers with a range large enough to include all your used rows, and the SUMIFS() function does not exist in pre-2007 versions of Excel.

    • =IFERROR(VLOOKUP([Site ID],Main_Frame_Table,Main_Frame_Table[CBPS INCLUDED IN PCM PRICE],FALSE),"") - This formula doesn't return any data with the table column reference put in.

    jeeped say:

    A similar method to the one proposed by Jean-Yves Tfelt would be using the table's headers which define the table's columns.

    =VLOOKUP(D7,Table1,COLUMN(Table1[Birthday]),FALSE)

    If you have a column with the header as Birthday then the above will retrieve that columns value from a match in the first column. Inserting a column between the first and Birthday will not make a difference.
