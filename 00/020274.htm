macropod say:

A mailmerge main document can be set up to generate a single letter, if that's what you prefer.

Basically, you'd set the letter up for a normal letter merge. Having done that, you'd then insert a field coded as:

{SKIPIF {MERGEFIELD Account_Code} <> {FILLIN "Account Code" \o}}

at the top of the letter, where Account_Code is the name of your Account Code field in the data source.

With the above setup, you'll be prompted for the Account Code whenever you execute (not preview) the merge and a single letter will be generated.

Note: The field brace pairs (i.e. '{ }') for the above example are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues.

Alternatively, a macro could be used to populate a dropdown content control from the data in your Excel workbook, with other content controls, etc. in the document being filled in according to the selection made. This is a lot more work to set up, though.

Typically, when changing the defaults fails, the underlying cause is some add-in that is preventing Word from saving the Normal template properly. That problem must be troubleshot and fixed separately.

As a workaround, open the Normal template as a document, change the defaults in the template, save and close. Restart Word.

The quickest way to open Normal is by using a line of VBA code as follows: Press Alt+F11 to display the Visual Basic Editor. Press Ctrl+G to activate the Immediate window (or click View | Immediate Window). Type

NormalTemplate.OpenAsDocument

and press Enter. Normal opens in a separate window. Close the Visual Basic Editor, change the document defaults, save and close.

Here's how you can change the document default: In the Styles pane (Ctrl+Alt+Shift+S), click the Manage Styles icon; on the Set Defaults tab, make the desired changes. Since you are editing the Normal template directly, leave the "Only in this document" radio button selected.

Function GetFolder(strInitialPath As String) As String
    Dim fldr As FileDialog
    Dim strItem As String, strCurDir As String
   
    strInitialPath = Replace(strInitialPath, "/", "\")
    If Right$(strInitialPath, 1) <> "\" Then strInitialPath = strInitialPath & "\"
   
    strCurDir = CurDir  'Store current directory
   
    Set fldr = Application.FileDialog(msoFileDialogFolderPicker)
    With fldr
        .Title = "Select a Folder"
        .AllowMultiSelect = False
        .InitialFileName = strInitialPath
       
        If .Show <> -1 Then
            strItem = strInitialPath
        Else
            strItem = .SelectedItems(1)
        End If
    End With
   
     'Reset directory before exit
    ChDrive strCurDir    ' Return to the original drive
    ChDir strCurDir      ' Resets directory for original drive
   
    If Right$(strItem, 1) <> "\" Then strItem = strItem & "\"
    GetFolder = strItem
    Set fldr = Nothing
End Function
