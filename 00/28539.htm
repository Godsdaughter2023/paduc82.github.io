steve rindsberg say:
These are all network printers, is that correct?

PowerPoint used to be sort of notorious for being cranky about network printers, or more exactly, about printing to network-installed printer drivers.   In other words:

Printer *driver* installed locally, even if printing to a network printer:  No problems.

Using printer driver that's installed on a network-attached PC, not the local computer:  GrumpyPPT.

Check to see if there are any differences in how the printers are installed.  For example, if the PCs where it's working have locally installed drivers and the problem PC doesn't, that could point to the problem.

Or conversely, if the problem PC has a locally installed driver and the others are using network-installed drivers, then reinstalling the driver on the network wouldn't change the driver on the PC with the locally installed driver.  

Also, check on whether the problem PC vs the others have different default printer drivers set.  That might, in some bizarre, convoluted way, affect things.  

Have a look here for more info and suggestions on how to tell where the driver's installed:

How to install a local printer driver
http://www.pptfaq.com/FAQ00605_How_to_install_a_local_printer_driver.htm

philipradler say:

Here's a long shot you could try. Go to File>Print>Printer Properties and click on the "Job Storage" tab (if there is one for the printer in question). Two settings there--"Personal Job" and "Stored Job"--will send the file to the printer, but the actual print command won't execute until the user triggers it at the printer control panel. I believe the "Personal Job" setting allows for setting a 4-digit personal ID number that will then be required to release the file, which will then be printed and immediately deleted from the printer's memory (that's good for protecting sensitive, private information). I think the second setting ("Stored Job") is designed to let the printer retain commonly used documents (such as forms) so anyone can readily print them out directly from the printer.

 

If someone had recently used either of those settings on a print job sent from User A's machine, that setting may have been retained in the local machine's PowerPoint installation. It's an unlikely cause, but probably worth checking since other diagnostic paths haven't yet cleared up the problem.

 

Since the exact same file will print from other machines, it definitely sounds like the file itself is OK, which in turn suggests that either (a) there's a problem with the printer driver on the A machine, as suggested by the highly skilled Mr. Rindsberg, MVP, or (b) there's a problem with a local setting on the A machine, as posited here.

martinwyatt say:

Hi again, it looks like I have solved my own problem.

The culprit is once again the network driver installed by Windows 10.

There is an updated driver on the Realtek site that I had downloaded and installed in previous builds.

The win 10 17751.1 and 17754.1 installer overrides this updated driver and installs an older version. This older driver causes the build to crash at startup. You must get to the control panel immediately after the upgrade (before any subsequent restarts) and update the driver for the Realtek network adaptor on the motherboard with the one supplied by Realtek, not the one supplied by Microsoft.

Hope this fix helps others,

Martin.

inserting a multi-page Word document as object into Outlook - Microsoft Community

robin ann say:

inserting a multi-page Word document as object into Outlook

I am attempting to insert a multi-page Word document into an Outlook email message, however it only inserts the first page. Would any know why this is happening? I am selecting Insert>Object>Create from File and then selecting the Word document.

Thanks.

graham mayor say:

The basic issues are twofold. (1) Objects cannot be multipaged; and (2) Word document format and HTML message formats are not the same and the differences between them will conspire against maintaining the original format. For example HTML format does not support pages, or headers and footers.

Chances are that marketing e-mails are not created as Word documents, but generated as html format.

To get a better idea of the issues involved, open the document in Word and view it in Word's web view.

You may have better success formatting the source document using tables.

Even if you get the document into the message body with similar formatting, you have no control whatsoever over how it is viewed by the recipient. Many users (especially business users) will have their mail clients set to show messages in plain text because of security considerations, and then all the effort is lost and the document may look even worse that way.
