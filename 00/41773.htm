Hello,

The problem seems to be because, your user profile is on a different drive of your system. I suppose that during the update, the system can't find the path of the profile. Surely, a different drive letter is attributed.

The solution for me:

Create temporaly a new user account whose profile is on the same drive as the system.
Restart the update from this account, it should work.

Maybe move your profile is enough. https://support.microsoft.com/fr-fr/kb/979463

If you have changed the registry key that indicates the path of user profiles, return it to its default value.

HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows NT \ CurrentVersion \ ProfileList
Change the key "ProfilesDirectory" and specify "% SystemDrive% \ Users".

rb.levow say:

My experience was that I need to have all profiles on the system drive, or at least make them appear to be there.  If you haven't already done it, the simplest is to create a junction (hard link) from C:\Users to the folder with all your profiles or from C:\Users\UserId to the individual profile.  This assumes C: is the system drive and UserId is some user id.  Then you need to edit the entries in the ProfileList key so that each one has the explicit path C:\Users\UserId.  My experience was that using %SystemDrive% to specify the drive produced many more new errors.

In any case, my thanks to Adrien for pointing me in the right direction after several weeks trying a number of other suggested but useless approaches.
