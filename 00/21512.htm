scottgem say:

But what is the relationship between Customer and Checkout? Can a customer checkout multiple costumes? 

it appears you have 2 many to many relationships. A customer can attend multiple events and an event can have multiple customers. A Customer can checkout multiple costumes for an event. So I would add 2 more tables:

tjxCustEvents

CustEventID (PK autonumber)

CustomerID (FK)

EventID (FK)

tjxCustCostume

CustCostumeID (PK autonumber)

CustomerID (FK)

EventID (FK)

CostumeID (FK)

I would create a mainform bound to your events table with 2 subforms bound to the two junction tables I suggested. So the main form would show the event info. The two subforms would be linked to the main form on EventID. The CustEvents subform would display info about the customer (see my blog on Displaying data from related tables) and the CustCostume subform would display info about the costume.

scottgem say:

I did not think about putting a subform in a subform.

There is a problem with nesting subforms. You can't nest a continuous form subform within another continuous form subform. 

If you need a customer centric view of this, then that is your best choice. However, you might consider using a report for that instead, if you don't need to do entry. A Report, grouped by Customer, then Event may work better. 

As for data entry, I would take a more Event centric view as I described. 

ken sheridan say: The alternative to nested subforms is to use correlated subforms.  This allows you to have both subforms in continuous forms view.  As you navigate from row to row in the first subform the second subform is requeried to show only those rows which reference the currently selected row in the first subform.  This is done by basing the second subform on a query which references a control bound to the key of the first subform as a parameter.

You'll find simple examples of both nested and correlated subforms, using data from Northwind, in CorrlatedSubs.zip in my public databases folder at:

In OneDrive for Business, we can see the shared folder via “Shared with Me”. However, it’s not feasible to sync the Shared with me folder to our local folder via OneDrive for Business sync client. As in OneDrive for Business, the “Shared with Me” folder is just a view not a real folder, it is used to view the files or folders which other users in the same organization have shared with us.
