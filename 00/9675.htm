I have been trying for the last few days to get my son's 6230i to connect to the PC. I too have a Bluesoleil bluetooth adaptor which recognises the phone but the NOKIA PC suite didn't. After reading around the web I have found the problem and then the solution - it all has to do with the bluetooth stack included with Windows XP SP2 which is not overwritten when you install the Bluesoleil drivers. You have to rename the original file and then it will get overwritten. To do this:

1. Click Start, click Run, type %windir%\inf in the Open box, and then click OK.
2. Right-click Bth.inf, and then click Rename.
3. Rename Bth.inf to Bth.bak, and then press ENTER.
4. Install the Bluesoleil driver.

I downloaded the up-to-date driver from Bluesoleil's website (not sure if I had to, but I did). This is the "BlueSoleil_1.6.1.4_release_050606.zip" on the support/downloads section of the website. barb way say:

Print Preview versus actual Print output is something to compare.  Some drivers do work better than others.

However, introducing conversions is almost always going to introduce fidelty loss.  Especially with JPG, which is a compressed format by default. If the JPG from AutoCAD looks good when you zoom in using a viewer, and also looks good when you zoom in with Visio, then try checking in the Printer options for a way to retain the graphic fidelity, perhaps.

  Also, in Visio, there is a Picture Compression function that you can check, as it might be automatically compressing the JPG imports.  Select one of the images in the drawing, and the Picture Tools ribbon should appear.  Select the Format tab, and then click on the expander at the bottom right of the Adjust group on the left to open the Format Picture dialog.  Select the Compression tab in the dialog, and make sure that "Compress pictures" is Unchecked, and also make sure that the "Change Resolution" option is set to "No Change", and click OK to save.  You will want to do this to each picture if it helps your print output, as it is saved for each item and there is no blanket "Change all pictures".   (If you have multiple pictures on the same drawing page, you can multi-select them and apply the changes to them all at once.  This is not possible for images on separate pages).

Having said all of that, I am concerned that you are seeing so many issues in trying to save the CAD drawing from AutoCAD 2005 to use in Visio 2010.  You should be able to import and convert CAD files saved from AutoCAD 2005.  When you say they 'disappear', that is a problem.  However, I have seen the CAD file relocate after importing/conversion. 

Try these steps with one of your simpler drawings:

1. Open a blank drawing in the format you want to use (scaled page, standard letter size, whatever you like)

2. Go to the Insert tab, and select the CAD Drawing option

3. Browse to your DWG file and select it, click Open

4. In the CAD Drawing Properties dialog, you can see the suggested scale that Visio will import with, you can change this later as well - but go ahead and set it as desired. Click OK to insert in your drawing.

5. By default, the drawing object should be centered in the drawing view.  If it doesn't appear to be present, try zooming out in the drawing using the Zoom Out option on the bottom bar.

 - TIP: One way to see if you are zoomed out as far as your largest object is to use Select All (Ctrl-A) to select every object on the drawing page.  Then keep zooming out until you can see all of the highlighted objects.

I don't know if this is the issue, but it's worth a try.  If you can figure out the import method for your CAD DWG files, you will have a much better print output.

By the way - are you creating files in the actual AutoCAD 2005 application, or in another application that creates DWG files in the 2005 format?  You can also try saving the files as "DXF" instead of "DWG", and see if you can import those as a work around. Visio supports up to AutoCAD 2007 format for both of these files, as far as I know.
