.\BulkUploadSharePointCSOM.ps1 -UserName "<UserAccountName>" -Password "<Password>" -SiteURL "<SiteCollectionURL>" -DocLibName "<LibraryName>" -Folder "<TheFilePath>" -O365 $true

rohn007 say:

Sorry, there is no easy way to automatically generate a TOC that looks like that.  We had a similar question a few weeks ago.

The closest I can suggest is a very manual process.

I assume that you are using Heading styles in the body of the document

I assume you are using auto numbering on the headings.

With those assumptions you can create a (very) manual TOC.

Insert a 3 column table in the right place.

Then you need to insert 3 cross references for each line, each "For which heading"

#1 is the "Heading Number"

#2 is the "Heading Text"

#3 is the "Page Number".

***********************************

Use <ALT><F9> shortcut to display the field codes

(hint: use the shortcut to revert the field codes back to values)

Bullet #1 is the Cross Reference command.

               The command displays the "Cross-reference" dialog

Bullet #2 shows the field code for the Heading Number

Bullet #3 shows the field code for the Heading Text

Bullet #4 shows the field code for the page number

You can speed up the process a little:

  • by inserting the middle column first.
  • copy the whole column,
  • paste in the first column
  • paste in the third column
  • change the field code to PageRef'
  • go back to first column, add the \r  parameter

Does anyone know what field to use in order to reference an endnote - Microsoft Community

lewis d. eigen say:

Does anyone know what field to use in order to reference an endnote number?

I write many academic papers where I use endnotes for citations of authority for different passages of text.  Normally, I have no difficulty.  However, the situation sometimes arises where in the text, I want to cite the SAME citation twice or even more frequently.  If I just create another endnote for the second citation, then there will be TWO citations for the same author/document in the endnotes.  This is considered bad form.

What I would like to do is enter a field as the second reference to a citation.  In the exact same place as I would enter a new endnote.  I could create a bookmark for the first endnote citation that is to be used for the second time.  If the field I insert could reference the bookmark, the same endnote number of the Bookmarked endnote would appear in the field, and I would have a second reference to the bookmarked endnote citation. 

So what field should I use for this, and should I bookmark just the Endnote number or the entire endnote?

Or is there a better way to handle this problem???

I have tried the Noteref field and the Ref field and neither of them works.

suzanne s. barnhill say:

A NoteRef field is not intended to take the place of a footnote or endnote; it is meant to reference that footnote or endnote in the document body or in another note. In traditional endnotes, you reference a previously cited source in a specific way depending on the style manual you are following. This might be by Ibid. and Op. Cit. (in the latter case, you can use a NoteRef to indicate the note in which the work was cited) or by the author name and short title, or some other method.

But it sounds like what you actually need is a numbered reference list. Using this method, you list all your sources in alphabetical order, with numbers, and reference them using the numbers. For instructions on how to do this, see https://wordmvp.com/faqs/formatting/FootnoteFAQContent.htm#AlphaEndnotes.

a. user say:

IMO the reply by SarahWilliams95 should do the trick, but if you prefer to insert your own fields, a NOTEREF field should do it.

You would need to

 a. select the Endnote reference number *in the body of the text* (not the endnote reference number in the Endnote area),

 b. insert a bookmark (say "myendnoteref"),

 c. click where you want the second reference to the same endnote, and insert a field such as { NOTEREF myendnoteref } (where the {} are the special field code brace pairs you can insert using ctrl-F9 on Windows Word.

If you use the cross-reference dialog box as SarahWilliams95 suggests, Word should create a suitable invisible bookmark and the appropriate NOTEREF field.
