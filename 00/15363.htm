john korchok say:

Editing embedded or linked content relies on OLE. OLE display is resolution-dependent. This is why you'll find a bazillion questions along the lines of "I changed my monitor and now I see a different number of columns in an embedded Excel sheet".

When you're running a system with multiple resolutions, OLE doesn't know which resolution to use. But it automatically falls back to a workaround: instead of editing the file in place (the normal consequence of double-clicking on an embedded selection), Office opens Excel and allows you to change your selection there so you can repaste it into PowerPoint.

In PowerPoint, you can click on File>Feedback, click the Send a Frown button. And describe the problem in detail, including all steps to replicate the issue. Submitting will send the report to Microsoft programmers. However, since they've already programmed a workaround, it seems likely they're already aware of this.

A readme file might help an individual understand how a folder and files got there, the digital signing is the only way for the operating system and security applications to check this programmatically, since this is a standard method to detect, display and verify (via the file's signature), both the developer as well as the integrity (not corrupted or modified) status of a file.

Introduction to Code Signing

Though not every file needs to be signed and even some Microsoft executables sometimes aren't, it's generally common to see the key executables for antimalware and other important security or key operating system files such as Explorer.exe for example which include them.  In this case, the very fact that the additional modules are stored in another folder and unidentified is what both bothered you personally and makes the act of signing them even more important.

rob koch say:

What's causing the confusion here is the common practice in security forums of concentrating on the registry cleaning functions of such questionable programs, when in reality it's the complete program and the often shoddy development practices of the companies which create them that are the larger issue.

In truth, Windows itself has contained most of the functions just itemized which provide true value for decades, Microsoft either has made these operations automatic and invisible or placed them in management interfaces that few typical PC owners learn how to use.

By bringing these "utilities" together in a single GUI and making them apparent to the non-technical user, some products easily convince the less knowledgeable person that they provide some added value.  Most technically knowledgeable users completely ignore such processes until we really need them and then simply use those built into Windows in most cases.  Only the uber-geeks seem to care to use a handful of well known 3rd-party products, in most cases because they feel the products do a better job, most often again because they provide a more visible representation of what is supposedly happening.

The choice in such cases is between the supposedly more reputable products and those of less repute, but since in most cases this is splitting hairs I personally feel the entire genre of maintenance utilities are simply snake oil.  If people wish to pay for something that Windows itself does for free, then more power to them for having the money to waste.  For myself, I'll keep my money and let Windows take care of this maintenance or look up the tool built into Windows if I ever really need it.

If you desire guidance to some of the Windows tools which provide the more important functions or the more reputable 3rd-party products which provide a more exposed interface, I'm sure JulietNE will have some recommendations.

Obviously this is my own opinion, so the decision of how to proceed on your own PC is entirely up to you.

As an aside to this question, have you examined the executable(s) within the main SparkTrust PC Cleaner application folder to see whether these follow the best practice of digital signing?  Beyond the reputation questions raised by JulietNE, if this company hasn't done something this basic then I question the true value of this application to provide security related functions like Privacy Protection and we already know that the vendor providing the Malware Protection has both ignored digital signing and has a questionable reputation as well.
