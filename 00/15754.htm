Sub Test()
  Dim Ws As Worksheet
  Dim Where As Range, This As Range
  For Each Ws In Worksheets
    Set Where = SpecialCells(Ws.UsedRange, xlCellTypeConstants, xlNumbers)
    If Not Where Is Nothing Then
      For Each This In Where
        If This < 0 Then This = Abs(This)
      Next
    End If
  Next
End Sub

Private Function SpecialCells(ByVal r As Range, ByVal Typ As XlCellType, _
    Optional ByVal Value As XlSpecialCellsValue = &H17) As Range
  'Avoid the SpecialCells-BUG to return all cells from the current region
  On Error Resume Next
  Select Case Typ
    Case xlCellTypeConstants, xlCellTypeFormulas
      Set SpecialCells = Intersect(r, r.SpecialCells(Typ, Value))
    Case xlCellTypeConstants Or xlCellTypeFormulas
      'Special feature: Return all used cells
      Set SpecialCells = Intersect(r, r.SpecialCells(xlCellTypeConstants, Value))
      If SpecialCells Is Nothing Then
        Set SpecialCells = Intersect(r, r.SpecialCells(xlCellTypeFormulas, Value))
      Else
        Set SpecialCells = Union(SpecialCells, Intersect(r, r.SpecialCells(xlCellTypeFormulas, Value)))
      End If
    Case Else
      Set SpecialCells = Intersect(r, r.SpecialCells(Typ))
  End Select
End Function

Sub ConvertNegtoPos()
    Dim Ws As Worksheet
    Dim FormulaCells As Range, ConstantCells As Range, Rng As Range, Cell As Range
      For Each Ws In Worksheets
        On Error Resume Next
            Set FormulaCells = Ws.Cells.SpecialCells(xlFormulas, xlNumbers)
            Set ConstantCells = Ws.Cells.SpecialCells(xlConstants, xlNumbers)
        On Error GoTo 0
        Select Case True
            Case FormulaCells Is Nothing And ConstantCells Is Nothing: Resume Next
            Case Not FormulaCells Is Nothing And ConstantCells Is Nothing: Set Rng = FormulaCells
            Case FormulaCells Is Nothing And Not ConstantCells Is Nothing: Set Rng = ConstantCells
            Case Not FormulaCells Is Nothing And Not ConstantCells Is Nothing: Set Rng = Union(FormulaCells, ConstantCells)
        End Select
        For Each Cell In Rng
            If Cell < 0 Then Cell = -Cell
        Next Cell
    Next Ws
End Sub

Basically, a "character" in text is actually a number: an integer that says which character it is.

However: there are two ways of encoding Chinese characters.  The current way is to encode the characters in Unicode (16-bit characters).  Unicode will describe any character known: 64,000 of them.

The old way was to specify a Chinese font and a Chinese encoding.  You needed to get both correct or the result was garbage, because within the encoding, any given number could represent several different characters, depending on the font in use.

jeeped say:
Your description of the circumstances and error message leads to believe that you have a Worksheet_SelectionChange event macro associated with that particular worksheet. Right-click the worksheet's name tab at the bottom of the Excel application window and choose View Code. Is there a VBA code procedure in the VBE's worksheet pane?

If macros are not important to the day-to-day operation of the worksheet, consider saving the workbook as an .XLSX. Code with only exist in a .XLSM (macro enabled workbook).

If macros are necessary, post back with a description of the worksheet, its operation and a copy of the Worksheet_SelectionChange procedure and it may be able to be reworked with error control that will halt the error messages you are receiving.

The offending code may actually reside in the VBE's ThisWorkbook as Workbook_SheetSelectionChange but it is more likely that is is localized to the single worksheet you are describing the problems with.

Finally, consider uploading a redacted a copy of your workbookto a public access file area like SkyDrive, DropBox or Google Drive and post a link to the publicly shared file back here. A firsthand look at the layout and any incorporated VBA would help immensely.
