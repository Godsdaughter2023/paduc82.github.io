Windows 7 disk defrag : What determines number of passes - Microsoft Community

charles g say:

Windows 7 disk defrag : What determines number of passes

rohit siddegowda say:

Hi Charles,

 

Disk defragmentation is the process of consolidating fragmented data on a volume (such as a hard disk or a storage device) so it will work more efficiently. During defragmentation, system performance will be degraded, and it is best to leave the computer alone during the process so that the defragmenter does not get confused by unexpected changes to the file system. Depending on the algorithm used it may or may not be advantageous to perform multiple passes.

What is disk defragmentation?

charles g say: You didn't really answer my question.....

bossdweeb say: I can't comment on the specifics of each pass but -
The number of passes is determined by (at least) two things other than capacity, and percentage
of that capacity in use on a HDD.
- The number of file fragments, and how poorly the HDD is optimised.
Optimising the HDD is what takes more time, and probably is where the number of
passes would be increased.
Optimising is the 're-arranging' of the HDD so all the files for the various software are placed
contiguously on the HDD  (as mcuh as possible).
This makes reading the program flies quicker.

The Win7 defragmenter apparently (I don't use it) optimises automatically.
 Other de-fragmenting software  (such as  AusLogics) sometimes have an option to Optimise,
or just do a 'basic' defrag.
In the case of a 'basic' defrag there is some optimising being done while fragments are being deleted, but you have to select the Optimise option to get a fully optimised HDD.

- If you shut down your PC when it's not in use, it could be difficult to arrange a time for an effective automatic defrag of the HDD, as I believe it only defrags when the system is idle if you are using an automatic defrag option.
For this reason a time in the early morning hours is commonly selected to run the defragmenter automatically.
- Also, as Rohit mentioned, it is best to avoid interfering with a defrag, so the best way to avoid
possible corrupt files it is really best to manually defrag if you don't leave the machine on over night,
or allow it to Sleep/Hibernate at other times,  so you can give the system over to the  defrag process.

Personally,  I run AusLogics  defragmenter manually after installing a new software or Windows Updates, and also run a manual defrag once every week or two.
The more frequent the defrag the less time it takes, regardless of which defragment software you use.

The Xbox 360 emulated games have a strange way to save. 

When you end a game session with a Xbox 360 backwards compatible game, make sure you really quit the game by going to the Home Screen, and then pressing the menu button on the running Xbox 360 game, and choose Quit from the menu.

Recover Devices in Windows 10 - Microsoft Community

publiccus say:

Recover Devices in Windows 10

Devices can fail, or become lost or stolen. In such cases, you should plan on a strategy to replace the device and recover your data. You should ensure that you are protecting user data, which in most cases means that you are storing it somewhere other than on the device.

When you need to recover the device, you should first attempt to use a nondestructive and fast recovery method, if it is suitable for the situation. For example, if your device cannot start, you should first start the Startup Recovery option from the recovery environment before considering the use of Reset this PC or System Image Recovery.

The latter options could also resolve the issue, but could cause some data loss and would take considerably longer. The order in which you should use device recovery methods depends on the specific situation, but as a rule of thumb, you should consider them in the following order:

  1. Startup Repair. If the hardware is functional, but the device does not start, you should consider the Startup Repair option from the recovery environment. This option is quite fast, and will automatically detect and fix most common startup issues, while leaving all the user data intact. Device can restart several times during the process.

  2. Driver Roll Back. If an updated device driver causes an issue, Driver Roll Back is the best option. You can access this option from Device Manager, whether you are in normal mode in Windows 10 or in safe mode. This option leaves the data intact. It can only fix issues related to device drivers.

  3. Safe Mode. This advanced startup mode starts Windows 10 by starting only basic services and using basic device drivers. You can use it to replace missing or damaged system files manually or to perform diagnostics and configuration changes that are not possible when Windows 10 is running in normal mode. You can also use System Restore and Driver Roll Back from safe mode. You can also use safe mode by selecting Safe Mode with Networking or Safe Mode with Command Prompt.

  4. System Restore. If System Protection is enabled and it has created restore points, you can use System Restore to revert system settings to an earlier restore point. You can use System Restore from Windows 10 running in the normal mode, from safe mode, or from the recovery environment. This operation is nondestructive, because it leaves user data intact.

  5. Command Prompt. This advanced startup option is not automated and it is suitable for experienced users. You can use it to perform diagnostics, which is not possible when the system is running. For example, you can use the command prompt to scan for rootkits, replace damaged system files, change the state of the services, and run third-party apps. This option is generally not destructive, but it could be, depending on your actions.

  6. Reset this PC. If you select this option from the recovery environment, keep in mind that it will remove apps that are not part of Windows 10, and reinstall the operating system. This option is faster than it used to be, but it still takes some time. Based on the options that you select, Reset this PC might also remove user data on the device during the reset process.

  7. System Image Recovery. You can perform system image recovery only if you already have the system image. This recovery process takes time and replaces all the data on the device with the system image content. Files that you created or modified since you created the system image will not be available in the system image.
