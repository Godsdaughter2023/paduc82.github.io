john - project say:

New Guy 74,

First of all, there are no stupid questions and, as it turns out, your question has been asked by various users over the years.

My take on the issue is this. In a good Project plan each and every task should be unique. There are no duplicates. Furthermore, summary lines are not tasks at all (even though they are called "summary tasks"). They simply categorize or summarize the activities (subtasks) under them. Those activities should be expressed in action verb format describing something that is accomplished using one or more resources. I like to refer to subtasks as "performance tasks" since the assigned resources perform work to complete the task.

In your particular case the "promote ourselves..." sounds more like a summary line that should contain specific development of articles for various media. For example, the first subtask might be "Write article for local newspaper". Another subtask might be "Create Facebook page". Another, "Send out weekly tweets". Those are performance tasks that can be assigned to a resource and can be measured to know when they are done.

By organizing the plan in that fashion, you might find that "customer relations" are automatically addressed by the above performance tasks but if there are other specific activities you could perform for your customer, then those can be shown under a separate customer relations summary.

But to answer your original question, yes there are ways to "mirror" tasks within a Project file but they are not necessarily straightforward.

Hope this helps.

jeeped say:

I agree that the gathering of the data (e.g. concatenation) would be best served as a UDF, but a helper column to store the result will make parsing the results a lot simpler as well. Consider,

 

The UDF used in G2 is,

Public Function udfMyGather(ByVal target As Range) As String
    Dim strConcat As String
    Dim c As Variant
    strConcat = ""
    For Each c In target
        strConcat = strConcat & "," & c.Value
    Next
    udfMyGather = Right(strConcat, Len(strConcat) - 1)
End Function

Usage: =udfMyGather(<range to gather>)   example: =udfMyGather(A2:E2)

... which can be Copied or Filled Down as needed. The formula in F2 is,

=COUNTIF(G$2:G2,G2)

... which uses both absolute and relative addressing to accomplish a progressive count.

Here is a workaround. Use next/previous records. This mimics the default mouse scroll a little better until Microsoft comes up with a fix.

Private Sub Form_MouseWheel(ByVal Page As Boolean, ByVal Count As Long)
On Error GoTo errorhandle
If Count < 0 Then
DoCmd.GoToRecord record:=acPrevious
DoCmd.GoToRecord record:=acPrevious
Else
DoCmd.GoToRecord record:=acNext
DoCmd.GoToRecord record:=acNext
End If
errorhandle:
Exit Sub
End Sub

andreas killer say:
The zoom state is sheet dependent, means you have to set the zoom when a) a file is opened and b) when a new file is created and c) when a sheet is activated.
So you have to create event routines for each of this events.

Replace the code in the code module ThisWorkbook of you PERSONAL.XLSB with the code below, save it, close and reopen Excel.

Andreas.

Option Explicit

Dim WithEvents App As Application

Private Sub App_NewWorkbook(ByVal Wb As Workbook)
  SetZoom
End Sub

Private Sub App_SheetActivate(ByVal Sh As Object)
  SetZoom
End Sub

Private Sub App_WorkbookOpen(ByVal Wb As Workbook)
  SetZoom
End Sub

Private Sub Workbook_Open()
  Set App = Application
End Sub

Private Sub SetZoom()
  On Error Resume Next
  ActiveWindow.Zoom = 91
End Sub
