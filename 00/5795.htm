ken sheridan say: You'll find a solution for this in CustomNumber.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes a form for numbering by group, gender in the demo.  The method employed is an extension of that published by Roger Carlson.  The next number per group is obtained in the gender combo box's AfterUpdate event procedure:

Private Sub cboGender_AfterUpdate()

    If Not IsNull(Me.cboGender) Then
        Me.NameID = GetNameID(Me.cboGender, Me.txtStartSequence)
    End If
    
End Sub

This calls the following function in the form's module:

Private Function GetNameID(strGender As String, intStartSequence As Integer)

    strCriteria = "Gender = """ & strGender & """"
    
    GetNameID = Nz(DMax("NameID", "NamesByGender", strCriteria), intStartSequence - 1) + 1
    
End Function

In the event of a conflict in a multi-user environment, if two or more users are inserting a new row simultaneously, the error is handled in the form's Error event procedure with:

Private Sub Form_Error(DataErr As Integer, Response As Integer)

    Response = IncrementField(DataErr)
    
Exit_Form_Error:
    Exit Sub

Err_Form_Error:
    MsgBox Err.Description
    Resume Exit_Form_Error
   
End Sub

This calls the following function in the form's module to increment the value by 1 and sert the retun value of the Response argument:

Private Function IncrementField(DataErr)

    Const KEYVIOLATION = 3022
    
    If DataErr = KEYVIOLATION Then
        Me!NameID = DMax("NameID", "NamesByGender", strCriteria) + 1
        IncrementField = acDataErrContinue
    Else
        IncrementField = acDataErrDisplay
    End If

End Function

PS:  Do not store the group value and number in a single column, but in two separate column's.  You can then concatenate them into the structured single value in a computed column in a query or a computed control in a form or report with and expression like this:

[Group] & Format([SeqNumber],"000")
