jay freedman say:

It's possible to assign a shortcut directly to the View > NavPane command, using the Customize Keyboard dialog (File > Options > Customize Ribbon > Keyboard shortcuts: Customize, choose the View Tab category and the NavPane command).

Alternatively, you can install the following macro (see http://www.gmayor.com/installing_macro.htm if needed) and assign a keyboard shortcut to it:

Sub ToggleNavPane()
    With Application.TaskPanes(wdTaskPaneNav)
        .Visible = Not .Visible
    End With
End Sub

In either case, Word has a quirk: If you open the pane with Ctrl+F and the search box still contains the search text, then the NavPane command doesn't do anything; you have to press Esc first to clear the search box, and then press your shortcut.

It appears that the most reliable keyboard-only method is to press and release Alt and then W to activate the View tab on the ribbon, and then press K to toggle the Navigation pane check box.

brian tillman [outlook mvp 2007-2019] say:

You cannot have a single set of folders when using IMAP accounts.  Each IMAP account MUST have its own folder set.  IMAP accounts keep their data on the server and the folders set you see will be a local cache of the server-side folders.  Since each IMAP server can have its own distinct folder set, so must each local cache be distinct.  Also, since IMAP accounts handle only mail items - no contacts, no calendar, no tasks, etc., Outlook will require a third, default folder set where it will keep the Calendar, Contacts, Tasks, Notes, etc.  So, with two IMAP accounts, the minimum number of folder sets you can have is three.  POP accounts can share folder sets, but IMAP accounts can't.
