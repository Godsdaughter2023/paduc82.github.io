frederik long say:

The command still works in Windows 7 but because you spawned a new shell when running it under elevated privileges, the output file ended up in the System32 folder. You must therefore be more specific when specifying the output file location, e.g. like so:

regedit.exe /e d:\myExportFile.reg HKEY_CURRENT_USER\Software\Microsoft\Office

    xircal say:

    Hi Irene,

    One more thing you can try before going the format route.

    Hit Windows key + X to open the power users menu and click "Power Options" near the top.

    Assuming you have the Balanced power plan, click the link to the right of it to "Change plan settings".

    In the next menu, click the link to "Change advanced power settings" underneath the buttons.

    Next, scroll down and expand the tree to "USB settings" --> "USB selective suspend" and change "Plugged in" to Disabled.

    Click Apply/OK to exit and then reboot.

    USB selective suspend works by suspending power to the port when it detects that it's not in use. However, when the user connects a device to a port to which power has been suspended, it can take a few seconds before it's restored. On some external devices, this could lead to it detecting the port as inoperable.
