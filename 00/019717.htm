Manual method:

Before you close up shop and start working on this reg fix, open the version of Word that you will want to be your default. In other words, say you have two versions of Word on your system. You want to use 2007, but you want 2003 to be the version that opens by default when you click on a DOC file. Open Word 2003 before you insert the fix below. This will ensure that Word 2003 is the last version opened and it will, if necessary, reset the registry to that version. (More on getting around this default in the next section.)

·         Now make sure Word (office apps) is closed

·         Start regedit  <Win  R> and type REGEDIT

·         Note that if this doesn't do anything, you're using a system where the administrator has taken away your ability to edit the registry so you don't mess it up!

·         Once within the registry, click the + (plus sign) to expand the /HKEY_CURRENT_USER/ folder.

·         Scroll down and locate the /Software/ folder and expand that one. We have a long journey ahead, so hang in there!

·         Once within the Software folder, scroll down that branch of the folder tree until you find the folder for /Microsoft/. Click the + to expand that one.

·         Now scroll down to locate the /Office/ folder.

·         There you'll see the numbered versions of the Office versions you have:

·         2010 is 14.0

·         2007 is 12.0

·         2003 is 11.0

·         2002 is 10.0e.

·         This fix can be used for all recent versions. Or at least, I know it works as far back as 2000. I can't confirm anything earlier. But if you're still using a version older than 2000, speed is obviously not an issue for you!<grin> Expand the folder for the version you want to speed up. (Note, this speed up fix will only help you if you are running multiple versions. So if you only have one version of Office on your system. Don't bother with this setting, as it won't do anything for you.)

·         Move to the /Word/ folder and expand that. Almost there!

·         Okay, locate the /Options/ folder and select that one. This is the folder where you'll be adding a new registry key.

·         With the Options folder selected in the route we took above, click Edit > New > DWORD Value

·         This will insert a new registry key into the Options folder. It will be highlighted in edit mode so you can go right ahead and start typing the name of this key: NoReReg

·         Hit the Enter key after typing the name to have the name accepted.

·         Now double click this new key to open it. The Value data will read 0 (zero) by default. Change that setting to the number 1

·         click OK to accept that value.

Then close the registry. You did it!

So what is it that you've done? Well, you went into the Word registry settings for the version of Word you selected and you set a new registry key. The key is an option that stops Word from reregistering itself if another version is the current default. The value of 1 means Yes. So the NoRereg (no reregistering) is set to Yes...i.e., don't do it! This will cause Word to ignore the default version and just open, without taking several minutes to go through the reregistering hassle.

Is there any chance where I can check how I can determine if the print preferences are incompatible with Internet Explorer 11?

It looks like this could be about a bunch of different but related objects.  E.g. using WMIC I would guess that it is creating a PrintJob.  Notice in there that there is something called Parameters and that is the closest thing that I can see which would be able to show you a PaperSize= value.

https://docs.microsoft.com/en-us/windows-hardware/drivers/print/standard-options 

You could try monitoring your two cases between stages to see if this is how it works.  Unfortunately it does not seem possible to modify a PrintJob once it is created.  E.g. that would seem to preclude a potential workaround such as adding or modifying a PaperSize= value to its list of parameters.

In any case, one method of checking would be to use a cmd window or PowerShell or just wmic.exe by itself to enter (assuming only one job exists at a time):

    wmic  printjob  list  /format:list

On a completely different tack there is apparently something associated with the printer itself which also could contain an indication of current PageSize

https://stackoverflow.com/questions/16601822/list-default-page-size-for-windows-printer-devices 

I don't think you need to use PowerShell to take advantage of the information implied there but it might help.
