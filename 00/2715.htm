th1238 say:

MS Access 2016 - network share drive security permissions

In previous versions of MS Access, using network shared drives, to access the back-end database users needed write access - read only access wasn't a sufficient level permission.  In MS Access 2016 is that still the case, or do you still need to give users write access?

scottgem say:
In reply to deleted message

Yes it is the case as long as you are using Access' ACE data store. The reason for it is that when you open an Access file, it create or modifies a locking file (laccdb or ldb) which manages record locking and performs other functions. So write/modify access is needed to manage this file.

You can, however, restrict the specific file to read only access. Another tool is using Traverse rights. See my blog on Securing Back Ends for details.

daniel pineault say:

Access needs, has always need, read/write/delete permissions to work properly. Nothing has changed in that regard.

Scott's suggestion of setting Traverse folder permissions is ideal when in a corporate environment allowing users to use the db but minimizing their ability to access the be directly.
