Switching cross references between paragraph number to paragraph text

I have a word document which is a questionnaire and it contains many cross references within the document.  The copy I have includes cross references displaying the paragraph number but I need a second copy with the paragraph text displayed.  Is there an easy way to switch all the cross references to paragraph text from paragraph number without changing each one individually?

doug robbins - mvp office apps & services (word) say:

If the document does contain the text \r or \n or \w in some other context, you could use a macro containing the following code:

Dim i As Long
With ActiveDocument
    For i = 1 To .Fields.count
        With .Fields(i)
            If .Type = wdFieldRef Then
                .Code.Text = Replace(.Code.Text, "\n", "")
                .Code.Text = Replace(.Code.Text, "\r", "")
                .Code.Text = Replace(.Code.Text, "\w", "")
            End If
        End With
    Next i
End With

suzanne s. barnhill say:

A cross-reference is a REF field. If you press Alt+F9 to examine the field code of a cross-reference that displays the paragraph number of a numbered item, it will look something like this:

{ REF _Ref448414332 \r \h }

A cross-reference that displays the bookmarked text will look like this:

{ REF _Ref448414332 \h }

The switch that causes the field to display the number may be \n or \w depending on which option you selected (see this article), but it will not be present in a cross-reference to the paragraph text.

If your document is unlikely to contain the text \r (or \n or \w) in any other context, you could just display field codes and use Replace to search for \r (or \n or \w) and replace it with nothing. After you do this, you'll need to toggle back (Alt+F9) and then select the entire document (Ctrl+A) and update field codes (F9) to display the new field results.

daniel pineault say:

A simple Google search quickly reveals it is merely a matter of switching the StartupForm propert and you will find code samples, such as:

 

'ChangeProperty "StartupForm", dbText, "YourFormName"
Function ChangeProperty(strPropName As String, varPropType As Variant, varPropValue As Variant) As Boolean
    Dim db As Database, prp As Property
    Const conPropNotFoundError = 3270
    On Error GoTo Change_Err
    Set db = CurrentDb
    db.Properties(strPropName) = varPropValue
    ChangeProperty = True
    Exit Function

Change_Err:
    If Err = conPropNotFoundError Then
        Set prp = db.CreateProperty(strPropName, varPropType, varPropValue)
        db.Properties.Append prp
        Resume Next
    Else
        ChangeProperty = False
        Exit Function
    End If
End Function

 

That said, you'd be better setting up an autoexec macro with a startup routine in which you control the form to be launched and so much more (relink your tables, apply security,...).
