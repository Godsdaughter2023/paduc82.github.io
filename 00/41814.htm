robert aldwinckle on forums say:
whenever i try to register a dll using the regsvr32 command in cmd, it gives an error that the "xxx.dll" was loaded but the entry-point DLLRegisterServer was not found! is there any other way i can register dll's? how do i fix this problem?


Then that probably means that "xxx.dll" does not contain a DLLRegisterServer entry point.   Hence you could not use regsvr32.exe with it.  It may have a  DLLInstall  entry point only.   In that case you could try using this command line syntax:

 

regsvr32.exe  /n  /i   xxx.dll

 

Tip:  use DependencyWalker  to find out what entry points a module exports.

 

Also note that in many cases you will only get a successful re-registration when possible when it is done in an elevated cmd window, aka Administrator Command Prompt.
