jllatham say:

With the help of the answers you gave to Mike H, I came up with this code for you.  To put the code into your workbook, open it and press [Alt]+[F11] to open the VB Editor.  In the editor choose Insert --> Module and then copy the code below and paste it into the code module presented to you.  Close the VB Editor.

To give yourself a button to click on to access the macro, either use the Command Button from the Forms Toolbox and right click on it and choose [Assign Macro] and pick the macro from the list.  Or you can draw a text box on a sheet and do the same right-click/Assign Macro to it if you prefer.

You'll be asked if you want to leave the workbooks where the invoice number is found open or not.

The code will ask you for the Invoice Number, if you don't enter one, it will quit.  It will next ask you to browse to the folder with the files to be searched and pick any file in that folder to begin the process.  If you don't pick a file, the process will quit.

Here's the code:

Sub SeekInvoiceNumber()
  Dim filesFolder As String
  Dim filesName As String
  Dim invoiceNum As Variant
  Dim foundFlag As Boolean
  Dim leaveOpen As Integer

  Dim testWB As Workbook
  Dim testWS As Worksheet
  Dim testCell As Range
  Dim foundCell As Range

  invoiceNum = InputBox("Enter the Invoice Number to find...", _
   "Enter Invoice Number", "")
  invoiceNum = Trim(UCase(invoiceNum))
  If invoiceNum = "" Then
    'no entry provided
    Exit Sub
  End If
 
  leaveOpen = MsgBox("Should workbooks with the invoice referenced " & _
   "be left open?", vbYesNo + vbQuestion, "Leave Books Open?")

  MsgBox "At the next prompt, select a file in the folder to be searched", _
   vbOKOnly + vbInformation, "File/Folder Selection"
  filesFolder = Application.GetOpenFilename
  If filesFolder = "False" Then
    'user cancelled
    Exit Sub
  End If
  'remove the filename, saving the path to the folder
  filesFolder = Left(filesFolder, _
   InStrRev(filesFolder, Application.PathSeparator))
  filesName = Dir(filesFolder & "*.xl*") ' any Excel file type
  Application.ScreenUpdating = False
  Do While filesName <> ""
    If filesName <> ThisWorkbook.Name Then
      Application.DisplayAlerts = False
      Set testWB = Workbooks.Open(filesFolder & filesName, False, True)
      ThisWorkbook.Activate ' be sure this book is the active one
      foundFlag = False
      For Each testWS In testWB.Worksheets
        Set foundCell = _
        testWS.Cells.Find(What:=invoiceNum, After:=ActiveCell, _
         LookIn:=xlFormulas, LookAt:=xlWhole, _
         SearchOrder:=xlByRows, SearchDirection:=xlNext, _
         MatchCase:=False, SearchFormat:=False)
        If Not foundCell Is Nothing Then
          'record the workbook as having the invoice number in it
          ActiveSheet.Range("A" & Rows.Count).End(xlUp).Offset(1, 0) = _
           testWB.FullName
          foundFlag = True
          Exit For ' exit testWS loop
        End If
      Next ' testWS loop end
    End If
    If Not foundFlag Or leaveOpen = vbNo Then
      Application.DisplayAlerts = False
      testWB.Close False
      Application.DisplayAlerts = True
    End If
    Set testWB = Nothing
    filesName = Dir() ' get next filename
  Loop
End Sub

kooolerking say:

Hello,

This bug has existed in both OneNote 2010 and OneNote 2013. It is related to a problem handshaking the OneNote audio with the Windows audio input device. In my experience, as you suggested, it fails to work correctly about 25% of the time, but appears to be random as to when. Unfortunately, there is no repair after the fact as OneNote apparently records a silent audio file with no input.

I've seen this same problem / question asked many times since 2010, and none of the official "fixes" suggested seem to work. The only work-around that I have found is as follows:

  1. set the OneNote audio input to "default device" under the options menu
  2. open the Windows sound control panel before starting the recording
  3. check the control panel tab "recording" to verify the desired input is the "default device"
  4. leave the control panel open to monitor the recording level
  5. start the audio recording capture within OneNote
  6. if there is no audio level showing in the control panel, stop the recording in OneNote
  7. restart the OneNote recording again and look at the audio level in the control panel - it should now show a level

What happens with the silent OneNote recording is that the input device sometimes gets either muted or bypassed, and it takes a second recording start to "wake up" OneNote" to actually correctly establish the handshaking with the Windows audio input.

Following the above process, I've never had another silent OneNote audio recording since.

Note: There is an option in the audio control panel to "allow applications to take control of the device", but from what I have experienced with this, whether checked or not, seems to have no apparent effect on the problem mentioned ... at least for Windows 7.

What is frustrating is that this has been reported, and the above work-around mentioned since 2010 but the problem has not been resolved yet in (2) subsequent OneNote version releases. There is a similar bug with adding meeting notes link from an Outlook meeting to OneNote ... it sometimes takes 2 or 3 tries for the function to work, but at least for it you get visual feedback for that issue.

It is almost as if OneNote goes to sleep or something and needs to be "woken up". I hope this helps.

Good luck!

As Doug has indicated it is not properly addressed in vba, but it is a built-in property (actually called Content Status) and can be written to with vba e.g. to set its value as 'Final'

Dim oProp As DocumentProperty

tisky zheng msft say:

Hi Hugocoffee,

 

To restore the deleted Team in Microsoft Teams, please follow the steps below:

 

1. Launch PowerShell as an administrator (Click Windows PowerShell, click Run as administrator).

2. Insert Install-Module AzureADPreview to install AzureADPreview module.

3. Insert Connect-AzureAD to connect to Azure AD. When prompted, sign in with your account.

4. Once you signed in, you will see the following:

5. Insert Get-AzureADMSDeletedGroup to display all deleted Office 365 groups.

6. Find the group you wish to restore, and copy the id from the Id column to the clipboard. Next, to restore the deleted group type the following where [ID] is the Id of the group and press Enter:

Restore-AzureADMSDeletedDirectoryObject -Id [ID]

7. Verify group was restored successfully by typing the following command, where [ID] is the ID of the group you restored:

Get-AzureADGroup -ObjectId [ID]

 

Note: The process of restoring an Office 365 group and thus restoring the team in Microsoft Teams can take up to 24 hours.

 

Regards,

Tisky
