You need a program on your system capable of reading Word documents.  Do you have MS Word/Office or the free Word Viewer http://www.microsoft.com/downloads/details.aspx?FamilyID=3657ce88-7cfa-457a-9aec-f4f827f20cac&displaylang=en or the free OpenOffice www.openoffice.org (which I would recommend in your current circumstance if you don't already have Word/Office) or something else that does the job (but you must be SURE it can read Word 2007 Macro-enabled documents.  Earlier versions of Word or Word Viewer (or even OpenOffice) might have problems with such a document unless they've been updated to handle the new format (and that can be done at the Microsoft Office Support center or by downloading the most recent versions of OpenOffice or Word Viewer).  Once installed, then you can then read Word documents and attachments.  Try opening the attachments again.

Perhaps there is a problem with associations.  Double-click on any Word file on your system (probably one in Documents or on your desktop) and see if it opens Word and displays the document.  In fact, try opening the file that was saved to see if you can.  If so, then the associations are working and that isn't the problem.  If not, go to Start / Default Programs / Associate a File or Protocol with a Program / search for .doc and check the default program - it probably isn't Word.  Click on Change and select Word.  If Word is not an option, click browse and search for the winword.exe file (somewhere in Program Files) and make it the default.  Do the same for .docx and .rtf (and any other file types you want Word to open).  Then try opening the attachment again (you may need to reboot for the changes to take effect).

If none of the above works, try posting your question in the Networking, Email, Getting Online Forum at: http://social.answers.microsoft.com/Forums/en-US/vistanetworking/threads where people who specialize in email problems like yours will be more than happy to assist you with your problem.

I hope this helps.

If you do not see an icon for Administrator in Safe Mode, post back for further instructions.

General Recommendations For Setting Up Users In Vista:

You absolutely do not want to have only one user account. Like XP and all other modern operating systems, Vista is a multi-user operating system with built-in system accounts such as Administrator, Default, All Users, and Guest. These accounts should be left alone as they are part of the operating system structure.

You particularly don't want only one user account with administrative privileges on Vista because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult. It isn't impossible to activate the built-in Administrator to rescue things, but it will require third-party tools and working outside the operating system.

The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. After you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

If you want to go directly to the Desktop and skip the Welcome Screen with the icons of user accounts, you can do this:

Start Orb>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).
