I also tried the Function tool, but again I either missed it or there's nothing suitable.

Could someone point this newb in the right direction. Thanks

 

sym wong-a-ton say:

The following formula works in InfoPath 2010.

count(../preceding-sibling::*) + 1

Use it as the Default Value for the autonumber field.

While I haven't tested the formula in InfoPath 2007, I'm assuming it would also work, otherwise you can always write code as described in this article: Autonumbering fields in a repeating table in InfoPath 2007 using VB.NET code
