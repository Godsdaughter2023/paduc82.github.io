ken sheridan say: The following function will return a string expression in the format mm:dd or y:mm:dd

Public Function GetAgeMonthsDays(varDoB As Variant, blnShowYears As Boolean, Optional varAgeAt As Variant) As Variant

    Dim intYears As Integer
    Dim intMonths As Integer
    Dim intDays As Integer

    If Not IsNull(varDoB) Then
        If IsMissing(varAgeAt) Or IsNull(varAgeAt) Then varAgeAt = VBA.Date
        
        intMonths = DateDiff("m", varDoB, varAgeAt)
        
        If Day(varDoB) > Day(varAgeAt) Then
            intMonths = intMonths - 1
            intDays = DateDiff("d", DateAdd("m", intMonths, varDoB), varAgeAt)
        Else
            intDays = Day(varAgeAt) - Day(varDoB)
        End If
        
        If blnShowYears Then
            intYears = intMonths \ 12
            intMonths = intMonths Mod 12
            GetAgeMonthsDays = intYears & ":" & Format(intMonths, "00") & ":" & Format(intDays, "00")
        Else
            GetAgeMonthsDays = Format(intMonths, "00") & ":" & Format(intDays, "00")
        End If
        
    End If
    
End Function

It would of course be a trivial task to amend the function so that the delimiters are forward slashes rather than colons.  You can see how it works in the debug (aka 'immediate') window, e.g. my age on 30th of this month would be:

? GetAgeMonthsDays(#1946-11-01#,True,#2017-04-30#)
70:05:29

Note that because a date value is the point in time at midnight at the start of the day the number of days returned is 29, not 30, i.e. the number of completed days.

As regards your second point, if you base a form or report on a query which calls the function, rather than on a table the duration can be returned in a bound control in the form or report.  You should not attempt to store the returned value in a column in a base table.
