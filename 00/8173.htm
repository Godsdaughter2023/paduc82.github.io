diane poremsky mvp (slipstick.com) say:

Open a new message - go to View and show the BCC field (so the CC field displays too). Then go to Tools, Forms, Design this form. Delete To and CC fields and drag the BCC field to the form. Then go to forms, publish form as and publish it to the Inbox. When you need to send messages only using BCC, go to Actions menu (while viewing the Inbox) and select your form. You can't set it as default and can't remove the To field if you use the address book to select messages. (Sample form here:http://www.slipstick.com/files/BCC_only.oft - right click on link and use Save target as... to open it, browse to it from Tools, Forms, Choose Form, user templates in file system.)

If nearly every message needs sent BCC, use VBA to ask if you really want to send - this (hopefully) will be enough to jar you into realizing a mistake (it will be annoying after a while):

Private Sub Application_ItemSend(ByVal Item As Object, Cancel As Boolean)
Dim intRes As Integer
Dim strMsg As String
strMsg = "Verify you used the BCC field before sending"
intRes = MsgBox(strMsg, vbYesNo + vbDefaultButton1, "Confirm Send ")
If intRes = vbNo Then
Cancel = True
End If
End Sub

The KB that Sagnik provided is for Office Pro Plus .  The "Plus" part is very important to the fix suggested.  It assumes that you got Office through a "downgrade" on an Office 365 subscription.  It is certainly worth testing and try, but I suspect that it does not apply.

Go into File menu > Help command. Look on the right side of the "Pane". It will show activation status (which is your current problem) and the Office bundle.  Confirm whether you have "2010 Pro" or "2010 Pro PLUS".

Another cause for this sort of error is having multiple copies of Office installed (a long shot in this case). Usually, the first copy is a free trial and the second one is one that the user installs and activates.  The fact that there are 2 copies means that often the activation system gets confused about which one you are actually trying to activate, and gets it wrong.  There is a simple check. Go into Control Panel > Programs and Features.  Look in the list of installed programs to see if there is more than one copy of Office installed.

IF there is more than one copy of Office installed, then uninstall the one that is not Office 2010 Pro (or Pro Plus) and do an Office Repair of your Office 2010, then activate 2010 again.

tomglass1 say:

I have found some methods of attaching pictures and adding descriptions. 

  1. You can insert a hyperlink that consists of a description of the picture (or file) and a link to that picture (or file). Right-click on the cell and pick "Link." Enter the description in the "show text" box and browse to the picture or file you wish to include. You can only have one link in a cell. This does not attach the picture; it just links it. If anyone knows how to get this easy process to result in an attachment instead of a link, I would like to know. 
  2. If you use Insert\Picture, you get a re-sizeable picture that you can put in a cell. If you make the picture tiny, you can also put some text in the cell with it. You can make the picture larger by stretching it. Verify that the Properties of the picture include either "move with the cell" or "move and size with the cell," and "maintain aspect ratio." You can put several attached pictures in one cell, but unless the pictures are shrunk down a lot, you will have trouble including them in a large spreadsheet.
  3. If you use Insert\(text)Object instead, you get the option of inserting an icon instead of a small picture. This works with other types of files, too, like Word, Excel, PDF, etc. The icons are small, and several can fit in a cell. They can even overlap. You can put text in the cell and position the icons next to the text. When you double-click on the icon, the attached document pops up at full size. I like to put in a short caption, press alt-enter to create another line, enter another caption, etc. Then I use Insert\(text)Object to put a file icon next to each caption.

For multi-lingual text entry, I think of this in four parts:

 a. how do I enter text in each script/language?

 b. what do I need in order to do proofing (spell/grammar checking etc.) in the language

 c. what do I need to enable any special features in Office for the language?

 d. do I want menus/help in the language?

For (a), AIUI Urdu's script is essentially the Arabic script with additional characters and ligatures for the sounds that occur in Urdu, and the standard Urdu keyboard has a different layout from the Arabic one. So how to proceed depends on what you have at the moment, how easy you find it to use one keyboard for another script, and so on. Whatever you do, I would make sure that you save a copy of your Word normal.dot/dotm if it has /any/ customizations/macros in it. Then you should go into Windows Control Panel->Region and language->Keyboards and Languages->Change keyboards...->Add, and add an Urdu keyboard, then go into (say) Word->File->Options->Language and add Urdu as an additional editing language. You have to restart Office after that. Personally I make use of the "On-screen keyboard" (Windows Start Button->All Programs->Accessories->Ease of Access->Onscreen Keyboard When you switch to the URDU keyboard in Windows you should be able to access special Urdu characters using that. But that's just what I'd do.

For (b), (c) and (d), I'd go with trip_to_tokyo's suggestion.

Which type of password protection is applied to these documents? Is it forms protection applied through the Review > Protect Document button, or is it a password to enable opening the document or one to enable editing?

For documents with forms protection, you can use the Insert > Object > Text from File command while the cursor is in the destination document. The content of the form will be inserted at the cursor, and it will be unprotected. The entire destination document must be unprotected until the last of the inserts is completed; then you can protected the whole document.

For the password to open the documents, there is no substitute for unprotecting the source document; until you supply the password, the document contents are encrypted. It would be possible to write a macro that will open each document using its password (which must be stored in or available to the macro) for the second or so needed to copy/paste it, and then close it.

To add a prepaid subscription, please use the product key from your reseller and redeem it on the site [View:https://office.com/setup365:550:50].

As mentioned before, in case of 2 subscriptions with licences for the same service, you do not need to reassign the licences: as long as you have the same or greater number of active licences (in total, regardless of a source subscription) for a given service as the number of user accounts assigned to that service, you will not experience any sort of disruptions.
