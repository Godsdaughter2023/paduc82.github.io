Until your Outlook.com account is migrated to the new Office 365 backend platform - you won't be able to create contact groups within the contact folder in the Outlook.com folder tree using OL'2013/'2016.

The "no" part of the response means that you are able to create Contact Groups in any other contact folder outside of the Outlook.com folder tree. Quickest way to do that is to create a new PST file and a new contact folder within that newly created PST file. Once done you can create a contact group in the new contact folder which can have members from any other contact folder in your profile. In this case, the contact groups(s) would only be local to the machine on which they were created (as PST files are local data stores). All comes down to how you work/what you need. Not everyone sends out emails from their mobile devices (as a for instance) in which case, not being able to sync the contact group the Outlook.com account itself is not something that is considered a problem. If you do need/want to send emails to a contact groups from any device using this account, then you will have to create the CG using the webmail interface. The downside of that is that contact groups created that way don't get sync'd as a group within OL'2013/'2016 (the name given to a contact group in Outlook.com becomes a category for those contacts in OL'2013/'2016)

Quickest way to determine if your Outlook.com account has been migrated is to log into the Outlook.com account using webmail (your browser). If it shows <Outlook Mail> it's been migrated.

For the sake of accuracy - the issue regarding Contact Groups relates to contact folders associated with Outlook.com accounts that have not been migrated to the Office 365 backend platform and as such are configured using the <Exchange ActiveSync> protocol (as opposed to <Exchange>) OR the account has been migrated and not re-configured in Outlook (re-configured = deleting the account and then re-adding it)

To determine if the account has been migrated, log into the Outlook.com via your web browser and it shows <Outlook.com> - it hasn't been migrated. Information regarding migrated accounts is contained in this thread (and countless others) so not going to repeat it here again.
