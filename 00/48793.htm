lemp say:
Both your initial question and FoxFifth's Answer were pretty light on specifics.

How is the Photosmart 3300 connected? 

My crystal ball is a bit cloudy, but I am guessing that the Photosmart 3300 is connected to your router by an Ethernet cable or by the printer's built-in wifi adapter. 

The most common problem with networked printers -- which especially arises when you do something like change your router -- is that the printer's IP address changes.

Here's the problem: 

Any device, whether it is a computer, printer, game box, or anything else, that is connected on the kind of network that virtually every Windows home user has must have an "IP address."  This is how the different devices know how to send data back and forth to each other.

In a home network, your router is responsible for assigning IP addresses to the devices on the network.  It uses something called a DHCP Server to do so, and by default, Windows, and many devices such as your HP printer, are configured to automatically accept the IP address assigned by the router.

When you first set up your printer, it connects to the network and is automatically assigned an IP address.  Later in the setup process, Windows configures its printing software to send print jobs for that printer to the IP address that was just assigned.

Now, for a variety of reasons, you turn on your printer, it connects to the router and is automatically assigned a NEW IP address.  The printer is connected to the network and can be detected, but Windows is still sending print jobs to the old IP address.  That old IP address is either unassigned or assigned to a device that doesn't know what to do with the print job and so when you click "Print," essentially nothing happens.

To confirm that the above is in fact what is causing your problem, do the following:

  • Use the front panel display of the printer to view or print out the wireless network settings and make a note of the IP address assigned to the printer.  See your User Guide at page 74 (pdf page 77) for directions.

  • On the computer, go to Printers and Faxes, select the icon for your printer, and select Properties
  • Click the Ports tab.
  • Click the Configure Port button.
  • You should see something like this --> http://3.bp.blogspot.com/_A8c50eN8tsk/S6fSg4qyQjI/AAAAAAAAB2g/OFl3ziQImwQ/s400/SS5.JPG
  • If the number in the box labeled "Printer name or IP address" is NOT the same as the IP address you noted from the printer, this is your problem.

At this point, you have two ways to fix things.  The quick and dirty way is to just enter into the box on the Port configuration screen the IP address that the printer reports as being assigned to it.  OK your way out and you should be good to go.

If that's all you do, you will be able to print for some time, but the problem may recur.  This usually happens if you turn off the printer and while the printer is turned off connect another network device.  The new device gets the printer's IP address (which isn't being used because the printer is turned off) and when the printer is next turned on, it gets a new IP address.

Some HP printer drivers have a little checkbox just below the "Printer name or IP address" which says something like "print to this printer even if its IP address changes."  If you have that box and check it, you may not run into this problem.

The sure-fire way to avoid the problem, however, is by manually configuring the printer with a "fixed" or "static" IP address.  The procedure for doing this is explained at page 75 of your user guide.  If you choose to do this, you must select a static IP address that is in the correct "subnet" but not in the range of addresses that can be assigned by your router.  You can determine these values from your router's configuration utility, but a method that will work most of the time for small home networks is to look at the currently-assigned IP address and make the static address different in the following way.

The IP address will have 4 numbers separated by dots.  For example: 192.168.1.101.  Your static IP address must have the same first 3 numbers and the fourth should be "far away" subject to the limits that it must be greater than 1 and less than 254.  So, if your printer was automatically assigned an IP address of 192.168.1.101, it would be safe to give it a static IP address of 192.168.1.200 or 192.168.1.25.

And, of course, when you change the printer to its new static IP address, you'll have to change both of your computers to use that IP address.
