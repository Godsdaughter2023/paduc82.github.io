isatv say:

How to force the Combo Box to Requery/refresh after typing some text in its control?

ken sheridan say:

But what about if he knows part of the code of the key only or part of the description in accnmim only?!

Then a combo box is not the right control type to use.  A combo box is predicated on the user selecting a distinct value from a list, not pattern matching.  The autoexpand mechanism allows progressive selection of matching rows as each character is entered, but only on the basis of the substring entered matching the left part of the value.  

If you want to use pattern matching then the appropriate control is an unbound text box not a combo box.  This can be combined with a bound combo box whose RowSource property is an SQL statement which correlates the control with the text box, e.g.

SELECT accno, accnmin
FROM [Accounts (Cust+CHO)]
WHERE accno LIKE "*" & Form!txtSearch & "*"
OR accnmin  LIKE "*" & Form!txtSearch & "*"
OR Form!txtSearch IS NULL
ORDER BY accnmin;

NB, Form in the above is the Form property, which can be used here rather than referencing the form by name as a member of the Forms collection.  The first two criteria in the WHERE clause restrict the result set of the SQL statement to rows where there is a substring match in either the accno or accnmin columns.  The third criteria is to return all rows where no value has been entered in the unbound text box.  This caters for existing records, and allows the user to select an item from the combo box in the usual way should they wish, rather than by entering a substring in the unbound text box.

The bound combo box should be requeried and a Null assigned to its value in the AfterUpdate event procedure of the unbound text box, e.g.

    Me.cboAccNo.Requery
    Me.cboAccNo = Null

Consequently, the bound combo box 's list would be restricted to the subset of rows in which there is a substring match on either column.  The user would then make the final selection from this shortlist.

A Null should also be assigned to the unbound text box's value, and the bound box requeried in the form's Current event procedure, e.g.

    Me.txtSearch = Null
    Me.cboAccNo.Requery

An alternative, and very simple, approach, which would allow the user to select either the accno or accnmin values, would be to have two combo boxes, both bound to the key accno column, with one listing the accno values and the other the accnmin values.  The user can then select from either, upon which the other would automatically show the corresponding value.  Or this dual bound combo box approach could be combined with the above method, correlating both combo boxes with the unbound text box, thus allowing the user to select from one of two shortlists on the basis of the value entered in the unbound text box.

Excel 2007 VBA Find and Replace

Hello,

 

How can I FIND and REPLACE certain values in Column G using Excel VBA?

 

For instances, if there are values in Column G that list values such as "Hamburger", "Fries" I want to FIND those two values in Column G and REPLACE it with "Fast Food".

 

Thank you.

mike h.. say:

Hi,

 

Try this code. Note that you add all the things you want to replace to the string s

 

Sub Replace_Things()
Dim LastRow As Long, r As Range
Dim s As String, v As Variant
Dim ReplaceString As String
'This is the string you want to put in the cell
ReplaceString = "FastFood"
'These are the string you're looking for
s = "Hamburger,Fries,Dogs"
v = Split(s, ",")
LastRow = Cells(Cells.Rows.Count, "G").End(xlUp).Row
For Each r In Range("G1:G" & LastRow)
    If Not IsError(Application.Match(CStr(r.Value), v, 0)) Then
        r.Value = ReplaceString
    End If
Next
End Sub
