hansv mvp say:

Yes, I see that behavior too. Access tends to mess up memo fields in general (not just rich text) in totals queries, so that doesn't suprise me.

 

This means that the problem is not DoCmd.TransferSpreadsheet, but the totals query. 

You may be able to work around it as follows:

 

1) Create a totals query that does not include the memo field. Make sure that it includes a unique identifier such as a primary key field (with the First option).

2) Create a query based on the totals query and the original table, joined on the unique identifier. Include the memo field in this query. It should display rich text instead of HTML code.
