But when I switch to my alternate computer (Samsung laptop running Windows 8.1, and does not experience this issue), then back I lose keyboard/mouse functionality, display works properly. This is agnostic to whether I use the keyboard hotkey or physical buttons to switch channels. Rebooting, with the Lenovo's channel selected, resolves the issue. Switching KVM channels then back again loses keyboard/mouse. Since my Windows 8.1 laptop is not experiencing this issue, it would seem to indicate that Windows 10 is somehow the culprit, or possibly something to do with the Lenovo hardware/drivers.

Troubleshooting measures attempted thus far, to no avail:

  • update all Lenovo drivers (BIOS, chipset, display, full boat)
  • ensure Windows 10 is current with updates
  • swap HDMI and USB cables between the 2 KVM channels
  • swap KVM channels
  • unplug all connections to KVM, plug HDMI/keyboard/mouse out back in, plug Lenovo channel HDMI/USB back in

Once the keyboard/mouse are lost after switching channels, the only way to recover them is to reboot.

Any thoughts are appreciated!

andrew---# say:

so I kept troubleshooting and finally found the source of the problem, but I'll keep this posted just in case anyone comes across this issue.

this has nothing to do with Windows 10, but rather USB 3.0. The KVM states it's USB 2.0, and it apparently doesn't like USB 3.0 ports. All the Lenovo T470s USB ports are 3.0, so what worked? connecting the KVM's USB cable through a USB 2.0 splitter or hub. Everything now works like a charm switching between channels.

andre da costa say:
Try simplifying your configuration by keeping only your mouse and keyboard attached, that means:

Disconnecting multiple display monitors.
Remove discrete video cards, use the on-board graphics if your motherboard supports it.
Swap out memory modules - a RAM module might be corrupt - you can also ensure RAM sticks are installed slots 1 and 3. Your RAM modules might be corrupt, try scanning them for errors.
If you have more than one hard disk/SSD installed, disconnect all except the target disk where Windows 10 will be installed.
Reset your BIOS Defaults.
Disconnect all external devices, since connected devices can sometimes confuse setup.

If setup is successful after performing these tasks, you can reinstall each removed component.

Another thing I would recommend, when you boot from the install media and arrive at the 'Install Now' screen, disconnect the USB thumb drive, go through setup and proceed with the normal partitioning and continue on with installation.

When ready, setup will prompt you to reconnect the original install media.
