bill manville say:

Not without creating a workbook containing a VBA macro to do the job.
The command would run Excel and open the workbook.  The workbook would contain a macro such as this:

Sub Auto_Open()
' don't try and do it in Auto_Open as some things don't work
' run Continue_Open when this workbook has been fully opened
  Application.OnTime Now, "Continue_Open"
End Sub

Sub Continue_Open()
  Workbooks.Open "C:\MyFiles\MyXMLFile.xml", ReadOnly:=True
  ' do some processing
  ' save as a suitable format for import to SQL Server....CSV?
  ActiveWorkbook.SaveAs "C:\Temp\MyCSVFile.csv", FileFormat:=xlCSV
  ActiveWorkbook.Close False
  If Workbooks.Count = 1 Then
    Application.Quit
  Else
    ThisWorkbook.Close False
  End If
End Sub

jay freedman say:
Usually tables in Word -- in body text or in headers or footers -- don't have a "limit" of any kind. They just expand vertically to hold whatever you put into them.

The one situation in which that is not true is when the row (or maybe all the rows) of the table is set to a row height of "Exactly" some measurement. Then the height of the row can't expand, and anything (text or graphic) that extends lower than the bottom of the row gets cut off.

Right-click in the row where this problem appears, and click Table Properties in the menu. In the Table Properties dialog, click the Row tab. In the Size section, if it says "Row height is: Exactly", change the dropdown to say "At least". Then click OK. Everything you put in the row should now be visible.
