Opening a Office 2010 files from a WebDAV folder results in an "is - Microsoft Community

andrew france say:

Opening a Office 2010 files from a WebDAV folder results in an "is not valid address" error, Office 2003 files work fine

I've got a strange issue with office files. When I try to open Office 2010 files from home through my work's WebDAV connection, it opens up Office and then I get the following error:

"The Internet address 'https://.........../New.docx' is not valid"

Any other file including older 2003 Office files work fine. I'm also getting this alert logged in the event viewer:

Microsoft Word
The Internet address 'https://........./New.docx' is not valid.
P1: 701727
P2: 14.0.4763.1000
P3: 
P4: 0x80070002

This occurs from a link in IE that opens up the WebDAV folder or a Network Location mapped in Computer

Any ideas welcome.
andrew france say: OK fixed this...

It was related to IIS in the server that's hosting the WebDAV folder. This is a pretty old server (2003 IIS6), so the issue was the new Office extensions aren't included in the MIME types.

Adding additional MIME types to the server fixed this problem...

Extension: .docx
MIME Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document

Extension: .xlsx
MIME Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet

Extension: .pptx
MIME Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet

Hope this helps someone
