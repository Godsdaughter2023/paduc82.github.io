johnm121 say:

I guess I don't understand why you'd have any entry at all in your internal DNS server -- on your domain's public DNS record, you'd have added the proper CNAME record for autodiscover.yourdomain.com pointing to O365, and clients would perform their lookup to that public DNS record whether they are on your LAN or connecting from home. The code you have mentioned should work for YouTube videos.  However, there is a catch:  it will only work on Embed cards, not on Video cards.  As soon as you commit the embed code, the card will automatically convert to a Video card.  If you want it to remain an Embed card, as soon as it converts, press the Undo button to turn it back into an Embed card.  That should allow autoplay to work.

Here are the steps:

  1. On youtube.com, get the iframe embed code
  2. In Sway, insert an embed card
  3. Paste the iframe embed code into the card
  4. At the end of the src property value (inside the ") add ?autoplay=1

Tummon,
No problem joining a thread especially if your problem is the same or very similar. Are you also getting a circular reference message? If so, it means that a link somewhere in your file is trying to tie back around on itself. Circular references can be a real pain to ferret out - it takes patience, time and a good deal of luck.

The best way to avoid the problem is by not putting any links (i.e. predecessors and/or successors) on summary lines as Jan eluded. However, circular references can also be inadvertently created if you change the indenture, (i.e. outdent a sub-task to create a summary line), and the task has links. Circular references can also occur in master/subproject structures wherein the user creates "backward pass" links between projects without realizing it.

If you are not having a circular reference issue but your tasks are simply disappearing as was mentioned in the original post, then obviously something else is at play.
