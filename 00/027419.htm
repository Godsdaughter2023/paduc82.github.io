Is MS Access an API? - Microsoft Community

vwp1 say:

Is MS Access an API?

Is MS Access an API?
scottgem say:

The relational database model has been in existence since the early 1970s. Access follows that model as do many other back end database engines that subscribe to SQL. SQL is the query language for relational databases. Oracle, MySQL, DB2, Progres and several other database engines are relational. 

There are other database models that are more focused on specific tasks. but they are not as well used as the relational model.

Being a relational database has nothing to do with APIs. 

scottgem say:

No, in fact Microsoft is actually a bit late in the Cloud. Amazon Web Services (AWS) is actually the premier player.

But the "Cloud" generally refers to storing your data online where it is accessible from anywhere over the Internet. It can also include running applications through a browser that can access data stored online.

When you use "code" to add records, then the link master + link child collum settings are NOT done for you. You have to add that part in code. So, you need to add this to your code:

.AddNew   

   !HvlEname = [Combo18].Column(0, 1)

   !HvlEcode = [Combo18].Column(1, 1)
   !NameOfCollumUsedToRelateToParent = me!PrimaryKeyCollumNameofParentTable .

Update

 

You have to replace the above names I used since I not aware of the collum names used to setup the relatonship. So, just keep in mind as noted you have set the value used for the relationship as Access does not do this for you when adding records via code.
