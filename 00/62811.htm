a. user say:

Most folks think that running sfc /scannow will replace missing or corrupted files in the system32 folder but that is not what sfc /scannow does and that is not what it is designed to do.

Even suggesting sfc /scannow as a way to resolve some problem since you believe it will "replace missing or corrupted files" is the same as saying "I have no idea what the problem is or how to fix it".

sfc /scannow is designed to run one time after Windows is first installed.  It's job is to populate the dllcache folder with backup copies of 3498 important Windows files (there is a list).

If anybody thinks sfc /scannow replaces missing or corrupt files in the system32 folder here is a test scenario you can safely run on your own system to prove that is not what sfc /scannow does.

We'll use the spoolsv.exe file in the system32 folder in this lesson.

Boot your system into Safe Mode or the Recovery Console so Windows File Protection is not running.

Rename your c:\windows\system32\spoolsv.exe file to something you can remember (like spoolsv.bak) so that the file is now "missing".

Since Windows File Protection is not running in Safe Mode, the missing file will not be automatically replaced like it would be if you were booted in Normal mode.

Reboot your system normally so Windows File Protection is running again (the spoolsv.exe file is still "missing").

Run sfc /scannow and it will not replace the "missing" spoolsv.exe file - because that is not what sfc /scannow does.

Now copy some bogus file like your boot.ini file into the system32 folder and call it spoolsv.exe:

copy  c:\boot.ini   c:\windows\system32\spoolsv.exe

In effect, now you have a spoolsv.exe file, but the spoolsv.exe file is now "corrupted".

Run sfc /scannow again and the corrupted file will still not be replaced since that is not what sfc /scannow does either.

Of course sfc /scannow is good for only one thing, but not these kinds of things and not what most people think.

Rename your spoolsv.exe ifle back when you are done.

Now you know what sfc /scannow doesn't do.

You can do this on any of the 3498 files that Windows File Protection and the System File Checker know about (yes, there is a list).
