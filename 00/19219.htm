jay freedman say:

Word can already do what you want.

  • Display all formatting marks (click the Â¶ button on the Home tab). The Find/Replace dialog can't find anything that's Hidden unless it's displayed.
  • Open the Replace dialog and click the More button to expand it.
  • In the Find What box, type the code ^p to find a paragraph mark.
  • While the cursor is still in the Find What box, click the Format button in the bottom right corner of the dialog and click Font in the menu.
  • In the Font dialog, click the Hidden option to put a check mark in the box. (All the other options' boxes should contain a black square, meaning "don't care".) Click OK.
  • In the Replace With box, type the code ^& for "the thing that was found". (Using this will preserve the style applied to the paragraph mark, which may vary from place to place in the document.)
  • While the cursor is still in the Replace With box, click the Format button in the bottom right corner of the dialog and click Font in the menu.
  • In the Font dialog, click the Hidden option twice to clear the check box. Click OK.

  • Click the Replace All button.

cbarnhorst say:

In the summer of 2015 MSFT ended special product keys for Insider builds and began requiring that all copies of Windows be normally licensed.  Preview builds, like all beta software, have expiration dates after which the build stops working.  Activation relies on an underlying license assigned to the device, not the build.  

Enterprise and Education editions are Volume License products, unlike Home, Pro, and Pro for Workstations which are digitally licensed. 

MSFT will not sell you a stand-alone license for Enterprise.  A Volume License Key (VLK) needs to be obtained from an owning enterprise or organization and activation is maintained by being connected to the org's activation server periodically.

You can obtain a six-month trial of Enterprise but it cannot be activated unless you obtain a VLK from an org. and maintain activation by connection to the server at least once in a six-month period.

don cuthbert say:

If your device does have a license for Windows 10 Pro, you can easily toggle to the Pro edition instead, using the generic Product Key for Pro, that is found on the official Windows 10 install media in \Sources\product.ini. You can open the file in Notepad to read the list of generic Product Keys.  If your device has no existing Windows 10 license, none of these keys will activate any Edition of Windows 10.

However, if your license is for Home edition, you will need to do a clean install of Home, as you cannot toggle editions from a professional edition to the Core/Home edition.

mbizup say:

As far as I know you cannot define the data type of an alias.

But if the length is always less that 100, you may be able to use the LEFT function to compare strings in your JOIN:

SELECT DISTINCT A.Column2 FROM Table1 AS A Left JOIN Table2 AS B on Left(A.Column2  & "", 100)  = Left("" & B.Column3, 100) where B.Column3 IS NULL

But why not just change the datatype of the underlying column from memo to text?

mbizup say:

Btw, there are a variety of functions that will have the same effect as the Left function.  The idea is perform the comparison on converted strings instead of directly on the Memo field.

This will work as well, without the 100 character limitation on the length of the strings being compared:

SELECT DISTINCT A.Column2 FROM Table1 AS A Left JOIN Table2 AS B on cstr(A.Column2  & "")   =  cstr("" & B.Column3) where B.Column3 IS NULL

(but again, if the data length is usually going to be less than 100, I would simply convert the field to Text, 250 or similar... keeping it text with allowance for some longer entries.)

ken sheridan say:
Query used

- SELECT A.Column2 FROM Table1 AS A Left JOIN Table2 AS B on A.Column2 = B.Column3 where B.Column3 IS NULL


To return rows from Table1 with no match in Table2 you might try this:

SELECT A.Column2
FROM Table1 AS A
WHERE NOT EXISTS
    (SELECT *
      FROM Table2 AS B
      WHERE B.Column3 = A.Column2);

As far as I can see, from trying this in Access 2007 at least, memo (long text) fields can in principle be used to correlate a subquery in this way, though whether the length of text in them will make any difference I cannot say.  Interestingly it also seems to work if the column in one table is Text data type, and that in the other is Memo data type.

However you do it performance is likely to be inhibited when comparing memo fields because of the inability to use the indexes.
