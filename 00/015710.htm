paul domaskis say:

vlookup: use column heading to specify column from which to get return value

I'm a big fan of vlookup in both excel formulas and vba.  However, using the column number to specify the column from which the return value is to be taken seems to be archaic -- like using numeric constants without naming them, or using cryptic variables without defining what they mean.  Ideally, the user should be able to specify the column heading in order to identify the column from which the return value should be taken.  (Granted, column headings could be lengthy and unwieldy).  I know this can be faked by using other formulas like Match (and I know that I could do it with a bit of reading and trial/error).  However, readability becomes severely compromised.  Is there best practice to achieve similar results?

One alternative to numerically specifying the column number is to define the column numbers as meaningfully named constants.  However, this leaves the task of ensuring that the column number assigned to a meaningful name always matches the actual column in the lookup table (I'm assuming that the user actually adds/removes/rearranges columns in the lookup table in the course of developing a solution).

Hi Paul

 

Replace the lookup_value and column_header with your actual lookup values

 

 

 =VLOOKUP("lookup_value",$A$1:$E$5,MATCH("column_header",$A$1:$E$1,0),0)

  • if the value for "Tier 1, 2, or 3" is 3, then Last Reviewed Date" and "Date Eval Completed" must be equal to or greater than two years ago from the current date, if not then they will be highlighted in red along with the title.

dhookom say:

When you have multiple conditions you might want to consider creating a user-defined function with all of the logic. For instance if you want to compare order, required, and ship dates along with freight a function might be something like:

Public Function GetPriority(vOrdDate As Variant, vReqDate As Variant, _
        vShipDate As Variant, curFreight As Currency) As Integer
    Dim intPriority As Integer
    If IsNull(vOrdDate) Then ' no order date then no priority
        intPriority = 0
      Else
        If Not IsNull(vReqDate) And IsNull(vShipDate) Then
            intPriority = intPriority + 1
            If vReqDate < Date Then
                intPriority = intPriority + 1
            End If
        End If
    End If
    If curFreight > 100 Then 'the greater the freight the higher the priority
        intPriority = intPriority + curFreight \ 100
    End If
    GetPriority = intPriority
End Function

You can set the Conditional formatting based on the value returned from this function. The Conditional Formatting rule would be

Expression Is: GetPriority([OrderDate],[RequiredDate],[ShippedDate],[Freight])=0

This would probably be low priority with a green background

Expression Is: GetPriority([OrderDate],[RequiredDate],[ShippedDate],[Freight])=1

This would probably be medium priority with a yellow background
