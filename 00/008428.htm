cnt lm say:

Hi

Windows phones by default use DHCP. In this process when your phone connects to a Wi-Fi router or access point it requests an IP address from the router or access point.

In this case, the router or access point give you that IP address. Routers and access point give IP addresses from a predefined pool of IP addresses that is configured in the router or access point.

The IP address you where given 192.168.206.10 it’s from a completely different range than the ones you mentioned 192.168.2.101 to 192.168.2.150. This point to the fact that something in the configuration of the access point or router that you are connecting to is misconfigured.

Are you sure the router/access point has not changed?

Trying to use a static IP address from the range 192.168.2.101 to 192.168.2.150 it’s not a good idea. As people access the router or access point the device will supply addresses from that IP pool, (in this case 192.168.2.101 to 192.168.2.150). Imagine you try to use the IP 192.168.2.120, if the router or access point already give this address to another device both devices will stop working because there is an IP conflict in that network, you cannot have the same IP twice in the same network.

The only way to resolve this is by explaining this issue to the person responsible for configuring the router or access point you are connecting to.

Do you now who that person is in your company?
