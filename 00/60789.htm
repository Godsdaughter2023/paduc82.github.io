praseetha k say:

Hi John,

 

On a 64-bit operating system, 32-bit applications run on top of an emulation of a 32-bit operating system that is called Windows on Windows 64, or WOW64 for short. Its primary purpose is to create a 32-bit environment that provides the interfaces required to allow 32-bit Windows applications to run unmodified in the 64-bit system.

 

On 32-bit Windows operating systems, programs are installed by default into a system folder called Program Files, while operating system components and shared libraries are installed into the System32 folder. On a 64-bit Windows operating system, there are separate versions of those folders for 32-bit and 64-bit software components. So, in addition to the Program Files folder, there is a Program Files (x86) folder for 32-bit applications. Also, in addition to the System32 folder, there is a SysWOW64 folder. Contrary to what the names may suggest, 64-bit operating system components and shared libraries go into the System32 folder, while 32-bit operating system components and shared libraries go into the SysWOW64 folder.

http://msdn.microsoft.com/en-us/library/aa384249(VS.85).aspx

Since this issue occurred while updating the video card drivers, I would suggest you to boot the computer in Safe mode and then perform a driver rollback.

 

Method 1: Rollback driver from Safe mode

a.    Once you’re in Safe mode, click on Start; in the start search type devmgmt.msc and Hit Enter.

c.    Right click on the display component and click on Properties

If the above method wasn’t of any help, then perform system restore from Safe mode

A parameter out of range may or may not indicate a potential hard drive problem as per Dell. Visit the link below for the same.

http://support.dell.com/support/edocs/systems/op330/en/ug/tools.htm 

 

Steps to perform a check disk from Windows Recovery Options

a.    In the command prompt, change the location to c: and then type chkdsk /r and press Enter.

b.   Answer Y when chkdsk asks you if you want to check the drive next time Vista boots (restarts).

Disclaimer:  /r = the /r switch tells chkdsk to attempt to recover any bad sectors of the hard drive if any are found. A bad sector is a spot on the hard drive that cannot hold saved data anymore. If you’ve any data on that sector, you’ll lose that.
