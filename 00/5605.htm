robertspinks1 say:

Access Form not saving all fields to the underlying table

I am running Access 365 accdb

It calculates on the form but does not save to the file, hence ends up with the same invoice number all of the time.

The other is to calculate a proposed discount if paid on time, Control Source =[DebitJ]*0.1

Although both calculate correctly on the form they do not save to the table.

These fields cannot be don at a later date or through query.

I would appreciate detailed Help.

Regards

RobertSpinks1

ken sheridan say: In the case of the invoice number the control should be bound to the InvoiceSNo column, and the return value of the expression assigned to the control in the form's BeforeInsert event procedure with:

    Me.InvoiceSNo = =Nz(DMax("InvoiceSNo","Journals"),0)+1

The discount should be applied in the same way only if it is functionally determined by the table's key, and can legitimately be amended, either in individual records in the form, or by a change in the standard level of discount  in the future.  Otherwise it should not be stored in a column in the table as that introduces redundancy and the consequent risk of update anomalies.  Instead it should be computed at runtime as you are doing.  The same should be done in reports, or it can be done in a computed column in the form's or report's RecordSource query.

The discount ratio should not be encoded in an expression.  It is data, and a fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.  It should be stored in a row in a separate discounts table, and looked up in the expression by means of the DLookup function.  If it is envisaged that the discount ratio might change at some time in the future, no matter how distant, it should also be stored in a column in the InvoiceSNo table, and assigned to the DefaultValue property of a control (which can be hidden)  bound to the column in the form's Open event procedure, e.g. if the Discounts table has only one row:

    Me.DiscountRatio.DefaultValue = """" & DLookup("DiscountRatio", "Discounts") & """"

The DefaultValue property is always a string expression, regardless of the data type of the column in question, which is why it is wrapped in literal quotes characters in the above expression.

The ControlSource property for the Discount control in the form would then be:

   =[DebitJ]*[DiscountRatio]

Returning to the method of computation of the invoice number, a simple expression in the form's BeforeInsert event procedure is fine in a single-user environment, but in a multi-user environment on a network there is the risk of a conflict if tow or more users are inserting an invoice record simultaneously.  You might like to take a look at CustomNumber.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the example for 'Sequential Numbering' resolves any such conflicts.  It also allows you to 'seed' the next number to be inserted, and/or to set the integer value by which the number is incremented.

scottgem say:

Please see my blog on Sequential Numbering (see below for blog address). It contains detailed instructions on generating Sequential Numbers in a variety of situations. 

The reason the value isn't saving is because, to save a value, the control source has to be a field in the underlying Recordsource. Expressions will not be saved. So you need to generate the number in code, not in the controls ControlSource. My blog explains how to do that.

VoIP in Asha Platform phones - Microsoft Community

sudude say:

VoIP in Asha Platform phones

Technical Level : Intermediate

Summary

Details

Nokia maintains a list of the phones running Asha Platform OS (v1.1 or newer) that have built in SIP VoIP capability in the firmware.

 

Configuring the phone for built in VoIP usage

 

There are three different ways how to configure built in VoIP in use in Asha Platform phones,

  • Using the built in Accounts setup which can be found from Accounts.
    • The list of VoIP service providers can be found under More accounts.
    • If you cannot find your VoIP service provider from the list, but know the SIP details for it, choose Other VoIP from Accounts and fill in the SIP details

 

  • Configuration over the air (OTA SMS). The VoIP service provider providing the OTA SMS configuration needs to have Asha Platform VoIP compliant configuration settings

 

  • Manual configuration requires following steps,
    1. Download settings example (XML file) compatible to your phone running Asha Platform OS
    2. Download wbxml2 tool
    3. Modify the XML file to suit your needs
    4. Use the xml2wbxml.exe (from wbxml2 tool) to convert the XML file to wbxml format and rename the.wbxml to .prov
    5. Scan and connect your phone over Bluetooth to your PC
    6. Send (OBEX PUSH) the .prov file to the connected phone. 

 

VoIP app

 

  • Downloading an app from Store (just choose your phone model from top of the screen to get relevant apps listed)

 

  • Developing an Java midlet with VoIP support
    • Asha SDK 1.1
    • Developer Guide to develop midlet with VoIP support
    • Example midlet with VoIP support

Hello Deepa,

 

Really appreciate your response, however none of this helped. I'll explain because we've resolved it so I hope this will help other people who are having issues as I know its a headache. Surprisingly simple resolution in this case.

 

(1) - In the beginning - Your first attept

You will try to install, the web will say installing, then it will go back to install after a short while. You think there is nothing happening and close your browser. You try again and then you get told you have a copy already installed and it needs to be removed from programmes (but its not visible). You need to get the M/S FIX I|T program. - The problem here is that something was happening in the background but it was not evident on the screen. It then starts to get confusing and messy. 

 

(2) - Clear up

Make sure you get the FIX IT to remove the existing partial deployment of the application

Make sure you do not have any other older office versions installed. his will also cause problems

 

(3) - OUR Problem

We use a proxy. The proxy was set in Internet Explorer for each PC, HOWEVER we discovered that it also needs to be set at HTTP level and must include the port. This is done as follows:-

 

Open Command prompt (Run as Administrator) <----Important

change directory  cd c:/windows/system32/   <-----For the 32 bit version

type    netsh winhttp set proxy YourProxyName-or-IP:3128 <--- Change to your port number

hit return to set it

 

Set it back to default no proxy when you are finished by using

netsh winhttp reset proxy

 

This worked for us - It took days to come to this conclusion. We had to get a network guy to come and look and he came up with this solution.

huni shen msft say:

Hi HenrikLawall,

 

Follow these steps to create your GitHub Enterprise connector.

 

Step 1:

In your GitHub Enterprise repository, in the Settings tab, select Hooks and services. Note that you will need to have administrative access to your repository to see this section.

 

Step 2:

Click Add WebHook button to access the form for creating a webhook.

 

Step 3:

Add the copied WebHook URL as the Payload URL.

In the Content type drop-down list, select application/json.

You can leave the Secret field blank.

 

Step 4:

In the Which events would you like to trigger this webhook? section, select the events you want to be notified for.

Note that only the following events are supported:

  • Issues
  • Pull request
  • Push
  • Issue comment
  • Commit comment
  • Pull request review comment

 

Step 5:

Ensure the Active check box is selected.

 

Step 6:

Click the Add WebHook button when you're done.

 

If followed the above steps and still get the error, I suggest you contact the Github support team and see if they can provide you some suggestions.

Huni
