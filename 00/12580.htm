(1) ThisWorkbook should contain the following code (I think you have this):
 
Option Explicit

Activate_Tab
 
End Sub
 
 
(2) Your workbook should contain the "Accessibility" module as per the
(Excel) sample on my site or as per the same code I posted here yesterday (I

(3) You should have another module as per the one called "Activation" in
the (Excel) sample now on my site, with this code (I'm not sure what you
have for this):
 
Option Explicit
 
Private Attempts As Long
 
Public Sub Activate_Tab()
 
Dim RibbonTab As IAccessible
 
Set RibbonTab = GetAccessible(Application.CommandBars("Ribbon"), _
ROLE_SYSTEM_PAGETAB, "Custom")
 
If Not RibbonTab Is Nothing Then
If ((RibbonTab.accState(CHILDID_SELF) _
And (STATE_SYSTEM_UNAVAILABLE Or STATE_SYSTEM_INVISIBLE)) = 0) Then
RibbonTab.accDoDefaultAction CHILDID_SELF
Else
MsgBox "Designated Tab is unavailable"
End If
Else
Attempts = Attempts + 1
If Attempts <= 10 Then
Application.OnTime Now + TimeSerial(0, 0, 0.5), "Activate_Tab"
Else
MsgBox "Custom Tab cannot be activated"
End If
End If
 
End Sub
 
 
 
The code in the "Activation" module now has a check to stop it running
forever if it fails to activate the tab. If the tab does not activate you
should get a message box telling you it failed - if you don't see that, then
it is not running and we'll have to work out why. I would suggest putting a
break (or a message box like the "8" I inadvertently included in one post)
in the code to see what's happening.
 
Note that the name of the tab should be whatever shows in the UI and is
case-sensitive. As you say it works when run manually, I guess you have this
correct.
 
The code in the "Activation" module does not run when included in
ThisWorkbook, giving the message you have seen.
