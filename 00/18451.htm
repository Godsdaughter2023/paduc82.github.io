jim gordon mvp say:

Here's a VBA code example that sets header and footer properties and includes the ability to set independent margins for header and footer. This code runs on Windows and Mac Excel. I highlighted the header and footer margin settings so you can see that they are set different from the page margins set just above them.

Sub IndependentMargins()
    With ActiveSheet.PageSetup
        .LeftMargin = Application.InchesToPoints(0.45)
        .RightMargin = Application.InchesToPoints(0.45)
        .TopMargin = Application.InchesToPoints(0.75)
        .BottomMargin = Application.InchesToPoints(0.5)
        .HeaderMargin = Application.InchesToPoints(0.8)
        .FooterMargin = Application.InchesToPoints(0.8)
    End With
End Sub

If you're not familiar with Visual Basic and the Visual Basic editor, it's pretty easy to use. Simply display the editor (it's like a funky word processor). Paste the code above into the editor, adjust the values as desired in the parentheses, then run your code. 

A shorter version of the code that adjusts only the header and footer margins is:

Sub IndependentMargins()
    With ActiveSheet.PageSetup
        .HeaderMargin = Application.InchesToPoints(0.8)
        .FooterMargin = Application.InchesToPoints(0.8)
    End With
End Sub

jay freedman say:
The first step in creating any kind of index in Word is to "mark" the entries in the main text. You can do this with the Mark Index Entry dialog (from the References tab, or by pressing Alt+Shift+X). This inserts XE fields into the document, which will be collected by the INDEX field that you insert at the end of the document. The usual steps are shown in http://www.writersservices.com/writersservices-self-publishing/making-index-using-word.

To make an index of names in surname, given name order, you have to create the XE fields properly. There are two ways to do that, which will result in two kinds of entries in the index when people have the same surname.

If you want the surname to appear for each person as in

Doe, John B
Doe, Mary

then you have to type the names in the Main Entry box of the Mark Index Entry dialog that way, including the comma. That will create XE fields that look like {XE "Doe, John B"}.

If you prefer to have the surname shown once, followed by the given names and page numbers indented below it, you have to put the surname in the Main Entry box and the given name(s) in the Subentry box. That results in XE fields that look like {XE "Doe:John B"}. The index then looks like

Doe
   John B
   Mary

If you've already marked all the entries in given name, surname order, it would be very tedious to change them manually, so you'll need a macro to modify the XE fields. If that's the case, post back.

bob jones aka: cybertaz say:

I'm afraid that if you never actually Saved the file it never was written to disk so there was nothing to be backed up by Time Machine or anything else.

AutoRecovery files are retained only in the event of a critical failure occurring while a document is open. When you close a file normally the AutoRecovery file pertaining to it is automatically deleted, regardless of whether you say Yes or No to saving any changes.

I don't discourage anyone from using the Always create backup option, but it should be noted that it has its down side as well. For example, even it doesn't apply to new files that haven't been written to disk... You still have to save in order for the backup file to be generated/updated. Further, it's always one 'save' behind, so if you go for an extended period without saving you're still going to lose a fair amount of work. That also means that if you close the file without saving, the backup will not contain any recent changes. It also means that at least twice the amount of disk space is required because there will always be at least 2 copies of the file.

I don't mean to preach :-) but the aphorism is as valid today as it was 30 years ago... "Save early, save often". Once the file is named & saved, all it takes is an occasional Cmd+S to keep it current as you work.
