bill manville say:

AutoFit doesn't work with merged cells.

 

The technique I use to autofit in such circumstances is to use a macro that sets a single cell to the width of the merged cells, copies the text to it and does an autofit on the row containing that cell. Then the macro sets the height of the original row.

 

Here's such a procedure

 

Sub SetRowHeights(Sh As Object)
' sets row heights in sheet Sh.
' Excel doesn't correctly set row height when merged cells have wrapped text
  Dim C As Range, rRow As Range
  Dim sHeight As Single
  Dim sBestHeight As Single
  Dim bUpdate As Boolean
  Dim bHid As Boolean
  Dim iHidCol As Integer
  Dim cSizer As Range
 
  ' switch off screen updating to speed up the process
  bUpdate = Application.ScreenUpdating
  Application.ScreenUpdating = False
 
 
  ' this process is only relevant to worksheets, not chart sheets
  If TypeName(Sh) = "Worksheet" Then
    If IsNull(Sh.UsedRange.WrapText) Or Sh.UsedRange.WrapText Then
      ' text wrapping done in some cells in the sheet
      Workbooks.Add xlWorksheet ' temporary workbook
      Set cSizer = Range("A1")  ' a cell to use as workspace
     
      For Each rRow In Sh.UsedRange.Rows
        If IsNull(rRow.WrapText) Or rRow.WrapText Then
          ' there are cells on this row with wrapped text
          If Not IsNull(rRow.MergeCells) Then
            ' no merged cells so can use Excel's autofit
            rRow.EntireRow.AutoFit
          Else
            ' row has merged cells and wrapped text
            sBestHeight = 12.75
            For Each C In rRow.Cells
              ' copy the content of the cell to a spare cell in Terms and Autofit there
              If C.Address = C.MergeArea.Range("A1").Address _
                   And C.WrapText And Not C.EntireColumn.Hidden Then
                ' first of a merged cell, or a single cell, with wrapped text
                ' and column not hidden
                ' set the single cell in Terms to match the (merged) cell here
                cSizer.Value = C.Text
                cSizer.Font.Size = C.Font.Size
                cSizer.Font.Bold = C.Font.Bold
                ' Width is measured in Twips and we can find the width of the MergeArea
                ' but we can only set the ColumnWidth which is measured in different units
                ' so scale the Width appropriately
                cSizer.EntireColumn.ColumnWidth = C.MergeArea.Width * cSizer.ColumnWidth / cSizer.Width
                cSizer.WrapText = True
                ' use AutoFit to find the right row height for this cell
                cSizer.EntireRow.AutoFit
                ' get the height
                sHeight = cSizer.RowHeight
                ' if the cell is merged vertically then we need less height than this
                If C.MergeArea.Rows.Count > 1 Then
                  ' adjust height down for later rows
                  sHeight = sHeight - (C.MergeArea.Rows.Count - 1) * (C.Font.Size + 2.75)
                End If
              Else
                sHeight = C.Font.Size + 2.75
              End If
              ' take the greatest height for this row so far
              If sHeight > sBestHeight Then sBestHeight = sHeight
            Next
            ' if the row isn't the correct height
            If rRow.EntireRow.RowHeight <> sBestHeight Then
              ' set it to the correct height
              rRow.EntireRow.RowHeight = sBestHeight
            End If
          End If
        End If
      Next
      ' close the helper workbook
      ActiveWorkbook.Close False
    End If
  End If
  ' restore screenupdating to its previous state
  Application.ScreenUpdating = bUpdate
End Sub

Current Data

Row 1, header, includes employee name, then a column for each class name (Orientation, Safety, Harassment, CPR, etc - total of 10 different classes)

tom ogilvy say:

Sheet2:

 

A2:  =OFFSET(Sheet1!$A$1,TRUNC((ROW()-2)/10)+1,0)

B2:  =OFFSET(Sheet1!$A$1,0,MOD(ROW()-2,10)+1)

C2: =IF(OFFSET(Sheet1!$A$1,TRUNC((ROW()-2)/10)+1,MOD(ROW()-2,10)+1)="",NA(),OFFSET(Sheet1!$A$1,TRUNC((ROW()-2)/10)+1,MOD(ROW()-2,10)+1))

 

then format C2 to display a date. 

 

select A2:C2 and drag fill down to row 5000

 

 Now select columns A:C and do a copy, then paste Values (choose paste values from the paste dropdown in the wizard).  This replaces the formula with the data they show.

 

Now go to column C and hit F5, choose special, choose constants and uncheck everything except Errors.  Click OK.

 

this select all those  #N/A  cells in column C where the person has not attended that class.

 

Now in the home tab, go to  the Delete dropdown in the Cells command button and choose  Delete Sheet Rows.

 

That should give you your list.

If the normal template shortcut has an exclamation mark then it is normal.dotm from Word 2007 or 2010 and merely implies that it is a macro enabled template.
You will not be able to delete the file if any Word processes are running. Start by closing both Word and Outlook, then check the Windows Task manager (CTRL+ALT+DEL) to ensure that there are no winword.exe processes still running. If there are, end them.

If there are orphan lock files present, you may still not be able to delete the file - seehttp://www.gmayor.com/what_to_do_when_word_crashes.htm

Note that you should never delete the normal template in the first instance, but RENAME it. That way if you screw something up you can always get your personalisations from it.

Ensure that you have the Word option set to prompt to save changes to the normal template when closing Word, so that you are in control of what is saved to that template, and check the option to always save backups, to enable you to quickly recover from such problems.

Sub insertSymbol()

Dim i As Integer
If ActiveWindow.Selection.Type = ppSelectionText Then

rich007a say:

Hi John, and thanks, but alas that falls into the same issues I was facing when I reached for help here... if you run your code, then type some text,  that new text takes on both the font and the font colour of the symbol.  I need the new font (an the tab) to have the same font and font colour of that the insertion point had before the macro was run.  (FYI, I noticed you edited your post - your first version is what was e-mailed to me - that did the same thing).

But.... taking your suggestion of .InsertBefore and .InsertAfter, I came up with the following:

Sub insertSymbol2()
    Dim otxR As TextRange
    Dim lnParaNum As Long
   
    If ActiveWindow.Selection.Type = ppSelectionText Then
        Set otxR = ActiveWindow.Selection.TextRange
       
        With ActiveWindow.Selection
             'Get paragraph number at start of selection:
            lnParaNum = .ShapeRange.TextFrame.TextRange _
                .Characters(0, .TextRange.start).Paragraphs.Count
            
             'First replace selected text with the tab:
            otxR.Text = vbTab
            
             'Then insert the symbol:
            With otxR.InsertBefore(Chr(149))
                .Font.Name = "Narkisim"
                .Font.Color.RGB = RGB(150, 150, 150)
            End With
           
             'Test if selection starts at start of a paragraph:
            If .TextRange.start = .TextRange.Paragraphs(lnParaNum).start Then
                
                 'Offset insertion point to start of third character in current paragraph:
                .TextRange.Paragraphs(lnParaNum, 1).Characters(3, 0).Select
               
            End If
        End With
    End If
End Sub

 

Note I found that if the insertion point is at the start of a paragraph, .InsertBefore did not leave the inserion point after the tab, hence the test at the end for this.  Credit to ZOW for his post here: http://www.vbaexpress.com/forum/archive/index.php/t-32289.html

So thanks for the nudge in the right direction.  Now for the next question (in a new post).

Rich
