Bug: Media Center update running, failing, despite no Media Center or mcupdate

SUMMARY

On my Windows 10 Pro 64 bit PC, the task scheduler tries to update Media Center, even though Media Center is not installed on my PC.  It does so by trying to run $SystemRoot\ehome\mcupdate which also doesn't exist, therefore the process fails.  Other side effects are that my PC may wake up in the process.  The update runs in Windows 7 mode, so this bug may be present in Windows 8.1 & 8 if they were upgraded from Windows 7, as well as other versions of Windows 10.

DETAILS

My Windows 10 64 bit PC started off with Windows 7 Home, which had Media Center.  I upgraded it to Windows 8 Pro, but did not get the Media Center addon.  I then upgraded to 8.1, then 8.1 update, and now have Windows 10 installed and fully up to date.

Every so often I notice that my PC is on despite putting it to sleep.  Usually when looking at the reason, I find something like: 

>powercfg /lastwake
Wake History Count - 1
Wake History [0]
  Wake Source Count - 1
  Wake Source [0]
    Type: Wake Timer
    Owner: [SERVICE] \Device\HarddiskVolume2\Windows\System32\svchost.exe (Syste
mEventsBroker)
    Owner Supplied Reason: Windows will execute 'NT TASK\Microsoft\Windows\Media
 Center\mcupdate_scheduled' scheduled task that requested waking the computer.

When looking at task scheduler, the mcupdate_scheduled task is triggered at logon and once daily.  It tries to run $SystemRoot\ehome\mcupdate.  However, that program is not present in C:\Windows\ehome.  There are lots of failures under history.

The scheduled task is configured for Windows 7, Windows Server 2008 R2, so maybe this task dates back to when the computer was running Windows 7.

A workaround is to disable both triggers.  Even the At log on trigger can wake the PC if the PC is put to sleep after logon but before mcupdate_scheduled is run (it is set to run after a delay).

spectre-4 say:
Hi,
There is a very limited option of doing this in Windows. If you go to Settings > System > Storage > Change where new content is saved. If you change your apps option here any Windows 10 UWP based apps. Namely those installed from the store will install here. However non Windows 10 apps will need to have their install locations changed manually when installing each one. There is not currently a global setting to ensure a location for all programs. So a per program basis is your only option.
I hope that answers your question.
