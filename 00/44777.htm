Should I compress a drive? - Microsoft Community

cynara4 say:

Should I compress a drive?

Original Title: compress drive__

Should I compress a drive?

 

ken blake - microsoft mvp say:

On Fri, 4 Feb 2011 17:00:19 +0000, cynara4 wrote:

Should I compress a drive?

I am almost always against drive compression. It puts all your eggs in
one basket, so that a problem anywhere on the drive can make
everything on that drive inaccessible.

A compressed drive requires that it be decompressed every time it's
read and compressed every time its written. That makes access to it
slower. But on the other hand it makes files smaller, and the drive
can read and write smaller files more quickly. Which is more
significant depends on the relative speeds of the CPU and the drive,
but in most cases these days, any difference is probably small.

Are you short of disk space? If not, don't even consider drive
compression. If so, I strongly suggest that, instead of compression,
you replace your drive with a bigger one, or buy a second drive.
Fortunately hard drives are very inexpensive these days.

Ken Blake (MS MVP)

jetta48 say:

Main advantage of compressing drive is saving space on drive. Main diasdvantage is that it will significantly slow access times for the drive as each file has to decompressed/recompressed eac time it read/written.

When you consider how much access your boot drive (usually C:) gets I therefore would strongly advise against the wholesale compression of C:. However,the compression of a lesser used drive (say an external hard disk or a non-boot partition) is more realistic. You could also consider compressing individual folders that contain a lot of your personal data - e.g. My Pictures.

Personally, I don't compress anything in the manner you suggest.

Save the new text file with a .reg extension to your desktop or a folder with a name you can remember, something like:

addnetman.reg

After saving the file, close notepad.

Locate the addnetman.reg file on your desktop and double click it.  If you have 'Hide extensions for known file types' enabled, you may only see the file listed as 'addnetman' without the .reg extension (that is okay).

If you double click the file and the file opens in something like Notepad or WordPad, close that and then right click the file and choose 'Open With...  Registry Editor' and then import the file.

Alternatively, you can right click the addnetman.reg file, choose Open With... and select the Registry Editor which will then import the file.

Alternatively, you can right click the addnetman.reg file, choose Merge which will then import the file.

Respond in the affirmative to the question... are you sure you want to add the information to the registry?

You should then see a message that the information was successfully entered into the registry.

Then register some DLLs.  Not all DLLs are registerable (pay attention Microsoft enagaged Support Engineer "experts) and it does not hurt to reregister a DLL that is already registered.

Click Start, Run and in the box enter the following commands one at a time pressing the Enter key after each command:

regsvr32  netshell.dll   (message: DLLRegister Server succeeded)
regsvr32  netcfgx.dll   (message: DLLRegister Server succeeded)
regsvr32  netman.dll   (message: DLLRegister Server succeeded)

I would reboot at this point to make sure the updates "stick".

Now you should see the Network Connections Service and you can right click it, choose Properties and (if it is not already) set the Startup type to Manual and start the service if it is not already running after the reboot.
