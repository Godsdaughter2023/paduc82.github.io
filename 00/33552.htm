sjiveru say:

The stacking function is provided by an input method editor program (IME), not by your keyboard layout. The keyboard layout outputs one jamo at a time, and the IME then takes that and converts in to whole blocks. So you have to find a way to send your layout's output through a Korean IME before it will properly combine jamo.

I'm not sure if there's a way to do this. Microsoft seems to treat layouts and IMEs together, and whether you can use a different input may depend on how the IME is written. I feel like some time ago I accidentally ended up changing the underlying layout for my Japanese IME, but I have no memory of how this happened; the Google IME provides a lot of settings, but I don't see anything in there about layout. I don't know if Korean IMEs have much in the way of user settings.

Alternatively, you can probably create an AutoHotKey script to take individual jamo and combine them automatically. You could also potentially abuse MSKLC's dead key functionality to create similar combining behaviour; but I'm not sure that MSKLC can handle dead key sequences involving three or more keystrokes.
