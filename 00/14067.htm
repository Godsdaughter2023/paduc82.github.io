hansv mvp say:

One way to do it:

  • Open the query in design view.
  • Use the View dropdown on the left hand of the Design tab of the ribbon to switch to SQL view.
  • Copy/paste the SQL text that I posted into the query window, replacing what's there.
  • Switch back to design view.

Another way:

  • On the Create tab of the ribbon, click Query Design.
  • Add tblContacts and tblContactNames, then close the Show Table window.
  • Drag FirstName from tblContacts and drop it onto FirstName in tblContactNames to create a join.
  • Drag LastName from tblContacts and drop it onto LastName in tblContactNames to create another join.
  • In the Query Type group of the Design tab of the ribbon, click to highlight Update.
  • This adds a new row 'Update to' in the query design grid.
  • Add the ContactNameID field from tblContacts to the query grid.
  • In the 'Update to' row of the ContactNameID column, enter tblContactNames.ContactID

Whichever way you used, you can now run the query by clicking the Run button on the Design tab of the ribbon.

This begs the questions:

  1. can I or can I not display animated GIFS in Outlook 2016 with new @outlook.com accounts?
  2. If I can't then what is the point of the GIPHY add in?

Just to add a bit of an explanation to round out <Suvana Govinda>'s response.

#1 - Any type of inserted animation (whether GIF or video) hasn't been supported in the Outlook <desktop email client> since OL'2007. To be more specific, animations are not displayed when viewing a message in the desktop client - doesn't matter what the <email account> is - whether Outlook.com or any other. Animated GIFs simply appear as a static image at best with a message. Outlook uses MS Word HTML engine to render render messages which is at the root of the issue.

#2 - The GIPHY addin for Outlook (desktop) is a search tool to insert curated animated GIFS into a message BUT that doesn't mean it will display as an animated gif within Outlook (desktop) - either yours or any recipient using Outlook. It will however show correctly in any email client that supports animated gifs or when viewed in a browser.
