soldiercide say:

By rent, I assume you mean through Nitrado; so, that's the assumption I'm going to operate off of.

First, let's talk Ark for a moment.  Ark is not fully optimized for console, or Windows 10 for that matter, and there have been issues with dashboarding for years.  If they are playing on an original Xbox One, the dashboarding issues is going to be worse.  This is not to say that is the root of the problem here, but it is a contributing factor.  I play on a One X and have only been dashboarded when I does something that exhausts the system resources.

Second, if you are running any server customization that increases the load, you are going to cause players on the One more issues.  Customizations that increase gathering rate substantially, without optimizing the gather rate through clamping and/or individual multipliers, are a major contributor to performance issues.  If you have increased the spawn rate of dinos, this could also be a contributing factor.  If you have substantially increased maturation and/or egg laying rates, those things can also contribute to performance issues.  Without knowing what's in your .ini files, I can't say for certain.  These are areas I would look at, however, as they do increase load on the client side.

Third, and perhaps more likely if they are playing on the same network, this is a connection issue.  Seems rather odd that connection issues will cause you to dashboard, but this is Ark.  If they are playing at the same time, and on the same network, there may be a traffic issue, particularly if they are using an older router (or a nasty gateway).  Multiple consoles connected to the same network has been an issue since the 360 days.  I have a router that can distinguish the traffic, and so I don't have any issues with the three consoles on my network.  Older, dumb, routers don't do so well with this.  The most consistent workaround I have found over the years is to port forward one of the consoles.  Someone might advise you to put one of the consoles in the DMZ, but you're just asking for trouble if you do that.  

Fourth, the most tried and true method for correcting Ark issues is a hard boot:

Hold down the power button until the console shuts down

Unplug the console for 10-15 seconds

Plug it back in and boot it back up

Ark has a rather interesting file and connection structure, and a host of things can cause performance issues in the game.  If you would like to respond with more details, I'll be happy to see if I can provide a more specific answer.

¡firedog say:

Storage folders are intended as local folders for saving stuff that you can't or don't want to save on the server. If you now want to save it on the server, you'll have to move (or copy) messages back to folders under the account. This will only work if the account is set up to use a synchronizing protocol - DeltaSync or IMAP. DeltaSync is the default for outlook.com accounts in Windows Live Mail, so unless you did something special when setting up the account, it will be using DeltaSync. Note that you can't move folders from Storage folders to an account - only messages.

Give the program plenty of time to adapt to the changes when you move or copy large numbers of messages. They'll all have to be re-indexed, and that will take time. It might be best to Work offline before starting, so that scheduled sync operations don't interfere with what you're doing. When you have some folders under the account ready to synchronize, click Send/Receive. It might take several sync cycles to get everything uploaded to the server again.

One the messages are back on outlook.com, they should synchronize with your iPhone if it's using an Outlook.com app or IMAP to connect.
