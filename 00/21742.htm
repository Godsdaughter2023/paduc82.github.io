Image size cannot be directly specified.  It is determined based on the original size of the image you upload, how it is cropped, use of focus points on it, the style/layout selected and the surrounding content, and the emphasis state of it.  The logo feature, however, is meant to be a small image at the top of your Sway and its size can't be affected much.  If you're going to use it, the best bet is to make your logo image as close to a square as possible to maximize its size (if the logo itself isn't square, fill in the image with transparent pixels to make it into a square to get it a little bigger).

We have definitely heard that type of feedback, Rich, and we are considering allowing the emphasis setting to affect logo images.  Just to give you some background, one of Sway's value propositions, is that Sways will have a great output no matter what device/screen size the reader views it on.  In order to accomplish that, anytime someone opens a Sway, a layout tailored to their environment is generated and content is resized and reoriented as necessary (you can see an example of this by viewing your Sway full screen on a laptop and then sizing the window down to a mobile phone size).  Instead of having authors specify things like size and positioning of elements, instead they specify the order of their content and give hints about what they want from their content.  It would far less flexible if specific sizes, for instance, had to be respected.  We realize this is definitely a change from what authors expect and that for projects requiring a lot of pixel perfect positioning other tools like PowerPoint might be a better choice.  Definitely keep the feedback coming and we'll do our best to improve the product!

There is probably a way to do this with Data Validation, but here is VBA code for the Worksheet_Change event in the worksheet module:

 

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Range("D1"), Target) Is Nothing Then
        If Not IsDate(Range("D1").Value) And Range("D1").Value <> "TBC" Then
            Application.EnableEvents = False
            MsgBox "D1 must either be a valid date or TBC", vbExclamation
            Range("D1").ClearContents
            Range("D1").Select
            Application.EnableEvents = True
        End If
    End If
End Sub

It sounds like you still have Data Validation turned on for those cells. The idea behind Hans' code is for it to take the place of Excel's Data Validation. Try clearing the Data Validation and see if that will make his code work for you.

Also, you may want to try this variation on Hans' code...

Private Sub Worksheet_Change(ByVal Target As Range)
     Dim cel As Range, BadCells As String, BadCount As Long
     If Not Intersect(Range("M6:M22"), Target) Is Nothing Then
         Application.EnableEvents = False
         For Each cel In Intersect(Range("M6:M22"), Target)
            If Not IsDate(cel.Value) And cel.Value <> "TBC" And Len(cel.Value) > 0 Then
                BadCells = BadCells & "," & cel.Address(0, 0)
                BadCount = BadCount + 1
            End If
         Next cel
         If BadCount Then
             MsgBox "Cell must either be a valid date or TBC", vbExclamation
             With Range(Mid(BadCells, 2))
                 .ClearContents
                 .Select
             End With
         End If
         Application.EnableEvents = True
     End If
 End Sub

While there is no set way to handle the validation you want for multiple cells (occurs when you paste several values into the range all at the same time), I thought the choice Hans made to stop the loop at the first bad value (thus leaving any bad data in the cells below the first bad data) might be improved upon. The code I posted above will actually validate all valid values and then alert you to those that are bad... you will get a MessageBox telling there are bad cells and then, when you dismiss the MessageBox, those cells with bad data will be cleared and selected so you can address the problem. I also added code to allow you to clear cells without popping up the warning message.

EDIT NOTE

Just to add to the above, there is a limit on how many bad cells the code will handle (the variable containing the addresses I accumulate has a 256 character maximum length) and I have not included any code to handle your having so many bad values being pasted in all-at-once as to exceed that limit... I am basically trusting you would never have a "ton of bad values" being pasted in all at one time. More than likely, you will only be putting one value at-a-time into the range, so you would never exceed the maximum; but, in case you think it is a possibility, then let me know and I'll try to figure out a way to account for that possibility.

girish m say:

You may add a New Column field ‘Timezone’ for the contacts to the Phone view in Outlook and then add the timezone for an individual recipient.

 

To add a column to a view, do the following:

1. On the View tab, in the Current View group, click View Settings.

2. Click Columns.

3. In the Available columns list, click the field ‘User-defined fields in folder’, and then click New Column.

4. Type in ‘Timezone’ and click OK. Make sure Type and Format is set to TEXT.

 

Once the above steps are done, then you may click the space created for Timezone next to the individual contact and enter the information.

 

You can also access the information and edit the same from within a contact once the above steps are completed. To do that, open an existing or a new contact, click on All Fields in the Show group under Contact tab. In the Select from, select User-defined fields in folder.

john mcghie say:
Hi Thomas:

It's not exactly Word that is circumventing the key processing, it is Apple OS X.

The Mac has a very different key mapping to Windows.  The key codes returned from the keyboard are not the same.

If you are developing code to sell to customers, you should abandon any thought of distributing keystrokes.  These are user-customisable, and vary from language to language.  You will never have any idea which keys are free, and your customers will not be happy if you or their Operating System conflicts with their key assignments.

Instead, create a toolbar or a user form.  A tool bar is easier.  Put your macros on that.  Your customers can then assign their own keystrokes if they wish, and they can be responsible for the complaints :-)

Cheers
