cybermaxx360 say:

Only way to get around it for now that I know of - don't direct connect to your cable modem but instead place a router capable of working with IPV4 and IPV6 and then change the DNS settings for the WAN ports there.

I have comcast myself and yes their IPV6 DNS servers are not ready for prime time.  Test performed using theirs mostly fail, while using googles Public iPV6 DNS servers always pass.

So, what I did - though the router will receive it's Public IP's (IPV4 and IPV6) and the DNS Servers to use. I instead changed the DNS settings from Auto to Manual - Manually adding Googles DNS Servers for both IPV4 and IPV6 but  leaving the Actual Addressing set to auto for both IPV4 and IPV6.

Thus: Anything DHCPing from my router (which acts as a DNS Forwarder) will receive and IP Address for my network along with the Routers IP Address as the DNS Server. Noted on my XBOX ONE that my Primary DNS Server is in an IPV6 Format and it's set to my Routers Local IP Address.  So what happens now AND DNS Requests will now pass DNS thru the router to what it was told to use as DNS servers which are now Googles Public DNS servers for both IPV4 and IPV6.

Problem solved.
