Colors fade when saving Powerpoint PPTX as PDF on iMac OSX Mojave - Microsoft Community

john runions say:

Colors fade when saving Powerpoint PPTX as PDF on iMac OSX Mojave

I've just had an epiphany. This is in response to an older locked thread reporting this issue. The color fading when exporting a Powerpoint .pptx file to PDF using Mac OSX Mojave was driving me crazy. It appeared similar to the way RGB files become apparently desaturated when converted to CMYK.

It turns out it wasn't an issue of color-space conversion or computer display color space at all. Rather, the Export as... PDF (on the print as PDF, or the Save as... PDF) machinery doesn't appear to handle images that have been made transparent using the 'Transparancy' tool in Powerpoint. I only realised this when I noticed that not all of the images in the original pptx appeared desaturated.

Solution, use Photoshop or comparable image editing program to make images appear transparent by, e.g. increasing 'Lightness' using the Image > Adjust > Hue and Saturation tool. Save the resulting image and place that into the Powerpoint file. Upon export to PDF, it looks perfect.

petercon [msft] say:

The "override Windows display language" setting does not affect anything besides the Windows shell UI. The language in which Store apps display is determined by the ordering of languages in your language settings (it is a priority list) and the set of languages into which the app has been translated.

Start by checking your language settings to ensure that the languages are ordered with Japanese as the first one (if Japanese is the language you would like the apps to use).

If they are ordered that way and the app doesn't come up in that language (you may need to restart the app --- but you don't need to reboot or log off/on), then the problem could be that the app's language package has not yet downloaded from the Store. So, the next thing to try is to check for app updates in the Store app. (Click on your user time next to the search box to get to the updates view.)

If the app is up-to-date and it still doesn't display in the first language in your list, then that would suggest that the app has not yet been translated.

Btw, the only time anyone should ever need to set the "override Windows display language" setting is if you want the Windows shell UI to display in a different language from your apps _and_ the Windows language pack for the language you prefer apps to be in has been installed on the machine. (App language packs install automatically, but Windows language packs require some user on the machine to manually request a language pack.)

jeeped say:
If you are returning a number from B!J:J, you can use SUMPRODUCT() to compare your text criteria.

     

The formula in Z:H2 is,
=SUMPRODUCT((B!A$2:A$9999=A!$E2)*(B!H$2:H$9999=A!$A2)*(B!J$2:J$9999))

I had to change some data around to provide at least one match, but you should be able to fill down with that formula as far as necessary. Note: this works well because there is only one row that matches both criteria. If more than one row matched, the result would be the sum of the results from column J.
