Driver verifier
Using Driver Verifier is an iffy proposition. Most times it'll crash and it'll tell you what the driver is.

*But sometimes it'll crash and won't tell you the driver.
*Other times it'll crash before you can log in to Windows. If you can't get to Safe Mode, then you'll have to resort to offline editing of the registry to disable Driver Verifier.
*I'd suggest that you first backup your data and then make sure you've got access to another computer so you can contact us if problems arise.
*Then make a System Restore point (so you can restore the system using the Vista/Win7 Startup Repair feature).

*In Windows 7 you can make a Startup Repair disk by going to Start....All Programs...Maintenance...Create a System Repair Disc - with Windows Vista you'll have to use your installation disk or the "Repair your computer" option at the top of the Safe Mode menu .

Here is the procedure:
- Go to Start and type in "verifier" (without the quotes) and press Enter
- Select "Create custom settings (for code developers)" and click "Next"
- Select "Select individual settings from a full list" and click "Next"
- Select everything EXCEPT FOR "Low Resource Simulation" and click "Next"
- Select "Select driver names from a list" and click "Next"
*Then select all drivers NOT provided by Microsoft and click "Next"
- Select "Finish" on the next page.
*Reboot the system and wait for it to crash to the Blue Screen.
*Continue to use your system normally, and if you know what causes the crash, do that repeatedly. The objective here is to get the system to crash because Driver Verifier is stressing the drivers out. If it doesn't crash for you, then let it run for at least 36 hours of continuous operation.
*If you can't get into Windows because it crashes too soon, try it in Safe Mode.
*If you can't get into Safe Mode, try using System Restore from your installation DVD to set the system back to the previous restore point that you created.

lemp say:

If the statement in your initial question is accurate --

For awhile if I turned off the computer with it plugged in and charging it will have correct time when turned on the next day. Now recently it loses about 5 hours when turned on.

then I suspect that something other than the CMOS battery is the problem.  The function of that battery is to keep the "Real Time Clock" running while the computer is powered OFF.  When the computer boots up, the time is retrieved from the RTC and time is then kept by Windows using the CPU clock and periodic syncs to an Internet time source.

If your computer is losing time while it is turned on ("it loses about 5 hours when turned on"), the problem is probably not the RTC or the CMOS battery that powers it.

I believe that the "Apply" button is not significant on the "Date and Time" dialog box.  The time resets when you click OK, doesn't it?

You wrote that the time "loses about 5 hours."  Does the time gradually get out of sync or does it abruptly jump to some incorrect value?

Unfortunately, clock problems that occur while the system is powered on are not easy to diagnose or fix.  You can try the following, which have helped in some situations:

  1. Enter the BIOS configuration utility and if there is a time display there, determine if that clock shows the correct time (assuming that your BIOS configuration does have a time display, that is the clock that is run by the CMOS battery).  If the BIOS clock isn't working, the problem is likely to be a bad motherboard.
  2. You can run a check for malware using MalwareBytes AntiMalware (you don't need to buy the Premium version) and the ESET on-line scanner
    https://www.malwarebytes.com/mwb-download/
    http://www.eset.com/us/online-scanner/
  3. You can check to see that you're correctly synchronizing the clock to an Internet time source.
    • Click on the clock and go to "Change date and time settings"
    • Select the "Internet time" tab
    • Click "Change settings"
    • Click "Update now" -- If the time doesn't update properly, chose a different time server
  4. If sync'ing to the Internet time server worked properly, you can configure the update to occur more often than the default once/week.  Don't do it more frequently than every 4 hours or so or you may get banned by the time server.  See https://www.pretentiousname.com/timesync/ for directions
  5. As a last ditch method, you may attempt to clear the CMOS.  This has been reported to work for some desktop motherboards.  I have no definitive information about how to clear the CMOS on a Toshiba netbook (or if that's even possible) or whether that would help your symptoms.  The following YouTube video purports to show how to clear a BIOS password on a Toshiba NB500 netbook.  That may or may not apply to your netbook, and even if it does, it may or may not have any effect on your clock symptoms.  On the other hand, it shouldn't harm anything, either (at least if you're careful).  https://www.youtube.com/watch?v=P516wdYnlYI

Good luck!

Just getting back to this, can't believe its a year old ...

Anyway, if someone comes across this problem, I have solved this; or rather “got it to work”.

 

To reiterate, I have a netbook (Gateway brand) with built in Wireless WAN (Air Card).  The service is through Verizon wireless.  The Wireless WAN card has 3G capability, so the speed is decent.  What I wanted was to share this internet connection; basically turn the netbook into a wifi hotspot.

 

Basically what I did was go through the windows setup to create wireless network.  So I created my own wireless network using my wifi card.  I also set up a password.  Next I went to my other laptop (Toshiba brand) and checked for wireless networks.  I saw the network I created, and connected to it.  I also made sure they were in the same workgroup.  Now both laptops could “see” each other and share files.

 

After that I went through the process where I enabled sharing with the Wireless WAN card.  You can find this on the card’s properties window.  I picked the wireless card as the connection to share the internet through.  When you do this, Windows assigns 192.168.0.1 as the IP address of the wifi card.  Subnet mask is 255.255.0.0.  There is no gateway address or DNS address on the wifi card.  To repeat, both of these are blank.  The wifi card just has the address and subnet mask.

 

OK, now I went back to my Toshiba.  For some reason the DHCP server on my netbook wasn’t working.  The result was that the Toshiba had a private IP address.  OK so for the Toshiba I manually entered the IP information.
IP address: 192.168.0.2
Subnet mask 255.255.0.0
Gateway: 192.168.0.1
DNS: 192.168.0.1

 

After that it still wasn’t working, so I went through a process of rebooting and reconnecting.  I never changed the IP information though.  Finally, I turned off the firewall on the netbook.  I disconnected the WLAN from the Verizon network and reconnected.  Then I rebooted the Toshiba.  Success!  Now both the netbook and laptop can get on the internet. 

 

I guess the netbook’s firewall is blocking the Toshiba from accessing the internet.  It sucks because I hate the idea of turning the firewall off.  There is probably a solution to keeping the firewall on and sharing the internet connection, but I haven’t found it yet.

 

And as a side note to anyone else who reads this, it works out best to set the wifi’s IP information as the alternate configuration.  So for my netbook, the IP address is automatic while on the alternate configuration tab I typed in 192.168.0.1 as the IP address and 255.255.0.0 as the Subnet mask.  That way, when I’m not using the Verizon WLAN card to connect to the internet, I can connect to a wireless network with the wifi card.  The wifi card will get the IP information automatically from the wireless network. 

When I connect to another laptop the alternate configuration is used.  For other laptops that I want to share my internet connection with, I just set them up like my Toshiba.  I just go to the other laptop and use the alternate configuration tab to type in the IP information.

 Public Sharing = Public Network (like Internet cafe) to reduce security risks.

 For best results log to each computer's System Screen and set all the computers to be on a Network with the same name while each computer has its own unique name.

http://www.ezlan.net/Win7/net_name.jpg


Make sure that the Software Firewall on each computer allows free local traffic. If you use 3rd party Firewall On, Vista/XP Native Firewall should be Off, and the active Firewall has to adjusted to your Network IP numbers on what is some time called the Trusted Zone (consult your 3rd Party Firewall instructions

General example,  http://www.ezlan.net/faq.html#trusted

Please Note that some 3rd party Software Firewalls keep blocking aspects of Local Traffic even it they are turned Off (disabled). If possible configure the Firewall correctly, or totally uninstall it to allow clean flow of local network traffic. If the 3rd party software is uninstalled or disables, make sure that Windows native Firewall is On .

shawn g - support engineer say: Kaitlyn07,

See if this works for that you are trying to do. 

First, place your footage in your editor’s timeline/

Second, cut the video at any place at least 20 frames from either end.

Next, move the second half so that it is before the first.

Finally, place a transition (something like  cross-dissolve), in between the two clips. You might need to trim the beginnings and ends to use the transition.

Now, since the clip ends with the frame before the frame that it begins with, it will loop seamlessly forever. You’ll see the dissolve (or whatever transition you used), but there will not be a jump cut.
