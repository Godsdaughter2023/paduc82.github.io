ossiemac say:

Hi Rhodze,

 

I have a feeling that if you knew enough to create and run a query then you would not be asking this question. Therefore I wonder if you would like a little more explanation of how to import the Excel data and run a query.

 

The following is the interactive method; not through code and writing SQL; although you could use code to invoke the query and requery it after you have created it.

 

Import Excel Worksheet to an Access Table:

 

·        Open Access project with the table that you want to filter according to the Excel worksheet.

·        Select Ribbon Tab ‘External Data’.

·        Click ‘Excel’ icon in the Import block.

·        Browse to the Excel file and select it.

·        Select either of following:

o  ‘Import the source data into a table in the current database’

o  ‘Link to the data source by creating a linked table’

·        Click OK (May be short delay)

·        Select the required worksheet in the Excel workbook and click Next.

·        Check or Uncheck ‘First row contains column headings’ depending on your data.

·        You may specify more info re the fields or just click Next.

·        Decide if you want Access to add a primary key (not essential) and click Next.

·        Insert Table name. (Defaults to worksheet name) and click Finish.

·        Click close. (Can leave ‘Saving steps’ to another time but well worthwhile learning.)

 

Create a Query:

·        Select ribbon tab ‘Create’.

·        Select ‘Query Design’ icon in Other block.

·        Select your main table and click Add.

·        Select the table created from Excel import and click Add.

·        Close the table select dialog box.

·        Highlight all of the headers in your main table. (Click first one and then hold Shift key and scroll down and click last one.)

·        Drag the selected fields into Field row in matrix.

·        In the Criteria row under the column to be matched insert the following formula:

o  =[Sheet2]![MyCol]    ‘Where Sheet2 is the table name and MyCol is the column name in which to find match.

o  You can set multiple Criteria if other columns also need to match.

·        Save the Query. (Coloured MS button top left and Save.)

·        Run the query. (Red exclamation icon in left block of ribbon.)

 

You can then use the Query name wherever you might otherwise use a table name.

sheenali_msft say:

Hi Stephen,

Some Office apps, like Skype for Business, automatically boot at startup and can sometimes disrupt your installation. It’s just a safety precaution.

Uninstalling Office 2013 before installing Office 2016 is indeed the best practice, as sometimes there can be problems with overlapping Office installations.

For deployment on multiple computers, Microsoft does suggest using GPO. For the GPO perspective, we suggest going to the Group Policy forum for expert help.

There is also a new attribute termed “ForceUpgrade” introduced in Office 2016. This attribute can be combined as part of Config.xml that will force the auto upgrade and install Office 2016 on end users machine.

This attribute can be used only with Office 2016 ODT download links and is an admin process.

Sample Config.XML

Below is a sample config.xml with “ForceUpgrade” attribute.

<Configuration>

<Add SourcePath="\\server01\office" OfficeClientEdition="32" ForceUpgrade="True" Branch="Business">

<Product ID="O365ProPlusRetail" > <Language ID="en-us" /> </Product>

</Add>

</Configuration>
