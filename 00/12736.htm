You can also look at each shape on the slide and test like so:

If oSh.HasTextFrame Then
  If oSh.TextFrame.HasText Then
     ' it's not empty
  Else
     oSh.Delete
  End If ' hastext

john - project say:
themandez47,
First of all, I'm not sure how the Status field is relevant to your calculation. The % Complete field indicates when a task is complete, so as far as I can tell you don't need the complexity of the Status field.

Second, a summary row is not a milestone. By definition, a milestone is a point in time. It has zero duration and is either 0% complete or 100% complete, there is no "partially complete" for a milestone. It sounds like your summary rows are phases, functions, or other grouped collections of tasks or milestones.

Third, what exactly is this "hidden numeric field"?

Fourth, without knowing more about what the tasks in your file are, I can't say if your "weighted averages" methodology will be a valid measurement of anything. It can all sound good on paper, but it may not be a valid metric. Likewise, it is also difficult to say if there is a more effective way to do what you want.

Fifth, you can implement the logic you want but it will require a VBA macro to do so.

Thanks for your help!

A

suzanne s. barnhill say:

If the actual paper size is 7 x 9, then you need to go to the Page Setup dialog (click the dialog launcher in the Page Setup group of the Page Layout tab or double-click at the top of the horizontal ruler) and select the Paper tab. Choose the correct paper size and click Default..., then answer Yes in the ensuing dialog.

If the 7 x 9 size merely represents the document body area (with 1" top margins and 1.25" side margins), then you may just need to change the view. See http://WordFAQs.mvps.org/WhiteSpace.htm.
