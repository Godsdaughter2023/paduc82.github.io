Trusted Root Certificates and Trusted Publishers Missing - Microsoft Community

dan- say:

Trusted Root Certificates and Trusted Publishers Missing

 

This is a fresh install of Windows 7 Professional.  Every website that uses https:, receives certificate error messages.

Legitimate sites such as Yahoo, Google or Microsoft mail signon pages, all receive certificate error messages.

Trying to run a plugin from the HP support website I receive the error message "Windows has blocked this software because it can't verify the publisher."

There are only 8 certificates listed in the Trusted Root Certificates and zero listing in the Trusted Publishers.

Where did all of the Trusted Root Certificates and Trusted Publisher entries go?  More importantly, how do I get them back?

 

dan- say:

It is now evedient to me that I did not provide enough information in the original post.

The time, date and time zone are correct.  Checked and re-checked.

The certificates required for the Microsoft Updates are missing along with almost all of the other certificates.

So, without the required certificates, you can not receive ANY updates from Microsoft Updates.  Unfortunately the ONLY way Microsoft offers Root Certificate Updates for Windows 7 is through Microsoft Updates.  Are you beginning to see the circular reference of it all?

Here is the solution;

Find a Windows computer, XP, Vista or 7,  that has all of its certificates intact.  Run MMC and add the certificates snapin for the local computer.  Browse to the Trusted Root Certificates and find the "GTE CyberTrust" certificates.  Highlight and select all of the GTE certificates and export them to a file.  Copy this file to a flash drive, email it to yourself, etc..

From the broken Windows 7 computer run MMC and add the certificates snapin for the local computer.  Browse to the Trusted Root Certificates and import the cert file that contains the GTE certificates.  Now you can receive Windows Updates and amazingly you will now get all of the missing certificates.
