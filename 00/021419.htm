Hi Jeffery,

I'd like to summarize my answers here so that users who have the same concern could easily get useful information from our thread:

Why a file has the uploaded date as created date?

This is a expected behavior, because the files are not moved to the document library, actually it's a copy of the file been created in the document library.

Can I change the date in the Created column?

No. Based on my test, the date in this column cannot be changed, even with the Full Control permission. Workaround:

1. Create a custom date column;

2. Click Quick Edit to edit the time in the custom date column.

rich michaels - mvp (greatcirclelearning.com) say:
Using your example the "find" would contain the following: <([0-9])???

The "replace" would  contain: \1xyz

Breaking it down on the find. The < character specifies the beginning of the word. Next is a left parenthesis and its closing partner is the right parenthesis. Within them it is saying this is the first item to evaluate and designate it as "item 1". More on that in a moment...  In the brackets [ ] are the range of numerics to allow for the first character in the string. Finally following the right parenthesis, which closes out item 1, is the ??? that represents any three characters.

On the replace side the \1 command is saying take "item 1" from the Find and then the "xyz" is your replacement string for the ??? on the find.

Sounds a bit cryptic. Trying to explain it is harder than doing it. But try it out and you'll see the result.

Hope this helps

Use the File>Properties menu in IE to determine which IE security zone your test/dev/production website maps to....

If this is an internal website (LAN) it should be mapped to your intranet zone not the internet zone....

If you are developing from your localhost, you need to manually add localhost to your Intranet sites list.... so that your development environment is using the same security zone as your production environment.

By default, hosts that map to the Internet and Restricted zones will only load (more secure) x64 bit versions of ActiveX controls, while the Intranet and Trusted zones still allow older x32 bit versions of ActiveX controls to load. (eg. Flash/Silverlight/WMP/QuickTime)

You should turn protected mode back on, and fix up your IE security zone mappings....

Your development/testing environment and IE settings must match your company's settings.. go Tools>Internet Options>Security tab, click "Reset all zones to default".

first go Tools>Internet Options>Advanced tab, check "Always record developer console messages".
