e.j. gun say:

The exact code you need: is below.  The first part goes into a General Code Module.  If you don't already have one, go into the Visual Basic Editor (VBE), and select Insert/Module.  You will see something like "Module1" appear.  Double-click on it and enter the first code below.

 

The second part goes into the code module for the specific worksheet you are trying to monitor.  Look for that worksheet's name in the list shown by the VBE.  Double-click on it and the code pane shows up.  Paste the code below into the worksheet module.

 

Now, whenever you change either $A$1 or $A$2 on that worksheet, if $A$1 is greater than $A$2, your sound will play.

 

HTH,

Eric

 

 

In a General code Module:

'===== BEGIN CODE =====

Option Explicit
'
' Code below from:
' http://stackoverflow.com/questions/9443801/how-to-play-a-wave-file-or-sound-file-using-vba
'
Public Declare Function sndPlaySound32 _
    Lib "winmm.dll" _
    Alias "sndPlaySoundA" ( _
        ByVal lpszSoundName As String, _
        ByVal uFlags As Long) As Long
'
Sub PlayTheSound(ByVal WhatSound As String)
    If Dir(WhatSound, vbNormal) = "" Then
        ' WhatSound is not a file. Get the file named by
        ' WhatSound from the Windows\Media directory.
        WhatSound = Environ("SystemRoot") & "\Media\" & WhatSound
        If InStr(1, WhatSound, ".") = 0 Then
            ' if WhatSound does not have a .wav extension,
            ' add one.
            WhatSound = WhatSound & ".wav"
        End If
        If Dir(WhatSound, vbNormal) = vbNullString Then
            Beep            ' Can't find the file. Do a simple Beep.
            Exit Sub
        End If
    Else
        ' WhatSound is a file. Use it.
    End If

    sndPlaySound32 WhatSound, 0&    ' Finally, play the sound.
End Sub

'===== END CODE =====

 

In the specific Worksheet module for the Worksheet you are monitoring:

'===== BEGIN CODE =====

Option Explicit

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim myRangeOfInterest As Range
    Dim mySoundFile As String
'
    mySoundFile = "C:\Documents and Settings\Jack\My Documents\Alert.wma"
'
' The code below is what determines if a specific cell (or range of cells)
' has changed, and then takes action.
'
    Set myRangeOfInterest = Me.Range("$A$1:$A$2")   ' Define a range that we want to watch
'
' Now see if any of the cells in that range changed...
'
    If (Not Intersect(Target, myRangeOfInterest) Is Nothing) Then
        If (myRangeOfInterest.Cells(1, 1) > myRangeOfInterest.Cells(2, 1)) Then
            PlayTheSound (mySoundFile)  ' Should include the path and file name
        End If
    End If
End Sub

'===== END CODE

Whilst I am sure that this topic will have been discussed in erudite detail elsewhere, I use my Personal workbook to hold code which I may use across a range of workbooks.

ken sheridan say:

I was thinking that select case might be a good option, but I don't know how to pass the parameter to the report.  :(

Assuming that you are not referencing the controls as parameters in the report's query, the code to filter the report would be along these lines:

 

Dim strCriteria As String

 

strCriteria = "[YourDateColumn] >= #" & _

    Format(Me.[txtDateFrom], "yyyy-mm-dd") & _

    "# And [YourDateColumn] < #" & _

    Format(Me.[txtDateTo], "yyyy-mm-dd") & "# +1"

 

Select Case [YourOptionGroup]

    Case 1

    DoCmd.OpenReport, "SomeReport", View:=acViewPreview, WhereCondition:= strCriteria

    Case 2

    DoCmd.OpenReport, "SomeOtherReport", View:=acViewPreview, WhereCondition:= strCriteria

    Case 3

    DoCmd.OpenReport, "YetAnotherReport", View:=acViewPreview, WhereCondition:= strCriteria

    ' and so on

End Select

 

Note how the date range is defined as 'on or later than the from date, and before the date following the to date'.  This is more reliable than a Between….And operation as it caters for any values in the table which might have a date on the final day of the range, but with a non-zero time of day.  Otherwise any such rows would not be returned.

 

Note also how the date parameters are formatted in the ISO standard for date notation of YYYY-MM-DD to ensure international unambiguity.  You would of course enter them in the form in your local date format.

 

scottgem say:

I was thinking that select case might be a good option, but I don't know how to pass the parameter to the report.  :(

You can pass parameters to filter a report in 2 ways. The first would be to use the WHERE clause of the OpenReport method. Ken's response illustrates this method. The second way is to use criteria in a query that represents the Recordsource of the report. You can reference values in a control on an open form using the syntax:

Forms!formname!controlname

So you might set the criteria to:

BETWEEN Forms!formname!txtStartDate AND Forms!formname!txtEndDate

(note Ken's comments on using BETWEEN for dates). 

Sometimes I use both methods in combination. For example. I will default the Start and End date controls to the Min and Max values in the table. And set criteria in the query. So if the user doesn't change the dates they get all the records. I then use additional criteria to set a filter in the OpenReport method. 

Finally, while you can use an Option group in a situation where you have only a few reports and don't expect to add more, using a combobox is more flexible.
