hansv mvp say:

The "normal" file type for an Excel 2007/2010 workbook is .xlsx. If you need to store macros, you need .xlsm instead of .xlsx.

For very large workbooks, the binary format .xlsb might be more suitable. It loads and saves faster than .xlsx/.xlsm and the file size is a bit smaller. It's not an "open" format though - you cannot manipulate it with other programs, at leasdt not easily.

The .xltx an .xltm formats are templates - you use them to create new workbooks from. If you double-click a template inj Windows Explorer, you get a new workbook based on the template.

ExecuteExcel4Macro("EXTRACT()")' - Microsoft Community

pencil16 say:

ExecuteExcel4Macro("EXTRACT()")'

For several years I have used 'result = ExecuteExcel4Macro("EXTRACT()")' in an Excel VBA macro to extract rows of data from a named range of cells on a worksheet, to several worksheets in the same work book. After installing Excel  2010, the same code will not function, and does not throw an error. The macro worked on all previous versions of Excel. Is there a solution?
pencil16 say:

I was able to stumble through to a solution. As it turns out EXTRACT still works with the older file formats, but doesn't work so well with the 2010 format, or at least not so well with criteria involving dates. AdvanceFilter works well with the 2010 format (with some criteria changes), but not so well with the older formats. I made the changes to the criteria as needed on the 2010 worksheets, then added an If...Then...Else to decided whether to call a new method to use AdvanceFilter, or stick with the ExecuteExcel4Macro("EXTRACT()") method for reports in the older files. It works for now.  At least it will buy me some time to change all the criteria statements.

 

'Update the report to the criteria dates
Sub UpDateReports()
  ActiveSheet.Unprotect
  On Error GoTo ExtractError
  If DefinedLocally(ActiveWorkbook, ActiveSheet.Name, "Extract") _
    And DefinedLocally(ActiveWorkbook, ActiveSheet.Name, "Criteria") Then
    If Left(ActiveWorkbook.Name, 17) = "Inventory Reports" Then
        result = ExecuteExcel4Macro("EXTRACT()")
    Else
        ExtractData ActiveSheet.Name
    End If
  End If
  ActiveSheet.Calculate
  ActiveSheet.Protect
Exit Sub

ExtractError:
MsgBox "Extraction error. Possible causes: Database," & _
   " Extract, or Criteria areas not defined."
End Sub

 

Sub ExtractData(theSheet)
'
    Sheets("INVDBASE").Range("Database").AdvancedFilter _
    Action:=xlFilterCopy, _
    CriteriaRange:=Sheets(theSheet).Range("Criteria"), _
    CopyToRange:=Sheets(theSheet).Range("Extract"), _
    Unique:=True

End Sub

doug robbins - mvp office apps & services (word) say:

It sounds like you have the display of White Space hidden.  There are two ways to deal with it:-

 

  1. Click on File, then Options, then on Display and check the box for "Show white space between pages in Print Layout view"
  2. Move the mouse cursor to the top of the page until it changes to two vertically opposed arrows and a tool-tip "Double-click to show white space" appears and then do that.

The mouse cursor needs to be between the left and right hand edges of the page, NOT on the vertical ruler, which would be where you adjusted the margins.
