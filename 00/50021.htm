frederik long say:

Your command launches a new Command Processor under the admin account. It does not bypass UAC. You can easily find out with this command:

dir > C:\test.xt

The command will work under elevated privileges but will fail with your command.

You get an error with Windows Explorer (and with the Control Panel, which is part of Windows Explorer) because you already have an active session of Windows Explorer, running under the original account. To run Explorer under the Admin account you would have to kill the existing process, then relaunch a new instance.
