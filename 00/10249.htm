Sub Text_to_Split(name As String)
    Dim objRange1 As Range
    Dim objRange2 As Range

    'Initial
    Sheets(name).Select
    Columns("B:CB").ClearContents
    Range("A1").Select
   
    No_Double_Space
  
    'Set up the ranges
    Set objRange1 = Range("A2:A20")
    Set objRange2 = Range("A21:A1000")

    'Do the first parse
    objRange1.TextToColumns _
      Destination:=Range("B2"), _
      DataType:=xlDelimited, _
      Tab:=False, _
      Semicolon:=False, _
      Comma:=False, _
      Space:=True, _
      Other:=True, _
      OtherChar:="-"

    'Do the second parse
    objRange2.TextToColumns _
      Destination:=Range("B21"), _
      DataType:=xlDelimited, _
      OtherChar:="x"
End Sub

If you point to C:\Onedrive\Addins\addinname.xla (say C:\Onedrive is your Onedrive folder location) Excel will automatically change the link to "https://d.docs.live.net/Onedrive/Addins/addinname.xla" and that's the problem, because such address doesn't exist. It basically replaces "C:\" with "https://d.docs.live.net/". So what you'll have to do is:

  1. Find your Onedrive ID, you can read it if you visit Onedrive.com, log in and take a look at the url, you'll see something like "https://onedrive.live.com/?cid=AbCdeFgH12345678", that "AbCdeFgH12345678" is the id you need (without ?cid=, mine is a 16 characters alphanumeric code)
  2. Open your workbook in excel, go to Data/edit links, select your problematic .xla, click on change origin, in the address bar type "https://d.docs.live.net/yourOnedriveID/Addins/" where "yourOnedriveID" is the code you have found in step 1. Click on the arrow next to the address bar, select addinname.xla and click on Open. (this supposes that you have saved your .xla in \Onedrive\Addins\). Now your function should work.
  3. However, everytime you re-open the workbook, you will find that your functions still don't update values and throw #NAMEs. To fix it just go to Data/edit links, select your problematic .xla, click on open origin.

You'll have to repeat this third step every time you open the workbook, unfortunately mine is a workaround, a band aid, rather than a fix... Hopefully though, it's a bit faster than what you have done so far (just click on open origin instead of re-pointing Excel every time). Besides it will actually make it possible for you to work with the same .xla from different devices.

I understand that you have chosen not to use Onedrive anymore, but just in case you ever come back, or anyone on this website is experiencing the same problem, I hope to be helpful.

Try following one by one and see after every step whether it resolves the issue-

1. Sometimes, add-ins can cause problems. To determine, if add-in is a problem start your application in safe mode and see if problem has gone away.

To start in Safe Mode -

Hold CTRL key and click on application icon and don't release the CTRL key till it asks you for Safe mode confirmation.

If yes, then start application normally and disable add-ins one by one and start application again every time you disable an add-in to determine the culprit add-in.

2. Try repairing your office installation (Quick repair is good enough). Try repairing your Office installation. First try Quick Repair (takes 5 - 10 mins) , if that doesn't help then try Online Repair (Takes more than 30 mins but less than 1 hour in general)

https://support.office.com/en-us/article/Repair...

3. Login to Windows with a new user profile and see if problem is resolved.

4. Sometimes, this is caused due to application not being able to access the default printer. Close all office applications and change your default printer through Start > Devices and Printers to another printer. It may be some physical one or One Note or XPS or PDF or anything else. If this is the problem, then you will need to update your printer driver for default printer.

5. Many third party applications also cause the problem like Tuneup Utilities, Abby Finereader, Kaspersky / Avast Anti Virus etc. See if you have any of them installed. Close them one by one and see whether it helps.
