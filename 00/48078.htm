a. user say:

Unless you are intimate with how System Restore works - and if you ever think you might need it to behave, I would stay out of the System Volume Information folder which is normally a hidden folder for your protection.

There are third party tools that will let you manage old Restore Points and delete them to recovery some HDD space, but for the most part, I would just let it take care of itself.

CCleaner is one popular tool:

http://www.piriform.com/CCLEANER

Uninstall CCleaner if you don't like it, but it has several nice features.  Most folks will advise to stay away from the registry cleaner stuff, but I have no problem with it.

If you think your SVI folder is taking up too much room, you can adjust it down to save space (My Computer, Properties, System Restore...) or adjust the settings to only keep a weeks worth (7 days) or 10 days, etc. via a registry edit.

System Restore Points "live" (by default) on a system for 7776000 seconds (90 days) based on the "Time To Live" registry setting, but you can adjust all that stuff if it it your desire.  Old RPs are deleted out the back end to make room for new ones as they are created.

If you have Microsoft Security Essentials (MSE) installed, you will probably get at least one new RP created every day since when MSE updates it creates a new RP that will show up in your SR calendar as Software Distribution Service 3.0 - and ou might have several of those in one day.

XP would like to create a new RP Checkpoint every 86400 seconds (24 hours), but depending on what is happening on your system, you could go days and days without a new RP checkpoint being created, but not having RPs created for days and days doesn't necessarily mean SR is broken. e.g If MSE updates and creates a new RP, that "resets" the 24 hour automatic checkpoint clock.

If you think you just want to get a "fresh start" you can use CCleaner to delete all but the last RP you have or if you disable SR and enable it again, XP will whack all your old RPs for you, then manually make a new RP.

It is prudent to test your SR from end to end one in a while by manually creating a new RP, restarting your system and then immediately seeing if you can restore your system to the RP you just made.

The day you need SR to work is not the day to find out it doesn't.

If your SR is broken, you can fix it.

lemp say:
I agree with everything that Elder L wrote. 

In addition, unless you have mistyped the location of your _Restore{GUID} folder, something strange is going on.

The System Restore folder is supposed to be in C:\System Volume Information\ and NOT in C:\Documents and Settings\All Users.

In any case, System Restore is not designed to be manually "cleaned out." At this point, you most likely have completely broken it, and those restore points you think you need are probably unusable.  Test it if you like:  if you do a System Restore while in normal Windows (i.e., not in Safe Mode), you can always "undo" the most recent restore.

My suggestion is that you open System Properties (e.g., right-click on My Computer and select Properties), click the System Restore tab, check the box to "Turn off System Restore", and click OK.

Then restart your computer.  You can now go back to the System Restore tab, uncheck the box, and configure System Restore to protect your system partition only and set the "Disk space to use" slider to allocate approximately 1 GB.

For more information about System Restore than you ever wanted to know, see http://bertk.mvps.org/

snejka say:

Hi again folks.

I found the solution on the problem.

From studying the eventlog and the help function I learned that two services are involved in the streaming process. Namely the services:

  • Windows Media Player Network Sharing Service
  • Host for UPnP-devices

 

On my laptop and stationary PC - both upgraded form Win Vista to Win 7 Home Premium, the "Windows Media Player Network Sharing Service" was Activated and set to Automatic-delayed.

The service "Host for UPnP-devices" was Deactivated!!!

I Activated "Host for UPnP-devices" and set it to Automatic, restarted the laptop.

Bingo the streaming worked.

I did the same operation on the stationary PC and now the streaming worked.

It looks like that when you do an upgrade to Win 7 the service "Host for UPnP-devices" by a mistake is set to Disable.

 

I hope that somebody else can learn from this experience.

file you wish to edit)
