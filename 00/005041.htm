There is no "windows 2013".

To transfer Office to a new computer you need 3 things:

  • a license type that allows transfers (ie a "retail" or FPP license)
  • your unique 25 character Product key
  • the generic installation file for the right Office bundle

EXACTLY which Office bundle came with your computer?

What do you mean install using an external drive?  If you have a local install version of Office 2013, then you can do a "custom" install that directs only part of the Office installation to an alternate location, like an external drive.  If you have a "Click to Run" installation of Office 2013 or Office 365 then you can't control where it is installed, it only installs on the C: drive.

Did they give you an actual 25 character Product Key, or did they give you a "Product Key Card" that has a 27 character PIN you had to submit to a web site to retrieve your Product Key?

If you have a Product Key Card type license you CANNOT transfer it to a new computer. That type of license is permanently tied to the hardware it is first activated on.  The builder is most likely to have given you a PKC type license.

If you have a "Retail" type license, then it can be transferred to a new computer.

Caption Autonumbering

I have a question about auto numbering captions. I have researched this to no avail. Can someone please help me with an answer?

Is there a way to change the caption auto numbering so that some of the tables reflect "Table 1, Table 2, Table 3, etc.," and some that reflect "Table 1-1, Table 1-2, Table 1-3, etc."? 

Can I create a Heading 0 (zero) with no numbering before the TOC and have the table reflect as "Table 1" or "Table 0-1" than have the other tables after the first chapter reflect Table 1-1?

Any help would be appreciated.

Terry

doug robbins - mvp office apps & services (word) say:

If you toggle on the display of the field codes in the document, you will see that the numbering of a Table caption is made up of the following fields

Table { STYLEREF 1 \s }-{ SEQ Table \* ARABIC \s 1 }

The numeral 1 in the above is a shorthand notation for Heading 1.

In the SEQ field, the \s 1 switch, causes the numbering to restart after an instance of a Heading 1 style.

That is why, after your second Heading 1, the caption is numbered Table 1-1 and not Table 1-2.  If you remove that switch from the caption after the second Heading 1, and you update the fields, that caption will change to Table 1-2

I would suggest that you do not mix the purpose and use of the Heading 1 style and that you should retain that for the numbered headings and use another style for the headings with no numbers.  You can cause those headings to appear in the Table of Contents in the same format as the headings to which Heading 1 is applied by assigning that style to the same level of the Table of Contents.

For your un-numbered Table Captions, you could delete the { STYLEREF 1 \s }- from the captions and change the { SEQ } field to

{ SEQ Tables \* ARABIC }

so that those numbers will belong to a separate sequence.

If you do that, you should not get any surprises when the fields in the document are updated when you go to print it.
