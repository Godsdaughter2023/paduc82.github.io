steve rindsberg say:

This was a fairly common problem with Windows PowerPoint a few years ago, but it seems to have been solved. 

Roaches will leave one apartment when it's fumigated and move into another that hasn't been.  Seems to work that way with software bugs too:  The problem has migrated to Mac PowerPoint.

Certain problems can cause PPT to lose track of images, and once the problem occurs and you get that "relationship id blah blah" error message, there's no way to get the images back.

Sounds like the presentation got corrupted when it was on the Mac.

Strong suggestion:  NEVER open PPT files directly from dropbox or any other cloud storage medium, or from thumbdrives.  ALWAYS copy the file to your local computer's hard drive, into a folder that's NOT synched with anything or automatically backed up.  Then open and edit it.  When done, save back to the computer's HDD, close it in PPT and only then, copy it back to the cloud/thumbdrive.

It's a bit of a pain, but nothing like the trouble it takes to rebuild a corrupted presentation. 

It's also good practice to save to a new file name every so often.  If I'm working on a project over several days, I'll save as SomeNameOrOther_2017-05-26.  Tomorrow, I'll open the file, immediately save as SomeNameOrOther_2017-05-27 and so on.  If I'm making lots of major changes, I'll save more often ... SomeNameOrOther_2017-05-26_001, 002 and so on.
