suzanne s. barnhill say:

It sounds like you're creating a new label definition in the Label Options dialog. You should certainly be able to save that; you'll find it later under Other/Custom (a vendor category that won't appear until you've saved a custom label). For your purposes, however, you should, after creating the custom label definition, click New Document to get a page of these labels. Save that page as your mail merge main document, which you'll reuse each time. Connect it to your data source and insert the merge fields in the first label. Then use Update Labels to propagate your merge fields to the rest of the labels on the page. Preview Results will show you the first page of labels, but you will need to compete the merge (Finish & Merge) to actually generate and print all the labels.

It sounds like you're getting a new data source every time. If it is possible to save this source in the same location, with the same filename, every time, then the connection to the mail merge main document should be retained. Otherwise, you will need to connect to the data source for each merge. This is a nuisance, and, in the case of some of my mail merge main documents, I haven't figured out how to avoid it despite the fact that the source and its location never change, but it is a minor one compared to starting from scratch.

·        Click Start

Once your computer has restarted, again test to see if the problem is still gone. If so, then the services you enabled are good to stay. Repeat the enabling process for half of the remaining disabled services and continue this process until the problem returns or all services are enabled.
