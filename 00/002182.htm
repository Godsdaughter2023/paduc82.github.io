jay freedman say:

I've tested every version of Word from 2003 through 2016. That error occurs in every version from 2007 onward when trying to use a .tif file in a custom watermark. The import works in Word 2003, but it appears Microsoft broke the watermark dialog's connection to the TIFF converter in 2007 and no one has noticed since then -- at least, I couldn't find any reference to it in a Web search. (There is an article here about errors importing .tif files, but it's about Office 2000 and isn't about watermarks.)

There are two workarounds. One is similar to the one you mentioned, inserting a picture anchored in the header -- the shape and fill steps aren't necessary -- with text wrapping Behind Text, and with the brightness set high (about 60% to 70%) and the contrast set low (about -25%). The other is to use a graphics program such as IrfanView to convert the .tif file to .png or .jpg before importing it as a watermark.

Eric,

While we're waiting then -

1     The type of file Outlook works with is either a pst file [for email accounts set up for pop retrieval or for Outlook set up without any email account] or an ost file [for email set up as imap or MSExchange]. 

2     Outlook creates the pst/ost file when it is opened for the first time.  So there should always be one type of file or the other.

3     I do not know what your first actions on opening Outlook for the first time were so I cannot tell you which type of file you should be looking for.  But the error message implies that you have set up / tried to set up IMAP or Exchange retrieval method [email accounts are not of themselves pop / imap / exchange, these are merely choices for how to access the information].  I really do not understand how this error can be appearing both before and after you have reinstalled Outlook - the implication is that exactly the same thing has gone wrong both times whilst you have been opening Outlook for the first time then trying to set up the email account.

4     I do not know which version of Office you are using so I cannot know where you should be looking for this file.  The folder you are looking in %userprofile%\AppData\Local\Microsoft\Outlook is the correct folder for Outlook 2007 [the strange syntax I use for the location is just to allow you to find it whatever the user name is].  You can find the appropriate location for your version of Office by searching Outlook help for configuration files.

5     For Outlook 2007, you can force Outlook to open as if it was its first time without actually reinstalling it.  This can save a lot of time when troubleshooting an email account problem such as yours.  You need to create a shortcut to it and add the switch /firstrun   So in my case I would create a shortcut to Outlook with this as its target

[64 bit Windows]

"C:\Program Files (x86)\Microsoft Office\Office12\OUTLOOK.EXE" /firstrun

or [32 bit Windows]

"C:\Program Files\Microsoft Office\Office12\OUTLOOK.EXE" /firstrun

and I believe that later versions of Outlook have similar capabilities [search Outlook help for command line switches].

6   There is a brief discussion of the differences between pop & imap in section 4 of Setting up MS Office Outlook 2007 for Outlook.com pop email [migrated & non-migrated accounts] - Wiki.

robert aldwinckle on forums say:

drag it over to my favorites icon that then expands with all my favorites folders and I can then drop the link in the folder I want.

You can do that with the Favorites bar (Ctrl-Shift-I).  But you do have to be patient while hovering over a folder to let it expand.  Also, I'm not sure but I think you can just drag on top of a folder and drop it without expanding it.   In general IMO it is faster in to use Ctrl-d and then find the folder you want to add to by first letter (or two, if you're quick) navigation.

BTW faster than drag-and-dropping a link on the Favorites Bar (aka Links button bar) is to use its Add to Favorites Bar button.  So another technique could be to add all the pages from your current session there that way and then open the Favorites folder itself (shell:Favorites) in File Explorer for further filing.
