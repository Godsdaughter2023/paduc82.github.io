max stewart say:

I trust the Moderators will let me requote a post I made in July elsewhere on this forum which might otherwise be difficult to find. It covers the context in which IMAP, CalDav and CardDav fit and where EAS competes.


=====


Business is business. And in Microsoft’s case, the business to be watched is Google.

 

Unsurprisingly, Microsoft wants you to use its products: desktop Outlook (chargeable), Windows Live Mail (free) and EAS (the DeltaSync protocol which underlies the Hotmail connector which existed as a separate free add-on in Outlook before Outlook 2013 and is now integrated). For businesses, desktop Outlook is often the de facto standard because it integrates seamlessly with Exchange Server which businesses use for internally handling mail. Exchange Server uses its own MAPI/RPC protocol for on-site connections and – usually – EAS for mobile devices which has a mail client (OWA - Outlook Web Access – is available for brower connections). DeltaSync was Microsoft’s proprietary replacement for WebDAV (see below) and it discontinued support for WebDAV in 2009.  

 

EAS is widely used for portable devices and is patent-licensed by Microsoft to Apple (for iPADs and iPhones) and to Google for Android phones, and is unsurprisingly available in Windows Phone 8.

The major advantage of EAS for domestic users is that it syncs not just email (Inbox, Sent box and folders within each) but calendars and contacts (including distribution lists). IMAP syncs email like EAS but stops there. IMAP does not support calendar or contact list syncing. And POP3 only syncs Inbox (and is only designed for one user with one device; otherwise you need to earmark one device to be the ‘master’ one that brings emails from the server and then deletes them whereas the other devices are all set up to leave incoming mail on the server).

If you want the added features, including use of your own domain name, that Microsoft provides with Hosted Exchange email as part of Office 365, you also get EAS, POP3 and IMAP. This is a service aimed more at businesses – and you pay for which bits you use (the cost per mailbox is similar to that of Google Application email  – see below).

 

Google offers free Gmail and chargeable Google Applications (which includes Gmail and calendar and contacts sync). In 2012, Google announced that in December 2012 they were withdrawing EAS support from free Gmail to push users towards Google’s own implementation of IMAP, WebDAV (i.e. HTTP)-based CalDAV (which uses the standard iCalendar formats) and WebDAV-based CardDAV (which uses standard vCard formats). This was their firmly declared ‘open systems’ strategy. (EAS support for Windows Phone was subsequently extended until 31st July 2013.) Then to most industry observers’ astonishment, Google announced in March that they were switching direction again and moving off CalDAV to their own proprietary Google Calendar API and that CalDAV support would be discontinued on 16th September 2013. But on 5th June, they backtracked and said that both CalDAV and the Google Calendar API would remain available.

 

Users of chargeable Google Applications can continue to use EAS to link desktop Outlook with Gmail + Google calendars + Google contacts (their implementation of EAS in Outlook is weird – you need a new Outlook profile – but works). The Google Applications elaboration of Gmail is a corporate mail service which allows administrators to create and control mailboxes for ordinary users (they charge by the mailbox) and a company’s domain own name can be used. It is a thorn in Google’s side that for corporate users, EAS and Exchange’s MAPI/RPC are controllable by administrators using same rich Active Directory policies which are used for Windows-based products.

 

Like Microsoft, Google would like you to stay with Google’s basket of products - in this case Gmail on Android. Although the Android mail APIs are published, it has never been clear (to me) what Google uses as the mail/calendar/contacts sync protocol between Android and their Gmail servers. There could be a shock lurking if Google make this protocol (as opposed to the APIs) available free to compete with licensed EAS.

 

So where does this leave IMAP?

Microsoft wants you to use the functionally far richer EAS. But on the desktop you need Microsoft clients (Outlook or Windows Live Mail). Since the new Mail client in Windows 8 has no support for POP3 (IMAP and EAS only) whereas the new Outlook.com has no support for IMAP (EAS and POP3 only), one can draw one ineluctable message from Microsoft: go EAS or be left high and dry.

Google wants you to run either totally online with the Gmail browser client, or use Android smartphones with integrated Gmail client functionality, or use IMAP / CardDAV / CalDAV from any other device, or to pay for a desktop Outlook EAS Gmail service via Google Applications (Windows Live Mail is not supported). Until of course they announce a free function-rich competitor to EAS for Gmail access that supersedes IMAP + CardDAV + CalDAV and runs on Windows (all common clients including at least Outlook, Windows Live Mail and Thunderbird), Mac, iPhone/IPad and Windows Phone...
