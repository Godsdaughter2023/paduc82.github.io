You can turn off hibernation if it is on and you don't use it..

When you hibernate your computer, Windows saves the contents of the system's memory to the hiberfil.sys file. If you don't use the hibernate feature and want to recapture the space that Windows uses for the hiberfil.sys file, perform the following steps:

- Start the Control Panel Power Options applet (go to Start, Settings, Control Panel, and click Power Options).
- Select the Hibernate tab, clear the "Enable hibernation" check box, then click OK; although you might think otherwise, selecting Never under the "System hibernates" option on the Power Schemes tab doesn't delete the hiberfil.sys file.
- Windows will remove the "System hibernates" option from the Power Schemes tab and delete the hiberfil.sys file.

Major difference between group and Organizational unit ?? - Microsoft Community

loki04 say:

Major difference between group and Organizational unit ??

Pls explain me what is the difference between groups and organizational unit in simple terms with a real time example.
graye say:

OK, I'll take a stab at it...

OU's are used for managing PCs (and accounts) from Active Directory.   So, you'd use an OU if you wanted to configure all PCs in an OU to have a same login screen, toolbars, password requirements, etc.  Only administrators can configure settings on OUs.   These settings are typically security-related.

Groups are use for managing permission on Files.  So, you'd use a group to allow all the folks in Accounting to have access to a folder on a shared network drive.  Any file owner can assign a permission to a group (although, it'd take an administrator to first create the group)

So, in summary:

OU - management of settings (typically on PCs)

Group - assignment of permissions (typically on Files)
