vlookup, using the LIKE function in the lookup_value - Microsoft Community

rich b87 say:

vlookup, using the LIKE function in the lookup_value

I have a situation where I have a list of items and another list of that contains the same items but the is additional data as a suffix to the items.  example list one contains item 123456 and the second list contains 123456xx.dwg.

Both lists contain 10's of thousands of records and I would to use the vlookup function to find list A in list B.  If the vlookup can not be used does anyone have a recommendation on how to do this?

Thanks

Rich 

hansv mvp say:

Let's say 123456 is in cell A1, and the other list (that contains 123456xx.dwg) is in C1:C100000.

The position of the first item that begins with 123456 is given by

=MATCH(A1&"*",C1:C100000,0)

If you want to return the value from column C or from an adjacent column, use a formula such as

=VLOOKUP(A1&"*",C1:D100000,2,FALSE)

If you want to search for values containing 123456 (not necessarily at the beginning), use "*"&A1&"*" instead of A1&"*".

vincentchoy say:

What you are asking for, are relatively advanced features....

One thing you need to understand - Office365 is first and foremost a family of cloud products. Within this family, there are mature, and there are new products. With new products, you would find that features you might expect in a "mature" product might not be there YET.

While Exchange and SharePoint are extremely mature and have been around longer than some of my friends...

Groups and Teams fall into the relatively new products category. For example -

The bulk of really useful Group management features were just announced in Ignite late last year. They are currently in Public Preview and are not even released yet. These are incorporated into Azure AD Premium P1 and P2 as Advanced Group Features. When released you will need to pay for Azure AD premium P1 or P2 to use them.


Guest access for teams was just announced in March 2018.

So while these products show a lot of promise, deep and advanced features are only progressively being rolled out over time.

So you are a bit too early for the features you are looking for.

My suggestion is to read / monitor / participate in Microsoft's Uservoice for Groups and Teams. The Uservoice would give you an idea whether a feature has been suggested, considered, put on development schedule, rolled out etc.

- You may set up a retention policy in Security and Compliance center for your groups. When a group expires, the group’s conversations in mail box and files in the group site are retained in the retention container for the specific number of days defined in the retention policy. Users will not see the group, or its content, after expiration however.

If there is a way to auto archive,  then are there any way to send a notification to owners saying the Team or group is about to archive.

-Office 365 Group Expiration Policy can help administrators remove inactive groups from the system and make things cleaner for their end users. When a group expires it is "soft-deleted" which means it can still be recovered for up to 30 days.  Administrators can specify an expiration period and any group that reaches the end of that period, and is not renewed, will be deleted. The expiration period begins when the group is created, or on the date it was last renewed. Group owners will automatically be sent an email before the expiration that allows them to renew the group for another expiration interval.
