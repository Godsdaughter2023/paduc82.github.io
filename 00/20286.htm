rob nicholson at informed group say: I've found the problem from this post in February last year:

http://www.etdot.com/2009/02/01/windows-mobile-activesync-error-80072ee2/

The problem is DNS related. On our internal network, we have the addresses of our DMZ based servers routed differently than when on the internet. For example, assume our mail server is mail.ourdomain.com. When connected to the LAN, this resolves to the internal IP address of the web server - so it's accessed directly instead of going through the internet. So when in the cradle, ActiveSync gets 10.1.1.1 back when it looks up mail.ourdomain.com.

However, ActiveSync caches this DNS value (not too unusual in itself) so ActiveSync keeps trying 10.1.1.1 even when on mobile wireless. This explains why recycling repplog.exe or rebooting fixes it as DNS is queried again and it gets the real internet facing IP address of our mail server.

The fix in the above article is to diable DNS caching which has a minor performance overhead but I can live with that.

ActiveSync is being a bit braindead here and it's a shame a simple fix couldn't have been applied to WM6.5 that did something like:

REPEAT
    Attempt connect to server
    IF timed our but we have a connection to internet THEN
       Flush DNS
    ENDIF
UNTIL failed for several minutes

Another workaround is for us to create another DNS record for our mail server that doesn't resolve to 10.1.1.1.

But very happy now my Diamond2 has joined the instant email world that I bought it for in the first place :-)

Cheers, Rob.

PS. I was feeling especially sad as all other OS based phones worked like Android and Nokia OS. Wasn't helping in the "My mobile phone is better than yours argument" :-)

graham mayor say:

Without access to the material it is difficult to be specific but ....

Start by getting the font from the message. Presumably you know what is being inserted at the start of the document, so having inserted the text, set the Inspector which will give you editing access to the document body. Set a range to the start of the document and the end of the inserted text. In the following example the document contains only the clipboard content, which you can format with the message font thus:

With olMail
        .BodyFormat = olFormatHTML
        .Display
        Set olInspector = .GetInspector
        Set wdDoc = olInspector.WordEditor
        Set wdRange = wdDoc.Range(Start:=0, End:=0)
        strFont = wdRange.Font.Name
        lngFont = wdRange.Font.Size
        If Not (wdDoc Is Nothing) Then
            .HTMLBody = MessageBody
        End If
        wdDoc.Range.Font.Name = strFont
        wdDoc.Range.Font.Size = lngFont
    End With
