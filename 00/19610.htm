Page Numbers, Skipping Pages, and Recognizing Text - Microsoft Community

coprau say:

Page Numbers, Skipping Pages, and Recognizing Text

I am trying to add page numbers throughout a document by using the Field option. I am need to add the page numbers in the body and not in a footer or header. Here are the quirky things I need.

 

1) I need to skip pages and just have blank pages with the page numbers resuming from where they left off. (i.e. 1, 2, blank, 3,...)

2) I need to have it number pages like this (12, 12-1, 12-2, 13, 13-1, 13-2, 14, 15)

jay freedman say:
You cannot get numbering of that kind by using the {PAGE} field -- it just doesn't work that way. Instead, you will need to use two series of {SEQ} fields.

The syntax of the {SEQ} field includes two pieces that you need to use.

  • One piece is an "identifier", which is just a name for a sequence. Every {SEQ} field must have some identifier. Two sequences that have different identifiers do not interfere with each other.
  • The other piece is one or more "switches" that control the way the numbering within a sequence behaves. You'll need to use the \c switch in some fields to repeat the preceding number, and the \r 1 switch to restart the numbering in other fields.
Here's how it will work: On a page that has only a single number (like 12), you'll insert a field with the code
    { SEQ  A }
On the first of a series of pages that have two numbers (like 12-1), you'll insert two fields separated by the dash:
    { SEQ A \c }-{ SEQ B \r 1 }
Because of the \c switch, the same number 12 will be repeated before the dash. Because of the \r 1 switch, the second number (in the B sequence) will show the number 1.

On the rest of the pages in the '12' group, omit the \r 1 switch:
    { SEQ A \c }-{ SEQ B }
The A number will continue to be repeated, but the B numbers will count up 2, 3, etc.

The next page after the '12' sequence will again need the field
    { SEQ  A }
so that number will increase to '13', and so forth.

Where you have a blank page, just don't put any fields there. When you get to the next non-blank page, just insert the same kinds of fields as above, and the count will continue from where it left off.

jay freedman say:
Do you want to start at 1, or at 12, or some other number? The \r switch can start (or restart) a sequence at any number you choose to put in.

If the first {SEQ A} field in the document has no \r switch, it will (by default) start at 1, and the next {SEQ A} will show 2, and so forth. If you want sequence A to start at some other number, use an \r switch -- only the first time! -- for example

   { SEQ A \r 12 }

to start at 12. Then all the other ones will have no switch, and they'll show 13, 14, etc.

If you wanted to, you could even skip around, for instance the series of fields

   { SEQ A \r 6 }  { SEQ A } { SEQ A } { SEQ A \r 20 } { SEQ A }

would show the values   6   7   8   20   21
