Delete Lookup Field Relationship - Microsoft Community

Delete Lookup Field Relationship

Hi,

ken sheridan say: In the table design simply set the data type of the column to a long integer number (which is the default number data type).  Then create a relationship between the table and the referenced table on the long integer column and an autonumber primary key column of the referenced table and enforce referential integrity.  This makes the column in the referencing table a foreign key.

In the referencing table's datasheet you'll now see the numeric values of the foreign key of course, which will be meaningless as they are arbitrary values merely serving to reference the relevant row of the referenced table.  This doesn't matter, as the place to interface with data is in bound forms, never in a table's datasheet.  In a form you would use a combo box as the control for the foreign key column, setting it up to hide the numeric value and show the referenced text value(s) from non key column(s) in the referenced table, e.g. if the column is EmployeeID, the combo box would be set up like this:

ControlSource:  EmployeeID

RowSource:     SELECT EmployeeID, FirstName & " " & LastName FROM Employees ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

Alternatively, you can concatenate the names so that the last name is first, which would be better with a large list of names as it allows the user to enter the initial characters of the last name and progressively go to the first match as each character is entered:

RowSource:     SELECT EmployeeID, LastName & ", " & FirstName FROM Employees ORDER BY LastName, FirstName;

How can I print on a 6x9 envelope - Microsoft Community

am16 say:

How can I print on a 6x9 envelope

I am not able to print from Word to a 6x9 envelope. I set the custom size but I keep getting an error message. This is a standard size envelope. What am I doing wrong?
graham mayor say:

6 x 9 is only a 'standard' size in Word if the printer driver says it is. Otherwise it is a custom size and Word is hopeless with custom sizes and will adopt a standard size that may be close to the requested size, because it relies on the printer driver to format the document.

You may be able to add the size as a form to the driver and that may allow you to select and print that size, if the printer itself allows it. Alternatively you can simply select any larger standard size paper and adjust the positions of the envelope address and return address styles so they print in the correct area of the envelope. See http://www.gmayor.com/changing_envelope_layout.htm  and the section on custom envelopes.

If that proves to be the approach you adopt, and you find creating a template is of benefit and then you should find http://www.gmayor.com/Envelope_Label_Add_In.htm useful.
