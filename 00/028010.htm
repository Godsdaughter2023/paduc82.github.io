young yang msft support say:

Hi Mitesh,

1. “Could you please provide steps for enabling SSO.”

For Single Sign-on (SSO), you can refer to this roadmap:

Single sign-on roadmap: https://technet.microsoft.com/library/hh967643.aspx

2. “Do I need to stop Directory Sync for SSO?”

No. SSO needs DirSync. You can just leave the DirSync deployment unchanged, and continue SSO deployment.

3. “Is there any extra cost involved with setting up SSO?”

The cost should be the certificate.

The officially supported scenario is to use ADFS to achieve SSO. So, we need an on-premise Windows Server. ADFS is a Windows server feature. If you already have Windows Servers, you can directly deploy ADFS.

To deploy ADFS/SSO, we need a 3rd party certificate.  

4. “Do I need any certificate?”

Yes, a 3rd party certificate is required.

5. “All users username on O365 are email addresses whereas their AD usernames are only their first name. Will that makes things complicated. How to setup this?”

Do you mean AD users only have First Name? If so, do not worry. This does not affect ADFS deployment.

In the ADFS/SSO environment, Office 365 users are synced with on-premise AD users. So Office 365 user names should be the same as on-premise AD user names.  

Moreover, Office 365 User Principal Name (UPN) should be the same as on-premise UPN. For example, both Office 365 UPN and on-premise UPN are: *** Email address is removed for privacy ***

Anything unclear, please let us know.

What is everyone else doing? Are you buying computers with a couple terra-bites of space?

Help.

jiaxing bian msft say: Hi Diane,

You should be syncing your SharePoint libraries with the OneDrive for Business legacy sync client (groove.exe).

Please understand we can’t sync SharePoint libraries to external hard drives with the legacy sync client (groove.exe). Groove will use the cache files to store authoring, versioning, and other important data, and it will take up almost the same size of storage as the files that you’re syncing.

Regarding the situation, please check if the cache files are taking up the same size of storage as the files.

If so, it should be an expected behavior. You can consider syncing your SharePoint libraries to another local disk on your computer, or replacing the current hard drive.

If not, there should be some corruption in the cache files. I suggest you use the “Easy fix” tool in this article to remove the cache files, and re-sync to check the outcome.

Warning: This tool can cause data loss. To reduce this risk, read this article in its entirety before you use this tool.

Let us know if there is any update.

Jiaxing
