tom van stiphout say:

Printing a PDF can be done with a bit of VBA code. You'll need the ShellExecute API. Here is an article that explains how to use it:http://www.suodenjoki.dk/us/productions/articles/vbashellexecute.htm

In your case for the second argument you put "print" rather than "open", and for the third argument the field containing the PdfFileName.

To print each record, you should create a report in Access. You run the report with DoCmd.OpenReport and in the Where-clause you specify the ID of the current record.

To put it all together you write a procedure in VBA which opens a recordset containing all ID values and PdfFileNames to print, loop over this recordset, and for each record print the report and call ShellExecute. Then relax and let the printer do the rest of the work.

 

-Tom. Microsoft Access MVP

You MUST - no option, no choice!!! - give each user their own individual copy of the front end. This is not dependent on Citrix or on local, Access database backend or SQL backend, oranything; the only exception (and it's arguable) would be if the users are all read-only users who will never need to update the tables (or, of course, make any design changes).

My Citrix users provide a shared backend (some in SQL, some use .mdb files) on a computer on the same fast stable LAN as the CITRIX server; each user gets a private folder containing a private copy of the frontend .mde file. When the frontend gets new features, a .bat file copies the new frontend into each folder. There's still only one frontend to maintain (the original .mdb); the users never touch that, only the compiled .mde files.

 

John W. Vinson/MVP
