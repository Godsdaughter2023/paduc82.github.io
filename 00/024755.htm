Diagnosing When Setup Stops Responding

At times, Office Setup stops responding (hangs), and you do not receive any error message. The letters in the value field can be in any order. Each letter turns on a different logging mode. Each letter's actual function is as follows for MSI version 1.1:

v - Verbose output
o - Out-of-disk-space messages
i - Status messages
c - Initial UI parameters
e - All error messages
w - Non-fatal warnings
a - Start up of actions
r - Action-specific records
m - Out-of-memory or fatal exit information
u - User requests
p - Terminal properties
+ - Append to existing file
! - Flush each line to the log
x - Extra debugging information. The "x" flag is available only on Windows Server 2003 and later operating systems, and on the MSI redistributable version 3.0, and on later versions of the MSI redistributable.

"*" - Wildcard, log all information except for the v and the x option. To include the v and the x option, specify "/l*vx".

Note This should be used only for troubleshooting purposes and should not be left on because it will have adverse effects on system performance and disk space. Each time you use the Add/Remove Programs tool in Control Panel, a new Msi*.log file is created.

When looking through the MSI logs we will typically want to look for a value 3 entry in the logs. Windows installer returns codes during the install which will indicate if a particular function was successful or not.

Value 1 = Success
Value 2 = Cancel
Value 3 = Error

No, that's not what AutoRecover is for :-) The purpose of the feature is to assist as bet as it can in recovering unsaved changes in the event of a critical failure of the software or your Mac. alex chen msft say:

Hi Sophia,

 

Thanks for your update.

 

The maximum storage for a site collection is 25TB. But the whole storage is based on your own tenant, the license you've purchased will affect this storage.

 

 

 

To check your site collection storage, you could go to the SharePoint Admin center > Active Sites, there should be a storage limit column, also, on the top-right corner, there is your whole storage limit for SharePoint Online.

 

 

The used storage in all the sub-sites are counted in the corresponding site collection.

 

To achieve the scenario you described, please follow my steps below:

1.Go to the sub site you created > click on the Gear icon on the top-right corner > Site permissions > Advanced Permission Settings.

2.Click on Stop Inheriting Permissions, you will be prompt to set up a new Group for this subsite. Select Create a new group instead of using the existing one.

3.After creating the new group, add the user to this sub site as a member/visitor.

4.Go back to the sub site advanced permission page again, delete those group inherit from the site collection. (Important)

In this way, they won't be able to see the content in the site collection and vice versa. 

Note: Make sure your site collection is in private. You will find this option at Gear Icon > Site information.
