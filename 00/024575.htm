dianeoforegon say: We see reports where Outlook will auto launch but not shut down. Check for a login item that might be involved. Possible culprits: MenuCalendarClock, Quicksilver, LaunchBar, iKey, Keyboard Maestro.

Log out under the Apple in the Menu bar. When you log in, hold down the Shift key. This disables all login items. Does Outlook quit? If no, then it's one of your login items.

If you fail to find,

1) Move your Outlook Identity to Shared folder.

Your Hard Drive/Users/Shared

2) Create a new User in System Preferences

3) Log into your new User.

4) Drag from Shared (it actually copies) your Main Identity and place it in the Microsoft User Data folder > Office 2011 Identities folder. (you'll need to launch Outlook first to create the new Microsoft User Data folder)

 ~/Documents/Microsoft User Data/Office 2011 Identities/your identity

5) Run Outlook for a few hours or a day to see what happens.

Do you still see the issue?

    If yes, then the problem is with your base files.
    If no, then the problem is in your User's folder.

scottgem say:
 I would like to set the parameters in my query for  a date performed greater that today's date + 30 days.  I tried this but it didn't work. <Date()+30

The criteria you show will return all dates less than 30 days from today's date. So it would be anything less that Nov 7th. If you wnat greater than Nov 7th you use:

>DateAdd("d",30,Date())

If you want greater than a month from today use:

>DateAdd("m",1,Date())

Use >= if you to include the date 30 days hence.
