and clover say: > when I look at the "Program and Features" screen in the control panel and then click on the "View Installed Updates" menu, I only can find one "Windows Internet Explorer 9" listed under the "Microsoft Windows" section.

That's normal. You have both 32-bit and 64-bit versions of IE9 installed as a single unit; you can't install IE9 64-bit without also replacing your 32-bit copy. Removing IE9 from Programs and Features will take out both copies.

The same goes for most of Windows, you essentially have two copies of all the system components, but they're tied together so you're always using the same version in 32- and 64-bit.

> My system seems to be set for the 32-bit version because the task bar icon loads the 32-bit

Yes, this is the default for now, probably because there are many browser plugins that are not yet available in 64-bit. For most people there is relatively little benefit in running IE in 64-bit mode as it tends not to need that much memory.

> If I uninstall this one, I assume that the system will have no IE registered

If you uninstall IE9 you'll revert to IE8, for both 32-bit and 64-bit instances.

Sub resetnumber()
Dim lngNum As Long
Dim osld As Slide
Dim oshp As Shape
lngNum = ActivePresentation.Slides.Count
For Each osld In ActivePresentation.Slides
For Each oshp In osld.Shapes
If oshp.Type = msoPlaceholder Then
If oshp.PlaceholderFormat.Type = ppPlaceholderSlideNumber Then

.InsertSlideNumber
.InsertAfter " of " & lngNum
End With
End If
End If
Next oshp
Next osld
End Sub

Although I'm replying to a nearly 2 year old post, I have a similar dilemma.  I purchase through an online retailer, get my products keys via email, but only want to associate them with a corporate email address and not the individual email address to keep control of the activation and licenses for the future.

I found a fix which is a bit of a hassle but works:

Open command prompt as administrator

switch folders to:

C:\Program Files (x86)\Microsoft Office\Office15

Run this command to result the last 5 digits of the current key:

cscript ospp.vbs /dstatus

Remove the old Key with this command:

cscript ospp.vbs /unpkey:xxxxx

where xxxxx is the last 5 digits from the first command

Run any MS Office product and it will let you re-enter the key that you want.
