The steps on how you can delete a Skype contact very depending on the version you're using.

  • On 6.35 version:
  1. To find the contact you want to remove, use the Search function at the top of the screen and enter their name, Skype Name, email address, or phone number. Alternatively, use the Contacts tab or the Recent tab to find the contact there.

  3. Select Delete contact.

In any case, if you don't have a large enough buffer for your cmd window (e.g. via Alt-Space,P) you can capture the output of commands whose output you would like to analyze.   E.g. enter

ipconfig  /displaydns   >displaydns.txt

and then browse it with

notepad  displaydns.txt

 

Your  ping  output above suggests that you may be using a standard user account.   I'm not sure about XP but even with NT I think it was once the case that you needed admin authority to do an  ipconfig  /flushdns.    If so, you would have to start your cmd window using  Run as Administrator and then you would also have to have write permission in whatever directory that started in.   The output filter I have given as an example above would be written in the current directory that the cmd window was showing.   If you did need to switch accounts you would not need to write in the default directory, e.g. you could either use a change directory (cd) command to switch to one you could write in or just give your output filter an explicit path.   It's also possible that a standard user could do a  ipconfig /displaydns  but not a  /flushdns.   In that case you could have two cmd windows.   Etc.

 

The point about the lookup is that if you are getting the wrong IP address, either via your DNS or via an override it could explain your symptom.   E.g. it would be an example of GIGO.   However, since your  ping  result is the same as mine I suppose lookup is not as critical an issue as it might be normally.
