sheen valentine ali msft say:

Hi Nipun,

It is not feasible to track changes in Excel Online. If you have the legacy workbook sharing feature (including the legacy track changes), you will not be able to open it in Excel Online.

The Shared Workbook feature has many limitations and has been replaced by co-authoring. Co-authoring doesn't provide the ability to track changes. However, if you and others have the file open at the same time (both Online and on the Desktop), you can see each other's selections and changes as they happen. Also, if the file is stored on the cloud, it's possible to view past versions so you can see each person's changes. Learn more about co-authoring.

If you would like to continue to work using the legacy Shared Workbook feature, then everybody has to work on Excel Desktop for it to work.

Additionally, we suggest you vote for this feature to work on Excel Online via UserVoice.

brian tillman [outlook mvp 2007-2019] say:

There are multiple differences between POP and IMAP. The main differences are that POP's intended behavior is to download messages from the server-side Inbox, storing them in a client-side Inbox after which they're removed from the server. Once they're in the client, the server no longer cares about them and you can't move anything back to the server from the client. IMAP, however, is designed to leave messages on the server at all times, perhaps making a local copy of what the client sees on the server for speedier access, but what's on the server is always kept in sync with what's on the client and vice versa. An IMAP client also understands server-side folder structure and maintains that structure locally as well so what the client presents is always in agreement with what the server knows and changes made either on the server or in the client will propagate between the two.

Windows Live Mail has an export function that will push directly to Office Outlook when both are installed on the same PC. If you were using POP in WLM, you can export the messages to Office Outlook. If you were using IMAP or DeltaSync (WLM's default connection type to an Outlook.com mailbox), however, mail doesn't need to be moved from one to the other. Just connect Office Outlook to the same mailbox used by Windows Live Mail and you'll see the same data because the data has stayed in the mailbox. You can export contacts from WLM to a CSV file and import that into Outlook, but if you were synching WLM with an Outlook.com mailbox, all the contacts should be on the server already and Office Outlook will just pick them up when you connect to the same mailbox.
