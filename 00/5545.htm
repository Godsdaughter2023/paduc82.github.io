jay freedman say:

In my earlier reply, I had not looked at the entire macro, just the one line that you asked about. This code has several problems, any one of which would mess up the results.

The reason the formatting isn't being copied is the line

Target.Range = Letter

If VBA interpreted that line literally, it would cause an error because it appears to assign one Range object to another Range object without a Set keyword. Instead, VBA interprets it as assigning the "default property" of one range to the default property of the other range. The default property of a Range object is its .Text property, which is an unformatted String. To copy the formatting together with the text, that line must be changed to

Target.Range.FormattedText = Letter.FormattedText 

I couldn't get the code to generate Error 5825, but for each section in the source document it did cause an Error 5941, "The requested member of the collection does not exist", because the Target document contains only one section and there is no Target.Sections(2). Try just commenting out or deleting that line.

The SaveAs command will put the file in whatever folder you specify in the FileName parameter. If you want to put it on the user's desktop, use the following syntax:

Target.SaveAs FileName:=Environ("userprofile") & "\Desktop\ServicePlanDocument" & i & ".docx"

That uses an environment string that Windows provides for each user, C:\Users\<user name>.

While assigning a non-default AttachedTemplate is not an error, it's more usual to specify it as part of the Documents.Add command, as in

Set Target = Documents.Add(Template:="TestConcept.dotm")
