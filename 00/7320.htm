steve rindsberg say:

>>The second--PPTools Hammer Tool--can adapt to any resolution but does require a mouse click. It is also only useful for full-frame display, I believe?


No, you can draw a rectangle of any size you like, memorize its size and The Hammer will whack other images/shapes to that size instead of the full size of the slide.


The way PPT handles incoming images is subject to change from version to version, but the last time I did any extensive testing, it worked something like this:

If the image is in a file format that allows for storing size/dpi info (not all of them do), then try to respect the stored info.  So if you have a 1000 pixel image at 100dpi/10", bring the image in at 10".

If the image is in a file format that does NOT allow for storing size/dpi, then look at the number of image pixels, divide by the current Windows display resolution (usually 96dpi but not always) to get the number of inches.  So the same 1000 pixel image stored as a BMP (doesn't allow storing size info) on my system (96dpi) would come in at 10.42" (1000 / 96 ).  

It sounds as though you're storing images in a format that retains your desired DPI setting.  In both cases, that'd explain your results.  

Personally, I'd put a higher value on the image size matching the projector resolution than on having the image come in full frame automatically, though if the two can be combined, why not?  ;-)

I have done more experimentation and want to report before anyone takes the notion of 131 PPI as a magic number seriously. It turns out that the magic number is 11 inches. PowerPoint 2010 will display an image at its full resolution as long as it is at a high enough PPI relative to the pixel dimension on the long side that it works out to 11 inches or less. Images of high resolution and/or low PPI work out to be longer than 11 inches, and PowerPoint 2010 does not display them at 100% scale. I can only imagine that this is related to printing and the size of 8 ½ x 11 paper.

 

The question is: What DPI makes the most sense? I had come to the conclusion that 131 PPI and higher works well because I was working on a WXGA projector. I hadn’t realized at the time the significance of the fact that 131 PPI at 1280 pixels (the width of WXGA) works out to 11 inches. (1280/131=11)  At another size in pixels, a different PPI would be required to keep the page size under 11 inches.

 

It’s too complicated to use a different PPI for different projectors. The highest-resolution projectors commonly found today, FHD, can project an image 1920 pixels wide. A PPI of 175 is required to keep such a large image under the 11-inch maximum page size. For ease of math, I suggest using a PPI of 200, which should work for all projectors.

 

The technique I would propose to insert an image into PowerPoint 2010 for a high-resolution projector (XGA and up) and have it automatically sized for full-frame is as follows:

 

    1. Crop and size the image to the projector’s full pixel resolution at 200 DPI.
    2. Divide each pixel dimension by 200 to determine the page size in inches and type those numbers in Page Setup. So an FHD image 1920 x 1080 pixels, for example, becomes 9.6 inches x 5.4 inches. (Checking Image Size in Photoshop or similar software will save you the calculation.)
    3. Create a new blank slide.
    4. Insert picture.
    5. Run the slide show at the full resolution of the projector, 1920 x 1080 in this example.

 

Does anyone have any thoughts on this?

I don't think so.  If I add, let's say, a 3000 pixel wide image to PPT, it's not going to display it at 3000 pixels, at least not unless my PC happens to have a monitor or projector capable of displaying that many pixels.

Seriously, if you insist on thinking in terms of DPI and inches, both of which are totally artificial constructs when applied to images on disk, PowerPoint will confuse you.  

In slide show view, it will *always* fill the available monitor with the slide, to the extent that it can do so w/o distorting the image.  If you have a 40x30 slide, it'll fill a 1024x768 projected image with the slide.  So you could insist that it's working at 25.6 dpi.  Project the same slide on a projector capable of 2048x1536 and now it's running at 51.2 dpi.  Keep the same projector but change the slide size to 4x3 inches and shazam, now we're at 512 dpi, even though nothing changed on screen.  

Hence my operating equation for PPT:   DPI = BS

It's really very very simple:  A full frame image's size in pixels should match the resolution in pixels of the projector/display you'll show it on.   If you need to work with a variety of displays, match the image to the highest resolution display you'll use.

It takes so little effort to whap the image to the size you want on the slide rather than fretting with getting it all to work out automatically that I don't think it's worth the mental effort.  

And at this point, if you want to keep gnawing on DPI and inches, more power to ya, but I'll drop out of the discussion.
