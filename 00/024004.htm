ken sheridan say: In a form for entering data into an Employees table, for the foreign key column which references the primary key of a JobTitles table use a combo box set up as follows:

ControlSource:  JobTitleID (this is the name of the foreign key column)

RowSource:     SELECT JobTitleID, JobTitle FROM JobTitles ORDER BY JobTitle;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

The combo box will show the referenced job title value, but its value, and hence that of the foreign key JobTitleID column will be the numeric key value of the relevant row in JobTitles.

For entering data into another table in which EmployeeID is a foreign key column you can use a combo box for the EmployeeID column set up as follows:

Name:              cboEmployee
ControlSource: EmployeeID

RowSource:     SELECT EmployeeID, JobTitle, FirstName & " " & LastName FROM Employees LEFT JOIN JobTitles ON Employees.JobTitleID = JobTitles.JobTitleID ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:    3
ColumnWidths:  0cm;0cm;8cm

In this case the first two columns are hidden, so only the name shows in the format Ken Sheridan.  If you prefer Sheridan, Ken then change the expression for the second column to LastName & ", " & FirstName.  By using an outer join any employees without job titles would also be included in the list.

Add an unbound text box to the form, with a ControlSource of:

=cboEmployee.Column(1)

The Column property is zero-based, so Column(1) is the second column.  When you select an employee, their job title will automatically show in the unbound text box.

doug robbins - mvp office apps & services (word) say:

It sounds like in some cases, your Sections may be defined to have a different first page header and that the primary header was set to be the same as previous.  That would explain why the first page of chapter 32 showed 32, but the second page (for which the header was set to be the same as that in the previous chapter) showed 31.

The way to do what you want is to have the Section Break before each chapter, make sure that the "Different First Page" box in the Options section of the Header and Footer Design tab of the ribbon is NOT checked (or on the Layout tab of the Page Setup Dialog) and that the Headers are NOT "Linked to Previous"

If the Chapter Number is all that is changing in the header, you may however be able to achieve what you want without using Section Breaks and instead make use of a StyleRef field.  To do that, you would need to have the Chapter Number in the body of the text of each chapter (it may be there in a heading). and have that number formatted with a specific Style that you reference in the StyleRef field.

jay freedman say:

I can think of several more complicated ways, but simpler ways are hard to come by.

One way that might work takes some setup:

  • Save your current document as a template. (I'm assuming here that most of the text in your documents is the same, with minor changes.)
  • At the bottom of the text, press Ctrl+Enter to insert a page break, which makes a second page.
  • Insert a bookmark around all the text on the first page except the page break. (It's easier to do this if you click the ¶ button to show nonprinting characters.) Give the bookmark a name, such as MyText.
  • Go to the second page, press Ctrl+F9 to insert a pair of field brackets { } and type the name of the bookmark between the brackets. Then press F9 to update the field, and it will show a copy of the text from page 1.
  • Open the footer for editing. The Header & Footer Tools tab appears on the ribbon.
  • On the ribbon, check the box for Different Odd & Event Pages.
  • Type "Customer copy" in the footer on page 1, and "Sales copy" in the footer on page 2.
  • In File > Options > Display, in the Printing Optio

    When you need a new document, make the necessary changes in the text on page 1. Then print the whole document. The act of printing will update the field on page 2 so it includes the changes in page 1, and each page will have its own footer. Because of the updated field, the document will need to be saved after printing.
