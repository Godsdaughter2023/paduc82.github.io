Try this modified version

Function Convert_Degree(ByVal Decimal_Deg As Double) As String
    Dim Degrees As Long, Minutes As Double, Seconds As String
    Dim s As Long
    s = Sgn(Decimal_Deg)
    Decimal_Deg = Abs(Decimal_Deg)
    'Set degree to Integer of Argument Passed
    Degrees = Int(Decimal_Deg)
    'Set minutes to 60 times the number to the right
    'of the decimal for the variable Decimal_Deg
    Minutes = (Decimal_Deg - Degrees) * 60
    'Set seconds to 60 times the number to the right of the
    'decimal for the variable Minute
    Seconds = Format(((Minutes - Int(Minutes)) * 60), "0.00")
    'Returns the Result of degree conversion
    '(for example, 10.46 = 10° 27' 36")
    Convert_Degree = IIf(s = -1, "-", Chr(160)) & Degrees & "° " & _
        Int(Minutes) & "' " & Seconds & Chr(34)
End Function

You can't open Freelance files in any newer versions of Office and even the Office 97 import filter was pretty weak; it generally worked better to start in Freelance and export to PowerPoint format, but unless you have FL, that's a non-starter, of course.

I wouldn't install Office 97 atop 2010, but you could install it in a virtual machine to keep it isolated.

However, the import filter for Office 97 only works with FL DOS and FL Windows versions 1 and 2.  They'd have to be really REALLY old files before this filter could open them.  The format of FL files changed after that and there's never been a PPT version that could open later FL files.

neha singh msft say:

Hi Michael,

 

If you suspect Ransomware is corrupting the files online which are synced on your local drive using the OneDrive for business sync client, then in this situation you must stop the OneDrive sync to save the data online. Disabling sync on OneDrive for business client will protect your cloud data from being updated. You can also enable versioning to recover the files, here’s an article which will give more information on versioning.

 

OneDrive for Business is included in SharePoint Online. Data protection services are provided to prevent the loss of SharePoint Online data. Backups are performed every 12 hours and retained for 14 days. If the data is infected, it can be recovered by contacting Microsoft support. Assisted support Engineers should help you evaluate the period which will be needed to recover the data.

 

Furthermore, if Outlook data is targeted by Ransomware, then in this situation you should disable ActiveSync temporarily to protect the data in the cloud from being targeted. To disable ActiveSync, you can run the script in PowerShell mentioned in the article under Step 2: Disable ActiveSync & OneDrive Sync.

 

Apart from the PowerShell script, you can use Exchange transport rules to protect users against emails with attachments vulnerable to Ransomware, read more about this from the above article. You can also back up Outlook data on regular basis.

Neha
