If an .ics file is not being imported either via double-clicking or drag and drop to calendar, it usually means the .ics file is malformed.  Different email clients have different tolerances for handling incorrectly formatted .ics files, so the .ics file might work in some email clients but not others.  You can try opening the .ics file in a text editor and looking for text that has improper line breaks (any blank line that does not have "\n" at the end), removing the line breaks, saving, and re-importing (do this on a copy of the original).

Is there any way to scan multiple physical drives simultaneously? - Microsoft Community

itgreybeard say:

Is there any way to scan multiple physical drives simultaneously?

My home office contains a Windows 7 64-bit tower being utilized as a engineering and video graphics engine and mini server. It has 3 internal physical drives (2 of which contain several partitions), and up to 8 external drives attached at various times (i.e. for offline storage, backups and load leveling). Altogether there can be 16TB of disk attached sometime each week.

It appears that MSE scans one partition at a time, which on this hyper-threaded multi-core machine results in tediously long scan times and terribly underutilized CPU and memory, at 15% and 30% respectively.

Is there any inherent technical impediment to an antivirus program scanning these multiple drives simultaneously? It's my assumption that, even with overlapping scheduler-driven batch scripts, MSE is not currently built to accommodate multiple instances of itself. But being a long-time systems hand, I don't sense that it would be a particularly difficult design change or programming project.

Am I missing the boat somewhere, or is MSoft missing the boat especially with much larger corporate data centers and server farms?

btw, I have considered setting up multiple virtual machines with scheduled runtimes and scanning tasks, but that seems quite wasteful of my time and of the machine resources.

Thoughts?  Thanks.
greginmich say:

There’s an old thread where Jeff Toole responded to a similar question:

 

http://answers.microsoft.com/en-us/protect/forum/protect_scanning/is-parallel-scanning-doable-in-mse/4c1935bc-69c5-47bd-8ade-06edac4b6998

 

It also looks like you can run parallel scans on different drives with MSE’s Command Line Interface. I just recently found the time to test this by running simultaneous command-line scans on two of my three internal hard drives, and the scans ran fine together, but took a good chunk of the total CPU. It’s easy to test this by using a one-line desktop batch file for each drive:

 

"C:\Program Files\Microsoft Security Client\MpCmdRun.exe" -Scan -ScanType 3 -File C:/

 

"C:\Program Files\Microsoft Security Client\MpCmdRun.exe" -Scan -ScanType 3 -File E:/

 

There are other possible solutions with Task Scheduler running these custom scans on a System account, but I don’t presently have the time to tinker with that idea.
