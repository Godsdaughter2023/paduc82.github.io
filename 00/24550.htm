joeu2004 say: Shailee wrote:
I am using microsoft office 2007. In Excel while entering $12345678912345678.00 the value changes to $12,345,678,912,345,600.00.

What government do you work for? :-)  Seriously, why do you need to enter 12 quadrillion (US) dollars?

By coincidence, you can enter that number with the formula:

=123456789123456*100+78

Excel displays it as 12,345,678,912,345,700 because Excel displays only up to the first 15 significant digits.

Nevertheless, it is represented internally exactly as 12,345,678,912,345,678.  You can confirm that by subtracting 12,345,678,912,345,600.

However, I reiterate that this is only "by coincidence".  In this case, because 12,345,678,912,345,600 is an integral multiple of 256 and because 78 is an even number.

In contrast, =123456789123456*100+77 results in exactly 12,345,678,912,345,676.  Note that the last digit is 6, not 7.

Generally, when you enter a number, Excel interprets only up to the first 15 significant digits, replacing subsequent digits with zero.

However, Excel can represent all integers up to the 9,007,199,254,740,992 (16 digits) exactly as a result of computations.  And as demonstrated, it can represent some even larger integers exactly.

How can I automatically create multiple worksheets in a single - Microsoft Community

blkprometheus say:

How can I automatically create multiple worksheets in a single workbook

I'm responsible for creating the monthly metrics for the Helpdesk team that I'm on.  Each month I receive a report, in Excel format, that has the following information in it: ticket number, date ticket opened, date ticket closed, name of person who closed ticket (lastname, firstname), ticket description, etc.

 

From this file I create a report for that month.  And I do this for every month in a year.  I sort the data by the name of the person who closed the ticket.  Then I create worksheets/tabs for each person on my team, and then copy the ticket information to their worksheet, for each ticket that they closed.  The name of the tab for each worksheet is the Helpdesk team member's last name.  There are currently 9 people on my team.

 

What's the best way to automate this process, as there are typically hundreds of tickets that we deal with each month?

 

Thanks in advance for your help.

 

jllatham say:

Ok, I think I have it.  Here's how I would deal with it:

I'd put the code I will provide below into a workbook with a single worksheet in it.  You could even put instructions on how to use it on that sheet.  Save the workbook as a macro-enabled template file, then each month you can use it to start a new month's report.

Each month you'd open both the provided file and start a new file from the template and copy the new data sheet into the monthly report file.  Then simply run the macro named CreateMonthlyReport().

Now, you cut off telling about columns at column E, so you need to make at least one change to the code below.  There is a statement that reads:

  Const lastColUsed = "F" ' rightmost column that can have data in it

You need/must change the "F" to contain the column identifier of the right-most column on the source data sheet that can ever contain data you need to SORT and/or copy over to the individual sheets.  Otherwise your data is going to get all kinds of squirrelled up.

All of this code goes into a regular code module.  Instructions for copying it into one can be found on this web page:

http://www.contextures.com/xlvba01.html

Here is the code - and yes, I've tested it and it works.  The only things it doesn't do are put in headers on new sheets beyond column E, or set up particular date/time formatting for the opened/closed columns.

Sub CreateMonthlyReport()
  'must start with the main data sheet selected
  'for all of this to work properly!
  '
  'redefine these Const values to match the
  'layout of the main data worksheet you receive
  '
  Const firstColUsed = "A" ' leftmost column that can have data in it
  Const lastColUsed = "F" ' rightmost column that can have data in it
  Const ticketNumberCol = "A" ' for sorting by name then ticket #
  Const closedByNameCol = "D" ' for sorting & sheet creating & data assignment
  Const firstDataRow = 2 ' assumes labels in row 1
  Const nameSeparator = "," ' MUST!!! be part of every "closed by" entry
'
' end of user defined values
'
  Dim srcWS As Worksheet
  Dim anyWS As Worksheet
  Dim srcLastRow As Long
  Dim namesList As Range
  Dim anyName As Range
  Dim uniqueNames() As String
  Dim LC As Integer
  Dim newSheetName As String
  Dim nextEntryRow As Long
 
  'is there any data to work with??
  srcLastRow = ActiveSheet.Range(closedByNameCol & Rows.Count).End(xlUp).Row
  If srcLastRow < firstDataRow Then
    'no data to work with, just quit
    MsgBox "No Names on the sheet.", vbOKOnly + vbExclamation, "Quitting"
    Exit Sub
  End If
  'go sort the data by name and ticket number
  SortSourceData ticketNumberCol, closedByNameCol, firstDataRow, _
   firstColUsed, lastColUsed
  Application.ScreenUpdating = False
  Set srcWS = ActiveSheet
  Set namesList = srcWS.Range(closedByNameCol & firstDataRow & ":" & _
   closedByNameCol & srcLastRow)
  ReDim uniqueNames(1 To 1)
  'build a list of unique names
  For Each anyName In namesList
    If Not IsEmpty(anyName) Then
      For LC = LBound(uniqueNames) To UBound(uniqueNames)
        If anyName = uniqueNames(LC) Then
          Exit For
        End If
      Next
      If LC > UBound(uniqueNames) Then
        'new name, must add to the list
        uniqueNames(UBound(uniqueNames)) = anyName
        'make room for another possible entry
        ReDim Preserve uniqueNames(1 To UBound(uniqueNames) + 1)
      End If
    End If ' end test for empty cells
  Next
  'if we found any names, uniqueNames() will have an extra
  'empty element in it, get rid of it
  If UBound(uniqueNames) > 1 Then
    ReDim Preserve uniqueNames(1 To UBound(uniqueNames) - 1)
  End If
  'now make sheets based on the unique names
  On Error Resume Next
  For LC = LBound(uniqueNames) To UBound(uniqueNames)
    'this is why a "," must be part of every name entry!
    newSheetName = Left(uniqueNames(LC), InStr(uniqueNames(LC), nameSeparator) - 1)
    'try to select the sheet, if we can, it already exists
    'if we can't then must create it
    Set anyWS = ThisWorkbook.Worksheets(newSheetName)
    If Err <> 0 Then
      'couldn't select it because it didn't exist
      'create it
      ThisWorkbook.Worksheets.Add after:= _
       ThisWorkbook.Worksheets(ThisWorkbook.Worksheets.Count)
      ActiveSheet.Name = newSheetName
      'add the header information to it
      With ActiveSheet
        .Range("A1") = "Ticket ID"
        .Range("B1") = "Creation Date"
        .Range("C1") = "Closed Date"
        .Range("D1") = "Closed By (lastname, firstname)"
        .Range("E1") = "Ticket summary description (short description)"
      End With
    End If
  Next ' end LC loop
  'all needed sheets have been created, now work through
  'list of names to move appropriate data to the individual sheets
  For Each anyName In namesList
    If Not IsEmpty(anyName) Then
      newSheetName = Left(anyName, InStr(anyName, nameSeparator) - 1)
      Set anyWS = ThisWorkbook.Worksheets(newSheetName)
      nextEntryRow = anyWS.Range(closedByNameCol & Rows.Count).End(xlUp).Row + 1
      srcWS.Range(firstColUsed & anyName.Row & ":" & lastColUsed & anyName.Row).Copy _
       Destination:=anyWS.Range(firstColUsed & nextEntryRow)
    End If
  Next ' end of namesList loop
  'make sure we end up back at the main worksheet
  srcWS.Select
  'all done, do some housekeeping
  Set anyWS = Nothing
  Set namesList = Nothing
  Set srcWS = Nothing
  MsgBox "Monthly Report Created", vbOKOnly + vbExclamation, "Task Completed"
End Sub

Private Sub SortSourceData(ticketCol As String, nameCol As String, _
 firstRow As Long, firstCol As String, lastCol As String)
  'this will sort by "closed by" names and when
  'there are more than 1 for a person, then by ticket #
  Dim sortWS As Worksheet
  Dim sortRange As Range
  Dim key1Range As Range
  Dim key2Range As Range
  Dim lastRow As Long
 
  Application.ScreenUpdating = False
  Set sortWS = ActiveSheet
  lastRow = sortWS.Range(nameCol & Rows.Count).End(xlUp).Row
  Set sortRange = sortWS.Range(firstCol & 1 & ":" & lastCol & lastRow)
  Set key1Range = sortWS.Range(nameCol & firstRow & ":" & nameCol & lastRow)
  Set key2Range = sortWS.Range(ticketCol & firstRow & ":" & ticketCol & lastRow)
  sortWS.Sort.SortFields.Clear
  sortWS.Sort.SortFields.Add Key:=key1Range, _
   SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
  sortWS.Sort.SortFields.Add Key:=key2Range, _
   SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
  With sortWS.Sort
      .SetRange sortRange
      .Header = xlYes
      .MatchCase = False
      .Orientation = xlTopToBottom
      .SortMethod = xlPinYin
      .Apply
  End With
  'housekeeping cleanup
  Set sortRange = Nothing
  Set key1Range = Nothing
  Set key2Range = Nothing
  Set sortWS = Nothing
End Sub
