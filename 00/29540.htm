It does state the Z is the EFI system partition.  The reason it is not showing is this command only shows partitions formatted as NTFS.  The FAT32 ones, like the EFI System partition, do not show in the listing.

The listing shows three partitions with no mount points.  This agrees with your showing three recovery partitions, so the OEM partition is probably formatted as FAT32 and was once the EFI System partition but was replaced.

At least you know it is not malware.  Maybe a future Windows Update, possibly today will have some effect on such situations.  As you can see there is already another thread about this.
