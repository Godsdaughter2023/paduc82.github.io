I had the same problem, this is what I did to fix it. Hope this helps:

Symptoms

  • All Skype sounds work okay except for incoming voice or the voice on the Echo test.
  • Contacts can hear me without any problems.

Fix

  1. Right-click Speakers (volume) on taskbar.
  2. Select sounds.
  3. Select Playback tab.
  4. You will see Speakers and other Playback devices.
  5. The problem I encountered was that, regardless of which Speakers are selected within Skype, Skype uses the wrong Playback device. If you test Echo in Skype you should see what Playback device Skype is actually trying to use (in sound). In my case is was trying to use Digital Audio.
  6. Disable the appropriate Playback device (if you are not using it for anything else of course).
  7. Go back into Skype Audio Setting and remove any devices not required in Speakers & Ringing if possible.
  8. May need to restart Skype or Reboot PC.

andreas killer say:
I don't know why this happen, but you can't fix it. And more: It's usual that Excel renames the chart by itself, i.E. if you move the chart from one sheet to another, or place it as standalone sheet and move it back into the sheet.

So it's not safe to identify a chart in a sheet due it's name.

Ideas to identify a chart:
- Use the chart title
- Store a string in the AlternativeText property of the shape object
- Identify the chart due the position (which cells are overlaped by the chart).

The chart title is lost if the user removes it.
AlternativeText property is lost if the chart is placed as standalone sheet.
The position is lost if the user moves the chart.

So an error treatment is indispensable.

Andreas.

ken sheridan say:
The user wants all events in the underlying query for the month before the report is run.

You can easily compute the range in code:

Dim dtmFrom As Date, dtmTo As Date
Dim strCriteria As String

' return first of previous month
dtmFrom  = DateSerial(Year(VBA.Date), Month(VBA.Date)-1,1)
' return first of current month
dtmTo = DateSerial(Year(VBA.Date), Month(VBA.Date),1)

strCriteria = "TDate >= # & Format(dtmFrom,"yyyy-mm-dd") & _
    "# And TDate < #" & Format(dtmTo,"yyyy-mm-dd") & "#"

DoCmd.OpenReport "Payments", _
    View:=acPreview, _
    WhereCondition:= strCriteria

Note the means of defining the range as on or after the start date and before the day following the end date.  The latter is more reliable than on or before the end date as it allows for values in the TDate column with a non-zero time of day.  Unless you have taken specific steps in the table definition to disallow such values you cannot confidently rule out the existence of such rows.  Any rows on the final day of a range would not be returned if the value included a non-zero time of day.

BTW in case you (or any lurkers) are wondering what will happen in January, a zero value for the month argument of the DateSerial function is interpreted a December of the previous year, e.g.

MyDate = #2014-01-15#
? DateSerial(Year(MyDate),Month(MyDate)-1,1)
01/12/2013

PS:  the above is in UK short date format of dd/mm/yyyy, i.e it's 1st December.
