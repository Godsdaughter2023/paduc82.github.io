john mcghie say:
Hi Reginald:

This is a hard call to make, actually.  It would help if I knew what YOU do with Word :-)

Word 2008 was built "down to a deadline".  They did their best to include the features most people actually use, but they had to rule a line when they hit the deadline when they had to ship the product.

The old rule is that 80% of users use only 20% of features.  So if you have 20% of the features of Word 2010, you should be good, right?  Sadly, "each" user uses a "different" 20% of a product's features, and that's where the discussion begins :-)

I do long documents and complex documents for a living.  I rely heavily on automation, styles, templates, drawing tools, fields and cross-references.  I am a Technical Writer.

Could I do my job with Word 2008?  No.  Could I do it with Word 2011?  Yes, I could, can, and do.

Given a Choice what would I choose?  Let's see:  Word 2000 is rock-solid stable, but doesn't do .docx well.  2003 is much faster  and very stable in .doc, won't do .docx properly.  2004 won't run on the current OS X, and it won't do .docx well either.  2007 has a non-customisable ribbon which makes it hopelessly slow and fiddly for my kind of work, but its styles management is much better than anything that came before it.

Word 2010 is fast and stable, with a semi-customisable ribbon.  It's a very good product.  A faster, more stable, more customisable version of 2007.  Word 2011 does not have all the bells and whistles of 2010, it's not as fast, it's not as stable, various high-end features don't work or aren't there.

However, it is my pick among all the versions, because the user interface is fully customisable.  I can fit it exactly to my job.  The drawing tools are poor, but I can get by with them.  VBA Automation has returned: not perfectly, but good enough to do my job.  Most of the templates are unusable, it's a little slow with long documents, and not very stable.  But it does everything I need it to, and it's a lot quicker to work in than 2008 or 2007.

To directly answer your question:  Word 2011 is a larger, faster, more powerful product than 2008.  It has power-user features 2008 doesn't have and will never get.  2008 is fine for general users doing shorter documents and school projects.  2011 is the "Tradesman's Edition".  It works almost exactly the same, but it also includes the high-end power tools that 80 per cent of users do not even know about, and so it enables "professionals" to do their jobs in an Office, Business, or Production setting.

Of course, the "next" version will be better.  I am hoping it will be substantially better.  And it's getting quite close.  So if 2008 is adequate to do what you need right now, waiting may be the best option for you.

Hope this helps

How to Suppress headers and Auto filter with an ODBC query using ActiveSheet.ListObjects.Add

I am running two queries to populate a single worksheet.  The first query returns the headers and creates an autofilter.  I need only the headers and not the autofilter.  The second query is set to return on the first empty line after the first queries data is returned, but is also returning headers and autofilter.

Question:  How do I turn autofilter and or headers off when running a query from VB.  Code snippet can be posted if required.

bill manville say:

I suggest using a QueryTable (as was used in earlier versions) rather than a ListObject.
No AutoFilter is created and you can switch off field names.
Here is code recorded in Excel 2002 to create such a table (slightly tidied up):

Sub Macro1()
'
' Macro1 Macro
' Macro recorded 21/01/2011 by Bill Manville
'

'
    With ActiveSheet.QueryTables.Add(Connection:="ODBC;DSN=MS Access Database;DBQ=C:\Clients\nostalge\data.MDB;DefaultDir=C:\Clients\nostalge;DriverId=25;FIL=MS Access;MaxBufferSize=2048;PageTimeout=5;", _
        Destination:=Range("A1"))
        .CommandText = "SELECT Company, CompanyName, PaymentID, ExportCode FROM Companies"
        .Name = "Query from MS Access Database"
        .FieldNames = True
        .RowNumbers = False
        .FillAdjacentFormulas = False
        .PreserveFormatting = True
        .RefreshOnFileOpen = False
        .BackgroundQuery = True
        .RefreshStyle = xlInsertDeleteCells
        .SavePassword = True
        .SaveData = True
        .AdjustColumnWidth = True
        .RefreshPeriod = 0
        .PreserveColumnInfo = True
        .Refresh BackgroundQuery:=False
    End With
End Sub
