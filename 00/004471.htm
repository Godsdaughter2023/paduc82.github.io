Can't get my flash drive to open when I use Excel - Microsoft Community

norbrozanski say:

Can't get my flash drive to open when I use Excel

We have Excel on our new Toshiba computer but I cannot get my flash drives to open.  What am I doing wrong?
rohn007 say:
Can you see the usb drive and the files in Windows Explorer?

If you can see the USB and files, then I suspect the problem is simply the change in design in 2013.

First thing you want to look at is your default options:
File menu > options command > Save option. Make note of the "AutoRecover File Location".  It defaults to your personal profile. Personally, I prefer to set it to the document default file location, to make it easy to see when autorecover files are "piling up".

I like to set the default file location to a specific "data driver" rather than "My Documents".

Now go to File menu > Open command > Computer command.  If you single click on Computer, you then have option of opening recent folder or using "Browse" button to display "old" Open dialog, or you can double click on the "Computer" command to immediately display the Open dialog.

Assuming your USB is not included in your "Libraries", at this point you will want to click on the "Computer" icon to see all drives, including your USB.

NOTE!!!

All of that being said, I have to follow up by saying DON'T DO THAT!!!

It is a VERY BAD IDEA to edit files directly from USB drives. Due to design limitations of the USB Flash drive structure, files are not actually saved directly to the USB. Files are actually staged to a "Cache" file on the local HD.  When a "block" worth of data is collected, only then it is written to the Flash drive.  If you forget to properly shut down the USB you WILL eventually leave part of the file behind on the local HD, causing file corruption.

By copying the file to the local HD you reduce wear on the Flash drive, and it encourages you to properly shut down the flash drive immediately after copying the files back to it at the end of the editing session.

jay freedman say:

It is possible to make the macro work with a button inside the document body. It just takes a small addition to the macro to change where it inserts the Building Block entry. The macro you have now inserts the entry at the Selection, which (because you click the button) is the button itself. Instead, make the macro like this:

<use the same Sub line as the current document, and the End Sub at the end>

Dim myRange As Range

Set myRange = ActiveDocument.Range

myRange.Collapse wdCollapseEnd

ActiveDocument.AttachedTemplate.BuildingBlockEntries("Leerprojekt 1.1").Insert _

        Where:=myRange, RichText:=True

Two additional comments:

  1. Make sure the building block entry includes a paragraph mark at the end, or a break of some kind, so the next entry inserted after it won't start on the same line.
  2. Instead of an ActiveX button, you may want to use a MacroButton field. ActiveX controls are "heavy" (they use more memory and are sometimes less reliable than fields).

john w. vinson say:

You can see a pretty good example of the OpenForm arguments by opening the VBA editor, pressing F1 for Help, choosing the "Offline" option on the lower right of the screen (otherwise you'll be thrown into the howling wilderness of a web search with 95% irrelevant hits :-{( ), and searching for "OpenForm Method".

 

In this case I think you want

 

Private Sub ScreeningID_DblClick(Cancel As Integer)
    DoCmd.OpenForm "[frm_Preschool Screenings]", _
        WhereCondition:="[ScreeningID] = " & Nz(Me.ScreeningID, 0), _

        View:= acNormal,
        WindowMode:= acDialog

End Sub

Since I have no idea what control on your form has the ID field, I don't know if the condition is right or not - this will look for a control on your form named ScreeningID and open frm_PreschoolScreenings with that ID as a criterion. Note that you need one more quotemark; the WhereCondition argument needs to be a text string like

 

[ScreeningID] = 312

 

so you need to build that string up from pieces - a literal text string and the value of a form object.

 

The acDialog setting means that your current form's code execution will stop until frm_Preschool Screenings is closed or made invisible.

 

Since you (unwisely, in my opinion) have a blank in your form name it's probably safest to use the brackets. I'd change the name to frm_PreschoolScreenings though.

On that worksheet, click in a cell and then press Ctrl + End. That will take you to what Excel thinks is the last cell of data.
