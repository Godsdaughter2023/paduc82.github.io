greginmich say:

How can we have a generated report on the results of a scan by Defender? 

Why does it take a long time when using Full Scan?  It was scanning after 13 hours and it was not completed yet.

Windows Defender doesn’t generate any scan reports per se. Scans are logged in Event Viewer in terms of the specific events enumerated below.

Right-click on the Start button and choose Event Viewer. Then navigate to Applications and Services Logs > Microsoft > Windows > Windows Defender > Operational:

Click on Filter Current Log in the Actions pane and enter “1000, 1001, 1002, 1005, 1116, 1117, 1119” for the Event IDs.

1000 – Scan started 

1001 – Scan completed

1002 – Scan stopped (scan canceled by user activity)

1005 – Scan terminated due to error

1116 – Threat detection

1117 – Threat remediation

1119 – Remediation failure

https://technet.microsoft.com/en-us/library/mt622090%28v=vs.85%29.aspx?f=255&MSPPError=-2147217396

Full scans can take a long time because they have a lot of ground to cover (a large number of files to scan); so general house cleaning and frequent use of the Disk Cleanup tool might help. A Full Scan also tends to take a long time because it takes the time to unpack archive files and scan their contents when it runs with the default setting for the -DisableArchiveScanning parameter. So you might want to disable archive scanning and see if that significantly reduces the scan time – but this is something that I wouldn’t generally recommend:

-DisableArchiveScanning<Boolean>

Indicates whether to scan archive files, such as .zip and .cab files, for malicious and unwanted software. If you specify a value of $True or do not specify a value, Windows Defender scans archive files.

The -DisableArchiveScanning parameter is actually set to False (Disabled False) by default, which means that archive scanning is enabled. You can confirm this by running the Get-MpPreference command in PowerShell. So in order to disable archive scanning you would run this command at the elevated PowerShell prompt:

Set-MpPreference -DisableArchiveScanning $True

https://technet.microsoft.com/en-us/library/dn433291.aspx?f=255&MSPPError=-2147217396

Also, initial scans tend to be very long because Defender is busy checking the “reputation” of files while it builds up a cache of trusted files that don’t need to be scanned. And of course there are some error states that result in excessively long scan times.

Users should also be aware that Defender’s scheduled scans still run with their CPU utilization capped at 50%, even though the Automatic Maintenance scan now runs strictly in idle time – where it has no impact at all on system responsiveness. So in order to get the Automatic Maintenance scan (or any independently scheduled scan) up to full speed, you would have to run this command at the elevated PowerShell prompt:

Set-MpPreference -ScanAvgCPULoadFactor 100

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/processor-speed-100/7306888a-9b1d-451c-b7c2-7e6e5e79b245

Eliminating the page number and hairline on one page of the document. - Microsoft Community

julius hack say:

Eliminating the page number and hairline on one page of the document.

I am editing a simple document, with page numbers in the footer. I need the last page to be totally blank.

But if I erase the last page number (an even number), all the even page numbers disappear throughout the document. Moreover, a hairline I have placed in the footer of every page is suddenly duplicated in the header, where it never was before. I want to get rid of both on that one page.

Any ideas?

Thanks for any help.

JH
suzanne s. barnhill say:

Part of this is easy. You use an IF field in the footer with this syntax:

    { IF { PAGE } < { NUMPAGES } "[Content you want to appear in the footer]" }

What goes between the quotes is whatever footer content you want. If this is particularly complex, it's usually easier to create it outside the footer and paste it into the field code. If it's just a page number, then all you need is:

    { IF { PAGE } < { NUMPAGES } { PAGE } }

As for the hairline, this will probably be a border, which you have added to the Footer style. Perhaps the Header style is based on or otherwise linked to the Footer style and has consequently inherited the border. It's easy enough to get rid of the border in the header on every page, but it will not be easy to get rid of it in either header or footer on just the last page as long as it is part of the Header/Footer style.

If all else fails, you can insert a section break before the last page and delete the header/footer content in the new section (after unlinking the header/footer from the previous section), but if your text must flow from the next-to-last page to the last, then this will not be a satisfactory solution.
