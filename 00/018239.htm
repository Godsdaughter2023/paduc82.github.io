ken sheridan say: An autonumber is a long integer number data type, so should not be enclosed in quotes characters when referenced.  I'm puzzled why you are referencing an autonumber as a literal value, however.  An autonumber is intended solely to provide a unique identifier.  Its value is arbitrary and has no semantic significance whatsoever, so the value should never be exposed to the user.

The value will often be used behind the scenes, e.g. as the hidden column in a combo box for entering data into a foreign key column in a referencing table, or in an unbound combo box used for searching for a record.  In either context you might have a combo box for selecting a city when inserting a value into a CityID column in a Customers table, or for searching for customers in a particular city.  In both cases the user would only see the city names, but behind the scenes the hidden CityID would either be entered into the table or used in an expression to navigate to the correct record.  The value of the CityID column would never be seen by the user.

If you could explain in more detail the context in which this problem is being encountered we will probably be able to advise more categorically.

scottgem say:
While I disagree with some people that an Autonumber should ONLY be used behind the scenes, In this case, I suspect there are some other value they can use to select a record. So they should use a combo that displays the value to select from but returns the autonumber key.

ken sheridan say:
............if the user enters a 30 as the contributor ID, they might end up with 301, 302 or ect.

Entered where?  And how is the number processed to return the contributor?  The only way this can occur is if the values are being processed as string expressions, not as numbers, and even then it would require pattern matching rather than by equality.  If the values are processed as a numeric data type, which an autonumber is, then there is no reason why entering 30 should return a row with a value 301, 3001 etc if such values exist in the column.  It will return the row with a value 30.  Most of us regularly work with tables in which the values in an autonumber columns are of much greater magnitude than this, and don't encounter any problems.
