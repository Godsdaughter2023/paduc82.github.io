You can't disable the keyboard command but you can change it to something else using Mac OS X's Keyboard Shortcuts feature.

Go to Apple menu --> System Preferences... --> Keyboard --> Keyboard Shortcuts tab --> Application Shortcuts.

Click the " + " button (plus) to create a new shortcut specifically for Outlook for Mac. Set the Menu title to "Mark All as Read". Enter this exactly as you see the menu item in Outlook.

In the Keyboard Shortcut field type a new shortcut such as Shift + Control + Option + Command + T. That will be much more difficult to accidentally type.

You should see the new command appear in Outlook right away.

cindy meister say:

Hi Tony

Whenever I read "the formatting changes", the first thing that comes to my mind is that the field code contains the switch \* MergeFormat. This switch enables the field to "remember" formatting applied to the field result when the field is updated. Unfortunately, what is remembered is the location, counted as characters from the starting point. So if something is added or removed, the formatting shifts. However, the switch is often useful for Excel tablestructure.

Traditionally, Word uses a LINK field to bring in data dynamically from Excel. If you press Alt+F9 in your document do you see LINK fields instead of the Excel content? Do these fields contain \* MergeFormat at the end? If not, try adding it to one of the fields and see if that stabilizes the column width to what you've set manually in Word.

As to the Excel content: Assign each Excel Range a NAME. Then substitute that name for the cell range in the LINK field. This is how a Named Range reference looks on my machine:"Tageskurse!AUD"

The part before the "!" is the name of the worksheet, the part after is the range name.

Cindy Meister, VSTO/Word MVP
