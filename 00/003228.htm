ken sheridan say: My guess is that the subform's recordset is not updatable, so when it returns no rows the subform's bound controls do not load.  In cases like this what I've done on occasions is put a message as a text box  beneath the subform control so that it is hidden.  Then in the parent form's Current event procedure hide the subform control with:

    With Me.SubformControlName
        .Visible = .Form.Recordset.RecordCount > 0
    End With

This hides the subform when no rows are returned, making the message visible.  SubformControlName is of course the name of the control in the parent form's Controls collection which houses the subform.  This might or might not be the same as the name of its source form object.

So I came across this issue and was at first relieved I found someone else with the same issue. But then I was also very discouraged as there has been no progress on this since 2009! Come on Microsoft!!

However, I have found a work around that does work. I will try my best to explain the process.

1. There is no way to change the link to the list if the URL character limit is over 63. But DO NOT DELETE the OLD SharePoint List, we will deal with this list later.

2. Create the new list you want (since the URL character limit is much higher for a new list).

3. Once the new SharePoint list has been created, right click and select "Rename"

4. Name the NEW SharePoint list EXACTLY like the OLD list.

5. MS Access will ask if you want to replace the OLD list with the NEW list. SELECT YES!!!!

6. All of your reports, SQL codes, Queries and everything stays linked!!! It's a beautiful thing.

This solution worked for me...hopefully it works the same for you.
