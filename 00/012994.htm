One way too overcome this problem of misisng pivot table function is to use the sql function together with the group function, count  or sum function.

To give an example below:

SELECT Table1.F1, Table1.F2, Count(Table1.F2) AS CountOfSite, Sum(Table2.Sep2014) AS SumOfSep2014

GROUP BY Table1.F1, Table1.F2;

That's merely a simple aggregating query.   Pivot table view is more akin to a crosstab query, e.g.

TRANSFORM FIRST(Evaluations.Recommendation)
SELECT Employees.Firstname, Employees.Lastname
 FROM (Employees INNER JOIN Evaluations
    ON Employees.EmployeeID=Evaluations.EmployeeID)
    INNER JOIN Positions
    ON Evaluations.PositionID=Positions.PositionID
GROUP BY Employees.Firstname, Employees.Lastname
PIVOT Positions.Position;

which returns the recommendation for each employee in relation to each position for which they have been evaluated, returning the evaluation at the inresection of the employee row and the position column.
