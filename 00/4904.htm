Calculation Settings in Excel 2016

I have a user that wants Excel 2016 to identify some files as Auto calculation and others as Manual. However, in the program, when you change the Calculation settings on one file, it applies program-wide. Is there an option for Excel 2016 to set Calculation Option by document or is it only a program-wide setting?

vijay a. verma say:
It is always program wise. If you set in one file, then every succeeding file will be opened with that attribute.

Say, you have 5 files open. Change the setting in one to manual. All 5 open files will become manual. Any succeeding file which will be opened will also be opened with manual mode.

Change to Automatic. All files will be changed to Automatic. Any succeeding file which will be opened will also be opened with Automatic mode.

davem121 say:
Hi Kevin, the calculation setting in Excel is set program wide.

What you can do is add a small macro to each Workbook that would set your users preferences for each individual workbook

In each workbook, press Alt + F11 to open the VBA Exitor

On the top left, double click ThisWorkbook

Paste wither of these macros into the resulting page

Private Sub auto_open()
Application.Calculation = xlAutomatic
End Sub

Private Sub auto_open()
Application.Calculation = xlManual
End Sub

Save the workbook as a Macro Enabled Workbook - xlsm

That's It!

jim gordon mvp say:

If you have all of Word's default settings, if Word freezes, is force-quit or crashes there is a temporary copy of each open document that was saved at least once. Word will recover the documents that were open the next time you open Word.

If you had ever saved the document you were working on, you could recover the temporary file, which would be no older than 10 minutes old.

There is a good explanation of how AutoRecover works and where the temporary file are stored in Word Help. Here's a link to that help topic:

Recover files in Office for Mac

The very idea of recommending uninstall/reinstall is ludicrous in this situation. Do not do uninstall/reinstall. It would be a complete waste of your time.

It wouldn't hurt to check to make sure your install is up to date. Go to the Word menu and choose About Word. If you have version 16.13.1 or 16.14 your install is up to date. If it's older, then do the following:

Ensure your Mac and Office have the latest updates. Here’s how to install updates:

 

Make sure your Mac has a working internet connection, then:

 

  1. Update Mac OS X. Open the App store and click on the Updates button. Install the updates.
  2. After, in any Office application go to the Help menu and choose Check for Updates to launch the AutoUpdate app. Let AutoUpdate install updates and then restart your Mac.

 

If you use Antidote, Webex, Silverlight, EndNote, Zotero or other add-ins there is a potential for crashes unless you also update your add-ins to the latest update. Install updates for these programs if you have them.

jerryduan_msft8021 say:

Hi Devante,

This is an expected behavior. The tokens that stores in Appdata\local\Microsoft\office\15.0\licensing are the licensing tokens users used in Shared Computer activation. These tokens need to be renewed every once in a while. Thus, users need to reactive the Office client after a time. The previous tokens and new created tokens all are stored under the licensing folder so you get a .previous extension files and new created files there.
