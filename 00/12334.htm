[sk] say:

I have spent MANY an hour on this problem on more than one computer.  I think I have finally found the cause of the problem and a solution.

The cause of the problem is due to the 2013 to 2016 upgrade process not cleaning up all the information related to 2013 ODFB and not properly configuring the 2016 ODFB.  In particular the Microsoft Office Document Cache (MSOSYNC.EXE) process for Office 2013 was still being loaded at startup after the upgrade to Office 2016.  This is easy to remedy via CCleaner or SysInternals tools but it is not the real problem.  The real problem is the Microsoft Office Document Cache Sync Client (CSISYNCCLIENT.EXE) was running after the upgrade.  If this process is not running 2016 ODFB WILL NOT SYNC.  NO matter how much you try to repair, reinstall, etc. 2016 ODFB will not sync.

This leads to the solution.  Find the CSISYNCCLIENT executable (C:\Program Files (x86)\Microsoft Office\root\VFS\ProgramFilesCommonX86\Microsoft Shared\OFFICE16\CSISYNCCLIENT.EXE), right-click on the file, and select Run as administrator.  This will launch the process.  You can use Task Manage (see attachment) to verify it is running.  After everything is running properly, reboot and ODFB should work fine.

One really cool tip I picked up in this whole process is to simply map a drive letter to ODFB.  In some ways this is much better than the "standard" ODFB process. The process is described here https://kb.niu.edu/page.php?id=46915 .

[sk] say:

    1. Default retention date

You shouldn't get this prompt with the default Mail Merge feature of Office 2010. Sub NameRows()
'assumes sheet with data is selected when you run this
  Const nameColumn = "D"
  Const firstDataColumn = "E" ' the first column to examine for data
  Const lastDataColumn = "I"  ' the last column to examine for data
  'this is first row of your data, may be 2 if you have labels in row 1
  Const firstDataRow = 1
  Dim namesList As Range
  Dim anyNameEntry As Range
  Dim columnPtr As Long
  Dim newName As String
  Dim newNameRef As String
 
  Set namesList = Range(nameColumn & firstDataRow & ":" _
   & Range(nameColumn & Rows.Count).End(xlUp).Address)
  For Each anyNameEntry In namesList
    If Not IsEmpty(anyNameEntry) Then
      For columnPtr = Range(firstDataColumn & 1).Column _
       To Range(lastDataColumn & 1).Column
        If IsEmpty(Cells(anyNameEntry.Row, columnPtr)) Then
          Exit For
        End If
      Next
      'columnPtr will always be 1 more than actual last used column
      columnPtr = columnPtr - 1
      'build up the reference formula for R1C1 type entry
      newNameRef = "='" & ActiveSheet.Name & "'!R" & anyNameEntry.Row & "C" & _
       Range(nameColumn & 1).Column & ":R" & anyNameEntry.Row & "C" & columnPtr
      newName = anyNameEntry.Text
      'some names may be invalid, test for that and if it is
      'then simply add _ to it to try to make it valid
      On Error Resume Next
      ActiveWorkbook.Names.Add Name:=newName, RefersToR1C1:=newNameRef
      Do While Err <> 0
        DoEvents
        Err.Clear
        newName = newName & "_"
        ActiveWorkbook.Names.Add Name:=newName, RefersToR1C1:=newNameRef
      Loop
      On Error GoTo 0
    End If
  Next
  MsgBox "All Done!"
  Set namesList = Nothing ' just good housekeeping

End Sub
