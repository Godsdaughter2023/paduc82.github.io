That may be how it seems to you and many others would have you see it that way. It is actually not true. The problem is you need to get your Windows Update settings correct to avoid an unwanted installation of Windows 10. On this Windows 7 computer and my other Windows 8 computer I have no problems and no nagging reminders.

A lot of problems arise from setting Windows Update to automatically update. You may prefer to change this setting. Select Start, All Programs, Windows Update, Change Settings. Under Important Updates is a box showing the current setting. Click on the Arrow down to the right and change the selection to “Check for Updates but let me choose whether to download and install them” and click on OK. Restart your computer.

Click on Install and click again on Install in the next window and the process of installing starts. You should let the process proceed without interruption. Multi-tasking at the time updating is in progress is unwise. The progress window will tell you what is going on. You need to be patient. Restart the computer when prompted to do so.

You may find that Upgrade to Windows 10 does not stay hidden. If that happens repeat the exercise.

In the next window place the cursor on GWX  1  and use the scroll  2  to change the behaviour to Hide icon and notifications  3  and click OK  4  . You may need to uncheck the box before Always show all icons and notifications on the taskbar to be able to change the GWX icon behaviour.

When you have changed your seetings we need to discover why you have failing updates.

Please restart your computer and allow 20 minutes for the system to run before uploading information. When examining Event Viewer log files many, not all, problems show in the period immediately after the computer has been booted.

Please upload the file to your OneDrive, share with everyone and post a link here. Reports in normal mode preferred. Please say if the report has been obtained in safe mode. Please upload and share with everyone a fresh copy of your System log from your Event Viewer to your One Drive and post a link here. It helps to avoid confusion if you remove any earlier copies of the log files from your OneDrive.

To access the System log select Start, Control Panel, Administrative Tools, Event Viewer, from the list in the left side of the window expand Windows Logs and select System. Place the cursor on System, select Action from the Menu and Save All Events as (the default evtx file type) and give the file a name. Do not provide filtered files. Do not place the cursor in the list of reports before selecting Action from the menu. Do not clear logs whilst you have a continuing problem.

palcouk say:
In a laptop or desktop with built in graphics, the graphics shares the installed system memory

dominicp say:

If you have a separate graphics card, no.

If you are using a laptop, or a desktop with graphics integrated in the motherboard, some of the system memory is assigned to graphics. In a proprietary system, the graphics memory allocation is often fixed by the system supplier. If you have a self-built or other system from a small system builder the graphics memory allocation can be varied in the BIOS.

If you have both onboard graphics and a separate graphics card you can sometimes combine them, if the BIOS permits it and you have a compatible graphics card.
