a. user say:

It is confusing! I'll try to straighten it out for you.

Your user account on a Windows computer should be password-protected. That's your Windows password, also known as your user account password.

When you create a user account on a Windows computer, you can make it a local account or a Microsoft account. A local account stores all your account settings and personal files on your own computer, also known as the local computer. A Microsoft account stores many of your account settings (but not all) and your personal files 'in the cloud,' i.e., on the internet, on servers owned by Microsoft.

So far so good? Okay, here's where it starts to get a little crazy.

You can have a Microsoft account even though you're not using it as your user account on Windows. That's what I do. My Windows user account is a local account. I have a Microsoft account, but I don't use it as my Windows user account. So I have a Windows password for my user account and a separate Microsoft password for my Microsoft account. If I wanted to, I could make my user account on Windows a Microsoft account instead of a local account. In that case, my Microsoft account password is also my Windows account password.

Clear as mud?

Now why would I want to have a local account as my user account, instead of a Microsoft account? Simply because I don't want to share my user account settings and personal data with Microsoft. To be more correct, I want to choose what I share. That's my choice. Other people feel differently and I respect that.

So I sign in to a local user account on Windows and when I want to use a Microsoft account service, like OneDrive or Office Online, I provide my Microsoft account username and password to the service, and I'm signed in to the service.

Sometimes when you're using Microsoft account services you're asked to provide your Microsoft account username and password several times. That's for security. To us it looks like one continuous thing but a Microsoft account is actually made up of several parts and you'll have to sign in to each one separately.

andre da costa say:

Its possible you setup the device with a Microsoft ID as your sign in (outlook.com, hotmail.com, live.com or msn.com), but you are using a different password for the sign in.

What are Local and Microsoft Accounts?

Since the release of the first version of Windows NT, the operating system requires a user account for authentication purposes. This keeps your personal files, privileges and settings separate from other accounts. A Local Account can have different types of privileges depending on factors such as the environment it is used in. If you are using your computer at home, your account can have certain privileges such as: Administrator, Standard Administrator, Limited User or Guest. If your computer is connected to a business network in an organization, it might need to authenticate on a Windows Server, where the account credentials are stored; this is called a Domain.

With the introduction of Windows 8, Microsoft introduced a new method of authentication called a Microsoft Account. A Microsoft Account uses an existing Hotmail.com, Outlook.com, Live.com or MSN.com domain to sign into Windows. The benefits of a Microsoft Account include the ability to keep your files and settings, such as themes, passwords and apps synced across devices running Windows 8 or later. A Microsoft Account is also secure, utilizing technologies such as two factor authentication, which prevents hackers from easily hacking into your computer. The Microsoft Account is also required to download Universal apps from the Windows Store.

Which ever option you choose, make sure your password is strong, kept safe and easy to remember. Temporarily write down your password during setup, memorize it, then destroy it after setup is complete. See instructions how you can recover your password if you forget it.

Proceed to create a password; make sure your password contains a mixture of upper, lowercase letters, and symbols. Avoid using names of places, people and birthdays. For more information about creating a secure password, review our article. Remember to preview your password to make sure you entered the right information.

If you continue experience sign in problems under your Microsoft Account, I would recommend changing the password for the account.

Press Windows key + R

Type: control userpassword2

Hit Enter

Select your account then click change password.

patrick barker say: Hi,



If you don't know where DMP files are located, here's how to get to them:

 1.    Navigate to the %systemroot%\Minidump folder.
 2.    Copy any and all DMP files in the Minidump folder to your Desktop and then zip up these files.
 3.    Upload the zip containing the DMP files to Skydrive or a hosting site of your choice and paste in your reply.

If you are going to use Skydrive but don't know how to upload to it, please visit the following:

http://www.wikihow.com/Use-SkyDrive

Please note that any "cleaner" programs such as TuneUp Utilities, CCleaner, etc, by default will delete DMP files upon use.

If your computer is not generating DMP files, please do the following:

1. Start > type %systemroot% which should show the Windows folder, click on it. Once inside that folder, ensure there is a Minidump folder created. If not, CTRL-SHIFT-N to make a New Folder and name it Minidump.

2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

4. Double check that the WERS is ENABLED:

Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. Also ensure that under Startup Type it is set to Automatic rather than Manual. You can do this by right clicking it, selecting properties, and under General selecting startup type to 'Automatic', and then click Apply.

How can i draw an angle of say 40 Degres in Paint with windows 7

How can i draw an angle of say 40 Degres in Paint with windows 7
jetta48 say:

Paint doesn't include a built-in way of creating lines at 40degs to each other. However, if you want it to look roughly right you could do the following:

1. Open Paint.

2. Click on View then put a tick against Ruler and Gridlines.

3. Find the point 100 along the horizontal and 600 down the vertical (call this Point A). Draw a straight line horizontally to the right from this point until you reach 700 horizontally.

4. Find the point 700 along the horizontal and 197 down the vertical. Draw a straight line that connects this point to Point A.

 

If my calculations are correct (and its a very long time since I did trigonometry at school), the angle between the 2 lines you have drawn will be approximately 40 degs.

Calculation based on:  Tan(40degs)=(503/600)

Personally, I would find a better application to do this in, particularly if accuracy is important.
