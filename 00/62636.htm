a. user say:

What to do if a machine is infected.

 

From not a long time ago, security used to be only a business concern, not anymore; it must be an awareness of all the IT players to stay at safe side.

On my view, once one have a machine infected no matter by what or how, I can find only two effective choices to deal with this and going confident on a machine again:

·         Recover the entire system from a previous backup, if it at all possible, from the first one right after the “clean install", updates, drivers and activation (Back up and restore ).

Or

·         Go into the entire “clean install” process (Installing and reinstalling Windows 7) or into a factory settings to be provided by the machine brand support site.

System restore could be another option also, but the problem stands when we realize that what time a machine is infected it is not at the equivalent precise time as the infection has gone underway, that is, modern attacks use hidden latent exploit technique to start it later. 

Workarounds give us a false sensation of safety by hiring leftovers that act as attack vectors no matter how creative those jobs are or the tools they use, the most common and evident symptom of vectors is when some Windows services begin to stop like as security center, and many others hard to precise as flashing black screen, missing drives… etc.

For the comings and as prevention is by far better than fixings, a reasonable behavior for end-users on security can be found here at Microsoft Online Safety  and practices both from TechNet Pros to general audience: UAC and Standard Account Use at Inside Windows 7 User Account Control by Mark Russinovich and Be Safer - Run as Standard User at  Jeff Jones Security Blog.

A security concern is not made to implement soft, instead, with determination for the reason that the safe side is the best place for all of us to stay.

The choice is yours anyway.

Best Regards.

I believe the issue with the mail and calendar apps was down to the fact that they needed an update but the issue with the store meant that those apps could not be updated so if you can get the store working then the other apps will follow

This is what worked for me:

  1. Completely uninstall (not just disable) your anti-virus software (AVG for me) and any other software related to viruses, etc leaving only Windows Defender to do its job and restart the PC [This solved the issue of not being able to access the store with the recent 'the server stumbled ****' but the app updates still stalled]
  2. type 'msconfig' into Cortana clicking the topmost highlighted item called 'System Configuration' and select 'Normal boot' and 'okay' in the opened dialogue box.
  3. Go to Settings > Accounts and create a 'Local Account' with full admin privileges if you don't already have one then sign out and log into that account. If you have not used this account in a long time or it's your first time then logging in may take a long time as Window sets up the account for Windows 10. After you log in you may have to restart and re-log in into the Local Account (LA) as the OS and apps including the store seem to adjust themselves for 'first use'
  4. Once in your LA open the app store and search for the app updates (do not log into your Microsoft account within the app) and the updates when found will finally start to download. Log back into your main account and you find that the store, mail and calender apps plus any other downloaded apps are working fine now - the updated mail and calender apps have quite a few neat improvements too. At this point make sure the latest windows updates have been installed.     andre for directly say:
    If you add a secondary language to Windows 10 version 1803 that shares the same base language as the Windows display language, neither language can be removed.

    For example: If English (United Kingdom) is the current Windows display language, and you then add English (India) to the Preferred languages list, afterwards neither English (United Kingdom) nor English (India) can be removed from the Preferred languages list.

    Note: The most notable effect of having multiple related languages in the Preferred languages list is additional entries in the text input selector. You can remove unwanted language list entries using these steps:

    1. From the Start menu, search for powershell.

    2. Right-click Windows Powershell in the search results and choose Run as administrator.

    3. Type or copy/paste the following command at the command prompt, and then press Enter.

    Get-WinUserLanguageList

    After typing this command, make note of the “LanguageTag” listed for the language you wish to remove, as you will need it in the next step.

    4. Type or copy/paste each of the following commands at the command prompt, and then press Enter. In the following commands, instead of typing <languagecode>, replace it with the language tag you wish to remove, such as en-in for English (India).

    $LangList = Get-WinUserLanguageList
    $MarkedLang = $LangList | where LanguageTag -eq "<languagecode>"
    $LangList.Remove($MarkedLang)

    Set-WinUserLanguageList $LangList -Force
