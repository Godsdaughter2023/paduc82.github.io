doug robbins - mvp office apps & services (word) say:

To do this in Word, instead of using the Line Numbering feature, you would need to anchor a text box in the header of the document and that text box would have to be located in the left margin and extend down the page.  However, you would have to have the text (including that in the Footnotes) all in the one font size and not have the paragraphs formatted to have space before or after them unless you did not mind if that space was numbered (the space would however need to equate to a single of multiples of a single line.  Such "line numbering" would restart on each page.

The following screenshot is a comparison between such line numbering (on the left) and the line numbering that is applied by the use of that facility in Word.

wendybw say: Thank you for your suggestion.  That is not a bad idea but this is something that I would have to create weekly in different documents and I didn't have much luck with alignment.  What about a macro or something that can do it each time without much fuss?

doug robbins - mvp office apps & services (word) say:
Rather than a macro, create a template that has such a textbox with the numbering in it and then use that template as the basis of the documents that you create each week.

stefan blom say:

As Doug suggested, your best option is to make use of a template with the "numbered text box" added to the header.

For numbers to line up with text, use fixed line spacing (in both the document and the text box) and make sure that spacing before/after (if you make use of it) are multiples of the line spacing. It still won't be perfect.

Never re-install firmware unless absolutely necessary.  You should always try everything that you can think of, and probably come here and ask others once you have done that.  Call Nokia or your phone operator if necessary.  If none of those work, then, you might want to try to re-install firmware.
