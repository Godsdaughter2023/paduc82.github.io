financial engineer say: Hello Sanjay

You can make use of the following UDF XIRR function that may resolve your issue.
Please note that it requires entering values for guess and compounding.
It was rewritten in VBA from an Excel add-in called tadXL which was coded in C

Values for compounding may be 1 for annual or anything less such as 1/12 for monthly and 1/4 for quarterly

Guess should begin at 10%.

I think the guess and compounding parameters can be made optional with a directive in the function call but as I said I am not a VBA programmer

' Author: Abraham A.
' (c) 2007-2013 http://tadxl.com


Public Function tadEFFECT(ByVal rate As Double, ByVal compounding As Double)
If compounding = 0 Then
tadEFFECT = Exp(rate) - 1
Else
tadEFFECT = (1 + rate * compounding) ^ (1 / compounding) - 1
End If
End Function

Public Function tadPVIF(ByVal rate As Double, ByVal N As Double, ByVal compounding As Double)
tadPVIF = (1 + tadEFFECT(rate, compounding)) ^ (-N)
End Function

Public Function tadPVIFbar(ByVal rate As Double, ByVal N As Double, ByVal compounding As Double)
If (compounding = 0) Then
tadPVIFbar = -N * tadPVIF(rate, N, compounding)
Else
tadPVIFbar = -N / compounding * tadPVIF(rate, N, compounding)
End If
End Function

Public Function tadXNPV(ByVal rate As Double, ByVal Values As Range, ByVal Dates As Range, ByVal compounding As Double) As Double

Dim rCell As Range
Dim i As Long
Dim t As Double
Dim npv As Double
Dim ValuesArr() As Double
Dim DatesArr() As Long

ReDim ValuesArr(Values.Count - 1)
ReDim DatesArr(Values.Count - 1)

i = 0
For Each rCell In Values.Cells
ValuesArr(i) = rCell.Value
i = i + 1
Next rCell

i = 0
For Each rCell In Dates.Cells
DatesArr(i) = rCell.Value
i = i + 1
Next rCell

npv = 0

For i = 0 To Values.Count - 1
t = (DatesArr(i) - DatesArr(0)) / 365
npv = npv + ValuesArr(i) * tadPVIF(rate, t, compounding)
Next i

tadXNPV = npv
End Function

Public Function tadXNPVbar(ByVal rate As Double, ByVal Values As Range, ByVal Dates As Range, ByVal compounding As Double) As Double

Dim rCell As Range
Dim i As Long
Dim t As Double
Dim npv As Double
Dim ValuesArr() As Double
Dim DatesArr() As Long

ReDim ValuesArr(Values.Count - 1)
ReDim DatesArr(Values.Count - 1)

i = 0
For Each rCell In Values.Cells
ValuesArr(i) = rCell.Value
i = i + 1
Next rCell

i = 0
For Each rCell In Dates.Cells
DatesArr(i) = rCell.Value
i = i + 1
Next rCell

npv = 0

For i = 0 To Values.Count - 1
t = (DatesArr(i) - DatesArr(0)) / 365
npv = npv + ValuesArr(i) * tadPVIFbar(rate, t + compounding, compounding)
Next i

tadXNPVbar = npv
End Function

Public Function tadXIRR(ByVal Values As Range, ByVal Dates As Range, ByVal guess As Double, ByVal compounding As Double) As Double
Dim f As Double
Dim fbar As Double
Dim x As Double
Dim x0 As Double
Dim i As Integer
Dim found As Integer

found = 0
x0 = guess

Do While (i < 100)

f = tadXNPV(x0, Values, Dates, compounding)
fbar = tadXNPVbar(x0, Values, Dates, compounding)

If (fbar = 0) Then
tadXIRR = (0) ^ (-1)
Else
x = x0 - f / fbar
End If

If (Abs(x - x0) < 0.000001) Then
found = 1
Exit Do
End If

x0 = x
i = i + 1

Loop

If (found = 1) Then
tadXIRR = x
Else
tadXIRR = (-1) ^ (0.5)
End If

End Function

financial engineer say:
This version of tadXIRR has 3 exit routes for a return value.

1st if the derivative of the function XNPV is 0, it returns a value 0^-1 which is a reciprocal of zero and Worksheet cell shows that as #VALUE since division by zero is not meaningful to the compiler.

The 2nd value is returned when an XIRR is found to 6 digit accuracy.

The 3rd value is returned when iterative calculations do not converege within 10^-6 digits of accuracy, in this case the return value is programmed as -1^0.5 which is the square root of a negative number and the Worksheet cells shows this as #VALUE error since the compiler can't handle this value.

Now there should have been more error checks such as when the function XNPV results in a division by zero, yet this I left out of the code.

When you encounter #VALUE error, and you think there might actually exists an XIRR, you can do a little trick and play around with COMPOUNDING value.

If there were divisions by zero in the calculations for annual compounding value of 1, it may be that a different compounding value will yield a valid XIRR.
