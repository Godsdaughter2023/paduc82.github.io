At first sight it seems the last two lines are wrong. You want to filter IndividualProjectsForm, right?

So write:

with Forms!IndividualProjectsForm

  .Filter = Criteria

  .FilterOn = True

end with


That said, there is another problem. It's better to use ID values rather than Names. Your listbox should have a hidden column with EmployeeID, given by the RowSource query, something like:
select EmployeeID, EmployeeFirstName & " " & EmployeeLastName as EmployeeFullName from tblEmployees Order By EmployeeLastName;

This will make the Filter expression much shorter and easier to deal with. Don't believe me? Enter an employee O'Brien and see how you fare with your code.


Even shorter than all the OR clauses would be an IN-clause:

AssignedTo in (123,456,789)

(note how I also renamed the [Assigned to] field taking out special characters such as [space] which otherwise can result in notational difficulties)

andreas killer say:
For your data sample we can calculate the numbers in column B using this formula:
=INT((RANK.EQ(A1,$A$1:$A$15)-1)/3)+1

You said you want to have it dynamic, so we have to detect the number of used cells in column A:
=INT((RANK.EQ(A1,OFFSET($A$1,0,0,COUNTA(A:A)))-1)/3)+1

Then we can use that numbers to apply 5 conditional formatting formulas to color the cells in column A, the formulas are simply =$B1=1 =$B1=2 etc. see screenshot.

BTW, it is not recommended to use the above formula directly in the conditional formatting, because that requires 5x more computing power, which can lead to a noticeable slowdown of the file in the case of a large amount of data.

Also using OFFSET (to make it dynamic) can have the same effect, because that is a volatile function. Means all this formulas are recalculated every time!

Andreas.
