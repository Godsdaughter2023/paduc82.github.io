angryvoucher say:

Microsoft Excel workbooks are read-only on the phone if they have:

  • Formulas that aren't supported on the phone.
  • Scenarios.
  • Data validation in one or more cells.
  • Array formulas.
  • Data tables.
  • Slicers.
  • Macros enabled.
  • Large column or row widths.
  • Formulas that contain links or references to other workbooks.
  • Formulas with structured references.
  • Formulas that contain references, content or formatting beyond cell IV16384.
  • Longer formulas, such as a formula that is longer than 512 characters or has more than 30 arguments in it.
  • Shared workbooks with tracked changes.
  • Workbooks with dialogue sheets.
  • XML markup that isn't formed correctly.
  • OLE objects or ActiveX controls.

zorvek say:

I've done a little more research.

The reason the date is displayed that way and that it cannot be changed is that it is how the "General" format has been configured by Microsoft and it has been so for years.

But it isn't quite that simple. If you enter a year then Excel will change the cell format to "Short Date". A date with a year can be either "1/1/15" or "1/1/2015".

Why it won't set the number format to date when only a month and day are entered is a mystery to me but it might have something to do with the frustration users experience when entering values that might be something other than a date ("1/2" or "1-2") that Excel choose to interpret as a date but which the user is not intending to be a date. Keeping the number format "General" in that case give the user the opportunity to alter the value without resetting the number format to "General" which is preferred for most values (for non-power users anyway.)

Note that Excel will also switch the number format to "Currency" if we enter "$123.45" and "Custom" with the format "h:mm AM/PM" if we enter "10:15 AM".

It's a bit random and, given that very aspect of the process, one would think Microsoft would give us more control over either how "General" displays things and/or how it changes the number format when various values are entered into a "General" formatted cell.

The bottom line is that you really want the cells' formats to be "Date" and you can either include the year when entering dates, or set the cell's format manually. It's not ideal for you but it's the best you can do.

You also asked about the default Values versus Formulas setting in the Find dialog. Two things. First, once set in an Excel session, the setting remains. Second, it kind of makes sense in that a value entered is treated as a formula when searching. The only reason you would select "Value" instead of "Formula" is when you specifically want to search for formula results in addition to actual values entered. It's not a perfect system - Microsoft has set the default to satisfy what they perceive are the most users and, if Microsoft gave us an option for every request like yours, they would never get the next release out ;-)

Kevin
