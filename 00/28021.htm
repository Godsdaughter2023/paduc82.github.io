qa guy commelec say:

It is not logical to multiply the year by 53!    That is  2016 * 53  which equals 106848.

Bear in mind that Access defaults the week starting on Sunday.

Current Week:    This has week of the year AND year to match ---

WHERE (DatePart("ww",[Days])=DatePart("ww",Date())) And (Year([Days])=Year(Date())));

Following Week:     This has week of the year to match current week plus 1 AND year to match --

WHERE  (DatePart("ww",[Days])=DatePart("ww",Date())+1) And (Year([Days])=Year(DateAdd("w", 1, Date()))));

Third Week:     This has week of the year to match current week plus 2 AND year to match ---

WHERE  (DatePart("ww",[Days])=DatePart("ww",Date())+2) And (Year([Days])=Year(DateAdd("w", 2, Date()))));

Table of contents page numbering: P1-1, P2-1 - Microsoft Community

mpennicott say:

Table of contents page numbering: P1-1, P2-1

Is it possible to generate a table of contents that draws in the Part number before the page number eg. for Part 1: P1-1, P1-2, P1-3, then for Part 2: P2-1, P2-2, etc.
suzanne s. barnhill say:

The table of contents reproduces the page number that is (or would be) printed on the page by the PAGE field alone. The only way to include something other than the actual page number in the PAGE field is by using the "Include chapter number" feature in the Page Number Format dialog. To be considered a "chapter number," the number must be the number of an autonumbered paragraph in one of Word's built-in heading styles. So if you use, say, Heading 1 for your part title and format the numbering of the heading as P#, then this could be included.

If your part titles are not actually numbered, you can add the numbering to the style and format it as Hidden (selecting "Nothing" for "Follow number with"); the PAGE field can still pick up the hidden numbering, but it will not print as part of the heading. If the part title is numbered but doesn't include P, you can still type P manually before the part number (in the page number) in the header or footer, but this will not appear in the TOC.

This is probably not the best approach. Make sure this is really what the printer wants. Most commercial printers will want the document in single-page sheets (you can make the page any size you like and still print it on Letter-sized paper).
