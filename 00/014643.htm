suzanne s. barnhill say:

There is indeed a longstanding bug in the "Book fold" implementation that implements "Mirror margins" backward. You'll just have to reverse your Inside and Outside margins (no need for a gutter).

In any document (not just "Book fold"), if you have either "Mirror margins" (enabled by default in "Book fold") or "Different odd and even" headers/footers enabled, you will see facing pages when you display two pages in Print Preview--but not in Print Layout view.

In order to get a Print Preview in which you can see two pages, however, you will need to use Print Preview Edit Mode. This command cannot be found on the Ribbon, but you can add a button to the QAT; find it under All Commands in the Customize Quick Access Toolbar dialog.

The reversed mirror margins bug has existed since "Book fold" was introduced in Word 2002; my guess would be that it has not been fixed because this would break documents created in earlier versions, though I would think this could probably be handled with a Compatibility Options setting.

kosh vorlon say:

Hi Franky,

 

You'll be fine running the MSE scan without being an administrator and lose nothing simply by being a standard user (but note that you won't be able to change the parameters/options/settings of the scan unless you run the program as an administrator {like exclusions or including removable drives or archived files} - and that may include access to files/folders/drives to select them to be scanned in custom mode or through Windows Explorer - but the latter doesn't apply to quick or full scans where no such selection exists).

 

You do not need to logout of the standard account and login to the administrator account to run MSE as an administrator.  I'm not 100% if this will work if MSE is already running (I suspect not), but you can try going to the program file in Windows Explorer (probably C:\Program Files\Microsoft Security Client\) and right click on msseces.exe and then click on Run as Administrator and then only MSE will be running with Administrator priviledges which is better than your entire profile). 

 

If it denies access because the program is running (it's possible, I've not actually tried to do this while the program was running) and you need to change paramenters before running the scan, then you can either disconnect from all networks and peripheral devices (to isolate the computer from external infection) and use Task Manager to end msseces.exe (and possibly also MpCmdEng.exe - possibly up to 3-4 times as it may keep returning for a bit).  Once complete and neither show in Task Manager and the icon is gone from the taskbar notification area, the above should work to open MSE ONLY in Administrative Mode to make the appropriate adjustments and then do what you want.  When done, if you want to return MSE to Standard Mode, you'll need to repeat the process except this time just double click the program icon instead of right clicking to use Run as Administrator and you'll be back to standard mode.  Then you can reconnect your network and peripheral devices and run the scan or do whatever you want to do.

 

If you don't have scan removable drives checked it means that only internal drives will be scanned with a fill scan.  If you have multilple internals drives or multiple partitions on one drive or you have an internal CD/DVD drive (or several with disks in them), then all such drives will be scanned with a Full Scan.  The Removable drive box will exclude external USB drives (hard drives or even an A drive with an old Floppy Disk in it) or Flash Drives or similar removable devices.  Just because you can remove a DVD from an internal drive does not mean that a Full Scan will treat the drive as removable if a disk is inserted (but if you remove the disk, it'll look, find nothing there, and move to the next drive if any).  If you have a System Restore Partition on your drive, it will scan that partition.  A Full scan also scans the registry and some other areas of the system that selecting a specific drive will not include.

 

It is not the same as right clicking C-drive and choosing to scan with MSE.  That will only scan C drive and works similar to a Custom Scan rather than a Full Scan.  It will not scan the registry and other areas that would be covered with a Full Scan.  If you have other internal drives you want to avoid scanning but want to do a Full Scan including C-Drive you can login as an adminstrator and exclude all those drives but C drive, run the Full Scan, and then include the other drives back again when done so they'll be protected real time.  Alternatively, you can run a Quick Scan (which will get the registry and other parts missed by a Custom Scan) followed by a Custom Scan of just C: Drive and that will accomplish the same as a Full Scan of C: Drive would have done if other internal drives or partitions exist and you don't want to scan them.

 

I hope this answers your question.  If I've missed some aspect of it that still concerns you, please post back and I'll be happy to clarify it.

 

Good luck!

 

Kosh

suznal say:

Yes

 

I will give you a working sample!!!

I have code behind Excel (you can modify as needed) because I enter data into the workbook, which then updates a bunch of other workbooks, after which i send each of them as an attachment in their own emails.

Like I said, you will have to modify the code for your purpose, but it is very do-able.

 

In my example, I got rid of all of the code that isn't pertinent to your question to make it easier to read...

 

 

 

Dim i As Integer
Dim OutLookApp As Object
Dim MItem As Object
Dim Subj, EAdd, Rcpt, EMBdy As String
Dim FlNme As String


EAdd = "*** Email address is removed for privacy ***"
Subj = "your weekly file"
Rcpt = "John"

EMBdy = "Dear " & Rcpt & ",<BR><BR>"
EMBdy = EMBdy & "Your weekly file is attached."

 

With Application.FileSearch
    .NewSearch
    .LookIn = "P:\temp"
    .SearchSubFolders = True
    .FileType = msoFileTypeMailItem
    If .Execute(SortBy:=msoSortByFileName, SortOrder:=msoSortOrderAscending) > 0 Then
        
        For i = 1 To .FoundFiles.Count
            FlNme = .FoundFiles(i)
            Set OutLookApp = CreateObject("Outlook.Application")
            Set MItem = OutLookApp.CreateItem(0)
            With MItem
                .To = EAdd
                .Subject = Subj
                .HtmlBody = EMBdy
                .Attachments.Add FlNme
                .Send
            End With
        Next i
    Else
        MsgBox "There were no files found."
    End If
End With

 

 

 

 

 

Enjoy!

 

 

 

Edit: Note that by "modify" I mean that you must edit the filepath as well as email subject, recipient, etc.,.

Additionally, since this is run from Excel, you don't have to set the application, bt rather just create a new mail item.

If you would like to review the email before it is sent, then replace

 .Send

with -

.Display

 

But should you just want to test the code as is - edit the filepath and place the code in Excel (or Word) and run it - and you will see that it works just fine!
