    Outlook 2010, I can no longer send from the outbox. Emails will no - Microsoft Community

    richard bazinet say:

    Outlook 2010, I can no longer send from the outbox. Emails will no longer leave the outbox and the date shows as 'none'

     

    Outlook 2010, I can no longer send from the outbox. Emails will no longer leave the outbox and the date shows as 'none'.

    Email messages only send only works in 'immediate' send. The outbox is no longer working.

     

    graham mayor say:
    This often happens if you work with the Outbox open, however all is not lost. The following Outlook macro will force the sending of all messages in the Outbox. The commented out sections are optional. Copying macro listings to Outlook is fairly straightforward and similar to other applications e.g. see http://www.gmayor.com/installing_macro.htm

    You may however have to use SelfCert.exe (which will be in the same folder as Outlook.exe) to create a certificate that can be used to sign the project, in order to avoid security messages when running the code.

    Sub SendMessages()
    Dim olItems As Outlook.Items
    Dim olItem As Outlook.MailItem
    Dim olNS As Outlook.NameSpace
    Dim olSycs As Outlook.SyncObjects
    Dim olSyc As Outlook.SyncObject
    Dim i As Long
    Set olNS = Application.GetNamespace("MAPI")
    Set olSycs = olNS.SyncObjects
    Set olItems = Session.GetDefaultFolder(olFolderOutbox).Items
    For i = olItems.Count To 1 Step -1
        Set olItem = olItems(i)
        olItem.sEnd
    Next i

    For i = 1 To olSycs.Count
        Set olSyc = olSycs.item(i)
        'If MsgBox( _
         "Do you wish to synchronize " & _
         olSyc.Name & "?", _
         vbYesNo) = vbYes Then
        olSyc.Start
        'End If
    Next
    cleanup:
    Set olItems = Nothing
    Set olItem = Nothing
    Set olNS = Nothing
    Set olSycs = Nothing
    Set olSyc = Nothing
    End Sub

    VBA code for displaying a message box for a max duration - Microsoft Community

    ab_c say:

    VBA code for displaying a message box for a max duration

    Hi

    I have a code in which at some instance I would like to display a MSGBOX with possible options for user to click upon.

    However, I would also like to add some logic that assumes a default value if the user has not clicked on any option till "nn" seconds (or minutes).  Thus, a kind of a timer whereby if beyond a certain time no option is taken then the msgbox disappears and the macro continues from next statement.

    Is there some technic to achieve this?  If it means replacing MSGBOX with something else then I am fine too.

    Thanks in anticipation

    graham mayor say:

    Create a userform with a combobox and a label.

    The following macro will populate the userform and will run a 20 second timer. If the user makes a selection from the  combobox the timer will stop and the userform will close. Add whatever processes you need:

    Private Sub UserForm_Activate()
    Const strMsg As String = "Seconds Remaining ... "
    Dim iDuration As Integer
    Dim StartTime As Variant

        Me.ComboBox1.AddItem "Item 1"
        Me.ComboBox1.AddItem "Item 2"
        Me.ComboBox1.AddItem "Item 3"

        iDuration = 20        'seconds
        Me.Label1.Caption = strMsg & iDuration
        StartTime = Timer
        While Timer - StartTime < iDuration
            Me.Label1.Caption = strMsg & iDuration - Round((Timer - StartTime) * 1, 0)
            If Me.ComboBox1.ListIndex > -1 Then
                iDuration = 0
            End If
            DoEvents
        Wend
        Unload Me
    End Sub

    DLookup on form does not automatically return value - Microsoft Community

    lakesidemom say:

    DLookup on form does not automatically return value

    I have a field that is filled by a DLookup function on a form. All other calculated fields fill in automatically but the DLookup field only populates after I enter the function builder and hit ok. What can I do to make it automatically fill the field?

     

    john w. vinson say:

    =DLookUp("[BoltHoleDiameter]","[tblBoltHoleDiameters]","[BoltSize]  Like [ABoltSize]")

    As written this will not reference the form at all - it will return those records in tblBoltHoleDiameters which have the field BoltSize equal to the field ABoltSize in that table (or an error if there is no such field). I think you want to pull the criterion out of the quotes. Assuming that BoltSize is a Text field then

    =DLookUp("[BoltHoleDiameter]","[tblBoltHoleDiameters]","[BoltSize] ='" & [ABoltSize] & "'")

    should work; if ABoltSize is a Number field drop the quotes:

    =DLookUp("[BoltHoleDiameter]","[tblBoltHoleDiameters]","[BoltSize] =" & [ABoltSize])
