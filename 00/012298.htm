moranyan say:

Hi GerardoSoto,

Regarding “I’ve learnt that the frequency of the queries is dictated by the TTL in the DNS response”, actually, the TTL in the DNS response has nothing to do with the frequency of the queries. The explanation of TTL (Time to live) is that if a DNS record was changed on the authoritative nameserver, DNS servers around the world could still be showing the old value from their cache after the change. For example: A TTL value of 86400 seconds would mean that, if a DNS record was changed on the authoritative nameserver, DNS servers around the world could still be showing the old value from their cache for up to 24 hours after the change.

About “is this 1-minute TTL imposed by Microsoft somehow? Maybe a default setting that customer can change in their O365 settings?”, I’d like to tell you the TTL of the DNS records is not imposed by Microsoft and the value of the TTL can be changed. The value of the TTL can be changed from 60 seconds to another value (e.g. 24 hours <86400 seconds>). Meanwhile, the suggested value of TTL that is related to Office 365 DNS records is 1 hour (3600 seconds).

If anything is misunderstood, please don’t hesitate to post back your detailed requirement so that we can help you further.

Thanks,

Mouran
