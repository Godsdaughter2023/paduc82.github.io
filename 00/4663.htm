Ex:

Values are in A1 through L1

I am trying to using IF statement as such:

=IF(AND(A1:L1>=1, A1:L1<= 2), "PASS","FAIL")  however I keep getting "#VALUE!"

You can use this:

=IF(AND(AND(A1>=1,B1>=1,C1>=1,D1>=1,E1>=1,F1>=1,G1>=1,H1>=1,I1>=1,J1>=1,K1>=1,L1>=1),AND(A1<=1,B1<=1,C1<=1,D1<=1,E1<=1,F1<=1,G1<=1,H1<=1,I1<=1,J1<=1,K1<=1,L1<=1)),"Pass","Fail")

Or can use a simple alternative formula:

=IF(AND(MIN(A1:L1)>=1,MAX(A1:L1)<=2),"Pass","Fail")

victor.r say:

How do I create links to different parts of the same word document?

I wrote a list of 120 references at the end of a large document. Now, I need to create links from other parts of the document to those 120 references at the end of the document.

Explanation:

1. Somewhere in the document I write something like this... "This issue was addressed in manual 23"

2. I need manual 23 to be clickable and take the reader to that reference section at the end of the document to the line that starts with manual 23

How do I do this?

suzanne s. barnhill say:

There are two ways to create clickable cross-references: with hyperlinks and with hyperlinked cross-references.

  1. If your reference list is autonumbered, then you could use a reference such as "This issue was addressed in manual 23 (4)," where (4) is a hyperlinked cross-reference to the paragraph number of the reference listing. To insert that, you would use References | Captions | Cross-reference. In the Cross-reference dialog, select "Numbered item" as the "Reference type" and "Paragraph number" for "Insert reference to." Then select the appropriate numbered item and click Insert. The "Insert as hyperlink" check box is checked by default.
  2. Although a hyperlinked cross-reference is by far the easiest approach, it does not allow you to specify display text different from the referenced item. For that you'll need an actual hyperlink. Unless your references use a built-in heading style, you'll have to bookmark them. This means that you have to select all or part of each reference item and manually insert a separate bookmark for each one (Insert | Links | Bookmark). The bookmark must be a single word; it cannot contain spaces, but you can use manual23 or manual_23. After you've bookmarked each reference, you can use Insert | Links | Hyperlink to hyperlink to it. If you select text before opening the Hyperlink dialog, it will be inserted as the "Text to display." Under "Link to," select "Place in This Document" and then click on the appropriate bookmark. You can also specify a custom ScreenTip if desired.

maor bracha say:

Hi Ashraf,

If I understand correctly, you intend on adding an Exchange 2013 to your existing Exchange 2010 Hybrid organization?

In that case, you should make sure to:

  1. Import the SSL certificate used for the TLS on your Exchange 2013 servers as well, and assign it the SMTP service.

  2. Add the Exchange 2013 server(s) to your Hybrid configuration either by rerunning the HCW, or manually adding where needed (Send\Receive connectors, as one example).

As far as accepted domains go- in short, yes. You would have to add and verify all of the domains in use in order for Office 365 to be able to use them.

Don't worry, though. You can use PowerShell in order to automate this process.

On your On-Premise server, run:

Get-AcceptedDomain | Export-Csv <Domain list file path>;

(Replace <Domain list file path> with the desired CSV output file path.)

Connect to Office3 65 PowerShell, and run:

$Domains = Import-Csv <Domain list file path>;

Foreach($Domain in $Domains)

  {

  New-MsolDomain -Name $Domain.DomainName -VerificationMethod DnsRecord;

  $VerificationTXTcollection+=@(Get-MsolDomainVerificationDns -DomainName $Domain.DomainName -Mode DnsTxtRecord;

  }

$VerificationTXTcollection | Export-Csv <TXT path>;

(Replace <TXT path> with the TXT records CSV output file>)

This would loop all your domains, add them to Office 365 and export the TXT records to a CSV file.

Once you added all your TXT records, run:

$Domains = Get-MsolDomain -Status Unverified;

Foreach($Domain in $Domains)

  {

  Confirm-MsolDomain -DomainName $Domain.Name;

  }

Cheers,

Maor Bracha

The answer is no. It is not feasible to import the SSL certificates.
