I thought it only did an autosave if you made a change since the last save, and I was careful to save my work before starting the presentation. How do I disable this feature to prevent it happening again? Thank you. 
steve rindsberg say:

PowerPoint doesn't actually do auto-saves.  Not in the sense that most of us assume "auto-save" means.  It's actually called AutoRecover, in fact.

It periodically saves the current presentation in a form that only it can read, then offers to restore the file if a) it crashes and b) it NOTICES that it's crashed.  Otherwise its autosaved file is about as useful as a tux on a tincan.

In the Options dialog | Advanced section, you can turn off AutoRecover altogether or set it to occur less often than the default of every 10 minutes.

jeeped say:

Any ideas?


You are going to need a helper column that will format the numeric portion of the alphanumeric entries into a sortable format.

                                         

Note that I've added a couple of purely alphabetic entries. The formula in B2 is,
=IF(ISNUMBER(1*MID($A2,ROW($1:$9),1)),TEXT(IF(ISNUMBER($A2),$A2,1*MID($A2,MATCH(TRUE,ISNUMBER(1*MID($A2,ROW($1:$9),1)),0),COUNT(1*MID($A2,ROW($1:$9),1)))),"00000")&RIGHT(A2,LEN(A2)-LEN(MID($A2,MATCH(TRUE,ISNUMBER(1*MID($A2,ROW($1:$9),1)),0),COUNT(1*MID($A2,ROW($1:$9),1))))),"00000"&A2)
... which is an  array formula and needs to be entered with Ctrl+Shift+Enter rather than just Enter. The formula can then be filled down to cover the entries in column A.
Select the entire range (e.g. above as A1:B11) and select Data, Sort. by the Helper column B. When you click OK, you may receive a Sort Warning.

                           

Use the settings as above and click OK. The results should mimic these,

                                         

There are a couple of things to note. Array formulas use computer resources exponentially. If you have thousands of rows, expect delays. I wasn't sure if you had any purely alphabetic entries. Chopping out the section that covers for that possibility would help if it is not needed. I used a 5 zero padding on the numbers. Increase that if you have numbers with more than 5 digits.

More information on the collection of numeric values from alphanumeric entries at,
Extracting numbers from alphanumeric strings
