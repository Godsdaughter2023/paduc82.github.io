¡firedog say:

My contacts list within live mail ...

You have at least two different contacts lists in Windows Live Mail:

 

  • When you're Home tab of the ribbon reads Sign in and the contacts list you see is the default list.
     
  • When you are signed in with a Microsoft Account, the right-most button on the Home tab of ribbon reads something other than Sign in and the contacts list you see is the one that belongs to the account you're signed in with. The list synchronizes with the one at contacts.live.com. Any change you make to this list in Windows Live Mail will propagate to the online list and vice versa.
     
  • Use the button to change your signed-in state and hence the contacts list you see. If you sign in with a different Microsoft Account, you'll see a different contacts list.
     
  • There is no connection between the different contacts lists you see in Windows Live Mail.
     
  • All of your contacts lists are stored locally on your computer in databases. The data they contain are only accessible through Windows Live Mail.

    Will I lose my roadrunner emails in storage folders when I cancel Time - Microsoft Community

    photofriend say:

    Will I lose my roadrunner emails in storage folders when I cancel Time Warner?

    I'm switching from TWC to AT&T and using Windows 7 Live Mail.  I have set up a new gmail email account but don't want to lose all my old roadrunner emails I have in storage folders.
    ¡firedog say:

    Depends what you mean by storage folders.

    If the folders you're talking about are under Storage folders below the Outbox in the folder pane, then these won't be affected at all by any change of provider or addition of mail accounts.

    If on the other hand they're under an account name above the Outbox, then they will be removed and their contents permanently deleted if and when you remove the account concerned. There's no need to remove the account, though; leave it active to pick up any messages from people who don't know you've changed address. When you start seeing password errors on it, indicating that TWC have closed it down, right-click on the account name in the folder pane and select Properties. On the General tab, deselect Include this account when receiving mail or synchronizing.

    Just reinstalled Windows, permissions on secondary drive are screwy now

    My 320GB Hard Drive which had Windows 7 installed on it crashed. I also had a secondary 1.5TB drive for storage. I bought a new 500GB drive and installed Windows 7 on it fresh. Now I think all the permissions on that secondary drive are messed up because they were originally owned by the previous installation. At first I was getting errors in programs that I could not write files because I did not have permission, so I told the hard drive to give me ownership of everything on it.

     

    That seemed to work for a while, but now - and this is possibly even more annoying - when I try to move or copy files, even though I am the owner, it keeps popping up this box (Which NEVER happened on the previous installation, by the way) that tells me I need Administrator Permission to move the files. The first time I saw this, my account was set up as the Administrator, so I was like "What the heck?" but I hit the little "Yes" with a shield next to it, and the files will move as I instructed.

     

    After spending a ton of time looking on the internet for information about this, it looked like every time someone asked something similar to this, they would immediately get told, before anything else, that they should not be on the admin account, so I figured that was probably a good idea anyway, and who knows maybe it'll fix the problem, so I made an admin account, logged into it, and set my main account to a Standard user. Now, logged into my account as a Standard User, when I try to move files similarly, the same box pops up, except now I have to enter the Admin account's password to move the files, too!

    I really do not want this obscene level of security on a single user computer, but I figure it all has something to do with the previous installation of Windows and the file system on that secondary drive.

     

    Long story short, I want the nagging box to go away, and I am just out of ideas.
    garylisk say:

    I solved my problem, I think.

     

    First: I removed that user, which after researching online, i found was just as I thought, an old leftover fromt he previous installation of windows. Removing it from one folder's permissions list seemed to work well, and removed it from everything.

    Second: Right clicked the D drive, went into security, hit Advanced, hit Change Permissions, opened my standard user, and cleared the permissions. I hit OK, checked the box on the next screen to replace all child object permissions, and hit OK. It took a while to change all the permissions, but then I checked various folders and files to confirm that the standard user was on none of the lists.

    Third: Once this was done, I did the same thing AGAIN. Right clicked the D drive, went into security, hit Advanced, hit Change Permissions, opened my standard user, and gave him FULL CONTROL. I hit OK, checked the box on the next screen to replace all child object permissions, and hit OK. It took a while to change all the permissions, but then I checked various folders and files to confirm that the standard user was on all of the lists with full control.

    Fourth: I changed back to the standard user account and tried to move one of the previously unmovable files, and it worked just fine.

    MY THEORY OF WHY THIS HAPPENED: When I reinstalled windows, I chose the same username for the main account again. This may have confused the file system, so I needed to get rid of all remnants of the old installation. Since my computer name and user name were identical, I just needed to trash the permissions for the old compname/username and then reset them again for the current compname/username.

    I could be WAY OFF with that theory, but this seems to have worked.

    One bit of research I could do, but am not going to do, would be to reinstall Windows again and pick a different computer name and user name to see if the same issues happened, or if the new permissions set properly. Or heck, even if the old permissions were still there, but once I took control initially, it would work. But I am not going to reinstall Win again, so I won't know that.

     

    Basically, I think when you have a situation like mine, it may be that just telling Windows to take ownership of the whole drive is not good enough, and you need to make sure all the old stuff is removed before doing so. (If your compname and username are the same thing again.)

    Seeing as how young Win7 is, I can imagine the percentage of people who have reinstalled like this is pretty low still, so this is probably a problem not often encountered. You may want to have some folks at MS do some testing on this!

     

    Thanks for all the assistance, I'll be sure to check back in if there's any other issues and let you all know!
