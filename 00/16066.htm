lz. say:

Hi

Problem very well explained (wish they would all be like yours…)

IMHO an IF isn't the best approach here. In both situtations (Activity/Status = Booked & Activity/Status = SomethingElse) you want to keep the 4th column.

Currently it's name is either [Column4] or [Activities] but as you pull information from an external source and you don't have the hand on it, sooner or later those names could change. So instead of testing (IF or anything similar) on the column names I think it makes more sense to do the following…

Assume the blue Table1 is my Source (the green one is my output - the 4th column):

Independently of the name of the 4th column (currently [gh]) my query will work if I do the following in the query code:

let
    Source = Excel.CurrentWorkbook(){[Name="Table1"]}[Content],

    FourthColumName = Table.ColumnNames(Source){3},
    RemovedOtherColumns = Table.SelectColumns(Source, FourthColumName)

in
    RemovedOtherColumns

Table.ColumnNames returns the column names of the referenced table (Source above) as a List. And Table.ColumnNames(Source){3} returns the 4th (List index starts at zero) item of the List, so "gh" in the above example

After that, instead of hard-coding the name of the 4th column ([Column4] or [Activities] in your case) in the Table.SelectColumns, use the name of the Step (FourthColumnName above): Table.SelectColumns(Source, FourthColumnName)

Hope this help & makes sense. Any question please let me know

If this solves your problem please mark as Answer - this might help others with a similar scenario
