Sub TestIt()
    Dim vntYear As Variant
    
    ' Test conditions.
    ' Empty,Null,"","abc",99,"99",1900,"1900",1904,"1904",4000,"4000",Date,Now()
    vntYear = 1900
    
    ' For Logical Purposes.
    If IsLeapYear(vntYear) Then
        MsgBox "Is a Leap Year"
    Else
        MsgBox "Is Not a Leap Year"
    End If
    
    ' For Display Purposes.
    MsgBox IIf(IsNull(IsLeapYear(vntYear)), _
               Chr(191) & CStr(Nz(vntYear, "")), _
               IIf(IsLeapYear(vntYear), _
               CStr(CBool(Not 0)), _
               CStr(CBool(0))))
    
End Sub


Public Function IsLeapYear(ByVal vntYear As Variant) As Variant
    
    IsLeapYear = Null
    
    ' Only process for Not (Null XOR Empty XOR Zero Length String).
    If Len(vntYear) Then
        If IsDate(vntYear) Then
            vntYear = Year(CDate(vntYear))
        End If
        
        If IsNumeric(vntYear) Then
            IsLeapYear = False
    
            Select Case True
                Case vntYear < 100 Or vntYear > 3999
                    IsLeapYear = Null
        
                Case vntYear Mod 100 = 0
                    If vntYear Mod 400 = 0 And vntYear Mod 4 = 0 Then
                        IsLeapYear = True
                    End If
        
                Case vntYear Mod 4 = 0
                    IsLeapYear = True
        
            End Select
        End If
    End If
    
End Function

Bad dates…
"0A1-001200" is a valid date as far as Access is concerned.
MsgBox IsDate("0A1-001200")  ‘ << True

This sort of ‘strange’ date is included simply because it is a valid date, as far as Access is concerned.
“0A1-001200” returns True as a leap year.

MsgBox IsDate("0A1-001201")  ‘ << True
This is also a valid date, as far as Access is concerned.
“0A1-001201” returns False as a leap year.

(They have now been included in the test conditions below.)

So, we go with the flow even if we don’t understand the date format.


Date range in Access…
Dates in Access are valid from year 100 to year 9999 inclusive.
I don’t understand the reasons for those limits but the lower limit also excludes two digit years. Perhaps that’s a good thing.
The upper limit of the leap year calculation is still not known for sure.
http://en.wikipedia.org/wiki/John_Herschel
However, the code below regards the years less than 100 and more than 3999 as unknown and therefore returns Null. Make of it as you will but the intention is to regard those years as logically False and display them as an inverted question mark with the passed argument, if there is a visible passed argument, else just display them as an inverted question mark.


Proposed solution…
The code supplied should correctly handle all passed arguments. (needs further testing.)

This means that, apart from the obvious special cases of Null and suchlike, it should handle a year as specified in the original question. A year may not be part of a Date but rather 1900 or “1900”. Note that this is the year of the 123 <chuckle>. The difference between 1896 and 1900 and 1904 is important (True, False, True.)

The code supplied should also supply a logical return and a display return.
A function which returns a Boolean can only return two states, False or Not False.
But because there is an unknown factor in the calculation it seems inappropriate to me to restrict the return value to one XOR the other. There are unknown dates and unknown arguments and to say that the unknown is either False XOR Not False would imply that the unknown is known.

To overcome that dilemma the function returns a triple state Variant. The function returns False, Not False or Null (Unknown).  Logically, since Null can not be Not False it is regarded as False. Hence it takes up position with other logically False returns.

But with the display, things are different, we can triple state. False is False and True is Not False and the third state (Null) can display the reason neither of the other two were displayed. Access normally displays a triple state being one or the other or error. This is an attempt to display the error state as the reason for the error, not just error.


IIF(Day(Dateserial(Year(Date()), 3, 0) = 29, "This is a leapyear", "This is not a leapyear")

That is not acceptable by the editor but can still be used to reduce the complexity.
It will still need pre conditioning to handle the passed arguments.

Sub TestIt()
    Dim vntYear As Variant
    
    ' Test conditions.
    ' Empty,Null,"",99,"99",1900,"1900",1904,"1904",4000,"4000"
    ' Date,Now(),"abc","0A1-001200","0A1-001201"
    vntYear = "0A1-001200"

    If IsLeapYear(vntYear) Then
        MsgBox "Is a Leap Year"
    Else

    End If

    ' For Display Purposes.
    MsgBox IIf(IsNull(IsLeapYear(vntYear)), _
               Chr(191) & CStr(Nz(vntYear, "")), _

               CStr(CBool(Not 0)), _
               CStr(CBool(0))))
    
End Sub

my Windows Defender does not offer a remove button - Microsoft Community

donaldpatterson1 say:

my Windows Defender does not offer a remove button

When and how does Defender clear the quarantine as it displays no option to remove its contents
greginmich say:

You can either click on the Remove all button or click on a down arrow in order to display the option buttons for each individual threat. But remember that the recommended action can sometimes be wrong when a harmless file gets quarantined by mistake (a false positive), and that's the reason why threats are quarantined rather than removed automatically. The items in the Quarantine could be safely left there indefinitely; but they'll be automatically removed after 90 days, by default (the documentation is wrong).

The number of days that the item is kept in the Quarantine can be reconfigured with this preference setting:

-QuarantinePurgeItemsAfterDelay

Specifies the number of days to keep items in the Quarantine folder. If you specify a value of zero for this parameter, items stay in the Quarantine folder indefinitely.

https://docs.microsoft.com/en-us/powershell/module/defender/set-mppreference?view=win10-ps

For example, to have quarantined items automatically removed after a month; just copy, paste, and enter this command at the Administrator PowerShell prompt:

Set-MpPreference -QuarantinePurgeItemsAfterDelay 30

GreginMich

how to unlock my windows phone lumia 1520

jack cook - aka help_line say:

I'm not sure if you are referencing the PIN code to unlock or if you want the recovery key to reset the device if it had device protection enabled.

If it is the PIN code to unlock the device and if you forgot it the only choice is to Hard Reset. If you DO have reset protection enabled then get the recovery key first by doing what is indicated below. After you get the key then hard reset

Here's what you can do to get the Recovery Key:

  • Go to account.microsoft.com/devices.
  • Next to the phone you no longer own, look for "Get Recovery Key"
  • Click on "Get Recovery Key"
  • Select the "I'm ready to get a recovery key. 

Now you can hard reset your device. (look here: http://resetwindowsphone.com/tag/nokia-lumia-1520/)

  1. It is a good idea to charge your phone to at least 50%
  2. Turn off your phone
  3. Hold down the Volume Down button during 3 seconds, and then hold down  and release the Power button, while still pressing the Volume Down button.
  4. Let go of the Volume Down button and go through the following sequence: VOLUME UP -> VOLUME DOWN – POWER – VOLUME DOWN.
  5. Windows Phone will now start the process to return your phone to factory settings. This process will take about 5 minutes. After the process is finished, the Nokia Lumia 1520 will re-start, and the device will be in the same state it was when it came out of the box.
