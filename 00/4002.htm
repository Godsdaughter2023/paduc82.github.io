If the original styles were not set to Automatically Update, the next most likely factor is that what appears to be several paragraphs actually is only one.

If you aren't doing so already, display the non-printing characters [¶]. They are indispensable when attempting to modify or revise existing documents. Having them displayed typically makes you aware of the potential impact of making a change before you make it rather than having to figure out what happened after making it. What you'll probably find is that what appear to be separate "paragraphs" are separated by a soft returns [aka, line breaks] ↵ rather than a paragraph break ¶.

There's no need for an "epic battle"... when something goes awry the typical way to recover is to use Undo [Command+Z]. Other helpful tools when reformatting are the View> Reveal Formatting command as well as the Clear Formatting option listed in the Styles section of the Formatting Palette.

As for the Style for following paragraph parameter: I'm afraid you seem to have the wrong impression :-) It does not pertain when changing the formatting of existing paragraphs. IOW, if you apply a style to an existing paragraph the next existing paragraph will not have the designated next style applied to it.

It works on an as you type basis so that when you apply the style to a paragraph then press return the next paragraph automatically has the specified next style applied. Also, in existing text which has the 1st style applied, if you press return at the end of that paragraph the one you're about to type will be formatted with the 2nd style.

guessed say:

The size of the header in any particular section is dependent on both the size of the size of the Page Setup Margins and the content that is in the margin. If you header content is larger than the margin allowed in the page setup then that header content just pushes the other page content further down the page. So the content of the header overrides the margin size (unless the header content is a floating text box or graphic)

You need to set the page setup margins to the 'smallest' margin that you want for the section.  Your first page header can contain any content you want to put in there. Hi Jorge,

This method is applied to you shared via right-clicking the inbox of the shared account. If you have assigned full access permissions in Exchange Admin Center, it will not work. So, in this situation, I suggest that you use an admin account to log into Exchange Admin Center firstly, and then check the permissions.

Another method: remove full access permissions via Power Shell:

Make sure Mailbox Full Access Permission is remove

Remove-MailboxPermission -Identity "your shared mailbox name" -User "the user name" -AccessRights FullAccess

Add-MailboxPermission -Identity "your shared mailbox name" -User "the user name" -AccessRights ReadPermission

Make sure only Reviewer permission is given to MailboxFolder Inbox, and others

Add-MailboxFolderPermission -Identity "your shared mailbox name":\inbox -user "the user name" -AccessRights Reviewer

Before that, use an admin account to connect to Exchange Online Power Shell.
