ronnie vernon say:

(Sorry if this isnt in the right topic)


Hi

 

The difference, if you disable the Search Indexer will be the speed of the searches you perform on the system. With the Search Indexer enabled, searches will be instantaneous, and with it disabled they will take a long time. This is the only performance issue if you disable that component.

 

If you have recently enabled the Search Indexer, it is normal for this component to use a lot of resources, however after the indexing is complete, the resources will be negligible.

 

The Search Indexer is also designed to perform the indexing when the computer is Idle. If you start using the computer, the Indexer will pause until it detects an idle state.

Port Forwarding

Login Information

  • Gateway - 192.168.2.1
  • Username - admin
  • Password - admin / admin1

Port Forwarding

  1. Open a web browser.
  2. Enter http://192.168.2.1 into the address bar. (Or click the link) Enter the username and password if the gateway asks. (admin/admin or admin/admin1)
  3. Select Advanced setup from the top menu.
  4. Select Port Forwarding from the left menu.
  5. Enter "3074" in both the Start and End boxes.
  6. For Protocol select "Both" from the dropdown. (If you don't have both create two of the exact same rules where you select UDP and TCP independently).
  7. Enter the Xbox’s IP address into the LAN IP Address area.
  8. Click Apply.

 

Disable Port Forwarding

  1. Open a web browser.
  2. Enter http://192.168.2.1 into the address bar. (Or click the link) Enter the username and password if the gateway asks. (admin/admin or admin/admin1)
  3. Select Advanced setup from the top menu.
  4. Select Port Forwarding from the left menu.
  5. Select the Edit button on the Applied Rule you would like to remove.
  6. Delete the rule.
  7. Click Apply.

I would like to inform you that you can use Event Viewer application included in Windows 10 to make a note of all the significant events. Event logs are special files that record significant events on your computer, such as when a user logs on to the computer or when a program encounters an error. Whenever these types of events occur, Windows records the event in an event log that you can read by using Event Viewer. Advanced users might find the details in event logs helpful when troubleshooting problems with Windows and other programs.

You must be logged on as an administrator to perform these steps. If you aren't logged on as an administrator, you can change only settings that apply to your user account, and some event logs might not be accessible.

Event Viewer tracks information in several different logs. Windows Logs include:

  • Application (program) events. Events are classified as error, warning, or information, depending on the severity of the event. An error is a significant problem, such as loss of data. A warning is an event that isn't necessarily significant, but might indicate a possible future problem. An information event describes the successful operation of a program, driver, or service.

  • Security-related events. These events are called audits and are described as successful or failed depending on the event, such as whether a user trying to log on to Windows was successful.

  • Setup events. Computers that are configured as domain controllers will have additional logs displayed here.

  • System events. System events are logged by Windows and Windows system services, and are classified as error, warning, or information.

  • Forwarded events. These events are forwarded to this log by other computers.

    Applications and Services Logs vary. They include separate logs about the programs that run on your computer, as well as more detailed logs that pertain to specific Windows services.
