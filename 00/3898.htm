  1. Open Outlook 2013 and then click to the File tab.
  2. Click Add Account.
  3. Choose Manual setup or additional server types.
  4. Select POP or IMAP.

- Removed the drivers that were installed from d-link cd-rom.
- Downloaded the MS Bluetooth Stack from

- added the following lines to the BTDongle.inf -file:

%D-Link1.DeviceDesc%=BCBTUsbDriverInstallXP, USB\VID_2001&PID_F111

and

D-Link1.DeviceDesc="D-Link DBT-122 USB Bluetooth Adapter"

- rebooted the computer and installed the adapter to the usb port
- when windows started to install the driver I manually selected the BTDongle.inf -file

This got my adapter working with the PC suite.

ken sheridan say: You don't say what type of automation operation you are undertaking with Word, but as an example, the following procedure, into which the full path to the Word template document is passed as the strTemplate argument, is for filling bookmarks in the document with the current record's data:

Sub CreateLetter(strTemplate As String)

    ' Opens a document in Word and inserts values from
    ' current record at bookmarks in Word document.
    ' Accepts: path to Word template file - String
    
    On Error GoTo Err_Handler
    
    Dim objWord As Object
    Dim objDoc As Object
    Dim frm As Form
    Dim strAddress As String
    
    ' return reference to form
    Set frm = Forms!frmAddresses
    
    ' if Word open return reference to it
    ' else establish reference to it
    On Error Resume Next
    Set objWord = GetObject(, "Word.Application")
    If Err.Number = 429 Then
        Set objWord = CreateObject("Word.Application")
    End If
    
    AppActivate "Microsoft Word"
    On Error GoTo Err_Handler
    
    ' open Word document in maximised window
    objWord.Visible = True
    Set objDoc = objWord.Documents.Add(strTemplate)
    objWord.WindowState = wdWindowStateMaximize
    
    ' insert text at bookmarks, getting values from form
    InsertAtBookmarks objWord, objDoc, "FirstName", frm!FirstName
    InsertAtBookmarks objWord, objDoc, "LastName", frm!LastName
    strAddress = (frm!Address + vbNewLine) & (frm!Address2 + vbNewLine) & _
        (frm!City.Column(1) + vbNewLine) & (frm!County + vbNewLine) & frm!PostCode
    InsertAtBookmarks objWord, objDoc, "Address", strAddress
    InsertAtBookmarks objWord, objDoc, "CurrentDate", Format(VBA.Date(), "d mmmm yyyy")
    InsertAtBookmarks objWord, objDoc, "ToName", frm!FirstName

    Set objDoc = Nothing
    Set objWord = Nothing
    
Exit_Here:
    On Error GoTo 0
    Exit Sub
    
Err_Handler:
    MsgBox Err.Description & " (" & Err.Number & ")"
    Resume Exit_Here

End Sub

The procedure calls the following procedure for each bookmark to be filled:

Private Sub InsertAtBookmarks(objW As Object, _
                        objD As Object, _
                        strBookmark As String, _
                        varText As Variant)
                        
    ' select bookmark
    objD.Bookmarks(strBookmark).Select
    ' insert text at bookmark
    objW.Selection.Text = Nz(varText, "")
    
End Sub

montana_x say:

Does PowerPoint 2010 allow Grayscale printing of Charts?

Has the Grayscale printing feature for Charts been removed and, is there a good workaround? ---I need to print slides that have a color background and white data labels on top the bars/pies. When print slides, the labels are not readable, because the grayscale setting are no where to be found.
mel turco say:

Try as you may, the grayscale control over certain items has been removed – intentionally or not. It may have to do with those objects being different from previous versions. I think it has to do with the technical type of container that encases the object, which prevents such interaction. It's even ignores a VBA direct command using the "BlackWhiteMode" property.

This is not only true for charts, but other objects as well such as tables, Excel worksheets, org charts, smartart, etc. You might be able to reformat objects, but the grayscale settings we knew and loved are gone for those objects.

Also, for charts, be careful about embellishing them with lines and shapes. Although lines and shapes can be controlled with the grayscale tools, if they are added INSIDE the chart, the tool no longer will control them. Add them OUTSIDE the chart and lay them over it for better control.

-Melina
