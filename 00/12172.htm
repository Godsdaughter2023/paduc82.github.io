Not sure about the solution but I was thinking .... (that could be dangerous)

Pocket Outlook suppports a set of COM objects, known as the Pocket Outlook Object Model (POOM).  It provides an application with access to the data stored in the Pocket Outlook applications. In addition, the POOM is used to add new contacts, schedule appointments,etc.  So if there is an issue with the POOM, I'm wondering if there is an issue with the data on your PIM.VOL file (such as a missing flag or pointer).

I certainly would try Prof Julie's and Linley's proposed solutions first but if that did not work I might try a few things.

First, back up your PIM data.  I would use two ways of backing up (MyPhone andPIM BACKUP).  Both are free, work well and will save you if there is an issue later.

I'm thinking that there is an issue with the PIM so I would now rename it to PIMOLD, soft reset the device and a fresh new PIM is created with no data. Now restore the PIM BACKUP file to the device.  Verify that your data is there and then try to sync to see if it works (I would definitely delete the old partnership on the device and PC.  

Keep in mind that it may all get messed up so playing around with it potentially can cause an issue.  However, that is why you have MyPhone and PIM BACKUP to save the day if there is an issue.

Of course if you have been syncing regularly and if your Outlook on your PC is reasonably up to date, when you create the new PIM, you could just delete the existing partnerships on your device and PC then sync with your PC and the data would be brought over to your device.  If it syncs, then the issue is resolved.

vanderghast say:

There are many possible solutions, but remember that the concatenated result must do less than 256 characters since 255 is the max, for a single field.

 

The solution that I propose is to first create a new table, which will hold the result at the end of the process:

 

SELECT DISTINCT field1 AS Item, iif(false, " ", null) AS concat INTO tempTable
FROM originalTableNameHere

 

And then, we have to concatenate the values together with a single update query:

 

UPDATE tempTable INNER JOIN originalTableNameHere ON tempTable.item = originalTableNameHere.field1
SET tempTable.concat = (tempTable.Concat + ", " )  & originalTableNameHere.field2

 

 

As previously said, the result is now in tempTable.

vac47 say:

SOLUTION to the problem - for the most part!

Thanks to those who offered advice.

As a result of a lot of Internet research and tons of trial and error on my part, I have come to the conclusion that in the latest Windows 10 Technical Preview (Build 9860), something has gone awry in the way Windows 10 handles Wi-Fi access during the start-up process for a significant number of users, but not all.

After the update, my computer lost its ability to access my wireless network on start-up. I need to mention here that I am using the Netgear Nighthawk R700 router broadcasting in both 2.4 GHz and 5 GHz. This will become more significant a bit later. Without getting into all of the natty-gritty details, I discovered that during the 9860 upgrade Windows somehow disables DHCP for Wi-Fi. For those who are having this problem, the solution is quite simple; you just need to re-enable this.

  • Locate your wireless adapter and right-click properties.
  • Select “Internet Protocol v4” and select properties again.
  • Make sure that the top button is selected: “Obtain IP address automatically.”

This should correct the problem of your computer not being able to access you wireless network on start-up PROVIDED it is a 2.4 GHz network. If you are trying to access a 5 GHz network, things get a bit more complicated. For some reason, the new build 9860 is not able to automatically recognize a 5GHz network—even after enabling DHCP for W-Fi. The only solution I have found is to manually access the 5 GHz network after you have been able to log on using the 2.4 GHz network. Once you manually enable the 5 GHz network, the 2.4 GHz network is automatically disabled.

Whether this is being done by design or there is a problem with the new build is uncertain. I know this much, given the same exact equipment, I never had this problem with Windows 8, Windows 8.1, or the first build of Windows 10.

I hope this has helped those of you who have lost your wireless access after the update.

VAC
