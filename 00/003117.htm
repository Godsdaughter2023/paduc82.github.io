Copy and Paste Data INCLUDING Hidden Columns - Microsoft Community

n1ckole say:

Copy and Paste Data INCLUDING Hidden Columns

1.    When I copy and paste data in a sheet with hidden columns, it will not paste the hidden columns, it is only pasting the visible columns. I have SEVERAL pre-made templates and it is extremely inefficient to unhide the number of necessary columns each time I need to copy and paste.

bernie deitrick say:

If you are copying entire columns, then the hidden column is copied - but the copy of the column remains hidden.

If you copy just a portion of the columns, then when the paste is performed, the cells from hidden columns are not hidden because 'hidden' is not a property of the cells but of entire columns.

Of course, filtered values are treated differently - though they look hidden, they are not: they are not visible, which is different.

hansv mvp say:

You could unprotect the sheet, then protect it again:

 

Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Cells.Count > 1 Then Exit Sub
    If Not Intersect(Target, Range("A5:A5000")) Is Nothing Then
        Me.Unprotect
        With Target(1, 2)
            .Value = Now
            .EntireColumn.AutoFit
        End With
        Me.Protect
    End If
End Sub
 
If the sheet is protected with a password, provide it both for the Unprotect line and the Protect line:
 
        Me.Unprotect Password:="secret"
        ...
        Me.Protect Password:="secret"

chip pearson say:

You can protect the worksheet via code with the UserInterfaceOnly flag set to true. In this case, protection applies to user input (keyboard and mouse) but allows VBA to do whatever it wants, regardless of the protection that might be in place.  There is no button or option to set this property -- it can be done only via code. Moreover, it does not survive a workbook being closed and then reopened, so you need to set it when the workbook is opened.  In the ThisWorkbook code module, use

 

Private Sub Workbook_Open()
    Me.Worksheets("Sheet1").Protect UserInterfaceOnly:=True
End Sub

This will protect Sheet1 such that users cannot change protected cells, but VBA can. Obviously, change the "Sheet1" reference to the actual sheet name.
