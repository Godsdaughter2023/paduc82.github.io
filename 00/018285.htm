For example:

- if the value in A1 is less or equal to 5 (=<5) - SUM a specific set of cells in column B .

- if the value in A1 is greater than 5 (>5) but less than or equal to 10 (=<10) - SUM a different set of cells in column B

- if the value in A1 is greater than 5 (>10) but less than or equal to 15 (=<15) - SUM a still different set of cells in  column B

=IF(A1<=5, SUM(B1:B100), IF(A1<=10, SUM(B101:B200), SUM(B201:B300)))

The logic "greater then 5 but less than or equal to 10" could be written AND(5<A1,A1<=10).

But the nature of the IF() expression makes that  unnecessary.  If A1<=5 is false, we know that A1>5 is true.  There is no need to reiterate that condition in the next nested IF() expression.

Note that I do not cover the condition A1<=15 explicitly.  I assume you mean to sum the third set of cells in column B (e.g. B201:B300) if A1>10 is true.

Otherwise, you need to add a condition for A1>15.  It might look something like this:

=IF(A1<=5, SUM(B1:B100), IF(A1<=10, SUM(B101:B200),

    IF(A1<=15, SUM(B201:B300), "ERROR!")))

PS....  You do not explain if and how the set of cells in column B might relate to the limiting values in A1.  In my example, we could avoid the nested IF() expressions because there is a linear relationship.  If you are more specific about the "sets of cells", we might offer better alternatives.

¡firedog say:

When a message bounces because it's undeliverable, the server doing the bouncing will include a Non-Delivery Report explaining why the message couldn't be delivered. The .eml file you received as an attachment to the bounce message will be the message that couldn't be delivered. You either need a mail client program that can read .eml files, or else you can simply open the file with Notepad. It might not make much sense, though. At least you'll be able to ascertain whether it is indeed a message that you sent.

Read the NDR to find out why your messages couldn't be delivered; this will guide you towards knowing what to do about it.
