joeu2004 say: Bernie wrote:
=MEDIAN(IF('RAW DATA'!A:A=B3,IF('RAW DATA'!X:X="Y",'RAW DATA'!H:H)))
In 2010, the only reason to not use full columns is if you really want to exclude data

And to significantly reduce Excel memory usage and (re)calculation time.

This is evident by watching peak memory usage in Task Manager and by measuring calculation time.  See details below.

@PAL, you are wise to use restricted ranges.  Following Bernie's paradigm, you might write (array-entered; press ctrl+shift+Enter instead of just Enter):

=MEDIAN(IF('RAW DATA'!$A$1:$A$13195=B3, IF('RAW DATA'!$X$1:$X$13195="Y"),
  'RAW DATA'!$H$1:$H$13195)))

Details....

Starting in Excel 2007, array-entered formulas with whole-column references like 'RAW DATA'!A:A usually result in creating and processing internal arrays with 1M elements.

There are some exceptions (e.g, SUMIF and SUMIFS, IIRC).  But this MEDIAN(IF(...)) paradigm is not one of them.

For an extreme example, compare the following two array-entered formulas in Sheet2 of new workbooks in separate instances of Excel.  They have identical functionality when Sheet1!:B1:C3 contain 1 and Sheet1!D1:D3 contain 2, 3, 4.

  1. =MEDIAN(IF(Sheet1!$B$1:$B$3=1,IF(Sheet1!$C$1:$C$3=1,Sheet1!$D$1:$D$3)))
    Peak memory:   about 61,000 Kbytes
    Execution time:  about 100 microseconds [1] (that's 0.0001 seconds)

  2. =MEDIAN(IF(Sheet1!$B:$B=1,IF(Sheet1!$C:$C=1,Sheet1!$D:$D)))
    Peak memory:   about 331,000 Kbytes
    Execution time:  about 438,500 microseconds (that's 0.438 seconds)

Those were the numbers on my (ancient) single-core single-processor computer using XL2010.  Your numbers might vary.  But the relative numbers should be comparable.


For example, on a 4-core dual-processor computer using XL2007 (multithreading enabled), the respective numbers were:  (1) 58,000 Kbytes [2] and 37 microseconds [1]; and (2) 363,000 Kbytes and 311,100 microseconds.


Of course, the difference (esp. in execution time) might not be significant if you have just one such formula.


But the use of whole-column references in a large number of formulas can result in very long (re)calculation times, resulting in a "Not Responding" Excel window.



-----

[1] 85% to 90% of the restricted-reference execution time is due to overhead in the Range.Calculate method.  The overhead is insignificant for the whole-reference execution time.  So the comparable times on my computer are closer to 15 v. 438,400 microseconds (generously rounding down the latter).


Also note that the execution times summarized above are the best of 10 samples in each case.  Execution time can vary greatly for a number of reasons.  Chief among them are:  (1) VBA overhead the first time code is executed after editing; (2) unrelated interrupt processing; and (3) variation in O/S process management and interprocess communication.


[2] Re: 61,000 v. 58,000 Kbytes using XL2007 on one computer (Win 7) and XL2010 on another computer (Win XP).  I would not try to read too much into such small differences.  I see similar variation even with XL2010 on the same computer.

dr. strangelove say:

If Security Essentials installed, you have the correct version.

What makes you believe you have the incorrect version?  Is there a green icon in the system tray?  If so, MSE is running.

Did you remove all existing antimalware before installing MSE?  If not, do so, and restart your PC.

What OS and service pack are you running?  If XP, check that Windows Defender is removed.  If Vista/W7, make sure it is disabled.

If you really did download the incorrect version, follow this procedure to re-download the incorrect version so you can uninstall it:

You should be able to uninstall from the “Add/Remove Programs”, or “Programs and Features”, depending on your OS.  If that fails, you can use this procedure provided by forum contributor Aiscer.

1.     Download the MSE installer file from the website and save the file on the Desktop.  This will allow you to find it easily after the download is done.

2.     Now press the Windows key (or Start key) and R both at the same time. This will open the RUN dialog box.

3.     Next, delete/clear any entries inside RUN, then drag the MSE installer file into the RUN dialog box.

4.     Depending on your Operating System, after dragging the file into RUN, it should appear as one of the three below:

·         C:\Documents and Settings\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-xp.exe

·         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-vista-win7.exe

·         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-amd64fre-en-us-vista-win7.exe

5.     Now put a SPACE at the end of the line, followed by a  /U  as seen below:

·         C:\Documents and Settings\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-xp.exe /U

·         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-x86fre-en-us-vista-win7.exe /U

·         C:\Users\[YOUR USERNAME]\Desktop\mssefullinstall-amd64fre-en-us-vista-win7.exe /U

6.     Click OK.

7.     Restart your system after the uninstall.

cortig say:
If your accounts are IMAP or Exchange, you should be able to use the little triangle on the left of the Inbox on the left pane to unfold it and reveal the individual inboxes.
POP accounts are different. By nature, all e-mails from POP are downloaded on your computer and all end up in the On My Computer inbox.
You can either create Rules to send the messages to folders of your choice but even better than that, you could create smart folders (saved searches) for e-mails coming from specific accounts and use them as the invidual inboxes (I use smart folders like that, adding the Unread criteria to only see new e-mails for instance).
