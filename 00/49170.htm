Windows UAC compliance - Microsoft Community

jpeters91 say:

Windows UAC compliance

I use an application that is not UAC compliant. Aside from the challenges that come with not being UAC compliant, is there a chance that Microsoft will say in the future that non-UAC complaint applications won't be allowed to run on future versions of Windows? 

Also, how does this affect applications running on the surface or mobile windows?

cmdrkeene say:

Microsoft has never said they intend to restrict non-UAC programs from running, and I don't think they ever will.  UAC is just a security technology so that programs don't get full privileges they don't need. That is, they don't run "as admin", even if the user themselves is an admin. Most programs work fine this way, because the program doesn't really need access to make low-level kernel modifications anyway.

They can't outright block you from running programs without UAC, because as long as your an administrator, you can do it.

Surface works just like a real PC, so there's no change there.

Mobile Windows can't run arbitrary x86 code anyway, so that's essentially moot.  A mobile app is always running restricted in a sandbox.

Hi,

Quite late, but useful for many people.

You have to say to Windows 10 that Firefox can handle "mailto" protocol :

"Win + R", lauch "regedit" and go to "HKEY_LOCAL_MACHINE\SOFTWARE\Clients\StartMenuInternet\FIREFOX.EXE\Capabilities\URLAssociations". Then add a string value named "mailto" with "FirefoxURL" as value.
