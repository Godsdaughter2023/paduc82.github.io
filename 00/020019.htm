willy denoyette say:

Don't do this, reducing all working sets trashes the Page file without any reason, you still have free memory right?.

When doing this all pages from the working sets are all written to disk. These pages have to be reloaded from the paging file whenever you re-activate a program, so you incur an IO penalty without gaining anything.

The system memory manager knows exactly what to do when there is memory contention and will trim the not recently touched pages of working sets of dormant processes. He knows exactly which pages should be cached, you don't.

jay freedman say:

In general, posts about Word 2013 are also applicable to Word 2016.

The first thing to look for is a file in the same folder with the name "Backup of <document>.wbk". That will be the file as it existed just before the most recent Save. The file will exist only if, before the Save in question, you had gone to File > Options > Advanced, scrolled to the section labeled Save, and checked the box for "Always create backup copy". If that box isn't checked now, you should check it to prepare for future disasters.

If it's possible that you made a change in the document and then closed it without saving, anytime the last 4 days, then click File > Open and click the "Recover Unsaved Documents" button at the bottom of the list of recent documents. This will show you all the unsaved documents, and you can pick one.

The best method, although sadly not used by most people, is to have set up Windows 10's File History feature before you needed it. If it was running when you overwrote the document, the previous version(s) will be recoverable from the history.
