Solution for Internet failure post Win 10 1511 build update. - Microsoft Community

tavi_dog_walker say:

Solution for Internet failure post Win 10 1511 build update.

Having had 3 very different hardware computers (2 wireless and 1 wired), that were working prior to the 1511 release in November, I have been investigating why all three computers have had intermittent/failing network connections since the 1511 upgrade and subsequent updates have been installed. Investigations focussed on what was common between the machines as all three were basically working except for the internet connection. Having first confirmer that I was using latest Network Adaptor drivers I proceeded to eliminate any Router caused problems by confirming its continuing functionality with non-Windows 10 machines (Windows 7 and I-pad etc.).  I then rebuilt one of the machines to remove all third party software (not even any 3rd party AV/Firewall) and, just to be sure, this was done on a formatted HDD.  I then proceeded to try all any of the many offerings in forums to investigate what, if any, impact they had.

I’m pleased to say I believe that, after a lot of trials and investigation, I have found a solution that works on all my machines plus two friends machines. I have now had internet connection running reliably for over 24hrs so far. A massive jump from the typical 5 minutes I have been seeing after resetting winsock, ipconfig /flushdns etc.

The solution appears to be adding specified DNS servers to the IP4 xxxxxxx instead of letting Windows 10 obtain DNS server automatically. An easy way to do this is

a)      Open Networking and Sharing Centre (right click on connection icon in the tray and select it)

b)      Click on Change adaptor settings.

c)       Open the properties of the connected Network adaptor (right mouse on it and click Properties)

d)      Select Internet Protocol Version 4 (TCP/IPv4)

e)      Select Properties

f)       Select Use the selected DNS Server addresses

g)      Add an appropriate DNS server address in the Preferred and Alternative boxes. You can use DNS server details from your router or use the easy to remember Google DNS addresses (8.8.8.8 and 8.8.4.4) or any other appropriate ones.

h)      OK your way out.

i)        Before re-booting I recommend applying the following from command Prompt (Admin). [Right mouse on windows button(bottom Left) and select]. Then enter;

Netsh winsock reset.  Wait till it finishes, ignore comment about re-starting and apply the following before restarting the computer.

Ipconfig /flushdns

After reboot, hopefully you will find your reliable network connection has been restored.

This is very much a work around for a problem that appears to be inherent within Windows 10 1511 to date.
