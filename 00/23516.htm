vanderghast say:

Ok. Have the table Obfuscations  with the two fields  Origine and Obfus. The first one is the primary key and the second one has an index not allowing duplicated value (the same obfuscation token should not be used for two different original name).

 

Have the following functions, in a standard module:

 

Public Function NewObfus() As String
'generate a new name
' 6 letters,  2 digits
NewObfus = RndLetter() & RndLetter() & _
        RndLetter() & RndLetter() & _
        RndLetter() & RndLetter() & _
       Int(10 * Rnd()) & Int(10 * Rnd())
End Function

Public Function RndLetter() As String
    RndLetter = Chr(97 + Int(26 * Rnd()))
End Function

Public Function MyObfuscation(Origine As String) As String
Dim obfus As Variant '  need a variant to handle possible null
   
    ' Look into the table to see if we can find the name
    obfus = DLookup("Obfus", "Obfuscations", "Origine=""" & Origine & """")
    ' if so, we get the obfuscated token, otherwise, it is null
    If IsNull(obfus) Then
        ' the name has not been obfuscated yet
        On Error GoTo tryAgain
tryAgain:
        ' find a new name
        obfus = NewObfus
        ' insert it in the table, so we will get it back if it comes again
        CurrentDb.Execute "INSERT INTO Obfuscations(Origine, Obfus) VALUES(""" & Origine & """, """ & obfus & """) ", dbFailOnError
        On Error Resume Next
    End If
   
    MyFofuscation = obfus
End Function

 

The CurentDb.Execute line may be split in two. It should be only one.

And use MyObfuscation(name) where you need an obfuscated token for the given name. If the name has already been obfuscated by a given token, that token will be returned, otherwise, a new token is generated.

Note:  The special error handling (on error goto tryAgain) and the code it covers should have been placed into a subroutine (an a counter added to avoid infinite loop) so it would have not disturbed your own error handling. It is there to handle the case where NewObfus would return an existing token, and the INSERT wlll then fail due to the duplicated obfus value. That would have make the illustration more complex, and I wished to illustrate the concept of obfuscation as simpy as possible, not of proper errror handling.

karl timmermans [outlook mvp 2012-2018] say:

To take your questions one piece at a time

1) <Delete Folder> - only active for folders that are not designated as a <Default> for the current profile or specific email account when there are more then email accounts at play. <Delete Folder> does exactly as advertised, deletes the entire folder which is moved to the <Deleted Items> folder. Folders that are (or have ever been) designated as a default folder cannot be deleted via the Outlook UI. That means that if you have deleted folders on one or more occasions, those would be ones that you have created.

2) <Delete All> - deletes all the items in the folder that has been right-clicked and the option is available. Those items are moved to the Deleted Items folder but the folder itself is not deleted. Ergo <Delete Folder> is completely different from <Delete All>

3) <Empty Folder> - option only available in the Deleted Items folder. Once the folder is emptied - the items are gone. There is no <Delete All> option for the <Deleted Items> folder.

For point of reference - the <Empty Folder> and <Delete All> options do not exist together for the same folder.

Private Function EnumAll(ByVal hWnd As Long, ByVal lParam As Long) As Boolean
  Dim ClassName$, Title$, i&, e&
  i = GetWindowTextLength(hWnd)
  Title = Space(i)
  GetWindowText hWnd, Title, i + 1
  ClassName = Space(256)
  i = GetClassName(hWnd, ClassName, 256)
  ClassName = Left$(ClassName, i)
  Debug.Print hWnd, ClassName, Title
  EnumAll = True
End Function

Sub SetSubjectWithAPI()
  Const ID_Subject = &H1005
  Dim hWndMail&, hWndFrame&, hWndEdit&, hWndSubject&
  hWndMail = FindWindowEx(0, 0, "rctrl_renwnd32", vbNullString)
  'EnumChildWindows hWndMail, AddressOf EnumAll, 0
  hWndFrame = FindWindowEx(hWndMail, 0, "AfxWnd", vbNullString)
  hWndEdit = FindWindowEx(hWndFrame, 0, "#32770", vbNullString)
  hWndSubject = GetDlgItem(hWndEdit, ID_Subject)
  SendMessage hWndSubject, EM_REPLACESEL, -1, ByVal "Subject"
End Sub

If you don't use filtering elsewhere on the sheet and you don't have Hide in any other ranges in BX you could just do

Range("BX1:BX551").AutoFilter 1, "<>Hide"

You will need BX1 to be non-blank for this to work.

  Dim C As Range

  ActiveSheet.Protect DrawingObjects:=False, Contents:=True, Scenarios:=False, AllowFormattingRows:=True, Password:="incomplete", userInterfaceOnly:=True

    For Each C In Range(Cells(408,76),Cells(414,76))
      C.EntireRow.Hidden = C.Value="Hide"
    Next
  End If
  ' etc

That code could be made more maintainable by naming the Hide ranges, e.g. BX25:BX31 as CGLPolicyHide and then
  For Each C In Range("CGPolicyHide")
will continue working if you insert or delete rows from the sheet in future.

Your formula uses the COUNTIF function which is appropriate when examining a range, not a single cell

Let write it as

=(E3>=80)+(F3<=60)+(Y3>133)

This will give a result of 0, 1 , 2 or 3 depending on how many conditions are met.

You wish to flag those with 2 or more conditions are meet

=IF((E3>=80)+(F3<=60)+(Y3>133)>=2,"Risk","")

michael herzfeld say:

In order for "Do Not Check Spelling" to work, you have to make a selection first. The easiest way to turn off spell checking for a specific document is:

  1. Press CTRL-A to select everything
  2. Click the Spelling button on the Home tab of the Ribbon and select Set Proofing Language.
  3. Check the Do not check spelling box, then click OK.

Unfortunately, this has to be done for each document. There isn't a way to turn it off globally.

Hi RABELL0811,

Microsoft Streets & Trips has been discontinued. Fortunately, you can use Bing maps and upload your map there by following these steps:

  1. Go to http://www.bing.com/maps
  2. Click on the "My Places" button in the upper left. Sign in if you haven't already.
  3. From the editor window that pops up on the right side, choose Import. Browse to your GPX file and choose which collection to load it into.
