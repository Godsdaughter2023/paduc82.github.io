j.e. mcgimpsey say: First, there's no such thing as a "VB scrollbar" - I expect you mean an ActiveX scrollbar (or a scrollbar from the Controls Toolbox in WinXL). ActiveX is a Win-only technology, and will not work with MacXL.

To work cross-platform you need a scroll bar from the Forms controls (or "Legacy controls" in later versions of WinXL). In MacXL, you can insert one by clicking on the Scroll Bar button on the Developer tab of the Ribbon and drawing it on your worksheet.

When you create one on your sheet, its default name will be "Scroll Bar 1", though you can change it in the Name box on the Formula Bar. In the worksheet code module for the worksheet it is placed on, you can refer to it as Me.Scrollbars(1) or Me.Scrollbars("Scroll Bar 1"). From other modules, refer to it as a child of the worksheet, e.g., Sheet1.ScrollBars(1) or Worksheet("Sheet 1").ScrollBars("Scroll Bar 1").

Second, your code makes no sense for either platform - you should use "With...End With" to refer to an Object on which you can perform multiple operations using the .Property syntax. That object could be Scrollbar1 (for ActiveX control) or Me.ScrollBars(1) (for Forms control), but not Me.ScrollBars(1).Max, since Max is a property, not an object. So you could write

   With Me.ScrollBars("Scroll Bar 1")
      .Max = Me.Range("max_h").Value
   End With

but since you're only referencing only one property of the ScrollBar, you could also write it more simply

    Me.ScrollBars("Scroll Bar 1").Max = Me.Range("max_h").Value

Third, since you need the scrollbar to work with negative numbers, you'll need to do some manipulation, since Forms scrollbars only work with non-negative integers.

One way would be to set the ScrollBar cell link (right-click the scroll bar and choose Format Control, then choose the Control tab in the dialog) to an out-of-the-way cell (perhaps in a hidden column/row), then set an offset in the cell you want displayed. For instance, to have cell A1 show -50 to +50 based on the scroll bar setting, set the scroll bar Min/Max to 0/100, set the cell link to, say, cell Z1, then in A1 enter
      =Z1-50

Fourth - I'm not sure why you're using the _SelectionChange event at all, since the value of Range("max_h") presumably won't change when the Selection changes. Unless  another event is changing Range("max_h")'s value, I'd think using the _Change event would make more sense, for instance:

    Private Sub Worksheet_Change(ByVal Target As Range)
        If Not Intersect(Target, Me.Range("max_h")) Is Nothing Then
            Me.ScrollBars("Scroll Bar 1").Max = Me.Range("max_h").Value
        End If
    End Sub

or, more efficiently, using With...End With

    Private Sub Worksheet_Change(ByVal Target As Range)
        With Me
            If Not Intersect(Target, .Range("max_h")) Is Nothing Then _
                .ScrollBars("Scroll Bar 1").Max = .Range("max_h").Value
        End With
    End Sub

FWIW Nicola saw the predecessor to the following two XSL stylesheets.

These stylesheets try to switch around the lastname and initials to be "Lastname I. J." format rather than "I.J. Lastname" format, for all "SourceTypes".

There are two versions: one where each initial is followed by a ".", e.g. "Lastname I. J.",at https://goo.gl/MNTYXu and one where there is no dot, e.g. "Lastname I J" at https://goo.gl/r4TaJT

A problem with the "Dot" format is that in some cases the bibliography entry can end in two dots. A fix for that would take a lot longer and/or might benefit from a complete re-think of the stylesheet coding approach.

rich007a say:

In the bottom left corner of Excel, where the arrows are that you use to scroll along the tabs, right-click and a handy menu appears which lists all the tabs.

Remember you can use Ctrl+PageUp and Ctrl+PageDonwn to step to the next left of right sheet.

To select the previous sheet, I think you'd need a macro:

With your workbook open, right-click on one of your tabs > View Code.

When the VBE opens, do Insert > Module

Paste the following:

Public strLastSht As String, strCurrentSht As String

Sub SelectPrevSheet()
' Shortcut key Ctrl+q
'
    Dim ws As Worksheet
   
    On Error Resume Next
    Set ws = ActiveWorkbook.Sheets(strLastSht)
    On Error GoTo 0
   
    If Not ws Is Nothing Then ws.Select
End Sub

_______________________________


Then, on the left, double-click on the ThisWorkbook module and paste the following:


Private Sub Workbook_Open()
    strCurrentSht = ActiveSheet.Name
End Sub

Private Sub Workbook_SheetActivate(ByVal Sh As Object)
    strLastSht = strCurrentSht
    strCurrentSht = Sh.Name
End Sub

_________________________________

Close the VBE and go back to Excel.

Do View tab > Macros

Select the procedure called SelectPrevSheet and click Options

Enter a q (or whatever other unused shortcut key you like)

Save your workbook as Macro Enabled *.xlsm

Now you can jump back to the previously active sheet by pressing Ctrl+q

Hope that helps.

Cheers
Rich
