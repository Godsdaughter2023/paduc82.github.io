This can happen when the new master or layout has more graphic elements than the original; PowerPoint has no way of knowing whether the new or old layout element count should be preserved, so it creates a new layout with the same name + _number.

If you tab through the layout before and after applying the new design, you'll notice that there are different numbers of shapes:  5 on Main Event, 9 on the original Berlin.

If you like, you can assign the Title and Content layout to the new slides, then delete any name + _numbered layouts.  In this case, applying the "original" layout doesn't seem to affect anything.

john korchok say:

Steve is right, plus:

Besides needing a matching name, a slide and its companion layout must have the same number and same type of placeholders as the layout it is to use. This includes the Date, Footer and Slide Number placeholders. It also must be marked in XML as the same type of layout.

In the XML, the layout must also have the same layout type (obj for a Title and Content layout). Unfortunately this latter property is not visible in the program interface. An XML type mismatch only occurs if the original layout is deleted and then a user or designer tries to re-create it. If you are confident this has never happened, you can rule it out.

steve rindsberg say:

Had to have a bit of a rummage in the Large Repository of Seldom-Accessed Old Code to find this one, but if you've got a whole lot of slides/presentations that you want to rid of these 1_Spurious layouts, have a go with this VBA (ON A COPY of your presentation, please; NEVER the original):


Sub RenameReplacementLayouts()
' When you add layouts to a presentation, PPT won't replace existing ones
' that have the same name, it renames the incoming new ones with 1_ + old name
' This routine:
'   - Reassigns all slides to the new 1_ version of the original layout, if it exists
'   - Deletes the original layouts (if no slides are left assigned to them), and then
'   - Renames the new 1_ versions back to the original names
'
' Result:
' If you had a "Two Content" layout originally, then brought in a new one,
' you'd now also have a "1_Two Content" layout.  Existing slides would still have the old
' "Two Content" layout.
'
' This macro assigns all the "Two Content" slides to the "1_Two Content" layout
' Then deletes the original "Two Content" layout
' Then renames "1_Two Content" to "Two Content"
' Leaving all your "Two Content" slides as "Two Content" slides, but with the new layout.
' Whew.

Dim oSl As Slide
Dim oDes As Design
Dim lLayoutNumber As Long
Dim lDesignNumber As Long

' first, assign each slide layout to its 1_ counterpart
For Each oSl In ActivePresentation.Slides
    With oSl
        Set oDes = oSl.Design
        ' does the 1_ version exist?
        lLayoutNumber = CustomLayoutNumberFromName("1_" & .CustomLayout.Name, oDes)
        If lLayoutNumber > 0 Then
            .CustomLayout = oDes.SlideMaster.CustomLayouts(lLayoutNumber)
        End If
    End With
Next

' delete the non- 1_ layouts; this will error if we try to delete
' a layout that has a slide assigned to it (ie, one without a corresponding
' 1_ version; ignore the error, don't delete it.

For lDesignNumber = 1 To ActivePresentation.Designs.Count
    With ActivePresentation.Designs(lDesignNumber).SlideMaster.CustomLayouts
        For lLayoutNumber = .Count To 1 Step -1
            Debug.Print .Item(lLayoutNumber).Name
            If Left$(.Item(lLayoutNumber).Name, 2) = "1_" Then
                ' leave it alone
            Else
                On Error Resume Next
                .Item(lLayoutNumber).Delete
                On Error GoTo 0
            End If
        Next    ' Layout
    End With    ' Custom Layouts
Next    ' Design

' then rename the layouts to remove the 1_
For lDesignNumber = 1 To ActivePresentation.Designs.Count
    With ActivePresentation.Designs(lDesignNumber).SlideMaster.CustomLayouts
        For lLayoutNumber = 1 To .Count
            ' only attempt to rename 1_ versions
            If Left$(.Item(lLayoutNumber).Name, 2) = "1_" Then
                ' does the non 1_ version still exist?  If so, we can't rename this one to match it.
                If CustomLayoutNumberFromName(Mid$(.Item(lLayoutNumber).Name, 3), ActivePresentation.Designs(lDesignNumber)) = 0 Then
                    .Item(lLayoutNumber).Name = Mid$(.Item(lLayoutNumber).Name, 3)
                End If
            End If
        Next    ' Layout
    End With    ' Custom Layouts
Next    ' Design

End Sub

Function CustomLayoutNumberFromName(sName As String, oDes As Object) As Long
' Returns the index of the layout named [sName]

Dim x As Long

For x = 1 To oDes.SlideMaster.CustomLayouts.Count
    Debug.Print oDes.SlideMaster.CustomLayouts(x).Name

    If oDes.SlideMaster.CustomLayouts(x).Name = sName Then
        CustomLayoutNumberFromName = x
        Exit Function
    End If
Next

' sName not found in Layouts collection, return 0
CustomLayoutNumberFromName = 0

End Function

If you will be adding many commands to the Quick Access Toolbar, you may well want to choose the "Show Below the Ribbon" choice. But when you click the arrow that displays the menu that shows that option, you should also see several options, including Quick Print, which is the equivalent of the Print button on the Standard toolbar in Word 2003 and earlier (it prints the entire document to the currently active printer with default settings). Click on this to select it (turn on the check), and it will be added.

You can access the Print tab of the Backstage by pressing Ctrl+P, but if you want to add a button to do this, you'll need to select More Commands... from the menu. Under "Choose commands from," select File Tab. There are two options here: Preview and Print and Print Preview and Print. The latter actually opens the Print tab of the Backstage. The former gives you a dropdown menu from which you can choose Print Preview and Print (the same as Ctrl+P), Quick Print, and Print Preview Edit Mode.
