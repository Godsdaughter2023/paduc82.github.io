m4e_team_k say: "error status received from server (12)"

The problem will be seen if you try a manual or interval (every 1 hour, etc) sync *after* you change the folders under 'Inbox' in Outlook. This affects users connected to an Exchange 2007 server.

The problem does not occur if Outlook folders are changed during 'Always on'.

Also, if the problem is encountered, MfE will 'fix' itself during the next 'Always on' period.

So in most cases Manual sync users are more likely to notice this.

You can fix this immediately if you perform a Options--Full_resync.

If you prefer not to perform a Full_resync because of data costs, put MfE into 'Always on', let it sync, and then put it back to Manual.

  3. Open the downloaded file and extract the files in it to the desktop;
  4. Open "configuration-Office365-x64.xml" with Notepad > delete the original contents > paste the following script into it > Save:

<Configuration>

    <Product ID="O365ProPlusRetail">

      <Language ID="en-us" />

    </Product>

  </Add>

</Configuration>

5. Type CMD in the search box of Windows > right click on Command Prompt > Run as administrator;

6. Check the pathname of your desktop. You may check it by right click on a file or folder in your desktop > Properties > check in “Location”. For example, my desktop pathname is C:\Users\Admin\Desktop;

7. Type the following command in the Command Prompt:

cd pathname

(In my case, it’s “cd C:\Users\Admin\Desktop”)

8. Type the following commands in the Command Prompt:

setup.exe /download configuration-Office365-x64.xml

(This command will make the Office Deployment Tool download the Office installation files to your computer, so it will take a while.)

setup.exe /configure configuration-Office365-x64.xml
