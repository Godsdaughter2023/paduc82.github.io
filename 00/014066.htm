john - project say:

ckandreou,

Let's start with the simple stuff. To get the cost to show as currency you need to add a Format statement. So, in your case it might be:

Text1="NRC Cost is : " & format([Cost1],"$##.00") & " EYRC : " & format([Cost2],"$##.00")

If you have the Cost1 and Cost2 fields set to rollup to summary level as a sum, then also set the summary line to use the formula.

Now the not so simple. It may be possible to create a group of custom fields to give time based cost data (I doubt it), but even if it were possible, the formulas would be unwieldy. I don't quite understand what you mean by "...show on the Gantt Chart, on top, the cost...". What is "on top" mean?

You can do the third part with VBA, depending on what "cost" you are referring to. If it's one of the normal cost fields (e.g. Cost, Actual Cost, etc.), then VBA can separate out the timescaled data and display that in extra fields (e.g. Cost1,Cost2, etc.). However, if the cost you want to parse into date ranges are a custom cost field (e.g. Cost1) that you have created (or entered), then how it was created will determine if it can be broken out by date periods.

Hope this helps.

John
