Adapted from Chip Pearson

If all you want is a count of the unique numbers in the Customer #
column, then, if all the entries are NUMBERS with no TEXT, you can
use:
 
=SUM(N(FREQUENCY(rng,rng)>0))
 
where rng is something like C2:C10000 (large enough to allow for
expansion).
 
If some are all of the customer #'s are TEXT, then use:
 
 
This formula must be **array-entered**:
 
=SUM(IF(FREQUENCY(IF(LEN(rng)>0,MATCH(rng,rng,0),""),
IF(LEN(rng)>0,MATCH(rng,rng,0),""))>0,1))
 
Ensure that rng does NOT include the Header cell, or subtract one from
the total.
----------------------------------------
 
To **array-enter** a formula, after entering
the formula into the cell or formula bar, hold down
<ctrl><shift> while hitting <enter>.  If you did this
correctly, Excel will place braces {...} around the formula.

jacques raubenheimer say:

Looks to me like it is the "spacing after" setting for the paragraph containing your code snippet.

What is probably happening is that the normal style has the "don't add space between paragraphs of the same style" setting on, so in the rest of your text, this space is not being added, but when you put in a caption and the style changes, then the space appears. That also explains why, when you delete the line, the caption changes, because it now becomes part of that normal style paragraph.

Solution:

Change the "spacing after" setting for the normal style.

Some additional recommendations:

1. Don't use Normal style. Rather use Body Text.

2. Use Body Text for your actual text, and create a new style (call it "Code") for your code sections.

3. Turn on your non-printing characters so that you can see the paragraph end marks (little pilcrow symbols at the end of each paragraph, and so, so many other little document bits that you actually need to see) which will help you see what is going on there.

Ah, see (excuse the pun), being able to see non-printing characters does help.
