I need it to return the count of unique values, in this case 3.

ronrosenfeld say: Here's a rather complicated array formula that would do that.  NAME your range of dates:  Dates; and your range of Names:  Names  eg.  Dates refers to:D1:d8 and Names refers to J1:j8.

Then use this formula:

This formula must be **array-entered**:

=SUM(IF(FREQUENCY(IF(LEN(IF(YEAR(Dates)>=2010,Names,""))>0,
MATCH(IF(YEAR(Dates)>=2010,Names,""),IF(YEAR(Dates)>=2010,
Names,""),0),""),IF(LEN(IF(YEAR(Dates)>=2010,Names,""))>0,
MATCH(IF(YEAR(Dates)>=2010,Names,""),IF(YEAR(Dates)>=2010,
Names,""),0),""))>0,1))

----------------------------------------

To **array-enter** a formula, after entering
the formula into the cell or formula bar, hold down
<ctrl><shift> while hitting <enter>.  If you did this
correctly, Excel will place braces {...} around the formula.

In the formula above, you could substitute a cell reference for the year, and, assuming you put your year in A1, change that part of the formula to :

YEAR(Dates)>=$A$1

How can I resize images from VBA in Access 2010

I know this is a long shot, but is there a way to resize a picture through VBA?  Windows Live Photo Gallery in Win7 has an excellent utility for resizing a group of pictures in batch to various sizes, and it would be great if there were a way to tap into that, assuming Access can't do the job itself, which I pretty sure it can't.

Using Access 2010, I have a database which manages packaging procedures for packaging product for shipment.  We have several hundred part numbers, and each part number has it's own procedure.  Each procedure needs to have 1 to 4 pictures on it, all of which will print on a single page, along with the procedure steps, so the pictures themselves will not be very big; maybe 4" x 3" a piece.

I have browsed through the forums regarding the handling of pictures and arrived at my current solution.  The user uses a front end packaged solution using the Access 2010 runtime only.  The back end (Access 2010 .accdb) and all photographs are stored on the file server.   The procedures table stores the path to the pictures, and when a record is opened to edit the procedure, the paths for the 4 pictures are loaded into the .picture property of the four image controls on the form, and the pictures are displayed.

I'll post the abbreviated code for the first picture below for posterity's sake.  [Pic1] is the field in the record source that has the path, and [Picture1] is the image control on the form.  Everything below works fine.

Private Sub Form_Current()
  
  Dim stPic                 As Variant
  
  stPic = [Pic1]
  If Not IsNull([Pic1]) Then
    [Picture1].Picture = [Pic1]
    [Picture1].ControlTipText = [Pic1]
    SysCmd acSysCmdSetStatus, "Image: '" & [Pic1] & "'."
  Else
    [Picture1].Picture = ""
    [Picture1].ControlTipText = ""
    SysCmd acSysCmdClearStatus
  End If
  
Exit Sub

Now that I've explained all that, here's the problem.  We already have hundreds of parts in production, so we've got someone running around taking pictures every time a part gets to packaging so that we have our photos of what it's supposed to look like.  The girl who's taking the pictures every time is not super tech savvy, and I don't want to rely on her to reduce the settings for the picture size on the camera before the picture is taken because so many people use the camera for tracking returned product and other things.  I just spent hours reorganizing the photos she had taken, and figuring out how to resize them from 2.5 MB a piece down to < 200k at 640x480 (THANK YOU Windows Live Photo Gallery!).  It would be fabulous if I could let her import all of the photos to one folder, then when she selects the photo from Access via the Application.FileDialog (below), it would resize the photo before moving it to a permanent (and organized) location on the server.  Everything works fine now, but I ant to resize the photos because together they are over 10MB, and trying to browse through records, or print them off on a network printer over the wireless connection, is slow and tedious.

Any ideas?

Thanks,

cevinmoses say:
I'll offer this up, again for posterity's sake.  Code for selecting a photo by double-clicking the image control and thus storing the path in the table is below.  Works really well.

(Like the code above, it has been adapted from several other people's code, which I wish I had done a better job of documenting for their sake, but I take no credit for the magic involved.)

Private Sub Picture1_DblClick(Cancel As Integer)
  On Error GoTo Err_Picture1_DblClick
    
   Dim fDialog As Object
   Dim varFile As Variant

   Set fDialog = Application.FileDialog(3)
   
   With fDialog

      .AllowMultiSelect = False ' Don't allow user to make multiple selections in dialog box
      .Title = "Please select a picture" ' Set the title of the dialog box.
      .InitialFileName = "\\Internetstorage\Access\PackagingPics\" & Me.CustShortName & "\" & Me.PartNo
      .Filters.Clear ' Clear out the current filters, and add our own.
      .Filters.Add "Image files", "*.bmp;*.gif;*.jpg;*.wmf"
      .Filters.Add "All Files", "*.*"

      ' Show the dialog box. If the .Show method returns True, the user picked a file
      ' If the .Show method returns False, the user clicked Cancel.

      If .Show = True Then
        'Originally coded as a loop through multiple files, but there should only be 1 since .AllowMultiSelect is set to False above
'Loop through each file selected and save the path in the [Pic1] field of the table, then set the .Picture property of the image control to that path.
         For Each varFile In .SelectedItems
            [Pic1] = varFile
            [Picture1].Picture = [Pic1]
            SysCmd acSysCmdSetStatus, "Image: '" & [Pic1] & "'."
         Next

      Else
         MsgBox "You clicked Cancel in the file dialog box."
      End If
   End With
Exit Sub

Err_Picture1_DblClick:
  MsgBox Err.Description, vbExclamation

End Sub
