Summary

Diagnose and Fix a Problematic Office Installation

If your Office application suite is giving you intermittent problems or not starting at all, the OffCAT diagnostic tool is the first place to start. While OffCAT is not a 'Fix-It-All' tool, it can identify problem areas from a frequently updated central storage of known problems and suggest steps to take where you arrive at a solution. The programs from the Office suite that are addressed are:

  • Access
  • Excel
  • InfoPath
  • OneNote
  • Outlook
  • PowerPoint
  • Publisher
  • Word

If is not the purpose of this article to simply reproduce the information already available through OffCAT's own screens or its accompanying DOCX ReadMe document. Rather, I will try to provide some Quick Start steps to guide you to some immediate results and suggest some direction for what you can do with those results. In the event of any confusion between this article and the official OffCAT documentation, the official documentation should be assumed as the authority. The most current documentation is available from the Microsoft Downloads' OffCAT page as a Word document (ReadMe_OffCATv1.2.docx).


Details

How do I run it?

OffCAT quickly identifies your Office application suite (or suites if multiple versions reside on the computer) and examines many facets of your computer's individual configuration against a set of known issues.

While there are some requirements for installing and running OffCAT, they are minimal and any system with one of the Office versions that OffCAT is designed to analyse *should* already meet those requirements. For the vast majority of users, this section can be skipped and you can get on with installing and running OffCAT. In the off-chance that your system will not install and run OffCAT, these are the minimum requirements and some links to help you get your system up to speed.

Quick Start

Go to the Microsoft Download Center's page for the Office Configuration Analyzer Tool (click here).

Choose the download you want. The only one you need to installâ´ the program to your computer is the OffCAT.msi.


Check the box beside OffCAT.msi and then click the blue Next button in the lower right corner of the screen. The download will start immediately and you will be paged through to a Thank you for downloading screen where you can review the installation instructions or close your browser.

Locate your downloaded OffCAT.msi file. This is typically in your Downloads folder but some browsers will differ. Double-click the file to begin installation.

â´If you do not have the permissions necessary to install a program on your computer or wish to run OffCAT from a portable media device like a thumbdrive, choose the OffCAT.zip archive file instead.   • OffCATv12.adm - a group policy file for system and domain administrators that govern the permissions of more than one user on a single computer or network.

Running OffCAT for the first time

When you initially open OffCAT, it will update its database of rules. It is important that you allow this to occur as even the documentation will not keep up-to-date with new rules being added all the time. When this is complete, you can get started.

  1. Read the general instructions on the opening pane. Note that there is a check box in the lower left to stop this information pane from displaying in the future.
  2. From the choices available down the left side, choose a Start a Scan. The right pane will give you the options to select a single Office application to scan the configuration for or select All (or selected) Office applications (the following screen will allow you to choose some of all of the programs available).
  3. If you have multiple versions (e.g. 2003/2007/2010/2013/365) of Office on your computer, this next screen will also allow you to choose which version to scan. While OffCAT can scan multiple Office programs, it can only scan one version at a time).
  4. If you selected All (or selected) Office applications, this screen also allows you to pick one or more of the available Office programs to scan the configuration of. This list will change depending upon the availability of Office applications by version (see step 3 above).
  5. You can also provide a scan label for the file name of your scan. If you do not supply one, one will be generated using the program(s) selected together with the date and time.
  6. If you want to review and/or change the options you selected on the previous screen, click  at the bottom of the page.
  7. If you are satisfied with your choices, click  near the bottom of the page.
  8. Once you've initiated a scan, the screen will change to a Scanning in Progress pane with a time estimate and visual gauge showing the process advancing. If you have to halt the current scan for any reason, click  and wait for the cycle to be interrupted.

Word 2010 - Autocomplete header based on title - Microsoft Community

tomas perez ponisio say:

Word 2010 - Autocomplete header based on title

Hi there,

I have this document in which I want to insert the document Title in the header, and if in case of updating the title, that the header auto updates. Does this make sens?

Any ideas?

Thanks!

suzanne s. barnhill say:

You can insert a TITLE field in the header, but this won't help unless the actual title of the document has been added to the Title property. And the field will update when other fields in the header are updated but not necessarily immediately. A better approach is to use the Title style for the document title and add a STYLEREF field in the header. This will update automatically and is the most efficient solution.

I'm assuming by "Title" you do mean a title that is in the document or the Title property, as opposed to the filename (for that you'd use a FILENAME field).
