Removing the battery only performs a soft reset (not a full reset).  When you are finished using an application and close it by tapping the X at the top of the screen, most of the time the application is not closed at all...it is minimized and keeps running in the background, taking up valuable processing memory. john mcghie say:
Hi Sally:

This is a well-known problem with Mac Word 2011.

"Insufficient disk space" is nonsense, it's simply Microsoft's best guess as to why the write to disk would be refused (which it has been).  In fact, it's a file server locking issue: the server thinks more than one user is trying to open the document.  (The "server" in this case, is part of the OS on your computer.)

The cause appears to be the number of files Word is trying to hold open.  Word has about 20 or 30 different files open while you are editing a document.  If you save the document three times to three different locations, that's 60 to 90 files it holds open.  If you never restart the computer, those files can stay open, until eventually the computer can't open any more and you get this error.

One of the first things to do when you get a "can't save" message is to re-start the computer, to clear all the dead file locks that have collected.

However, the other thing that will give you a "Can't save" error is Tracked Changes.  Word goes to save the document, and notices a few seconds later that nothing got written to disk.  The reason is that while you are working, Word loads the file in its internal format for speed.  When you save, Word first converts  that internal format to .docx, then tries to convert the .docx to .doc if you are working in the obsolete format.  A .docx can contain a heap of things that a .doc simply cannot store.  And Tracked Changes can make the internals of the document so complicated that Word can't read it.  Either way, one of the conversions fails, so the save fails, so you get the message.

Make sure you are up at Update 14.1.4 (look in Word>About Word for the version number).  Microsoft has put out several fixes to try to improve this behaviour.

You have NOT set up an "auto-save"; Word doesn't have one.  You have set up a save of "AutoRecovery Data" every five minutes.  This feature is not a lot of use: it will restore the document only if Word knows that it has crashed (not hung...) and only if it can read the original document.  It's not saving the document, it's saving a list of changes to the document, so if it can't read the original, you're hosed.  And if you quit Word, it will wipe out the autorecovery store.

A better mechanism is Word>Preferences>Save "Always make backup".  That will work only if you MANUALLY save.  But it actually saves the previous version of the document, in the same folder, labelled "Backup of..." every time you save.

If you save every time you pause to think, you will never loose more than a few minutes worth.  You can simply open "Backup of..." it's a complete document, not a list of changes.  Save it as a new file name, and off you go again.

Compare Document will give you character-for-character markup of the changes you have made: the only thing you lose is the exact time that each edit was made: not a big loss.  But your documents are much less likely to fail if you turn off Track Changes.

Sadly, there's not a lot that Technical Help would be able to do for you.  The cause is bugs in the Word program, which are being triggered by the way you work.  Microsoft is still working on these issues: but they are very complex, and they may never be able to fix them all in the current version of Word.  Sorry: but if I told you anything else, I would be misleading you.

We can, of course, hope that the next version of Word will have replaced the file writing mechanism so it doesn't do this.  We have been hoping this for three versions now...

Hope this helps

bob jones aka: cybertaz say:

There's certainly no need for a template to do this. In fact, a template is an awkward way to go about it unless you want to be able to generate new documents from the template with the watermark already in place. Additionally, I'm having no problem here recording a macro triggered by a keyboard shortcut to insert either a built-in or custom Text Watermark. It also makes no sense to me that such a macro would create a new document.

Try following these steps to see if you have success;

  1. Create a new blank document
  2. Go to Tools> Macro - Record New Macro or click the Record button on the Developer tab
  3. Name the macro using no punctuation or spaces
  4. Click the Keyboard button
  5. Press the key combination you want to use, making sure that [unassigned] appears beneath the editable field
  6. Click the Assign button
  7. Confirm that Normal.dotm is indicated in the Save changes in: list
  8. Click OK
  9. Go to Insert> Watermark... or use the Watermark button on the Layout tab
  10. Click the Text option
  11. Select a built-in watermark or type one you prefer
  12. Set the formatting options
  13. Click OK
  14. Go to Tools> Macro - Stop Recording or click the Stop button on the Developer tab

rohn007 say:
Update Offers Updates for Programs you do not have installed

http://support.microsoft.com/kb/830335

Windows Update often offers updates for programs that you think you do not have installed. The link above gives 3 reasons why this happens and some possible solutions. Another reason is that the updates are often not only for the single product in the description.

 

Updates are often bundled for more than just the app identified in the label. More than one program or bundle may be using the same component.

 

For me, the short answer is, just let Update "do its thing” because it has better insight into the deep details of your system than you do. The only control I sometimes exert is limiting the number of update I let it apply at one time. For example, when installing Windows or an application Update may find very many updates so I limit it to 5-10 at a time.
