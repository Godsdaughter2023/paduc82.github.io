Exchange Basics

The Inbox syncs more frequently than any other and should be kept as clean as possible. (note PCs seem to handle large Inboxes where Mac does not.)

Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.

ken sheridan say: You don't actually need to create and save an 'append' query for this; you can execute the SQL statement in code in the Click event procedure of the button, e.g.

Dim strSQL As String

strSQL = "INSERT INTO YourLocalTable " & _
    "SELECT * "
     "FROM YourQuery " & _
    "WHERE CustomerNumber = " & Me.CustomerNumber

CurrentDb.Execute strSQL, dbFailOnError

This assumes that CutomerNumber is a number data type.  If it's a text data type amend the code with:

    "WHERE CustomerNumber = """ & Me.CustomerNumber & """"

This inserts a row from the query whose CustomerNumber matches that of the form's current record.  You should ensure that (a) the local table's columns have exactly the same names as those returned by the query, and (b) the local table has one or more appropriate columns defined as the table's primary key, or otherwise uniquely indexed, to ensure that the same row isn't inserted more than once.  From your description it sound like the CustomerNumber column is a candidate key.

ian ser say:

Hi kjb90038,

This error usually appears when the program you're trying to install still has remnants of its previous installation found on your device. I recommend the following steps to completely remove the program and after that, retry the installation:

  1. Press Windows key on your keyboard, type Control Panel and click the search result.
  2. Under Programs click on the item Uninstall a program.
  3. Windows then lists all programs that were installed using Windows Installer. Select the program you want to uninstall by clicking on it. 
  4. Click at the top on Uninstall/Change.
  5. In the next message box, confirm the uninstall process by clicking on Yes.
  6. Windows now starts the uninstall process. After completion you'll receive a status message, which you confirm by selecting OK.

If it was not found on the program list, try to get in touch with the developer of the app. They may have a complete uninstaller for the app available on their website.
