macropod say:

In this simple case:

{IF{=SUM(ABOVE)}> 9 "Yes" "No"}

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues.

shadoshryke say:

Bonnie is up the right track, but if you are working in Windows 7 Pro, I found the file not in the location she listed, but in ("show hidden files/folders" must be active):

 

C:\Users\(User Name)\AppData\Local\Microsoft\Windows\1033\

 

Note that this same issue can cause your Windows Search to fail or just do nothing.   Once I found and deleted this file, both my Windows Search (from the Windows Explorer Search) and the MS Outlook Search worked flawlessly.


What I sleuthed out is that when the computer in question had a shutdown (power interruption) while a search was open, the .bin file because marked as corrupted and a second one was created as StructuredQuerySchema(1).bin .  One showed as encrypted and compressed and one showed normal.   For some reason, neither was acceptable to Windows Search (which is the component that allows both Windows Explorer and Outlook to search) and so it would cause Outlook 2010 to crash and restart, and Windows Explorer search to do nothing.
