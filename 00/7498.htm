Embedded font Use - Microsoft Community

nicole-2140 say:

Embedded font Use

Once a font is used and embedded into a PowerPoint presentation or template, then circulated to the team, can team members edit and use the ppt as is, or do they need to download the font onto their computers as well? Thanks

steve rindsberg say:

As long as the font is successfully embedded (not all fonts or font types CAN be embedded), the other team members won't have to do anything but open the PowerPoint file.  No need to download the font ... embedding is meant to eliminate the need to do that.

General info that might be useful:

Some fonts are "edit-embeddable", meaning that the font can be used to edit the document it's embedded into, but not with other documents.

Other fonts (not nearly so many, in my experience) are "install-embeddable", meaning that when the user opens the document containing the font, the font is installed into Windows and then becomes available to any Windows program, even after the original document is closed.

Countifs with booleans - Microsoft Community

**** nasty say:

Countifs with booleans

I have a worksheet with 3 columns -- column “A” contains “Yes” or “No”, column “B” contains a name, and column “C” contains a name.  I want to count the number of records where column “A”=”Yes” and column “B” OR column “C” is blank/empty.  I’ve played with countifs and can get the count when both are empty using “”, but struggling with using the “OR” boolean…  (Win XP, Office 2007)

 (Win XP, Office 2007)

Thanks in advance…

hansv mvp say:

If you want to include rows where both column B and column C are blank in the count:

 

=SUMPRODUCT((A2:A100="Yes")*(1-(B2:B100<>"")*(C2:C100<>"")))

 

If you only want to count rows where either column B or column C is blank but not both:

 

 =SUMPRODUCT((A2:A100="Yes")*((B2:B100="")+(C2:C100="")=1))

 

Adjust the ranges as needed.

    renegade10005 say: I got this answer from our IT staff whic is exactly how it happened on my laptop:

    If you import calendar items into Outlook that are duplicates of items that already exist in your Calendar folder, and you select the Allow duplicates to be created option in the Import and Export Wizard, you can delete the duplicate calendar items after the import process is completed. Removing the duplicate calendar items is a manual process, but the following is the easiest way to do it:

    1. In Calendar, select the calendar folder where there are duplicate calendar items.
    2. On the View menu, point to Arrange By, point to Current View, and then click By Category.
    3. Click the Subject column heading to arrange all calendar items by subject.
    4. Hold down CTRL while you click each duplicate calendar item in the list.
    5. When you have selected all the duplicate calendar items, press DELETE.

    Note  If you do not want duplicate calendar items when you import calendar items into Outlook, select the Do not import duplicates option or the Replace duplicates with items imported option in the Import and Export Wizard.

    I understand that you would like transport rules to be directly applied to all concerned addresses that are not in a particular group rather than firstly applied to all recipients and then with some exception.

    I’d like to clarify that currently it’s not feasible to achieve that. Actually, catch-all addresses are not supported in Exchange online/Office 365 for security reasons. Due to recipient filtering in place to protect against potential spam messages, email addresses that do not exist in your Office 365 tenant will be rejected. In this scenario, you may like to let the #4 rule be in the top of the #1 rule.
