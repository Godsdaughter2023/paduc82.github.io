    andreas killer say:
    I also noticed that when I tried to remove the reference to the Microsoft Forms 2.0 Object Library, I would get this error message:
    That's okay, this reference could not be removed once it has been set.

    Make a new file, open the VBA editor, insert a normal module, paste in this sub and execute it:

    Private Sub SetRef()
      On Error Resume Next
      'Microsoft Forms 2.0 Object Library
      Application.VBE.ActiveVBProject.References.AddFromGuid _
        "{0D452EE1-E08F-101A-852E-02608C4D0BB4}", 2, 0
    End Sub

    After that copy all code below and paste it in the same module above the existing sub.
    Write a formula in cell A1 and execute this sub:

    Sub Test()
      SetClip Range("A1").FormulaLocal
      Range("A2") = "'" & GetClip
    End Sub

    Andreas.


    Option Explicit

    Private MyData As DataObject

    Function GetClip() As String
      If MyData Is Nothing Then Set MyData = New DataObject
      MyData.GetFromClipboard
      GetClip = MyData.GetText
    End Function

    Sub SetClip(ByVal S As String)
      If MyData Is Nothing Then Set MyData = New DataObject
      MyData.SetText S
      MyData.PutInClipboard
    End Sub

    a. user say:

    OK, first I re-checked to ensure it was working here, which it is. 

    You do have to put a real email address where I have put <put the email address here>

    The error returned by the MacScript function usually just tells you that an error has occurred, without providing any detail. There could be several types of error. So you will need to dive deeper into Applescript than you probably wanted if you have never used that language/environment.

    If possible, I suggest that you find and open the Applescript editor (in Yosemite, it's not just for AppleScript, it's in the Applications folder, and it is called "Script Editor") and try to run the code in there.

    The Applescript generated by the above is as follows - you will need to substitue the email address and document path/name. Or, you can copy the text generated by the "debug.print s" VBA statement from the VBE's Immediate window into the Applescript Editor.

    tell application "Microsoft Outlook"
       tell drafts
            set theMessage to make new outgoing message with properties {subject:"Yard Observation Form", plain text content:"Please see attached form"}
            make new to recipient at theMessage with properties {email address:{address:"<put the email address here>"}}
            make new attachment at the end of theMessage with properties {file:("<put the full path and file name of the dcoument here>" as alias)}
          send theMessage
        end tell
    end tell

    In the Applescript editor, I would compile that. If there are errors, you will need to fix them (you can post back here as necessary) and eventually copy them back into your VBA. Then try running the routine. If that works, there is probably a security/permissions error somewhere.

    The alternative is to try to work entirely in the VB Editor (I don't recommend it). In that case, the first step is to look at the generated applescript (In the VBE Immediate window) and see if you can spot any obvious errors (two lines joined into one, missing quotation marks, invalid email address etc.)

    Then cut the script down to the bare minimum by commenting out all the "s = s &" lines except the initial "tell application" and final "end tell" lines. Run that. If that fails, I would also guess a permission error of some kind. If the VBA succeeds, nothing will happen (Outlook will not start if it has not already been started).  Move to the next step...

    1. Change the line

    Debug.Print Macscript(s)

    to 

    MsgBox Macscript(s)

    2. Insert the following line after the "tell application" line:

    s = s & "return ""after tell application""" & vbCrLf

    Run the code again.  This time, Outlook should start even if it has not started. The result of the Macscript call should be "after tell application", which should be displayed in a VBA dialog box.

    If all is OK, you can remove the "return" statement and successively add the "tell drafts/endtell", then each of the remaining statements in turn.
