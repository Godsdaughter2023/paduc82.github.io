"Some of our developers had renamed some registry keys in HKCR\Installer\Products\ in order to deactivate the auto-repair functionality of Windows Installer for our product.

This was impeaching the FEP installer to search for competitive products. The installer would then abort with error code 0x800700ea.

So here is what we did:

  1. We backuped the registry key HKCR\Installer\Products by exporting it to a file.
  2. Then we tried to look for the faulty key: We deleted the each subkey of HKCR\Installer\Products one after the other, and tried to install FEP after every deletion.
  3. Once we found out which subkeys were faulty, restored the whole HKCR\Installer\Products key from the exported file, fixed the faulty key and succesfully installed FEP.

Some more comments that might be useful:

  • Other people have tried to rename then whole HKCR\Installer\Products key, but that leads to other problems.
  • Marek Nowicki had exactly this problem because of the HKCR\Installer\Products\9D99FDDE3EF917847ABD1D25C215EEA9 key. This key corresponds to the .NET Compact Framework 2.0 (see here, it's in Polish, so use an online translation tool).

To conclude, it seems that the FEP installer aborts as soon as it finds an invalid/incorrect registry subkey in HKCR\Installer\Products\ when looking for competitive products.

It looks like a bug in the FEP installer to me. I think the FEP installer should not abort in such situations; it might display a warning, or a more explicit error message.

But hopefully, the workaround described above will help."

Windows phone -> MS Store: App language not freely selectable - Microsoft Community

maarten de vries say:

Windows phone -> MS Store: App language not freely selectable when system language not available for app.

This is most likely a bug submission, as I don't expect an immediate solution for this issue. (although would be nice)

Bug description:

Apps in the MS App Store can be published in multiple languages. For example, an app was published with English and Swedish as supported languages. If your phones system language is not in the list of available app languages, the MS Store will only offer the app in one language. This is most likely a hidden (to end-users) default language for that app.

This means that you can only find the app in an probably inconceivable language and not the preferred one.

If you change your phone's system language to one on the apps supported languages list, it immediately offers the app in that language instead. (Instead of the default language I mentioned earlier)

Use case:

https://www.microsoft.com/nl-nl/store/p/the-mindfulness-app/9nblggh0k5h6#

This app has been published to the store in both Swedish and English. Swedish is most likely the default language, as it's a Swedish developer.

Step 1. If I have Dutch as the only system language on my phone:

- App is only available for download in Swedish. (Store page also contains Swedish app description text, and Swedish screenshots)

- I have no way of selecting the English version instead.

Step 2. If I add and change my system language to English:

- After restart, the app on my phone is still in Swedish and I can only start that version.

- I look for the app in the Store. App is now available for download in English. (Store page also contains English app description text, and English screenshots)

- Having bought and installed the app prior to the system language change, the Store immediately starts to download the English version of the app as an update.

- After update completion, I can start and use the English version of the app.

Step 3. If I change back the system language to Dutch (because I'm Dutch and that's still my preferred language for all other purposes) (By moving the Dutch language up in the list of languages and thus making it active)

- After restart, the app is still in English.

- I look for the app in the Store. The store doesn't automatically start updating to install another language version of the app. (Store page also contains English app description text, and English screenshots)

- In all honesty, this is the place I'd like to be without having to do two restarts in the first place.. But, for completeness sake..

Step 4. If I remove the English secondary system language: (I don't use it, why keep it around? It also takes up 10 MB drive space.. That's like 3 MP3's!)

- No restart required. App is still available in English.

- Open the Store again, it notifies me that the store needs to be refreshed due to changes to my Language settings.

- The Store hates me, it updates back again to the Swedish version because the Store likes being a little *****. :D

(Store page also contains Swedish app description text, and Swedish screenshots)

Edit: the store doesn't actually force the Swedish version back again. Probably, both languages have already been installed at this point and now the Store just leaves it up to me whether I want to revert back to Swedish. This seems really inconsistent with Step 2 behavior, where visiting the page will automatically start the update to English. Good stuff!
- App is obviously only available again in Swedish.

Now, I could've skipped that last step to prevent me having to do steps 1 and 2 again, but oh well. At least you've got a full test report this way. (See earlier edit)

This bug has been found on a Lumia 640 XL, Windows 10 fully updated. I spoke about this issue over chat to a support representative in the Netherlands (he will also file an internal report), and the problem has also been submitted earlier for a Windows 950 XL. I think it'd be pretty safe to assume that this is not hardware related but platform (Windows Phone as a whole) related, but I added this information just in case..

Edit 2: Step 1-3 can be abbreviated to just adding a new language to your language settings. Normally, the newly added language will also be the default one after adding, but if you then immediately move up the language that you want, no restart will be required. I don't even think the language pack for the extra added language will even be installed. You only added a language to your list of available system languages without actually installing it. However, it seems that for Store purposes, just the newly added system language being there is enough for the Store to trigger a refresh and offering the app in the preferred language.

This is a mediocre but at least workable workaround for the situation until you can choose in what language you want to download a multi-language app.
