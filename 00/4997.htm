marshall barton say:

I'm not convinced that dot vs bang makes a difference here.  The most common reason for that is the form name, text box name was changed or that the form was closed right after opening the report.

 

Another thing that can cause that is if the form does not have a module.  The Form_ syntax requires that the form must have a module, even if the module is empty.  Technically, you should not be using the Form_ syntax to refer to a form object.  That syntax is intended to represent a class in an object variable's declaration statement and, in some advanced situations, may return an unexpected value.  Instead, you should stick to the standard Forms!formname syntax.

 

Regardless of the problem you asked about, it's a better design practice to avoid having a report (or another form) refer back to the "calling" form to "pull" values.  The  better practice is to use the OpenArgs argument to "push" the values, or, at a minimum, the name of the form to the report.  This way the report can be totally independent of any forms that open the report.

ken sheridan say: I'd endorse what Marshall has said with regard to passing values to a report by means of the OpenArgs mechanism rather than referencing controls in a form.  The OpenArgs property is a string expression, but you can pass multiple values by tokenising the string and parsing it.  Some years ago Stuart McCall published a module which does this, and I subsequently developed it further to allow for numbered or named arguments to be passed, which would probably be the best solution in your case.  You'll find the module and a demonstration of how it works Args.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

The demo passes values to a form, but it can be done to a report in exactly the same way.

One or two points about your existing code:

1.  You are declaring the tmpdate and tmpBoxNo variables as Public in the report's module's declarations area.  As it appears that these are to be used only in the report's module, however, there is no need to declare them as Public.

2.  The  tempdate variable is declared as a date/time data type, but by calling the Format function you are assigning a string expression to it.  Assuming the value is entered in the calDate control in the local regional date format, you can use the CDate function to assign a date/time value to the variable:

tmpdate = CDate(Form_fSendOffSite.calDate)

though if you follow our advice and pass the value to the report by means of the OpenArgs mechanism  you would get the value from the OpenArgs property e.g. using my module:

Dim args As String

args = Me.OpenArgs
tmpdate = CDate(Arg(args, "MyDate"))

where MyDate is the name which has been given to the argument in the code in the calling form's module.

    abi99 (2) say:

    Apps are signed by the author. However, such a Code-Signing certificate is valid only for a certain amount of time (one year, three years, …). Or stated differently: The app is broken; the phone does not miss any certificate authority. With such an ‘expired’ error message, one approach is to change the date back, just change the year and start with 01-01-2016. Then, you try to install that app again. Then, you go for 01-01-2015 and so on. You did that already, just as explanation.

    However, when it comes to Facebook, I cannot recommend to go for the generic variant because if I am not mistaken that is not maintained anymore and even older than the one installed on your phone. Therefore, first try the variant built-into your phone.

    Please, do not apply the tricks of Nokia Series 40 or Symbian/S60 blindly to your Nokia Asha 503. The Asha Software Platform (introduced with the Nokia Asha 501) was written from the ground up and is its very own operating system, with its own bugs, limitations, and issues. Many things work like on previous phones, because the very same programmers created this Asha Software Platform. However, when it comes to apps, make sure to go for an app tested on Nokia Asha Platform, like through the Opera Mobile Store. Java on this operating system differs a lot (!) from previous Nokia Series 40 and apps might not behave as expected if not tested with the Asha Software Platform

    When it comes to the list of certificates authorities, these steps you found, do not help, because those certificates apply only to E-mail and VoIP. The already built-in list should be sufficient for E-mail. Opera Mini and UC Browser access the Internet via a proxy server and by this circumvent the certificate-authority list on your phone. Consequently, you are not about Internet certificates but Code-Signing certificates.

    Could not connect to server: certificate was issued by an unrecognized entity.

Depending on where this happens, this could indicate an error in the file system itself and require a recover via the Recovery Tool or Care Suite. Furthermore, the list at traud.de/ta is quite complete. It would be interesting which certificate authority creates this error message. Therefore, which server/service/app did you use?     Workbook.Saves FileFormat Enumeration (file type code) - Microsoft Community

    rjbayha say:

    Workbook.Saves FileFormat Enumeration (file type code)

    The programs I have developed work in Windows Excel 2000 thru 2010, and Mac Excel 2004. I am updating the programs to work in Mac Excel 2011. My projects allow the user to export worksheets to another workbook file having no VBA code included.

    The file format enumeration is -4143 for saving workbooks as XLS using Excel version 2000 thru 2004 (windows and mac).
    The file format enumeration is 56 for saving workbooks as XLS (pre 2007 Excel version) for using Excel versions 2007 and 2010.
    The file format enumeration is 57 for saving workbooks as XLS (pre 2007 Excel version) for using Excel version 2011.

    The following is a sample of code that works in all Excel versions 2000 thru 2011.

    Option Explicit
    Private mNewBook As Workbook, mFileName As String, mFileType As Integer


    Sub File_Type()
        Select Case True
        Case Val(Application.Version) < 12
        mFileType = -4143
        Case Left(Application.OperatingSystem, 3) <> "Mac" And _
            Val(Application.Version) >= 12
        mFileType = 56
        Case Left(Application.OperatingSystem, 3) = "Mac" And _
            Val(Application.Version) >= 12
        mFileType = 57
        End Select
    End Sub

    Sub Export_Instructions()
    ... a lot of code before

    Call File_Type
    mNewBook.SaveAs Filename:=mFileName, FileFormat:=mFileType, _
            CreateBackup:=False, AddToMRU:=True

    ... a lot of code after
    End Sub
