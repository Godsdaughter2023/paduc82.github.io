tom ogilvy say:

 

I am guessing for each row that you loop through in column C, you are only interested in the value in column F (variable d) for that same row.  If that is correct, then this should do that.

 

 

Sub stuff()
Dim ws As Worksheet
Application.ScreenUpdating = False
Application.Calculation = xlCalculationManual
ActiveSheet.Range(Cells(2, 2), Cells(65536, 2).End(xlUp)).Select
Selection.Copy
Range("C2").Select
ActiveSheet.Paste
Set ws = ActiveWorkbook.Sheets("CMRData")
Dim c As Range
Dim d As Range
For Each c In ActiveSheet.Range(Cells(2, 3), Cells(65536, 3).End(xlUp))
  Set d = c.Offset(0, 3) '<=  use this instead
  If Not IsNumeric(d.Value) Then
     If Len(c.Value) = 6 Then
        c.Value = Left(c.Value, 5)
     End If
  ElseIf Not IsNumeric(d.Value) Then
     If Len(c.Value) = 5 Then
        c.Value = Left(c.Value, 5)
     End If
  Else
     c.Value = ""
  End If
Next c
Application.ScreenUpdating = True
Application.Calculation = xlCalculationAutomatic
End Sub

damian shields say:

Hi,

Can I suggest a slight variation on this, which makes it more flexible and usable:

When unlocking a section, allow any other sections with the same password to be unlocked (and searched) automatically.

The reason I suggest this is because locking/unlocking entire notebooks would require you to reorganise your notebooks according to access rights/groups, so you have one notebook per group, per project, per customer, etc.

By using automatic password unlocking, you can maintain your existing notebook structure the way you're using it, but control access simply by using different strong passwords for different sections and only giving that password to certain groups.

Examples for using this would be:

  • To share a single customer-related section with a colleague, without having to break it out from the other 50 customers in my "Customers" notebook.
  • Not have to worry about managing rights if I move/copy a section from one place to another within the same notebook
  • Not have to go through the process of:  creating a new notebook, setting the password for that notebook; and setting the access privileges for that notebook, individually - and THEN have to manually add that notebook for synchronisation to all my other synced devices!

Of course, if password locking isn't secure enough, I can always move the section to another notebook, and only share that notebook with specified users at the OneDrive level.

Don't get me wrong - password-protecting a notebook is still better than the per-section process at the moment, because it at least gives me the ability to encrypt everything as zero-knowledge, so that it's safe from data leakage or government access, and access it more easily than having to type a password for every single section individually.

The idea of caching a password is a little strange, but if the password is kept locally to the device, then I would imagine it's no different to manually typing it in - just a lot faster. There are plenty of examples of other security products that use this method of automatically trying cached credentials against anything a user tries to unlock: Truecrypt caches keys and passwords if you try opening more volumes, and Putty with Pageant also stores a set of keys and tests them all when you log into a new system.

Is SafeSear.ch a Threat to Windows 10? - Microsoft Community

michaeljcarroll say:

Is SafeSear.ch a Threat to Windows 10?

The situation is simple.  A family member introduces a browser hijacker like safesear.ch.

In the past, cleanup was pretty easy.  The first step almost always involved downloading and installing adwcleaner, a free adware cleaner that is the major step in removing something like safesear.ch.

With all other applications shut down, adwcleaner scours all the usual places for signatures, marks them, and -- with your permission -- removes them.  To do this, since safesear.ch and its brethren lives (like a virus) hidden within all apparently healthy apps, adwcleaner has to shutdown a number of applications.

But now, with Win 10, this is a red flag -- nothing is going to be allowed to shut down any background applications, even infected ones or malware itself.  The solution is the familiar BSOD (blue screen of death).  Win 10 simply won't let you remove the infection.

Alright, then, will Win 10 take over and do the job itself?

Nope, it can't.  While programs like Malwarebytes will find and remove the overt components of safesear.ch, the hidden components of safesear.ch simply (like DNA hiding within a germ cell) self-replicate with the next launch; and other programs, such as Windows Defender, simply go "malware? what malware?" and tootle on clueless to any corruption under the slick surface.

So Win 10 insistence that "I can take care of myself" actually provides a rich landscape for an entire breed of malware.

What I find sad -- and of great concern -- is how easily it was done.  Here we have the Goliath of Win 10, and there this insipid little piece of Malware, inadvertently clicked on by an unsophisticated child, and the whole structure is threatened.  We are less than three weeks from official launch and it seems Win 10 is designed to prevent vital support just in those very cases where we most would need it.

Now, I can hear a chorus saying "we'll do it," as MS services steps up to the bat.  "But since you brought it on yourself, you are going to have to pay for the treatment."

And I will say, "but I have always dealt with these issues myself."

To which MS will say, "and it's people like you who cause these problems, tinkering around where you don't belong.  Let MS fix MS products."

Okay, before I get too carried away, someone step in and convince me otherwise.

You can achieve this with array formula formulas based upon SMALL() and LARGE().

                      

The array formula in C2 is,
=IF(LEN(C1),IFERROR(LARGE(IF(LEN($A$2:$A$999),IF($A$2:$A$999<IF(ISNUMBER(C1),C1,1E+100),$A$2:$A$999)),1),""),"")
... which requires Ctrl+Shift+Enter↵ and the array formula in D2 is,
=IF(LEN(D1),IFERROR(SMALL(IF(LEN($A$2:$A$999),IF($A$2:$A$999>IF(ISNUMBER(D1),D1,-1E+100),$A$2:$A$999)),1),""),"")
... which requires Ctrl+Shift+Enter↵. Once entered correctly, these can be filled down as necessary.

Note that these formulas need to have one row above them, preferably non-numeric and this row is referenced with the formula. The formulas essentially provide a unique list of sorted data, one ascending and one descending. If you want the first three highest or lowest values, fill down the formulas for three rows. I've provided a couple of error controls so they will not return #N/A and they will transition zero values.
