ken sheridan say: You might (and I hope you are) be using the phrase 'lookup field' in a more generic sense than that of the rightly deprecated lookup field wizard.  If you are simply referring to a foreign key column which references the primary key of another table, then that's fine as it's a fundamental mechanism of a relational database.  It's the wizard which should be avoided like the plague.

It's your reference to a 'dual' lookup field which makes me think that you might not be referring to the wizard , but just what you mean by 'dual' is open to a number of interpretations:

1.  You could be referring to a table with a composite foreign key made up of two columns, which is referenced by a two-column foreign key in the referencing table.  In a case like this you cannot use the NotInList event procedure of a combo box in a form based on the referencing table to insert a row into the referenced table as you would in the case of a single foreign key column.  It is possible to do so, however, by means of code in the form's BeforeUpdate event procedure.  You'll find an example in Prescriptions.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the BeforeUpdate event of the fsubDrugStrengths subform is as below.  this inserts a row into the referenced Strengths table if new Strength and Unit values (in combination) for a drug have been inserted into the subform.

Private Sub Form_BeforeUpdate(Cancel As Integer)

    On Error GoTo Err_Handler

    Dim strMessage As String
    Dim strSQL As String
    Dim strCriteria As String
    
    strMessage = "Add new strength: " & Me.cboStrength & _
        " " & Me.Unit
    
    strCriteria = "Strength = " & Me.cboStrength & _
        " And Unit = """ & Me.Unit & """"
    
    strSQL = "INSERT INTO Strengths(Strength,Unit) " & _
        "VALUES(" & Me.cboStrength & ",""" & Me.Unit & """)"
        
    ' if currently selected/entered strength/unit not in Strengths table
    ' after user confirmation insert new row into table and requery comboboxes
    If IsNull(DLookup("Strength", "Strengths", strCriteria)) Then
        If MsgBox(strMessage, vbYesNo + vbQuestion, "Confirm") = vbYes Then
            CurrentDb.Execute strSQL
            Me.cboStrength.Requery
        Else
            Cancel = True
        End If
    End If

Exit_Here:
    Exit Sub

Err_Handler:
    MsgBox Err.Description & " (" & Err.Number & ") ", vbExclamation, "Error"
    Resume Exit_Here
    
End Sub

2.  Another interpretation of 'dual' could be that you have a single foreign key column in the referencing table, but are showing two or more values from separate columns in the referencing table, either in separate controls in a form based on the referencing table or as a single concatenated value in a single control.  The foreign key could be one of these values, or it could be a hidden numeric key referencing a 'surrogate' autonumber primary key of the referenced table.  In this case the NotInList event of the combo box bound to the foreign key can be used, usually by opening a form bound to the referenced table in dialogue mode and passing the new value to it, or by passing the concatenated value and parsing it into two or more values when passed to the dialogue form.  You'll find examples of both in the NotInList demo in my same SkyDrive folder.

karl donaubauer say:

Lou79 wrote:

is run off of this to identify a report which matches the criteria
chosen, the button I have opens the query in a separate window. I

Form called frmSearch, query called qryResults.

There are several ways to achieve this.

You could use qryResults as record source of the subform
and in your button click procedure update the subform with:

Me!MySubForm.Requery

Another method would be to assign the source in the button
click procedure:

Me!MySubForm.Form.RecordSource = "qryResults"

If you don't need "qryResults" as a saved query, you could
also use the SQL string for both methods.
