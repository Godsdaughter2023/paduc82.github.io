a. user say:

Hello SB,

Thank you for posting.

As per my understanding you want the list of inactive groups in your yammer network.

So by exporting the data by navigating to yammer home network as a verified admin rights > Network admin > Export data.

In the export data .zip folder you will find a file groups.csv with difeerent columns

By this each column you can differentiate the group as below: -

  1. ID: -  you will find one more file in export data .zip folder as messages.csv.
    1. By the group id you can search in the message.csv that which messages are posted.
  2. Name: - This column will indicate the name of the group
  3. Description: - if there is any group description mentioned than it will maintain in this column.
  4. Private: - If the group is private than the value of private column will be true.
  5. created_by_id: - this column contains the group creator ID which can be found in users.csv file.
  6. created_at: - In this column date of group creation will be mentioned.
  7. Deleted: - if the group is deleted than value of this column will be TRUE, if the group is not deleted than value will be FALSE.
  8. External: - if the group is external group than value will be TRUE for internal groups it will be TRUE

So with the group ID which would be having less number of messages can be considered as inactive group.

Please make a note if the data retention policy is kept as Hard delete than Deleted data is permanently removed and not visible in data exports.

That can be checked by login in yammer network as verified admin.

Navigating to network Admin > Data Retention > Hard delete or soft_delete.

You can refer below links for your reference purpose: -

https://support.office.com/en-us/article/Monitoring-your-Yammer-data-Yammer-admin-guide-8c4651fa-12c2-4ced-b4ea-2200c0a630ed

https://techcommunity.microsoft.com/t5/Yammer/How-to-Export-the-Members-of-a-Group-into-a-Spreadsheet/td-p/9418
