cmdrkeene say:

You can't import anything into the Mail app.  In fact, the Mail app doesn't hold or store any data at all, it's just a "viewer" that lets you see what's in your online email account.  Sure it temporarily caches emails so you can read them offline/disconnected, but it's just a cache (like a browser uses) and temporary.  All your emails stay and will always stay on the online email server.

One way you can accomplish something similar is to use Outlook 2016 (the full program that you buy with MS Office) to add those emails to your online email account, so that they show up inside the Mail app.

There is no back up of the data in the Mail app, because it's all backed up online.  If you drop your computer and it breaks, you can just get a new computer and login to the Mail app again, and all your data will show up the same, no backup required.

boffies say:

storage pools in Windows 8

Hi,

I have a home server and have just purchased 2 * 6Tb hard drives.  I already have a 2Tb and a 3 Tb.  So I have 4 hard drives that I want to add into the pool. I will install Windows on another drive as this cannot be in the pool.  I originally intended to do a RAID-5 configuration, partitioning the 2 six Tb hard drives into 2 partitions each and creating one RAID drive using 3 disks (6,6 and 3) with 3Tb and another with the other 2 partitions on the six Tb hard drives plus the 2 Tb disk.  I have now found out about the storage pool feature in Windows 8 and would like to use that instead.

I have the following questions before I start:

  1. If I place a disk into the pool, can I partition the disk? In other words, can I create 2 virtual drives with the 4 disks in the pool, both using parity.
  2. Can I place disks into the pool if they already have data on them?
  3. If I ever need to reinstall windows in the future, will the new installation recognise the storage pool and the virtual drives in it, or will it be complex to get access to the data in the storage pool?

Thank you very much.

Andreas

robert mitchell [msft] say:

I have the following questions before I start:

  1. If I place a disk into the pool, can I partition the disk? In other words, can I create 2 virtual drives with the 4 disks in the pool, both using parity.
  2. Can I place disks into the pool if they already have data on them?
  3. If I ever need to reinstall windows in the future, will the new installation recognise the storage pool and the virtual drives in it, or will it be complex to get access to the data in the storage pool?

Thank you very much.

Andreas

1. Yes.  When you put disks into a pool, you are just grouping them together.  Then when you carve out a virtual disk (aka space), you will see it show up in disk management as a normal disk.  So you will be able to partition it, format it...use it like a normal disk.

2. No.  If you are going to be using disks in storage spaces, they should start out empty.

3. Yes..sort of.  When you install the fresh copy of Windows, it will read the metadata for the disks in the pool and recognize that they came from a different instance of Windows.  Simple right click on the pool and enable 'Read Write Access'.  Then the pool will belong to the new installation of Windows.  Much like with dynamic disks where you had to import the disks.
