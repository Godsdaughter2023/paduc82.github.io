jeeped say:
I accidently dropped my laptop and now all my programs only open with microsoft word. What should I do to get my programs back the way they were?

I've never heard of this happening when you drop a laptop, but from what you've stated above, I believe the file association for LNK files has been misdirected to MS Word. I've seen this happen when upon opening a program through a shortcut (e.g. .LNK file), you are asked what program to open it with. After selecting MS Word (or some other program) via the Select a program from a list of installed programs, and checking the Always use the selected program to open this type of file (circled in red below), the document or program referenced by the original shortcut opens in MS Word as it should.

The trouble is that now all shortcuts (e.g. .LNK files) on the desktop or throughout Explorer will want to be opened by MS Word and use the Word icon to display the shortcut regardless of what the shortcut was originally intended to open. This is known as a file association and default icon. The same condition could occur if you performed those 'Open With' actions on a .JPG or even an .EXE, but the .LNK explanation is far more likely for items on the desktop.

The good news is that a small free utility has been written to undo the .LNK file association that was created. There is also a registry hack to accomplish this, but I recommend the unAssoc utility for its ease of use. I used to send people to another site to download this utility, but that site has become confusing with all the ads and false download links. I've put a copy on by own SkyDrive here.


                                    

Just download, then use the utility to unassociate the .LNK files for the User or globally Delete the file type and let Windows rebuild a new one to defaults.

This isn't an issue with the images themselves, it is usually a problem with the gallery database. Sometimes the database may be totally corrupted (which looks like your case), sometimes an image or small number of images may be moved or deleted but the gallery not updated so it still tries to find an image that isn't there. You will probably find that your images a fine if you access them from File Manager.

 

As a solution, reformat the phone but when you do the backup first, don't include the 'user files' - just make a separate copy of them to PC using mass storage mode - this way you should not reimport the issue when you restore the backup.

 

In case the issue is with images stored on memory card or mass memory, you may need to format that separately.

A fix for links not working in OneNote Online

I've found that sometimes links to another page within the same section (created in OneNote 2016) don't work when using OneNote Online: "Sorry, OneNote Online can't open this link".

To replicate, create a new page in the "welcome" section of classnotebook using OneNote 2016. Go to this page and type something. Right click on page name and select copy link to page, paste into 1st page of welcome section.

Edit this link and delete &section-id={.......}, so url starts with onenote:#Contents&page-id={.....

Once synced the link now works with onenote online.
