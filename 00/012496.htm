Update on the problems reported above:

Problem (1) (Default window size and position) behavior has changed some but the problem remains in build 15014.  Apps which are opened in a window and then exited with the window at the desired position/size retain this setting for the next use only.  After that, the window reverts to a very small size positioned at the upper left of the Desktop.  This behavior is sporadic, not happening in all cases (even with the same app).  The apps I notice this problem with include Chrome and Outlook 2016.

Problem (2) (Desktop icons rearranging) seems to have been fixed in the current preview build 15014.

Any suggestions appreciated.
