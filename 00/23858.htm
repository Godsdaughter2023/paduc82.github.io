This happens because your are sending via the SMTP server of Gmail and have Outlook configured to save a copy of each Sent Item.  

As all the messages that you sent via the Gmail SMTP server also automatically store a copy in your Sent Items folder of the Gmail account, you’ll end up with duplicates. There is no option within Gmail to turn off this behavior.

To prevent Sent Items from being duplicated, you’ll need to disable the Outlook option to save a copy of your Sent Items in your Gmail account.

ken sheridan say: The condition for the macro could be:

DLookUp("YourDateField","YourDateTable","YourDateField < Date()") Is Not Null

The action to import the data would be conditional on this, as would be the action to update the value of the date field to the current date.

While macros are cheap and cheerful, they are very limiting, and generally used only rarely by experienced developers.  Code in the button's Click event procedure would be the more usual approach:

Const MESSAGETEXT = "Data already uploaded today."
Dim strCriteria As String
Dim strSQL As String

strCriteria = "YourDateField < #" & Format(VBA.Date,"yyyy-mm-dd") & "#"

If Not IsNull(DLookUp("YourDateField","YourDateTable",strCriteria)) Then
    ' code to import data goes here
    strSQL = "UPDATE YourDateTable " & _
        "SET YourDateField = #" & Format(VBA.Date,"yyyy-mm-dd") & "#"
    CurrentDb.Execute strSQL
Else
    MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"
End If

If you are not sure how to write the code to import the data then you could run your current macro at this point in the code with:

    DoCmd.RunMacro "YourMacroName"

But I'd recommend that you invest some time in learning how to write VBA code.  Sooner or later you'll find that you can't do what you want with macros, but even where a macro can be used, code has a lot of advantages.

If you are unfamiliar with entering code in event procedures, this is how it's done:

1.  Select the button and open its properties sheet if it's not already open.

2.  Select the On Click event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new lines between these.

However you do it you'll need to first manually 'seed' the date table with a single row of a date before the current date.
