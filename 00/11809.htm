jeeped say:

... each range is named (eg ProcessA, ProcessB, ProcessC etc) ...


Here s some code to get you started,

Sub mcrAdd10()
    On Error Resume Next
    For r = 65 To 90
        If Range("Process" & Chr(r)) Is Nothing Then
            Exit For
        Else
            For Each c In Range("Process" & Chr(r))
                c.Value = c.Value + 10
            Next c
        End If
    Next r
End Sub

 I would probably reserve two blank cells somewhere. One to hold a value for amounts to add and one for a value to multiply. The code routine could look that up easily. Likewise, the above code could easily be copied and modified into a second macro routine for multiplication purposes.

Hi Mattwyld,

 

Please note that the storage limits of shared mailbox is 50 GB rather than 100GB. You can see proof in this official article: https://technet.microsoft.com/en-us/library/exchange-online-limits.aspx?f=255&MSPPError=-2147217396.

 

Since all other mailboxes has been migrated, as a workaround, you can please give the Full Access permission of the shared mailbox to one normal mailbox and then set the normal mailbox in Outlook client. Then use the PST export function of Outlook client to export the shared mailbox data to a PST file.

 

Note: if the normal mailbox is a on-premise mailbox, then Automapping will add the shared mailbox in Outlook client automatically. If it’s an online mailbox, Automapping doesn’t work. In Outlook client > File > Accounting Settings > double click the mailbox > More Settings > Advanced > Add the shared mailbox manually.
