sharonbmcgard say:

pull data from one sheet into another sheet in same workbook.

I have a spreadsheet with approx. 470 rows of data.  Columns A - W. 

 

Column N contains a formula, multiplying 3 other columns.

 

What i want to do, is have Excel look at Column N and find the top 25 values.  Then take the enitre 25 rows and copy them into a separate sheet.

 

For example:

 

If Row 200, Column N contains the highest value in column N, copy the entire row 200 into a spreadsheet in the same workbook..

If Row 3, Column N contains the 2nd highest value in column N, also pull the entire row 3 into the spreadsheet.

and so on, for the highest 25 column N Values.

jeeped say:

... The problem i am having is that if there are 5 of the same values in column N, it does pull over all 5 rows, the problem is that it duplicates the first row for all 5 rows ...


Interesting problem and one that I probably should have accounted for. I had endeavored to avoid the use of array formulas as they can quickly become very CPU intensive. However, the need to overcome the significant number of ties in the N column values would seem to supersede calculation conservation. So with data like this,

     

You would like results like this,

     

This will require a standard formula in column N and an array formula for processing a count of possible duplicates. The standard formula in column N is required for the remainder of the formulas so put that in first. N2 becomes,
=LARGE(Sheet1!$N$1:$N$999,ROWS($1:1)) 
Next, put this array formula into A2,
=INDEX(Sheet1!A$1:A$999, SMALL(IF(Sheet1!$N$1:$N$999=$N2,ROW($1:$999)),COUNTIF($N$2:$N2,$N2)))
... and confirm it with Ctrl+Shift+Enter↵ rather than just Enter↵. Once entered correctly, it can be filled right to column M, and you can Copy then Paste Special, Formulas into O2:W2.

With all of that in place, A2:W2 can be filled down to A26:W26 for the top 25 entries.

I've uploaded that sample worksheet to my SkyDrive here for your reference. Post back with any difficulties that you have. I added a red fill/highlight warning in the case that the 25th entry has duplicates in the 26th, 27th, etc.
