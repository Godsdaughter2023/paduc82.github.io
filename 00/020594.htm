rob koch say:

Most of the files found in the x86 folder are related to Multilingual User Interface (MUI), and other client display portions of the program, which likely make these portable between the 32-bit and 64-bit versions of the program.

Since creating x64 specific versions of these would create additional overhead or may not even be possible in some cases due to operating system or library support, this isn't really surprising.  We're still caught in the conversion between 32-bit and 64-bit operating systems/applications and likely will be at least until the next operating system release, which might finally drop full 32-bit (only) support.

Note that if you compare the installations on a 64-bit and 32-bit system, you'll find that these files are always found in the x86 Program Files folder and never within the main 64-bit Program Files folder.  That means they are inherently 32-bit files on all systems, regardless of operating system type.

< EDIT > Note that is also implies that the files found within the x64 Program Files folder are probably all specific to the operating system they support, either x86 or x64.
