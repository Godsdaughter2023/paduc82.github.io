young yang msft support say:

Hi Pierre,

Thanks for the information. So my assumption is correct? One user has both an on-premise mailbox and an Exchange Online mailbox?

Yes, you’re right. The 2 mailboxes are separate, and they are not synced. And this scenario is not officially supported.

Another question, “Is possible for these emails to also go to Exchange Mailboxes?”:

Short answer is no. For your current situation, there is no official solution to achieve this.

From my experience, I suggest you deploy the normal/typical hybrid deployment:

a. If the mailbox is still hosted in on-premise Exchange, do not create Exchange Online mailbox for this user. Please sync this user to Office 365, then this user will exist in Exchange Online as a mail user (not mailbox).

In hybrid, any emails sent to this mail user will be routed to the corresponding on-premise mailbox.

b. If you want to host the mailbox in Exchange Online, please migrate the mailbox to Office 365 via hybrid remote move migration. Then, the Office 365 mail user becomes a mailbox, and the corresponding on-premise mailbox becomes a mail user (also called remote mailbox).

In hybrid, any emails sent to the on-premise mail user will be routed to the online mailbox.

This is the officially supported hybrid deployment.

Anything unclear, please let us know.

Creating Directory from Access - Microsoft Community

help mel say:

Creating Directory from Access

I have created an Access database with all individuals of our organization in individual rows.  Each individual is part of a family which I am trying to group in order to create a phonebook-like directory listing each by family.  I have created a column in order to place a common family name for each individual but am having trouble creating a query/report or even using Word to create this directory.  Any help would be immensely appreciated!  Thank you in advance for your time.
graham mandeno say:

Create a report and group it by family name.  You can then show the family name in the group header and list the family members in the detail section.

You may have more than one family with the same name, so this would work better with a separate Families table, with an autonumber primary key (FamilyID) and a text field (FamilyName).

In your Individuals table, store only the FamilyID value.

Then, base your report on a query on these two related tables, sorting by FamilyName and grouping by FamilyID.  That way, different families with the same name will get their own group headers.

Best wishes,
Graham
