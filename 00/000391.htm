copy and paste a rwo multiple times - Microsoft Community

saeideh saedikhosroshahi say:

copy and paste a rwo multiple times

Hi

How can I select a row (range) in sheet1 and paste it 10 times in sheet2 and go to next row in sheet1  and repeat the task r times.

Thanks

ossiemac say:

Try the following. I have assumed that at the destination you want to find the next blank row for each new paste.

Sub RepeatCopyAndPaste()
    Dim S As Long   'Selected row
    Dim R As Long   'Number of rows to copy from source
    Dim D As Long   'Number of times to duplicate row at destination
    Dim L As Long   'Loop for rows to copy
    Dim rngDestin As Range
   
    R = 5       'Number of rows to copy from Source
    D = 10      'Number of times to duplicate row at destination
   
    If Selection.Rows.Count <> 1 Then
        MsgBox "Only select 1 row. Processing terminated"
        Exit Sub
    End If
   
    With Worksheets("Sheet1")
        S = Selection.Row
        For L = S To S + R - 1
            With Worksheets("Sheet2")
                'Get the next blank cell in column A.
                Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
            End With
            .Rows(L).Copy Destination:=rngDestin.Resize(D, 1).EntireRow
        Next L
    End With
       
End Sub

Query Criteria Using A List Box In Form - Microsoft Community

turbon5.0 say:

Query Criteria Using A List Box In Form

Hi,

How can I apply the following criteria using a list box? >"Survey Week 4" And Not Like "baseline *" And Not Like "bonus *"

 I tried to just place this within the list box but I think the characters are making the query come up blank. I don't know how to make the query see the above as is.  I think it sees it as a string of text only.

Thanks.
ken sheridan say: In the list box put meaningful plain English string expressions which identify the criteria you are attempting to apply, e.g. for this:

'Survey week 5+, but not baseline or bonus'

Then in the query use parenthesised Boolean expressions to apply the relevant criteria for the item selected in the list box, e.g.

SELECT *
FROM YourTable
WHERE (Forms!YourForm!ListBox = "Survey week 5+, but not baseline or bonus"
AND YourField > "Survey Week 4" AND YourField NOT LIKE "baseline*"
AND YourField NOT LIKE "Bonus*")
OR  (Forms!YourForm!ListBox = "Some other string expression"
AND <some other criteria>)

If you want to return all rows if nothing is selected in the list box just add:

OR Forms!YourForm!ListBox IS NULL

to the WHERE clause.

If you are combining this with restrictions on one or more other columns in the query then you'll have to be careful to parenthesize the expressions in the WHERE clause correctly to force the expressions to evaluate in the correct priority.  Don't try and do it in query design view, however; the logic is far easier to apply in SQL view.  And be sure to save the query in SQL view or Access will move things around, and at best the logic will be obscured, at worst the query will be too complex to open.

graham mayor say:

I am not sure what you mean by Out of Town for two weeks every Tuesday - which would suggest she is away constantly?

However the following Outlook macro will mark the Calendar for 15 days from the selected day categorised with the Red Colour category.

Sub AddSusanAway()
    Dim objExpl As Outlook.Explorer
    Dim objFolder As Outlook.MAPIFolder
    Dim objCB As Office.CommandBarButton
    Dim objAppt As Outlook.AppointmentItem
    Dim objNewAppt As Outlook.AppointmentItem
    Dim sTime As Date
    On Error Resume Next
    Set objExpl = Application.ActiveExplorer
    If Not objExpl Is Nothing Then
        Set objFolder = objExpl.CurrentFolder
        If objFolder.DefaultItemType = olAppointmentItem Then
            Set objCB = objExpl.CommandBars.FindControl(, 1106)
            If Not objCB Is Nothing Then
                objCB.Execute
                Set objAppt = Application.ActiveInspector.CurrentItem
                sTime = objAppt.Start
                objAppt.Delete
                Set objNewAppt = objFolder.Items.Add
                With objNewAppt
                    .Start = sTime
                    .End = sTime + 15
                    .ReminderSet = False
                    .AllDayEvent = True
                    .Subject = "Susan Away"
                    .Categories = "Red Category"
                    .BusyStatus = olFree
                    .Save
                End With
            End If
        End If
    End If
    Set objCB = Nothing
    Set objAppt = Nothing
    Set objNewAppt = Nothing
    Set objFolder = Nothing
    Set objExpl = Nothing
End Sub

graham mayor say:

The following uses the above code to set the repeating calendar appointment every other Friday starting 'today'. Though frankly for a single repeating appointment a macro is hardly necessary. It would be simpler to create the appointment manually.

The Out of Office replies to your e-mails is less straightforward and how you do it will depend on the type of e-mail account. Unless you have a PC sitting on a desk somewhere running your Outlook account on those Fridays and automatically answering your incoming mail, this is going to have to be done at the mail server. Most server types provide such a function.

Sub AddAway()
Dim objExpl As Outlook.Explorer
Dim objFolder As Outlook.MAPIFolder
Dim objCB As Office.CommandBarButton
Dim objAppt As Outlook.AppointmentItem
Dim objNewAppt As Outlook.AppointmentItem
Dim oPattern As RecurrencePattern
Dim sTime As Date
    On Error Resume Next
    Set objExpl = Application.ActiveExplorer
    If Not objExpl Is Nothing Then
        Set objFolder = objExpl.CurrentFolder
        If objFolder.DefaultItemType = olAppointmentItem Then
            Set objCB = objExpl.CommandBars.FindControl(, 1106)
            If Not objCB Is Nothing Then
                objCB.Execute
                Set objAppt = Application.ActiveInspector.CurrentItem
                sTime = objAppt.Start
                objAppt.Delete
                Set objNewAppt = objFolder.Items.Add
                With objNewAppt
                    .Start = sTime
                    .End = sTime
                    .ReminderSet = False
                    .AllDayEvent = True
                    .Subject = "Out of Office"
                    .Categories = "Red Category"
                    .BusyStatus = olFree
                    Set oPattern = objNewAppt.GetRecurrencePattern
                    With oPattern
                        .RecurrenceType = olRecursWeekly
                        .DayOfWeekMask = olFriday
                        .Interval = 2
                    End With
                    .Save
                End With
            End If
        End If
    End If
    Set objCB = Nothing
    Set objAppt = Nothing
    Set objNewAppt = Nothing
    Set objFolder = Nothing
    Set objExpl = Nothing
End Sub

If the mail accounts are set up as pop accounts you have to enable the setting to 'retain a copy on the server' in the email setup.

The default setting for pop mail is to download all mail from the server, and under these circumstances, mail will be downloaded on the first device opened.

Perhaps better to set up the mail accounts as IMAP on all devices, which is a copy of what is on the server

ken sheridan say:
As you want to save the selected data you will need a third table into which you can insert records.  Each record will represent one of your saved 'documents'.  This table will have two foreign key columns, e.g. EmployeeID and LocationID referencing the primary keys of the two existing tables.  In relational database terms this table is modelling a many-to-many relationship type between the two existing tables, by resolving the relationship type into two one-to-many relationship types.

The question now arises as to whether this third table should include any other columns, into which data from one or both of the two existing tables is inserted.  This will depend on whether the data in each of the two referenced tables can change, while the corresponding data in each 'document' should remain fixed as that at the time the 'document' record was created.  If this is the case then the new table should contain columns in which that data will be stored.  If not, then the new table should not contain such columns as the values can be obtained at any time by joining the three tables in a query, or by referencing the Column property of a text box bound to the relevant foreign key column.

You'll find examples of both situations in DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly. 

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file, you'll find a form in the section on building the user interface for entering orders.  This form includes an order details subform.  In the parent form, when a customer is selected, their address data is shown in unbound controls which reference the Column property of the bound combo box to show data from the various columns of the combo box's RowSource query.  If the customer should change their address this form will consequently always show the customer's current address.

In the order details subform on the other hand when an item is selected in the combo box, the UnitPrice control, rather than being an unbound control which references the item combo box's Column property is a bound to a UnitPrice column in the OrderDetails table.  Its value is inserted when an item is selected by the following code in the AfterUpdate event procedure of the item combo box:

    ' assign current unit price from Items table
    ' to UnitPrice column in OrderDetails table
    Me.UnitPrice = Me.ItemID.Column(1)

Consequently the unit prices in each order will remain fixed as those current at the time the order was created, regardless of subsequent changes to the unit price of items in the Items table.  In the language of the database relational model  the UnitPrice columns in both the Items and Products table is said to be functionally determined by whole of the primary key of the relevant table, which means there is no redundancy as each table is normalized to Third Normal Form (3NF).  See Normalization.zip in the same OneDrive folder for a simple introduction to normalization.

While in a form, data from referenced tables can be pulled in by referencing the Column property of a combo box, this should not be done in a report.  Instead, the report should be based on a query which joins the relevant tables and returns the relevant columns from the referenced tables as well as the relevant columns from the referencing table.
