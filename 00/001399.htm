That is by design. Integer fields can only hold whole numbers - that's what the word integer means. The format settings will be ignored.

Choose DataType=Number, then FieldSize=Single or Double.

You could also use Currency, which is a bit of a hybrid between Integer and Double, and has a fixed 4 decimals, but you can format as two.

per l bylund say:
When a user deletes an email while it is open, it moves to Deleted Items as an Unread message.  Is there a setting somewhere that governs this?  The message should be marked as read if it was open in the first place, correct?


Your description of the problem is too limited to say what could be the issue. I see several potential solutions:

 

(1) You have the default setting for marking read, which marks messages as read only when navigating away from the message. This setting keeps the message marked as unread while reading it in the preview pane, and I guess it may be kept as unread if deleted before navigating to another message. (I always change this setting to marking as read as soon as I've stayed on the message for two or three seconds, since I find it annoying to have all messages marked as read when quickly stepping through the messages in my inbox).

 

(2) You have the setting that counts all messages rather than just unread messages, which means the deleted items folder lists the number of messages (such as "Deleted Items (12)"). I use this setting to keep track of the "size" of my deleted items folder so that I empty it before it gets too large. This does not mean the individual messages are explicitly marked as unread though, if you open the deleted items folder they may be read or unread (but both are counted). So from the inbox they may appear to be unread but aren't.

 

Neither of these are, as I understand it, bugs. Both of them are settings that you can easily change (the former through email settings in the Tools menu, the latter through right-clicking your deleted items folder and change properties).

 

Hope this helps!

sajith nirmal say:

Well the default behavior of Outlook is when the mail item loses focus it will be marked read. If you select an item and delete it, it will go to the "Deleted Items" folder as Unread. You can change by doing the following

  • In Outlook, click on File
  • Click on Options
  • Select the Mail tab
  • Click on the "Reading Pane..." button
  • Check the box "Mark items as read when viewed in the Reading Pane"
  • Change the Wait to 0
