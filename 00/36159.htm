Batch Help - Microsoft Community

eddiedunlop say:

Batch Help

I want to write this line to text file

from a batch file.

echo &#160; >temp.txt

I won't work, I have tried  echo ^&#160;

and echo ^&^#160;

Help

lemp say:

What are you trying to do?

The string &#160; is HTML for a non-breaking space (also written &nbsp;)

The command interpreter doesn't understand HTML.

The command interpreter recognizes the ampersand (&) as a "command separator" which simply means that it treats whatever follows the & as a command.  Thus, if you type the following in a command prompt

   echo &#160;

the command interpreter attempts to interpret #160; as a command -- and it can't.  Similarly, the command interpreter recognizes the semicolon (;) as a blank space (I think).

C:\Users\LemP>echo &#160; > temp.txt
ECHO is on.
'#160' is not recognized as an internal or external command,
operable program or batch file.

The caret character (^) "escapes" the ampersand, allowing the command interpreter to treat it as just another character.  Thus the following

C:\Users\LemP>echo ^&#160; > temp.txt

results in the creation of a file named temp.txt that contains the 6 characters &#160;

So ... depending on what you want your batch file to accomplish, there may (or may not) be a way to achieve that goal.

The following pages may be of interest:

http://judago.webs.com/batchoperators.htm

http://www.robvanderwoude.com/escapechars.php

greg carmack - windows mvp 2010-2020 say:
Hi Mr. K. I"m Greg, an installation specialist, 9 year Windows MVP and Independent Advisor, here to help you.

The error normally means that someone has activated Windows with a Volume License Key meant only for business or organizations that activate a large number of PC's, which need to check in with the organization's server regularly to reactivate.

Sometimes this is done by shops reinstalling Windows, unnecessarily because the PC either came with a Windows Product Key embedded in the BIOS/UEFI firmware, or it was already activated for free Upgrade and would have reactivated after a reinstall had they given it the chance and not interrupted this by installing the illegal key.

If you know Windows was originally activated legitimately by one of these methods then to fix the problem what I would do is go offline to remove the Volume License key and then restore the Generic Product Key that is used to activate with the Digital License issued by Upgrade or Factory install. Then we'll see if it activates:

1) Go offline by opening Action center at the right end of Task Bar, then clicking on Airplane mode to turn off internet.

2) Next type CMD in Start Search, right click to Run as Administrator, then right click to copy and paste this command in and press Enter: slmgr -upk

3) Restart PC and then insert Generic Product Key for WIndows 10 Pro from here: http://winaero.com/blog/generic-key-to-install-...

If the PC originally came with Windows 10 from factory, then you can also install ProducKey: http://www.nirsoft.net/utils/produkey-x64.zip to look for an OEM-BIOS key (only!) and copy/paste it into Activation Settings. But be aware that these embedded keys often will convert to Digital Licenses using only Generic Version key, as can a retail license.

This sets everything back to default so that if there is a Digital License issued during a proper Upgrade it should reactivate within a few days. If not we will need to try something else:

What Windows Operating System came on that exact model PC originally - you can google it's make and model to read the specs to find out? If it says Windows 7, 8 or 8.1 Home version then that is a clue you need to reinstall Home instead of Pro to try to activate. We might even be able to still activate it fresh with a Windows 8/8.1 Product Key read from the BIOS or on the Windows 7 COA sticker on the PC, or any of those retail keys.

If nothing else works you may need to reinstall the licensed Windows edition that came with that PC, or buy a copy of retail Windows 10 to reinstall (if necessary) and activate it. I would buy only from a major retailer who backs their sales or Microsoft here: http://www.microsoftstore.com/store/msusa/en_US...

If you were sold a Volume license product key or the PC came from seller activated with one, then It is piracy when someone resells Windows with no way to legally activate it. You can report this to Microsoft here: https://www.microsoft.com/en-us/howtotell/cfr/r.... In the past when victims cooperated with an investigation they would sometimes give them a free copy as thanks.

You can also contact the seller and let him/her know you are taking these actions and that to avoid prosecution for felony theft and piracy they need to make the activation good.

I hope having all possible options helps.

Feel free to ask back any questions and let us know how it goes. I will keep working with you until it's resolved.

Redundant backups against EULA? - Microsoft Community

ronm54 say:

Redundant backups against EULA?

In the EULA for Windows 7 it says "10.    BACKUP COPY. You may make one backup copy of the software. You may use it only to reinstall the software on the licensed computer."

So if one makes a redundant backup of my OS then one is breaking their licensing agreement?


Microsoft is against the good backup policy of redundancy? 

hairyfool say: It is specifically refering to the installation media not a backup of your running O/S. So in the terms of the EULA the System Image backup does not count.

ronm54 say: Why does it say that it refers to "desktop operating system" then?

From the EULA defining software: "Software. The software includes desktop operating system software. This software does not include Windows Live services. Windows Live services are available from Microsoft under a separate agreement."

ronm54 say: And as far as that is concerned, the EULA implies that one has to choose between backing up the installation software or the installed software running on the computer.  If one did a backup of both then they would be making more than one copy of that software because there is going to be code from the install media that is installed on the computer itself.

palcouk say:

There are no limitations on any licence for creating a backup or Image, you can in fact make multiple backups or Images

Niether a backup or Image can be used for anything other than reinstalling/recovering the origonal software or data

andre da costa say:
The backup refers to installation media, not the installed operating system itself. You can create multiple offsite backup copies of that if you wish which you can use to restore on the same computer.

ronm54 say: Why does the EULA specifically say that when it refers to software that it refers to the desktop operating system.  It doesn't say anything about software referring to installation media only. 

From EULA:
"a.    Software. The software includes desktop operating system software. This software does not include Windows Live services. Windows Live services are available from Microsoft under a separate agreement."

It says includes not excludes. If I am for example changing the registry in Windows am I not effecting the desktop operation system software?  What would be the name of the installed software if not "desktop operating system"?

hairyfool say:

"The software" ie the product for which you have bought the license for and what the EULA applies to.

 

It can only refer to that as it cannot refer to anything else. Your disk will have other software, data files, music files, pictures, customisations etc. which it cannot include in any definition and will have its own copyright restrictions. On the disk for which they have given you a license to make a backup copy is the only place it will exist in isolation.

 

To a certain extent they are moving to a situation which you are postulating. Windows 8 has moved away from complete system backups as the norm to what is called file history and that does not include "The Software"
