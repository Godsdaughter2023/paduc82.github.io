a. user say:

Hi Rou,

A SPF record is used for the receiving side to check whether the sender’s IP address is approved by the sender’s domain. For example, when domain2.com received an email from domain1.com, it will firstly check whether there is a SPF record, if so, then it will check whether the sender’s IP address matches the IP predefined in the SPF record from domain1.com, if so, that means the IP address which sent the email has the permission to send as doamin1.com. This record intends to prevent spoofing.

So usually we don’t recommend our users to add external domain into SPF record to avoid spam block. If you’d like to let the external domain to be recognized as a safe domain, you can use the safe domain function in EAC. EAC (Exchange admin center)>protection>spam filter>allow list>under Domain allow list you can add the external domain to it.
