lemp say:
To give a little insight to the "install a new driver" solution discussed in the Technet forum thread, it seems that the problem here is the same as one commonly encountered when attempting to update printer drivers:  a user downloads the new driver and installs it but the problem remains.

What is happening is that if the old driver is not first removed, it is not replaced by the new driver.

Windows installs hardware devices using INF files.  These are located in %windir%\inf.  In reality, the previous statement is incomplete.  The first time a device is installed, Windows uses the INF file to create a corresponding PNF file located in the same directory.  The PNF file is a "precompiled" version of the INF file and Windows setup actually uses the PNF file because it is "more efficient" to do so.

If you download a new driver and only replace the INF file and do not delete the old driver files, when you "update" the driver, Windows setup looks at the old PNF file which points to the old files and uses them.

So ... either completely uninstall the device or at least delete the appropriate PNF file before trying to install an updated driver.

And get the new driver either from Toshiba or from manufacturer of the wireless adapter in the computer.  Do NOT get it from Windows Update.
