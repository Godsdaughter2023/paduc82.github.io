don cuthbert say:

The System partition is not normally the same as the Boot partition. System only needs 100-450 MB.  Windows (C: drive) is the Boot partition. It is best to remove ALL partitions before a clean install of Windows to a hard disk, unless you are trying to preserve the existing OS (or a data partition) like you were.

Deleting a partition does not actually wipe all of the data, but you would need to use recovery software to try to recover anything from a deleted partition. There are various solutions available online to try to recover a deleted partition. If you really need to recover anything, the important thing is not to write any data to the disk. Anything recovered has to be written to other storage media.

Sub CopyPasteWBks()
Dim WB As Workbook, myWB As Workbook
Dim sDate As String
Dim myWS As Worksheet
Dim X As Long, N As Long
Dim v As Variant
sDate = InputBox("select date", , "01-17-12")
If sDate = "" Then Exit Sub
Application.ScreenUpdating = False
On Error Resume Next
Set myWB = ThisWorkbook ' Target Wb
Set myWS = myWB.Sheets("TargetSheet") ' change name
N =2
v = Array("ABCD ", "EFGH ", "WQER ") 'change names
For X = LBound(v) To UBound(v)
Set WB = Workbooks.Open(ThisWorkbook.Path & "\" & v(X) & sDate & ".xlsx")
WB.Sheets("Sheet1").Range("A2:I15").Copy
myWS.Cells(N, 1).PasteSpecial xlValues
N = myWS.Cells(Rows.Count, 1).End(xlup).Row + 1
Application.CutCopyMode = False
WB.Close False
Next X
myWB.Save
On Error GoTo 0
Application.ScreenUpdating = True
End Sub

tom van stiphout say:

The reason this is complicated starts with incorrect database design. In a relational database (unlike a spreadsheet) you should not define column values as attributes this way. Think skinny rather than wide:

tblPerson

PersonID

PersonName

maybe email address, other contact info

tblProducts

ProductID

ProductName

Now you can see the huge benefit: if you ever decide to sell another product, all you have to do is add a row to this table, not add additional fields and fix up existing queries.

tblLikes

PersonID long int PK

ProductID long int PK

Now people can have one or more likes, and your queries become trivial.

Probably also need:
tblStores

StoreID

StoreName

StoreLocation

tblStoreProducts

StoreID long int PK

ProductID long int PK

This table will currently have 1 entry per store, but that could change if a store in future will sell two products. All you will have to do then is add one row in this table, and everything else stays the same.

And don't forget to enforce relations in the Relationships diagram.

ken sheridan say: As Tom points out, you are 'encoding data as column headings'.   A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.

For an illustration of how to automatically recast a badly designed table like this into a set of correctly related tables see UnencodeColumns.zip in my public databases folder at:

a. user say:

Hello James,

Thank you for your post.

Kindly ask the CIO to post announcement in internal group than every user subscribed to that group will receive email notification.

But I would request you to kindly go through below points for better understanding regarding the email notification:-

  1. The user should be subscribed to the notifications he wishes to receive that can be set under edit settings -> Notification in user profile.

  1. If the user is online in Yammer when the post was created, Yammer checks if the user has read the message or not. If the user reads the message, logged into yammer with any device than in notification button it will show next to message button, no notification email will be sent.

If the user does not read the message, a notification will be sent after 15 minutes.

  1. If the user is not online in Yammer, the notification is sent immediately. By not online we mean he is not logged in to Yammer from any device.

Microsoft is making some changes to how notifications work in Yammer, here is a summary as below:-

  • logic has been added that holds email (15 minutes) and push notifications (2 minutes) while you are active on the Yammer web site or in the mobile app

  • After that delay Yammer will check that a message thread is still unread before sending an email or push notification to avoid notifying you about things you've already seen on the web or in the mobile app

  • To make it easier to triage your Yammer emails by providing very explicit explanations for why you're getting the emails

  • Some changes to the format of email notifications, Tuck has provided Microsoft with feedback on what we did not like about examples we have seen

Developed new business logic for what triggers email notifications and will send notification emails for the following scenarios:

  • new announcements
  • when someone mentions you on Yammer
  • when someone is added to a direct message
  • when a non-group participant is added to a private group conversation
  • when a message thread conversation in your Yammer Inbox goes from read to unread
  • when someone replies directly to your message
  • when someone sends you a new direct message

With the exception of the above scenarios, Yammer will stop sending email notifications if you haven't read a conversation thread since receiving a previous notification for the same conversation. In other words, lack of engagement will be seen as lack of interest.

Of course this is all subject to your notification preferences. If you disable email notifications then no notifications will be sent.
