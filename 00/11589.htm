steven. s say:

Hey RPMM,

 

I don’t know of a command to use in the Target: of the shortcut to always run the shortcut as another user. You can get this to work by using a .CMD script to launch Internet Explorer as another user. This can be done by:

·        Click  Start

·        Click  All Programs

·        Click  Accessories

·        Open  Notepad

·        Type/paste:

o  runas/user:*insert the account path here* "\"c:\program files\internet explorer\iexplore\" "

·        Click  File

·        Click  Save As…

·        Choose the location you would like the save the file (probably the Desktop)

·        Change  Save as: type to All Files

·        Type  IE.CMD into File name:

·        Click  Save

 

Some examples of the runas command would be:

·        Domain name=TestDomain & User Name (on domain)=TestUser

o  runas/user:TestDomain\TestUser "\"c:\program files\internet explorer\iexplore\" "

If you wanted to use runas for another local (not attached to the domain) user account:

·        Computer name=TestPC & User name=TestUser

o  runas/user:TestPC\TestUser "\"c:\program files\internet explorer\iexplore\" "

 

Then open the IE file on your desktop. You’ll be prompted for the password to the user account and then Internet Explorer will open as the other user.

 

Hope that helps, look forward to hearing back from you.

Hi,

I had same problem (same error but with "Micrososft Store" and "Xbox" apps). Today I finded out solution. Maybe it will be helpfull also for you.

Main step was to remove stored credentials (remembered username with password ) of problematic MicrosoftAccounts from Windows 10. I am not talking about removing account from OS. When I removed stored accounts informations, applications asked me for my username and password even they suggested me to use one of remembered accounts. And this approach worked for me to solve 0x80048103 error

Steps to solve this problem are: 
1. run Powershell
WindowsButton + R
Type "Powershell" -> click OK
2. run app "Stored User Names and Passwords" 
In Powershell type command
"rundll32.exe keymgr.dll,KRShowKeyMgr"
Command opens little window with title "Stored User Names and Password"
3. remove problematic MicrosoftAccount
Select problematic account (click on it) in list (microsoft accounts starts with prefix "MicrosoftAccount:user=" )
Click at "Remove"
Close "Stored User Names and Passwords". Click at "Close" button
4. restart preoblematic app and try to login
When app, after restart, will suggest you existing account, dont worry. After choose suggested account, it will ask you for password (maybe also fingerprint, and so on ... )

Good luck. Hope it help.
