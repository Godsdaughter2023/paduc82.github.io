jay freedman say:

Since you've gotten the theme to fix your problem, I don't want to put you to more work than necessary. If you happen to run into more problems or want help with a macro, you know where to find me.

For educational purposes, here's some background for my comments about Shape and InlineShape objects...

You may already be aware that the "object model" in Word -- the organization of things you can work with in VBA -- consists of objects of various kinds: documents, ranges, paragraphs, and so forth. Each object has a set of properties (things that have a value, such as font size) and a set of methods (things that manipulate the object, such as a Delete command).

There are also collections of objects, usually all of the same kind. A document has a collection of paragraphs, a collection of words, and so forth. Of particular interest here, a document has two collections named Shapes and InlineShapes. Each collection may contain zero or more objects (usually graphics such as pictures, rectangles, etc. but "text boxes" are also included). The InlineShapes collection contains objects whose text wrapping is "In line with text", while the Shapes collection contains objects with text wrapping of "Square", "Top and Bottom", and all the other kinds that are not in line (sometimes called "floating").

When you want to do something to all of the Shape objects in a document, you can use a loop to operate on each member of the Shapes collection -- that's what my macro was meant to do. If there are no Shape objects in a document, then the property ActiveDocument.Shapes.Count has the value 0, and the loop ("from 1 to 0") does nothing at all.

Hi Pawel,

I've been tracking causes of slowness in Excel for Mac. The list has grown since the question you replied to was answered.  Here's what I have right now:

Slow Excel

  • Copy (Cmd+C) causes spinning beach ball delay for 3-4 - Microsoft Community (HandOff issue)
  • Another user reported the problem goes away in a new Mac OS user account. A corrupt user account can cause Excel to be slow
  • Zoom – set zoom to 100% or less
  • Another user reported the problem goes away when menu manager is turned off. The name of the manager was not disclosed. The general case is that a 3rd party application intercepts keyboard and mouse input, performs actions, and then passes the keyboard and mouse activity on to Excel resulting overhead and delay.
  • A different user reported uninstalling QuicKeys input manager fixed the problem.
  • Page Layout view is slow. Use Normal View, instead
  • Another user reported that inserting pictures caused Excel to be slow and/or hang
  • Not enough free memory. Use Activity Monitor App (it’s in Applications > Utilities) to monitor memory usage while you work in Excel. If you are using swap space, it means you ran out of actual RAM and are using virtual RAM, which is very slow.

 

 The symptoms you describe are most likely attributable to a 3rd party app intercepting your mouse and keyboard input. Booting to Safe Mode by holding the Shift key down as your Mac starts up from a restart or cold boot will tell you whether or not this is a prime suspect. Booting to Safe Mode takes a long time, so don't be worried about your Mac seeming to take forever to boot. If speed is good during the cold boot, that's a solid indicator a third party app is the culprit. If the speed does not improve, something else is going wrong.

 Are you using a regular mouse and keyboard? Is there anything unusual about your hardware setup?

 Did you install anything else when you first noticed the slowness?

how can I create a .csv file via the TransferText macro command - Microsoft Community

lac55 say:

how can I create a .csv file via the TransferText macro command without double quotes around each field

I have a database that creates a .csv file via the TransferText maco command that adds double quotes around each field.  How can I create the same file without the double quotes?

ken sheridan say: Is this in a context where VBA is ruled out, e.g. a Web database?  If not I've always used the following function:

Function ExportToText(strQuery As String, _
                    strExportTo As String, _
                    strDelim As String, _
                    blnQuoteText As Boolean, _
                    Optional blnHasFieldNames As Boolean = True)
                    
'Accepts:
' strQuery - Text - name of query to be exported
' strExportTo - Text- path to file to export to
' strDelim - Text - delimiter character(s) to separate fields
' blnQuoteText - Boolean - True to enclose text or memo data in quotes
' blnHasFieldNames - Boolean - (optional) - True (default) to export field names as first line

   Dim dbs As DAO.Database
   Dim rst As DAO.Recordset
   Dim qdf As DAO.QueryDef
   Dim prm As DAO.Parameter
   Dim fld As DAO.Field
   Dim n As Integer
   Dim strPrintList As String
   Dim strQuote As String
   
   Set dbs = CurrentDb
   Set qdf = dbs.QueryDefs(strQuery)
   
   ' evaluate query's parameters, if any
   For Each prm In qdf.Parameters
       prm = Eval(prm.Name)
   Next prm
    
   Set rst = qdf.OpenRecordset
   
   If Dir(strExportTo) <> "" Then
       If MsgBox("Overwrite " & strExportTo & "?", vbQuestion + vbYesNo, "Export Query") = vbYes Then
           Kill strExportTo
       ElseIf MsgBox("Append rows to " & strExportTo & "?", vbQuestion + vbYesNo, "Export Query") = vbNo Then
           Exit Function
       End If
   End If
           
   With rst
       If Not (.BOF And .EOF) Then
           Open strExportTo For Append As #1
           
           If blnHasFieldNames Then
                ' include column headings in text file
                For n = 0 To qdf.Fields.count - 1
                     strPrintList = strPrintList & strDelim & qdf.Fields(n).Name
                Next n
                ' remove leading delimiter
                strPrintList = Mid$(strPrintList, Len(strDelim) + 1)
                Print #1, strPrintList
                strPrintList = ""
           End If
           
           Do While Not .EOF
               For n = 0 To qdf.Fields.count - 1
                   Set fld = .Fields(n)
                   strQuote = IIf(blnQuoteText And (fld.Type = dbText Or fld.Type = dbMemo), """", "")
                   strPrintList = strPrintList & strDelim & strQuote & _
                   .Fields(n) & strQuote
               Next n
               ' remove leading delimiter
               strPrintList = Mid$(strPrintList, Len(strDelim) + 1)
               Print #1, strPrintList
               strPrintList = ""
               .MoveNext
           Loop
           Close #1
       End If
   End With

Exit_Here:
   rst.Close
           
End Function
