To address the points raised in this and your preceding post, replace your code with the following version:

'===========>>
Option Explicit

'----------->>
Public Sub CopyRangeFromMultiWorksheets()
    Dim SH As Worksheet
    Dim DestSh As Worksheet
    Dim Last As Long
    Dim iRow As Long
    Dim CopyRng As Range
    Const sName As String = "Summary"   '<<=== Change to desired name of summary sheet

    With Application
        .ScreenUpdating = False
        .EnableEvents = False
    End With

    'Delete the sheet "RDBMergeSheet" if it exist
    Application.DisplayAlerts = False
    On Error Resume Next
    ActiveWorkbook.Worksheets(sName).Delete
    On Error GoTo 0
    Application.DisplayAlerts = True

    'Add a worksheet with the name of the sName constant
    With ActiveWorkbook
        Set DestSh = .Worksheets.Add(Before:=.Sheets(1))
    End With
    
    DestSh.Name = sName
    'loop through all worksheets and copy the data to the DestSh
    For Each SH In ActiveWorkbook.Worksheets
        If SH.Name <> DestSh.Name Then

            'Find the last row with data on the DestSh
            Last = LastRow(DestSh, DestSh.Range("A:B"))

            'Find the last row with data on the detail sheet

            iRow = LastRow(SH, Sh.Range("A:B"))
            
            'Fill in the range that you want to copy
            If CBool(iRow) Then
            Set CopyRng = SH.Range("A1:B" & iRow)

            'Test if there enough rows in the DestSh to copy all the data
            If Last + CopyRng.Rows.Count > DestSh.Rows.Count Then
                MsgBox "There are not enough rows in the Destsh
                GoTo ExitTheSub
            End If

            'This example copies values/formats, if you only want to copy the
            'values or want to copy everything look at the example below this macro
            CopyRng.Copy
            With DestSh.Cells(Last + 1, "A")
                .PasteSpecial xlPasteValues
                .PasteSpecial xlPasteFormats
                Application.CutCopyMode = False
            End With

            'Optional: This will copy the sheet name in the C column
            DestSh.Cells(Last + 1, "H").Resize(CopyRng.Rows.Count).Value = SH.Name
            End If
        End If
    Next

ExitTheSub:

    Application.Goto DestSh.Cells(1)

    'AutoFit the column width in the DestSh sheet
    DestSh.Columns.AutoFit

    With Application
        .ScreenUpdating = True
        .EnableEvents = True
    End With
End Sub

                       SearchOrder:=xlByRows, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Row
    On Error GoTo 0

ken sheridan say: If I understand you correctly, what you want is more 'as you select' than 'as you type', i.e. as each selection is made in an unbound combo box, or an unbound check box is checked/unchecked in the case of the Boolean (yes/no) columns, the form's data will be refreshed progressively.  The simplest way to do this is to base the form on a query which references each of the controls as parameters.

Let's assume you have a table MyTable with two text columns FooText and BarText and two Boolean columns IsFoo and IsBar.  Your form is named frmMyForm and the unbound controls are named cboFoo, cboBar, chkFoo and chkBar.  The query would be like this:

SELECT *
FROM MyTable
WHERE (FooText = Forms!frmMyForm!cboFoo
    OR Forms!frmMyForm!cboFoo IS NULL)
AND (BarText = Forms!frmMyForm!cboBar
    OR Forms!frmMyForm!cboBar IS NULL)
AND (IsFoo = Forms!frmMyForm!chkFoo
    OR Forms!frmMyForm!chkFoo IS NULL)
AND (IsBar = Forms!frmMyForm!chkBar
    OR Forms!frmMyForm!chkBar IS NULL);

The FooText and BarText columns could equally well be of a number or date/time data type; the SQL would be exactly the same.  Note that the chkFoo and chkBar check boxes would need to be triple state so that they could be left Null, which shows as greyed out.

You have two options when designing the form:

1.  The form can be bound to the above query, and be in continuous forms view, with the unbound controls can be in its header.  In this case you'd requery the form in the AfterUpdate event procedure of each combo box with:

Me.Requery

2. The form containing the unbound controls can be unbound, and within it would be embedded a subform based on the above query.  In the AfterUpdate event procedure of each unbound control, instead of Requerying the form, the subform control (i.e. the control which houses the subform) would be requeried, e.g.

Me.sfcMySubform.Requery

The subform can be in datasheet view or continuous forms view.
