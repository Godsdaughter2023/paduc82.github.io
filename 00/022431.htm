lenard van der maas say:

Make VBA scroll until a specific column/row is the last one visible (on any screen size)

Hi everyone,

I recently came across a problem for which I couldn't find a solution online, but after a lot of research, I figured out a solution, so I thought I'd share it with you.

Question:

How do I let VBA scroll through an excel sheet until a specific colum/row is the last one visible on the screen?

Example:

Here's the spreadsheet on which I wanted to apply this macro.

The blurred columns on the left are frozen. Then there are 103 columns to the right of that, each one representing 1 week, in which I enter data every week. VBA automatically hides the columns that represent future weeks so the last week visible is the current week. Then to the right of that, there are 2 more columns with some extra information about each row of data.

What I wanted:

As soon as anyone opens the file, VBA should scroll through the columns to make show as much of the column I prefer to be the most right column (column 'DI'), while showing as little of columns to the right of column 'DI' as possible. All this regardless the size of the window.

Why this is difficult:

All the methods for scrolling in VBA make the destination column the center or leftcolumn, except 1, which is

ActiveWindow.SmallScroll

The problem with it is that it's relative to the current scroll. It simply takes the most left column/top row applicable for scrolling (it ignores frozen cells) and scrolls up/down/left/right a specific amount of cells, so that if row 9 is the first row visible and you use ActiveWindow.SmallScroll to scroll up 1 row, it will make row 8 the first visible row, etc.

How I solved it:

I came across this way of getting the visible range of cells of the active window. That is: the range that is actually visible on the screen.

ActiveWindow.VisibleRange

Now I saw this as a way of checking what the right most visible column is. So I took the last column of this range and took it's column number:

currLastVisibleCol = ActiveWindow.VisibleRange.Columns(ActiveWindow.VisibleRange.Columns.Count).Column

Now the only thing I had to do is scroll through the columns in a loop until the last row visible is the one I want:

    prefLastVisibleCol = numberOfTheLastColumnToBeVisible
    currLastVisibleCol = ActiveWindow.VisibleRange.Columns(ActiveWindow.VisibleRange.Columns.Count).Column
    lastActualScrollCol = ActiveWindow.ScrollColumn + 1
    While currLastVisibleCol <= prefLastVisibleCol And lastActualScrollCol <> ActiveWindow.ScrollColumn
        lastActualScrollCol = ActiveWindow.ScrollColumn
        ActiveWindow.SmallScroll 0, 0, 1, 0
        currLastVisibleCol = ActiveWindow.VisibleRange.Columns(ActiveWindow.VisibleRange.Columns.Count).Column
    Wend
    lastActualScrollCol = ActiveWindow.ScrollColumn + 1
    While currLastVisibleCol > prefLastVisibleCol And lastActualScrollCol <> ActiveWindow.ScrollColumn
        lastActualScrollCol = ActiveWindow.ScrollColumn
        ActiveWindow.SmallScroll 0, 0, 0, 1
        currLastVisibleCol = ActiveWindow.VisibleRange.Columns(ActiveWindow.VisibleRange.Columns.Count).Column
    Wend

Now let me explain what this does:

It first sets prefLastVisibleCol to the number of the column that you prefer to be the last one visible.

Then it stores the number of the column that is currently the last one visible in currLastVisibleCol.

Then it takes the current scrollcolumn, increases it by 1 (I'll explain why a little later) and stores it in lastActualScrollCol .

Then it starts a while loop that runs as long as

currLastVisibleCol is smaller of equal to prefLastVisibleCol. This is to make sure that it scrolls right until the column right of the desired column is visible

AND

lastActualScrollCol is NOT equal to the current Scroll Column that is retrieved through ActiveWindow.ScrollColumn. The lastActualScrollCol is updated every loop, just before it executes the smallscroll. So if the smallscroll succeeded, the lastActualScrollCol value should not be equal to the actual current scroll column. If it is, that is probably because smallscroll somehow didn't succeed. In that case, it should stop the loop, because if it doesn't, it will just keep looping until excel stops the process and that's not what we want. That's why we need to add this to the while loop condition.

In this while loop:

lastActualScrollCol is set to ActiveWindow.ScrollCol

'ActiveWindow.SmallScroll 0, 0, 1, 0' is executed to scroll 1 to the right.

currLastVisibleCol is set to the last visible column by doing:

currLastVisibleCol = ActiveWindow.VisibleRange.Columns(ActiveWindow.VisibleRange.Columns.Count).Column

After that, another while loop is started that runs as long as

currLastVisibleCol is greater than prefLastVisibleCol. This is to make sure that it scrolls left until the column right of the desired column is not visible anymore

AND

lastActualScrollCol is NOT equal to the current Scroll Column that is retrieved through ActiveWindow.ScrollColumn as explained before.

And that's actually everything there is to it.

It works with any window size, zoom settings and column width.

I hope this was helpful. I've been searching for a solution for hours and I don't want anyone else to have to do the same.

charles kenyon say:
I had believed you could put other form fields inside the table inside the IF Field but was mistaken.

The underlying problem, of which I was unaware, is that if a Form Field is inside an IF field it is not accessible as a form field. This is true even when the document is unprotected. Only when the field codes are displayed does Word see it as a Form Field. Then Word treats it as a Form Field. You can double-click on it to get the properties.

In one of my forms I use on-exit macros that make changes in the document including enabling fill-in on other text form fields (that are otherwise invisible to the user when the document is protected). These could probably insert a table with form fields using AutoText to hold the table. Such a macro needs to unprotect the form, make the change, reprotect the form, and go to the appropriate field.

This is more programming than I would want to undertake without a powerful reason, though.

Should you choose to undertake it, though, here is the code to protect and unprotect a password protected form. You would call these procedures from your on-exit macro.

Sub UnProtectForm()
    Dim sPass as String
    With ActiveDocument
        sPass = "MyPassword"
        If .ProtectionType <> wdNoProtection Then .Unprotect (sPass)
        ' End If
    End With
End Sub

Sub ProtectForm()
     Dim sPass as String
     With ActiveDocument
        sPass = "MyPassword"
        If .ProtectionType = wdNoProtection Then .Protect Type:=wdAllowOnlyFormFields, NoReset:=True, Password:=sPass
            ' End If
    End With
End Sub


greg maxey say:

Since Charles has brought you this far, we might as well go the full course ;-)


1. Define TableA with your appropriate formfields, select it, press ALT+F3 and save it as an AutoText entry named TableA. 


2. Define TableB with your appropriate formfields, select it, press ALT+F3 and save it as an AutoText entry named TableB.


3. Delete tables A and B from the document.


4. Add a bookmark where table A or B should appear.  Name it bmTableRange.


5.  Add the following code to a standard code module in the document.  If required, see: http://gregmaxey.mvps.org/word_tip_pages/installing_employing_macros.html for instructions to employ the VBA code provided:


Sub InsertTable()
Dim oRng As Word.Range
With ActiveDocument
  If .ProtectionType = wdAllowOnlyFormFields Then
    .Unprotect
    Set oRng = ActiveDocument.Bookmarks("bmTableRange").Range
    If oRng.Tables.Count = 1 Then oRng.Tables(1).Delete
    If .FormFields("Dropdown3").Result = "A" Then
      Set oRng = .AttachedTemplate.AutoTextEntries("TableA").Insert(Where:=oRng, RichText:="True")
    ElseIf .FormFields("Dropdown1").Result = "B" Then
      Set oRng = .AttachedTemplate.AutoTextEntries("TableB").Insert(Where:=oRng, RichText:="True")
    End If
    .Bookmarks.Add "bmTableRange", oRng
    .Protect wdAllowOnlyFormFields, True
  End If
End With

End Sub


6. Set an InsertTable to run OnExit from your Dropdown3 field.  You must have another valid field following dropdown3 to tab or click into so the exit event fires.


7.  Lock the form and toggle between "A" and "B" and exit the field.

How to disable Exchange email, Skype for Business, One Drive and Delve for Tenant level?

Hi,

I am trying to disable the Exchange email, Skype for Business, One Drive, Delve and other office 365 features are at tenant level. When i search online to disable them i don't see any options for tenant level.  Let me know if there is any way to disable them at tenant level or do they need to be done at user level only?

william zeng msft say:

Hi SharePoint User 426,

  1. For Exchange email, Skype for Business and Sway, I suggest you assign/remove Exchange Online license, Skype for Business and Sway licenses from users.
  2. For Office Store, you can go to Admin center > Service & add-ins > Office Store to turn ON/OFF it.
  3. For OneDrive, you can go to the SharePoint admin center and select user profiles, click Manage User Permission under People. Then, Add Everyone except external users and uncheck Create Personal Site (required for personal storage, newsfeed, and followed content) and click OK to save the change.

        

      4. For Delve, you can go to the SharePoint admin center and select Settings, under Office graph, select Don’t allow access to the Office graph to disable it for users.


In addition, you can also use PowerShell to disable these kind of services. Please refer to the following articles:

Disable access to services with Office 365 PowerShell

Office Delve for Office 365 admins

Regards,

William

william zeng msft say:

Hi SharePoint User 426,

You are right. OneDrive can’t be disable if it has been provisioned for users while Delve can be disable for existing users.

If you don’t allow access to the Office Graph, Delve will be removed from the Office 365 app launcher. When users visit a person’s page, for example by clicking on a person in OneDrive for Business, that person’s page will contain only user profile information. No documents will be shown. Users can still search for other people, but not for documents or boards.

William

sylvie liang msft say:

Hello Erica,

Thanks for your reply. I am monitoring this thread by now. Based on my knowledge, when you make someone else as your delegate, the user is just able to access your Inbox, calendar, Tasks, Contacts, Notes. Other folders (such as Inbox sub-folders) are not available.

If you want to lock calendar, contacts folder, etc., you can change the delegate permission to None via File > Account Settings > Delegate Access > Permissions. For your reference: Change permissions for a delegate

If you are referring to some other folders, you don’t need to lock it as delegates cannot access them by default.
