jeeped say:
The recommended best configuration for your M2N68-AM or M2N68-SE motherboard is a matched pair of DDR2 PC2-8500 (e.g. DDR2-1066) 2Gb ram modules. There will be a significant improvement with a matched pair rather than a single ram module. At a little over 80 bucks for 4Gb (e.g. 2 × 2Gb), it doesn't make much sense to buy an individual module and fill the second slot later. Another option is to use the lower DDR2-800 modules at a little over 60 bucks for a matched pair (e.g. 2 × 2Gb).

For your memory compatibility and upgrade questions, you cannot get better information than the Crucial Memory Advisor. I always start there when considering a memory upgrade. Check out the Memory Advisor reports for the M2N68-AM motherboard here and for the M2N68-SE motherboard here.

Hi,

 

You might not have changed the display language to Chinese and so only you are facing the issue. When you change the display language, the text in menus and dialog boxes for some programs might not be in the language that you want. This happens because the program might not support Unicode.

 

Follow these steps to change the display language.

a.      Press Windows key + X and select Control Panel.

b.      Click Clock, Language, and Region, and then click on Region and Language.

c.       Click on Keyboards and Languages tab.

d.      Under Display language, choose a language from the list, and then click OK.

 

The system locale determines the default character set (letters, symbols, and numbers) and font used to enter and display information in programs that don't use Unicode. This allows non-Unicode programs to run on your computer using the specified language. You might need to change the default system locale when you install additional display languages on your computer. Selecting a different language for the system locale doesn't affect the language in menus and dialog boxes for Windows or other programs that do use Unicode

 

Follow these steps to change the system locale.

a.      Press Windows key + X and select Control Panel.

b.      Click Clock, Language, and Region, and then click on Region and Language.

c.       Click the Administrative tab, and then, under Language for non-Unicode programs, click Change system locale. If you're prompted for an administrator password or confirmation, type the password or provide confirmation.

d.      Select the language, and then click OK.

e.      Now restart the computer.

¡firedog say:

There is no quick fix.  Create a new folder and copy messages into it.  Wait and then create another folder.


If the folders that used to be directly under Storage folders and are now in Recovered items are intact, you can drag the folder back to its original location. You don't have to create folders and move messages so long as you're operating exclusively under Storage folders.

You can't drag folders out of Storage folders and in under an account. This is when you'd have to create new folders and move messages.

Items are recovered when the database discovers some corruption in its records. They will often find their own way back where they belong at the next program shutdown/start; only if they can't will they stay put under Recovered items. Since yours date from a couple of months ago, the messages involved obviously didn't make it home.
