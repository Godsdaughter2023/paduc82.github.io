Adding area code to a list of phone numbers in MS Access - Microsoft Community

luke.m55 say:

Adding area code to a list of phone numbers in MS Access

is there a way to add an area code to list of phone numbers already in my access database. the area code will be the same for every single number and they are formatted like 555-5555. tried find and replace but that doesnt use regex, apart from manually pasting it in i'm at a loss. if i cant add the area code to the table then i at least would like it to show up in the query/report. thanks
dirk goldgar say:

Your mention of formatting raises the concern that maybe the Format property is set, and might be obscuring the actual text content of the field.  But in principle, you can use an update query to prepend whatever you want to each field.  So for example, if the PhoneNumber field is a text field and each field's content is similar to "555-5555", you can make them all like "(909) 555-5555" by executing an update query like this:

    UPDATE MyTable SET PhoneNumber = "(909) " & PhoneNumber
    WHERE PhoneNumber Is Not Null

You could make them like "909-555-5555" by executing an update query like this:

    UPDATE MyTable SET PhoneNumber = "909-" & PhoneNumber
    WHERE PhoneNumber Is Not Null

If your PhoneNumber field is actually -- unwisely -- a number field, just formatted to look like text, then you would have to do something like this:

    UPDATE MyTable SET PhoneNumber = 9090000000 + PhoneNumber
    WHERE PhoneNumber Is Not Null

Note, though, that that could only work if the field is of a number type that can accept values that big.

Of course, you should make a backup copy of your database before executing mass-update queries like these.

To recap what you are going to have with an IMAP account in terms of data files using Outlook '2010 or earlier

#1 - One PST file which will contain the email folders as found on the server. 

#2 - Second PST file which will be configured as the default data file containing the default contact/calendar folders. This data file will have a full folder set but the only significant folders are your contact/calendar/task folders

You cannot remove the "default data file" from an Outlook profile.

#3 - The best way to reset the PST file containing your email folders would be to rename the PST file (with Outlook closed). If Outlook cannot find the file defined in the Outlook profile, it will create a new one and resync the data from the server. The reason that I suggest renaming vs deleting the email PST file is that it will allow you to revert back to it if there is any reason to do so. Once Outlook has re-sync'd with the server and everything is the way you want, you can delete the original email PST file (the one that was renamed)
