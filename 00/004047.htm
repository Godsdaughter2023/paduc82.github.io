Excel rounding problem

I am entering a numbers in a spreadsheet - 20 columns by 100 lines.  Cells are formatted to currency, 2 decimal places.  At the end of each line I have a sum for that line, followed by the running total, which is the running total from the previous line + the sum from the current line.  At one point the previous running total is 1178.07, the sum for the current line is 40.90 and the new calculated running total is 1218.96.  Wrong!  Why is this happening?  I tried the same thing on Excel 2003 and had no problem.

jeeped say:
You are displaying each value as Currency, 2 decimals but the underlying raw value may have a trace decimal above or below the displayed amount. Using ROUND(SUM(...),2) for each sum line should rectify your end row totals but each line may be ±0.01 due to the values within the row still having trace remainders. Try the following for each row,

=SUMPRODUCT(ROUND(A1:Z1,2))

This will ROUND() each value to 2 decimals from A1 to Z1 before adding them together.

rick rothstein, mvp - excel say:

thank you all...I am learning things

now that I have a lot of attention:

is there any reason not to use:

Arr = Range(Cells(1, 1), Cells(1, 1).End(xlDown))

instead of:

Arr = Range(Cells(1, 1), Cells(Rows.Count, 1).End(xlUp))

since I see the xlUp being used far more often than the xlDown??

With xlDown, you will be starting on a cell filled with data, so it will stop at the non-filled (that is, empty) cell it comes to in the column even if there are filled data cells after that empty cell, so you will not reach the end of your data using it if your column could have empty cells scattered about. On the other hand, xlUp, because it comes up from the bottom, starts in an empty cell and ignores all empty cells above it until it finds a non-empty cell and then it stops there. Just so you know, the End(xl...) property finds the same cell that pressing <EndKey><ArrowKey> on the keyboard would find.
NOTE: Please mark the message or messages (yes, you can mark more than one) that answer your question as the "Answer" so that others will know your question has been resolved.

hansv mvp say:

Cells(Rows.Count, 1).End(xlUp) is the last non-blank cell in column A.

Cells(1, 1).End(xlDown) moves down from cell A1 and stops before the first blank cell.

If there are no blank cells in between the filled cells in column A, the result is exactly the same, but if there are, Cells(1, 1).End(xlDown) will not return the really last non-blank cell.

Example:

A1: Name

A2: John

A3: Mary

A4: Dave <--- Cells(1, 1).End(xlDown)

A5:

A6: Anne

A7: William <--- Cells(Rows.Count, 1).End(xlUp)
