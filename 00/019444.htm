daniel pineault say:

If you are setting the RowSource Type to Value List then is is no harder than

Me.YourListBoxName.AddItem "1" & ";" & " haha2"

Where you separate multiple column values by a semicolon (;)  This will add it to the listbox for the active session, but will not be kept in memory for future use.  For that you'd have to use a table/query as you source.

 

If you are, however, using a table/query as your source then you'd have to use code to add the new value to the underlying table and the requery the listbox.

hansv mvp say:

List boxes in Access are mainly intended to be used with a table or query as row source. To add/remove/edit list items, you manipulate the row source.

For list boxes with a list of values as row source, you can only add entire rows in one go: the AddItem method accepts a semi-colon delimited list of values:

Me.lstDemo.AddItem "23;Green;Ireland"

To refer to a specific item, you can use the Column property: Me.lstDemo(3, 2) retrieves the element in the 4th column, 3rd row (the column and row index are zero-based). This is a read-only property - you can't use it to set a value.

How do I publish a major version in SharePoint 2007 via my form created in InfoPath 2007?

  2. User updates the InfoPath form and clicks Submit. This is a custom Submit button. The Submit in InfoPath is disabled.
  3. Submit uploads the form to the library then sends an email.

Users can approve the form or reject it. Each update, including approval/rejection, is uploaded to the library.

Is it possible that when the user approves a form, this is uploaded to the library as a major version instead of a minor version? It is not an option to use the publish major versions only functionality in the library as we also need to track minor versions which are used for tracking the changes to the form before it is submitted for approval.

daisycao say:

Hello Redge,

 

The approval workflow works as following:

 

When you use a workflow to manage content approval in a library, the content approval workflow starts whenever a user checks in a document as a major version, or when a user selects the Publish a Major Version command for a document. Although the workflow is in progress, the document remains a minor version with a status of Pending.

 

If the document is approved through the content approval workflow, its status is updated to Approved, and the document is published as a major version, making it visible to anyone with permission to view documents in the library.

 

If the document is rejected through the content approval workflow, its status is updated to Rejected, and it remains a minor version that is visible only to the document author and to those users who have permission to manage lists.

 

It did uploaded as major version after the approval, if you still have problems, feel free to post the question on TechNet SharePoint forum: http://social.technet.microsoft.com/Forums/en-US/category/sharepoint2010,sharepoint
