rohn007 say:

In theory, Office 365 applications only need to connect to MS to validate the license/subscription once every 30-40 days.   But they do need to connect to MS servers for various "online" features. And of course things like email / outlook and OneDrive fundamentally do need an internet connection to MS to work.

It sounds like there is some sort of delay making the connection to MS servers. That is a "network" problem and is likely to be very difficult to fix yourself. The problem may be at MS, or at one or more "servers" between you and MS, that you simple do not have any control over.

One thing you can try is changing your "DNS Server".  This this the computer that translates web addresses like https://answer.microsoft.com into the number code (IP Address) that the internet uses to identify where a computer is. If your DNS service is slow to respond, or if it provides a bad routing to MS, that can have the effect you are seeing.  Here is an article describing how to change your DNS server.  Make sure to make note of your original setting before changing it.

https://www.lifewire.com/double-internet-speed-with-one-settings-change-2377750

Other than that, yes, it seems like you will have to wait until the connection is made before getting to work. Inconvenient, yes.

    From one old fart to another. You are right. That is not what you need :)

    You seem to have a combination of stuff there including the code to update fields which was used as an example on my web pagehttp://www.gmayor.com/installing_macro.htm

    Select all of that code that you have posted in your vba editor and replace it with the macro from the graphics web page which is

    Sub DuplicateLabels()
    Dim oTable As Table
    Dim oField As Field
    With ActiveDocument

         'Change the document type to mailing label
         .MailMerge.MainDocumentType = _
            wdMailingLabels

         'Propagate the labels
         WordBasic.MailMergePropagateLabel
         'Check each table in the document
         For Each oTable In .Tables

              'Check each field in each table
              For Each oField In .Fields

                   'If the field is a {NEXT} field,

                   If oField.Type = wdFieldNext Then

                        oField.Delete  'Delete it!
                   End If
              Next oField
         Next oTable

         'Revert the document type to a normal document
         .MailMerge.MainDocumentType = _
             wdNotAMergeDocument
    End With
    End Sub

    Then run the macro as previously instructed.
