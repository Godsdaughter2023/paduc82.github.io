Word 2010 spell check - Microsoft Community

a. user say:

Word 2010 spell check

I have compiled a 6000 word vocabulary list with pronunciations in brackets. This caused spell check to shut down because of "too many misspelled words." How can I make Word 2010 spell check ignore everything in brackets?
jay freedman say:

<Edit: Despite what may seem quite different approaches between Suzanne's answer and this one, they accomplish similar results, and either will work. The big difference is between applying a character style and applying direct formatting of the same property. The style approach is preferable for a document that will be updated frequently.>

You can mark all of the text in brackets as "Do not check spelling or grammar". To do that:

  • Open the Replace dialog.
  • Click the More button to expand the dialog.
  • Check the box for "Use wildcards".
  • In the Find What box, copy/paste this:  \[([!\]]@)\]  (Note that this assumes that "brackets" means the characters [ and ], not parentheses or curly braces.)
  • In the Replace box, copy/paste this: [\1]
  • While the cursor is still in the Replace With box, click the Format button at the bottom of the dialog, and click Language in the menu.
  • In the Find Language dialog, put a check mark in the box for "Do not check spelling or grammar". You can leave the "(no language)" item selected, or select a specific language. Click OK.

  • In the Replace dialog, click the Replace All button. When the replacement is complete, dismiss the message box that says how many replacements were made, and close the Replace dialog.
  • Save!!

For explanation of the Find What and Replace With expressions, see http://www.gmayor.com/replace_using_wildcards.htm.

1)      There will be letters in the Excel file that are not in the tblLetter table. The action needed is to import the letters and enter new letter in the status field

2)      There will be letters in the Excel file and in the tblLetter . The action needed don’t import the letter enter open letter in the status field

3)      There will be letters in the tblLetter and not in the Excel file. Action needed enter closed letter in the status field

bill mosca say:

It sounds like what you need to do is import the Excel file into a temporary table. The two tables should have a unique index on whatever constitutes a unique record.

Then create an update query with the 2 tables. The join should be an inner join (only include records where the joined fields are equal). Mark all recs in the Access table as open. 

Then create an append query with a left join (all records from the excel table and only matching records from the Acc table). Run the query to add the excel records and put "new" in the status field. Only the new ones should go in and you will get a warning that not all records were inserted due to a key violation.

Then create one more query to update as per your #3. In this case, the join will be a right join (include all recs from Access table and only matching ones from the Excel table. Update the status as closed where the excel record is null.

 

There might be a way to combine a couple of the above, but this way you can see what is happening at each step.

rick rothstein, mvp - excel say:

I am retired several years now, so I don't have access to a network system to test this, but the documentation I have read seems to indicate this code will work for local and network drives, but you will have to test that for yourself. The code makes use of a function that returns the drive letter or network drive designation. This is the function...

Function GetDriveLetter(Path As String) As String
  Dim FS As Object, Drv As Object
  Set FS = CreateObject("Scripting.FileSystemObject")
  Set Drv = FS.GetDrive(FS.GetDriveName(Path))
  GetDriveLetter = Drv.DriveLetter
End Function

and here is the code (it replaces my previously posted code) that does all the work, calling the above function as needed...

Dim OldDir As String, OldDrive As String, DefaultPath As String
DefaultPath = "D:\Original Files\Documentation\ENG"
OldDrive = GetDriveLetter(CurDir)
OldDir = CurDir
ChDrive GetDriveLetter(DefaultPath)
ChDir DefaultPath
Application.GetOpenFilename
ChDrive OldDrive
ChDir OldDir

a. user say:

Hi ,

Thank you for posting your query on Microsoft Community.

Please be informed that by upgrading to Outlook.com, you are only upgrading the user interface of Hotmail to the new user interface hence all your emails, contacts, etc. should be migrated.

Here are the possible reasons why emails are missing from your account:

 1. Emails got deleted due to Inactivity.

When users don't sign in for more than 365 days or within the first 10 days after signing up for an account, the system will flag it as Inactive. After an account becomes inactive, all messages, folders, and contacts are deleted. Incoming messages will be sent back to the sender as undeliverable.

Also, if you were accessing your emails on a mail client such as Windows Live Mail or through your mobile device, but didn’t log into your web account for over 365 days, you may lose emails due to inactivity in web account.

 2. Account has been hacked or compromised.

You may lose emails if your account was hacked and the hacker had already deleted emails in your account.

 3. Conversation threading is enabled.

If the feature Conversation threading is turned on wherein all emails with the same subject line are grouped together. You can click

Please be aware that there is no guarantee that the messages that you are looking for will all be recovered. You can refer to this

Note: The system can recover messages but not the folder itself. Recovered messages will be presented in your Deleted folder. Also, emails will not be recovered via recovery link if they were deleted from email clients like Windows Live Mail, Microsoft Office Outlook, etc.

Hope this information was helpful and do let us know if you need further assistance. We will be glad to assist.
