HUB's, Cooling fans, External Generic Backup device'(s) are only a few reasons this message persists.  In some cases, the message will pull you back to the Desktop interrupting whatever operation was in progress.  A simple patch can be made but not installed by the PC's operator.  The instructions for the patch should be to ignore usb items that do not normally answer the constant system check or at least post or return a no message & sound event.  NOTE: This problem can be seen in the "show hidden icons", in the systems tray, but cannot be corrected in the properties (depending on your Windows version), by selecting it and changing the notification properties. It will reset itself and just keep notifying you because the current Windows script does not have a routine to ignore these usb items. Or at lest, a reliable patch to enable  or disable this.

dondurst say: The problem arises because some MS modules embedded in the QB software have changed with the Windows 10 Anniversary Update and Intuit does not update versions older than 2014. Q.B. is "web enabled" so that the Quick Books home office senses a problem when the program starts and immediately shuts it down. Q.B will run as it did before if you disconnect from the internet or just block communication with Intuit by making a new rule in your firewall (Windows firewall or a anti-malware program's firewall if you're using one). Intuit of course will urge you to upgrade to the 2017 version but we found that doesn't seem to be necessary.
