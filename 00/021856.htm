Actual Durations are Displaying as Fractional - Microsoft Community

suec421 say:

Actual Durations are Displaying as Fractional

I have a project plan in MS Project 2007.  I have a baseline duration, start and finish dates.  I also have actual start, finish and duration dates.  When I enter my actual start and finish dates it calculates the actual duration.  However in the Level 2 summary task it is showing a fractional number.

For example I have tasks with actual durations of 17, 12, 10, 5 and 1 day  and the Level 2 summary task is showing me 34.23.  

The corresponding baseline durations are 17, 12, 1, 14 and 0 days. 

 

I've looked through my plan and cannot find any dates that would cause the fractional duration I'm seeing.  How can I correct this problem with my actual durations. 

 

Thank you,

Sue

john - project say:
SueC421,
Level 2 summary means nothing but I will assume you are looking at the summary line data for the tasks you have listed. When looking at actual duration for a summary line you have to use a formula to calculate its value.

You must have some lag or other link relationships among the performance tasks you show in your example, but that's not important to illustrate the formula. I entered your tasks with the durations you show. I will assume for the sake of simplicity that your plan dates and baseline dates are identical, meaning that your baseline duration values are the same as your duration values. We need the Duration field values to use the formula. With no links between tasks (i.e. they all start on the same day), and given the durations you show along with the actual duration values you show, the summary line duration is 17 days and the actual duration will be 14.17days. Your question is, how is that latter value derived.

For a summary line the % complete is defined by the formula:
summary % complete = (sum of subtask actual durations)/(sum of subtask durations) * 100%

The actual duration for a summary line is then calculated as:
summary actual duration = summary duration * summary % complete

If you apply the above formulas to your project, you should get a summary actual duration of 34.23 days.

This is the simple case. If any of your tasks are split or any of your resources assigned to the tasks have different calendars, then the formulas get much more complex.

Hope this helps.

John

This problem was solved as follows (with help from the folks at CrashPlan.com):

Turns out that a Windows 7 Environment Variable for Java Applets was set to a maximum size that was too small, so the Java Testing applet in the browser (as well as other Java based programs on my PC, including CrashPlan) was not getting any memory to run in. This Environment Variable is usually not set system-wide and I don't know which of my programs put it in.

Also, this Environment Variable is not visible from the Java Control Panel.

To check if there's a limit set on the amount and size of Java applets find the Environment Variables, as follows:

  1. From the Desktop, right-click the Computer icon and select Properties. If you don't have a Computer icon on your desktop, click the Start button, right-click the Computer option in the Start menu, and select Properties.
  2. Click the Advanced System Settings link in the left column.
  3. In the System Properties window, click on the Advanced tab, then click the Environment Variables
