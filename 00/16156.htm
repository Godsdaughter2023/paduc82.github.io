starkeeper say:

To have a header style show up in the navigation pane, you need to make sure that the style is marked as "Outline Level 1." This is because the navigation pane uses the outline levels to mark content.

In Word 2010:

  1. Open up the "Styles" menu so that it on the right side of your window. 
  2. Find the heading style that you are using for your headers and right click on it to open up the options menu.  Then select "Modify."  This opens a "Modify Style" menu box.
  3. Click on the "Format" button in the lower left corner.
  4. Select "Paragraph."
  5. In the "Outline level" selection list, make sure that "Level 1" is selected. (Or whichever level you want this header to be. If the selection box says "Body Text," then nothing will appear in the Navigation Pane.)
  6. Select "OK"

You should now see the headers of this style in the selection pane.  Any subheads based off of this header should now also be showing up in the correct nesting configuration.

stefan blom say:

It is a known limitation in Word that headings inside table cells won't appear in the Navigation pane. For what it's worth, the same thing applies to the predecessor of the Navigation pane, the so-called Document Map, in older versions of Word.

As you have noticed, Word properly adds headings inside a table to the TOC, fortunately.

Sub ListAllCF()

When you create the code to delete the CF you need to edit the returned formula to include double quotes around the entire formula (including the leading equals sign) and then the double quotes within the formula must be 2 double quotes as per the example code.

Also the formula comparison is case sensitive so  stuff like "rows" must be in the same case as the formula used in the CF. In the example I have alleviated the problem from this with Ucase on both .Formula1 and the actual formula string so it is comparing upper case to upper case.

Sub DeleteConditFormat()
    Dim cf As FormatCondition
    Dim ws As Worksheet
   
    For Each ws In Worksheets
        For Each cf In ws.Cells.FormatConditions
            With cf
                If .Type = 2 And UCase(.Formula1) = UCase("=OR(CELL(""row"")=CELL(""row"",A1))") And .Interior.Color = 9359529 Then
                    .Delete
                End If
            End With
        Next cf
    Next ws
End Sub

Further to the above, you should include a test for the CF already existing before allowing the code to apply it again otherwise you will have multiple CF's with the same condition. See my modified code below.

Private Sub Workbook_SheetBeforeDoubleClick(ByVal Sh As Object, ByVal Target As Range, Cancel As Boolean)
   
    Dim cf As FormatCondition
   
    'The following determins if the CF is already applied so that multiple CF's not applied
    For Each cf In Sh.Cells.FormatConditions
            With cf
                If .Type = 2 And UCase(.Formula1) = UCase("=OR(CELL(""row"")=CELL(""row"",A1))") And .Interior.Color = 9359529 Then
                    Exit Sub        'CF already exists so exit the sub don't create another
                End If
            End With
    Next cf
   
    Cells.Select 'Selects entire worksheet
   
     'Highlight ROW
     Selection.FormatConditions.Add Type:=xlExpression, Formula1:= _
         "=OR(CELL(""row"")=CELL(""row"",A1))"
     Selection.FormatConditions(Selection.FormatConditions.Count).SetFirstPriority
     With Selection.FormatConditions(1).Interior
         .PatternColorIndex = xlAutomatic
         .ThemeColor = xlThemeColorAccent6
         .TintAndShade = 0.399945066682943
     End With
     Selection.FormatConditions(1).StopIfTrue = False
        
   End Sub

Private Sub Workbook_SheetSelectionChange(ByVal Sh As Object, ByVal Target As Range)
    Sh.Calculate
End Sub
