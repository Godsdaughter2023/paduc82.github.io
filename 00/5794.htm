Hi Sunny,

 

Thank you for posting your query in Microsoft Community.

 

"Windows Defender", anti-spyware tool before Microsoft Security Essentials, Microsoft acquired a third-party anti-spyware product in 2004, first renaming it "Microsoft AntiSpyware", and later "Windows Defender": this wasn't an anti-virus, but an anti-spyware solution.

 

It was also available as a free download for Windows XP, and shipped included into both Windows Vista and Windows 7.

 

In 2009, Microsoft released MSE ("Microsoft Security Essentials"), which was an anti-virus software, whose functionality crossed over Windows Defender: this is why Windows Defender is disabled once you install MSE in Windows 7 and earlier versions.

 

MSE isn't bundled with any version of Windows, but it's available as a free download from Windows XP through 7.

 

Windows Defender running in Windows 8 / 8.1 / 10 For the first time, Microsoft shipped an anti-virus package built into the operating system with Windows 8: it is called "Windows Defender", and does basically the same thing as MSE.

 

 You won't be lost using Windows 8 / 8.1 / 10 if you are familiar with MSE.

 

So, to keep things clear, Windows Defender for Windows 8 and later is mostly the same thing as Microsoft Security Essentials for Windows 7 and earlier. We suggest to just turn on Windows Defender because it is the updated one and same thing with MSE.

john w. vinson say:

This kind of "intelligent key" is, as Bill correctly says, not an ideal design. I'd (strongly) suggest using an Autonumber field in this table as a primary key, and changing all the related tables to use a Long Integer foreign key linked to it. You can keep the three fields in the work order table, using a Subform with UniqueIdentifer as the master/child link to populate that field, and code in the form's BeforeInsert event to populate the other two fields:

 

Private Sub Form_BeforeInsert(Cancel as Integer)

Me!FiscalYear = <some expression to calculate the fiscal year from today's date>

Me!WorkOrderNumber = NZ(DMax("[WorkOrderNumber]", "WorkOrders", _

  "UniqueIdentifier = " & Me!UniqueIdentifier & " AND FiscalYear = " & Me!FiscalYear)) + 1

End Sub

 

If you insist, you can then concatenate the three fields into one text string, but it's neither necessary nor is it good design - you can always just concatenate them in the control source of a textbox for display and use the hidden autonumber for linking.

A GUID contains some very specific components, such as the MAC address of the computer on which it's generated. A true GUID is stored in a specially formatted memo field, if I understand aright. I'm not experienced with their use.
