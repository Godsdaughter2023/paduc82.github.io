Hi Myles,

I realize the inconvenience you are experiencing with Headphone on Windows 10. I will certainly help you.

This issue might occur if the audio drivers are corrupted or due to incorrect audio settings. I suggest you to try the following troubleshooting steps and check if it resolves the issue.

Step 1: Uninstall Audio drivers

Uninstall your audio driver: open Device Manager, right-click (or tap and hold) the audio driver, and select Uninstall. Restart your device and Windows will attempt to reinstall the driver.

And if that didn't work, try using the generic audio driver that comes with Windows. Here's how:

Open Device Manager, right-click (or tap and hold) your audio driver > Update driver software... > Browse my computer for driver software > Let me pick from a list of device drivers on my computer, select High Definition Audio Device, select Next, and follow the instructions to install it.

Step 2: Set default device

If you're connecting to an audio device using USB or HDMI, you might need to set that device as the default.

Open Start, enter Sound, select it from the list of results, and then, on the Playback tab, select a device, then Set Default.

Step 3: Turn off Audio Enhancements

Follow these steps:

1. Go to Start, enter Sound, and select Sound Control Panel from the list of results.

2. On the Playback tab, right-click (or tap and hold) the Default Device and select Properties.

3. On the Enhancements tab, select the Disable all enhancements check box and try to play your audio device.

4. If that doesn't work, select Cancel and, on the Playback tab, select another default device (if you have one), select

    the Disable all enhancements check box, and try to play audio again. Do this for each default device.

Step 4: Try different audio formats

1. Go to Start, enter Sound, and select Sound Control Panel from the list of results.

2. On the Playback tab, right-click (or tap and hold) the Default Device, and then select Properties.

3. On the Advanced tab, under Default Format, change the setting, and then retest your audio device. If that

    doesn't work, try changing the setting again.

  4. Click Network Connections.

    When Internet Connection Sharing is enabled, your LAN adapter will be set to use IP

    your network. If these other computers have static IP addresses, it is a good idea to set them
    to obtain their IP addresses automatically. Are you sure you want to enable Internet
    Connection Sharing?
  11. Click Yes.

lemp say:

I'm glad that things are working now.  You're not in as bad shape as I had feared.

In Windows 7, the built-in account named "Administrator" is hidden and disabled by default.  It can be turned on ... but you shouldn't do that unless you have no other options.

Although the security experts have long recommended that you shouldn't use an account with "administrator" rights for daily use, in fact most people ignore this recommendation.  For that reason, in Windows 7, even accounts with administrator rights don't normally run with those rights active.  If you're using a "Standard user" account, when you need administrative rights, Windows will prompt you for the name and password of an "administrator" account; if you're already using an administrator account, Windows will just prompt you to see if you really want to do the action.  See http://windows.microsoft.com/en-us/windows/what-is-user-account-control#1TC=windows-7

It's a good idea to create a "spare" account with administrator rights.  That way, if the account you use daily becomes corrupted -- and this does occasionally happen -- you can log in to your spare administrator account and fix things.  See the "workgroup" instructions here --> http://windows.microsoft.com/en-us/windows/create-user-account#create-user-account=windows-7

The Guest account is even more limited than a standard user account.  Microsoft says, "People using the guest account can't install software or hardware, change settings, or create a password. Because the guest account allows a user to log on to a network, browse the Internet, and shut down the computer, you should disable the guest account when it isn't being used."  Following this advice, the Guest account is disabled by default in Windows 7 (although unlike "Administrator" it's not hidden as well).

The ASP.Net account was more commonly found on XP machines and is related to Microsoft's .NET Framework version 1.1.  If you upgraded from XP to Vista to 7 (or perhaps from Vista to 7) the account may have been brought along during the upgrade.  Otherwise, you've probably installed some very old program that was built using .NET Framework 1.1.  Take a look at Control Panel > Programs and see if you have installed Microsoft .NET Framework 1.1:

The various .NET Framework versions are installed when various third-party applications that are built using these frameworks are installed.  The ASP.Net Machine Account is harmless, but you can remove it with no effects from Control Panel > User Accounts, even if you do have .NET Framework 1.1 (if you don't have that version, you certainly can remove the account).  See http://forums.asp.net/t/758994.aspx?Why+is+there+an+ASP+NET+account+on+my+machine
