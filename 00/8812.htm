MailNickname not updating in Exchange Online for synced users - Microsoft Community

alexandru cazacu say:

MailNickname not updating in Exchange Online for synced users

Although this topic has been discussed in the past under different circumstances, I thought that I should give it a go and publish my version of a solution to this type of scenario, where in a hybrid deployment, your Office 365 synced user mailboxes/mail users, have their aliases (mailNickname attributes) different than what you see in your local Active Directory/on premises Exchange.

 

How would one try to fix this issue, without contacting the Microsoft Support engineers, you might ask. Well, the solution is quite simple and logical.

 

To give you a short description on how Exchange Online works, you have the Azure AD platform which is the backbone of Office 365, while each workload in an Office 365 tenant (SharePoint Online, Exchange Online, Skype for Business Online, Teams) has its own Active Directory instance. 

 

So, in Office 365, the infrastructure is quite similar to a hybrid environment, in the way that your on premises AAD Connect server, will sync the local users to Azure AD, and then a cloud sync process will go ahead and sync the objects further on, in the Active Directory instance that runs in the background of the Exchange Online workload.

 

In our scenario, the issue appears in the 2nd stage, the cloud sync process between the Azure AD platform and the EXO AD instance. Given that we now know where the issue lies, it's time to get down to business:

 

1. Requirements:

 

NOTE:  Make sure that all users have the mailNickName attribute populated in the local Active Directory; mailNickName is an Exchange property and it doesn't exist by default in Active Directory, so if you never had a local Exchange installed, the mailNickName attribute doesn't exist on the user's properties. The AD Schema must be extended with the Exchange schema, more details here: https://docs.microsoft.com/en-us/Exchange/plan-and-deploy/prepare-ad-and-domains?view=exchserver-2019

 

- make sure that you have a Windows 10 machine ready

- open Powershell with right click and Run As Administrator 

- run Set-ExecutionPolicy unrestricted

- type Find-Module AzureAD | Install-Module, choose Y to install it

- type Find-Module Msonline | Install-Module, choose Y to install it

 

To have access to Find-Module and Install-Module commands in Powershell on Windows server 2012/2012R2/2016, make sure you go here https://www.microsoft.com/en-us/download/details.aspx?id=54616 and install the appropriate update according to what operating system you have. Windows 10 has these Powershell commands by default.

 

 

2. Open 2 Powershell windows and type the following:

 

- in the first PS window, Import-Module Msonline and then Connect-MsolService (provide your Office 365 tenant admin credentials)

- in the second PS window, Import-Module AzureAD and then Connect-AzureAD (provide your Office 365 tenant admin credentials)

- from now on, we are going to refer to the first PS window as to MSO and to the second one as AzureAD

 

 

In the AzureAD window, follow the steps in this article https://blogs.technet.microsoft.com/dpickett/2017/06/04/alias-in-the-cloud-not-synced-with-on-premises/ to confirm that the mailNickName is correctly synced from the local AD to Azure AD. Use a command like Get-AzureADUser -SearchString <UPN of the user> | fl mailNickName or Get-AzureADUser -SearchString <UPN of the user> | fl.

 

In the MSO window, for a user that has the incorrect alias in Exchange Online but the correct one in Azure AD, type the following commands:

 

Get-MsolUser -UserPrincipalName <UPN of the user> | fl ObjectID

 

Redo-MsolProvisionUser -ObjectID <guid from previous command>

 

Now, the Alias should be correct in Exchange Online.

 

If you want to re-sync all the users in MSO window: 

 

$users = Get-MsolUser -All | select ObjectID

 

$users | % {Redo-MsolProvisionUser -ObjectId $_.ObjectID}

 

Useful articles:

 

https://blogs.technet.microsoft.com/o365support/2016/01/18/user-re-provisioning/

https://docs.microsoft.com/en-us/powershell/module/msonline/redo-msolprovisionuser?view=azureadps-1.0

https://docs.microsoft.com/en-us/Exchange/plan-and-deploy/prepare-ad-and-domains?view=exchserver-2019

https://www.microsoft.com/en-us/download/details.aspx?id=54616

https://blogs.technet.microsoft.com/dpickett/2017/06/04/alias-in-the-cloud-not-synced-with-on-premises/

https://support.microsoft.com/en-us/help/2824766/mailnickname-or-alias-attribute-in-exchange-online-doesn-t-match-what

PS: Just to clarify something, MSO and AzureAD are one and the same thing, but I am referring to them as 2 separate identities, just because of the way we can interact with the platform, by using 2  different Powershell modules and different sets of commands.

A4:
=MATCH(TRUE,INDEX($A$2:$BD$2>0,0),0)

B4:

=TEXT(SUMPRODUCT(INDEX($A$2:$BD$2,A4):INDEX($A$2:$BD$2,B4),

A5:

B5:
=MATCH(TRUE,INDEX(INDEX($A$2:$BD$2,A5):$BD$2=0,0),0)+A5-2

C5:
=TEXT(SUMPRODUCT(INDEX($A$2:$BD$2,A5):INDEX($A$2:$BD$2,B5),
10^(B5-A5+1-COLUMN($A$1:INDEX($1:$1,B5-A5+1)))),"0")

Select A5:C5 and fill down as far as needed.

If the combinations could exceed 15 numerals in length, practicality requires using user-defined functions in VBA. Press [Alt]+[F11] to open the Visual Basic Editor, run the menu command Insert > Module to create a general module, and add the following code to it. Note: this has no error checking.

Function combo(rng As Range, n As Long) As String
  Dim k As Long, t As String * 1

  'concatenate all cells, replacing 0 with space
  For k = 1 To rng.Cells.Count
    t = rng.Cells(k).Value2
    combo = combo & IIf(t <> "0", t, " ")
  Next k

  'trim off leading spaces, if any, and add a space at the end for simplicity later
  combo = LTrim$(combo) & " "

  'remove combinations before the n_th combination
  For k = 1 To n - 1
    combo = LTrim$(Mid$(combo, InStr(1, combo, " ")))
  Next k

  'prune everything after the n_th combination
  combo = Left$(combo, InStr(1, combo, " ") - 1)

End Function

Then use the following formulas for results in D4 and the cells below it.

D4:
=combo($A$2:$BD$2,ROWS(D$4:D4))

Select D4 and fill down as needed.

I figured out how to get Outlook contacts to sync on the Pixel 3.

1) Go into the main settings -> Apps & Notifications ->Outlook

3) Open the Outlook app

4) In Outlook settings, under Account, click the name of the account for which you want to sync contacts.  It's probably in the form of an email address

6) Open the phone contacts app

7) Click the account icon in the top right of the Search bar

8) Pick the account for which you want to see contacts (your Outlook account)

This should get all your Outlook contacts synced!
