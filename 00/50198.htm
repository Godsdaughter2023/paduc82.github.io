When you purchase a Skype number, you are basically purchasing/leasing a land line. A landline unless packaged with an outbound calling subscription, will only receive inbound calls.  If you purchased one or multiple Skype numbers, that would only cover the cost of inbound calls to route them to your Skype account, voicemail or another Skype account. Forwarding calls to another phone number would still require on outbound calling subscription or Skype credit that covers the cost of the call (including connection fees) from your Skype account to that destination. Each forward is viewed equivalent to you making a standard outbound call in Skype. 

I personally have a US Skype number and an unlimited US calling subscription that I pay for yearly. Purchasing/renewing any calling subscription first allowed me to obtain a discount for my Skype number yearly purchase. In the opposite order I would not receive the discount. I have been paying for both subscriptions for over a decade. If I forwarded calls to another US number, I would not be charged anything additional; not even a connection fee. If I forwarded all inbound calls to an international location and I did not have an unlimited or minute subscription that would cover the destination and duration of all the forwards made, I would be charged pay as you go rates with a connection fee per forward.

lead3 say:

Tripplite removed the light bulb when it created a version for Vista and it is still missing from the newest version.

If you download the older version and you can get pastatus.exe from that version. This is the program that puts the lightbulb in the notifications area.

I copied it from my XP install.

This file must be copied into  C:\Program Files\Tripplite\Console.

If you are running Windows 7 x64 it must be copied to C:\Program Files (x86)\Tripplite\Console

Then you must create a Task in the Task Scheduler that runs pastatus.exe at startup.

The task must be run at logon with highest privileges.

Also, if you are running Windows x64, you must load the 32 bit JAVA as well as the 64 bit for it to work. You should really do this anyway.

You will then have the lightbulb. Obviously this is not supported by Tripplite.

I have had bad luck with the latest version of Poweralert.

 I use version 12-04-045-1
