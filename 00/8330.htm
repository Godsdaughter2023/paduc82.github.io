karl timmermans [outlook mvp 2012-2018] say:

where would the contacts file be by default? when this started I lost my calendar and contacts.

By "default" - the default contact folder is located in the data store designated as the default in the Outlook profile

File --> Account Settings --> Data Files tab

It's the data stores that are actually contained in the Outlook Nav Pane - not email accounts. Just mentioning that since many people mistakenly believe that the Outlook Nav pane only represents email accounts. Every email account will be associated with one data store according to the following rules

1) POP accounts - can use a single PST file or be created so that each have their own

2) IMAP - configured to use an .ost file. If this is the first account created, there will only be one data file. If this is not the first account, there would be another data file designated as the default for the profile, in which case, the IMAP folder tree would not have any folders marked with "This Computer Only" (i.e. contacts/calendar).

If <contact/calendar> info was contained in folders identified with "This Computer Only" and the email account got reconfigured, the data in those folders would be lost.

3) Exchange ActiveSync/Exchange - use a single OST file complete with all folders all the time regardless of which data file may already have been designated as the "default data file" for the profile

Hi Adam,

 

The reason your JPEG files are most likely not showing up is because they need a specific OLE Server on the machine in order to render them.  The program in the past that provided this OLE Server was Microsoft Photo Editor, which is no longer included with Office 2007.  The article below discusses this:

 

What to do when you see “Package” instead of “Bitmap Image” in your database tables - http://office.microsoft.com/en-us/access-help/what-to-do-when-you-see-package-instead-of-bitmap-image-in-your-database-tables-HA001151381.aspx

 

But you should be able to still see bitmaps assuming Microsoft Paint is still setup as the OLE Server for these on your machine.  As a test, what application opens to display your picture when you open one of your tables that contains these OLE Objects and you double click on one of your .bmp files?
