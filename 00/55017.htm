In this case, opening AirPort Utility will confirm that an update is available, and will give you the option to install it.
  • AirPort Utility 5 will display the firmware available and the option to update. AirPort Utility 6 will have a red indicator next to the device indicating a firmware update is available.
  • Even though you may not have AirPort Utility open, it will automatically do a weekly check to see if an update is available.

Case-sensitive errors with File History in Windows 8.1 - Microsoft Community

paul coddington say:

Case-sensitive errors with File History in Windows 8.1

There is a peculiar bug in the File History feature of Windows 8.1 (x64).

Certain folder names seem to be "special", and File History decides how they should be capitalized, ignoring all attempts to correct them.

So, when I restore data from a File History archive, even on a new machine with empty user data folders, certain folders are consistently renamed:

"Games" becomes "games"

"legacy" becomes "Legacy"

"resources" becomes "Resources"

"Scripts" becomes "scripts"
"templates" becomes "Templates"

There may be other names that are "magic" that I have not encountered. I am not sure if these are special words, or if File History gets stuck on the capitalisation it first encounters for a folder then applies it to all other folders with the same name.

The folders are renamed regardless of where they are in the file system, so:

"%UserProfile%\Documents\Folder 1\Scripts"

"%UserProfile%\Documents\Folder 2\Folder 3\Scripts"

"%UserProfile%\Downloads\Folder 4\Scripts"

...will all be renamed to "scripts".

Apart from the nuisance value of potentially having to check all folders for capitalisation errors after a restore, this problem is especially significant for source code projects where naming is always case-sensitive (although source code control systems will highlight the errors as spurious changes, making them easier to find than would be the case in other parts of the file system).

So far I have not observed any filenames change, only folder names.

Posted as a heads up to others and a request to Microsoft to fix.
