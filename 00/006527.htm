bob jones aka: cybertaz say:

What you are referring to is called a filename extension. For the most part, extensions inform the user of the type & structure of the file. However, they also are important to the operating system, application programs & file transmission protocols that handle the file. Without them the file type easily can be misinterpreted even though files also are identified internally. That can cause the file to not be opened or possibly become damaged & rendered unrecoverable.

FWIW, my interpretation is diametrically opposed to Daniel's... There was a trend to get away from extensions simply because many users considered them to be 'confusing', 'annoying' wastes of space, primarily [I don't mean to be snobbish] due to the fact that they didn't understand the significance of extensions. In Office 2008 & prior the user had the option to not include an extension when naming a file. The practice was supported by the operating system as well. That led to an epidemic of the problems mentioned above.

Effective with Office 2011 that option no longer is available. All programs in that version as well as Office 2016 automatically append extensions even if the user explicitly tries to exclude one. The same is true of other "recent software" such as Apple's Pages, Numbers, etc. Some versions of the OS will permit hiding extensions, but files cannot initially be saved without them. They can later be deleted in Finder, but there is a warning if you attempt to do so.

My guess is that your files that have no extension were created by a different version of Word & probably are older files. If I were in your position I'd save new versions in the updated format complete with extensions.

IMHO, extensions always should be used, rarely should be removed & even less often should be changed unless there is a valid technical reason for doing otherwise.
