(READ THE UPDATE BELOW BEFORE ATTEMPTING TO DO THIS)

1. Run cmd as an administrator.
2. In the command prompt, run "sfc /scannow" and wait until its completion. It will take a while.
3. Start powershell by typing "powershell" in the same command prompt (with administrative privileges).
4. Run the following command in the powershell:
Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}
5. Ignore any errors it will report in the process.
6. Restart the computer

UPDATE (25 Jan 2016): It appears that the command above MAY BE HARMFUL and may break existing Windows Store apps, including Windows Store. Even thought it did help me resolve the freezing notifications issue at the time and did not cause too many problems with only a few Windows Store apps not working, I WOULD NOT RECOMMEND running it on Windows 1511. After researching forums on the topic, I found out that there seems to be a bug in Windows 1511 that may corrupt your apps when you run this command. If you are also facing this issue while upgrading to Windows 10, following steps will help you:

STEP 1: Run Windows Media Creation Tool as Administrator

First of all try to run the tool again as Administrator as mentioned below:

Right-click on the Media Creation Tool's EXE file and select "Run as Administrator" option.

Now it should not give "Something happened" error message.

STEP 2: Change Locale and Language Settings to English (United States)

This problem may also occur if the locale and language settings in your computer are different than the downloaded Windows 10 setup files.

Just follow these simple steps to change locale and language settings in your computer to fix the problem:

2. Now go to "Administrative" tab and click on "Change system locale" button.

3. Now select English (United States) option from the drop-down box and apply changes.

Restart your system and now run the tool again. It'll work without any problem this time.

The AVI extension for video can be a canister for a wide array of video formats. Formats that perhaps not be supported by Windows Live Movie Maker. And if the AVI file isn't a pure AVI format, you'll need a matching codec to be able to properly read the AVI format you're using.

If you haven't tried already, (as a test) try converting the video to WMV before using it in Windows Movie Maker. 

WMV is Microsoft's video format, Windows Movie Maker should handle the WMV just fine. Finding a proper codec is optimal because you don't have to do any conversion before using video clips in Windows Movie Maker; saving time— but converting video before using it in Windows Movie Maker usually is a logical and stable option.
