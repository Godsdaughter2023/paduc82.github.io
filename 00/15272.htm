daniel pineault say:

Add a new standard module to your database and copy and paste in:

'Source: http://www.pacificdb.com.au/MVP/Code/ExeFile.htm
Public Const SW_HIDE = 0
Public Const SW_MINIMIZE = 6
Public Const SW_RESTORE = 9
Public Const SW_SHOW = 5
Public Const SW_SHOWMAXIMIZED = 3
Public Const SW_SHOWMINIMIZED = 2
Public Const SW_SHOWMINNOACTIVE = 7
Public Const SW_SHOWNA = 8
Public Const SW_SHOWNOACTIVATE = 4
Public Const SW_SHOWNORMAL = 1

Public Declare Function ShellExecute Lib "shell32.dll" Alias "ShellExecuteA" _
    (ByVal hWnd As Long, ByVal lpOperation As String, ByVal lpFile As String, _
    ByVal lpParameters As String, ByVal lpDirectory As String, _
    ByVal nShowCmd As Long) As Long

Public Sub ExecuteFile(sFileName As String, sAction As String)
    Dim vReturn As Long
    'sAction can be either "Open" or "Print".
   
    If ShellExecute(Access.hWndAccessApp, sAction, sFileName, vbNullString, "", SW_SHOWNORMAL) < 33 Then
        DoCmd.Beep
        MsgBox "File not found."
    End If
End Sub

 

Then on your form add your button and add an onclick event with code, such as:

ExecuteFile "c:\...\Your.doc", "Print"

shinmila h say:

Hi stogneri,

 

There seems to be a bug in the Sun Java installation file. The SSVAGENT.EXE referred to is Java’s update agent, which runs every time the browser runs.

 

Detailed inforamtion:

 

http://bugs.sun.com/view_bug.do?bug_id=6635144

 

Step 1:

To resolve this issue, follow the steps below.

 

1.     Click Start, select All Programs, and right-click Internet Explorer.

2.     Here, select Run as Administrator.

3.     Next, click Allow in the User Account Dialog box

4.     In Internet Explorer, go to Tools, select Manage Add-ons, and then select Enable or Disable Add-ons.

5.     Here, click SSVHelper Class and select Disable under settings, and click OK.

 

That should do the fix.

 

More information on add-ons:

 

How do browser add-ons affect my computer?

http://windows.microsoft.com/en-US/windows-vista/How-do-browser-add-ons-affect-my-computer

Step 2:

 

If the icons still don't work, create a new short icon for Internet Explorer from the source location and verify.

Here's how:

 

1. Go to C:\Program Files\Internet Explorer.

2. Right-click on iexplore and select Send to Desktop (create shortcut).


See if the new shortcut icon works.
