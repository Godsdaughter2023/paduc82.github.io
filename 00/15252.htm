Es,

There are still old values in the cache of the pivottable(s), as there is "HORN,EA,MR".

This is not an adviser (anymore; he became "HORNE,EA,MR"), and results in the error.

To empty the cache for all PivotTables, use this code once:

Private Sub ResetPivotCaches()
    Dim Pt As PivotTable
    Dim Ws As Worksheet
    Dim Pc As PivotCache
    For Each Ws In ActiveWorkbook.Worksheets
        For Each Pt In Ws.PivotTables
            Pt.PivotCache.MissingItemsLimit = xlMissingItemsNone
        Next
    Next
    For Each Pc In ActiveWorkbook.PivotCaches
        On Error Resume Next
        Pc.Refresh
    Next
End Sub

then the property MissingItemsLimit is set for all PivotTables to None.

I think this is is enough for future use, but I'm not sure of that.

If not, you can repeat this code on opening the workbook.

To be able to use check boxes in the same document in both Word 2003 and 2010, you should use the "legacy forms" check box.

On the Developer tab, click the Legacy Tools button and choose the check box from the top row of the gallery:

That will insert a check box at the current cursor position. You can then click the Properties button (to the right of the Legacy Tools) or just double-click the check box to make any needed changes.

In order for these check boxes to work, the document must be protected for "Filling in forms" on the Restrict Editing and Formatting pane.

graham mayor say:

The protection options are further to the right on the Developer Tab in the 'Protect' Group. In particular Restrict Editing. If you put section breaks between the sections with the form fields and the free text areas you can select which sections to protect.

Note that documents protected for forms do not have the full range of editing functions as unprotected documents.

There is no ideal solution in a document for use with Word 2003. Content controls which would achieve your aim with Word 2010/2013 are not compatible with 2003.

The only other solution is to use macros to set the check boxes and they create a whole new set of problems. If you want to explore that route see http://gregmaxey.mvps.org/word_tip_pages/add_interactive_toggle_objects_to_document.html

john w. vinson say:

You can add these numerically-adjacent zipcodes which actually exist (avoiding those wehre theare IS no such zipcode) with a couple of simple Append queries: Update queries won't help because they only change the values in records which already exist, and you want to add new records.

Try

INSERT INTO ZipcodeServiced(ResellerID, Zipcode)
SELECT ResellerInfo.ResellerID, AllZips.Zipcode
FROM ResellerInfo INNER JOIN AllZips
ON Format(Val(ResellerInfo.ResellerZip) + 1, "00000") = AllZips.Zipcode;

Then do the same query changing +1 to -1.

You may want to put a unique two-field Index on ZipcodeServiced on ResellerID and Zipcode to prevent duplicate entries (and ignore the warning message when you run the append query).
