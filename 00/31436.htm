danielmonge- [o365] say:

I had the same thing happen for the second time. Once several months ago then just this week. My  only solution was to use PowerShell to remove the duplicating event. You need to have access to a Windows computer or a Virtual Machine running Windows (that's how I did it) and run the commands as follows (your milage may vary):

1. Connect to Office 365 through PowerShell: [View:https://technet.microsoft.com/en-us/library/jj984289(v=exchg.160).aspx:550:50]

2. Find the event: Search-Mailbox -Identity "name.here" -SearchQuery "Kind:meetings",'Subject:"Event Title"' -EstimateResultOnly

This will give you the number of events and the total size. You're limited to searching for 10,000 at a time.

3. Delete the Event: Search-Mailbox -Identity "name.here" -SearchQuery "Kind:meetings",'Subject:"Event Title"' -DeleteContent

Note: The "name.here" is what the name of the mailbox is. If you don't know the name of the mailbox type the command Get-Mailbox before running the search. Make sure you have the Event Title as the exact name of the event. Use EXTREME caution when deleting the event! Incorrect Title or any other wrong commands could potentially delete the wrong events or mailbox.

Reinstall Windows Preinstalled On Computer

OEM computer manufacturers are required to provide at least one way for you to reinstall Windows if you buy a computer with Windows preinstalled - your computer should have included either a restore partition on your hard drive, or Windows install/restore disks. ﻿Most computer manufacturers also have a feature to make your own install disks.
