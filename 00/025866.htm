You have the opposite of the usual situation.

In your case, the range E3:E17 contains numeric values in cells that are formatted as Text.  Thus, =ISTEXT(E3) is FALSE and =ISNUMBER(E3) is TRUE initially. When we select E3, press f2, then Enter, the value is converted to text. Thus =ISTEXT(E3) becomes TRUE.

Moreover, the lookup range D4:D18 contains text values, although the cells are formatted as General.

That explains why VLOOKUP fails initially, then succeeds after E3:E17 is converted to text.

Your comments suggest that you used Text To Columns to "isolate" the first word in column A (second worksheet), presumably into E3:E17. When I do that straight-forwardly, the "isolated" values are numeric and the cell format is General.

But if I select Text for the column in the TTC menu, the "isolated" values are numeric, but the cell format is Text.

I suspect that is how you encountered your initial situation.

I am surprised by the TTC behavior. But it is what it is. The question is: how can you fix it?

-----

I would change column A of the first worksheet to contain numeric values. TTC will do that if you select General (default) for the column in the TTC menu.

Then I would also select General for the column that becomes E3:E17 in the second worksheet.

Of course, then you must "refresh" the Pivot Table.

[EDIT] But when I refresh the PT after converting column A to numeric values and General format, D4:D18 is still text.  I confess that I know nothing about PTs. If you cannot make it work, perhaps someone who knows PTs can help you.
