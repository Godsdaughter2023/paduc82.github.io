greginmich say:

The rationale for using a context-menu (on-demand) scan is that this type of scan will unpack and scan the contents of container files (archive files). That’s something that real-time (on-access) protection simply doesn’t do. There’s a very easy batch-file workaround that provides both drag-and-drop scanning for files, folders, and drives, and context-menu scanning for files and folders.


Open Notepad and copy and paste these two lines:

 

"C:\Program Files\Windows Defender\MpCmdRun.exe" -Scan -ScanType 3 -File %1

Pause

 

Then name the file something like “Defender Scan”; give it a .bat file extension; and save it to your desktop. Now simply drag and drop the icon for any single file, folder, or flash drive onto the batch file icon for a scan. If you’d prefer to use the context menu, then move the batch file to the Send To folder (C:\Users\<account name>\AppData\Roaming\Microsoft\Windows\SendTo), as suggested by manfeildman in this thread:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-you-scan-a-file-in-windows-8/3277ad77-b400-47c1-8508-1dfbf1729d3d

here's one approach:

Use the after update of the combo box and assign the values you're wanting to the text box.

if me.combobox between  1 and 5 then

    me.textbox = date()          'Current date

else
