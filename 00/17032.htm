suzanne s. barnhill say:

Duplexing (printing on both sides of a sheet) is different from printing "2 pages per sheet" (two pages on one side of a sheet). Duplexing is controlled by your printer Properties. On the Print tab of the File backstage, beneath Printer click on "Printer Properties." This will open a dialog generated by your specific printer driver. This is not part of Word but will vary depending on the printer, so we can't give specific instructions on where to find the duplexing setting, but it will be somewhere in the Properties if your printer has the ability to duplex automatically.

 

Additionally, if your printer is capable of duplexing automatically, then the dropdown list under Settings that by default says "Print One Sided" will also offer options for duplexing; mine has "Print on Both Sides, Flip pages on long edge" and "Print on Both Sides, Flip pages on short edge."

 

If your printer is not capable of duplexing automatically, then yes, you must duplex manually. Under Settings: Print One Sided, the dropdown list will offer only "Manuall Print on Both Sides" as an alternative to "Print One Sided."

rohn007 say:
Just becasue the name of the program is the same does not mean that the underlying code is the same. For a long time Word for Mac and Office where programmed completely separately. It was actually only January of this year that they really were finally based on the same program code.

The point is, even now it is entirely possible for a file to open in one but not the other.

The error message you captured it typically Microsoft vague. It could be a bad file name, (I'm assuming you have tried changing the file name) or the file is corrupt. The type of corruption indicated by this error, and the message you got from Google Docs is an "XML Error". Sometimes Word gets confused and mixes up the XML codes and then the next time it can't open the file. Normally the error message should show a row number, 2 in this case and a character number, which points to the place that Word got terminally "confused" when it tried to open the file. When we are given that second number, you / we can crack open the file and find the spot the problem appears. With some experience we can find the problem codes and fix them. But there is no second number, so we can't do that.

Sometimes Google Docs can get past errors that Word can't, then it saves the file in the correct order so that Word can work with the file afterwards. LibreOffice is another program that can fix some of the minor problems.

At this point, it is time for drastic measures. For the moment you can still open the file in Word Windows, that could change at any time. So you going to have to risk losing some time to avoid losing the whole document.

One of the causes of this sort of problem is Equations. If the file was originally created with equations before the new year, it was using the old equation editor which was known to cause this sort of problem, sometimes.

So there are a few trouble shooting options left to you.

******************

Use SAVE AS to RTF format close the file. Open the RTF, then Save AS back to DOCX format may clear up the problem.

*************

. * Create a new blank file
. * Open the problem file
. * Turn on the Show/Hide feature so you can see the paragraph marks.
. * Add a couple of blank paragraphs to the very end of the document.
. * excluding those new blank para marks at the bottom, select from bottom to the top of the document using <CTL><SHF><PAGEUP>
. * copy
. * paste into the new blank document
. * save it
. * does that fix the problem?

*****************
This next process will hopefully allow you to narrow down on the problem code. But it is slow and repetitive. Save a chunk, look for problem, save another chunk, look for problem.
. * Select the first half of the document
. * paste into a blank document
. * Save it
. * Test it
. * If it is good, make a copy of the document with the good half, (so you don't mess it up).
. * copy half of the remainder
. * paste into one of the files with good first half
. * save again
. * test again

Keep repeating until you find a small enough section to recreate. Yes, it is not pretty, it is the fastest non code way to find the bad part of the file and fix it.

***********************

You've tried Google Docs, next try LibreOffice.
