I should add that putting the second instance into "add" mode is a matter of setting its DataEntry property to True.  So that bit of the code would look like this:

    Set frm = New Form_AssessmentHeader  ' maybe

ken sheridan say: As Dirk says, this is not trivial, but on the other hand it's not overly difficult.  You'll find an example of how to open multiple instances of a form as MultiForms.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

In this little demo file a form is opened as multiple instances of the class (hence the need for a class module), filtered to a different row in each case.  The trick is that, not only is a new instance of the form opened, but that it is added to a collection.  It is thus possible to identify the individual instances by third position in the collection.  In your case you would open the form normally, and then open a new instance and add that to the collection, so the collection would have just the one  member, so to close the second instance you simply have to move focus to the member of the collection with index, and call the Close method of the DoCmd object.
