john w. vinson say:

If you're using an expression like this as a Control Source, it must be preceded by an = sign. 

As an alternative to the nested IIF statements, you can use the Switch() function - it's sort of a many-branch IIF. It takes arguments in pairs, evaluates them left to right, and when it finds a pair with the first argument TRUE it returns the second:

=DateAdd(Switch([UOM]="days", "d", [UOM]="month", "m", True, "yyyy"), [CalTimeInterval], [LastCalDate])

You could also consider storing the Access interval code itself in the UOM field so you can use it directly.

I can get around the problem by "sharing" the attachment in pdf form but I would rather send it in Word format with the track changes visible.  Any thoughts please? 
doug robbins - mvp office apps & services (word) say:

The action of emailing a file as an attachment does NOT change that file in any way.

Perhaps when you are opening the document, it is in the "No Markup" view.

You can change that via the "All Markup" dropdown in the Tracking section of the Review tab of the ribbon.

You might also want to Lock Tracking, which prevents other authors from turning off Track Changes.  That can be done via the Track Changes drop down, also in the Tracking Section of the ribbon.
