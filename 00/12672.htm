jeeped say:
... Whenever I try to print (i click file, then the print button), a save menu pops up ...

This sounds like the default printer on your computer has been set up as some sort of XPS or PDF document creator. These are typically installed as pseudo-printers and a Save As dialog opens to request the name of the output file.

Check your Devices and Printers from the Windows Start command. If you have a hard copy (e.g. paper) printer you wish to use, right-click and Set as default printer. Reopening Word should have this printer set as the new default.

Edit: once you get this sorted out, you may want to retrace your steps and delete the print-to-file copies you created earlier.

Hey all. I have what I believe is a relatively straightforward spreadsheet. It's 6 worksheets total less than 3,000 rows. The whole thing is 660k. Created with Office 2010 on a PC. Runs on Win7.

The functions are still there and they still look correct. They just no longer work. Various things solve the problem. Sometimes deleting the row I just inserted puts things back. Sometimes inserting another row works. Sometimes nothing solves it except for this little piece of VBA I wrote:

Private Sub restoreSheet()

    Dim vCell As Variant
   
    For Each vCell In Range(D10:D1048576)
        vCell.Formula = vCell.Formula
    Next

End Sub

Amazingly enough, this works every time. Somehow resetting the cell's formula kick-starts it into relearning how to work. I think the fact that I am setting the formula to itself proves that my formula is correctly constructed.
