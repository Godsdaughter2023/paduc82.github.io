yangzheng_msft6300 say:

Hi Shivbhaghi7619,

Based on my research, there is no related command to check the emails sent from and received by your users in one day.
There is a workaround for an Office 365 admin to check the sent and received emails report in Office 365 Admin Center. However, this report may updated with a delay. To check the report, please follow the steps below:
1. Login to Office 365 Admin Portal.
2. Admin->REPORTS, click Sent and received mail under Protection.
3. Choose Custom Date selection and change custom date range to one day.
4. Click View table to see the mail amount in the table view.

brian tillman [outlook mvp 2007-2019] say:
The only way to get a client-side rule is to run a mail client.  Since Outlook.com is web-based, accessed using a web browser, usually, all rules are server-side.  There are no client-side rules, unless you access Outlook.com using some mail client installed on your PC.  Snce you don't say you have done that, it's not possibe to explain what you see.

brian tillman [outlook mvp 2007-2019] say:
Which client is that?

As far as I can tell, there is no ability to add server-side rules from the client side.  There are no mail clients that provide Outlook.com sync services that include rules.  The only rules you can create on the server side are those you create by opening your Outlook.com mailbox with a web browser and clicking Options>More mail settings>Rules for sorting new messages.

You cannot reply to incoming messages via rule, however.  There is no "reply" action in the rule set.  Instead, you can create an away message using Options>More mail settings>Sending automated vacation replies. With Outlook.com, that's the best you have.
