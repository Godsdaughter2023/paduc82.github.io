dale howard [mvp] say:

Steve --

 

First, you would need to add the PM to the project team in the Resource Sheet view and assign a Standard Rate for the PM.  Second, you would need to assign the PM to an extra task in the project than spans the current Duration of the entire project.  Some people refer to such a task as a Level of Effort (or LOE) task.  You might call this task something like Project Management, and then assign the PM to the task with a Units value that represents the approximate percentage of an average working day that the PM will work on this task.  If he/she might spend 2 hours per day on average managing the project, you would set the Units value to 25% on this task.

 

If the PM has PTO, you can edit the PMs calendar to show that.  If the Duration of the project increases, you would simply need to increate the Duration of the Project Management task.  That would be the simplest and easiest way to handle this, in my opinion.  Hope this helps.

=lookup not working with a =left formula - Microsoft Community

avia1234 say:

=lookup not working with a =left formula

Hi,

 

I'm doing a simple =lookup formula on a cell where I only want to use the first character in the string, so I want to use the =left function to extract it but I'm getting N/A returned. E.g. "4B" is in cell A1 and I want to do the lookup on the 4 only. The named range for the lookup is "KS2GdPts". The formula I'm using is:

=lookup(left(A1,1),KS2GdPts)

 

I've broken down the forumla into separate cells to see which bit is not working. The =left is fine and returns the 4. However, when I try the =lookup on the result it returns N/A but if I overtype the =left formula to the 4 the =lookup works! I swapped the =left to =mid on the offchance but the same problem. Why can't I get the two to work together? It's driving me mad that a simple thing like this isn't working.

 

Feedback on this gratefully received, thanks

 

mike h.. say:

Hi,

 

I think the clue here is when you say if you type the 4 it works.

 

=left(a1,1) returns a text value and it sounds like you're looking up a number so a text "4" is not the sane as the numeric 4. Try it like this

 

=LOOKUP(--LEFT(A1,1),KS2GdPts)

 

the -- coerces the "4" into a number 4

ashish mathur say:

Hi,

 

Try this

 

=lookup(left(A1,1)*1,KS2GdPts)

 

Please note that for the LOOKUP() function to work, the values in KS2GdPts should be sorted in ascending order.
