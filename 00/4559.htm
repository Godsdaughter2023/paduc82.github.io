Sub ConvertData()
Dim ws1 As Worksheet, ws2 As Worksheet

Dim v1 As Variant, v2 As Variant, v3 As Variant
Dim r As Long, t As Long
Set ws1 = Sheets("Sheet1") '<<< change name as needed
r = ws1.Cells(Rows.Count, 1).End(xlUp).Row

keith peck (kdpeck) say:

Outlook for IOS : how to open .eml file attachments

Here is the answer to a closed thread about Outlook for IOS : how to open .eml file attachments saved to your desktop

This is using Microsoft Outlook for Mac (Office 2019, and OSX 10.14.1 Mojave) and does not require a third party application.

E.g. you previously dragged a message from Outlook onto your desktop, or you saved an EML attachment to your desktop and now you want to open the file. By default when you try to open the saved message you will see an error/security alert - "[message name].eml” can’t be opened because it is from an unidentified developer”

  1. Right-click (or hold the "control" key and click) on one of the EML files you have saved to your desktop.
  2. From the pop-up menu click on “Open With >” and then click “Other…” from the fly-out menu list.
  3. A Finder window will open in the Applications folder with a header: "Choose an application to open the document "[message name].eml".
  4. Immediately below the list of Applications it shows “Enable: Recommended Applications” – Change the drop-down list setting to be to Enable: “All Applications”
  5. Check the box immediately underneath the Enable drop-down for “Always Open With”
  6. Scroll down the list of Applications and click on “Microsoft Outlook”
  7. Click on the “Open” button at the bottom-right of the window.
  8. The EML file you selected in step #1 should now open with Microsoft Outlook for Mac, and from this point on any time you double-click on an EML file it should then also open in Outlook.

I don't have a copy of Office XP handy and don't know if 2003 behaves exactly the same way, but:

Rightclick the icons and choose Edit Sound Object.  If the resulting dialog box shows a file path at the bottom, it means it's a linked sound file.  You'll have to do some extra fiddling to get it to work correctly on other systems.  More info on that below.  If there's no file shown, it's an embedded WAV and should work wherever you try to use it, provided the needed CODEC is available on the computer where you're playing it back.
