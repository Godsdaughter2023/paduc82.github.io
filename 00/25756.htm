Sub DeleteRowsBasedOnCriteria()
   Dim i   As Long
   Dim j   As Long
   Dim del As Range

   Application.ScreenUpdating = False
   Set del = Cells.Find("*", , xlValues, , xlByRows, xlPrevious).Offset(1).EntireRow

   For i = del.Row - 1 To 2 Step -1
      For j = i - 1 To 1 Step -1
         If Cells(i, "P") = "Order" Then
            If Cells(i, "Q") = "Waiting Payment" Then
               If Cells(i, "K") = Cells(j, "K") Then
                  If Cells(i, "J") = Cells(j, "J") Then
                     Set del = Union(del, Rows(i))
                     Exit For
                  End If
               End If
            End If
         End If
      Next
   Next

   del.Delete
   Application.ScreenUpdating = True
End Sub

Here is a starting point.  Assuming you want to  add it to all first level subfolders of the main folder, try this

 

Option Explicit

Sub AddSubfolder()
'Under tools, add reference to Microsoft Scripting Runtime
Dim FSO As Scripting.Filesystemobject
Dim RootFolder As Object
Dim SubFolder As Object
Dim myFolder As String
Dim myNewFolder As String

Set FSO = CreateObject("Scripting.FileSystemObject")

myFolder = "C:\Users\User.Name\Documents\"   'Change to identify your main folder
Set RootFolder = FSO.GetFolder(myFolder)
For Each SubFolder In RootFolder.SubFolders
    Debug.Print SubFolder.Path
    myNewFolder = SubFolder.Path & "\2012"
    If Not FSO.FolderExists(myNewFolder) Then
        MkDir (myNewFolder)
    End If
Next SubFolder
End Sub

Ranking only showing top 50 - Microsoft Community

kyleethan say:

Ranking only showing top 50

I am first doing a qry to rank the product id's by revenue.  this I can do. Then I want to do another query to show only the top 50 results as there are hundreds of product id's.  How can I do this.

djsteele, mvp say:
While you have the query open in design view, look at the Properties window associated with the query. (Make sure you'll looking at the query's properties, not the properties of a field in the query). One of the properties is named Top Values, and by default it'll be set to All. Simply change that to 50 to select the top 50 results. Note that you must have the sort order set correctly for the query. And be aware that if there's a tie, you may get more than 50 records returned.

qa guy commelec say: Post the query SQL that shows your ranking.

The SQL can be seen by opening the query in design view, clicking on VIEW icon and selecting SQL View, highlight all, copy, and paste in a post.

ken sheridan say: It depends how you are computing the rank.  If you are doing so by means of a join, which is the most efficient method, then you simply need to add a HAVING clause to the query.  Here's an example using a table of transactions ranked by amount in descending order:

SELECT T1.TransactionDate, T1.TransactionAmount,
COUNT(T2.TransactionID)+1 AS Rank
FROM Transactions AS T1 LEFT JOIN Transactions AS T2
ON T2.TransactionAmount>T1.TransactionAmount
GROUP BY T1.TransactionAmount, T1.TransactionDate, T1.TransactionID
HAVING COUNT(T2.TransactionID)+1 <=50
ORDER BY T1.TransactionAmount DESC;

Here's the equivalent using the less efficient method of a subquery:

SELECT T1.TransactionDate, T1.TransactionAmount,
    (SELECT COUNT(*)+1
     FROM Transactions AS T2
     WHERE T2.TransactionAmount > T1.TransactionAmount) AS Rank
FROM Transactions AS T1
WHERE (SELECT COUNT(*)+1
     FROM Transactions AS T2
     WHERE T2.TransactionAmount > T1.TransactionAmount) <=50
ORDER BY T1.TransactionAmount DESC;

Note that these both return true ranks, so in the event of ties for the 50th rank more than 50 rows will be returned, correctly of course.
