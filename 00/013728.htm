tom_hha say: Hi all,

I finally got it working ! The clue came out while using an AP to connect instead of the Wireless Controller.
I had no problem while authenticating from a single AP to the IAS Radius server (with the ValidateServerCert trick)

I compared the two methods and noticed that the wireless controller was sending out EAP-Identity-Request packet very quickly (1 per second), so the time I typed my pass on the PDA, it has already received 5+ EAP-Requests and when I pressed OK, it was sending my Identity with Request-ID=1 and was rejected because the controller was already expecting a greater Request-Id.

I adjusted the timeout and voilà !!! Here is the command line for Cisco Wireless Controller 4402  (the value was set to 1s !) : config advanced eap identity-request-timeout 30
save config

 Thanks everybody for investigating.

john - project say:

DonnlyH,

I'm guessing you made the mistake of using paste links. Unfortunately those are prone to corruption and it's likely there are several rogue link fragments as a result of various editing steps to the Project and Excel files.

I'm also assuming you are using a Project version later than Project 2007. In older versions of Project the Edit Links command was readily available via menu drop down however with the release of Project 2010, that command is now hidden. You need to add that command back to a custom ribbon tab or to the Quick Access Toolbar. These steps will show you how to add a custom tab to the ribbon.

1. Right click in a blank area of the ribbon or go to File > Options > Customize ribbon

2. Click the New Tab button below the right side selection list

3. Select the New Group that appears under the new tab

4. In the selection list on the left scroll down to find the Edit Links command. Note you may need to expand the list to show all commands

5. Select the Edit Links command and then hit the Add button in the middle

6. Hit OK

You should now see a new tab on the ribbon. Select it and you should see the Edit Links command. Click on it and you will see all paste links in the file. Select each one and hit the Break Link button. After all links are successfully broken and you exit the Links window, the Edit Links command will be greyed out. Then save the file. That should solve the problem on the Project side although you may still have link fragments in the Excel file. I'm not that familiar with Excel's options but you can look for a similar process on that end.

If you still have issues with Project after using the above process, save the Project file as XML. Then re-open the XML file in Project. Note, you will lose an custom formatting with this XML to Project process.

Hope this helps.

John

graham mayor say:
Only macros that are capable of being run from the macro list will be shown in that list. Functions and Macros to which you are required to pass parameters are not listed.

e.g. Take the following two macros

Sub AEvents(oDoc As Document)
MsgBox oDoc.name
End Sub

Sub BEvents()
AEvents ActiveDocument
End Sub

Only the second macro will be listed as the first requires you to pass the identity of the document (oDoc) to the macro e.g.; by calling it from the second macro as shown.
