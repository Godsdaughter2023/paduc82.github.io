tony jollans say:

First off, all modern computers have fonts that will display Russian and the fact that you can view Russian web pages confirms that you have such fonts. If you have unicode-encoded Russian text in your database, it should display in Russian (one caveat: I don't know what kind of font substitution might happen in Access Forms if you happen to be missing a font that is required, but I would be surprised if that were the cause of your problem).

 

Forgive me if this is all known to you, but for clarity for all ... text in computer systems is encoded. Historically only one byte was used per character, which allowed for only 256 different characters to be in use at any one time. All computers used the same display characters for the first 128 of those but the second 128 were displayed according to a 'code page' in use. For example, the character encoded as 193 would display as Á on an English computer but as Б on a Russian one.

 

Most modern software will allow, if not try to enforce, the use of unicode which uses a variable number of bytes (usually two) for each character and unicode characters should display the same on all computers all over the world.

 

If your data is encoded in the pre-unicode way you will see exactly what you describe and if you change your locale setting your view should be 'correct', yet you say that didn't happen. Just to be sure, on Windows 7 > Control Panel > Clock, Language and Region > Region and Language > Administrative tab, it shows the current language for non-unicode programs and there is a button "Change system locale" - it is this that you need to change.

 

Changing your data to unicode, if that is the problem, requires you to examine each character and if it has a code of between 127 and 255 deciding if it needs changing, what it should be changed to, and then changing it. I have never done this nor do I know if there are any facilities in Access, or Windows, or third party utilities that might help.
