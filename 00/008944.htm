Open Office is a free suite of programs that are similar to -- but not the same as -- Microsoft Office. The programs can read and write the same file formats as Microsoft Office, but their features are different.

Open Office shouldn't just appear on your computer, but it's possible that it was downloaded along with something else you installed because you didn't notice a checkbox that offered it.  Because Open Office can use the same file formats, it takes over the file assignments from Microsoft Office -- but in this case it appears to have done something wrong.

If you prefer Office 2003, go to the Windows Control Panel and open the Programs & Features page. Click on the Open Office item and click Uninstall. When that's finished, click on the Microsoft Office 2003 item, click Repair, and let that finish.

OK, it sounds like you're halfway there -- at least you don't get the error messages any more, right?

marshall barton say:

That's a lot of code and I'm not going to analyze all of it.  The key point here is that there is only one subform control and it can only contain one source object at a time.  So, as I think I explain more clearly in the newsgroup, when you switch the subform control's source object property you are kind of like closing the form and then opening it with another source object.

At a glance, I don't see anything wrong with the collection, but you need more than that if you want all the forms (Events) to stay open until the users closes them.  The one thing I checked your code was to see how you are filtering these forms to just the one user selected Event.  I did not see it, which I think means the user did the selection after the form was "opened" (via setting the subform control's SourceObject property).  And that is a serious problem because the previous "instance" was closed so you can not get back to any other "open" event.

When using real form instances, you can use the main form's WindowLeft, WindowTop properties plus the offsets within the main form to position a real form instance where your subform control used to be.  The WindowLeft/Top should tell you where the main form currently is located in the Access window.  Remember to do this every time you make a form instance visible just in case the user drags the main form around.  The code sequence should be something like:

   TempForm.Visible = True
   TempForm.SetFocus
   DoCmd.MoveSize Me.WindowLeft + 3000, Me.WindowTop + 1440

Change the 3000 and 1440 to the relative position within the main form, but you need to add in the main forms border thickness and title bar height so there will be some guesswork involved.
