    After adding RAM, I get a BSOD stop:0x0000007e - Microsoft Community

    jonosinsalacious say:

    After adding RAM, I get a BSOD stop:0x0000007e

    I have a Intel DP35DP motherboard with an Intel Q9450 Core2 Quad processer running Windows 7 64bit.  I installed 4 sticks of 2GB DDR2 800MHz RAM from KomputerBay and get BSOD stop:0x0000007e (0xFFFFFFFFC0000005, 0xFFFFF8000C39D56D, 0xFFFFF880027B4408, 0xFFFFF880027B3C60) .  It gets to the "Starting Windows" screen, freezes, then BSOD.  It was booting and running when I had one channel of the 2GB sticks (KomputerBay) and one channel of 1GB 667MHz sticks (Kingston).  I've tried setting the timmings in BIOS to 5-5-5-12 as per Intel website, but still get BSOD.  I hope to find a fix from this forum.  How bad would it affect performance if I were to have three slots of the 2GB 800MHz sticks and one 1Gb 667MHz stick if it turns out to be a bad stick?

    bossdweeb say: If the new RAM is correct for the system then you may have a bad stick, so run memtest on each
    stick individually to determine if the RAM is good.

    I would return the lot for exchange if it tests bad. All the major RAM mfg's have good warranty.
    If the RAM is not from a major mfg or does not have a good warranty then, no surprise.

    If you have 'tweaked' the RAM settings in BIOS then set them back to default/automatic.
    Default settings are the best guarantee of stability.

    Running an odd stick will likely cause problems particularly when it has a different speed, latency
    or voltage. Using an odd stick is also likely to cause the RAM to run in single channel mode and
    reduce performance somewhat as a result.
