The file type .msi abbreviation stands for Microsoft Installer, which is a file type that can be used only in Microsoft Windows. For the convenience of Windows users, some developers package their add-ins so that an installer properly places the add-in in Office for Windows. That's not to say the add-in itself won't work on a Mac - just that you can't install it because it is locked into a Windows installer program.

Ask the developer to give you the .xlam file without all the rigamarole. Then in Excel use Tools > Add-ins and then click the Add button to activate the add-in. Unfortunately, if the developer went to all the trouble to put it into a Windows installer, I fear the developer likely used Active-X controls and hard coded Windows-only file paths into the add-in, which would render it useless on a Mac. First try the ..xlam and if it doesn't work, point the developer to this forum and we can help her make her add-in code cross-platform.

Kellie,

The article you referenced was due to a recent bug.  There are only two workarounds that work for that bug.  

The first is to try and fix the PST data file path.  One of the previous builds had a bug that caused the bad path.  In Outlook click File, Account Settings, Account Settings.  Click Data Files.  Under Location if the PST path is bad it will have a UNC path such as \\?\C:\users\username\Documents\Outlook Files\file.pst.  To fix it click Settings and then click OK.  Under the Location you should see the path is fixed and no longer has the \\?\.  After you fix the PST path restart Outlook and give it some time to index.  

The other thing you could try is from the same steps above note the location and name of the PST file that you cannot search.  Remove the PST file, then restart Outlook, and then add the PST back.  Then restart Outlook and give it some time to index.

If those solutions do not work the only other resolution is to create a new Outlook profile, https://support.office.com/en-us/article/Create-an-Outlook-profile-F544C1BA-3352-4B3B-BE0B-8D42A540459D.
