philip taron (seattle) say:

Hey tstoons, glad you found a solution that works for you. Those ...-Sigs folders are required for correctly extracting out the signatures at the moment, or, if they're not there, then there's one cycle of failures before the problem rights itself. (You can see the results in http://img.photobucket.com/albums/v323/marc57/WindowsUpdate.jpg.)

We do our best to try to detect every possible case where your signature installation might fail and mitigate it, but, occasionally there are situations which we are either not able to detect (this happens to be one of them) or we've overlooked.

The good (?) news is that we're constantly trying to improve the signature delivery experience, so problems like this will right themselves once we verify a fix. For now, your workaround of not cleaning out the %windir%\temp folder looks like a fine solution.

Philip

As the dev behind the decision to use the %windir%\temp folder, I do have to defend it slightly: we need a location that's transitory -- no content other than a log file remains there after a successful signature update -- and secured for read by elevated admins only by default. %windir%\temp fills that need beautifully. The failure was in the assumption that once a directory was created there, it would remain for some time, given that it was empty. However, as the traffic on this topic shows, for some users that assumption isn't true at all.

stephen boots say:

Here is my understanding of the "solution" and the cause based on Philip's response above.

1. MSE brings down incremental updates only most of the time. They are smaller.

2. Weekly (?) (cycle not clear to me) a larger update comes down that includes bits needed for the install of the incremental smaller updates.

3. If the incremental update is downloaded and the previoudly downloaded core bits needed for subsequent updates within that cycle are missing, an error occurs.

4. if you then do a 2nd manual check for updates, the larger update gets brought down and can be applied/installed. (not clear if this is always true)

So, the solution is to *not* clear the %windir%\temp folder at all or be prepared to get an error on updating MSE signatures.

The cause is not ccleaner, but the act of deleting the contents of the %windir%\temp folder, thereby causing the update to fail as the expected sub-folder is missing.

I believe that the MSE team is indeed looking to change the behavior to not throw the failure, but to create the extraction folder and download the core bits if they are found to be missing. I might be mis-representing that explanation, though.

I don't know if this is something that will need to wait for the next version upgrade of MSE or if it is behavior included in the engine/database that typically gets refreshed monthly.

For now, I'm marking this post as the Answer along with Philip's reply above. However, I'll leave this thread and the other similar one as a sticky until the issue is fixed.

create border around stuff - Microsoft Community

jeffs_programs say:

create border around stuff

Hi,

What is the easiest way to create a border around a group of shapes that describe a program flow.

What is the best way to create shapes to show the program flow?

I am using word 2007, saved in word 2003 compatability mode.

 

Thanks!

jay freedman say:

To make the flowchart, on the Insert ribbon click the Shapes button. A bit below the middle of the gallery is a group of flowchart symbols. Combine these with the arrows and lines. Right-click any
shape and choose Add Text to be able to type into them. It's useful to use the Shift key when drawing lines to keep them exactly horizontal or vertical; and you can use the arrow keys to nudge the
selected shape into position with finer control than you can get with the mouse. Use Ctrl+drag to make same-size copies of shapes you already have.

Draw a rectangle shape to enclose the whole diagram. Right-click it and choose Format AutoShape; in the dialog, set the fill to No Color, and choose the desired color and style for the border line.

When you've drawn and labeled the whole diagram, go to the Home menu and click Select > Select Objects, and drag a bounding box around the whole thing (including the border) to select all the shapes
at once. Right-click and choose Group > Group to make the whole thing into one object, so it won't come apart if you move some piece.

If you need to work on flowcharts very frequently, or very complex ones, you'd do well to get a copy of Visio. Drawings made there can be imported into Word.
