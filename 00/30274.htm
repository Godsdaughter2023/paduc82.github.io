Unless your computer has a GPS chip, Windows has no idea where it's located. Instead, Windows can only guess where it is, based upon what it knows about how you are using your computer. Those guesses are often comically wrong.

The better thing to do is to establish a default location for your computer, like this:

Start menu -- Settings app -- Privacy -- Location -- Set default location

Even if your computer moves around with you - if it's a laptop - Hulu only needs to know what country you're in.

rob koch say:

Actually, that's clearly a keyboard issue, though exactly what's causing it and whether it's early symptoms of a hardware failure or instead a problem with supporting software such as drivers, some component of the operating system or other interference isn't as clear.

Since keyboard contact scanning works across rows and columns, with groupings of several keys both across and angle wise (e.g. o and 0 in the same column) down the array, your descriptions fit these physical concepts precisely.  That's why this is clearly somehow related to the keyboard, it's just not clear the true cause.

It could also be something as simple as a modifier key like a Shift, Ctrl or Alt key causing flaky input, which could interfere with proper input, but by the description I would more suspect something within the array or keyboard scanning hardware beginning to fail.

I've mostly had such issues due to liquids being spilled in the keyboard, which often occur days later as the sugars or other components of the liquid begin to dry and cause keys to stick.  However, this again doesn't seem to explain issues with adjacent keys, since this typically results in problems with only the affected key(s) themselves.

So what seems most likely is either something metal physically moving around within the keyboard or as mentioned above early signs of a hardware failure randomly affecting only portions of the keys.

Since you didn't specify the type of PC, I'll just state that the easiest way to confirm this is to temporarily replace the keyboard with another, though with a laptop that may not always work, since the built-in keyboard might still interfere with an external one attached via USB.  However, that's always worth a try as the quickest way to prove the problem is either physical or at least somehow related to the current keyboard device.

FYI, these problems almost never have anything to do with malware, though since they are supported by software including drivers and the operating system itself, flaky interactions with these by other software of any sort can contribute to such unexplained operation in rare cases.

I'd always try the simple explanation first though, since as hardware ages such problems can seem to come out of no where, even with hardware that up until that point had been rock solid in its operation.

alexfayads say: The key to your problem might lie right here: "So when in doubt, shut it down :) and turn it on again (a techno phobics answer to everything)" and here: "'windows resume loader'". Shutting it down is fine but it has to be a proper shutdown. When you see the "Resume" message then you did not turn off the PC but you put it to sleep (=hibernation). To really turn it off, do one or all of the following:
  • Press the power button for 10 seconds.
  • Unplug the power cord (if this is a desktop PC)
  • Remove the big battery (if this is a laptop)

If this fails then a System Restore might fix the problem. Do you have a Windows installation DVD? A Windows Repair CD? If the answer is No to both questions then youmust burn a Repair CD now! It's done via the Control Panel / Backup and Restore.

 

 

Here’s how I fixed it:

  1. Connected my external USB DVD drive and booted the Windows 7 disc. You’ll also need a USB key with your BitLocker recovery key if you encrypt your drive.
  2. Access the recovery options and bring up Command Prompt.
  3. You need to delete the hibernation data (hiberfil.sys). This is a protected file; however, and it is not easy to delete even at the administrative recovery command prompt. Change to the drive letter for your hard drive. You’ll then need to enter the following commands:

    takeown /f hiberfil.sys
    icacls hiberfil.sys /reset
    attrib -h -s hiberfil.sys
    del hiberfil.sys

    (Those commands are necessary since by default even the administrative user doesn’t have access to modify the file).

  4. Reboot the machine.

The system will then boot normally (but without the hibernation data, it’s a cold boot, so you’ll lose anything you had running when the system originally hibernated)

Creating a bootable CD from a .ISO file is not the same as just copying the .ISO file to a blank CD.  You have to use software that understands how to burn a .ISO file to a CD to create a bootable CD.

In the Hiren's ZIP file are the BurnToCD.cmd file that you can double click to launch it, but I have never used it and prefer to use ImgBurn.

If you need a free and easy CD burning software package, here is a popular free program:

http://www.imgburn.com/

Like many third party programs you might install, the ImgBurn installation defaults to installing things you probably don't want installed on your system in the form of extra Internet browser toolbars or may make other adjustments to your browser.  You don't want to install any of that so you have to pay attention during the installation.

When clicking through the installation screens be sure to pay attention to the screens and always choose a Custom install and UNcheck the following (or any other things like it):

UNcheck:

Install the AVG toolbar and set AVG Secure Search as my default search provider
Set AVG Secure Search as my homepage and newly opened tabs

Choose custom installation again and UNcheck the following:

Uncheck:
Install QuickShare

There could be other things too.

Here are some instructions for ImgBurn:

http://forum.imgburn.com/index.php?showtopic=61

It would be a good idea to test your new bootable CD on a computer that is working.

You may need to adjust the computer BIOS settings to use the CD ROM drive as the first boot device instead of the hard disk. 

These adjustments are  made before Windows tries to load.  If you miss it, you will have to reboot the system again.

When booting on the Hiren's CD you will see a menu of options.  Choose the Mini XP option.  Then it will appear that Windows is being loaded and you will be presented with a desktop that has the look and feel of the Windows Explorer interface you are already used to using.

Remember:  You did not boot on your hard disk - you booted into the Hiren's desktop.

From Hiren's you could also browse you HDD and manually see if you can get through KB307545 file rename, delete, copy process, but that is alot of trouble and they already have a tool in there to do that for you.

From the Hiren's HBCD Menu, choose Programs, Registry, Registry Restore Wizard and it will show you all your current System Restore Points (each RP has a copy of the registry for that RP). and go back to a day when you think your system was working okay. 

Since the listed RPs are sorted by NUMBER and not by date, be careful to look at the dates of the RPs offered and not necessarily the number of the RP - so don't accidentally chose RP9 when you really need to choose RP90 - so be sure to pay attention to the dates.

Unless you are absolutely 100% sure what you are doing, never choose "Original Installation Settings".

Note that the Registry Restore Wizard is not equal to an XP System Restore.  Only the XP registry is restored.

You may start with the most recent Restore Point prior to the incident and if that doesn't work, go back to the next most recent one, etc. and keep your fingers crossed.

In this situation, you can use DISM to repair the Windows image (.wim), which then it will allow you to use the System File Checker to fix any issue with your Windows 10 installation.
