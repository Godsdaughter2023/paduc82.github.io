john korchok say:

When cloud storage syncs with a local folder, it has to add tracking attributes to the file kept in the local folder. This way the cloud storage knows what files have been edited locally and need to be synced with the server. This is accomplished either by keeping a hidden database folder in the local folder, or by adding hidden file attributes to each file in the local folder. It appears that Google Drive uses the latter system, adding attributes to the files.

When Word and PowerPoint edit files, they create hidden work files in the same folder where the file is stored. As so as these work files are created, the local Google folder thinks they are new files that you have created that should be synced with the server. Then when you close the file being edited, the temp file is deleted by Word and Google Drive gets confused. As an example, I tried editing a Word document in a local Google folder. As soon as I closed it, I got a warning:

You removed a Google Doc from your Google Drive folder. This file is a shortcut. Content online will be lost if Drive Trash is emptied.

Obviously, I did nothing of the kind, I just opened and closed a Word file in that folder. Just as clearly, this folder does not operate the same as a normal Windows folder.
