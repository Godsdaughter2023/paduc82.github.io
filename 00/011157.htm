So, as I said, you can disable the Close button in the form's properties. Or you can (and should) put code in the On Close event:

Best way to convert a POP e-mail to IMAP?

Recently, I've decided it would be better if I could access this account from multiple systems, so I am trying to make the transition to using this account as an IMAP.  Have had no trouble getting the IMAP setup, and sending and receiving new e-mails works just fine.

 

The real problem I'm having is how to move all my e-mails from the old PST file to the IMAP server.  I've tried moving a couple folder manually, but I'm am constantly plagued by errors.  So what I'm wondering is if there is an efficient way to import all my e-mails over to the IMAP server???

 

Appreciate your thoughts.

brian tillman [outlook mvp 2007-2019] say:
There is an import command (File>Open>Import), but if you import a PST you don't get the option to choose or change folder names.  Don't try to move folders.  With the old PST in your mail profile so you can see it's folders, visit each folder, select it's content with Ctrl+A, then hold Ctrl and click the selection.  Drag it to the IMAP folder where you want it.  The cursor will have a plus sign next to it signfying a copy instead of a move.  Release the mouse button to copy all of the items in the selection to the destination folder.  If you don't have a folder with that name on the IMAP server, create it manually, then copy the contents of the source folder.
