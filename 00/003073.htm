What is the fastest and easiest way to alphabetize labels in Word? - Microsoft Community

natly say:

What is the fastest and easiest way to alphabetize labels in Word?

Original Title: "Alphabetizing labels"
Does anyone know the fastest and easiest way to alphabetize labels in Word? Or if its even possible.?. Thank you so much
daniel g. grau say:
Labels are actually a table, and tables can only be sorted by columns.

If you have a single column of labels, click on the table, select Table > Select > Table and then Table > Sort.

If you have multiple columns, when sorting, each row will carry over all its cells as it reorders, so that a second column of names won't be vertically sorted. This is a bit more complex, because the table has to be disassembled, sorted, then rebuilt:

  • Make a copy of your document, or select the table and copy it to a blank document
  • Do a global replacement of ^p (paragraph break) to ^l (line break). This will keep all info on each label in a single paragraph.
  • Select the table (Table > Select > Table) and do Table > Convert > Convert table to text. Choose "Paragraph marks" and each label will become a separate paragraph.
  • Do Table > Sort to reorder the paragraphs (which are still selected).
  • Do Table > Convert > Convert text to table to convert the paragraphs back into a table. Choose "Number of columns" equal to the initial number of columns.
  • Select the resulting table and paste over the original.

chocogiggles say:

Using the Find button & once found what you are looking for making it a different colour/border outline thickerr

When using the Find button, I want to make the cell that I have found a different colour/making the outline of the cell thicker, is there a way of doing it?

ossiemac say:

This can be done with a combination of Conditional formatting and a small VBA code (Macro)

Not sure of your expertise so here are detailed instructions so you will not be held up asking for further information to implement.

  • Select entire worksheet. (Click where the row numbers and column Id's meet.)
  • Select "Conditional formatting" (Home ribbon)
  • Select "New Rule"
  • Select "Use a formula to determine which cells to format"
  • Copy the following formula and paste into the field "Format values where this formula is true"

    =AND(CELL("row")=ROW(),CELL("col")=COLUMN())

  • While still in Conditional formatting, Click Format button.
  • Select the required tab at top of dialog. (I suggest the Fill tab. Thick borders are not supported in Conditional Formatting)
  • After setting the color, click OK and OK until dialog closes.
  • Next Right click the worksheet tab name
  • Select "View Code" (Opens the VBA Editor at the worksheets module)
  • Copy the 3 lines of code below and paste into the VBA editor (don't change the Sub name)

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Application.ScreenUpdating = True
End Sub


  • Close the VBA editor (Red cross very top right of the editor window)
  • Ensure macros are enabled as follows.
  • If Developer ribbon not displaying then right click anywhere in one of the ribbons and select "Customize the ribbon".
  • On the right side of the dialog check the box against "Developer" and click OK.
  • Select Developer ribbon.
  • Select Macro Security (In the Code block at left end of Developer ribbon)
  • The dialog that displays should default to Macros in left column.
  • Select required security option button. (Option to "Disable all macros with notification" should be OK.)
  • Click OK to close the dialog.
  • Now each time you select a cell then it will be highlighted (Including if selected with Find)

The workbook must be saved as "Macro Enabled".

inconsistent margins - Microsoft Community

gibranshah say:

inconsistent margins

I'm trying to set the margin between the body of my pages and the footer to 1cm:

http://www.shahspace.com/setting%20bottom%20margin%20to%201cm.png

But I'm getting inconsistent results:

http://www.shahspace.com/bottom%20margin%20inconsistent.png

Does anyone know what's going on here?
suzanne s. barnhill say:

There are two issues here:

  1. Top and bottom margins (as set on the Margins tab of Page Setup) don't actually apply here because they are superseded by your header and footer margins (set on the Layout tab or in the Position group on the contextual Header & Footer Tools | Design tab).
  2. The footer on the third page shown uses a larger font and has an upper border, causing it to push the bottom margin up.

To correct this error, you must first understand how Word sets margins. The default top and bottom margins are 1" (2.54 cm). The header and footer margins are 0.5" (1.27 cm). That means that the TOP of the header and the BOTTOM of the footer are 0.5" (1.27 cm) from the edge of the page, and, with a one-inch top and bottom margin, the header or footer can occupy half an inch of space before affecting the top or bottom margin. If it exceeds half an inch, it will push the top down or the bottom up.

When you set your bottom margin to 1 cm without changing the footer margin, you are not accomplishing anything because the footer is already 1.27 cm from the edge of the page and is pushing the bottom of the page up accordingly. With a 1.5 cm top margin, you aren't allowing your header much space, either.

You can solve the immediate problem by making the footer on the third page the same as the ones on the other two, but a better solution would be to straighten out the header and footer margins and then set more sensible top and bottom margins to allow for the header and footer.

For more information on headers and footers, see http://wordfaqs.ssbarnhill.com/headerfooter.htm and http://wordfaqs.ssbarnhill.com/DifferentMargins.htm
