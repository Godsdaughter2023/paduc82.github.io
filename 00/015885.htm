lz. say:

Hi Kate

While Zixuan solution definitively works I would suggest a slightly different approach for 2 reasons:

  1. If you select columns I through L (10 workshops) before setting up the Cond. Formatting formula rule, the calculation (COUNTIF) will apply to > 1 million of rows (* 10 columns) while 800 only are of interest. And although COUNTIF is very efficient why doing unecessary evaluations?
  2. If below your 800 rows you enter values (or formulas) that are equal the cells will color although they should not

So suggestion is to format your data as a Table. Tables offer many many benefits, 1st one being dynamic ranges: if you enter any data in the 1st row below the table - so row 801 in your case - the Table will auto-adjust and include that new row.

If your Cond. Formatting rule applies to columns I through L of the Table it will stop at the bottom of that Table. And as the Table auto-adjust when adding new data, no need to update the Cond. Formatting rule. Same goes for formulas, data validation rules… within a Table - no need to copy them, that's automatically done with the Table adjustment

Cond. Formatting formula rule:

=COUNTIF($C2:$L2,C2)>1 (no reason to include row 1) that currently applies to =$C$2:$L$800

Sample available here. Hope this all makes sense & helps

Highlight row based on active cell within range

With range (columns AT:BB), if a cell is selected highlight the active row orange. If any other columns outside the range is selected - no highlights.

jllatham say:

Single cell solution - goes into the specific worksheet's code SelectionChange() event. The problem with this is that the entire row's cell shading is either set to orange or removed and that may affect other shading you have in use.

 

 

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
  If Target.Column >= Range("AT1").Column And _
   Target.Column <= Range("BB1").Column Then
    Target.EntireRow.Interior.Color = 44907 ' code for orange
  Else
    Target.EntireRow.Interior.ColorIndex = xlNone
  End If
End Sub

Try this variation




Dim LastRow As Long
Private Sub Worksheet_SelectionChange(ByVal Target As Range)
On Error Resume Next
   If Target.Column >= Range("AT1").Column And _
    Target.Column <= Range("BB1").Column Then
     Target.EntireRow.Interior.Color = 44907 ' code for orange
     If Target.Row <> LastRow Then
     Rows(LastRow).EntireRow.Interior.ColorIndex = xlNone
     End If
   Else
     Target.EntireRow.Interior.ColorIndex = xlNone
     Rows(LastRow).EntireRow.Interior.ColorIndex = xlNone
   End If
   LastRow = Target.Row

On Error Goto 0
 End Sub

When somebody fill out the cell, it looks 22/01/2019. I tried to use format cells date setting again, format painter that cannot change the date format to 22/01/19.  The only thing that work is to type the date on the cell again but it's really time consuming to do so. 

Just wonder if the date format problem came from the improper "Cut & Paste" from other excel file go with 22/01/2019.

eli qian msft say:

Hi Gladys,

Thank you for your data.

Based on checking the workbook, I found that this issue was caused by improper input. Generally, the build-in date function consists of years, months and days. The month should be in the front of the days so that the date function can identify it as a date. Because if the day is in the front of the month whilst the days is over 12, the value cannot be identified correctly.

Back to the issue you are experiencing, I create a date function to fix this issue. For example, you could enter =DATE(RIGHT(B2,4),MID(B2,4,2),LEFT(B2,2)) in column I. You will get the desired format in I2. Then drop down and right to correct the corresponding date. After that, copy the function values and paste it with Paste Option "Values" to get rid of the date function in another range. Finally, you could use the values to replace the incorrect format value in original place.

(Note: You need to set the desired date format in the function range.)

Hope this workaround helps. For any concerns, please feel free to let us know.

palcouk say:

If the email tab shows only the single mail account, but when you start Outlook itself it indicates that there are two mail accounts then your outlook Profile is corrupt, it probably occurred as a result of what ever method you used to "transfer" from one PC to the new.

On the new PC, Control Panel (View by large icons)>Mail applet (mail32bit) in the resulting window>Profiles>add new Profile, set this as default, then add the single mail account using the auto setup page where only the mail address & password is required, once verified, finish the process then start OL under this new Profile, and allow to fully sync before doing anything else
