Leading Zeros

A frequently asked question is "Why does Excel remove the zero in my zip code?" The reason is that the zero is not required to display the value of the number. However, since number formats are not about the value, they are about the appearance, you can use a special format to retain the leading zeros, (use as many zeros in the number format as you want digits in your number). There is even a Zip Code number format listed in the Special category.


What if you don't want to show leading zeros? For example, you want a number to appear as .123 instead of 0.123. We saw above how a number format of uses the pound sign to hold places that indicate the comma thousands separator should be used, but if the places only contain leading zeros, they are not used and the comma is not needed. In the same way, the pound sign in the number format holds the place for the decimal point, but if it is a leading zero, the place contains no zero digit.


You can use the pound sign after the decimal point, to indicate how many digits to show if they are not trailing zeros. The format shows up to three decimal digits, except for trailing zeros. The following table shows how treats various numbers. Notice that the decimal point is displayed for the first two examples even if there are no nonzero decimal digits. Also notice how zero is displayed: since the zero is a leading zero, all that is displayed is a lone decimal point.

Are you sure that the current open window has focus?

What I mean by that is you have to make sure you've clicked in a specific window and made it the active window... we in the developer world call that having "focus." 

So the scenario is... you click on an open window and copy something from it, but you might need to click once to get the system to recognize that this new window is the window to fave "focus" or "attention" or whatever adverb you'd like to use. Once the window has focus the Operating System will recognize from where to place the selected text onto the system's clipboard.
