a. user say:

 

H,
re:  find cell with a particular format

Finds top most cell in columns "H:L" with Red interior color.
Additional "FindNext" code required to find more cells.
SearchFormat is True in order to utilize FindFormat setting.

'---
Sub FindFirstColor()
  Dim rngFound As Range
  Dim rngToSearch As Range
  Set rngToSearch = ActiveSheet.Columns("H:L")

  Application.FindFormat.Clear
  Application.FindFormat.Interior.Color = vbRed
  Application.FindFormat.Locked = True
  Set rngFound = rngToSearch.Find(what:=vbNullString, searchformat:=True)
  rngFound.Select
End Sub

Formula for a value when the tab is not yet present - Microsoft Community

patrickmcnicholas say:

Formula for a value when the tab is not yet present

Hi,

I have a workbook with a summary tab that pulls data from another tab in the workbook.  The name of the second tab changes monthly (January, February, etc.).  The graph on the summary tab uses the data from a table which pulls data from the second tab whose name changes each month.  I would like to set the table up to pull data from the second tab but am having difficulty  with the tab name change.  Specifically if February hasn't occurred yet then there is no February tab for the data to pull from.   Is there a way to enter a formula to pull data from a tab that doesn't exist yet without returning a #REF! and disturbing the graph?

Thanks! 

jllatham say:

You could perhaps use the IFERROR() function to 'wrap' your existing formula in and return a value of zero (or anything you want).

Let us say you want the formula

=February!A1

but because the February sheet doesn't yet exist, you get the #REF! error.  You could write the formula like this:

=IFERROR(February!A1,0)

Now, you still get the prompt to identify where February is, but now instead of #REF!  you see 0.  The IFERROR() says "if this formula creates an error, return the optional value otherwise return the results of the formula".  This only works in Excel 2007 and later, if you're using 2003 or earlier, a different form of the formula would have to be used.

Testing here shows me that the formula still won't automatically recognize the February sheet when you do add it -- :(

But this will, but creates some maintenance headaches:

=IFERROR(INDIRECT("February!A1"),0)

Will show 0 until you create the February sheet, and when you do, it will automatically pick up the value from that sheet.

The problem is that because "February!A1" is a text literal, it doesn't update as you fill it up/down/left/right on the sheet.  But if there's a pattern to using those and you do need to fill it we could do some math to 'fix' it.

For example, lets say that formula is on a sheet in row 2, we could write it this way:

=IFERROR(INDIRECT("FEBRUARY!A" & Row()-1), 0)

The ROW() would evaluate to a 2 (because the formula is in row 2, then 2-1=1 and it gets added on to FEBRUARY!A" to form the address FEBRUARY!A1

Hope this helps, and doesn't leave you feeling lost.

Caution - Word document edits NOT saved when managing documents via - Microsoft Community

dmhunter_5 say:

Caution - Word document edits NOT saved when managing documents via OneNote (or other apps such as Outlook)!

There is a problem with the way OneNote for iOS and Word for iOS interact. It might seem to be a little obscure, but it caused a very real problem for me when I lost a great deal of work on a document. If I have missed an obvious solution to this problem or someone knows how I might recover my lost work, then I would love to know. Sorry for all the detail below, but I wanted to be very clear about the problem.

 

My physical office building had a scheduled power shutdown last Sunday. I knew I had to work on a document over the weekend, so I dropped a copy of the Word document from my PC into my OneNote page for the matter on which I would be working … Here are the steps:

 

  1. Drop an Word document into a OneNote page on desktop Windows 10 PC
  2. Later, open that Word document on an iPad from that page on OneNote for iOS
  3. Start editing the document and, into the editing, tap on the icon on the Word toolbar that is second over the the top left (iie. the one that looks like a piece of paper with a top corner folded over and has "…" at the bottom) to bring up the Word File menu
  4. Check the Word AutoSave setting. You will see that AutoSave is set to "On" and below that setting is the message "Changes are automatically saved to iPad"
  5. Continue editing until your work is done
  6. When done exit Word by tapping the iOS "<Back to OneNote" message that appears at the very top left of the screen, or by double tapping the iPad's home button and then swiping up on the Word app (Note: the problem does not occur if you exit Word by tapping on the leftmost icon on the Word toolbar (i.e. the one that is a circle with a left-pointing arrow in it)
  7. Later (an hour later, the next day … whatever), go back to the OneNote page and re-open the document on which you have been working. Word will open, but the version of the document that opens will be the original, without any of the edits
  8. If you had happened to re-open Word directly, without going back to the OneNote page, and you have not worked on any other Word documents in the interim, then your document with all of your edits will still open when you open Word; it is stored in the iPad's cache for Word. However, opening the document in question or any other Word document from other document icons in OneNote (or any other app such as Outlook) will wipe that cache, including all your edits
  9. Go to the Word file list and top on the icon to "Open" a file. One of the file locations listed there, below OneDrive and any other locations that you added, will be "iPad" - where Word said your changes would be saved automatically. You'll see that your document is not there. Look for it in OneDrive, in recent documents, … anywhere else.  It will be gone and all your edits will be lost!
  10. There are at least two ways to avoid this: 1. to close/exit Word by tapping on the leftmost icon on the Word toolbar (i.e. the one that is a circle with a left-pointing arrow in it) which will jump you back to OneNote, but will also, as an intervening step, save your changes to the version of the document stored in OneNote; or 2. go into the Word File menu and manually "Save a Copy" of the document to your iPad or OneDrive before closing Word

 

Again, the problem is that a user can be misled in a material way and with potentially very problematic results when they see that AutoSave is set to "On" and, below that setting, the message "Changes are automatically saved to iPad". It is so easy to put down one's work for a while, return to their iPad and re-open their document from their OneNote page. That simple act wipes out all of their work.  It is not being saved to their iPad.

gary's student say: In A1 and B1 put the real and imaginary components of the number.
In C1 put:
=COMPLEX(A1,B1)
to create the complex number
In D1 put:
=IMPRODUCT(C1,C1)
to square the number.
