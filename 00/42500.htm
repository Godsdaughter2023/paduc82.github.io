jkinglet say:

Thanks. Figured it out finally. Essentially the same way you outlined with a couple of twists and turns.

Set up hidden Administrator (command admin prompt) with password (net user administrator password /active:yes). Logged in with the hidden administrator then set up a new local account and transferred everything from the corrupt login folder on C:drive/users into the new local login file under users.Then used regedit to ditch the corrupted Microsoft Account login (.bak)

Changed MS account login password, restarted, logged in with the new local account, went to settings, accounts, logged in with Microsoft Account ("Old [Windows] Password" by the way was the password I used to set up the local account an hour earlier. "Old password" is meaningless. And the "If you don't have a Windows [old password again] click next" does not work..never has). 

Synched up everything and back in business. Only took a day out of my life!
