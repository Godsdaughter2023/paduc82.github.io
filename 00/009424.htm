Bug (Excel Mac 2016 on Sierra): Text import Wizard doesn't scroll so cannot set data format

The Text Import feature allows you to set the data format for each column.

The UI in this version (Excel 2016 latest, on Sierra latest, Mac) appears to be broken, see below.

The default is "General", but there's an option to manually override. To do that you need to be able to select columns. There is no way to do that outside the window shown, so, if you have more columns, you're out of luck. Tried scrolling, tabbing, using arrow keys, resizing the dialog box, nothing works.

In this case, the import is genealogical data, so it's a classic example where you have to override Excel - since Excel cannot handle dates prior to 1900, it converts those to strings, but then tries to "interpret" the others as date. That's useless, a user of this would want all the data handled as text because Excel can't be trusted with complicated dates.

I had this same issue, IE11 would retain the tabs until a reboot.    Found the answer.

I had it set to disabled for Personal List.   Selected enabled, now my tab history survives the reboot.

To try to auto recover your email do the following
1. Sign in to your account.
2. Go to your Deleted folder.
3. Look {towards the bottom}for "Lost a message?", and click "recover deleted messages".

We can only recover recently deleted emails, specifically not more than 5 days old.
Beyond that won't be possible.
