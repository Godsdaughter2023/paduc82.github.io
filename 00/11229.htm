suzanne s. barnhill say:

Unless Track Changes was turned on while the changes were made, and the personal data has not removed from the document on saving, you will not be able to see a record of the changes or who made them. If you have a copy of the original document, however, you can use Compare Documents to see what changes have been made (but not who made them).

With either of the documents open, display the Review tab and click the Compare button (in the Compare group, toward the right end of the Ribbon), then choose Compare again. You'll get a dialog allowing you to choose the document to be compared against and specify which changes you want to see and how you want them displayed (in the original document, the revised document, or a new document).

I would imagine the issue is partially caused by the fact that the server is not authenticating you on the workstation.  My expertise in this area is very limited, but basically when you start work you should login to the server so the server knows who is who.

I am not sure why you chose AFP as your transport protocol.  I believe it is likely that you will have better luck with SMB.

It could help to ensure that you do not have more than one connection mounted to that server.  For example: if you have an Icon to "Documents" on the server on your desktop, and another to "Documents:Projects" that can get things a little confused.

graham mayor say:
The suggested method would work with any styles in the document, however if you have a list of allowable styles, you could check whether any other styles are used and either change them to a particular style or prompt the user to change them - at its simplest maybe something like


Dim oPara As Paragraph
For Each oPara In ActiveDocument.Paragraphs
    Select Case oPara.Style
        Case "Body Text", "Heading 1", "Heading 2"
            oPara.Reset
        Case Else
            oPara.Range.Select
            If MsgBox("This paragraph style is not allowed. Do you wish to change the style?", vbYesNo) = vbYes Then
                Exit Sub
            End If
    End Select
Next oPara

Add the list of allowable stylenames separated by commas to the first case statement.

When you run the macro the first style not to match that list is selected and the user warned. The paragraph is selected to give the user the opportunity to change it. The macro can then be run again to find the next inappropriate style.

If you want prompts to select from a list of allowable styles, you are going to need additional code
