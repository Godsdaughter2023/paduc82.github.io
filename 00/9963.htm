Call function in control's event - Microsoft Community

questionboy say:

Call function in control's event

I am trying to use a control's event to use a function to format a string

in the BeforeUpdate (also tried the afterupdate event) event I have

=FormatTelNo([Tel])

I have tested the output of the function itself and it is being triggered and returning the proper value, but the form's value is not changing.  Why?  How do I get it to take the function returned value?

QuestionBoy

marshall barton say:

We will need to see a Copy/Paste of the function's code to tell what it's doing.

The BeforeUpdate event can not change the control's value, so AfterUpdate is the event to use.  Code like this directly in the event procedure can do what you want:

   Me.telnum = FormatTelNo([Tel])

To do it without an event procedure by calling the function directly from the AdterUpdate property, the function needs to change the value of the control and thus must be able to figure out which control is using it at the time.  You can use the form's ActiveControl property for that, but then the function would need to know which form contains the control.  That's easy if the function is Public in the form's module, just use Me for the form object:

   Me.ActiveControl = whatever

But if the function is in a standard module so it can be used in more than one form, then you need to pass the form object as an argument to the function.

The AfterUpdate property would then be:

   =FormatTelNo(Form, Tel)

and the function would look like:

   Public Function FormatTelNo(frm As Form, Tel As String)
       . . .
      frm.ActiveControl = whatever
   End Function

Notice that the function does not return a value.  It has to put the result back into the control's value.
