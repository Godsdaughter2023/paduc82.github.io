When you log in to an Admin account, you are still a Standard user.  The prompts that you get are the means of authorising your Admin-level permissions for the process being authorised [only that one].

The most immediate concern I can see is that there is a problem with your user profile [the login problem].  Do not delay, create two additional password-protected Admin accounts [poss NormalAdmin, ReserveAdmin] the first time the computer lets you log in.  Write the passwords down somewhere secure.  Then, if the user profile does get so corrupted that you cannot log in to that account any more you will have an Admin account to log into to rescue the situation.

Since all of the symptoms you describe might be caused by user profile corruption, I think you should also create a new password-protected user account for your day-to-day tasks.  And write the password down somewhere secure.

Little is known about what causes user profile corruption but one generally-accepted cause is turning the computer off without letting Windows go through its full shutdown procedure.  This can happen if

  • Hold down the power button for 10 seconds to force it off deliberately [possibly because the computer has locked up so you have no choice].

No, no, no - keep them as local user accounts.  Local user accounts can do everything that you want to do on the computer and they cannot be affected by external changes such as problems with MSAccounts.  Make another one so you have a total of two Admins that are purely local accounts.  [Don't bother to change or delete what you have already done though as it might be useful for other purposes - see below]

At the bottom of the dialog box when you create a new user account is a control for making them local accounts.  So the full instructions are:-

  1. Start button,
  2. Settings,
  3. Accounts,
  4. Family & other users,
  5. Add someone else to this PC,
  6. I don't have this person's sign-in information [at the bottom of the dialog],
  7. Add a user without a Microsoft account [at the bottom of the dialog],
  8. Fill in the username, password, password hint,
  9. Next,
  10. [Admin prompt appears]
  11. wait a moment as the account is created & you are returned to the Family & other users dialog,
  12. click on the entry for the new account in the list,
  13. Change account type,
  14. [Admin prompt appears]
  15. Select Administrator from the dropdown list,
  16. OK,
  17. now you can see it listed as an Administrator - Local account.

You do not need to transfer anything over to the new Admin accounts.  They are just there to do work when you need them to - authorising installations, fault investigations, ... .

You also want to make a new standard/admin account for your own use [because I cannot entirely dismiss the notion that the fault symptoms are because of user profile corruption].  This is the one that you can choose to link with your MSAccount [but only if you want to - I have an MSAccount for my email but it is not linked to my user accounts on the computer]

You need one MSAccount-linked user account but it does not have to be the user account you use for your day-to-day activities, it could be this new Admin account since you have already linked it to an MSAccount  - you only need an MSAccount so you can update the "modern apps" such as the calculator & maps. 

The position you want to reach is

  • two password-protected Admin-level local user accounts
  • one new standard/admin user account for each human user.  These can be either local accounts or MSAccount-linked ones as you desire.  If one of these is an MSAccount then that's all you need.
  • If none of your human users' accounts are MSAccounts then keep that new Admin that you have already linked to an MSAccount. 

Your new accounts for your human users will need files to be transferred across.  You can just copy all the files from the C:\Users\<OldAccount>\Documents, \Music etc to the new account folders [these new folders appear when you first log in to a new account rather than when the account is created].  If you have MS Office Outlook then its files are, by default, in an awkward place so I can give you separate instructions for finding them.

The additional accounts can be left as they are or can be deleted.  As long as they are password-protected then there is no rush to decide.  You do not want to delete them until you are sure that you have copied across everything you need from them anyway.
