FWIW, the behaviour is the same here in Word 2011, and I agree with your conclusions about the processing of hyperlinks. (From what I recall, there are some problems with Word+hyperlinks on the Windows platform too - to do with special characters such as "?" and characters that have to be represented by their hexadecimal equivalents. )

I wondered whether there was a difference between protocol scheme names (http, ftp etc.) that are registered in Mac OSX and one that are not (because, e.g. callto does not appear to be registered on my system). But it appears to make no difference - a link to an afp:// filename is also converted into a file://localhost name that does not work either.

Doing stuff like locking the field makes no difference.

I also wondered whether it might be possible to intercept the HyperlinkOpen command using VBA to change the hyperlink address. That means ythat you write a VBA sub called HyperlinkOpen and Word then runs that command instead of the built-in one. However,
 a. Word 2011 only runs the macro when you right-click on the link and choose Hyperlink->Open Hyperlink. It doesn't run the macro when you click (or cmd-click, whatever) on the hyperlink
 b. When you try to insert the address, Word modifies the link to be file://localhost as before. Using Word's own "FollowHyperlink" method doesn't work either.

An alternative is to create that HyperlinkOpen macro and get it to use AppleScript to unmangle the skype address and invoke Skype. Something like this:

Sub HyperlinkOpen()
'
' HyperlinkOpen Macro
' Open
'
Const strSignature As String = "file://localhost/callto/::"

'MsgBox "we're in"
Dim strAddress As String
Dim strResult As String
Dim strScript As String
With Selection.Hyperlinks(1)
  strAddress = Trim(.Address)
  If Left(LCase(strAddress), Len(strSignature)) = strSignature Then
    strScript = "" & _
      "tell application ""Skype""" & vbCrLf & _
      "  activate" & vbCrLf & _
      "  send command ""CALL " & Mid(strAddress, Len(strSignature) + 1) & """ " & " script name ""Word VBA""" & vbCrLf & _
      "end tell"
    'Debug.Print strScript
    strResult = VBA.MacScript(strScript)
    'Debug.Print strResult
  Else
    ' Follow the unmodified link (there are various options you can add)
    .Follow
  End If
End With

End Sub


As I said before, it will only work if you use the Hyperlink->Open Hyperlink content menu option to open the link. You could also assign a keystroke to the HyperlinkOpen macro and open the link by selecting it then pressing the keystroke, and so on. Not nice, but the best I can do right now.

Level 4 & 5 headings, Word 2010 - Microsoft Community

mefallit say:

Level 4 & 5 headings, Word 2010

Level 4 & 5 heading issue. How can I get the paragraph that follows this level of heading to have Normal formatting so that it does not automatically interpret the entire paragraph to be part of the heading?

     Anticipating table of contents issue. If I can get the paragraph to have Normal formatting, will the TOC automatically display my Level 4 & 5 headings properly?

Appreciate your help. I'm working on a dissertation and sure do thank you for your help.

pamcaswell say:

If it is the style separator you want,  you first need to create separate paragraphs for heading 4 (or 5) and the text that follows it.  Then, with the cursor anywhere in the heading paragraph,  press Alt+Crl+Enter  or  click the style separator icon.  Word will apply the style separator to the next paragraph mark it encounters.   Make sure that the paragraph and line spacing are the same for both styles.  If they are not, you may get odd spacing after the first line of the paragraph. 

Another method is to enable linked styles. Apply the body text style to the entire paragraph and then select and apply the heading style to the beginning portion.  This method works if the heading styles are not linked to a multilevel list.  I would not use this method if the file might be opened in W2003 & before because the linked styles will break and the headings would then not be recognized by the W2003 TOC.  Linked styles are the easiest to apply, but because of the limitations, I still use the style separator most of the time.  

Pam

 

stefan blom say:
Note that when working with style separators it's a good idea to display nonprinting marks, which you can do by clicking the ¶ button on the Home tab. "Ordinary" paragraph marks will show as pilcrows (¶) and style separators as pilcrows surrounded by a dotted border.
 
Moreover, if backward compatibility is crucial, neither style separators nor linked styles should be used as older versions of Word don't support either. Instead one would have to format an "ordinary" paragraph mark as hidden (it would look correctly only when you are not showing hidden text).

ronrosenfeld say:

Not entirely clear what you want.  This is what I interpreted, but you can expand or clarify:

For a result like this:

 

 

Use these formulas:

A1:  =TODAY()  (or enter any date you wish for testing)
   Custom Format:  Right click on cell:  Format/Cells/Number/Format  Type:   "as of "d-mmm-yyyy

D1:  =AVERAGE(D4:D1000)
   Change 1000 to be able to encompass at least the largest number of entries you might have in this column

D4:  =IF(COUNT(B4:C4)=0,"",IF(C4>0,MIN(C4-B4,$A$1-B4),$A$1-B4))
