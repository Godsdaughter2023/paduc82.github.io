Hi,

Try this.

Place your numeric values in cell A3 downwards.  In this example, the data is in A3:A11.

In cell B1 enter:    100                      

In C1 enter:          =B1+100               then drag across to cell Bi2  (to give 6000).

In B2 enter:          =B1&"-"&B1+99     then drag acroos to cell Bi2

In B2 enter:          =COUNTIFS($A3:$A11,">"&B1,$A3:$A11,"<="&B1+99)

then drag that across to cell Bi3.

 

That will give you counts of occurances within ranges of 99.  In your question you say "e.g. 100-199, 200-299 so on and so forth until 5000-6999"  that last one doesn't really fit into an obvious pattern...

gary's student say: You need to anticipate the need for an ESCAPE and build the functionality into your design.

Consider:

Dim StopLooping As Boolean

Sub InfiniteLoop()
StopLooping = False
Do
    DoEvents
    If StopLooping Then Exit Sub
Loop
End Sub

Sub LoopStopper()
StopLooping = True
End Sub

Once started, the InfinitLoop macro will loop forever.  However, the macro shares focus with the user.  While the macro is running, the use can perform many worksheet functions.  If the user wants InfiniteLoop to stop running, the use will run the LoopStopper macro.  InfiniteLoop will detect the flag change and exit.

chip pearson say:

I usually set the break key to raise an error (error 18), then test the error and take appropriate action. For example

 

Sub AAA()
    Dim B As Boolean
    Application.EnableCancelKey = xlErrorHandler
    On Error GoTo ErrHandler:
    Do Until B = True
        DoEvents
        ' your code here
    Loop
   
    Exit Sub
ErrHandler:
    If Err.Number = 18 Then
        MsgBox "Break"
    Else
        MsgBox "Other error: " & vbNewLine & _
            CStr(Err.Number) & " " & Err.Description
    End If
End Sub

This will loop forever until the break key is pressed, at which point execution jumps to the line following ErrHandler. The code tests the error number to ensure the error is from the break key and if so take appropriate action.

A variation on this is shown below. Break raises an error and the error handler sets the loop control variable to True, causing an exit from the loop.

 

Sub AAA()
    Dim B As Boolean
    Application.EnableCancelKey = xlErrorHandler
    On Error GoTo ErrHandler:
    Do Until B = True
        DoEvents
        ' your code here
    Loop
   
    Exit Sub
ErrHandler:
    If Err.Number = 18 Then
        B = True
    Else
        MsgBox "Other error: " & vbNewLine & _
            CStr(Err.Number) & " " & Err.Description
    End If
End Sub
