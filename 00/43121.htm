wunders say:

It sounds like there is a misunderstanding about terminology.  I had thought that you had already been successful in connecting these two computers but just wanted it automated better.

There must be some way for computer A to communicate through the network with computer B.  Otherwise, you can't do what you're trying to do and "sneaker-net" is the only solution.  It is OK to be on two different "networks" (subnets?) but these subnets must be able to communicate to each other in some way.  Usually both subnets are connected to a common internet and the Internet devices around the world find other devices by using IP address of the machine (or router) they are trying to connect to.

When a folder on one Windows 7 machine is "shared", then that machine becomes a "server" by definition because other machines (clients) will attempt to connect to this machine, and a machine accepting connections from other devices is a "server".  A given Windows 7 computer can be a server and a client at the same time depending on which machine initiates the connection to the other.

Network topology is all-important and so far nothing has been said about how these machines connect to each other network-wise.  Computer "A" might be my own machine at home connected to my home network behind a router then through the router to my ISP via a cable modem.  If computer "B" were my friend's computer in a different city connected the same way at his end, then it would be extremely difficult to share files/folders directly between these two computers because routers typically will block unexpected incoming packets such as occurs when a client machine initially attempts to contact a server machine.  Servers behind routers require that the routers be properly configured using either the "DMZ" feature or the "Port Forwarding" feature on the router to make sure that initial incoming packets find their way to the proper server/computer.  In addition, the Terms of Service (TOS) of some ISPs (typically cable companies) may prohibit servers on their networks and actively block traffic on these ports.

Solutions like having both machines connect to a common "Cloud Server" or establishing a VPN connection between the two networks may be easier than dealing with the network configuration required to directly connect one machine to the other.  Without knowing the networking, I really cannot comment much further.
