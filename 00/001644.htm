Hi Stephen

You pretty much summed it up. Lack of category support has to do with the IMAP protocol and not Outlook (the email client).

Aside from it not being recommended to store contact/calendar data in an IMAP .ost file where the folders have the added words of "This Compute Only" in the folder name, the alternate solution would be to

#1 - create a new PST file
(go to Home Ribbon --> <New Items> --> <More Items> --> Outlook Data File

#2 - make the newly added PST file the default data file for the profile which is where the default contact/calendar folders will be stored
(Go to FIle --> Account Settings --> Data Files tab --> highlight the newly added PST file --> set it as the default

#3 - Copy the contact/calendar data from the IMAP folders to the corresponding folders in the newly added PST file. Categories will be enabled for contacts in the PST file

Now for some background info

- an IMAP .ost (offline storage) file is a mirror of the email folders on the server. Contact/calendar folders do not sync with the server (as defined by the IMAP protocol)
- if you ever remove the IMAP account from the Outlook profile, the OST file will also be deleted from the system and data in any "This Computer Only" folders will be lost
- also, if Outlook cannot find/open the .ost file, it will automatically create a new one, email folders will be sync'd and the "This Computer Only" folders will be empty
