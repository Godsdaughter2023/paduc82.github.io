Possible solution for a red MSE icon after startup. - Microsoft Community

micro111 say:

Possible solution for a red MSE icon after startup.

Possible solution for a red MSE icon after startup:
------------------------------------------------------------------

I am using Windows XP SP3.  MSE appears to have to be in sync
with another program: Microsoft Antimalware Service.  This service
can be found by Control Panel -> Administrative Tools -> Services.
Select Microsoft Antimalware Service.  Using "Restart the service"
causes the Microsoft Antimalware Service to stop then start.  If the
MSE icon is currently red then the icon should turn green.  If the MSE
icon is green then MSE should flash a stopped message and turn
green again.

Solution 1: Restart the computer to see if MSE and Microsoft Antimalware
Service can sync properly.

Solution 2: Copy Services from Administrative to the Desktop.  This allows
easier access to Services and restarting the Microsoft Antimalware Service.

Solution 3: Use a script that runs the command line program sc.exe
to automate the restart of the Microsoft Antimalware Service.  To
try sc.exe use Run -> Cmd to get to the command line.

Type with a return after each line:
sc stop msmpsvc
sc start msmpsvc

This is the equivalent of using the restart from within Control Panel ->
Administrative -> Services.  The name msmpsvc can be found from
viewing the properties of Microsoft Antimalware Service in Services.
The sc.exe sends a request to Microsoft Antimalware Service to stop or
start, which takes 1 or 2 seconds.  So, a delay is required between
requests.  If a script is used then the delay has to be added.

I used the free program AutoHotkey from www.autohotkey.com to
write a script.  This script can be placed in Startup to restart the
Microsoft Antimalware Service each time MSE loads.  The MSE
icon should then be green.

Example script for AutoHotkey made with Notepad:
Filename: Startup1.ahk

; Title: Restart the Microsoft Antimalware Service
;          for Microsoft Security Essentials.
;
; Notes: If the MSE icon is currently red then the icon should turn green.
; If the MSE icon is green then MSE should flash a stopped message
; and turn green again.
;---------------------------------------------------
; Sleep, 1000 is a delay of  1 second.

Sleep, 3000
Run, sc stop msmpsvc, , hide
Sleep, 3000
Run, sc start msmpsvc, , hide
Sleep, 3000
;---------------------------------------------------
; End
;---------------------------------------------------

The first sleep allows MSE to load and the icon to appear (red or green).
The second sleep allows the stop to be processed.
The third sleep allows the start to be processed.
The hide parameter prevents the command line window from flashing
on the desktop when the script runs.

This script can also be placed on the Desktop and clicked on to
manually restart the Microsoft Antimalware Service.

Solution 4: Microsoft can include an advanced option in the MSE gui
to restart the Microsoft Antimalware Service when MSE loads.
------------------------------------------------------------------

It's not an issue of "sync" -- the service is *part* of MSE.
The issue is that something is preventing the service to start in the allotted time at Windows startup. Using SharePoint Online As a Collaborative, Archiveable Notebook

I'm a food scientist.  I'm also a "power user" of SharePoint, but I don't have any formal training in computer science.

 

My old "information storage and retrieval system" was a physical notebook in which I would jot notes regarding projects and insert articles, emails, and hard copy printouts of Word, Excel, and PowerPoint documents.  The notebook ended up looking like a scrapbook.  When it was full, I'd tuck it away into a box with a table of content of the projects contained inside pasted on the box and I'd forget about it until the project I worked on became resurrected 3 years later because an exec had the idea to make a new product... which matches the product I made 3 years ago.  I'd have to remember where it was, get it out of storage, and then review the notes at that time and make a new project... in a new scrapbook/notebook that has room.  I went from "just a food scientist" to "food scientist/project manager", and I'd have to collaborate with 10 other people on the project to commercialize it, and they were in different parts of the country. 

 

Then I discovered SharePoint online.  It's a very powerful solution to my many needs. I needed to:

  1. Communicate to a controlled group of people.
  2. Separate projects so that groups of people weren't spammed with information about other projects.
  3. Write out and assign tasks.  Have those tasks be formatted to build a timeline.
  4. Share files and concurrently update the files.
  5. Allow for online and offline access to information and files.
  6. Make all information searchable.

SharePoint Online met these needs, and I was able to use the Collaboration/Team Site template to make a sub-site for each and every project that I'm collaboratively working on.  I have a timeline (with deadlines and tasks), newsfeed, documents, and notebook associated with each project.  Collaborating with others means that there are valuable conversations being created, notebooks being filled with information, documents created, and documents concurrently edited.  SharePoint has effectively replaced my notebook/scrapbook and has added value through the ability to search the information generated with each project and has added the collaborative capability to creating and maintaining documentation related to the projects.

 

What I haven't been able to do, so far with my sites, is archive them somehow.  I go through 40-50 projects a year.  Not all remain active during the entire time.  Sometimes, they are commercialized or they are killed.  The information generated from each one is valuable to us and needs to be able to be accessed (like my old notebooks of the past) for the information stored within.  This leads me to my problem: in about 5 years, I'm going to have ~250 subsites with lots of information that is going to be very difficult find a specific active project quickly in the sea of inactive projects.  The user will have to scroll through many pages of sites to eventually find the active project.

 

I've looked into several different options for managing a large number of subsites under the parent site.  I've been using option #1 below, but I've been reading up and finding that there are other options too.

  1. Using naming to designate active/non active projects.  Adding a "z_" to the name shifts the subsite to the end of the list, leaving the active projects at the beginning of the list.  This is the It's been working so far and allows for searching of information.
  2. I read about the ability to move sites via the "Content and Structure" function.  I'm thinking that moving the sites to an "Archived" site via this function would be great and would mean that the sites would still be searchable and the active project site could be de-cluttered.
  3. Using a document center site.  From what I'm reading, there is the ability (records management?) to archive documents into this site via a predetermined “switch” in the settings that will take any document that is X [days, months, years] old and move it to another place. 
  4. Use in-place archiving.  This is basically doing nothing and letting the projects/sites exist in the project site.
  5. Use the quick launch area for the active projects and leave the inactive ones under the “subsites” area. 
  6. 3rd party archiving.  There are a couple of companies that claim that they can archive the information to another site.

I’m using option #1 in combination with #4 and #5.  #2 intrigues me because I’m used to the thought process of physically archiving the notebook/scrapbook and bringing it back when it is necessary.  I’m afraid of any potential copy errors that would degrade information from moving it around.  #3 sounds interesting too, but I’m afraid conversations, timelines, and notebooks would be lost.

Does anyone have any suggestions for options or best practices for using SharePoint as my collaborative notebook/scrapbook and archiving the information for retrieval for a later date?

bob jones aka: cybertaz say:

Well, I hate to suggest that you 'start over' but you might be able to make it easier on yourself :-)

Just about any version of WordPerfect can save in a variety of file types, & RTF is one of the most common denominators. If you can have your Dad save the file in that format you could simply open it with Word to do whatever you intend to do. That may very well eliminate the unwanted breaks unless they were actually created using the return key. IOW, it may be that VueScan is creating the line breaks, so avoiding the scanning could be key -- not to mention avoidance of OCR errors in spelling, etc.

Another thought is that even if you need to scan the hard copy there should be an option in VueScan to save all pages to a single RTF file. That wouldn't eliminate OCR errors but at least you could simply open the RTF with Word in order to revise as necessary. [I'm just lazy, but having to assemble 80 separate files into a single document seems like more trouble than should be necessary :-)]

If you find that you do have to deal with 80 files, copy/paste is about as tedious & slow as you can go. Instead, use Insert> File in Word to insert the RTF content into a new blank Word document... Unfortunately, you can insert only 1 at a time, but it's still faster than copy/paste -- and it may also avoid the breaks if they were generated by the OCR process.

Now, if you can't [or choose not to] do any of the above or still need to remove the breaks, I'm afraid there is no 'single-click' solution. You can't "get rid of all nonprinting characters". The paragraph mark [¶] is just one of several, most of which are necessary (e.g., spaces between words/sentences, tabs) & all paragraphs would run together making it difficult to discern where one is supposed to end & the next one begin.

What you can do is use Word's Edit> Find> Advanced Find & Replace in order to expedite removing the unwanted breaks. In the Find What field use the code ^p which identifies the ¶s in the document. In the Replace With field you'll need to type whatever the ¶ is to be replaced by... it may be a space in some instances or perhaps nothing in others. For the reasons mentioned above I'd suggest that you avoid the temptation to use Replace All. Instead, go from one to the next which also allows you to change the Replace With parameter as necessary rather than having to run the process multiple times for differing instances.

macropod say:

The following macro cleans up text pasted from emails, websites, scanners etc, that insert paragraph breaks at the end of every line. Note that the code assumes there are at least two such paragraph breaks between the 'real' paragraphs. If your document doesn't already have that, you'll need to insert the extra breaks before running the macro.

Sub CleanUpPastedText()
'Turn Off Screen Updating
Application.ScreenUpdating = False
With ActiveDocument.Range.Find
  .ClearFormatting
  .Replacement.ClearFormatting
  .Forward = True
  .Wrap = wdFindStop
  .Format = False
  .MatchAllWordForms = False
  .MatchSoundsLike = False
  .MatchWildcards = True
  'Replace single paragraph breaks with a space
  .Text = "([!^13])([^13])([!^13])"
  .Replacement.Text = "\1 \3"
  'Replace all double spaces with single spaces
  .Execute Replace:=wdReplaceAll
  .Text = "[ ]{2,}"
  .Replacement.Text = " "
  'Delete hypens in hyphenated text formerly split across lines
  .Execute Replace:=wdReplaceAll
  .Text = "([a-z])-[ ]{1,}([a-z])"
  .Replacement.Text = "\1\2"
  .Execute Replace:=wdReplaceAll
  'Limit paragraph breaks to one per 'real' paragraph.
  .Text = "[^13]{1,}"
  .Replacement.Text = "^p"
  .Execute Replace:=wdReplaceAll
End With
'Restore Screen Updating
Application.ScreenUpdating = True
End Sub

For Mac macro installation & usage instructions, see: http://word.mvps.org/Mac/InstallMacro.html

If you'd prefer to run the macro against just a selected range, change:
ActiveDocument
to:
Selection

Here is what I would try:

1)  Open the entire RTF file from the OCR

2)  COPY all of the text

3)  Go to the Word document and Edit>Paste Special>Unformatted text...
