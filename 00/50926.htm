Hi,

Thank you for posting your query in Microsoft Community.

I'm sorry to hear that. I can imagine that must be frustrating.

I would suggest you to try the methods below and check if it helps.

Method 1: Uninstall update

Follow the steps below and check if there is a KB3135173 update installed in the computer. This update has been known to cause this issue sometimes.

1. Press Windows key + R to open the run command.

2. Type appwiz.cpl and press enter.

3. Click "view installed updates".

4. Locate the KB update and uninstall it if its installed.

Method 2: Re-assign default apps manually

As the app default was restored to default, you will have to manually re-assign the preferred default apps again. Follow the steps given below:

1. Press Windows key + I to open settings.

2. Click system > default apps

3. Re-assign the apps accordingly.

Hi
Thanks for contacting Microsoft Community.

Windows, while shutting down, checks for any open programs. If there is any, it sends a request to the program to close and wait till the program closes safely, to prevent any loss of data. But, Windows waits for some specified 

If you are sure that none of the programs are visibly open, you can try the following steps:
Press Windows key+R. Type regedit and pres enter. If UAC prompts, click Yes. Go to HKEY_LOCAL_MACHINE->expand System->expand CurrentControlSet->click Control (do not exapnd this). When you click (or select) Control, you can see WaitToKillServiceTimeout. Double click it to edit the value. Set the value to 500 or less than that. Click Ok and close the Registry Editor. Now restart the PC. If the issue persists, restart again and check.

Be careful while accessing registry as incorrect values/deleting any unknown keys may prevent Windows from booting and will require repair.
