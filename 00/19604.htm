a. user say:

Hello RPJ3337:

Thank you for posting to the forum regarding the question you have about changing SD cards.  From my findings and experience there is not going to be an easy way to do this.  There is not going to be a way to move these apps from one card to the other, at least in a working order.  That leaves you with essential two ways to get these to the new card.

  1. You can move the apps from the card to the phone storage and then remove the card, enter the new card, format the card and move the apps to the new card. 
  2. If you cannot move all your apps to the phone storage the next option is a bit more labor intensive but will get the job done.  You can move one or two apps to the phone storage.  Safely remove the current card, http://www.windowsphone.com/en-us/how-to/wp8/phones-and-hardware/use-an-sd-card-to-add-space .  Add the new card to the phone and format this card.  Move the apps on your phone to the card and safely remove the new card.  You can then repeat the above steps until you have all the apps transferred from one card to the other card.

Please let us know if you have any other questions or concerns we can assist you with.

Hi Roney,

 

My assumption would be that when you secured your database you either were too restrictive with your permissions and have locked yourself out of the database or Access is not connected to the proper .mdw file when you are opening the file.

 

If you used the wizard to secure the database, which option did you choose when it asked what you wanted to do with the new workgroup information file that was being created?

 

+>I want to make this my deafault workgroup information file.

+>I want to create a shortcut to open my security-enhanced databse.

 

If you choose the second option, then make sure you are opening your database via the shortcut that was created.  If you chose the first option, then I would check to ensure Access is using the proper .mdw file.  You can go through the steps in the KB article below to make sure it is connected to the proper .mdw file.

 

918583            How to use the Workgroup Administrator utility in Access 2007

http://support.microsoft.com/default.aspx?scid=kb;EN-US;918583

 

Note:  If you link Access itself to this .mdw file, then every database you open on this machine will prompt you for a username and password.  If you only want to open specific databases with this .mdw file, then you need to create a shortcut that points to the custom .mdw file to open the file.  The article below explains this.

 

305542            Understanding the role of workgroup information files in Access security

http://support.microsoft.com/default.aspx?scid=kb;EN-US;305542

 

If the above steps don’t help, then are you able to open the backup unsecured copy of the database that the wizard creates when securing the file?

What happened to user-level security?  - http://office.microsoft.com/en-us/access-help/what-happened-to-user-level-security-HA010234561.aspx
