bbao say: Do you know a "hidden" E72 feature under Menu > Log? Simply pressing the Right key may allow you to see the full history of voice calls, SMS, and of course data traffic whatever it is WLAN or packet data. The details for data traffic include Access Point (AP) name, direction, type, duration and data usage (sent and received).

I know you are looking for a way to monitor data usage per app, not for all apps in total. I believe base on the above feature, after properly configuring AP settings, you can get what you expect in a different way without running any 3rd-party app.

The trick is to define a new AP dedicated for the app that you want to monitor, and of course the app should be configured to use this new AP only. The AP can be a packet data AP, or a WiFi AP if appicable. The new AP may share the same settings as other workable APs, it actually just need a unique name.

This method does have limits. For example, it is ideally for you to monitor a specific single app, not for monitoring all apps as obviously you can't define a unqiue AP for each app.

Excel 2010 - Complex formulas: Need to be able to "cap" - Microsoft Community

istumped say:

Excel 2010 - Complex formulas: Need to be able to "cap" cell result @ 100% if value is over 100%.

I have the following formula in a cell that is accurately returning a value that is over 100%.

 

=IF(ISERROR(SUMIF('Customer Item Detail'!$A:$A,$B4&$C4&I$1,'Customer Item Detail'!$AE:$AE)/SUMIF('Customer Item Detail'!$A:$A,$B4&$C4&I$1,'Customer Item Detail'!$AC:$AC)),0,SUMIF('Customer Item Detail'!$A:$A,$B4&$C4&I$1,'Customer Item Detail'!$AE:$AE)/SUMIF('Customer Item Detail'!$A:$A,$B4&$C4&I$1,'Customer Item Detail'!$AC:$AC)).  

 

In this spreadsheet, if the value returned is over 100%, we only want the cell to show that the error percent is 100%.  How can I modify the formula to "cap" the value at 100%?

ashish mathur say:

Hi,

 

If the user/recipient of this file is a Excel 2007 or higher version user, then you may use this shorter formula

 

=IFERROR(MIN(1,SUMIF('Customer Item Detail'!$A:$A,$B4&$C4&I$1,'Customer Item Detail'!$AE:$AE)/SUMIF('Customer Item Detail'!$A:$A,$B4&$C4&I$1,'Customer Item Detail'!$AC:$AC)),0)

 

Format this cell as a % age.

 

Also, as a good practice, avoid giving entire columns as references in your formula.  So instead of A:A, AC:AC and AE:AE, use the explicit data range instead i.e. A2:A100, AC2:AC100 and AE2:AE100

 

Hope this helps.

jeeped say:
The IFERROR() wrapper catches errors such as #DIV/0! (division by zero) and returns an alternate value. In the case of the formula posted in your last reply, this alternate value would be 0. This is easily changed to 1 so that if the denominator is 0 then the result would be 1.

=IFERROR(MIN(1,SUMIF('Customer Item Detail'!$A:$A,$B10&$D10&F$1,'Customer Item Detail'!$X:$X)/SUMIF('Customer Item Detail'!$A:$A,$B10&$D10&F$1,'Customer Item Detail'!$V:$V)),1)

Taking care of a 0 for a numerator is a bit trickier but not impossible. The result of the numerator SUMIF() can be compared to zero and if equal then the denominator can be substituted. This would have the effect of returning 1 as well.

=IFERROR(MIN(1,IF(SUMIF('Customer Item Detail'!$A:$A,$B10&$D10&F$1,'Customer Item Detail'!$X:$X),SUMIF('Customer Item Detail'!$A:$A,$B10&$D10&F$1,'Customer Item Detail'!$X:$X),SUMIF('Customer Item Detail'!$A:$A,$B10&$D10&F$1,'Customer Item Detail'!$V:$V))/SUMIF('Customer Item Detail'!$A:$A,$B10&$D10&F$1,'Customer Item Detail'!$V:$V)),1)

Does that get things working according to plan?
