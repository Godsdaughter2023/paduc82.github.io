ken sheridan say: There are a number of points you need to cover when doing this:

1.  The main form's current record must be saved before you add a record in one of the linked forms or referential integrity will be violated.  You can do this by setting the form's Dirty property to False before opening the linked form.

2.  The linked form's underlying table should include a ClientID column as a foreign key, but this must not be an autonumber (the Clients table's primary key ClientID can, and normally would be an autonumber, however).

3.  The linked form's underlying table must not include LastName or FirstName columns; this would introduce redundancy and the possibility of update anomalies.  You can show the values of the LastName or FirstName columns in the linked form from the matching row of the Clients table instead.  To do this, in the linked for use a combo box for as the control bound to the ClientID column and set it up as follows:

Name:    cboClient

ControlSource:    ClientID

RowSource:     SELECT ClientID, FirstName & " " & LastName FROM Clients ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column and that the second is at least as wide as the combo box.

4.  When you open the linked form from the main form you filter the linked form to the current client, and pass the current ClientID value to it via the OpenArgs mechanism, so that it can be used by the linked form as the DefaultValue property of the cboClient combo box.  I'd also suggest that you open the linked form in dialogue mode; this will force the user to close it (or strictly speaking at least hide it) before returning to the main form.  So, the code in the Click event procedure of a button to open a linked form named frmMyFirstLinkedForm would be as follows:

Const FORMNAME = "frmMyFirstLinkedForm"
Const MESSAGETEXT = "No current client record."
Dim strCriteria As String

strCriteria = "ClientID = " & Me.ClientID

' be sure a record has been entered
If Not IsNull(Me.ClientID) Then
    ' save current client record
    Me.Dirty = False

    ' open linked form in dialogue mode
    DoCmd.OpenForm FORMNAME, _
        WhereCondition:=strCriteria, _
        WindowMode:=acDialog, _
        OpenArgs:=(Me.ClientID)
Else
     MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"
End If

5.  You now need to put code in the linked form's Open event procedure to set the DefaultValue property of the cboClient control:

If Not IsNull(Me.OpenArgs) Then
    Me.cboClient.DefaultValue = """" & Me.OpenArgs & """"
End If

Note that the DefaultValue property is always a string expression, regardless of the data type of the column to which the control is bound, so should be wrapped in literal quotes characters as above.

If you decide not to open the linked form in dialogue mode by omitting the WindowMode:=acDialog, _ line, which would enable you to move focus between the open forms, you should refilter each linked form and change their ClientID's DefaultValue setting in the main form's Current event procedure with:

' ignore error if linked form not open
On Error Resume Next

frmMyFirstLinkedForm.Filter = "ClientID = " & Nz(Me.ClientID,0)
frmMyFirstLinkedForm.cboClient.DefaultValue = """" & Nz(Me.ClientID,0) & """"

frmMySecondLinkedForm.Filter = "ClientID = " & Nz(Me.ClientID,0)
frmMySecondLinkedForm.cboClient.DefaultValue = """" & Nz(Me.ClientID,0) & """"

frmMyThirdLinkedForm.Filter = "ClientID = " & Nz(Me.ClientID,0)
frmMyThirdLinkedForm.cboClient.DefaultValue = """" & Nz(Me.ClientID,0) & """"

frmMyFourthLinkedForm.Filter = "ClientID = " & Nz(Me.ClientID,0)
frmMyFourthLinkedForm.cboClient.DefaultValue = """" & Nz(Me.ClientID,0) & """"

And you should also put exactly the same code in the main form's AfterInsert event procedure.
