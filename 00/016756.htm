UDF Not Working Properly - Count Comments

    z = z + 1
  Next
  CountCmts = z
End Function

Any reason / flaw in the same?
rick rothstein, mvp - excel say:
As Gary's Student said, SpecialCells does not work correctly when used in a UDF; however, if you really want a UDF for this functionality, you can do it by attacking the problem completely differently. This function can be called either by other VB code or directly from a worksheet as a UDF...

Function CountComments(Rng As Range) As Long
   Dim Cell As Range
   Application.Volatile
   For Each Cell In Rng
     CountComments = CountComments - (Not (Cell.Comment Is Nothing))
   Next
 End Function

Note: When used as a UDF, this function will not update automatically as inserting or deleting comments does not force the sheet to recalculate. So, to update the cell after inserting or deleting a comment, you will have to press F9 (or wait until some other action forces the sheet to recalculate).


The above code, which I posted earlier, will return the number of comments in the specified range whether the parent cell for a comment is hidden or not. In looking at the code you tried and the code Gary's Student posted, I realize you wanted the count of the number of comments only for cells that are visible. Here is the above code modified to do that...

Function CountComments(Rng As Range) As Long
   Dim Cell As Range
   Application.Volatile
   For Each Cell In Rng
     CountComments = CountComments + (Not (Cell.Comment Is Nothing)) * _
                                                                   (Not Cell.EntireRow.Hidden)
   Next
 End Function
