If (for example) your system is running chkdsk on every reboot, that means that your HDD has been marked "dirty" and until you clear the dirty bit on the volume, XP is automatically going to run chkdsk on every reboot to try to clear it - but sometimes that won't do it.  Sometimes the automatic chkdsk will not clear the dirty bit.

No amount of registry restoration is going to fix that.

You also do not describe any symptoms consistent with a need to implement KB307545, and that will not fix your chkdsk problem anyway (assuming you make it all the way through it without making things worse).

A sure way to set the dirty bit on a volume is to use the power button or reset button when you think XP is hung but sometimes you gotta do what you gotta do.

The only thing that can clear the dirty bit is a clean run of chkdsk with error correction and sometimes you are going to have to do that from the XP Recovery Console or a Hiren's boot CD.

You can query the dirty bit on a volume from the command prompt window.

Click Start, Run and in the box enter:

cmd

Click OK to open the command prompt window.

To query the dirty bit on drive C, type:

fsutil dirty query C:

Sample output:
Volume C: is dirty
Volume C: is not dirty

Type 'exit' to close the command prompt window.

If the volume is dirty, chkdsk is going to run again when you reboot - that is what it is supposed to do...  XP wants to try to fix the problem.  You can tell XP NOT to check the dirty bit, but wouldn't you rather really fix it if you can?
