Best way to input weather delays in Project 2010

Hi all

To get an accurate idea of the schedule I have to input weather delay (too cold, rain etc) days in the Project schedule, what is the best way:

dale howard [mvp] say:

JL --

I really like this question, and I thank you for asking!  :)

Although I am not a big fan of putting tasks into a project that are not real tasks to be performed by your team members, I could see a real possibility in using tasks for weather delays.  If the weather delay occurs after one task is completed and delays the start of its Successor task, add a new task named Weather Delay, and then link it with the two tasks.  If you do add a task for a weather delay, I would recommend that you apply cell background formatting to the weather delay task to highlight it.  For example, I might use a light red color to make the task really stand out in the project.

If a weather delay causes a work stoppage in the middle of an in-progress task, you can apply a task split during the time period for the weather delay.  Start the task split on the date the weather delay begin, and end the split on the date that work resumed.  For example, when I used to live in Florida, where a tropical storm stopped construction work in the area for a full week.  If I were a PM managing a construction project at that time, and had an in-progress task that was occurring during the tropical storm, I would introduce a 5-day split, beginning when the storm came ashore and ending when construction work was able to resume (when the sun came out and the rain water drained away).

In either situation, I would strongly recommend that you add a note to the weather delay task or to the split task, documenting the reason for the delay.   Hope this helps.

rob koch say:

On the Power Options control panel page select the "High Performance" power plan, which may be hidden under the Show Additional Plans selection on that page.  This plan is set to never sleep the PC by default, so that should remain running until you re-select a different plan to return to normal.

If this still isn't working, then there's a possibility the PC may be shutting down to an overheat condition, which if the fans start running at high speed shortly after starting a scan might be the likely issue.

You mentioned PC, but not whether this device is a laptop or desktop, though the latter seems implied.  Overheat problems are more common for a laptop, though they can happen to either.

There is no "best" antivirus/ antimalware, since this is always inherently someone's opinion related to their own usage patterns and experience.  What matters is what you prefer in terms of notifications, technical information displayed or required to configure and simply comfort level with the program you use.  Others will likely provide links to possible aid with choosing a program, but don't ever believe anyone who tells you there's an absolute best, since that indicates a lack of understanding of the above.

I'm not clear what your final sentence means, since I've never heard of "kill the page" so be careful to always use exact error messages or other textual information including context when asking technical questions in a forum, such as specifically what application you meant by the "Internet".

Rob

greginmich say:

Full Scans can run for a very long time if they have a lot of ground to cover (a large number of files to scan), or in cases where the Antimalware engine is preoccupied with rebuilding the cache of trusted files after an OS update. Since there’s no way to keep the scan running when the PC is in a sleep state, you have to either manually adjust the sleep timeout in Power Options to accommodate the length of the scan (and then set it back to the default afterwards if you're conscientious about saving energy) – or – run the scan with a batch file that automates the power configuration changes, as described in this thread:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-can-i-run-windows-defender-walk-away-and-have/7b57798e-b2da-40a6-9c74-afd9ff2dff11

 

Or schedule your Full Scan in Task Scheduler and have the scheduled task automatically control the power configuration and the shutdown. In Version 1607, you currently have to skip step 7, because running the task under the system alias will cause it to fail:

https://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02?page=1

 

 

GreginMich
