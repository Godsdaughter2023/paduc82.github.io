jackthedog say:

Using Mail Merge for 20+ Page Contract

Our agency prepares 20+ page contracts with around 25 pieces of unique information found in 90+ locations throughout the document which is currently a moderately heavily formatted Word doc.  The unique information is available in existing Access tables.  It looks like exporting a query through Mail Merge is the best option, but it seems a little clumsy to me, because we only do contracts for one client at a time.

Is there a better solution?

tom van stiphout say:
You could use Automation instead, but I would not. MailMerge is well supported in Word, is easy to use by any Word power user, and it's the perfect solution for you.
-Tom. Microsoft Access MVP

graham mandeno say:

Hi Jack (?)

One method is to use bookmarks in your Word document template at the locations where the Access data is to be inserted.  You can then write some code to open a new documnt based on the template and find each bookmark in turn, replacing it with the data.

This is fiddly for a number of reasons.  One is that the bookmark names have to be known by the Access application and therefore need to be either hard-coded or stored in a table.  Another is that bookmarks are normaly invisible when editing a Word document, so it is difficult to see where the place-holders are.

Because of this, I only use the bookmarks method where a mail merge cannot wrk - for example, when I want to insert records from a subquery in a table in the Word document (for example, the items in an invoice).

However, the "classic" Word merge where the merge master document is linked to an Access query is also cumbersome, especially if your query is filtered for one record, for example.

The solution I use is to create a temporary delimited text file containing the merge data, then open the merge master document (through code) and set the merge DataSource on the fly to the temp file.  Then execute the merge (to printer, or to a new document) and finally close the merge master without saving changes.

MVP Albert Kallal has some very similar code here: http://www.members.shaw.ca/AlbertKallal/msaccess/msaccess.html

Scroll down to "Super Easy Word Merge", about half-way down the page.

Good luck!

Graham Mandeno [Access MVP]
