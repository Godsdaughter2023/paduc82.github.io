theloniousm say:

After further exploration, I've found it only happens when the sheet is protected.  Whenever I unprotect it, the edges of the sheet become normal like in any other sheet.  Does that make any sense?  Thanks!

bill manville say:

Yes, that makes a lot of sense.

When you protected the sheet you must have indicated that you would prohibit selection of locked cells.

So when you use an arrow key it takes you to the next unlocked cell in the direction you chose.

If you don't need this restriction you could remove it by unprotecting the sheet and then re-protecting allowing selection of locked cells. 

Otherwise you could lock the cells that you don't want the user to access.

If you have a macro that needs to write to those cells (which is why you left them unlocked) you could change the macro to protect the worksheet using UserInterfaceOnly:=True.  That allows your code to do most things on the worksheet while protecting it against the user.  The UserInterfaceOnly attribute is forgotten when the file is closed so you would need to set it up again in the Workbook_Open event.

Sub Test()
   MsgBox ServiceStatus("BITS")

   StopService "BITS"
   Application.Wait Now + 5 / 86400
   MsgBox ServiceStatus("BITS")

   StartService "BITS"
   Application.Wait Now + 5 / 86400
   MsgBox ServiceStatus("BITS")
End Sub
 
Sub StartService(service As String)
   Dim svc As Variant

   For Each svc In GetObject("winmgmts:{impersonationLevel=impersonate}!\\.\root\cimv2") _
         .ExecQuery("SELECT * FROM Win32_Service WHERE Name='" & service & "'")
       svc.StartService
      Exit For
   Next
End Sub
 
Sub StopService(service As String)
   Dim svc As Variant

   For Each svc In GetObject("winmgmts:{impersonationLevel=impersonate}!\\.\root\cimv2") _
         .ExecQuery("SELECT * FROM Win32_Service WHERE Name='" & service & "'")
       svc.StopService
      Exit For
   Next
End Sub
 
Function ServiceStatus(service As String) As String
   Dim svc As Variant

   For Each svc In GetObject("winmgmts:{impersonationLevel=impersonate}!\\.\root\cimv2") _
         .ExecQuery("SELECT * FROM Win32_Service WHERE Name='" & service & "'")
      ServiceStatus = svc.State
      Exit For
   Next
End Function

I don't have a clear picture of what your named ranges "RefersTo" values look like, but perhaps this code will help you get started, or you can provide more details later and we can refine the code.

This code would go into the code module for the sheet that has your cities lists on it.  To get to that sheet's code module, simply select the sheet and right-click on its name tab and choose [View Code] from the popup list.

Private Sub Worksheet_Change(ByVal Target As Range)
  If Target.Cells.Count = 1 And Target.Row = 1 And _
   Target.Column = UsedRange.Columns.Count And _
   IsEmpty(Target.Offset(1, 0)) Then
    'a change was made to a single cell in row 1 and
    'it is in the right-most used column on the sheet and
    'the cell below (row 2) is still empty then
    'we assume it is a new city name entry
    '
    'the basic code to add a named range to the workbook
    'using the active cell and it's contents
    ActiveWorkbook.Names.Add Name:=Target.Value, _
     RefersTo:="=" & Me.Name & "!" & Target.Address
  End If
End Sub

Here's one way but it doesn't actually create a named range for each city. There's just 2 named ranges, one for the source for the city names list and one for the source for the client names list.

Let's assume you use Sheet1 as the source location for the list data.

In row 1, A1:C1 you have the city names listed:

City1 City2 City3

Create a dynamic range for these city names as:

  • Name: CityList
  • Refers to: =Sheet1!$A$1:INDEX(Sheet1!$1:$1,COUNTA(Sheet1!$1:$1))

So, on Sheet1 you also have the client names under each city name:

City1 City2 City3
Smith Jones Black
Brown King
Biff

Let's assume that you will never have more than 10 clients in any one city.

Create this dynamic range for the client names:

  • Name: ClientList
  • Refers to: =OFFSET(Sheet1!$A$2,,MATCH(Sheet2!$A$1,Sheet1!$1:$1,0)-1,COUNTA(INDEX(Sheet1!$2:$11,,MATCH(Sheet2!$A$1,Sheet1!$1:$1,0))))

On Sheet2 in cell A1 you want the drop down for the the city names.

Set up the validation list and as the source use this formula:

=CityList

On Sheet2 in cell B1 you want the drop down list of client names associated with the city selected in cell A1.

Set up the validation list in cell B1 and as the source use this formula:

=ClientList

As you add new cities/names to Sheet1 they will automatically get "picked up" in the validation lists.

jllatham say:

Here is a variation of the code (goes into the same place, would be used instead of the previous code) that might serve you better.  The comments explain it fairly well - it won't create a new named range until there is at least one entry under a city name and then it will create a named range using the city entry in row 1 and the named range will refer to the column beginning at row 2 and continuing down the column to the last entry in that column.  If the named range already exists, then its "RefersTo" is updated.

Private Sub Worksheet_Change(ByVal Target As Range)
  'this code will create or revise the named range
  'for a city under these conditions:
  ' a change to a single cell on the sheet was made
  ' there is an entry in row 1 of the column the
  ' change was made in AND there is at least one
  ' entry in a row below 1.
  ' The "RefersTo" for the range will be from
  ' row 2 to the last used row in the column.
  ' A name will not be created until there is
  ' at least one entry below the city name in row 1.
  '
  Dim lastRow As Long
  If Target.Cells.Count = 1 And _
   Not IsEmpty(Cells(1, Target.Column)) Then
    lastRow = Cells(Rows.Count, Target.Column).End(xlUp).Row
    If lastRow > 1 Then
      ActiveWorkbook.Names.Add Name:=Cells(1, Target.Column).Value, _
       RefersTo:="=" & Me.Name & "!" & Cells(2, Target.Column).Address & _
       ":" & Cells(lastRow, Target.Column).Address
    End If
  End If
End Sub
