DISCLAIMER:

Modifying REGISTRY settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of REGISTRY settings can be solved. Modifications of these settings are at your own risk.

1.       Click Start

niton_ say: Michael Bauer provides this VBA solution. (Also useful when you find an item in a search and want to know the folder, with full path, where the item is stored.)

Folder: Get the path  http://www.vboffice.net/sample.html?lang=en&mnu=2&smp=65&cmd=showitem

Public Sub GetItemsFolderPath()
    Dim obj As Object
    Dim F As Outlook.MAPIFolder
    Dim Msg$
    Set obj = Application.ActiveWindow
    If TypeOf obj Is Outlook.Inspector Then
        Set obj = obj.CurrentItem
    Else
        Set obj = obj.Selection(1)
    End If
    Set F = obj.Parent
    Msg = "The path is: " & F.FolderPath & vbCrLf
    Msg = Msg & "Switch to the folder?"
    If MsgBox(Msg, vbYesNo) = vbYes Then
        Set Application.ActiveExplorer.CurrentFolder = F
    End If
End Sub

#1 - To be honest - been a long time since I've done a side-by-side install of Office versions and the last one was with OL'2013. The following MS article  covers that topic

Install and use different versions of Office on the same PC
https://support.office.com/en-us/article/Install-and-use-different-versions-of-Office-on-the-same-PC-6ebb44ce-18a3-43f9-a187-b78c513788bf

#2 - If your data is stored in a PST file that is currently configured in OL'2007 - it will be used by OL'2016. What happens when you install OL'2013/'2016 is that a copy of the OL'2007 profile is automatically made for use by '2013/'2016. Once that's done, making any changes to an OL'2007 Outloook profile will NOT show up in the equivalent '2015/'2016 profile (and vice versa). To be clear about something, installing OL'2013/'2016 doesn't "import" anything - what it does is use the existing settings related to <Email Accounts> and <Data Files> found under <Account Settings> - difference being the IMAP OST file(s) as noted earlier.

As an aside - generally no need to "import" a PST file since it can be opened directly in any version of Outlook.
