File History has a predefined set of items that will be backed up automatically: all your libraries (both default libraries and custom libraries you created), the Desktop, your Contacts, Internet Explorer and Edge favorites and, finally, OneDrive. The idea behind File History is that it allows you to retrieve earlier versions of a particular file. a. user say:

The client uses its own port 68 as the source port with port 67 as the destination port on the server to send the request to the server. The server uses its own port 67 as the source port with port 68 as the destination port on the client to reply to the client. Therefore the server is listening and sending on its own port 67 and the client is listening and sending on its own port 68. This can be confusing when you consider which way the message is going. To be clear on this, I quote RFC 1531 which states "DHCP messages from a client to a server are sent to the 'DHCP server' port (67), and DHCP messages from a server to a client are sent to the 'DHCP client' port (68)"

http://www.comptechdoc.org/independent/networking/guide/netdhcp.html

This thread may help

 

1. Find out your IP address e.g. go to a website like http://www.whatismyip.com/ it will show you your IP.
2. Copy It.
3. Open Network & Sharing Center
4. Open the Properties of the network you what your IP to change. In my case it is a EVDO modem.
5. On the Networking tab double-click TCP/IPv4 which will open a window to edit IP and DNS IP address.
6. On this window Click on "Use the following IP address" and paste that IP you copied at Step 2.
7. Change the last digit of the IP to any value above it e.g. if its 192.168.23.123 edit the last digit to something like 122 so that the IP becomes 192.168.23.122.
8. Click OK until all the windows close.
9. Disconnect from that network.
10. Reconnect to that network and if it goes as planned the ISP should force a Dynamic IP. Check http://www.whatismyip.com/ if it has changed. If you cannot browse go to Step 6 and select "Obtain an IP address automatically" then Disconnect -> Reconnect the network.

marilyn o say: Hello RalphBaehr,

If you go to the right top of the screen and your cursor is at the top of the corner, it should display the Charms bar with a transparent background. Then pull your cursor down the right side of the page, the background of the Charms Bar should become black instead of transparent and you will see the clock, day of the week and date on the lower left side.

It takes a little practice to swipe and get the Charms bar to show up. There is a shortcut you can use to bring up the Charms bar. Hit the Windows key + C and it will display the Charms bar.


The Charms Bar should show up the same way as when you are on the Desktop.  

Let us know if this helps or if you are still having trouble displaying the Charms bar.


Sincerely,


Marilyn

Edit 6-4-13:  Verified Cause:  I verified that the issue was adding a KVM using VGA adapters and VGA cable.  With a DVI cable, Windows 8 works without problems.  The cable choice also affects Windows 7.  I have not found a good explanation on the Internet, but will ask NVIDIA for an explanation when following up on my trouble report (NVIDIA GeForce GTX 650Ti and ASUS VE247 monitor).  Windows 7 was tested with NVIDIA GeForce 8800 GT and ASUS VE247 monitor, so I think it is display adapter independent.

 

Edit 6-6-13:  NVIDIA Response:  I received this response from NVIDIA:

 

“Hello Steve,
Thank you for your reply.
Such issues with display may happen if you use KVM switch. KVM switch sometimes blocks the EDID report which is the identity and contains the resolution charts of the display. If this report is not corrected passed or corrupted then the display resolution may get improper. I would recommend you to please use direct connectivity from the graphics card as KVM switch seems to be causing the issue.
We apologize for the inconvenience caused.
Please feel free to contact us for any further queries or issues.
Best Regards,
NVIDIA Customer Care”
