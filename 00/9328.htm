If, else... - Microsoft Community

ceceliafedd say:

If, else...

I am creating a mail merge for resumes and I want a section to collapse/disappear if it does not populate with data...can I do that? 

I am working with Microsoft Word 2010

macropod say:

You can use an IF test coded along the lines of:

{IF{MERGEFIELD MyField}<> "" "Variable text"}

or

{IF«MyField»<> "" "Variable text"}

where 'MyField' is a field in the data source that determines whether the variable text is to be displayed. The variable text can include multiple paragraphs, tables & pictures if need be.

In the above example, it is assumed you want nothing displayed if 'MyField' is empty. Changing the '<>' to '=' inverts the test. Thus, if you only want the variable text to display if 'MyField' has a specified value (e.g. 'MyValue'), you could use a field coded like:

{IF{MERGEFIELD MyField}= "MyValue" "Variable text"}

or:

{IF«MyField»= "MyValue" "Variable text"}

Note: The field brace pairs (i.e. '{ }') for the above example are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues. Likewise, you can't type or copy & paste the chevrons (i.e. '« »') - they're part of the actual mergefields, which you can insert from the mailmerge toolbar. The spaces represented in the field construction are all required.

calculate compound interest on 100000 @28% pa for 32 months - Microsoft Community

abhik ranjanthakur say:

calculate compound interest on 100000 @28% pa for 32 months compounded annually in excel 2007?

In excel 2007 when I am calculating the CI using FV formula =FV(28%,32/12,0,-100000) the result is 193149.35. But when I am calculating the same using manual CI formula =100000*(1+0.28)^2*(1+8/12*0.28) the result is 194423.4667. why the two is not same? Which one should I take?

vijay a. verma say:

I have provided the explanation in your original thread. Your compounded interest balance formula is wrong.

Compounded Balance = Principal x (1+rate)^N

Hence, your formula has to be 

=100000*(1+0.28)^(32/12)

Now you will get same result.

joeu2004 say:

Abhik RanjanThankur wrote:

In excel 2007 when I am calculating the CI using FV formula =FV(28%,32/12,0,-100000) the result is 193149.35. But when I am calculating the same using manual CI formula =100000*(1+0.28)^2*(1+8/12*0.28) the result is 194423.4667. why the two is not same? Which one should I take?

Your formula assumes that simple interest is earned throughout the year, then

That is certainly one possible interpretation of "compounded annually".

In contrast, the FV formula assumes, in this case, that interest compounds monthly at the rate (1+28%)^(1/12) so that the annual yield is 28% compounded annually.

That formula can be written as:

=100000*(1+28%)^2 * (1+28%)^(8/12)

But as Vijay notes, that is algebraically equivalent to the simpler formula:

=100000*(1+28%)^(32/12)

Note that that might be different from 28% compounded monthly.  In some countries, that results in a monthly rate of 28%/12 and an annual yield of (1+28%/12)^12.

Only you can answer the question:  "which one should I take?".  It depends on a careful understanding of the terms of investment.

Rate = Annual Rate / 12

pmt = payment you will make in the end of each period (assume type is omitted), i.e. 0 in your case

Is it possible to prevent duplicates in Excel, including special - Microsoft Community

heatley999 say:

Is it possible to prevent duplicates in Excel, including special characters

I have been using the following formula in 'data validation' (custom) to prevent duplicate text entries in a particular column in excel:

=COUNTIF($C:$C,C1)=1

This has worked well preventing a duplicate entry of, for example, 'abc'.

However, if I were to attempt to enter 'a?c' into the column that already has 'abc' as an existing entry, the formula appears to recognise this as a duplicate.

Does anybody understand why this is happening, and do you have a solution that would work even if special characters such as '?' are used?

Many thanks

ossiemac say:

The reason it occurs is that both the asterisk and question marks are wild cards and Conditional formatting recognizes wild cards as a match.

A preceding tilde (~) with a wild card disables the wild card. Therefore need a formula that inserts the tilde just for the calculation without changing the value in the worksheet.

Try the following. Note the test for blank or it formats all of the blanks. I initially tested with > 1 because I thought that you wanted to format the duplicates where as  = 1 will format just the unique values so I have include both the formulas.

Format the duplicate values:

=AND(NOT(ISBLANK(C1)),COUNTIF($C:$C,SUBSTITUTE(C1,"*","~*"))>1,COUNTIF($C:$C,SUBSTITUTE(C1,"?","~?"))>1)

Format the unique values:

=AND(NOT(ISBLANK(C1)),COUNTIF($C:$C,SUBSTITUTE(C1,"*","~*"))=1,COUNTIF($C:$C,SUBSTITUTE(C1,"?","~?"))=1)
