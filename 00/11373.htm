Even Outlook 2010 didn´t support categories in an IMAP account. What´s new since Outlook 2013 is that the data file for the IMAP account can (locally) store contacts, calendars etc. whereas that file in previous Outlook versions only held emails; for all other item types another PST file was used.

You can get this behaviour back by adding another data store via account settings, and make that store the default data store.

If you don´t want the other items in a second file, or if you also want categories for emails in the IMAP account, try Category-Manager:

To install Office applications without signing in to Office 365 portal, you can use Office 2016 Deployment Tool as you mentioned in your original thread Use the Office 2016 offline installer. You can follow the steps below to install the Office 365 through Office 2016 Deployment Tool.

<Configuration>
