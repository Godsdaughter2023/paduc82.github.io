a. user say:

Renaming your wuaueng.dll does not fix the problem and will break your Windows Update Service.

Renaming the wuaueng.dll fixes the symptom of the problem and now you have broken the Windows Update Service.

Fixing the problem and fixing the symptom of the problem are not the same thing.

You will see many svchost.exe Processes running in Task Manager and under each of those will be one or more important Windows Services.   It is normal to see several svchost.exe Processes running in Task Manager and that does not mean there is a problem.  Important Windows Services run under each svchost.exe Process you see.  There might be 6 or 8 or 10 svchost.exe Process running...  it depends on how the system is configured.

When you see a svchost.exe process consuming too much CPU the problem is (usually) not with the svchost.exe Process itself - the problem is what is running under that svchost.exe Process and you can figure out what that is in several ways.

It is alleged that sometimes malicious software can disguise itself as a svchost.exe Process and show up in Task Manager so assuming your system is not afflicted with malicious software you can figure out what is going on.

The Windows Update Service runs under one of the many svchost.exe Processes you see in Task Manager. 

You can notice the PID column of the afflicted svchost.exe Service in Task Manager, right click the afflicted Process and choose Services (Process ID PID 1044 is running amok in this bad example):

To see the Windows Services that are running under that svchost.exe process, right click the offending Service and choose Go to Services:

Then in the Services Tab, click the PID column to sort by PID and you can see all the important Windows Services that are running under that svchost.exe process - but that still doesn't show you the one Service that is using so much CPU:


To see which Service is using the most CPU click the Performance tab and choose Resource Monitor (you may have to minimize the Task Manager window):

In Resource Monitor expand the Services window, click the PID column to sort by PID and scroll down to the PID of the offending Service and now you can see the CPU column which will tell you how much CPU each Service under that PID is using:
