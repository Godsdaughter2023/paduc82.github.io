Private Declare Function OpenClipboard Lib "user32" (ByVal hwnd As Long) As Long
Private Declare Function EmptyClipboard Lib "user32" () As Long
Private Declare Function CloseClipboard Lib "user32" () As Long
Private Declare Sub Sleep Lib "kernel32" (ByVal dwMilliseconds As Long)

  'Get access to the clipboard
  Do
    Result = OpenClipboard(0&)
    If Result <> 1 Then
      CloseClipboard
      DoEvents
      Sleep 10
    End If
    Count = Count + 1
  Loop Until Count = 10 Or Result = 1
  If Result <> 1 Then _
    Err.Raise 70, "", "Can not open the clipboard"
  EmptyClipboard
  CloseClipboard

Sub SaveVBACode()

'This code Exports all VBA modules
Dim i As Integer
Dim mName As String
Dim Fname As String


    With ThisWorkbook
        For i = 1 To .VBProject.VBComponents.Count
            If .VBProject.VBComponents(i).CodeModule.CountOfLines > 0 Then
                mName = .VBProject.VBComponents(i%).CodeModule.name

Column A is a spool count. 
The first (and only) editable cell in the in the column is cell A6, the remainder of the cells (cells A7+) contain this formula:

for automatically incrementing the spool number if there is a value in column B.

The VLOOKUP returns the value from this column successfully.

Column V is for Employee ID # and is entered manually.
The VLOOKUP returns the value of this column successfully.

Column Q is for total footage run.
It contains a formula that looks like this (as the column goes down the row numbers in this formula increment):

=SUM(B6:P6)

The VLOOKUP formula does not return the value displayed in the last cell with a value in column Q.

Any direction on why the formula is behaving this way and potential alternatives/workarounds are greatly appreciated.
amit tandon say:

Your formula "=INDEX(Q6:Q256, MATCH(0, Q6:Q256, 0)-1)" will return the value of cell which is one above the FIRST zero. If this immediately preceding cell is blank then your formula will return zero.

To return the LAST NON-ZERO value in a column, you may use below formula:

=LOOKUP(2,1/(Q6:Q256<>0),Q6:Q256)

steve_p---01 say: You'll need a 3rd party File Manager Application like FExplorer in order to see all of your phone's system files. If you use PC Suite or the Nokia file manager you will be limited to viewing only a number of certain 'safe' directories. Having said that, the Nokia file managers will prevent you from causing any potential harm to your handset by, say, accidentally deleting a crucial system file.

Preinstalled ringtones are stored in:
Z:\Nokia\Sounds\Digital
Z:\Nokia\Sounds\Simple

Where Z: is FExplorer's reference to the phone's ROM memory.

Preinstalled Themes are located in:

Z:\System\skins

Each theme will be stored in its own directory called something like 101f84b9 or similar. Each directory will contain 3 files, *.mbm, *.mif and *.skn, these 3 files should all have the same name which may or may not be the title of the theme. e.g. Series60Skin.mbm, Series60Skin.mif and Series60Skin.skn

There will also be corresponding ini files stored in the phone's memory: C:\System\skins e.g. 101f84b9.ini in C:\System\skins\101f8b49

Downloaded themes can either be stored in phone memory:
C:\System\skins

or be stored on your phone's removable memory card:
E:\System\skins

In the case of downloaded themes you will only see *.ini, *.mbm and *.skn files, there is no *.mif file.

S.

qryApplicableAdminCosts
SELECT First("7181") AS AccountNo, tblPayProfitCenter.ProfitCenter, Sum([PercentTime]*IIf([payperiod]="week",[salary]*52/12,[salary]*40*52/12)) AS Cost, qryApplicableEmployeesAll2.BudgetMonth
FROM (((qryApplicableEmployeesAll2 INNER JOIN tblJobsDepartment ON qryApplicableEmployeesAll2.JobTitleID = tblJobsDepartment.JobsID) INNER JOIN tblJobsPay ON tblJobsDepartment.JobsID = tblJobsPay.[JobID]) INNER JOIN tblPayProfitCenter ON tblJobsDepartment.Department = tblPayProfitCenter.DeptCode) INNER JOIN tblJobs ON tblJobsPay.[JobID] = tblJobs.ID
WHERE (((tblJobsDepartment.Department)<>"ADMIN") AND ((tblJobsDepartment.Category)="ADMIN") AND ((tblJobsPay.DateEffective)<=[BudgetMonth]) AND ((tblJobsPay.TermDate) Is Null Or (tblJobsPay.TermDate)>[BudgetMonth]))
GROUP BY tblPayProfitCenter.ProfitCenter, qryApplicableEmployeesAll2.BudgetMonth;


qryApplicableCOGS
SELECT tblCOGSPercent.AccountNo, tblPayProfitCenter.ProfitCenter, [Cost]/[LaborPercOfCOGS] AS TotalCOGS, ([Cost]/[LaborPercOfCOGS])-[Cost] AS NonLaborCOGS, qryApplicableDirectLaborCosts.BudgetMonth
FROM (qryApplicableDirectLaborCosts INNER JOIN tblPayProfitCenter ON qryApplicableDirectLaborCosts.ProfitCenter = tblPayProfitCenter.ProfitCenter) INNER JOIN tblCOGSPercent ON tblPayProfitCenter.DeptCode = tblCOGSPercent.Department
WHERE (((tblCOGSPercent.BudgetNo)=[Forms]![frmBudget]![txtBudgetNo]) AND ((tblCOGSPercent.EffectiveDate)<=[tblTempBudgetMonths].[BudgetMonth]) AND ((tblCOGSPercent.TermDate) Is Null Or (tblCOGSPercent.TermDate)>[tblTempBudgetMonths].[BudgetMonth]));

qryApplicableRevenue
SELECT tblGrossProfitPercent.AccountNo, qryApplicableCOGS.ProfitCenter, Sum([TotalCOGS]/(1-[GrossMargin])) AS Revenue, qryApplicableCOGS.BudgetMonth
FROM (qryApplicableCOGS INNER JOIN tblPayProfitCenter ON qryApplicableCOGS.ProfitCenter = tblPayProfitCenter.ProfitCenter) INNER JOIN tblGrossProfitPercent ON tblPayProfitCenter.DeptCode = tblGrossProfitPercent.Department
WHERE (((tblGrossProfitPercent.BudgetNo)=[Forms]![frmBudget]![txtBudgetNo]) AND ((tblGrossProfitPercent.EffectiveDate)<=[tblTempBudgetMonths].[BudgetMonth]) AND ((tblGrossProfitPercent.TermDate) Is Null Or (tblGrossProfitPercent.TermDate)>[tblTempBudgetMonths].[BudgetMonth]))
GROUP BY tblGrossProfitPercent.AccountNo, qryApplicableCOGS.ProfitCenter, qryApplicableCOGS.BudgetMonth;
