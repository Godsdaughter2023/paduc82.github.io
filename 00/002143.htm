Excel 255 Character Path Limitation - Microsoft Community

jggtn say:

Excel 255 Character Path Limitation

Apparently Excel limits file path lengths to ~255 characters. This is neither an OS X HFS or Windows NTFS limitation, rather it is a limitation of the Excel application itself.  In todays world, this path length limitation is unacceptable.

Back in the day when file and folder name lengths were limited to 8.3 characters, It was possible to save Excel files hierarchically, much like one would organize an filing system. When VFAT came along in 1995, it appears the powers that be did not take into consideration that extending the filename length would substantially reduce the hierarchal depth of the directory structure where files could be stored.

File systems, have evolved, but apparently Excel has not, and this is a serious problem. I do not want to have to conform my workflow to an arcane and out dated file management structure.  Rather, my expectation is for Microsoft to conform their product file system behavior to conform to how their customers (me in particular [because the world revolves around me] but I'm not the only one) want to use it.

This is no longer 1985. RFC 0959 is long outdated.

When can we expect Microsoft to lift the path length limits from its apps and provide some robust path aliasing so that we can store and retrieve our work from where we want, when we want it?

Naysayers will raise the red flag of backwards compatibility (when in fact it is a forwards compatibility issue for older iterations of Excel). Frankly, I don't care. It's time to move forward instead of living in the past.
