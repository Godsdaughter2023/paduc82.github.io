Private Sub Workbook_Open()
Dim MyDate As Date
Dim Pword As String, response As String

If Date < MyDate Then
    Pword = "MyPass"

    Pword = "Anotherpass"

Application.DisplayAlerts = False
ThisWorkbook.Close
End If

chip pearson say:

You can modify the TimeBombWithDefinedName procedure to display a message box informing the user of the number of days until expiration. At the end of that procedure is the following code:

 

If CDate(Now) > CDate(ExpirationDate) Then
    MsgBox "This workbook trial period has expired.", vbOKOnly
    ThisWorkbook.Close savechanges:=False
End If

 

Replace these lines with the following:

 

If CDate(Now) > CDate(ExpirationDate) Then
    MsgBox "This workbook trial period has expired.", vbOKOnly
    ThisWorkbook.Close savechanges:=False
Else
    MsgBox "You have " & CStr(ExpirationDate - CLng(Now)) & _
        " days before expiration."
End If

don cuthbert say:

Restart is a more thorough boot process than Shutdown and Power-up with Windows 8 and 10.  If you have the capability in your BIOS to hardware-enable NX (which goes by various names in BIOS) Build 10130 may work.  It has to do with hardware Data Execution Prevention (DEP).

On an Acer Netbook that I installed 10130 on, there is no option to enable NX in BIOS.  However, UEFI Secure Boot requires NX.  I found that choosing BIOS Defaults was the only way to enable Secure Boot and after doing so, I was able to boot the ISO on a USB flash drive (which is quicker than using disk media).  There is an F12 option that can be enabled, which allows the choice of boot device at power-up, on the netbook I did.  This netbook has a 64-bit AMD processor.

Normally we can force Repair mode by powering off 2 or 3 times, just after power-up, as soon as the Windows logo flag appears or dots begin to circle.  Usually the power button has to be held in to do this.  The next boot should launch Repair.  But most of those with issues with Build 10130 were in a Windows Repair bootloop, so Repair was not launching properly either.

It is quick and easy to make a Repair USB flash drive and only requires a minimum 512MB device.  Run recoverydrive.exe from within Windows 8 or 10 to make one.  With this flash drive you can get to a Command Prompt to assess and repair the boot environment if the auto-Repair process fails.
