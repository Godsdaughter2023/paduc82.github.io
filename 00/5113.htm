timliu_msft1261 say:

hi ron,

using workflow to move contents is not supported in sharepoint online. so, we don't have send to repository action in sharepoint 2013 work flow. thanks for your understanding.

you may try the steps below to check if it can achieve your requirement. we use content organizer, drop of library. a and b are two site collections. la is a library in a. lb is a library in b. we want to move document from la to lb.
1.we navigate to b and click the setting gear>site settings>manage site features.
2.find content organizer and click activate.
3.after this feature is activated, we automatically get a drop of library in b.
4.return to site settings page>content organizer settings> set up for your requirement.
4.navigate to sharepoint admin center>records management>new connection> create a new connection. now you get a send to connection in all the site collections which will send a document to the drop of library in b.
note:for the send to url, you can find it at the bottom of content organizer settings page.

5.now we need to set up a rule for the drop of library. it can move the files in the drop of library to lb. return to the site settings page of b>content organizer rules>new item>configure the settings as you want.
6.if you want the files in la to be sent to lb automatically. you can also create retention policy for la. navigate to la>library tab>library settings>information management policy settings>choose a content type>tick the check box of enable retention>after the page refresh>click add a retention stage>configure it as you want.
note:1.in the action section, we choose transfer to another location, and then we can choose send to connection created in step 4. it may take 24 hours for the new created connection being chosen here.


2.the retention policy runs weekly, it may need a week to take effect. to learn more, see timer job reference (sharepoint 2013).

in summary, we create send to connection for moving a file to drop of library. the content organizer rules move the files in drop of library to the destination library (it runs every day). the retention policy move the files in other library to a drop of library automatically.

here is another similar workaround if we consider the documents as records:implement records management.

if the workarounds mentioned above are not what you want, we suggest you post a new thread in our sharepoint msdn forum to check if there are workflow workarounds to achieve your requirement. it's a specific forum that handles this kind of questions and issues.
