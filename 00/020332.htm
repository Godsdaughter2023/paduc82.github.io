i will add on to John's answer that Acccess goes to great lengths to interpret a wide variety of dates that might be entered by users, and I find that very convenient, both as a user and as a developer.

It would be possible to use an input mask to restrict the sorts of date entries that users can make, but it still wouldn't help you in keeping a user from entering "13/12/2012".  And you could use code in the text box's BeforeUpdate event to examine the actual text the user entered and possibly reject and entry such as "13/12/2012", but:

john w. vinson say:

That is a perfectly correct and normal date in most of the world. The United States convention of month/day/year is used in some other places, but most nations prefer the more logical day/month/year. Access recognizes both; since its programmers were from the US, they made month/day/year the default, and as a result 7/9/12 will be recognized as the 9th of July; but if a date entry doesn't make sense as month/day/year the program will try the other alternative.

Note that a date is actually stored in the table without any particular format; instead, it's stored as a double float count of days and fracions of a day since midnight, December 30, 1899. The format is applied when that number is displayed or entered.

don cuthbert say:

BIOS/UEFI recorded Windows 8 or 8.1 product keys can be retrieved using this command at an Administrator Command prompt:

wmic path SoftwareLicensingService get OA3xOriginalProductKey

If needed, you can do the install of Windows 8.1 with no product key and then retrieve the product key with this command to copy and paste it in the Windows activation screen.

There is really no promise to give free Windows 10 to Insiders.  However, the build 10240, which we got in mid-July is the same build as the Windows 10 RTM and free upgrade that is being distributed to Windows 7 and 8 users who upgrade.  You can have multiple devices installed with Insider Preview under the same Microsoft Account e-mail Windows login.  Activation does not require the use of an MSA (Microsoft Account).  The MSA login was only to get the new Insider builds via Windows Update.  The current Slow Ring build for Insiders who have remained on the Insider builds is the RTM build 10240, so installing the RTM build just brings Insider computers up-to-date.

The license for Windows 10 free upgrade is tied to a device and uses no product key to activate.  This enables future clean installs on that same device without a product key, as it has been registered by Microsoft activation servers as a licensed Windows 10 device.  When upgrading or clean-installing Windows 10, no product key is to be used at any time or it may prevent auto-activation.

Product keys for Windows 7 or 8 are only of use to activate Windows 7 or 8.  They are never used to activate Windows 10.  Windows 10 (free upgrade) activates by detecting that a valid upgrade has been done, not by product key.  Only purchased Windows 10 licenses would use a product key to activate.

Older activated builds of Windows 10 Insider preview can be upgraded to the same Edition of auto-activated official Windows 10 (Pro, Home, Single Language, Home N or Pro N) as the Insider Preview Edition they have activated, by using the matching Edition Windows 10 ISO, running setup.exe, choosing to keep Windows settings and not entering any product key.

We have been told that computers that did not originally upgrade from Windows 7 or 8 to Windows 10 Insider Preview and then to Windows 10 build 10240 will eventually expire, but it has not been said when or how this will happen.  A complete wipe or replacement of the hard disk and reinstall of Windows 10 from the official ISO will auto-activate on such devices, as long as no product key is entered.  They will show permanently activated, with no expiry date.  So there is no way to tell them from one that was upgraded from Windows 7 or 8.  Perhaps the activation server has some record of the original OS that was upgraded (Windows 7, Windows 8 or Windows 10 Insider Preview).

http://blogs.windows.com/bloggingwindows/2015/06/19/upcoming-changes-to-windows-10-insider-preview-builds/

To clarify:  All Windows 10 free upgrade product keys are blocked, because a product key is never used to activate the free upgrades.  Attempting to activate Windows 10 free upgrade with any product key may actually prevent auto-activation.
