How to use metadata to organize files, instead of folders?

Hello

My company has a problem: we all need to share files, but we all want them organized differently. One person wants a folder of all contracts in 2017 across multiple clients, while another person wants a folder of all contracts across multiple years, but with a particular client. Of course everyone is creating the folders they need, so we have version control issues and it's a mess.

I am not sure what Sharepoint is capable of, but I am hoping it is powerful enough to make metadata entry easy or obvious if not mandatory for file creation or upload. And I am hoping it can easily create (and let you bookmark) "folders" displaying and allowing you to manipulate lists of different documents with different attributes. 

if we can get this going, all our documents can be in a big pile on the front page of Sharepoint, and we will all be able to find things.

tim liu msft say:

Hi Esther,

I assume all users upload their files in one document library and they have different requirement for filtering those files. In this situation, my suggestions is to create different views for different users.

1. You can create all columns they want in the library.

2. Click Setting gear>Library settings>Create View. Users can create Personal View which can only be seen by themselves or you can create Public Views for them. 

3. Select the columns they want to filter the file and can change the order of columns.

4. When they access the library, they can switch to their views. So, they can see files they want.

If it cannot fully achieve your requirement, you may follow up this thread to see if other community members have better suggestions.

Best regards,

Tim
