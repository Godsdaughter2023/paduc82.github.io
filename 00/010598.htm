The “400 Bad Request error” is an HTTP status code that means the request you sent to the website server (i.e. a request to load a web page) was somehow malformed. Therefore the server was unable to understand or process the request.

I’d suggest you to check for errors in the URL. The most common reason for a 400 Bad Request error is because the URL was typed wrong or the link that was clicked on points to a URL with some kind of mistake in it.

Step 1:Turn off friendly HTTP error messages.


a. Open Internet Explorer, click the settings button, and then click Internet Options.
c. Click the Advanced tab, and then scroll down to the Browsing section.
d. Clear the Show friendly HTTP error messages check box, and then click OK.
e. Click ok to apply the changes and restart the browser to check if the issue persists.

Step 2: If the issue still persists, uncheck HTTP settings.

Open Internet Explorer, click the settings button, and then click Internet Options.
c. Click the Advanced tab, and then scroll down to the HTTP 1.1 settings.
d. Uncheck "Use HTTP 1.1

Displayed color in conditional format - Microsoft Community

pcmk say:

Displayed color in conditional format

I have a model that uses conditional formats to display header and sub-header rows in various colors depending on the filter selection.

All works well, except that I want to be able to extract a portion of the sheet into a new worksheet, without leaving links to the original sheet, but keeping the display colors as they are indicated by the conditional format.

Is there a function in Excel or a VBA property that will tell me the color being displayed at that point in time for a specific cell ?

hansv mvp say:

The DisplayFormat property of a range returns the formatting of the cell as currently displayed, taking conditional formatting into account.

For example, Range("A2").DisplayFormat.Interior.Color returns the RGB fill color of cell A2 as currently displayed.

Remark: DisplayFormat doesn't work in functions used in a worksheet formula. It does work in macros.

kosh---01 say:

Hi siddu3728,

Welcome to the forum and thanks for posting.

When first started with SIM inserted a Lumia Windows Phone will read Internet and MMS settings from the SIM if available. After that when you change SIM only Internet settings are read. To set Internet and MMS settings you can use the app 'Network Setup' which you will find in the Nokia Collection on Marketplace through the phone when availabe for your region. MMS is a data service and as such your mobile network data connection  (Settings>mobile network>Data connection) must be turned on to use MMS.

This app has settings for many carriers pre-set and also offers the option to enter the settings manually. You usually will be able to find these settings on your carriers website. When setup using the app is complete you should remove any settings in the manual apn settings from Settings>mobile network.

Hope this helps, let us know how you get on with this. 
Kosh
