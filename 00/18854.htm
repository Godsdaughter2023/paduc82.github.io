doug robbins - mvp office apps & services (word) say:

Click on the More button in the Find dialog and then check the Wildcards box and then insert the following into the find what control

\([0-9]{1,#}[A-z]{1}\)

If it's a single digit number, omit the ,#  Otherwise, replace the # with the maximum number of digits.    If the letter is uppercase, you can replace the z with Z, which will restrict the search to only uppercase letters.  Similarly, if it is lower case, replace the A with a.

While different email "clients" operate differently (where differently = user interface) - the method (i.e. POP, IMAP, EAS/MAPI etc) by which email "accounts" are accessed are "universal" regardless of email client used. For example using the POP or IMAP protocol to retrieve email data, the same data is accessed the same way in any email client. The data accessed by each of the protocols (i.e. POP, Imap, EAS/MAPI, Exchange etc) is governed by the email protocol - not the email client. See

Understanding Email Protocols (POP, Imap, MAPI, EAS)
http://www.contactgenie.info/understanding-email-protocols-pop-imap-mapi-eas-exchange

Email clients interact with email accounts and if the same email clients are configured in each email client - then the same data will be accessed by each (POP3 being a possible exception in terms of whether each client is configured to "Leave a copy of messages on the server" or not).

Consider "sync'ing" in one of two ways

1) email client with email accounts (vertical)

2) email client to email client (horizontal)

(in the case of item #2 - there is no functionality for direct Outlook email client to Outlook email client sync'ing without the use of a 3rd party sync tool.

The following article describes the various sync scenarios

Understanding Sync'ing versus Importing
http://www.contactgenie.info/understanding-outlook-syncing-vs-importing

union join truncates field - Microsoft Community

zenmasta say:

union join truncates field

I have two queries (q1 and q2). The queries are exactly the same except for the table names they reference and so the output structure between the two is identical.

I then created a union query of q1 and q2 so I could export the two queries as a combined single file.

However, I noticed a field is being truncated to 255 characters and obviously this is no good.
When I view q1 and q2 individually the field is not truncated.

My union sql:

SELECT q1.*
FROM q1;
UNION SELECT q2.*
FROM q2;

john w. vinson say:

The UNION operator checks the two tables for duplicate data and displays only one instance if it finds duplicates. In order to do so efficiently, it truncates Memo fields to 255 characters (a Memo could contain a billion bytes of data, making duplicate checking a very slow operation!)

Try replacing UNION with UNION ALL: this keyword change will skip the duplicate checking and will return the full memo field contents.
