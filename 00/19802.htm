I'd like to address a married couple like this "Mr. Abe & Mary Lincoln".  But I'd like to address an unmarried couple like this "Abe Lincoln & Mary Swift".  I was wondering how to do this.  I tried the IF THEN rule, but I don't know how to insert mail merge variables into the formula.

IF Last_Name = Guest_Last_Name THEN Title First_Name & Guest_First_Name Last_Name.  But I don't know how to do this in word mail merge.  I just get the actual typing that I put in.  It doesn't know that I want to access the variables from the linked data source of the excel spreadsheet that I've linked my mail merge to.

Use the following field construction, where you use Ctrl+F9 to insert each pair of field delimiters { }

{ IF { MERGEFIELD Last_Name } = { MERGEFIELD Guest_Last_Name } "Mr. { MERGEFIELD First_Name } & { MERGEFIELD Guest_First_Name } { MERGEFIELD Last_Name }" "{ MERGEFIELD First_Name } { MERGEFIELD Last_Name } & { MERGEFIELD Guest_First_Name } { MERGEFIELD Guest_Last_Name }" }

or, you can insert the mergefields from Mailings tab of the ribbon with the title, quotes and &s as follows

«Last_Name» = «Guest_Last_Name» "Mr. «First_Name» & «Guest_First_Name» «Last_Name»" "«First_Name» «Last_Name» & «Guest_First_Name» «Guest_Last_Name»"

and then select all of it and then press Ctrl+F9 and you will get

{ «Last_Name» = «Guest_Last_Name» "Mr. «First_Name» & «Guest_First_Name» «Last_Name»" "«First_Name» «Last_Name» & «Guest_First_Name» «Guest_Last_Name»" }

and then insert IF as shown below:

{ IF «Last_Name» = «Guest_Last_Name» "Mr. «First_Name» & «Guest_First_Name» «Last_Name»" "«First_Name» «Last_Name» & «Guest_First_Name» «Guest_Last_Name»" }

and then press Alt+F9 (probably twice) to hide the field codes.

On the developer tab, insert a form-control button, then right-click it and assign this macro to it.

Sub MakeFolderAndSaveFileMacro()
    Dim strRet As String
    Dim strFolder As String
    Dim strFName As String

    strFolder = Worksheets("Sheet1").Range("C3").Value

    'strFolder = Worksheets("Sheet2").Range("C3").Value  'Or use this line if the folder name is on Sheet2

    strRet = Dir("C:\" & strFolder, vbDirectory)

    If strRet = "" Then
        MkDir "C:\" & strFolder
    End If

    'Choose one of the next two lines
    'The first one save the file as an xlsx, the second as a macro-enabled .xlsm

    'ActiveWorkbook.SaveAs "C:\" & strFolder & "\" & strFName & ".xlsx", FileFormat:=xlOpenXMLWorkbook
    'ActiveWorkbook.SaveAs "C:\" & strFolder & "\" & strFName & ".xlsm", FileFormat:=xlOpenXMLWorkbookMacroEnabled
End Sub

Mail merge using IF rule with multiple options - Microsoft Community

daniknox1 say:

Mail merge using IF rule with multiple options

I'm looking to create a tailored report based on people's responses in an individual questionnaire. Essentially, people fill out questions that have multiple options (e.g. Age 18-30, 31-45, 46-60, 61+). Once they have completed this, I want to run a mail merge in which a report is produced based on their answers selected. So, if they have clicked option 1, one sentence is inputted. If the second option, a different sentence is inputted and so on until all options have been set up. The questionnaire would contain 20 or so questions each with different options. The hope is to run the mail merge each time a questionnaire is filled out and the participants are then sent a tailored report based on their given answers. 

Is this even possible? The IF rule only works for true or false type statements so only two options.

Thanks for any help.

doug robbins - mvp office apps & services (word) say:

If the number of options for each question is no more than a handful, you can "nest" the If...then...Else constructions.

For the purpose of providing an example, if the answer to Question 1 is in a field named A1 and assuming that the possible responses are Option1, Option2, Option3, Option4, and the sentence for each option is S1, S2, S3, S4, respsectively, you would use

{ IF { MERGEFIELD A1 } = "Option1" "S1" { IF { MERGEFIELD A1 } = "Option2" "S2" { IF MERGEFIELD A1 } = "Option3" "S3" "S4" } } }

You must use CTRL+F9 to insert each pair of field delimiters { } and press ALT+F9 to toggle off their display.

If there is a much larger number of possible answers to a question, you can run into issues when the fields get nested too deeply in which case, you can use a construction such as

{ IF { MERGEFIELD A1 } = "Option1" "S1" }{ IF { MERGEFIELD A1 } = "Option2" "S2" }{ IF MERGEFIELD A1 } = "Option3" "S3" }{ IF { MERGEFIELD A1 = "Option 4" "S4" }

suzanne s. barnhill say:
Since the options are mutually exclusive, it isn't necessary to nest the IF fields, and it is much more straightforward just to have a separate field for each option. Using Doug's suggested syntax, this would be { IF { MERGEFIELD A1 } = "Option1" "S1" }{ IF { MERGEFIELD A1 } = "Option2" "S2" }{ IF { MERGEFIELD A1 } = "Option3" "S3" }{ IF { MERGEFIELD A1 } = "Option4" "S4" }, etc.
