Hi Matt,

  • Alt+F11 to open the VBA editor
  • Alt+IM to insert a new code module
  • In the new module, paste the following code
'=========>>
Option Explicit

'--------->>
Public Sub MySave()
    ThisWorkbook.Save
End Sub

'--------->>
Public Sub MySaveAs()
    Application.Dialogs(xlDialogSaveAs).Show
End Sub
'<<========= 
  • Alt+Q to close the VBA editor and return to Excel
  • Add two buttons from the Form Controls to the worksheet of interest

  • Assign the above macros to the two buttons
  • Save the file as a macro enabled file with an xlsm extension.

ken sheridan say: Let's deal with the easy one first.  In the conditional formatting dialogue for the staff name text box put:

Expression is        InStr([StaffName],"(Faculty)")>0

and set the fore colour to red.  You cannot have two different colours in one control, so if you wanted Faculty in blue you'd have to use two adjacent text boxes.  If you do this I'd suggest aligning the name text box left, and the status text box right. I think this might look neater, but you can experiment to see what you like best.  If you do have separate controls the conditional formatting for the name text box would now be:  

Expression is        [Status] = "(Faculty)"

and that for the Status control would be:  

Field Value is        "(Faculty)"

substituting your real control names of course.

Now for the trickier stuff.  You won't find the Line method in the properties sheet. To draw a line you have to write code in the report's module.  Here's an example of code in a report's Page event procedure which draws two vertical lines:

Private Sub Report_Page()

    ' draws two 9 inch vertical lines on page,
    ' starting 1 inch down in print area,
    ' with first line 2 inches from left of
    ' print area, and second 4.5 inches from left

    Dim lngColor As Long
    Dim sngTop As Single, sngLeft As Single, sngHeight As Single

    Const TWIPSPERINCH = 1440
    
    ' set line width to 8 pixels
    Me.DrawWidth = 8
    ' Set scale to twips.
    Me.ScaleMode = 1
    ' Top = 1 inch
    sngTop = Me.ScaleTop + TWIPSPERINCH * 1
    ' left = 2 inches
    sngLeft = Me.ScaleLeft + TWIPSPERINCH * 2
    ' height = 9 inches ( 10-1)
    sngHeight = Me.ScaleTop + TWIPSPERINCH * 10
    ' colour = black.
    lngColor = RGB(0, 0, 0)
    ' draw line
    Me.Line (sngLeft, sngTop)-(sngLeft, sngHeight), lngColor
    ' move 2.5 inches to right and draw another line
    sngLeft = sngLeft + TWIPSPERINCH * 2.5
    Me.Line (sngLeft, sngTop)-(sngLeft, sngHeight), lngColor
   
End Sub

A twip is 'one twentieth of a point' and is the basic unit used when setting many properties.  A point is a typographical unit of 1/72 inch, so there are 1440 twips per inch.

By experimenting with the settings in this code you should be able to draw lines between your columns, of the right length, width and in the right positions.

Automatic Numbering in Descending Order - Microsoft Community

katyg2016 say:

Automatic Numbering in Descending Order

I was wondering if there is a way to make word number automatically in descending order, so far example if I have ten things in my list the 1 is at the bottom and 10 is at the top and for word to automatically keep this format. I have tried using the 'SEQ list' code but it when I update the list it automatically changes the list back to 1-10 instead of 10-1 I need the list to keep adding or getting rid of numbers like a normal list for each item I add but I cannot get word to do so.

stefan blom say:

You can use a SET field to define the starting number and then subtract 1 in a formula field which also includes a SEQ field. Each new formula field will then decrease numbering by 1.

The SET field won't display a result in the document; it only defines the variable with the starting number (33 in my example).

The other fields will subtract an increasing number from the starting number, resulting in 32, 31, etc. being displayed.

katyg2016 say: Do you have to manually type in the start number each time you change the number of items in the list or will it update itself and then update the list ? Thank you!

doug robbins - mvp office apps & services (word) say:

You need to replace the 33 in the { Set StartNo 33 } in Stefan's example with the number of items in the list.

Note, also that you must use CTRL+F9 to insert each pair of field delimiters, but that after entering the first { = {Ref StartNo }-{ SEQ Tryout } } you can copy and paste that for the other numbers in the list.  Use ALT+F9 to toggle off the display of the field codes.

Also note that if you want to use such numbering with more than one list in a single document, you will need to use different terms for StartNo and Tryout.  It does not matter what they are, but it might be a good idea to use a convention such as Set StartList1 and SEQ List1, Set StartList2 and SEQ List2.

stefan blom say:

I suspect that a macro would do a better job and dynamically apply the numbering in decending order.

A very basic approach would be the following, which doesn't take into account if numbering has been applied to the selected paragraphs previously:

Sub TestMacro()
'Basic macro created by MVP Stefan Blom, June 2018
Dim i As Long
Dim ParaCount As Long
Dim r As Range

Set r = Selection.Range

ParaCount = r.Paragraphs.Count

For i = 1 To ParaCount
r.Paragraphs(i).Range.InsertBefore ParaCount - i + 1 & ". "
Next i
End Sub

doug robbins - mvp office apps & services (word) say:

To be able to re-apply the numbering after an item has been added to\inserted in the list, you would need to use:

'Basic macro created by MVP Stefan Blom, June 2018, modified by Doug Robbins

Dim i As Long

Dim ParaCount As Long

Dim r As Range, rdel As Range

Set r = Selection.Range

ParaCount = r.Paragraphs.Count

For i = 1 To ParaCount

If IsNumeric(Left(r.Paragraphs(i), 1)) Then

    Set rdel = r.Paragraphs(i).Range.Duplicate

    rdel.End = rdel.Start + InStr(rdel, vbTab)

    rdel.Delete

End If

r.Paragraphs(i).Range.InsertBefore ParaCount - i + 1 & "." & vbTab

Next i
