kyleg454 say:

We finally managed to solve this issue.

 

As we all already know, Outlook 2011 (Mac) defines fonts by px rather than pt. Upon rendering the email, Outlook (Win) will convert the px to pt. This will cause slight variation, as theres not always an exact match, but for most purposes, it will be close enough. 14px Calibri becomes 10.5pt.

 

The issue we encountered, where fonts were dramatically changing is actually caused by how the process above interacts with zooming. The rendering process seems to be zoom -> convert px to pt. This is handled by Word (optionally in 2003).

 

This means when Word converts the px to (zoomed) pt, the pt which matches the px reduces. At 200% zoom, 14px Calibri matches 4pt Calibri. Interestingly, this seems to be fixed in Outlook 2007, but the windows display DPI setting produces the same behaviour.

 

As it stands, I don't think we have any option but to have all zoom settings at 100%. Ideally the rendering process would be convert px to pt -> zoom or at least base the px to pt conversion on a 100% base regardless of user zoom. Or just have Outlook 2011 use pt, like every other Outlook product but maybe that's too simple!

 

I hope this saves a few other people from the headaches we experienced.

chip pearson say:

That formula looks familair. I don't think you can do what you want with conditional formatting. The formula, if possible at all, would end up being frighteningly complicated and unmaintainable. I would use a VBA procedure to color the cells.

Sub FormatCellFill()
    Dim R As Range
    Dim LastRow As Long
    Dim ColorIndex As Long
    Dim InFill As Boolean
    Dim N As Long
    Dim ColorWidthColumns As Long
    Dim StartCell As String
    Dim DataTestColumn As String
   
    ColorIndex = 3 'red '<<<<<< Set color index
    InFill = True '<<<<< True -> Color first group, False-> No color first group
    ColorWidthColumns = 10 '<<<< how many columns wide to format
    StartCell = "A5" '<<<< where to start the banding
    DataTestColumn = 4 '<<<< column number containing data to test
   
    Set R = Range(StartCell)
    With R.Worksheet
        LastRow = .Cells(.Rows.Count, "A").End(xlUp).Row
    End With
    N = 0
    Do Until R.Row > LastRow
        N = N + 1
        If R.EntireRow.Cells(1, DataTestColumn).Value <> vbNullString Then
            If InFill = True Then
                R.EntireRow.Cells(1, "A").Resize(, ColorWidthColumns).Interior.ColorIndex = ColorIndex
            Else
                R.EntireRow.Cells(1, "A").Resize(, ColorWidthColumns).Interior.ColorIndex = xlColorIndexNone
            End If
        Else
            N = 0
            InFill = True
        End If
        If N >= 30 Then
            N = 0
            InFill = Not InFill
        End If
        Set R = R(2, 1)
    Loop
End Sub

Change the lines marked with '<<<< to the appropriate values.ColorIndex is the color that is to be used for shading the row. InFill, if True, means that formatting is applied to the first thirty rows. If False, no shading will be applied until the second group of thirty rows.ColorWidthColumns is how many columns to shade, starting in column A. StartCell is the address of the first cell to which shading might be applied. DataTestColumn is the column number of the cells whose value determines whether to highlight missing data.

tushar mehta say:

Recently I had reason to do some fairly sophisticated banding for an Excel report for a large financial institution and was rather thrilled that I did it without code.  Inspired by that accomplishment {grin}, here's how to solve your problem without code.

Based on your post, I assumed you have data starting with row 5 and column D being space is the trigger for a reset.  If no reset occurs within 30 consecutive rows, you want the band color to change every 30 rows.

In E5 enter the formula =IF(D5="","",0)

In F5 enter the array formula =IF(D6="","",IF(E5="",0,IF(MOD(ROW()-MAX(ROW($E$5:E5)*($E$5:E5=""))-1,30)<>0,E5,MOD(E5+1,2))))

To enter an array formula complete data entry with CTRL+SHIFT+ENTER rather than just the ENTER or the TAB key.  If done correctly, Excel will show the formula bracketed by curly brackets { and }

Now, format the row based on the contents of column E, which will have only one of three values "", 0, or 1.

The above is lightly tested as compared to what I did for my client. {grin}

jay freedman say:
The package in the kb931125 article is explicitly only for Windows XP. As explained in the section of the article headed "Windows Vista and Windows 7" (which also applies to Windows 8), the same effect is achieved in those versions by automatic Windows Update downloads. There is very little probability that the problem you see on your new machine has the same cause (lack of root certificates) as the one on Windows XP.

It is more likely that your laptop is running the 64-bit version of Windows 8 (almost all new machines do), while the ActiveX component in the third-party software is 32-bit (very few ActiveX components are available as 64-bit versions). Making that work requires special steps to get the right entries in the registry. Point your IT folks to an article such as "Using 32-bit or 64-bit ActiveX Components on x64 Windows". Also, as Aravinda says, you should check with the vendor of the third-party software for updates or advice.
