greginmich say:

Any and all security or virus notifications that appear in a web browser and ask you to call a telephone number for assistance are totally bogus. These fake alerts are known generically as Tech Support Scam webpages and most of them run a malicious script that immediately loads a modal alert dialog and then just repeatedly reloads that same dialog every time you click on the OK button in order to close it – which effectively prevents you from regaining access to the browser. The only way to get out of this dialog loop in some browsers is to end the browser process in Task Manager. In other browsers, like Microsoft Edge, there’s a built-in Dialog Loop Protection feature that allows you to break out of the malicious dialog loop with a checkbox that appears on the fake alert dialog after your first attempt to close the alert fails:

This Dialog Loop Protection feature is especially important for Microsoft Edge, because Edge is unique in having an AutoRecover feature that can’t be bypassed – so if you attempt to escape from a scam page by ending the browser process in Task Manager, the malicious page will just be automatically reloaded whenever you restart the browser. Edge’s automatic malware-reloading feature caused an incredible amount of frustration and anxiety for the first users of Microsoft's flagship  browser:

https://answers.microsoft.com/en-us/protect/forum/protect_other-protect_scanning/system-error/dcccc74e-fb98-43a8-8579-956692a40982

And although Dialog Loop Protection was certainly a step in the right direction here, I won’t personally consider the Tech Support Scam issue resolved until all of these scam pages are consistently prevented from locking up the browser in the first place, so that absolutely no action (or learning) is required on the part of users:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/defender-ineffective-against-the-windows-warning/dfa1ffd3-56cf-403d-b8a9-96f2a4846909#LastReply

Public Sub ImportTextFile_new()
    Dim the_file As String
    Dim LineData As String
    Dim ICDraw As String
    Dim cnn As ADODB.Connection
    Dim rs As ADODB.Recordset
    Dim strSQL As String
    Dim x As Integer
    Dim aryICDraw As Variant

    Set cnn = CurrentProject.Connection
    Set rs = New ADODB.Recordset
    ' Open the text file

    the_file = open_dialog
    Open the_file For Input As #1


    ' Open the table to insert the text file into
    strSQL = "Select * from local_file"
    rs.Open strSQL, cnn, adOpenDynamic, adLockOptimistic
    ' Read a line of data.
    Input #1, LineData
    ICDraw = Trim(LineData)
    Close #1
    aryICDraw = Split(ICDraw, vbLf)
    For x = 0 To UBound(aryICDraw)
        rs.AddNew
        rs!the_code = aryICDraw(x)
        rs.Update
    Next x

    Set rs = Nothing
    Set cnn = Nothing

End Sub

This function will do the job.

Function SumByColor(InRange As Range, WhatColorIndex As Integer, _
    Optional OfText As Boolean = False) As Double
'Chip Pearson code
'You can call this function from a worksheet cell with a formula like
'=SUMBYCOLOR(A1:A10,3) 'sums by cell color
'=SUMBYCOLOR(A1:A10,3,TRUE) 'sums by font color
Dim rng As Range
Dim OK As Boolean

Application.Volatile True
For Each rng In InRange.Cells
    If OfText = True Then
        OK = (rng.Font.ColorIndex = WhatColorIndex)
    Else
        OK = (rng.Interior.ColorIndex = WhatColorIndex)
    End If
    If OK And IsNumeric(rng.Value) Then
        SumByColor = SumByColor + rng.Value
    End If
Next rng

End Function
