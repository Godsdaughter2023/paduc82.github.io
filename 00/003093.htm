Hi James,

Chromium is a nuisance, and could be classed as malware it did not come with Windows or from Microsoft - it is installed automatically when you install 3rd software and is extremely difficult to remove . . .

The best way to remove it is to manually delete it:

Open Windows file explorer

On the view ribbon, check the box marked 'Hidden Items'

Navigate to (replace username with your login name):

C:\Users\username\AppData\

Open the Local and Roaming folders, and if you see a Chromium folder inside either or both of them, delete it

After a reboot, go into Control Panel and check for any other unwanted software that may have also come down with Chromium - it is generally bundled with other PUP's . . .

a. user say:

On balance, the simplest approach is usually to rearrange your data source as has been suggested, especially if your example is a simple example of something that is actually much more complicated.

But if you don't have a large number of records and...

a. ...your data source actually has a sequential record number for each row, or one could be added, e.g.

K   A          B           C             D

1  Name   Mon     Target   Material 

2  Kate     Jan       455        Burger

3  Biren    Jan       540        Pizza

4  jack      Jan       630        Hotdog

5  Angel   Jan       710        Puff

c. ...If necessary, you can add two empty rows at the end (e.g. 

6 (followed by the appropriate number of field separators)
7 (ditto)

...then you might be able to get away with a mail merge main document with field names like this:

{ SET DBFN "the full pathname of the data source file" }{ SET TNAME "the table name"
}{ SET name2 { QUOTE { DATABASE \d "{ DBFN }" \s "SELECT Name FROM { TNAME } WHERE K = { MERGEFIELD K } + 1" }
}{ SET target2 { QUOTE { DATABASE \d "{ DBFN }" \s "SELECT Target FROM { TNAME } WHERE K = { MERGEFIELD K } + 1" }
}{ SET name3 { QUOTE { DATABASE \d "{ DBFN }" \s "SELECT Name FROM { TNAME } WHERE K = { MERGEFIELD K } + 2" }
}{ SET target3 { QUOTE { DATABASE \d "{ DBFN }" \s "SELECT Target FROM { TNAME } WHERE K = { MERGEFIELD K } + 2" }
}{ MERGEFIELD Name } your target of { MERGEFIELD Target }{ IF "{ Name2 }" = "" " is the highest" " is far less than { Name2 }'s ({ Target2 }){ IF "{ Name3 }" <> "" " and { Name3 }'s ({ Target3 })" }" }, so request to concentrate on your { MERGEFIELD Material } sales.

(You will probably need to adjust the text in the special case at the end where Name2 is blank)

If you try this approach, I strongly advise that you test with a very small number of data rows - if there are any errors in the coding, you will find yourself clicking through a large number of error messages.

*All* the {} must be the special field code brace pairs that you can enter using ctrl-F9, not the ordinary {} characters you can type on the keyboard.

Instead of { SET DBFN "the full pathname of the data source file" }, you need to set the "full name" of the data file, with backslash characters doubled up. So if your data file is an Excel workbook, you might need something like

{ SET DBFN "c:\\my excel files\\theexcelwb.xlsx" }

Make sure there aren't any space characters at the beginning and end of the path - if you copy/paste a pathname into Word, you are likely to end up with

{ SET DBFN " c:\\my excel files\\theexcelwb.xlsx " }

Instead of { SET TNAME "the table name" } you need to set up the name of the table your data is coming from. If the data is in a Word document, you can use
{ SET TNAME "X" }

If it is in an Excel worksheet, say "Sheet1", then you need

{ SET TNAME "[Sheet1$]" }


(also avoiding spaces at the beginning and end).

If it is in an Excel named range such as "myrange", or an Access table called "myrange", you would (probably) need

{ SET TNAME "[myrange]" }

If it's elsewhere, you might need something else.

If your source is an Excel worksheet, you may be able to get away without adding 2 records at the end (There are in any case other ways to handle that problem). But for a Word document, and perhaps other types of source, you will either need those dummy records, or more complex mergefield coding. But because you have two extra rows, when you merge, you should specify that you want to merge reords 1 to the number of non-dummy rows (in the case of your example, 1 to 5, not 1 to 7). 
 

doug robbins - mvp office apps & services (word) say:

If you have the data in Excel and run a macro containing the following code

Dim i As Long
With Sheets(1).Range("A1")
    For i = 1 To .CurrentRegion.Rows.Count - 3
        .Offset(i, 4) = .Offset(i + 1, 0)
        .Offset(i, 5) = .Offset(i + 1, 2)
        .Offset(i, 6) = .Offset(i + 2, 0)
        .Offset(i, 7) = .Offset(i + 2, 2)
    Next i
End With

It will modify the data as shown below and if you then add some field names in the first row of columns E through H, you will then have data in the necessary format to do what you want

Name Mon Target Material
Kate Jan 455 Burger Biren 540 Jack 630
Biren Jan 540 Pizza Jack 630 Angel 710
Jack Jan 630 Hotdog
Angel Jan 710 Puff
