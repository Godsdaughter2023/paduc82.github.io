How can I put 2 style ( title4 and normal) on the same line. - Microsoft Community

gchaput say:

Every time I tried ''office word'' change all the line for normal. or all the line for title4.

daniel g. grau say:
You can't do that, if both are paragraph styles. Let me elaborate:

  • A paragraph basically contains two kinds of attributes: those that affect the way the characters look (point size, color, bold, font…) and those that affect the way the paragraph looks (indent, tabs, alignment, spacing…).
  • A paragraph style (such as Normal and Heading 1) contains both types of specifications (character formatting and paragraph formatting). This is why you cannot have two paragraph styles applied at the same time.
  • A character style only contains format information for characters.

You should define one or more character styles that look the way you need. To do this:

  • Select Format > Style > New 
  • Give it a name
  • In the dropdown menu at the top right, select "Character"
  • Define the parameters of your characters (font, size, color, etc.)
  • Before you click OK, click on the bottom dropdown menu to select Shortcut. This will let you define a key combination to recall the character style. Just about all Control + Letter combinations are available.

You can also assign shortcuts to paragraph styles.

Combining spreadsheets with a macro - Microsoft Community

chrisl_778 say:

Combining spreadsheets with a macro

I am trying to figure out if it is possible to combine spreadsheets using a macro.

To simplify what I am trying to do:

I would have a folder with two files for each day titled (May 1 2017, May 1 2017b, May 2 2017, May 2 2017b etc.)

I would want to create one file for each day called May 1 2017, May 2 2017 etc.  The files would be saved into a different folder and would consist of the two files mentioned above in two separate worksheets.

Is there an easy way to do this?

bernie deitrick say:

This macro requires a reference to MS Scripting Runtime - save the file with the macro into the same folder as the double files, and the macro will create a new subfolder named "Combined Files" with the new files in it.

Sub CombineFiles()
    Dim objFSO As Scripting.FileSystemObject
    Dim objFolder As Scripting.Folder
    Dim colFiles As Scripting.Files
    Dim objFile As Scripting.File
    Dim wkbkB1 As Workbook
    Dim wkbkB2 As Workbook
    Dim i As Integer
    Dim NewFolder As String
    
    Set objFSO = CreateObject("Scripting.FileSystemObject")
    Set objFolder = objFSO.GetFolder(ThisWorkbook.Path)
    Set colFiles = objFolder.Files

    NewFolder = ThisWorkbook.Path & "\Combined Files"
    If Dir(NewFolder, vbDirectory) = "" Then
        MkDir NewFolder
    End If
     
    For Each objFile In colFiles
        If objFile.Name <> ThisWorkbook.Name Then
            If Right(objFile.Name, 6) <> "b.xlsx" Then
                If Left(objFile.Name, 1) <> "~" Then
                    Set wkbkB1 = Workbooks.Open(objFile.Name)
                    Set wkbkB2 = Workbooks.Open(ThisWorkbook.Path & "\" & Replace(objFile.Name, ".xlsx", "b.xlsx"))
                    For i = 1 To wkbkB2.Worksheets.Count
                        wkbkB2.Worksheets(i).Copy after:=wkbkB1.Worksheets(wkbkB1.Worksheets.Count)
                    Next i
                    wkbkB2.Close False
                    wkbkB1.SaveAs NewFolder & "\" & objFile.Name
                    wkbkB1.Close False
                End If
            End If
        End If
    Next objFile

End Sub

ken sheridan say: If there is a FwdID autonumber column and a Forwarder text column in the Forwarder form's underlying table (which I'll assume is named Forwarders in the code below), and the value shown in the combo box is that of the Forwarder column, you can do everything directly from the FwfID combo box's NotInList event procedure without the need for double-clicking:

Private Sub FwdID_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control
    Dim strMessage As String
   
    Set ctrl = Me.ActiveControl
    strMessage = "Add " & NewData & " to list?"
   
    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        DoCmd.OpenForm "Forwarder", _
            DataMode:=acFormAdd, _
            WindowMode:=acDialog, _
            OpenArgs:=NewData
        ' ensure Forwarder form closed
        DoCmd.Close acForm, "Forwarder"
        ' ensure forwarder has been added
        If Not IsNull(DLookup("FwdID", "Forwarders", "Forwarder = """ & _
            NewData & """")) Then
        Response = acDataErrAdded
        Else
            strMessage = NewData & " was not added to Forwarders table."
            MsgBox strMessage, vbInformation, "Warning"
            Response = acDataErrContinue
            ctrl.Undo
        End If
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If

End Sub

In the Forwarder form'sOpen event procedure put:

    If Not IsNull(Me.OpenArgs) Then
        Me.Forwarder.DefaultValue = """" & Me.OpenArgs & """"
    End If

Note that assigning the new value as the Forwarder control's DefaultValue property does not in itself add a new record.  This will only happen once the user inserts further data into the Forwarder form, e.g address data etc.

If the value shown in the combo box is not taken from a single column in the Forwarders table, but is the concatenation of more than one column, e.g. FirstName & " " & LastName it is still possible to do this by parsing the OpenArgs property.  Here's an example which does this for adding a new contact entered in the format Ken Sheridan:

Private Sub lstFindContact_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control
    Dim strMessage As String
   
    Set ctrl = Me.ActiveControl
    strMessage = "Add " & NewData & " to list?"
   
    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        DoCmd.OpenForm "frmContacts", _
            DataMode:=acFormAdd, _
            WindowMode:=acDialog, _
            OpenArgs:=NewData
        ' ensure frmCities closed
        DoCmd.Close acForm, "frmContacts"
        ' ensure contact has been added
        If Not IsNull(DLookup("ContactID", "Contacts", "FirstName & "" "" & LastName = """ & _
            NewData & """")) Then
        Response = acDataErrAdded
        Else
            strMessage = NewData & " was not added to Contacts table."
            MsgBox strMessage, vbInformation, "Warning"
            Response = acDataErrContinue
            ctrl.Undo
        End If
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If
   
End Sub

With the following in the frmContacts form's Open event procedure:

Private Sub Form_Open(Cancel As Integer)

    Dim strFirstName As String, strLastName As String
   
     If Not IsNull(Me.OpenArgs) Then
        strFirstName = Left(Me.OpenArgs, InStr(Me.OpenArgs, " ") - 1)
        strLastName = Mid(Me.OpenArgs, InStr(Me.OpenArgs, " ") + 1)
        Me.FirstName.DefaultValue = """" & strFirstName & """"
        Me.Lastname.DefaultValue = """" & strLastName & """"
     End If
       
End Sub
