    mike h.. say:

    Hi,

    As others have said you cannot directly read the colour of a Conditionally formatted cell but it can be done. The code below was written by Rick Rothstein, I liked it and kept it and you may be able to use it. The code in bold is simply used to call the main routine. Please see the comments in the code.

    Sub test_Colour()
    Dim r
    r = DisplayedColor(Range("A1"), 1, 0)
    End Sub
    ' Arguments
    ' ----------------
    ' Cell - Required Range, not a String value, for a **single** cell
    '
    ' CellInterior - Optional Boolean (Default = True)
    '                True makes function return cell's Interior Color or ColorIndex based on
    '                the ReturnColorIndex argument False makes function return Font's Color or
    '                ColorIndex based on the ReturnColorIndex argument
    '
    ' ReturnColorIndex - Optional Boolean (Default = True)
    '                    True makes function return the ColorIndex for the cell property determined
    '                    by the CellInterior argument False make function return the Color for the
    '                    cell property determined by the CellInterior argument
    '
    Function DisplayedColor(Cell As Range, Optional CellInterior As Boolean = True, _
                            Optional ReturnColorIndex As Long = True) As Long
      Dim X As Long, Test As Boolean, CurrentCell As String
      If Cell.Count > 1 Then Err.Raise vbObjectError - 999, , "Only single cell references allowed for 1st argument."
      CurrentCell = ActiveCell.Address
      For X = 1 To Cell.FormatConditions.Count
        With Cell.FormatConditions(X)
          If .Type = xlCellValue Then
            Select Case .Operator
              Case xlBetween:      Test = Cell.Value >= Evaluate(.Formula1) And Cell.Value <= Evaluate(.Formula2)
              Case xlNotBetween:   Test = Cell.Value <= Evaluate(.Formula1) Or Cell.Value >= Evaluate(.Formula2)
              Case xlEqual:        Test = Evaluate(.Formula1) = Cell.Value
              Case xlNotEqual:     Test = Evaluate(.Formula1) <> Cell.Value
              Case xlGreater:      Test = Cell.Value > Evaluate(.Formula1)
              Case xlLess:         Test = Cell.Value < Evaluate(.Formula1)
              Case xlGreaterEqual: Test = Cell.Value >= Evaluate(.Formula1)
              Case xlLessEqual:    Test = Cell.Value <= Evaluate(.Formula1)
            End Select
          ElseIf .Type = xlExpression Then
            Application.ScreenUpdating = False
            Cell.Select
            Test = Evaluate(.Formula1)
            Range(CurrentCell).Select
            Application.ScreenUpdating = True
          End If
          If Test Then
            If CellInterior Then
              DisplayedColor = IIf(ReturnColorIndex, .Interior.ColorIndex, .Interior.Color)
            Else
              DisplayedColor = IIf(ReturnColorIndex, .Font.ColorIndex, .Font.Color)
            End If
            Exit Function
          End If
        End With
      Next
      If CellInterior Then
        DisplayedColor = IIf(ReturnColorIndex, Cell.Interior.ColorIndex, Cell.Interior.Color)
      Else
        DisplayedColor = IIf(ReturnColorIndex, Cell.Font.ColorIndex, Cell.Font.Color)
      End If
    End Function

    thestickman---01 say:

    I have successfully used my 6500C as a GPRS or 3G modem for my computer in other countries, but never managed in the UK on Vodafone Pay as you talk.

     

     

    After a geeky couple of hours of research, I found that Vodafone (and some other providers) use different access points for contract and Pay as you Talk, but the Nokia PC suite offers only the contract access point.

     

     

    Before going any further, check that you can view a web page on your phone- this usually means that you have at least some valid settings and that you are subscribed to use GPRS or 3G.

     

     

    It's possible to get the network provider to send settings to your phone, but PC suite doesn't use them. So you need to get the settings sent to the phone- you can either go into a shop, or do it via their web site- then dig into the settings on the phone to find the access point name.

     

    Alternatively, you can make use of the altruistic efforts of another geek who kindly made the access point names for several networks available on http://www.filesaveas.com/gprs.html

     

     

    To find the access point name on a 6500C:

     

    Select Configuration

    Select Preferred access pt.

    ... at this point, the phone lists DESCRIPTIONS of access points, not names of them....

    Move the cursor to the most suitable sounding one, for example Vf PreWAP for Vodafone prepaid, BUT DON"T SELECT IT!

    Press Options

    Select Details

    The parameter entitled 'Packet data access point' is the one to use: for Vodafone prepaid, it's pp.vodafone.co.uk

     

     

    To use this with Nokia PC suite

    Select 'One touch access'

    Quickly click on the spanner, before it tries to connect.. you have only ten seconds!

    Select your phone from the list of modems

    Ignore the network operator list, and instead select the 'Configure the connection manually' option

    Type in the access point name

     For some providers, you also need to enter a username and password (see the web site listed above)

     

    By the way, when you are experimenting, you may get a message saying "Subscribe to packet data first". This doesn't mean that you need to subscribe- it could mean that the access point name is invalid, or that you have a valid access point name but it's the wrong one for you (contract/prepaid, for example) or that the username and password are invalid.
