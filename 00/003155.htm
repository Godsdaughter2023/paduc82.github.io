Desktop shortcuts, hyperlinks (in Outlook, etc) not working after uninstalling Google Chrome

Hey everyone,

We've heard many reports from you that desktop shortcuts, hyperlinks (in mail programs), etc aren't working after uninstalling Google Chrome. Please note that if you set Google Chrome as your default browser and later uninstall it, your computer can no longer locate what was the default browser (i.e. Google Chrome) for opening links or file types associated with webpages such as .html, .htm, etc and therefore generates an error.

This can be remedied by setting another browser (such as Internet Explorer) as your default or making some changes to your registry keys (please only edit reg keys if you are familiar/comfortable with your system). Thanks to users 'Nakro' and 'nondescript' for their helpful instructions provided below! If these steps don't solve your problem, please let us know!
Thanks,
Toni

===========================

Option 1: Setting Internet Explorer as your default browser:

1. Open Internet Explorer
2. Click on Tools menu
3. Select Internet Options -> Programs
4. Click on button Make Default

If you have MS Outlook open, close it after you have set Internet Explorer as your default, and re-open it for the changes to take effect.

Note: If you would like to set Firefox as your default, you can find instructions here: http://support.mozilla.com/en-US/kb/How+To+Make+Firefox+The+Default+Browser


Option 2: Restore registry settings in the User Profile:

1. Click Start, click Run, type Regedit in the Open box, and then click OK.
2. Locate the following subkey: HKEY_CURRENT_USER\Software\Classes\.html
3. On the File menu, click Export.
4. In the Export Registry File dialog box, enter HKCU_Classes_HTML_Backup.reg and click Save.

Note: This will create a backup of this registry key in the My Documents folder by default.

5. Right click the (Default) value for the .html key and select Modify...
6. Change the value from "ChromeHTML" to "htmlfile"

Repeat steps 1-6 for .htm and .shtml keys if they exist, remember to change the name of your backup file for each export.

   '************************************  BEGIN VBA CODE  *************************************
Function BoxNumbCapture(S1 As String) As String
   '
   'Starting point:
   'http://www.excelforum.com/excel-programming-vba-macros/681379-extract-s-and-names-from-p-o-box-and-street-addresses.html
   '

   ' NOTE:  "S1" is typically a full Street-address String

   '
   Dim Matches As Object
   Dim RE As Object
   Dim S2 As String

   Dim S3 As String
   Set RE = CreateObject("VBScript.RegExp")
   '
   'The following code line MUST Begin AND End with quotation marks AND be on ONE line!
   '  work-wrapping on these Forum sites can cause unnecessary challenges 
   '
   RE.Pattern = "\b[P|p]*(OST|ost)*\.*\s*[O|o|0]*(ffice|FFICE)*\.*\s*[LOCK|lock]*\.*\s*[B|b][O|o|0][X|x]\b(?! St)"
   S1 = WorksheetFunction.Trim(WorksheetFunction.Clean(S1))
   S2 = S1
   '
   'Worksheet Functions are generally NOT case sensitive; VBA Functions are
   '
   'Edit "0" for "O" (zero for letter O)
   S1 = Replace(S1, "P0", "PO")
   S1 = Replace(S1, "p0", "po")
   S1 = Replace(S1, "P 0", "P O")
   S1 = Replace(S1, "p 0", "p o")
   S1 = Replace(S1, "P.0", "P.O")
   S1 = Replace(S1, "p.0", "p.o")
   S2 = Replace(S2, "P0", "PO")
   S2 = Replace(S2, "p0", "po")
   S2 = Replace(S2, "P 0", "P O")
   S2 = Replace(S2, "p 0", "p o")
   S2 = Replace(S2, "P.0", "P.O")
   S2 = Replace(S2, "p.0", "p.o")
   ---
   '&&&&&&&&&&&&&&&& Edit Box# run-together  &&&&&&&&&&&&&&
   If S1 Like "*Box#*" Then
      'Exception to the "rule" that Worksheet Functions are NOT case sensitive
      S1 = WorksheetFunction.Substitute(S1, "Box", "Box ")
      S2 = WorksheetFunction.Substitute(S2, "Box", "Box ")
   End If
   If S1 Like "*BOX#*" Then
      S1 = WorksheetFunction.Substitute(S1, "BOX", "BOX ")
      S2 = WorksheetFunction.Substitute(S2, "BOX", "BOX ")
   End If
   If S1 Like "*box#*" Then
      S1 = WorksheetFunction.Substitute(S1, "box", "box ")
      S2 = WorksheetFunction.Substitute(S2, "box", "box ")
   End If
   ---

  'Testing the string prevents errors if there are no matches

   If RE.Test(S2) = True Then
      'Return all matches set by the Pattern Property
       Set Matches = RE.Execute(S1)
      'Place the first number in the first column to the right of the search cell
       S3 = Matches(0).Value
        'Locate the data relative to the number
         If Matches(0).FirstIndex <> 0 Then
           'Number starts the string - Data is to the right
            If InStr(1, S1, S3) >= Matches(0).FirstIndex Then
               S2 = Replace(Mid(S1, InStr(1, S1, S3)), S3, "")
               'Uses another function to strip non numerics
               S2 = " " + RemoveNonNumeric(S2)
            Else
               'Uses another function to strip non numerics
               S2 = " " + RemoveNonNumeric(S2)
               S2 = Left(S2, Matches(0).FirstIndex)
            End If
         Else
            'Uses another function to strip non numerics
            S2 = " " + RemoveNonNumeric(S2)
            If Replace(S2, " ", "") = "" Then
               'Next line makes an attempt to add back a Non-Numeric Box Number like "Box H"
               S2 = Mid(S1, Len(S3) + 1, Len(S1) - Len(S3))
            End If
         End If
         BoxNumbCapture = Trim(S3 + S2)
   End If
   Set RE = Nothing
End Function

---

Function RemoveNonNumeric(str)
   Dim Matches As Object
   Dim RE As Object
   Set RE = CreateObject("VBScript.RegExp")
   RE.Pattern = "\d+"
   If RE.Test(str) = True Then
       Set Matches = RE.Execute(str)
       str = Matches(0).Value
   Else
      str = ""
   End If
   RemoveNonNumeric = str
   Set RE = Nothing
End Function

Locky is currently being distributed via phishing e-mails that will contain Word document attachments with malicious macros (macro-malware); which when opened, a message is to be displayed stating that you should enable the macros if the text is unreadable/scrambled.

Macro threats, as payload couriers, seem to be an effective infection vector. But unlike exploit kits, these macro threats require user consent to run.
