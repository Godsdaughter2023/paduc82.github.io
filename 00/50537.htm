septimusfry say:

A bit late for a reply, but someone else may have same question. So these are the 'rules' which you must be aware of.

  • First, resynching MUST NOT be interrupted. If you reboot it will start again
  • Second, resynching does not (AFAIK) use full disk r/w rates. In my case, disks which normally operate at more than 70MB/sec, resynch at about 7MB/s. With 3GB drives, this will likely take up to 120hrs.
  • If you have to zap your machine down for some reason, you will almost certainly cause a resynching.
  • If  you are lucky, pressing F5, or selecting 'refresh' from More Actions in Disk Management console, will produce a % figure. Sometimes it obstinately will not.

There are some who think that the resynch hassle is not worth it, and hardware RAID is a better proposition. I am not so sure, as HWR has its own problems, especially in the case of failed controller. At least with W7 s/w raid, one can continue to work while it sorts itself out and it will work its way through eventually !

    ¡firedog say:

    The junk filters in Windows Live Mail react to both the content and the origin of messages it scans, not just the sender. If a message from a trusted sender (i.e. one on your safe senders list or in your contacts) contains something suspicious, the filter will move it to the junk folder just in case. This will avoid harm caused, say, by someone using one of your contacts' addresses fraudulently, or by malicious software on the sender's computer that is automatically sending out spam or worse. 

    You can examine the source of a message in the junk folder to see whether it's genuine: right-click on it and select Properties. On the Details tab, click Message source. You can then compare this with the source of a known good message from the trusted sender. There will often be headers indicating the results of scanning the message at the server you retrieve the message from. These will vary from server to server, but you may spot something to explain why the message was consigned to junk.

     

    Repairing the installation will reset the junk filters. If they're still acting up, there is probably something about the messages concerned that triggers the filter. This could be something added by the mail server that you retrieve your mail from.

    In any event, it's better to be safe than sorry; rather a good message in the junk folder than a nasty one in the inbox.
