ken sheridan say: Using a form with an embedded subform as Karl describes is the simplest approach and requires no code or macro whatsoever, but if you do wish to open a second 'linked' form then I'd recommend that you open the frmContacts form in dialogue mode, and pass the StudentID to it as its OpenArgs property.  I'd also suggest that you do it with code in the button's Click event procedure.  Macros are cheap and cheerful, but limited, and writing VBA code is a lot less scary than you might think.

So the code in the Button's Click event procedure to open frmContacts would be:

Const FORMNAME = "frmContacts"

DoCmd.OpenForm FORMNAME, _
    DataMode:=acFormAdd, _
    WindowMode:=acDialog, _
    OpenArgs:=(Me.StudentID)

Opening a form in dialogue mode means that it must be closed before the user can return to the calling form.  This isn't always a good idea, but in this case I think it's probably appropriate, but if you'd prefer not to just remove the 'WindowMode:=acDialog, _' line from the above code.

Having passed the StudentID to frmContacts you now want it to be automatically assigned to the StudentID control in that form.  The best way to do this is to make it the control's DefaultValue property.  This puts the value in the control when the form is at a new record, but does not initiate the creation of a record as such until the user adds more data in the form.  Consequently they can back out if they wish simply by closing the form before adding any more data.  To do this put the following code in the Open event procedure of the frmContacts form:

If Not IsNull(Me.OpenArgs) Then
    Me.StudentID.DefaultValue = """" & Me.OpenArgs & """"
End If

The DefaultValue property is always a string expression regardless of the data type of the field in question, which is why the OpenArgs value is wrapped in literal quotes characters in the above code.

If you are unfamiliar with entering code in event procedures this is how it's done in form design view:

1.  Select the form (by clicking on the small black square in its top left corner) or the control and open its properties sheet if it's not already open.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new lines between these.
