rohn007 say:

You are talking about a process known as ETL, Extract - Transform - Load, not "normalizing".   Normalizing a database is the process of optimizing the DESIGN of the database, not the data.

You already have the "extract" part down, you are getting data from external sources to load into your spreadsheet/database.

Now you want to work on the "transform" part. Bascially what you want to do is read an incoming record, check the value in the partner name against your list of variations. If you find a match, insert the 'standard' value and write the record out to the "good" file.  If you don't find a match, write the record out to the "error" File. If you don't want to split the incoming file into 2 file, you could set an "error" flag.  After finishing the transform you have to go back and "manually" look in the error file, or rows marked as error.  A person has to determine what the correct value is and either change or better, add a new row to the list of variations.  Then you can re-run the transform to fix the remaining problem row.

Finally you load the corrected rows to the final location.

There is a technique you may want to look into. It is called "Soundex". It is a technique originally designed for the US Census for translating names into a standard format based on the sound of the name. The benefit of the technique is that it reduces the effect of simple misspelling to group the names closer, if that is an issue with your data. You would create a new helper column for the soundex translation, then sort the file on the soundex column before doing the transform.

You have 19 accounts in your profile each with their own data file or 1 data file with 19 suggested contacts folders?

The first (1 suggested contacts folder per data file or mailbox) is by design. If you have more than 1 per data file, something is wrong.

The purpose of suggested contacts is save addresses you send mail to who are not in your contacts - without cluttering up your contacts folder. You can configure Outlook to not save these contacts but if you delete the folder (using MFCMapi) it will come back.

You can hide the folders you don't use on the navigation modules by creating groups (i call my groups "Hide") and dragging the folders to the group and collapsing it.
