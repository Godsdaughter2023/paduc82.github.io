It is possible to determining word versions of documents from the - Microsoft Community

ora1 say:

It is possible to determining word versions of documents from the computer that originally created?

Hi,

i need some help, It is posoble to determining word versions of documents from the computer that orignaly created?


macropod say:

Ignore Joan Pin's advice: it's completely unreliable. Documents with the .doc extension can be created in any Word version and documents with the .docx extension can be created in any Word version from Word 2007 on, plus Word 2003 & earlier if the docx converter has been installed. As if that's not enough Word can easily convert documents between the doc and docx formats - or even from formats belonging to other word-processors (e.g. Wordpad, WordPerfect, Open Office Writer, etc.).

The simple fact of the matter is that there is no reliable way of determining which version of Word a document was created with, or even if it was created in Word. There can sometimes be ways of determining what the earliest possible version a Word file was last saved in, but that relies on the document containing features not supported in earlier Word versions. For example, table centring was added in Word 2000, ISTR, so a document containing that property won't have been last saved in Word 97 or earlier. But that still leaves nearly 20 years of Word versions it could have been saved in. And that still doesn't tell you whether the document was created in Word 97 or earlier.

=EOMONTH(A1,CHOOSE(1+MOD(MONTH(A1),3),-3,-1,-2))

But I wonder if there is something simpler.

jeeped say:

I wouldn't say this is so much simpler as merely an alternative method of the same result,

=DATE(YEAR(A1),FLOOR(MONTH(A1)-1,3)+1,0)

If you are playing with dates, it may be of worth to note that the EOMONTH() function is imitated here by the practice of placing a zero in the day parameter of the DATE(<year>,<month>,<day>) formula as in,

=DATE(2010, 3, 0) returns 28-Feb-10

It's just something I've found handy.
