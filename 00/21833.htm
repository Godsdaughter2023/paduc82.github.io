Sub SheetDelete()
    Dim i As Long
    For i = Sheets.Count To 1 Step -1

ossiemac say:

If I am interpreting your question correctly then I think minus 12 will be better applied directly to the date before formatting it to text and then use Select Case.

When using a formula for Select Case then set it for a formula returning True as per the following example.

Ensure you have a backup of your workbook in case it does not work as desired because you cannot get deleted sheets back.

Sub SheetDelete()
     Dim i As Long
     Dim bolDeleteIdentifier As Boolean
    
     For i = Sheets.Count To 1 Step -1
       
        bolDeleteIdentifier = False 'Initialize to false
       
        Select Case True
       
            Case Sheets(i).Name Like (Format$(Date - 12, "yyyymmdd")) & "*"
                bolDeleteIdentifier = True
            Case Sheets(i).Name Like (Format$(Date - 11, "yyyymmdd")) & "*"
                bolDeleteIdentifier = True
            Case Sheets(i).Name Like (Format$(Date - 10, "yyyymmdd")) & "*"
                bolDeleteIdentifier = True
            Case Sheets(i).Name Like (Format$(Date - 9, "yyyymmdd")) & "*"
                bolDeleteIdentifier = True
        End Select
       
        If bolDeleteIdentifier Then
             Application.DisplayAlerts = False
             Sheets(i).Delete
             Application.DisplayAlerts = True
        End If
     Next
 End Sub

ossiemac say:

Or you might prefer the following method.

Sub SheetDelete()
     Dim i As Long
     Dim d As Long
    
     For i = Sheets.Count To 1 Step -1
       
        For d = 12 To 1 Step -1
            If Sheets(i).Name Like (Format$(Date - d, "yyyymmdd")) & "*" Then
                 Application.DisplayAlerts = False
                 Sheets(i).Delete
                 Application.DisplayAlerts = True
            End If
        Next d
       
     Next i
    
 End Sub

Excel is exactly the same in all editions of Office 2010 except the Starter Edition (which is very limited).

 

If a cell contains more text than fits in the cell, and if text wrapping has not been turned on in that cell, the text will spill (overflow) into empty cell(s) to the right. justin cavida msft say:

Hi Allan,

This issue probably caused by the blank rows in your sheet. When you select just the header of a table when you invoke the filter, Excel tries to be smart and includes what it calls the "Current Region" into the filter area.

As a workaround, you can select the entire sheet first by pressing Ctrl + A before enabling the filter in Home tab.

Justin
