Long story short - Using a BIOS direct from the mobo manufacturer hosed my OEM-supplied pc's Windows activation because it not contain my OEM's SLIC table.  The OEM (Microcenter/PowerSpec) supplies the pc with a "tattooed BIOS" so that the OEM SLP license self-activates.  If Microcenter would have had BIOSes available on their website for my model, I never would have gone direct to ASRock in the first place (and I would have been able to fix myself a lot sooner in the second place).

(Additional hard drives had nothing to do with it.)

!!

This is addressed to everybody with the same issue. 

The last person who actually was any help was the support guy at Microcenter (who makes PowerSpec brand pcs) via chat and email.  My email response and thanks to him (not Andre) is pasted below.  All the relevant info is there. 

OK so maybe I was not so familiar with new Windows licensing and activation techniques, technology and terminology in the beginning, but I sure did get an education about all of it.  Sorry about all the rambling, but I hope that all of the detail helps others. 

Begin paste of email response to Microcenter here:

+++++++++++++++++++++

Thank you for your help. 
I solved my problem on my own.  (My problem was that Windows had become de-activated, and would not re-activate.)
 
I'll tell you what I did in detail below, but to shorten a long story, if we would have been able to get a BIOS supplied by Microcenter/PowerSpec, we would have been fixed. 
 
Although the information at the link (http://forums.mydigitallife.info/threads/10950-howto-Enable-the-quot-activate-by-phone-quot-option) you provided me did help me to get Windows to generate the 54-digit Installation ID that is needed by the Phone Activation Operator, the operator said that my number was not correct or not able to produce an activation.  He went so far as to tell me that my copy of Windows was probably counterfeit and even if it is not, that the counterfeit reporting process would eventually lead the OEM manufacturer to correct my situation.  Sounded like BS to me, but I must confess that I was desperate and I did start the process.  However I received an email about the next steps to follow which included mailing to Microsoft the Windows installation discs (with no ability to have them returned), and I gave up on that process.
 
What led me to my solution were other threads at http://forums.mydigitallife.info/. 
It turns out that my SLIC table was blown away as was noted in the output of MGADiag.exe ("BIOS valid for OA 2.0: yes, but no SLIC table").  This had to have happened when I installed a newer BIOS downloaded from the ASRock (motherboard) website.  Without the SLIC table, Windows could not self-activate.  (See http://en.wikipedia.org/wiki/SLIC_table#SLIC.) 
 
Luckily for me, I had another PowerSpec pc at home and so was able to use tools referenced in other threads at http://forums.mydigitallife.info/ to do a dump of its 374-byte SLIC table, save it and then "tattoo" the clean ASRock BIOS with the SLIC table and reinstall it.  The process is not for the faint-of-heart and is used widely to enable illegal installations of  Windows.  The tools needed to do this were quarantined by Norton Antivirus, however I was able to restore and create exceptions for them. 
 
When we were on the phone, we had suspected that getting a BIOS from Microcenter/PowerSpec would probably fix me, but you had no way to get me one.  There are no BIOSes available on your websites and you did not have one to send me.  The reason I installed a new BIOS in the first place was because the version out of the box was real old - version 1.x (don't recall exactly) versus the current 2.7.  If you don't have current versions of PowerSpec-tattooed BIOSes, you need to at least have old versions available.  Many people will do what I did because they will want an up-to-date BIOS.  Then they will need to downgrade to an old version again to re-activate. 
 
When you asked your people how to fix me, you were told I have to take the pc to a Microcenter store so that they can re-insert the SLIC table.  Well as I now realize, all this means is getting a BIOS tattooed by the OEM, so that the automatic OEM Activation system works (OA 2.0 I think).  At the store all they would have done is install a proper PowerSpec-tattooed BIOS.  I should have been able to download it myself, or you should have been able to send me one. 
 
I had backed up my Windows product key originally using a VB script provided by Microsoft before upgrading my BIOS thinking that this was all I needed.  I was aware that there are no COA stickers anymore.  Turns out that the Product key is an OEM key (sort of a volume key) and not a unique key.  It is known as a SLP license key (System Locked Pre-installation).  Attempting to "Enter a different product key" did not work with my actual product key because OEM SLP keys HAVE to self-activate using the SLIC-tattooed BIOS from the same OEM (and other factors such as a certificate).  You can't force activation over the internet without a valid matching SLIC table (the error will mean invalid key or key already in use) and phone activation won't work either.  That is why the option to phone-activate is not present on OEM SLP installations when you do "slui 4."  As far as I can tell (I could be wrong on this), an OEM SLP-licensed pc seems to self-activate without requiring connectivity (internet or phone).  If it ever does "phone home" it will probably be only to report that it is activated already. 
 
A symptom to look for in the future with other customers:  When the pc was new out-of-the-box, the BIOS flashed a PowerSpec logo during boot.  After flashing the clean non-PowerSpec-tattoed BIOS from ASRock, the logo was ASRock.  After restoring the SLIC table to the BIOS (tattoing), the PowerSpec logo was back. 
 
I hope all this rambling detail will help you and future customers. 
 
I would like to re-iterate that having BIOSes available on your website would have let me (us) fix my issue more quickly.

Hi,

 

Is the Recent Items in the start menu empty?

 

Method 1:

 

Follow these steps.

 

a.       Click on Start and then on run.

b.      In the open box type in the following and hit enter.

 

%AppData%\Microsoft\Windows\

 

c.       Right click on the folder “Recent” and select delete.

d.      Click the Start, type "shell:recent"  (without quotation marks) in the Start Search box and press Enter.

Note: The above command re-creates the Recent folder, along with a new desktop.ini file and opens the folder.

 

e.      Restart the computer.

Perform a system file checker scan on your computer and check for the issue.
