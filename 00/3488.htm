Ripping music to collection-audio format - Microsoft Community

windypilot say:

Ripping music to collection-audio format

Can anyone tell me what the differences are between the four selectable audio formats when ripping music to my collection? Is one format better than the other and will it affect quality if I elect to burn any of the tunes to a cd? Thanks
nsilmike say:

windows media format is microsoft's creation.  MP3 is the original digital audio format, invented back in the 1970's I believe... wikipedia will probably have a decent article about it.

wma is generally higher quality/more compact than MP3, at similar bit rates.

MP3 is universal, while wma is widespread but certainly not universal

cbr means constant bit rate (it rips at a constant bit rate or constant 'resolution.')  vbr means variable bit rate, so you get more efficient file sizes (bit rate goes up when necessary and down when possible.) Bit rate or 'sampling' resolution tells you how 'close' to the original analog signal you can get... but higher resolution means bigger files- as bit rate goes up, the system is sampling and converting the original signal at a faster rate (samples per second.)  Lossless is well...lossless.  none of the information in the original analog has been lost due to the compressed digital format.  BIG files.  The losses in digital conversion are due to both quantization error... look at wikipedia again for this if you really want to get this deep...which is unavoidable, and also due to compression techniques. (Lossless conversion means only that compression losses are avoided by the way.)

If you rip then burn to a data CD, you will have no further loss of info in the burning process (you are just copying the file bit by bit.)  If you burn to an audio CD, you are changing formats, and you will have further losses....thus starting from the highest quality digital would give the best result.

Primary key autofill - Microsoft Community

whitney f say:

Primary key autofill

I am creating a form for entering in new personal. I am using the username as my primary key. Right now, I have to fill in the username for every table used to ensure the persons information will show in all 7 tables used.

 

I am looking for a shortcut that will allow me to enter in the username once in the main table's field and duplicated it to the other 6 tables. Any ideas??

 

Thanks

ken sheridan say: 1.  If you do use UserName as the primary key of the main table (let's call it Employees) you would also have a Username column in the 6 referencing tables.  T o enter these in forms bound to each of these tables you would use a combo box as the bound control, with a RowSource of:

SELECT UserName  FROMmployees ORDER BY UserName

2.  If you have an EmployeeID autonumber as the primary key of Employees, then you'd have an EmployeeID column in each of the 6 tables, but not an autonumber this time.  You would not have a UserName  column in theses tables.  The combo box would now be set up as follows:

ControlSource:    EmployeeID

RowSource:     SELECT EmployeeID, UserName FROM Employees ORDER BY UserName;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column.

You now simply select the Username from the combo box's list.

Whichever way you do it, if you use subforms within the Employees form for the other 6 tables you don't need to select the Username at all for the 6 subforms.  By linking the parent form and the subforms on the keys (Username or EmployeeID) the value will be automatically inserted into the referencing table when you enter other data in the subform.

Windows Registry Editor Version 5.00
