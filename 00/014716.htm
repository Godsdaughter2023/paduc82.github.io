bernie deitrick say:

You have a lot of options, and the easiest is to use a column of formulas with the first table.

In E2, use the formula

=IFERROR(MATCH(A2,F:F,False),COUNTA(F:F)+ROW())

Then copy down to match your first table. Sort A:E based on E ascending, and all the data that matched will be sorted onto the same row as its matching value. Then just copy A:D over to F:I for the rows where F:I are blank and you're done.

Hello Arlaan,

There are some reasons why you would not see the Restore function under Settings>Phone>Update.

  • If this computer you are connecting the phone to has never updated through the Zune Software, specifically on this computer, the Restore function would not be present.
  • If you have uninstalled and reinstalled the Zune Software, the Restore function would be missing, as the backup would have been erased during the uninstall.
  • If you have more than one user profile on this computer, and the phone was updated on a different user profile, Restore would not be available.

jim gordon mvp say:

For once Arshad is on the right track.

The format that is saved when Excel exports as .csv depends on the cell formatting.

Click on the letter for a Column and press command-1 to see the current formatting for that column. Excel's default is General format, where Excel figures out what kind of data it is and applies default formatting. However, since you did not make the source workbook, you have to look at the column format to see what it is.  

In the default General format, Excel displays and will export date as dd/mm/yy

If you want Excel to export dd/mm/yyyy you must first select the column and apply that Date format instead of General format.

You can verify the result by opening the saved .csv file in TextEdit.
