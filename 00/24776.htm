albert d. kallal say:

It not quite clear of your context.

It was “very” common in VB6 to use the access database system.

Remember, Access has two parts:

The “UI” part. This lets you write VBA code, build an application, and build reports.

And then there is the database “engine” part.

As far as I know, the choice and ability to use Access as the database is un-changed.

So in terms of compatibility, you are STILL free to use Access as the database with your VB6 application. Nothing has really changed in this “long time” choice of writing an application in VB6, and using the database engine from Access.

About the only change is that the default database engine used with VB6 was called JET, and now the database engine is called ACE. The main difference is that ACE can handle the newer ACCDB file formats, were as the default and “often” used JET

So if you question is in the context of using Access as the database system with VB6, then that choice remains un-changed, and this “long time” history of using the Access database engine with VB6 choice remains a viable choice.

The other issue is now office and Access offer true x64 bit versions of Access, and these versions cannot be used with VB6, since only an x32 bit version of VB6 is available. Thankfully, the vast majority of Access (and office) installs are x32 bits, even on x64 bit windows.

However, over time, the industry is slowly moving towards x64 bit office and Access. These versions of Access cannot be used with VB6. I can’t say when this changeover will represent the vast majority of office installs. I will say I am seeing more and more companies go with x64 bit versions of office, and that is a problem for VB6.

grschinon say:

SIP is the protocol (Session Initiation Protocol) that is used to set up a point-to-point session for real-time communications. It can be used for several things, one of which is VoIP.

 

The communication takes place over an internet-type connection, so you need to have either of the following for it to work:

 

1) WLAN access (the E72 does have this)

 

2) Packet data with sufficient speed (3G or 3.5G) and authorization from the network operator to use it for VoIP (most of the time it is forbidden).

 

You need to get service from a VoIP provider. SIPGate is popular in several countries. I use voiptalk.org myself.

 

Optionally, if you have multiple SIP-capable devices at home (I do), you can set up a server running asterisk, have that connect to your VoIP provider and use it as a PBX to which all your SIP phones are connected, allowing you to pick up the call with any one of them and transfer calls between them, set up conference calls etc.

 

The SIP profile on the E72 is a collection of settings that allow your phone to connect to the SIP server: login, password, server name, proxy, transport protocol etc.

 

In order to be able to use VoIP over this SIP connection you need a small application from forum.nokia.com available here (you want the SIP VoIP 3.x Settings app for the E72). This will set up a new "Advanced VoIP Settings" entry under menu > control panel > net settings.
