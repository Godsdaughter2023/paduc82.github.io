pamcaswell say:

You can use find and replace to change from one style to another.  And you can also use the select all instances feature in the styles pane to change the selected items by applying your style. With either method, start with the less used and specialty styles (captions, headings, numbered and bulleted paragraphs, and such) and end with the most regular text paragraphs (like normal).  If you don't  have a lot of manual formatting that must remain, you can use either of the methods to change the  regular text style to you style  If you do have manual formatting,  then making the change paragraph by paragraph may be better.

I agree with you that switching style sets is a faster way to bring styles into a document. But If you apply styles from the styles gallery, make sure the style set has all the styles (and style settings) you need.  Also, since changing style sets also changes the document defaults (styles pane > manage styles > defaults tab), watch out for unexpected results.  I suggest you change the defaults to what you want them to be in your documents before you create the style set. 

You mentioned renaming "default styles".  Note that renaming built-in styles just adds an alias to the built-in name.

Pam

I’m always interested in algorithms, so if you don’t mind going off topic a little, here are some thoughts.  This problem has 24 input variables, about half are negative.  One can do a brute-force method by looking at all subsets (2^24, or 16,777,216).  This takes about 1 minute, but it’s not very efficient because of the large array.  It’s actually not a problem, but it is a little large.  Another ‘issue for this method is that for some reason, this particular function can’t split the workload  using parallel computation.  (My particular multicore computer has 8 kernels).   I’m not really sure as to why this particular one can’t split the workload.


Another option for math programs is to limit the size of our search to a particular smaller size, and hope we can find a solution.  Combin(24,7) = Combin(24,24-7) = 346,104.  These sizes are much more manageable.  We don’t find a solution with up to 7 values, but we do get lucky in finding a solution with 17 values.  With this method, it took about 2 seconds!  This is much faster.  What we hope to avoid is having a solution with 24/2 = 12 values.  This is the biggest search, as there are over 2.7 million possible values.  By avoiding these, we speed it up quite a bit.


So, the question is .. can we beat 2 seconds, while searching the entire possible list?


Suppose we had a list {1, 2, 3, 4, 5, -6, -7}, and we want a total of -2.  This is similar to your problem.

Now, we look at just the negative values in our list.  Suppose we ‘assume that only -7 is in our final answer, and not -6.  What we have is:

{1,2,3,4,5, -7}  = -2.

If we add 7 to both sides of the equation, we have

{1,2,3,4,5} = 5


So now, we are looking at only the positive values that add to 5.

They are:

{5, {4,1}, {3,2}


Now, we add the -7 back in:

{5,-7},  {4,1,-7}, {3,2,-7}


So, what we do in split the input into 2 smaller arrays of the following sizes.

1024, and 16384.

These can be done almost instantly, and avoids 1024 * 16384 = 16,777,216.

We pick the intersection of these 2 arrays, which means left side = right side of our equation.

We then join the values from both the positive and negative list.


Done in this way, the solution is found in a whopping 0.029 seconds.

Second, I don't know why Harlan Grove's macro takes 7 minutes for 24 values.  I never looked at it.  My VBA implementation below is relatively straight-forward, and it takes less than 60 seconds.  It takes less than 0.5 second for data with 16 values.  Of course, those times vary from computer to computer.

Arguably, that is not impressive compared to the processing time that Dana reports using Mathematica.  On the other hand, I wonder if Mathematica stops after finding the closest sum using the fewest values.  That requires iterating over all 2^24-1 sums [1].

If you are looking for a Solver alternative that can be used with Excel, you might try the VBA function below.  It allows for up to 31 values [3].  But for 31 values, I estimate it would take nearly 3.5 hours(!) on my computer.  Not very practical, IMHO.

Usage.....

Suppose the data are in A1:A24, and the goal value is in C1.  Select B1:B24 (or B1:B25), type =bestsum31(C1,A1:A24), and

The result is the list of values that result in the best sum, as defined above.  If you select B1:B25 (one more cell), B25 contains the processing time in seconds.

You might also enter the formula =SUM(B1:B24)-C1 to check the result.

The VBA function.... [2]


Option Explicit

Type bestT
    diff As Double
    cnt As Long
    n As Long
End Type

Function bestSum31(goal As Double, rng As Range)
Dim st As Single, et As Single
Dim v As Variant, nv As Long
Dim i As Long, x As Long
Dim mySum As Double, myCnt As Long
Dim n As Long, maxN As Long, t As Long
Dim diff As Double
Dim best As bestT

st = Timer
v = rng              ' input range of values
nv = UBound(v)
ReDim res(1 To nv + 1, 1 To 1) As Variant
For i = 1 To nv: res(i, 1) = "": Next
If nv > 31 Then res(1, 1) = CVErr(xlErrValue): GoTo done

' find best sum
maxN = 2 ^ nv - 1
best.diff = 1E+308
For n = 1 To maxN
    mySum = 0: myCnt = 0
    t = n: x = 0
    Do
        x = x + 1
        If t And 1 Then
            myCnt = myCnt + 1
            mySum = mySum + v(x, 1)
        End If
        t = t \ 2
    Loop Until t = 0
    diff = Abs(mySum - goal)
    If diff <= best.diff Then
        If diff < best.diff Or myCnt < best.cnt Then
            best.diff = diff
            best.cnt = myCnt
            best.n = n
        End If
    End If
Next n

' output best sum
i = 0
t = best.n: x = 0
Do
    x = x + 1
    If t And 1 Then
        i = i + 1
        res(i, 1) = v(x, 1)
    End If
    t = t \ 2
Loop Until t = 0

done:
et = Timer
res(nv + 1, 1) = et - st
bestSum31 = res
End Function

  2. Copy the VBA text above, and paste into the VBA editing pane on the right.

  3. You can close the VBA window now.

  4. Save your file with the "xlsm" or "xls" extension.
