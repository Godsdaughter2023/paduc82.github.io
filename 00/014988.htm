scottgem say:
An option is a list control. It only allows you to select ONE item from the list. You can mimic the look of an option group as DBGuy describes, or maybe you need a MVF or a junction table. It would help to understand the data you are modeling.

ken sheridan say: If the control is an unbound control, used for searching on the basis of multiple values for instance, use a multi-select list box.  You'll find examples in MultiSelect.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

If you attempting to use an option group as a bound control to select multiple values in the context of a many-to-many relationship type, e.g. multiple courses for which a student is enrolled, you cannot do so.  It is possible to use an unbound multi-select list box in this context, but it requires code behind the scenes to read and write date from/to the table.  The StudentCourses demo in the same OneDrive folder includes an example.  Far simpler, and code-free, is to use the conventional interface of a subform based on the table which models the many-to-many relationship type.  The StudentCourses demo includes an example.

Hello Martin,

Please be informed that Windows Temporary files are created by the operating system during the normal course of its running.

The Temporary Files in Windows are typically found located in two locations:

  • %systemdrive%\Windows\Temp
  • %userprofile%\AppData\Local\Temp

Once you access the above folders, you will see that most of its contents are .tmp, .temp and .txt files.

In order to locate temporary folder in Windows, follow the below steps:

  1. Click Start, in the search box, type CMD.
  2. Right click on CMD and choose run as Administrator.
  3. In the command prompt type ECHO %Temp%.

This displays the full path of the actual temp folder and you can follow the path and identify its location by copying the entire path and pasting it in Run command or in search options.

You can deploy the Disk Clean up utility to free up some space. Follow the below steps.

1. Press the Windows + R keys to open the Run dialog, type cleanmgr, and click/tap on OK to open Disk Cleanup.
2. If you have more than one drive or partition on your PC, then select the Windows C: drive or the drive letter you wanted to clean, and click/tap on OK.

john - project say:
VR TC,
It's Thursday morning and I mulled over the response I prepared below and realized I had made a couple of errors and some clarification was in order, so this is the edited version of last night's response.
 
Okay, it's later and I took a look at your example. First let me make a couple of comments on your first task.

1. Leveling is not something Project does on its own. Leveling is a function selected by the user under Resource/Level group. So, you do NOT need to set the Max Units to 10,000%
Edit: Setting a very large max units will prevent the resource from showing as overallocated but will NOT turn leveling on or off. However, overalloccation is designed to tell you something, namely that the resource is assigned to work more hours than there are in his/her work day.

2. You mention that everything in the "Resource area" (I assume by that you mean the Resource Sheet), is Project default yet I see a Base Calendar of "FiveDayWeek". That is not the default. The default calendar is "standard". I asked you about that in my initial response.

3. Why do you set the remaining duration at all? Project calculates that automatically based on the difference between the duration and actual duration. In your case the actual duration is being calculated by Project based on you using the Update Schedule function under Project/Status group/Update Project icon

4. You can use manual scheduling mode if you wish but it isn't necessary simply because you have no task dependencies. I find that manual scheduling is actually more confusing, probably because I'm so used to the auto-schedule mode that has been a basic function of Project for many many years. If I were you, I'd use auto-schedule

Now to answer you question about the weird data you are seeing. I actually found out something that I did not know but alas I'll probably forget it after I explain it to you. I'll use your task one (Oak street) as the example. Your screenshot is too small so I can't readily discern what status date you are using so I'll just pick the 1st Friday in November (i.e. 11/4/13). 

WIth no grouping applied, create a summary line wherein the Oak street task with "G" as the sole resource is the only task under the summary. If you then use the update status to the status date, Project will show a value of 95% in the % Complete field, even though the actual value is 94.596% (i.e. 350d actual duration/370d duration). The summary task will show the same value because percent complete for a summary line is the sum of the actual durations of all subtasks divided by the total duration of all subtasks. Again, Project only show percent complete in whole numbers. So far, so good.

Edit: I made an error in my explanation, this is the corrected version.
Now here is where it gets interesting. If you do basically the same thing but use grouping to create the summary line the results are a little different. For the group summary line, Project does NOT calculate percent complete using the same formula as for a summary line, rather the group summary line will take the rounded value of percent complete value, (i.e. 95% instead of the actual 94.596%), and use that to calculate the actual duration. Thus the group summary line will show 351.5d of actual duration. And of course remaining duration is simply the difference between duration and actual duration, so in this case it will be 18.5d.

I didn't dig in to the 3 task set, but the above analysis will certainly flow into the reason behind the values you are seeing for group summary lines.

So how does all of this relate to your end goal of keeping track of field staff? Well, I wouldn't use grouping, I'd use the Resource Usage view. It details the assignments of each resource.

Hopefully I haven't completely confused you. I'm already confused most of the time so having two of us in that same state would be just too much to handle, especially so close to the holidays.
