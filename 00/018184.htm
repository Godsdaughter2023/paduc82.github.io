helenmcclaine say:

Nested IF using dates

We have a spreadsheet showing when vouchers are issued and used.  We want a count by month of the open vouchers.  So if the formula works you'd see:

V Issued  |   V Used      | Aug | Sep | Oct | Nov

8/3/2010  | 11/7/2010   |    1  |  1   |  1   |   0

At the bottom of the list we'd then be able to SUM Aug, Sep, Oct, etc.

We're sure it's going to require nested IFs with (>=8/1/2010) and (<=8/31/2010), but every configuration we've used hasn't worked.

joeu2004 say: Helen McClaine wrote:
We have a spreadsheet showing when vouchers are issued and used.  We want a count by month of the open vouchers.  So if the formula works you'd see:

V Issued  |   V Used      | Aug | Sep | Oct | Nov
8/3/2010  | 11/7/2010   |    1  |  1   |  1   |   0

At the bottom of the list we'd then be able to SUM Aug, Sep, Oct, etc.

We're sure it's going to require nested IFs with (>=8/1/2010) and (<=8/31/2010), but every configuration we've used hasn't worked.

I interpret your requirement to be:  count as one any voucher that is still unused by the end of the month; count as zero any voucher that is used during the month.  Hence, Nov is zero because the voucher was open in or before Nov and used during Nov.

There are many ways to express that logic.  One way for Aug, assuming "V Issued" and "V Used" are in columns A and B starting in row 2:

=1*AND($A2<=DATE(2010,8,1),OR($B2="",DATE(2010,8,31)<$B2))

The 1* converts that boolean values TRUE and FALSE to the numbers 1 and 0.

You cannot write <=8/1/2010 because in that context, 8/1/2010 is interpreted as 8 divided by 1 divided by 2010.

You could write <=--"8/1/2010" and --"8/31/2010"<.  Note the double negation (--), which causes Excel to interpret the date strings as numeric dates.  But that can cause problems if the Excel file is used by someone in another country with different Regional and Language settings.

If the titles Aug, Sep, etc are actually dates formatted as Custom mmm, the following formula would make it easier to copy down and across, assuming that Aug is in column C:

=1*AND($A2<=DATE(2010,MONTH(C$1),1),OR($B2="",DATE(2010,1+MONTH(C$1),0)<$B2))

If you are only counting open vouchers in 2010 and the title dates are 8/1/2010, 9/1/2010, etc, you could write:

=1*AND($A2<=C$1,OR($B2="",EOMONTH(C$1,0)<$B2))

Example:            page 10                                       Conclusion:

                         doc 1 xxxxxxxxx                             xxxxxxxxxx

                                   xxxxxxxxx                             xxxxxxxxxx.  (See page 10) 

                        

                         doc 2 xxxxxxxxx                                   xxxxxxxxx

                        

                         doc 3 xxxxxxxxx                                   xxxxxxxxx

suzanne s. barnhill say:

This is a two-step process:

  1. Bookmark the text you want to reference. Using Insert | Links | Bookmark, you can either add a bookmark around selected text or insert the bookmark at a single point. [By default, you will not see any representation of this in the text. If you opt to display bookmarks (File | Options | Advanced: Show document content), you will see a heavy grey I beam for a point bookmark and heavy grey brackets around selected text.]
  2. Then use Insert | Links | Cross-reference to insert your cross-reference. You'll need to type "See page" because the cross-reference will be just the page number. In the Cross-reference dialog, select Bookmark as the Reference type and Page number under Insert reference to. This will insert a PAGEREF field, which will update if the bookmarked text moves to another page. [PAGEREF fields don't update dynamically (the way autonumbered paragraphs or notes do when you add or remove text), but they do update automatically when you update fields by printing, switching to Print Preview, or using F9.]

So you made the text box unbound? An unbound text box will display the same value in all records on a continuous form. Only bound controls can display a value that depends on the record.

john w. vinson say:

As Hans says, there's only ONE textbox on a continuous form; it's displayed repeatedly.

To display a different formula value on each row, do the calculation in the Query upon which the form is based, rather than in the control source of the textbox: in a vacant field cell in the Query design window type

SomeName: MyFunction(<arguments>)

The function must have at least one table field as an argument; otherwise Access will just call the function once and display the same value for each row.

trevor rabey say:

Well, that's a great question. MSP doesn't optimise anything, mainly because it cannot know what your objective function is, ie doesn't know what you are trying to max or min.

And the scheduling algorithm in the engineroom, whatever it is, is probably not an application of linear programming.

It is true that the longest path problem is often presented as a classic LP problem, and solved in that way in classrooms. Then there is the classic "crashing" based on the idea of finding out which critical tasks can most effectively be pulled off the critical path. Again, in classrooms, but rare in real life.

Stephen Devaux, in his book Total Project Control, identifies that partitioning a set of tasks into critical and noncritical is a good first step but after that you don't know anything about the critical tasks, comparatively, except that they all have zero total float, so there is a need for some way to grade the critical tasks.
