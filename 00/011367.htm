macropod say:

Hi MickiTin,

For the landscape table, add a 'Next page' Section break after the table (but still within the field code) to restore the portrait layout. This may result in blank pages being produced at the end of the records requiring the table, but I suspect that's the best you'll do. Those blank pages can be deleted after the output document is created.

If you want to run the whole process from Excel, you'll need to use a macro, but once you start doing that, you really need to consider why you'd use Word's mailmerge and not simply use Excel to do all the work of populating the documents, saving to separate files, emailing and so on.

As for filtering records and restricting how many records are processed in a mailmerge, Word can do both

For sending mailmerge emails with attachments, see: http://word.mvps.org/FAQs/MailMerge/MergeWithAttachments.htm
