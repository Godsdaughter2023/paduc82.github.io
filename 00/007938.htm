What events are triggered by Requery - Microsoft Community

kevthu say:

What events are triggered by Requery

When a subform control is requeried, what events on the subform does the requery action trigger?

marshall barton say:

Quite a few.  If the current record is dirty, then you might get a control's Before and After Update events followed by the form's Before and After Update and maybe After Insert events and the Dirty property is set to False.  Oh wait, that already happend before the requery because the focus moved back to the main form when you clicked anywhere in the main form.

 

The subform's record source is reloaded, which triggers the subform's Current event as the first record is made current.  There may be some painting, etc. events too.  A good way for you to check this is to add a Debug.Print "name of event" to every possible event procedure in the subform (tedious, but faster than trying to get a definitive reply from a forum).

 

I always use the syntax  Me.subformcontrol.Form.Requery  to requery the subform so I'm not sure requerying the subform control does the same thing.  It used to in older versions, but I haven't tried requerying the subform control in a long time.
