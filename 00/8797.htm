hansv mvp say:

I would test whether there are matching records in the On Click event procedure of the command button on form(1) that you use to open form(2).

Let's say:

  • The text box is named txtSearch.
  • The command button is named cmdOpenForm2.
  • The client table is named tblClient.
  • The field you search on is named Surname.
  • The name of form(2) is frmClient.

The code could look like this:

 

Private Sub cmdOpenClientForm_Click()
    Dim strWhere As String
    ' Assemble where-condition from text box
    strWhere = "[Surname] Like" & Chr(34) & Me.[txtSearch] & Chr(34)
    ' Check if there are matches
    If DCount("*", "[tblClient]", strWhere) = 0 Then
        ' If not, select the search box
        Me.[txtSearch].SetFocus
        ' Display a message
        MsgBox "There are no clients matching the criteria that you entered!", vbExclamation
        ' And get out
        Exit Sub
    End If
    ' If we get here, there are matches
    ' So we can open the client form
    DoCmd.OpenForm FormName:="frmClient", WhereCondition:=strWhere
End Sub

Hi SimonMallmoa

It sounds like you company could be using a certificate that it created internally for the OWA or the certificate that it's using is from a certificate authority that's unknown to the device.  If you try to access the OWA site from your home computer or any computer that's not a part of your company do you get a certificate error?  If you do get that error if you click on the warning you should be able to veiw the crtificate the server passed to you.

The error you're getting on your device when you try to access the site via IE indicates that there's still something about the certificate that the device can't verify.  It's possible that there's some extra certificates that may go with the one you put on the device already. 

Using IE to access the OWA site is a great way to verify your device trusts the OWA site.  Once you can access the OWA site through IE without any warnings you should be able to sync with Exchange Active sync just fine.

eric hicks say:

Hi SimonMallmoa,

 

Ok upon first try I did get the certificate error.  I checked and your server does have a valid root certificate and server certificate as well.  The problem comes from the intermediate certificate.  It's a valid certificate but your device doesn't have that certificate installed and without that certificate the server certificate can't be verfiied.  I was able to access the page without issue from my desktop though.  So here's how to get it working.

As a note this procedure was done using Internet Explorer

Using your computer's web browser goto your companies webmail site, you don't have to login.  Once at the main page openinternet options in IE and click the Content  tab.  Under Certificates click theCertificates button.  Click th tab named "Intermediate Certification Authorities" and scroll to the bottom of the page and look for "Verisign Class 3 Secure Server CA - G2". Select this certificate then click Export.  This will launch the certificate export wizard.  ClickNext  on the cert wizard welcome page.  On the next page make sure "DER encoded binary X.509 (.CER)" is selected then clickNext.  Select the location for the exported cert to go using theBrowse button and also give it a name and click Save.  ClickNext on the file to Export page then Finish and now you should have a certificate file waiting for you in the location you selected.  Copy that certificate to your device and using the Windows Mobile File Explorer click on that file.  It should install onto the device.  Open the Certificates manager on the device which is located under the System Folder in Settings.  Tap theIntermediate tab and you should see the cert you just imported.  It will say "Verisign Trust Network".  Now it's time to make sure the device likes it.  Using Internet Explorer on your device navigate to your webmail page.  If all goes well you shouldn't be presented with any certificate warnings.  Once this works try syncing you device again.  Now you should be able to setup Exchange activesync minus the /owa in the address name and get connected.

MVP-Windows Phone
