Sub Project/Master Project and What NOT to Do Advice in MSProject - Microsoft Community

sandra500 say:

Sub Project/Master Project and What NOT to Do Advice in MSProject 2007

My business is discussing ways to reorganize our 45,000 task MSProject 2007 schedule file that is 8 years long.  We'd like to break it up into about 8-10 subprojects that include subproject links.  Each subproject will have it's own manager.  We attempted to do this same thing about four years ago on another job using MSProject 2003 and it caused a lot of headaches.  One of the most difficult issues were the inadvertant creation of circular links, even though we followed best practices (no summary links and no file moves).  I know we're not the ony company who deals in schedules this large. 

I would like to ask the user community how they handle extremely large files and lengthy programs.  What should we NOT do when breaking the file into subproject as well as manage them?  What are the pitfalls using this method?

Thank you for your imput.

deanlc say:
  1. Keep all files together. Whenever you make a copy, copy every file that is linked to any file that will be edited. Do not save any of the unedited files.
  2. Make a separate consolidated schedule for each manager. Set the Subproject Read Only field to yes for all subprojects not owned by the manager. This will allow the manager to analyze network impacts and not lock files owned by another manager.
  3. When you open the master with Subproject Read Only field equal to no for all subprojects, do not rely on opening the master to control the subprojects. The master may have a subproject collapsed, and you have not opened the subproject read/write until it is expanded. After showing all outline levels, save the file and respond to each subproject save box to assure that you have control of the entire schedule.
  4. At least once a week, open each subproject file individually, look at the Links Between Projects dialog box and resolve differences.
  5. Standardize custom fields, setting aside specific fields for subproject usage. Periodically use Organizer or a macro to copy standard fields to all subproject files.
  6. Create a subproject field that concatenates subproject file name or standard code and Unique ID. This will give you a stable Unique ID.
  7. At least once a week, display the master in the network diagram view. If it can be displayed then you do not have circular relationships. 
  8. At least once a week, open the master then open the Visual Basic Editor. If you get a file not found error, you have a corrupt file, most likely caused by mixed (2003 versus 2007) version edits to the same file.
  9. If you maintain y to the server.

john - project say:

Sandra500,

I'll share some of my experience with you, hopefully others will also.

First of all, whether you are working with a single 45,000 task schedule or the equivalent broken into multiple subprojects and a master, there will always be issues. Have you considered Project Server? It might be a better match for your needs.

You mentioned your previous problems with circular references when attempting to create a master/subprojects structure. With that type of structure it is quite easy for links between files to get "over done". I've seen user files that had a plethora of unnecessary links to a separate milestone file and other passing of information between files that really didn't relate to logic flow. Quite often this can lead to "backward pass" links which may not cause problems at the subproject level, but can immediately trigger a circular reference fault at the mater level.

You've already noted that you understand the "rule" not to link summary lines. Good for you. That is the number one cause for circular references. You also understand the "rule" about moving file in a master/subproject structure. Again, good for you. That is the number one cause for file corruption. You should also avoid "saving off" periodic copies of the master/subproject structure for backup as this can create duplicate links, particularly if there are interfile links.

Hope this helps.

John

jllatham say:

Here is code that should work for you.  I've based everything on what you've posted here.  So I think the only thing that you will need to edit is the server name and top level folder name which is associated with the Constant value serverPath in the code.  Now, you said beginning at row 9, but your graphic shows row 8 as first possible entry, so I've coded it that way.  Again, you can change any of the Const values as needed to match your setup.

 

To put the code into your workbook (a test copy of it would be good to work with at first) - open the workbook, select the proper sheet, then right-click on the sheet's name tab and choose [View Code] from the list that appears.  Copy the code below, paste it into the code module presented to you, edit it as required.  Then you can close the Visual Basic Editor by either clicking it's red-x or pressing [Alt]+[Q].  Save the workbook as a macro-enabled workbook, type .xlsm or .xlsb.

 

Take it for a test drive, we'll have the sales papers all ready for your signature when you return, and our interest rates are really low this week. :-)

 

Private Sub Worksheet_Change(ByVal Target As Range)
  'change these Const values as needed to match
  'your worksheet layout
  Const hLinkDataCell = "B7"
  Const firstHLinkRow = 8
  Const hLinkColumn = "B"
  Const ID_Column = "A"
  Const displayText = "Yes"
  'be sure you put the final \ as part of the
  'hard coded server path!
  Const serverPath = "\\YOUR_SERVERNAME\FOLDER_NAME\" ' <=== Change as required!
 
  If Target.Cells.Count > 1 Then
    Exit Sub ' do nothing unless just 1 cell changed
  End If
  If Target.Row < firstHLinkRow _
   Or Target.Column <> Range(ID_Column & 1).Column Then
    'ignore, it's above row 9, or not in column A
    Exit Sub
  End If
  If IsEmpty(Target) Then
    Range(hLinkColumn & Target.Row).ClearContents
    Exit Sub
  End If
  'do we have something to make a hyperlink with?
  If IsEmpty(hLinkDataCell) Then
    MsgBox "No document to set link to in " & hLinkDataCell, _
     vbOKOnly + vbExclamation, "Quitting"
    Exit Sub
  End If
  Range(hLinkColumn & Target.Row).Formula = _
   "=HYPERLINK(" & Chr$(34) & serverPath & Trim(Range(hLinkDataCell)) _
   & Chr$(34) & "," & Chr$(34) & displayText & Chr$(34) & ")"
End Sub
