jay freedman say:

For the "embedding tags" part, you can put any text into an image's Alt text: right-click the image, click "Edit Alt Text", and type in whatever you need. The text is saved in the document file.

Searching the Alt text is a lot harder. The built-in Find and Advanced Find functions don't see Alt text at all. You'll need a macro that reads the Alt text of each image until it finds a match for the expression you're looking for, and then selects that image.

This can get quite complicated for a few reasons:

  • Word VBA separates images into two collections, InlineShapes (those with text wrapping set to "in line with text") and Shapes (all other text wrappings). You can cycle through one type or the other but not both simultaneously; trying to work through them in order of their locations in the document is very difficult. It would be best if all the images in your document have the same wrapping type.
  • Your users may want to search for a single word or short phrase that's only part of the Alt text in many images. In other words, you want a Find Next function like that in the Advanced Find dialog. The macro needs to stop and select an image where it finds the match, then continue from there toward the end of the document. You should probably also have a Match Case option.
  • You may want to approach this differently by creating a list box containing the Alt texts of all the images, and letting the user click on one to jump to the corresponding image.

Post back with some more details if you need help creating the macro.

andreas killer say:

A hyperlink into an other sheet needs to be a real hyperlink.

Andreas.

Sub Test()
  Dim Where As Range, LinkTo As Range
  Dim MyValue
  'Refer to the cell
  Set Where = Sheets("MasterSheet").Range("A1")
  'Get the contents
  MyValue = Where.Value
  'Be sure
  If Not IsNumeric(MyValue) Then
    MsgBox "No number"
    Exit Sub
  End If
  'Do your calculation
  MyValue = MyValue / 5
  'Temporary disable errors
  On Error Resume Next
  'Refer to the destination cell
  Set LinkTo = Sheets("Table" & MyValue).Range("A1")
  On Error GoTo 0
  'Link okay?
  If LinkTo Is Nothing Then
    MsgBox "'Table" & MyValue & "' doesn't exists"
    Exit Sub
  End If
  'Create the hyperlink in the cell
  Where.Value = MyValue
  Where.Hyperlinks.Add Where, "", LinkTo.Address(External:=True), "Jumps to " & LinkTo.Parent.Name
End Sub

Filter Multiple Fields in Subreports - Microsoft Community

dutchman78 say:

Filter Multiple Fields in Subreports

I have a report with four subreports.  The subreports are linked to the master report via the Subreport Field Linker with a field called GroupNumber. I need to filter the report and subreports as follows:

1. By Group Number (common to all subreports)

2. By Date (Three of the subreports have date fields, but they are not related or linked - I just need to be able to filter all of them by the date range or the current FY).

3. By JobClosed - this is a yes/no field for the fourth subreport that does not have dates.

Ideally I would have a form or section at the top of the report with an input the groupnumber, the date range, and a checkbox for whether the job is closed or not.  What is the cleanest way to do this?  I am a novice and not great with VBA or SQL.

I appreciate your assistance.

ken sheridan say: Open the report from a button on an unbound dialogue form in which you have controls to select or enter the group number, start and end dates for the date range, and the JobClosed value.  The first can be a combo box, the second two will be text boxes formatted as short date to force the date picker to pop up when the control receives focus, and the last will be a check box.

In the relevant RecordSource queries reference the controls relevant to each subreport as parameters.  If the parent report is also to be restricted by group number then you would include a parameter along the following lines as the criterion for the group number column:

  Forms!YourDialogueFormName!cboGroupNumber

You would then not need to include this parameter in the subreports' queries as this would be catered for by the linking mechanism.  In a subreport restricted by the date range you would put the following (as a single line) as the criterion for the relevant date column:

  Between Forms!YourDialogueFormName!txtStartDate And Forms!YourDialogueFormName!txtEndDate

For the JobClosed column the parameter would be like this:

  Forms!YourDialogueFormName!chkJobClosed

Instead of opening the report directly, you would open the dialogue form, select/insert the parameter values in the controls and then click the button to open the report.

ken sheridan say: Be sure the form is open when the report opens and that the form and control names in the parameters are exactly correct.  If you want to close the form automatically do so in the report's Close event procedure.
