Ms Acess VBA Insert Query - Microsoft Community

nicksonlukalaga say:

Ms Acess VBA Insert Query

As per the attached image. I need to only choose a particular candidate from the enabled fileds and store them in a database. However i need to create a table with two columns( VoterID and Candidate). How should i script an insert into  query that will handle all the combo boxes and store their values to only two columns when either is enabled.

ken sheridan say: If you want to use a form like this then you will need to write VBA code to determine which controls in the form provide the values to be inserted into the Voting table, and then build and execute an INSERT INTO statement in code.  An interface which mirrors the data structure would be much simpler, but first we need to examine the correct model for the database.

In relational database terms what you have is a many-to-many relationship type between Elections, Students, Forms, Streams and Candidates.  So in broad outline the model would comprise the following tables:

Elections
….ElectionID  (PK)
….ElectionType
….ElectionDate

Students
….StudentID  (PK)
....FirstName
....LastName
….StreamID

Forms
….FormID  (PK)
….Form

Streams
….StreamID  (PK)
….Stream
….FormID  (FK)

Candidates
….CandidateID   (FK)
….StreamID  (FK)
….ElectionID  (FK)
 
The primary key of the last table is a composite of the three foreign keys.  CandidateID references the primary key StudentID of Students

The relationship type between the tables is modelled by a table which resolves the many-to-many relationship type into a set of one-to-many relationship types:

Voting
….ElectionID  (FK)
….StudentID  (FK)
….FormID  (FK)
….StreamID  (FK)
….CandidateID  (FK)

Again this table is 'all key' i.e. its primary key is a composite of all columns.

For data entry you would use an unbound form in which there are two combo boxes, cboStudent and cboElection say, for selecting the student and election.  The BoundColumn of each would be a hidden column StudentID and ElectionID respectively, with each showing the student's name and the election type and date respectively in their visible columns.

Within this form would be a subform, in continuous forms view, based on the Voting table.  This would be linked to the parent form by setting the LinkMasterFields property to cboStudent;cboElection and the LinkChildFields property to SudentID;ElectionID.

In the subform would be combo boxes bound to the FormID, StreamID and CandidateID columns.  Again the BoundColumn should be a hidden column, with the visible columns showing the form, stream and candidate names.

I assume the head boy, head girl etc elections are school-wide.  These would be modelled by separate tables like this:

Positions
….PositionID  (FK)
….Position

PositionVoting
….ElectionID  (FK)
….StudentID  (FK
….PositionID  (FK)
….CandidateID (FK)

Again the table is 'all key'.  Data can be entered into this table via a second subform in the same parent form.

I should point out that the model described above is simplified.  It will work, but does not cover all constraints.  In reality there are other relationship types between relationship types (an relationship type is just a special kind of entity type) which should be modelled by further tables.  I don't want to complicate the matter by going into this in more detail, however, but if you want to see an example take a look at Relationships.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

This little demo illustrates how relationships are built up to achieve the final model, using a simple medical prescriptions database as its example.

Not sure this is what you are looking for but you can identify the defined names, including the auto created ones, like the following. You should be able to adapt your delete code from it.

Sub IdentifyNames()

Dim defName As Name

With ThisWorkbook 'Entire workbook
  For Each defName In .Names
    MsgBox defName.Name
  Next
End With

With Sheets("Sheet1") 'Scoped to sheet only

    MsgBox defName.Name

End With
