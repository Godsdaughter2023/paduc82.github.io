Pam,

Your question prompted us to dig deeper into our 3 Word books - and my manager solved the mystery.  The styles for headings - headings that display in the TOC - must have Outline level assigned - to match the TOC level.  Some of our styles already had an Outline level assigned because they are based on Heading 1, Heading 2, and Heading 3.  But other styles that we defined  - that are not based a built-in heading style - did NOT have an Outline level. 

  • We modified the styles by assigning the correct Outline level.
  • Edited the TOC definition by selecting Show Levels as 9, left Styles and Outline levels checked
  • All of our custom heading styles were set with the correct TOC level - in the dialog - we did not have to assign a TOC level

The style definition is the key to generating the TOC.  We should be able to use many styles - each needs to be assigned a TOC level from 1 through 9.

The TOC generates with the correct headings - however, it appears that we have encountered a bug - entries are in the TOC that are not headings - such as body text, table descriptions, table text, etc.  This appears to be a bug.  When I reapply the style and update the TOC the entries go away.

a. user say:

Hi Michael,

 

I have finished the test on my side and got the same result as you said - it requires basic auth to be enabled in WinRM. In this situation, if there is no way to convince your IT department, you may need to manually disable the blocking in registry as a workaround:

 

<HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\WinRM\Client> change AllowBasic from 0 to 1.

 

Please note this method requires you to change it every time your machine restarts or it get refreshed, but at least you will be able to connect by doing so.
