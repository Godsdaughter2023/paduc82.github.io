Visual C++ compiler (cl.exe) performance degradation - Microsoft Community

paul eremeeff say:

Visual C++ compiler (cl.exe) performance degradation in preprocessor mode (/P) under Microsoft Security Essentials (or Windows Defender on Windows 8) antivirus

Utilizing Microsoft Security Essentials (or Windows Defender on Windows 8) as antivirus significantly affects the performance of Visual C++ compiler (cl.exe process) while using it for preprocessing C/C++ source files (a /P and /c switches) in parallel compilation mode on a system with a multi-core processor. Other antiviral software (Kaspersky for instance) does not cause such an odd behavior.

For instance, we take a 4-processor system with active MSE/Defender antivirus and initiate a build of a solution containing several dozens of projects. Visual Studio will use 4 cl.exe processes by default, and each process will utilize approximately 25% of total processor execution time, for a total of ~100%, of course. Next, let’s add the /P and /c (preprocess to i-files and no linking) switches for every project in the solution, and then start the build again. Contrary to our expectations, we see that each cl.exe process utilizes something like 4 to 5 percent of processor’s exec power, for an approximate total of 25-30%. Disabling MSE/Defender returns the numbers to 25% for a single process and 100% for the total cap, even in cl.exe preprocessor mode. Starting 4 cl.exe processes from 4 bat files manually results in the same. Even setting the affinities for each cl.exe process to different CPU cores through WinAPI does not affect anything. Adding cl.exe and all of the utilized directories to the exclusions in MSE does not help either.

We’ve been able to replicate this issue on clean Windows 7 install with MSE running as VirtualBox  guest with 4 active cores, as well as on a clean non-virtualized Windows 8 install (Defender comes here by default) on a different 8 core machine. Funny thing is that on the 8-cores machine, while 8 cl.exe processes have been launched,  guest with 4 active cores, as well as on a clean non-virtualized Windows 8 install (Defender comes here by default) on a different 8 core machine. Funny thing is that on the 8-cores machine, while 8 cl.exe processes have been launched, their total execution was still at the 25% level, each one now utilizing only 2 to 3 percent, decreasing from the results on a 4-cores system! Disabling the Defender bring this to ~100% total (12.5 for aech cl.exe) as expected.

find and replace macro - Microsoft Community

a. user say:

find and replace macro

I'd like to create a macro that does a find and replace that deletes the line wherever I have a bullet with no content afterward, and one that deletes the table row (in a word NOT excel table) in any row that has an empty cell. How can i do this?

doug robbins - mvp office apps & services (word) say:

The paragraph containing the bullet could be in any of a number of styles so you may need to change the style that is being looked for in the following

Dim i As Long, j As Long, k As Long
Selection.HomeKey wdStory
Selection.Find.ClearFormatting
Selection.Find.Style = ActiveDocument.Styles("List Paragraph")
With Selection.Find
    Do While .Execute(FindText:="", Forward:=True, _
    MatchWildcards:=False, Wrap:=wdFindStop, MatchCase:=True) = True
        With Selection.Range
            If .Text = vbCr Then
                .Delete
            End If
        End With
    Loop
End With
With ActiveDocument
    For i = .Tables.Count To 1 Step -1
        With .Tables(i)
            For j = .Rows.Count To 1 Step -1
                With .Rows(j)
                    For k = 1 To .Cells.Count
                        MsgBox Len(.Cells(k).Range.Text)
                        If Len(.Cells(k).Range.Text) < 3 Then
                            .Delete
                            Exit For
                        End If
                    Next k
                End With
            Next j
        End With
    Next i
End With

graham mayor say:
You can use a macro to force the required view e.g, in the normal template save the following code either as an AutoOpen macro or if an AutoOpen macro exists in the template add the code to that existing macro - see http://www.gmayor.com/installing_macro.htm.  Put the document name where indicated in bold.

    On Error Resume Next
    With ActiveWindow.View
        If Err.Number = 4248 Then Exit Sub
        If ActiveDocument.name = "documentname.docx" Then
            .Type = wdNormalView
        Else
            .Type = wdPrintView
        End If
    End With

Whenever you open a document Word will check whether it is your particular document and then if it is will open it in Draft view. Other documents will be opened in Print view.

els76uk say: I fixed this problem with the help of alexofindy on the Netgear forum

There is a serious bug in Word 2011 for Mac such that it will not save to a share whose name contains the string "file" (without the quotes). Thus, Word 2011 for Mac cannot save to a share named "fileserver" or "file" or "myfiles". The error message is exactly the one the OP mentions in the initial post of this thread. The bug only affects Word 2011, it does not affect Excel 2011, Powerpoint 2011, or Word for Mac 2008. It occurs with both SMB and AFP access to shares. It occurs regardless of which Mac OS is being used (to the best of my knowledge). It has nothing to do with the readynas, it will occur using any network file server, including a Windows-based server. The latest patches to Word 2011 from MS do not fix the bug. The bug has nothing to do with permissions. 

http://www.readynas.com/forum/viewtopic.php?f=28&t=55283&p=315123&e=315123

My share was simply called "files"; renaming the share to "nas" fixed the problem.
