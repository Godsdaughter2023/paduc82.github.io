jay freedman say:
It's still where it was in previous versions of Word: In the Open dialog, open the Files of type dropdown and click Recover Text from Any File (*.*). Select the file and click the Open button.

This will work only with Word 97-2003 (*.doc) formatted files, not with Word 2007-2013 (*.docx or *.docm) files. For the latter, don't use Word. Instead, in Windows Explorer:
  • Make a copy of the document file.
  • Rename the copy to change its extension from .docx or .docm to .zip. (The file really is just a zip file with a different extension.)
  • Double-click the zip file to see the folders within it. Double-click the word folder.
  • Copy the file document.xml from the zip file to a regular folder on the disk, and open it with a suitable program.

A web browser such as Internet Explorer or Firefox will probably be the best way to extract the text from the XML coding, although a dedicated XML editor will do it well if you have one. Strangely, Word 2013 will refuse to open it, complaining of an unspecified error, but Word 2003 handles it perfectly well.

dave peterson say:

If you already have the outline/subtotals/autofilter applied, you can protect
the worksheet in code (auto_open/workbook_open??).

Option Explicit
Sub auto_open()
    With Worksheets("sheet1")
        .Protect Password:="hi", userinterfaceonly:=True
        .EnableOutlining = True
        '.EnableAutoFilter = True 
        'If .FilterMode Then
        '   .ShowAllData
        'End If
    End With
End Sub

It needs to be reset each time you open the workbook.  (Earlier versions of
excel don't remember it after closing the workbook.  IIRC, xl2002+ will remember
the allow autofilter setting under tools|Protection|protect sheet, but that
won't help when you're filtering via code.)
