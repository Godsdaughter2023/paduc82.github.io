hutt0 say: Found a way to fix windows explorer crashing when looking at picture thumb nails in windows 7 64 pro. Couldn't find anything on the net but I worked out this fix.

1. open windows explorer, press ALT, TOOLS, FOLDER OPTIONS, view (tab), select the 'always show icons never thumbnails. you should be able to look at the files structure without it crashing at this point. But to fix the issue continue. You have to log out of this user to over write the .db files, which follows...

2. Open control panel.

3. Users, create a new user with admin privileges. (START, CONTROL PANEL, User Accounts and family Safety, Add or remove user accounts, create a new account, Create Username and check admin, Select Create Account.)

4. Log out of current user, log into new user.

5. open my computer, turn on show hidden system files (press ALT, TOOLS, FOLDER OPTIONS, view (tab), select the 'show hidden files folder and drives)

6. go to c:\users\New_Username\AppData\local\Microsoft\Windows\explorer
copy all the .DB files

7. go to c:\users\Old_Username\AppData\local\Microsoft\Windows\explorer
paste all the .DB files to the problem login.

8. Log out of the current user, log back into the other user.

9. open the picture folder, uncheck the always show icons, never thumbnails as per step 1. Now the folder will rewrite new thumbnails over the copied ones, and it will hopefully show you thumbs without crashing. Remember to delete the account you no longer need.

Hope this works for you, as it did for me. Cheers Brett.
