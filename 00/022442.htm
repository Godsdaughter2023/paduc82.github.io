mike h.. say:

Hi,

 

That formula won't accurately return the leap year day (29) for every year. For example if you enter 1900 in a1 it will return 29 which is incorrect because 1900 wasn't a leap year.

 

For a year in a1 you can use this to get either 28 or 29

 

=IF(OR(MOD(A1,400)=0,AND(MOD(A1,4)=0,MOD(A1,100)<>0)),29,28)

 

 

and in VB

 

Function isLeapYear(ByVal YY As Long) As Variant
isLeapYear = IIf(YY Mod 100 = 0, YY Mod 400 = 0, YY Mod 4 = 0)
If isLeapYear Then
isLeapYear = 29
Else
isLeapYear = 28
End If
End Function

 

 

ronrosenfeld say:

In VBA, you can do it with a single line of code:

 

To enter this User Defined Function (UDF), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

 

 

To use this User Defined Function (UDF), enter a formula like 

 

=IsLeapYear(A1)

 

 in some cell, where A1 contains the year in question

 

=========================================

Function IsLeapYear(yr As Long) As Boolean
    IsLeapYear = (Day(DateSerial(yr, 3, 0)) = 29)
End Function
===============================

 

That's not actually equivalent to your worksheet function, as it returns true or false depending on a leap year.  If you want to return the number of days, then:

 

==================================
Function DaysInFeb(yr As Long) As Long
    DaysInFeb = Day(DateSerial(yr, 3, 0))
End Function
====================================

 

Unlike the worksheet, VBA does not suffer from the 1900 leap year problem

harrycanyon say: I figured it out!!!

On one of the sheets within the workbook there were thousands of "conditional formats". 

Here's how it happened...  I had given conditional formatting to a 4 column by 20 row section that I had used to place an order with one of my suppliers.  Column 1 was the item name, column 2 item price, column 3 item quantity, column 4 total price.  I had made column 3 cells turn pink if quantity was zero, and gave column 4 a color scale by value format.  Since I copied and pasted this same 4 x 20 section(even though the order was not the same I wanted to keep the formatting) every time I placed an order the number of conditional formats began to increase dramatically.  In the last year my order lists had reached over 2000 rows, and each row had 2 conditional formats!

If anyone ever has a slowness problem with Excel find a blank cell on the sheet, click it, press Ctrl+a, click "Conditional Formatting", then click "Manage Rules".  This will list all the rules on the entire sheet.  When I did this on my suspect sheet the list was so long it wouldn't even load!  As soon as I cleared all the rules everything was fine.

 Click OK.

3.     To create the dword value:

·          Highlight the “general folder”.

·          Click on edit

·          Select new->dword value (32 bit) for 32 bit operating system and dword(64bit) for 64 bit operating system.

john w. vinson say:

Well, what is your business flow? Do you start with a Grievance and then assign People to it, or vice versa? Could you explain (perhaps with an example) how People, Grievances and Roles interact?

I suspect you will indeed want to have a Form based on Grievances with a subform based on Roles, with a (bound!) combo box on the Subform to select which person is involved; or the other way around, a Form based on People with a subform based on Roles, with a combo box to select which Grievance. You might even have both forms, depending on the workflow and how you want to look at it.

The original problem - with the unbound combo box repeating itself - is typical for unbound controls on a Continuous Form. Since there's nothing to tie the unbound control to any data, Access just shows it as many times as there are rows in the Subform.
