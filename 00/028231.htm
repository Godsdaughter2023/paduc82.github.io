Scheduling a scan by adding a trigger to the Windows Defender Scheduled Scan task doesn’t work:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/how-do-i-schedule-a-defender-scan/9857bdcc-ef25-40f9-a08a-2e9b3d244f4f

It is still possible to schedule a Windows Defender scan by calling the Command Line Utility (MpCmdRun.exe) or the old Windows Defender UI (MSASCui.exe), but the rules for this could change “without notice” at any moment – as they’ve done so frequently in the past:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/windows-defender-how-to-schedule-a-scan/d4ba70ce-a70c-4d82-bb26-18902b4e8f9e

Now these issues certainly aren’t creating any kind of crisis for Windows Defender users, because the priority for running scheduled scans with Windows Defender is actually pretty low. Defender’s real-time protection will generally find any active malware that Defender is capable of detecting, and will only leave inactive malware (archived malware) or inactive malware remnants for scheduled scans to pick up. So running periodic manual scans with third-party malware-removal tools is usually a more productive strategy – just because these apps work with different signature sets, and consequently have the potential for finding active malware that Defender simply can’t detect:

https://answers.microsoft.com/en-us/protect/forum/protect_other-protect_scanning-windows_other/list-of-malware-removal-tools/d824b9af-ebd8-4c47-94e2-8ee6c544c100
