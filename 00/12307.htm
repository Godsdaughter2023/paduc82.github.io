ken sheridan say: I'd add one thing to Scott's reply; you should also have a table for Aisles:

tblAisles
....Aisle (PK)

You are doubtless wondering why, as the Aisle value per category is already in the Aisle column in tblCategories?  The reason is that by having one row per Aisle in tblAisles, and creating a relationship between tblAisles and tblCategories on Aisle, enforcing referential integrity, this guarantees that only valid Aisle values can be entered into tblCategories.   Similarly you should create an enforced relationship between tblItems and tblCategories on CategoryID for the same reason.

The representation of each entity type by a table and the creation of enforced relationships between them in this way is the fundamental basis of the database relational model.  While it might at first sight seem strange to think of it in this way, the Items entity type is, in database terms, a many-to-many relationship type between aisles and categories.

A query for use as the RecordSource of a form in which you can enter new items, edit existing ones, and search for items' location, would join tblItems and tblCategories on the CategoryID columns.  In the query return all columns from tblItems and the Aisle column from tblCategories.  In SQL view it would be:

SELECT tblItems.*, Aisle
FROM tblItems INNER JOIN tblCategories
ON tblItems.CategoryID = tblCategories.CategoryID;

In your form you need a text box bound to the Item column, a combo box bound to the Category column and a text box bound to the Aisle column.  Set the Enabled property of the Aisle text box to False (No) and its Locked property to True (Yes) to make it read only.  Set up the CategoryID combo box as follows:

ControlSource:   CategoryID

RowSource:     SELECT CategoryID, Category FROM tblCategories ORDER BY Category;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column.

When you select a category in the combo box its, and hence that of the underlying column in the table, will be the value of the hidden CategoryID column.  The Aisle text box will automatically show the aisle for the selected category.  Note that it's important that the CategoryID column in Items be indexed (allowing duplicates) in the table design.

To implement Scott's suggestion of a combo box to search for an item you can add this to the form header.  Call it cboFindItem say, and set it up like so:

ContrlolSource:    leave this blank.

RowSource:     SELECT ItemID, Item FROM tblItems ORDER BY Item;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

In the combo box's AfterUpdate event procedure put:

    Dim ctrl As Control
    
    Set ctrl = Me.ActiveControl
    
    strCriteria = "ItemID = " & ctrl
    
    With Me.RecordsetClone
        .FindFirst strCriteria
        If Not .NoMatch Then
            Me.Bookmark = .Bookmark
        End If
    End With

In the form's Current event procedure put the following to keep the cboFindItem control in sync with the form if you navigate from record to record in another way, e.g. via the navigation bar:

    Me.cboFindItem  = Me.ItemID

In the form's AfterUpdate and AfterDelConfirm event procedures put the following to update the combo box's list:

    Me.cboFindItem.Requery

If you are unfamiliar with entering code in event procedures, this is how it's done in form design view:

1.  Select the form or control as appropriate and open its properties sheet if it's not already open.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new lines between these.

I'm not sure I follow the question. Each table should have a Primary key. It is generally better to use an Autonumber as the primary key because that's what it is designed for. Also, when doing joins numeric keys perform better. And there is also a storage issue. An autonumber takes up 4 bytes. A text value takes up 1 byte per character. So if you have an 8 character category name it takes up twice the space. If you repeat the name in your table, you are wasting space.

The tblCategories I described earlier is a lookup table. If you already have one, you can use it.

hansv mvp say:

Activate the Developer tab of the Ribbon (if you don't see it, you can display it via Office button > Excel Options).

Click Insert in the Controls section.

Under ActiveX controls, you'll find a ComboBox button.

Click it, then click on the worksheet.

This will insert a combo box and turn on Design Mode.

Right-click the combo box and select Properties.

In the ListFillRange property, enter the address of the list, e.g. Sheet1!A1:A1000

In the LinkedCell property, enter the address of the cell that will contain the selected item.

Set the MatchEntry property to 1 -fmMatchEntryComplete.

Now click the Design Mode button on the ribbon so that it is not "on" any more.

You can now use the combo box.

timjwilliams say:

For a form with a list box  "lstNames" and a text box "txtName":

'**********************************

Option Explicit

Dim arrNames

Private Sub txtName_Change()
    FillList Trim(Me.txtName.Text)
End Sub

Private Sub UserForm_Activate()
    With ThisWorkbook.Sheets("Names")
    arrNames = .Range(.Range("A1"), .Range("A1").End(xlDown)).Value
    End With
    FillList ""
End Sub

Sub FillList(sFilt As String)

    Dim i As Integer, v As String
    Me.lstNames.Clear
    For i = 1 To UBound(arrNames)
        v = arrNames(i, 1)
        If Len(sFilt) = 0 Or v Like sFilt & "*" Then
            Me.lstNames.AddItem arrNames(i, 1)
        End If
    Next i

End Sub

'*******************************

That's just the filtering part: you can add a double-click handler to the listbox to add the selected name to the sheet and hide the form
