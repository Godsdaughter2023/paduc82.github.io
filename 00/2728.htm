hansv mvp say:

A simple but not 100% secure way to get the login name in VBA is

Environ("UserName")

Someone who knows how to set environment variables could manipulate the value. A more complicated but also more secure way is

Private Declare Function WNetGetUserA Lib "mpr.dll" _
    (ByVal lpszLocalName As String, ByVal lpszUserName As String, lpcchBuffer As Long) As Long
 
Public Function GetUser() As String
    Dim lpUserName As String, lpnLength As Long, lResult As Long
    'Create a buffer
    lpUserName = String(256, Chr$(0))
    'Get the network user
    lResult = WNetGetUserA(vbNullString, lpUserName, 256)
    If lResult = 0 Then
        GetUser = Left$(lpUserName, InStr(1, lpUserName, Chr$(0)) - 1)
    Else
        GetUser = "-unknown-"
    End If
End Function

The GetUser function returns the login name.

How to use this in the pivot table depends on where the username is used - as a row field or column field, or as a filter field.

    Set R = R.Offset(1).Resize(R.Rows.Count - 1)
    'Get the visible cells (filtered data)

    'Show message if no data & close macro
    If R Is Nothing Then

      GoTo ExitPoint

      .UsedRange.ClearContents
      'Copy the headings
      Header.Copy .Range("A1")
      'Copy the data
      R.Copy .Range("A2")
    End With

ExitPoint:

Private Function SpecialCells(ByVal R As Range, ByVal Typ As XlCellType, _
    Optional ByVal Value As XlSpecialCellsValue = &H17) As Range
  'Avoid the SpecialCells-BUG to return all cells from the current region
  On Error Resume Next
  Select Case Typ
    Case xlCellTypeConstants, xlCellTypeFormulas
      Set SpecialCells = Intersect(R, R.SpecialCells(Typ, Value))
    Case Else
      Set SpecialCells = Intersect(R, R.SpecialCells(Typ))
  End Select
End Function

ccbbbb1 say:

I have a solution. This is an issue with the build and it is also in the latest builds unless you fix the issue yourself, then it won't reappear.

You need to manually remove the driver through the command line, then reinstall it.

1. Open Search, and search for PowerShell. Right click it and click Run as Administrator.

2. Type "pnputil /enum-drivers | Out-File oem_all.txt"

DON'T CLOSE POWERSHELL!

3. Open File Explorer and in the address bar, type "%systemroot%\system32\oem_all.txt", click enter.

4. Open it with Notepad if it doesn't automatically.

5. In Notepad, in the top menu bar, click Edit, then click Find. In the box. type: "netbridge"

6. In the block of text with "netbridge" in it, you should see something like Published Name. Next to it, there should be text looking something like "oemXX.inf". In my case it's "oem28.inf"

7. Copy the "oemXX.inf" text.

8. Open PowerShell again, and this time type "pnputil /delete-driver oemXX.inf" (replace XX with your number).

9. Open File Explorer, and in the address bar, type "%programfiles(x86)%\VMware\VMware Workstation" and scroll down and right click "netbridge.inf" and click Install.

10. Open Control Panel, and navigate to Network and Internet, then Network and Sharing Center, and on the left, edit the adapters.

11. Right click your adapter, and click Properties. Click Install, then Protocol, then click VMware Inc, then click VMware Bridge Protocol, then Install.

12. Check VMware Bridge Protocol, then Apply and close.

13. Open Search, and search Virtual Network Editor. Right click it and click Run as Administrator.

14. Click Add Network. Choose number 0. Add it.

15. Select Bridged. Set the adapter as your adapter. Then Apply and close.

16. Restart your computer.

17. Set the VM(s) as Bridged again.

18. Start the VMs!

NOTE - you may not be able to start the virtual machines in build 18290 and 18298, it may give a KERNEL SECURITY CHECK FAILURE GSoD.

3. On the Show drop-down menu, select All add-ons to display all add-ons that are installed on the computer.
4. For each item in this list, select the add-on, and then click Disable in the Information window.
5. When you have disabled all the items in this list, click OK.
6. Exit and then restart Internet Explorer.
7. If issues do not reoccur, repeat steps 1 through 3.
8. Click Enable for a single add-on.
9. Repeat steps 6 through 8 until you determine which add-on causes errors to occur.

After you have used this process to determine which add-on is causing errors, you can disable that add-on. Or, you can uninstall the software that installs the add-on. We also recommend that you contact the software vendor that provided the add-on for additional troubleshooting and support.

Method 2:

If you have Internet Explorer 7 or Internet Explorer 8 installed, you can repair damaged files or missing registration information in Internet Explorer 7 or in Internet Explorer 8. To do this, follow these steps:

1. Start Internet Explorer.
2. On the Tools menu, click Internet Options.
3. On the Advanced tab, click Reset.
4. In the Reset Internet Explorer Settings dialog box, click Reset to confirm.

Please note that running Reset Internet Explorer Settings will reset all user-defined settings including those set by installed extensions, toolbars and other add-ons to IE Defaults. This includes all Security, Privacy and Zone settings. Also this will clear browsing history, delete all temporary Internet Files, cookies, form data and especially all stored passwords.
