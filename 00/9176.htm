I would like to confirm all files are in cmyk.  I can name the cmyk files uniquely but I  can't see the file names in the dox. 

 Is there a way to see the inserted  jpg file name in the dox?

Original Title: jpg file name.

macropod say:

Add .zip to the end of the file name.  That "converts" it into a zip format that Explorer will recognize.  Then you can double click on the zip to explore the underlying file structure.  Go into the Word folder then the Media folder to see the inserted image files.

That won't tell you the original filename - they simply get given names like image1.jpeg, image2.jpeg, etc.

Unless the images have been inserted as links to the original files, there is no direct way of retrieve the original filenames in a docx file. An indirect way is to do the zip conversion then look in the document.xml file. Original picture names will be within tags looking like:
<pic:cNvPr name="MyPic.JPG" id="0"/>

If the images have been inserted as Links, you can circumvent all that circumlocution by going to File|Info>Edit Links to view the filenames there.

formula available? - Microsoft Community

moko79 say:

formula available?

In an excel sheet, I need to calculate some products that include 2 different taxes, GST and PST and some services that only include GST. I want to add G/P letter behind the money amount to differentiate the taxes included in each amount. However, is there a formula that automatically recognizes that different tax symbols i put in associate with the money amount and calculate the percentage automatically for me?? thanks  

jeeped say:
As soon as you add a letter to a cell containing a number, the numebr becomes a textual representation of a number and is useless for mathematical calculations. While it is possible to parse the number back out of the text and convert it to a number again, it is much easier all around to simply use another column for the G, P or G/P. With the amount in A1 and either G, P or G/P in B1 then the calculation in C1 becomes,

=A1+(ISNUMBER(SEARCH("G",B1))*A1*0.07)+(ISNUMBER(SEARCH("P",B1))*A1*0.06)

... where your GST is 7% and your PST is 6%. This works better when you put 7% in another cell (e.g. Z1) and 6% in another cell (e.g. Z2),

=A1+(ISNUMBER(SEARCH("G",B1))*A1*Z1)+(ISNUMBER(SEARCH("P",B1))*A1*Z2)

rohn007 say:
I like your use of search. I used IF

I'm just wondering if there is any functional reason for using ISNUMBER() rather than IF()?

anannya podder say:

Hi,

 

"Software rendering" depends on the CPU to do generate the screen elements.


"GPU rendering" hands off much of the screen drawing to the Graphics card. In theory this can be better/faster if you have a high power graphics card for a few reasons:

The advantage of using GPU can be better performance. However, some people find that Internet Explorer can be more unstable if the GPU rendering is being used.


You can choose either and see which gives you the best results. If you decide to go for GPU rendering keeping your graphics drivers up to date is a good idea.
