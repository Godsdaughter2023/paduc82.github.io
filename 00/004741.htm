Say that you have the dealer names in column A and the dealer URLs in column B of Sheet2, then if you have some dealer names in column A of Sheet 1 and want the corresponding URL in column B of  sheet 1, the you can try this VLOOKUP formula in cell B1 of Sheet 1:

How use ANSI 1992 SQL syntax in MS-ACCESS-2010 outer joins where - Microsoft Community

paul farseth say:

How use ANSI 1992 SQL syntax in MS-ACCESS-2010 outer joins where selection criteria for subordinate table test for literal values?

In SQL programming, selection criteria for the master table in an outer join go in the WHERE clause.  BUT the selection criteria for the subordinate table must be put in the the OUTER JOIN conditionals.  This is standard in ANSI SQL 1992 but seems mostly not to work in ACCESS when the selection criteria involve testing an attribute in the subordinate table for a literal value.  Is there a way to fix this?  (The lack of conformity to the ANSI standard makes use of other non-Microsoft database products seem more attractive.)  Here's an example of the problem SQL:

SELECT [Clients].[ClientID], [ClientNames].[PostalName]

FROM Clients LEFT JOIN ClientNames ON ([Clients].[ClientID]=[ClientNames].[ClientID]) AND ([ClientNames].[LEFT(PostalName,1)] < "K")

WHERE [Clients].[ClientState] = "MN"

What ACCESS 2010 (and earlier versions) will not accept is the join conditional subpart: ([ClientNames].[LEFT(PostalName,1)] < "K") which is ANSI SQL 1992 compliant and essential to proper processing.  So how does one persuade the jet database engine to accept the ANSI 1992 SQL syntax here?

 

 

john w. vinson say:

I believe the problem is the square brackets. Access is interpreting your function call as a fieldname and not finding it. Try

SELECT [Clients].[ClientID], [ClientNames].[PostalName] FROM Clients LEFT JOIN ClientNames ON ([Clients].[ClientID]=[ClientNames].[ClientID]) AND LEFT([ClientNames].[PostalName],1)) < "K" WHERE [Clients].[ClientState] = "MN"

And WHERE clauses are perfectly legitimate on child tables:

SELECT [Clients].[ClientID], [ClientNames].[PostalName] FROM Clients LEFT JOIN ClientNames ON ([Clients].[ClientID]=[ClientNames].[ClientID])  WHERE [Clients].[ClientState] = "MN" AND LEFT([ClientNames].[PostalName],1)) < "K" should work fine.

John W. Vinson/MVP
