daniel jackson say:

Hello Tom,

Thank you for posting your query in Microsoft Office Community.

I understand your concern and will help you with this issue.

After uploading a Publisher file on OneDrive, you cannot access its Page Parts from a different device as they would be stored in the following location on the local computer on which it was originally created.

\Users\<username>\AppData\Roaming\Microsoft\Publisher Building Blocks.

Note: The AppData folder is hidden by default. Unhide the same by following steps mentioned below.
         
a) Open the User Account folder and click on the “View” tab.

b) Now, select “Options” and then “change folder and search options”.

c) Select the “view” tab and then place a dot next to “Show hidden files and folders”.

d) Click on “Apply” and then “Ok”.


Try the following steps and see if it helps.

1) After creating the Page Part, upload the same from Publisher Building Blocks folder as mentioned above.

2) Now go to the other device, download and save the Page Part file (*.pbb) from OneDrive to Publisher Building Blocks folder.

3) Now open Publisher > go to Page Part and check if you’re able to find the Page Parts which you’ve saved.

Hope the above information helps. Keep us posted on the status of the issue and we will be glad to offer our assistance.

Thank you.

jay freedman say:

It does depend -- indirectly -- on your profile, in the sense that the culprit is most likely an add-in that loads into Word in your original profile but not in others.

In your original profile, try starting Word in safe mode by holding the Ctrl key while clicking its icon or its name in the Start menu. When a message box appears

click Yes. That suppresses all add-ins and the (possibly customized) Normal template.

If documents open at 100% in safe mode, then either an add-in or -- less likely -- a macro in the Normal template is responsible. If there are no macros in the Normal template, start disabling or removing add-ins one at a time to find the cause.

If all else fails, you can download and install the SaveView2013.zip add-in from http://jay-freedman.info/. It contains macros that save the current zoom and view when a document is closed, and force the same zoom and view when it's reopened. A possible drawback is that it will change the file's Modified date, even if there are no other changes in the document.

rich porter - microsoft say:

PowerPoint accesses a registry entry to find the active Word installation for the handout feature. This error is usually an indication that there is a problem with that key. Here's how to check it. These steps only work if you also have Word 2010 installed. If you are using an earlier version of Word, then the values below will be different.

From Start go to Run and type regedit. If you don't see the Run option, then right-click the taskbar and go to Properties - Start Menu tab - Customize. You can turn on the Run command in the list there.

Browse to this key in the Registry Editor.

HKEY_CLASSES_ROOT\Word.Application\CLSID

The value for (Default) should read {000209FF-0000-0000-C000-000000000046}. If it doesn't, double-click (Default) and modify it.

The next key to check is HKEY_CLASSES_ROOT\Word.Application\CurVer. The value for (Default) should read Word.Application.14.

You will also want to check that there is a key present at HKEY_CLASSES_ROOT\Word.Application.14, and that the (Default) value under CLSID is the same as above.

Please give this a try and see if it resolves the error. Good luck!
