However, if I use the countif function, [for example =COUNTIF(B4:UM4,"true") or =COUNTIF(B4:UM4,"false")], both return 0.  Clearly, the words are there but countif doesn't count them.  I have other rows with "True/False" and "Multiple Choice" in some of the cells, and those cells are correctly found using both the find and countif functions.  I've tried removing formatting, but don't have any other ideas as to why this is happening, or how to correct it so I can use the countif function.  

Can someone please help? Thank you!

ronrosenfeld say:

The COUNTIF function will interpret the string "true" or "false" as a Boolean.  This is, although maybe not desired, consistent with the manner in which it interprets other values which are included within quotes.

One way I know of to force COUNTIF to count the string "true" is to use a wildcard:

=COUNTIF(B4:UM4,"*true")

This isn't foolproof, as it will count any word ending in true, but it may be useful.

If you require a more robust counting, you could try:

=SUMPRODUCT(--(B4:UM4="true"))

cindy meister say:

Hi WD

Could you please describe in more detail what you mean by "template help"? This isn't ringing any bells with me...

Is this some kind of custom template, that's linked to "some Word documents"? A template that contains macro tools or similar?

If that's what we're talking about, then it would be best to discuss it with the person or company who supplied the custom template...

Or, if you don't need any of the tools anymore, you could unlink the template. In order to do that, go to Tools/Template and Add-ins. In the box at the very top of the dialog, delete the template path and type in: Normal

That would link the document to the default Normal Template (Normal.dot) - but it will mean that any tools supplied by the template will no longer be available.

Or, you could hold down SHIFT when opening a document. That would keep any AutoOpen macro from running (which is what's happening, I assume). That would leave the template linked in, but (probably) stop the dialog box from showing.

How to Stop Slow Add Ins from Being Disabled in Outlook 2016

Hello,

There is a setting that states 'always allow add in' - however after a number of restarts outlook decides to disable it again.

Has anyone had any experience getting around this behavior?

barry xie msft say:

Hi Kris,

 

Many thanks for your information.

 

Outlook will protect the user experience by disabling an add-in it considers to be misbehaving. Please refer to the following steps to re-enable them:

1) Click on the Windows Start button, and run regedit.exe.

2) Open the following key in the registry:

HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Outlook\Addins (name_of_the_addin) where (name_of_the_addin) is usually something similar to: +ArchiveOutlookAddin.

3) In the right portion of the screen, you should see the values stored in this key. One of the values is "LoadBehavior". If the Data of this key is 2, then the add-in is disabled. You will need to set the value to 3 to re-enable it, and then restart Outlook.

 

Also, you can try uninstalling and re-installing the affected add-ins. If the problem still persists, please let us know the detailed version of your Outlook client.

 

Thanks,

Barry
