john_barnett say:

Hello MichelleBoyes,

You are now seeing a 'dual boot' menu. Firstly you need to delete the partition which contains the second (accidental) copy of Windows then you need to remove the second entry from the boot menu.

The first is relatively simply, just locate the partition with your accidental copy of Windows installed and either reformat the partition or delete the partition if you do not use it for anything in particular.

 

The second part is a little more difficult if you reply upon editing the BCDedit file. However, there is an easier way and that is to download the free version of EasyBCD http://www.softpedia.com/get/System/OS-Enhancements/EasyBCD.shtml

Install EasyBCD then run it. Once EasyBCD is running click on the Edit Boot Menu option. Delete the operating system menu option thta you accidentally installed (before you do this it is best to make sure you have the right one by checking how they appear on the dual boot menu) and then click the Save Changes button. Now exit EasyBCD and reboot your PC. You should now boot straight to your original operating system and the dual boot menu should be gone.

A file shortcut in Microsoft Windows is a small file containing a target URI or GUID to an object, or the name of a target program file that the shortcut represents. The shortcut might additionally specify parameters to be passed to the target program when it is run. Each shortcut can have its own icon. Shortcuts are very commonly placed on a desktop, in an application launcher panel such as the Windows Start menu, or in the main menu of a desktop environment. (.LNK) as the filename extension for shortcuts to local files, and (.URL) for shortcuts to remote files, like web pages. Commonly referred to as "shortcuts" or "link files", both are displayed with a curled arrow overlay icon by default.

 

Generally, the effect of double-clicking a shortcut is intended to be the same as double-clicking the application or document to which it refers, but Windows shortcuts contain separate properties for the target file and the "Start In" directory. If the latter parameter is not entered, attempting to use the shortcut for some programs may generate "missing DLL" errors not present when the application is accessed directly.

 

 Although shortcuts, when created, point to specific files or folders, they may break if the target is moved to another location. Microsoft Windows has standard algorithms for fixing up shortcuts when they are moved. Windows use a simple search algorithm to fix broken shortcuts. On Windows NT-based operating systems and the NTFS file system, the target object's unique identifier is stored in the shortcut file and Windows can use the Distributed Link Tracking service for tracking the targets of shortcuts, so that the shortcut may be silently updated if the target moves to another hard drive. File shortcuts in Windows can store a working directory path besides the target path. Environment variables can be used.

Well, I can help with at least this one small part of the puzzle...

The Windows "System Restore" feature is a bit hard to explain, but it goes like this:   Some activity (typically the install of a new software, driver, or Microsoft update) creates a "restore point".   This initial "snapshot" is like a miniature backup of a lot of the key files that Windows needs.   So, after the restore point has been first created, it's usually not very big.

But, here is the strange part...  All disk write activity on your PC after that restore point was created causes the original disk content to be automatically added to the restore point.   So, essentially, a restore point will continue to grow forever until you create another one.

So, in a sense, its is "slow moving backup" of what the files looked like at the time of the snapshot.  For example, if you made a System Restore snapshot immediately before do a huge restructuring of your files... moving, copying, restructuring folders, etc.,  then the restore point could grow to a huge size.   On the other hand, if you didn't do much on your PC after a restore point, it might stay very small for a long time.


The "gotcha" here is.... if the restore point "overflows" the amount of storage that is allocated to it, then the system will silently start deleting older restore points to make more room.   If after all of the old versions are deleted, and there still isn't enough room, then the current restore point is deleted also (leaving you with NO restore points!).
