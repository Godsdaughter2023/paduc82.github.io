Files that start with ~$ are the lock files that Word creates when a file is opened.  If the file is saved and closed in the normal fashion, those files should be deleted.

Hi Vegar,
 
Welcome to Microsoft Community.

I completely understand your concern with error message "irql not less or Equal (netio.sys)" BSOD on your system and we will assist you to fix the issue.

Before we proceed, I need more information to help you better.
 
1. Did you install any third party security software on your system?
2. Are you able to boot to normal mode on your system?

When your computer crashes with a Blue Screen of Death (or BSOD) the events leading up to the crash are held in memory until you restart your computer, when they are written to a file (called a DMP file).  This file is critical in our diagnosis.

This indicates that a kernel-mode driver attempted to access pageable memory at a process IRQL that was too high.
 
A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high. This bug check is usually caused by drivers that have used improper addresses.
 
By default, the fault of the crashes is NETIO.sys which is the Network I/O Subsystem. This is not the true cause of the crash, and usually when we have network related crashes like this, it's caused by one of two things:
 
1. Network drivers themselves need to be updated. 
2. 3rd party antivirus or firewall software causing NETBIOS conflicts.
 

If you are able to boot to normal mode on your system, follow these steps below:

Method 1: 

You may update the Network adaptor driver from Device Manager manually and check, if this helps.

1. Press (Windows key + X key) and click on “Device Manager”.

2. Expand “Network adaptor”.

3. Right click on the Network adaptor driver and select “Update Driver Software”. Once this is done, restart the computer and check, if it works.

If the issue still persists, follow the Method 2.    

 
Method 2:

rob koch say:

Rick,

The problem with displaying an example of the Defender notification is that it varies slightly on each version of Windows in which Defender operates, while the GUI portion of the program is basically the same.

The methods used to perform notifications are actually determined by Windows itself, since it's the "Toast" or similar notification pop-outs that are most often used, while even these have evolved slightly over time to, for example, slide out from the side panel rather than the lower-right notification area where they used to display.

The best way to be aware of the current method of display is to perform a "test" of your Defender or other Antivirus application by using the EICAR provided eicar.com test file(s) that were designed and supported by most major AV programs for this purpose.

Download ° EICAR - European Expert Group for IT-Security

http://www.eicar.org/85-0-Download.html

Note that the warning on that page is referring to how your AV program interacts with the eicar.com file and not the file itself, since that's actually nothing more than a well known Anti-Malwaare Testfile text string.  This is easy for AV developers to create a signature to detect and generate a [false] warning for a severe threat in order to perform an otherwise completely safe test.

As for the clearly false Trojan warning you received, the notable difference for all of these fakes is that they typically fill as much of the entire page as possible and often include loud obnoxious sounds in order to attempt to get you to call the fake number displayed.

The entire point of these fake notifications is to make people panic and not take a moment to think about what they're seeing, since if they do most would stop and realize that they're simply being conned.

The other recognizable item is that these fake notifications are always some form of a browser popup, since that's how they're getting into the PC in the first place and also why few are ever detected by most AV products, since they aren't actually malware and simply a web page (e.g. browser hijack).

Rob
