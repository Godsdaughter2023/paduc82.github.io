pa bear - ms mvp say:

graye say:

I'm in IE11 and  all my favorites folder's are there, but they are all empty.  Any idea on how to get them back?

Regardless of the reason why they went missing...   we can still help you get them back

If you haven't been backing your files yourself, then don't panic... there most likely is still a way to get your favorites back.  Windows has a built-in feature called System Restore that takes a "snapshot" of your system before it does an install, update, etc.

So, that means that this snapshot will most likely have the missing favorites.  And now the questions are: "Do I have a viable snapshot", and "How do I get my favorites from the snapshot"


There is supposed to be a feature in File Explorer that allows you to right click on a file (or directory) and go to properties, and select "Previous Versions".  Unfortunately, that doesn't work for most versions of Windows 10.

Step 1 – Do I have a viable snapshot?

The first step is to see if you have a recent snapshot. So, type "create a restore point" at the Cortana prompt to bring up the settings... then click on the "System Restore" button to get to the next prompt.


This will show the list of available restore points.  Hopefully, you’ll have a recent one to use. 


Step 2 – How do I get my favorites from the snapshot

So, there are two approaches... the first is a "broad approach" that just restores everything on your PC from that snapshot.  However, that might not be the best approach, since the snapshot may be several days old... and you'd lose everything that you did from that point in time.  You'd also have to reinstall whatever update that originally caused this problem.

So, look at the date of the restore point, and decide if you want to restore everything from the snapshot or not.  If so, just click on the Next button to start the restore.  Warning, this takes a lot of time to complete the process (at least 20 minutes or so).

The other approach is to extract only parts of the data from the restore point.  This would probably be the most appropriate since you won't lose anything else you've done since the last snapshot.  Plus, it will take a heck of lot less time to do.

Windows doesn't have a "selective restore" option, but there are some free 3rd-party applications that can do this.  Personally, I use one called ShadowCopyView.  It will allow you to view the snapshot as a hard drive... and then copy the specific files/folders that you want to restore.   See the website http://www.nirsoft.net/utils/shadow_copy_view.html for more information.

In this case, you'd be wanting to copy the contents of the \Users\YOUR_NAME_HERE\Favorites folder from the backup to the same location on your C:\ drive.    Also, this particular utility doesn’t overwrite existing files, so you should delete (or rename) that folder first.

alexsun_2012 say:

Hi Chad,

I’d explain that it is expected that the sent message isn't saved to the Sent Items folder of the shared mailbox. This is because, in Office 365, a shared mailbox is designed to receive emails, not to send emails. It doesn't require a license and can't be added to Outlook as an independent mailbox. You need to sign in to your own mailbox and then open the shared mailbox. When you send or reply to a new message from the shared mailbox, Outlook automatically sends or replies from the sender's account. Therefore, messages are stored in the Sent Items folder of the sender's mailbox.

From Outlook client, you can modify the local registry to display the emails in the shared mailbox sent items folder. However, it is only a local behavior. Other users cannot sync the changes on their clients. See: support.microsoft.com/.../2843677.

ken sheridan say:

You seem to have ignored the But….. in my last reply.  You do not close the form in the calling procedure at all.

 

Let's start from square one and rearrange your code a little to make it more readable:

 

Private Sub SSNLookUp_AfterUpdate()

 

    Const MESSAGE_TEXT = "The SSN already exists do you want to open that record?"

    Dim strCriteria As String

 

    strCriteria = "[Social Security Number] = """ & Me.SSNLOOKUP & """"

   

    ' ensure SSN has been entered in SSNLookUp control

    If Not IsNull(Me.SSNLookUp) Then

        ' if SSN does not currently exist in table, open bound form at an empty new record

        If IsNull(DLookup("[Social Security Number]", "Members", strCriteria)) Then

            DoCmd.OpenForm "frmAddNewMember", DataMode:=acFormAdd

        Else

            ' if SSN does currently exist in table, get user confirmation to open form

            If MsgBox(MESSAGE_TEXT, vbYesNo,"Confirm") = vbYes Then

                ' if user answers yes, open form filtered to SSN

                DoCmd.OpenForm "frmAddNewMember", WhereCondition:=strCriteria

            End If

        End If

    End If

 

End Sub

 

That should do as described in the comment lines.  We are assuming that the Social Security Number column in Members is a text data type.  This has to be the case with UK SSNs, which are alphanumeric, but I don't know about US SSN's.  Even if it is totally numeric it should be a text column as it has no cardinal or ordinal significance as a number.

 

BTW I notice from your screen shot that you appear not to be requiring the declaration of variables.  That is a bad idea as it would easily allow a variable name to be incorrectly entered as a result of a simple typo.  Go to Options on the VBA tool bar and check the relevant option.  You should then review all other existing modules and ensure that their declarations area includes the line:

 

Option Explicit

 

immediately below the line:

 

Option Compare Database

 

If you then try and compile the project you'll get an error for any undeclared variable, which you should declare with a Dim statement at the start of the procedure or function, or in the declarations area of the module if it's a module level variable or, in the case of a standard nodule, one exposed throughout the database.
