st3velawr3nce say:

I've also had the same issue, running the exact same version of Office 365. My issue started this morning after being away for the weekend. The software updated on its own, and then stopped working.

The problem is that after the update, you will find that Outlook will crash within a second of displaying the Outlook splash screen (you may not even get to the splash screen).

It then sends a report to Microsoft. You try to re-open Outlook a second time, and are greeted with a "Do you want to start Outlook in Safe Mode?" prompt. Click yes, and you see that Outlook has crashed again. A quick repair / online repair for this issue does not fix the problem.

The problem herein lies within the Profile settings. For some reason, the Outlook program sees the profile as corrupt, and cannot open. If you are running an exchange address, you have no problem creating a new profile and loading Outlook. If you aren't on Exchange however, you will need to figure out how to export the .pst file and re-import. The only other option here is to revert back to a non-Insider preview of Office.

THE SOLUTION FOR THIS ISSUE IS:

Close all Outlook applications.
Navigate to: Control Panel - Mail for Microsoft Outlook (32-bit)
Under profiles, click "Show Profiles"
Click "Add", and create a new profile with your information. Make sure the name of this profile is completely different than the one that you already have.
Once the new profile is created, click "Prompt for a profile to be used".
Click OK.

Close the windows, and restart your computer.

Once Windows has restarted, start Outlook. You may get the prompt to start in Safe Mode, if the program crashed the last time before you created a new profile. Be patient, however, as eventually, the profile selection window will appear. Select the new profile you have created, and voila! Outlook will communicate to the exchange server and start importing all your information.
