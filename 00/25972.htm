marshall barton say:

You do not want to combine two things in one field.  It would be a violation or the rules of normalization for no significant benefit.  It's easy enough to display the combined values in a form or report text box whenever you need to display them to users so they will neither know or care how it is stored.

An alternative to using an AutoNumber field (which should not be displayed to users), I would use some code in the Client form's BeforeUpdate event.

   If Me.NewRecord Then
      Me.ID = Nz(DMax("ID", "tbleClients"), 0) + 1
   End If

This way of creating rhe ID number will not leave gaps in the sequence when users start to create a new client record and the undo it.

This way, you won't get a duplicate ID whether it is or is not the primary key so maybe you want to make the ClientID (or client name?) the primary key??

john mcghie say:

Harry: We need to do a little more digging here.

In a Word document, the styles are stored internally.  The document makes no further reference to the template after it is created. So if the style ever was in the document, it still is -- unless it has been specifically deleted.

What is likely to have happened is that the style is set to Invisible so you cannot see it, even though it remains in the document.  Additionally, it is likely that the font the style specifies is also missing, so the text does not look correct.

First we should check to see if the style is really gone. Word makes it really difficult to find styles sometimes: the interface is incomplete in Mac Word, so it works really badly.

  1. Open one of the documents
  2. Go to the Format menu (you can't do this from the Ribbon)
  3. Open the Style item
  4. On that dialog click the Organizer button.

This brings up a two-column dialog, with the styles in the document listed in the left column and the styles in the template listed in the right column.

The list is alphabetical by name, and shows ALL styles in the document.  Check the list to see if Kanchi appears.  If it doesn't, you need to work out who deleted it, because they would need to delete it very specifically from every document (styles don't just go "missing", once they are in the document, they are there for life).

Check the template also, in the right column.  Since you have upgraded Word, chances are it's definitely NOT in there. If you find the style in the document, use this opportunity to copy it to the other side to add it to the template, this will save you time later.

Now Close the Organizer.  

If you found the style, perform the next 4 steps.  If you didn't, skip them.

  1. we now need to make it visible.  
  2. Go to Format>Style and select the Kanchi style.  You may need to change the "List" drop-down from Styles in Use to All styles, in order to be able to see Kanchi.
  3. Select the Kanchi style, and click the Modify button.  
  4. At the bottom of the dialog that appears, check the Add to quick styles list checkbox.

That will mean you can see Kanchi style in the Styles list when you are working.  Sadly, you will need to do this in every document...


We now need to check the document.

  1. Select some text you think should be in Kanchi style.
  2. Go to Format>Font
  3. In the dialog that opens, the name of the applied style will be highlighted.

If Kanchi is still in the document, its name will appear in that dialog and be highlighted.

If the Kanchi style has been deleted from the document, Word will fall back to Normal style (text cannot have "no style", if you delete a style, Word re-assigns the style as Normal).

These two investigations will show you what you need to do: if the style is still in the document but the Font is missing, installing the font on each machine will magically make all of these documents come right.

If the style itself is missing,you need tore-create it.  But then you will have to re-apply it to all the paragraphs that need it.

If you can find a document that still has Kanchi in it, you can use the Organizer we looked at before to open that document on one side and copy the Kanchi style across.

But before you do all that work, you should carefully investigate who deleted the style and why. Deleting a style from a document is a very specific act: deleting it from all the documents would have been a lot of work.  Why did they do that?

I think it more likely that the font is all that is missing.

Hope this helps
