mainsoft say:

Windows Installer information in the registry

Hi,

In 2004, when I still was a development MVP, I posted the following message in a Microsoft newsgroup:

With time, I'm pretty sure that some of the registry entries created by MSI on my system are corrupted or useless (just because some applications didn't uninstall correctly). However, AFAIK, we don't have any tool to check the consistency of this information in the registry (something like MSI RegClean). And "Windows Install Clean Up" may not be enough.
...

Meanwhile, nothing has changed. A thorough look at the Installer key in my registry shows that a lot of irrelevant information is stored there. This has been caused by failed Windows Updates, by applications not cleanly removed or updated, by badly written installers, etc. Anyway, we still don't have any tool, whatever the Windows version, that would allow us to effectively cleanup the registry and remove all this useless information in the Installer key. Windows Install Cleanup is a rather primitive tool and it is not able to check the MSI information consistency. It's just a "kill it or keep it" tool.

Note that it is not a trivial matter. You'll be amazed by the amount of useless data cluttering the registry and therefore affecting the system's performance. Microsoft WIndows Installer is a complex technology but I'm wondering why nobody in the MSI team ever considered writing a tool that could check that the MSI information in the registry is valid and healthy and possibly do a real cleanup. Or is writing such a tool just impossible? In which case the credibility of this technology would be affected.

Any thought about this?

Thanks in advance.

I did my first system backup using the back up and restore in Windows 7.  I had it copy the file to my external hardrive.  It comes up as a .VHD file.  How do I open that?  I tried to use the restore option but it says there is no backup to restore. Is it because the backup was not completed due to some files being skipped?  I do see the large .vhd file on my external harddrive. 

 

 

mouneshwar r say:

Hi Ronda,

 

The reason why you’ve a VHD file is because you’ve created a system image backup. A system image includes Windows and your system settings, programs, and files. You can use a system image to restore the contents of your computer if your hard drive or computer ever stops working. When you restore your computer from a system image, it is a complete restoration; you can't choose individual items to restore, and all of your current programs, system settings, and files are replaced.

 

You wouldn’t be able to restore VHD files using backup and restore center. You can mount the image by following the steps below.

 

Method 1: Steps to mount VHD files in Windows 7

 

a.    Open Start Menu, right-click on Computer and click Manage.

b.    Click on Disk Management.

c.    Click Action -> Attach VHD and specify the external hard disk location on your computer.

 

Check out the section in the link below about how to extract files from a system image (advanced)

http://blogs.technet.com/b/filecab/archive/2009/11/12/recovering-your-files-in-windows-7.aspx

 

Method 2: How to restore your computer from a system image backup

 

http://windows.microsoft.com/en-US/windows7/Restore-your-computer-from-a-system-image-backup

Learn more about system image backup

Backup and restore: frequently asked questions
