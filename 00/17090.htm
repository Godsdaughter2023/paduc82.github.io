Resource allocation - Microsoft Community

prashantanand say:

Resource allocation

Hello 

I am working on a construction schedule on MSP 2010 .

I am having difficulty in figuring out how to assign multiple units of same resource on a task to crash its duration.

For example,

I have a task named "Form work installation for terrace slab" and it has a duration of 7 days and i want to assign 7 carpenters 7 helpers,3 scaffolders and 10 labor for misc job.

Can any one show me how to go with it properly in MSP 2010.

Thanks in advance .

Prashant

dale howard [mvp] say:

Prashant --

Good question.  Do the following:

  1. Before you assign resources to tasks, navigate to the Resource Sheet and set the Max. Units value as 100 multiplied by the number of each type of resource you have.  For example, if you have 9 carpenters available for the project, enter 900% in the Max. Units column for the Carpenter resource.  Do the same for every non-human resource.  You should enter 100% for each named human resource, but you can set the Max. Units value above 100% for generic resources, such as the Carpenter resource for example.
  2. Apply the Gantt Chart view.
  3. Right-click anywhere in the Gantt Chart pane and select the Show Split item on the shortcut menu.  This will display the Task Entry view, with the Gantt Chart view in the top pane and the Task Form view in the bottom pane.
  4. Select a task in the Gantt Chart pane.  In the Task Form pane, delete any resources assigned to the task and then click the OK button.
  5. In the Task Form pane, enter the correct Duration value for the selected task and then click the OK button.
  6. In the Task Form pane, make sure the Task Type value is set to either Fixed Units or to Fixed Duration.  DO NOT select the Fixed Work task type, and then click the OK button.
  7. In the Task Form pane, selec Units value to 1000%.  Click the OK button to assign all of these resources.  DO NOT click the OK button until you have selected all of the resources.

Hope this helps.

.9881 - .0007 = .9874

ronrosenfeld say: Well, you just move the decimal over two places in all of the functions:

=--(TEXT(TRUNC($A2+B$1,1),"0.0") &3+4*(RIGHT(INT(($A2+B$1)*100),1)>="5"))

A2:  current value  (e.g. 78.52)
B1:  Amt to add  e.g:  0.07   or -0.07

neha singh msft say:

Hi Dave,

I found the way to hide the calculated column from the display form, you will have to use SharePoint Designer 2013 for this task. For this, please follow the steps below:

  1. Open the site in SharePoint Designer, ensure you have the appropriate level of permissions.
  2. Click on Lists and Libraries
  3. Select the List you wish to modify.
  4. Under Content Types, select the Content Type that has the calculated column.
  5. In the content type screen, click "Edit content type columns" under Customization
  6. On the columns editor, click on the calculated column you wish to edit.
  7. On the Property column, click ONCE on the word ‘Optional’ and wait. It will take 2-3 seconds before you see a drop-down box.

Note: Even though it's greyed out, it should pop up a drop-down list where you can select "Hidden".

      8. Select the Hidden option, click Push Changes to Sites and Lists and Save.

The above steps will hide the column from the display form only, but you will still see the calculated column in the list view. To hide the column from the list view, follow the steps mentioned in my last reply i.e. All Items> Edit current view and uncheck the columns.

To summarize, you have to use both the SharePoint designer 2013 and the browser to completely hide the calculated column.

Note: Before you proceed with above steps, please make sure you have installed SharePoint designer 2013 from the official site.

For the other issue, where adding calculated column in the list reverts the display to the class form, we’ll suggest you submit your feedback in our SharePoint UserVoice. UserVoice is the best way to make the product better for you and others.

Thanks,

Neha
