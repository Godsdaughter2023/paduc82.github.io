Most common error made when importing from an Excel file is that there is either no <named range> or the <named range> is incorrect. In the latter case, most typical problem is that the <named range> does not encompass a <header row> - the first row in the

Re: CSV file error

Most common cause to that type of error with a CSV file is that either

jeeped say:
You've stated that the range is not always A1:Z180. What else could it be? Is A1 always the cell in the top left corner? Are there any completely blank rows or completely blank columns?

The following is one possibility. It will look at A1 and grab all rows and columns in a contiguous block around the A1 cell. This is known as the .CurrentRegion. and is the same as a single Ctlr+A keystroke. It will halt at any completely blank row or completely blank column.

Sub my_sort()
    With ActiveSheet.Range("A1").CurrentRegion
        .Cells.Sort Key1:=.Columns(13), Order1:=xlAscending, _
                Orientation:=xlTopToBottom, Header:=xlYes
        .Cells.Sort Key1:=.Columns(1), Order1:=xlAscending, _
                Key2:=.Columns(7), Order2:=xlAscending, _
                Key3:=.Columns(8), Order3:=xlAscending, _
                Orientation:=xlTopToBottom, Header:=xlYes
    End With
End Sub

This method of sorting by code can only perform 3 levels of sorting criteria so the M column is sorted first then the A, G and H are done in a second sweep.

Excellent question, my friend.  Your Standard calendar will be the working schedule for all on-shore resources.  You will need to create a second calendar named something like Off Shore and set the holidays for your off shore people as nonworking time in that calendar.  Then in the Resource Sheet view of your project, you will need to set the Base Calendar value to Off Share for each of your off shore resources.  Regarding your second question, Microsoft Project will take care of the scheduling, based on the calendar used by each resource in your project.
