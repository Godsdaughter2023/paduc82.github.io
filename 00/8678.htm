suzanne s. barnhill say:

This will depend on how you select the text in each cell. When you click to select a table cell, the entire cell contents are selected. If you tab into the cell, the contents are selected, but the end-of-cell marker is not included. You can accomplish the same thing by displaying nonprinting characters so that you can see the end-of-cell marker and carefully selecting text without it.

If you copy this selection, then you will be able to paste into another cell provided that you haven't inadvertently selected the entire contents of that cell: again, if you tab into the cell, or just click in it, instead of clicking to select it, the copied content can be pasted without overwriting the current cell contents.

You can disable the Up-/Down-Arrows in the KeyDown event.

Andreas.

Private Sub DisableArrows(ByVal KeyCode As MSForms.ReturnInteger)
  Select Case KeyCode
    Case 38 'Arrow-Up
      KeyCode = 0
    Case 40 'Arrow-Down
      KeyCode = 0
  End Select
End Sub

Private Sub TextBox1_KeyDown(ByVal KeyCode As MSForms.ReturnInteger, ByVal Shift As Integer)
  DisableArrows KeyCode
End Sub

Private Sub TextBox2_KeyDown(ByVal KeyCode As MSForms.ReturnInteger, ByVal Shift As Integer)
  DisableArrows KeyCode
End Sub

    While id(x) <> 0
        On Error GoTo 0

        Set Conn = ActiveSheet.Shapes.AddConnector(msoConnectorStraight, 100, 100, 100, 100)

noMore:
    Set Conn = Nothing

The "On error goto noMore" will NOT catch the "Subscript out of range" - why not ???

ronaldoonenil say:

Yes, but the error trap will only be on when x is 0

Your On Error Goto 0 is inside your While Loop so when x becomes 1 and goes back up to the While, the error checking is off.

Edit: The On Error is not for a particular line of code but is just for the position or sequence that the code executes, that is why it is Off when you go back up to the While statement
