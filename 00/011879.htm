BUG in Access 2007 Switchboards - Microsoft Community

rmittelman say:

BUG in Access 2007 Switchboards

Switchboard pages on multiple-page switchboard are arbitrarily showing the wrong description on some, but not all, pages.  They are showing the description of an item on the page, rather than the description of the page itself. Changing the order of the items on a page in switchboard manager changes which "wrong" switchboard page title appears.

I think the DLookup function in the embedded macros is not written properly, and is finding an arbitrary record for that switchboard page to display at the top.

In the embedded macro, on the lines that fill in the switchboard page's label captions, I added: " And [ItemNumber] = 0" (without the quotes) between the 2 closing parentheses of the DLookup functions, and the problem disappeared.

This must be done twice, once for the button click event, once for the label click event.
