ken sheridan say: The problem is not with the relationship type.  A one-to-many relationship type is really a one-to-many-or-to-one-or-to-zero relationship type.  More technically it is a unary relationship type.

What the error means is that when you apparently think you are inserting a new row in to the SYSTEM table, you are in fact trying to edit an existing row.  You have not explained the context in detail, but a basic sort of interface I might expect would be a form based on the SYSTEM table and within it a subform based on the PRI_BACTERIA table.  Given the relatively large number of columns in each table both would be in single form view, with, to maximize the use of available space, the subform on the second page of a tab control, of which the first page contains controls bound to secondary columns in SYSTEM, with controls bound to the principle identifier columns from SYSTEM above the tab control in the parent form so that they remain visible whichever page of the tab control has focus.
