rick rothstein, mvp - excel say:
Thanx again Rick! However, won't the following code be more appropriate so as to return the intersection of the last ROW as well as the last COLUMN?

Sorry, I misread your question to be asking for the last piece of data. See if this code does what you want...

Function LastCell(Optional SheetNumber As Long) As String
  Dim LastUsedRow As Long, LastUsedCol As Long, WS As Worksheet
  Application.Volatile
  If SheetNumber = 0 Then SheetNumber = ActiveSheet.Index
  Set WS = Worksheets(SheetNumber)
  LastUsedRow = WS.Cells.Find(What:="*", SearchOrder:=xlRows, _
                SearchDirection:=xlPrevious, LookIn:=xlFormulas).Row
  LastUsedCol = WS.Cells.Find(What:="*", SearchOrder:=xlByColumns, _
                SearchDirection:=xlPrevious, LookIn:=xlFormulas).Column
  LastCell = Cells(LastUsedRow, LastUsedCol).Address(0, 0)
End Function

barkoosh say: It's the same old UNICODE/ANSI problem. In order to fix it,
- Right click on the Start button and choose Control Panel.
- If the view is by Category, click Clock, Language, and Region. Under Region, click Change location. If the view is by Small icons/Large icons, click Region.
- Click the third tab Administrative.
- Under Language for non-Unicode programs, click Change system locale...
- From the list under Current system locale, choose Hebrew. Click OK. Restart.

This way the text in the resulting comma/tab delimited files will appear correctly, although in ANSI encoding. If needed, you can convert them back to UNICODE (UTF-8) with Notepad.
