Our enterprise business installs o365, by default, without Microsoft Access included.  We use the following XML format:

<Configuration>

    <Add OfficeClientEdition="32" Channel="Targeted">

        <Product ID="O365ProPlusRetail">

            <Language ID="en-us"/>

            <ExcludeApp ID="Access"/>

        </Product>

    </Add>

    <Updates Enabled="TRUE" Channel="Targeted"/>

    <Logging Level="Standard" Path="c:\Temp\"/>

    <Property Name="FORCEAPPSHUTDOWN" Value="TRUE"/>

    <Property Name="SharedComputerLicensing" Value="0"/>

    <Property Name="PinIconsToTaskbar" Value="FALSE"/>

    <Display Level="None" AcceptEULA="TRUE"/>

</Configuration>

 

For the initial install (no access) we pre-download the office installation files using the /download switch with the Office Deployment Tool and then install with the above xml file using this command line:

setup.exe /configure install-no-access.xml

 

 

When a user requests Microsoft Access, we simply use the Office Deployment Tool, without pre-downloading the files (basically, an internet based installation) and give it the following xml file:

<Configuration>

    <Add OfficeClientEdition="32" Channel="Targeted">

        <Product ID="O365ProPlusRetail">

            <Language ID="en-us" />

        </Product>

    </Add>

    <Logging Level="Standard" Path="c:\Temp\" />

    <Display Level="Full" AcceptEULA="TRUE" />

    <Property Name="FORCEAPPSHUTDOWN" Value="TRUE" />

    <Property Name="SharedComputerLicensing" Value="0" />

    <Property Name="PinIconsToTaskbar" Value="FALSE" />

    <Updates Channel="Targeted" Enabled="TRUE" />

</Configuration>

The only difference between the two XML files is that our initial installation excludes Access and the installation we use to add it, removes that specific XML tag.  

 

This USED to add the Access software to our existing installation, and even update the installation if there was a newer version available.  However, once we updated from 16.0.9126.2275 to 16.0.10730.20102, this method stopped working.  We now receive the error: "The following product(s) can’t be installed at the same time:   O365ProPlusRetail"

Currently, our techs have to uninstall the Office Suite fully, and then re-install it using the "with Access" .xml file.

 

Why did this change in the latest versions?  This was so convenient for our techs to add Access to people who needed/requested it.
