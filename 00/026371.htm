I had the exact same issue after a windows 10 upgrade: every time i launched excel or an excel file, the windows installer would launch, and then request office 2010 installation media.

Detection of product '{90140000-003D-0000-0000-0000000FF1CE}', feature 'ProductNonBootFiles' failed during request for component '{BBE9774C-E551-439A-93D7-756DF1FFD033}'

Detection of product '{90140000-003D-0000-0000-0000000FF1CE}', feature 'ProductNonBootFiles', component '{B8F92320-3FBC-4C8B-88ED-A9A9E590D05A}' failed.  The resource 'C:\windows\Fonts\MEIRYO.TTC' does not exist.

The problem is the missing font. For whatever reason I don't have meiryo.ttc. Not feeling like downloading it, I simply created a dummy file. Windows disables some functions in the fonts folder, so I had to do this via the command prompt:

1: Open a command prompt with admin priviledges

2: Navigate to c:/windows/fonts/

3: Create a new .txt file by typing:   start notepad meiryo.txt

4: Click yes to create the file and then close it to return to the command prompt.

5: Change the file extension by typing: rename "meiryo.txt" "meiryo.ttc"

Done! No reboot necessary, problem solved. With an empty dummy file in the folder, the font isn't identified as missing, and the installer doesn't launch. Presumably the installer was launching in order to repair an installation that was missing a needed font file.

Merging time-series data with different time intervals - Microsoft Community

jbk545 say:

Merging time-series data with different time intervals

I'm trying to merge time-series data from multiple sources. The main difficulty is that the different data sources, which I'm combining, record time at different intervals. So for example, values for Variable A are recorded every 15 seconds; values for Variable B are recorded every 10 seconds; values for Variable C every 1 minute; and values for one variable are even recorded at sporadic intervals of, say, 5-20 seconds.

Ideally, I'd like to merge all the data so that the main time interval is 1 minute. Variables that have multiple values per minute, then, can be averaged over the entire minute. So for minute 1, under Variable A, you'd see the average of 4 values taken at 0:15, 0:30, 0:45, and 1:00 times. For Variable B, you'd see the average of 6 values taken at 0:10, 0:20, 0:30, 0:40, 0:50, and 1:00. That way you can see all the variables along the same timeline.

Anyway, it seems like there must be a decent solution using Excel, but I'm stumped and not sure how to go about it. Any ideas?

Thanks
joeu2004 say: jbk545 wrote:
the measurements do vary within a minute, and we have more fine-grained data. So that's why I'd like to average the values over a minute.

I understand.  And if you label the 1-minute data as "average 1-minute data", I would have less of a problem with your methodology.  Be that as it may, you are the expert in how you want to represent your data.

jbk545 wrote:
it would be nice to have one timeline at 1 minute intervals, and then to place all the other variables along that one timeline. So at time 1, you could see what Variables A, B, and C are

Following the paradigm that I presented before, I would do the following.

For variable A, quarter-minute data starting in Sheet2!B2, enter the following simple non-array formula into B2 and copy down [1]:

=AVERAGE(OFFSET(Sheet2!$B$2,4*(ROW()-ROW($B$2)),0,4,1))

For variable B, 10-minute data starting in Sheet3!B2, enter the following simple non-array formula into C2 and copy down [1]:

=AVERAGE(OFFSET(Sheet3!$B$2,6*(ROW()-ROW($C$2)),0,6,1))

For variable C, 1-minute data starting in Sheet4!B2, enter the following simple non-array formula into D2 and copy down:

=Sheet4!B2

For variable D, data at irregular ("sporatic") intervals starting in Sheet5!B2, I assume that Sheet5!A:A contains the sample time, as does A:A in the sheet containing these formulas.

Both times must have the same semantics, namely:  either (1) they are [m]:ss, where [m] is minutes since the start of the data capture, not time of day; or (2) they are the date/time of the data capture.

Then one way to compute the average of samples at irregular intervals within the same minute is:  enter the following simple non-array formula into E2 and copy down(assuming Sheet5!B2:B1000 contains only numbers, no empty cells or cells with text) [2]:

=SUMPRODUCT((INT(Sheet5!$A$2:$A$1000*1440)=INT(A2*1440))*Sheet5!$B$2:$B$1000)
/SUMPRODUCT(--(INT(Sheet5!$A$2:$A$1000*1440)=INT(A2*1440)))

Note:  That also assumes that there is at least one sample for every 1-minute interval.  If that is not the case, the following might be sufficient for the Excel 2007 user:

=IFERROR(SUMPRODUCT((INT(Sheet5!$A$2:$A$1000*1440)=INT(A2*1440))*Sheet5!$B$2:$B$1000)
/SUMPRODUCT(--(INT(Sheet5!$A$2:$A$1000*1440)=INT(A2*1440))),"")

Do these formulas satisfy your needs?

 

-----
Endnotes

[1] For Excel 2003, SUM does not seem to be needed in the AVERAGE(OFFSET) context.  As I noted previously, I cannot explain why ROW()-ROW($B$2) sometimes does and sometimes does not result in an array.  If you do not want to determine this empirically, there is no harm in always using 4*SUM(ROW()-ROW($B$2)).

[2] INT(x*1440) should work for Excel 2007.  However, in Excel 2003, I have encountered some time values for which INT(x*1440) does not have the expected result due to an inconsistency in the way that Excel 2003 converts time constants and performs arithmetic (binary floating-point).  I am told the inconsistency does not exist in Excel 2007.  Nonetheless, if you encounter problems with INT(x*1440), you might consider INT(x)+60*HOUR(x)+MINUTE(x).
