don cuthbert say:

UEFI is more modern and has various advantages, such as Secure Boot. Plus it requires a GPT-initialized hard disk. But if you enable UEFI, it requires a FAT32 file system to boot from, so the USB stick needs to be formatted as FAT32. Also a GPT hard disk has a FAT32 System partition, but a Legacy MBR-initialized disk does not. GPT is considered a more robust partitioning scheme than MBR, so it is recommended to enable UEFI and use a GPT disk.

There is a utility (MBR2GPT.EXE) to convert your hard disk from MBR to GPT. The way it works is to convert the disk, and then you must switch your BIOS to full UEFI mode, which enables it to boot from the newly created FAT32 System partition.

https://docs.microsoft.com/en-us/windows/deployment/mbr-to-gpt

So unless your USB stick is formatted FAT32, you cannot boot from it in UEFI mode. Rufus can create a bootable USB for UEFI from an ISO file. https://rufus.akeo.ie/

don cuthbert say:

If your first goal is to recover data, you can leave it in Legacy BIOS mode and boot from an NTFS-formatted USB stick with Windows install on it. If you can get the Windows install to load, instead of going through it, use SHIFT-F10 to get a Command Prompt after the first screen of Windows setup loads. Then you can use diskpart to shrink C: to make enough room to install another copy of Windows to a small partition. However, an MBR disk has a limit of 4 primary partitions.

You can also use the bootsect and bcdboot commands to fix your System files. By default, they will be on the first partition, which is usually a 500MB NTFS partition containing System files and Windows Recovery Environment.

If you can't get the USB stick to boot, you might be able to boot to your Windows RE partition. When you first power on and see the Windows logo, hold the power button in for 5 or more seconds to power off. Do this twice. On the next boot it should load into Windows Repair. From there you can get a Command Prompt and could potentially install Windows to a small partition or use Commands to fix your System files.

The drive is locked error can be tricky to fix, but it can be done. Below is a video about it, but there may also be simpler fixes to try.

karl timmermans [outlook mvp 2012-2018] say:

I have read that Outlook 2010 cannot co-exist with Office 365.  Is that true? Is there any way that I can re-install Office 365 but keep Outlook 2010? 

Unless you are installing the stand-alone <.msi> version of Office 2013 which would only be the case if you had a subscription to MSDN or are otherwise provided a copy - Office 2013 (aka Office 365) will install side-by-side with any earlier version of Office including Outlook.

In short - you will be able to run both Outlook '2010 and Outlook '2013 but be aware that if you have some 3rd party programs that interact with Outlook - those may only recognize one version (usually the older version).
