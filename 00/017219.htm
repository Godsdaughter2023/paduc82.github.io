bob jones aka: cybertaz say:
Well, Phillip is correct so far as ActiveX Controls are not supported on a Mac in OS X... Technically they will work on a Mac if you're running Win Office on a Windows VM :-) ActiveX is the less likely possibility, though, because you have to go into a special dropdown [Legacy Tools] on the Developer tab in Win Word to get to them.

The more likely devices used are those created by the tools displayed openly on the Developer tab. Since they are the ones most readily selectable (as well as being the newer, flashier kids on the block) they are probably the ones that were used. They create what are called Content Controls which, also, are unsupported in Mac Office.

The devices supported on both platforms are what are created by the Legacy Forms Tools which are also accessed from the top row of the Legacy Tools dropdown mentioned above -- the ActiveX Controls tools being on the bottom 2 rows. Those are the same Form Controls tools provided on the Developer tab in Office 2011.

As for the lack of notification, most of the Content Controls & the ActiveX Controls do not require the use of Macros or VBA. They can be saved in .docx format if no VBA is used so there would be no reason for the notification. You can also save in .docm format even if the file contains no Macros/VBA, in which case there would still be no notification. Another possibility is that even if there were VBA, saving the file in .docx format effectively strips the code out -- Again, that would eliminate any notification. Of course, if the file is saved in .doc format Content Controls would die as well.

ericyang_msft6286 say:

Hi Leonard,

Thank you for the information.

SharePoint can’t recognize the pages in a OneNote notebook file, so the minimum level where we can set permission control is the file level. Therefore, if you need to control the permission for each page, we suggest you created separated OneNote files to store each page.

Regarding your specific requirements for the permissions, I suggest you create custom permission levels to control the permission in a more granular way. A permission level is built up from the basic permission such Read, View, Delete, etc. Please refer to this article for more details: Understanding permission levels in SharePoint. You can set up two custom permission level, one for the file owner and one for other users.

If you want the permissions to be set up automatically when the files are created or moved into the library, you may need to set up custom SharePoint workflow in SharePoint Designer to do the automated job. If you need help on creating custom workflow, we suggest you turn to our Developer Network SharePoint forum for the further help.
