luismarsano say:

Are you logged in as a standard user or an administrator?

If you're logged in as a standard user and the file is in any directory under your user profile (including your downloads directory), then try this.

It sounds idiotic, but try it anyway.

Open File Explorer then open the root of the drive your file is on (e.g., go to C:\).

Then navigate through each subdirectory down the path to your file (i.e., Users → userID → Downloads) and mount your ISO file.

Select an administrator in the UAC.

Don't go straight from your profile (where File Explorer opens) to your file.

Do it the long way.

If this works, then the problem is the file is located under your profile, and File Explorer opens in the current context's profile at the variable path %USERPROFILE%.

However, the UAC switches this context to the selected user's, so %USERPROFILE% becomes C:\Users\selectedUser instead of C:\Users\loginUser, which mounts the wrong file path.

Navigating to C:\Users\loginUser, however, statically places File Explorer in that path, which doesn't vary with context change.

This definitely happens in my case, and I'd like to report the issue to Microsoft.

The only workarounds I know are to do the above every time, move the files to any place not under the logged in user's profile, or log in as an administrator.

frederik long say:

The command "pause" will pause the batch file until you press a key. To pause it for 30 seconds you need to use this command:

timeout  30

I might ask why you have a timed pause of 30 seconds plus a 5 second pause with shutdown.exe. Why not delete the pause, then make the shutdown delay 35 seconds?

frederik long say:

app.exe might not return control to the batch file. If so then you must launch it in a separate shell like so:

@echo off

start  /b  "App"  app.exe -exit

shutdown.exe

I note that you do not include a path for app.exe. This is mandatory for batch files that are invoked by some other application. Without a fully qualified path you never know which app.exe gets invoked.

Here are some notes that may help resolve problems encountered when installing Windows Updates.

The process of installing updates comprises downloading, installing and configuring the operating system to use the new updates. Some updates require the computer to be restarted and when this occurs configuring occurs as the system is shut down and is completed when the computer is restarted.

 

It is important not to interrupt the configuration process as configuration involves making changes to the registry. If interrupted, the result can be a computer that will not boot.

 

A lot of problems arise from setting Windows Update to automatically update. You may prefer to change this setting. Select Start, All Programs, Windows Update, Change Settings. Under Important Updates is a box showing the current setting. Click on the Arrow down to the right and change the selection to “Check for Updates but let me choose whether to download and install them” and click on OK.

 

If you encounter problems updating or you are not using the Automatic Update you should manually update by selecting Start, All Programs, Windows Update, Check for Updates. Check the boxes before the updates you want to install, click on Install and click again on Install in the next window and the process of installing starts. You should let the process proceed without interruption. Multi-tasking at the time updating is in progress is unwise. The progress window will tell you what is going on. You need to be patient.

If you have a number of updates to install break the task of updating into two or more tasks.

argel obm say:

Updating your network adapter is the same as updating your Windows. When you update your driver, it can resolve the issues you're having on your internet connection and also improve your network adapters performance. You can update your network adapter by knowing the manufacturer and model of your system. You can do this by doing these steps:

  1. Press the Windows key and the letter R on your keyboard.
  2. In the Run window, type msinfo32.
  3. In the right pane of the System Information window, you'll see System Manufacturer and System Model. Take note of those two information.

After knowing the manufacturer and model of your system, you can visit the manufacturer's website and search for your system model using the website. You'll be able to download the latest update for your network adapter from there. You can also contact your manufacturer if you want further assistance on how to update your network adapter.
