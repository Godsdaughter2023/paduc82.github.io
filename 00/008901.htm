brian tillman [outlook mvp 2007-2019] say:

Do the messages appear in your Sent Items folder?

 

Just because the messages arrive when using web mail, that's not proof the problem lies with Outlook.  It could still lie with the mail provider.  When using web mail, SMTP is not involved and so the SMTP server the mail provider uses could have a problem and web mail wouldn't be able to detect that.

 

If the messages are appearing in your Sent Items folder, then they were definitely sent, as far as Outlook is concerned.  In other words, Outlook was able to make a connection to a system that answered when Outook initiated its SMTP session with the outgoing server and that system accepted the message from Outlook and acknowledged to Outlook that it had accepted the message.  At that point, the SMTP session ended and Outlook moved the message to Sent Items.  So, if the message is in Sent Items, delivery is complete.  You must now look at the system that accepted the message from Outlook.  The fact that you say Outlook Express demonstrates the same problem is pretty good verification that the problem is at the server end.

 

Try enabling diagnostic logging and examine the trace of the SMTP session.  It may give you a clue as to where the problem lies, but more likely, it will give you proof that the problem is not within Outlook.  http://support.microsoft.com/kb/300479
