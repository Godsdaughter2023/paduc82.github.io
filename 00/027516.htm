Limitation to geomean function? - Microsoft Community

rgalyen say:

Limitation to geomean function?

I'm trying to use the geomean function in Excel on a large dataset (n>1000) and keep getting a #num error.  After some experimenting, this error is returned when n>100.  Just want to verify this is indeed true and I will look somewhere else to find the geomean.

thanks, Robert

joeu2004 say:

Robert wrote:

I'm trying to use the geomean function in Excel on a large dataset (n>1000) and keep getting a #num error.  After some experimenting, this error is returned when n>100.  Just want to verify this is indeed true and I will look somewhere else to find the geomean.

You are correct.  The problem is not so much the number of terms, but the computational limitations of Excel arithmetic.  That is, the products might become too large or too small to represent in binary floating point.

The alternative is to average the log values, then compute the antilog.  So instead of GEOMEAN(A1:A1000), use either of the following array formulas [*]:

=10^AVERAGE(LOG(A1:A1000))

=EXP(AVERAGE(LN(A1:A1000))

Note that if you had used the array formula GEOMEAN(1+A1:A1000), you would use LOG(1+A1:A1000) or LN(1+A1:A1000) in the formulas above.  This is typical when computing the geometric average of daily stock returns, for example.

[*] An array formula is entered by pressing ctrl+shift+Enter instead of Enter.  In the Formula Bar, you will see curly braces around the entire formula, viz. {=formula}.  You cannot type the curly braces yourself; this is just the way that Excel distinguishes the appearance of array formulas.  If you make a mistake, select the cell, press F2, edit as needed, then press ctrl+shift+Enter.

The fact that your messages are moving directly into the "recover deleted items" area is troubling. To get your message out of the recovered deleted items area you might want to try copying them rather than moving them to see if that works.
