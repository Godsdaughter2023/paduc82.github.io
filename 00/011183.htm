daniel g. grau say:
Don't select the image, because that causes Word to enable the Format Picture options. Instead:

  • Click on the elsewhere same paragraph, or
  • Select the picture and press the left or right arrows to place the cursor into a text-typing position (immediately to the right or left of the image), or 
  • Move your cursor to the left margin until the arrow points "northeast" (right and up), then click to select the complete paragraph with the inline picture.

Then:

  • Center the picture with the formerly grayed-out Center Text button (note the name!) in the Ribbon's Home tab (you'll be actually centering the paragraph with the inline picture)

Easier still, click on the picture or select its paragraph, or click on the paragraph that contains the inline picture, then:

  • Press ⌘E to center, ⌘L to left-align, ⌘R to right-align (and ⌘J to justify, but this will only be evident in a paragraph with text)

I suspect that the paragraph-centering button in Focus View works only because that mode does not offer options to modify pictures, so Word automatically gives you the paragraph options.

Outlook Converts Inline PNG Image to JPG

Hello,

As part my job I often have to share simple graphics (such as screenshots of schematics or simulation plots) with my coworkers.  PNG is the format of choice because it offers better image quality and compression than JPG for this type of image.  However, I have found that Outlook 2013 converts my embedded PNG images to JPG when I send an email.

I am adding the images inline in my email (instead of attaching) so that I can type captions with each image.  I do this by going to the Insert tab -> Pictures -> select .png image file -> Insert.  The image appears inline in my email still as a PNG (no visual artifacts) and I can adjust its dimensions if needed.

mattlanahan say:

I've worked around this issue on my own, but I am posting my findings in case anyone with the same issue stumbles across this thread.

After a bit of experimentation I found that if I open my PNG image in mspaint and re-save it as a PNG, then embed that new PNG into my email, then Outlook does not convert it to a JPG.  I used ImageMagick to compare metadata for the two PNG images: the original screenshot was type TrueColor (24-bit/rgb), while the one saved by mspaint was TrueColorMatte (32-bit/rgba).

I perhaps should have mentioned in the original post that although I am using Outlook 2013 to send my emails, that is running inside a Windows 7 VM which is solely used for running MSOffice.  The host machine that I work on is actually running CentOS 6.7, and I use gnome-screenshot to capture the PNG images that I'm trying to embed.  Evidently gnome-screenshot and ksnapshot only save PNG24 images (or at least the versions I have installed do).

I did a quick test with the Windows Snipping Tool and it does save PNG32, so screenshots taken with this tool work fine in Outlook.  Unfortunately I can't use it to take my screenshots because all my work is all done on the host, not inside the VM.

Fortunately there is a simple workaround: ImageMagick can convert images to PNG32.  So I now have a simple shell script for converting batches of images.

for FILE in $*

do

convert $FILE PNG32:$FILE

done
