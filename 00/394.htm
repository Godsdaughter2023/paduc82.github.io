kerwinyang_msft7476 say:

Hi Thomas,

To rename the old BPOS URL from <company>.microsoftonline.com to <company>.sharepoint.com, a rename tool has been published, you can follow the steps below:
1. Set up the SharePoint Online Management Shell Windows PowerShell environment
2. Open the SharePoint Online Management Shell and run the following commands:
    $adminURL = 'https://company-admin.sharepoint.com'
   $bposURL = 'https://company-1.sharepoint.microsoftonline.com'
   $targetURL = 'https://company-portal1.sharepoint.com'

   Connect-SPOService $adminURL # enter credentials when prompted
   $site = Get-SPOSite $bposURL
   Set-SPOSite $site -newUrl $targetURL

The first 2 URLs should be the tenant admin URL and the BPOS URL. The 3rd is the target URL, it must start with the tenant name and hyphen but the highlighted part can be portal1 and portal2 according to the BPOS URL.

The command should execute quickly but the actual migration can take a bit longer for large sites. In this case the site’s state should be ‘Renaming’ while the migration is in progress.

rob_l [msft] say:

In your original post, you say that you apply your document's "normal 9pt style".  Does this mean that your normal style includes a definition of 9pt as the font size?

If this is the case, then that normal style will always override the paragraph formatting set in your table style. (Property application in tables are "Default font properties + Table wide font properties + Table conditional font properties (like first row/etc) + Paragraph style font properties + Character style font properties + Direct formatting font properties".

Since the paragraph style font properties are after the table style font properties, they will always win.

In Word 2007 or later, you can change the Default font properties for your document which will allow you to change all the text in your document to be 9pt by default, but still allow table styles to override that formatting. 

In the format/font dialog (you can get to this by clicking the dialog launcher button on the font ribbon (red circle below:

Then change the font size to your desired font size and click "set as default".  Word will ask if you want to set the default for all documents, or just this document. Choose only this document unless you want this to be a global change to all your newly created documents.

Once you change your document defaults, these will be the default font properties for everything in your document -- every style, etc, even if those styles aren't based on the Normal style. 

NOTE that when you are working with documents that already have customized the normal paragraph style (and other styles), then things can get pretty difficult to manage if you start mixing document defaults and normal paragraph style changes. I would recommend trying this in new documents or in copies of existing documents. With documents that have existing style customizations, you will probably need to spend a little time making sure your styles look the way you want them to after you change your document defaults. 

rob_l [msft] say:

I haven't tried these next steps out, but I wanted to get the idea out there (if you try them, be sure to backup your document!)

During the steps in my answer, there's a little "magic" that happens in the latest versions of Word.  The problem you are hitting is that your normal style had "9pt" defined in it, which by rule overrides the table style definition of 8pt.  The default font of 9pt would make this work, but only if the 9pt in your normal style is gone. Until that's gone, you will still have the problem (actually, its not just your normal style -- its any style that you apply that has 9pt in its definition).

In my steps, when you "change your default to 9pt", Word also goes and redefines the normal style to remove 9pt to avoid the problem. So, in theory, you might be able to force the normal style to get updated by changing your default size to something different (like 7pt, or some other size you have never used before), OK out of all the dialogs, and then change it to 9pt and OK all the dialogs, you might get the normal (and other styles) fixed up as well.

As for table styles, I'm sort of biased -- I'm a big fan of them. Early versions were a bit rocky (especially around banding with custom band sizes, custom heading row sizes, and special corner cells), but those have been worked out.
