When you save a TOC as a Building Block, you are only saving the TOC field. Formatting of entries in the TOC is determined by the TOC styles in the target document (these styles are called TOC 1, TOC 2 etc.). It's better to distribute a template with the desired styles, as John suggested.

hal hostetler - mvp-outlook say:

FWIW, restoring (or backing up for that matter) an .OST file is a waste of time because, unlike a .PST file which is a data repository, an .OST file is only a mirror image of the folder set on the email server and is there for only one purpose, to give you your existing email, contact, calendar, etc. when you are NOT CONNECTED to the Internet.  If it becomes corrupt or damaged, you merely delete it.  The next time Outlook starts, it will completely rebuild the file using the data on the server (the server folders are the data repository in this case).

Pst-files vs. pst-archives vs. ost-files

The big risk is that it will be too old and outlook will create a new one, but with regular backups this shouldn't be a problem.
