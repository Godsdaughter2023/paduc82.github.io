Use

Private Sub enter_cmd_Click()

    Dim SPKRNAME As Range
    Set SPKRNAME = ActiveDocument.Bookmarks("SPKRNAME").Range
    With SPKRNAME

         .Text = Me.spkrname_txtbox.Value

    With SPKRTITLE

   Me.Hide

Changing the default font will require different steps in different Office applications. In Word, you can change the defaults as follows: Create a new, blank document, for example by pressing Ctrl+N. In the Styles pane (Ctrl+Alt+Shift+S), click the Manage Styles button. On the Set Defaults tab, make the desired changes. Select "New documents based on this template" before clicking OK. If the above doesn't work, something is preventing you from saving to the Normal template. Usually, that "something" is a poorly designed add-in, which must be dealt with separately.
 
In the meantime, a workaround is to open the Normal template as a document and making the changes directly in the template.

jim gordon mvp say:

You can't move an application from the Trash to the dock.

The Excel application has to be in the Microsoft Office 2011 folder inside the Applications folder. It will not work properly anywhere else.

After you move the Excel application into the proper location, you can drag its icon to the dock.

Now you should be able to open Excel. Go to the Excel menu and choose About Excel and see what version you have. You should have version 14.7.7

If you need to install updates to get to 14.7.7 here's what to do:

Make sure your Mac has a working internet connection before checking for updates. Install Mac OS X updates followed by Office updates.

Install Mac OS X updates first:

To get Mac OS updates, go to the App store on your dock, and then click the Updates button to reveal available updates.

Then Install Office Updates

To get the update:

  1. Open any Office application
  2. Go to the Help menu and choose Check for Updates
  3. Let AutoUpdate check for and install updates
  4. Restart your Mac

If you have any add-ins such as WebEx, Mendelay, EndNote, Zotero, TypeIt4me, etc it is essential that you install the latest updates for your add-ins.

jay_answers say:

I don't quite know how contacts synced with exchange would work, for this is mostly a consumer support forum and not enterprise support forum. However, there are times when the caller contact won't show up and this is what I have observed.

the windowsphone PEOPL hub - responsible for showing the caller profile - does get confused easily.

For instance, suppose a contact Jay is stored on the phone. His number is - 5552613. However, when Jay actually calls the phone owner, his number is +91-5552613. As you can see, the prefix is a area code or company code or some other code while his number is the same. However, PEOPLE hub will take into account the prefix and think of that as a different number.

The solution for this is to open Jay's profile on the phone. and then delete the number 5552613. However, this will not remove his profile from the phone. Just the number. Then, come to call records, and tap on +91-5552613, and then tap SAVE. After this, a list of contacts on the phone, include Jay will show up. Add this number - with the prefix - to Jay's contact.

Now, next time when Jay calls, the PEOPLE's hub will recognize Jay calling.  This works for regular phone consumers but I have no idea if it will work in a exchange scenario. So, do let me know if it worked.
