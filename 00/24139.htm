How do a get to a reference value in an Access expression? - Microsoft Community

leith mccombs say:

How do a get to a reference value in an Access expression?

I need to understand the expression syntax for calling a reference value in a form.

 

In my sales database, the Leads table includes a ComboBox field, "FoundBy", that takes values from the "Person" field of the "ListValues" table. In my reporting form (based on Leads), I have a field to show a count of all leads found by a given person. Unfortunately, while this syntax works fine for tallying value types (string, number, bool), it errors when I try to use it with a reference field.

 

=Count(IIf([FoundBy]="Bob", 1, null))

 

I also tried specifying the source table but got the same result, "#Error" on the form.

 

=Count(IIf([ListValues]![FoundBy]="Bob", 1, null))

 

As far as I can tell, this scenario isn't documented either. If I could find a way to read the specific error messages I'm getting I might be able to piece it together, but I can't find that either.

 

Any ideas?

ken sheridan say: My guess would be that the FoundBy column is in fact of long integer data type, as a foreign key referencing the primary key, most likely an autonumber, of the ListValues table.  If so, while you see the text value from the referenced table's non-key column in the combo box control, its value will be a hidden number.  This will be the case if the dreaded Lookup Field Wizard was used when designing the referencing table.  This feature is generally deprecated by experienced developers.  For reasons why see:

http://www.mvps.org/access/lookupfields.htm

In this scenario to count the number of rows in the form's recordset which reference the 'Bob' row in ListValues table you'd need to extend the expression:

=Count(IIf([FoundBy]=DLookup("ID", "ListValues", "FoundBy = ""Bob"""), 1, Null))

Where ID is the name of the primary key of ListValues.  Note the use of pairs of contiguous quotes characters to represent literal quotes characters in the string expression.

NB:  this will count the matching rows in the form's recordset, so will take account of any filter on the form, or restriction on the form's underlying query.  If you want to count the matching rows in the table regardless of any filters or restrictions, call the DCount function rather than using the Count operator:

=DCount("*","ListValues", "FoundBy=" & DLookup("ID", "ListValues", "FoundBy = ""Bob"""))

zorvek say:

A different VBA solution. This one differs from Vijay's in that it validates any cell in a named range, in this case the name of the range is "EntryValues". It also includes a macro to reset (clear) the cells in that range. To name the range, select the cells that you want to control and enter the value "EntryValues" in the Name Box to the left of the Formula Bar and press ENTER. The code below must be placed in the worksheet's code module. To run the reset macro, you can add a command button to the worksheet and tell it to run the macro when clicked.

To add a command button to a worksheet that runs a macro, navigate to the Developer tab and select Insert in the Controls group. From the menu select the Button control from the Forms controls (top set). With the mouse, click where the upper left corner of the command button is to be placed and drag the mouse down and to the right to create the shape of the command button. A dialog is displayed asking for the macro to run when the command button is clicked. Select the desired macro and click the OK command button. If the macro is Private then it won't be listed but it can still be attached to the command button by manually entering its name.

Private Sub Worksheet_Change(ByVal Target As Range)

    Dim NewValue As Variant
    Dim NewSelection As Range
    
    If Not Intersect(Target, Range("EntryValues")) Is Nothing Then
        If Target.Cells.Count = 1 Then
            Application.EnableEvents = False
            Set NewSelection = Selection
            NewValue = Target.Value
            Application.Undo
            If NewValue > Target.Value Then
                Target.Value = NewValue
            Else
                MsgBox "Enter a value higher than " & Target.Text, vbExclamation + vbOKOnly
            End If
            NewSelection.Select
            Application.EnableEvents = True
        End If
    End If

End Sub

Public Sub Reset()

    Range("EntryValues").ClearContents

End Sub

The usedrange doesn't have to start in Row 1 (or column A).

If you want to select the rows under the usedrange, you could use:

Option Explicit
Sub testme()
    Dim NextRow As Long
    Dim wks As Worksheet

    Set wks = ActiveSheet

    With wks
        With .UsedRange
            NextRow = .Rows(.Rows.Count).Row + 1
        End With
        If NextRow > .Rows.Count Then
            MsgBox "No rows under that used range!"
        Else
            .Rows(NextRow & ":" & .Rows.Count).Select
        End If
    End With

      Rows(Rows.Count)).EntireRow.Select

Sub SelectColumnsOnly()
   Range(ActiveSheet.UsedRange.Offset(, ActiveSheet.UsedRange.Columns.Count), _
      Columns(Columns.Count)).EntireColumn.Select

Sub SelectRowsAndColumns()
   Union( _
      Range(ActiveSheet.UsedRange.Offset(ActiveSheet.UsedRange.Rows.Count), _
         Rows(Rows.Count)).EntireRow, _
      Range(ActiveSheet.UsedRange.Offset(, ActiveSheet.UsedRange.Columns.Count), _
         Columns(Columns.Count)).EntireColumn).Select
