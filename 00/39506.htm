It may just be with the way Apps appear now on Windows 8.1

 

Unlike in Windows 8, where every installed app lands up on the Start screen, Windows 8.1 does not automatically pin all installed apps and programs to the Start screen. Instead, all apps and programs appear in Apps view.

 

For Windows 8.1, you need to manually pin your favourite apps and desktop programs to the Start screen. As of now, there is no option or workaround to automatically pin all installed apps and desktop programs to Windows 8.1 Start screen. However, you can manually pin your favourite apps, apps that support live tiles, and desktop programs to the Start screen.

 

1.  Switch to the Start screen. Move the mouse pointer to the lower-left of the screen to see a small Down arrow icon and click on the same to see Apps view.

 

2.  Right-click on the app name that you would like to pin to the Start and then click Pin to Start option to pin the selected app to the Start screen.

 

3.  And if you wish to pin multiple apps to the Start at once, you can select multiple apps by holding down Control key and then clicking on app names one-by-one (touch screen users can select multiple apps by touching an app and then moving it up).

A fairly easy regedit should remove an application from the 'recommende programs' section. Here is a walkthrough on how to do that. Unfortunately, I don't think that will solve your problem. What actually appears to have happened is either your .exe or .lnk (or perhaps both) have inadvertantly been associated with WMC. Taking it out of the recommended programs list for the Open With dialog won't change that. In fact, the next time you open the program it will open in WMC and automatically add it back as a recommended program in the open with dialog.

There really shouldn't be anything that you open a .lnk or .exe with because you don't really want to open them...you want to run them. To determine whether the .lnk or .exe is associated incorrectly you can either look at the file extention associations in the Control Panel > Programs, or you can see if it's a shortcut or the actual executable that is doing it. reece coombes say:

Hi,

In most cases, the D: drive is the disk drive or USB drive. If it is a USB drive not disk drive, go straigh to method 1 and ignore this paragraph. So the problem that you are having is that your disk is full. Disks typically have a storage capacity of 4.7GB but can go up to 8.5GB. The solution depends on how your disk is set up. If it is set up like a USB drive, in that you can add and remove any files as you wish, the fix is easy. However, if you set it up for media like a DVD, there is no way other than to buy a new disk. If you set it up like a USB, this is how to fix it:

Method 1: Format the Disk

  1. Take note: This will wipe the disk. If you have any important data, back it up first or if you do not with to erase all files, go to method 2
  2. Find the device in File explorer
  3. Right click it and click 'Format...'
  4. Leave the settings as they are and click 'Start'
  5. Once it is finished, you will have an empty D: drive

Method 2: Manually get rid of files you don't need

  1. Navigate to the D: drive
  2. Go through it and delete any files you don't need

  • Disable (preferrably uninstall) your Antivirus utility before attempting the upgrade.
  • Restart a few times and try again.
  • Disable General USB Devices (example - Smart Card Reader). 
  • If you are using a SCSI hard disk, make sure you have drivers available for your storage device on a thumdrive and it is connected. During Windows 10 setup, click the Custom Advanced Option and use the Load Driver command to load the appropriate driver for the SCSI drive. If this does not work and setup still fails, consider switching to a IDE based hard disk.
  • Do a clean boot and try again.
  • If you are upgrading using the .ISO file, disconnect from the Internet during setup, if you are connected by LAN (Ethernet) or Wi-Fi, disable both and attempt setup again. 
  • If you are updating through Windows Update, when the download reaches 100% disconnect from the Internet LAN (Ethernet) or Wi-Fi and proceed with the installation.
  • If that does not work, try using the .ISO file to upgrade if possible.
  • If you are connected to a domain, switch to a local account
  • If you have any external devices attached to the machine, disconnect them (example, gaming controllers, USB keys, external hard disk, printers, non-essential devices).
