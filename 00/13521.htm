(are there still companies that track pencils issued?)

Do I understand correctly that you have a table like this:

tblStuffIssued

ID autonumber PK

EmployeeID long int required FK

Pencil yesno

Stapler yesno

Pen yesno

etc...

That is a really bad idea and a main reason why you're having difficulty. You have what's called a Repeating Group with one field for each office supply item.

If you later decide that you will also issue say Desk Lamps do you really want to go back to this table and add a new field, add a field to the queries on this table, to the forms and reports? Of course not.

The correct design is obtained by realizing that you have a many-to-many relation: each employee has many supplies, and each kind of supply item is issued to many employees. So this requires 3 tables. You already have tblEmployees. Now you need:

tblStockListOfSupplies

SupplyID autonumber PK

SupplyName text(50) required uniqueindex

tblStuffIssued

EmployeeID long int required PK

SupplyID long int required PK

Maybe this table will also have a DateIssued field.

This is often called the "junction table" and has PK over both fields, and one row per supply item issued to an employee.

Now adding the Desk Lamp at a later time is trivial: add a row to tblStockListOfSupplies and you are done.

Please go back and fix your database design. Then see if the answer to your original question materializes. If not, post back.

In this little demo file think of stores as analogous to your employees and stocks as analogous to your items issued.  The many-to-many relationship type between Stores and Stocks in my demo is modelled by the StoreStock table and this is the basis of the subform in the frmStoreStock form.  You should mirror this set up in your database.

The 'Select Criteria for Report' button on the form opens a dialogue form with a multi-select list box in which one or more items can be selected.  Option buttons in the dialogue form enable you to output the stores which stock all or any of the selected items.  When the report is opened it is filtered to the relevant stores and selected items in the report are shown in red amongst the other items stocked by each store.

Under the skin what is being done here is 'relational division'.  There are standard SQL solutions for this, and my code works on the same basis, but instead of using a subquery to count the selected items, in my case it uses the Count property of the list box's ItemsSelected collection.  Logically the methods are much the same, however.

The code to filter the report is in the Click event procedure of the 'Open Report' button on the dialogue form and is as follows:

Private Sub cmdOpenReport_Click()

    Const REPORTCANCELLED = 2501
    Const MESSAGETEXT = "No matching items to report."
    Dim varItem As Variant
    Dim strStockIDList As String
    Dim strCriteria As String
    Dim ctrl As Control
    
    Set ctrl = Me.lstStock
    
    If ctrl.ItemsSelected.Count > 0 Then
        For Each varItem In ctrl.ItemsSelected
            strStockIDList = strStockIDList & "," & ctrl.ItemData(varItem)
        Next varItem
        
        ' remove leading comma
        strStockIDList = Mid(strStockIDList, 2)
        
        If optAnyOrAll = 1 Then 'any item stocked
            strCriteria = "StoreID IN(SELECT StoreID " & _
                "FROM StoreStock " & _
                "WHERE StockID IN(" & strStockIDList & "))"
        Else                    'all items stocked
            strCriteria = "StoreID IN(SELECT StoreID " & _
                "FROM StoreStock " & _
                "WHERE StockID IN(" & strStockIDList & ") " & _
                "GROUP BY StoreID " & _
                "HAVING COUNT(*) = " & ctrl.ItemsSelected.Count & ")"
        End If
        
        On Error Resume Next
        DoCmd.OpenReport "rptStoreStocks", _
            View:=acViewPreview, _
            WhereCondition:=strCriteria
         Select Case Err.Number
            Case 0
            ' no error
            Case REPORTCANCELLED
            ' anticipated error
            MsgBox MESSAGETEXT, vbInformation, "Warning"
            Case Else
            ' unknown Error
            MsgBox Err.Description, vbExclamation, "Error"
        End Select
    Else
        MsgBox "No items selected", vbInformation, "Warning"
    End If
End Sub

Try and work out for yourself how this code works, but even if you can't get your teeth completely around it, if you adapt my forms to your own purposes you should be able to copy my code and amend the object names as necessary without too much trouble.

john - project say:
Noin-RCaviator,
As Jan said, a classic Gantt Chart display shows duration time on the Gantt bars. It includes additional information such as non-working weekends and holidays in the display. If you want to show a Gantt type bar for the total effort (i.e. work) for a task, that can be done but you need to decide then how that bar will react to the non-working periods on the timeline. Would you show work hours per the working calendar or would you show total hours as if they were in terms of a 24 hour clock? For example, let's say the work content for a task is 20 hours and the task starts on a Thursday morning at 8:00 am. Should the Gantt bar representing those 20 work hours end on the following Monday at noon (working calendar), or should the bar end at 4:00 am on Friday morning (20 continuous hours after it started)?

Here's something you might want to consider. You can create a custom Gantt bar that will display a time span equal to the hours in the Work field. Create a custom date field (e.g. Finish1) with the formula:
Finish1=ProjDateAdd([Start],[Work])
Note: this formula assumes you want the bar to use the working calendar. It also assumes the standard calendar. If you have a custom work calendar you would reference that as the third argument in the formula.

After you have set up the custom field. Create a custom bar that goes From the Start field To the Finish1 field. If you add your custom bar to the end of the existing Bar Styles list, your new "work" bar will be superimposed over the normal duration based Gantt bar.

Just a thought.

Even though you’ve disabled the pop-up blocker as mentioned in the error message, you’re still not able to open window.

This seems to be a problem with the AOL web browser. You probably have script debugging turn on. This is why it appears that you’re not able to open window.

   1. Open Internet Explorer. Click on the Tools and click on Internet Options.
   2. Click on the Advanced tab and there you will find options to 'Display notification about every script error ', uncheck this option.
   3. Also, check on Disable script debugging.

Now check if you’re able to open the window. I would also suggest you to turn off your security software and then give a try to open the window.
Make sure you turn it back on once you figure out the issue.

If you've installed any toolbars, uninstall it and check.
