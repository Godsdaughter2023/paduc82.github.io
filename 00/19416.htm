doug robbins - mvp office apps & services (word) say:

As I mentioned in my response to Greg Maxey, the prevention of the row following the header row from being split across pages takes precedence over the Keep with next setting of the header row.

That is, if the row that follows the header row has the attribute set so that it cannot break across the page, and the content of that row is such that it and the header row(s) will not all fit on the one page, and even if you have the paragraphs in the header row(s) set to be kept with next, the row(s) of the table that are set to be the header row(s) will be marooned on a page by themselves and the next page will contain the repeating header row(s) and ONLY as much of the next row as will fit on that page. 

That is if you are in page layout view, the balance of the text in the row after the header row(s) will not appear in the document.  If you switch  to Draft View, it will however appear.

doug robbins - mvp office apps & services (word) say:

If the row that follows the Header row is set so that it cannot be split across the pages and that row will not fit on the same page as the header rows, you cannot make it stay on the same page as the header rows by setting the header rows to be kept with next.

There is nothing that you can do with a macro to prevent this from occurring.
