TabIndex and SetFocus are properties/methods of forms.  You can have forms in PowerPoint VBA as well, but PowerPoint *slides* are not forms. 

You can do something like this to step through the shapes on a slide though:

Sub Test()

    Dim oSl As Slide
    Dim oSh As Shape
    
    Set oSl = ActivePresentation.Slides(1)
    
    For Each oSh In oSl.Shapes
        oSh.Select
        MsgBox oSh.Name
    Next

End Sub

Exit button ... what do you want this to do?  Again, if it were a form, it'd close the form and return control to the host application or vba routine.  Slides are not forms;  do you want to close the presentation or the application or ???

Sub createAnODBCCOnnection()

' Create a new sheet and select it

' Where we are going to put our table

Const RangeString As String = "$A$1:$B$2"

' Our connection string. This one is for an Excel sheet,

' so all we need is the DSN name. The DSN points to the correct

' Excel workbook, and the query selects from a specific table

' (i.e. Sheet or named range) in that file). For other DSN types

' we might need more info. in the Connection string.

Const ConnectString As String = "ODBC;DSN=acc1"

' Our Query. Has to be a valid query for the data source I think

Const QueryString As String = "SELECT * FROM Sheet1"

Dim i As Integer

Dim lo As Excel.ListObject

Dim qt As QueryTable

' Remove all ListObjects on the sheet

With ActiveSheet.ListObjects

  For i = .Count To 1 Step -1

     .Item(i).Delete

  Next

End With

With ActiveSheet.QueryTables

  For i = .Count To 1 Step -1

    .Item(i).Delete

  Next

End With

'Set lo = ActiveSheet.ListObjects.Add(xlSrcRange, Range(RangeString), , xlYes)

'lo.Name = "NewTable"

Set qt = ActiveSheet.QueryTables.Add(Connection:=ConnectString, Destination:=Range(RangeString), Sql:=QueryString)

With qt

  '.Name = "NewQuery"

  .BackgroundQuery = False

  .Refresh

  .RefreshStyle = xlOverwriteCells

  .UseListObject = True

End With

Set qt = Nothing

'lo.Delete

'Set lo = Nothing

End Sub

Indexing Problem: How to remove a large number of incorrect - Microsoft Community

keithrimmer say:

Indexing Problem: How to remove a large number of incorrect entries at one time

I am indexing a large document, and have discovered that I have made a mistake on one index entry.  The mistake occurs at 70 places throughout the document.

How can I remove ALL of the 70 entries at one time, and start-over?

CAN I in fact remove the entire 70 entries at once, or must I go through the entire document and replace/correct each entry one at a time, which will be very time-consuming?

Thanks

Keith


suzanne s. barnhill say:

Display Hidden text, which will make the index entries visible. Then search for

 

    ^d XE "<whatever is in the incorrect entry>"

 

For example, say you indexed the word "the" (!). You'd search for

 

    ^d XE "the"

 

If you display the Find dialog (you can do this through the Navigation pane, but I find it easier just to press Ctrl+H to open the Replace dialog and then select the Find tab), you can select Find In: Main document. This will not just highlight but actually select all the found instances. Change the focus back to the document (without deselecting the entries) by clicking on the title bar of the document. You can then press Delete to delete them all at once.

 

If you prefer to replace them with the correct entry, this is even easier. Insert one instance of the correct entry, select it, and Copy it to the Clipboard. Then, in the Replace dialog, enter the incorrect entry (as above) in the "Find what" box. In the "Replace with" box, type ^c (which is the code for the Clipboard contents), and click Replace All.
