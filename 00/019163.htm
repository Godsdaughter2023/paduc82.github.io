In general, typing a hyphen between two spaces will autocorrect to an em dash after completing the following word: for example, "one - two" will autocorrect to "one – two". If, however, the preceding word is followed by a footnote, the autocorrection does not occur (so, "one[1] - two" does not autocorrect to "one[1] – two").

This is such a common occurrence in academic writing that I always expected it to be fixed at some point, but some seven years into my academic career this is yet to happen, so I feel inclined to ask: is anybody aware of a way to fix this, possibly using manual autocorrections?

suzanne s. barnhill say:

What you are describing is not an em dash but an en dash (an em dash requires two hyphens), but I noticed (in Outlook) the same issue with an em dash following a fraction. I suspect that this is related to Word's failure to capitalize the first letter of a sentence if the previous sentence ends with a number. There was a time (previous to Word 2003, which is the oldest version I currently have installed) when Word would perform this AutoFormat As You Type action, and it occurred even after the numbers in numbered lists. Apparently a large number of corporate users complained because they wanted numbered lists that were not capitalized. When this situation was corrected, it apparently extended to all numbers everywhere, even in running text (the Law of Unintended Consequences), and I'm guessing that the "correction" also applies to other AutoFormat actions.

Yes, this is incredibly annoying, but there are workarounds and solutions. The best is to either use the built-in keyboard shortcuts for en and em dashes (which I consider clumsy) or create your own. I have been using my own keyboard shortcuts since before Word included any (and long before AutoFormat As You Type), so it's usually easier for me to press Ctrl+Shift+N to get an en dash than to count on Word to do it (which in fact it won't in most of the situations where I use it).

Try this one.

Select any two sheets that have hidden sheets between them.  Run macro to unhide all.

Sub UnhideSheets()
'// = = = = = = = = = = = = = = = = = = =
'// Select a sheet on each side of hidden sheets.
'// Macro will unhide sheets between selected sheets.
'// Dana DeLouis  Aug 2nd, 2008
'// = = = = = = = = = = = = = = = = = = =
   Dim a As Long
   Dim b As Long
   Dim J As Long
  
   With ActiveWindow.SelectedSheets
      a = .item(1).Index + 1
      b = .item(2).Index - 1
      For J = a To b
         Sheets(J).Visible = True
      Next J
   End With
End Sub
