Got it fixed, but it probably won't help the other people in this thread.
The first hint that lead me in the right direction was the error 80070005 I got, which is different from that 80070490 spectrawomyn reported (next answer below mine).
KB2862330 is an USB driver update, and xxxx0005 makes it an "access denied" error.

About a year and a half ago, I had a problem with USB devices, apparently caused by a third party software setup deleting the file USB.INF from my system.
(see http://answers.microsoft.com/en-us/windows/forum/windows_7-hardware/previously-known-usb-devices-stop-being-recognized/b9951796-1b8d-4786-8c94-a7ca5a91cc26)

I got it fixed it then by copying that file over from another system. To make sure someone couldn't delete it again, it looks like I also made it read only (and forgot all about that later).
That was the reason for my access denied errors this month: removed the read only status from usb.inf, and the update succeeded.
