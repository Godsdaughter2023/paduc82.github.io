APA Headings - Level 2

I have been having an issue lately separating my paragraph from my heading.  Under APA guidelines, the third level heading is supposed to be with the rest of the paragraph.  Unfortunately, when I apply the third level heading to the first portion of the heading, it automatically applies it to the rest of the paragraph, making it look atrocious on my table of contents.  If you have the secret to fixing this, I would much appreciate it!

jb54321 say: Edit: title should be level 3

suzanne s. barnhill say:
Create your heading in a separate paragraph, applying the Heading 3 style (or whatever other style you're using for that level). Make the rest of the text a second paragraph in Body Text style (or whatever style you're using for body text). Then click in the first (heading) paragraph and press Alt+Ctrl+Enter. This will insert a style separator. If you're displaying nonprinting characters (formatting marks), you'll see, between the heading part and the body text part, a paragraph mark with a dotted line around it. The text will behave like a single paragraph, but only the heading part will appear in the TOC.

Multiple Master Projects in Project Online with Project Pro - Microsoft Community

james2600 say:

Multiple Master Projects in Project Online with Project Pro

Have an interesting situation here and could use some advice.

At a software firm and implementing Project Pro to manage resources.  We're a large company and have multiple teams and initiatives.  One of these teams is QA.  Here, QA works like a horizontal service. That is, other teams request that code is checked by QA, and the QA lead will set the schedule. 

Im trying to come up with a scheme where our team managers can set tasks dependent on QA while allowing QA to make their tasks dependent on each other as well.

As far as I can tell, this would require, at minimum, the following projects:

Team Project, where programming tasks will live

Team QA project, where QA tasks related to the Team Project will live

Master Project, to consolidate the information

QA Master Project, for QA to consolidate their information

Where Team QA would be a sub project of both the Team Project, and the QA Master, and, Team Project would be a sub project of Master Project.

In our situation, we would have about 35 Team Projects, and thus, 35 Team QA projects.

The QA Master Project would allow our QA lead to consolidate all of the mini Team QA projects.

Currently, we're trying to test this configuration out, but Project immediately gave us warnings about a Team QA project existing in different master projects and publishing.

Has anyone dealt with anything like this before?  Is there a suggested workaround, or perhaps a better way to set all these projects up?

We're a pretty big shop with something like 35 projects to keep track of, and three separate horizontal services, such as QA, that would all need to be accounted for.

Thanks for reading!

dale howard [mvp] say:

James2600 --

My approach to this problem would be to avoid creating and saving master projects at all.  Instead, what I would do personally is to create TEMPORARY master projects whenever needed.  It is very simple using the Project Center page to select the subprojects and then to create a temporary master project for analysis and reporting.  That would eliminate the warning messages you are seeing.  When you finish working with a temporary master project, you simply close it and do not save the changes.  If you made changes to any of the subprojects, you are then asked if you want to save the changes to those subprojects as well.  Just a thought.  I believe others in this forum will share their thoughts with you as well.

Hope this helps.

james2600 say:

Hadn't considered using temp master projects.  My concern would be that some of the users are not super technical and the work required to select a fair number of projects in order to set dependencies would cause usage/compliance to fall.

Thank you for taking the time to answer, I'll absolutely add this method to our tests to see how it works out!
