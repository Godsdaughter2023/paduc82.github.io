What does Windows as a Service entail?

Since the introduction of Windows Update as part of Windows, Microsoft has delivered updates as way to keep Windows updated and secure. This has always been the way the company keeps commercial versions of Windows on the market current.

With Windows 10, Microsoft is making significant changes to this model by delivering new methods of servicing Windows. Taking into account we now live in a mobile, constantly connected world, where threats happen all the time. Part of the solution to this is to keep users current by providing the latest updates as soon as they are available in addition to delivering features as soon as they ready too.

Microsoft will do this using three types of service branches for Windows 10:

  • Security Updates, Features and Fixes are automatically applied.
  • There is no option to delay or customize these updates.
  • Windows 10 Home
  • CBB includes the requirements of the Current Branch, but also provides the option of customizing when and which Security Updates, Features and Fixes are applied, similar to how Windows Update works today in current versions of Windows. 
  • Updates cannot be deferred indefinitely.
  • Windows Updates can be managed using enterprise management tools such as Windows Update for Business and/ WSUS.
  • Windows 10 Pro
  • Windows 10 Enterprise
  • Windows 10 Education
  • Only available to volume license customers running Windows 10 Enterprise.
  • Flexible options for managing Windows Updates.
  • Windows Updates can be customized to only accept Security Fixes.
  • Windows Updates can be managed using enterprise management tools such as Windows Update for Business and/ WSUS.
  • Windows 10 Enterprise

a. user say:

Hi,

This issue might occur if the Direct Memory Access (DMA) for a drive is turned off or the write caching for the drives for the policies.

Let me explain the details about the role of DMA during a data transfer within a drive. DMA means transfer of data between IO and disk without intervention of CPU.

The IO processor gets the control of IO buses along with control buses. Suppose you have connected your peripheral device, e.g., a flash drive on your computer. Now you have started copying some data from hard disk to that pen drive.

Now to transfer data, the CPU must have to control that. But using CPU only for copying data is not efficient. Here comes DMA controller.  It takes the charge of data transfer between devices and make CPU free for other tasks. But remember, the hardwire should actually support DMA & It can only be enabled or disabled using the operating system.

I would suggest you to enable write caching for the second HDD and check if it fixes the issue.

Follow the methods:

Method 1:

Follow the steps to enable write caching on the device:

Note: Make sure that you have logged-in as an administrator on the computer before following the steps:

  • Press Windows+X keys and select Device Manager.
  • Locate Disk Drives and expand the icon. Right Click on the second HDD and choose Properties.
  • Navigate to Policies tab and check the box which says “Enable write caching on the device”.
  • Click Apply and then on OK.

Restart the computer and verify if the speed still decreases while transferring files on the Western Digital hard drive between the partitions. If yes, proceed to the next method.

Method 2:

Follow the steps to check if you are able to find the option to enable DMA under IDE ATA/ATAPI controllers in Device Manager;

  • Press Windows+X keys and select Device Manager.
  • Locate IDE ATA/ATAPI controllers and expand the device icon.
  • Right-click on any label where “Channel” is present & Click on Properties.
  • Check if you are able to find Advance Settings tab on the window. If yes, Navigate to that tab & enable DMA by checking “Enable DMA” & then click on OK.

Please let us know if the issue still persist.
