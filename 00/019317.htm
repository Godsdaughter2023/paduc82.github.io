Cases like this are much more rare than they used to be.

 

In the past, when this was discussed on a daily basis, the majority of cases where of people having set passwords at some dim-and-distant point in the past and having forgotten. If you Windows cannot read your card, then it suggests that there really is a password on there - reformatting the phone does not perform any actions on the memory card, there is proven history any phone (Nokia or otherwise) adding passwords to memory cards without the user having initialised the process from the memory menu functions. If Windows doens't see the device at all, that is more suggestive of the card having been corrupted than a password being active it is not the normal behaviour of Windows to totally ignore protected media, it would more usually be expected to demand that security requirements are met (ie: password entered) before allowing the card to be read.

Try it like this. I have assumed that your list in column A starts in A2 and has no blank cells until the last filled cell in column A.

Sub FillInternetForm2()
    Dim IE As Object
    Dim rngC As Range
    
    With ThisWorkbook.Sheets("sheet1")
        
        For Each rngC In .Range(.Range("A2"), .Cells(.Rows.Count, "A").End(xlUp))
            
            Set IE = CreateObject("InternetExplorer.Application")
            
            IE.Navigate "https://safer.fmcsa.dot.gov/companysnapshot.aspx"
            IE.Visible = True
            
            While IE.busy
                DoEvents 'wait until IE is done loading page.
            Wend
            
            IE.document.all("query_string").Value = rngC.Value
            IE.document.forms(0).submit
            
            Set HTML = IE.document
            Set icstatus = HTML.getelementsbyclassname("queryfield")
            
            rngC.Offset(0, 1).Value = icstatus(1).textcontent
            
            IE.Quit
            Set IE = Nothing
    
        Next rngC
    End With
End Sub

  5. Restart and retest Word.

How do I use VBA to count the number of images with specific - Microsoft Community

imjon say:

How do I use VBA to count the number of images with specific names?

If I have a folder with a bunch of image names starting with IMAGE, is there a way for me to use VBA to count how many images in the folder contains the word IMAGE in the file name?

Thanks
john sr wilson say:
This code should count the names in the folder and any sub folders. If you don't want to count sub folders delete or comment out the parts indicated.

Sub getfiles()
Dim fso As Object
Dim objfolder As Object
Dim objfile As Object
Dim lCount As Long
Dim strpath As String
Dim objsub As Object 'not needed if no sub folders
strpath = "C:\Test\" 'Path to your folder
Set fso = CreateObject("Scripting.FileSystemObject")
Set objfolder = fso.GetFolder(strpath)
For Each objfile In objfolder.Files
If UCase(objfile.Name) Like "*IMAGE*" Then lCount = lCount + 1
Next objfile
'Optional delete this block if no sub folders ********************
For Each objsub In objfolder.SubFolders
For Each objfile In objsub.Files
If UCase(objfile.Name) Like "*IMAGE*" Then lCount = lCount + 1
Next objfile
Next objsub
'********************************************************
MsgBox lCount & " Files have 'IMAGE' in their name."
End Sub

If you ONLY want files that START with IMAGE use Like "IMAGE*"

willy denoyette say:

Why does this happen - don't Microsoft know about this problem?

You erroneously assume everyone has this problem which is not the case.

You have to turn on, one by one, everything you have turned off and watch when the disk start performing excessive IO commands. Do this by means of the Task manager > Performance > Open Resource monitor. Watch the Disk Read/Write counters and look which process is consuming most of the IO's. Excessive IO's are in the order of MBytes/sec for long periods of time (say several seconds/minutes).

Don't disable superfetch or prefetch, they are there to enhance performance and have nothing to do with your problem.
