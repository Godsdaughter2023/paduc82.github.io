  • User-defined browser settings

For example, the Reset Internet Explorer Settings feature resets security settings, privacy settings, and zone settings for all users on the computer. Any settings that are set by add-ins or toolbars that you downloaded are also reset.

    If any settings were created by the OEM before you received the computer, the Restore Internet Explorer Settings feature restores Internet Explorer customizations that were applied by the OEM in the initial package.

  • Browsing history

Alternate email domains with Office 365/GoDaddy starter email - Microsoft Community

edwardhoffman say:

Alternate email domains with Office 365/GoDaddy starter email package product

It took me a bit to get this going. Partially due to some incorrect information floating around the www's. I wanted to come back here and post the exact steps I took to help others who are trying to add alternate email domains to GoDaddy's O365 products.


Adding alternate email domains to your GoDaddy-Office365 starter email package.
This may only work if the alternate email domains are also under the same account or are also hosted at GoDaddy (mine are all under the same GoDaddy account). Can someone confirm?

Run from 64-bit OS - Windows 7 SP1 or Server 2008 R2 SP1 or higher (requires .NET 3.5.x)
Install required software -
MS Online Sign-in Assistant - go.microsoft.com/.../
Azure Active Directory Powershell module - go.microsoft.com/.../

Open Azure Powershell and run the following commands -


PS C:\Windows\system32> $usercredential=get-credential


[At the prompt enter your Office365 credentials]


PS C:\Windows\system32> connect-msolservice -credential $usercredential


[If you receive an error here the supplied credentials are probably incorrect]


PS C:\Windows\system32> new-msoldomain -Authentication managed -name contoso.com

Name Status Authentication
---- ------ --------------
contoso.com Unverified Managed


PS C:\Windows\system32> get-msoldomainverificationdns -DomainName contoso.com -mode dnstxtrecord


Label : contoso.com
Text : MS=ms11111111
Ttl : 3600


[Add TXT record to external DNS for contoso.com and wait for DNS propagation]


PS C:\Windows\system32> confirm-msoldomain -domainname contoso.com

Availability AvailabilityDetails EmailVerifiedInformation
------------ ------------------- ------------------------
AvailableImmediately The domain has been successfully verified for your account.


[Wait for GoDaddy propagation? I waited over an hour before I could add an email address with the new email domain. YMMV]


Log into GoDaddy.
I don't have access to the DOMAINS section in Admin center but the alternate email domain now appears as an option when adding secondary email addresses to users.

jasmine cao msft support say:

Hi JJ,

When you send emails to an alias, exchange server will identify it as the primary email. There is no such inbox rule that can sort emails for aliases. You can use a workaround, which needs to involve the administrator if you’re not the one. Here are the steps:

1.Create a mail user: Admin > Admin Centers > Exchange > recipients > contacts > Mail user
About Mail user, please refer to this document. Here Jasmine2 is the alias of user1.

2.Create a transport rule to redirect all emails sent to the mail user to your mailbox: Admin > Admin Centers > Exchange > mail flow > rules.

3.Create an inbox rule to move emails sent to this mail user to another folder of your mailbox: Go to the mailbox (user1 in this test) > Setting > Options > Inbox and sweep rules > create a rule.


Now, every email sent to the mail user Jasmine2 will be delivered to Junk Email folder of user1.

Best regards,
Jasmine
