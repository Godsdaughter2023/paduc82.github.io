"...The bottom right corner of the front page does not print and even disappears on the monitor..."

This suggests that you might have the text outside the printable area on the page, or in other words you are too close to the edge of the paper. Reference: http://office.microsoft.com/en-in/publisher-help/common-printing-issues-HA010247849.aspx#BM5 and http://ed.mvps.org/Static.aspx?=Publisher/horidiag 

If this is the problem you need to move the text and/or the text box into the printable area of the page by changing your page layout and the text box position, reducing the size of your font, the number of words or perhaps flowing the text to a text box on the next page. If you cannot see the text when you do the print preview then it will not print on the page.

If this is not the problem then do you have some other design object overlapping the text and thus obscuring it? You can select that object > Arrange > Order and send to the back, or the opposite, select the text and bring forward.

Bug in Frequency function

The Frequency function can return unreliable results when the data array contains errors. This can lead to incorrect values in formulas. Try entering in a cell of a new workbook:

=FREQUENCY(0,{#N/A})

Now Change the 0 to 1, 2, and 3 pressing enter after each change. The value returned can changes to an arbitrary non-zero integer in my version [Windows 8.1, Excel 2013 (15.0.4771.1001) 64-bit].

For example to find the closest match in a range (B2:B6) to a given value (5) you can try.

=LOOKUP(2,1/FREQUENCY(0,(B2:B6-5)^2),+B2:B6)

but this returns unreliable results if the range contains nonnumeric values. I believe this worked ok in previous versions.
