john korchok say:

Here is PowerPoint code that illustrates recoloring an EPS. There are 2 ungroups, 1 to turn the graphic into a PowerPoint shape object and the other to break apart the graphic to recolor the components.:

Sub RecolorShapes()
  Dim myShapeRange As ShapeRange, x As Integer
  With ActivePresentation.Slides(1).Shapes
    .AddPicture FileName:="C:\PathToYourFile.eps", LinkToFile:=False, SaveWithDocument:=True, Left:=100, Top:=100
    .SelectAll
  End With
  ActiveWindow.Selection.ShapeRange.Ungroup.Select
  ActiveWindow.Selection.ShapeRange.Ungroup.Select
  Set myShapeRange = ActiveWindow.Selection.ShapeRange
  For x = 1 To myShapeRange.Count
    myShapeRange(x).Fill.ForeColor.RGB = RGB(255, 102, 0)
  Next x
End Sub

Some EPSs include bounding boxes that become a shapes when ungrouped. Most of the time, this is shape 1. If your EPSs have this, you'll have to modify the loop that applies color to skip shape 1.

steve rindsberg say:

Depending on the source of the EPS and how it's been saved, you might be able to write VBA code to modify the colors in the EPS files themselves directly.  You'd need something between a nodding acquaintance and a deep familiarity with the PostScript language, minimum, to work out where and how the colors are defined in the EPS.  

Because EPS is actually more of a programming language than a graphics format, it's impossible to give examples of how you'd go about this; each program's EPS exports will be different.

You won't be able to affect the colors of images, but as long as your files are all vector graphics, it might just work.  I did something similar in the past to convert PowerPoint's RGB PostScript output to CMYK so that it could be color separated.
