It was not easy to locate because it is not in a cell, so displaying equations and then LEN()-counting their length did not find anything.  When the worksheet concerned is not hidden, Excel's "Formulas | Name Manager", showed the "Value" as {...}, which seems to be used for some kinds of references as well as for vector-populated variables.  And the "Refers To" showed a long string of numbers ending in ... to indicate more than can be seen here.

For this, it took one of the reports from j-walk's PUP7 -- their "Utility | Name Lister" which puts all workbook names (whetherthe sheets are hidden or not) into a separate spreadsheet that you can analyse.  Lo and behold, one variable was 8240 characters long.  This particular variable retained all the contents of the last vector operated on until its macro was called again.  So, no matter what I did, I could not get back below the limit until I used the macro on some dummy numbers to flush out the variable.  That done, I could easily save my .xlsm without any complaints.

jumpingcrab say:

I came across the same error today and after digging around for a while found the issue.

I have a shared workbook that contains an indented Bill Of Materials containing some 7000 lines and up to 6 indent levels at various sections of the BOM.

I had created a macro that automatically outlines the BOM using the indentation levels, so users can open up the BOM to a certain depth by using the outline level buttons.

For the users' convenience, the shared workbook saves the user outline filter settings (Review->Share Workbook->Advanced). These settings are stored in hidden names per user and when a lot of hidden indent levels exist, the filtered row list (stored in a single name) can grow beyond 8291 characters.

The following macro will find the user filter names with long values in the shared workbook, which will help you find the culprit (modify as required to tweak your search :) ).

Sub FindLongNames()
Dim nName As Name
For Each nName In ActiveWorkbook.Names
    If InStr(nName.Name, "wvu.Rows") And Len(nName.Value) > 1000 Then
        If MsgBox(nName.Name & vbCrLf & nName.Value, vbOKCancel) = vbCancel Then
            Exit Sub
        End If
    End If
Next nName
End Sub

Turning off user filters saving in Shared mode solves this particular issue.

Select cells with inconsistent formula error. - Microsoft Community

tedmh say:

Select cells with inconsistent formula error.

Go to special lets you select error formulas, but this doesn't seem to work for soft errors.  Is there a way to select all errors in a worksheet, including the soft errors such as inconsistent formula errors?
ronrosenfeld say: You can do this with a VBA Macro.

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.

It will select all cells on the active sheet that have errors, including some that are not marked as such.
The list of possible errors that will be flagged by the "Item" index (1 to 7) below include:

  1. The cell evaluates to an error value.
  2. The cell contains a text date with 2 digit years.
  3. The cell contains a number stored as text.
  4. The cell contains an inconsistent formula for a region.
  5. The cell contains a formula omitting a cell for a region.
  6. The cell which is unlocked contains a formula.
  7. The cell contains a formula referring to empty cells.
You may want to omit some of them, depending on your worksheet setup.

This selects ALL errors:
===========================
Option Explicit
Sub SelectAllErrors()
    Dim rg As Range, c As Range, rErrors As Range
    Dim i As Long
Set rg = ActiveSheet.UsedRange
For Each c In rg
    For i = 1 To 7
    If c.Errors(i).Value Then
        If rErrors Is Nothing Then
            Set rErrors = c
        Else
            Set rErrors = Union(c, rErrors)
        End If
        Debug.Print c.Address, i, c.Errors.Item(i).Value
    End If
    Next i
Next c
rErrors.Select
End Sub
=========================

This would select a subset of the errors:

==================================
Option Explicit
Sub SelectErrors()
    Dim rg As Range, c As Range, rErrors As Range
    Dim i As Long
Set rg = ActiveSheet.UsedRange
For Each c In rg
    For i = 1 To 7
    Select Case i
        Case Is = xlEvaluateToError, xlNumberAsText, xlInconsistentFormula, _
            xlOmittedCells
        If c.Errors(i).Value Then
            If rErrors Is Nothing Then
                Set rErrors = c
            Else
                Set rErrors = Union(c, rErrors)
            End If
        End If
    End Select
    Next i
Next c
rErrors.Select
End Sub
=============================

You may want to do something different, but the above should get you started.

If you are refereeing to add Windows Task Manager (or any program) to your Startup folder (that will make it work each time you start the windows and you will see it in the system tray), please follow the below instructions:

 

a. Click Start>Programs, then right click on Startup folder and choose Open.

b. On the Startup window, go to File menu and choose New>Shortcut.

c. In the dialog box click the Browse button to specify the Task Manager program, it's by default at C:\windows\system32\taskmgr.exe (Where C is the partition that your windows XP is installed on).

d. After specifying the program path, click next and finish. Then you will see a new shortcut (computer icon) in the Startup folder.

e. Right click on that shortcut and choose Properties, select Minimized from the Run combo box, and click Ok, then close the Startup folder.

f. At the next time you start your computer the program will run automatically and you will see it in the system tray.
