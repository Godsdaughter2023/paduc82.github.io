a. user say:

Hi James,


It might have happened that some of the Windows files might have gone corrupt. As a result of which, Automatic Repair operation has started on the computer.

  1. Which is the program that you had uninstalled prior to these issues? Is it any way related to Windows program?
  2. Are you able to boot the computer normally after repair gets over or it a loop that is happening every time you start Windows?

If it’s still automatic repair screen, don’t interrupt and allow the repair operation to complete. But, if it happening every time, you can use a hard reset to shut down the computer and proceed further with troubleshooting methods.


Scenario 1:

If you are able to boot the computer normally, I would suggest you to run SFC scan and check if there are missing or corrupt system files getting replaced or fixed. Refer to the link to run SFC scan:

Follow the steps from section "Windows 8.1 or Windows 8":

http://support.microsoft.com/kb/929833/en-us  


Scenario 2:

If you are unable to boot the computer normally, I would suggest you to perform System Restore using Windows 8 installation media.


If you do not have a Windows 8 media, refer to the link to create a Windows 8 installation media using a different Windows 8 computer:

“How to create installation media for a PC refresh or reset”

http://windows.microsoft.com/en-IN/windows-8/create-reset-refresh-media  

Note: You would require the product key for the version of Windows 8 that is installed on your computer while preparing an installation media.


Boot to Recovery Environment with the installation media:

Follow the steps:

  • After you boot your computer using Windows DVD or System Repair Disc, a black screen appears with gray text "Press any key to boot from CD or DVD". Press any key.
  • Select the correct time and Keyboard type.
  • Click Repair your computer in the lower left corner.
  • Select Troubleshoot from Choose an option screen.
  • Click Advanced options in Troubleshoot screen.
  • Select System Restore.

Please get back to us with the status of this issue.
