The document map  in W2002 to 2007 assigns heading paragraph levels to  paragraphs that it "thinks" should be headings.  It does this as direct formatting, so you can clear it  from an affected paragraph by selecting it and pressing Ctrl+q (to clear direct paragraph formatting).  Or go to home tab > paragraph group > launcher arrow (lower right) > select body text as the paragraph level.

Hi Kelly,

Try as follows:

  • Alt-F11 to open the VBA editor
  • Alt-IM to insert a new code module
  • In the new module, paste the following code 
'=========>>
Option Explicit

'--------->>
Public Sub Tester()
    Dim srcWB As Workbook, destWB As Workbook
    Dim srcSH As Worksheet, destSH As Range
    Dim srcRng As Range, destRng As Range
    Dim arrIn As Variant
    Dim aStr As String, sFileName As String
    Dim LRow As Long, LCol As Long
    Dim i As Long, iRow As Long
    Const sStr As String = "Payee Total:"
    Const iFirstDataRow As Long = 3                               '<<=== Modify
 
    Set srcWB = ActiveWorkbook
    Set srcSH = srcWB.Sheets("Sheet1")                          '<<=== Modify   

    Application.ScreenUpdating = False
    With srcSH
        LRow = LastRow(srcSH, .Columns("A:A"))
        LCol = LastCol(srcSH)
        Set srcRng = .Range("A1").Resize(LRow, LCol)
    End With

    arrIn = srcRng.Value

    iRow = iFirstDataRow
    For i = 1 To UBound(arrIn, 1)
        If arrIn(i, 1) Like sStr & "*" Then
            aStr = Trim(Replace(arrIn(i, 1), sStr, vbNullString))
            aStr = Replace(aStr, Chr(160), vbNullString)
            sFileName = aStr & ".xlsx"
            Set srcRng = srcSH.Range("A" & iRow).Resize(i - iRow + 1, LCol)
            Set destWB = Workbooks.Add(xlWBATWorksheet)
            Set destRng = destWB.Sheets(1).Range("A2")
            srcRng.Copy Destination:=destRng
            destWB.SaveAs Filename:=sFileName, FileFormat:=51
            destWB.Close SaveChanges:=False
            iRow = i + 1
        End If
    Next i
    Application.ScreenUpdating = True
End Sub

'--------->>
Public Function LastRow(SH As Worksheet, _
                        Optional Rng As Range)
    If Rng Is Nothing Then
        Set Rng = SH.Cells
    End If

    On Error Resume Next
    LastRow = Rng.Find(What:="*", _
                       after:=Rng.Cells(1), _
                       Lookat:=xlPart, _
                       LookIn:=xlFormulas, _
                       SearchOrder:=xlByRows, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Row
    On Error GoTo 0
End Function

'--------->>
Public Function LastCol(SH As Worksheet, _
                        Optional Rng As Range)
    If Rng Is Nothing Then
        Set Rng = SH.Cells
    End If

    On Error Resume Next
    LastCol = Rng.Find(What:="*", _
                       after:=Rng.Cells(1), _
                       Lookat:=xlPart, _
                       LookIn:=xlFormulas, _
                       SearchOrder:=xlByColumns, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Column
    On Error GoTo 0
End Function
'<<=========

symbols in mail merge - Microsoft Community

llib.wf say:

symbols in mail merge

Is there a way for mail merge to retrieve symbols that replace text fractions such as, 1/2, 1/3, 1/4 from Excel database to the merged Word document? The normal procedure fails to print the fraction symbols.

john mcghie say:
No.  Only ANSI characters will cross the mail merge boundary.

What you COULD do is send a string that identifies the character you want, and use Find/replace to substitute it on the Word side.

Of course, that's double-handling and gets a bit laborious for a large mail shot.

You can run up a chunk of VBA to automate the Find/Replace if you're OK at coding.

Hope this helps

bob jones aka: cybertaz say:
Extending on John's reply, Fractions in Excel are not real symbols [i.e., characters] The appearance is imposed as a display formatting attribute. If you select a cell which displays 3/4 you'll see in the Formula Bar that what is actually stored in the cell is a value of .75, not the fraction. Formatting is not retained in a merge.
