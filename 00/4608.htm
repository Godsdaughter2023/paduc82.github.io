joeu2004 say: Stephens wrote:
Need help updating this formula, need to add this criteria. Where the data in B Column is equal to $AB$1 or $AE$1 ( Inv!$B$3:$B$14068=$AE$1)

It works as it is, but when I try to add the additional criteria it returns N/A
=AVERAGE(IF(Inv!$B$3:$B$14068=$AB$1,IF(Inv!$M$3:$M$14068={"01","06","08","12","41","42","43","44","62","64"},Inv!$P$3:$P$14068)))

Try:

=AVERAGE(IF((Inv!$B$3:$B$14068=$AB$1) + (Inv!$B$3:$B$14068=$AE$1)>0,
IF(Inv!$M$3:$M$14068={"01","06","08","12","41","42","43","44","62","64"},Inv!$P$3:$P$14068)))

Technically, ">0" is not needed in this context because the two conditions are mutually exclusive; that is, both cannot be TRUE at the same time.

However, it is good to get into the habit of writing ">0" because mutual exclusion is a subtle distinction, a requirement that you might easily overlook in the future.

BTW, are the values in Inv!$M$3:$M$14068 truly numeric text like "01", "06", etc?  If not, you should remove the quotes.  That is, I wonder if the array should be {1,6,8,12,41,42,43,44,62,64}.

Finally, beware that the formula will return a #DIV/0 error if the conditions are not met in at least one row.  If that might be a possibility and if you do not require Excel 2003 compatibility, it would behoove you to write something like:

=IFERROR(AVERAGE(IF((Inv!$B$3:$B$14068=$AB$1) + (Inv!$B$3:$B$14068=$AE$1)>0,
IF(Inv!$M$3:$M$14068={"01","06","08","12","41","42","43","44","62","64"},Inv!$P$3:$P$14068))),"")

That returns the null string ("") if the AVERAGE fails, presumably with a #DIV/0 error.  You might prefer to replace the null string with 0 (zero).

[EDIT] Food for thought....

I doubt that you need absolute references in this context.  That is, it does not seem like you intend to copy this formula down or across, since as written, nothing would change.  So it might be easier to type the following:

=IFERROR(AVERAGE(IF((Inv!B3:B14068=AB1) + (Inv!B3:B14068=AE1)>0,
IF(Inv!M3:M14068={"01","06","08","12","41","42","43","44","62","64"},Inv!P3:P14068))),"")

Alternatively, if you do intend to copy the formula down a column, perhaps only the references to AB1 and AE1 should be relative, to wit:

=IFERROR(AVERAGE(IF((Inv!$B$3:$B$14068=AB1) + (Inv!$B$3:$B$14068=AE1)>0,

IF(Inv!$M$3:$M$14068={"01","06","08","12","41","42","43","44","62","64"},Inv!$P$3:$P$14068))),"")

cortig say:
In the Windows version of Outlook, you have different choices for
displaying the rich text content of your emails.
This is one of the options, but the problem is that only Outlook for
Windows can display it. All other clients (like Mail or Outlook for Mac)
will fail to understand it, they will display the plain text version of
the email and the rest will be displayed as a winmail.dat attachment
(and some apps out there can decode that and display the content, but it
sure isn't built in). And worse: the attachments in the email are
embedded and hiddent within this winmail.dat archive.
 
The solution is to turn off the option to send the email as rich text
using this format. Use HTML instead of you want (which should work
everywhere).
There's nothing you can really do on the recipient side.
