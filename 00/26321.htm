Have you lost files, or changes you made when editing a file are - Microsoft Community

bob grosh say:

Have you lost files, or changes you made when editing a file are gone? I think i FINALY found out why !!!

This is a copy of a feedback I submitted on 10TP. Open for discussion. See additional comments below the feedback.

FEEDBACK

----------------------------------------------------------

Photo App eats files. (build 9926)
When you save a edited photo it disappears, sometimes a few seconds after you save it back to the original, sometimes a day later. Sometimes all the files you edit, rotate, crop or otherwise adjust in the app all disappear a day or two later, If you monitor the folder where you saved them, for a few ays in File Explorer, you can see this happen.

They are there for days, then one at at time, just seconds apart, they blink out of existence.

Impossible? Read on.

There are hundreds  of post since late 2011 reporting this mystery with various photo editing programs, including Photo Galery. No answer from Microsoft has ever found the real problem. It has been blamed on anti virus programs, or claimed that the files are not lost, but simply hidden or moved to the appdata folder. Those solutions only work for a very small minority.

I am not running any anti virus program except the built in one on Windows 10.

The vanishing pictures are not on the machine they were on, anywhere! Not the recycle bin, not the appdata folder, not anywhere. They are not hidden or changed to a system file. I am using a clean install of Windows 10 to the unpartitioned space on a HD with no other partitions. No easy transfer or recovery from a backup.. No registry hacks, no attempt to repair problems on 10 by inserting old versions of files or code. no third party software.

Still say it is impossible? What if I tell you photos disappear, sometimes days later if you change the rating, rotation or tag using Windows File Explorer?

OK, this has been driving me crazy for months.

I can re-create it.
I think I can even explain it.

FIRST HINT!
I have several personal computers, desktops, laptops and tablets. SOME OF THEM STAY OFF FOR A DAY OR TWO AT A TIME. One is running Windows 10TP build 9926. the others are Windows 8.1 and Windows 7 Ultimate. !!! NO XP  or other OS's!!!

SECOND HINT!
I use the library feature of explorer, and, I use OneDrive to sync photos.

----------------------
THE ANSWERS
On 10 Pick a photo from your Photo Library, one that is on a local file under the OneDrive folder. Edit it
Save it. (Use the smaller icon with the down arrow under it, NOT the Save Copy icon.
If you are also monitoring that folder using File Explorer on the same Windows 10 PC, you will see the APP save a temporary file, then delete the original photo and finally rename the temporary file back to the original name.
If your other PC's are all off, It will stay there. If other PC's are on, and depending on how the libraries are set up, the file will vanish seconds or perhaps a minute later. Even if the file stays put where you saved it, it might disappear later when you turn on one of the other PC's.

What happens when you have a windows 7 PC TURNED ON when you save the edited a photo that is synced to OneDrive?
Well, on the second (idle, not sleeping) PC, if you have added the Pictures Folder, (or Camera Roll or any folder you were editing) on OneDrive to your Library, and leave the default save folder for the library as it's default local folder, then here is what happens: You can see the temporary file copying down by OneDrive, but it goes to the default save location, not the OneDrive folder you added to the library. Then the original local copy of the file you edited is deleted from OneDrive and the deletion of the file is replicated to this second PC. depending on the timing, and other PC's you have online, the deletion gets replicated to other machines, back to the cloud and to the new version you just edited on the original PC. In this case, the original file might be in the trash bin of one of the other PC's you have synced with OneDrive. OR - It might be on the default save location of any Library, on any of the PC's that were turned on when you edited the picture.
(Note: the problem is not a fault of OneDrive, it is doing exactly what it is supposed to do. Besides, I replicated this problem after replacing OneDrive with  Dropbox or Google Drive.)

If another computer is running 8.1, the result is the similar, however the SmartLinks that are used on the OneDrive Modern App may change the symptoms. Files may get deleted quicker and end up no place at all, not on any computer in any default save folder of a library or any trash can. Since my desktop has huge conventional drives, but the tablet has smaller SSD's, both the deleted original and/or the modified copy have a high probability of NOT get backed up to the History.

Turning on or off the batch mode of OneDrive may change the number of files that are lost, or make you think you cured your problem, only to find outthat the problem is still there, it just happens a little differently.

NOT ONEDRIVE

As I said, the problem is not OneDrive, Dropbox or Google Drive. monitoring the edited folders on several system and observing the files tells me they are all doing exactly what they are designed to do. Besides, You can still lose files without syncing to the cloud. Take some folder on another PC in your Homegroup and add it to a Library. Don't set that folder as the default save location, Edit the photo. It will still be in the same library, but in the default save folder.

NOT THE PHOTO APP

The problem is not the Photo App. I still lose files using Photo Gallery or even rotating them with Windows Explorer. If you edit photos on a local drive, all is good with the world. Life is a bowl or roses. You can even edit files in OneDrive folders and not lose files. Picasa, Flicker, Sony Motion Browser, and Microsoft Photo Gallery all work fine on OneDrive Folders. AS LONG AS YOU DON'T INCLUDE THEM IN A LIBRARY and forget to make the synced folder the default save location. More importantly, you need to make the same folder the default save location on every computer you sync to the cloud.

NOT LIBRARYS

OK, so everything so far points to Libraries as the culprit. But, they are doing exactly what they are designed to do.

THE CULPRET

It is not unusual for a user to add folders from multiple drives or even different PC's to a Library. Not understanding the unintended consequences may be a little to much to expect from the average user. Even experienced users, who, after all, are the ones likely to use all of these features to make things quick and easy, might miss the fact that little used tablet or laptop sitting unnoticed in a brief case might one day get turned on and start sucking up pictures and tossing them around like candy and treats you hid for your kids to find on Christmas morning.

The problem, aside from poor user understanding of the unintended consequences of sophisticated configurations is compounded by timing. Since this happens through the cloud, over the net, on a mixture of slow conventional drives, fast SSD drive and a variety of processors, the replication of this behavior is erratic. I edited 10 photo in 10 minutes on one PC. Seven of the ten vanished. Two of those 7 were found in a tablet Library. three more were found in various trash cans.

-------------------------------------------------------------------------------------------------------

 END OF FEEDBACK

I guess it is no wonder that the OneDrive App is removed in the early 10TP.  The Libraries are being revamped and have been deemphasized in 10. Even the Photo App has just started to reappear. I look forward to those features coming back. MS has already acknowledged as much on these forums. They will be a lot more robust. From what I've seen so far, feedback works. Windows 10 will be awesome.

jason (office engineer) say:

Captions will only show in grids if they are big enough.  You are using small images in the grids and they are set to subtle emphasis.  The only way to get the captions to show up is to make the grid bigger.  Try adding your captions back, selecting the grid UI on the storyline, clicking Options, and changing to Moderate.  That should get you captions on most screen sizes.
