That does not happen the way I have written this formula apparently. When I add a row to Sheet One the whole original 10 x 100 range shifts down one row instead of remaining anchored at the top and expanding down one row.

erica blair say:

Hi TheGnomon,

If I understand your problem correctly, your range in your formula is moving when you insert cells. This is inevitable with any hard-referenced range when you insert cells. The only way in your case to make your range grow is by inserting a row within the range, any inserts before will slide it down and after will be ignored.

What it looks like you need is an Indirect() function that generates a range for you, please try the following:

=COUNTIF(INDIRECT("Sheet One!$B2:$K"&COUNTA(B:B)),A36)

This formula relies on there being at least some data entered in all rows in column B on sheet one for which you want to search, if any blanks are present, either switch the B:B to another column that has no blanks (such as possibly column A) or instead try:

=COUNTIF(INDIRECT("Sheet One!$B2:$K"&MAX((A:K<>"")*(ROW(A:K)))),A36)

This will always use the range starting at 2 down to the highest numbered row for which any data exists in columns A:K.

NOTE: This second formula is an array formula and must be entered using ctrl+shift+enter!

Please let me know if either of these helped and if you need more assistance,

suzanne s. barnhill say:

If you saved the AutoText entry in the default location (that is, not in the Headers gallery), it should have been saved in the Normal template. There are two possible reasons for its not being saved:

  1. If you have "Prompt before saving Normal template" checked at File | Options | Advanced: Save, then, when you quit Word, you would have gotten a message box asking for permission to save the Normal template. If you said no, then it would not have been saved, and your AutoText entry would have been lost.
  2. What is much more likely (since I think the "Prompt" setting is not enabled by default) is that something is preventing the Normal template from being saved. Usually this is a poorly written add-in, and the ultimate solution is to find out what add-in it is and disable it (unless it's something you really, really need). As a temporary workaround, however, you can open the Normal template directly for editing and make your changes there.

There are two ways to open the Normal template for editing:

  1. If this is something you envision having to do often, you may as well figure out where the template is located. By default, the Normal template, Normal.dotm, is stored in the user templates folder at C:\Users\<Profile Name>\AppData\Roaming\Microsoft\Templates. You can navigate to that folder (I think that by default it's hidden, so you'll have to tell Windows you want to see Hidden and System files and folders) and open the template from there. You will probably want to add this folder to your Favorites to make it more accessible. Note that this is just the default folder; it's unlikely that you've changed it, but you can see the correct path at File | Options | Advanced: File Locations: User templates.
  2. For just a one-off change, you'll find this procedure simpler:

    In Word, press Alt+F11; the Visual Basic Editor opens. Press Ctrl+G to activate the Immediate window. In that window, type

         NormalTemplate.OpenAsDocument

    and press Enter. Normal opens, but note that it may not be on top of all windows. You can close the Visual Basic Editor. Make the desired changes to the Normal file, and then save and close it. If the changes you have made have not been recognized as changes (the Undo button is not activated), press Spacebar, then Backspace to "dirty" the file, thus ensuring that the template will actually be saved.

suzanne s. barnhill say:

What you don't want to do is put this information in the header of the Normal template (as that causes problems with labels). What you do want to do is create the header you want and save it as an AutoText entry, as follows:

  1. Double-click in the header area to open the header pane for editing. For your purposes, this is better than using any built-in building block (even the Blank one, which is usable but adds an extra empty paragraph).
  2. By default the empty header has two tab stops, a center-aligned one in the center and a right-aligned one at the right margin. These are "alignment tabs," which means that they will move to the center and right margin if you change the margins; you want this.
  3. You can insert tab characters to use these tab stops to place your FILENAME and DATE fields. For purposes of example, let's say you want the filename at the left margin, the date in the center, and a page number on the right.
  4. At the left margin, type "FILENAME \p" (without the quotation marks). Select what you have just typed and press Ctrl+F9 to convert it to a field.
  5. Press Tab once to get to the center-aligned tab stop, where you'll insert your date field. If you know how to use date fields (see http://WordFAQs.mvps.org/DateFields.htm) you can create this one manually as well, but you will probably want to use the Field dialog for this purpose.
  6. Use Insert | Text | Quick Parts | Field. In the Field dialog, select the type of field you want. It is unlikely that the Date field will be suitable; you'll probably want to choose CreateDate, SaveDate, or PrintDate. Select the date format you want to use and click OK to insert it.
  7. Press tab again to get to the right-aligned tab stop at the right margin.
  8. Press Alt+Shift+P to insert a PAGE field.
  9. When you have inserted all three fields, select the entire paragraph you have created and press Alt+F3 to open the Create New Building Block dialog.
  10. Unless you specifically want to save this as a Header building block, all of the default settings will be correct except for the name of the entry. Choose the name you want to use for the entry--for example, myheader.
  11. When you want to use this header, double-click in the header area to open the header pane, then type "myhe" and you will get an AutoComplete tip offering to insert the AutoText entry. Press Enter to insert it.
  12. If you have saved the header in the Headers gallery of built-in building blocks, then you can use Insert | Header and select it from the gallery. If you choose this route, you may want to use a name beginning with _ so that the header will sort to the top of the gallery.
