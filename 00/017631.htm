jay freedman say:

The first thing that I'd be suspicious of is the beginning of the macro, the statement RecentFiles(1).Open. You would have to be certain that the first file in the Recent Files list -- that is, the most recently opened file -- really is the one you mean to copy your tags from. If it's any other file, the copying would grab the wrong text, or possibly fail altogether if there is less text than expected. Instead, you should use the Documents.Open command with the name of the file (including the folder path if it isn't the default Documents folder).

I'll suggest, though, that you're using the wrong feature of Word for this, and you shouldn't use a macro at all. If you once store your tags in AutoText entries or AutoCorrect entries in your Normal.dotm template, they'll be available in all documents. Instead of going to the Office clipboard to get them for insertion in a document, you would type the "name" of a tag (which could be just a few characters). If the tags are AutoText entries, you would then press F3 to expand the name to the complete tag; if they're AutoCorrect entries, you need only press the space bar.

For information about AutoText, see the second half of  http://www.word.mvps.org/FAQs/Customization/AutoText.htm. For information about AutoCorrect, see http://wordfaqs.mvps.org/AutoCorrect.htm.
