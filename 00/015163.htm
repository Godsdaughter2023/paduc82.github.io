vijay a. verma say:
1. SUMPRODUCT Issue - Somehow, EOMONTH is not array friendly in the normal fashion. That is forcing you to create the formula in 3rd way. I had thought that good old way (when EOMONTH was not there) to derive last date of the month was dead. The old way was DATE(YEAR(A1),MONTH(A1)+1,1)-1.
It came to our rescue here. So, below will work which is definitely an improvement over 3rd one.

=SUMPRODUCT(--(A1:A3=DATE(YEAR(A1:A3),MONTH(A1:A3)+1,1)-1))

2. All characters have a Unicode associated. That can be known by UNICODE function. Hence, they can be easily translated (That's how I had made the challenge)

For ASCII (256 characters)
Excel functions = CHAR and CODE
VBA functions = CHR and ASC

For Unicode (65,536 characters)
Excel functions = UNICHAR and UNICODE
VBA functions = CHRW and ASCW

So, you need to extract the first and last char, convert through UNICODE to know the code and replace it through UNICHAR.

But in fact there is another solution. You need not take the route of UNICODE and UNICHAR.
