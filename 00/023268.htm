IMO the problem started with Onenote 2010 (my version, BTW).

Since this version MS splitted the Onenote cache (in the 2007 version only one file) into the "real" cache for text, ink etc., and a new picture cache. In Onenote 2010 its path is:

C:\Users\<user>\AppData\Local\Microsoft\OneNote\14.0\OneNoteOfflineCache_Files

If you substitute 14.0 by 15.0 you should find it. With all your pictures.

It's only a temporary split while a notebook is open. So there must be pointers from the real cache to the outsourced pictures. 

I think there are bugs in the handling of those pointers which result in a red X.

So it's not surprising, that in the most cases the red X disappear by closing and re-opening a notebook.
