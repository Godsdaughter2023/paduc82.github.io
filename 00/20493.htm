suzanne s. barnhill say:

"Mirror margins" and "Different odd and even" cannot be selected for a single section; they're all or nothing settings (unlinke "Different first page," which can be enabled for selected sections). Word's assumption is that, if you're doing to duplex the document, you're going to duplex all of it. There's no requirement that the mirrored margins be different, so you can certainly set them to the same on both sides, but if you are wanting a larger left margin on all pages, then that can't be done.

Since you're going to need two separate print runs, anyway, however, you can turn "Mirror margins" off after printing the duplex section.

john w. vinson say:

One way would be to open the table (not the form) in design view and remove the 0 that Access all-too-helpfully puts in the Default Value field. Leaving it blank will cause new records to contain a NULL (no value, undefined, don't know what's there) value rather than a zero. You would then probably also need to run an Update query updating the fields from 0 to NULL.

If it's just a matter of display - e.g. you want to type 0 into a field, or do a calculation that gives a zero, and have that result shown as a blank, you can use the rather obscure "four value format". Set the Format property of the form textbox to

"Currency";"Currency";"";""

to display (in order) positive values using your computer's current regional setting for currency values; negative values the same; zeros as an empty string; and NULL as an empty string. Each of the four formats can be customized; for example you could use

"$#,##0.00";"($#,##0.00[Red])";"No Balance";"UNKNOWN"

if you wanted to get fancy.

robert aldwinckle on forums say:

When I open and close IE tabs I'm not recognizing any clue about which process might be interfering with the IE lockup. 

Does this happen with all sessions?  If not, watch what happens with normal ones.  The connections are quickly dropped.  Usually after a few seconds you won't see anything there.   In any case, sort by Remote address.   If you see any there which are 127.0.0.1  that would be a clear sign that you were connected to a local proxy.   Then whatever was listening on that port would be your other target.  Note that you can monitor all tasks at once by clicking the box for the Image column header.  Then you can uncheck ones that you aren't interested in seeing listed.  You could also then sort the Listening Ports by Port to see if there were any there with 80.  Hmm... it may not be that easy.  I just used Fiddler.exe to check for more ideas and saw that in fact the listening address is shown as IPv4 Loopback and the Port does not have to be 80.  YMMV.

 

Good luck

Robert
