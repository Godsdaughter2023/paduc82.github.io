john w. vinson say:

Well, you don't WANT to append. You said so yourself!!!

Appending does just that: it adds a new record, with (in this case) a duplicate StudentID. You don't want a new record because you only want one record per studentID. It would be counterproductive to create duplicate studentID records and then do a second process to combine the duplicates!

What I think you want instead is an Update query to update the existing old record. If you want to just add the values from TheSourceTable into the blank fields in ExistingMaster, you can use the Update query John suggests, or do it all in one pass with a query like

UPDATE ExistingMaster INNER JOIN TheSourceTable
ON ExistingMaster.StudentID = TheSourceTable.StudentID
SET [ExistingMaster].[LastName] = NZ([ExistingMaster].[LastName], [TheSourceTable].[LastName]),
[ExistingMaster].[FirstName] = NZ([ExistingMaster].[FirstName], [TheSourceTable].[FirstName]),

<etc for all the fields that you want to update>

If a field already exists in ExistingMaster, the NZ() function will ensure that it gets updated to its already existing value rather than getting overwritten by the value from the new data. If you WANT the current value to be always overwritten, except not by a NULL value, reverse the logic:

SET [ExistingMaster].[LastName] = NZ([TheSourceTable].[LastName], [ExistingMaster].[LastName]),

You'll need to come up with some business decision about how to handle conflicts; if the existing record for StudentID 222222 has "Jane Doe" and the new table has "Bill Richardson", which do you want to trust?

When you say <MAPI> do you actually mean <IMAP>? (there are no email accounts configured using MAPI in Outlook 2019/365).

For IMAP accounts where the contact/calendar data is stored in folders with the added description of "This Computer Only", categories are nor supported. However, really not recommended that you maintain your default contact/calendar data in "This Computer Only" folders.

You don't need to configure the account as a POP account. Just need to add a new PST file to the profile (from Home ribbon --> New Items --> More Items --> Outlook Data File).

When the PST file is added, then go to File --> Account Settings --> Data Files tab --> select the newly added PST file --> set it as the default data file for the profile. This will now be the location where default contact/calendar folders will be stored.

After the above is done, copy your contact/calendar data from the IMAP "This Computer Only" folders to the corresponding folders in the PST folder tree.

As a FYI, reason you don't want to maintain data in "This Computer Only" folders is because if the IMAP account is ever removed from the profile, the OST file associated with that account will also be deleted and info in the "This Computer Only" folders will be lost. Similarly, if Outlook cannot find the OST file for the account, it will create a new one, sync the email data but the "This Computer Only" folders will be empty.

Is there a method to convert a mergefield to UPPER CASE?
macropod say:

Hi Jethro,

Select the mergefield and press Shift-F9. You should see something like {MERGEFIELD MyField}. Replace anything after 'MyField' with ' \* Upper', so that you end up with {MERGEFIELD MyField \* Upper}.
