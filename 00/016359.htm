suzanne s. barnhill say:

The Styles task pane should indicate, by a border, which style is in use. The exception would be when the style in use is not displayed in the task pane (because of an Options setting). Another problem is that, if the list contains more styles than can be displayed without scrolling, the task pane doesn't automatically scroll to display the current style. This is probably by design, as the scrolling might be distracting, but I find it very unsatisfactory.

If you want to be able to see what style is applied in any given paragraph, even without using the Styles task pane, you can add the classic Style dropdown to the QAT. In the Customize Quick Access Toolbar dialog, select Commands Not in the Ribbon and scroll down to the Style entry that shows a dropdown (when you mouse over it, it will say "StyleGalleryClassic." Select that and click Add>>.

The Style dropdown will show the same list of styles that you have opted to display in the Styles task pane, but the box on the QAT will always display the current style at the insertion point. The dropdown will show the styles formatted (even if you have chosen not to do so in the Styles task pane), but the box itself will show the style name as plain text.

On the minus side, the Style dropdown also displays all the table styles, which is a bit of a pain, but you can skip over them.

On the plus side, you can force the Style dropdown to display All Styles by pressing Shift while clicking the dropdown arrow.

macropod say:

With Word 2013’s introduction, non-breaking spaces inserted via Ctrl-Shift-Space (ASCII 160) will now expand/contract in justified text. They previously behaved as a fixed-width spaces as well. The new behaviour conforms to the CSS space rules. Given that the new paragraph justification can also lead to smaller spaces between words on some lines, the ASCII 160 behaviour could look odd with the larger fixed-width non-breaking spaces Word previously inserted.

For fixed-width non-breaking spaces you may now need to use one of the other non-breaking space characters (e.g. the Narrow No-Break Space (202F, Alt-X)). If you use a Narrow No-Break Space is that, it appears as a space ( ) without the . or º that appears for other spaces when the formatting display is turned on. You can assign the a keyboard shortcut to the Narrow No-Break Space directly through the Symbol dialog (or the Customize Keyboard dialog).

macropod say:

Try:

Sub SplitDoc()
Application.ScreenUpdating = False
Dim Rng As Range, DocSrc As Document, DocTgt As Document, StrNm As String
Dim Ext As String, FlPth As String, Tmplt As String, Fmt As Long
Set DocSrc = ActiveDocument
With DocSrc
  Ext = "." & Split(.Name, ".")(UBound(Split(.Name, ".")))
  FlPth = .Path & "\"
  Tmplt = .AttachedTemplate.FullName
  Fmt = .SaveFormat
  With .Range
    .InsertAfter vbCr & "///"
    With .Find
      .ClearFormatting
      .Replacement.ClearFormatting
      .Text = "[//]{3}*^13"
      .Replacement.Text = ""
      .Forward = True
      .Wrap = wdFindStop
      .Format = False
      .MatchWildcards = True
      .Execute
    End With
    Do While .Find.Found
      If .End = DocSrc.Range.End Then GoTo Finished
      Set Rng = .Duplicate
      With Rng
        .MoveEndUntil "///", wdForward
        StrNm = Split(Split(.Paragraphs.First.Range.Text, vbCr)(0), "///")(1)
        .Start = .Paragraphs.First.Range.End
        Set DocTgt = Documents.Add(Tmplt)
        With DocTgt
          .Range.FormattedText = Rng.FormattedText
          .SaveAs2 FileName:=FlPth & StrNm & Ext, Fileformat:=Fmt, AddToRecentFiles:=False
          .Close False
        End With
      End With
      .Collapse wdCollapseEnd
      .Find.Execute
    Loop
  End With
End With
Finished:
Undo: Set Rng = Nothing: Set DocTgt = Nothing: Set DocSrc = Nothing
Application.ScreenUpdating = True
End Sub

jim gordon mvp say:

Hi RWilson,

The reason for not being able to install on new hardware is software piracy control. If the installer would work on any other hardware other than for which it was originally installed, the installer would be a universal installer and would work for everyone everywhere and Microsoft would probably get only a few paying customers.

Going back to the 2011 version, there used to be a phone number you could call and if you were polite and convincing, you could coax the support person to give you a new product key. Those days are over, though, because beginning with Office 2016 you can get a subscription.

One of the important features of a subscription is that you can control your installations yourself. You get up to 5 concurrent installs with an Office 2016 subscription (more now in 2019). You can activate and deactivate your subscriptions yourself using a control panel and always have 5 instances of Office running on the devices you specify.

The stand-alone install is primarily designed for IT shops that want a version of Office that doesn't have changing features. Many corporate and education installs require that the organization provide support for the product instead of Microsoft (they get a price break for this), but these places do not want to have to keep up with changes.
