steve rindsberg say:
>> DPI is a measure of image resolution. Physical dimensions of an image can be calculated by division of its width/height in pixels by DPI.

Its width/height AS PRINTED OR DISPLAYED ON A SPECIFIC DEVICE, yes.  

Otherwise, no.  An image in a disk file is nothing more than an orderly collection of numbers that represent the color of each pixel ... each sample ... in the image.  A red pixel might be represented as the numbers 255, 0 and 0.   They have no physical size until displayed.

>> On the other hand, as far as I understood, Excel measures physical dimensions in points that in turn correspond to some number of pixels (but the relationship between points and pixels depends on system running Excel).

And on how large Excel is displaying the image.  The same image might be added to a small cell or might cover the entire spreadsheet.  The spreadsheet might be displayed in a small window or fill the computer's screen. The computer might be displaying the image on a 640x480 screen or on a 2048 by whatever screen.   Each of these variables will affect the number of pixels the same image might occupy when displayed.

>> What I want is to evaluate number of pixels in a point and vice versa from within VBA macro.

You can't.  It's not a fixed quantity.

You may be able to work out how many pixels are occupied by a given shape/image in Excel on a specific computer at a specific time.  

To do that, you'll need to work out the current Windows display resolution (via calls to WinAPI, or if you know the system you'll use, you can hardcode the values).  Then work out how much of the full display Excel is using and how much of Excel's space the image is using.

cristind say:

So I spent some time on this, and isolated the problem to outlook.com.  Basically, every time an email goes through outlook.com from gmail (as an SMTP relay), it would send a copy of the mail as expected.  7-10 minutes later, it would send additional emails which would correspond to the # of recipients on the mail.  Example ... 7 recipients, 7 copies of the mail would go out (to everyone).  If you only had one recipient, you don’t notice the issue.

I contacted Outlook.com support, Google Support, and O365 support.  We isolated to outlook.com (it couldn’t be reproduced on O365).  The issue is not on the Gmail end.

There is no “real” support for outlook.com.  They don’t talk to the public, apparently (neither does gmail to be fair - I talked to their g suite support).

BUT

The guys at O365 (ironically) found this article:

https://kinsta.com/knowledgebase/free-smtp-server/

Basically,you CAN get your email to go out with your custom email address WITHOUT using outlook.com.  You use the Gmail SMTP service.  You put in your desired custom domain name on the first screen, and on the 2nd screen (for authentication) you put in the GMail SMTP server and your Gmail user name/password.  I’ve tested it all day and it is working like a champ.

I was concerned that putting in my gmail info, it would label it with my gmail email address (which I didn’t want).  But it doesn’t. 

HTH.  Weird way of solving it, but happy I can stay on Gmail without moving my outlook.com custom domain account.
