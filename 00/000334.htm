gustavbrock say:

You can convert to a Variant holding a date value for date strings and Null for zero-strings, and reduce it a little:

VariantDate = CVDate(IIf([PSTNG_DATE]="00000000", Null, Format([PSTNG_DATE], "@@@@-@@-@@")))

Or:

VariantDate = IIf([PSTNG_DATE]="00000000", Null, CDate(Format([PSTNG_DATE], "@@@@-@@-@@")))

Or, as an expression for a textbox to which you can apply normal formatting of dates:

=CVDate(IIf([PSTNG_DATE]="00000000",Null,Format([PSTNG_DATE],"@@@@-@@-@@")))

neman syed say:

Yes, this is an old question, but it's still relevant. One answer, which works very effectively for Word's built-in PDFing, is to:

  1. Click File > Save As
  2. Set Save as Type > PDF
  3. Click Options > PDF Options > turn on ISO 19005-1 compliant (PDF/A)
  4. OK, Save

(Instructions from Word 2010 (Windows), but should be quite similar in current versions)

Upsides:

  • It works every time to preserve hyperlinks
  • It's native to Word - no extra software required, and it'll work on anyone's computer who's using Word (this can be really handy in corporate environments)

Downside: It can generate larger files compared to, say, Nitro PDF. My test two-page document had embedded fonts, one extra colour, and a few paragraph borders. 

  • Word: 3537 Kb
  • PDFed by Nitro: 96 Kb
  • PDFed by Word: 356 Kb

Yes, those are small numbers in 2017, and I haven't tested whether those ratios actually mean much, but just do a moment of testing in case you end up needing to PDF a large file. Not all your recipients are on desktop computers with unlimited bandwidth. :-)

Hope this helps!

That link provided by RickCP is for making general feedback, not for conversing with anyone at Microsoft directly to get a response, which typically isn't possible.

 

What you are asking is really; "Why didn't MSE detect whatever was creating the ads?" correct?  So what you need to undertand is that all antimalware applications from Microsoft have certain criteria, which if applications such as adware stay within the boundaries won't be classified and/or removed as malware by Microsoft programs such as MSE.

 

How Microsoft antimalware products identify potentially unwanted software

http://www.microsoft.com/security/portal/shared/objectivecriteria.aspx

 

So in this particular case, though the ads themselves may have been irritating and undesired, if you accepted the installation of the Freeze.com or another toolbar application which provided the ads and it otherwise stayed within the boundaries of the Objective Criteria at the URL above, MSE would not include detections to remove it.

 

From a search for Free Offers from Freeze.com I found that it is often incuded with screensavers and other "free" applications that you might easily have downloaded and installed.  In fact the top listed search result from Bing was actually an ad offering a screensavers.com download which is titled "Freeze.com Screen Savers", so obviously they must be considered to be following the above criteria.

 

The problem here is the difference between following the criteria, which are reasonable, and doing what the user wants.  Adware at its best has always ridden the fine line between these two often opposing choices.  While anti-adware applications often remove such "on-the-edge" applications, because that is their entire purpose, many anti-malware applications avoid this since it takes resources away from their core purpose of protecting against malware.

 

In other words, Microsoft isn't anti-advertising, since in fact that's a portion of it's own business and to begin randomly selecting potential competitor's products for removal without the strict criteria above would quickly start a war between them and those 3rd-party advertisers.

 

In fact, one of the above listed products found on your own PC, Babylon, is a perfect example of an item that at one point was detected by MSE and has since been removed from detection once the offending situation was corrected.
