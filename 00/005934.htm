dirk goldgar say:

CLng(date1)

Note that, if the date value in question includes a time, then CLng(date1) will round it up or down.  For example (in the Immediate Window):

    ?CLng(#2017-4-14#)
     42839
    ?CLng(#2017-4-14 11:59 AM#)
     42839

    ?CLng(#2017-4-14 12:00 PM#)
     42840

Internally, the Date data type is stored as a double-precision floating point number:

    ?CDbl(#2017-4-14#)
     42839
    ?CDbl(#2017-4-14 11:59 AM#)
     42839.4993055556
    ?CDbl(#2017-4-14 12:00 PM#)
     42839.5

If you want just the date portion of a date/time value that may contain a time as well, use the Fix() or Int() function to drop the time:

    ?CLng(Fix(#2017-4-14 12:00 PM#))
     42839

Fix() is better than Int() if your dates might occur before the "zero date", because Int() truncates downward where Fix truncates upward:

    ?Int(#1800-4-14 12:00 PM#)
    4/13/1800
    ?Fix(#1800-4-14 12:00 PM#)
    4/14/1800
