lemp say:

What is the name and version of your antivirus program?

What are the names of any other "security" programs you have installed?

Here's a little background about what I believe is going on with your system.

Whenever a printer is used as a network device (and this is what a "wireless" printer means), the printer must be assigned an IP address on your local network so that the computer (which is also on your local network) can find it. 

By default, any Windows computer or Windows-network-compatible device is configured to obtain its IP address from the router that is at the center of the network.  For computers and mobile phones, this setup works fine.

When a network printer is installed on a Windows computer, part of the installation process configures the computer's printing system to direct its output to the IP address assigned to the printer. 

When the router assigns an IP address, that address is valid for a specified period of time (the "lease" period) which usually is at least a day but can be longer or shorter.  If the network device (e.g., the printer) is disconnected from the router (e.g., because the printer is powered off) for longer than the lease period, the router may reassign that IP address to the next network device that asks for an address.  When the printer finally reconnects, the router assigns a new IP address.  The problem now is that the Windows printing system is still directing print output to the old IP address -- so nothing prints.

Knowing that this situation is relatively common, HP developed something it calls the "HP network re-discovery port monitor," which sweeps the network looking for your printer (the Port name is the printer's serial number) so that in the event the printer's IP address has changed, the print job will be sent to the new, correct address instead of the old, incorrect address.

Sounds good, right?

The problem is that the HP network re-discovery port monitor may be blocked by security software on your computer (Windows firewall or a third-party security program) or it sometimes may just fail to work properly.

My suggestion is to get rid of HP's "solution" and do things the old-fashioned way, which is known to work.  There are two parts to this method.  First, assign a "static" IP address to the printer (this means that the same IP address is always assigned to the printer), and second, configure Windows to print to the static IP address.

A.  Assign a Static IP Address to the Printer

There are two parts to this step. 

First, you need to find out what IP addresses are valid for your network.  To do this, you will need to access your router's web configuration utility.  You would be looking for the range of IP addresses used by the router.  See the screenshot below.  If you don't know how to do this for your router, please provide the make and model as shown on the label on the back or bottom of the router and I'll try to find the router's User Guide and give you step by step instructions.

There are two constraints for selecting a static IP address.  It must be on the same subnet (for virtually all home routers, this means that the first 3 "octets" must be the same; in this example, 192.168.1.xxx) and it must not be in the range that might be used by the router.  The largest number that can be used for the last "octet" is 254.  Thus, for this router configuration, a static IP address could be 192.168.1.2 through 192.168.1.99 or 192.168.1.151 through 192.168.1.254.  If you look at the "Ports" tab screenshot in my earlier post, you'll see two static IP addresses assigned to two HP 6988 printers:  192.168.1.25 and 192.168.1.26.

Now that you've selected a static IP address, you need to configure the printer to use it.  Your printer's User Guide can be found at http://h10032.www1.hp.com/ctg/Manual/c05050200  The process is described at pages 104-106:

To change IP settings
The default IP setting for the IP settings is Automatic, which sets the IP settings automatically. If you are an advanced user and want to use different settings (such as the IP address, subnet mask, or the default
gateway), you can change them manually.
TIP: Before setting a manual IP address, it is recommended to first connect the printer to the network using Automatic mode. This allows the router to configure the printer with other needed settings like subnet mask, default gateway and DNS address.
CAUTION: Be careful when manually assigning an IP address. If you enter an invalid IP address during the installation, your network components cannot connect with the printer.
1. From the printer control panel, touch or swipe down the tab at the top of the screen to open the
Dashboard, touch ( Setup ), and then touch Network Setup.
2. Touch Ethernet Settings or Wireless Settings.
3. Touch Advanced Settings.
4. Touch IP Settings.
A message appears warning that changing the IP address removes the printer from the network. Touch OK to continue.
5. To change the settings manually, touch Manual (Static), and then enter the appropriate information for the following settings:

          ● IP Address
          ● Subnet Mask
          ● Default Gateway
          ● DNS Address
6. Enter your changes, and then touch Done .
7. Touch OK .

Change ONLY the IP address.  Leave all the other settings as they are.

B.  Configure Windows To Print To The Static IP Address

  • Go to the "Ports" screen as described in my earlier post.
  • Click "Add Port ..."
  • Select "Standard TCP/Port" and then "New Port"
  • Complete the "Add Standard TCP/IP Port Wizard.  When you get to the screen to enter the printer name or IP address, type the static IP address you selected into the box for "Printer name or IP address."  The bottom box will automatically fill with the same

    At this point, you should be able to print successfully.  You can start by clicking the "Print Test Page" button on the "General" tab of the printer properties screen.
