hansv mvp say:

Microsoft Access supports the use of macros and VBA code. In theory, it is possible to create a "macro virus" in an Access database, but in more than 15 years of working with Access, I have never actually seen one. If you take reasonable precautions to restrict who can open the database, it is extremely unlikely that you'll be hit by a macro virus.

 

There is an inherent risk in Access, however: it requires that all users have full permissions (Read, Write, Create, Delete) to the folder that contains the database (both for the frontend and for the backend, if you use a split database design). That means that any user can delete the database by accident or intentionally if it is not in use. So backups are essential.

 

Although Access can handle tables with millions of records, executing queries on the data may well become slow. It might be a better idea to use Access as a frontend for a SQL Server database. SQL Server is more secure than Access, it can handle larger databases and it supports many simultaneous users. (Access works best with a limited number of simultaneous users, say up to 10.)

shane devenshire say:

I suppose the common thought would be a range that can change without you having to manually change it.  The advantage is it is automatic. 

=OFFSET(Sheet1!$A$1,COUNTA(Sheet1!$A$2:$A$1000),,-24)
=OFFSET(Sheet1!$B$1,COUNTA(Sheet1!$A$2:$A$1000),,-24)
If these are given name like X and Y then you can change:

=SERIES(Sheet1!$B$1,Sheet1!$A$2:$A$50,Sheet1!$B$2:$B$50,1)

to this:
 
=SERIES(Sheet1!$B$1,Sheet1!X,Sheet1!Y,1)

In which case the chart which is being plotted can automatically adjust as more data is added or removed from the range.

  4. Select the default Delete group and click Remove. The group you added should be selected.
  5. Select Main Tabs from the Choose commands from field on the left.
  6. Expand Home (Mail) then expand the Delete group (on the left).
