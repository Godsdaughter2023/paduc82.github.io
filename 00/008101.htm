    Is there something something I need to do to the database or do I always have to change the shortcut?

    Thanks in advance.

    tom van stiphout say:

    It sounds like you have two versions of msaccess.exe on your machine: one is a runtime version (limited options) and one is a full version.

    There is a way to tell Windows you want .mdb files to open with the second one. Search the web for "Windows file association".

    bully319 say:

    Here's what I've discovered.

    Microsoft Office 2016 C2R(Click-to-Run) is not compatible with Access 2016 Runtime MSI.  In order to use an OLEDB with Office 2016 C2R installed, you need to install Access 2013 Runtime MSI. https://blogs.office.com/2015/12/16/access-2016-runtime-is-now-available-for-download/   

    Once you've install the Runtime 2013, the shortcuts refer to the Runtime and you can not change the file type associations back to Office 2016.  The only way you can use a shortcut to open an Access database using Office 2016 is to use the complete path stated in the original post.

    Fortunately, I have found a solution.  If you install the Access Runtime first and then install Office 2016 C2R, then the shortcuts will work and there will be less conflicts.  The original problem had other side-effects besides the shortcuts.
