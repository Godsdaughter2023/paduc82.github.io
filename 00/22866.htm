This is what I did to resolve the issue. Other people have the same problem. Refer to:

How to temporarily prevent a driver update from reinstalling in Windows 10

and

How to Hide Updates in Windows 10 (Basic Method) - Microsoft Community

Download the file   wushowhide.diagcab   mentioned in the first link above. Do not execute the file since update information is cached in the system and must be reset, first.

1) Turn off the "Windows Update" service and disable the service so it does not start when rebooted.

2) Reboot

3) Delete the folder \windows\SoftwareDistribution

4) Attempt windows update manually and an error will occur, validating the service is not running. Also if it was running, you would not be able to delete the folder.

5) Reenable the "Windows Update" service setting it to manual and start it.

6) Quickly execute the file wushowhide.diagcab that was downloaded. Do this quickly to make sure the operating system does not initiate the update process first.

7) Allow the tool to search for updates. When presented with a list of available updates, place a checkmark next to the one giving the problem. In my case it was KB2267602.

8) Allow the tool to finish and close it.

9) Attempt windows update manually and there will be no issues.

Every once in a while when thinking of it, you might run the tool again, unhide the problem update, then manually invoke a windows update. Maybe the problem will have gone away by then.

It is not good concealing a problem this way, but the problem is a nuisance, and the failed update may be preventing other updates from being installed.
