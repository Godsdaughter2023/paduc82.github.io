That's a workaround.  There can be more than one instance of explorer.exe when you open a File Explorer window.  The one you want doesn't have any command line arguments.  If you open another drive or if you open a File Explorer window before you have a Desktop (e.g. after you kill it) then it will be a separate process having a command line.

I can demonstrate this with wmic but haven't found equivalent PowerShell syntax yet.

wmic.exe  Process  where  Name="explorer.exe"  Get  CommandLIne, ProcessId

Public Const SW_SHOWMINIMIZED = 2
 
Public Sub MySub()
    'LATE BINDING TEST 1
    Dim objWord As Object
    Dim blnWordWasOpen As Boolean
    Dim hWnd As Long
   
    On Error Resume Next
   
    'Get a reference to Word if it's open
    Set objWord = GetObject(, "Word.Application")
 
    'If Word isn't open, create an instance of it
    If (Err <> 0) Then
        Set objWord = CreateObject("Word.Application")
    Else
        blnWordWasOpen = True
    End If
   
    objWord.Visible = True
   
    'Get a handle to Word
    hWnd = FindWindow("OpusApp", vbNullString)
   
    'Maximise it
    ShowWindow hWnd, SW_SHOWMAXIMIZED
 
    'Do whatever you want
   
    If Not blnWordWasOpen Then objWord.Quit 0
    Set objWord = Nothing
End Sub
