ken sheridan say: This is not an expression, it's the complete query; just paste it straight into the query in SQL view and terminate it with a semi colon. I notice that you have no space before the WHERE clause however, and table or column names which include spaces or other special characters must be wrapped in square brackets.  To make the query more readable you can use carriage returns and indenting like so:

SELECT [CCB_HSHLD_CLI_ID], [Current Age]
FROM [NCWest_Members with Hshld ID+]
WHERE [Current Age] =
    (SELECT MAX([Current Age])
     FROM [NCWest_Members with Hshld ID+] AS T2
     WHERE T2.[CCB_HSHLD_CLI_ID] =
     [NCWest_Members with Hshld ID+].[CCB_HSHLD_CLI_ID]);

However, the main point I'd make is that a person's current age as a value in a column will soon become out of date.  If you store their date of birth on the other hand you can compute their current age at any time.  the following link has a couple of methods:

http://www.mvps.org/access/datetime/date0001.htm

But I generally use this simple function:

Public Function GetAge(varDoB As Variant, Optional varAgeAt As Variant) As Variant

    If IsMissing(varAgeAt) Then varAgeAt = VBA.Date
    
    GetAge = DateDiff("yyyy", varDoB, varAgeAt) - _
        IIf(Format(varAgeAt, "mmdd") < Format(varDoB, "mmdd"), 1, 0)

End Function

Paste it into a standard module in your database and you can call it in a query if you have a DoB column of date/time data type rather than your Current Age column, e.g.

SELECT [CCB_HSHLD_CLI_ID], GetAge([DoB]) AS [Current Age]
FROM [NCWest_Members with Hshld ID+]
WHERE [DoB] =
    (SELECT MIN([DoB])
     FROM [NCWest_Members with Hshld ID+] AS T2

darlenez say:
This is not an expression, it's the complete query; just paste it straight into the query in SQL view and terminate it with a semi colon. I notice that you have no space before the WHERE clause however, and table or column names which include spaces or other special characters must be wrapped in square brackets.  To make the query more readable you can use carriage returns and indenting like so:

SELECT [CCB_HSHLD_CLI_ID], [Current Age]
FROM [NCWest_Members with Hshld ID+]
WHERE [Current Age] =
    (SELECT MAX([Current Age])
     FROM [NCWest_Members with Hshld ID+] AS T2
     WHERE T2.[CCB_HSHLD_CLI_ID] =
     [NCWest_Members with Hshld ID+].[CCB_HSHLD_CLI_ID]);

However, the main point I'd make is that a person's current age as a value in a column will soon become out of date.  If you store their date of birth on the other hand you can compute their current age at any time.  the following link has a couple of methods:

http://www.mvps.org/access/datetime/date0001.htm

But I generally use this simple function:

Public Function GetAge(varDoB As Variant, Optional varAgeAt As Variant) As Variant

    If IsMissing(varAgeAt) Then varAgeAt = VBA.Date
    
    GetAge = DateDiff("yyyy", varDoB, varAgeAt) - _
        IIf(Format(varAgeAt, "mmdd") < Format(varDoB, "mmdd"), 1, 0)

End Function

Paste it into a standard module in your database and you can call it in a query if you have a DoB column of date/time data type rather than your Current Age column, e.g.

SELECT [CCB_HSHLD_CLI_ID], GetAge([DoB]) AS [Current Age]
FROM [NCWest_Members with Hshld ID+]
WHERE [DoB] =
    (SELECT MIN([DoB])
     FROM [NCWest_Members with Hshld ID+] AS T2
     WHERE T2.[CCB_HSHLD_CLI_ID] =
     [NCWest_Members with Hshld ID+].[CCB_HSHLD_CLI_ID]);

Reminders at midnight can (and are) a huge pain in the butt and have been a source for complaint for a long time.  Some folks like to get a reminder in advance, but don't want it to go off at midnight, so they set the reminder for a certain number of hours.  For example, if you want the reminder to come a week ahead of schedule then use 156 hours (6.5 days).  That setting would remind you at noon, 7 days before the event.
