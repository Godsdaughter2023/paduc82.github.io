How can I do this with project 2010? What task type would be most appropriate? Moreover, how can I consider focus factors for resources?

dale howard [mvp] say:

Kurt --

OK, now that I know what you want to do, follow these steps instead:

  1. Right-click anywhere in the Gantt Chart pane (on the right side of the Gantt Chart view) and select the Show Split item on the shortcut menu.
  2. Select a task on which you want to assign a resource who will expend limited effort (work).
  3. In the Task Form pane at the bottom, select the resources who will work on the task, enter a Units value for each resource indicating the focus factor value (like 50% that you noted in your last post), then enter a Work value for each resource as well.
  4. In the Task Form pane, click the OK button.

Using the new set of steps, you will not need to change the Task Type and Microsoft Project will automatically calculate the Duration value for you.  Frankly, the method I am giving you in this post is the method I prefer to use personally.  On a Fixed Units value, if you provide the Units value and the Work value, Microsoft Project will calculate the Duration value for you.  Hope this helps.

How to configure IMAP in an existing profile in Outlook 2013 - Microsoft Community

fredbk say:

How to configure IMAP in an existing profile in Outlook 2013

I would like to switch from using POP3 to IMAP in Outlook 2013. Can someone please explain what happens when an IMAP connection is setup in an existing profile. That is: what files are created and how are previous pst files involved?
karl timmermans [outlook mvp 2012-2018] say:

Scenario #1 - POP first - Imap second

- this will create a PST file for the POP account (or use an existing one if that's what you've selected) and becomes the 'default" location for your contacts/calendar items

- an OST file will be created automatically when configuring the Imap account and the PST file assigned to the POP account will contain your contacts/calendar items etc

>Since Outlook is currently setup to use POP3 I think this is the applicable scenario.

>What folders are created on the IMAP server?

>What, if any, synchronization is there between the PST file and the OST file?

>I have 3 mail accounts setup all using the same PST file. Is there an OST file setup for each account?

>How do the OST folders get marked as "This Computer Only"?

>Since the contacts/calendar, etc. are in the PST file how can I sync them between different computers, other than a third party program?

>I have rules setup to automatically move incoming files to other folders in the PST file. Can I use similar rules when using IMAP?

>Thank you for your quick response to my question.

Whether you use your existing profile to add your POP accounts as Imap or create a completely new profile configuring only the Imap accounts from scratch is your choice. What happens depends on what sequence you follow - a lot of different permutations and combinations. One thing to keep in mind, regardless of which way you approach it - does not impact the data located on your server so you can't damage anything. POP3 / Imap is purely a "protocol" used to access the data on the email server.

#1 - Nothing gets "created" on the server by virtue of using the Imap protocol - what email folders get included will depend on the email provider (for example - Imap GMail will have a root folder called "GMail" with sub-folders located underneath it. You can create additional folders but keep in mind that these will never be accessible when you access the account via POP3 which is exclusive to the <Inbox>.

#2 - There's no synchronization between the PST and OST

#3 - Yes there is one OST file created for each Imap account

#4 - The one folder that gets created all the time with "This Computer Only" is the <Sync Issues> folder. Whether or not the other folders (Calendar, Contacts, Drafts, Journal, Notes, Tasks) get created depends on whether or not you have another InfoStore (i.e. PST file) designated as the "default" data file for the profile. The "default" data file is the default location for these item types.

#5 - Folders with "This Computer Only" are automatically created (if relevant/required) when you first configure the account - not under your control. Keep one thing in mind - an OST file is specific to the email account/profile which created it. If you recreate the email account - a new OST file will be created and any data in folders marked with "This Computer Only" will be lost. You can't open an .OST file on its own as you can with PST files.

#6 - If you want to sync non email folders between computers/devices - you will need to use an email account type (Exchange ActiveSync or full Exchange). Outlook itself does not sync <sideways> (I.e. computer to computer - it syncs "vertically" (i.e. with the email account on the server). Since I don't try to sync data "sideways" - have no idea what 3rd party options exist or how well those would work.

#7 - Rules work the same way

Just to make things easier - if you post any follow up questions using "reply with quote" - please put those questions outside of the quoted area (you can also delete whatever is not relevant in the quoted area) - makes things a lot easier and faster to deal with.

karl timmermans [outlook mvp 2012-2018] say:
I would like to switch from using POP3 to IMAP in Outlook 2013. Can someone please explain what happens when an IMAP connection is setup in an existing profile. That is: what files are created and how are previous pst files involved?

What happens depends completely on the sequence of events that are followed when configuring the accounts.

For the following description - these assumptions apply

- this description is exclusively for Outlook '2013

- this is in reference to a new Outlook '2013 installation - not an upgrade of a previous version

- this only references one email account (may not make a difference depending on scenario)

Scenario #1 - POP first - Imap second

- this will create a PST file for the POP account (or use an existing one if that's what you've selected) and becomes the 'default" location for your contacts/calendar items

- an OST file will be created automatically when configuring the Imap account and the PST file assigned to the POP account will contain your contacts/calendar items etc

Scenario #2 - Imap first - POP second

- this will create an .OST file for the Imap account which becomes the "default" infostore for the profile and contain the local folders for contacts/calendar (anything in the folder tree with "This computer Only" in the folder description

- a PST file will be created for the POP account (unless you assign one that already exists). If a new PST file is created - it will not have the contacts/calendar folders. The folders contained in the .OST file will be utilized as the default

*** caveat - any data in OST folders with that includes "This Computer Only" in the folder description will be lost if you have to recreate the Imap account and you don't have a backup of this data. OST files can only be opened in the original profile containing the account that created it. (A serious oversight in my mind by the designers when they made this change and am presuming that they were thinking of Exchange accounts which would contain this info at the server level)

You just insert a tab character from the keyboard. You'll need IF fields, however, to prevent printing the label data if the field is empty. Something like this:

{ IF { MERGEFIELD Productnumber } <> "" "Product Nr.:<tab>{ MERGEFIELD Productnumber}" }

where <tab> represents a tab character.

FWIW, I find "Product Nr.:" very ugly. The best solution would be to spell out "Number." If you want to abbreviate, the standard abbreviation for "Number" is "No." But to avoid the period followed by a colon, you might use # instead.
