rdhw say:

I eventually fixed this particular problem (but then ran into another which prevented the upgrade, but that's another story).

Having exhausted all possibilities concerning plug-and-play drivers associated with physical devices, in the existing Vista installation, I used Process Explorer to investigate what other non-plug-and-play drivers/services were being loaded.  I discovered a handful of ancient (XP-era) drivers being loaded at boot-time, despite no device needing them, as Vista had in-box drivers for the entire system.  The most suspicious of this handful were nvatabus.sys and nvata.sys, which were the IDE/SATA drivers supplied by nVidia to enable XP to drive the disks.  Vista uses the in-box driver pciide.sys instead.  These drivers should never have been migrated from XP to Vista in the first place.  Worse, the Windows 7 installer was attempting to migrate them to Windows 7 as well, and nvatabus was even being migrated in C:\Windows\System32\DriverStore\FileRepository.

In the end, I audited all oem*.inf files in C:\Windows\inf to determine whether each one referred to some driver for which Vista had a more up-to-date in-box driver.  If the oem*.inf file was definitely not required any more, I executed pnputil -d on it, to make quite sure that it was not available for migration to Windows 7.

If there were drivers being loaded at boot-time for out-of-date .sys files, I executed sc delete on them.  There were things like admtools.sys left over from long-ago uninstalled AMD processor monitoring utilities.

As each upgrade cycle, blue-screen, and rollback took over two hours, I wasn't very scientific about testing these suspects one by one: I just got rid of the whole lot in one go, and was mightily relieved to see the Windows 7 upgrade then progress past the driver installation stage.  So we shall never know for sure which of the outdated boot-time drivers was the culprit causing the blue-screen.

My relief was short-lived, as the upgrade program subseqently hit a FatalError and rolled-back again: I have started a new thread on that issue at:
http://social.answers.microsoft.com/Forums/en-US/w7install/thread/3e8d10a0-880b-4f6a-b43e-172a511f89fc

Thanks for listening, and for the ideas/input.
