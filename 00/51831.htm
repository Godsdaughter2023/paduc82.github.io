The printer manufacturer (HP) is not likely to help you in this situation.

Microsoft used to have an automated Fix it routine that reset the printing subsystem to the state it is in when Windows is first installed.  Rather unhelpfully, Microsoft has "retired" that Fix it, so you'll have to attempt to do that manually. 

Before you do that, however, see if the following procedure helps.

A.  Clean out anypending print jobs

  • Open a Command Prompt window (Start > Run > cmd > OK)
  • Type the following in the black Command Prompt window and press Enter after each line

                           net stop spooler
                           del /Q "%windir%\system32\spool\PRINTERS\*.*"
                           net start spooler
                           exit

B. Clean out the corrupt printer driver

    
  • Uninstall printer-related software from Control Panel > Programs and Features
  • Open Devices and Printers, right-click on the offending printer, and select "Remove device"
  • Open an Elevated Command Prompt (search for "cmd.exe"; right-click and select "Run as administrator")
  • Type the following in the black Command Prompt window and press Enter

            printui /s /t2
            exit
  • Select the driver corresponding the the problem printer and remove both the driver and the driver package.

C.  Reinstall the printer using a freshly-download file from HP.

  • http://ftp.hp.com/pub/softlib/software11/COL30777/mp-76232-1/AIO_CDA_Net_Full_Win_WW_130_140.exe  (290 MB, so I hope you have a fast Internet connection!)
  • Follow the installation directions here --> http://h10032.www1.hp.com/ctg/Manual/c00737614

Reboot and see if the printer works properly now.

If that didn't help, you'll have to take more drastic measures. 

The description of what the former Fix it routine did is here --> https://blogs.technet.microsoft.com/askperf/2012/02/24/microsoft-fixit-for-printing/  The references to Vista in the article are also valid for Windows 7.

As you'll see, there are numerous registry modifications, unfortunately without much detail.  Before you do ANY registry modification, make a System Restore point or otherwise back up the registry.

From a little Internet research, the specific error you're experiencing appears to be related to the following step under "Full mode," so I suggest that you try it first, without attempting the registry modifications.  Removing the driver as described above should have been sufficient, but perhaps there's some other file in this folder that's causing the problem.

Vista: delete all files under “%windir%\System32\spool\drivers\<arch>\3” except for driver files associated with “Microsoft XPS Document Writer” and “Microsoft Shared Fax Driver”.  Ignore subdirectories.

If you have a 64-bit version of Windows 7, <arch> is x64.  If you have a 32-bit version, it is W32X86.

Because you probably won't be able to determine which files in that directory are those you should keep, what I suggest is that you copy the entire folder to some other location and then delete all files in the original folder with names beginning with "HP".

If Windows balks at letting you delete files in this folder, stop the spooler service as shown in Part A above.  If that doesn't work, reboot to Safe Mode.

If after all of the above you still have the problem, post back and I will attempt to determine what the specific registry modifications are.
