-- Check if your Outlook installation is up-to-date. If not, update it by installing the latest service pack.

-- Remove stored credentials and force Outlook to use your Windows desktop credentials, follow these steps.

1) Click Start, click Control Panel, and then click Credential Manager.

2) Locate the set of credentials that has Outlook in the name.
3) Click the name to expand the set of credentials, and then click Remove from Vault.
4) Repeat step 3 for any additional sets of credentials that have the word Outlook in the name.

-- Manually reset the login settings for Outlook through Windows Credentials Manager

1) Go to Control Panel -> Credentials Manager -> Account and Family Safety
2) Look for the set of credentials that has “MS Outlook” (or something similar) as its name for mail server
3) Click on the name to expand the set of credentials and then click on “Remove from vaults”
4) Repeat above steps for all the set of credentials that start with word “Outlook”
5) Close the Credentials Manager and Control Panel
6) Launch Outlook

-- Enable the “Remember Password” Option

1) Launch Outlook and go to File Tab -> Account Settings

2) Select the Account under the Email Tab

3) At the bottom of the window that appears, you will see a “Remember Password” option. Make sure this option is selected

Be accurate with changes registry.

Using Task Scheduler ( XP ) to do daily updates - Microsoft Community

falcon04 say:

Using Task Scheduler ( XP ) to do daily updates

Is there a way to run the MSE .exe file via "Scheduled Tasks" to do a scheduled periodic update ? I did this for AVG to beat the once a day limitation an also for the weekly Spybot S&D update for immunization files update.
greginmich say:

Yes, you can schedule a daily update with Task Scheduler to do extra update checks. In the Wizard, select “Microsoft Security Essentials”, and then replace“msseces.exe” with the command line for a signature update: MpCmdRun.exe -SignatureUpdate

 

So the paths will be:

Run: C:\PROGRA~1\MICROS~2\MpCmdRun.exe -SignatureUpdate

Start in:C:\PROGRA~1\MICROS~2

 

The update activity will be logged in C:\Documents and Settings\<Account Name>\Local Settings\Temp\MpCmdRun.log

 

The command line syntax is presented here:http://social.answers.microsoft.com/Forums/en-US/msescan/thread/809b0b84-9396-49b2-95c4-68e7ec539fc7

 

 

 

Update for Version 2 (Edit):

 

The method presented above was a shortcut intended to avoid the need for following a file path in setting up the update task in the Windows XP “Scheduled Task Wizard”. This method won’t work with Version 2 of Security Essentials, because the Command Line Utility (MpCmdRun.exe) and the user interface (msseces.exe) are no longer in the same folder. In the XP Scheduled Task Wizard, you will now need to browse to MpCmdRun.exe, and then add the command line for a signature update. In the Windows 7 “Create Basic Task Wizard”, you can copy and paste the "Program/script" and "Add arguments" lines.

 

 

In Windows XP, the Task Scheduler "Run" line for a signature update would now be:

"C:\Program Files\Microsoft Security Client\Antimalware\MpCmdRun.exe" -SignatureUpdate

 

And the "Start in" path would be:

"C:\Program Files\Microsoft Security Client\Antimalware"

 

 

In Windows 7, the Task Scheduler "Program/script" path would be:

"C:\Program Files\Microsoft Security Client\Antimalware\MpCmdRun.exe"

 

And the "Add arguments" command line for a signature update would be:

-SignatureUpdate

 

So the "Start a program" action line would be:

"C:\Program Files\Microsoft Security Client\Antimalware\MpCmdRun.exe" -SignatureUpdate

 

GreginMich
