Bullet formats won't apply - Microsoft Community

a. user say:

Bullet formats won't apply

This new document will not allow me to apply multilevel numbering. I've tried so many ways, to add/edit/clear/remove; the doc keeps defaulting to list paragraph, too, which I don't want it to do.

Suggestions?

I've used Word for years. This is making me insanely frustrated.

john mcghie say:

Hi Melissa:

It's not that bad, but you do have to understand how it works to get it to be reliable.

Sadly, Shauna died before she could really address List Styles, but List Styles were the mechanism created to stop this rot.

In Word, you have a bunch of paragraphs.

Those paragraphs are formatted by different paragraph styles.  Styles are stored in the document itself, unless you add them to a template. Styles will not be copied with the text into a document that already has styles of the same name.  So the first key to this is to always "Add to template" each time you change a style.

You will never get this reliable if all of the paragraphs have the same style.  So use the built-in styles List Number through List Number 9. Customise their formatting as you like, and make sure you check the Add to template button to save the styles back to your Normal template.

If you miss that step, your customised styles are not there to be copied, when you create a new document.

Then you apply numbering by choosing the model ("List Template") closest to the one you want, from the Outline Numbering button.

But if you want to copy reliably between documents, you instead hit "Define new List Style".  Again, check the "Add to template" button or it won't be there to copy.

Give it a name, then click the Format button and choose "Numbering".

Choose the model closest to the one you want and click CUSTOMISE.

Now here is the SECRET...

At the bottom left of the Customise dialog there is a tiny un-labelled downwards-pointing arrow.  Click it to reveal the "Link level to style" field.

For each of the Levels you want to use (in the long vertical box in the top left of the dialog) choose a Style Name in the bottom right in "Link level to style". You do not have to do all nine levels, but do one more than you think you will need.

You can use each style name only ONCE, and you CANNOT use that style name for any other purpose. I recommend the built-in styles List Number to List Number 9 -- that's what they are there for.  If you need more than one list in a document, you can create more paragraph styles at will.

Once you OK your way out of there, the List Style will be stored in your template.  When you create a new document, it will be copied into your new document and available for use. When you use it, your numbering definition will "just work" reliably in every document.

Later, you can change the "text" formatting by updating the paragraph styles.  You can change the "numbering" formatting by updating the list style.  If you make a change, don't forget to "Add to template".

To USE a List Style, you simply apply any of the paragraph styles it contains to a paragraph.  You never need to apply, or even think about, a list style (unless you need to change it).  The list style sits in the background and applies the correct number formatting (or bullet formatting...) to the paragraphs with its paragraph styles. It exists purely to enable numbering to be copied reliably between documents.

The List Styles mechanism was invented to respond to exactly the frustration you are writing about.  Sadly, the designer of the feature was not a Word user (they thought they were... everyone 'thinks' they are a Word user, but a startlingly small proportion of the world's people actually do know how to use Word...).  So the design is a real mess.  It's on the list for fixing, but the fix is large and expensive, and there are more important things to get done.

The core of your problem is that while styles can be copied between documents, "Lists" cannot.  Each numbered format in a document (there can be several hundred) is required to be a unique part of the document code. If you try to paste one in to a new document, you corrupt the document so Word can't read any of it...  The designer did not understand Word...

What a List Style does is store the INSTRUCTIONS for making a list.  When it gets to a new document, the List Style creates a new list; but one that has the formatting you designed.  It's a terrible Heath-Robinson botch-up, but it works.

The key to it is that you must associate paragraph styles with the list style.  That's optional, but it won't work reliably without them. And you must not use those paragraph styles for anything else (if you do, Word removes them from the list style).

Add what I just said to what Shauna said, and you will be one of the world's experts in Word numbering.  Disregard the differences between PC Word and Mac Word: in the near future, those differences will disappear.  Currently in Mac Word we are running a copy of the PC Word code.  It updates whenever PC Word does.  There is now basically only one team writing Word, working on the same code.  Currently, quite a few bits are switched off on the Mac because they have not yet been adjusted to work reliably. But they are steadily being switched on as they get them to work...  Sorry:  It's a big job, so it will be a year or so yet before we have everything.  But eventually we will get it all...

Hope this helps
