How do I use hlookup and vlookup together to find specific value in a - Microsoft Community

treytucker say:

How do I use hlookup and vlookup together to find specific value in a table

Using a table with column headings of 10, 20 or 30 years and row headings of 4%, 5%, 6%, 7% or 8%; with dollar values in the field of the table and wanting to enter those variables and find the specific dollar amount that corresponds to the length of time and percentage.  Which function or combination of functions would be best to use?

benishiryo say: hi TreyTucker.  say your column headers are in B1:D1 (10, 20, 30) & row headers in A2:A6 (4%-8%).  if the years you are looking for is in F1 & the % in G1, then MATCH gives you the position of where F1 or G1 is at.  Say F1 is 20 & G1 is 6%
=MATCH(F1,B1:D1,0)

the above would return 2 because it's the 2nd position from B1:D1.  the 0 at the end indicates i want an exact match.  and that brings to another question.  is there a case where you want something in between?  like if F1 is 15?

for G1, i can also use:
=MATCH(G1,A2:A6,0)
so that will give me 3, since it's the 3rd position from A2:A6

you can now use INDEX.  notice if you type INDEX, it asks you for the array, row_num & column_num.  we have got both the row & column number.  the array is where the results you want are.
=INDEX(B2:D6,MATCH(G1,A2:A6,0),MATCH(F1,B1:D1,0))

so from B2:D6, i want the 3rd row, 2nd column

edwin tam say: You may use a combination of VLOOKUP and MATCH.

Suppose your data is in range A1:D6 as shown in the picture below.
To find the value for 7% and 30 years, you can use:

=VLOOKUP(7%,A$1:D$6,MATCH(30,A$1:D$1,0),FALSE)


The MATCH function actually finds the column number of 30.) The "0" in the function means exact match.

Then, we can use VLOOKUP to locate the value for 7%. The FALSE in the VLOOKUP function also means exact match.
(Note that you may use "$" in the address of the range to "lock" the range when you drag the formula.)

trevor rabey say:

Um.. what do you need? what's the problem? Get costs on the tasks, baseline the plan, draw the graph. Then track progress, input the actual cost for each task, draw the graph.

Once you build the model you have the data so even if you can't get what you want out of the MSP2013 dashboards, you can always copy the data from the task usage or resource usage views, and build the graph in EXCEL.

You will be assigning cost type resources?

The only other way that a task can acquire a cost is by assigning fixed cost in the cost table. If you only assign one cost type resource for a certain amount of $ to a task it is more or less the same thing as assigning a value for fixed cost. But it is an advantage to be able to assign several different cost type resources to a task because it allows you to break up the overall cost of a task.
So if you have a task called say "lay 10000 bricks" you could assign bricks (material, $10000) and sand (material, $1000) and the bricklayer (work, $50/hour).

Or instead you could assign "cost of bricks", "cost of sand", "cost of bricklayer", all cost type resources at whatever value.

Or you could use a mixture of work, material and cost type resources, as well a fixed cost, for any given task.

Using just the fixed cost without any resources is a shortcut way to get $ against tasks, but it is rough.

Using just the cost type resources is also a shortcut to getting the costs on the tasks, but it is rougher than using work and material type resources. It is useful as a first pass.

Using just the cost type resources is a disadvantage because you can't input the quantities of materials.

Using just the cost type resources is a disadvantage because you can't track the work of the bricklayer.
