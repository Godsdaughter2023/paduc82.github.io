merlin unwin say:

Mailmerge ignore blank fields

How do I get Word 2016 to ignore blank address fields? Labels have blank lines - how do I stop this happening?

a. user say:

Update:

The \b switch mentioned by Doug should now work in Mac Word 2016 (from version 16.10 and possibly earlier versions numbered 16.something). 


Original post:

If this is Mac Word 2016 as the category suggests, then unfortunately the \b switch mentioned by Doug will not help.

Another way to solve this problem (using the same example as Doug)  is to use

{ MERGEFIELD Name }¶

{ MERGEFIELD Address2 }" }¶

(*All* the {} need to be the special field code brace pairs that you can insert in Mac Word using cmd-F9 (command-F9) or fn-cmd-F9, depending on how your keyboard is set up. Also, where I have put ¶, you need a normal paragraph mark, i.e. the thing you will get if you press the enter key.).

The idea is that if Address2 is blank, the IF field inserts nothing so in effect you have

{ MERGEFIELD Name }¶

{ MERGEFIELD City } { MERGEFIELD State } { MERGEFIELD Zip }¶

but if Address2 is not blank, the IF field inserts a paragraph mark followed by Address 2, so in effect you have

{ MERGEFIELD Name }¶

{ MERGEFIELD Address2 }¶

{ MERGEFIELD City } { MERGEFIELD State } { MERGEFIELD Zip }¶


So all you really need to do is extend that to cope with all the fields that might be blank in your address.


By default, Mailmerge should suppress blank lines anyway in some cases. The following conversation lists some of the rules for that...

https://answers.microsoft.com/en-us/msoffice/forum/msoffice_word-mso_winother/criteria-for-paragraph-mark-after-merge-field/987d7627-b342-46ee-8d8d-f21294691ac1
