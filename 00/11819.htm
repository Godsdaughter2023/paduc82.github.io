barb way say:

This kind of error is generally seen when a file has some corruption, possibly very minor - meaning that you can still open and print, for instance, but some action, like copying shapes, causes an error.

If you have one or two files that seem to see this problem, try cleaning them up (after making backup copies) using suggestions from this article:http://support.microsoft.com/kb/826418 / "How to troubleshoot damaged drawings in Visio"

Specifically, try the "Methods to try if you can open the drawing" section:

Method 1: Save the file as an XML drawing

Save the drawing as an XML drawing (.vdx file), and then save the XML drawing as a drawing (.vsd file). To do this:

  1. Start Visio, and then open your drawing.
  2. On the File menu, click Save As.
  3. In the Save As dialog box, click XML Drawing (*.vdx)in the Save as type box, and then type a file name in theFile name box. Specify a location where you want to save the drawing file, and then clickSave.
  4. Close all the drawings that are currently open.
  5. On the File menu, click Open.
  6. Locate the XML drawing (.vdx) that you saved earlier in step 3, and then clickOpen.
  7. On the File menu, click Save As.
  8. In the Save As dialog box, click Drawing (*.vsd) in the Save as type box, and then type a file name in the File name box. Specify a location where you want to save the drawing file, and then clickSave.
  9. Close the drawing.
  10. On the File menu, click Open
  11. Locate the drawing (.vsd file) that you saved earlier in step 8, and then clickOpen.

There are other options in that section, but this one is the most effective.

Barb

don cuthbert say:

I find that it takes a while after install for all of the configuration, device drivers and updates to complete.  If you can open and watch the full list of Apps in  the Start menu, most of them gradually update and install over the first while.  Windows Update meanwhile is pulling device drivers and other updates.  Usually it's best to let things sit a while and it also takes a few restarts to ensure that all of the installs and updates have completed.

I find Build 10130 to be pretty stable compared to previous Builds and have installed various versions of it on 4 computers multiple times.

Spartan still seems unready for general use.  It does usually crash and just vanish sometimes at the beginning, but seems to become more stable after a few launches.

Networking seems to be a problem, as I understand that a lot of the basics underlying it in the OS have been redesigned.  Keeping this working with so many different network adapters and older hardware is a challenge.  Usually these things can be resolved without needing to rollback to another OS version or Build.

There can be a lot of problems and even malware that survives these Build updates and causes problems too.  Sometimes a clean install is best.  I have my main PC that has been upgraded through many versions and Builds, but I also will do a clean install (dual boot) to another drive partition on this computer to compare performance and see if issues persist.  I also reinstalled Windows 8.1 to it's own partition to ensure that my OS license for Windows 8.1 on this PC is upgraded to Windows 10 next month.
