Yes, like this:

Sub CopyModule()
  Const strTextFile = "Dummy.bas"
  Dim strPath As String
  Dim wbkSource As Workbook
  Dim wbkTarget As Workbook
  Set wbkSource = Workbooks("Book1")
  Set wbkTarget = Workbooks("Book2")
  strPath = Environ("Temp")
  If Right(strPath, 1) <> "\" Then
    strPath = strPath & "\"
  End If
  wbkSource.VBProject.VBComponents("Module1").Export strPath & strTextFile
  wbkTarget.VBProject.VBComponents.Import strPath & strTextFile
  Kill strPath & strTextFile
End Sub

Modify the names as needed. Both workbooks must be open.

IF statements in form fields - Microsoft Community

wrierson say:

IF statements in form fields

What is the proper syntax for using IF statements in word form fields? I want a legacy form field to compare two fields, then do a simple division calculation with the results in the field that has the if statement in it.

doug robbins - mvp office apps & services (word) say:

With the document unprotected, use Alt+F9 to toggle on the display of field codes and you will see

 

{ FORMTEXT }

 

Place the selection after the last T in FORMTEXT and then using Ctrl+F9 to insert the necessary field delimiters, construct you expression in a similar way to the following example

 

{ FORMTEXT { IF Text2 > 0 { =Text1/Text2 } 0 }}

wrierson say:

The formula I used is {formtext {if cra=curvea{ {=dda/mda }}

wrierson say:

cra and curvea are text fields. dda and mda are numbers that have to calculate based on the results of cra and curvea

 

wrierson say: Also, there are four fields that cra has to look at to determine which number field to pull from, so can I do nested IF statements to check all four fields?

doug robbins - mvp office apps & services (word) say:

Actually, you do not need the IF field construction in a formfield,  it can just be in the document wherever you want the result to appear.

 

There is a problem with your construction (which may be the transcribing to the forum) as you have 4 opening { and only two closing }

 

Doing it without it being in a formfield, you would use

 

{ IF cra = curvea { = dda/mda }   }

 

To get the result to appear, you need to check the Calculate on Exit box for the formfields.

 

You can nest IF field constructions, but beware of doing it too deeply.  The alternative is just to have the negative condition show nothing and just place the constructions one after the other

 

{ IF cra = curvea { = dda/mda } "" }{ IF cra1 = curvea { = dda1/mda1 } "" }

macropod say:
The fact the OP has differing #s of { and } also indicates that the { and } are being typed. Typed braces won't work - they need to be inserted as matched pairs via Ctrl-F9.

FWIW, you don't need to specify the empty False conditions. You are correct, though, that the OP's fields won't work as they are. The following will work, though:

{if cra= curvea {=dda/mda*100} {if cra= curveb {=dda/mdb*100} {if cra= curvec {=dda/mdc*100} {if cra= curved {=dda/mdd*100}}}}}

{if cra= curvea {=dda/mda*100}}
