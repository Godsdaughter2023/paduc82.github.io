what is the maximum disk space that windows mobile 6 supports - Microsoft Community

eli-sha say:

what is the maximum disk space that windows mobile 6 supports

what is the maximum disk space that windows mobile 6 supports
chris de herrera say:

Hi,
Windows Mobile supports 2 different storage spaces:

1. Internal storage - the OEM decides how large this is and how it is used. This is flash and can be shared with the "rom" applications based on the OEM's design.

2. Secondary storage - Windows Mobile 6 supports secondary storage that uses the FAT32 file format. There are limitations in the drivers for SD/MicroSD/SDHC/MicroSDHC cards that limit them based on the timeframe in which the device was released and the OEM's testing.  If you use 3rd party hardware such as a Redfly, you can connect a USB hard drive and access it if it's formatted with FAT32.

So the short answer is tens of gigabytes however it depends on the OEM's implementation.

john mcghie say:

Hi Namnoot:

Well I am going to sit firmly on the fence here :-)

Like Daniel, I have used Master Documents in production for very large documents.  They are OK in the hands of a very skilled Word user.  I suspect Namnoot you are such a user (anyone who can keep a 1,000+ page document alive for 5 years is very skilled).

However, I could never recommend using the obsolete .doc format.  Not only is it incapable of storing the content that modern versions of Word can create, but it is fragile and given to lots of irritating side-effects, particularly to do with cross-reference.

I have Word 2008 and Word 2011 on this machine. Word 2011 is somewhat slower, very slow compared to the PC version.  If I open my 500-page test document (a complex document with lots of tables) using File>Open here are the timings:

Word 2008 file in .docx 28.5 seconds

Word 2011 file in .docx 51.8

Word 2008 file in .doc 16.6 seconds

Word 2011 file in .doc 28.5 seconds.

To make it fair, I timed until the page count at the bottom of the Word window stopped spinning.  That means that Word has fully paginated the document.  There's a bit of subterfuge to display the first two or three pages immediately, but Word hasn't "REALLY" loaded the document until pagination is complete.

Speed was a known issue before they shipped the product.  They asked us whether we would rather have it slow and "now" or wait another 12 months while they optimised it for speed.  Obviously, we chose to get it now {grin!}

Now, in terms of getting that document to go faster, there are a number of tricks you can try:

The first, of course, is to make sure your Office installation has updated itself to 14.4.1 or better.  When you first install, the install media is at base level.  There have been a few speed and stability tweaks since then.

For slinging around documents of that size I use a Mac Pro, and I suggest that you do also.  We should not that despite its very sexy appearance, the iMac is really just a laptop turned on edge.  It's very nice to use, and the big ones move along quite nicely.  But it really does not have the sheer power of a Mac Pro when the heat rises in the kitchen :-)  It's not even the processor that's t he problem, it's the motherboard: it just doesn't move data fast enough.

Let's assume that, unlike me, you can throw unlimited money at the problem!  You could go out and buy a MacPro.  Get the quad-core 3.7 GHz.  Word and nearly all other applications are single-threaded, so more cores will not make it go faster, but more MHz will!

Assuming you're not going to splash out for a MacPro, then fill the iMac up with memory.  Word will run a bit quicker if it has plenty of room to spread itself around.

Add an SSD if you have not already done so: it makes a dramatic difference to Word.

However, the bottom line is that Word on the Mac is a slug.  We are waiting for the next version and hoping it will be quicker. The PC Version 2010 is dramatically quicker! 18 seconds and 14 seconds on that same document, and that's running in Parallels on the same MacBook Pro with Word 2011 still running at the same time.

It is certainly worth questioning the use of "Tables".  Word slows down dramatically if any one table stretches beyond about 100 pages.  If that's all one table, split it into chunks: Word will run dramatically faster.

"Floating" graphics are another thing that slows Word down dramatically.

Styles are dramatically quicker than direct formatting in a long document.  Do all of your formatting in styles and things will speed up nicely (provided you remove the direct formatting).

Saving that document out to .docx then bringing it back will force an internal clean-up of the code and make it run quicker.

I have used Master Documents.  Personally I think they are just too much trouble and too flaky, so these days I normally use a half-way technique: "Referenced Documents".  Look up RD fields: these enable you to set references to a collection of "Chapter" files so you can build your TOC and Index without needing a contiguous document.  Fast and stable without all the breakages.  But it means folio by chapter, which I also hate.

Sometimes I will create the document as a set of independent files, then assemble it as a Master Document after I have finished editing and immediately before I print.  This prevents the crashes, hangs, and blowings-up yet still gives contiguous page numbers.

But yes, the bottom line is that Word 2011 is slow.  We had a choice: we chose to get the new features now.  The next version is not far away: I will be surprised if that were not much, much faster.
