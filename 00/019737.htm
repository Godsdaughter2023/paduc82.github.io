This is how to do the whole process from scratch:

  1. You need to have an open document (shouldn't be so)
  2. In the main menu, select View > Citations
  3. Click on the + button at the bottom to add your citations (these are not added to your document, but to a Master List)

OK, So what's stopping you? All that is well within Access capabilities.

The first thing you need to do is design the tables that will capture the information that you want to send via E-mail. These tables will need to be normalized properly. Once that is done, you can design a form that will enable the user to enter/select the information that will be included in the e-mail. If you need help along the way we would be glad to help, but you need to ask specific questions.

scottgem say:
Well first you are working backwards. Forms do NOT have fields. They have Controls that may or may not be bound to fields in a table. So you don't start with a form. Like I said you start by creating a Normalized table design. Make a list of all the data points you want to capture, then organize those data points into normalized tables. Only then do you start designing a form.

Access is different from any other app. with Word, for example, you can open up a document and start typing a letter. But with Access you need to understand something about Relational database design before you start.
