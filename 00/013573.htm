phillip m jones say:

Chance are with files that size you have some .jpg Pictures. That have not beed saved in Proper size and DPI before inserting into Word Document. 

2011 Do have in File Menu a command called Reduce file size that Greatly helps. But sometimes even that doesn't help a great deal.

The Proper way is go back to Original sized Picture and open in an editor such Graphics Converter or PhotoShop Elements and Edit.

  1. First Tweak sharpness so you get best Sharpness.
  2. Next adjust size say from 8" x 10" to 4 by 5, or 3 by 5.
  3. Now Save in PNG Format.
    PNG format is a Lossless format each time a PNG file is open It retains all detail and even if you edit further in Office it still doesn't lose detail.
    Jpeg is on other hand a Lossy Format and each time the file is open and closed detail is lost (due to compression/decompression issues)
  4. Now save ne file under a slightly different name
  5. The replace pictures with the edited version. 

Should Greatly reduce file sizes.

IMHO, yes, you're looking at a calculation bug with the end date but you also need to understand that dates can includes times.

An all day event is from 12:00 a.m. to 12:00 a.m. according to Outlook. You're calculating a date for next week but that variable includes a time as well. For example, "(current date)" returns:

Thursday, January 10, 2013 9:47:00 PM

not

Thursday, January 10, 2013

You want to get the "date string" of your date to get just the date.

Try changing this part of your script:

set theDate to (current date)

set nextWeekStart to date string of (theDate + (24 * 60 * 60 * 7))

set nextWeekEnd to date string of (theDate + (24 * 60 * 60 * 8))
