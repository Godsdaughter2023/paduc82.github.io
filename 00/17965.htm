Synchronizing Outlook and Windows Live (i.e., Hotmail) contacts/calendar with My Phone

To avoid potential sync conflicts, My Phone does not synchronize Outlook and Windows Live calendar or contact items.  To synchronize these items, you will need to utilize the synchronization mechanism supplied for those programs.  For Outlook, you will need to use Windows Mobile Device Center (WMDC) or Exchange Active Sync (EAS); for Windows Live you will need to use the Windows Live Mobile client.

By default, My Phone automatically syncs contacts, calendar items, tasks, text messages, browser favorites, photos, videos, music, and documents. You can choose not to sync any of these content types ( instructions ).  If you are concerned about having enough space for the various types, then consider archiving some of your data. The reason for this: Feature upgrades are only coming to Office 365 subscribers while the perpetual Office remains at the status of its original release in 2015 and will only get security patches and bug fixes. So if you change from the most recent program versions that come with the subscription to a perpetual version you might miss some of the newer features (not that many at the moment but there will be more).

stop annoying message on cancelling a report

   msgbox "no data to print"

   Cancel=true

 

This works fine; the message is shown on screen.  then the message "The OpenReport action was canceled" appears, requiring another action by the user (locating and clicking on the ok button).  Is it possible to bypass the second message?  The first lets the user know there is no data to print.  All I can deduce is to generate a temporary table, populate it, and then check the record count.  Thanks in advance.

 

John

 

bill mosca say:

John

 

You have to trap th error like this in the procedure that opens the report. I do it this way:

 

Private Sub Report_NoData(Cancel As Integer)
    Cancel = -1
End Sub

 

'Button on form that opens report.

Private Sub cmdOpenSummaryRpt_Click()
    On Error GoTo err_PROC

    DoCmd.OpenReport "RptSummary", acViewPreview


exit_PROC:
    On Error Resume Next
    Exit Sub

err_PROC:
    Select Case Err.Number
        Case 2501   'Canceled report
            MsgBox "No Data for selected filter"
            Resume Next
        Case Else
            Dim strErrMsg As String
            Dim lngIcon As Long

            strErrMsg = "Error " & Err.Number & " (" & Err.Description & ") " _
                        & "occurred in procedure cmdOpenSummaryRpt_Click of " _
                        & "VBA Document Form_frmCustomers"
            lngIcon = vbOKOnly + vbInformation

            MsgBox strErrMsg, lngIcon, "Error"
    End Select
    Resume exit_PROC

End Sub
