bob jones aka: cybertaz say:

Glad you figured it out :-) but to further clarify... Your solution actually isn't that you selected a "different date format", it's that you selected a valid date format.

Excel picks up it's default Date formats from your OS X System Preferences. The dot (.) is not a valid date separator unless it is set in the Formats of the OS X prefs for a Region which uses that separator or is entered as a Custom format that differs from the Region's default.

Regardless of the format you apply in Excel, if the content of the cell does not comply with the format the content is treated as text. (E.g., a cell can be formatted as Percentage but if words are entered into the cell the Percentage Formatting has no effect.) In your case, use of the dot separators did not comply with any of the valid date formats listed in Format> Cells - Date. That's also why the prior suggestion did not work for you.

jack - support engineer say:

Hello Alessandro,

Thank you for posting.  Yes, you can set IE to always open in the InPrivate Browsing mode.  You will need to create a new shortcut to IE.

1.     Click on Start

2.     Click on All Programs

3.     Right click on Internet Explorer

4.     Mouse over Send To

5.     Click Desktop (create shortcut)

6.     Go to your desktop

7.     Right click on the Internet Explorer shortcut

8.     Click Properties

9.     Locate the Target line

10.  Add a space –private after the .exe extension.  The command should look like this: "C:\Program Files\Internet Explorer\iexplore.exe -private"

NOTE: Make sure that you have the quotation marks at the beginning and end of the command line.  I hope this information is helpful.

ken sheridan say: You can standardise the style of forms by creating an unbound form in the style in which you wish new forms to be created, including a variety of controls of different types, and saving this, e.g. as frmTemplate.  Set the Form Template option for MS Access to frmTemplate.  If you then copy this form into any database, new forms in that database will adopt the style of the template when created.  NB: this does not apply to forms created by wizardry.

If you want new forms to adopt the style of an existing form in the database, just save a copy of that form as frmTemplate.  Any new form will then adopt its style.  It can then be deleted once you've finished with it.

The same can be done for reports.
