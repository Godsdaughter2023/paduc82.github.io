Enter Date Range One Time - Microsoft Community

mj_955 say:

Enter Date Range One Time

Hi,

I have a report based on 3 underlying queries (all are querying different datasets). Each query has a date range entry requirement. How do I design the report so the user only has to enter the date range once?

scottgem say:

Create an unbound form with 2 textboxes to enter the date range. Then reference those controls in your queries using the syntax:

Forms!formname!controlname

Add a button on the form to open the report.

scottgem say:

Here's the expression I put in the queries:

Between [Forms]![frmdatefilter]![txt]![EnterStartDate] And [Forms]![frmdatefilter]![txt]![EnterEndDate]

I was hoping that there was a way a user could click on a report and it would prompt once for a date range even though there may be multiple underlying queries that require a date range...

No, you cannot have the user click on a report and be prompted once. What difference is there to open a form, enter the dates then click a button to open the report? Maybe one extra click. 

The problem here is that when using a parameters query (which I rarely use), each time you run the query, whether directly or called in another query or report, it has to prompt you for the parameter. So you could be prompted for the same value several times. However, if you use a form reference, as I described, then you enter the parameters once in the form, and the queries look to those controls on the form for the parameters.

So I gave you the syntax for a form reference:

Forms!formname!controlname

You don't have a control named [txt]!EnterStartdate]. that's why you are getting prompted. So you might have a control named txtStart so the form reference would be:

Forms!frmdatefilter!txtStart

So if you set the criteria in your query to:

Between [Forms]![frmdatefilter]![txtStart] And [Forms]![frmdatefilter]![txtEnd]

then enter the dates on the form and THEN run the report, the report should open with no prompting for dates.

I had similar message returned trying to open my mp3 files from hyperlinks and resolved it by adding the root mp3 folder to Trusted Locations, after which I didn't get the message any more.

If you haven't already tried that it's worth a go.

Hi Arpit Patel,

If you want to deploy AAD Connect after the Cutover migration, you need to convert the Exchange 2010 mailboxes to mail-enabled users. 

To convert mailbox to Mail-Enabled User Mailbox, the process is to collect the attributes you want to keep, disable the mailbox, mail-enable the user, and replace the attributes. It's certainly scriptable. If you want this script strongly, refer to the script for exchange 2007 or ask question on Exchange development forum.

https://social.technet.microsoft.com/Forums/exchange/en-US/home?forum=exchangesvrdevelopment
