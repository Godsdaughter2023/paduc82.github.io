e.j. gun say:

If only it affected all documents, rather than just those based on the Normal template,

 

You could add a short macro to your Normal template, and add a button to the QAT that, when pressed, sets the background color to blue.  This would at least make it much easier to do this for each document.

'
' SetBlueBackground Macro
' This macro sets the background of the active Word document to
' "Dark Blue, Text 2, Darker 25%".  This automatically sets the
' text color to White.  The printed output will still be black &
' white, unless the user goes to the Word Options/Display/Print Options
' and checks the "Print Background Color and Images" box
'

Sub SetBlueBackground()
    ActiveDocument.Background.Fill.ForeColor.ObjectThemeColor = _
        wdThemeColorText2
    ActiveDocument.Background.Fill.ForeColor.TintAndShade = -0.25
    ActiveDocument.Background.Fill.Visible = msoTrue
    ActiveDocument.Background.Fill.Solid
End Sub

HTH,

Eric

Sub PrintSelectionWithoutBorders()
  Dim rSelected As Range
  Set rSelected = Selection
  Workbooks.Add xlWorksheet  ' create new single sheet workbook
  rSelected.Copy
  With ActiveSheet.Range(rSelected.Address)
    .PasteSpecial xlPasteAllExceptBorders  'paste into the new workbook without its borders

  ActiveWorkbook.Close False   ' close the temporary workbook

The following VBA Script:

1. Turns on "No Borders" for the current cell range selected.

2. Prints the selection.

3. Turns on "All Borders" for the current cell range selected. 

NOTE: The script would need to be customized if you have varying types of borders in your selection.

Sub PrintNoBorders()
'The following turns off borders on the cells currently selected
    Selection.Borders(xlDiagonalDown).LineStyle = xlNone
    Selection.Borders(xlDiagonalUp).LineStyle = xlNone
    Selection.Borders(xlEdgeLeft).LineStyle = xlNone
    Selection.Borders(xlEdgeTop).LineStyle = xlNone
    Selection.Borders(xlEdgeBottom).LineStyle = xlNone
    Selection.Borders(xlEdgeRight).LineStyle = xlNone
    Selection.Borders(xlInsideVertical).LineStyle = xlNone
    Selection.Borders(xlInsideHorizontal).LineStyle = xlNone
    Selection.Borders(xlDiagonalDown).LineStyle = xlNone
    Selection.Borders(xlDiagonalUp).LineStyle = xlNone

'The following prints your selection to your default printer
    Selection.PrintOut Copies:=1, Collate:=True
   
'The following turns on thin black borders on all cells selected.
    With Selection.Borders(xlEdgeLeft)
        .LineStyle = xlContinuous
        .ColorIndex = 0
        .TintAndShade = 0
        .Weight = xlThin
    End With
    With Selection.Borders(xlEdgeTop)
        .LineStyle = xlContinuous
        .ColorIndex = 0
        .TintAndShade = 0
        .Weight = xlThin
    End With
    With Selection.Borders(xlEdgeBottom)
        .LineStyle = xlContinuous
        .ColorIndex = 0
        .TintAndShade = 0
        .Weight = xlThin
    End With
    With Selection.Borders(xlEdgeRight)
        .LineStyle = xlContinuous
        .ColorIndex = 0
        .TintAndShade = 0

End Sub

Further enhancements to GSM networks are provided by Enhanced Data rates for GSM Evolution (EDGE) technology. EDGE provides up to three times the data capacity of GPRS. Using EDGE, operators can handle three times more subscribers than GPRS; triple their data rate per subscriber, or add extra capacity to their voice communications. EDGE uses the same TDMA (Time Division Multiple Access) frame structure, logic channel and 200kHz carrier bandwidth as today's GSM networks, which allows it to be overlaid directly onto an existing GSM network. For many existing GSM/GPRS networks, EDGE is a simple software-upgrade.

EDGE allows the delivery of advanced mobile services such as the downloading of video and music clips, full multimedia messaging, high-speed colour Internet access and e-mail on the move.

Due to the very small incremental cost of including EDGE capability in GSM network deployment, virtually all new GSM infrastructure deployments are also EDGE capable and nearly all new mid- to high-level GSM devices also include EDGE radio technology. ken sheridan say: You need a UnitPrice or similar column in both tblProducts and tblPurchaseOrderDetails.  When inserting a row into the latter in a form assign the current price from tblProducts  into the Unitprice column in tblPurchaseOrderDetails by putting code along these lines in the AfterUpdate event procedure of the ProductID control:

Dim strCriteria As String

strCriteria = "ProductID = " & Nz(Me.ProductID,0)

Me.UnitPrice = DLookup("UnitPrice", "tblProducts", strCriteria)

This assumes that the ProductID keys are of numeric data type.

ken sheridan say: I'm not clear whether you have only the one VPrice column (field), that in tblProducts, or whether you also have one in tblPurchaseOrderDetails.  You must have both, both must be of currency data type, and the latter must be assigned a value only when a new row is inserted into tblPurchaseOrderDetails via the form.  This will assign the current price to the new record, but all previous records will remain unaltered.

If you do have the column in both tables, but the values in that in tblPurchaseOrderDetails are all reflecting the current value in tblProducts then the only thing I can think of is that the column in tblPurchaseOrderDetails is a 'lookup field'.  If so its actual value will not be the currency value at all but a hidden numeric value referencing the key of tblProducts.  This would be a highly unusual scenario, but it's possible the tblPurchaseOrderDetails table could have been inappropriately designed that way.

You mention 'running' a query, which to me suggests an 'action' query which updates data rather that a SELECT query which returns data.  What is this query and where does it fit into the picture?

For an example of how it should work take a look at the Order Subform for Order Details subform in Northwind.   The AfterUpdate event procedure of the ProductID control, while more complex than that I gave you, does much the same at the end of the day.  You'll see that in Northwind the Products table contains a List Price column and the Order Details table a Unit Price column.  In essence the current value of the former is assigned to the latter when a new row is inserted into Order Details.
