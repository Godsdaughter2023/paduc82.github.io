tjk_001 say: After much frustration with attempting to open my own Click Once Application using Access VBA I stumbled upon the correct, and thankfully simple, answer. 

First you need to know that the file extension is actually .appref-ms
I discovered this little fact out when the auto complete feature in the address window of a folder showed that to me. Saved my life, not really but after hours of frustration it was the light at the end of the tunnel.

Second, the normal Shell function in Access VBA does not work with the .appref-ms file extension. While someone else may have figured out how to get that to work, I was unable to return anything but an empty string and a file not found error. 

Lastly, what does work is the Application.FollowHyperlink function. This turns out to be easier and simpler to use than the Shell function so another saving grace there.

Put it all together and that should solve your problem.

Application.FollowHyperlink "file_path_including_file_name.appref-ms"
You can pass parameters to this function really easily by doing the following:

Dim path
Dim filename

path = "yourfoldercontainingtheClickOnceApplication"
filename = "Filename.appref-ms"

Application.FollowHyperlink path & filename

'or you could do:

Application.FollowHyperlink "C:/Documents and Settings/User/ClickOnceApplication/" & filename

'or any combination you require.

Just make sure to enclose static locations in quotes, include the ampersand (&) between the quoted information and the parameters and the function will put it all together.

To answer your questions:
