The inetmgr.exe is a type of executable file used to run a program on your PC. The inetmgr.exe errors can be attributed to possible file corruption, invalid registry entries, or a virus and malware infection. To verify, have you tried enabling the hidden Administrator account, and see if you are to run or install the previously blocked app/ software? If not, follow these steps:

  1. Open an elevated command prompt. Press Windows key + X, and then right-click on Command Prompt (Admin).
  2. At the command prompt, type the following command, and then press Enter: Net user administrator /active:yes
  3. Access the newly enabled built-in Administrator account by clicking your user account name in the Start, and then click the Administrator account. It will also appear on the login screen of Windows 10.
  4. Launch the software that you want to run or install.
  5. After installing the software, sign out from the hidden Administrator account, and then sign in using your original account.
  6. Check the software that was installed on the hidden Administrator account, it should also appear on your original account too.
  7. Once confirmed, disable the hidden Administrator account by typing Net user administrator /active:no in the command prompt box.

yaqub k say:

Hi Sarah,

When you use the System Refresh feature, Windows is still technically starting over with a fresh install of the operating system. The main difference is that the Refresh process automatically sets aside your data, Windows settings, and Metro apps, then puts them back where they belong once the operating system is reinstalled. This will result is a clean slate for your operating system, but with your configuration settings and data intact.


Refreshing your PC goes like this:
1.The PC boots into Windows Recovery Environment.
2.Windows Recovery Environment scans the hard drive for your data, settings, and apps, and puts them aside (on the same drive).
3.Windows Recovery Environment installs a fresh copy of Windows.
4. It restores the data, settings, and apps it has set aside into the newly installed copy of Windows.
5.The PC restarts into the newly installed copy of Windows.

vinod n say:

Hello Willem,

Give a try with the steps below:

- Click on start (bottom left of desktop)
- Right click on "computer" and select properties.
- On the left side, just below Tasks, select "Device Manager".
- Locate your device, right click it and select properties.
- Select the "driver" tab and click uninstall.
- Click ok and reboot the computer.

If not, the only workaround for this issue is to install the latest driver from the manufacturer website to overwrite the current driver.
