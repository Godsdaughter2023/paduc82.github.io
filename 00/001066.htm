kbullkar say: I think I found the solution.

You need to be comfortable editing files within the application package (right-click the Microsoft Word application, "Show Package Contents".

THIS IS EDITING PACKAGE FILE CONTENTS - THERE IS DANGER WHEN YOU DO THIS - ONLY FOLLOW THESE INSTRUCTIONS IF YOU TAKE PERSONAL RESPONSIBILITY FOR YOUR ACTIONS - BACKUP AND REALIZE YOUR BACKOUT OPTION IS TO RESTORE FROM BACKUP - I DON'T PROVIDE TECHNICAL SUPPORT

Browse: Contents -> Resources -> Style

Put a copy of the APASixthEditionOfficeOnline.xsl file (from Word 2010 SP1) into this directory.
Open the file with a text editor (e.g. TextWrangler).
The following two lines need to be changed (first line and ~seventh line)
  • <?xml version="1.0" encoding="ISO-8859-1"?>
  • <xsl:output method="html" encoding="us-ascii"/> 
Change them to:
  • <?xml version="1.0" encoding="utf-8"?>
  • <xsl:output method="html" encoding="UTF-8"/> 
The following commands are based on TextWrangler.
  • Go to file, save as.
  • You need to change the Line Breaks from: Windows (CRLF) to Unix (LF).
  • You need to change the Encoding from: Western (ISO Latin 1) to Unicode (UTF-8).
Now, when Word 2011 opens, it should recognize this style file, but you aren't done yet... the DOI field still won't show up when you add a reference. In order to do this, you need to edit the BIBFORM.XML file in the Contents -> Resources Directory.

At the end of each source entry (right before </Source> I added the DOI tag as follows:

   <Tag>
     <Label>DOI</Label>
     <DataTag>b:DOI</DataTag>
     <Sample>Example: 10.1000/182</Sample>
     <Row>11</Row>
     <Col>1</Col>
     <Width>5</Width>
    </Tag>

MAKE SURE TO SET THE ROW VALUE TO BE ONE HIGHER THAN THE HIGHEST ENTRY IN THAT SOURCE. IT IS DIFFERENT FOR EVERY SOURCE

There are 17 source types to edit.

I would post the BIBFORM.XML and APA6.XSL files (I renamed the latter), but I don't think we are allowed to do that here, so you will have to make the edits manually.

It seems to be working for me. I am still going to look for the built in bibliography styles (currently supports "Bibliography" and "Works Cited", but not "References" - this is easy to change manually in the document. If I figure this out, I will post.
