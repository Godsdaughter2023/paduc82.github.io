davidf2 say:

It is possible that you may be seeing the old pages from your local cache. When you view the page key CTRL + F5 which will force a reload.

It is also possible that you are not over-writing the old html files. Go to Tools > Web Page Options and change the title to what you want and ok out. Then do a web page preview of the page. In IE or the browser you are viewing the page go to View > Source and the source code for the page should open in NotePad or some other default code editor on your local computer. Look down about 15 lines or so and find <title> and you should see your title in that line of code. If so then go ahead and Publish to the Web to produce the filtered web files, delete the old web files on your host server, and upload the new ones. You should be good to go.

If you continue to have problems, post back with the URL of your site.

Problem Importing a .pst file from Outlook 2010 into Outlook 365 - Microsoft Community

meagan01 say:

Problem Importing a .pst file from Outlook 2010 into Outlook 365

When I try to import a .pst file that was exported from Outlook 2010 into Outlook 365, I receive the following message:

"The Outlook data file (.pst) is already in use in the current profile. "

karl timmermans (outlook mvp 2012-2018) say:
>>
THIS IS EXACTLY RIGHT!!
<<

OK - with that in mind, going to split this up to deal with this in two parts (Email & Contact.calendar data)

Before starting anything - click on the <Send/Receive> tab on your ribbon and click on <Work Offline> to go Offline (this is an on/off toggle switch). The reason for doing this is to avoid Outlook starting the sync process while you're copying your data.

Next to simply references, please set the Outlook Nav Pane to show the full folder list by pressing CTRL+6

Lastly, as a FYI, during the following <Move> instructions, am going to advise you to select <Copy to Folder> instead of <Other folder>. The reason for this is so that the source items don't get deleted from the source folder in case something goes awry during the Copy process.

Part #1 - Email data

#1 - Click on the <Inbox> folder in the PST folder tree
#2 - Press CTRL+A to highlight all the items in the <Inbox> folder
#3 - Right-click on the highlighted items --> select <Move> --> select <Copy to Folder> --> select the <Inbox> in the IMAP folder tree

For any other email folders you may have created in the PST folder tree

#4 - Right-Click on each email folder --> select <Copy Folder> --> select the applicable location in the IMAP folder tree
#5 - Rinse and repeat #4 for each folder you want to move to your IMAP acount

#6 - When done - set Outlook to work online by clicking on the <Work Offline> button again
#7 - Click on <Send Receive all folders> to start the Outlook sync process. This may take a while depending on how much data you have to sync up to the server. (To emphasize, be sure you have enough space on the server with this email provider otherwise this is all going to be a waste of time)

Part #2 - Contact/calendar data (this assumes you have contact/calendar data in your PST file that you want to use/keep)

#8 - Contact/calendar data doesn't sync with the server so there's no way to share that data with your other devices just by using an IMAP account
#9 - One approach to use for sharing your contact/calendar data with other devices is to create a free Outlook.com account (go to https://www.outlook.com)
#10 - Add this account to your Outlook profile - just enter your new Outlook.com email address/password - this will add it as an Exchange account
#11 - Once the account is added to the profile, go to File --> Account Settings --> Data Files tab --> highlight the .ost data file for the Outlook.com account and set it as the default data file for the profile (this is where your "default" contact/calendar folders will be located)
#12 - Select the contact folder in your PST file and repeat steps #2 & 3 for this folder (destination folder is going to be the Contact folder in the Outlook.com folder tree)

For you calendar data

#13 - Select the calendar folder in the PST file
#14 - Click on the <View> tab on the Outlook ribbon
#15 - Select <Change View> --> select a "List" view
#16 - Perform steps #2 & 3 to copy the calendar items to the Outlook.com calendar folder

When you're all done with the above and happy that everything has been copied and looks right, you can remove the PST file from the profile

#17 - Right-click on the <Root> of the PST file --> Close

You're now all good to go with this machine.

#18 - Just need to configure both your email and Outlook.com account on the other devices by just entering the email address/password for each account. Each will be configured automatically (email will be IMAP and Outlook.com will be Exchange or Exchange ActiveSync to allow you to share you contact/calendar data - there's no requirement to use your Outlook.com for email.

When all this is done, you're good to go with all your devices :-)
