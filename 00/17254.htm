>>

..............

The fact that the 3000 emails were downloaded again means one of two things
1) she was using a POP3 account with the option "Leave Messages on Server" checked OR
2) the account was configured as an IMAP account. In this case, if emails were saved to other folders, those would not be downloaded if the account is now configured as a POP3 account which only downloads items in the <Inbox>

In either case, contacts are not stored on the server. The fact that you found an .ost file would seem to indicate that the account was originally configured as an IMAP Account AND the factory reset you performed saved the data files. Unfortunately, an .ost file can only be used by the original Outlook profile/email account that originally created it. If it was indeed originally configured as an IMAP account, the contact/calendar folders would also be stored in the .ost file in folders that had the words "This Computer Only" in the folder description. You are going to need a 3rd party OST2PST conversion utility to access the data in the .ost file.

I have 5 calculated fields in an Access query and I need to determine the minimum number of all 5 fields.  I'm sure I could do a giant IIF statement but it seems there could be a better way.  I am calculating the difference between production and production levels.  If the person has already hit the highest level, the number should be 0 (zero). I am not a programmer so can anybody help me with this? 

ken sheridan say: You have confirmed our suspicions.  In a relational database each level should not be represented by a column in a table, but by a row in a related table.  For each row in a promotions table there would therefore be up to five rows in a referencing table of levels whose columns would be a foreign key referencing the primary key of the promotions table, a column to identify the level, whose value could simply be a number within the range 1 to 5, and a third column to quantify the value achieved at that level.  The primary key of the referencing table would be a composite one made up of the first two of these columns.  It is then possible to compute any aggregations over the set of levels reached per promotion by use of the standard SQL aggregation operators like SUM, COUNT, MIN, MAX or AVG.

workinghour say:

I have to agree with Steve on this one. Generally, the format for getting videos to work in PPT2007 somewhat consistently from machine to machine seems to be:

1) Have a local copy of the video file

2) Convert the video to WMV for best results

3) Make sure the WMV file and the PPTWhatever file are in the same folder

4) Insert the video

5) Zip the folder containing the PPTwhatever file and the WMV

6) Have the next user download the ZIP folder to their machine and unzip it

I've gotten the most consistent results with this approach. As for getting the YouTube video onto your machine, you can look for a ripper program like RealPlayer or a browser plugin (eg, https://addons.mozilla.org/En-us/firefox/addon/download-youtube/) to get the video, and a transcoder program like FormatFactory or Handbrake to convert it to WMV.

(Mind you, you should probably check with your legal department to see how flexible they are with this approach as it's not exactly copyright-friendly.)

Good luck!

-Eric
