    lFirstSpace = InStr(1, sFilename, " ")

rick rothstein, mvp - excel say:
It will only return tomorrows date if the current time is after 12 noon. CLng rounds fractional values using Banker's Rounding (0.5 rounds to the nearest even integer value, all other numbers round normally). Since VB stores date/times as a floating point number where the whole number is an offset number from a "date zero" (12/30/1899) and the decimal part is the fraction of a 24-hour day represented by the time, then that fractional part will be greater than 0.5 for any time values after 12 noon; hence, CLng will round them up to the next whole number which will be the offset value for the day after today's offset value... in other words, tomorrow's date. To get what I think you want, use the Int function instead of the CLng function.

If your intention is to determine if Cells(ActiveCell.Row,1) is part of "today", where "today" means between today at 11am and tomorrow at 11am, perhaps the following is what you need.

Dim findDate As Date, startToday As Date, startTomorrow As Date
findDate = Cells(ActiveCell.Row, 1)
startToday = Date + #11:00:00 AM#
startTomorrow = Date + 1 + #11:00:00 AM#
If startToday <= findDate And findDate < startTomorrow Then
       MsgBox findDate & " is part of " & Date
Else
       MsgBox findDate & " is not part of " & Date
End If

Some notes:

1. findDate = Int(Cells(ActiveCell.Row, 1)) would extract the date without time -- effectively, midnight on that date.  That does not seem to be what you need if your day starts at 11:00 AM.

2. You should never write Int(Now()) if you only want the current date.  Use Date() instead.  Now() returns the current time as well as date.  Int(Now()) extracts just the current date.  But that is what the Date() function returns.  Why use two functions when one will do?

3. The time constant #11:00:00 AM# was actually entered as #11:00 AM#.  VBA changes it.  Alternatively, you can write TimeSerial(11,0,0).

4. Note the use "<=" and "<" in the If statement.

5. As Rick explained, you should never use CLng(...) to extract the integer date.  CLng rounds its argument.  Int(...) truncates its argument to an integer.  That is what you would use if you wanted to extract the integer date.  But it appears that is not appropriate for you do after all.  (But see errata in a follow-up response.)

joeu2004 say: Errata....  I wrote:
startToday = Date + #11:00:00 AM#
startTomorrow = Date + 1 + #11:00:00 AM#

Since your day is between 11:00 AM one day and 11:00 AM the next day, you might be running this code after midnight.  So it might be better and more reliable to write:

Dim findDate As Date, startToday As Date, startTomorrow As Date
findDate = Cells(ActiveCell.Row, 1)
startToday = Now()
startToday = Int(startToday) + #11:00:00 AM# - IIf(Hour(startToday) < 11, 1, 0)
startTomorrow = startToday + 1
If startToday <= findDate And findDate < startTomorrow Then
       MsgBox findDate & " is part of the day starting " & startToday
Else
       MsgBox findDate & " is not part of the day starting " & startToday
End If

Note that there is only one reference to Now (or Date).  This is important if the code were run just before midnight.  It is possible for two calls to Now (or Date) to return different date/times, one before midnight and one after midnight, because the system clock is updated in between.

(Arguably, the probability is very low.  But it does happen.)
