How do I delete User-Defined Number Lists in Word 2016 - Microsoft Community

mike_cavaliere say:

How do I delete User-Defined Number Lists in Word 2016 (Mac)?

I created a user-defined numbering list that I would like to delete and I cannot figure out how to. I do not have the option under format to "reset" as it is greyed out.
john mcghie say:

Hi Mike:

I am afraid Arshad has completely failed to understand your question (and he doesn't know the answer...).  Just ignore what he said.

Now: the "Reset" button in the Bullets and numbering dialog is greyed out because that position is already set to default.  You need to click in a sample in the numbered list gallery to select its numbering scheme, then you will be able to reset it.

When you get into it, the Bullets and Numbering dialog is very confusing: it is not showing you "all the numbering schemes in the document"; it is showing you "the eight most-recently used" schemes (depending where you look). I see they've done some fixes in this dialog, it now works for bullets and numbering, which it didn't in the previous versions of Word 2016.

The way Word's numbering mechanism works means that it is NEVER possible to actually delete a numbering scheme once it has been created. The best you can do is simply "not use" the numbering scheme you do not want. This doesn't make much difference: each numbering definition is only a few bytes, so just leave it there and let Word take care of it.

Word will clean up the unused numbering schemes automatically on document close, once it finds that there are more than about 200 schemes in the document. It's quite rare for a document to ever reach this count: there may be thousands of numbered paragraphs in the document, but hopefully they all belong to only "one" numbering scheme.

The reason we can't delete them is that Word stores these things in a spreadsheet-like structure that is just a simple list, in order of the sequence in which they were created.  If there are 27 in a document and you deleted number 8, the list would renumber and all of the numbering using schemes of 8 or greater would instantly be corrupted (chances are, so would the document be, so it wouldn't open). To trim the number of schemes, Word has to do a very complex operation to determine the ID number of the scheme to be deleted, run through the document and change the pointers of all of the numbered or bulleted items to point to the new numbers after the list has been renumbered, then delete the numbering scheme, then re-number all of the numbering schemes.

Not something you would want to try for yourself :-)

If you just leave it there, Word will never notice.  Eventually that numbering scheme will fall of the list of the eight that get displayed, so you won't even be able to see it any more.

There: That was a rather long and complex way of saying "Just don't use it, it will automatically disappear eventually!"

Hope this helps

scottgem say:

Ok, first there are two things going on here. Do you need to sequentially number the records or do you just need a count for the month? These are different things.

If you just want a count then use the Dcount and yes you enter it as the ControlSource, This will show a total count for the current month.

However if you want sequential numbers that reset after each month, you need to use the techniques in my blog. So look at the example you pulled. You need to change the criteria of the DMax. So you want to test whether the Date of Issue is equal to the Date of issue of the previous max record.

The Dmax says to return the highest number in the Sequence field where the criteria matches. The NZ function returns a 0 if there are not matching records. It then takes that number and increments it by 1. So if the criteria doesn't match it returns a 0 and adds 1, restarting the number.

"Year([InquiryDate]) = “ & Year(Me.[txtInquiryDate]) & " AND Month([InquiryDate]) = " & Month(Me.txtInquiryDate)

Its very logical if you stop to think about it.

The tables are probably "floating"; that is, in the Table Properties dialog box, the text wrapping is set to "Around". Instead, click the "None" button, which forces the table to be "in line with text", as part of a regular paragraph.

If this doesn't give the formatting you want, one possible solution is to combine all five tables into one long table. Include an extra empty row between each pair of original tables, and turn off the borders in that row so that they appear to be separate.

Another possible solution is to create a table that has one column and five rows. Then cut/paste the original tables and paste each one into a separate cell of the new table. The new outer table can be floating, but each of the original tables must be converted to "None" wrapping to keep them inside the cells.

Thanks  - I ended up using this thanks to Suzanne S. Barnhill!

 
{ MERGEFIELD Address1 }<line break>
{ IF { MERGEFIELD Address2 } = "" "" "{MERGEFIELD ADDRESS2}<line break>" }
 
The way that will actually appear in your document is:
 
{ MERGEFIELD Address1 }
{ IF { MERGEFIELD Address2 } = "" "" "{MERGEFIELD ADDRESS2}
" }
 
By putting the line break inside the FalseText quotes, you ensure that thereis a line break only if the line contains text; otherwise, there will be no line.
