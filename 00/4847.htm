ericyang_msft6286 say:

Hi Bernard,

We can’t force deleting a content type and need to first remove it from all sites before it can be deleted. After some research, I have found the following PowerShell script which can to use to find where a content type is being used:

Connect-SPOservice
$site = Get-SPOSite("<site_collection_url>");
foreach ($web in $site.AllWebs) {
   $ctype = $web.ContentTypes["<content type name>"]
   $usages = [Microsoft.Sharepoint.SPContentTypeUsage]::GetUsages($ctype)
   foreach ($usage in $usages) {
      Write-Host $usage.Url
   }
}

Replace <site_collection_url> with the URL of your site collection. Replace <content type name> with in-use content type name.
