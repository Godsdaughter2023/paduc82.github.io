diane poremsky mvp (slipstick.com) say:
It's not an uncommon problem with SMTP - and yes, internet issues are usually at the root. Outlook doesn't get confirmation that the smtp server accepted the message and keeps retrying until it gets a reply back.

Deleting the account won't fix it (as you have discovered). There isn't a good solution - a possible work around is to show the total folder count on the outbox and disable send immediately. Set the send/receive settings to 5 - 8 (or higher) or change the settings so messages are only sent when you press Send/Receive or Send All. After a send, move any messages left in the outbox to another folder.

vasukikarthi say: If i do this how can i know message in outbox is sent or not, if a message sent to recipient it should automatically goes to sent folder, am i right? Please give suggestion for that  

diane poremsky mvp (slipstick.com) say:
>> if a message sent to recipient it should automatically goes to sent folder, am i right?
Correct... but if outlook doesn't get the final OK to confirm acceptance, it won't move it to the sent folder. It appears that Outlook is not getting the final OK when the internet connection is slow though.

Raising the send/receive time and turning off send immediately can help as Outlook won't still be sending a message when the next send receive pass starts - this may be enough to 'fix' the problem. But if not, Outlook will keep trying to resend - which leaves the only option of moving the message out of the outbox so it doesn't send multiple copies of the message.

VLOOKUP Closest Match - Microsoft Community

scenicaz say:

VLOOKUP Closest Match

I have a table array with Hydrometer readings and the percentage of charge as the two columns:

1.073    10%

1.098    20%

1.124    30%

1.148    40%

1.172   50%

1.195    60%

1.217    70%

1.238    80%

1.258    90%

1.277   100%

My VLookup works except that the value of 1.250 will return 80%, yet it is closest to 90%.  The same hold true with other values, i.e. 1.235 will return 70%, yet it is 18 points from 70% and only 3 points from 80%. 

So, the question is:  How does VLookup determine the "closest" match?  By looking at the first encountered value, or is it really by "closest" and I'm doing something incorrectly?

hansv mvp say:

If you specify TRUE as 4th argument in VLOOKUP, or omit the 4th argument, Excel looks for the largest value that is less than or equal to the first argument. So If the lookup value is 1250, the matching value is 1238 because it's less than 1250, and there are no larger values in the column that are less than or equal to 1250.

If you want the closest match, you can use an array formula confirmed with Ctrl+Shift+Enter:

Let's say your table array is in A2:B11 and the lookup value is in D2. The formula would be

=INDEX(B2:B11,MATCH(MIN(ABS(A2:A11-D2)),ABS(A2:A11-D2),0))

The { } around the formula indicate that it is an array formula. You shouldn't type { } yourself; they are the result of confirming the formula with Ctrl+Shift+Enter.
