karl timmermans [outlook mvp 2012-2018] say:

You raise a bunch of different things so will take this in pieces

#1 - O'2007 didn't <force> the use of different PST files for each account - that could have been changed after the account was configured so that all POP accounts goes to a single <default> PST file. The difference with the O'2013 is that "stipulate" using an existing PST file when you initially configure the account as opposed to doing it after the fact.

#2 - Question re: "Should or shouldn't all email go to a single PST file etc?"

There are actually 2 primary issues/questions involved (1) where the email is received and (2) how the data is maintained long term. The problem is not "where" the items are received but rather - <keeping> everything in a single PST file allowing it to grow to a very large size. My 2 cents in that regard is as follows:

a) I have all my POP accounts go into a single PST file where I have the <Inbox> grouped by <Email Account as the primary grouping followed by another grouping which in turn is sorted according to my personal preference. That let's me see all inbound mail very quickly in one <Inbox>. Each person will have their own preference as to how to organize the <Inbox> in this case - no right or wrong answer which is the reason not going into anything beyond the <Primary> group.

BUT......................!!!!!!

b) I do not store all emails in a single PST file - never have with historical going back more than a decade where my PST files created and organized by how I look for information - no right or wrong answer but in my case - just "archiving" by date is of no use - would take forever to locate things but other people are very comfortable organizing everything by date.

You can create all kinds of secondary PST files which are not attached to a given email account but would suggest avoiding creating 100's simply because there was no thought put into data organization. I know people who feel they are "completely" organized but can't find anything since they create a new PST file when the better solution would have been to be create specific topic folders and sub-folder within a major topic PST file.

To summarize - wholeheartedly agree with opinions stating that the entire universe should not be stored in a "single" PST file both for a performance and safety standpoint. Extremely large PST files will show things down just as having to open "hundreds" of individual small PST files.

jay freedman say:
In the Label Options dialog, first make sure the Label Vendors dropdown is set to "Avery US Letter".

In the Product Number list, scroll down through all the numbers that start with the digit "1". The 2160 comes right after 18660. (The items in this list are arranged in "alphabetic" order, not in numeric order.)

suzanne s. barnhill say:

That's the size that 2160 is; it's a mini-sheet. If your labels are on a Letter-sized sheet, then they're not 2160 (or they're not Avery). The same size labels on a Letter-size sheet are 5160.
