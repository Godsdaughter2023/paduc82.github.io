jim gordon mvp say:

Compatibility mode is invoked when you open a Word document that was saved in one of the old file formats, such as .doc or when you open a document saved in a current file format that was saved by a word processor other than Microsoft Word.

Compatibility mode turns off many features in Word's interface so that you don't accidentally add a feature that is not supported by old versions of Word or other word processing applications. The presumption is that you want to share the file with users having old versions of Word or other word processors and you want to maintain compatibility with them.

If you are not sharing with someone having access to old versions of Microsoft Office (prior to 2004) or other word processors (LibreOffice, TextEdit, etc.) you can turn on all of Words features by choosing File > Convert Document, which updates the file to the current format for word processing documents saved by Microsoft Word. 

This article explains the file formats for Word 2011, and they are unchanged in 2016:

Office 2011 for Mac: Compatibility among Word File Formats

john korchok say:

As Jim has mentioned, you can change a file to the current 2016 format by choosing File>Convert Document, which will make [Compatibility Mode] to disappear from the title.

If the document is an older format with a .doc file ending, using File>Convert Document will result in a small but noticeable shift in many page elements. This is most noticeable with text boxes, where approximately 0.02" padding is added to the interior of the text box, moving the contents to the right.

If the document is .docx format from Word 2007, 2008, 2010 or 2011, document elements mostly stay in the same position when coverted to 2016. The principle difference is that in the older format, table borders jut out into the left margin by the amount of cell spacing applied to the table (0.08" by default). This means text inside the table aligns with text outside the table. When you use File>Convert Document, tables are converted to Word 2013/2016 rendering where the border of the table aligns with the left margin and text is indented by the amount set for cell margins.

If you're a design professional, the way to create tables that render identically in all versions is to set cell margins at 0" and indent table text with paragraph indentation instead.

Philip's information about editability of 2016 documents is incorrect. 2016 documents can be freely edited in any version of Word back to 2007, but when they are saved, they will revert to an earlier version of the file format.

Address spoofing happens to most people at one point or another and, despite what some people may say, it cannot be stopped.  The computer sending the mail can be anywhere in the world.  It is extremely easy to use any arbitrary address (such as yours or mine) as the sending address for spam.  The only consolation is that spammers change the sending address frequently, so it's likely that, given time, the spoofing will move on to some other person's address and you'll stop receiving non-delivery reports for messages you didn't send.

It's best if you have two-step verification enabled for your account.  That way, even if the bad folks guess the password accidentally, they still won't be able to log into your account.

The following example returns the next row after the bottom of the AutoFiltered range.


Explanation of code:

ActiveSheet.AutoFilter.Range is the generic reference of the filtered data. It includes column headers plus all rows of both visible and non visible rows. Therefore the next row after the last row will always be the empty row below the AutoFilter range.

Note the leading dot with  .Cells and  .Rows that ties them to the With statement.

    Dim Nx As Long
    With ActiveSheet.AutoFilter.Range
        Nx = .Cells(.Rows.Count, 1).Row + 1
    End With 

Without using the With/End With statements the line of code would be like the following.

Nx = ActiveSheet.AutoFilter.Range.Cells(ActiveSheet.AutoFilter.Range.Rows.Count, 1).Row + 1
