¡firedog say:

Windows Live Mail doesn't delete attachments. Select one of the messages in the message list and press Ctrl-F3 to reveal the message source. Attachments will be at the bottom, separated from other parts of the message by a blank line followed by a line starting with at least two hyphens. They will normally have a Content-Type: declaration telling you what sort of attachment you're dealing with.

This is most likely nothing to do with servers and everything to do with clients. If a message with an attachment is sent using Microsoft Office Outlook or Microsoft Exchange Client and prepared using those clients' proprietary RTF format, then only one of those two clients will be able to decode the attachment. Some webmail services can decode them (Hotmail and Gmail, for example), but no other client. Which one did you use ('if I use another mail client other than Windows Live, the mail and attachment is accepted as expected')?

Viewing the message source will also probably tell you which client was used to compose and send the message. Outlook for example will include a header like
X-Mailer: Microsoft Outlook 14.0

Does any of this look likely?
