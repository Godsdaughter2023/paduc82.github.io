Hello mohankrishna_658,

I've recently had the same issue (be it due to another action, Windows Updates). The wininet.dll has been corrupted. Many programs and services depend on the file, so it does cause a lot of trouble.

This helped for me:

  1. There are 2 wininet.dll files that might be corrupted. Get a copy of both files from a working pc using the same version of Windows. Put them on a usb stick and make sure you don't mix them up! You can find both files in:
    1. C:\Windows\System32
    2. C:\Windows\SysWOW64
  2. Restart you're pc and rapidly press F8 till you see the boot menu. Choose Safe Mode.
  3. Login as an administrator, then open a command prompt (Start Menu, type cmd into the run field).
  4. Type:
    1. takeown /f C:\Windows\System32\wininet.dll
    2. takeown /f C:\Windows\SysWOW64\wininet.dll
  5. Type:
    1. cacls C:\Windows\System32\wininet.dll /G adminusername:F (replace adminusername with your admin account name)
    2. cacls C:\Windows\SysWOW64\wininet.dll /G adminusername:F (replace adminusername with your admin account name)
  6. Open an explorer window and go find C:\Windows\System32\wininet.dll and C:\Windows\SysWOW64\wininet.dll and rename it into wininet.dllx. Copy paste the good files from your usb stick. Again, don't mix them up!
  7. Restart your pc. The problem should be fixed.

For the people who want to know what they have done:

You can't normally replace or alter a system file. So how did we do it? We started in Safe Mode (2), took ownership of the files (4) and granted permission to change them (5).

I hope this helps for some of you! Good luck.
