Dear Eion,

Since march, I guess you found an answer. Nevertheless, I guess many other Linux users will have the same question.

A good thread to find a comprehensive answer :

http://askubuntu.com/questions/564496/trying-to-dual-boot-ubuntu-14-10-with-windows-10-last-one-not-detected

And here is how I solved the issue.

I installed first Windows 10, and then Ubuntu. GRUB did not find Windows and did not add it correctly to the menu.

First, use lsblk to find out on where is your windows located :

root@glitch-VPCF12S1E:/home/glitch# lsblk
NAME   MAJ:MIN RM   SIZE RO TYPE MOUNTPOINT
sda      8:0    0 119.2G  0 disk
├─sda1   8:1    0   500M  0 part
├─sda2   8:2    0 104.1G  0 part
├─sda3   8:3    0     1K  0 part
├─sda5   8:5    0  12.1G  0 part /
└─sda6   8:6    0   2.6G  0 part [SWAP]
sdb      8:16   0 465.8G  0 disk
└─sdb1   8:17   0 465.8G  0 part
sr0     11:0    1  1024M  0 rom  

I know that my Windows partition size is 100 GB => this is sda2.

**BUT** In fact, the working bootloader was on sda1 (I tried to add msdos2, did not work).

Once you know that, you can edit the grub customization file :

root@glitch-VPCF12S1E:/home/glitch# vim /etc/grub.d/40_custom

And add this at the end of the file :

menuentry 'Microsoft Windows 10' {
set root='(hd0,msdos1)'
chainloader +1
}

...in my case, hd0 because sd*a* is the first disk and msdos1 because the working bootloader is on sda*1*.

Finally, run update-grub2.

=> working.

Hope this helps...

As I suspected, two of the series, the lines on the primary axis, nave no X values. Their series formulas are

=SERIES('Break Even Model'!$A$12,,'Break Even Model'!$B$12:$G$12,3)

=SERIES('Break Even Model'!$A$11,,'Break Even Model'!$B$11:$G$11,2)

The two series on the secondary axis are

=SERIES('Break Even Model'!$A$9,'Break Even Model'!$B$1:$G$1,'Break Even Model'!$B$9:$G$9,1)

=SERIES('Break Even Model'!$A$10,'Break Even Model'!$B$1:$G$1,'Break Even Model'!$B$10:$G$10,4)

The chart has primary and secondary Y axes, and only primary X axis. The primary X axis uses the X values for the primary series, which are not included in the series formula. Since they are not included, Excel just uses 1, 2, 3, etc.

copy the X values

'Break Even Model'!$B$1:$G$1

from one of the secondary axis series formulas, and paste it into either of the primary axis series (either one, Excel fills in both), and now Excel knows what to plot on the primary X axis.

Problem solved.

karen_[01] say:

Hello Leo5659,

Welcome to Skype Community forum!

If you're having an audio issue, it's possible that your sound card driver is out of date or there might be some incomplete update in Skype. You may follow the steps below and let us know if the issue persists.

To update the sound card driver, follow these steps:

  1. Click Start, click Control Panel, and then double-click System.
  2. Click the Hardware tab, and then click Device Manager.
    If you are prompted for an administrator password or for confirmation, type the password, or click Continue.
  3. Click Sound, video, and game controllers to expand.
  4. Under Sound, video, and game controllers, double-click the entry for your sound card, and then click the Driver tab.

    Note If a Driver tab is not present, the entry that you clicked under Sound, video, and game controllers may not be the correct entry for your sound card. Try clicking any additional entries listed under Sound, video, and game controllers.
  5. Click Update Driver Software.
  6. Follow the Update Driver Software Wizard to update the sound card driver.
  7. You may also use plug in microphone and speaker and check if that will work.

Completely uninstall Skype.

  1. Quit Skype.
  2. Press Windows key + R on your keyboard at the same time and type %appdata% in the Run dialog box.
  3. Open the Skype folder and delete shared.xml file.
  4. In the same folder, look for your Skype name and open it then delete config file.
  5. Restart your computer.
  6. Re-launch Skype and make a test video call.
