In the VBA editor, insert > Userform. Now insert a Label into the userform.

 

In the code sheet for the Userform insert this:


Private Sub UserForm_Initialize()
Me.Label1 = Format(Range("A4").Value, "0.00")

In the code sheet for ThisWorkbook insert this:

Private Sub Workbook_Open()
UserForm1.Show (False)

Private Sub Worksheet_Calculate()
UserForm1.Label1 = Format(Range("A4").Value, "0.00")
End Sub

tom van stiphout say:

This should not be done with one table. You need at least these two:

Products

ProductID autonumber PrimaryKey

ProductName text(50) required uniqueindex

StockInventory

ProductID long integer required PrimaryKey

StockTakeDate datetime required PrimaryKey

Quantity integer required

See also: http://allenbrowne.com/AppInventory.html

scottgem say:

To expand on what Tom said, your products should have fields that describe your products. An ID, description and whatever else you need. Maybe supplier, maybe reorder level, there may be other items you need to record about your products.

The daily inventory numbers should be in a separate table that lists the ProductID as a foreign key, the date and the quantity.

The way this is usually presented is in a mainform/subform. The Mainform is bound to the Product table, the subform to the daily numbers table, probably in a continuous form. So you press the add record button on the subform. The productID is entered automatically, you just enter the date (you can default to current date) and the quantity.

What you are describing, btw, is a typical misunderstanding of newbies. When you open an Access form it generally displays the first record in the sort order. If you just start typing you are overwriting that record. You have to start a new record to enter new data,

ken sheridan say: For data entry I'd suggest that, using the tables described by Tom, you create a form, frmStockTake, in whose header you have an unbound text box, txtDate, formatted as Short Date.  The form should be in continuous forms view and based on the following query:

PARAMETERS Forms!frmStockTake!txtDate DATETIME;
SELECT StockInventory.*
FROM StockInventory INNER JOIN Products
ON StockInventory.ProductID = Products.ProductID
WHERE StockTakeDate =Forms!frmStockTake!txtDate
ORDER BY Product;

The form should include the following bound controls:

1  A combo box set up as follows:

ControlSource:    ProductID

RowSource:     SELECT ProductID, ProductName FROM Products ORDER BY ProductName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

2.  A text box with a ControlSource property of Quantity.

In the AfterUpdate event procedure of the txtDate control in the form header requery the form with:

    Me.Requery

When you enter a date in the text box, which you can do with the date picker, and press the Enter key, the form will be requeried to show any existing data for the selected date.  Where no data for the date has yet been entered the form will show only one empty new row.  To automatically enter the product list add a button to the form header and put this in its Click event procedure:

Const MESSAGE_TEXT = "Please select a date first."
Dim strSQL As String

If IsNull(Me.txtDate) then
    MsgBox MESSAGE_TEXT,vbExclamation, "Invalid Operation"
Else
    strSQL = "INSERT INTO StockInventory(ProductID, StockTakeDate, Quantity) " & _
                  "SELECT ProductID, #" & Format(Me.txtdate,"yyyy-mm-dd") & "#, 0 " & _
                  "FROM Products"

    CurrentDb.Execute strSQL, dbFailOnError
End If

Me.Requery

This will insert the product list with a zero quantity for each product, ready for you to overwrite the zeros with the current stock levels.

The existence of such a form does not preclude the existence of a form/subform such as that described by Scott.  The above provides an interface for entering or viewing the stock levels for all products on any one day, Scott's for editing/viewing the stock levels over all dates for any one product.  The two are complementary.

If you are unfamiliar with entering code into a form's or control's event procedures, this is how it's done in form design view:

1.  Select the form or control as appropriate and open its properties sheet if it's not already open.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new lines between these.

Finally, note that Tom has shown 'primary key' against both the ProductID and StockTakeDate columns in his definition of the StockInventory table.  This is very important, and means that the two columns make up the composite primary key of the table, not that it has two primary keys, which is not possible.  One other important thing you should do is create a relationship between the two tables on ProductID and enforce referential integrity in the relationship.
