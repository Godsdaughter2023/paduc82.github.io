talking moose say:

Part 1:

Mail for all POP accounts flows into the Inbox under On My Computer. You cannot remove this folder. It’s there permanently. You can hide the On My Computer folders by choosing Outlook menu -->  Preferences... --> General and enabling Hide On My Computer Folders.

Part 2:

Messages located in any folders under On My Computer are only on your computer. Nothing here syncs with mail server servers. Deleting items here will permanently remove them from your computer once you empty your Deleted Items folder.

The only way you can recover POP messages is to restore the entire identity from backup. You’ll need to determine when you last had the messages and then restore this folder from that date:

/Users/<your name>/Documents/Microsoft User Data/Office 2011 Identities/Main Identity

The “Main Identity” folder is the default name. If you’ve ever created a new identity with a different name then you’ll need to restore that folder instead.

Once restored, Outlook will rebuild its database the first time you launch it. This is normal.

FAQ: Microsoft Mathematics Add-in for Word and OneNote

The Microsoft Mathematics Add-in for Word and OneNote makes it easy to create graphs in 2D and 3D, calculate numerical results, solve equations or inequalities, and simplify algebraic expressions in your Word documents and OneNote notebooks.

 

Q: How much does it cost?

A: Mathematics Add-in for Word and OneNote is available as a free download, found here.

 

Q: How can I get support for the Mathematics Add-in?

A:Questions about using the Mathematics Add-in in Word should be asked in the “Working with Graphics and Inserts” forum on Answers. Questions about using the Mathematics Add-in in OneNote should be asked in the“OneNote: Sharing, Creating and Managing Notebooks”forum.

 

Q: Which versions of Word or OneNote work with the Microsoft Mathematics Add-in?

A:The Microsoft Mathematics Add-in is compatible with Microsoft Office Word 2007, Word 2010, and OneNote 2010. A full list of requirements is available on the download page.

 

Q: Can I install the Microsoft Mathematics Add-in if I have a previous version of the Math add-in already installed?

A:Yes.If you install the Microsoft Mathematics Add-in on a computer that also has Microsoft Math Add-in for Microsoft Office Word 2007 installed, Microsoft Mathematics Add-in treats the installation as an upgrade and replaces the older version of the add-in.

 

Q: Where do I find the add-in after I install it?

A: The add-in appears as the Mathematics tab in Word and OneNote.

 

Q: Why aren’t all the Compute and Graph actions available?

A: Your cursor must be in an equation or mathematical expression for the actions to be available. Also, Compute and Graph actions are available only when they can be applied to the selected expression.

 

Q: Can I edit a graph after I insert the picture in my document?

A: After inserting a graph, you can change it at any time. To edit the graph, double-click the graph and adjust the settings in the Graph dialog box.

 

Q: Can I use the Microsoft Mathematics Add-in with notebooks that are in the OneNote 2007 file format?

A:No. When you upgrade from OneNote 2007 to OneNote 2010, notebooks aren’t automatically updated to the new OneNote 2010 file format. The feature to insert math equations is available only in OneNote 2010 format notebooks. If you can’t insert equations, you may need to convert your notebook to OneNote 2010 format. For information about how to do this, see OneNote Help.

    This works great for the month and the year. But I cant get the MID numbers to pull the day of month correctly. it only pulls the first number. So from the example above it shows me 2/2/2018 but I want it to show 2/21/2018. The problem lies in here FIND(MID(A12,2,1),
    "123456789abcdefghijklmnopqrstuv") but not sure how to fix it. any help would be awesome! thanks

    justin

    ossiemac say:

    Hello Justin,

    Yes! the formula as is can only extract one digit for the day of month. The formula you posted actually returns an error because it is attempting to find an upper case B in a lower case string and FIND function is case sensitive. Should use SEARCH because it is not case sensitive and I changed it in my testing to the following.

    =DATE(2010+SEARCH(RIGHT(A12,1),"123456789abc"),SEARCH(LEFT(A12,1),"abcdefghjklm"),SEARCH(MID(A12,2,1),"123456789abcdefghijklmnopqrstuv"))

    However, I need to see some more examples of the date code that includes codes where the day of month is only one digit (ie. 1 to 9). If it is represented as 01, 02, 03 up to 09 then it is fairly easy to adjust the formula as follows. Note the inclusion of IFERROR because it will not find the zero in the string to search. Alternatively, the zero could be included in the string and subtract one from the search result.

    The formula uses code in cell B12 in lieu of A12. See the Screen shot below where the code is adjusted in column B to include a zero if not already included in the date code. 

    =DATE(2010+SEARCH(RIGHT(B12,1),"123456789abc"),SEARCH(LEFT(B12,1),"abcdefghjklm"),IFERROR(SEARCH(MID(B12,2,1),"123456789abcdefghijklmnopqrstuv"),0)&SEARCH(MID(B12,3,1),"123456789abcdefghijklmnopqrstuv"))

    However, if the code for Feb 9, 2018 is B98 (only 3 characters) then the string needs to be tested for the length of the code and insert a zero after the B if only 3 characters long. Easiest method is with an extra helper column beside the code column and use the following formula. Again, refer to the screen shot below.

    Formulas are the same for both B12 and B13 but the zero is inserted in B13.

    Formula in cell B12 (Insert zero not required):     =IF(LEN(A12)=4, A12,LEFT(A12,1)&"0"&RIGHT(A12,2))

    Formula in cell B13 (Insert zero required):      =IF(LEN(A13)=4, A13,LEFT(A13,1)&"0"&RIGHT(A13,2))
