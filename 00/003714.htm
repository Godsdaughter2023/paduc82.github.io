Try taking an resource's salary and dividing it by 2088 which is to total number of working hours in 2015 and assigning that as their hourly rate (assuming you have used the default calendar settings)  Example: A resource earning $100k would have an hourly rate of 47.89272 per hour (100000/2088 = 47.89272).  If you run the Cost / Budget report you will find that employee makes $99,994.32 a year if they are fully allocated for each working day.  I think the difference is due to a rounding issue on the application.

I don't think you will ever get a perfectly matching amount due to the rounding issue but you should get close.

Hi ITTeam

I was having the same issue though I don't have a local AD.

As part of my trouble shooting I found the same settings for the send as and access in the O365 admin centre(OAC) as well as the Exchange admin centre(EAC).

If you change the settings in the OAC, it is reflected in the EAC.  The reverse however does not appear to be true.

I ended up getting it working by removing the access settings for my shared account from the EAC and then adding them in the OAC.  

  • In the EAC, go to recipients>shared and select the account to edit.  Click the pencil icon to open the edit screen
  • In mailbox delegation, remove the user from both the full access and send as sections.
  • Save
  • Go back to the OAC
  • Select Groups>Shared Mailboxes
  • click the required shared mailbox to open the info panel
  • Click the edit to the right of "Members" and in the dialogue, add the required user
  • In the "Members" section, click "customize permissions"
  • Add the the "Read and manage"
  • Add to the "Send as" or "Send on behalf" depending on requirements.
  • Save

You should now be able to test.  You should also have the settings back in the EAC if you check.  The account will also appear in the users outlook.

Hope this helps.
