david-fink say:

Your formatting example displays 100503 as 10+05+03L, so in the following I will assume that is what you want and not the10+03+3L you stated. 

There are three solutions. 

The first solution is if your values are in one cell and you want to display this special format in a different cell. The second and third solutions are if you want the cell you type your value into to immediately display the special format. 

The third will only work if your inputs are integer numbers.

-----

First solution:

The trick is to create the output text you want as text with a formula.

I will first treat your two formats case separately, then combine them into one formula.

The solution will also depend on how your 0500 and 100503 are created. I put them in cells A1 and A2, with those cells Formated as Text:

Select cells, then menu

Format > Cells > Number > Text

If the cell format of A1 and A2 are not Text, as soon as you enter the value, Excel will change it, and the rest of this may not work.

To have cell B1 display the 0500 in cell A1 as 5+00L, use this formula in B1:

  =TEXT(A1,"##+##""L""")

Note that inside a quote, the quote character is entered as two quotes.

To have cell B2 display 100503 in cell A2 as 10+05+03L, use this formula in B2:

 =TEXT(A2,"##+##+##""L""")

It appears you want to have either 0500 or 100503 in the source cell and have the formula select the correct format. If the choice is made depending on whether the number of characters in the source cell is 4 or 6, then this nested IF will work, choosing the first form for length 4 and the second for length 6:

 =IF(LEN(A1)=4,TEXT(A1,"##+##""L"""),IF(LEN(A1)=6,TEXT(A1,"##+##+##""L""")))

If you have Office 365, there is a new IFS function to simplify nested IF statements.

Instead of 

  IF(test2(value1,IF(test2,value2,IF(test3,value3,other))))

and having to keep track of where the IF functions end the the correct placement and number of ")", this new IFS function can be used

  IFS(test1, value1, test2, value2, test3 value3)

Then the above simplifies to 

 =IFS(LEN(A1)=4,TEXT(A1,"##+##""L"""),LEN(A1)=6,TEXT(A1,"##+##+##""L"""))

-----

Second solution:

(This is a slight variation of the one given above by Andreas Killer.)

If you want to type 0500 or 100503 into a cell and have THAT cell display your two formats, a solution is to use Conditional Formatting. The strategy is to set the format of the cell to one of the formats, and set a condition that chooses the other format.

You need to have both custom formats already created:

  Format > Cells > Custom

and in the Type box, put 

  ##+##"L"

Click OK, then do it again for 

##+##+##"L"

Select your cell and set its format to ##+##"L", the one for length 4.

Now set the conditional formatting:

  Menu Format > Conditional Formatting...

  Show formatting rules for: Current Selection

Click the + to the left of "Edit Rule..." to create a rule,

In the New Formatting Rule dialog,

  Style > Classic > Use a formula to determine which cells to format

If we are working on cell B2, type in the formula

=LEN(B2)>4

Click OK

  Format with > Custom Format...

That opens the Format Cells dialog, and pick for format for length 6:

  Custom > ##+##+##"L"

OK

OK

OK

-----

Third solution:

If you want to type 0500 or 100503 into a cell and have THAT cell display your two formats, if your inputs are actually integer numbers, and the format switch occurs if the number is greater than 9999.

Excel supports conditional custom number formats. Number formats can have three sections separated by semicolons. Normally the first section is the format for positive numbers, the second section for negative, and the third section for any other entries (zero or a non-number), but by using  [condition value], you can have the three sections serve any purpose. The condition can be any numeric comparison (<, >, =, >=, <=, <>), and the value can be any number. 

Here we will switch formats if the cell value is less than 10,000.

Format > Cells > Custom

and in the Type box, put 

[<10000] ##+##"L"; ##+##+##"L"

jay freedman say:

There's a well-buried option: Go to Office button > Word Options > Trust Center > Trust Center Settings > Privacy Options > "Remove personal information from file properties on save".

According to the help, "The  Remove personal information from file properties on save (disabled)  option is available only if you are working with a document that was created in an earlier version
of Office, and you used this option in that version to remove personal information. To remove personal information from this document, click Inspect Document."

I suspect that in the document you have, this option has been enabled -- possibly even if the conditions stated by the help topic don't apply, and even if the option is still disabled. To test this,
press Alt+F11 to open the macro editor, press Ctrl+G to open the Immediate Window, and type or paste this line:

?ActiveDocument.RemovePersonalInformation

With the cursor still on that line, press Enter.

If the response is "False" then my guess is probably wrong. If it's "True", though, change the line to this and press Enter again:

ActiveDocument.RemovePersonalInformation = False

Then try adding more Tracked Changes items, and see what happens when you save.
