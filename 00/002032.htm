roady [mvp] say:

Exchange ActiveSync is a protocol intended for (smaller) mobile devices to connect to an Exchange server or to mail server that has implemented Exchange ActiveSync. Outlook doesn't support Exchange ActiveSync but offers Exchange account support directly on-site or via RPC over HTTP (Outlook Anywhere) for off-site access.

If your company doesn't support Outlook Anywhere, then you'll need to make a direct connection to the Exchange server. If you are off-site, this can still be achieved via a VPN connection. If you company also doesn't offer that, making a connection via Outlook isn't possible via any built-in means.

You could try setting up a forward to an external address so you can still collect your mail from elsewhere but again; if it is company policy not to support email access via Outlook, why go against it? jeeped say:
Can you please add (Microsoft) minimum ifs (=MINIFS) and maximum ifs (=MAXIFS) to Microsoft Excel?

Native MINIF(…) and MAXIF(…)  (and probably MINIFS(…) and MAXIFS(…)) functions will likely be added in a future version as an equivalent formula is a very common request. You can make suggestions for Microsoft Office improvements on the Office web site here:

Office Feedback

If the interim, here are some equivalent standard and array formulas.

           

Array formula¹ in D2,
=MIN(IF($A$2:$A$99="Cat",$B$2:$B$99))
Array formula¹ in E2,
=MAX(IF($A$2:$A$99="Cat",$B$2:$B$99))
Array formula¹ in D5,
=MIN(IF($A$2:$A$99=1,$B$2:$B$99))
Array formula¹ in E5,
=MAX(IF($A$2:$A$99=1,$B$2:$B$99))
Standard formula in F2,
=MAX(INDEX(($A$2:$A$99="Cat")*($B$2:$B$99),,))
Standard formula in F5,
=MAX(INDEX(($A$2:$A$99=1)*($B$2:$B$99),,))

¹ Array formulas require Ctrl+Shift+Enter↵ rather than just Enter↵. Note that a psuedo-MAXIF(…) can be created using MAX(INDEX(…)) as a standard formula, but this methodology does not apply for MIN(INDEX(…)).

More information is available in my pair of local Wiki artlicles.

MINIF, MAXIF and MODEIF with Array Formulas
MINIF, MAXIF and MODEIF with Standard Formulas

vaidya darpan msft say:

Hello AgnibhaNath,

Have a good day. Thanks for your post in forum.

Regarding your description, we cannot set any rule about spam in Outlook because in Outlook we can only send or receive email. In Office 365 spam is detect from the SCL (Spam Confidence Level). You can refer this article for SCL: https://docs.microsoft.com/en-us/office365/securitycompliance/spam-confidence-levels

Also, for protection email message, we can make spam filter policy from Office 365 Admin Center under Protection. By default spam filter policy for bulk email action is 7 and we can change as per our requirement. We can also create new spam filter policy.

For Bulk email action: we must need to enable "Mark email as spam" option under Bulk email, after that it will work for spam.

Following are the steps and screenshot about default spam filter policy in Office 365 Admin Center.

•                     Login Office 365 portal.

•                     Go to Admin Center.

•                     Click on Exchange option.

•                     Go to Protection.

•                     Click on Spam filter.

•                     Double click on Default.

You can also refer the article about: Configure your spam filter policies.
