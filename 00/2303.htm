bob jones aka: cybertaz say:

You're referring to a gradient, right? I'm afraid that isn't going to happen. Unlike Watermarks & Borders, Background Colors are not intended for printed output.

They were added when Microsoft was trying to convince the world that Word could effectively be used for web design :-) They are fine for viewing on screen, but the Gradients simply do not translate to the printer. Since generating a PDF is a print process, the same result occurs as if sent to a physical printer.

If you need a gradient background, draw a rectangular Shape in the Header/Footer layer & apply a gradient fill to the rectangle. It will most likely not run edge-to-edge in the PDF, but at least the gradient effect will be retained.

kergga say:

 

I can give you some pointers, but I can't give you exact formulas because of your data format (discussed below).

 

I would sort by driver, then date, then time.

I would then add a helper column (column G), with this formula:

=IF(A2<>A1,1,(D2-D1)+(E2-E1))

This gives you a default value of one trip the first time a driver is seen, then all subsequent values are calculated to see how much time has elapsed.

 

Then I would add this formula, on the first row that each rep appears in your data (in a second helper column, column H)

First rep shows up starting in row 2:

=COUNTIFS(A:A,A2,B:B,B2,G:G,">.0104")+1

The +1 is because the first row on which any rep shows up will have an elapsed time/date of zero, so we are adding one for that row.

.0104 is something like 14 minutes, 59 seconds .... so any values over that are your elapsed 15 minutes.

 

Some things to consider; your date format appears to be dd/mm/yyyy. My PC is set up as mm/dd/yyyy so the elapsed time formula on my PC has trouble with resolving the numbers, but it should be ok on your PC since that is your date format.

 

There is probably an opportunity here to keep your date and time all in one column, but I don't know enough about how you collect and enter your data to really give a strong recommendation.

Hi,

 

I have answered a similar question at this link on my site - http://www.ashishmathur.com/count-uniques-with-conditions/.

 

The specific answer to your question would be:

 

1. Suppose your data is in range B2:G40 (headings are in row 2)

2. Copy B2:C2 to cell J2:K2

3. Select cell J8 and go to Data > Advanced > Copy to another location

4. In the List range, select B2:G40

5. In the copy to box, select range J2:K2

6. Check the box for Unique Records only and click on OK

7. In cell L3, enter the following array formula (Ctrl+Shift+Enter) and copy down

 

=SUM(IF(($B$3:$B$40=J3)*($C$3:$C$40=K3)*(SUMPRODUCT(1/COUNTIF($D$3:$D$40,$D$3:$D$40))),1/COUNTIF($D$3:$D$40,$D$3:$D$40),0))

 

By the way, if you were using Excel 2013, this result would have simply come with using a Pivot Table.  In Excel 2013, if the Data model option is selected, then one can use Distinct Count in the Value area section of the Pivot Table.  My answer from the formula above tallies with the result of the Excel 2013 generated Pivot Table.

 

Hope this helps.

By default, Internet Explorer automatically detects proxy settings.

b)   Click the Tools button, and then click  Internet Options.

e)   Click Apply and  OK
