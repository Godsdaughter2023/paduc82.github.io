rohn007 say:
Yes, you can run multiple versions of Office, with a little work.

I currently have 2003, 2007, 2010 and 2013 installed on my old laptop.

The "best" way to ensure compatibility back that far is to write the macros in Office 97, then import them into 2007.  Even so you may have some problems because that is a large span and there are some substantial changes in VBA support between those 2 versions.

Dual Install - Working with Multiple Versions of Office

https://office.microsoft.com/en-us/word-help/keep-earlier-versions-of-office-programs-when-installing-office-2010-HA102597134.aspx

This tip describes the specific steps and choices during the 2010 custom install. After you have the multiple versions installed you still have some additional steps to do. They are described in this tip:

http://pubs.logicalexpressions.com/pub0009/LPMArticle.asp?ID=762

With multiple versions installed, there is one issue you can’t avoid, Default Windows File Associations. Only one app can be used to open the files by double clicking in Windows Explorer. You have to pick which one.

FYI: NoReReg key works with Office 2010 / 2013 installed on Windows 7 and 8 (I tested it myself Rohn007)

Note: in user accounts, when you log in to do the NoReReg hacks, DO NOT start REGEDIT using “Run As Admin”. You will be editing the wrong “Current User” key <grin>

 

Office 2010 configures each time you start

http://support.microsoft.com/kb/2528748

In Method 1, this tip has a Fix it, 50780, that applies the required Registry fix.

Thank you for your reply, whilst possible it seems silly that this is the only way to write backwards compatible macros. Still I have older versions of Excel and will install one if that is the only road to go.

 

rohn007 say:
It is not the only way, but it is the easiest way to quickly test them in both environments before sharing them with others.

In theory, you could do it strictly on paper. Investigate and learn the differences between the 2 and write to the lowest common denominator. But you will still have to test them at some point.

If you want to ensure complete isolation between the 2 versions, use a virtual environment for one of them. If you were dealing with 2010 you could go with a click to run installation (if you could find one). Otherwise you can use a tool like VirtualBox or VMWare (or others).
