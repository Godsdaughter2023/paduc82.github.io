wkrasl say:

Excel CSV Import error for time-formatted data

When I import a CSV file exported by Garmin, Excel is often (but not always) incorrectly reading data in "mm:ss" format (mm minutes and ss seconds) as if it was entered as "mm:ss:00" and then interprets the data as mm hours, ss minutes, and zero seconds.  HOWEVER it does so inconsistently, and what I have observed is that the error appears to occur only when the CSV data is between "24:00" and "59:59" inclusive. 

In other words, when the field is 58:01 or 26:02 it treats it like hours and minutes and zero seconds, but when the time is 18:15 or 13:59 it treats it correctly as  minutes and seconds.  It also correctly treats time of 1:00:00 or greater correctly as hours, minutes, and seconds.  In any case, Excel stores the data as the number of days; e.g. 56:08 is stored as 2.34 days instead of 0.03898 days.

  5. Delete the old column C.
  6. Repeat above for next time column but substitute the correct column in the formula.

hi off2work,

yes, we can use term sets as the refiner in search result type, but we still need to create managed metadata columns to get use of the term sets:

1. go to site settings > term store management (under site administration) > hover your mouse above site collection - xxxx.sharepoint.com-sites-xxx > click the down arrow and select new term set.


2. create a managed metadata site column using the term set created above. ( select yes under require that this column contains information:)


3. add this column to a library and select add to default view. upload the files and fill the column value for each file.


4. map a crawled property – ows_columnname to one of the refinablestring managed property, say refinablestring03. (sharepoint admin center > search > manage search schema)


5. manually reindex this site (site settings > search and offline availability under search > reindex site) and wait for about 15 minutes.

6. go to the search site (https://domain.sharepoint.com/search) > settings > edit page > edit the refinement web part and then click on "choose refiners…" button > select refinablestring03 > add > click move up to to put this new navigator at the top of the "selected refiners" list in the refinement web part, the countryname refiners will be displayed for search results and available to filter the results to the appropriate selection > click apply > save.

for steps 4-6, we can also refer to this blog: refiners for managed metadata columns in sharepoint online.

randlesc say:

Autofilter does not recognized entire column

I have a worksheet that I export from Access.  After exporting, I apply an autofilter; however, the autofilter only recognizes about 1500 of the 5000 records.  I can find anything different about the second portion of the report that would cause it to do this.

 

Any suggestions?

ron coderre say:

If you mean that the AutoFilter dropdown list does not display *every* unique value, that is by design. You wouldn't want to scroll through tens of thousands of items in a list, would you. The AutoFilter will still filter ALL of the records.
Note: Excel 2007 displays the first 10,000 unique items...are you using Excel 2003?

What you need to do for "unlisted" items is:

• Click the dropdown list
...Select: Text Filters (or Number Filters...whichever displays
...Enter the value you want to find...eg Equals Frisbee
...(you can also use wildcards (? and/or *) to find items that contain a value)
...Click: OK

The other possibility is that your data has a blank row, causing Excel to assume your data list ends at that row.
If that's the case, either
• Delete the blank row
or
• Select ALL of your data list before you engage the AutoFilter.


Does that help?
