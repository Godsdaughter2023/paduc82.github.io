ken sheridan say:

The other line of code which Gina suggested would either look up the maximum score value for the criterion in question by calling the DLookup function, or will assign the maximum score value from a second column in the combo box's RowSource query.

 

Another approach would be to do the work in the form's query like this:

 

SELECT Results.*, MaxScore, ScoreType

Score/MaxScore*100 AS ScorePercent

FROM (Results INNER JOIN Criteria

ON Results.CriteriaID = Criteria.CriteriaID)

INNER JOIN ScoreType

ON ScoreType.ScoreTypeID = Criteria.ScoreTypeID;

 

In the AfterUpdate procedure of the CriteriaID combo box you can then enable/disable the score text box, and set the value of the PassFail control:

 

Select Case ScoreType

    Case Is = "PassFail" Then

    Me.PassFail.Enabled = True

    Me.Score.Enabled = False

    Case Is = "PointBased"

    Me.PassFail.Enabled = False

    Me.Score.Enabled = True

    Me.PassFail = False

    Me.Score = Null

    Case Else

    Me.PassFail.Enabled = False

    Me.Score.Enabled = False

    Me.PassFail = False

    Me.Score = Null

End Select

 

In the AfterUpdate event procedure of the Score control:

 

Me.PassFail = (ScorePercent >= 70)

 

However, there is a design deficiency in this model; the pass threshold of 70% is being encoded in the above line of code.  This is data.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1).  This requires that all data be stored as values at column positions in rows in tables, and in no other way.  The value of 70 should therefore be stored as a value in a column in a table.  It does not appear to be functionally determined by the key of any of your existing tables so should be in a one row/one column table of its own:

 

PassPercents

....PassPercent (PK)

 

This table can be constrained to a single row, but the technique is a little advanced, so rather than confuse you any further I'll let that lie for the moment.  While the value in the table will record the current pass threshold, one should not rule out the possibility of it changing over time, so rather than simply looking it up, it should also be stored in a PassPercent column in the Results table, and a value assigned to this in your form's BeforeInsert event procedure:

 

Me.PassPercent = DLookup("PassPercent","PassPercents")

 

The code In the AfterUpdate event procedure of the Score control would then be:

 

Me.PassFail = (ScorePercent >= Me.PassPercent)

ossiemac say:

On my real form, I have a large number of controls. For this approach to be effective, will I have to capture the _Enter event for every control on my form

No. You do not have to do that.

Some notes in case you are not conversant with TabIndexes (Order of selecting controls using the Tab key).

TabIndexes start at zero and are set in the Control Properties.

In the Exit event of the ListBox, set the focus to the next TabIndex control after the ListBox TabIndex.

In the Enter event of the control to which focus is set, insert the code to hide the ListBox.

It then does not matter how you exit the ListBox, it will always go to the same control. The disadvantage is that if you really want to go to another control then you will need to click it again.

I have uploaded an example workbook to the link below. The file is zipped so download and unzip to extract the workbook file.

Link to example workbook

 I have entered the default text in the textboxes to display their TabIndex. TabIndex 2 is assigned to ListBox1.

The Exit event of the ListBox sets the focus to the TextBox with TabIndex 3 (Next one after the ListBox). The code to hide the ListBox is in the Enter event for the TextBox with TabIndex3.

Irrespective of how you exit the ListBox , the SetFocus command in the ListBox Exit event sends it to the next control. Even if you click the Command Button to exit the ListBox, it still goes to the next control and the code associated with the command button is not executed. It is then necessary to click the command button again to execute the code associated with the command button.

If you do not set the focus to the next TabIndex control and the code to hide the ListBox in the Enter Event of that control then the VBA code errors.
