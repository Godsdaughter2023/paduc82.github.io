The problem may be the Distributed Transaction Coordinator Service (MSDTC).

If you are not using distributed transactions or don't even know or care what that means, see how your ntbackup behaves with DTC disabled:

Click Start, Run and in the box enter:

services.msc

Click OK to open the Services applet and locate the Distributed Transaction Coordinator Service, right click, Properties and change the Startup type to Disabled.  Click OK to save the adjustment, reboot and see how your ntbackup behaves now.

mem-tek say:

Did you install any new antivirus or backup software, or perhaps add another hard drive recently, or did you have a couple of USB drives plugged into the computer when the shadow copy failed in backup? Basically, all of these things and other things each require an instance of low level I/O access to your computer's hardware, yet by default Windows XP has its IRP Stack Size set to 15. The first safe number to set this to is 39 (27hex). Here is a regfile which will do this for you:

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\lanmanserver\parameters]
"IRPStackSize"=dword:00000027

Copy and paste the above into Notepad. Then save the file as:

Change IRP Stack Size To 39 (27hex).reg

Then right-click on the saved file and select Merge to merge it into XP's registry. You will have to reboot your computer for the new IRP Stack Size setting to take effect. This should prevent any future problems with creating shadow copies in windows Backup or other third party backup programs for XP.

Over the years and whenever I installed XP, this was the first thing which I did before installing additional hard drives or programs, or setting up networking and mapped drives.
