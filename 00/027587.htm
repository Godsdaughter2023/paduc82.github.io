The STYLEREF field is used to do that.

In your case { STYLEREF "Heading 1" }

john mcghie say:

Sadly, Microsoft has not documented the specifics of this mechanism for years, so no, we cannot point you to "documentation".

Nobody has mentioned the Update styles on opening setting yet, so I will.

It's hard to find in Word 2016, because it's on the Developer tab under the Add-ins button. That pops up the "Attach template dialog" and you will see the "Automatically update document styles" checkbox immediately below the "Attach" button.

If that box is checked, the entire document style table is over-written by the one in the attached template each time the document is opened.

Now:  Here's the thing:  Some years ago, Word was modified so that setting would not "stick" if the attached template was Normal.dotm.  However, documents older than that change could hold that setting despite having only the global Normal.dotm as their template.

And as I say: each tim the document is opened, all of the styles are reset to whatever is in the template.  The behaviour you are describing is exactly the way this works.

It is possible that certain choices of document "Theme" could also do this, but I have not experimented with this because I don't use themes.

As the others have mentioned, it is important to control the cascading of properties within your style set.  I use the built-in styles almost exclusively (it's a good practice in corporate environments where other users will not find or understand custom styles).  I set the formatting of those styles according to the corporate standard.  One of the things I set is the "Based on" property.  I tend to create three "sets" of styles: "Headings", "Numbering" and "Body".  I disable the link to Normal style for the first style of each set, then base the rest on the one above.

I suggest you check the setting of Automatically update document styles.

Hope this helps
