Please help me.

 

 

 

 

 

joeu2004 say: Tobie wrote:
What do you mean by "The multiplication (*) converts TRUE and FALSE in the first term to 1 and 0."? 

Sorry, I don't speak excel or mathmatics that well.  Everything I've learned in excel has been self taught.


Perhaps it is better to ignore my comment.  If you understand the formula without the answer, that really might be all you need to know.

I mentioned it because it is a common follow-up question.  I also mentioned it in anticipation of the following alternative suggestions:

=SUMPRODUCT(--($F$2:$F$15=A2),$B$2:$B$15+$D$2:$D$15-$E$2:$E$15)

=SUMPRODUCT(--($F$2:$F$15=A2)*($B$2:$B$15+$D$2:$D$15-$E$2:$E$15))

Nothing wrong with the first alternative.  In fact, use it if it makes more sense to you.

But in the second alternative, the double negative is unnecessary and just plain silly.  The multiplication (any arithmetic operation) is sufficient.

(See below for an explanation of the double negative.)

-----

Let's break down the original formula.

=SUMPRODUCT(($F$2:$F$15=A2)*($B$2:$B$15+$D$2:$D$15-$E$2:$E$15))

$B$2:$B$15+$D$2:$D$15-$E$2:$E$15 returns an array of the row-wise calculations that you want, namely B2+D2-E2, B3+D3-E3, etc.

($F$2:$F$15=A2) returns an array of TRUE or FALSE for each row-wise comparison, namely (F2=A2), (F3=A2), etc.

So ($F$2:$F$15=A2)*($B$2:$B$15+$D$2:$D$15-$E$2:$E$15) multiplies an array of TRUE and FALSE times an array of numbers, returning an array of the products.

For many people, that is bewildering.  "How can we multiply a number times a logical value?"!

The answer is:  in any arithmetic expression, Excel translates TRUE into 1 and FALSE into 0.

Suppose F2=A2 is TRUE, but F3=A3 is FALSE.  The SUMPRODUCT formula computes 1*(B2+D2-E2) plus 0*(B3+D3-E3), etc.

Anything times 1 is itself; so the first term is simply B2+D2-E2.

Anything times 0 is zero; so the second term is zero.

Likewise, in the alternatives above, --($F$2:$F$15=A2) is an arithmetic expression.  So if F2=A2 is TRUE, that is equivalent to -(-TRUE).  Excel translates -TRUE into -1; and -(-1) is 1.

Hope that helps.

bernie deitrick say:

"Don't move or size with cells" means that it stays with the cells and doesn't move or change size if you delete or insert or resize rows or columns beneath it. It still scrolls with the displayed cells.

Either use freeze panes and place the textbox on a frozen area, or create a userform with your textbox, then show it using in modeless mode:

 UserForm1.Show vbModeless
