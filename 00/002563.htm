Web Data Query - Mac Excel 2011 / Microsoft Excel 2010 - help - Microsoft Community

stuht say:

Web Data Query - Mac Excel 2011 / Microsoft Excel 2010 - help greatly appreciated!

I have written the macro below, created in Microsoft Excel 2010, to pull back exchange rate data from xe.com. My issue is that when the spreadsheet is opened using Mac Excel 2011 the macro falls over.

Question: Is there any way in which this code can be adapted to make it cross compatible for both Windows and Mac OS?

One final point of note, in case of relevance, is that I have my actual code set up so that the user can input the required date in a cell (23 January 2013 in the below example). This however is not the cause of the compatibility issue, but is a key requirement i.e. the URL is not static but driven by user input of the required date.

I have sought to look at all the other similar questions posted and the responses given, but cannot find a solution which would work here.

Many thanks.

Andrew

----------------------------

Sub ExchangeRateDownload()

With ActiveSheet.QueryTables.Add(Connection:= _
        "URL;http://www.xe.com/currencytables/?from=USD&date=2013-01-23", Destination:=Range("A1"))
        .Name = "?from=USD&date=2013-01-23"
        .FieldNames = True
        .RowNumbers = False
        .FillAdjacentFormulas = False
        .PreserveFormatting = True
        .RefreshOnFileOpen = False
        .BackgroundQuery = True
        .RefreshStyle = xlInsertDeleteCells
        .SavePassword = False
        .SaveData = True
        .AdjustColumnWidth = True
        .RefreshPeriod = 0
        .WebSelectionType = xlEntirePage
        .WebFormatting = xlWebFormattingNone
        .WebPreFormattedTextToColumns = True
        .WebConsecutiveDelimitersAsOne = True
        .WebSingleBlockTextImport = False
        .WebDisableDateRecognition = False
        .WebDisableRedirections = False
        .Refresh BackgroundQuery:=False
        
    End With

End Sub

bobgreenblatt say: Well, the short answer to your question is: NO, you can't make this code cross compatible for both windows and the Mac, because the Mac does not support query tables. However, it does support web queries. You can use conditional assembly to use either the Mac code or Windows code; or use a runtime determination like application.operatingsystem to see which set of code to run.

This may give you some help:

rick song msft say:
Hello Saleaway,

 

I have checked the information you provided. The postmaster@domain.com is auto created by Exchange Online to send you emails like NDRs. In your case, what I can see is: The spam sender sent email to your organization with malwares using your company’s domain. But the sender’s address is an invalid address. The anti-malware filter detects it and blocked the email, delete the attachments. However, the message already be sent to your organization. The Anti-malware filter used postmaster@domain.com to send you a message to inform you that the message was blocked.

 

My suggestion is that you can block the sender’s IP. In the message trace results, you can see the spam sender’s IP address.

 

Regards,

Rick

Hello Saleaway,

 

Thanks for your prompt reply. it is difficult to confirm whether the user account is hacked. In this case, the spam senders may send the spoof emails using your domain.

 

In my opinion, you can try the suggestions below to enhance the security.

1. Block the sender’s IP address.

3. You can also choose do not send notice to external senders who sent malware in Anti Malware setting if you want.
