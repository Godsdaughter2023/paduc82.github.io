Bug in Word/Sharepoint integration: Opening document in - Microsoft Community

chris ten den say:

Bug in Word/Sharepoint integration: Opening document in Read-Only mode with DIP open modifies the document.

Issue:

Date (not DateTime) metadata are shifting date on their own when the Document Information Panel is open and the user opens the document in Read Only mode.  Version History is updated reflecting the erroneous change.

Background Information:

There is a known bug when opening a document amongst two timezones.

  • Open a new document using a field that has a Date only Metadata field and is shown on the document.
  • Fill in the Date, Save the document.
  • Have another user (in a time zone more eastern than yours) open the document and modify and save.
  • Reopen the document and the date will have changed back a day.

This is because the Date metadata does have a time portion but is never shown to user.  Opening in another timezone sees the data as a prior day and updates the document with that date.

Reproduction Steps:

In a Sharepoint list, where the Document Information Panel is enabled on document load:

Open a new document using a field that has a Date only Metadata field and is shown on the document.
Fill in the Date 2018-01-10

Save document
The date shown in the SharePoint list where the document is saved shows 2018-01-10
Doc Version History shows only the 1st version. (So far so good)

(You can repeat this step any number of times)
User opens document in Read-Only mode
The date shown in the document is 2018-01-10.
User closes document.
The date shown in the SharePoint list where the document is saved shows 2018-01-10.
Doc Version History shows only the 1st version. (So far so good)

User opens document in Edit mode
The date shown in the document is 2018-01-10.
User makes a small change to document
User saves document.
The date shown in the SharePoint list where the document is saved shows 2018-01-10
Doc Version History shows only the 2 versions. (So far so good)

User opens document in Read-Only mode

The date shown in the document is 2018-01-10.

User closes document.

The date shown in the SharePoint list where the document is saved shows 2018-01-10.

Doc Version History shows 3 versions. (This is very wrong!)  The latest version shows a change to the date.  (This happened because of the Date issue in the background)

User opens document in any mode

The date shown in the document is 2018-01-09.  (WOW!)

Work Around:

I know this is related to the DIP, so the quick fix is to turn it off.

The real question is why does the DIP write back to the document in read-only mode? 

I know you can't change the metadata in the DIP during read-only, but the above steps does make it write back.
