As Bob mentioned, you can only use the workgroup administrator and User Level Security with a .mdb file.  You won’t be able to use User Level Security with an accdb file or an adp file.  In an adp file, the security on your SQL Object (tables, stored procedures, function, etc.) will be enforced by your permissions on the SQL Server itself.  Then if your form or report is based off this data on the SQL Server and they don’t have permissions to it, they will receive an error when trying to open the form or report.

You can NOT have both Norton and MSE installed at the same time on the same computer.  This will cause conflicts, problems, freezes, slowdowns, and make neither product work to it's full capacity.  Instead of more protection, you will actually have less.  I strongly suggest switching to MSE.  To do that, you need to unininstall and then remove Norton with the special Norton removal tool (https://www-secure.symantec.com/norton-support/jsp/help-solutions.jsp?lg=english&ct=united+states&docid=20080710133834EN&product=home&version=1&pvid=f-home).

You MUST use the cleanup/removal tools to get rid of these programs properly as the normal uninstall process leaves remnants behind that cause problems with any other AV product you install later including MSE.  This means not only current ones but any EVER (including free trials you may never have used) or done years ago and possibly forgotten. Other programs "package" deals like this as well with this or other products and if you've recently done a clean install (a restore to factory conditions) you may have also re-installed trial software that you long since forgot was there.

-Make sure that the Windows Firewall is enabled. No other firewall truly offers better protection or is needed. This one is perfectly fine.

malabui say:

How do I insert page 'x of y' and 'File name & Path' into my Footer?

This used to be easy to do in previous versions of Word as I could just insert it from a list of options, but it seems to be impossible in this version.

beth melton say:

There isn’t a building block for the filename and path. You can either download a template and copy the building blocks that have already been created to your personal building blocks and use it or insert the field manually.

To download the template and follow the directions to copy the building blocks for the file name and path to your personal building blocks:
http://office.microsoft.com/en-us/templates/TC300002951033.aspx

Then to insert it at a specific position, place your insertion point in the desired location in the footer, right-click the building block in the Header or Footer gallery and then click Insert at current document position.

To insert the field manually:

  1. Press Ctrl + F9 and inside the field braces type: FILENAME  \p
  2. Press Alt + F9 to toggle off the field codes
  3. If the file name and path doesn’t display, select the field and press F9 to update it. (Or right-click the field and then click Update Field)

Page X of Y is an available building block. However, to insert it in a specific location, either use the Page Number/Current Position gallery, or find it in another gallery and use the right-click, Insert at current document position method.

 

~Beth Melton Microsoft Office MVP

myall_blues say:

I am not sure how you intend to 'pick' your bundle of items, but if for example the total weight of those items was in a cell then VLOOKUP with the [range_lookup] parameter set to TRUE will do what you want.

VLOOKUP with TRUE looks up your value and finds the nearest value that is less than or equal to your value.

So for example with your total weight in G2, then the shipping cost in H2 will be given by the formula

=VLOOKUP(G2,E3:F29,2,TRUE)

If you add an additional value at the top of your list with weight 0 (zero) and cost $2.12, it will handle the values where the weight is less than 1.
