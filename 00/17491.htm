Should I be using an unbound or bound form?

tom van stiphout say:

An unbound subform is not possible, so there is your answer.

Your statement "if two users are assigned the same record" conflicts with "especially not be able to view other user data", or is it really possible that multiple users are assigned the same record? What is your database design in this area?

scottgem say:

Unbound forms certainly have their place, but this isn't it. Access was designed to be multi-user so it includes record locking that will manage access to individual records. In an unbound form, the second submission will overwrite the first.

You can't link a subform in datasheet view to an unbound mainform.

Have you split the database? Each user will have their own local front end, correct?

Using a datasheet view how are you going to prevent seeing historical data?

There seems to be a lot of questions bout your design.

daniel pineault say:
Just an FYI. although technically feasible, a standard Access database (FE & BE) probably will not run smoothly with 100 users.  Typically when you are facing this number of users it is seriously time to start thinking about SQL Server, ... for the back-end.

scottgem say:

I agree with Daniel, however, that depends somewhat on concurrency. You can have more than 100 users but if only 15-20 will be logged in at the same time, that shouldn't be an issue. Your statement; (let's just say 100 users at a time) will be likely be keeping a single form open throughout the day" would indicate this isn't the case. So you probably should consider migrating to SQL Server for your back end.

I would be inclined not to let users sit idle in the app all day. You can have them forced off after a time.

tasos_k say:

Hi,

try this sample,

in your workbook:

sheet1, sheet2, sheet3,....sheet10

sheet1 tab always visible (first sheet tab, no matter the name)

in sheet1, in cell B1 drop down list...sht names: sheet1, sheet2,......,sheet10

select a name from list

double click on ThisWorkbook (in vba) and paste in the following:

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
On Error Resume Next

If Not Intersect(Target, Range("B1")) Is Nothing Then

With Sheets(1) '<< first sheet
.Range("B1").Locked = False
'.Range("B1").FormulaHidden = False
.Protect "1230" ', DrawingObjects:=True, Contents:=True, Scenarios:=True
End With
For x = 2 To Sheets.Count
Sheets(x).Protect "1230"
Sheets(x).visible = xlSheetVeryHidden '<< visible only via vba
Next

With Sheets(Sheets(1).Range("B1").Value)
.visible = True
.Select
End With

End If

End Sub

note: password is ..."1230"
