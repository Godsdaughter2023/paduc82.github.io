ken sheridan say: Technically the Me keyword returns a reference to the current instance of the class in which the code is being executed, but think of it simply as a short way of referring the current report.

The HasData property is a property of a report object, which in this case is the subreport's source report object.  It is not a property of any of the objects in the parent eport, so you won't find it anywhere in the properties sheets; it is used solely in the VBA code.

The code needs to go in the Format event procedure of the relevant section in the parent report, so:

1. In report design view select the section in which the subreport is located (probably the detail section) and open its properties sheet if it's not already open.

2. In the Event tab select the On Format event property, and click on the 'build' button (the one on the right with three dots).

3.  Select 'Code Builder' and click the OK button.

4.  The VBA editor will open at the section's Format event procedure with the first and last lines already in place.  Enter the code as a new line between these two lines.

5.  Return to the report design window and save the report.

A variation on the code I posted would be if the subreport is the only object in the section in question.  In this case, rather than hiding the subreport, you can cancel the whole section with:

Cancel = Not Me.[YourSubreportControl].Report.HasData

dhookom say:

Ken's solution using the HasData property is the most efficient. If you have a control on the main report that depends on a value from a subreport, you can use something like:

=IIf([YourSubreportControl].Report.HasData, [YourSubreportControl].Report.txtControlName, 0)

Howdy, so I can answer part of your issue. Firstly, since you have so few email's on this app I will recommend you just start from scratch. Read all before proceeding.

1. Delete the Outlook app. If it asks if you wish to delete all app data, choose yes.

2. Re-download the app. Now stop. Do not even open the app.

I want you to go to your Microsoft account, and change your outlook email address to your "primary alias".

*Disclaimer: Sometimes Microsoft will prevent you from repeatedly changing your primary alias, so you could be stuck for a week or two with outlook.com as your primary Microsoft alias. If this is a problem, don't proceed. Usually it is not an issue, however, and they will generally let you switch back and forth twice before stopping you again, which is enough to get the problem fixed and get you back to normal.*

Change alias via the following steps.

1. Login to your Microsoft account and go to the normal account homepage - it should automatically direct you there.

2. Select "Your info" from the blue bar at the top.

3. Select "Manage your sign-in email address"

4. Now you will see a list of your emails associated with the account. On the outlook.com email address, there will be an option that says "make primary". Select this. If it asks you to confirm, do that.

If you had trouble changing your alias, visit the following page: http://www.thewindowsclub.com/create-add-use-outlook-email-alias

Now return to the app, which you may now open.

1. Add the outlook.com account first.

2. Add any other applicable accounts.

Problems? Here are some of my tips for possible issues:

1. If you're worried this process may fail (or if it does!), delete the OneDrive app as well, then re-download it after you've completed everything, and sign in with your Microsoft account you had been using. Sometimes OneDrive interferes with things because, well, it likes to make us miserable sometimes.

2. Password failing for an account? You may need to generate an app-password, particularly if you use two-step verification. Search google for "How to generate an app password for (email type, e.g. icloud)", there are lots of reliable step-by-step guides.

P.S. Once your outlook.com email no longer has the long series of random numbers and letters, you may change the primary alias on your Microsoft account back to what it was.
