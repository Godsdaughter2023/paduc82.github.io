Tom wrote:

Although any of those methods might seem to work, we really should explicitly round any time arithmetic to the desired

Otherwise, binary arithmetic anomalies might result in infinitesimal differences that affect comparisons with time constants, notably in lookup functions like MATCH and VLOOKUP.

This is especially problematic when we propagate time arithmetic in a range, as Ian is doing ("A3 = A2 + 60 minutes

So we should write one of the following in A2, for example:

=--TEXT(A1+TIME(0,60,0), "[h]:mm")


The double-negative converts text to numeric time.  The format "[h]" instead of "h" allows for the possibility that A1 exceeds 24 hours; for example, A1 might include a date.

Note that =ROUND(A1*1440+60,0)/1440 in A2 is not reliable, in general.  (Although it might work with Ian's numbers.)  For example, if A1 is 15:28, A2 is 16:28, and B2 is =ROUND(A1*1440+60,0)/1440, the formula =MATCH(A2,B2,0) and even =MATCH(A2,B2,1) return the #N/A error (no match) because B2 is infinitesimal larger than A2.

It may need some time to complete the deletion, if the site still display in the frequent site section after several hours, you can try the following steps to unlock the site then remove it.

 

1. Ensure that SharePoint Online Management Shell is installed.

SharePoint Online Management Shell

 

2. Run SharePoint Online Management Shell as administrator

 

3. Connect to the tenant

Connect-SPOService (https://technet.microsoft.com/en-us/library/fp161392.aspx)

 

4. Release the hold for the group site

Set-SPOSite -LockState UnLock (https://technet.microsoft.com/en-us/library/fp161394.aspx)

 

5. Delete the site

Remove-SPOSite (https://technet.microsoft.com/en-us/library/fp161377.aspx)

 

After the above steps are finished, please wait for 24 hours to see the result.
