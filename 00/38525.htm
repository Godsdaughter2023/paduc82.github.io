Workaround

open notepad

insert the following lines:

@echo off
mountvol E: /d
C:\Windows\System32\shutdown.exe /s /t 0

(0 is the number zero which represents a delay in seconds before shutting down)

Save file as shutdown.bat in a suitable folder eg C:\MyScripts

 Right click on desktop select new>shortcut

At this point if you want to pin a button to the start or taskbar then enter:

C:\Windows\System32\shutdown.exe /s /t 0

in the window for the location of the item, otherwise use

C:\MyScripts\shutdown.bat (or whatever you filed the .bat as)

and click next, give the shortcut a name  eg. shutdown and click finish.

Optional - Right click the new desktop icon and click pin to taskbar and/or start.

Right click the new desktop icon and select properties

Target the shortcut to C:\MyScripts\shutdown.bat (or whatever you filed the .bat as)

Click advanced and check run as administrator and click ok

Click change icon and ok

Select a suitable icon and click ok and ok again

It is now safe to delete the shortcut from the desktop if you like, provided it has been pinned to the start and/or taskbar

You can do a similar thing for a restart button but the syntax on line 3 would be


C:\Windows\System32\shutdown.exe /r /t 0

Note: if you want to edit your .bat file in future, navigate to the folder and right click the file and select edit. If you left double click the file will execute and shutdown your PC

ALT+F keyboard shortcut stopped working - Microsoft Community

ronczapala say:

ALT+F keyboard shortcut stopped working

For as long as I can remember, in Windows Notepad (any many other programs) you could hold down the ALT key and press the "F" key and then the "S" key (while the ALT key is held down) to save a file.
Also, just the ALT+F combination would open the File dropdown menu.

This still works on my XP computers, but on my Windows 7 Pro x64 computer it recently stopped working.

In Notepad the ALT+E combination does open the Edit dropdown, ALT+O opens the Format dropdown, ALT+V opens the View dropdown, etc
but ALT+F does not work (the underscores do appear under all of the menu selections when ALT is pressed).

If you press and release the ALT key, the menu items' underscores appear and then you can press "F" to get the File dropdown.

I am pretty sure this is a recent issue since I use Notepad frequently and the ALT+F, S key sequence.

I searched the web but the only article I found was here:  ALT+F no longer opens file menu
and it indicates that "Most recent system update (around 01 Apr 2013) seems to have fixed the issue" but I didn't have WIN7 until
this year and all updates are applied.

P.S. - I know CTRL-S also saves a file in notepad but old habits are hard to break...
ronczapala say:

I tracked down the source of this problem.

I had installed a serial terminal program called TeraTerm to configure some microcontroller WiFly modules from RovingNetworks (now owned by MicroChip).

This program apparently hooked into the keyboard messages and prevented the ALT-F key combination from reaching other programs.

Uninstalling TeraTerm resolved the issue.
