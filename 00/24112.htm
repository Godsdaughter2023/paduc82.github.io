psyberfox say:

Hi there,

First of all I need to mention that the SQL Data Transfer function is very sensitive re data types, so if you have a date in the wrong format the whole import will fail and not specifically point you to what exactly the problem is.

Just on the information you provided it would probably be easier to simply open your access database, link the SQL table and do a straight copy>paste. Doing it this way is a whole lot easier and also will point to exactly where problems are encountered. So, I am assuming the table has already been created in SQL (make sure the fields are exactly the same and also in the same order).

You need to set up an ODBC link first, so let's go do that: from the start button simply type ODBC and you should see at the top of the list of available applications something called Data Sources (ODBC). Select that - it will open a screen with a few tabs: User DSN, System DSN, etc. You can either use a system or a user DSN - for the purposes of this link it shouldn't matter because it's a once-off issue. So, on the system DSN, select Add and on the following screen go right to the bottom and select SQL server. You need to know the name of your SQL server as well as the database name - just follow the wizard to setup a system DSN called "mydb" or something to that effect.

Then open your db in Access, and from the ribbon at the top select external data, more, odbc database. Then select link as you simply want to link the table(s) and not import the data. On clicking OK it will then go to the Select Data Source screen, select machine data source and select the data source you set up as above... Follow the wizard once again to select which table you want to link - and voila!

You now have the SQL table linked into access, so whatever data you add will automatically be available in SQL server...
