techie007l say:

    No problem.  I am assuming that you will be the only user of the PC.  Also, I will not be using the new Windows 10 built-in function to specify the storage device as it is a kludged-together hack that doesn't actually move the existing files or change the real save location of your files.  All it does is change the default save location of the Libraries, which means that programs that remember the previous save path, or read the default save path from the registry will still save documents to the SSD by default, causing files to get inconsistently saved to either drive.  So I use the older method that's much more reliable, but is unfortunately a little more involved to set up.

    First, let's make sure that the drive letters are correct, and then erase and format the HDD:

  1. Right-click the Windows/Start button and click Disk Management.
  2. In the bottom pane, find your CD/DVD drive and make sure its drive letter is (O:), not (D:).  If so, skip to step 5.
  3. Right-click where it says "CD-ROM 0" and click Change Drive Letter and Paths.
  4. Click [Change], choose O: from the dropdown list, and click [OK].
  5. Close Disk Management.
  6. Right-click the Windows/Start button and click Command Prompt (Admin).
  7. Type the following command and press [Enter]: diskpart
  8. Type the following command and press [Enter]: list disk
  9. You should see the SSD as Disk 0 (111GB), and the HDD as Disk 1 (931 GB).  (The sizes may not be exact, but the order and numbering of Disk 0/1 must be exact.  Otherwise, stop and post what you have here before proceeding.)
  10. Type the following command clean
  11. When it finishes in a couple seconds, close the Command window.  The HDD is now cleared.
  12. Right-click the Windows/Start button and click Disk Management.
  13. A window should pop up asking if you want to initialize Disk 1.  Follow the prompts to go ahead and initialize the disk.  The defaults should be fine.
  14. Back in the bottom pane, you should see Disk 1, with all its space marked as Unallocated (with a black colored bar).  Right-click its unallocated space and click New Simple Volume...
  15. Follow the prompts to create a partition that spans the entire disk (the default size should accomplish this).  Label the drive "Files", format as NTFS, and choose to quick format.  When asked what drive letter to assign, choose D:.  If D: isn't an option, let me know and we'll try to figure out why.
  16. After a couple seconds when the format completes, you should see Disk 1 now showing a 931GB partition called Files (D:) in the bottom pane of Disk Management.
  17. Close Disk Management.

    Now that the HDD is ready for your files, let's set up Windows to store your files on it instead of the SSD:

  1. Open File Explorer and expand This PC.
  2. Under This PC, right-click Documents.
  3. Click Properties and go to the Location tab.
  4. Replace the "C:\Users\Jeff\" portion of the path with "D:\".  For example, when you're done, the Documents folder should read "D:\Documents".
  5. Click [OK].  It should ask if you want to move the files from the old location to the new location.  Click [Yes].
  6. Back in File Explorer, right-click Downloads, and repeat steps 3 to 5.
  7. Back in File Explorer, right-click Music, and repeat steps 3 to 5.
  8. Back in File Explorer, right-click Pictures, and repeat steps 3 to 5.
  9. Back in File Explorer, right-click Videos, and repeat steps 3 to 5.
  10. I usually leave the desktop on the SSD so Windows isn't always waking up the HDD just to show the desktop.  This setting should be fine for you unless you plan to store many gigabytes of data on your desktop, in which case, do the above for the Desktop folder too.

    That's it!  Note that if you choose to create another user account in the future (or if you have other user accounts on this PC now), you will need to repeat this last set of steps so that their user files will also be stored on the HDD.  Just keep in mind that you would probably want to store another's files in a subfolder on the D:\ drive (e.g. D:\OtherUser\Documents and so on) to keep them separate from your files.  If you forsee this being a thing, you might want to store your files in D:\Jeff instead of directly on D:\ just for consistency's sake.
