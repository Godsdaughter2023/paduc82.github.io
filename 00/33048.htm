You might experience Black screen issues in Windows for a number of reasons. Some common issues that can cause these problems are:

  • The video output is being sent to an unexpected port.

1. On a working PC, go to the Microsoft software download website and create Windows 10 installation media.

3. On the Install Windows screen, select Repair your computer. (If you’re not seeing this screen, your PC might not be set up to boot from a drive. Check your PC manufacturer's website for info on how to boot from media, and then try again.)

5. After your PC restarts, select 5 or F5 to restart you PC in Safe Mode with Networking. If your PC won’t start in Safe Mode with Networking, try steps 2-4 again, and then, after your PC restarts, select 3 or F3 to Enable low-resolution video.

6. Once your PC is in Safe Mode, open Device Manager by typing device manager in the search box on the taskbar, select Device Manager, and select the arrow to expand Display adapters.

1. If you see two items under Display adapters:

  • To roll back the driver, open Device Manager and select the arrow to expand Display adapters. Under Display adapters, right-click the item and select Properties. Select the Driver tab, and then select Roll Back Driver > Yes.
  • To uninstall the display adapter, open Device Manager and select the arrow to expand Display adapters. Under Display adapters, right-click the item and select Properties. Select the Driver tab, select Uninstall > OK.

7. Shut down your PC, disconnect your installation media, and then turn your PC back on and check if it resolves the issue.

how to find the hard drive date stamp - Microsoft Community

nikkiwilliams say:

how to find the hard drive date stamp

i need to find the hard drive date stamp to check that a new one was installed when my laptop was in repair. As i do not believe that it has been replaced as it still has the same errors and files on it.
frederik long say: The manufacturing date stamp, if it exists, won't help you at all because disks are produced in batches. A replacement disk might easily have the same date stamp as the one that has failed. To be sure that the disk is replaced you need to compare serial numbers. The code below, when saved as Serial.vbs, will tell you the serial number of a disk.

set objWMIService = GetObject("winmgmts:\\.\root\CIMV2")
Set colItems = objWMIService.ExecQuery( _
    "SELECT * FROM Win32_DiskDrive",,48)
For Each objItem in colItems
  sNumber = objItem.SerialNumber
  for i = 1 to len(sNumber)
     sNum = sNum & mid(sNumber, i, 1)
     if i mod 5 = 0 then sNum = sNum & " "
  next
  msgbox  "Hard disk serial number: " & sNum
Next

Examining files or folders won't help you either. If the repairer cloned the flawed disk to the new disk then the file and folder dates will remain intact.

Your best bet is to run the CheckDisk program. If it reports errors then you should get the disk replaced.
