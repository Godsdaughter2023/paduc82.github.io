First Condition:
The only way I know of for connections to work is inside of the "Drawing Canvas". That applies in 2013 and back to 2003 (actually 2002) also.  If I create shapes outside of the drawing canvas, the connection lines won't "stick".

In 2003 you turn on automatic creation of the canvas in Tools menu > Options command > General tab > "Automatically Create drawing Canvas when inserting autoshapes"

In 2013 you do File menu > Options command > Advanced option > Editing options section > make sure this option is turned on: "Automatically Create drawing Canvas when inserting autoshapes"

Second Condition:
You have to use specific "connection lines", not just any lines. In 2003 it was easier to identify them because you when you clicked on the "autoshpes" command (in the drawing toolbar) there was a "connections" category". I don't see that category any more in the ribbon versions.  Instead, in ghe shapes gallery when you hover over a line you have to wait to see the title and pick a line that includes "Connection" in the name.  You can visually tell the difference between the 2 types of lines by looking , very carefully, at the little "handles" at the ends of the lines.  Regular lines have clear circle handles.  "Connection" lines have green filled circle handles.  When you drag a connection handle over a shape, the connection points are highlighted as tiny circles that are easy to miss.

I've recently encountered a new concept in PowerPoint. "Relational" or "Non-Linear" Presentations. The idea is that with some preparation, you no longer have to follow a top to bottom flow through the slides. rajesh c. say:

You cannot use COUNTIF for multiple conditions comparison.

COUNTIF is useful only when you have a single condition to count.

For Multiple conditions (like in your case), you have to use COUNTIFS function.

Try the formula 

=COUNTIFS($B$2:$B$100,"y",$A$2:$A$100,">"&DATE(2018,1,1),$A$2:$A$100,"<"&DATE(2018,3,31))

The same result can also be achieved by using the SUMPRODUCT function  as follows:

=SUMPRODUCT(--($B$2:$B$100="y"),--($A$2:$A$100>=DATE(2018,1,1)),--($A$2:$A$100<=DATE(2018,3,31)))

Checking for Updates - Microsoft Community

hosch250 say:

Checking for Updates

Technical Level : Basic

Summary
Explains how to check for and install updates in Office 2013. Basic skills required.
Details

Office 2013 has a new feature - automatically checking for and downloading updates instead of having you do it manually.  However, if you have your user account controls (UAC) set to not allow programs to run without your permission, the updates will not be allowed to install.  To check if any updates are available, you will need to open any Office application, Word will work excellently, and open the "File" tab.  Then choose the Account tab on the side:

If there are updates, it will say "Updates are available, click here to install."  To install these updates, you will need to activate the built-in Administrator account.  Sign out of all your accounts and sign in to the Administrator account.  Open Word and tell it to install the updates.  Allow it to close the applications when it asks you to; this is why you need to be in this account - the other accounts do not have enough power to close all the Office files, and it is impossible to do it manually.  I have tried to do it from an ordinary administrator account, closed all the Office programs, opened the Task Manager and killed all the Office processes, told it not to allow them to run on system boot, and re-booted the computer, and it still failed.

Let the updates install and do not use Office until it tells you that the updates have been applied.  Sign out of the Administrator account and disable it.

Edit:

With later updates, you no longer need to go into the Admin account and you can check for updates manually:
