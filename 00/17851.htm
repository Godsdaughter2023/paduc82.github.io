doug robbins - mvp office apps & services (word) say:

The following code inserts a Next Page Section Break at the end of the active document and then inserts the file containing the footer (in this case Doc18,doc after that section break

Dim rng As Range
Set rng = ActiveDocument.Range
rng.Collapse wdCollapseEnd
rng.InsertBreak wdSectionBreakNextPage
Set rng = ActiveDocument.Range
rng.Collapse wdCollapseEnd
rng.InsertFile "C:\Users\Doug\Documents\doc18.doc"


-- Hope this helps.

Are you sure it's undoing the main form? When you're working with subforms Access will save the main form when you first move into the subform, and you can't undo the main form. You can generally undo the subform - or at least the record you're working on in the subform, if you have a datasheet or continuous form.


You could use temporary tables which are exact recreations of your live tables, and bind your forms to them. When the user needs to edit data you fill those temp tables with the data, and then open your form. When the user finishes editing (or adding), you write the data back to the live tables. This gives you complete control over when/how your data is saved, but it comes with a good bit more work.

qa guy commelec say: Append and Update are two different things and therefore will need two queries.

Your are correct for update - I set the Join Properties to  "1. Only include rows where the joined fields from both tables are equal", then the update query will not append new records with new UIDs to the existing data table. It will simply change the existing data to match the new data in records where the UID are equal.

For append do this (Your statement edited) -  I read elsewhere online that if I set the join properties to "2. Make sure to backup your database before running the queries.
