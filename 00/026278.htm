duplicating a "Sway" project but not renaming the title - Microsoft Community

roncahalan say:

duplicating a "Sway" project but not renaming the title page?

Hello,

I need to duplicate a sway presentation for different sales staff. The title should not change BUT, I need to be able to identify each individual (duplicated) presentation somehow. The title page would be the same for each person but the contact info would be changed for each sales-person to use. Any ideas how this "identification" can be accomplished?

jason (office engineer) say:

Hi Ron,

While we are considering some ways to do this for the future, there is currently no way to do this exclusively within Sway if the title page must remain exactly the same.  One suggestion is to add a suffix/prefix to the title (e.g. Sales Presentation - Jim).

If you are using a personal account or your organization admin has allowed it, Sways can be published to docs.com through the Share dropdown and then viewed by others within your organization.  If you do that, you can add some metadata to the published Sways like description and tagging.

The only other suggestion I can make is that you could create a single parent Sway and then embed all the sales presentations into it.  Next to each embedded Sway, you could add a text card describing who it belongs to.  This would give you a kind of table of contents for all the different Sways.  The main problem for this is that it would need to be managed manually, but I suppose you're already doing that by duplicating the Sways.  If you decide to do this, open one of the sales presentation Sways, open the Share dropdown and get the embed code, then create a new Sway, add an Embed card, paste in the embed code into it, and then do the same for all the other Sways.  This might be a way to get closer to what you want.

Hope this helps!

Jason
