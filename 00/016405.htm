You would only see the Greek letter E symbol at the top of the query screen in design view.  Why do you run it from there?  

Are you saving it after clicking on the TOTALS before running it?   You might have saved after accidently clicking it.

 

Your query is not using the parameter.  If you did as below then you would have only one pivot column.

PARAMETERS [Forms]![frm_Switch_Board]![Batallon] Text ( 255 );
TRANSFORM Count(qry_Obreros_Por_Pais_ReporteFinal1.ObreroID) AS CountOfObreroID
SELECT qry_Obreros_Por_Pais_ReporteFinal1.[Pais Asignado], Count(qry_Obreros_Por_Pais_ReporteFinal1.ObreroID) AS Total_IDs
FROM qry_Obreros_Por_Pais_ReporteFinal1

GROUP BY qry_Obreros_Por_Pais_ReporteFinal1.[Pais Asignado]
ORDER BY qry_Obreros_Por_Pais_ReporteFinal1.[Pais Asignado], qry_Obreros_Por_Pais_ReporteFinal1.Batallon
PIVOT qry_Obreros_Por_Pais_ReporteFinal1.Batallon;

If I may briefly join in, please ... because the built-in roll back provision doesn't always succeed, and because hard drives all eventually fail, etc. one of the most important precautionary steps that one can/should take is to make a disk image type backup of the OS partition and save it on a separate hard drive, so that by booting to the related rescue disk, one can, in a very few minutes, get back to where he/she was before the failure, rather than having to start over from scratch.

At my place, that's a basic requirement, to be done for every build on every machine as soon as the debloating and operational adjustments have been accomplished.

We keep on hand at least the previous build and the last public release for each machine.

There are 3rd party programs for such that are free for personal use, and since they don't require any OS to restore to a saved image, they aren't inhibited by the failure of either a build or a hard drive.

It only takes 10 minutes, more or less, to make the image, and about the same to restore to whichever of the saved images one chooses.

That, combined with the keeping of one's data in a separate data partition, redundantly backed up, allows one to concentrate on testing and reporting, rather than on rebuilding from the ground up.
