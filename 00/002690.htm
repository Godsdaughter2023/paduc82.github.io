Weird.  I've not been able to reproduce the behaviour myself, in Access 2007 at least.

Rather than shrinking multiple controls for an address I generally use a single one-line deep control, which can be allowed to grow (as can a shallow subreport control in the parent form in your case), by concatenating the lines into a single string expression with the CanShrinkLines function from:

http://office.microsoft.com/en-us/access-help/eliminating-unwanted-blank-lines-in-labels-and-reports-HA001149472.aspx

though I've amended it as follows to avoid a trailing carriage return/line feed:

Public Function CanShrinkLines(ParamArray arrLines())

    ' Pass this function the lines to be combined
    ' For example: strAddress =
    ' CanShrinkLines(Name, Address1, Address2, City, State, Zip)
    Dim x As Integer, strLine As String
   
    For x = 0 To UBound(arrLines)
        If Not IsNull(arrLines(x)) And arrLines(x) <> "" Then
          strLine = strLine & vbCrLf & arrLines(x)
        End If
    Next
     
    ' remove leading carriage return.line feed
    CanShrinkLines = Mid(strLine, 3)
 
End Function

If one line contains values from multiple columns these should be concatenated as a single value when passed into the function, e.g.

CanShrinkLines(Title & " " & FirstName & " " & LastName, Address1, Address2, UCase(City), County)

It does have one disadvantage, however, in that individual address lines cannot be differentially formatted, so the practice of putting the city name in bold face for instance, which some people like, (though I don't, preferring to show it in upper case as above if it is to be emphasised) is ruled out.

How to write a sheet name in a formula to avoid #REF! error - Microsoft Community

raph.b say:

How to write a sheet name in a formula to avoid #REF! error

Hello

I'm using a huge formula that refers the a sheet (let's say "Cat")

 

When ever the sheet "Cat" is deleted,  i get a #REF! error, even if I create anohter sheet name "Cat" is created a second after,

 

I want to know how to write the reference the sheet "Cat" in the formula so that if the sheet is deleted, i will get a #REF! error but the moment a new sheet named  "Cat" is created, the formula will work with the new sheet named "Cat".

zorvek say:
You don't have to do it that way:

=INDIRECT("cat!"&ADDRESS(MATCH($B$1;INDIRECT("cat!$A$1:$A$54");0);MATCH($B$2;INDIRECT("cat!$A$1:$OP$1");0)))

And, if your tab name has spaces:

=INDIRECT("'cat!'"&ADDRESS(MATCH($B$1;INDIRECT("'cat'!$A$1:$A$54");0);MATCH($B$2;INDIRECT("'cat'!$A$1:$OP$1");0)))

Of course you can put the tab name in a cell too but I would make all tab names reference the cell, not just the second two:

=INDIRECT("'"&G7&"!'"&ADDRESS(MATCH($B$1;INDIRECT("'"&G7&"'!$A$1:$A$54");0);MATCH($B$2;INDIRECT("'"&G7&"'!$A$1:$OP$1");0)))

Kevin

zorvek say:
In your case, assuming you wanted cell Cat!A1, the INDIRECT function is:

=INDIRECT("Cat!A1")

If you delete Cat the function result will be #REF! but the function will re-calculate the desired result once Cat is added back into the workbook.
