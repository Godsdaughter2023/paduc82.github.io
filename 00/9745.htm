    Sub Worksheet_SelectionChange(ByVal Target As Range)
    'Sheet module code only, like: Sheet1.

    'In this case all the lists are in rows 1 to 8 only
        If Target.Row <= 8 Then

    Don't know why no one will specifically address your actual question, since the problem with your statements there seems completely obvious.

    Your original post in this thread mentioned that your weekly "deep scan" used to take an hour or two and resulted in around 12,000 files being scanned, which made no sense since the typical Windows 10 system contains at minimum something like a half million files.

    I assume that by deep scan you meant a full scan, which performs scanning on every file contained on all physical drives attached to a PC, as well as any remote drives mounted via sharing.  The default now I believe is to also expand and scan every file contained within any type of compressed file such as zip, Windows update packages and those like ISO copies of system installer or other mountable CD or DVD image files.

    In fact, the recent monthly cumulative update package that first began installing on systems this last Tuesday, likely contains an additional several thousand files if not more and would typically remain on a system for roughly a month or possibly longer to allow for its manual removal or repair if requested.

    In my own case a Windows 10 system that contains less than 5,000 personal files results in over 600,000 files when a full scan is performed, while the quick scan results in either roughly 43,000 or around a quarter of that depending upon whether the system has been rebooted recently.  This latter difference occurs because the status of individual files can't be guaranteed once a system is rebooted, so Defender assumes it must rescan all of the files involved rather than only those which haven't been scanned since the PC was last restarted.

    So if there are either external flash or hard drives, a phone or other mounted devices attached, compressed files containing either application installers or data files stored on the system or any other packages that aren't part of Windows itself, it's not uncommon for a Windows 10 system to contain far more than the 600,000+ files that my own desktop indicates.

    Now please understand that once a full scan has been completed on a system, it will also cache some results of the previous scan just as with a quick scan, but as I recall this typically only includes those regularly scanned by that smaller quick scan.  This is because the full scan is specifically considered a request to scan everything on a system, so other than that relatively small number contained within areas commonly known to contain malware, which were already known to have been scanned since the last system restart, most other files will be scanned again.

    I haven't personally tested the precise way this scan results caching currently operates with Windows 10 Defender, so some of this may have changed, but these same symptoms have caused lots of confusion in the past for those (most everyone) who don't truly understand how Defender operates behind the scenes.

    You need to use a Directory type mail merge main document to which you attach the Excel spreadsheet as the data source.  Leave the page orientation of the main document as Portrait and via the Page Setup section of the Page Layout tab of the Ribbon, the number of columns to 2.  Then in the first column, arrange the merge fields in the configuration that you want them, I would suggest using Shift+Enter to start each new line and after the last merge field, press the Enter key so that there is one empty paragraph after the paragraph that contains the merge fields.  Format the paragraph that contains the merge fields so that the lines are kept together.

    Your setup of the merge fields should look like

    «Name»
    «Address»
    «City» «State» «Zip»¶

    ¶

    Then when you execute the merge to a new document (use the Edit Individual Letters item in the Finish & Merge section of the Mailings tab of the ribbon), the new document that is created will contain the address details for each member, arranged in columns with each address separated from the next by the space that you have set to appear after paragraphs in the Format Paragraph dialog.  Then, via the Page Layout tab, go to the Margins tab of the Page Setup dialog and from the Pages section of the dialog, select Book fold and you document will then be formated so that when you print it, you will have the pages for a booklet with four columns of addresses.

    Excluding cells in a range based on criteria - Microsoft Community

    alec w 2169 say:

    Excluding cells in a range based on criteria

    I have a competition score card where each competitor has a row of scores. Some are integers, some are decimals (this is to denote different aspects of the competition - the final score must contain at least four of the decimals, and good logic says it has to be the largest four). Expected values are between 15 and 50.

    So range C5:X5 may look like:

    30.0 38.1 35.1 37.1 35.1 29.1 36.1 36.1 36.1 32.1 32.0 19.1 35.0 27.0 32.0 22.0 29.0 32.0 35.0 29.0 30.0 30.1 30.0 13.0

    The object is to find the top 4 decimal scores, then the top 16 scores from the remainder of the range, be they decimals or integers.

    My first goal is to find the top four decimal scores, which I have done easily enough by starting in cell AC5 with the following: 

    =IFERROR(INT(AGGREGATE(14,6,$C5:$X5/($C5:$X5<>INT($C5:$X5)),1)),0)

    then repeating in cells AD5, AE5 and AF5, incrementing the ranking value as I go.

    My question is, without using macros, how do I determine and sum the 16 largest values in the range C5:X5 whilst excluding the four values identified above? 

    I know I can do it by manually changing all the decimals except the top 4 to integers, then ranking them with:

     =IFERROR(INT(AGGREGATE(14,6,$C5:$X5/($C5:$X5=INT($C5:$X5)),1)),0)

    but that is hard work! 

    amit tandon say:

    Refer below image:

    Please note that you have given the values for range "$C$5:$Z$5" instead of "$C$5:$X$5" and hence below formulas use "$C$5:$Z$5" ... you may update the range accordingly in the formulas.

    Enter below formula as an array formula  (CTRL+SHIFT+ENTER) in cell B8:

    =SUM(LARGE(ISNA(MATCH($C$5:$Z$5,LARGE(IF($C$5:$Z$5<>INT($C$5:$Z$5),$C$5:$Z$5),{1,2,3,4}),0))*$C$5:$Z$5,ROW(INDIRECT("1:16"))))

    This formula will SUM the largest 16 values (decimal or whole nos) after excluding largest 4 decimal values in the range $C$5:$Z$5. HOWEVER, because of duplicates in the large 4 decimal values ie. the largest 4 decimal values are 38.1, 37.1, 36.1, 36.1 and then 36.1 again appears so the formula ignores 5 (38.1, 37.1, 36.1, 36.1, 36.1) instead of 4 largest decimal values.

    To overcome the above and to exclude ONLY 4 largest 4 decimal values (38.1, 37.1, 36.1, 36.1), you may use below formula - enter below formula as an array formula  (CTRL+SHIFT+ENTER) in cell B10:

    =SUM(LARGE(ISNA(MATCH($C$5:$Z$5+COLUMN($C$5:$Z$5)/100000,LARGE(IF($C$5:$Z$5<>INT($C$5:$Z$5),$C$5:$Z$5+COLUMN($C$5:$Z$5)/100000),{1,2,3,4}),0))*$C$5:$Z$5,ROW(INDIRECT("1:16"))))
