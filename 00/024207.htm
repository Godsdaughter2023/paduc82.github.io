Yes, unfortunately the "Field language" just does not provide the facilities that you need for this. I wish it did. 

There are two different possibilities here: 
  a. if you only need to filter Excel records for inclusion in or exclusion from the merge, then I think you should probably stick to trying to define appropriate functions in Excel that (for example) populate a column called "Include", then use that column to filter the records in the merge.

WHERE mycode LIKE '%E07%'

teylyn say:

The label option "Value from cells" is a new feature in Excel 2013. SharePoint 2010 and its Excel Web Services / Excel Web Access Web Part don't understand that new feature and will ignore it. 

Your best approach would be to create the labels with Excel 2010, since that is version-compatible with the SharePoint 2010 web part. If you no longer have access to Excel 2010, you can still use the same technique you would have used in that version, i.e:

  1. Create labels with one of the name or value label options that you can choose from, NOT with "value from cells".
  2. Edit a single label
  3. click in the formula bar and enter a = sign
  4. click the cell in the worksheet that has the text you want to show in that labels
  5. repeat from 2.) for each label. 

If you find that a bit tedious, you can download and install the free XY Chart Labeler add-in. Don't be thrown off by the title. It creates labels for all Excel charts, not just XYs. The add-in does exactly what I've described above, just automated with VBA. The workbook that has the charts will be saved with all the labels in place and will not need the tool if the workbook is opened by other users or displayed on SharePoint. 

You can download the add-in from the author Rob Bovey's web site here: http://www.appspro.com/Utilities/ChartLabeler.htm 

Let me know how that works for you.
