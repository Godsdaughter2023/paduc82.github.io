macropod say:

Ok, here's an example of what you can do:

1. Suppose you have one or more text content controls for numeric input and you want their content to be formatted as $ (e.g. $1,234.56).

2. Edit each of those content control to assign them the tag name 'Currency'.

3. In the document's 'ThisDocument' code module, insert the following macro:

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
With ContentControl
  If .Tag = "Currency" Then
    With .Range
      If IsNumeric(.Text) Then .Text = Format(.Text, "$#,##0.00")
    End With
  End If
End With
End Sub

From now on, inputting a number into any of those content controls then exiting it will cause the number to be reformatted is the $1,234.56 format, just as you might get with a formfield.

macropod say:

There are many ways of approaching this problem; the optimum way(s) depend on precisely what you want to achieve. Doug's shown one approach. Here's a different one that might work better if, say, you have one set of values you want to deduct from another. In that case, you might give one set the Title 'Credit' and the other set the Title 'Debit', with the aggregate being output to yet another, titled 'Total':

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
Dim SngCr As Single, SngDr As Single, i As Long
SngCr = 0: SngDr = 0
 With ContentControl
   If .Tag = "Currency" Then
     With .Range
       If IsNumeric(.Text) Then .Text = Format(.Text, "$#,##0.00")
     End With
   End If
 End With
 With ActiveDocument
  With .SelectContentControlsByTitle("Credit")
    For i = 1 To .Count
      If IsNumeric(.Item(i).Range.Text) Then
        SngCr = SngCr + .Item(i).Item(i).Range.Text
      End If
    Next
  End With
  With .SelectContentControlsByTitle("Debit")
    For i = 1 To .Count
      If IsNumeric(.Item(i).Range.Text) Then
        SngDr = SngDr + .Item(i).Item(i).Range.Text
      End If
    Next
  End With
  .SelectContentControlsByTitle("Total").Item(1).Range.Text = Format(SngCr - SngDr, "$#,##0.00")
 End With
 End Sub

Note: If your debits are input as -ve numbers, you'd need to change 'SngCr - SngDr' to 'SngCr + SngDr'.

Customer header/logo spacing guidelines - Microsoft Community

christinahaller say:

Customer header/logo spacing guidelines

I need to create a header with a logo, a line beneath, then an address with specific spacing guidelines.  The logo is in a portable network graphic file type.  Here are the guidlines

3/4" height of logo

logo 1 1/4" distance from left edge

logo 3/4" distance from top edge

line size 1 point

line distance 1 5/8" from top edge

line distance 1/2" from right edge

address 1 3/4" from top edge

address 1/2" from right edge

What is the easiest way to go about accomplishing this?

suzanne s. barnhill say:

In the header, insert the logo and create the line without concern about positioning. Then select each in turn and use the controls on the Position tab of the Layout dialog to fine-tune the positioning Relative to Page. You'll want the wrapping to be almost anything except In Line With Text. Access this dialog as follows:

  • For the line, on the contextual Drawing Tools | Format tab, in the Arrange group, click Position and choose More Layout Options. You can also use the Drawing Tools to fine-tune the length of the line if necessary.
  • For the logo, on the contextual Picture Tools | Format tab, in the Arrange group, click Position and choose More Layout Options.

For the text, you could use a text box, but I prefer to keep the text in the text layer whenever possible. For this purpose, you will use Spacing Before/After and other controls in the Paragraph dialog. If you have retained the default 0.5" header margin, then your header text starts 0.5" from the top, which means that you need to have the address 1¼" from the top of the header. So, in the Paragraph dialog, in the Spacing Before box, type 1.25". Word will convert this to points for you.

If the address is to be right-aligned, then you will either set the document right margin to 0.5" or (if this isn't practical) add a negative right indent to the paragraph to get it there. For example, if the right margin is 1", then add a half-inch negative indent to place the text at 0.5" from the edge. If the address is to be left-aligned, then you'll need to type it in the required font (again, with a negative right indent if necessary) and then adjust the left indent to place the text so it just reaches the half-inch mark.

If you've maintained a normal (say, 1") top margin for the document and put this letterhead (as you should) in the First Page Header, then you will also need to add some Spacing After to your address paragraph to generate some "breathing room" between the header and the document body. When a header exceeds the space between header margin and top margin (that is, the distance between 0.5" and 1" in a normal Blank Document), it will push the document text down, but there will be no space between the header and document unless you build some in.

Another way to approach this, though it would be much more difficult to fine-tune placement, would be to use a two-row, two-column table with a border between the rows. The logo would go in the top left cell and the address in the bottom right. Since Word requires an empty paragraph after the table, space between header and document is built in (you may want to change the spacing or size of that paragraph).

For more on creating letterhead, see http://wordfaqs.mvps.org/LetterheadRibbon.htm.

Limit max length of text in a textbox by centimetres rather than - Microsoft Community

josh craig say:

Limit max length of text in a textbox by centimetres rather than characters

I have a textbox on a VBA form where users enter a response in text.

I want to be able to limit the total width of text in the textbox. I know I can use the MaxLength property to limit the number of characters in a textbox, but how would I limit it so the total width of the characters doesn't exceed say, 3 centimetres?

Josh

e.j. gun say:

Here is a simple example that you can adapt to your needs.  It uses a Label on a UserForm to determine the current width of the text in the TextBox.  If the width of the Label (plus a bit more for margin) is greater than the width of the TextBox, then the last character typed is removed, effectively limiting the text width to the width of the TextBox, while allowing a variable number of characters.

You could place the label outside the form so the user does not see it.  Label1 can be used to limit multiple TextBox objects as required.

This example limits the length of text to whatever width you set for TextBox1.  You could also modify it to limit the length to a specific number of inches, centimeters or points.

EDIT:  This example assumes that MultiLine is False for the TextBox!

EDIT2:  I did this in Excel, and I just realized you're talking about Word.  Let me know if it doesn't work...

EDIT3:  Finally, this example assumes that you have SelectionMargin set to False for TextBox1.

HTH,

Eric

  • In a new Excel file, create a UserForm (UserForm1)
  • On UserForm1, add TextBox1 and Label1
  • In the UserForm1 code module, add this code:

Function MyTextWidth(sText As STring, sFont as String, LFontSize as Long) as Single

    With UserForm1.Label1

        .AutoSize = False

        .Font.Name = sFont

        .Font.Size = LFontSize

        .Width = 5000

        .Caption = sText

        DoEvents

        .AutoSize = True

        MyTextWidth = .Width

    End With

End Function

Private Sub TextBox1_Change()

    If (MyTextWidth(TextBox1.Text, TextBox1.Font.Name, TextBox1.Font.Size) > (TextBox1.Width - 7)) Then

        TextBox1.Text = Left(TextBox1.Text, Len(TextBox1.Text) - 1)

    End If

End Sub

  • Insert a new Module
  • Add the code below and run it.

Sub TestIt()

    UserForm1.Show vbModal

End Sub
