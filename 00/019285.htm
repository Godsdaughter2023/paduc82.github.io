In a normal Autodiscover query, Outlook will try to contact the following locations in order:

1. the SCP record for Autodiscover

2. the standard DNS namespaces:

  1. https://<smtpdomain>/Autodiscover/Autodiscover.xml

  2. https://autodiscover.<smtpdomain>/Autodiscover/Autodiscover.xml

  3. http://autodiscover.<smtpdomain>/Autodiscover/Autodiscover.xml

3. SRV record query for _autodiscover._tcp.<smtpdomain>

Office 365 use the https://autodiscover.<smtpdomain>/Autodiscover/Autodiscover.xml or    (usually it won’t), then Outlook will not be able to connect to Office 365.

Each row has a person's name associated with the project as well as the hours they are working on that project

 

Example:

 

A1 = Audrey,  B1 = 4 Hours

A2 = Ben,  B2 = 6 Hours

A3 = Charlie, B3 = 8 Hours

A4 = Audrey, B2 = 8 Hours

A5 = Charlie

A6 = Audrey, B6 = 3 hours

A7 = Ben

 

Desired lookup and calculation in single cell:  Type in Audrey, Result returned is a  sum of B1, and B4, and B6

bernie deitrick, excel mvp 2000-2010 say:

It depends. If the cell's contents are actual hour counts, like 4, (number of hours), then

=SUMIF(A:A,"Audrey",B:B)

or

=SUMIF(A:A,C1,B:B)

where C1 contains Audrey.

If the values are times - like  4:00, then

=SUMIF(A:A,C1,B:B)*24

IF the cell contents actually is a string "4 Hours" then an array formula (entered using Ctrl-Shift-Enter) is needed, like

=SUM(IF(A1:A100=C1,VALUE(SUBSTITUTE(B1:B100, " Hours",""))))
