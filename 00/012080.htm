cindy meister say:

Hi JMsinKY

You'd need to open the file as a pure TEXT file. You could do this in Notepad, for example.

If you want to do it in Word, go to the "Word Options", "Advanced" tab. In the section "General", activate Confirm conversions on open. Word will then prompt you about how the file should be opened, and you'd choose "plain text".

Be warned, however, that saving a Word document as HTML could provide HTML code that your blog does not recognize. When Word saves as HTML, it usually saves to a "round-trip" HTML file format, which means a lot of Word-specific stuff is included. Your blog site may not support the Word-specific (X)HMTL.

Bob Jones AKA: CyberTaz's reply to your question Fonts & adobe was marked as an answer.

I'm not sure why others are running you around like this... The simple fact is that Mac Office does not support font embedding in documents. Having you look for someplace to turn on font embedding is a waste of your time because no such setting exists. All the more, it isn't an Option/Preference issue in the first place :-)

Generating a PDF is a print function... Microsoft only stuck their PDF command in the Save As path for the purpose pf pandering to users who can't seem to understand that :-) When using the Save As method it invokes the Microsoft PDF generator, which provides 2 options: If you use the Best for electronic distribution and accessibility option fonts are not embedded in order to minimize file size. If you use the Best for printing option you'll get a different result and what could be a significantly larger PDF file.

Also, the File> Share method invokes the OS X PDF generator as does using File> Print - PDF. The common problem is that both of these methods embed all fonts used in the document in their entirety, rather than subsets, which can result in an extremely large PDF file. Most other programs [such as Pages] use the OS X PDF generator.
