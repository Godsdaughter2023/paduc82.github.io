scottgem say:
I have to ask why? There are several questionable design issues involved here. 

First, one should not store Age. What someone's age is changes daily. So if you need to know age, you should be storing date of birth and calculating age. 

Second, Name is a reserved word in Access and shouldn't be used for object names. Also, people names should be broken into their components. You should, at least be storing first and last names. I generally use 5 fields for people names; Salutation (Mr, Ms, etc.) first, Middle, Last and Suffix (Jr, Sr, etc.).

Third, you don't have fields on a form, you have controls that may or may not be bound to fields in a table. This is a subtle but important distinction. But what I don't follow is why, if you have a subform that displays the name and age, that you need to display them on the main form as well. 

So before we can really help you we need to know why you are doing this, what the recordsource of the main form is, what the purpose of the app is and more about the structure of your database.

jay freedman say:

Since Word doesn't have a Reset button for content controls by default, I assume that you have a macro that runs when you click that button (whether a MacroButton field in the form or a button that was added on the Quick Access Toolbar or the ribbon).

If that's the case, then the macro needs some additional code. If it now searches through the collection ActiveDocument.ContentControls, that will find only controls that are in the main body of the document but not any that are in text boxes, headers or footers, footnotes or endnotes. There's a technique, explained at http://www.word.mvps.org/FAQs/Customization/ReplaceAnywhere.htm, for searching through all those places, and it can be adapted to reset content controls. If you're concerned only with controls in text boxes, the added code can be as simple as this:

    Dim CC As ContentControl
    For Each CC In ActiveDocument.StoryRanges(wdTextFrameStory).ContentControls
        CC.Range.Text = ""
    Next CC

If you don't want to mess with a macro, I find that if I just delete all the text from the Date Picker (or any other content control) and then click anywhere outside that content control, the placeholder text reappears.
