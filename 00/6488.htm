IE9 Tracking Protection blocks well-known jQuery CDN URLs - Microsoft Community

carl.camera say:

IE9 Tracking Protection blocks well-known jQuery CDN URLs

Hi, this is going to be a big problem for lots of websites.  I am very supportive of Tracking Protection and I especially like that IE9 automatically counts how many different sites are pointing to the same javascript files across domains, then disables access to that javascript file automatically.  THIS IS GREAT! But there's a problem. 

There are a handful of well-known jQuery javascript CDN (Content Delivery Network) urls out there that many of us web designers point to in order to improve performance.  The beauty of a CDN is that if there are many sites that all point to the same jQuery library at ajax.googleapis.com then the javascript library is more likely to be cached on the client browser when he or she visits my page.  However, it appears that these jQuery CDN libraries -- because they are in fact javascript files used across several domains -- are being disabled automatically by IE9 Tracking Protection.  The result is dramatic loss of functionality on many, many, many websites.

Example websites:

  • http://youversion.com  (the home page shows "loading" -- entire website functionality is gone.)
  • http://ohiomcr.com (the carousel is non-functional - hover states over navigation do not appear.)

Incidentally, the google map javascript on the ohiomcr.com site is not blocked by Tracking Protection.

  • http://maps.gstatic.com/intl/en_us/mapfiles/310c/maps2.api/main.js GET 304 text/javascript 226 B 16 ms <script> 140 0 16 0 0 374

The workaround, I assume, is to have each domain host their own jQuery library. This is what I plan to do. But expect some backlash from developers. I think the IE9 Team should be aware of this problem and, frankly, you need to alert web developers because many sites will be affected.

 

Steps to recreate:

1. Enable Tracking protection in IE9

2. Visit 20 sites that invoke the same jQuery CDN (http://ajax.googleapis.com/[............]jquery.js)

3. Open IE Developer Toolbar

4. Enable "Net" tab in IEDevBar

5. Revisit sites in Step 2 -- note that the js is blocked by Tracking protection.

Sub MyAddInMacro(Argument)
  MsgBox Argument

Sub Test()
  Application.RecordMacro "Msg = ActiveCell"
  Application.RecordMacro "MyAddInMacro Msg"
  Msg = ActiveCell
  MyAddInMacro Msg

jllatham say:

Try the code below.  I've set it up with Const values for worksheet names, column IDs and first data row on each sheet to make it easy to adapt if needed.

It looks for last used row in F:H on source sheet to determine how many rows to copy, and will work even if all 3 don't have entries in all cells in 'last' row.  In determining where to put it on the destination sheet, it looks for the last used row anywhere on that sheet and copies F2:H# to column A just below last used row on the sheet in any column.

Sub CopyRevisedData()
'NOTE: Does not prevent data being copied multiple times
  Const srcWSName = "Worksheet1"
  Const destWSName = "Worksheet2"
  Const firstPossibleRow = 2
  Const firstColToCopy = "F"
  Const lastColToCopy = "H"
  Dim rangeToCopy As Range
  Dim anyLastRow As Long
 
  'find last used row of F:H
  On Error Resume Next
  anyLastRow = Worksheets(srcWSName).Range(firstColToCopy & ":" & lastColToCopy).Find(What:="*", _
                  After:=Worksheets(srcWSName).Range(firstColToCopy & 1), _
                  Lookat:=xlPart, _
                  LookIn:=xlFormulas, _
                  SearchOrder:=xlByColumns, _
                  SearchDirection:=xlPrevious, _
                  MatchCase:=False).Column
  On Error GoTo 0
  If anyLastRow < firstPossibleRow Then
    Exit Sub ' no data to copy
  End If
  Set rangeToCopy = Worksheets(srcWSName).Range(firstColToCopy & firstPossibleRow _
   & ":" & lastColToCopy & anyLastRow)
  'find next available row on destination sheet
  On Error Resume Next
  anyLastRow = Worksheets(destWSName).Cells.Find(What:="*", _
                  After:=Worksheets(destWSName).Cells(1), _
                  Lookat:=xlPart, _
                  LookIn:=xlFormulas, _
                  SearchOrder:=xlByRows, _
                  SearchDirection:=xlPrevious, _
                  MatchCase:=False).Row
  If Err <> 0 Then
    anyLastRow = firstPossibleRow
    Err.Clear
  Else
    anyLastRow = anyLastRow + 1
  End If
  On Error GoTo 0
  rangeToCopy.Copy
  Worksheets(destWSName).Range("A" & anyLastRow).PasteSpecial xlPasteValues
  Application.CutCopyMode = False
  Set rangeToCopy = Nothing
End Sub
