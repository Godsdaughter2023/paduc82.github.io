Drop down list using jpegs or images as select options - Microsoft Community

a. user say:

Drop down list using jpegs or images as select options

Hi all,

I'm wanting to create multiple drop down lists in a Word template I'm creating using jpegs/images.

Is this possible? Can it be done using a graphic or shape instead?

Thanks in advance for the advice and help.

jay freedman say:

Word doesn't offer exactly what you're asking for, but there is something close. Note that it will be usable only in Word versions later than 2003, and I'm not certain it will work in Word for Mac.

First you have to store the images as building blocks in a single gallery in your template. There are five otherwise unused galleries named Custom 1 through Custom 5, as well as a Custom Quick Parts gallery. Choose one of these for your project. Then:

  • Insert the images in a document based on your template.
  • Select one image and press Alt+F3.
  • In the Create New Building Block dialog that appears, enter a name for the image. Choose your selected gallery. [Optional: choose or create a category. You could create a new category for each "dropdown" so each one will have only its own image list.] Make sure the Save In box is set to your template, not to Normal.dotm. Click OK.
  • Repeat the previous two steps for the rest of the images. 
  • Close the document, and answer Save when asked whether to save changes to the template.

Now you can set up the "dropdowns" in the template.

  • On the Developer tab of the ribbon, insert a Building Block Gallery content control.
  • While the content control is selected, click the Properties button.
  • At the bottom of the Properties dialog, select the custom gallery (and category, if needed) that contains the image building blocks. Click OK.
  • If you want a different placeholder text instead of "Choose a building block", click the Design Mode button and edit the text; then turn off Design Mode.
  • Save the template.
  • Repeat for the other "dropdowns".


In a document based on the template, a user can click in a content control, click the down arrow in its tag, and click the desired image to select it. Unfortunately, the space for each building block is rather large, and finding the desired image in a long list can take some scrolling.

1) Press Windows key + R, type regedit in Run box.

2) Go to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\HomeFolder\NameSpace\DelegateFolders

Guys, my apologies. I have led you on a wild goose chase as I was confusing myself.

Right. Once more with feeling....
The date is put into a textbox on a UF when it is initialised thus:

       Me.TextBox1.Value = Format(Date, "dd/mm/yyyy")

This shows as expected and, today, shows as 10/02/2011
When data is then copied to another sheet it is from this TB, not another sheet:

That said, I have jut re-tested with both 10/02 and 20/02.
20/02 comes out as 20/02/2011 but 10/02 still comes out as 02/10/2011


I think you are running into what is known as "evil type coercion". I believe in the VB world, String dates are assumed to be in US format unless that would raise an error, then it attempts to find a date format that does not raise an error and then uses that instead. This is why using String dates is a bad idea. There is no need to pass a formatted date to the worksheet's cell...dates in memory, for both Excel and VBA, have no format... dates are stored as floating point numbers with the whole number part representing an offset from "date zero" and the fractional part representing the fraction of a 24-hour day that the time represents. The formatting of dates is only for the convenience of the human viewer... Excel and VB do never really see them when they work with the dates. So, the solution to your problem should be an easy one... don't pass the String representation housed in the TextBox to the worksheet cell; rather, pass the actual Date value to it. Just change this statement...

.Range("B" & lngNewRow).Value = Me.TextBox1.Value

to this...

.Range("B" & lngNewRow).Value = Date

IF there is the possibility that your code could be used across midnight (that is, the TextBox is filled with its date before midnight, but the cell gets filled after midnight, then you would need do use an intermediary Date variable to bridge the gap. Something like this...

Dim DateRightNow As Date
DateRightNow = Date
Me.TextBox1.Value = DateRightNow
....
....
.Range("B" & lngNewRow).Value = DateRightNow

This would make sure both dates are always the same.

shyam pillai say:

Here is a macro which will overlay your existing table with textboxes containing the text from the tables.

You can then animate them. It does not do borders or fills just text. But you can get the individual cells and fills from the ungrouped meta file.

Select the table shape and run the command.

Sub OverlayCellText()
Dim I As Integer
Dim J As Integer

Dim TableShp As Shape
Dim Shp As Shape
Dim Tbl As Table
Dim txtBox As Shape

Set TableShp = ActiveWindow.Selection.ShapeRange(1)

If TableShp.Type = msoTable Or TableShp.HasTable Then
    Set Tbl = Shp.Table
    
    For I = 1 To Tbl.Rows.Count
        For J = 1 To Tbl.Columns.Count
            
            Set Shp = Tbl.Cell(I, J).Shape
            
            Set txtBox = ActiveWindow.Selection.SlideRange(1).Shapes.AddTextbox(Shp.TextFrame2.Orientation, _
                    Shp.Left, _
                    Shp.Top, _
                    Shp.Width, _
                    Shp.Height)
            
            If Shp.TextFrame2.HasText Then
                Shp.TextFrame2.TextRange.Copy
            
                txtBox.TextFrame2.AutoSize = Shp.TextFrame2.AutoSize
                txtBox.TextFrame2.HorizontalAnchor = Shp.TextFrame2.HorizontalAnchor
                txtBox.TextFrame2.MarginBottom = Shp.TextFrame2.MarginBottom
                txtBox.TextFrame2.MarginLeft = Shp.TextFrame2.MarginLeft
                txtBox.TextFrame2.MarginRight = Shp.TextFrame2.MarginRight
                txtBox.TextFrame2.MarginTop = Shp.TextFrame2.MarginTop
                        
                txtBox.TextFrame2.TextRange.Paste
                
            End If
        Next
    Next
Else
    MsgBox "Please select a table shape and then run this macro.", vbExclamation
End If
End Sub
