"Upgrading to Color Categories" creates duplicate categories FIX

EDIT: I added links and fixed poor writing.

Hello, 

I am migrating a user who heavily relies on color categories in his calendar events from Exchange 2007 to Office 365. Since it is just one user, I am exporting his outlook data from his Exchange2007 profile as a PST and importing it into his O365 profile. However, when I import, I am left with events that are all the same color. This is because none of his imported categories are part of the "Master Category list". So, I used this tutorial to upgrade him to color categories.

However, once I do this, I get additional categories added to events. 

For example, if I originally have an event categorized under the "Personal Category", once I "upgrade to color", I get a second category that is called "Personal" added to the same event. Or an event categorized as "Needs Preparation" also has "Heating" added to it. Completely erroneous data. 


I nearly went insane trying to figure out why the data got corrupted. I still don't know what causes it. However, thanks to a fluke, I found out that if you have to add a single note with your custom categories into a brand new Exchange Mailbox and upgrade to color categories BEFORE you migrate any other data, then you won't have any other problems. 

This is what I did. 

 Go to this website to learn how to backup and restore categories using the "empty note" method. 

In essence, you backup your categories by creating a blank note in outlook (so, in my example, Exchange 2007), and add ALL of your custom categories. Then drag it to your desktop so that it is backed up (it saves itself as a .msg file). Then close outlook, setup a new Mail profile for outlook that will be logging into your new Exchange service (in my case, Office365). Then you drag the note that you created earlier back into  outlook. Next, go through the process of "Upgrading to Color Categories."

After that, THEN you can import your PST (I haven't tried this with a remote migration). All of your calendar events should be color coded according to the category that they originally had, and there will be no more duplicate categories. The colors will be different, however. So you will need to change them back manually. Click this link and find "Choose a color for a color category" heading to learn how to change colors manually. This is where having the old mail profile comes in handy, you can go back and do a screen capture of your old categories list again as a reference for your new profile so that it looks exactly like it did before. 

The important thing is that the Exchange service CANNOT have any data already in it. You have to start with a fresh, new mailbox and mail profile on your computer. If someone has already attempted a migration, or if you already tried importing PST data, this won't work. You will have to erase that mailbox and create it again. 


I really wish Microsoft supported color categories in Outlook PST's better... period. Some people with very mobile lifestyles NEED color categories to let them know when they need to arrange travel, need to create a presentation, review business numbers before a meeting, or know when they are going to a doctors office, or when they will be seeing a longtime friend, etc.

scottgem say:
Hi DJ,
In Access macros and VBA are two separate, distinct and different programming environments. This is not true of the other Office apps. In all the other apps the macro language is VBA.

Prior to 2010, the macro language was a simplistic tool that was very inflexible. While it has improved since 2007, it still has nowhere near the flexibility of VBA. The main reason for the improvements was because of trying to WEB enable Access apps.

I would not recommend wasting time learning the macro language. Anything you can do in a macro can be done in VBA almost as simply. For the most part, Macro Actions are GUI forms to enter VBA methods. VBA's Intellisense does, basically the same thing.

For example, you want to open a form you type
DoCmd.
and Intellisense gives you a list of the methods. You select OpenForm and Intellisense prompts you for the parameters in almost the same way the macro Action does.
