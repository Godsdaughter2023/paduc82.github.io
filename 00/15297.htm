tara sl say:

Default Value of a field in a form.

Fairly beginner w/ Microsoft Access 2013. Is there a way to make the default value of your form field be be the previous item entered?

For instance, I have a form that people will fill out monthly and one of the fields is "Watch Items".

The next month when they open a blank form, I want the "Watch Items" to prefill with what was entered the previous month in case there are no changes. If there are changes I want to allow edits. Is there an easy way to do this? Thanks in advance.

scottgem say:

Well, the problem here is your question indicates a design problem. Relational databases are setup according to a set of rules called Normalization. A properly normalized database is the foundation of an efficient database. Without proper normalization you may be applying bandaids over an over to fix the design problems. 

A field with a name like Watch Items indicates that you are storing multiple pieces of data in a field and that is not good design. Neither is repeating data over again. 

So before we answer your question (and there are a number of ways that this can be done), it would help if you explained what your application does and what your table structure looks like. Then we can suggest the best way to accomplish your goal (though it may mean a redesign.

mimmo57 (d. laurenza) say:

Hi,

perhaps you can use a DlookUp function to take Watch Items using as critera the previous month:

=DLookUp("Watch Items","yourTableName","Format("oldDate","yy-mm")= Format(Datediff("m",-1,Date()),"yy-mm"))

Ciao Mimmo

ken sheridan say: Yes, it can be done, and you'll find an example in Defaults.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the option to 'Open Form to Set and Save Default Values for Current and Subsequent Sessions' allows you to electively carry forward the last entered Department and/or City values to the next session.  it does this by storing the form name, control name and the value to be carried forward in a Defaults table.

As you apparently wish to do this automatically rather than electively you won't need the check boxes used in my demo to show which controls' values are to be carried forward, and the code in the form's AfterInsert event procedure would not need to be conditional on the check box's value being True.

Alternatively, as your requirement seems to be a simple one you could do as Mimmo suggests, and look up and assign a value to the control in the form's Load event procedure, though I'd prefer to look up the value for the row with the latest (Max) date rather than making the assumption that each row will fall in a distinct calendar month, and the sequence of months will be unbroken.

However, I share Scott's reservations.  Your use of the plural 'Watch Items' does suggest that you might be storing multiple values at one column position in a row in a table.  This would mean that the table is not normalized to First Normal Form (1NF), whose definition is:

First Normal Form: A relvar is in 1NF if and only if, in every legal value of that relvar, every tuple contains exactly one value for each attribute.

Loosely speaking, in the language of the relational model, a relvar (relation variable) equates to a table, a tuple to a row (record) and an attribute to a column (field).
