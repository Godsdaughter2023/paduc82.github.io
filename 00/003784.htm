charles kenyon say:

The options you mention are Styles and would not appear in the Font (Police?) area. Numbering is best done using styles as well (and not the list styles). See How to create numbered headings or outline numbering in Ribbon Versions of Word by Shauna Kelly. If for some reason you are using the Filling in Forms restriction, I believe that is a legacy restriction and will not pay attention to your options. It is really not intended for Content Controls (although, for some purposes, it works fine). You would, instead, want "No changes" with exceptions.

For further help, you would want to explain exactly which method of "protecting" your document you are using.

charles kenyon say:

Try use "No changes (with exceptions)" instead of Filling in Forms protection. You need to make each Content Control an exception for everyone.  The styles restriction should be in each Content Control.

Again, look at the Shauna Kelly article and consider using numbering styles set up that way. I do not know that the list styles will cause problems, but am more comfortable doing it that way.

If the "No changes" restriction does not work, consider saving an abbreviated version of your document with any confidential or proprietary information removed to OneDrive or Dropbox and posting a link here.

stefan blom say:

Are you using the "Limit formatting to a selection of styles" option at the top of the Restrict Editing task pane? Note that it requires that you create paragraph and character styles for all of your formatting needs. In particular, you'll need to add numbered styles to be able to insert numbered paragraphs.

In addition, I wouldn't try to use "Limit formatting..." together with any of the "Editing restrictions" in Step 2 (assuming that is what you are trying to do).

stefan blom say:

You can configure a Rich Text content control so that it uses a specific style. Click in the control and then click Properties on the Developer tab.

When you are using content controls, you can decide which areas can be edited by using the Group command in the Controls section of the Developer tab.

In other words, assuming that I have understood what you are trying to accomplish, there is no need to combine styles restriction and forms protection.
