ken sheridan say: Add the following function to the form's module:

Private Function ZeroSelection()

    With Me.ActiveControl
        .SelLength = 0
        .SelStart = 0
    End With

End Function

If you want the insertion point to be at the end of the value rather than the start amend the code as follows:

        .SelStart = Len(.Value) + 1

Then set the On Got Focus event property of each control in which you want to suppress the selection when the control receives focus to:

    =ZeroSelection()

However, this means that if the user then types a value this will be inserted before/after the existing value and not overwrite it, which is what would normally be wanted.  So rather than the above you might think about setting the BackColor and ForeColor properties of the control to colours which will allow greater legibility.

PS:  Having read your post again I see that it's when a selection is made that you wish to suppress the selection, not just when the control receives focus.  I don't think there's any event which would allow this, so you'd have to amend the colours to make it more legible.

hansv mvp say:

I don't think the code for LastRow is correct. It will almost always return 1.

Try changing it to

 

Function LastRow(sh As Worksheet)
    On Error Resume Next
    LastRow = sh.Cells.Find(What:="*", _
                            SearchOrder:=xlByRows, _
                            SearchDirection:=xlPrevious).Row
End Function

 

The value FieldNum = 41 assumes, as is mentioned in the accompanying comment, that the range starts in column A. Your range, however, starts in column AI, not in column A. So FieldNum should be 7.

 

You copy the unique values to column A in the new sheet:

 

        My_Range.Columns(FieldNum).AdvancedFilter _
                Action:=xlFilterCopy, _
                CopyToRange:=.Range("a1"), Unique:=True

 

But then you look for values in column AI:

 

        Lrow = .Cells(Rows.Count, "Ai").End(xlUp).Row

 

That column is empty, of course.

 

There is probably more, but start by correcting the above inconsistencies.

stevenshanemoore say:

Hi SakuraL, restting your computer to factory settings would remove any viruses, malware or spyware.  A factory reset would remove all applications and data (pictures, files, programs) you have installed before the reset.

 

It would be good to note that restoring any files from a backup made prior to the newly installed 'clean' Windows 7 OS could bring the virus, malware or spyware problem back if those files are infected with the unwanted malware.

The issue where you get alphanumeric letters in the email address when sending happens if your Primary Alias is not an Outlook.com account.  For example, if you go to this page https://account.live.com/names/Manage make sure your Outlook.com account shows as Primary Alias there.  It will show as your email with (primary alias).  If that is not the case then locate your primary Outlook.com account on that list and click Make Primary.  After you make this change you should remove your account from Outlook and add it back, or create a new Profile and add it during that process.  This will enable Outlook to pick up the change.

kindly let me know how to report this bug or will microsoft take cognizance and fix this ?

chrispeng_msft6626 say:

Hi Raj,

This is not a bug but an expected behavior of Outlook.

The Daylight saving time(DST) ended on October 25 2/3 am in Netherlands. For multi-day view, the display of the time zones is determined by the  start time of the period. So for the work week October 26-30, Outlook adjusts Netherlands time since it returned to standard time. As the DST still takes effect on October 25 , for the week October 25-31, the Netherlands time zone is displayed as DST.
