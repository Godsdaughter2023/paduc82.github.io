john korchok say:

Normally image editing programs and previews will try to display images at dot-for-dot resolution (In Photoshop, this displays as 100%). Each dot in the original maps to 1 pixel on your display. They ignore the resolution setting of the file.

But if you've set the file to something like 600 dots per inch resolution, when you place the file on the page, it will appear quite small. Word reads the resolution setting and places what appears to be a small version of the picture.

You can test this by saving your picture in GIF format, then place that in Word. GIFs do not have resolution information, so Word assumes they are 72 dots per inch. I think you'll see a larger version of the picture. But this is just a demonstration, the GIF file will be relatively low quality, since it can only hold 256 colors.

If the above got you a larger image, open the picture in your image editor, lower the resolution so the same number of pixels cover a larger area, then resave. This is different from resampling the image which creates or destroys pixels. If you post what program you use to edit pictures, I can give you step-by-step instructions, should you need them.

suzanne s. barnhill say:

The simple answer is that the appendix title/heading must be in a style that is included in the TOC (one with an outline level of 1, 2, or 3 for the default TOC) or you must explicitly select that style in the Table of Contents Options dialog to be included. If the appendix headings are numbered, the numbering will be included in the TOC; otherwise, you can apply numbering to the TOC level assigned to the appendix entries.

See http://wordfaqs.ssbarnhill.com/TOCTips.htm#Content for the first issue and http://wordfaqs.ssbarnhill.com/TOCTips.htm#UnnumberedHeadings for the second.

john mcghie say:

PCs with Word 2007 or above certainly will have Calibri.  If they do not, they will automatically substitute Arial.

That's not your problem.  You have copied from a PDF that contains some odd font.  The obj boxes are where Adobe has converted characters that are not contained in Arial into outlines so they can be displayed.  Always try to get the Word original of sources you work with to avoid this problem.

The best way to overcome this is to paste as Unformatted Text, then re-format the text when it gets into Word (it will adopt the formatting of the paragraph you paste into).

Hope this helps
