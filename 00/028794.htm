john mcghie say:

Hi Hauns:

The short answer is "DON'T" attempt to share templates between different versions of Word.

Sadly, Word 2011 is kind of "Word Lite", it does not have all the powerful tools and features that Word 2010 has.  While documents are almost completely interchangeable, templates are not.

As you have discovered, the template path is stored in the document.  While the PC can resolve the Mac path, the Mac can't resolve the PC path.

One issue you will run into is the concept of "Trusted Locations". On the PC, you can "Trust" locations so Word will load templates from them, ignoring the presence of macros.  On the Mac, the trusted locations are fixed, you can't customise them.

For your purpose, you need a Global Add-in (more convenient than an Attached Template, since the Global Add-in is available to all documents).

If you place an alias to your template in the Startup folder on the Mac, Word will load the template as a global add-in whenever it launches.  On the Mac, the Startup location is
 /Applications/Microsoft Office 2011/Office/Startup/Word

I don't think you can change that location: I haven't tried recently, but I seem to remember that if you try to change the Startup folder location, it doesn't work.

Of course, on the PC, you can load this as a Global Add-in and it will stay loaded.

Now:  I would set the file Read-Only and edit it only on the PC.  PC Word adds extra sauce to templates; Mac Word will interpret that as "damage" and may strip it if you attempt to save changes to the template from the Mac.

I used to share templates between Word 2008, 2007, 2010 and 2011 on my computer, but eventually I found it was just too much trouble.  I now copy the macros to the Normal Template in each version of Word.  Anoying, having to remember to update the macros in four different templates.

If I wasn't so lazy, I could write a macro to load a Global Add-In on Execute, and put that in each template. That macro would never change, and each time it would load the single maste copy of the macros.

One reason I don't is because of the mayhem that would result if I ever introduced a 64-bit version of Word.  The VBA in the 64-bit versions is very definitely NOT compatible, you end up with some serious code-arounds on some statements to cover the different functions available.

Hope this helps

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
    Dim someCells As Range
    With ActiveSheet
    Application.ScreenUpdating = False
        If ActiveCell.Column <> 3 Then Exit Sub
              Set someCells = ActiveSheet.Range("C4", _
              Cells(.Cells.Count))
        With someCells
            .Sort Key1:=Range("C4"), Order1:=xlAscending, Header:=xlYes, _
             OrderCustom:=1, MatchCase:=False, Orientation:=xlTopToBottom, _
             DataOption1:=xlSortNormal
        End With
    End With
    ActiveSheet.Cells(Rows.Count, 4).End(xlUp) _
            .Offset(1, 0).Select
    Application.ScreenUpdating = True
