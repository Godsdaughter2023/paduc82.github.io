One important piece of info that you left out is the type (POP3 or IMAP) that this PST belongs to.

In short, unless you are moving from a POP3 account on the old machine and "intentionally" moving to an IMAP account in OL'2016 - then there would be absolutely no reason to import anything.

If this involves an IMAP account in both '2010 and '2016 then there is nothing for you to do in terms of email data since the email folders will sync automatically. You will however need to move the "default" data file defined in OL'2010 to the new machine so that you retain your contact/calendar info. To determine what data file is the "default data file" in OL'2010, from the menu --> File --> Account Settings --> Data Files tab --> check to see if the default data file is your 50gb PST file

If this involves a POP3 account, one thing I would suggest is that you break up your 50gb PST file in smaller logical files to make things easier and safer to deal with. Am going to presume that you're not backing up this file on any kind of regular basis (if at all). If that's true, you risk losing everything should that file ever become corrupted for any reason. Whether you create multiple PST files along the lines of "Date" using the Outlook Archive function (i.e. one file for a specific yr or set of years) or along the lines of some logical "subject group" is a matter for you to decide. Once you get that all done, make a copy of those new PST files for backup purposes.

Rather then get into an encyclopedic response with all the different possibilities, will wait for the answer to the initial question in this post and what you want to end up doing.

sys-eng say:

I see this a lot and had to solve it a long time ago for my computer business.  This solution should help people who have moved their computer or trying to use it on different Internet Service Provider (ISP) connection.

Many outgoing e-mail (SMTP) server addresses are ISP specific. When changing ISP, even temporarily, the outgoing server address must be changed. Every ISP that I know of has a mobile setting that will work on any network. Try that one and see if it solves your problem.  Many of the ISP tech support websites hide this information because they just assume that the customer is always on thier netowork.

 

EXAMPLE

If I am working on a computer that was on a typical Time-Warner Roadrunner connection, its Outlook program will recieve messages but not connect to the outgoing e-mail server because it detects that it is no longer on the Roadrunner network.  My shop is on AT&T DSL.  To resolve this, I change the outgoing server address to mobile-smtp.roadrunner.com and the outgoing server port to 587.  This resolves this problem every time.

 

I hope this helps.

ken sheridan say: Are you referring to a LastModified date-stamp column or similar in a table referenced by the query, or to the querydef object's Modified property?  If the latter, this is the date when the query definition was last modified, not its data, and could be something as simple as changing the width of a column in the query's datasheet.

If you want to record when data in a row in a table was last modified then you need to add a column of date/time data type to record this, and assign the current date/time value to it when data is changed.  Note that this is not the same as when the row was updated, which can happen without any changes to the values in the row.

This should be done in a form in which the data is entered or edited.  In fact, regardless of this issue data should only ever be entered or edited by a user in a form, never in a table or query's raw datasheet.  You might like to take a look at ChangedRecordDemo.zip in my public databases folder at:

Yes, the one key problem with many of Greg's posts are that he is at some level a developer, so he generally writes his posts as if he's talking to others with knowledge of programming, since these are the people who most often complain about various technical design issues that also interest him.

However, you misunderstand my response that you might be better off to consider another security application for the same reason you misunderstand my own lack of concern about the currently inconsistent scan frequency of Windows 10 Defender.

The reason is that I've understood for a few years now that any manual or automated scanning on a regular basis is unnecessary due to the real-time operation and so rarely do this anymore myself.  Microsoft had always intended the full scan, for example, as only to be used when malware was either known to have recently attacked or suspected.  Over time I thus moved away from anything other than a scheduled daily quick scan which due to my sporadic use of machines may only run one or two times a week, which is fine with me.

However, the differing experiences of others regarding malware and their levels of comfort with less regular scanning may mean they still retain the generally obsessive tendency to scan regularly as I used to as well.  The history of Windows malware makes this understandable, but since Windows 10 Defender isn't really failing, only the scan's task scheduling is being inconsistent, the options to either perform an occasional manual scan in its place or simply ignore it completely are also fine with people like myself.

So this is the only reason I recommended the option to select another security application in Defender's place, since many of these 3rd-party applications are tuned more specifically to these obsessive tendencies and in fact were actually instrumental in creating them, primarily in order to aid in the sale of these products to the public.

There's nothing wrong with making this switch if that is what you require in order to feel comfortable with the security of your system, I only believe that it's best that people be aware that this is the reason they are making this choice.
