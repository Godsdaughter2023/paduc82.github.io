jennifer_foster say:

Excel to Word mailmerge

I have an excel sheet with client records this data file is merged with the same word file on all occasions.  Is there a macro shortcut key or some visual basic form, a simple one please that I could use to start the merge whilst in Excel.  I only merge the last record in the excel sheet?

graham mayor say:

I think this is one of those instances where if I was going to that destination, I would start from where you are starting from. I don't feel mail merge is the best approach for a single record. I would create a copy of the Word document, unlink the data source and save it as a Word template.

Replace the mergefields with docvariable fields or simply bookmark each field with a unique bookmark name that reflects its content.

It is then a simple matter to create a new document and write the values from the last record, used in the document to from the cells to those docvariables or bookmarks. The following uses a function to write bookmarks, but the values could just as easily be written to docvariables and the docvariable fields in the document updated:

Option Explicit

Sub MergeWorkSheet()
Dim wdApp As Object
Dim wdDoc As Object
Dim lastRow As Long
Dim strBook As String
Dim xlSheet As Worksheet
Dim lngCell As Long
    On Error Resume Next
    Set wdApp = GetObject(, "Word.Application")
    If Err Then
        Set wdApp = CreateObject("Word.Application")
    End If
    On Error GoTo 0
    Set wdDoc = wdApp.Documents.Add(Template:="D:\My Documents\Test\Example.docx")    'The word template
    wdApp.Visible = True
    Set xlSheet = ActiveWorkbook.Sheets(1)
    With xlSheet
        lastRow = .Cells(.Rows.Count, "A").End(xlUp).Row
        FillBM wdDoc, "bkProp_ID", .Cells(lastRow, 1)
        FillBM wdDoc, "bkOwner", .Cells(lastRow, 2)
        FillBM wdDoc, "bkEmail", .Cells(lastRow, 3)
        'etc
    End With
lbl_Exit:
    Set wdApp = Nothing
    Set wdDoc = Nothing
    Set xlSheet = Nothing
    Exit Sub
End Sub

Public Sub FillBM(oDoc As Object, strBMName As String, strValue As String)
'Graham Mayor - http://www.gmayor.com
Dim oRng As Object
    With oDoc
        On Error GoTo lbl_Exit
        Set oRng = .Bookmarks(strBMName).Range
        oRng.Text = strValue
        oRng.Bookmarks.Add strBMName
    End With
lbl_Exit:
    Set oRng = Nothing
    Exit Sub
End Sub

Even when you choose to delete your out-of-date security info, MS don't fully remove it for 30 days.

How to select which Contact folders to sync via ActiveSync on a - Microsoft Community

saxnix say:

How to select which Contact folders to sync via ActiveSync on a Samsung Galaxy S4 running Android 4.2.2. It currently syncs EVERY contact in all folders??

I have just updated to a Galaxy S4 running 4.2.2 from a G3 on 4.2.1. 

On the S3, syncing with an Outlook Exchange account would only sync the Contacts in the default "Contacts" folder of "My Contacts".

With the S4, it syncs all contacts from EVERY Contact folder. This brings in hundreds of contacts I have that I don't want on my phone.  Is there a way to choose which Contact folders you want to sync to the phone, or at the very least, have it just sync the default Contacts folder?

Thanks for any help or advice you can provide.
saxnix say: Thank for replying and providing the info on where to better to post the question.

I have an Exchange Server account I am syncing on the phone.

I actually had contacted Samsung directly and was informed by them that it has been a change in design - to now include all contacts in all folders.  

However, I was able to figure out a workaround for the problem.

  • In Outlook/Contacts use Categorize to designate each contact a Category.  My tip here is to create new Categories for designating types of Contacts.  I have preceded each of my contacts with a "c".  Eg. cFriends, cFamily, cBusiness etc as it will make it slightly easier when setting up the Galaxy S4.   
  • Either one by one, or by selecting multiple contacts, or even entire folders of contacts, right click on selected Contact Card(s), select Categorize, and then chose the category you wish for this/these contacts.  
  • Do this for all your contacts.
  • On the phone select Contacts/Menu button/Contacts to display/Customised list (and the settings cog beside this title. 
  • Select the relevant Microsoft Exchange ActiveSync account and this will give you a drop down box listing several default categories, along with the categories you set up in the steps above.  
  • Simply select the check boxes for the Categories (of Contacts) you wish to appear on your phone.  Any contact belonging to a group you do not select will now not appear in your phone's contacts.
So although it would be nice if Samsung had the functionality to give you the choice of which folders of Outlook's My Contacts you wanted to display on the phone, this workaround shouldn't take too long and achieves the same result.

rob koch say:

Microsoft isn't rated in the tests because starting in 2012 most testing groups started treating it as their baseline, since Microsoft provides all of the major antivirus vendors with much of their sample and other telemetry data in an attempt to help them improve their products, so of course the other products should perform well with this free "leg up".

However, Microsoft also commissioned AV-Comparatives to perform a special analysis of a recent set of results taking prevalence of samples into account, based upon the telemetry data that Microsoft collects via their telemetry from all of their security products.

Microsoft Goes From Cellar to Stellar in New Antivirus Test

http://au.pcmag.com/opinion/23571/microsoft-goes-from-cellar-to-stellar-in-new-antiv

Note that though even Neil Rubenking, the author of the story and a regular critic of both MSE and Microsoft in general, thinks this sounds like a good idea, nothing has changed in the way results are analyzed and displayed for current tests.

This is because nothing about this would help the vendors to sell their AV products any more easily and in fact might actually require them to work harder to make their products more truly effective at dealing with real-world threats, so why would they want the tests to change?

Microsoft originally created its first commercial product Windows Live OneCare and later Microsoft Security Essentials in order to "set the bar" for what a good basic security product should consist of and do in order to protect a consumer PC.  It wasn't ever trying to "beat" any of the existing products or replace them completely, only provide a truly simple and effective protection for those who might not otherwise even purchase one, whether due to cost or simply apathy.

Microsoft still recommends that you find the security product that works most effectively for you, which is exactly what AV-Comparatives recommends in their own disclaimer section in every single report they have released.

However, for me and several friends and relatives I support that has been OneCare, MSE and now Windows 8 Defender for several years and all of us have had less security (e.g. malware) and general operating issues using these than any of the commercial products we'd used in the past.
