    First, you can check if the file is checked out or not. Because when the file is checked out, other users can’t view or edit the files. To check that, you can refer to

    Second, you can go to Library Settings-> Versioning Settings-> Draft Item Security, see which item you checked.

    In the meantime, another possible cause can be the specific item permission. Because we can set individual file permission not to inherit its folder permission, and users can only see items they are allowed to access. To check specific item permission, you can select the item > click the “!” icon on the top right of the page > Manage access > Advanced > check permissions > input the specific external user account having the issue > check whether he has the correct permission.
