scottgem say:

As Bill is telling you, as a general rule, you should NOT store a calculated value. Lets take a look at this. Lets say you add a transaction today. The closing balance on the previous record was $123. The current transaction was $23, leaving a balance of $100. Now you carry this forward and at the end of the week  and several transactions you now have a balance of $232. Then you realize you made a mistake and today's transaction was $22, not $23. Now you have to fix that record and all the subsequent records. 

That's why you don't store a calculation. Balance is a calculation arrived at by adding all credits and subtracting the debits. So if a value changes, you just need to change that record and everything automatically recalculates.

cbarnhorst say: Windows Server for DataCenter has unlimited licensing for VMs.  Consumer Windows does not.  Each VM requires its own license because VMs are portable.  IT managers are thus able to move VMs between host servers for load balancing.  If a VM relied on a host's activation it would require re-activation every time a manager shifted a VM to a different host.  That would be chaotic.  Keep in mind that virtualization is an enterprise-oriented set of features, not a consumer one, so business decisions virtualization providers make are meant to convenience IT.  That we consumers get to use virtualization is a plus for us.

cbarnhorst say: You have to have a new license for Windows to activate the new VM.  Your MS account link does not provide a portable license that you can use on multiple VMs in the way you are hoping.  A VM can be exported and imported but you don't have access to the old VM to do that.

Custom Quick Parts - Distributing - Microsoft Community

peggy1974 say:

Custom Quick Parts - Distributing

We're trying to create a template that can be loaded to Sharepoint, for everybody in a team to create various documents.  The template would have standardized text which we've figured out how to save as Building Blocks.  I've read to save the template in my Application Data folder.  The first time I use that template to create a Word document, I can find the Blocks to use then.  But, after saving, closing and reopening the document, they are no longer in the Quick Parts Gallery.  I saved the document on my desktop, but could need to save to a network drive, or on a sharepoint site, and come back later to edit, and perhaps use more of the Blocks. 

So, how to get the template off of sharepoint, where to save it as a user, then how to use in the document created from the template?  We're stuck.

Thank you.

jay freedman say:

Make a separate template for just the building blocks, and save it under a different name than the template used for basing new documents. Store the template with the building blocks in the folder

%appdata%\Microsoft\Document Building Blocks

(the %appdata% part is an environment variable that points to the Application Data folder on Windows XP or the equivalent AppData\Roaming folder on Windows Vista or Windows 7).

Any building blocks in any template stored in that particular folder will automatically be loaded into Word each time the program starts, and will be available in all documents.
