greg maxey say:

This issue is finally resolved for me.

This so called "Mapi failure - unspecified error," actually has some specification. Prompted by a tip in another forum, I began searching the Windows event logs for anything related to this issue. I found in the Windows Application event log that Outlook was generating an Error ID 27:

"MAPISendMail: Failed to submit message. Attachments to the message exceeded the size limit set by your Administrator," each time I tried sending an attachment from another office suite application.

Now I had never set a size limit. Not with OUTLOOK 2013 or any earlier version. It appears that the when OUTLOOK 2013 was intalled to replace my earlier OUTLOOK 2010, then the "unlimited' attachment size setting is misread and recorded as "0" size in the OUTLOOK 2013 registry setting.

Changing this registry key: HKEY_Current_User>Software>Microsoft>Office>15.0>Outlook>Preferences>MaximumAttachmentSize from "0" to some larger value e.g., "5000" (5 megabyte) has resolved the issue.

This would be inline with an emerging trend towards "Software as a Service" (SaaS) and I would not be surprised to see many software purveyors gravitate towards this model.

I suspect the inspiration behind SaaS is that many software producers either feel as if they are not getting legitimate compensation for all the copies of their software in use, OR they feel the model offers improved opportunities for "secure" software, because they can "push" updates out to End User more readily.

Even some Linux user are [sort of] encountering this already with "Live" Distros because the O.S. must be downloaded as an .ISO onto media for use or installation, and this could be "cut off" at any time.  Period.  End of the "free ride".

Ultimately, if a company like Microsoft was able to produce a high-quality Network Attached Device that could boot up and download its "full" Operating System from a remote server, then they would have complete control over the security vulnerabilities.  It is a bit Catch-22, because despite having an improved security model, I think many Users are "hesitant" [at best] to trust their personal data is safe flying back and forth between Home and Host.  However, what if all your data could remain locally cached and be directly exchanged with the websites you choose to interact with (since this is what is happening now anyway..), and only the O.S. or word processing program (now "App") were the thing being loaded from the remote Host?

There are a great many Users in the world that paid $99 or more to upgrade their Windows, and I do wonder how many of them would have agreed to $7 a month (~ approx. = 14 months) for the same exact thing?

Just one person's thinking on this topic.

dennis_c-jr. say:

See also:

(I think this portion of the Wikipedia page provides a good statement on the future of Windows)

http://en.wikipedia.org/wiki/Windows_10#Update_system

The focus would be meeting the needs of multiple "communities" of potential Users:  One community would be consumers receiving the new version of Microsoft Windows already installed on the hardware they buy (whether PC, laptop, hybrid, tablet or phone), and who absolutely DO NOT want to be worried about the safety and security of their device.  So the updates would be automatic and "transparent" (behind the scenes) to these users; following the logic, 'Just keep my data safe.'

The other community of User is businesses, whether small, medium or large (Enterprises), whose systems are often composed of many vertical (single purpose) applications installed on top of the O.S.  In business the priority is keeping the "business in motion", or keeping the data entry people going, keeping the HR Specialists going, and keeping the Project Managers going, so the revenue stream is not interrupted.

In some cases these vertical applications are (literally) customized in-house by Systems Engineers, Programmers, Database Administrators and the supporting personnel and the programs are so dependent on customizations specific to the Enterprise, an "untested" Update could cause difficulties.  The businesses of run their own Windows Update server in-house to function as an intermediary step in the release (or installation) of updates.

a. user say:

Hi,

The filter option "contains word" differs from "contains" in that it looks only for full-word matches, and not letter matches. Example: if you are trying to filter emails containing the word "Hotmail" in the subject line, an email whose subject is "I have a hotmailaccount" would be filtered with "contains", but would not be filtered with "contains word", because in that case, "Hotmail" is part of
a non-existing word ("hotmailaccount"), and not a standalone word. Put differently: "contains word" actually means "contains these letters as a separate word".

For your 2nd concern It is not possible to create a rule with multiple words, you have to create a rule separately. It is not also possible to scan the content of th email messages.

jan karel pieterse say:

The easiest is to use some VBA user defined functions.

- Open the VBA editor and insert a module (Insert, Module)

- Paste this code:


Function GetState(orng As Range)
    Dim v As Variant
    v = Split(orng.Value, " ")
    GetState = v(UBound(v, 1) - 2)
End Function

Function GetZip(orng As Range)
    Dim v As Variant
    v = Split(orng.Value, " ")
    GetZip = v(UBound(v, 1) - 1)
End Function

Function GetAddress(orng As Range)
    Dim v As Variant
    Dim lCt As Long
    Dim lHouseStart As Long
    v = Split(orng.Value, " ")
    For lCt = 0 To UBound(v)
        If IsNumeric(v(lCt)) Then
            'found house number
            lHouseStart = lCt
            Exit For
        End If
    Next
    For lCt = lHouseStart To UBound(v) - 3
        GetAddress = GetAddress & v(lCt) & " "
    Next
    GetAddress = Trim(GetAddress)
End Function

Next to your data, enter these formulas in three columns:

=getzip(A2)

=getstate(A2)

=GetAddress(A2)

Copy down as far as needed.

Now save file as excel workbook with macros to ensure the user defined functions stay where they are.

Alternatively, buy Excel 2013 and use the new flasfill option.

ronrosenfeld say:

Assuming the entire portion of the string preceding the name of the city is the "Street", one would need a list of cities in order to do this reliably.

Assuming a list of cities in a vertical array someplace, named CityList, the segments can be extracted with the following formulas:

With the address string in A2:

Street:  =LEFT(A2,MIN(SEARCH(CityList,A2&CityList))-1)

City:     =INDEX(CityList,MATCH(TRUE,ISNUMBER(SEARCH(CityList,A2)),0))

State:  =TRIM(MID(SUBSTITUTE(TRIM(A2)," ",REPT(" ",99)),(LEN(TRIM(A2))-LEN(SUBSTITUTE(A2," ",""))-1)*99,99))

Zip:    =TRIM(RIGHT(SUBSTITUTE(A2," ",REPT(" ",99)),99))

The formulas for Street and city are array formulas.  That means that after pasting or typing them in, you need to hold down CTRL-SHIFT while hitting ENTER.  If you did this correctly, Excel will place curly braces {...} around the formula in the function bar.
