Hi Everyone,

Sub ReplaceOnAllFilesInFolderAndSubFolders()

    Dim flpath As String

    Dim macroname As String

    flpath = GetFolder

    If flpath = "" Then Exit Sub

        Application.ScreenUpdating = False

    Call ProcessFolder(flpath)

    Application.ScreenUpdating = False

End Sub

 

Sub ProcessFolder(ByVal p As String)

    Application.ScreenUpdating = False

    Dim fso As Object

    Dim fld As Object

    Dim fl As String

    Dim sfl As Object

    

    Set fso = CreateObject(Class:="Scripting.FileSystemObject")

    Set fld = fso.GetFolder(p)

    On Error GoTo ErrHandler

    

    If Right(p, 1) <> "\" Then p = p & "\"

    ' Process files

    fl = Dir(p & "*.docx")

    Do While fl <> ""

        Call ProcessFile(p & fl)

        fl = Dir

    Loop

    ' Process subfolders

    For Each sfl In fld.SubFolders

        Call ProcessFolder(sfl.Path)

    Next sfl

    Set sfl = Nothing

    Set fld = Nothing

    Set fso = Nothing

    Exit Sub

ErrHandler:

    MsgBox Err.Description, vbExclamation

End Sub


 

Sub ProcessFile(ByVal f As String)

    Application.ScreenUpdating = False

    

    Dim oCell  As Integer

    Dim from_text As String, to_text As String

    Dim doc As Word.Document

    Dim wb As Excel.Workbook

        

    Set doc = Word.Documents.Open(f, AddToRecentFiles:=False, Visible:=True)

    Set wb = Excel.Workbooks.Open("C:\Users\elisabetta\Desktop\culo.xlsx")

    

    For oCell = 1 To 400

    from_text = wb.Sheets("1").Range("A" & oCell).Value

    to_text = wb.Sheets("1").Range("B" & oCell).Value

    With doc

        Set myRange = .Content.Tables(2).Cell(4, 5).Range

        With myRange.Find

            .Execute FindText:=from_text, ReplaceWith:=to_text, Replace:=1

        End With

    End With

Next oCell



    doc.Close SaveChanges:=True

    Set doc = Nothing

    

End Sub

 

Function GetFolder() As String

    With Application.FileDialog(4) ' msoFileDialogFolderPicker

        If .Show Then

            GetFolder = .SelectedItems(1)

        End If

    End With

End Function

jay freedman say:

Number formats in Excel change what is displayed in Excel cells, but the value stored in the cell is not affected by the format. When the value is extracted into a VBA variable, what you get is the stored value, not the displayed text/number.

What you can do is run the from_text and to_text variables through the Format function to apply the custom number format to those strings before they're used in the .Find.Execute command.

charles kenyon say:

I'm writing to address a problem you did not ask about.

Word does not work with flash drives. Using Word with documents on flash drives is the #1 cause of document corruption. You need to act as if you do not have any flash drives while you are using Word. You can use Windows to copy any document on a flash drive to your hard drive, use Word with it, and when Word is closed, copy it back if you wish.


This means:

  • Do not open a document on a flash drive from Word or with Word.
  • Do not save a document to a flash drive from within Word.
  • Do not print a Word document while it is on a flash drive.

There is a problem with "fake" flash memory and USB memory sticks. I do not believe that this is your problem, but, believe it or not, when a memory stick or flash memory reports a capacity, Windows takes its word for it and doesn't check. When the memory has reached its real limit, which is usually far short of it advertised one, it either throws away the overlimit data, or overwrites the oldest files. Either way can result in file corruption. Up to 1/3 of some brands, being sold at cut rate prices and not through legitimate channels that is, are "fake."

For instance you may buy flash memory or a USB memory stick that is advertised as 16GB but is really only 2 GB. When the 2GB limit is reached, either one of those two strategies will occur ending up with either corrupt older files or the one that you are currently working on. The image is from the article in the link below.

 You can download tools to check if you have fake memory and to find out more, here:

http://www.rmprepusb.com/tutorials/-fake-usb-flash-memory-drives

My information may be outdated. If it is, I hope someone will correct me.

zorvek say:

Determining if a cell contains a date can be challenging for the uninitiated. The quickest and easiest way I have found to determine if a cell definitely contains a date is to, in another cell, enter "=A1" where A1 is the cell containing the suspect date. Format that the formula cell as a number and, if it is a number in the low to mid 40K such as:

42762.00

then it is a date.

Another way to tell is to select the cell and, if the value in the formula bar looks like a date but is not the same text as is displayed in the cell, then it's a date:

If the value in the formula bar shows the same text as displayed in the cell, then it's probably not a date but a text string:

When entering dates, it is best to enter the date in the same format as your system has been configured to display short dates. To determine this format, go to your settings or control panel and find the region settings application. Or press the Windows key and S at the same time, type "date format", and press ENTER. Select the drop down menu for "Short date" to see all of the possible formats. Note that the selection of formats changes based on the region selected. For example, "English (United States)" provides mostly month-day-year formats but "English (United Kingdom)" presents mostly day-month-year formats.

Dates can be tricky but, with some experience, are easily managed. One difficult situation is when we receive dates from another country where they use a different date format than ours such as d/m/y when ours is m/d/y and we have to paste that accepted format that is never misinterpreted. Some people think that formatting the dates with a spelled out month makes them universal but months are spelled differently in different regions.

A final note about formatting dates. A feature of Excel is that, assuming you have entered a valid date, Excel will let you format that date any way you want. If you really want your dates to look like "Jan-27-2017" assuming you are entering them using more conventional means so Excel knows they are dates, you can use custom formatting.
