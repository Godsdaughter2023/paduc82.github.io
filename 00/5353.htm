    I dont know what else to try. 

    It still sounds that you did not do it correctly.  This is the process that gets an O/S onto a newly built PC so it has to work.  I used this process twice in recent weeks.

    The process is fairly simple but also can be highly confusing.  In simple terms, you must do these steps -

    1   Change BIOS to load the USB. You can confirm this by USB activity at the start.   The BIOS may also assume that if an USB is not present, then just ignore the BIOS  setting.  Just my opinion.

    2   You must get by the partition setup.  I have found this fairly easy since it will reject obvious bad choices.

    3   It is my understanding that the USB data gets dumped to the hard drive.  This sort of makes sense in that this   process takes some time.  It must be moving data.

    4   You are told at a point to remove the USB and re-start.  This is  often a tricky move.  There is a  point where you must change the BIOS load back to Drive C.  I don't keep notes but if the USB data is now on Drive C and you have been told to remove the USB, this makes sense.

    5  At some point, you want to hit Setup and it is either from USB or Drive C.  I think it is now Drive C.

    If I was you,  I would retrace my steps.  I know it works.  I likely did not get lucky on my first attempt.   You should be able to see the USB activity and Drive C activity.  If there is activity on both fronts, then Drive C should now have the ISO.   Some PCs may default to Drive C if no USB is present.  Some PCs could remain tied to BIOS setting.  The BIOS setting needs a change along the way.

    Give it another whirl.  I know it works.

    Mel

    1-Your browser is set to delete cookies on exit
    2-A new security feature has been added

    Please follow these steps and edit this setting.
    1. Open your Internet Explorer.
    2. Press and hold Alt key then press "T", you will go to Tools menu of Internet Explorer.
    3. Select Internet Options.
    4. Under General tab look for Browsing history.
    5. Untick Delete browsing history on exit.
    6. Click Apply and Ok.
    Close your browser and start a new session.
     
    It is one of the new security changes.
    Part of the change in service include some increased security measures to cut down on the frequency and ease of hacking.

    The new log in ensures a more secure way of preventing unauthorized access to your Live ID accounts.

    Bold Heading but not bold tab stops and page numbers - Microsoft Community

    jane_p say:

    Bold Heading but not bold tab stops and page numbers

    Currently my style heading 2 is bold. When I created the TOC, I also made TOC2 Bold as well. When I refresh the TOC the entire line is bolded. Does anyone know if there's a way to display that heading on the TOC so that the item is bold, but the tab dots and page numbers are not?

    Thanks, PJ

    jay freedman say:
    Sorry, Word doesn't work that way. The TOC styles are paragraph styles, meaning that they apply to an entire paragraph or not at all.

    You could remove the bold setting from the TOC 2 style so that the whole line is not bold, and then manually apply the character style named Strong to the text of the items. That will give you the appearance you want, but it will be wiped out if you update the table of contents (which always applies just the TOC styles to its result).

    jane_p say: Jay, thanks,

    I tried that, and instead of manually bolding/stronging the items in the TOC, I bolded the actual headers manually instead. That way when you refresh the TOC nothing gets wiped out.
