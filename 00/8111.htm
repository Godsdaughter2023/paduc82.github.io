bernie deitrick say: Here's two ways.

1) Let's say that you have values in A2:A10. Instead of putting your SUM formula in A11 and using =SUM(A2:A10), put your SUM formula in A12, and use =SUM(A2:A11). When you want to add new values at the bottom, copy row 10 and select row 11 (which is blank) and insert the copied row. The new value will be included in the SUM result when added to your table in the new row 11.

2) If you convert your range to a table, the formulas at the end of the rows in the last 2 columns will be automatically copied to any newly inserted row. But, if you do use Tables, don't manually insert the SUM functions, but use the table tools to do it after you create the table, and any inserted rows will be included, without using the trick above.

a. user say:

1. If exporting to .csv preserves the amount how you need it, that is probably the quickest way to overcome this in the short term - export, then use the .csv as the data source. (I haven't verified whether or not it would help in this case).

2. Otherwise, use Alt-F9 to reveal the field codes in your letter and use so-called "formatting switches" to apply formatting.

For example, if your amount is ina column called amount, the mergefeild will look like this

{ MERGEFIELD amount }

or perhaps

{ MERGEFIELD amount \*Mergeformat }

If you change that to 

{ MERGEFIELD amount \#"£0.00" }

then alt-F9, select the field, F9 to update it and preview, you should get the format you describe.

If you need thousands separators, use

{ MERGEFIELD amount \#"£,0.00" } (It doesn't really matter where the comma is).

If you search around you will find more on these formatting codes. One starting point is Graham Mayor's page at http://www.gmayor.com/formatting_word_fields.htm

john w. vinson say:

Try three escalating steps if Compact fails:

1. Make a copy of the database file; DELETE the offending form (in the copy); compact and repair. Then use External Data... Access... Import to import the form. If it then works, just use the (fixed) copy and delete the old database.

2. Open the VBA editor (Ctrl-G will do it) and in the Immediate window type:

Application.SaveAsText "MyFormName", "MyFormName.txt"

to write the entire structure of the form out to disk as a text file. Delete the form, compact, and then use

Application.LoadFromText "MyFormName", "MyFormName.txt"

to read it back in.

3. Sigh deeply and create a new form. You may be able to copy and paste controls and labels from the old form, but doing so might import the corruption.

Good luck!
