mike middleton say:

(B) Then the Actual_range is from the cell with value 19 to the cell with value 0.

(C) You must construct the Expected_range yourself. Excel Help says: "Expected_range Required The range of data that contains the ratio of the product of row totals and column totals to the grand total." One approach is to make a copy of the current table somewhere to use for the Actual_range later. If the original table to be modified for Expected_range is in the top left of the sheet with the Bone Quality cell in A1, enter the following formula into B2 (which contains 19): =$K2*B$6/$K$6. Select cell B2, and click and drag the fill handle to J2. With cells B2:J2 selected, click and drag the fill handle down to J5. Each cell in B2:J5 now contains RowTotal*ColumnTotal/GrandTotal, the expected frequency.

(D) My result was a p-value of approximately zero, so I reject the null hypothesis of independence.

(E) Some statisticians recommend the guideline that each expected frequency should be greater than 5, in which case you could combine two columns into a single 80-99 column.

when i send event invites, some recipients using office 2003 receive only a blank email (with no included message). Any settings in need to do in office 2010?
shashirekha say:

On the client (Outlook 2010), click file, options, calendar.  Under the Section Calendar Options, deselect the option that says (second from bottom) "When sending meeting requests outside of your organization, use the iCalendar format".

This feature will depend on both the sender's and the recipient’s mail server. Both the servers need to support Transport Neutral Encapsulation Format (TNEF) which is responsible to retain certain properties for the message which includes meeting invites. The following link provides detailed explanation regarding this feature:

http://support.microsoft.com/kb/290809

schmoov say:

ActiveSync Error Code 85002009

Why do I get this error during synchronization?

schmoov say:

Possible Cause:

The synchronization key data on the device is not valid because a data removal failed.

Possible Resolution:

Try again.  May require user to de-select sync items (Inbox, Contacts, Calendar, Tasks) and re-select them to reset the sync key.
