doug robbins - mvp office apps & services (word) say:

Firstly, for any document, there is no hierarchy of templates.  A document is based on a single template.

I would try a different approach.

  1. Having found a need for MySpecialTemplate.dotm, create a new document from any other template in which you want the styles from MySpecialTemplate.dotm to be available.
  2. Via the Developer tab of the ribbon, access the Templates and Add-ins dialog by clicking on the Document Template item in the Templates section of the ribbon.
  3. Use the "Attach..." button to attach MySpecialTemplate.dotm to the document and check the "Automatically update document styles" box. Then click on OK to close the dialog.
  4. Re-open the Templates and Add-ins dialog and uncheck the "Automatically update document styles" box.  Then click on OK to close the dialog.
  5. Use File>SaveAs to save the document as a template (.dotx), over writing the original template from which the document was created or using a new name.

If the original template did not contain any macros, there is no need to save it as a macro-enabled template (.dotm).  If it did contain macros or user forms, before overwriting the template, use the Visual Basic Editor to export any modules\userforms from the template and then after Step 5 above, use the Visual Basic Editor to import the modules\userforms into the template, which would then need to be saved in macro-enabled format ('dotm).

jay freedman say:
The prompt you're getting on immediately closing the document is caused by the macro printing the document as the last thing. Word will have automatically updated the last-printed date/time, which causes ActiveDocument.Saved to become False. Since Word always checks this value on close, you're being asked to do another save.

In either Doug's macro or mine, you can avoid this by adding this line just before the Exit Sub statement:

  ActiveDocument.Saved = True

If you make any changes in the document after running the macro but before closing, that will  change it to False again and trigger a Save prompt. But if you close immediately, there won't be any prompt.

bob jones aka: cybertaz say:
As for the 2nd part of your question: There is no format specific to Office 2007 or any other version of Word :-)

Prior to Office 2007 [PC] & Office 2008 [Mac] all versions of Office for both platforms used a binary file type. The file name extensions of .doc, .xls, .ppt denote files of that type.

Starting with Office 2007 [PC] & Office 2008 [Mac] Microsoft changed the file type used by Office apps to an XML-based format. Files saved in these types have such extensions as .docx, .xlsx, .pptx and so on. The Office 2011 programs use these latter file types which are identical to the default file types used by Office 2007 & later.

Just make sure that your Save As dialog in Word 2011 indicates a Format: of Word Document (.docx) rather than Word 97-2004 Document (.doc). All current versions of the Office apps can use either file type, but the old binary file type cannot  support many of the newer features... especially graphics. If you "back-save" to the old binary format it can cause undesirable changes to occur as well as possible loss of content.

cody c. say:
Outlook for Mac and Outlook for Windows use different point systems to measure font size. This can result in a font size appearing larger on one mail client than the other. Outlook for Mac does it's best to convert to the closest font size to match it's Windows counterpart. This means that if you choose a font size of 9 on Outlook for Mac, the font size may be received on Outlook for Windows as a size 11. However, the text should be very close to the same size. 

Unfortunately, this is by design. It's also not possible to adjust the size of the Outlook for Mac interface. I'd recommend sending your feedback on this by choosing Help > Send Feedback... in any Office application.
