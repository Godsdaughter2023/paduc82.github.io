david m. connell say:

I think it's normal. Memory management these days tends to focus on using your RAM to improve performance rather than just having free RAM. More memory gets allocated to background processes and stuff when idle, and when you need it, the extra memory is taken from those processes and allocated to your use.

The basic idea is that unused RAM is wasted RAM, so it's better to put it to use than to have a high "free memory" number.

It's not memory leakage, it's just Windows using your memory effectively.
