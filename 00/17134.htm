kseaver say:

I am trying to create a dynamic word document such that a selection from a drop down box populates a text field that is unique to that selection. How do I do this?
greg maxey say:

So if I understand you correctly, your respect for someone and your opinion of them is tied to what that person will or will not do for you?  Ok.

While I am not going to write your specific code for you, here is a general example showing how to define and write text to a targeted text field based on a selection made in a dropdown list:

 

1.  Create the dropdown list content control, title it "Term", and add the terms.

2.  Create and title "Definition" the text field.

3.  Add code to the Document ContentConrolOnExit event procedure:

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
Dim oCCTargetForDef As ContentControl
Dim strDef As String
  Select Case ContentControl.Title
    Case "Term"
      Set oCCTargetForDef = ActiveDocument.SelectContentControlsByTitle("Definition").Item(1)
      Select Case ContentControl.Range.Text
        Case "Respect"
          strDef = "positive feeling of esteem or deference for a person. "
        Case "Professional"
          strDef = " A professional is a person who is paid to undertake a" _
                  & "specialized set of tasks and to complete them for a fee.  " _
                  & "The traditional professions were doctors, engineers ..."
        Case "Spoiled (i.e., child)"
          strDef = "A spoiled child (also called a spoiled brat) is a child that" _
                 & "exhibits behavioral problems from overindulgence by his or her parents.  " _
                 & "Spoiled children may be described ..."
        Case "Freeloader"
          strDef = "A freeloader is someone who exploits chances to get free stuff whenever possible."
      End Select
      With oCCTargetForDef
        .LockContents = False
        .Range.Text = strDef
        .LockContents = True
        End With
  End Select
lbl_Exit:
  Exit Sub
End Sub

Good luck.

Change the "J" to the appropriate drive letter.

If the drive letter may change but the volume label is always the same, use code like the following in the ThisWorkbook code module:

Private Sub Workbook_BeforeClose(Cancel As Boolean)
    Dim DriveLetter As String
    Dim VolumeName As String
    On Error Resume Next
    Application.EnableEvents = False
    ThisWorkbook.Save
    VolumeName = "YourVolumeName"
    DriveLetter = GetDriveLetter(VolumeName)
    If Len(DriveLetter) > 0 Then
        ThisWorkbook.SaveCopyAs DriveLetter & ":\" & ThisWorkbook.Name
    End If
    Application.EnableEvents = True
End Sub

 

Private Function GetDriveLetter(VolumeName As String) As String
    Dim FSO As Object
    Dim DD As Object
    Set FSO = CreateObject("Scripting.FileSystemObject")
    For Each DD In FSO.Drives
        If DD.IsReady = True Then
            If StrComp(DD.VolumeName, VolumeName, vbTextCompare) = 0 Then
                GetDriveLetter = DD.DriveLetter
                Exit Function
            End If
        End If
    Next DD
End Function

Change VolumeName to the appropriate volume name.

In the last issue of WordTips there were a number of readers that got the idea that I was suggesting that people should use the FastSave feature. In fact, I got quite a bit of feedback on that tip, some of it quite pointed. (According to one reader, I was even “irresponsible.” Another stated that I was doing a disservice to readers.)

I am sorry if I gave the impression that I thought FastSave is a great idea. That is far from the truth; I do not. There are several other “features” of Word that I think are ill conceived or totally unnecessary, as well. It doesn’t change the fact that they are features and people can use them, if they desire. WordTips are not designed to promote one method of doing something over another, but to provide alternative ideas that people can incorporate into their Word use, if they so desire.

With that being said, let me point out specifically why you should not use the FastSave feature.

·         Other word processors may not be able to open Word documents saved with the FastSave feature enabled. For instance, WordPerfect says it can open Word documents, yet some versions of WordPerfect cannot open FastSaved Word documents.

·         Third-party software designed to work with Word documents (such as some grammar checkers, document management programs, and desktop publishing software) will not work correctly with FastSaved Word documents.

·         Find File (either in Word or in Windows) may not work correctly with FastSaved Word documents.

·         FastSave results in more complex files being saved on disk. Any time you increase complexity, you run the risk of corrupting the files easier.

·         FastSaved files take up more disk space.

·         There have been reports of macro viruses “living on” in FastSaved documents, even though you thought you had them removed.

The list could go on and on, but you get the idea. In short, unless you have a specific need to use FastSave (such as if you are only creating simple documents using Word 6 on a slow 286 machine), it is probably a good idea to turn it off. The FastSave feature is turned on by default in all versions of Word, even in the original Word 97. Only if you are using Word 97 SR-1 is it turned off by default.

As a side note, according to the Microsoft Knowledge Base there are several circumstances under which a FastSave is not done by Word, even if you have enabled the feature. These circumstances include the following:

·         When you save a document for the first time. (This makes sense, right?) This includes when you choose Save As to save under a new name or in a new location.

·         When your document is saved on a network server or remote volume. (Word only performs a FastSave to a local volume.)

·         When Word reaches the limit of how much information it can save using the FastSave option. In this case Word does a normal save, and then again starts using FastSave for future saves.

graham mayor say:
The backup files are ALWAYS created in the same folder as the original document. There is no mechanism to allow them to be saved elsewhere. If the flag is checked in Word's options to always create a backup copy and that copy is not being created, it is possible that the word data key in the registry where these options settings are stored is corrupt or not being updated through reasons of Windows security.

You could try creating a document and saving it a couple of times, checking for the presence of the backup in the same folder that you saved the document. If that bacdkup is not present, try starting Word as Administrator, turn off the option to save a backup, then turn it back on, to rewrite the registry value. Then test again. If that doesn't fix it, delete the registry key - HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Word\Data - with Word closed, then restart Word which will cause a new default key to be created. Reset the option to create a backup and see if it works now.

As always check for orphan temporary files which can cause all sorts of grief - http://www.gmayor.com/what_to_do_when_word_crashes.htm

The backup is only created if you click Save. It does not create a backup if you SaveAs an existing filename and elect to overwrite at the prompt. The chances of recovering that original document are minimal. Forensic data recdovery may be able to rescure some of it, but I wouldn't hold my breath.

Once you close the document the undo stack is wiped from that document.

jay freedman say:
The first thing is that the zoom factor is 53%. Use the slider in the lower right corner to set it closer to 100%, or click the box that shows "53%" and choose the percentage you want.

The other part is that you have the "white space between pages" hidden, which will also hide any headers or footers you create. You can change this by putting the cursor at the top of the page, just below the ruler, and when the cursor becomes a pair of white blocks, double-click. Alternatively, go to Office button > Word Options > Display and check the box for "Show white space between pages in Print Layout view".

Macro to print specific sheets in workbook

I have an Excel workbook with many worksheets. One of the worksheets is called "INDEX". On the "INDEX" worksheet, these are some menu names in column C and against each menu names, there is either "YES" or "NO" in column F (see table below). Each menu has a corresponding worksheet with the recipe.

Example:

MENU Available Today
CHICKEN BIRYANI YES
VEG BIRYANI NO
STEAMED RICE YES
FRIED RICE YES
CHICKEN KABSA NO
SPAGHETTI BOLOGNESE YES
STIR FRY NOODLE NO

hansv mvp say:

You should either rename the sheets to the menu names (CHICKEN BIRYANI etc.), or create an extra column that lists the sheet names corresponding to the menus.

If you rename the sheets, you can use the code as posted.

If you list the sheet names in another column, you only have to change the line

                a(n) = .Range("C" & r).Value

to for example

                a(n) = .Range("H" & r).Value

if H is the column with the sheet names.
