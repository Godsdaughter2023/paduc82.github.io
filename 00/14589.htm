jay freedman say:
There shouldn't be any difference in the document files created by 32-bit and 64-bit versions of Word 2010. The only place where the difference is important is in executable files -- that is, Winword.exe itself, the printer driver, and any COM add-ins.

Of course, it's possible that a bug affecting the saving or opening of documents could occur in one version and not in the other, but I haven't heard of that (yet).

To make saving in .doc format the default, go to the Options dialog (from the Office button in Word 2007 or the File menu in Word 2010), click the Save tab on the left side, and set the "Save files in this format" dropdown to "Word 97-2003 document (*.doc)". You'll still be able to save a .docx file when you want to, by changing the "Save as type" dropdown in the Save As dialog.

One other thing: the people with Office 2003 can download a Compatibility Pack that will let them open and save .docx files. The download is at http://www.microsoft.com/downloads/details.aspx?FamilyId=941B3470-3AE9-4AEE-8F43-C6BB74CD1466.

Excel 2003 to Excel 2010 Upgrade

I recently upgraded from Excel 2003 to Excel 2010.  Most files run fine.  But one, a fairly large file at 3.65 GB, seems to run very slowly.  When trying to run it I find that Excel itself pauses before execution, even when I'm atrying something simple like just selecting a cell.  My PC is a Lenovo T60 with 2.0 GB RAM and running at 1.83 GHz.  Do I need to upgrade my PC to get this file to run?
rohn007 say:

At that size you are within about 10% of the maximum file size that 32bit Windows can handle. I would be very careful with that file, starting with lots of duplicates / backups. You don't want to loose that much data.

The reason it is frequently pausing is definitely related to file size and RAM. Your 2 GB RAM is not dedicated to the file, the OS and other apps also use it. So to work with the file Windows will be using the swap file, aka Virtual Memory. But the swap file is also on the HD.

First thing I'd suggest is considering reducing the file size. Can you archive off some of the old data? Can you split the file into sub files?

If you do decide to upgrade your computer because you can't shrink the file go with 64 bit Windows and 64 bit Office. Handling large files (4GB +) is one of the specific circumstances when MS recommends 64 bit Office.   But, before you do that, confirm that ny Office Addin tools you use all have 64 bit versions. Lack of 64 bit comptible addons is the main reason MS isn't pushing 64 bit Office.

Suggestion. if you are looking for a new computer, add "USB 3" to your wish list. It is the new USB standard that works about 10 times faster with external USB devices. Just something to watch for.
