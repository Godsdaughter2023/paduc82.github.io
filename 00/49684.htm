¡firedog say:

 Server Response: -ERR [SYS/TEMP] Server error - Please try again later.

As you see, that is a temporary system error at the server, so there's probably not much you can do about it. However, BT accounts are notoriously temperamental when accessed from a mail client as opposed to via webmail. You may, though, be able to take a few steps to make this sort of fault less likely.

The first is to make sure you've entered the new password on the Servers tab of the account's properties dialogue. To open the dialogue, right-click on the account name in the folder pane and select Properties.

Then, it's wise to do a bit of housekeeping at the server end. Sign in to the account at bt.com and start by emptying the folders for spam and deleted items. Create a new folder or folders and move all of the messages out of the inbox and into the new folder. When the inbox is empty, select the account from the dropdown list on the bottom half of the send/receive button in Windows Live Mail.

Assuming you were able to connect to an empty inbox, return to webmail and move half of the messages that haven't yet been downloaded by Windows Live Mail back into the inbox and again try send/receive for this account in Windows Live Mail. If the fault recurs, we now know that there's a dodgy message and also which half of the messages it's in. Repeat the process, taking half of the messages each time until the dodgy message can be identified and dealt with in webmail.

Don't use the webmail inbox as a storage folder. Move any messages for keeping on the server into archive folders, leaving only a few days' worth of messages in the inbox at any one time. Empty the spam and deleted folders regularly.
