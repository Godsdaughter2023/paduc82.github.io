karl timmermans [outlook mvp 2012-2018] say:

To start with the easy portion of your question

#1 - Sort order of User1 & User2 fields

These are text fields which are going to be sorted as per text sort orders (left to right as per characters found). This is true of any "text" field.

#2 - The underlying problem and solution

The reason you were not able to import data from a text file for an Outlook date field is because the format of the date does/did not conform to standards. Would be too involved a process to go through all the acceptable variations. Your solution would be to

a) open the CSV file in Excel

b) convert the column that contains the date values to a "date field" value. If Excel cannot convert the information to a correct date value, you are going to have to modify the values to a valid format (for example, a value of <Dec. 1 1951> will not convert correctly whereas the same value changed to <Dec 1, 1951> will - note the change in punctuation - no period after the month and a comma after the day).

c) once all the dates have been changed to a valid format that can be converted to a standard internal date value, you can then save the worksheet as a CSV file or an Excel worksheet and import the file type of your choice into Outlook - either one will work. If you opt for using an Excel worksheet, ensure that it is saved as an .xls file and that the worksheet includes a valid "named range".
