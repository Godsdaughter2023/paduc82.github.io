ambroz06 say:

Rhialto has the point here.

The problem is that Microsoft's email sending violates the SPF (Sender Policy Framework) spam protection.

Namely, Microsoft sends invitation email from their hotmail.com mail server but with your own email address designated as the sender.

Many mail servers reject these kind of emails as SPF violation. They receive mail from hotmail.com server but designated as sent from you, through your own email provider. This is a typical spammer's way of faking the address so they reject it as spam.

A workaround (tested and proven), until Microsoft corrects this and indentifies their mail as sent from hotmail.com and not from you:

1. A workaround works, if the person you would like to invite, has other email address, that is not subject to SPF validation.

2. Send invitation to this other email address.

3. The invitee will now receive the invitation. The invitee should accept the invitation. You will see it in your Office account.

4. The invitee during/after the installation of MS Office connects to his/her Microsoft/Office account with his "primary" or "intended" email address.

graham mayor say:

The following should do the job

Sub ExtractEMailAddresses()

'Graham Mayor - http://www.gmayor.com - Last updated - 26 Oct 2017
Dim oDoc As Document
Dim oNewDoc As Document
Dim oRng As Range
Dim hLink As Hyperlink
    Set oDoc = ActiveDocument
    Set oNewDoc = Documents.Add
    Options.AutoFormatReplaceHyperlinks = True
    oDoc.Range.AutoFormat
    For Each hLink In oDoc.Hyperlinks
        If InStr(1, hLink.Address, "@") Then
            oNewDoc.Range.InsertAfter hLink.Range & vbCr
        End If
    Next hLink
    oNewDoc.Paragraphs.Last.Range.Delete
    oDoc.Close wdDoNotSaveChanges
    oNewDoc.Range.Sort
    oNewDoc.Range.Find.Execute FindText:="(*^13)@", _
                               MatchWildcards:=True, _
                               ReplaceWith:="\1", _
                               Replace:=wdReplaceAll
lbl_Exit:
    Exit Sub
End Sub

How to display ordinals in date field in mailmerge Word 2010 from - Microsoft Community

w.m.mitchell say:

How to display ordinals in date field in mailmerge Word 2010 from Excel data source?

I've looked for last couple of days for solution to issue but haven't had luck beyond the basic date display.

A mailmerge was bringing up a date problem that is quite common.  The display wasn't right as I was getting a numerical display where Word 2010 was just taking the basic date in Excel without the Excel formatting.

Solution easy to find on net.  Didn't have to follow complicated way of changing anything in options and confirming any conversion business, there's the much easier option of just changing the mergefield coding, like this:

     { MERGEFIELD Date '\a "dddd, MMMM d, yyyy}

That gives me this display on today's date:

     Wednesday, August 5, 2015

That's great and it's simple!  However, I actually need ordinals, like so:

     Wednesday, August 5th, 2015

I found that there is a possible switch in \ * Ordinal, but only found examples on regular field codes and nothing that actually worked in the mailmerge scenario.

Can anyone help, please, in getting ordinals in the syntax so that all the dates show the appropriate one according to the number?  i.e., st for 1st, nd for 2nd, rd for 3rd, th for 4th, etc., etc.?? 

I understand from some posts and results via googling that superscript may not be possible (?) and that is secondary.  Certainly ordinals in superscript would be ideal, but I can live without that.

Thanks for any help!

doug robbins - mvp office apps & services (word) say:

The following combination of fields:

{ MERGEFIELD Date \@ "dddd, MMMM" } { MERGEFIELD Date \@ "d" \* ordinal }, { MERGEFIELD Date \@ "yyyy" }

will give you Wednesday, August 5th, 2015

macropod say:

I found that there is a possible switch in \ * Ordinal, but only found examples on regular field codes and nothing that actually worked in the mailmerge scenario.

Can anyone help, please, in getting ordinals in the syntax so that all the dates show the appropriate one according to the number?  i.e., st for 1st, nd for 2nd, rd for 3rd, th for 4th, etc., etc.?? 

I understand from some posts and results via googling that superscript may not be possible (?) and that is secondary.  Certainly ordinals in superscript would be ideal, but I can live without that.

Superscript, too, is possible! You can use a field coded as:

{QUOTE{SET MergeDate {MERGEFIELD Date}}{MergeDate \@ "dddd 'the' d"}{IF{=(MOD({={MergeDate \@ d}+89},100)>2)*(MOD({={MergeDate \@ d}+9},10)<3)}= 1 {=MOD({MergeDate \@ d},10)-2 \# rd;st;nd} th}{MergeDate \@ "' of' MMMM, yyyy"}}

where 'Date' is the name of the field containing the date in the data source. Note the superscripted ordinals in the field code.


Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues. For a macro to convert either of the above into working field codes (after which you'd need to re-apply the superscripting), see Convert Text Representations of Fields to Working Fields in the Mailmerge Tips and Tricks threads at:
http://www.msofficeforums.com/mail-merge/21803-mailmerge-tips-tricks.html
or:
http://windowssecrets.com/forums/showthread.php/163017-Word-Mailmerge-Tips-amp-Tricks
