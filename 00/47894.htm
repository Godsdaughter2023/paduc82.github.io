Being a USB adapter - a few things can make or break the connectivity.
 
There is the simple location factor.  Is it connected in such a way  that it is up high with as unobstructed path to the router itself as possible?  If needed - some sort of USB extension could be used, preferrably one that is not too long or an active extension cable.
 
There are the motherboard chipset drivers.  Since this is a USB device, if there is some communications issue between the USB ports and the operating system you have chosen to utilize - that can cause issues like this.  Make sure your motherboard chipset drivers are current via the motherboard manufacturer's (or computer manufacturer's OR chipset manufacturer's) web page.
 
There is, of course, the wireless device drivers themselves.  There seems to be two different versions of the product (conveniently version 1.0 and version 2.0) and I can only hope, looking at the drivers, that you have version 2.0 (if you do not - your problem could be simply that.)  Even version 2.0 hasn't been well taken care of (the last driver update for Windows 7 was 11/09/2009 --> which tells me that support for this device has pretty much died and they likely consider this a legacy product and were already starting to consider it that way then.)  In any case - make sure you have the latest driver from here:
http://homesupport.cisco.com/en-apac/support/adapters/WUSB100

santosh y say:

Hi hackneylad,

This issue can occur if the folder containing the files does not have “SYSTEM” permissions or if it was removed somehow.

To troubleshoot this, do the following:

1. Right-click the folder which is not returning results.

2. Select the Properties option.

3. Click the Security tab.

4. If SYSTEM appears in the Group or user names box, system permissions is not the problem.

5. If SYSTEM does not appear in the Group or user names box, click the Edit button to open the Permissions dialog.

6. In the Permissions dialog, click the Add button.

7. In the Select Users, Computers, Service Accounts, or Groups dialog, enter system in the Enter the object names to select box and click the Check Names button.

8. Select SYSTEM from the list of Matching names and click OK.

9. Check if the issue is fixed.

Reference Link: Windows Search 4.0 Troubleshooting Guide


Hope this helps. Let us know if you need further assistance.
