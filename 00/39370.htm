The default GINA DLL for XP is msgina.dll and there is no reference in the registry for it and there is not supposed to be - it loads by default if there is no reference in the registry for a GINA DLL to load.

Some software you install wants to load it's own GINA DLL and when that software installs, it adds an entry in the registry to point to their GINA DLL so theirs loads instead of the default msgina.dll.

If their DLL comes up missing somehow - perhaps from a reinstall, update or uninstall of their software - or maybe it was quarantined (effectively deleted) by your unspecified antivirus/antimalware on your system of unspecified make and model with unspecified XP version or Service Pack, the registry is trying to load the file that has come up missing hence the error message.

cmdrkeene say:

If you are just typing a single word, you're pretty much done. Only documents that contain that word will be shown.  If you type more than one word, only files that contain both words are listed.  But of course those two words could be pages separate from each other, so you can get better results by using quotes.  It's still an exact match, but instead of files that just contain all the words, it's an exact match for that phrase specifically, verbatim.

Unless you use quotes, multiple words are always considered to be "AND'ed" together. So nsight teresa (without quotes) gives the same results as if you had typed nsight AND teresa to find files that absolutely have both words.  But you can also do exclusions, nsight NOT teresa, or teresa NOT nsight.

You can also consider that perhaps that word is being found in some other meta-data, filename path/directory, or something other than the body of the document. So, try this:

contents:"keyword"

Only files that contain the given keyword inside the file's contents (not meta data) will be listed in the results.
