Possible to move XP to new computer without installation CD? - Microsoft Community

kevin short say:

Possible to move XP to new computer without installation CD?

I have an old computer that I got from a friend who didn't want it anymore, but now it's so outdated that there's not much I could do with it. It has Windows XP installed, and I would like to know if I could move XP without the installation CD to a new computer I recently built. I'm currently running Linux on my new PC.
databaseben say:

you won't be able to because moving the hard drive to a totally different system will engage xp's anti piracy feature.

 

you see, when xp is initially installed, it gathers information about all the hardware of that computer and builds a security hash.

 

afterwards, every time xp is booted up it gathers information about the hardware and compares it to the hardware hash to ensure little or no variances with the hardware.

 

so basically, if xp is booted up on that newer computer, the hardware hash it has stored on the disk will not match the information it will gather on the new computer.

 

however, if you had the xp installation cd and the product number matches to the o.s. on the hard drive, then you can do a repair installation and a new hardware hash would be created.

 

also, keep in mind that the old xp may not run on newer computer systems anyway.  so not only is the hardware hash a factor for xp, but incompatible hardware might be as well.

 

as a suggestion, you might want to network the old computer with your new one.

What to do if you forget your Windows password

If you've forgotten your Windows password and you're on a domain, you should contact your system administrator to reset your password. If you're not on a domain (most home users are not on a domain), you can reset your password by using a password reset disk or by using an administrator account.


Create a password reset disk:

http://windows.microsoft.com/en-us/windows7/Create-a-password-reset-disk

Reset your Windows password:
http://windows.microsoft.com/en-US/windows7/Reset-your-Windows-password


If you forget the administrator password and don't have a password reset disk or another administrator account, you won't be able to reset the password. If there are no other user accounts on the computer, you won't be able to log on to Windows and will need to re-install Windows.

Reinstall Windows Preinstalled On Computer


OEM computer manufacturers are required to provide at least one way for you to reinstall Windows if you buy a computer with Windows preinstalled - your computer should have included either a restore partition on your hard drive, or Windows install/restore disks.


If the restore partition is still there and was not removed, it will reinstall Windows 7. There is usually a prompt when you first start the computer telling you what key to press to start the restore﻿ - check your manual if you're not sure how to. This will remove any personal files on the computer, back them up first.


If you don't have the restore partition:

Contact the OEM computer manufacturer and ask them to send you an install disk for your computer - they may or may not do it. If they agree to send you one they may charge a small fee.

jaymo1978 say:

Hey DebtlessMoon - I totally understand, the repair process (specifically, the shipping) can be extremely frustrating, I've been there. I do know that when a tracking number shows "Shipped," it does not always mean that it has left on its way to you. Often, USPS says "Shipped: Status - Shipping Information Received," and UPS will say something like "Electronic shipping info has been transmitted." I know Fed Ex also has some version of this, but the bottom line is, this wording means that the shipping process has started by notifying the carrier that a pickup is ready, but the package hasn't been scanned at its initial pickup yet.

Now, that said, I had a couple of times when I was waiting for a 360 to be shipped back and the package actually got to me before the tracking info updated. When the FedEx rep tells you "No, we don't have it yet," they're just giving you the info they have based on the tracking. I had this experience also.

So, long story short, the 2-business-day shipping usually refers to the time in-transit once it is picked up/scanned by FedEx. So, you may get it soon, if FE didn't scan properly, or within 2 days after they scan it.

What does your tracking say, specifically?

stevew_f say: Months later... I concur that my M51 running Windows 7 (x86) is happy now that I updated the driver of the unknown device.

- Download the exe file found at the link in the original message of this thread.
- Run the exe file that you downloaded. It's a self-extracting executable, meaning that its sole function is to extract a bunch opf files from within itself, placing them in a location of your choice on your computer. Note that location when you run the exe file.
- Start > right-click "Computer" > Properties > Device manager
- Scroll down the list of devices until you find an unknown device, right-click it > Properties > Details, Property = Hardware IDs
- Check that the hardware ID reads "ACPI\NSC1100". If it does, then continue; otherwise, the driver you're looking for is elsewhere (suggestion: web-search the text that you find under the Hardware IDs).
- Now choose the "Driver" tab > Update Driver > "Browse my computer..."   and navigate to where the files were extracted from the exe file that you'd downloaded > Next.

The unknown device should then become identified as "Winbond Trusted Paltform Module 1.1"  (to see it, you may have to click the "View" menu in your Device manager, and turn on "Show hidden devices".

Note that the source page of the download does NOT list the M51 as one of the models to which this driver applies. Kudos to the person who figured out that the driver does nevertheless apply to the M51  (which, at release date, was "Designed for Windows XP").
