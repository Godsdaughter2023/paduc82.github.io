ken sheridan say: Why bother with the subform's recordset at all?  In the subform's Current event procedure disallow additions by default with:

    Me.AllowAdditions = Me.NewRecord

In the Click event procedure of the button to enter a new record allow additions and move to an empty newrecord:

    Me.AllowAdditions = True
    DoCmd.GoToRecord Record:=acNewRec

In the Click event procedure of the button to move to the next record handle the error if the form is at the last record:

    Const CANNOT_MOVE = 2105
    Const MESSAGE_TEXT = _
        "Sorry, this is the last goal.  To create a new one click the New Goal button."
    
    On Error Resume Next
    DoCmd.GoToRecord Record:=acNext
    Select Case Err.Number
        Case 0
        ' no error
        Case CANNOT_MOVE
        ' anticipated error
        MsgBox MESSAGE_TEXT, vbInformation, "Invalid Operation"
        Case Else
        ' unknown error
        MsgBox Err.Description, vbExclamation, "Error"
    End Select

rohn007 say:

OK, that is definitely fluky.  Frankly I'm shocked it worked.

Office is normally VERY fussy about matching the bit version of all Office elements.

The disadvantage as I see it is unpredictability.  While it appears to work now, it may crash at any time. And, I expect that the next time you try install some other Office component you could run into an error complaining about mixing 32 and 64 bit installations.

Play it safe, get the 64 bit Proofing tools.

Question, why are you switching to 64 bit. MS still recommends (and defaults) to 32 bit because many addons (including/especially  ones from MS) still do not have 64 bit versions.

MS only recommends going to 64 bit if file size on the HD is approaching 2GB or RAM use in the application is approaching 2GB in the application with the file open. Most typically this happens in Excel, with large data sets downloaded from the internet.

The data file structure is not affected by the application bitness, so you can share other files with 32 bit users without problems.  Consider only moving specific users who explicitly need 64 bit.
