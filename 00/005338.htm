For question 1, you'd do something like:

=Left([YourFieldName], InStr([YourFieldName],"-")-1)

 

For question 2, you need to create a VBA procedure to do what you need in which you will utilize functions like Split(), Mid(), InStr(), InStrRev(), Left(), Right(), ...

Hi Doug,

I've been running some tests on an Intel SSD, with Windows 7 installation and Windows 10TP. I have also used Macrium Reflect Free, to Image a back up of installations to HDD and restore to SSD.

What I've discovered is: If you fresh install W7 or W10TP directly to SSD, you have a really fast and efficient installation. The OS will have configured the installation to support the SSD (It will also create the correct system alignment that SSD requires). It is different in configuration to an installation to HDD which is then imaged to SSD. (I've just read that back to myself and I'll try to clarify)

An installation to HDD is different to an installation to SSD. There are changes to system config and services and the way they run.

SSD installations are a bit leaner and it seems to involve less needless writes to the drive.

Just a suggestion: Back up your W7 activation with a tool like Advanced Tokens Manager. Do a fresh install of W7 to the SSD - Re-activate and then add the W10TP you want to dual boot to. You will be seriously impressed.

Sorry it's not the answer to your question, but if you are able to do this, it will make a noticeable difference to performance and prolong the life of your SSD.

How can I restore my Word 2003 Autocorrect backup doc (.doc) - Microsoft Community

randycantrell say:

How can I restore my Word 2003 Autocorrect backup doc (.doc) to Word 2010?

My wife's old PC bite the dust. Thankfully, I had a full .doc backup of her thousands of auto-corrections. 

New PC with Windows 7 and MS Word 2010 enter the picture.

Am trying to find way to get the .doc backup file into Word 2010. I do not have the support.dot file, but I do have the complete backup file in .doc format. I tried a macro I found, but it failed. 

I searched for ways to convert a .doc file to a .acl file (used by Word 2010), but found nothing. 

Thanks in advance for any help.
jay freedman say:
Open the .doc backup file and look at how it's laid out. If it's a table with three columns, where each row has the name of the entry in column 1, the text of the entry in column 2, and either False or True in column 3, then you can use it.

Download http://jay-freedman.info/AutoCorrect2007.zip and unzip the AutoCorrect.dotm template from it to your Templates folder (%appdata%\Microsoft\Templates). Start Word 2010 and make a new document based on that template. In the dialog that appears, click the Restore button. When the Open dialog appears, choose the backup .doc file.

The macro in that template has rarely been used with "thousands" of entries, and I suspect it may crash at some point. If that happens, try splitting the job by copying portions of the table from the original backup document to new blank documents, and then running the Restore on one portion at a time.

jay freedman say:
Don't worry about it. The macro only adds entries to the installed items, it never removes any. So if you get entries named A, B, and C from the first chunk of the backup, and then you get D, E, and F from the second chunk, the result is you have all six entries in the .acl file.

The only time you get something different is if two or more items in the backup file (and/or in the current .acl) have the same name. Then the final result is one entry having the the value that was in the last entry loaded by the macro. If all your entries have unique names, that won't happen.
