graham r seach say:
>>Why was replication discontinued in ACCESS 2007?
Because it was incredibly buggy! You were just lucky with the the process you were using. There are a bunch of technical problems with Jet replication that would eventually have led to your
 
Seriously though, Jet replication never did work properly and professional developers avoided it like the plague. Microsoft has moved to a different replication architecture now; one which is much more stable and scalable. That architecture is SharePoint. Aside from writing your own synchronisation engine, the only way you can get it working in 2007 or 2010 is to push all your data to SharePoint, and use the native synchronisation. Then you just distribute a copy of your Access database, which will then be in 'offline mode'. When you get it back, it will automatically sync with SharePoint, and everybody's happy.
 
Another option is to make SharePoint available to all your remote users. That way, you don't need to distribute copies of your database. They just use the copies they have, and it becomes a wide-area multi-user system.
 
I know that's going to cost you since you'd then have to either purchase and install SharePoint, or pay for SharePoint hosting, but there's no other (easy) way.

I can answer some of your questions.

For distributing themes, you can make your life a little simpler by opening a new file in PowerPoint, applying your Font, Color and Effect themes to it, then saving it as a Theme file (.thmx). This theme file can be applied to Excel and Word documents and it has the same effect and applying the 3 themes types separately.

While you can apply a theme manually to to any open document, it's a manual operation that must be repeated every time a user creates a new file. The simple way to get around this is to apply the theme to each user's Normal template and resave, (it can end with either .dotx or .dotm), but you've already said you don't want to do this.

In Excel, you can create a template called book.xltx, apply the theme, then save it to Excel's XLSTART folder to set it as the default template. To set a default presentation in PowerPoint, you create a template called Blank.potx (you can just resave your Theme file with a different name and file ending) and save it to the Templates folder.

Custom Document Templates is usually created by Word when templates have been migrated from a previous version, so it may not exist on every computer. You can place user templates anywhere in the Templates folder.

jay freedman say:

Specifically regarding the Custom Document Templates folder, which is within the “C:\Users\%username%\Documents" folder rather than the ...\Microsoft\Templates folder:

By default, when the user creates or modifies a template and uses the Save As dialog, Word 2013 will automatically switch to the Custom Document Templates folder. This is an (in my opinion misguided) attempt to make users' templates "roam" (by copying them to OneDrive) so they're available on any device where the user logs in with the same Microsoft account; and possibly to make it easier to back up only user data and not the default stuff from the Office installation.

To make Word ignore the Custom Document Templates folder and save all templates to the Templates folder: Go to File > Options > Save. Type the path of “C:\Users\%username%\AppData\Roaming\Microsoft\Templates" into the "Default personal template location" box. (I suspect the lack of a Browse button for this box is meant to make it harder to change this setting.)

One other observation: Besides not being able to hide the default themes, I believe there's no way to prevent users from creating new themes or modifying any existing theme. This will have to be a topic of user education.
