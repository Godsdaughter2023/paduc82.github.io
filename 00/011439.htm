john mcghie say:
Microsoft Word doesn't have a default font: that's part of your difficulty.  What you need to do is set the font in the Normal style in the template you are using to create documents.

As you say, that's usually the Normal template.

But it's not Normal.dot, and it's not in the Microsoft User Data folder.

The file is named Normal.dotm, (note the 'm' in .dotm) and it's stored in:
 /Users/ ~ /Library/Application Support/Microsoft/Office/User Templates

Where ~ is your login name.

The easiest solution will be to delete that file: because I suspect it has picked up bad settings from the old Normal template in your MUD folder.  But before deleting Normal.dotm will work, we need to get rid of the old templates in the MUD, otherwise, Word will just make a new copy on startup.

So:  Quit Word,

Then delete any files named Normal or Normal.dot that you can find.  There may be one of each in the Microsoft User Data folder.  You said there was one in  > Apps > Microsoft Office 2011 > Templates > My Templates?  If there is, delete that one too.

Now go to " /Users/ ~ /Library/Application Support/Microsoft/Office/User Templates" and delete the one you find in there.

Now re-start Word and wait for the blank document to appear.  A clean new Normal.dotm file will be automatically created in this step.

Go to Format>Style, make sure the Normal style is selected, and click "Modify".

Set the font as you wish, and at the bottom of that dialog, check the "Add to template" box, to make sure your change is written back to the template.

Now save, and close Word (to ensure that your change is also saved to the Normal.dotm template).

When you restart Word, any new documents you create should have the font setting you require.

For existing documents, you will need to re-set the font in the Normal style,a nd you may also need to reset any non-style formatting that has been applied.

Hope this helps  

daniel g. grau say:
You are getting confused with so many Normal templates. This is where each version of Word stores its Normal template ("~" represents your home folder):

  • Word v.X: Applications/Microsoft Office X/Templates
  • Word 2004: ~/Documents/Microsoft User Data
  • Word 2008 & Word 2011: ~/Library/Application Support/Microsoft/Office/User Templates

You have a Normal template in Microsoft Office 2011/Templates/My Templates, which you must have copied from Word v.X. Word 2011 does not use this folder.

Important concept: Clicking on any template will create a document based on that template. When you modify the font through Format > Font and click on Default, you are modifying the template on which the document is based. According to your description, then, you have been modifying the wrong Normal templates.

What you need to do is double-click on the correct template for Word 2011, as indicated in the bulleted list above.

To create a blank document based on Word 2011's Normal template, it's easier to type Command N.

tom van stiphout say:

Before I try to answer that, the screenshot seems to indicate you do not have Referential Integrity between the two tables. That is a REALLY BAD idea that should be corrected right away. Put both tables on the Relationships window, right-click the link line between them, and in the ensuing dialog check the box to enforce. Do that for all your relations. Not checking this makes it just a notational nicety without any teeth.

Your expression Forms!subfrmWorkOrderDetails!workorderdetailsID seems to indicate that workorderdetailsID is on a subform, on some unknown parent form. That is a problem, because say you have two form/subforms open with this same subform, and different records are selected in the subform, which one is Access supposed to use when you run the query? It can't and this is illogical.
Rather you are supposed to include the parent form as part of the expression, like this:
Forms!myParentForm!subfrmWorkOrderDetails.Form!workorderdetailsID
This says to use the subform on myParentForm, then step into that subform (with .Form) and get the value of the workorderdetailsID.

InfoPath 2010 - How to show an optional section based on user - Microsoft Community

davesbonnylass say:

InfoPath 2010 - How to show an optional section based on user selection in a control?

I have a repeating table that contains a drop-down list called "AbsenceType". If the value of the user's selection = "S", then I want another optional section to appear. I've already created the optional section, but don't know what to do next. Step-by-step instructions are required as I'm new to this!

jimmy rishe say:

First, I would suggest using a regular section and not an optional section as optional sections can't be removed or added from and to the form without code.  Once you've got your regular section control:

  1. Click Manage Rules on the Home tab to open the Rules panel if it's not already shown.
  2. Click the section you want to conditionally show
  3. In the Rules pane, click New -> Formatting
  4. Click the blue None text under Condition: and use this to set up the condition AbsenceType is not equal to S
  5. Check the Hide this control checkbox.

That should do it!
