daniel pineault say:

why you reccomend me to store in other folder than C drive?

Because it is simply not the way to do things.  You can find tons of discussions on the subject by Googling.  For one thing, newer OS won't even let you install files directly in the root drive like that without getting into UAC security problems!

I personally privilege using the Documents or %AppData% folders with a sub-folder for the specific database.

"While it's logical to think of an Access application as a program file, the Access model breaks that logical model, because it has to be writable. The only proper place for any Access app (since Windows 2000, in fact) is the folder the %AppData% environment variable points to." - taken from http://stackoverflow.com/questions/3610409/how-to-change-data-in-ms-access-database-saved-in-program-files

also of interest:   http://stackoverflow.com/questions/3665605/appdata-for-ms-access-installs  and many more if you search.

Beyond the security issues, think of it this way for a second.  In your house (apartment, wherever you live) do you just drop everything at the front door in one big pile or does everything go in its' own place in different room, closets,...  There needs to be a clear organization for things to work properly.  Same goes for your computer.

hansv mvp say:

Right-click the mailmerge field in the merge document in Word.

Select Toggle Field Codes from the context menu.

You will see something like

{ MERGEFIELD FieldName }

or

{ MERGEFIELD FieldName \* MERGEFORMAT }

where FieldName is the name of the mailmerge field.

Make the field code look like this:

{ MERGEFIELD FieldName \# ",0.00" }

You can use more or fewer decimal places if desired.

Press F9 to hide the field code and to update the result.
