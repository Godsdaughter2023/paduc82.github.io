gordonshowers say: There are two sets of logs, one of which you can delete individual entries from & the other you can't.

When you first enter the "Log" application, you have "Recent calls", "Call duration" and "Packet data". Entering these will allow you to delete individual entries (as the manual describes).

However, the comprehensive list you get by pressing the right-key will only allow deletion of ALL entries. This will also clear all entries in the other lists.

The only way around it is, as you say, deleting all entries. Or setting the log duration to a shorter interval (1 day, for example).

You need to ask yourself: how important is a log at all?? Especially if one or two inappropriate entries are in the list!

I have the following macro code to update a brief table with Date, Time and the users logged on initials, so that I can track who accesses the workbook and when.  However, sometimes the macro works and then sometimes it doesnt and returns a Runtime error '1004' Application-defined or object-defined error.  I have checked that the table is being referenced and the cell range names are correct as is the table name.  The same snippet of code works in another macro I have to update more than 3 cells in a different named table.

 

Can anyone suggest where I have gone wrong.

 

Public Sub Login()

Dim Login As Variant
Dim ActiveUser As Range
Dim reply1 As Integer
Dim oSh As Worksheet
Set oSh = Sheets("UserActivity")

Application.ScreenUpdating = False
Range("ActiveUser").ClearContents
Login = InputBox("Enter your initials to login...", "Logging on...")
Range("ActiveUser").Value = Login
    With oSh.ListObjects("UserActivity")
        .Range.Select
    End With
   
    Dim oNewRow As ListRow
    Set oNewRow = Selection.ListObject.ListRows.Add(AlwaysInsert:=True)
    oNewRow.Range.Cells(1, 1).Value = Format(Now, "dd-mm-yy")
    oNewRow.Range.Cells(1, 2).Value = Format(Now, "hh:mm:ss")
    oNewRow.Range.Cells(1, 3).Value = Range("ActiveUser") & "-IN"

Application.ScreenUpdating = True
reply1 = MsgBox("You are now logged in.  Remember to log off (Ctrl + Shift + M) when you are finished!", vbOKOnly, "Logged in Successfully!")
reply1 = vbOKOnly

End Sub

Rethink if it makes sense to add a row to ListObjects("UserActivity") every time, because you clear any cell before, that's a contradiction.
