ossiemac say:

Hello Matt,

This answer is a little more than you requested but I thought you might find the extra helpful.

The code example you posted doesn't seem to gel with your following statement so I have assumed the following statement and coded accordingly. "So if I hit a button on the row of 6, it would delete cells G6,K6,M6."

 

If you use the Forms Control buttons in lieu of ActiveX buttons then all of the buttons can call the same code. Application.Caller identifies the calling button.

The first Sub in the following examples will create the forms control buttons on each of the rows. See my comments where you might need to edit the code. The buttons are named "btnRow" & r (eg. btnRow6) where r is the row number to be cleared by the button. Application.Caller identifies this number for the called code to determine which row to clear.

Note the code where Move and Size are set. (Same as right click the button and selecting format control -> Properties.) Uncomment your required option in the code and comment out the other 2 options.

The second sub is the code that is called by the button. You might want to edit this to add your MsgBox etc.

The third sub is a utility to delete all of the buttons. You might find this helpful during development.

Sub CreateButtons()
    Dim ws As Worksheet
    Dim lngLeft As Long
    Dim lngTop As Long
    Dim lngHeight As Long
    Dim lngWidth As Long
    Dim r As Long
    Dim c As String
    Dim shpButton As Shape
   
    Set ws = Worksheets("Creator")   'Edit "Creator" to required sheet
    c = "E"     'Edit "E" to required column for buttons
   
    With ws
        lngLeft = .Columns("E").Left          'Assumes left and width will all be the same
        lngWidth = .Columns("E").Width
        For r = 6 To 26     'Edit 6 to 26 to required rows for buttons
            lngTop = .Rows(r).Top               'To will change for all rows
            lngHeight = .Rows(r).Height     'Height could change depending on row height
                                                                   
            Set shpButton = .Shapes.AddFormControl(xlButtonControl, lngLeft, lngTop, lngWidth, lngHeight)
            With shpButton
                .Placement = xlMoveAndSize      'Move and size with cells
                '.Placement = xlFreeFloating     'Don't move or size with cells
                '.Placement = xlMove             'Move but don't size
                .Name = "btnRow" & r
                .TextFrame.Characters.Text = "Clr Row " & r
                .OnAction = "ClearCells"
            End With
        Next r
    End With

End Sub

Sub ClearCells()
    'this is the code that is called by the buttons
   
    Dim r As Long
   
    r = Mid(Application.Caller, 7)     'Get row number from suffix of button name
   
    With ThisWorkbook.Worksheets("Creator")
        .Cells(r, "G").ClearContents
        .Cells(r, "K").ClearContents
        .Cells(r, "M").ClearContents
    End With
   
End Sub

Sub DeletShapes()
    'Utility to delete all buttons
    Dim shp As Shape
   
    For Each shp In ActiveSheet.Shapes
        If shp.Type = msoFormControl Then
            If shp.FormControlType = xlButtonControl Then
                shp.Delete
            End If
        End If
    Next shp
End Sub

rich007a say:

In the file you uploaded, I assume the offending cell is Sheet1 L3.  First off, I found this formula had been entered as an array formula (confirmed with Ctrl+Shift+Enter, and thus being contained in {}).  There is no need for this, just type the formula and hit Enter.

Second, the formula is   =VLOOKUP(F3,Sheet2!A2:K1220,6)

So it's looking for the value in F3 which is   "ABL-BLA"

It is going to search for that value in Sheet2!A2:A1220, i.e. the left column of Sheet2!A2:K1220.  The first occurance of "ABL-BLA" in column A is on row 8 (A8).

The 6 at the end of your formula tells it to return from the 6th column from the left of A:K, which is a, b, c, d, e.... F.  So since "ABL-BLA" is found in Sheet2 on row 8, the value of Sheet2 F8 will be returned, which is.....

BLANK!

If you want the formula to return from column H ("productdescription") then use:

=VLOOKUP(F3,Sheet2!A2:K1220,8)

but looking at the title of Sheet1, L2 "METADESCRIPTION", I suspect your formula with 6, not 8, is correct.  It's just that cells F4 and below are blank.  That's why they are not returning a value.

Does that fix it?

One final point.  If you plan to drag this formula down to other cells below L3, you need to set absolute references in the table_array.  You should use:

 =VLOOKUP(F3,Sheet2!A$2:K$1220,6)

(correct the 6 to return the required column, 6 will return from column F).

Cheers
Rich

jay freedman say:

Using any web search facility to look for "free fonts" will turn up hundreds of sites claiming to have tons of free fonts. There are two major concerns when looking at these sites, whether they're "safe" (not likely to pass along malware) and whether the fonts are really free.

The first is easier: Most of the sites with lots of traffic, which occupy the top of the search results, are safe. Good choices include fonts.com, dafont.com, fontzone.net, and others. You should also be using an always-on antivirus program that will block access to sites that are known to be risky.

When you're looking at specific fonts, be aware of which ones are really free, which ones are trials that are free for a limited time, and (this is harder to determine) which ones are really copyrighted fonts that shouldn't be free at all.

Beyond those considerations, some sites have better organization than others, such as searching for categories of fonts, and showing all the characters in each font rather than a short sample.
