jay freedman say:
> Jonathan may have been writing in a simpler time when this worked better

Indeed. The article dates from Word 2003 or earlier. To try to stem the onslaught of Office macro viruses, Microsoft imposed increasingly stringent requirements on macro-bearing files. Starting in Office 2007, "trusted locations" were introduced. If there are macros in any document or template stored in an untrusted location, Office refuses to run those macros unless they're digitally signed (or if the user is dumb enough to enable all macros).

If you want to use macros in templates stored on a server, you have to have each user do this:

  • Click File > Options > Trust Center > Trust Center Settings > Trusted Locations.
  • Check the box at the bottom of the dialog for "Allow Trusted Locations on my network (not recommended)".
  • Click the Add New Location button.
  • Navigate to the server location where the template is stored.
  • Click OK.
  • In Options > Advanced > File Locations, set the Startup location to point to the server location.


The reason Microsoft says trusted locations on a network are "not recommended" is that they're more easily hacked, with greater spread of damage, than local folders. It seems unlikely to me, though, that someone who can hack into your server will do so with the intention of replacing your add-in templates with ones containing malicious macros; there's so much else more valuable.

dianeoforegon say:
Try deleting this file in your User Library/Preferences folder:

com.apple.internetconfig.plist

Log out/in.

If you still have problems setting as default...

1)  Use one of these two utilities:

RCDefaultApp... RCDefaultApp is a preference pane that allows a user to set the default application used for various URL schemes, file extensions, file types, and MIME types. (look under Apps -> iCal)
http://www.rubicode.com/Software/RCDefaultApp/

http://www.flickr.com/photos/dianeross/6199079996/

MisFox - Missing Internet Settings For X
http://mac.clauss-net.de/misfox/

2) Set file association:

    Set Mail to open ".eml" filetypes
    Locate a message file by opening Mail and  drag a message to the desktop.
    Do a Get Info (Cmd + I) Under the "Open With" section, select "Mail" from the drop-down menu (or locate the application if it is not in the menu) and then click "Change All..." to associate all .eml filetypes with the Apple Mail.
