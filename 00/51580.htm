The user profile folders (C:\Users\Username) are protected system
folders and _cannot_ be renamed, without risky registry editing, even if
the associated user account has been renamed, which is what I surmise
you did. So, if you don't really want to restore the machine to its
original, ex-factory state, your best course of action would be to log
on using the built-in Administrator account, create a new user account,
with the username desired. Once you've logged in using this new account
(and transfered any data you need to preserve, you can then delete the
old user account(s).

That is the way it works - and also why people often fret when their system appears to be hung on mup.sys when booting in Safe Mode.  There is nothing wrong with mup.sys for it has already loaded and XP has moved on to the next thing.  Instead of the system being "stuck", it is probably really just running a chkdsk with error correction.

If you have scheduled a chkdsk to run and reboot in Safe Mode, or if, when XP checks the volumes on the next reboot it finds that one of them has been marked "dirty" and XP runs the chkdsk with error correction, the chkdsk is going to run after mup.sys loads that and it could take a long time.  The system will appear to be stuck on mup.sys, but it is really running the chkdsk you scheduled, so you will have to wait it out.  If the HDD lite is blinking, it is doing something, so wait.

This disk checking can take a long time (perhaps several to many hours) depending on the size of the volume, the amount of data on the volume and what the disk checking finds to do.

Depending on the the speed of your system, the size of the volume, the amount of data on the volume and what chkdsk finds to do, it may take a long time for chkdsk with error correction to complete (several to many hours) or it may appear to be 'stuck'.  Be patient.  If the HDD light is still flashing, chkdsk is doing something so don't interrupt it.  It will finish eventually one way or the other.

When a system appears to hang on mup.sys many folks set about a time wasting effort to replace mup.sys but mup.sys is not the problem for it has already loaded (pat attention Microsoft engaged Support Engineer "experts").  The problem is what come after mup.sys - and that is the time XP checks to see if a chkdsk with error correction needs to be run.

If the system is running a chkdsk with error correction either because one has been scheduled to run on the next reboot (then you boot in Safe Mode) or because XP has determined that the volume has been marked "dirty" one can observe the HDD access LED if there is one or "listen" to see if the HDD sounds like it is "working" on something...  then it is bet to just wait it out.

To see the "problem" schedule a chkdsk with error correction to run on the next reboot, then boot in Safe Mode.  The last thing you will see on the screen is mup.sys having loaded and then your system is hung until the chkdsk with error correction finishes and that could take a very long time.

One common reason a HDD can appear lame is if it is an IDE drive and the Transfer Mode is set to PIO instead of some kind of DMA mode (check the Transfer Mode settings in Device Manger).  If that is the case, everything will run slowly - and the chkdsk with error correction can take an extra long time to complete.

Another reason a system can appear to hang up mup.sys is an afflicted USB device so if one is pretty sure the problem is not a running chkdsk with error correction the troubleshooting method is to disconnect every external device from the system (especially USB devices) leaving only the keyboard - and preferably a PS2 keyboard since you only need a keyboard to boot and operate your system.

If the system behaves with just a keyboard then start adding the devices back one at a time, reboot after each device is added and it the system hangs again, the last device you added back to the configuration is the suspect.

This is my generic how to for proper driver updates : (BIOS and major drivers) 

This utility makes it easy to see which versions are loaded :

DriverView - Free - utility displays the list of all device drivers currently loaded on your system.
For each driver in the list, additional useful information is displayed: load address of the driver,
description, version, product name, company that created the driver, and more.
http://www.nirsoft.net/utils/driverview.html

For Drivers check System Maker as fallbacks and Device Maker's which are the most current.
Control Panel - Device Manager - Display Adapter - write down the make and complete model
of your video adapter - double click - Driver's tab - write down the version info. Now click UPdate
Driver (this may not do anything as MS is far behind certifying drivers) - then Right Click -
Uninstall - REBOOT this will refresh the driver stack.

Repeat that for Network - Network Card (NIC), Wifi, Sound, Mouse and Keyboard if 3rd party
with their own software and drivers and any other major device drivers you have.

Now go to System Maker's site (Dell, HP, Toshiba as examples) (as rollback) and then Device
Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get their latest versions. (Look for
BIOS, Chipset and software updates at System Maker's site while there.)

Download - SAVE - go to where you put them - Right Click - RUN AD ADMIN - REBOOT after
each installation.

Always check in Device Manager - Drivers tab to be sure the version you are installing actually
shows up. This is because some drivers rollback before the latest is installed (sound drivers
particularly do this) so install a driver - reboot - check to be sure it is installed and repeat as
needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER - check
manually by model.

Manually look at manufacturer's sites for drivers - and Device Maker's sites.
http://pcsupport.about.com/od/driverssupport/ht/driverdlmfgr.htm

Installing and updating drivers in Windows 7 (updating drivers manually using the methods above is
preferred to ensure the latest drivers from System maker and Device makers are found)
http://www.sevenforums.com/tutorials/43216-installing-updating-drivers-7-a.html

If you update drivers manually then it is a good idea to disable Driver Installations in Windows
Updates, this leaves Windows Updates ON however it will not install drivers which will usually be
older and cause issues. If Updates suggests a new driver then HIDE it (Right Click on it) and then
go look for new ones manually if you wish.
