a. user say:

You have to find out where the high pitched tone is coming from. Think of what typically spins in a computer: the power supply fan, the case fan, the CPU fan and the hard drive's disk platters.

If your computer has an optical drive, that's the first thing to check. Typically, the optical drive spins up briefly when a computer reboots, as a test that it's working properly. That does make a sound. The same is true of a floppy drive, but hardly any computers have one of those.

As your computer restarts, stick your ear up against the case and try to locate more or less where the sound is coming from. That will tell you which of those 5 devices is the likely culprit.

If it's the power supply fan, the case fan, the CPU fan or the optical drive, it may simply be the result of dust and dirt. Blow it out  - carefully - with a can of compressed air. If that doesn't help, you'll probably have to replace the affected part. OEMs often use the cheapest parts they can find. I've already replaced my case fan and my power supply. (Oh, that was fun.)

If the sound is coming from the hard drive, act quickly! Contact the hard drive manufacturer as soon as possible.

Hi Mehdi,

Thank you for using windows 10.

The error message usually indicates that a kernel-mode process tried to execute an incorrect or unknown processor instruction. This error handler is a default error handler that catches errors not associated with other specific error handlers. The error could happen due to buggy device driver or incorrect system service settings or compatibility issue with any new hardware installed.

I suggest you to try following methods and check if that helps:

Method 1:
Perform SFC scan and check if that helps.

System File Checker is a utility in Windows that allows users to scan for corruptions in Windows system files and restore corrupted files.

You can follow the steps in the following article:

http://answers.microsoft.com/en-us/windows/wiki/windows_other-system/system-file-check-sfc-scan-and-repair-system-files/bc609315-da1f-4775-812c-695b60477a93

Method 2:
Run the Deployment Image Servicing and Management (DISM) tool which fixes the Windows corruption error if there are any. Follow the steps:

  1. Press Windows+X.

  2. Click on Command Prompt (Admin).

  3. Type the command DISM.exe /Online /Cleanup-image /Restorehealth and hit Enter.

You may refer to the following link: https://support.microsoft.com/en-us/kb/947821?wa=wsignin1.0      (Note: Although the link is for previous version of Windows, but the commands can be used)

Method 3:
Perform Clean Boot and check.

A clean boot is performed to start Windows by using a minimal set of drivers and startup programs. If everything works fine after a clean boot, you can infer that some third party services are causing the problem. Proceed with the remaining steps to pin-point out the third party service. After you figure out the problem causing program, you may have to update or install a higher version of the program, if you seldom use that software you can consider uninstalling it.

Refer to the following Microsoft Article for performing Clean Boot:

https://support.microsoft.com/kb/929135

Disclaimer: Once you perform clean boot do refer to the section “How to reset the computer to start normally after clean boot troubleshooting” to boot the computer in normal mode.

If the issue gets resolved in Clean Boot, then refer to the section “How to determine what is causing the problem by performing a clean boot” in the above mentioned article to identify the program or service which is causing the error.

Hope the information helps. Get back to us for further assistance and we’ll be happy to help.
