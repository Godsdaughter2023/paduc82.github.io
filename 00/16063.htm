maor bracha say:

[New Office 365 portal] Directory synchronization, part #1- Enabling directory synchronization using the readiness wizard

Technical Level : Basic

Summary

After adding your domains to Office 365 (as described here), the next step you might want to consider is directory synchronization.

Directory synchronization allows us to replicate users and groups from our local AD environment to Office 365 and Azure AD, in order to work using centrally managed unified entities.

More information about directoroy synchronization tool and implementation methods here

In order to configure directory synchronization we will be required to prepare our local AD environment, install a sync client and enable Office 365 for synchronization. 

In previous versions, enabling an Office 365 tenant for directory synchronization was just a simple toggle switch.

With the new Office 365 portal things has changed a little, and in this article we will go over the basics of enabling directory synchronization using the readiness wizard.


Details

Directory readiness tool

Navigating to the wizard

As described in the summary- the new Office 365 portal provides an improved experience when it comes to directory synchronization preparation. In order to enable synchronization, we first have to go through the readiness tool.

To run the tool you will be required to logon to the Office 365 portal from a domain machine, preferably with a privileged user.

Go to users, click more > Directory Synchronization.

A new interactive box will appear (figure #1) > Click "Go to DirSync readiness wizard" (Figure #2)

Figure #1 - Navigating to the directory synchronization menu

Figure #2 - Navigating to the readiness wizard.

Running the wizard

Once on the wizard page, you will be asked for your organization size- please select the appropriate figure and continue.

A "Start Scan" button will appear (figure #3).

You will be taken to a new window, and an application permission and download request will appear. Please allow it to run. (Figure #4)

Figure #3 - Starting scan

Figure #4 - Running the wizard

When the scan completes, you will be notified (figure #5) and taken to the summary page by clicking next.

Figure #5 - Scan Complete

Figure #6 - Scan summary

The above page allows us to brief the summary and make sure the result estimate is correct.

By clicking next we will be taken to the domain verification page, which will notify us about possible issues with our domains that may effect directory synchronization (Figure #7)

Figure #8 - Domain summary

You may noticed in the above example (figure #8) that my domains show as none-routable.

o365lab.knowit.se show as such since there is no MX record, but the one we should notice is "Knowit.local" - this is a local domain and therefor cannot be synced to Office 365 \ Azure AD. Objects using this domain will either be excluded or routed to the "@X.onmicrosoft.com" (or other default domain, if changed).

Cleaning up your environment

Moving on- you will be taken to the IdFix page. This page will allow you to download the IdFix tool in order to scan your local AD environment for possible issues, for instance:

•Duplicate items (such as duplicate aliased, email addresses, UPN)
•Top level domains (email or UPNs containing local domains, which cannot be synced)
•Corrupted items (such as ambiguous letter in alias, missing UPN, etc.

For example, let's refer to the "knowit.local" domain that showed a "none routable" error on the page (figure # 8).

The user *** Email address is removed for privacy *** (figure #9) will be synced to Office 365, but will be assigned a UPN of *** Email address is removed for privacy *** as that is the default domain in our tenant.

Please refer to part #2 of this article series - "Using IdFix to identify and remidiate possible issues with your on-premise AD"

Come back to this article after completing the described steps in part #2.

Figure #9 - Faulty user example

Run Azure Active Directory Connect

Please refer to part #3 of this article series - "Directory Synchronization, part #3: AD-Connect- Installation, configuration and running a first sync cycle"

Come back to this article after completing the described steps in part #3.

Completing the readiness wizrd

After completing the steps described in chapters 1 & 2 you are ready to complete the readiness wizard.

Click "Next" in the window prompting you to install AD-Connect (Figure #10)

Figure #10 - Move on from AD-Connect installation

Make sure sync worked as expected

The next window will suggest that you do some manual verification, on top of Microsoft's checks, to verify your first sync cycle was successful.

To do so in the most basic level you can go to the active users windows and make sure your newly synced users show up as expected (Figures#11 & 12)

 

Figure #11 - Make sure sync cycle worked as expected

Figure #12 - Active users

Activate Synced users

If you have licenses you wish to assign- you can use this page to do so, or skip by clicking "next" and license users using the "active users" panel later on.

All Set!

Congratulations! You now activated directory synchronization and performed your first sync cycle!

Office 365 will now take you to the directory synchronization health monitor to provide some overview. You can reach this monitor at any stage and time by navigating to "Health" and then "Directory Sync Status"
