john - project say:

Spongmonger,

First let me give you a caution about linked projects. Unfortunately linked structures in Project, (i.e. master/subproject, resource pool/sharer files and combinations thereof), are prone to corruption. None of the files in the structure should ever be renamed, moved, overwritten, or saved off to another location. All files should reside in a single directory on a local drive and the structure should not be operated over a network.

A static master (i.e. uncheck the "link to project" option) is not a linked structure and therefore not prone to corruption.

Okay, now that you've been warned, let me answer your question on duration. Project only allows one definition for a day so no, it is not possible to have a 7.5h task appear as 1 day duration and a 8h task also appear as 1 day duration. If you want to steer clear of that limitation, change the units of the Duration field to hours.

Hope this helps.

John

ken sheridan say: A few comments:

1.  Equality comparisons in Access are by default case-insensitive, so calling the UCase function serves no purpose unless you have set the Option Compare statement to make binary comparisons.  Alternatively the strComp function can be used to make case-sensitive comparisons, e.g. in the debug window:

? strComp("XXX","xxx",vbBinaryCompare)
-1
? strComp("xxx","XXX",vbBinaryCompare)
 1
? strComp("xxx","xxx",vbBinaryCompare)
 0

Note that the function does not return a Boolean value, but -1 or 1 if the first string is less than or greater than the second respectively, and 0 if they are equal.  So a return value of zero would indicate a case-sensitive equality of the two values.

In the context of this thread case-sensitive comparison serves no purpose whatsoever, however.

2.  If branching rather than in-line error handing is used then the error handler should be called immediately before the first executable line in the procedure.

3.  The line On Error GoTo 0 should not be included either following that which calls the Execute method or in the error handler, which should at the very least also handle unanticipated errors.  I only included it in my original crude example to turn off error handling, and allow the system to report any errors.
