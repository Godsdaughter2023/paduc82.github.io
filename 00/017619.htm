mike h.. say:

Hi,

It works fine in Excel 2007 IF  you implement it correctly. Put this code in aGENERAL module. i.e. right click 'ThisWorkbook' insert module and paste the code in there

Sub ToggleCutCopyAndPaste(Allow As Boolean)
     'Activate/deactivate cut, copy, paste and pastespecial menu items
    Call EnableMenuItem(21, Allow) ' cut
    Call EnableMenuItem(19, Allow) ' copy
    Call EnableMenuItem(22, Allow) ' paste
    Call EnableMenuItem(755, Allow) ' pastespecial
    
     'Activate/deactivate drag and drop ability
    Application.CellDragAndDrop = Allow
    
     'Activate/deactivate cut, copy, paste and pastespecial shortcut keys
    With Application
        Select Case Allow
        Case Is = False
            .OnKey "^c", "CutCopyPasteDisabled"
            .OnKey "^v", "CutCopyPasteDisabled"
            .OnKey "^x", "CutCopyPasteDisabled"
            .OnKey "+{DEL}", "CutCopyPasteDisabled"
            .OnKey "^{INSERT}", "CutCopyPasteDisabled"
        Case Is = True
            .OnKey "^c"
            .OnKey "^v"
            .OnKey "^x"
            .OnKey "+{DEL}"
            .OnKey "^{INSERT}"
        End Select
    End With
End Sub
 
Sub EnableMenuItem(ctlId As Integer, Enabled As Boolean)
     'Activate/Deactivate specific menu item
    Dim cBar As CommandBar
    Dim cBarCtrl As CommandBarControl
    For Each cBar In Application.CommandBars
        If cBar.Name <> "Clipboard" Then
            Set cBarCtrl = cBar.FindControl(Id:=ctlId, recursive:=True)
            If Not cBarCtrl Is Nothing Then cBarCtrl.Enabled = Enabled
        End If
    Next
End Sub
 
Sub CutCopyPasteDisabled()
     'Inform user that the functions have been disabled
    MsgBox "Sorry!  Cutting, copying and pasting have been disabled in this workbook!"
End Sub

Now double click 'ThisWorkbook' and paste this code in

'*** In the ThisWorkbook Module ***

 
Private Sub Workbook_Activate()
    Call ToggleCutCopyAndPaste(False)
End Sub
 
Private Sub Workbook_BeforeClose(Cancel As Boolean)
    Call ToggleCutCopyAndPaste(True)
End Sub
 
Private Sub Workbook_Deactivate()
    Call ToggleCutCopyAndPaste(True)
End Sub
 
Private Sub Workbook_Open()
    Call ToggleCutCopyAndPaste(False)
End Sub

Now very important  close VB editor, save the workbook as a macro enable workbook (.xlsm extension) close the workbook and re-open it ensuring you enable macros and it will work fine.

al borges say:

Try this:

'looks like [Mnths] is a number
DoCmd.RunSQL "SELECT QryMagoyeSalary.* AS PayeFin FROM QryMagoyeSalary WHERE QryMagoyeSalary.[ZZDone] = " & [Mnths]  &  ";"

'if [Mnths] is a string expression
DoCmd.RunSQL "SELECT QryMagoyeSalary.* AS PayeFin FROM QryMagoyeSalary WHERE QryMagoyeSalary.[ZZDone] = " & Chr(34) & "*" & [Mnths] & "*" & Chr(34) & ";"

dhookom say: Can you provide the full sql view that includes the DLookup() in it? Is the DLookup() one of the selected columns? What are the ****.

ken sheridan say: The fact that you are calling the DLookup function rather than a domain aggregation function implies that there is only one matching row in QryMagoyeSalary  for the value of Mnths passed into the function.  From this it follows that you can use a join, which is far more efficient than a subquery:

SELECT SELECT QryMonzeSalaries.EmployeeID, QryMonzeSalaries.NapsaFinal,
QryMagoyeSalary.Paye AS PayeFin
FROM QryMonzeSalaries INNER JOIN QryMagoyeSalary
ON QryMagoyeSalary.ZZDone = QryMonzeSalaries.Mnths;

This would reproduce what the DLookup function is doing, but I'm surprised that neither the function's criteria nor the join criteria of the above include the EmployeeID.

scottgem say:

 but I'm surprised that neither the function's criteria nor the join criteria of the above include the EmployeeID.

ken sheridan say: I'm aware of the bug, which, in the examples I've seen, appears to arbitrarily substitute the four asterisks for a comma.  What surprises me is that something which needs 'to pull 3000 employee details'  does not include the EmployeeID in the join criteria.  However, the main point here is that calling the DLookup function is an inefficient way of effecting a join operation, which is most efficiently done as such.
