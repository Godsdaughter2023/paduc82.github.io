ken sheridan say: Mea culpa!  It should have been:

SELECT *
FROM [YourTableNameGoesHere]
WHERE NOT(State = "Texas"
AND Code = "BKCL");

As I'd written it the Boolean AND operations required both columns to be NOT equal to the values, whereas what's needed is for both columns to be equal to the stated values, and then for the NOT operation to be applied to the result of the AND operation, which is what the above statement does.

It can also be expressed as a Boolean OR operation:

SELECT *
FROM [YourTableNameGoesHere]
WHERE State <> "Texas"
OR Code <> "BKCL");

Office 365 Planner - permission on folder level

We what to create a "plan" which should contain a lot of files - we will create several folders -

Question : Can we control permission on a folder and/or file level - we dont what all "members" to see all files on the plan...

jiaxing bian msft say: Hi Anders,

Yes, we can maintain unique permissions for specific folders and files. Here are the detailed steps:
1. Open the plan in Planner, click the ellipsis behind Notebook and select Files to open the group document library.
2. Select the folder or file, and then click Share > Shared with > ADVANCED.
3. In the advanced permission window, click Actions and select Edit permissions to stop inheriting permissions from its parent. When prompted, click OK.

 

4. Repeat Step 2 to open the advanced permission window, and now we can customize the permission for users and groups.
For example, select the Members' and Visitors' groups, and then click Actions > Remove User Permissions to remove their permission.

 

5. After that, only the Owners' group will be able to see the folder.
