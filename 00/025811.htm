marshall barton say:

That use of ! is not part of the SQL spec.  It should be a dot, but Access tries to help people that are confused by the syntax in different contexts.

Technically, bang is used to specify a member of a collection while dot is used to specify a field in a table and a property or method of an object.  The confusion is made worse because a control in a form/report is both a member of the form/report's Controls collection and a property of the form/report object.

The whole idea behind Applescript is that ordinary users can automate applications in Mac OS using a relatively straightforward sentence-like programming language. While posting to a site that is primarily for Windows Office developers can't hurt I think this forum is the correct forum for questions such as yours. amit tandon say:

Try below formulas, these will not count blank cells as unique and should give you the correct result:

Valid for both numbers or text in column:

=SUMPRODUCT((A$2:A$100<>"")/COUNTIF(A$2:A$100,A$2:A$100&""))

or

this is valid only for numbers (not text):

=SUM(IF(FREQUENCY(A$2:A$100,A$2:A$100)>0,1))

The above formulas return the number of unique values in column A (so that duplicates are counted as 1).

If you directly want to count the number of values which are duplicates (occur more than once), you may use below formula as an array formula (CTRL+SHIFT+ENTER):

=SUM(IF(FREQUENCY(IF($A$2:$A$100<>"",MATCH(A2:A100,A2:A100,0)),IF($A$2:$A$100<>"",MATCH(A2:A100,A2:A100,0)))>1,1))
