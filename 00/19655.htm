hansv mvp say:

Please note that C2:G35 contains only 34 rows.

Can the selection be any range of 5 columns and at most 35 rows? Or should it be within columns C:G? If it can be anywhere:

 

Sub MoveSelection()
    Dim rngSrc As Range
    Dim rngDst As Range
    Set rngSrc = Selection
    If rngSrc.Rows.Count <= 35 And rngSrc.Columns.Count = 5 Then
        Set rngDst = Range("C2").Resize(rngSrc.Rows.Count, 5)
        If Intersect(rngSrc, rngDst) Is Nothing Then
            rngSrc.Copy
            rngDst.PasteSpecial Paste:=xlPasteValues
            rngSrc.ClearContents
        Else
            MsgBox "Source range overlaps destination range!", vbExclamation
        End If
    Else
        MsgBox "Source range doesn't have correct dimensions!", vbExclamation
    End If
End Sub

Completely bypassing your actual question, which John will address perfectly if you provide enough information, your plan has several other severe and serious problems which you should be aware of.

You have predecessors and successors on the summaries. This is universally considered a poor practice for several reasons. Mainly, a summary is not a task. It is just a way to organise the tasks in the WBS. Only a task can be a predecessor or a successor. Also, it will create myriad difficulties when tracking progress.

Also, you have some tasks with no predecessor at all, and others which do not have a FS predecessor.

Every task should have at least one FS0 predecessor and at least one FS0 successor.

Also, one task has a date constraint, which looks like it might be a MFO constraint. Tasks should not have date constraints, especially the strong ones (MSO, MFO), for various reasons.

Also, you have durations in weeks. It would be better to confine durations to days or hours.

Also, your line 1 task is a manually scheduled task (summary, actually). Better if it is auto scheduled.

Also, you do not have line 0. On the format tab ribbon, at the right hand end, is a check box show project summary. It should be checked.

The instructions to move an IMAP data file to another hard drive are shown here; https://www.howto-outlook.com/howto/movingostpstoutlookcomimap.htm

If this is a usb connected drive, prior to shutting down win, or rebooting, you have to use the safely remove process to eject the usb.

Likewise when booting up the usb should not be connected.

You connect the usb only when win has finished loading, you then start Outlook.

A usb connected drive was never designed to be connected during a win shutdown or start-up, if you do you run the risk of any data file being corrupted.

A usb connection is also slower than an internal drive, even if you use a ssd or sata.

There would be no impact with any other data files, or programs being located on C:\
