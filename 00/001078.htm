Need help summarizing fiscal year and fiscal quarter data - Microsoft Community

lesliedalton say:

Need help summarizing fiscal year and fiscal quarter data

In an access 2003 database, I'm trying to design a report that lists Fiscal Year (FY), Fiscal Year Quarter, (FYQ) and a Claim Count. I've designed the query which includes nested IIf expressions for both the FY and FYQfields. I created a summary query to count the number of claims.How do I list all four quarters for each fiscal year even if there were no claims? I would like to return a count of 0. Right now, I'm only getting the FY and FYQ for those quarters that have claims.

 

ken sheridan say: While you could return the accounting years and quarters a little more efficiently using the Format function as Karl suggests (or simple customized functions, which is how I've always done it), as your original queries are working successfully, on the basis of if it ain't broke don't fix it, you can stick with them (subject to the removal of the HAVING clause as explained below).

To solve the problem of the absent accounting years/quarters you'll need to provide these by creating a separate all-inclusive table.  This can be a general purpose calendar table, from which you return distinct rows for each accounting year/quarter, but it would be more efficient to create a table specifically for this purpose, FYearsQuarters say, with columns FY and FYQ, which would have rows like this:

2011-2012        1
2011-2012        2
2011-2012        3
2011-2012        4
2012-2013        1
and so on

You can then LEFT JOIN this to your final summary query, calling the NZ function to return a zero in place of a Null for the absent quarters, and computing the calendar year from the accounting year/quarter:

PARAMETERS [Starting fiscal year:] TEXT(9),
[Ending fiscal year:] TEXT(9);
SELECT NZ(CountOfClaimID,0) AS ClaimCount , Department, Division,
FYearsQuarters.FYQ, IIF(FYQ>2,Right(FY,4),Left(FY,4)) AS CalendarYear,
FYearsQuarters.FY
FROM FYearsQuarters LEFT JOIN qryYourSummaryQuery
ON FYearsQuarters.FY = qryYourSummaryQuery.FY
AND FYearsQuarters.FYQ = qryYourSummaryQuery.FYQtr
WHERE FYearsQuarters.FY BETWEEN [Starting fiscal year:] AND [Ending fiscal year:];

As you are using parameters in this to restrict the results to particular accounting years, you can remove the HAVING (((tblClaims.InjuryDate) Between [New Injury Start Date] And [New Injury End Date])) clause from your qryExpNewInjuriesDir query.  You must do this as otherwise, if there were a discrepancy between the two date ranges defined, the final result set could inaccurately report some quarters as having no claims when this is not the case.  Note also that it is important to declare the parameters as text in the above query to avoid their being misinterpreted as arithmetical expressions.
