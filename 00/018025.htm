MS Project 2010 Task Updates: Zero hours done, what do I do? - Microsoft Community

mrhmeek say:

MS Project 2010 Task Updates: Zero hours done, what do I do?

When updating a task in PWA for MS Project Server 2010, and no hours have been done on a task as identified, do I need to put insert that 0 hours have been done and send an update - or by not inserting anything it gets the message through?

We have a single entry mode setup whereby time-sheets and task updates can be done at the same point.

Thanks

dale howard [mvp] say:

MRHMeek --

When no work has been done on a task, I recommend you tell your team members not to enter any progress for that task in their timesheet, and then submit the timesheet at the end of the week.  They do NOT need to type zeroes in the timesheet to show no progress.

On the flip side, you as the PM would need to reschedule uncompleted work on that task from last week's reporting period to this week.  After you have approved task updates from last week, open the enterprise project.  For any tasks on which no progress was entered last week, you can reschedule the remainder of those tasks by doing the following in Microsoft Project 2010:

  1. Click the Project tab to display the Project ribbon.
  2. In the Properties section of the ribbon, click the Project Information button.
  3. In the Project Information dialog, enter the date of the LAST DAY of last week's reporting period in the Status Date field.  For example, if last Friday was the last day of the reporting period for last week, enter the date for last Friday.
  4. Click the OK button.
  5. In the Status section of the ribbon, click the Update Project button.
  6. In the Update Project dialog, select the Reschedule Uncompleted Work to Start After option.
  7. Select the Entire Project option and then click the OK button.

After doing this, Microsoft Project 2010 will reschedule all uncompleted work from last week to this week.  If a task was supposed to start last week, but no work was done, the software will move the entire task.  If a task was in-progress prior to last week, the software will create a task split, and only move the uncompleted portion to this week.

Hope this helps.

dale howard [mvp] say:

MRHMeek --

No, rescheduling uncompleted work from the past to the current reporting period will not work like resource leveling.  If the rescheduling operation causes a resource overallocation, then you must level the overallocation.  Hope this helps.

Try this as a test.

Open this presentation.

Choose Design (Deseno) then Slide Size , Custom Slide Size.

Write down the Width and Height of the slide then close the dialog box.

Insert a rectangle on the slide.

Change the position to 0 horizontal and 0 vertical.
