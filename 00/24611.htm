PMFJI, but just to shed some light on what you're up against :-)...

The core challenge is that you are attempting to use Word for a purpose for which its design is diametrically opposed. The type of project you're describing is handled far better by a Desktop Publishing program. Such programs provide an object oriented environment which readily supports page layout features.

Word is designed to produce manuscript-type documents: Its document structure is based on text flow rather than object orientation. Inclusion of graphic objects can be done, of course, but that is an ancillary capability introduced to the program long after its foundation was firmly established. All objects are attached to - and therefore controlled by - the flow of text. Objects, essentially, are foreign bodies injected into the document bloodstream (i.e., text flow). The greater the number of objects the more difficult it is to manage the flow of the document accordingly as well as maintain the placement & integrity of the objects.

terfar say:
Last Tuesday was Windows Update day. Did this happen after the updates installed? If so the following may be appropriate.
 
 
That's a symptom of a Word Data Key corruption - usually because Word has
been running when updates are installed.
 
Run Regedit and expand by double-clicking to the key below. With DATA
selected, press DELETE. Word will automatically recreate a new key when it
is restarted.
 
HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Word\Data
 
Symptoms of Data Key corruption include:
 
   •    The mouse does not work when you use Word.
   •    You cannot open a Word document from the Search window in Windows
Vista.
   •    You cannot open a Word document from Windows Desktop Search.
   •    Word crashes when you try to start or exit Word.
   •    Word crashes when you open the Open dialog box.
   •    Word crashes when you save a document.
   •    Word crashes when you close a document.
 
If you feel uncomfortable editing the registry yourself, Microsoft have a
fix that will do this for you.
http://support.microsoft. com/kb/940791 and press FixIt

Hi

There are four computers in a small network where everyone can access shared Word 2016 Office 365 documents like the same templates.

charles kenyon say:

***

I want to go to my client to put this template file which will be on a flash disk onto all 4 computers on their small network so when any PC is used and Word 2016 is opened on them they can insert the Autotext entries that are on this template file i created.

so how do I distribute the template file from my flash disk  as a global template so any user on an auy of the 4 computers on their network can go into Word and insert the autotexts easily?

once its distributed and one of the users then creates a new additional Autotext on their own, I want it to be then available on all personal computers automatically ... will that occur automatically once I distribute this template file onto their network as a global template?

Craig,

I'm sorry. It is not that easy.

  1. While I could set something up that should install your template on their computers fairly automatically, that would be a one-time thing and probably is not for the best. It would be far from foolproof and a fair amount of work. Instead, I would recommend...
  2. Find a network-saavy person at your client's location.
  3. Store your template in a shared location on their network.
  4. If this is a formal server-client network add something to the login scripts for each user to update the template on each user's local drive in the user's startup folder.
  5. Any changes made need to be made to the master file on the network server. Have one or two people able to make those changes.

If there are no network logins then it is a bit tougher.

You could put your template on a network folder to which all users have rea d-write-edit-delete access.

Then put a shortcut to that template in each user's local startup folder.

In this setup, any new AutoText can be saved from any computer, but it is possible that the file will be locked for saving changes if multiple people have Word open.

(The ability to save changes depends on network configuration and this is waaay over my head.)

Document what was done. Although AutoText has been a part of Word for more than 20 years, many users have little if any knowledge about it. It is seen as Word magic.

Usually I document global templates using text in the template itself. For examples of such documentation, take a look at 2003 WordArt Add-In for 2010-2016 . That is a template that both adds AutoText and puts it in a custom gallery with an icon on the Quick Action Toolbar. For more on galleries, look at Greg Maxey's Build & Deploy Custom Building Block Galleries.

In addition prepare a cheat-sheet document that users have available explaining which AutoTexts are available, how to call them, and how to add to the list. Be sure that they pay attention to naming conventions and which gallery new AutoText should be stored in. Please note that you can store AutoText in galleries other than AutoText and still have them available. The advantage of some of the other galleries is better preview of content than in the traditional AutoText gallery.

steve rindsberg say:
If the text is in a single string in the WMF, it comes in as a single string when ungrouped in PPT.

I don't believe that WMF supports the kind of character adjustment that'd be needed for good typography though, so most likely what's happening is that Illustrator, in trying to give you better typography/letter spacing, is creating each letter as an individual shape that it can position as needed.

I don't know if it's possible to change this behavior (not an Illo user).  But if you're adjusting character spacing or kerning on the type, try adding some more text w/o any adjustments and see if it comes through WMF into PPT as a single piece of text.
