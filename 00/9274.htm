suzanne s. barnhill say:

Styles are saved in documents and templates. Ordinarily, you would create a template with your custom styles and make it available to your co-workers by storing it in the folder designated as the "Workgroup templates" folder. If you want the style to be available when users create a new Blank Document, then the style must be saved in the Normal template. Because the Normal template is used to store personal settings, it should not be shared, nor should you ask users to replace their copy with yours. Instead, you can furnish them with a document from which they can copy the style into their Normal templates.

There are several ways to copy a style from a document or another template into Normal.dotm, but probably the easiest way is as follows:

  1. Open a document containing the style and create text to which the style is applied.
  2. Copy this text.
  3. In Word, press Alt+F11; the Visual Basic Editor opens.
  4. Press Ctrl+G to activate the Immediate window. In that window, type

         NormalTemplate.OpenAsDocument

    and press Enter.
  5. Normal opens, but note that it may not be on top of all windows. You can close the Visual Basic Editor. Paste the copied text into the template, then delete it. If you want the new style to be the default style for text in a new document, apply it to the remaining empty paragraph in the template. Save and close the template. The style should now be stored in Normal.dotm.

Defender Full Scan - Microsoft Community

thembankoana say:

Defender Full Scan

Lately when I run a full scan on defender it takes too long, sometimes over two days. What is causing this? It never used to take this, an hour or two at the most?
greginmich say:

Seven things to keep in mind when your Full Scans seem outrageously long:

1. Full Scans are paused when the System idles to sleep (after 30 minutes by default), and are not resumed until you wake up your PC:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/defender-full-scan-still-running-after-22-hours/cb69f242-a673-4dee-837d-317785a1b57a

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-can-i-run-windows-defender-walk-away-and-have/7b57798e-b2da-40a6-9c74-afd9ff2dff11

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-do-a-set-and-forget-full-scan-with/a1fb5e32-f227-42b6-b455-05ae41f40bf6

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/initial-scan-with-windows-defender-in-windows-10/eb9efd51-7a74-4f50-8a40-9cde2db5c25c

2. Running Quick Scans or Custom Scans will help build up Defender’s cache of trusted files and shorten up your Full Scans when the cache has been reset by an OS Update:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-10-1511-10586-windows-defender-full-scan/436b7fd5-8e87-46c4-99fa-db4a0cf09fc1

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/wd-scan-still-running-after-more-than-10-hours/4d34fbbc-c459-46b4-8378-8bc9843420a7

3. Several different types of system damage or corruption can abnormally increment both the length and CPU utilization of Windows Defender Full Scans; so it’s always a good idea to run the standard Windows 10 system repair tools whenever your scans aren’t behaving normally:

https://answers.microsoft.com/en-us/windows/forum/windows_10-update/system-file-check-sfc-scan-and-repair-system-files/bc609315-da1f-4775-812c-695b60477a93

4. Scans will be slowed (and CPU utilization will be elevated) by the presence of any other real-time security app, or by any remnants thereof. So make sure that you’ve run the cleanup utilities for any previously installed or preinstalled AV apps:

https://answers.microsoft.com/en-us/protect/forum/mse-protect_start/list-of-anti-malware-product-removal-tools/2bcb53f7-7ab4-4ef9-ab3a-6aebfa322f75

5. Uninstalling unused programs; deleting unused files; and using the Disk Cleanup tool will lighten the workload for your scans and thus shorten their run times.

6. In addition to physically removing files, you can further reduce the scan’s workload by excluding files. You can add your personal work files, media files, archived backups, disk images, etc., to the list of trusted files simply by creating either path or type exclusions for them:

https://support.microsoft.com/en-us/instantanswers/64495205-6ddb-4da1-8534-1aeaf64c0af8/add-an-exclusion-to-windows-defender

7. Turning off Archive Scanning will exclude most types of archive files in one simple step, and it will have a pronounced effect on the run times for your Full Scans:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-can-i-exclude-archive-files-for-a-scan-in/2d515e9d-a0cb-4759-8cb5-9fd7f5cc84ca
