You're using a relational database - use it relationally!

 

You need one, or a couple of lookup tables (note: lookup TABLES, not lookup FIELDS - see http://www.mvps.org/access/lookupfields.htm for a critique of the Lookup Wizard). You could have a table like

 

tblStatusCodes

  Status (text)

  ProgramType (text)

  DisplayText (text)

 

it could contain records such as

 

Status; ProgramType; DisplayText

Active; p; Primaries Approved

Active; o; Options Approved

 

and so on; you could then simply join this table to the Status and ProgramType fields to display the desired text, with no IIF coding at all.
