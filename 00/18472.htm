I won't pretend to understand how UEFI BIOS extensions have been implemented on all platforms,

I have trashed and rebuilt both computers in the past two weeks and found a few things about using Recovery USB sticks:

In Win 8.1: Settings > Recovery > Boot from other Device / UEFI Device will boot a Win 10 DVD and do an install / update

In Windows 10: Settings > Recovery ... most options will not allow booting from a Win 8.1 USB Recovery drive... it tends to insist on a Windows 10 Recovery.

What does work is to either force a recovery mode BIOS recovery.... pull the plug / force a hardware reset a few times and the software UEFI extensions give up and drop into a low level hardware BIOS from which the boot device options will show the 8.1 Recovery drive (if plugged in) and then will boot from that.

Less brutal is to use the Settings > Recovery options to find the Advanced options and then the BIOS UEFI option to get down into the low level hardware BIOS boot options... and then select the USB drive.

It varies from computer to computer.... the main thing is to get into the old fashioned hardware BIOS to enable choosing the USB Recovery drive as the boot device.

Word 2011 for Mac Indexing

When making an index using Word 2011 for Mac, is there a way to move from one term being indexed to its next occurance in the text while leaving the Mark Index Entry box open and keeping the entry the same, so that I don’t have to keep retyping the entry (which is different from the highlighted term in the text) over and over?  For example, "Pynchon" in the text needs to be entered as "Pynchon, Thomas." john mcghie say:

Jeraldine:

PLEASE read the question before replying...

Paula:  There is no "good" way to do this, because as you have discovered, the next selection replaces the previously-entered text.

If only the Main Entry is involved, you can COPY it, then PASTE into the dialog box each time.

To specify all of the entries in the dialog, you can record a little throw-away macro to fill it in for you.  You need to display the Developer tab to record macros.

You could, of course, use the "Mark All" facility to mark every occurrence of the search term in a single command; but of course, a good indexer would never publish such an index :-)  However, it may be quicker for you to use Mark All, then use Search to revisit all the tags and delete the ones you do not want.

If you Search for the main term as '^d XE "jumps' you will find only index entries whose main term begins with "jumps".  ^d is the code for "a field", XE is the code for a field of type Index, and the quote begins the main term.

Hope this helps

bob jones aka: cybertaz say:
I'm continuing to use Snow Leopard & have Office 2008 installed. Excel's File> Print menu structure does not contain a Print Preview command. It hasn't disappeared, it never was there in that version. It was eliminated from the program starting with Office 2008 (although Word continued to include a Print Preview feature in that version). The only method of previewing how the document will print offered by Excel is to use Page Layout View which is not listed in the File menu structure. It's accessible from the View menu or by clicking its icon in the lower left corner of the document window [next to the Normal View button].

Once the Print dialog window opens there are 2 'preview' options provided by Apple:
  • The Quick Preview feature displays a scaled down preview of the document which appears within the Print dialog window. In SL there is a check box below the Quick Preview area to turn it on/off. I don't use 10.8 but I believe the feature works the same in that version of OS X.
  • Use Apple's Preview program to view the document in full size [actually displayed as a PDF but without having to save as one].

I believe it may be the latter of those options that you're unable to locate. In OS X 10.6 there was a Preview button at the bottom of the Print dialog window -- right next to the PDF button. As I understand it, that button was eliminated in the OS X 10.8 Print dialog window. You now have to click the PDF button & select Open PDF in Preview. Preview a document before you print

macropod say:
Often, it is also necessary to switch to Draft view to see them. Of course, if all you want to do is to delete them, you can do that via Find/Replace, where Find = ^b

robert19 say:

1. Press Ctrl+End to go to the end of the document.

2. Go up to the first Header from the bottom of the document, and double-click the Header area to bring it to the foreground.

3. Use the ribbon to unlink that Header from the previous Header.

4. Remove all text from that final Header.

5. Double-click the document main body text to get out of the Header area.

5. Press Ctrl+Home to go back to the start of the document.

6. Are there any remaining Headers? If so, repeat steps 1 to 5 above for each specific Header.


Repeat steps 1 to 5 above for the Footer areas.


HTH.

Deleting the Section breaks, which can be done via Find/Replace, will address that issue and is also the simplest way of getting rid of all the unwanted header/footer content, whilst at the same time ensuring every Section has the same page layout. the only issue that might need addressing is the need to ensure the page layout & headers/footers from the first Section are preserved.
