Hi Jonathan,

First, what I’m trying to say is that there is an Rules Quota limit (64 KB)for the users in an Office 365 organization.
Only Office 365 tenant admins can increase the limit by using PowerShell.

To view the current Rules Quota limit for a specific user or mailbox, you can run the following command:
get-mailbox -identity <mailbox> | fl *rulesquota*

Note: where <mailbox> could be the user’s UPN (User Principal Name) or full e-mail address, like *** Email address is removed for privacy ***

Based on your scenario, it seems that you need to choose to keep the server-side rules or the client-side rules to prevent the error message poping out.

Note: If your messages are stored in an Exchange Server mailbox, a majority of your rules are server-side rules. If your messages are stored in a Personal folder, most of your rules are client-side. Once you have decided which side to use, you may need to be re-adjusted some rules so that they can function properly.

Please consider the option carefully before you make the decision.

If the error message continues to display, you may have to delete all the rules and re-import them.

Please export a copy of your rules to an .RWr file so that you can import them later.
To do this:
1.On the Outlook HOME menu, click Rules > Manage Rules & Alerts.
2.Click the Options button.
3.Click Export Rules.
4.Browse to where you want to save the .RWZ file, give it a file name that is easy to remember, and then click Save.
5.Click OK twice.

After you have exported a copy of your rules, use the following steps to delete all your rules, and then import them from the .RWr file that you exported them to in the previous steps:
1.On the Outlook HOME menu, click Rules > Manage Rules & Alerts.
2.In the Rules and Alerts dialog box, click to select a rule, and then click the Delete button.
3.Repeat step 2 until there are not any rules remaining.
4.Close and restart Outlook.
5.On the Outlook HOME menu, click Rules > Manage Rules & Alerts.
6.Click Options.
7.Click Import Rules.
8.Find the .RWr file that you exported you rules to, and then click Open.
9.Verify that the rules are configured correctly.
10.Click OK

If you encounter any issue during the process, please feel free to post back and we will further assist you.

Best Regards,
Lance

In the meantime, you could try this as a workaround. It's a little complicated, but this is a generic technique you can use any time you run into issues with a self-extracting installer that contains drivers. Basically, what you're doing is installing the drivers into the system so that Plug and Play can take care of things. The side effect is that you don't get any of the value added software installed, which can mean you have limited functionality in some areas.

1. Run the setup software from HP

2. Right click on the icon in the task bar, then right click on the application's icon in the jump list. Choose properties. This will open a familiar properties dialog which tells you where the installer is located. It's most likely somewhere in your user profile's temp directory.

3. Select file path in the location entry, and right click, then copy

4. Open a PowerShell console as an administrator

5. cd to the directory that you copied from the properties page

6. Run the command pnputil -a *.inf (see below for an example of the PowerShell commands and output)

7. At this point, you can close the setup software from the vendor.

8. Plug in the printer (or add using the Add Printer Wizard)

I you choose Insert | Object | and do not check the "link" box, the Word file is embedded. (You can click "display as icon" in that dialog box after you select the file.)

 

Then, to open it during the presentation, add an Object Action animation.

1. Click Animations tab, Custom Animations pane.

2. Select the Word doc icon on the slide.

3. In the Custom Animation Pane select Add Effect, Object Animation, Open.

4. Double-click the animation in the animation pane and select the Timing tab.

5. Click Triggers, then Start Effect on Click Of  ... and choose the object. (In my case it's Object 4.)

 

This makes PPT open the Word doc when you click the icon during the presentation. Because it's embedded, you won't have to send it separately and worry about the links.
