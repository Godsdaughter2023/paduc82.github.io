In VBA, you cannot set the intial value of a variable on the same line of code that you declare the variable's type on (you have to set the variable's value on a separate code ling). Just so you know, VB automatically sets the initial value of numeric variables to 0 and effectively sets the initial value of String variables to an empty string ("")... well, technically, String variables are defaulted to a "null string" (needed for certain types of external, to VB, function calls)  which are indistinguishable from an empty string within the VB code you will be working with.

 

When you Dim or ReDim an array using a single number between the parentheses, that number is used for the upper bound of the array... the lower bound for the array is set automatically depending on the Option Base in use (the default Option Base is 0 if not explicitly specified). When you used the Option Base 1 statement, you automatically set the lower bound to 1, but you set the upper bound 0... since 0 (the upper bound) is less than 1 (the lower bound), VB considers it an error. By the way, you can specify both the lower and upper bound when you declare an array's limits... just connect them with the keyword "To", like this...

 

ReDim MyArray(0 To 1)

 

Doing it this way makes your intent much clearer, but more important, guarantees the array will be declared the same way no matter what Option Base setting is used.
