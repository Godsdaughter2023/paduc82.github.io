jolisdegats say: I finally found a tip but I don't think it could be useful to everybody.

To resume, I have a workbook named "Mydocument.xlsm". I work on it, save the file and then run my applescript. My Applescript create a new folder, duplicate "Mydocument.xlsm" in it, open this copy and is supposed to run the macro "Macroexporttxt()" which is in a module code.

To fix my problem, I wrote my macro again. 

Firstly, I put the macro in the workbook code of my "Mydocument.xlsm" file and rename it "workbook_open()". This operation allows my macro to run automatically when I open the workbook.

So, there is no more macro to launch manually as it runs itself and it fixes the problem of the "run VB" action that doesn't seems to work anymore.


Then I put a "if" function at the beginning of the macro "workbook_open()"that looks like that : 

    If ActiveWorkbook.Name = "mydocumentcopie.xlsm" Then

(... the rest of the macro)

Else
        Exit Sub
    End If


Then I modified my Applescript by adding a line to modify the name of the duplicated file from "my document.xlsm" to "mydocumentcopie.xlsm" before launching Excel and open the document "mydocumentcopie.xlsm" (and no more "mydocument.xlsm" as it did before).

So when I open "mydocument.xlsm" to modify my data nothing happens but when "mydocumentcopie.xlsm" opens, the macro runs. It was simple like that.

Now, it works fine for me. Maybe it could help you to find your own tip !

Hi ThunderLight420

Open 2 Windows File Explorer Windows (press Windwos Key + E twice)

Re-arrange those windows to split your screen

In one Window, create the folder you want all mp3 files to be in

In the other File Explorer window, click 'This PC' in the left Pane, then double click your hard drive

Wait for that search to complete (it will take time)

Then click into the resulting files and press Ctrl + A to select all mp3 files and drag those into the folder in the other window

That's It!

jeeped say:
The 'trust' must come from each workstation/login. You cannot 'push' a trust. To take this to an extreme, it would be the equivalent of a web site forcing 'trusted' malware on a user without the user's consent. If you have a number of these, consider setting up a self-certification and a 'Trusted Publisher' with SelfCert.exe.

OFF2000: Using SelfCert to Create a Digital Certificate for VBA Projects
Sign your own macros for stronger security
Digitally sign a macro project

Windows Update stuck downloading updates

If your Windows Update is showing available updates and your computer starts downloading them, it may happen that at some point it just gets stuck and stops downloading. It will show a constant figure but there will be no progress. The figure in your case could be 0%, 23%, 33%, or any other, but every time you try to download the updates, you find that they are stuck on that particular figure, even if you leave it for an hour or two.

If you find that your Windows Update is stuck downloading updates at 0 % or any other figure in Windows 10, Windows 8.1, Windows 8, Windows 7 and Windows Vista as well, then please refer to the link below.

http://windows.microsoft.com/en-in/windows/troubleshoot-problems-installing-updates#1TC=windows-8
