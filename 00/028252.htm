joeu2004 say: Al-Amin wrote:
Can you give any calculation, where i get trend & growth formula in excel. i want to know trend & growth formula.in details.  where i use trend & growth formula. how to calculate it.

The TREND function is based on a linear regression model of the form y = m*x+b.  Refer to details explained in the LINEST help page. [1]

The GROWTH function is based on a linear regression model of the form ln(y) = x*ln(m)+ln(b).  Refer to details explained in the LOGEST help page. [1]

For more information about the "simple linear regression" method, refer to http://en.wikipedia.org/wiki/Simple_linear_regression.

It is important to note that the TREND and GROWTH functions are based on linear regression curves, which are derived from the actual data, but which usually do not fit the actual data exactly unless you have only two data points.

http://answers.microsoft.com/message/4244a772-b7b3-48dc-9a01-adf730644c04?threadId=59b36c50-28f1-4317-b472-a2b4215916bb for an example of how to compute the GROWTH coefficients without using LOGEST.  Similar formulas can be derived for LINEST.

The original posting needed the methods in order to implement them an SQL SRS product.  I mention this in case you have a similar need.


-----
[1] I am not saying that the TREND and GROWTH functions actually use the LINEST and LOGEST functions or internal implemenations.  I don't know whether or not they do.  But the same concepts apply.

1. Create a new blank document in .docx format

3. Copy it.

4. Paste in the new document.

5. Save under a new file name and close all, then re-open.


This technique for de-corrupting is known as "Doing a 'Maggie'", after

Margaret Secara from the TECHWR-L mailing list, who first publicised the

technique.

scottgem say:
The way I read it your need for a many to many relationship is temporary. You have two tables with an Ingredient field, but in those two tables, the field is not standardized. So you need to create a translation table. This table would look like this:

tblIngredientTranslation
IngredientID
IngredientText

Now you have to populate that table with a list of the unique Ingredient text values from both tables. To do this, Create a union query (name qryIngredients):

SELECT DISTINCT Ingredients FROM Products

UNION SELECT DISTINCT Ingredients FROM DMFs;

The next step is to run an Append query:

INSERT INTO tblIngredientTranslation (IngredientText) 
SELECT DISTINCT Ingredients FROM qryIngredients;

You also need a new Ingredients lookup table:
tluIngredient
IngredientID (PK Autonumber)
Ingredient

Next, you need to MANUALLY populate the IngredientID field in tblIngredientTranslation with the appropriate IngredientID. Now you have your translation table. 

The final steps will be to add the IngredientID field to the two tables and run Update queries matching on the current Ingredients field to the translation table and updating the IngredientsID field from the translation table. 

Once this has been completed and verified, you can delete the Ingredietns fields int he two tables, as well as the union query and translation table.
