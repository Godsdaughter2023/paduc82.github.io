Function ColLetter(ColNumber As Long) As String
    On Error Resume Next
    ColLetter = Application.Substitute(Application.ConvertFormula("R1C" & ColNumber, xlR1C1, xlA1, 4), "1", "")

Function Col2Num(ByVal ColName As String) As Long
  'Returns the column number for this column identifier => "ABC" to 731
  Dim j As Integer
  ColName = UCase(ColName)
  Col2Num = (Asc(Mid$(ColName, Len(ColName), 1)) - 64)
  For j = 1 To Len(ColName) - 1
    Col2Num = Col2Num + (Asc(Mid$(ColName, j, 1)) - 64) * 26 ^ (Len(ColName) - j)
  Next
End Function

Function Num2Col(ByVal ColNumber As Long) As String
  'Returns the column identifier for this column number => 731 to "ABC"
  Dim i As Integer
  Do While ColNumber > 0
    i = (ColNumber - 1) Mod 26
    Num2Col = Chr(i + 65) & Num2Col
    ColNumber = (ColNumber - 1) \ 26
  Loop
End Function

davidf2 say:

It would be helpful if we could see your work. Can you post a link to it?

There are some things that Publisher will choke on when trying to convert to html and the result is the text being converted to an image. Without seeing it I can only guess.

If the text box is grouped with other design elements on the page, Publisher will combine them into one image. Ungroup the text box.

Use standard web fonts. Non web fonts and word art can be converted to images.

Some 'fancy borders' will result in the text being converted. Use none or simple single lines.

Some text box fill colors including gradients will choke Publisher. Once again don't use them or try something different.

If the formatting of the text box is the problem one workaround you can try is to strip out all the formatting from the text box. Then create a second text box exactly the same size as the original...you can create it on top of the original, apply the formatting to that second text box and then under the Arrange tab > send to the back. By layering the two, you will get the same effect but the orignial text will remain text when you convert to html.
