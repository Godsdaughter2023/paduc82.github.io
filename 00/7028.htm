julie sheets say:

You can use the exiting Cost Variance and Finish Variance fields in formulas.

For the Cost Variance.  Add a spare Text field to your table and customize it with the following formula:

IIf([Cost Variance]<=0,"Green",IIf([Cost Variance]<[Cost]*0.1,"Yellow","Red"))

Ensure that you have set "Use formula" in the "Calculation for task and group summary rows".

Once you have the correct values as a result of the formula, select "Graphical Indicators" and set them appropriately.

For the schedule try:

IIf([Finish Variance]/[Minutes Per Day]<=0,"Green",IIf([Finish Variance]/[Minutes Per Day]<=5,"Yellow","Red"))

Neither formula tests for a saved baseline, so if a baseline is not saved, both will show "green" as the baseline variance and baseline costs will be zero.

scottgem say:

First Company Name is not a good idea for a key field. Names can change and text fields don't perform that well in Joins. You should have an Autonumber field for your CompanyID. That should be the Foreign Key in your Contacts table.

If you use a main/sub form combination, linked on companyID, then the FK will be automatically populated when you enter a record in the Contacts subform.

Mote, that there should be no other fields in the contacts table from the company table. For a more detailed discussion of that please see my blog on Displaying Data from Related Tables.

hello77 say: helpline,

I found a solution (thanks to a little help from a friend over at mobile planet); but not a real solution because you have to go into the registry editor to do it; but it will make it so your screen does not turn off during a phone call. Here what was suggested to me:: 


\HKLM\Drivers\BuiltIn\RIL\EnableFastDormantDisplayDuringCall

 

change "1" to "0"

Here is what i did though in a simplified format (i dont fool around with registry entries to often):

Open a registry editor (i downloaded resco explorerer which includes a registry editor)

In editor go to

hkey local machine

then to drivers

then to builtin

then to ril

then to enablefastdormantduringcall

change 1 to 0 and your done

after you do this you must then do a soft reset

It is a little confusing at first but it can be done (though must say you need to be real careful not to accidently change a register entry as it could be detrimental to your phones health).
