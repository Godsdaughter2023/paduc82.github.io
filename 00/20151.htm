    Best practices are to isolate devices like these on their own VLAN and only run Citrix or terminal server type applications on them, avoiding the issue of direct access to the device by malware entirely.

    ossiemac say:

    Finding dates with VBA is often a problem. To the best of my knowledge the method used in the following code should always find the date irrespective of the actual format of the date in the worksheet. However, if you have a problem then please let me know and include an example of the date format you are using plus the default Short Date format for your region.

    Sub FindDates()
        Dim strDate As String
        Dim rngToFind As Range
       
        strDate = Format(Date + 9, "Short Date")
       
        With ActiveSheet.Range("K:K")
            Set rngToFind = .Find(What:=CDate(strDate), _
                            LookIn:=xlFormulas, _
                            LookAt:=xlWhole, _
                            SearchOrder:=xlByRows, _
                            SearchDirection:=xlNext, _
                            MatchCase:=False)
                           
            If Not rngToFind Is Nothing Then
                With ActiveSheet
                    .PageSetup.PrintArea = Range(.Cells(5, "A"), rngToFind).Address
                End With
            Else
                MsgBox "Date " & strDate & " not found." _
                    & vbCrLf & "Print Area not set."
            End If
        End With
               
    End Sub

    john korchok say:

    To create a new template from the old one, you would open the old template, choose View>Slide Master, delete all graphics on the slide master (the larger slide at the top of slide master view), but leave the text placeholder there, so the bullet styles are maintained. Then create new graphics for the master slide to match your new corporate look.

    As an alternative, you can transfer settings manually. Follow these steps:

    1. In the old template, choose View>Slide Master and choose the Slide Master.
    2. Click on the first text level.
    3. If Home>Paragraph>Bullets and Numbering is not yellow, a picture was used for the bullet and there isn't an easy way to re-create it, just proceed to the next text level.
    4. Assuming Bullets and Numbering is yellow, click on it, then click on Bullets and Numbering... at the bottom of the dropdown.
    5. Note the Size and Color settings used.
    6. Click on the Customize... button.
    7. Note the Font and Character used. In large unicode fonts, it can be helpful to note the Character code at the bottom of the dialog. OK out.
    8. Now choose Home>Paragraph and open the Paragraph dialog. Note the Indentation and Spacing settings.
    9. Switch to your new template, open View>Slide Master and select the first text level.
    10. Click on Home>Paragraph>Bullets and Numbering>Bullets and Numbering....
    11. Apply the size and color you noted in the old deck.
    12. Click on Customize...
    13. Set the same font and character that you found in the old presentation. OK out.
    14. Open Home>Paragraph and open the Paragraph dialog. Set the Indentation and Spacing settings to match the old presentation. OK out.

    The first text level of the new file should now look just like the first text level in the old one. Repeat the steps for each text level. Slide Layouts can have different bullets than the Master, so you might have to transfer some text levels for them as well.

    john korchok say:
    That link applies to Word. But I believe you should be aiming at formatting the Slide Master text levels rather changing the styles displayed in the Bullets and Numbering dialog. Slide masters are a more reliable way to transfer and use bullet styles company-wide.
