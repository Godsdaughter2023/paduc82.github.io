What is the reason for wanting the text to be red? Must it remain red forever? If so, the default option for pasting within and between documents is Keep Source Formatting. Even if you have selected a different option, if you have left the Paste Options button enabled, you can select Keep Source Formatting after you paste. This assumes that the text to be pasted is already formatted as red. If not, then the suggestions the others have given will be helpful.

If, on the other hand, you just want inserted text (any inserted text) to be displayed as red so that you can see what you have inserted, then the option you're looking for is Track Changes. With Track Changes enabled, you can opt to have insertions displayed in red (usually they're also underlined). For the basic principles of Track Changes, see http://www.shaunakelly.com/word/trackchanges/HowTrackChangesWorks.html; for specific information on Track Changes in Word 2013, see http://nssit.com/how-to-use-the-new-track-changes-feature-in-word-2013/ and https://support.office.com/en-US/Article/Change-options-for-Track-Changes-fee8a93d-9172-42dd-b0f3-e3b39b6c5840?ui=en-US&rs=en-US&ad=US (note that you have to click Advanced Options... in the Track Changes Options dialog to get to the dialog where you can specify the formatting for insertions, deletions, etc.

doug robbins - mvp office apps & services (word) say:

If you copy the text to be inserted to the clipboard and then use a macro containing the following code, it will paste the text at the location of the selection and format it as red:

Dim rng As Range
Set rng = Selection.Range
rng.Paste
rng.Font.ColorIndex = wdRed

or, if the text does not already exist, if you put the selection where you want to insert the text and then run a macro containing the following code, it will ask for the text and then insert it at the selection and format it as red:

Dim rng As Range
Set rng = Selection.Range
rng.Text = InputBox("Enter the text to be inserted.")
rng.Font.ColorIndex = wdRed

How do you add a static distro group to a dynamic distro group? - Microsoft Community

a. user say:

How do you add a static distro group to a dynamic distro group?

How do you add a static distribution group to a dynamic distribution group via power-shell for a hybrid (AD intergration) office 365 setup?

I'm looking for the command to do this in powershell - anyone know?

vasil l. michev say:

If the 'static' DG doesn't match the criteria you used for creating the Dynamic one, you basically have to edit the recipient filter and add either the DG explicitly or expand the filter to include group objects as well. Here's an example:

Set-DynamicDistributionGroup TestDG -RecipientFilter {((SKUAssigned -eq 'True') -and (Company -eq 'blabla')) -or (Name -eq "DG")}

where TestDG is the name of the dynamic group, and "DG" is the name of the normal one I have explicitly added to the filter (note the -or condition).

jay sun msft say:
Hi Nislander14,

As the folders you mentioned are default folders, you can only hide or show these folders by settings in Outlook > Preferences > General, just like what you did. And it’s impractical for users to delete them. If you quite want to be allowed to delete the unwanted folders or other way to hide them, you can go to this feedback page to vote it and your comments will be valuable. 

Besides, if you want the incoming mails move to the “On My Computer” inbox, you can create a rule by following steps:

1. Click Rules on the Tools menu.
2. Choose account type under Client Rules in the left pane of Rules (As all your accounts are IMAP account, please choose IMAP) and click Add + at the bottom of the right area.
3. Type the rule name in the Rule name box.
4. Click the arrows under When a new message arrives and choose All Messages.
5. Click the arrows under Do the following and choose Move Message, choose the destination Inbox (On My Computer).
6. Click OK.

As the previous Operating System (Windows 7 or earlier) were not using a Microsoft Account, hence when accessing Microsoft Services, such as the Microsoft Community, MSN etc. you were supposed to login into them separately.
