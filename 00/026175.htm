jjjsheridan wrote:
How is the frequency below working?

Your data seems to come from the Excel offline help page, but you made some changes that are inconsistent with the description of the results.  That might be confusing you.

Suppose the data are in B2:B10 (79, 85, 78, 85, 50, 81, 95, 88, 97).  And suppose your "bins" (really, the bin limits) are in C2:C4 (70, 80, 90).

To use FREQUENCY, select __4__ unused cells -- say, D2:D5 -- and enter the following array formula:

=FREQUENCY(B2:B10, C2:C4)

Note:  Enter an array formula by pressing ctrl+shift+Enter instead of simply Enter.  If you make a mistake, select the entire range, press F2, edit the formula as needed, then press ctrl+shift+Enter.  If you want to change the number of cells for the result of FREQUENCY, I think you need to select the original range, delete the formula, then select the new range and enter the array formula.

With your data, coincidentally FREQUENCY will return 1, 2, 4 and 2 into D2:D5, as in the help page.  But the interpretation is slightly different from the help page description, which you copied, because your bin limits are different.

D2 reflects the number of values in B2:B10 less than or equal to the bin limit in C2 (<=70).  D3 reflects the number of values in B2:B10 greater than C2 and less than or equal to C3 (>70 and <=80).  D4 reflects the number of values in B2:B10 greater than C3 and less than or equal to C4 (>80 and <=90).  D5 reflects the number of values in B2:B10 greater than C4 (>90).

Note that in B2:B10, there is 1 value <= 70 (50), 2 values >70 and <= 80 (78, 79), 4 values > 70 and <= 90 (81, 85, 85, 88), and 2 values >90 (95, 97).

The important thing to note is:  the values in the "bins" array contains the __upper__ limit for each bin; i.e. for each cell in the FREQUENCY result.  Thus, the count in each FREQUENCY result is the count of the data less than or equal to the corresponding bin limit (and greater than the previous bin limit).

A common use of FREQUENCY is creating data for a histogram (chart).  For that purpose, rather than have a mysterious 4th FREQUENCY result that is simply greater than the previous bin limit (using your example), I prefer to have 4 elements in the "bins" array, where the last element (C5) is the formula:  =MAX(B2:B10).  Then the FREQUENCY array formula would be (still selecting D2:D5):  =FREQUENCY(B2:B10,C2:C5).

h0gwarts say:

Is it possible to create and share templates in Outlook 2010 that contain building block entries?

Building Blocks exist in both Microsoft Word 2010 and Microsoft Outlook 2010.  The building blocks for each are kept in their normal templates: normal.dotm and normalemail.dotm, respectively.  C:\Users\##\AppData\Roaming\Microsoft\Templates

In addition, in Word 2010, building block galleries can be used from templates, such as Built-in Building Blocks.dotx.  These templates are kept in: C:\Users\##\AppData\Roaming\Microsoft\Document Building Blocks\1033\14

The advantage to these templates is that they can be edited independently of the Normal.dotm, and will survive if the Normal.dotm must be replaced for any reason.  They can also be shared between users without affecting their individual Normal.dotm.

Question: Is it possible to create and share a template containing building blocks usable in Outlook 2010, like it is in Word 2010, so that these entries are separate from the Normalemail.dotm, and can be distributed to several users?  If so, how would that template be created, and which folder in Appdata/Roaming would it be kept to be accessible to Outlook 2010?
