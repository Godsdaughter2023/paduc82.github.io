It was happened with me as well, and I found this solution:

1. From Format Object (right click over the excel table).
2. Then Select Layout tab then Advanced
3. Select Position tab then and from Horizontal select Absolute position enter the Vertical and keep to the right of same Column and below same Paragraph then from Options mark Move object with text and Allow overlap (it's up to you) then press OK.

bob jones aka: cybertaz say:

File encoding is determined by the program creating/saving the file. Excel cannot arbitrarily open a file using a different decoding method from that in which it was encoded by the creator program. Besides, doing so often can cause the file to be misread. The only time the user has any option is if Excel cannot determine the encoding of the file being opened, usually because the file is damaged or the encoding method is not supported.

IMHO, the preferable approach would be to open the file, then use Save As to create a version using the preferred encoding method. AFAIK, anything else - if possible - would have to be done programmatically.

Option Explicit

'
    Dim wBook       As Workbook
    Dim wSheet      As Worksheet
    Dim oList       As ListObject
    Dim olRow       As ListRow
    Dim NbRows      As Long
    Dim destRow     As Long
   
    ' For demo:
    Dim add_nom         As String:  add_nom = "my name"
    Dim add_prenom      As String:  add_prenom = "my fist name"
   
    On Error GoTo ERR_HANDLER
   
    Set wBook = ThisWorkbook
    Set wSheet = wBook.Worksheets("Accueil")
    Set oList = wSheet.ListObjects("Table1")
   
    ' NOTE: Should not happen but 1st row of DataBodyRange
    ' can be deleted - still displayed/visibled in the UI though...
    ' In such case DataBodyRange.Rows.Count fails and raises an Err = 91
    On Error Resume Next
        NbRows = oList.DataBodyRange.Rows.count
        If Err.Number = 91 Then NbRows = 0
    On Error GoTo 0

        Select Case NbRows
            Case Is >= 1
                ' If last DataBodyrange row has no value use that row
                If Application.WorksheetFunction.CountA(oList.ListRows(NbRows).Range) = 0 Then
                    destRow = NbRows

                Else
                    Set olRow = oList.ListRows.Add(AlwaysInsert:=True)
                    destRow = olRow.Index
                End If
   
            Case Else
                Set olRow = oList.ListRows.Add(AlwaysInsert:=True)
                destRow = olRow.Index
        End Select

        With oList
            .ListColumns("Nom").DataBodyRange(destRow) = add_nom
            .ListColumns("Pr?nom").DataBodyRange(destRow) = add_prenom
            ' ...
        End With
    wSheet.Protect Password:=motdepasse

ERR_HANDLER_EXIT:
'
    Set oList = Nothing
    Exit Sub

ERR_HANDLER:

    MsgBox "Something went wrong" & vbLf & _
        Err.Number & ": " & Err.Description
    Resume ERR_HANDLER_EXIT

End Sub
