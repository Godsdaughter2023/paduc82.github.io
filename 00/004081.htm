bob jones aka: cybertaz say:

I can't explain the technicality of why it has to be that way, but Word tables must always be followed by a paragraph... That paragraph may or may not have content. (If it has no content it appears only as a ¶ when nonprinting characters are displayed.) When the Table is near the bottom margin there may not be sufficient space on the same page for that ¶, which causes it to flow to the next page - adding a new page if necessary at the end of a document.

Either of many approaches can be used to resolve the issue, generally by tightening up the content that precedes the Table. Some of the most common are:

  • Eliminate any "empty" paragraphs that have been used for vertical preceding the Table (pounding the return key is not the preferred way to control spacing or text flow :-)), or
  • Reduce the Line Spacing and/or Paragraph Spacing of the content preceding the Table, or
  • Reduce the Row Height used in the Table, or
  • Slightly decrease the Top or Bottom Margin. This is usable in a short (~5 pages or less) document, but may adversely affect the overall flow of longer documents, or
  • Select the ¶ that follows the Table, then go go Format> Paragraph & set the;
    1. Line Spacing: Exactly At: .7 pt
    2. Spacing Before: 0
    3. Spacing After: 0, or
  • Select the ¶ then change the Font Size to 1 pt

Other techniques can be used as well - Most advanced users have their own preferred method & the optimal choice often depends on the specific situation.

Access lookup from table to return true or false - Microsoft Community

nancybaumunk say:

Access lookup from table to return true or false

I am an Access novice and have been doing online training and I think I don't know the proper terms to use.  I have a table showing all students enrolled in classes.  I have additional tables showing the students enrolled in specific classes.  On the larger table, I want to add a field for each of the individual classes and I want to look up to find if a particular student is enrolled in that specific class.  Each student has an ID number.  For each record (student ID), I want to tell Access to look at the smaller table of those enrolled in a specific class and see if it can find that same ID number is listed.  If so, I get a TRUE (or Yes) result, if not, a FALSE (or No) result.  I can't figure out how to structure this.  Can anyone help?  Thanks in advance. 

john w. vinson say:

Stop. Turn back! You are going the wrong way!

Your tables are incorrectly structured. A separate Table for each class - OR a separate field for each class - is not correct for a relational database. A class enrollment system is a classic "Many to Many" relationship; the correct structure has three tables:

tblStudents

  StudentID (primary key)

  LastName

  FirstName

  <other relevant biographical data>

tblClasses

  ClassID (primary key)

  ClassName

  <other info about the class, e.g. schedule, instructor, prerequisites>

tblEnrollment

  EnrollmentID (autonumber primary key)

  ClassID (link to tblClasses, which class is this student in)

  StudentID (link to tblStudents, who is enrolled)

  <other info about THIS student in THIS class, e.g. enrollment date, comments, grade>

You're using a relational database. Use it relationally! Read up on the concept of Normalization.

electrical_king say:

Encrypted word files?

I'm working in a project that needs to encrypt MS Word documents to be distributed only for those who buy the documents, now I need distribute the MS Word as a portable to view and edit the encrypted files, is that is legally accepted? Also is there is any ready made solutions for my issue?
Thanks in advance.
graham mayor say:
There is no foolproof way of doing this, but you can make things a little more difficult for potential misusers by supplying the documents as password protected PDF format documents. PDF is an industry standard and users can get the Adobe reader viewer as a free download.

As with all such measures it won't stop plagiarism by those who have the password.

If you make each copy of the document ever so slightly different from the others, if someone does release the document widely you can trace back who had the version by examining the difference you included and take action as appropriate. You will of course need a log of the differences, and you need to make sure that the difference is not obvious to the reader, so using serial numbers isn't going to be worth a stuff.

jay freedman say:
If you have an method (independent of Word, such as PGP) for encrypting and decrypting document files, but your recipients don't have Word, they can use the Word Viewer program (http://support.microsoft.com/kb/891090). The download page for the Viewer says that you can distribute it.

As Graham says, this sort of solution does absolutely nothing to prevent redistribution of the documents.
