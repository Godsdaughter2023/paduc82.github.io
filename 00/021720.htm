scott mcdaniel say:

You're calling OpenReport from an object named "objAccess", but you're calling OutputTo from the default instance of Access. If you're automating this (i.e. you've opened another session of Access for your "objAccess"  variable), then you should preface the Outputto with that same object:

objAccess.Docmd.OutputTo

If you are NOT automating Access, then don't use that objAccess variable in the OpenReport method:

DoCmd.OpenReport sReportname etc etc

Also, you should NOT include both a Filter and a Where argument, and in later version of Access including an invalid or incorrect filter argument will cause that to fail. A properly formed OpenReport would look something like this:

DoCmd.OpenReport sReportName, acViewPreview, , strWhere

In later versions, the Filter argument only accepts a "string expression that is a valid name of a query", according to the MSDN article:

https://msdn.microsoft.com/en-us/library/bb238032(v=office.12).aspx

I believe the freezing issue is caused by the large volume of emails in user's mailbox so that Outlook will take a much longer time to search the emails.
