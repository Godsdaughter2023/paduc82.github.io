myall_blues say: Conditional formatting can take some getting your head around. I find the easiest way is to select the entire range, and then write the CF formula as you want it applied to the FIRST cell in the range. So:
Select B13:M513 (or wherever)
Conditional formatting->New Rule->Use a formula...
In the formula box enter either:
=$E13="ICT P" if you want the formatting of B13:M13 to be dependent on the value in E13
=B13="ICT P" if you want formatting of B13 to be dependent on B13, C13 to be dependent on C13 and so on.
Set the font and shading to green as you want it.
Click OK.

dirk goldgar say:

It is possible via the Windows API to rename the buttons on the MsgBox dialog, so that's a possible workaround if you don't need more than three buttons.

If you need more than that, can create your own dialog form with some preset maximum number of buttons, all of which would be initially hidden and stacked on top of one another.  You would then write your own function that would be passed a set of options to be mapped to the buttons, and which would open the form hidden, set the button's captions and visibility, and resize the form and reposition the buttons that are to be displayed.  Clicking each button would cause the form to close and return to the function a number corresponding to the option chosen.  So once you have this form and function set up, you can call it from anywhere you need it in your application. 

Note that there's more to it than this, but that's one approach to the problem.
