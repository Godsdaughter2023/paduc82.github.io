a. user say:

As far as I can tell, this problem occurs no matter which referencing style you use - even APA, for example. It would need a fix from Microsoft, but the fix should be pretty easy.

Looking inside the .docx file leads to the source of the problem - the XML that defines the referencing style specifies the .xsl file that contains the style information. However, the file is specified relative, with a path separator at the beginning. Windows Word uses the standard Windows path separator, so for APA the file name might be given as "\APA.xsl". Mac Word uses the standard Unix/Posix separator, so would store "/APA.xsl". However, WIndows Word also recognises "/APA.xsl" whereas Mac Word does not recognise "\APA.xsl"

(This may be because Windows/DOS has always recognised "/" as a path separator as well as "\", but I do not know for sure).

Personally, I would have thought pathnames inserted by Word (i.e. not directly by the user) and stored in XML should always have the Unix/Posix style separator and that the Windows version probably ought to be changed to do that, but because older Windows-produced files will have "\" I think Microsoft is going to have to fix Mac Word to deal with this one.

Word 2011 may have a similar problem, but also uses different XSL file names from the 2016 Mac/Windows versions so it is more difficult to tell.

As far as I can tell you can work around this by re-selecting the correct referencing style and saving the document. Actually looking inside the .docx and changing the "\" to "/" works but is so difficult I doubt if it is worth it.
