randy k. say: For those who might be interested -

Regarding my original problem, Arial font not present in older Word documents created in Version X when opening them in Word 2011:

Following installation of Office for Mac 2011, 4 fonts in my /Users/Me/Library/Fonts folder were duplicates:

Arial-->Regular; Italic; Bold; Bold Italic
Times New Roman-->Regular; Italic; Bold; Bold Italic
Verdana-->Regular; Italic; Bold; Bold Italic
Wingdings--Regular

These old fonts were installed in my /Users/Library/Fonts folder by Office for Mac version X many years ago.  (Brought over in Lion by Migration of my User Account.)  Duplicate, newer versions were installed by Office for Mac 2011 in /Library/Fonts/Microsoft.

The resulting font conflict caused my old documents created with Arial to have no text visible when opened with Word 2011.  Manually dragging the old fonts into the trash fixed the problem and I was able to open the old documents in Word 2011 (compatibility mode) and all text appeared.  However I did go farther and empty the trash.

I then restarted in Safe Mode, by holding down the Shift key after the start up tone until the Apple symbol and spinning gear appeared.  I then logged into my existing account.  Once the computer fully started up to my desktop, I then restarted normally.  This cleared Font Book's database and my user account cache files and restarting normally rebuilt them.

I then used FontNuke http://www.jamapi.com/pr/fn/ to clear my font cache files (including /Library/Preferences/Microsoft/Office 2011/Office Font Cache).  I first clicked "Nuke Font Caches" at the upper left corner, then I had to scan for cache files before proceeding.  Once the list of cache files populated, I clicked Nuke Font Caches again.  FontNuke finished by automatically restarting my computer once more.

That completed the process.  The list in Font Book is updated and accurate and the problem with Arial text appearing in old documents opened with Word 2011 seems permanently fixed.

P.S.:  As far as other font conflicts that I found in Font Book:

A.  Duplicate fonts that are turned off (disabled) in Font Book are ok, and do not cause a conflict.

B.  Therefore I left the fonts that were duplicates in /Library/Fonts/Microsoft untouched as they were already turned off in Font Book (apparently by the Office for Mac 2011 installer).

C.  Using Font Book to try to remove fonts was not always fully effective.  Manually dragging them to the trash was best.

D.  Fonts placed in /Library/Fonts Disabled by the Office for Mac 2011 installer do NOT create any conflicts, even though they are duplicates.

E.  I did not remove ANY fonts from /Library/Fonts or /Library/Fonts/Microsoft.

I hope this information is helpful to other users of Office for Mac 2011, as installation WILL create font conflicts with older Office versions that placed fonts in User Libraries.  These will  have to be corrected.

Good luck!  - Randy

james atkinson (james2016) say:

ossiemac say:

Try the code below.

The filtered data does not need to be copied to another worksheet. Just assign the visible data of the Employee column (empnum column D) to a range variable and then turn off AutoFilter and paste the visible range to cell G4 (column header cell) on the same worksheet.

The duplicates are then removed from the Employee list with the code.

The code then inserts the formula in range I5 and copy down to row at bottom of the unique Employee list.

Your biggest problem was with the formula to perform the counts.

Because the formula is being copied down, the ranges being counted must be in absolute format (with the dollar signs) so that the ranges do not alter as the formula is copied down.

If you use relative referencing instead of absolute (with the $ signs), if you are counting a range C2:C3399 then when the formula is copied to the next row the range becomes C3:C3340 then the next row the formula will be C4:C3341 etc.

Also the same thing will occur with the Start and Finish dates. First formula will reference H2 and I2 and when copied down the formula will reference H3 and I3 etc.

I have assumed that where you added 1 to the Finish date for the formulas is to work with combined dates and times. It is correct to use start date with >=. However, for the end date where 1 is added to the end date, use only < than and not <= because you don't want to include anything on the additional day; only to the end of the day prior to it.

The actual formulas that are inserted are as follows. Note that the only relative references are cell G5, G6 and G7 that refer to the employee in column G. The formula concatenation builder in the VBA code will alter the last row number (3339) to suit the source data by using the variable lngLastInput.

Cell I5    =COUNTIFS($C$2:$C$3339,G5,$D$2:$D$3339,">="&$H$2,$D$2:$D$3339,"<"&$I$2+1)
Cell I6    =COUNTIFS($C$2:$C$3339,G6,$D$2:$D$3339,">="&$H$2,$D$2:$D$3339,"<"&$I$2+1)
Cell I7    =COUNTIFS($C$2:$C$3339,G7,$D$2:$D$3339,">="&$H$2,$D$2:$D$3339,"<"&$I$2+1)

If you want to see the formulas then comment out the code to copy and PasteSpecial Values between the asterisk lines and the formulas will remain for you to select the cells and view the formulas.

Install the following code in a workbook similar to the one you uploaded.

Sub Macro1()
    Dim wb As Workbook
    Dim ws As Worksheet
    Dim strPrompt As String
    Dim strTitle As String
    Dim strStart As Variant
    Dim strFinish As Variant
    Dim dteStart As Date
    Dim dteFinish As Date
    Dim lngStart As Long
    Dim lngFinish As Long
    Dim rngVisible As Range
    Dim lngLastInput As Long
    Dim lngLastOutput As Long
   
    Set wb = ActiveWorkbook
    Set ws = wb.ActiveSheet
        
    ws.Columns("G:I").Clear     'Optional to prepare for next report
   
    Do
        strPrompt = "Enter the Start date."
        strTitle = "Date entry"
       
        Do
            strStart = Application.InputBox(Prompt:=strPrompt, Title:=strTitle, Type:=2)
            If strStart = False Then
                MsgBox "User cancelled. Processing terminated."
                Exit Sub
            End If
           
            If IsDate(strStart) = True Then
                dteStart = CDate(strStart)
                Exit Do
            Else
                strPrompt = "Invalid Date. Re-Enter Start date as m/d/yy format."
            End If
           
        Loop
            
        strPrompt = "Enter the End date."
       
        Do
            strFinish = Application.InputBox(Prompt:=strPrompt, Title:=strTitle, Type:=2)
            If strStart = False Then
                MsgBox "User cancelled. Processing terminated."
                Exit Sub
            End If
           
            If IsDate(strFinish) = True Then
                dteFinish = CDate(strFinish)
                Exit Do
            Else
                strPrompt = "Invalid Date. Re-Enter End date as m/d/yy format."
            End If
           
        Loop
       
        If dteFinish >= dteStart Then
            Exit Do
        Else
            MsgBox "Start date must be prior to End Date. Please re enter dates."
        End If
       
    Loop While dteFinish < dteStart
        
    'Following is generic.Works from bottom of worksheet irrespective of number of rows in worksheet
    With ws
       lngLastInput = .Cells(.Rows.Count, "D").End(xlUp).Row
      
       .AutoFilterMode = False     'Turns off Autofilter if previously set
      
       'Convert dates to DateSerial for use in AutoFilter dates for positive results in all date formats
       lngStart = DateSerial(Year(dteStart), Month(dteStart), Day(dteStart))
   
       'Adds one day to Finish so that Less than can be used due to times associated with the dates
       lngFinish = DateSerial(Year(dteFinish), Month(dteFinish), Day(dteFinish)) + 1
       
       'Next line turns on AutoFilter as well as sets the criteria
       .Range("$A$1:$E$" & lngLastInput).AutoFilter Field:=4, Criteria1:=">=" & lngStart, Operator:=xlAnd, Criteria2:="<=" & lngFinish
   
       'Assign visible range of 3rd column to a range variable
       With .AutoFilter.Range.Columns(3)    '.AutoFilter.Range is generic reference without knowing the actual last row.
           Set rngVisible = .SpecialCells(xlCellTypeVisible)
       End With
      
       .AutoFilterMode = False     'Ensure AutoFilter is turned off
      
       .Cells(1, "H") = "Start"
       .Cells(1, "I") = "Finish"
       .Cells(2, "G") = "Date Range"
       .Cells(4, "I") = "Jobs Completed"
       '.Cells(4, "G") = "Employee"   'Gets inserted after Duplicates removed
       .Cells(2, "H") = dteStart
       .Cells(2, "I") = dteFinish
   
       'Use following line before Copy Command.
       'Recent changes to Copy can cause problems if line of code not used before Copy command in VBA
       Application.CutCopyMode = False
       rngVisible.Copy      'Copy the range (3rd column) assigned to the variable
      
       'Paste the Employee list to cell G4
       .Cells(4, "G").PasteSpecial Paste:=xlPasteValues, Operation:=xlNone, SkipBlanks:=False, Transpose:=False
      
       .Range(.Cells(4, "G"), .Cells(.Rows.Count, "G").End(xlUp)) _
              .RemoveDuplicates Columns:=1, Header:=xlYes       'Remove duplicates from Employee list
   
       Application.CutCopyMode = False
   
       .Cells(4, "G") = "Employee" 'Overwrites the column header "empnum"
      
       'Insert the formula in cell I5
       .Cells(5, "I").Formula = "=COUNTIFS($C$2:$C$" & lngLastInput & ",G5,$D$2:$D$" & lngLastInput & ","">=""&$H$2,$D$2:$D$" & lngLastInput & ",""<""&$I$2+1)"
      
       lngLastOutput = .Cells(.Rows.Count, "G").End(xlUp).Row   'Find last row of output from the Employee column
      
       .Cells(5, "I").AutoFill _
              Destination:=.Range(.Cells(5, "I"), .Cells(lngLastOutput, "I")), Type:=xlFillDefault  'Copy the formula down
             
       '***********************************************************************************
       'Comment out the code between the asterisk lines if you want to view the formulas in Column I
       .Range(.Cells(5, "I"), .Cells(lngLastOutput, "I")).Copy      'Copy the formulas
      
       .Range(.Cells(5, "I"), .Cells(lngLastOutput, "I")).PasteSpecial _
                  Paste:=xlPasteValues, _
                  Operation:=xlNone, _
                  SkipBlanks:=False, _
                  Transpose:=False
       '***********************************************************************************
      
       Application.CutCopyMode = False
      
       'Tidy up the output with column width and centreing the values
       With .Range(.Cells(1, "H"), .Cells(lngLastOutput, "I"))
          .HorizontalAlignment = xlCenter
       End With
       .Range(.Cells(1, "G"), .Cells(4, "I")).Font.Bold = True
       .Columns("G:I").Columns.AutoFit
      
       .Cells(1, 1).Select
      
    End With
End Sub
