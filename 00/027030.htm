trevor rabey say:

Assuming all auto-scheduled tasks.

I definitely recommend using the assign resources dialog (resource, assign resources button) to assign resources.

Don't just pick them off the drop down in the resource column.

Setting the max units is only to provide a trigger for determining the level when a resource goes over-allocated.

It is not necessary to change max units in order to make whatever resource assignment you want to make.

However, max units does affect the default resource assignment in the following way:

  • If you leave max units at 100% which is the default, then when you assign the resource to the task the default assignment will be at 100%.
  • If you set max units to 30% (or any number less than than 100, except 0) then when you assign the resource to the task the default assignment will be at 30%.
  • If you set max units to any number greater than 100) then when you assign the resource to the task the default assignment will be at 100%.
  • If you set max units as low as 0% (allowed, but why would you?), then when you assign the resource to the task the default assignment will be at 100%.

You don't have to make the default assignment, but can specify some % in the assign resources dialog.

If you do make the default assignment, you can then change it.

However, if you change it, you must be aware of the task type and effort driven settings which will affect the result of the change.

This is probably the source of your misery. I used the default fixed units, effort driven off, and when I change the assignment units from 100% to 30%, duration changes to 16.67 days, which is as expected. So I just over-type in the correct 5 days and all is well.

trevor rabey say:

Ok. Let's start from the beginning. There are, let's say without counting them, a lot of fields attached to each task and also a lot of fields attached to each resource, and each one has a specific definition. It is essential to have a crystal clear understanding of the definition of each field, or at least the most commonly used ones. The first stop for the definition is by hovering over the heading of the column, then read further, and then practice. It is risky to rely on intuition or assumption about what the fields mean and how they should be used and many people get into deep trouble simply by misunderstanding a definition or the intended purpose and use of a particular field, or assuming them.

Max Units:

MSP determines if a resource is over-allocated by comparing max units, which is a limit set on how available the resource is, to the current overall assignments.

For example, in the simplest case, if the same resource is assigned at 100% to each of two 1 day tasks which are scheduled for the same day (and time), then the resource is assigned at 200%, 200 > 100 so the resource is over-allocated.

For example, if max units is 100%, and the resource is assigned to one task only at 200%, the resource is over-allocated.

For example, if max units is 100%, then it could be assigned to one task at 100%, or two simultaneous tasks at 50% each, without being over-allocated. If assigned to one task at 50% then the remaining 50% of availability could be assigned to other simultaneous tasks up to the limit, say even 50 tasks at 1% each.

The result of these examples can be most clearly seen in the task usage and resource usage views, with the correct zoom on the timescale, and you can zoom right down to minutes.

Apart from having an effect on the default assignment as described in my previous reply, max units has nothing to do with how you make the the assignment. Whatever max units is, you can make whatever assignment you want, and you don't even have to know what max units is when you do it. When you make an assignment you either care or don't care whether you are over-allocating a resource. It is easier to not care initially about over-allocation, with the intention of letting the leveling shake out the over-allocation, but in some case you may need to care.

trevor rabey say:

By entering 50% in the Units box, she’s now assigned to work 50 percent on the “Build the fence” task.

This question is about resource assignment. max units will come into it but not yet. max units is about resource availability but it is possible, certainly easier and probably best, to ignore availability when making a resource assignment. Let's assume availability ie max units = 100%.

By entering 50% in the units box she is now assigned at 1/2 of every hour. This leaves 50% remaining availability which can be assigned to other simultaneous tasks before the resource is over-allocated.

If the resource had been assigned at 100% there would be no more remaining availability for other simultaneous tasks, and if any further assignment is made the resource would be over-allocated.

Where you say,

"I would expect that MS Project would schedule the work with the selected resource, based on priority, resource availability, and predecessors",

this is only partly a correct expectation. Mainly, MSP does not schedule work based on resource availability. It schedules work based on resource assignments.

It is best to avoid any expectation at all and instead accept what you get, ie what MSP does and doesn't do.

MSP follows instructions, doggedly.

It does not make decisions about project management.

It does not optimise, but it can produce a feasible plan (one with no resource over-allocation) from one which is not feasible before levelling.

MSP does not know what your intentions are and it cannot read your mind.

It does not ever automatically adjust resource assignments (except in connection with task type, effort driven and W=ExD.

MSP will indicate a problem but does not make decisions that the PM has to make about solving it.

For example, depending on the particular circumstances, there are many possible solutions to over-allocation and leveling to delay tasks is one of them, and that can be done in a number of ways according to the options chosen.

The obvious solution to over-allocation is not level at all (because tasks and maybe even the project will be delayed), but to remove some of the assignments of the over-allocated resource and instead assign someone else who is not yet fully allocated/assigned up to the limit of their max units, but that someone else may not exist, so that solution is not feasible. That means tasks have to be delayed by leveling, but which ones and by how much?
