hal hostetler - mvp-outlook say:
That's a common result of placing a .PST file anywhere but on the local hard drive.  Network shares or any online storage of these files is not supported.  Additionally, one does not sync Outlook on one machine with Outlook on another machine, the way syncing is done is to sync both machines with the server.  What kind of email account is this, POP, IMAP, Exchange ActiveSync (Outlook.Com/Hotmail/Live mail), Exchange Server, something else?  POP does not support syncing, but can be "fudged" into doing it in a very limited fashion.  IMAP supports email sync, ActiveSync adds support for Calendar and Contacts, Exchange Server is about the only service that syncs everything, but Exchange Servers are primarily a business entity and not available to the general public except by a subscription hosting service.

You will need to create a partition and make sure to install it to that partition to do the dual boot.

Here is what I did.

I opened partition manager (just search partition).

I opened up free space, then created a new partition. If you don't know how use google :p

I downloaded the .iso for the tech preview, and I downloaded daemon tools lite (it's free)

I then mounted the .iso onto a virtual drive. Again if you don't know google.

I went into explorer, opened up the virtual drive. Shows up as a blu-ray drive.

Inside you will find a folder called sources. Open that folder.

Use the setup.exe that is there.

When it's done you will have dual boot automatically, however the Tech preview will take priority, so if you want to change that, you can do it during that menu. Hope this helps.

jay freedman say:

Uninstalling/reinstalling Word (or Office) may or may not remove cached credentials, depending on how thorough the uninstaller is. It's probably a waste of a couple of hours.

Instead, first try the Key Manager built into Windows. This description is based on https://jespermchristensen.wordpress.com/2008/03/27/remove-cached-credentials-in-windows/, but I've used it in later versions up to Windows 10 and it still works.

  • Start a command window (cmd.exe). If you aren't logged in as Administrator, open the command window "As administrator".
  • If the command window prompt indicates that the current folder is anything other than C:\Windows\System32, use the CD command to go to that folder.
  • Enter the command rundll32.exe keymgr.dll, KRShowKeyMgr and press Enter. The Key Manager interface opens:

  • Select the OneDrive credential and click the Edit button.

  • Enter the correct user name and password, and click OK.
  • Test the login scenario.
