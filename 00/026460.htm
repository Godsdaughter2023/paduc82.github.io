bill manville say:

It seems to me that Ctrl+Shift+u does not autofit the formula bar height to the contents.
Instead it toggles between 1 line and the number of lines (>1) that the formula bar was last set to display.

SendKeys is inherently unreliable - if there is another program gobbling characters from the keyboard other than Excel it might get them first.

The only other thing i can think of is to do something smart by putting the formula in an autosize text box and setting its width to the width of the formula box to see how high the box ends up. And you probably would need to use Windows APIs to find out the width of the formula box since the boundary between name box and formula box is variable.  Not something that sounds worth the effort. 

It could be worth making the suggestion for some future version that setting FormulaBarHeight to 0 or -1 or some such should auto-fit to the contents.
