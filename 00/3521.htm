Deleting files on Sharepoint

The problem is that I now have two copies of the tables on Sharepoint, and I can't delete any of them because the tables are relationally linked.

How do I delete the files from Sharepoint and start over? Or can I move the files to a sub site and then delete the sub site and then start over? 

dr peter oz say:

Hello everyone,

Thank you for all your comments and suggestions. Taking a bit from everyone, together with MS Support, the problem was solved.

When you try to delete a table (or Sharepoint list) that is relationally connected to another table on Sharepoint, you have to delete one of the tables first, before you can delete the other. That's pretty obvious.

However, what was not so obvious, is precisely how to do it. The set up is a bit different from MS Access on my hard drive.

Firstly, if you try and delete the table (Sharepoint list), you get a message at the base of the table saying "We're sorry, we had trouble removing this. You can try again from the settings page." 

So, you go to the settings page, and attempt to delete the table (list) and you get this error message, "You cannot send this list to the site Recycle Bin. This list has the following related lists ...." (It then goes on to list the MS Access tables which are relationally connected.

So, the trick for me was to delete the tables slowly in order, and one by one. And, in the "one to many" relationships, I had to delete the "one side" before I could delete the "many side". However, to delete a relationally connected table, you have to go to the settings page, then scroll down to almost the bottom of the screen. You then see the table properties, which may have the relational connection by virtue of "Lookup" or "Number" attribute. Delete that connection, and then you can delete the related table. As simple as it may sound, scrolling down the screen to find the table properties to delete seemed to elude me.

I had around 5 or 6 very lengthy conversations with MS Support on the problem. The technician also sought support from other colleagues. MS seemed rather perplexed as well. After the problem was resolved, I suggested that Sharepoint documentation and help material be updated to make it a lot easier for others treading the same path.

Thank you everyone.

ossiemac say:

You can only directly use the ListFillRange in another workbook while the other workbook is open so it is best to set up a link to the source data list and this link will update when the source workbook is not open. This is the best method even if you want to have both workbooks open simultaneously.

  1. Open both workbooks
  2. Select the workbook containing the list
  3. Select the range containing the list and Copy
  4. Change workbooks to the one with the ComboBox
  5. Select the first cell of the range where you want the list (Can be on different worksheet to the ComboBox)
  6. Right click and select Paste Special -> Paste Link.
  7. Now the ListFillRange will be something like the following (Which from your question you already know.)

Sheet2!A1:A30

If you have any spaces in the worksheet name like the following then enclose the sheet name in single quotes.

'Sheet Two'!A1:A30

If you always want both workbooks open then you could use something the following for the ListFillRange

'[Example List.xlsm]Sheet1'!A1:A30

You can get most of the syntax from the first cell of the previous explanation. Select the first cell of the linked data and copy the formula in the formula bar (but NOT the equals sign). It will only have the first cell of the range so just edit it in the ListFillRange field and add the colon and last cell of the range.
