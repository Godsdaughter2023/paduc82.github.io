cbarnhorst say:

You are correct.  You are asked for a product key only if you start the computer with the installation media (clean install).  You are not asked for a product key if you insert the media while Windows 7 is running (upgrade).  You need to do an upgrade in order to recieve Windows 10 for free.  

After the computer has been upgraded you may perform a clean install and will not be asked for a product key.  Go to Settings/Upgrade and Security/Activation and see if your computer is activated.  If it not you need to reverse the process and roll back to Windows 7 and make sure you upgrade back to 10.  As for your Windows 7 key, product keys never cross version boundaries.  Only a Windows 10 key works with Windows 10, etc.

Microsoft is not providing a product key that can be used to activate Windows with the Get Windows 10 (aka GWX) program  The dummy key you see with KeyFinder or other such programs is blocked on the activation servers just like OEM keys are.  The dummy key is the same for everyone (for the same edition) that upgrades.  The dummy key tells Microsoft that your copy of Windows 10 is upgrade licensed.

You do need to wipe the old Win 7 harddrive when finished.  The Win 7 license is bound to the Win 10 license in the GWX Program and cannot be used to activate Win 7 anymore unless you rollback.

As to how you would know when the upgrade is ready and it is time to switch back to the other hard drive, you wouldn't because the GWX program on the other hard drive is of course not running.

All of this old/new/old again hard drive scheme is not necessary.  You're presumed to be upgrading a working system while waiting for the GWX notification.  It is unnessary.  The more so because you plan to clean install after the upgrade anyway.

tekkpc say:

Well, I now know its a Windows issue. I did a test by booting the laptop in CentOS. The connection to the LAN and the Internet was automatic. Both with the wired and wireless adaptors. Moving on since I'm feeling like a dog chasing its tail. Anyways, thanks for your response.

I lied! :-)  I figured one more day of troubleshooting might just yield a solution. And it did. Putting the search parameters the right way pointed to this URL. Windows 7 Network Awareness: How Windows knows it has an internet connection

How does it work?

Windows does indeed check a Microsoft site for connectivity, using the Network Connectivity Status Indicator site. There are a few variations of the connection checking process:

NCSI performs a DNS lookup on www.msftncsi.com, then requests http://www.msftncsi.com/ncsi.txt. This file is a plain-text file and contains only the text Microsoft NCSI.

NCSI sends a DNS lookup request for dns.msftncsi.com. This DNS address should resolve to 131.107.255.255. If the address does not match, then it is assumed that the internet connection is not functioning correctly.

The registry for this to work is:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\NlaSvc\Parameters\Internet

Under the Internet key, double-click EnableActiveProbing, and then in Value data, type: 0.The default for this value is 1. Setting the value to 0 prevents NCSI from connecting to a site on the Internet during checks for connectivity.

Mine was set to "0". I set it to "1" restarted the laptop and boom! The network connected to the Internet without delay.
