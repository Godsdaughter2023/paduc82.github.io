mike h.. say:

Hi,

try this macro

Sub Blank_Rows()
Dim i As Long
With Application
    .Calculation = xlCalculationManual
    .ScreenUpdating = False
LastRow = Cells(Rows.Count, "A").End(xlUp).Row
    For i = LastRow To 1 Step -1
        If WorksheetFunction.CountA(Rows(i)) = 0 Then
            Rows(i).EntireRow.Delete
        End If
    Next i
        .Calculation = xlCalculationAutomatic
        .ScreenUpdating = True
    End With
End Sub

A Solution... updating to Build 17063 - Microsoft Community

georgeogden say:

A Solution... updating to Build 17063

My last successful install was to 17035 after that 17040, 17046 and then 17063 all failed when it came to the reboot - the download and install appeared to work OK - just the Windows logo but no spinning dots. On restarting I found the error condition was 0x80240034

Even an "in place" install from an ISO failed with an error 0xC1900101 - 0x20017 ......

This was reminiscent of problems I'd experienced with several builds after 16199 (all until 16278) during the "Fall Creators" updates which I eventually tracked down, with a 99.9% certainty, to missing or corrupt motherboard (Nvidia iForce) drivers so, after several attempts with clean installs, updates and "in place" updates I went hunting for a solution.......

Looking into "C:\Windows.old\Cleanup" directory  I found that it contained three folders (I assume one for each of the last 3 failed attempts) named 0000.~BT, 0001.~BT and 0002.~BT which each contained a directory "....\NewOS\Windows\System32". Therein were two DLLs (nvsvc64.dll and nvsvcr.dll) and a further directory path to "...\DriverStore\FileRepository" which contained the directory "nvakwu.inf_amd64_0b3c1a15295d17ee" (you may find yours has a different name).

I then retried the update (from 17035) again BUT BEFORE restarting I copied and replaced only "nvakwu.inf_amd64_0b3c1a15295d17ee" into the "C:\$Windows~BT\NewOS\Windows\System32\DriverStore\FileRepository" as both the two DLLs were present and the same in the "...System32" directory.

I then restarted and the update ran to completion..... the usual restarts and a couple of anxious moments with blank screens later ......

OK this got me to 17063 but this just isn't good enough!!! Microsoft appear to have a totally random approach to handling drivers when it come to Insider builds .... the graphics drivers are a year old while my production PC is only a month or two behind with the OEM's release.

Anyway, I hope this helps others get past Microsoft's inadequacies and on to build 17063!
