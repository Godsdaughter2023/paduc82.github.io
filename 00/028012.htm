There is no good Help documentation for Pub 2013 so reference: 

Tips and troubleshooting for sending a publication page as an e-mail message (2003):

http://office.microsoft.com/en-us/publisher/HP010390591033.aspx?pid=CH062524821033

http://office.microsoft.com/en-ca/publisher/HP010390591033.aspx?pid=CH062524821033

You can go into File > Options > Advanced > Web Options and check "Send entire publication page as a single JPG...". This will increase the size of the email significantly and if the user has their email client set to not view images or text only, then they won't be able to read it.

You can deconstruct your newsletter to a simple text box of a web friendly text, no borders, no infill, no special formatting and test. If successful then add one level of formatting at a time testing as you go until you find what causes the problem. Follow the KISS principles. This is an older publication but still relevant to the conversation:

While you are at it, if you didn't choose an 'email template' you might want to resize your publication to be 5.818" or less in width which is the width of the Publisher 'email templates'. gord dibben say:

You can do that with sheet event code.

First, Format the cells you want use for date entry as Unlocked

Add this event code to sheet module.  As written when you enter data in a cell in the range A2:A47 that cell becomes locked from further change.

Private Sub Worksheet_Change(ByVal Target As Excel.Range)
    Dim ocell As Range
    On Error GoTo enditall
    ActiveSheet.Unprotect Password:="justme"
    Application.EnableEvents = False
    For Each ocell In ActiveSheet.Range("A2:A47")
        If ocell.Value <> "" Then
            ocell.Locked = True
        End If
    Next
enditall:
    Application.EnableEvents = True
    ActiveSheet.Protect Password:="justme", DrawingObjects:=True, _
            Contents:=True, Scenarios:=True
End Sub

tom ogilvy say:

Rob,

Excel is set up to allow entry of data in cells.  A Formula or Formulas that refer to the entered value can calculate a result.   It doesn't directly support they types of actions you describe you want.  Thus Gordon's suggestion of using a macro. 

A non macro approach (which I don't recommend because of the situation I describe):

If you want A51 to calculate a value 1 time and to not change that value, then you could put in a formula in A51 that uses intentional circular references.   (A51 would need to be locked as well)

A51:  =if(A51>0,A51,If(A50=0,0,A50*100))

This will raise a circular reference error unless you go to the Workbook Calculation settings under formulas in options and check the "Enable iterative calculations" checkbox.  (note that this is an application level setting)

If you don't have control of the instance of excel where this will be used, then it is unlikely that Iterative calculations would be enabled in options and you would get the circular reference error. 

The alternative is to use macros.  My previous contribution was not to avoid using macros.  I just was illuminating that use of macros to control user behavior can be easily overcome and that approaches that encourage the enabling of macros could be frustrating if the users security settings have them disabled automatically without notification.   As long as you understand that I think macros are your only practical solution as previously stated. 

Note that I said "When you are battling your users" in my initial post.  If your users are behaved and you are only trying to help them not make mistakes, then there should be no problem.
