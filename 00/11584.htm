karl timmermans [outlook mvp 2012-2018] say:
I am trying to copy a .pst file that is 18GB to a 64GB flash drive and I get the error message that the file is too big. What am I missing?

This has nothing specifically to do with it being a <.pst> file but rather how your flash drive has been formatted which is likely has a FAT32 volume which can handle a max file size of 4gb.

Reformat your flash drive as a NTFS volume and the issue will be resolved.

How do I format a USB Flash Drive to NTFS file system?
http://www.ntfs.com/quest22.htm

*** be sure to save anything that already exists on the flash drive since everything will be gone after the drive is reformatted.

The proposed code (1) saves the existing calculation setting, (2) changes the calc mode to manual, (3) code runs, (4) changes the calc mode back to the saved mode.  If the calc mode was manual when the code starts it remains manual when code execution completes.  In that case, nothing is calculated, even if some values were changed (the worksheet would be wrong).

Your best bet may be to just add one line to each appropriate sub or function.

It would be located just before the sub or function exits.
It would look like...
If Application.UserName = "uyentule01" Then Application.Calculation = xlCalculationManual

(requires your actual user name)

That would allow the existing code to automatically calculate, but if the code were run on your machine,
it would switch back to manual calc when the code completes.
Note that not all sub/functions should have that line added.  If you can't make that decision comfortably then don't do it.

Please reconsider and look again at adding the button to the QAT to toggle calculation.

jhadurd say:

Linking information from title page into headers and footers

Hi, I have text on my title page that I need to have included on my header and footer.  This information changes with each revision of the document.  Currently I have to change the information on my title page, then go to the header and footer and change the information there as well.  Is there a way to link the text on my title page to the header and footer so I only need to change the title page and the header and footer update automatically?

 

Eventually this document is going to become a template that other people will be using and the simpler the better.  :)

suzanne s. barnhill say:

Although you can use linked content controls in Word 2010, that is probably overkill for what you're doing. A method that is 100% backward-compatible is a StyleRef field. Apply the Title style to your title and then, in the header/footer, insert a StyleRef field referencing the Title style. Repeat for any additional information, using a character style instead of a paragraph style as required. See http://WordFAQs.mvps.org/StyleRef.htm.

 

If you're doing this in a template, a useful approach is to insert MacroButton NoMacro fields (see http://word.mvps.org/FAQs/TblsFldsFms/UsingMacroButton.htm) for the title and other elements. For example, you can have a Title-style paragraph containing only a field that displays [Type title here]. For elements within a paragraph, you can have [Type <whatever> here] and format the MacroButton field with the <whatever> character style.

 

This is the technique used by many of the templates shipped with earlier versions of Word, and most users find it very easy to use. Keep in mind that you can create a single-page template that includes First Page, Even Page, and Odd Page headers and footers; the header/footer content is stored in the template until needed as pages are added.
