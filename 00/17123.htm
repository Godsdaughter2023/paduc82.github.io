j.e. mcgimpsey say: There's certainly no "built in formula" for rebasing, but it's not too hard to do it using separate cells.

Assume: 
  • Data to be rebased in cells A2:A100
  • Rebased values in cells B2:B100
  • Transition point original value in J1
  • Transition point rebased value in J2
Then in B2, enter

   =A2 * $J$2 / $J$1

Copy B2 down to B100.

One could easily make it flexible using lookups, etc., but without any idea how your data is laid out, it's impossible to make specific suggestions.

Follow these steps:
  1. If the Developer tab isn't displayed, click Office button > Word Options > Popular and check "Show Developer tab in the Ribbon".
  2. On the Developer tab, in the Controls group, click the fourth button in the bottom row ("Legacy Controls") and click the second button under "ActiveX Controls".   3. With the text box selected, click the Properties button (or right-click the box and choose Properties from the menu).
  4. In the properties list, set these values: ScrollBars = 3, MultiLine = True, WordWrap = False. The other properties can keep their defaults.
  5. Drag the box's sizing handles to the desired size.

  7. Paste the text of the script into the box.
  8. Close the Properties dialog, and turn off the Design Mode button. Save the document.

Here are the mistakes you made: Using Windows Easy Transfer (it does not work with Outlook) and assuming that just copying the files to the default location would magically connect them to Outlook. Word 2016 Chicago Style (Author-Date System) sorts References - Microsoft Community

aaron w. hsu say:

Word 2016 Chicago Style (Author-Date System) sorts References list incorrectly

When sorting the References list for the Chicago Style in Word 2016 using the Author-Date system, the references list should be sorted alphabetically by author and date, not author and title, as in the Notes and Bibliography system. The Bibliography Style provided by Word 2016 is incorrect and sorts on the basis of Author and Title instead of Author and Date.

This can be fixed by adding the following lines around line 6365 in the %APPDATA%\Microsoft\Bibliography\Style\CHICAGO.XSL file:

<xsl:if test="string-length(b:Year) > 0">
  <xsl:text>&#32;</xsl:text>
  <xsl:value-of select="b:Year" />
</xsl:if>

This content should be inserted into the b:SortingString block and should occur after the elements dealing with the Author, and before the elements dealing with the Title.
