Hi,

The answer to your first question: What exactly is SuperFetch? Does it have anything to do with drivers?

SuperFetch is a service that was introduced with Vista and has been part of Windows ever since. It tries to speed up your computing experience by pre-loading parts of programs into memory with sometimes detrimental effects as you had to experience. It tries to determine which software you use and stores parts of it to the Windows folder. SuperFetch is a technology which allows Windows to manage the amount of random access memory in the machine it runs on more efficiently. SuperFetch tries to make sure often-accessed data can be read from the fast RAM instead of the slow hard drive. you're currently using over background tasks and adapts to the way you work by tracking the programs you use most often and pre-loading these into memory. With SuperFetch, background tasks still run when the computer is idle.  However, when the background task is finished, SuperFetch repopulates system memory with the data you were working with before the background task ran. 

Answer to second question: Why is SuperFetch causing such high CPU usage?

As you are using Solid State Drive (SSD), there are possibilities that the SuperFetch service is conflicting with the performance of SSD thus causing high CPU usage.

Method 1: Run the Hardware troubleshooter

 

a. Type troubleshooting in search bar and click it

b. Click Hardware and Sound

c. ClickHardware and Devices and follow the instructions 

 

Method 2: Check if the hard drive is detected in Device Manager

 

a. Press the Windows key and the R key to get the Run prompt

b. In the Run Prompt, type devmgmt.msc and press Enter

c. Browse to Universal Serial Bus Controllers

d. See if the external hard drive is detected or not. If yes, right click on it and click on Uninstall

e. You may be prompted to delete the drivers completely from the computer

f. Select Yes and click on Ok

g. Once the driver is uninstalled, restart the computer

h. After the restart, windows will automatically try to install the drivers for External hard drive

 

If the issue still persists, follow next method.

 

Method 3: Assign drive letter from Disk Management and check:

 

1. Press Windows key + X, click Disk Management

2. Check if external hard disk is detected, if itâs detected assign different drive letter and check

3. To assign a drive letter if one has not already been assigned, click Add, click the letter that you want to use, and then click OK

4. To change a drive letter, click Change, click Assign the following drive letter, click the letter that you want to use, and then click OK

5. Once done, check if the external hard drive is detected or not

lead3 say:

Most all USB "stick" drives are formatted FAT32 since they then can be used on most devices,

Mac's, PC's, etc. (not all devices can use NTFS).  While NTFS has some advantages over FAT32, I would not change the drive unless you have a reason to have it formatted as NTFS., e.g., the need to store a file larger than 4GB.
