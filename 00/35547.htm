¡firedog say:

For POP3 accounts, the main difference between using folders under the account or folders under Storage folders is that if the account is removed, all its folders and their contents will be permanently deleted. This comes as a surprise to users who are used to other applications like Outlook or Outlook Express, where removing an account has no effect on message folders. The program warns the user that this will happen, but many users have ignored the warning with disastrous consequences:


 

      'Remove account' warning

For the careful user, it is often more convenient to use folders under the account because these remain near the top of the folder pane and not at the bottom of it as Storage folders do. User-created folders, whether within an account or under Storage folders, will be shown in alphabetical order below the 'system' folders: 


Folders sort alphabetically after a program re-start.


Right-click on a folder and select Rename to re-arrange
folders to suit. The characters '( )' are not allowed
in folder names, but '[ ]' are.


If a POP3 account is configured not to Leave a copy of messages on the server, then messages once downloaded will be removed from the server and will not be available in webmail or to other devices accessing the same mailbox, like a smart phone. To change this option, right-click on the folder name in the folder pane and select Properties. The setting is at the bottom of the Advanced tab.

It's different with accounts that synchronize folders between client and server, i.e. accounts configured to use IMAP or HTTP (DeltaSync) (Hotmail-type accounts). Messages moved from an account folder to a folder under Storage folders will be removed from the server at the next sync. This makes it possible to retain local copies of messages while removing them from the server, so they will not be available in webmail.
