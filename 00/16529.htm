Does the Outlook Junk Mail provide feedback to the spam filter? - Microsoft Community

marcrohde say:

Does the Outlook Junk Mail provide feedback to the spam filter?

Does Office 365 (Exchange) use the end user feedback to "tune" the enterprise spam filter or are the settings all retained on the client?

For example if a user indicated "block sender" or sets a safe sender list will the organization either inherit that or use it to "vote" for likely spam?

We have been using another spam filter with some inherent machine learning and are looking at using Exchange (cloud) with the advanced threat features but I want to understand if the Outlook client data will help tune the engine.

gary zhu msft support say:

Hi Marc,

 

First, I’d like to talk about mail flow in Office 365. For inbound email, the incoming email will go through external network, Exchange Online Protection (EOP) and the user’s mailbox. The spam filter exists in EOP. It means that the incoming emails will  be restricted by Spam filter. Please see the following two different situation:

  • If we want to the email pass through the spam filter, we should set the spam action as “Move message to Junk Email folder”, “Add X-header” or “prepend subject line with text”. In this situation, the organization will “inherit” the end user’s side (go to Junk Email folder or not) when they block or whitelist a sender or domain.
  • If we set the spam action as “Redirect message to email address”, “Delete message” or “Quarantine message”, those incoming emails will not be delivered to the end user’s side. It means the end user will “inherit” the organization when the sender or domain is added in the block list of spam filter.

Gary

macropod say:

Given a range, the following codes are not working.

range.Font.Underline = WdUnderline.wdUnderlineSingle;

range.Font.ColorIndex = WdColorIndex.wdDarkRed;

When working with fields, the font attributes are managed by applying either:

• the required formatting to the entire field, including the field braces (i.e. not to just the field's result); or

• adding a Charformat switch to the field (delete any Mergeformat switch that's present) and applying the desired formatting to at least the field code's first character (e.g. { MyVar \* Charformat }).

Neither approach allows you to apply formatting to only part of the field. Any other approach will result in the formatting being lost when the field is refreshed.
