Creating a Bootable USB for Windows Insider ISOs - Microsoft Community

jason[msft] say:

Creating a Bootable USB for Windows Insider ISOs

Hello Windows Insiders!

One of the questions that comes up from time to time is how to create a bootable USB with a Windows Installation (from an ISO file).  Having this can be helpful in various scenarios, such as bypassing a particular issue or bug, or even to quickly setup a new device into the Windows Insider program.

Should you want (or need) to create a bootable USB drive from a Windows Insider ISO file, please see the directions below:

*NOTE*  There are numerous third-party software applications that are capable of converting a USB drive into a "bootable" format.  For the purpose of this example, we've selected Rufus as it is a free application and lightweight on system resources.  You can use this or any other software of your choosing that can complete this functionality.  Whatever software you choose, it should be able to properly format the USB into a bootable format as well as convert the ISO into the decompiled installation files.

  1. Download the latest Windows Insider ISO here -> LINK
    • Ensure you save this file to a location you can easily remember

  2. Connect the USB drive that you'll be converting to a bootable USB
    • IMPORTANT:  Any data on this drive will be erased.  Back up any important data before proceeding.
    • USB drive size/capacity should be at least 8GB

  3. Launch the Rufus software (or other software client of your choosing)

  4. Configure the Rufus menu options as seen in the image below.  Please take note of the following:
    • If you have multiple drives connected via USB (external hard drive, thumb drive, etc.), please ensure you select the proper drive from the drop-down list.
    • File system should be set to NTFS.  If for some reason this does not work on your PC, you may re-create it using FAT32
    • Provide the "New Volume Label" of your choosing.
    • Ensure "Create bootable disk using" is set to <ISO Image>.  Click the disk icon next to this line and navigate the path to the ISO file that you downloaded from step 1.




  5. Click "Start" and allow Rufus (or other software client) to format your USB drive and decompile the ISO as needed.

  6. Once the process is completed, you're ready to go.  You now have a bootable USB.


That's all there is to it!  You now have a USB drive that is formatted into a bootable state and has a Windows Install on it that you can use.

If you are using this drive to upgrade a device that already has Windows on it, insert the thumb drive into the target PC, open File Explorer, locate the drive, and launch setup.exe from there.

If you are using this drive to do a clean Windows install which overwrites and clears the hard drive, insert the thumb drive and reboot your PC.  You may also need to adjust your BIOS or UEFI settings to allow booting from a USB-based drive.

I hope this has been helpful.  If you have any questions about this process, please share your comments below.

Thank you as always!
Jason

¡firedog say:

Why am I receiving MAIL DELIVERY SUBSYSTEM errors? I have not sent any emails.

These emails are called bounce messages, or more properly Delivery Status Notifications (DSN). A DSN is simply informing you that a message you apparently sent couldn't be delivered. If you examine the DSN in detail, it will tell you who the message was addressed to and why it couldn't be delivered. The reason is often that the address no longer exists.

Since you state that you haven't sent the messages referred to (e.g. there is no sign of them in your sent items folder), then a criminal has used your email address as the sender address when sending out spam to many recipients. The addresses he sends from and to are obtained in a variety of ways, e.g. by hacking someone's account and getting hold of its associated contacts list or by simply buying them from another criminal. 

Because the sender address on the spam message he sends out is yours, the mail server for the addressee will notify you if the message couldn't be delivered. If you're quite sure that you didn't send the message, you can simply delete the DSN and forget about it. Please don't report it to anyone: it's a legitimate message sent by a mail server, so you would be 'shooting the messenger.' Mail servers are in fact required by the Internet rules to inform the sender that a message couldn't be delivered, but not all of them do it all the time.

This is a common annoyance, it constitutes no danger to you and there's nothing much you can do about it. Changing your password would only help if your computer had been infected with malware that sends out spam; if your computer is clean as you say, you're not the one at fault. If you really wanted to help, you could search the DSN to find the IP address the original message (the one that bounced) came from, then look up that IP address to see which network it belongs to, then report the case to the network's abuse contact. However, by the time you've gone through that procedure, the criminal will most probably have moved on to use a different network for his nefarious activities, and it won't stop him. 

tl;dr

If you didn't send the email referred to, you don't have to worry about it.
