Hi Joseph,

Basically, you can create a list/task list to achieve your request.
Features:
1. Unique ticket number assignment
You may use the existing ID (start from 1 in the list) or create a workflow to generate an auto-incremental ticket number based on your requirement.
2. Update capabilities (i.e. open, assigned, in process, closed, etc.)
You can create one or more columns to maintain the status of the tickets.
3. Archiving abilities
You can set up an archived site and create a workflow to copy closed tickets/items to archived location. Looking back to closed tickets is also feasible.

It is optional to create an InfoPath form associated with the list for end users to submit the detailed information. So that a new item (i.e., ticket) contained the details in the form can be automatically created in the connected list. In this case, a license for InfoPath applications is needed.
