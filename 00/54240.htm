At a command prompt, type: diskpart to start the Diskpart utility: "Diskpart version 0.0" is displayed as well as a Diskpart prompt.

  1. At the prompt, type: list to list the mounted hard drive volumes.
  2. At the prompt, type: select x (where x is the number of the hard disk drive that you want to convert to GPT). The following information is displayed: "Selected Disk = x".
  3. At the prompt, type: new gpt to add the GPT information into the non-volatile RAM (NVRAM).

    NOTE: This step does not create the GPT partition information.
  4. At the prompt, type: create name="efi system partition" type=efisys Size=102 to create the Extensible Firmware Interface (EFI) partition.

    NOTE: The name can be anything you want, but if the name has spaces in it, the name must have quotes around it. The type must equal either Efisys, Msres, or Msdata. If a size is not specified, the partition uses the remaining unpartitioned space. To install Windows XP, the Msdata partition must contain at least 2 gigabytes (GB) of free space.
  5. At the prompt, type: create name="microsoft reserved" type=msres Size=31 to create the MSR partition.
  6. At the prompt, type: create name="basic data" type=msdata to create a partition for the remaining unpartitioned space.
  7. At the prompt, type: inspect to display the partition information.
  8. At the prompt, type: exit to quit the Diskpart utility.

a. user say:

 

Hi Kumar,

 

I understand that you want to remove write protection on your Flash drive which you are trying to access on your Windows 8 computer. For better troubleshooting please provide the following information:

 

1) What is the make and model of the flash drive?

 

2) Do you remember the changes made, which might have caused the issue?

 

3) Did you try connecting the pen drive on another computer and check if it works?

 

 

Method 1: I would suggest you to follow the steps and check if the issue persists.

 

a) Open 'My computer'.

 

b) Right click on USB drive and select 'properties'.

 

c) Uncheck the option 'Read only' if it is marked. Note: Click on yes when the option pops up saying "Apply changes to subfolders and files writing to this folder"

 

d) Click on 'Apply' and 'Ok'.

 

e) Try to delete the files.

 

 

Also try to run the virus scan and check:

 

Refer to the following link:

http://windows.microsoft.com/en-us/windows-8/how-find-remove-virus

 

Disclaimer: There could be a loss of data while performing a scan using Microsoft safety scanner to eliminate virus if any.

 

 

I would also recommend you to contact the flash drive manufacturer and try to remove the write protection from the flash drive.

 

You can also perform the following steps and try to remove the write protection from the flash drive by making changes in Registry files.

 

a) Press Windows + R to open the Run Window and then type regedit and press Enter to open the Registry Editor.

 

b) Navigate to HKEY_LOCAL_MACHINE -> SYSTEM -> CurrentControlSet -> Control. Search for StorageDevicePolicies. If that doesn’t exist, then add a new key named StorageDevicePolicies. Now navigate to that and create a new D-WORD(32 bit), Write Protect.

 

c) Change the value of the key Write Protect to 0.

 

 

Important Note: Sometimes this problem is caused by two Windows registry entries that have become corrupted. To fix the problem, you have to use Registry Editor to delete the corrupted registry entries.

However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs.

 

For more information about how to back up and restore the registry, follow the steps as in the KB Article:

http://support.microsoft.com/kb/322756/
