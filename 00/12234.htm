A PPAM or PPA can be installed in pretty much any folder, but in order to make it easier for the user to find and install it, it's best to put it in the default folder. 

Typically this is:

C:\Users\[username]\AppData\Roaming\Microsoft\AddIns

The default trust center settings will normally work ok.

In Trust Center | Add-ins, none of the options should be checked.

In Trust Center | Macro Settings, "Disable all macros with notification" is probably best; when they install the add-in the first time, users will be warned that there are macros and will have to give permission to install.  After that, the add-in will quietly load at PPT startup.

Actual Duration and Remaining Duration if the task has been started, not finished.

Can not shape text in arch or circle - Microsoft Community

larryw7260 say:

Can not shape text in arch or circle

IN Word 2007 you are able to arch words or type words in a circle I cannot do this in Word 2010
jay freedman say:
The lack of text labels on some of the ribbon buttons is an unfortunate side-effect of a design decision that Microsoft made.

If you have a wide enough monitor that you can set the horizontal resolution to 1680 pixels, and you expand the Word window to that full width, then every button on the ribbon (except the ones in the Font and Paragraph groups on the Home tab) will have a text label.

As the window width is reduced from there, first some of the text labels are hidden, leaving only buttons (which must be identified by their tooltips when you mouse over them). With further reductions, more of the buttons are stacked two or three high, and then entire groups are collapsed to single buttons with dropdown galleries containing the commands. Widening the window restores the stacked buttons and then the labels.

A further unfortunate side-effect comes from the fact that the points where these transitions occur for each ribbon tab were hand-coded into the Word program. We've been told that it would be too difficult to change this mechanism to allow the ribbon to be oriented vertically on the sides of the window, as could be done with toolbars in Word 2003 and earlier.

a g broadhurst say:

TOC goes beyond edge of page for book 6x9

I have a book 6x9 for which I need a table of contents but the TOC extends beyond the right edge of the document and I cannot figure out how to get it to show the page numbers on the page.  Is there something I cannot see in the format for TOC?

suzanne s. barnhill say:

This won't happen if you set the page size and margins before generating the TOC; you may find it easier just to delete the existing TOC and insert another one, but you can correct the current one by modifying the tab settings. The easiest way to do this in Word 2010 is as follows:

  1. If not already displayed, display the Horizontal Ruler (View | Show | Ruler).
  2. Switch to Draft view by choosing the Draft icon on the right side of the status bar or by using View | Document Views | Draft.
  3. Draft view will allow you to see the TOC entries extending off the page. You will also see the tab stop for the page numbers off the end of the ruler. You won't be able to drag that tab stop to where it belongs, but you can set a new one. Click the tab type button on the left end of the ruler until it shows a right-aligned tab (two clicks if it's set at the default Left Tab). Then click on the ruler just at the right margin (you'll probably need to click just left of the margin and then drag into place).
  4. You can click on that tab stop to open the Tabs dialog, where you can delete the tab stop that's beyond the margin. You'll also need to add a leader (#2) for the tab you set, which won't have one.
  5. Since TOC styles are set to update automatically, making this change in any entry at a given level will cause all the other entries at that level to update, but you will need to repeat the process for one of the entries at each other level (TOC 1, TOC 2, TOC 3, etc., for however many levels you have used).
