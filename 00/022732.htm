john korchok say:

You can quickly add your cover sheet to any existing document if you place it in AutoText:

  1. In Word, choose Word>Preferences>View and under the Show Non-Printing Characters section, check All. OK out. This makes breaks visible on-screen.
  2. Create your cover page and end the page with a Page Break (Insert>Break>Page Break).
  3. Select the page, including the Page Break, but nothing after the Page Break.
  4. Choose Insert>AutoText>AutoText.
  5. Give the entry a suitable name like CoverPage in the Enter AutoText entries here: field.
  6. Very important: click on Add. The AutoText is added to the template attached to your document, usually Normal.dotm. OK out.

Then when you want to insert the cover:

  1. Click just to the left of the first character on the first page.
  2. Choose Insert>AutoText>Autotext.
  3. Choose CoverPage and click on Insert. The cover page is added to the beginning of your document.

To streamline this further, you can give the entry a keyboard shortcut:

  1. Choose Tools>Customize Keyboard.
  2. In the Categories list, scroll down to AutoText and select it.
  3. In the Commands list, find CoverPage and select it.
  4. Click in the Press new keyboard shortcut field and enter your preferred keyboard shortcut. Control + Option + C is one that's not assigned to anything else.
  5. Very important: click on the Assign button. OK out.

Now you can insert a cover in any document created from the same template by clicking before the first character and pressing Control + Option + C at the same time.

robert aldwinckle on forums say:
 

Ok. how can I view the expiration date of cookie?


Where is the expiration date?

 

In the Cookies\index.dat  but it is binary in there so there isn't something you could read.   E.g. Run...  shell:Cookies  to find the path and then open its index.dat in Notepad and you can see what I mean.

There may be an API to access it.   In which case you could perhaps execute a script which displayed it.   Perhaps then even via PowerShell...   Must check.   ; )

When the OS is XP you can use the TIF Viewer  (e.g. from IE, press Alt-T,O,Alt-S,V) to see both TIF and Cookie data extracted and displayed.   I can't remember if XP has a separate  desktop.ini  file for Cookies.   If it does you might even be able to see it displayed as a special folder even with the above  shell:Cookies  command.

Hmm...  I'm so used to seeing nothing useful in W7's TIF Viewer that I just surprised myself and noted that some Cookies are still being displayed that way in it.   Who knew?   So, even if you don't have it in Details view, with the Expires column enabled you can at least right-click Properties on Cookies there and see Expires:  that way.
