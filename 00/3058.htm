Visio Data Set Issue

Hi

I have found a problem using data sets with custom Visio stencil shapes. The procedure below describes how to reproduce the problem.

We use Visio to describe detailed network objects . By applying multiple data sets we can build up a description of that object.  For computer this might be,  Host configuration, Network interfaces etc. Applications etc.

Because we use compound data sets it means I cannot change/edit any data set once its applied to a drawing.

To setup the document

 

Create a new document

 

Add basic shapes to the document : a  square & a circle

 

Select a shape – select shape data from the menu and then select shape data sets.

 

Create 2 x new data sets with the following custom properties:

 

DataSet1 : property 1 ; property 2;  property 3

 

DataSet2 : property 4; property 5 ; property 6

 

 

Create new custom stencil & edit.

 

Move objects from page to stencil to create master shapes.

 

Assign both datasets 1 & 2 to each stencil shape and save the stencil

 

Test shape data:  Test shape data: 

Drop shapes onto page – each will show properties 1- 6 when shape data view is enabled.

 

The Issue:

 

Edit either of the data shape definitions. Add a new property (property 7). Close the data set.

 

Do not apply to any shape in the stencil!!!!

 

Drop a shape onto the page and view shape data.

 

Shape now only shows properties from the data set that has been edited.

 

Switch to design view in developer mode: shape now shows both data sets associated with it.

1. Each font may have different support for different Unicode ranges. For example, here on Windows Vista+Office 2007 SP2, Arial has the IPA Extensions (or at least some of them) but the "default font" Calibri does not.

2. There is a lot more about the versions of fonts provided with each version of each Microsoft product at

http://www.microsoft.com/typography/fonts/

When you look up a font there, you will see that in some cases there are many different versions. For each version, you can see which Unicode ranges are supported. Perhaps the site will also tell you how you can get an updated version of a common font (I don't know, sorry).

Even then, you may need to consider what will happen if you use IPA Extensions with say, Calibri, and supply a document to someone who does not have the same font version as you.

Finally, "Windows Vista SP2 and Windows Server 2008 SP2 x86 retail symbols" may provide you with some updates to some of the fonts, but this is in effect "by accident". The "symbols" in the title of this package have nothing to do with the symbols/glyphs in fonts/typefaces. These "symbols" are for use in source code debugging and are, for example, the names of variables, entry points, etc. in the source code of the Windows products.
