bernie deitrick say:

Filtering has changed somewhat with the advent of tables. Before, you could only have one filter on a sheet - now you can have each table have its own filter (though it is better to make sure that the tables are stacked vertically so that filtering does not 'cross-talk').

So, you are not using a sheet filter on a range, but a table filter on a defined table. To find out a table's filter status, use code like this for "Table1"

Sub TestFilterStateOfTable1()
Dim boolF As Boolean

On Error GoTo NotActive
boolF = ActiveSheet.ListObjects("Table1").AutoFilter.FilterMode
MsgBox "Table1 has filters " & IIf(boolF, "and is currenlty filtered", "but is not currently filtered")
Exit Sub

NotActive:
MsgBox "Table1 is NOT filtered"

End Sub

scottgem say:

A "text field with a drop down menu" is called a combobox. You don't have fields on a form, you have controls that may or may not be bound to a field in a table. This is a subtle but important distinction. Each control has a type and a specific name for the type.

It sounds like you used the combobox wizard to create a search combo. Such a combo is, by necessity, unbound. So if you change records using any other method than the search combo, the combo does not update. 

If you want to keep them in sync this is easy. in the On Current event of your form:

Me.cbosearch = Me.PKcontrol

where cbosearch is the name of your search combo and PKcontrol the name of the control bound to your primary key field.

john mcghie say:
Hi Lisa:

You can stop your disclosure right there :-)

ActiveX does not exist on the Mac, so anything you send to someone on a Mac containing ActiveX will behave as a picture.

However, provided that the document leaves you in .docx format and returns to you in .docx format, the ActiveX control will not actually change into a picture on the Mac.

In .docx, Word has been trained to leave unchanged code that it doesn't understand.  So unless the Editor saves to the old format, your ActiveX controls will remain usable in PC Word.  If he saves as .doc, then Mac Word will downgrade the ActiveX to pictures.

Yes, .docx is the default on Word 2007 unless you change it.  It's also the default in Word 2008 unless he has changed it.  Best idea is to look at the extension on the file name: it should read .docx if that's what the file contains.

You should also ask a very specific question of the Editor about HIS update levels.  Ask him to open Word>About Word and tell you what his version number is.  If he is fully up-to-date, he will see 12.3.1.  Mac users are not very good about putting in their updates :-)

I need a formula that will highlight a date if it is becoming close to expiration, so I can inspect the item.  I would like it to highlight when the "date" is 15 prior, 30 days prior, and 45 days prior.  Is this possible?

Date Calibrated Next Inspection Due
12/30/2014
11/3/2015 11/3/2016
11/3/2015 11/3/2016

teylyn say:

Hello,

create a conditional formatting rule that uses a formula. This is the formula you need

=A1<=TODAY()+15

Adjust A1 to reflect the cell you want to check. Adjust the number at the end to reflect the days prior. If you want different colors for different time frames, you will need a rule for each time frame.

Let me know if that helps.
