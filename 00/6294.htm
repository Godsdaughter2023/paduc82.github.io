Ransomware & OneDrive - Microsoft Community

aitch52 say:

Ransomware & OneDrive

Hi folks,

Am I correct in assuming that, should a pc be hit by a ransomware virus and the files are encrypted then files in the local OneDrive directories would be encrypted and these encrypted files would be synchronised with the remote ones rendering both versions useless? This would equally apply to any other cloud storage.

rob koch say:

In addition to the link already provided, here's another explanation that refers to either the business or personal versions of OneDrive, along with the specifics for Office documents vs. other file types.

OneDrive vs. Ransomware (Updated)

Also, note that different ransomware have been known to perform the encryption in different ways, some by encrypting using a different file type, while others use the same exact name in order to attempt to overwrite any cloud or other backups.

Still others have been known to either overwrite the files multiple times or force a vssadmin.exe command in order to overwrite the local Shadow Volume Copies of files that also might be used to recover from an attack.

So the true answer to your question isn't a simple yes or no, it's more precisely a question of which type of ransomware you've encountered and whether it's been designed to damage your local and online backups or synced files.

The general recommendation today is to create some sort of offline backup often enough to recover from the worst case loss of all online copies.

The questions you need to ask yourself are how much of your data truly exists nowhere else and to what extent does this data change?  If most of your data are merely copies of financial or billing records which are already duplicated on the websites of those other businesses, then they're already backed up.

If, however, you create much of this data personally and it exists nowhere else, then the change or churn rate as well as value of this data can help you decide how often to create an offline backup.  If there's no other way to recover the data, then the value may be high, while if the files are only updated rarely, then it may not need to be backed up very often.

Remember that this offline backup option is only intended as a hedge for a catastrophic event where both the local and OneDrive or other online backup copies are both lost, which should only occur in a case like extremely malicious ransomware, so you need to balance the typically small possibility of this occurring with the extra effort required.
