jllatham say:

It's difficult, if not impossible, to do all you want with worksheet functions, formatting and data validation.  But I think this bit of VBA code might do the trick for you.  It is Worksheet event handling code, so it has to go into the worksheet's code module.

 

To get to the worksheet's code module, open the workbook, select the particular sheet you need this to work on and RIGHT-click on its name tab.  Choose [View Code] from the list that comes up.  That will open the VB Editor and present that worksheet's code module to you.

 

Copy the code below and paste it into the code module.  Change the value of Const tgtAddress to become the address of the cell you need to deal with, be sure you use the $ symbols.  Close the VB Editor and toss some test entries into that cell and see if you like it or not.

 

If you want to keep it, be sure and save your workbook as "Macro Enabled ..." type .xlsm, not as .xlsx.

 

Here's the code:

 

Private Sub Worksheet_Change(ByVal Target As Range)
  'address of cell you want to watch over
  Const tgtAddress = "$E$2"
  'ignore it if they deleted the entry
  'or if not the proper cell at all!
  If IsEmpty(Target) Or Target.Address <> tgtAddress Then
    Exit Sub
  End If
  'next we deal with entries like 123a or b221 or HI!
  If Not Application.WorksheetFunction.IsNumber(Target) Then
    MsgBox "You did not enter a number into the cell.", _
     vbOKOnly + vbCritical, "Invalid Entry"
    'force back into the cell but leave their mistake visible
    Target.Select
    Exit Sub
  End If
  'test if negative and just warn them
  If Target < 0 Then
    If MsgBox("You have entered a negative value.  Do you wish to KEEP the entry?", _
     vbYesNo + vbQuestion, "Negative Entry") <> vbYes Then
      Target.ClearContents ' clears their entry
      Target.Select
      Exit Sub
    End If
  End If
End Sub

I was hoping for a bunch of bug fixes, but so far the only obvious improvement has been the Google/Internet connectivity in the Search plugin (the "Home screen application" plugin on the Standby screen) coming back to life.  But I feel this was fixed more by the installer flushing some corrupted file rather than replacing bad code.  A user on the t-mobile.com forums suggested that WiFi now works normally with non T-Mobile SIM cards, but I haven't confirmed that.  Here is a list of smaller bugs I've found, that I wanted to post publicly somewhere.  Bugs related to Messaging, Email, WiFi, Maps, Voice functions, UMA, VPN, VOIP, etc. are not here, either because they're too time consuming to test or because I haven't used those features yet. :smileywink:

I am hoping other E73 owners will either post some of their own bugs here or post feedback on my list.  And I am curious how many of these bugs also plague the E72?

Browser bugs

1) Go "Back" a page in the History (pressing the "3" key works well) but press Stop before the new page can load.  Immediately try going "Back" again, and this page is now gone from the History.  The page before it loads instead.  Use the right soft key to choose a page farther back, but again immediately press Stop.  It appears nothing has changed, but now press the right soft key to flip through the History, and you will see two copies of the current page.  One is the last page in History (what you would expect), but there is now a copy which has replaced the original page you wanted to load.  Press "3" and you will load the History page previous to it.

2) Ctrl-A and Ctrl-V don't work in HTML text input or textarea fields.  (Ctrl-X and Ctrl-C do work.)

3) The layout of this very web site (http://discussions.europe.nokia.com) is slightly worse when viewed in "Full" (not "Mobile") mode.  View an individual post and the "Options" DHTML drop-down menu now spills into the page.  Followups to that post (and much of the rest of that page) now appear centered.

4) When replying to a post on this web site in "Full" (not "Mobile") mode with the "Reply" (not "Quick Reply") editor, several strange things happen.  Typing in the HTML textarea field with predictive text off, the "capitalize first letter" feature (i.e. "Abc" in the header field) can get stuck on, resulting in all caps.  Typing with predictive text and "Word auto-completion" on results in the auto-completed word being taken as the correct word.  That is, continuing to type does not change the auto-completed word, but simply appends to it.  For example, trying to type the word "something" types this instead

s

sometimese
sometimese2
sometimese26
sometimese26i

This happens with "abc" still appearing in the header field.  Since the firmware upgrade, there has been a slight change.  Instead of forcing numbers, it now activates browser keypad shortcuts.  So instead of a "2" appearing, the "Find keyword" search field appears instead.

Additionally, since the firmware upgrade I sometimes find myself unable to type at all.  The cursor remains flashing in the textarea field, but no variation of "abc", "123", etc. appears in the header field, so no characters appear.

All problems described in 4) appear to be exclusive to http://discussions.europe.nokia.com.  They do not appear when typing on other sites!  My initial feeling is that I would prefer having browser 7.2.6 back. :smileysad:

8) I cannot say for sure for the specific situation of yours, but the phones generally handle the cards inefficiently. The data is written to them in such a way that it becomes fragmented very soon. Since the cluster size is pretty big, each of the files which is not an exact multiple of a power of 2 (the cluster size) uses much more memory than its actual size due to multiple fragments. Since the reported free space is total space minus the total file size, it only shows the free space that should be present as if the files were packed most efficiently without any gaps. But since this is not the case, the memory equal to the reported free space is often not available.

In your case, the card's size is 8 GB and it shows only 340 MB free space. That means most of the space is occupied. Assuming a great many number of fragments, it's not surprising to say that it might actually have only a 100 MB or so worth of usable free space left, maybe even less. To determine more exactly, try copying a 330 MB file over to the card. If the copying fails, try a 320 MB one. If it fails too, try a 310 MB one, and so on.

When you backup the card on a computer, format it, and then copy the files back over to it, the computer writes the files one after the other. The data is not fragmented here immediately. As a result, the reported free space is very close the actual free space available for writing the files (you can check this like above). Probably that's why the problem goes away for the moment when you just backup, format, and restore the data on the card.

This change is part of Microsoft's decision to stop packaging any kind of clip art with Office. You have a few alternatives:

  • Find a friend who is still using Office 2010 or earlier, who can copy the Bullets folder from their computer to a disk or USB drive. For Office 2010, that folder is at C:\Program Files (x86)\Microsoft Office\MEDIA\OFFICE14\BULLETS (for earlier versions, change the "14" to the appropriate version number. Then copy that to any convenient folder on your computer, from which you can select bullets using the "From a file" method. [Caveat: this is a copyright violation.]
  • Download a collection of bullets from the Internet. A useful search term is "free clipart bullets". An example of what you may find is http://www.wilsoninfo.com/bullets.htm.
  • Buy a clipart collection. These often have a large selection of high-quality bullet images.
