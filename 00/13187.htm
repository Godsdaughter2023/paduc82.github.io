Is there a way to combine tasks planned backwards and tasks planned - Microsoft Community

mdubose say:

Is there a way to combine tasks planned backwards and tasks planned forwards on the same project?

I'm working as a Project Coordinator with a software group. For each feature they're working on, they need to provide the requirements group with a "drop dead" date for each set of feature requirements to come in in order to meet the software due date, based on resource assignments and estimates of work for each feature. This seems like a candidate for As-Late-As-Possible scheduling.

However, at a certain milestone, these dates are locked down and the software work (and late added content) needs to be tracked forward from those dates to see if we are on track to meet the software due date, or by how much we're missing it.

Since each software phase uses the same resource pool and they need to be leveled appropriately, (I think) each phase needs to be tracked on the same project - even though some phases must be tracked As-Late-As-Possible and some phases must be tracked As-Soon-As-Possible.

Obviously, it's a nightmare trying to manage resource leveling while using both of the soft constraints at once... Is there a sane way to manage this scenario in Project?

rod gill say: Don't ALAP. You need to know the critical path and have as much flexibility as possible. Schedule the work so as to meet the required date then set the deadline date for the deliverable. This still allows the milestone to move past the date, but flags a red diamond in teh indicator column as an exception report. You now know you have a problem and can re-schedule to recover lost time. This is exactly what's needed. Doing ALAP scheduling will introduce you to a world of grief including losing control of what's happening.

trevor rabey say:

Yes, well, vote with Rod on this one.

Whenever anyone comes across a situation like yours it seems to be "a candidate for As-Late-As-Possible scheduling", and that often turns out to be possibly the worst scheduling mistake you could make.

Don't do it.

Adopt the critical path method, which fundamentally is about scheduling forwards from the start date. Without knowing where your critical path and critical tasks are, you have nothing to manage and no chance of exercising any control over your schedule.

CPM first, resource leveling last.

toby tu msft support say:

Hi Gerald,

If you have “Full Access” permission to another mailbox, you can open the other person’s mailbox in a separate browser window.

  1. Log in to your mailbox using Outlook Web App.
  2. In the Navigation bar on the top of the Outlook Web App screen, click on your name. A drop-down list will appear. Click Open another mailbox.
  3. Type the email address of the other mailbox that you want to open, and click Open.
  4. The other person’s mailbox opens in a separate Outlook Web App browser window.

doug robbins - mvp office apps & services (word) say:

If Word 2010 is not running, to force the use of that version, you will need to use:

Sub Test()
    Dim mwApp As Object 'Word.Application
    If MsgBox("Is Word 2003 running", vbQuestion + vbYesNo) = vbYes Then
        MsgBox "Please shut it down and then re-run this code."
        Exit Sub
    End If
    On Error Resume Next
    Set mwApp = GetObject(, "Word.Application")
    If Err Then 'Word was not running
        Shell "C:\Program Files\Microsoft Office\Office14\winword.exe", vbMaximizedFocus
        Set mwApp = GetObject(, "Word.Application")
    End If
    On Error GoTo 0
    'Show open file dialog
    'mwApp.Dialogs(wdDialogFileOpen).Show
    'Open a file
    mwApp.Documents.Open "C:\temp\test.doc"
End Sub
