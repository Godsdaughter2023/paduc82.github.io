What is happening is that when you remove spaces, some of your values are converted to legitimate time entries.  Time entries are stored as as the fraction of 1 day (24 hours).  for example, your 11:00:00  is being displayed as

11/24 = 0.4533333333

if you format the cell as time, then you will get your time back.   The results are inconsistent because removing spaces does not make all your values legitimate time values that Excel recognizes as Time.

pete_uk say:

Dharmesh,

the problem here is due to formatting. When I copy your data into a blank sheet all the cells with a colon in them (except the two with dates) are treated as proper time values by Excel, so your formula to remove spaces won't work on those values - it is the formatting of those values which is introducing the spaces.

You are looking for something along the lines of:

IF(A1 is a number (i.e. date, except for the last one) then copy A1 and its formatting but remove any spaces from the format string, otherwise take the text value from A1 and remove any spaces (which is what your formula does now))

Unfortunately, a formula cannot act upon the formatting of a cell, so you would need some VBA solution to do this.

Incidentally, the two cells with dates have character code 160 (non-breaking space) between the date and time part, which is why you get a "space" in your output for those cells.

Hope this helps.

Pete

Auto Backup of an Access Database - Microsoft Community

tpstanley say:

Auto Backup of an Access Database

It there a way that I can have an auto backup of my Access Database when I close out of the database?  I am thinking of a VBA code that would do this upon closing the database.
ken sheridan say: Even though your database is operating in a single user environment, it is strongly advised, as the others have pointed out, that you split it into separate front and back ends (which the built in wizard will do for you) as this has a number of advantages, irrespective of making backing up the data easy.  You can, for instance, keep a 'master copy' of the front end in a safe location, and if the operational front end is corrupted, then you simply have to replace it with a fresh copy of the 'master.  It also means you can do further development work on a copy of the front end without worrying about inadvertently damaging the operational front end, which can then be replaced with the new one once you've finished and tested any changes you've made.

As regards backing up all you have to do from the front end is make a copy of the back end file.  To  do this when the database is closed make sure you have an opening form, which remains open in the background, hidden if you wish, while the database is open.  The code to copy the back end can then go in the opening form's Close event procedure, which will execute when you close the front end.

You'll find an example of suitable code in BackupDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

I'd recommend the code for creating a dated backup, but if you are likely to close the file more than once in one day, to make sure that it backs the data to a new file each time, format the date part of the file name as 'yyyymmddhhnnss' rather than merely as 'yyyymmdd' as in my code.  You'd also have to store the paths to the back end and backup files in a FileLocations table as in my demo.  In my case they can be entered via the backup form, but as you want to automate things, just make sure the paths are in the two column in the table beforehand.  You'll also need the two modules from my demo.

ken sheridan say: You can either change the Windows API function calls in the modules to 64 bit versions, or more simply:

1.  Instead of using Bill Wilson's BrowseForFileClass module use the FileDialog property of the Application object.  See the BrowseDemo file in my same OneDive folder for an example.

2.  Instead of the Windows API CopyFile function call the CompactRepair method of the Application object.  As well as copying the back end file this will also compact and repair the copy.
