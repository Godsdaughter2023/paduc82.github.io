trevor rabey say:

To expand a bit on John's excellent example,

We can't have scheduled duration in the past or progress in the future.

So when the progress in the future of a, e and f is shaved back so that progress only goes to the status date (or the end of the Friday in this case), and task b is either 100% or moved to the status date, then task b is either on schedule or a future task, and the summary is on schedule.

Then

if task b is moved entirely to the status date, it is a future task but not late, and the summary is on schedule.

If task b is 50% complete and the remaining 50% of the duration is moved to the status date, it is late, but the summary is on schedule.

Even if task b is 99% complete and the remaining 1% of the duration is moved to the status date, it is late, but the summary is on schedule.

j.e. mcgimpsey say: First - Are you sure your model is really valid? By multiplying each term, i.e.:

m *W  * n * X * o * Y * p * Z = C

and assuming C has units of currency $, then W will have units of (the fourth root of currency / sq feet), X will have units of (the fourth root of currency / number of bedrooms), etc. I'd think you're more likely to want this instead...

    mW + nX + oY + pZ = C

where W is in $/sq foot, X is in $/bedroom, etc.

Regardless, that's not something that XL has a single function to do. It's really more a problem of algebra:

   m1W + n1X + o1Y + p1Z - C1 = 0
   m2W + n2X + o2Y + p2X - C2 = 0
   m3W + n3X + o3Y + p3X - C3 = 0
   m4W + n4X + o4Y + p4X - C4 = 0

which you can solve for W,X,Y,Z in terms of ms, ns, os, ps, and Cs.

If you install the Solver add-in, I'd expect you could set up a scenario where you could Enter your m,n,o.p & C values in a table, along with the result of the equations above, and use 4 variable cells for W,X,Y & Z and constrain each of the equations above to zero
