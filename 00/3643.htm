Replacing Bookmarks with Custom Word Properties via VBA / Looping - Microsoft Community

la lawyer say:

Replacing Bookmarks with Custom Word Properties via VBA / Looping through all of the Word Documents in a Folder

I have decided to replace all of the Word fields referring the Bookmark "Today" (which has a text value) with fields referencing a Word custom property, also called "Today".  I have found, for my application, that custom properties just work a lot better than referencing a bookmark.

How can I use VBA to search and replace the fields which reference the bookmark "Today"?

Also, since this problem exists in a number of documents in a single folder ("c:\wordforms"), I'd like to be able to (1) loop through all of the documents, (2) replace these fields as necessary and, if necessary, (3) save the corrected document.  How is that done?

TIA
jay freedman say:
This is a combination of the macros in http://www.word.mvps.org/FAQs/Customization/ReplaceAnywhere.htm and http://www.word.mvps.org/FAQs/MacrosVBA/BatchFR.htm plus a little bit specific to your requirement of replacing Ref fields with DocProperty fields.

Public Sub ReplaceTodayFields()
  ' Change the pathToUse value to point to your folder!
  Const pathToUse = "C:\Users\Jay\Documents\"
 
  Dim thisFile As String
  Dim thisDoc As Document
  Dim rngStory As Word.Range
  Dim fld As Field
  Dim lngJunk As Long
 
  'Prevent AutoOpen macros from running
  WordBasic.DisableAutoMacros 1
 
  On Error Resume Next
 
  thisFile = Dir$(pathToUse & "*.doc*")
  While thisFile <> ""
    StatusBar = "Checking " & thisFile
    Set thisDoc = Documents.Open(pathToUse & thisFile)
   
    'Fix the skipped blank Header/Footer problem as provided by Peter Hewett
    lngJunk = thisDoc.Sections(1).Headers(1).Range.StoryType
   
    'Iterate through all story types in the current document
    For Each rngStory In thisDoc.StoryRanges
      'Iterate through all linked stories
      Do
        For Each fld In rngStory.Fields
          If (fld.Type = wdFieldRef) And (InStr(LCase(fld.Code.Text), "today")) Then
             fld.Code.Text = "DocProperty Today"
             fld.Update
          End If
        Next fld
        'Get next linked story (if any)
        Set rngStory = rngStory.NextStoryRange
      Loop Until rngStory Is Nothing
    Next
   
    thisDoc.Close SaveChanges:=wdSaveChanges
   
    'Get next file in folder
    thisFile = Dir$()
  Wend

  'Re-enable AutoOpen macros
  WordBasic.DisableAutoMacros 0
End Sub

btfu say:

SUCCESS!!  (I hope…)

I think I’ve finally found a way around this.  It involves using Notepad to “sanitize” the Excel data.  The latest procedure is below…

I'm going to call the offending file MicrosoftSux.xlsx (in honor of the headaches this has given me) and the new, clean file Clean.xlsx.

1.       Open MicrosoftSux.xlsx. 

2.       Open a new instance of Excel (just to be safe).  You can hold the Windows Key  + R to bring up the Run dialog box, type in Excel and hit OK. 

3.       In MicrosoftSux.xlsx, convert all formulas to text by using Find and Replace (Ctrl + H).  [All of the following steps are to be performed without the quotes.  The quotes are used here only for clarity.] In “Find what:” put “=”, in “Replace with:” put “#=”.  Under “Options >>”, ensure “Look in:” is set to “Formulas”, and click “Replace All”.  Give it a while. 

4.       Open notepad (Win + R à type "notepad" à OK)

5.       In MicrosoftSux.xlsx, select all data and copy. 

6.       Paste the data into notepad.

7.       In notepad, select all data and copy. 

8.       In Clean.xlsx, rename the first tab to the same name as the tab you were working in on MicrosoftSux.xlsx. 

9.       Select cell A1, and Paste the data copied from notepad. 

10.   Repeat steps 3 – 9 for each tab that you want to save from MicrosoftSux.xlsx. 

11.   Change back your formulas from before using Find and Replace (Ctrl + H).  [All of the following steps are to be performed without the quotes.  The quotes are used here only for clarity.] In “Find what:” put “#=”, in “Replace with:” put “=”.  Under “Options >>”, ensure “Look in:” is set to “Formulas”, and click “Replace All”.  Give it a while. 

12.   You may have to correct other items in the new sheet.  As an example, if you had dollar amounts in MicrosoftSuxs.xlsx that were zero, they would have shown up as “$   -“.  This text will be copied directly into Clean.xlsx, so you’ll need to use Find and Replace to fix these also.  ("Find what:” put “$   -”, in “Replace with:” put “0”.)  You may have some formulas that have a space appended to the front of them.  ("Find what:” put “ =”, in “Replace with:” put “=”.)  And so on.  It’s not perfect, but it works. 

13.   Repeat steps 11 – 12 for each tab. 

Now, this is where I am skeptical, but I’ve had good luck so far…  Formatting.  (Now, this worries me because it lets some of the “information” from MicrosoftSux.xlsx back into our new Clean.xlsx.  But, like I said, I’ve had good luck so far with it.)

14.   Save and close Clean.xlsx, and reopen it in the same window as MicrosoftSux.xlsx. 

15.   On your first tab, select all and hit the format painter (the little paint brush). 

16.   Switch over to Clean.xlsx and click the paintbrush cursor on cell A1.

17.   Repeat steps 14 – 16 for each tab. 

18. Pray that you never have to deal with this again....

 

Hopefully this solutions holds up and works for you also.
