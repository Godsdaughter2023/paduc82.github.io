christopher hastings say:

How can I insert a building block using a check box?

I am trying to find out how to insert a building block by using a check box. I looked at this Community post, but when I used a Legacy Control the check box would not let me "check it". When I tried a Content Control, there was no Enter/Exit option. Same problem with an ActiveX Control. Anyone know what I need to do to fix this? Here is an image of the Legacy Checkbox:

Even though it looks like the checkbox is selected, I just clicked off of it several times.

jay freedman say:

If you use a legacy check box, you must protect the document for "filling in forms" in the Restrict Editing pane to make the check box field respond to clicks. This has many consequences: The rest of the document cannot be edited except in other legacy form fields, and several features (e.g., spelling checker and inserting graphics) become disabled even in sections that are not protected. Not the least, if you want the check box to run an exit macro, then the macro must be written to unprotect the document, insert the building block, and reprotect the document (because inserting a building block or anything else is disabled while protection is in force).

If you use a content control check box, the document doesn't need any sort of protection. You're correct that there isn't any exit macro feature for it. Instead, you must write an "event handler" macro in the ThisDocument module of the document or its template. First, in the control's Properties dialog, enter a unique value for the Title or the Tag or both -- that becomes the control's name.

After double-clicking the ThisDocument module in the Project pane of the macro editor, set the left dropdown at the top of the code pane to Document and then select ContentControlOnExit in the right dropdown. That inserts the first and last lines of the macro:

This event handler will run whenever the cursor exits from any content control in the document, so the first thing the macro must do is determine whether the control that was exited (contained in the parameter named ContentControl) is the check box you're interested in. Do that by writing a statement like 

    If ContentControl.Title = "MyCheckBox" Then

followed by the code you want to run if that is the correct control, and ending with an End If statement. Caution: the condition written this way is case-sensitive; the name in quotes must exactly match the value you put in the Properties dialog. The same is true if you use ContentControl.Tag in the If statement.

I would recommend that you stay away from ActiveX controls. Compared to form fields and content controls, they occupy more memory and are sometimes less reliable.

One last comment: If the location of the building block will always be occupied rather than having it completely absent when the check box is unchecked, you can use a building block gallery content control to let the user choose the building block.

graham mayor say:

If this document has been created by mail merge then each envelope will be separated from the next by a section break. The following macro (one of Dougs) will treat each section as a separate print job.http://www.gmayor.com/installing_macro.htm

Sub SplitMergeToPrinter()
Dim Letters As Long
Dim Counter As Long
Letters = ActiveDocument.Sections.Count
Counter = 1
While Counter < Letters
    ActiveDocument.PrintOut Background:=False, Range:=wdPrintFromTo, _
    From:="s" & Format(Counter), to:="s" & Format(Counter)
    Counter = Counter + 1
Wend
End Sub

If that doesn't work, start again using one of the envelope templates that you can download from the end ofhttp://www.gmayor.com/changing_envelope_layout.htm
Create a new document from the template. Change the page size to match the envelope size that you are using.
Create a letter merge document (not an envelope merge) using the on screen envelope as the 'letter'.http://www.gmayor.com/merge_labels_with_word_2007.htm
Attach your data source and insert your merge fields in the Envelope Address frame
Merge to the printer (or to a new document and use the above macro to print it).

I would also like to add a second criteria for those results that bring back duplicate rankings.  For instance, if cell D15 and Cell D25 end up with the same rank I need it to look at cell E15 and E25 respectively for the new rank for each of them.  Something like that anyway.  This is a ranking fo survey results and when two dealers end up with the same survey score we want to then look at their respondent volumn to weight the results further to keep any dealer from winding up with the same ranking position.

pete_uk say:

As you have marked Mike's post as the answer, I'm not sure if you still want my response, but here goes anyway:

 

I can't see your workbook, so I haven't a clue how your data is laid out. What I was suggesting is that you use an amendment to the score which takes account of all tie-breaks. It might be at the moment that you only have two scores which tie for first place, but in future this might not be the case. As you do not have a formula in column D which can be amended, then I would suggest that you use another column to combine the score with the volume, and to apply the RANKing to that new column. So, put this formula in X15, for example:

 

=D15+L15/10000

 

and copy this down to X167. You can then delete 2 dells in every three, so that it matches your other format. Now you need to amend that formula in K15 so that it is looking at column X instead of column D. An easy way is to select D15:D16 and then CTRL-H (Find & replace) and:

 

Find what:           D

Replace with:     X

then click Replace All

 

Now this will change the MOD to MOX, and so you will probably get a #NAME error in D15 and D16, so you just need to change that back to MOD and then you can copy the formula down as before.

 

Note that if two suppliers have the same score AND the same volume, then you will still get the two tied, so in that case you might like to think about a third tie break, so an amendment to the formula in X15 might be:

 

=D15+L15/10000 - ROW()/10000000

 

which would then maintain the order in which they appear on the sheet.

 

Hope this helps.

 

Pete

jim gordon mvp say:
The Library folder is not visible by default in Lion. To make it visible, run Terminal application (You'll find Terminal in the Utilities folder inside the Applications folder).  Paste this into the Terminal window and then press Return

chflags nohidden ~/Library/

You can then quite the Terminal Application and you will find the Library folder where you expected it in the User directory.
