jeeped say:
Is there a conversion available in excel to move this data into minutes?

In Excel a day is equal to 1 and time is a decimal portion of 1. Noon (12 hours) is 0.5 and 1:29 is 0.0618. Number formatting turns the 0.0618 into 1:29. You can easily convert the decimal time into a number representing the number of minutes by multiplying by the number of minutes in a day (e.g. 1440 or 24Ã—60). Remember that if you want to use that number when dealing with time (e.g. adding or subtracting), you must divide by 1440 to get back to the decimal or use it within the TIME( <hours>, <minutes>, <seconds>) function.

I don't believe that Excel's number formatting allows for representing minutes over 59 as a true time (e.g. decimal number).

pasha... say:

Security Alert outlook.office365.com

Office 365 users may get this Alert while connecting from Public/Home network on outlook 2010,2013 or 2016.

I would like to explain why this Alert is presented.

This issue may occur under the following circumstances:
  • The user tries to create a new profile in Microsoft Office Outlook.
  • The user tries to start an Outlook client.
  • The issue occurs intermittently when the Outlook client is running.

If the user clicks Yes, the user can continue the operation. However, if the user clicks No, Autodiscover lookup fails. The failure of Autodiscover lookup prevents the following features from working as expected:
  • Automatic creation of an Outlook profile by using Autodiscover
  • Out of Office (OOF) Assistant
  • Free/Busy information

Why it happens: Generally, this issue occurs when the URL that you are trying to access is not listed in either the Subject or the Subject Alternative Name (SAN) of the Secure Sockets Layer (SSL) certificate for the website. Although different organizations' configurations may differ slightly, this issue typically occurs because the organization's Autodiscover Domain Name System (DNS) records are configured incorrectly.

Why Outlook client presents this error:

In the back ground outlook locates the autodiscover service with FOUR fundamental operations or steps.

1. The Service Connection Point

2.User's SMTP Domain

3.User's SMTP domain is per-pended with Autodiscover like https://techsupporto365.com

4.Outlook client tries to locate a DNS service (SRV) Record for the Autodiscover service in the DNS zone that matches the user's SMTP domain like _autodiscover._tcp.techsupporto365.com

In each operation it tries to find the A record of the domain , so it connects there to get exchange information.Once it finds A record, it connects with exchange online to get the xml autodiscover settings.Now if the URL that is presented to exchange online doesn't match the SSL (Subject Alternative Name in the SSL certificate SAN) certificate with exchange online, the office365 certificate Alert will be sent back to client.

To fix it for whole organization you need to change the SAN with your domain name. I will post another blog for "How to change the SAN in SSL certificate)

Quick FIX for a single client.

To fix this issue you need to export and import this certificate into IE trusted sites.

1. Export the certificate on your desktop.

2. Open IE as admin and from Gear icon Go to InternetOptions

3.Click COntent tab and than Certificates tab

4.click Import and Browse to exported certificate.

5.Select trusted root certification authorities and click ok to install the certificate.

That's it.

If your SAN needs to be edited, I will post a another POST for that.
