The first thing I would try would be to start Excel with the /regserver switch. This causes Excel to recreate its file associations, registry values, references, and other internal plumbing. With Excel closed, go to the Windows Start menu and choose Run. There, enter the following:

 

"C:\Program Files (x86)\Microsoft Office\Office14\Excel.exe"  /regserver

 

Of course, change the folder path to your installation. You can get this folder name from the VBA Immediate window. Open VBA and press CTRL G to display the Immediate window if it is not already visible at the bottom of the screen. In that window, enter the following and press ENTER:

 

?Application.Path

 

This will return the folder path that you should use in the Run command described above.

 

If you have multiple versions of Excel on the same machine, do this procedure for each version of Excel, starting with the earliest version then moving on to later versions.  The /regserver switch can absolve Excel of a multitude of sins.

a. user say:

There are PowerShell script that will do what you want.

First you have to go through all your users and turn off all forwarding.

Then you can turn off Forwarding for all your users and then none can forward.

Something like:

  • Get-Mailbox –Filter { RecipientTypeDetails –eq 'UserMailbox' –and ForwardingSmtpAddress –ne $Null } | Set-Mailbox –forwardingsmtpaddress $null
  • Get-RemoteDomain | Set-RemoteDomain –autoforwardenabled $false
