anitatesar say:

One and two pager documents with different headers and footers in word

Good day. I have a two page document in word. The first page has a header and footer, the second page has a different header and footer. I used a section break to do this. If I delete the second page, just want to send a one pager out to clients I loose the header and footer that was on the first page as well. Any way to get around this, or should I have a one pager and two pager stationery document.

doug robbins - mvp office apps & services (word) say:

You can have a First Page Header\Footer and a Primary Header Footer in which case, you do not need to use a Section Break.

However, with that set up, when you delete the second page, you must NOT delete the final paragraph mark (¶) as that Header and Footer information is stored in that paragraph mark.

WIth your present document, when you delete the second page, you MUST leave the Section Break, but you will want to change it to a Continuous Section Break so that you do not have an empty second page.

charles kenyon say:

See:

  • How can I get a different header/footer on the second page?
  • Setting Up Letter Templates - Ribbon by Suzanne Barnhill, MVP
  • Using Headers and Footers by Suzanne Barnhill, MVP
  • Header/Footer Settings Recap

Note, these are Windows-version pages. While the document structure is the same on the Mac, the location of controls is a bit different.

john korchok say:

Here are the steps to fix this in 2011:

  1. Make a separate copy of the document.
  2. Delete the section break. The header and footer on page 1 assumes the appearance of the one on page 2, this is normal.
  3. Choose Format>Document and click on the Layout tab.
  4. Check Different first page and OK out.
  5. Open the header and footer of page 1 and delete the header and footer.
  6. Open the old document and copy the page 1 header and footer to page 1 of the new document.

Windows phone 8 mp3 as ringtone - Microsoft Community

ashik aliyar say:

Windows phone 8 mp3 as ringtone

I want to set mp3 songs in my phone (windows phone 8) as ringtone. But i didn't saw any option for that. Is it possible to do that? If its possible then how to do that? Any one please help me...
[ pete ] say:

Please follow this how-to article.

Adding ringtones to your Windows Phone

====================

To add a ringtone using a Windows Vista, Windows 7 or Windows 8 PC

  1. Connect your phone to your PC, then do one of the following:

    • On a Windows Vista or Windows 7 PC, click Start, and then click Computer.
    • On a Windows 8 PC, click Desktop, click File explorer on the taskbar, and then click Computer.
    • On a Windows 8.1 PC, click Desktop, click File explorer on the taskbar, and then click This PC.

  2. Under Portable Devices (Devices and drives on Windows 8.1), double-click your phone, double-click the Phone folder, then double-click the  Ringtones folder.

  3. Find a sound file on your PC (for example, music files are often in the Music folder and organised by artist and album), then drag the file into the Ringtones folder.

    Note

    To add or rename a ringtone, you need to move a file from your computer to your phone. You can't add a ringtone by moving a file from a folder that's already on your phone or renaming an existing ringtone.

  4. On your phone, go to Settings > Ringtones + sounds, then tap Ringtone to see a list of your available ringtones. Your new ringtone will appear under Custom in the list.

Copy Field Names from one Table to Another - Microsoft Community

sancarloscyclist say:

Copy Field Names from one Table to Another

I have 2 tables and I want to copy the field names from one table to another without losing data from the receiving table.

Below is an example:

Table 1: No data with field names that I want to copy to Table 2

Table2: 175K rows of data.

My goal is to copy the field names from Table 1 to Table 2 without losing the data in Table 2. is there a simple way to do that? I considered doing an append query, but that is difficult since there are 98 fields in the tables. Your help is GREATLY APPRECIATED

dirk goldgar say:

It's not a macro, it's a VBA procedure.  In Word and Excel, what they call "macros" are VBA procedures but in Access, macros and VBA procedures are different animals.

Here's what to do:

1. On the Create tab of the ribbon, click "Module" (not "Class Module") to create a new code module.  The VB Editor window will open, with the new module (probably named "Module1" displayed in a window inside it.  

2. In the module window, the code pane will contain the line, "Option Compare Database", at the top.  Underneath that you may see another line, "Option Explicit", depending on your settings, but probably there'll just be the one line.

3. Copy the code I posted, from the "Sub CopyFieldNames ..." line to the "End Sub" line, and paste it into the module window, below the "Option ..." line(s).

4. Click the Save button on the toolbar to save the module.  You can either leave the name of the module unchanged, or you can change it -- but don't call it "CopyFieldNames", because Access gets confused if a module and a procedure have the same name.

5. On the menu bar, Click Debug -> Compile.  If any error messages popup, you'll have to deal with them, but if that's all the code in your database it ought to compile without errors.

6. If you haven't made a backup of the database already, do it now!

7. To call the procedure, press Ctrl+G to open the Immediate Window.  Into the Immediate Window, copy and paste the line:

    CopyFieldNames "Table 1", "Table 2"

and change the table names as appropriate.  For "Table 1" substitute the name of the table you want to copy field names from, and for "Table 2" substitute the table whose fields are to be renamed.  Leave the table names enclose in quotes.  When you've made the necessary changes, click at the end of that line and press Enter to execute it.

8. If any messages are displayed, that will mean something went wrong.  Otherwise, check the table in design view to see the result.
