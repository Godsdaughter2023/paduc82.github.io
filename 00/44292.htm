with winders ape and up
you must force a full shutdown of the system so that you can get into BIOS / UEFI,
also this is the only correct shutdown method when adding new hardware internally
fake shutdown (due to fast start up being enabled), sleep, or hibernate,
are not the correct methods of "shutdown" when changing or adding new internal hardware.


a> from an administrative user account, uninstall the graphics drivers
b> when it says reboot choose NO!
c> force a full shutdown (shift + shutdown at the power icon)
d> unplug and disconnect all cables
e> open the case and insert the new graphics card
f> close the case and connect all the cables
g> boot the system
h> press the appropriate BIOS / UEFI "entry" key
i> verify that the primary / first etc. graphics card is PCI-e
- some BIOS  / UEFI don't have a setting for this
and will disable the integrated as soon as an add-on card is detected
but still may require at least one  BIOS / UEFI entry to complete the changeover process

exit BIOS / UEFI saving changes and boot into windows,
install the correct drivers for the new graphics card.

DLL ("dynamic link library") files such as dnsapi.dll are small programs, similar to EXE ("executable") files, which allow multiple software programs to share the same functionality.

Most dnsapi.dll errors are related to missing or corrupt dnsapi.dll files. Because dnsapi.dll is an external file, it presents a great opportunity for something undesirable to occur.

Improperly shutting down your PC or getting a virus infection could corrupt the dnsapi.dll, which could lead to DLL errors. When your dnsapi.dll file becomes corrupt, it cannot be loaded properly and will present an error message.

I suggest you to update your Windows Defender (or Antivirus) and check. Also scan your PC with the Windows Defender.

palcouk say:

You should be aware that Libraries are simply pointers to the original location and as such data may 'appear to be' in several places, when in fact it isn't.

Storing data in other than the library locations can cause problems. You would be advised to use the Libraries to store your data, and what folders you create in the various libraries is entirely up to you.

And if you create data within a program, when you chose to save that data the Library locations will open in an explorer type window, by default

.

You can collapse or edit the Favourites 'folder' set and Collapse the others

The Desktop location shows shortcuts

The Libraries>Documents folder show what locations are included, and you can add/remove locations
