elizabeth23 say:

 The issue is that the clipboard has to be compatible with all legacy programs that might need to use clipboard data, so the clipboard format has not changed.  The clipboard format in windows is (for the most part) a 16-bit format, but windows fudges this up to 32-bit so it doesn't run out of memory handles too fast.  The problem is that you're running out of 32-bit memory handles, and the OS is having to struggle to find a memory handle to assign to your clipboard request.  This makes ctrl-C/ctrl-V operations slower (just like when any other 32-bit program starts running out of memory handles), or eventually unresponsive (just like when your 32-bit machine would hang or crash from running out of memory handles).  Restart your machine regularly, so that you always have a large number of 32-bit memory handles available for the clipboard, and the problem will go away. =)  Also, look into what 32-bit apps (or any other legacy apps) you may be running.  Some may be memory hogs, not correctly releasing memory back to windows when they are done.
