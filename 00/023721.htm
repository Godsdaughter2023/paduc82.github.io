joeu2004 say:
Ron wrote:
[Rick wrote:]
I guess it means I will have to "bite the bullet" and learn about Pivot Tables now... that is an impressive speed for sure.
I'd sure like to know how they do it.

It occurred to me that the PT implementation might use a hash table to process the reference list.

That allows the concordance to be built and counted with only one "pass" over the reference list.  And the "right" hash function would sort the concordance while counting it.  (Note that the PT result is sorted.)

I used a slightly larger example than Ron:  a concordance of 35,165 words occurring in a random list of 457,000 references, retaining the approximately 13-to-1 ratio in Ron's example.

On my computer, a C++ implementation of a hash algorithm takes less then 0.25 sec to build and count the entire concordance of 35,165 words from the list of 457,000 references.

(Compare that to about 80 sec for a C++ implementation of Rick's algorithm, which assumes knowledge of the 35,165 words in the concordance.)

A VBA implementation of the same hash algorithm takes about than 2.4 sec to build and count the entire concordance on my computer.

(Compare that to more than 25 min that I estimate for an improved implementation of Rick's algorithm.)

In comparison, the PT operation that Ron implemented takes about 2.6 sec on my computer.  That is consistent with Ron's observations; he wrote:  "So
The VBA implementation is below.  Some notes....

1. The reference list (457,000 words) is presumed to be in column A starting in A2 of worksheet "Sheet1".

2. The result is copied into columns G and H of "Sheet1".

3. Beware that column C of "Sheet1" is cleared because in my usage, that might have irrelevant formulas that would affect the execution time.

4. myTimer and myElapasedTime encapsulate my use of the QueryPerformance* functions.  With the long run-times of these routines, there is no significant loss of measurement precision for the counting portion if myTimer is changed to Timer and myElapasedTime(et-st), for example, is changed to simply et-st.  (But don't measure time across midnight. ;->)  Measurement precision loss might be significant for the much smaller non-counting portions (copy-in and copy-out).


-----


Option Explicit

Type wListT
    hlist As Long
    cnt As Long
    wd As String
End Type

Private Sub test5()
Const nHashtab As Long = 256& * 256
Dim hashtab(0 To nHashtab - 1) As Long
Dim rList
Dim rN As Long, wN As Long
Dim w As Long, r As Long, i As Long, x As Long
Dim h As Long, prv As Long, nxt As Long
Dim minHT As Long, maxHT As Long
Dim s As String
Dim st As Currency, et As Currency
Dim et0 As Currency, et1 As Currency

Worksheets("sheet1").Activate
Columns("c").Clear
Columns("g:h").Clear
Range("g1") = "Test5"

' copy list of references in from Excel
st = myTimer
rList = Range("a2", Cells(2, 1).End(xlDown))
rN = UBound(rList, 1)
ReDim wList(1 To rN) As wListT

et0 = myTimer
minHT = nHashtab: maxHT = 0
For r = 1 To rN
    s = LCase(rList(r, 1))
    ' hash function (h) is simply the first 2 characters of word
    h = 256 * Asc(s)
    If Len(s) > 1 Then h = h + Asc(Mid(s, 2, 1))
    ' search ordered list in the hash bucket
    prv = 0: x = -1
    nxt = hashtab(h)
    Do While nxt > 0
        x = StrComp(s, wList(nxt).wd)
        If x <= 0 Then Exit Do
        prv = nxt
        nxt = wList(nxt).hlist
    Loop
    If x = 0 Then
        ' reference to previously-found word
        wList(nxt).cnt = wList(nxt).cnt + 1
    Else
        ' reference to "new" word
        wN = wN + 1
        wList(wN).cnt = 1
        wList(wN).wd = s
        wList(wN).hlist = nxt
        If prv > 0 Then
           ' insert within ordered list in the hash bucket
            wList(prv).hlist = wN
        Else
            ' insert at front of list in the hash bucket
            hashtab(h) = wN
            If h > maxHT Then maxHT = h
            If h < minHT Then minHT = h
        End If
    End If
Next
et1 = myTimer

' copy concordance out to Excel worksheet, already
' in ascending order.
' hash table is ordered by first 2 characters of words.
' each hash bucket list is ordered by words therein
ReDim res(1 To wN, 1 To 2)
w = 0
For i = minHT To maxHT
    nxt = hashtab(i)
    While nxt > 0
        w = w + 1
        res(w, 1) = wList(nxt).wd
        res(w, 2) = wList(nxt).cnt
        nxt = wList(nxt).hlist
    Wend
Next
Range("g2:h2").Resize(wN) = res
et = myTimer

' wN is number of words in concordance
MsgBox "wN:  " & wN & _
    vbNewLine & "copy in:  " & Format(myElapsedTime(et0 - st), "0.000 sec") & _
    vbNewLine & "count:  " & Format(myElapsedTime(et1 - et0), "0.000 sec") & _
    vbNewLine & "copy out:  " & Format(myElapsedTime(et - et1), "0.000 sec") & _
    vbNewLine & "total:  " & Format(myElapsedTime(et - st), "0.000 sec")
End Sub

Method 2: To isolate the issue, I suggest you to create another user account and check if the issue persists. This will determine if the issue is a profile corruption. To do that, follow the steps below.

  1. Go to Start menu.
  2. Click on Settings.
  3. Choose Accounts.
  4. Select Other User Accounts towards the left.
  5. Select Add an Account.
  6. Towards the bottom, select “Sign in without a Microsoft account.”
  7. Choose Local Account.
  8. Enter a user name and hit Next.
  9. Click on Finish.
  10. Sign out from the current Account and Log into the new account.
