qa guy commelec say: If you want a descending sort then include a calculated field to convert Risk to a number like this --
Risk_Sort:  IIF([Risk] = "High", 1, IIF([Risk] = "Med", 2, 3))

ken sheridan say: Firstly you should have a separate table, RiskLevels, say in which the values High, Medium and Low are Stored in a primary key column, OveralRisk say.  This should be related to your operational table on OveralRisk and referential integrity enforced.  This then ensures the integrity of the data by forcing only valid values to be entered into the OveralRisk column in the operational table.  Add a further column RiskLevel to the RiskLevels table, with values of 1, 2 and 3 in the rows for High, Medium and Low respectively.  In the report's RecordSource query join the RiskLevels table to the operational table on OveralRisk  and return the RiskLevel  column in the query.  You can now use RiskLevel as the reports first sort level in the Group and Sort window in report design view.

For data input into the operational table you can use a combo box in a form based on the table, with a RowSource property of:

SELECT OveralRisk FROM RiskLevels ORDER BY RiskLevel;

By the above data is being stored as explicit values at column positions in rows in tables, as Codd's Rule #1, the Information Rule requires, and not hard code in expressions or value lists.
