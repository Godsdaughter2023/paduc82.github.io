    Is there a way to do this to work around this error message without affecting other users?  This is a multi-user database.

    Thank you.

    daniel pineault say:
    Although there are ways of copying a access database file when it is open, it is ill-advised.  You should never copy a database file while in use.  Most of the time it will be ok, but you could copy it in a state, when it is being used (records being edited, transaction not committed, ...) and it could end up being corrupted.  You are much better off with a technique that either copies the file for the first user entering the db, or the last person exiting the db, or a mechanism that boots everyone out to allow a backup to be performed.

    andreas killer say:

    ActiveSheet.Shapes.AddChart2(332, xlLineMarkers).Select

    The AddChart2 method was added in Excel 2013, means this code does not run in previous versions.

    Please never use SELECT, SELECTION, ACTIVECELL, it is slow and error prone. Always refer to the objects directly.

    The bold part in the code below is not necessary when Sheet1 is the active sheet, and can be removed.

    Andreas.

    Sub MacroChrt()
      Dim C As Chart
      Dim S As Shape

      Set S = ActiveSheet.Shapes.AddChart2(332, xlLineMarkers)
      Set C = S.Chart
      C.SetSourceData Source:=Worksheets("Sheet1").Range("G10:G40")
    End Sub
