External USB drives go offline when in active use - Microsoft Community

rgbell say:

External USB drives go offline when in active use

I have USB external drives attached to my desktop PC that are used with Acronis True Image 2019 for backups. If I have hard drives set to turn off after 20 minutes in Power Settings, the drives will go offline when they are actively being written to by Acronis. Shouldn't they stay online when in active use? How will USB Selective Suspend Settings affect this issue? I would like to be able to have my hard drives suspend when not in use.
junaid_a say:

Hi,

Thank you for writing to Microsoft Community Forums.

For most USB storage devices and USB host controllers, the power-saving mechanism works without issues. However, a pre-existing problem in the USB drive or USB host controller might cause the device to go offline after the specified idle period expires.

I would suggest you to check if the firmware on the USB host controller or the USB storage device is out of date. If so, get the latest firmware from the system or device manufacturer. A firmware update might resolve the issue.

If the firmware update doesn’t fix the issue, you may try the steps listed below to check if it works:

  • Plug in the USB device and obtain the hardware ID of the same.

    (Search Device Manager from the Start menu and select View > Devices by connection from the Device Manager menu. Right click the USB Mass Storage Device node under which the device appears and select Properties > Details. On the Details tab, select Hardware ID from the dropdown menu.)

  • Make a note of the 4 digits occurring after “VID_” and “PID_” and close Device Manager.

  • Run Registry Editor (regedit.exe) as administrator by searching for “regedit” in the Start menu. Right-click the regedit icon, and select Run as administrator.

  • In Registry Editor, navigate to: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\usbstor

  • Right-click usbstor and create a key. The name must be VID and PID strings obtained previously. Leading 0’s are significant.

  • Right-click the new key and create a DWORD entry named DeviceHackFlags . Set the value to 400 hexadecimal.

  • Disconnect and reconnect the device to see if the issue persists.

Note:

Serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. To backup and restore registry key, refer to this link.
