karl timmermans (outlook mvp 2012-2018) say:
First for the sake of accuracy, what you are referring to as "books" are in fact "contact folders" visible in the Outlook Navigation Pane.

To merge the data from the 2 contact folders, set the Outlook Nav Pane to show only contact folders by pressing either CTRL+3 or clicking on the People icon (or the word People depending on how things are set).

Select the contact folder you want to copy FROM and press CTRL+A to select all the contacts. The right-click on the highlighted contacts --> select <Move> --> the select <Other Folder> --> select the contact folder you want to copy TO.

If you are going to move a large number of contacts, would suggest that instead of selecting <Other Folder> in the process above select <Copy to Folder>. The reason for this is that in the event that something goes south during the process, Outlook won't delete anything from the source folder which occurs during the <Move> (i.e. <Other Folder>) process

No that method isn't reliable because it assumes that the action of a filter will actually filter out rows which it might not because no rows might meet the filter criteria. So in short we could have a filter and a criteria set and your method would return TRUE,

Try it this way

Dim bn_CriteriaApplied As Boolean
If ActiveSheet.FilterMode Then
    bn_CriteriaApplied = True
End If

EDIT...Note that FILTERMODE tests if a filter criteria is in place compared to AutoFilterMode which simply tells is if the sheet has a filter on it.

But also note that FILTERMODE does not tell you if the applied criteria filtered out any rows, to establish that you would need to use your counting method in addition to this test.

jim thomlinson say:

An XL4 macro command will greatly speed things up. Here is some code from John Green that I use.

Sub test()
Call PageSetupXL4M(CenterFoot:="Tada")
End Sub

Public Sub PageSetupXL4M( _
     Optional LeftHead As String, Optional CenterHead As String, _
     Optional RightHead As String, Optional LeftFoot As String, _
     Optional CenterFoot As String, Optional RightFoot As String, _
     Optional LeftMarginInches As String, Optional RightMarginInches As String, _
     Optional TopMarginInches As String, Optional BottomMarginInches As String, _
     Optional HeaderMarginInches As String, Optional FooterMarginInches As String, _
     Optional PrintHeadings As String, Optional PrintGridlines As String, _
     Optional PrintComments As String, Optional PrintQuality As String, _
     Optional CenterHorizontally As String, Optional CenterVertically As String, _
     Optional Orientation As String, Optional Draft As String, _
     Optional PaperSize As String, Optional FirstPageNumber As String, _
     Optional Order As String, Optional BlackAndWhite As String, _
     Optional Zoom As String)

  'based on a post by John Green in
  'microsoft.public.excel.programming
  'on 21 January 2001:
  'http://google.com/groups?selm=VA.00000b2f.0028c7e5%40mara9"
  Const c As String = ","
  Dim pgSetup As String
  Dim head As String
  Dim foot As String
  If LeftHead <> "" Then head = "&L" & LeftHead
  If CenterHead <> "" Then head = head & "&C" & CenterHead
  If RightHead <> "" Then head = head & "&R" & RightHead
  If Not head = "" Then head = """" & head & """"
  If LeftFoot <> "" Then foot = "&L" & LeftFoot
  If CenterFoot <> "" Then foot = foot & "&C" & CenterFoot
  If RightFoot <> "" Then foot = foot & "&R" & RightFoot
  If Not foot = "" Then foot = """" & foot & """"
    
  pgSetup = "PAGE.SETUP(" & head & c & foot & c & _
    LeftMarginInches & c & RightMarginInches & c & _
    TopMarginInches & c & BottomMarginInches & c & _
    PrintHeadings & c & PrintGridlines & c & _
    CenterHorizontally & c & CenterVertically & c & _
    Orientation & c & PaperSize & c & Zoom & c & _
    FirstPageNumber & c & Order & c & BlackAndWhite & c & _
    PrintQuality & c & HeaderMarginInches & c & _
    FooterMarginInches & c & PrintComments & c & Draft & ")"
  Application.ExecuteExcel4Macro pgSetup
End Sub
