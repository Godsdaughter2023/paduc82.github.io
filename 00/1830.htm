Vlookup is not working for text starting with "~" - Microsoft Community

karthik dale say:

Vlookup is not working for text starting with "~"

Hi All,

I have some data, in that some cells are starts with "~". Vlookup function is not working for text which starts with "~". Formula is working fine with all other text. What could be the issue ? Your help in this regard highly appreciated

Regards,

Karthik Dale

james o hunt say:

Hi,

I recreated your issue, and I was able to address it with an IF statement.

Let's say that the Lookup_value containing the tilde is contained in Cell A1,  and the Table_array is contained in Column B of the worksheet.  For the purposes of this example, let's assume that the text is Cell A1 is a follows: "~Something is rotten in the state of Denmark."

Given that the tilde is a wildcard, its presence causes the function to yield an error. In essence, the function thinks that it is looking up a string of text followed by "Something is rotten in the state of Denmark."

In order to correct this, a second "literal" tilde needs to be added. You can do so as follows:

=IF(LEFT(A1,1)="~",VLOOKUP("~"&A1,B:B,1,FALSE),VLOOKUP(A1,B:B,1,FALSE))

This function looks to see if the contents of the cell begin with a tilde. If so, the function adds an additional tilde to the beginning. If the cell does not begin with a tilde, the function uses a standard VLOOKUP function. I believe this should resolve your issue.

jay freedman say:
At each point where you need to go from a page without a border to a page with a border, or go from a page with a border to a page without a border, you must insert a Next Page Section Break (use the Breaks button on the Page Layout tab of the ribbon).

For example, you need a Next Page Section Break at the start of page 2 to separate it from page 1. You also need a section break at the start of page 3 to separate it from page 2. The next one will be at the start of page 8, and then one at the start of page 10, and so on.

When all of the needed section breaks are in place, you can insert the page borders. Put the cursor on page 2 and open the Page Borders dialog. At the lower right, open the Apply To dropdown and choose "This section". Select the border type and click OK. Move the cursor to page 8, and again apply a border to "This section". Repeat for each of the other sections that need borders.

Project Pro 2013 - Different tasks to show in different timeline views - Microsoft Community

gb_seattle say:

Project Pro 2013 - Different tasks to show in different timeline views - is it possible?

Is it possible to show different tasks in multiple timeline views? For example, I have tasks A, B, C, D and Timelines T1 and T2. Is it possible to show A and D in T1, and B & D in T2? If so, how?
dale howard [mvp] say:

GB_Seattle --

Yes, this is very possible.  What you need to do is to create additional Timeline views for each set of tasks about which you want to report.  To create a new Timeline view, complete the following steps:

  1. Click any View pick list menu (such as the Gantt Chart pick list menu on the Task ribbon) and select the More Views item.
  2. In the More Views dialog, click the New button.
  3. In the Define New View dialog, select the Single View option and click the OK button.
  4. In the View Definition dialog, enter a name for the new view in the Name field.
  5. Click the Screen pick list and select the Timeline item.
  6. Click the OK button and then click the Close button.

After creating the new Timeline view, next you must apply it by doing the following:

  1. Apply the Gantt Chart view if it is not already applied.
  2. Click the View tab to display the View ribbon.
  3. In the Split View section of the ribbon, select the Timeline checkbox.
  4. Click the Timeline pick list to the right of the checkbox and select your new Timeline view (it will be blank and ready for you to add tasks to it).

Using this technique, you can use any many Timeline views as you need.  BTW, a new feature in Microsoft Project 2016 allows you to create multiple timelines in the same Timeline view.  This feature is not available in the 2013 version of the software; thus, you need to create multiple Timeline views instead.  Hope this helps.
