            Next cell

For Each cell In ActiveSheet.Range("D2:D" & lastrow)
is causing this issue would be greatly appreciated!

hansv mvp say:

Since you're cutting rows, you should loop backwards (upwards):

 

    Dim r As Long
    For r = LastRow To 2 Step -1
        If Range("D" & r).Value <> "HHH" And Range("D" & r).Value <> "" Then
            Range("D" & r).EntireRow.Cut Sheets("Transfer").Range("A" & Rows.Count).End(xlUp).Offset(1)
        End If
    Next r

tom_bpc say:

Yes these numbers are all stored in the project model, but custom field formulas do not have access to them.  As Trevor highlighted, these formulas are practically limited to evaluating data across a row (i.e. within a particular task or resource record).  There are only four "Project-level" numbers that are available to custom field formulas, including total resources (count) and total tasks (count).

Escaping from these limitations requires vba - not that hard once you get over the initial hump.

Below I've pasted a little snippet to get you started.  It sets the Text3 and Text4 fields to the proper percentage of the overall project work and the immediate-parent summary task's work, respectively.  This is far from a general-usage tool and would fail under a number of circumstances (including any time your project is not sorted by default WBS order).  In addition I would imagine that the second column would need some additional work to target exactly which summary tasks get rolled up and which don't.  (Use a flag field in addition to the summary field to conditionally re-set "denom1".) 

The actual coding here is almost trivial (5-10 minutes), and it is much simpler than a corresponding custom field formula would be (if it were even possible).


Sub WorkPercent()

    Dim t As Task
    Dim denom As Double
    Dim denom1 As Double

    denom = ActiveProject.ProjectSummaryTask.Work
    denom1 = 0

    For Each t In ActiveProject.Tasks
        If Not t Is Nothing Then
            t.Text3 = Format(t.Work / denom, "0.00%")
            If t.Summary = True Then denom1 = t.Work
            If denom1 > 0 Then
                t.Text4 = Format(t.Work / denom1, "0.00%")
            Else
                t.Text4 = Format(0, "0.00%")
            End If
        End If
    Next t

End Sub

trevor rabey say:

The total work for civil is 64 hours, and this is calculated from the calculated by MSP sum of the work for the individual tasks under the summary "Civil", 40 + 24.

Formulae in spare fields in MSP can only calculate using fields across the row, and cannot be used to take data from another row.

In order to avoid using VBA if you can, you would need to use another spare field into which you can input 64 for each task under the civil summary.

Of course, you would have to manually input 64 into the custom field for each task, and manually update it if 40 and/or 64 should change. You could write a VBA macro to populate the field for each task with the summary work for the summary of each task.

However, ignoring that small difficulty, you can then easily make a formula to calculate this "weight factor" as a decimal or a % fraction of 64 for each of the tasks.

Outline level is a field so it could be inserted as a column, and from there, well, .... etc etc.

Someone has probably written the VBA for this and will post it here in response to your question.

However, I have to ask, what is the point? Why introduce this "weight factor" and then go to all this trouble to calculate it? What will it be used for, and how does it help?

And, BTW, you would be much better off using worker 01, worker 02, worker 03 etc so that each resource represents the smallest divisible unit of workers, ie 1 person. The way you have it, you will definitely have problems with levelling, obtaining a head count etc and will give up a degree of control of the resources assignments.
