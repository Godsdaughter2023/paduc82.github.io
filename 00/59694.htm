Hi Roger at CCCC,

 

·        Do you get the “safe mode” option when you boot the computer now?

One of the reasons for a disk read error may be due to physical connections. You may try the following and check if it helps.

 

Method 1:

Step 1: Check to see if the computer`s hard drive is detected under BIOS (Basic Input\Output System). You need to boot into BIOS (Basic Input\Output System) in order to make these changes. b)      Use the arrow keys to choose Safe Mode in the Windows Advanced Boot Menu Options and then press ENTER.

*EMBARRASSED*

A friend just identified what was causing the problem.

Apparently the ALT key to the right of the space bar was getting stuck. It was not visibly stuck down, but by pressing and holding the key or striking it hard it would "lock on" although the actual button came back up. 

While ALT is "engaged" (as if being pressed), if you use escape to escape out of whatever, it did not escape, instead it did what ALT-ESC does, which is switch between open applications much like ALT-TAB.

Likewise, if you try and type, nothing appears in any text window, instead the keyboard was trying to interpret ALT-(letter) combinations.

Problem solved...when it happens now I know to tap on the right ALT key until it releases. I will eventually try cleaning it to see if the problem goes away, if not I will just get a new keyboard.
