slance310 say:

Hi,

You may need to change your Bios settings to set the SSD as the 1st HDD Boot device... You may want to set the Flash Drive as the 1st actual boot device... Or select it using the "Boot Options" at startup... F9, F11, f12, ESC, or whatever your machine uses...

Try doing a "Custom Install"... You can create your own partitions and\or delete the ones you don't want, or all of them... Deleting everything would be the best choice..

Windows will create at least two partitions...

The OS Partition and a "Recovery" Partition...

If you have a machine running UEFI Bios it will create four partitions...

The 3rd will be a small "EFI" Partition of around 100mb that is formatted as "FAT32"...

There will be a fourth small partition... I don't remember what it is called, but it is normally hidden, and is 16mb-128mb in size... This partition is for the partition tables on a GPT type drive... This is what allows the drive to have more than the four partition maximum as on older machines...

You only normally see "WindowsBootManager" when using UEFI Bios...

That is because the boot files are on the "EFI" partition... Not the OS partition...

Your SSD would need to be a "GPT" type drive for UEFI Bios... Deleting everything on it and selecting the unallocated space for a Custom Install will cause Windows to do what is needed to complete the installation..

If in fact you are using UEFI Bios, your installation Media Flash Drive must be configured accordingly... It must be created as a "GPT" type drive, and formatted "FAT32"..

If yours were not correct, you would probably getting a installation failure mentioning the type of drive, so that is probably OK as is...

a. user say:

Most retail hard drives come with the software and the instructions to use it. Or you can find the software and instructions on the manufacturer's website.

Briefly: An image (also known as an image backup) is an exact copy of a hard drive, compressed into a single file. When you restore the image of the HDD to the SSD, you've transferred an exact copy of your computer to the SSD. Then, just replace the HDD with the SSD. It's like taking a dollar bill from your left pocket and putting it into your right pocket. Same dollar, different pocket.

If the manufacturer of your SSD doesn't offer the software, you can get it on the internet. But first, check the package from your SSD. If you don't see the software (probably on a disk) then check with the SSD manufacturer.

Metadata Cache Management, reports the use of the metadata caches files like SPP Metadata Cache that store information about the three types of metadata caches: objects, indexes, and databases.

bossdweeb say: If you are using the DirectX Web Installer it will install all 'missing' Direct X files -

Download details: DirectX End-User Runtime

If you have run the Web Installer and get a message to the effect that you have newer or
the same versions then you know they are all installed, but there may be a corrupt file that
is causing your problem.
If the error message specifies Direct X 11 which may be corrupted - not the same as "some directx3d"  - then you can delete the d3dx11_42.dll  and d3dx11_43.dll  files from System 32 and SysWOW64 and the DX Web Installer will replace them with new copies.
Note that the DX Web Installer does not overwrite existing DX files.
[btw - Assasins Creed 3 can use DX9, 10 or 11,  and Win7 has them all.]

If the error does not specify a specific Direct X file, or files then it is best  to use the full
Direct X Redistrubitable (June 2010) as it will overwrite the existing Direct X files (except for the couple of newer DX10 and 11 files that it does not have).

Download Details - Microsoft Download Center - DirectX Redist (June 2010)

Another option is to Open the game disk ,  find the DirectX installer on the disk and
run the setup from there.

dominicp say:

If the keyboard or mouse will not wake the machine press the power button to wake, not the reset button.

Check that you have the latest drivers for your keyboard and mouse. In Device Manager, check that the Power options for the keyboard and mouse allow wake and that the "Turn off to save power" option is not enabled.

 

The keyboard is still controlled by the BIOS at the boot selection menu.

Have you enabled the USB keyboard options in the BIOS? If so, a BIOS update is needed to resolve the problem.
