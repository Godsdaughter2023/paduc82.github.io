malkeleah say: You probably won't be able to do anything from your user account since you made that rather large mistake. You now need to log into the built-in Administrator account. This account is normally disabled in Vista.

1. First see if Administrator is available in Safe Mode. Restart the computer and as it is starting up repeatedly tap the F8 key. This will get you to the right menu to choose Safe Mode. You will use your arrow key to choose Safe Mode; the mouse doesn't work here. In Safe Mode, if you see an icon for Administrator on the Login Screen, click it to log on. Once in, do a System Restore to undo your mistake.

2. If you don't see the icon for Administrator as described above, you will need to use NTpasswd to enable the built-in Administrator.

Enabling this account will put an icon for it on the logon screen so you can then log on.

http://home.eunet.no/~pnordahl/ntpasswd/

Download the CD image (.iso) and burn as an image, not as data. You will need third-party burning software. Then boot with the CD you created and follow the directions. After you've finished with NTpasswd, remove the CD and reboot the computer. Log into the Administrator account you enabled and run System Restore. You will then want to go into a user account with administrative privileges and disable the built-in Administrator account for security purposes. Here's how to do this:

Start>Search box>type: cmd
When cmd appears in Results above, right-click and choose "Run as administrator". Supply the password/OK to the UAC prompt. Now you'll get the command prompt. Type:

net user administrator /active:no [enter]

Exit the command prompt.

After you have fixed your mistake, it would be wise to set up user accounts as described below. Also, if you want help with how to properly restrict your grandaughter, please post back with what you'd like to do so we can help you do it right.
You absolutely do not want to have only one user account. Like XP and all other modern operating systems, Vista is a multi-user operating system with built-in system accounts such as Administrator, Default, All Users, and Guest. These accounts should be left alone as they are part of the operating system structure.

You particularly don't want only one user account with administrative privileges on Vista because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult. It isn't impossible to activate the built-in Administrator to rescue things, but it will require third-party tools and working outside the operating system.

The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. After you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

If you want to go directly to the Desktop and skip the Welcome Screen with the icons of user accounts, you can do this:

Start Orb>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).

Windows Update error 80004002

If you receive Windows Update error 80004002, it means that some required registry keys are missing. To fix the problem, reinstall the Windows Update client by following these steps:

  1. Open System by clicking the Start button , right-clicking Computer, and then clicking Properties.

  2. Under System, note whether the computer's System type is a 32-bit or 64-bit version of the Windows operating system.

  3. Click one of the following links to reinstall the Windows Update client:

    • Click this link if you have a 32-bit operating system.  If you're prompted for an administrator password or confirmation, type the password or provide confirmation.

    • Click this link if you have a 64-bit operating system.  If you're prompted for an administrator password or confirmation, type the password or provide confirmation.

    Once you've finished reinstalling the Windows Update client, follow these steps to check for updates:

  4. Open Windows Update by clicking the Start button . In the search box, type Update, and then, in the list of results, click Windows Update.

  5. In the left pane, click Check for updates, and then wait while Windows looks for the latest updates for your computer.

  6. If any updates are found, click Install updates.  If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
