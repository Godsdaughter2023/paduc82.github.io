smittypro1 say:

When you delete rows with code, you need to go from the bottom up with Step -1. When you delete rows from the top down, then the row beneath the row you delete takes the place of the deleted row, so Excel skips over it. E.G. If you delete row 1, then row 2 is now row 1, but your code is looking at row 2, which has been replaced by row 3. It's a bit confusing, but that's why you're seeing it skipping rows.

Better than a loop would be to selet the range in question, add autofilter with Criteria1:="=", then Go to > Special > Visible cells only > Delete. Something like this:

Sub foo()
    Dim rng As Range
   
    Set rng = Range("Total_Cost_2")
   
    With rng
        .AutoFilter Field:=1, Criteria1:="="
            .Offset(1).SpecialCells(xlCellTypeVisible).EntireRow.Delete
        .AutoFilter
    End With
   
End Sub

Word Image and Table Resizing Macro

Prefacing my first post with what may turn out to be an obvious statement - I don't know anything about macros.  My firm uses software to document construction issues in the field that is essentially a big database tool.  It can export this information into a report using Mail Merge fields in a word document template, which is great.  The problem is, Word is very bad at controlling image sizes on the document creation.  So portrait, landscape, cropped, different quantity of image, etc. result in a pretty nasty end result.  I have figured out through the help of these forums how to write a macro that just resizes all the images to a fixed width - that's good enough, I'll take it.  However, if this ends up shrinking some of them, the table rows are not automatically shrinking themselves to fit the images.  I checked the document template and none of the rows have row heights set.  I tried changing the "at least" row height to be a very small number as well.  In either case, after the macro shrinks images, you're left with "fat" hanging off the bottom of some of the cells below the images.  

Here's the simple image resizing macro I minimally adapted from posts on this site:

Sub FixPictureWidths()

Dim oShp As Shape, oILShp As InlineShape
With ActiveDocument
    For Each oShp In .Shapes
        With oShp
            .LockAspectRatio = True
            .Width = InchesToPoints(3.25)
        End With
    Next
    
    For Each oILShp In .InlineShapes

        With oILShp

            .LockAspectRatio = True

            .Width = InchesToPoints(3.25)

        End With

    Next

    

End With

End Sub

Any help with the part to go back and resize/refit the row heights would be greatly appreciated.  Thanks in advance, looking forward to being educated!

macropod say:

I need to select all the tables in the document (including nested tables) and set row height to auto as well - that's the part I can't get worked out.

Try:

Sub Demo()
Application.ScreenUpdating = False
Dim Tbl As Table, iShp As InlineShape
For Each Tbl In ActiveDocument.Tables
  For Each iShp In Tbl.Range.InlineShapes
    With iShp
      .LockAspectRatio = True
      .Width = InchesToPoints(6)
      If .Height > InchesToPoints(4) Then .Height = InchesToPoints(4)

      .Range.Cells(1).HeightRule = wdRowHeightAuto
    End With
  Next
Next
Application.ScreenUpdating = True
End Sub
