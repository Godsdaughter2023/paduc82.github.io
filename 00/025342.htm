Hi,

From the help page on data validation: " Data validation is designed to show messages and prevent invalid entries only when users type data directly in a cell. When data is copied or filled, the messages do not appear. "

In fact, normal copy-paste overrides data validation settings (it pastes the data validation settings of the copied cell). 

A solution is to press F2 and then pasting.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim rngInterest As Range, rngCell As Range
    Dim boolUndo As Boolean

    Application.EnableEvents = False
    Set rngInterest = Intersect(Target, Range(c_dvrange))
    If Not rngInterest Is Nothing Then
        vDVType = rngInterest(1).Validation.Type
        If Not IsEmpty(vDVType) Then
            For Each rngCell In rngInterest
                If rngCell.Value <> "" Then
                    If Not rngCell.Validation.Value Then
                        boolUndo = True
                        Exit For
                    End If
                End If
            Next rngCell
        Else
            boolUndo = True
        End If
        If boolUndo Then
            MsgBox "Action Not Allowed - Reversed", vbCritical, "Computer Says No"
            Application.Undo
        End If
    End If
    Application.EnableEvents = True
End Sub

bill '-) say:

Well, where do we start...

Firstly it's not because of any fundamental superiority/inferiority of Windows Vs Android/iPhone from a user perspective, whatever anyone likes to pretend; if you can successfully use one, you can use either of the other two.

It's probably mostly about origin, timing, price and marketing.

Windows Phone OS effectively began earlier (around 2002 as Windows CE) and evolved in a series of steps starting from the PDA market (meaning lots of baggage), the other two did not. You can see this in the way Windows Mobile discarded and added features (and changed names) down the years. This has slowed down Microsoft and made it look sluggish and/or indecisive.

Most of the phone manufacturers who went down the windows path already had a family of more "traditional" mobile phones (and/or traditional PDAs) and moved stepwise towards what I'd describe as the "sweet spot" smartphone product spec (i.e. based on a touchscreen display of around 4 inches), the other two went there directly and a little earlier. Look at the complex evolution and diverse variations in the Nokia phone range and how it has lately slimmed down.

The Windows hardware/OS products were generally starting from an installed base at the higher end of the market and were often underpinned by a range of lower spec/lower price "entry products" which tended to keep pricing artificially higher, the others did not or were much less so. Apple, made a virtue of being high price/high end.

Marketing had to fit in with an extensive existing product range and thus was constrained (couldn't risk damage to little/big sibling products) and the message was diluted or confusing as a result, the other two had much better focus. Apple just had whatever was the current incarnation of the iPhone, so a very clear message.

I suppose one may/must speculate that Windows 10 might be Microsoft's last "big push" for Windows mobile.  Windows 10, unifying at least in marketing terms, desktop and mobile (in a multi manufacturer hardware environment) is probably a strong step in the right direction (and a Microsoft marketing USP), but time will tell and it's too early to call.

Do Microsoft have the determination and stamina to keep fighting on?  If they do, then I believe they will succeed, but they'll need to be prepared to spend a lot of time and money between now and then and it may be a long haul.
