¡firedog say:

if I then close down my Windows Mail and then reopen, the deleted items reappear. All that shows is a heading – Message cannot be found – and then text of the messsage – The contents of this message cannot be found. 

This sounds like a synchronizing account - one that tries to keep folder contents both in Windows Live Mail and at the server identical. If you're not sure what type of account it is, right-click on an account name in the folder pane and select

 

Who is the Email Service Provider (usually apparent from the part of the email address after the '@' symbol)?

If it is an IMAP account, then when you delete a message, it's flagged as deleted and (in your case)moved to the Deleted items folder. The account will then try and transmit this flag to the server to keep folder contents there up-to-date. Meanwhile, Windows Live Mail will create a log entry to reflect the change. Both of these actions take time - usually only a few seconds, but possibly longer if there are other things going on at the time.

It sounds as if you may be interrupting this process by closing down Windows Live Mail and then reopening. The log entry may not be written to the message database until after the program has been closed down, so you should always wait a few minutes after closing down for this housekeeping to finish before re-opening the program or shutting down Windows. The message list is constructed from the database index; when you select a message for viewing, the program will consult its index to find out where the message is located and then retrieve it for display.
