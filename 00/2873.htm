This copies the selected range to a new sheet, sorts it then puts it back in the correct columns

 

Sub sort_Areas()
Dim area As Range
Dim s As String, v As Variant
Source = ActiveSheet.Name
For Each area In Selection.Areas
    s = s & area.Cells(1).Address & ","
Next
v = Split(s, ",")
Selection.Copy
Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = "Tmp"
Sheets("tmp").Range("A1").PasteSpecial
Selection.Sort Key1:=Range("A1"), Order1:=xlDescending, Header:=xlGuess, _
        OrderCustom:=1, MatchCase:=False, Orientation:=xlTopToBottom, _
        DataOption1:=xlSortNormal

For x = 1 To Selection.Columns.Count
    Sheets("Tmp").Columns(x).Copy
    Sheets(Source).Range(v(x - 1)).PasteSpecial
Next
Application.DisplayAlerts = False
Sheets("Tmp").Delete
Application.DisplayAlerts = True
End Sub

lars-Åke aspelin say: If we can assume that the selected columns are never adjacent, i.e. each "area" is only one column wide, try these macros:

Sub sort_non_contigous_columns(ascending As Boolean)
   nrows = Selection.Areas(1).Rows.Count
   For Each a In Selection.Areas
     If a.Columns.Count <> 1 Or a.Rows.Count <> nrows Then
       MsgBox "Wrong input! All areas must be one column wide and of equal height."
       Exit Sub
     End If
   Next a
   Application.ScreenUpdating = False
    For i = 1 To nrows - 1
     For j = i + 1 To nrows
       If ascending Xor (Selection.Areas(1)(i) < Selection.Areas(1)(j)) Then
         For Each a In Selection.Areas
           tmp = a(i)
           a(i) = a(j)
           a(j) = tmp
         Next a
       End If
     Next j
   Next i
   Application.ScreenUpdating = True
 End Sub

Sub sort_ascending()
  sort_non_contigous_columns (True)
End Sub

Sub sort_descending()
  sort_non_contigous_columns (False)
End Sub

    Unable to add certain members to Distribution lists and Mail Enable - Microsoft Community

    nuno-tavares say:

    Unable to add certain members to Distribution lists and Mail Enable Security Groups using the Office 365 Admin Center

    If you constantly use Office 365 Admin Center to manage users and groups on your company, you have/would most likely encounter this behaviour.

    Certain objects cannot be found when using the search feature, even though we can see them in the drop-down list.

    After some tests, I confirmed that the search feature is case sensitive. The search applies to the “Display Name” as well as the “Email Address”.

    See the example below:

    Select any Distribution list and click on “Edit”

    Click on “Add Members”. Make sure the user is displayed under “All (…)”

    Search for the object in question using an upper case (e.g. “T”)

    Search for the object in question using a lower case (e.g. “t”)

    NOTE:
    If both the “Display Name” and the “Email Address” start with lower and upper case individually, then the search will work with whatever you decide to search (upper or lower).

    If both the “Display Name” and the “Email Address” only have one (either lower or upper), then the search will fail for whichever is missing.

    Hope this helps.

    Storing numbers as text without losing the format - Microsoft Community

    cassiefuhr say:

    Storing numbers as text without losing the format

    I need to store a combination of text and numbers as text format; however, Excel automatically uses scientific notation and adds the E and + sign when I format to text.  How do I prevent this?  The original data comes from an export and will be re imported as CSV.
    If I store the combination (example: ABC-12345-21345.1) as number I am concerned it will interfere with my import later on.  Thank you!

    joeu2004 say: Cassie wrote:
    I need to store a combination of text and numbers as text format; however, Excel automatically uses scientific notation and adds the E and + sign when I format to text.  How do I prevent this?  The original data comes from an export and will be re imported as CSV.
    If I store the combination (example: ABC-12345-21345.1) as number I am concerned it will interfere with my import later on.

    You are misinterpreting the problem.  The problem is not how Excel (or any application) writes the data to the CSV file.  The problem is how Excel, in particular, reads the data; that is, how Excel interprets it.

    Unfortunately, in general, we cannot change the way that Excel reads data in a CSV file.  Even we write the data surrounded with double-quotes in the CSV file, Excel will try to interpret numeric text as a number.  If we prefix the data with a single-quote (apostrophe) in the CSV file, Excel will not try to interpret the numeric text that follows.  But the apostrophe becomes part of the cell value, unlike what happens when we enter it manually.  (If we "re-enter" the cell value by pressing F2, then Enter, the apostrophe becomes hidden as usual.)

    If all of the text data is in a column, use Get External Data to import the CSV file instead of opening the CSV file directly.  In that last dialog box, the Import Text wizard allows you to assign the Text format to each column.

    [EDIT] PS:  You would have no problem with text of the form ABC-12345-21345.1 per se.  You would only have problems with text that Excel might interpret as numbers or dates, subject to the Regional and Language Options control panel setting on the receiving computer.  For example, the text '1E123, entered with a leading single-quote in Excel, then saved to a CSV file.
