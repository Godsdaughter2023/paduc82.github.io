john_barnett say:

Hello CM_979

 

Dump files are created when a problem occurs with your PC - usually when your PC unceremoniously crashes. They help Microsoft troubleshoot any problems. The only problem is the dump files can take up a whole lot of disk space (usually equivalent to the amount of system memory you have). If you have numerous dump files then it will seriously eat into your available disk space.

Using disk cleanup is one of the quickest ways of locating these dump files. I assume your PC is working okay? If so then yes, by all means, delete the dump files to reclaim your lost disk space.

Right click command prompt, and run as administrator. Then type in the following command, which will disable hibernation mode: 
 powercfg /h off

This worked for me, but doesn't seem to work over a whole night. (i.e. 8 hours or so) But it absolutely keeps the device able to wake from sleep for many, many hours, which basically fixes the issue. For day to day use, when you have to put the computer down for a while and then come back to it within a few hours, it wakes right up now with this fix.

It's a good workaround until they fix the issue, which they must be working on, based on their response.

Again, if you leave the laptop asleep overnight, it doesn't wake up the next morning, but if you're just leaving it for a few hours, it will now wake from sleep if hibernation mode has been disabled in the way I described above. On my HP Envy 360, this has "fixed" the issue pretty much.

bossdweeb say:

You didn't say which option you used, but -

Clear the tick box next to 'Automatically manage paging file size for all drives", and reboot your machine.

Or, If you are not using that option an want to clear a PF  previously set on one or more HDD's, 

first tick the box next to the drive letter with a PF you want to disable, then select the 

"No Page File" option - reboot.

If your OS is on an SSD and you have a HDD as well, disable the PF on the SSD,

and enable a PF on  the HDD . This will help prolong the life of the SSD, if the PF gets much work.

Take your pick of which option to choose. There are a couple of trains of thought on which is best.

- Enable the "Automatically manage....", or "System managed" size option, and reboot.

- Or, create a "Custom size". Using the same value for both min and max (was popular with XP).

 (Control Panel > System > Advanced System Settings > Performance Settings > Advanced >

Virtual memory Change )
