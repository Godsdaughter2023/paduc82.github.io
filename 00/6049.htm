aravinda balakrishna say:

Currently this is how '.oft' template open in Outlook by design.

 

However, registry can be tweaked to prevent the signature to be added automatically when opened the message template or custom form (oft - file).

 

Note: Backup the registry before you modify it.

 

Follow the steps below:

1.       Click start > In Run type ‘Regedit’ (Without the quotes) and hit OK.

2.       Navigate to ‘HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Outlook\Options’ key.

3.       Create a custom 'DWORD' by right clicking on the options and selecting New > DWORD (32) bit Value with the name ‘NoAutosigOnCustomForms’ with the Value type ‘REG_DWORD’ and with the value ‘1’.

Note: After setting this key upon opening an oft-file, no signature will be added. New messages, replies and forwards will still have a default signature as configured. If you have templates that still require a signature to be added, you can do this manually or choose to update the template with a signature included.

daniel g. grau say:
I don't have an answer for the slow typing problem, but I've found a workaround for the large PDF issue.

The three methods you mentioned for obtaining PDFs are equivalent.

Somehow, .docx documents printed/saved as PDFs include a subset of the Math Cambria font for every equation subgroup or unit (as selected with Shift + Right Arrow). So, a simple equation like:

                                           

will have five subsets of the font, while the PDF of this .docx document:

                   

contains 31 copies of Cambria Math font subsets. This behavior can be checked in Acrobat Pro through File > Properties > Fonts. Acrobat Pro is unable to remove the redundant copies, although "Reduce file size" yields a small improvement.

What you can do is this:
  1. Save As your final document as .doc. All equations will convert to low-resolution screen graphics. Furthermore, if they were originally centered, they may become left-aligned, so this is something you'll have to fix by hand or through a global replacement:
    1. In the "Find what" field, type "^g" (the escape sequence for graphics).
    2. In the "Replace with" field, specify Format > Paragraph > Alignment > Centered using the dropdown menu at the bottom.
  2. Convert to PDF using any method. The quality of the equations will reconstitute and the PDF will be much smaller.
In my test, the 5-equation .docx produced a 524KB file, reduced to 25KB through the .doc version.

By the way, re-saving the .doc as .docx does reconstitute the equations to Word 2011 format, but results once again in a bloated PDF.

Code sample that worked:

Sub Main()
Dim xl As Object 'Excel.Application
Dim wb As Object 'Excel.Workbook
Dim ws As Object 'Excel.Worksheet
Dim rng As Object 'Excel.Range
Dim cl As Object  'Excel.Range
Set xl = CreateObject("Excel.Application")
Set wb = xl.Workbooks.Open("C:\Users\Joel Madero\Documents\WordList.xlsx") '## Modify as needed
Set ws = wb.Sheets(1) '##Modify as needed
Set rng = ws.Range("A2:A3")

Dim blnBold As Boolean

For Each cl In rng
 
    Call searchAndReplace(cl.Value, cl.Offset(0, 1).Value, cl.Font.Bold, cl.Font.Italic)
    
Next

End Sub


Sub searchAndReplace(findText$, replaceText$, boldlnBold As Boolean, italicslnBold As Boolean)
'
'
    Selection.Find.ClearFormatting
    'Selection.Find.Font.Bold = blnBold
    Selection.Find.Replacement.ClearFormatting
   
    With Selection.Find
        .Text = findText
        .Replacement.Text = replaceText
        .Forward = True
        .Wrap = wdFindContinue
        .Format = True
        .Font.Bold = boldlnBold
        .Font.Italic = italicslnBold
        .MatchCase = True
        .MatchWholeWord = True
        .MatchWildcards = False
        .MatchSoundsLike = False
        .MatchAllWordForms = False
    End With
    Selection.Find.Execute Replace:=wdReplaceAll
    Selection.Find.Execute

End Sub
