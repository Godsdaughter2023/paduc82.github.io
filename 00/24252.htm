Blank cells aren't blank - Microsoft Community

terremelton say:

Blank cells aren't blank

When I sort all columns in a worksheet, columns that appear blank sort to the top.  I checked them with an if(isblank(), and it came back false.  They were created with a formla that said something like =if(c2=b2,"",c2) where it returned a true.  If this is the problem ("" doesn't create a blank cell), how can I create a blank cell in a formula?  What is in there when I use ""?
hansv mvp say:

The very fact that the cell contains a formula makes it "not blank". A zero-length string "" looks blank, but it is still a string.

The zero-length string will sort with other text values, i.e. after all numbers and dates, and before non-empty text values (if you sort in ascending order).

 

If you want to sort the empty strings at the end, add a column with formulas such as =D2="", and sort on this column, then on the column you originally sorted on. TRUE sorts after FALSE, so cells containing "" will be moved to the end.

john - project say:
Intern_Ashley,
When you say the summary portion of each project is dynamically linked from information below, exactly what kind of dynamic link are you using? If it is via paste links then you are in trouble. You should be using normal predecessors.

How about the dynamic links between the summary portion of each project and the master? Again, if those are paste links, you're in trouble. For links between projects there are basically two choices - external predecessors or a master/subproject structure. The former probably has less of a chance of developing problems but still isn't foolproof.

External linking in Project is not very robust. Paste links are the most prone to corruption and external predecessor, master/subproject, and resource pool/sharer structures are close behind. It can be done but requires a whole lot of very disciplined file setup and maintenance to avoid corruption. The most notable issues that lead to eventual linked structure corruption are: renaming files, moving files, working over a network and improper setup/breaking of the links themselves.

For a seasoned Project user (who has the discipline) I would suggest a master/subproject structure or external predecessors. That will give dynamic updates but still requires files to be opened for the update. A much safer approach would be to use a static master. It won't give dynamic updates but a new updated static master is easy and quick to build. Simply uncheck the "link to project" when you insert the subprojects into the master each reporting period.

If you are currently having problems with updates it could be that you already have file corruption or it may simply be technique. However to make the best impression I believe the static approach is your best bet. It may not seem as sophisticated but it is the smarter approach for someone trying to make an impression.

And thanks for a well worded clear post. You already made an impression on me.

John

Ashley,
If you used paste links to connect tasks in the lower part of a file with your summary portion, it should be fairly safe and easy to remove those and replace with normal predecessors. You will be able to see all the paste links within the file by going to Edit/Links. Select each one and then hit the "break link" button. Then use Save As back to the same file name to insure both ends of the link have been properly captured (i.e. broken). Once all links are broken you can set up the necessary predecessors to create your summary portion.

If some of those paste links are to the master file, then things get a bit more complicated but the process is basically the same. I would open the master and all the projects that are linked to it. In the master go to Edit/Links, select each link and break it. Yes, that is what I mean by static master. When you create a static master you create a whole new independent file so deleting unneeded tasks in the master will have no effect on those same tasks in the project from whence they came. And yes, you should be able to record a macro while creating the static master such that your team to re-create it more easily.

John

First code

   


 ' Access the GetUserNameA function in advapi32.dll and
     ' call the function GetUserName.
     Declare Function GetUserName Lib "advapi32.dll" Alias "GetUserNameA" _
     (ByVal lpBuffer As String, nSize As Long) As Long

     ' Main routine to Dimension variables, retrieve user name
     ' and display answer.
     Sub Get_User_Name()

     ' Dimension variables
     Dim lpBuff As String * 25
     Dim ret As Long, UserName As String

     ' Get the user name minus any trailing spaces found in the name.
     ret = GetUserName(lpBuff, 25)
     UserName = Left(lpBuff, InStr(lpBuff, Chr(0)) - 1)

     ' Display the User Name
    ' MsgBox UserName
     Range("D1") = "User " & UserName & " @ " & Time
     End Sub
