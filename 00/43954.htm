boarder2 say:

More than likely it is because the Surface Pro is a high DPI display which means you'll run at a higher scaling mode for things to be sized correctly.  When an external display is attached that is not high DPI, windows has to do some less-than-optimal scaling to make things look the same size across both displays.  Ultimately if all screens attached are LCDs and of differing DPI, one of them will look pretty poor.

Modern applications handle this scaling better and might look better, but older applications will look pretty bad.  It's just the way of things right now, unfortunately.

Your Lenovo and Dell laptops are probably not high DPI displays and therefore wouldn't have this issue.

If your display is FullHD (1920x1080) you may need to adjust the scaling settings on your device. The SP4 comes configured with 200% scaling by default, meaning that it will draw each pixel in a UI element as a 2x2 pixel on screen. You can configure that scaling per display by going to "Settings -> System -> Display", selecting the display where you need to adjust scaling and set the slider titled "Change the size of text, apps, and other items:" to 100%.

Don't adjust the settings for the SP4 otherwise items onscreen will become unusably small.

In addition you may want to configure the external display as your primary display when the device is connected. To do that in the same section of the Settings app, select the display you want to be the primary and check the checkbox that says "Make this my main display". That will mean that the start menu and Cortana will primarily located in the external display (although by default the start menu is also replicated on the secondary) and new applications you launch will go on that display by default.

Windows will remember these settings so that every time you plug the device to the dock it will go back to using the external display as your primary.

EDIT: Fixed now, after many hours of experimenting. Turns out it's the Wi-Fi. All you have to do is disable it in BIOS, and the installation will complete without any errors. Hit F2 when you turn on the computer, enter the BIOS Setup, find the Wi-Fi card (in mine it's under Advanced, and the line says "Integrated WLAN [Enabled]"), disable it, save changes and restart. Now, attempt Windows 8.1 install once again and it should work.

crappish say: Alright. Finally managed to get it working. The cause in my case seems to be not just ASUS wifi cards, but any that use Ralink chip, so if you're having similar problems, check for the Ralink chip. If you have laptop, you probably have to disable the thing from BIOS.

The solution for me was to remove the ASUS card, remove all existing Ralink drivers, put the card back in, boot up and install the device manually.

NOTE: You might be able to do this without physically removing the drivers, just by booting to safe mode, heading to Device manager -> unknown devices. If the card pops up in there, you might be able to install the ASUS drivers from there. However, as I finally got it working, I rather not test if that works as well. :P

Removing the drivers:
Open command prompt with admin rights, type "pnputil.exe -e". Write down all the Ralink driver filenames, go to C:\windows\inf and move all said files to temp folder (or delete, by moving you can still "roll back" if something goes awry). Then plug the card back in, boot up and go to Device manager -> Unknown devices. The wifi card should be there and just install ASUS provided drivers to said card - Don't use the ones provided from Windows Update.
