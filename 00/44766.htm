mouneshwar r say:

Hi Jam,

 

Is your computer up-to-date with the latest patches and updates?

 

This issue occurs when the Windows Reliability and Performance monitor fails to detect the installed anti-virus product.

 

In Windows Vista SP1/2008 Server/Windows 7, the WMI (Windows Management Instrumentation) Namespace "root\SecurityCenter" is deprecated, and programs are supposed to use the new Namespace "root\SecurityCenter2". However, the Windows Performance Monitor seems to use the old namespace and fails to detect the installed anti-virus product.

 

If your computer is not up-to-date with the latest patches and updates, then turn on windows update and check for any available updates.

 

Step 1: Install Windows updates in Windows 7

http://windows.microsoft.com/en-US/windows7/Install-Windows-updates

 

Step 2: Provide the Event log information from the event viewer for further troubleshooting

http://windows.microsoft.com/en-US/windows7/What-information-appears-in-event-logs-Event-Viewer

Search results (for Settings) in different language

Hey.

I was facing an MUI issue in Windows 10 Enterprise 2016 LTSB (Version 1607, Build 14393.2007) after installing the german language pack.

The issue seems to apear (after googling)  also on other editions of windows but mainly on 1607.

After installing i have set up the default language for Current User, New Users & Login Screen to the new de-DE language pack.

The UI language changed correctly.

The issue is that the Search results was still english than!

Example: Searching for "probl" (first chars for german word Problembehandlung) find english results:

Have Tried many sugestions from other Thread like: https://answers.microsoft.com/en-us/windows/forum/windows_10-win_cortana/search-results-in-different-language/c9f52147-7b2b-4953-b24e-e470226285e1

This was:

  • manual Rebuild the search index -> issue not solved
  • scf /scannow, DISM -> all with no errors -> issue not solved
  • De-/Reinstalling the de-DE language pack -> issue not solved
  • Change Language, Region settings back and forth -> issue not solved
  • ...and many other things...

Nothing solved the issue until i became an tip of my OEM partner, that this is a known issue and there is a workaround which FINALLY solved the issue for me. Here is the info:

Workaround:

    1. Run “mcbuilder” from an admin command prompt, give it about a minute or so, then restart explorer. Give it another 2-3 minutes.
    2. Navigate to %localappdata%\Packages\windows.immersivecontrolpanel_cw5n1h2txyewy\LocalState and verify it is not empty.
    3. Navigate to %localappdata%\Packages\Microsoft.Windows.Cortana_cw5n1h2txyewy\LocalState\DeviceSearchCache\ has SettingsCache.txt that is not empty.
    4. Try searching for settings from settings app and cortana.

Lösung von Microsoft:

Summary:

The initial issue happens because a scheduled task called LPRemove gets falsely executed once an additional MUI gets installed. Thus removes accidently also

important resources for Search to function correctly. Once this task has completed unfortunately there is no other option than to rebuild the MUI components by

using MCBuilder. The false setting for that task came due to an error in the Language Pack Setup executable manifest which created the same.

 

Resolution:

This issue does not happen with all versions greater than Windows 10 LTSB 2016. To prevent the task on LTSB 2016 there is an easy way in editing the LPRemove

task configuration.

  1. Navigate to “C:\Windows\System32\Tasks\Microsoft\Windows\MUI” and open the Security Properties of LPRemove file to take admin ownership.
  2. Edit the following part of that file with example Notepad

                <MaintenanceSettings>

               <Period>P3D</Period>

               <Deadline>P4D</Deadline>

                <Exclusive>true</Exclusive> ::: this needs to be set to false

false

  1. Save the file.
  2. Change the Security back to System ownership.

 

These steps should prevent that this task damages new MUI pack installation and also there will be no more need to run MCBuilder.

The Solution didnt solved the issue but the Workaround FINALLY does!

Hope this helps others facing this issue
