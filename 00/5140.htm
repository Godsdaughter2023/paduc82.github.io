dirk goldgar say:

OK, so now it's working, but the filtering is slow.  Then I requery the form, it's 1 sec max, when I filter the form via the toggles, it's roughly 5 sec.  Typical?

Depends on how many records are returned by the SP.  When you requery the form, Access calls the SP, SQL Server does all the query processing, and Access just takes the records it was given and displays them.  When you filter the form, Access can't (as currently written) go back to SQL Server to say "give me the records filtered by these criteria", because it doesn't know how those records were produced.  So all the filtering is done on the Access side, and since it can't an index, that means scanning through all the records in the recordset.

Now, 5 seconds seems like a long time, unless there are a lot of records to go through.  Roughly How many records does the unfiltered form display?

If this turns out to be the issue, then one workaround might be to add an optional parameter "DocumentStatus" to the stored procedure on the SQL Server side, and modify the SP so that, if that parameter is passed, it returns only records with that status, but if not, it returns all records.  Then you modify the Access code so that, when the form is initially opened, in its Open event, it sets the .SQL property of the pass-through query to execute the SP with no parameter, or with whatever default parameter value you want.  When you filter the form via the option group, you then modify the pass-through query to pass the appropriate DocumentStatus parameter, and requery the form.  There'd be no need to tell Access to filter the form, because you modifed its recordsource query and requeried it instead.

rich michaels - mvp (greatcirclelearning.com) say:
Yes it is quite possible.

The template would have a AutoNew and AutoOpen macro in it that executed as their name suggests. The would capture from the system clock the current second of time that it was; you'd thus have 61 possibilities of random numbers.

In a folder accessible to the template are 61 image files each with the same base name such as "Image" and ending with numbers 00 to 60. Based on the current second, the template would then insert the proper image into the background of the document it creates.

How to change the email address associated with a Bing Places for - Microsoft Community

elselotus654577 say:

How to change the email address associated with a Bing Places for Business Account, ie transfer the listing

I have been trying to figure this out, and it seems a lot of other people have had the same problem over the years, only to receive generic support replies linking them to pages that do not answer the question (and waste more of everyone's time in the process).

The simple answer is that you can't (at the time of this post).  But you can create (and then have to verify) a new listing.

I wanted to post this to hopefully help anyone else who comes along with the same issue.  I found this by clicking "Troubleshoot" on the dashboard page.  Under "Others" it has the option to select: "How can I transfer my listing to another account"

Link: https://www.bingplaces.com/TroubleShoot/TransferLocations?BusinessId=844424932041403

Here are the directions from that link:

Change Microsoft account or transfer listing

You cannot change the Microsoft account associated with your business listings. However you can create new listings using a new account and delete the listings from the old account:

  1. Sign in to Bing Places for Business with the new Microsoft account.
    • If you have not already created the new Microsoft account, you can do it here.
  2. Add your business as a new listing.
    • If you are a chain business (with multiple locations) or an agency managing listings on behalf of one or more clients, make sure to choose the appropriate option on the Bing Places home page.
  3. Complete verification for your new listing.
  4. Sign out of Bing Places and sing in using your old Microsoft account.
  5. Delete the old listings.

This is just silly.  They need to make a way to change an email address without scrapping the entire listing and having to re-verify.
