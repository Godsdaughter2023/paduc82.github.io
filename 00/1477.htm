But the reason I asked the question I asked is because when you log into the PC remotely via the Microsoft app the PC looks like it's switching to another user and that to me implies that maybe I can have different settings for each user type.

cmdrkeene say:

Nope, there's no other user account, it just looks like that because of licensing restrictions.  If you were to remote-desktop into a server, nothing would happen.  But the "individual user" versions have some code that makes sure that only 1 person can see the screen at a time, and so when you connect it looks like it's switching, really it's just locking the screen with a command like this:

rundll32.exe user32.dll, LockWorkStation

But yeah, the resolution for a remote connection is determined by the connecting client.  Basically the OS will do anything the app asks, because only the app knows how many pixels it needs (since there's no physical monitor being used, Windows won't know and won't store how many pixels the resolution should be for a remote connection).

pesach shelnitz say:

To add a grave accent mark (`) to a letter, such as e, press Ctrl+` (Ctrl together with the key to left of the 1 key) and then press e. The result is è.

To add an acute accent mark (´) to a letter, such as e, press Ctrl+' and then press e. The result is é.

For a complete list of the shortcut keys for adding accent marks, see the part near the end of the table inShortcut Key Reference on my website.

If the values had been formatted as General before, you might have noticed that the alphanumeric values were left aligned and the numbers were right aligned. hansv mvp say:

See if this works for you

Sub SvMe1()
    Dim strPath As String
    Dim newFile As String, fName As String
    strPath = "C:\Users\User\Documents\" ' base path
    With Worksheets("report1")
        Select Case .Range("AF22")
            Case Is <= 2
                strPath = strPath & "Folder_1\"
            Case Is <= 4
                strPath = strPath & "Folder_2\"
            Case Is <= 6
                strPath = strPath & "Folder_3\"
            Case Else
                MsgBox "Incorrect value in AF22"
                Exit Sub
        End Select
        fName = .Range("AF22") & " " & .Range("AE2") & " " & _
            .Range("M4") & " " & .Range("D8") & " " & _
            .Range("G8") & " " & .Range("J8") & " " & .Range("M8")
    End With
    newFile = strPath & fName & " " & Format$(Date, "dd-mm-yy") & ".xlsx"
    ActiveWorkbook.SaveAs Filename:=newFile, FileFormat:=51
    Sheets("report1").Select
    Range("B2").Select
    Save_Excel_As_PDF_1 'this is another module incorporated in the SvMe macro
End Sub
