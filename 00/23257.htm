with a result of

Is there a way to set up an equation that will allow me to subtract a finish time from a start time to get a total hours worked?

jeeped say:

Excel treats dates as 1 for every day past Dec 31, 1899. Time is a decimal portion of a day. Today happens to be 41,878 and today at noon would be 41,878.5. The 0.85 that you are looking for is 0.85 of an hour not a day. The difference between the two is 0.035 of a day.

If you want the proportion of an hour, multiply by 24 (e.g. the number of hours in a day).

0.035 × 24 = 0.84 (note 0.84 not 0.85)

  ... or,

0.0354166666666668 × 24 = 0.85

Addendum: alternately, you could use a custom time format of [hh]:mm:ss to produce a result of 00:51:00 which is the more accurate and preferred method of describing the difference between two times unless you have a specific need for something else.

Third,

(saved from an old post)

If you want to change to a folder on a mapped drive, you could save the current
location, and use ChDrive and ChDir.

But if you want to change to a folder on a network share that isn't mapped, you
can use a Windows API -- in fact, it works with mapped drives, too.

(Saved from an old post)

ChDrive and ChDir are limited to those mapped drives.

But Windows supplies an API that'll work for UNC paths as well as mapped
drives.  So you could use something like this instead:

Option Explicit
Private Declare Function SetCurrentDirectoryA Lib _
    "kernel32" (ByVal lpPathName As String) As Long
Sub ChDirNet(szPath As String)
