andreas killer say:
IMHO (7 - 5) / 0.4 is divisble. ;-)

But some more decimal places on the values shows the issue, see the code below.

There are 3 possibilities:

1) Calculate a MajorUnit that is divisible.
That could lead to a value with many decimal places.

2) Calculate a MaximumScale that is >= the High value and the difference to Low is divisble by Gap.
That could lead to a MaximumScale that is in the range High < High+Gap

3) Calculate MinimumScale and MaximumScale, so that the difference is divisble by Gap.
That could lead to a MinimumScale that is in the range Low-Gap < Low
That could lead to a MaximumScale that is in the range High > High+Gap

Andreas.

Option Explicit

#Const Version = 3

Sub Test()
  Const Low = 5.123
  Const High = 7.456
  Const Gap = 0.4

  Dim i As Integer
  Dim C As Chart

  Range("A1") = "Value"
  For i = 2 To 10
    Range("A" & i) = Low + Rnd * (High - Low)
  Next

  If ActiveSheet.ChartObjects.Count = 0 Then
    Set C = ActiveSheet.Shapes.AddChart.Chart
    C.ChartType = xlLineMarkers
    C.SetSourceData Source:=Range("A1:A10")
  Else
    Set C = ActiveSheet.ChartObjects(1).Chart
  End If

  With C.Axes(xlValue)
#If Version = 1 Then
    'Calculate the Gap value
    .MinimumScale = Low
    .MaximumScale = High
    .MajorUnit = (High - Low) / (Fix((High - Low) / Gap) + 1)
#End If
#If Version = 2 Then
    'Calculate the High value
    .MinimumScale = Low
    .MaximumScale = Low + (Fix((High - Low) / Gap) + 1) * Gap
    .MajorUnit = Gap
#End If
#If Version = 3 Then
    'Calculate the Low & High value
    .MinimumScale = Fix(Low / Gap) * Gap
    .MaximumScale = .MinimumScale + (Fix((High - .MinimumScale) / Gap) + 1) * Gap
    .MajorUnit = Gap
#End If
  End With
End Sub

    Any help or recommendations would be greatly appreciated.

    thanks,

    Jackie

    Moved from:  Office / Excel / Windows 10 / Office 2016

    bernie deitrick say:

    Copy the employee name data onto a new sheet, in column A. Copy the salary values into column B, both with headers in row 1.  Then in cell C2, array-enter (enter using Ctrl-Shift-Enter) the formula

    =IFERROR(IF(COUNTIF($A$1:A2,$A2)=1,INDEX($B:$B,SMALL(IF($A$2:$A$2000=$A2,ROW($A$2:$A$2000)),COLUMN(B$1))),""),"")

    If you have more than 2000 rows of data, increase those references before entering the formula.

    Then copy down to match your data, and copy column C to the right until all rows start to return blanks. (If you have, say, at most 20 records for any one employee, you will need to copy at least until column V.)

    Then copy all your formulas and paste values, sort based on column C, delete all rows where column C is blank, and you are done.
