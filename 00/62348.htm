wunders say: When you logon to your computer, there are a lot of programs, processes, and services that launch.  Chances are that one or more of these processes is contributing to your delay.  The first test would be to boot to a minimal system.  This is called "Safe Mode".  When you boot your computer, repeatedly tap the F8 key until a black and white screen comes up, then select "safe mode" or "safe mode with networking" with the arrow keys and start up.  If your computer starts up fairly quickly, then it is the non-critical items that are contributing.

To debug this, you use a program that will show you all the programs that begin at startup and allow you to temporarily turn off some of them to determine which one(s) is/are causing the problem.   Microsoft has a built-in program called "msconfig" or you can download another program called "autoruns" from Microsoft
     Autoruns:  < http://technet.microsoft.com/en-us/sysinternals/bb963902.aspx >

Instructions on how to startup and use msconfig are contained in this article:
   "How to troubleshoot configuration errors by using the System Configuration utility in Windows XP"
   < http://support.microsoft.com/kb/310560 >

Once you find what is causing your problem, the action is usually obvious.  Sometimes un-installing and reinstalling is all that you have to do.
