malkeleah say: Don't bother changing the built-in Administrator's name. Make yourself a new user account and copy the settings from Administrator to it.

Create a New User Profile in Windows XP Professional

   1. Log on as the Administrator or as a user with administrator credentials.
   2. Click Start, and then click Control Panel.
   3. Click User Accounts.
   4. Click the Advanced tab, and then click Advanced.
   5. In the left pane, click the Users folder.
   6. On the Action menu, click New User.
   7. Enter the appropriate user information, and then click Create.

You'll need to have three users on the system so create an extra one called "Tech" or "Extra' or the like (which you can delete later if you like) and the user account you're going to work from daily. Log into your new daily user account once so the profile files/folders are created and then log back out into "Tech". From there:

Copy Files to the New User Profile

   1. Log on as a user other than the user whose profile you are copying files to or from.
   2. In Windows Explorer, click Tools, click Folder Options, click the View tab, click Show hidden files and folders, click to clear the Hide protected operating system files check box, and then click OK.
   3. Locate the C:\Documents and Settings\Administrator folder.
   4. Press and hold down the CTRL key while you click each file and subfolder in this folder, except the following files:
          * Ntuser.dat
          * Ntuser.dat.log
          * Ntuser.ini
   5. On the Edit menu, click Copy.
   6. Locate the C:\Documents and Settings\New_Username folder, where New_Username is the name of your new daily user account.
   7. On the Edit menu, click Paste.
   8. Log off the computer, and then log on as the new user.

Note: You must import your e-mail messages and addresses to the new user profile before you delete the old profile. For additional information see 313055 OLEXP: Mail Folders, Address Book, and E-mail Messages Are Missing After You Upgrade to Microsoft Windows XP .

You're way overthinking this problem.

You're also a bit confused.  The printer drivers used in Windows 7 are user mode drivers.  The drivers are also "version 3" drivers (although Windows 7 labels them "type 3").  Version 4 drivers were introduced in Windows 8.  These are also user mode drivers.  See, e.g., https://blog.thinprint.com/the-new-microsoft-v4-printer-driver-model/  (which shows that Win 8 also uses "Type" rather than "Version").

AFAIK, Windows 7 does not support version 4 "class" drivers.  You can't do what you are attempting to do.

I'm a bit confused about what you did.  Your second screenshot, which shows the Add a printer Wizard, appears to be similar (but not identical) to the screen you get if you select "Have disk" and point the wizard to an *.inf file.  On the other hand, you said that you had downloaded the "full software and driver" package.

Did you try two different methods:  first ran the full package and got the error when the s/w told you to connect the printer and second, downloaded the "driver only" file (LJPro_MFP_M125-126_drv-only_15309.exe (15.1 MB) ) and tried using "Have disk"?

I'm also surprised that Windows Update couldn't find a driver.  There is one there:  http://www.catalog.update.microsoft.com/Search.aspx?q=LaserJet%20Pro%20M125 (the first one is the 64-bit driver and the second is 32-bit).  At a quick glance, it appears that these are identical to the "driver only" file mentioned in the preceding paragraph.

I asked about other printers because there's a "last ditch" method that resets the Windows 7 printing subsystem to its default state, but this will remove all of your printers and you probably don't want to go there just yet.  I'll give instructions for less intrusive methods first.

1.  Disconnect the printer from the computer

2.  Open Devices and Printers, right-click on the icon for the M125 printer, and select "Remove device."

3.  Click on any icon in the "Printers and Faxes" section.  Then click on the link for "Print server properties" that appears in the toolbar at the top of the window.  Select the "Drivers" tab.  If there is a driver listed for the M125, click the "Change driver settings" button (which may require a UAC response) and then "Remove" the driver.  When asked, remove both the driver and driver package.

4.  If you haven't already done so, download the "HP LaserJet Pro MFP M125/126 Series PCLm Driver Only (no Installer)" file.  On the main download page, this is one of the "basic drivers" downloads.  Run the downloaded file to extract the compressed files and make a note of the location of the extracted files.

5.  Run the "Add a printer" wizard.  Choose to add a local printer to an LPT1 port (you'll "remove" that printer after the installation, but the driver will remain installed).  When you get to the list of printer manufacturers/models, click "Have disk" and point the wizard to the folder where you had extracted the driver files in step 4.  Choose the hpcm125126.inf file and complete the wizard.

6.  Assuming that the install completed successfully, right-click and "Remove device" and then connect the printer's USB cable.  It should install using the already-installed driver.

If those steps didn't work, here's the next part.

Microsoft used to have an automated "Fixit" utility that dealt with printer problems.  This is described here:  https://support.microsoft.com/en-us/help/2793718/you-experience-print-spooler-error-messages-after-installing-or-upgrad  Note that there are two "modes."  I suspect that you'll need the "full" mode, but this uninstalls all printers from the computer and requires a reboot, so you may want to use the "light" mode first.

If you want to try the "light" mode first, use "light" rather than "full" in the command at step 6.  You don't need the restart at step 7.  I suggest that at step 8 you use the "basic driver with installer" package: LJPro_MFP_M125-126_basic_15309.exe (88.2 MB) because the scanner driver and other software will still be present.

If you use the "full" mode, I suggest uninstalling everything first, including any packages listed at Control Panel > Programs and Features as well as the scanner.  Reboot before following the instructions below. Then at step 8 use the "full software and drivers" package.

In its zeal to move everyone to Windows 10, Microsoft "retired" the Fixit utility described above (as you'll discover if you click on one of the links in the Microsoft article describing the Fixit).  Fortunately, the developers of the utility preserved the guts as a Powershell script that you can download and run.

  1. Disconnect the printer from the computer if connected by USB
  2. Go to https://github.com/jdickson289/Print-Reset-Tool/ and download print-reset.ps1 and make a note of where you save the file
  3. Click Start, type PowerShell into the Search box, right-click on the link to Windows PowerShell, select "Run as administrator" and respond as needed to the UAC prompt 
  4. Type the following command into the PowerShell window and press Enter
          Set-ExecutionPolicy Unrestricted
  5. In response to the "security risks" question, type Y and press Enter
  6. Type the following command into the PowerShell window and press Enter.  In typing the command, use the full path of the location where you saved the print-reset.ps1 file.  In the example shown below, print-reset.ps1 was saved in C:\Temp
          C:\Temp\print-reset.ps1 -full
    Type R to run the script
  7. After the script completes, restart the computer
  8. Download a fresh copy of the driver package. 
  9. Run the installation package and follow the on-screen connections to connect the printer
  10. Go to Devices and Printers and confirm (a) that the printer is installed and set as the Default Printer and (b) right-click on the icon, select Printer properties, and click "Print Test Page"

This is what the PowerShell window should look like as you proceed through steps 3 through 6 (I didn't enter R because I didn't want to reset my own printing system).  Don't forget to use the actual location where you saved the print-reset.ps1 file rather than C:\Temp as shown below.

Note that in step 6 if you omit the -full switch and run the script with no switches, you'll get a complete description of what the script does.  See the screenshot below.
