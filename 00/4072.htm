jay freedman say:

Word saves custom AutoText entries in templates, by default in Normal.dot (in 2003 and earlier) or Normal.dotm (in 2007 and later). Starting with Word 2007, the entries that come with Office are stored in a Building Blocks.dotx file. (More background information is at http://wordmvp.com/FAQs/Customization/AutoText.htm.)

What isn't often mentioned is that Word 2007 and later will load AutoText (or other building blocks) from any .dotx file stored in the same folder with Building Blocks.dotx. They can then be used directly, or moved to a different template by using the Building Blocks Organizer.

The first step is to get a copy of the Normal.dot template from Word 2003 and put it in some folder (such as the Documents folder -- but Mode]".

Click File > Info > Convert. Answer the prompt to convert the template to 2010 format. Then save the template, which will give it a .dotx extension. Close Word. (Note: the original Normal2003.dot will still exist, along with the new .dotx copy. You can delete it if you won't need it again.)

Now copy or move the Normal2003.dotx file to the same folder that contains the Building Blocks.dotx file. This is a hidden folder, which you can reach by pasting %appdata%\Microsoft\Document Building Blocks in the title bar of the Windows Explorer and pressing Enter. If you don't see the file there, you need to open the folder with your language identifier (such as 1033 for English US), and possibly within that folder open the folder named 14 (Word 2010 is also known as version 14).

Once the Normal2003.dotx template is in place, open Word 2010 again. Click Insert > Quick Parts > Building Block Organizer and examine the list of AutoText entries. You'll probably see a lot of duplicates, where one entry comes from Normal.dotm and the other comes from Normal2003.dotx. You can delete the duplicates from Normal2003.dotx, and keep only the ones that are unique to that template.
