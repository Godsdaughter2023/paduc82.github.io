john w. vinson say:

It would be good to find out how the change occurred first - this is NOT normal!

That said: copy the table to a new empty table by choosing the "structure only" checkbox. Change the field to Autonumber in the new empty table; you can ONLY do this if the table is empty. Then run an Append query appending all the data (including this integer field) into the new table. Remove all relationships from the old table, reestablish them in the new table, and then delete the old one. Compact and repair the database when you're done - it will have bloated by at least the size of this table and only compacting will recover the lost space.

ken sheridan say:

..................it does not come up with the next number in sequence.

Another factor is whether an unbroken sequence is important.  Usually the value of a key is irrelevant so long as it is distinct, and should never be exposed to the user.  If sequential numbering is required, however, then don't use an autonumber as you cannot assume it will maintain an unbroken sequence (all it needs is for a user to abort the insertion of a new record by pressing the Esc key), but compute the next number to be inserted when a row or rows are inserted.

a. user say:

Let me try to help. First off The error you are getting is commonly caused by one of two things.

1. The dll file is missing or corrupted. A call to locate the memory address (Using GetProcAddress or similar api) failed to locate the procedure at that ordinal (Nth index into a list of procedure stored in the library). In this case replacing the file may fix it however see #2.

2. You have a version mismatch of the specified file. This is actually much more common especially when moving to a newer (or older) version. The game expects there to be (at least) 43 procedures in the dll and that procedure 43 be a particular one. A new version of that dll may have that procedure removed, moved due to new procedures added or older ones removed, or altered in such a way that ORD 43 is now un locatable. The best bet in this case is to reinstall the game. that will ensure that the correct version of every file it uses is in place and that they are not corrupted. Fear not on gamesaves you have some options you may not be aware of.

First the obvious. Many games have game saves built in and you can simply back those up to preserve your gameplay. There may be an export option in the game options. If not then the game save (which it must have) is stored in a manner thats transparent to you. Typically this data is stored in a special hidden folder called AppData. You can access it with the following.

Press Win+R to open the run dialog. type %appdata% with the % signs in the run field and press enter. In the appdata folder you will see a number of other folders. They will typically have the name of various programs installed on your machine. For example you may have a WinRar folder in there if you have Winrar installed. Look around in there for a folder with your games name. Be careful as it may be named by the game companies name instead. If you find it then back that folder up PRIOR to reinstalling the game itself. Simply replace the folder after the reinstall.

If this doesnt work for you or you cant find a folder another trick you can do is install a second copy of the game. Perhaps in a Virtual Machine or on another hdd/partition. locate the missing/corrupted file in the new installation and copy it over to the old one. Dont forget to uninstall the second copy to save space. This method is a long shot and if you're not comfortable with it then the next thing would be to contact the game's developers. They could better diagnose and provide you will files you may need. Also ask them about methods for backing up your games in case this happens to you again.

Sorry if my post sounds a bit juvenile. I like to keep it simple as I dont know the extent of everyones computer knowledge. If you already know this then disregard what ive said.

Hope this helps you get back to gaming Good Luck.
