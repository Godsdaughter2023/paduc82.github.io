jay freedman say:
Yes, Microsoft made a mess of something that used to be very simple. The key now is that when you use Alt+F3 to create a new AutoText entry, you must pay very careful attention to the dropdown labeled "Save in". That's because the bubble (in MS jargon, a "tooltip") appears only in two special cases:
  • if the AutoText is saved in your Normal.dotm template
  • if the AutoText is saved in the template on which the current document is based (so the tooltip will appear when you reopen that document or base another document on the same template).

If you let Word save the AutoText in Building Blocks.dotx, or anywhere else, you get no tooltip. You can, if necessary, use the Building Block Organizer to move entries from one template to another by opening an entry's Properties dialog and changing the "Save in" setting.


If the AutoText is available at all (in Normal.dotm, in the document's base template, or in Building Blocks.dotx), then F3 will work provided that you've typed at least four characters of the entry's name and enough characters to be unique. For example, if you have entries named AutoFour and AutoFive, you'll have to type at least six characters of the name before pressing F3 will get you the response you want.


The business of entries disappearing when you exit Word points to a possible problem of Normal.dotm not saving properly. That could be due to damage to the file itself, or incorrect permissions on the folder (usually %appdata%\Microsoft\Templates) where it's stored. After you create an entry and you're sure you specified saving it in Normal.dotm, go to that folder and check the file's Modified date and time to see if it updated.

How do I increment the column references in a formula but a set number - Microsoft Community

clintfinch say:

How do I increment the column references in a formula but a set number of columns?

Hello,

I have a spreadsheet that tracks teams, projects assigned and their allocation to each project each sprint.  Projects are listed in rows.  Teams are listed in columns - 12 teams in all and each sprint is a total of 12 columns.  what I'd like to do is increment the following formula by 12 columns.

=IF(SUM(Details!C$5:C$65)=0,0,1)

So, when I copy this over to the next column, I'd like it to increment to =IF(SUM(Details!O$5:O$65)=0,0,1) etc...

I think I can do this w/offset but I don't know how to use that function well enough to get it to evaluate correctly.  Any help is greatly appreciated.

andreas killer say:
The distance from C to O are 12 columns, therefore you need a multiplier in OFFSET, Example:

=IF(SUM(OFFSET(Details!$C$5:$C$65,0,A1*12))=0,0,1)

If A1 = 0 then you access C5:C65
If A1 = 1 then you access O5:O65
etc.

Now let us eliminate the A1 term and I move the formula to D2
=IF(SUM(OFFSET(Details!$C$5:$C$65,0,(COLUMN()-4)*12))=0,0,1)

COLUMN() returns 4 in this case (because we are in the 4th column) and -4 = 0
If we copy that into column E we get 5 - 4 = 1

That's it.

1. Open Control Panel.

2. Choose on Modes.

3. Scroll down to E-mail notifications.

4. Under Mailbox change from "Mail for Exchange" to "None.

 

This removes email notification on Home screen and disables vibration :-)
