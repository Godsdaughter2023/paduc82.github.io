davidmcandy say:

You are making a common mistake in logic. You are assuming two things are related because they are close in time.

Defrag does not change the amount of Disk Space. It neither increases or decreases space used or free.

Windows Defrag runs every three days and optimises program and system startup loading. This is not controllable. This is also not a data file defrag.

Using another defragger will fight with Windows.

NTFS (and FAT32) are fragmentation resistant. Most fragmented files are log files and will fragment 1 second after defragging.

Windows only writes files where there is a lot of space to write preventing fragmentation. Files that don't change don't fragment. Disk caching makes fragmentation irrelevent.

All recommendation about free space are based on best guess on how an average person uses a computer. Lack of free space doesn't prevent the computer working till it's zero. However most people like saving their work and some Windows watchdog goes spastic just before you can't do anything.

As to why you observed a change in disk usage.

1.  System Restore - this could be it.

2. Disk Cleanup - it will run by itself is space is critically low

3. Log files are created, deleted, written to, etc

4. Windows repairs physical disk errors automatically. If defrag was having problems you may want to check there was no auto repairing of files.

This is how to see what Disk Operations may have occured.

Start - All Programs - Accessories - Right click Command Prompt and choose Run As Administrator. Type (or copy and paste by right clicking in the Command Prompt window and choosing Paste).
 
edit c:\bootex.log
 
Does this file exist with something in it?
 
The following commands query the event logs.
 
For boot time checking using source name for any Windows version (as it varies).
 
wmic /append:"%userprofile%\desktop\DiskEvents.html" PATH Win32_NTLogEvent where  (sourcename='Autocheck' or sourcename='Winlogon' or sourcename='WinInit') get /format:HForm
 
For running chkdsk within Windows.
 
wmic /append:"%userprofile%\desktop\DiskEvents.html" PATH Win32_NTLogEvent where (sourcename='Chkdsk') get /format:HForm
 
For warnings about disk problems detected during normal operations and automatic repairs made by Windows at the time the problems were discovered.
 
wmic /append:"%userprofile%\desktop\DiskEvents.html" PATH Win32_NTLogEvent where  (sourcename='NTFS' or Sourcename='Disk') get /format:HForm
 
Then to view the file created.
 
start "" "%userprofile%\desktop\DiskEvents.html"

lemp say:

The only thing I have to add at this point is that there are two aspects to moving to a larger hard drive.

One is the physical/mechanical part which involves opening the computer, removing the old hard drive, and installing the new one.  On a tower such as your computer, that's actually pretty easy.  The manual is HERE (see Serial ATA Hard Disk).

The other and more import part is copying everything -- including Windows, your application programs, and your data files -- from the old hard drive to the new one.  There are various methods to do this, and as Frederik Long suggested, the procedure for Windows 7 is not as straightforward as it used to be for Windows XP systems.

Read through the following linked pages and you can decide if you want to try to do things yourself or hire a professional. 

As Ken Blake suggested, you can buy hard drives on Amazon or from a reputable online vendor like Newegg.  You need a 3.5-inch SATA drive.  Your motherboard supports SATA-1 (1.5 GB/sec).  These days, you're more likely to find SATA-2 (3 GB/sec) or SATA-3 (6 GB/sec) drives, but the newer versions are backward compatible -- they will just be limited to the 1.5 GB/sec speed of your motherboard's connection.  However, if you decide to have an independent shop do the work, they'll probably want to sell you the drive in addition to charging for the labor.  Use the online shops as a price guide.

You can use a search engine such as Google or Bing to search for how to install a larger hard drive windows 7

but I suggest you read this page by J W Stuart, a knowledgeable and frequent contributor to these forums:  http://www.pagestart.com/win7bckuprstrnhd072610.html  Note that you'll need an intermediate backup location that's about the same size as your current hard drive, i.e., about 75-80 GB.

There's at least one catch:  because your hard drive includes the special HP Recovery partition, which is used to restore the computer to its "fresh from the factory" state, if you want to keep that partition you won't be able to expand your C: partition into the empty space without using another special program to move the HP Recovery partition to the "end" of the new disk.  Fortunately, there are several free programs available that will do this.  I haven't checked these out lately, but the first two are well-known and I've seen several online tutorials that recommend the last one:  http://www.thewindowsclub.com/free-partition-managers-windows
