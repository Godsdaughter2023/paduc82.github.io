Try working through this 3 step process: CHKDSK / DISM / SFC

******CHKDSK – CHECK DISK STATE **********
https://windowsinstructed.com/run-chkdsk-windows/

http://www.thewindowsclub.com/command-line-chec... (and 8 and 10) – There are a number of “paramters” you can pass to ChkDsk to ontrol wht it does.

There could be a small problem with your Window installation. The Check disk command cleans up small sections of the disk that may have been “orphaned” / corrupted when Windows is not shut down normally. The SFC command will check and repair Windows system files, then the DISM commands will repair any other minor problems with Windows.

In start menu type “PowerShell”
Right Click on the result and select “Run as Administrator”

run next command to schedule a check and fix the disk the next time the computer is restarted. At the end of the run it will display a status report.
CHKDSK C: /F

You will have to restart the computer for that check to work.

********DISM - DISM - Deployment Image Servicing and Management ************

After ChkDsk is done go back to the admin prompt and run following commands, one at a time to Repair The Installed Windows Image

this command checks whether the image has been flagged as corrupted by a failed process and whether the corruption can be repaired:

DISM /ONLINE /CLEANUP-IMAGE /CHECKHEALTH

this command scans the installed windows for component store corruption:

DISM /ONLINE /CLEANUP-IMAGE /SCANHEALTH

this command performs Window repair operations automatically:

DISM /ONLINE /CLEANUP-IMAGE /RESTOREHEALTH

DISM creates a log file (%windir%/Logs/CBS/CBS.log) that captures any issues that the tool found or fixed. %windir% is the folder in which Windows is installed. For example, the %windir% folder is C:\Windows.

https://docs.microsoft.com/en-us/windows-hardwa...

******* SFC – System File Checker **************

https://support.microsoft.com/en-us/help/929833... - Process shown with pictures

In start menu type “PowerShell”
Right Click on the result and select “Run as Administrator”

Type (or paste) the following command and press <enter> to Scan the integrity of all protected system files and replace incorrect versions with correct Microsoft versions.

SFC /SCANNOW

bigbenx88 say:

Hello again Metal.  :)

This has been an issue with some systems before, usually because there is an error in the system, usually caused from a bad install, and that bit of corrupted content can cause issues especially concerning power.  give these couple things a shot:

1)  Make sure we have the power set up correctly.  As I've mentioned before, we never want to plug the system into a power strip or surge protector.  A lack of adequate amperage can cause start up issues.  If you have it plugged into one of those, go ahead and unplug it from there and from the back of the Xbox and then plug it into a wall outlet first, and then back into the Xbox.  When you do this, watch the light on the PSU (the power brick) and if the light turns white when you plug it into the console for 5-10 secs, you should be good to go.  Turn it back on with the sensor on the console, then turn it off with the controller, give it a minute, and turn it back on with the controller.

2)  If you did the above, then don't worry about this because doing the above will do this for you automatically.  If not, then what we need to try is a hard reset.  Go up to the front of the console, hold your finger on the power sensor for 10 secs, and let it go.  Let it sit for about 10-15 secs, and then tap the sensor on the front to turn it back on.  When you power cycle it like this, it will run a maintenance on the system that will look for errors and corrupted content and attempt to fix them automatically as well as clear you system cache.

Hopefully one of these resolves your issue, if it doesn't then you know what to do.  :)
