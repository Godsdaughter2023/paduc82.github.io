I, too, was frustrated.  But I found a solution - and it isnt intuitive.

First, go to the directory LOTRO is installed to.  Search for .config.  You should see two files.  Edit each one, and where you see 'required version' and 'supported version', change the number to 4.0.  What this does is take care of your .Net 1.1 problem.

This step should let the launcher run.  Now - the problem I had was that the launcher would just hang and crash here.  Did the usual turn off firewall/virus protection, turn off ad blockers, renamed old user preferences to regenerate them, even a clean install.  Nothing seemed to work.  The issue is that you'll need to agree to two new EULAs, and for whatever reason, the updates that come with the new launcher are crashing the program.

The solution that worked for me:  in windows, create a new/different local account.  Log in to this new account and launch the game.  While I did initially get an error message here, the new launcher downloaded, the updates downloaded, I signed into my Turbine account as normal, and the game started.

Once you have done this, exit the game.  Log out of your 'new' Windows account, and log in to your machine as normal.  Head to Users/your account/documents/Lord of the Rings Online, and delete  your user preferences.  Launch the game.  You'll probably have to agree to the EULAs again, but it should launch just fine.

One last thing - if you get a black screen that doesnt go away when you run the game, its sometimes associated with DX11.  Choose DX9, and you should be off and running.

Technical information

Error 80240016 indicates that Windows Update cannot access the user token.

The user token contains the basic credentials that enable Windows Update to make changes to your computer on your behalf without requiring you to log in every time. This error can occur if you're running your computer for long periods without restarting and the token expires.

It might also occur when Automatic Update is running in the background and Windows Update is run in the foreground.


Starting and stopping the Automatic Update service will reset the service and acquire the token again
