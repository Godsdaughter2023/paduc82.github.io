Hi there, I think I know what's going on here.

The Bandwidth Meter displays your current bandwidth in megabits per second but what you are downloading is showing the speed at megabytes. These two data measurements are different.

bit is 1 bit

byte is 8 bits

This means that the speed what you are seeing in the bandwidth meter is 8 times larger than what you are seeing in your download.

I hope this helped and that this is what you were confused about. However, if you already knew this and I misunderstood you, sorry for wasting your time! :)

taidara say:

Bandwidth usage always takes measures of how much data (Kilobyte, Megabyte, Gigabyte etc.) you've actually downloaded, as opposed to download- and upload speed, where these are measured in bits per second (Kbit/s, Mbit/s, Gbit/s etc.). However, these two correspond with one and the other, as per Term Royal's previous post.

1. Start-->Control Panel-->System and Security-->System

2. Next click on the System protection link located in the upper left.

3. In the System Properties window click on the System Protection tab.

4. In the Protection Settings box click on the drive letter (usually C:).

5. Now click on the Configure button to change the Disk Space Usage Restore Settings.

6. Move the slider to increase the amount of disk space used.

7. Click Apply and then the OK button.

Hi Kayla,

Thank you for posting in Community.

We understand your concern and we will provide the assistance required to help you resolve the issue.

Before we proceed, I would like you to provide the information stated below:

Do you see the 100% memory usage in your Task Manager?

System and Compressed Memory in Windows 10, is  a new concept in the Memory Manager called a compression store, which is an in-memory collection of compressed pages. This means that when Memory Manager feels memory pressure, it will compress unused pages instead of writing them to disk. This reduces the amount of memory used per process, allowing Windows 10 to maintain more applications in physical memory at a time. This also helps provide better responsiveness across Windows 10. The compression store lives in the System process’s working set. Since the system process holds the store in memory, its working set grows larger exactly when memory is being made available for other processes. This is visible in Task Manager and the reason the System process appears to be consuming more memory than previous releases.

For now, I suggest that you open task manager and click on the Open Resource Monitor link  and check which process or service is causing the above high usage. I would also suggest that you refer to the article below to try and speed up your computers performance
