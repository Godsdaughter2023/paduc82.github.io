john w. vinson say:

Well... this can be done, but WHY would you want to!? Storing data in Field Names is bad design; storing data in Table Names is much worse! If you want to see data for a given week, simply store all the data in one table, and use a Query to select that week's

I'd really suggest you reconsider, or at least explain why you feel that this is necessary!

ken sheridan say: You would have to do one of two things, in each case calling the Date() function to name the table:

1.  Build and execute the SQL statement in code.
2.  Create and save a query, and change its SQL property in code before executing the query.

However, as John points out, this is bad relational database design.  All you need to do is to have a single table which includes a column of date/time data type which allows Nulls.  Having inserted data into other columns in new rows you can then simply update the table whenever you want to date-stamp the new rows with:

UPDATE [YourTableName]
SET [YourDateStampColumName] = DATE()
WHERE [YourDateStampColumName] IS NULL;

rohn007 say:
OK, then the answer to your question is there is no way of "deactivating" your old installation.

"Deactivation" is only required to comply with the license terms that you limit your number of concurrent usable installations.  A "dead" computer, and/or one that has been returned to the vendor/manufacturer meets this license term.

If the computer is still working, the requirement is to uninstall it. BUT, the uninstall is completely a local process. Unfortunately it does not contact the MS activation system at all.  This has changed with Office 365/2013, but ONLY with the 365 installation so it is not relevant to your question.

There is one more consideration. Did you buy a "Product Key Card" or "Retail" license? PKC licenses cannot be transferred to new computers, Retail licenses can (although initially Office 2013 did not allow this, MS agreed to change the license terms to allow it again).

What you will have to do is install on the new machine. Then do a Phone activation. When you phone in, don't answer the initial prompts. They will just give you the same "too many activations" type error.  When you talk to the service rep tell them you are transferring the installation from a dead computer. They should allow the activation to complete.
