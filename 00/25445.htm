Thank you for providing us the message header that was being asked.  Upon investigation, Error code 5.7.1 can occur if one or more of the following conditions is true:

  • The sender of the message does not have the privileges that are required to complete delivery.
  • You try to relay your mail by using a second server, and the second server does not let you relay mail. The remote server returns a 5.7.1 error code.
  • You do not have a recipient policy configured for the domain to which the message is sent.
  • The recipient has mailbox delivery restrictions enabled. For example, the recipient's mailbox delivery restriction is configured to receive mail only from a specified list. Other mail is rejected.
  • A distribution list is configured to restrict mail delivery to messages from authenticated users. Mail that is sent from an anonymous session is rejected.
  • Your Exchange computer is on an unsolicited commercial email list. Your Exchange computer may be listed as an open relay.
  • The DNS feature is configured incorrectly.
  • Anonymous access to the SMTP virtual servers is disabled.

Make sure that mail exchanger (MX) records point to the correct SMTP virtual server. If the DNS feature is configured incorrectly, incoming SMTP connection attempts may randomly connect to the wrong SMTP virtual server.

    ranmo_msft say:

    Hi Deepak,

    As directory synchronization is enabled and source of the authority is in on-premises, most of the attributes cannot be managed from Exchange Online and must be managed from on-premises. If you decommission the last Exchange server, you will lose the ability to edit the Exchange attributes of the objects from on-premises. Although you can use ADSIEdit to edit those attributes on premises and then sync to Office 365, it is not supported. It will put you in an unsupported scenario.

    Therefore, if DirSync is a requirement, then we strongly recommend you keep at least on Exchange server in on-premises.

    If anything is unclear, please feel free to post back and we glad to assist you.

    Ran
