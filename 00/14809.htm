davidf2 say:

When you File > Publish to the Web you have the option of saving the home page as index.htm or index.html. If you choose .htm then all the other pages of your website will also end in .htm and will be within the index_files folder. If you chose index.html then the other files will end with .html. Both are ok, but both are also a default name. If you have both an index.htm and an index.html file on your server then that could be what is creating the problem. If I try to see your home page then my browser may not know which file to open and then when I try to follow the links to your other pages you might have two different versions of each page.

Use FileZilla to log on to your host and find all the index.htm and index.html files and just delete them along with any index_files folders you find. Now start over and File > Publish to the Web and direct your web files to your local computer where you can find them. Decide on .htm or .html and be consistent from now on. Then upload your new index.htm file and your new index_files folder.

If your web host does not have good instructions on how to FTP your files using FileZilla (look under FAQs or Support section) then you can review the directions at:

http://wiki.filezilla-project.org/FileZilla_Client_Tutorial_(en)

http://wiki.filezilla-project.org/Using

And a couple more resources for using FileZilla:
http://help.godaddy.com/article/104
http://www.siteground.com/tutorials/ftp/filezilla.htm
http://www.apluskb.com/scripts/How_do_I_upload_with_answer3668.html

I would focus on learning how to use FileZilla as once you learn how to use this FTP tool it will be easier in the long run.
