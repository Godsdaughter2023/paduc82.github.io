Firstly, I’d like to confirm if you mean an Exchange 2010-based hybrid deployment. I suppose so because “#hybrid” is included in the tags under your post.

In an Exchange Online environment, only a licensed Exchange user can see and use a site mailbox. For your situation, please be aware you cannot just assign an Exchange Online license to a user who already owns an Exchange on-premises mailbox. By default, one user can only has one mailbox, either at on-premises or cloud side.

As to the shared mailbox, if you create it from the online side, on-premises Exchange users cannot see or use it. You can create a shared mailbox at Exchange on-premises and move it to online side, so users from both sides can see this shared mailbox and send emails to it. However, cross-premises mailbox permission is not supported, therefore, a user from one side cannot access or send as a shared mailbox which is located at the other side.

I ended up going a different route.

Instead of having access try to find the file(s), which might take a long time through my connection to the network drive and since all the folders and files follow a pre determined naming convention I made it so that Access can deduce the exact location and name of the file.

I then loop through a recordset that contains the exact map to the files and copies them to the location I selected.

This method I believe is more reliable and much faster than looking through folders and subfolders to maybe find what I want.

I did come across some issue where maybe the name is incorrect, so I added an If to the code to get things moving, next I want to add a print of the files it could not find.

In case this might help someone:

Code to select where to copy the files:

    Dim vFile As Variant
    Dim SQL As String
    Dim rs1 As DAO.Recordset
    Dim FSO As Scripting.FileSystemObject
       
        With Application.FileDialog(msoFileDialogFolderPicker) 'opens dialog box to select folder where to save
      
        .Title = "Select Folder to Export"
          .ButtonName = "Select Folder"
           If .Show = -1 Then
              vFile = .SelectedItems(1) & "\"
          Else
           MsgBox "Process Cancelled"
           Exit Sub
        End If
      End With

Code for the loop through the recordset:

Set rs1 = CurrentDb.OpenRecordset(SQL, dbOpenSnapshot)
    
     If Not rs1.BOF And Not rs1.EOF Then
   
    rs1.MoveFirst
   
    While (Not rs1.EOF)
   
    Set FSO = CreateObject("Scripting.FileSystemObject")
    If FSO.FileExists(rs1!Location) Then
    FSO.CopyFile rs1!Location, vFile
    Set FSO = Nothing
    rs1.MoveNext
          
    Else
   
    rs1.MoveNext
       
    End If
   
    Wend
   
    End If
