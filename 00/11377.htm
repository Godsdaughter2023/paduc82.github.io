What's tricky here is that the In and Out values are in different rows. I take it that you want to exclude the third row in your screenshot if 5147397 exists anywhere in the Out column in the table? Do you want to exclude the fourth row as well?

If so I think you need a "unmatched self join" query:

SELECT <whatever you want to see>
FROM mytable AS X LEFT JOIN mytable AS Y

ON X.[Trailer Number In] = Y.[Trailer Number Out]

WHERE Y.[Trailer Number Out] IS NULL;

This "frustrated outer join" query will attempt to match In with Out values, and display the record only if there is NOT a match.

If it's the latter, and the page for each record differs, You might try:

Sub ADiocese1()
Dim i As Long, strPages As String
With ActiveDocument.MailMerge
  .Destination = wdSendToNewDocument
  With .DataSource
    For i = 1 To recs_2_merge
      .FirstRecord = i
      .ActiveRecord = i
      .LastRecord = i
      strPages = strPages & "p" & .DataFields("PAGE") & "s" & i & ","
    Next
    .FirstRecord = 1
    .LastRecord = recs_2_merge
  End With
  .Execute Pause:=False
End With
strPages = Left(strPages, Len(strPages) - 1)

  .PrintOut Range:=wdPrintRangeOfPages, Pages:=strPages

End With
End Sub
