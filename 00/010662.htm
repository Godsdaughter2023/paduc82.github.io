    On Error GoTo Fallthrough

Desktop SSO is the process that's used to obtain seamless sign-in to Office 365 resources through AD FS from a domain-joined computer that's inside a company network. When desktop SSO fails, users may be unable to activate Office 365 ProPlus or users may be prompted for forms-based logon even though they are using domain joined-computers from inside the company network.

Modern authentication (ADAL) with AD FS requires the /adfs/services/trust/13/windowstransport endpoint to be enabled.

The token mentioned in the message you receive is about licensing token. Office 365 ProPlus contacts the Office Licensing Service on the Internet to obtain a licensing token for the user. To determine whether the user is licensed to use Office 365 ProPlus, the Office Licensing Service has to know the user's account for Office 365. In some cases, Office 365 ProPlus prompts the user to provide the information. For example, the user might see the Activate Office dialog box.

If the user is licensed for Office 365 ProPlus, a licensing token is stored on the computer in the user's profile folder, and Office 365 ProPlus is activated. Please see Overview of shared computer activation for Office 365 ProPlus  for detail information.

To check whether shared computer activation is enabled on the computer that has Office 365 ProPlus installed, open any Office 365 ProPlus program, such as Word. Go to File > Account > About Word. On the second line from the top, underneath the MSO version number, do you should see Shared Computer Activation or Product ID?

On the shared computer, after you open any Office 365 ProPlus program, go to the following folder:

%localappdata%\Microsoft\Office\16.0\Licensing

If activation succeeded, there are some text files in the folder.
