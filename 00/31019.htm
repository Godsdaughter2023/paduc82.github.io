ronnie vernon say:

I have a solid state drive C on my system that is now full, and the only folders are program files, users, program data, and windows.  My system will not let me move any of those folders to my D drive, where there is room.  How can I do that?

Hi

To free up disk space on the System Drive (C:) you can Move the physical user folders (such as Documents, Pictures, Music, Videos, etc) to a different drive.

1. Go to the target drive where you want to Relocate the folder and create a new folder with a similar name.

2. On the C: Drive navigate to the folder that you want to move. Right Click the folder and select Properties/Location Tab.

3. Click the Move button and select the new folder that you just created in the target drive and click the Select Folder button. Click the Apply button to move the folder. Click to confirm the move and click OK.

You can easily move any folder that has a Location Tab.

Let us know if you have more questions.

windows 8.1 update error 80092004 code about kb4054566 (NetFW 4.72) - the solution :

it is linked to a previous KB4340558 update which contains 3 kbs : 4338415 4338419 and 433424.

The 4338419 often (no always) fails using WinUpdate when looking at my error log.

THE solution is to uninstall the KB4340558, then install the KB4054566 before reinstalling the KB4340558 using DiSM instead of WinUpdate.

Here is the KB4340558 : https://www.catalog.update.microsoft.com/Search.aspx?q=KB4340558

and the KB4055466 : https://www.catalog.update.microsoft.com/Search.aspx?q=KB4054566

Create a folder (using windows explorer) C:\updatefix and the 4 subfolders C:\updatefix\expanded4xxxxxx (see the numbers below)

Download the whole KBs to C:\updatefix according your OS / 32-64 bit and rename them as below (mine is x64 so take care about the command lines to remove then add)

Extract each *.msu to the expanded file using the command prompt in Admin mode :

expand -f:* "C:\updatefix\kb4054566.msu" C:\updatefix\expanded4054566\
expand -f:* "C:\updatefix\kb4338415.msu" C:\updatefix\expanded4338415\
expand -f:* "C:\updatefix\kb4338419.msu" C:\updatefix\expanded4338419\
expand -f:* "C:\updatefix\kb4338424.msu" C:\updatefix\expanded4338424\

Remove the KB434558 (using command prompt) in this order :

dism /online /remove-package /packagepath:C:\updatefix\expanded4338424\Windows8.1-KB4338424-x64.cab
dism /online /remove-package /packagepath:C:\updatefix\expanded4338419\Windows8.1-KB4338419-x64.cab
dism /online /remove-package /packagepath:C:\updatefix\expanded4338415\Windows8.1-KB4338415-x64.cab

then the KB4054566 using :

dism /online /remove-package /packagepath:C:\updatefix\expanded4054566\Windows8.1-KB4054566-x64.cab

Now you will be able to install it without problem using :

dism /online /add-package /packagepath:C:\updatefix\expanded4054566\Windows8.1-KB4054566-x64.cab

Same way for the 3KBs of the KB4340558 :

dism /online /add-package /packagepath:C:\updatefix\expanded4338415\Windows8.1-KB4338415-x64.cab

dism /online /add-package /packagepath:C:\updatefix\expanded4338419\Windows8.1-KB4338419-x64.cab

dism /online /add-package /packagepath:C:\updatefix\expanded4338424\Windows8.1-KB4338424-x64.cab

You may be asked to reboot but i did all in one pass (say no at the end of each when asked), then I rebooted going back with WinUpdate to download the langage pack.

You won't see the results using the WinUpdate story but you will see them in installed updates.

If some have a better idea, share.

DiSM is better and allow to verify your system files.

October 15th Update :

KB4340558 (july) has been Updated by the KB4345592 (august) then by the KB4457920 (in september), so look at these first in your WinUpdate history, then in your error Log, in order to know exactly what to do next, meaning the right way.
