greginmich say:

CFA is a new Windows Defender anti-ransomware feature, and if you don’t want any ransomware protection, then you can just follow the steps in this article to turn off CFA:

https://support.microsoft.com/en-ph/help/4046851/windows-10-controlled-folder-access-windows-defender-security-center

However, that article is purporting to tell you how to:

“Add an app to the list of safe or allowed apps to prevent them from being blocked.” 

But in fact it's just providing the steps for turning off CFA; which will allow your friendly apps through to be sure, but right along with unfriendly apps, malware, ransomware, and data wipers – so it’s important to understand how logically distorted this article is. The Microsoft article that actually describes how to allow a blocked app through CFA is right here:

https://support.microsoft.com/en-us/help/4012987/windows-10-virus-threat-protection-windows-defender-security-center

When you see this Unauthorized Changes Blocked notification; follow these steps to allow the app through Controlled Folder Access:

1. Click on the notifications icon at the far right in the taskbar.

2. Write down the file path provided in the notification.

3. Click on the notification, and then on the Add button (+).

4. Navigate to the file identified in the notification, then select and “open” it.

The file path in the Unauthorized Changes Blocked notification tends to be truncated – although there’s usually enough of it there to locate the app without any trouble if you just assume that it’s in the Program Files or Program Files (x86) directory. If the file path is truncated to the point where you can’t locate the blocked app in the file-pickerdialog; then open Event Viewer; navigate to the Windows Defender Operationallog; and locate the blocking event (Event ID 1123):

1. Right-click on the Start button and select Event Viewer.

2. Navigate to Applications and Services > Microsoft > Windows > Windows Defender > Operational

3. Filter for (or just look for): Event ID 1123

More information here:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/windows-defender-is-blocking-too-much-saying-it/83a1d803-c687-4ddc-800b-5bb53c6151d1 

If CFA starts blocking apps when you haven’t enabled it, that might be the effect of a compatibility bug that seems to be activating CFA in the presence of third-party AV apps. In this case, the first thing to try would be disabling the third-party AV to see if this activates Windows Defender. If so, then toggle CFA on, and then back off again – and then enable the third-party AV in order to turn Defender back off.

Additionally, some third-party AV apps that are capable or running in tandem with Defender (in its real-time mode) might cause problems with CFA. For example, we’re not sure if the anti-exploit protection packaged with Malwarebytes Premium is compatible with Windows Defender’s CFA.

If you can’t turn on Defender by disabling your third-party AV app, then you can either attempt to troubleshoot the issue by removing the most common sources of Defender’s operational issues (recommended):

1. Remove any undetected malware by scanning with several third-party malware-removal apps, starting with Malwarebytes Free:

https://answers.microsoft.com/en-us/protect/forum/protect_other-protect_scanning-windows_other/list-of-malware-removal-tools/d824b9af-ebd8-4c47-94e2-8ee6c544c100

2. Remove any antimalware remnants by running the cleanup utilities for any preinstalled or previously installed AV apps:

https://answers.microsoft.com/en-us/protect/forum/protect_other-protect_start-windows_other/list-of-anti-malware-product-removal-tools/2bcb53f7-7ab4-4ef9-ab3a-6aebfa322f75

3. Run the standard Windows 10 system integrity checks:

https://answers.microsoft.com/en-us/windows/forum/windows_10-update/system-file-check-sfc-scan-and-repair-system-files/bc609315-da1f-4775-812c-695b60477a93

Or you can just proceed directly to uninstalling and then reinstalling the third-party AV app. Also note that these troubleshooting steps can be applied in cases where there isn't any third-party AV app installed and you're seeing erratic behavior with CFA:

And also be advised that reinstalling the third-party AV app is now optional, since Windows Defender is no longer playing “second fiddle” to any third-party AV app:

http://chart.av-comparatives.org/chart1.php?chart=chart2&year=2018&month=2&sort=0&zoom=2

Hi Derek,

Try replacing your function code with the following version:

'=========>>
Option Explicit

'--------->>
Public Function SheetNumSinceStart() As Integer
    Dim DaySheetNum As Integer
    Dim sStr As String
    Dim SH As Worksheet

    sStr = Application.Caller.Parent.Name
    Set SH = ActiveWorkbook.Sheets(sStr)
    DaySheetNum = SH.Index
    SheetNumSinceStart = DaySheetNum - 1
End Function
'<<=========

ken sheridan say: Your subform, in continuous forms view, should be based on your m2mLocationResource table, which will include two foreign keys, referencing the primary keys of the tblLocations and tblResources tables respectively.  The subform will be linked to the parent form on LocationID and will include a combo box bound to ResourceID, but set up to hide the BoundColumn and show the resource name.

You'll find an example as StudentCourses.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates how to represent a many-to-many relationship type with a variety of interfaces, but I'd strongly recommend the conventional form/subform.  Note how in the courses subform a new course not currently represented in the database can be inserted by typing the name of the course into  the combo box, which invokes the NotInList event procedure, whose code inserts a new row into the courses table.
