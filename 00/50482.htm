bossdweeb say:

The only way to change the amount of system RAM that is allocated to an integrated graphics processor

is via the machines BIOS. The amount that can be allocated varies between machines.


"At least to 512 Mb."

Your post shows - Total Available Graphics Memory: 1792 MB- . This is the maximum amount of

RAM that is allocated for gaming,or doing other graphics intensive tasks, and includes the 32MB VRAM

(assuming this 32MB is a VRAM chip and not a small portion of the system RAM 'dedicated' to video).

So you should still be OK with the current 'dedicated 32MB' as long as you don't do heavy multi-tasking.

It would take some serious multi-tasking to use 12GB RAM.

The main difference here is the 'dedicated' memory may be a VRAM memory chip on the motherboard,

vs using a portion of the system RAM (shared).

VRAM is much faster than system RAM, but there is no way to upgrade this, whether using integrated graphics

or a dedicated graphics card.

- The other issue is some games do not recognise shared system RAM as an alternative to VRAM.

You are out of luck in this instance.

kamikazze say:

Don't know why nobody still haven't found this yet, but solution for this issue is quite easy and obvious.

Home Group Listener service default configuration is incomplete, and missing services which it depends on.

So service tries to start too early and fail as not all conditions met for it to start properly.

To fix it you should do the following:

1) Start Registry Editor from Administrator.

2) Navigate to HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\HomeGroupListener

3) On right pane, right click Create -> Multi String (REG_MULTI_SZ)

4) Name this parameter "DependOnService"

5) Fill in the following lines in it:

RPCSS
RpcEptMapper
MpsSvc
HomeGroupProvider

6) Make sure no blank lines in parameter value, only filled strings.

7) Click OK to save parameter value.

8) Now make sure "Start" parameter is set to 3 (Manual), if not - set it to 3.

Close regedit and reboot. Check Events log and Services. Home Group Listener should be running right from start and error is gone.
