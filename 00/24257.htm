myall_blues say: Perhaps I am missing something in your description, but I have never seen formulas "auto adjust to the end cell" as you describe.

Perhaps what you are seeing is the adjustment that occurs when you paste in different rows or columns. For example, if your formula =SUM(AM9:AM14) is in AM15, and you then copy/paste it to AN16, it will become =SUM(AN10:AN15), i.e. the row and column both increment by one. However, note that as well as the end row changing, the start row has also changed.
When you copy to a different worksheet it will exhibit the same behaviour. so to the same relative cell the formula will remain the same, to a different row the formula will increment appropriately.

To some extent you can limit this by using the $ symbol in formulas to "lock" rows and columns. In general:

A1: Row and column will change when copy/pasted to different location
A$1: Column changes, row remains the same
$A1: Row changes
$A$1: Neither row nor column changes.

Printers require room to grip the paper. How much room then need depends on the specifications of the printer. graham mayor say:

The following macro will create any missing folders in the named path so you can call it from your macro that names the sheet e.g.

With Sheets(Sheets.Count)
       .Name = Cells(rw, 1).Value2
       CreateFolders "C:\Projects\" & Cells(rw, 1).Value2

       .Cells(1, 1) = Cells(rw, 2).Value

End With

Public Sub CreateFolders(strPath As String)
'A Graham Mayor/Greg Maxey AddIn Utility Macro
Dim oFSO As Object
Dim lngPathSep As Long
Dim lngPS As Long
  If Right(strPath, 1) <> "\" Then strPath = strPath & "\"
  lngPathSep = InStr(3, strPath, "\")
  If lngPathSep = 0 Then GoTo lbl_Exit
  Set oFSO = CreateObject("Scripting.FileSystemObject")
  Do
    lngPS = lngPathSep
    lngPathSep = InStr(lngPS + 1, strPath, "\")
    If lngPathSep = 0 Then Exit Do
    If Len(Dir(Left(strPath, lngPathSep), vbDirectory)) = 0 Then Exit Do
  Loop
  Do Until lngPathSep = 0
    If Not oFSO.FolderExists(Left(strPath, lngPathSep)) Then
      oFSO.CreateFolder Left(strPath, lngPathSep)
    End If
    lngPS = lngPathSep
    lngPathSep = InStr(lngPS + 1, strPath, "\")
  Loop
lbl_Exit:
  Set oFSO = Nothing
  Exit Sub
End Sub

Of course, however you will have to call the function twice to insert create two sub folders (not three times as the function creates all the missing folders in the path).
