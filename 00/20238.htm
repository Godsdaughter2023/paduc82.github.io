my Windows Defender quick scan only looks at a few files when it - Microsoft Community

jimakers say:

my Windows Defender quick scan only looks at a few files when it customarily scanned 86 thousand

Still having problems with Windows Defender .... Usually when I would run a quick scan if WD hadn't done it for a week automatically I would get up to 186,000 files scanned ...now when I do a manual Quick scan it only picks up 13,000 files and says I don't have a threat ... why is there that much of a discrepancy ?
rob koch say:
Still having problems with Windows Defender .... Usually when I would run a quick scan if WD hadn't done it for a week automatically I would get up to 186,000 files scanned ...now when I do a manual Quick scan it only picks up 13,000 files and says I don't have a threat ... why is there that much of a discrepancy ?


Though the thread answer by Quietman7 is very informative about the amount of time a scan might take it states absolutely nothing about the number of files which are scanned during each type of scan.

 

The actual reason for the different number of files during a Quick Scan is that during the first scan after Defender Real-time protection (or the PC itself) has been shut down, Defender will scan all of the files in the areas most commonly known to be infected by malware.

 

For any subsequent scan after the first until the PC or Defender Real-time protection are again turned off, Defender will only scan a subset of these files, usually about 10 to 20% of those scanned during the first scan.

 

The reason for this is that Defender maintains a cache index of all the files it has scanned already, so it only needs to scan a small number of regularly accessed critical files each time once the first scan has completed.  However, when the PC/Defender is turned off and then back on again, the state of this cache isn't known, so Defender must scan all of the files again to refresh the cache and insure none have become infected while Defender was shut down.

cbarnhorst say:

Hula_Baloo is right.  Ram is always a key in improving performance because it reduces the amount of disk activity in a 4GB system.  Disk activity is a major factor in performance.  What is happening is the system is constantly saving memory pages to the page file to free up memory for new requests.  Adding memory greatly reduces the system's reliance on the page file.  Memory is hundreds of times faster than an HDD.  Not having the system wait for the HDD to finish is a biggie.

My philosophy concerning ram is that if I turn on my computer and the lights don't dim, add more ram.  But I'm a bit geekish about it.

More memory is cheap these days.  However, in your case, since you are buying a new computer you may not want to spend money on the old system.

Obviously an SSD is far preferable to even a 7200 rpm HDD but you would be at $300 USD for that and the old system is not worth half that.  I use SSDs for all my system drives and HDDs for secondary storage and backup drives.  Now with the new NVMe drives and optane accelerators things have gotten very fast indeed.  But those are out of reach of older systems.

brochlass say:

WORD 2010 - Spacing between numbered list and a sub level

I have a list which needs to be a numbered list and has a sub level at the end of the last bullet.   When I click on the Numbering button and choose whichever style, it automatically numbers each item accordingly and the final item has a sub level which is fine.   However, I need a line space between each number and when I insert a line, it automatically creates a new number for a blank line (hope this makes sense). How can I fix this?  All I want is a numbered list with line spacing between each numbered bullet.

 

 

stefan blom say:
Don't press Enter twice to create vertical spacing. Instead, select the paragraphs and right-click; on the context menu, click Paragraph. In the Paragraph dialog box, specify the desired amount of Spacing Before (you'll find the option on the Indents and Spacing tab). Clear the "Don't add space between paragraphs of the same style" option if it is selected.

scottreed111 say: What if I want to create a list such that the line spacing at various levels is predetermined?  E.g., for Level 1, I might want 10pt/10pt (before/after), but at Level 2, I might want 0pt/6pt, and at Level 3, I might want 0pt/0pt.  

I know I can manually adjust for this spacing in every part of the list.  However, I'm wondering if there is an automatic way to pre-set these spacing requirements by level.  When creating a list, I usually just use the Tab and Shift-Tab keys to go back and forth between levels.  I'd like it so that using the Tab and Shift-Tab keys also achieves my pre-set spacing requirements as noted above.  Is this possible, or am I asking too much?

I'm also using Word 2010 on a Win7 Pro system

stefan blom say:

To each level of the list, assign a paragraph style with the desired spacing. For the general principles of numbering with styles (which is the recommend approach to multilevel numbering anyway), take a look at http://www.shaunakelly.com/word/numbering/numbering20072010.html.

After you have created numbering this way, you most likely have to (re-)apply it to each individual paragraph manually. Obviously, this may be time-consuming.

The upside is that numbering created with styles can easily be re-used in new documents, if you save the document as a template.
