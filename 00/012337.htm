I need to allow edits to a spreadsheet but don't allow to save - Microsoft Community

abigail.sanchez say:

I need to allow edits to a spreadsheet but don't allow to save

I'm using an Excel spreadsheet to allow users to search for information, for example - enter a name and the address, phone, etc information pops up in different fields.  I need to keep the data proteced and not allow users to save the document as an unprotected copy.  Is there any way to do that?

 

Thank you!

chip pearson say:

You can use the Workbook_BeforeSave event to cancel a save operation. Use code like the followning in the ThisWorkbook code module:

 

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, _
            Cancel As Boolean)
    If Worksheets("Sheet1").Range("Z1").Value = False Then
        Cancel = True
        Me.Saved = True
    Else
        Worksheets("Sheet1").Range("Z1").Value = False
    End If
End Sub

This has a "back door" so that you can save your work as you develop the workbook. Set cell Z1 on Sheet1 to TRUE to allow saving. Set Z1 to FALSE or empty to prevent saving. Keep in mind that, like all protection related measures in Excel, this code can be overridden by anyone with an intermediate knowledge of VBA and Excel.

Microsoft Word 2010- Protected Forms that users can fill in - Microsoft Community

melinda. say:

Microsoft Word 2010- Protected Forms that users can fill in

When I protect a form in Word 2010, is there a way to setup the text fields so that the users entering the data are able to use the bold, italic, underline font features for example, or change the font color so that they may highlight parts of the data they are entering?
jay freedman say:
With the requirement that everyone who wants to fill in the form must use Word 2007 or later (and not the Word Web App), this can be done.

Replace all of the form fields with the equivalent content controls (see http://gregmaxey.mvps.org/word_tip_pages/content_controls.html). In particular, replace the text fields where formatting is desired with rich-text content controls (not the plain-text content controls, whose button is the next one to the right on the Developer ribbon).

When that's done, do not use the "Filling in forms" type of protection. Instead, select the "No changes (Read only)" type. Then select each of the content controls (you can hold the Ctrl key and click one content control after another so they're all selected at once), and check the box for Everyone. Click the Start Protection button, and save the form. Any place in the form that wasn't selected when you clicked Everyone will be protected. The rich-text controls will accept formatting, inserting pictures, and almost anything you could put into a regular document.

First off... Tables, and table datasheets, are not intended for data presentation or interaction. They're for data STORAGE. You can use a Query to select data from one or more tables, and use the query's Order By feature to sort the data any way you like.
