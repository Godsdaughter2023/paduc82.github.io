stefan blom say:

If your goal is to have continuous page numbering through-out the document, visit the Page Number Format dialog box for each section (Insert | Page Number | Format Page Numbers) and select the "Continue from previous section" option.

Note that each time you delete a section break you are also deleting the related section formatting (page setup, newspaper columns, page setup), and that may certainly seem confusing. A section break stores the formatting for the preceding section.

Each Word section can have up to three different headers (and three different footers). This applies if you selected both "Different first page" and "Different odd and even" in the Page Setup dialog box.

For more on headers and footers, see http://wordfaqs.ssbarnhill.com/HeaderFooter.htm.

For more on sections, have a look at https://wordmvp.com/FAQs/Formatting/WorkWithSections.htm (note that the article isn't updated with information on the most recent versions of Word).

doug robbins - mvp office apps & services (word) say:

Almost certainly, you have some Continuous Section Breaks within the document.  In a document with such Section Breaks, via the user interface, it can be difficult to access particular footers and to set particular page numbering, it is sometimes easier to use a macro that will access the footer of each Section of the document.

FOR THE RECORD: Here's how your code would look if we left it using the same methodology of jumping back and forth between the two workbooks, with one change to prevent screen flickering.  You could replace the code you provided with all of it starting with the first Dim statement down to, but not including the End Sub

 

Sub ModOriginalCode()
'assumes Piedmont Natural Gas 3272 Coding.xlsx is active and
'that proper sheet with data is selected and other workbook is open
  Dim lastRow As Long
  Dim rowPtr As Long
  lastRow = Range("N" & Rows.Count).End(xlUp).Row

  'this will improve performance and reduce screen flicker
  Application.ScreenUpdating = False
  For rowPtr = 4 To lastRow
    Range("N" & rowPtr).Select
    Selection.Copy
    Windows("Copy of COA Cross Reference with macro_MASTER.xlsm").Activate
    Range("B9").Select
    ActiveSheet.Paste
    Windows("Piedmont Natural Gas 3272 Coding.xlsx").Activate
    Range("O" & rowPtr).Select

No, and there is not (really) a chance to accomplish that.

I've written a source many many years ago with the help of Michael Schwimmer, has tons of API code, but the end result is not really usable. The issue is not to "draw" the rectangle, the issue is to get the point (in chart coordinates) where you click inside the chart.

There are only 2 ways to zoom or shift the data in a chart:

a) Reduce the data that the chart shows

The most mentioned solution for that is formula based, using OFFSET inside the name manager, search the web for "excel chart zoom", there are also videos on youtube. IMHO a very cryptic solution.

The other solution is to use VBA to fill the real data into the chart, means don't use formulas. That is possible, you have to go that way when you want do create a "simulation", means a XY-Chart that shows a figure that moves.

b) Change the Axis scale values

When you change the MinimumScale and/or MaximumScale values of the axis in a chart leads to the result that it seem that the data in the chart zoom or shift.

But for a zoom feature, your chart must have 2 scalable axis, which is not the case of a line chart (in your case, you can scale the vertical axis, but not the horizontal axis which is based on categories).

If you like I can post some low level routines for that, give me a sign.

ceroberts75 say:
paul is right.

sounds like you are confusing the two.

the settings for the 5MP+34MP (MegaPixel) are in reference to how many pixels/inch are in the RESOLUTION to the image you are taking, not in direct reference to the size of the actual image in MB (MegaByte)

so your typical hi-res picture of the 34MP image, depending on the quality of the image and the amount of light (CZ Lense is awesome), will be somewhere between 8-12MB (MegaByte)

your 1020 takes two images at the same time with this setting.

it takes your hires image, which you only have access to view when tapping on the "open with nokia camera" or by connecting to your pc with a usb cable and pulling the image directly from your pictures folder.

it also takes a second, 5MP (MegaPixel) image that is an over-sampled duplicate of the larger image. though the detail is not as great, it is a great image and is there so that your image can be sent via email/sms/social media with a size-friendly/Data friendly way.

These regular 5MP images are typically between 1-6MB


If you are using a different setting for the JPEG (5MP) + DNG, then your file will be MUCH MUCH larger and only accessible for altering with connecting to a pc with usb and using the software that suggested.

by the sounds of it, I WOULD DEFINATELY RECOMEND YOU NOT USE THIS SETTING UNLESS YOU ARE GOING TO LEARN HOW TO HANDLE RAW FORMAT IMAGES.

tom ogilvy say:

Debbie2297, 

Do Alt+F11 to get to the Visual Basic Editor

in the project explore on your left, find the entry for ThisWorkbook for you project.   This is where workbook level event code is placed.  Right click on the thisworkbook entry and choose view code.

in the resulting module, there will be two dropdowns at the top.  In the left dropdown select  Workbook

in the right dropdown select  BeforeSave.  This should put a the beforesave event in the module:

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)

End Sub

put in code like this

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
With Worksheets("Sheet1")
  .Range("A1:M200").Sort Key1:=.Range("C1")
End With
End Sub

This sorts the range A1:M200  using column C.   If you don't need to change any of the settings, then just remove the Key1 argument so it looks like:

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
With Worksheets("Sheet1")
  .Range("A1:M200").Sort
End With
End Sub

If that doesn't work for you, under the developer tab you can turn on record macro and then manually sort your data.  Then turn the macro recorder off and see what the recorded code looks like.  The sort method I used is the old method because it is very simple.  You will probably record the new method which is more complex. 

By default, not providing any arguments to the old Sort method should cause it to inherit the existing settings since they are persistent.

In any case, change the sheet name name and range to sort to reflect the range in your workbook where you want this to occur.

--

Regards,

Tom Ogilvy

Why is it not automatically deleting archive to make room for current data?  If you are not able to do that, why is there not an option to clear archives?  Deleting items one page at a time is not an option.


Keep in mind that MyPhone was designed as a means to preserve your valuable data. The archive was set for several reasons; one being that id data was accidently removed from the device, it would be preserved unless (or until) you manually deleted it from the MyPhone web portal.  Having it automatically remove archived data would absolutely defeat it purpose.

The real issue IMHO is the amount of space available.  They are looking into providing more space.  As for me, I only sync Contacts, Calendar and tasks. 

As for deletion of items, once you select an entire page for deletion, an option appears to SELECT ALL enabling you to delete the entire section.
