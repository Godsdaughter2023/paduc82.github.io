It's going to be really hard to help when you don't supply any details at all.

To start, let's talk about how you save documents. In Word, when you open the Save dialog, look at the top just above the list of files. If you have Windows Vista or Windows 7, the path will appear there; the dropdown shows previously used folders. Look in the most recently used folder for your documents.

Hint: When you use the Save dialog, pay close attention to what folder is selected. If necessary, write it down!

To search for a file, click the Windows Start button and choose the command to open the file management program. In various versions of Windows that program will be called Windows Explorer or My Computer or Computer. In Windows XP there will be a Search button on the toolbar, and clicking it will open a view where you can type some word or phrase that you know appears in the document. In the other versions, there's a Search box in the upper right corner for the same purpose. In either case, you can start the search in a particular folder that you think is likely, or you can start at C:\ and search the whole disk.

Let's say that the Products table has a numeric primary key named Item (linking to Item in the Sales table), and a text field ItemName. You could use

ken sheridan say: I think you probably want something like this:

SELECT Product.Item, SUM(Sales.Quantity) AS TotalQuantitySold
FROM Sales INNER JOIN Product
ON Sales.Item=Product.ItemID
WHERE YEAR(Sales.Date)= 2014
GROUP BY Product.Item;

Where ItemID is the primary key of Product and its Item column is a non-key column of text data type.

It's this sort of confusion which makes many of us so hostile to the use of the 'lookup field' wizard when designing a table in MS Access.  For the gory details see:

http://www.mvps.org/access/lookupfields.htm

BTW I'd strongly recommend you change the name of the Date column to SaleDate.  As the name of a built in function Date is a 'reserved word' and should not be used as an object name.
