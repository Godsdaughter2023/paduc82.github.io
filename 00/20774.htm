dirk goldgar say:

One way to check if the query is a totals query is to open it in design view and look at the state of the Totals button on the ribbon (on the Design tab).  If the button appears to be "pushed in" or highlighted, then Access recognizes the query as a totals query.  Another way to tell from design view is to look to see if the "Total:" row in the field grid is showing.  If it is, it's a totals query.

Or you could look at the query in SQL view.  If it contains the keywords "GROUP BY", it's a totals query.

If the query is a totals query, it can never be updatable, because each row returned by the query may represent more than one record in the table.  So to answer your question, "how to resolve this?", I need to ask you why you want to update the Nest_Code field in the records returned by the query, and whether it is appropriate to be trying to do it on this particular form, which seems to be designed as some kind of summary form.  In order to update data, you would need to go to some other form, which is based on a non-totals query -- or else totally modify the current form to be something different from what it is.

hansv mvp say:

You don't need a section break for this, a page break is sufficient. You can specify that the first page of the document has a different header than the rest.. While in the header, tick the check box "Different First Page" in the Options group of the Design tab of the ribbon under Header & Footer Tools:

You have to do that only once.

Here is a macro you can run:

Sub SetHeaders()
    Dim s As String
    s = InputBox("Enter the header for the first page")
    ActiveDocument.Sections(1).Headers(wdHeaderFooterFirstPage).Range.Text = s
    s = InputBox("Enter the header for the second page")
    ActiveDocument.Sections(1).Headers(wdHeaderFooterPrimary).Range.Text = s
End Sub

hansv mvp say:

I would save each of the headers as an AutoText entry: Insert > Explore Quick Parts > AutoText > Save Selection to AutoText Gallery. Name them as (for example) Header A and Header B.

You can then use

Sub SetHeaders()
    ActiveDocument.AttachedTemplate.AutoTextEntries("Header A").Insert _
        Where:=ActiveDocument.Sections(1).Headers(wdHeaderFooterFirstPage).Range, _
        RichText:=True
    ActiveDocument.AttachedTemplate.AutoTextEntries("Header B").Insert _
        Where:=ActiveDocument.Sections(1).Headers(wdHeaderFooterPrimary).Range, _
        RichText:=True
End Sub

The insertion point does not need to be in the header to run the macro.
