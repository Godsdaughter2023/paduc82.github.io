lukechung say:

The LACCDB should appear on the front-end and back-end database. The LACCDB appears on the back-end database when you create a lock on it. 

A lock is created when your front-end database opens a table linked to a table on the back-end database. It can also occur if you create an explicit link to the back-end database. For performance reasons, your front-end database should have code that does this when it starts. That eliminates the repetitive creation and deletion of the lock file. rich michaels - mvp (greatcirclelearning.com) say:

Hi Adam,

 

As you probably are aware, the Format Painter is not fully implemented on the Mac in the same way as on the PC. And by that I mean the automatic next selection clicks of the mouse or as you have indicated the mouse button down and up, which would apply the formatting.

 

And you are correct that the Mouse Up event for Mouse Click does normally trigger the PasteFormat function.

 

However on the Mac you can use Cmd+Shift+C to copy the formatting (this executes the CopyFormat function) and follow it with a Cmd+Shift+V (this executes the PasteFormat function) to apply the formatting to a newly selected text somewhere else in the document. Not quite a slick as the Mouse only action but very workable.

 

As for a VBA solution you could replace the CopyFormat subroutine and turn it into a toggle like event utilizing the same shortcut keystroke for copying or the CopyFormat toolbar button. To use this code you need to declare a Public Boolean variable of cpyFormat.

 

Sub CopyFormat()

    If cpyFormat = False Then

        Selection.CopyFormat

        cpyFormat = True

    Else

        Selection.PasteFormat

        cpyFormat = False

    End If

End Sub


And a third alternative...

 

If you already have the Mouse events coded then instead of using a timer to activate I would suggest the Public Boolean variable method that executes the Selection.PasteFormat command if it encounters a True condition on the public variable.

 

Hope this helps
