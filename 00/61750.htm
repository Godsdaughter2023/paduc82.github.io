bill smithers say:

Remember that you can't create a Restore Point today for yesterday, last week, etc.

You create a restore point today to use at some time in the future.

And your system automatically creates restore points.

This should assist you.

https://www.eightforums.com/threads/restore-point-create-in-windows-8.4690/

@@@@@@@@@@@@@@@@@@@@@@@@

Also read through this information I put together:

"How to Do a System Restore in Windows 8 and 8.1"

http://www.eightforums.com/tutorials/4692-system-restore-how-do-windows-8-a.html

But it does not mean that Restore Points that you created 6 months ago will still be there.

It all depends on the amount of space allocated for System Restore.

New restore points are being created automatically/regularly and as that happens older ones are deleted to fit that space allocation.

If you find that you need earlier restore points as they are no longer there you will have to alter that space allocation to allow that in the future.

You can't alter the restore points that are available presently. What's there is there, what's gone are gone.

But remember that you use too much hard drive space if you wish to keep restore points back to the year "dot" for your machine and which you would have to keep increasing to accommodate them and it is not practical to do that as you would lose installed programs/app, Windows updates, computer settings, etc. when you do a SR onwards from that earlier restore point.

"How to Change Maximum Disk Space Used by System Protection and System Images in Windows 8 and 8.1"

http://www.eightforums.com/tutorials/4691-system-protection-change-maximum-disk-space-usage-windows-8-a.html

Cheers.

Problems with the Narration Audio Track

Another Bug?

Operating System:
Win 7
Movie Maker 2012

Try these steps:  Simple picture and audio test

1. Layout a series of sample pictures on your story board.

2. Add an audio file anywhere under picture #1 on the storyboard. This is not the "Add Music" option, but the "Add Sound" option in the drop down menu on "Record Narration".

3. Edit Pic #1 by changing the duration and notice that now the audio file only displays to the new duration of Pic #1.   And will play no further than the end of Pic #1.   If you Save, Exit, and re-open the project, the audio track then looks normal and plays normal.

4. But edit Pic #1 again and the same problem occurs. If you add a second audio file to the end of the first audio file that was shortened, followed by a save, exit, and re-open, the second audio file gets kicked to end of the now "normal" audio file #1. Or audio file #2 no longer displays on the story board if not enough pics to cover the overall storyboard time duration.

Next, try this:
1. Repeat steps 1 and 2 above.
2. Grab Pic #1 and move it to another location on the storyboard. Notice that the audio went with it.

It seems that there is an umbilical cord tied between Pic #1 and the audio file located beneath it.

One work around seems to be:
1. Move the audio file to Pic #2 location, or later on the storyboard.
2. Edit Pic #1. 
3. Then move the audio file back where you want it under Pic #1.

I have not seen any problems such as this with the "Add Music" option.

considerate_guy say:

I was easily able to reproduce the symptoms using your steps.


Here are some further findings from my tests:

  1. Even if the duration was lengthened, the associated audio was shown stopping at the end of the picture.  It did not continue to the end of the added audio.  And like your test, if I saved the project and reopened, then it showed the audio continuing.
  2. If I trimmed either the beginning or the end of the audio, even by 0.03 seconds, then when changing the photo duration, the audio was not truncated to the end of the picture.  It continued past the photo as before.
  3. The same symptoms happen if the audio is beneath video.  If the video is trimmed or split, then the audio is truncated to match the end of the video.  Again, saving and reopening, the audio goes beyond the end of the clip.
  4. Dragging a clip around, the associated audio moved with it.
  5. Trying these same steps with added music, I could not get the same symptoms.

So in summary:

  • If you add music (using the add music button), then it is not locked to the clip (photo or video) which it is below.  Dragging clips around will not affect the music.
  • If you add sound (using the narration dropdown and add sound), then it is locked to the clip (photo or video) which it is below.  Dragging the clip around, the sound moves with it.

I'm okay with that being the case, though it would be nice to document that.  What is not good is the following:

  • When a clip (photo or video) is trimmed in some way (change duration, set end point, split), the associated sound file is being truncated to the end, at least during that editing session.
  • Saving and reopening the project will reset the durations as expected (possibly moving other audio around, however).

I think this should be escalated to a technical specialist because this looks like a defect to me.
