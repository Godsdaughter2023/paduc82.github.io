Hi Callum,

If you have previously enabled SharePoint Server Publishing feature for the subsite, it is not feasible to create a template from the site. this is because the publishing feature creates site elements that are not supported as part of a template, and these elements remain even when the feature is disabled: https://support.microsoft.com/en-us/kb/2492356.

In this case, you may need to move the content of the site to a new site. You can use the following methods to manually move the data:

1. Use the OneDrive for Business sync app to back up files from the /portal site collection, and then upload them to the new site collection. For more information about how to do this, go to the following Microsoft website:
Sync OneDrive for Business or site libraries to your computer
 
2. Use the Open with Explorer command for each library, copy the files to a local location, and then use Open with Explorer to move the files to the new location. For more information about Open with Explorer, go to the following Microsoft Knowledge Base article:
2629108 How to use the "Open with Explorer" command and how to troubleshoot issues with this option in SharePoint Online

3. Manually download the files from the /portals site collection by using the web browser or by using a method from the previous options. Then, upload the files to the new site collection. For more information, go to the following Microsoft website:
Upload files to a library
