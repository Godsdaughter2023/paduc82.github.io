teylyn say:

Hello,

looks like you want to jump by four rows for the sum when the formula is dragged down. 

Normally, a formula is adjusted by one row, so you need a more complex formula to adjust the sum by 4 rows. Try

=B3+SUM(INDEX('Tab 2'!C:C,(ROW()*4)-9):INDEX('Tab 2'!C:C,(ROW()*4)-6))

Dragging this down will adjust B3 to B4 etc, and sum the data you indicated above.

Page numbering starting at zero is used when you add an envelope to a document, so it has its uses.

Run the following macro to ensure that page numbering is set to "Continue from previous section" for each section in the document:

Sub SetContinuousPageNumbering()
Dim s As Section
For Each s In ActiveDocument.Sections
s.Headers(wdHeaderFooterPrimary).PageNumbers _
    .RestartNumberingAtSection = False
Next s
End Sub

For assistance with installation, see http://www.gmayor.com/installing_macro.htm.

So, if an employee's wages for January are $2000, the SUTA is $27.80.  If the employee's wages for February are $3,500, the SUTA is $48.65.  If the employee's wages for March are $2,400, only $2,200 are subject to SUTA because the maximum annual wages subject to SUTA tax are $7700.  The SUTA tax on $2200 is $30.58.

joeu2004 say:

Although I believe the previous formula is correct, there might be a penny-off error in the sum of monthly SUTA amounts.

And more generally, that sum might exceed the statutory max total SUTA, which might be rounded arbitrarily (e.g. to the dollar).

The following demonstrates a more reliable calculation in row 8, to wit:

=MIN(ROUND(B6*$B$1,2), MAX(0, ROUND($B$3-SUM($A7:A7), 2)))

Note that $7700*1.39% is $107.03, but the statutory limit is $107.00, rounded to the dollar.

The formula in B3 is =ROUND(B1*B2,0)

Thus, the max SUTA in Mar is $30.55, not $30.58.

Of course, if you want to allow for $30.58 in Mar, then the statutory limit must be $107.03.

The formula in B3 would be =ROUND(B1*B2,2).
