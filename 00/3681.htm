Word does not let you select a specific color for a specific user. If you let Word assign colors (the "By author" option in the Advanced Track Changes dialog box), they will be randomly assigned. On the other hand, if you choose specific colors, they will be the same for all users.

Now, when a user goes to select a name from the list in the combo box, it will only display current employees. But on the older records, where a former employee entered his/her name in the combo box, I will still like those values to display. Is that possible?

The only way I can think to do it would be to make a 'fake' combo box which actually is a really short combo box next to a text box. The combo box filters to current employees, but the text box is not filtered & thus shows all values. But this seems cumbersome to do in 50+ different locations. Any other ideas?

Thanks in advance.
scottgem say:
If you change the RowSource of the Combobox to filter for only Current employees and you have combos on forms that will display non current employees, you will have an issue.

The way I handle this is in the Got Focus event of the combo. By default the RowSource is unfiltered. So when a user views a record, they see the employee assigned to that record. But in the Got Focus event I change the RowSource to a filtered SQL statement, so the user can only select from current employees.

Me.cboEmp.RowSource = " SELECT EmployeeID, Lastname & ", " & Firstname AS Fullname FROM tblEmployees WHERE Current = True ORDER BY Lastname, Firstname;

Complex math from 2 Tables in Word Document

I generate reports in Word with a combo of text and tables. I need to do a calculation with these variables:

1) weight in kg

2) a diameter in cm

Then I need to index the diameter to the weight using the formula DIAMETER/(WEIGHT^.294)

In my reports I have Table 1 at the top. In cell D5 is a value in lbs and in Cell E5 I have a formula that converts weight from lbs to kg (=PRODUCT(LEFT)*.453)

In Table 2 further down the report I have the diameter in D3, in D4 I want to have the result of the formula above, but I can't get it to work. I can get the formula to work within the same Table but not from one table to the other. I keep getting syntax errors, I've attempted to use bookmarks and still can't get it to work 

Any idea how I can get it work. Maybe word just can't do this but some how I want it to:

[Table 2 D3]/(Table1 E5)^.294) = Table D4 (Table 1 E5 is a calculated value using formula (=PRODUCT(LEFT)*.453), were the left cell is a value in pounds.

Is this possible? Maybe using bookmarks wrong? I'm getting tired of having todo  the math every time I need the indexed diameter.

Thanks

doug robbins - mvp office apps & services (word) say:

Select the first table (using Table Tools>Layout>Select>Table) and assign the bookmark Table1 to it.  Then in Cell D4 of Table 2, insert the following formula

{ =(D3/Sum(Table1 E5)^.294) }

You must use CTRL+F9 to insert the field delimiters { } and use ALT+F9 to toggle off their display

For the full bottle on Word field maths, see Macropod's Word Field Math Tutorial for which there is a zip file on the following page of Graham Mayor's website

http://www.gmayor.com/downloads.htm
