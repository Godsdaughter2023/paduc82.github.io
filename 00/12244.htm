palcouk say:

Usually means the mail server is unavailable, can be caused by a currupt outlook profile or a currupted win system (see below, NB Telnet is not enabled by default under win7)

 

The client connects to a port that is blocked or unavailable on the mail server. If you are attempting to send a message, the blocked port is 25 (SMTP), and if you are attempting to retrieve a message, the blocked port is 110 (POP3). Refer to the error message for the protocol and port numbers.

First check your Outlook settings, making sure that your POP3 and SMTP usernames, passwords and servers are correct. Also, make sure that your account and domain name are still active.
If the settings are correct and your account and domain name are active, one possibility is that you may not be connected to the Internet or the connection is bad. To test your connection follow the procedures below :

For incoming mail server (port 110)

    1. Click the "Start" button then "Run". Type "telnet <Incoming Server> 110" and press Enter. You will then be connected to the email server
    2. Type in "USER" followed by your email address (example: USER *** Email address is removed for privacy ***) and then press Enter. You should get a "+OK"
    3. Type in "PASS" followed by your password (example: PASS secretpass) and press Enter. If it is correct, you will get another "+OK"
    4. If you can do this, you have connected to the mail server, so the problem lies somewhere with your Outlook
    If you get "-ERR authorization failed" at step 4, it is possible that your password has somehow become corrupted. Try changing it in your account's control panel and try again.

For outgoing mail server (port 25)

    1. Click the "Start" button then "Run". Type "telnet <Outgoing Server> 25" and press Enter. You will then be connected to the email server
    2. If you see "220 mail.somedomain.com ESMTP", you have connected to the mail server, so the problem lies somewhere with your Outlook
    In the case you receive "connection refused" message or timeout, you need to contact your ISP with the request to open port 25 in their firewall (which they will probably not do). If they don't want to open the port you have to use their SMTP server for your outgoing mail connection. You can obtain their SMTP settings from either their tech support or possibly their web site.
    The following service providers are currently known to block SMTP connections and require your use of their SMTP server: 1st.net, ATT WorldNet, Earthlink, Lineone.net, MSN, Mindspring, Sympatico, Bellsouth.

    ken sheridan say: Regardless of the current issue you should split the database as Mimmo advises.  Sharing a single database file is a recipe for corruption.  And be sure to back up the back end very regularly, preferably to at least two secure locations.

    As regards the conditional formatting, use an expression where the date difference is 30 or more days as one condition, and where it is is 60 or more, as the second.  The 60+ condition should be the first as if this evaluates to True the second, 30+, condition will not be implemented.  Do this for all controls in the form's detail section.

    scottgem say:

    You need to understand better the concept of sharing an Access database. By default an Access database is designed for multi-user Access. The only way to change that is to open it Exclusively. And then it is exclusive only during that session.

    But this shouldn't be necessary. The recommended way to deploy an Access database in a multi-user environment is (as Mimmo and Ken suggest) as a split application with the a back end containing only the tables and a front end containing everything else. The back end only is placed on a shared network drive and the front end is installed locally for each user. 

    There are several advantages to this setup, but the advantage that deals with your question is you can now work on a development copy of the front end (preferably linked to a test copy of the back end) and make modifications to it. Once these modifications have been completed and tested, you can then deploy copies of the front end to each user.

    The only reason you shouldn't be able to make changes is if someone has the object open already. In such a case you should be getting a message that you you need to open the app exclusively to make changes. But if you are getting greyed out features on a copy that no one else has open, then there may be another issue.
