I'm using a split database.  Front ends are NOT .mde/.accde files.  My idea is that if I distributed a limited vesion of the front end for users with just input forms and then create another version with reports and things in it that only administrators might use, that this might be away of instilling a type of pseudo-user-level.  I'm obviously using the word "level" extremely loosly and I know that anyone (security speaking) who has access to the backend has access to all the data and can create their own reports and whatnot in the front ends they have--I'm not worried about it.

Your thoughts? 

~Dax

marshall barton say:

I have used a hokey way to deal with that kind of situation using only one front end file.  The funky trick was to name the front end one thing (e.g. prog.mdb) before copying it to regular user systems and something else before copyimg it to admin systems (e.g. admins.mdb).  Then any form that has a button or whatever that only admins should use, has code in the Open event to hide those features.  For eample. the code to hide a comand button could be::

   Me.cmdDoReport.Visible = (Right(CurrentDb.Name,10) = "admins.mdb"

This approach to your problem more or less requires that you hide the database window (or nav pane) and all built in menu/tool bars (or the ribbon) so users can only get to things such as reports through your forms..

suzanne s. barnhill say:

From the R300 manual (http://files.support.epson.com/pdf/r300__/r300__u1.pdf), it appears that there are color settings to select B&W (see p. 58). Also, in the Printing Preferences dialog shown on page 51, there's an Advanced button that may produce more options. There are also some forum discussions that suggest the the R300 normally prints B&W photos and other images that are already greyscale using composite black, which is not very satisfactory. I think the bottom line here is that this printer is really intended to be a photo printer rather than a "business" printer.

Did you try the recommended Compatibility Option setting?
