john korchok say:

Unfortunately, the information in that article is inaccurate. All modern fonts from major vendors are OpenType, including those distributed with Microsoft Office and Windows. Fonts with a .TTF ending are TrueType-flavor OpenType and fonts with an .OTF ending are PostScript-flavor OpenType.

You can verify this using the Font Book application in MacOS. Open a .TTF font and look at the Kind attribute in Font Book. It says OpenType TrueType. Now look at Kind in an .OTF font. It read OpenType PostScript.

Microsoft has dropped it's support for PostScript fonts and has buggy support for .OTF. There have been many reports on these fora of bad display of .OTF fonts, especially in cross-platform environments. Typical symptoms are bold italic displaying as bold, or italic, but not both, or bold displaying as bold italic. Often this weird display is evident only on one operating system and not on another.

This happens because Microsoft programmers have not be able to program the correct handling of an .OTF font family. The linked article papers over this issue by referring to .OTF as unsafe, which is BS.

Unfortunately, the only solution is to install a .TTF version of the font on all computers. You can probably obtain a .TTF version from the font vendor. Explain to them the Office display bug problem and they may have mercy on you and give you the alternate format for free.

VBA code for if then else

ossiemac say:

If you would prefer VBA code then the following. Right click the tab name on Sheet1 and select View Code. Copy the code below and paste into the VBA editor.

Note the comment re the Case Else which will blank Cell B1 if no matches.

The code assumes that the validation selection is in Cell A1 of Sheet1. If not, then change the following line to the correct cell address.

 If Target.Address = "$A$1" Then

You can add case statements with further ranges. Because I have used With Sheets("Sheet2") then ensure that you tie the range to the With statement by starting with a dot. The range for where value is entered in B1 has no dot because it is on the ActiveSheet (Which is Sheet1).

 

Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Address = "$A$1" Then
        With Sheets("Sheet2")
            Select Case Target.Value
                Case .Range("A1"), .Range("A3"), .Range("A5")
                     Range("B1") = 200  'Note this is Sheet1 (No dot preceding Range)
                Case .Range("A2"), .Range("A4"), .Range("A26")
                    Range("B1") = 400
                Case Else       'Not sure from question if this is required. If not, delete
                    Range("B1") = ""
            End Select
        End With
    End If
End Sub

 

Added with Edit after initial post.

It might be easier to simply create a 2 column table so that you use the first column for the data validation and in the 2nd column enter the required values for the selection and and then in B1 use Vlookup to find the value.

Access: generate a message box when 2 fields contain specific - Microsoft Community

uk4uk say:

Access: generate a message box when 2 fields contain specific entries

Hi - I've been trying to work out how to have Access generate a message box when 2 fields in a form contain specific values.

The first field is a combobox (CmbStatus) in which the user uses a dropdown to select the status of a project (e.g. in queue, active, project complete). The second field (AdminBilled) is also a dropdown in which the user indicates whether a project is billed (Yes) or not (No).

What I'd like to have happen is one of 2 things:

#1) when the user selects "Project complete" in CmbStatus, if the AdminBilled entry is "No" for a Message Box to pop up that says "Project has not been billed, please confirm"

Or even better: #2) Access will not let the user select "Project complete" in CmbStatus if the AdminBilled entry is "No". The Cmbstatus entry remains unchanged and a Message Box pops up that says "Project must be billed before marking complete"

I'd appreciate any ideas about how to do this. Thanks in advance.


 
ken sheridan say:

In the CmbStatus control's BeforeUpdate event procedure set the return value of the Cancel argument to True if the user selects 'Project Complete' and the value of AdminBilled is currently not 'Yes':

    Const MESSAGE_TEXT = "Project must be billed before marking complete."

    If Me.CmbStatus = "Project complete" And Nz(Me.AdminBilled,"No") = "No" Then
        MsgBox MESSAGE_TEXT, vbExclamation, "Invalid Operation"
        Cancel = True
    End If

You then need to cater for the corollary of this and prevent the user selecting 'No' in AdminBilled after selecting 'Project complete' in CmbStatus.  Again this is done in the AdminBilled control's BeforeUpdate event procedure:

    Const MESSAGE_TEXT = "Project has already been marked complete."

    If Me.AdminBilled = "No" And Me.CmbStatus = "Project complete" Then
        MsgBox MESSAGE_TEXT, vbExclamation, "Invalid Operation"
        Cancel = True
    End If

PS:  I've assumed that AdminBilled is a column of text data type rather than a Boolean (Yes/No) column.  If it's the latter test for the Boolean constants TRUE or FALSE rather than the string expressions 'Yes' or 'No'.
