Target: "C:\Program Files (x86)\Google\Chrome\Application\chrome.exe" http://suspiciouswebsite.com

Target: "C:\ProgramData\AmgSFAGMKD\TotallyNotAVirus.bat"

Start in: "C:\ProgramData\AmgSFAGMKD\"

If you do find something, quarantine/delete, fix your shortcuts, restart in safe mode, scan again, restart in regular mode, scan again, check your shortcuts again. If it's still not fixed you might need some specialized tools like HijackThis or SmitfraudFix.

suzanne s. barnhill say:

Yes. As you can see, there is a tab character (the arrow) before "Chapter One," and the only tab stop in the paragraph is the one at the right margin. That explains what's happening, but it doesn't explain how the tab character got there. If the Chapter One heading in your document is right-aligned using a tab instead of right paragraph alignment, that would partially explain the inclusion of the tab, but only if it was created by autonumbering. Ordinary tabs are eliminated in the TOC unless the TOC field includes the \w switch (see http://wordfaqs.ssbarnhill.com/TOCSwitches.htm).

It's possible the tab character was inserted inadvertently in the TOC itself. Have you updated the TOC (press F9) to see whether that changes anything?

suzanne s. barnhill say:

I see how this can happen. If you select a numbered heading style and then set the numbering to (none), a tab character remains. But I suspect what you did was format the numbers as Hidden so that you could still use them in the lower levels. Instead of doing that, you can create the Chapter One numbering in the Define New Multilevel List dialog. Type Chapter before the default 1, followed by a period. Then set the numbering style to One, Two, Three and "Follow number with" to Space. For the lower levels (in order to show 1 instead of One), check the box for "Legal style numbering."
