Any help is greatly appreciated

gord dibben say:

Getting late but this may get you started.

Add a Toggle Button from the Control Toolbox

Edit the Caption to read   "Show HR"  (no quotes)

Place this code in the worksheet module

Private Sub ToggleButton1_Click()
      If ToggleButton1.Caption = "Show HR" Then
          ActiveSheet.Rows("2:10").EntireRow.Hidden = False
          ToggleButton1.Caption = "Hide HR"
    Else
        If ToggleButton1.Caption = "Hide HR" Then
            ActiveSheet.Rows("2:10").EntireRow.Hidden = True
            ToggleButton1.Caption = "Show HR"
        End If
    End If
End Sub

Click button to toggle show/hide rows 2:10

You can amend the code to hide/show rows in which column B contains HR

The latter issue is a Windows 7 setting. Control Panel - Ease of Access Center - Make the mouse easier to use, enable "Prevent windows from being automatically arranged when moved to the edge of the screen".

For the former issue, you can programmatically control the position--i.e., by using an AutoOpen macro that repositions window to the desired location--but I don't think there is a built-in way to do it without resorting to VBA. The problem with this approach is that it would affect all documents, not just the ones that are on the left. It could test for the current position, however, and then move only the desired amount (or not at all). For that... you will need more VBA guidance than I can provide.

herb tyson say:

If you haven't created one, you won't have one. Conveniently, this is the correct forum for discussing macros! Have you used macros in the past? If not, there will be at least a little bit of learning that's needed.

Actually...  it turns out that the macro I was thinking of won't move your document if it's already at the right. So, this approach should work for you.

To record the macro, first right-click the status bar and ensure that Macro Recording is checked.

Open a document that isn't where you want it to be.

Click the macro recording tool on the status bar.

Name the macro AutoNew (you will need one for new documents and another for existing documents... once you create AutoNew, you can simply copy its "guts" to one you'll name AutoOpen). Click OK.

To record the steps, you have to use the keyboard--Word won't record mouse actions. Press Alt+Spacebar, S so you can size the windows as desired. Don't worry about the position of the window at this point. Use the cursor arrows to make the window the desired default size. Press Enter when the window is the desired size.

Press Alt+Spacebar, M. Use the cursor keys to move the window to the desired location. Press Enter when it's where you want it.

Click the macro recording tool on the status bar to turn the recorder off.

Press Ctrl+N (new document) a couple of times to verify that the correct size/location window is created.

In the Developer tab, click Macros, type AutoOpen as the name, and click Create. This will open the VBA editor.

Right above this macro, you should see the AutoNew macro. It will look something like this:

Sub AutoNew()
'
' AutoNew Macro
'
'
    Application.Move Left:=126, Top:=14
    Application.Resize Width:=901, Height:=589
    Application.Resize Width:=909, Height:=582
    Application.Move Left:=134, Top:=22
End Sub

Copy the four Application.Move lines into your AutoOpen macro.

If you've done everything right, when you open an document based on Normal.dotm, it should always open in the same location/size.

If prompted to save changes in Normal.dotm, say yes. Come back here if the macro doesn't continue to run after Word is closed/reopened. You might need to adjust your macro security settings to trust your own macros.

If you don't see the Developer tab, you'll need to enable it. Right-click the ribbon, and choose Customize the Ribbon.

Under Main tabs, place a check next to Developer, and click OK.

Hopefully... I didn't leave anything out. I guess we'll see.
