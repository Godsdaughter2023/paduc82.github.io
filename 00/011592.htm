Hi Luke,

The reason this happens because the Office installations in this machine is activate by the formal employee’s account. The solution to this would be removing the license key  of Office in this machine and activate it again with the logged in user’s account.

Using OSPP tool to remove the license key:
1. Open Command Prompt: Start > Typing Command Prompt > right click the Command Prompt and choose Run as administrator
2. Run this Command:

    C:\program files <x86>\Microsoft office\office16>cscript ospp.vbs /dstatus, 
3. Make note of value for “Last 5 characters of installed product key”

4. Then run this command with the key you get from last command:

    C:\program files <x86>\Microsoft office\office16>cscript ospp.vbs /unpkey: XXXXX(Key from last command)
5. Restart the computer.

Activate office with the logged in user’s account, refer to this:  Activate Office
