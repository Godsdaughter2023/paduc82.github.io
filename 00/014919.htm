tryx3 say:

Petie,

I have my Excel 2007 set up to have only one instance irrespective of the number of spreadsheets that are open.

To do this, you need to start  Excel differently. 
1  You cannot use the entry that its installation routine puts in the Start menu [or by selecting from its Jump list] because that starts a new instance.
2  Start Excel by opening the xlsx/m files themselves. 
3  To start Excel with a new blank file, create a shortcut to an Excel template file such as C:\Program Files (x86)\Microsoft Office\Office12\XLSTART\Book.xltm [you cannot pin such a shortcut to the Start menu because the Start menu was designed by people who hate users - I have these and all other shortcuts in a Taskbar toolbar of my own*].

To close all open workbooks in one go, click on the Office button then Exit Excel.  You can make this a bit more convenient to use by putting a macro [Application.Quit] on the Quick Access Toolbar.  When you want to close a single workbook without affecting the other open ones, click on the red x as you do now.

I noticed that you explained your aim was to share macros across all open files.  I moved to my current scheme for similar reasons, I have macros that involve counting open workbooks and the count cannot cope with multiple instances. 
Do please note that all the macros you write in C:\Program Files (x86)\Microsoft Office\Office12\XLSTART\Book.xltm are available to all xlsm files in Excel so you can achieve your aim even without going as far as I have [unless, of course, your macros also involve counting the number of open workbooks]. 
One other way to share macros is to put them in an Excel add-in xlam file [I use an xlam as a CustomUI that adds an additional tab to the Excel Ribbon with buttons for the macros that have been written in that xlam - this method has the added advantage of making the macros available to xlsx files as well as xlsm files. 
Another way to share macros is to use a "Personal macro workbook" but I found that extremely awkward in practice when I last tried it a decade ago and have not tried since.

* I have a folder called Menus that has all the shortcuts to files and applications grouped as I need them into subfolders .  For example, Menus\Office contains shortcuts to my Word and Excel templates, shortcuts to Office Picture manager and shortcuts to some individual Excel & Word documents.  The Windows XP Start menu could cope with having such folders visible; the Windows 7 Start menu cannot.  To activate the Menus, I right-click on the TaskBar then select Toolbars, New toolbar and point at my menus folder.  Menus behaves like an XP Start menu, subfolders fly out when hovered over.  See http://answers.microsoft.com/message/ccdb9b15-099b-4630-a854-89074a9b13a1?threadId=5c785b0f-3ec6-498f-b0dc-61f776c79928
