Looks to be a roaming profile issue. User path changes from user to user.domain messing up the pst default path. Added this registry fix into group policy to force the PST path to the %userprofile% directory. Created one for office 2013 also (change the 14.0 to 15.0)

Software\Microsoft\Office\14.0\Outlook\ (15.0 for office 2013)

Value = ForcePSTPath type = REG_SZ

Value data = %userprofile%\AppData\Local\Microsoft\Outlook

garthok01 say:

I recently had a similar issue, outlook 2013 on windows 10 attached to a domain. It turned out to be a bad path in the registry.

The registry key:

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders\

The Personal key was pointing to an obsolete network location. I changed it to the new location, rebooted and outlook was able to open the dialog to browse to the users PST files.

Revision Bars - How to retain for printing after "Accept - Microsoft Community

donldd say:

Revision Bars - How to retain for printing after "Accept Change"?

I wish to publish my .docx revision with Revision Bars (vertical lines in the left margin) still visible.  However, when I "Accept Change" the track changes markups disappear, including the "Changed lines" markup in the left margin.  How then to retain the "Changed lines" as Revision Bars for printing/publication?
suzanne s. barnhill say:
Aside from replicating the change bars with graphic lines, there's no way to do what you ask. There is, however, a way to display only the change bars for printing. In Track Changes Options, leave "Changed lines" enabled, but change the format of insertions to None and Deletions to Hidden. The text will then be displayed as it would be without showing markup except for the change bars. Note that this will be true only on your own system; if you send the document to someone who has different settings in effect, the recipient will see the document according to his own settings. But for printing (to paper or PDF), you will have achieved your object.

afwings say: Hey there Steve; Bob here...

Abhilash, one of the apps that's installed with MathType for Windows is the Equation Conversion Manager. To use this, you must first close Word, PowerPoint, and Outlook, if they're running. Next, go to the Windows Start menu and type Equation. You should see Equation Conversion Manager in the results. Click it. (If this doesn't work, navigate through All Programs > MathType 6 > Equation Conversion Manager. When the Equation Conversion Manager dialog opens, click on Microsoft Equation 3.0 and click Move>>, then click Close.

You should now be able to use both Microsoft Equation 3.0 and MathType in the same Word document or PowerPoint presentation.
