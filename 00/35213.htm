1)  If the discs are Windows Vista upgrade discs, you would need to reinstall Windows XP then upgrade to Windows Vista.  When filling out the registration, you may use any username/password you choose.

Start by exporting your current BCD settings for backup.

  1. Boot into Windows Vista
  2. Log in with an administrator account
  3. Open up an elevated command prompt
  4. Type bcdedit /export c:\bcdbackup and press

This will create a file named bcdbackup on your C disk. Note that there is no file extension in the file name. No file extension is needed here. You can change this location or the name of the file if you like, but having it stored on the root of C will make it easy to find if needed.

This will return a list of all the operating systems that the computer can boot from.

  • Enter bcdedit and press
  • Enter bcdedit /v and press By using the V switch the list will give you the UUIDs

You need to use the V switch to see the UUIDs for the different operating systems.

Do you see any entry for Windows.old on the right side of the description headline? If so, then you need to delete it. And to delete it, you need to know its identifier.

The identifier is the UUID number you see on the right side of the identifier headline, including the braces. Braces are the curly brackets at the start and the end of the number.

  1. If you have not done so, type bcdedit /v and press
  2. Locate the identifier UUID for the entry that has "Windows.old" as description.
  3. To delete the entry type bcdedit /delete {UUID} and press .

Depending on how many operating systems you have installed, this could take anywhere from 4 minutes to 15 minutes. Just let it do its thing, you will see a confirmation message when it's done.

When you see the confirmation message, type bcdedit /v again and press . Make sure that there is no entry for Windows.old.

Warning! Make sure you type in the correct UUID! You need to type in the UUID for Windows.old. If you type in the UUID for Windows Vista then you will not be able to boot into Windows Vista, at least not until you restore it using the backup file you created earlier.

Anywho, to make a Hiren's boot CD, do this:

I am going to recommend you use Hiren's boot CD (it will also go on a USB drive).

This is good for you because it has many more tools on it that on the XP Recovery Console CD, does not care about your Administrator passwords and you will not have to futz around in your BIOS if any afflicted system has SATA drives - Hiren's can deal with that.

http://rufus.akeo.ie/

FINALLY! HOURS AND HOURS OF SLAMMING KEYBOARD... I found a way to stop that god forsaken VIA HD Audio from replacing your working audio driver-- for me, at least.

To begin with, windows will always attempt to update and install new drivers, recently there's some words roaming about Windows 10 Audio not working or something, I guess VIA HD Audio is made to replace it?

So to fix it, simply refer to this: http://www.howtogeek.com/223864/how-to-uninstall-and-block-updates-and-drivers-on-windows-10/ or from this quote to disable windows from automatically installing new driver:
OR refer to this quote which is taken from the link (in case the link is down or whatsoever preventing you from opening the website page:

  • "Open the Control Panel by right-clicking the Start button and selecting Control Panel. Navigate to System and Security > System > Advanced system settings. Click the Hardware tab, click Device Installation Settings, and select the “No, let me choose what to do option. Select “Never install driver software from Windows Update.”"
  • Now to uninstall VIA HD Audio Device, Right click Start > Device Manager > Expand "Sound, video and game controllers" and look for VIA HD Audio, double click it to bring up properties > Switch to Drivers Tab > Uninstall > Tick/Check/Mark uninstall/delete software device > OK.

It's a troubleshooter gotten from Microsoft that “hide” them, preventing Windows from automatically installing them.
  • Finally to reinstall High Definition Audio Device driver (for me, not sure about other users), return back to Device Manager > Action button > Scan for hardware changes... and it will install the driver.

As a warning, disabling the option to automatically installing recommended drivers....... wait, I'm not sure if disabling that is a bad idea, but it's causing problem either way.
