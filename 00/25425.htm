suzanne s. barnhill say:

You don't need the Avery templates. Instead, on the Mailings tab of the Ribbon, click on Labels in the Create group. In the Labels dialog, click on Options. In the list of vendors, select the appropriate Avery listing (A4/A/5, US Letter, or Zweckform, and then find your label stock number from the product number listing. Note that the numbers are sorted as text rather than numbers, so you have to scroll through all the numbers that begin with 1 before you get the ones that start with 2 and so forth.

When you have found the correct business card stock number, select it and click OK, then New Document to create a document in which you can set up your business cards. Note that the document will be set up as a table, so make sure you have table gridlines displayed so you can see the card boundaries. You can set up one card, then copy it, select the entire table, and paste. If there are graphic elements that don't paste correctly, you can use the macro at the end of http://www.gmayor.com/graphics_on_labels.htm.

steve rindsberg say:
This bit of code will work on a Windows PC; give it a try on your Mac.  The first subroutine simply shows you the link.  The second one shows it to you and allows you to edit it.  If you try to link to a file that isn't where you say it is, the code will error out.  PowerPoint doesn't allow you to link to non-existent files.

Sub ShowMeTheLinkSource()

    With ActiveWindow.Selection.ShapeRange(1)
        MsgBox .LinkFormat.SourceFullName
    End With

End Sub

Sub EditTheLinkSource()
    Dim sTemp As String
    With ActiveWindow.Selection.ShapeRange(1)
        sTemp = InputBox("Edit the link", "Edit link", .LinkFormat.SourceFullName)
        
        ' if user deleted the link text, don't change anything
        If Len(sTemp) = 0 Then
            Exit Sub
        End If
        ' if user left link text unchanged, don't change anything either
        If sTemp = .LinkFormat.SourceFullName Then
            Exit Sub
        End If
        
        ' user made a change to the link; apply it
        .LinkFormat.SourceFullName = sTemp
        
    End With

End Sub

bob jones aka: cybertaz say:
You should be able to determine the path to the linked Excel file by opening the presentation & going to File> Properties - Contents. What you'll find listed under Links is something such as: C:\Documents and Settings\....

That information cannot be modified, however. As long as the file is on a Mac the Link is essentially 'dead'. The path cannot be edited because Mac PPT does not support linking & therefore does not have the features to edit links. Nor can the Linked object be updated because the Mac version cannot parse the DOS/Windows path syntax.

As long as the file is on a Mac the linked object is basically a snapshot of what the content was as of the last refresh while on a PC. Any change or update of the link will have to be done on a PC.

steve rindsberg say:
For the record, I'm with Bob.  OTOH, the code I supplied *will* work on a Windows version of PPT (pretty much any version back to and including PPT 97), so if you have access to one, you could fix it there.

On the other hand, it might not be worth the trouble, since, even if fixed, the link would still not function back on the Mac.
