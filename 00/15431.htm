dianeoforegon say:
When you move messages from the IMAP folders to folders "On My Computer" it is copying from the server to your computer. 

Use command-drag or select the Move to icon in the Ribbon. You can select either "Choose" to move a message or select "Copy to Folder" to move to another folder under IMAP.

This will force a move instead of a copy. it works just like drag in files in the Finder - between different servers (and moving from an exchange or IMAP folder to a local folder is moving between server) the file/email is copied by default. 

  • Cmd-drag forces a move between servers/volumes.
  • Option-drag will copy on the same server/volume. Dragging between different folders on the same server moves by default, but a copy can be forced by holding down the Option key when dragging.

macropod say:

The references to .Item(1) & .Item(2) are to the two fields in the set spanned by the cell range.

 

For the progress status, try something based on:

Option Explicit
Dim SBar As Boolean           ' Status Bar flag
Dim TrkStatus As Boolean      ' Track Changes flag

 

Private Sub MacroEntry()
' Store current Status Bar status, then switch on
SBar = Application.DisplayStatusBar
Application.DisplayStatusBar = True
' Store current Track Changes status, then switch off
With ActiveDocument
  TrkStatus = .TrackRevisions
  .TrackRevisions = False
End With
' Turn Off Screen Updating
Application.ScreenUpdating = False
End Sub

 

Private Sub MacroExit()
' Clear the Status Bar
Application.StatusBar = False
' Restore original Status Bar status
Application.DisplayStatusBar = SBar
' Restore original Track Changes status
ActiveDocument.TrackRevisions = TrkStatus
' Restore Screen Updating
Application.ScreenUpdating = True
End Sub

 

Sub Demo()
Call MacroEntry
Dim WhichTable As Long, WhichRow As Long, lCount As Long, lTotal As Long
With ActiveDocument
  For WhichTable = 1 To .Tables.Count
    'Update status bar
    Application.DisplayStatusBar = "Updating Table " & WhichTable & " of " & .Tables.Count
    'Defaults
    lCount = 0: lTotal = 0
    With .Tables(WhichTable)
      'Get Total Number of Games that can be Downloaded
      lTotal = .Rows.Count - 2  '-2 To Remove Title Row & Totals Row (First & Last)
      For WhichRow = 2 To lTotal
        'Get Number of Games I've Downloaded
        If Len(.Cell(WhichRow, 6).Range.Text) > 2 Then
          lCount = lCount + 1
        End If
      Next WhichRow
    End With
    'Set the Values
    .Variables("Count").Value = Format(lCount, "#0")
    .Variables("Total").Value = Format(lTotal, "#0")
    'Inserting the Numbers into the Correct Tables
    With .Tables(WhichTable).Cell(WhichRow, 1).Range.Fields
      .Item(1).Locked = False
      .Item(2).Locked = False
      .Update
      .Item(1).Locked = True
      .Item(2).Locked = True
    End With
  Next WhichTable
End With
Call MacroExit
End Sub

 

The output will be displayed on the status bar for as long as it takes the macro to run. Depending on how many tables you have to process, it may not be there for long. The MacroEntry & MacroExit subs are generic ones I use for multiple macros. They probably do more than you had in mind.
