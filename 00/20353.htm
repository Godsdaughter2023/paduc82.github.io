There is no need to have two cells.  If you reference Range("C2") in VBA, you can use it either as a number or as a date.

First, it is unclear what the format of C2 is.  By default, =DATEVALUE(B2) is formatted as General if the cell was originally General.  And in that case, C2 would display "just the number result" (39330) already.

In that case, Range("C2").Value is type Double.  If you want to use it as type Date, you can use CDate(Range("C2")).

However, from the context of your question, it appears that you might set the format of C2 to Date.

In that case, Range("C2").Value is type Date, and Range("C2").Value2 is type Double.

But you can always use CDbl(Range("C2")) [or Range("C2").Value2] and CDate(Range("C2")) to ensure that Range("C2") is used as the intended type regardless of the Excel cell type.
