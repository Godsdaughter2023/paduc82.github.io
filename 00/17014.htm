ossiemac say:

The Hrs, Mins and Secs are in text format instead of time format.

To convert to time format:

  1. Use Number format and select Custom and set the format of the column to [hh:]mm:ss
  2. In any empty cell enter 1 (ie. the numeral one)
  3. Copy the cell with 1 in it.
  4. Select the column of times and then Paste Special -> Multiply. 
  5. After previous step, the times could appear as simply decimal numbers but use Number format again to re-format as per step 1.

Note that the square brackets around the hh in the number format instructs Excel to not convert the hours to days and time but retain as hours.

Multiplying by one does not change the value but performing a mathematical operation on a text number forces it to become a number (Times are only numbers displayed in an alternative format. If less than 24 hours it is a decimal number but more than 24 hours it is a whole number and decimal)

Idle Session Timeout in SharePoint and OneDrive Introduction & - Microsoft Community

tom van gaever [msft] say:

Idle Session Timeout in SharePoint and OneDrive Introduction & Frequently Asked Questions

Idle Session Timeout in SharePoint and OneDrive Idle session timeout provides an Office 365 administrator to configure a threshold at which a user is warned and subsequently signed out of SharePoint or OneDrive after a period of inactivity.

Idle session timeout policies allow Office 365 administrators to automatically sign out inactive sessions preventing the overexposure of information in the event a user leaves a shared system unattended.

Note: In scenarios where Keep me signed in is selected at authentication, the client will not honor the idle session timeout.   

More information about this feature can be found in the techcommunity article and in the Ignite recorded session

.

Feel free to read

  1. Introducing Idle Session Timeout in SharePoint and OneDrive (Coming to Preview) by Bill Baer (MSFT)
  2. Default Session Timeouts in Office 365
  3. Enforcing idle session timeout restrictions in SharePoint Online article written by MVP Vasil Michev regarding this topic

Frequently Asked Questions

Q:When will idle session timeout start rolling out as preview?
A:November 6, 2017

Q: Is idle session timeout enabled by default, can I control the settings?
A: No.  Idle session timeout is disabled by default.  The warning and timeout timespans, as well as enabling idle session timeout are administrator controlled.  Instructions will follow as we start to roll out this feature.  


Q: Does the policy effect existing signed in sessions? 
A: No, only new sign-ins to new browsers  


Q: How long does it take to effect?
A: Approx. 15 minutes  

Q: What is considered a managed device?
A: A device is managed if Azure Active Directory indicates to SharePoint Online that the device state was evaluated and the device is at least one of the following: Domain joined or Compliant. Device state claims are not passed in Google Chrome or when using inPrivate mode – device claims are only available on Internet Explorer or Microsoft Edge on Microsoft Windows.  


Q: Can I hide the Keep me signed in prompt? 

A: During the public preview period of the new sign-in experience, the updated “Keep me signed in” prompt will only show when users opt into the new sign-in experience. Users using the old experience will continue to see the checkbox and will not get the prompt. Admins can choose to hide this new prompt for users by using the “Show option to remain signed in” setting in company branding.  

NOTE : Existing configurations of this setting will carry forward, so if you previously chose to hide the “Keep me signed in” checkbox in your tenant, we won’t show the new prompt to users in your tenant.   This change won’t affect any token lifetime settings you have configured.  


Q: When will idle-session timeout be generally available? 

A: Late CY2017

Q: How do I view the idle browser settings? 

A: Use the GET-SPOBrowserIdleSignOut cmdlet via SharePoint Online Management Shell.

Q: How do I set the idle browser settings? 

A: Use the SET-SPOBrowserIdleSignOut cmdlet via SharePoint Online Management Shell. Example: 

Set-SPOBrowserIdleSignOut -Enabled $true -WarnAfter (New-TimeSpan -Seconds 1200) -SignOutAfter (New-TimeSpan -Seconds 1500)
