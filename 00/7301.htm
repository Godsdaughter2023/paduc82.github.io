bofra say:

Shanmugam,

DL sync to O365 is a 3 step process:

1. Sync from local AD to SQL database in AAD Connect

2. Sync from AAD Connect database to Azure

3. Sync from Azure to Exchange Online

(Not mentioning the OAB sync here)

I generally start investigation from the last sync, as I've often seen there to be the failure.

How?

1. From O365 powershell, run: "Get-DistributionGroupMember <DL name/email address>" and confirm that you don't see all the members as you see when you run the same cmdlet from on-premise (this confirms there indeed is an issue; but if you see all DL members here then issue is between GAL to OAB update)

2. From O365 pwershell, run: "Get-DistributionGroup <DL name/email address> |select ExternalDirectoryObjectId" to get the ObjectId of the DL

3. From Azure powershell (Connect-MsolService), run: Get-MsolGroupMember -GroupObjectId <ObjectId value received from step-2>" and verify if the DL members are showing accurate in this output

If step-3 shows accurate DL members, issue is between Azure to EXO sync. This will require Office 365 support team to submit a SOP for RecipFullSync (this basically forces an Azure-EXO sync). Alternatively, making a modification to the DL property from local AD (like Fax, Notes, etc.) will force a new sync process and thus forcing Azure to re-sync with EXO.

If step-3 also shows inaccurate DL members, then the issue is local: Either AAD Connect did not import the DL members correctly from local AD, or did not export correctly to Azure.

Let us know how it goes.

    charles kenyon say:
    Nested {IF} Fields

    One {IF} can be used inside of another one to give logical branches and twists.

    {IF {PAGE} = 1 "This is the first page." "{IF {PAGE} < {NUMPAGES} "Page {PAGE} of {NUMPAGES}" "{NAME \p}" }" }

    This is called "nesting." The nested IF field is used if the first expression is false.


    You can also use the merge field selector inside your IF field. To do this you need to turn field display on (Alt-F9) and not simply toggle field codes. Toggling field codes usually shows field results rather than codes for fields inside other fields.
