jay freedman say:

It's often a bad idea to install an older version of a program while the newer version is already installed. The older installer, being ignorant of the existence of newer versions, sets registry entries (particularly those in the HKEY_CLASSES_ROOT hive) to point to its own files. Because OLE depends critically on registry entries to know what to execute, you run into problems like this.

 

It might be sufficient to rerun the installer for Origin 8.6 if it has a Repair option; that should rewrite the necessary registry entries. If that doesn't work, it would be necessary to uninstall both versions, then install the partial version 7, and finally the complete version 8.6.

hansv mvp say:

You can create an On Click event procedure for the command button. With a command button named cmdOpenQuery and a query named qryMyQuery:

 

Private Sub cmdOpenQuery_Click()
    DoCmd.OpenQuery "qryMyQuery"
End Sub

 

By the way, I'd prefer to create a form based on the query, and to open that from the command button. End users shouldn't work directly with tables or queries.
