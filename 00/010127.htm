john mcghie say:

I am going to take a wild guess and suggest that somehow it is attempting to access your contacts, and crashing when it does. I have no idea why it would be doing that.

Let's carry out a couple of diagnostic steps, to see if we can narrow this down.

  1. Create a new blank document.
  2. Type one word (anything)
  3. Save and close.

Chances are the crash won't happen.

  1. Create a new user ID (it's temporary, you can delete it later).
  2. In Word, try the test above.  Chances are, it won't crash.
  3. Now, open the problem document.
  4. Make a change (add or delete a space)
  5. Save the document.
  6. Close the document.

Chances are it WILL crash there.

You can now delete the new User ID.

My guess is that you have a "circular autorecovery crash".  Something about a particular document crashed Word, but got saved to the autorecovery file.  Every time Word opens, it "recovers" the Autorecovery file, and promptly crashes again.

You may be able to break the circle by:

  1. Quit Word
  2. Navigate to Macintosh HD ▸ Users ▸ <your user name> ▸ Documents ▸ Microsoft User Data ▸ Office 2011 AutoRecovery
  3. Delete any files you see in there

When you open the trouble document, chances are you will find some text missing: that would be where the corrupt code that was crashing Word was.

Hope this helps

The handwritten example suggests that you are interested in charts of the following form.

For Economy A:








For Economy B:





In all cases, the procedure for drawing the chart is as follows, using Econ B as an example.

1. Select the X and Y data, namely B2:C4.

2. Click on the Insert tab on the ribbon.  Then in the Chart section, click on Scatter, then Scatter With Straight Lines.  That creates the basis chart.

3. Right-click on the line in the chart, then click on Add Trendline or Format Trendline, depending on the state of the chart, and click on Display Equation On Chart.

Normally, we should only need the X and Y data in B2:C2 and B4:C4.  Recall:  "it takes two points to define a line".

However, if we did that, Excel becomes confused:  it assumes that B2:C2 is X data, and B4:C4 is Y data.

To avoid that confusion, we need at least 3 points.  B3 is midway between B2 and B4; and C3 is midway between C2 and C4.

Alternatively, we could create the chart Series Data manually, eliminating B3:C3 first.

Finally, we do not need the graph and trendline at all.  We can compute the linear equation of the form y = mx+b directly.

Suppose B16 is some X (e.g. 100).  Then we can compute Y in C16 as follows:

=B16*SLOPE({200,0},{0,200})+INTERCEPT({200,0},{0,200})

See the Help pages for SLOPE and INTERCEPT for an explanation of the parameters.

And of course, the array constants {200,0} and {0,200} could be replaced with cell references.  Generally, that is preferred.
