Select Start, All Programs, Accessories, right click Command Prompt, and click Run as administrator from the drop down menu. Type or copy and paste: findstr /c:"[SR]" %windir%\Logs\CBS\CBS.log >"%userprofile%\Desktop\sfcdetails.txt" and press the ENTER key. You will find an sfcdetails.txt file on your Desktop.

 

Please note there are three spaces in the command: Between findstr and /c, Between [SR]" and%windir Between CBS.log and >"%userprofile

 

Please upload a copy of your sfcdetails.txt file to your OneDrive as a shared file and post a link here. Enter sfcdetails.txt in the Search box above the Start button and press the ENTER key to locate the file created.

 

Note The Sfcdetails.txt file contains details from every time that the System File Checker tool has been run on the computer. The file includes information about files that were not repaired by the System File Checker tool. Verify the date and time entries to determine the problem files that were found the last time that you ran the System File Checker tool.

You should not press the Power button to shut down the computer or to resolve a freeze. Click on the Orb in the bottom left of the Desktop and select Shutdown. If you encounter problems shutting down then select the keyboard shortcut Ctrl+Shift+Esc to bring up Task Manager and click on the Applications tab. Place the cursor on a running application to select it and click on the End Task button. As the application is removed from the list repeat the procedure on the next application in the list until all are gone from the list. If you use the Taskbar to access programs simply close all open windows. You should then be able to shut down your computer.

Uninstalled App Keeps Trying to Update - Microsoft Community

misterfocus say:

Uninstalled App Keeps Trying to Update

There appears to be a flaw or  bug in the windows store app update process.

 

I have uninstalled Bing Sports, yet the store continues to prompt me to update it. I have updated it, uninstalled via right click, and this did not fix it.

 

I then went to powershell and tried to uninstall it this way. It also failed. It told me the app package wasn't installed even though I had just used get-AppxPackage to confirm that the app package listing was there! Apparently, the right-click uninstall sometimes fails to remove the listing and the store checks this listing when prompting for updates, therefore it will prompt you to update an app that's already installed (but erroneously listed as still installed)

 

That's my guess, but here's the solution:

 

This finally fixed it:

 

1) reinstall the app via the store, as usual

2) open powershell, use the command get-AppxPackage -allusers

3) find the app package you want to uninstall for good

4) double click the package name, put mouse on title bar, right click, select edit > copy (trying to type in the long package IDs will only make you swear)

5) enter command Remove-AppxPackage [insert package name here by right clicking title bar, selecting edit > paste

 

You should then get a confirmation that the package is removed and your live tile should stop giving you erroneous update info for at least this package.
