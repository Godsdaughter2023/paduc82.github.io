Even though officially you can open Word files in different versions than they were created, to play it safe, you should try to use the same version as the file was created in.  LONG ago, I was able to consistently  corrupt my document when editing it on 2 different machines using different versions.  I have not heard of that being a current issue, but you never know.

Regarding which one to use.  Generally Access people prefer Access and Excel people prefer Excel.  So I will stick my neck out and make some tentative observations:

1. Depending on how many question and what type of data - numerical or verbal (text) Access can store larger volumes of data.  But it doesn't sound like there is a lot of data, so size won't dictate the use of one application over the other. 

2. Without knowing the types of questions and types of data you will be collecting its hard to lock down one app or the other.  Keep an open mind - you can actually store the data in Access and analyze it in Excel through a refreshable connection. 

3.  Access gives you relational database capability, which is limited at best in Excel.  This means you have such things as primary key fields, control of data types and the ability to use multiple tables.  (With PowerPivot Excel has gained some genuine relational database capabilities - but few people know this new tool).

5.  Access' report writing capability would be judged by most to be superior to Excel's.

6.  Both programs provide a Pivot Table tool to analyze your data, unfortunately knowing one doesn't mean you know the other, because they aren't identical.

  I haven't seen your data but I would be tempted to input and store the data in Access and connect to it from Excel via pivot tables, data table connections and/or powerpivot for my analysis.

jay freedman say:
I think you misunderstand the article (http://support.microsoft.com/kb/970020). That registry entry doesn't exist -- for any of the Office programs -- until you save a document in ODF, check the "don't show this again" box in the warning, and click Yes or No. Then the program creates the entry in the Options hive with the value 1 (meaning "disable the warning").

If you do that, the warning won't appear again, so you can't see the box to uncheck it. The information in the article is about how to change the registry value to 0, so the warning will appear again.

Of course, if you want to prevent the warning from appearing even the first time, you can do that by manually creating the DisableSaveAsLossWarningOpenDocumentText entry in each of the three Options hives (right-click > New > DWORD) and setting its value to 1.

dlponline say:

I came up with a solution. It requires using ActiveX controls instead of regular form controls.

To get tabbing to work correctly between cells and controls, do the following:

  • Use ActiveX controls instead of form controls.
  • Change the TabOrder array to be an array of arrays. What you need to end up with is a two dimensional array where the first item in each subarray is a designator for the type of information in the second item in that subarray. For example, I used "a" to indicate that a subarray specifies and ActiveX control and "c" to indicate that a subarray specifies a regular cell. For the second item in each subarray, I specified either the control name, such as "combo1," or the cell reference, such as "C1."
  • Then modify the rest of the code to check the designator and then do what is appropriate for cells for ActiveX controls. For example, you may want to get rid of merged cells for regular cells but you don't need to do that for ActiveX controls. In the end, the tabbing occurs by calling the Activate member for the control or the cell.

OK found the issue for this problem and will add a fix here to help others with webcams with built in hardware encoding. The new skype is using the NGC codec (next generation codec) which ignores if your high end camera has hardware encoding so uses your graphics card as the encoding device and in turn turns your graphics card to full base clocks and wastes electricity and heats up your GPU for no good reason or benefit.  

Now if you have a camera with hardware encoder built in you can simply rename a file in skype to turn off the NGC codec and go back to the Silk codec that uses your hardware encoder in the camera and allows your GPU to idle at normal speeds again.

In C:\Program Files (x86)\Skype\Phone or wherever you installed skype there is a file called RtmCodecs.dll all you need to do is rename it to something like RtmCodecs.dll.NOTUSED. Restart skype and it will say there is an error with skype and says to download again or cancel, click cancel and use skype as normal. All will be fine apart from this error message on start up that can be ignored.
