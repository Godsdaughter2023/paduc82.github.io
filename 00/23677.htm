Others are reporting similar issues during install. The select keyboard screen is not normal during an upgrade.

I would try to eliminate any existing traces of the new build. Files used for the upgrade reside at C:\Windows\SoftwareDistribution and a hidden folder at C:\$WINDOWS.~BT. I find it easiest to boot with Windows 10 install media, such as a bootable USB with Windows 10 install on it. From the first screen after install loads, SHIFT-F10 will get to Command Prompt. From there you can delete those folders and their contents using the remove directory command (but check if C: is actually pointing to your usual Windows volume, as sometimes it varies in this mode):

rd /S /Q C:\Windows\SoftwareDistribution

rd /S /Q C:\$WINDOWS.~BT

Also, while at this Command Prompt, check your System volume to see if there is plenty of room. You will have to assign it a drive letter using Diskpart first. See the example below, where you can see that S: has about 75MB out of 100MB as free space. If you show very little free space, you might consider recreating the System volume. You could also check this by running diskpart now if you like, using an Administrator Command Prompt. If you assign drive letter S: using Diskpart, the free space on S: will also show up in File Explorer.
