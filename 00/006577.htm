ken sheridan say: It's hard to see why you are experiencing his behaviour if the form is set up as two pages in the normal way, i.e.

1.  A page break control is inserted in the detail section at the point you want the first page of each record to end.

2.  The form's DefaultView property is Single Form

3.  A control bound to the column containing the 'record number' is included in the form footer section.

4.  The form's AutoResize property is set to True (Yes).

The same value should show in the control in the footer for both pages of a record.

However, rather than two-page form, have you considered putting the controls on two pages of a tab control?  The control bound to the 'record number' column can be in the report footer, or in the detail section immediately below the tab control.  The use of a tab control like this is probably more intuitive to a user these days than paging down from the keyboard.

Another point here, which Karl touched upon earlier, is how you are computing the 'record numbers'.  The concept of a record number is really completely foreign to a table in a relational database, which is a set and as such has no intrinsic order.  A form can be ordered by the value in a column, however, by basing it on a query ordered by the column in question.  If you are using the built in autonumber facility to return the numbers, this is not really a reliable method to produce a sequential sequence of numbers as if a user begins to insert a new record, then abandons it, the number will not be re-used unless the database is compacted/repaired before the next record is added, so gaps will appear.  If you want the basis of the 'record number' to be the order in which the rows were inserted into the table, therefore, and you wish to avoid gaps in the sequence then you really need to compute the number in code rather than relying on the autonumber feature.  You'll find a demo of this as CustomNumber.zip in my public
https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

You might have to copy the text of the link into your browser's address bar (not the link location).  For some reason it doesn't always seem to work as a hyperlink.

The file includes three different scenarios, the appropriate one of which would be 'Sequential Numbering', which is an extension of the method published by Roger Carlson at:

http://www.rogersaccesslibrary.com/forum/topic395.html

If on the other hand the sequence of the 'record numbers' is determined by some other value, e.g. a date column in the table, then it should not be stored in a column but computed on the fly in a query.  The following is an example of an updatable  query which does this for transactions by date:

SELECT DCOUNT("*","Transactions","TransactionDate  <=  #"
& Format(TransactionDate,"yyyy-mm-dd") & "#  
AND (TransactionID <= " & TransactionID & "  
OR TransactionDate <> #" & Format(TransactionDate,"yyyy-mm-dd") & "#)") AS RowCounter,
Transactions.*
FROM Transactions
ORDER BY TransactionDate, TransactionID;

Note how the autonumber primary key TransactionID is brought into play here as a tie-breaker in the case of two or more transactions on the same day.  These number will not necessarily be static of course, and will change if a row is inserted or deleted out of date order, which does
raise the question of what purpose does numbering rows like this serve?
