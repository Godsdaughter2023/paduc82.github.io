larrysuper say:

I just found it or overlooked it

The error code 0x80070035 indicates that the network path is not found. This behavior occurs if the NetBIOS over TCP IP is not enabled in Windows Vista. Please follow the steps below.

 

1. Click "Start", input "NCPA.CPL" (without quotation marks) and press Enter.

2. Right click on the connection that you use for the local connection, and then click "Properties".

3. Click Continue button to verify the administrator permission.

4. Click to select "Internet Protocol Version 4 (TCP/IPv4)", and then click "Properties".

5. Click "Advanced" button in the open window. On the "WINS" tab, click to select "Enable NetBIOS over TCP/IP", and then click "OK" to save the setting.

 

The following article introduces NetBIOS over TCP/IP

 

NetBIOS over TCP/IP

http://technet.microsoft.com/en-us/library/bb727013.aspx

 Now it is working!

john - project say:

Gehani,

Well there is a bit of information missing from your requirements. For example, what if the finish date is equal to the status date. Note, the status date is as of 8:00AM on the status date whereas the finish time of a task on that same date is at 5:00PM, assuming a standard calendar.

Also, it doesn't make much sense comparing the status date to the baseline finish, since the baseline is usually fixed when the baseline is set so it doesn't move with the dynamics of the plan. I think you really want to compare the status date with the scheduled finish.

Whenever you do a formula like this you should always test for the completed tasks first, then test for other conditions. This is one version of formula which should work for your scenario. If you want to address the condition when the status date and finish date are the same you will need to add a little complexity due to the reason cited above. At any rate here is the formula (I use the Number1 field), and a simple test file to show what it does.

Number1 = IIf([% Complete]=100,1,IIf([Finish]<=[Status Date],2,3))
