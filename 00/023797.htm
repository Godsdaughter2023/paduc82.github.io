On the off chance that your data is not always filled across your three rows (that is, as an example, Column B could have the last piece of data in it while the cells in Columns A and C next to it are blank), then you will need a more general method. The following line of code will find the last used cell no matter what column it is in...

ActiveSheet.Cells.Find(What:="*", SearchOrder:=xlRows, SearchDirection:=xlPrevious, LookIn:=xlValues).Select

And no, the problem is not with any background stuff since the CPU issue is happening in Windows Services and the Groove App.
