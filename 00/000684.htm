dale howard [mvp] say:

Barb --

In Microsoft Project 2010 or 2013, open a project and then click File > Save As. On the Save As page in the Backstage, select the Computer icon and then click the Browse button.  At the bottom of the Save As dialog, click the Tools pick list and select the General Options item.  In the Save Options dialog, there is an Always Create Backup checkbox.  Select it and click the OK button.

Every time you save a project, the software will create a same-named project file, but with the BAK file extension.  This file can actually be opened in Microsoft Project, and it is a copy of the project.  The BAK file is saved in the same folder as the original MPP file, which I realize will not fully meet your requirements.  You can at least get a backup of each project when you save it.  Hope this helps.

john - project say:

Barb,

Along with Dale's suggestion, you could use a Save Event macro to save the file to whatever location you want.

Oh, and one more thing. I don't remember your file structure but it your files use a resource pool, dynamic master or any other linked structure, you do NOT want to automatically, (or manually), save any of the files to another location. That will essentially guarantee corruption. If all the files in a linked structure are in a single folder then it is okay to save the entire folder to another location as a backup.

john - project say:

Barb,

Okay, if you're game, I worked out some code to do what you want

I wanted to keep it simple enough, (Event macros can be fairly simple or rather complex to set up), so I elected to use a Project level Event. First of all, since we'll be using the Project_beforesave Event with an embedded save operation in the code, we have a recursive process and therefore must provide an "exit" so we don't end up in a continuous loop. To facilitate that, the code uses Flag20 on the Project Summary Task as the control.

In my test example, I set up a simple project file which normally resides in my documents folder. To test the code, I set it up to save that file to my desktop any time a Save operation is initiated by the user. You will need to modify the SaveAs path to whatever location you want your backup.

To set it up follow these steps.

1. In the file you wish to have auto-saves to another folder, set the Project Summary Task Flag20 field to "yes". This is necessary to allow the below code to be saved with the file without triggering the backup save.

2. Go to Developer/Code group/Visual Basic to open the Visual Basic Editor window

3. Hit the Project Explorer icon

4. If the VBAProject (xxxx) is not expanded, expand it

5. Double click on the ThisProject (xxxx)

6. Insert the below code into the code window. Modify the path as needed in the SaveAs Method

7. Go to File/Save xxxx. That will save the save the code to the file and also execute the macro for the first time (without saving a backup)

8, That should do it. Now when you hit Save at file level, the code with execute and save a backup to the location you specified.

Private Sub project_beforesave(ByVal pj As Project)
    Application.DisplayAlerts = False
    If ActiveProject.ProjectSummaryTask.Flag20 = False Then
        ActiveProject.ProjectSummaryTask.Flag20 = True
        ActiveProject.SaveAs Name:="c:\users\John\desktop\backuptest.mpp"
        FileCloseEx Save:=pjDoNotSave, noauto:=True
    Else
        ActiveProject.ProjectSummaryTask.Flag20 = False
    End If
End Sub

john - project say:

Barb,

You're welcome. First of all, Event macros are more complicated than regular macros. Second, since the process your asked for creates a recursive procedure, (i.e. procedure that calls itself), the level of complexity raises significantly.

Doing a Save As will initiate the Event macro just like a regular Save. And yes, you could modify the code to solicit user input (e.g. "Do you want to do a backup save to xxx?"). Just remember though that with Event macros, they will always fire when the event occurs, so if you add a message, that message will always pop up when doing a save. In that case, it seems to me that you'd be better of just using Save As for your backups with no Event macro at all. You can prevent the macro from doing anything if you set the Flag20 field of the Project Summary Task to "yes" before doing a save. The Event macro will still run but it will not do any backup save. However it will clear the Flag20 field so you would have to remember to set the flag each time you don't want to do a backup save.

If you haven't already, you might want to do a test run of the macro to see how it works on a simple test file rather than on your "live" file.

Your original question was, "how do I do an auto-save to another location?" and Dale and I both answered that, I think. If so, please consider marking either or both or our responses as the answer.
