I called Microsoft Finland but they couldn't help, they directed me to here. ;)

Question: Is it possible and how to add this kind of data to Outlook 2010 calendar?

It could look like national holidays look at the moment, nice tidy "plate" at the top. Any ideas?

adam vero say:

You could convert the provided data to a text file suitable for import into Outlook. National Holidays use a plain text file with the extension .hol, which simply requires a list of the names for the days and their dates, with a heading row for the country name and the number of included dates.

Get your data in two columns in Excel and save as a csv file, then change the name to "MyNameDays.hol" (or whatever) and open in notepad. Add a heading of "[Finnish Name Days]" and the number of entries in the list.

The dates are single days, though, there is no implied recurrence  pattern but you could create these for multiple years with Excel formulas relatively easily, or just use find-and-replace on the finished text file in notepad to do this (and end up with a series of files which you then copy and paste together - don't forget to update the number at the top or it will ignore any extra ones).

An example of part of a .hol file would look like this (the 172 identifies how many entries there are in this country's section):

[Australia] 172
Anzac Day,2009/4/25
Anzac Day,2010/4/25
Anzac Day,2011/4/25
...

More information on adding holidays to Outlook 2010:

How to add national holidays in Outlook 2010

john - project say:
SHM72,
First I applaud your company for deviating from the old adage that "time is money". I think more in terms of how much work has been done, the cost is what it is. I've always felt that the EV metric of schedule variance being expressed in terms of cost is a misnomer.

Okay, enough of the soapbox. Yes you can do earned value metrics in terms of hours. It can be really simple or rather involved. The good thing about expressing earned value in terms of cost is that everything, labor and material, is rolled into the equations. But if you want to do earned value in hours, having tasks with both labor and material assignments complicates the calculations.

Let's say you either have no material or your schedule has the material tasks separate from the labor tasks. If there is only one labor rate, (unlikely but not unusual if generic bid rates form the labor rate structure), then you could create custom fields wherein BCWP, BCWS are simply divided by the single labor rate. The Actual Work field will replace ACWP. That's the simple solution.

However, if your schedule has multiple resources with multiple labor rates, (more typical), then it gets a bit more complicated. Actual work will still replace ACWP but replacements for BCWS and BCWP must be calculated using cumulative cost as of the status date and that can't be done with formulas in custom fields. It will require VBA.

That's my help/guidance/ideas. Others may chime in with theirs.

John

Mail Merge Excel to Word Labels Spacing and Blanks

I have blank spaces in my Excel 2010 database. When I do a Mail Merge with Word 2010 for Labels, the blank spaces appear on the labels.  How can I get the merge program to move the lines on the labels to fill in the blanks.  I just moved from XL and did not realize I had to program.
doug robbins - mvp office apps & services (word) say:

You will need to use an If...then...Else field construction

Say you have the following fields Address1, Address2, City and sometimes Address2 might be blank, you would use

{ MERGEFIELD Address1 }¶
{ IF { MERGEFIELD Address2 } <> "" "{ MERGEFIELD Address2 }¶
{ MERGEFIELD City }" "{ MERGEFIELD City }" }

Where you use Ctrl+F9 to insert each pair of field delimiters { } and you use Enter (or Shift+Enter) where the ¶ appears in the above construction.

Instead of starting from scratch, you could insert the mergefields and text as follows

«Address1»¶
IF «Address2» <> "" "«Address2»¶
«City»" "«City»"

and then select the second and third lines and press Ctrl+F9.  Then press Alt+F9 a couple of times to toggle off the display of the field codes.

You only need to set this up in the first label on the sheet and then use the Propagate Labels facility to propagate it to the other labels on the sheet.

Hope this helps.

greginmich say:

There actually is an option to view the logs, but it isn’t very friendly. When you create a Support Log, two of the MpCmdRun logs that are included in the cabinet file are also created in the Support directory, and they can be inspected for records of signature updates, threat detections, and scans. The files are MpCmdRun-NetworkService.log and MpCmdRun-System.log. These same events are also logged as System Events (Microsoft Antimalware) and can be viewed in the Event Viewer, but this provides less information, and requires more interpretation.

 

GreginMich
