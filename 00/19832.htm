davidf2 say:

No offense but what a convoluted way to go about this.

It used to be that Publisher produced 'round trip' code that allowed it to open Publisher websites in Publisher fully formatted but that was taken away as of Pub 2007 if memory serves. As of Pub 2010 MS started deprecating the web building capability and stripped even more functionality as of Pub 2013. Bottom line I know of know way for you to use Publisher 2013 to do directly what you want to do.

Was the manual text only or were there pictures and other objects? When you produce a HTML version of a Publisher page it produces not only the .html file but also copies of all the images on the page. Unless you change the default settings Publisher also produces a 'thicket' folder or subfolder along with the index.html page that contains all the supporting images. The index.html file and the index_files folder together were your complete webpage. You can change that option and all the image copies will be produced at the same directory level as the index.html file. However, which ever way you go the code in the .html file contains the instructions for the layout for the page...where to load the images, all the formatting. If your translator changed the text from English to another language that is going to throw off the whole layout.

Bottom line is you are probably going to have to redo each page layout. Try opening the .html file your translator sent back in Publisher. It will at least give you the text. Then you can copy and paste the new text into the old Publisher files and tweak the layout as necessary. Perhaps if you substitute the new index.html file with the old it will open with the images. Note that there is no easy way to combine multiple Publisher files so you might want to build the manual as one file, or plan ahead as to how your will append or combine mulitple pdf files if you generate individual pages.

I'm sorry but I just don't know of any easy way for you to accomplish what you want. Maybe someone else will using a full fledged html editor. Maybe you can go to your original Publisher files and use the 'Edit Story in Word' feature open the text boxes as Word files and edit those English files, copy and pasting the other language text. Maybe someone else will have a better idea...

hansv mvp say:

Windows uses a specific character as the so-called list separator.

In countries such as the USA and UK, the comma is used as list separator. Hence the name "CSV" which stands for Comma-Separated Values.

The list separator character is also used to distinguish the arguments of a function in Excel.

In countries that use comma as decimal separator, such as many continental European countries, using a comma both as decimal separator and as list separator would be very confusing: does 3,5 mean the numbers 3 and 5, or does it mean 3 and 5/10?

So in such countries, the semi-colon ; is used as list separator.

If you want to open a CSV file with a list separator that does not conform to your regional settings, you can change it extension from .csv to .txt. When you open the file in Excel, the Import Text wizard will start. It lets you specify the delimiter.

Although the thumbnail is generated by Word and stored inside your document, the actual decision on showing it is made by your operation system.

You have to decide what you want. Either you turn of the displaying of thumbnails for all file types or you start saving your Word documents without thumbnails.

jay freedman say:

If, as is typical, each appendix is a separate section (beginning and ending with a section break), it's probable that any corruption is in the code represented by the section break at the end of a section. (Word's construction is backwards from what any sensible person would expect -- the information about formatting a section is held at the end of the section, not at the beginning.)

So the first thing to try is to copy a complete appendix except the section break at its end (which you should be able to see by clicking the ¶ button to display nonprinting characters) and paste that into a new document. If the equation editor works there, you can insert a section break at the end of your new working document (starting with the copy of the main body) and paste the appendix after it. Then repeat with the next appendix, and so on.
