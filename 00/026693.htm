hansv mvp say:

I don't think that it is possible without VBA. You could use the following user-defined function:

Function LinkAddress(c As Range) As String
    If c.Hyperlinks.Count Then
        LinkAddress = c.Hyperlinks(1).Address
    End If
End Function

Let's say you have hyperlinks in B2 and down. In (for example) C2, enter the formula

=LinkAddress(B2)

and fill down as far as you want.

hansv mvp say:

You could create the following macro in a standard module:

Sub PlayWav()
    Dim wsh As Worksheet
    Set wsh = ActiveSheet
    ' Playing the sound will activate Sheet1
    Worksheets("Sheet1").OLEObjects(1).Verb xlPrimary
    ' Return to original sheet
    wsh.Select
End Sub

where Sheet1 is the sheet in which the .wav file is embedded.

Place a command button from the Form Controls on each sheet, and assign PlayWav to it.

Access 2007 upgrade to 2010 - Microsoft Community

marsmaed say:

Access 2007 upgrade to 2010

Will Office Access 2010 64bit convert my current access 2007 32bi databases t to the access 2010 64bit version.

hansv mvp say:

You don't have to convert databases to work with them in 64-bit Access 2010.

If you have VBA code that uses Windows API calls, you will have to go through them - many of the declarations will have to be modified for the 64-bit version.

And - but this is not specific to the 64-bit version - Access 2010 doesn't come with the Calendar control, so forms that use this control won't work any more. You'll have to remove the calendar control and the corresponding code.

If possible, I'd go for the 32-bit version of Access (and Office) 2010 instead of the 64-bit version. You'll have fewer compatibility problems.

hansv mvp say:

No, the 64-bit version of Access doesn't have any advantages over the 32-bit version.

The only compelling reason for installing the 64-bit version of Office 2010 would be working with extremely large Excel workbooks. The increased memory capacity of 64-bit Excel 2010 helps there.

Other Office applications don't profit from being 64-bit.

For that reason, Microsoft itself recommends using the 32-bit version for most users. See http://office.microsoft.com/en-us/word-help/choose-the-32-bit-or-64-bit-version-of-office-HA010369476.aspx
