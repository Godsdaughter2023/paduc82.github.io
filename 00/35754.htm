v5point0 say:

How does the Windows Store work?

Changing stores are sometimes inevitable due to studying/living/earning abroad or for a certain app availability. Thus, using the same Microsoft Account, if I were to purchase a globally available app from the US Windows Store and when I switch to the UK Windows Store, will I receive automatic updates assuming the same app is available in the UK Store? Does the Windows Phone Store work similarly? The reason I ask this is to get a better understanding of how Windows Store works in order to aid purchases in the future.

In comparison:

Google's Play Store is actually one single storefront with apps and content available across regions at the discretion of Google and the app developer/content publisher. Reviews and ratings are global. All card payment methods are accepted irrespective of physical origin and store country selection. A single purchase history exist showcasing all apps/content ever downloaded/purchased irrespective from which Play Store country. Apps currently installed on the device will receive automatic updates and user is able to leave reviews again irrespective of download/purchase origin.

Apple App Store/Mac App Store/iTunes Store/iBooks Store are actually segregated into country stores which all share separate purchase history. An app/content downloaded first in one country will only appear in the purchase history of that country despite being available worldwide. Any app/content currently installed on the device but not appearing in the purchase history of the current country store will not receive automatic updates and users are not able to leave reviews. Only on iOS will free apps currently installed on the device receive automatic updates despite not originally being downloaded from the current country store, however users still can't leave reviews nor does this apply to paid apps.

Of the two, I prefer Google's implementation as it provides more freedom and a richer review/rating structure - so how does the Windows Store work?

a. user say:

That message indicates that your NT File System (NTFS) is likely corrupted and you will not be booting in any mode until you fix it by running a chkdsk with error correction.

You said when booting in Safe Mode "it fails on the driver Mup.sys" but do you mean that it just hangs with that being the last thing you see on the screen? 

If that is the case, there is nothing wrong with your Mup.sys and what is happening is XP has detected the file system is corrupted and it running the chkdsk with error correction to try to fix it.  That is one of the checks that are made after mup.sys loads.  And, there is nothing wrong with Mup.sys - it has already loaded (sorry Microsoft engaged Support Engineer "experts") .

Even if you schedule a chkdsk to run on the next reboot and then boot in Safe Mode, your system will appear to be stuck on mup.sys as the scheduled chkdsk runs to completion.  This fools alot of people.

XP checks all volumes for corruption when your system restarts and in Safe Mode, after mup.sys loads, that is when the chkdsk is going to run (if it needs to) so your system may appear to be stuck or hung but if you look at the HDD access light to see if it is blinking or listen, you can probably see the system is "working" and that could take a very long time to complete.  Turning off your system with the power switch or unplugging it could make things worse.

You should make a Hiren's Boot CD and run the chkdsk with error correction from there so you can see what is is doing.  Here's how:

If I were you I would start by running a chkdsk with error correction (always first for that error).

Look at this example which looks remarkably like yours:

http://answers.microsoft.com/en-us/windows/forum/windows_xp-system/0x24-blue-screen-issue-sata-drives/0f8726f0-f377-4871-8602-4b0f9b5c68e9#_self

If you are not sure what you have for bootable media, make yourself a Hiren's Boot CD, boot on that into the Mini XP mode and run the chkdsk from there.

Make yourself a Hiren's Boot CD which you can download from here:

http://www.hirensbootcd.org/download/

The ZIP file is large, so the download will probably take a little while to complete bit it is worth it.  Then unzip the download to extract the Hirens.BootCD.ISO file that will be used to create your new bootable CD.

Creating a bootable CD from a .ISO file is not the same as just copying the .ISO file to a blank CD.  You have to use software that understands how to burn a .ISO file to a CD to create a bootable CD.

In the Hiren's ZIP file are the BurnToCD.cmd file that you can double click to launch it, but I have never used it and prefer to use ImgBurn.

You may need to adjust the computer BIOS settings to use the CD ROM drive as the first boot device instead of the hard disk. 

These adjustments are  made before Windows tries to load.  If you miss it, you will have to reboot the system again.

This disk checking can take a long time (perhaps several to many hours) depending on the size of the volume, the amount of data on the volume and what the disk checking finds to do.  You may have to run it more than once.
