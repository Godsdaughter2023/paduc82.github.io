Hi Robby, to get your system booting correctly, would require a clean install of Windows 10 on the SSD

The method I use is:
Pull the HDD out of the laptop until Windows is loaded on the SSD
(this is important, otherwise the UEFI boot partition and/or page file can end up on the HDD instead of the SSD)

Insert the Bootable Installation Media, then go into your UEFI/BIOS and make the following changes:
1. Disable Secure Boot
2. Enable Legacy Boot
3. If Available enable CSM
4. If Required enable USB Boot
5. Move the device with the bootable disc to the top of the boot order
6. Save BIOS changes, restart your System and it should boot from the Installation Media

Install Windows 10 on the SSD

When the installation completes, you can re-connect the HDD

Check your Internet connectivity

If any users are like me that used unattend.xml to move the ProgramData folder to another (non-SSD) drive, the WindowsApps program doesn't use the %ALLUSERSPROFILE% environment variable - it is still hard-coded to C:\ProgramData\Microsoft\Windows folder instead of [YOURDRIVELETTER]:\ProgramData\Microsoft\Windows.  To fix this, I made a symbolic link to that folder, mine is [S:]:

ajay k say:

Hi Tazota,

This problem may occur if any one of the following conditions is true:

· The Wmnetmgr.dll file is unregistered.
Note The Wmnetmgr.dll file is located in the C:\Windows\System32 folder.

· The Wmsdkns.xml file is corrupted.
Note The file is Windows Media Player's network namespace file.

· The Wmsdkns.xml file has incorrect permissions.

I would suggest you to try the following steps, to perform these steps you need to run the computer as an administrator

Step 1:Register the wmnetmgr.dll file in elevated command prompt.

a)   Click on  Start > Type in cmd in the search box

b)   Right click on  cmd that appears on the start menu under programs and hit   “Run as administrator”

c)   Now run the command  regsvr32 wmnetmgr.dll

Step 2:Delete the Wmsdkns.xml file

Location of the file

C:\Users>your username>AppData>Local>Microsoft>Windows Media>12.0 folder

After you delete it, Windows Media Player will re-create the Wmsdkns.xml file. You may restart the computer once and check to see if the issue persists.
