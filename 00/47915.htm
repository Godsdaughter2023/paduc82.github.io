¡firedog say:

When the name of a virtual folder in Windows Live Mail is taken into use as a real folder name in the file system, it is truncated to 15 characters long. This isn't corruption; it's just the way the software is programmed. The ordinary user won't be aware of this anyway - the store folder is hidden by default.

  • If you use the Export function to back up your messages and folders, each folder contains a file giving the folder name as used in the program. Using Import against such exported folders would put messages into their original virtual folders in Windows Live Mail.
  • If you back up by any other method, then you'll only have the (possibly truncated) folder names to go by. Importing from any such backup would put all the imported folders and messages into a folder called Imported folders under Storage folders.

If this is only a safety backup in case of data loss, then it shouldn't make much difference what the folders are called. If it's an archive - that you intend to use to read, search and sort messages - then exportation, or drag-and-drop, would be the preferred method.

¡firedog say:
... not all folders are corrupted from the 12th character

If you look a bit closer, you'll see the rule:

  • If the folder name is 15 characters or less, leave it as is
  • Otherwise, take the first 11 characters, then add a space and three random hex digits.

One exception: if there would be more than one folder with the same name, the latecomers will have a space and serial number appended in parentheses in addition to the 15-character name. I've never gone so far as to see whether the serial number is decimal or hex.

¡firedog say:

... which way do you recommend to backup emails in WIndows Live Mail 2012?

Again, that depends on what the purpose of this backup is. It also depends to some extent on your OS. Windows 7's Previous versions feature using a volume shadow is a great backup for minor incidents. As far as I can tell, this feature doesn't exist in W8 except in watered-down form.

  • If it's only for security, so that you still have all your messages in case of accident on your computer, then I would simply copy the store folder to a safe place. Any smart copy utility should be able to transfer only new and changed files.
  • If it's to be able to transfer folders and messages to a new installation elsewhere, then I'd use Export. This wouldn't be a nightly exercise.
  • If it's to create a searchable archive, then drag-and-drop might be preferable. The other methods leave all messages with cryptic numerical filenames, whereas drag-and-drop substitutes the message subject for the filename.

Keeping the store folder in your Documents folder may be the best way of ensuring regular security backup. You might even consider putting it in your OneDrive folder. This would provide another real-time backup in the cloud.
