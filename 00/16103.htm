moranyan say:

Hi Stuart,

OWA junk filter is a mailbox level function, the EOP anti-spam policies are applied to all mailboxes in your organization, which means EOP anti-spam policy is an organization level function. When an email is sent to you from outside your organization, the email will first pass the EOP anti-spam filter such as connection filtering, content filtering, etc. After it passes the EOP, it will pass the OWA junk filtering. Finally it will be delivered to your inbox after passing all the anti-spam filtering.

Regarding “So, it would seem that I can bypass EOP policy setting by enabling or disabling the automatic junk filter in OWA”, actually, the answer is no, EOP filtering is in front of the OWA junk filtering. So, you are not able to bypass EOP policy setting by enabling or disabling the automatic junk filter in OWA.

About “Would the EOP settings then still apply”, the answer is yes, if you only use Outlook rather that OWA, the EOP settings will still apply because EOP filtering is in front of the mailbox-level junk filtering.

technet.microsoft.com/.../jj723119(v=exchg.150).aspx

If you are using POP3, outlook won't use the server filter - it will often filter mail tagged as spam, but that's because it matches outlook's junk filter too.
