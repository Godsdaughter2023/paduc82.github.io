Tony: I don't know the answer to the original question, but perhaps some clarification will help.

With embedded objects, the container (word) has to maintain displayable/printable views of the object in case, for example, the document is copied to a system that doesn't have the server for the contained object (such as Excel). If you look inside the .docx, you see that this image is saved in Enhanced Metafile (.emf) format.

If this were a .doc + .xls, then how that works is that Word is saving its document in a Structured Storage file, and (AFAIK) it creates a Storage within that file and gives it to Excel to store its data. AFAICR Excel is responsible for storing its file (i.e., a .xls) within that Storage, but it is Word's responsibility to get and store the display/print renderings in its own Storage(s).

I don't know how that translates when were in .docx/.xlsx territory, but it sounds more to me like Word is doing its work properly, but Excel is not saving updates to its Storage (or whatever it is).

I wonder if there could be a "trust" problem?
