ken sheridan say: As Scott pointed out, a variable is not exposed to a query.  You have to do so via a function.  In your case you'd declare the vMID variable in the declarations area of a standard module with, assuming the Manager ID column is a number data type:

Public vMID As Long

Then in the same module create a function:

Public Function GetMID() As Long

    GetMID = vMID

End Function

In the query call the function by putting the following in the 'criteria' row of the Manager ID column in design view:

GetMID()

However, this is an unnecessarily complex way of going about it when you can simply open a form (or a report for that matter) bound to an unrestricted query as Scott describes, filtering it to the value of the variable by passing this as the WhereCondition argument of the OpenForm or OpenReport method.  In any application worth the name data should only be exposed to the user via forms or reports, never as a raw datasheet of a table or query.
