doug robbins - mvp office apps & services (word) say:

In Excel 2016, for an .xlam to be loaded automatically when Excel starts, it needs to be saved in 

C:\Users\[User Name]\AppData\Roaming\Microsoft\Excel\XLSTART

If you have it in

C:\Users\[User Name]\AppData\Roaming\Microsoft\AddIns

which is where you would have had it with Excel 2007, while it will appear to have a tick mark against it in the AddIns dialog, suggesting that it has been loaded, that is not really the case and you would need to untick it, close the dialog and then tick to get it to load.

This change results from a security patch, released in July 2016 as a result of which Microsoft changed the behavior of Excel so that it will not load certain file types (including .xlam) when they are untrusted, or in an untrusted location.

While you could set the Microsoft\AddIns folder as a Trusted Location, seeing as how there is now an XLSTART folder, similar to the Word\STARTUP folder, which is where Word Add-ins are stored, you might as well use it.

Strangely, an Add-In that is saved in the XLSTART folder, while it is loaded, does not appear in the File>Options>Add-ins dialog, or in the Developer tab>Excel Add-ins dialog.
