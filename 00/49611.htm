a. user say:

Hello Jersey,

Thank you for posting your query in Microsoft Community.

Ideally, the 350 MB system reserved partition contains the boot files. Disk Manager does not allow you to remove this partition because removing this partition could cause the system to not boot. The installer creates a 350MB System Reserved partition to hold the boot files and install Windows on a C: partition. This primary partition is always 'C' and cannot have any other drive letter.

The operating system uses the Boot Configuration Data stored (BCD Store) in this partition to boot from. If the partition were deleted you would be unable to boot to Windows. Bitlocker (if you are using it) also makes use of this partition.

This partition must be the first partition restored to either a new or existing drive, usually followed by the C: Windows partition.

So, I would suggest you to once verify in the Disk Management if the System Reserved is available.

  • Press Windows key + X, click Disk Management.
  • Check if you have the System Reserved partition listed (you may also get the name System for the System Reserved partition).
  • If the partition is already available, check if you find the difference between the Available capacity and free space in the partition. If so, the partition hasn't been formatted.

If the system is working fine, it states that the System Reserved partition has not been harmed yet and system health is restored.

Hope this information is helpful. Do let us know if you need any further assistance, we'll be glad to assist you.
