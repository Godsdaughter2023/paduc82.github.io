Hi Clyde:

Phillip's reply is quite possibly the problem.

Like you, I suspect there is nothing wrong with the disk.

The problem we have is that Mac OS X does not return an informative error when a write fails.  It returns the computer equivalent of "Nup!".  Which makes it very difficult for the calling application to figure out what is wrong. 

It is usually a deadlock (two or more users attempting to access the same file).  Note that the Mac user must be authenticating to the PC (the user must be logging in to the PC) otherwise these kinds of deadlocks are quite likely.

It can also be that an un-patched copy of Word 2007 wrote into the file some illegal XML. Word 2007 can save it just fine, but nothing else can.  The cure could be to open the file on a patched version of Word 2007 or 2010, make a small edit and save and close it.  That should correct the error.  This is rather clutching at straws, since the bug that caused this was patched two or three years ago.
