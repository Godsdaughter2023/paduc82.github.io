john sr wilson say:
When you say beside it - it has to be in front or behind.

Assuming it is selected (no error chex)

Something like:


Sub Dupe()
Dim oshp As Shape
Dim zPos As Long
Set oshp = ActiveWindow.Selection.ShapeRange(1)
zPos = oshp.ZOrderPosition
With oshp.Duplicate
While .ZOrderPosition > zPos + 1
.ZOrder (msoSendBackward)
Wend
End With
End Sub

If you know the name of the shape you could just change the reference for oshp
Set oshp = ActiveWindow.Selection.SlideRange(1).Shapes("myShape")

roady [mvp] say:

Outlook gives that warning message when an add-in or application wants to send an email or access Outlook data in an unexpected or insecure way.

Initially, it is up to the add-in developer to properly integrate his/her solution with Outlook.

Which virus scanner do you have installed? If it reports its status to Windows and its status is "Valid" then Outlook 2013 will only show that prompt in very rare cases.

To see whether your virus scanner status is "Valid", in Outlook go to:
File-> Options-> Trust Center-> Trust Center Settings...-> Programmatic Access

When it reports "Valid", the (greyed out) settings above don't matter.

As an add-in developer, you can work with Redemption to prevent triggering such prompts.

As an end-user, you can install MAPILab Advanced Security to mark specific applications as Safe so they will no longer trigger that security prompt.
