zorvek say:
First of all, the only way I know of that you can get a #REF! is if the tab names are not as represented in your formula. So please check again for leading and/or trailing spaces or other characters.

Second, if you are looking up a value associated with the value in column A of MLD, then you need to include more columns in the look up range and specify the column containing the desired value. For example, if the desired value is in column C then:

=VLOOKUP(IDMS!C1995,MLD!$A$1665:$C$2988,3,FALSE)

Resolved my Start Menu, Cortana, Project Spartan Problems

First, I'm going to say, It's a doozie. A lot of you probably aren't going to like it.

Back story

Before I jump into how I resolved my problem I want to give some back story leading up to the fix. (If you don't want it you can jump to the Resolution Section, but the backstory might help ensure your situation is similar).

Like many of you, my experience started with Windows 10 Preview back in October. I didn't use it for too long but did play with some of the 98** builds. For what they were I didn't have any issues. I reverted back to Windows 8.1 Enterprise back in November 2014. In January I re-engaged myself into Windows 10 TP with 9926. I've been running Windows 10 TP Enterprise ever since.

I noticed in build 10041 that Cortana was having some fits getting me to show her UI to do searches. That never did get resolved during that build. Everything else was working as expected. When build 10049 came along, I started noticing that Cortana's voice wasn't playing (at all). I would still receive her UI and reminders but when I would say "Hey Cortana ...." she would only respond with text in her UI, no voice response."

After build 10049 came, (Project Spartan FTW) Cortana no longer functioned. I was still able to get searches working but that was it. No reminders, nothing. Not even if I typed them out the way she likes it. Project Spartan worked just fine although crashed a lot. Lets get to build 10061.

After Build 10061 was installed I noticed Project Spartan was no longer functioning. When I would open it, it's loading UI would appear only long enough to see that it would show up but would crash before the brain could process it's existance. (200ms maybe?). After several attempts to "repair" it thru powershell, I decided to try the Profile method thats been recommended by everyone. So I created another profile (admin) and proceeded to login to it. Project Spartan started working! However, soon after trying I noticed that my start menu wouldn't open when clicking the start button. Cortana also wouldn't work. So I tried some other powershell scripts that were recommended for the start menu on the forums but still had no resolve. After re-creating another user account (this time local, admin) and still receiving the same result I decided a refresh was in order. Unfortunately the refresh provided no resolution to the problem. I did a few more "work from home" things (running things from task manager is hard work if you can't remember where everything is at. lol) and before going to bed decided a hail mary was in order. I decided to do a Factory Reset on Windows 10. I want to be clear, I did not reinstall 10049 from ISO then update to build 10061. I went to Settings > Recovery > Remove Everything and reinstall Windows. Why? Because I thought that there might be a upgrade issue between builds. If there was, I didn't want those issues to re-surface.

Results of Findings

During the course of trying to find a fix for the problem I was able to come up with these findings:

1. After trying to use the "Create new Account" method, I determined that the Project Spartan issue is a profile issue.
2. After using the "Create new Account" method, I determined that the start menu and cortana problems may be related to new account creation. It's possible that systems being upgraded between builds might cause the system to corrupt new profiles during creation (starting ISO build was 9926, final build upgrade thru MU was 10041-10061)
3. Start menu and Cortana issues (after build 10061 upgrade) are system issues, not profile issues. This was determined since a new user account was created which caused the issue to surface. (profiles that were created from previous build that were upgraded to build 10061 were not affected).
4. Refreshing the system does not resolve the issue. It seems the affected system components are not reset during this process.

Resolution

In order to resolve the problem the use of "Remove everything and reinstall Windows" must be used. It's important to say that this was only tested on build 10061. Furthermore, I should say I did not reinstall Windows 10 TP from an ISO and upgrade to build 10061. This was run directly from build 10061.

1. Back up all of your data (because you will lose everything).
2. Open settings. If you cannot do this follow "Alternate Resolutions steps" section.
3. Go to Update & Recovery
4. Choose "Recovery" on the left.
5. Under "Remove everything and reinstall Windows" click "Get started".
6. Follow the prompts.

1. Back up all of your data (because you will lose everything).
2. Right-click the start menu button. If you cannot do this follow "Alternate to the Alternate Resolution steps" section. (Yes, that was a "The Office" reference).
3. Hover over "Shut down or sign out". A sub-menu should appear.
4. Press and hold "shift". Click restart.
5. "Click Troubleshoot".
6. Follow the steps on screen to perform "Remove everything and reinstall Windows".

Alternate to the Alternate Resolution Steps

2. Open task manager by pressing [CTRL] + [ALT] + [DEL] and clicking "task manager".
3. click the file menu then click "Run new Task"
4. Type "Control Panel" into the box and click ok.

6. Click "User Accounts".

8. In the settings window click the Back arrow next to "Accounts".
9. Click "Update & Recovery".
10. Click "Recovery" at the left.
11. Under Remove everything and reinstall Windows click "Get started".

Actually, Publisher is poorly suited to "publishing documents" since it is page layout rather than text flow software. Using large quantities of text (as in a book or report) is much more complex in Publisher than in Word.
