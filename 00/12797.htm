First - I am not a Visual Basic developer. That said, is there a way that I can generate a calculated value based on the total number of pages in my document. greg maxey say:

Greg Maxey --- Visit my website at: http://gregmaxey.mvps.org/word_tips.htm

jay freedman say:

There are two ways to approach this. Each has its advantages and
drawbacks, so pick the one that makes the most sense to you.

(1) You can do arithmetic in a field, subtracting 1 (if it's just the
cover page) or another number from the NUMPAGES field with a syntax
like

{ = { NUMPAGES } - 1 }

To make a nested field like this, you must use Ctrl+F9 to make each
pair of field braces.

The advantage of this is simplicity -- it's once and done. The
drawback is that if you add more frontmatter, you have to change the
field to subtract a different number, which can be difficult if the
frontmatter includes a table of contents that may expand
unpredictably.

(2) You can put a bookmark on the last page of the document, and
replace the NUMPAGES field with a PAGEREF field pointing to the
bookmark. It will display the page number of the last page, whatever
that may be.

The advantage of this is that you can ignore expansions of the
frontmatter. The drawback is that bookmarks are notoriously easy to
overwrite or delete, which will cause the PAGEREF field to display an
error message when updated. Or you could inadvertently add more text
after the bookmark, which would make the PAGEREF show the wrong
number. It can help to check the option to display bookmarks.

Jay Freedman
