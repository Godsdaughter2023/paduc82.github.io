Filtering a query used by DoCmd.TransferSpreadsheet - Microsoft Community

emerald77 say:

Filtering a query used by DoCmd.TransferSpreadsheet

One nice thing about using DoCmd.OpenQuery is that you can then execute DoCmd.ApplyFilter WhereCondition, and the query will be filtered based on the value of WhereCondition.

Unfortunately, DoCmd.TransferSpreadsheet does not have a WhereCondition parameter, and DoCmd.ApplyFilter WhereCondition cannot be used with it.

Does anyone know how I can somehow "pass" a WhereCondition to DoCmd.TransferSpreadsheet?  Thanks in advance for any assistance.
hansv mvp say:

Yes, you can get the SQL string of a query, but unfortunately, DoCmd.TransferSpreadsheet won't accept a SQL string in its TableName argument, only the name of a stored table or query.

You could create a temporary query just for export, then delete it again:

 

    Const strTemp = "TempQuery" ' or whatever you like
    Dim dbs As DAO.Database
    Dim qdf As DAO.QueryDef
    Dim strSQL As String
    Set dbs = CurrentDb
    ' Get the SQL of the existing query
    Set qdf = dbs.QueryDefs(strReportName & "Query")
    strSQL = qdf.Sql
    ' Now modify strSQL
    strSQL = ...
    ' Create a new temporary query with the modified SQL string
    Set qdf = dbs.CreateQueryDef(Name:=strTemp, SQLText:=strSQL)
    ' Export the temporary query to Excel
    DoCmd.TransferSpreadsheet TransferType:=acExport, _
        SpreadsheetType:=acSpreadsheetTypeExcel12XML, _
        TableName:=strTemp, _
        Filename:=strFilePath & ".xlsx"
    ' Delete the temporary query
    dbs.QueryDefs.Delete strTemp

 

Edited to remove QueryDefs.Append line.

hansv mvp say:

You could apply the filter in the query definition by changing its SQL programmatically.

 

Or open the query and filter it, then use DoCmd.OutputTo. If you specify acOutputQuery as object type, but omit the object name argument, it will output the currently active query.

Disadvantage is that this exports to an old-fashioned Excel format.

    Pointing Office 365 to new ADFS

    We have an existing hybrid configuration with Azure and use the on-premise ADFS (adfs.domain.com) for federation with Azure. We would like to point Office 365 to our new ADFS Server (newadfs.domain.com). Looks like we just need to run the below commands:


    1. Connect-MsolService –Credential $cred

    2. Set-MsolADFSContext –Computer newadfs.domain.com


    Is there anything else we need to do to point Office 365 to our new ADFS server ? Thank you in advance.

    vasil l. michev say:

    Those cmdlets are not enough. You will also need to update/recreate the federation trust with O365. This can be performed via the following cmdlet:

    Update-MsolFederatedDomain -DomainName domain.com

    Some variations might be in order, depending on your configuration.

    prashanthpalakollu5136 say:

    Vasil's original suggestion worked perfectly ! We ran the below three commands and waited for few minutes for the changes to propagate...

    1. Connect-MsolService –Credential $cred

    2. Set-MsolADFSContext –Computer newadfs.domain.com

    3. Update-MsolFederatedDomain -DomainName domain.com
