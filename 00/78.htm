ken sheridan say: I suspect you may have fallen victim to something which I think Microsoft introduced into the Access report designer with Access 2007.  When you group on a date/time field it allows the grouping to be by quarter, month etc.  In my copy of Access 2007 it defaults to quarter.  You want it to group by day, so in report design view open the grouping window and select the group, then select 'by day' as the group interval.

amjad ali siddiqui say: Thanks a lot. The problem was exactly as you had identified and now stands resolved. Now please help in another problem.
I want to use parameter in Order Date field in a query linked to a report so that I get a print out of the desired date only. ken sheridan say: There are several possible causes of the problem:

1.  It could be that the parameter is being misinterpreted as an arithmetical expression.  This can be avoided by declaring the parameter in the query.  If you open it in design view and then switch to SQL view, you can add a PARAMETERS clause to the start of the query, so it would begin like this:

PARAMETERS [Please enter the Date] DATETIME;
SELECT....etc

The query's WHERE clause would be something like this:

WHERE [Order Date] = [Please enter the Date]

Even if this is not the cause of the problem it is always best to declare parameters, especially those of date/time data type.

2.  Another possibility is that the Order date column's values contain date/time values with a non-zero time of day.  This can easily happen without you realizing it, a common reason being the inappropriate use of the Now() function to automatically insert the current date; it in fact inserts the current date and time.  If you are using the Orders table from Northwind it does this in my 2007 version, which is a bad piece of design by the Microsoft developers (not the only one in Northwind I'm afraid).  This can be overcome by firstly setting all the values in the column to dates with a zero time of day by executing the following update query:

UPDATE [Orders]
SET [Order Date] = DATEVALUE([Order date])
WHERE [Order Date] IS NOT NULL;

Then in table design view add the following Validation Rule to the Order Date column:

DateValue([Order Date])

Then set the Default Value to Date() rather than Now().

I suspect 2 is the cause of the problem, but if neither solve it copy the query's SQL statement and paste it into your reply here.
