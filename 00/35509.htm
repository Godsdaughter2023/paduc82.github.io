a. user say:

Hi,

I can understand the inconvenience that you are facing at the moment. If you receive a prompt to change the Windows password at a regular interval, it could happen if the Password policies under Local Security Policy is enable to change the password after a certain number of days.

Here’s the explanation about the options that are present under Password Policy:

Enforce password History: Prevents users from creating a new password that is the same as their current password or a recently used password. To specify how many passwords are remembered, provide a value. For example, a value of 1 means that only the last password will be remembered, and a value of 5 means that the previous five passwords will be remembered. You can set a value of greater than “1” for this option. We will discuss further how to change the value.

Maximum password age: Sets the maximum number of days that a password is valid. After this number of days, the user will have to change the password. Set the same value (0) for this option as you don’t want Windows to remind you to change the password. You can change the password whenever you want.

Minimum password age: Sets the minimum number of days that must pass before a password can be changed. If the minimum password age is set to 0, the user can change their password six times on the same day and begin reusing their original password on the same day. Set the minimum password age to at least 1 day.

There are other options that may not be related to the issue. But, here’s an article that will provide you all the insights regarding the options:

http://windows.microsoft.com/en-us/windows/change-password-policy-settings#1TC=windows-7

Note: The explanation for the policy settings in the article also apply for Windows 8.

Now, open Local Security Policy by opening “Run” (Windows+R keys) and typing “secpol.msc”. Hit Enter. I would suggest you to change the option for “Maximum password age” to “0” if you don’t want Windows to prompt you to change the password again (or you want to do it manually). You can also set a value of “70” days as recommended in the article and you can change the password after 70 days.

Hope it resolves the issue.

Please feel free to let us know if the issue still persists. We will be glad to assist you further.
