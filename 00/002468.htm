terfar say:
File Locations settings are found in Word Options, Advanced tab at the very bottom. Your template should be in a Work Group folder on the server (the folder can be called anything you like, but Workgroup Templates or something similar is an obvious choice). All the users should point their File Locations, Workgroup Templates to the location on the server. When they start Word and go to File New, the templates in that folder will be available to them in the same way that their local templates are available.
 
When a user opens a new document from the template, the template will be locked for access on the server. This will not stop other users from opening a new document from the template, but it will stop anyone from editing the template. There is no reason for anyone to want to open the template, but making it write protected will ensure that no one accidentally alters it.
 
To write protect the template is the same as write protecting any document you create in Word. File, SaveAs (F12) slick on Tools, General Options and add a password in the Modify box.
 
But as I said before, you will not be able to edit the template if anyone on the network has a document open that was created from the template. So you need to edit a copy of the template and then quickly substitute when everyone is not using the template.
 
 
-- Terry Farrell - MSWord MVP

jllatham say:

Why is Next used without a reference?

This is actually a way to improve performance.  The VBA engine keeps track internally of the sequence of such loops and it can actually more rapidly find the appropriate value for Next if it is not referenced than if it is.  If you do reference it, then the engine has to look it up in the internal table and apply the appropriate value.

I normally leave the reference off of them but will add a comment at the Next line to help with examining or debugging the code later; for example:

For X = 1 To 100

   For Y = 23 To 2 Step -1

    'do some work

   Next ' end Y loop

Next ' end X loop

The UpperLeft object: this does seem to be a left-over, unused variable in the code.  Perhaps from an earlier version of the code where it was used.  I suspect that it may have been used to determine where to paste the selection to be copied in an exact fashion.  It ends where it begins: it was a one-cell reference.

The reason for the subtraction is to get the positioning for the paste correct.  But you are correct, 10-10 = 0 just as 44-44 = 0.  You could test this theory by changing line 44 to become simply:

SelAreas(i).Copy PasteRange  (but I don't think that'll work properly - see next paragraph).

But we need to look closer at what TopRow and LeftCol actually are when the loop begins, as earlier in the code those are gradually set to the top-most and left-most row/column of any of the selected ranges - in effect giving you the UpperLeft location of where any given selection is to be pasted, combined with the offset it will position each individual copied selection to the right and below that location by some number of rows/columns.
