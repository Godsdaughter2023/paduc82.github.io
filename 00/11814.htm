merging spreadsheets

teylyn say:

Hello,

there is no such thing as an .xml spreadsheet.

.xls is the file format of Excel 2003 and earlier. It can contain macros or not. The file extension does not tell.

XML is a data storage format. Nothing to do with Excel.

.xlsx is the file extension of Excel 2007 and later Excel workbooks without VBA code

.xlsm is the file extension of Excel 2007 and later Excel workbooks that are VBA enabled

.xlsb is the file extension of Excel 2007 and later files in binary format and may or may not contain VBA macros.

Yes, it is possible to combine workbooks of different types. Open both workbooks in Excel, move or copy the sheets you want to combine into the target workbook. 

If the target workbook contains VBA code it must be saved as a macro-enabled workbook with the .xlsm extension (or alternatively as .xlsb)

Does that answer your question?


julievandermeulen1 say:

ok kind of.

So the only way i can merge the .xml file with a .xls file is to copy the .xlm sheet to a sheet in the .xls file and then merge them?

teylyn say:

Sorry, but your use of the wrong file extensions is confusing. There is no .xlm and there is no .xml when it comes to Excel file names. 

If you want to merge a macro-enabled file with the file extension .xlsm with a macro-free file with the file extension .xlsx, then the easiest way is to copy or move all sheets from the macro-free workbook to the macro-enabled workbook. 

You CAN do it the other way around, but then you also have to copy the code modules from the macro-enabled workbook to the macro-free workbook, and you will have to store the previously macro-free file as a macro-enabled workbook with the .xlsm extension. 

From .xlsx to .xlsm is faster. Just select all the sheets, right-click, move or copy, select the other workbook and confirm. Then save the other workbook.

There are three possibilities here:

1. A faulty Office installation;

2. A corrupt document; and/or

3. A corrupt table.

Try repairing the Office installation (via Start > Windows Control Panel > Programs > Programs & Features > Microsoft Office (version) > Change).

Corrupt documents can often be 'repaired' by inserting a new, empty, paragraph at the very end, copying everything except that new paragraph to a new document based on the same template (headers & footers may need to be copied separately), closing the old document and saving the new one over it.

Similarly, corrupt tables can often be 'repaired' by saving the document in RTF format, closing the document then re-opening it and re-saving in the doc(x) format
