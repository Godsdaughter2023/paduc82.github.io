Run time and compiled have different meanings. In the context you appear to be using it Run time means being able to run an Access application without having a full Access License. microsoft distributes a free Run time version of Access. This version can be used to run any Access app (assuming the proper versions), even if the user doesn't have a full licensed version of Access. it doesn't matter whether the file is mdb, mde, accdb, accde or any version of the Access format. There is a version of the file format (accdr), that forces a full license to run as if it were a runtime license. This is mostly for developers to test with. An accdr is created simply by renaming the file.

Compiled, on the other means converting an Access app to an mde/accde format. This conversion compiles the code within the app and then removes it so the user can't get into the design of forms, reports or any code modules. This is done mostly to protect the design from prying eyes.

scottgem say:

First splitting of the database has nothing to do with the user interface (UI). Splitting has to be done when you are ready to deploy, but you can leave it as one file until then.

You don't purchase software, you purchase a license to use that software. The terms of that license are spelled out in the End User Agreement that almost no one reads, but everyone agrees to before the software will install. A full license for Access means that you can do everything to create, and develop applications. The runtime license only permits running an application with no access to development tools.

Compiling, in Access terms, means to convert the code in an application so that it can no longer be accessed by a user. So a user of a compiled app cannot get into Design mode of most of the objects and cannot access the VBA ormacro code. You do not need to purchase a compiler. Using the Publish and Send functionality you simply convert to accde.

To hide the ribbon use the VBA code:

DoCmd.ShowToolbar "Ribbon", acToolbarNo

I'm not entirely clear whether you want:

  1. The footnote reference left-aligned, with footnote text beginning at 0.25" from the left margin.
  2. The footnote reference indented 0.25" (i.e., a first-ilne indent).
  3. The footnote text hanging, with a left indent at 0.25", footnote reference flush left, and footnote text starrting at the 0.25" indent.

kathleen orland say: Outlook 2010 - Empty The Deleted Items Folder

This article explains how to empty the Deleted Items Folder in Outlook 2010.

Items that you delete are moved to the Deleted Items folder, but are not permanently deleted until that folder is emptied. Over time, the contents of the Deleted Items folder can consume a large amount of your data file or email mail account server storage quota.

The Deleted Items folder appears in the Navigation Pane when you are using the Mail or the Folder List views. Outlook can be configured to automatically empty the Deleted Items folder, or you can manually empty the folder at any time.

Automatically Empty the Deleted Items folder
  1. Click the File tab.
  2. Click Options.
  3. On the Advanced tab, under Outlook Start and Exit, select the Empty Deleted Items folder when exiting Outlook check box.
  4. To be notified before the Deleted Items folder is emptied automatically, on the Advanced tab, underOther, select the Prompt for confirmation before permanently deleting items check box.
Manually Empty the Deleted Items Folder
  1. To open the Folder List view, at the bottom of the Navigation Pane, click Folder List or press CTRL+6.
  2. Right-click the Deleted Items folder, and then click Empty Folder. You can also do this procedure inMail, in the Navigation Pane.

dim wks as worksheet

If turn off just means that you show all the data and keep the arrows:

dim wks as worksheet
set wks = worksheets("Somesheetname")
with wks
   'show all the data
   If .FilterMode Then
      .ShowAllData

end with
