Hi Sam,

Did you use the OneDrive sync client on your Windows 10 computer? If so, it’s an expected behavior that local OneDrive folder will use your hard drive space. Because the sync client will sync files from the OneDrive.com to your local OneDrive folder. At the same time, it will also sync your files in local folder to the cloud. To avoid it, I suggest you unlink the OneDrive sync client by right clicking OneDrive icon in the task bar> Settings> Account and clicking Unlink this PC under “OneDrive – Personal”. Then, delete the local OneDrive folder. After that, every time you need to back up your files, login to OneDrive.com and just drag and drop your files into the browser.

I have a question about using multiple folders for Outlook for Mac.  I'm considering moving from PC to Mac.  I use multiple current psts to manage large amounts of emails.  I create and maintain separate psts for each project file that I handle so that I have permanent correspondence folders that can be accessed and searched later.  I probably have 250 of them, with about 50-75 open at any given time.  This enables me to avoid the risk of crash from an excessively large single pst and also to compartmentalize the potential damage from a crash of any one individual pst folder.  I separately back up all psts and then when a project file is completed, I close the particular pst.  It also is important for me to be able to retain permanent records of the particular chain of correspondence in each of my separate project files, and I often have to go back, sometimes years later, to search them again.

 

The question I have is whether there is a way to similarly structure Outlook for Mac.  I know there are no psts in Outlook for Mac.  It looks to me from the discussions that I have seen in the various forums that while you can create multiple folders "on my computer," those are not recommended as permanent long term file storage.  I also cannot tell if there is a maximum size/risk for folder size within Outlook for Mac. 

 

Thanks,

 

Peter

 

cody c. say: Hi Peter,

You're correct that there's no way to attach PST files. There's also not a way to attach OLM files (the PST equivalent in Outlook for Mac). It's only possible to import these files, which will create a separate folder within Outlook with the contents of the OLM file.

Keeping folders inside On My Computer is going to be your best solution. It's just the local database for Outlook and there's no issue with keeping files in there long term. I don't believe there's a published maximum database size, but as long as you're under 10 or 15 GB you should be fine. Keep a backup as a safety precaution, but Outlook should handle the size just fine.

Install Windows to C Drive & Data to D Drive partitions at Install - Microsoft Community

djrd say:

Install Windows to C Drive & Data to D Drive partitions at Install

Will Windows 10 allow the installation of Data to a separate partition on install on PCs?

On a previous laptop with a 1 GIG HD I set up a second D: DATA partition and installed all my data there. I moved the folders over, but it was tedious and created issues with programs like iTunes.  Recently I purchased a new laptop with a 256 GIG SSD and didn't bother going through the rigamarole.

I wondered why it is not possible and done at the install?

With hard drive storage becoming immense, and even SSDs getting bigger, I feel this should be an important part of the future.

The main reason? After a quick reinstall of windows, the users and the User's Data are still on the PC and immediately available after the reinstall. The data is not lost as it would be if it were on C Drive with the operating system.

The basic idea is that the Operating system and user's Data remain completely segregated. OS on C Drive (ie the default Hard Drive - mine is called C: CORE) and Data on D Drive (ie D: DATA).

Therefore, at the time of installing windows the user/installer nominates (and creates, if necessary) one or multiple partitions. One partition (usually C:) is nominated as the OS partition. The second, as a Data partition. And in most circumstances nowadays, there will also be reserved, recovery partition that already have the Win10 install built in.

It would go something like this: at the time of Initial all the Admin folders are installed onto D Drive -- All paths and security are automatically set up. Future users are also installed onto D drive. Subsequent install will already have a Path to D Drive.

It might mean that there is still a User Folder on C Drive that Contains the OS's (User) AppData folder etc, but the main data that the User uses is on a completely separate partition. For me this creates much more security and much less stress around the users data, especially when reinstalling Windows.

While it is not a substitute for off PC backups, and catastrophic HD failure it goes a long way to bringing a degree of security at time Windows needs to be quickly reinstalled to C drive -- the users Data will be there on the D partition, untouched by the reinstall.

I know it is possible to go through and move everything over to a D partition, or even to create a D partition and Back Up data there, but it takes time and I'm not so sure the Data is as secure (without encrypting it) as it could be when done this way and as I said earlier it is a PITA.

So, will WIN10 allow the creation of a separate secure Data Partition at Install and automatically find Users at reinstall?
