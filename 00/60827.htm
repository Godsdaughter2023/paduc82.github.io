charrodine say:

I have had this exact problem also and it manifests itself as Mybookworld not being accessible over wifi from either the web interface or via a mapped drive.  Somtimes you can access the top level folders but never go deeper down.  If you try this wired usually you will be OK.

The problem is actually down to MTU size set on the WiFi card and the fact that the Mybookworld can only handle an MTU of 1500 or less.

To fix the problem you need to set the max MTU size to anything less that 1500 on the wifi card.

If you have Windows 7 this can easily be done using netsh command.

to view existing interfaces run

netsh interface ipv4 show subinterfaces

this will show all interfaces and there names - now you need to set the mtu speed on the wifi connection in my case the command would be (note actual MTU is upto you but I find 1458 works well)

netsh interface ipv4 set subinterface "Wireless Network Connection" mtu=1458 store=persistent

Once you run the commsnd th MTU is set and you should be able to access Mybookworld

If you have not got Win7 there are tools in WinXP or you may even be able to do it in the driver under the advanced settings but you would need to look into this.

Why this happens I can only conclude is down to the wifi card not picking up on a max mtu and so sending out jumbo frames which the mybook cant handle

Hope this helps!
