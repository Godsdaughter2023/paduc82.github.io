bill manville say:

Further to Jeeped's reply which I endorse, if you really want to enter time values but leave out the : then you can use a Worksheet_Change event procedure for the worksheet that converts your input.

Suppose that the range of cells into which you will enter time values is named TimeInput.
In the Visual basic editor (Alt+F11 from Excel), in module of that worksheet enter the following

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim C As Range
  If Intersect(Target, Me.Range("TimeInput")) Is Nothing Then Exit Sub
  For Each C In Intersect(Target, Me.Range("TimeInput")).Cells
    If C.Value >= 1 And Int(C.Value) = C.Value Then
      Application.EnableEvents = False  ' prevent second call
      C.Value = TimeSerial(C.Value \ 100, C.Value Mod 100, 0)
      Application.EnableEvents = True  ' prevent second call
    End If
  Next
End Sub

jeeped say:

If you truly want a time input, then you need to use a true Excel time (BASE60) format for input.

However if you simply wish to express hours, minutes and second in a 00:00:00 displayed format while retaining the BASE10 numerical values, use a Custom number Format like,

00":"00

... or,

00":"00":"00

... neither of which will translate to hours and minutes and seconds in Excel's mathematical interpretation of date and time, e.g. you will not be able to accurately add, subtract or rank them. This is not to say that you cannot do some of these tasks some of the time.

Personally, my vote is the true date/time format. For the record, BASE10 inputs can be translated to proper hours, minutes and seconds through formulas and 'helper' columns.

From the Home tab's Number Group, choose More Number Formats, then Custom from the list on the left. Use one of the supplied formats above.

The Page NameU is not displayed anywhere in the Visio 2007 GUI, but running a module with the following VBA code will output the NameU names into the Immediate Window:

Public Sub Pages_Example()

    Dim intCounter As Integer
    Dim vsoDocument As Visio.Document
    Dim vsoPages As Visio.Pages

    'Get the Pages collection for the active document.
    Set vsoPages = ActiveDocument.Pages

    Debug.Print "Page names for document: "; ActiveDocument.Name

    'Iterate through the pages and print the page name
    'in the Immediate window.
    For intCounter = 1 To vsoPages.Count
        Debug.Print " "; vsoPages.Item(intCounter).NameU
    Next intCounter

End Sub

doug robbins - mvp office apps & services (word) say:

I believe that the official limit for nesting of If...then...Else fields is 19 levels, though I am aware of situations where it has been possible to exceed that number, though the exceptions are very much hit and miss and what works with one version of Word on one machine, may not work with the same version on another machine.

You can usually work around the limitation by using a series of IF fields one next to the other. Just have the last "false" entry in a preceding set be an empty string: "" then no one will know that set of IF fields is even there.
