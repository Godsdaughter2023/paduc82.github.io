don cuthbert say:

If you have Windows 10 install media on USB, boot with that (first disable Secure Boot in BIOS/firmware).  From the first screen of Windows setup, use SHIFT-F10 to get a Command Prompt.  The drive letters are not always the same in this mode, but it does not matter, as long as you determine which letter Windows and the Users folder is on.  Normally on a UEFI computer, drive C: has the expected letter, but it often changes on a Legacy BIOS computer in this mode.

You can create Windows 10 install USB using the Media Creation tool from here:

https://www.microsoft.com/en-us/software-download/windows10

Or download a full Insider ISO from: https://www.microsoft.com/en-us/software-download/windowsinsiderpreviewiso and use Rufus to create the bootable USB from the ISO file http://rufus.akeo.ie

You can completely recreate the FAT32 System partition, if necessary, using diskpart, bootsect and bcdboot commands.

Your command line, if it was as you typed here, has a typo.  BCEDIT should be BCDEDIT.  Windows PE creates a RAM drive X: that it runs from, which includes all of the commands that you might need to check your drives and Windows.

sfc /scannow /OFFBOOTDIR=C:\ /OFFWINDIR=C:\Windows

If sfc finds errors that it cannot fix, run DISM:

DISM /Image:C:\ /Cleanup-Image /RestoreHealth

Adjust the drive letter C: above to use whatever letter has Windows and Users folder.

bcdboot C:\Windows

The above should recopy System files and rewrite the BCD.  Also, just entering bcdedit and pressing Enter should give a short summary of the existing BCD.

don cuthbert say:

I have that multi-sign-in issue on this website at times also.  Sometimes I just need to sign-in again to Reply.  Sometimes I have to use CTRL-F5 to refresh the page, or close and reopen the browser tab.

Yes, I always forget the ScratchDir part.  There is not enough space in the Temp folder on the RAM disk, that the Command Prompt is running from, for temp files, so it needs some physical disk space to use instead.

You can just create any folder on F: for that:

md F:\Scratch

Then use the command as it says:

DISM /Image:F:\ /Cleanup-Image /RestoreHealth /ScratchDir:F:\Scratch

However, since sfc did not find any issues, there will probably be nothing for DISM to fix.

It sounds like there is an issue with your System partition, while your Windows partition is OK.  System partition can be rebuilt.  It is pretty easy on a GPT disk, because System and Recovery are separate partitions.  On Legacy BIOS they are usually combined, so you have to relocate the Recovery files before you can delete and recreate the System partition.  System/EFI partition on a UEFI GPT disk is not drive C: and usually does not show up that way.

You can completely rebuild the System partition on a GPT disk with diskpart, bootsect and bcdboot commands.  First, be sure that your computer is in UEFI BIOS mode.  Your USB stick has to be formatted FAT32 to boot UEFI and often Secure Boot needs to be disabled.

Once you are able to boot and run diskpart, you can list volume to see all of the volumes.  If the FAT32 EFI partition has a drive letter, you can remove it by first selecting that volume number and entering remove to take away the drive letter.  But it helps to have a drive letter on it for the bootsect and bcdboot commands.  I usually use drive S: for the EFI partition and change the one I know to be C: from whatever letter it is to C:.  If C: is in use, I first select that volume, remove the volume letter and then assign C: to where it should be. The FAT32 System/EFI partition is normally only 100MB in size.

Once you are sure that you have selected the 100MB FAT32 volume, it can be deleted. If you select it and then list volume it will show an asterisk beside the selected volume.  list partition will also then show an asterisk beside the EFI partition.  Since the plan is to delete it, be sure you have it properly selected first.

If you want to first copy data elsewhere, such as to another drive or to a USB stick, you can do that from the Command Prompt too, but deleting the System partition does not delete anything from the Boot/Windows partition.

Once you can see that the 100MB System/EFI partition is selected in diskpart by the asterisk, you can delete it.  delete partition override

Now it can be recreated:

create partition EFI

format quick FS=FAT32

assign letter=S

Now you have a new, empty, FAT32-formatted EFI partition at drive S: and can exit diskpart.  Make sure that some drive letter is assigned to the EFI partition, such as S: and also a drive letter is assigned to the desired Windows partition, preferably letter C:, although it doesn't matter the letters used in this mode, as long as you know what partition/volume the letters refer to.

To rebuild the EFI contents run bootsect and bcdboot:

bootsect /nt60 S: /force

bcdboot C:\Windows /s S: /f UEFI

Now run bcdedit and it should show you a proper BCD contents pointing to C:\Windows.  If your drive letters are different than C: and S: use your letters.

bcdedit

All of these commands have help available, so you can understand what they do and the correct syntax, with bcdboot /? or bootsect /help.

Assuming this is all successful and works as planned (and your Windows volume is good, as it appears to be from sfc) you might be back into Windows after a Restart.
