Activation depends on what type of license you have of Windows 7/8/10.

If Windows 7/8/10 came preinstalled with the computer, that is called an OEM license. For an OEM license, the license dies as soon as Motherboard dies. So essentially it is non-transferable. If that is the case you need to get a new license.

Run the activation troubleshooter on the new device using the below steps:
Go to Settings>Update and security>Activation and run the troubleshooter. and select I recently changed hardware on this device.

      bethany rogers say:

      I know this is an old thread, but just in case someone is still looking for an answer and hasn't found it yet...

      Typically this means OneNote "Side Note" has started. SideNote is a specific component of OneNote (versus the entire program).

      If you want to completely stop it from automatically launching at startup, you can use your computer's System Configuration tool.

      There are 2 ways to do this:

      Using "msconfig.exe" to access Startup Configuration:
      1. Click on the Start button
      2. In the Search box type: msconfig.exe
      3. Then hit "Enter", or you can click on "msconfig" in the search results list
      4. A new window should open called System Configuration
      5. Click on the Startup tab
      6. Look for "Microsoft OneNote" in the item list.
      7. Un-check the box next to Microsoft OneNote. This will prevent it from automatically launching at startup. You can still open OneNote manually any time after startup.
      8. Click the Apply and/or OK button
      9. Restart your computer to confirm your desired setting was applied.

      OR

      Using the Control Panel to access Startup Configuration:
      1. Click the Start button
      2. Click on Control Panel (on the right side of the Start Window, where there's a blue background)
      3. Click the "System and Security" category. (If the control panel is in "icon" view, skip this step and go to step 4 below)
      4. Click on Administrative Tools
      5. Click on System Configuration
      6. Then follow steps 4 through 9 on the list above.

      Here's a really good help article, too, including screen images for each step:
      http://www.elmajdal.net/win7/Disable_Enable_Windows_7_Startup_Programs.aspx

      Hope this helps!

      When you upgrade to Windows 10, the previous Windows files and folders are automatically stored in C:\Windows.old folder. This folder is kept by Windows only for one month. After one month Windows 10 will automatically delete the folder. So you can restore previous Windows version only within one month after Windows 10 upgrade. Also if you run Disk Cleanup in Windows 10, make sure to uncheck "Previous Windows installation(s)" option.       Explorer.exe WILL NOT stay pinned

      I recently updated to the Anniversary update. Most things are alright, just had to reinstall the sound drivers that windows decided I didn't need.

      However, one problem persists: explorer.exe will not stay pinned to the taskbar. All other shortcuts remain pinned

      The steps to reproduce the issue, at least on my hardware, are to

      1. Pin explorer.exe as normal via a right click > pin to taskbar
      2. Close explorer
      3. Restart explorer.exe
      4. Open explorer
      5. Browse to %APPDATA%\Microsoft\Internet Explorer\Quick Launch\User Pinned\TaskBar
      6. Explorer will be unpinned shortly if not instantly
      • A simple reboot will also unpin explorer.exe, but having to constantly reboot while troubleshooting is slow and cumbersome

      Fixes I've tried

      • Deleting %APPDATA%\Microsoft\Internet Explorer\Quick Launch\User Pinned\TaskBar and allowing windows to rebuild it
      • Deleting the "NoStartPage" key in HKLM\SOFTWARE\Classes\Applications\explorer.exe
      • Collapsing the %PATH% environment variable as it may have been too long (script 1)
      • Clearing the tile cache (script 2)
      • Shawn's answer (script 3)
      • sfc /scannow
      • dism /online /cleanup-image /restorehealth
      • Clean boot (All non-microsoft services and startup items disabled)
      • Obviously just unpinning and repinning it

      Script 1

      @echo off

      SET MyPath=%PATH%
      echo %MyPath%
      echo --

      setlocal EnableDelayedExpansion

      SET TempPath="%MyPath:;=";"%"
      SET var=
      FOR %%a IN (%TempPath%) DO (
          IF exist %%~sa (
              SET "var=!var!;%%~sa"
          ) ELSE (
              echo %%a does not exist
          )
      )

      echo --
      echo !var:~1!

      then

      setx path <OUTPUT OF ABOVE>

      Script 2

      @echo off

      taskkill /f /im explorer.exe

      taskkill /f /im shellexperiencehost.exe

      timeout /t 3 /NOBREAK > nul

      del %localappdata%\Packages\Microsoft.Windows.ShellExperienceHost_cw5n1h2txyewy\TempState\* /q

      timeout /t 2 /NOBREAK > nul

      start explorer

      @echo on

      Script 3

      DEL /F /S /Q /A "%AppData%\Microsoft\Internet Explorer\Quick Launch\User Pinned\TaskBar\*"

      REG DELETE HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\Taskband /F

      taskkill /f /im explorer.exe

      start explorer.exe

      ken blake say:

      See LemP's reply and let me add two points:

      1. A backup of your personal files will be useful, but a backup of your program files will not be. Your programs will need to be reinstalled from their original media.

      2. Your question suggests that you do not regularly backup your personal files to external media. If that's the case, you've been playing with fire. Almost everyone should be backing up to external media regularly. It is always possible that a hard drive crash, user error, nearby lightning strike, virus attack, even theft of the computer, can cause the loss of everything on your drive. As has often been said, it's not a matter of whether you will have such a problem, but when.
