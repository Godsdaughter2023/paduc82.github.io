add sequential # to each copy of excel page printed - Microsoft Community

nitengaleacct say:

add sequential # to each copy of excel page printed

In excel I have a form that I am printing 50 copies of.  I would like to number each copy.  Is there a formula that I can insert into one of the cells to calculate a page #?  Or, any other suggestions on how to accomplish this?  Thanks.
jllatham say:

Try this macro.  If you enter 0 or just press [Enter] at the prompt, no copies will be printed.

 

Sub PrintMultipleCopies()
  Dim CopiesToPrint As Integer
  Dim CopyNumber As Integer
 
  CopiesToPrint = Application.InputBox("Enter # of copies to print:", "Copies", 0, , , , , 1)
  If CopiesToPrint = 0 Then
    Exit Sub
  End If
 
  'these refer to the sheet/form you are going to print
  'and to the cell you want to put the copy # information into
  For CopyNumber = 1 To CopiesToPrint
    'put the phrase "Copy # of ##" into cell R20 of Sheet1
    '  the # will increment, the ## will be the # of copies you requested.
    Worksheets("Sheet1").Range("R20") = "Copy " & CopyNumber & " of " & CopiesToPrint
    Worksheets("Sheet1").PrintOut copies:=1 ' do them 1 at a time
  Next
 
End Sub

To put the code into the workbook for use: open the workbook and press [Alt]+[F11] to get into the Visual Basic Editor.  In the VBE, use Insert --> Module from its menu bar and copy the code above and paste it into the empty code module presented to you.

Edit the "Sheet1" entries in it to hold the actual name of your worksheet with the form on it (include the quotation marks at each end of its name).  Change the "R20" to the cell address where you want the copy number shown, and modify the

= "Copy " & CopyNumber & " of " & CopiesToPrint

to put what you want into that cell.  If you just wanted the copy number then it would read:

    Worksheets("Sheet1").Range("R20") = CopyNumber

Close the VBEditor and run the macro from the [Developer] or (I think it is 2007 on the) [View] tab Macros selector.

Hope this helps.

ossiemac say:

You will need to use Advanced Filter. I have set up a small example with only a few columns to demonstrate. (the screen shot becomes difficult to read with lots of columns)

I suggest that you insert a few lines above your actual data to allow for the filter entries like I have done in the following screen shot and Freeze panes at cell A7 so that the data column headers and the criteria are always visible.

Where I have entered Col2 in cells A1, B1 and C1 have the Criteria in row 2 under the headers. The headers must be identical to the header in the data to be filtered so you will use the column header of column 24 of your data. For "And" operator the criteria must all be on the same row and in separate columns with multiple matching column headers. ("Or" operators have one column and the criteria on separate rows.)

The Criteria as I have shown will filter all of Col2 data that does not contain 3, 9 or 2.

I suggest that you filter in place and then copy the visible data to another worksheet and then copy the worksheet to a new workbook for emailing. This can all be done with VBA (See my example code below)

The following code example will filter the data as per the criteria and then copy the visible cells to another worksheet. You will need to edit your code as per the comments in the example to match your data start and finish cells. Note that a space and underscore at the end of a line is a line break in an otherwise single line of code.

Feel free to get back to me if any problems following my instructions.

Sub Macro1()

    Dim rngToFilter As Range
   
    With Worksheets("Sheet1")     'Edit "Sheet1" to your worksheet name
        'Edit 6 in the following line to first row of your data (Header row of data)
        'Edit "E" in following line to last column of your data
        Set rngToFilter = .Range(.Cells(6, "A"), .Cells(.Rows.Count, "E").End(xlUp))
    End With

    rngToFilter.AdvancedFilter _
                Action:=xlFilterInPlace, _
                CriteriaRange:=Range("A1:C2"), _
                Unique:=False
   
    rngToFilter.SpecialCells(xlCellTypeVisible).Copy _
                Destination:=Worksheets("Sheet2").Range("A1")   'Edit "Sheet2" to your output sheet name.
               
    Worksheets("Sheet2").Columns.AutoFit
End Sub

ossiemac say:

Alternative code to extract the filtered data to another worksheet.

In the interactive mode, Excel does NOT allow advanced filter to copy the extracted data destination to be on another worksheet; must be on same worksheet or use filter in place.

In VBA Excel does allow the extracted data destination to be on another worksheet so by using that method it is not necessary to copy the visible cells because it can all be done in the Advanced filter line of code like the following example.

Sub Macro2()
    Dim rngToFilter As Range
   
    With Worksheets("Sheet1")
        'Edit 6 in the following line to first row of your data (Header row of data)
        'Edit "E" in following line to last column of your data
        Set rngToFilter = .Range(.Cells(6, "A"), .Cells(.Rows.Count, "E").End(xlUp))
    End With
   
    'In following code edit "Sheet2" to your sheet name
    rngToFilter.AdvancedFilter _
                Action:=xlFilterCopy, _
                CriteriaRange:=Range("A1:C2"), _
                CopyToRange:=Worksheets("Sheet2").Range("A1"), _
                Unique:=False
   
    Worksheets("Sheet2").Columns.AutoFit
End Sub

As you can see, there are 4 files: two Excel files (.xls and .xlsx) and two Word files (.doc and docx.)
So I created simple documents or I added a form:

  • a blue round for Word files,
  • and a yellow rectangle for Excel files.

I'm not talking here about the preview of the document, which is another feature of the Windows Explorer view.

This problem exists only for Office Open XML files (.docx, .xlsx, etc.).

As you can see, there is no problem to older format files (.doc, .xls, .ppt). In these cases, the thumbnails appear.

I checked several things:

  • About the procedure for recording thumbnails: I have saved my files each time by clicking on the "Save Thumbnails" option. I also check "Save thumbnails for all Word documents", which is located on the "Advanced Properties" tab of the "Information" tab.
  • About the settings of the folder option: the "Always show icons, never thumbnails" option is not checked.

I also did several manipulations and tests:

  • change Visual Effects: on Computer => Properties => Advanced tab =>Performance area => verify that Show thumbnails instead of icons is checked,
  • perform a clean boot state,
  • perform System File Checker Scan (SFC\SCANNOW),
  • restore thumbnail cache and icons,
  • perform a system restore,
  • perform a simple repair of Office 2016,
  • perform an on-line repair of Office 2016,
  • create a new user account,
  • perform a complete reinstallation of Windows 10 Professional (1703) with a clean installation of Office 2016 (without without third-party software like Ccleaner or antivirus).

I also noticed that this problem disappears by installing an older version of Office 2016.

What is interesting is to see how the recording of these thumbnails works.

For example, for a .docx file, Word creates an .emf file (vector image file). You can easily find this file by decompressing a .docx file (ZIP for example). There are three files:

  • _rels
  • DocProps
  • word

The docsProprs folder contains a file named thumbnail.emf. You can open this file with any image software, such as Paint. It is this file that allows Windows Explorer to display an image in the icon.

The problem must therefore come from either Windows Explorer, which can't open this .emf file, or from Office 2016 ...

Have other people been affected by this problem?

If someone to follow, I am taker.

Thank you in advance for your answers

P.
