You cannot copy the PST while Outlook has it open.  Outlook cannot open the PST if another program has it open.  Outlook must have exclusive access to the PST while it is running in order to ensure the integrity of the data structures within the PST.

wangyanyu say:

hi siow,

we can use our emails and website with the same domain in office 365. to achieve it, please follow the steps below:

if office 365 manage your dns records for office services,

1. add the domain to office 365 and select yes, i want to transfer dns management in the next step. then, the office 365 add dns records for email services automatically. please refer to add your users and domain to office 365.


2. sign into office 365 with your work or school account > office 365 admin > users > domains > select the domain > domain settings.
3. add an a record via clicking new > a(address) > host name or alias: type the following: @ > ip address, type the static ip address for your website where it's currently hosted > ttl > save.
4. add a cname record via clicking new > cname(alias) > host name or alias, type the following: www > points to address, type the fully qualified domain name (fqdn) for your website (for example, contoso.com) > ttl > save.

if you manage your domain’s office 365 records at your dns hosting provider, your website stays where it is and people can still get to it.

1. add the domain to office 365 and select no, i have an existing website or prefer to manage my own dns records.
2. add your dns records for your exchange service at your dns hosting provider. please refer to add your users and domain to office 365.

for more details, i would like to share with you an article: update dns records to keep your website with your current hosting provider. 

if anything is unclear, please let us know and we will continue to discuss the situation based on the update.

After you have located the Normal.dotm file, to open it, you have to right-click its icon and choose Open (as opposed to New, which is the default action) from the context menu. stefan blom say:

When I try to save Normal.dotm, it wants to save the file as "Doc2.docx."


This suggests that you haven't really opened the file. When Normal is open as a file, the title bar says "Normal.dotm" (or just "Normal" if you haven't instructed Windows to display file name extensions).

suzanne s. barnhill say:

As Stefan says, this means you haven't actually opened the Normal template for editing. If you have found Normal.dotm in Windows Explorer and just double-clicked on it, then you will have gotten a new blank document based on the Normal template because the default action for .dotm and .dotx (template) files is New. You would have to right-click on the file and choose Open. But here's another way to easily open and edit the Normal template:

In Word, press Alt+F11; the Visual Basic Editor opens. Press Ctrl+G to activate the Immediate window. In that window, type

     NormalTemplate.OpenAsDocument

and press Enter. Normal opens, but note that it may not be on top of all windows. You can close the Visual Basic Editor. Make the desired changes to the Normal file, and then save and close it. If the changes you have made have not been recognized as changes (the Undo button is not activated), press Spacebar, then Backspace to "dirty" the file, thus ensuring that the template will actually be saved.
