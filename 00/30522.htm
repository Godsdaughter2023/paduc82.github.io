Requiring a recovery key helps ensure that only an authorized person can unlock your PC and restore access to your encrypted data.

     b. If you saved the key to a USB flash drive, insert the USB flash drive and follow the instructions on your PC. (If you saved the recovery key as a file on the USB flash drive, you’ll need to open the file and manually enter the recovery key.)

    brian tillman [outlook mvp 2007-2019] say:

    I'd caution you against making a separate profile for each identity.  Switching profiles in Outlook means you'd have to shut Outlook down and restart it in order to choose a new identity.  Instead, if you really need to keep the information separate, just use a separate PST for each identity.  If I were doing it, I'd just use different folders in the same PST.  However, you're the one who need to use it, so you'll need to decide what's best.

    How are you transferring the data from OE to Outlook?  As far as I can tell, only one identity (the default identity)transfers at one time, since you have to switch identities in order to make another identity's data visible.  In Outlook, use File>Import and Export>Import from another program or file and choose Outlook Express.  That will import the default identity's folders into Outlook.  You can then add another PST to Outlook, make that the delivery location, change the default identity in Outlook Express to another identity, then import that data, continuing in this fashion until all of the identities' data has been transferred.

    Because Desktop wallpaper images have varying colors, it is easy for the font colors to be unreadable if they are on top of the same color as a wallpaper image. For instance a black font on top of a dark part of the background image or a white font on a white part of a background image.

     

    The Desktop icon fonts are dynamic and will show white on dark backgrounds and change to white with a black shadow effect on a light background.

     

    In previous versions of Windows, it did not matter what color you picked for the Desktop fonts, some would always be rendered invisible because of the background color of a desktop wallpaper. The user was constantly moving icons around the desktop to a different colored area where the font could be read. This was the source of a large number of complaints from users asking for a font that was always visible, regardless of the color that was behind the font.

     

    However you may change the theme and see if that helps:

    Change desktop theme:

    http://windows.microsoft.com/en-us/windows-vista/Change-desktop-theme
