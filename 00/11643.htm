tom van stiphout say:

The way I would typically do this is as follows:

* First get the db design correct, so the Events table has a ClientID field.

* Create a query on the Client table, perhaps:

select ClientID, FirstName & " " & LastName as FullName from tblClients

I'll name this query qrycboClients

* Design the Events form, put a combobox on it (I'll name it cboClientID), and set the ControlSource to ClientID and the RowSource to qrycboClients. Set the columncount to 2, and the ColumnWidths to 0;1

Using such combobox is very common in Access. It satisfies computers that like to work with numbers, and humans who like to work with text.

If there was an extra field about clients that I wanted to display on the Event form, let's say the City where they live, I would:

* Change qrycboClients to include the City field.

* Update the combobox properties to columncount=3, columnwidths=0;1;0

* Add a textbox, and set ControlSource to:

=cboClientID.Column(2)

This will display the value in the 3rd column (index = 2)
