macropod say:

Frankly, if you're trying to replicate an entire document, which is all your code does, you'd do better to make a copy of the file itself. Otherwise:

Sub Demo()
Dim DocSrc As Document, DocTgt As Document, HdFt As HeaderFooter
Set DocSrc = ActiveDocument: Set DocTgt = Documents.Add
With DocTgt
  .Range.Characters.Last.InsertBreak Type:=wdSectionBreakNextPage
  With .Sections.Last
    For Each HdFt In .Headers
      If DocSrc.Sections.First.Headers(HdFt.Index).Exists Then
        HdFt.LinkToPrevious = False
      End If
    Next
    For Each HdFt In .Footers
      If DocSrc.Sections.First.Footers(HdFt.Index).Exists Then
        HdFt.LinkToPrevious = False
      End If
    Next
  End With
  .Range.Characters.Last.FormattedText = DocSrc.Range.FormattedText
  With .Sections.Last
    For Each HdFt In .Headers
      If DocSrc.Sections.Last.Headers(HdFt.Index).Exists Then
        .Range.FormattedText = DocSrc.Sections.Last.Headers(HdFt.Index).Range.FormattedText
      End If
    Next
    For Each HdFt In .Footers
      If DocSrc.Sections.Last.Footers(HdFt.Index).Exists Then
        .Range.FormattedText = DocSrc.Sections.Last.Footers(HdFt.Index).Range.FormattedText
      End If
    Next
  End With
End With
End Sub

Restricting Access With Legacy Form Fields & Images - Microsoft Community

alepak say:

Restricting Access With Legacy Form Fields & Images

I am creating a form that will be used as a report. Basically, it includes dates that one of the employees can check, data they can fill in with text fields, drop-down controls that can select yes/no responses and images that they can upload in order to print the report.

Restricting access doesn't completely work. The only thing I can do is actually fill in the text fields. Dates won't change. Images won't upload. And trying to select something from a drop-down means that it pops open the Properties box.

Since this is a report that should be locked, is there a way to get images to upload and save? Is there a way to get dates and drop-down choices to stick without opening property boxes?

In unrestricted mode, I can actually get everything to work property EXCEPT the drop-down boxes, which still pop open the Properties box whenever you try to select something.

I am about at my wit's end trying to figure this out. If there are any suggestions, I would appreciate them.

macropod say:

Your description suggests you have a mix of formfields and content controls or ActiveX controls. You should only use the one kind of control in a document. Content controls and formfields, especially, don't play nicely together. For dates to change, they'd have to use one of the date fields (e.g. DATE, CREATEDATE). Obviously, DATE fields will only update when the date actually changes and CREATEDATE fields only update when a new document is created from a template or a document is saved via SaveAs. In a document protected for forms (which is what you need for formfields) you cannot access any of the image tools. Dropodown formfields should work just fine. Your description of the dropdown behaviour suggests it may be an ActiveX control.

There is a workaround for images, but it may not be viable for your scenario. That workaround is to use a second, unprotected document, and have an INCLUDETEXT field in the form document pointing to that document. You can then insert & manipulate the picture in the second document and have it display in the form. The INCLUDETEXT field could be inserted by the user into an unprotected Section, giving them control over which source file to use, or into a protected Section, meaning the user has no control. The former also requires the user to create the link via Insert|Quick Parts|Field>IncludeText, but it also means they can break the link, so the image becomes part of the form in its own right, by selecting its range and pressing Ctrl-Shift-F9. If no manipulation is needed, the same process could be used to insert the picture directly, using an IncludePicture field.

suzanne s. barnhill say:
To add to what Paul has said, if you use only content controls, you do not need to protect the document for forms, so all the controls (including date pickers and Picture Content Controls) will work correctly. If you want to protect the static text from editing, you can include the entire document in a group control (select all the text and click on the Group button).
