Hi,

Thanks for posting your query to Microsoft forum.

This issue can occur if Printer Spooler Service is stopped or anti-virus application is blocking the service.

I suggest you to follow the methods provided below and check if it helps.

Method 1:

Delete the pending files in printer folder and check: 

  1. Press Windows key + R and type services.msc in the search field and press enter.
  2. In the Services listed stop the Print Spooler service.
  3. In Windows Explorer, navigate to the folder below, and open the PRINTERS: C:\Windows\system32\spool\PRINTERS
  4. Click on Continue or Yes (if prompted by UAC) to approve opening the PRINTERS folder.
  5. Delete all files in the PRINTERS folder until it is empty, and then close Windows Explorer. Note: DO NOT DELETE the PRINTERS folder itself. Only delete its contents.
    Open Services, and start the Print Spooler service.

Method 2:

Let's start the all the dependencies service for the printer spooler service. Follow the steps provided.

  1. Press Windows key + R and type services.msc in the search field and press ENTER.
  2. Scroll down to Print spooler.
  3. Right click and click properties, click the tab “Dependencies”
  4. Note all of the dependent services listed, and then close the Print Spooler Properties window.

If any of the dependent services are missing or cannot be started, try uninstalling all installed printer drivers and then reinstalling them.

Method 3:

  1. Stop and Start the Print Spooler service at command prompt 
    Press Windows key + X.
  2. Right click on Command Prompt (Run as administrator)
    If you are prompted for an administrator password or for a confirmation, type the password, or click Allow.
  3. Type the following commands at the command prompt:

    net stop spooler
    net start spooler
    Exit

Note: Press ENTER after each command line. Perform these recommended steps and post back if you are still experiencing the issue.

Always thinking about how to undo any system adjustments if something goes haywire...

There is no quit or undo when modifying the registry so be sure you can undo any adjustments later if you need to.

itonme say:

Why these many "svchost.exe" files are running in win-10?

Why these many "svchost.exe" files are running in win-10?, and I never ever had like this on my pc. if anyone can answer pls? here is attachment...

bossdweeb say:

You may have more 'Services' running than previously ( I have 16 svchost.exe  running atm).

Windows, or software functions that uses Dynamic Link Libraries (.dll files ) interact with the Service Host .

You can find out which Services are being run via  svchost.exe  by R/clicking on each instance of the

svchost.exe Process in Task Manager and then selecting  Go to Service(s).

I believe that Win10 may run more svchost.exe than previous Windows as part MS optimising Win10 to make

better use of the many gigabytes of system RAM that most machines have these days.

.

a. user say:

The Host Process is a container for services. It's a completely normal part of Windows and it was actually designed to make Windows more efficient in its use of RAM. There is no number of host processes that is too high or too low.
