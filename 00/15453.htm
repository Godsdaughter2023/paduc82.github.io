This doesn't directly answer your question, but maybe you'll find a helpful clue in these tips:
Silent Install

http://www.appdeploy.com/packages/detail.asp?id=1802

Using MSP file

You need to create an MSP file first by running from the Office 2010 folder setup.exe /admin

You can customise the MSP from there. But to make sure you get a complete silent install:

In the "Licensing and User interface" section, tick "I accept the terms in the license agreement", Display level is set to "none", "completion notice" is un-checked, "Suppress model" is ticked.

Once the MSP file has been created you can run a silent install by: setup.exe /adminfile <the .MSP file you created>

I did find that Office 2010 wasn't activated by default so when a user opens office for the first time it would ask to activate. You can do this automatically by adding the following to a batch file:

C:\WINDOWS\system32\cscript.exe "C:\Program Files\Microsoft Office\Office14\OSPP.VBS" /act

The VB script can be found in the Office 2010 folder.

 

Using XML file

Configure a silent installation of Office 2010 by using Config.xml http://technet.microsoft.com/en-us/library/dd630736(office.14).aspx

 

Silent Uninstall using XML

To silently uninstall Office 2010, you need a xml file. I have used this:

command: setup.exe /uninstall ProPlus /config SilentUninstallconfig.xml

The contents of the xml file I use is:

<Configuration Product="ProPlus">

<Display Level="none" CompletionNotice="no" SuppressModal="yes" AcceptEula="yes" />

</Configuration>

(Obviously you will need to place the relevant file paths for your location of the Office 2010 setup files / folders)

harry stevens say:

Hi Ansonii,

Microsoft had already disabled the option of using @Hotmail.com as an alias for Microsoft Accounts. But, your existing @Hotmail.com address can still be used for your login. There is an option of disabling and enabling the login of specific aliases for your account, so it's possible that the hotmail address was disabled there.

To check if it was disabled or not:

1. Login to https://account.live.com

2. Click on "Your Info" on top of the page.

3. Click "Manage your sign-in email or phone number

4. Once you're at the "Manage your account aliases" page, click on "Change sign-in preferences".

5. It should list out all your account aliases. If it's unchecked, that means the login for it is disabled.

Let us know if this fixed up your issue. Thanks!

joeu2004 replied:

If the values are in A1 (actual closings), A2 (actual sales) and A3 (projected sales), then:

=IF(A1="",IF(A2="",IF(A3="","",A3),A2),A1)

Adds the condition after those stated in the first requirement: if Projected Sales appears blank, then return the null string (appears blank).
