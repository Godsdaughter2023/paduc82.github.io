andreas killer say:
You are right, the RefEdit works only if the userform is shown modal.

And I'm facing the same issue, I can't select a cell in a different workbook, even not with Application.InputBox. (XL2002 and XL2010 on WinXP)

So the userform must be shown modeless and we can't use the RefEdit control.

An easy way is to use a TextBox and a few events from the application to show the current selection. The only difference is that the TextBox did not update during the selection, but after the mouse button is released.

The example below needs a UserForm with at least one TextBox.

Andreas.


Option Explicit

Dim WithEvents App As Application

Private Sub App_SheetActivate(ByVal Sh As Object)
  'Update if the user select a different sheet
  TextBox1 = SelectionAddress
End Sub

Private Sub App_SheetSelectionChange(ByVal Sh As Object, ByVal Target As Range)
  'Get the address of the current selection
  TextBox1 = Target.Address(External:=True)
End Sub

Private Sub App_WorkbookActivate(ByVal Wb As Workbook)
  'Update if the user select a different workbook
  TextBox1 = SelectionAddress
End Sub

Private Sub TextBox1_Enter()
  'Enable the event
  Set App = Application
  TextBox1 = SelectionAddress
End Sub

Private Sub TextBox1_Exit(ByVal Cancel As MSForms.ReturnBoolean)
  'Disable the event
  Set App = Nothing
End Sub

Private Function SelectionAddress() As String
  'Get the cell reference of the current selected cells
  If ActiveCell Is Nothing Then
    'This happens if a chart sheet is selected
    SelectionAddress = ""
  ElseIf TypeOf Selection Is Range Then
    'Cells are selected
    SelectionAddress = Selection.Address(External:=True)
  Else
    'A shape is selected
    SelectionAddress = ActiveCell.Address(External:=True)
  End If
End Function

e.j. gun say:

That's difficult because the Application.InputBox seems to be "semi-modal", allowing you to select from different workbooks, but not allowing you to change workbooks through the taskbar.  The mod below provides one possibility - temporarily show all the open windows, allowing the user to select from any of them, and then going back to the original view.

 

HTH,

Eric

 

Sub test()
    Dim theRef As Range
    Dim saveWB As String
'
    saveWB = ActiveWorkbook.Name
    ActiveWorkbook.CustomViews.Add viewname:="Temp999"
    Application.Windows.Arrange xlArrangeStyleTiled
'
    Set theRef = Application.InputBox(Prompt:="Select a range from another workbook", _
                                      Title:="Select Range", _
                                      Type:=8)
    MsgBox ("You selected the following range:" & Chr(10) & Chr(10) & _
           "Address : " & theRef.Address & Chr(10) & _
           "Sheet   : " & theRef.Parent.Name & Chr(10) & _
           "Workbook: " & theRef.Parent.Parent.Name & Chr(10) & _
           "Path    : " & theRef.Parent.Parent.Path)
'
    Workbooks(saveWB).Activate
    ActiveWorkbook.CustomViews("Temp999").Show
    ActiveWorkbook.CustomViews("Temp999").Delete
End Sub

Is there any difference of the font size between Word and Visio? - Microsoft Community

shuai gong say:

Is there any difference of the font size between Word and Visio?

The font size in Visio ends with 'pt' but font size in Word does not have it. Are they still the same font size?
gene_gh say:

Font size is all the same; however, depending on how you import a Visio file into Word, the font may appear larger or smaller than the text in the document.  This is because Word imports the file as an illustration that can be resized and thus no longer retains it's 1:1 ratio.

If you want to retain as close to a 1:1 ratio, then decide the size the image will take in your Word document and then create your diagram in Visio based off that constraint.

I prefer to design the diagram in Visio, export it as an emf, and then insert it into my Word documents.  That way if I have a large Visio file with multiple tabs, all that extra content won't be in the document, it protects the content from alterations, it maintains the vector graphics (unlike a png, gif, jpg), and in many cases (so long as the you didn't use any images in the original Visio file) it will be much smaller file size than other file types.
