robert aldwinckle on forums say:

When the pop up happens how can I see where it is going and what IP address I need to add to proxy server?  


Try using the Resource Monitor's Network tab.   E.g. via link in Task Manager's Performance tab.

 

Alternatively, you may find enough clues in a cmd window from  ipconfig  /displaydns

 

Another simple to use tool which would definitely capture IP addresses but might be a bit difficult to interpret is ProcMon, e.g. for its Network events monitor.

 

Otherwise, you could have to do some kind of network trace.   If you're lucky you could try inserting Fiddler2 into your proxy chain and see what addresses it shows.   And if all else fails you could try using NetMon or Wireshark for their TCP traces.

 

 

Good luck

 

Robert Aldwinckle

if outlook launches in safe mode:
  a) click file, then click options, and then click add-ins.
  b) select com add-ins, and then click go.
  c) click to clear all the check boxes in the list, and then click ok.
  d) restart outlook, but not in safe mode.
if outlook starts and the error does not appear, this indicates a conflict with an add-in. begin adding the add-ins one at a time, restarting with each add-in that is enabled. this will allow you to figure out which add-in is causing the issue.

2. create a new user profile in outlook
  a) enter control panel, click mail > show profiles > add.
  b) in the profile name box, type a new name for the profile.
  c) specify the user name, the primary smpt address, and the password, and then click next.
you may receive this prompt: allow this website to configure alias@domain server settings?
in this prompt, click to select the don’t ask me about this again checkbox, and then click allow.
  a) in the add account dialog, click finish.
  b) under always use this profile, select the profile name that you used above.
  c) click ok.

3. run the /resetnavpane command. click start, type outlook.exe /resetnavpane, then click ok.

4. repair your outlook data files. you can follow the article below:
repair outlook data files (.pst and .ost)

if the issue persists, please share with us more details about the situation, such as your last operation and the error message.
