    burrwalnut say:

    Have you tried doing the association using the commands assoc and ftype?

     

    I don’t use Visual Studio or files with the .PDB extension, but check these out:

     

    Click the Windows Orb (Start) > All Programs > Accessories and right-click Command Prompt, then ‘Run as Administrator’. Note the spaces in all these commands:

    1. Type assoc .pdb and press Enter. This should display the current filetype.

    2. Type ftype xxx (where xxx is the filetype returned by (1) above and press Enter. This returns the currently used program.

     

    3. Change the filetype association by typing assoc .pdb=XXX (where XXX is the correct filetype) and press Enter.

    4. Change the associated program by typing ftype XXX=path of your program name %1 and press Enter.

     

    An easier to understand example of the above is a .txt file, which we all use in notepad:

    assoc .txt should return txtfile

    ftype txtfile should return %SystemRoot%\System32\notepad.exe %1
