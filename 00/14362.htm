jay freedman say:
Click Insert > Symbol. In the dialog box that appears:
  • At the top left, set the Font dropdown to "(normal text)" unless you want a specific font.
  • At the bottom right, make sure the "from" dropdown is set to "Unicode (hex)".
  • At the top right, set the Subset dropdown to "Combining Diacritical Marks".


In the center of the dialog, starting at the highlighted square, you'll find a series of the symbols you want -- accents, a macron, a breve, and so on. Each of these "combining" characters, when inserted, will combine with the character to its left.

For each character, you can use the combination shown at the bottom of the dialog as the shortcut key; for example, type the letter o followed immediately by the number 0304 and then press Alt+X to create a letter o with a macron (the "long vowel" mark) over it. However, if you expect to use these marks often, click the Shortcut Key button and assign a shortcut that's easier to remember and use such as Ctrl+_.

One other note: While the Symbol dialog is open, you can click in the document and continue to edit, coming back to the dialog when needed. You don't have to keep closing and reopening the dialog to insert multiple symbols.

While you can use the Combining Diacritical Marks that Jay refers to, it's much simpler to use the characters that already contain the macron and breve. You can find these in the Latin Extended-A character subset, as follows:

 

Latin capital letter A with macron: 0100

Latin small letter a with macron: 0101

Latin capital letter E with macron: 0112

Latin small letter e with macron: 0113

Latin capital letter I with macron: 012A

Latin small letter i with macron: 012B

Latin capital letter O with macron: 014C

Latin small letter o with macron: 014D

Latin capital letter U with macron: 016A

Latin small letter u with macron: 016B

 

In that same character subset you'll find many other useful characters. If you need the schwah, the capital is at 018F, the lowercase at 0259.

suzanne s. barnhill say:

I see I got carried away with the macrons (macra?) and didn't note the breves (brevia?), but of course you'll have discovered them immediately following the macrons.

 

If you can use a single opening quote for the rough breathing (dasia), then you can insert it via Ctrl+', '. But if you need the mark above a vowel, then you can use U0314, Combining Reversed Comma Above. Alternatively, the Greek Extended character subset includes vowels with all the required accents, breathings, and combinations. You could assign keyboard shortcuts to them.

 

As for the vowels with macron and breve, it is easy to assign custom keyboard shortcuts for them. In the Symbol dialog, click Shortcut Key, and then type in your desired shortcut and click Assign. I would suggest the following:

 

Ctrl+- (hyphen), a for lowercase a with macron (Word will show this a Ctrl+-, A)

Crtl+-, A for capital a with macron (Word will show this as Ctrl+-, Shift+A)

Ctrl+(, a for lowercase a with breve

Ctrl+(, A for capital a with breve

 

And so on for the rest. Unfortunately, Word won't accept Ctrl+), which would be more intuitive.

I had a different problem from the OP Expera, but mine is possibly related.

Sharing my notes below in case it helps Expera and others:

 

First, me before 1803: Surface Pro 4 with Sandisk Ultra SDXC card formatted as exFat to hold my PDFs, and Windows Search using PDF-XChange iFilter.

 

After I upgraded to 1803 10.0.17134:

 

1) All my PDF thumbnails  were removed.

2) Explorer would adequately re-build the PDF thumbnails (if I manually opened the folder containing the source PDFs), but no matter how many times I let Explorer rebuild the PDF thumbnails they were blank the next day and I had to rebuild them all over again.

3) I could no longer search and find content inside my PDFs using Explorer.

 

#2 is frustrating because it takes a long time to generate the thumbnail if the PDF is large. I have many large PDFs (book library) and I rely on the thumbnail for quick visual identification.

 

#3 is frustrating because I rely on Windows Search to find content inside my PDFs.

 

Fixing #2

I had this problem before 1803, and had fixed it, but after 1803 this problem returned.

I repeated the following regedit fix and it worked (again):

 

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\VolumeCaches\Thumbnail Cache

 

Change the Autorun DWORD to 0

 

HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Explorer\VolumeCaches\Thumbnail Cache

 

Change the Autorun DWORD to 0

 

Fixing #3

After 1803 the PDF handler for Windows Search was "Reader Search Handler" (Windows Settings, Indexing Options, Advanced Options, File Types) which I believe is the default index-handler for Windows (?). I re-installed PDF-Xchange Viewer and doing so installed my original (PDF-XChange) PDF iFilter again, but the re-installation did not fix problem #3.

 

  1. I tried other iFilters:
    1. PDF-XChange Viewer installs their iFilter. (not the Editor)
    2. I discovered that the old Adobe iFilter did erratically work but was very slow!
    3. FoxIT is really only for servers and is expensive.
    4. TET is free and probably the best. Great docs.
  2. But the real problem came down to my D: (SDXC) where my PDFs were stored. The SDXC card was formatted as exFat. I copied a folder of PDFs to C: (the SSD inside my Surface Pro 4) and noticed that the iFilter worked fine for that folder on C. I backed up D: and reformatted D: as NTFS and re-copied the files, then installed iFilter (TET), and everything worked fine (I could quickly search inside the PDFs).

 

So, bottom line, I believe something in Windows Update 1803 (Spring Update):
- Erased/overwrote/disregarded my thumbnail cache.

- Reset the Thumbnail Cache DWORDs to 1

- broke the indexing engine for Windows Search and it could no longer index exFat PDF files on SDXC cards.

 

Recommendation to MS:

For upgrades:

  1. Do not reset the thumbnail cache during upgrades.
  2. Do not change Thumbnail Cache Autorun DWORDs back to 1 if they are 0.
  3. Do not overwrite a PDF iFilter if it exists and is set as the default.
  4. exFat problems should be looked into.

Hi Ryan,

 

Thanks for your feedback and sharing your research on this issue.

 

You can refer to this article to know more detail about this fixed known issue: Unable to edit Contacts, Appointments, Meetings, or Tasks within Outlook

 

And I would like to make a summary here so that more people can know more about this issue.

 

About the cause:The reason for this phenomenon is because some items contain internet header information that causes Outlook to believe these items should not be editable. And it will affect not only contacts, appointments, meetings or tasks may also be involved.

 

About the fix:

a) Monthly Channel fix was released on April 29th in Version 1904 Build 11601.20144.

b) Semi Annual Channel will release ~May 14th in Version 1808 Build 10730.20340.

c) The fix for Outlook 2016, Outlook 2013, and Outlook 2010 (also known as perpetual versions) is planned for the August 13th 2019 Public Update. 

 

About the workarounds:

Outlook users: Create, view, and edit contacts and contact lists in Outlook.com

Office 365 users: Create, view, and edit contacts and contact lists in Outlook on the web

Fix contacts in bulk: Export all of your contacts, then import them so the internet headers are removed. This process marks the contacts as editable.
