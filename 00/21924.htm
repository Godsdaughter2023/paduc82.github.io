rohn007 say:

What you are asking could be rephrased as "can changes to PPT slides in Sharepoint be 'pushed' to the PPT presentations they are copied to".

Short answer, appears to be yes (to my surprise), kinda. It is more of a "Pull" initiated by the presentation, rather than a "push" initiated by Sharepoint.

Short form the process is:

  • can create a "PPT Slide Library" in Sharepoint. 
  • Add slides to the library
  • add slides from the library to your presentation, with a link back to the library. 
  • After that, the slide in the library may be edited.
  • Open the presentation, PPT gives you the option to check the library for changes, skip the check or disable the check.  If you check and changes are found they are uploaded to replace or compare to existing slide.

I found this article in the PPT 2016 online help under "reuse slides". It should give you a start on the subject:

ossiemac say:

Not exactly sure if my code emulates what you are attempting to achieve but you can set the position of the button as the last code lines after processing the Hide/Unhide. I have used a single button and just changed its caption and use the caption to decide whether to hide or unhide.

 

Private Sub CommandButton1_Click()

    If CommandButton1.Caption = "Hide Columns" Then
        Columns("Y:AF").Hidden = True
        CommandButton1.Caption = "Un-Hide Columns"
    Else
        Columns("Y:AF").Hidden = False
        CommandButton1.Caption = "Hide Columns"
    End If
   
    'Add following as last lines of code when button is clicked.
    CommandButton1.Top = Range("Y2").Top    'Edit "Y2" to required location
    CommandButton1.Left = Range("Y2").Left  'Edit "Y2" to required location

End Sub

 

Caveat: If column to the left of Y are hidden then the button will appear at the left of the hidden columns because after all hiding columns is really just making then all zero width so the left of the hidden column will be the right of the last visible column to the left.

How else would the web site know where to send the files if they didn't know your IP? A bank can't send you mail unless it knows your street address.

You could use a proxy to hide your IP like a PO box would hide your street address, however, for the most part, your ISP does this for you by you:

Your PC > request sent to Router IP > Sent to ISP Router > sent to web server > back to ISP Router > through modem to Router > to your PC

Web server knows the IP of the ISP Router but not your IP, unless you have a static IP which then you told the ISP you want everyone to have a direct line to your IP. With this they can get the public recorded IP of the ISP which can narrow down your state (maybe) and less likely a city.

If you really want to hide you use a proxy. So everything is the same except you have a server that collects your requests sends them to the ISP then collects the returned data and sends it to you.

This doesn't do much use unless you are trying to hide from the FBI, and even them once they have access to the proxy they can find where the data is sent. This also doesn't do much for privacy unless you run your own proxy because now you are adding one more server to see your data. Then SSL gets a little odd in there too.

InPrivate browsing is not a tool for terrorist and other malicious user to hide their location from the authorities
