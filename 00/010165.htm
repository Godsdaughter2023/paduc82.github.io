anula d'souza say:
Right click on the copied PST file click on Properties and check if you get any option 'Unblock' if yes click on it and then try to open the PST file in Outlook and check if it helps.
 
Are you able to access the PST file in the old system?
If yes, Export the PST file and save it on the system, copy the exported PST file to the new system and then try to open the PST file in the new system and check if it helps.
To export the PST file:

·         Open Outlook

·         click on File< Open< Import

·         Highlight Export to a File and click Next

·         Highlight Outlook Data File (.pst) and click Next

·         Highlight Inbox and click Next

·         Browse to the location where you want to save the file and click Finish

As scanpst cannot recognise the PST file, the PST file might have gone corrupt beyond repair.

russ valentine say:

State by what method you copied and transferred this PST file. Ensure it did not accquire a Read only attribute when doing so. See what happens if you try to repair this file with the Inbox Repair Tool: http://support.microsoft.com/kb/272227/en-us

The OS is not in play here. I would disagree with the recommendations that you export and import a PST file. That's just another good way to corrupt it. Just copy it to your local hard driveand open it in the new installation of Outlook and open it with Outlook's File > Open > Outlook data file... path.
