swischner say:

Not a solution to someone (really?) having used your gmail-adress, but my personal workaround for anything like this:


I don't know if this would help you but here is what I am doing to be able to send mails to my OneNote without having to fiddle around with authorizing mail adresses. Prerequisite is that sending mails from my MS account main address to me(at)onenote.com is working.

1. In outlook.com i signed in with my MS account and created an alias address for that mailbox. Lets say my MS address would be "SW(at)outlook.com". I added an alias like "SW-OneNote(at)outlook.com". Now all messages sent to this alias are going to the inbox of the main account ("SW(at)outlook.com").

2. Next (still on outlook.com) I create a forwarding rule. Every message which has been send to the alias address instead of the main adress (e.g. "SW-OneNote" in TO: field) is forwarded to me(at)outlook.com.

This way mails to me(at)onenote.com always are coming from my MS account mail adress and the OneNote redirection works flawless. At the same time I can send to OneNote from any address without authorizing it first in the OneNote-mail settings. I just have to send them to that alias adsress I created ("SW-OneNote(at)outlook.com").

Keep the alias address a secret! Everybody who knows it could send messages to your OneNote too. If you happen to believe the alias adress has been compromized you can always delete it on outlook.com, create another one and edit your forwarding rule accordingly.

A, It can not be done through formulas. Though you can use =NOW() formula but it will keep updating. But you can copy and paste it as value to retain the timestamp.

B. Otherwise, you will have to use VBA.

1. Make a backup of your workbook.
2. Open your workbook and ALT+F11
3. Locate your Workbook name in Project Explorer Window
4. Double click on your worksheet name
5. Copy paste the Macro code given below
6. Save your file as .xlsm

Sample file can be downloaded from @ https://1drv.ms/x/s!Akd5y6ruJhvhjzpdGq9XQwE7oYqR 

Private Sub Worksheet_Change(ByVal Target As Range)
    On Error GoTo ExitSub
    If Intersect(Target, Me.Columns("D")) Is Nothing Then Exit Sub
    Application.EnableEvents = False
    If Target = "Completed" Then
        Target.Offset(0, 1) = Now
    End If
ExitSub:
    Application.EnableEvents = True
End Sub

gord dibben say:

You can do this with VBA worksheet event code which inserts a static time stamp.

Right-click on the Sheet Tab and "View Code".

Paste the following code into that module.

Alt + q to go back to the sheet.

Choose a value from DV List in a cell in D to get a timestamp in E

You can adjust the D1:D10 range and the date/time format in the code

Private Sub Worksheet_Change(ByVal Target As Range)
    On Error GoTo ws_exit:
    Application.EnableEvents = False
    If Not Intersect(Target, Me.Range("D1:D10")) Is Nothing Then
        With Target
            If .Value = "Completed" Then
                .Offset(0, 1).Value = Format(Now, "dd mmm yyyy hh:mm:ss")
            End If
        End With
    End If
ws_exit:
    Application.EnableEvents = True
End Sub

An alternative approach which if I've understood correct seems to do what you want.

 

 

=INDEX(F1:F4,MATCH(1,(A1:A4=K3)*(--(RIGHT(E1:E4,6))=--INDEX(RIGHT(E1:E4,6),MATCH(MIN(ABS((--RIGHT(E1:E4,6))-J3)),ABS(--(RIGHT(E1:E4,6))-J3),0))),0))

=INDEX(F1:F4,MATCH(1,(A1:A4=K3)*(--(RIGHT(SUBSTITUTE(E1:E4," ",""),6))=--INDEX(RIGHT(SUBSTITUTE(E1:E4," ",""),6),MATCH(MIN(ABS((--RIGHT(SUBSTITUTE(E1:E4," ",""),6))-J3)),ABS(--(RIGHT(SUBSTITUTE(E1:E4," ",""),6))-J3),0))),0))
