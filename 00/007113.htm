EDIT After reading the comment of Dana.D, here is the improved version.Function MySum(R As Range)
   Dim RE As Object
   Dim objMatchCol As Object
   Dim objMatch  As Object
   
   Set RE = CreateObject("vbscript.regexp")
   
   With RE
      If Application.DecimalSeparator = "," Then
         .Pattern = "\[[0-9]+\]|\[[0-9]+,[0-9]+\]"
      Else
         .Pattern = "\[[0-9]+\]|\[[0-9]+\.[0-9]+\]"
      End If
      .Global = True
      .IgnoreCase = True
   End With
   
   If RE.Test(R) Then
      Set objMatchCol = RE.Execute(R)
      For Each objMatch In objMatchCol
         MySum = MySum + Replace(Replace(Replace(objMatch, "[", ""), "]", ""), ".", ",") * 1
      Next objMatch
   Else
      MySum = ""
   End If
End Function

I've tested it with number formats #.##0,0 & #,##0.0.

Enter in B2: =MySum(A2)

The advantage of this approach is that the number of values to sum is almost unlimited.

What it does is looking for any occurrence of a to z (and because of IgnoreCase =TRUE, also valid for A to Z. (Short notice is [a-z].

The "|" stands for an OR function. Because "[" & "]" are used in RegExp to indicate a range like [a-z] but here I wan to explicitly refer to the literals [ & ], I have to escape them with "\".

So, putting it all together, the pattern says to look in the string for any character a to z or the characters [ or ] and finally replace them by "", keeping 2020 + 3800 + 550 as remaining string which can be handled by the EVALUATE function.

Function SumInBrackets(S As String) As Double
Do
  On Error GoTo Finish
  P = InStr(P + 1, S, "[")
  L = InStr(P, S, "]") - P - 1
  SumInBrackets = SumInBrackets + CDbl(Mid$(S, P + 1, L))
  Loop
Finish:
End Function

That Netgear UTM50 isn't just a firewall, it includes threat management which can block things based on signatures, so it's likely these additional features which are blocking the MSE updates.

I've seen this issue with another competitor's UTM firewall in the past, though typically these cleared themselves up within less than a day once the firewall company realized the issue.

You need to contact whomever provides support for that firewall and ask them to look at its logs, then modify either the firewall policies or other configuration as necessary to stop blocking the Microosft signatures.
