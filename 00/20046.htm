Houston:

You "do" have QuickParts, but they are not very developed in Word 2011, and they are not the weapon you want for this task.

You have four options for doing this.

1)  Paul's and Peter's suggestion of using DOCPROPERTY fields is the simplest, that's the way we have been doing this for decades, it's simple, fast, and reliable.

Go to File>Properties and you will see the built-in properties on the Summary tab.  These are usually enough for normal use, but if you need more you can go to the Custom tab.  Note:  Most of us repurpose the Category and Keywords fields into something useful.  We've been doing that so long I don't think I have ever seen those fields used for their intended purpose :-)

On the Custom tab you have a wide variety of built-in Custom fields.  These fields have been defined but they do not yet exist until you put something in them.  When you do enter a value, you can also use the Type drop-down to set a data type for them.  Most people don't bother.  Caution: Trap for young players -- when you assign a value, you MUST click Add to create the field with its value.  Remember, these things do not exist until you click Add.

You cannot re-name the built-in custom fields, but you can create your own custom fields, and assign any name you like, simply by over-typing the name field, but I have almost never needed to create one.  If you create these fields in a document template, they will be copied to new documents created from that template.

http://office.microsoft.com/en-au/word-help/set-or-change-file-properties-HP005189753.aspx

2)  Rich's idea of custom Character styles and StyleRef fields is very handy, especially if you need your variables to appear in headers or footers.

http://office.microsoft.com/en-au/word-help/field-codes-styleref-field-HP005186193.aspx

3)  For heavy-duty use you can use Bookmarks and Cross-references.  You bookmark the variable, and the cross-references copy the text wherever you want it.  You can use an ASK field to automatically prompt the user to specify the variables at Document Open.  The ASK field populates the bookmark the cross-references then use.

http://www.wikihow.com/Add-a-Bookmark-in-Microsoft-Word and http://office.microsoft.com/en-au/word-help/inserting-cross-references-HA001186680.aspx

4)  Finally, for industrial purposes you can use a Document Variable.  A DOCVARIABLE is exactly the same as a document property with the added advantage that normal users cannot change them. You must use VBA to create or change document variables, the users can never get at the value from the user interface (unless they know enough to build themselves a macro to do it).

http://support.microsoft.com/kb/306281

Hope this helps

'aircode warning

Sub Macro1()
Dim FileNum As Integer
Dim strPara As String
Dim strNum As String
Dim strMachine As String, strMachineID As String
Dim strProgrammer As String, strDate As String
Dim strItem As String, strSeq As String, strTape As String
Dim strDraw As String, strRev As String
Dim strOD As String, strID As String, strLen As String

Dim iLine As Integer: iLine = 1
    FileNum = FreeFile()
    Open "C:\Path\Filename.txt" For Input As #FileNum

    While Not EOF(FileNum)
        Line Input #FileNum, strPara
        Select Case iLine
            Case 1
                strNum = Split(strPara, "(")(0)
                strNum = Mid(strNum, 2)
Debug.Print strNum
            Case 5
                strMachine = Split(strPara, "-")(0)
                strMachine = Trim(Split(strMachine, "=")(1))
                strMachineID = Trim(Replace(Split(strPara, "=")(1), ")", ""))
Debug.Print strMachine & vbTab & strMachineID
            Case 7
                strProgrammer = Split(strPara, "=")(1)
                strProgrammer = Trim(Replace(strProgrammer, "DATE", ""))
                strDate = Split(strPara, "=")(2)
                strDate = Trim(Replace(strDate, ")", ""))
Debug.Print strProgrammer & vbTab & strDate
            Case 8
                strItem = Split(strPara, "=")(1)
                strItem = Trim(Replace(strItem, "SEQ #", ""))
                strSeq = Split(strPara, "=")(2)
                strSeq = Trim(Replace(strSeq, "TAPE NO", ""))
                strTape = Split(strPara, "=")(3)
                strTape = Trim(Replace(strTape, ")", ""))
Debug.Print strItem & vbTab & strSeq & vbTab & strTape
            Case 9
                strDraw = Split(strPara, "=")(1)
                strDraw = Trim(Replace(strDraw, "REV", ""))
                strRev = Split(strPara, "=")(2)
                strRev = Trim(Replace(strRev, ")", ""))
Debug.Print strDraw & vbTab & strRev
            Case 10
                strOD = Split(strPara, "=")(1)
                strOD = Trim(Replace(strOD, "PART ID", ""))
                strID = Split(strPara, "=")(2)
                strID = Trim(Replace(strID, "PART LENGTH", ""))
                strLen = Split(strPara, "=")(3)
                strLen = Trim(Replace(strLen, ")", ""))
Debug.Print strOD & vbTab & strID & vbTab & strLen
            Case 11
                strTime = Split(strPara, "M")(1)
                strTime = Trim(Replace(strTime, "E", ""))
Debug.Print strTime
            Case Else
        End Select
        iLine = iLine + 1

ross jaburg say: Let me see if I can clarify:


You are assigning a default action to perform.

If you leave it set to "Recommended Action," when MSE encounters the specific alert level it will perform whatever action MS (the developers) think is the best course of action. This may be removal or quarantine. I have not seen a list anywhere of exactly WHICH action it will take in specific cases.
 
If you manually set it to "Remove" or "Quarantine" it will do that when it encounters something. Whatever you want to do is fine.
If you are concerned about possible "false positives," you may want to have it default to "Quarantine" so nothing is deleted.

But I will warn you, MSE's quarantine takes up a disproportionate amount of space.
I had two quarantined trojans totalling less than 10MB of file size taking up about 1GB of disk space.
According to support (my trouble ticket) response, this behavior is "by design."
