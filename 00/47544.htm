jamesgritton say: I ran into this problem too.  It turns out to be related to active window tracking (focus follows mouse).  I had set this with the registry in Windows 8, and noticed that when I went to the trouble of creating a new user after upgrading, the new user worked fine - until I made that same registry setting.

I managed to fix it by setting active window tracking the "right" way, via the "ease of access center" control panel.  Unchecking the checkbox ("Activate a window by hovering over it with the mouse" in "Make the mouse easier to use"), and then rechecking it did the trick.  Another noticeable effect was that there's a half-second delay before the window under the mouse gets the focus, the same delay that makes task bar context menus work again.

On a further look, the problem is likely related to something else about active window tracking that 8.1 broke: if you move the mouse outside a window (i.e. onto the desktop), then the window loses its mouse focus.  Pre-8.1, the window the mouse used to be in kept the focus until you moved to a different widow.

The old behavior is correct; the new 8.1 behavior is broken.  Microsoft: please fix that bug.
