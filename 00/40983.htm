chkdsk is a Windows utility that scans your hard drive(s) and depending on the options specified may make an effort to correct problems or inconsistencies in the NT File System (NTFS).

There is a similar message with Windows XP:

Windows XP could not start because the following file is missing or corrupt: \WINDOWS\SYSTEM32\CONFIG\SYSTEM

Always, always, always the first thing to do when you see a message like that is to run a chkdsk with error correction (chkdsk  /r) because most of the time the files of the registry are not really missing or corrupt...  the operating system just can't find them because there is some corruption in the file system.

The file system can get corrupted from things like an aborted restart, ungraceful shutdown, power failure, power interruption, using the reset button, using the power button, plug pulling, battery pulling, etc.

Did any of those incidents take place prior to this incident?

If the system used to work properly, what do you think happened to it?

entegy say:

XP Mode (and Microsoft Virtual PC) has no graphics acceleration. It's not designed for gaming. No matter what your host hardware is, you will not get your games running. The best you could manage is maybe some old Windows 9x era text games.

If you absolutely need XP for your games and are not comfortable dual-booting, then VMware has some virtualization solutions with 3D acceleration for some gaming and 3D graphics apps. I do believe the free VMware Player will work for you, and Oracle's VirtualBox has OpenGL support and experimental DirectX support.
