¡firedog say:

Why did you add the process no more rules? 

That action only applies to the message being dealt with at the time. When a message arrives, rules are processed in order from the top down. If there is a rule lower down the list which also applies to the message being processed, it might override the first rule. If your rule, say, is intended to delete a message from a .date address, but a later rule has a different action, it's the later one that will win the contest. Unless there is a good reason not to, 'move' and 'delete' rules should always end with Stop processing … to be sure of getting the right result.
