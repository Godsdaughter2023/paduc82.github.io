One computer is running Vista, the other Windows 7.  Both OS are fully updated and patched, as is Office/Outlook 2010.  This problem only occurs with e-mail downloaded from AOL.  Gmail, Hotmail, and e-mail from SBCGlobal.net do not experience the same problem.

billhutchinson say: Problem Solved!

It turns out the problem appears to have been with the syntax of the User IDs in question.  Previously, login to the AOL e-mail servers was  not case sensitive.  Apparently some recent improvements that AOL made to its e-mail system changed this in some way, at least as it relates to the User ID.  Since the User ID/Names in question were originally created in AOL Mail as a mix of upper case, lower case, and numbers, using all lower case in creating the Outlook accounts resulted in the problems I was experiencing.

Recreating the accounts using the original AOL Mail user names seem to have resolved the problem.  I have no idea why the same problem did not occur in Mozilla Thunderbird.  It most have something to do with how that program resolves User ID conflicts...

hansv mvp say:

Try this:

Sub ResetShapes()
  Dim shp As Shape
  Dim ishp As InlineShape
  For Each shp In ActiveDocument.Shapes
    shp.Glow.Radius = 0
    shp.Reflection.Type = msoReflectionTypeNone
    shp.Rotation = 0
    shp.Shadow.Visible = False
    shp.SoftEdge.Type = msoSoftEdgeTypeNone
    shp.ThreeD.Visible = False
  Next shp
  For Each ishp In ActiveDocument.InlineShapes
    ishp.Reset
  Next ishp
End Sub
