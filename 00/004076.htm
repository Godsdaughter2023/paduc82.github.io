See the following Wiki article: Microsoft Security Essentials Update FAQ


Unless you also have a WSUS server to deploy signatures and other updates, you will need to do this manually.


You can manually download the (large file with engine and full definitions) latest definitions from the portal here: https://www.microsoft.com/security/portal/default.aspx


This download is large and there are x86 and x64 versions. If your PCs are all the same architecture, you can get away with one download. You would need to download this file to a shared location and then create a scheduled task on each machine to execute the downloaded file, which updates the engine and database.

reza ameri say:

stephen boots say:

You may also consider Windows Update catalogue to get definitions:

http://catalog.update.microsoft.com/


I don't recommend this. The download via the catalog will yield a full set of the most recent base definition set and every single delta patch update released since then. There is no way to select only the individual patches. The engine and base are typically updated monthly.


If the users are not connected to the Internet, update frequency can be limited to every few days for replacement of the engine/definitions via the portal download and 70 megs or so. The portal download can be renamed to be the same time each time that it runs as it will replace the installed database and engine.


Via the catalog, the download will become incrementally larger every day and you would need to manage the delta patches individually to tracks which were installed per machine (with the corresponding name changes).


-steve

bob jones aka: cybertaz say:

It sounds like the Style idea is working, ... but it's precisely due to those "make it easy" features that you're seeing that result override the Style :-)

Specifically, Default Tab Stops combined with the fact that Word has a built-in AutoCorrect feature to have a press of the Tab key set a First Line Indent @ the first tab stop (.5" is the first default tab stop) when the insertion point is at the start of an existing paragraph.

There are several ways to avert the behavior. Which one is preferable is dependent on the intent of pressing Tab at the beginning of the paragraph;

  • With the feature active use Option+Tab to tab to the point at which the Hanging Indent is set. If there is none set the keystroke moves the insertion point to the first available tab stop.

Or,

  • The feature can be disabled by going to Tools> AutoCorrect - AutoFormat as You Type to clear the check from Indent with Tab or Backspace. With the feature inactive, pressing Tab will indent to the same point as the Hanging Indent. Using Option+Tab applies a First Line Indent as described above.

IOW, Option+Tab reverses the behavior of whatever you have chosen as your default AutoCorrect setting.

BTW: Although I can understand how you may have gotten that impression based on the somewhat different context of your other message, typing the entire document isn't exactly what I suggested. The point was that if you're going to format on the fly, it's far less painful to apply the deviations selectively once the target content is in place. Otherwise, you run the risk of overriding what you had originally done, which is why re-applying your Style has no effect unless you use the Clear Formatting tool remove the direct formatting first.
