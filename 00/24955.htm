#1 - Re: Autosave. What you're referring to is the "auto-complete" cache which gets populated by the recipients of any email message that you send. That is completely separate and distinct from saving those as contacts in a contact folder

#2 - A contact folder would be created for any Exchange/Outlook.com account. This will occur for every EX/Outlook.com account created

#3 - The first email account created in the Outlook profile would result in the data file for that email account to be set as the "default data file" for the profile. This data file contains the "default" contact/calendar folders for the profile

#4 - Once a default data file is created/set, then any subsequent POP3 or Imap account will not have any contact/calendar folders created within the folder tree for that account

karl timmermans [outlook mvp 2012-2018] say:
Re: #1 - AutoSave - Email Address selection

There are two ways to select a recipients email address to populate the To/CC/Bcc boxes

a) First - is to click on the actual To/CC/Bcc buttons which opens the Outlook Address book. The Outlook AB is a specialized view of all the contact folders that have been set to be a member of the AB. There is no automatic saving to an actual contact folder

b) Starting to type the first few characters in the To/Cc/Bcc boxes when composing/replying to an email message. This presents the auto-complete cache of all recipients to whom you have sent a message. As mentioned, the Auto-Complete cache has nothing to do with any contact folder

Re: #2 - Number of contact folders

Your starting point to determine the number of contact folders in your profile starts with the Outlook Nav Pane (list of folders on the left hand side when you have Outlook open). Press CTRL+3 to just see the contact folders or CTRL+6 for the complete folder list which will clearly show in which folder tree each contact folder tree is located

Re: #3 - Default contact folder

Your default contact folder is located in the "default data file" as configured in Outlook. From Outlook menu, go to File --> Account Settings --> Data Files tab. One of the data files is set as the "default" which should also be the first folder tree showing in the Outlook Nav Pane.

Your screenshot shows "All Users" the Global Address List which is part of an Exchange account and has nothing to do with the topic at hand so will skip this just for the sake to keep things simple at this point. The Contact folder

If the first email account configured in your profile was the Exchange account, there would not be any contact folders created in the folder trees for the POP and IMAP accounts. The exception to this would be if you used an old PST file when configuring a POP3 account and that PST file contained a contact folder

Re: #4 - Screenshot AB default folder

That shows that you have set the Offline Global Address List as the default list to show when the AB is opened. What you want to do is change to show the actual default contact folder where your contacts are located. Can't tell you which one that is since the screenshot shows that you have 2 "contact folders". The steps to change what s shown first when you open the AB is done by

a) open the AB from the Outlook ribbon
b) from AB menu --> Tools --> Options

Finally, to be very clear about something, where you are saving your contacts has nothing to do with the settings of the AddressBook. If you actually save a contact either by adding a new contact to a contact folder it will be saved to that specific folder regardless of whether or not it's the default. If, on the other hand, you right-click an email address from a rec'd message and selecting "Add to Contacts" - it will be added to the contact folder that is the default for the profile

Solving problems unique to a machine can be an arduous task for both the end-user and the help desk. That’s why Win7 introduces the Problem Steps Recorder, a screen-capture tool that allows the end-user to record the problems they’re having step-by-step. If the problem is repeatable, the recorder will capture what you do in text form.

It’s as simple as hitting “record” then adding in comments as needed. A HTML-based file is converted to a .ZIP folder, which is easily passed on to the help desk.  The program is accessible from the Control Panel under “Record steps to reproduce a problem” or run psr.exe from Explorer.

andreas killer say:
If you had a set-up that did a lot of debug.print-ing, the code runs much slower (although I don't know if there is any difference in speed if the VBE is closed).
Theoretical it's possible to enumerate all open windows etc. to detect this state, but a conditional define is much easier and fail safe too.

And more, if you want to suppress Debug.Print in a loop you have to use an IF statement and this costs time and resources. To clarify, your code can look like this:

Dim UseDebugPrint As Boolean

Sub Test()
  Dim i As Long
  For i = 1 To 1000000
    If UseDebugPrint Then Debug.Print i
  Next
End Sub

but it's slow. Safer and faster is this:

#Const Develop = False

Sub Test()
  Dim i As Long
  For i = 1 To 1000000
#If Develop Then
    Debug.Print i
#End If
  Next
End Sub

If you open the VBA editor and need your Debug.Print, set Develop to True.

There are buttons to switch between views in the right-hand side of the status bar at the bottom of the Word window. Click the first one, Print Layout View.

jay freedman say:
The short way is to point to the very bottom of the Word window, to the left of the zoom percentage -- there are five little buttons there that change the view. When you hover the mouse over any one of them, a popup will tell you which view it opens. The one on the left is "Print Layout", so click it.

The long way is to click the View tab on the ribbon and then choose Print Layout at the far left end.
