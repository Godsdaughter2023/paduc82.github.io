frankpjb say:

It could be an issue with how the text is formatted in the notes page. The PDF printer will have no choice but to rasterize the text if you use a font or font style that cannot be embedded.

For example, many fonts (e.g. Tahoma) have no italic form. When you italicize them in an Office application you are actually applying a faux italic directly to the roman (upright) version. The problem is this can't be embedded in the PDF because it simply doesn't exist.

Or, you might simply be using a font that does not permit embedding.

If the notes use different formatting from the slides, this will explain why the slides look OK but the notes do not.

Are you using the same font as you used in Office 97? If so, were the fonts rasterized there too? If not, then perhaps you could switch back to the older font and see if that resolves the issue.

ronrosenfeld say: It is difficult, using formulas alone, to concatenate a series with an undefined number of repetitions.  If you know the maximum number of repetitions, you can develop a formula with a number of "If" statements.

But the problem is easily solved using a UDF (User Defined Function), which can also add considerable flexibility

For example:  =GetValues(String, Prefix)     -->  GetValues(A1,"ABC")
  would be the formula to be entered in some cell

To enter this User Defined Function (UDF), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

The code assumes that the substrings are always in the format of "ABC"  (or some other prefix) and then followed only by digits; until getting to the <space> or <comma>.  Using regular expressions, as below, allows one to easily change the format definition, if such should not always be the case.

======================================
Function GetValues(s As String, prefix As String) As String
    Dim re As Object, mc As Object, m As Object
    Dim v() As Variant, i As Long
Set re = CreateObject("vbscript.regexp")
    With re
        .Pattern = "\b" & prefix & "(\d*)"
        .ignorecase = True
        .Global = True
    End With
If re.test(s) = True Then
    Set mc = re.Execute(s)
    ReDim v(0 To mc.Count - 1)
    For i = 0 To mc.Count - 1
        v(i) = CLng(mc(i).submatches(0))
    Next i
    GetValues = Join(v, " & ")
End If
===================================

ronrosenfeld say: Here's another UDF that does not use regular expressions.  It also depends on the format of the substrings being as described in my first response:

===========================
Option Explicit
Function GetValues(s As String, prefix As String) As String
    Dim v
    Dim i As Long
   
v = WorksheetFunction.Trim(Replace(s, ",", " "))
v = Replace(v, prefix, "")
v = Split(v)

For i = 0 To UBound(v)
    If IsNumeric(Left(v(i), 1)) Then
        GetValues = GetValues & " & " & Val(v(i))
    End If
Next i

GetValues = Mid(GetValues, 4)
End Function

A mobile phone must not restart randomly.
Possible defects are the battery, SIM card, SD memory card, or the phone.

I suggest try rule out the phone itself by removing the SIM and SD (if any), connect the charger and see if it still restarts randomly while used and connected to WiFi?
If all is well disconnect the charger, check again.
If all is well add the SIM, check again.

If all is well add the SD, check again.

If the phone itself appears faulty then backup, apply the recovery tool, start up the phone without SIM, without SD, without adding your Microsoft account, and check again.
This is to rule out any software or configuration issues. Next step is the place of purchase and a warranty claim if the restarts continue after a recovery.

Replace the battery, SIM or SD if any of those appear to cause the restarts.
