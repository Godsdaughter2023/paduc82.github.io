If you do not want to have multi-boot option, the safest way to do this is to disable the list of operating system on start-up. Please follow the steps mentioned below to disable the list of operating system:

1.   Open System by clicking the “Start” button , right-clicking “Computer”, and then clicking “Properties”.

    I decided to do some cleanup to see if it would improve things a bit. And it did.

    For the benefit of all those looking for a solution out there, here is the list of actions I took on the remote machine, in no particular order:

    • Uninstalled OpenVPN
    • Uninstalled Docker
    • Uninstalled the NordVPN client
    • Uninstalled VMWare Player
    • Uninstalled Hyper-V (Start menu, type "feature" and select "Turn Windows features on or off", then uncheck the whole "Hyper-V" group)

    The rationale behind this cleanup routine was to get rid of the many (about 10) virtual network adapters that were installed on the machine as a result of all the virtualization tools I was using over the years.

    Once all the above was done, I was left with a few Hyper-V virtual network adapters. Still following the same rationale, I removed the remaining Hyper-V adapters using the method described here: 

    https://blogs.msdn.microsoft.com/jjameson/2011/03/14/removing-stale-network-adapters-in-hyper-v-vm/
    For that particular part, I encourage you to be extra careful, so that you don't delete a device that shouldn't be deleted.

    All these (or a combination thereof) made the issue disappear, and I was able to use Remote Desktop without any other issues.

    This is what worked for me, but I recognize that it might not work for you, because you have none of this software installed (or need to keep it), so here is a workaround that might work for you: use Parsec (https://parsecgaming.com/) to achieve a similar result (not a sponsor). It's free to use, has clients for Windows 7 and up, Mac OS, Ubuntu Linux, so it's pretty versatile that way. And I never had any trouble using it.
