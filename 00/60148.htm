Are the Performance Graphs Accurate? - Microsoft Community

miketer say:

Are the Performance Graphs Accurate?

When I purchased my new Intel Core i5-3330S @ 2.70GHz system a year ago, I expected it to be faster than my old Intel Pentium T4200 @ 2.00GHz system but it seemed significantly faster than I expected. There seems to be a general consensus on the internet that Windows 8 does use multiple cores and it is up to the applications to use multiple cores to make applications run faster.

Comparing processor benchmarks:

  • Using single thread (assume that means single core, single thread), the i5 is 2.15x faster.
  • Comparing all cores, the i5 is 4.92x faster.
  • Comparing all cores of the i5 against one core of the Pentium is 7.28x faster.

Intel Pentium T4200 @ 2.00GHz Benchmarks

Intel Core i5-3330S @ 2.70GHz Benchmarks

Although memory, disk access, and graphics may be faster in the i5 system, the benchmarks should give a reasonable comparison between the way the average application runs.

I believe in older Windows versions, when Windows booted up, system processes was serial where one process was run to completion before another process was started using a single core. I suspect that under Windows 8 after basic functionality of Windows has been established, then system processes can be run on all cores allowing any core to be executing multiple system processes and task switching between processes. It also appears that when background applications are loaded on boot up, those can also be run the same way. If this is true, then that would explain why Windows 8 boots up so much faster than my old system.

To try to determine if this was likely happening, I initiated the task manager and displayed the performance of all cores. I can't do this during the boot operation but what I noticed is that when the task manager is run, all 4 cores were executing near 100% when initializing. In order for that to occur, at least 1 process per core and probably more would need to be running during initialization. The first spike in the following graphics are the performance of the 4 cores when the task manager is invoked. 

That doesn't prove that all Windows processes are running independently but it seems to give a good indication that system processes are running simultaneously on different cores.

That seems to confirm what may be happing during boot up but I wondered if Windows 8 could somehow use multiple cores when running applications. Since Norton virus scan was really bogging down my old system and a single core on my new system was only 2.15x faster, I wondered why I never noticed when Norton virus scan was running when using my new system. I didn't think it would be possible to use more than one core unless the Norton application was written to use multiple cores since basically the Norton application scans memory for viruses and then reads a file, scans it for viruses, and then reads the next file and scans it for viruses. Since Norton can't scan for viruses until the file is in memory and doesn't initiate another file read until the previous file was scanned, I didn't expect to see multiple cores running but to my surprise multiple core were executing.

In the above graphs other than the 1st spike, Norton virus scan is running and there is a point where processor usage for the 1st core peaks to about 70% usage 3/4ths into the graph. If the graphs of the other cores are inspected during the same time, 2 are indicating about 20% usage and the 4th core indicates about 10% usage. That would be over 100% of a single core usage. The core usage seems to be attributed to Norton since during idle time, very little usage (near 0%) is indicated. This could explain why I've never noticed Norton virus scan running on my new system.

To try to understand more about how Windows 8 works, I decided to invoke three browsers (IE11, Firefox, and Chrome) one after another closing one before launching the next one and watch the usage of the cores. I decided to use three different browsers since it was possible that IE11 was designed to use multiple cores but I doubted that all three were designed to use multiple cores. The following are the graphs of that operation and it can be seen that there are three main spikes in all cores when each browser is initiated which seems to indicate that Windows 8 is using multiple cores for those applications. 

Does anyone understand how Windows 8 is able to utilize multiple cores for applications? If Windows 8 really is utilizing multiple cores for applications that are not designed for multiple cores, then the I'll be paying much more attention to the performance of all cores instead of primarily one core when I purchase a new laptop.

miketer say:

After further investigation, it appears that Windows 8 is not doing anything special to support multiple cores than what any other operating system does including Windows Vista and Windows 7 which are the first two Windows operating system to support multiple cores. Although Windows XP and Windows Server 2003 supported multiple processors, applications were limited since those operating systems weren't designed to handle "race" conditions. Windows Vista was Microsoft's first attempt to handle "race" conditions that multiple processors or cores could cause.

From my investigation, it appears that Norton and possibly the browsers are designed to use multiple cores. When running a full Norton virus scan on my old Windows Vista system with dual Pentium processors, initially it appeared that both cores were utilizing about 30% of the processor power but then after 15 minutes, utilization wet to 100% on both cores and flat lined there for about another 15 minutes before utilization dropped but within a couple of minutes, utilization returned to 100% and flat lined again. This repeatedly occurred until the virus scan completed. You can see in the following graph that both cores start having high utilization and then both cores hit 100% and remain there.

Surprisingly Windows 8 on my new system also flat lined at 100% utilization for all 4 cores but where my old Windows Vista system took over 2 hours to complete the virus scan with over 75% of that time was flat lining at 100% utilization, my new Windows 8 system only took about 15 minutes to complete the virus scan with only about 2 minutes flat lining at 100% utilization. The following are the graphs of Windows 8 flat lining at 100% utilization for all 4 cores.

The reason that I believe thta Norton is designed to use multiple cores is that when I looked at the CPU usage of all the processes, the Norton process was utilizing almost 100% of the CPU power that was being used. Processes such as the interupt handler, the display manager, system, and one of the Service Host processes were using some CPU power but the total typically accounted for less than 1% of the CPU power.

It is unknown how many cores Norton virus scan can use but it appears to be at least 4 since we can see Norton using 4 cores on my new system. In order to run that on a two core system, the Norton application would likely create two threads for each core. If the Norton application was designed for 8 cores, then four threads would be created for each core. The operating system would likely see each thread as a task and would task switch between threads on each core. This may be why I see such a slowdown of applications on my old system when Norton is running. If Norton was running 4 threads per core with 100% utilization and my application was a single threaded application, my application would only get 1/5th of the CPU usage causing it to run 1/5th slower than normal.

I probably never noticed the slowdown on my new system since the virus scan uses significantly less time and core utilization at 100% is only for about 2 minutes.

fahimulla k say:

Hi,

The Network Access Protection (NAP) agent service collects and manages health information for client computers on a network. Information collected by NAP agent is used to make sure that the client computer has the required software and settings. If a client computer is not compliant with health policy, it can be provided with restricted network access until its configuration is updated. Depending on the configuration of health policy, client computers might be automatically updated so that users quickly regain full network access without having to manually update their computer. By default the Startup type of Network Access Protection (NAP) agent service is Manual under services.msc.

If you’ve been trying to put Linux or Windows on Mac hardware but receiving, “Select CD-ROM Boot Type”, and then subsequently your keyboard becomes unresponsive to actually input any value, try this:
