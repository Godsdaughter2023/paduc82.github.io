Text Message Notification Sound - Microsoft Community

evansto say:

Text Message Notification Sound

Why does my text notification sound change when I receive a 2nd message from the same person?  My first notification sound is the tone I set, but the 2nd subsequent sound is just a "ding" sound and (1) I don't know why it changes  - I would prefer my default sound and (2) where is the setting that determines the 2nd sound?  The issue is the 2nd sound is too low in volume and I miss text messages.
albuquerquefx say:

The "text notification" only plays if you are not actively looking at that conversation. The "alternate notification" (ding!) plays when a message is added to the conversation that is currently being viewed.

Here is an example:

  • Your phone is in your pocket and the screen is off
  • Your phone makes the "text notification" sound.
  • You pick up your phone, activate the screen, pull down the action bar, and touch your new message
  • You are now inside the messaging interface, viewing the message.
  • IF you stay on this screen, all subsequent messages to this conversation will result only in the "alternate notification" (ding!) sound
  • If you leave this screen (another app, or the home screen) your "text notification" sound will play

This behavior cannot be altered in the current version of the software. My suggestion is to either simply read your messages from the notification area and swipe them away, or else read your messages from the app and then return to the home screen.

joeu2004 say:

My guess is that Excel is busy recalculating. To demonstrate (but not a solution), temporarily set Manual calculation mode, and see if the problem goes away.

Some formulas are especially susceptible to recalculation delay. Notably, formulas that use "volatile" functions like OFFSET, INDIRECT, RAND, RANDBETWEEN, etc.

And if those formulas refer to whole-column ranges like A:A instead of limited ranges like A1:A10000, each recalculation might be processing 1+ million rows, even if there is no data to process.

Finally, some functions cause recalculation delays, even neither of the above applies. In Excel 2003, I encountered that when I pasted large amounts of data into a column referenced by COUNTIF. Oddly, I could not duplicate the problem when I tried; but I know that COUNTIF was the culprit.

I'm pretty sure that particular problem was fixed in Excel 2007 or 2010. But there might be others like it.

Bottom line.... Review your formulas. Avoid "volatile" functions or large amounts of cell that depend on cells with time-consuming "volatile" functions. And most importantly: avoid using whole-column ranges unless you truly believe you might have 1+ million rows of data(!).

If that does not help, there are tools out there (for a price) that claim to isolated bottlenecks. I cannot vouch for any of them.

Good luck!

PS.... I also encountered instances where charts sometimes cause recalculation delays. And of course, any VBA function (UDF) is a potential culprit.

macropod say:

You should be able to use something like:
{IF{MERGEFIELD Annual_Summary_NW_PS \* Caps}="Y" {INCLUDETEXT "V:\\Murray\\Feb2014_Eoy\\2014 Mail Merge Data\\1_CEO_Summary.docx"}}

Note that you don't need the MERGEFORMAT switches. Note also that all the field brace pairs (ie '{ }') are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message.

If, however, you'd prefer to have all the variable content in a single document, you could use:
{QUOTE{INCLUDETEXT "V:\\Murray\\Feb2014_Eoy\\2014 Mail Merge Data\\1_CEO_Summary.docx" "BkMk{IF{MERGEFIELD Annual_Summary_NW_PS \* Caps}="Y" "A" "B"}"}}

where 'BkMk' is a bookmark's prefix and the 'A' and 'B' are bookmark suffixes for that bookmark.

When it comes to handling multiple records in the same document, things start getting complicated. You could, for example, use Word's Catalogue/Directory Mailmerge facility for this (the terminology depends on the Word version).
