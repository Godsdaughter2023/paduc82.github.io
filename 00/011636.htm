macropod say:

Unfortunately, you can't control the format of text inserted via the \b or \f MERGEFIELD switches - you'd need to use an IF field for that. Thus, taking the examples from your previous thread (http://answers.microsoft.com/en-us/office/forum/office_2010-word/remove-an-unwanted-return-in-a-list-with-mailmerge/b8899458-25ec-49a0-a2d1-d206120f9c4c), you could use fields coded as:

{IF{MERGEFIELD TolSitMin}<> "" "Sitting: {MERGEFIELD min}¶
"}{IF{MERGEFIELD TolStndMin}<> "" "Standing: {MERGEFIELD min}¶
"}{IF{MERGEFIELD TolWlkMin}<> "" "Walking: {MERGEFIELD min}¶
"}Content following mergefields

where ¶ is a real paragraph break or line break. Note how the final mergefield is attached to the start of the paragraph following the series.

suzanne s. barnhill say:
If this is the same merge that you described in a previous post, where your mergefield was part of an IF field, you can use "Sitting: { MERGEFIELD "min" }" as the TrueText or FalseText in the IF field.
