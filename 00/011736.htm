Security Essentials scan time lengthy

Idahosteve

greginmich say:

The scan time for Security Essentials on my windows vista is about 3hrs and 30 min for a full scan.  Malware bytes can do a full scan in 23 minutes now. Is there a download or fix to speed up the Full Scan Time on Security Essentials.

Thanks,

Idahosteve

There isn’t any “quick and easy” fix for this issue because there are so many different factors involved. First, you need to keep in mind that an MSE Full Scan will be paused if the system idles to sleep during the run, and that the run time will be proportionately extended. If this is a significant problem, then you might consider scheduling your own Full Scan task in Task Scheduler, where your task will have the option to ignore the “sleep-after” setting, along with other important options that will allow the scan to run completely unattended; to run on restart if a scheduled start is missed; etc.:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/mse-scan-schedule-off-by-days/f0e68e51-02d6-4fce-bf0b-999b1f0268c8

Manageable Factors

1.) The length of a Full Scan quite obviously depends on the number; size; and type of files being scanned. So some of the things that you can do to reduce the run time for a Full Scan would be:

a.) Clean up your disk, including Restore Points, to reduce the overall number of files scanned.

b.) Use path exclusions for large collections of media files, backup files, etc., to reduce the number of large files that get scanned.  

c.) Turn off the scanning of archive files (in the “Advanced” settings) to eliminate the considerable overhead involved in unpacking and scanning the contents of container files.

2.) Any conflicting (real-time) AV program will radically slow down an MSE Full Scan by causing MSE’s on-access module to continuously rescan the conflicting scan engine and all of the files that it’s scanning (on–access) at the same time that the Full Scan is trying to scan your files. Truly excessive scan times indicate either the presence of a conflicting process or a corrupt installation. Note that the paid version of Malwarebytes does include an on-access module, so the potential for conflict exits. I believe that the recommendation for minimizing the conflict with MSE involves setting up mutual process exclusions to prevent each of the scan engines from scanning the files accessed by the other engine.

3.) The most important manageable factor in determining the length of a scheduled MSE Full Scan isn’t quite so obvious, and it typically gets left out of the equation entirely – it’s MSE’s unique “CPU throttle” setting. This feature is actually doubling the run time for a scheduled Full Scan when it’s set at the default level. So turning off CPU throttling is definitely worth a try if the objective is to reduce scan times.

jasimali say:

MS Outlook 365ProPLUS error Out of memory or System Resources .Close

Dear Microsoft Support

I can't open any PST file in  Outlook365 Pro Plus  ,  It Shows  following error while open PST Archive file .

"Out of memory or System Resources .Close  some windows or programs and try again "

I tried following solution methods . Still not solved the Problem.   How do I resolve this issue.

Solution : Reduce the Memory Cache Size for Each PST File

If  still wish to keep all the PST files in Outlook but the Outlook error persists, here is another approach. You can reduce the memory cache size which Outlook assigns to each PST files. Take the following steps:

  1. For a start, close Outlook and head to “Start Menu”.
  2. Then type “regedit” in search box and press “Enter” key button.
  3. In the Registry Editor, browse to the keys below: “HKEY_CURRENT_USER” > “Software” > “Microsoft” > “Office” > “<version>” > “Outlook” > “PST”.
  4. Next right click on “PST” key and choose “New” > “DWORD Value”.
  5. Name the newly created value as “UseLegacyCacheSize”.
  6. Later specify its value data as “1”.
  7. Ultimately close Registry Editor and restart Outlook.

Thanks

Jasim
