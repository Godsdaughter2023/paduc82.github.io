john w. vinson say:

There are two types of parameter: a prompt such as you used, and - much more useful! - a form reference.

Create a little unbound form named, say, frmCrit. Put a combo box on it named (say) cboProduct. Put a command button on the form to launch a Report or a Form displaying the data that you want to see.

The criterion would be

=[Forms]![frmCrit]![cboProduct]

Do not use the asterisk - it's usable only as a wildcard in a LIKE query, which is not appropriate for a numeric (concealed) Lookup Field ID number.

See http://www.mvps.org/access/lookupfields.htm for a critique of the Lookup Field datatype, which a lot of us consider a "misfeature".

john w. vinson say:

You can indeed filter a form using a combo box on the same form, and you don't even need a button.

Instead, use some VBA code in the AfterUpdate event of the combo box. (If that's what you call "advanced methods", well, you're doing an advanced task!). Let's say the combo box is named cboFilter (it doesn't matter in this case what the form is named). Open the form in design view; select the combo box; view its Properties; and click the ... icon next to the line that says "After Update" on the Events tab. Choose "Code Builder". Access will give you the Sub and End Sub lines for free - edit the code to:

 

Private Sub cboFilter_AfterUpdate()
If IsNull(Me!cboFilter) Then
  Me.Filter = ""  ' Clear the filter if the user clears the combo
  Me.FilterOn = False
Else
  Me.Filter = "[ProductID] = " & Me!cboFilter
  Me.FilterOn = True
End If
End Sub

This assumes that the relevant fieldname is ProductID, and the combo box has the same product ID as its bound column.

ken sheridan say: You might like to take a look at the fileComboDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

This little demo file is intended to illustrate the use of correlated combo boxes, i.e. where a selection in one limits the items listed in another, and includes an option to open a form for 'drilling down' through hierarchies.  This form has three combo boxes, via which the rows returned in the form can be restricted progressively.  It does this by means of parameters in the form's query which reference the combo boxes.

In your case, as you wish to use only a single combo box the query would be much simpler, but in principle the same, e.g.

SELECT *
FROM [YourTable]
WHERE (ProductID =[Forms]![frmCrit]![cboProduct]
OR [Forms]![frmCrit]![cboProduct] IS NULL);

The important thing to note here is that the expression in the WHERE clause tests not only for the ProductID value being equal to the value of the combo box, but also for the combo box being Null.  This means if nothing is selected all rows will be returned, if a selection is made only rows which match the selection will be made.

In query design view you'd enter:

[Forms]![frmCrit]![cboProduct] OR [Forms]![frmCrit]![cboProduct] IS NULL

in the criteria row of the ProductID column.  The cboProduct combo box in the form would be set up along these lines:

RowSource:     SELECT ProductID, Product FROM Products ORDER BY Product;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the dimension is zero to hide the first column.  In the combo box's AfterUpdate event procedure requery the form with:

Me.Requery

My Email provider is ATT--and I do find that--while my Email is synced wirelessly over their 3G network, my Calendar tasks are NOT being synced.  I have to physically connect to my laptop--and then my Calendar will sync.  (Note: This I perceive to be a limitation of the Email service and not of the phone itself. I'm not really connecting to an "Exchange Server"--just a POP Email server.)  If you are lucky enough to connect to a true "Exchange Server" (e.g. corporate server), I think your Calendar will sync without issue.

So, syncing with the PC may be the best you can do--to get your Calendar items.  What follows are some things to check if you wish to use a hard-wired connection to your laptop and synch with your "Windows PC".

In the Windows Mobile Device Center (WMDC), go to Mobile Device Settings/Change Content Sync Solutions.  (If you don't have WMDC running on your computer, you will need to locate it online and download it.  It gets patched occasionally.  So--even if you have it, you might download the latest version.)

In this Change Content Sync Solutions dialogue,

Make sure "Calendar" box is checked

Click on "Sync Settings"

Make sure "Windows PC" is checked 

Then---plug your phone into a USB port and sync should begin.

If it does not begin, you can initiate it manually--

From the phone, click the (middle) Microsoft menu button (at bottom of phone).  On this menu click the green circle with two arrows (Sync icon).  If you don't have this icon, click "All Applications" and go find it there.

I hope this helps.  Have faith--once you work through some configuration issues--and understand the limits of your Email server, this is a pretty good phone.

Bonus Information: While you are in "All Applications", check out the game called "Teeter".  I am not much for games but recently ran across it.  Pretty nifty game--when you have some time to kill.
