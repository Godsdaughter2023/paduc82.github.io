In your Normal.dotm template, create an AutoOpen macro containing the following commands:

Sub AutoOpen()

With ActiveDocument
    .TrackRevisions = True
    .SpellingChecked = False
    .CheckSpelling
End With

End Sub

If you do not want those settings for some documents, hold down the SHIFT key while opening them as that prevents AutoMacros from running.

This isn't happening anywhere else on my system. Emails display the ampersand correctly and Wordpad displays them, properly. Doesn't matter which font either.

James

hansv mvp say:

The & character is used to specify the so-called accelerator. For example, if you set the caption of a label or command button to 'Last &Name', it will be displayed as 'Last Name' and you can press Alt+N to activate the control associated with the label or click the button.

If the character after the & is a space, that space will be underlined, in other words you'll see an underscore _.

 

To include an ampersand in a caption, double it: 'First && Last' will be displayed as 'First & Last'.
