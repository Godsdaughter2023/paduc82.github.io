Macro to strikethrough parts of text in a cell - Microsoft Community

adammk say:

Macro to strikethrough parts of text in a cell

I want to list several dates within the same cell in excel using alt+enter to separate the dates. The problem is I want all but the last  date to be struckthrough. Essentially, anytime a new date is typed into the cell I want the older dates to become struckthrough leaving the new date the only non-struckthrough date.

The contents of the cell would look something like this:


hansv mvp say:

Let's say the cell is C6.

Right-click the sheet tab and select View Code from the context menu.

Copy the following code into the code window that appears:

 

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim s As String
    Dim p As Long
    If Not Intersect(Range("C6"), Target) Is Nothing Then
        Application.EnableEvents = False
        Range("C6").Font.Strikethrough = False
        s = Range("C6").Value
        ' Position of last line break
        p = InStrRev(s, vbLf)
        If p > 0 Then
            Range("C6").Characters(1, p - 1).Font.Strikethrough = True
        End If
        Application.EnableEvents = True
    End If
End Sub

 

Switch back to Excel.

Remarks:

  • You will have to save the workbook in a macro-enabled format (.xlsm, .xlsb or .xls, but NOT .xlsx).
  • You will have to enable macros.

wayne-i-m say:

Format your table fields as Long Time
MyStartTime  MyEndTime

 

Then in your query use this
TimeElapsed: Format(DateDiff("s",[MyStartTime],[MyEndTime])/86400,"hh:nn:ss")

 

Of course if you don't need this much detail (ie. seconds) you can always use
DateDiff("n",[MyTable].[MyStartTime],[MyTable].[MyEndTime])\60 & " Hours " & DateDiff("n",[MyTable].[StartTime],[MyTable].[MyEndTime]) Mod 60 & " Minutes"

 

There are lots of ways to do this but it depends on the use you will put the results to and how much details you need as to which method you use

marshall barton say:

You need to enter the time, not just a minute.  If the time starts at, say, 9:54:15 and ends at 10:03:12 you will have a problem subtracting your proposed 54:15 from 03:12 and if you should ever run into an odd situation where an interview takes more than an hour, you would be lost.  If you enter the times in hour, minutes and seconds, the only difficulty would occure if an interview crosses a midnight boundary (in which case you should also enter the date part.

With the hour, minutes and seconds, you should not have a problem entering the times.  You could then use the DateDiff function to calculate the number of seconds, or just subtract to get the interview duration as a time value that can be formatted anyway you like.

If you ignore all the problems with just entering minutes and seconds (e.g. 54:13) then Access will think you entered hous and minutes.  That could then be converted to minutes and seconds by dividing the "time" by 24
