To recreate the System partition, you would have to boot with Windows 10 install media, choose Repair your computer and navigate to a Command prompt.  Assign a drive letter to System and check if your Windows partition shows as C: or another letter.  Usually I choose S: for the System partition to make the commands simple for recreating the System environment.

format quick fs=fat32 label=SYSTEM

eric legault (microsoft mvp) say:

All items that are deleted in Outlook go to the Deleted Items folder (but you may know that), but you don't get a warning.  If an item is deleted in the Deleted Items folder, or if you choose to permanently delete an item, you DO get a warning (unless that option is turned off).

If you want to write code to intercept when an item is deleted, you can get a handle to the message's object (like the MailItem object) and capture that object's BeforeDelete event.  Every folder also has an associated Items collection, and you can trap the Items.ItemRemove event as well.

However, keep in mind that all these events are not 100% failsafe, and can require quite a bit of effort to implement properly.

Other than that, your best bet is to ensure that you have backups of your .pst files.  If you are using Exchange, administrators have the ability to restore Mailbox data.  Some Exchange environments also provide third-party e-mail archiving and recovery tools that can aid in data retention.

Give this macro a try (after first changing the three Const statements to match your actual setup)...

Sub SplitOutFixedLengthFile()
  Dim X As Long, Z As Long, LastRow As Long, FileNum As Long
  Dim TotalFile As String, Records() As String
  Const StartRow As Long = 1
  Const Source As String = "c:\temp\FixedLengthFile.txt"     'Full path
  Const DestinationPath As String = "c:\temp\"                      'Note the required trailing backspace
  FileNum = FreeFile
  Open Source For Binary As #FileNum
    TotalFile = Space(LOF(FileNum))
    Get #FileNum, , TotalFile
  Close #FileNum
  Records = Split(TotalFile, vbCrLf)
  TotalFile = ""
  LastRow = Cells(Rows.Count, "A").End(xlUp).Row
  For X = StartRow To LastRow
    FileNum = FreeFile
    Open DestinationPath & Cells(X, "B").Value For Output As #FileNum
    For Z = 0 To UBound(Records)
      If Cells(X, "A").Value = Mid(Records(Z), 3, 2) Then Print #FileNum, Records(Z)
    Next
    Close #FileNum
  Next
End Sub
