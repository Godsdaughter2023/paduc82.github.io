Preventing Corruption of Normal.dot
 
Applies to Word 97, 2000, 2002, 2003
 
 
You are not alone in your problems, John. It seems that others have had these same problems. Invariably, the one suggestion that everyone gave is that it is unsafe to make changes to either the Standard or Formatting toolbars. Doing so, while certainly possible, increases the likelihood of corrupting the Normal.dot file. Instead, custom toolbars should be created, and then all your custom tools placed in the custom toolbar. The custom toolbar can be placed right beside the Standard or Formatting toolbar, if desired.
 
The other guideline is to limit the changes you make to Normal.dot. The best way to do this is to save macros and other customizations to your own global template. You can use any name you want for the template; there is nothing unique about a global template, other than it is stored in the Word Startup folder. Place it there, and it is loaded when Word is loaded. This means that everything in the global template is loaded and available to all your documents, just like Normal.dot.
 
The advantages to using a global template should be immediately obvious. You run less risk of corrupting Normal.dot, you can easily create backups of the global template, and you can more easily transfer the global template file to another user on another system. (This makes sharing macros and toolbars much easier.)
 
Another idea is to create two global templates, hypothetically named Global.dot and GlobalTest.dot. When you are developing macros and you want to test something, store it in the GlobalTest.dot template. When it is debugged and running smoothly, transfer it to the Global.dot template. This makes it less likely that an errant macro or questionable customization will mess up your “production” templates—Normal.dot and Global.dot.
 
(Thanks to Judy Stein, Phil Rabichow, Kim Murdock, and Charles Kyle Kenyon for contributing to this tip.)

Finding In-Use Styles in a Macro
 
 
 
Last week I featured a macro that indicated how to determine, in a macro, how many styles are available in a particular document. In the tip, I also included code on how to exclude from the count the built-in styles, and gave the impression that this would result in the number of styles “in use” in a document.
 
This is not the case—the macro code I provided returns the number of styles defined by the user, and nothing more. It does not provide any indication of whether the style is actually in use or not.
 
The only way to get that information using a macro is to do a series of Finds for each of what Word counts as the “in use” styles, to find out which of those styles are actually applied to text in the document. The macro required to do the Finds is itself more complex than one would expect, because unlike the Edit Find dialog, VBA Finds don't search all of the so-called “stories” in the document (the main document, the headers, footers, etc.) in one go. Public Sub DeleteUnusedStyles()
 
 
 
'Simon Jones, MillStream Designs Ltd, 05/03/2007
 
'Email: *** Email address is removed for privacy ***
 
 
 
Dim objStyle As Word.Style
 
Dim intBuiltInStyles As Integer
 
Dim intStyle As Integer
 
 
 
'Count the number of built-in styles in this document
 
 
 
For Each objStyle In ActiveDocument.Styles
 
  
 
    If objStyle.BuiltIn Then
 
        intBuiltInStyles = intBuiltInStyles + 1
 
    End If
 
  
 
Next objStyle
 
 
 
'Show summary and ask permission to continue
 
 
 
If MsgBox("There are " & intBuiltInStyles & " built-in styles " & vbCrLf & _
 
          "and " & ActiveDocument.Styles.Count - intBuiltInStyles & _
 
          " custom style(s) in this document." & vbCrLf & vbCrLf & _
 
          "Delete unused custom styles?", vbOKCancel + vbExclamation, _
 
          "Delete Unused Styles") = vbOK Then
 
        
 
    'OK to continue
 
    'Run through all the styles in the document BACKWARDS -
 
    'so we only delete styles from the end of the collection
 
  
 
    For intStyle = ActiveDocument.Styles.Count To 1 Step -1
 
      
 
        If Not ActiveDocument.Styles(intStyle).BuiltIn Then
 
            'This is a custom style
 
            'See if it is in use
 
          
 
            With ActiveDocument.Content.Find
 
                .ClearFormatting
 
                .Style = intStyle
 
                .Execute Format:=True
 
              
 
                If Not .Found Then
 
                    'Style is not in use
 
                    'We can delete it
 
                   
 
                    With ActiveDocument.Styles(intStyle)
 
                        'If this style is linked to another we have to unlink them -
 
                        'otherwise both styles will be deleted which causes problems -
 
                        'this style may be linked to a built-in style which will cause -
 
                        'an error if we try to delete it.
 
                       
 
                        If .Type <> wdStyleTypeCharacter Then
 
                            'Character styles can't be linked from, only linked to
 
                          
 
                            'The Linked property only exists in Word 2007 and above -
 
                            'for previous versions, comment out the "If" and "End If" lines below
 
                          
 
                            If .Linked Then                 '### Word 2007+ only
 
                                'The LinkStyle property only exists in Word 2002 and above -
 
                                'for previous versions, comment out the lines below
 
                              
 
                                If Not .LinkStyle Is ActiveDocument.Styles(wdStyleNormal) Then '### Word 2002+ only
 
                                    .LinkStyle = wdStyleNormal                  '### Word 2002+ only
 
                                End If                                          '### Word 2002+ only
 
                              
 
                                'LinkStyle is a very badly behaved property and this is the -
 
                                'closest we can get to unlinking styles short of copying all -
 
                                'text except the last paragraph mark to a new document.
 
                            End If                          '### Word 2007+ only
 
                          
 
                        End If
 
                       
 
                        'Ignore any error caused by trying to delete
 
                        'a style whose name begins with a space
 
                        On Error Resume Next
 
                        .Delete
 
                        On Error GoTo 0
 
                    End With
 
              
 
                End If
 
              
 
            End With
 
          
 
        End If
 
  
 
    Next intStyle
 
  
 
    'Display summary
 
    MsgBox "There are " & ActiveDocument.Styles.Count - intBuiltInStyles & _
 
           " custom style(s) left in this document.", vbInformation, "Unused Styles Deleted"
 
End If
 
 
 
End Sub

I have this macro that deletes unused user styles from a document. Here it is:
 
This is how to clean out a document of all user styles that are not actively in use—the Style.InUse property is of no help here since InUse is TRUE if the style was EVER used.
 
So, here's a function that checks if a style is actually in the document.
 
The sample routine nukes user styles that are not in the document, and hides built-in styles from the drop down list if they are not actively in use.
 
Sub NukeStyleNotInDocument()
 
 
 
‘ Macro: NukeStylesNotInDocument
 
‘ by Guy Gallo
 
‘
 
' Purpose: Remove all user styles that are not ACTIVELY
 
' in use in the current document
 
' Remove built-in styles that are not ACTIVELY in use
 
' from the style list
 
 
 
'Adapted from a macro by Paul Cornell
 
'http://office.microsoft.com/assistance/2002/articles/pwWordStyles.aspx
 
 
 
Dim strTitle As String
 
Dim astrStyles() As String
 
Dim objStyle As Style
 
Dim objDocument As Word.Document
 
Dim intCount As Integer
 
 
 
' Get the title of the active Word document.
 
 
 
strTitle = _
 
 
 
ActiveDocument.BuiltInDocumentProperties(wdPropertyTitle)
 
 
 
' Store, in memory, each style in the active document,
 
' including whether the style is in use.
 
 
 
 
 
For Each objStyle In ActiveDocument.Styles
 
ReDim Preserve astrStyles(intCount)
 
 
 
'GG Mod
 
 
 
If objStyle.BuiltIn = False Then
 
   If Not StyleInDoc(objStyle) Then
 
      Debug.Print "Removing " & objStyle
 
      objStyle.Delete
 
   Else
 
      Debug.Print "In use: " & objStyle
 
      astrStyles(intCount) = objStyle & " (Style in Use)"
 
      intCount = intCount + 1
 
   End If
 
Else
 
   If objStyle.InUse And Not StyleInDoc(objStyle) Then
 
      objStyle.Hidden = True
 
      Debug.Print "Hiding: " & objStyle
 
   Else
 
      Debug.Print "In Use: " & objStyle
 
      astrStyles(intCount) = objStyle & " (Style in Use)"
 
      intCount = intCount + 1
 
   End If
 
 
 
   '**********
 
End If
 
 
 
Next objStyle
 
 
 
' Create a new Word document to report the results.
 
 
 
Set objDocument = Word.Documents.Add
 
With objDocument.Range
 
 
 
   ' Report the results.
 
 
 
   .InsertAfter "Styles in " & strTitle & ":"
 
   .InsertParagraphAfter
 
   .Collapse Direction:=wdCollapseEnd
 
 
 
   For intCount = 1 To UBound(astrStyles)
 
      .InsertAfter astrStyles(intCount)
 
      .InsertParagraphAfter
 
      .Collapse Direction:=wdCollapseEnd
 
   Next intCount
 
 
 
End With
 
 
 
MsgBox Prompt:="Done!"
 
 
 
out:
 
End Sub
 
 
 
Function StyleInDoc(sty)
 
 
 
'gg
 
'Check if a style is in active use.
 
 
 
Dim myRange As Range
 
Set myRange = ActiveDocument.Content
 
 
 
With myRange.Find
 
   .ClearFormatting
 
   .Style = sty
 
   .Text = ""
 
   .Replacement.Text = ""
 
   .Forward = True
 
   .Wrap = wdFindContinue
 
   .Format = True
 
   .MatchCase = False
 
   .MatchWholeWord = False
 
   .MatchWildcards = False
 
   .MatchSoundsLike = False
 
   .MatchAllWordForms = False
 
   StyleInDoc = .Execute(FindText:="", Forward:=True, _
 
   Format:=True) = True
 
   .ClearFormatting
 
End With
 
 
 
End Function

marshall barton say:

Required is a quick way of setting the field's Validation Rule to"  Is Not Null.  Because the NewCriteria field might be Null, you can not make it required.  Not only that, but because two fields are involved you can not use a Validation Rule on the NewCriteria field.

The preferred approach is to use a table Validation Rule (in the table's properties):

    [Choices] <> "Other" OR ([Choices] = "Other" AND NewCriteria Is Not Null)

Instead of the GotFocus event, the form's Current event and the combo box's AfterUpdate event will need the same line of code:

   Me.txtNewCriteria.Visible = (txtChoices = "Other")

You can use Enabled instead of Visible if you agree that it is less confusing than using Visible.

Last page footer after mail merge. - Microsoft Community

plotguru say:

Last page footer after mail merge.

Hi,

I am having a headache trying to get a footer to appear on the last page of a document which has a variable page count depending on the merge data.

I have tried {IF {PAGE} = {NUMPAGES} "True" "False"}

Which works perfectly in the template, but when the document is produced via my app (basically runs a mailmerge to the template) the {PAGE} element in the statement is set at 1 for every page.

I tested this with the following:

{IF {PAGE} = {NUMPAGES} {PAGE} {PAGE}} {PAGE} {NUMPAGES}

This resulted in the footer on each page appearing as - 1 1 3 - 1 2 3 - 1 3 3 with a 3 page result as an example.

Is there any other possible way to ensure a footer appears on the final page of a document that goes through a mail merge process?

macropod say:
Hi , The problem you're having is caused by the fact that IF fields are resolved to their final values during a mailmerge. Accordingly, as each record is merged, the IF fields pertaining to that record get evaluated and, of course, the last page in that record is the last page in the document. Hence the result not being what you're after. You can work around this by using a field coded along the lines of: {={NUMPAGES}-{PAGE} \# "'True';;'False'"}
