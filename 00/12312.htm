nakiamatthews say:

Can I Exclude Certain Domains or Contacts From the Directory?

yangzheng_msft6300 say:

Hi Nakia,

From your description, my understanding is that firstly you don’t want the accounts in the 2 domains to see the accounts in the other 4 domains in the directory or address list. If so, you can create a separated address list for the accounts in the 2 domains so that they can only see the accounts in the 2 domains. Here are the detailed steps:
1. Make sure that you admin account has the Address List role permission. You can check this in Office 365 EAC (Exchange Admin Center)->permissions->admin role, and add the Address List role to the admin role group where your account is a member.
2. Connect to Exchange Online with PowerShell.
3. Turn on address book policy routing.
4. Run the command below to create a new room list.
New-AddressList -name "Roomlist1" -RecipientFilter {(CustomAttribute1 -eq "domain1") or (CustomAttribute1 –eq “domain2”)}
5. Run the command below to create a new Address List which will contain the accounts in domain1 and domain2.
New-AddressList -Name Addresslist1 -RecipientFilter{(CustomAttribute1 –eq “*** Email address is removed for privacy ***”) or (CustomerAttribute1 –eq “*** Email address is removed for privacy ***”)}
6. Run the command below to create a new Global Address List which will contain the accounts in domain1 and domain2.
New-GlobalAddressList -name "GAL1" -RecipientFilter {(CustomAttribute1 -eq "*** Email address is removed for privacy ***") or ( *** Email address is removed for privacy ***)}
7. Run the command below to create a new Offline Address Book.
New-OfflineAddressBook -Name "OAB1" -AddressLists "GAL1"
9 Run the command below to create a new Address Book Policy.
New-AddressBookPolicy -name "ABP1" -AddressLists "Addresslist1" -OfflineAddressBook "\OAB1" -GlobalAddressList "\GAL1" -RoomList "\Roomlist1"
10. Then run the command below to assign the new policy to the accounts in domain1 and domain2.
Get-Mailbox -Filter {(CustomAttribute1 -like "*** Email address is removed for privacy ***") or (CustomAttribute1 –like “*** Email address is removed for privacy ***)} | Set-Mailbox -AddressBookPolicy ABP ABP1

Then you also don’t want some contacts to show up in the address list. If the contacts are listed in as mail contacts in the Office 365 EAC, you can do this by running the command below.
Set-MailContact -Identity *** Email address is removed for privacy *** -HiddenFromAddressListsEnabled $true

bob jones aka: cybertaz say:
OK, glad you were able to resolve it, but just for further clarification :-)

What you've been looking at is item #4 in my previous response -- the Quick Styles Gallery. It has nothing to do with what Styles are actually available. Any Style [built-in or user-defined] can be set to be included in the gallery based on a check box in the Format> Styles - Modify dialog for the Style. However, just because a Style does not appear in the gallery doesn't mean that it isn't available in the document. Likewise, removing a Style from the gallery does not delete the Style. The gallery basically is a tool for grouping/organizing Styles into sets. Also, it is a work-in-progress making its Mac debut appearance in Office 2011. See  Apply, change, create, or delete a Quick Style set for more info.

There is no single switch or reset button to restore Word to default settings. Word is a complex program & different aspects of it are stored in different locations. It's also a part of an integrated suite of applications with shared features, so some of those settings are not directly controlled by any of Word's dedicated user files... E.g., Themes & User Info are shared by Excel & PowerPoint.

As for the Preferences property list [.plist] I have a feeling you looked in the Library folder on the main level of your Macintosh HD. None of the preferences specific to your User Account are stored there. The Preferences folder for each User Account is stored in that account's Library folder. See this article for one way of accessing it.

As I emphasized before, however, not all of Word's settings & customizations are controlled by Preferences. Many are stored in the Normal.dotm template, which is the model for the generation of most new documents. Toolbar & Menu customizations, Styles, AutoText Items & Macros are among the content typically stored in the template.

nayana s m say:

Hello Denny,

What type of ‘SimpleSYN’ did you use?

There is no automatic way to sync contacts between Outlook programs on two Computers.

As you have configured POP3 email account type in Outlook then only the emails which are on Inbox folder on server will be downloaded to Outlook. Hence the contacts and calendar items are not downloaded to Outlook. So there is no automatic sync for contacts or calendar items to get synced between Outlook programs on two Computers which holds the same email account.

You may refer to the thread link mentioned below which discusses the similar issue:

http://answers.microsoft.com/en-us/office/forum/office_2010-outlook/i-need-to-sync-outlook-contacts-mail-from-desktop/966770a3-ca5e-e011-8dfc-68b599b31bf5  

For further queries, reply and we would be happy to assist.

Thank you.

roady [mvp] say:

Outlook syncs between mail servers and not between clients.

What you can do is configuring your POP3 address as an Outlook.com address which does offer support synching mail, contacts and calendar between multiple clients including Outlook.
