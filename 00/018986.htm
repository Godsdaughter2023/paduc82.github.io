That's probably a sign of a permissions problem.  I would run ProcMon to trace the registry accesses the regsvr32.exe command is trying to make.  That could show you if there was a key which needed to have its permissions changed.  Etc.  Another possibility would be to check if there are any extra diagnostics created to supplement that error code.  E.g. perhaps in the Event log?  ProcMon could help with that too, e.g. show you if any files were being written to just after the error was detected.
