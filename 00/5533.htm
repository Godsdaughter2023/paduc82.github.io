Hi Pat,

Gord has already provided you with an answer to your substantive question.

However, looking at your code may I make one or two observations?

As written, your Worksheet_Change procedure will convert lower case entries in column Z to the corresponding upper case values, as intended. As an unintended consequence however, this case conversion itself represents a change and this latter change will engender a recursive call to the procedure and initiate a recursive loop. To avoid such recursive loop problems, the standard solution is to deactivate Excel events before any change instruction in the code and restore event handling before the event code is terminated.

Thus, your code might become:

'==========>>
Option Explicit

'---------->>
Sub Worksheet_Change(ByVal Target As Range)
    On Error GoTo XIT
    Application.EnableEvents = False
    If Not Intersect(Range("Z:Z"), Target) Is Nothing Then _
       Target.Value = UCase(Target.Text)
XIT:
    Application.EnableEvents = True
End Sub
'<<==========

However, to allow for the possibility of multiple entries in the relevant column or the entry of a formula which returns a text string, a more flexibly version of this code might be:

'==========>>
Option Explicit

'---------->>
Sub Worksheet_Change(ByVal Target As Range)
    Dim Rng As Range, Rng2 As Range
    Dim rCell As Range

    Set Rng = Intersect(Target, Me.Range("Z:Z"))
    If Not Rng Is Nothing Then
        On Error GoTo XIT
        Application.EnableEvents = False
        For Each rCell In Rng.Cells
            If Not rCell.HasFormula Then
                With rCell
                    .Value = UCase(.Text)
                End With
            End If
        Next rCell
    End If
XIT:
    Application.EnableEvents = True
End Sub
''<<==========

These observations are, however, intended merely as side notes on the code and do not represent an answer to the question which you posed; as already indicated, this has been addressed and answered by Gord.

===

Norman

Your "default contact folder" is located in the PST file on your "D" drive based on your screenshots. That's the folder where you want your primary contacts to end up. Steps to do that are

1 - Open whatever folder contains the contacts you want to keep

2 - type CTRL-A to highlight all the contacts

3 - move mouse over highlighted area - right-click

4 - Select <Move> followed by either <Other Folder> or <Copy to Folder> (would suggest copying instead of moving)

5 - Select the destination folder where you want everything to end up - done

If you have no purpose for the "backup" PST file - then simply remove it from the profile. Either highlight the <Root> of the backup PST file --> right-click --> close OR go back to the Data Files tab in Account Settings and click on <Remove> - either way ends up with the exact same result. Removing the second PST file will result all of the folders associated with that PST file being removed from the Outlook nav pane

Re: the word <Profile>

An Outlook profile contains all the critical setting related to the current email accounts in use. You can have multiple email accounts/data files etc associated with a given profile but Outlook only works with one profile when it's opened. You cannot create a new profile or change which profile is in use from within Outlook - that must be done using the Control Panel <Mail> app.

Can anyone help me try to get this set up using 'Define new multilevel list'? suzanne s. barnhill say:

To make Heading 3 restart in each chapter, you'll need to include Heading 2 (unnumbered) as Level 1 in your multilevel list. Heading 3 will be Level 2 and Body Text Level 3. If you set Level 3 to restart after Level 2 (Heading 3), then it should be unaffected by the lower-level headings. It appears that Heading 4 will also restart after Heading 3, and Heading 5 after Heading 4. Since Heading 6 will be unnumbered, it can be omitted from the list.

The question is where to put the (a) and (i) levels (which could use styles called Body Text 2 and Body Text 3). Assuming they restart after Body Text (Level 3), then you could put them at Levels 4 and 5, but I think I'd be inclined to use those levels for Headings 4 and 5 and drop the Body Text paragraphs to Levels 6 and 7, but set to restart after Level 3.

This will probably require some trial and error with the "Include level from" and "Restart after" settings, but it should be doable. Keep in mind that font formatting and paragraph formatting in general are done in the style, but all indents on the left side (Left, Hanging, First-line) must be set in the Define New Multilevel List dialog.
