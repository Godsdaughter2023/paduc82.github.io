doug robbins - mvp office apps & services (word) say:

Contrary to popular opinion, Word has no integral means of automatically saving the current document, nor of backing-up the current document.

The AutoRecover\AutoSave facility in Word provides the basis to recover a document (or part of it) in the event of a system crash, particularly in pre 2010 verstions of Word, it should not be relied upon to save your bacon.   In Word 2010 however, a facility was introduced to allow you to select that the last autosaved version of a file be kept in the event that you close the file without saving. (File>Options>Save)

It's not that Word 2010 is not regularly creating AutoSave versions of your document, but that the 13 ghosts backup is failing to retrieve those files.

See the following pages of fellow MVP Graham Mayor's website:

http://www.gmayor.com/automatically_backup.htm 

http://www.gmayor.com/SaveVersionsAdd-In.htm

To open an ASD file, you should place it in the following folder

C:\Users\[User Name]\AppData\Local\Microsoft\OFFICE\UnsavedFiles

and then in Word, use File>Info and click on the Manage Versions drop down and then select Unsaved Documents, which will take you to that location.  When you select an .asd file in that location, Word knows how to deal with it, which it may not do if you try and use it by using File>Open.
