hansv mvp say:

When you export to Excel, Access names the exported range. By default, it gives the range the same name as the table.

However, a name in Excel may not begin with a R1C1-style reference.

C1 is the R1C1-style reference for column A.

Similarly, table names such as R37_Test and R5C8_Test will cause the same problem.

To get around this, give the range a different name:

DoCmd.TransferSpreadsheet acExport, acSpreadsheetTypeExcel12Xml, "C1_test", "C:\C1_test.xlsx", True, "MyRange"

This will cause the exported range (and the sheet) to be named MyRange instead of C1_Test.

julesklap say:

If an Excel cell contains certain text, then

If cell A1 contains "ABC" anywhere within that cell, put "Yes" in cell B1.

Or if cell A1 begins with "ABC", put "Yes" in cell B1

Can I put multiple If/Then/Contains statements in cell B1?

myall_blues say:

For contains "ABC":

  Case sensitive - =IF(ISNUMBER(FIND("ABC",A1)),"Yes","No")

  Not case sensitive - =IF(ISNUMBER(SEARCH("ABC",A1)),"Yes","No")

For begins with "ABC"

  Not case sensitive - =IF(IFERROR(SEARCH("ABC",A1),0)=1,"Yes","No")

 Change SEARCH to FIND to make it case sensitive.

  Can also use (not case sensitive) =IF(LEFT(A1,3)="ABC","Yes","No")

You can put multiples by nesting the IF statements within one another.
