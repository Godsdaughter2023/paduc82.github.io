I just bought a new HP computer with Windows 10 Pro.  I want to impost my Outlook files (contacts, old emails, etc) from the old computer which is still on the network, but when I navigate to the old Outlook directory Windows 10 reports that the folder is empty, but it is not because I can see it and use it on the old computer.  How can I import the data if it is not visible?

frederik long say:

Here you go:

  1. Use Windows Explorer on the new machine to navigate to the networked Outlook folder on the old machine.
  2. Mark all .pst files, then press Ctrl+C.
  3. Use Windows Explorer on the new machine to navigate to some Outlook folder of your choice.
  4. Press Ctrl+V to paste the .pst files.
  5. Open Outlook and tell it where to find its .PST files. If unsure how to do this, ask the experts in an Outlook forum:
    http://answers.microsoft.com/en-us/office/forum/office_2010-outlook

pjp1532 say:

Yes, this would be the steps I would take, however when I navigate on the new computer to the networked Outlook folder on the old machine, the Outlook.pst file is not displayed and the Windows 10 computer reports that the folder is empty, hence there is no .pst file to copy or import.  I have gone into Windows 10 settings and made all hidden and all system files visible, but "no joy."

But on the two other networked computers (running Windows 7), the Outlook.pst file in question is visible and working.

I tried the reverse approach, copy the old.pst file using a Windows 7 networked computer and paste it into a temporary directory on the Windows 10 computer, but when I access the temporary directory on the new machine, the Outlook.pst file is not to be seen and the computer reposts the temporary directory is empty!

frederik long say:

Let's keep it simple:

  1. Format a flash disk with the FAT32 file system.
  2. Copy the files to this flash disk.
  3. Paste them into a suitable folder on the new PC.

Problem solved!

malacath say:

I have a similar TP-LINK but my model is TL-WR841N.

Now to get an open NAT on that you have to rely totally on UPnP.

Also to assign a static IP you need to do it from within the routers settings as setting a static IP in the xbox settings seems to stop UPnP from working.

So in your xbox network settings make sure that the ip address setting is set to automatic.

To set a static IP address you need to go into the router settings and find the address reservation section.

In address reservation you input the xbox mac address then type your desired IP address. You can find the xbox mac address in the xbox one settings app. Just use the correct one for wireless or wired depending how you connect.

Make sure that there are no port forwards settings and make sure the DMZ is not set.

After setting a static IP you need to switch off the xbox then power down the router for a minute.

It gets complicated though if you also own an xbox 360 as the tp-link routers don't seem to work well if you have 2 Xboxes connected at the same time.

I found that I could rarely get an open NAT in both consoles at the same time. I regularly had to turn the router off and on again to fix the NAT.

malacath say:

Just go to the settings  app on xbox, select Network then  Advanced settings.

Make a note of the mac address.

There are two mac addresses listed they look something like XX-XX-XX-XX-XX-XX

Make a note of the correct mac address depending on whether you are using wired or wireless.

go to address reservation and click 'Add New'

Type the mac address in the top box.

Type the IP address in the second.

The IP address you type depends on the address range of your router.

.

So if you type 192.168.0.1 to log into the router settings you could use 192.168.0.100 as the reserve address.

If you type 192.168.1.1 to log into your router you could use 192.168.1.100 as your reserve address

After doing this select save.

After it's saved you should power down the router for about a minute then switch back on. Make sure your xbox is switched off during this time as well and make sure the xbox is set to automatic ip address

Hello Sheyenne,
 
Thank you for posting to Microsoft communities.
I can understand how frustrating it could be when things don't work as expected.
 
The probable cause for this issue could be because of using a parallel port to USB adapter for printing.
It's because the printer is operating on a different USB port from where it was defined, or you are using a USB hub.
 
You can fix this extra S1M Pages printing issue by turning off bidirectional support:
 
1. Press Windows key + X key together on the keyboard.
2. Select Control Panel.
3. Make View Small icons and select Devices and Printers.
4. Right click your printer, select properties
5. Go to the ports tab.
6. Uncheck Enable bidirectional support.
Note: Your printer has to be turned on before printing.

This steps stops extra page "S1M" printing.

To Clear and remove the print job from Spooler follow the steps:

1. Click on the Windows button in the lower left corner of the screen. Select "Control Panel," then "Hardware and Sound" then "Printers."

2. Double-click on the icon for the printer to which you sent the print job. Find the print job in the list that opens, right-click on it and select "Cancel."
3. Press the "Cancel" button on the printer's control panel if the job still does not come off the spooler. Switch the printer off and then on again to see if that will delete the print job.

4. Clear the entire spooler by clicking on the Windows icon, then type "Command." Right-click on "Command Prompt," then click "Run as Administrator."

5. Type "net stop spooler" and press "Enter." Type "del %systemroot%\System32\spool\printers\*/Q," then press "Enter."

6. Type "net start spooler," then press "Enter." Type "exit" and press "Enter" to exit the command prompt window.
