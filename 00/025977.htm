jonbiggs say:

Macs removing ACLs from Word documents

This is 100% replicable.

 

When a Windows platform creates a Word document or modifies a Word document and then a Mac later modified that same file, it removes all ACLs that are not inherited from the top-level share.

 

Example scenario: I create a share called SharedFolder and give 'Domain Admins' Full rights. I create a subfolder called Subfolder1 and give Accounting Read/Write to that. From a Windows machine I authenticate to the share as a member of Accounting and create a Word document. At this point the file contains ACLs for 'Domain Admins' and Accounting, as it should. But then I access the same file from a Mac (using SMB) authenticating as another member of Accounting, modify and save it. I check the ACLs and Accounting is gone. Only 'Domain Admins' have access now. No more access for Accounting to that file. Also, the user that authenticated from the Mac has now taken ownership (even though they can no longer access it).

 

We've tried this with Mac OS 10.68 and 10.82. We're using Office 2011 for Macs and Word 14.2.4. This happens with a share on Server 2008 R2 as well as our EMC file server. If there are multiple groups that have access to the subfolder (in this case, Subfolder1) then all of those groups are removed after the Mac modifies it.

 

In our multiple platform environment, this will certainly cause problems. Has anyone else experienced this and have you found a solution?

No, you setup your application to eliminate standard menus and substitute your custom ones (if any). The Runtime license just runs what you have setup in your app.

So you just give them access to the forms and have buttons to directly print the reports (no print preview). 

You have a full version of Access so when YOU open the app with the full license you have full access.

The runtime is actually a version of Access that does not allow access to development elements. The runtime version is install to users without a full Access license. In either case, you still have your own copy of the MDB for development.
