If you just want to input the number, and have the name replace it, you can use a VBA Change event. You'll need to tweak the ranges to suit:

Private Sub Worksheet_Change(ByVal Target As Range)
    '   Code goes in the Worksheet specific module
    Dim x As Long
    Dim rng As Range
        '   Set Target Range, i.e. Range("A1, B2, C3"), or Range("A1:B3")
        Set rng = Target.Parent.Range("A5:G5, A9:G9,A11:G11,A13:G13,A15:G15,A17:G17,A19:G19")
             '   Only look at single cell changes
            If Target.Count > 1 Then Exit Sub
            '   Only look at that range
            If Intersect(Target, rng) Is Nothing Then Exit Sub
            '   Action if Condition(s) are met (do your thing here...)
            x = Application.Match(Target.Value, Range("J:J"), 0)
            If IsNumeric(x) Then
                Application.EnableEvents = False
                    Target.Value = Cells(x, 9).Value
                Application.EnableEvents = True
            End If
End Sub

Sorry, I can't tell how you got that result, I can only tell you how color themes work:

  • Any color scheme that shows under the Custom tab is showing you a color theme file that was saved to the computer on which you see it.
  • An open presentation that contains a custom color theme does not display that theme under the Custom heading. It will not display it under Custom until you click on Customize Colors..., give it a name and save it.

john korchok say:

Client A's color theme is not in the template. Here is how color themes work:

  1. You choose Customize Colors... in PowerPoint, set the colors, give the theme a name and press Save.
  2. The theme colors are saved in the current document. At the same time, they are also saved as a color theme file to your hard disk.
  3. The next time you click on the Colors dropdown, the Custom area at the top shows all color theme files that have been saved to your disk. These themes are not part of the document until you select them and apply. It's simply a convenient way to access previous themes you've created.

If you don't ever intend to do work for client A again, you can delete their color theme file from your computer by following these steps:

  1. Close PowerPoint.
  2. In the Finder, hold down the Alt key while clicking on the Go menu and choosing Library. Your user Library folder opens.
  3. Navigate to ~/Library/Group Containers/UBF8T346G9.Office/User Content/Themes/Theme Colors.
  4. Delete Client A's color theme and close the window.

zeconte say:
No other security is installed on the machine.
After validating Windows , the software states that it cannot complete the installation.
System is Windows 7 RC.
I have found a possible solution to this issue.

Go into regedit (Start > Run > regedit or regedt32)

In the registry, go to HKEY_CLASSES_ROOT\Installer\Products

There will be a list of folders with hexidecimal naming schemes, scroll through them until you find the folder that has Microsoft Security Client listed as the ProductName.  Make a note of the hexidecimal folder name it is found in, then goto HKEY_CLASSES_ROOT\Installer\UpgradeCodes.

Scroll through those folders until you find the one with registry key that matches the folder name the Microsoft Security Client key was found in.

Delete both folders (the one in Products that has the MSC key in it and the one in UpgradeCodes that has the key that matches the folder name you found in Products).  You may want to look through all the folders in both Products and UpgradeCodes just to make sure there isn't more than one for Microsoft Security Client (or potentially Microsoft Security Essentials, I found a folder that had a key that simply said "essentials" but this one DOES NOT need to be deleted), or more than one folder in UpgradeCodes that has a matching key for the MSC Product key folder.

Just to be safe, export each folder before you delete them.  At first, I only deleted the key out of HKEY_CLASSES_ROOT\Installer\Products, and after I did that, I got error code 0X80070645, so I had to restore it to find the right key to delete from UpgradeCodes as well, but once I deleted both, MSE successfully installed.

This may also work if it's not a corrupted version of MSE that's messing up the install.  Just look for any other anti-virus you might have had on the computer in the ProductName registry key and do the same thing.

Note: After installing, I could not turn on Real Time Protection, receiving error 0x800705b4.  At this point, all I did was uninstall MSE, restart, and reinstall to fix this issue.

You may or may not be able to recover this document since the changes are saved in your Temporary Internet files folder. Note that you can't just open your Temporary Internet Files folder directly and find it due to the way Windows handles your Temporary Internet files but if it's still available, here's how you can get it back:

  1. Open another Word document sent to you as an attachment from an email. If you don't have one, send an email with an attachment to yourself. (While you should be able to open the same email attachement since another copy of the file is created in your Temporary Internet Files folder, I don't like taking a chance like that.)
  2. Click the Office Button, point at Prepare, and then click Properties. The Properties pane will display below the Ribbon.
  3. On the far right of the Properties pane you'll see a file path and the document name. Select only the file path and press Ctrl + C to copy it.
  4. Press the WinKey + E to open the Windows Explorer. (Or use an alternate method to open it)
  5. Paste the copied file path in the Address bar and press Enter.

These steps should put you either in the same location as your revised file or near it. You may need to dig around in other folders to locate it.

Good luck and I'll keep my fingers crossed for you. :-)
