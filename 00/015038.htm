Macro involving F9 recalculation to obtain random number series - Microsoft Community

howard1414 say:

Macro involving F9 recalculation to obtain random number series

I have four variables that i wish to have as random numbers between limits fed into an equation to produce an output that is then read to a matrix and the process repeated, with all output stored.  How do i do this, bearing in minfd that i need the recalculate procedure to get the next set of random numberd?

joeu2004 say: Howard wrote:
I have four variables that i wish to have as random numbers between limits fed into an equation to produce an output that is then read to a matrix and the process repeated, with all output stored.  How do i do this, bearing in minfd that i need the recalculate procedure to get the next set of random numberd?

Your question is unclear to me, although it might be clear to someone else.

First, what do you mean by "random numbers".  Are you looking for random values between 0 and 1 (excluding 1) -- the result from RAND()?  Or are you looking for random values between a specific range of integers -- the result from RANDBETWEEN()?  Or something else entirely (explain)?

Second, you say you want the random results "fed into an equation".  I presume you mean an Excel formula.  But your subject line mentions "macro", which implies VBA, not Excel.  What exactly is your design?  Are you writing a macro?  Does the macro produce the values of the maxtrix?  Or does the macro product formulas?

Finally, what is the problem with the "recalculate procedure", specifically?  I infer that you want the random results to be regenerated when F9 is pressed.  But that is the normal behavior when RAND and RANDBETWEEN are used in Excel formulas.  Are you trying to avoid regeneration of random results when you merely edit other cells in any worksheet in the Excel file -- that is, limit regeneration to only when F9 is pressed?

Those questions are intended to be suggestive, not exhaustive.

But let me take a wild-a.s.s guess about your needs....

Instead of using RAND() directly in Excel formulas, I use a UDF (VBA function) of the following form:

Function myRAND(Optional r As Range) As Double
myRAND = Evaluate("RAND()")
End Function

That can be used as a direct replacement -- use find/replace to change RAND() to myRAND().

Then the random results are not changed until the worksheet is recalculated -- for example, when ctrl+alt+F9 is pressed.

Alternatively, you can use myRAND(X1), where X1 is any cell or range reference.  Then the random results are not changed until X1 is recalculated, as well as when the worksheet is recalculated by pressing ctrl+alt+F9.

Is that going in the direction of a solution you are looking for?  Or have I missed your problem altogether?

 

PS....  I use Evaluate("RAND()") because Excel RAND is better than VBA Rnd in some respects, especially in Excel 2010.  However, if you are generating many thousands of random values, you might prefer to use:

Function myRAND(Optional r As Range) As Double
Static first As Long
If first = 0 Then Randomize: first = 1
myRAND = Rnd
End Function

It performs 100s of times faster on my computer.  And it is sufficient for most purposes.

I don't know anything about your defined pages thing, but if you execute the merge to a new document and then use the following macro to print that document, it will send each letter to the printer as a separate print job which may work for you:

Dim i As Long
With ActiveDocument
   For i = 1 To .Sections.Count

End With
