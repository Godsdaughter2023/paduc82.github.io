lars bremer say: We were having the same issue, some time ago. The problem (in our case) was a combination of Windows XP and Office 2010. It has to do with mail encryption, which doesn't seem to be compatible with XP (or maybe older versions of MS Server as well). Check your registry for these keys:

[HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Common\Policy]
"EnableClassifications"=dword:00000001
"TrustClassifications"=dword:00000001

Set them to 0 or delete them. Then try again. Of course, you won't be able to use Outlooks own encryption any more, but maybe you don't need it.

Hope this helps,

glenprouty say:

Hi Berk,

As I interpret your post, it sounds like your Ransomware protection is what is causing

your problem.

If so, there are two ways that you can achieve a resolution. Kill your Ransomware

protection, (dumb) or allow the Apps that are being blocked, to pass through "Controlled

Folder Access". (smart).

Your Ransomware protection works by disallowing Apps, that are not on the whitelist,

from modifying the "protected" folders in your PC. W10 v1809 makes whitelisting the

Apps that you trust, fairly simple and quick. Once an App is on the whitelist, it will no longer

be blocked. Consequently, as time goes on, those that are blocked become fewer

and fewer.

In v1809 you can allow an App to pass by clicking on the blocking notification, or on a

notification that is described in the taskbar. When you do that, you are taken straightaway

to the "add an App" routine. By clicking the + sign to add an App, you are presented

with a list of all recently blocked Apps.

Select the list and you should see most of your Apps that have been blocked already.

A single click on each App, that you want to pass, will add it to your whitelist, and it will

no longer be a problem for you. Use caution, there may be some in the list that you do

not want to pass.  That is what this feature is for!

You can prevent Apps from being blocked the dumb way, if you consider yourself to be

Ransomware proof. You can turn off Controlled Folder Access from the Ransomware

Protection screen in Windows Defender. No more blocks, but also no longer protected.

Regards,  Glen

jcomputer21 say: In reply to deleted message

That setting determines if information about suspicious files is sent to Microsoft, files themselves are only sent if they are Quarantined, not a personal file (e.g. document, email, calendar), and you allow it at the time.

You can find more information, or change the setting by going to Windows Defender itself (Security Essentials in Windows 7) from the Start menu and clicking on settings, MAPS:

The basic setting tells MS where suspicious files originated and how Defender dealt with it. Advanced will send logs of what the file did (if it was a program, personal files don't create these) which is needed to create the virus definition file.
