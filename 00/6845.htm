DoCmd.ShowToolbar "Ribbon", acToolbarNo
 
The resulting form will look like:
 
 
In the above you see NONE of the access interface. The background above is my desktop.

The settings for the above are
 
hide nav pane, no record selector, no status bar, tabbed windows (uncheck show tabs). and then simply specify the form on startup, and in the forms on load command execute the one line of code above.
 
Note that you need to use a regular form to do this. If you attempt to use a popup form, then none of the above works.
 
I should also note that you can't have all these settings change while the application is running, but you can most certainly create a access application that hides all of the UI and does so with great ease.

for forms on load, I perhaps should have been more clear and was meaning you simply specify a startup form.
 
file->options->current database
 
Then choose display form (it is a combo box). This is typically referred to an startup form. This is simply the first form that will display when you launch the application.
 
And then you want to make sure you using Tabbed documents option as opposed to overlapping windows.
 
The other options you want to set in current database are actually up to you. However, you likely want to turn off display status bar, uncheck navigation pane.
 
So, the above options are all found in current database.

don't allow that startup form to load + run. So, you cant develop, or really
modify things when you run your application with the startup settings...so you
must shift-by-pass them when you want to work.

You are getting a message because Word's VBA is finding an error in its macro code. There are two possibilities:
