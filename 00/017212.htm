Hi Trieu,

Nice hearing from you. I’d like to add more details regarding your questions:

1. When and why the two accounts are added in the user list?
When you sharing anything on your SharePoint Online Team Site to an external user for the first time, the external user will receive an email including a link to accept the invitation. When the user opens the link and login in with a Microsoft account or Office 365 work or school account, an account for the user will be created in Office 365. This account is intended to help Office 365 control the permissions and other properties of the external user to make it function correctly. When you don’t want the external to have access to your Office 365 tenant, you can delete the account from the list.
2. What happen if I delete those 2 user accounts in our users list?
The external users will no longer have access to your Office 365 tenant until they receive the invitation email and accept it again.
3. What is Office 365 Azure AD?
Office 365 Azure AD (Active Directory) is the Active Directory services designed for and applied to Azure, the cloud services which Office 365 is based on. Azure Active Directory function similarly to the on-premises Active Directory which is commonly running Windows Servers. It provides an integrated service to manage and connect the user accounts and other resource such as groups, mailboxes. The external users accounts will also be added to and controlled by the Azure AD. Here is an article introducing the Azure AD in detail: What is Azure Active Directory?

doug robbins - mvp office apps & services (word) say:

The rId# is a referencing system used in the document.xml file in the docx document type.  If you change the file extension to .zip and extract the files from the archive, the missing image may be in the word\Media folder in the archive

The images in that folder are sequentially numbered, but I don't think that the numbers assigned the images are an exact match to the rID number as I believe that rID references also relate to embedded oleObjects that are located in the word\embeddings folder in the archive and are also sequentially numbered.
