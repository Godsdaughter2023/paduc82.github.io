john - project say:

CombustibleKen,

I think the operative word here is "filter". You are trying to use the grouping function when simple filtering should do the trick.

Filtering could be accomplished in multiple ways. If you identify your releases by number, then using an extra number field (e.g. Number1) will work fine. If you identify them by name, such as release 1, release 2, etc. then using an extra text field (e.g. Text1) is a better choice. Add the extra field as a column in the view. Then enter the release number/text into the summary line for that software release and use fill down to populate that field for all subtasks under that release summary.

I've mocked up your example for some screen shots. Note that good practice dictates each performance task, (i.e. tasks that have resources assigned), be expressed in action verb form. Non-performance tasks (e.g. summary lines) are in noun form. The first screen shot is the Gantt and the second screen shot is the interactive filter which when applied via View/data group/filter selection box, will pop up a user input box asking for the release number.

Hope this helps.

John

john sr wilson say:

This might fix SOME things. Make sure you work on a copy

Sub BW()
    Dim oshp As Shape
    Dim osld As Slide
    For Each osld In ActivePresentation.Slides
        For Each oshp In osld.Shapes
            If oshp.HasTextFrame Then
                If oshp.TextFrame.HasText Then
                    If oshp.Fill.ForeColor.RGB = vbBlack Then oshp.BlackWhiteMode = msoBlackWhiteGrayScale
                End If
            End If
        Next
    Next
End Sub
