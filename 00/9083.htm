ken sheridan say: The best way to do this is to use the control's NotInList event procedure to (a) open a form bound to the referenced table in dialogue mode, which pauses the calling code's execution until the form is closed, and (b) pass the value of the NewData argument to the form via the OpenArgs mechanism.  When the code execution recommences check that a new row has been inserted into the referenced table and, if so, set the Response argument's return value to the acDataErrAdded constant to requery the control.

You'll find an example as NotInList.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the NotInList event procedure of the CityID combo box in the frmContacts form is:

Private Sub City_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control
    Dim strMessage As String

    Set ctrl = Me.ActiveControl
    strMessage = "Add " & NewData & " to list?"
    
    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        DoCmd.OpenForm "frmCities", _
            DataMode:=acFormAdd, _
            Windowmode:=acDialog, _
            OpenArgs:=NewData
        ' ensure frmCities closed
        DoCmd.Close acForm, "frmCities"
        ' ensure city has been added
        If Not IsNull(DLookup("CityID", "Cities", "City = """ & _
            NewData & """")) Then
        Response = acDataErrAdded
        Else
            strMessage = NewData & " was not added to Cities table."
            MsgBox strMessage, vbInformation, "Warning"
            Response = acDataErrContinue
            ctrl.Undo
        End If
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If
    
End Sub

and the Open event procedure of the frmCities form is:

Private Sub Form_Open(Cancel As Integer)

    If Not IsNull(Me.OpenArgs) Then
        Me.City.DefaultValue = """" & Me.OpenArgs & """"
    End If
    
End Sub

However, it's only necessary to open a form if there are values other than that entered into the combo box to be inserted, e.g. in my frmCities form it is necessary to select or insert a value in the RegionID combo box.  Where only the single value needs to be inserted into a new row in the referenced table it can be done transparently in the NotInList event procedure.  The RegionID control in my frmCities form illustrates this:

Private Sub RegionID_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control
    Dim strSQL As String, strMessage As String
    
    Set ctrl = Me.ActiveControl
    strMessage = "Add " & NewData & " to list?"
    
    strSQL = "INSERT INTO Regions(Region) VALUES(""" & _
            NewData & """)"
   
    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then
        CurrentDb.Execute strSQL, dbFailOnError
        Response = acDataErrAdded
    Else
        Response = acDataErrContinue
        ctrl.Undo
    End If
    
End Sub

shane devenshire say:

If you are asking how to lock and unlock cells and how to merge and unmerge cells:

1. Lock:  Select the cells you don't want locked and choose Format, Cells, Protection and uncheck Locked.  Then choose Tools, Protection, Protect Sheet.

2. Unlock: Choose Tools, Protection, Unprotect Sheet

3. Merge:  Select the cells you want to merge and click the Merge and Center toolbar button on the Formatting toolbar

4. Unmerge:  Click the same button while selecting the merged cells

?5. undelet:  You can use Undo to recover the contents of a cell you just deleted.
