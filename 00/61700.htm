timothystellar say:

The source of the error reported by OP above it is in TipTiles.xml, default path:

C:\Program Files\Microsoft Mouse and Keyboard Center\TipTiles.xml

(replace path with yours, search for folder name to find it, then the xml file inside)

Once inside the xml file (open it in notepad, no need for XML editors, though some of those can highlight in red some errors, but I only used Notepad to find it, so no need for more), go to this line:

  <!--All Touch-->-->

and make it look like this (delete only --> in the end, only 3 characters, don't touch empty places at all), so it will look like this:

  <!--All Touch-->

Backup you original file (copy file, paste in same folder, rename copy to BackupOfBugged.TipTiles.xml.txt), then save the corrected xml with notepad.

Test:

Empty Application logs (see OP above), then start again the "bugged" program, by going into Control Panel, Keyboard, start on one of those tabs the Microsoft Mouse and Keyboard Center, play with things a bit (no need to save or apply anything, then close it), then go and check in Application logs again (select another log to refresh it) and you can see that the error is gone.

Also the program started faster too, before seemed to take 1-2 second longer (on an SSD over the top system). Also the system boots a bit faster (it's scheduled to initialize macros and such).

This was (it is) a minor bug, but affected system performance a bit and a major annoyance too see red in logs.

Credits go to OP and his friend, was a great help to pointing me where to try to find the fix. It's sad that Microsoft doesn't care much... Just an option, because the center may help novice users, but the rest of us hate to loose precious seconds on that slow graphical hog, also hate to loose performance, even less than a second. I'm a gamer, and want more than 100% of the system to work for the game, not wasted on the Operating System doing whatever in background, ever.

a.       Click Start, type Device Manager in the Start Search box, and then press ENTER.

b.      Click Continue. The Device Manager Dialog box is displayed.

c.       If you are prompted for an administrator password or for a confirmation, type the password, or click Continue.

d.      In Device Manager unknown devices would be listed as such and marked out with a yellow question mark.

e.      Right on yellow warning sign click on properties check for error codes.

Method 4:

You may uninstall and reinstall the USB drivers and check if you are able to connect USB condenser microphone. You may disconnect all USB devices and then perform the following steps:

a.       Click Start, type Device Manager in the Start Search box, and then press ENTER.

b.      Click Continue. The Device Manager Dialog box is displayed.

c.       If you are prompted for an administrator password or for a confirmation, type the password, or click Continue.

d.      In Device Manager, double click to expand "Universal Serial Bus controllers", right click on the Host Controller, click "Uninstall" and click "OK".

Restart your computer and Windows 7 will reinstall all USB controllers automatically.
