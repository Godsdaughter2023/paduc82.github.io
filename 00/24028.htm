   said blah blah blah.

That is, the text in the second line should be directly beneath the text in the first line. In the example, the first 's' in said is supposed to be directly beneath the 'S' in Smith.

I've tried using hanging indents but they have to be different for each footnote and have to get updated if the number of footnotes changes (i.e. when the above footnote becomes footnote 11 rather than 1 the text would get pushed across and throw out the indent.

Is there any simple way to get consistent indents either with or without the use of a macro?

Josh
macropod say:

Insert a tab between the footnote # and text, together with a hanging indent and a tab-stop at the desired position for the first line's text. Update your footnotes Style with that format. You'll still need to insert the tab characters manually (or use Find/Replace to do it for existing footnotes), but that should give you the consistent layout you need.

 

FWIW, you shouldn't need different indents for footnotes 1-9, compared to 10-99 or 101+ - they should all be the same as for the largest number you'll require.

kosh vorlon say:

Hi Gordon,

 

I'm not sure why it doesn't appear (perhaps the tech support department at HJT can explain), but the real bottom line here is that HJT isn't the best program to do that.  Here's how to get it done properly and easily (and includes several options which offer different features and what I do and why):

 

There are at least three other and more appropriate ways to manage start-up programs (possibly more, but these are enough). I use all three depending on the circumstances.

 

1. Go to Start / All Programs / Accessories / Run and type msconfig.exe and enter. Go to the Startup Tab and you can check or uncheck the programs you want to run at startup. The two drawbacks are that you get little information about the programs and you can't add programs that aren't already there to the listing. Then again, if you need to run a clean boot, this is by far the easiest way to go about it.

 

2. Download the free WinPatrol: http://www.winpatrol.com/download.html and you will not only have control over those programs, but you can also add new ones AND, what I like best, you can allow it to run in the background and it will alert you if anything tries to add itself to the startup menu without your knowledge (as sometimes happens with some downloads and installations). You don't have to use that feature if you don't want, but I find it convenient. It also provides more information than msconfig. I find it better than Defender and would use for startup-program control it even if it were possible to run Defender.

 

3. Download the free Autoruns: http://technet.microsoft.com/en-us/sysinternals/bb963902.aspx. This is the gold standard for controlling startup programs and everything else at startup with loads of details and options and different views. The danger here is that it is so good and powerful, that if you're not careful, you can remove something you really don't want to remove and hose your system (but if you pay attention and show use caution and don't change anything unless you know EXACTLY what you are doing and the ramifications of doing it, you'll be fine).

 

I rarely use msconfig. I mostly use WinPatrol. When I'm really into cleaning things up and need to know exactly what is happening a boot and startup, then I go to Autoruns to get all the details I need.

 

This will cover managing start-up and boot programs and processes and everything related much better HJT. And if you choose to run WinPatrol only on-demand and not with real-time scanning, then none of them will take up any resources unless you are using them.

 

I hope this helps.

 

Good luck!
