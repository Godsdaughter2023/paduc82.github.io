rich michaels - mvp (greatcirclelearning.com) say:

Hi EdlastWG,

The font size of the references are not linked. Superscripts in the body of the document are always proportionaly smaller than the reference text they are marking and they have no direct link to the font size of the footnote/endnote they are notating.

Body text set at a 12 pt font is actually occupying a space equivalent to the text only height of a 14 pt font. The remainder of the space above and below the actual text 12 pt characters is reserved for subscript and superscript height above or below the text they reference.

A simple test to prove this is to set your footnote style at 20 pt and you will see that nothing changes to the superscript reference height in the body of your document.

Hope this helps

stefan blom say:

The Footnote Reference style is a character style which is used, by Word, for both the reference in the main body of the document and for the reference in the footnote area.

To modify the size of footnote text, what you should do is modify the Footnote Text paragraph style.

tom_bpc say:

Jonathan,

Apparently John and I are seeing your question slightly differently....

I don't think that there is a built-in single-command technique to collapse all summary tasks at once.  Here is a 4-command technique that works equally well in Project 2010 and 2016.

  1. Open up the entire outline tree using View | Data | Outline -> "++ All Subtasks"
  2. Clear all filters
  3. Select all tasks by clicking the blank header of the first (locked) column, normally where the task ID is listed.
  4. Collapse the entire outline tree using View | Data | Outline -> "- Hide Subtasks".

One or both of the first two steps could easily be left out, but the summaries that are left hidden will not be touched.

Alternately, you could do it with a single click attached to this little macro:

Sub OutlineCloseAll()

Dim t As Task

    For Each t In ActiveProject.Tasks

        If Not t Is Nothing Then

            If t.Summary Then t.OutlineHideSubTasks

        End If

    Next t

End Sub

Both approaches apply the same "OutlineHideSubTasks" method to summary tasks, but the macro doesn't have to go through the steps of making them visible first.

Good luck, tom
