Hi WaelAyoub

Within any Office application, select File > Options > Language.
The dialog box where you can add, select, or remove the language Office uses for editing and proofing tools.

If your language already appears among the editing languages click Set as Default. Otherwise select the language from the Add additional editing languages list then click the Add button. The new language will appear in the list of editing languages.

If the Keyboard Layout shows as Enabled you're all set for editing. If it says Not enabled in the Keyboard Layout column follow these steps to enable it:

Click the Not enabled link.

In the Add Languages dialog box, click Add a language, select your language in the list, and then click Add.

Close the Add Languages dialog box. Your language should display as Enabled under Keyboard Layout in the Choose Editing Languages section.

Under Proofing, if your language shows as Installed you're ready to go. Otherwise click Not installed and Office will take you to a page where you can download the necessary Language Accessory Pack for that language. Follow the instructions on that page to download and install the pack, then come back to the Office Language Preferences dialog box to confirm that it's now installed.

Note: If you're using Office in the Microsoft Store, clicking Not installed will take you to Windows Language Settings where you can select your desired language and it will be downloaded, if necessary, automatically.
Display and Help Languages

Within any Office application, select File > Options > Language.
The dialog that allows you to select the language that Office will use for it's buttons, menus and help.

If you've already set your Microsoft Windows display language to be the language you want Office to display in you can leave this set to Match Microsoft Windows. Otherwise select your language and click Set as Default to move that language to the top of the list. The language that is listed first is the one Office will use for its menus and buttons.

Note: If you're using Office in the Microsoft Store and you want to change your default display language go to the Region and Language settings in Windows 10. Click the language you want to set as the default, and choose Set as default.

Repeat to configure your help language if you want to.
If the language you want isn't available...

Download the proper Language Accessory Pack for the language you want to use from this page: Language Accessory Packs for Office (https://support.office.com/article/82ee1236-0f9...). The Language Accessory Pack contains the display and proofing tools for most languages, but you can still choose to have your display in one language and editing in another.

Correcting Floating Point Arithmetic - Problems with Rounding - Microsoft Community

jolsef say:

Correcting Floating Point Arithmetic - Problems with Rounding

Hello community! I wanted to see if anybody had come up with this issue and had any other resolution for it? Scroll straight to the bottom to see my solution for this problem (second to last formula).

The issue lies in Excel's ability to perform basic arithmetic functions such as subtraction and addition and to a lesser extent, multiplication. When subtracting two numbers and rounding the difference to particular decimal places may result in incorrect values. See the example below:

      A                B            C   

1    124.15       124.7      =ROUND(B1-A1,1)=0.5

2    124.015     124.07    =ROUND(B2-A2,2)=0.05

3    124.5         124.7      =ROUND(B3-A3,1)=0.2

In this example, rounding B1-A1 to one decimal point should result in a value of 0.6 and rounding B2-A2 to one decimal point should result in a value of 0.06. When Excel performs the math, C1 will have a true value of 0.549999999997 due to the floating point errors. 

There are a couple of potential fixes for this problem. However, not all of them appear to be bulletproof because they start to fall apart at certain numbers. One option is to round the difference to 12 decimal places. 

      A                B            C   

1    124.13       136.7      =ROUND(A1*B1,12)=16968.5709999999

We can see here that with an integer large enough, 12 decimal places is too many places and does not address the error. Changing the decimal places temporarily addresses the issue until another number with either too many or not enough decimal places causes the error to return. The second option is to convert both numbers into an integer form. 

      A                B            C   

1    124.15       124.7      =(B1*10000-A1*10000)/100000 = 0.55

2    4.19           4.74        =B2-A2 = 0.55

3    4.19           4.74        =(B3*10000-A3*10000)/100000 = 0.549999999...

4    124.0915   124.0074=(B4*100-A4*1000)/1000 = 0.0844999999...

However in the example above, there are two concerns. The first one is not having enough 10's to convert the inputs into a whole number. The second is in certain cases, the conversion to a whole number actually introduces floating point errors into the arithmetic. Subtracting 4.74 from 4.19 results in 0.55, but when multiplying each of the inputs by 10e3, the error arises when otherwise it would not have occurred. 

The third option is to enable Excel's "Set Precision as Displayed." This does indeed correct the issue with floating point arithmetic, but in the example above, column C cannot be formatted in a single way. C1:C3 can be shown with two decimal places, but C4 must be shown with 4 decimal places. By not showing enough decimal places, one runs the risk of using incorrect values when performing calculations further down. 

So the goal here is to come up with a formula that one can enter into the column C and work for any and all inputs that are entered into columns A and B. By performing the calculations by hand, we find that the expected result in the first example for B1-A1 is 0.55 and for B2-A2 is 0.055. A trend is observed here that the expected result will have the same number of decimal places as the input with the greatest number of decimal places. We can see that the floating point error shows up at the last decimal of expected result. If your one of your inputs have two decimal places and the other input has one decimal places, the error will show up at the 2nd decimal place. If one input has three decimal places and the other decimal has two decimal places, the error will show up at the 3rd decimal place. To generalize this:

     1. Your expected result will have number of decimal places equal to the number of decimal places of the input value with the greatest number of decimal places when adding or subtracting. 

     2. We can even generalize multiplication by counting the number of decimal places in a myriad of calculations. The expected result will have a number of decimal places equal to the sum of the number of decimal places of all inputs. 

     3. If your expected result has n number of decimal places, then the Excel calculated number should be rounded to the nth place.

Thus the problem here arises how to come up with a formula that can dynamically count the number of decimal places of the inputs. My solution for this involves the following utilizing Example 1:

     MAX(0,LEN(ABS(A1))-LEN(ABS(INT(A1)))-1,LEN(ABS(B2))-LEN(ABS(INT(B2)))-1).

For an equation with multiple inputs, the underlined portion should be copied as necessary within the MAX() function to account for each input's decimal places. To account for the decimal point, the "-1" is introduced. MAX(0,[...] is necessary because inputs that are whole numbers do not have the decimal point and result in a value of -1. By introducing MAX(0,[...]), whole numbers are automatically chosen as 0 decimal places. The final equation in Excel would look something like:

     =ROUND(B1-A1,MAX(0,LEN(ABS(A1))-LEN(ABS(INT(A1)))-1,LEN(ABS(B2))-LEN(ABS(INT(B2)))-1)).

This function so far works with values with varying decimal places as well as whole numbers and eschews IF() statements. There are a couple different ways out there for counting the number of decimal places in a number, but this is the one I've come up with. In example 1, if we wanted to display the final value to one decimal place, we could either format the cell to display with one decimal place or we would ROUND() again to one decimal place. For example:

     =ROUND(ROUND(B1-A1,MAX(0,LEN(ABS(A1))-LEN(ABS(INT(A1)))-1,LEN(ABS(B2))-LEN(ABS(INT(B2)))-1)),1).

The reason I chose this method is so people without VBA experience can utilize this formula to correct the error. This also requires minimal adjustment for different numbers and can be copied down a cell. A similar fix is available for AVERAGE(), SUM(), multiplication over a range, and summation over a range. If you'd like to know how that formula looks, please let me know below. 

Try it out for yourself and let me know in the comments below if this works for you or if you notice any issues with the formula! If you have your own solution for correcting floating point arithmetic, I'd love to see how you correct this issue. If you have an easier way to determine number of decimal places, please share below in the comments.

JoL.
