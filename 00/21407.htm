You then use an append query to shuffle the data from the table to the production tables, and at that point delete the tables. This would also allow some cleanup of data, and it would be quite risky to allow data to go striating into your all-important production tables. This way, if the import messes up or freezes half way, it could be deleted and started over again without having to figure out how far the import went and thus what was added to the main working table would be unknown.

You can manually import xml and save the import.  Then once done, in code you can go:

docmd.RunSavedImportExport "name of import export"

So you can still use the built in xml import and let it crate the tables. Just use execute an append query to move the data.
