vijin v say: Hi,
Thanks for asking this question in Microsoft Community!
I understand that you need to extend the record time on Windows XP.
I would suggest you to refer the following steps and check if it helps.
To increase the maximum recording time in the Sound Recorder utility:
a. Make sure that you have installed a functional microphone, and then start the Sound Recorder utility.
b. Click Record, and then let Sound Recorder run for 60 seconds.
c. After Sound Recorder stops recording, click Save As on the File menu, and name the file Blank.wav.
d. To increase the maximum recording time in Sound Recorder, click Insert File on the Edit menu, and then insert the Blank.wav file that you saved in step 3. When you do this, the maximum recording time is increased by 60 seconds. You can repeat this step once for each additional minute of recording time that you want to add. For example, for a 5-minute recording time, insert the Blank.wav file five times.

To make Sound Recorder use the extended length of recording time automatically:
a. Right-click Start, and then click Open.
b. Double-click the Programs folder.
c. Double-click the Accessories folder.
d. Double-click the Multimedia folder or the Entertainment folder.
e. Right-Click Sound Recorder, and then click Properties.
f. Click the Shortcut tab.
g. In the Target box, press the END key, press the SPACEBAR, and then type the location of the Blank.wav file.

For example, if the target is "C:\Windows\sndrec32.exe", and the location of the Blank.wav file is C:\Blank.wav, the new line in the Target box is "C:\Windows\sndrec32.exe C:\Blank.wav"
The next time you start Sound Recorder, the Blank.wav file opens automatically. After you record your new .wav file, you can click Delete After Current Position on the Edit menu to remove the extra blank space in the file.

Note: When you finish recording and editing your new .wav file, save the file with a new file name.

That advice is peculiar and I suspect further assistance you offered will be needed since for that message, I don't think one is going to get past step 1 in the article you posted (could be wrong though).

Notice the difference in the status message that the OP posted 0xc0000135 is not the same as the status message in the the KB article you posted which is this: 0xc0000034

I have in my notes several reasons you might see a message reported by the OP:

STOP: c000021a {Fatal System Error}
The Windows Logon Process terminated unexpectedly with a status of 0xc0000135 (0x00000000 0x00000000).
The system has been shut down.

(Note that those are zeros and now ohs)

None of the examples I have for the message reported by the OP involve winlogon.exe or csrss.exe, but I do know the messages you will get it either are missing or corrupt:

If the winlogon.exe file is missing, you will get this message:

STOP: c000021a {Fatal System Error}
The Windows Logon Process terminated unexpectedly with a status of 0xc0000034 (0x0000000 0x0000000).
The system has been shut down.

If the winlogon.exe file is corrupt, you will get this message

STOP: c000021a {Fatal System Error}
The Windows Logon Process terminated unexpectedly with a status of 0xc000012f (0x0000000 0x0000000).
The system has been shut down.

If the csrss.exe file is missing, you will get this message:

STOP: c000021a {Fatal System Error}
The Session Manager Initialization System Process terminated unexpectedly with a status of 0xc0000034 (0x0000000 0x0000000).
The system has been shut down.

If the csrss.exe file is corrupt, you will get this message:

STOP: c000021a {Fatal System Error}
The Session Manager Initialization System Process terminated unexpectedly with a status of 0xc0000020 (0x0000000 0x0000000).
The system has been shut down.

Bluetooth stopped working after connecting to WIFI (on a 2.4 GHz - Microsoft Community

jess can say:

Bluetooth stopped working after connecting to WIFI (on a 2.4 GHz WIFI Band)

After connecting to a new WIFI network, suddenly all my Bluetooth devices stopped working, and I can no longer pair any new Bluetooth devices.  After connecting to a new WIFI network, suddenly all my Bluetooth devices stopped working, and I can no longer pair any new Bluetooth devices.  How can I resolve this issue?
jose [msft] say:

Microsoft has received reports that some Bluetooth devices cease to work while engaged in a 2.4GHz WIFI connection. Similarly, there are reports that Bluetooth devices function normally until they switch their computer from a 5.0 GHz WIFI connection to a 2.4GHz WIFI connection.

Many WIFI access points or hotspots are 2.4 GHz connections.  So, for example, you may experience this issue if you connected to a WIFI hotspot, or if your wireless router automatically switches to a 2.4 GHz connection to maintain a proper connection.

In this case, switching back to the 5.0 GHz network does not fix the issue.  Furthermore, a new Bluetooth device cannot be paired once in this state.

These scenarios are related to a known issue for Intel Driver version 19.30.  Intel has fixed this issue by releasing updated driver versions 19.50+.  To resolve this issue, install the updated Bluetooth driver.  If you do not have an Intel Wireless Bluetooth driver, and your Bluetooth is not working, try the troubleshooting steps at Fix Bluetooth problems in Windows 10: FAQ.

How to install an updated driver for your machine:

  1. Right-click the Start menu icon and select Device Manager.
  2. Locate the Bluetooth icon and click the drop-down to identify your Bluetooth radio adapter. If your Device Manager does not look like the picture below, click View > Devices by Type.


  3. Right-click your Bluetooth adapter and select Update Driver.


  4. Select Search automatically for updated driver software.

If you’re not able to install the updated driver with these steps, you can also search for a Bluetooth driver update on your computer manufacturer’s support site or install the driver directly from Intel’s download center.

Typically, WIFI connections are 2.4GHz connections. If one of your access point enumerates more than one SSID and one of the SSIDs includes “5.0” in its name, the access point is indicating that both options are available.  To learn more about installing and updating hardware drivers on Windows 10 PC’s click here.

jonh_sonic say:

I had the same problem and I was able to resolve the issue with Microsoft by doing the following steps:

  1. Go to the registry (RegEdit in the search)
  2. Add ALL APPLICATION PACKAGES Read privileges to the groups below.  To do this Right-Click on the folders named below and then click on Permissions.  Click on Add, and then type the word 'All' (make sure the location is set to your local computer for those on a domain) and click Check Names and it should automatically pull up ALL APPLICATION PACKAGES group.  Click OK and then make sure to give that group Read Permissions.
    1. HKEY_CURRENT_USER
    2. HKEY_LOCAL_MACHINE > HARDWARE
    3. HKEY_LOCAL_MACHINE > DRIVERS (If you have this folder)
    4. HKEY_LOCAL_MACHINE > SAM
    5. HKEY_LOCAL_MACHINE > SECURITY
    6. HKEY_LOCAL_MACHINE > SOFTWARE
    7. HKEY_LOCAL_MACHINE > SYSTEM
    8. HKEY_USERS
  3. Then on your windows installation drive, e.g., C:\, right click on "Program Files" folder and go to the Security tab, click on Edit, and again add (if its not there already) ALL APPLICATION PACKAGES group but this time you are giving it the following permissions (defaulted but check anyways), "Read & execute", "List folder contents", "Read". 
  4. Click OK and it will chug through applying the permissions.  You might get a popup with permissions errors for a few folders (most likely for Windows Defender) but just click continue several times and you'll get through it.
  5. That's it, you should now be able to open up all your apps.

Just a note, some of the registry folders already had the ALL APPLICATION PACKAGES group but some didn't so you're just finding the ones that don't have that.
