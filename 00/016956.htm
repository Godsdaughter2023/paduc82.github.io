video editing time format

joeu2004 say: Brad wrote:

I am creating a simple table to keep track of my comments on some videos.  The time format needs to be year, month, date, hours, minutes, seconds, frame #.  I know how to do everything except the frame number, which is 0 - 30 in 30th of second increments.  I don't need to do complex math by it, but I would like to be able to sort, so if I used a 100th of second that would be ok.  So far I have yyyymmddhhmmss, but what do I do for the frame number so that Excel will recognize it?


It would be better if you entered date and time in the form mm/dd/yyyy hh:mm:ss.fff, where mm/dd/yyyy should match your Regional and Language Options short-date configuration.

[EDIT] That is, use whatever your short-date confirguation is.  If not m/d/yyyy, perhaps yyyy/m/d or d/m/yyyy.

You can format the date and time with Custom yyyymmddhhmmss.000.  Note that fractional seconds must be ".000", with the period radix separator.

"fff" represents 1/1000th second.  If you only want to enter time to the 1/100th second, that's fine.  Just enter as hh:mm:ss.ff and format as hhmmss.00.

Caveat:  When you look at the time in the Formula Bar, the fraction seconds will not appear.  In fact, fractional seconds will be rounded to the second.  Consequently, you cannot simply edit incorrect data entry.  You must retype the entire time part.

The advantage is:  Excel stores date and time as a number, so sorting is quite natural.

[EDIT] How do you intend to enter "frame number" as "100th of a second"?  If you mean ss.00, ss.01, ss.02 up to ss.30, that might be sufficient for displaying and even sorting.  You say you "don't need to do complex math".  But that form will complicate even simple addition and subtraction.  If you need at least simple arithmetic capability, you should enter "frame number" as the decimal fraction equivalent of x/30.  For example, ss.00, ss.03, ss.07, ss.10, ss.13, ss.17, ss.20, etc.

I can also filter for only blank cells, by using ="=" as filter criteria - this will return only rows with blank cells in the column where ="=" was used as criteria.

 

But to make my sheet tidy, I don't want to enter data directly into the criteria range. Instead I want to use cell links in the criteria range, and have all criterias collected somewhere else. And this is where the problem enters:

 

A blank cell in a filter criteria is interpreted as "any value including blank"

A cell in a filter criteria with a reference to another cell will only work if there is something entered in the referenced cell, if it is blank, the criteria will not work as a blank cell. 

 

To better describe the problem, see examples below:

 

A B C
1 name make model
2 james
3
4 name make model
5 lars volvo v70
6 lars saab 9-5
7 james tesla
8 james MB e350

 

Run advanced filter, and set up A4:C8 as list range, and A1:C2 as criteria range. The result is a list showing rows 7 and 8, i.e. rows with search criteria "james" in column A.

 

If you now instead set this up with links in the cells in row 2, it will work if you have values in all cells, but not if you leave them blank:

 

 

A B C D E F G
1 name make model
2 =E2 =F2 =G2 james
3
4 name make model
5 lars volvo v70
6 lars saab 9-5
7 james tesla
8 james MB e350

=T(E2) - looks like a blank cell, but filter wont work

=IF(E2="";"";E2) - looks like a blank cell, but filter wont work

=IF(E2="";"=";E2) - if E2 is blank, A1 is set ="=" meaning an emty cell. But then it searches only for emty cells, not the rest.

 

So, the only way I can figure out to fix this is to write a macro that populates cells A1:C1 with the values I want to filter with. This works, but I want to avoid macro's. Does anyone know a way to reference a blank cell in a way so that the cell with the reference it is treated exactly as a blank cell?

Hi,

 

Welcome to Microsoft Community and thank you for posting. By the description, I understand that you need assistance in moving the e-mails and folders from PST file to IMAP Account (OST file).

 

The only way to transfer the e-mails from the POP (PST file) into IMAP is by copy, drag and drop the files and folders in Outlook. The PST files cannot be uploaded in order to sync with the webmail.

 

You will have to manually drag and drop or copy the e-mails to IMAP folder by selecting few e-mails at a time. An IMAP account does a complete sync between your mail client (Outlook) and the server every time data is moved or transferred. Meanwhile if there is any other activity initiated, there are chances that Outlook freezes because it would be taking too much resource at that given point of time.

 

Since you already have setup the IMAP account, you may open the PST file in Outlook and access whenever required.
