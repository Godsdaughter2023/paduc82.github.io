rich michaels - mvp (greatcirclelearning.com) say:
Hi Paul,

Here is a macro that performs the tasks you desire. 

BTW - The function that records macros recording on the Mac and the PC have not been kept updated since Word 2003 (PC) version. You can never be certain what it will and will not record.

Sub CleanupUpEdittingMistakes()
    Dim doc As Word.Document, rng As Word.Range
    Dim para As Word.Paragraph
    
    Set doc = ActiveDocument
    Set rng = doc.Content
    'remove four leading spaces
    With rng.Find
        .ClearFormatting
        .Format = False
        .Forward = True
        .Text = "^32^32^32^32"
        .Replacement.ClearFormatting
        .Replacement.Text = ""
        .Execute Replace:=Word.wdReplaceAll
    End With

    Set rng = doc.Content
    'replace two empty spaces with one
    With rng.Find
        .ClearFormatting
        .Format = False
        .Forward = True
        .Text = "^32^32"
        .Replacement.ClearFormatting
        .Replacement.Text = "^32"
        .Execute Replace:=Word.wdReplaceAll
    End With

    Set rng = doc.Content
    'remove emply space at end of paragraphs
    With rng.Find
        .ClearFormatting
        .Format = False
        .Forward = True
        .Text = "^32^13"
        .Replacement.ClearFormatting
        .Replacement.Text = "^13"
        .Execute Replace:=Word.wdReplaceAll
    End With

    Set rng = doc.Content
    'remove single empty space at start of
    'next paragraph
    With rng.Find
        .ClearFormatting
        .Format = False
        .Forward = True
        .Text = "^13^32"
        .Replacement.ClearFormatting
        .Replacement.Text = "^13"
        .Execute Replace:=Word.wdReplaceAll
    End With
    
    Set rng = doc.Content
    'replace En dash with Em dash
    With rng.Find
        .ClearFormatting
        .Format = False
        .Forward = True
        .Text = " - "
        .Replacement.ClearFormatting
        .Replacement.Text = " ^+ "
        .Execute Replace:=Word.wdReplaceAll
    End With

End Sub

embeddedengineerhta say:

I've decided to post a fix for the Lumina 1520 that worked well for me.  I've been reading on the web various solutions to the 1520 Phantom touch issue, and none had the following fix that worked for me.  My symptoms were the same; random touches, till the touch stopped working in which I power cycled the phone and it worked again.  This was the clue I needed.  I've seen this exact issue before on other embedded touch devices.

Since this is a touch issue, I opened my phone and removed the main CPU board and located the touch IC.  Look for a ribbon cable coming from the screen.  My phone uses the Synamtics touch IC, and as I expected it is a BGA (Ball Grid Array) soldered onto a ribbon cable.  Since the ribbon cable is bent over from the touch panel to connect into the main CPU board there is a small constant stress on the BGA solder connections.  That's why it worked great for about a year and then started exhibiting the issue.  Flexing the screen will possibly give you temporary relief, but the problem would come back again.

Having a BGA soldered onto a stressed ribbon cable is a notorious problem in the touch industry, and requires a lot of expertise to do it correctly.  Using a rework station I reheated (with hot air) the BGA till the BGA solder connections re-flowed.  This essentially re-soldered the touch chip.  Reassembled the phone and no more touch issue.  It's been working great for the last month.

This worked for my phone, hope this helps someone else.

doug robbins - mvp office apps & services (word) say:

You will need to put the selection in the Section in the two column Section of your document and format that Section so that it has a single column layout,  rather than two columns.  Then you should be able to removed the Section break without it affecting the first part of your document.

To correct your terminology, in the first instance, it wasn't a "two-column break" that you added.  What you would have done, was formatted a section of your document so that it had two columns as a result of which a Section Break would have been added before that section of your document.   Adding a column break is similar to a page break in that it terminates the text in one column of a multi-column layout so that it will continue in the next, or other column.
