pete_uk say:

If your date is in A1, then you could convert it to a text representation using this in a helper column:

=TEXT(A1,"mm/dd/yyyy")

and with percentages in B1:

=TEXT(B1,"0.0%")

If you fix these values and copy/paste them over the originals (then delete the helper columns) then they will not be converted to pure numbers.

Hope this helps.

Pete

pete_uk say:

The formula I gave you assumed that the date was in A1. If you have other dates in the same column, then you can just copy the formula down and they will be converted. If you have dates in lots of different places on your sheet, then you can still use the formula by copying it and then pasting it into a cell in the same relative position compared to where it was copied from. Suppose your date was in A1 and your formula was in E1, and that you have another date in C6, then you can copy the formula into cell G6 to give the same effect.

diane poremsky mvp (slipstick.com) say:

How are you syncing with the BB? It's easier (and safer) to disable it in the Desktop manager. (BES only syncs with the default calendar but if you use POP3, you aren't using BES.)

Basic instructions for using mfcmapi are here (with screenshots) http://www.outlook-tips.net/howto/delete_rr_mfcmapi.htm 

More specific instructions to delete folders - i'll have to double check this against a pop account in 2010 as this is how its done with Exchange accounts - I'm pretty sure its identical.

  1. Go to Session-> Logon and Display Store Table.
  2. Select the outlook profile of the user and double-click on the mailbox name where you want to delete the folder 
  3. It will pop up a window, expand "Root Container" and then expand "IPM_SUBTREE"
  4. Delete folders, such as Calendar, Task, and Drafts etc by right clicking and choose delete. 
     

Note that you should make a backup copy of the pst files before doing this as you can make of mess of things. Plus, the folders may be autocreated by outlook at a later date. For this reason, its better to control syncing in the BB software, not by deleting folders.

william zeng msft say:

Hi Criag,

I want to explain that desktop.ini can’t be uploaded to SharePoint Team Site Libraries normally by OneDrive for Business while desktop-xxxxxxx.ini can. So, the desktop.ini should be uploaded manually by the user through the browser. And he also syncs the new one during initial synchronization of the new folder. OneDrive for Business treated this as a version conflict, and starts syncing multiple versions of desktop.ini. So, it add some computer name to the desktop ini file. This is why we can see different kinds of desktop-%computername%.ini files.

Given this situation, I suggest you follow the steps below.

  1. Stop syncing a folder associated with users’ PCs.
  2. Manually remove the duplicate files (desktop.ini & desktop-xxx.ini)from the SharePoint Online library.
  3. Find the user who is in the original desktop.ini “Created by” column of the library and manually remove his sync folder from the local computer.
  4. Start syncing the library again on users’ computers.

@Ljatodd please refer to the symptom and suggestions above. If you have other questions, please share more details.

Regards,

William
