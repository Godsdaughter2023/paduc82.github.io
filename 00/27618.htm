rohn007 say:
What version of Office?  What type of license.

It is something of a grey area.

Assuming you have a retail license that allows install on more than on more than one device then in theory you should be OK.

Trying to install a PKC license on your computer and a virtual machine may work, but it would be stretching the terms of the license, because in theory you would be running 2 separate installations.

<snip 2010 MS License Terms>
d. Use with Virtualization Technologies.

Instead of using the software directly on the licensed device, you may install and use the software within only one virtual (or otherwise emulated) hardware system on the licensed device.

</snip>

That says you can have one copy of Office running per license, either installed directly on your local machine or running inside a virtual machine inside your computer.

Retail licenses allow you to install on 2 machines, so you could argue you should be in compliance by installing one copy on directly on the machine and the second on a virtual machine on the computer.

I've seen comments suggesting that people were able to use a PKC product key to install Office more than once simultaneously on a single machine in multiple partitions/OS's.   Is that legitimate? MS lawyers probably say no, but it seems that the activation system may have no problem ...

It's not clear exactly how you're making these settings. 

PowerPoint will retain settings during a session but when you quit and restart it, it'll return to the Windows defaults EXCEPT that you can save certain print settings with a PPT file and if you open the file and print w/o visiting the print dialog box, it'll use those settings instead of your printer defaults.

See if there's anything unusual in File | Options | Advanced | When printing ...
