thetao---01 say: Question: were the messages from August and later that you are trying to salvage all stored on the memory card? The memory card uses the FAT32 filing system, and so looks just like a DOS/Win95/Win98/etc. drive to a compatible
disk tool. Theoretically, you could put the phone in Mass Storage mode and use some Windows "recover my files" tool to undelete the messages. BTW, Symbian phones store messages in \Private\1000484b\, although the format details are likely undocumented. The problems doing this are

1. If the files are actually deleted (we won't know until you look), any new files saved to the card risks overwriting the ones you are trying to recover! This means any new SMS, new email, new files, new programs, etc. plus files written by the OS. And we don't know when this problem actually began. If you want to stand the best chance of recovering something, remove that card NOW and don't use it in the phone again until you either recover something or give up. (I am undecided if having messages stored on the phone, which uses a totally different filing method, would have been better or worse.) So depending on how much activity your card sees, there may not be much to recover anymore.

2. I can't recommend a good FAT32 recovery program. Months ago I contributed to a thread on memory card recovery, and recall another poster also not knowing of a really good program. Then there is the other approach of using a disk editor, where you are accessing the raw FAT32 structure yourself...but I can't recommend a good Windows-based track and sector editor either. :(

3. Each message is stored in its own separate file, and I believe the file names are just numbers. If you find a good recovery program and it doesn't list the files by directory, expect a lot of guesswork. Actually, a "recover absolutely everything and sort through it later" strategy is probably your best bet.

But first, install a binary editor (the free "Hex Editor Neo" will do...use a web search) and start going through everything in \Private\1000484b\. Just drag-and-drop to the editor. You'll figure it out. Keep in mind that you may be viewing email files or various who has already assembled the right tools (maybe contact a users group?), this might be worth a little money to know the job is done thoroughly and done right.

Per your comment about the sender, time/date, and some text of the messages still being visible...this doesn't convince me. Due to the very large potential size, the messages "database" must use an index file (think: a table of contents) to prevent the Messaging program from taking forever to load. When you run Messaging, it reads only the index, which contains a summary of each message. Only when you view a message does the full file get loaded. So the index file remains but the individual messages are either disassociated or deleted, which I think totally explains the problem.

When you say you were unable to copy and paste folders from the E5, please explain what you were doing. I.e. was the phone connected to your computer in Mass Storage mode? Did an error message appear?

Once again, good luck! :)

Microsoft Word 2010 spacing - Microsoft Community

prestonkeys say:

Microsoft Word 2010 spacing

In Microsoft Word 2010, I'm unable to lessen the space between (1) and an "open quote symbol."  Yet, in the same document, I can increase or decrease the space between (2) and an "open quote symbol".  (See below.)  Setting the cursor just before "One resident" and back-spacing just one space wipes out the (1).

jay freedman say:

Turn on nonprinting characters by clicking the ¶ button in the Home ribbon. Examine the characters between the numbers and the following words. Do they look like this, with an arrow in paragraph (1) and a dot in paragraph (2)?

If so, it indicates that paragraph (1) has automatic numbering turned on, but paragraph (2) is manually numbered. (The arrow represents a tab character, and the dot represents a space character.)

Most likely that was created by an AutoFormat As You Type option. Go to File > Options > Proofing > AutoCorrect Options and click on the AutoFormat As You Type tab of the dialog (NOT the one that's just "AutoFormat"). On the right side, is "Automatic numbered lists" checked? If so, clear that check box. Click OK in each dialog.

Now you can put the cursor before "One resident" and backspace, removing the tab and the (1). Then just type the (1) and a space, and it will be spaced like the (2). However, the paragraph will still have the List Paragraph style (applied by automatic numbering), and you'll have to apply the Normal style instead to align (1) with (2).

eaton guo say:

Issues Importing Calendar from PST

I recently upgraded my computer and was trying to restore my email accounts in Outlook. One of the accounts is an IMAP Yahoo account. However, this caused the calendar to appear as a folder under my yahoo email account. If I click on the calendar section of outlook, it is not listed with my other calendars.

I also attempted to add the calendar by adding the PST as a data file. However, this calendar appears to be offline because when I attempted to send a calendar invite to myself and accepted it, the response was not registered on the calendar itself.

karl timmermans (outlook mvp 2012-2018) say:
>>
I also attempted to add the calendar by adding the PST as a data file. However, this calendar appears to be offline because when I attempted to send a calendar invite to myself and accepted it, the response was not registered on the calendar itself.

I am using Microsoft 365 ProPlus.
<<

You posted the question in the Outlook.com forum focused on accessing an Outlook,com account via your browser but since you mention using Office 365, going to assume that the question is related to Outlook desktop (part of the Office suite).

in terms of the reference to the "calendar being offline" - not entirely certain what exactly you're asking about or want to accomplish so with that in mind, some background info on how "calendar's" work.

#1 - <IF> you are looking for the calendar items to get sync'd back to a server, you will need to have your email account configured as an Exchange account which includes Outlook.com. This does not include Yahoo accounts.

#2 - To be more specific, email accounts configured as either IMAP or POP3 accounts do not sync back to any server if that's what you mean by "calendar being offline".

#3 - There is (always) only one default contact and calendar folder in an Outlook profile. That is determined by the data file that is set as the "default data file" for the profile. You can determine which data file that is by going to Outlook menu
File --> Account Settings --> Data files tab

#4 - If you add a PST file with a calendar folder and want that to be your primary (default) calendar folder, you need to set this PST file as the "default data file" (this will also add all standard Outlook folders like <Inbox>, <Contacts> etc to the data file. In this case, The folder named <Calendar> will be the default calendar folder for the profile. You cannot set any other calendar folder in the data file as the default calendar folder.

#5 - If you have the account configured as IMAP that has a calendar folder with the added words "This Computer Only", means that either the ost data file for the account is the default data file for the profile OR that ost file was set as the default at some point in time. In either case, you really don't want to maintain contact/calendar data in any "This Computer Only" folder since if you remove the account from the profile, the OST data file will also be deleted from the system. Additionally, if Outlook can't open/read the .ost file, it will automatically create a new ost file resulting in the data in the "This Computer Only" folder being lost.
