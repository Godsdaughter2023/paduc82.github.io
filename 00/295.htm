ken sheridan say: The usual way to email a form's current record is not to attach the form, but to base a report on a query which references a control in the form bound to the form's recordset's primary key as a parameter.  The control can be hidden if desired.  The report will thus be restricted to the current record.  This method also has the advantage that it enables you to design the report independently of the form's layout, which is almost always preferable.

You'll find an example in InvoicePDF.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the report attached to the email as a PDF file is restricted by means of its query referencing the invoice forms invoice number control as a parameter.  My file uses the button's Click event procedure, but it could equally well use a macro as its In Click event property.

ken sheridan say: You can either reference the control in the form as the ControlSource property of a text box in the report, or you can reference it in the report's query as a parameter, in this case in the query's SELECT clause and then bind a control in the report to the column returned by the query.

For examples take a look at DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

In this little demo file the section on 'Retrieving data from the database' includes a form from which two versions of a report can be opened.  In the form are two combo boxes containing dates.  In the first version of the report the combo boxes are referenced by the report's RecordSource query and a text box in the report's header is bound to the column, in the second version they are referenced directly by the text box in the report's header.

From your screenshot, it appears that the message appears when you're trying to save a file with fonts embedded.
