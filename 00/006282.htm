jeeped say:
If you have a large number of cells to check, I would recommend using the COUNTIFS() function (introduced in Excel 2007) rather than the less efficient SUMPRODUCT() workhorse.

Using the same number of rows (1:999) I cycled through each of these 50×, comparing column A to B and the COUNTIFS() was 25% of the processing time for SUMPRODUCT() (10 seconds vs 41 seconds).

The formula for your CF rule would look like,

=COUNTIFS($A$2:$A$999, $A2, $B$2:$B$999, $B2)>1

In the interest of efficient use of processing, full column references (e.g. A:A or B:B) are never a good idea.

COUNTIFS function

Edit: COUNTIFS() does not require a third condition to exclude blanks.
