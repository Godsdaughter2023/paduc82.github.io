How do i run a module in Access? - Microsoft Community

thomasal say:

How do i run a module in Access?

I am using Access 2010 and am running a query to search for a specific substring within a field to return the records that contain only that substring.  i have created a module that would be able to test for the substring and return a value of 1 or 0 depending on if it was found but i do not know how to integrate that module into the query.  is there a way in SQL view that i can call the module, and would i have to pass values to it from the table to run properly?
all help is greatly appreciated, thank you.
ken sheridan say: You would actually call a function from the module, not the module itself, which is just a container for functions and sub-procedures.  It's important that you don't name a module with the same name as a function, though.  Giving module names a prefix such mdl guards against this being done inadvertently.

A function declared as Public can be called in a query.  You can do this in design view or in SQL view.  In design view you'd call it in the Field row of an empty column and in the column's criteria row enter the 'true' return value, 1 in your case, and uncheck the Show check box.  In SQL vie you call it in the WHERE clause.

So if you have a function named FindSubstring say, which takes a string expression as its argument, and you want to apply this to a field in the table named MyField you'd call it in design view with:

Expr1: FindSubstring([MyField])

Access inserts Expr1: or similar automatically.

In SQL it would be along these lines:

SELECT *
FROM [MyTable]
WHERE FindSubstring([MyField]) = 1;
