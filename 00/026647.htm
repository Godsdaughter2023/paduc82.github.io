ken sheridan say:

You can do it with a parent form in single form view, and an unlinked subform in continuous forms or datasheet view.  You can include controls bound to some columns in the parent form, and bound to others in the subform.  However, you cannot have controls bound to columns whose Required property is True unless these appear in both the parent form or subform, or the columns have a non-Null DefaultValue property, as failing to enter a Required value in one or the other would raise a data error.

In the single form view parent form's current event procedure put code like this:

    With Me.YourSubformControl.Form.RecordsetClone

         .FindFirst "YourID = " & Nz(Me.YourID, 0)

         If Not .NoMatch Then

             Me.YourSubformControl.Form.Bookmark = .Bookmark

         Else

              Me.YourSubformControl.SetFocus

              DoCmd.GoToRecord Record:=acNewRec

         End If

    End With

And in the subform's Current event procedure put code like this:

    With Me.Parent.RecordsetClone

         .FindFirst "YourID = " & Nz(Me.YourID, 0)

         If Not .NoMatch Then

             Me.Parent.Bookmark = .Bookmark

         Else

              Me.Parent.SetFocus

              DoCmd.GoToRecord acForm, Me.Parent.Name, acNewRec

         End If

    End With

The first keeps the subform in sync with the parent form, the second keeps the parent form in sync with the subform.  YourSubformControl is the name of the subform control in the parent form which houses the subform.  This might or might not be the same as the name of its source form object.  YourID is the name of the table's primary key.

In the parent form's AfterInsert and AfterDelConfirm event procedures put:

    Me.YourSubformControl.Requery

In the subform's  AfterInsert and AfterDelConfirm event procedures put:

    Me.Parent.Requery

PS:  The advantage of this approach over a conventional linked subform is that the navigational process is 2-way.  Whichever record you move to in either the parent or subform will take you to the same record in the other.

PPS:  Note that if you are embedding the parent form as a subform of another form, the following lines should be omitted:

         Else

              Me.Parent.SetFocus

              DoCmd.GoToRecord acForm, Me.Parent.Name, acNewRec
