Xbox Video to External hard drive - Microsoft Community

bruceleroy_ say:

Xbox Video to External hard drive

Is it possible to download purchased videos from Xbox Video to an external hard drive?

a. user say:

BruceLeroy_ -

Thank you for posting.

You can change the default save folder for videos, through the Windows Libraries.

The easiest way to do this is to open File Explorer from the desktop.  You should see Favorites, This PC, Libraries, and Network as headers on the left.  If you don't see Libraries, right-click in the blank area below Network and select Show Libraries.

From there, right-click on Videos and select Properties.  Click the Add... button, and you'll be able to add any folder your PC can see as a monitored folder for the video collection.  Once the folder is added under Library Locations, highlight it and click the "Set save location" button below that.  This will set that folder as the default save location for the Xbox Video app.

One caveat to doing this.  Setting an external drive as a save folder for license-restricted DRM content like movies purchased from Xbox Video can sometimes be problematic, if you hot-swap that drive a lot between systems.  While the files themselves are stored on the drive, the license is part of your Windows profile.  If that association gets lost, it can use up one of your download licenses for the purchase (or make a rental expire early) in order to play it again later.

Why is Outlook.com Breaking DKIM? - Microsoft Community

ravenstar say:

Why is Outlook.com Breaking DKIM?

I have my own domain and my own vanity email server.  The server signs it's mails using DKIM

I was testing a problem with DKIM sending the same mail to a number of addresses

outlook.com
yahoo.co.uk
blueyonder.co.uk

The DKIM results showed as follows.

outlook.com - fail

yahoo.co.uk - pass

blueyonder.co.uk - pass

I downloaded the source of each mail and verified that each server was reporting correctly by using DKIMPY's dkimverify command.

On investigating the mail and comparing the mail in my sent folder with the mail in my Inbox - I found that Outlook.com's mail servers had actually changed the contents of the email body significantly, thereby breaking my DKIM signature.  Surely this is contrary to the mail RFC's.  After all RFC5321 states that the only alteration allowed is the addition of trace headers.
