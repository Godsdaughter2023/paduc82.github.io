The misinterpretation might be because you use VBA Now to measure elapsed time.  That is incremented only each second on the system clock.  So the delta time can be off by as much as 1 second.

The problem might be compounded by the fact that you are including several unrelated Excel operations in the total time.  The time for each operation can vary greatly due to process management and interrupt handling.

Finally, I have noticed that there can be a large VBA overhead the first time a code path is executed.

Since you are looping only 3 times and, in some cases, each iteration might be close to 1 second (on my computer; YMMV), I wonder if those factors could explain such a huge difference intermittently.
