jay freedman say:

I've told you this a couple of times already, but maybe not explicitly enough: You not print anything -- all it does is run your code.

Instead of trying to figure out what's going on by looking at just a MsgBox, do this -- just try it, step by step:

  • Open the template's code in the macro editor and change it to this, which includes the code you proposed:

Private Sub oApp_DocumentBeforePrint(ByVal Doc As Document, Cancel As Boolean)

    Cancel = True

    Dim Response As Integer

    Application.PrintOut FileName:="", Range:=wdPrintRangeOfPages, Item:= _
        wdPrintDocumentContent, Copies:=1, Pages:="1", PageType:=wdPrintAllPages, _
         Collate:=True, Background:=False, PrintToFile:=False

    Response = MsgBox("Pick up printed sheet" & Chr(10) & Chr(13) & _
        "Insert into paper tray", vbOKOnly, "IAM")

    Application.PrintOut FileName:="", Range:=wdPrintRangeOfPages, Item:= _
        wdPrintDocumentContent, Copies:=1, Pages:="2", PageType:=wdPrintAllPages, _
         Collate:=True, Background:=True, PrintToFile:=False
End Sub

  • Save the template and close it.
  • Create a new document based on the template, and add two pages of text to it.
  • Go to File > Print (or press Ctrl+P) and click the Print button.
  • Observe that the first page prints immediately, and the "Pick up printed sheet" message appears.
  • Follow the instruction to put the sheet back in the tray.
  • Click OK in the message and observe that the second page prints. Also observe that nothing else happens -- Word's regular print routine does not run.

As a bit of extra advice, if the users are as inexperienced as you described, it would be best to make the message box more explicit: "Pick up the printed sheet. Insert it into the paper tray face up. Do not click OK until the paper tray is closed."

davidf2 say:

I was able to download the file and recreate your problem.

When I opened the file in both Pub 2007 and 2010 I had no problem accessing the text in the 'Follow-up Appt.' box but did with Pub 2013. It appears that Pub 2013 treats the layers differently.

First of all I would suggest that you go to View and check Boundaries. Now you will be able to see the text box boundaries without mousing over them. I am not sure why this is turned off by default but it is. It is much easier to see and understand your layout when you can see the boundaries.

Now click directly below the 'Follow-up Appt.' box in the empty space. When you do you highlight or select the rectangle shape that you have used to create a border around all your checklists. Your cursor will stop just to the right of 'Optical'. Now send that rectangle shape to the back. Go to Drawing tools format tab > Send Backward > Send to the Back. This was the topmost or upper rectangle shape.

Now again try clicking on the 'Follow-up Appt.' box and you will highlight another rectangle shape that you drew around those three columns. Once again send that rectangle to the back.

Now when you click on the 'Follow-up Appt.' box you can select the text. It appears that when you layer multiple rectangle shapes on top of text boxes they take focus away from the text in Pub 2013 while they didn't in earlier versions. You can see this more clearly if you temporarily insert a fill color into one of the rectangle shapes before you send it to the back. You will see that it obscures the  'Follow-up Appt.' box. Then send it to the back and you will see the 'Follow-up Appt.' box,

If you have this happen again be sure to click on the outermost, or the top layer and send it to the back and work down from the top.
