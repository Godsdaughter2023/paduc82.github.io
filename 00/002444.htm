sst 1.0 say:

numberformat in array elements vba

Hello,
an array loaded with numbers from an excel sheet (with cells formatted eg. [h]:mm:ss), does not retain the number format when fed to a chart. Is there anyway to "preserve" the original formatting so that the chart does not show silly numbers?
I know I can format the series labels but the series is loaded from data with different formats ( 4 to be exact).
I could create several series but I was trying to avoid that.
Thanks
rich007a say:

The "silly numbers" that you mention are the real value of date / time information that is formatted as date or time.  Time data has a value of 1=24hours, so 12hrs = 0.5, 1hr=1/24, etc.

 

So I think all you can do is "copy" the number formatting from the source data to the relevant chart elements:
 
    ActiveChart.Axes(xlValue).TickLabels.NumberFormat = Worksheets("Sheet1").Range("A1").NumberFormat
 

Why do you need to move the data as an array?  Can't you point the series at the range of the source data?  Then you could use:
 
    ActiveChart.Axes(xlValue).TickLabels.NumberFormatLinked = True
 
which is the equivalent of checking the Linked to Source option in the Format Axis > Number dialog.

 

Hope that helps.

Cheers
Rich

jan de messemaeker - project mvp say:

Hi,

 

As suggested in my previous post, one look at the Task Usage view explains it all.

Work is equally divided between the three resources, each picking up 33,33 hours; then duration is calulated by assignment and for the third resource this results in a duration of 66,67 hours, which equals 8,33 days. This being the longest duration of the three it becomes the task duration.

When you only assign 2 resources @ 100% each, they pick up 50 hours each and indeed duration becomes 6,25  days. As long as the units for the resources assigned are equal you will get the expected result.

There are several alternative ways to assign these resoruces that do lead to the expected result of 5 days duration. First, make sure the task is fixed units, effort driven of fixed work. Now assign the resources one by one, allowing Project to calculate after each assignment.

Another way is to go to View ribbon and click details. This will allow you to enter Work for eachof the resouces as well as their units.

Greetings,
