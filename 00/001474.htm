rena yu msft say:

Hi Paul,

 

The 5000 items limitation is a view threshold. Not a document library storage threshold. Therefore, you don’t have to reduce the size of the document library. You can just organize files to folders and ensure the items in the view don’t hit the threshold. Or you can create index and filter to get rid of the threshold also.

If you what you want to do is reducing the size of the document library, you can move some contents to another document library. The best way is using the open with explorer button to transfer contents among libraries. You don’t need to use PowerShell to do this.

In Internet Explorer > go to the library >click library tab>Open with Explorer

 

You can open multiple libraries and move contents on your computer conveniently.

 

Thanks,

Rena

gisle say:

Hi, Aaron!

I had this problem too.

The solution for me was to manually enable the windows recovery environment:

1. Copying the winre.wim file from c:\windows\system32\recovery\ to a folder on a recovery partition. (The partition need to have a drive letter at this point.)

2. Set the RE Image path. Exaple: reagentc /setreimage /path r:\recovery\windowsre

3. Enable RE with command: reagentc /enable

(You may remove the drive letter for the recovery partition after you have done this.)

This made me able to enable bitlocker.

What I troubled most with was that I tried to use the recovery partitions windows had made for me. Either the one that was 300MB or the one that was 333MB. But it turned out that both of them were too small. I made a new one of 450MB and that worked.
