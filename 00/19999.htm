Windows Defender

rob koch say:

ElderN,

You have the reasons behind Microsoft's design of its security products a bit twisted, possibly because you listen to articles in places like PC Mag, which those of us with a security background avoid like the plague since their primary purpose is to sell advertising for 3rd-party products.

The most fundamental basis behind the Microsoft security products is their objective criteria regarding malware, which is required when an operating system vendor is also attempting to provide protection from malware and use an even hand in deciding which applications are truly malicious.

Microsoft Malware Protection Center - How Microsoft antimalware products identify potentially unwanted software

However, as time wore on and both Microsoft and the 3rd-party security applications began to succeed in better controlling the true malware, many of these same malicious actors moved into the gray areas of spyware and adware.  This meant that Microsoft had to adjust their focus on these generally less malicious applications, but retain a similar objective criteria when doing so, unlike those 3rd-party products which may simply remove anything in sight that is possibly unwanted by someone.

adware A new approach – Microsoft Malware Protection Center

The problem for Microsoft is both that the PC belongs to its owner and the applications are developed by vendors with varying objectives as well.  While for some PC owners, the idea of having any adware on their systems my be a bad thing, for others the preference to use some for the free products or services these might provide may make them desirable.  So Microsoft must walk a fine line between what is truly providing a service that someone may want versus which are acting in a malicious manner or at least not providing the PC owner with proper and effective control over the operation of their own systems.

As a final note, I also tread in relatively dangerous areas of the Internet in order to determine how well the protection of Windows Defender and the other security settings provided by Microsoft within Windows and their browsers operate in protecting against current malware.  The difference is that instead of becoming infected, my goal is to avoid this in the first place, since an infection means that protection has failed.

The key difference here is that I never directly install malware on purpose, since that's a failure in personal choice that no security product can truly protect from, since the only recourse is to try and clean up after the fact.  This is never the primary goal of any security product and is instead an "if all else fails" situation that is often only partially successful.

In the 10+ years that Microsoft has been providing their own security products, I have yet to get a system infected while using the combination of Windows and Internet Explorer security settings with whichever Microsoft security product is currently supported on that system.  My only use of 3rd-party security in that time has been occasional manual scanning with products like MBAM, ESET and a few other online scanners to sanity check that nothing has gotten past this core protection, which it hasn't.

For the OP in this thread, if you regularly browse to questionable websites or install software from various 3rd-party (e.g. questionable) websites, or don't pay close attention while installing free software you'll likely get adware or worse as a result.  This is a choice and not really preventable by any security product, though same may try to protect from this.  There are security products which attempt to protect a user from themselves, but these will all eventually fail, so the question becomes how much you are willing to spend in order to not take the time to think before clicking.

The transient issues with the Edge web browser where pop-up announcement screens may be displayed on even relatively safe websites via advertising is a temporary anomaly of the fact that the Edge browser itself is so secure.  Unfortunately, the trickery used to display these screens is good enough that they confuse many less technically knowledgeable users into believing they are something more than an easily removed announcement.  In this case true adware isn't really involved, so there's nothing for a security program to detect or remove and the key is to learn how the new checkbox that Microsoft has provided to help close these is meant to be used.

Now that we know how the document is being populated (i.e. via a DOCVARIABLE field), we can provide a solution, which is to use a field coded as:

{QUOTE
{SET ID {DOCVARIABLE MasterAccountMerge.Account.Client.ServiceBilling.Agcy.Branch.BrnchFax.FaxNumber*6*152}
{SET EXP1{=-{=-{ID}-ID}/2}}
{SET EXP2{=-({ID}*(-1)-{ID})/2}}
{SET EXP3{=({ID}*(-1)-ID)/2}}
{=EXP1*1000000+EXP2*10000+EXP3 \# "000'.'000'.'0000"}}

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. The spaces represented in the field construction are all required, but the line breaks aren't - they just make the field code more readable.
