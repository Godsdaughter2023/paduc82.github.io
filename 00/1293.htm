a. user say:

Found a solution finally.

There's a machine-level policy under C:\Windows\assembly that tells an application that is backwards-compatible with older versions of Office to to be redirected and use the appropriate newer interop assembly associated with the more current version of Office that is installed, such as 365.

Copying this policy code and going to the application-level, where the application is installed there may be a config file for the executable (e.g. APPNAME.exe.config) if you open this file in notepad or a visual studio viewer there is likely a section dedicated to dependent assembly instructions. 

Add this to the end of that section before it moves on to other instructions:

<dependentAssembly>

<assemblyIdentity name="Microsoft.Office.Interop.Excel" publicKeyToken="" culture="neutral" />

<bindingRedirect oldVersion="12.0.0.0" newVersion="15.0.0.0" />

</dependentAssembly>

Save the file, launch the program again, test the process to populate an Excel template with bookmarks again. You may see an additional error that is for version 15.0.0.0 and states "access is denied."

Run Office online repair and try it again. Should be resolved.

bill manville say:

I don't understand "the address bar of the graph".
Maybe you were trying to create a linked picture in the way I suggested.

You can't use a formula as the source of a linked picture; it has to be a direct reference to a cell.

I don't think there is a way to do what you appear to be trying to do - use a formula to create a sparkline in a cell which matches the sparkline in another cell whose address you create as a string.

You could however use a macro to change the source of the linked picture - e.g. a Calculate event.

Private Sub Worksheet_Calculate()
  Me.Pictures(1).Formula = "=" & Range("L1") & Range("M1")
End Sub
