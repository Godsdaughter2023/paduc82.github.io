The only thing you're "missing" :-) is that the 2 operating systems are different, so the data connection works differently. On a Mac Word reads the data source from disk but holds the data in RAM... To succeed in obtaining an activated Insider Preview Build, you must upgrade from within an activated Windows 10 Build.

The only way to reliably do this using an ISO is to:

  • mount the ISO in the running activated Windows 10 device
  • run setup.exe from the mounted ISO
  • choose to keep Windows settings and files
  • do not enter any product key during or after the upgrade
  • since it does not use a product key to activate, it auto-activates after detecting a valid upgrade has been done.

I wonder if you have disk partitions that are posing a problem with the upgrade.  Would you be able to do a clean install of the official Windows 10 ISO, by booting with it and first deleting all partitions from your hard drive?

If your licensed Windows 10 is back up and running and is activated now, you could try testing a clean install to a small partition first to ensure that a clean install will auto-activate.

Is your PC UEFI or Legacy BIOS mode?  This changes the desired disk initialization and partition layout.  If Legacy BIOS / MBR disk you are limited to 4 primary partitions.

A standard Windows 10 clean install Legacy/MBR disk layout is a single combined 500MB NTFS System and Recovery partition containing System files and Windows RE, followed by a single NTFS partition for Windows.

A standard Windows 10 clean install UEFI/GPT disk partition layout has a 450MB NTFS Recovery partition (Windows RE) followed by a small 99MB EFI/System FAT32 partition, a 16MB Microsoft Reserved partition and then the main NTFS Windows partition.
