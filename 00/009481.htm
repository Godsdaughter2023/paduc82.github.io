    sFormat = Options.AutoFormatAsYouTypeReplaceQuotes
    If sQuotes = vbYes Then
        vFindText = Array(Chr(147), Chr(148), Chr(145), Chr(146))
        vReplText = Array(Chr(34), Chr(34), Chr(39), Chr(39))
        Options.AutoFormatAsYouTypeReplaceQuotes = False
        For i = LBound(vFindText) To UBound(vFindText)
            Set oSearch = oRng
            With oSearch.Find
                Do While .Execute(vFindText(i))
                    oSearch.Text = vReplText(i)
                    oSearch.Collapse wdCollapseEnd
                Loop
            End With
        Next i
    Else
        Options.AutoFormatReplaceQuotes = True
        oRng.AutoFormat
    End If
    Options.AutoFormatAsYouTypeReplaceQuotes = sFormat
    oRng.Select
End Sub

shane devenshire say:

Let me answer the question about switching to a scatter plot - in a scatter plot Excel determines the horizontal position by the value on the x axis.  in a regular line chart that is not how Excel works, in that case Excel considers the first datapoint of any series to correspond to the first item on the x axis, regardless of its value.  This is because each x axis elements are spaced equidistant from each other.  Consider the following data

1/1/2000       6

1/2/2000       11

3/1/2000       4

On a line chart with dates on the x axis all three points are equi-distant from each other with no regard to the value of the date.  On an xy scatter the first two items would be one unit apart but the third item would be displayed far to the right (about 60 units away). 

In your chart, done as a line, all series begin with the first data point being over the first item on the x axis, but you don't want that.

Regarding the dragging of the handles verses editing the formulas - the top selection handle of the series is also the bottom selection handle of the title.  So you have to work very carefully to get the right one, most often one gets the title sizing handle and nothing appears to happen.  Editing the formula just eliminates the problem.
