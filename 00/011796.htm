john - project say:
S.A.,
You asked if there is a better way of doing this. It seems the tasking a a little backwards. Perhaps a more logical sequence would be a milestone indicating when the panel order is placed (e.g. "place order for Solar Tech panels"), followed by a "x" day lead time for the panel manufacturer to do his thing (this could also be simply shown as a lag following placement of the order since the "task" of manufacturing the panels is at the panel supplier). This then feeds a milestone called "panel delivery". If you normally have 30 days to pay the invoice I suppose you could put another lag into the sequence for a "payment due" milestone but that's more of an accounting item, not a scheduling item. Remember Project is not designed to be accounting software, its planning and scheduling software.

As far as the material rate goes, when you identify a resource as a material resource, the Standard Rate field becomes the cost per unit, where the type of unit is identified in the Material Label field on the Resource Sheet. For example, let's say that each panel costs $100. That is entered into the Std. Rate field and "panels" would be entered into the Material Label field. Then when you assign the material resource to the "place order for Solar Tech panels" milestone you indicate how many panels are in the order, let's say 5. The Cost field will then show a total cost for 5 panels of $500. Or if the cost is by KW, then that is what you use for the material label.

Hope this helps.

John

jllatham say:

The fact that you've made the contents of the data validation in the Source exactly the same as the names of the worksheets the entry would go to makes it kind of slick to work with.

Below is worksheet event code for your Master List sheet that should do the job for you.  To put it into your workbook:  open the workbook, select the Master List sheet and RIGHT-click on its name tab and choose [View Code] from the list that pops up.  copy the code below and paste it into the code module presented to you.  Save the file as a macro-enabled file type (type .xlsm or .xlsb).

You'll want to make entries left to right so that choosing the source of the 'call' is the last thing you do because when you make that selection, all the data is going to be copied to the appropriate sheet.  The routine does not guard against double-entry on any sheet nor does it verify that you made the right selection.  If you find you've made double entries on the other sheets that you don't want, you can use the Remove Duplicates feature on the [Data] tab to clean things up.

Here's the code:

Private Sub Worksheet_Change(ByVal Target As Range)
  'change these Const values as needed/when needed
  Const nameCol = "A"
  Const addrCol = "B"
  Const phoneCol = "C"
  Const callCol = "D"
  'first row with data entry on it
  Const firstDataRow = 2
 
  Dim destWS As Worksheet
  Dim destRow As Long
 
  'did a change take place at row 2 or lower and in
  'column D?
  If Target.Column = Range(callCol & 1).Column And _
   Target.Row >= firstDataRow Then
    'assumes other sheets named EXACTLY like entries
    'in the type of call selection list
    On Error Resume Next
    Set destWS = ThisWorkbook.Worksheets(Target.Text)
    If Err <> 0 Then
      'worksheet does not exist, QUIT!
      Err.Clear
      On Error GoTo 0
      Exit Sub
    End If
    On Error GoTo 0
    'do the actual data copying
    With destWS
      'determine next empty row on destination sheet based
      'on entries in the type of call column.
      destRow = .Range(callCol & Rows.Count).End(xlUp).Row + 1
      .Range(nameCol & destRow) = ActiveSheet.Range(nameCol & Target.Row)
      .Range(addrCol & destRow) = ActiveSheet.Range(addrCol & Target.Row)
      .Range(phoneCol & destRow) = ActiveSheet.Range(phoneCol & Target.Row)
      .Range(callCol & destRow) = Target.Value
    End With
  End If
  Set destWS = Nothing
End Sub
