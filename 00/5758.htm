Sub A_line_and_space_breaks()
'
' pagebreaks Macro
'
'
    Selection.Find.ClearFormatting
    Selection.Find.Replacement.ClearFormatting
    With Selection.Find
        .Text = "^p^p"
        .Replacement.Text = "^p"
        .Forward = True
        .Wrap = wdFindContinue
        .Format = False
        .MatchCase = False
        .MatchWholeWord = False
        .MatchWildcards = False
        .MatchSoundsLike = False
        .MatchAllWordForms = False
    End With
    Selection.Find.Execute Replace:=wdReplaceAll

    With Selection.Find
        .Text = "  "
        .Replacement.Text = " "
        .Forward = True
        .Wrap = wdFindContinue
        .Format = False
        .MatchCase = False
        .MatchWholeWord = False
        .MatchWildcards = False
        .MatchSoundsLike = False
        .MatchAllWordForms = False
    End With
    Selection.Find.Execute Replace:=wdReplaceAll
End Sub

nette s say:

I would do it like this (you can adjust the number of times the code looks for extra spaces by adjusting the last number in the line For i = 1 to 5):

Sub RemoveParagraphsAndSpaces()
    Dim i As Long
   
    On Error GoTo RemoveParagraphsAndSpaces_Error
    Application.ScreenUpdating = False
   
    For i = ActiveDocument.Paragraphs.Count To 2 Step -1
        If ActiveDocument.Paragraphs(i).Range.Characters.Count = 1 Then
            ActiveDocument.Paragraphs(i).Range.Delete
        End If
    Next i
   
    For i = 1 To 5
        Selection.Find.ClearFormatting
        Selection.Find.Replacement.ClearFormatting
        With Selection.Find
            .Text = "  "
            .Replacement.Text = " "
            .Forward = True
            .Wrap = wdFindContinue
            .Format = False
            .MatchCase = False
            .MatchWholeWord = False
            .MatchWildcards = False
            .MatchSoundsLike = False
            .MatchAllWordForms = False
        End With
        Selection.Find.Execute Replace:=wdReplaceAll
    Next i
   
    With ActiveDocument.Range
        .Font.Name = "Times New Roman"
        .ParagraphFormat.Alignment = wdAlignParagraphJustify
        .Paragraphs.LineSpacingRule = wdLineSpaceSingle
    End With

    On Error GoTo 0
    Application.ScreenUpdating = True
    Exit Sub

    Err.Clear
    Resume RemoveParagraphsAndSpaces_End
End Sub

tswnet say:

Lookup Field Populates Other Fields in Same Table?

Can you us a lookup field in table 1 (that points to column 1 in table 2) to populate another field in table 1 with column 2 in table 2 data?

marshall barton say:

No!  Lookup fields by themselves are a BAD thing that leads to all kinds of confusion and mistakes (follow the link in the second commandment at http://access.mvps.org/access/tencommandments.htm)

 

What you are asking is a bad idea too.  Data needs to be stored in one and only one place because any duplicated data can get out of sync and wreak havoc on your database.

 

The "proper" way to do that kind of thing is to first change that lookup field back to a text box so you can see what is really in the field.  Then create a form to display the table's records and use a combo box to enter/display the field.  Set the combo box's RowSource to a query based on the lookup table and include the ID field, the Text field you see now and the other field you want to see.

set the combo box's ColumnCount to 3, BoundColumn to 1, and ColumnWidths to 0;;0  to make the combo box display the text while the stored value is the ID in the field.  Then you can display, but not store, the other field by using a text box with an expression like  =thecomboboxname.COLUMN(2)
