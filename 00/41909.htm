Which edition of Windows Vista are you attempting upgrade from?

If they editions don't correspond, you won't be able to do an in place upgrade:

When making significant changes to your computer such as an upgrade, it is always recommended you backup your system prior to installing a new version of Windows. In your case, you can upgrade to Windows 7 depending on edition and architecture of Windows Vista you have installed. If you are running a 32 bit version of Vista and want to upgrade to a 64 bit version (edition of Windows 7), you will have to a do clean install, this applies vice-versa.

If you are running consumer editions of Windows Vista such as Home Basic or Premium, you can only upgrade to either Windows 7 Home Premium or Ultimate. Also, you cannot do downgrades, which means, if you are running Windows Vista Ultimate now, you cannot downgrade to Windows 7 Home Premium. You must do logical upgrades:

Windows Vista Home Basic > Windows 7 Home Basic, Premium or Ultimate

Windows Vista Home Premium > Windows 7 Home Premium or Ultimate

Windows Vista Business > Windows 7 Professional, Enterprise or Ultimate

Windows Vista Enterprise > Windows 7 Enterprise

Windows Vista Ultimate > Windows 7 Ultimate.

¡firedog say:

There are now three users with the same symptoms using the same version of the program on the same OS. You all appear to have taken the right steps to conduct the export, yet it no longer works as it used to. I have merged the two threads to make the case more manageable.

There is a suspicion that a recent Windows Update may have changed something to cause this to happen. It would help to know, therefore, which updates were recently installed, starting with this month's Patch Tuesday ones released on 9 February and installed within the next couple of days. I haven't been able to spot a likely culprit, but since Windows Live Mail relies on .NET 2.0 for much of its functionality, changes here are always prime suspects.

It's very tedious to troubleshoot this kind of fault. The procedure would be to uninstall all recent updates, reboot and test to see whether the fault still happens. If it does, it wasn't one of those updates that caused it. If it doesn't, it was (most probably) one of those updates. In that case, install the updates again one at a time, reboot and test again. This is what I would expect product support staff to do, but there aren't any.

I would try first uninstalling KB3127222 on the off-chance that this update to .NET 3.5 (which includes .NET 2.0) is responsible. The quick way to uninstall an update: in an elevated command window, type

wusa /uninstall /kb:nnnnnn

To open an elevated command window, press Windows key + X and select Command Prompt (Admin)
