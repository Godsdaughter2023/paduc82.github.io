marshall barton say:

No.  This the same as trying to use a parameter as a table name in the FROM clause.

Parameters can only supply a value to an expression, they can not affect the structure of a query.  Your use of a parameter in the ORDER BY clause returned a text string (it's irrelevant that the string just happens to be the name of a field) and because the string is the same for every record, it has no effect on sorting.

Using an equation in Excel to create a graph

Hi,

I'm trying to make a graph on Excel of an exponential equation for a physics lab report and I don't know how to do that. Can someone help? The equation is Y=6-6e^(-t/1.298).

joeu2004 say: Kira wrote:

I'm trying to make a graph on Excel of an exponential equation for a physics lab report and I don't know how to do that. Can someone help? The equation is Y=6-6e^(-t/1.298)

First, determine the range of t for which you want to evaluate the equation (curve).  For example, 0.2 through 20 with increments of 0.2.  (Wild guess!)  Enter those values into T1 through T100.  (You might use the Fill Series feature.)

(On second thought, it might be better to start closer to zero and with smaller increments.)

Then in Y1, enter the formula =6-6*EXP(-T1/1.298).  Copy Y1 and paste into Y2:Y100.

Note:  1.298 is probably an estimate for some computation.  It would be better to use the computation instead of the estimate.  The same might be true of the two 6s.

In the Name Box (upper left), enter T1:T100,Y1:Y100 to select both columns.  Then click on Insert, Scatter Charts.

That's a start.

I was only able to find this information in regards to Excel that would explain this behavior:

When you change the width of a column to a fractional number, the column width may be set to a different number depending on the font used in the Normal style. For example, with a Normal style font of Arial, if you attempt to change the width of a column to 8.5, the column is set to 8.57 or 65 pixels. This behavior occurs because of the translation of font characters to pixel units. Fractional pixel units cannot be displayed; therefore, the column width rounds to the nearest number that results in a whole pixel unit.

bill mosca say:

Beth

I can verify that Access does indeed change widths based on the font used but also on the Grid numbers of a form. And not only columns, but controls such as text boxes, too. I've seen it happen lots of times over my many years of developing Access applications. You might be able to prevent it by increasing the Grid X and Grid Y properties of the form. The higher the number, the more accepting the form is to small fractions.

By the way, if you show your teacher that Access is actually changing your settings he really has no leg to stand on.
