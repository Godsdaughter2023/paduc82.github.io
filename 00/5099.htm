Hi CDB & MLK - sorry to see we are inflicting pain as you work with Office. I'm going to give you a bit of background on why things are set up the way they are and then hopefully get you working the way you want to work.

First - we look at cloud storage as having three big advantages:

1/ "safe storage" - your PC explodes, you don't lose your data

2/ "global storage" - you can get to your files from any device, and even replicate it locally for offline access

3/ "shareable storage" - documents stored in the cloud can be shared with other folks.

========================================

We feel that safety (1/) alone is enough for us  to make the cloud the default, to ensure that people who don't know/don't care where their data lives will usually have a safe backup and the ability to revert to older versions after those "oops" moments -  our aim is to deliver "it just works" experiences and keep our customers safe.

Replication (2/) is useful for folks who move from device to device, and you've noticed that we fix up the File Open MRU "most recently used" list (which roams with most users) to target the cloud copy so that all of your devices can share/open these documents from the MRU and you don't need to type/remember long URL for OneDrive or SharePoint stores.

And, the last of the background, as part of (3/) we work to provide maximum concurrent access to the document to allow multiple users to open the same, shared document simultaneously. This support varies from client to client (sometimes just the Web client, sometimes just the desktop) but is a very effective way of a team of folks coming together in a document and being productive.

=========================================

In your case I understand that you want to work with the benefits of (1/) & (2/) but not (3/) (your feedback is that server-hosted file save and even the initial open is just too slow) - I think you can get set up the way you want by doing a couple of things:

A/ In the OneDrive sync client (exposed as the white cloud icon in the system tray area) ensure that Office integration is off. We're pushing through a lot of sync optimizations here at the moment so the UI to do this is pretty fluid - for example on my own PC there is an "Office" tab, with a single option to uncheck.

There is a downside to this (which applies to other cloud storage providers as well) - if you change the locally synced file at the same time as another user changes the server file/another synced copy then you'll see "forks" (multiple copies of the file with a version number added to the name) and you'll have to reconcile the changes yourself by comparing files.

B/ In the Office app clear your MRU (file open, right mouse on the recent files, then "clear unpinned documents") - you need to do this as otherwise we'll remember that they are server files and open from the server directly.

C/ Now open sync'ed documents by treating them like other local files and navigating to the sync folder via "This PC" and opening the document/or starting from Windows File Explorer.

I hope this helps.

One last data point for you, we are working on substantial performance improvements in this area, we've started with the Enterprise "OneDrive for Business" client and will pivot to the OneDrive client soon (next couple of months).

We'll be changing the flow to open the locally synced copy (and get you started with your read/edit session) and then syncing up with the server without getting in your way. Big architectural changes are required for this but things are looking much faster.

Thanks for reading, and again, sorry for the pain....pc
