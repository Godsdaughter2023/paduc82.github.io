a. user say:

This is because Word exports the text parts using quoted-printable format, which is a "content transfer encoding" that uses "=" as an "escape" and also imposes a maximum line length with an "=" at the end - see, e.g. 

http://en.wikipedia.org/wiki/Quoted-printable


In other words, Word is making a correct encoding - it just isn't the one you want.

I don't think Word has any options to let you use an encoding such as 7bit ot 8bit, with longer lines etc.

Depending on what you are trying to do, I suppose you could instead save your file as Encoded text or find a utility designed to transform/decode MIME formats. (I don't have any suggestions on that, sorry).

Automatically Updating Fields in Access 2016 - Microsoft Community

rodneyheck say:

Automatically Updating Fields in Access 2016

Beginner question time:

I will keep this as simple as possible.

Lets say I have a Table called TechnicianTable with three pieces of information; The Key, FirstName and LastName. On the MasterTable I create the fields TechFirstName and TechLastName. I then link all the information from TechnicianTable to the field LastName on MasterTable. Is it possible to use the "Row Source" under the "Lookup" tab MasterTable in design view to populate that field in that record? If so, what is the syntax?

hansv mvp say:

Your MasterTable should contain neither TechFirstName nor TechLastName, it would be redundant information. Instead, create a field TechID (or similar), and create a relationship between the tables on TechID vs the key field of TechnicianTable.

You can then create a query based on MasterTable and TechnicianTable, joined on the fields mentioned above.

Double-click the join line and select the option to display all records from MasterTable, and only those records from TechnicianTable for which the join field matches.

Add all fields from MasterTable plus the FirstName and LastName fields from TechnicianTable.

You can use this query as record source for forms and reports.

ken sheridan say: For selecting a technician when inserting a row into MasterTable the usual control would be a combo box set up as follows:

ControlSource:   TechnicianID

RowSource:     SELECT TechnicianID, FirstName & " " & LastName FROM TechnicianTable ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

Alternatively, you can concatenate the names so that the last name is first, which would be better with a large list of names as it allows the user to enter the initial characters of the last name and progressively go to the first match as each character is entered:

RowSource:     SELECT TechnicianID, LastName & ", " & FirstName FROM TechnicianTable ORDER BY LastName, FirstName;

While this can be done as the DisplayControl property of a column in MasterTable, it is not a good idea to do so.  You should only interface with the data via bound forms, never in a table's datasheet, so in a form bound to MasterTable you would set up a combo box as above.

For a report use a query as Hans describes which joins the Master table to TechnicianTable on TechnicianID.  You can then bind text box controls in the report to the first and last name columns, or you can return the names as a single concatenated string expression in the query with:

TechnicianName:[FirstName] & " " & [LastName]

in the 'field' row of a blank column in query design view.  A single text box in the report can then be bound to the computed TechnicianName column.
