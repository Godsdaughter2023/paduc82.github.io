mbizup say:

Actually a fairy simple approach to what you're trying to do is to:

1. Create an unbound form, a main form to hold both your current main "edit" form and your datasheet form as subforms.

2.  Set the Master/Child links of your editable subform as follows:

      Master:  Forms!frmMain!frmDataSheet!ID

      Child: ID

     (You'll have to type this into these properties directly.  The "..." doesn't work with an unbound main form)

3.  In the Current Event of the datasheet subform, requery the edit form:

          

         Me.Parent.frmEdit.Form.Requery

With this set up, your edit form will automatically go to the record selected in the datasheet, with no additional code or custom navigation buttons.

peterlipecka say:

Just to let everybody know that I was able to change my navigation code to navigate inside the SubForm instead of the Form itself with very little code.  Everything works the way I want !

Code used:

Move To First Visible Record:

SubFormName.Form.Recordset.Move -SubFormName.Form.Recordset.AbsolutePosition

Move To Next Visible Record:

SubFormName.Form.Recordset.Move 1

*Note: Verifying the AbsolutePosition before moving Forward, so I don't navigate to RecordCount + 1.

Move To Previous Visible Record:

SubFormName.Form.Recordset.Move -1

*Note: Verifying the AbsolutePosition before moving backwards, so I don't navigate to Record 0.

Move To Last Visible RecordCount:

SubFormName.Form.Recordset.Move SubFormName.Form.Recordset.RecordCount - SubFormName.Form.Recordset.AbsolutePosition - 1

Current Event of the SubForm:

FormName.Form.Recordset.FindFirst "FieldName = Form![ColumnName]"
