Tagging .png and .gif files - Microsoft Community

hylia say:

Tagging .png and .gif files

Is there a way to add image tags to .png files? I know you can do it in windows photo gallery, but I wanted it just in the image information itself. I can add tags to .jpg files easily, but it seems that .png is not supported, the same applies to .gif files. If it can't  be done at this time, will it be added in the future?
jarrardi say:

Microsoft Windows XP and later editions, most image tagging programs (including Windows Photo Gallery), digital cameras, scanners, and pretty much any other picture-handling hardware or software in existence, store non-photo information (collectively called metadata) into a special section of image files that has been specifically defined by international standards organizations. These standards are specific to image formats (e.g. JPEG or TIFF); if no metadata standard has been defined for a specific format (e.g. BMP or GIF) then there is no way to save metadata (including tags) to it. You can store tags to .JPG files because JEIDA specifically wrote tag support into the EXIF standard for the Joint Photograph Expert Group file format. You cannot store tags to .GIF files because metadata was not included in the standard for the CompuServe Graphics Interchange Format. You could theoretically store tags to .PNG files because a lot of versatility was written into the Portable Network Graphics format, but to date no single standard for storing metadata to PNGs has arisen.
Windows Photo Gallery overcomes these limitations by storing the tags that you assign to metadata-less image formats into its own special database which it can consult any time you view the picture. Because this is a Windows Photo Gallery thing, not an operating system-wide thing or something stored in the image itself (as with JPEG and TIFF), tags saved in Windows Photo Gallery can only be viewed in Windows Photo Gallery for BMPs, GIFs, ICOs, PNGs, RAWs, TGAs, and other formats.

So in answer to your question: The .gif format will likely never support tags because it's fast becoming obsolete (being replaced by .png, ironically). The.png format probably will support tags someday when the standards organizations get around to it (and more importantly, when developers for all the products that work with PNGs gets around to adopting said standards), but we're at the mercy of large corporations and international committees of experts, neither of which are particularly noted for speed.
