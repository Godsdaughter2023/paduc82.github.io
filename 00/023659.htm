rohn007 say:

It appears you have 2 separate problems. 

First is the invalid product key message.

Second is the constant configuration.

2010 is not managed through the MyAccount website so you have very limited (essentially none) options to validate your product key.

Actually, both of your errors can be different symptoms of the same problem.

The configuration message happens when the registry is not setup for your specific version of Office. This can happen in 2 ways.

The first is that you have multiple copies of Office installed.  For example, most ne computers come with a "free trial" of Office 365 installed. Even if you don't use it, it is there.  Then you installed your copy of Office 2010.  Now, when ever you launch applications in the different versions, ie 2010 or 2016, during the application startup it checks to see if the registry is setup for it's version (ie 2010 for Outlook(?)) If it is not, then the configuration is triggered.

The other situation where the configuration message shows up is if Windows gets a little "stupid" and does not save the configuration information during Windows shutdown, so next time you restart, it is forced to "configure" again.

Lets look at the first situation, because it can also cause the invalid product key message. When you have 2 versions that have not been activated installed, if you try to activate the second one, ie 2010 in your case, the system submits your product key to the not activated version with the earliest install date, ie 365.  That automatically causes the invalid key message because a 2010 key will NEVER work for 2016/365.

The solution for this problem is to uninstall one of the versions, in this case taht would be the factory trial of 365.  Go to Control Panel > Programs and Features.  Scroll through the list of installed programs looking for more than one Office installation.  If you find more than one, uninstall the other(s) AND then do an Office Repair of the remaining one, Office 2010 in your case.  With a little luck it will already be activated, otherwise you will have to activate it one more time.

If you had only one version of Office installed and are getting the configuration message it could be a situation where Windows is not giving itself the required "Access Rights" during the configuration process. That means when you close and Windows tries to save your configuration it fails, so next time you start Windows, you do NOT have the appropriate configuration in the registry, triggering the message again (and again, and again ....).

In this case on possible fix is to, restart windows, then start any Office application using "Run As Admin". Let the configuration finish. Close the application. Close Windows normally. Start Windows normally and start the Office application normally to confirm that the problem has been fixed.

An increase from 23.78 to 25.00 is an increase of 1.22.  1.22 is 5.13% of 23.78; that is, 1.22 / 23.78 = .0513 (rounded).

 

A decrease from 25.00 to 23.78 is a decrease of 1.22.  1.22 is 4.88% of 25.00; that is, 1.22 / 25.00 = .0488.

 

It's all to do with which number you are using as the basis of calculation.

Mail Merge "Rule" on Word 2016 for Mac - Microsoft Community

lattedonna say:

Mail Merge "Rule" on Word 2016 for Mac

I am trying to use mail merge, using Word 2016 for Mac.  I want to merge a field, but also want to change the information that it inputs. I want to change it from an abbreviation to the full spelling.  I tried using the "if/then" rule, but can only see how to do that for one item.  If I have more than one item I want to change to a full spelling, how would I do that?  For example... if I have 4 status abbreviations (A,B,C ad D) and I want A= Alpha, B= Boy, C= Cat, D= Dog.. how can I accomplish that?

Thanks in advance for your help!

macropod say:

To do that, you could either use four separate fields, or three - or four - nested fields.

Separately:

{IF{MERGEFIELD Status}= "A" "Alpha"}{IF{MERGEFIELD Status}= "B" "Boy"}{IF{MERGEFIELD Status}= "C" "Cat"}{IF{MERGEFIELD Status}= "D" "Dog"}

or:

{IF«Status»= "A" "Alpha"}{IF«Status»= "B" "Boy"}{IF«Status»= "C" "Cat"}{IF«Status»= "D" "Dog"}

Nested:

{IF{MERGEFIELD Status}= "A" "Alpha" {IF{MERGEFIELD Status}= "B" "Boy" {IF{MERGEFIELD Status}= "C" "Cat" {IF{MERGEFIELD Status}= "D" "Dog"}}}}

or:

{IF«Status»= "A" "Alpha" {IF«Status»= "B" "Boy" {IF«Status»= "C" "Cat" {IF«Status»= "D" "Dog"}}}}

alternatively:

{IF{MERGEFIELD Status}= "A" "Alpha" {IF{MERGEFIELD Status}= "B" "Boy" {IF{MERGEFIELD Status}= "C" "Cat" "Dog"}}}}

or:

{IF«Status»= "A" "Alpha" {IF«Status»= "B" "Boy" {IF«Status»= "C" "Cat" "Dog"}}}}

Note: The field brace pairs (i.e. '{ }') for the above examples are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. Likewise, the chevrons (i.e. '« »') are part of the actual mergefields - which you can insert from the 'Insert Merge Field' dropdown (i.e. you can't type or copy & paste them from this message, either). The spaces represented in the field constructions are all required.

Hi Khalid,

To clarify the situation, I’d like to confirm how you “block one particular users email ID” and if there is a related PowerShell cmdlet.
Generally, we can use the PowerShell cmdlet Search-AdminAuditLog to search the administrator audit logs.
For example, you can find all the administrator audit log entries that contain either the New-RoleGroup or the New-ManagementRoleAssignment cmdlet via the following cmdlet.
Search-AdminAuditLog -Cmdlets New-RoleGroup, New-ManagementRoleAssignment
Also, you can use the following cmdlet to export all the admin administrator audit log entries to a CSV file for checking.
Search-AdminAuditLog | Export-Csv C:\logs.csv
