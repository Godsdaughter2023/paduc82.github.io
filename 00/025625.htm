john w. vinson say:

As Tighe says, no can do! A Date/Time value is actually stored internally as a Double Float number, a count of days and fractions of a day since midnight, December 30, 1899:

 

?now; cdbl(now)
1/8/2014 10:54:02 AM  41647.4541898148

As such, a Date/Time must refer to a specific instant in time; there is no single number that corresponds to "Sometime in October of 1934".

 

I dislike masks for dates in any case - Access is quite good at interpreting any real date, in various formats - but in this case it would appear that not only will a mask not work, neither will a date/time field. Could you explain the nature of the data, and what you plan to do with it?

 

ken sheridan say:
.......some of my data is only month and year, or sometimes only year.

In that case I'd use three separate columns of integer number data type for the year, month and day of month (YearVal, MonthVal,and DayVal say).  For data input of months, if you prefer to use month names, use a combo box setup as follows:

RowSourceType:  Value List

RowSource:  1;January;2;February;3;March;4;April;5;May;6;June;7;July;8;August;9;September;10;October;11;November;12;December

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

If you need to work with real date/time values you can convert the values of the three columns to a date with:

DateSerial(YearVal,Nz(MonthVal,1),Nz(DayVal,1))

If only a year is entered the date returned will be 1 January, if only a year and month are entered it will be the first of the month in question.

macropod say:

Your description suggests the footnotes themselves have the Footnote Reference Style applied; they should only have the Footnote Text Style (or another suitable Style) applied. The following macro will re-format all footnotes with the Footnote Text Style, leaving the references superscripted (and in the Footnote Reference Style).

Sub RepairFootnoteFormats()
Application.ScreenUpdating = False
Dim FtNt As Footnote
With ActiveDocument
  .Styles("Footnote Text").Font.Superscript = False
  .Styles("Footnote Reference").Font.Superscript = True
  For Each FtNt In .Footnotes

    FtNt.Range.Style = "Footnote Text"
    FtNt.Range.Font.Superscript = False
    FtNt.Reference.Style = "Footnote Reference"
  Next
End With
Application.ScreenUpdating = True
End Sub

For PC macro installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm
For Mac macro installation & usage instructions, see: http://word.mvps.org/Mac/InstallMacro.html

Do be aware that any intended superscripting in the footnotes will have to be re-done after running the macro.

Communication Site - Best Permissions? - Microsoft Community

tiffani preston say:

Communication Site - Best Permissions?

I have a Communication site that I'm needing to distribute to roughly 75 people in 5 groups. These documents do not need to be edited but shared for information. I also have macros that I am sharing. I found when giving read only access- the teams can only open the macros in excel online, VBA doesn't run on excel online. I also found that with read only acess -users can upload directly into the document library - if they do this with the same name as the original - it will replace it. I'm trying to create a permission to be able to download the documents, view but no upload option. Any ideas? SharePoint permissions are pretty new to me. 
tim liu msft say:

Hi Tiffani,

You could create a custom permission level in that site. Grant users that custom permission. Below are some steps for creating that permission level.

1. In that site, click setting gear>Site Permissions>scroll down to the bottom>click Advanced Permission settings.

2. Click Permission levels>Edit permission>scroll down to the bottom>click Copy Permission Level>it will create a copy for Edit permission.

3. Give the copied permission a new name, and then uncheck some permissions and make is the same as the following screenshot.

4. Return to the Permission list and change user's permission to this new permission.

5. They will only be able to download the file and cannot upload new files.

(test account can download but no upload option)

You could also try other setting options for the custom permission level for different requirements.

Regards,

Tim
