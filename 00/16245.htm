scottgem say:

There is no "Payment Type" field in Access. What it sounds like is that you downloaded an Access template that has a table with a Payment Type field in it.  I also suspect, like Gina, that this is a "lookup field". 

It is not recommended to use lookup fields on the table level. The reason for this is that it masks what is actually stored in the table. 

The proper way to design this is to create a table like:

tluPaymentType

PaymentTypeID (PK Autonumber)

PaymentType

Then add a PaymentTypeID field to your data table as a foreign key. Finally, on your form, use the combobox wizard to place a combo bound to your PaymentTypeID foreign key where you can select payment type.

As to the lists you mention, they don't seem to be data that you would store, but actions that you would process. So it would help if you explained more about your app and what it does.
