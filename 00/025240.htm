Difference between trend line and graph? - Microsoft Community

friendlyvampire say:

Difference between trend line and graph?

I have made a graph that is growing steeper at first, and then grows linear. I have added a trendline to the linear part of the graph, and now I have to find out on which x-value the difference between the trendline and the graph is exactly 18. Can anyone help me?
jim thomlinson say: Unless your graphed line is created from an equation then there is no exact answer. It would depend on how you are interpolating between the points. Assuming that you are using linear interpolation between your points then this is not too bad. You need to determine between which 2 points you are going to find your answer. To do this you need to get the equation for the trend line which you can do by right clicking the trend line and selecting to view the equation. Now just plug in your known x values to that eqations to get the corresponging y values. Subtract these y values from your known y values to find the 2 points where the difference is close to 18 (one side above and the other below). Now it is just a matter of doing a linear calculation to determine the point where the difference is 18.

Ok, follow this steps if you have an activation issue:
Access any of the Office products (Word, Excle, Outlook etc) and go to:
File \ Office Account
In the panel on the right, you will see the option to activate the license in yellow.

Try activating from there with the product key (if it is MSI) or with username and password (if it is click to run)

In case that the activation of an error continues it will be necessary to pause the antivirus and also perform a cleaning of the registry keys with the following steps:

  Remove product key
1. Locate the one in the path where the "ospp.vbs" file is located, usually it is:
1. C: \ Program Files (x86) \ Microsoft Office \ Office16
2. <C: \ Program Files \ Microsoft Office \ Office16>
2. Open a command console by specifying with CLIK RIGHT "Run as Administrator" and execute the command cd <C: \ Program Files \ Microsoft Office \ Office16> (or C: \ Program Files (x86) \ Microsoft Office \ Office16 depending on the path where the "ospp.vbs" file is located).
3. Execute the following command in the console: cscript ospp.vbs / dstatus
4. * Take note of the results of "product key", these would be 5 uppercase characters, for example 2PK9C
5. Execute the following command in the console: cscript ospp.vbs / unpkey: 2PK9C
 
Edit the record to delete cached credentials. To do this, follow these steps:
1. Click Start >> Run, type regedit, and then click OK.
2. In the Registry Editor, locate the following registry subkey: HKEY_CURRENT_USER \ Software \ Microsoft \ Office \ 16.0 \ Common \ Identity \ Identities
3. Remove all keys listed within this record.

Restart the system and try the activation again.

Let me know if the problem was solved with these steps.
