considerate_guy say:

I've seen several reports of problems trying to create a movie from .mp4 footage from various GoPro cameras.  The recommendation has been to do a conversion of the .mp4 video into .wmv format and use that instead.

I understand your aversion to starting over at square one, so there are a couple ways to work around that.  First however, you'll need to convert your footage from .mp4 to .wmv.  I've used Freemake Video Converter (freemake.com) for conversions, but you can use your favorite search engine to find other options.  If you do install Freemake, just install the program, not any add-ons it might offer.

When converting the files, I'd suggest putting them in the same original folder.  So you'll have two files with the same name (one with an .mp4 extension, one with a .wmv extension).

Now to trick WLMM into using the new file you have a couple options:

1) Rename all your .avi files to different names and then name the .wmv files so they have the extension .mp4.  This may trick WLMM into using the converted files in place of the originals.

2) Instead of renaming all the source files, make a manual edit of your project file in Notepad.  If you do this, I'd highly recommend doing it in a copy of your project file.  Right-click on the copy of your project file (.wlmp) and choose 'Open With' and pick Notepad.  What you want to do now is replace every file name ending in .MP4 with the corresponding file name ending in .WMV.  The quick way to do that is Edit > Replace.  Type .MP4 to be replaced with .WMV and click 'Replace All'.  When you now open your project file (the copy), it will reference the converted .wmv files instead.  It won't look any different, but your project should now save as a movie.

System Restore:  The error message: 80071A91

 

This problem occurs because, after the System Restore is performed, the Common Log File System (CLFS) transaction logs may be left in an inconsistent state. When the CLFS transaction logs are in an inconsistent state, you may experience the problems that are described in the "Symptoms" section.

 

Windows Update, I would suggest you to try the following methods and check if it helps.

Method 1:

Unregister and re-register the Windows Update

Unregister:   

a.     Click Start, click All Programs, click Accessories, right-click Command Prompt, and then click Run as administrator. If you are prompted for an administrator password or for a confirmation, type the password, or click Allow

b.     Type the following command, and then press ENTER:  wuauserv /UNREGISTER

Re-register:

a.     Click Start, click All Programs, click Accessories, right-click Command Prompt, and then click Run as administrator. If you are prompted for an administrator password or for a confirmation, type the password, or click Allow

b.     Type the following command, and then press ENTER:  wuauserv /REGSERVER

c.     Now, try your installation again. 

 

Method 2:

Empty the software distribution folder:

a. Click Start, click Run, type services.msc, and then click OK.

Note On a Windows Vista-based computer, click Start, typeservices.msc in the Start Search box, right-clickservices.msc, and then click Run as administrator.

b. In the Services (Local) pane, right-click Automatic Updates, and then click Stop.

c. Minimize the Services (local) window.

d. Select all the contents of the Windows distribution folder, and then delete them.

Note By default, the Windows distribution folder is located in thedrive:\Windows\SoftwareDistribution folder. In this location,drive is a placeholder for the drive where Windows is installed.

e. Make sure that the Windows distribution folder is empty, and then maximize the Services (local) window.

f. In the Services (Local) pane, right-click Automatic Updates, and then click Start.

g. Restart the computer, and then run Windows Update again.


Method 3:

Run the Update Troubleshooter.

Windows Update troubleshooter: http://windows.microsoft.com/en-US/windows7/Open-the-Windows-Update-troubleshooter
