1.   For example, I take a flight from VHHH to NZAA and before going on the flight, I file a flight plan, set up the flight at VHHH (with fuel, pax load, etc.) and then save that flight plan to desktop.   After completing that flight, I save the flight and exit FSX, and that plan is still there on desktop (Leave it there).   If I want to then go from NZAA to KLAX, I launch FSX using the saved flight and make up a new flight plan (as above) and save that to desktop.   So now I have two flight plans on desktop - don't delete the first flight plan as FSX still has to reference the old one before you can take off using the new plan.   After you finish this flight to KLAX, then the old plan (VHHH - NZAA) can be deleted.    And you can continue to do this for future flights.

2.   However, if I have misunderstood what you're doing and you are having a series of old flight plans on desktop (or in a folder), and you are simply going to a very old flight plan, thinking you can just use that to start on a flight without having to file a plan, then this won't work as your aircraft will be at a wrong airport as per the data on the plan.

3.   Or, if you're flying on a saved flight plan and then exit FSX.   Later on, you want to get back to that flight and flight plan, then this won't work properly as your aircraft will have moved.   In this case, before you exit FSX, save the flight (giving it a specific name) and then you can come back to it at its navigation point, altitude, etc.

Hope something here has answered your question.
