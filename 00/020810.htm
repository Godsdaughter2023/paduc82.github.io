j.e. mcgimpsey say: I rarely open a workbook from an unknown source, so I can't verify your data, but I'm certain that the problem is not with your data, but the size of your range.

I don't think I've seen a published XL2011 limit, but previous versions of MacXL (and all WinXL versions prior to XL07) limited the number of cells in a validation dropdown list to 2^15 - 1 (32,767) - i.e., the maximum value that can be stored in an Integer variable. Sounds like XL07 may have switched to a Long.

Is scrolling through 32K entries really something you expect your users to do? Even in a sorted list, that takes a significant time just to get to the end (30+ seconds in my informal test).

jim gordon mvp say:
I confirm Bob’s explanation as to what is happening. The List form control supports only 32,767 rows. The fact that the control fails to validate data as expected in Windows Excel is probably a bug in the Windows version of Excel. The control should either work right or not at all, and that is what the Mac version of the List control does. On the Mac, the List control works up to 32,767 rows and when that number is exceeded, it refuses to budge when it is clicked. 

Other built-in controls also contain the same row limits on both Mac and PC. 

Excel used to support a worksheet command called SQL.SELECT. This command would have been just the ticket to accomplish your task. To save money, Microsoft dropped support for the ODBC add-in that included this worksheet command. Microsoft saved money, but now you can’t use this command any more. If you want to suggest to Microsoft that they restore this command, visit Excel’s Help menu and choose the Send Feedback option to display the suggestion form. Explain that not having SQL.SELECT is costing you time and money.

The only way I know around the 32,767 limit is to use a SQL parameter query. This lets you have up to 65,536 rows in your source data and validates the entry against the source data. Doing it this way you would save the source document in .xls file format (this is where the 65,536 limit comes in), and then use a second workbook as your form. There is a detailed tutorial on how to set this up.

There's various ways to go about it - your way is fine, but rather than "ignore" the error I might be inclined to trap it and set the output value accordingly.

Dim Vtest As Variant
