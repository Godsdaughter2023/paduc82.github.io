jay freedman say:

There is no way to use an environment variable such as %LocalAppData% in a hyperlink address in Office. The hyperlink code doesn't contain a reference to the Windows routine for evaluating the environment variable. To invoke that, you need a bit of VBA macro code.

You can install this macro, which uses the Environ function to evaluate the variable and then launches File Explorer to display the folder. The template or document will have to be saved as a *.dotm or *.docm file, respectively, and stored in a folder that's specified as a trusted location.

Sub EdgeFavoritesLink()
    Dim favPath As String
    favPath = _
        Environ("LocalAppData") & _
        "\Packages\Microsoft.MicrosoftEdge_8wekyb3d8bbwe\" & _
        "AC\MicrosoftEdge\User\Default"
    Debug.Print favPath
    Shell "explorer.exe " & favPath, vbNormalFocus
End Sub

Then you can put a MACROBUTTON field in the body of the template or document to run the macro. You can apply the Hyperlink style to the field's display string to make it look like a hyperlink.

My feedback on the Windows 10 Preview at this moment. Bugs, - Microsoft Community

jochemkuijpers say:

My feedback on the Windows 10 Preview at this moment. Bugs, suggestions and annoyances

Hi Microsoft / Community,


I have been using Windows 10 in a virtual machine for the last 24 hours, about 4 hours of solid testing.

Here is my list of bugs, annoying things and suggestions for improvement:


Bugs I consider "breaking" or "major' and should be fixed at release:

  • Windows Explorer (the file browser) cannot handle other DPI values than 100%. It will render the whole window blurred and scaled.
  • Snapping a window to the side/corner of the screen and moving it back to the center (using a mouse drag) will not restore the original window size.
  • Merging a directory with a sub directory with the same name cannot be restored by Undo (Ctrl+Z).
    • To reproduce, create a directory name 'X' (e.g. C:\X)
    • Create a subdirectory named 'X' (e.g. C:\X\X)
    • Optional: Throw some files in there (e.g. C:\X\test.txt and C:\X\X\another.txt)
    • Now cut the subdirectory and paste it in the same directory as the 'outer' directory is placed in.
    • It will first of all not even mention for a file operation conflict (merging should NEVER be done without explicit instructions)
    • When pressing Ctrl+Z (Undo), the file system will not recover completely from the merge operation. In some cases where I tried this I will not even receive an error message something went wrong, just nothing happens.
    • THIS IS BAD.
  • "Frequent folders" in the Windows Explorer "Home" screen may show directories that are no longer available or have been deleted/renamed/moved.

"minor" Bugs:

  • When I change the width of the Start Menu and start searching, the default width is used instead of the current width (which is too wide in my opinion)
  • Connecting Windows 10 to a Microsoft account cannot be skipped during the installation. A workaround is disabling the wireless antenna, not connecting the the wireless network or unplugging the ethernet cable and restarting the installation configuration (which is stupid).
  • The slide animation of the taskbar context menu (the one with the colored bar at the top) can "bounce", meaning that it will show the slide-down animation followed by the slide-up animation. To reproduce:
    • Right click a pinned or active application on the taskbar so the context menu will appear.
    • Left click anywhere not on the task bar
    • Quickly right click a pinned or active application. The "new" context menu will first finish the slide down animation of the previous context menu.
  • Taskbar (pinned/active application) context menus have unnecessary colored bars at the top. Is this intentional or should it contain the application title or something else?
  • Selecting a dark window color that matches or is very similar to the window title color will make the title unreadable. Can either automatically or manually the title color be changed when this occurs?

Annoyances:

  • The Search box in the Start Menu has too much padding. It's a desktop environment, use desktop style rules, not "metro" style rules. Same goes for Start Menu context menus. Use the desktop ones!
  • "Recent Files" clutters the screen. Can I turn it off?
  • I can't remove Search from the task bar?
  • I can't remove Task View from the task bar?
  • There is no keyboard shortcut to move windows from one virtual desktop to another? (I suggest Shift + Win + Left/Right)
    • Virtual desktops should in this case be automatically removed (optional/customizable) when it is left empty.
    • Virtual desktops should be added when a window is moved to a new virtual desktop and the current virtual desktop will not be left empty.
  • There is no way telling which virtual desktop I'm currently watching without opening Task View. Can we have (optional) visual feedback when switching virtual desktops (Ctrl + Win + Left/Right)
  • There are in general no settings to customize virtual desktop (combined with multi-monitor setups) behavior? We need customization. The group of people using multiple monitors is small, but demanding. It should work our way or it shouldn't be there at all. Windows 8 really lacked on multi-monitor support
  • Using any of the window reordering tools (cascade, side-by-side, stacked) from the task bar context menu will not take in account that some windows may be on other virtual desktops. All windows will be organized as if they were on the same desktop.especially the task bar, third party tools somewhat made it acceptable but never 100% satisfying.
  • The Search app. WE DO NOT WANT IT. It's a stupid Internet Explorer window without any controls which loads a Bing page based on the thing I'm searching. I can see some people would find it useful, but I do not. Please allow me to nuke it out of existence if I want to. If you really really really really want to make a separate search button, please use my default browser with my default search engine. I don't use it because I like the link colors and fonts, I use it because it is adapted to my personal search profile and furthermore, some search engines (not to be named here) perform better than Bing.
  • Not able to only show active applications on the taskbar of the currently active virtual desktop.

Suggestions:

  • In general: more customization. Not just colors, also really "nerdy stuff", as the media puts it. We want more control over our system. Let us control every aspect that is already implemented as registry settings. Just give us a nice dialog to do this. Allow to disable the lock screen, remove the Search button from the task bar, display active applications only on the taskbar of the virtual desktop they are in, etc. etc.
  • The current close-minimize-maximize, etc. context menu on application windows may display: “move to other virtual desktop.
  • Name virtual desktops, assign start up applications to these virtual desktops and start them at their corresponding virtual desktop.

That's all I have for now.


Please at least consider each point as I have spent a lot of time writing these down and figuring out what is wrong. It would be nice if a developer can give some feedback as this will motivate me to test the current and future builds.


Kind Regards,

Jochem Kuijpers
