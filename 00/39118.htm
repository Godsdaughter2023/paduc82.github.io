To access it, you will need to boot into when you start your computer by pressing a function key. This can be either F1, F2, F9, F10, F11, F12 or even DEL or Tab key. Consult the owners manual that came with your PC for instructions about how to reinstall Windows.

     

     

    This is how the recovery partition is accessed for the most popular makes

    For Dell, press CTRL + F11 directly after powering up the unit

    For HP, press F11 directly after powering up the unit

    For Toshiba, press and hold '0' BEFORE and during the power up

    For Acer, press and hold ALT + F10 as soon as you see the logo

    For Asus, press F9 as soon as you see the Asus logo.

    For Advent, restart your computer. Then Press F10 repeatedly until the message "Starting System Recovery" is displayed

    lemp say:
    When "System Idle Process" indicates a high percentage of CPU use, it means that your CPU is doing almost no work -- it is idle.

    "Svchost.exe" is a generic process that "hosts" one or more system services that are implemented as DLL files rather than exe files.  If one of the svchost.exe instances peaks in usage after you disable your network connection, it may be hosting a service that is continually attempting to reach the Internet (e.g., in order to update something).

    Open Task Manager and click View > Select Columns
    Check the box for "PID (Process Identifier)" and click OK

    If you have XP Pro, open a Command Prompt window (Start > Run > cmd > OK)
    In the black Command Prompt window, type the following and press Enter
         tasklist /FI "imagename eq svchost.exe" /svc

    You'll see a list of running services like this:



    Match the PID of the instance of svchost.exe shown in Task Manager as using 90% of your CPU cycles with the PID in the list from running the tasklist command.  If you need help in determining what is causing the problem, provide the list of services for that instance of svchost.exe.

    If you have XP Home, Microsoft did not include the tasklist command for you.  Download tasklist.exe here --> http://www.computerhope.com/download/winxp.htm  and save the file in C:\Windows\system32.  Then proceed as described above for XP Pro.

    lemp say:
    The situation you're reporting now is not a problem.

    1.  That amount of memory in the "working set" of a process (10,000 K = 10 MB) shouldn't be a problem for most systems.  Even an older computer running XP probably has at least 512 MB of RAM installed.

    2.  PIDs are specific to your system.  Reporting that a process has PID 912 doesn't mean anything to anyone else.  The instance of svchost.exe that you mention hosts the most services of all the svchost instances in a typical Windows XP system (it is the "netsvcs" group of services), so it's not at all surprising that it needs the most memory. 

    Buried in that long list of services is wuauserv, which is the Windows Update Automatic Update service.  It wouldn't surprise me at all if that was the service that was using a lot of the CPU cycles when you switched off your Internet connection.   

    If you "end process" for this instance of svchost, most of the things that will stop working are network-related, which may not matter to you if you have disconnected your network connection.  Not all, however, are network-related.  For example, your sound will no longer work (AudioSrv). 

    For more information, go here --> http://www.blackviper.com/service-configurations/black-vipers-windows-xp-x86-32-bit-service-pack-3-service-configurations/  The names shown when you run tasklist /svc are the "service name" rather than the "display name," so click the header in the second column of Black Viper's table to sort by service name.  You can click on the "display name" of any of the services to find out more about it.
