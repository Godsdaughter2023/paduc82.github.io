saltgrass say:

As you probably realize, the message is saying your boot device is not being seen.  When this happens, my first though is something has changed what the bios sees as the Primary Boot device.

It may be something simple such as adding or removing another drive, plugging in some type of external drive or something else which causes the bios to lose its settings.  So the first thing is to get into the bios setup and confirm which drive or Boot Manger is selected as primary.  You could even try booting directly to a specific device from the bios.  While you are there, make sure your system time is correct in case the CMOS battery might be having a problem.

There could also be more serious causes such as drive corruption/failure, a SATA controller error or even a connection problem to the drive.  If you have altered any partitions on a drive it could be relevant.

I have seen a couple of threads where the OP eventually restarted the system the next day and it booted fine.  If you can boot into recovery media, use the Startup Repair section to try to correct the problem.  If it doesn't work after two or three attempts, hit the cancel button if you are offered one.  It might be bootable at that point.

Step 3: Determine whether the problem returns

Step 4: Enable half of the Startup items

Step 5: Determine whether the problem returns

If the problem does not occur, repeat step 1 and step 4. In step 4, select only half of the remaining check boxes that are cleared in the Startup Item list. Repeat these steps until you have selected all the check boxes.

If only one startup item is selected in the Startup Item list, and you still experience the problem, the startup item that is selected in the list is the service that is causing the problem. Go to step 6.

If no startup item causes this problem, a Microsoft service most likely causes the problem. To determine which Microsoft service may be causing the problem, repeat step 1 and step 2 without selecting the Hide all Microsoft services check box in either step.

Step 6: Resolve the problem

After you determine the startup item or the service that causes the problem, contact the program manufacturer to determine whether the problem can be resolved. Or, run the System Configuration utility, and then click to clear the check box for the problem item.

roger h. grant say:

Here is the solution to "error code 0x80070035 network path was not found" on Windows Vista and Windows 7 Computers. Click on the "START" button, select "CONTROL PANEL", and go into "DEVICE MANAGER". Click on "NETWORK ADAPTERS", then click on "VIEW", and select "SHOW HIDDEN DEVICES". In the expanded view you will see a long list of numbered "MICROSOFT 6to4 ADAPTER". My Windows 7 Professional desktop had 200 of them. Right click and select "DELETE" on all but 1 of them. You can only do 1 at a time so it does take a while. When you have only 1 left, restart computer and enjoy being able to see your other network computers, including HOMEGROUP files.
