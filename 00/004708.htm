suzanne s. barnhill say:
Actually, you can set the grid interval. Word 2007 and above make it much easier to turn the grid on and off (except in Word 2010 Starter, which has no View menu), but if you access the classic grid dialog, you'll find all the controls. On the Page Layout tab, in the Arrange group, click Align | Grid Settings. In the Drawing Grid dialog, you can set the grid interval. The default is 0.13" (which represents 0.125" or 1/8"). You can also choose to display gridlines at an interval other than the grid spacing. The default display is one gridline for every grid interval, but you can choose 2, 3, 4, etc. So, with the default spacing, if you set the horizontal and vertical to 4, you'd have gridlines every half inch.

suzanne s. barnhill say:

Are you referring to text generally or specifically text generated by the equation editor?

 

In general, Word's line spacing is determined by the font. Every font, by design, includes a certain amount of leading. For Times New Roman, Single line spacing is equivalent to about 120% of the nominal point size. So, as you have observed, 10-point TNR has Single line spacing of 12 points. This was doubtless by design since it produces 6 lpi, the conventional line spacing of typewriters. When word processing was new and many companies were converting from typewriters to Word, consistent line spacing allowed documents to, for example, meet court requirements for briefs, match spacing on printed forms, etc.

 

But TNR is just an example; the line spacing of Arial, Calibri, Cambria, and other fonts will vary. That's just one reason for using Exactly line spacing--to make spacing consistent when fonts are mixed. Another reason is that Double spacing often becomes too "spacey" at larger point sizes. For 12-point TNR, I prefer to use Exactly 24 points rather than Double spacing. This reproduces double spacing on a typewriter, which is the same whether you're using Pica (10-pitch, 12-point) or Elite (12-pitch, 10-point) type.

 

Whenever you are likely to have raised or lowered characters, you may want to use Exactly line spacing to keep the spacing consistent (though there are Compatibility Options that deal with this as well), or you may want to experiment with At Least spacing.
