shyamal debnath msft support say:

Hi Deneen,

1] A shared mailbox is a mailbox that multiple users can use to read and send email messages. Shared mailboxes can also be used to provide a common calendar, allowing multiple users to schedule and view vacation time or work shifts.

A shared mailbox doesn't have its own user name and password. Users can't log into a shared mailbox directly using Outlook or Outlook Web App. You must first add them to the shared mailbox (full access permission to a shared mailbox), and then they access it using Outlook or Outlook Web App. You don’t need to assign licenses to shared mailboxes, except when they are over their storage quota of 50 gigabytes (GB).

For more detailed information, please refer to the articles:
Create shared mailboxes in Office 365
Open and use a shared mailbox in Outlook Web App

2] A proxy (alias) address lets a user receive email that’s sent to a different email address. Additional email aliases are not mailboxes. Email sent to an additional email alias arrives in the user’s Inbox.

Adding email aliases to a user is helpful when:
• You want users to receive email at their primary email address such as *** Email address is removed for privacy *** and also an address like *** Email address is removed for privacy ***.
• A user changes their name, and you want to add an email alias with their new name while maintaining their primary email address.
• Email aliases don’t require separate user licenses. There are no fees associated with additional email aliases.

For your reference: Add or remove email addresses for a mailbox

If you have any further questions, please do not hesitate to contact us.

Best regards,
Shyamal

Hi Deneen,

No, you didn’t miss anything. This is an expected behavior.

Since the shared mailbox is different from our normal user mailbox, we need to assign send as or send on behalf right to users. The sender is ourselves instead of the shared mailbox. So the sent item does not appear in the shared mailbox.

We can manually enable this feature via PowerShell, so that we can  control  Sent Items when using shared mailboxes.

Connect to Exchange Online using remote PowerShell

Run the following cmdlets t can be enabled by setting feature enable flag to TRUE:

For emails Sent As the shared mailbox:

set-mailbox <Shared mailbox name> -MessageCopyForSentAsEnabled $True

For emails Sent On Behalf of the shared mailbox:

set-mailbox < Shared mailbox name> -MessageCopyForSendOnBehalfEnabled $True

If you then want to disable the feature again, you can do it as follows:

For emails Sent As the shared mailbox:

set-mailbox < Shared mailbox name> -MessageCopyForSentAsEnabled $False

For emails Sent On Behalf of the shared mailbox:

set-mailbox < Shared mailbox name> -MessageCopyForSendOnBehalfEnabled $False

gary's student say: Assuming the same tab name.

Create a third workbook call macro.xlsm and include the following macro in it:

Sub CopyData()
Dim wb1 As Workbook, wb2 As Workbook
Dim sName As String
Dim r As Range
Set wb1 = Workbooks("Weekly Metrics.xlsx")
Set wb2 = Workbooks("Chart.xlsx")
sName = "Previous Data"
wb1.Activate
Sheets(sName).Activate
For Each r In ActiveSheet.UsedRange
    If r.Value <> "" Then
        addy = r.Address
        r.Copy wb2.Sheets(sName).Range(addy)
    End If
Next
End Sub

Then insure that all three workbooks are open in the same instance of Excel and run the macro.
