tom van stiphout say:

No problem. Here is a function to get you started:

Public Sub CheckReferences()       
    On Error GoTo Err_Handler
    Dim ref As Access.Reference
    Dim strMsg As String

    For Each ref In Access.Application.References
        If ref.IsBroken Then
            strMsg = strMsg & ref.Guid & vbCrLf
        End If
    Next ref

    If Len(strMsg) > 0 Then
        MsgBox "Fatal Error: Unable to start application because of broken references. Quitting." & vbCrLf & "Broken GUIDs: " & vbCrLf & strMsg
        DoCmd.Quit
    End If

End Sub

Now, for some reason, I see the indents only in outline view; in print layout view they are all flush left.  Thanks for any advice you can offer.
john korchok say:

You can check that the headings are styles with the following steps:

  1. Click on a heading
  2. Choose Format>Style. If the dialog is reading Normal, the selected text is not actually a style. If it reads something like Heading 1 or Heading 2, you have verified that styles have been applied.

You can reactivate the indentation with these steps:

  1. Click on the first heading.
  2. Choose Format>Style.
  3. Click on Modify.
  4. At the bottom left of the dialog, change Format to Paragraph.
  5. Use Indentation>Left to set the correct indent. OK out.
  6. Repeat for all heading styles.

These steps should correct the document unless local formatting has been applied to the text over the styles. In this case, select the text and press Command + Option + Q to reset the paragraph formatting to the style.
