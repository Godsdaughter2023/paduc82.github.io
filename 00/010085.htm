Hi,

Some sites have not been updated to properly detect and identify IE11. It's a website problem that you can't fix, but there's  workaround, F12/Developer Tool.

On affected sites, Compatibility View may be the culprit.

Compatibility View tells websites that you're running an earlier version of Internet Explorer.

From IE> Tools (or Alt + t)> Compatibility View Settings, verify that the site is not in the list, that "Use Microsoft compatibility lists" is unchecked and that "Display Intranet Sites in Compatibility View" is unchecked.

 If Compatibility View is not the issue, you can use Developer tool to emulate/spoof another version of Internet Explorer.

Use the F12 key to open Developer Tool and click on Emulation then select another version of Internet Explorer in the User Agent String box. Usually IE10 or IE9 will be a better choice than IE8 or IE7.

To get the IE window in front of Developer tool, click the unpin icon in the upper right of the Developer Tool window. Once it's unpinned you can click on the IE window to bring it to the front.

The emulation/User Agent spoof will only last while the Developer Tool window is open.

Using NumberFormat to pad number out with leading zeros changes locale - Microsoft Community

ossiemac say:

Using NumberFormat to pad number out with leading zeros changes locale

When setting custom number format to 3 zeros the system changes locale to Chinese (Taiwan). If change the locale back to English (Australia) then lose the custom number format.

To re-create, set a custom number format to 3 zeros and close the Number Format dialog. With the formatted cells still selected, open the Number Format dialog again and locale is incorrectly set.

If custom format set to 4 zeros then the local changes to Portuguese (Angola)


This not an isolated problem. If Googled then there are quite a lot of hits like the following link that describes it fairly well with screen shots.

https://www.mrexcel.com/forum/excel-questions/764584-format-cells-occasionally-showing-chinese-taiwan-locale.html

gloria gong msft say:

Hi CJDAVISUK,

Thanks for your reply.

When you create a new Communication Site—top navigation is added to the site by default.

Technically the top navigation is the local navigation of the site (quick launch). No left side navigation option is available on the pages you create. The top navigation is a static list of headers and sub links and can have two levels. When new pages are added to the site—no new item for this page is added to the navigation. So maintaining the navigation is a manual action in Communication site.
