Yes, It's one way of "creating the data source that you need."

But bear in mind that
 a. you only get 255 or 511 characters of SQL, maximum. So if you need to transform a number of fields in the same way, you can run out of characters very quickly. Using table aliases can help quite a lot there.
 b. you have to live with the limitations of the query language supported by the data source, and some additional problems imposed by Word (for example, Word will probably insist that you quote the table name even when it is not required syntactically. Jet SQL is quite powerful, but some of the features available in Access will not work when you query via OLE DB. There can also be irritating treatment of long string fields (longer than 255 characters), columns with mixed data types and so on.

In more difficult cases, you might be able to go one step further by linking the .csv (or whatever) into an Access database and using a query stored in Access to retrieve the data. You can set that up without Access by using VBA, ADO and ADOX. You then don't have the query length limitation. But it's not something I would attempt unless I really had to.
