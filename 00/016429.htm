So do you understand now that if you want a negative result when SUM(C13:E13) > B13, you need to write B13-SUM(C13:E13) instead of SUM(C13:13)-B13?

B13-SUM(C13:E13) and B13-C13-D13-E13 are mathematically equivalent.  So is B13-(C13+D13+E13).

If you are seeing a difference of $50, I believe there are two likely explanations [1].

  • You entered one formula incorrectly.  I assume that actually unlikely because you mentioned that you are "pasting" formulas, a very good idea.  Presumably you mean copying from the Formula Bar and pasting into your posting.
     
  • Some of the cells in C13:E13 contain text that appear to be numbers, not bona fide numbers.

The SUM function ignores any text, despite appearances.  In contrast, when you reference a cell in an arithmetic expression, Excel will interpret text as numbers if possible.

To confirm, enter =TYPE(C13) somewhere for each of C13:E13.  If it returns 2, you have text.

Alternatively, simply go with the assumption and correct it as follows.  Put the number 1 into some cell and copy it.  Then select C13:E13, right-click and click on Paste Special > Values > OK.  You can now delete the number 1.

PS:  A third possible explanation:  you are formatting numbers so that they round to thousands or greater.  Formatting affects only the appearance  of values; it does not change the actual values.  For example, if A1 is 1149 and A2 is 1099, and they have the Custom format [<500]0;0,"000" , they both appear  as 1000, so you might expect their difference to be zero.  But A1-A2 is 50 because their actual  values remain 1149-1099.

Endnotes

[1] One other very unlikely source of the $50 error:  the magnitude of the sum of C13:E13 is a 16-digit integer.  Hah!
