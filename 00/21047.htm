jim gordon mvp say:

Documents that arrive as email attachments open as Read Only because Apple requires Microsoft to "sandbox" Office 2016. For a detailed explanation of what sandboxing is, visit this link:

https://developer.apple.com/library/content/documentation/Security/Conceptual/AppSandboxDesignGuide/AboutAppSandbox/AboutAppSandbox.html

What it means for users is that Apple and Microsoft now prevent you from accidentally infecting your Mac from malicious code that could be part of a macro-enabled Word document. Sure, Word's default is that you will receive a dialog asking you to enable macros, but Apple thinks that's not enough. Now, you have to actually save your attached file to your documents folder or other approved directory before you can open an attached Word document normally.

This applies to everyone using Outlook 2016 regardless of subscription or license type.

Hi Dipak,

Thank you for posting your question in Microsoft Community. I shall assist you with the issue related to accessing of the files created in Excel 95 file format in Excel 2010.

 

Microsoft Excel 5.0/95 Workbook is saved in the .xls format, hence can be opened in Excel 2010.

 

You may try the steps below to open Excel files in Excel 2010 and work with them:

Open Excel 2010 Click on file, open, browse to the location where the Excel 95 file is saved and open it.

 

Once the file is opened you may save the file in 97-2003 format in order to open the file on different computer using Excel 2003 or save it in default 2010 format (.xlsx).

 

For more information you may refer to the link below to know the file formats supported in Excel 2010:

hendrikschmieder say:

It looks like I found  a way

I take a look at the registry:
 
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Addin
 HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Addin\CLSID
 
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Application
 HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Application.12\CLSID
 
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Sheet\CLSID
 
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Sheet\CurVer
 HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Sheet.12\CLSID
 
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.OpenDocumentSpreadsheet
 HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.OpenDocumentSpreadsheet.12
 HKEY_LOCAL_MACHINE\SOFTWARE\Classes\ExcelWorksheet\CLSID

I made sure that
 
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Sheet\CurVer has value Excel.Sheet.12
 
and

HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Sheet\CLSID
 
has the same value as
 
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Sheet.12\CLSID
 
I then checked the paths in HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Sheet.12 and
 
for HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Excel.Sheet.12\CLSID .
 
Siniliar for the other keys above.

You should also check for the same paths under HKEY_CURRENT_USER.
They could be there but mustn't be there.

Process monitor was of help here.
