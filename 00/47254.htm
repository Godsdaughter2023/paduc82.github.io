The Xbox is dependent upon the router to provide it an IP address and to allow the Xbox to go online.


"When I test the internet connection, it has shown me a lot of different error messages, e.g., Teredo, DHCP error, etc."

 - It means that the router is not communicating with the Xbox


Here's a few things to try.


Hard reset the Xbox by holding down the power button on the front of the Xbox for 15 seconds. After the 15 seconds let go of the power button then restart the Xbox.


Try out your Xbox again.


If that fails, try these steps.

Power cycle your network by unplugging the power to the modem & router for about 60 seconds. Then plug your router back in and try your game again.

Go to Settings > Network Settings > Advanced Settings > Alternate MAC Address > Clear > let the Xbox Restart


Try out your Xbox again.

If that fails, try these steps.


If you're using a wireless connection to your router.

On your Xbox go to Settings > Network > Network Settings > Forget Wireless Network


If wired connection go straight to this.

Go to Advanced settings > DNS Settings > Manual - Set the Primary DNS as 8.8.8.8 and the Secondary as 8.8.4.4

Go to Settings > Network Settings > Advanced Settings > Alternate MAC Address > Clear > let the Xbox Restart


After the Xbox Restarts go back and reconnect to your Wireless Network and try your Xbox again.


If that fails, consider resetting your router back to defaults.

bossdweeb say:

"Or to definitively test whether older versions of DirectX are functioning as promised?"

- I can assure you that all versions of DX still work well on Win10.

In regard to specific DX file numbers, there is no such thing as 'older' versions of those DX files (the 'old' versions have not been updated).

There are still a few superseded DX installers, but the most recent DX installer has the same 'old' DX9\10 files as do the older installers.

The only thing 'new' are some DX9\10\11 files that were first released with the

Download Details - Microsoft Download Center - DirectX Redist (June 2010)

"deleting various DX.dll things then running sfc/scannow "

When a DX error specifies a particular DX file, eg- d3dx9_33.dll , if you delete that file and run the 

Web Installer from Official Microsoft Download Center  -  it will replace the now 'missing' file with a new copy.

- Rather than use System File Checker when corrupt DX is suspect and there are no files specified, reinstalling DX with the

full installer package (June 2010)  is the better option. This will overwrite all DX files (except DX12).

"That's how I found out that I got a system error when installing DirectX"

This issue with DX errors, particularly when there is an error when trying to reinstall DX (using the official  DX Redst -June 2010,

or the DX Web Installer) has been a problem on Win10.

Assuming your issue is the same, it's not to do with DX at all.  The problem is actually due to a corrupt .NET installation.

( DX12 may use a newer .NET version that does not get corrupted ? I believe most posts I've seen with this bogus DX error

have been with games that use DX9\10 ).

-Give the repair .NET tool a go....

https://docs.microsoft.com/en-us/dotnet/framework/install/repair

Other potentials...

- If you have done a Win10 version update you will likely also have some games from Steam and other online retailers that will need

'repairing'. I have a number of games that have need repair after most of the version updates.

If you have not already, try running Steams Verify Integrity of Game Files tool on the offending games.

(R\click games title in your Steam Library > Properties > Local Files tab). Origin, etc have similar repair options.

- If the games that do not launch are on disc and you haven't tried them on Win10 before the problem is very likely due to

DRM Blocking. This is a 'security feature' MS Built into Win10 that prevents disc based games that use SafeDisk copy protection

form launching. I have more details regarding DRM Blocking and some other DRM issues in my post at    ---this thread---

( I don't have data on newer versions but the original TW  series used SafeDisk )

- I don't know what the change in the value in that Registry key is supposed to do, but it may just be hiding the real problem

rather than fixing it ? I have seen another Registry fix that changed both binary value and edit string value data, but don't know

exactly what it's suppose to do.

However, if the altered key didn't help it's probably safer to revert to default values.

- For something more radical... Someone else with this DX error deleted his whole DX Registry folder, and rebooted.

The folder apparently rebuilt itself and it fixed his issue (no follow up on long term effect though).

Hi Gary,

There are several factors that we consider why we're unable to receive our security code. One of those factors is because of our alternate email address end in @outlook.com, @hotmail.com, @live.com or @msn.com. If you're using one of those alternate email addresses, this can make it tricky to keep track of which one you're signed in to. When you sign in to the second account (to get the code sent to that address), most browsers automatically sign you out of the first account (the one that's actually requesting the code). To assist you in receiving your security code, you may follow the steps below:

  1. Sign in to the second account with a browser in privacy mode. This option will let you stay signed in to both accounts at the same time.
  2. When you're prompted to enter the security code that was sent to your alternate email address, don't close the browser window.
  3. Open a new window in privacy mode. Ctrl + Shift + P is the shortcut for InPrivate Browsing in Internet Explorer and Edge. If you're using a different browser, check that browser's help for info about privacy mode.
  4. In the new window, sign in to your alternate email account and look for the message from the Microsoft account team. Copy or write down the security code from the message.
  5. Return to the window that is asking you for the security code. Enter the code and follow the instructions.

You may also refer to Paul Seymors's answer posted in this thread.
