In this little demo file, if you click the 'Select Criteria for Report' button, and then, in the dialogue form, select more than one item in the list box and select the 'All Items' option the report will show (in my case) stores who stock the items selected, highlighting the items in question in red in the report.

While my demo opens a report, it would be a simple matter to open a form in the same way of course.  The code which opens the report and restricts it in this way is:

Private Sub cmdOpenReport_Click()

    Const REPORTCANCELLED = 2501
    Const MESSAGETEXT = "No matching items to report."
    Dim varItem As Variant
    Dim strStockIDList As String
    Dim strCriteria As String
    Dim ctrl As Control
    
    Set ctrl = Me.lstStock
    
    If ctrl.ItemsSelected.Count > 0 Then
        For Each varItem In ctrl.ItemsSelected
            strStockIDList = strStockIDList & "," & ctrl.ItemData(varItem)
        Next varItem
        
        ' remove leading comma
        strStockIDList = Mid(strStockIDList, 2)
        
        If optAnyOrAll = 1 Then 'any item stocked
            strCriteria = "StoreID IN(SELECT StoreID " & _
                "FROM StoreStock " & _
                "WHERE StockID IN(" & strStockIDList & "))"
        Else                    'all items stocked
            strCriteria = "StoreID IN(SELECT StoreID " & _
                "FROM StoreStock " & _
                "WHERE StockID IN(" & strStockIDList & ") " & _
                "GROUP BY StoreID " & _
                "HAVING COUNT(*) = " & ctrl.ItemsSelected.Count & ")"
        End If
        
        On Error Resume Next
        DoCmd.OpenReport "rptStoreStocks", _
            View:=acViewPreview, _
            WhereCondition:=strCriteria
         Select Case Err.Number
            Case 0
            ' no error
            Case REPORTCANCELLED
            ' anticipated error
            MsgBox MESSAGETEXT, vbInformation, "Warning"
            Case Else
            ' unknown Error
            MsgBox Err.Description, vbExclamation, "Error"
        End Select
    Else
        MsgBox "No items selected", vbInformation, "Warning"
    End If
    
End Sub

ken sheridan say: That is not a valid model.  What you have is a many-to-many relationship type between farmers and products.  A binary (2-way) many-to-many relationship type is modelled by a table which resolves it into two one-to-many relationship types.   Price is an attribute of the relationship type, and hence a column in this table.  There is also redundancy in that city determines county, which in turn determines state, so decomposition of the table into a normalized set of related tables is required.  Assuming there is only one farmer per farm, farm is an attribute of farmers.

So, in broad outline the model is made up of the following tables:

Farmers
….FarmerID (PK)
….FirstName
….LastName
….Farm
….Address
….CityID  (FK)
….Zip

Cities
….CityID  (PK)
….City
….CountyID  (FK)

Counties
….CountyID  (PK)
….County
….StateID  (FK)

States
….StateID
….State

Products
….ProductID  (PK)
….Product

and to model the relationship type between farmers and products:

FarmerProducts
….FarmerID (FK)
….ProductID  (FK)
….Price

The primary key of the last table is a composite one comprising the two foreign key columns.

Diagrammatically the model would thus be like this:

Products---<FarmerProducts>----Farmers>----Cities>----Counties>----States

In each relationship referential integrity should be enforced.  US state names are distinct, so the State column is a candidate key and should be indexed uniquely.  City names, and I'd imagine county names in the USA (not in the UK), can legitimately be duplicated, however, so are not candidate keys.

    The "ignore all" information is not saved in the document file, so it doesn't survive shutting down Word and later reopening the document. All versions of Word behave this way.

    To ignore certain words and phrases permanently, you need to apply the "do not check spelling and grammar" setting from the Set Language dialog to each occurrence. The preferred way is to make that setting part of a character style, which you can apply either per-occurrence with a keyboard shortcut or by using the Replace function. The setup is described at http://wordfaqs.mvps.org/MasterSpellCheck.htm#ExemptingText.
