ashish mathur say:

teylyn say:

Ashish's suggestion uses the Indirect() approach I mentioned. 

Summarizing this for the benefit of readers who don't want to click through to three different sites to find how it works:

- On each sheet, have one cell that has the name of the previous sheet, for example cell A1 on Sheet2 has the text "Sheet1"

- use Indirect() and build a cell address with the sheet name mentioned in A1 like this:

=Indirect("'"&$A$1&"'!B20")

This formula will resolve to =Sheet1!B20 and retrieve the cell value of B20 on the sheet that is listed in cell A1.

Note that INDIRECT() is volatile, though and can cause slowness, especially in large datasets with many formulas, because any time that ANY cell in the workbook changes, ALL formulas in the whole workbook will be re-calculated. In small files, that won't make much of a difference, but in some files you will find yourself staring at the "Recalculate" message a lot.

Find and replace when creating the new sheet might be a better option.  

ashish mathur say:
You obviously have not read my solution carefully.  INDIRECT() is only one of the functions that has been used there.  There are other XLM 4.0 macro functions such as GET.CELL that have been used.  Read that article carefully.

teylyn say:

I understand the technique, Ashish. 

Please allow me to just point out the performance drain that INDIRECT causes in large datasets. Of course, using Get.Cell and Get.Workbook adds another complication, since it also requires the file to be saved as a macro-enabled file.

If going the macro way is not a problem, then a quick VBA routine that deals with references to the previous sheet will be another option to avoid the INDIRECT performance drain.

Selecting a cell with a hyperlink:

When using the mouse, select the cell, but hold the mouse left key down momentarially (about helf a second?) until the cursor changes from a hand back to the normal plus. You would then need to use the F2 and control-shift-home combination to copy the text of the link.

Alternatively you can adjust the row height (suggest by at least 1.5 times) you will then be able to enter the cell by clicking just above the hyper link text, which will mean you can use the double click method - double click into the cell just above the link text, hold the right mouse button down and drag down below the text to select the hyperlink text.

Word document reverting to normal .dot from my template - Microsoft Community

archway debbie say:

Word document reverting to normal .dot from my template

Hi

I am creating a document based on a template on my network. sometimes but not always when I send it to a client it reverts to normal .dot so the client can't see the formatting from my template. Why does this happen?

jay freedman say:

In your template and in each existing document that's based on that template, do the following:

  • If you don't see a "Developer tab" on the ribbon, go to File > Options > Customize Ribbon and click to put a check mark next to Developer in the right-hand list.
  • On the Developer tab, near the right end, click the Document Template button.
  • In the resulting dialog box, make sure the "Automatically update document styles" box is NOT checked.
  • Click OK and then save.

Here's what's happening. When you send a document to a client who doesn't have access to the template on your network, their copy of Word automatically attaches their Normal.dotm (not .dot) template to the document. If the "Automatically update" box is checked, all the styles in the document that have the same names as styles in their Normal template will be redefined. You don't want that to happen.

graham mayor say:
You have a section break after the first page. Each section can have a different set of page numbers. Double click in the second page footer (the one with the second page 1) and from the Header and Footer Tools context sensitive tab, click the Page Number dropdown then select Format Page numbers and check 'continue from previous section' or set the start number to whatever you wish.
