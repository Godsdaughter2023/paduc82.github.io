jllatham say:

A question or two to get a better picture of what you think your workbook is going to look like.

 

You say you have 20 cards and you mentioned "spreadsheets" - does this mean you will have one sheet in a workbook for each of the 20 cards?  I think this would be a good way to do it from the standpoint of being able to easily see the status of any of the cards, and makes it easier to add more cards later if required.

 

You mention 3 fare levels, I presume that the actual amounts of these changes based on the to-from points of travel involved?

 

---

I think that your plan to use a single letter or number to represent the start/finish points will save you some work, and it is probably going to push you toward using a VLOOKUP() formula.  It not only makes that possible, but it makes it very manageable should rates change or more routes are added: you just change the content of the lookup table but the formulas using VLOOKUP() don't have to be altered.

 

Let us say you had a table something like this on a sheet by itself, and the sheet is named RouteInfo :

     A        B            C            D                E                  F

1  ID     Start      Finish    On Peak    Off Peak     Tap Fine

2   A       Pt1         Pt2         2                 1                  3

3   B       Pt1          Pt3        2.5               1.25             3.5

 

On one of the card's sheets you enter your ID into column A, then in column B you put the date and in column C you have the start time of the trip.   In column D you want to show the starting point, so you could have a VLOOKUP() formula like this (for row 2)

=VLOOKUP(A2, RouteInfo!A:F, 2, False)

What that does is take the entry in cell A2 and compare it to the entries in the first column of the lookup table, and when it finds a match it returns the value from the 2nd column of the table on that row.  The "False" parameter, in effect, says that your entries in the first column of the table do not have to be sorted for it to find the proper match.

 

You could even set it up to tell you if you entered a route ID that doesn't exist:

=IF(ISNA(VLOOKUP(A2, RouteInfo!A:F, 2, False)),"Invalid Route ID",VLOOKUP(A2, RouteInfo!A:F, 2, False))

 

You can still use a variation of the VLOOKUP() to pick out the proper fare rate from the table using a Choose or IF() formula to determine which column to return the fare rate from, something like this:

=VLOOKUP(A2, RouteInfo!A:F, CHOOSE(MATCH(A2,{"A","B","C"},0),4,5,6) , False)

I have used CHOOSE(MATCH(A2,{"A","B","C"},0),4,5,6) as just an example to show that a formula can be used to pick which fare column should be used.  You may have another column or two in use as "helper" columns to determine if the fare should be "on peak", "off peak" or "Tap fine" rate, and use the value of that to make the choice.

 

Remember that the column number to return information from is "relative" to the columns in the table and doesn't have anything to do with column numbers for columns on the sheet.  To make this clearer:  If your table used columns A through F, then a ,2, reference would refer to column B (the second column of the table), but if your table was in columns R through W, then ",2," would refer to column S on the sheet (still the 2nd column of the table).

 

I hope this helps with your understanding of VLOOKUP() should you need to use it for your solution.  Tables and VLOOKUP() are very powerfull tools that can really make things easier for you, especially when lists (the content of the table) get very long.

 

A nice side effect of using a table like that is that you could use the entries in the first column of the table as the data source for data validation on the individual card sheets to help make sure that people only make valid route ID entries on those sheets.

jllatham say:

First, in the MATCH() portion: MATCH(A2, {"A","B","C"},0)

The curley braces {} indicate that what is inside of them is an array, or group, of information which causes whatever is in A2 to be compared to each of them, one at a time to see if there is a match.  The ,0 tells the MATCH() function that we want an exact match, not just something close.

 

The 4, 5, 6 is part of the CHOOSE() function.  The Match() will return 1, 2 or 3 which indicates which of the choices to pick: 1st (4), 2nd (5) or 3rd (6).  That is used as the offset into the VLOOKUP() table to determine which cost to pick up.

 

But even though you cannot bring another workbook into work, perhaps you can download this one at your home and look at it.  It uses several "helper columns" to determine what choices to make.  Those helper columns are out to the right side of the main work area on the sheet(s) and they can even be hidden from view if desired.  There's one sheet with the lookup table to set up for routings and costs, and it has a 'table' to hold a list of the Public Holidays that must be taken into consideration.  Those tables and pieces of them are referenced by 'name' in the formulas on the 2 example costs sheets I have in the workbook.  My email address is on that tables sheet, so if you would like to work with something like that, then you have a way to get in touch with me.

jay freedman say:

As to the other question, about the separation of groups with the same surname (or, in general, the first-level entries in any index), the source is not the font but the capitalization of the words in the XE fields created by the Mark Entry dialog. The Index field sees {XE Wright} as a different entry than {XE WRIGHT}.

To fix this (assuming you want all the names to be initial-cap only), turn on nonprinting characters by clicking the ¶ button and use the Find dialog to visit each all-caps name in an XE field and press Shift+F3 twice (the first press makes it all-lower-case). When all the names are cased the same, update the index field.

If you want to use a macro to fix the whole index with one click, install this and run it:

Sub FixIndexCaps()
   Dim rg As Range
   Set rg = ActiveDocument.Range
   With rg.Find
      .Text = "^19 XE "
      .Forward = True
      .Wrap = wdFindStop
      While .Execute
         rg.Case = wdTitleWord
         rg.Collapse wdCollapseEnd
      Wend
   End With
   ActiveDocument.Indexes(1).Update
End Sub

Carefully check the result for fragments that should not be capitalized at all, such as "van" or "de", and correct those by manually changing their XE fields.

suzanne s. barnhill say:

Hi, neighbor!

It is incorrect to say you "have never used Styles." In Word, you cannot avoid styles. If you never apply any other style than Normal (the default body style), then most of your document will be in Normal style with direct formatting. This is not only a burden on Word (resulting in larger file sizes) but also a very inefficient way to operate. If you don't use specific styles for headings, for example, you can't generate an automatic TOC, and you gain many advantages by using the built-in heading styles (modified as desired).

But even if you don't take advantage of styles for the main part of your document, Word will, by default in some cases and unavoidably in others, use specific styles for specific document elements. The index is one such case, but Word also uses built-in styles for headers, footers, footnotes, endnotes, footnote and endnote reference marks, TOCs, captions, and so on. You need to be aware of this; not knowing it can cause problems.

As to the capitalization of the index entries, this reflects the formatting of the XE (index entry) field, not the actual name in the document. When a document contains index entries, which are formatted as Hidden, Hidden text is displayed by default, so you are probably aware of these fields, and Jay has given you a macro to make them consistent.
