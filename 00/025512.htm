The “400 Bad Request error” is an HTTP status code that occurs when Internet Explorer is able to connect to the web server, but the webpage cannot be found because of a problem with the web address (URL). This error message often happens because the website address is typed incorrectly.

 

I’d suggest you to check for errors in the URL. The most common reason for a 400 Bad Request error is because the URL was typed wrong or the link that was clicked on points to a URL with some kind of mistake in it.

 

You may try the following steps and check:

Step 1:

Turn off friendly HTTP error messages

a. Open Internet Explorer by clicking the Start button, and then clicking Internet Explorer.

b. Click the Tools button, and then click Internet Options.

c. Click the Advanced tab, and then scroll down to the Browsing section.

d. Clear the Show friendly HTTP error messages check box, and then click OK.

e. Click ok to apply the changes and restart the browser to check if the issue persists.

 

You may refer the following link for more information:

Get help with website error messages (HTTP errors)

http://windows.microsoft.com/en-US/windows7/Get-help-with-website-error-messages-HTTP-errors

 

Step 2:

If the issue still persists, uncheck HTTP settings

a. Open Internet Explorer by clicking the Start button, and then clicking Internet Explorer.

b. Click the Tools button, and then click Internet Options.

c. Click the Advanced tab, and then scroll down to the HTTP 1.1 settings

d. Uncheck "Use HTTP 1.1

e. Click ok to apply the changes and restart the browser.

f. Check if the issue persists.

suzanne s. barnhill say:

Your screen shot shows that the page width (actual paper size) is 8.5", but if you look at the Margins tab of Page Setup, you'll probably see that the left and right margins total 1". This means that the document body area is 7.5" wide, and that's what the ruler shows. (The default margins are 1" all around, which allows a text width of 6.5", so your margins must have been changed.) Your text will be confined to the 7.5" width unless (a) you reduce the page margins or (b) you add negative indents to selected paragraphs, which will allow them to extend into the margins.

Similarly, inline images (those with a text wrapping style of In Line With Text) will be constrained to the margin area unless you apply negative indents. If you change the wrapping style, you can drag the images into the left and right margins; in all versions except Word 2013 (which does not permit it), they can also extend into the top and bottom margins.

steve rindsberg say:

Audio and video files use special software called CODECs to compress the files; there are many different CODECs.  If your audio/computer uses a CODEC that isn't installed on the other PC, it won't be able to play the audio.

Try this on your computer:

Choose FILE | INFO

Click "Optimize Compatibility"

Save the file under a new name and send that to the other user(s) who have trouble with your file.  With luck, the audio will now play for them.
