jay freedman say:

The fundamental problem here is that checkboxes are a poor substitute for option buttons (a.k.a. radio buttons) because they don't automatically enforce their own exclusivity. Furthermore, an On Exit macro executes only when the user actually exits the control; if the value of the checkbox changes but the user then just saves and closes the document, the On Exit action never occurs. While you can beat this problem into submission with additional code, it will always be fragile.

A much better solution is to create userform, in which you can insert real option buttons. Each option button has a property called GroupName, and all the buttons with the same value of GroupName automatically become exclusive.

Further, controls in a userform don't have to rely on entry and exit macros. The code behind the userform can contain functions that fire immediately when the value of a control changes. As a simple example, consider a userform with six option buttons. The first three all have the GroupName set to "green" and the last three have it set to "red".

Write the following code in the userform.

Private Sub OptionButton1_Change()
   OptionButton4.Value = OptionButton1.Value
End Sub

Private Sub OptionButton2_Change()
   OptionButton5.Value = OptionButton2.Value
End Sub

Private Sub OptionButton3_Change()
   OptionButton6.Value = OptionButton3.Value
End Sub


When the form is displayed, clicking the first option button in the green group (OptionButton1) causes that one and the first one in the red group (OptionButton4) to become selected, and all others to be deselected.

If you change your mind and click OptionButton3, then OptionButton6 will also be selected, and buttons 1 and 4 are automatically deselected.

For information about transferring the results of the user's choices back to the body of the document (which you do in the CommandButton1_Click macro), look at http://gregmaxey.mvps.org/word_tip_pages/create_employ_userform.html.

This type of prompting for firewall access will not typically occur when the application is simply requesting outbound access to the Internet, so I'd want to be more certain of what is actually being set up by the application.

I note from this VLC Setup Guide – How to Set up VLC for Media Streaming, Remote Control document that it's possible to allow video or audio streaming or remote control between networked computers.  Since this would require opening a port inbound to the PC in order for other computers on the network to stream media files from it, this might be the reason for this firewall prompt and warning.

As the firewall prompt is indicating, this action wouldn't be recommended on Public (e.g. airport, hotel or other open networks), since these are typically insecure and you'd be unlikely to use these features while connected to these anyway.

On the other hand, if your home or work networks are secure and you might be interested in streaming between individual computers on these, then enabling these abilities there might make sense.  However, only you know whether you have other devices you might wish to stream to or from, so the decision is really yours as to whether this ability is useful or should be blocked in all cases.

From your responses above; you only use this single HP PC and no other devices and you apparently also put that PC in Private Network mode.

If this is true, then this is less secure than it could be, since the only reason to use the Private Network setting is if you will be sharing the PC on the local network for access by other devices.

I also contributed to that thread with Geoffrey: http://answers.microsoft.com/en-us/office/forum/officeversion_other-office_other/lines-appear-around-photos-when-published-to-the/f5634250-b3e6-4003-a5a7-8503faa53aa6

 

As I read your question, I am not sure your lines are the same. As Geoffrey pointed out, we were talking about lines that were created by Publisher when web pages were generated, not printed pages.

 

Are your pictures edge to edge or full coverage of the page? Are the lines appearing when you print the images? Could it be a limitation of the printable area of your printer...the printer cannot print 'full bleed'? 

 

Are you creating transparent PNG images and the transparent parts of the image showing? Did you try to use the transparency tool in Publisher?

 

Are the lines straight edge lines?

 

How did you create your images? Did you use a third party image editor to join the three images?

 

Please explain in more detail what you are doing or maybe post a copy of the image somewhere so we can see it.

 

Did you consider or try adding a thin border of the same color as the background to the image? How about creating a vertical line of the same color as the background and nudging it to the edge of the image, bring forward, so it screens/covers the lines you are seeing?

 

I have also seen lines around objects that have been cropped out of the background of the original image such as a face that ended up with an irregular line around the hair, that showed up when that cropped image is pasted on a contrasting background. This is because the image editor is not able to exactly follow the hair lines and some of the orginal background still shows. Two workarounds for this is to change the background to the color of the line so that the line blended in, or to edit the image with the image editor and 'paint over' the line in a color that matches the hair, or the background color that you want to use. If your lines are a part of your combined image then this might be the best approach for you. Open those images in your image editor, zoom in and 'paint' that line a color that matches the image or the background you are using, or change the background to match the lines.

 

DavidF
