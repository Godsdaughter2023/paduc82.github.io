mbstr8k say:

Mail Merge into shapes vs. letters or labels

Is it possible to merge data into a shape, such as a star?  If so, what process would I use?  I would like to print a sheet(s) of stars with different data in each one.  The data is in an excel spreadsheet right now.

a. user say:

It depends on exactly what you mean.

In essence, merge is probably always going to merge into a rectangular area. So, if you need to merge onto (e.g.) label stationery with stars, it might be difficult to make the text fit the star.

If you just want to pick an autoshape and merge into that, then

 a. again, a rectangular piece of text will likely overlap the star (unless it's a big star!)

 b. you can select the shape, click Add text, and then you can insert text and field codes as in normal text. You may find it easier to add field codes by using ctrl-F9 to insert the special "{}", then insert everything else in the usual way

 c. I'd do that for one copy of the shape, then duplicate it as many times as you need.

 d. however, you then have the problem that these shapes are in the graphics layer and the merge fields generally take the same value (i.e. you cannot get { NEXT } fields to work). I think the only reasonably simple workaround for that is to organise that your data source has enough data for the entire page in each row. So for example, if you have 8 stars per page, and you want 1 field in each star, then you would need an eight-column sheet with the following columns (for example):

star1data, star2data,...,star8data

So you would then need to modify the field code in each star so that the first has

{ MERGEFIELD star1data }

the second has

{ MERGEFIELD star2data }

 and so on.

Peter Jamieson

If you have Excel 2007 or 2010, you could use this formula:

=SUMIF(E:E,"Y",C:C)

If you are using Excel 2003 or earlier, you must specify the rows to include and of course they should at least cover the rows with your data in them, such as:

=SUMIF(E2:E500,"Y",C2:C500)

This is also more efficient in 2007/2010, but might need attention from time to time if your data goes down beyond the last row in the formula (500).

NOTE: in your post, first you said column E had the Y/N entries, later you said F, so I have used E in my example, change to F:F or F2:F500 if it really is column F.
