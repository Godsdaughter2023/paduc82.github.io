The behavioral monitoring components have been included or being added to Defender and MSE over the last few years and the telemetry they provide, including possible malware samples have been collected during that time as well.

The page I referenced above shows quite clearly that the Advanced Threat Protection (ATP) is simply a localized storage and analysis capability for enterprises, allowing them to more directly collect and analyze their own attack situation using the data from their own organization's endpoints as well as the analysis aid of Microsoft and its partners via their threat intelligence knowledge.

All of this has been developed using the existing infrastructure that's supporting every Windows client running a supported and enabled version of a current Microsoft security product.  All that ATP is doing is segmenting the local organization's portion of this telemetry data to allow an enterprise customer a better view into its attack profile including additional monitoring points such as AppLocker and Device Guard.

For consumers, this really changes nothing since the segmented analysis requires additional server and other resources to allow for the data collection and analysis on a local basis by the enterprise.  Microsoft performs all of this currently within their cloud systems, collecting this data anonymously from those systems connected to MAPS and evaluating it in an aggregate manner in order to detect, monitor and analyze the profiles of any new attacks, whether network or file based across various regions and the world.

This is why I've always stated that no one else has a monitoring and analysis capability that's anything close to Microsoft's, since this system collects information from every machine that runs the monthly Malicious Software Removal tool, Microsoft Security Scanner, Microsoft Security Essentials, Windows Defender (any version) and any of the other commercial security products using the Microsoft engines and definitions.  Nothing else comes close to the mass data collection and analysis capability of this huge cloud based system.

All that ATP is really doing is exposing the system that already exists to the world a little more clearly, since anyone who's been involved with the MVP Security programs or similar access has known about these systems since the time of the original Windows Defender, OneCare and the acquisition of GIANT Company Software Inc., including their GIANT AntiSpyware product and SpyNet, upon which all of the Microsoft security products and MAPS are based.

ken sheridan say:

Here's an example of a simple model:



And here's a query to return each person's antecedents down to their great-grandparents:

SELECT
FamilyMembers.FamilyMemberID AS ChildID,
[FamilyMembers].[FirstName] & " " & [FamilyMembers].[LastName] AS Child,

    FamilyMembers.LastName AS ChildLastName,
    FamilyMembers.FirstName AS ChildFirstName,
    FamilyMembers_1.FamilyMemberID AS ParentID,
[FamilyMembers_1].[FirstName] & " " & [FamilyMembers_1].[lastName] AS Parent,
    FamilyMembers_1.LastName AS ParentLastName,
    FamilyMembers_2.FamilyMemberID AS GrandParentID,
[FamilyMembers_2].[FirstName] & " " & [FamilyMembers_2].[LastName] AS GrandParent,
    FamilyMembers_2.LastName AS GrandParentLastName,
    FamilyMembers_3.FamilyMemberID AS GreatGrandParentID,
[FamilyMembers_3].[FirstName] & " " & [FamilyMembers_3].[LastName] AS GreatGrandParent,
    FamilyMembers_3.LastName AS GreatGrandParentLastname
FROM FamilyMembers LEFT JOIN (((((Parentage
LEFT JOIN FamilyMembers AS FamilyMembers_1
ON Parentage.ParentID = FamilyMembers_1.FamilyMemberID)
LEFT JOIN Parentage AS Parentage_1
ON FamilyMembers_1.FamilyMemberID = Parentage_1.ChildID)
LEFT JOIN FamilyMembers AS FamilyMembers_2
ON Parentage_1.ParentID = FamilyMembers_2.FamilyMemberID)
LEFT JOIN Parentage AS Parentage_2
ON FamilyMembers_2.FamilyMemberID = Parentage_2.ChildID)
LEFT JOIN FamilyMembers AS FamilyMembers_3
ON Parentage_2.ParentID = FamilyMembers_3.FamilyMemberID) ON FamilyMembers.FamilyMemberID = Parentage.ChildID;

A report base on the above is very easy to build, merely requiring three group levels, child, parent and grandparent.  Great-grandparents are returned in the detail section.  I should point out that I produced this as a demonstration of the use of an adjacency list table (Parentage in this case), not with any serious intention of genealogical modelling.
