If the model number is RM-356 then you have the global version, which won't work in the US because its 3G frequency bands are 900/2100 MHz. In the US, they use 850/1900MHz for that.

stefan blom say:
I'm creating a new thread for your question, since it is unrelated to the rest of this thread.

pamcaswell say:

CHGN,

Table styles should work for you.  There are just a few things you must know before you try them--more later.   This site has the instructions for getting started:  http://www.tech-recipes.com/rx/45623/word-modify-and-create-custom-table-style/ . 

Play around with it to see if you can get what you want.  Write back if you get stuck or have questions. 

The instructions don't mention that the formatting that you apply to elements (like header row, total row, first column and such) of the table style can be turned off or on on an individual table.  Use the check boxes on the Table Tools > Design tab >Table Style Options to do that.

Also, to get heading rows to repeat when a table lasts past a page, select the top or top few rows and go to Table Tools > Format tab and click Repeat header row. 

Now for he things you should know:

  1. Table styles work best in a new theme-ready document.  By theme ready I mean when the normal paragraph style itself has not been modified either in the document or its template (usually normal.dotm) and when normal carries the settings of the defaults (& so they match).  In W2007 and later, the best way to change fonts, line spacing, etc., of the main text style (often normal) of a document is to change the defaults. The defaults can easily be changed through the Change Styles button on the Home tab in W2007 and -10 or the Design tab of W2013 and -16.
  2. If the document is not theme ready, table style styles can still be made to work reliably by going to Styles pane > Manage Styles tab > Set Defaults and changing the settings there to whatever is set in the normal style.
  3. Even if you never use normal or any other built-in style, you need to  be sure that  normal and the defaults match.

I know these three things seem like a lot to have to bother with.  But if you work often with longer documents, knowing these things can pay off in fewer pasting errors, tables that behave as expected, and more reliable fonts in long documents.

abi99 (2) say:

Android requires not the Bluetooth Dial-up Networking profile (DUN) but the Bluetooth Personal-Area Networking profile (PAN). With Nokia, neither Symbian, MeeGo/Maemo, nor Windows Phone but only Series 40 offer this. Consequently, you use the correct operating system on your Nokia. Unfortunately, Dual-SIM variants of Series 40 do not offer PAN. Those only offer DUN.

 

If your phone would work, the steps would be:

  1. Nokia Series 40 » Menu » Settings » Connectivity » Bluetooth » Phone visibility » Temporarily visible
  2. Android » Menu » Settings » (All ») Bluetooth » (Search for devices ») your Nokia Series » Settings » Use for Internet access

If the Bluetooth icon gets visible and you get the notification on your Nokia, your Android would be connect, however, the connection drops: You have to create a preferred access-point in Nokia » Menu » Settings » Configuration » (Options ») Add new » Access point » Access point settings » Bearer settings » Mobile data access point » here, you enter the APN for Internet of your network provider.

 

Again, no help, because your Nokia 515 Dual SIM does not support PAN. tecy, you used PAN with your old Nokia, didn’t you? Which one was it?
