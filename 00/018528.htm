Found out what's causing this issue on IE and Edge in our environment.  There was a policy that we implemented to prevent the use of weak MD5 and 3DES encryption algorithms using IISCryptoCli.exe (https://www.nartac.com/Downloads/IISCrypto/IISCryptoCli.exe) we remove the policy applied on the effected computer, then ran "IISCryptoCli.exe /template default", reboot the computer and IE and Edge can now access the site we were having problem accessing. 

If you're having the same issue, make sure that you don't have any policy that prevent the use of weak MD5 and 3DES encryption algorithms, download the https://www.nartac.com/Downloads/IISCrypto/IISCryptoCli.exe, then run "IISCryptoCli.exe /template default" as an admin in command prompt and then reboot your computer, you can maybe run the whole command with the reboot "IISCryptoCli.exe /template default /reboot".  Hope this help.

michael bauer (vboffice.net) say:

This small macro asks for every outgoing email whether or not to change the sender address. Setting the SentOnBehalfOfName property works only with Exchange, and if you have the SentAs permission for that address:

Private Sub Application_ItemSend(ByVal Item As Object, Cancel As Boolean)
  If MsgBox("Change sender?", vbYesNo) = vbYes Then
    Item.SentOnBehalfOfName = "*** Email address is removed for privacy ***"
  End If
End Sub

If the prompt on every ItemSend is annoying, you could add some more logic to decide automatically if the address should be changed. Moving the sent email to a folder in another mailbox, and changing the signature is more tricky. This Addin (SAM) supports all that based on rules that you can create easily.
