SELECT qry_Ownership_Final_Data.LEGAL_ENTITY_ID, qry_Ownership_Final_Data.LE_COUNTRY, qry_Ownership_Final_Data.LE_TAX_TYPE, qry_Ownership_Final_Data.PARENT_LE_ID, qry_Ownership_Final_Data.PARENT_COUNTRY, qry_Ownership_Final_Data.PARENT_TAX_TYPE, qry_Ownership_Final_Data.OWNER_PERCENT AS OWNER_PERCENT, [Forms]![frm_Main]![txt_StockTrackDate] AS SPOT_DATE

marshall barton say:

Using a Formatting function on the text box value converts it to text that just looks like a date.  You could use the query field's Format property instead.  If the query is used as a form/report's record source, you could use the Format property of the text box bound to the field.

OTOH, sometimes Access can not determine the desired data type of the query field without some kind of clue.  Possibly, just specifying the form text box's Format property will do it.  Or using a date function in the query:

   SPOT_DATE: DateAdd("d", 0, [Forms]![frm_Main]![txt_StockTrackDate])

should leave the query field as a Data/Time data type instead of converting it to Text.
