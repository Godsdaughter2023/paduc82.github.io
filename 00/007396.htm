marshall barton say:

That is a seriously complex problem and you will have to do a lot of homework to put all that together.  The first thing is to learn all you can about queries and how to put together criteria expressions.  Next. you will need a form with text boxes where you can enter the criteria values.  For example, searching in either ID field can be done by entering an ID value into the text box named txtID on the form named frmSearch and then running a query that looks something like:
   SELECT * FROM yourtable
   WHERE ID1 = Forms!frmSearch.txtID OR ID2 = Forms!frmSearch.txtID

Simiilarly, you can use other text boxes on the form to search for city and country.  Take a look athttp://allenbrowne.com/ser-62.html for an example of how to create and use a form to create the query using optional criteria.

The question of using a fuzzy match for the name can get as complex as you are willing to go.  You can start out by becoming familiar with using wildcard patterns in the criteria.  For example,
  WHERE NegativeName Like "*[OU]sam?*"
will find Osama. Usame, Osamo, etc anywhere in the name field.

I think the next step beyond that is to find/create a function that converts a name into some kind of canonical form that you can store in your table and use in the search.  A popular one is called SoundX and has several variants that you can learn about by searching the internet.  An example of using this kind of thing would be:
    WHERE SoundX(Forms!txtNegName) = NegativeNameSoundX

I'm unclear on whether you want to:
  1. Sum column C, substituting the corresponding values from column B if there is an x in column C or,
  2. Sum the rows in column B when there is an x in the corresponding row from column C

If you are looking for the first then try,
