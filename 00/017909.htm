I do the following for OneNote 2010 on Windows 7.

If you are afraid if anything bad happen to any data, then do *not* do the following.

BackUp… BackUp… BackUp… Your Data.

 

==============================================

 

You can make a new unfiled notebook and unfiled section for various pages you create. Later, you can move the pages to other notebooks for better organizing.

 

CREATING A NEW UNFILED NOTEBOOK AND SECTION

 

OneNote's Menu > File > New > ...

1. Store Notebook On : click "My Computer"

2. Name: (write the name) "MyUnfiledNotebook"

3. Location: (select the folder you want. Your Unfiled Notebook will be saved as the sub-folder of the folder you select now.)

- Write the title for that section: "MyUnfiledSection"

 

Now, you have a notebook: "MyUnfiledNotebook" and it has a section: "MyUnfiledSection"

 

 

SETTING THE NOTEBOOK's SECTION AS THE DEFAULT UNFILED SECTION

 

Now, you should tell OneNote to use the above notebook and its section for unfiled pages.

 

OneNote's Menu > File > Options > Save & Backup.

As long as you use "MyUnfiledNotebook" and its section "MyUnfiledSection" as your unfiled notebook and for saving your various pages, then *never* close it.

It's a shame though, that there appear to be fonts on Mac (and they are TT fonts) with the correct glyphs AFAICS, but which simply don't appear to work in Word however you use them.

AFAICS the Unicode approach on this is a bit weird, in that
 a. the code points from 0x30 to 0x39 are supposed to be for the "European" digits
 b. there are markers to allow selection of a "National" glyph set, but these markers are deprecated:

"Numeric Shape Selectors. The numeric shape selector format characters allow the selection of the shapes in which the digits U+0030..U+0039 are to be rendered. These format
characters do not nest.
From the point of encountering a U+206E national digit shapes format character up to
a subsequent U+206F  nominal digit shapes (if any), the European digits (U+0030..
U+0039) should be depicted using the appropriate national digit shapes as specified by
means of appropriate agreements. For example, they could be displayed with shapes such
as the arabic-indic digits (U+0660..U+0669). The actual character shapes (glyphs) used
to display national digit shapes are not specified by the Unicode Standard.
From the point of encountering a U+206F nominal digit shapes format character up to
a subsequent U+206E  national digit shapes (if any), the European digits (U+0030..
U+0039) should be depicted using glyphs that represent the nominal digit shapes shown in
the code tables for these digits. This state (nominal) is the default state in the absence of any
numeric shape selector or a higher-level protocol."

So it isn't completely clear from that how anyone is supposed to write a piece of text that will be unambiguously displayed by any piece of software unless they always use precisely the correct Unicode code points for the appropriate national digits (including "European"). 

AFAICS some of the necessary code point substitution stuff may be built into the OpenType (and perhaps TrueType) standards, but even then, in Word, it seems to be difficult to impose the correct language characteristics on any given piece of text - typing using the correct keyboard seems to be the only sure-fire option and that's only on the Windows version.

lemp say:
Having multiple versions of Office at the same time is not recommended, although it will work.  It's my recollection that when I did that, the version of Office that was not the most recently used one would go through an "install" process when you started it (unless you make the registry changes described in the KB article below).

Here's Microsoft's advice for Office 2007:  http://support.microsoft.com/kb/928091

If you want Word 2003 to open when you double-click a *.doc file, first determine where winword.exe (the executable for Word) is located for Word 2003.  If I recall correctly, that should be c:\Program Files\Microsoft Office\Office11.  Then find a *.doc file, right-click and select Open with > choose Program.  Browse to the Word 2003 executable, select it, and check the box to "Always use this program to open this type of file."  Click OK.

You can use the same process for Excel.  The executable file for Excel 2003 should be C:\Program Files\Microsoft Office\Office11\Excel.exe

Remember that Office 2007 defaults to using *.docx and *.xlsx files (the newer XML-based format) rather than the older *.doc and *.xls formats.

For more advice, ask the Office experts in the Office 2007 forum.
