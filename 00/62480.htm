steve winograd say:

A computer can be controlled locally (at the keyboard) or remotely (through Remote Desktop), but not both at the same time.  If one user is connected and another user connects, the first user will be automatically disconnected.

Using Remote Desktop over the Internet requires some advanced setup.  You'd have to:

  • Sign up with a dynamic DNS service to give a URL to your Internet modem or router.
  • Assign a static IP address on the local network to the computer that you want to access.
  • Create a rule in your modem or router's firewall to pass Remote Desktop traffic to the computer.

A much simpler solution is to use a remote access program like TeamViewer , which is free for personal, non-commercial use.  It lets both users connect at the same time, and it requires no setup beyond installing the TeamViewer program.

I use the paid version of TeamViewer to give remote support to my consulting clients.

Hi John,
 

Thank you for replying with the status of the issue, sorry to know the USB stopped working issue is still not resolved, I appreciate your efforts to resolve this issue.

I suggest you to follow the steps below and check if it helps.

1. Press Windows Key + X key on the key board.

2. Select Device manager.

3. When the Device Manager window opens, you have to seek the USB device with which you’re facing Unknown device issues.
4. Right click over this entry, pick Properties.

5. In the device properties window shown below, switch to Details tab. Now click the drop-down menu for Property and select Device Instance Path.

Note down the corresponding Value because we will need this value in further steps. By the way, this value iscombination of three IDs; namely Vendor ID (VID), Product ID (PID), Instance ID.

6. Press Windows Key + R combination, type put regedit in Run dialog box and hit Enter to open the Registry Editor.
7. Navigate to the following location:
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Enum\USB\<Device Instance Path>\Device Parameters
Substitute the Device Instance Path part (after USB\) obtained from the step 3.
8. In the right pane of Device Parameters key look for the DWORD named EnhancedPowerManagementEnabled which must be showing its Value data as 1. Double click on the same.

9. In the above shown box, change the Value data to 0. Click OK. You may now close the Registry Editor as well as Device Manager.

10. Now check if the USB ports works, for best result restart the computer and check.
