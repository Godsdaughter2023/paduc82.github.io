dirk goldgar say: The combo box has a List Rows property, on the Format tab of its property sheet.  Is that what you're looking for? 

preschool mike say:

Not sure, does it change the height of the row list in the combo box?

dirk goldgar say:

Did you try it, or look up the property in the online help?  It controls the number of rows that will be shown in the list when it drops down -- not the total number of rows in the combo, but the number of rows that will be shown at one time.

From the help:

ComboBox.ListRows Property (Access)
You can use the ListRows property to set the maximum number of rows to display in the list box portion of a combo box. Read/write Integer.
 Syntax

expression. ListRows

expression A variable that represents a ComboBox object.

Remarks

The ListRows property holds an integer that indicates the maximum number of rows to display. The default setting is 16. The setting for the ListRows property must be from 1 to 255.

preschool mike say:

Guess I'm not being clear.  I'm not talking about the quantity of rows seen, but the actual height of those rows.  Adjusting the height of the combo box does not increase the height of the rows in the list.  I need, if possible, to increase the height of the row list in the combo box so I can seem all the data.  Each row contains about one or two sentences and I'm only able to see a portion of the row.  This would be similar to what the List Width does, but I need a List Height - is this possible, even programmatically?

dhookom say: There is no word wrap in combo box items. You can set the column widths to display larger strings. 

dirk goldgar say:

Ah, I see -- you're talking about what I would call "Row Height" -- but there's no such property.  The height of the rows is automatically set according to the font height of the control, so that the text fits.  However, the text will never wrap -- it will always be rendered on one line.  You can make that line wider by setting the List Width property, but you can't get the text to wrap within the line.

If you need something like that, you might consider creating your own popup, continuous form to display the rows, with code to capture the user's selection and set the value on your original form.  That's a bit of coding, but not impractical.

Problem: If a rule is setup where the action is to forward to multiple addresses, the rule engine appears to show discrete addresses but upon being triggered it amalgamates the multiple addresses into one text string separated by semi-colons but no spaces.
