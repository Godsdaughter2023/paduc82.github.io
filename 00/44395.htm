Most phones will appear as a 'drive' on your computer, and you can browse the files on them simply by opening them.

g)    Click in the folder window opened.

a. user say:

 Hi,
 
Thank you for posting in Microsoft community. I am glad to assist you.
  
Follow the steps below and check if it helps:
 

Method 1:
I suggest you to run the troubleshooter for Printer and check if it helps.

Refer the link below to run the troubleshooter:
http://windows.microsoft.com/en-us/windows-10/fix-printer-problems

If the documents have got stuck in the queue, you can turn off the print spooler service, delete the pending documents and then restart the service again to print the documents.

Method 2:
To delete the documents, you have to stop the Print Spooler service (software that temporarily stores print jobs on the PC's hard disk).
You can stop print spooler service and then delete all the pending documents.
In this ways you start printing the documents again.
Follow the below steps:
• Press Windows key + R on the keyboard, and type Services.msc and press Enter.
• Click on Print Spooler service, right click, and select Stop to stop the Print Spooler Service if it is running.
• Check whether the Startup type is set to Automatic. If not, then set it to Automatic.
• Go to the following location and delete all the pending documents from the Printer.
C:\WINDOWS\system32\spool\PRINTERS
• Delete all the queued print jobs (select and delete all the files) in the folder so the folder is devoid of pending print jobs. The folder should be empty when you are done.
• Restart your system and check the Print Spooler Service again to be sure the Print Spooler Service restarts automatically on reboot (since the Startup type is set to Automatic).
• After rebooting, check the Print Spooler service again and the Status column for the Print Spooler should read Started.  

Hope this helps. Get back to us if you need further assistance.

You'll probably be stuck having to get a parallel to USB adapter cable.  This is a well known problem that started with the 1511 release of Windows 10.  Microsoft basically broke parallel port functionality with the update and has yet to fix it.  I've also tested it on various machines and can safely verify that the 1511 update breaks parallel printing every time.    It's a huge deal for me because the business uses multiple parallel port receipt printers.  At $300+ to replace, this is a huge deal.

A temporary fix, which may only work until the next reboot, or until the next Windows update is to go into the Device Manager and then go to the properties of the LPT Port.  Then click the "Port Settings" tab and put a check mark in "Enable legacy Plug and Play detection".
