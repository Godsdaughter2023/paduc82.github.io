jllatham say:

Ok, the use of ActiveWorkbook and ActiveSheet and some other workbook/sheet references leads me to this question:  Is all of this in a single workbook?  That is, does all of the code only work with the "ActiveWorkbook" - or perhaps that Sub we haven't seen is doing something with another workbook?

With the assumption that you are going to say that everything is dealing with just one workbook, and that all of the code is in that one workbook, then I would change all references to ActiveWorkbook to become ThisWorkbook.

ActiveWorkbook refers to whatever workbook happens to be active in the same instance of Excel when you have more than one workbook open.  With only one workbook open, it is the same as ThisWorkbook.

But ThisWorkbook is very explicit: it is saying to use the workbook that contains the code being executed.  And it takes any question of how many workbooks are involved in this out of the equation.

-------

Along with that I want to revisit your assignment statement:

sh_name = ActiveWorkbook.Sheets(1).Name

I understand why you're doing that - it's to be used later in building the formulas.  And since you don't allow for another workbook name in those formulas I'm assuming that this statement is referring to Sheets(1) in the same workbook with the code.

A problem arises when referencing Sheets/Worksheets (Worksheets are a subset of Sheets which can also contain other types of sheets such as chart sheets) by position index:  what happens to your code if a user moves some of the sheets around in the workbook???

A safer way to make such a reference to a worksheet in the same workbook with the code is to use the worksheet's 'code name'.  If you look at the VBA Project window at the Excel objects listed in it you see that each sheet in the workbook shows two names, like:

Sheet1 (Sheet1)

The Sheet1 entry is the code name for the sheet, the (Sheet1) entry is showing you the name of that sheet as it appears on the sheet's name tab.  Now a user can obviously easily change the name as it appears on the name tab.  But you can only change the code name for a worksheet in the VBA Editor.  So if you were to change that statement to:

sh_name = Sheet1.Name

(assuming its code name is Sheet1) then even if the user changes the sheet's name as displayed on its name tab, or if they move it around within the workbook later, the code continues to function as you intended it to.
