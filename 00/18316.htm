carolejeanx say: Automation errors usually occur when a workbook is trying to run some macro that contains objects they are not included in the references section of the VBA editor. One example that would produce this is having some controls on a form which require a specific DLL to be correctly registered on a client PC. If these DLLs are on your pc, the workbook is fine. Put this same workbook on another PC without the DLLs, and an automation error will occur.

brooken say:

Hi Frank,

Thank you for the report. We have identified the problem and are working on a fix now.

This will affect data sources in Excel Workbook (.xlsx) format.

A workaround for this for the time being would be to save a copy of your Excel Workbook file in "Excel 97-2003 Workbook" format (.xls) and use that as the data source. That should not hit the problem. To do this, load your .xlsx file in Excel, then choose "Save As..." from the File menu, and change the file format to "Excel 97-2003 Workbook", and save a copy. You can then use that file when you select recipients in Word.
