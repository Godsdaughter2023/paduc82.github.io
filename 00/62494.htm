Shameful limitation: File Open dialog cannot return more than 1000 - Microsoft Community

zexx say:

Shameful limitation: File Open dialog cannot return more than 1000 files

This is a limitation in Windows Common Dialogs. It was discovered in an image processing application, but it can be reproduced in any application that uses Windows File Open Dialog to open multiple files:

1. In the Windows Open File Dialog select 2500 or more files using the Shift-click method.

2. Click the Open button.

 

Result:

The dialog reports an error, or retrieves a smaller number of files than selected.

 

Cause:

Windows File Open Dialog retrieves file names in a buffer that is up to 65536 bytes long. This is an old 16-bit limitation, however it still exists today. If the total sum of file name lengths exceeds 65536 bytes, the dialog will display an error.

This limitation exist in Windows 3.1 (which was a 16-bit operating system). However, Windows 95 was a 32-bit OS, so there was no reason for the limitation anymore. Nevertheless, this limitation exist in Windows 95, Windows 98, Windows NT, Windows 2000, Windows Millenium, Windows XP, Windows Server 2003, Windows Vista, Windows 7 and Windows Server 2008.

 

That's right, 64-bit Windows 7 still has this ridiculous 16-bit limitation.

 

So, does Microsoft plant to fix this in one of Windows 7 service packs?

Any excuses for not doing it in any of the past 18 years?
