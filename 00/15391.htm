Click Start, Run. Type regedit.exe and press {ENTER}

Navigate to the following branch:

HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows \ CurrentVersion \ Explorer \ MyComputer \ NameSpace

Right-click on the subkey {416651E4-9C3C-11D9-8BDE-F66BAD1E3F3A} and choose Export

Type nokiaphonebrowser.reg as the file name, and click OK to backup the branch

Exit the Registry Editor.

karl timmermans [outlook mvp 2012-2018] say:

The <Private> field is a member of the <Sensitivity> group or to put it another way, <Private> is nothing more than a separate view of one of the <Sensitivity> settings - the intrinsic values of which are

 

0 - Normal

1 = Personal

2 = Private

3 = Confidental

 

While the value for >Sensitivity> is displayed in the descriptive form, it is stored as a numeric value as shown above. Change the Sensitivity value to anything other than <Private> and the <private field> will be shown as <No>

 

Importing to the sensitivity field using the appropriate numeric value OR the appropriate descriptive name as shown above will resolve your problem. Importing a value to <both> the Private and Sensitivity fields will yield unpredictable results depending on which field is processed first and what values you're using for both of those fields (in essence, importing to both fields is importing the same info twice).
