ericbergan say:

You are having a more broad problem than I had, It's a bit of a long shot, but perhaps what I found might help you.

For several builds the UWP apps (store, mail, news, etc.) would not run on my system. If you tried to run them, basically nothing would happen - no window would open, etc. The tile would turn briefly black, and then return to normal.

The problem was specific to a user login. If I created another user and logged in, it could start the apps.

I eventually found a hint that c:\<user>\appdata\local\Microsoft\windows\notifications might be the issue. This turns out to be a cache database that is used by the UWP apps for posting notifications. Mine had become corrupted. The apps would see the corrupted

Given the other user-centric issues you are seeing with other MS tools, you might have additional corruption or deletions from c:\<user>\appdata\local\Microsoft\windows? There are lots of folders for various Microsoft tools in there, presumably storing the user's configuration and history for those apps. If you have a backup of that folder from before the problems started, and there are significant differences from what you see in the current folder, I would "very" carefully (i.e. a folder at a time, backing up the current in case you need to roll back) see if restoring pieces helps. BTW, to do that, you will probably need to be logged in as a different user, since many of those files are locked when you are logged in as that user.

Again, just offering this as a possible hint. I was certainly surprised that this actually fixed my problem.

Hi Aimee,

As I understand based on your description, the user who has been removed from the distribution group(DG) can still receive the emails sent to this DG.

Given this situation, I’d like to confirm how long you removed the user. If you delete a member from a group, the system takes about 24-48 hours to update.

Meanwhile, you can also run the following PowerShell command as an admin role to get the DG list:

Connect to Exchange Online PowerShell

Example

Get-DistributionGroupMember -Identity "Technical Support"

This example returns the existing distribution group members for the distribution group named Technical Support.

Check whether the user was actually removed from the DG list. If not, you can refer to the following steps to remove him:

Log into Exchange Admin Center > recipients > groups > double click on the distribution group > membership > remove this user > save

Or you can also run the following PowerShell command to remove users from DG:

Example

Remove-DistributionGroupMember -Identity "Technical Support" -Member "Jan Dryml"

This example removes Jan Dryml from the distribution group named Technical Support.

If the issue persists, I suggest you try to run a message trace and check if the message was forwarded to the user. Message trace shows what actions were taken on the message before it reached its final status.
