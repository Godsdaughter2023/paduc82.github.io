scottgem say:

Your Assets table is incorrect. The Checked out  fields should be ONLY in the Transaction Log. You should not be storing data in multiple tables. 

You Assets table should look like this:

Assets

AssetID (PK Autonumber (note do't accept the default name of ID, give it a meaningful name, usually the table name + ID)

SerialNumber (don't use spaces in object names, use either camel notation (as shown) or underscores I.e. Serial_Number)

CalibrationValidity

Description

CategoryID (FK)

LocationID (FK)

I left off quantity since you should have one record for each item. I lkeft out the dates because they belong in the Transaction Log.

So your Transaction Log should look like:

TransactionLog 

TransactionID (PK Autonumber)

AssetID (FK)

CheckOutBy (FK)

CheckOutDate

CheckInDate

to determine whether an asset is checked out you do a query on the TransactionLog for all transactions with a null CheckInDate. So there is no need to update the Asset table at all. that is just redundant data. 

A couple of more tips. Name is a reserved word in Access and shouldn't be used as an object name. Plus person names should be broken down into their components; at least first and last. 

The Combobox wizard can create a search combo to allow you to select an Asset and move the pointer to that record. You then have subform bound to the Transaction log where you select the employee from a combobox and fill in the date(s).

ken sheridan say:

How do I link the two asset details fields in such a way that one the user looks up the asset either by the serial or by the description, the other field automatically updates? 

Not fields, controls.  The  field is the column to which the control is bound.  In this case both combo box controls should be bound to the AssetID column.  That to select by serial number would be set up as follows:

ControlSource:    AssetID

RowSource:     SELECT AssetID, SerialNumber FROM Assets ORDER BY SerialNumber;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

That to select by description would be set up as follows:

ControlSource:    AssetID

RowSource:     SELECT AssetID, Description FROM Assets ORDER BY Description;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

Once you select a value in one the other will automatically show the corresponding value.

Sorry, but nothing obvious comes to mind, especially without being able to see it.

Mark on track is just a shortcut way of letting MSP fill in the actual start and actual duration, by forcing it to assume that the task started as scheduled and was in continuous progress from the actual start to the status date.
