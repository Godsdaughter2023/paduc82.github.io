lemp say:
Although it's entirely possible that your Ethernet network adapter got fried by the lightning as well as the modem, that should not affect a separate wireless adapter in the desktop (your "Wireless 8 PCI Adaptor").

It sounds as if you've got the new router configured to use WEP encryption (which these days is hardly better than no encryption).  Unless you have very old hardware, you should be using WPA2 (or at least WPA) encryption. 

Is your XP installation (you posted in an XP forum) up to date with service pack 3 and subsequent security updates?

If for some reason you must use WEP (e.g., you have some obsolescent hardware connected to your wireless network), you need to look in the router's configuration utility and get the actual hexadecimal key rather than any "password" that you may have entered in the router.  Often, routers will "generate" a hex key from a user password ... but Windows will probably not know how to get from that "password" to the hex key. 

But you really should not use WEP.

For more detailed help, either look in the User Guide for your router or post the router's complete make/model, including any version, here.

EDIT

On re-reading your question, I see that you write that the router "has a 16 number key," which doesn't sound like WEP after all.  On the other hand, "computer is asking for a 10 or 26 character key" IS WEP.  Please post the complete text of whatever message leads you to believe that you should be entering a 10 or 26 character key.  The problem may well be that your installation if XP is not up to date.  Please right click on My Computer, select Properties, and report which version of Windows you have, including service pack.

andre for directly say:
Press Windows Key+X
Click Windows Powershell (Admin)

Enter the following command then hit Enter

Get-AppxPackage -allusers Microsoft.SkypeApp | Remove-AppxPackage

Open start menu
Type in "powershell"
Right click on "Windows PowerShell"
Run as Administrator

Then paste the following command

get-appxpackage -AllUsers Microsoft.SkypeApp | foreach { Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml" }

Press Enter

After the install process finishes, you have the Windows 10 Skype app back
