Hi,

There are two group commands, Get-Group which would return all Exchange groups and Get-MsolGroup which would return all groups within your AD.

Using the portal is the equivalent of running "Get-MsolGroup", which would show all available security groups.

Using  Get-Group (or Get-DistributionGroup , for that matter), would return all mail enabled groups.

If you're using DirSync\AD-Connect, Please make sure that your groups are populated with Exchange attributes (Email, ProxyAddresses, TargetAddress, Alias, etc.) then run a sync in order to mail enable your groups on Office 365.

If you're using Office365 as standalone, you need to use "New-DistributionGroup -Type Security" in order to create mail enabled distribution groups.

Hi WesleyDe,

Before moving on, I’d like to clarify some information.

In Office 365 Exchange Online, to use a security group (SG), the SG must be mail-enabled.

There are 2 official ways to create mail-enabled SG in Office 365:

1. Directly create SG in Office 365 Exchange Admin Center -> recipients -> groups. By default, SG created here is automatically mail-enabled.

2. To sync on-premise SG to Exchange Online, the officially supported way is: Create a mail-enabled group from on-premise Exchange (or mail-enable a normal AD SG from on-premise Exchange), and then sync this group to Exchange Online. alison wang msft say:

Hi Sheila,

Thank you for your update.

It is likely that one of the add-ins is causing the performance issue. To troubleshoot possible problems that are caused by COM add-ins or Exchange Server client extensions, disable any items that you do not use regularly. Then, restart Outlook. If that does not resolve the problem, disable the remaining add-ins one by one. If that does not resolve the problem, the issue may not be caused by an add-in.

To disable COM add-ins, follow these steps: 
1. In Outlook, click the File tab.
2. Click Options, and then click Add-ins to display a list of the COM add-ins that are installed.
3. Select COM Add-ins in the Manage list, and then click Go.
4. In the COM Add-Ins dialog box, click to clear the check boxes for any COM add-ins that you want to disable.
5. Click OK, and then restart Outlook.
