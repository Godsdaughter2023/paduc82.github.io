andreas killer say:

Option Explicit

Sub ShiftData()
  Dim Last As Range, All As Range
 
  'Find the last cell in the sheet
  Set Last = Cells.SpecialCells(xlCellTypeLastCell)
  'Map to column J (Saturday J10 in this sample)
  Set Last = Range("J" & Last.Row)
  'Refer to all data
  Set All = Range("E2", Last)
  'Refer to the data rows next column after All (K2:K10 in this sample)
  Set Last = All.Resize(, 1).Offset(, All.Columns.Count)
  'Move the 1st column to the right end
  All.Columns(1).Cut
  Last.Insert xlShiftToRight
  'Done
  Application.CutCopyMode = False
End Sub

gina whipp say:

ken sheridan say: This is not a matter of querying the subform, but of restricting its recordset.  Each subform should be based on a query which restricts the rows returned to those for the service in question.  You say that the Service column is a 'lookup field', so its value will not be that which you see, but a hidden numeric foreign key referencing the key of a referenced services table.  To restrict the result table of the query on service, therefore, the referencing table should be joined to the referenced table on the keys and restricted on the non-key text Service column from the referenced table, not on the foreign key column in the referencing table.  The non-key text Service column should not be returned as a column by the query, however, i.e. in query design view the 'show' check box of the column should be unchecked.

BTW most experienced Access developers recommend that the lookup field wizard should be avoided like the plague.  For reasons why see:

http://www.mvps.org/access/lookupfields.htm
