As mentioned in the article below, you should avoid using “#” in any fields names within an Access database since it is a reserved symbol.  Because of this, I would expect to see behavior similar to what you are describing.  I also tested this on my end and was able to reproduce the issue.  I did find if I checked the box to “Export data with formatting” it will work.  If you would prefer to not have the formatting, then you most likely will need to come up with another way to change the field name in your CSV file.
