Section 2 - admin area, includes export subform to excel functionality

ken sheridan say: You need to change the query's WHERE clause to:

WHERE  ((StartDate > Forms!frmOvertimeAdmin!srchStartDate
OR Forms!frmOvertimeAdmin!srchStartDate IS NULL)
AND (EndDate < Forms!frmOvertimeAdmin!srchEndDate
OR Forms!frmOvertimeAdmin!srchEndDate IS NULL))

If the query is restricted on any other columns you'll also need to include these criteria in the WHERE clause of course.  Note that the positions of the parentheses in the above are crucial to force the expressions to evaluate in the correct priority.

Make these amendments in SQL view, not design view, and be sure to save the query only in SQL view.  If you do so in design view Access will move things around.  At best the logic will be obscured, at worst the query might be too complex to open.

BTW, one thing we've assumed so far is that the start/end dates must be later/earlier than the parameter start/end dates.  If, on the other hand, you want rows returned where the range might start on the parameter start date or finish on the parameter end date then change the operators to >= and <= respectively.

suzanne s. barnhill say:

When you enable "Different odd and even" headers and footers, the setting is for the entire document, and Word will force the correct kind of section break if you are restarting numbering in a new section. That is, if you have inserted an Even Page Break and want to start the new section with 1, Word will change the break to Odd.

Whenever you need to change the section break type, it is better not to try to replace an existing break. Instead, go to the Layout tab of Page Setup and change the "Section start" setting to the desired type.

Where is the Office Upload Center? - Microsoft Community

bittsel say:

Where is the Office Upload Center?

I have a continual "Inbox: uploading..." problem that is using up system resources.  I saw that I can clear the document cache in Upload Center - would that help and if so, how do I find the Upload Center?
dianeoforegon say: It sounds like you are using Exchange. If you have not upgraded to 14.2.1, I suggest you upgrade then start over in a new Identity. Before you download into Outlook, log into your account with your browser and clean up your Inbox.

If you have IMAP, Outlook is not as picky as Exchange sync, but if you have thousands of messages in your Inbox, move the older ones to an archive folder.

The only data that would need to be moved to a new Identity would be under "On My Computer" messages, contacts, calendar events, notes and tasks.


The Inbox syncs more frequently than any other and should be kept as clean as possible. (this is the number one cause of problems with sync). Do NOT make subfolders under the Inbox.

Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.
