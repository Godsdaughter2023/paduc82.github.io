scottgem say:

Probably the reason why you are having trouble is you are not clear on what you are doing. Field width can be misleading. For one a "database" in Access terms is a collection of objects including tables, forms and reports. So its not clear whether you want to change the size in a table or the displayed width on form or report.


There is actually no field width property in Access. A field has a SIZE property which can be easily changed in Table Design mode. However, the size can only be changed for text datatypes. With other datatypes, it is either fixed (like date/time) or is changed by choosing a subtype (like number).


Each control on a form or report has a width property which can also be changed in the Property dialog in Design mode. This property is on the Format tab.

scottgem say:
Again you are using the wrong terms which is causing your problem. 

"I opened the file in "Design" view". You can't open a "file" in Design view. An Access file is, as noted, a container for a number of objects. What it sounded like you did was open a FORM in Design View. You then selected a CONTROL (not a field) and looked at the Width property. This property controls the width of the control on the form, but does not affect the number of characters it can hold. 

What you need to do is open the TABLE in Design mode. And select the field. Under Field Properties at the bottom, in the General Tab is a Field Size property. For Text Datatypes, this will be a number from 1 to 255 which will be the number of characters the field can store. 

Additionally, you can also limit the number of characters using a validation rule.

Any easy way to copy tab settings to paragraphs with mixed fonts? - Microsoft Community

mike 1430 say:

Any easy way to copy tab settings to paragraphs with mixed fonts?

In my 800 page technical opus, I have hundreds of lines of text that have both arial & TNR fonts.

(Seemed line a good idea at the time, and it DOES LOOK GOOD.)

Now that I (sort of) understand tabs, I would like to get one line the way I want it and copy the tab settings to other lines.

I would MUCH RATHER USE A STYLE, but I believe there is no such thing as a style that doesn't change font characteristics.

My only solution would be to write down the settings, then record a macro to first clear all tabs, then set new ones.

Any better suggestions?

suzanne s. barnhill say:

You should be able to just select all the text (Ctrl+A), then open the Tabs dialog, clear existing tab stops, and set new ones--no macro needed.

If you're wanting change only selected lines, then you may be able to use Copy Format and Paste Format. These commands work sort of like the Format Painter but more precisely. If you select text and press Ctrl+Shift+C, all of the properties of the selected text (including the font) will be copied, but if you just click in a paragraph and press the shortcut key, you'll copy only paragraph formatting; you can then use Ctrl+Shift+V to paste the formatting to other paragraphs. Again, you can't select the text; instead, click at the end of the paragraph (not within a word) and paste. You will then be able to repeat the process by clicking at the end of another paragraph and pressing F4. For this a macro probably would be more efficient.

Your inclination to use styles is well-founded. Why not define two styles, one in each font, with the desired tab settings, and then apply them as required? You can assign a keyboard shortcut to each style to speed up the process, or use the Quick Styles buttons on the Home tab to apply them.
