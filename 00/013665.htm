jllatham say:

Here's my modification of Per Jessen's code modified some more to deal with the 3 things you needed:
1) Remove all passwords - the code to protect the sheets is still there, just commented out for now
2) Removes ", " entries when they are created for empty apartments
3) Sorts the Resident List-Alpha tab by resident name
Code assumes that data starts on row 4 of both sheets (that there is a header/label row at row 3 on both)

I've set up a list of Const (constant) values at the start of the routine that you can change if needed in the future if your sheet names change or if the layout of the sheets changes.  All you have to do is change the value of the appropriate Const entries to keep the code working properly.

Private Sub Worksheet_Activate()
'place in the [Resident List-Alpha] worksheet's code module
'change these Constants to adapt the code to your
'worksheet's names/layouts - this way you can
'alter it easily in the future
  Const pWord = "JustMe"
 
  Const shAName = "Resident Information"
  'this assumes a header/label row at row 3
  Const FirstRowShA = 3 ' header/label row on shA
  Const shAAptNumCol = "B"
  Const shALNameCol = "F"
  Const shAFNameCol = "G"
 
  Const shBName = "Resident List-Alpha"
  'this also assumes a header/label row at row 3
  'with actual data entries to begin at row 4
  Const FirstRowShB = "4" ' first DATA row on shB
  Const shBAptNumCol = "A"
  Const shBNamesCol = "B"
  'these are needed for the sort by name feature
  'need the first & last column IDs that contain
  'any data at all, for now I've assumed only
  'columns A and B (Apt# and full names) are on the sheet
  Const shBFirstUsedCol = "A"
  Const shBLastUsedCol = "B"
'*** End of user definable Const list ***

  Dim shA As Worksheet
  Dim shB As Worksheet
  Dim InformRng As Range
  Dim DestCell As Range
  Dim ApartNumList As Range
  Dim cell As Range
  Dim f As Range
  'these will hold offset from Apt# col to Last & First names on shA
  Dim off2Lname As Integer
  Dim off2Fname As Integer
  'used in the sorting
  Dim sortRange As Range
  Dim sortKey As Range
  Dim lastRowToSort As Long

'now the work actually begins
  Set shA = Worksheets(shAName)
  Set shB = Worksheets(shBName)
  'insure that there is data to work with on the [Resident Information] sheet
  If shA.Range(shAAptNumCol & Rows.Count).End(xlUp).Row <= FirstRowShA Then
    Set shA = Nothing
    Set shB = Nothing
    Exit Sub ' no data to work with
  End If
  Application.ScreenUpdating = False
  off2Lname = Range(shALNameCol & 1).Column - Range(shAAptNumCol & 1).Column ' F-A = 4
  off2Fname = Range(shAFNameCol & 1).Column - Range(shAAptNumCol & 1).Column ' G-A = 5
  Set InformRng = shA.Range(shAAptNumCol & FirstRowShA, _
   shA.Range(shAAptNumCol & FirstRowShA).End(xlDown))
  Set DestCell = shB.Range(shBAptNumCol & FirstRowShB)
  shA.Unprotect pWord
  shB.Unprotect pWord
  InformRng.AdvancedFilter xlFilterInPlace, Unique:=True
  InformRng.Offset(1).SpecialCells(xlCellTypeVisible).Copy DestCell
  shA.ShowAllData
'remove the ' from next line when you want to start protecting
'the [Resident Information] sheet again
  'shA.Protect pWord
  Set ApartNumList = shB.Range(shBAptNumCol & FirstRowShB, _
   shB.Range(shBAptNumCol & FirstRowShB).End(xlDown)) 'Range variable
  Set DestCell = DestCell.Offset(0, 1)
  For Each cell In ApartNumList
    Set f = InformRng.Find(what:=cell.Value, _
     after:=shA.Range(shAAptNumCol & FirstRowShB - 1), lookat:=xlWhole)
    If f.Value = f.Offset(1).Value Then
      If f.Offset(0, off2Lname).Value = f.Offset(1, off2Lname).Value Then
          DestCell = f.Offset(0, off2Lname).Value & ", " _
           & f.Offset(0, off2Fname) & " & " & f.Offset(1, off2Fname)
      Else
          DestCell = f.Offset(0, off2Lname).Value & ", " _
           & f.Offset(0, off2Fname) & " & " & f.Offset(1, off2Lname) & ", " & f.Offset(1, off2Fname)
      End If
    Else
      DestCell = f.Offset(0, off2Lname).Value & ", " & f.Offset(0, off2Fname)
    End If
    'remove entry if it is only ","
    If DestCell = ", " Then
      DestCell.ClearContents
    End If
    Set DestCell = DestCell.Offset(1)
  Next
  'sort the list by resident names
  lastRowToSort = shB.Range(shBAptNumCol & Rows.Count).End(xlUp).Row
  If lastRowToSort < FirstRowShB Then
    lastRowToSort = FirstRowShB
  End If
  Set sortKey = shB.Range(shBNamesCol & FirstRowShB)
  Set sortRange = shB.Range(shBFirstUsedCol & FirstRowShB & ":" _
   & shBLastUsedCol & lastRowToSort)
  sortRange.Sort Key1:=sortKey, Order1:=xlAscending, Header:=xlNo, _
      OrderCustom:=1, MatchCase:=False, Orientation:=xlTopToBottom, _
      DataOption1:=xlSortNormal
'remove the ' from next line when you want to start protecting
'the [Resident List-Alpha] sheet again
  'shB.Protect pWord
 
  'housekeeping/cleanup
  Set ApartNumList = Nothing
  Set InformRng = Nothing
  Set DestCell = Nothing
  Set sortKey = Nothing
  Set sortRange = Nothing
  Set shA = Nothing
  Set shB = Nothing
End Sub

per jessen say:

In my code, I assumed that you have headings in row 1 and data starting in row 2. From the formula you posted, I guess that your data start in row 4 in both sheets? I have modified the code a, with variables for first row in each sheet, so you can change them as required.
The code will overwrite your formula with generated data. I have changed the code, so it will only fire if a row has data in column B, F and G.

ApartNumList is a range variable used to hold the address for all apartments in Residen List-Alpha sheet.

Here's the updated code:

Private Sub Worksheet_Change(ByVal Target As Range)
Dim shA As Worksheet
Dim shB As Worksheet
Dim InformRng As Range
Dim DestCell As Range
Dim ApartNumList As Range
Dim pWord As String
Dim FirstRowShA As Long
Dim FirstRowShB As Long

pWord = "JustMe"
Set shA = Worksheets("Resident Information")
Set shB = Worksheets("Resident List-Alpha")
Set isect = Intersect(Union(Range("B1:B" & Rows.Count), Range("F1:G" & Rows.Count)), Target)
If Not isect Is Nothing Then
    FirstRowShA = 3 ' Header row just above first data row
    FirstRowShB = 4 ' First row to populate with data
    If Not WorksheetFunction.CountA(Range("B" & Target.Row), Range("F" & Target.Row), Range("G" & Target.Row)) = 3 Then Exit Sub
    Set InformRng = shA.Range("B" & FirstRowShA, shA.Range("B" & FirstRowShA).End(xlDown))
    Set DestCell = shB.Range("A" & FirstRowShB)
    shA.Unprotect pWord
    shB.Unprotect pWord
    InformRng.AdvancedFilter xlFilterInPlace, Unique:=True
    InformRng.Offset(1).SpecialCells(xlCellTypeVisible).Copy DestCell
    shA.ShowAllData
    shA.Protect pWord
    Set ApartNumList = shB.Range("A" & FirstRowShB, shB.Range("A" & FirstRowShB).End(xlDown)) 'Range variable
    Set DestCell = DestCell.Offset(0, 1)
    For Each cell In ApartNumList
        Set f = InformRng.Find(what:=cell.Value, after:=shA.Range("B" & FirstRowShB - 1), lookat:=xlWhole)
        If f.Value = f.Offset(1).Value Then
            If f.Offset(0, 4).Value = f.Offset(1, 4).Value Then
                DestCell = f.Offset(0, 4).Value & ", " & f.Offset(0, 5) & " & " & f.Offset(1, 5)
            Else
                DestCell = f.Offset(0, 4).Value & ", " & f.Offset(0, 5) & " & " & f.Offset(1, 4) & ", " & f.Offset(1, 5)
            End If
        Else
            DestCell = f.Offset(0, 4).Value & ", " & f.Offset(0, 5)
        End If
        Set DestCell = DestCell.Offset(1)
    Next
    shB.Protect pWord
End If
End Sub

albert d. kallal say:

You can do two things:

 

First ensure that your install does NOT include the runtime. Then  when you create a update, make sure you modify/change the GUID’s that are created when you make the install package. (so when using the package wizard blank out the product code and update code. (bottom of last wizard panel).

 

I believe if you just blank out the upgrade code and then tab out, and then tab back in, you get a NEW code. (but try both, I recall I had to change the product code to eliminate the un-install issue).

 

The result is that you don’t actually ever un-install the previous database, but are always overwriting and updating to a new version.

 

It is A VERY VERY (but very^12 times) good think that you cannot with ease un-install the previous version – since it would be FAR TOO easy to wipe out and delete the data file on the customers computer! So be VERY thankful this is not easy to do!

 

At the end of the day, I not really should I would use or bother with the installer to issue upgrades. However the “trick” overall here is to NOT include the runtime with your front end, and then by modifying the Product Code in the package wizard, you “can” coax the system to overwrite your existing database.

 

At the end of the day, updates to your data base is really ONLY a file copy. I thus have some reservations about using the package wizard to achieve what amounts to a simple file copy.

 

I also provide a sample inno script and explain how I update my applications in the following article:

 

Deploying updates to your software in a Runtime environment for Access

http://www.kallal.ca/RunTime/Index.html

 

So if you deploy without runtime, and ALSO change the product code, then you can "get around" the un-install requirement.

graham mayor say:
The following macro will offer the option to format all cross references in lower case. It will intercept the built-in command to allow you to select the item you wish to cross reference. The formatting will occur when you lose the dialog.

Sub InsertCrossReference()
Dim oRng As Range
Dim bLower As Boolean
With Dialogs(wdDialogInsertCrossReference)
    If MsgBox("Format reference in lower case?", vbYesNo) = vbYes Then
        bLower = True
    End If
    .Show
End With
If bLower Then
    Set oRng = Selection.Range
    oRng.MoveStart wdWord, -1
    oRng.MoveEnd wdWord, 1
    'oRng.Select
    With oRng.Fields(1)
        .Code.Text = .Code.Text & " \*lower"
        .Update
    End With
End If
End Sub

greg maxey say:

There are many occassions when the macro recorder simply doesn't work.  Steps involving dialog boxes is one of them.

 

Graham has posted some code that meets your objective for formatting the individual fields as you create them.  Or you an continue as you have leaving the formatting to the end and use something like:

Sub ScratchMacro()
Dim oFld As Word.Field
For Each oFld In ActiveDocument.Fields
  'Is it a REF field?
  If oFld.Type = wdFieldRef Then
    'Is is a crossreference field?
    If InStr(oFld.Code, "_Ref") > 1 Then
      oFld.Code.Text = oFld.Code.Text & "\*lower"
      oFld.Update
    End If
  End If
Next oFld
End Sub

 

This of course would be applied to "all" crossreference fields in the main text story of the document which may or may not be desirable.
