When I use Firefox, I'm not facing the same problem.

victor noguedad say: First, is it on every page or in certain pages?

Probably what has happened is this: The site you're trying to access has configured apache to tell IE not to cache pages. This means that the contentmenu in the admin interface sends a large burst of HTTP requests. The ISA firewall has a feature to limit the number of HTTP requests per second that it passes through to try to limit worm and virus activity, but you are hitting this limit.

In ISA it is set in the General node underneath Configuration. There is a selection on the right called Define Connection Limits. By default it is set low like  600. It is possible to either increase the limit for all users, or create an exception list so specific machines.

Also, if youre accesing inside a corporate network the issue might be in the servers providing internet access. Check with your administrator.
