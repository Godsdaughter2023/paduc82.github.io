hippman257 say:

Windows 10 reverts to Windows 8 start menu during OS deployment

Anyone attempt deploying the Windows 10 technical preview via OS deployment instead of manual install?

I have seen that the Start Menu reverts back to the Windows 8.1 style of Start menu.  I found a workaround via the HKCU registry settings and also discovered a few other discrepancies when comparing a "clean" install versus a custom deployment.

Clean install via USB media vs. deployment via install.wim using setup and unattend.xml:

 

Only on clean install:

 

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer]

"GlobalAssocChangedCounter"=dword:00000001

"GridPaneRowCount"=dword:00000004  (added automatically to custom once "EnableStartMenu" value is created.

 

Only on custom install:

[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer]

"AllowStartMenuToDefaultOn"=dword:00000001

 

Differences:

[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer]

"TelemetrySalt"=dword:00000000 (clean)

"TelemetrySalt"=dword:00000002 (custom)

 

"DisplayToastAtBottom"=dword:00000000 (clean)

"DisplayToastAtBottom"=dword:00000001 (custom)

 

 

 

[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced]

"EnableStartMenu"=dword:00000001 (clean)  - This is the setting that enables the new Windows 10 start menu.

"EnableStartMenu"=dword:00000000 (custom)

When you are learning to create macros, most of them don't work. However, if you can re-open your MyMacros.dot file and run the macro while the template is open, Word will often give you some indication of what went wrong.

If you created this file very recently, try looking in your Recent Files list and re-opening the template from there. If it's not there, then open the Word Startup folder to look for it.

A couple of handy techniques to test your macro include inserting a temporary MsgBox command followed by a macro variable in your code. When you run the macro, the message box will pop up and display the variable value, telling you whether the variable is getting the right information.

It's also helpful to click in the left margin of the VBA code window so that a brown bar is displayed across one line of code. When you run the macro, the code will halt at that line. You can run the macro one line at a time by pressing the F8 key. Floating your mouse over the variables in your code will reveal their value at each step, so you can see where a step goes wrong.

In addition, you can post your macro code with a description of its intended operation in the Programming section of this forum and skilled programmers will help you.
