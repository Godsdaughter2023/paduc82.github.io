john visio mvp say:

The creator may have requested that the group shape be protected.

You should be careful when ungrouping. When a collections of shapes are grouped, a new shape is created to control the group and the creator can add text and shape data to this new shape. the subshapes could be mosdified to use attributes from this group shape. If you ungroup, this "grouped" shape is deleted along with the extra information and the links to the group shape.

The normal way is to NOT ungroup, but edit the subshapes directly. If you click on a group, you select the group, if you click again, you select the sub shape beneath where you clicked.

So be careful ungrouping. If it is a non Visio shape, then you should be able to ungroup with no worries.

doug robbins - mvp office apps & services (word) say:

The code to remove any type of protection from a document is:

ActiveDocument.Unprotect

It is not necessary to specify the type of protection and it is that, which is causing Word to not recognise the command.  If a password was used when applying the protection, it would be necessary to supply the password by using:

If ActiveDocument.ProtectionType <> wdNoProtection Then
    ActiveDocument.Unprotect Password:=strPassword
End If

The If...Then...End If construction is optional, but will prevent an error from occurring if the document is not protected.
