Rounding Time Totals to nearest 1/4 hour - Microsoft Community

premierdmp say:

Rounding Time Totals to nearest 1/4 hour

I am tracking "check-in" times, "check-out" times which give me a column with accurate "time onsite" in hours, represented  in decimal form such as 2.16, 3.67, 1.93, etc. My need is to have a formula that will convert these values (in a separate column) to the nearest 15 minute increment, i.e. - 2.0, 2.25. 2.5, or 2.75 for accounting/ invoicing, based on the specific criteria below. (Is this possible?)

If value is less than 2.0 hrs then it should convert it to 2.00 (as that is minimum)

If the decimal value is between x.01 and x.12 it reverts to x.00 (where "x" is the whole number)

If the decimal value is between x.13 and x.37 it should be x.25

If the decimal value is between x.38 and x.62 it should be x.50

If the decimal value is between x.63 and x.87 it should be x.75

If the decimal value is x.88 or above, it should round up to next whole number, (x+1.00)

jllatham say:

Given that it's a dynamic range, how would you decide where to start/stop?  Would you have a rule "starting 20 rows from the beginning, select the next 20 rows" or something like that.

 

But the basic statement in VBA is pretty simple, there are a couple of ways:

 

Select the entire rows

Range("20:40").Select  ' but that gets the whole row (20 to 40) not limited to any columns

 

In code you'd probably have a couple of constants to identify the columns and a couple of variables to deal with the rows (or maybe the other way around, I don't know - you said the range was dynamic, but didn't say what could expand/shrink).

Const startCol= "C"
Const endCol = "M"

Dim startRow As Long
Dim endRow as Long
you do some stuff somehow to assign values to startRow and endRow, I will simply assign values:

startRow = 20
endRow = 40
Range(startCol & startRow & ":" & endCol & endRow).Select

 

then do what you need to with that range, copy it, delete it, whatever.

But you can actually work with it directly without selecting it, let's say you want to copy it to cell A1 on another sheet:

 

Range(startCol & startRow & ":" & endCol & endRow).Copy Worksheets("Sheet2").Range("A1")

 

would do that.

How can I get a cell to only accept a value that is a multiple of 5 - Microsoft Community

mysticd say:

How can I get a cell to only accept a value that is a multiple of 5 (ex: 0, 5, 10, 15, 20, 25; but not 2, 3, 7, 14, 22, etc.)

I actually figured it out and just wanted to share.

Under data validation, set it to custom and type in the formula.

=A1=MROUND(A1,5)

 - where A1 is the cell that you want to only have the multiple of show.

 - 5 is the multiplication you want to see, can change to 2, 10, 100, 25, 50, etc.
