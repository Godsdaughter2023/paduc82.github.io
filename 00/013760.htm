The contents of a text box on a userform are text, even if the user enters a number, so you can use Len. You could also use

or, to take into account that the user might have entered one or more spaces:

jllatham say:

Text Boxes don't really distinguish what's in them.  The numeric entry is actuall a text entry of a string of numeric characters.  I created a simple userform with a text box and a command button.  The code for the button:

Private Sub CommandButton1_Click()
  If Len(Me!TextBox1.Text) = 0 Then
    MsgBox "Box is empty"
  End If
  If IsNumeric(Me!TextBox1.Text) Then
    MsgBox "Entry starts with numbers."
  Else
    MsgBox "Entry is NOT numeric."
  End If
End Sub

If you type "abc" or even "abc1234" into the text box, the "Entry is NOT numeric." will appear.  If there is nothing in the box, the "Box is empty" message appears.

For entries like 13414148 or even 134ABC or 123xyz987, it would interpret them as numeric.  You'd have to test each character individually to see if they are all numeric.  Here's modified code for my button that does that type of thing:

Private Sub CommandButton1_Click()
  Dim LC As Integer ' Loop counter
  Dim testText As String
 
  If Len(Me.TextBox1.Text) = 0 Then
    MsgBox "Box is empty"
    Exit Sub
  End If
  testText = Me.TextBox1.Text
  For LC = 1 To Len(testText)
    If Not IsNumeric(Mid(testText, LC, 1)) Then
      MsgBox "Entry is not entirely numeric."
      Exit Sub
    End If
  Next
  MsgBox "Entry is numeric."
End Sub

jllatham say:

Assuming 1 date is in column A and the other date is in column B and entries start at row 2.

In the column where you want OVERDUE to appear, but this into cell on row 2

=IF(OR($A2<TODAY(),$B2<TODAY()),"OVERDUE","")

That should take care of the auto-text part of things.  Now for the formatting. 

Select all cells on the rows concerned from column A to last column you want highlighted.  This should begin in our sample at cell A2.

Select Conditional Formatting, New Rule, and "Use formula..."

For the formula enter this, which can actually be copied out of the formula above

=OR($A2<TODAY(),$B2<TODAY())

and set up the format you want.

If it doesn't seem to work, double check the rule to make sure Excel did NOT change it to:

="OR($A2<TODAY(),$B2<TODAY())"

bernard liengme say:

This formula will work only if the workbook has a cell named x and another named y.

Try this:

In A1 enter the text x , in B1 the value 4

In A2 enter the text y , in B2 the value 6

Select A1:B2; on the Formulas tab, use Create from Selection (see snip)

Now IF(x=y, "yes", "no") will work

I think i understood what is going on, you are currently using smart quotes:

You need to use the block double quote character "
