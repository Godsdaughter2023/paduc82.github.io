I see you mentioned installing your Nvidia Graphics drivers by booting into Safe Mode.  I've also had to use Safe Mode

after the 1709 update with my desktop Nvidia GTX980. I my case the machine does not crash, the driver install just fails

Once this hurdle is over, games all run as well as they did on earlier Win10 incarnations ( which is about as good as on Win7 )

- With Win10 it's generally necessary to uninstall the drivers completely by using Display Driver Uninstaller or Driver Fusion.

Run one of these uninstaller tools before booting into Safe Mode to better assure Windows does not install another driver version

by stealth when you reboot (I don't know if Windows does this in Safe Mode, but it generally does when rebooting into Windows normally)

- You can tell if Windows Updates has stuffed around with your Nvidia graphics driver during a big version update because 

Nvidia Graphics Driver will no longer be listed in Programs and Features after the update, or if it is still listed, it may show an older version number than what you were using prior to the version update.

- Laptop users.

The best driver has always been the one provided by the mfg of the machine, not Nvidia (Nvidia also mention this on their site).

The machine mfg's may want to lay the blame on Win10, but it's up to them to continue driver support for their hardware, and traditionally

laptop mfg's fail at this after the first couple of years.

I would be surprised that ASUS would not have a driver released to cover the 1709 update for a machine with the current model gpu though.

If they don't = slack.

- You may also find that ASUS has an automated driver update tool on their site that will work well with your machine.

- Nvidia do a pretty good job of driver support for desktop cards, and I've never had an Nvidia (or AMD) driver that is flawed,

but the laptop mfg's modify how the Nvidia gpu gets used, and this can make the  generic Nvidia driver less than optimal when

used with the laptops configuration at best, and can cause BSOD at worst.

- Not applicable to the a machine with a GTX1050, but ....

I've seem may people posting on these forums with laptops and desktops that didn't have proper WIn10 drivers from day one,

and while their machines may have been able to run Win10, many don't get as good gaming performance as they would have if they

had stayed with Win7 and suitable drivers.  As time goes by issues with these legacy gpu's will likely only grow.

"I figured out it was something with the GPU when it runs fine in safe mode, though installing from Nvidia direct even in safe kills it. "

Safe Mode disable the drivers, so the gpu is not functioning in Safe Mode. It is only acting as a pass through for the generic Windows

VGA drivers which run can display Windows desktop and similar low end graphics when the gpu is not functioning.

Technically, installing the drivers in Safe Mode should not cause any problem until you reboot after they are installed.

If the machine BSODs straight away after you install the graphics drivers and reboot there is a good chance of an issue with the card.

The 5 sec interval is probably about how long it takes for the Nvidia driver to fully load when booting into Windows.

- I would first pull and reseat the card and verify both power cables are fully seated.

If the machine still BSOD's after doing that, the easiest way to verify the issue is not due to a problem with the motherboards

PCIe slot/circuit, power supply, or the card itself is to try another card, if you have one.

Or, if you don't have access to another card, you can verify if the card if faulting by testing it in another machine, if possible.

[EDIT: Also, if the card is an upgrade a motherboard BIOS update may help ]

If these options are not available you you may need to apply  to the mfg of the card for an RMA (if desktop upgrade),

or apply for warranty repair if laptop or new machine that is still within warranty period.

Or take the machine to a local tech for testing.

If it is a Retail licence, then it will have transfer rights to a new PC, if it is OEM it will not . ..

Since you previously had Windows 10 installed and activated on that PC during the installation process skip the steps which ask for a product key and select the option 'I am Reinstalling Windows 10 on this PC', and activation will not be an issue, your PC will have a digital entitlement stored on the Microsoft Activation Servers
