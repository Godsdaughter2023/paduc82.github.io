bernie deitrick say:

Normally, in VBA you could use specialcells to identify filtered cells, but that does not work with Functions.

So you need to use the SUBTOTAL function, like this:

Function Countcolour(rng As Range, colour As Range) As Long
    Dim c As Range
    Application.Volatile
    For Each c In rng
    If Application.WorksheetFunction.Subtotal(3, c) = 1 Then
        If c.Interior.ColorIndex = colour.Interior.ColorIndex Then
            Countcolour = Countcolour + 1
        End If
    End If
    Next
End Function

drdiagram say:

How can I batch convert doc to docx for my MSWord docs?

Original title: doc to docx?

daniel g. grau say:
New attempt

How about a script? Paste this one in AppleScript Editor and run it. It will Save As all open files into the .docx. format and add that as an extension. The resulting files are placed next to the original. Try it first with just a few files, then on increasing numbers while Word does not choke.

As mentioned in my previous post, you can do a Finder's search for names containing .doc, open those files right there from the results window and then run the script. Finally, repeat the find operation in the Finder, Command-Click on the files you no longer need (duplicates), then trash them.

I think you will be surprised at how little space you end up saving. Yesterday I batch-converted hundreds of space-hungry .RTF files into .docx format. They were on the disk of an old, storage-starved Mac. The disk's free space went from 6.1 GB to 6.3 GB. Although 200 MB were recovered, this is a very small percentage of the total free space.

--------------SCRIPT TO PASTE FOLLOWS--------------------

tell application "Microsoft Word"

   repeat with i from 1 to the count of (documents)

      save as the front document file name path of the front document & ".docx" file format format document

      close front document saving no

   end repeat

end tell



lou brucker say:

People reading this thread should be aware that renaming a document (even changing its ending) is not the same as a conversion.  In the case of Microsoft Office files like .doc > .docx some things change or add features when you truly open a file and save as... and change the format to .docx.  One of these things is opportunistic locking functionality.

A real .docx file stored on an AFP (mac environment) server can not be accidentally opened by 2 users at the same time - an ending renamed file as suggested above can, as it has not changed its format just its name.  

This can be very important in certain organizations ESPECIALLY since renaming it to .docx implies that op-locking would work, when in fact it doesn't.

Contacts have a built-in field called CustomerId. After you run this you can add the CustomerId column to the Contact Folder view.  Call this on a New Item event for the contact folder to make it automatic.

 

Sub SetCustomerIDs()
    Dim ol As Outlook.Application
    Dim olns As Outlook.NameSpace
    Dim oConItems As Outlook.Items
    Dim oCurItem As Object
    Dim I, J, iNumItems, iContactId As Integer
    Set ol = New Outlook.Application
    Set olns = ol.GetNamespace("MAPI")
    Set oConItems = olns.GetDefaultFolder(olFolderContacts).Items
    iNumItems = oConItems.Count
    iContactId = 0

    For I = 1 To iNumItems
        Set oCurItem = oConItems.Item(I)
        If TypeName(oCurItem) = "ContactItem" Then
            J = oCurItem.CustomerID
            If IsNumeric(J) Then
                If J > iContactId Then
                    iContactId = J
                End If
            End If
        End If
    Next

    iContactId = iContactId + 1

    For I = 1 To iNumItems
        Set oCurItem = oConItems.Item(I)
        If TypeName(oCurItem) = "ContactItem" Then
            If oCurItem.CustomerID = "" Then
                oCurItem.CustomerID = iContactId
                iContactId = iContactId + 1
                oCurItem.Close olSave
            End If
        End If
    Next
    Set oConItems = Nothing
    Set olns = Nothing
    Set ol = Nothing
End Sub

I think you mean the Items.ItemAdd event. (There isn't a NewItem event.) In any case, SnooziSuzi wanted to integrate this with her custom form. One way to do that would be to include code in the Item_Open event handler to get the last number used (see function below) and then assign that to the CustomerID property on the new item. 

For I = 1 To iNumItems

        Set oCurItem = oConItems.Item(I)
        If TypeName(oCurItem) = "ContactItem" Then
            J = oCurItem.CustomerID
            If IsNumeric(J) Then

                    iContactId = J
                End If
            End If
        End If
    Next

    iContactId = iContactId + 1

There's no need to loop through all the items in the folder twice. In place of the first loop (above), you can simply sort on the CustomerID field in descending order and get the number from the first item. The same code, with minor modifications to account for the differences between VBA and VBScript, could be used in VBA to return the last number used for the bulk SetCustomerIDs operation and in VBScript behind the form to get the next number for the new item. Here's a VBA version:

Function GetLastNumber(itms as Outlook.Items) as Integer
    Dim cItems as Outlook.Items
    Dim strFilter as String
    Dim cont as Outlook.ContactItem

    ' use the message class for your custom form
    strFilter = "[MessageClass] = ""IPM.Contact.YourForm"""
    Set cItems = itms.Restrict(strFilter)
    If cItems.Count > 0 Then 
        cItems.Sort [CustomerID], True
        Set cont = cItems.GetFirst
        If cont.CustomerID = "" Then 
            GetLastNumber = 0 
        Else
            GetLastNumber = CInt(cont.CustomerID)
        End If
    Else
        GetLastNumber = 0
    End If

    Set cont = Nothing
    Set cItems = Nothing
End Function
