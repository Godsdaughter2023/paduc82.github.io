The Envelope Address style, which is used for the address of an Envelope, contains a Frame.  Modify the width of that Frame in the Modify Style dialog for the Envelope Address style.

graham mayor say:
Envelopes are formatted using the Envelope Address and Envelope return paragraph styles from the active document which in turn were derived from the template that document was created from. As has already been reported the Envelope Address paragraph style features a frame with fixed dimensions. You can modify that frame size and save the results to your template and that will then be used next time - see http://www.gmayor.com/changing_envelope_layout.htm. If you set the width of the frame to 'Auto' then the lines will never wrap as long as there is space on the envelope for the frame, so it is perhaps better to set its width to a measurement that will allow your longest address lines to fit.

When modifying styles don't forget top check the box 'New Documents based on this Template' or the changes will apply only to the current document (which appears to fit the issue you describe in your latest reply).

Alternatively, you could use a template for your envelopes, formatted as you wish. To facilitate that see http://www.gmayor.com/Envelope_Label_Add_In.htm  which comes complete with a sample envelope that you can modify to your taste.

rohn007 say:
No, the encryption settings are Office version specific. Changing to 256 bit for one will not affect the other. The default setting only affect the creation of new encryption. Both versions will automatically adjust to the encryption level that a file has been saved in as long as they have the encryption software to handle it, which both do.  Changing the default encryption level will NOT change the encryption on existing files.

Encryption

Encryption in 2007 and 2010 can be "better" than the default, in that they can be set to use 256 bit AES. Here are links to more specific information:

Enabling password rules for Office 2010 (includes “time to hack” table) - https://blogs.technet.com/b/office2010/archive/2009/10/16/enabling-password-rules-for-office-2010.aspx

This article includes a “time to brute force” a password chart. The thing to remember about this is that it works on a few assumptions:

·         Average computer speed

·         Average number of CPU’s in a computer

·         Single computer

·         “general” computer

Computer specs rise very quickly, so do hacking techiques. There are two very important things go be aware of. “Cloud” computing has come to hacking. Hackers can now by time on clouds of hacked computers with 100’s or even 1000’s of computers, splitting the load.  Hackers have learned to use video graphics cards to do hacking. They are MUCH faster due to the way they are designed to process images, doing MANY similar calculations at the same time.

The bottom line, is that you want to use full “complexity” (mix of character types) and the longest possible password you can.

Security and protection for Office 2010 - http://technet.microsoft.com/en-us/library/cc179135.aspx - links to many detailed articles
