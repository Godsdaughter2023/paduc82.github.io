Formula - Calculate Only to Certain Amount (Figure Ceiling) - Microsoft Community

rpay say:

Formula - Calculate Only to Certain Amount (Figure Ceiling)

Hello,

I a have a two tiered calculation that I am looking to do. The number applies a certain percentage up to a certain base amount and then applies a different one to anything past that. For example: $10-200 applies 0.1% for each dollar added and anything past $200 applies at 0.2%. If a certain dollar figure is input, I need to find the right amount added.

I have done this before with a single tier/surcharge rate, but never with two. I have set up two tables for the data and have a third to add them together.

The upper table I have functioning properly with a sumif function added to make sure negative numbers are excluded, but the lower table still needs work. For example, if I had a dollar figure of $300, it is applying 0.1% on the whole $300, not just the first $200. When added with the 0.2% on the $100 past that, it is far too high.

How do I set up a limit on the one table so it only calculates to amount?

Thanks.
joeu2004 say: RPay wrote:
I a have a two tiered calculation that I am looking to do. The number applies a certain percentage up to a certain base amount and then applies a different one to anything past that. For example: $10-200 applies 0.1% for each dollar added and anything past $200 applies at 0.2%.
[....]
For example, if I had a dollar figure of $300, it is applying 0.1% on the whole $300, not just the first $200. When added with the 0.2% on the $100 past that, it is far too high.

There are many ways.  Two simple ways, assuming the amount (e.g. $300) is in A1:

=A1*0.1% + MAX(0,A1-200)*0.1%

or

=MIN(200,A1)*0.1% + MAX(0,A1-200)*0.2%

In the first formula, note that the second 0.1% is 0.2%-0.1%; that is, the difference between the two pecentage rates.

Whatever formula you use, you probably want to round to the penny.  For example:

=ROUND(MIN(200,A1)*0.1% + MAX(0,A1-200)*0.2%, 2)

Formatting to 2 decimal places alone rounds the displayed value, but not the underlying actual value.  Leaving the actual value unrounded might cause problems later on, e.g. with sums of columns of such values.

PS:  Note that this will charge $0.01 to amounts as low as $5.  Do you really need to limit the 0.1% rate to $10 or more arbitrarily.  Or did you choose $10 as a lower limit simply because 10*0.1% is 0.01?
