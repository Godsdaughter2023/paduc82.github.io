Which permissions override when using Folder permissions and NTFS permissions?

sarahkong say:
I teach this all the time.
The way it works is the most restrictive applies when combining share and NTFS permissions.
What that means is if you share a folder as read and then you set NTFS permissions as read and write you are only going to get read. Because it is the most restrictive when combined.
So basically set your share permissions for your group as full control then use NTFS permissions to create the restrictions.
So if you want a group to be able to read and write but not delete then set share as full control from that group. Then go to NTFS permissions and set explicit permissions for that group to only read and write.
