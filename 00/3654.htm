gina whipp say:
Error handling won't handle errors with code itself (VBA).   It's not a bug, as far as I know, it has always been this way.

andreas killer say:
The most common cause is the incorrect use of a error handling.

Copy the code below into a regular module and run both subs to notice the difference.

Andreas.


Sub Wrong()
  Dim i As Integer, a As Integer, b As Integer
  On Error GoTo Ignore
  For i = 1 To 2
    a = i / b
Ignore:
  Next
End Sub

Sub Right()
  Dim i As Integer, a As Integer, b As Integer
  On Error GoTo Errorhandler
  For i = 1 To 2
    a = i / b
  Next
  Exit Sub

Errorhandler:
  Select Case Err.Number
    Case 11
      'Ignore this error
      Resume Next
    Case Else
      Debug.Print "Source     : " & Err.Source
      Debug.Print "Error      : " & Err.Number
      Debug.Print "Description: " & Err.Description
      If MsgBox("Error " & Err.Number & ": " & vbNewLine & vbNewLine & _
          Err.Description & vbNewLine & vbNewLine & _
          "Enter debug mode?", vbOKCancel + vbDefaultButton2, Err.Source) = vbOK Then
        Stop 'Press F8 twice
        Resume
      End If
  End Select
End Sub

Private Sub Document_ContentControlOnExit(ByVal CCtrl As ContentControl, Cancel As Boolean)

What is web page filtered? - Microsoft Community

healee say:

What is web page filtered?

Does Microsoft Word let us read the souce code or edit the file when it opens an html file?  I don't seem to be able to change anything.  Is it meant to be read-only mode?  When we save a document as a web page with Microsoft Word what happens to the html code?  What is web page filtered?

rohn007 say:

When you save to "web page" or "single file web page" MS includes all of the formatting and style information found in normal.dot, and it includes a lot of "extra", repetitive explicit html formatting around the text.

When you save as "Web page filtered" word strips out most of the extra html code preserving close to the minimum required to preserve the look of the file. The resulting file is much smaller than the equivalent DOC/DOCX and also smaller than an unfiltered HTML file saved from Word.

WORD is a WYSIWIG (sp?) editor, it translates the html into display formatting. I've never tried to edit existing HTML code natively in word. If you want to look at the underlying HTML code, download one of the many free html IDE tools that gives you multiple views of a html file. The simplest html code editor provided with windows is Notepad.

If you have a control on your form called Email which contains the email address and (for example) another called Client (with the client’s name in) you could use something like this

Private Sub cmd_send_email_Click()
If Not IsNull(email) Then
 On Error GoTo cmd_send_email_Click_Err
    Dim strTitle As String
    Dim strEmailAddress As String
    Dim strClientName As String
     strEmailAddress = Me.email
     strClientName = Me.client
            strTitle = "This is a message to “ & strClientName
            DoCmd.SendObject acSendNoObject, "", "", strEmailAddress, "", "","strTitle", "", True, ""
cmd_send_email_Click_Exit:
    Exit Sub
cmd_send_email_Click_Err:
    If Err.Number = 2501 Then
        MsgBox "The e mail hasn't been sent", _

    End If

End If
    MsgBox "You can't send an e mail to someone" & Chr(13) + Chr(10) & _
    "without an e mail address ?", vbInformation, "ooops  :-)"
End Sub
