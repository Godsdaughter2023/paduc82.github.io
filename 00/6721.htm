=Switch([Year] = "1-106" Or [Year] = "1-115" Or [Year] = "1-118" Or "[Year] = "1-125" Or [Year] = "1-343", "Pilot #1", [Year] = "2-114" Or [Year] = "2-116" Or [Year] = "2-117" Or [Year] = "2-303" Or [Year] = "2-344", "Pilot #2", [Year] = "3-111" Or [Year] = "3-302" Or [Year] = "3-341" Or [Year] = "3-342", "Pilot #3", True, "-")

The arguments are evaluated from left to right.  As soon as the first in a pair of arguments, which must be a Boolean expression, evaluates to True, the second argument of the pair is returned by the function.

BTW Year is a bad choice for an object name in Access.  As the name of a built in function it is a 'reserved keyword', so should be avoided as an object name.

PS:  I should have mentioned that, in a relational database, hard coding data in an expression like this is not good practice.  The relationship type between pilot numbers and year values should be modelled by a table in which one column is the year value, the other the pilot number.   The latter will be repeated in multiple rows of course.
