mark l. ferguson say: Rather than a problem with running regedit, I would guess that there is some Permission lacking in the exact registry key you are trying to write with a reg file on startup.

Just as an example, the startup command might look something like this:

regedit /s "C:\myfolder\myreg.reg"

In myreg.reg there would be some key location:

HKEY_LOCAL_MACHINE/SOFTWARE/Microsoft/Windows/Current Version/SomeKey

Opening that key in regedit, you might go to the Edit menu, and see the Permissions entry for it, adding your user to the list.

If, on the other hand, it is just opening regedit you want, without the UAC prompt, create a new shortcut in the startup folder, and rightclick it for Properties, and put a check in the checkbox to 'run as administrator' (Shortcut Tab, Advanced button)

a.    Click Start. Click Run and type regedit.exe in the Run dialog box and press Enter.
