Hi,

If you are automating Excel (i.e. manipulating Excel from code running in MSP), then you need to include the xlAp. when you refer to any Excel object (like Range).

 

                       With xlApp.Range("A1:C4")
                             .Borders.LineStyle = xlContinuous
                             .BordersAround Weight:=xlThick
                        End With
 
Without the "xlApp" there, VBA will be assuming "Range" is a method belonging to MSP.
 
What Bernie was saying is do not name your Range object "XLRange"
                 Dim xlRng As Excel.Range

                             .Borders.Color = vbBlack

Another one of the applications found a file that slipped by McAfee's defenses. It's called "update.vbe" and was found in "C:\Users\<username>\AppData\Roaming\Origin". I have removed Origin a long time ago and cleaned all remaining files using Revo Uninstaller, so I was suspicious. After uploading it to VirusTotal, it turns out that it was a Bitcoin miner. I opened the file using Notepad and found out it was encrypted.

I then stumbled upon a site in which the person who got infected with "update.vbe" has decrypted the file. After reading through the text, I can make out that the infected "svchost.exe" process may be tied to the .vbe file. The tasks in the file seem to prove it clear.

Let me hear your say about this, I don't think this will be easy to solve.

PS. I shredded the "update.vbe" file upon discovering it. It somehow reappeared after a restart. However, after removing some files using the tools I mentioned and rebooted, the Origin folder is now empty.
