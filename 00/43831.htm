First let me say that I'm very sorry to hear this but rest assured that temporarily disabling Defender & then restarting the computer in a "clean boot" environment should not have caused such problems - assuming all had been well with the computer beforehand.

It would be highly unusual - in fact, exceedingly rare - that a Defender Defs update would have caused the BSODs - assuming the computer was not (already) infected when the Defs version was installed.

Assuming the computer was not infected, the most likely cause of your problem (i.e., the sudden appearance of the BSODs & your current situation) is Automatic Updates installing one or more third-party driver updates which your computer probably didn't need.

Background: When Automatic Updates (AU) is set to the default Automatic configuration, AU downloads & installs all Important (AKA Recommended)updates which include Optional, non-security Software (e.g., .NET Framework 4 Client Profile) and Hardware (i.e., third-party driver) updates. [Nice, huh?]

Moving on...

You cannot uninstall driver updates (primarily since they're not listed in Installed Updates) but you can "roll back" to the previously-installed driver version - if you could start the computer (e.g., in Safe Mode). See...

   • How to Roll Back a Driver in Vista
      http://windows.microsoft.com/en-US/windows-vista/Restore-a-driver-to-its-previous-version

So what do (or can) you do now?

OPTION A: If you can start the computer (e.g., in Safe Mode), you could try rolling back the driver-in-question. The "fly in the ointment" is that I'm not sure you'd be able to identify which driver may have been upgraded. However, the experts in the Hardware subforum may be able to help you with that: http://answers.microsoft.com/en-us/windows/forum/windows_vista-hardware

OPTION B: If you can start the computer (e.g., in Safe Mode), you could try doing a Repair Install (AKA Startup Repair AKA In-place Upgrade) of Vista. See this tutorial: http://www.bleepingcomputer.com/tutorials/repair-windows-with-windows-startup-repair/

OPTION C: Residents of North America (US & CA), the UK (and perhaps other regions now) may obtain Microsoft-sponsored Premium (i.e., paid) Support via the Answer Desk => www.answerdesk.com [1] [2]

OPTION D: If all else fails (and I as much as I sincerely regret having to say this), your only remaining option would be a clean install of Vista. See the (my) ANSWER post in this thread and follow those instructions (to-the-letter & in order! ) to return your computer to a secure & functional state: http://answers.microsoft.com/thread/c44429fb-3f7c-4646-8529-5a97bb3cd0eb

lemp say:
That's a really sketchy way of reinstalling Windows

One of the first steps in a clean install of Windows (which is what you're trying to accomplish) is to format the hard drive.  Plainly, if you format the hard drive, you'll lose all of the files that you're in the middle of using to install Windows and everything will simply stop.

Because whatever you did resulted in two installations of Windows, I suspect that you may have created a new partition and installed Windows there.  Or you may have installed Windows in a separate folder in the original partition.  Now that you have edited you're boot.ini file, it's difficult for me to ask you to look at it for information.

However, whether you deleted a folder in the current partition that had "old Windows" in it, or created a new partition and then used Disk Management to delete the original partition, you haven't securely wiped the old installation.  There are readily-available free tools on the Internet that permit one to undelete folders or partitions.

If you're certain that there is no restore partition AND you have no access to a installation CD for the same version Windows XP and is from the same computer manufacturer AND you don't want to just wipe the drive as I suggested earlier, then create a bootable XP installation CD using C:\i386, reboot from that CD, repartition the hard drive into a single C: partition, format it, and install Windows there.  This will overwrite your old information and render it unavailable to anyone short of highly sophisticated forensic recovery experts.

I don't have a favorite method for creating a bootable XP install CD from the C:\i386 folder.  See https://www.google.com/#q=create+xp+install+cd+from+i386

Placing your system in Clean Boot state helps in identifying if any third party applications or startup items are causing the issue. Check this issue in clean boot state.

1.   Press Windows key + X key. And select Run.

2.   Type msconfig in the Run box and hit Enter.

3.   On the Services tab of the System Configuration dialog box, tap or click to select the Hide all Microsoft services check box, and then tap or click Disable all.

4.   On the Startup tab of the System Configuration dialog box, tap or click Open Task Manager.

5.   On the Startup tab in Task Manager, for each startup item, select the item and then click Disable.

6.   Close Task Manager.

7.   On the Startup tab of the System Configuration dialog box, tap or click OK, and then restart the computer.

Refer: How to perform a clean boot in Windows
http://support.microsoft.com/kb/929135

Note: After clean boot troubleshooting step, follow the section "How to reset the computer to start as usual after troubleshooting with clean boot" in the provided link to return your computer to Normal startup mode.

Steps to uninstall the Graphics Card Drivers.

  1. Press “Windows Logo” + “X” keys on the keyboard.
  2. Click on “Device Manager” from that list.
  3. Search for the Graphics Card from the device list, right click on it and then select “Uninstall”. On the un-installation window, if you have an option: “Delete the driver software for this device” you may select that and then remove the corrupted drivers from the computer.
