Try deleting and re-creating the Registry key referred to in the following instructions:

To use the Send To option in Office applications with Windows Live Mail, make the following changes to the Window Registry.
Note: Just in case something goes wrong, you should first back up the registry by following the steps in the following article:
http://windows.microsoft.com/en-US/windows7/Back-up-the-registry
Now, back to the original issue:
1. Click on Start > All Programs > Accessories > Run > Type regedit
2. Browse to HKEY_LOCAL_MACHINE > Software > Client > Click on Mail
3. Check what appears in the right pane. It should read as below:
   (Default)         REG_SZ         Windows Live Mail
4. If Windows Live Mail does not appear, right click on (Default) and click on Modify and in the Value Data box, type
Windows Live Mail
Note: If you don’t find the Mail item, right click on Client and Click New > Key and rename the key to:
Mail
Once the Mail key is created, right click on Mail > New > String Value. Name the value as:
(Default)
Right click on (Default) and Click Modify. In the Value Data box type
Windows Live Mail

suzanne s. barnhill say:

In my experience, this is purely a display issue. At some Zoom ratios, it appears that Word allows more characters on the line than I would expect. Are you experiencing this in justified or ragged-right text?

If you truly need to fit more characters onto a line, you can use the Condensed setting on the Character Spacing tab of the Font dialog. I have a toolbar button (in Word 2003) that condenses selected text by 0.1 point, which is usually enough to make a dramatic difference.

In previous versions, you could enable the Compatibility Option to "Do full justification like WordPerfect 6.x for Windows," but Word 2013 has removed this option, primarily because something like it has been implemented by default.

Formula as follows:

=INDIRECT("[" & Table1[[#Headers],[DECEMBER]] & ".xls]DATA!"&[@TAG])

 

VBA Code as follows.

 

VBA Code has been modified since initial post. The formula to insert should have contained the variable "Report" instead of the fixed column name so that it is dynamic to reflect the month of the column being inserted.

 

Sub AddColumn()

    Dim WS As Worksheet
    Dim tbl As ListObject
    Dim col As ListColumn
    Dim Report As String
   
    Report = "DECEMBER" 
    
    Set WS = Worksheets("Sheet1")
    
    Set tbl = WS.ListObjects("Table1")
    
    Set col = tbl.ListColumns.Add
    
    col.Name = Report
   
    tbl.ListColumns(Report).DataBodyRange = _
        "=INDIRECT(""["" & Table1[[#Headers],[" & Report & "]] & "".xls]DATA!""&[@TAG])"
   
End Sub

 

Following also added since initial post. It finds the last used month in the headers and creates a new column for the next month. See comments where it tests for the existance of the column before inserting a new one.

Sub AddColumn_2()

    Dim WS As Worksheet
    Dim tbl As ListObject
    Dim col As ListColumn
    Dim Report As String
    Dim myDate As Date
   
    Set WS = Worksheets("Sheet1")
    
    Set tbl = WS.ListObjects("Table1")
   
    'Find and save the last existing column header name
    Report = tbl.ListColumns.Item(tbl.ListColumns.Count).Name
   
    'In the following line of code: _
     1 can be any valid day for the month (safe between 1 and 28) _
     2000 can be any year for a valid date (safe between 1900-9998)
    myDate = DateValue(Report & " 1 2000")
   
    'Calculate the next month after last used month
    Report = Format(WorksheetFunction.EDate(myDate, 1), "mmmm")
   
    'Test if the header already used and if used, terminate processing.
    If WorksheetFunction.CountIf(tbl.HeaderRowRange, Report) > 0 Then
        MsgBox "Column header " & Report & " already exists." & vbCrLf & _
                "Processing terminated."
        Exit Sub
    End If
   
    Set col = tbl.ListColumns.Add
   
    col.Name = Report
   
    tbl.ListColumns(Report).DataBodyRange = _
        "=INDIRECT(""["" & Table1[[#Headers],[" & Report & "]] & "".xls]DATA!""&[@TAG])"
   
End Sub

ossiemac say:

In case you have started to work with my answer, please note that the code has been edited since my initial post. I submitted this post becasue unfortunately there are no notifications when a post is edited.

 

Also you might be interested in the following guidelines for creating indirect formulas.

 

How to create an indirect formula.

Enter the formula as follows by entering the equals sign and then changing workbook/worksheet etc and then clicking the cell for the link. I assume you did this to get the basic formula that you posted as follows.

=[DECEMBER.xls]DATA!ABCD

Now look at what you want to replace with variables (cell addresses). They are DECEMBER and ABCD. Therefore those names need to be replaced with the variables and everything else enclosed with double quotes and concatenated with the variables using ampersands.

Start by adding INDIRECT and open the parenthesis to commence the indirect function parameters.

=INDIRECT([DECEMBER.xls]DATA!ABCD

Then enclose the square bracket in double quotes and add an ampersand.

=INDIRECT("["&DECEMBER.xls]DATA!ABCD

Now highlight DECEMBER only and then click on the cell containing the word DECEMBER and it will be replaced with the address (in this case the Table address.)

=INDIRECT("["&Table1[[#Headers],[DECEMBER]].xls]DATA!ABCD

At the end of the address add an ampersand and then open the double quotes to start enclosing the next string.

=INDIRECT("["&Table1[[#Headers],[DECEMBER]]&".xls]DATA!ABCD

Now move along to the next string to be replaced with a variable (ABCD) and immediately before it add the closing double quotes for the preceding string and add another ampersand.

=INDIRECT("["&Table1[[#Headers],[DECEMBER]]&".xls]DATA!"&ABCD

Highlight the ABCD and then click on the cell that holds the value to replace it and you are finished.

=INDIRECT("["&Table1[[#Headers],[DECEMBER]]&".xls]DATA!"&[@TAG])
