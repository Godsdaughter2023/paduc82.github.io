rob koch say:

Searches find this Heurengine.zerodaythreat detection by Spyware Doctor in threads going back several years, typically either indicated as a possible false positive or in threads that terminated completely with no response from the original poster.

Since the name implies heuristics, it's common for such detections to contain false positive results, since the intent of the heuristics is to attempt to detect future items that aren't an exact match for currently known malware.

I'd personally not trust any results from such less well known products, regardless of whether I believed that a system might actually contain additional malware.

In a case where I was questioning whether undetected malware remained, I'd more likely try the Malwarebytes Anti-malware free product, since that has a far better reputation at finding the types of annoying, but not always truly damaging adware or other potentially unwanted software that often comes via either free software downloads or attached to spam email.

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/while-in-sleep-mode-security-essentials-will-not/4beb26be-ed1d-4793-8b75-e4cb8991aee5

To change Automatic Maintenance settings: Right-click on the Start button > Control Panel > Security & Maintenance > Maintenance > Automatic Maintenance > Change maintenance settings > Allow scheduled maintenance to wake up my computer at the scheduled time

That being said, we certainly don’t need to leave our PCs on all night, or even reschedule the scan, in order to get the scheduled scan up and running (like we did with MSE) – because Automatic Maintenance will always run the maintenance tasks on the next startup whenever the scheduled time is missed. So if you just leave the default scheduled time alone, and turn off your PC normally at the end of the day, then scheduled maintenance will be triggered the next day shortly after startup, and the maintenance tasks will be launched whenever you take a break from your work and let the system enter an idle state.

teylyn say:
Instead of a direct reference, you could use a lookup formula, for example Index/Match. Set up the cities in column A and the months in row 1 of the Summary sheet. Then you can use in B2

=index(Data!$A$1:$ZZ$1000,match($A2,Data!$A$1:$A$1000,0),match(B$1,Data!$A$1:$ZZ$1,0))

copy across and down. Note the placement of the $ signs. The first Match will always use column A, but in the current row, and it will try to find it in column A of the Data sheet. The second match will use the value in the current column, in the first row, and will try to find it in row 1 of the Data sheet.

        Name myNewFN As myFN
    Next myFN
    End
End Sub

Hi,

Firstly, I know nothing about this object...  with that in mind...

Looks like .ResponseText returns a string.  So c00 is a Variant object in this case (because you didn't declare it with a Dim statement) that contains a string.

Right-click on c00 and Add A Watch.  Look at it's properties and values in the Watches pane in the VBE (View > Watch Window) as you step through the code.

You can also do:
    Debug.Print c00
in your code and look in the Immediate window (press Ctrl+G in the VBE).

The formula then performs a series of operations on that string to split it into 4 cells.

Now, when I try your code, it fails, but this might be to do with my firewall settings.

You might find a better way to muddle your way through this is to write your code like this:

    Dim req As Object         '<--add a Watch on the req object to see what properties are available.
    Dim vResp As Variant    '(or: strResp As String ??)

    Set req = CreateObject("WinHttp.WinHttpRequest.5.1")
    req.Open "GET", "[URL]http://www.coupons.com/coupons/?zip=77477[/URL]", False
    req.Send
    vResp = req.ResponseText

    Debug.Print vResp

I don't know whether you need the [URL] .... [/URL] around your url....

The above code uses late binding.  You may find that adding a reference (VBE>Tools>References...) to the "Microsoft WinHTTP Services, version 5.1" library then allows you to declare "req" like this:

    Dim req As WinHttp.WinHttpRequest

Now when you type req. in your code you will get intellisense dropdown with all available properties and methods for that object type.

See here for more (but it isn't VBA based, the info should still apply):

http://msdn.microsoft.com/en-us/library/windows/desktop/aa384106(v=vs.85).aspx
