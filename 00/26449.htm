robinbrush say:

Access 2016 - Adding a temporary tick box to records on a form

My next idea was to use a multidimensional array as in

  Dim tempvar(DCount("*","queryname")) as Long

and somehow then set the relevant element of tempvar depending on which record was chosen.

However, for that to work I need a function that returns the recordnumber of the current record/row in a continuous form, and I also need to know how to attach a separate independent tickbox to each record/row.

Any suggestions or help would be appreciated.

scott mcdaniel say:

With a Continuous or Datasheet form, you would have to bind that checkbox to an underlying field, otherwise every checkbox on the form would be the same, and checking one would check all the others.

The simplest way is to just add a Yes/No field to your table, and use that. Problem with that is if you have a multiuser application, then you could have users overwriting another users selections.

You could also add a "Selections" table to your database, and Join that table to your form's recordsource. Your "Selections" table would have a Yes/No field in it, and some sort of joining mechanism. When you load the form, you'd have to fill that Selections table with the Primary Key of the "other" table, then Join those two, then use the Yes/No field from Selections in the form. When the form closes, you clear that table. This avoids the multiuser issue above, if you put the table in the Front End of your application.

BitSqueezer has a demo database with an interesting example of using a class to do this:

http://www.ccedv.de/downloads/start.php?lang=en

Look for the "SelectRecordsV2" download and check that.

scottgem say:

Another alternative here is using a local table for the selections. This eliminates the multi-user problem. Then, rather then include the Selections table in the form's Recordsource use either a select button or the Double click event of the form to run code like:

Dim strSQL as String

strSQL = "INSERT INTO tblSelections (RecordID) VALUES(" & Me.PKcontrol & ");"

CurrentDB.Execute strSQL

PKcontrol would be the name of the control holding the primary key.

pedro_lane say:

hi

 

802.1x is just the standard way to enable EAP authentication and port access authorization. so WPA EAP and 802.1x are two terms for the same things, at least in wireless network.

 

anyway your config seems ok. what i see in your e7 screenshots is that you're using EAP-PEAP-MSCHAP with a trusted CA. this is the most common combination and the main one supported by MS Windows.

 

the CA certificate "mobility root ca mobility" is not a "public" CA, so N9 doesn't trust it. N9 should ask you if you trust this CA and then allow to connect. but N9 doesn't seem to behave this way.

 

then you have to manually import this CA (settings > security > certificates > add) to allow N9 to trust it. you need the certificate in a file (pem, der, p12 etc).

 

ask your network admin to provide you with.

 

good luck
