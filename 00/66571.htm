Is Java Necessary in Windows 7?

Split from this thread.

Java update in Window 7 - Is Java necessary in Windows 7? How can I disable automatic updates in Java

lemp say:

1.  No, Java is not necessary and many security experts suggest uninstalling all versions of Java.  At the very least, you should uninstall all old versions of Java.  Java has a tool for this purpose.  See https://java.com/en/download/faq/remove_olderversions.xml

2.  On the other hand, there are still some applications that require Java in order to work.  If you remove Java, these applications will no longer function.  You can uninstall Java and then re-install it if you find some necessary application is no longer functional.

3.  You should not disable automatic updates for Java.  The reason that Java gets frequent updates is because security holes keep getting discovered and then patched.  If you keep Java and disable its automatic update function, then you will have to check for updates manually and frequently.

4.  If despite #3 you still want to disable automatic Java updates, here's how:

-- Go to Control Panel and in the upper right corner, select "Small icons" or "Large icons" next to "View by"

-- Double-click the "Java" icon in Control Panel

-- Select the "Update" tab and make the changes you desire.  You can disable updates entirely or change the schedule for checking to daily, weekly, or monthly.

-- I suggest that you NOT mess with the other settings without thoroughly researching what they do

-- OK your way out

Hi,

Copying files from one partition to another is much slower than copying to a different drive...

Two different partitions still use the same read\write heads internally on the HDD...

They need to take turns reading and writing...

It is somewhat normal for there to be a fast burst when starting to copy that then slows down... This is due to the files being buffered in the Systems memory, which is considerably faster...

It eventually slows down to match the speed the drive being written to is able to handle...

In addition to all of this, I have noticed that the drivers provided by Microsoft for the "Intel Chipset" on my machine were not performing well... The HDD speeds were slower, and the USB 3.0 speeds were slower than USB 2.0 speeds...

In my case, I checked with the motherboard manufacturer and found a newer version of the Intel Chipset "INF" driver which updates all of the Intel Controllers...

I found an even newer one at Intel by checking Support\Drivers for my specific CPU... Both worked...
