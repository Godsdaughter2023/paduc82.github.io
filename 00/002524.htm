doug robbins - mvp office apps & services (word) say:

Delete the duplicate Dim i As Long and replace

lastnum = .Paragraphs(.Paragraphs(.Count)).Range)

with

lastnum = .Paragraphs(.Paragraphs.Count).Range

(not having a document to test it on, I had not tried to run it.)

However, rather than trying to split your present document (which is likely to lock up your computer for a significant time, if not permanently, requiring you to end the Word process), it may be better to use the following code to create each document from scratch

Dim i As Long
Dim j As Long
Dim k As Long
Dim doc As Document
k = 0
For j = 1 To Int(1852083 / 16384)
    Set doc = Documents.Add
    For i = 1 + 16384 * k To 16384 * (1 + k)
        doc.range.InsertAfter i & "; "
    Next i
    doc.SaveAs filename:=1 + 16384 * k & "-" & 16384 * (1 + k) & ".docx"
    k = k + 1
Next j
Set doc = Documents.Add
For i = 1 + 16384 * k To 1852083
    doc.range.InsertAfter i & "; "
Next i
doc.SaveAs filename:=1 + 16384 * k & "-1852083.docx"

macropod say:

If the numbers are sequential, it would be far faster to simply create the required files afresh, using a simple loop. For example to create the output as plain text files:

Sub MakeFiles()
Dim i As Long, j As Long, StrOut As String
j = 1
For i = 1 To 1852083
  StrOut = StrOut & i & "; "
  If i Mod 16384 = 0 Then
    Call CreateFile(StrOut, j & "-" & i)
    StrOut = ""
    j = i + 1
  End If
Next
Call CreateFile(StrOut, j & "-" & i)
End Sub

Sub CreateFile(StrOut As String, StrName As String)
Open "C:\Users\" & Environ("UserName") & "\My Documents\" & StrName & ".txt" For Output As #1
Print #1, StrOut
Close #1
End Sub

To output as Word documents instead, replace the 'CreateFile' sub with:

Sub CreateFile(StrOut As String, StrName As String)
Dim wdDoc As Document
Set wdDoc = Documents.Add
With wdDoc
  .Range.Text = StrOut
  .SaveAs FileName:="C:\Users\" & Environ("UserName") & "\My Documents\" & StrName & ".docx", _
    Fileformat:=wdFormatXMLDocument, AddToRecentFiles:=False
  .Close
End With
Set wdDoc = Nothing
End Sub

About MSDN or TechNet Product Keys

They are genuine Microsoft product keys for retail licenses, but not for Retail sale. They are intended for a particular product channel either the Microsoft Software Developer Network (MSDN) or TechNet for IT Professionals who pay a subscription fee. The main purpose is for evaluation purposes. It is for one person only to use and no one else. Unfortunately, regardless of the licensing terms, persons still abuse the program, either giveaway product keys or resell it on auction sites. 

What probably happened is, you bought an MSDN licensed key, which carries up to 10 activations unlike full packaged retail licenses which only carry 1 activation. The person who sold it to you probably sold it to 10 other persons. Somewhere along the way, one of those persons might have installed it on a second system, activated it, because it went past 10 activation threshold, Microsoft detected it that it was being abused and blocked the key from further use.
