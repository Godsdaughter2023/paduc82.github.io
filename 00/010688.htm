Any advice would be greatly appreciated. Please let me know if you need additional information.

Using: Access 2010.

marshall barton say:

First  :-)  you can not use the First function to find the earliest date.  First finds the first date in the returned records and because records are returned in whatever order the database engine thinks is efficient, the records can be returned in any order, especially after many records have been added, deleted or edited (the same can be said about the Last function).  Instead of First, use the Min function to get the earliest date (Max for the latest date).

Second, just concatenating # signs around a date value will not always result in the date in the field.  The concatenation forces Access to convert the date value to a string that looks like a date.  But, Access uses each user's Windows regional settings to convert a date to a string.  That means users can change their settings to use non USA style dates and after you concatenate the # signs (to tell Access the date string is a USA style date) you end up with Feb 5 being changed to May 2.  The safe way to do what you want is to use the Format function so the date is converted to a string that is the correct representation of the date value.  For example, you could use:

Expr1: IIf([Final_Court_Date] Is Not Null, [Final_Court_Date], DMin("[Mondays]", "Monday_Court_Dates", "[Mondays] >" & Format(DateAdd("d",190,[Violation_Date]), "\#yyyy-m-d\#")))

Or, you can use the Nz function to make that a little simpler:

Expr1: Nz([Final_Court_Date], DMin("[Mondays]", "Monday_Court_Dates", "[Mondays] >" & Format(DateAdd("d",190,[Violation_Date]), "\#yyyy-m-d\#")))

If using the Format function doesn't fix the syntax error, post back with the new expression you are then using.
