larry val say:

I have a very large MS Word 2010 doc - how do I split it into 2 separate documents?

I have a very large MS Word 2010 document - how do I split it into 2 separate documents?  
deanh say:

How large is very large?

MS Word can handle documents of thousands of pages as well as a fairly large file size.

If your problem is file bloat then see the following suggestions that can help.

Sometimes file bloat can be indicative of the file starting to corrupt and that none of the below suggestions worked, then possibly the copying all but the last paragraph mark into a new document is a common cure for common bloat.

 

Forgot to say, to split into 2 documents, if you still need to that is; open the file, do a save as, rename file appropriately, decide on a point of separation, delete one half of the document, save. Open the other version, delete the other half, save. Job done.

 

There are a number of reasons for excessive file size, including:

1. Fast Saves: Disable this at on the Save tab of Tools | Options.

2. Preview Picture: Clear the check box on the Summary tab of File | Properties.

3. Versions (File | Versions): Make sure "Automatically save version on close" is not turned on.

4. Revisions (Tools | Track Changes):

   Highlight Changes: Make sure "Highlight changes on screen" is turned on (or that "Final Showing Markup" is displayed).

   Accept/Reject Changes: If "Accept All" or "Reject All" is available then revisions are present; accept or reject all changes, then turn Track Changes off.

5. Keep track of formatting (Tools | Options | Edit). This is reportedly a major cause of file bloat in Word 2002 and above.

6. Embedded True Type fonts (Tools | Options | Save); embedding fonts should be avoided wherever possible.

7. Embedded linguistic data (Tools | Options | Save).

8. Embedded graphics: When feasible, it is preferable to link the graphics.

That is, when you insert the graphic, click the arrow beside Insert in the Picture dialog and choose Link to File rather than Insert or Insert and Link.

9. Embedded objects: These are even worse than ordinary graphics saved with the document. If you see an { EMBED } code, the graphic is an OLE object.

Unless you need to be able to edit the object in place, unlink it using Ctrl+Shift+F9.

10. File format: Make sure you are saving as a Word document; in some cases .rtf (Rich Text Format) files are significantly larger than .doc files.

11. Document corruption: See http://www.mvps.org/word/FAQs/AppErrors/CorruptDoc.htm.

Create a Default Template in PowerPoint 2010 for Windows - Microsoft Community

geetesh bajaj say:

Create a Default Template in PowerPoint 2010 for Windows

Technical Level : Intermediate

Summary

Learn how you can set up any template or Theme in PowerPoint 2010 to be the default for new presentations you create -- this can be very helpful when you want to use your corporate, organization or company template as the default.


Details

Do you want to use a template or Theme other than the blank white design that PowerPoint 2010 provides by default? True, the basic white design is a great starting point -- but there are situations when you need to use the approved template of your company or organization for all your presentations. If that's true, follow these steps to create a default template:

  1. Open the template file or even any actual presentation that is based on the template you want to use as your default.

  2. Thereafter choose the File menu to bring up Backstage view. Now choose the Save As menu option.

  3. Within the resultant Save As dialog box, select the PowerPoint Template (*.potx) option from the Save as type list.

  4. PowerPoint will on its own choose the correct folder for your template. Do not change the folder at this point of time and provide Blank.potx as the name for your template. Remember, the template must be named Blank.potx and nothing else!


  5. Click the Save button.

Now whenever you create a new presentation, it will be based on your default template.

Note: Is your template not working as the default? That may be because you may have set a default Theme -- to learn more about what's different between a default template and a default Theme -- and to learn how you can create a default Theme, look at the Change the Default Template or Theme in PowerPoint 2010 page (external link).

Well first you don't want to copy data from one place to another. That creates redundant data.

I assume your contact database has a title field and you use that to search for Treasurers. So how to handle this depends on how you search for a contact. One scenario might work like this:

Using a "search" combo to select a contact, you would add the title field to the Rowsource of the search combo. Lets say this is the 4th column. In the After Update event of the search column you might add code like:

If Me.cboContactSearch.Column(3) = "Treasurer" Then
    Me.cboTitleSearch.Enabled = False
Else
    Me.cboTitleSearch.Enabled = True
End If

This will disable the title search if the contact has the title of Treasurer.
