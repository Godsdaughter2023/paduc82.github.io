karl timmermans [outlook mvp 2012-2018] say:

You can't create a "contact's" folder in an IMAP folder tree.

Two options

#1 - Move all the contacts to the contact folder marked "This Computer Only". Those contacts will appear in the AB

or - the recommended approach

#2 - Open the PST file in from your old system that contains your contacts folder and make it the "default" for the profile at which point new contacts will be added to this folder by default and should also appear in the AB (if not - right-click as noted in your post)

So why is option #2 my recommended approach?

O'2013 keeps IMAP data in an .OST file which can only be opened/accessed by the email account/Outlook profile that "originally" resulted in the creation of the file. If the email account is deleted, you will lose all local data contained in the .OST file. Simply re-adding the same email account back in the same Outlook profile will not result in a previously created .OST file being used - a new one will be created with an <empty> "Contacts - (This Computer Only)" folder

If you want to eliminate the extraneous folders marked with "This Computer Only" - then <AFTER> you have assigned a PST file as the new "default data file" for the profile which will now be the home for you default contact/calendar folders etc <AND> you have moved whatever is required from the IMAP OST file to the equivalent folders <THEN> you can delete and re-add the IMAP email account to the profile. A new .OST file will be generated, the email folders will get sync'd and you won't have the extraneous "This computer only" folders in the IMAP folder tree. You will have extraneous folders still showing in the default data file (i.e. <Inbox> etc but those won't be used for anything)

This sounds like it is related to your previous question.  What is not working?  When you color a series the legend entry for that series automatically gets that color.

drawyah say:
Looks like I've answered my own question and fixed the problem
 
WM6 & AS 4.5 must be using the http service
 
I couldn't find the http service listed in Services in the control panel So I had a look thru the registry and I found
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\HTTPservices
under there is a key ImagePath which is set to System32\Drivers\HTTP.sys
 
So I went looking for this file. Someone (probably me) had renamed it to http.old
 
Renamed it back to http.sys reconnected the PDA and it sync'ed up 1st time.
 
Thanks to everyone for looking
 
Hope this is use to anyone with the same problem
 
Richard
