In the userform code, you would need something like the following. The example assumes two text boxes a check box and two command buttons all with default names.
The code determines the next row of the table (it assumes a header row at least is present in row 1 and that there are values in Column A).

When CommandButton1 is clicked the values in the fields are written to the next row of the worksheet - here Columns A, B & C. (1, 2, & 3).

The other command button (Cancel) unloads the form.

The last piece of code disables the red cross in the top right of the userform.

You can use this to relate to your own textboxes etc. by changing the names and addresses as appropriate


Option Explicit

Private Sub CommandButton1_Click()
Dim xlSheet As Worksheet
Dim NextRow As Long
Set xlSheet = ActiveWorkbook.Sheets(1)
NextRow = xlSheet.Range("A" & xlSheet.Rows.Count).End(-4162).Row + 1
With xlSheet
    .Cells(NextRow, 1) = Me.TextBox1.Text
    .Cells(NextRow, 2) = Me.TextBox2.Text
    .Cells(NextRow, 3) = Me.CheckBox1.Value
End With
Unload Me
End Sub

Private Sub CommandButton2_Click()
Unload Me
End Sub

trevor rabey say:

Your question identifies important issues in progress tracking and progress measurement, and you should get very clear about how to do this so that you get valid results, before you start heading down the wrong track.

First, you don't have to care much about "ahead %" or behind %". It is not a good measure of progress. It will always be ambiguous and subject to mis-interpretation.

You don't have to input % of anything, and you should not.

MSP will calculate all percents (% complete which is about duration and % work complete) provided you have a good robust procedure for tracking.

Have you set a status date in project information?

Have you displayed the status date as a vertical red line in the chart (format,gridlines)

Have you input actual start and actual duration in the tracking table?

Insert the status field and the status indicator field in the table (entry or tacking, whatever) but it won't tell you anything that you can't just see just by looking at the tracking table and bars in the chart.

ken sheridan say: The place to sort rows is in a query, not in a table's datasheet.  As the column by which you want to order the rows is alphanumeric, it will sort alphabetically, so to sort in numerically you'd have to parse the value in the query's ORDER BY clause, e.g.

SELECT *
FROM [YourTableNameGoesHere]
ORDER BY LEFT([YourColumnNameGoesHere],1), VAL(MID([YourColumnNameGoesHere],2));

This assumes that the column's value is always a single letter followed by a number.

ken sheridan say:

........but my column's value will have a TR1 (example) for trailers in it.

Then you need to parse the value into the alphabetic and numeric elements.  Add the following functions to a standard module:

Public Function GetAlpha(strVal As String) As String

    Dim i As Integer
    i = 1
   
    Do While Not IsNumeric(Mid(strVal, i, 1))
        GetAlpha = Left(strVal, i)
        i = i + 1
    Loop

End Function


Public Function GetNumber(strVal As String) As Integer

    Dim i As Integer
    i = 1
   
    Do While Not IsNumeric(Mid(strVal, i, 1))
        GetNumber = Val(Mid(strVal, i + 1))
        i = i + 1
    Loop

End Function

Then call the functions in the query:

SELECT *
FROM [YourTableNameGoesHere]
ORDER BY GetAlpha([YourColumnNameGoesHere]), GetNumber([YourColumnNameGoesHere]);
