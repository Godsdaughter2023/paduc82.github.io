ArrayList is not part of VBA, so I assume you made a reference to the .NET System.Collections namespace, or you're using GetObject?

jeeped say:
You are bouncing back and forth between the recorded macro method of selecting something before using it and real VBA programic direction of commands without selecting. Further, the rng2 is an integer representing the last non-blank row in Sheet2's column A. It cannot be used as a destination without applying it in a Range construct and you probably want to offset it down one row so you are using teh next blank row and not overwriting the last non-blank row.

If I am interpreting your intentions correctly, try this:

Sub copypastesp()
    Dim c As Range
    Dim rng1 As Long
    Dim rng2 As Long
    Dim cols As Long
    Dim s1 As Worksheet
    Dim s2 As Worksheet
    
    Set s1 = Sheets("sheet1")
    Set s2 = Sheets("sheet2")
    
    rng1 = s1.Cells(Rows.Count, "A").End(xlUp).Row
    cols = s1.UsedRange.Columns.Count
    
    For Each c In s1.Range("A2:A" & rng1)
        If c.Value = "BAC" Then
            rng2 = s2.Cells(Rows.Count, "A").End(xlUp).Row + 1
            s2.Range("A" & rng2).Resize(1, cols) = c.Resize(1, cols).Value
        End If
    Next c

    Set s2 = Nothing
    Set s1 = Nothing
End Sub

jeeped say:
       **   rng2 = s2.Cells(Rows.Count, "A").End(xlUp).Row + 1

This starts at the last row in column A (A1048576) and looks up to see the first populated cell. This is better than starting at A1 and looking down because there might be a blank cell between cells with values. It takes the row number and adds one so that when it is used ("A" & rng2) to construct a range reference, it will be describing the first blank row and not the last row with a value in column A.

      ** s2.Range("A" & rng2).Resize(1, cols) = c.Resize(1, cols).Value

The For Next, If Then code that wraps this has gone through each cell in Sheet1's column A and determined whether or not it contains the text string "BAC". If we have reached this point it does contain "BAC" and we need to copy it and any associated data in columns to the right over to Sheet2 (aka s2).

The number of columns containing data was assigned to cols in this statement above,

    cols = s1.UsedRange.Columns.Count

Instead of copying and pasting data, we will perform a direct value assignment from one sheet to the other. This is exactly the same as Copy, Paste Special, Value. and a little more efficient I believe. The only trick to this method is that the source and the target need to be exactly the same size and we have cols for the number of used columns so we can use .Resize() to expand the cell noted by c to a range of cells.

So rng2 is the first unused row in Sheet2's column A and cols is the number of cells to copy/assign across. The rest kind of speaks for itself.
