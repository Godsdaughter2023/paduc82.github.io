    steve rindsberg say:
    Alt+F11 accomplishes two things in this case:

    1) The fact that it opens the VBA editor (aka IDE) proves that you do have VBA installed; in some versions of Office, it's an optional install item; if it's not installed, there'd be no sense in displaying the Dev tab.

    2) You now have a workaround that gets you into the IDE w/o needing the Dev tab until you figure out why it won't appear.

    Here's something to have a look at in the registry if you're so-minded:

    HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\PowerPoint\Options

    Under Options, there's a DeveloperTools (REG_DWORD)
    If it's set to 1, I have a developer tab.  If it's set to 0, I don't.

    Looks like there's the same DeveloperTools entry under Excel\Options and Word\Options

    If you decide to try changing this, do it while PowerPoint and the other Office apps are shut down, otherwise, the changes won't take effect and won't stick.
