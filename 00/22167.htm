suzanne s. barnhill say:
As I point out in http://wordfaqs.mvps.org/BottomLine.htm, 24-pt line spacing is the equivalent of double spacing on a typewriter and should satisfy your publisher. "Single" line spacing varies depending on the amount of leading built into a font. Line spacing for Times New Roman is approximately 120% of the nominal point size, which means that it is 12 points for 10-point TNR and about 14.4 points for 12-point. Word's "double" spacing for TNR always looks too spacey for me; I routinely use Exactly 24 points for "double-spaced 12-point TNR text. I've never had anyone object (and I have produced copy for many publishers).

peterfarrelly say:

I have had this problem too with Windows 7 and Office 365.  I was comparing fonts using paragraphs of lorum ipsum text and single line spacing. Different versions of the same typeface were very different in line spacing, varying by nearly 2 points for a 12pt setting.  This is a property of the individual font. When a type designer digitises a font, the designer makes decisions about the character spacing and line spacing. As well, applications like MS Word make their own decisions; different versions of MS Word have used different values like 1.0, 1.08, 1.1, or 1.15 times the type size. Some applications (e.g. Serif PagePlus) tell you what "single" spacing means. The same text set as "Single" in MS Word will also be Single in PagePlus, except that text in Calibri will be set in lines of 14.656pt, and the same text in lines of 13.792pt if the font is in Times New Roman.  Based on 3 paragraphs of 507 words, 3423 characters (incl. spaces), the Calibri setting is 6% longer than the Times New Roman setting, which is about the same as the difference in the line setting. 

One should remember the the Single and Double line settings are carryovers from typewriters. You will be better off testing different settings yourself and satisfy yourself on matters like readability, or page colour, and leave the "Single" line setting alone.

karl timmermans (outlook mvp 2012-2018) say:
>>
Please advise why it doesn't change to an excel format but keeps the commas as separators even though I save it as an excel workbook.
<<

Hello AngelaH7

Going by your question, am going to assume that you are in a region where a character other then a comma is used as the list separator (field delimiter) character. - is that correct?

Only Excel used the List Separator as the field delimiter. Whenever 3rd party programs create a CSV file, in 99.9% of the cases, those CSV files will be created using a comma. A CSV file is a text file and 3rd party programs do not use a region's list separator character.

The above explains the why. A quick work-around to deal with a CSV file created using a comma as the field delimiter character in a region that uses something else (most common would be the semi-colon ; character) is as follows



#1 - With Excel closed, open Control Panel --> Select <Clock & Region> --> Select <Region> --> Change the format to any region that uses a comma - i.e. English - United States. Close Control Panel

#2 - Open the CSV file in Excel and save it as an Excel workbook. Close Excel

#3 - Open the Control Panel and change your region back to what it was and then close Control Panel

#4 - Open Excel and the workbook you just saved. If required, re-save it as a CSV file if you want that file to be saved using the list separator character for your region. *** most programs outside of Excel (including Outlook for importing) will expect any CSV file to be saved using the comma as the separator
