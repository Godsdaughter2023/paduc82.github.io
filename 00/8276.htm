john w. vinson say:

Not quite. You do need to call the Format() function is you want to format the field; and the third argument to DLookUp needs to be a valid SQL WHERE clause specifying which record in tblMealPrices you want to look up. "Breakfast" is not a valid SQL WHERE clause;

 

"[Meal] = 'Breakfast'"

 

might be, though I don't know the structure or fieldnames of your table. Try

 

IIf([Breakfast]=-1,Format(DLookUp("Breakfast","tblMealPrices","[Meal] = 'Breakfast'"),"Currency","$0.00")

 

to display the text string $3.75 if that's the price of breakfast, or $0.00 if the field [Breakfast] is not true.

 

If tblMealPrices is incorrectly normalized and has fields named [Breakfast], [Lunch] etc. and has only one record, then you don't need any criterion at all:

 

IIf([Breakfast]=-1,Format(DLookUp("Breakfast","tblMealPrices"),"Currency"),"$0.00")
