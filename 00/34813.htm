Documents Files - Microsoft Community

indy mike say:

Documents Files

I am set up as admin on this computer and now have several document files ( My Documents, Document Library ect) and want one document file. How do I do this without causing a problem on my computer?

ronnie vernon say:

I am set up as admin on this computer and now have several document files ( My Documents, Document Library ect) and want one document file. How do I do this without causing a problem on my computer?


Hi Mike

The default folders configuration on Windows 7 takes some explaining to fully understand what you are seeing.

1. Libraries: Libraries are 'virtual folders'. They do not physically duplicate folders or files, they simply 'mirror' the data that is in the actual folders. They also do not take up any space on the hard drive.

By default, the Documents Library has 2 folders, the logged on users My Documents folder and the Public Documents folder (All Users). If you open this library and click the Locations item at the top/left of the View window, you will see these 2 folders have been added to that library.

When you view the properties of a folder or file in a Library, you wil see the information from the actual, physical folder or file. You can open, edit, rename, move, or delete a file or folder from within a library and the action will take place on the physical file.

Why Libraries: Because recently more and more users want to keep all of their personal data on a separate drive to reduce the space used on the system drive, make backing up their personal data easier, and protecting that data in case the system drive crashes.

2. Junctions: If you have the option to Show hidden files, folders, and drives enabled, you will see what looks like duplicates for many of the common user folders in several different locations. These Junctions have a couple of things in common. They have a folder icon with a shortcut arrow overlay, and they cannot be accessed.

They look like this:

 

Why Junctions: Since Windows XP, there have been many changes to the file system in Windows. Many of the default user folders have had name and location changes. The problem is that there are thousands of older, legacy, software programs that are hard coded to look for the old folder names and locations. These programs will fail to install if they cannot find the folder in the old location.

For backward compatibility, Junctions are placed in those older locations and they redirect these older software installations to the new location and allow them to install properly. This is the sole purpose for a Junction.

Junctions have their permissions set to Everyone = Access Denied. This prevents them from being moved, deleted, or changed. Junctions do not contain any files or data. If the permissions are changed on a Junction, this can have unexpected and unpredictable results, such as installed programs stop working and the file system being corrupted.

Hope this helps.

Regards,

Ronnie Vernon MVP – Windows Desktop Experience

Note:

Event logs are special files that record significant events on your computer, such as when a user
logs on to the computer or when a program encounters an error. Whenever these types of events occur,
Windows records the event in an event log that you can read by using Event Viewer. Advanced users might
find the details in event logs helpful when troubleshooting problems with Windows and other programs.
Event Viewer tracks information in several different logs. Windows Logs include:

• Application (program) events: Events are classified as error, warning, or information, depending on
 the severity of the event. An error is a significant problem, such as loss of data. A warning is an
 event that isn't necessarily significant, but might indicate a possible future problem. An information
 event describes the successful operation of a program, driver, or service.
•Security-related events: These events are called audits and are described as successful or failed
 depending on the event, such as whether a user trying to log on to Windows was successful.
•Setup events: Computers that are configured as domain controllers will have additional logs displayed
 here.
•System events: System events are logged by Windows and Windows system services, and are classified as
 error, warning, or information.
•Forwarded events: These events are forwarded to this log by other computers.
