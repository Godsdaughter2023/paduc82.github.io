greg maxey say:

Word documents are made up of sections.  Each sections has three headers ranges (first page, primary page, and even page) regardless if they are used and contain text.  To get the first two lines (provided that lines are defined by separate paragraphs) of section 1, primary page header use:

Sub ScratchMacro()
'A basic Word macro coded by Greg Maxey
Dim oRng As Word.Range
  Set oRng = ActiveDocument.Sections(1).Headers(wdHeaderFooterPrimary).Range
  ActiveDocument.Variables("HeaderLine1").Value = Left(oRng.Paragraphs(1).Range.Text, Len(oRng.Paragraphs(1).Range.Text) - 1)
  ActiveDocument.Variables("HeaderLine2").Value = Left(oRng.Paragraphs(2).Range.Text, Len(oRng.Paragraphs(2).Range.Text) - 1)
  Debug.Print ActiveDocument.Variables("HeaderLine1").Value & vbCr & ActiveDocument.Variables("HeaderLine2").Value
End Sub

a. user say:

I would guess that the reason that no-one has attempted an answer is that everyone else's experience is similar to yours, i.e. even when you think you have set everything up how you want. something destabilises the arrangement and you (more or less) have to start over. 

If you are the only author/maintainer for the document(s) you are creating, in my view you will be able to create a much more stable and predictable system if you use numbering based on SEQ fields. It's not ideal (e.g. you have to select and update all the field codes whenever you do an insertion/deletion, and you may not be able to take advantage of some of Word's built-in features such as cross-referencing) but you do get more control. If other people are working on the document(s) it's not likely to go so smoothly.

e.g. if you just have two levels of heading, and you want

1. top level

  (a) second level

  (b) second level


2. top level

  (a) second level

etc., you can use the following fields each time you insert a "top level" heading

{ SEQ L1 }{ SEQ L2 \r0 \h }. Your top level heading 1

and the following fields each time you insert a "second level" heading

({ SEQ L2 \*Alphabetic }) Your second level heading. 

If you want

1. top level

  1.1 second level

  1.2 second level


2. top level

  2.1 second level

You can use

{ SEQ L1 }{ SEQ L2 \r0 \h }. Your top level heading 1

{ SEQ L1 \c }.{ SEQ L2  }. Your second level heading. 


and so on.

stefan blom say:

Generally, to have style A restart the numbering of style B, these two styles must be part of the same multilevel list, and style A must be higher up the hierarchy than style B. You can have another style, say C, mimic the behavior of a numbered style if you base style C on some numbered style.

To get started, you first have to set up a multilevel list. See https://www.shaunakelly.com/word/numbering/numbering20072010.html. The article uses the built-in headings as examples, but you can use any set of nine styles. For example, you can mix heading and non-heading styles if you wish.

Let us assume that we have a multilevel list where level 1 is Heading 1 and level 2 is Heading 2. You can then create a new style which is based on Heading 2, and that new style will also restart following a Heading 1 paragraph.
