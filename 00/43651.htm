Hello,

Thank you for keeping us updated about the issue.

This issue happens when the Display Driver installed in the computer is not able to handle the "Fade effects" set for the right click Context menus in Windows. This management is done by the "Desktop Window Manager" process running in the computer all the time.

There are two workarounds that can help in this scenario:

Note: These are not resolution but only workarounds.

Method 1

Try to reset "Desktop Window Manager" process when this issue occurs:

  1. Click on Start.
  2. Type run in search box
  3. Type in "tskill dwm" (without quotes) and hit Enter.

Method 2

Try to Disable Fade Effects for menu items:

  1. Click on Start.
  2. Type run in search box
  3. Type "sysdm.cpl" (Without quotes) and hit Enter
  4. Now open advanced tab
  5. Click "Settings" under "Performance"
  6. Then Uncheck "Fade out menu items after clicking"
  7. Click Apply and hit OK

However if these workarounds didn’t work then try to install a graphics driver that can handle these fade effects properly.

andre da costa say:
 there is nothing wrong with using it, but it has some restrictions in regards to licensing;


OEM versions of Windows 7 are identical to Full License Retail versions except for the following:


- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel


- OEM licenses are tied to the very first computer you install and activate it on


- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard


- OEM versions cannot be used to directly upgrade from an older Windows operating system


The OPK is not required, but here is some information about it:

OEM licenses are to be installed by professional system manufacturers only. Under Microsoft's OEM License Agreement, they are not to be sold to end-users under any circumstance, and are to be preinstalled on a computer using the OEM Preinstallation Kit (OPK) before shipment to the customer, and must include at the very least the manufacturer's support contact information. They are, therefore, designed for installation only on a single computer and are not transferable, even if the original computer is no longer in use. This is not usually an issue for users who purchase new computer systems, because most pre-assembled systems ship with a preinstalled operating system. There are few circumstances where Microsoft will allow the transfer of an OEM license from one non-functioning system to another, but the OEM System Builder License Agreement (SBLA), as well as the OEM End User License Agreement (EULA) do not contain any allowance for this, so it is entirely up to Microsoft's discretion, depending on the situation

http://www.microsoft.com/oem/en/licensing/sblicensing/pages/licensing_for_hobbyists.aspx



There are two types of OEM vendors. Royalty OEM vendors can distribute Windows products by using System-Locked Preinstallation (SLP) Product Keys so that end users can bypass product activations. All other OEM vendors distribute Windows products by using the standard Product Keys that require the user to activate the Windows products.


　

DSP OEI means "Delivery Service Partner, Original Equipment Installer." It is an OEM version of Windows distributed to smaller computer manufacturers and system builders. According to the license, it is only supposed to be pre-installed on a new PC that will be resold.

dominicp say:

Sometimes there can be a long delay (several minutes) before installation proceeds after the initial background appears.

Changing from 32 to 64-bit Windows (or vice versa) is always a clean install. Upgrading is not possible.

You need to back up your data and settings with the Easy Transfer Wizard http://windows.microsoft.com/en-US/windows7/products/features/windows-easy-transfer before installing 64-bit Windows 7.

 

Check that your motherboard does not need a BIOS update or other changes to its settings.
