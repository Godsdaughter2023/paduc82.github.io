isatv say:

ken sheridan say: All date time values are stored as a 64 bit floating point number.  The format in which you see the value is just that, so the same value could be shown as US Format(mm/dd/yyyy), UK format (dd/mm/yyyy), ISO format(yyyy-mm-dd) etc.  By default dates are shown in the short date format set in Windows Control Panel.

To show a date in the format of your choice in a control in a form or a column in a table's datasheet set the Format property of the control or column to whatever format you wish, e.g. dd/mm/yyyy.  This will omit the time of day, even if it is non-zero, but bear in mind that the value of the control will still include the time of day, which is represented by the fractional part of the underlying numeric value.  If you wish the value to be the date only you must convert the value to an integer, so the expression would be like this:  =Int([MyDateTimeField]) and format it as dd/mm/yyyy or similar.

Note, however, that a date literal must be in US short date format or an otherwise internationally unambiguous format.  If I were to use the following expression for instance:

=DLookup("ContactsID", "Appointments", "ApptDateTime = #" & Me.txtApptDateTime & "#")

and the appointment I was seeking was at 11.00 AM on 4th July 2017 the expression would give the wrong result because it would interpret 04/07/2017 in UK format as if it were in US format, i.e. 7th April.  The value must therefore be formatted accordingly, for which I'd use the ISO format:

=DLookup("ContactsID", "Appointments", "ApptDateTime = #" & Format(Me.txtApptDateTime,"yyyy-mm-dd hh:nn:ss") & "#")

andreas killer say:
You can use a CASE statement like this:
   Select Case R
        Case "Andreas Killer"

But it is wise to have all names hardcoded in the VBA code?
What about to create a setup sheet with all names that should grouped togeher?

Let us assume that this sheet as this values:

A1:  Group1
B1:  Patrick Retelle
C1:  Jen Jones
A2:  Group2
B2:  Bob Smith
C2:  Andreas Killer
D2:  Dan Miller

Each row holds all names for one group.

So if we evaluate the data, we can search the Setup sheet for the name we found in the data, get the group name and add the values to that group.

Andreas.


Sub Test()
  Dim R As Range, C As Range, First As Range, Last As Range
  Dim Dict As Object 'Dictionary
  Dim GroupName As String
  Dim Result(1 To 4) As Double
 
  'Get the first value (this is "001")
  Set First = Range("B1")
  'Get the line with "Total"
  Set Last = First.End(xlDown)
 
  Do While Last.Row < Rows.Count
    'Initialize our dictionary
    Set Dict = CreateObject("Scripting.Dictionary")
    'Visit each data cell in column C (line after "001" before "Total")
    For Each R In Range(First.Offset(1, 1), Last.Offset(-1, 1))
      'Search the name in Setup
      Set C = Sheets("Setup").Cells.Find(R.Value, LookIn:=xlValues, LookAt:=xlWhole)
      'Found?
      If C Is Nothing Then GoTo SkipName
      'Get the group for this name
      GroupName = C.Offset(0, 1 - C.Column)
      'Group already found?
      If Not Dict.Exists(GroupName) Then
        'No, associate the group with this value
        Dict.Add GroupName, R.Offset(0, 1).Value
      Else
        'Yes, sum the values
        Dict(GroupName) = Dict(GroupName) + R.Offset(0, 1).Value
      End If
SkipName:
    Next
   
    Select Case Left$(First, 2)
      Case "00"
        If Dict.Exists("Group1") Then _
          Result(1) = Result(1) + Dict("Group1") 'X2
        If Dict.Exists("Group2") Then _
          Result(2) = Result(2) + Dict("Group2") 'Y2

Please let me know whether it is possible.

Thanks a lot :)

 

brian tillman [outlook mvp 2007-2019] say:

You cannot report any message as junk in any version of Outlook because there's nothing to receive any such reports.  Also, you cannot mark any messages as junk in any version of Outlook.  Outlook does not have a learning filter.  Your options are to add the particular sender to the Blocked Sender list or add the particular sender's domain to the list.  Since these options work on specific addresses, it's not possible to select multiple messages, since the senders and/or their domains are likely all different.

On top of this, adding particular sender addresses to the Blocked Sender list is nearly pointless in combating junk mail because junk senders are constantly changing the sending address.  Adding a domain to the Blocked Sender list is marginally more useful.

Brian Tillman [MVP-Outlook]
