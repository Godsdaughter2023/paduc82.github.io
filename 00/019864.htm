Ok, first thing I should have said was to check you're aware of the Developer Tools in Internet Explorer?
In IE, go Tools > Developer Tools, then Find > Select element by click...
Then go to IE and click on the element of interest.  Switch back to the Developer Tools and the relevant piece of html will be highlighted.
That's how I found the "cite" element id.

I couldn't figure out (a) how to get only the main links and not the "on-every-page" google links.  That's why there are so many more cells filled in columns A and B than in C, and (b) I couldn't figure out how, using the HTML object model, one can extract the href address, so I think you'll have to do that in vba using the likes of InStr, once to find the start of the URL and a second time to find the end (search for ">  starting at the start of the URL).

Now, to loop through several pages, you just need to repeat the process while feeding in the URL for the next page in google.
The tag  &start=10  at the end of the google URL dictates what page is returned (depending on the settings you have applied in google's settings for number of returns per page, which defaults to 10, and you can't change this setting if you have google instant enabled).  Note the first result is number zero so 10 returns the second page, 20 returns the third page, etc.  If you disable google instant, you can use
http://www.google.co.in/search?q=obama&start=0&num=50  to return 50 results on one page, and to disable google instant you add   complete=0    to the URL, so you could use:
http://www.google.co.in/search?complete=0&start=0&num=50&q=obama

So you should be able to use this (not tested) to loop through pages:

    Dim strSearch As String
    Dim lnPage As Long
    strSearch = "obama"
    For lnPage = 0 To 4
        IE.navigate "http://www.google.co.in/search?q=" & strSearch & "&start=" & lnPage & "0"
        'etc...
    Next lnPage

or just use this to load one page with 50 results:

        IE.navigate "http://www.google.co.in/search?complete=0&start=0&num=50&q=" & strSearch

Hope that helps (some more).
Do post back if you ever figure out how to extract the url by using the HTML object model, and if you figure out how to identify a result vs. the general google chaff.
