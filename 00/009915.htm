bob jones aka: cybertaz say:
OK, let's simplify it if we can :-)

First, the Default ACL [language specific] files are not what you are looking for, as John indicated. These are global ACL files used by all Office apps in all accounts on the same Mac, and contain the built-in Auto-correct entries. These are the ACL files located in:

Macintosh HD:Applications:Microsoft Office 2011:Office

Regardless of what Finder indicates, these are not user accessible files & do not store any AutoCorrect entries you create. Nor are they involved if you modify/delete any of the built-in AutoCorrect entries. [See Note below]

Second, User AutoCorrect modifications are stored in 1 of 2 locations -- which location is dictated by whether it is an unformatted entry or it is a formatted entry.

Location 1: If it is an unformatted entry [e.g., Replace: pront With: print] it is stored in the Microsoft Office ACL [language specific] for the user account in which the entry is created. The AutoCorrect entries stored here are available to all Office apps. These ACL files are located in:

Macintosh HD:Users:username:Library:Application Support:Microsoft:Office:Preferences:Office 2011

The Microsoft Office ACL files also contain the changes that user makes to the built-in AutoCorrect list stored in the Default ACL file 9essentially, I believe, as exceptions). This is the file you need to copy to a different User Account or different installation of Office in order to have the same custom unformatted AutoCorrect entries.

Location 2: Formatted AutoCorrect entries [e.g., Replace: print With: print [bold, italic, red)] are stored in the Normal.dotm template for that user account, so they are only available in Word. In order to have these available to another user a copy of the Normal.dotm containing them needs to be used to replace the other user's Normal.dotm template.

There are some technical differences, but the process works the same in the Windows versions.

Note: If you double-click one of them the OS will attempt to open it using the associated app, but it will fail to open. My guess is that since they are a proprietary format used only by Office apps, OS X really doesn't know what type of file they are. It has to identify (register) them in some way, though, so it just guesses & assigns a parent app based on some sort of 'nearest neighbor' algorithm. Installed on OS X 10.6.8, mine are shown as Entourage Preferences files, On my MBP installed on 10.5.x & later upgraded to 10.6.x the ACL files are identified as AdobeDevice Central Preferences Document files... go figure‽ In any case, the ACL files can be opened by specifying TextEdit, but IMO trying to edit them is risky business.

anula d'souza say:

Hi Stephen,

Thank you for posting in Microsoft community. Before starting the troubleshooting steps just a quick question:

Is the system on network or domain?

 

Open Excel in safe mode and check if Excel Works fine. Follow the steps listed below to open Excel in safe mode:

·         Close Excel.

·         Click on Start< All Programs< Accessories< Run

·         In Run window and type Excel /safe

Note: There is a space between Excel and /.

 

If you are able to work with Excel without any problem then the issue might be due to add-ins. In order to disable the add-ins follow the steps given below:

·         In Excel, click on File< Options< Add-ins.

·         Under Manage select COM Add-ins.

·         Click on GO options beside Manage, uncheck all the add-ins and click Ok.

·         Try to open Excel and check if it works fine.

 

I hope the above suggestions helps to resolve your issue. If you need any further assistance reply and we will be glad to assist you.

Thanks.
