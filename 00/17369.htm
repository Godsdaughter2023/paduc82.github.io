bob jones aka: cybertaz say:

Yes, it is a bit confusing in the Mac version as well.

All Styles have an option to be added to the Quick Style List which is presented as a gallery on the Home tab of the Ribbon. By default it contains the 15 Recommended Styles which also appear in the Style Task Pane. A user-defined Style automatically is included in the Style Task Pane but is not added to the Quick Style gallery unless the box is checked [Add to Quick Style list] in the New/Modify dialog. This is all reasonably straightforward.

The confusion lies with what is found on the Design tab. This is where you find the group named Document Formatting. That group also is presented as a gallery which contains a variety of what are hinted at as being Style Sets. (The only clue is that there is a listing in the dropdown for that gallery to Save as New Style Set).

Each of these sets contains nothing more than variations on the same 15 Recommended Styles referred to above. Also, custom styles or those built-in styles modified to include the Add to Quick Style list property are not appended to any of them. From what I can tell, they cannot be modified. To have a set that includes other styles you must save a new set.

Part of the confusion stems from how the Document Formatting dropdown is labeled. It consists of 3 segments plus 2 commands at the bottom [Reset to the Default Style Set: Save as New Style Set]. The segments are labeled:

  1. This Document, which simply shows a representative thumbnail. When you point to it, though, the tip generically states This Document's Style Set, not the actual name of the set.
  2. Custom, which contains 12 thumbnails representing what actually are the built-in sets. These have names such as Elegant, Manuscript, Perspective, etc. & those names appear in the tip when you hove over each. However, saved custom sets are not added to that grouping.
  3. Built-In, which actually is where any user-defined custom sets appear.

My only guess as to the Custom segment is that it means sets that differ from the default even though not created by the user. I have not found any documentation at all, let alone any that contradict/confirm my impressions on any of this.

Try this modification

Public Sub InsBelow()
Dim lngStartRow As Long
Dim lngEndRow As Long
Dim m As Integer
Dim n As Long
With ActiveSheet
lngStartRow = Range("T" & CStr(Application.Rows.Count)).End(xlUp).Row
lngEndRow = 1
For n = lngStartRow To lngEndRow Step -1
If IsNumeric(Range("T" & CStr(n)).Value) And Range("T" & CStr(n)).Value <> "" Then
    Rows(n + 1).Resize(Range("T" & n).Value - 1).Insert
     Rows(n).Resize(Range("T" & n).Value).FillDown
        End If
    Next n
End With
End Sub
