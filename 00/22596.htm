ran mo msft support say:

Hi Pavle,

 

Yes, the cutover migration’s one prerequisite is that no Directory Sync, no existing users in Online. So, what you heard is correct. But considering your final deployment is pure online and Hybrid deployment is relatively complex which is idea for large scale of orgs, cutover is easier for you to perform.

 

What you need to do if you decide to do the cutover in your current deployment is disable Directory Sync, completely delete the users from Office 365 and change the MX to on-premises (after migration, can be changed to cloud again). Not much work to do before performing and the cutover migration itself is also easy. The following cmdlets are used to completely delete users from office 365.

Get-MsolUser | Remove-MsolUser –Force (Remove all users account)

Get-MsolUser -ReturnDeletedUsers | Remove-MsolUser -RemoveFromRecycleBin –Force

 

But anyway, it is fine to do the Hybrid if you wish. For the autodiscover authentication question, you can use the following cmdlet

Set-AutodiscoverVirtualDirectory -Identity 'Autodiscover (Default Web Site)' -WSSecurityAuthentication $true
