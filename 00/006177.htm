I am sure you've already figured this out by now, but for the benefit of anyone finding this in a web search... A "Guest mail user" is an external user who was invited to an Office 365 group, a SharePoint site, or (I think) a OneDrive folder.

When you send an invite to an external user for those things, a user is created in Azure Active Directory with the user type of Guest. You can't create them directly in Exchange admin dashboard, but you can from Azure Portal. But typically, these are created automatically when users in your organization send an invite.

Hello,

With the issue description, I understand that you are experiencing issue with msn homepage. I will certainly help you to fix this problem.

This behavior can occur if either of the following conditions exists:

English (India) [gb] is the only language selected in the Language Preference dialog box.
English (India) [gb] is a higher priority language than English (United States) [us] if both languages are listed in the Language Preference dialog box.

If some other language is selected as the default language, you may be redirected to the site corresponding to the default language.

I may suggest you to try the resolution from the Article below, use settings for English (India) at the Place of English( UK) as mentioned in the article.
