sue mosher say:

Sorting or grouping on the Read field is not possible, probably because the information about the read/unread state actually is calculated from a field that holds multiple bits of message state information. In other words, at the actual raw property level of Outlook, there is no single property that represents the read/unread state of an item.

In addition to GordonBP's good suggestion of using the Unread Items search folder, if you're interested mainly in a particular folder, you could create separate Read and Unread views for that folder, filtering on the Read field.

That is not the same as your original question, however ...
 
When you copy things to the clipboard, several things can happen depending
on what you have copied. In some circumstances the last thing you copied can
trigger that warning and the way to stop it is to ensure that the last thing
you copied is trivial. If you are working in the UI all you can do is swap
sometimes having to answer "No" to the dialog with always doing a more
complex procedure of selecting and copying a small amount of text before
closing.
 
If you are working in VBA, the procedure is the same - just before closing
make sure a small text-only string is copied to the clipboard as this will
overwrite the possibly more complex construct copied before. HOWEVER the
clipboard really belongs to the user and you should make serious efforts not
to use it in VBA yourself to begin with.
