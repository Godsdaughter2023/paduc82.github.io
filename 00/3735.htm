Excel formula: round up sum of several cells - Microsoft Community

ericchiarelli say:

Excel formula: round up sum of several cells

Hi,

I'm trying to figure out the formula to multiply a cell by a percentage and then round up the value to the nearest whole number. For example, I currently have the formula as =sum(g2*.1). The cell g2 in this particular example is 613, so instead of the result of this formula being 61.3, I need it to be 62.

Thanks!

Eric

joeu2004 say:

Eric wrote:

I'm trying to figure out the formula to multiply a cell by a percentage and then round up the value to the nearest whole number. For example, I currently have the formula as =sum(g2*.1). The cell g2 in this particular example is 613, so instead of the result of this formula being 61.3, I need it to be 62

Ostensibly:

=ROUNDUP(SUM(G2*0.1) ,0)

But you don't need to use the SUM function in this context.  The purpose of the SUM function is to, well, sum (add) values.  For example, instead of =A1+A2+A3+A4+A5, we could write =SUM(A1:A5).

You're mutiplying, not adding.  So your best formula is simply:

=ROUNDUP(G2*0.1 ,0)

Problems in using nested IF function - Microsoft Community

lina148 say:

Problems in using nested IF function

I am trying to use nested "IF" functions, excel is recognising and calculating correctly for first two "IF"s but then after that it uses the multiplier from second "IF" (C8) to calculate from there on totally disregarding  the formula in subsequent "IF"s (C9 etc). 

Formula: =IF(A18<=A7,A18*C7,IF(A18<=A8>A7,A18*C8,IF(A18<=A9>A8,A18*C9,IF(A18<=A10>A9,A18*C10,IF(A18<=A11>A10,A18*C11,IF(A18<=A12>A11,A18*C12,""))))))

How do I get the formula to calculate correctly?
joeu2004 say: Lina wrote:
I am trying to use nested "IF" functions, excel is recognising and calculating correctly for first two "IF"s but then after that it uses the multiplier from second "IF" (C8) to calculate from there on totally disregarding  the formula in subsequent "IF"s (C9 etc).  Formula: =IF(A18<=A7,A18*C7,IF(A18<=A8>A7,A18*C8,IF(A18<=A9>A8,A18*C9,IF(A18<=A10>A9,A18*C10,IF(A18<=A11>A10,A18*C11,IF(A18<=A12>A11,A18*C12,""))))))
How do I get the formula to calculate correctly?

The problem is your use of expressions of the form A18<=A8>A7.

I know what you want to write, but that is not the proper form.  To explain why you always get the second multiplier when A18>A7, the expression is evaluated as (A18<=A8)>A7.  That translates into either FALSE>A7 or TRUE>A7.  In either case, Excel treats a logic value (TRUE and FALSE) as greater than anything in A7 except an Excel error [1].

You might want to write AND(A18<=A8,A8>A7).  Or you might have meant AND(A18<=A8,A18>A7).

Your intent is unclear.  But based on the structure of the IF expression, I suspect you meant the latter.  In that case, your IF expression can be simplified as follows:

=IF(A18<=A7,A18*C7,IF(A18<=A8,A18*C8,IF(A18<=A9,A18*C9,IF(A18<=A10,A18*C10,
IF(A18<=A11,A18*C11,IF(A18<=A12,A18*C12,""))))))

It is not necessary to test A18>A7 in the second IF expression, for example, because that is implied by the fact that it failed the first test, A18<=A7.  That is, If A18<=A7 is false, A18>A7 must be true.

It might be possible to simplify your expression by redesigning it entirely.  I have not given that much thought.

Even if that is true in this case, I thought it was important for you to understand why your original expression failed and how to write nested IF expressions in the future, should the need arise.


-----
[1] Also TRUE is greater than an empty cell, but FALSE is not.

joeu2004 say: I wrote:
Your intent is unclear.  But based on the structure of the IF expression, I suspect you meant the latter.  In that case, your IF expression can be simplified as follows:

=IF(A18<=A7,A18*C7,IF(A18<=A8,A18*C8,IF(A18<=A9,A18*C9,IF(A18<=A10,A18*C10,
IF(A18<=A11,A18*C11,IF(A18<=A12,A18*C12,""))))))

Your original IF expression, as I interpret it above, works only if the values in A7:A12 are in ascending order, e.g. 7, 8, 9, 10, 11 and 12.

If you can reverse the order so that A7:A12 are in descending order, e.g. 12, 11, 10, 9, 8 and 7, you could replace the nested IF expression with the following:

=IFERROR(A18*INDEX(C7:C12,MATCH(A18,A7:A12,-1)),"")

Note that IFERROR is valid only for XL2007 and later, as you are using according to the "applies to" information in your original posting.

    dianeoforegon say:
    Evidently this sounds like an issue the Outlook team is investigating. I need some followup info. 

    What size is this folder? 

    The info I've received indicates this error occurs on Exchange accounts where a folder is too large. Keeping all folders under 2GB will give you the best results overall. While you can put thousands of messages in a folder when you need to sync, export as .mbox or try to rebuild it will fail. The Inbox especially should be kept as clean as possible. With advanced search you don't need to keep all messages in one folder to do a search.

    You should never rebuild an Identity with an Exchange account unless the Identity fails to open. See rebuild basics below.

    If you log into your account with your browser and do some cleanup, does the error go away?

    Exchange Basics

    The Inbox syncs more frequently than any other and should be kept as clean as possible. (this is the number one cause of problems with sync). Do NOT make subfolders under the Inbox.

    Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

    Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.


    Rebuild Basics:

    • Do Not Rebuild Identities with Exchange and IMAP accounts. Delete the accounts (this deletes the cache files in Outlook). Do the rebuild then add back accounts. Better yet start over in a new Identity. If the rebuild does not work then you could sync unwanted changes or deletions to your server and lose data.
    • Make sure you have plenty of free space on your drive. Make sure that you have at least three times as much hard disk space available as the current size of your Outlook identity PLUS 10% free for virtual memory.
    • Rebuilding does not fix all issues. It can make some problems worse. Even though the Database Utility will make a backup, I advise making a copy in the Finder first.
    • You can delete the rebuild version after you are satisfied with the results. This copy will be named like this: Main Identity [Backed up 1-21-2012 13.35]   

    Tip: If your Identity will open, before you rebuild, check "On My Computer"Inbox and Sent folders to be sure they are under the 2GB limit. Generally this is less than 10,000 messages but with large attachments could be less. Make subfolders e.g. Inbox 2011, Inbox 2010, Sent 2011, Sent 2010 etc to help reduce the number of messages in a folder. (Check for any large archive folders you might have in addition to Inbox and Sent).

    I have found that often a simple fix for issues with the Windows Insider page in Settings is to delete a Registry Key:

    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\WindowsSelfHost

    The Key should be automatically re-created and hopefully that Insider Settings page will then work right.

    You can first right-click the Key to Export it to a text file first if you wish.

    Launch regedit.exe (right-click Start and select Run). You can copy and paste the line I quoted into the address bar in regedit and press Enter to find it fast. To Export to a file, right-click WindowsSelfHost in the left column and choose Export, or else just right-click and select Delete.
