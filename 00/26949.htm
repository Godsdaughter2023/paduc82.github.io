Method 1: Disable proxy Connection

If you are using any Proxy connection, then try disabling the proxy connection and check. Follow the steps to disable proxy connection:

  1. Click the Gearbox at the top right corner in Internet Explorer.
  2. Click the Tools button, and then click Internet Options.
  3. Click the Connections tab, and then click LAN settings.
  4. Uncheck the Use a proxy server for your LAN check box.
  5. Click on ok.

Method 2: Manually reset the Windows Updates Components

I suggest you to manually reset the Windows Updates Components. Please follow the below steps to reset the Windows Updates Components:

  1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from      the menu.
  2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

   3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

   4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

   5. Type Exit in the Command Prompt to close it.

Please revert if you face any issues with Windows in future.

adam bean say:

Excel - copy formula to increase column, not row

Hey all,

Trying to build a summary table on a very wide table. Very simplistic, doing a =COUNTIF(C9:C48,"In Progress"), =COUNTIF(C9:C48,"Complete"), =COUNTIF(C9:C48,"Fail"). This needs to be repeated per column.

When I drag my formula down, it's changing the numerical values (row) =COUNTIF(C10:C49,"In Progress"), not the character values (columns) =COUNTIF(D9:D48,"In Progress")

How can I accomplish this?

Thanks!

bernie deitrick say:

Change, for example, 

=COUNTIF(C9:C48,"In Progress")

to

=COUNTIF(INDEX($C$9:$Z$48,,ROW(A1)),"In Progress")

Note that if you need more columns than out to column Z just change the Z to whatever you need - you could also ignore the rows so that if your range expands, the formula won't miss the extra cells:

=COUNTIF(INDEX($C:$Z,,ROW(A1)),"In Progress")

adam bean say:

The second formula, =COUNTIF(INDEX($C:$Z,,ROW(A1)),"In Progress")

Putting that in a second table under my main data set.

So basically, I just want to allow for future columns using that formula. All results to be counted start at C9 and end at BA48 currently.

Want to future proof myself if I add more rows or columns.

What does the comma represent?

bernie deitrick say:

If you have it under your data, you cannot use complete columns in your INDEX function reference range.

If you want to add more row, make sure that you insert them within whatever range you use

=COUNTIF(INDEX($C$9:$BA$48,,ROW(A1)),"In Progress")

So if you need more columns, insert them between C and BA; if you need more rows, between row 9 and row 48.

The comma means to use all cells within the column of the reference range instead of a specific row.

Possibly you have a pdf of the same name opened in Adobe Acrobat.  That would cause that error.

If that is not the cause of the problem, a work around would be to download and install the free PrimoPDF converter which will appear as a Printer and then use that to "print" the documents to a pdf.
