When you do that, are you able to navigate to an empty new record in the form?  If so, a trick would be to open the form filtered to a non-existent record, e.g.

    DoCmd.OpenForm "YourFormName", WhereCondtion:="OrderID = 0"

As no record with an OrderID value of zero exists, the form will open at an empty new record.  The user could turn off the filter of course, but you could reduce the possibility of their doing so by setting the form's NavigationButtons property to False (No).  To save the record and show an empty new record ready for entering another order you could include a button on the form with the following in its Click event procedure:

    Me.Requery

Requerying the form has the effect not only of saving the record, but removing it from the form's current recordset, so the user will not be able to navigate back to it.

I believe what Bob G is suggesting is that you need separate columns for;

  1. Functional Range Low
  2. Functional Range High
  3. Lab Range Low
  4. Lab Range High

In Excel, 75-85 cannot be parsed as a value. It's treated as a Text String, so it cannot be used to determine whether an actual value falls within that "range". If the range limits vary from one row to the next, you'd have to do it that way.
