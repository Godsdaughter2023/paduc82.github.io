ajeesh_nair say:

I am not sure, whether it is possible without vba code.

 

The code that I used...

 

Sub ToggleCutCopyAndPaste(Allow As Boolean)
     'Activate/deactivate cut, copy, paste and pastespecial menu items
    Call EnableMenuItem(21, Allow) ' cut
    Call EnableMenuItem(19, Allow) ' copy
    Call EnableMenuItem(22, Allow) ' paste
    Call EnableMenuItem(755, Allow) ' pastespecial
    
     'Activate/deactivate drag and drop ability
    If Allow = True Then
      If Application.CellDragAndDrop <> True Then Application.CellDragAndDrop = Allow
     End If
     'Activate/deactivate cut, copy, paste and pastespecial shortcut keys
    With Application
        Select Case Allow
        Case Is = False
            .OnKey "^c", ""
            .OnKey "^v", ""
            .OnKey "^x", ""
            .OnKey "^d", ""
            .OnKey "^r", ""
            .OnKey "+{DEL}", ""
            .OnKey "^{INSERT}", ""
        Case Is = True
            .OnKey "^c"
            .OnKey "^v"
            .OnKey "^x"
            .OnKey "^d"
            .OnKey "^r"
            .OnKey "+{DEL}"
            .OnKey "^{INSERT}"
        End Select
    End With
End Sub
 
Sub EnableMenuItem(ctlId As Integer, Enabled As Boolean)
     'Activate/Deactivate specific menu item
    Dim cBar As CommandBar
    Dim cBarCtrl As CommandBarControl
    For Each cBar In Application.CommandBars
        If cBar.name <> "Clipboard" Then
            Set cBarCtrl = cBar.FindControl(ID:=ctlId, recursive:=True)
            If Not cBarCtrl Is Nothing Then cBarCtrl.Enabled = Enabled
        End If
    Next
End Sub

Hi,

By default Office installs on the respective drive where the Operating System is installed and cannot be customized.

 

However for Office 2013 suite (perpetual License), you also get the setup file (Complete Setup file) using which you can install on the D drive.
Though you install the Office 2013 on D Drive, all the registry entries or some of the installation files will be installed on the C Drive where the Operating system is installed.

Reinstalling almost never fixes such problem. Fixes that work include:

  • Deleting the Data key
  • Deleting Normal.dotm
  • Removing add-ins

Have you tried starting Word in Safe mode? Do do this, hold down the Ctrl key when starting Word, and click Yes to the Safe mode prompt. If Word works okay in Safe mode, then one or more of the above almost always fix the problem.

In Word 2010, the data key is here in the registry:

HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Word\Data

In Word 2010/Windows 7, Normal.dotm usually is located here:

C:\Users\[user name]\AppData\Roaming\Microsoft\Templates

To disable add-ins, start Word as administrator, choose File - Options - Add-ins, set Manage: to COM Add-ins, and click Go. Remove the checks next to all add-ins shown, and click OK.

In your case, I would close Word, delete the Data key, delete or rename Normal.dotm, then start Word as admin and turn off all add-ins. I would then close Word and restart it, and see if the problem recurs.

If the problem recurs, I would next try a Repair, which is done from Programs and Features in the Control Panel.

If the problem still recurs, then I would try creating a new Windows user profile and see if the problem still occurs.

If the problem still occurs, I would then try screaming.
