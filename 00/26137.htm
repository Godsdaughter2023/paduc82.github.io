PC Flight: "Your security settings need attention" - Microsoft Community

jason[msft] say:

PC Flight: "Your security settings need attention"

In taking the latest build, a small number of users are seeing an error on their update screen when trying to check for the new build.

"Your security settings need attention"

This issue is caused by changes in the servicing pipeline for which the configurations did not land successfully on a particular PC.

If you hit this issue, click "confirm" and let the system try to self-heal.  Work through the on-screen dialogue, reboot if necessary, and then check for updates again.

If the system was unable to self-heal and you see this same error again, worry not!  Depending on what steps you may have tried to fix this issue, you will need to complete one of two following paths to finish resolving this issue.

Path #1

If you *have not* tried to opt out of Flighting, try these steps:

  1. Open Settings > Accounts
  2. Delete/remove the Microsoft Account (MSA) you have registered for Flighting
  3. Open Settings > Update & Security > Windows Update > Advanced Options
  4. Select "Stop Flighting"
  5. Reboot your PC
  6. Open Settings > Update & Security > Windows Update > Advanced Options
  7. Choose "Enable Insider Builds"
  8. Re-enter your MSA information that is registered for Flighting
  9. Follow on-screen directions, reboot if needed

Path #2

If you *have* tried opting-out of Flighting, you will need to try these steps:

  1. Open Settings > Update & Security > Windows Update > Advanced Options
  2. Select "Stop Flighting"
  3. Reboot the PC
  4. Click start, type "regedit" and open the Registry Editor
  5. Navigate to:  HKEY_Current_User > Software > Microsoft > WindowsSelfHost > Applicability
  6. Delete the following registry entries:  "Error State", "PilotInfoRing", and "LastHR"


  7. Exit the Registry Editor
  8. Reboot the PC
  9. Open Settings > Update & Security > Windows Update > Advanced Options
  10. Choose "Enable Insider Builds"
  11. Re-enter your MSA information that is registered for Flighting
  12. Follow on-screen directions, reboot if needed

After making these changes, it may take up to 24 hours for you to see the new build become available.  This is due to your PC being re-targeted for this flight due to the settings changes.

Thanks,
Jason

Hi,

 

Unless the use of RegEx is an end in itself then I'd do it differently. Try this.

 

Function Sentence(s As String) As String
Dim I As Long, MyChar As String
Begin = True
For I = 1 To Len(s)
    MyChar = Mid(s, I, 1)
    Select Case MyChar
        Case "."
            Begin = True
        Case "!"
            Begin = True
        Case "?"
            Begin = True
            Case "a" To "z"
            If Begin Then MyChar = UCase(MyChar): Begin = False
            Case "A" To "Z"
            If Begin Then Begin = False Else MyChar = LCase(MyChar)
        End Select
        Mid(s, I, 1) = MyChar
Next
Sentence = s
End Function

In F1, enter the heading Jan unique, and in G1, enter Feb unique.

=IF(C2=0,0,IF(COUNTIFS($E$2:$E2,$E2,$C$2:$C2,">0",$B$2:$B2,$B2)>1,0,1))

john w. vinson say:

I'd personally go with meters and decimals... <g>

If you're stuck with mixed units like this, I'd really suggest storing the lengths in feet and decimal fractions of a foot (12'6" = 12.5, 8'4" = 8.333333333333) for ease in sorting and in calculations. You can display and enter these values on a form by using three textboxes - one with Visble=False bound to the field, and two visible ones for feet and inches. You can populate them in the form's Current event:

Me!txtLenFt = Fix(Me!txtLen)
Me!txtLenIn = 12*(Me!txtLen - Fix(Me!txtLen))

and similarly for the other dimensions; and use code in the AfterUpdate events of the feet and inches textboxes to add feet + inches/12 to calculate the decimal length.
