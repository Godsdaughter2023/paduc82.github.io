What am I doing wrong?

The SQL:

DELETE tbl_Main_Log.MainRecPKID, tbl_Daily_Log.ChargeAccID

I tried this also: (Using * for all fields)

tom van stiphout say:
You can only specify one table in the Delete clause.

efandango say:

You can only specify one table in the Delete clause.

That's a pain because the key to knowing which records in the parent table to delete is found in the child table tbl_Daily_Log with the field criteria of 33 for ChargeAccID

I suppose I could create a make table query using the same criteria, and use that to denote which records to delete, but that feels longwinded (and I have other versions of this delete process to do, and would end up with a whole lot of tables)

tom van stiphout say:
You misunderstood my reply. I said *clause*, not *statement*. The FROM clause certainly can have multiple tables.

ken sheridan say: Your second attempt should work if you remove the reference to tbl_Daily_Log.ChargeAccID from the DELETE clause:

DELETE tbl_Main_Log.*
FROM tbl_Main_Log INNER JOIN tbl_Daily_Log
ON tbl_Main_Log.MainRecPKID = tbl_Daily_Log.MainRecFKID
WHERE tbl_Daily_Log.ChargeAccID = 33;

Or you can use the EXISTS predicate rather than a join:

DELETE *
FROM tbl_Main_Log
WHERE EXISTS
    (SELECT *
      FROM tbl_Daily_Log
      WHERE tbl_Daily_Log.MainRecFKID = tbl_Main_Log.MainRecPKID
      AND ChargeAccID = 33);

You will of course need to enforce cascade deletes in the relationship between tbl_Main_Log and tbl_Daily_Log.  Otherwise referential integrity would be violated by the attempt to delete rows from the referenced table only, without deleting the matching rows from the referencing table.
