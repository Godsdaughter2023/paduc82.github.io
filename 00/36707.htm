¡firedog say:

  User is authenticated but not connected.

This appears to be a feature of the Outlook Mail IMAP implementation that has more than one trigger. It has been happening more frequently recently for users of different mail clients, so it's nothing to do with Windows Live Mail or wrong account settings. If you don't get anywhere with the measures outlined below, you might consider seeking assistance through Outlook Mail. The Help and Feedback mechanisms there give ways to contact Outlook staff, as does the dedicated UserVoice site.

One of the triggers is 'calls from a chatty client' where the program tries to open a new connection before an earlier one has closed. To see whether this is the cause, ensure that Windows Live Mail is not set to check for new messages more frequently than every 20 minutes or so. Set the interval on the General tab of the program options dialogue (Ctrl-Shift-O). Ensure, too, that no other client is attempting to synchronize with this account via IMAP meanwhile; this might include smart phones or tablets not using the Outlook app. Note that selecting an unread message in the message list will cause the program to attempt to access the server, as will deleting a message, so it's as well not to do this while you're trying to troubleshoot the error.

The obvious trigger is 'wrong password'; the IMAP server first checks that the username exists. This should be the full email address; unless you've made changes to the account, this won't have changed. Check it on the Servers tab of the account's properties dialogue. Once the server has established that the username is a valid one, the user is apparently authenticated. In normal circumstances, this will be enough to make the connection. However, just sometimes the server will also check the password; if this check fails, the status becomes authenticated but not connected. Removing the password and re-entering the correct one may help, but otherwise there's little that can be done other than wait half an hour and try again.  

One last avenue to explore is unverified access to the Microsoft Account concerned. Sign in at https://account.live.com/Activity and approve any sign-in from your own IP address.

hairyfool say:

The minimum spec for 64bit is 2GB so the "upgrade" has effective reduced your available RAM to 1GB. These values are not precise but reasonably correct.

 

It is even debateable if there is much benefit with 4GB let alone 3 to upgrade unless you have a specific need for 64bit. Without having the actual model I can't be precise but older laptops are limited to 4GB in many cases.

 

However the low frame rates and excessive heat are going to be down to the CPU and graphics spec as much as the memory. Some systems will allocate more RAM to integrated graphics chips if more system RAM is available but not all.

 

The simple option is to install more memory depending on the capabilities of the laptop.
