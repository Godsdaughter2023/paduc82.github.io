Hamish,

You can use a web browser to display an HTML file that is still on your computer, using the file:// prefix and the file's path and name instead of the http:// prefix and Internet address.

For anything else, you do need a host that runs a web server (Internet Information Server on a Windows host or, typically, Apache on a Linux host). jay freedman say:

Hi Hamish,

Microsoft's Office 365 includes OneDrive, which is a file storage service from which you can share files with other people. It isn't intended as a website host, but I suppose you could share an HTML file and it would open in the user's browser.

There are lots of organizations that offer hosting, some of them for free (generally supported by advertising that they add to your pages). Look at http://www.google.com/search?q=web+site+hosting and do some research.

Word offers a "save as web page" option because that file can then be uploaded to your site on a host, from which other users can display it. It's nothing more than that. And as Suzanne mentioned, the HTML that Word creates tends to be bloated by comparison to the same page built in a real HTML editor. (The reason for that is to preserve information that allows you to reopen the HTML file in Word with all the formatting and other internal stuff that was in the original document.) There's also an option in the same list to save as "web page, filtered", which produces a file that's somewhat less bloated but still not really "cleaned up".

To have someone else use a browser to access an HTML file on your computer, you need two things: (1) Run a web server such as IIS or Apache on your computer, and store the HTML file within a folder that you make known to the web server. IIS comes with most versions of Windows, but usually isn't installed by default -- you have to go into the Control Panel and install the feature. (2) The other user has to know the IP address that's assigned to your computer, which you can send to them in an email, and perhaps the name of the HTML file. This setup isn't very difficult, but it's more than I can put into this post. Start reading some of the articles listed at http://www.google.com/search?q=how+to+setup+a+local+website.

graham mayor say:
Unless you check the email address against the available e-mail addresses then as far as Outlook is concerned the e-mail address is valid. The fact that it is bounced is a reflection on the receiving system rather than the sending system. The sender would not know whether the receiving system is aware of the address until the message has been sent and delivery attempted.

If the addresses are in the Outlook contacts list and that list is up to date then you can validate the addresses against the contacts list (which will be slow if there are lots of addresses in that list), but I suspect that the contacts list may not be any more up to date than the worksheet, but if you want to explore that path you could call a function to validate the address against the contact list e.g. as follows which will return 'True' if eAddress exists. But if your contacts lists are not rigorously maintained I don't see how you can overcome this problem.


Function CheckAddressExists(eAddress As String) As Boolean
Dim olApp As Outlook.Application
Dim olNS As Outlook.NameSpace
Dim olContactFolder As Outlook.MAPIFolder
Dim olContact As Outlook.ContactItem
Dim olItems As Outlook.Items
Dim bFound As Boolean, bStarted As Boolean
On Error Resume Next

Set olApp = GetObject(, "Outlook.Application")
If Err <> 0 Then
    Set olApp = CreateObject("Outlook.Application")
    bStarted = True
End If

Set olNS = olApp.GetNamespace("MAPI")
Set olContactFolder = olNS.GetDefaultFolder(10)
Set olItems = olContactFolder.Items
For Each olContact In olItems
    bFound = False
    If eAddress = olContact.Email1Address Or _
       eAddress = olContact.Email2Address Or _
       eAddress = olContact.Email3Address Then
        bFound = True
        Exit For
    End If
    Err.Clear
Next olContact
If bFound Then
    CheckAddressExists = True
End If
CleanUp:
If bStarted = True Then
    olApp.Quit
End If
Set olItems = Nothing
Set olNS = Nothing
Set olContactFolder = Nothing
Set olContact = Nothing
End Function
