ken sheridan say: I find it better, when including this sort of validation, to provide a means for the user to explicitly save a record via a command button rather than relying on it being saved when the user closes the form or navigates to another record.  It can be tricky to handle the error if the validation fails when the user tries to save the record by closing the form.

You'll find an example of how to do this as SaveDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

This little file includes validation of the type you are doing, in this case rejecting a record where the contact name is already in the database.  This is unrealistic of course as personal names can legitimately be duplicated, but it serves to demonstrate the methodology.  Note that the validation in the form is reinforced in the table definition by a unique index containing the two columns.

Look carefully through all the code in the form's module and you should be able to see how it works.

One thing worth mentioning is that when you do test for an existing record with the same PartnerID it's better not to examine the form's NewRecord property as this excludes the possibility of an existing record being edited by changing the PartnerID value to one which already exists in another row.  A better approach is to test for a PartnerID of the same value as the current record, but with a different primary key value (ProfileID?).  My demo does this, testing for the same first and last names, but a different ContactID.

If you don't wish to force the user to save a record via a button, but simply by closing the form, and at the same time require the validation on the PartnerID column, you'll need to flag up the cancelling of the update with a module level variable, and examine this in the form's Unload event procedure, which has a Cancel argument whose return value can be set to True to prevent the form closing if the validation criterion is not met.

jeeped say:

What formula(s) can I use to calculate the number of shipments for each month based on the following criteria: every new date for the month indicates a new shipment, provided that the shipment date is for the same order. But if the Customer order is different, even though the ship date may be the same on two rows, then two shipments should be counted.


Your data seems contradictory. The Number of Shipments for the Month total seems to come from the Actual Ship Date. If you want unique Order Nos with Actual Shipping Date then Aug should be 2, 2, 0. There are similar discrepancies for other months. The numbers as provided can only be returned by changing the conditions of uniqueness from column to column. Here is what I came up with,

                            

I based all dates as actual numerical dates with formatting as either mm/dd/yy or mmm. I prefer to deal with numerical dates, oly converting to textual representations when absolutely necessary.

The formula in A3 is,
=DATE(YEAR(C3),MONTH(C3),1)
... formatted as mmm. I was unsure whether this should have been referencing D3, so I duplicated your own sample. F3 is =A3 formatted as mmm and F4 is =EDATE(F3,1).

The formula in G3 is,
=SUMPRODUCT((TEXT($D$3:$D$99,"mmmyyyy")=TEXT($F3,"mmmyyyy"))*(MATCH($B$3:$B$99&"×"&$D$3:$D$99,$B$3:$B$99&"×"&$D$3:$D$99,FALSE)=ROW($1:$97)))
... which returns a unique (Order × Actual Date Shipped) count of the shipments made in Aug (i.e. F3).

The formula in H3 is,
=SUMPRODUCT((TEXT($D$3:$D$99,"mmmyyyy")=TEXT($F3,"mmmyyyy"))*($D$3:$D$99<=$C$3:$C$99)*(MATCH($B$3:$B$99&"×"&$D$3:$D$99,$B$3:$B$99&"×"&$D$3:$D$99,FALSE)=ROW($1:$97)))
... which returns  a unique (Order × Actual Date Shipped) count of the shipments made in Aug that were on time.

The formula in I3 is,
=SUMPRODUCT((TEXT($D$3:$D$99,"mmmyyyy")=TEXT($F3,"mmmyyyy"))*($D$3:$D$99>$C$3:$C$99)*(MATCH($B$3:$B$99&"×"&$D$3:$D$99,$B$3:$B$99&"×"&$D$3:$D$99,FALSE)=ROW($1:$97)))
... which returns  a unique (Order × Actual Date Shipped) count of the shipments made in Aug that were late. The third row of data (i.e. row 5 as above) was late but the shipment was actually made in Sep so it will be counted in that month. The counts in Dec also illustrate this point.

Hi John,

Outlook.com lets you connect up to five other email accounts, such as Gmail or Yahoo! Mail, and view them from your Outlook.com inbox. To connect your email accounts to Outlook.com, kindly follow these steps:

  1. Sign in to Outlook.com.

  2. On the navigation bar, select  > Options.

  3. In the left pane, choose Mail > Accounts > Connected accounts.

    NOTE: Not all features are available in every region. If you don't see the Connected accounts option, this feature might not be available for your account.

  4. If you're connecting a Gmail account, choose Gmail. To connect any other type of email account, such as an Outlook on the web (Office 365) account, choose Other email accounts.

Word 2016 Redline / Review - Microsoft Community

david sirota say:

Word 2016 Redline / Review

I often have to redline documents and then, subsequently, have to make additional changes or make changes to my original redline comments.  I'm looking for a way I an change the color of redline changes on a revision by revision basis.  If I change the redline color in Options, it changes the color on the previous redline changes too.

Thanks for your help.

jay freedman say:

Word can already do this, but the method isn't obvious.

First, in the Track Changes Advanced Options dialog, set the colors for the kinds of changes you want to track to "By author".

Second, for each session that you want to show in a different color, click the Change User Name button in the Track Changes Options box, or change the user name in File > Options > General. (The button just opens the General page.) You can add a number to the end of your usual user name, or change the user name to something completely different such as the session number. Also be sure to check the box below the user name for "Always use these values regardless of sign in to Office".

The Track Changes feature will see each variation of the user name as a different author and pick a different color, up to 14 or 15 colors. Unfortunately, you can't specify which colors they are, and at least white and yellow are really bad choices.

Remember to restore your preferred user name when you've finished marking up the document.
