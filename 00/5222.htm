john - project say:

DanHan55,

Project is "dumb". If you say a task will require 40 hours of work but then status it by saying the actual work was 48 hours, Project will rightly conclude that the original 40 work estimate was wrong.

You need to consider two things, the original plan and the current plan. Once you have developed a plan, save a baseline. The baseline will capture several Project fields for later use in comparing to the current plan. The current plan reflects what is actually happening in addition to the future part of the plan. When updating work you should input the actual work and remaining work. Project will update the current plan such that work is the sum of actual and remaining.

Let's say you have a task that is estimated to take 40 hours over a 1 week span (40h duration). Three days in the actual work is 24 hours (one resource working full time) but you realize that task is only half done and you estimate that it will take another 24 hours to complete so you set the remaining work for 24 hours. Project will then update the original estimate of work to agree with reality and set it at 48 hours and extend the finish date by one day. No! you say, the task has to absolutely positively be done in 5 days so you either use overtime for the single resource or find and assign another resource to help. This is all part of the dynamics of project management.

Because you saved a baseline at the onset, that original plan is available for comparison to the current plan using several of Project fields such as Duration Variance, Start Variance, etc. And you can show the comparison graphically with various views such as the Tracking Gantt view.

If you want to have tasks with a fixed duration, then set the task type to Fixed Duration. Assign resource, monitor progress and adjust resource assignments as necessary.

Hope this helps.
