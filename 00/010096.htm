macropod say:

Simpler than using a macro, would be to embed a FILLIN field within the QUOTE field, thus:

{QUOTE "*{FILLIN "What is the barcode ID?"}*" \* Charformat}
Now, whenever you want to update the barcode, simply select it and press F9 - you'll get a prompt for an update, with the current value displayed.

Since you're using Word 2007, you could add the field to a custom Quick Part named, say, Barcode. Then, whenever you want to insert a barcode, insert the Barcode Quick Part and press F9.

The only potential drawback I see with this, but it could also be seen as a benefit, is that anything that causes fields in the body of the document to update (eg Ctrl-A then F9, or printing with the 'update fields' option checked) will trigger the FILLIN field.

As to a macro solution, you could use something like:

Sub InsertBarcode()
Dim StrCode As String
StrCode = Trim(InputBox("What is the barcode ID?", "Barcode Insertion"))
If StrCode = "" Then Exit Sub
With Selection
  .Collapse wdCollapseStart
  .Text = "*" & StrCode & "*"
  .Font.Name = "3of9"
  '.Font.Size = 24
End With
End Sub

With the macro, you'll need to add the actual font name and, possibly, specify an appropriate point size (there's a commented-out line in the code that shows how to do that). to use the macro, you'd need to add it to whatever template the document is based on (eg Normal.dot). For instructions on installing macros, see: http://www.gmayor.com/installing_macro.htm

Sorry, I was referring to Autonumbering paragraph format, not {AUTONUM} fields.

With a regular paste, the {DOCVARIABLE} field is rendered null because the variable does not exist in the new document.

You need to unlink fields before copying, in order convert them to their equivalent values. After selecting text In the original Word document:

    Selection.Fields.Unlink 'Convert to field results
    Selection.Copy
    ActiveDocument.Undo 'Restore fields

In case you just opened the source document to copy text from, you'll be able to close it without a "Save?" prompt using this statement:

    ActiveDocument.Saved = True 

Then switch to the temp document and use a regular Selection.Paste to preserve the format.

daniel g. grau say:
By default, to skip-move a picture, select it and press the arrow keys or nudge it with Option Arrow. Of course, you have to be in Print Layout View AND the picture has to be floating (i.e., not "in line"). Check this attribute by Control-Clicking/Right-Clicking on the picture, then choose Wrap Text to make sure "Inline with text" is not selected.

Again by default, Command Right/Left Arrow rotates the picture.

You have not picked the wrong command, what's happening is you've reassigned Option Right/Left Arrow to something else. You need to reinstate the proper shortcuts to these commands: WordRight (Option Right Arrow) and WordLeft (Option Left Arrow).

I wasn't able to assign shortcuts to some of the picture-handling commands. However, those that work as expected are DrawNudgeUp, DrawNudgeDown, DrawNudgeLeft and DrawNudgeRight. As stated, these four are already being invoked though Option Arrows when a floating picture is selected.

rohn007 say:

Ignore Rhen Pal, re-install is irrelevant to this problem.

Short answer, NO.

Office no longer has such setting. There used to be an option to emulate WordPerfect's blue background. But that has been removed.
Bright white background is your only option in Office. "MS knows best".

There is a Win 10 setting you can try: Control Panel > Personalization.

In there you want to try the various Themes.  The only defaults that seem to affect what you want changed are the "High Contrast Themes". But since you don't want high contrast they will not do the trick. BUT, if you try them, you'll see that even the Word 2016 document background is switched from White to Black, proving it can be changed.   Now, "all" you have to do is search the internet for a free theme that changes the colors the way you want OR find out how to create a custom theme to do this.

If you find either, please let us know. Other people have asked the same question.

There is another option if you don't mind distorted color display. In settings > Advanced display Settings > Color managment or Color calibration you can adjust the color display so that white is not so bright, maybe slightly "off color" by selecting a different "color temperature".

Yet another option is  tool like f.lux: https://justgetflux.com/  (or Asus EyeCare Switcher). f.lux makes the color of your computer's display adapt to the time of day, warm at night and like sunlight during the day. You can use this adjustment to make the white on screen appear less contrasty.
