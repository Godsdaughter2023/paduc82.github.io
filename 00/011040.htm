Hi TrinityTSS,

For the scenario that local AD will be re-built, our suggestion is:

1. First, deactivate/disable Directory sync from Office 365 admin center. Office 365 user status will be changed from “synced with AD” to “In cloud”. This is used to make sure local AD change will not affect Office 365 online users.

2. Now, you can complete local AD re-build.

3. This is the key step. Now, for existing Office 365 users and new AD users, we need to match them. It is recommended that you use Hard Match to match Office 365 user with the corresponding new AD user. Then, Office 365 users will be synced with new AD users.

If we do not match the users, there might be duplicate issue during the sync.

Office365 and Exchange will maintain continuity for all Office365 settings

hector mad say:

Hello,


If you are getting the "something went wrong" error when trying to search for emails in Outlook, you may need to check for and Install Office updates, or include Microsoft Outlook in the Indexing Options by doing the following:

  1. Close Outlook.
  2. Type in Indexing Options on the Cortana search field, and hit Enter.
  3. Click on the Modify button.
  4. Check on the box beside Microsoft Outlook.
  5. Select OK.
  6. Launch Outlook, and try searching again.

If the issue persists, or the box is already ticked, try rebuilding your Instant Search catalogs. If you are still getting the error, try repairing the Outlook data files. For instructions, see the following links:

  • Fix search issues by rebuilding your Instant Search catalog
  • Repair Outlook Data Files
