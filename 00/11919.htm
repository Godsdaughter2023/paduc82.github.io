Lower priority question is: why can't I write simple click-as-you-go macros for graphic objects?

Thanks,

Fanciulla in Seattle

jay freedman say:

Here's the macro you want:

Sub ResizeInlinePix()
Dim ILS As InlineShape
Dim ratio As Single

For Each ILS In ActiveDocument.InlineShapes
ratio = InchesToPoints(2#) / ILS.Height
' resize only if original is more than 2" high
If ratio < 1# Then
ILS.ScaleHeight = 100 * ratio
ILS.ScaleWidth = 100 * ratio
End If
Next
End Sub

You can't record this because the recorder has never been able to handle mouse selections, nor will it ever create a For-Next loop.

doug robbins - mvp office apps & services (word) say:

That's a pretty standard mail merge operation.  However, I think that you might have made a bit of muddle of the description of what you are trying to achieve.

Do you have field names in the cells in the first row?

If not, insert a row at the top of the worksheet and into that row, insert field names such as those shown in the following screen shot of the fields used for the Address Block.

Then, use the Sorting facility in Excel to sort the records into the order that you require.

Then, in Word, use the Page Setup facility to set up the page with two Newspaper style columns and then go to the Mailings tab of the ribbon and from the Start Mail Merge dropdown, select Directory and then use the Select Recipients facility to select you Excel Workbook as the data source.   Then, insert an Address Block field into the document (going through the Match Fields routine if necessary).  Then save the mail merge main document and click no the Finish & Merge dropdown in the Finish section of the Mailings tab and select Edit Individual documents and check the output to see if it is what you require.

Using the Address Block field should take care of any records that are missing data,  If it does not, you may then need to use an If...then...Else field construction to deal with it.

doug robbins - mvp office apps & services (word) say:

You will need to use the Match Fields facility to match the fields in your data source to those used by the Address Block.

Alternatively, you can use the fields from your data source with If...then...Else field constructions to overcome the issue of blank lines.

For example, if you have the following fields Address1, Address2, City and sometimes Address 2 is empty, use

{ MERGEFIELD Address1 }{ IF { MERGEFIELD Address2 } = "" "" "¶

{ MERGEFIELD Address2 }" }¶

{ MERGEFIELD City }

You must use CTRL+F9 to insert each pair of field delimiters { } and use ALT+F9 to toggle off there display.   Where the ¶ appears in the above, press the Enter key.

macropod say:

If the Address2 field is on a separate line from the Address1 field, or in a separate paragraph per your field code, the mailmerge should suppress it when empty. Either way, an IF test isn't needed. Instead, one you could use:

{ MERGEFIELD Address1 }{ MERGEFIELD Address2 \b "¶

"}¶

{ MERGEFIELD City }

where ¶ represents a real paragraph break or line break. For more mergefield formatting advice, the OP should see the Mailmerge Tips and Tricks thread at:
http://www.msofficeforums.com/mail-merge/21803-mailmerge-tips-tricks.html
or:
http://windowssecrets.com/forums/showthread.php/163017-Word-Mailmerge-Tips-amp-Tricks
