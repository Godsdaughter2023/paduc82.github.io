albert d. kallal say:
I don't believe the fact of using one module or 150 modules has ANY relationship to the write conflict error at all, in fact, they are completely and entirely unrelated to each other.
 
Write conflict errors when connecting to SQL server are usually result of not having a primary key, or are that that table does not have a time stamp column. Access really needs both of these to function correctly when working with SQL server.
 
The third and even more common issue for unexplained write conflict errors when using SQL server is of course when you have a true/false collum, and allow null values. Check out:
 
Write Conflict Error When You Try to Update Records in a Linked SQL Server Table
http://support.microsoft.com/?id=280730
 
So check out the above issues, the fact of using one module, or 200 modules is likey not relevant at all.
 
Keep in mind also that if you use a subform based on a one particular form one time or 20 times inside of a form, each subform is a separate class object instance of that form. And each separate instance of that form, is going to have its own copy of variables for the particular instance of the form, you can think of this is much like a session in a web form, each web browser loaded has its own to session state.
 
Simply using one module, or 10 models out of the blue has absolutely no relationship in any way shape or form to that of some write conflict error.
