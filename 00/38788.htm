balaji kundalam say:

Hi

Thanks for contacting Microsoft Community.

 

The reason for the error "Application not found" is that your USB drive was infected with some virus/malware and an antivirus program removed the virus/malware. But, the autorun.inf file is still present in the drive. Autorun.inf file gives information to Windows about the programs/commands to be execcuted. Therfore, the virus made an entry that when the drive is opened, the virus program should be executed. But, the virus has been removed, so there is no application for execution. So the error comes. Note that autorun.inf file is not a virus, so the antivirus program did not remove it.

 

If it is opened as portable device, the autorun.inf file will not be recognised. So it works out.

 

Therefore, try deleting the autorun.inf file from the root of the USB drive (suppose E:\ is the USB drive, E:\autorun.inf is the files location). Also, note that the file will be hidden, so you need to go to the folder options->under view tab, select Hidden files, folders and drives->also, remove the tick on Hide Protected Operating System files (Recommended). Click Yes if a Warning dialog prompts. Click Apply and OK. Now you can view the file and delete it.

 

If you can't delete it, backup your files from the USB drive (excluding the autorun.inf file) and format (complete, not quick) it.

 

Note that autorun is different from autoplay. When you connect a flashdrive and a dialog comes with options like Open Folder to View files or Open with Windows Media Player, it is called autoplay. It works only if autorun.inf file is not present or any program execution path is not defined in the autorun.inf file (like only the icon file is defined).
