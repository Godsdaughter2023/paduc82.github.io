Little bit late to answer, but I ran this week in the same issue and solved it (lucky me ). Did almost the same installation (W10 with Office 2013) on two different machines, with only one difference.

Custom lists are filled with a few default values (weekdays and months) when installing Office. These values are depending on the region settings (not language, but region) at the moment of installation. When you change the region settings after installation you can't add a custom list anymore. Solve the problem by changing the custom setting to the one at the moment of the Office installation.

Click 2 Run is the MS marketing name for a "Virtual environment" that contains the Office, or Outlook installation. The programs are not actually installed on your machine, they are downloaded from MS servers EVERY TIME you start them.  The C2R environment is setup inside of the Q:\ drive.  As you've found, the Q:\ drive is defined in such a way to prevent users from tinkering with them.

Here are a couple of options to help uninstall it:
Update, repair, or uninstall Office Click-to-Run CTR products

http://office.microsoft.com/en-ca/excel-help/update-repair-or-uninstall-office-click-to-run-products-HA010382089.aspx

Note: Repairing Office Click-to-Run products require an Internet connection to be available during the entire repair process.

1.     In Windows 7, click Start and select Control Panel from the right-hand list then select Programs.

2.     Select Microsoft Office Starter 2010 from the list of installed programs and them click Change from the commands available above the list.

3.     Check Remove Office user settings if you want to reset all options to factory defaults. Uncheck it if you wish to retain any customization that you have performed on the program(s).

4.     Click Repair and follow the procedure through to its conclusion.

 

Unable to uninstall Click2run using MS FIX IT - solution

http://answers.microsoft.com/en-us/office/forum/officeversion_other-office_install/unable-to-uninstall-click2run-using-ms-FIX IT/a51729cd-9b41-41eb-b035-90c35ec87037

When you try to uninstall Office click2run 2010 using kb290301 (MS FIX IT), you get an error stating problem with client virtualisation.

Solution:

Go to: Start> Run> Services.msc> Client virtualisation> properties> startup type> Disable\manual> Stop the service> Ok

Now try restarting system and then try using FIX IT from kb290301
