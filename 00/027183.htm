Instead of commenting the increasing mess which is made by MS regarding the storing and syncing issues of OneNote, OneDrive and Apps I'll just try to clarify the status quo a bit.

Edge Broswer is in no common way supporting OneNote notebooks (neither locally stored like IE does nor those on OneDrive / OneDrive for business). The Share-Button does not save the content into a notebook but directly to the OneNote Universal App instead. This means you would have to open the OneNote App everytime to make it sync the new content into OneDrive and be able to see it in OneNote 2016 after that one synced too.

Edge does not make use of the "old" API which "talks" to the office OneNote directly (as IE or Outlook do).

Edge does not use the "new" API either, which "talks" to the OneNote notebooks stored in the cloud.

Edge does not support Microsofts OneNote WebClipper yet which would use the "new" Web API.

But there is a workaround to install WebClipper into Edge until officially supported: Install Webclipper (www.onenote.com/clipper) into Internet Explorer and import IE bookmarks to Edge afterwards. Webclipper will be transferred too (together with all other bookmarks in IE so you may want to clean them up completely before if you are not using IE at all).

Note that the Webclipper wont support annotations / drawings you made on a webpage (as far as I know).
