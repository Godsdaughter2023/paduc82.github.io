Keep Excel file from closing? - Microsoft Community

michael rosa say:

Keep Excel file from closing?

Is it possible to password protect an Excel 2010 file to keep it from being closed? The file needs to stay open 24/7 and there are times where someone will accidently close it out.
chip pearson say:

Clearly you need some mechanism to allow the workbook to be closed under certain circumstances. Use the code below in the ThisWorkbook code module:

 

Private Sub Workbook_BeforeClose(Cancel As Boolean)
    If StrComp(ThisWorkbook.Worksheets("Sheet1").Range("A1").Text, _
            "password", vbBinaryCompare) <> 0 Then
        Cancel = True
    End If
End Sub

 

Here, the workbook cannot be closed unless cell A1 on Sheet1 contains the word "password". To close the file, enter "password" in cell A1.

rohn007 say:

Here are a couple of possible  ways of fixing the problem:

Microsoft Office 2010 Professional Plus configures each time you start - http://support.microsoft.com/kb/2528748 Office Apps reconfigure every time they are started

 

Look at the event handler for error messages. In one case it was showing an error that HKLM/Software/Classes/.msg could not be accessed.

This is how I fixed it:

·Using Regedit navigate to the reg key identified in the error

·When accessing Permissions, I got an error message, but ignored that.  Open the Permissions window

·click on the "Advanced" button.

·Then select the "Owner" tab,

·set the owner as "Adminstrators".

·Click Apply and OK.

·Close Regedit,

·Run the application again, the Configuration process will run.  Then close Outlook.

·Open Regedit again

·locate the same Classes/.msg key.  You should now see a number of sub keys. 

·For each sub key , right click on it to select Permissions.

·Click on the "Advanced" button,

·then the "Owner" tab,

·set the owner as "Administrators",

·click Apply and OK.

·On the security tab, click on the Add button. 

·Add "SYSTEM" as a new user, and put a tick next to "Full Control". 

·Click Apply and OK, and you should then see other users added.

·Repeat the above for all sub keys.

·Close Regedit,

·reboot.

If there is no error message, ie normal operations try adding the NoReReg key, ie manual method:

Before you close up shop and start working on this reg fix, open the version of Word that you will want to be your default. In other words, say you have two versions of Word on your system. You want to use 2007, but you want 2003 to be the version that opens by default when you click on a DOC file. Open Word 2003 before you insert the fix below. This will ensure that Word 2003 is the last version opened and it will, if necessary, reset the registry to that version. (More on getting around this default in the next section.)

·Now make sure Word (office apps) is closed

·Start regedit  <Win  R> and type REGEDIT
Note that if this doesn't do anything, you're using a system where the administrator has taken away your ability to edit the registry so you don't mess it up!

·Once within the registry, click the + (plus sign) to expand the /HKEY_CURRENT_USER/ folder.

·Scroll down and locate the /Software/ folder and expand that one. We have a long journey ahead, so hang in there!

·Once within the Software folder, scroll down that branch of the folder tree until you find the folder for /Microsoft/. Click the + to expand that one.

·Now scroll down to locate the /Office/ folder.

·There you'll see the numbered versions of the Office versions you have:
2010 is 14.0
2007 is 12.0
2003 is 11.0
2002 is 10.0e.

·This fix can be used for all recent versions. Or at least, I know it works as far back as 2000. I can't confirm anything earlier. But if you're still using a version older than 2000, speed is obviously not an issue for you!<grin> Expand the folder for the version you want to speed up. (Note, this speed up fix will only help you if you are running multiple versions. So if you only have one version of Office on your system. Don't bother with this setting, as it won't do anything for you.)

·Move to the /Word/ folder and expand that. Almost there!

·Okay, locate the /Options/ folder and select that one. This is the folder where you'll be adding a new registry key.

·With the Options folder selected in the route we took above, click Edit > New > DWORD Value

·This will insert a new registry key into the Options folder. It will be highlighted in edit mode so you can go right ahead and start typing the name of this key: NoReReg

Hit the Enter key after typing the name to have the name accepted.

·Now double click this new key to open it. The Value data will read 0 (zero) by default. Change that setting to the number 1

·click OK to accept that value.

Then close the registry. You did it!

So what is it that you've done? Well, you went into the Word registry settings for the version of Word you selected and you set a new registry key. The key is an option that stops Word from reregistering itself if another version is the current default. The value of 1 means Yes. So the NoRereg (no reregistering) is set to Yes...i.e., don't do it! This will cause Word to ignore the default version and just open, without taking several minutes to go through the reregistering hassle.

Shortcut method:

·<win R> to start run dialog

·enter this command

reg add HKCU\Software\Microsoft\Office\14.0\Word\Options /v NoReReg /t REG_DWORD /d 1

repeat for each office app and each version installed

bob jones aka: cybertaz say:

I'm afraid I don't have a solution, but perhaps this may help point you in the right direction :-)

It's highly improbable that the issue has anything to do with the Office software, itself. Mac Office 2016 does not have any sort of Clipboard of its own. It uses the OS X Clipboard which is implemented independently on each device. In fact, it's specific to each User Account on the Mac, so if you log in as User A & copy any content then switch to User B on the same Mac, what User A copied is not available to be pasted... IOW, User B's Clipboard will be empty.

Further, Cut/Copy actions are not 'remembered' by the document files in which they are performed. Once the file is closed the history of those actions is purged from memory (although the content can be retained on that user's Clipboard for additional pasting elsewhere). If the document is opened by a different user or a separate installation of the software there is no record of those actions retained in the file.

Some thoughts on what to pursue:

  • Are you using any sort of 3rd party Clipboard management software, such as Keyboard Maestro, iClip, iClipboard, CloudClip, etc.?
  • If none of the above, it may have something to do with network configuration set to automatically back up the Macs. Using the same credentials could be contributing to the confusion.

You might check in the Apple Community for your version of OS X to see if there are any reports of similar behavior.
