Preventing the Mail Merge security message from being displayed - Microsoft Community

johncrawford_893 say:

Preventing the Mail Merge security message from being displayed

There have been a number of questions regqrding the security message that appears whenever a mail merge document opens. The normal response is to alter the security in the registry. I have found what, I think, is a relatively easy. It does, however include a macro.

This is the code:

==============================================================

Private Sub Document_Open()
  Dim OutputFileNAme As String
  Dim DataSourceName As String
  Dim FullFilePath As String
 
  OutputFileName = "MergedFile.docx"
  DataSourceName = "MergeSource.csv"
 
  filepath = ActiveDocument.Path                             'Get the path to this document
  Application.ScreenUpdating = False                         'stop screen updating
   
   'Make sure the Data source exists
   If Dir(filepath & "\" & DataSourceName) = "" Then
     MsgBox "The Merge Data source is missing." & vbCrLf & _
            "This document will now close.", vbCritical Or vbOKOnly, "Missing Data source"
    Else
                 
     'Set up the Mail merge
     With ActiveDocument.MailMerge
       .OpenDataSource (filepath & "\" & DataSourceName)    'Opens without security message
       .Destination = wdSendToNewDocument
       .SuppressBlankLines = True
       With .DataSource
         .FirstRecord = wdDefaultFirstRecord
         .ActiveRecord = wdLastDataSourceRecord
         .LastRecord = .ActiveRecord
         .ActiveRecord = .FirstRecord
       End With
       .Execute Pause:=False                                'Do the Mail Merge
       'Close the data source, otherwise it cannot be deleted
       .DataSource.Close
     End With
    
     'Output file name. .docx file
     FullFilePath = filepath & "\" & OutputFileName

     'Save the merged file
     ActiveDocument.SaveAs2 FileName:=FullFilePath, _
                           FileFormat:=wdFormatDocumentDefault
     
     ActiveDocument.Close (wdDoNotSaveChanges)    'Close the new document
        
     'Cannot remove the data source unless the source has been closed
     Kill (filepath & "\" & DataSourceName)                  'Remove the data source
        
   End If
    
   Application.ScreenUpdating = True                         're-start screen updating
   Application.Quit (wdDoNotSaveChanges)               'Close Word
End Sub

==============================================================

The first thing you need to do is to remove the datasource. Whan you open the document, select No when presented with the security message. Save the document. This will then remove the data source.

Next add the macro. It:

  1. The macro must be placed in "ThisDocument" under "Microsoft Word Objects", and named "Document_Open()". Otherwise it will not run when the document opens
  2. Makes sure the data source exists. If not a message will be produced and the app will close
  3. Next the Merge process is set up.
    1. First the Merge source must be opened. I use a .csv, but any source should work.
    2. Next the merge is set up. This is needed because, since the source is not attached, it must be created.
    3. Next Execute the merge
    4. Close the datasource. I delete the source at the end of the process, and if the datasource is not closed it cannot be deleted.
  4. Save the new merged document, then close it
  5. I then delete the data source
  6. I restart the screen updating that I stopped at the beginning. It is not really necessary, just to me it is better.
  7. Then close Word without saving.

There may be better ways of coding this, but it works for me in 5 or six different merge documents.

jp ronse say: Hi André,

Why not using VBA to do this?

Sub IP_Gen()
   Dim i, j, k, l
   Dim strIP As String
   Dim x
   x = 1
   For i = 0 To 255: For j = 0 To 255: For k = 0 To 2555: For l = 0 To 255
      strIP = i & "." & j & "." & k & "." & l
      Cells(x, "A") = strIP
      x = x + 1
   Next: Next: Next: Next
End Sub

Will generate all IP addresses. Remark: Excel does not have enough rows to add them all.

You can limit the range by changing the 0 to the starting value you want and this for each byte.

Note: it is not needed to enter/add the zero's. 10.10.10.10 is equal to 010.010.010.010

Using 3 digits contains a little risk. If you paste 010.010.010.010 in a cell, it will be displayed as 10.010.010.010 but the content will be 10010010010 because Excel will interpret the dot as the thousand separator.

I make this remark because your name sounds as from someone of the same country where we use . for 1000 & , for decimal.

rich michaels - mvp (greatcirclelearning.com) say:

Hi David,

Try something like this:

Sub mcr_Open_DOCX_Document()

    Dim wdDOCX As Word.Document
    Dim wdAPP As Word.Application
    Dim sFn As String
    
    On Error Resume Next
    Set wdAPP = Word.Application
    If Err.Number <> 0 Then
        Err.Clear
        Set wdAPP = CreateObject("Word.Application")
    End If

    sFn = ThisWorkbook.Path & "\My_document.docx"
    Set wdDOCX = wdAPP.Documents.Open(Filename:=sFn, ReadOnly:=True)

    wdAPP.Visible = True

    wdAPP.Activate

    'wdDOCX.Close SaveChanges:=wdDoNotSaveChanges
    'wdAPP.Quit
    Set wdDOCX = Nothing
    Set wdAPP = Nothing
End Sub
