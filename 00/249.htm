Hello Richard,

To change the email notifications, follow these steps:

  1. Sign in to your Yammer account. In the upper-right corner of the screen, click the ellipsis (…), and then click Edit Profile.
  2. In the column on the left side, click Notifications.

    Note If you're a member of multiple networks, click the name of the network that you want to change the email notifications for. You'll have to change the notification settings for each network individually.
  3. In the Send me a digest of message activity drop-down list, select Never, Daily, or Weekly. This sets the frequency at which you'll receive a digest email message that summarizes the recent activity in your network.
  4. Under the Email me when... section header, clear the check box next to each update type that you don't want to be notified about.
  5. Under the There is new activity in the following Groups section header, select the check box next to each group that you want to receive real-time email updates for. You can clear a selected check box to stop updates for that group.
  6. Click Save at the bottom to put all the changes that you've made into effect.

    query from a list box with multiple selections - Microsoft Community

    sg0824 say:

    query from a list box with multiple selections

    I have a form that will populate the criteria for a query.  The form has two fields to enter a start date and an end date.  It also has checkboxes to choose active or inactive.  The problem I have is a list of events that the user needs to be able to choose from.  I want them to be able to select one or more that will then show in the results of the query as well as the dates and active/inactive.  I honestly have no idea how to go about this.  Could someone out there please offer some suggestions?  Thank you in advance!
    ken sheridan say: There are a number of ways you can do this.

    1.  The simplest is not to open a query as a datasheet at all, but to open a form (which can be in datasheet view of course, though continuous forms view allows you to present the results in a consistent 'house-style).  The form is unfiltered by default and filtered by means of the WhereCondition argument of the OpenForm method.

    2.  The same can be done with a report.

    3.  You can open a temporary query as a datasheet by building the SQL statement and passing this to the following function:

    Public Function OpenTempQuery(strSQL As String)

        Dim qdf As DAO.QueryDef
        Static n As Integer
        
        n = n + 1
                   
        ' delete temporary querydef object if exists
        On Error Resume Next
        CurrentDb.QueryDefs.Delete "Temp" & n
        Select Case Err.Number
            Case 0
            ' no error
            Case 3265
            ' temporary querydef does not exist
            ' ignore error
            Case Else
            ' unknown error
            MsgBox Err.Number, vbExclamation, "Error"
        End Select
        ' create temporary querydef object
        Set qdf = CurrentDb.CreateQueryDef("Temp" & n)
        CurrentDb.QueryDefs("Temp" & n).SQL = strSQL
        ' open query and then delete temporary querydef object
        DoCmd.OpenQuery "Temp" & n
        ' delete temporary querydef object
        CurrentDb.QueryDefs.Delete "Temp" & n

    End Function

    4.  Create and save a query which references the controls on the form as parameters.  You cannot reference a multi-select list box directly, however, as it has no value.  You need to loop through its ItemsSelected collection in code, in a way similar to that below, and build a value list which you then assign to a hidden text box control on the form.  This text box is referenced as the parameter, but for this the IN operator would normally be used in the query, and this does not accept a parameter as its argument, only a literal string.  There are a couple of ways of simulating this at the following link though:

    http://support.microsoft.com/kb/100131/en-us

    The second option at the link, using the InParam and GetToken functions is more reliable as it handles strings which might be substrings of another value correctly; these can give false results with the first method, using the InStr function, unless it is further modified.

    5.  Create and save a query and then in the form's module build the complete SQL statement for the query and modify the querydef object's SQL property.

    I'll confine my detailed comments to 1 and 3 above, though in my view 1 (or 2 for a report) is by far the best solution.

    1.  In addition to the existing controls on the form add a button to open the form (frmEvents) bound to the unrestricted query, using code like the following, for which I've assumed that the query includes columns EventID (number), EventDate (date/time) and Active (Boolean, i.e. yes/no), and the dialogue form includes a list box lstEvents, a check box chkActive and text boxes txtStartDate and txtEndDate:

        Dim varItem As Variant
        Dim strEventIDList As String
        Dim strCriteria As String
        Dim ctrl As Control
        
        Set ctrl = Me.lstEvents

        ' build criteria for Active and EventDate columns
        strCriteria  = "Active = " & Me.chkActive & _
            " And EventDate >= #" & Format(Me.txtStartDate,"yyyy-mm-dd") & "#" & _
            " And EventDate < #" & Format(Me.txtEndDate,"yyyy-mm-dd") & "# +1 And "
        
        ' build value list for IN operator
        If ctrl.ItemsSelected.Count > 0 Then
            For Each varItem In ctrl.ItemsSelected
                strEventIDList = strEventIDList & "," & ctrl.ItemData(varItem)
            Next varItem
            
            ' remove leading comma
            strEventIDList = Mid(strEventIDList, 2)
            
            strCriteria = strCriteria & "EventID In(" & strEventIDList & ")"
            
            DoCmd.OpenForm "frmEvents", WhereCondition:=strCriteria
        Else
            MsgBox "No events selected", vbInformation, "Warning"
        End If

    I've not included any validation code for the txtStartDate and txtEndDate controls, which would be advisable, firstly to be sure valid date values have been entered, and that they are not inconsistent, i.e. the end date doesn't precede the start date.

    3.  This is essentially the same as above, but builds an SQL statement and passes it to the OpenTempQuery function.  It assumes the query is based on a table Events:

        Dim varItem As Variant
        Dim strEventIDList As String
        Dim strCriteria As String
        Dim ctrl As Control
        Dim strSQL As String
        
        Set ctrl = Me.lstEvents

        ' build criteria for Active and EventDate columns
        strCriteria  = "Active = " & me.chkActive & _
            " And EventDate >= #" & Format(Me.txtStartDate,"yyyy-mm-dd") & "#" & _
            " And EventDate < #" & Format(Me.txtEndDate,"yyyy-mm-dd") & "# +1 And "
        
        ' build value list for IN operator
        If ctrl.ItemsSelected.Count > 0 Then
            For Each varItem In ctrl.ItemsSelected
                strEventIDList = strEventIDList & "," & ctrl.ItemData(varItem)
            Next varItem
            
            ' remove leading comma
            strEventIDList = Mid(strEventIDList, 2)
            
            strCriteria = strCriteria & "EventID In(" & strEventIDList & ")"
            
            ' build SQL string and pass to function
            strSQL = "SELECT * FROM Events WHERE " & strCriteria  & _
                " ORDER BY EventDate"
            OpenTempQuery strSQL
        Else
            MsgBox "No events selected", vbInformation, "Warning"
        End If

    One thing to note about the OpenTempQuery function is that it can open multiple datasheets simultaneously, so you could change the criteria in the form a number of times and click the button, in which case the datasheets for each set of criteria would remain open simultaneously.  This is really what  wrote the function for, as a cheap and cheerful way of entering several 'what if?' scenarios and then being able to compare the results in multiple datasheets.  The same could be done with a form, but that would be a little trickier as it would require creating a Collection in code and assigning multiple instances of the form's class to the collection.
