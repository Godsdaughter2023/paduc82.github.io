Books that show up on Zune were synced there either via Audible software or Overdrive software.  These books are usually DRM protected, which means that even if you could copy them directly to your computer, you probably would not be able to read them, since the DRM protection would still be in place.

Eventually, you may purchase new books and you can use the Audible or Overdrive softwre to transfer the books to your Zune.

Word page width - Microsoft Community

john p_213 say:

Word page width

I have a chart on Word which I've created using the maximum Custom page size width of 55.87cm.  I want to add to the chart and need more width.  Is there a way Word will let me increase the size of the page past this seemingly arbitrary value?

hansv mvp say:

No, that is not possible, the 55.87 cm = 22 inches is a 'hard' limit.

Explanation:

Internally, page width and height are stored in a 16-bit integer with twips as unit, where 1 inch = 1,440 twips.

The maximum positive number that can be stored in a 16-bit integer is 32,767.

22 inches = 22 * 1,440 twips = 31,680 twips

23 inches = 23 * 1,440 twips = 33,120 twips

As you see, 23 inches does not fit in a 16-bit integer, so 22 inches is the maximum.

I was also very disappointed that IEEE format wasn't available, in Word 2016. However, I found the solution, and so far, it's worked for me. (No crashing, no unpleasantness, etc.).

Here's what I did.

How to Add New Reference Styles to Word 2016 on OS X

  1. Download the IEEE format (or any other referential format) XSL file of your choice from BibWord.
  2. Open the Terminal.app (from the control panel, using spotlight type "Term" and hit enter, whatever you prefer)
  3. Input these commands
    1. NOTE: you'll be asked for your password, this will be the password you use to login to your Mac. When you type in your password, it'll look like nothing is happening. That's what's supposed to happen. It doesn't show your password (or show you typing it with dots) for security reasons.
    2. You'll replace the "IEEE_Reference.XSL" with the name of the file that you downloaded from BibWord.

cd /Applications/Microsoft\ Word.app/Contents/Resources/Style/

chmod 664 IEEE_Reference.XSL

sudo chown root:wheel IEEE_Reference.XSL

Then you can close the Terminal, and restart Word! Then the next time you open word, check under Reference and the dropdown for the format and you should see the reference style you just added!

Sources for the Commands

  • Good on you for being careful, but I'm not trying to wipe your hardrive ;)
  • Here is a breakdown of the commands and the links to their man(ual) pages so you can confirm what I'm saying.
  1. "sudo" is the "super user do" command. To modify (e.g. add) something inside the Application directory, we have to be a "superuser".
  2. "mv" is the "move" command. We're moving the file you downloaded to the proper Style folder within the Word.app (you can't do it in Finder, because finder won't open any *.app directory).
  1. "cd" is the "change directory" command. To make our lives easier, we changed to the Word directory where we just moved the referential style file (that way we don't have to type /Applications/Micro... blah everytime).
  2. If you're wondering why I linked to the "linux" manpages, that's because Apple's manpage for cd sucks.
  1. "chmod" is the "change mode bits" command. What does that mean? It means change the permissions of the file.
  2. "664" Is the decimal representation of the permissions we're setting on the file. It means "The owner has permission to read & write to the file (6), the group has permission to read & write (6), and everyone else has permission to read (4)".
  3. Why are we setting the permissions? Because all the other XSL files that Word comes with are set with these permissions, so it's to be consistent.
  1. See above for "sudo" link. To set the owner to "root" (the super user) and group to "wheel", we have to be the super user.
  2. "chown" is the "change owner" command. We're using it to change both the owner and "group" the file belongs too.
  3. "root:wheel" is the "user:group" specification of the "chown" command. "root" is the super user, and "wheel" is the group.
  4. We're setting the user to root and group to wheel because all the other reference files that Word came with are set to root:wheel.
