¡firedog say:

OK, thank you. I wonder if you're misunderstanding the function of the Apply now button. If you set up a rule to act on received messages, it will start doing so as soon as you close the Rules dialogue. However, you might also want to perform the same action on messages that have already been received. That's what Apply now does. Not all types of rule can be 'applied now', but I was wrong to state in an earlier post that Forward rules couldn't be applied now. They can, because I just did it.

When you see that the rule has defaulted to storage folders, all you're seeing is the folder that would be used if you elected to Apply now. This is clearly a design decision; the box could have been left blank, ready for population by the Browse function, but this would have required an error message in case the user didn't browse and clicked Apply now anyway.
