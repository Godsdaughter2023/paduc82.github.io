bob jones aka: cybertaz say:

Further to Paul's reply, Content Controls are not supported in Mac Office. There are some that will work as designed if included in a document/template created using a Windows version, but they cannot be added or modified on a Mac. The display format of the Date Control is a property of the control - Neither your system settings nor your Default Date & Time in Word have any influence on it.

Additionally, in the past, quite a few template files were supplied with the Mac Office software & physically stored on the local HD. That no longer is the case... Office 2016 includes only 15 Word templates compared to more than 75 Print Layout templates included with Office 2011. There were another ~80 For Publishing Layout. Those template files readily could be opened & modified or customized. If you did download one of the online templates you did exactly that - the template file actually was downloaded to your local HD. That is no longer the case.

Although you still can modify the few that are locally installed, the vast majority of the available templates are stored on the Microsoft servers. When you select one of the online templates what you get is a new document based on that template. The template file, itself, is not copied to the local system. Even if you go to the site in your browser, the same is true.

Graham's macro doesn't display a dialog; it just saves the document with its current path and document name but in PDF format. It does what you described in your original post.

With your additional description of what you want to do, the code you need is only this:

    With Dialogs(wdDialogFileSaveAs)
        .Format = wdFormatPDF
        .Show
    End With

You can put this in a macro with some otherwise unused name, such as Sub SaveAsPDF(), and create a button on the Quick Access Toolbar or in a custom ribbon group to run it.
