brookzhang_msft8047 say:

Hi TT,

Email recall is an Outlook client feature. It’s not feasible to use it in OWA.

Besides that, it requires that both sender and recipient are using Exchange account in the same organization. Hotmail is not an Exchange account and outside your organization, so it’s not possible to use the recall for it.

Regarding how to perform email recall, please refer to this article: https://support.office.com/en-us/article/Recall-or-replace-an-email-message-that-you-sent-81c1ae4a-1ea3-4355-b05f-91785773ac15?CorrelationId=c0a22434-27c0-4892-8756-9d6bc1a1404d&ui=en-US&rs=en-US&ad=US

When an email is successfully recalled (you'll receive an email), recipients cannot read the sent email.

Also, email recall may be not successful if one or more of the following conditions are true:

  • The recipient is not using Outlook.
  • The recipient is not logged on to the mail service provider.
  • The recipient is using Cached Exchange Mode and is working offline.
  • The original message is moved from the Inbox. This can occur when rules are used.
  • The original message is opened first and marked as read. This can occur when the message is displayed in the Preview Pane or Reading Pane.

Brook

  1. 1.     The mail merge main document must be of the Letter type, though that does not mean that the output cannot be sent as an e-mail message.
