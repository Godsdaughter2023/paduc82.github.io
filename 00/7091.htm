JP Ronse,

I'm not following your instructions to do the group highlighting.  Can you please explain in more detail for me?

"This table in A1:C10, add in D1 FALSE...."
In your table, add a column (we will call it the helper column) and in the first row in that column, put in the text value "FALSE".  If you do NOT have a header row, insert a blank row at the top and put FALSE in the top row/column you added.

"and in D2: =IF(C2=C1,D1,NOT(D1)) and drag down."

When reading this formula, you need to understand what data is in each column.  Column C, in this case, is our column containing the duplicate values.  Column D is our helper column.  Row 1 is our header row, and the first row in your helper column you are hard setting to false.  Now to explain the formula.  Because the first row is our header row, and we've defaulted the first row helper column to false, we need to write the formula in the first row helper column.  In this example, that is cell D2.  So the formula for D2 says "Check the value of C2 (our duplicate data column) to C1 (the one above that).  Are they the same?  If so, set the column D value for this row to the column D value of the row above it.  If not, set the value for this row column D to the value OPPOSITE the value above it. 

"Select the range A2:C10, conditional format, use formula, =$D2 and apply desired format."

Select the columns you want to apply the conditional formatting to and click conditional formatting > new rule > Use a formula to determine which cells to format.  In the formula box, type =$(helper column letter)(row number that starts the dataset you selected to highlight).  It is important to know why the original author of this solution said "=$D2 and apply desired format".  The original author instructed the user to select cells A2 through C10.  That means that D2 would be the first row in our selected dataset to apply conditional formatting to.  Lets say you selected your entire worksheet and all columns/cells.  The first row in your selection is row 1, so your formula should say "=$(helper column)1".  If your selected dataset started on row 3, your formula would be "=$(helper column)3".  The conditional formatter will take this formula, apply it to the first row in your selected dataset, then increase the number in the formula. 

For example, lets say you selected your entire dataset and you put your formula in the conditional formatting as "=$(helper column)1".  The conditional formatter starts with row 1, the first row in your selected dataset, and say Ok, the value of (helper column) row 1 (the current row) determines whether or not I should highlight this row.  Because we set the value in our header row helper column to false, it would not highlight this row.

Now lets say you selected your entire dataset, but you did exactly what the original author said, and in your conditional formatting, you put in the formula "=$(helper column)2".  You've selected your entire dataset, so your first selected row is 1, not 2.  The formula will start on row 1, and say Ok, the value of (helper column) row 2 (even though row 1 was your first row in your selected dataset) determines whether or not I should highlight this row. On line 2, the conditional formatter looks at helper column row 3 (incremented by 1 for each row processed), so on until your formatting has been applied to your range.  This results in your highlighting to be off by 1 row.

Hopefully this explains what the original author wrote.  Credit to the solution should go to that author, this is just an explanation to implement the solution into your own scenario properly.
