While I mentioned a macro in that thread, there isn't any macro there -- and even if there were, we were discussing how to create the XE fields and not the issue of adding or repeating main entries at the tops of columns in the index.

jay freedman say:
The index you see is created by a single INDEX field, and you can get only what that field is able to produce. Unfortunately, the programmers who wrote the code to implement the field apparently gave no thought to this problem -- there is no switch or modification of the field that will ensure that each column begins with a main entry.

The only solution is completely manual: Once the INDEX field has been inserted and updated, place the cursor in the index and press Ctrl+Shift+F9 (the UnlinkFields command) to change it from a field to plain text. Then you can manually insert/repeat main entries and insert column breaks (Ctrl+Shift+Enter) until the index looks the way you want it.

If anything changes in the document -- if you add or remove text that includes index entries, or if repagination changes the page numbers that should appear in the index -- then you can manually edit the index as needed, or it might be quicker to delete the entire index, insert a new INDEX field, and redo the unlinking and all the subsequent changes.
