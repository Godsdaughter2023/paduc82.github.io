jllatham say:

I Have column Number, Need letter ID for the column

This probably isn't original (I seldom have original thoughts), but for a long time, to convert from a number representing a column to it's letter identifier was typically done with a bunch of MOD() math and the code I've seen for those all only involved columns with no more than 2 letters as the identifier.  The UDF below should work in pretty much any version of Excel you toss it into and it doesn't care how many letters need to be returned as the answer: that's up to the version of Excel you are using.

 

The function below will let Excel itself tell you the letters for the column based on a number representing the column.  By using Cells(row, column).Address and some footwork with text manipulation, we let Excel give us the answer rather quickly.

 

Anyone is welcome to include the code in any project they need it for - in other words, I'm putting it into the public domain.  Enjoy and I hope someone gets some good use out of it.

 

Function GetColumnID(columnNumber As Long, Optional returnAbsolute As Variant) As String
'Author: J.Latham 03 September 2013
  'INPUT:
  '       columnNumber (typed as Long for future change potential)
  '       a number from 1 to maximum number of columns
  '       permitted in the version of Excel being used.
  '       Optional returnAbsolute
  '       If TRUE (or non-zero number)then will return column letter ID with $ absolute
  '       If FALSE (or zero)then it returns column letter ID without the $.
  '
  'NOTE: using numbers other than 1 to represent TRUE for the
  'optional returnAbsolute argument can result in a performance hit.
  '
  'OUTPUT:  Letters representing the column idenfication
  '         If returnAbsolute = True (or non-zero), then $ is included
  '
  'ERROR:   columnNumber is always converted to a positive number,
  '         but if columnNumber is greater than the maximum
  '         number of columns permitted, then "INVALID!" is returned.
  '
  Dim tempID As String
 
  If IsMissing(returnAbsolute) Then
    returnAbsolute = False
  End If
  columnNumber = Abs(columnNumber)
  If columnNumber > Columns.Count Then
    GetColumnID = "INVALID!"
    Exit Function
  End If
  tempID = Cells(1, columnNumber).Address ' gives absolute reference
  tempID = Left(tempID, InStr(2, tempID, "$") - 1)
  If returnAbsolute Then
    GetColumnID = tempID
  Else
    GetColumnID = Right(tempID, Len(tempID) - 1)
  End If
End Function

Microsoft dropped support for databases in Access 97 and earlier .mdb format in Access 2013.

Access 2016 can still open databases in Access 2000 and Access 2002-2003 .mdb format without needing conversion.

a. user say:

The other fairly simple way to do it is to use the Caption mechanism, with some additions. 

The main difference is that that mechanism uses field codes which

 a. do not auto-update (you can select all the text using ctrl-A, then update all fields using F9)

 b. are slightly harder to change if you need to change the level (e.g. you had [1.A-1], [1.A-2], but you need to change them to [1.1.A-1] etc.0

For example, if your 1.1.A paragraph is a Heading 3 paragraph, then you can get "1.1.A" using

{ STYLEREF 3 \s }

and you can get "1", "2", "3", restarting at each new Heading 3 paragraph, using 

{ SEQ T4 \s 3 }

So you can get [1.1.A-1] using

[{ STYLEREF 3 \s }-{ SEQ T4 \s 3 }]

(All the {  } need to be the special field code brace pairs that you can insert using ctrl-F9 on Windows Word.)

a. user say:

[ - trim: the character "[" (following your example)

{ - open special field brace

STYLEREF - field code to reference a text with a specific style

3 - short for "Heading 3" - useful for internationalisation because the builtin Heading styles change 

\s - an undocumented switch - strips out any text from the Heading's autonumber, e.g. if the number is "Chapter 1", you get "1". It's equivalent to \r \t or \n \t. (These switches also strip some types of trailing delimiters such as "."

} - close special field brace
- trim: the character "-"

SEQ - sequence field code

T4 the sequence name for this sequence (could be any sequence name allowed in Word - I happened to call it "T4")

\s 3 - restart the numbering of the specified sequence (i.e. T4 in this case) at the Heading level 3

] - trim - the character "]"

doug robbins - mvp office apps & services (word) say:

The \s switch causes the sequence number to reset (restart at 1 in the absence of an \r # switch) after the Heading Level that follows the s (heading 3 in this case).

The T4 is the unique identifier for that sequence.

Following on from Karl, bob and Ken's answer, try this

Open your database
Create a new Module
Cut and past this into the new module then save it as anything you want but NOT Num2Roman - your can't use this name so save it as something else like  modNumberToNumeral

 

 

Cut and paste everything from the next next line

 

 

      Function Num2Roman(ByVal N As Integer) As String

        Const Digits = "IVXLCDM"

        Dim I As Integer, Digit As Integer, Temp As String

        I = 1
        Temp = ""
        Do While N > 0
          Digit = N Mod 10
          N = N \ 10
          Select Case Digit
            Case 1
              Temp = Mid(Digits, I, 1) & Temp
            Case 2
              Temp = Mid(Digits, I, 1) & Mid(Digits, I, 1) & Temp
            Case 3
              Temp = Mid(Digits, I, 1) & Mid(Digits, I, 1) & _
                     Mid(Digits, I, 1) & Temp
            Case 4
              Temp = Mid(Digits, I, 2) & Temp
            Case 5
              Temp = Mid(Digits, I + 1, 1) & Temp
            Case 6
              Temp = Mid(Digits, I + 1, 1) & Mid(Digits, I, 1) & Temp
            Case 7
              Temp = Mid(Digits, I + 1, 1) & Mid(Digits, I, 1) & _
                     Mid(Digits, I, 1) & Temp
            Case 8
              Temp = Mid(Digits, I + 1, 1) & Mid(Digits, I, 1) & _
                     Mid(Digits, I, 1) & Mid(Digits, I, 1) & Temp
            Case 9
              Temp = Mid(Digits, I, 1) & Mid(Digits, I + 2, 1) & Temp
          End Select
          I = I + 2
        Loop
        Num2Roman = Temp
      End Function

 

 

The above line was the last line of the code to cut-n-paste

 

Next
open the query the report is based on in design view
I assume the number field you want to convert to numerals is call MyNumberField - of course yours will be something else
In a blank column (in the Field Row - thats the top row) add this

FormatedNumber: Num2Roman([MyNumberField])

Don't forget to change MyNumberField to what it really is in your database

Next

Open the report in design view and from the list of available fields select FormatedNumber and drag it to where you want it in your report

Local administrators have the option for configuring the operational state of Windows Defender with Group Policy. You can use the Local Group Policy Editor (gpedit.msc) to toggle Defender on or off in Windows 10 Pro – but in the Home edition, you either have to edit the Policies key manually – or – if you’re not really comfortable with editing the registry, you can use a REG file to make the edit:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

Or, if you're not afraid of the command line, you can make the whole process quick and easy by using these REG commands. Just copy, paste, and enter the appropriate line at the elevated (admin) command prompt: 

Determine the present status of the Group Policy switch:

REG QUERY "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntispyware

Turn Off Windows Defender:

REG ADD "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware /t REG_DWORD /d 1 /f

Turn On Windows Defender:

REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware
