a. user say:

It is unlikely the hal.dll file is really missing or corrupt, but you can certainly try to replace it if you want.

First you would have to understand a lot about your hardware and motherboard and then figure out which of the 7 possible hal.dlls that comes with XP goes with your system.  When XP is installed, it figures out which of the 7 possible hal.dlls you need and installs the correct one from the installation CD on your hard disk.  Maybe you will get lucky and pick the correct one, or you could just try them all.

You can also try replacing the hal.dll file with some of the many files of similar name that are probably on your hard disk and maybe you will get lucky there too.

I think you will certainly get frustrated if you just start trying things that might work.

You don't need to try things, you need to fix things.

The more likely problem (which happens a lot) is that your c:\boot.ini file has somehow gotten messed up and XP is being told to look for the file in the wrong place and just can't find it.  It is easy to create a c:\boot.ini file that recreates this problem, then when you see it, you can learn how to fix it without trying things.

You can examine and fix your c:\boot.ini file easily using a bootable CD.  If you don't have a bootable CD, make one.

Lately, I am leaning towards the Hiren's BootCD which you can download from here:

http://www.hiren.info/

If you follow the directions, it will work the first time.  You don't need an XP installation CD and you don't need to try to learn anything about Linux.  It uses an interface that you are already used to.

After you follow the directions and make the CD, boot your afflicted system on it and choose the "Mini Windows Xp" option and you will see a familiar looking Windows desktop and interface that has just about everything you need to fix a non booting system.  There are lots of useful tools included in the CD that you can use to fix just about any XP won't boot problem - even lost or forgotten passwords (always popular).

It would not be a bad idea to use the Hiren's CD to run chkdsk on your HDD either.  Your problem could just simply be file system corruption (any power interruptions lately?)  If you have had a power interruption, aborted restart or ungraceful shutdown, your system may be restored to operation just by running chkdsk - it will not hurt to run it even if there is no file system corruption.

It really doesn't make any difference what you boot on, but somehow you need to fix your c:\boot.ini file.

From the Hiren's desktop use the Explorer icon on the desktop to navigate to c:\ and either fix your boot.ini using Notepad or just rename the c:\boot.ini file to something like c:\boot.ini.bak (so in effect you don't have one).  Then reboot on your HDD.

A single partition XP installation does not need a c:\boot.ini to start (it will complain but still boot), so if you can't figure out what is wrong with the one you have by looking at it and fix it, post the contents of the suspicious c:\boot.ini back here for analysis.

If you boot on something besides the Hiren's CD, run chkdsk /r and just rename your c:\boot.ini file to c:\boot.ini.bak, reboot without one and then fix the one that is broken.

Here is an example of a general purpose c:\boot.ini file that works in most single partition configurations:

[boot loader]

timeout=5

default=multi(0)disk(0)rdisk(0)partition(1)\WINDOWS

[operating systems]

multi(0)disk(0)rdisk(0)partition(1)\WINDOWS="Microsoft Windows XP Professional" /fastdetect /NoExecute=OptIn

 

It is easy to take that example c:\boot.ini and break it in such a way that your error message comes up.  When you know how to do that and understand it, it makes it much easier to fix when you come across it on some other system without trying things.

Once you get your system running, you need to figure out what happened to it and fix the problem that got you here in the first place.  It could have been a self inflicted wound or malicious software.  

Once you get running again, I would do this:

Perform some scans for malicious software, then fix any remaining issues:

Download, install, update and do a full scan with these free malware detection programs:

Malwarebytes (MBAM):  http://malwarebytes.org/

SUPERAntiSpyware: (SAS):  http://www.superantispyware.com/

They can be uninstalled later if desired.
