japanzaman say:

If you want to build animation in show mode successfully, you need to use an event handler so you can correctly control the the sequence of your game. The reason your shapes are disappearing is because a macro will effectively end an animation that it follows. In order to fire code after an animation you need to use the event SlideShowOnNext so the macro can pick up after the animations are finished firing. Here's the general idea.

1) Player chooses first card and your macro identifies this as the first card selected. It builds the exit animation for that card and fires it.

2) Your event macro then fires and clears the animation dock and hides that card.

3) Player choses the next card and you determine whether or not the cards match. If they don't bring the first card back into view and then set up an animation sequence to have the chosen card disappear and then have the two cards come back into view. Your event handler will then clear the animation dock after the cards come back into view.

If they match then you simply set another disappear animation, then hide the other two cards after with the event macro.

 

The point is that you should try to build, run, and clean all your animations up within your sequence. As long as everything is self contained then you can work animations into your game, keep score, etc. This is kind of complicated but you look like you should be able to do it. The trick is to have a sequence handler that instructs the macro what to do after a given animation fires. You can call this from a class module and use  a Select Case to choose from different sequences that the macro needs to follow depending where you are in the game. Your game isn't that complicated so there won't be too many sequences to mess with.

You can see the game I built here if you want to see how I put the event handler into the game.

The folder you have referenced is a normal Windows 8.1 folder and has nothing to do with the similarly named "Crypto virus".

The so called Crypto malware actually encrypts various common data file types in an attempt to extort you to pay for the decryption of these files.  None of the antimalware programs that I am aware of are able to decrypt these and instead simply remove the malware application itself, sometimes crippling any possibility of paying the ransom even if you wish to.

Generally the best way to protect from this is to have offline backups of all important files on either CD or disconnected USB drives at all times, since the malware will encrypt all files it can access if the writable drives are connected.

I've seen some indications that certain versions of the Crypto encrypted files might be recoverable, but no certain confirmation, so someone else will have to confirm the options if you must try to recover the encrypted files.
