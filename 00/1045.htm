per jessen say:

Try this one, assuming that your filtered data is extracted to Sheet1:

Sub Macro6()
' Macro6 Macro
' Keyboard Shortcut: Ctrl+r
Dim FilterRange
Dim SortRange As Range
Dim DestSh As Worksheet

With Worksheets("Data")
    Set FilterRange = .Range("A1", .Range("C" & Rows.Count).End(xlUp))
End With
Set DestSh = Worksheets("Sheet1") 'Change to destination sheet
DestSh.Activate

FilterRange.AdvancedFilter Action:=xlFilterCopy, _
    CriteriaRange:=Worksheets("Data").Range("Data!Criteria"), _
    CopyToRange:=DestSh.Range("A7:C7"), Unique:=False

Set SortRange = DestSh.Range("A8:C8", Range("A8:C8").End(xlDown))
DestSh.Sort.SortFields.Clear
DestSh.Sort.SortFields.Add Key:=DestSh.Range("B8", DestSh.Range("B8").End(xlDown)), _
    SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
   
With DestSh.Sort
    .SetRange SortRange
    .Header = xlNo
    .MatchCase = False
    .Orientation = xlTopToBottom
    .SortMethod = xlPinYin
    .Apply
End With
With DestSh
    .Range("A8") = 1
    .Range("A9") = 2
    .Range("A8:A9").AutoFill _
        Destination:=.Range("A8", .Range("A8").End(xlDown)), Type:=xlFillDefault
End With
End Sub

shane devenshire say:

You can treat it when it gets into Excel - create a new column

=--(C1&D1)

This assumes the $1 is in C1 and the 200 in C2.  It also assumes that the quotes were stripped.  Copy the formula down and then convert to values.

If the quotes came over do a Find & Replace to get rid of them and then the above formula.

If it won't come in correctly then clean it up in Excel.  Excel is great for that.  If you don't want to try my previous suggestion then try this one, assume here that the 1 goes into column A and the 800 goes into column B. In an empty column create the formula

=VALUE(A1&B1)

copy this formula down as far as needed and then copy all the formulas and choose Home, Paste, Paste Values and format to currency.  Then delete columns A & B.

cjgade say:

 

I also noticed this problem when I tried to import a CSV file and I am sad to see that no one at MS has fixed this issue yet.

When you try to import files containing ; , or other standard separator characters Excel, 2007 at least, will assume this to be a separator and put the following text in to a new column. This is when using file import. My only conclusion is that the import routine is bugged, it is not possible to get around this without changing the text in the files and few people would bother to do this.

The workaround for this in my case was to load the file in to notepad, select all (CTRL+A) and paste it in to Excel. This worked in my case at least. Though my issue was that I had some fields containing semi colons (;) in a comma separated file.

 

Carl

 

 

bill manville say:

When Excel's standard method of dealing with a CSV file doesn't give the results I want, I change the file extension to .TXT and use the text import wizard.  That generally gives good results if I set the correct parameters. 

Some software creates badly formed CSV files (for example one containing
Sales,$1,800,Up
instead of
Sales,"$1,800",Up
or
Sales,$1800,Up
)
In that case you can either import into Excel as text in column A and sort out the mess using Excel formulas (along the lines that Shane suggested) or you have to edit the file to make it comform to CSV format.
