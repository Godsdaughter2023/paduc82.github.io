If the IE9 Welcome page titled “Meet your new browser” keeps coming up and you want to disable it, use the local Group Policy editor and set PREVENT PERFORMANCE OF FISRT RUN CUSTOMIZATION SETTINGS:

  1. run GPEDIT.MSC from the Start button search bar;
  2. Expand USER CONFIGURATION, ADMINISTRATIVE TEMPLATES, WINDOWS COMPONENTS, INTERNET EXPLORER
  3. Double click “PREVENT PERFORMANCE OF FIRST RUN CUSTOMIZATION SETTINGS” (in the right pane)
  4. Set it to ENABLED and set GO DIRECTLY TO HOME PAGE from the drop down.

scottgem say:

Is it possible that InvTotal is Null in any records? That could cause this result.

And by the way this is not good design. As a general rule we don't store calculated values. Such values can be calculated on the fly in queries as needed. Why would you even need an OutstandingInvoices table? You should have an Invoices table and a query that shows Invoices that haven't been paid in full. You can show the total for an Invoice by adding the line items. You can show the balance by adding the Payments and subtracting from the calculated Invoice Total.

So the real answer to your problem is that you shouldn't be using Update queries like that. I would strongly suggest that you review your table structure and redesign.

Conditional formatting - Microsoft Community

trippap say:

Conditional formatting

If you create two conditional formatting rules for some range that result in the range being formatted with data bars base on cell values, and the types/colors of the data bars that result from application of the rules are different I would expect the second rule to govern the resulting format, but that is not the case. Only the first rule is applied, even though the Stop If True box is not checked, and although the Stop If True box is not grayed-out, you cannot check it for either of the two rules.

Reproduce the problem:
1) Select a range containing numeric values.
2) Create two conditional formatting rules for the selected range that result in different data bar formats.
3) Observe that the first rule governs the resulting format, even though Stop If True is not selected.
hansv mvp say:

When you have two or more rules that apply the same type of formatting, the first rule (counting from top to bottom) that evaluates to True determines the formatting, regardless of the setting of the "Stop if True" check boxes.

 

The Stop if True check box only comes into play if the rules apply different types of formatting that can be combined.

If the first rule sets the font color and the second rule sets the fill color of the cells, and if a cell satisfies both rules, both the font color and the fill color will be applied, unless the "Stop if True" check box of the first rule is set - then only the font color will be applied.

 

In your example, both rules apply data bars. A cell can't have two different data base at the same time, so the first rule has precedence. And since data bars cannot be combined, ticking the "Stop if True" check box would be meaningless.
