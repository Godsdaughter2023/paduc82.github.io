The error can be caused by a malware which changed the security settings of your .exe files (programs). Kindly run the Windows Defender or you may also run the Microsoft Safety Scanner to check and remove viruses, spyware, and other malicious software. (Note: Any data files that are infected may only be cleaned by deleting the file entirely, which means there is a potential for data loss.)

If you are still experiencing the same after scanning your system for viruses and malwares, you would need to change the security settings of the affected files by doing the following:

  1. Right-click the file/program which you are having errors with.
  2. Click Properties.
  3. Click Security tab.
  4. Click Advanced. A new window for the Advanced Security Settings should pop up.
  5. To the right of Owner field, click Change. The Select User or Group window should pop up.
  6. Under Enter the object name to select, type Administrators then click Check Names.
  7. Click OK.
  8. Click OK on Advanced Security Settings window. It will route you back to the Properties window.
  9. Click Advanced again.
  10.  Click Change permissions.
  11.  Look for Administrators under Permission entries, double-click it.
  12. Put a check on Full control and click OK all the way out of Properties.

Re-run the program afterwards and check if the issue persists. If so, then the next course of action is to get a fresh copy of the installers which you are having issues with by re-downloading those.

bruce chambers say: Dima7g wrote:
I have bought Win7 Home English edition. Can I use my product key and install a Russian version of Windows instead? If so, where can I get it?
No.  Product Keys are bound to the specific type and language of
DVD/license with which they are purchased.  An OEM Product Key will not
work to install a retail product.  An Italian Product Key will not work
with an English DVD. Bottom line: Product Keys and DVD/license types
cannot usually be mixed & matched.
