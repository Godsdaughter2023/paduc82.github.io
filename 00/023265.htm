MS Publisher Font/Line Spacing Issue with every Windows update - Microsoft Community

shaunc78 say:

MS Publisher Font/Line Spacing Issue with every Windows update

Since upgrading to Windows 8, and now with Windows 10, we've encountered an issue with every single update that Windows rolls out where the paragraph and font spacing/kerning in text boxes shrinks, leaving about 20% more empty space in the text box.  Needless to say, this throws off formatting of files, especially when aligned with graphics (which don't change their position on the page with this issue).  It's become an annoyance that we just anticipate every time that infamous blue screen appears with a Windows update.  This has happened consistently since the release of Windows 8 and has continued through 8.1 and 10.

I'm not sure if it's the way fonts are recognized and rendered from the operating system or what the deal is, but we use Publisher for our weekly church bulletins, and the file goes between 3 of us on staff each week.  When the "font issue" creeps on to one of our machines, the formatting looks very different on that person's screen, so that when they try to "fix" the spacing and send it on to one of us without the issue, we have to "fix" it again.  It's particularly noticeable when we have musical responses pasted into the service at particular times.  When the spacing issue occurs, sometimes the music ends up in a completely different spot due to the line spacing shifting.  Where the title of the song was originally right above the music, after the update, it can be as many as 3 or 4 lines above where it was supposed to be and the music then appears in the middle of a spoken response, etc.

After months of searching online a year or 2 back, the only solution that I've found that works is to have the Windows 7 default fonts saved in a folder on the machine.  You copy and paste those fonts into the regular C://Windows/Fonts directory, making sure to check that it's ok to replace the fonts (all 426 of them), and magically when you open Publisher again, the font spacing is back to normal.

I'm not sure why Microsoft hasn't put out a fix for this as it literally happens to all 3 of our computers every time Windows runs an update.  I've probably run this fix 5 or 6 times between my desktop and laptop over the past 2 years and we've done the same on the other 2 staff members' machines.  We have over 10 years worth of files that we refer back to when planning our services and newsletters, and without this little "fix," opening those older files is like a guessing game as to what will show up on screen.

davidf2 say:

When MSFT introduced Win8 Publisher users found that text rendered a different size than it did in Win7 and older. This meant that if you opened a Pub file that was created on a Win7 machine or older, on a Win8 or Win10 machine with Pub 2010 or newer you had a formatting and layout issue as you described. If you used Pub 2010 or newer on a Win8 or Win10 machine the one workaround we found was to install Win7 fonts. You could run Pub 2007 and 2003 on Win10 and the formatting was preserved. Here is just one reference that describes this text size issue:

July 2015 - Reference: Publisher 2007 and Publisher 2013 paginates differently: http://answers.microsoft.com/en-us/office/forum/office_2013_release-publish/publisher-2007-and-publisher-2013-paginates/2ff3812a-5874-46dd-8c57-87aa3f86a321 - Well tested and documented thread showing the text rendering issue continues in Win10, but is limited to Pub 2013 and probably 2010. If you install Pub 2007 or 2003 in Win10, Win8, Win7, WinVista or WinXP then the files still render text the same and are cross OS compatible.

Based on what you are reporting the Win7 fonts are now being replaced/updated every time you do a Win10 update. That is new information and if true then I fear the Win7 font workaround is no longer viable. Your only recourse is to reformat and tweak the layout of your old files using the new Win10 fonts and text size. This also means that everyone that edits that file must be using Win10 or Win8 machines and Pub 2010 or newer as the newly formatted files will not render correctly on Win7 or older machines.
