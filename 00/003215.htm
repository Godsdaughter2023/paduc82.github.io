joeu2004 say: I wrote:
I was going to suggest the use of non-volatile VBA UDFs.  But it appears that the formulas that call them are recalculated when the file is saved if Automatic calculation mode is selected.

Aha!  My bad!  I was making changes in another VBA module just before the saves, and apparently that causes all formulas with UDFs to be recalculated when the workbook is subsequently changed or saved.  (Makes sense.)

So you might consider using non-volatile VBA UDFs like myRAND and myRANDBETWEEN below.

Call myRAND and myRANDBETWEEN the same way that you call Excel RAND and RANDBETWEEN, namely RAND() and RANDBETWEEN(low,high).  Thus, you can use Find-and-Replace to change all existing formulas.

The first set of UDFs below use VBA Evaluate to utilize the Excel RAND and RANDBETWEEN functions.  I do that to take advantage of any improvements in random number generation, notably in XL2010.

However, VBA Evaluate is  very  slow relatively.  If performance becomes an issue, consider using the second set of UDFs below, which use VBA Rnd instead.

Caveat:  As noted above, apparently changes to  any  VBA module (in the VBA project?) causes expressions that call myRAND and myRANDBETWEEN to be recalculated.  I suspect the only way to have absolute (manual) control over when random expressions are recalculated is to use copy-and-paste-special-value as I suggested previously, or to use macros to generate either a set of random constants or the value of the random expressions themselves as constants (i.e. implement the "formulas" in the macro).  Presumably, the latter would also be the fastest approach as well.

-----

UDFs using VBA Evaluate....

Function myRAND() As Double
myRAND = Evaluate("rand()")
End Function

Function myRANDBETWEEN(lo As Double, hi As Double) As Double
myRANDBETWEEN = Evaluate("randbetween(" & lo & "," & hi & ")")
myRANDBETWEEN = Int(lo + (hi - lo + 1) * Evaluate("rand()"))
End Function

EDIT....  myRANDBETWEEN using Evaluate("rand()") is 85% faster than using Evaluate("randbetween(...)") on my computer with XL2003.  This implementation also provides some flexibility that XL2003 RANDBETWEEN does not have.  YMMV with XL2007 and XL2010, where RANDBETWEEN is integrated into Excel.

 

UDFs using VBA Rnd....

Private first As Long

Function myRAND() As Double
If first = 0 Then Randomize: first = 1
myRAND = Rnd
End Function

Function myRANDBETWEEN(lo As Double, hi As Double) As Double
If first = 0 Then Randomize: first = 1
myRANDBETWEEN = Int(lo + (hi - lo + 1) * Rnd)
End Function

Report junk email and phishing scams in Outlook on the web

As for someone fraudulently providing this to a reputable financial institution - not likely but very likely a phishing scam. There are any number of ways for a spammer to get your email address for the purposes of sending any kind of spam message (phishing or otherwise). One way would be that you used your email address to subscribe to something and the database containing the subscription info has been compromised.

One way to check to see if your email address comes up on a compromised database is by checking it on the following website (created by a Microsoft MVP so can definitely be trusted)

';--have i been pwned?

*** while the spammer/hacker has gained access to your email address and the password that was used to sign up to the applicable website where the database has been breached, they would not get access to your email account unless it's using the same password as that shown in the email message. Also, you should not assume that the above website is a comprehensive list of every site that has been breached.

If your email address does come up, you certainly want to ensure that you have not re-used the same email address/password on critical sites/services (email, finiancial, Amazon, PayPal, NetFlix, Apple etc etc etc). If you have, strongly suggest that you change the passwords for those critical sites.

somasundaram pandiaraj say:

There are some added and improved features that have, because of the way they're implemented in Word 2007, changed the methods used in prior

versions.

Basically, if an item in Word 2007 has white pearl sizing handles (new graphics object) it usually can't be directly grouped with an

item that

has flat square sizing handles (old/classic graphics engine object) or with other 'white pearl handled' items.

Pictures inserted via Insert=>Picture into the body of a Word document need to have their Text wrapping style set to something other

than 'inline with text' to be able to group them (right click on an inserted graphic to use Text Wrapping settings or double click

the graphic to get to the drawing/picture tools in the ribbon).

In 'old' Word versions (or 'Classic' as MS calls the last version before 2007) the graphics/pictures and shapes from

the MS Drawing Objects engine were managed within Word's capabilities.  For Office 2007 there was a new graphic engine created and,

unfortunately for us, it was only partially implemented in Word 2007 (for example Word doesn't have 'new' WordArt, only classic

while Excel and Powerpoint have new WordArt <g>).

Because of that partial implementation, the inserted pictures and SmartArt are managed by the new Office graphics engine, while the

inserted shapes and  shape handling is still managed in Word.  The basic result is that Word's 'classic' grouping and multiple

object selection tool don't 'see' the inserted pictures to allow you to group them with the shapes.  If you select a picture that

you inserted and right click, copy then on the Home tab look at the Paste Special Dropdown the graphic type is

"Microsoft Office Graphics Object", rather than the Word classic

"MS Office Drawing Object".

When you save in 'compatability' mode (.doc) the pictures are reset to 'classic' types, but if you then upvert (save as the new file

formats) the pictures will tend to break away from the grouping.

There are several workarounds (and hopefully one of the folks here who still program <g> will find a macro solution to this as

well).  Here are some you may want to consider.  The workarounds give the 'new items' the appearance to Word of being 'classic'

objects.


1. Work in 'compatability mode' (i.e. save in Word .doc format).  Lets you work as before, but turns off some of the newer features

in Word 2007 (as Herb mentioned)

2. Work within a Word drawing Canvas:

Use Insert=>Shapes=>New Drawing Canvas (at the bottom of the shapes dialog) and Insert all of your items to group into the canvas.

A canvas automatically groups/contains the pictures and shapes within a single frame and within the canvas you can selectively have

subgroups (i.e. graphics are 'equal' within a drawing canvas.) Separately, a canvas can be grouped with drawing items outside of the

canvas.

3. Revert to 'classic' by extracting items from a drawing canvas.

A variation of item 2 allows you to move items out of a drawing canvas back into Word's "regular" area where they can be handled as

Word 'classic' objects, even in .docx files.

While in the drawing canvas created in 2., above, after applying shadows, styles, etc to the items, use the shift key and click on,

for example an inserted picture and a shape, right click and group them.

4. Use a textbox shape to hold the inserted picture.

Use Insert=>Textbox=>Draw a Textbox (bottom of the gallery), then inside the text box use Insert=>Picture or double click the

textbox and use Textbox Tools=>Format=>Shape Fill=>Picture.

Use conditional formatting to shade the rows.

 

Select the rows where you want this then:-

 

Format|Conditional format|Select formula is and use the formula below and apply the fill colour you want

 

 

To shade even number rows

=MOD(ROW(),2)=0

 

or this if you want to shade odd numbered rows

=MOD(ROW(),2)=1
