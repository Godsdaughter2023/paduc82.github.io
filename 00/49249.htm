cbarnhorst say:

The items in Program Files and Program Files (x86) are programs that have to be installed in order to function.  Just copying them to corresponding folders in Windows 10 does not install them.  That has to be done in Windows 10 the same way it was originally done in Windows 7.  That is, you would have to run the programs installer.  If you disturb items in the windows.old folder you may not be able to rollback to Windows 7 within the 31-day grace period.  

When you "move" items in the windows.old folder, do not use the MOVE function.  Use the COPY function.  Right click on the item and keep the right button down.  Then drag to the new folder and release the button.  A menu will appear.  Choose MOVE.

It is important to make a system backup on external media, like an external usb drive, before upgrading from one Windows to another so that you can return to the old Windows without depending on the rollback feature, windows.old folders, and such.  Everyone should have a backup plan for regular backups anyway.  I hope you did.

       3.      When you move a folder or file within the same partition, it retains its original permissions.

       3.    When you move a folder or file within the same partition, it inherits the destination folder permissions.

Is this correct?  I've searched online, but I can't find anything that mentions a difference in NTFS permission handling between XP and Windows 7.  This is, however, what I've observed.  I just want to make sure this is normal behavior, and not some fluke on my machine.  I actually rather like the new behavior.  It makes more sense that when copying or moving a file, regardless of whether it's on the same partition or not, it will inherit the destination folder permissions.

 

 

shekhar s say:
Hi,
 
 
This question seems to be a difficult one but the answer is very tricky and short. It depends upon the file system rather than the operating system. A file or a folder may inherit the destination folders permission or the parent folders permission and this depends upon a setting.
Here is this setting for Windows Vista and Windows 7:
a. Right click on the folder and go to properties.
b. Now browse to the security tab.
c. Click on advanced under Permission for SYSTEM.
d. Below the option change permissions place a checkmark next to Include inheritable permissions from this objects parent.
 
Now if you copy or move this folder to a different location it carries the permissions as on the parent folder and if this option is unchecked the folder will inherit destination folders permission.
 
For Windows XP you may refer to the articles below.
How permissions are handled when you copy and move files and folders
http://support.microsoft.com/kb/310316/en-us
 
How to set, view, change, or remove special permissions for files and folders in Windows XP
http://support.microsoft.com/kb/308419
 
Hope this helps.
