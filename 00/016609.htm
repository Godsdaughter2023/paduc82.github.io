Sub somesub()

bernie deitrick, excel mvp 2000-2010 say:

There is an issue with specialcells if the number of cells or areas (I forget which) is greater than 8,192.  Any way, there is no need to use that method in your code. To delete all the rows where a cell in a column is blank, simply sort based on that column (either way, Excel will put blanks at the bottom), then go to the bottom of the column, go up to the first blank, select from there to the bottom and delete rows. If overall order is important, re-sort your data, either based on its original sort key or one that you added prior to the first sort.

In code, if you don't need to re-sort:

With Sheets("Sheet1")
    .Cells.Sort Key1:=.Range("B1"), _
        Order1:=xlDescending, _
        Header:=xlYes, _
        Orientation:=xlTopToBottom
    .Range(.Cells(Rows.Count, 2).End(xlUp)(2), _
        .Cells(Rows.Count, 2)).EntireRow.Delete
End With

 Note, too, that this is _way_ faster than deleting rows that are separated, often by a factor of 100 or more, depending on the size of your file.

HTH, Bernie

jim thomlinson say:

I am assuming that your issue is that you have more than 8000 blanks. A range can not have more than 8000 disconnected areas. This should keep you under that limit...

Sub DeleteRows()
    Dim rngToSearch As Range
    Dim rngBlanks As Range
    Dim lng As Long
    Dim lngLast As Long
    Dim wks As Worksheet
   
    Application.Calculation = xlCalculationManual
    Set wks = Me
    lngLast = wks.Cells(Rows.Count, "B").End(xlUp).Row
   
    lng = 10000
    Do While lng < lngLast
        With wks
        Set rngToSearch = .Range(.Range("B1"), .Cells(lng, "B"))
        End With
        On Error Resume Next
        Set rngBlanks = Nothing
        Set rngBlanks = rngToSearch.SpecialCells(xlCellTypeBlanks)
        On Error GoTo 0
        If Not rngBlanks Is Nothing Then rngBlanks.EntireRow.Delete
        lngLast = wks.Cells(Rows.Count, "B").End(xlUp).Row
        lng = lng + 10000
    Loop
    Application.Calculation = xlCalculationManual
   
End Sub

greg maxey say:

Charles,

 

You're right. As long as you can meet your needs with the general AutoText gallery then no separate code is needed.  If you want to use one of the ohter building block galleries and unlimited categories then you would use something like this:

Option Explicit
Private Sub cmdOK_Click()
On Error Resume Next
'Delete AutoText Entries and repopulate
With ThisDocument.AttachedTemplate
  .BuildingBlockTypes(wdTypeCustom1).Categories("Email").BuildingBlocks("MyEMail").Value = Me.txtBB1.Text
End With
Me.Hide
End Sub
Private Sub UserForm_Initialize()
'Get current AutoText values and populate text boxes
On Error Resume Next
With ThisDocument.AttachedTemplate
  Me.txtBB1.Text = .BuildingBlockTypes(wdTypeCustom1).Categories("Email").BuildingBlocks("MyEMail").Value
End With
End Sub

 

Of course this can't be used in Word2003.
