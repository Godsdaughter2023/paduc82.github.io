For example: one patient has been seeing Dr. A for 6 times within 2 years and another patient also has been seeing Dr. A for 8 times. I wish that It will show 2 in the CN's group footer instead of 14.

Chi

dhookom say:

I typically use Ken's solution but you can also do this with a group by query that groups by CN#. This query must use the same filtering as the main/existing report.

 

qgrpCN

 

SELECT Count(*) as NumOfCNs
FROM (SELECT DISTINCT [CN#]
FROM tblYourTableName);

 

Add this query to your report's record source and don't link it to any other query/table. Add the column [NumOfCNs] anywhere in your report.

ken sheridan say: To count the number of distinct instances of a patient in the period covered by the report include a hidden text box, txtPtCounter say,  in the group header or footer of the patient group, with a ControlSource property of  =1, and a RunningSum property of Over All.  Then in the report footer, not the group footer, you can include a visible text box whose ControlSource property references the hidden text box with =[txtPtCounter ].

I am not sure if this has already been discussed but in the Windows Technical Preview, Cisco AnyConnect VPN does not work properly. Although I have found a fix, it's easy if you know what your doing but if you don't know what your doing it's not very simple.

I use the VPN software to allow remote connection to my PC at work when I am not there. I am currently using the Microsoft Surface Pro 3. When I installed the VPN it worked fine on Windows 8.1 but when I upgraded to the Windows Technical Preview it kept giving me a connection error, the error said "Failed to initialize connection subsystem." 

If you have the error the fix is to make a small edit to the registry, and I did find myself to be pretty bold when doing that on the Surface Pro 3 since this thing isn't even 3 weeks old yet.

Please follow the Step-By-Step Procedures:

Step 1: Navigate to Run by going to Start and searching "Run"

Step 2: Next type "regedit" and hit the enter key

Step 3: Navigate to the provided area:

HKEY_CURRENT_USER\Software\Microsoft\Win

Step 4: Create a New Key that says GlobalOfflineUser

Step 5: Give the key a value of 1

Step 6: Restart your system, load the VPN and it should be able to connect.

Thanks,

Ian
