It does give you an 8 bit value, but with only 7 significant bits. If you want a 7 bit answer, you cannot convert any number beyond 128. If you just want to drop the LSB, simply divide by 2. This will work up to 255:

=DEC2BIN(INT(A1/2),7)

Help with Random Sampling and Monte Carlo Simulation - Microsoft Community

bulle_lon say:

Help with Random Sampling and Monte Carlo Simulation

I have 100 columns of data with 100 observations in each. The numbers are essentially randomly generated using Rand(). Imagine each column represents historical monthly mutual fund performance data

I calculate the average of each column (to get the monthly average performance of each mutual fund)

I now want to run a Monte Carlo simulation, where I select three averages at random and calculate the average of the three. I want to repeat this experiment 1,000 times.

Obviously the Data Table Function is helpful, when performing the Monte Carlo Simulation.

My problem is that I have used the Sampling Function in the Data Analysis Utility Pack, which doesn't contain a formula. hence, when running the Data Table the derived sample remains static as does the average.

I suppose I could record the function as a Macro, but I would have to repeat this 1,000 X 100 = 100,000 times, which exceeds my limit for patience.

Any suggestions?

mike middleton say:

If you want the contents in the 100-by-100 range to be dynamic, you could use =RAND() in each of those cells.

Or, use a transformation of the uniformly-distributed RAND in each cell where the transformation reflects the historical data.

For example, for a normal distribution (bell-shaped density function) use

     =NORMINV(RAND(),mean,stdev)

in each cell, where mean and stdev are numbers or references to cells containing the numerical values.

This approach uses only worksheet functions (which are relatively fast) and avoids using VBA (which would be relatively slower), but it still may exceed your limit for patience.

I suggest you consider testing this approach using fewer than 100 columns, fewer than 100 observations, and fewer than 1,000 replications. For example, try 20 columns, 20 observations per column, and 100 replications.

Disclaimer: I'm the author of SimVoi, a Monte Carlo simulation VBA add-in. SimVoi has many random number generator functions for distributions shapes other than uniform and bell-shaped normal, and SimVoi automates the Data Table replications and creates summary statistics and charts. But once you have set up your worksheet model, SimVoi would not perform the simulation faster than using a Data Table. Excel's native functions are generally faster than a VBA add-in.

paulhermans say:

I think you could do this using the RANDBETWEEN(), INDIRECT() and ADDRESS() functions in Excel.

Something like this (not pretty but will be good for explanation):


Cells B2:E10 are all random #s using the RAND() function.

Cell B12 is  =AVERAGE(B2:B10)

Cell C12 is =AVERAGE(C2:C10)

etc (could go on for many, many columns)

Then rows 15 through 24 represent a trial, in which I chose 3 columns randomly and then averaged the averages of those columns.

B15   =RandBetwee(1,4)               used to generate a random column "number" to use

C15   =RandBetween(1,4)

D15   =RandBetween(1,4)

Then comes the fun...

E15  =INDIRECT(ADDRESS(12,B15+1))

F15  =INDIRECT(ADDRESS(12,C15+1))

G15 =INDIRECT(ADDRESS(12,D15+1))

Let me explain the formula in E15 and the same logic applies to the others.

B15 contains the column number we are interested in, but since I started with column 1 in B, I need to add 1 to make them match.

Address(12,B15+1) means to generate a string for the address of the cell on row 12, with column of the value of B15+1

In this case that is like saying Address(12,5)  which would produce $E$12

Then the INDIRECT("$E$12") would get the value of the cell E12

E15  =INDIRECT(ADDRESS(12,B15+1)), together these end up being the value in column 4, row 12.....or the average of column 4's data.

That was a rough explanation, but hopefully it makes sense.

Finally, H15   = Average(E15:G15)   or the average of the averages of the randomly selected columns of random numbers :-)

Then I just copied the row 15 down and made another trial.....could copy this down many times. I did it about 16,000 rows down and it only took about 1/2 second to calculate all of it.

Hope this is what you were asking about :-)
