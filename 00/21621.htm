Master documents in Word are a kludge to create a larger document from smaller parts. Publisher doesn't really need this because it is a more modular program. That is, you can insert and delete pages and sections without affecting the neighboring content. This is more difficult in Word.

davidf2 say:

Generally most people find Word the best choice for books and large documents as it offers some features and functionality not found in Publisher.

Some people will break down books and such into chapters, or separate files, when they use Publisher in order to keep the file size down and make it more manageable. However there is no easy way of combining multiple Publisher files when it is time to submit the project to a printer or publisher. You can submit multiple PDF files or use a PDF tool that allows you to append a PDF file and combine your multiple Pub files that way.  I like the freebie primopdf for that purpose. spartacus38 say: I was thinking along the lines of being able to link in the files while not having to follow the link to look at it. I want to have them able to be edited outside of the main document and update inside. Also, I wanted cohesive page numbers without having to do it manually (pdf option).

john korchok say:

Those features work better in Word. The Master Documents feature has a bad rep online because people try to use them as permanent document organizers. When used this way, you can experience document corruption over the long term.

The best idea is to create a folder of Word documents, say one for each chapter. These are updated individually. Then at print time, create the Master document with continuing page numbers (and a table of contents, if needed) and print the file. Then dump the Master document.

It's possible to write a macro that creates the Master document from the Word files in the folder, applies the page numbering and prints the file. This makes the process fast and easy.
