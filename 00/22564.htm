Meanwhile, try this:
1. On a laptop running Windows 7, connect to the wifi at your office. Confirm that you have internet access on the laptop.
2. Go to Control Panel/Network and Internet/Network and Sharing Center
3. In the middle of the screen, in the right-hand column, it should say "Connections: Wireless Network Connection". Click on this.
4. Click "Wireless Properties"
5. Click "Security"
6. From top to bottom, you should see: WPA2-Enterprise, AES, Protected EAP (PEAP). Click on "Settings"
7. You should now see a check mark next to "Validate server certificate" and "Connect to these servers:". (If "Validate server certificate" is not checked, check it manually and reconnect). Write down the web address of the server, in case you need it later. Scroll down on the list of "Trusted Certification Authorities" until you see one that has a check mark next to it. Write down the name of the certification authority (e.g., "Equifax Secure Certificate Authority"). Confirm that "Select Authentication Method" is "Secured password (EAP-MSCHAP v2) (don't change any settings, just make a note if these are not your settings).
8. On your Nokia phone, go to EAP-PEAP settings/Authority certificate. From the pull-down menu, select the same certification authority that had a check mark next to it in step #7 above.

Try to connect now and let me know if it works.

 

This worked for me and allowed me to connect my Nokia C7 to the wireless network at my workplace, a major university in the U.S. 

 

The rationale behind the above method is that most servers do have a certificate even though they allow connecting initially without the certificate, as long as your wireless device also allows it (Symbian does not, unfortunately). You just need to find out what that certificate is.

kj55552009 say:

If you cannot find the correct certificate on your phone in step #8 above, you will have to download the certificate to your phone and install it.

 

To do this, use a laptop running Windows 7 to connect wirelessly to the WPA2 network. Go to step #7 above and scroll through the pull-down list. Write down the names of the certificates that have a check mark next to them. If there is only one certifcate with a check mark, that is the certificate you need to have on your phone. If there are several certificates with a check mark, uncheck them all, then reconnect. Windows may ask you to provide your userid and password to connect. After you are connected, go back to the list of certificates. You should now see only one certificate with a check mark next to it.

 

Again using your laptop, click on the Start button, type certmgr.msc into the Search box, and press ENTER. Find the certificate that you identified above. It should be under Trusted Root Certification Authorities or Third Party Root Certification Authorities.﻿ Right-click on the certificate, the click All Tasks/Export. Export the certificate as a DER (.cer) file to your desktop. Copy the exported certificate file anywhere to your phone.

 

On your phone, use the file manager to navigate to the directory where you copied the certificate file. Tap on the file to install it. Now go to step #8 above, and you should see the new certificate in the pull-down menu. Tap on it to select it.

 

Your phone should now be able to connect to the WPA2 network. If your WPA2 connection works fine for months, and then one day stops working, a likely reason is that the person managing the wireless server decided to change the certificate on the server. (This happened to me recently). This is not a problem for anyone else except owners of Nokia phones, as all other wireless devices will automatically download and install the new certificate. But if you have a Nokia phone, you will have to follow the procedure above to find out what the new certificate is, and then either activate it on your phone or download and install it if necessary.

rich007a say:

Try this:

Option Explicit

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
    Dim Myname As String, Mypath As String
    Dim a As Long

    On Error Goto ErrorHandler
    Application.EnableEvents = False
    Myname = Application.ActiveWorkbook.Name
    Mypath = "J:\Apg\Proposals\Resource Input\"
    a = MsgBox("Do you really want to save the workbook?", vbYesNo)
    Cancel = True
     If a = vbYes Then
        Application.DisplayAlerts = False
        Application.ActiveWorkbook.SaveAs (Mypath & Myname)
        Application.DisplayAlerts = True
    End If
ErrorHandler:
    Application.EnableEvents = True
End Sub

 

Your "Application.ActiveWorkbook.SaveAs (Mypath & Myname)" causes the BeforeSave event to fire, whichis why you see it twice.  Disable events with Application.EnableEvents = False at the start and re-enable at the end.

 

Hope that helps.
