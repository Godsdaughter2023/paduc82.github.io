jllatham say:

I believe you're correct in saying that Access doesn't work as a web data source.  I've used it at a site as a local database from which some .php pages referenced the database there.  But it would seem to be a bit of extra effort to create the database, install it on a web-accessible location and then build a couple of web pages to act as a front-end to it.  Could be done, but a bit of an effort.

I think that IF all the places that a student could be checking in are on the same network, the best way to go is probably to convert the Excel data into a simple Access database - probably a single table with their ID as the key field in it along with a form to check in the students.  Separate the backend from the front-end (split the database), put the backend with its table in a network accessible folder and install the front-end on each of the 3 machines and work that way.

One argument you can use as a convincer to go this route is that of security of personal information: putting names and student IDs on the web presents a potential security problem, keeping it all within the school's network allows more security control.  I'd also consider setting up the Access database to at least require login to use it at all.
