If I grant Send As, User A cannot access the Inbox (Outlook says "Cannot display the folder. The Inbox folder cannot be found")

robert__p say:

Hi Anthony,

By default, if we grant the full access permission of a mailbox to another user, the mailbox will appear on the user’s Outlook client automatically. This function is called automapping.

To remove the automapping, please firstly remove the full access permission from the user and then use PowerShell to grant full access permissions back to the user for the mailbox, but don't enable automapping. Here are the steps:

1. Connect to Exchange Online Using Remote PowerShell.

2. Remove full access permissions for the user from the mailbox. This removes automapping.

Remove-MailboxPermission -Identity <Mailbox ID1> -User <Mailbox ID2> -AccessRights FullAccess

3. Grant full access permissions back to the user for the mailbox, but don't enable automapping.

Add-MailboxPermission -Identity <Mailbox ID1> -User <Mailbox ID2>-AccessRights FullAccess -AutoMapping:$false

Here is the article for your reference:

https://support.microsoft.com/en-us/kb/2646504

Then the user can sign in to Outlook Web App (https://outlook.office365.com/owa) with the Office 365 account. Click the picture on the upper right corner of the page, select Open another mailbox. Enter the address of the mailbox and click Open.

Regards,

Robert
