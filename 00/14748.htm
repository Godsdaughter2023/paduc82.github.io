graham mayor say:
This becomes much simpler if you use a range. Insert the field into a range and then unlink the fields in the range e.g. the following sets the selection point as a range and inserts the field into the range

Dim oRng As Range
Set oRng = Selection.Range
oRng.Fields.Add oRng, wdFieldIncludeText, """c:\\path\\filename.docx""", False
oRng.Fields.Unlink

You could also insert the whole file without using the field.

Dim oRng As Range
Set oRng = Selection.Range
oRng.InsertFile """c:\\path\\filename.docx"""

For the basics on ranges see http://gregmaxey.mvps.org/VBA_Basics.htm

a. user say:

Have you verified it's the correct time zone?  If you set it manually but the time zone is wrong, it may automatically change the time to match the time zone instead of maintaining the time from your manual time setting.

If the time zone is correct, then can you verify that the NTP server is working and that it is updating?

If the time zone is correct, but it's automatically setting the wrong time, you may want to choose a geographical location with the same time zone as you, to see if the problem is unique to what Windows thinks it should be for your location but it's correct for other locations with the same time zone.

a. user say:

Howdy MarkusEx,

Thank you for posting on the Microsoft Support forums.

It's possible that the problem causing the few apps to fail to start is also causing the update issue. Because of that, use of the Nokia Software Recovery Tool may be required. There are a few things to try before that to see if any behavior changes or red flags appear:

First thing to do would be to uninstall/re-install the Preview For Developers app and ensure the Enable Preview For Developers checkbox is checked inside the app itself.

Secondly, manually set the date/time. Sometimes the date on the phones can become incorrect which will cause issues with security certificates among other things. On the phone, go to Settings > Date + Time and turn Set Automatically to Off. Then proceed to manually correct the date and time.

Lastly, check the following small things to ensure they aren't the problem:

  • There is enough free space on the device's internal storage (not including an SD card). 2 GB is normally enough.
  • That the device is connected to a Wi-Fi network.
  • That the battery is over 50% charged.

Once those have been, try checking for an update once more and see if the behavior has changed. If it hasn't then using the Nokia Software Recovery Tool would be the next step since a hard reset has already been tried.

 

I hope this information helps to address your issue. As always if you have any additional questions or concerns, please feel free to reply to this post and we'll be happy to assist.

Regards,
Bryce B
