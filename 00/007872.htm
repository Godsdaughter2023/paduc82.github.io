jsv_450 say:

LibSecret is a backend service. Both Kwallet and Gnome use this. But there is a critical difference to how it is used, and Skype (or some other program) needs to access KWallet specifically to get to Libsecret.

Gnome opens the Libsecret wallet at login. This means all processes have access to the contents of the wallet as long as they originate from the proper user account. This means Skype just needs to access the contents and it works.

But in KDE, the wallet is only opened when at least one process accesses the wallet. So if Skype has no features to get to Kwallet, it will never get to Libsecret.

I imagine, if you start your computer, then start a program that accesses kwallet, then start Skype, the feature you are looking for will work. For it to work properly, Skype will need to see which keyring/wallet is in use and take the appropriate measures (just get to the keyring, or open kwallet first, as needed).

I forgot to mention, ksecret service uses Libsecret. I considered them equivalent.
