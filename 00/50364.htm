kris-nz say:

This issue was created by conflicting methods for calculating time by the two operating systems.

Linux was always using UTC time to calculate local time. Win 10 was using local time specifically. Both systems change the cmos clock to their preferred time. A reboot to linux always showed the correct time because it validates the network time immediately on start up. So regardless of whatever win 10 did to cmos time, it was always corrected for use with the linux OS. However, win 10 doesn't immediately check network time on start up and continues to use the current cmos time (as set by linux to UTC). When win 10 does check the network time, which can be hours or days later, the time is corrected and cmos time updated. That updated time would be lost whenever a boot to linux occurred but because the Linux OS corrects time immediately, it gave the appearance that this was an win 10 issue specifically. And it is a win 10 issue in relation to dual-boot systems. Win 10 could avoid this by validating network time at boot.

The final fix is to check the windows time service and ensure it is set to automatic, not manual, and then sync the two systems to use the same time. They must both be either UTC or local but whatever one is chosen must be the same on both systems.

For me, it was easier to change the time on the linux system to use local time and thereby match the same time as win 10. I did this in Linux Mint by adding "UTC=no" to the file "/etc/default/rcp". Of course I could have chosen to set win 10 to use UTC time instead, but it just seemed so much easier the Linux way.

The result is that both systems now save cmos time as local time.

Let’s try with the below methods and check if he issue resolves.

Method 1: Run an inbuilt Sound troubleshooter.

  1. Press Windows + X and select the Control panel.

  2. Type troubleshooting and select it.

  3. Left pane select view all and select the Playing audio.

  4. Run the troubleshooter and follow the instructions.

  5. Restart the system and check.

If the issue doesn’t resolve then try with the next Method. (Even though the below steps are already done I request you to follow the same and check if it resolves)

Method 2: try uninstalling the sound driver and reinstalling it.

  1. Press Windows + X and select Device Manager.

  2. Select Sound, video and gaming controllers.

  3. Uninstall the High definition audio driver

  4. Restart the system and check.

Method 3: Reinstall the driver from the manufacturer website and check updating the device.

Refer to the below link to check how to install and update.

How to Install and Update drivers in Windows 10: http://answers.microsoft.com/en-us/insider/wiki/insider_wintp-insider_devices/how-to-install-and-update-drivers-in-windows-10/a97bbbd1-9973-4d66-9a5b-291300006293
