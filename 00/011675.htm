hansv mvp say:

Option buttons have a property GroupName. Option buttons with the same GroupName act as a group, i.e. only one of them can be "on".

By default, the GroupName property is blank, so all option buttons act as one group.

You can change the GroupName for each set of three option buttons to a unique name, e.g. Q1, Q2, etc. to make the sets act independently of each other.

 

To change the GroupName, right-click an option button and select Properties from the context menu.

ossiemac say:

Are you interested in a VBA code solution? Depending on the layout of your form, it may not be suitable.

If you don;t want to use it then no problem.

The code processes the Option buttons first across the document and then Down (and across and down etc. until finished)

It sets the GroupName for each set of 3 buttons and also sets the Captions in groups of Yes, No and N/A.

You DO need to create all of the option buttons first but you know how to do this with Copy/Paste.

Make a backup copy of your document in case the code does not perform as you expect.

To install the code:

Ensure that Macros are permitted to run. (See Help for how to do this. (Set to Disable all with notification.)

Ensure that Design mode is turned off after creating the Option buttons.

Alt and F11 to open the VBA editor.

Select Menu item Insert then Module.

Copy the code and paste into the VBA editor.

Click anywhere in within the pasted code and press F5 to run.

Alt and Q will close the VBA editor. (Alt F11 will open again)

If you want to save the workbook with the code then SaveAs a Macro Enabled workbook. However, if you do not want to keep the code then in the VBA Editor select Module1 in the Project Explorer (left side) and Right click and Remove. Answer No re exporting etc.

 

Sub GroupOptionButtons()
    Dim fld As Field
    Dim lngCounter As Long
    Dim lngGroup As Long
    Dim arrCaption(1 To 3)
    Dim lngArray As Long
   
    arrCaption(1) = "Yes"
    arrCaption(2) = "No"

    lngGroup = 1

    For Each fld In ActiveDocument.Fields
        If fld.Type = wdFieldOCX Then
            If TypeOf fld.OLEFormat.Object Is MSForms.OptionButton Then
                lngCounter = lngCounter + 1 'Count buttons
                lngArray = lngArray + 1     'Set array element
                fld.OLEFormat.Object.GroupName = "Grp" & lngGroup      'Applies GroupName
                fld.OLEFormat.Object.Caption = arrCaption(lngArray)    'Applies Caption
                If lngCounter Mod 3 = 0 Then    'Change group every 3 buttons
                    lngGroup = lngGroup + 1     'Next Group number
                    lngArray = 0    'Re-Start arrCaptions element number
                End If
            End If
        End If
    Next

Thank you for posting to the form regarding the issue you are having with OneNote.  I am afraid the short answer to your question is it is very unlikely you will be able to recover this content.  If there is an option to do this you would want to post to the following forum as this would only be possible via OneDrive, http://answers.microsoft.com/en-us/onedrive . 

The way the phone works is that it works with the cloud for things like OneDrive, your contact list, calendars, emails, etc...  This means that when an action is taken on the phone, like adding a new note, editing an existing note or deleting a note, that action will sync with the cloud and update your current account. 

If you do not want this to work this way and would prefer that you have a copy for the phone that is not tied to OneDrive you will want to be sure you download that note before making changes and then edit via the documents on the phone.  In a case were you are going to have the phone sent for repair or you are getting rid of the phone to upgrade, you can simply do a factory reset of the phone that will remove the account syncing to the phone.  This will ensure that your documents are not on the phone but also stay on the OneDrive cloud.
