e.j. gun say:

I don't think you'll be able to do this without some VBA in the background.  Data validation is limited to using only built-in Excel functions (as demonstrated above), and none of those functions allow you to do character by character examination of strings.

You could add some code to the WorkSheet_Change event to detect when the contents of a cell change due to user actions, and then check each of the changed cells to see if they violate your rules.  If any of the cells do, you can Undo the action and send the user a message.

Something like the code below, which goes in the Worksheet code module (right-click on Worksheet tab, select "View Code"):

Option Explicit

Private Sub Worksheet_Change(ByVal Target as Range)

    Dim myRange as Range, intRange as Range, myCell as Range

'

    Set myRange = Target.Worksheet.Range("A1:A20") ' What range do you want to restrict?

'

' See if any of the changed cells are in the range of interest...

'

    Set intRange = Intersect(Target, myRange)

    If (Not intRange Is Nothing) Then ' changed cells intersect range of interest

        For Each myCell In intRange.Cells

            If (isUpperOrNumber(myCell.Text) = False) Then

                MsgBox "These cells are limited to UPPER CASE characters and 0123456789 ONLY!", vbCritical + vbOKOnly, "Disallowed Input"

                Application.Undo

                Exit Sub

            End If

        Next myCell

    End If

End Sub

Private Function IsUpperOrNumber(theString as Variant) as Boolean

    Dim i as Long, j as Long

    IsUpperOrNumber = True

    For i = 1 To Len(theString)

        j = AscW(Mid$(theString, i, 1))

        If (Not ((j >= 65 And j <= 90) Or (j >= 48 And j <= 57))) Then

            IsUpperOrNumber = False

            Exit Function

        End If

    Next i

End Function

This library has never been available for the Mac. Active-X is open source software that was never ported to the Mac. You will never get this library because it does not exist.
