ossiemac say:

It is not possible to assign non contiguous rows to an array. It will stop at the first non-contiguous row.

 

There are 2 ways of achieving this. One is to copy the visible cells to a temporary worksheet, assign the UsedRange to the array and delete the temporary worksheet when finished. (Not advisable to re-use a temporary worksheet by clearing the contents because VBA gets confused with the UsedRange. Best to create new worksheet and then delete after assigning the UsedRange to the array.)

 

The other way is to loop through the range of visible cells and assign each to the array which if you only have a few visible cells is probably the best.

 

The following code is the first method of copying to temporary worksheet.

 

Note that AutoFilter.Range is a dynamic name for the entire AutoFiltered range and the last row of the AutoFiltered range can be extracted from it.


Sub VisibleRangeToArray()
    Dim SourceData As Worksheet
    Dim lngLastRow As Long
    Dim rngToCopy As Range
    Dim SourcePerNrs As Variant
    Dim wsTemp As Worksheet
   
    Set SourceData = Worksheets("Sheet1")       'Edit "Sheet1" to your worksheet name
   
    With SourceData.AutoFilter.Range
        lngLastRow = .Rows(.Rows.Count).Row   'Last row of AutoFilter.Range
    End With
   
    Set rngToCopy = SourceData.Range("A15:A" & lngLastRow).SpecialCells(xlCellTypeVisible)
   
    Set wsTemp = Sheets.Add(After:=Sheets(Sheets.Count))
   
    rngToCopy.Copy Destination:=wsTemp.Cells(1, 1)
   
    SourcePerNrs = wsTemp.UsedRange     'Assign to array
   
    Application.DisplayAlerts = False
    wsTemp.Delete
    Application.DisplayAlerts = True
   
    'Test array contents
    'Note when a range is assigned to an array,
    'the array must be addressed by both dimensions even though
    'there is only one dimension for a single column
    Dim i As Long
    For i = LBound(SourcePerNrs, 1) To UBound(SourcePerNrs, 1)
        MsgBox SourcePerNrs(i, 1)
    Next i
   
End Sub

 

 

the following to assign values with a loop

 

Note can use for each loop but cannot not use for i = 1 to whatever for the non-contiguous rows because again it will stop after the first non-contiguous row.

 

Sub VisibleRangeToArray_2()
    Dim SourceData As Worksheet
    Dim rngCel As Range
    Dim SourcePerNrs()
    Dim lngLastRow As Long
    Dim rngToAssign As Range
    Dim k As Long
   
   
    Set SourceData = Worksheets("Sheet1")       'Edit "Sheet1" to your worksheet name
   
    With SourceData.AutoFilter.Range
        lngLastRow = .Rows(.Rows.Count).Row   'Last row of AutoFilter.Range
    End With
   
    Set rngToAssign = SourceData.Range("A15:A" & lngLastRow).SpecialCells(xlCellTypeVisible)
   
    k = 0
    For Each rngCel In rngToAssign
        k = k + 1
        ReDim Preserve SourcePerNrs(1 To k)     'Dimensions One based array
        SourcePerNrs(k) = rngCel.Value
    Next rngCel
   
   
    'Test array contents
    Dim i As Long
    For i = LBound(SourcePerNrs) To UBound(SourcePerNrs)
        MsgBox SourcePerNrs(i)
    Next i
   
End Sub

a. user say:

Hi Douglas Levy,

 

It's not feasible to show an external connected account separately in OWA (Outlook on the web). The emails sent to your external account will go to same inbox as your primary account. Given the situation, you can create an inbox rule to route the incoming emails from your GoDaddy account to a subfolder in OWA. You can create a rule like when the emails are sent to a specific recipient (your GoDaddy account), then route them to a subfolder like "GoDaddy inbox", then stop processing more rules.

 

About you want to send on behalf of the GoDaddy address question, please check if the following method can solve it: Click New to create a new email>click … on the top bar then choose Show From, then you can see From box displayed> right click the address showing in From>Remove>then enter your GoDaddy email address in the box and check if the email can be successfully sent out.

 

If the issue persists and this is not the scenario, please double check if you have successfully added your account to OWA, are there any error messages during the process?
