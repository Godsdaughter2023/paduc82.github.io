I have a database that we use to track students receiving meals at the hostel. They are allowed only one lunch meal per day. I would like to achieve the following:

2. I have a second table that stores the scan (RFID and Date)

3. I would like to when the RFID scan, it compares the RFID number against the table (Student Info) to validate that the student is a hostel student and if the number is not present int he Studentinfo table it will not allow the scan.

4. The second step validation should be that the student cannot scan a second time on the same date. This should then be done with a message expelaining why the scan is not accepted.

daniel pineault say:

You can't stop the scan as the scan had already taken place to be able to get the RFID to validate within your database.

You should be able to use a simple DCount to validate of the RFID exists in your database already and even validate of it was used within a certain timeframe.  Then you can act accordingly, pop-up message, ...

ken sheridan say: If the StudentInfo table includes only authorised students at the hostel, then you merely need to create a relationship between the tables and enforce referential integrity.  It will then not be possible for a row to be inserted into the RFIDScans table where there is no match in the StudentInfo  table.  The error can then be trapped, and appropriate action taken.

To prevent an authorised student getting more than their one meal allowance per day you can apply a CHECK CONSTRAINT to the RFIDScans table as follows:

CurrentProject.Connection.Execute "ALTER TABLE RFIDScans ADD CONSTRAINT DailyScan CHECK((SELECT RFID FROM RFIDScans GROUP BY RFID, INT(ScanDateTime) HAVING COUNT(*) > 1)IS NULL)"

Enter the above as a single line into the 'immediate' window (press Ctrl+G to open it) to apply the constraint.  Again you can trap the error and take appropriate action.

If you need to remove the constraint enter the following in the 'immediate' window:

CurrentProject.Connection.Execute "ALTER TABLE RFIDScans DROP CONSTRAINT DailyScan"

You will need to change the table and column names in the above to your actual names of course.  If you have used spaces or other special characters in the object names be sure to enclose them in square brackets [like this], but it's best not to do so in the first place; use CamelCase or represent a space by an underscore character instead.  And do not used Date as a column name; as the name of a built in function it's a 'reserved' keyword, and should not be used as an object name.

Sub DiagonalToRows()
'ensure used range in source sheet is a square range which will be copied to the destination sheet

Application.ScreenUpdating = False

Dim lLastCol As Long, lLastRow As Long, lFirstCol As Long, lFirstRow As Long
Dim lTotalRows As Long, lTotalColumns As Long, count As Long
Dim wsSource As Worksheet, wsDest As Worksheet
Set wsSource = Sheets("Sheet1")
Set wsDest = Sheets("Sheet2")

wsDest.UsedRange.Clear

lFirstRow = wsSource.UsedRange.Cells(1, 1).Row
lFirstCol = wsSource.UsedRange.Cells(1, 1).Column
lLastRow = wsSource.UsedRange.Cells.Rows.count + lFirstRow - 1
lLastCol = wsSource.UsedRange.Cells.Columns.count + lFirstCol - 1
lTotalRows = wsSource.UsedRange.Cells.Rows.count
lTotalColumns = wsSource.UsedRange.Cells.Columns.count

If lTotalRows <> lTotalColumns Then
    MsgBox "Range should be a square ... exiting sub!"
    Exit Sub
End If

For i = 1 To lTotalRows
    For lCol = 1 To lTotalRows
        If i Mod 2 = 1 Then
            wsSource.Cells(lLastRow, lLastCol).Offset(((i - 1) - (i - lCol)) * -1, (i - lCol) * -1).Copy
            With wsDest.Cells(i, lCol)
                .PasteSpecial xlValues
                .PasteSpecial xlFormats
            End With
        ElseIf i Mod 2 = 0 Then
            wsSource.Cells(lLastRow, lLastCol).Offset((i - lCol) * -1, ((i - 1) - (i - lCol)) * -1).Copy
            With wsDest.Cells(i, lCol)
                .PasteSpecial xlValues
                .PasteSpecial xlFormats
            End With
        End If
    Next lCol
Next i

count = -1
For i = lTotalRows + 1 To (lTotalRows * 2) - 1
    count = count + 1
    t = i - 1
    For lCol = 1 To (lTotalRows * 2) - i
        If i Mod 2 = 1 Then
            wsSource.Cells(lLastRow, lLastCol).Offset((t - (t - lCol - count)) * -1, (t - lCol - count) * -1).Copy
            With wsDest.Cells(i, lCol)
                .PasteSpecial xlValues
                .PasteSpecial xlFormats
            End With
        ElseIf i Mod 2 = 0 Then
            wsSource.Cells(lLastRow, lLastCol).Offset((t - lCol - count) * -1, (t - (t - lCol - count)) * -1).Copy
            With wsDest.Cells(i, lCol)
                .PasteSpecial xlValues
                .PasteSpecial xlFormats
            End With

        End If
    Next lCol
Next i

Application.CutCopyMode = False
Application.ScreenUpdating = True

End Sub

Sheet1 is the source sheet from which the data is copied to Sheet2. The used range in Sheet1 (numbers or formats) must be a square. Even if numbers are not entered, the formats will be copied. Presently the range to be copied to in Sheet2 starts from first row & first column ie cell A1.
