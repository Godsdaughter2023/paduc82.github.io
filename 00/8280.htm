a. user say:

You're probably going to have to modify the field codes that Word uses to insert the page numbers and page counts.

For "Page n of m" Word uses field codes something like this:

"Page { PAGE } of { NUMPAGES }

You should be able to see the codes using option-F9 (but it depends on whether the keyboard shortcut has been appropriated by something else in Mac OSX).

What you can do is
 a. select { NUMPAGES }
 b. use cmd-F9 to insert another pair of the special "field code braces" around that field, so you have
{ { NUMPAGES } }
 c. type an "=" sign before the inner field, and " - 1" after it, so you have
{ ={ NUMPAGES } - 1 }
 d. Select that lot, and press F9 to update the field value
 e. use option-F9 to redisplay the results.

There is also a { SECTIONPAGES } field that you can sometimes use for this purpose instead of { NUMPAGES }, i.e. if the number you want to display happens to be the number of pages in a particular section. But it's not usually a good idea because Word sections are used for things other than "document structure" (e.g., just to switch from portrait to landscape layout you have to introduce a section break)

    Sometimes, for various reasons (including sunspots and hemline heights) the official technique to change defaults does not work.  In that case, open the NORMAL.DOTM template for editing, change the margins but don't bother with using the "Default" button.  Close the document (NORMAL.DOTM) and save the changes.  The new margins should now apply to all new documents.

         Application.NormalTemplate.OpenAsDocument

    and press Enter. Normal opens, but note that it may not be on top of all windows.
