rseiler say:

UAC (set not to dim) sometimes puts its dialog behind Explorer

Back in the release notes for 14352, MS said:

"We fixed an issue that resulted in the User Account Control dialog sometimes appearing behind other windows instead of in the forefront."

But this is not true here. In fact, I'm not sure that I ever saw it happen before then--if I did, it was rare. Ever since then though (through all released builds), it's been a far more prevalent problem.

I've also tested this with clean installs, so it's not an installation issue, and others have reported it in various places, too.

It does not happen with UAC on its default (dim).

To reproduce:

1) Set UAC to not dim the screen (this is the third notch down in UAC settings).

2) Try to run UAC-dependent apps via Explorer, like regedit.exe, Sysinternals tools, or whatever else.

It won't happen every time, but it should happen within the first minute of trying various programs. Also, there's no consistency or pattern to it that I've seen, and the same apps run different ways (e.g. File/Run) will never show the problem.

ken sheridan say: This is a fundamentally a modelling issue.  When referencing a tree-structured hierarchy only the lowest level in the hierarchy should be referenced.  For example, if you have tables Countries, States and Cities then a referencing table, e.g. Locations, would include only a CityID foreign key column, not StateID or CountryID columns.  In the States table, in addition to real States, there should be a rows with a value N/A or similar, one for every Country.  Similarly in the Cities table, in addition to real Cities, there should be a rows with a value N/A or similar, one for every State.  The primary keys must be 'surrogate' numeric keys, usually an autonumber, CountryID, StateID and CityID of course, not the names of the countries, sates or cities.

Consequently, in the Locations table to record a location only by Country and State you would select the N/A value for the State in question.  To record a location only by Country you would select the N/A value for the N/A sate of the country in question.

ken sheridan say:

I tried to explain that in my first reply, but apparently was not clear enough, so I'll try and do so with a hypothetical example.  The important principle to grasp is that the hierarchy is always continuous, so every location references a row in Cities, which in turn references a row in States, which in turn references a row in Countries.  To maintain the continuity of the hierarchy in cases like that which you cite, there is a N/A state for every country and a N/A city for every state, including the N/A states.

So let's assume there is a location Much Binding in the Marsh which you wish to record as being in the United Kingdom, but not in any specific City or State.  First there would be rows in Countries like this:

CountryID    Country
1                 N/A
2                 France
3                 Spain
4                 United Kingdom
etc

and in States, there would be rows like this:

StateID        State            CountryID
41               N/A              1
42               N/A              2
43              Ardennes       2
44              Ariège           2
123            N/A               4
124            Lancashire     4
125            Warwickshire  4
etc

In Cities there would be rows like this:

CityID        City                StateID
510           N/A               42
511           N/A               43
512           Sedan            43
512           N/A               44
513           Foix               44
514           N/A               123
515           N/A               124
516           Lancaster       124
517           Bury              124
518           N/A               125
519           Warwick         125
520           Leamington    125
etc

So the row in Locations (Elements in your case) would be like this:

LocationID    Location                              CityID
245              Much Binding in the Marsh    514

This maps to the N/A row in Cities with CityID 514, which maps to the N/A row in States with CountryID 4, which maps to the United Kingdom row in Countries.
