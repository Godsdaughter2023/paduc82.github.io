You can turn off hibernation if it is on and you don't use it..

When you hibernate your computer, Windows saves the contents of the system's memory to the hiberfil.sys file. As a result, the size of the hiberfil.sys file will always equal the amount of physical memory in your system. If you don't use the hibernate feature and want to recapture the space that Windows uses for the hiberfil.sys file, perform the following steps:

- Start the Control Panel Power Options applet (go to Start, Settings, Control Panel, and click Power Options).
- Select the Hibernate tab, clear the "Enable hibernation" check box, then click OK; although you might think otherwise, selecting Never under the "System hibernates" option on the Power Schemes tab doesn't delete the hiberfil.sys file.
- Windows will remove the "System hibernates" option from the Power Schemes tab and delete the hiberfil.sys file.

steve cochran say:

Its quite likely that the files got filled with zeroes where the messages were.  This happens a lot when users let their files get above a few hundred megabytes.  Your 1.52 gigabyte file was a disaster waiting to happen.

You can check the Recycle bin and message store for bak files that are backups of the dbx files that you can restore, if present.  See www.oehelp.com/OETips.aspx#2 and www.oehelp.com/backup.aspx#imp1 .  If there are no bak files then your only option is to try and extract the messages from the files (which may not contain anything anymore except zeroes -- you can try zipping them and if they zip down to nothing, then that's what's in the file) or if they are no longer in the files, then DBXpress has an exclusive extract from disk feature that might recover them (www.oehelp.com/DBXpress/) off the hard drive directly.

In the future it's wise to keep the size of the dbx files below a few hundred megabytes and backup frequently as this happens to OE users on a daily basis.
