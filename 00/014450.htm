Excel 2013 Crashes when you scroll - OSF.DLL (Office / Visio DLL File) - Stop “annimations” & uninstall Avast AV

http://blogs.technet.com/b/the_microsoft_excel_support_team_blog/archive/2013/11/04/excel-2013-crashes-when-you-attempt-to-scroll-down-the-page.aspx

4 Nov 2013 10:39 AM

We have seen several reports of Excel crashing when you scroll down, even on a brand-new sheet. The Application Event log points to a crash in OSF.dll.

We have found that at least one cause of this particular crash is a conflict with Avast! Antivirus. Essentially, if you install Avast! and then open any workbook, Excel will crash if you scroll down using the scroll bar or the mouse wheel. This issue only occurs if you are logged in using a Microsoft Account (previously known as Live ID).

Unfortunately, the crash continues to happen even if you uninstall Avast! and run an online repair. We have been able to reduce the instance of issues by Uninstalling Avast! Free Antivirus, rebooting, and then turning off animations in Windows. To do that:

1.   In Control panel, go to the Ease of Access Center

2.   Click on Make the computer easier to see

3.   Scroll down to Turn off all unnecessary animations (when possible)

4.   Check that, and Apply.

Note: Just turning off animations or removing Avast! Antivirus is not enough to make a difference. You need to do both.

Alternatively, if you have the restore point created before you installed Avast! Antivirus, you can restore your system to that point.

We contacted Avast! product support, and have received word that they are working on a fix. They should release a patch soon.

davem121 say:
Hi miabullard, the general form of a VLOOKUP function is:

=VLOOKUP(value, range containing the lookup value, the column number in the range containing the return value, optionally specify TRUE for approximate match or FALSE for an exact match)

So your formula should looks something like this:

=VLOOKUP( Cell, A15:C18, XX, TRUE)

Where XX should be:
1 if Interest rate is in column A
2 if Interest rate is in column B
3 if Interest rate is in column C

Cell should be:
The Borrowers Name - "John Doe"
or a Cell Reference - A16

john korchok say:

You can apply the hyperlink character style to a selection in a multilevel list and it won't change the list style. This makes it looks like a hyperlink without actually being one.

By default, the Hyperlink style doesn't show in the Style Gallery or Syles list, but here's how you make it accessible:

  1. With you document open in Word, press Alt + Ctrl + Shift + S to open the Styles list.
  2. Click on the Manage Styles button at the bottom of the dialog.
  3. Click on the Recommend tab.
  4. Scroll 2/3s the way down until you see last  Hyperlink (Hide Until Used). Click on it.
  5. Click on the Show button. OK out. The style is added to the Styles list.
  6. If you would like to see it in the Style Gallery, right-click on it in the Style list and choose Add to Quick Style Gallery. OK out.

julie sheets say:
You should save a baseline for the project - or phases in the project - before you start working on the tasks.  When you save a baseline, Project captures the start, finish, work, duration, and cost for each task.  You can then compare the schedule to actuals.  Project also automatically calculates variance (difference between baseline and actuals) for you.

I hope this helps.
Julie

julie sheets say:
No need to calculate - the start variance, finish variance, work variance, cost variance, and duration variance fields are all ready calculated for you.

If you want to see how late John is on a single assignment, I'd add the Start Variance and/or Finish Variance fields to the Task Usage view or Resource Usage view.

You may also add Baseline Work to the timescaled (right side) of either of those view to look at when work was scheduled (baseline) and when it is occuring.

julie sheets say:
Hi Kevin,

I guess I understand a bit better about what you want.  You are correct, Baseline Start and Baseline Finish do not apply to resources - rather assignments - but again, if John is late - he's late working on an assignment.

There is a cumulative Work field you may add to the timescaled portion of the Resource Usage view - but it does not show you how many hours were planned to date versus accomplished to date as once you start recording Actuals, it is the Actual work used for Cumulative - not baseline work.  There is not a cumulative baseline work field - which I think is what you are seeking.
