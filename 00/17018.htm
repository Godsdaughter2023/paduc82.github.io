ken sheridan say: Normally one would expect the combo box to be set up as you want if you drag the field to the form from the field list.  However, failing that, there is a very easy way to do it if you are not familiar with setting up the properties of such combo boxes manually.  First select the table in the navigation pane, and then click on Form in the Create ribbon.  This will create a very simple form bound to the table, and will include a combo box bound to the relevant column, set up in the way you want.

In form design view copy the combo box to the clipboard and paste it into your operational form, also in form design view.  In the operational form you can then position the combo box where desired, and amend its properties to whatever size, colours etc you want.

You don't have to save the simple form which Access creates.  Once you've finished with it you can close it without saving.

Having said that, think carefully about whether a 'multi-valued field' is the best way to represent a many-to-many relationship type like this, rather than using the conventional method of modelling the relationship type by a table which resolves it into two one-to-many relationship types.  You'll find an example of a simple many-to-many relationship type modelled like this as StudentCourses.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file various possible interfaces are illustrated, though a conventional form/subform is usually to be recommended, and is very easy to implement.  A major disadvantage of a 'multi-valued field' is that the relationship type which it models cannot have any other attributes, such as the Status attribute in my demo.  In a conventional model this simply requires another bound control in the subform.

OneNote files are not automatically encrypted on OneDrive by default, there is no at rest encryption on OneDrive

swischner say:
No, they aren't. While there is a transport encryption (meaning they cannot be read by someone "listening" to the data transfer from and to Microsofts servers), notes are stored unencrypted on the servers unless you password protect the section(s). In fact the servers need to be able to read the notes to build a search index. On encrypted sections search does not work. So either password protect sections with critical data in them or don't store such data in OneNote.

karl timmermans (outlook mvp 2012-2018) say:
>>
I cannot recover my contacts using the Outlook repair tool. Where are the contacts kept, in the pst file?
<<

Hello JoleneZ

Answer to your question depends on a couple of different things

#! - The type (POP3, Imap, Exchange) of email account that the contacts belonged to
(if not sure, from Outlook menu, go to File --> Account Settings --> Email tab --> right-hand column shows the type of email account i.e. POP3/SMTP / Imap/SMTP / Exchange)
OR
#2 - The default data file defined for the Outlook profile
(to confirm which data file is configured as the default, go to File --> Account Settings --> Data FIles tab)

<IF> this involves,a POP3 account AND the PST file for the POP3 account is configured as the default, the contacts would be located in the PST file

<IF> this involves an IMAP account AND the default data file for the IMAP account is set as the default, then the contacts would be stored in the OST file associated with the IMAP file and located in a contact folder that would have the words "This Computer Only" in the folder description

<IF> this involves an Exchange account, the contacts are stored on the server and sync'd to the contact folder in the OST file associated with the EX account.

Whatever the account type, the default location for contact info is in the contact folder in the data file set as the default data file for the profile.

<IF> you have an IMAP account configured AND the OST file is the default for the profile AND the contact folder is suddenly empty, the cause for that would be due to Outlook being unable to open the OST file for the account at which point, the email folders would be sync'd with the server but the folders labeled as "This Computer Only" folders would be empty (previous data in those folders would be lost).
