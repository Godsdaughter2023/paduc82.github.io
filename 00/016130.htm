This time, despite that the Modal property is set to "No", the form still does not appear! I have to set the "Pop-up" property to "No" also to view my form (which gets "tabbed", what I don't want).

 

Any idea on this wierd behavior?

 


Conceivably the form's window position has been moved off the screen somehow.  Do you use a multiple-monitor setup?  Switching between a multiple-monitor desktop and a single-monitor desktop has been known to get Access confused.


After opening one of these forms, try entering a statement like this in the Immediate Window:


    Forms("YourFormName").Move 0, 0


(Replace "YourFormName" with the name of your form, enclosed in quotes as shown.)  If the form has just been moved offscreen, that statement ought to move it to the top left corner of the Access document window.  Then you can move it where you want and save it by pressing Ctrl-S. 

 

Or, if the form opens in design view, you can set its Auto Center property to Yes, and that ought to make it open in the center of the screen.

 

Another possibility, though less common, is that the form's height and width have been set to 0.  In that case, you could use an alternative form of the Move method to size it to visible dimensions.

Mass Email from Excel

I have 50 worksheets in one workbook.  Each worksheet must be sent to a different provider based on the email in A1.  The subject in A2 and the Body of the email in A3.  The remaining information that is individualized is in cell $A$3:$E$51.   What is the best course of action to accomplish that task?  I know how to write macros for emailing through outlook one at a time, but not sure how to do 50 different worksheets to 50 different people.  The excel file can be in pdf format.

Any assistance is greatly appreciated.

[Moved from: Office / Excel / Windows 10 / Office 2016]

james.zh say:

Hi Rose,

You said you know how to send one email at a time from Excel, so I guess all you need is some coding which can loop through the 50 sheets?

Hope the coding below (or something like that) will solve your problem. I exported the information in A3:E51 to a pdf file first before attaching it as an attachment to the email. Let me know if it doesn't work.

Sub MassEmails()

Dim olApp As Object
Dim olMail As Object
Dim x As Integer, y As Integer

On Error Resume Next

Set olApp = GetObject(, "Outlook.Application")
If Err.Number <> 0 Then
    Set olApp = CreateObject("Outlook.Application")
End If

y = ActiveWorkbook.Worksheets.Count

For x = 1 To y

    Sheets(x).Activate

ActiveSheet.Range("$A$3:$E$51").ExportAsFixedFormat Type:=xlTypePDF, _

        FileName:="C:\Desktop\" & x & ".pdf" 

    Set olMail = olApp.CreateItem(olMailItem)

    With olMail

        .Subject = ActiveSheet.Range("A2").Value
        .To = ActiveSheet.Range("A1").Value
        .Body = ActiveSheet.Range("A3").Value
        .Attachement.Add "C:\Desktop\" & x & ".pdf"
        .Display

    End With

    Set olMail = Nothing

Next x

End Sub
