Hi,

you can add an extra shape (like an arrow) between 2 points
but not fit exactly

scanario

add a shape between 5 and 6 point

before

after

vba macro

Sub Chart_Add_Extra_Shape()
On Error Resume Next
Const sArrow$ = "myArrow" '<< arrow name
Dim ws As Worksheet
Set ws = ActiveSheet '<< chart in active sheet
Dim obj As ChartObject
Set obj = ws.ChartObjects("Chart 1") 'chart name
Dim c As Chart
Set c = obj.Chart
Dim sLine As Shape
ws.Shapes(sArrow).Delete ' << delete old shape arrow/if exists
Dim x1 As Double, x2 As Double, y1 As Double, y2 As Double
Dim p1 As Integer, p2 As Integer, nSer As Integer
nSer = 1 ' << series 1
p1 = 5: p2 = 6 ' << add an arrow from point p1 to point p2
x1 = c.SeriesCollection(nSer).Points(p1).Left + obj.Left
y1 = c.SeriesCollection(nSer).Points(p1).Top + obj.Top
x2 = c.SeriesCollection(nSer).Points(p2).Left + obj.Left
y2 = c.SeriesCollection(nSer).Points(p2).Top + obj.Top
Set sLine = ws.Shapes.AddConnector(msoConnectorStraight, x1, y1, x2, y2)
sLine.Name = sArrow
With sLine
.Visible = msoTrue
.Line.EndArrowheadStyle = msoArrowheadOpen
.Line.ForeColor.RGB = vbRed
.Line.Weight = 3
End With
End Sub

    a. user say:

    Your result is consistent with the following situation:
      1. The mechanism that Word uses to get data from Excel (an "OLE DB provider") is determining that your amount column is not numeric or "currency" but "text". That could happen even if almost all the values in your amount column are correctly formatted as "currency", but just one of the values in the first 8 rows has been typed in as a text value. There may be other ways it could happen.
     2. the amounts are coming across as text values with  "US" formatting, i.e. as 242,129.05. That could happen if for example you saved/closed the .xlsx while the US regional settings were switched on (e.g. Excel is not even open) or if you are editing the sheet with US settings on, then change the regional settings, then open the word document.
     3. as a consequence, the amount is coming across like this

    242,129.05

    in which case with the German regional settings, the text will be treated as two numbers, 242,129 (i.e. just over 242 euros) and 5. These numbers wil be added together giving you 247,129, then rounded up to 247,13

    The key thing is either to try to ensure that all the amounts are amounts, in which case I think they wil come across formatted correctly as "German" amounts as long as your regional settings are the German ones, or that they are all formatted as texts with the "German" formatting.

    It is difficult to advise precisely how to ensure either of those things is true, because it depends partly on exactly what you have in your Excel worksheet, but for example if what you see in Excel is what you need in word, it may be easier to copy/paste your worksheet data into a new Word document and use that as the data source.
