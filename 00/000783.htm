Subtotal function does not sum correctly in Office 2010 - Microsoft Community

hanief edwards say:

Subtotal function does not sum correctly in Office 2010

The FunctionNumber 9 and 109 for the SubTotal function work the same in Office 2010, when applied to Tables.  Using 109, filtered rows should be excluded from the result, while using 9 should include all rows. Office 2007 handled this correctly.
ie
=Subtotal(109,[Amount])
and
=Subtotal(9,[Amount])
produce the same result for a column named Amount, whether or not rows are filtered
ashish mathur say:

Hi,

 

A SUBTOTAL() function with function argument of 9 will exclude only invisible cells of a filtered range.  A SUBTOTAL() function with function argument of 109 will exclude invisible cells of a filtered range as well as manually hidden rows.

 

When the range is unfiltered, 9 and 109 will show the same result.

Hi Kate,

   The desktop version of Excel being more single user focused remembers the last tab that was active when the document was saved. In a more collaborative world Excel Online does not. However, at its core it is still the same Excel so you can take advantage of hide/unhide sheet to change what the underlying "active" sheet is.

  • Select the sheet you would rather have open up on start up,
  • right click on that sheet tab and choose Hide,
  • then right click again on a different sheet and choose "Unhide..."
  • unhide the sheet you want to be the "active" sheet.
  • if you exit back to onedrive now the autosave will preserve the sheet as the "active" sheet.

Currently un-hiding a sheet has the side effect of making it the "last active" sheet.

Hope that helps,

The correct term (jargon) is Building Blocks. Microsoft conflated a bunch of features under the term "Quick Parts."

You need to be sharing the Document Template or Global Template not just a file based on the template. Unless a document is attached to the template, component parts like Building Blocks or macros will not be available in the document.

jllatham say:

Add the code below to your project and then call it like this

  strPath = BrowseFolder("Select Folder")
  If strPath = vbNullString Then
    'user cancelled the operation
    Exit Sub
  End If
  'the return will not have final \ yet, so...
  If Right(strPath, 1) <> Application.PathSeparator Then
    strPath = strPath & Application.PathSeparator
  End If
  'continue on here with the path selected
  MsgBox strPath ' just to show what was selected

Here is the routine, with notation of the source that you can visit for other ways to do the same thing:

Function BrowseFolder(Title As String, _
  Optional InitialFolder As String = vbNullString, _
  Optional InitialView As Office.MsoFileDialogView = _
    msoFileDialogViewList) As String
'source: http://www.cpearson.com/excel/browsefolder.aspx
  Dim V As Variant
  Dim InitFolder As String
  With Application.FileDialog(msoFileDialogFolderPicker)
    .Title = Title
    .InitialView = InitialView
    If Len(InitialFolder) > 0 Then
      If Dir(InitialFolder, vbDirectory) <> vbNullString Then
        InitFolder = InitialFolder
        If Right(InitFolder, 1) <> "\" Then
          InitFolder = InitFolder & "\"
        End If
        .InitialFileName = InitFolder
      End If
    End If
    .Show
    On Error Resume Next
    Err.Clear
    V = .SelectedItems(1)
    If Err.Number <> 0 Then
        V = vbNullString
    End If
  End With
  BrowseFolder = CStr(V)
End Function

The question of returning a blank or null value has come up several times in this forum and I have found the answer to my immediate need.  However, I was curious about the differences between various methods/formulae.  There were other suggestions (such as conditional formatting to change the text colour to the same as the cell background, which I rejected)
I am using a simple total or SUM as an example.

1)  =SUM(C14:AG14)

Entire sheet set to return a blank if content (or formula result/product) is zero, in Options.


2) =IF(SUM(C31:AG31),SUM(C31:AG31),"")

This is the automatic translation in a recent conversion from a Lotus 1-2-3 spreadsheet (c. 2000).  The original is a template with some fairly fancy formulae which I did not enter myself or understand, but which do the job admirably.


3) =IF(COUNT(C48:AG48)=0,"",SUM(C48:AF48))

A solution given in these forums, which might be similar to (2).  Without knowing a  lot about how Excel formulae are structured, I think I can see the logic of this one (3).


In the thread covering (3) there was a mention of different effects when, e.g. calculating things like AVERAGEA, i.e. if the blank cell is counted or not.

Comments on the differences welcome, specially between (2) and (3).
