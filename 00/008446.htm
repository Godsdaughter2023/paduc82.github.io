A no-width non-break, which is also called a zero width joiner (ZWJ), is a nonprinting character that prevents a word, number, or phrase from breaking if it falls at the end of a line.... SELECT AS_PT.ID, AS_PT.[Active ], AS_PT.[Product ], AS_PT.Precursor FROM AS_PT ORDER BY AS_PT.[ID];

I want that the values from the other fields to be displayed to help selection of the tables.

However I only get the values from the ID from these other fields.

Can someone help?

scottgem say:

This is one of the main reasons why we do NOT recommend using Lookup fields on the table level. Because it masks what is actually stored in the field. Lookups should be done on the form level using list controls.

It is correct for you to store the ID values as your foreign keys. Typically you would use a query to display the text values. For example the typical properties for combo would look like this:

RowSource: SELECT ProductID, ProductName FROM table ORDER by Product;

BoundColumn: 1

Column count: 2

Column Widths: 0";1"

This would display the Product name for the user to choose from, but store the ID. If you needed to display the Product Name, then you link your query to the Product table and bring in the Product name column.

reg add HKCU\Software\Microsoft\Office\12.0\Word\Options /v NoReReg /t REG_DWORD /d 1
