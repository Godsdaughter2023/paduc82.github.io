ken sheridan say: If you have have used the preferred method of opening each instance of the form as a member of a collection, then to close a single instance of the form you should first move focus to the form, and then call the Close method of the DoCmd object.

You'll find an example of how to open multiple instances of a form as MultiForms.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file opens each instance of the form as a member of a colPopups collection, so to close a specific instance the following procedure could be added to database:

Public Sub CloseForm(instance As Long)

    'set focus to form
    colPopups(instance).SetFocus
    ' close form
    DoCmd.Close
    
End Sub

To close the 4th instance of the form for instance the procedure would be called with:

    CloseForm 4

Bear in mind, however, that a closed form is removed from the collection by the following code in its Close event procedure:

 Private Sub Form_Close()

    Dim n As Integer
    Dim CurrentInstance As Integer
    
    For n = 1 To colPopups.Count
        If colPopups(n).CompanyID = Me.CompanyID Then
            CurrentInstance = n
            Exit For
        End If
    Next n
    
    colPopups.Remove n
    
End Sub

In this, the current instance is identified by its primary key, CompanyID, and then removed from the collection, so the index for each member of the collection above that removed is reduced by 1.  So if the function is called again with:

    CloseForm 4

This would close the instance which previously had an index of 5, so when calling the function the instance has to be specified as that in the current collection, not the original one.

If Word is working properly when you start WINDOWS in Safe Mode, that indicates that some background program that is started at boot time is having a bad interaction with Word 2013.  Back with Office 365/2013 was first released there were several Word Addins that were causing various random problems too. For the most part they have been fixed.

To fix your problem, you will have to investigate the programs, including services, that are automatically being started with Windows in normal mode.

The first place to look is in MSConfig or Task Manager.  Since you have to boot windows to test this, a "binary" search is best. Turn off half of the programs.  Then test word.  If the problem goes away, it is in the half that are turned off.  If not, it is PROBABLY in the other half, so turn off the ones on and turn on the ones you turned off. Boot again and test again.  If the problem still persists the it is being caused by something else, like a service.  If the problem is gone, then turn half of the applications back on and restart.  Keep this up until you identify the specific program.  They you can decide if you need it to do your job or not. If you need it, contact the maker to see report the problem and see if they have a fix for it.

If the problem is not in one of the autostart programs, it may be starting as a "Windows Service". Contact your technical support people for help with this.

jay freedman say:
Your terminology is a bit confusing, but it sounds like you aren't seeing something that should be there. On the Equation Tools ribbon, click the Bracket button. The gallery should have three sections, labeled Brackets, Brackets with Separators, and Single Brackets. The first section should contain three rows of four items, each of which consists of an opening and closing pair of "brackets" (including braces, brackets, parentheses, angle brackets, and other types) surrounding a dotted box.

You should start by inserting the desired pair of brackets. Click in the dotted box; then you can type any expression, including using other buttons on the ribbon for things such as built-up fractions or large operators. As the expression grows taller, the brackets will grow to match.

The other way to do this is to create the expression first. Then select the entire expression, go to the Brackets button on the ribbon, and choose a pair of brackets; they'll be inserted at the correct size (and will continue to grow or shrink if you further edit the expression).

Win 8 Office - Microsoft Community

leo dodge say:

Win 8 Office

I already own Office Prof. 2007.  Just installed Office 2010 Home & Budget on my new HP Win 8.1 machine.

Is there any way to install my Publisher from Office Prof. 2007 onto my new computer?

Leo Dodge

 Ellicott City, MD

davidf2 say:

Yes if it is a retail version, you have the installation media and the keycode and you have retired the old computer, you can install on two computers.


When you start the installation you will get the option to use a default or a custom installation. Always choose a custom. You will also get the option to overwrite other Office installations. Be sure to choose the option to keep other installations.


Once you choose the custom installation you will have the option to install into a default directory or folder. I would suggest you create a custom folder and install there. For example the default is likely to be the Program Files\Microsoft Office folder. Change that to Program Files\Publisher2007 as an example. It will make it easier to manage and help insure that you do not overwrite anything in the other installation.


At some point you will also get the option of what parts of Office to install. You will want to uncheck everything except Publisher. See this reference and scroll down about half way and you will see the options under the Installations Options tab. http://www.howtogeek.com/howto/17342/upgrade-office-2003-to-2010-on-xp-or-run-them-side-by-side/#comment-107651


Anything you do not want to install you choose the Not Available option. Technically you can run multiple versions of Office (I run 5 versions of Publisher) but it is best to not install multiple versions of Word or Excel and you cannot run multiple versions of Outlook. But to keep things clean you probably should just install Publisher unless you have a need for Access, etc.


Here are a couple other references for more details:

http://support.microsoft.com/kb/2121447/en-us

http://pubs.logicalexpressions.com/pub0009/LPMArticle.asp?ID=762

All the referenced articles may discuss different versions but the principles are the same.
