Hi,

Follow the Methods mentioned below.

 

a)      Click on Start. Type cmd in the Start Search box, and then press ENTER. 

b)      At the command prompt, type the following command, and then press ENTER:

c)      ipconfig /flushdns (there is a space between ipconfig and / )

d)      It will show the message successfully flushed the DNS resolver cache.

e)      At the command prompt, type the following command, and then press ENTER:

f)       ipconfig /registerdns (there is a space between ipconfig and / )

g)      At the command prompt, type the following command, and then press ENTER:

h)      ipconfig /all (there is a space between ipconfig and /)

i)        The ipconfig /all command displays Windows TCP/IP settings for all your network adapters.

j)        If your IP address for your Local Area Connection is either 0.0.0.0 or 169.x.x.x (where x is any number), then your computer is not getting an IP address from the router. 

k)      If that is the case, try this:

•                ipconfig /release in the command prompt and press ENTER

•                ipconfig /renew in the command prompt and press ENTER

l)        Test it again by typing ipconfig /all to see what the address is?  (If you are on a home router, it should start with either 192.168.1.x or 192.168.0.x or 10.x.x.x)

I can think of several reasons why this would happen:
  • The font specified in the template is missing from your system and is being visually replaced by another. Check this in Word > Preferences > Compatibility > Font Substitution.
  • The text has scaling or condensed attributes. Hightlight the problem text, then select Format > Font > Advanced and set the scale to 100% and Spacing to "Normal."

ronrosenfeld say: Most likely, the trailing space is a nbsp which is ascii character 160.

You can use substitute(a1,char(160),"")
You could use find/replace, but in the find box, hold down the <alt> key while entering 0160 on the numeric keypad.
