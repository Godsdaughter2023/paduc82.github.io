john mcghie say:
No, you can't do that with Fields, they do not have the needed LEN, INSTR, LEFFT$, MID$ and RIGHT$ functions you need.

Use VBA:  There it is quite easy:

For Each aField In Snapshot.Fields
   If aField.Type = wdFieldIncludeText Then ' Print document name so if it won't open we know which one
      StatusBar = "Opening " & Right(aField.Code.Text, Len(aField.Code.Text) - 14)
      aField.Update
      aField.Unlink
   End If
Next ' aField

That simply iterates a table of fields and updates the status bar with the right-most 14 characters of the string in each field (which just happens to contain the path and file name of a list of documents, but we knew that...)

johnny zhang support moderator say:

Hi RichyInnes,

I will answer your questions one by one.
1. “Is it possible to run more than one instance of Azure AD connect tool e.g one instance in each of the source forests and sync the identities? Are there any limitations to this?”
A:  No. If the target is a single tenant, you cannot deploy multiple AAD connect server.

2. “Is it supported to implement multiple Hybrid configurations from separate AD forest to a single tenant, if so do they all need to be Exchange 2013 SP1?”
A:  Yes.  At least one Exchange 2013 SP1 server with the Client Access server role must be installed in each Active Directory forest configured for hybrid deployment.

If the numbers to be averaged are in column A, try this array formula,
=AVERAGE(IF(A:A>0,A:A))

=AVERAGE(IF(A:A<>0,A:A))

{ =AVERAGE(IF(A:A<>0,A:A)) }

I'll approach from a different angle.  Assume your numbers (and zeros, etc) are in column A, beginning at row 2 and ending at row 1052

=SUMIF(A2:A1052,">0") / COUNTIF(A2:A1052,">0")

=SUMIF(A2:A1052,"<>0") / COUNTIF(A2:A1052,"<>0")
