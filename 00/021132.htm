A template that is saved as a dotx file cannot contain macros.

 

Even though you created documents from the templates, if the macros were in the Normal template, they would be available to those documents.  In fact, they are available to ALL documents.

DOTX format does not support macros - which is rather the point. Templates with macros must be saved as DOTM (macro enabled) format. If you record a macro, by default it will be saved in the normal template. To save it in the document template ensure the document template is first opened in Word and select it from the record macro dropdown.

I understand that this will take much of your time but this is the only option that you have to do if you wish to have them back to your lists.

Please refer to the steps below on how to add the recipients again to your safe/blocked senders list:

How to add the email address to your Safe Senders list

1. Sign in to your account. Go to inbox.
2. Click the Gear icon located at the upper-right side of the page.
3. Select Options.
4. Under Preventing junk email, click Safe and blocked senders.
5. Click Safe senders.
6. Enter the email address or domain on the box labeled Sender or domain to mark as safe: and click Add to list >>.

How to add the email address to your Blocked senders list
1. Sign in to your account. Go to inbox.
2. Click the Gear icon located at the upper-right side of the page.
3. Select Options.
4. Under Preventing junk email, click Safe and blocked senders.
5. Click Blocked senders.
6. In the Blocked email address or domain list, enter the email address or domain: and click Add to list >>.

Feel free to post back in this thread if you have other issues or clarifications and we'll be happy to assist you. Your patience is highly appreciated.

Type MIsmatch Error In Worksheet_Change - Microsoft Community

macuseronline say:

Type MIsmatch Error In Worksheet_Change

I'm trying to populate a cell whenever H6 is cleared. But with this code, I am getting a Runtime Error -13; Type Mismatch Error. What am I doing wrong here? Thanks.

Private Sub Worksheet_Change(ByVal Target As Range)    
    If Target = Range("H6") Then If Target = "" Then Range("G1") = "Test"
End Sub
j.e. mcgimpsey say: Not sure exactly why you're getting the error (haven't tested it), but you're using default properties in an extremely loose way, and it almost certainly isn't doing what you intended.

Your code, when expanded to include the default property for a Range object (i.e., the .Value property), is equivalent to

   If Target.Value = Range("H6").Value Then _
      If Target.Value = "" Then _
         Range("G1").Value = "Test"

So every change made to any cell in the worksheet will test that cell's value against the value in cell H6. If comparing the value in Target's cell against whatever value is in cell H6 causes a Type 13 Mismatch error (e.g., if the changed cell has an error value, etc), you'll get the runtime error.

You'd also find that if H6 is blank, that clearing any OTHER cell will also cause "Test" to be written to G1 (including if you tried to clear G1, which leads to a nice little loop).

This code is one better way (there are, of course, many others). It first determines if H6 is included in the changed cell(s), and only if it is does the code proceed to check whether H6 is blank (i.e., its .Value property is Empty). Note that using the With...End With block means that any property starting with just a '.' refers to the object in the With statement (in this case, the worksheet's cell H6):

    Private Sub Worksheet_Change(ByVal Target As Range)
        With Me.Range("H6")
            If Not Intersect(Target, .Cells) Is Nothing Then _
                If IsEmpty(.Value) Then _
                    Me.Range("G1").Value = "Test"
        End With
    End Sub
