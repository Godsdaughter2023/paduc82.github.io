kim benzon say:

If for example the overall sum or value is 8000, it will tell me, that the average is four, but the correct result is 20... That's my problem. lz. say:

Hi Kim

A bit of reading may help understand. See AGGREGATE function:

  • 16 is PERCENTILE.INC sub-function
  • 17 is QUARTILE.INC sub-function
  • 6 is the option that ignores error values

+ you need to understand a bit array calculations. Example:

AGGREGATE(17,6,1/(A2:A8<>0)*A2:A8,1)

  • 1/(A2:A8<>0) generates an array of {1,#DIV/0!,1,#DIV/0!,#DIV/0!,1,#DIV/0!}
    The 1s represent the position - in A2:A8 - of values <>0
    Values equal to 0 do not meet the condition <>0 and consequently = FALSE
    1/FALSE generates a #DIV/0! Error
  • Above array *A2:A8 generates a new array: {10,#DIV/0!,5,#DIV/0!,#DIV/0!,10,#DIV/0!}

AGGREGATE option 6 will ignore the #DIV/0! Errors in the last array and will compute the QUARTILE.INC on {10,5,10} only

Hope this helps
