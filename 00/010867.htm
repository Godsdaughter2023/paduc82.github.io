Instead of trying to do it the way you have envisioned it.  What the way to do it is to do something like this:

 

Dim strSQL As String

Dim lngClientID As Long

Dim rst As DAO.Recordset

 

strSQL = "Select Distinct ClientID From TableNameHere Order By ClientID"

 

Set rst = CurrentDb.OpenRecordset(strSQL)

 

Do Until rst.EOF

   DoCmd.OpenReport "ReportNameHere", acViewPreview, WhereCondition:="[ClientID]=" & rst("ClientID"), WindowMode:=acHidden

   DoCmd.OutputTo acOutputReport, "ReportNameHere", acFormatXLS, "C:\Temp\" & rst("ClientID") & ".xls"

   DoCmd.Close acReport, "ReportNameHere", acSaveNo 

   rst.MoveNext

Loop

 

rst.Close

Set rst = Nothing

 

 

So, the idea is you open a recordset of all of the groups (I used ClientID as my example) and iterate through that, and the code opens the report filtered on that group and then outputs it to Excel.

steve rindsberg say:

>> Of course, if you move the file to a computer that does not have Visio there may be problems. I just don't know exactly how it will react.

That's actually quite predictable.  When you have OLE content (linked stuff like this) in PPT or any other program, what you actually see is a Windows metafile representation of the graphics; under the hood, there's a pointer to the original application/file, which is responsible for creating an updated metafile for display purposes whenever there's an update.

If the original application or file are not available, you still have the same metafile as always; it just won't update anymore, and PPT will bark at you if you double click in an attempt to edit the linked contents.

3. In mailbox features, check the role assignment policy assigned to the mailbox.

4. Back to EAC, in permissions->user roles, find the role assignment policies assigned to the mailbox.
5. Double click on the policy and tick on the MyMailSubscription role.

Hello Karen

Re: #1 - Whether the accounts are the same is something only you can know since you would have to configure the account(s) in each device. For the sake of this reply, going to assume that you have configured the same email account on both your computer and iPhone

Re: #2 - In terms of sync'ing, the most important item to be aware of is that the iPhone/Computer don't sync with each other. Sync'ing is accomplished between the device and the email account server based on which protocol was used to configure the <email account> on each device.

<IF> the account was configured as a POP3 account, only the <Inbox> data is downloaded to the device on a one-time uni-directional basis. If the option was not checked to <Leave messages on server> was not checked then the items would be immediately deleted from the <Inbox> meaning that no other device would have access to that data. No other folders of any type (email, contact or calendar is sync'd with the server.

<IF> the account was configured as an IMAP account, all email folders would be sync'd bidirectionally with the server. Any item deleted on a device would be deleted from the server and subsequently from any other device where the account was configured using IMAP.

<IF> the account was configured as an Exchange account which requires that the email account server supports that protocol, all folders are sync'd with the server (email, contact, calendar).

Sub Mail_As_Attachment(strDocName As String)
Dim oApp As Object
Dim oMail As Object
Dim olInsp As Object
Dim wdDoc As Object
Dim oRng As Object
    On Error Resume Next
    'Get Outlook if it's running
    Set oApp = GetObject(, "Outlook.Application")
    'Outlook wasn't running, start it from code
    If Err <> 0 Then
        Set oApp = CreateObject("Outlook.Application")
    End If
    On Error GoTo 0
    Set oMail = oApp.CreateItem(0)
    With oMail
        .to = "*** Email address is removed for privacy ***"
        .Subject = "Subject Line Here"
        .Attachments.Add strDocName
        .BodyFormat = 2
        Set olInsp = .GetInspector
        Set wdDoc = olInsp.WordEditor
        Set oRng = wdDoc.Range(0, 0)
        oRng.Text = "Hello" & vbCr & vbCr & "Main text of email."
        .Display 'Retain this line
        '.Send        'restore after testing
    End With

    Set oMail = Nothing
    Set oApp = Nothing
    Set olInsp = Nothing
    Set wdDoc = Nothing
    Set oRng = Nothing
End Sub
