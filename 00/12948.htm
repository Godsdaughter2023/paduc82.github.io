bernd p. say: My impression is that you have a misconception of the way Onenote works.

The basics:

Each notebook exists on exactly two places:

  1. the original or master notebook, consisting of one or more *.one files (the sections of the notebook)
  2. as part of the always local cache (a file which is normally invisible to you)
These two places are synced by Onenote (if connected).

The place for a master notebook can be your computer, another computer connected over a LAN (including Wi-Fi) or a Skydrive server.

There is a builtin trap:
If you create a new notebook at first on your computer and later "move" it to Skydrive, Onenote unfortunately copies the local master notebook to Skydrive and leaves the old local master behind, as an orphaned child.
This former original is now just useless and will never be updated.
But some people think (as I know from this Forum) they have two working copies now. That's wrong.
And so is no valid scenario using Skydrive AND Dropbox parallel. Such a scenario will produce the symptoms you describe.

The problem may not have been related to patch then. I would suspect something happened to corrupt that particular database. 

kerryjones[alti-2] say:

I found the problem.  The failure's help tip indicated the database may have been created on a newer version of Access.  I found both of the failing machines had Access 2007 installed on them at some previous point.  When the update ran, it changed the default application for MS Access file extensions.  Under HKEY_CLASSES_ROOT, Access extensions (accdb, accdr ,accdr etc.) were pointing to Access 2007 ('Access.ACCDCFile.12').  The Access 2010 was a run-time installation done a year ago.  Using the registry editor, I changed  THE MS Access file extension defaults from 'Access.ACCDCFile.12' to 'Access.ACCDCFile.14'.  This pointed the default application for these extensions back to the Access 2010 run-time version.  I probably could have re-installed the 2010 run-time to accomplish this as well. 

Also, I was incorrect earlier when stating my database version.  It was re-compiled using Access 2010 not 2007.

Problem is resolved..

Hello oi12000oi12,

At every beta release of Windows (regardless of version) there has always been software that will find it hard to integrate with the current beta version. Anti virus applications are notoriously fickle when it comes to beta version software. It is a testing time for both Microsoft and the Software developers of anti virus software. An anti virus application that works on the current version of Windows 10 may become broken when the next version of Windows 10 is released.

While we all have our 'pet' anti virus application it is worth remembering that Windows 10 comes with its own built in anti virus application, Windows Defender. Windows Defender is quite capable of 'defending' your system against anti virus threats and, personally, Windows Defender is the only anti virus application that is installed on my copy of Windows 10.

By the time the final release of Windows 10 hits the shelves you will find that Avast will have ironed out the problems they are having and everything will be harmonious again.

Rather than using third party anti virus applications at the moment I would suggest you stick with Windows Defender.
