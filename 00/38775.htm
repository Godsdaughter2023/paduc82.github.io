In windows 10 the permissions are a little bit changed. You as administrator don't have by default access to all the things in the computer. In regard to renaming the C: drive, I think it worked after you provided the administrator permission. However in regard to Windows Apps folder, you are not the owner and you don't have read access to that folder either. Of course, since you are administrator you can change this by going in properties at the security tab, changing the owner of the folder to you and changing the permissions. Is not at all a bug, I think these things were implemented just to avoid beginners to delete various important files. Some of them not, but you can set them to run by default with administrator rights by right click > properties and then compatibility.

Is not a bug, it was just implemented to prevent unauthorized access to your computer. Probably there is a command line syntax or a registry hack to enable run as administrator by default in all other programs, software but I don't know it.
