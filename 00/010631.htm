marshall barton say:

THe author's combo box needs to use it's NotInList event procedure to add the new author to the Authors table.  The author's combo box should not play games to get a value in the Books table without first storing the author info in the Authors table.

Just to be clear, the Authors table needs to have a primary key that should be an AutoNumber field (because two different authors may have the same name, making the name an invalid candiafte pk) along with what ever other fields are specific to an author.  Then, the Books table would only have the one autonumber field for the author information, event if the Authors table only has one field for the author name.  (We'll leave aside the arrangement needed to deal books that have multiple authors.until you feel compelled to deal it.)

With all that said, the combo box's NotInList event procedure can add an author's name to the Authors table using some VBA code along these lines:

   db.Execute "INSERT INTO Authors (AuthorName) VALUES(""" & NewData & """ ")
   Response = dbErrorAdded

An alternative if you need to add additional author information in other fields at this same time is to open a form bound to the Authors table where you can enter all the information.  This form would be opened this way:

   DoCmd.OpenForm "Author form", WindowMode:=acDialog, DataMode:=acFormAdd, OpenArgs:= NewData

and insert the author's name in its locked text box with a line of code in the form's Open event procedure:

   Me.txtAuthorName.DefaultValue = """" & Me.OpenArgs & """"
