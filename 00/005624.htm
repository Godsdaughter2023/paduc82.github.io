This is the solution:

1. It's Microsoft's fault; their formula works, but only if you do certain things.

2. The chart MUST be "X Y Scatter" type - it isn't in the insert chart options, you have to insert a chart, click Change Chart Type, then change to X Y Scatter.

3. You need more digits for the formula to be useable (in my case, the accuracy was enough, except that it went into scientific number format so the 5 digits just showed the E01.1 and that was about it). To do this, right-click the formula box on the chart; click Format Trendline Label; change Category to Number; and increase the number of decimal places.

I would agree that Microsoft could document this better.  But they do document it in https://support.microsoft.com/en-us/kb/211967 (click here).

If you copy-and-paste coefficients from the trendline label, I suggest that you choose the Scientific format with 14 decimal places.  That ensures that you have the best representation that Excel permits regardless of the magnitude of the number.

Alternatively, you might try using LINEST.  It appears that you removed your example Excel file, so I cannot give you an example tailored to your problem.  But for an order-6 polynomial trendline, you might select 7 cells in a row (e.g. A1:G1) and array-enter the following formula (press ctrl+shift+Enter instead of just Enter):

=LINEST(Y1:Y100, X1:X100^{1,2,3,4,5,6})

Then A1 is the coefficient for x^6, B1 is the coefficient for x^5, etc.

Caveat:  Sometimes, LINEST returns different coefficients than you see in the trendline label, even when formatted to display 15 signficant digits.  But usually, the estimated y-values are the same.  However, occassionally, LINEST returns bogus coefficients or an Excel error.  In that case, copy-and-paste the trendline coefficients.

Sub TriDonnées_AK()
  Static Path As String
  Dim Filename As String
  Dim R As Range, All As Range
  Dim Data
  Dim FirstAddress As String

  'Ask the user for the path to the CSV files
  With Application.FileDialog(msoFileDialogFolderPicker)
    If Path = "" Then Path = ThisWorkbook.Path
    .InitialFileName = Path
    .Title = "Sélectionnez le dossier qui contient les fichiers CSV:"
    If Not .Show Then Exit Sub
    Path = .SelectedItems(1)
    'Add a backslash if necessary
    If Right(Path, 1) <> "\" Then Path = Path & "\"
  End With

  'Let the user know that we work
  Application.Cursor = xlWait
  Application.StatusBar = "Removing data..."

  'Prepare our sheet, remove existing data
  Set R = Range("Mesures").CurrentRegion
  Set R = R.Offset(8 - R.Row)
  R.ClearContents
  'Import starts at C8:
  Set R = R(1, 1)

  'Screen off, runs faster
  Application.ScreenUpdating = False

  'Get the files
  Filename = Dir(Path & "*.csv")
  Do While Filename <> ""
    'This loop can take a long time...
    Application.StatusBar = "Processing " & Path & Filename
    '...so give some time slice to windows
    DoEvents

    'Read the file
    If ReadCSV(Path & Filename, Data) Then
      'Store the data into the sheet
      R.Resize(UBound(Data) + 1, UBound(Data, 2) + 1) = Data
      'Next cell
      Set R = R.Offset(UBound(Data) + 1)
    Else
      If MsgBox("Invalid file: " & vbCrLf & Path & Filename & vbCrLf & _
          vbCrLf & "Continue?", vbOKCancel, "TriDonnées_AK") = vbCancel Then
        Exit Sub
      End If
    End If
    'Next file
    Filename = Dir
  Loop

  'Remove duplicate dates
  Application.StatusBar = "Remove duplicates..."
  Set R = Range("Mesures").CurrentRegion
  R.RemoveDuplicates Columns:=Array(1, 2), Header:=xlYes

  'Sort the data by the first 2 columns (date and time)
  Application.StatusBar = "Sorting..."
  Set R = Range("Mesures").CurrentRegion
  R.Sort _
    Key1:=R.Cells(1, 1), Order1:=xlAscending, _
    Key2:=R.Cells(1, 2), Order2:=xlAscending, _
    Header:=xlYes

  'Search for "DELETE" in the first column
  Application.StatusBar = "Delete invalid data..."
  With Range("Mesures").CurrentRegion.Columns(1)
    Set R = .Find("DELETE", LookIn:=xlValues, LookAt:=xlWhole)
    If Not R Is Nothing Then
      FirstAddress = R.Address
      Do
        If All Is Nothing Then
          Set All = R
        Else
          Set All = Union(All, R)
        End If
        Set R = .FindNext(R)
      Loop Until R.Address = FirstAddress
      All.EntireRow.Delete
    End If
  End With

  'Generate the min/max formulas
  Set All = Range("Mesures").CurrentRegion
  'Refer to the data cells only
  Set All = All.Offset(1).Resize(All.Rows.Count - 1)
  'For each cell in the min/max region
  For Each R In Range("C4:C5,E4:L5")
    If R.Row = 4 Then
      R.Formula = Replace("=MIN(ref)", "ref", _
        Intersect(All, R.EntireColumn).Address(0, 0))
    Else
      R.Formula = Replace("=MAX(ref)", "ref", _
        Intersect(All, R.EntireColumn).Address(0, 0))
    End If
  Next

  'Done
  Application.Cursor = xlDefault
  Application.StatusBar = False
  Application.ScreenUpdating = True
End Sub

Private Function ReadCSV(ByVal Filename As String, ByRef Data As Variant) As Boolean
  'Read a CSV file, return a 0 based 2D array with formatted data
  Dim fs As Object 'Scripting.FileSystemObject
  Dim ts As Object 'Scripting.TextStream
  Dim Buffer As String
  Dim Lines, Temp
  Dim i As Long, j As Long, k As Long

  'Read the text file in unicode mode
  Set fs = CreateObject("Scripting.FileSystemObject")
  Set ts = fs.OpenTextFile(Filename, Format:=True)
  Buffer = ts.ReadAll
  ts.Close

  'Find the "Message" marker
  i = InStr(1, Buffer, "Message", vbTextCompare)
  If i = 0 Then Exit Function
  'Find the next linebreak
  i = InStr(i, Buffer, vbCrLf, vbBinaryCompare)
  If i = 0 Then Exit Function
  'Remove the headings
  Buffer = Mid$(Buffer, i + 2)
  'Remove empty lines
  Do
    i = Len(Buffer)
    If Right$(Buffer, 2) = vbCrLf Then Buffer = Left$(Buffer, i - 2)
    If Right$(Buffer, 1) = " " Then Buffer = RTrim$(Buffer)
  Loop Until i = Len(Buffer)

  'Split the rest by line breaks
  Lines = Split(Buffer, vbCrLf)

  'Create 2D array
  ReDim Data(0 To UBound(Lines), 0 To 0)
  For i = 0 To UBound(Lines)
    'Split each line into words
    Temp = Split(Lines(i), ";")
    'Expand the 2D array if necessary
    If UBound(Temp) > UBound(Data, 2) Then
      ReDim Preserve Data(0 To UBound(Lines), 0 To UBound(Temp))
    End If
    'Copy the words into the array
    For j = 0 To UBound(Temp)
      Data(i, j) = Temp(j)
    Next
  Next

  'Convert date and time values
  For j = 0 To 1
    For i = 0 To UBound(Data)
      If IsDate(Data(i, j)) Then
        Data(i, j) = CDate(Data(i, j))
      Else
        'Mark this row for deletion
        Data(i, 0) = "DELETE"
      End If
    Next
  Next

  'Numerical values
  For j = 2 To UBound(Data, 2)
    For i = 0 To UBound(Data)
      If IsNumeric(Data(i, j)) Then
        Data(i, j) = Val(Data(i, j))
      Else
        'This happens if the row contain "???":
        'Mark this row for deletion
        'Data(i, 0) = "DELETE"
      End If
    Next
  Next

  ReadCSV = True
End Function

marshall barton say:

The DoMenuItem method is a hold over from A1 or 2 (20+ years ago) and has been deprecated for at least 15 years.  There is a newer DoCmd.SaveRecord method, but it is more definitive (because you specify which form object has the record) to use a line of VBA code instead of any DoCmd method:

   If Me.Dirty Then Me.Dirty = False

The Dirty property is True if the record has been modified and needs to be saved, so setting the Dirty property to False forces the record to be saved.

BTW, I have used the technique Duane suggested many times and find it far preferable to the alternatives.  Since the NumCopies field is in the report's record source query, it can be used to number the copies (e.g. a page header/footer text box with an expression like  ="Copy " & NumCopies & " of 3")

If you want to specify the number of copies instead of always using 3, add an unbound text box (named txtNumberOfCopies) to the form and use Forms![your form].txtNumberOfCopies instead of the 3 in your query.

jay freedman say:

Legacy form field protection interferes with quite a few features throughout the document, even in unprotected sections. There is no way to change that.

If everyone who uses your template has at least Word 2007, use content controls instead of legacy form fields for the fill-ins. To "protect" the instructor's text, select that entire section and click Developer > Group > Group. Repeat for each instructor section, leaving the unprotected sections without grouping. Then comments will be enabled in the unprotected sections, but disabled in the protected sections (including the content controls).
