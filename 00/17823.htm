    In addition to the comment made by Graham Mayor, as posted, your field code also have two sets of double quotation marks around Required Text, which would mean that if the value of Tran(s)Number was "INT", the result would be blank, and if the value of Tran(s)Number was not "INT", the result would be Required.

    Assume the data range is A1:Z100, the date is in named range DATE, and transaction type is in named range TYPE. If I wanted to count the number of transaction types =X that occured in July 2011, I know the following formula would work:

     

    =SUMPRODUCT(--(TYPE="X"),--(YEAR(DATE)=2011,--(MONTH(DATE)=7)

     

    Is there a more efficient way of looking at dates when only month and year are being considered?

    David

    =SUMPRODUCT((TYPE="X")*(YEAR(DATE)=2011)*(MONTH(DATE)=7))

     

    or

     

    =SUMPRODUCT((TYPE="X")*(DATE-DAY(DATE)+1=DATE(2011,7,1)))

     

    might be marginally more efficient.

    zhang_yi say:

    Hi Saurabh,

    This kind of mapping a property is not available in SharePoint online. To enable the directory service properties in SharePoint online, it requires us to configure the profile synchronization at first. This is available in SharePoint server, but not in SharePoint online. Currently, we can only use the default user property mappings listed in the following article. https://technet.microsoft.com/en-us/library/hh147510.aspx.

    As a workaround, you can add the field as a SharePoint Online only attribute, then manually fill in the field for your users, or have them to fill it themselves, but the attribute cannot be mapped back to local AD.
