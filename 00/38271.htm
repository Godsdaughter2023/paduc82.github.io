a. user say:

hazren,
To answer your first question, DFS Replication can be used to keep files synchronized across multiple computers on a network. It can update just the changed blocks from a file, so it is very efficient. If you don’t plan to synchronize files across multiple computers, you can disable the service but it is set to manual by default so should not be running unless being used, which obviously it is.  If you have multiple computers DFS keeps the shared files synchronized so if you use the "People Near Me" feature all users will see the same files and folders that are being shared.

There could be several reasons why DFS stopped working.  You will need to check the entries in the Event Viewer logs for System to see what is causing the errors.  If you are not on a network or don't need to share files between computers you can disable the service to avoid the pop up error notification. 

Here is how to disable the service:
Start> type "services" in the search box and hit enter > if it asks for permission click continue > in the services panel find the DFS Replication service and double click it to open its properties sheet > Click the down arrow next to Startup Type and select "disabled".  Click "Apply" and close out of the Services management console.

To find the errors in Event Viewer:
Click Start > type "event viewer" in the search box and hit enter, click Continue if asked for permission.  Expand the Applications and Services Logs section and click on DFW Replication.  You will see all the events logged by the DFS service.  Click on any events that show red error or yellow warning read the detailed description.  This will tell you what caused the service to stop.  You can post that information back on this forum so that we can further assist you.
