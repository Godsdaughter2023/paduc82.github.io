    Item 1:  that's easy, if the Win7 box is joined to AD, then a Domain User account (other than a member of the Domain Admins group) is not an administrator of local machine by default.  You can also use a Restricted Groups GPO setting to enforce membership of the local administrators.

    Item 2 and 3:  a UAC prompt is looking to accomplish a credentials elevation.  even though you are prompted, depending on what you are trying to do, you may or may not actually require admin level credentials.  assuming the action does require admin credentials, those credentials only have to be an administrator on the local machine - not neccessarily a Domain Admin or even a Domain account.  UAC just happens to call elevation prompts "administrator"

    For example, on my systems, I have a local computer account each on each win7 box named "support01" that is a member of the local administrators group on the win7 box (and that membership is enforced via a restricted groups GPO).  If a Standard User requires local administrative credentials to accomplish a goal - my first preference is to have a support staffer do it for them - if that is not possible (such as a user on the road and we cannot get remote connectivity to him) - then I can divulge the password of the local "support01" account to the standard user to allow him to accomplish his task - assuming that task is acceptable to me;  (I then use a GPO prefernece setting to change that password so he cannot use that account again without asking - at least once he hs reconnected to the managed network and picked-up the new GPO value).

    This "support01" account is essentially "hidden" from a Standard User in the sense that on a domain joined machine, you don't get the silly User Logon Icons - you must press ctr-alt-delete, then explicity type the userID and password with which you wish to authenticate.  Also, a Standard User would not be able to see the account in the computer's Local Users and Groups UI.

    Item 4: becomes a mute point.

    Based on the breadth of your original question - I would recommend you take a couple of Microsoft Official Curriculum courses relatd to both AD and Win7.  That would go a long way to getting you the core info you need to design the correct solution for your enviroment.

    Last but not least, and account in the local administrators group of the computer is an Administrator of that computer - domain joined or not.  Meaning any Standard User can install software on the computer if he has the userID and Password of ANY account (domain or local) that is a member of the local adminstrators group.

    remitchell say: To be clear:  a Restricted Groups GPO is used to manage WHO is a member of any given group.  Once defined, any userID manually removed from the defined group will be automatically added back at the next policy refresh - and any userID manually added to the defined group will be automatically removed.

    If you wish to use a GPO to manage a local userID's password, then you must create a Group Policy Preference entry for the userID;  this is supported by default in Win7, but as I recall would require the "Group Policy Preference Client Side Extensions" update be installed on an XP box to get the same functionality.
