(This is where I think I have found a breakthrough in determining the cause of this bug as no-one mentions it in linked articles below.)

  1. Create a fresh install of windows 10 Home (I used a VM purposely created to chase down this bug) with a single offline admin account (we'll call it Account).

  2. Set the lockscreen to a non default image (anything to differentiate it from built in windows images).

  3. Create a new offline user account (we'll call it Test).

  4. Give Test admin rights.

  5. Sign out of Account and log in to Test.

  6. From Test set a lockscreen image different to Account or just leave it as default.

  7. Log out of test.

  8. The lockscreen will now be that of the Test account. Select Account, the background will now change to that of Account. All is well...

a. user say:

You can't backup individual programs, like games. You can backup game data, if you know which files store the data.

However, if you're sending your computer away for repair, what you really want to do is make a disk image. An image is an exact copy of your hard disk, stored in a single file. You'll need disk imaging software to do this, and the technique takes a while to master. But it's well worth the effort. No matter what happens to your computer while it's out of your sight, you still have an exact copy of your hard disk that you can restore at any time.

Something else you should definitely do if you're sending your computer for repairs is to backup all your personal data. That means all your documents, pictures, music, videos, email, favorites ... everything that's personal to you. Once you've backed up your personal data, and tested your backup to make sure you can still access it, then you should erase everything that you backed up. Do not allow your personal information to fall into unknown hands.
