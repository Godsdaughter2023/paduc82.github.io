vanderghast say:

It is definitively a matter of how we order (ascending or descending) within  the two sub-groups, and between the two sub-groups. The following seems to produce exactly what you seek:

 

SELECT a.mach, a.so,
   Count(*) AS Rank,
   Last(a.shipDate) AS OfshipDate,
   LAST(a.startDate) AS OfstartDate
FROM Table1 AS a INNER JOIN Table1 AS b ON a.Mach = b.Mach
WHERE ((a.shipDate<= now() ) AND ( a.shipDate >= b.shipDate AND  b.shipDate<= now()))
     OR ((a.shipDate > now() ) AND ( a.startDate>= b.startDate  OR  b.shipDate<= now()))
GROUP BY a.mach, a.so;

 

Note that all the difference is in the use of < or >, as it fits. I added the shipDate and the startDate in order to more easily test the validity of the solution.

 

I added a proper inner join, so that the rank will be computed inside each given mach value (you won't have to run different queries for different mach values), but the main modif. I did is really in the WHERE clause, with the > and <  signs.

rick rothstein, mvp - excel say:

As long as the array is two-dimensional (even though you specified part of a single row, VB captured a two-dimensional array), you can just assign it to a range (of the same dimensions)...

myArray = Range("A10:D10")
myArray(1, 6) = "New Value"
Range("A10:D10") = myArray

The first dimension is the row offset and the second dimension is the column offset. So, you could just as easily do this with a rectangular array...

myArray = Range("A10:F20")
myArray(4, 5) = "New Value"
Range("A10:F20") = myArray
