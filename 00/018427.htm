Add unwanted names to the Blocked Senders List

Messages from addresses or domain names in the Blocked Senders List are always treated as junk. Outlook moves any detected incoming message from senders in the Blocked Senders List to theJunk E-mail folder, regardless of the content of the message.

 Note    The Outlook Junk E-mail Filter does not stop junk e-mail from being delivered, but instead diverts suspected spam to the Junk E-mail folder instead of your inbox. You can make the Junk E-mail Filter more strict by changing its level of protection, or some third-party solutions are available which can be more aggressive.

 

To add a specific sender to the Blocked Senders List, click a message from the sender. On theHome tab, in the Delete group, click Junk and then clickBlock Sender.

  1. On the Home tab, in the Delete group, clickJunk and then click Junk E-mail Options.
  2. On the Blocked Senders tab, click Add.

trevor rabey say:

When you say "20% of the time" you mean 20% of the resource's time.

The resource's max units in the pool is 50%.

Try a simple test.

One file, the pool, with 1 resource having max units 50%.

Two files, Project A and Project B, each with 1 x 10 day task, Task A and Task B, both tasks scheduled for the same 10 days.

Project A and Project B, both share resources from the pool, pool takes precedence.

Assign the resource at 20% and 30% to Task A and Task B respectively.

No problem. Resource is not over-allocated.

If you go to the resource sheet in project A and edit the max units down to 20%, the resource is immediately over-allocated.

If you check the pool, you will see that you have edited the pool.

Then run the same test, but with "sharer takes precedence".

It makes no difference. Resource still goes over-allocated.

So if this is what you are trying to do, it is not controlled by "sharer takes precedence".

"On conflict with calendar or resource information"

But over-allocation is not a conflict, just over-allocation.

You will need a different test to check what "sharer takes precedence" really does control.
