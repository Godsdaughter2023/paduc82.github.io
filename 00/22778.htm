On restart I got the eternal "Loading profile..." blue box.

I spent lots of time trying various "fixes", some suggesting creating a new profile which wasn't an option as I'm not an exchange user and have lots of imap accounts plus calendar data etc.

It wouldn't start in safe mode. I tried new registry keys relating to graphics acceleration, I tried repairing all the ost and pst files. Nothing worked.

I then tried simply deleting the ost file that related to the new failed imap account.

The problem was as soon as I tried to launch Outlook it recreated the ost file and then it sticks at loading profile again.

So my solution was this:

  1. Make sure Outlook isn't trying to start.
  2. Browse to the folder containing your ost files. In Windows 10 this should be:
    C > Users > [Username] > AppData (hidden) > Local > Microsoft > Outlook
  3. Copy the full name (including extension) of the new ost file that relates to the failed account.
  4. Delete the same problematic ost file
  5. Now to prevent it being recreated with bad data, right click and select New > Text Document. 
  6. Rename the text file using the name of the ost file on your clipboard
  7. Start Outlook
  8. "Remove" the offending email account via Outlook (File > Account Settings)

You're now back to where you were hours ago and ready to try and add that email account again. My advice is, let it detect settings and add the account automatically. It hung for me in the first place as I tried to manually add all the mailbox settings. So, the second time around I let Outlook do all its fancy detection (incorrectly) and then I went in and adjusted the settings. 

I hope this works for other people and prevents wasted time.
