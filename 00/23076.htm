ken sheridan say:

This is an unusual request.  You appear to want to return the values from an incorrectly designed table in the way that a correctly designed table would be structured.  More commonly we are asked how to return the data horizontally in the way your table is structured from values in a correctly designed table.

To return the values in the vertical layout you illustrate base the report on a UNION ALL query:

SELECT [Code], [North] As Company,
"North" As Location, 1 As LocationIndex
FROM [YourTable]
WHERE [North] IS NOT NULL
UNION ALL
SELECT [Code], [South], "South", 2
FROM [YourTable]
WHERE [South] IS NOT NULL
UNION ALL
SELECT [Code], [East], "East", 3
FROM [YourTable]
WHERE [East] IS NOT NULL
UNION ALL
SELECT [Code], [West], "West", 4
FROM [YourTable]
WHERE [West] IS NOT NULL;

Beacuse a query like this recasts the data into a correct format, it is sometimes colloquially referred to as a 'normalizing' query.

You can include the Location column or omit it from the report as desired.  The LocationIndex column should not be included, but used to sort the report consistently by ordering it first on Code and then on LocationIndex.  Don't be tempted to order the query; reports by and large ignore a query's ORDER BY clause.  Use the report's internal sorting and grouping mechanism to order it.

How do I subtract consecutive records from within the same field - Microsoft Community

jcall1 say:

How do I subtract consecutive records from within the same field

I have a table that contains records that have a date/time stamp field.  I want to subtract the following record from the prior record.  Easily done in Excel but I'm struggling to find a method in access.  See the data example below:

Data:

START_DATE_TIME
10/2/12 5:44 AM
10/2/12 5:47 AM
10/2/12 5:49 AM
10/2/12 5:50 AM
10/2/12 5:52 AM

Desired Result:

START_DATE_TIME Cycle
10/2/12 5:44 AM 0:03:16
10/2/12 5:47 AM 0:01:49
10/2/12 5:49 AM 0:01:46
10/2/12 5:50 AM 0:01:49
10/2/12 5:52 AM 0:01:47
john w. vinson say:

SELECT A.[Start_Date_Time], DateDiff("s", A.[Start_Date_Time], B.[Start_Date_Time]
FROM yourtable AS A, yourtable AS B
WHERE B.[Start_Date_Time] = (SELECT Min(C.[Start_Date_Time]) FROM yourtable AS C WHERE C.[Start_Date_Time] > A.[Start_Date_Time]);

will get an integer count of seconds. If the time lapse will never exceed 24 hours you can probably get away with just subtracting A.[Start_Date_Time] from B.[Start_Date_Time].

The problem is that a relational table doesn't really have a usable construct "the next record". The records in a table have no meaningful order; they should be viewed as just a bucket full of records. If you want to compare fields in two records then you must use something like the above subquery to identify WHICH record.
