macropod say:

Assuming the sources are all contained in the document as bibliography entries (i.e. they appear if you use References>Bibliography>Insert Bibliography), you could use a macro like:

Sub SourcesImport()
Dim oSrc As Source
'In case the source is already present
On Error Resume Next
With ActiveDocument
  For Each oSrc In .Bibliography.Sources
    Application.Bibliography.Sources.Add oSrc.XML
  Next
End With
End Sub

Access 2007, identify current user - Microsoft Community

lesleew say:

Access 2007, identify current user

I am creating a database and on my computer the CurrentUser()=Admin.  How was Admin setup?  I need to be able to identify the CurrentUser on a system used on a network and I do not know how to retrieve the correct information.  Can anyone help me?

hansv mvp say:

Admin is the default Access user - it has nothing to do with the user's login name. To get the latter, place the following code at the top of a standard module:

Private Declare Function WNetGetUserA Lib "mpr.dll" _
    (ByVal lpszLocalName As String, ByVal lpszUserName As String, lpcchBuffer As Long) As Long

Public Function GetNetUser() As String
    Dim lpUserName As String, lpnLength As Long, lResult As Long
    'Create a buffer
    lpUserName = String(256, Chr$(0))
    'Get the network user
    lResult = WNetGetUserA(vbNullString, lpUserName, 256)
    If lResult = 0 Then
        GetNetUser = Left$(lpUserName, InStr(1, lpUserName, Chr$(0)) - 1)
    Else
        GetNetUser = "-unknown-"
    End If
End Function

You can use it in a query as GetNetUser() and in the control source of a text box on a form or report as =GetNetUser()
