rohn007 say:

Word 2010 and 2007 both use DOCX/DOCM file format by default. 2010 has some features that 2007 does not. DOCX Files created by 2010 have a slightly different structure to support those new feature.

2007 can read files created by 2010 without any addon required. But IF you use any of those new features, ie "Text Effects" on the Home tab, they will be disabled. Some of the features, like Text Effects, are stripped out while others are changed into static none-editable images.

2010 can read files created in 2007 (and 2003 DOC) with no problem. When you open one of them you will see [Compatibility Mode] beside the file name in the title bar. At the same time, any new 2010 features that the file format does support will be disabled. For example the Text Effects button is grayed out.

If you wish to ensure that only 2007 features are used in a 2010 installation you have to open a 2007 file. You could do this by creating a blank file on 2007 and copying it to your 2010 machine. Then when you need a new file, copy and rename it in explorer before opening the new copy in 2010.

alexsun_2012 say:

Hi Sven,

There is no official migration solution for POP connector.

However, if the emails of the mailboxes are kept in the server side, you can also set it up as a connected account in Office 365. Firstly, you need to create an Exchange Online mailbox. In Outlook Web App, you can try to add the on-premises email account via POP3. Once the connection is established, all the emails will be synced to the Office 365 mailbox from your mail server, but the calendar events and contacts are not.

If the connected account doesn't work for you, the only workaround is to use manual PST migration. Meanwhile, other community members with the related experience are welcomed to share their valuable insights.
