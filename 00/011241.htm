Sub GrammarOneSpace()

    End With

    ActiveDocument.GrammarChecked = False

tulaw07 say:

Word 2011 - Using Columns for only Part of the Document

I am trying to use columns in only part of a document.  I have one column at the very beginning of the document.  Then I switch to three columns -- the first is 2.75", the second is 1.5", and the third is 2.75", with no spacing in between.  Basically, I have text in the first, then I have a line of colons (:) in the small second column centred on the middle of the page, then I have a bit more text in the third column.

I get the document to format properly.  (I enter the text for the initial single-column part.  I then go to Columns and I select 3 columns & enter in the measurements.  Then I choose to apply to this point forward.  I then enter my columnar text.)

At this point, I want to stop using three columns & go back to a single column for the remainder of the document.  So I select Columns, one column, choose to apply to this point forward.  I'm able to enter ONE LINE of text.  I want a centred heading.  I centre, bold, and underline what I want.  But when I hit enter for the hard return, what is in my third column above disappears.  I have to hit ctrl-z a few times to get it back.  I can't figure out what I'm doing wrong.  It always starts to work, but then when I want to have a hard return after I've gone back to the single-column format, it takes out what's in the third column, and only what's in the third column.  It leaves the other two columns and the text earlier (before going to 3 columns) alone.

Please help me fix this.  It is driving me insane.
john mcghie say:
First, turn on your non-printing characters display, so you can see what you are doing.

In Word, all formatting is contained in the terminating character of an object.  In the case of columns, the column specification is contained in the section break following the columned part.

So first you need to ensure that the section break that ends the three-column component is not disturbed by your typing.

Now:  Section breaks come in three types: Left, Right, and Continuous.  Respectively they force the page that their section begins with to be a left page, a right page, or to have no page break.

The trick is that a Section Break of type "Continuous" is also a signal to Word to "balance" the columns that end with that section-break.  So as you change the text in a section that ends with a Continuous break, Word will move text from column to column to try to have all of the columns in the section ending at the same depth down the page ("balanced").

For what you describe, I think "Columns" is the wrong mechanism.  I think you would be better off using a "Table" with three columns.  With a table, you can hide all the borders except the centre one, which you can make dotted, to achieve the same effect.

With a table, Word will not attempt to move text from column to column as the text changes.

Hope this helps

  Dim FolderName As String ' Name of the folder

Sub Scan()

  Dim strDateiname As String

  Set objCommonDialog = New WIA.CommonDialog
  Set objImage = objCommonDialog.ShowAcquireImage
