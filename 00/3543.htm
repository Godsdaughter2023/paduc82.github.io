emilyftw say:

Keep "toggle filter" from clearing out all of my filter criteria

I have a form.  I want to both (1) allow the users to filter the data by form, selection, etc. using the filter menus, and (2) maintain one of several "base" filters on the data.  The base filters represent certain large-scale cross sections of the data such as: (A) projects which are open, (B) projects which are not open, (C) projects which are not open but which were closed within the last 21 days, etc.  The user specifies which "base" filter they are applying through a combobox control.

If a user chooses their base filter (say they are looking at all open projects), it properly displays only those records.  And if the user then (say) applies a filter by form, Access properly displays the records that meet both the base filter criteria (that was already applied) and the new form-filter criteria.  BUT...  if the user then wants to remove the filter-by-form criteria by clicking the "toggle filter" button, then both the base filter and the form-filter criteria are removed, and it just shows every record in the table.

My thought for how to fix this was to find an event that fires after the user clicks the "toggle filter" button and put some code in there such as the following:

'  If current filter is blank, then apply the filter stored in the myCriteria string

That doesn't work.  I've tried it in the following events: ApplyFilter, AfterUpdate, Current.  I also stuck a MsgBox line in there to display for me what the filter value was.  The MsgBox displayed for the ApplyFilter and Current events, but it always showed the criteria that existed BEFORE the toggle (such that the clearing out of the criteria must happen after those events somehow).  The MsgBox didn't display at all when I tried the AfterUpdate event, so an AfterUpdate event must not fire when a filter is toggled (which doesn't surprise me).

But I'm stuck.  Is there an event that it would work in?  Or any other suggestions for how to deal with this?

Thanks!
Emily

dirk goldgar say:

What I would probably do, for this sort of functionality, is not apply the "base filters" as filters at all, but rather use the chosen filter to modify the form's RecordSource, swapping among SQL statements that apply the filter criteria via WHERE clauses.  For example:

Private Sub cboBaseFilter_AfterUpdate()

    Select Case Me.cboBaseFilter

        Case "Open Projects"
            Me.RecordSource = "SELECT * FROM PROJECTS WHERE CloseDate Is Null"

        Case "Closed Projects"
            Me.RecordSource = "SELECT * FROM PROJECTS WHERE CloseDate Is Not Null"

        Case "All Projects"
            Me.RecordSource = "SELECT * FROM PROJECTS"

        Case "Recently Closed Projects"
            Me.RecordSource = "SELECT * FROM PROJECTS WHERE CloseDate > (Date() - 21)"

    End Select

End Sub

Then the user can apply or remove their own filters at will, but those filters will always be applied on top of the current recordsource, so will have no effect on the "base filter".

macropod say:

Select Case and nested Ifs should be used differently.

 

Where the processing of a single variable depends on multiple possible values, as in OssieMac's example, Select Case is the better approach.

 

Nested Ifs can be more efficient than And & Or tests. For example:
If A then

  If B then

    If C then 'Do something
  End If

End If

is more efficient than:

If A And B And C then 'Do something

as the nested If only tests the B & C conditions after establishing their predecessors to be true, whereas the compound And test has to test all three conditions, regardless of the truth of A & B. Significant improvements can be gained by carefully ordering the If tests.

ossiemac say:

Firstly the Select Case is much easier to follow. The other big advantage I find is that it is better than if statements where Or operator is used. ie. you can list multiple items separated with a comma.

 

The example can be used 2 ways. Include the sheets that match or do not put any code in that case and code in the case else will process the sheets that do not match.

 

Sub test()
    Dim ws As Worksheet
   
    For Each ws In Worksheets
   
        Select Case ws.Name
       
            Case "Sheet1", "Sheet4", "Sheet7", "Sheet8", "Sheet10"
                'code here if any of above match and to be included for processing
                'If the names in the list are to be excluded then no need to
                'actually have code under this Case.
            Case Else
                'code here if sheets that do NOT match to be processed
        End Select
    Next ws

End Sub
