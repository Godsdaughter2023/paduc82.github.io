doug robbins - mvp office apps & services (word) say:

If the references to a requirement are in the same document, then by using the cross reference facility, the references will be updated to reflect any changes in the numbering of the target of the cross reference.


If the references are in another document, you could bookmark the target of the reference and then use and IncludeText field to create the reference.


In both cases, it will be necessary to update the fields in the document(s) for the references to point to the correct items.

rohn007 say:
It depends in part on how you want to handle it.

If you use the automation that Doug describes, bookmarks and cross references then the numbers will be automatically updated to reflect changes when you insert new requirements.

On the other hand, if after the initial document creation you want the requirement numbers to remain static everywhere in the document (ensuring constant numbers when people are referring to printed documents) , you will have to pick a point in time to convert the the autonumbers to Static text (say document sign off).
Then you can manually add new custom numbers between existing static text numbers when additional requirements are added. 

You can add the "Convert List Text" command to the QAT to do the autonumber to text conversion for you.

jim gordon mvp say:

There are two ways to open .csv files

Double-click the .csv file in Finder

Excel will use a default set of rules and attempt to display the data. Sometimes this works fine. Others, not so good, depending on the file contents.

In Excel, use Data > From Text

When you open a .csv file using this method, the Text Import Wizard displays.

Step #1: If you know the data was delimited, choose delimited, otherwise choose Fixed Width. If you know the headers are not on the first row, set the import to begin on the row having the headers.

Step #2: Assuming the .csv file is, in fact, delimited, Excel will show you a preview on the bottom of the wizard. Excel usually guesses correctly even when the delimiter is not a comma as promised by the file provider. You can adjust the delimiter settings as needed. Many web sites lie about comma separated value (.csv) files and use other delimiters instead of commas.

Step #3: Notice that the default format for data coming in is General. From your description of the problem, you should change the setting to TEXT or DATE as appropriate. The Advanced button lets you specify number formats.

Once you have your .csv file imported, you can use help columns having formulas that use LEFT, RIGHT, MID, FIND, SEARCH, IF and other formulas that manipulate text. You can try Text to Columns. You may find that FLASH FILL is a very powerful tool to help you parse the data into columns.
