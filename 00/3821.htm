xl_geek say: D17 & E17 (which are hidden) contain the values 43102.

hansv mvp say:

OK, so D17 contains 43102 and D20 contains 2. The other cells in the range D6:D21 are empty. So =SUM(D6:D21) returns 43102 + 2 = 43104. SUM includes both visible and hidden cells, so this result is to be expected. If you want to exclude hidden rows from the sum, either adjust the range so that it contains only visible cells, or use the AGGREGATE function:

=AGGREGATE(9,5,D6:D21)

The first argument 9 tells Excel to calculate the sum, and the second argument 5 tells Excel to ignore hidden cells.

See https://support.office.com/en-us/article/aggregate-function-43b9278e-6aa7-4f17-92b6-e19993fa26df for more information about AGGREGATE.

rohn007 say:

This sounds like a typical new computer problem.

The new computer comes with a factory installed "free trial" of Office 365. The user buys, and installs their own copy of Office. With 2 unactivated copies of Office on the machine the activation system has "issues".

So if this is a new computer, go to Control Panel > Programs and Features.  Look through the list of installed applications for more than one copy of Office.  If you find them, uninstall the factory version AND do an Office Repair of your installed version.  You MAY have to do one more activation (but probably not).
