Unprotect the form, right-click the reference field, and click Toggle Field Code. You'll probably see something like

{ REF Text1 \* MERGEFORMAT }

Change it to

{ REF Text1 \* CHARFORMAT }

and make sure the field -- or at least the R in REF -- is formatted the way you want it. Press F9 to collapse the field, relock the form, and try it again.

Office 365 license provisioning status clarification

May i know what is the difference between PendingActivation and PendingInput provisioning status for a service plan?

I have earlier never seen a status called PendingActivation, is this status newly introduced by Microsoft in recent times?

biang_0 say:

Hi RoutineUser,

Here are some updates:

PendingActivation basically means work load is available for user/tenant but need to backfill/provision.

For example, for Yammer, if the license has been assigned to the user but the status is PendingActivation, it means we will provision the service when the user starts using the service(just in time provisioning). After the provisioning starts, it will go to PendingInput or Success depending on the results.

PendingInput means the license assignment info has not been synced to the corresponding service’s data store.

Success means the whole provisioning completes successfully.

Disable means the license is not assigned to the user.
