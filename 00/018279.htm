What you say is true only if what you have selected is not part of the document body (e.g. a textbox); in that case, only fields in the selected object will be updated. ossiemac say:

Do you have any protected worksheets and controls on the userform with Control Source set to a cell on the protected worksheet? If so, unprotect the worksheet/s and see if you still get the same problem. If this is a problem then unprotect the worksheet/s before showing the userform or remove the ControlSource property from the control/s and use change event code with the control that will unprotect the worksheet, update the cell and then re-protect the worksheet. (When a Userform opens it attempts to update the control source cell/s even if they don't really need updating and therefore the worksheet must be unprotected before showing the userform. Cannot do this in the Userform Initialize event because the userform is already open by the time the initialize event runs.)

If  no protected worksheets then do you have controls on the userform with Control Source set to a cell on the unprotected worksheet and updating of the cells causes a workbook calculate.

Check that you don't have any code associated with the Userform that changes cells on worksheets and then other event code that re-changes the cells after update on worksheets and effectively setting an eternal loop going.
