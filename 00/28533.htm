tryx3 say:

Peter,

The methods that ought to be available to you for restoring an image in Windows 10 are

1  Select F8 during booting to get to the Startup settings menu then pressing F10 and selecting Launch Recovery environment.  Then, on the Choose an option menu, select Troubleshoot, Advanced options, System image recovery*.

2  Boot from the Win10 installation disk, choose language-keyboard, select Repair your computer then, on the Choose an option menu, select Troubleshoot, Advanced options, System image recovery*.

3  Boot from the Recovery Drive and doing something similar.

[In Windows 7, you could use an installation disk or  a System repair disk to get at the same tools - the "System recovery options" menu contained a link to a Command prompt and a link to restoring a system image.  You could also get at these tools from within Windows if your setup provided for that.]

- - - - - - - - - - - - - - - -

* To check that you know what drive letters the temporary boot environment has allocated to the disks & partitions on your computer, you can make these selections the long way round:-

On the Choose an option menu, select Troubleshoot, Advanced options, Command prompt, [mess around checking drive contents** until you know which drive letter has been allocated to which drive], exit Command prompt, {you are flung back into the Choose an option menu}, select Troubleshoot, Advanced options, System image recovery

- - - - - - - - - - - - - - - -

** I have created an empty file in the root folder of every drive [ThisIsOS7, ThisIsOS10, ThisIsData, ThisIsArchie , ThisIsBackup, ThisIsTom, ThisIsDick, ThisIsHarry, ...].  This allows me, despite the fraught situation of having a corrupted system, to be certain of what drive letters are being used for everything.  If you do not have markers like these you will need to jump around and use dir commands to identify each drive [you have to ignore the X drive as it is a virtual drive even though it looks similar to your Windows 10 drive].
