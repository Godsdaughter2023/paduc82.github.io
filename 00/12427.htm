a. user say:

I don't have this particular product, but
 a. print/scan products frequently come with
    - the manufacturer's own software for performing basic scan/OCR functions
    - third-party scan/OCR software, such as Adobe Acrobat. This is often a "Lite" version or an old version which the software manufacturer is willing to licence to the hardware manufacturer at reduced cost. Sometimes the features in the Lite or old version are all you need, and may not even have changed in more recent versions. Anyway, I would guess that Fujitsu put this hardware/software package a few years ago. What has also changed is that Mac OSX has gone through several versions in the past few years, and may have changed in ways that make it harder to use some of the software features.

Sometimes, you just don't need the 3rd party software. 

 b. With scanners, you can typically either 
  - "just scan", which captures each page as an image. You can't edit that text using (say) Word. Or
  - "scan and OCR", which converts the text on the page into plain text (it isn't in image format any more) and puts it in formats such as plain text, Word format, etc. so you can open the document in Word and edit it. OCR software capabilities vary - some products just recognise text, others try to recognise text areas, image areas and try to produce output that matches the original as closely as possible, but with editable text and individual images. 
  - "scan, and OCR putting the plain text 'behind' the image. That's used to create a searchable .pdf file (and I think it will always be .pdf, but again, you cannot typically edit the text of the output file. 

If you do not need any kind of OCR (i.e. you do not need to edit the scanned text), then you probably need to choose a feature that does "just scan". My guess is that the manufacturer's own software will do that. If you want a multipage document to be scanned into a single output file, the simplest option is usually to choose Acrobat format, i.e. .pdf. I don't know whether Fujitsu's software will let you do that. The only other common "image" format that supports multi-page (AFAIK) is mutli-page TIFF (.tiff or .tif). I get the impression that Fujitsu's software will also save as Word .doc format, but I think in that case it is probably doing OCR rather than putting page images in the .doc file. 

If you have to produce a .doc file, either with "just scan" or "scan and OCR", then it should open in Word 2011. If it doesn't, then either something is producing a .doc format for use by another Word Processing package (".doc" is, or was, used by other packages), or it is incorrect Word .doc format, or it is an old version of .doc (Word 2011 prevents some old versions from opening). I would actually be surprised by any of those things, but if that's what you are seeing, that's what you are seeing. You could look to see if .docx format is supported, and/or you could see if Mac OSX's TextEdit program can open the .doc documents.

horatio26 say:

Following up on Bernard's reply -- using your example:

You will use Conditional Formatting on cell A2 to set its text font color depending on the value in A1

Enter your text in A2

First, set the default font color of A2 to Black if not already that color

in cell A2, Select the Conditional Formatting tab (found in Home:Styles}

Select New Rule

Choose "Use a formuls to determine which cells to format" option

In the formula field type: =a1=100 where, say 100 is the value of (X) you want to test for.  the formula can be quite complex.  Note there is no "IF" in the formula.

Select the Format button and choose the font color - in your case choose a red color

Select OK

Now depending on your value in A1 being =100 or not the color of any text in A2 will be red or black

You can have a number of formatting rules for one cell to change its appearance depending on the value or values of other cells.

If there are multiple rules for one cell, the rules are evaluated from top to bottom, so you can control the appearance by changing the order of the rules using the Manage Rules ... option
