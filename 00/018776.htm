macropod say:

Is there a reason you're marking them to be ignored instead of adding them to your custom dictionary? Seems like a whole lot less work than running a macro afterwards to (a) re-set the document's spell-checked state; (b) compile a list of all errors found with a new spell-check; re-mark the words as no-proofing; then (c) output the list of 'errors' to another document so you can add them to the custom dictionary at a later time. AFAIK, all Word does with skipped spelling errors is to mark them as checked. That said, the following macro will re-check the document and output a list of all the spelling errors in a new document.

Sub GetSpellingErrors()
Application.ScreenUpdating = False
Dim i As Long, StrIgnored As String
StrIgnored = " "
With ActiveDocument
  With .Range
    .SpellingChecked = False
    For i = 1 To .SpellingErrors.Count
