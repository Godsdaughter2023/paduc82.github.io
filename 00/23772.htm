charles kenyon say:

I was using cross-references to bookmarks rather than to figures. I suppose that could make a difference.

The following macro should unlink only Cross-Reference fields in a document.

Sub UnlinkAllRef()
'   Unlink all Ref fields in a document, even if in headers/footers or textboxes
'   Based on code at http://www.gmayor.com/installing_macro.htm
'   Charles Kenyon
'   30 August 2018
'
    Dim oStory As range
    Dim oField As Field
    '
    For Each oStory In ActiveDocument.StoryRanges
        For Each oField In oStory.Fields
            If oField.Type = wdFieldRef Then oField.Unlink
        Next oField
        '
        If oStory.StoryType <> wdMainTextStory Then
            While Not (oStory.NextStoryRange Is Nothing)
            Set oStory = oStory.NextStoryRange
                For Each oField In oStory.Fields
                    If oField.Type = wdFieldRef Then oField.Unlink
                Next oField
            Wend
        End If
        '
    Next oStory
    '
    Set oStory = Nothing
    Set oField = Nothing
End Sub

Is it possible to auto-populate form fields between Word - Microsoft Community

gsyx say:

Is it possible to auto-populate form fields between Word documents?

I'm looking for a form solution and would like to know whether Office for Mac supports what I'd like to do. 

Can I create a Word document containing a form where the user inputs text in form fields, and the data auto-populates in fields in the other linked forms? If so, do all the forms have to be in the same Word file, or can they be separate files? If they have all be in the same file, can I create tabs for the separate pages/ forms in a similar fashion as worksheets in Excel? 

Thanks for your help

a. user say:

To do forms with any complexity in Word you need to use VBA. At that point, what is possible largely depends on your programming ability, ability to acquire VBA.

In Mac Word, there are arguably two types of form:
 a. the type that uses form fields in the Word document surface
 b. VBA Userforms (which would be much the same whether you did them in Word, Excel, or AFAIK, PowerPoint)

Assuming that you are really talking about (a), then

> and the data auto-populates in fields in the other linked forms?

Autopopulation only really exists for numeric fields. The results of other fields can be inserted using Word { REF } fields, but these are not fillable form fields. But with VBA, you can do stuff whenever a user tabs out of, or into, another form field, so you can generally do what you want reasonably easily.

> If so, do all the forms have to be in the same Word file

For autopopulation, they would have to be in the same Word document, yes.

> can I create tabs

There is no forms feature for that. (There is in the VBA Userforms world). You could probably emulate it by, e.g. using MACROBUTTON fields and some VBA to switch to the correct part of the document (On Mac Word, it would be nice if you could develop forms in the Notebook view where there's a nice UI mechanism for switching between sections, but I don't think you can "protect for forms" in that mode)
