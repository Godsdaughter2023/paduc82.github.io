Save large Word file - Microsoft Community

laoandy say:

Save large Word file

Most often when I save a large file (81.2 MB) I either get the message, "This is not a valid file name. . . . " and/or "You cannot save the file in use by another process. . . . " It tries to save the file with the name ".docx.." How can I fix this?
john mcghie say:
There is no reliable fix for this condition.

It indicates that your system has reached the limit of its resources and the Operating System is struggling to handle the demands placed on it.

If you ensure that your system is fully up-to-date by running "Check for Updates" from the Help menu in Word, it will happen less often.

If you LOGOUT each day when you finish work (instead of simply closing the lid or walking away) it will happen less often. (This gives the system a chance to clean up its temporary files.)

If you SHUTDOWN once a week (say, on Friday night) and thus do a cold re-start on Monday morning, it will happen less often.  (This gives the system a chance to do a full spring-clean of its memory and temporary storage.)

If you keep your individual files below 32 MB and do all of the above it will almost never happen (because the time the file is 'in transit' is shorter).

But there is no way to cure it entirely.

"Not a valid file name" means the system thinks this is the problem:
http://word.mvps.org/Mac/cantsavefilenames.html

If that really were the problem, it would happen with all files, and it was fixed in an update.  So it seems that the system is having such a struggle that it is not correctly reading the file name.

"In use by another process" means the OS has not released a previous lock on the file.  Before anything is allowed to write to a file, it must request a "lock" from the OS, to ensure nothing else can write at the same time.  When the write (save) has finished, the OS should release the lock, ready for the next save.  If things are running too slowly, it sometimes does not do that in time.  This can mean that Spotlight or Time Machine gets hold of the file and has not released it by the time you are ready to save.

If you go to Word>Preferences>Save and enable "Always create backup copy" that will reduce the number of opportunities for this to happen, because when you do that, Word actually writes a brand new file for each save, and re-names the original out of the way after the write completes.

But the best cure of all would be to ask Santa Clause for a much larger, faster computer.  The latest 27" iMac should fit the bill quite nicely, if you bring it up to 16 GB of RAM.  Personally I use a Mac Pro for this kind of thing: with files that large, you really are in workstation-class territory.

Hope this helps

I don’t believe anything has been fixed. The bug is still there. The problem is subtle. I saw this with Microsoft Outlook 2011 on a Mac running OS X 10.9.1. Outlook sends mail as “multipart/alternative”, with two alternative forms:

Content-type: text/plain; charset="ISO-8859-1"
and
Content-type: text/html; charset="ISO-8859-1"

Most modern mail clients display the text/html variant, which renders okay and hides the problem. If the receiver switches to text/plain view, the problem becomes visible. In the text/plain variant the apostrophe is wrongly encoded as character B9, and character B9 is not apostrophe in any character set I know, and certainly not in ISO-8859-1.

The problem also becomes visible on mailing lists where the mailing list forwarder is stripping out the text/html and only forwarding the text/plain, so people then see the garbage variant that’s usually not shown.

The workaround described by Peter Yee works: Change Outlook to use UTF-8, and you avoid the bug in its ISO-8859-1 encoder.

shane devenshire say:

As Ron says you need to give us details:

Suppose your two cells are A1 and B1 here's what you should have

To conditionally format your cell(s):

In 2003:
1. Select the cells you want to format A1
2. Choose Format, Conditional Formatting
3. Choose Cell Value is  from the first drop down
4. Pick Greater than in the second

4b. In the third box enter =B1

5. Click the Format button
6. Choose a color on the Patterns tab (or any available option)
7. Click OK one.
8. Click Add
9. For Condition 2, from the first drop down pick Cell Value Is
10. In the second box choose less than or equal to

10b. In the third box enter =B1

11. Click the Format button
12.  Choose a color and click OK twice.

Now if you type S into A1 and Z into B1 A1 will turn Green

If you type Z into A1 and S into B1 A1 will turn Red

If you type 1 into A1 and 2 into B1 A1 will turn Green

If you type 2 into A1 and 1 into B1 A1 will turn Red

If you type A into A1 and 50000000000 into B1  A1 will turn Red

If you type 50000000000 into A1 and R into B1 A1 will turn Green

Text is always interpreted as larger than numbers.
