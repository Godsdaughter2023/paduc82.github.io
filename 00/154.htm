rich michaels - mvp (greatcirclelearning.com) say:
To answer your question about changing picture links in one operation. It requires you writing a custom macro or getting your hands on an add-in product called zAPPs-docTools. It's got a 7-day free trial period so if you need it for just this one thing you can download it and fix up the links.

Full disclosure... I wrote the product. You can get from our website which is listed in the footer of this note.

More on pictures inserted into documents...

There are three types, Embedded, Linked, and Linked & Saved. 

An embedded picture is the default. Its advantage is that it is always there in your document so you can easily share the document with others. The disadvantage is document file size and not easily updated. 

Linked pictures are not stored in the document, they are stored separately either in the same folder as the document or a different folder from the document. Linking to pictures versus embedding them keeps the document file size smaller and the pictures can be easily changed/updated. However, in order to share the document and for the recipient to see the pictures the whole folder must be sent. If the pictures are stored in a separate folder, the linked path is an absolute path to the folder that contains the pictures. Absolute path links must be changed if you are going to share the document.  A document with relative linked pictures, meaning pictures stored in the exact same folder as the document, can easily be shared as long as the entire folder is forwarded.

Linked & Saved pictures provide the best and bad of both embedded and linked picture options. The embedded version of the picture is the "default" image if Word cannot find the picture in the absolute or relative link that is associated with it, the default image is shown. Since there is an embedded picture the document file size will be larger. However, since there is also a link, any updates to the image will be used when available.

Hope this helps

shelleeeee say:

Advanced Find and Replace Q

I have created tests with hundreds of questions that I need to upload into Blackboard. In multiple choice questions, Blackboard will recognize the correct answer IF there is an asterisk in front of the letter or number choice.

Ex:

C. Orange

Failing that, is there an efficient way that I can bold all answers with asterisks at the end, which I can then use find and replace to recognize and add an asterisk to the front of those answers?

robert19 say:

Your screen capture explained it all! We had assumed that your test “lines” were ending with paragraph end marks. With the exception of the last item, they are all actually ending with “line breaks”.

You need to use the following wild card search/replace:

Find What:

([A-Z].[!^l^13\*]{1,})(\*)([^l^13])

Replace With:

\2\1\3

It is working on the following test strings where all the “lines” end in “line breaks” and the last item ends in “paragraph end mark”:

A. Supine
B. Prone*
*C. Dorsal
D. Posterior
E. Anterior*

The results are as follows:

A. Supine
*B. Prone
*C. Dorsal
D. Posterior
*E. Anterior

Any asterisk character that Word finds at the end of any of your test lines will be moved to the front of the line.

Again, depending on your system locale, you might need to change {1,} to {1;} in the above Find What expression.

Here are procedures to correct the WMI repository.

 

When running this in Vista7, you must run the procedures as an administrator, elevated.

Method A

Start>Run>services.msc [enter]

Scroll down to Windows Management Instrumentation and double-click it.

Now click on the "Pause" button. Leave that window open and double-click

My Computer. Navigate to %systemroot%\Windows\System32\wbem
(where %systemroot% is the drive where XP is installed). Delete the Repository
folder and *only* the Repository folder. Now go back to the WMI service
window you left open and restart the service.

This will rebuild the Repository and hopefully straighten out the
incorrect entries for all your duplicates.

In order to see the Windows files, you may need to unhide them:

Make sure you are able to see all hidden files and extensions (View tab in Folder Options).

Check "Display the contents of system folders".

Check "Show hidden files and folders".

Uncheck "Hide protected operating system files" and click "OK" to the dialog box.

 

Method B from MVP Torgeir Bakken (more elegant)

Open a command window (Start/Run --> cmd.exe) and run the following commands:

net stop winmgmt

cd /d %windir%\system32\wbem

ren repository repository.old

(or delete it using the command "rd /s repository" instead of the ren command)

net start winmgmt

It may take a minute or so to complete while WMI rebuilds the database.

 

Alternatively, see this blog post: http://ashwinrayaprolu.wordpress.com/2011/05/13/repairing-fixing-wmi-repository/
