critterlescp say:

How would I add multiple CSV files into the same Access Table?

Every day I am going to receive 1 CSV file with the same structure as the day before but with new data. There are probably multiple ways to do this so I wanted to ask in the forum. How would you suggest I import these daily files into the same table? Ultimately I would like to automate this procedure so it starts every day at a set time.

hansv mvp say:

If the Access table doesn't exist already, create it by importing a CSV file with the option "Import the source data into a new table in the current database" in the Get External Date - Text File dialog. From then on, append the data to the existing table by selecting the option "Append a copy of the records to the table" and selecting the table.

The first time you do this, go through all the steps of the Import Text Wizard, and just before you click Finish, click Advanced...

You should see a summary of the choices that you made. Click Save As... to save the import specification. Make a note of the name that you give to the specification.

Next time, you can click Advanced... immediately, then click Specs... to load the import specification that you saved. You can then click Finish without having to go through the intermediate steps.

When this works to your satisfaction, you can create a VBA procedure (or a macro) to automate the import. The VBA code looks like this:

    Dim strFile As String
    strFile = "C:\MyPath\MyFile.csv"
    DoCmd.TransferText _
        SpecificationName:="MySpec", _
        TableName:="MyTable", _
        FileName:=strFile

where C:\MyPath\MyFile.csv is the path and filename of the text file, MySpec is the name of the import specification that you saved, and MyTable is the name of the target table.

jay freedman say:

You can't do what you ask with just text boxes.

You can do it with a table. If you click Table Tools > Layout > AutoFit and choose Fixed Column Width, then a picture that's wider than the table cell will be resized to fit the cell's width. If you also want to constrain the picture's maximum height, click Table Tools > Layout > Properties > Row and set the row height, changing "At Least" to "Exactly".

Note that pictures that are smaller than the cell width (or height) will not be resized larger to fit the cell.

If the path is set to somewhere that doesn’t exist the computer will display error messages like the ones you are receiving. Follow these steps to check for and fix this problem:

 

Modifying REGISTRY settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of REGISTRY settings can be solved. Modifications of these settings are at your own risk.

 

·         Click Start

·         Click Computer

·         In the address bar, paste this location: %userprofile%

·         Press Enter on your keyboard

·         Make sure you have a Favorites folder listed in the %userprofile% folder (from your original post this should be the location of C:\Users\Jill\)

·         If there isn’t a Favorites folder listed, create a new folder and name it Favorites

·         Press Windows key + R on your keyboard to open the run dialog

·         In the Open: field, type regedit

·         Click OK

·         Click Yes on the User Account Control (if prompted)

·         Navigate to HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Shell Folders

·         Right click Favorites

·         Click Modify…

·         In the Value data: field, paste %userprofile%\Favorites

·         Click OK

·         Exit Registry Editor
