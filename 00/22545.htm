doug robbins - mvp office apps & services (word) say:

You would use the following field construction

{ IF { REF bookmarkname } = "some text" "True Result"  "False Result" }

where each pair of field delimiters is inserted by using CTRL+F9 and you use ALT+F9 to toggle off their display.  It will also be necessary to select the field construction and press F9 to cause the field to be updated to show the result.

However, basing an If...then...Else field construction on the content of a bookmark can be very risky for the following reasons:

  1. When inserting text into the bookmark, it is very easy to destroy the bookmark
  2. At lease for the True condition of the If...then...Else, the text that is entered must match the condition exactly.

If you can tell us what you are trying to achieve, we might be able to suggest a more robust method of doing it

Dim Doc As Word.Document
Dim objWOrd As Word.Application

    .Bookmarks("GRDate").Range.InsertAfter Format(Date, "mmmm d, yyyy")

    If rs!CollectionMode = "CCARD" Then
        .Bookmarks("CollectionMode").Range.InsertAfter "your payment was rejected by your Master Card...."

        .Bookmarks("CollectionMode").Range.InsertAfter "your check was returned to us ...."
    End If
    'Then for the text to go in the footer of the document, use
    If .Bookmarks.Exists("Footer") Then
        .Bookmarks("Footer").Range.InsertAfter ISMPageFooter
    End If
    If .Bookmarks.Exists("LetterName") Then
        .Bookmarks("LetterName").Range.InsertAfter strFooter
    End If
    SaveAs2 "F:\DocStorage\LETTERS\ONLINE\NSF1.Doc"
    .Close
End With

How to Encrypt-Decrypt database programatically

Does anyone know how to programatically Encrypt(Password-protect) and  Decrypt (remove password protection) for a database?

christw1 say:

I found the solution. Here is the code to decrypt/encrypt a database programatically:

 

Function DecryptDb()        'Decrypt an existing encrypted DB
    Dim ws As Workspace
    Dim db As Database
    Set ws = DBEngine.Workspaces(0)
    Set db = ws.OpenDatabase("MyDb", True, False, "MS Access;PWD="test")
    db.NewPassword "test", ""
    db.Close    
    DecryptDb = True
End Function

 

Function EncryptDb()        'Encrypt an existing non-encrypted DB
   Dim ws As Workspace
   Dim db As Database
   Set ws = DBEngine.Workspaces(0)
   Set db = ws.OpenDatabase("MyDb", True, False)
   db.NewPassword "", "test"
   db.Close    
   EncryptDb = True
End Function

Here is a bunch of reading material on Windows 8 backup options, including Refresh


Win 8 Total Backup System: File History + RecImg + Refresh + SlimImage

Win 8 has refined your backup and restore options. Before Win 8 you could create an Image Copy backup of the full drive (or selected directories), but it was hard to separate backups of the OS, Applications and personal Data unless you created a separate data partition.  I still think it is a good option.

 

Out of the box, MS has been advertising the “Refresh” and “Reset” options. Personally, I think they missed a valuable selling point by not mentioning RecImg.EXE in conjunction with Refresh. It removes my main concern/objection to Refresh.

 

Reset is equivalent to using the “Recovery Partition” on previous versions of Windows. The factory would create a “Recovery Partition” that you could use to “reset the computer to factory fresh”. It would typically include all of the factory “crapware” and it would wipe out all of your data files and installed applications.  A less than desirable outcome but an appropriate reaction to a bad malware infection.  The “recovery partition” type of restore typically had to be accessed at boot time while Reset can be accessed from within Windows 8.

 

Refresh is roughly equivalent to an image copy backup.  A Reset will restore a backup of the OS and Metro applications. It will wipe out “legacy” / x86 desktop type applications BUT it will not affect your personal data files, which is much better than a Reset.  But there is still that issue of wiping out your installed desktop applications. I do not have documented confirmation, but I’m guessing that Metro apps are treated like personal data and those directories/folders are simply not reset back to the old condition.

 

RecImg.EXE is a backup tool that ships with Windows 8. It creates the backup “image” that is used by the Refresh tool. This image that was created at the factory is why using Refresh will wipe out installed desktop programs in the “C:\Program files” and “C:\Program Files (x86)” folders because they haven’t been backed up. However, using RecImg.EXE you can create new images that will include recently installed desktop applications.

 

RecImg Manager is a third party tool the provides a GUI to access the RecImg.EXE command line tool and backup files.

 

File History is a new Win 8 feature. It is an extension to the “Previous Versions” and “Volume Shadow Copy” features. File History continuously monitors files stored in Libraries, Desktop, Contacts, and Favorites folders. It runs on a predetermined schedule, and when it detects changes in any file, it makes a backup of that file to an External drive. It isn’t enabled by default.
