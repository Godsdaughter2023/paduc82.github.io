size a watermark top fit into 6 sections on an 8x11 sheet of paper

daniel g. grau say:
Labels are just fixed-size tables. Watermarks are placed in the document's Header, so in order to reuse it you'd need to go to the Header, make the watermark smaller, replicate it 5 times, and painfully reposition each watermark so it will fall exactly behind a cell (label). That's too inexact.

I tried using an identical floating table in the Header, but Word won't allow the document's main text (i.e., the labels) to overlap. I think thus this is the easiest method. We'll work with just one table cell (one label), then duplicate it:

  1. Create your labels
  2. Create a watermark
  3. Go to the document's header, resize the watermark to make it a lot smaller and Cut it
  4. Exit the header, click on the first cell and paste the watermark
  5. Position it on the cell (don't pay attention to the label's text being pushed around, just position the watermark where you need it).
  6. Once it's positioned, make it "Behind text" (the "behind text" attribute is not required when the watermark is in the header, so you need to add it). Once the watermark is behind the text, it will be very hard to reselect it to correct its position. (You need to add an extra tool that Word does not include by default.)
  7. Select the cell (Table > Select > Cell) and Copy or Cut
  8. Select the whole table and Paste

doug robbins - mvp office apps & services (word) say:

See the following page of Greg Maxey's website (the same code can be used for a combobox

http://gregmaxey.mvps.org/Populate_UserForm_ListBox.htm

Or, if your form uses formfields in a document that is protected for filling in forms,

Dim xlapp As Object
Dim xlbook As Object
Dim xlsheet As Object
Dim myarray As Variant
On Error Resume Next
Set xlapp = GetObject(, "Excel.Application")
If Err Then
    bstartApp = True
    Set xlapp = CreateObject("Excel.Application")
End If
On Error GoTo 0
Set xlbook = xlapp.Workbooks.Open("[path\filename of workbook")
Set xlsheet = xlbook.Worksheets(1)
myarray = xlsheet.Range("A1").CurrentRegion.Value
If bstartApp = True Then
    xlapp.Quit
End If
Set xlapp = Nothing
Set xlbook = Nothing
Set xlsheet = Nothing
For i = 1 To UBound(myarray)
    ActiveDocument.FormFields("DropDown1").DropDown.ListEntries.Add Name:=myarray(i)
Next i

Changes to outlook "sweep" function - Microsoft Community

jamesbankert say:

Changes to outlook "sweep" function

I just tried to use the sweep function in outlook and discovered it has changed.  The dialog box now gives 4 options that are all variations of "delete."  Before, I was able to move all messages from a sender to a designated folder, and also set it to move future emails from the sender to that folder.  This was great for keeping the inbox organized.  But the new dialog box makes it seem that this cannot be done anymore.  Am i correct about this, or is there still a way to use the very useful sweep archiving features as before?
xjiox say:
I just tried to use the sweep function in outlook and discovered it has changed.  The dialog box now gives 4 options that are all variations of "delete."  Before, I was able to move all messages from a sender to a designated folder, and also set it to move future emails from the sender to that folder.  This was great for keeping the inbox organized.  But the new dialog box makes it seem that this cannot be done anymore.  Am i correct about this, or is there still a way to use the very useful sweep archiving features as before?

I think Microsoft is further defining and isolating what the Sweep service will mean, so it has moved the "Move all mail from ..." feature to the "Move to" function, so when viewing an email from a sender whose emails you want to have automatically bypass the inbox and sorted into a folder, do the following:

  1. Click "Move to (or press "v")"
  2. Click "Move all mail from ..."
  3. Then choose from:
    • Move from Inbox
    • Move from Inbox and move all future email
    • Always keep the latest one, move the rest
    • Always move email older than 10 days

Microsoft should have been clear about this change or at least made contextual prompts appear when viewing your email once the change went live so we knew what to do or were at least aware.  I didn't receive an email about this change, either.  I hope this helps.
