Excel NETWORKDAYS.INT - Microsoft Community

ac montalvo say:

Excel NETWORKDAYS.INT

I am using the networkdays.int function to include Saturdays in the work week.  The issue I'm having is that when I changed from not including Saturdays to including Saturdays in the work week the formula displays a number "1" when there are no dates entered in the range to compare.  Even if I enter a NULL date it still displays a 1.  It does not affect the data, it is as if it has a null value.  I just don't like it for aesthetic reasons. 

The number 11 in the formula (second to last entry represents "only Sunday off") if I change to 1 which is Saturday and Sunday off the "1's" go back to zero.  If I select any other combination of days off whether it is one or two days the "1's" will display. Any help would be appreciated.

hansv mvp say:

If the start date and end date are the same, NETWORKDAYS and NETWORKDAYS.INT return 1 if that date is a working day, and 0 if it is a non-working day.

An empty cell is equivalent to 0, and 0 happens to correspond to a Saturday in Excel's date system. So an empty cell counts as a non-working day for the old NETWORKDAYS function, and for NETWORKDAYS.INT with 1 as third argument, and hence they return 0 if the start date and end date are empty.

But with 11 as third argument, an empty cell counts as a working day and hence NETWORKDAYS.INT returns 1.

You could get around this by using

=IF(OR(@[Start Date]="", @[End Date]=""), "", NETWORKDAYS.INT(@[Start Date], @[End Date], 11, lstHolidays))
