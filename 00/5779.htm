Setting up a default footer for all documents? - Microsoft Community

goatberg say:

Setting up a default footer for all documents?

Hi.  How can I setup a default footer for all documents that are created?  Also if I can do this in Word, is it possible to also do this in other Office applications and would the steps involved be any different?

This is for Office 2007.

cindy meister say:

Hi Goatberg

The steps for other Office applications would be different, yes.

Word creates each document from a template. By default, it uses the template Normal.dotm. Anything you put in the file (template) Normal.dotm will appear on any new document you create using Ctrl+N or the "Blank template" options in File/New. If you use another template - one not created from this Normal.dotm - it will not contain the footer.

By default, Normal.dotm is stored in your personal "Templates" folder. On Windows XP, I think this was something like C:\Users\[Profile name]\AppData\Office\Templates (or it might be Application Data rather than AppData). In some configurations the template and folder have the "hidden" attribute, so you may need to activate the Option in the Windows Explorer to show hidden files and folders.

When you find the template, right-click it and choose the command "Open". (If you double-click a template that creates a new document rather than opening the template.) Make the changes you require, then SAVE and close.

Please note that a common trouble-shooting step for the Word application is to rename/remove the Normal.dotm template. That's because this template file can become damaged (corrupt), so it's a good idea to save a back-up copy if it contains things you don't want to lose.

An alternate approach would be to create a different template and use that for new documents. But it wouldn't be as convenient as the default Normal.dotm...

Cindy Meister, VSTO/Word MVP

Office 2010 Upgrade, 32 or 64 bit? - http://devonenote.com/2010/05/office-2010-upgrade-32-or-64-bit/

14 May, 2010 | Category: Office | By: emma

Office 2010 is the first release of Office that contains 64 bit edition. It’s recommended to install the 32 bit version Office 2010 due to compatibility issues however if you still want to try 64 bit version keep these in mind.

·         32 bit version works on both 32 bit and 64 bit windows operation systems. But 64 bit version Office 2010 must be installed on 64 bit windows operation systems.

  • You can’t upgrade from 32 bit Office 2007 to 64 bit Office 2010 directly, you have to uninstall existing 32 bit Office products prior to install 64 bit Office 2010. (It’s okay to upgrade Office 2007 to 32 bit Office 2010 on either 32 bit or 64 bit operation systems)
  • You can’t run 64bit and 32bit office products side by side. For example, you can’t have side by side installation of 32 bit OneNote 2010 and 64 bit Outlook 2010
  • Some of 32bit Office Add-Ins might not work on 64 bit Office 2010.

 

Choose the 32-bit or 64-bit version of Microsoft Office - http://office.microsoft.com/en-us/word-help/choose-the-32-bit-or-64-bit-version-of-microsoft-office-HA010369476.aspx?CTT=1

64-bit Office 2010 product backup is available at http://www.office.com/backup  . However, we recommend most users install the 32-bit version of Office 2010. The 64-bit installation of Microsoft Office 2010 products are for users who commonly use very large documents or data sets and need Office 2010 programs to access greater than 2GB of memory. There are some technical limitations with the 64-bit version.

 

Compatibility Between the 32-bit and 64-bit Versions of Office 2010 - http://msdn.microsoft.com/en-us/library/ee691831%28office.14%29.aspx

 

64-bit editions of Office 2010 - http://technet.microsoft.com/en-us/library/ee681792.aspx

The recommendations for which edition of Office 2010 to install are as follows:

·         If users in your organization depend on existing extensions to Office, such as ActiveX controls, third-party add-ins, in-house solutions built on previous versions of Office, or 32-bit versions of programs that interface directly with Office, we recommend that you install 32-bit Office 2010 (the default installation) on computers that are running both 32-bit and 64-bit supported Windows operating systems.

  • If some users in your organization are Excel expert users who work with Excel spreadsheets that are larger than 2 gigabytes (GB), they can install the 64-bit edition of Office 2010. In addition, if you have in-house solution developers, we recommend that those developers have access to the 64-bit edition of Office 2010 so that they can test and update your in-house solutions on the 64-bit edition of Office 2010.
  • Side-by-side installations   No support is provided for side-by-side installations of 64-bit and 32-bit editions of Office, including across applications. For example, there is no support for side-by-side installations of the 2007 Microsoft Office system 32-bit with Office 2010 64-bit, or for Microsoft SharePoint Workspace 2010 64-bit and Microsoft Excel 2010 32-bit.
