brian tillman [outlook mvp 2007-2019] say:

A POP client's default behavior is to download messages from the Inbox and then send a Delete command to the server so that those messages are deleted from the Inbox. This keeps the server disk space to a minimum and keeps all messages on your local PC.

Many peope didn't like this, so some mail providers, like Microsoft and Google, ignore the Delete command the client sends. Outlook.com's behavior is to move the messages to a subfolder of the Deleted Items folder, named POP. This way, the messages are removed from the Inbox as normal, but are still retained in the mailbox do you can always go back to find them if you need to.

If you log into https://outlook.live.com/owa/?path=/options/popandimap, you'll see that when POP is enabled for your Outlook.com mailbox, there is also a setting that controls whether Outlook.com will move your messages to the POP subfolder or obey the client's Delete command and remove them altogether.

For a 95% confidence interval, instead of hardcoding 1.96, an approximation, I would use -NORMSINV((1-C1)/2), where C1 is the confidence level (95%).
