Introduction to the Office Macro Editor, Part 2 - Microsoft Community

jay freedman say:

Introduction to the Office Macro Editor, Part 2

Technical Level : Basic

Summary
Know the parts of the Visual Basic for Applications (VBA) macro editor in Microsoft Office programs: Access, Excel, Outlook, PowerPoint, Publisher, and Word. Applies to Office 2003, Office 2007, Office 2010, and Office 2013/Office 365 (not Office Online or Office RT).
Details

The first part of this wiki article, describing the Project Explorer, the Properties pane, and the code window, is at Introduction to the Office Macro Editor, Part 1.

Menus and toolbars

Most of the menus in the macro editor are self-explanatory. Some of the commands may be unfamiliar:

  • File -> Export File saves the item selected in the Project Explorer. The code in a module or class module is saved as a *.bas file. The code of a userform is saved as a *.frm file, and the visual design of the form is saved in a *.frx file. You can edit the suggested names of the files and choose the folder in which to save them.
  • File -> Import File reads a *.bas or *.frm file and adds the code or userform to the selected project in the Project Explorer. You can export a module or userform from one template and import it into another template.
  • View -> Tab Order lets you choose the order in which cursor moves through the controls on a userform as you press the Tab key.
  • The items on the Format menu apply only to controls on a userform.
  • The items on the Debug menu help you to understand how an existing macro works or to find out why it’s failing. The items related to the Locals window and to breakpoints are especially useful. The commands for controlling the execution sequence can also be found on the Debug toolbar.
  • Tools -> References opens a dialog box containing a list of the programs and other files that offer additional tools in Visual Basic for Applications. For example, you can write a macro in Excel that opens and edits a document in Word; to use the Word objects in your Excel macro, you must open the References dialog box and click the check box next to the Microsoft Word Object Library item.
  • Tools -> Project Properties enables you to view and change a few properties related to the project that is selected in the Project Explorer pane. The Protection tab of the dialog box lets you lock the project and optionally assign a password, so other users can’t view the code of the macros and userforms. You might want to do this if the macros contain passwords for opening restricted documents.

Object Browser

The View -> Object  Browser menu item opens a window in which you can view all the objects in the Visual Basic object model of the application, of Visual Basic itself, and of any extensions that you choose in the Tools -> References dialog box. To search for a term, type any part of it in the text box to the left of the Search icon   and click the icon.

Userform design window

Each userform consists of a custom dialog box that you create, and the code that makes the userform work. The macro editor displays the userform’s code in the same kind of code window as other macros, and it shows the userform’s visible surface in a separate design window.

In the design window, the userform surface appears at a default size, with a grid of dots to help you position controls that you add. You can make the form larger or smaller by clicking on it and dragging its sides or corners, as you would with a picture in a document. The dots won’t appear when the userform executes.

When you work in the design window, you’ll also see a floating Toolbox containing buttons for the kinds of controls you can put on the userform—labels, text boxes, dropdown lists, command buttons, and so on. Click a button in the Toolbox and then drag the mouse diagonally in the userform surface to the approximate size you want the control to be. You can then resize and reposition the control, and use the items on the Format menu to get the best appearance.

As you add each control, VBA assigns a default name consisting of the control’s type and a sequence number, as in Textbox1. You should use the (Name) box in the Properties pane to give the control a meaningful name related to its purpose, such as txtLastName. This will help you to use the correct references in the userform’s code. For controls that have static text, such as labels and command buttons, also change the Caption property.

The quickest way to see the code related to a control on a userform is to double-click the control. For example, clicking the command button named cmdOK opens the code window with the cursor in the cmdOK_Click procedure. (If the control doesn’t have any related code yet, VBA will create an empty subroutine for the control’s default event, usually the Click event.)

Other ways to get to the code window include clicking the View -> Code menu item (shortcut F7), or right-clicking the userform’s icon in the Project Explorer and clicking View Code.

Immediate window

The Immediate window enables you to run one or a few VBA commands without having to construct a complete macro. Click View > Immediate Window (shortcut Ctrl+G) to open the pane at the bottom of the code window, type a line of code, and press Enter to execute it.

The question mark at the start of the line is an abbreviation for the Debug.Print command. The result of the Print command appears on the next line in the Immediate window. If the code you execute changes the content or formatting of an open document, it takes effect immediately.

You can enter several commands in a single line of code by separating them with colons.

Locals window

The Locals window displays the values of all the variables in the currently executing macro, which is useful for debugging. Click View -> Locals Window to open the pane at the bottom of the code window. Then use the commands on the Debug menu or toolbar to execute part of the macro and stop at a point of interest. The values at that point appear in the Locals window.

Watch window

The Watch window displays values of variables, similar to the Locals window, but you can choose individual variables and some other expressions to display.

Sub splitter()
' splitter Macro
' Macro created by Doug Robbins to save each letter created by a mailmerge
' as a separate file, retaining the header and footer information.
Dim i As Long, Source As Document, Target As Document, Letter As Range
Set Source = ActiveDocument
For i = 1 To Source.Sections.Count
    Set Letter = Source.Sections(i).Range
    Set Target = Documents.Add
    Target.Range = Letter
    Target.Sections(2).PageSetup.SectionStart = wdSectionContinuous
    Target.SaveAs FileName:="Letter" & i
    Target.Close
Next i
End Sub
