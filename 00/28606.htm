If you want to merge tenant B to tenant A, you need to cancel the tenant B’s plan firstly and then buy more licenses of the tenant A’s plan for users who use the tenant B domain. a. user say:

If your computer has Windows 7 or 8.1, and you signed up for the Insider Program, you got an update to prepare your computer for the Tech Preview. If you uninstall the update, you will stop getting offered Tech Preview.

 

To uninstall the update, go to Add or remove programs, and remove KB2990214 (if you’re on Windows 7) or KB3014460 (if you’re on Windows 8.1).

 

If the update doesn’t show up there, press the Windows key + R, and type the following:

 

(for Windows 7)

wusa /uninstall /kb:2990214

 

(for Windows 8.1)

wusa /uninstall /kb:3014460

 

…and then press Enter. This should uninstall the update.

I do my best to be patient, to help out Microsoft, but engineers, you need to indicate SOME progress on the counter, even if it's to only 1 or 2%. I have a fast hard drive (RAID10 array of 4 Mushkin SSD's), a fast CPU, and 8 gigabytes of DDR3 memory. Data collection should not take a long time.

By the way, here is the information on DPC_WATCHDOG_VIOLATION:

http://msdn.microsoft.com/en-us/library/windows/hardware/jj154556(v=vs.85).aspx

Bug Check 0x133 DPC_WATCHDOG_VIOLATION

The DPC_WATCHDOG_VIOLATION bug check has a value of 0x00000133. This bug check indicates that the DPC watchdog executed, either because it detected a single long-running deferred procedure call (DPC), or because the system spent a prolonged time at an interrupt request level (IRQL) of DISPATCH_LEVEL or above. The value of Parameter 1 indicates whether a single DPC exceeded a timeout, or whether the system cumulatively spent an extended period of time at IRQL DISPATCH_LEVEL or above.

More info at link above.
