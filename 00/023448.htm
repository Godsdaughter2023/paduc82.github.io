Though you believe the problem is not your printer, always check for an updated printer driver. Publisher is more picky about drivers than other programs. Try an older driver if the newest one does not work.

A good workaround can be to convert your Publisher file to a PDF and print that.

As per the files going corrupt, where are the files stored? It is best to open from and save to a local hard drive to prevent corruption. If you want to backup your files on a network, USB, or any external or removable drive then copy the file to and from...don't save to or open from.

Adopt a backup protocol with your files. Save often and do a Save As a new file name before or after each significant edit. If I know I am going to make a significant edit to a file I will do the Save As a new file name immediately after opening the file. That way if the new version becomes corrupt I can go back to the last copy. Remember that it is not a matter of IF a file will go corrupt or drive will fail, it is a matter of WHEN.

If you have a problem with a file remember the %temp% trick. Type %temp% in the address bar in File Explorer or Run, find a recent file with the pub prefix, change the .tmp extension to .pub, try opening the file.This approach works best if you try to recover the file immediately. The more time passes, the less likely it will work.

talking moose say:
You need to try to restore the entire Microsoft User Data folder not just parts of it from an identity.

Look for this folder and copy it from your old drive:

/Users/<yourname>/Documents/Microsoft User Data

After it's copied Launch Outlook with the Option key pressed. This will launch the Database Utility application. Select an identity (usually called "Main Identity") and use the menu button at the bottom of the window to set it as default. Quit the Database Utility and launch Outlook normally.

Hope this helps!
