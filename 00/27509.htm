bob jones aka: cybertaz say:

Just going by the sequence as you wrote it in your message: You need to Select All before opening the Language dialog. If the dialog is already open Cmd+A does nothing. The procedure needs to be:

Use Cmd+A first, then go to the Language dialog to make the changes.

Also, the version you cited is several moths behind. Version 15.34 is the present update level. That very well could be having an influence.

Please disregard reference to setting the Default Language - That has no bearing whatsoever on existing documents. It determines the language used by default for new documents you create. Other than that, the dialog is used to change the language of selected content in a document to something other than the attribute it currently carries.

Similarly, standard Spelling & Grammar checking is independent of the 'as you type' settings in the Preferences pane. IOW, the full Spelling & Grammar check does its thing regardless of whether the automatic features are being used.

Another possibility is that the document you're referring to may be corrupt. Are you experiencing similar issues with other documents?
