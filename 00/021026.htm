Splitting Cells within a Column - Microsoft Community

trisha0714 say:

Splitting Cells within a Column

 I have information in a column that  I would like to split in 2 cells without adding another column.  For example:

ABCD  1234
EFGH   5678

 

  And would like to split the cells between ABCD and 1234 within the column without adding a column.

norman david jones say:

Hi Trisha,

Try something like:

  • Alt+F11 to open the VBA editor
  • Alt+IM to insert a new code module
  • In the new module, paste the following code 

'=========>>
Option Explicit

'--------->>
Public Sub Tester()
    Dim WB As Workbook
    Dim SH As Worksheet
    Dim Rng As Range
    Dim arrSplit As Variant
    Dim arrIn As Variant, arrOut() As Variant
    Dim LRow As Long
    Dim i As Long, j As Long
    Const mySheet As String = "Sheet1"                 '<<=== Modify


    Set WB = ThisWorkbook
    Set SH = WB.Sheets(mySheet)

    With SH
        LRow = LastRow(SH, .Columns("A:A"))
        Set Rng = .Range("A2:A" & LRow)
    End With

    arrIn = Rng.Value
    ReDim arrOut(1 To UBound(arrIn, 1) * 2)
    For i = 1 To UBound(arrIn, 1)
        j = j + 1
        arrSplit = Split(arrIn(i, 1), " ")
        arrOut(j) = arrSplit(0)
        arrOut(j + 1) = arrSplit(1)
        j = j + 1
    Next i
    Rng.Resize(UBound(arrOut)).Value = Application.Transpose(arrOut)
End Sub

'--------->>
Public Function LastRow(SH As Worksheet, _
                        Optional Rng As Range)
    If Rng Is Nothing Then
        Set Rng = SH.Cells
    End If

    On Error Resume Next
    LastRow = Rng.Find(What:="*", _
                       after:=Rng.Cells(1), _
                       Lookat:=xlPart, _
                       LookIn:=xlFormulas, _
                       SearchOrder:=xlByRows, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Row
    On Error GoTo 0
End Function
'<<=========

  • Alt+Q to close the VBA editor and return to Excel
  • Alt+F8 to open the macro window
  • Select Tester | Run

 Change Sheet1 to the name of your sheet.

dale howard [mvp] say:

Brent and all --

I saw this issue many years ago when there were hundreds of projects connected to a shared resource pool file.  The calendar duplication issue was caused by a corrupted shared resource pool file.  If that is the source of your problem, the way to resolve it would be as follows:

  1. Open the shared resource pool file Read/Write.
  2. Open every one of the sharing projects.
  3. Disconnect every sharing project and disconnect it from the shared resource pool file.
  4. In the Organizer dialog for every project, delete the duplicate calendars.
  5. Save and close every sharing project.
  6. Save and close the shared resource pool file.
  7. Open a new blank project and save it using a name that would indicate it is a shared resource pool file.
  8. Individually open each project and connect it to the new blank project as a sharing project.
  9. Save and close each project.
  10. Save and close the new shared resource pool file.

After completing these steps, open one of the sharing projects.  Do you still see the duplicate calendars?  Please let us know and we will try to help you.
