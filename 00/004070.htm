OK so if you are still getting an error it could be because there does not appear to be a space before one of the _ at the end of the second bold line...

        ...ersion12).CreatePivotTable_
becomes
        ...ersion12).CreatePivotTable _

This is quite a subtle change, but important. The space then underscore tells the application that the code line continues on next line

Screenupdating is a function that can be used to improve the speed of macros as the application doesn't update the screen during the running of the process eg

Sub someCode()
Application.ScreenUpdating = False
       ' put your code here
Application.ScreenUpdating = True
End Sub

So it is something that you can add, and will actually work with the code you have too...
