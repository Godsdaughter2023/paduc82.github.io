irixion say:

DNS lookup has nothing to do with this.

Using a lot of switches can present issues, because it's difficult to narrow what the issue can be. I do suggest grabbing a Windows based laptop, and setting up your virtual WiFi adaptor. Enable Internet Connection Sharing, and use your laptop as a "router" essentially. If everything checks out, it's a hickup somewhere on your network.

Edit:

If it is a network hickup, there's not much I can do without knowing the specific structure. Though, you can use a utility like WallWatcher to sniff packets to see where exactly they're dropping off to narrow down the possibilities. This can be helpful, especially if you have a complicated and/or large network to deal with. Depending on the quality/length of your RJ45 cables, there could be some kinks or signal degradation somewhere, despite a link being established at any given speed. You could also test large file transfers in Windows between Windows to Windows networks to see if there's any issues with that, though keep in mind SMB is pretty limited, especially when using different versions (Windows 10, say vs Windows 7, which uses an older version of SMB protocol).

ken blake say:

Please note that space on the hard drive is not called "memory." The term "memory" is used to refer to the place where the computer holds the program and data it is working with at the moment. It's normally random access memory (RAM), but that can be extended with the page or swap file on the hard disk. The word "memory" is never used to refer to the space on a drive (whether that's a hard drive, SSD, diskette, CD, DVD, or thumb drive). Space on a disk drive is normally called "storage" or simply "disk space."

Calling disk space "memory" isn't so much wrong as just unconventional. In a sense, disk space is, of course, memory, but the important thing to realize is that the word is never used that way, and if you do so, you do it at the risk of seriously confusing your audience.

With so little detail in your question, it's hard to be helpful, but I did want to point out that if that's the message you're getting, it does *not* refer to your hard drive.
