I narrowed the image (just to be sure that it was indeed an image) and then I deleted it and also cleared it from the Clipboard. As soon as I did that, I had no more problems with copying/pasting rows.

In this case the error message was valid rather than there being a problem with too many temporary files or the Clipboard being full (it was empty).

alan wu msft say:
Hi Bernd,

I’d like to answer your questions one by one:
1. After enabling the “auto-expending” of the Archive mailbox storage, the storage limit of Recoverable Items is still 100 GB. 
2. Only if the mailbox’s Litigation Hold is enabled, all the data in that mailbox will be kept on the server side. The Recoverable Items/Archive mailbox won’t delete the data.
3. We can only manually move the data in the Recoverable Items to Archive. There is no such a rule in EAC. 
4. You can use “Get-MailboxFolderStatistics -Identity user@contoso.com –Archive” to view the mailbox user@contoso.com (here, it is just an example) archive folder (mailbox) size. For more information, you can refer to https://technet.microsoft.com/en-us/library/jj984357(v=exchg.150).aspx.

Alan
