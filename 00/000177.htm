scottgem say:
Ok, I have to reiterate this. The first rule of Normalization is that data should be defined by values in rows and columns and by no other way. That's not what you are doing by using multiple tables. You are defining the data by using a table name. Instead you should have a field that defines the date the expense is for. You can then use that date to determine this year and last year or any year.

Do you need to maintain a history of the budgets and forecasts? If so, you can use just ONE table. If you are only using budgets for current year and forecasts for future year, then I would use a separate table for those. So your expense table becomes:
ExpenseID (Don't accept the Default name)
AccountNumber (don't use spaces in object names)
CostCentre
ExpenseTypeID (Actual, Budget, Forecast)
Period (Should be the first of the month So period 5 would be 5/1/18 Or you can use the actual data of the expense.)
Value

Now to do your query, you will need multiple queries. So your First query would join the cost centre table to get the mappings field. then you would use a Group By query to group by Mapping, CostCentre, and sum the month. The Criteria would be records for this year. You can then build on that query to get your YTD amounts.
