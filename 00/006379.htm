Count Unique with SUMPRODUCT() Breakdown - Microsoft Community

jeeped say:

Count Unique with SUMPRODUCT() Breakdown

Summary
You can quickly and efficiently create a count unique function using SUMPRODUCT(…) together with a standard condition that excludes blanks and a reiterative COUNTIF(…) function. This formula's framework has been around for quite a while in one form or another and is fully backwards compatible to Excel 2000-2003 worksheets. Even a beginning Excel user can usually transcribe a sample for their own purposes but not everyone fully understands the processes that generate the unique count. This article is an attempt to explain the mechanism.
Details
    Background: 
    The SUMPRODUCT(…) function is used as it supplies cyclic calculation akin to that provided by an array formula without requiring Ctrl+Shift+Enter. Cyclic processing differs from a standard formula in the same way that counting the occurrences of a single number found in a block of cells differs from counting the occurrence of each cell within the block to the same block of cells.

    It is also important to appreciate how Excel treats boolean values, particularly how they are regarded when used in a mathematical calculation. Excel treats a FALSE return as 0 (zero) when used in a calculation and TRUE¹ is regarded as 1 (one). Anything multiplied by zero is zero and anything multiplied by one remains unchanged so we can say that any number multiplied by FALSE will be zero and any number multiplied by TRUE will be that number. To take that concept one step further, any fraction with a numerator of zero will equal zero and any fraction with a numerator of one becomes the reciprocal of the denominator.

    Lastly, it must be noted that a cell containing an empty string (e.g. "") is not the same thing as a blank cell. An empty string is considered a value; specifically a text value of zero length. A blank cell is considered to contain nothing at all. The difference between the two becomes important when COUNTIF(…) is used on a range of cells that may contain blanks (see step 1 below).

    Scenario: 
    A financial worksheet has a column identifying which regional currency is being used for that row's transaction. A count of the different currencies used is required. Due to space considerations, the formulas will only deal with rows 2 through 9 but this method can be applied to any number of rows².
        
    Breakdown: 
    The formula used in C2 of the sample worksheet is,
        =SUMPRODUCT((A2:A9<>"")/COUNTIF(A2:A9,A2:A9&""))
  1. SUMPRODUCT(…) produces the iterative processing that utilizes COUNTIF(…) to cycle through each cell in the range, computing its frequency of occurrence in the same range. This produces an array of values in the manner of,
        COUNTIF(A2:A9, A2&"")
        COUNTIF(A2:A9, A3&"")
          ···
        COUNTIF(A2:A9, A8&"")
        COUNTIF(A2:A9, A9&"")
    The empty string is appended to the value in A2, A3, … A8, A9. This fools COUNTIF(…) into avoiding a #DIV/0! error that would otherwise arise if the cell was truly blank.
  2. Each of the COUNTIF(…) cycles is divided against the boolean result of whether the cell in A2:A9 is blank or not. This produces an array of fractions.
        A2 has CAD and there are two CAD's in A2:A9 so the calculation of <not blank>⁄<count> will be ¹⁄₂
            A3's calculation result is ¹⁄₃
            A4's is the same at ¹⁄₃
            A5 is another CAD so it is ¹⁄₂
            A6 is the only USD so this is ¹⁄₁
            A7 is another EUR at  ¹⁄₃
            A8 and A9 are both blank so each resolves as ⁰⁄₂
  3. SUMPRODUCT(…) takes those values and totals them.
            ¹⁄₂ + ¹⁄₃  + ¹⁄₃  + ¹⁄₂ + ¹⁄₁ + ¹⁄₃ + ⁰⁄₂ + ⁰⁄₂ = 3
    There are 3 unique currencies being listed in A2:A9.

    Footnotes: 
    ¹ In Excel's VBA, True is treated as (-1), not 1. False is still regarded as 0.
    ² Functions such as SUMPRODUCT(…) or any array formula consume calculation resources at a logarithmic rate. Avoid full column references when creating formulas with these functions. I've included a few blank cells to show that this formula can include a buffer zone to accept future row additions without being constantly modified, but the number of empty cells being referenced should be kept under control.

    Function Reference (2010): 
    SUMPRODUCT function   Returns the sum of the products of corresponding array components
    COUNTIF function   Counts the number of cells within a range that meet the given criteria

    More Information: 
    You can get more insight into the inner workings of these formulas by setting one up for yourself and stepping through the processing sequences with your Excel's Evaluate Formula command from the Formula Auditing group found on the Formulas tab (Tools, Formula Auditing, Evaluate Formula in Excel 2000-2003).

@Sadasiv Padhiary;

When providing an array formula solution, it is probably best to include a note that explains how the formula will need to be finalized with Ctrl+Shift+Enter rather than simply Enter. Failure to provide this critical information usually results in unnecessary replies stating that the formula does not work. This explanation can also include information on how Excel will warp the formula in 'mathematical braces', noting it as an array formula. The user does not type the braces in themselves (see next discussion point).

Another point would be your inclusion of the array formula 'braces' that Excel provides to wrap the formula when it is correctly finalized with Ctrl+Shift+Enter. Since new users looking for a solution to their problem will typically copy a formula directly out of a web page and transcribe the range(s) for their own purposes, including the braces that wrap the formula can generate confusion.

FWIW, I always try to provide standard formula (e.g. non-array) solutions first as worksheets get passed around and a new user unfamiliar with an array formula can easily 'break' it by attempting to complete a minor edit with just Enter. SUMPRODUCT(), INDEX() and other functions can provide array-style processing without requiring Ctrl+Shift+Enter.

These are the lessons I've learned from supplying several hundred (if not thousand) array formula solutions in these forums. I see you are a new user with a new Excel based web site. Welcome to the Microsoft Community.
