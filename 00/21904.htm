jllatham say:

OK, having given you that macro, I personally don't see any gain over using [Ctrl]+[Shift]+[:] other than you can control the format of the date/time display better.

 

If you want truly hands-off operation, if there is a cell somewhere else on a row that the user would type data into before getting to the date/time cell, or for that matter, pretty much anywhere else on the row, we could do it totally automatically with a Worksheet_Change() process.  It would be really nice if the timestamp column was at the far (right) end of the rest of the data and not in column A otherwise you have to teach them not to type into column A.

 

To put this code to work, open (a copy) of your workbook and select the sheet that this needs to function on.  Then RIGHT-click on the sheet's name tab and choose the [View Code] entry.  Copy the code and paste it into the code module presented to you and close the VBEditor after changing the column ID as needed (and the date/time format if desired).  Try it out: type something into the identified column and nothing special happens; but type into another column on a row and the date/time will be entered.  BUT once a date/time is entered, then it will not be updated with added entries on the same row.

 

After adding the code to the workbook, be sure to save it as a "macro enabled..." workbook, type .xlsm or the code will be deleted and unavailable the next time you open it.

 

Private Sub Worksheet_Change(ByVal Target As Range)
  'this should be the column ID where the date/time stamp goes
  Const timeStampCol = "F" ' change as required
  Const firstDataEntryRow = 2 ' assume labels in row 1
 
  'allows you to type in rows above the data entry
  'section without putting unwanted date entry there!
  If Target.Row < firstDataEntryRow Then
    Exit Sub
  End If
  'if they typed something into the timestamp cell,
  'leave it alone!
  If Not Application.Intersect(Target, _
   Range(timeStampCol & ":" & timeStampCol)) Is Nothing Then
    Exit Sub
  End If
 
  On Error GoTo RecoverFromError
  Application.EnableEvents = False
  'if the timestamp cell has nothing in it, put in the time stamp.
  If IsEmpty(Range(timeStampCol & Target.Row)) Then
    With Range(timeStampCol & Target.Row)
      .NumberFormat = "hh:mm:ss AM/PM" ' change as desired
      .Value = Now()
    End With
  End If
RecoverFromError:
  Application.EnableEvents = True

  On Error GoTo 0
End Sub

stefan blom say:

If you want to prevent grammar checking on your Word installation, deselect "Check grammar as you type" and "Check grammar with spelling" in Tools | Options | Spelling & Grammar. Note, however, that this won't prevent the green lines on another computer.

To suppress the checking completely, also when the document is opened by someone else, select all the text (Ctrl+A) and then click Tools | Language | Set Language. Check the "Do not check spelling or grammar" option, and click OK.

suzanne s. barnhill say:

You can fully configure the grammar checker to allow contractions or any other "errors." Go to Tools | Options | Spelling & Grammar and, under Grammar, click Settings... Note that you can choose to check "Grammar Only" or "Grammar & Style." Contractions are actually a Style issue, but note the check boxes below which allow you to select which issues you want checked for.

The settings you make in this dialog will apply to all documents, but you can also suppress the display of grammar errors in a given document by checking the box for "Hide grammatical errors in this document." This won't prevent you (or anyone else) from running a grammar check manually, but it will suppress the display of the green wavy underlines. This is useful when you are sending a document to someone else who may have grammar checking enabled (even if you don't).

For more on this, see http://sbarnhill.mvps.org/WordFAQs/MasterSpellCheck.htm#HidingErrors.
