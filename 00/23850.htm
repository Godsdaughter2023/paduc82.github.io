jeeped say:

You are correct in thinking that variable is the defining order with anything not found (like numbers) reverted to their strict Unicode identifier to avoid errors. Â, â, Î and î are not actually Unicode, just high ASCII so they do not strictly require the ChrW() function to be represented in the string assignment.

Your Excel 2013 can actually generate the numbers (Unicodes) that ChrW() is looking for with the UNICODE() function. To Reverse the Unicode to a Unicode character, the worksheet can use UNICHAR().

The formula in B2 is,

=UNICODE(A2)

The process is proofed by reversing the result in C2 with,

=UNICHAR(B2)

Fill down as necessary. The UNICODE() and UNICHAR() functions were introduced in Excel 2013 so Excel 2010 and previous Excel versions will not be able to use them. If you have an earlier version, the Windows utility Character Map (charmap.exe) can locate the characters and the Calculator utility (calc.exe) can resolve the hex designation to dec in Programmer Mode.

Hyperlink Description
UNICODE function Returns the number (code point) that corresponds to the first character of the text
