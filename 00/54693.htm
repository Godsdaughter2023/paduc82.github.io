mlg63 say:

Call it System Image instead of backup. It is not a true backup because it is an "all or nothing" image that must be installed intact with no single file available to be chosen. It is only used after major changes and never updated regularly like backups. It is for disaster recovery like installing a new hard drive or reverting to a previous windows installation and requires a bootable recovery disk to access it if you cannot boot to the repair menu. A System Image should be stored on an external hard drive, not on your hard drive and not on dvds or thumb drives. Setup a system image from the File History page, not your backup program which might have a similar image name but is not the same thing. Each system image will replace the previous one unless you use a workaround. And the date on the file is misleading. Mine has the wrong date too.

    I know this is an OLD thread but with no answer I wanted to put something here in case others have the same issues.

    I've had similar issues in the past. What all appeared to be a WMP problem turned out to be a collection of minor file problems. I probably don't have THE answer for you and most people probably don't because it's not a single problem. Instead, I'll give you some information I've learned (some of which my actually be wrong but is my assumption) to work with so you can hopefully sort it out.

    WMP works from a database and not from the files themselves. It scans your files and adds information to the database. The database has rules as to how the information should fit into it. I don't have those details but for instance, if only certain characters are allowed in a specific field, then if your file has disallowed characters in that matched field, it can bypass and ignore the file entirely.

    Likewise, there are certain expectations of file formats. So if you have a wma file that's really an mp3 or similar, it could (but won't always) overlook it.

    WMP is only "OK" at overlooking file errors. And over time, files can accumulate errors. Over the last several years, for security reasons, software has become less and less tolerant of errors in data because it could possibly be real attempts at doing bad things. VLC for instance seems to just not care. A file that is corrupted totally unplayable in WMP can often be played in VLC without a problem and maybe with a slight hiccup or two. Likewise, the database in WMP my simply ignore a file with corrupted header information and not add it to the database, but when manually played, may go ahead and give it a try and it  may play fine.

    As time has passed, WMP itself seems to have become more selective as to how many and what kinds of errors it will allow in the database. So it's possible to upgrade and then upon scanning of the database, lose files that previously showed up, not because WMP is doing something wrong, but because there are file problems that were ignored before. Or because file corruption happened after it was added to the previous database but since it was already there it was never removed.

    My advice would be to create a folder on your hard drive and then copy all files that are missing from the database to this new folder. If you have a LOT, you might want to either take note of the locations or copy entire folder structures.

    If you have whole folders being missed, copy the contents to a brand new folder and delete the old folder as there may be properties in the folder itself causing the problem such as the length of the path.

    Next, go through each file and folder and check it out in detail. Look at the properties. Eliminate any long file names. I once had a mystery file that would never show up and after weeks of troubleshooting found out that the problem is that the file was located at:

    L:\Family_Share\Drive\Users\Kevin\Media\Music\The_Presidents_of_the_United_states_of_America\The_Presidents_of_the_United_states_of_America_Ten_Year_Super_Bonus_Special_Anniversary_Edition_bonus_tracks\Naked_and_Famous_(New_York_Demo)"_(New_York_ 1987)–-2:08.mp3

    Obviously this was a problem. I couldn't even delete the damned file. lol

    Other issues I've found were files and folders marked as hidden and/or archived since I always show hidden files and folders.

    Another issue can be bad information in the ID3 tags. I've had issues where the ID3 information of a song matched another song even though it was a totally different song and artist. I have no idea how the ID3 info got mixed up. I fixed it and the song showed right up. I use a program called ID3AudioShell for that. I can right click and select "properties" on a file and in addition to normal properties, I can see and edit all of the ID3 info, Great little program, almost no overhead.

    Next is the possibility of problems with the codecs. Many different codecs could have been used to code the file in question and if that codec isn't on your machine, it won't play and it won't be imported. Or it may be close enough to play, but not good enough to get imported into the database. So I would download the windows media center codec pack  that's available free out on the internet. I would give you a link but there are some bad ones out there that are fake and/or infested and I don't have time to run down the good one and I don't want to link you to a bad one. So do a search for "Windows Media Player Codec Pack" and use your best judgement finding a good clean one.

    By now, most of your file issues should be solved. Anything remaining, convert to a different format. I use dbpoweramp for it since I can multi-select, right click, and then convert to anything I want. If they are wma, convert to mp3, or mp3 to wma. stay away from anything odd like wma variable bit rate, wma lossless, OGG, FLAC,etc. These are good formats but they are more complex and what you want to do is simplify the file. A good re-encoding should solve the last of the problems but be sure to rename the files as well so that the database doesn't just overlook them. Put them in new folders and re-scan. You should be good to go.

    Then of course, if all else fails, then go re-rip or "re-obtain" the files and replace them.

    Hope this helps. I've given up on expecting things to work right. lol
