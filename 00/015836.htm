I have therefore inserted an extra empty page, so the prologue starts on an odd page. This screws up the page numbering, with the first prologue page starting with II (2) instead of I (1).

I've been messing around to no avail. Does anybody know a way to denote a left (even) page with I (1)

Many thanks
jay freedman say:
Hi Jay,

This seems to work quite well for arabic numerals. For some reason I cannot get it to work with Roman numerals. I get e.g. !Undefined bookmark, I
This might be a problem if Word interprets roman numerals as letters rather than numbers. Any ideas?

Yes, that's exactly what's happening -- the roman page number ii is being seen as the name of a bookmark, consisting of the letters "ii", and that bookmark doesn't exist.

The fix is to force the inside page number field to be numeric, and then force the outer field to show a roman-numeral result.

Open the header or footer pane, and press Alt+F9 to expand all the field codes. You'll see

   { = { PAGE } - 1 }

Edit that to look like

   { = { PAGE \* arabic } - 1 \* roman }

The upper/lower case doesn't matter except for the word "roman" -- if that's lower case you'll get ii, and if it's upper case you'll get II.

When you've edited the code, press Alt+F9 again to collapse the fields.

By the way, although I've told you the mechanics of forcing Word to do your bidding, I agree generally with Suzanne's assertion that you shouldn't be doing it. Books have several centuries of tradition in the way they're constructed, and book lovers aren't very tolerant of attempts to change that.

john w. vinson say:

I have to quibble with your instructor: Now() does not return the date, it returns the date and time, accurate to the second. You don't need the time to count years.

Now() will work (though Date() is probably better). Take a look at the help topic for the DateDiff function to get an idea for how you would call Now().

marshall barton say:

Just type Now() wherever you want to use the current date/time value in your calculation.

Be aware that Now() includes the time part as a fraction of a day.  You may want to use the Date() function instead because then the time part is 0 (midnight) and you don't have to be concerned with fractions of a day in your age calculation - see http://access.mvps.org/access/datetime/date0001.htm

 

tony chen chn say:

Use DateDiff() and getDate() function to get a result you want.

The method is:

DateDiff(year, EmpEntryTime, getDate())

The "EmpEntryTime" is a calculated field of each employee's entry time in the database.

 

e.g(select the years worked for each employee which is greater than 3 years):

SELECT

         EmpID,

         EmpFirstName,

         EmpLastName,

         EmpEntryTime,

FROM EmpTable

WHERE DateDiff(year, EmpEntryTime, getDate())>3

 

Thanks.

john w. vinson say:

Tony, I think your syntax is off for a Microsoft Access application; the first argument of DateDiff should be a text string with a value "yyyy" to calculate years, and the builtin function for today's date is Date() rather than getDate().

Of course if you have defined a string variable named "year" with a value "yyyy", and defined a public custom function getDate() to return the date, your expression will work.
