jllatham say:

Mike H..'s and Tom Ogilvy's formulas should do that for you - when today is the last day of the month Mike's will show 0 days left, but tomorrow (say February 1st) it would show 27 days left.  You could add 1 to the result to get days left including current day.  Not exactly sure without testing how Tom's would appear if the last day or two of the month are on a weekend.

Because they use TODAY() and as the basis for the date and the month, they are always measuring current real-time date and month. 

Use Mike H..'s  if you want to include weekends and holidays - use Tom Ogilvy's if you want to count workdays only.

The only catch to either, I think, is that at least once a day you're going to have to go to the Excel sheet and do an [F9] or take some other action to cause it to recalculate and pick up the current date for the formula if the Excel file is left open 24/7.  Otherwise when you open it, it will pick up the current system date and update the result - but that would not be updated if the file then stayed open with no interaction for some period of days.  This potential shortcoming could be overcome by a short On Timer macro that would do something like cause that sheet to recalculate at least once a day - or even once an hour.  That would run in the background and make it a hands-off deal for you.

Not a reason.  If you used styles properly it would be automatic.

I suspect that these are not even numbered styles.  Still, you you insist.

Sub AddCrappySpaces()
Dim oPara As Paragraph
For Each oPara In ActiveDocument.Paragraphs
   Select Case Left(oPara.Range.Text, 1)
      Case 1 To 9
         oPara.Range.InsertAfter vbCrLf
   End Select
Next
End Sub

This checks every paragraph, and if it starts with 1 to 9, adds one of those horrid "extra" paragraphs after it.  This is a BAD use of Word.

At least you could make a Style, with whatever format and spacing (Before, After...whatever) you want, and then apply it using the same logic.

Sub AddCrappySpaces()
Dim oPara As Paragraph
For Each oPara In ActiveDocument.Paragraphs
   Select Case Left(oPara.Range.Text, 1)
      Case 1 To 9
         oPara.Style = "Fat_Paragraph"
   End Select
Next
End Sub

This checks for the same thing, starting with 1 to 9, and then applies the style Fat_Paragraph.  This is simply as Suzanne suggested:  a use of Before/After attributes.

ossiemac say:

I have resolved this issue by looping through the references as per the following code.

However, if anyone knows how to simply use a single line of code without the loop then I am still interested and will appreciate the reply.

Sub RemoveReference()
    Dim i As Long
    'Dim ref As Reference   'Code will error if Reference already removed
    Dim ref As Object       'Late binding with Object eliminates the error
   
    For i = 1 To ThisWorkbook.VBProject.References.Count
        Set ref = ThisWorkbook.VBProject.References.Item(i)
        'Debug.Print ref.Name       'For testing only
        If ref.Name = "VBIDE" Then
            'Following only unchecks reference in Tools -> References
            'Does not entirely remove the Reference from Excel VBA (Which was referenced in my original question)
            ThisWorkbook.VBProject.References.Remove ref
        End If
    Next i

End Sub

a. user say:

Hi,

 

You can use the keep me signed in feature on your computer. Once you check “Keep me signed in” you'll be kept signed in with your Windows Live ID until you explicitly sign out. This also has the nice benefit of giving you a much faster page load time, so that getting to your inbox or whatever service you are using is now much faster.

 

Please try to re-enable refer to the steps below: 

1.       Visit http://live.com.  

2.       Click Sign in located on the upper right of the screen.

3.       Windows Live sign in page will appear,

4.       Tick the box for Keep me signed in and enter your credentials.

Note: Optimizing your browser will disable this feature. Make sure that your browser is not set to Delete browsing History on Exit. To check:

  1. Click on Tools.
  2. Select Internet Options.
  3. Remove the check on Delete browsing History on Exit.
  4. Click ok.

Since mymsn.live.com requires the Windows Live ID credentials, you will be asked by default to log in (login.live.com) unless the keep me signed in feature is not enabled.
