greg maxey say:

To populate a ContentControl, modify Dougs code as follows:

 

Sub FillCCLsitWithExcelData()
Dim xlapp As Object
Dim xlbook As Object
Dim xlsheet As Object
Dim i As Long
Dim subject As String
Dim bStart As Boolean
Dim ffield As FormField
Dim oCC As ContentControl
  On Error Resume Next
  Set xlapp = GetObject(, "Excel.Application")
  If Err Then
     bStart = True
     Set xlapp = CreateObject("Excel.Application")
  End If
  On Error GoTo 0
  Set xlbook = xlapp.Workbooks.Open("D:\Data Stores\Data Source.xlsx")
  Set xlsheet = xlbook.Worksheets(1)
  With xlsheet.Range("A1")
     Set oCC = ActiveDocument.SelectContentControlsByTitle("Names").Item(1)
       For i = 2 To .CurrentRegion.Rows.Count
         Debug.Print .Offset(i - 1, 0)
         oCC.DropdownListEntries.Add Text:=.Offset(i - 1, 0), Value:=.Offset(i - 1, 0)
       Next i
  End With
  xlbook.Close
  If bStart = True Then
    xlapp.Quit
  End If
End Sub

a. user say:

In fact they can be created on Mac Word, but there are no User Interface options to do so in the out-of-the-box product, and there is no support yet for creating them or manipulating them in Mac Word VBA.


In addition, even if you create them, they cannot currently function the way that they do on Windows Word, primarily because there is (as far as I know) no support for the content control events and xml data store events that are available on Windows.

The one way that you might be able to create them without coding would be if the Office App Store had an Addin to do it. I did have a look, and there does not appear to be anything that is designed for that purpose except a thing called "Word Content Control Wizard" - unfortunately, that currently appears to be a very crude piece of experimental code that just shows how you can insert a single Rich text control and modify its properties. Perhaps one of the more commercial "Document Builder" addins actually lets you do it - they mainly make you sign up for their services so I haven't actually looked.

SO at the moment if you wanted to do it, you'd have to write your own Addin using the Javascript APIs, which do support the insertion of content controls and modification of their properties. Serious developer skills and quite a lot of effort required, I think!

Other ways you could probably do it if you wanted (other than the easy way, by creating the things on a machine with Windows Word) include the following, but they all require dev. skills, familiarity with the relevant SDKs and formats, and probably lots of effort:

 a. write code that generates a complete Office Open .docx file (or the single file .xml equivalent). On Windows you can use the .NET Office Open SDK to help with that. I believe there is a project to port that SDK to Mac, but I do not know what its current status is. You may be able to find out on GitHub. I guess you could either use the MonoDeveloper environment or the new Visual Studio for Mac to create some C# code to do it. 
 b. Create a control of each type on Windows Word and save it as an OOXML .xml format file, then strip out all the unnecessary bits. You then have a set of templates that you could edit to insert the control names, tags and other properties before inserting them into Word via the Insert->Text->Object->Text From File option or using VBA. At least I think that would work.

As for *why* they are not fully implemented, like Jim I can make some guesses but that's about it.
