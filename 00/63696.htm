mouneshwar r say:

Hi Cornell,

 

·         Are you talking about one single update which failed for the first time and got installed successfully the second time it was offered or there are two different important updates wherein one installed and other got failed?

 

If you’re talking about one particular update which was failed initially with error code 80246007 and then got installed successfully the second time, then everything is good. The Windows update history just keeps track of the status of each update that was offered and attempted to install. If you want to remove only the update that displays failed status in the update history, it is not possible. However, you can manually erase the update history which erases the record of all the update irrespective of their status (successful or failed).

 

Note: Erasing the update history will just erase the record of the updates and doesn’t remove the updates completely. In other words, this action will not affect the currently installed updates on your computer. Subsequent updates will appear in the history list.

 

Steps to erase the update history completely

Note: If you don’t want to erase the complete update list, then don’t follow these steps.

a.    Click on Start; in the start search type cmd.

b.    Right click on cmd.exe and choose to “Run as administrator”.

c.    Type the following command and press Enter after typing each command

            net stop wuauserv

            net stop bits

d.    If you’re at C:\windows\system32 prompt, type cd.. to get back to c:\windows

e.    Once you’re at c:\windows, type the following command and press Enter.

            ren softwaredistribution softwaredistribution.old

f.     Restart the stopped services by typing the following commands

            net start wuauserv

            net start bits

g.    Close the command prompt.

 

Now open Windows update and check the view update history.

 

Additional information:

Suggest that you first rule out malware as the cause of the issue by doing the following:

2) Run this tool from Kaspersky: How to remove malware belonging to the family Rootkit.Win32.TDSS <http://support.kaspersky.com/viruses/solutions?qid=208280684>


If no malware was detected, there is a Fixit on this page that will reset Windows Update components:

How do I reset Windows Update components <http://support.microsoft.com/kb/971058>

Suggest you download and save it. Then close all open programs and web browsers.

After it's finished running, see if the return code appears again when you check for updates.

Restart once it's finished running and see if the return code still appears when checking for updates.
