ken sheridan say:
................is not a good idea to use calculated field in a table, it's a redundance.

That's not really the case.  Redundancy really means the storing of data which is determined by one or more non-key columns, giving rise to the risk of update anomalies.  The whole point of a calculated field is that the value is not stored but calculated dynamically, so no update anomalies are possible.  I would not use one myself, but instead use a computed column in a query or a computed control in a form or report, but I see no reason to discourage others from using them if they wish.

In the current case of course a computed column in a query or a computed control in a form or report is the only solution.

A different approach...

Function bFileExists(strFile As String) As Boolean
  Dim lngAttr As Long
  On Error Resume Next
  lngAttr = GetAttr(strFile)
  bFileExists = (Err.Number = 0) And ((lngAttr And vbDirectory) = 0)
End Function
