kevinmccartney1 say:

Export access report and subreports to separate pdfs

In Access 2016, I have a form with a button that opens a report. That report is exported to pdf as one file currently.

The change request is to instead export the (parent) report and its subreports into separate pdf's. This needs to be done from a new button on the form. The subreports exist in the footer and there are 5 of them. So I need to export the parent and loop through each of the footers to export the subreport therein. How do I refer to the footers and subreports in VBA in order to call DoCmd.OutputTo? And how do I refer to the parent report standalone i.e. without the subreports?
ken sheridan say: You would have to output each of the subreport's source report objects independently by calling the OutputTo method five times, naming the report object as a literal string expression in each case.  

As the subreport's are in the report footer I imagine that they are not linked to the parent report by means of the LinkMasterFields and LinkChildFields properties.  If they are linked, however, then the method of linking would need to be changed.  Rather than doing so by means of the LinkMasterFields and LinkChildFields properties, the RecordSource of each would need to be restricted by referencing a control in the calling form as a parameter.  This is necessary because the OutputTo method, unlike the OpenReport method, has no argument by which a report can be filtered.

To output the parent report you would need to create a copy of the current parent report, omitting the subreports, and output this.  This again would need to be restricted by means of a parameter if the parent report is currently filtered by means of the WhereCondition argument of the OpenReport method.

In most cases, the Office activation issues can be resolved if you repair the Office program from the Control Panel and then try to reactivate it. Refer the article below for more information. If the error matches any of the errors that’s listed in the article, follow the steps provided in the ‘Workaround‘ section.

http://support.microsoft.com/kb/982750

 

Another main cause for this issue is the presence of other versions of Office on the computer, which may include trial versions as well. If you have any previous versions of Office, you may uninstall them, then restart the computer and check if you’re able to activate Office 2013.

 

Refer the section ‘Remove Microsoft Office 2010 suites manually’ provided in the article below to manually remove all traces of Office 2010:

http://support.microsoft.com/kb/290301

Note: You may take a backup of the registry before troubleshooting.

 

Refer the section ‘Remove Microsoft Office 2007 suites manually’ provided in the article below to manually remove all traces of Office 2007:

http://support.microsoft.com/kb/928218

Note: You may take a backup of the registry before troubleshooting.
