General info on resetting to factory condition:

Reinstall Windows Preinstalled On Computer

OEM computer manufacturers are required to provide at least one way for you to reinstall Windows if you buy a computer with Windows preinstalled - your computer should have included either a restore partition on your hard drive, or Windows install/restore disks. ﻿Most computer manufacturers also have a feature to make your own install disks.

If the restore partition is still there and was not removed, it will reinstall Windows. There is usually a prompt when you first start the computer telling you what key to press to start the restore﻿ - check your manual if you're not sure how to. Or if you have install disks - insert them, restart and boot from them to start the reinstall process.

If you restore it to factory condition, it will be like it was when you first bought it - all preinstalled programs will be installed. Anything you installed after buying it must be reinstalled. This will remove any personal files on the computer, back them up first.

How do I stop Windows Backup from backing up D: ? - Microsoft Community

eric kolotyluk say:

How do I stop Windows Backup from backing up D: ?

My backups used to work OK, but now for some reason it insists on backing up stuff from D: too.

I have configured my backup with custom settings, and checked to make sure that the D: partition is NOT selected. However, when I run my backup it takes too long. When I check to see what is going on I can see it copying files from D: to by backup drive F:

What is wrong with Windows Backup? Why can't is simply do what it is configured to do? Why does it insist on backing up files from D: when I have specifically configured it not to do so?
eric kolotyluk say: OK, I finally figured out what the problem was. There are actually a few problems involved.
  1. Libraries. Ever since this feature was added to Windows I have never felt comfortable with it. It is one of those things that is really not very obvious. A library is a collection of a bunch of places in the file system, but it is not clear where all those place really are - it is not clear what is actually in the library.
  2. In the standard backup settings, when you select "Let me choose" from the "What do you want to back-up?" dialog, you get two sets of options (a) Data Files, which contains a bunch of libraries with Vuse and WindAmp Remote to look in various places in D:
  3. Unbeknownst to me, this has a side effect of putting stuff from D: into my libraries, so them my backups started trying to back up all the media (that I did not want backed up). Real software developers all know the perils of side effects - but sadly at Microsoft any real software developers who might exist do not get make real decisions.
  4. If you drill down into the Data Files settings in Windows Backup there is absolutely nothing in the UI to indicated the explicit structure of what will be backed up - the whole thing is completely implicit. up so that the user has some clue to what is really going on.
  5. Further, the fact that well intentioned experts were having me try other ways to configure a backup shows how poor the design of the standard mechanism is as they could not fathom what the actual problem was.
