peter.r. say:

Hi Linda,

I recently had what appears to be the same problem, which I was able to resolve with a simple tweak of the Registry.

Before I offer the solution however, I will elaborate for the benefit of others with this problem.

My problem computer was using Windows 7 (32-bit) with Internet Explorer 11 and Windows Live Mail 16.4.3508 (2012).

The first symptom was the constant flashing of the messages 'Updating calendar...' and 'Your calendar has been updated' in the status bar of Live Mail.

Another symptom was that the Live Mail Calendar was actually failing to synchronise despite the spooler Tasks tab (click on the flashing status bar message) indicating [task] 'Update all calendar events' [Status] 'Completed (it was actually continuously alternating between [Status] 'Executing' and [Status] 'Completed'.

Yet another symptom was that my Outlook.com (HTTP) account was failing to synchronise, with the spooler throwing the following error after I stopped the perpetual synchronisation attempts:

Unable to send or receive messages for the [outlook.com] account.

Server Error: 3204

Server: 'https://mail.services.live.com/DeltaSync_v2.0.0/Sync.aspx'

Windows Live Mail Error ID: 0x8DE20003

I knew this was a User Account problem on my computer because another User Account on this computer had no such Live Mail problem with the same Outlook.com account.

So as instructed at  http://support.microsoft.com/kb/2842520  I deleted the relevant registry entries that referred to SSL under the registry key:

HKEY_CURRENT_USER\Software\Microsoft\Windows Live\Communications Clients\Shared\Policies

I also ran the Microsoft Fix it offered at  http://support.microsoft.com/kb/2842520

Sadly, this fix failed to solve the problem.

In any case, I solved the problem by correcting an inaccurate registry Value Data for the registry Key:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Internet Explorer\Version Vector

On a 64-bit system, this key is:

HKEY_LOCAL_MACHINE\software\Wow6432Node\Microsoft\internet explorer\version vector

With the registry Value Name:

IE

I noticed that my Value Data for the Value Name IE was 9.0000 despite me having Internet Explorer 11 installed, so I changed this Value Data to 11.0000

This immediately solved the problem.

Thus, I suggest that you also change this inaccurate registry Value Data to the same major version number of your system's Internet Explorer. Hopefully this will fix your system.

More information and instruction on changing this registry Value Data can be found at  http://support.microsoft.com/kb/969148  (see ' Method 2: Change the registry' - however, it only refers to IE version 8.0000).

I suggest that you close Windows live Mail before changing this registry Value Data, and restart your computer after changing it.

Please create a System Restore Point before you venture into the Windows Registry.

Cheers,

Peter.R.

    Follow the steps:

    a.       Go to Start and type internet options in Search box.

    b.      Click on Internet Options.

    c.       Click on “Connections” tab.
