hal hostetler - mvp-outlook say:
The problem is due to changing from POP to IMAP. IMAP requires its own folder set because it is server based, like an Exchange Server; your email lives on the server and the folder set on your computer is nothing more than a mirror image of what's on the server. POP, on the other hand, is client based; mail is downloaded from the server and removed from it by the client. IMAP was created to solve the problem of multiple computers needing to be in sync and it does this well. The shortcoming is that it must live in its own world.  Outlook.Com, unless you connect to it with POP protocol, has the same limitation. It uses Exchange Active Sync and is like both Exchange Server and IMAP in that it is server based and must live in its own world.

mateiarutp say:

I managed to solve this by booting into windows recovery cmd (advanced startup options - command prompt), loading the SOFTWARE hive from C:\Windows\System32, deleting the values in TemporaryPaths, unloading the hive then rebooting. It might be just me, but I think the process of allowing Windows Defender to detect malware should be a little less complicated and more user friendly.

EDIT: To get into advanced startup options:

- Win + X

- Shut down or sign out

- Hold shift & click on restart

- Troubleshoot - Advanced Options - Command Prompt

Then login to your user, in the command prompt window type "regedit", then select HKEY_LOCAL_MACHINE, then from the File menu select Load hive. Here go to your Windows drive (should be C:), then in Windows\System32\Config\, here select the SOFTWARE file. Next, you will be prompted to enter a name for the key this hive will be loaded as. Enter any name that isn't already in use (I entered WindowsSoftware). Next, open your WindowsSoftware (or however you named it node), go to Microsoft\Windows Defender\Exclusions\TemporaryPaths. Here, delete the values (paths) that you want removed from exclusions. Finally, select your WindowsSoftware (or, again, however you named it), then go to File - Unload hive. After unloading, close regedit then the cmd window. In the new screen that appears, select Continue. Your computer should now boot normally into Windows, and the TemporaryPaths values will be removed.

Hope this helps!
