I would recommend checking the Bluetooth drivers and the device drivers

How to update drivers
Go to Device Manager > select mice and other pointing devices> right click in the mouse > Update driver

Uninstall and install the drivers

Please try these methods
Method 1
Restart Bluetooth Support Service

Check if the Bluetooth Support Service is running, to do this do the following:

Press Windows key + R and type services.msc.
When the Services window opens search for Bluetooth Support service, right-click it and select Restart. Bluetooth support service
Right-click Bluetooth Support Service again and this time chooses Properties from the list.
When the Bluetooth Support Service Properties window opens set the Startup type to Automatic.
Method 2
Enable Bluetooth Audio Service
Press Windows key + X and select Control Panel from the list.
In Control Panel select Devices and Printers.
Next, locate the Bluetooth device and right click the device. Choose Properties.
When the Properties window opens go to Services tab.
Method 3
Update your Bluetooth drivers
go to your Bluetooth adapter manufacturer’s website, and check if there is a Windows 10 driver available. If no Windows 10 driver is available you can just download the latest driver.
Select the Start button, type Device Manager, and select it from the list of results.
Expand a category (like Monitors or Network Adapters) to find the name of your device, then right-click (or tap and hold) it, and select Update Driver. For example, to update a graphics card driver, expand the Display adapters category, right-click your graphics card, and select Update Driver.

sir rujak say:

I attempted to get the T-16000M FCS working today and the main issue is that the adaptive controller does not correctly interpret the joystick position. Rather than allowing both vertical and horizontal motion the mapping allows it to only use the horizontal movement and maps that onto a diagonal. As such at the moment I wouldn't consider it compatible. I have submitted this as an issue on the Xbox insider program so hopefully they will patch it in an update.

P.S. I tried it on Windows 10 as well and while windows can correctly interpret the input directly hooking the joysticks in through the adaptive controller gave the same issue. Probably means that it needs a new mapping config in the controller soft/firmware.
