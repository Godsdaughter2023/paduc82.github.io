hal hostetler - mvp-outlook say:

Big companies use either IMAP or Exchange Server, since both are server based rather than client based.  In your IMAP system, your mailbox world lives on the server and your devices are means by which you manage this mailbox.  When an email comes in, it will appear on both devices and webmail at pretty much the same instant.  If you read it on one, it will show as being read on all three.  If you delete it on one, it will be gone from all three because you're manipulating the server in all cases.  The beauty is that all devices are always in sync with each other.  If one of your devices has a catastrophic failure, you lose zero email because it all lives on the server, not the devices.  Exchange Severs add to this by allowing you to also sync Calendars, Contacts, To-Dos, and a number of other items that IMAP doesn't support.

POP, OTOH, doesn't sync anything.  With two devices and a POP account (without using the "Leave a copy of email on the server" fudge that most email clients provide these days), the first device to login gets all the new email and it's deleted from the server  If you immediately login with the second device, you'll find either nothing or maybe an email or two that arrived in the time it took you to get the second one fired up and connected.

In a multi email enabled device world, the current trend is for all to be in sync with each other to the maximum extent possible.

hal hostetler - mvp-outlook say:
"if one computer is asleep and not constantly polling the server, how will it be able to see what

The other computer will not have grabbed it off, that's a POP thing, not an IMAP thing.  When the sleeping computer wakes, it will sync and show you exactly what the first computer and webmail both see.   Say that one is on the other is asleep and you're watching the one that's on.  Five emails arrive and you read two then wake up the sleeper and let it sync, it will show five new emails with two of them marked read.

"Do I have to create folders that match on the server and both machines so that when I move something on one it moves it on all three?  I moved some stuff from the inbox to a folder I created in Outlook using a Rule, and could not then find them on the server."

Folders created on one device will sync back to the server and, from there, to the rest of the connected devices, if all three are setup as IMAP.  I have several IMAP accounts, if I create a folder in Outlook, it will appear on both my phone and in webmail as soon as Outlook syncs with the server.  It is important that all devices are set the same way; if one device is setup as POP and the rest are IMAP, things will get messy fairly quickly.

I don't know how much wisdom I have...less and less some days, but I will throw out another idea...

I haven't had to deal with more than a few layers and so saving as an image has never been an issue. Try grouping the elements together as you build the 'object'. In other words after you have added a layer/object to the base layer/object, group the two together before you add a third, and then group the third to the original group before you add the 4th etc. By making sure each added element is grouped with the group as you build your 'object' then perhaps by the time you are done you will be able to right click and save as an image as you want.

If you find that works, it may be that you can group 4 or 5 layers together, then build a seperate object of 4 or 5 layers grouped to together and then drag that one on top the first group, arrange and group. In other words do things in batches.

I suggest a variation of this work flow because if you have 10+ objects you are layering, then obviously you are not able to group them all together by conventional means if some are left behind when you right click save as an image.

By the way, as J brought it up, I assume you know that you cannot group elements that are on the Master page with elements that are not...
