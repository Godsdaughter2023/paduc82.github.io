andre da costa say:

Should I upgrade my existing version of Windows to Windows 10 Preview?

Since Microsoft made available the first pre-release build of Windows 10 in early October 2014, there has been some misunderstanding that this is not a product you should use to replace a production installation of Windows such as Windows 8 or 7 or Vista. Instead, if you must test it on a single machine, the best recommendation is to create a partition or have an a spare hard disk ready where you can install it. Windows 10 in its current pre-release state will expire and at the same time, it is unfinished work. So if you must evaluate it, the best way is by installing it beside your commercial installation of Windows.

If you do choose to upgrade your existing version of Windows to Windows 10, please be warned, it will make your recovery partition in operable, requiring that you have separate recovery media for reinstallation if necessary.

Where can I get more information about preparing my computer safely for testing Windows 10?

  • How to install the latest Windows 10 build

dustin_015 say: Hi All,

I just had the same problem described in this thread and wanted to share my solution in case it may help anyone else. 

I had a tab of data, and I wanted to move this data to new tabs. All my charts were working fine, but then I went to delete the original tab of data, and got the same error message described above. I went through every single chart, data series by data series to try and find what was referencing the tab I had deleted. There was not a single one. I ended up wasting a lot of time, I deleted every chart but 1 at a time, and then would delete my data to isolate which charts were causing the issue.

I found two charts that were still referencing the bad data. Both of these charts were set up with a secondary axis. Even though I had edited the "Horizontal axis labels" to reflect the labels on the new tab, the secondary axis was still trying to reference the tab I had deleted. 

My solution is to temprarily move all data series to the primary axis. Then update your axis labels, and then restore the data series to the secondary axis. 

Took me about an hour and a half to get to this, so I wanted to share in case it may be able to help someone else out.
