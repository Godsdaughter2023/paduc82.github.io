Can I stop user to add new items into a list app - Microsoft Community

li nen say:

Can I stop user to add new items into a list app

I have a list app in Sharepoint online and I want to stop using it, but just update the remaining item's state.

So how can I do to prevent users(even for me) to add new items into it, but permit them to modify the available items?

felix tao msft say:

Hi Li Nen,

 

If you are the site admin, you can't prevent yourself from adding new items. You can manage other users' permissions by creating a new permission level. To do that:

1.Create a new permission level in the site.

Go to the SharePoint Online where is the SharePoint list is>Click the gear on the upper right corner and select Site settings>Under Users and Permissions, choose Site permissions>Click Permission Levels under PERMISSIONS>Access Contribute permission level>At the bottom of the Edit Permission Level page, click Copy Permission Level>In the Copy Permission Level page, give the new permission level a name and uncheck Add items and Delete Items as below>At the bottom of the page, click Create.

2.Give the SharePoint list a unique permission and remove users' permissions from it, and then assign the newly created permission to all the users

a.Go to the SharePoint List>If you are in the new UI, click the gear on the upper right corner to access List settings. If you are in the classic UI, click LIST in the top ribbon to find and access the List settings>Under Permissions and Management, select Permissions for this list>Click Stop Inheriting Permissions. Note: After clicking Stop Inheriting Permissions, it would create a unique permissions for this list. Changes made to the parent site permissions will no longer affect this list.

 

b.Remove all the users' permissions for this list: Select all the SharePoint Groups, and then click Remove User Permissions as below.(after that, other users can't access the SharePoint list).

c.Assign the newly created permission level to all the users: Click Grant Permissions>In the Invite people section, input Everyone except external users>Expand the Select a permission level to find the newly created permission level.>Click Share.>Let the users edit the items to check the outcome.

 

 

Please note that the site admins still can add new items or delete items from the list.

Thanks,
Felix

brian tillman [outlook mvp 2007-2019] say:

When you change versions of Outlook, particularly when downgrading, you will need a new profile.  Moreover, the fact that you say you imported the old PST indicates that you created a new Unicode PST (the format native to Outlook 2003) and imported your old PST into it.  The new format PST cannot be read by Outlook 2002 or earlier.  The fact you imported, though, should save you.  This means that you should still have your original PST and it will be in the same folder as it was before you updated.  For Windows XP, that location is %UserProfile%\Local Settings\Application Data\Microsoft\Outlook.  Open Windows Explorer, enter this exact path into the Address field and click Go.  You should see your PST there.

With Outlook 2000, it's somewhat problematic to create a new profile.  Outlook 2000 has two configuration modes that can access Internet mail: "Internet Mail Only" mode (the default) and "Corporate/Workgroup" mode.  My personal choice was always to use C/W mode because then I have the menu item Tools>Services when I could control the mail profile, rather than Tools>Internet Settings where I could not.   Make sure Outlook 2000 is installed in C/W mode so that the Mail applet in Control Panel allows you to create a new mail profile and add the PST originally used the last time you hade Outlook 2000 installed.

steve rindsberg say:

>> The lack of the overtype function is a reason I generally refuse to translate PowerPoint docs of any length or complexity. I'm not going to recreate the formatting in a new doc, and adding text within the existing doc before deleting the existing text creates a giant mess.

While I agree that the loss of overtype is unfortunate, I think you may have missed Bob's point:

If you first select the text that you want to replace, then type the text you want to replace it with, you've overtyped the original for all intents and purposes, and in general the original formatting should be retained.  

I think you might actually find it less error-prone, in fact, since you can select exactly the word or phrase you're translating and type in the replacement this way; if the replacement word/phrase happens to be longer or shorter than the text in the original language, you don't end up overtyping or leaving behind unwanted characters.
