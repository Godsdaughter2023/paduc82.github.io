Drb555,

Hello and thanks for your post.

This behavior can occur if the original file was moved. Briefcase keeps track of the original location of all the files that you copy to Briefcase. However, it cannot track the files if they are moved to a different location. After you move a file on the primary computer, the file stored in Briefcase becomes an orphan file, and therefore it can no longer be synchronized with its original counterpart outside Briefcase. This is the case even if the file is moved back to its original location.

To resolve this issue, first rename the modified version of the file, and then copy the original file to Briefcase. Open and save the modified version of the file with the same name as the original file that you just copied to Briefcase, and then delete the orphan file.

BIOS and chipset drivers).

DriverView - Free - utility displays the list of all device drivers currently loaded
on your system. For each driver in the list, additional useful information is
displayed: load address of the driver, description, version, product name,
company that created the driver, and more.
http://www.nirsoft.net/utils/driverview.html

For Drivers check System Maker as fallbacks and Device Maker's which are the  

most current. Control Panel - Device Manager - Display Adapter - write down 
the make and complete model of your video adapter - double click - Driver's tab -
write down the version info. Now click UPdate Driver (this may not do anything
as MS is far behind certifying drivers) - then Right Click - Uninstall - REBOOT this
will refresh the driver stack.

Repeat that for Network - Network Card (NIC), Wifi, Sound, Mouse and Keyboard
if 3rd party with their own software and drivers and any other major device drivers
you have.

Now go to System Maker's site (Dell, HP, Toshiba as examples) (as rollback) and
then Device  Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get their
latest versions. (Look for BIOS, Chipset and software updates at System Maker's
site while there.)

Download - SAVE - go to where you put them - Right Click - RUN AD ADMIN -
REBOOT after each installation.

Always check in Device Manager - Drivers tab to be sure the version you are
installing actually shows up. This is because some drivers rollback before the
latest is installed (sound drivers  particularly do this) so install a driver - reboot -
check to be sure it is installed and repeat as needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER

If the iertutil.dll file is missing, you will see a message like this:

Explorer.EXE - Unable To Locate Component
This application has failed to start because iertutil.dll was not found. Re-installing the application may fix this problem.

The iertutil.dll file is required for explorer.exe to start.  For the moment, think of the running explorer.exe as your desktop since it is necessary to display all of your desktop icons, taskbar, etc.  If explorer.exe is not able to run, you will not be able to do much with your system until the file is replaced.

There is no copy of iertutil.dll in the dllcache folder since that is not one of the 3498 files that Windows File Protection cares about.

There is also no copy of iertutil.dll on any XP installation CD for you to expand to replace the missing file.

There are different versions of iertutil.dll for different versions on Internet Explorer and one of each is on my SkyDrive:

IE7_iertutil.dll   for IE7
IE8_iertutil.dll   for IE8

From your working system, download the one you need (or both if you are not sure) and then copy the files to a floppy disk or USB thumb drive so you can carry them over to the afflicted system using sneakernet (that means you walk it over). 

Let us say that you are using a USB thumb drive since that is what most people will have handy.

On the afflicted system, press Ctrl-Alt-Del to bring up the Task manager and then choose Shutdown, Restart and start tapping the F8 key as Windows is loading to get to the Advanced Boot Options menu.  If you miss the F8 windows of opportunity, let the system finish booting and try again until you do see the Advanced Boot Options menu.

From the F8 Advanced Boot Options menu, boot the afflicted system into Safe Mode with Command Prompt - the Command Prompt will still work even if the iertutil.dll file is missing.

Then copy the correct file from your floppy or USB thumb drive to the afflicted system and call the file:

iertutil.dll

For example, if on the afflicted system Windows is installed on your C drive and your USB thumb drive is drive E and you are running Internet Explorer 7, you would enter a command like this:

copy  e:\IE7_iertutil.dll   c:\windows\system32\iertutil.dll

You should see a message that one file(s) was copied.

For example, if on the afflicted system Windows is installed on your C drive and your USB thumb drive E and you are running Internet Explorer 8, you would enter a command like this:

copy  e:\IE8_iertutil.dll   c:\windows\sysmte32\iertutil.dll

You should see a message that one file(s) was copied.

Then from the Command Prompt, enter the following command to launch Windows Explorer:

explorer.exe

Then you should be reminded that you are booting into Safe Mode (click Yes) and you will see the Safe Mode desktop and then from there you can remove the USB thumb drive, enter 'exit' in the Command Prompt window to close it and just choose to restart your system normally and you will be good to go.

If you copy the wrong version of iertutil.dll and try to launch explorer.exe, you will see a message like this:

Explorer.exe - Ordinal Not Found
The ordinal 681 could not be located in the dynamic link library iertutil.dll

That message means you copied in the wrong version of iertutil.dll, so copy in the other version of the file from your thumb drive and follow the other previous instructions to launch explorer.exe and restart your system.
