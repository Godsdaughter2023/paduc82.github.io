suzanne s. barnhill say:

That article is misleading. You can apply individual borders to paragraphs and table cells but not to selected text. No matter what you select, as long as Text is selected for "Apply to," it will be a box border.

There is a way to get individual borders. For a "bottom border," you could perhaps make do with an underline. For other sides, see http://wordfaqs.mvps.org/Overbar.htm#UpperBorders. But the EQ \x field doesn't allow any customization of the distance from text, so it is not always satisfactory.

If you're trying to get a bottom border on just the text of, say, a centered heading, and the underline is not satisfactory, you can apply a bottom border to the entire paragraph and then drag the right and left indent markers to make the paragraph exactly the width of the text. See also http://gregmaxey.mvps.org/word_tip_pages/banner_text.html.

In older versions of Word, I remember a check box for "suppress blank fields" but I can't seem to find that in 2007.
suzanne s. barnhill say:
In general, blank lines are automatically suppressed, but only if they end in a paragraph break, not a line break. Consequently, the default is not a solution for me and perhaps not for you, either. So you have to use an IF field. Assuming your two address lines are the merge fields Address1 and Address2, you'd have something like this.
 
{ MERGEFIELD Address1 }<line break>
{ IF { MERGEFIELD Address2 } = "" "" "{MERGEFIELD ADDRESS2}<line break>" }
 
The way that will actually appear in your document is:
 
{ MERGEFIELD Address1 }
{ IF { MERGEFIELD Address2 } = "" "" "{MERGEFIELD ADDRESS2}
" }
 
By putting the line break inside the FalseText quotes, you ensure that thereis a line break only if the line contains text; otherwise, there will be no line.
