Introducing Clubs and LFG for the Xbox app!

Hey everyone,

We're happy to introduce two new features on the Xbox beta app! Check them out today and let us know what you think! 

Introducing Clubs on Xbox Live

Check out the Community tab on the dashboard to find the new entry point for Clubs on Xbox Live. Create your own Club or explore the Clubs the community has created. You can request an invite to a Club, share content to the Activity Feed, and check out Clubs on Game Hubs and your friends profiles. We’ve got a lot more functionality coming soon!

Release Notes & Known Issues

  • Currently, mobile device users will not receive notifications regarding Clubs.  These will be coming in a future update.
  • Clubs cannot be deleted on PC or mobile
  • You cannot change club related privacy settings on Xbox.com. These will be coming in a future update.
  • You cannot view a user’s profile from a request to join or recommendation on PC or mobile
  • You wont be able to create or customize clubs on mobile devices
  • Club admins cannot manage reports on mobile devices
  • Cannot invite users to clubs on mobile

Introducing Xbox Looking for Group (LFG)

Need to find players to help unlock an Achievement, complete a weekly quest, or build a team?  Xbox Looking for Group allows you to post a “want ad” that other players can see and answer.  Each post can contain a description, number of players needed, and searchable tags.  You can tag a post with communication and playstyle preferences like “Mic required” or “Cooperative”, Achievements you want to pursue, or custom tags created by the community for each game.  Once you’ve found other players who share your goals, you can quickly and easily form into a party and get into your game.

We’ll be adding new features and upgrades to Looking for Group over the course of the preview, but we’re excited to get this build into your hands so you can start giving your feedback.

Note that because of the limited size of the preview program, you may not find Looking for Group posts for every game.  In general, we recommend trying out the Looking for Group with multiplayer games that involve co-operative gameplay or team competitions, such as:

  • Halo 5: Guardians
  • Minecraft: Xbox One Edition
  • Destiny
  • Call of Duty: Black Ops III
  • Tom Clancy’s Rainbow Six Siege
  • Overwatch: Origins Edition

 

Where to find Looking for Group

  • The easiest way to get started with Looking for Group is to open the Parties tab in the sidebar on the right side of the Xbox App. 
  • By clicking the “Looking for Group” button and selecting a game, can search for all LFG posts for a game on its Game Hub.  There you can search available posts using tags or create your own post.
  • When you Host or request to join an LFG post, you can find it in the “Upcoming” list on the Parties tab of the sidebar. 
  • If you’re the leader of a Party and need more players, you can also create a Looking for Group post directly from the Party roster.

 

Release Notes & Known Issues

  • LFG posts do not yet show up in user profiles, Clubs, or the friends list.
  • LFG posts can currently only be posted to all of Xbox LIVE, visible to everyone on each Game Hub’s “Looking for Group” page.
  • LFG posts can currently only be posted for “right now”.  These will timeout if you disconnect from Xbox LIVE for more than 5 minutes.
  • You can currently only host one LFG post at a time; however, you can request to join as many as you want.
  • On the Game Hub “Looking for Group” page, posts do not yet update in real-time. You’ll need to hit the refresh button or change your search tags to get fresh results.
  • On the Game Hub “Looking for Group” page, a maximum of 25 posts are shown.  This is a bug that will be fixed in a future update.
  • On the Game Hub, if you filter using a custom tag that contains a capital letter, no filtering will occur.  This is a bug that will be fixed in a future update.
  • On the Game Hub, all posts have the game art instead of the Gamerpic of the Host.  This will be addressed in a future update.
  • Currently, all LFG posts for a game are visible in all regions and languages.
  • Currently, custom tags cannot start with a number.
  • When creating an LFG post, creation will fail if your custom tags or description contain profanity. 
  • When creating an LFG post, you can only set a maximum of 5 players needed.  This is a bug that twill be fixed in a future update.
  • When selecting tags to create or search for an LFG post, Achievement tags are not yet sorted by whether they are locked or unlocked.
  • Currently, Party Chat sessions linked to an LFG post can be joined (without having to use Looking for Group) if the Party is set to “Joinable”.
  • The Posted Time is not yet shown when browsing LFG posts on a Game Hub; however, posts are sorted with the newest ones at the top.
  • To see or create LFG posts, your privacy & online safety setting, “Others can communicate with voice, text, or invites” must be set to “Everybody”.  Don’t worry, Looking for Group will let you know if this setting isn’t set to “Everybody” and will help you change it if you like.
  • You cannot currently view previous LFG posts.
  • Currently, only Xbox LIVE-enabled games support LFG posts.
  • Currently, LFG posts for Xbox 360 games will not show up in the Game Hub, but will be visible to you and your friends in the Party tab of the Guide.
  • Currently, when choosing a game for a Looking for Group post, search may not find a game if you haven’t played it before.

zoltán farkas (wilczek) say:

Hi Andrew,

I had the same problem, but I found a solution, maybe it will help you as well.

In my case the problem was that when VS2017 wanted to deploy the UWP app locally it was reading files from the build folder of another UWP app.

Here are the steps I used:
1. Attach a VS debugger on the VS instance (devenv.exe) you want to use to deploy the app.
2. In the VS instance you attached to devenv.exe open "Exception Settings" (Ctrl+Alt+E) and in the search field type "XmlException".
3. Check the checkbox in front of the System.Xml.XmlException entry.
4. Now try to deploy the UWP app in the other VS instance and wait for the exception breakpoint to be hit.
5. Now open the call stack and go back to the point where you can see the last method call in the Microsoft.VisualStudio.TailoredProjectServices.dll. If necessary load the symbols from the MS symbol server.
6. Double click on that entry and open "Locals" (Debug->Windows->Locals). There check if the folder being used belongs to another UWP app. If yes, go to that given folder and delete the entire "bin" folder.
7. Now if you try to deploy the app again it should deploy without any issues.

Best regards,
Zoltan
