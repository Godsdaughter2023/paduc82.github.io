To answer your question:

PS:  Suppose you have nested loops and you want to exit the outer loop from an inner loop.  Then for example:

Do While condition1
    Do
        ....
        If condition2 Then GoTo done
        ....
    Loop Until condition3
Loop
done:

The days of goto-less code ended decades ago, and even Dijkstra recanted his categorical "goto's are dangerous" position.  Generally, structured statements are preferred, and goto's should be used sparingly.  But there are times when overuse of nested structured statements obfuscates the design and is grossly inefficient, and the use of a few "structured" goto's is acceptable.

tasos_k say:

Do you want to sort a filtered area ?

try this..

Sub Sort_02()
'Jun 26, 2015

On Error Resume Next

Dim MyWorkSheetRangeA As String
Dim MyWorkSheetRangeB As String
Dim MyWorkSheetRangeC As String
Dim MyColumnRange As String

Dim r As Long

Dim ws As Worksheet
Set ws = ActiveWorkbook.Worksheets("detail")
ws.Select
'ActiveSheet.ShowAllData << disable this line

r=ws.Cells(Rows.Count,"A").End(xlUp).Row

MyWorkSheetRangeA = "E2:E" & r
MyWorkSheetRangeB = "A2:A" & r
MyWorkSheetRangeC = "B2:B" & r

MyColumnRange = "A1:H" & r

With ActiveSheet.Sort
.SortFields.Clear
.SortFields.Add Key:=Range(MyWorkSheetRangeA) _
, SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
.SortFields.Add Key:=Range(MyWorkSheetRangeB) _
, SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
.SortFields.Add Key:=Range(MyWorkSheetRangeC) _
, SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
.SetRange Range(MyColumnRange)
.Header = xlYes
.MatchCase = False
.Orientation = xlTopToBottom
.SortMethod = xlPinYin
.Apply
End With
End Sub
