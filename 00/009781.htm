The way that first came to mind was to insert a cross reference. But that requires something to "refer to".  A heading or a bookmark or something. The problem is that the concept of a "page" in word is very fluid.  "Stuff" moves from one page to another as you edit the document. That is the whole point of "word processing". You worry about content, the program takes care of putting it on logical and physical pages. 

There is another simpler option. Just use the "GOTO" function, shortcut <CTL><G>.  When you invoke it, you see the GoTo dialog which includes the "go to what" category. graham mayor say:

A footer, by implication, applies to several pages and there can be up to three different footers for each section of the document. Having a link in the footer is inappropriate because you would have to open the footer range in order to access the link and in any case footers are not tied to individual pages. Your plan therefore does not work.

You would have to put separate links on each page and they would be subject to movement when the document is edited, so good luck with that.

It might be appropriate to put a button on the QAT or Ribbon to run a macro to determine where the cursor is and then select the page as appropriate e.g.

Sub JumpToPage()
'Graham Mayor - http://www.gmayor.com - Last updated - 03 Jul 2017
Dim iPage As Integer
    iPage = Selection.Information(wdActiveEndPageNumber)
    Select Case True
        Case iPage >= 10 And iPage <= 20
            Selection.GoTo What:=wdGoToPage, which:=wdGoToNext, Name:="3"
        Case iPage >= 21 And iPage <= 30
            Selection.GoTo What:=wdGoToPage, which:=wdGoToNext, Name:="4"
        Case iPage >= 31 And iPage <= 40
            Selection.GoTo What:=wdGoToPage, which:=wdGoToNext, Name:="5"
        Case iPage >= 41 And iPage <= 50
            Selection.GoTo What:=wdGoToPage, which:=wdGoToNext, Name:="6"

I found the solution to this problem in the meantime.

After some testing, I found out that my name appeared wrong only if the message didn't contain any special characters (like 'á', 'é', etc.). If the message contained any of these characters, my name appeared correctly.

It turned out that Outlook would choose the wrong encoding for these messages. So the solution was to make Outlook send all messages in UTF-8 encoding instead of choosing the encoding automatically.
