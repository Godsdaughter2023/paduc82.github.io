omniplex say:

Im guessing its most likely a registry setting bit or corrupted desktop.ini.

    

On my box, where explorer works as it should for all mounted volumes, there is no C:\DESKTOP.INI in the root directory of C:.  I've restarted explorer with ShowHidden and ShowSuperHidden, i.e., I can see both the public (all users) and my own DESKTOP.INI on the desktop, and I see no DESKTOP.INI on C:.  If you have a C:\DESKTOP.INI you can delete or rename it to see if that helps.  

I used REGEDIT to check HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\MountPoints2\C.  The key exists, there are no values under this key on my box.  There is no corresponding Explorer\MountPoints2 under HKLM (HKEY_LOCAL_MACHINE).  If your problem is really a registry problem limited to which volumes explorer shows on "my computer" it should be somewhere below one of these two (HKCU and HKLM) ...\CurrentVersion\Explorer keys.

You can get the GUID (wannabe-globally unique ID) for all mounted volumes with MOUNTVOL on a command line.  The output will be something like this (but different on your box):

    \\?\Volume{e553a5f9-6c65-11e0-93f0-806e6f6e6963}\
        C:\

The GUID is the stuff between the curly braces, you could search for all occurences of {e553a5f9-6c65-11e0-93f0-806e6f6e6963} in the registry, hoping to find something unusual.  Trying that on my box I found the GUID in the Explorer\MountPoints2 and BitBucket keys for various users, in a bunch of keys below VSS (presumably lists of restore points), and in one global list of mounted devices.

Unfortunately you can't click on the properties for C: if C: does not show up. ;-)  But while you wait for better answers you could open a command line (elevated CMD.exe with admin rights), change the working directory to C:\ with "CD /D C:\", and START . (internal command start with argument dot).  The default action for any "started" folder including . (dot) is to open explorer, you should see the content of the C: root directory.  

On my box the path (or address) in this explorer window is C:\ below "Computer".  
On your box it might be different in some interesting way, because you have no C: in "Computer".
