bossdweeb say:

"...random Windows restart requirements"

Windows has no random restart requirements, but restarting is often required to complete various software and

update installs. It's also advisable to restart you machine occasionally just so Windows can sort out things such

as changed settings and clear caches, etc.

If you leave you machine on, and connected to the internet all the time it will likely reboot after a Windows Update

(not all updates require reboot).

I don't use Sleep, but believe that Windows Updates can wake the machine from sleep. You could modify this

behavior by blocking Windows updates for up to 12 off duty hours via the "Change Active Hours" option in

Windows Updates settings. Set it up so the active hours are not in a time frame that you would normally have

the machine in sleep mode.

- While walking around with the machine in your backpack there could be other issues.

- Loose lid causing the machine to boot ?

- Is there a Start on LAN, or other non power button/lid boot option in the machines BIOS ?

- There is an option in Control Panel > Power > Advanced Power Settings that enables a sleep wake timer,

which may be waking your machine (never used it - just guessing). Is this disabled ?

- If you have 'Ready Boot' enabled shutting the machine down is much the same as the old hibernate mode, so

it should come back even more quickly than a full reboot from your SSD, but Ready Boot can have problems,

so with the quick boot times of an SSD why not just shut the machine down properly ?

Sleep mode is always going to have potential for 'random' reawakening.

cmdrkeene say:

It's probably an issue with it reading the browser's User Agent String and not being prepared for what it finds since it was changed in IE11.


Long story short: because IE 5-8 were basically awful, most websites look for the IE user agent and send different code to it than it would send to another browser like Firefox or Chrome.


IE 9, 10, and 11 have gotten much better at rendering sites correctly with standard code, but most sites continue to send outdated IE-specific code anyway, thus causing issues. Microsoft tried to get around these issues by changing the IE11 user agent string. In most cases this works fine, and you get the same experience in IE11 as you would in another browser. But in some cases the site can't figure it out.
