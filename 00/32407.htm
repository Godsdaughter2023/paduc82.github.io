Hello Kaustubh,

Noting the details that you have provided.

Generally speaking, the most common reason for this error is due to a misconfigured bootloader file. While installing Ubuntu in Windows, a part of Grub lives in the Master Boot Record (MBR), while the other parts live in the Linux boot partition. That is to say, although the Ubuntu partition is deleted, Grub is still in MBR. In addition, the active partition is not set correctly. As a result, on the startup of operating system Windows 10, you receive such an error of no such partition grub rescue.

To resolve this, let us go ahead and try to run the automatic Startup Repair feature using the Windows installation media that you have. If you do not have the Windows 10 installation media, you can click on this link to download it first. Afterwards, you can follow the steps below on how you can perform the Startup Repair:

  1. Boot your computer using the installation media by inserting the DVD or USB flash drive and restarting your computer. 
  2. Once you see a message saying Press any key to boot from DVD, press Enter. (If you do not see this message, you might need to change the boot priority order in your computer’s BIOS settings to have the DVD as the first boot priority.) 
    • Note: Modifying BIOS/complementary metal oxide semiconductor (CMOS) settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of BIOS/CMOS settings can be solved. Modifications of the settings are at your own risk. 
  3. Once you see the Install Windows page, click Repair your computer
  4. On the Choose an option screen, select Troubleshoot. 
  5. Click Advanced options. 
  6. Click Startup Repair.

Once the repair is finished, you should be able to boot your computer as normal. However, if the issue persists, we recommend that you reset your computer. The link below would give you the steps on resetting your PC without having to lose your personal files saved on your computer:

Possible??

Yes, it's possible.

You probably wanted to ask HOW to do it. Try this:

  1. Save the code below on your desktop as Camera.vbs. The .vbs extension is essential!
  2. Grab any folder containing pictures with your mouse and carefully drop it on Camera.vbs.
  3. Examine the desktop file Tags.xls. It contains a recursive list of all .jpg files plus the camera that you used. You can now sort the list by camera type.
'Extract the camera name from picture meta data
'24.7.2017 FNL
'----------------------------------------------
Const Camera = 30
Set oFSO = CreateObject("Scripting.FileSystemObject")
Set oShell = CreateObject("Shell.Application")
Set oWshShell = CreateObject("WScript.Shell")
sTarget = oWshShell.SpecialFolders("Desktop") & "\Tags.xls"
Set oTarget = oFSO.CreateTextFile(sTarget, True)
oTarget.WriteLine "Picture Path" & vbTab & "Camera"

ProcessFolder(WScript.Arguments(0))
oTarget.Close

Sub ProcessFolder(sFolder)
    Set oFolder = oShell.Namespace(sFolder)
    For Each oFile In oFSO.GetFolder(sFolder).Files
        If LCase(Right(oFile.Name, 4)) = ".jpg" Then
            sCamera = oFolder.GetDetailsOf(oFolder.Parsename(oFile.name), iCamera)
            oTarget.WriteLine oFile.Path & vbTab & sCamera
        End If
    Next
    For Each oFldr In oFSO.GetFolder(sFolder).SubFolders
        ProcessFolder(oFldr.Path)
    Next
End Sub
