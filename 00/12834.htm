How can I get Yammer to remove non-active users automatically? - Microsoft Community

mikebaker7910 say:

How can I get Yammer to remove non-active users automatically?

People are complaining that they are getting spammed by Yammer recommending contacts who have left the company. We do not immediately delete users when they leave, we deactivate them for a period of time (usually 90 days) before we delete them. How can I get Yammer to remove non-active users automatically (I cannot afford for this to be a manual process)?

mylenealluvida_msft say:

Hi Mike,

We understand the importance of this inquiry and we would be glad to address this matter.

Are you referring to “Suggested People” list? If so, here’s how “Suggested People” lists are being pulled out:

“Suggested People” list uses an algorithm that suggests people for you to follow. It is based on who you already follow, who they follow, etc.

Yammer serves up two suggestions at a time of people you may want to follow. The first suggestion is an active user on the network whom you can follow. By following people who are relevant to you, you will see more relevant content in Yammer. The second suggestion is a pending user or imported contact who has not yet joined the network. Users will receive an email letting them know you are following them, and encouraging them to log-in. Therefore, if a former member or mailing list have been invited to the network and are being stored as pending users, those addresses are eligible to show as the second suggestions in Suggested People. Even if you mark a suggestion as a non-employee or mailing list, if the address is sent a new invitation or if a pending account exists for that person under another alias, it can appear in Suggested People.

Pending users are users that have been invited to Yammer but have not completed the signup process. If the email address is incorrect, then the signup process will never be complete. Pending users will indeed appear on the mentioned list unless removed through bulk update.

“Pending users” may be due to the following reasons:
1. A user might be synchronizing contacts to Yammer which has the incorrect email address.
2. A user may be inviting the user’s outdated/incorrect email address.
3. Address book upload with the outdated/incorrect email address.
4. If someone imports their address book from their mobile device, those users are stored as "Pending Users" and are suggested to current members of the network as suggested people.
These are all user initiated actions. Yammer does not have a feature that will automatically invite users. It is highly recommended to educate users within the network on the reasons stated above to help avoid this. Otherwise, a bulk update is necessary to remove these pending users from Yammer.

We definitely understand how troublesome it is to keep deleting these users when they appear on the suggested people list. So our current solution to avoid former members to be shown as Suggested Users, the admin of the Yammer Network must make sure to periodically do a bulk upload to the "Suggested People", based on the latest list of valid Yammer network users. You need to monitor pending accounts.

To determine which accounts are pending, you would perform a user data export. When you have the file, you'd sort by accounts there that do not have joined_at nor deleted_at/suspended_at dates. These are your pending accounts.

Hope this helps. Please let me know if anything is unclear.

Access 2007 - Microsoft Community

keke_m say:

Access 2007

Is it possible to convert a query to a table?

hansv mvp say:

Open the query in design view.

On the Design tab of the ribbon, in the Query Type group, select Make Table.

Specify the name for the table that you would like to create, then click OK.

Still on the Design tab of the ribbon, in the Results group, click Run.

Access will ask for permission to create the table. Click Yes.

If you save the query and run it again later, Access will ask for permission to overwrite the existing table.

scottgem say:
Its possible, see Hans' reply. But can you explain why you want to? While there are a few good reasons to do so, a query functions the same way as a table, so it is usually unnecessary.

ken sheridan say: Like Scott, I'd first ask why.  As an example of when it's not valid to do so, and when it is, a few years back I helped a friend in California with a price list database she was developing for the local office of a large printing supplies company.  The no need to save the result table of the query as a base table; whenever the report was opened it showed the current price list.

However the company's sales staff could adjust prices and/or margins at their discretion for specific customers, and the brief included a requirement for these adjusted price lists to be saved and date/time stamped.  There could be several generated within a single day as negotiations between the sales staff and the customer progressed.  For this a base table was generated from the query and saved under a name which included the current date/time and the CustomerID.  Consequently the report could be opened at any time to show the price list generated for a specific customer at a specific date/time, simply by setting the relevant table as its RecordSource.

So unless you have a situation along the lines of the second scenario above, where the values returned by the query need to be fixed for possible later use regardless of subsequent changes to the base data, generating a base table from the query's result table is unnecessary.
