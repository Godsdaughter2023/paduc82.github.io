Hi,
 
Cause for this error message.
 
1.The Web server (running the Web site) thinks that the HTTP data stream sent by the client (e.g. your Web browser) contains a URL that is simply too large i.e. too many bytes.
 
2. Typically Web servers set fairly generous limits on length for genuine URLs e.g. up to 2048 or 4096 characters.    If your URL is particularly long, you can usually try shorter variations to see roughly where the limit is. If    your long URL is indeed valid, then the Web server may need to be reconfigured to allow your URLs through. Understand that Web servers have to set some reasonable limit here, because they have to deal with badly programmed clients trying to give them huge URLs.
 
This can be fixed by:
 
1.This error seldom occurs in most Web traffic, particularly when the client system is a Web browser. The URLs in this case are typically standard hyperlinks found on Web pages.
2.These links tend to be too large if they are simply wrong i.e. the Web page containing the link has been badly coded.
3.If your client system is not a Web browser, the problem can only be resolved by examining what the client is trying to do then discussing with your ISP why the Web server rejects the size of the URL sent by the client   system.

  If Application.CustomListCount > 4 Then

Any PC with a language other then english has at min. 8 default custom lists!

Run the sub below, after the first run you'll see that your custom list is increased.

Sub Test()
  Dim i As Integer
  Dim Arr, MyList
  With Application
    'Get current amount of custom lists
    i = .CustomListCount
    'Try do add a cusotm list
    MyList = Array("cogs", "sprockets", "widgets", "gizmos")
    'MyList = .GetCustomListContents(1)

    If i = .CustomListCount Then

      For i = 5 To .CustomListCount
        If StrComp(Join(MyList), Join(.GetCustomListContents(i)), vbTextCompare) = 0 Then

We've had exactly this problem and I've spent several hours trying to sort it. I found out that the problem has nothing to do with missing name and DOB despite the error messages telling me that. The problem is that the billing region on my account (the parent account) is US and in the US they insist on a credit card payment to verify that you are over 18. The catch is that it doesn't recognise a UK credit card and despite giving UK address details it thinks you are in the US. Aaaaargh!

There are threads on here that say you can change your billing address region since Jan 2013 but we didn't manage to get that to work despite help from online support. In the end I created a new parent account choosing UK as the region. Then did a factory reset on the phone and set up the child account as UK too. This seems to have worked.

Hope this helps others with same problem.
