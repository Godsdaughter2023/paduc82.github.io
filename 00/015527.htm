You do not need to reference a table name in the WHERE clause of the OpenForm or Open Report methods. The fieldname does have to be in the Recordsource of the Form or Report object. So the Syntax is:

"Fieldname = value"

"fieldname = " & variable

where variable will be the name of a variable containing the value or a reference to a control on an open form. it could also be a Tempvar. Or it could be hard coded.

While using 'Name' as a variable in a worksheet macro, excel see it as the name property of the worksheet which you change by the statement. You should always avoid using variable names which are reserved for use in the VBA language.

Sub cmdBlank.Click()()
Dim DestCell As Range
Dim MyName As String
Sheets("Supplier").Select 'Not needed Supplier worksheet is always the active sheet when user hit the button.

With Sheets("All Orders")

    If .Range("C3") = "" Then
        Set DestCell = .Range("C3")
    Else
        Set DestCell = .Range("C2").End(xlDown).Offset(1)
    End If
End With

DestCell.Value = MyName
ordernum = DestCell.Offset(0, -1)
With Sheets("Blank Order")
    .Range("A7") = MyName
    .Range("I7") = ordernum
