ken sheridan say: You can write a simple little function:

Public Function RemoveExtension(varInput, ParamArray varTokens())

    Dim varToken As Variant
    Dim intTokenPos As Integer
    Dim RetVal As String
    
    If Not IsNull(varInput) Then
        RetVal = varInput
        For Each varToken In varTokens
            intTokenPos = InStr(RetVal, varToken)
            On Error Resume Next
            RetVal = Left(RetVal, intTokenPos - 1)
        Next varToken
    
        RemoveExtension = RetVal
    End If
    
End Function

You can see how it works in the debug window:

? RemoveExtension("98765-16","-",".")
98765
? RemoveExtension("98765.16","-",".")
98765
? RemoveExtension("98765","-",".")
98765

So in a query you'd call it like this:

UPDATE [Table 1]
SET IDNumber = RemoveExtension(IDNumber,"-",".");

You can of course pass as many token values into the function as necessary should there be others than "-" and "."

john w. vinson say:

Do you want to permanently and irrevocably remove the extension from the field in Table one? Or just ignore it for the purpose of joining the two tables? Either can be done but the consequences are obviously different!

The expression you want to extract the untrimmed value is

Left([IDNumber] & "-", InStr([IDNumber] & "-", "-") - 1)

The spinach with concatenating "-" is to allow for the possibility that some values in Table one might not HAVE an extension.
