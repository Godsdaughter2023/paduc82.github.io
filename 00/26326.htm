john mcghie say:

What do you regard as a "Section"?

If it is text between two section breaks, Word considers that a section.

However, human beings consider a section to be the text under a heading.  And you can move those around simply by dragging in Outline View.

Outline View is the most powerful editing tool in Word: it's what sets Word apart from the also-rans. To use it, first make a copy of your document: until you get the hang of it, it's possible you might get one or two 'accidents' so have a backup :-)

Now:  Outline View works on the Outline Level of the paragraphs.  You could set these manually, but I would hate to imagine how much work you would make for yourself doing that.

The built-in styles "Heading 1" through "Heading 9" have their outline levels properly set.  So use those for your headings (you can change any other formatting in them to your tastes). To get Outline View to work properly, you must format each heading with one of the Heading series of built-in styles.

Now hop into Outline View.  Ignore the appearance: Outline View is designed not to be WYSIWYG, it's a structured editing view where the appearance of the text bears little relation to how it will print.

When you select a Heading 2 paragraph, the selection will automatically extend down to but not including the following Heading 2 in the Document.  If you select a Heading 3, same thing: you will select all the way to the following Heading 3.  To move whole chapters, select the Heading 1.

Now:  Simply drag, and drop it where you want it.  The heading and all its subordinate text will come too.

As John suggested, it's critical to turn on your non-printing characters, so you can see what you are doing, otherwise you could drop whole chapters on the middle of a paragraph of text.

As far as the section breaks are concerned, they will be included in the selection, but you will probably need to fix them up later.

I would also ask why you have "hundreds" of section breaks in the document?  That suggests to me that you are not using some of the power features of Word that you should be using on a document of that size.  It also suggests to me that that document is likely to fail quite soon.  Sections breaks are very complex formatting containers, and the more of them you have in a document, the sooner it will fail. I try never to have more than one per chapter: but you need to use more if you are printing landscape pages.

If you are using section breaks to adjust headers and footers, I can show you a way to do that without all the section breaks :-)

With a little practice, you will find you can completely restructure a document of thousands of pages (up to about 2,000 pages in Mac Word: PC Word will go higher) in a few minutes, just by dragging the headings around in Outline View.

Hope this helps: Don't hesitate to ask for more information if you need it.  Remember: the key is to use the built-in Heading styles. You can set the Outline Level in any style, if you have not used the built-in styles.  But you need to work carefully: only headings should have an outline level, everything else should be Body Text level.  And there should be only one style at each level, forming an unbroken chain from 1 down to 9.  It's usually quicker to use the built-in Heading styles, which have the correct levels already baked into them.

Cheers

Hi David,

The problem looks like a font cache issue.
Go to the ~/Library/Preferences/Microsoft folder and in there, find and trash the Office font cache.
Now if you launch Word for instance, it should create a new font cache and hopefully the problem will be gone.
As for the preferences, there are 2 locations. Each app have their own set of preferences (starting with com.microsoft.) in ~/Library/Preferences, but Outlook is a bit different.
The data for Outlook (and the settings for the accounts, etc) is stored in ~/Documents/Microsoft User Data.

If the corruption spreads from one account to the other, then it could be the System cache that's corrupted. There are tons of utilities out there that can clear it for you (e.g.: Onyx). Clear the cache and reboot.
Rebooting pressing the Shift key down (until you see an indication that you are safe-booting) will also clear the cache.
After that, launch /Applications/Font Book and resolve duplicates (an additional frequent source of font problems).

I hope it helps :-\

graham mayor say:

It is difficult to validate a dropdown form field because the macro necessary has no way of telling whether the selected item in the dropdown is correct or not.

One possibility, assuming the users can be bothered to select their own names from the dropdown, would be to use conditional word fields (not form fields) to automatically fill the remaining data based on the original selection. e.g. assuming you have three entries Sarah, Graham & Bill.

If the users are each logged in to their own PC and are using the form on their own behalf, it should be possible to abandon the dropdown and select the user information freom the user's profile, but for the moment, let's stick with what you have.

The first line of field codes will enter the job titles.
The second line of field codes will enter the e-mail address
and the third line will insert a graphical representation of the signature in the form of a jpg file named after the dropdown entry eg Sarah.jpg (and here called from the D:\Temp folder.

All field codes are inserted with CTRL+F9 for each of the field bracket pairs {}

The first two lines only need the calculate on exit checkbox of the dropdown field to be selected in order to update the field when you tab out of the dropdown field. The IncludePicture field will require updating by macro run on exit from the dropdown field. I would suggest using the sample code at http://www.gmayor.com/installing_macro.htm  The form should be saved as a template and new documents created from it.

{ IF { REF Dropdown1 } = "Sarah" "Programmer"}{ IF { REF Dropdown1 } = "Graham" "Helper"}{ IF { REF Dropdown1 } = "Bill" "Minion"}

{ IF { REF Dropdown1 } = "Sarah" "*** Email address is removed for privacy ***"}{ IF { REF Dropdown1 } = "Graham" "*** Email address is removed for privacy ***"}{ IF { REF Dropdown1 } = "Bill" "*** Email address is removed for privacy ***"}

{ IncludePicture "D:\\Temp\\{ Ref Dropdown1 }.jpg" }
