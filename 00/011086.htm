    Case Is < 18

    Case Else
      ActiveCell.Offset(0, 6) = "?? currentAge = " & currentAge
  End Select
  ActiveCell.Offset(1, 0).Activate
Loop

I like the Select Case block more than the many If Else ElseIf structure because it give you the catch-all ability of Case Else - where a value or problem you didn't expect injects itself.  You can always leave it out if you have really convinced yourself all bases are covered, although I'll usually leave it in even then, but with no statement under it as a do-nothing to just show that I think I've covered all bases, like this:

   Select Case ValueOfOne

     Case Is = 1

         msgbox "All is still right with the universe"

     Case Is = 2

         MsgBox "Bend over, place your head between your knees..."

     Case Else

         ' probably all over by now, do nothing

  End Select

Notice in the logic for this Case structure, we start off with the highest numbered group and ask if the value is >= to that.  Since only a true statement will be executed, then just those work, but if the age is less than 60, then the others are tested in turn until one of them is found to be true.

In the setup I've provided, the Case Else is going to tell you what the value of currentAge is when it seems to have failed by placing the "?? currentAge = " along with the value of currentAge out in column J.
