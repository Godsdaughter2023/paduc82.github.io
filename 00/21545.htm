greginmich say:

I tried to do this update but I keep getting:

Definition Update for Windows Defender - KB2267602 (Definition 1.223.1683.0) - Error 0x80070643

When Windows Defender definition updates fail with the 0x80070643 error code, this usually means that Windows Defender didn’t get switched off in its registry key when a third-party AV app was installed. The technical details for this issue can be found here:

http://answers.microsoft.com/en-us/windows/forum/windows_10-update/definition-update-for-windows-defender-kb2267602/d4edd314-a438-4ac2-b7a9-dc1c45a1ef62

This is an extremely common issue, but we’re unfortunately not making any headway with it because threads keep getting diverted and deadlocked by people who insist on treating it as a Windows Update issue. This is not a Windows Update issue. It’s an issue with the registry switches that disable Windows Defender. So the quick and easy fix is to just download and run the Turn_Off_Windows_Defender.reg file, which will disable Windows Defender in the Group Policy registry key:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html  

On the other hand, if you’re seeing this error code when there isn’t any third-party AV app installed, then it's possible that this might actually be a Windows Update issue – so you should download and run the Update Troubleshooter:

http://windows.microsoft.com/en-us/windows-10/windows-update-troubleshooter

jay freedman say:
The reason that option is in the 'Set proofing language' dialog is that the proofing language and the 'Do not check' are properties of individual characters, not of the document as a whole. You could mark a single word, or even one or a few characters within a word, as having a different language or as not being subject to spelling checking. (It really doesn't make much sense to mark a few characters, but the program has only the fuzziest idea of what a 'word' is, and a whole paragraph is too coarse as a minimum.)

A typical example is a short story or novel in which the characters' names won't be in the standard spelling lexicon. You could make a character style whose only nondefault setting is turning on the 'Do not check spelling' option. Applying that style to the names would prevent them from being marked as misspellings, while allowing checking of the rest of the text.

If some of the document is marked as 'Do not check' and there are no misspellings in the rest of the document, you'll get that 'check complete' message -- because it is complete. Previous versions of Word included the statement 'Text marked with "Do not check spelling or grammar" was skipped.' and we used to get a regular parade of users asking 'What the #%^& does that mean??'
