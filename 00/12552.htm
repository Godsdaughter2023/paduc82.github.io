elemantal say:

davidf2 say:

I have a multipage Publisher file which I think doesn't have page numbers. When I create a PDF for two of the pages, the PDF says "(1 of 2)" but also shows "Page 10" from the original file.

Other than breaking up my Publisher file into separate files for each subset of PDF pages I want to create, how can I create PDFs that would show Page 1 instead of its page number from the original document?

You cannot print a subset of pages using the built-in Office PDF tool in Publisher so that says you are using a different PDF tool to create the PDF.

You also cannot automatically set Publisher to show 'Page 1 of 2" or '1 of 2' which also suggests you are using a different PDF tool.

Both these things also suggests that you should look to the options within the PDF tool you are using to turn off the automatic numbering of the PDF pages. If you cannot find the options to control this numbering then use a different PDF tool as suggested by Elemental. I prefer PrimoPDF myself.

You can insert page numbers into the footer or the header of a Publisher 2010 document two ways. Insert > Page Number and drop down to choose the location on the page where it will appear. Or you can View > Master page and click in the Header or Footer and insert page number. You can also type around the '#' symbol and manually create 'Page # of 2' or some such if you want. And finally it is on the Master Page view that you can select the '#' and delete it.

DavidF

davidf2 say:

Since Publisher seems to be designed to create a catalog/newsletter, is there a way to use it for what I'm seeking which is individual documents?

Can multiple publisher files share the same Master Page settings and styles so that a change in one place would be reflected in other documents when they were opened the next time?

You can link to Excel data and such and this will update as you change the data in Excel, but there is no Master Page settings that will automatically update across multiple publications to my knowledge.

You can create and save custom templates which might help you accomplish what you want to do. Here is a link for Pub 2003 that describes the process in general: http://office.microsoft.com/en-us/publisher-help/create-branded-templates-for-your-business-from-scratch-in-publisher-HA001097229.aspx?CTT=1

and here is a link for templates using Pub 2010: http://office.microsoft.com/en-us/publisher-help/find-create-or-change-a-template-in-publisher-HA101846238.aspx?CTT=1

One feature you might look at that might help is the ability to save 'building blocks' that are available from all Publisher publications. Create a text box, format it as you want and save it as a building block and you can insert that text box as a 'page part' into all new publications. This saves some time.

Reference: Use building blocks to quickly create a publication: http://office.microsoft.com/en-us/publisher-help/use-building-blocks-to-quickly-create-a-publication-HA101823829.aspx

I don't know if that answers your question or not, but perhaps it will lead to a solution.

By the way, one quick workaround for obscuring page numbers showing from the Master Page is to simply create a small text box or rectangle object and fill it with the same fill color as the background; usually white, and then drag that text box over the page number. When you are done you can simply delete that filled text box or drag it into the scratch area to use again later. If you are only printing a few pages from within the publication this might be faster than moving the page(s).
