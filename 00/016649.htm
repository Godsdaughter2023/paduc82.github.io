jeeped say:
You can quickly make a pseudo-button out of a cell by using the sheet code module's Worksheet_BeforeDoubleClick event macro. For example, if you wanted to run MyMacro when cell A1 was double-clicked, it would look something like the following.

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    If Target.Address = "$A$1" Then
        Cancel = True
        Call MyMacro
    End If
End Sub

The Cancel = True statement stops the user from entering in-cell-edit mode which might be set in the user's options.

Sway storage--Can I create folders to organize my Sways? - Microsoft Community

joseph valentine esq say:

Sway storage--Can I create folders to organize my Sways?

I'm planning to create many Sway "modules" that I will use in my weekly classes. Because there will be many, I would like to organize them in folders, as I do in Windows Explorer. Can that be done?
rohn007 say:

If you don't use Jason's suggestion and want to keep the modules as separate files to make them easier to re-use there is another option.  Use "prefixes" on the file names to simulate folders.

So, since you can't create a "folder1", just add that to the start of the module name, ie:

folder1- module1

folder1- module2

folder2- module1

folder2- module2

yes, it is unwieldly, but it gets the job done.  

I use a system of keyword prefixes along with dates IN the file name to group files by sub-subject and date within a folder.

jason (office engineer) say:

Hi Joseph,

Sway doesn't support grouping or folders yet.  However, one suggestion I have is to make use of the ability to embed them.  Make a master Sway for each module and then embed all the Sways for that module in that master Sway.  Depending on how you want them to be viewed, this could be a very effective way to organize them.  If the organization is more for you, the author, I would suggest making the master Sways be in the vertically scrolling layout.  If the organization more for the viewers, the layout optimized for presentation might be optimal as one Sway would show on the screen at a time and the viewer can just move through each Sway, one at a time.

Hope this helps!
