ronrosenfeld say:

It can be difficult to define what is "similar'.  And I'm not sure how you need your results reported.

 Here's one approach which computes the levenshtein distance between the subject cell and the rest of the range.  It reports the cell address and contents of the most similar, provided that you don't need to replace all of the contents of one to get the other.  It ignores exact duplicates, as you indicate you already have that in hand.

To enter this User Defined Function (UDF), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this User Defined Function (UDF), enter a formula like

B2:    =minld(A2,$A$2:$A$14)

and fill down as far as required.

=========================================
Option Explicit

'use one or the other of the next two lines depending on
'if you want a case sensitive comparison
'Option Compare Binary   'For case sensitive comparison
Option Compare Text   'case INsensitive comparison
Function MinLD(s As Range, r As Range) As String
    Dim c As Range
    Dim LVD As Long
For Each c In r
    LVD = Abs(Len(s) - Len(c))
    If c.Address <> s.Address And c <> s Then
        If LD(s, c) <= LVD Then
            MinLD = c.Address & ": " & c.Text
            LVD = LD(s, c)
        End If
    End If
Next c
End Function

Public Function LD(ByVal s As String, ByVal t As String) As Long
Dim d() As Long ' matrix
Dim m As Long ' length of t
Dim n As Long ' length of s
Dim i As Long ' iterates through s
Dim j As Long ' iterates through t
Dim s_i As String ' ith character of s
Dim t_j As String ' jth character of t
Dim cost As Long ' cost
 
  ' Step 1
  n = Len(s)
  m = Len(t)
  If n = 0 Then
    LD = m
    Exit Function
  End If
  If m = 0 Then
    LD = n
    Exit Function
  End If
  ReDim d(0 To n, 0 To m) As Long
 
  ' Step 2
  For i = 0 To n
    d(i, 0) = i
  Next i
 
  For j = 0 To m
    d(0, j) = j
  Next j

  ' Step 3
  For i = 1 To n
    s_i = Mid$(s, i, 1)
   
    ' Step 4
    For j = 1 To m
      t_j = Mid$(t, j, 1)
     
      ' Step 5
      If s_i = t_j Then
        cost = 0
      Else
        cost = 1
      End If
     
      ' Step 6
      d(i, j) = Minimum(d(i - 1, j) + 1, d(i, j - 1) + 1, d(i - 1, j - 1) + cost)
   
    Next j
  Next i
 
  ' Step 7
  LD = d(n, m)
  Erase d
End Function

'*******************************
'*** Get minimum of three values
'*******************************

Private Function Minimum(ByVal a As Long, _
                         ByVal b As Long, _
                         ByVal c As Long) As Long
Dim mi As Long
                         
  mi = a
  If b < mi Then
    mi = b
  End If
  If c < mi Then
    mi = c
  End If
 
  Minimum = mi
                         
End Function

possibly when excel doesn't have the focus, it doesn't have priority access to resources.

in VBA, a person might do

 

sheet1.Activate

Range("b1").Select

Selection.Value = 12

 

but the better way would be

 

worksheets("Sheet1").Range("B1").Value = 12

 

the first way would cause the screen to flicker.  The second would not change the screen at all if sheet1 wasn't the active sheet.

Application.Screenupdating = False

These examples are in VBA, but you should be able to do something similar from you application.
