minorgeek say:

when you runsfc (system file checker), does it complain about tcpmon.ini as corrupt and unfixable?  here's a solution.

i found a workaround for this same problem.  seems to be a vista bug (since many have reported it).  i have no idea what caused the problem.  seemed to appear after i did a norton ghost restore, which i've done before with no problems. 

problem in short:  runningsfc.exe results in it complaining that tcpmon.ini is corrupt and it can't fix it 'cause the backup is also corrupt.  [corrupt means the hash doesn't match.  hash is a "summary" kind of thing...think of it as looking at the file as  a sequence of numbers and adding them all up modulus 2^32 (32 bits long)]  sfc = system file checker, which checks all the system files.  if it finds any corrupt, it replaces them, perhaps fixing whatever's wrong. 

workaround: restore the original files from your installation disk.  details:
 open your vista install disc, open folder sources, and dbl click (open) fileinstall.wim.  that's a compressed archive (like .zip) but ms created a new format.  i found that7zip opens it quite well.  maybe other things, but 7zip is free and a great program for archiving, more formats, etc.  get it at the usual places; e.g., downloads.com, majorgeeks.com, tucows.com, etc.  within 7zip file browser, open what you need, find the file(s) you need, copy 'em out (extract), then copy them where they go in the system.  here are the ones i copied: (copies of tcpmon.ini and its manifest and deleting the manifest cache):
   \windows\system32\tcpmon.ini
   \windows\winsxs\x86_microsoft-windows-p..rtmonitor-tcpmonini_31bf3856ad364e35_6.0.6001.18000_none_d278d2aaa749970e\tcpmon.ini    [folder and file]
   \windows\winsxs\manifests\x86_microsoft-windows-p..rtmonitor-tcpmonini_31bf3856ad364e35_6.0.6000.16386_none_d04210aeaa5e863a
   \windows\winsxs\manifests\x86_microsoft-windows-p..rtmonitor-tcpmonini_31bf3856ad364e35_6.0.6001.18000_none_d278d2aaa749970
   \windows\winsxs\ManifestCache\6.0.6002.18005_001c11ba_blobs
[n.b. the second one above, copied into winsxs, is a folder containing a backup copy of tcpmon.ini.  copy folder and its content.]
