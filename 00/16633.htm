Bob

I don't have 2013 so I can only go by what is in 2010. This is a web app, isn't it? Web app objects are very limited in what you can do with them. All you get is what is available in the Ribbon

You can import the tables into a new, blank desktop database and they will come in as desktop tables. You can get to the properties sheet in the new DB. The other objects need to be saved as text. While in the web app, press Ctrl+G to open the immediate window and save each object as text. Then load those text files into the new DB.

Example:

SaveAsText  acForm,"CampaignDetails", "C:\Temp_C\CampaignDetails.txt"

Then open the text file in NotePad and change the 3rd line from:

PublishOption =2

-to-

PublishOption =1

Then go to your new db's Immediate window and type:

loadfromtext acForm,"CampaignDetails","C:\Temp_C\CampaignDetails.txt"


It's a lot of work, but it is the only way I know to convert a web app to a desktop db.
