tom van stiphout say:

Maybe you can have a criteria form where you prompt for the date. If you format that text box as Short Date you will automatically get the date picker.

Then based on that date in its AfterUpdate event you can set two hidden textboxes txtBeginDate and txtEndDate to the first and last day of the week.

Then you can have a query that "looks back" on those date fields and filters data accordingly.

ken sheridan say: You can easily get the start date of the week for any date by subtracting the value returned by the Weekday function from the data and adding 1.  So, in the debug window, if we use 4th July this year as an example, specifying Monday as the first day of week (NB dates are returned in UK format below of dd/mm/yyyy, but will be returned in whatever your local short date format is in your case)

dtmDate = #2014-07-04#
? dtmDate - Weekday(dtmdate,vbMonday) + 1
30/06/2014

To make life a little easier you can wrap this up in a little function:

Public Function WeekStart(intStartDay As Integer, Optional varDate As Variant)

    ' Returns 'week starting' date for any date
    
    ' Arguments:
    ' 1. intStartDay - weekday on which week starts, 1-7 (Sun - Sat)
    ' 2. vardate - optional date value for which week starting
    '   date to be returned.  Defaults to current date
    
    If IsMissing(varDate) Then varDate = VBA.Date
    
    If Not IsNull(varDate) Then
        WeekStart = DateValue(varDate) - Weekday(varDate, intStartDay) + 1
    End If
    
End Function

Using the function this time:

? WeekStart(vbMonday,dtmdate)
30/06/2014

To get the end of the week just add 6:

? WeekStart(vbMonday,dtmdate) + 6
06/07/2014

With this you can use a date picker without worrying about which day in the week the user picks, so you can either assign the values to hidden controls in the form as Tom suggests and reference these as parameters in a query; or you can open a report or form (or filter a currently open form) by building a string expression in code, referencing the control whose value you insert via the date picker, e.g. to open a report in print preview:

Dim strCriteria as String
Dim dtmStartDate As Date, dtmEndDate As Date

' get start date and day following end date of week
dtmStartDate = WeekStart(vbMonday,dtmStartDate)
dtmEndDate = dtmStartDate + 7

' build string expression by which to filter report
' formatting dates in ISO standard format to
' ensure international unambiguity
strCriteria = "OrderDate >= #" & Format(dtmStartDate,"yyyy-mm-dd") &  _
    "# And OrderDate < #" & Format(dtmEndDate,"yyyy-mm-dd") & "#"

DoCmd.OpenReport "rptOders", _
    View:=acViewPreview, _
    WhereCondition:=strCriteria

Note that the range is defined as on or after the start of the week, and before the day following the end of the week.  This is safer than a Between.....And operation as it caters for any OrderDate values which might inadvertently contain a non-zero time of day element, which you cannot confidently rule out unless you've taken steps to prevent such values in the table definition.
