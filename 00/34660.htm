How do I transfer unused memory from one partition to another? - Microsoft Community

davidhatch say:

How do I transfer unused memory from one partition to another?

I am running low on memory on my C partition and need to transfer unused memory from my D partition to the C. How, step by step, can I do this, p[lease!?

 

 

a. user say:

Hi, David,

 

Look in Control Panel > Uninstall Programs

Uninstall any applications you do not use to free up memory

 

Control Panel -> System -> "Advanced" tab -> "Performance" button -> "Advanced" tab -> "Virtual memory" button. In the new window, in the "Virtual Memory" area, press the "Change" button.

 

At the bottom of the new window there is a "Recommended paging file size." Decide how big you want the swap file on D Drive to be, and be prepared to add a swap file to C Drive which, added to the one on D Drive, totals the "Recommended paging file size."

 

At the top of the window there is a scroll box. Select D Drive, then down below click on the "Custom size" circle checkbox. Put in the size of the swap file you want on D Drive in the "Initial Size" and "Maximum Size" entry boxes, the same number in each box.

Repeat the same steps with C Drive.

Click "OK" on each of the windows to exit, and agree when Windows asks you to reboot the computer.

 

After rebooting your system should have a "virtual memory" swap file on both D Drive and on C Drive, with part of the recommended paging file size on one drive and the rest on the other.

NOTE: This is not recommended unless absolutely necessary, as one continuous "virtual memory" swap file works a lot better than two separate swap files.

matt h - say:

Hi Brad,
What you are referring to is called multi-homing.

Vista by default is not going to let you multihome it. IPv6 and the way Vista uses it will always try to get the best connection and force all the connections to use it.

But if you go into your network connection properties and set a static IP for each connection using a proper network mask, I think that should work.

Go to network connections, right click and choose properties, then pick manage network connections.
Right click over each connection and go to properties and choose (TCP\IPv4) and then click properties.

 

Manually input your IP address here.

Then choose a correct subnet mask and gateway.

 

Like 192.168.1.10 for NIC 1

255.255.0.0 for the subnet mask

 

192.168.2.10 for NIC 2

255.255.0.0 for the subnet mask

You should be able to have separate connections now.

 

From here it's just a matter of telling the programs you have, which connection to use.

 

Also, you may have to disable IPv6 if it tries to take over.
