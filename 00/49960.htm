This is a permissions issue on your C Drive:

I helped the user bring up the Action Center but the tiles displayed were not what I expected.  Trying to launch Display Properties resulted in "This file does not have a program associated with it".  An Internet search suggested that the user's profile was corrupted.  I was able to get a command prompt through Task Manager and brought up the new Metro application for handling users.  Again, a slight difference - I had an 'Add a user' link but the failing laptop had an 'Add a user in PC settings'.  This failed with "explorer.exe: class not registered".  

I managed to get the user to bring up the Desktop Control Panel, add a new account and give the account admin access.  At first things looked better - the expected taskbar icons reappeared.  However, the Start button was still not working and the same file association errors occurred.  

I was about to call it it night when I noticed that the screen displayed after a Ctl-Alt-Delete had a shutdown button.  On the failing laptop, clicking on the button showed that an update needed a restart.  Once the update had been applied, the laptop rebooted properly and everything was once again working.  

I have not determined what update needed a restart but suspect "Cumulative Update for Windows 10 Version 1511 for x64-based Systems (KB3124262)" based on the time taken to shutdown and then reboot.  The failing laptop has the default Windows update setting and the user always shutdowns down the laptop when it is not being used.   The user did not recall seeing any notifications about a pending restart.  It is possible that leaving the laptop up overnight would have allowed the update to be properly applied.

It is not clear what triggered the problem.  The problem happened while the laptop was being used.  The user thinks she accidentally hit Ctl-P instead of Caps-P but it is hard to see how this could be related.

jonathanlunman say:

Good news!  I have a fix!!!

After further investigation I discovered that the web service that is failing is not actually responsible for the search itself.  All it does is provide Media Center with a bit of meta information (the "search taxonomy") about what its Internet TV search providers are, and what sort of filters they support.  Most of those were for the old Internet TV channels that are all gone anyway.  All that really remains is Netflix.

It turns out you can override the search taxonomy URL with a registry entry.  So I created a simple Windows Service that acts as a local substitute for Microsoft's broken server, and serves up a dirt-simple SearchTaxonomy document with the minimum necessary information to allow Windows Media Center to initialize the Netflix search.

You can download it here.  Extract the files to a local directory (not on a network drive!), and run Install.bat with the "Run As Administrator" command.

Once installed, the netflix search should just work.

The code is hosted on GitHub, if you're curious about the technical details.

Enjoy!

Binaries: https://github.com/revengineering/MceNflx/releases/download/v1.0-beta/MceNflx.zip

Source: https://github.com/revengineering/MceNflx
