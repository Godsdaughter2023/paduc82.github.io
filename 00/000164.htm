Word VBA - Overflow Error multiplying 135 * 256 and Assigning to - Microsoft Community

ha567 say:

Word VBA - Overflow Error multiplying 135 * 256 and Assigning to Long

Hello,

I have the following data type:

     Private Type Entry
         tagNum(1 To 2) As Long
         dataFormat(1 To 2) As Long
         numComponents(1 To 4) As Long
         dataValue(1 To 4) As Long
     End Type

When the following line of code executes and byteTemp = 135, I get an overflow error. 
     ifdEntry.tagNum(2) = byteTemp * 256

My variable declaration section includes:

     Dim byteTemp as Byte

     Dim ifdEntry as Entry

For smaller numbers the code runs without a hitch and provides the correct results.

Any light you can shed on my error would be greatly appreciated.

Best Regards,

Chris

ha567 say:

Hello,

After some more searching for answers I have found it.

     ifdEntry.tagNum(2) = byteTemp * 256

In the above line of code the compiler has to store what I think is an intermediate calculation.  When you multiple 256 * 135 the compiler attempts to store the result in an integer which it overflows, thus the error.

The workaround is to cast one of the values as a long by appending L or &.

In my case I changed the formula to

     ifdEntry.tagNum(2) = byteTemp * 256&

and it now works.

Chris
