[....]
The code below is giving me errors I do not understand..One of the errors says "unable to get the vlookup property of the worksheet function class" (i do not have the slightest idea what this is telling me..I also get errors on the "set" statement.

LastTableRow = Sheets("Translatetable").Range("A55536").End(xlUp).Row

Set LookUpTblRange = Sheets("Translatetable").Range("A1:B & LastTableRow")
ReturnValue = WorksheetFunction.VLookup(ItemDescription,LookUpTblRange, 1, True)


First, the Set error is because of a misplaced double-quote.  The statement should be:

Set LookUpTblRange = Sheets("Translatetable").Range("A1:B" & LastTableRow)

Once that is fixed, the WorksheetFunction error might be because the lookup fails.  One way to fix that is to use Application instead WorksheetFunction, to wit:

ReturnValue = Application.VLookup(ItemDescription,LookUpTblRange, 1, True)

That returns a #NA error if the lookup fails, just like Excel's VLOOKUP.  Alternatively, you could use On Error one way or another.  For example:

On Error Then Resume Next
Err.Number = 0
ReturnValue = WorksheetFunction.VLookup(ItemDescription,LookUpTblRange, 1, True)
If Err.Number <> 0 Then
    ' .... lookup failed; do something here ....
End If

Finally, some other observations that may or may not be of interest.

1. Shouldn't VLookup(ItemDescription,LookUpTblRange,1,True) be VLookup(ItemDescription,LookUpTblRange,2,True)?  You wrote:  "Column B contains the return value".

2. Should Range("A55536") be Range("A65536")?  Note that A65536 is the last row in an XL2003 worksheet.

3. Does the sheet name "Acct & Wireless Charges Detail " really have a trailing space as written, or should it be "Acct & Wireless Charges Detail"?

4. Beware that if column A actually has 65536 values, Range("A65536").End(xlUp) is Range("A1"), not Range("A65536") as you might expect.  This is true even if 65536 is 55536 as written.

regile reb say:

Hi .dafil,

The 30-day waiting period helps Microsoft stop unauthorized people from trying to completely take over an account that isn't theirs. For example, an unauthorized person obtaining an account password through phishing or shared passwords across third party platforms. This waiting period gives the real account owner enough time to be alerted of any major changes and to cancel it if they didn't make those changes.

During this 30-day period, you'll still be able to access most Microsoft services. You won't, however, be able to do things that require a security code, like changing your password, viewing billing information, or buying new apps or games from Microsoft Store or on Xbox.

After the process of resetting your security information has started, you must wait the full 30 days for it to complete. However, if you regain access to a piece of your original security info, you can cancel the replacement yourself.
