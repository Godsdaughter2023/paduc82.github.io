i have done lots of researches. the link you provided is the only one official document about licenses. i will try my best to explain it and if anything is unclear or you still have any questions, please feel free to let me know. i will provide the further help.

generally, we have two kinds of mailboxes: user mailboxes and non-user mailboxes (such as distribution group, security group, room mailbox, shared mailbox.)

users use user mailboxes with licenses to send or receive emails. it means licenses must be assigned to user mailboxes.

for non-user mailboxes, you don’t need to assign licenses to them. you only need to assign related permissions (such as send as or send on behalf) to non-user mailboxes like the screenshot shows below. after that, you can use them to send or receive emails.

 

each mailbox has the storage limits. if they are full, you may need to enable archive, this feature must have exchange licenses. so if non-user mailboxes are full, you need to assign licenses. here is a reference:

Excel hangs at the Green Screen

Many people in my company are having this problem, although not every time they try it. When opening Excel, it will sometimes freeze at the green splash screen. To fix the problem, they must exit out of the splash screen (by clicking on the X in the upper-right of the splash screen), then either re-boot, or they can kill a background Excel process using the task manager. The next time they open Excel, it'll work and they'll have a Recovered file that they can just get rid of.

This can happen when opening Excel before opening a file, or it can happen when clicking on a desktop shortcut to an Excel file.

Some people seem to run into this more often than others. I see it maybe once every other week. I don't see any other correlations. Nor did I see it on the list of known problems and fixes.

jan de messemaeker - project mvp say:

Hi Nancy,

Yes, Project will not show a finish date different from the actual finish date.

When you want to keep the planned finish date for comparison purposes you have to keep it as a BASELINE.

On the Project ribbon, click the "set baseline" button, once more set baseline, then OK.

THis copies alll velues of Start into the BaselineStart fields, and Finish into the Baseline finish field. You can then see the differences in the Tracking Gantt view and have them alfanumerical when you change to the Variance table. (View ribbon, Tables, select Variance)

Mark on Track is about in the middle of the task ribbon just right of the 100% button.
