The MIN function can take up to 30 parameters, each of which can be an array.  So simply write the following array formula:

=MIN(IF(range1>0,range1),IF(range2>0,range2))

rich michaels - mvp (greatcirclelearning.com) say:

Hi Dominic,

I can share with you what I did to get mine working again. However, I can't guarantee how long it will continue to work.

I've asked three times now for help from the Microsoft Development Team about this problem that you, I and others are having with the Recent function not working properly. Since the Microsoft Word for the Mac internal product team has chosen to ignore my requests for help that go back to February, I can only conclude that they don't know what the problem is or how to fix it. :-( 

In frustration I decided to try some experiments and figure out a fix... so far what I discovered appears to work. There are consequences though because if you do this you will have to reestablish all of your preferences for not only Word but also Excel, PowerPoint and Outlook.

Before starting the process you must quit all Office applications, including Outlook, Excel, Word and PowerPoint... and remember "closing" is not "quitting" on a Mac. If you are unsure about whether an Office application is still running it would be best if you restarted your computer. You will also need to know your sign in ID for your Microsoft Office account if you obtained it via an Office365 subscription.

When you are ready, go to this location and move to the trash the MicrosoftRegistrationDB.reg file that is located there. This is a locked file so you will need to confirm your request to delete it.

/Users/<your  home directory name>/Library/Group Containers/UBF8T346G9.Office

After deleting the file open Word and if you still have Finder open to this Group Containers folder you will see the database be recreated. Word will now open to the document gallery page and you will notice that you are not signed in. You should go ahead and do that with your Microsoft Office sign-in ID.

Then click the new blank document image and go to File > Open Recent. The recent list will probably not be populated at all but go ahead and open something or save a new document. Then check the recent list again, hopefully it will repopulate with not only your new document, but also your former list of recent documents... that's at least what happened on my system.

My conclusions about this are that the recent list of files is not being held in this MicrosoftRegistrationDB.reg file but some important pointer(s) are and they have corrupted. By deleting the file and allowing it to rebuild, clears the problem... at least for now.

Hope this helps
