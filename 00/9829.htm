Powerpoint internal Hyperlinks

steve rindsberg say:
Powerpoint has a limited amount of storage space for hyperlinks; once the space is filled, hyperlinks start to break.

How many hyperlinks are there in the presentation?

Do the slides they link to have long titles?  The title text becomes part of the hyperlink; the longer the text, the more quickly the available hyperlink space gets used up.

Also:  

Choose File | Info

On the right, click the downward-pointing triangle next to Properties and choose Advanced Properties.

Click the Custom tab of the properties dialog box that appears.

Normally there are no properties or only a few defined here, but if there are lots of them, they might be causing hyperlinking problems.  Custom Properties and hyperlinks share the same limited storage space.

marshall barton say:

When you change the GroupLevel control source, you should also change the group header text boxes' control source:

Private Sub Report_Open(CanCel As Integer)
   Select Case Forms!frmActionPlanFromUnitChangingGroupOrder!cboChangingGroup
   Case 1
       Me.GroupLevel(0).ControlSource = "NghiepVu"

       Me.txtGroup0 = "NghiepVu"

       Me.GroupLevel(1).ControlSource = "UnitSupplied"

       Me.txtGroup1 = "UnitSupplied"

 

   Case 2

       Me.GroupLevel(0).ControlSource = "UnitSupplied"

       Me.txtGroup0 = "UnitSupplied"

       Me.GroupLevel(1).ControlSource = "NghiepVu"

       Me.txtGroup1 = "NghiepVu"

   End Select
End Sub

I used a generic name for the header text boxes because they will not always display the same record source data field.  If the group headers and/or footers include controls that are related to the grouping field, their control source would also need to be changed.

Don't forget that a text box expression might also be related to the grouping field.  E.g. a group total text box would have to be changed to:

    Me.txtTotal0 = "=Sum(UnitCost)"

    Me.txtTotal0.Visible = True

and make the other footer's text box invisible:

    Me.txtTotal1.Visible = False

So, while the code to change the GroupLevel is very simple, there may be a lot of related things that also need to be changed.

Greetings!   Welcome to the Windows Mobile Owners Circle forums!

 

Your problem is likely being caused by a setting in ActiveSync (AS) which is resetting your clock on the device.  Go to AS, click on the OPTIONS button, and on the Sync Options (tab) should be the option "Synchronize mobile device clock upon connecting".  Since your time is consistently being reset, I would expect you to find this option already checked.
