For Windows 7, you need to use the command line utility "cmdkey.exe"  Full directions are at http://technet.microsoft.com/en-us/library/cc754243%28WS.10%29.aspx

The quick start:

Open a DOS window

cmdkey /delete:<<<The target info for the key you want to delete>>>

 

But be careful.  There is no undo, so if you don't have the password, you can break logging in with those credentials.

Definitely.  There are two possibilities from that description.

First, find out where the OS thinks your  Favorites  are.   E.g.  Run...  (press Win-R and enter:)

     shell:Favorites

Assuming that causes an Explorer window to open, use  Alt-d  to see the path to it.

Does that look like your Favorites and where you would expect them to be?

Try avoiding using the Add to Favorites dialog and see if you can add to that directory in other ways, e.g. press F11 to make the Explorer window Fullscreen, then open an IE window in normal mode on top, press Shift- and drag a URL out of the Address bar and then drop it on the Explorer window.   Does that create a new Favorite (.URL file) there?

Alternatively, you could try using IE's Add to Favorites Bar button, then refresh your background Explorer window, drill down into the Favorites Favorites Bar subdirectory and see if you had a new copy of it there too.

If you can do any of the above the second possibility will not be required but the reason why it might be needed would be due to an NTFS permissions issue caused by the way that you may have created your Favorites directory...

http://blogs.technet.com/b/james/archive/2007/02/26/vista-favorites-problem-and-solution.aspx

The only other possibility that I can think of is that there is a problem with the UI but that would be a separate issue which could be dealt with after you find out whether your Favorites are where you think they should be and whether you can use them in any way without involving the UI.
