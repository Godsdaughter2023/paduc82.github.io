Show message instead of opening form when query requiring parameter entry to run returns no results

Hi - I have a form containing a command button that opens a second form (Frm_Qry_Search) based on a query (Qry_Search). The query is set up to prompt for the search parameter as follows:

Like "*" & [Enter Search Criteria] & "*"

This searches 2 concatenated fields as follows:

[Item name] & " " & [Alternative name]

All is working well... except that when the query returns zero results the form opens and shows blank records. Not a major problem, but I'd prefer to have a message box pop up.

I've done some digging and found a number of potential solutions using the DCount Function. I've tried using the following code in the ON Load event of Frm_Qry_Search, but this isn't working:

If DCount("*","Qry_Search") = 0 Then
Msgbox "No Records"
Cancel = True
End If

The error messages indicates the following:

Run-time error '2741':

The expression you entered as a query parameter produced this error: '[Enter Search Criteria]'

Does anyone have any ideas how I can figure this out, please? Thanks in advance.

ken sheridan say: Dispense with the parameter in the query and filter the form by means of the WhereCondition argument of the OpenForm method by adding an unbound text box, txtCriteria say, to the search form, in which the criteria can be entered by the user before clicking the button.  As you appear to be already filtering the form to a string expression assigned to a strCondition variable, you need to extend the button's code as follows:

     If Len(strCondition) > 0 Then
        strCondition = "(" & strCondition & ") And "
     End If
     
     strCondition = strCondition & _
        "[Item name] & "" "" & [Alternative name] Like ""*" & _
        Me.txtCriteria & "*"""

     On Error Resume Next
     DoCmd.OpenForm _
          FormName:="Frm_Qry_Search", _
          View:=acFormDS, _
          WhereCondition:=strCondition

     Select Case Err.Number
          Case 0
          ' no error
          Case 3709
          ' anticipated error, do nothing
          Case Else
          ' unknown error, inform user
          MsgBox Err.Description, vbExclamation, "Error"
     End Select

In the Frm_Qry_Search form's Open event procedure extend Tom's code as follows to clear the text box in the search form:

    If Me.RecordsetClone.RecordCount = 0 Then
        MsgBox "Yo! No results for this search. Try again", vbExclamation
        Cancel = True
        Forms!NameOfSearchForm!txtCriteria = Null
    End If

In the Frm_Qry_Search form's Close procedure clear the text box in the search form with:

        Forms!NameOfSearchForm!txtCriteria = Null

Hi,

Word builds the Page X of Y AutoText entry from a pair of fields: Page and Numpages.

If you need to show the current page number and the total number of pages in a document, you can use the Page Number Gallery, or you can use Quick Parts tool with field codes.

You can control the Numpages field to return the page count you're after and use Start At setting to display page 1 for the page following your page.

For a complete instruction for these settings, kindly refer to the support article below.
