The formula that you have looks at Col A and Col B when it is calculating for column A and then at col B and col C when calculating for column B due to relative referencing.

Use absolute addressing for the columns in the formula like the following.

=(LEN($A2)+LEN($B2))>=40

Mail Merge INCLUDEPICTURE is repeating the same picture! Please - Microsoft Community

kevin.rose say:

Mail Merge INCLUDEPICTURE is repeating the same picture! Please Help!

I have my customer info on a spreadsheet.  One field includes a URL to the public record's picture of that customer's house.

 

On the mail merge document I inserted... { INCLUDEPICTURE"{MERGEFIELD "HOUSE_PIC" }" }

 

The first record's picture displays, but then repeats the itself for every following record, even though I'm using the NEXT command.  The other fields go to the next record, but not the picture.

macropod say:

You can use a field construction like:

{INCLUDEPICTURE {IF TRUE "«HOUSE_PIC»"} \d}
or:
{INCLUDEPICTURE {IF TRUE "{MERGEFIELD "HOUSE_PIC" }"} \d}

Note: The field brace pairs (ie '{ }') for the above example are created in the body of the document via Ctrl-F9 (Cmd-F9 on a Mac) - you can't simply type them or copy & paste them from this message. Likewise, you can't type or copy & paste the chevrons (ie '« »') - they're part of the actual mergefields, which you can insert from the mailmerge toolbar.

 

Even with the above construction, though, the images remain linked to the image files, which can be an issue if you later delete the image or you need to send the merged output to someone else. The following field construction addresses that issue as well:
{IF {INCLUDEPICTURE {IF TRUE «HOUSE_PIC»} \d} {INCLUDEPICTURE {IF TRUE «HOUSE_PIC»} \d}}
or:
{IF {INCLUDEPICTURE {IF TRUE {MERGEFIELD "HOUSE_PIC"}} \d} {INCLUDEPICTURE {IF TRUE {MERGEFIELD "HOUSE_PIC"}} \d}}
