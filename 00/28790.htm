doug robbins - mvp office apps & services (word) say:

I assume that your data source has just one record for each family with separate fields for names and age of each of the possible 4 children

In that case, for each merge field that may be empty in the data source, you need to use a field construction such as the following:

{ IF { MERGEFIELD Male_Adult } <> "" "{ MERGEFIELD Male_Adult } ({ MERGEFIELD Male_Nat })" "" }{ IF { MERGEFIELD Female_Adult } <> "" " & { MERGEFIELD Female_Adult } ({ MERGEFIELD Female_Nat })" "" }

Where you must use Ctrl+F9 to insert each pair of field delimiters { } (Note, there is no hard carriage return between the above lines.  All of that construction needs to be on one line though it may word wrap automatically)

Use Alt+F9 to toggle off the display of the field codes.

You can include all required punctuation in the appropriate places in the field constructions so that they will only appear if required (as in the case of the & (with a space before and after it) in the above example.

You should probably use Shift+Enter to create each new line in the data for a family so that it is all in the one paragraph and then you can format the paragraph in your mail merge main document so that the lines are kept together and then the data for a family will not be split over a page.

Unless you want to produce labels, you should be using a Directory type mail merge main document, in which case, you need only set up the required merge fields the one time and insert an empty paragraph after the paragraph that contains the merge fields.  You do NOT then need to use a «Next Record» field.

Hope this helps.

Please follow the below steps to reset the Windows Updates Components:

  1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.
  2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

   3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

   4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

   5. Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and check if the above steps resolve the issue.
