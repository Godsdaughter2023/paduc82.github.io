brian tillman [outlook mvp 2007-2019] say:

By definition, webmail is mail you access using a web browser. So to be pedantic, Outlook 2010 doesn't handle webmail. Only a web browser can do that.

The only account type in Outlook that has the ability to download messages from the server and then delete them, keeping only a local copy, is POP3. The setting for that is on the Advanced tab of the account settings. Click File>Account Settings>Account Settings. Double click the account to open the account properties, then click More Settings and choose the Advanced tab.

If you're not using POP3 but, instead, using IMAP, there is no ability to download the message locally and then delete it from the server. IMAP always shows you a server-based mailbox. The server always holds the messages and Outlook makes a local cache that is supposed to always reflect what's on the server. It is a two-way protocol so that anything you do server-side is reflected in the mail client and vice versa.
