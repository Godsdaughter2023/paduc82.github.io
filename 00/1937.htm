We have had the delight in trying to solve the issue of root certificates with Entourage 2008 and Exchange 2007 (Microsoft Small Business 2008)

Amazingly, with lots of people on the web with the same problem, no-one has posted a solution? As a small business we at Alpine hope this saves you the time and expense that we have incurred trying to solve this problem!

Here goes with the solution for how to install a root certificate on Mac OS X, Entourage 2008 and Microsoft Exchange:

First we need to create a certificate:

Error message: unable to establish a secure connection to XXX.XXX.XXX.XXX because a correct root certificate is not installed.

Within IIS console on SBS 2008

  • Server Certificates
  • Create self signed certificate i.e. mail.myserver.com or buy one!
  • SBS web applications
  • Bindings
  • Edit - where it says SSL certificate - ensure correct certificate is selected - or select view drop down menu.
  • View
  • Details
  • Copy to file
  • Next
  • Yes, export the private key
  • Next
  • Choose Personal Information Exchange
  • Tick boxes for;
  • "include all certificates in the certification path if possible"
  • and
  • "export all extended properties"
  • Give it a password
  • Save the certificate.
  • Go to location where you saved the file/certificate and email your users or put on an accessible network share.


Save certificate to your mac (desktop).

Installing the certificate on mac OS X.


We need to install the certificate in two places:

On mac
Open hard drive
Search for Microsoft cert manager (choose files to search) and open it
Import
Locate saved certificate - open
Enter password you gave to certificate earlier.
Certificate now imported.
Quit cert manager


Install certificate on Keychain login
Now search for "Keychain access"
Highlight login
File
Import
Browse for certificate
Make sure you are importing into login key chain
Enter certificate password

You should now have 2 certificates that you have imported. One is expandable.
On both certificates:
Double click
Expand trust
Under field "when using this certificate"
Choose "always trust"
Then enter MAC password

Now go to to Entourage 2008, connect to your exchange 2007 server.


Open entourage and note no certificate problems!

Try these 2. These are both ARRAY formula, see below for how to enter them.

Count if in scope (4)

=COUNT(1/FREQUENCY(IF($C$1:$C$100="Yes",IF($A$1:$A$100<>"",MATCH($A$1:$A$100,$A$1:$A$100,0))),ROW($A$1:$A$100)-ROW($A$1)+1))

Count of started (3)

=COUNT(1/FREQUENCY(IF($D$1:$D$100="Started",IF($A$1:$A$100<>"",MATCH($A$1:$A$100,$A$1:$A$100,0))),ROW($A$1:$A$100)-ROW($A$1)+1))

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

jay freedman say:

When you say "click on the end of the table", exactly where is that with respect to the table structure? Is it within the last cell, between the last cell and the final row marker, or really below the table? Each of these locations gives a different result when you press Enter.

It's easiest to tell what the location is if you click the ¶ button on the Home tab to display nonprinting characters, including the cell markers.

If you click inside the cell, you just get a new paragraph inside the cell:

If you click outside the last cell, but somewhere in the margin to the right of that row, the cursor goes between the table and the final row marker, or it might select the row marker:

Then pressing Enter creates a new row, which is probably what's happening to you:

What you really want is to click at the start of the next paragraph below the table:

Pressing Enter there adds a new paragraph between the table and the existing text:

Similarly, if you want to add a paragraph mark before a table, click at the end of the existing paragraph before the table, and press Enter there. A special case is when the table is the first thing in the document, and there is no paragraph above it. For that, put the cursor at the left of the first character in the first cell, and click Table Tools > Layout > Split Table. That will create a new paragraph above the table.

jack cook - aka help_line say:

error code 0X85010014 indicates a problem in Outlook on your desktop. Run Detect & Repair in Outlook under the Help menu. If that does not help, you will need to uninstall Outlook and Activesync and reboot your PC. Reinstall Outlook first, then Activesync and connect again.

Here is a list of common issues relating to error 0x85010014 (Credit: PocketPCFAQ)

List of General Issues

  1. If you are seeing the error 85010014 (a general error) then try uninstalling and reinstalling Outlook. The error in ActiveSync is actually from the Outlook APIs.  You will need to reconfigure Outlook unless you backup your settings.  To uninstall Outlook click on the Control Panel -> Add or remove Programs -> Office 2003 and chose to modify the installation by adding or removing components. Uncheck Outlook and updated it to remove Outlook from Office 2003. Then click on Office 2003 again and re-check the option to install Outlook to add it back to Office 2003. You may need your Office 2003 CD. Then reboot.
  2. If you are seeing the error 85010014 (a general error) and are using Exchange2003 and folder redirection with Outlook, you may have to turn folder redirection off to sync or upgrade toActiveSync 4.2.
  3. If you are seeing the error 85010014 and you are using Outlook in  off-line mode with Exchange, you must be online with Exchange to sync.
  4. If you are seeing the error 85010014 and you have Outlook script blocking setup with your antivirus oranti-spywareprogram enabled you must configure your antivirus or anti-spyware program to allow ActiveSync to send scripts to Outlook.
  5. If you are using an unregistered or illegal copy of Outlook you may see the 85010014 error.

You can save a webpage as an .html document and attach it as a link in your email..

 

Open your webpage on the Desktop Internet Explorer.

From the top menu bar, select "File"  (If theres no menu bar, right-click on the top blue border)

Select "Save As"

Name the file and select the .html extension.

 

 

You can do a scrrenshot too, but the quality is not always that good., and you have to designate a program to open it with.

Press the Windows Key and prt sc at the same time.  (your screen will dim momentarily)

The screenshot will go into your Pictures library. 

 

 There is also a "Snipping Tool" available amongst your Start Screen Apps that can be used to crop a webpage.  Right-click on Start screen, select "all Apps" from the bottom ribbon.
