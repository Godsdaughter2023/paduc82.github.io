Win 8 Total Backup System: File History + RecImg + Refresh + SlimImage

Win 8 has refined your backup and restore options. Before Win 8 you could create an Image Copy backup of the full drive (or selected directories), but it was hard to separate backups of the OS, Applications and personal Data unless you created a separate

 

Out of the box, MS has been advertising the “Refresh” and “Reset” options. Personally, I think they missed a valuable selling point by not mentioning RecImg.EXE in conjunction with Refresh. It removes my main concern/objection to Refresh.

 

Reset is equivalent to using the “Recovery Partition” on previous versions of Windows. The factory would create a “Recovery Partition” that you could use to “reset the computer to factory fresh”. It would typically include all of the factory “crapware” and it would wipe out all of your data files and installed applications.  A less than desirable outcome but an appropriate reaction to a bad malware infection.  The “recovery partition” type of restore typically had to be accessed at boot time while Reset can be accessed from within Windows 8.

 

Refresh is roughly equivalent to an image copy backup.  A Reset will restore a backup of the OS and Metro applications. It will wipe out “legacy” / x86 desktop type applications BUT it will not affect your personal data files, which is much better than a Reset.  But there is still that issue of wiping out your installed desktop applications. I do not have documented confirmation, but I’m guessing that Metro apps are treated like personal data and those directories/folders are simply not reset back to the old condition.

 

RecImg.EXE is a backup tool that ships with Windows 8. It creates the backup “image” that is used by the Refresh tool. This image that was created at the factory is why using Refresh will wipe out installed desktop programs in the “C:\Program files” and “C:\Program Files (x86)” folders because they haven’t been backed up. However, using RecImg.EXE you can create new images that will include recently installed desktop applications.

 

RecImg Manager is a third party tool the provides a GUI to access the RecImg.EXE command line tool and backup files.

 

File History is a new Win 8 feature. It is an extension to the “Previous Versions” and “Volume Shadow Copy” features. File History continuously monitors files stored in Libraries, Desktop, Contacts, and Favorites folders. It runs on a predetermined schedule, and when it detects changes in any file, it makes a backup of that file to an External drive. It isn’t enabled by default.

daniel g. grau say:
Estás totalmente equivocado. Yo trabajo con inglés y español, y puedo decirte que los diccionarios de Word (definiciones, sinónimos y ortografía) son muchísimo más complejos en español que en inglés. Piensa solamente en la cantidad de alternativas que tenemos en español con respecto al inglés:

  • Verbos: en inglés un verbo se define por el infinitivo, la tercera persona (con "s" al final), el pasado (generalmente "ed") y el gerundio ("ing"). Cuatro opciones. En español, tenemos las seis pronombres (yo, tú, él, nosotros, vosotros, ellos), más las deformaciones del voseo (tienes = tenés), más ocho tiempos verbales, más infinitivo, participio y gerundio. En total, cada verbo tiene unas 50 variaciones. Súmale los dativos me, te, se, la, le, los, las (témase, témame, témale, témeles…), más otras combinaciones (témaselas, témaseles, témaselos...) y fácilmente superamos las 500 variaciones por verbo.
  • Adjetivos: en inglés hay una sola variante. En español, tenemos las formas masculina, femenina y sus respectivos plurales.

Estas herramientas fueron hechas en Chile, según recuerdo. Pero vayamos a tu pregunta.

Para agregar una palabra, primero tienes que crear un diccionario (y luego activarlo):

  • Vas a Word > Preferences > Spelling and Grammar
  • Aprietas el botón Dictionaries
  • Aprietas el botón New y le das un nombre descriptivo

¿Para qué querrías más de un diccionario? Para poder agregar vocabulario específico por tema y por idioma. En mi caso, los documentos médicos tienen su propia terminología (enfermedades, remedios, siglas, unidades...). Una sigla como "ALS" puede ser correcta allí, pero constituir un error de tipeo en otro contexto ("LAS").

Cada diccionario puede habilitarse o no con una marca de verificación, y puede tener un idioma asociado (controlado por medio del menú desplegable Language). De ser así, solamente se utilizará en textos que estén escritos en ese idioma. Si no tiene ningún idioma asociado, se utilizará en todos los textos, cualquiera que sea el idioma.

Cuando estés verificando la ortografía y Word no reconozca una palabra, tendrás la opción de agregarla por medio del botón Add. En ese mismo diálogo, el botón Options te permitirá cambiar de diccionario, en caso de que tengas varios activados al mismo tiempo.

Por último, el foro en español al que el moderador te refirió es para Windows y, si bien los participantes tienen buena voluntad, no manejan Mac. La solución más generalizada es siempre "reinstalar Office".

sql2 = ""

sql4 = "Into [dbo].[tbl_KOHLS_EV_2010_abenit01] FROM tblv_EmailViewer WITH (NOLOCK) WHERE MasterRec ='177'  Order by SentDate desc"

"Driver={SQL Native Client};Server=ATL20AS1100SQ02;Database=AS_KHL_2010_2011;Trusted_Connection=yes;"

 

function:

Function SQL_PassThrough2(ByVal ConnectionString As String, _
                              ByVal sql As String, _
                              ByVal sql2 As String, _
                              ByVal sql3 As String, _
                              ByVal sql4 As String, _
                              Optional ByVal QueryName As String)

    Dim dbs As DAO.Database
    Dim qdf As DAO.QueryDef
   
    Set dbs = CurrentDb
    dbs.QueryTimeout = 300
    Set qdf = dbs.CreateQueryDef
   
    With qdf
        .Name = QueryName
        .Connect = ConnectionString
        .sql = sql & sql2 & sql3 & sql4
        .ReturnsRecords = (Len(QueryName) > 0)
        .ODBCTimeout = 300
        If .ReturnsRecords = False Then
            .Execute
        Else
            If Not IsNull(dbs.QueryDefs(QueryName).Name) Then dbs.QueryDefs.Delete QueryName
            dbs.QueryDefs.Append qdf
        End If
        .Close
    End With
   
    qdf.Close
    Set qdf = Nothing
    dbs.Close
    Set dbs = Nothing

End Function
