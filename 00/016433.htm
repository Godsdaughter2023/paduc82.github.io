cmcdonald64 say:

Move PST to OneDrive and change from POP to IMAP

I've been running Office 365 Home Premium for over a year on my laptop and my wife's laptop (separate accounts on each set up with respective POP email.)

My wife recently bought a Surface Pro 3 so I installed Office 365 on it without issue.

Now what I'd like to do is find the exact steps to move HER account PST to OneDrive so she could access her account from either machine. I assume I will need to change it from POP to IMAP to accomplish this.

I find numerous reference to this issue, but so far I have not clearly understood exactly how to accomplish this. I've been searching for answers for a couple weeks now and seem to find more confusion than answers.

Her PST is relatively small, 200 MB.

Both machines are on Win 8.1

karl timmermans [outlook mvp 2012-2018] say:

#1 - PST files are not supported on external sources (network drives / OneDrive etc). Size ha s nothing to do with it and the approach could very easily cause corruption.

#2 - Changing to Imap will result in the email component of the account being stored in an .OST file which mirrors the content of the account on the server. The caveat is that contacts/calendar/tasks are maintained locally only - in other words - those item types don't get sync'd across devices. To have everything sync'd across multiple devices, you would need to use either an Outlook.com account (which is free) configured as an EAS (Exchange ActiveSync) account or a full Exchange account (has a monthly cost associated with it)

The other thing worth noting is that configuring an Imap account in Outlook '2013 where it is the first account configured in the profile - the contacts/calendar/tasks will also be contained in the .OST file but will show the designation (This Computer Only) - ergo - locally stored. If the email account gets deleted - you will loose everything in the OST file since OST files can only be opened by the email account that originally created it in a given Outlook profile. To put it another way, you can't open an OST file on its own nor can you use it with another Imap account like you can with a PST file.

On the other hand, if the Imap is configured as the second (or subsequent) email account in the profile, the contact/calendar/task items will be stored in the respective default folders for the profile (located in the data file designated as the default).
