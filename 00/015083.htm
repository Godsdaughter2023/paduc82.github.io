hansv mvp say:

The standard export to csv will create the same number of commas in each line, so that lines with fewer phrases will have more extra commas at the end.

Here is a version that uses | as delimiter (customizable) and that doesn't produce extra commas.

Sub SortRows()
    Const FirstRow = 1
    Const FirstCol = 1 ' A
    Const strSep = "|" ' Separator
    Dim LastRow As Long
    Dim CurRow As Long
    Dim LastCol As Long
    Dim CurCol As Long
    Dim strLine As String
    Application.ScreenUpdating = False
    Open "C:\MyFolder\Test.csv" For Output As #1
    LastRow = Cells(Rows.Count, FirstCol).End(xlUp).Row
    For CurRow = FirstRow To LastRow
        LastCol = Cells(CurRow, Columns.Count).End(xlToLeft).Column
        If LastCol > FirstCol Then
            Range(Cells(CurRow, FirstCol), Cells(CurRow, LastCol)).Sort _
                Key1:=Cells(CurRow, FirstCol), Header:=xlNo, _
                Orientation:=xlLeftToRight
        End If
        strLine = ""
        For CurCol = FirstCol To LastCol
            strLine = strLine & strSep & Cells(CurRow, CurCol).Value
        Next CurCol
        Print #1, Mid(strLine, Len(strSep) + 1)
    Next CurRow
    Close #1
    Application.ScreenUpdating = True
End Sub

If the email account is configured as an IMAP type, then by default you would not have the option “Leave a copy of messages on the server” available in the More settings Advance tab. "Drag and Drop", you probably are doing it without knowing it by that name. It is easiest if you use 2 Windows Explorer windows (but you can do it with one). One explorer is open to the source location (C: or D:) and the other is opened to the desired folder on the destination drive (D: or H:).  In the source location, you click on the file you want to move and hold the mouse button down while you "drag" the mouse pointer to the destination folder and release the mouse button. Windows will move the file from source to destination.  If you click and hold the right mouse button (instead of the left button), then drag and drop, you will be offered the option to copy rather than move the file.

Drag and drop may be more challenging than you want to deal with. There is another option. In the source folder, click to select the file you want, the use <CTL><C> to copy it (or <CTL><X> to cut it) into the "clipboard". Then you can simply click on the destination folder to make it active and use <CTL><V> to paste the file into the destination.

sanwin2 say:

As Rohn says, there is nothing you can do about it.  If it has hit your sent items it has left your computer.  From you it will go into the send queue at your ISP, which may take a minute or two (or even more) in busy periods, it then travels from server to server over the internet, possibly even bouncing off a satellite or two, until it reaches the incoming server on your recipient's ISP, where it will hit the incoming queue.  Once it reaches the head of the queue it will be put in the recipient's incoming queue where it will sit until their email program polls for new email - very often 5 minutes or more between polls.  (In fact a lot of ISPs frown on polling more often than every 15 minutes.)

 

Any server on the route will add a delay, so expecting instantaneous email delivery is unrealistic.

 

To see how long things can be try sending yourself an email.  With my ISP this can take anything from 10 seconds to several minutes.
