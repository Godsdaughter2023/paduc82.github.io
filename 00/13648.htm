jay freedman say:

That error occurs if a template-type add-in contains a macro that has a programming error or is not compatible with Word 2016. The fix is to remove the add-in or find an updated version that is compatible.

After closing Word, use the File Explorer to go to the folder %appdata%\Microsoft\Word\Startup. Move every file you find there to some other folder. Restart Word and confirm that the error doesn't occur. If there were two or more files, you can move them back into the Startup folder one at a time, restart Word, and look for the error. The one template that causes the error to return is the cause of the problem.

When you know which template is the cause, that should tell you which add-in installed it. You can then decide whether to do without the add-in or try to get an update.

hal hostetler - mvp-outlook say:
The POP folder appears when you have an email program connected to Hotmail set up with the POP protocol and set to delete email from the server after downloading it.  Hotmail does this so other email clients connected to the account have access to the "deleted" email.  You can fix this by using any of the other protocols that Hotmail supports, IMAP, Exchange ActiveSync, or Exchange Server (once your account is migrated to the Office 365 Exchange Online servers), by telling Outlook's POP service not to delete email from the server after downloading it ("File > Account Settings > Account settings", select your Hotmail account, click "Change", click "More Settings", and click the "Advanced" tab), and by logging into your Hotmail account using a web browser.  Click the Options gear to the left of your name at the upper right of the web page, choose "More mail settings", then click "POP and deleting downloaded messages".  You'll see a radio button labeled "Do what my other program says—if it says to delete messages, then delete them".  Select it, then click Save.  Now Hotmail will obey the delete command

Using mail merge to create different numbers of each label - Microsoft Community

eric leopold say:

Using mail merge to create different numbers of each label

I am trying to use Word to print out labels where the label info is stored in an Excel spreadsheet.  These labels are for greenhouse plants we are starting, hence each label may need to be printed out from 10-150 times depending on how many of each variety we are planting.  I suspect I may have to write a macro that repeats each line the specified number of labels required (based on info in the source file for each variety).  Is there an easier way to do this?

[Moved from: Office / Word / Windows 10 / Office 365 Home]

doug robbins - mvp office apps & services (word) say:

If you have the number of labels required for each record in the last column of the data source, and you run a macro containing the following code when the worksheet containing the label data is the active sheet, it will add a sheet to the workbook that contains the required number of records for each label so that when you use that added sheet as the data source, you will get the number of each labels that you require

Dim SourceSheet As Worksheet

Dim TargetSheet As Worksheet

Dim i As Long, j As Long, k As Long, r As Long, numlabels As Long

Set SourceSheet = ActiveSheet

Set TargetSheet = Sheets.Add

With SourceSheet.Range("A1")

    For i = 0 To .CurrentRegion.Columns.Count - 2

        TargetSheet.Range("A1").Offset(0, i) = .Offset(0, i)

    Next i

    r = 1

    For k = 1 To .CurrentRegion.Rows.Count - 1

        numlabels = .Offset(k, .CurrentRegion.Columns.Count - 1)

        For j = 1 To numlabels

            For i = 0 To .CurrentRegion.Columns.Count - 2

                TargetSheet.Range("A1").Offset(r, i) = .Offset(k, i)

            Next i

            r = r + 1

        Next j

    Next k

End With
