lynneford say:

I don't pretend to have a lot of knowledge here, but I ran into this same problem on a new PC with Win 10 that I am moving to and this is how I fixed it. Hope this helps you if you are still having a problem.

When I checked the Epson driver security settings against my old PC from the control panel I noticed that my user account (which is an admin account) was missing the check mark on "Manage this printer" on the new PC. I checked this box it to match my old PC. 

The real solution however is in the way I then forced the custom settings down to the user account to be available in the Adobe and other applications from my user account. Basically I set the custom settings up from the control panel version of the driver, exported the settings. Next I went to an application, opened the print setting panel and imported the settings to make them available to my user account. Details follow.

First, launch the Epson P800 print settings box from the Control Panel (go to Control Panel/ Choose Devices/ Choose Printers/Choose Epson Printer/Click on Manage.) Click on Printing Preferences - this opens the Print Settings box. Make any default changes here that you want (it comes up as "glossy, I always print on Luster or Matte, so I choose luster here, made my additional changes such as "Mode: No Color Adjustment". Click OK. These will become your new defaults. Close the panel.

Second, open the panel again, confirm your default values are still there and and set up your custom settings using the Save/Del button to designate a name for the setting. Before you close this second panel, click on EXPORT and export the setting using the same name to some place you can remember. Close this print setting panel. Repeat the Save/Export steps for additional setups.

Third, go to Photoshop and open an image. Select File/Print to open the print panel. Make sure Epson P800 is selected and click on "Print Settings". Up comes the now familiar panel. Click on the Save/Del button and at the bottom of the new window, click on IMPORT. Navigate to the .ssf file you just exported and click "Open". The setting will appear in the list at the top left. Close this window and check that the saved setting is now available from the top "Select Settings" drop-down. This should now look familiar to you. Repeat for each setting you want to bring in.

These settings will now be available to any application that this user has access to. I know this is a lot more work initially then the way it used to work, but short of uninstalling a lot of stuff and trying figure out what was conflicting to cause the driver to install this way, this workaround gave me a fast workaround to get the two setups in that I needed immediately. Hope this helps.

This is an FAQ. The problem is that different HTML editors have different ways of converting line breaks into HTML, i.e. what code is generated when you press the Enter or Return key. Some of the possibilities are:

  1. A line break code <br> or <br />
  2. A paragraph code </p> ... <p>
  3. A divider code </div> ... <div> 


These all have different effects on the way HTML will be displayed in the viewer. There may be different styles pre-defined for the different elements, and some of those styles may not be available in another HTML editor, which has then to find a workaround to try and reproduce the text in a manner as close as possible to the original. For example, the style definition for <p> in the original editor may include padding to produce white space before or after a paragraph, while the <div> doesn't.

Windows Live Mail has a simple HTML editor with limited formatting capability. To keep thins simple, when you paste HTML markup from another application, all <p> and <div> tags are reproduced as <div> and assume the default style as specified at Options > Compose.  Empty <div>s may be removed or replaced by <div>s containing a single non-breaking space.

If you're using MS Word, then it's quick and easy to use Find/Replace (Ctrl-H) to change paragraphs (^p) to newlines (^l) in your prepared texts.
