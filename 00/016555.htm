You can use the below Function to check for the File...

Function FileExists(ByVal strFile As String, Optional bFindFolders As Boolean) As Boolean
    'Purpose:   Return True if the file exists, even if it is hidden.
    'Arguments: strFile: File name to look for. Current directory searched if no path included.
    '           bFindFolders. If strFile is a folder, FileExists() returns False unless this argument is True.
    'Note:      Does not look inside subdirectories for the file.
    'Author:    Allen Browne. http://allenbrowne.com June, 2006.
    Dim lngAttributes As Long

    'Include read-only files, hidden files, system files.
    lngAttributes = (vbReadOnly Or vbHidden Or vbSystem)

    If bFindFolders Then
        lngAttributes = (lngAttributes Or vbDirectory) 'Include folders as well.
    Else
        'Strip any trailing slash, so Dir does not look inside the folder.
        Do While Right$(strFile, 1) = "\"
            strFile = Left$(strFile, Len(strFile) - 1)
        Loop
    End If

    'If Dir() returns something, the file exists.
    On Error Resume Next
    FileExists = (Len(Dir(strFile, lngAttributes)) > 0)
End Function

Then use a If, Then Else statement for what you want to.

tom ogilvy say:

Just to add to the wonderful solutions already provided, the problem is that you specified the argument to your VBA functions would be in R1C1 format and then provided an A1 formatted formula.   the solutions used just the Formula function which looks for an A1 formatted formula.  If however, you wanted to use FormulaR1C1 then you could have done this:

Sub abc()
Range("P2").Select
ActiveCell.FormulaR1C1 = _
        "=VLOOKUP(RC[-10],'Supplier list'!C[-15]:C[-14],2,0)"
End Sub

that produces the formula  =VLOOKUP(F2,'Supplier list'!A:B,2,0)  in cell P2.   Not that the cell references in the result are relative as you originally appeared to want.

Nesting Functions in Excel 2013 - Microsoft Community

demomantwo say:

Nesting Functions in Excel 2013

I need an "OR" function within an "IF" function to read:

If (H322 OR J322 <> 0, J322 + H322 - I322, 0)

In other words:

IF H322 or J322 contains any data then I want it to take: J322 plus H322 and subtract I322.

Otherwise leave the cell blank.

I also tried

IF(OR(H322<>0, J322<>0), J322 + H322 - I322, 0)

jeeped say:
A blank cell equates to 0 when used mathematically or logically. You need to test for blank, length or numbers in both cells. There are literally dozens of ways to check for this. I'm not entirely sure whether you want both H322 & J322 to contain numbers or if one is enough. Here are a few examples that check for both cells containing values or specifically number values.

=IF(COUNT(H322, J322)=2, J322 + H322 - I322, 0)
=IF(AND(LEN(H322), LEN(J322)), J322 + H322 - I322, 0)
=IF(OR(ISBLANK(H322), ISBLANK(J322)), 0, J322 + H322 - I322)
=IF(AND(ISNUMBER(H322), ISNUMBER(J322)), J322 + H322 - I322, 0)

It is important to note that if H322 or J332 contain formulas that may return an empty string (e.g. "") then these are not considered blank by ISBLANK(). You need to test using one of the other methods. Likewise, testing for LEN() does not preclude that one of the cells may contain text that will produce #VALUE! errors when used mathematically.

COUNT function Counts how many numbers are in the list of arguments
LEN function Returns the number of characters in a text string
ISBLANK function Returns TRUE if the value is blank
ISNUMBER function Returns TRUE if the value is a number
OR function Returns TRUE if any argument is TRUE
AND function Returns TRUE if all of its arguments are TRUE
