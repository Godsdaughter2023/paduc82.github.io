rick rothstein, mvp - excel say:

A worksheet has two names... Name, which is the name you see in the tab, and CodeName which is its (almost) unchanging name that you referred to. The Name property is what you use as an argument for Worksheets, such as Worksheets("Sheet1"), and it is a String value. The CodeName property is a worksheet object and can be used directly in code, such as, Sheet1.Activate... notice, because it is a worksheet object, it works standalone directly in code without needing a call to Worksheets. Now, some of the confusion on this is that in a new workbook, both the Name and CodeName appear to be the same; however, you can change the Name property during execution of the program whereas you cannot change the CodeName for the sheet during execution of the program (which is why you can use the CodeName worksheet object directly in you code because VB knows you won't change it while the program is running. So, let's say you rename Sheet1 to be called Summary. Then, in code, you could select, say cell B2 on the worksheet (as but just one way to use these properties) either like this...

Worksheets("Summary").Range("B2").Select

or like this....

Sheet1.Range("B2").Select

It is this later usuage that your original message was referring to. Now, if we were always trapped to CodeNames of Sheet1, Sheet2, etc., it might be kind of hard to keep track of them. So, while you cannot change the CodeName from within your project, you **can** change it in the VB editor. From our previous example, select the Summary sheet (which has CodeName of Sheet1) in the Project Explorer window and press F4 to display the Properties window (assuming it is not already displayed). Now, because you selected the Summary sheet in the Project Explorer, it will be the selected sheet in the Properties window as well. Notice the this worksheet has to "Name" properties... the first one is shown in parentheses and is the CodeName (it should be showing Sheet1) and further down the property list is the Name property (no parentheses) which should be showing Summary, what we renamed the sheet to. Well, just like you can change the Name property to Summary or whatever else you want, you can also change the CodeName, that is, the (Name) property as well... just click into the property item and type, say, MySummary. You can now use MySummary as the CodeName inside your program. So, after doing this, you can perform our B2 selection either like this...

Worksheets("Summary").Range("B2").Select

or like this....

MySummary.Range("B2").Select

I hope this gives you what you are looking for.

jim thomlinson say:

To add to Rick's thread the code name is the ideal solution to reference worksheets within the same workbook. To continue Rick's example if you type in MySummary and then the dot you will notice that you get an intellisence drop down of the properties and methods of the worksheet. The same is not true using Worksheets("Summary"). The reason for this has to do with a concept called binding. Using the code name we are binding directly to the specific worksheet at design time so the compiler knows the properties and methods of a worksheet. Using the second method the type of sheet will not be know until the code starts to execute. You do not get intellisence because the sheet could be a worksheet, chartsheet or macrosheet.

Now for the drawback to refering to the sheet by code name... You can not create a link directly to a sheet in another workbook. At design time the workbook you are in does not know anything about the sheets that exist in other workbooks. So you can not easily use the code name of a sheet in another workbook in your current workbook. You need to establish that link at run time and now you are at the mercy of the tab name or sheet number. If the sheet you are looking for has been renamed then it will not be found. Worse still if you used the sheet number then if worksheets were added or deleted you may be referencing the wrong sheet... There are a few techniques that you can use to get around this depending on your specific situation. 

HTH... Jim Thomlinson

Function GenUniqueRand()

'Source - http://eforexcel.com/wp/article-37-vba-generati...



    Const Min As Long = 1000

    Const Max As Long = 9999



    Dim LastRow As Long

    Dim i As Long, j As Long, Temp As Long, Number As Long

    Dim Arr

    Number = Max - Min + 1

    ReDim Arr(1 To Number, 1 To 1)

    ' Generate all possible number between Min and Max

    For i = Min To Max

        Arr(i - Min + 1, 1) = i

    Next i

    Randomize

    ' Shuffle the array generated above randomly

    For i = 1 To Number

        j = Int((Number - i + 1) * Rnd) + i

        Temp = Arr(i, 1)

        Arr(i, 1) = Arr(j, 1)

        Arr(j, 1) = Temp

    Next i

    

    For i = 1 To Number

        If Unique(Arr(i, 1)) = True Then

            GenUniqueRand = Arr(i, 1)

            Exit Function

        End If

    Next i

    

End Function



Function Unique(InNumber)

    Dim RegX As Object

    Set RegX = CreateObject("VBScript.RegExp")

    With RegX

        .Global = True

        .MultiLine = True

        .Pattern = "^(?:([0-9])(?!.*\1)){4}$"

        'Source - https://stackoverflow.com/questions/20205653/re...
    End With
    Unique = RegX.Test(InNumber)
    Set RegX = Nothing
End Function

Hello Anthony,

Thank you for posting back. How are you doing? Hope everything is working fine.

Let me give you more clarifications regarding App password and two-step verification (TSV) process. Here's the thing that you need to understand.

Enabling TSV feature on your account will make your account more secure. This will give hackers/hijackers a hard time accessing your account with a stolen password. Once you've enabled TSV, you should generate and use an App password for your account so you could access it using an email client or through the default Mail app of your devices. Note that you can use the same App password for all of your devices.

If you already configured your account to the default Mail app of your devices (regardless if you use the App password or your account's password), you don't need to sign back in to your account because it's already configured. Once your phone connected to the Internet, it will automatically sync your emails.

Now if you have intentionally sign out your account or clear the device cookies, you will need to enter your credentials (username and account's password or username and App password - for TSV enabled only) again if you want to access/login to your account.
