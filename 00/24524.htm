When the fields are collapsed Excel no longer sees them as subtotals, they are just the values for that category. (there is no "sub" to total!) (Unless you are using Compact Form as your layout with subtotals on same row as labels)

So I would strictly speaking call this a workaround, but it might be perfectly sufficient for your needs.

1) select the subtotals. By the sound of things you already know how to do this by clicking when you have the black arrow which points to the right. Selecting subtotals for one category selects all subtotals for the same "level" of hierarchy. Format as required.

2) Collapse that entire field (from right click menu or on the PivotTable tools|Options ribbon. Now use the same right-pointing arrow to select all the categories (click against first, drag down to last and let go, or use Shift+click). Now format those the same as the subtotals.

Voila! Now they look the same whether expanded or collapsed.

Note, in step 2 you could use the PT Tools|Options > Select button to choose to only format the labels, the values or both (the default) if you want to keep the labels different from the values (eg coloured text on labels but not values).

As for using styles, that is a whole other question, but yes, you can use cell styles to apply the formatting above, and yes, you can create your own cell styles which will apply only the formatting you choose (eg one style applies borders and cell fill, another applies font and colour and a third applies number format and alignment). jay freedman say:

A few observations:

  • You aren't "saving memory". For a given picture format, the size of a document in memory is the same whether the picture is inserted or linked. The difference is only in the size of the document's file on disk -- because the link itself is only the few characters in the link field, and you aren't storing both the original picture file and copy of it in the document file.
  • Unless you have timing data to add to your size data that shows the opposite, I doubt that a document full of links to picture files is any faster to load than the same document with the pictures embedded in it. Even if it is faster, I doubt that the difference is more than a few seconds.
  • With the large size and low price of hard drives now, saving a few hundred KB -- or even a few hundred MB -- isn't worth bothering. Even low-end laptops these days have 500 GB drives.
  • If you plan to share your documents with other people, or to store them on a CD or DVD, then Word's bad habit of converting all relative links to absolute links that don't work will drive you crazy. There is a solution (http://windowssecrets.com/forums/showthread.php/154379-Word-Fields-and-Relative-Paths-to-External-Files) but it's tedious.
