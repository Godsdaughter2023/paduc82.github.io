hansv mvp say:

In the following code, change Range("C2:C100") to the range that you want to split. You can use a variable of type Range, of course.

 

    With Range("C2:C100")
        .TextToColumns Destination:=.Range("A1"), DataType:=xlDelimited, _
            TextQualifier:=xlDoubleQuote, ConsecutiveDelimiter:=False, Tab:=False, _
            Semicolon:=False, Comma:=False, Space:=False, Other:=True, OtherChar:="_", _
            FieldInfo:=Array(Array(1, 9), Array(2, 1)), TrailingMinusNumbers:=True
    End With

bernie deitrick, excel mvp 2000-2010 say:

Using Text-to-columns will do it, but may cause problems if you have other columns of data.

For Each myC In Range(Range("A2"), Cells(Rows.Count, 1).End(xlUp))
myC.Value = Split(myC.Value, "_")(1)
Next myC

Sorting tables in Word - Microsoft Community

oldtimer2012 say:

Sorting tables in Word

I want to sort a table in Word 2016.  The entries to be sorted are in decimal format - ie.  2.3.1.7; 3.2.5.7; 1.9.2; 5.1.4.3.2 - you get the idea!  I've been unable to sort in this manner.  Is there a way to do it without renumbering the items?   Thanks. Oldtimer.

doug robbins - mvp office apps & services (word) say:

Select the column containing the entries and then use Edit>Replace to replace the . with nothing.  Then sort the table, then to reinstate the . use Edit>Replace to replace

^#

with

^&.

You will end up with an additional . after the last number.  If you want to remove that, select the range of cells that contain the numbers and run a macro containing the following code:

Dim i As Long
Dim rng As Range
With Selection.Columns(1)
    For i = 1 To .Cells.count
        With .Cells(i)
            Set rng = .Range
            rng.End = rng.End - 2
            .Range.Text = rng.Text
        End With
    Next i
End With

macropod say:

Have you tried sorting as text? Do note that no built-in sort method will compensate for the absence of leading 0s, as in:

1.9.2

1.10.2

1.9.12

2.3.1.7

10.9.2

3.2.5.7

Whichever built-in sort method you use will result in 1.10.2 and 10.9.2 being miss-sorted.

Replacing the periods, as per Doug's suggestion would only be viable if none of the levels goes to double digits; otherwise it'll make a complete mess of the sort.

macropod say:

The following macro should sort your table correctly, regardless of how many numbering levels and the number of digits in each level you table might have and regardless of whether there is text in the cells following the numbering.

Sub TableSortDemo()
Application.ScreenUpdating = False
Dim ArrData(), StrTmp As String
Dim i As Long, j As Long, k As Long, l As Long, m As Long
With ActiveDocument.Tables(1)
  'Find out how many array elements we need
  For i = 1 To .Rows.Count
    'Exclude anything after the first space, paragraph break or the end-of-cell marker
    StrTmp = Split(.Cell(i, 1).Range.Text, Chr(13))(0)
    If Len(StrTmp) > 0 Then
      j = UBound(Split(Split(Split(StrTmp, vbTab)(0), " ")(0), "."))
      If j > l Then l = j
    End If
  Next
  'Dimension the sorting data array
  ReDim ArrData(.Rows.Count, l)
  'Populate the sorting data array, excluding anything after the first
  'space, paragraph break or the end-of-cell marker
  For i = 1 To .Rows.Count
    StrTmp = Split(.Cell(i, 1).Range.Text, Chr(13))(0)
    If Len(StrTmp) > 0 Then
      j = UBound(Split(Split(Split(StrTmp, vbTab)(0), " ")(0), "."))
      For k = 0 To j
        ArrData(i, k) = Split(StrTmp, ".")(k)
      Next
    End If
  Next
  'Reformat the sorting array data elements with as many leading 0s as might be required
  For i = 0 To l
    k = 0
    For j = 0 To UBound(ArrData)
      If Len(ArrData(j, i)) > k Then k = Len(ArrData(j, i))
    Next
    StrTmp = ""
    For m = 1 To k
      StrTmp = StrTmp & "0"
    Next
    For j = 0 To UBound(ArrData)
      If ArrData(j, i) = "" Then ArrData(j, i) = "0"
      ArrData(j, i) = Format(ArrData(j, i), StrTmp)
    Next
  Next
  'Add a temporary column to the table
  .Columns.Add .Columns(1)
  'Populate the temporary column the the sorting data
  For i = 1 To .Rows.Count
    StrTmp = ""
    For k = 0 To l
      StrTmp = StrTmp & ArrData(i, k) & "."
    Next
    .Cell(i, 1).Range = StrTmp
  Next
  'Sort on the temporary column
  .Sort ExcludeHeader:=True, FieldNumber:=1, _
    SortFieldType:=wdSortFieldAlphanumeric, _
    SortOrder:=wdSortOrderAscending
  'Delete the temporary column
  .Columns(1).Delete
End With
Application.ScreenUpdating = True
End Sub

As coded, the macro assumes you have a header row that is not to be sorted. If you don't have a header row, change:

ExcludeHeader:=True

to:

ExcludeHeader:=False

For PC macro installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm
For Mac macro installation & usage instructions, see: http://word.mvps.org/Mac/InstallMacro.html
