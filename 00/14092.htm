mike h.. say:

Hi,

I'm assuming the named range 'date' is columns C&D

Your formula is fine, it's your data. Basically it's not finding a match for A4 in column C and I think the clue is when you tell us the data are from an external source. Importing data can produce dirty data.

Note that all the numbers in Col C are 6 digits. Put this formula in a helper column and drag down and it tells us how many digits are actually in the cell

=Len(C4)

What we're looking for is when the formula tells us something we dont expect i.e there are 7 (or more) digits in the cell.

If as i suspect this is the case one method that 'sometimes' solves the problem is to put a 1 in an empty cell and copy it. Select all the data and then Right click - Paste Special - select multiply - OK

lets see if that works

EDIT,

The data you posted has an invisible character to the left of each number in column C. It's character 160.

In a helper column enter this formula and drag down

=MID(C4,2,LEN(C4))

Copy this column and then

select C4, right click, paste special, select 'VALUES', OK
