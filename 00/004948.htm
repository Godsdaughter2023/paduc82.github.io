bernie deitrick say:

It depends on what is in column D - is it a function that pulls external data, or are you manually entering the values?

If you are using functions, you could use the worksheet calculate event to compare all the values: copy this code, right click the sheet of interest, select "View Code" and paste the code into the window that appears, and save the file as a macro-enabled .xlsm  - this assumes column A has a name as the label.

Private Sub Worksheet_Calculate()
    Dim c As Range
    
    For Each c In Range(Range("D2"), Cells(Rows.Count, "D").End(xlUp))
        If c.Value > c.Offset(0, -2).Value Then
            MsgBox c.Offset(0, -3).Value & " is now at $" & c.Value & _
            ", higher than its previous high of $" & c.Offset(0, -2).Value
        End If
        If c.Value < c.Offset(0, -1).Value Then
            MsgBox c.Offset(0, -3).Value & " is now at $" & c.Value & _
            ", lower than its previous low of $" & c.Offset(0, -1).Value
        End If
    Next c
    
End Sub

rob koch say:

Could you please provide the complete path to the object that Defender identified with this detection?  This would be found at the bottom of the log entry for the item in the Defender History tab within the All detected items log.

My generic answer to this would be that the item detected was created in a place that Defender's Real-time protection doesn't normally monitor, since there are many unsuccessful attacks that occur which are blocked by the operating system itself, which since they never reach a location of true risk will never be detected via Real-time protection.  If Defender detected and alerted to these unsuccessful attack attempts you'd be receiving additional notifications for which there is no real concern, simply adding to the noise and aggravation without providing any true increase in protection.

The difference with a Full scan is that it scans all files found on the machine, regardless of context or location.  What this means, for example, is that an item which was delivered via an infected web page might be successfully placed in the Temporary Internet Files folder, but then the exploit(s) it contains are blocked by the security within Internet Explorer or another browser itself.  However, the temporary file left behind, if not automatically deleted by the browser could still be detected during a later Full scan, since this doesn't differentiate between an infection truly installed and operating as a quick scan does, or one that is currently performing an initial attack using either exploits or a file dropped into a more critical download folder such as Downloads.

This is the confusion that often results from using a Full scan and why Microsoft only recommends using this once an infection is already suspected, since in that case it is best to try and determine whether any other undetected malware might also be present on a system.  Using the Full scan outside this situation or without understanding the ramifications mentioned above typically only creates confusion for those unskilled in the knowledge of how security applications actually work.

Rob
