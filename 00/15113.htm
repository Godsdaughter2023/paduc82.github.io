This is probably happening because your Normal.dotm has macro code that is generating an error. This can be caused by two things:

  • You've caught a PC macro virus that fails to execute completely on the Mac because of an invalid reference. For instance, it might be trying to access disk C:, which does not exist on the Mac.

In either case, you need to get rid of that code. Access is as you did before, then delete it. If it looks like actual programming code (that you did not write), kindly paste it here so I can take a look and tell you if you need to further disinfect your computer.

It is safe to open infected documents as long as you refuse macro execution.

andreas killer say:

Use this code in Sheet1

Private Sub Worksheet_Change(ByVal Target As Range)
  Select Case Target.Address(0, 0)
    Case "B1"
      With Sheet2
        .FindWhat = Target
        .LastAddress = ""
        .FindNext
      End With
  End Select
End Sub

and this code in Sheet2

Public FindWhat As String
Public LastAddress As String

Sub FindNext()
  Dim R As Range
  If LastAddress <> "" Then
    Set R = Me.Cells.FindNext(Me.Range(LastAddress))
  Else
    Set R = Me.Cells.Find(FindWhat, LookIn:=xlValues, LookAt:=xlWhole)
  End If
  LastAddress = ""
  If R Is Nothing Then
    MsgBox "Not found"
  Else
    Me.Activate
    R.Select
    LastAddress = R.Address
  End If
End Sub

rbrian say:

A Curious WINB10 USB / BIOS / CMOS Problem

I have a 4-year old Shuttle SH55-J2-BK-V1 Intel i5 PC which has been happily running WIN7 with no problems - which I decided to use as a Windows 10 Technical Preview test system. All was going nicely – then at some point when I attempted to boot the PC in the A.M. (after being off all night) I couldn’t enter the BIOS User password. It appeared that the USB ports were dead. I attached a different USB keyboard with similar results (so it wasn’t the keyboard). I then reset the BIOS CMOS and was able to boot the computer after setting the proper BIOS options, including the need to enter a BIOS User password. Again, after running WIN10 and shutting the PC down I was unable to use the USB keyboard to enter the BIOS User password at re-boot. I again reset the CMOS and configured the BIOS to not require a USER password. At that point I was able to repeatedly boot and re-boot after a hard power shutdown. And the keyboard and mouse worked O.K. once in WIN10. So, the USB ports were really not dead – only to the BIOS at boot time. (I also loaded the BIOS with the most recent release – still no happiness under WIN10 after a power shutdown.)

I then deleted the WIN10 boot and OS partitions and reloaded the WIN7 boot and OS partitions. All was well again. No problems entering a BIOS User or Administrator password at boot-time after shutting down the PC.

So – why would WIN10 prevent the PC from allowing the entry of a BIOS User or Administrator password during boot-time? What in WIN10 would cause USB ports to be unresponsive after a power-down reboot until the CMOS was reset?

For Windows, below is the solution (For Mac, I don't know how to trace the path but "application.startuppath" should give you the path)

Please look into C:\Users\User_Name\AppData\Roaming\Microsoft\Excel (replace user name with your user name) and see if is contains few files / folders. Move those files to some other location (i.e create a backup of those files / folders and delete all files / folders from here). Hence, make Excel folder blank.

Now open the file and see if the problem disappears or not. 

Note - to find Excel path, you need to find XLSTART path. Excel folder contains XLSTART folder in itself.

1. Open Excel
2. ALT+F11
3. If Immediate Window is not visible, type CTRL+G. Same can be also be accessed through View > Immediate Window
4. Type "? application.StartupPath" without quotes and press enter
5. You will have your path below.
