Try this procedure...

patricia yu say:

Hi,

Before anything else, verify if the Outlook Address Book service is added to the account. To check, follow the steps below:

  1. Click on File, and then select Account Settings.
  2. Click Account Settings, and then the Address Books tab.
  3. If the Outlook Address Book service has already been added, mark your contacts folder an as Outlook Address Book.
  4. If it's not been added yet, click on New or Add.
  5. Select Additional Address Book and then click on Next.
  6. Select Outlook Address Book, and then click Next.
  7. There will be a prompt saying that you must close and restart Outlook for changes to take effect.
  8. Click on OK, close, and then restart Outlook.

 Also check if there's a .PST file in the PC. Search for *.PST files and check if you find any that will have your contacts in Outlook. If you find it, you can simply open the data file in Outlook.

Here's how:

  1. Click File, and then Open.
  2. Select Open Outlook Data File.
  3. Locate the PST file and then click on Open.

Regards.

It's the same problem. You just have to place the code in a place where it is still running when your file is already closed.

Add this code into the code module ThisDocument of your NORMAL.DOT

--- schnipp ---
Option Explicit

Public WithEvents App As Application

Private Sub App_NewDocument(ByVal Doc As Document)
  Doc.ActiveWindow.DocumentMap = False

Then add a normal module to your NORMAL.DOT and add this code

--- schnipp ---
Sub AutoExec()

Save the file, exit and reopen Word. And remove the Sub Document_New() from your document, it's superfluous.

rich007a say:

Hi,

I can't test this, as I don't have Word 2010, but I thought I might post this just in case.

    If Application.CommandBars.GetPressedMso("NavigationPaneShowHide") Then
        Application.CommandBars.ExecuteMso ("NavigationPaneShowHide")
    End If

Does that do what you need?

Use GetPressedMso to see if a built-in toggle button on the Ribbon is pressed.

Use ExecuteMso to push a built-in toggle button on the Ribbon.

The text ("NavigationPaneShowHide" in this case) to use in the brackets is the idMso of the control.

You can download a complete list of 2010 idMso's from here:

http://www.microsoft.com/downloads/details.aspx?familyid=3F2FE784-610E-4BF1-8143-41E481993AC6&displaylang=en

Alternatively, right-click on a control and add it to the QAT.  Then right-click on the ribbon again and Customize the ribbon.  If you hover over a control in the list it tells you the idMso in the pop-up.

Hope that helps.
