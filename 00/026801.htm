jeeped say:
UAC or User Account Control is built in to the Windows 7 operating system unless it has been disabled. Due to everyone under the sun running their computer logged in as either the or an administrator, many drive-by viruses and/or malware found their way on people's systems and to make it worse, they were installed with administrative rights as these were inherited from the currently logged on user.

Microsoft created UAC to counteract this and gave the Administrator of a computer no more rights than what was previously assigned to a Power User. If any action of real consequence was to be performed, the UAC would stop the process and ask for confirmation to proceed. If the current user was not a member of the Administrators user group, it would also ask for the username and password credentials of an administrator.

If you are not getting a UAC confirmation when you right-click and run as administrator then your UAC is likely turned off. I have no idea how this would affect the acceptance of the Office 2013 license agreement under Windows 7 but I would advise you to turn to back on again unless you are absolutely sure that you have sufficient expertise to never click on something you shouldn't.

Hi Marten,

 

About the Use Office 2016 to sync Office files feature, to answer your questions:

1.  We won't say "conflicts". If we turn off the feature in OneDrive sync client, when we edit an Office file in OneDrive local synced folder, we can't see any changes that other people make to the document in real time , and the changes you and othe people make to the frile will no longer merge together automatically. But when we try to save the file, it will prompt us to refresh the document to update the document with the changes made by others manually.

2.  Yes.

3.  Yes, but it will sync the changes before we close the file.

 

About the Microsoft Upload Center and "Why does Office save the files to OneDrive Online", we can understand in the following way:

1.  When we try to upload a file to a web server, Microsoft first saves that file locally to the Office Document Cache before it starts upload.

2.  If there is no problem when connecting to server, it will upload the cache from the Office document cache to Server directly.

3.  But if we encounter some problem when connecting to server, since the changes have been save to the Office document cache and we can continue working. The Microsoft Office Upload Center lets you keep track of how uploads are progressing, and whether any files need your attention. When we can connect to the server, it will upload the changes to server again. For the detailed information, I would suggest you refer to Microsoft Office Upload Center.


 

About the checkbox "Save to PC by default", it does not affect the above things. It matters when we try to save a new document created in Office application:

1.  If it is checked, when we click Save button in Office application, it will choose to save to This PC and the corresponding location by default.

2.If it is not checked, when we click Save button, it will choose to save to Recent locations by default, not This PC.
