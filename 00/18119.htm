Microsoft Access 2013 - Changing Null value to "N/A" - Microsoft Community

yok33 say:

Microsoft Access 2013 - Changing Null value to "N/A"

I have a database in Access 2013.  There is a number field in my table that is set to fixed 2 place decimal.  I also have a form for entering the data into this table.  I am wondering if there is any way to design either the table field or the form field or both so that if nothing is entered in this field (Null), then the form and table would automatically fill the field with the text value of "N/A".

ken sheridan say: Not if the column is of a number data type.  You would be able to return N/A in an unbound control in a form or report by setting the ControlSource property to:

=Nz([YourColumnName],"N/A")

but you would be unable to edit the data in the control.  To achieve both you would need two controls in the form, one bound, one unbound.  You could then switch the visibility of each control so that the bound control becomes visible only when you wish to edit the control, e.g. by having a small button beside the control which toggles the Visible property of each between True and False.

john w. vinson say:

You can't store a text string in a Number field (actually I'd suggest changing the datatype to Currency; the default Number type is Long Integer and doesn't support decimal places). But you can display N/A using a Format for the textbox on the Form or Report. One obscure option on Format lets you set four formats for a number field - positive, negative, zero and NULL; so you could use

"#0.00";"-#0.00";"0.00";"N/A"

ken sheridan say: Good answer.  I'd forgotten about the ability of the Format property to do that.
