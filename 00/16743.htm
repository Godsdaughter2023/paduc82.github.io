a. user say:

What if you use the following field code and format the entire field as 12pt (where "Text1" is the bookmark name?

{ REF Text1 \*lower \*caps \*charformat }

Is that the format you need? Does it still go wrong? (It has been OK here, so far). If so, try removing the "REF"

{ Text1 \*lower \*caps \*charformat }

Normally the thing where the existing formatting is preserved and additional text is formatted differently is a consequence of a \*mergeformat switch that Word has a habit of inserting. Although { REF Text1 } and { Text1 } are supposed to be two different ways to code a REF field, Word does occasionally treat them differently. macropod say:
Your description suggests your cross-reference fields have the \* MERGEFORMAT switch applied. You should delete that switch. As for the cross-reference formatting in general, you could format the 'REF' part of the cross-reference field's code with the desired font characteristics and apply the \* CHARFORMAT switch to the field. Be aware, though, that whatever attributes you apply via a \* CHARFORMAT switch apply to the entire field. Within limits, you can also change a cross-reference field's capitalisation via other switches. doug robbins - mvp office apps & services (word) say:

You would need to use a little bit of VBA code as follows


With ActiveDocument
    If .FormFields("Check1").CheckBox.Value = True Then
        .Variables("varAltText").Value = vbCr & "some text"
        .range.Fields.Update

   Else

        .Variables("varAltText").Value = " "
    End If
End With


Insert a { DOCVARIABLE varAltText } field before the end of paragraph mark (¶) of the paragraph before where you want the alternate text to appear.


You must use Ctrl+F9 to insert the { } and Alt+F9 to toggle off the display of the field codes.
