    Auto updating of a date specific field - Microsoft Community

    mauriceeggleston say:

    Auto updating of a date specific field

    I have a small database application (Access 2016)  that must display the number of days since a person was admitted to the hospital.  Everytime the database is opened it must update all of the current files with the new value for days since admission.  I do not have any problem calculating the difference in dates for a current record but cannot figure out how to automatically update all current records when the database is opened anew.  Thanks

    john w. vinson say:

    Since this value will change daily, it should be calculated on demand in a Query or in the Control Source of a form or report textbox whenever it's needed. It need not - indeed SHOULD not - be stored anywhere in any table. Just use an expression like

    =DateDiff("d", NZ({AdmissionDate], Date()), Date())

    to display the number of days, or 0 if the patient has no recorded admission date yet.

    If your users are looking at data in the Tables... well, don't do that. Tables should be kept "under the hood"; users should see data only through Forms and Reports.
