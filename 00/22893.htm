suzanne s. barnhill say:
There are two approaches you can take here.
  1.  You can specify a default path for "Clipart pictures" at Office Button | Word Options | Advanced: File Locations. If you do this, Word will always default to the specified folder every time you use Insert | Picture. If you choose this option and, for a given job, are repeatedly using a different folder, you can temporarily add that folder to your Favorites.
  2.  If you prefer to have Word default to the most recently used folder (for pictures, not the MRU folder for documents), you will need to leave the path unspecified in File Locations. By default, when Office is installed, the path IS unspecified, but if you have at any point chosen a path, then you can't restore the "no selection" status through the File Locations dialog. Instead, you must edit the Windows Registry to remove the entry that tells Word where to look.
Open the Registry with Start | Run | Regedit (in Windows 7 you can use Windows Key + R to open the Run box), navigate to the key HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Word\Options, and look for the PICTURE-PATH entry, which should contain the clip-art location shown in Word's File Locations dialog. Delete just that entry (not the whole key).

Here is a very simple user-defined function

Function JOINCELLS(ByVal ref As Range) As String
    Dim C As Range
    For Each C In ref
        If Len(C) > 0 Then
            JOINCELLS = JOINCELLS & C & ", "
        End If
    Next
    JOINCELLS = Left(JOINCELLS, Len(JOINCELLS) - 2)
End Function

steve rindsberg say:

A caveat and a bonus:

 

Once you've created the link from Excel to PPT, the content will update in PPT as long as the XL file stays open.  As you make changes to the XL file, the changes will appear in PPT too. 

 

If you close and reopen the XL file, the PPT will cease updating automatically.

 

If you close/reopen the PPT with the XL file open, the connection gets re-established and automatic updates start happening again.

 

Now the handy bonus trick:

 

If you select the XL content you want to work with and give it a range name before linking to PPT, the link points to the rangename, not a specific set of cells.  For example, it might look like this if you named the range LinkMe:

 

C:\Users\[y'all]\Documents\LinkTest.xlsx!Sheet1!LinkMe

 

"So?" I hear you ask.  So if you insert more rows/columns within the area you've assigned the range name, they become part of the range.  And you get more rows or columns in PPT. 

 

Ex:  If LinkMe was originally assigned to Col A, rows 1 through 4:

 

Bob

Carol

Ted

Alice

 

and I later change that to

 

Bob

Steve

Carol

Glenna

Ted

kking7796

TripToTokyo

Alice

 

 ... then all of us appear in the linked table on the PPT slide.

 

Magic.  Very useful, cool magic.

glenna shaw say:

Actually it's select the cells in the Excel spreadsheet, then click Format As Table on the Excel toolbar.  Then it formats as an Excel table.  PPT honors the Table features once it's paste/linked because any changes made are in an instance of Excel.  But it doesn't respect the boundary of the table as it does in Named Ranges.  It maps directly cell by cell.  This is especially problematic with Pivot Tables which can dynamically change as data is updated.

 

If you've formatted the cells to look like a table instead of using Excel's table or Pivot table features, then the way to go is definitely Steve's method with the Named Ranges.  And it'll still honor the Excel formatting you've done.

 

It's just strange the way PowerPoint handles Excel Cells on a slide whether they're linked or embedded.  When you're editing them, it's in an instance of Excel, but when you're not editing them it's like PowerPoint renders them as an image which can easily be distorted if you don't maintain the Aspect Ratio.  It doesn't matter if you format them before or after you paste/link them into PowerPoint, you'll always be formatting them in Excel since PowerPoint will open an instance of Excel if you want to make changes.  That's why I always recommend folks do the formatting in Excel before paste/linking.  It's just a little easier.

 

Sucks indeed if you're trying to create a professional looking presentation.

According to http://support.microsoft.com/kb/2460049, formats for APA 6th and MLA 7th are definitely part of the SP1 package, and they do appear in my copy of Word 2010. Given those things, I suspect that something went wrong with applying SP1 on your computer.

Download a fresh copy of the SP1 installer from that page (in the Method 2 section, and make sure you choose the correct "bitness", 32 or 64) and reapply it. jay freedman say:
Most likely what you have is 32-bit Office running on 64-bit Windows, for which the correct download is the 32-bit SP1 (that is, you must match the "bitness" of Office, not Windows). To be certain of this, in any of the Office applications, click File and then click Help, and look on the right side under "About <product>". For example, my copy of Word shows "Version: 14.0.6129.5000 (32-bit)".

Next, click the "Additional Version and Copyright Information" link below that line. In the dialog that opens, the top line will include "SP1" if the service pack is already installed. If that doesn't appear, then my original guess was correct, that the previous attempt to apply the SP failed. In that case, just apply the new download.
