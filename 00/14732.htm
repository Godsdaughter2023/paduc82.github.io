Creating a chart using Macro - Microsoft Community

fameyyusoff say:

Creating a chart using Macro

Hi all,

I have to create charts, more specifically the box and whiskers charts a lot from different data set, is it possible to do it using macro? I have tried recording function but it doesn't work. Thank you.

Famey.

andreas killer say:

In other words, I need the script, if possible though because I tried the recording but it doesn't work.

Have a look at the code below, it works, but not the way you like, resp. expect.

Note:
  It seems not possible to have different font/size for each axis title.
  The color of the plot area border is as the color of the line of the axis, but what you see on screen is different.

IMHO there is no way to solve that now, maybe in a few years when the VBA object model supports more and better adjustments.

Not all what you can to manually is supported, resp. possible with the VBA object model.

Andreas.

Sub Test()
  Dim Sh As Shape
  Dim C As Chart
  Dim Ay As Axis, Ax As Axis
  Dim DataRange As Range
 
  'Be sure the data is selected
  Set DataRange = Range("A1").CurrentRegion
  DataRange.Select
  'Create the chart with default settings
  Set Sh = ActiveSheet.Shapes.AddChart2(407, xlBoxwhisker)
  'Access the chart
  Set C = Sh.Chart
  C.HasLegend = False
  'Get the axis
  Set Ax = C.Axes(XlAxisType.xlCategory)
  Set Ay = C.Axes(XlAxisType.xlValue)
  'Scale the vertical axis
  With WorksheetFunction
    Ay.MinimumScale = .RoundDown(.Min(DataRange.Columns(2)), 0) - 1
    Ay.MaximumScale = .RoundUp(.Max(DataRange.Columns(2)), 0) + 1
  End With
 
  '1. to get rid of the Primary Major Horizontal Gridlines
  Ay.HasMajorGridlines = False
 
  '2. tick marks for both axis to be outside the line
  Ay.MajorTickMark = xlTickMarkOutside
  Ax.MajorTickMark = xlTickMarkOutside
  Ay.MinorTickMark = xlTickMarkOutside
  Ax.MinorTickMark = xlTickMarkOutside
 
  '3. title for vertical axis: font - Arial 10; no Bold
  With Ay
    .HasTitle = True
    With .AxisTitle
      With .Format.TextFrame2.TextRange.Font
        .Name = "Arial"
        .Size = 10
        .Bold = False
      End With
      .Caption = DataRange.Cells(1, 2)
    End With
  End With
 
  '4. title for horizontal axis: font - Times New Roman 12; Bold
  With Ax
    .HasTitle = True
    With .AxisTitle
      With .Format.TextFrame2.TextRange.Font
        .Name = "Times New Roman"
        .Size = 12
        .Bold = True
      End With
      .Caption = DataRange.Cells(1, 1)
    End With
  End With
 
  '5. Border for the plot area with line thickness and color matches the horizontal line axis
  With C.PlotArea.Format.Line
    .Style = msoLineThinThick
    .ForeColor.RGB = Ax.Format.Line.ForeColor.RGB
    .Visible = True
  End With
End Sub

john w. vinson say:

Well, you do need to get your tables and relationships right before you start designing forms. The first step is to define the relevant "Entities" for your project: real-life things, people or events relevant to your project. Each type of Entity gets its own Table; the fields in that table are "attributes" of that entity, discrete bits of information that you need to know about any instance of that entity.

So far the entities that I see are Spaces (650 records); Inspections, which would be related one-to-many to Spaces since each Space can be inspected several times; Treatments, ditto. I'd assume that the space would have a unique combination of building and apartment number, though you may want to use an Autonumber SpaceID as the primary key for convenience in linking to other tables. You may want an "adjacency" table with fields SpaceID1 and SpaceID2, where a record indicates that these two spaces (which might have quite different IDs and apartment numbers) are physically close (e.g. Apt 202 might be up a flight of stairs from Apt 118).

The Inspections table would have an autonumber Primary Key, a link to the SpaceID, an inspection date, the ID of the inspector perhaps, and any other fields pertaining to the inspection event itself.

The Inspections table would also probably be related one to many to a table of Findings (cockroaches, fleas, Republicans, Socialists, or other pests... <g>), each perhaps with fields for severity, recommended treatment, and so on.

It would also be related one to many to a Treatments table indicating the treatment date, outcome and followup.

Once you have the tables set up - THEN you can work on a form to enter data into them; and only then would you start building Reports.

 

Good luck and don't hesitate to post back with more!
