Why unchecked programs in the exceptions list of windows firewall are - Microsoft Community

saul issus say:

Why unchecked programs in the exceptions list of windows firewall are still able to maintain internet connection?

Original Title : firewall exceptions

Can anyone explain how after I uncheck programs in the exceptions list of windows firewall, they are still able to maintain an internet connection ? I use XP sp3.

lemp say:
1.  What makes you think that these unnamed programs are still able to maintain an Internet connection?

2.  Windows firewall blocks unsolicited inbound connections.  Thus, for example, if you (or some malicious software), set up an FTP server running on your computer, Windows firewall will block a request from the Internet to your computer seeking to establish a connection and download some of your files. 

On the other hand, when your web browser sends a request to a website server to load a page, the incoming traffic from the web server is allowed because it is in response to a request made on your computer.

This distinction between inbound and outbound traffic is why some people insist that third-party firewalls, which typically block/monitor traffic in both directions are necessary.  The scenario is that you inadvertently introduce malware to your computer (in any number of ways), and then the malware "phones home" to establish a link with the bad guys, who then proceed to do whatever they want on your computer.
