dave peterson say:

If you look at VBA's help for .autofill, you'll see this:

Performs an autofill on the cells in the specified range. Variant.

expression.AutoFill(Destination, Type)
expression    Required. An expression that returns one of the objects in the
Applies To list.

Destination    Required Range object. The cells to be filled. The destination
must include the source range.

Type   Optional XlAutoFillType. Specifies the fill type.

==========

That note that the destination must include the source range means that you're
going to have to be careful with your selection.

So instead of adding a bunch of checking or beeping if there's an error, maybe
you can let your code determine what to fill.

For instance, if I want B2:B4 filled to match the last used cell in column A, I
could do:

Dim LastRow as long
with activesheet
  lastrow = .cells(.rows.count,"A").end(xlup).row
  'even add a small check
  if lastrow < 5 then
     beep
     exit sub
  end if
  .range("B2:B4").autofill destination:=.range("B2:B" & lastrow), _
        Type:=xlFillDefault
end with

greginmich say:

All of the methods for single file scanning in Windows Defender use the command line interface, so they don’t look quite as nice – but they do get the job done. We came up with this simple (no downloads or registry hacks) method for scanning single files and folders in Windows 8 – but this should also work fine in Windows 10. I actually like the drag-and-drop version of this best; and it wasn’t so long ago that this kind of drag-and-drop scanning was one of the “advanced” features in Kaspersky AV products. But this method can also be easily adapted to the context menu – so you can use it either way, or both ways:

 

 

Open Notepad and copy and paste these two lines:

 

"C:\Program Files\Windows Defender\MpCmdRun.exe" -Scan -ScanType 3 -File %1

Pause

 

Name the file something like “Defender Scan”; give it a .bat file extension; and save it to your desktop. Now simply drag and drop the icon for a single file, a folder, or a flash drive onto the batch file icon for a scan. If you would prefer to use the context menu, then simply move the batch file to the C:\Users\<account name>\AppData\Roaming\Microsoft\Windows\SendTo folder, as suggested by manfeildman in this thread:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-you-scan-a-file-in-windows-8/3277ad77-b400-47c1-8508-1dfbf1729d3d

accidentally deleted auto text (file and path) Word 2003 - Microsoft Community

teegwen say:

accidentally deleted auto text (file and path) Word 2003

I accidentally deleted auto text (file and path) Word 2003 and cannot get it back.  Have followed instructions on Microsoft website.  But all it get is the "text": 

file and path - NOT the "formula":  C:\My Documents\research .  How do I add it back in so that it will insert the file and path into my footer?  Help!

suzanne s. barnhill say:
  1. In a header or footer (so that it will be in the Header/Footer style), press Ctrl+F9 to insert a pair of field delimiters (the things that look like braces but can't be typed from the keyboard).
  2. Between the braces, type FILENAME \p so that you have this
    { FILENAME \p }
  3. Select the entire field and press Alt+F3. This will open the Create Auto Text dialog.
  4. Type a name for your entry (the original entry is called "Filename and path").
  5. Click OK.
  6. You should now see the "Filename and path" AutoText entry on the AutoText menu in headers and footers, and you can insert it anywhere by typing "filename" and pressing F3.
