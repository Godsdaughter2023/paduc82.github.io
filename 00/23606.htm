don cuthbert say:

If you Google to find it, there is probably some Russian website that has made it available for download.

I run Insider Preview builds in Hyper-V which is using a virtual machine.  Sometimes I will install it on another partition on my hard disk if I want to test my specific hardware with the preview build.  This way I always have the public, stable build to use day-by-day for normal use.  The preview builds do not get as thoroughly tested as they would if I depended on them, but this way, the virtual machine can be downloading a new build, etc. in the background, while I do other things.  You could use Oracle VirtualBox if you don't have Hyper-V.  Hyper-V is in Windows 10 Pro.

The dual-boot option is another way that is recommended.  You don't need to dedicate much space for the Insider Preview partition - maybe 32-64GB.  Then if there is a Preview Build that barely works on your equipment you can just switch back to the public build until a new Preview comes out that might work better for you.

So one way to get the latest build is to install Windows 10 public build in a virtual machine and join that VM to the Fast Ring.  Once it downloads the latest Insider Preview, you have the Install.esd file on the VM.  You can copy that to your physical C: drive to use for creating an ISO for the latest build.  Then you can install it anywhere.  That is what I always do for each new Insider Build released.  That way I can clean-install any build that comes along if I choose to.

So I created a new document (We'll call it Test1), and made a copy of the sheet containing all of the pivot tables (from the original document) in this new file. Then I created a new query from a file that referenced the master excel file (call that Master1) with all the relevant data. This populated a separate sheet in the Test1 document in sheet 2. I then changed the data source of the Test1 pivot tables (on sheet 1) to reference the information that was brought in to sheet 2.

I set the both the pivot tables in sheet 1 and the referenced data in  sheet 2 to AutoRefresh on start, but this is where I ran into my problem. If I change data in Master1 then save the changes and then go and open my Test1 document, I find that sheet 2 (that is pulling it's data from Master1) has reflected the changes I made, but the Pivot tables in sheet 1 do not. jackcozart say:

Tom! 

You're the best! I've been banging my head against the wall about this for hours. Thank you so much for your help.

To anyone with this problem in the future: 

-I went into the sheet with the imported data and selected the connections properties, and in that sub menu I unchecked "Background refresh".
