In this case then I agree that the link I gave you is not appropriate.

 

- - - - - - - - - --

I think that the first steps, given these symptoms, should be checks for updated drivers. 

 

Go to Control panel, [View by - Small icons, if necessary], right-click on Device manager & select Run as Admin. 

Make sure you have an internet connection.

Select your display, right-click, select Update driver.

Select your hard drive, right-click, select Update driver.

 

- - - - - - - - -

You could also run the Control panel, Troubleshooting, Power troubleshooter but all it has ever done for me is argue the toss about how long I've set before my display turns off.  It is also at http://windows.microsoft.com/en-US/windows7/Open-the-Power-troubleshooter

 

- - - - - -- - -

I don't think powercfg is going to help you much.  If you just use it to report a list of properties then you can manually check through the list to see if the intended settings are being saved correctly.  I think they will be from your comments. 

 

Make sure you are in the power plan that you are investigating then enter this in a Command prompt:-

 

powercfg -qh >C:\Users\YourUserName\ThisPlanProperties.txt

 

then repeat this to give you something to compare it to.  So switch to the Balanced power plan & enter this in a Command prompt:-

 

powercfg -qh >C:\Users\YourUserName\BalancedPlanProperties.txt

 

[by the way, the -q argument is documented but -qh is not; -qh just makes sure that "hidden" properties are included in the powercfg output.]

 

Open the text files & paste them each into a column of a spreadsheet so you can see them side by side [you might have to insert a few cells here & there to get everything properly lined up].  Then look through them for the entries you want and look through the plans' properties using the normal Control panel, Power plan, Advanced settings route as well to help you understand what each one is.  

 

To understand the entries better, you can also refer to PMPolicy_Windows.docx, see Power Policy Configuration and Deployment in Windows.  This document will make you want to tear your hair out if you look at it now but will make more sense after you have seen the list of properties produced by powercfg.

a. user say:

Cindy,
The first error from the event log indicates that the VZAccess software is failing to connect, but it does not say why.  The second error is not related to this issue, it is about a missing Real Player file.  The third error is unrelated and can safely be ignored.

I did find this on a forum for Verizon mobile broadband users:
Open up your VZAccess Manager, press ctrl, T, on your keyboard.
a command prompt should come up
type in 2 commands,
type "AT$QCMIPEP=1" then hit enter on your keyboard.
Then, type "AT$QCMIP=1" and hit enter on your keyboard,
it should say ok after each command
press Alt, F4 to close the terminal

If that does not work, try doing a system restore to a date before the problem occurred.
To restore the operating system to an earlier point in time, follow these steps:
Click, type system restore in the Start Search box, and then click System Restore in the Programs list.
If you are prompted for an administrator password or confirmation, type your password or click Continue.
In the System Restore dialog box, click Choose a different restore point, and then click Next.
In the list of restore points, click a restore point that was created before you began to experience the issue, and then click Next.
Click Finish.

The computer restarts, and the system files and settings are returned to the state that they were in at the time that the restore point was created.

Let us know if any of this helps or if you need further assistance.
