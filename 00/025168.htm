bob jones aka: cybertaz say:

As you seem to realize, creating a PDF is a print process. A PDF is subject to the same constraints as if printing to paper. Although Word will permit laying out the document edge-to-edge it's the print driver which is raising the objection. If you do not have a print driver installed which offers Borderless options you may not be able to do what you are attempting to do.

The default print driver [Any Printer] does not offer Borderless & observes generic minimum margin settings. Further, it will not print content that extends beyond those margins.

The only suggestion I can offer is to install a driver which provides an A4 Borderless paper size option. It isn't necessary to have the printer physically connected to the Mac. Select that printer when generating the PDF.

a. user say:

I've posted this in a couple of places, since it is the only thing that worked for me.

Maybe it will help someone else.

I too had such serious problems with build 15014 and then trying to get to build 15019. On win 10 build 15014, I could barely run with it, never do an update of it to 15019, and finally could not even boot. Fortunately I had an image backup of the C drive from 3 days earlier of 15014 that was restored. But first I followed the advice in "TIPS" from this site:

https://www.tenforums.com/tutorials/2570-esd-iso-create-bootable-iso-windows-10-esd-file-73.html#post886726 .

The KEY for me was to use the recommendations to change the registry to disable UUP as suggested and to use the DISK CLEANUP as suggested.

Here were the suggestions in case you can't find them:

Both of the following DWORDs have by default value 1 (UUP enabled). You must change both values to 0 (UUP disabled) to keep getting the ESD file and full build download:

Key: HKEY_LOCAL_MACHINE>SOFTWARE>Microsoft>Windows>CurrentVersion>WindowsUpdate>Orchestrator
DWORD: EnableUUPScan

Key: HKEY_LOCAL_MACHINE>SOFTWARE>Microsoft>WindowsUpdate
DWORD: SupportsUUP

Then for my system, I did the DISK CLEANUP on my system C drive, and I cleaned absolutely everything (since I still had a three day old image backup available).

Then, I did the windows update to 15019 with no problems, and everything is working fine now!!!
