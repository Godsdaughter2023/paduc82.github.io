OK - here is the answer *if* you installed Windows Media Center. 

Go to Task Scheduler > Task Scheduler Library > Windows > Media Center.

You'll see a task called "mcupdates_scheduled" it is set to "Run whether user is logged on or not" and in "conditions" it is set to "Wake the computer to run this task".

Hex is base 16. 

The built-in Windows 7 Calculator can work in Hex.  Click on View menu, Programmer then select the Hex entry on the left-hand side of the window.

DefaultSecureProtocols Value Protocol enabled

Don't be put off by letters appearing as Hex uses them for digits greater than 9.

lemp say:

The key word in the article is "bitmap" as in "The registry value is a DWORD bitmap."

Computers still operate using only two states:  on and off.  These are represented using binary numbers (base 2).  To make large binary numbers slightly easier for humans to work with, programmers started using base 16 numbers or hexadecimal.

0x is a standard hexadecimal notation.  Each hex "digit" represents four binary "digits"

Considering only the right-most three hex digits:

0x008 (hex) translates to binary as 0000 0000 1000

0x020 (hex) translates to binary as 0000 0010 0000

0x080 (hex) translates to binary as 0000 1000 0000

0x200 (hex) translates to binary as 0010 0000 0000

0x800 (hex) translates to binary as 1000 0000 0000

Note that the "flags" are carefully chosen so that there is only 1 in any given place.

Adding them all up, you get 1010 1010 1000

Translating back to hex results in A A 8
