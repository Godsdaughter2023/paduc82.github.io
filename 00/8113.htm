qa guy commelec say:

Was missing closing parenthesis --


TRANSFORM Sum([CO_DEPT_EXP UNION].SUMOFDEPTCOST) AS SUMOFDEPTCOST
SELECT IIF([CO_DEPT_EXP UNION].COST_CENTER LIKE "MAINT" & "*", "MAINT DEPT",  [CO_DEPT_EXP UNION].COST_CENTER)  AS [Cost Center]                                                                
FROM [CO_DEPT_EXP UNION]
GROUP BY IIF([CO_DEPT_EXP UNION].COST_CENTER LIKE "MAINT" & "*", "MAINT DEPT",  [CO_DEPT_EXP UNION].COST_CENTER)  
PIVOT [CO_DEPT_EXP UNION].TBL;

ken sheridan say: You could leave the tables as at present and return a computed column in the UNION query with an expression:

Switch(Cost_Center = "MAINT DEPT","MAINTENANCE", Cost_Center = "MAINTENANCE DEPT","MAINTENANCE", True, Cost_Center)

Then use the computed column in the crosstab query.  But Ideally I would suggest that you correct the inconsistencies in the data by changing all instances of MAINTENANCE DEPT and  MAINT DEPT in the base tables to MAINTENANCE for instance.  This can be done with a simple 'update' query' per base table:

UPDATE YourTable
SET Cost_Center =
Switch(Cost_Center = "MAINT DEPT","MAINTENANCE", Cost_Center = "MAINTENANCE DEPT","MAINTENANCE", True, Cost_Center)

If for some reason you have to retain the original names, then you could add an additional column to the base tables, CostCenterGroup say, and then update this by executing a similar 'update' query per base table;

UPDATE YourTable
SET CostCenterGroup  =
Switch(Cost_Center = "MAINT DEPT","MAINTENANCE", Cost_Center = "MAINTENANCE DEPT","MAINTENANCE", True, Cost_Center)

Then use the CostCenterGroup column in your UNION and crosstab queries rather than the Cost_Center.

mike (microsoft) say:
Prof Julie has the right idea. When you sign into Windows Live there is an authentication token the phone gets. The token validity is determined server-side, and is based on whatever token policies the server has implemented. Often servers will accept tokens up to 8 hours old. So, whether or not you are presented for your password is a result of whether or not you do something that hits a service using a token older than the policy interval.

tasos_k say:

Hi,

try this...

 

Sub FindReplace_Array_01()
Const myCol As String = "B" '<<< search in column B
Dim x As Long
v1 = Array("ABC", "DEF", "GHI")
v2 = Array(123, 456, 789)
For x = 0 To UBound(v1)
Range(myCol & ":" & myCol).Replace What:=v1(x), Replacement:=v2(x), LookAt:=xlWhole
Next
End Sub
