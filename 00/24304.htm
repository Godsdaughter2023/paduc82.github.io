Try this:

FIND returns #VALUE! error if it doesn't find a reasult, so we can use the IFERROR function to handle this (the IF statement will never return "NO", but the IFERROR function will).

dhookom say:

Apparently you want the borders to all grow based on the height of another control. The method I use is to hide the borders and use code in the On Print event of the section to draw a rectangle around the unbound text boxes. Enter "Border" in the tag property of every control you want to grow. Your code in the report might look like:

Private Sub Detail_Print(Cancel As Integer, PrintCount As Integer)
    Dim intMaxHeight As Integer
    Dim ctl As Control
    intMaxHeight = Me.Goal.Height
    'Draw a box around each control in Detail _
      that has a tag property of "Border"
    For Each ctl In Me.Section(0).Controls
        If ctl.Tag = "Border" Then
            Me.Line (ctl.Left, ctl.Top)- _
                Step(ctl.Width, intMaxHeight), vbBlack, B
        End If
    Next
 
End Sub

Note: VBA doesn't run in all view modes.

rick rothstein, mvp - excel say:

Yes, the UserForm has a ScrollBars property which allows you to show either the horizontal scroll bar, vertical scrollbar or both (or none, of course, which is the default). You control the size and position of the scroll area within the UserForm's visible area using the ScrollLeft, ScrollTop, ScrollWidth and ScrollHeight properties. These properties are all available in the Properties Window for the UserForm.

Thanks for this useful information Sir.

Accutually i want to add this scrollbars at design time Bcz i've a lot of controlls on my user form, so what should i do.

rick rothstein, mvp - excel say:
Thanks for this useful information Sir.

You are quite welcome.

Accutually i want to add this scrollbars at design time Bcz i've a lot of controlls on my user form, so what should i do.

You will be setting the ScrollBars property at design time using the Properties Window (right click the UserForm and select Properties from the popup menu that appears).

so what should i do.

It kind of depends on how big you will need the UserForm to be. First, set the ScrollBars property to whichever of its settings will apply to your needs (horizontal, vertical or both... kind of depends on your layout and how you want the user to traverse your UserForm). Next, if you can enlarge the UserForm large enough to accomodate your controls, then do so and place your controls on the UserForm, then resize the UserForm to the final, smaller size that you want. If you can't make the UserForm big enough, then you will have to set the ScrollHeight and ScrollWidth properties to values large enough to accomodate the controls you want to put on it (don't be afraid to make it larger than you think you will need as you can change the ScrollHeight and ScrollWidth properties later on so that the form is "just the right size") and then put your controls on it (once you set the ScrollBars property, you will actually be able to scroll the form using the scrollbars that appear in order to get to the "hidden" areas). From here, you would create the event code for the controls just like for any other UserForm project.
