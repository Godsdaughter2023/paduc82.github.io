When you do the free upgrade, and activate with Windows 10, a digital hardware fingerprint of that PC and what version of 10 was installed is saved on the activation server. Then when you clean install that same version on that same hardware (PC) the activation server rechecks the hardware hash. When they match it activates that PC. It's refered to as a digital entitlement. PC's activated with a digital entitlement get a generic key. There is one for Home and one for Pro etc. My for PC's here at home were all upgraded from 8.1 Pro to 10 Pro. They all have the same generic Pro product code. 7 Home gets you 10 Home. So you would install 10 Home and click skip when asked for a key. Home should install and activate with the original digital entitlement you got with the free upgrade. I've done it here at home and verified it works, only in my case I select Pro. Nothings perfect though and some have had issues. Are you installing build 10240 or 10586? 10240 media creation tool downloads are Home or Pro. 10586 media creation tool downloads are multi edition, Home and Pro. You get a selection menu and have to pick which one to install.

TrustedInstaller is used by a service called “Windows Module Installer” which is used to install Windows 7 system modules. The system modules include Windows Update and Automatic Updates which automatically scans your system to check for any new updates and hotfixes available. So when scheduled Windows Update is performing its update scan or check on your system, the TrustedInstaller.exe process increases the CPU usage.

Also, the CBS.logs data file keeps on updating itself after each SFC Scan. The SFC.exe program writes the details of each verification operation and of each repair operation to the CBS.log file. The CBS.persist.log is generated when the CBS gets to be around 50 meg in size. CBS.log is copied to cbs.persist.log and a new cbs.log file is started.

These logs are useful only for troubleshooting issues. If you are sure your system is running fine, you can delete this file. SFC.exe will create a new one automatically, next time it is run.

The simplest resolution for the TrustedInstaller high CPU usage problem is to wait for a while for the TrustedInstaller.exe to complete its job. Once it finishes running the tasks, the CPU usage rate will return to normal.

However, if you want to stop TrustedInstaller.exe from taking up all your CPU resource, here’s a suggestion that may help.

Step 1: Disable Windows from automatically updating.

Here’s how: Turn automatic updating on or off
http://windows.microsoft.com/en-us/windows/turn-automatic-updating-on-off#turn-automatic-updating-on-off=windows-7

After disabling automatic updates, depending on your preference, you can change the way Windows notify you about updates.

Refer this article: Change how Windows installs or notifies you about updates

arminatorx say:

I'd be careful of transferring hard drives from "fat" consoles to a slim console, as the hard drives have a serial number on their security sectors, and in theory the could then detect that you are using a "fat" drive in a "slim" console. 

If your slim console did not come with a slim compatible hard drive, you will have to buy an original drive just to be sure. 

However, if you want to increase the storage of your Xbox 360, you can simply connect any external USB hard drive to the console and use that as "Memory Unit" storage. This is not hard drive storage, so original Xbox games won't work with USB drives only, but if you can get a small Slim-compliant hard drive for cheap, and just want more storage space, you can simply add up to two 2TB USB hard drives to the console for additional download storage. 

If you have content on your old "fat" drives, you can try to get a transfer cable to connect these old drives via USB on your new console, and then transfer your stuff over to the slim's hard drive.
