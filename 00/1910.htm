marshall barton say:

The standard way to do that kind of thing is to avoid lookup fields like the plague.  They cause far more problems than any slight convenience you might gain.  Besides, you should never work directly in a table where you have no way to prevent a slight mistake from turning into a disaster.

Instead, use a form with a combo box for the city.  The combo box can then use your query as its row source.

john w. vinson say:

Lookup fields are misleading and very limited. I don't believe you can do this with a Table lookup field, though it's possible.

However, if you work with Access properly - and use a Form, rather than a table datasheet, for all your interaction with the data - it's dead easy; you would simply put a combo box on the form with a Control Source of the city, and a RowSource such as

SELECT DISTINCT City FROM [Patient] WHERE [Patient].[State] = Me![State] ORDER BY City;

Note that it's important to specify the state first - there are a whole lot of cities named Springfield in this country, not to mention at least three cities named Las Vegas.

You'll want to set Limit to List to No if you do this, so you can add a new city.
