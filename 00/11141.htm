rohn007 say:

As Stefan says, you will want to put a section break between each of the files when you finally merge them. You should already have section breaks between the portrait to landscape orientation changes. If that is how you split the files, by orientation, then be absolutely sure you have a section break separating the  files when you merge them (not using the Word "Merge" feature, that is something else altogether)

One way to merge them, when you are ready

  1. Open a new blank doc
  2. go to Insert tab / Text group / Object ddl /Text from File option
  3. select the first file name to merge
  4. <CTL END> to jump to end of document
  5. Page Layout tab / Page Setup group / Breaks ddl / select a Section break type, you probably want Next Page
  6. start at step 2 for next file

 

Another way is to create a blank file and insert a "RD" (Reference Document) field code for each file to be merged.

In theory there is another mechanism you could use, the "Master/sub" document feature. However, personally I have not had much luck with it. I first tried to use it in Word 97, and tested in 2010 beta. I simply wasn't satisfied with the experience. MS keeps trying to sell it, but ever since they copied it from WordPerfect (where it worked) in the late 1980's, it has not worked very well.

What are "go microsoft link id's"?

I see go microsoft link id numbers on my cookies history and am wondering what this is about??? Thank you
网游 say:

Hi,

Instead of listing the full web address, some web sites (like Microsoft) will use a database of the full (complete) web addresses of pages on their site.... This means that when the (real) web address changes all of the links and Favorites that are stored in client browsers are not broken.

eg... some of the menu links in IE use a web address to reference online documentation for help with IE.

it may actually be a web address like http://www.microsoft.com/ie/6/7/8/9/{some secret folder name}/HelpIndex.aspx

but is stored locallly as http://www.microsoft.com/?linkID=1234

If the 'real' web address changes in the future, to say HelpIE9.aspx then only the Microsoft database has to be updated.
