    Making a record of queries as they are completed - Microsoft Community

    scorpiorc say:

    Making a record of queries as they are completed

    I have a macro that runs a large number of queries, some of which take a very long time to run.  There are times when a in the macro query will error out, and I need to know what query was the last to be completed so I can start the macro again from that point.  Is there some way I can write VBA code so that I can have the name of a query be appended to a table after it completes?  Or maybe some other way to track the queries as they finish?
    john w. vinson say:

    I'd suggest a table-driven approach. You could have a ToDoList table with the names of your queries; it might have other fields such as the date that this query should be run, etc.

    Your code would open a Recordset based on this to-do list, sorted by some field or fields specifying the required order of execution, and then step through the recordset using MoveNext. Each query would be executed in turn, and a record written to a log table indicating success. Untested air code:

    Dim rs As DAO.Recordset
    Dim rsLog As DAO.Recordset
    Dim db As DAO.Database
    On Error GoTo Proc_Error
    Set db = CurrentDb
    Set rs = db.OpenRecordset "ToDoList", dbOpenDynaset ' or a query on ToDoList
    Set rsLog = dbOpenRecordset "LogTable", dbOpenDynaset
    Do Until rs.EOF
      db.Execute rs!QueryName, dbFailOnError
      rsLog.AddNew
         rsLog!QueryName = rs!QueryName
         rsLog!DateRun = Now
         rsLog!ItemsAffected = rs.ItemsAffected
       rsLog.Update
       rs.MoveNext
    Loop
    Proc_Exit:
      Exit Sub
    Proc_Error:
      <report and handle any query errors>

     

    This approach would give you more flexibility with which action queries get run, as you could encode all that logic in your ToDoList table rather than having it embedded in a macro or VBA routine.

     

    john w. vinson say:

    You do indeed need both tables: a To Do list table and a log table. I don't know what names would make sense to you, I just chose names that make sense to me: ToDoList with (at least) a field QueryName, and a table LogTable with fields QueryName, DateRun and ItemsAffected. I'd suggest changing the fieldname in your ToDoList to QueryName since Query is a reserved word; you should also be sure to have a line

    Option Explicit

    at the very top of your code window (before the first Sub line) to help trap code errors.
