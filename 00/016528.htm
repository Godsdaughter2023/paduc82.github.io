chip pearson say:

You've hit a rounding problem. Not all fractional numbers can be stored exactly in a computer. This is analogus to regular math. For example, it is impossible to express 1/3 + 1/3 + 1/3 = 1 with a finite number of decimal places. No matter how far out you go, you'll never get exactly 1; the best you'll get is 0.99999...... The same thing is happening with yoru formulas. The value 1.2 cannot be stored exactly in binary math. You get an approximation to about 15 digits. This is my no means exclusive to Excel. Nearly all software use a data type called a Double Precision Floating Point, whose structure and operations are standardized by the Institute Of Electrical And Electronic Engineers.

Professional programmers are well aware of this limitation and code around it. Rather then

If 1.2-(0.12*10) = 0 Then

we use code like

If Abs(1.2-(0.12*)) < 0.0000001 Then

You can use the ROUND function to round the operands and/or the small epsilon method of testing < 0.00000001.

In the end, though, it is not a bug. It is just the nature of the machine.

See www.cpearson.com/Excel/Rounding.htm for more information about rounding and how to cope with it.
