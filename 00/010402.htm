After checking the SR, the case has been closed and there’s a workaround I’d like to share with you:

• Remove-MailboxPermission “Shared Mailbox Email ID” –User “User email ID” –AccessRights ‘FullAccess’
• Add-MailboxPermission “Shared Mailbox Email ID” –User “User Email ID” –AccessRights ‘FullAccess’ –Automapping $false
• Remove / Rename OST and XML files from the following locations
• C:\users\User name\Appdata\local\Microsoft\Outlook
• Configure a new profile using SARA.
https://support.microsoft.com/en-us/kb/3073666
• In Outlook – File – Account settings – Account settings – Select the account – Change – More Settings – Advanced – Add the shared mailbox – Apply – Ok
