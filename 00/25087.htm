FIXED 12/17/15 - Word crashes if you use custom keyboard - Microsoft Community

anneliese [msft] say:

FIXED 12/17/15 - Word crashes if you use custom keyboard layouts

******************************************************************************************************************************

UPDATE: On12/17/15, we released a fix for the Word-specific custom keyboard layout bug. You can get this latest update by clicking Check for Updates on the Help menu (if you don't already have it by now), or by downloading the update here. Thanks for your patience.

*****************************************************************************************************************************

We've heard your reports that after installing update 15.17, Word will crash if you have customer keyboard layouts. We're working on the fix now. In the meantime, if this bug affects you, you can try one of the following workarounds:

Workaround #1: Disable custom keyboard layouts

  1. Go to System Preferences > Keyboard > Input Sources.
  2. Select a standard keyboard.

Of course, this workaround means that you can't use your custom keyboard layout until a fix is out. If that's not acceptable, you can try the following workaround.

Workaround #2: Enable only the .key layout file from custom keyboard bundle

  1. Go to your ~/Library/Keyboard Layouts folder.
  2. Find the custom keyboard layout file that you want to continue to use. It should end in ".bundle".
  3. Move it out of that folder to the desktop or another folder.
  4. In its new location, Control-click on the file and choose Show Package Contents.
  5. Find the file that ends in ".keylayout" in the folder that appears (most likely under Contents > Resources.)
  6. Make a copy of the file that ends in ".keylayout" and put it in the Keyboard Layouts folder you opened in step 1.
  7. Log out and log back in.
  8. Add the custom keyboard layout in the Input Sources tab of Keyboard System Preferences. (It will be listed under Other.)

Thanks!

suzanne s. barnhill say:

The default in Word 2010 and earlier is to align text in a table with text outside the table, which is desirable when table are borderless. But this means that table borders extend into the margin by the amount of the outside cell padding (left on the left side, right on the right).

In Word 2013, a change was made to align table borders (instead of the text in the table) with the page margins. For borderless tables, it is necessary to remove the outside cell padding in order to align the text inside the table with text outside it.

If you just want to achieve a simply send/receive function in Outlook, you don't need to create an empty archive folder for this.   2. You are moving from PC/Mac to PC/Mac.
