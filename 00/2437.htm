Grouping and Sorting must be done in the report, not in the query. The only character besides letters and numerals in table and field names should the underscore.  There is no need to enter today's date.

 

SELECT [Vendor].[Name], [Contract].[Contract_#], [Purchase_Order].[Purchase_order_#], [Contract].[End_Date], [Contract].[Original_Lease_Amount], [Purchase_Order_Line].[Description], [Cost_Center].[Cost_Center], [Cost_Center].[Department], Sum([Contract].[Monthly_Rental_Amount]) AS [Monthly_Cost_Center_Rental_Amount]

FROM (((Vendor LEFT JOIN [Purchase_Order] ON [Vendor].[Vendor_#] = [Purchase_Order].[Vendor_#]) LEFT JOIN  [Contract] ON [Contract].[Contract_#] =  [Purchase_Order].[Contract_#]) LEFT JOIN [Purchase_Order_Line] ON [Purchase_Order_Line].[Purchase_order_#] = [Purchase_Order].[Purchase_order_#])

WHERE[Contract].[End_Date] <= DateAdd("m", 6, Date())

GROUP BY [Vendor].[Name], [Contract].[Contract_#], [Purchase_Order].[Purchase_order_#], [Contract].[End_Date], [Contract].[Original_Lease_Amount], [Purchase_Order_Line].[Description], [Cost_Center].[Cost_Center], [Cost_Center].[Department];

hansv mvp say:

I think what you want is a weighted average. The weighted average of the y-values is

 

=SUMPRODUCT(z-values; y-values)/SUM(z-values)

 

Let's take the example

 

             A           B
1      11000      1,46
2      12000      1,48
3      13000      1,49

 

The average of 1,46, 1,48 and 1,49 is 1,476666, but that doesn't produce the correct sum.

The weighted average is

 

=SUMPRODUCT(A1:A3;B1:B3)/SUM(A1:A3)

 

and this returns 1,4775. If you use this factor, you get the correct sum.

greginmich say:

When you install a third-party AV app, this will disable Windows Defender in order to prevent conflicts. But the conflicts between AV apps are limited strictly to their real-time protection modules, which use kernel-mode file system filters that can result in conflicts and performance issues when they’re stacked. So once you’ve installed a third-party app, you’ll have the option for running Windows Defender in its Limited Periodic Scanning mode, which includes all of Defender's usual features, with the exception of real-time protection and its dependencies. The dependencies include Controlled Folder Access, Attack Surface Reduction and Network Protection.

https://docs.microsoft.com/en-us/windows/security/threat-protection/windows-defender-antivirus/limited-periodic-scanning-windows-defender-antivirus

https://cloudblogs.microsoft.com/microsoftsecure/2016/05/26/limited-periodic-scanning-in-windows-10-to-provide-additional-malware-protection/?source=mmpc

The effect that installing a third-party AV app will have on the system's overall level of protection will depend on whether the real-time protection offered by the third-party app is better or worse than the default real-time protection offered by Windows Defender, and that in turn will depend largely on the way that Defender's real-time protection options have been configured:
