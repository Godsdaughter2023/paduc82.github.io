Is there an error message on-screen after the upgrade fails and you log back into 14915?

You could make an ISO for build 14931 from the ESD file on the VM, in the hidden folder at C:\$WINDOWS.~BT\Sources\Install.esd.  There is a utility (ESD Decrypter 4) that does this quite easily, described in this old discussion from April, 2015.

Create a new expandable virtual hard disk for your Windows 10 Insider Preview VM and configure it to use the new empty disk.  Your existing hard drive remains intact this way and can be used if you want to switch back to it.

Boot the VM with the 14931 ISO mounted in its DVD drive and you can install 14931 to the empty new hard disk. Assuming that the VM retains its Digital License for Windows 10, it should auto-activate once connected to the Internet.

___________________________________________________________________

Alternative setup:

Create the new empty virtual hard disk and add it to the VM in addition to the existing one.  Boot to build 14926 on the VM.  Use Disk Management or Diskpart on the VM to quick format the new hard disk as NTFS and give it a drive letter. Let's assume the new hard disk is drive E:

Go to a Command Prompt (Admin) on the VM.  Use DISM to put 14931 on the new hard disk using the commands below:

DISM /Apply-Image /ImageFile:C:\$WINDOWS.~BT\Sources\Install.esd /Index:4 /ApplyDir:E:\ /Verify

bcdboot E:\Windows

Exit the Command prompt and restart the VM and it should go through clean-installing build 14931 (on the new hard disk only).  Once it is done, (assuming that 14931 will run on that VM) it will have a dual-boot setup between 14931 and 14926.

If at some point you want to replace 14926 with the public build of Windows 10 (so that you always have a stable, activated build of Windows 10 version 1607 available to run) you can either install that by mounting it's ISO on the VM's DVD drive or format that virtual disk's Windows partition and use DISM to put the public build files on that virtual disk.
