Using Access with SQL Server Express - Microsoft Community

eric b. smith say:

Using Access with SQL Server Express

Hi,

What's involved in creating, distributing, and maintaining MS Access front ends with a SQL Server Express database as the back end?  Also, can SQL Server Express be installed on a server with a full version of SQL Server already on it?  Please don't ask why I would want to do this.  Let's just say the organization I work for has their own unique issues.

Eric

albert d. kallal say:

>What's involved in creating, distributing, and maintaining MS Access front ends with a SQL Server Express database as the back end? 

Very little changes from say a typical “split” database with an acccDB front end and an accDB back end.

If the front end is pre-linked to the database (an accDB back end, or SQL server back end), then when you distribute + install this front end, then they should be connected to the back end without any re-linking needing to occur.

>can SQL Server Express be installed on a server with a full version of SQL Server already on it?

If it is a “different” version of SQL server, then as far as I know this is possible.  However, I not sure if the same edition (say 2012 Full and express can be installed on the same machine. But if they have 2012, then install 2008 express for example.

So for the front end, one typical will have “built up” a set of routines that allows one to re-link the database to a different back end.

The “exact” same concept exists with SQL server back ends – you build a new set of re-linking routines that quite much re-links to the back end database of your choice.

While in most cases distributing a pre-linked database should work, one often will adopt some re-link routines anyway.

If you link (DSN less) to the SQL server, then you not need to re-link that front end when you distribute it. (and the same goes for a simple split FE/BE even without SQL server).

When you rename an account, the old account name will be the ALIAS of the new one.

 

Example:

*** Email address is removed for privacy *** was renamed to *** Email address is removed for privacy ***

*** Email address is removed for privacy *** will become the alias of the *** Email address is removed for privacy ***

 

In this case, you need to login using your new account because you won't be able to login using an alias account.

 

Please be informed that there is no way to rename an @Outlook.com account back to an @MSN.com account at this time. If you rename an MSN account, you can't undo the rename.
