Many to one Mail Merge for Mac

Are there any simple explanations or fixes for this that are compatible with a Mac? I'm using Office 2011. The page provided by Microsoft is pretty confusing.

a. user say:

1. Macropod's reply in the following conversation lists the approaches commonly suggested for windows word users in this community:
http://answers.microsoft.com/en-us/office/forum/office_2010-word/mail-merge-from-an-excel-spreadsheet-containing/d53b4370-357e-4c60-adef-9ea12772c19b

Of those, only the one by macropod will work unmodified on Mac, as it's mostly based on fields. (I don't have the latest version, but if you need to merge one-many to email, VBA code is also needed, and I haven't checked that on Mac). So if you have to do something quickly, that's probably your best bet. 

2. The problem with the other solutions is that they use features that are either not available on Mac at all (I.e. would need to be done a completely different way) or which need more minor modification to work on Mac. I would be willing to do some work on that if you are interested and not in too much of a hurry-let me know.

3. There are other windows word approaches, e.g.
- use a DATABASE field to insert the "many" rows for each record. It might be usable on Mac in some very simple situations, but on Mac, the way connections work with Excel (say) can be a problem.
- use MailMerge events and VBA. Not available on mac, where Word does not have those events.
- approaches based on content controls, .NET and so on. Not available on Mac, partly because it doesn't let you work with content controls.

4. Some people "roll their own" merge using VBA or some other language to automate Word. Also do-able on Mac, but a lot of work is involved to do it in a general-purpose way: easier if you have a single requirement.

(Extra: the other approach is to do a one-many report in a database product such as FileMaker, or perhaps one of the open source products. I'm not really up-to-date on what's available on that front)
