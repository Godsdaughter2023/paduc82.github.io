Field types in Access - Microsoft Community

ellensilverberg-brennan say:

Field types in Access

Do fields have to be "numeric" if you want to
create calculations in reports or queries?

How do I get the number to show 2 decimal places?

dirk goldgar say:
Do fields have to be "numeric" if you want to
create calculations in reports or queries?

How do I get the number to show 2 decimal places?


Arithmetic calculations can only be performed on numeric data.  However, under many common circumstances, if you use a string value in such a calculation, it will be converted to what the program thinks is a suitable numeric data type before performing the calculation.  For example, if you were to enter this in the Immediate Window:

 

    ?1 + "2"

 

The response would be:

     3

 

In general, if one of the operands is a string and the other is a numeric type, the string will be converted to a number (if possible).  But you have to watch out!  If you enter this in the Immediate Window:

 

    ?"1" + "2"

 

You'd get this result:


     12

 

That's because the "+" operator is also used for concatenating strings, and since you gave it two strings to work with, that's what it did.

 

In general, you should avoid storing numeric data as text if you are going to calculate with it.  Calculations will be more efficient and accurate if the values to be used are stored as the correct data types.

 

The number of decimal places displayed is a formatting option, and so is most easily controlled by the Format and Decimal Places properties of the text box that will be displaying the value.  If you choose one of the numeric formats, then the Decimal Places property controls how many decimal places are displayed.
