scottgem say:

First, if you are using a bound form you do not need to expressly save the record. The record is saved when it loses focus.

Second, If you set the Data entry to Yes, the form can only be used to enter new records. You cannot view existing records once you move off the record.

Third, Me.Requery is a VBA command and has to be run in a code module. So you need to use Code Builder, not just type the line in the Event box.

Finally, you said it only clears the left side of the form, not sure what you mean by that. Are you using a split form? Main form/sub form, or what?

ken sheridan say: The code must be entered in the Click event procedure, not directly into the properties sheet.

If you are unfamiliar with entering code into a form's or control's event procedures, this is how it's done in form design view:

1.  Select the form or control as appropriate and open its properties sheet if it's not already open.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new lines between these.

If you wish to force the user to explicitly save a record via a button, preventing the user from moving off an unsaved record or closing the form until the record has been saved via the button the means to do this are illustrated in SaveDemo.zip in my public
https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

scottgem say:

First, may I suggest that you not accept the default names for Controls (or other objects). Give your objects meaningful names. Six months from now you will not remember what Command31 is supposed to do whereas cmdSave will be clear.

 Second, yes, if you have unbound controls. (not fields), you have to clear them out using code. Going to a new record won't do it.

ken sheridan say: You only need one button and one event procedure for this.

To clear the unbound controls first set the Tag property of each to ClearMe.  Then in the form's Current event procedure put the following code:

Dim ctrl As Control

For Each ctrl  In Me.Controls
   If ctrl.Tag = "ClearMe" Then
       ctrl= Null
   End If
Next ctrl

If a value entered in one control immediately appears in another control then it suggests that both are bound to the same column in the form's RecordSource table or query.

Can this be done?

Thank you,

Cooz

greg maxey say:

Sub GetLabel(control As IRibbonControl, ByRef ReturnedVal)
  Select Case Application.International(wdProductLanguageID)
    Case 1033 'US English
      Select Case control.ID
        Case "custTab1": ReturnedVal = "U.S. English Tab Name"
        Case "Grp0": ReturnedVal = "U.S. English Group Name"
        Case "Grp0Btn1": ReturnedVal = "U.S. English Button Name"
      End Select
    Case 3804 'French Canadian
      Select Case control.ID
        Case "custTab1": ReturnedVal = "French Canadian Tab Name"
        Case "Grp0": ReturnedVal = "French Canadian Group Name"
        Case "Grp0Btn1": ReturnedVal = "French Canadian Button Name"
      End Select
    Case 4105 'English Canadian
      Select Case control.ID
        Case "custTab1": ReturnedVal = "English Canadian Tab Name"
        Case "Grp0": ReturnedVal = "English Canadian Group Name"
        Case "Grp0Btn1": ReturnedVal = "English Canadian Button Name"
      End Select
  End Select
End Sub

stanimal21 say:

When you pin the IE11 icon to the start screen it always uses the same icon whether it's the desktop version or the modern app version of Internet Explorer. You can, however, configure Internet Explorer 11 to only open on the desktop mode even when you click that "modern" looking icon:

  1. Go to Desktop mode and then open Internet Explorer on the desktop
  2. Click the Gear cog on the top-right and then select "Internet options"
  3. Click the "Programs" tab and then click the drop-down list under "Choose how to open links:"
  4. Choose "Always in Internet Explorer on the desktop"
  5. Select the checkbox "Open Internet Explorer tiles on the desktop"
  6. Select "OK" to save your settings
