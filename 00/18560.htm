herb tyson say:

What you're referring to are non-printing formatting characters. They can be toggled on/off by pressing Ctrl+Shift+8, or by clicking the paragraph symbol in the Paragraph section of the Home ribbon tab.

They can be extremely useful when trying to figure out why a document doesn't do what you expect it to do. For example, stray tabs and other non-printing characters can make spacing hard to understand, as can the presence of unbordered tables. So, while you might find the marks annoying right now, you might eventually find them to be useful.

VBA coding help - make a sheet name equals to a cell (loop from a - Microsoft Community

ll1100000 say:

VBA coding help - make a sheet name equals to a cell (loop from a certain sheet to the end)

I have some workbooks that each of them contains 20 to 30 worksheets.

I want to change the sheets name equals to cell A2 from the 4th sheet (count from left) to the last sheet in this workbook.

Then change the sheet name equals to cell B2 again from the 4th sheet (count from left) to the last sheet in this workbook.

The sheet name should equals to cell B2, but unfortunately, I can't just simply do a loop to change it to B2 directly because we can't have 2 sheets have the same sheet name in one workbook...

So I have change it to A2 first, then after all sheets (from the 4th sheet to the last) have been changed to A2,  then change it to B2.

Thank you.

Also, if I want to this macro to be executed when opening the workbook, how to add the coding for this....  Thank you.

hansv mvp say:

Change the line

            .Name = .Range("B2").Value

to

            If .Range("B2").Value = "" Then
                .Name = "Blank"
            Else
                .Name = .Range("B2").Value
            End If

Keep in mind that this will work once only - you cannot have multiple sheets named "Blank".

The line Application.ScreenUpdating = False temporarily stops Excel from updating the screen. This speeds up macro execution. At the end, Application.ScreenUpdating = True causes Excel to show the result of the changes.

The line On Error Resume Next suppresses all error messages from that point to the end of the macro. If we didn't use that, you'd get a Debug message each time B2 contains a character that isn't allowed in the name of a worksheet, such as [, ], /, \, ? or *. With On Error Resume Next, the error is simply ignored (and the sheet is not renamed).
