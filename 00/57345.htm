If you see event ID 7023 — Service Stop Operations log, that indicates that the Service Control Manager (SCM) stops services and driver services. It also reports when services terminate unexpectedly or fail to restart after it takes corrective action. Usually service control manager (SCM) is started at system boot. It is a remote procedure call (RPC) server, so that service configuration and service control programs can manipulate services on remote machines. It maintains a database of the installed services and driver services that allow the operating system to start successfully, and provides a unified and secure means of controlling them. The database, which is stored in the Windows system registry, includes configuration and security information about each service or driver service.

 

Troubleshooting this error will ask you to review the Windows error code that the Service Control Manager encountered when trying to stop the service, and then review the Event logs for other related events that have been logged by the Service Control Manager (SCM) Event log Provider.
