I need to add  ="selected cell"*(1-$A$10)  to the selected cells.

 

What function can I use?

 

Thank you


RXA887

teylyn say:
Hello,

a cell can either hold a value or a formula. If a cell has a value and you want to change it to have a formula, then you will need to edit the cell.

There is no formula to take an existing number value of a cell and replace it with a formula that encompasses the original number.

To achieve what you describe, you would need to run a VBA macro along these lines:

Sub test()
Dim cel As Range
For Each cel In Selection
    cel.Formula = "=" & cel.Value & "*(1-$A$10)"
Next cel
End Sub

Use Ctrl click to select the cells, then run the macro.

cheers, teylyn

4. Switch back to the Excel workbook and use Ctrl-click to select the cells you want to change.

5. Select the VBE window, click somewhere inside the code and hit F5. This will run the code on the selected cells.

Custom paper size errors - Microsoft Community

julie_vega say:

Custom paper size errors

We use a 4x6 index card to print a mailer. We never had any problems printing this custom size in Word 2007. Now that we have Word 2010 it will only print the last 1/3rd of the document on the card. There's an exclamation point next to the custom size. Any help would be greatly appreciated as I really don't want to re-create the card in a different program.  We have not changed our printer.

graham mayor say:
The issue is not so much with the printer per se, but its driver software. Word (all versions) format the document according to what the printer driver will allow. The driver sets the available page sizes and most are hopeless with custom paper sizes and Word 2010 appears to be even less tolerant than earlier versions. Check the printer driver manufacturer's web site for a driver update (for HP printers checkout also http://www.hp.com/pond/ljbeta/ )

Your best option for individual cards is to set the next larger available standard page size and make up the difference with the margins.

jllatham say:

I can't leave the files with the ".xls" substring because they end up having Office 1997-2003 and that format is not desirable.

And that is the answer to the problem - the copied versions do not offer a clue to the system as to what application to open them with.  Since you are simply copying and renaming them, the file format itself is still going to be Excel 97-2003, so you might as well leave the .xls as part of the file name (or more accurately, add it back to the new filename).

The only way to get them into a later format would be to have your code actually open each one and save the file to the new destination as the proper format type you desire, be it .xlsx, .xlsm, .xlsb or other.

The internal file formats for Excel 97-2003 and later versions are completely incompatible with one another: in 97-2003 everything is actually in a single file; with 2007 and later, what you actually have is a zipped file with several folders within it and files within those folders.  If you want to see this for yourself, take a .xlsx or .xlsm file and change the .xlsx or .xlsm to .zip and then open it in WinZip or 7Zip.  Just change the name back and it'll "become" an Excel 2007-2013 usable file again.
