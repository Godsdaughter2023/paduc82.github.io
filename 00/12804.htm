ken sheridan say: For the second combo box's RowSource property use a query which joins the T_Client_Join table to the T_Client_Entity on Entity ID and restrict it by a parameter on the ClientID column which references the first combo box.  You then just need set the second combo box to Null and requery it in the first combo box's AfterUpdate event procedure.

ken sheridan say: I assume the button to open the form is on the main plaintiff form.  As well as filtering the form to the current plaintiff You can pass their ID value to the form as its OpenArgs property, so the code would be something like this:

' open second form at new record,
' passing current plaintiff's ID to it
' as OpenArgs property
DoCmd.OpenForm "YourFormName", _
    DataMode:=acFormAdd, _
    WindowMode:=acDialog, _
    OpenArgs:=(Me.ID)

' requery subform to include new record
Me.YourSubformControl.Requery

The DataMode setting of acFormAdd cause the form to open at a new record.  The WindowMode setting of acDialog causes it to open in dialogue mode, which means the user must close it before returning to the main form, and it also causes the code execution to pause until the second form is closed, so the Requery method is not called until then.  The OpenArgs argument is used to pass the current ID value to the second form.  Note that YourSubformControl is the name of the control in the main form's Controls collection which houses the subform.

In the second form's Open Event procedure the value (if any) passed to it via the OpenArgs mechanism is assigned to the DefaultValue property of the PltfID control on the second form with:

If Not IsNull(Me.OpenArgs) Then
    Me.PltfID.DefaultValue = """" & Me.OpenArgs & """"
End If

If you don't want the PltfID to show in the second form just set the Visible property of the control to False (No).  Note that the DefaultValue property is always a string expression, regardless of the data type of the field in question, so should always be wrapped in literal quotes characters as above.

Excel 2007's Conditional Formatting are applied sequentially. So, if you create your rules in the right order, the CF will stop at the first one that matches.

Example:
with the cells to be colored in A2:L5000
and
Col_J contains the dates that determine the band color
and
O1: 04/16/2010 <-The beginning of the YELLOW band
O2: 06/02/2010 <-The beginning of the NO FILL COLOR band

• Select A2:L5000
• Home.Conditional_Formatting.Manage_Rules
...Click: New Rule...Use a formula to determine which cells to format
...=$J2<$O$1
...Click the Format button...set the fill color to RED...Click: OK


...Click: New Rule...Use a formula to determine which cells to format
...=$J2<$O$2
......Click the Format button...set the fill color to YELLOW

• Done

All rows with Col_J dates that are less than 04/16/2010 will highlight: RED
All rows with Col_J dates that are less than 06/02/2010 will highlight: YELLOW

sum columns in a range vba

Hello,

given a range Rng=Y6:AI377 (which has 11 colums), how could I choose what colum to sum or max?

(Rng is moved around a sheet with an offset function)

 

At the moment I am using sum + index, but was wondering if I could assign each colum with a number (1 to 11) and say something like:

Results= Application.sum(Rng (colum6)) or

Total= Application.sum(Rng(10))

 

 

Thanks

 

 

tom ogilvy say:

SST 1.0

 

here is a demo from the immediate window just to show how you can achieve what you want. 

 

set Rng=Range("Y6:AI377")
? rng.columns(6).Address
$AD$6:$AD$377


? application.Sum(rng)
 208213
? application.Sum(rng.columns(6))
 18453

 

so you don't have to number the columns - they are already "internally" numbered starting with 1. 

 

I did the sum of the whole range just to show that the sum of column 6 is a subset of the whole range.

stefan blom say:
On the Review tab, look at the Display for Review drop down (which you'll find in the Tracking group). If the selected option is "Simple Markup," change it to "All Markup."

suzanne s. barnhill say:
Clicking on one of the change bars ("changed lines") should toggle from Simple Markup to All Markup.
