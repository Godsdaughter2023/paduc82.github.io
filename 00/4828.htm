john w. vinson say:

The size of text fields actually isn't too important. Access trims off trailing blanks so if you have a 255 byte Text field containing "abc" it actually stores only 3 characters.

I don't think you can specify the field size in an Import, except with an import spec (from Text, not from Excel). But unless you will then be entering data into the table, the field size really doesn't matter.

gina whipp say:
Yep, you're right... been so long since I used an Import Specification never realized it didn't apply to Excel.  That said why is the size important?  As John pointed out it won't store *blank* spaces.  If you are trying to control how much data is in a field, you could import into a Temp table and then append to the live table...

Right-click in the first line of the works-cited page (which I guess is a title for the page) and click Paragraph in the menu. In the Paragraph dialog, go to the Line and Page Breaks tab and check the box for "Page break before". Then click OK.

That will make sure that paragraph will always be at the top of a new page, no matter how much or how little text is on the page before it.
