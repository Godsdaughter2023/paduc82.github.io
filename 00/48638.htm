argerrit say:

Some POP3 servers haven become unstable since the latest Outlook.com migration. There are three common problemens:

1) This one, which happened over the weekend for the first time The key in the error message is this: FederatedStsUnreachable

The POP3 server tries to check the password and username, but the Active Directory is not reachable. Probably something crashed, or got misconfigured on the Microsoft side. So POP3 cannot check the password, returns 'authentication failed' and the client (your email program) then 'just' assumes the password is incorrect. But it isn't.

Note this is a per server issue, so only some accounts are affected, while others are just fine. The same error occurs also with IMAP on affected accounts, as it is not a POP3 issue, but authentication forwarding.

Webbased and Exchange access to the same accounts will continue to work, in the mean time.

2) Another issue, only affecting POP3 it seems. On occasion, since the migration, any POP3 client (e.g. Windows Live Mail, Outlook 2010 or even a manual login using open_ssl/telnet) will see its connection dropped by the POP3 proxy server for no reason.

The behaviour is that the client will experience a loss of connection right after the Authentication fase. Logging manually in using open_ssl/telnet shows authentication actually works, but the first command issued (e.g. POP3 STAT for those familiar with the low level basics. But I tried various, it doesn't matter pretty much any command will disconnect) will result in a remote server disconnect.

This is a transient issue, and waiting 30 minutes or so makes it go away. Also logging in on the web interface and logging out, also seems to help. Exchange based access will work wile this issue is active, and I believe IMAP does as well. I experience this issue roughly 3 or 4 times a week.
 
Using a different IP (e.g. VPN) does not resolve the issue, so it really is a server-issue. This is further proven, by that this issue only affects certain accounts. And it seems to be the same set of accounts that are this weekend affected by the first issue. So it looks as if certain servers are just more buggy than others.

3) Just for completeness. When migrated, Microsoft seems to reset a lot of settings, so POP3 acces may have been disabled. So always log in with your web interface once and check POP access has not been reset. Typically though, the error message will actually shows this. If it is enabled, and you still have issues, you are likely experiencing issue (1) or (2).
