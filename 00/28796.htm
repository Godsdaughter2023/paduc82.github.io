gustavbrock say:

There were some copy-paste error, I can see.

This is how it should read:

    TrueDate = DateSerial(Split(Split(TextDate, " ")(0), "/")(2), Split(Split(TextDate, " ")(0), "/")(0), Split(Split(TextDate, " ")(0), "/")(1)) + CDate(Split(TextDate, " ", 2)(1))

gustavbrock say:

First, if you have the power, make them export the date/time using 24-hour ("military") time as: yyyy-mm-dd hh:nn:ss.

That won't fail.

Split splits the textdate by the first space.

First part (Split(TextDate, " ")(0) )is split in its three parts - year, month, day - and used in DateSerial to create the date.

Second part (Split(TextDate, " ", 2)(1) ) is simply converted to a time value with CDate. The last parameter, 2, limits the split to two part, or it would split the last AM or PM into a third part. We don't want that.

Finally, the date part and the time part are added to build the return value.
