If you have a DVD burner in your PC, it will have come with DVD burning software.  Read the documentation that comes with your DVD burning software.  It will explain how to create a data DVD.

You don't need to use a DVD if the purpose is to make a backup copy.  You can also use a flash memory USB device if it's large enough to hold the installation program, or you can use an external hard drive, or you can create a WIndows Live account and store the installation on your SkyDrive or you can sign up with a service like DropBox and keep it there.  Any copy that is not on your hard drive, in addition to the copy that is on your hard drive, is a backup.  My personal opinion is that you shoud have the original, a local copy, plus one on a cloud service like DropBox or SkyDrive.

doug robbins - mvp office apps & services (word) say:

Assuming that your combobox contains two columns, one for the address and one for the city, you need to obtain the data from each column to "build" the actual address

Dim straddress As String
straddress = cboOfficeAddress.List(cboOfficeAddress.ListIndex, 0) & vbCr & cboOfficeAddress(cboOfficeAddress.ListIndex, 1)

Then I would use a DocVariable field to display the address and set the value of the field by using

With ActiveDocument
    .Variables("varOfficeAddress").Value = straddress
    .PrintPreview
    .ClosePrintPreview
End With

scottgem say:

First, you need to understand that the format is immaterial. A format just DISPLAYS the data. The date/time datatype stores the data as a double precision number where the integer portion is the number of days from 12/30/1899 and the decimal portion is a fraction of a day (i.e. 6am = .25).

So if you have a text box control bound to a filed in the Form's Recordsource and set with a Short Date format, then the data that you enter will be stored as the number. If you are not using a bound control you are making life harder on yourself. 

john w. vinson say:

Scott's right. The Format is JUST FOR DISPLAY.

Since the programmers who wrote Access were doing so in America, they (perhaps unwisely) used the American mm/dd/yyyy format by default. Any literal date, a text string bounded by # - such as that constructed in your INSERT query - will be interpreted in that way, regardless of your computer's date/time setting or the screen format. You must either use a bound control (by far the simplest approach), or if you must construct a date literal in a SQL string, use either mm/dd/yyyy format or the unambiguous ISO format yyyy-mm-dd:

strSQL = "INSERT INTO yourtable (datefield) VALUES " & Format([Forms]![YourForm]![txtDate]), "\#yyyy\-mm\-dd\#)" & ";"

Just for some info, an Access Point (AP) changes wired internet protocol stuff into wireless. A Router is what sends the bits of information to the right place when you have one connection going to the internet and a lot of PCs, or devices behind it. The typical "wireless router' you buy at Best Buy from Linksys or Belkin or Netgear, actually contains all these components in one box. If it is done in software on a laptop, it could be called a Virtual AP, or Virtual Router, but it performs essentially all the functions.
