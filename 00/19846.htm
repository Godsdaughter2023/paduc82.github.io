bill manville say:

I guess when doing m pages wide, Excel has to compress each page by the same amount and keep entire columns on one page.

If I were programming this algorithm I would start by working out the compression factor needed to get the entire width onto 3 pages. I would then apply that factor, find that some pages were not entirely filled (because of entire column width considerations) and that some columns therefore would fall off the end onto a fourth page.  I would then try reducing the factor a bit and try again, until I got a value which would fit all.  I wouldn't try every possible factor (there are a large number of possible values) to see which fitted the best. 

A particularly difficult case would be where in the initial attempt a wide column would want to start just before the edge of a page.

If your column widths are reasonably static try setting an explicit compression factor and adjusting page breaks and column widths to get the best result you can, rather than relying on Excel to do the best it can.

 

Bill Manville. Excel MVP, Oxford, England. www.manville.org.uk

Function MatrixMulta(ByVal aMatrix As Range, ByVal bMatrix As Range, ByVal cl As Double, ByVal price As Double)

  End With
  MatrixMulta = Sqr(cMatrix(1, 1)) * cl * price
