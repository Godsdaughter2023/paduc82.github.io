I finally got my C6-00 call log working again! I want to document my finding here, just in case some detail could be useful to anyone else.

 

The short story: Go into the Configuration menu (green button > Options > Configuration > Log duration) and set it to "No log". Then go back to your preferred duration.

 

Behavior my phone exhibited before I got it fixed:

  • Upon entering the Call log, the screen displayed "Retrieving Data", but that was all.﻿ No data. Sometimes I could switch between tabs, sometimes not.
  • I could enter the Call log, but could not exit it. The only way to close it was to switch to another program, select Options > Show open apps and close it from there.
  • The phone showed the "someone has tried to call you" if I missed a call, but pressing it just sent me to the described situation above.
  • It seemed to me that the battery was draining a little faster﻿, but I'm not really sure.

Methods suggested here and other forums that I tried but did NOT work:

  • Turning the phone off and on again
  • Updating the firmware
  • Resetting the phone
  • Restoring the phone to factory settings
  • Turning it off and taking the battery out for a while
  • ﻿Turning it off and taking the battery and the SIM card out﻿, and let it cool for about an hour or so...

EXACTLY what I finally did:

  1. I was giving another try to the "taking﻿-the-battery-and-SIM-card-out" method, but instead of just let it that way for a while, I put on the battery and turned it on without the SIM card. The phone showed "Disconnected" status.
  2. I went to the Call log. No data, but I could navigate the tabs more freely.
  3. Changed the Log duration to "No log".
    NOTE: This step always seemed nonsense to me before, as I was trying to get my calls logged, not disabling the log (just like Lightning McQueen﻿, turning right to go left...)
  4. The phone warned me of having all my log data erased permanently. I accepted.
  5. After this step, tabs displayed the message "No data" and the app could exit normally.
  6. So, I went to the Log duration option and put it back to "30 days".
  7. I made a call, and receive a call and they were logged. Now the phone is working normally again.

I don't thing it is necessary to do it exactly as I did, but I wrote it all down, just in case what it says in "the short story" above don't work for you. BTW, sorry for the length of this post.

Sub TestGetActiveItemInMasterView()
' Enables code to do actions on the active Master or Custom Layout
' in Slide Master view.
'
    Dim var As Variant, ActiveLayout As CustomLayout, ActiveMaster As Master
       
    If CurrentViewType = ppViewSlideMaster Then
        Set var = GetActiveItemInMasterView
       
        If TypeName(var) = "Master" Then
            Set ActiveMaster = var
        ElseIf TypeName(var) = "CustomLayout" Then
            Set ActiveLayout = var
        End If
       
        If Not ActiveLayout Is Nothing Then
            Debug.Print ActiveLayout.Name
            'Do other stuff with ActiveLayout...
        End If
       
        If Not ActiveMaster Is Nothing Then
            Debug.Print ActiveMaster.Name
            'Do other stuff with ActiveMaster...
        End If
    End If
End Sub

Function GetActiveItemInMasterView() As Variant
    Dim lo As CustomLayout
    Dim des As Design
   
    For Each des In ActivePresentation.Designs
        If des.SlideMaster.Name = ActiveWindow.View.Slide.Name Then
            Set GetActiveItemInMasterView = des
            Exit Function
        End If
        For Each lo In des.SlideMaster.CustomLayouts
            If lo.Name = ActiveWindow.View.Slide.Name Then
                Set GetActiveItemInMasterView = lo
                Exit Function
            End If
        Next lo
    Next des
End Function

Public Function CurrentViewType() As PpViewType
' From http://support.microsoft.com/kb/285436
    If Application.ActiveWindow.View.Type = ppViewNormal Then
        CurrentViewType = ActiveWindow.Panes.Item(2).ViewType
    Else
        CurrentViewType = Application.ActiveWindow.View.Type
    End If
End Function

aerdian say:

csrss.exe is a legitimate Windows core system file. However, with two running, that could likely mean one of them is legitimate and the other is malware that is pretending to be that file. If Windows Defender has shut off and McAfee has disappeared, that is an almost sure-sign that you have been infected.

While it's not recommended to install an AV program onto an infected system, you essentially have no choice, considering that the malware disabled both AV programs. My recommendation would be to download Bitdefender Free (or another good program like Kaspersky or Panda, etc.). If Bitdefender is removed as well and you are unable to scan anything, you could try downloading some secondary AV programs to see if they're removed. Malwarebytes, Zemana, and HitmanPro are good options. 

If nothing works, I would recommend doing a clean install of your system. You can, however, do a full system reset. 

To do a system reset:

  • Press Windows Key and type Settings. Press Enter.
  • Select Update and Security.
  • Choose Recovery.
  • Click Get Started under Reset this PC.
  • Choose Remove everything.
  • Then, Remove files and clean the drive.
  • Choose Next, then Reset.

This will take a significant amount of time to reset your computer to factory settings. Note that ALL files will be deleted, so hopefully, you have a backup of your important files. I would not recommend completing a backup now, as your files may be infected. That said, the infection may return after the reset if any of your files are infected.

jan roelof say:

As far as I can tell from the XML of the OneNote files, each page contains definitions of all tags used in that page. The tags that are used in a page and the tags that are in your tag list are independent from each other, but you can add a tag used in a page to the tag list by right-clicking the tag icon and then click "Add To My Tags". The tag list is not automatically updated to contain all tags used in a notebook or anything like that - unfortunately. This might be a good idea for a PowerToy, though .. mmmm.

Jan Roelof

In short, the tag list is a list of (custom) tags that is saved in preferences.dat. The list is for your convenience only. There is no relation between the tags in the list and the tags actually used.
