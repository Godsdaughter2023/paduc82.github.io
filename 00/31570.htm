cmdrkeene say:

I'm certainly at a loss for how this situation could arise, but I do know that Fast Startup is the same as hibernation.  When Fast Startup is enabled, choosing "Shut down" does not actually shutdown the system. Instead, it logs-off the user account, then hibernates (the restart and sleep options are not affected by this).

As a side note: one very interesting artifact about this is that, if you're ever experiencing a problem, a shutdown won't necessarily help. Since you aren't actually shutting down (just hibernating), whatever flaw/memory stack corruption would still be present when you booted back up and restored the hibernated state.  This is very non-intuitive for most users, since typically "turn it off and back on" is a good troubleshooting tactic. But starting with this new feature, a "restart" is actually a full restart, whereas a "shutdown" is, essentially, a lie.

But I digress.

Anyway, I think my best option for you would be to clear out this state and try it fresh.  Please leave Fast Startup disabled/off for now, while you run these steps:

  1. Close all open programs, because this will restart your computer twice.


  2. Open Command Prompt in administrator mode.
    To do this, find Command Prompt, right-click it, and choose "Run As Administrator". You'll know it's correct if the title bar caption of the command prompt has the word "Administrator" at the beginning.


  3. Type the following commands, pressing Enter after each one:

    shutdown /r


  4. At this point, your computer will restart and the hiberfile.sys (hibernation file) will be cleared and deleted.


  5. Open an administrator command prompt one more time, and this time type:

    shutdown /r


  6. At this point, the computer will restart and hibernation should now be available.

Now all that's left is to turn Fast Startup back on, and test it out.

For Drivers check System Maker as fallbacks and Device Maker's which are the most current.

of your video adapter - double click - Driver's tab - write down the version info. Now click UPdate
Driver (this may not do anything as MS is far behind certifying drivers) - then Right Click -
Uninstall - REBOOT this will refresh the driver stack.

with their own software and drivers and any other major device drivers you have.

each installation.

shows up. This is because some drivers rollback before the latest is installed (sound drivers

needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER - check
manually by model.
