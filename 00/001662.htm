john korchok say:

Office 2016 for Mac's support for third-party PostScript-flavor OpenType fonts has been shaky from the beginning. It's possible you are running one of those type bugs that displays the wrong variant of a font. Please choose Word>About Word and check the version number. You should be on at least 15.29, if it's less than that, please choose Help>Check for Updates to make your software current.

Back in the day, most Mac fonts were single PostScript fonts. When you're using single fonts, you must change from Roman to Bold by changing the font, not by clicking the Bold button. With single fonts, every variant shows in the font menu and it's simple to assign a keyboard shortcut to the variant you want.

Today, most fonts come in OpenType families. With a family font, you specify the base font with the font dropdown, then get to the variants by clicking on bold and or italic buttons, or by choosing a variant from the font submenu for the chosen family. These are not so easy to assign a keyboard shortcut to, because the Fonts list in Customize Keyboard lists only the base font name.

The way around this is to create a Style. This could be a paragraph-based style from applying to an entire paragraph. Or it could be a character style, to apply to only a word or two in a paragraph. In the style, you can specify both the base font and the variant (bold, italic, bold italic, etc.). Finally, you can apply a keyboard shortcut to the style so that you can have the workflow you miss.

Styles have significant advantages or local formatting (your old technique of selecting text and applying a font). One example is if you decide to change all your bolding from Myriad to Frutiger. With styles, you just edit the style and all instances are automatically updated in your document.

a. user say:

The filters will save, but if your process means that the user needs to attach a different data source, the filters will actually be lost at that point. Ways to deal with that include:

 a. before opening the mail merge main document, copy the data source file onto the one that the mail merge main document expects

 b. create a small VBA routine that re-imposes the sort/filter conditions and run that each time a new data source is attached. That has some chance of working if the data source is always the same type of file, e.g. always an Excel worksheet. If it could vary between being a Word document and an Excel worksheet, some things may not work so well.

For (a), what I mean is that if the mail merge main document is attached to datasourceX.xlsx, and you want to use datasourceY.xlsx, copy datasourceY.xlsx to datasourceX.xlsx before opening the mail merge main document.

To use (b), the complication is usually not in the VBA itself but in how you make it easy to use. Further, I do not know precisely what will and will not work on Mac. So this is just an outline.

Word takes your filter/sort settings and turns them into an SQL SELECT statement (even for "non-database" sources such as Word documents). At present with Mac Word, that is all you really need to know because you cannot select individual records as you can on Windows Word.

So for example, once I have attached my data source and applied the filter settings, I can discover the SQL query using this VBA (e.g. executed in the VBA Editor's Immediate window"

?activedocument.mailmerge.datasource.querystring

In my test example, this returns

SELECT * FROM /Volumes/pjj1/axfer/2017121501 skipif multitest/mmds1.xlsx WHERE ((Postal_Mail_OptOut = 'False') AND (Mailing_Address_Line_1 IS NOT NULL ))

As far as I can tell, the current version of Word does not really care what you are selecting "from", so it seems that you could use exactly that SQL statement to set up the filters even if you used a data source called /Volumes/other/mmds2.xlsx.

so you could for example use a VBA routine such as this to save the initial query string into a document Variable (which is an object stored invisibly inside your document

Sub saveMailMergeFilters()

With ActiveDocument

  .Variables("theQueryString").Value = .MailMerge.DataSource.QueryString

End With

End Sub

After the user attaches a new data source, they would then need to run something like this

Sub setMailMergeFilters()

With ActiveDocument

  .MailMerge.DataSource.QueryString = .Variables("theQueryString").Value

End With

End Sub


macropod say:

You could just use two SKIPIF tests, such as:

{SKIPIF{MERGEFIELD Postal_Mail_OptOut}= "True"}{SKIPIF{MERGEFIELD Mailing_Address_Line_1}= ""}

or:

{SKIPIF«Postal_Mail_OptOut»= "True"}{SKIPIF«Mailing_Address_Line_1»= ""}

Otherwise, try:

{SKIPIF{={IF{MERGEFIELD Postal_Mail_OptOut}= "True" 1 0}+{IF{MERGEFIELD Mailing_Address_Line_1}= "" 1 0}}> 0}

or:

{SKIPIF{={IF«Postal_Mail_OptOut»= "True" 1 0}+{IF«Mailing_Address_Line_1»= "" 1 0}}> 0}

Note: The field brace pairs (i.e. '{ }') for the above examples are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. Likewise, the chevrons (i.e. '« »') are part of the actual mergefields - which you can insert from the 'Insert Merge Field' dropdown (i.e. you can't type or copy & paste them from this message, either). The spaces represented in the field constructions are all required.

flaviub say:

Phone-Car syncronisation

How do I syncronize my Nokia 6120 Classic with my car's phone. I have to establish a connection from the phone, but how do I do that?if anyone knows please reply.

cjlim say:

The procedure depends on the car you have but is usually as follows:

 

1. Activate bluetooth on the phone

2. Activate bluetooth on the car and set to pairing mode

3. Car should find phone and ask you to set a pin

4. A message should appear on the phone requesting a pin

5. Enter the pin and Bob's your uncle

 

If the phone is fully compatible with the car, the contacts on the phone will download to the car as well. If this is the case, do not use the phone for a few minutes (depending on how many contacts you have) for the phone to download contacts.

 

The above may not work first time. If it doesn't, unpair the both car and phone and repeat the above again.
