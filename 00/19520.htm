brian tillman [outlook mvp 2007-2019] say:

It is your mail provider's problem.  The message comes from whatever SMTP server you have configured in the account.  It is not CenturyTel's problem and it's not Outlook's problem.  The only address Outlook cares about is the address of the SMTP server.  If it can reach that server, that's all it needs.  It sent the message to the server, which tried to look up the domain and failed.

Open a command prompt window and enter

C:\> nslookup centurytel.net

If you get an IP address back, then the problem can't be with your DNS server, it must be with the mail provider's server.  4xx errors are usually transitory in nature, so it might clear up on its own.

The file name for the backup is the same as the original; it does not use the .wbk filename extension.
