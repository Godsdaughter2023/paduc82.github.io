Sub Consolidate_from_Excel_files()
Application.ScreenUpdating = False
Dim sep As Integer
Dim file_name As String
Dim rng_copy As Range
Dim count_files As Integer
With Worksheets("Query and file listing")
count_files = Application.WorksheetFunction.CountA(.Range("A7:A157")) - Application.WorksheetFunction.CountBlank(.Range("A7:A157"))
End With
For n = 1 To count_files
With Worksheets("Query and file listing")
    Workbooks.Open Filename:=.Range("A1") & .Range("A" & 6 + n)

Set rng_copy = Worksheets("QPI Tracking").Range("A1").CurrentRegion
ThisWorkbook.Activate
Worksheets.Add After:=Worksheets(Worksheets.Count)

ActiveSheet.Name = file_name

Workbooks(Worksheets("Query and file listing").Range("A" & 6 + n).Value).Close
Next n
Application.Goto reference:=Worksheets("Query and file listing").Range("A2")
Application.ScreenUpdating = True

graham mayor say:
I think the following will do what you require. It will prompt for the folder containing the workbooks, then if any workbook in that folder contains a worksheet called 'Tracking', that worksheet will be appended to a new workbook and named with the filename of the workbook (without extension) it was drawn from. The workbook will contain all the tracking worksheets in the order they were accessed. If you want only part of the filename then you will have to further modify strfilename.

Sub MergeTrackingData()
Dim xlBook As Workbook
Dim xlNewBook As Workbook
Dim xlSheet As Worksheet
Dim xlNewSheet As Worksheet
Dim strfilename As String
Dim strPath As String
Dim strSheetname As String
Dim fDialog As FileDialog
    Set xlNewBook = Workbooks.Add(-4167)
    Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)
    With fDialog
        .Title = "Select folder and click OK"
        .AllowMultiSelect = False
        .InitialView = msoFileDialogViewList
        If .Show <> -1 Then
            MsgBox "Cancelled By User", , _
                   "List Folder Contents"
            Exit Sub
        End If
        strPath = fDialog.SelectedItems.Item(1) & "\"
    End With
    strfilename = Dir$(strPath & "*.xls?")
    While Len(strfilename) <> 0
        Set xlBook = Workbooks.Open(strPath & strfilename)
        For Each xlSheet In xlBook.Worksheets
            If UCase(xlSheet.Name) = "TRACKING" Then
                xlSheet.Copy After:=xlNewBook.Sheets(xlNewBook.Sheets.Count)
                Set xlNewSheet = xlNewBook.Sheets(xlNewBook.Sheets.Count)
                strSheetname = Left(strfilename, InStrRev(strfilename, ".xl") - 1)
                xlNewSheet.Name = strSheetname
                Exit For
            End If
        Next xlSheet
        xlBook.Close 0
        strfilename = Dir$()
    Wend
    Application.DisplayAlerts = False
    xlNewBook.Sheets(1).Delete
    Application.DisplayAlerts = True
    Set xlSheet = Nothing
    Set xlNewSheet = Nothing
    Set xlBook = Nothing
    Set xlNewBook = Nothing
End Sub

I am trying to make Excel behave differently too and made some tests.


I think that the "use office 2016 to sync files that I open" option in onedrive for business 2016 forces Excel to connect to the server directly each time it's online because Office 2016 is able to merge changes in files when concurrently editing them using the onedrive sharing options. What happens when you save the file when offline, is that MSO upload center just caches the file but if it senses a conflicting change when back online, it will prompt you either to open it in Excel anyway to merge it or to discard your changes. It would be nice that Excel always used MSO UC and reverted to direct upload only when MSO UC senses a conflict, but it doesn't sound like this feature it has been implemented

unselecting the sync option makes Office + Onedrive for Business fly like a bullet but you lose the concurrent editing ability

the fact that you still see the modal dialog may be due to the fact that your excel is still opening the server version. That's what happened to me. Once unselected the above mentioned checkbox, I then navigated to the local filesystem and opened the local version, and Excel stopped trying to connect directly to the server

I hope this works for you, it's my own experience not the result of extensive research....
