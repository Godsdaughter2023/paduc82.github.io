Issue Description
================
Customer cannot sync his files using OneDrive for Business 2016 with the “Credential Needed” pop-up.

Root cause
================
Msosync is attaching a FedAuth cookie along with X-IDCRL_ACCEPTED header but not the SPOIDCRL cookie and that is causing request to fail since SPO does not look at the FedAuth cookie if X-IDCRL_ACCEPTED header is present - since that indicates that the request is coming from the Office clients.

Solution
================
Product Group has already fix the problem and the fix will be released in the version: 16.0.6413.1000. If the problem has not been resolved after the new release 16.0.6413.1000, please contact back to the escalated engineers and we’ll keep working on the issue.
