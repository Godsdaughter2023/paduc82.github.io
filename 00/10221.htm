If you are on Lion

Word 2004 won't run because Apple has suppressed the Rosetta application required by Intel Macs to emulate the old line of Power PC CPUs. Rosetta was first introduced in 2006, when Apple changed to Intel CPUs.

You have two courses of action:

  • Rosetta is still supported in Snow Leopard, so you could install this version of the OS to run Word 2004. See How to downgrade from Apple OS X Lion to Snow Leopard.
  • You can partition your disk (basically, convert it into two separate units) and install Snow Leopard in one, leave Lion in the other. This will let you do a dual boot. See Using Partitions to Downgrade from Apple OS X Lion to Snow Leopard.

I wouldn't do any of the above unless I had a must-have application that was never released for Intel CPUs.

What I would do is purchase a three-license home version of Office 2011 for $150, so that's $50 per user. Not expensive, really. You will find Word 2011 to be 2 to 3 times faster in everyday use than Word 2004, and if you use macros, they run at least 10 times faster.

filter with like - Microsoft Community

tobemaster say:

filter with like

Hi,

I want to use 'Filter' to search by 'First Name' in my tblEmployees.

 

Me.Filter = " strFirstName= Like ' " &txtFilterFirstName& " * ' "

 

I want to add 'Like ' to use wildcard.

 

Do you think this is right code?

dirk goldgar say:

I want to use 'Filter' to search by 'First Name' in my tblEmployees.

 

Me.Filter = " strFirstName= Like ' " &txtFilterFirstName& " * ' "

 

I want to add 'Like ' to use wildcard.

 

Do you think this is right code?


It seems to me that it's almost right, except that the equals operator (=) and the Like operator are mutally exclusive, and you seem to have extra spaces in the filter string.  In some places an extra space doesn't matter, but in other places it does.  I would write this:

 

    Me.Filter = "strFirstName Like '" & txtFilterFirstName & "*'"

 

That assumes that "strFirstName" is actually the name of the First Name field in your table.  Note also that this code will fail if txtFilterFirstName contains a single-quote or apostrophe (').  Maybe it would be better to use double-quotes around the specified name fragment, but to do that, you have to double up the double-quotes, like this:

 

    Me.Filter = "strFirstName Like """ & txtFilterFirstName & "*"""

Hi,

 

With IE10 on Windows 7, the IE shortcut on the Start Menu should be the 64-bit version so that you will, normally, get a 64-bit manager process and 32-bit tabs. (See links below)

 

You can check the target of your IE shortcut by... Right-click the shortcut> Properties> Shortcut tab> Target.

 

Your iexplore.exe files are located at:

C:\Program Files (x86)\Internet Explorer\iexplore.exe . . . (32-bit)
and

C:\Program Files\Internet Explorer\iexplore.exe . . . (64-bit)

 

You can create a shortcut to either of those for the desktop or Start Menu.

 

IE> Help> About doesn't provide bitness like it used to.

To see whether 32-bit or 64-bit IE10 has opened, open Task Manager (Ctrl+Alt+Delete> Task Manager) and look at the Processes tab.

 

64-bit IE10 will appear as iexplore.exe.

32-bit IE10 will appear as iexplore.exe *32.

 

You may see both processes  with only 1 window and 1 tab open- the 64-bit Manager Process and a 32-bit tab.

 

To get the 64-bit Manager Process to open automatically, go to Internet Options> Advanced and check "Enable Enhanced Protected Mode." I've also seen reports that 64-bit IE10 cannot be accessed/run (using any method) if Enable Enhanced Protected Mode is not checked.

 

Also, in Internet Options> Security, verify that Protected Mode is enabled.

 

Also, from Control Panel> Action Center, Change User Account Control Settings, verify that the slider is not set to "Never notify." If User Account Control is set to "Never notify" Protected Mode and Enhanced Protected Mode can't be enabled.
