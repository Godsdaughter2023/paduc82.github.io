jay freedman say:
There is no way for a document to save the setting that controls whether highlighting appears in the editable regions.

A macro can turn off the shading. The key to avoiding security alerts is to put the macro in the template attached to the document, not in the document itself, and to store the template in a trusted location (which includes the default User Templates folder). With that caveat, store the following macros in the letterhead's template:

Sub AutoNew()
   ActiveWindow.View.ShadeEditableRanges = False
End Sub

Sub AutoOpen()
   ActiveWindow.View.ShadeEditableRanges = False
End Sub

The appropriate macro will run automatically when a new document is created from the template or when an existing document is opened for further editing.

The main drawback to this arrangement is that if the template isn't accessible when the document is opened, as when the document is sent to another organization, the macros won't be there and the shading will be visible. If the recipients don't need to edit the document, the best solution (even without the shading issue) is to create a PDF from the document and send that instead.

There's another, more complex macro solution at http://www.word.mvps.org/FAQs/Customization/ProtectWord2000PlusHeader.htm, but it has the same drawback.
