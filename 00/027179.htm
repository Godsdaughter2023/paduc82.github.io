Hi Evaldas,

If you use PowerShell to set the AllowConflits to true, when a user sends a meeting request for the room at a time when it is already booked, it advises in the mailtip area that the meeting conflicts with another in place, but the booking will still take place at the same time.

To set the value to be true, the full command should be:

Set-MailboxCalendarSettings -Identity "Pxxxxxxxxs"(the identity shows on the image you provide) -AllowConflicts $True

And the AllRequestOutOfPolicy parameter is to specify whether to allow all users to submit out-of-policy requests. Out-of-policy requests are subject to approval by a resource mailbox delegate. You can set it to 0/False to automatically send a declined email to the meeting requestor for all “Out of Policy” Meeting requests, which includes conflicts.

An article for your reference: Set-MailboxCalendarSettings

Just a footnote to what you've already discovered: There's no way for Word 2003 to do a Find/Replace on a .dtox or .dotm template.

Even with the Compatibility Pack applied, Word 2003 won't open .dotx or .dotm templates. I'm not sure why that MS made restriction, but it was a deliberate design decision. If you try to use File > Open in 2003 to open a .dotx or .dotm, you get the File Conversion dialog that asks you to choose an encoding; but of course there are none that work because all it sees are the compressed bytes of the zip file.

I'm not sure what Word 2003 VBA would do if you used Documents.Open on a .dotm, did some kind of edit on whatever appeared, and then did a save -- but I'm pretty sure the result wouldn't be recognizable by WinZip or Word.
