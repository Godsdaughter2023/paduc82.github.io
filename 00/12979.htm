joeu2004 say:
I need to work with data that includes dates in the mm/dd/yyyy format. When I try to convert the dates to serial number using the n function, I get a value of "0" no matter what I do. If I manually change the data to the dd/mm/yyyy format, the n function works fine.
 
Anyone know an easy way to fix this?

Be sure that the "dates" in the form mm/dd/yyyy are not actually text.  N(text) always return zero.

I suspect that N() works with the manually entered dates because your system date form is set to dd/mm/yyyy.  I am not familiar with Win7, but in WinXP, you find the system date form in the Control Panel under Regional and Language Options.

Note that in that case, dates entered in the form of mm/dd/yyyy will be interpreted as dd/mm/yyyy.  So if you entered 12/1/2010 intended to be interpreted as 1 Dec 2010, the N() function will work, but the date serial number will correspond to 12 Jan 2010.

In contrast, if you entereed 12/30/2010 intented to be interpreted as 30 Dec 2010, the N() function returns zero because the data is treated as text since Excel cannot interpret that as the 12th day of the 30th month.

joeu2004 say:

PS....

If my presumption is correct about your system date form, you cannot really "work with that includes dates in the mm/dd/yyyy format", at least not without some care.

The point is, as I alluded to previously:  some data of the form aa/bb/dddd will be treated as a date in the form dd/mm/yyyy, and some data will be treated as text because it cannot be interpreted as a date in the form dd/mm/yyyy.

Consequently, you cannot simply use string functions like LEFT(A1,2), MID(A1,4,2) and RIGHT(A1,4) to parse the intended parts of the "date".

That works when aa/bb/dddd is treated as text, but it does not work when aa/bb/dddd is treated coincidentally as a date of the form dd/mm/yyyy.

Of course, the simplest solution is to change your system date form.  But that might not be a viable approach for you.

An alternative might be:  if you import the data, set the "date" column to type Text.  But you do not say how you acquire the data that you "work with"

Other alternatives may be needed and are possible.  But you do need to explain how you acquire the data in the first place.

If you right-click the circle you have drawn and choose Add Text, an integral text box will be added to the shape (that is, the text box cannot be manipulated separately from the shape). It will still be square, however.
