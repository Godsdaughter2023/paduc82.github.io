rohn007 say:

15 MB should not be "too large".

Do you have 32 or 64 bit Windows?

Do you have 32 or 64 bit Office?

How much RAM do you have installed?

How much RAM does excel use when you open the file.

Even if you have 64 bit Windows, Office installs 32 bit by default.  32 bit Office applications are limited to using only 2GB of RAM. Obviously the file size itself is no where near 2GB, but your formulas may be written in a way that use up RAM approaching that limit.

First thing you can try is starting Excel in Safe Mode. This will free up a little RAM by not loading any addins.  Then close the file immediately.  If that works, make 2 additional copies of the file (so you have 3 in total).  Keep one as untouced "original".  Open one and delete roughly the first half of the data. Close and Save.  Open the other and delete the second half of the data.  At least this way you can work with the data.   Or is there some other way you can reduce the amount of data rows involved in calculations.  Maybe there is some "old" data you can archive and delete/

Or, assuming you have a relatively new computer with 64 bit Windows and 32 bit Office, you could try installing 64 bit Office. This will make some more RAM available to the application, hopefully avoiding the crash problem.  Or you could find a friend with 64 bit Office installed to see if it will help.

Either way you should look into "optimizing" your calculations.  Excel often allows you more than one way of achieving a result.  Some are more efficient than others, but this difference does not show up until you have spreadsheets with "complex" computations and lots of data.  By tweaking your calculations you may be able to resume using the data as it is.
