andreas killer say:
If you add something to a range in Word the range grows automatically. If you add a paragraph from a range, the returned paragraph includes the range. You can prevent this if you collapse the range to it's end.

I don't know what's the equivalent for VB's "Chr(13)" in C#, change this in the code below to a carriage return.

Andreas.

foreach (Section sec in aDoc.Sections)
{
  Microsoft.Office.Interop.Word.Range rng = sec.Footers[Microsoft.Office.Interop.Word.WdHeaderFooterIndex.wdHeaderFooterPrimary].Range;
  rng.Collapse 0
  rng.InsertAfter Chr(13)
  rng.InsertAfter "My Text";
  rng.Font.Name = "Arial";
  rng.Font.Size = 8;
  rng.Font.Hidden = 1;
}

Looks like results could be unpredictable while working with rng.Text. I have a table in the footer, tried to delete an extra Chr(13) and that converted my table into a text.

andreas killer say:

I don't know what you want to accomplish, but this should answer your questions.

Andreas.


  // Is this range within a table?
  If rng.Information(Microsoft.Office.Interop.Word.WdInformation.wdWithInTable)
  {
    // get the table
    Microsoft.Office.Interop.Word.Table ThisTable = rng.Tables(1)
    // insert a row before last row
    Microsoft.Office.Interop.Word.Row NewRow = ThisTable.Rows.Add(ThisTable.Rows(ThisTable.Rows.Count))
    // write a text in the first cell of this row
    rng = NewRow.Cells(1).Range
    rng.Text = "MyText"
  }
  Else
  {
    // More then one paragraph?
    if rng.Paragraphs.Count > 1
    {
      // get the next to last paragraph
      rng = rng.Paragraphs.Last.Previous.Range
    }
    // collapse the range to the endpoint
    rng.Collapse Microsoft.Office.Interop.Word.WdCollapseDirection.wdCollapseEnd
    // insert a line afterwards
    rng.InsertAfter "My Text" + Chr(13)
  }

kim s. say:

Need to be able to make slide "#1" start on the fourth slide.

When I insert slide #'s, it starts on the first slide.  I need for slide "#1" to begin on the fourth slide which follows the table of contents. I also need for each slide to correspond correctly with my table of contents. 

I can manually enter page #'s in text box's at the bottom of each slide, but as I add new slides or change the order, I have to manually change each slide #....time consuming.  There has tobe an easier way.  Thanks!

chirag. say:

One of the ways to get this done is as follows:

  1. Place your actual first 3 slides at the end of the presentation and start your presentation from your 4th slide. PowerPoint would number your slides from 1 and that would be the correct number for your 4th slide.
  2. Remove the slide numbers from the last 3 slides (which are your actual first 3 slides).
  3. Create a Custom Show such that your last 3 slides are placed as the first 3 slides in the Custom Show and the remaining slides follow those 3 slides. To create a Custom Show, useSlide Show | Custom Shows... menu item.
  4. Next, tell PowerPoint to display the custom show when you start the slide show. To do this, selectSlide Show | Set Up Show... menu item, select Custom show in the Show slides section and select your custom show in the list, click OK button.

Now when you start the slide show, PowerPoint will show your actual first 3 slides without slide numbers and slide #1 from your actual slide 4 onwards.

david a matson say:

Greetings Savvas Sopiadis!  Welcome to the Windows Phone forums!

Unfortunately, there are a great many reasons why these issues exist, and most of them are actually on purpose.

To the best of my knowledge, in order for most devices to be visible to other devices, BOTH devices have to have:

  1. Some kind of NetBIOS agent running to process external requests (pings).
  2. Some kind of policy that stipulates whether or not pings are acknowledged or not.
  3. An open port on each device's firewall for TCP/IP services to properly access the device.

    Additionally, your Router can have settings that allow the user to setup the Router for home/residence where sharing is allowed and endorsed, or as a Cafe Hotspot where each connection is compartmentalized and sharing is actively prohibited.  Depending on how you have your network setup, this could certainly impact your abilities to share or access your laptop.

    If you don't have your laptop set to "share" its resources, that would be a good first point to investigate further.

Good luck!
