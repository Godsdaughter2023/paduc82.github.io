charles kenyon say:
You say you are new to Word 2010. Have you used previous versions of Word? The switch can take a bit of an adjustment. This option used to be right on the print dialog. It is now hidden in a drop-down menu.

With the print dialog displayed in the backstage, click on the down triangle next to your printer name. At the bottom is the option to print to file. This should be unchecked.

If all else fails, the following macro will print your document to your default printer.

Sub PrintCurrentDoc()
'
' PrintCurrentDoc Macro
' Macro recorded 11/30/2011 by Charles K. Kenyon
'
    Application.PrintOut FileName:="", Range:=wdPrintAllDocument, Item:= _
        wdPrintDocumentContent, _
        Copies:=1, _
        Pages:="", _
        PageType:=wdPrintAllPages, _
        ManualDuplexPrint:=False, _
        Collate:=True, _
        Background:=True, _
        PrintToFile:=False, _
        PrintZoomColumn:=0, _
        PrintZoomRow:=0, _
        PrintZoomPaperWidth:=0, _
        PrintZoomPaperHeight:=0, _
        OutputFileName:="", _
        Append:=False
End Sub

swischner say:

As far as I know there is no way to map size of the contents of a notebook in detail. You can get the size of each notebook itself via file info on web frontend of OneDrive. Although you just see links to the notebooks there the actual size of the (whole) notebook content is displayed.

There is a somewhat clumsy way to validate the size of single sections though if you are using OneNote desktop version (OneNote 2016 free would do):

Export the whole notebook into a ONEPKG file. It doesnt matter if the notebook is stored on OneDrive.

As that is nothing else but a ZIP fie with a different file extension you can peek inside it with an unzipper (rename the .onepkg to .zip first if needed)

You'll discover a .one file for every section in the notebook. The sizes may give you a clue where the biggest content is sitting.

You may keep the ONEPKG files as backup (you would need an office version of OneNote to restore notes from that!) or delete it.

Unfortunately I know of no way to determine the content sizes on page level. 

If that all seems too complicated I can give you the alternative too of course: Buy additional OneDrive space or subscribe to Office 365 :)
