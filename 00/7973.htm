cevapcici say:

MS Project 2013 - record a task as late

Many thanks for your assistance.

trevor rabey say:

Baseline is definitely recommended, but is not essential, or not enough. When I update progress status of tasks in a plan, I always save a copy of the file first, so that I can refer back to that to see what the plan was before the update. I end up with a lot of incremental versions of my plan. In the construction business you need to keep this kind of progressive history for the inevitable dispute.

Baseline is needed in order to calculate variances and, if you must, EV.

You mention "expected date" and "scheduled date". The scheduled date is not necessarily the "expected date", and a task is not necessarily "late" just because it does not start and finish as scheduled. The plan is not a bus timetable. Most actual start and actual finish dates will be different from the scheduled dates, usually later, rarely earlier. The scheduled dates are just the earliest possible dates. Every task is scheduled asap, and it would be nice if they all started and finished asap, but they don't.

A task can start and finish anywhere within its total slack without delaying the finish date of the project.

Whether an actual start and actual finish which are later than the scheduled asap dates is a problem, whether it is something serious to be concerned about, depends on the effect of those actuals on the rest of the plan.

You need the schedule table to see the earliest dates, latest dates and the free and total slack.

Do not simply set % complete to 100% because that forces MSP to assume that the task started and finish as scheduled, because it has nothing else to go on.

As you have seen, this results in incorrect actuals. If they are in the past, relative to the status date, then this means you are just reporting incorrect actuals but they are plausible, because they are at least in the past. It is worse if the scheduled dates are in the future, because then you would be reporting progress made in the future, eg "we laid the bricks next week".

Do not ever type in % complete at all. This is one of the most common mistakes made in using MSP.

Use the tracking table to input the actuals, and if necessary re-estimate the remaining duration if the task is not yet actually finished.

Log into windowsphone.com and use the Find My Phone feature to set a new lock code
  1. Go to http://www.windowsphone.com and sign to the Microsoft account associated with your phone.
  2. Select "Find My Phone" from  the drop down menu that appears when you click on your name.  Or just go to https://www.windowsphone.com/en-us/my/find
  3. Click the "Lock" link
  4. Enter a new 4 digit PIN code and press the Lock button.
