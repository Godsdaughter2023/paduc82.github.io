rohn007 say:

Normally that error is associated with files in shared folders on the company network.  But it can also display when you try to open a file after Word crashed with that file open.

First, go to File menu > Options command > General option.

Look to see what text/name is entered in the "User Name:" field. It should be your name, but could be a generic text too. 

If your name is in the "user name" field, but is not displaying in the messages, then it is clear that the "lock" files are not the cause. And since the problem is happening with all (?) .DOC files the lock files are unlikely to be the cause.

Go back to the Trust Center (where you set Trusted Folder).

Click on the "File Block Settings". 

Make note of which "Open Behavior ..." is selected. I suspect you will find you have the first option selected.  Pick one of the other 2.

Then look in the list of file types to find "2003 binary documents and templates" (MS's wordy way of saying 2003 DOC and DOT files!).

If there is a check in the "open" column, you could remove it to stop the message.

    sean in mi say:

    Pivot Chart needs to change size automatically

        The volume of data in my chart can change dramatically. I can drag/drop for large volumes, but it looks terrible when the volumn is reduced. Does anyone have a code that will change the size of the entire chart to accommodate any volume (that has been selected by pivot options).

        So it will grow from a small chart to a very large chart as data options change. Note: I did move this chart to it's own worksheet called "Chart1".

teylyn say:

Hello,

Jon Peltier has a good introduction to resizing charts with VBA here.

You can take this as a starting point. As a general suggestion for an approach: take a look at your chart and determine how wide/high it needs to be if it shows 5 or 10 or 50 or 100 etc. data points.  Divide the width of the chart by the number of data points. That will be your sizing factor. When the pivot changes, count the number of data points (or rows in the pivot table) and multiply by the sizing factor. That number can then be used by VBA to size the chart.
