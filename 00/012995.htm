ken sheridan say: Your expression is not a completely accurate means of returning a person's age in years at a given date.  To return an accurate value add the following function to a standard module in the database:

Public Function GetAge(varDoB As Variant, Optional varAgeAt As Variant) As Variant

    If IsMissing(varAgeAt) Then varAgeAt = VBA.Date
    
    GetAge = DateDiff("yyyy", varDoB, varAgeAt) - _
        IIf(Format(varAgeAt, "mmdd") < Format(varDoB, "mmdd"), 1, 0)

End Function

Change the Name property of the text box bound to the DoB column to txtDoB to differentiate it from the name of the column and remove any ambiguity.  Then call the function in the text box's ControlSource property with:

    =GetAge([txtDoB], #2017-08-31#)

julie fka profjulie say: This is a limitation with the calendar application on WM6.1 standard devices.  I don't know if it has been addressed in WM6.5.

Even though the Calendar application on a WM6.1 standard device, such as the Samsung Jack, does not display categories and there is no way to include a category when inputting a new appointment into the Calendar application on the WM6.1 standard device, the category field is included in the database (you just don't see it or have access to it through the Calendar application). 

However, if you use a third party PIM application (I use Agenda One on my WM6.1 standard phone), you'll see that categories are included with appointments and you can use categories when inputting new appointments directly into the device, you can filter by category, and do just about anything else you are accustomed to doing with categories on appointments.

You first need to add the required ShapeData to the shapes and then create a custom stencil with all those shapes (http://office.microsoft.com/en-in/visio-help/create-save-and-share-custom-stencils-HA102749233.aspx).
