afwings say: Bob, when you copy anything from MathType, it puts several formats of data onto the Windows clipboard. One of those formats is always MathML, no matter which of the settings you've chosen. In fact, it's easier than that because MathType's default format -- "Equation object" -- is what you'd be using if you're using MathType with Word or PowerPoint.

So, to answer your questions...

As to your first question, it doesn't matter which of the formats you choose from those you listed above, but my recommendation is to just leave it set at whatever format you're using in something else. If you're working with TeX, leave it set there. If you're creating a web page with MathML for the math, leave it set there. If you're working with Word, leave it set at the default setting. OneNote doesn't much care, since it will always pull the MathML off of the clipboard anyway.

So that also answers your second question -- MathML is the format that OneNote uses -- regardless of whether you have Microsoft Mathematics installed or not. Note this assumes OneNote 2010, which is what I assume you're using since this is an Office 2010 forum.

This is consistent with our article on the subject.

Excel Column1 Formula - Microsoft Community

kothaimanickam say:

Excel Column1 Formula

Hi! When you are using [Column1] in part of your formula, how do you keep [Column1] as a fixed reference point?

When I drag the formula from cell B2 to B3 the [Column1] becomes [Column2].

bernard liengme say:

Suppose in B2 I have =A2*B1

If I drag this to B3 I get =B2*C1

Change the formula to =$A$2*B1 and drag right, you get =$A$2*C1 because the $ (nothing to do with money!!) hold both the A and the 1 constant when we drag to the right

But if we drag down we get =$A$2*B2

Change the formula to =$A2*B1 and drag right, you still get =$A2*C1 but if we drag down we get =$A3*B2

For more on this do an internet search using Excel relative references as search term.

You can type the $ or use the F4 key. Type =A4 into a cell, select the cell and repeatedly press F4; you get $A$4, A$3, $A3 and then A3.

ossiemac say:

Try the code below.

I have written the code so that it will process correctly irrespective of which is the ActiveSheet. In fact, neither sheet needs to be active for the code to run.

Note that when using the With / End With that the Range and Cells have a leading dot that ties them to the worksheet in the With statement. If the With statement is not used then it is necessary to repeat the worksheet name at each place where you see the leading dot on the range and cells.

Setting range variables to the range to be copied and the destination allows the Copy and Paste commands to remain together. there are circumstances if these are separated with other code then Excel loses the Copy and therefore there is nothing to paste. Normally only occurs if there is some complex codeing between the Copy and Paste but it is a good idea to get used to using the range variables as I have done.

Sub test()
    Dim rngToCopy As Range
    Dim rngDestin As Range
    Dim index As Integer    'Var linhas do ciclo for (1-365)
    Dim num As Integer
    num = 1
   
    'Will run much faster if screenupdating is turned off while processing
    Application.ScreenUpdating = False
   
    For index = 1 To 365
        With Worksheets("Folha2")
            Set rngToCopy = .Range(.Cells(index, 1), .Cells(index, 24))
        End With
       
        With Worksheets("Folha1")
            Set rngDestin = .Cells(num, 1)  'Don't require Range when only one cell
        End With
       
        rngToCopy.Copy
        rngDestin.PasteSpecial Paste:=xlPasteAll, Transpose:=True
       
        Application.CutCopyMode = False
        num = num + 24
   
    Next index
    Application.ScreenUpdating = True
End Sub
