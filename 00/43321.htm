Windows 10 1709 Fall Creators Update failing due to Paragon HFS+ - Dell Vostro 1700

Just thought I would post this in case anyone else hits the issue.

My trusty old Dell Vostro 1700 failed to install the Fall Creators update. The system would keep reverting to the previous version. The error occurred at 2nd reboot with the following errors:-

0xC1900101 - 0x40017

The installation failed in the SECOND_BOOT phase with an error during boot operation.

The failing module was listed as CLASSPNP.SYS

In the registry, I noted there is a section in HKLM\System\Setup that lists the reason for reverting that refers to CLASSPNP.SYS and the other associated modules which in this case were DISK related. I noted that one of these was apmwin.sys which is for Apple formatted USB drives and I remembered that recently I had installed Paragon's HFS+ Apple file system add-on. I uninstalled this and re-applied the update and it went through OK.

Dell have produced a list of their devices that they have tested with FCU and this is too old to be on it but I can confirm that aside from this self inflicted issue, it does work OK on it. I assume therefore it will be OK others in the family such as the Vostro 1500, and Inspirons 1520 and 1720.
