barb way say:

What version of Visio are you using?  Did you upgrade the operating system in place, with the installed applications (like Visio) remaining on the system?  You could try to repair Visio, from the Control Panel, to see if that clears up any issue in the conversion of the CAD drawings.

Also, how are you bringing the portions of the CAD drawings in to Visio?  Copy/Paste?  Or are you using Insert > CAD Drawing, or opening the DWG/DXF files directly from disk?  Try changing the method and see if you get better results.

Finally - I have seen examples where the 0,0 point and drawing scale become confused when converting the CAD object.  Try this to see if the objects are somewhere out of sight in the workspace in Visio: 

  • Select All (Ctrl-A) to select all objects in the drawing workspace.
  • If you cannot see the outline of the selection box, zoom out until you can see it.  Sometimes objects change scale, and end up 'surrounding' the drawing page, this should highlight them if they are still present.

If none of those help, try a sample drawing that ships with Visio Professional.  Locate the Visio installation folder, something like :

  C:\Program Files\Microsoft Office\Office12\Samples\1033                         (Note: 1033 is the English language install folder, other languages have different language IDs)

There should be a sample CAD file called "Blocks.DWG".  Test importing it to Visio, then using the CAD Drawing Object > Convert action to turn it into Visio shapes and see if the conversion works as expected. 

Barb

1. Make a backup copy of your front-end, in case this goes awry.

2. Open the front-end.

3. Delete the two linked tables that are causing the problem.

4. On the ribbon's External Data tab, in the "Import and Link" tab group, click Access (assuming your back-end is an Access database).

5. Click the option, "Link to the data source by creating a linked table".

6. Browse to and select the back-end database.

7. Click the OK button.

8. In the Link Table dialog, select the tables to be linked from the back-end, then click OK.

If all goes well, the tables will now appear as linked tables in your front-end.  Hopefully, the problem will have disappeared.  But if there's a problem linking the tables, let us know.
