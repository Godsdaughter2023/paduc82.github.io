rajesh govind say:

 

Hi Laura,

 

 

It would be great if you can provide more information about this issue. Please answer to these questions to assist you in a better way.

 

1)       Is this issue confined to a particular browser?

2)       If yes, did you try opening this website with another browser? Are you getting the same error with that browser as well?

 

A certificate error message on Internet Explorer occurs when the browser detects a website's security certificate contains invalid information or if it has some problem with it. We cannot just ignore this error and proceed to access the website. Internet Explorer helps keep your information more secure by warning about certificate errors.

 

One of the main common reasons for the certificate error is the incorrect system date and time. If the date and time are not synched properly with the time server and it shows an incorrect value, then there are high chances that you will get Certificate Error on all the websites that you try to visit.

 

Thus, if this error occurs with almost all the websites that you try to visit, then I would first suggest you to check the system date and time and make sure that they are updated.

 

If you face this issue even after updating the system date and time and if this happens with a particular website, then either the certificate of the website has really expired or the site is not trustable.

 

If you encounter a certificate error at a website you have visited before without any errors, it's possible the website is not configured correctly. Try retyping the address you are using. For example, if you are typing the address as https://example.com , try changing it to https://www.example.com.

 

Thus, the certificate errors are website-related, and cannot be corrected in Internet Explorer. If you are getting errors on a website you have previously visited successfully, contact the website or webmaster to report the problem. I would also like to inform you that it is not ever safe to ignore a certificate warning and continue to a website. A website's certificate provides identification of the web server. If the certificate has an error, it might indicate that your connection has been intercepted or that the web server is misrepresenting its identity.

 

For more information on the “Certificate Errors”, I would suggest you to refer to the article: “Certificate errors: FAQ” from this link:

 

http://windows.microsoft.com/en-IN/internet-explorer/certificate-errors-faq#ie=ie-10

2016 office for Mac installation on Mac Book Air - Microsoft Community

mw_mba say:

2016 office for Mac installation on Mac Book Air

Hi 

I just installed Office 2016 on my new Mac Book Air and the storage room was occupied 30 GB, is this normal?

paul bowden [-msft-] say:

A full Office 2016 install occupies ~7.5 GB of disk space. I'm not sure if you're 30 GB calculation included the core OS and any other apps you might have installed already.

A couple of thoughts about minimizing the amount of space used by Office:

1. If you don't use some of the Office 2016 apps (e.g. you don't use Outlook because you prefer the native Mail client), then you can safely move that app from /Applications to trash and reclaim some disk space.

2. You may find that temporary files build up in the /tmp folder over time. You can either close all apps, and move those files to Trash, or if you reboot your laptop, the OS will remove files from/tmp automatically.

3. If you previously had Office 2011 installed on the machine, and then installed Office 2016, both will live side by side. If Office 2016 suites all your needs and you no longer have a use to launch Office 2011 apps, you can reclaim some disk space by dragging the '/Applications/Microsoft Office 2011' folder to the Trash. This will reclaim 99.9% of the disk space previously occupied by Office 2011.

Hope this helps!

Thanks, Paul.

Captions for tables and figures - Microsoft Community

gvm7777 say:

Captions for tables and figures

The requirement on me is that captions appear above a table but below a figure. So I modified the caption style (and renamed it accordingly) for tables by formatting the paragraph to 'keep with next'. The idea being that the caption will stay with the first row of the table. Secondly I created a new style and called it 'figure caption' and formatted the paragraph to NOT 'keep with next'. I have two problems:

1. Rows in a table will split across pages even though the table properties are set so that they don't. Instead of this happening, it would be better if the table caption automatically moved to a new page.

2. There doesn't seem to be a 'keep with next' option for images so that a figure consisting of an image will always be seen above its caption, i.e. instead the caption may appear on the next page.

How do I fix this please?

TIA ... Greg

doug robbins - mvp office apps & services (word) say:

If a table will not fit on a page, there is nothing that you can do to prevent it from being split somewhere

For the first issue, if your table does not have heading rows that you need to be repeated, the caption will be more likely to remain with the table if you insert it into a row at the top of the able and from that row, you remove the top, left and right borders.

For the second issue, if you have the images inserted so that the layout is "In-line with text", you can format the paragraph that contains that image so that it is kept with next, which is done by selecting the image and then accessing the Paragraph dialog.
