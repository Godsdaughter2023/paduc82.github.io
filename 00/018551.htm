ossiemac say:

Hello Anne,

Try the following. Note that Target is the selected range so you do not reset it to the ActiveCell and the code tests that only one cell has been changed.

Application.EnableEvents is set to false so the code is not called again each time another cell is changed with the insertion of the formulas.

The on error routine is so that events do not remain disabled if an error occurs in the code and stops it before reaching the code to tune events back on because when events are disabled, they remain disabled until re-enabled with code or Excel is re-started.

Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Cells.Count = 1 Then     'Only process if single cell changed
        On Error GoTo ReEnableEvents
        Application.EnableEvents = False    'Suppress recursive calls due to formulas being inserted
        If Target.Address = "$E$5" Then
           Range("G5").FormulaR1C1 = _
           "=IF(OR(LEFT(R[0]C[-2],4)=""RNAM"",LEFT(R[0]C[-2],4)=""RLAM""),"""",LEFT(R[0]C[-2],4))"
          
           Range("H5").FormulaR1C1 = "=IF(IFERROR(FIND(""RNAM"",R[0]C[-2],1),""RLAM"")=1,IFERROR(SUBSTITUTE(1,1,""RNAM""),""RNAM""),""RLAM"")"
        End If
    End If
   
ReEnableEvents:
    If Err.Number <> 0 Then
        MsgBox "Error occurred in Private Sub Worksheet_Change sub"
    End If
    Application.EnableEvents = True
End Sub
