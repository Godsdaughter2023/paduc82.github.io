These new programs are: Cortana, Groove Music, Photos, Movies & TV, etc. (any of the newer programs not seen in previous versions of Windows seem to cause this problem as well).

3. Sometimes when you first logon to or restart your computer before you hit the login screen and/or the screen where you have a picture just before you put your password in. If you don't hear the Windows Startup Sound here you might be having this Critical Error about to occur but sometimes this doesn't happen all the time. Microsoft has Windows Update install Drivers for Windows 10 but these aren't the ones distributed by your Computer Manufactures on the products page for your computer, instead Windows Update gives the generic version of these drivers, these generic versions of such drivers can be also found at the driver hardware manufacture's websites as well. Here is an example of what I mean about generic drivers, Intel makes the driver and the driver updates for the Intel graphics GPU Driver inside my personal computer, but HP my computer manufacturer modified both the driver hardware as well as the driver software so when attempting to install the driver updates through Intel's Update program it won't allow me to install the driver software and it gives me a notice to go get the updated driver software from my computer manufacturer (HP). This is also bad if you do a Clean Install of Windows 10 for you wipe the hard drive including the manufacturer's installed driver software, then since Windows Update installs driver software, you're getting all generic drivers without any warnings to get them from your computer manufacturers. Such generic drivers can thus cause other problems so the Critical Error problem "could" be connected to such depending on your pc drivers. When I installed Windows 10 back in August on my personal computer Windows Update installed one generic driver and both HP and the driver's manufacturer don't have an updated driver but I always check to see if there is one.

Steps to get rid of the Critical Error TEMPORARILY:

On a last note, following these steps has gotten past the error:

Method 1: Open an Administrator CMD window.  If you can’t find a way, use these steps: 

1.Press CTRL+Shift+ESC to open Task Manager.  
2.Click File > Run New Task  
3.Make sure you have a check mark beside “Create this task with administrative privileges”  
4.Type CMD 

Type the following command at the CMD prompt: 

sfc /scannow

Type the following commands at the CMD prompt: 

dism /online /cleanup-image /restorehealth
   


Method 2: If dism and sfc both completed and show no remaining issues, run these commands in powershell:

Get-AppXPackage -AllUsers |Where-Object {$_.InstallLocation -like "*SystemApps*"} | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

You can also try this powershell command as an alternate one:

$manifest = (Get-AppxPackage Microsoft.WindowsStore).InstallLocation + '\AppxManifest.xml' ; Add-AppxPackage -DisableDevelopmentMode -Register $manifest 

Method 3: Try booting your computer up into Safe Mode:

There are a few ways to get to Safe Mode

Pressing F5 on startup can get you there in Windows 10.

You can also press and hold the Shift Key well clicking on Restart via the Power Icon on the Security Settings Screen or the Login Screen where you put your password.

Another but longer way to get to Safe Mode is to follow these steps:
1. CTRL ALT DEL to get to the Security Settings page

2. Click on Task Manager

3. When the Task Manager window pops up click on "More Details" on the bottom
4. Click File and "Run New Task"

5. Type in the new window that comes up: msconfig (check the box: Run this task with administrative privileges)

6. In the new window that pops up, click the Boot Tab and then check "Safe Mode" and also check "Network". Then click Ok.
7. Restart your computer

Doing it this way requires you to leave Safe Mode or you'll always boot up into it. To Exit Safe Mode, follow these steps:
1. Right Click on your Task Bar, click on Task Manager

2. Click "More Details" if you aren't able to see the "File" menu

3. Click on file and Run New Task

4. Type msconfig again

5. In the window that comes up, click the Boot tab and uncheck "Safe Mode"

6. Restart your computer and the Critical Error should be gone.

Steps to get rid of the Critical Error TEMPORARILY:

On a last note, following these steps has gotten past the error:

Method 1: Open an Administrator CMD window.  If you can’t find a way, use these steps: 

1.Press CTRL+Shift+ESC to open Task Manager.  
2.Click File > Run New Task  
3.Make sure you have a check mark beside “Create this task with administrative privileges”  
4.Type CMD 

Type the following command at the CMD prompt: 

sfc /scannow

Type the following commands at the CMD prompt: 

dism /online /cleanup-image /restorehealth
   


Method 2: If dism and sfc both completed and show no remaining issues, run these commands in powershell:

Get-AppXPackage -AllUsers |Where-Object {$_.InstallLocation -like "*SystemApps*"} | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

You can also try this powershell command as an alternate one:

$manifest = (Get-AppxPackage Microsoft.WindowsStore).InstallLocation + '\AppxManifest.xml' ; Add-AppxPackage -DisableDevelopmentMode -Register $manifest 

Method 3: Try booting your computer up into Safe Mode:

There are a few ways to get to Safe Mode

Pressing F5 on startup can get you there in Windows 10.

You can also press and hold the Shift Key well clicking on Restart via the Power Icon on the Security Settings Screen or the Login Screen where you put your password.

Another but longer way to get to Safe Mode is to follow these steps:
1. CTRL ALT DEL to get to the Security Settings page

2. Click on Task Manager

3. When the Task Manager window pops up click on "More Details" on the bottom
4. Click File and "Run New Task"

5. Type in the new window that comes up: msconfig (check the box: Run this task with administrative privileges)

6. In the new window that pops up, click the Boot Tab and then check "Safe Mode" and also check "Network". Then click Ok.
7. Restart your computer

Doing it this way requires you to leave Safe Mode or you'll always boot up into it. To Exit Safe Mode, follow these steps:
1. Right Click on your Task Bar, click on Task Manager

2. Click "More Details" if you aren't able to see the "File" menu

3. Click on file and Run New Task

4. Type msconfig again

5. In the window that comes up, click the Boot tab and uncheck "Safe Mode"

6. Restart your computer and the Critical Error should be gone.
