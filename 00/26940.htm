hansv mvp say:

In the first place, a SELECT query merely returns values. Opening a SELECT query does not in itself modify any value in the underlying table(s).

You could use a SELECT query as Record Source of forms and reports to display "No Vendor" instead of a blank.

In the second place, any field name (and table name) that contains spaces or punctuation must be enclosed in square brackets [ ].

In the third place, constructs such as CASE WHEN, COALESCE and IFNULL work in SQL Server, but not in Access.

The correct SQL for a SELECT query to be used as record source for forms and reports would be

SELECT [Project Name], Nz([Vendor_Name], "No Vendor") AS [Vendor Name]
FROM [Table]

If you want to give the calculated field the same name as its source field, you must specify the table name explicitly to avoid a circular reference:

SELECT [Project Name], Nz([Table].[Vendor_Name], "No Vendor") AS [Vendor_Name]
FROM [Table]

If you want to replace blanks with "No Vendor" in the table itself, you must run (execute) an UPDATE query:

UPDATE [Table] SET [Vendor_Name] = "No Vendor"
WHERE [Vendor_Name] Is Null

After running this query, all blanks in the Vendor_Name field will have been filled in.

Problem with Renew Office365 - Microsoft Community

general electronic commerce say:

Problem with Renew Office365

We'd 15 licenses for office accounts and wanted to re-new only 12 accounts from vendor. But instead of reduce account to 12 as we preferred they added another 12 accs.

Can you please suggest us for renew the account?

lance cai msft say:

Hi General Electronic Commerce,

Please kindly note that you will get two options while redeeming the product key: Add licenses and Renew your subscription.

  • If you choose Add licenses, licenses quantity will be added up and the expiration date for all licenses will get extended according to certain calculation formula:

Example (The customer adds two users to an existing five-user prepaid subscription):

Customer has a prepaid subscription with five users and 300 days remaining on their subscription.

Customer adds two users. The two new product keys have 365 days.

(5 users * 300 days + 2 users * 365 days) / 7 users = 318.57 rounded up to 319 days for the subscription

In this example, the customer's subscription's end date will be moved out 19 days, because the additional time is spread among all users.

Note: Once the key was redeemed all of the licenses will share one common expiration date.

  • If you want choose Renew your subscription, the assigned license quantity must be equal or less than the license quantity you purchased then you will be able to renew your subscription for a whole year.

For example, now you have 15 licenses and assigned all of the licenses to your users already and all the licenses will get expired on 9/1/2017 and you need to renew the licenses for a whole year.

However, you just purchased 12 licenses then you need to un-assign the licenses first and make the assigned licenses quantity equal or less than 12 then you will be able to see Renew your subscription option.

Hence, you will get 12 licenses in total and use it for a whole year.

Note: You could not renew your licenses separately because whenever you redeem the key, the previous licenses will be covered and all of the licenses will share one common expiration date.
