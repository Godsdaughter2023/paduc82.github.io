¡firedog say:

This is a long-standing problem. Any file attached to a mail message has to be encoded, meaning converted to plain text so that it can be passed through the email system. This is normally not a problem, because the encoded material will be decoded by the recipient's mail client or service. However, certain types of encoding don't decode properly and the resulting file is not a well-formed PDF. The usual remedy is to ask the sender to zip the file before sending (in Windows: right-click on the file to be sent and select Send to > Compressed (zipped) folder), then send the resulting .zip file. I've not seen a detailed explanation of why some encodings don't work properly.

The sender may be able to change the Content-Transfer-Encoding used to transmit the attachment. Base 64 may be the most reliable, but I've no idea whether this makes any difference.

Senders can usually avoid this type of problem by uploading the file to a file-hosting site like SkyDrive and then sending a link to the uploaded file to the recipient.
