zaphod_beeblebrox say: Hi Mikael (and Group),

Holy Moly, I just figured out a solution!

I'll do some experimenting to figure out if this work with other file combinations, but I found a thread talking about this and it worked!

Problem:
- Colleague sends me a .XLS which has an embedded .PDF on one sheet
- Office: Mac 2011 will not open the .PDF because of lack of OLE support

Solution:
- Copy the .XLS file to a new file
- Rename the file from <filename.xls> to <filename.zip>
- Open the .ZIP with Archive Utility
- Browse into the resulting "decompressed" folder to filename>xl>embeddings
- Find the file of interest, named something like <oleObject2.bin>
- Rename to <oleObject.pdf>

VOILA! The PDF is recovered in its fully glory.

On my 8-year-old, non-UEFI, Legacy BIOS mode laptop, a clean install of Windows 10 on an MBR disk with no partitions, places a 500MB NTFS combined System/Recovery hidden partition at the beginning of the disk, followed by a single NTFS partition for the rest of the disk.  The System partition also contains Windows RE for repairing Windows when needed.  So Windows 10 does not need a GPT disk, since my laptop has no way to use one.

From reports I have seen, it appears that Windows 10 install is confused by a UEFI computer that is not setup optimally with a GPT disk.

There is a limit of 4 primary partitions on an MBR disk, so if there are any OEM partitions, they could be using up that partition limit.  Multiple Windows versions could instead be installed on multiple logical drives in an Extended partition (which should keep their restore points separate, if each install is told where to keep its Restore points).  If they are configured to use the same partition for restore points, then that would be a problem, as perhaps you have experienced.

It looks like mine are completely disabled at the moment.  As I mentioned, it's not something I have used more than once or twice.
