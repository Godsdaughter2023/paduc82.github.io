Hello,

So, I got the W10 Anniversary Update and everything has been running smoothly. However, I noticed that my fans would never turn off on my laptop and even on complete idling, the processor would not rest. The processor 'hog', I discovered from the task manager was the 'Windows Audio Device Graph Isolation' i.e. audiodg.exe.

This process is used by/for software enhancements which your audio hardware supports and previously and in previous versions of windows the advice has always been to 'Disable All Enhancements' from the Sound Settings. However, this only applies if you are using certain drivers (mainly windows sound drivers) which has these options.

I use a VAIO i5-3rd Gen, and have Realtek's own drivers and their own enhancements. I tried to disable them and reinstall all drivers and wasted a whole morning before finding the solution.

In the new update, Cortana has a feature which allows her to respond to , 'Hey Cortana' from any screen without opening Cortana or pressing the Mic button. The OS has to keep audiodg.exe in use and open at all times for this to function (it seems! Windows Techs, any light on this?). Turn it off! and no more audiodg.exe processor hog. This is by no means a 'solution', its just a workaround- and for Cortana functionality Windows needs to fix this.

john rubdy say:

Hi Corey,

Welcome to Microsoft community forums.

This error occurs because the remote procedure call service does not wait for all the asynchronous RPC calls to be returned before the service reallocates the structure. Because of this behavior, the asynchronous RPC calls access an incorrect reallocated address. Therefore, an access violation occurs, and the RPC service crashes.

Method 1: A supported hotfix is available from Microsoft to fix this issue refer this article:

Remote procedure call service crashes on a computer that is running Windows Server 2003 SP2, Windows Vista, Windows Server 2008, Windows 7 or Windows Server 2008 R2

http://support.microsoft.com/kb/2401588/en-us

If issue persist, proceed to next method.

 

Method 2: Type Services into the start menu search box.

 

  1. In Services, scroll down to "Remote Procedure Call" and make sure the status says "Started" and set to Automatic.
  2. The second "RPC Locator" should be set to "Manual".

 

Method 3: If the issue persists, as a workaround you may perform scan on your computer using Microsoft Safety scanner and check if it helps:

http://www.microsoft.com/security/scanner/en-us/default.aspx

Note: The Microsoft Safety Scanner expires 10 days after being downloaded. To rerun a scan with the latest anti-malware definitions, download and run the Microsoft Safety Scanner again.

Disclaimer: Any data files that are infected may only be cleaned by deleting the file entirely, which means there is a potential for data loss.

 

Hope this helps. If you need further assistance with Windows, let us know and will be glad to help.
