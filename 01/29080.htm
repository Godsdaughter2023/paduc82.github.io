praseetha k say:

Hi användare,

Step 1: You may try to update the drivers. If you are facing any issues to update the drivers, then try to install the driver setup in compatibility mode.

1. Locate the executable file for the driver installation program.

2. Right-click the file, and then click Properties.

3. In the PackageName Properties dialog box, click the Compatibility tab.

4. Click to select the Run this program in compatibility mode for check box, click Windows XP (Service Pack 2) in the Run this program in compatibility mode for list, and then 5. click OK.

6. Double-click the executable file to start the installation program.

If the driver is not available in setup file format then can manually update the drivers from device manager

Step 2: Manually install the driver

1. Click Start, right-click Computer, and then click Properties.

2. In the Tasks pane, click Device Manager. If you are prompted for an administrator password or confirmation, type your password or click Continue.

3. In Device Manager, locate the device for which you want to install the driver.

Note To display hidden devices, click Show hidden devices on the View menu.

4. Right-click the device and then click Update Driver Software.

5. Click Browse my computer for driver software, click Let me pick from a list of device drivers on my computer, and then click Have Disk.

6. In the Install From Disk dialog box, click Browse, locate an .inf file for the device driver that you want to install, and then click Open.

7. Follow the steps in the Update Driver Software DeviceName dialog box to update the driver.

Hope this information is useful.

    Hi Soph,

    The most common cause of an out-of-date account is an incorrect password. Remove your account from the Mail app then change your password. Re-add your account afterwards.

    Tip: To change your account password for your account, visit the website of your email provider.

    If the password isn't the problem, you may have a certificate error. If so, there should be another message that says, "There is a problem with the server's security certificate. The security certificate is not from a trusted certifying authority."

    This error often happens when you don't use SSL to secure your email account. To fix this error, use the following steps.

    1. Choose Settings > Manage Accounts.
    2. Choose the account that's out-of-date to display the account settings dialog.
    3. Choose Change mailbox sync settings > Advanced mailbox settings. You may have to scroll down in the dialog to see Advanced mailbox settings.
    4. Check the boxes for Require SSL for incoming email and Require SSL for outgoing email then choose Done > Save.

    WordPad in Windows corrupted - Microsoft Community

    hillbilly08 say:

    WordPad in Windows corrupted

    If I install the latest version of WordPad will I lose files stored in the old edition?               
    frederik long say:

    If I install the latest version of WordPad will I lose files stored in the old edition?               

    No, you won't.

    ken blake say:

    Your WordPad data files are *not* stored in the "old edition." They are probably stored in your documents folder, but it really doesn't matter where they are stored. They were stored *by* WordPad, but they are not part of WordPad.

    These files are probably .rtf files, but they could be some other types too. The important thing to realize is that these files are not WordPad files. In fact they have nothing to do with WordPad other than that WordPad is one of *many* programs that can read, edit, and write them.

    So to answer your question, no matter what you do to WordPad (even if you remove it entirely), it has no effect on your existing files. Think of it like a screwdriver that you used to assemble a piece of furniture; if you throw away the screwdriver or replace it with a new one, you don't lose the furniture. Losing the tool (the screwdriver or WordPad) that built it doesn't lose what it built (the furniture or  data file).

    One other point: your question suggests that you don't do regular backups of your important data to external media. If that's the case. you've been playing with fire.  Almost everyone should be backing up to external media regularly. It is always possible that a hard drive crash, user error, nearby lightning strike, virus attack, even theft of the computer, can cause the loss of everything on your drive. As has often been said, it's not a matter of whether you will have such a problem, but when.
