Found a solution. But first, need to let you guys know what the fault was. What was the fault? I am not sure, but what finally solved my issue was something to do with the user profile. Apparently, some kind of software was installed that caused the failure to connect or reconnect to the shared folders from NAS drives.

A new user was created with standard and a new user was created with admin rights. The current user was logged off and the standard user had been logged into the system. The NAS shared folders were available from Windows Explorer and was also able to be accessed. Meaning that files could be copied, cut, pasted... in other words could be managed like any other folder. This is true with the administrator as well. The shared folders were able to be accessed without any issues, but when logged into the system with the original user, the shared NAS folders were not available. The dead links from the windows explorer network section had faulted where no access could be observed. Only when new users were created did the shared network folders have access to those shared folders.

Final results to resolving this issue. As stated before, something occurred either when a software application blocked the shared folder path or either a configuration change had interrupted the path to the shared folders. From trial and error for months and months, I could not be specific as to what actually caused the shared folder to fail.

In conclusion, I decided to blow off the OS and rebuild the system from a clear install of Windows 10 Pro x64 OS. After a clean installed, the shared folders were available and could be accessed from Windows Explorer. All network devices could be pinged and function as expected.

Regards to all those that are experiencing the same or similar issues. I wish I could pin point what was causing the issue to occur, but after months and months of configuration changes, trail and error testing, nothing seem to work. I thought a couple of times I was close, but after a reboot, the problem resurfaced.

For me, the fix was simple and pure. Blow off the drive, rebuild Win10 and live life again.

Good luck to you all...
