1. A driver has inadvertently, or deliberately, modified critical kernel code or data.

2. A hardware corruption occurred.  For example, the kernel code or data could have been stored in memory that failed.

  2. A dialog box will open type “msconfig”
  3. Locate and select the Boot tab and select Safe with Minimal as option.
  4. Restart Computer

Note: Follow the same procedure to undo Safe Mode, uncheck the Safe option and restart your system.

  • You will be prompted to Restart the system, click on Restart.
