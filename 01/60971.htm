The Dynamic Host Configuration Protocol (DHCP) is a computer networking protocol used by devices (DHCP clients) which dynamically distributes the IP address to the destination host. RFC 1531 initially defined DHCP as a standard-track protocol in October 1993. The next update, RFC 2131 released in 1997 is the current DHCP definition for Internet Protocol version 4 (IPv4) networks. The extensions of DHCP for IPv6 (DHCPv6) were published as RFC 3315.

i.              Click on start button

ii.            Type in services.msc in search box.

iii.           Search for DHCP client and set it to automatic.

2.    Check the manufacturer’s website of your network adapter and router for the latest drivers and update to the drivers.

3.    Disable the anti-virus or firewall on your computer and the router.

4.    Check if DHCP service is enabled. If not then continue these steps.

a. user say:

Here's the procedure: http://www.howtogeek.com/howto/windows-vista/using-windows-vista-system-restore/.  Be sure to check the box to show more than 5 days of restore points (this could be why you only see a couple of days of restore points).  If the first attempt fails, then try an earlier point or two.  NOTE: You will have to re-install any software and updates you installed between now and the restore point, but you can use Windows Update for the updates.  If checking the box still shows no more than two days of restore points, then the others have somehow been deleted because they WILL show up here if they exist - there's no archive.  I do not know why they no longer exist.  They could have been overwritten or deleted - it's hard to tell since it doesn't keep a log of restore point activity.

You could be infected with malware - malware frequently attacks System Restore since that's one way to recover from them and they are protecting themselves.  While in Safe Mode with networking (click repeatedly on F8 during the boot process) try to run anti-malware programs. To fix this problem (if it is a problem) download,  install, and run the following two programs: http://www.malwarebytes.org/mbam.php and http://www.safer-networking.org/en/index.html.  You may also want to try a free trial of Pandasoft (www.pandasoft.com).  Before downloading pandasoft, uninstall whatever anti-virus program you are currently using (because installing and running two at the same time can cause conflicts, freezes, and all sorts of problems).  Once you've uninstalled your current AV program, download, install, update, and scan using Pandasoft.  I think you might be amazed at how much it finds that the other software missed.  Once complete, uninstall pandasoft and re-install your current AV program (unless you decide to switch which I did when I first tried it about 4 years ago).  You may also want to try the new, free Microsoft Security Essentials http://www.microsoft.com/security_essentials/default.aspx (with the same caveat that only ONE AV program can be installed and running on your system at any one time). Reboot after completing all the scans.  You may also want to try the free OneCare at http://onecare.live.com/site/en-us/default.htm and let it run all the options because that’s good maintenance (it will take some time to complete but can be done in the background).

If you can't find any further restore points, you will have to try to download and install the correct drivers for your devices (yes, I know you're having trouble with that but we'll have to get past that trouble somehow).  Go to Device Manager by going to start / search box and type device manager and enter and then double-click on the program icon that appears.  Check each device for a red x, yellow ! or white ?.  These identify devices with problems (probably drivers, but also conflicts or something else).  Click on each for further details and troubleshooting tips.  If you need to get drivers, do so from the computer vendor or device manufacturer (NOT from Microsoft Updates).  In fact, you should turn off automatic driver updates from Windows Update as follows: http://www.addictivetips.com/windows-tips/how-to-disable-automatic-driver-installation-in-windows-vista/. You should fix these devices as well as the ones where you deleted the then present drivers.  Proceed as follows: http://pcsupport.about.com/od/driverssupport/ht/driverdlmfgr.htm.  Once you have the drivers, you can install them through Device Manager as follows: http://www.vistax64.com/tutorials/193584-device-manager-install-driver.html.  If you can't install the new drivers, there may be something else wrong with your system (and it could be further evidence of malware).  Post back here if none of these procedures work and we'll try to talk you through installing the drivers (you'll need to tell us the device manufacturer and model number and version number of the devices).  And we'll need to know the version of Vista you are using (32-bit or 64-bit)..

Another option is to rollback device drivers to the version that was in place before the version currently in use.  This may restore your old drivers which were working.  Here's the procedure: http://www.howtogeek.com/howto/windows-vista/roll-back-troublesome-device-drivers-in-windows-vista/.  I'm not sure this will work if you deleted the drivers (in fact, I don't think it will), but it's worth a try (on at least one driver as a test) if nothing else is working.

I hope this helps.

Good luck!
