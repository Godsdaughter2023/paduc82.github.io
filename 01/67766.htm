UAC - "Unknown Publisher" alert with Microsoft programs and - Microsoft Community

fran- say:

UAC - "Unknown Publisher" alert with Microsoft programs and system settings after FixIt 50202

I have had some issues with the Windows Update list being empty . Since a simple reboot did not help in my case, I searched around and found FixIt 50202 - "How do I reset Windows Update components?"

I run the fix once some time ago, and it solved the problem. However the issue came back, so after a month or so I decided to run it again, this time in (d'oh!) aggressive mode.

The issue was solved again, but after restarting UAC has begun to prompt for every single action that requires elevation, even those that should fall under the "changes I make" class. My UAC settings officially have not changed, and I have tried moving them around, restarting, and so on.

What I get, for each and every single program or setting that requires elevation is the standard UAC dialog asking "Do you want to allow the following program from an unknown publisher to make changes to this computer?"

An example?  The services dialog, i.e. services.msc, no shows up as


Program name:    mmc.exe

Publisher:            Unknown

File origin:           Hard drive on this computer

 

Has UAC, or my system, lost the ability to recognise the Microsoft signature? Any registry tweak to fix this?

 

Oh, I have no restore point before the FixIt...

fran- say:

I think I have solved the problem :-)

Among the steps taken by FixIt 50202 there is

  • Rename the following folders to *.BAK:  [...] %systemroot%\system32\catroot2

For what I understand, this folder contains the Security Catalogs installed on the system, that is, signatures of files and com objects, and certification infos. Renaming/removing then was not a safe choice, and left me without signatures for most of the windows exectuables.

I searched for these files on the Windows 7 DVD, and I could not find them. However, using 7zip I was able to browse the install.wim archive on the dvd, and extract the contents of the catroot2 directory. Copying this file from the cd made the trick:

C:\Windows\System32\catroot2\{F750E6C3-38EE-11D1-85E5-00C04FC295EE}\catdb

The file was locked by the Cryptographic Services, so I had to stop the service, overwrite the file, and restart the service.

I still have a few files that appear as "Unsigned" using sigcheck *, which I suppose were not part of the initial installation. I hope the issue will solve itself with future updates, but now think are working normally again.

[*] before, almost all the files in my System32 directory were "Unsigned" !

 

[edit] all done now: I copied some backup .cat files from Windows\WinSxS to the catroot\{F...} directory, and that fixed the signature of the last few files!

Hi,

You will find that it is the Graphics Drivers that is the problem, not the Monitor.

Go to either your Computer or Graphics Card Manufacturer's website > look for a Drivers and Download Section > look for your Computer or Graphics Card Model Number > your new Operating System > Drivers for it: either Video / Graphics / Chipset (depending on wording there) > download and install the 8.1 Drivers.

If there are no 8.1 Drivers available, the upgrade should not have been done.
