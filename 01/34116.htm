The folder C:\Documents and Settings\<username>\Local Settings\temp normally will have a number of files with names such as
     Microsoft .NET Framework 2.0-KB2836941_20140212_231457562-Msi0.txt

     Microsoft .NET Framework 2.0-KB2836941_20140212_231457562.html

The KB*.NET Framework*.txt files are the installation logs from .NET Framework patches applied by Windows Update. The only reason to keep any one of them is if Windows Update notifies you that a patch failed to install, and if you want to find out what error caused the failure. The logs are hard to read, so the corresponding .html file will show a summary in your web browser. If lots of patches fail to install, you would have to find someone who understands them to examine the big log files to find out what's going wrong.  Once an update patch has successfully installed, there's no reason to keep these files any longer, and it won't damage anything to delete them.

None of these files should be 125 GB.  What is the name of the file you see that is that large ... or are you referring to a total of all such files?

Run Disk Cleanup (Start > Run > cleanmgr.exe > OK)

This may or may not remove the .NET Framework related files.  If it doesn't just delete them manually.

With respect to your second issue, please elaborate "unable to properly install updates."  If you get error messages, please provide the complete text without paraphrasing.

lemp say:

One of the things I like about trying to answer questions in these forums is that I learn new things all the time. 

I hadn't noticed that particular temp folder before.  It is the temp folder for the system account. 

As I wrote before, the files that begin with "Microsoft .NET Framework" are log files generated when updates to the various .NET Frameworks are installed.  It makes sense that these files are here because the installation is done by the System account.

Yes, it is safe to delete all the files in this folder, including those other files shown at the top of your screenshot. 

The reason that the Local Settings folder is "ghost like" is because it is a hidden folder.  You have Windows Explorer configured to view hidden files and folders or you wouldn't see it at all.

I'm not expert enough with respect to Windows Update to answer your second question.  Post a new question in the Windows Update forum --> http://answers.microsoft.com/en-us/windows/forum/windows_update?tab=Threads

skeeter quill say:

Could be a number of things that cause this. Poor WiFi connection and/or interference, a bad Ethernet cable, faulty router, hardware issue in the console, other devices hogging the bandwidth, bad line from your service provider.


First thing to try would be rebooting both your console and router, and disconnecting all other devices that are connected to your router, and if you are connected via Ethernet then try replacing the cable. If none of that helps, contact your service provider and ask them to run a check on the line.


When resetting your router and console, it's also a good idea to disconnect them from the power source for 30 seconds or so too.


Failing those, you could try to factory reset your console. The last option after that would be to send your console in for repair.
