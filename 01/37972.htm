Here are four ways:
 
1. Go to a command prompt and issue the command
 
dir [drive:folder] > c:\tempfilename (you can use any name and put it
in any folder you want)
 
Then open notepad, open tempfilename, and print it from there.
 
2. Write (for example in Notepad) a 1-line text file:
DIR %1 /O >LPT1:
 
Save it as "printdir.bat" in the "Send To" folder.
 
Then, to print list of files in any folder, right-click that folder
and select Send to | printdir.bat
 
To include subfolders, change the command to DIR %1 /O/S >LPT1:
 
3. Go to
http://support.microsoft.com/default.aspx?scid=kb;EN-US;Q321379 and
follow the instructions there.
 
4. Download and use any of the several freeware/shareware utilities
that can do this, such as the popular
http://www.karenware.com/powertools/ptdirprn.asp

lemp say:
The best clue I have is that the laptop performs perfectly in Safe mode with networking. I tried booting with all services disabled (it was a Windows boot option) but that just broke everything. 

Given that you're in IT, you probably know most of the following, so I apologize if this is too simplified.


Because safe mode with networking disables pretty much everything except the services you need for networking, your observation suggests that something that is running in normal mode is causing the problem. 

Generally, the most intrusive software is your antivirus and other security software.  These are usually disabled in safe mode (with or without networking).  You may not be able to completely disable your a/v while running in normal mode.  I suggest disconnecting the router from the Internet and then uninstalling the a/v.  Many a/v programs (especially Norton and McAfee) require special uninstaller tools.  There's a list here --> http://kb.eset.com/esetkb/index?page=content&id=SOLN146

If uninstalling security software doesn't improve the normal mode performance, the only approach I know of to deal with this is brute force (I suspect there's some more sophisticated method, but I don't know of it).  It's very tedious.

Start the system in normal mode

Run Task Manager and make a list of all the processes and services that are running

Restart in safe mode with networking and make a list of all the processes and services that are running

Compare the two lists and determine what is NOT running in safe mode.


Restart in normal mode.

If you start with the services rather than the processes, you can research what the different services do here --> http://www.blackviper.com/service-configurations/black-vipers-windows-7-service-pack-1-service-configurations/

Start the Services applet (type services.msc in the Search box). 

Systematically, stop the services that run in normal mode but not in safe mode.  Keep in mind that some services depend on other services.  You need to be aware of these relationships (which are listed at the BlackViper site above) in order to disable services individually.

You can't do it the other way -- run in safe mode with networking and start things up one by one -- because most of the services that are disabled in safe mode simply won't let you start them while in safe mode.
