i-panda say:

This is as close as you are likely to get to the Guest a/c of old:-

Firstly, open an elevated command prompt by going to Start menu/All Programs/Windows System, right-click Command Prompt, select More, select Run as administrator, then;

  1. Enter "net user Visitor /add /active:yes" (ignore quotes) to create a new account. You can replace “Visitor” with whatever name you wish, but be sure to change it for every command on this list.  “Guest” is reserved by the Windows system.
  2. Enter "net user Visitor *" (ignore quotes) to create a password for the account.
  3. Press Enter key twice to create a blank password.
  4. Enter "net localgroup users Visitor /delete" (ignore quotes) to remove Visitor from the regular Users group.
  5. Enter "net localgroup guests Visitor /add" (ignore quotes) to add Visitor to the guests user group.

lemp say:
This may be the result of your antivirus or antimalware application mistakenly deleting or quarantining a necessary Windows system file.

What antivirus application is installed?  Take a look in its log or quarantine area and see what files may have been removed around the time the problem started.

Have you recently done a scan with another antimalware application?  If so, look in its log file as well.

Do you have a genuine Microsoft Windows XP install CD?  If so, what service pack is it and what service pack is currently installed on your computer (right-click on My Computer and select Properties)?

If you see files in your a/v quarantine or log file but can't determine if they are Windows system files (Google is helpful here), list the file names in a Reply here BEFORE you try step C below.  Especially look for files with the SYS file extension such as IPSEC.SYS or TCPIP.SYS.

The following procedures may or may not fix the problem.

A.  Reset the TCP/IP stack (from http://support.microsoft.com/kb/299357)
  1. To open a command prompt, click Start and then click Run. Copy and paste (or type) the following command in the Open box and then press ENTER:
    cmd
  2. At the command prompt, copy and paste (or type) the following command and then press ENTER:
    netsh int ip reset c:\resetlog.txt
    Note If you do not want to specify a directory path for the log file, use the following command:
    netsh int ip reset resetlog.txt
  3. Reboot the computer.
B.  Reset winsock (needs at least Service Pack 2) (probably will not help, but won't hurt) (from http://support.microsoft.com/kb/811259)
  1. To open a command prompt, click Start and then click Run. Copy and paste (or type) the following command in the Open box and then press ENTER:
    cmd
  2. At the command prompt, copy and paste (or type) the following command and then press ENTER:
    netsh winsock reset
  3. Reboot the computer.
C.  If neither of the commands above work AND YOU HAVE NOT BEEN ABLE TO IDENTIFY A WINDOWS SYSTEM FILE THAT WAS REMOVED BY YOUR ANTIVIRUS, then

  • if you do have a real Windows XP CD, backup all of your data and do a repair install following the directions here --> http://smokeys.wordpress.com/2008/07/20/how-to-recover-a-really-dead-windows-xp-sp2sp3-tcpip-stack/

  • if you do not have a Windows XP CD or do not want to do a repair install, follow the directions at the link above under "Hard core method when nothing else is working."  This procedure will make significant changes to the Windows Registry.  I suggest making a backup before following this procedure.  ERUNT is a good tool for doing so.
