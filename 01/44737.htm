Any device with a 32 GB SSD is not good enough to run Windows 10, regardless of the system specs put out by Microsoft.

I honestly don't know why the manufacturers produced these devices.

The Windows 10 folder with a couple of updates will very quickly eat up 20 - 25GB of your hard drive, mine is currently 23GB

https://www.microsoft.com/en-us/software-downlo...

If you have problems booting from a boot disc, you may have UEFI BIOS:
Insert the Bootable Installation Media, then go into your BIOS and make the following changes:
1. Disable Secure Boot
2. Enable Legacy Boot
3. If Available enable CSM
4. If Required enable USB Boot
5. Move the device with the bootable disc to the top of the boot order
6. Save BIOS changes, restart your System and it should boot from the Installation Media

If I do an Explorer wildcard search for e.g. 123-4567-??A I get results for files ending -01A, -02A, -03A, etc. The same works for a search ending -??B and -??C. However searching for 123-4567-??D returns all files in the directory matching 123-4567- presumably because of the 'D' in the .doc suffix. Even if I search for 123-4567-??D.doc I also get a match against files ending -01A, -01B, -02C etc.

cmdrkeene say:

Hmm, what about surrounding the search string in quotes so that the D is terminated by a quotation (that way the OC in DOC won't get matched).  I'd try this super specific search text:

filename:"*??D.doc*"

That should get anything with at ends with two characters followed by a D as long as it's a doc or docx file type.  Although you could probably take out the doc but keep the dot and asterisk before the closing quote.

stevewilson_55 say:

Thanks, the working solution is to include the .doc extension inside the quotes, having only the filename part in quotes still results in a match against all files.

Katt,

 


System Restore can take up to 15% of your partition's space but that can be reduced. It's a trade-off. The less space you allocate, the fewer restore points you will be able to retain - but many people reduce this to save some space while ləaving enough to save 3-4 restore points at ləast (which is usually enough - until you need them). Here's the procedure:

<<http://www.howtogeek.com/howto/5482/make-system-restore-use-less-space-in-windows-7/>>

 

 

To store restore points, you need at least 300 megabytes (MB) of free space on each disk that is 500 MB or larger.


System Restore might use between three and five percent of the space on each disk. As the amount of space fills up with restore points, it deletes older restore points to make room for new ones. System Restore doesn't run on disks smaller than 1 gigabyte (GB).
