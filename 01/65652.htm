Hi,

Here we will try to install Windows 7 by copying the contents of the DVD to the hard drive on your computer.

After copying the contents, you need to make a USB flashdrive bootable device and you may start the installation procedure.

Requirements:

*USB Flash Drive (Minimum 4GB)

*Windows 7 or Vista installation files.

Follow the below steps to create bootable Windows 7/Vista USB drive using which you can install Windows 7/Vista easily.

1. Plug-in your USB flash drive to USB port and move all the contents from USB drive to a safe location on your system.

2. Open Command Prompt with admin rights. Use any of the below methods to open Command Prompt with admin rights.

*Type cmd in Start menu search box and hit Ctrl+ Shift+ Enter.

Or

*Go to Start menu > All programs > Accessories, right click on Command Prompt and select Run as administrator.

3. You need to know about the USB drive a little bit. Type in the following commands in the command prompt:

First type DISKPART and hit enter to see the below message. 

Next type LIST DISK command and note down the Disk number (ex: Disk 1) of your USB flash drive. In the below screenshot my Flash Drive Disk no is Disk 1.

4. Next type all the below commands one by one. Here I assume that your disk drive no is “Disk 1”.If you have Disk 2 as your USB flash drive then use Disk 2.Refer the above step to confirm it.

So below are the commands you need to type and execute one by one:

SELECT DISK 1

CLEAN

CREATE PARTITION PRIMARY

SELECT PARTITION 1

ACTIVE

FORMAT FS=NTFS

(Format process may take few seconds)

ASSIGN

EXIT

Don’t close the command prompt as we need to execute one more command at the next step. Just minimize it.

5. Next insert your Windows7/Vista DVD into the optical drive and check the drive letter of the DVD drive. In this guide I will assume that your DVD drive letter is “D” and USB drive letter is “H” (open my computer to know about it).

6. Maximize the minimized Command Prompt in the 4th step.Type  the following command now:

D: CD BOOT and hit enter.Where “D” is your DVD drive letter.

CD BOOT and hit enter to see the below message.

7. Type another command given below to update the USB drive with BOOTMGR compatible code.

BOOTSECT.EXE /NT60 H:

Where “H” is your USB drive letter. Once you enter the above command you will see the below message.

8. Copy your Windows 7/Vista DVD contents to the USB flash drive.

9. Your USB drive is ready to boot and install Windows 7/Vista. Only thing you need to change the boot priority at the BIOS to USB from the HDD or CD ROM drive. I won’t explain it as it’s just the matter the changing the boot priority or enabling the USB boot option in the BIOS.

Note: If you are not able to boot after following this guide means you haven’t set the BIOS priority to USB.

igfxpers.exe belongs to software Intel® Common User Interface and is sometimes used by nVidia graphic cards drivers. It is responsible for enabling display resolution settings to persist when a monitor is disconnected or turned off. Although it might not be essential for desktop computers, many notebook devices might require this process to run on startup. This process can easily be stopped from running on startup from the msconfig and is not essential to Windows 8.1 operating systems.

Typical file (disk image) location:

C:\Windows\System32\igfxem.exe

I would suggest you to start the computer in safe mode and then place the computer in clean boot state.

Step 1: Start the computer in safe mode

Safe mode is a troubleshooting option for Windows that starts your computer in a limited state. Only the basic files and drivers necessary to run Windows are started. The words Safe Mode appear in the corners of your monitor to identify which Windows mode you're using.

a. When you hit the power button in Windows 8.1 (through Charms Bar), hold down the Shift key when clicking the Restart button.

  • F5 for Safe Mode with Networking
  • F6 for Safe Mode with Command Prompt

To help troubleshoot error messages and other issues, you can start Windows by using a minimal set of drivers and startup programs. This kind of startup is known as a "clean boot." A clean boot helps eliminate software conflicts.

zacabeb say:

That is a great question.

There are a couple of things that the Xbox One must be able to do because of its ability to loop through video from a set top box and overlay graphics. Firstly, it must be able to genlock onto the incoming signal to match its exact timings. Secondly, it must accept a multitude of input resolutions. Thirdly, it must accept both YCbCr and RGB for input. It should have settings for output color space and resolution like the Xbox 360, and hopefully they'll work correctly.

This means that they must do a much better job handling video output than they have with the Xbox 360. Or let's say, they have to do as good a job or better than they did initially with the Xbox 360. Before they botched its video output, it handled things correctly. The big problem was game developers using poorly calibrated TVs and monitors in development, working in environments that weren't light controlled, and not understanding the difference between sRGB and the piecewise linear gamma approximation used by the Xenos GPU.

But when it comes to settings, the only thing that should be needed is an in-game or global gamma adjustment, and brightness. Everything else should follow spec. In YCbCr at 8 bits per channel, black is at code 16, white is at code 235 with "super white" being able to extend up to code 254. In full range RGB at 8 bits per channel, black is at code 0 and white is at code 255. There isn't any room for pumping up contrast or saturation without clipping. In xvYCC there is, but will that even be used by devs? I think they will stick with targeting 24-bit sRGB and having the Xbox One convert it to whatever the display expects.

Adjustments for brightness, contrast and saturation would add on top of those of the display and potentially make things worse. The source component should follow spec, allow for the display to be calibrated to match those specs, and the content should be designed following those specs. Adjustments to compensate for deficiencies in the viewing environment should be fairly strategic. Valve are the only ones this generation who have done fairly correctly.

In plain English: Many current games are too dark even if the settings are correct. It's because of developers, developers, developers, developers doing things wrong. If they don't, things should work.

There's more though. Because of the analog sunset, the Xbox One doesn't have any analog AV output. Thus they don't have to worry about NTSC, YPbPr or VGA, only HDMI and HD. That might help both Microsoft and developers get things right.

zacabeb say:

Sony probably does have more dedicated video techs involved in designing software and firmware. The update for the Xbox 360 which broke its handling of video levels back in the day seems like it was the result of having people inexperienced with video handling things based on API's already written.

Color management in computers and printers is a necessity because of limitations in technology and differences between additive and subtractive reproduction. It's been a must have, whereas television has been the Wild West since the beginning.

Reproduction gamma for instance is not standardized very well at all. Set-top boxes and tv's don't handle things correctly. This has become ever more apparent to me lately in designing a new test pattern for Swedish Television - in developing it we set out knowing standards were being followed very poorly, and came out realizing they were being followed even more poorly than we had imagined. Things such as set-top boxes converting the output to RGB and causing banding, instead of leaving it in YCbCr (SD to HD requires a color space conversion from BT.601 to BT.709, but it's a different story).

I don't know what is going to happen with the Xbox One handling the output of a set-top box that flags its output incorrectly, or what additional deterioration could happen as it overlays graphics. But I'm dreading it could be a mess.

And gamma is a big issue in itself. Control for gamma is certainly needed for game graphics, but what is reference display gamma? The EBU has tested professinal monitors (such as Sony's BVM range with Trinitron CRT's) and found that their display gamma roughly equals 2.35, and regards that as a target display gamma. This is undercompensated in cameras according to BT.709/BT.1847, with a the Opto-electric Transfer Function or OETF that roughly equals 0.51.

The result is a system gamma - a difference in between display gamma and encoded gamma - by 1.2. This system gamma and display gamma assumptions are met by the OETF specified in BT.709/BT.1847.

However, modern displays and other CRTs but the Trinitrons used in Sony's BVM range may have very different reproduction. Screens are bigger, we watch tv in brighter rooms, and people are divided on whether display gamma should be considered this or that. Some swear by 2.22, some swear by 2.35. I consider the latter to be more appropriate since that's what production monitors use, and compensation for the viewing environment should be done relative to that. But I'm far from an expert on gamma.

So in essence, games must have a gamma control unless they're bright enough that variations in gamma are not a problem. Games with darker environments need a gamma control, and potentially a brightness control to compensate for the dip in the darkest shades on many displays.

But unless developers adhere to standards, such controls may themselves turn out useless as they will alter gamma and brightness in a way that doesn't help.
