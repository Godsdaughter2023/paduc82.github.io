ianinfrance say:

From what I've seen elsewhere in these forums, this is due to a really dumb mistake in the way in which the copy of the reserved system partition is taken. It was discovered a long time ago and should have been fixed in SP1.

If Windows is using more than 50% of the space on the drive, or of that partition, then the initial copy can't be taken - so it can't then be moved to your destiination disk.

As the reserved partition is relatively small at 100Mb it's usually this that causes the problem. The best solution I've seen suggested so far was to increase the size of that partition. But that can't be done directly, because the main windows partition is next to it.

So it's a two step solution. First you need to provide some free space between these partitions, by moving "up" the start of the main partition and then you can increase the size of the reserved system partition. This reserved partition must not be as large as 500mb. I used 200mb and it was fine. It's always nerve racking fiddling around with partitions especially on your boot disk, but there's a free program called Easeus available here:- http://www.partition-tool.com/ which did it with as little fuss as one could ask for.
