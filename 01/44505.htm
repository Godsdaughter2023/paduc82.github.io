Just to make sure everyone is satisfied - here is the content of the article:
  

  1. Open the Start menu and click your user name to open the User folder
  2. Right-click the personal folder you want to redirect to another location.
  3. Select “Properties”
  4. Click the tab “Location”
  5. The dialog box shown below will open
  6. Click the button “Move”
  7. An Explorer dialog “Select a destination” will open
  8. Browse to the location where you want to redirect this folder. You can select another location on this computer, another drive attached to this computer, or another computer on the network
  9. Click the folder where you want to store the files (not the root of a drive)
  10. Click the button “Select Folder”
  11. Click “OK”
  12. In the dialog that appears, click “Yes” to move all the files to the new location.

List of personal system folders where this is applicable:
  

Not all system folders can be redirected this way. On my system these system folders located under Users\{User name}\ had a tab “Location” in their properties sheet and could be redirected:
  

  • Contacts
  • Desktop
  • (My) Documents
  • Downloads
  • Favorites
  • Links
  • (My) Music
  • (My) Pictures
  • Saved Games
  • Searches
  • (My) Videos

If you wish to put folders back in their original location, use the button “Restore Default” shown in the graphic.

You would think Windows would be smart enough NOT to open apps and position their windows UNDER the Start Bar when it's positioned on top.  You can't move the windows at all because the move area of the window is under the start bar.

This problem has ALWAYS existed since Windows XP.  That's a long time for such a gross oversight.  However, back in the XP days, you could right click and select "MOVE" and use the arrow keys to move the window out from under the Start Bar.  That option doesn't exist anymore in Windows 7.

Is there anyway to finally fix this issue?

jeeped say:

I can only reproduce this behaviour if Auto-hide the taskbar is on while theTaskbar location on screen is Top. For what it is worth, the behaviour is mirrored vertically but only if both if Auto-hide the taskbar is on while the Taskbar location on screen is Bottom and it is probably true for each side as well. Turn off Auto-hide the taskbar and the application windows reposition themselves accordingly. I would suppose this is by design because if you need the vertical real estate enough to hide the taskbar, it makes no point in repositioning windows away from that real estate strip.

As for moving application windows around a Windows 7 desktop, there are a couple of things you might wish to try.

First, the age old ALT+Spacebar still gains access to the Application Control Menu. When you have it visible, type M to initiate Move, then use the arrow keys to reposition and hit <enter> when you are satisfied with the result. There are a few applet windows that do not appear to have an application control menu (e.g. Desktop, Personalize) but still respond to this legacy command.

Second, you can right click an Aero Peek Thumbnail to get the same Restore, Move, Size, etc menu. This is not the taskbar icon but the thumbnail displayed above the icon when using Aero Peek.

So it would appear that all is not lost. I hope you can find something here to help your Windows 7 experience.
