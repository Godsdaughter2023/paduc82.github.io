wunders say: My guess is that you were using the built-in administrator account for your first account.  When you then add any other account, the built-in administrator account is then hidden from the "Friendly Welcome" login screen. (The expected behavior is that the first account created be an administrator account for you with other accounts to follow -- the internal administrator is then hidden for emergency purposes)   Since the contents of folders under "Documents and Settings" for other users are by default protected, you can't see or access them from your other user.  All this is expected.  Chances are your files are still there and still recoverable.

To verify that your other account still exists, bring up a command prompt window (Start -> Run -> "cmd") and enter the command:
    net users
This will print a list of all users that currently exist on your computer.  To login under your old administrator account, you may have to temporarily disable the Friendly Welcome login screen and manually provide the user name and password in the "Classic" logon screen.  The procedure to do this is given here:

"How to change the logon window and the shutdown preferences in Windows XP"
  < http://support.microsoft.com/kb/291559 >

which basically will tell you to hit Alt-Ctrl-Del twice at the Friendly Welcome screen.

I can't help you if you end up having password difficulties.  But a last resort might be to boot your machine from a free Live Linux CD such as Knoppix, back up your files to a USB drive, and reinstall Windows.  Other solutions might be Googled from non-Microsoft sources.

The KERNEL_MODE_EXCEPTION_NOT_HANDLED bug check is a very common bug check. To interpret it, you must identify which exception was generated.

Common exception codes include the following:

  • 0xC0000005: STATUS_ACCESS_VIOLATION indicates that a memory access violation occurred.

Resolving the Problem

If you are not equipped to debug this problem, you should use some basic troubleshooting techniques:

  • Make sure you have enough disk space.
  • If a driver is identified in the bug check message, disable the driver or check with the manufacturer for driver updates.
  • Try changing video adapters.
  • Check with your hardware vendor for any BIOS updates.
  • Disable BIOS memory options such as caching or shadowing.

If you plan to debug this problem, you might find it difficult to obtain a stack trace. Parameter 2 (the exception address) should identify the driver or function that caused this problem.

If you do not know the specific cause of the exception, consider the following items:

  • Hardware incompatibility. Make sure that any new hardware installed is listed in the Microsoft Windows Marketplace Tested Products List.
  • Faulty device driver or system service. A faulty device driver or system service might be responsible for this error. Hardware issues, such as BIOS incompatibilities, memory conflicts, and IRQ conflicts can also generate this error.

If the bug check message lists a driver by name , disable or remove that driver. Also, disable or remove any drivers or services that were recently added. If the error occurs during the startup sequence and the system partition is formatted with NTFS file system, you might be able to use Safe Mode to rename or delete the faulty driver. If the driver is used as part of the system startup process in Safe Mode, you have to start the computer by using the Recovery Console to access the file.

If the problem is associated with Win32k.sys, the source of the error might be a third-party remote control program. If such software is installed, you can remove the service by starting the system by using the Recovery Console and then deleting the offending system service file.

Check the System Log in Event Viewer for additional error messages that might help identify the device or driver that is causing bug check 0x1E. You can disable memory caching of the BIOS to try to resolve the error. You should also run hardware diagnostics, especially the memory scanner, that the system manufacturer supplies. For more information about these procedures, see the owner's manual for your computer.

The error that generates this message can occur after the first restart during Windows Setup, or after Setup is finished. A possible cause of the error is lack of disk space for installation and system BIOS incompatibilities. For problems during Windows installation that are associated with lack of disk space, reduce the number of files on the target hard disk drive. Check for and delete any temporary files that you do not have to have, Internet cache files, application backup files, and.chkfiles that contain saved file fragments from disk scans. You can also use another hard disk drive with more free space for the installation.

irfan h say:

Hello eastcoastgirl_28,

Well there is an option that you can use with the DVD Maker to sync the Audio size with your Picture, if it is not working with Windows Movie Maker.

Here is what you can do. Open Your Pictures with the DVD Maker instead of Movie Maker.

Adjust the settings of the Slideshow in Windows DVD Maker

 To adjust the settings of the Slide show follow these steps.

1.     Open Windows DVD Maker.

2.     Add required Photos and Audios files and click Next.

3.     On the Ready to Burn screen, select Slide show.

4.     Check the length of the Slide show and the Music length duration.

5.     Check the box “Change Slide show length to match Music length” to sync both audio and video.

This will sync the Music Length with the Pictures.

Once you do this, save the Movie. Then try opening with Movie Maker or Media Player.

syed n say:

Hi,

1.       Is the icon for Local Area Connection missing from the Network and Sharing Center?

2.       Did you experience this issue earlier?

3.       Did you check the device manager?

First, you can check for the latest Windows Updates and install any latest drivers available. To do this, follow these steps:

1.       Click Start, All Programs and then Windows Update.

2.       Click Check for Updates on the left of the screen.

3.       If any updates are available, go ahead and install them.

Then, you also can check if the Network Adapter is disabled in the Device Manager. Do this as stated below:

1.       Open Device Manager by clicking the Start button, clicking Control Panel, clicking System and Maintenance, and then clicking Device Manager.‌
If you are prompted for an administrator password or confirmation, type the password or provide confirmation.

2.       Look for the Network Adapters and expand it. Now, right click on it and check if you have an option Enable.

3.       If yes, click Enable.

4.       Restart the computer after the installation and check.

At the same time, try checking the Device Manager for any missing drivers. If you find any exclamation signs, update the drivers. To do this, follow these steps:

1.       Open Device Manager by clicking the Start button, clicking Control Panel, clicking System and Maintenance, and then clicking Device Manager.‌
If you are prompted for an administrator password or confirmation, type the password or provide confirmation.

2.       Look for the Network Adapters and expand it. Now, right click on it and select Properties.

3.       Click the Driver tab and click Update Driver. Windows will now try to find the latest drivers.

4.       Restart the computer after the installation and check.

Also, when I check the HP website, I see that HP does not provide Network Drivers for Windows 7 for your Computer model. In this case, if the drivers are not available even from Windows Updates, I would suggest you to download and install Windows Vista 64 bit drivers to see if that helps in any way.
