So, before you enabled it, it was "not configured"!

About the NAS: do you have set user accounts on it? I would assume that you don't have any user accounts on it. Maybe everybody has access to it? If you create the exact same user account with same password on the NAS that you use on your Windows machine and enable password-protected sharing in Windows you should be able to access the file shares just fine.

a. user say:

If you are the only one using it then there is no reason to use a public share. A public share is for being able to access a share from numerous PCs without authentication (and you are not able to provide everyone with a user/password access to the NAS). That means if someone hacks your router (which they do by the millions) or intrude in your network/PC by other means they can access this public share (no matter what you set in Windows). If you use it from "outside" your network (as they mention "Cloud") you should definitely not use a Public share.

You should be able to move the files to the private share area or to expand the private share area to it. No? I don't know how their devices work.

Also note, that "enable insecure guest logons" is not the same as enabling the guest account in Windows as the IA seems to think. It is *not* for allowing unauthenticated access to your (usually disabled) guest account in your Windows system!

This setting tells the client (your PC) that it can try an insecure unauthenticated login at *another* server (your NAS). That's all. And, as I said, it should be "enabled" by default as "not configured" means "enabled".
