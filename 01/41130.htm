cmdrkeene say:

Try this.

  • Press +R to show the Run box, type cleanmgr (short for 'cleanup manager'), and click OK.

Also, technically it's all safe to delete.  By safe I mean, "will not cause any harm".  But that doesn't mean it's actually beneficial to delete.  Here's why:  your computer caches information it needs (makes a temporary copy).  When you use a cleaner, you delete that cache (the temp files) and it has to rebuilt them all again from scratch -- which is slow (that's why it tried to cache them so it didn't have to build them!), and causes your computer to get slower and slower every time you "clean" it.  Consider your temporary internet files.  Those speed up your Internet use (that's why they exist).  If you delete them, all that data has to be downloaded again, which slows down your internet access and your computer while it rebuilds.

Also consider that some of this data is for diagnostic purposes.  Such as the "problem reports" in the list.  If you delete them, then your computer will no longer be able to check for solutions to those problems, which means that if a solution comes available someday for a problem, you won't get the update.  Of course you can go check yourself for updates, but it's nice for the computer to be able to help you out by checking for new solutions and performance improvements.

So my recommendation is to only use the clean up tool if you are concerned about retrieving more space on your hard drive at all costs, giving up the benefits of the caches.

¡firedog say:

... the problem is only on the new mail app of Windows 10, which makes the app useless. 

On the contrary, the problem is most probably with the sender.

... sending email is from exchange server.

If a message is sent from Exchange Server (or Office Outlook) using their proprietary RTF format, the recipient will have to have special software to be able to decode any attachments. That software is included with Office Outlook (and Exchange Client) but no other mail client that I know of. Some webmail services are also able to decode that kind of attachment.

If you sign in to the account at mail.live.com, you can right-click on a message with an attachment that doesn't appear in the mail client and select View message source. Scroll down to the bottom of the text file, then scroll back up to the last blank line in the message. That's where you'll most probably see the start of a MIME part whose Content-Type is given as application/vnd.ms-tnef or similar. The important bit is TNEF, the encoding method used.

If you have the possibility, you should instruct the sender only to use RTF when sending to correspondents known to be using Office Outlook. You'll find an explanation and instructions in this article: How e-mail message formats affect Internet e-mail messages in Outlook
