a. user say:

triron;

I work with the Microsoft account team, and I acknowledge it is inconvenient to have to wait the 30 days, however there is no mechanism available to circumvent that time.

If it helps, let me explain why the 30 day window is important to all our users.  The information added to your account is very important to both ownership and recovery, and the 30 days is to ensure if a hacker should gain access to your account they could not go in and remove your information and add theirs without you as the rightful owner having the opportunity to cancel that action.  Should a hacker successfully replace your information, they would then own the account, and there would be nothing that we could do to help.

ricardo;

 

If you have one piece of information that shows as Confirmed (there is no confirm link after the name) you can use that to remove the others.  If all your information shows the confirm link, or what is confirmed is a PC that you no longer have access to (this would include a PC that was once "trusted", but has been reloaded since), your only option is to mark all as lost and wait the 30 days for them to roll off.

Here'a an article on how to diable automatic Defragmentation in Vista.  Perhaps these procedures will work if they differ from those you've been using: http://vistasupport.mvps.org/disable_automatic_defragmentation.htm.

Here's another article with a slightly different procedure.  If the above doesn't work, give this one a try (or you can do it anyway - it won't hurt to do it in both places): http://www.tech-recipes.com/rx/1598/automatic_defrag_disable_and_enable_vista/.

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

But WHY is disk defragmenter causing your system to freeze?  That's another issue.  Let's check your system files and hard drive for corruption:

Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator (you can skip this step if using the disk).

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.  Try to post any corrupted files here so we can see if they can be repaired with good copies from the installation disk (unless there are too many).

While in Command Prompt, type chkdsk /f /r and enter and let it run.  It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a potential cause.

If that doesn't work, then please post any repeating error messages from the Event Viewer concerning disk defragmenter (Start / Control Panel / Administrative Tools / Event Viewer – or typing eventvwr from the command prompt).  Here's how to use Event Viewer: http://www.petri.co.il/vista-event-viewer.htm.

If that doesn't work (or if SFC finds corruption it cannot repair) and you want to be able to defragment again (assuming one of the above procedures resolved the problem of it running automatically), we'll need to do a system repair/upgrade using the genuine Windows Vista Installation Disk (one you own or one you can borrow from ANYONE).  Here's the procedure: http://www.vistax64.com/tutorials/88236-repair-install-vista.html.  Although this will not affect your data, settings, or programs, you should still backup your data before starting just to be on the safe side. If the other procedures don't work, THIS one is almost certain to work.  You may have a lot of updates to re-install (including any service packs you had to remove).  If the version on the system came with SP1 or SP2 pre-installed and the disk is an earlier version, then you'll need to make a slipstream disk as follows: http://www.vistax64.com/tutorials/151606-vista-sp1-slipstream-installation-dvd.html.  

Hopefully one of these procedures will resolve your problem.  If not, post back and we'll try something else.  Post back in any case so we'll know how it turned out.

Good luck!

charlie klausen say:
Hi Adil,

Let’s give this a shot:

Hide A Nearby WiFi Network

You already know the name of the network you want to hide from the list of WiFi networks. Open Command Prompt with administrative rights. Enter the following command, and replace “WiFi network you want to remove” with the name of the network you want to remove from the list.

netsh wlan add filter permission=block ssid="WiFi network you want to remove" networktype=infrastructure

davem121 say:
Hi Adil

Note down the exact names of the WiFi networks you want to hide

Click your Start Button, then just type cmd

From the resulting list, right click Command Prompt and choose 'Run as Administrator'

For each network you want to hide, run this command and hit Enter, (each time change NetworkName to the appropriate network name)

netsh wlan add filter permission=block ssid="NetworkName" networktype=infrastructure

When that completes, close Command Prompt

After a restart, you should no longer see those networks . . .
