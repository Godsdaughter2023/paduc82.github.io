donovan t. say:

I was able to overcome the Windows Backup error on Windows 10 by performing the following commands:

  1. Disable Hibernate and remove hyberfil.sys file: powercfg.exe /h off
  2. Boot Disk Check (scheduled during next reboot): chkdsk C: /F/R
  3. System File Check (should report back "did not find any integrity violations"): SFC /scannow
  4. Windows Component Store Integrity Check (should report back "No component store corruption detected"): dism /Online /Cleanup-Image /ScanHealth
  5. Repair Windows Component Store (should report back "restore operation completed successfully"): dism /Online /Cleanup-Image /RestoreHealth
  6. Validate System Files again if needed: SFC /scannow
  7. Enable Hibernate: powercfg.exe /h on

I disabled/enabled hibernation based on previous comments above, but I don't know if it is necessary.

The most important commands are #2 through #5 as they will remedy most Windows 8, 10, 2012, 2016 operating system integrity issues. I perform these commands once a year, or when a problem arises with the system, to ensuring there are no file integrity or OS issues present.
