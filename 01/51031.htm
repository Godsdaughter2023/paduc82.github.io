bossdweeb say:

- hiberfile.sys is created if you use Hibernation, and will be deleted if you disable Hibernation.

I use an SSD  for my Windows drive,  which does not use Hibernation,  and don't recall if there

is an option to adjust the maximum size of the  hiberfile.sys  file via the Hibernation options.

- The Page File will be cleared if you disable it and re-boot. You can then use the 'custom' size option

and set both Minimum and Maximum at 2048, which will keep the PF from growing too large

over time. This small size should only be used if you have 8GB or more system RAM.

- If you have not already done so, you may also be able to save some HDD space by deleting

System Restore points and re-setting the amount of HDD space allocated to System Restore.

After deleting 'old' restore points do make a new one manually.

- Regardless of if you do regular backups, System Restore is a very useful tool, particularly if

you are inclined to delve into Windows files and settings, so it is not advisable to disable it.

- However clearing the  files above will not improve performance if that is your goal, and trying to

make more space on a HDD by deleting Windows files is not the safest option, and is really

only going to be a stop gap measure.

9GB of data on modern machines is not a lot. A larger, or second HDD, either internal or external, 

is most likely going to be required sooner than later.

Yes, they should be kept on an internal disk.
The Page File can be on a separate internal HDD from the OS , but this should not be necessary
with a  platter HDD (It's sometimes done when the OS is on an SSD to limit the number of writes ).
- I'm not sure if this can be done with the hiberfile.sys  file though.
- Once your data has been moved to the new internal HDD there should be enough free space on the
OS HDD to not have to deal with these system files again.

"... 1 gigabyte of capacity"

I assume this was a typo.    1 Terabyte (TB), or larger are becoming common, but may not be

necessary, depending on the data you need to store.  500GB HDD's may suit your needs,

but 1TB is cost effective per GB.

- It is best practice for important data to be backed up from your internal HDD to an external HDD,
so having one each of the same capacity is a good idea .
