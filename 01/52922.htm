https://blogs.msdn.microsoft.com/oldnewthing/20071121-00/?p=24433

In my particular case, I had a batch file in DOS-866 encoding with ~20 doskey commands.
The path to this batch file ("c:\Program Files\somefolder\somefile.bat" ) was in the "AutoRun" REG_SZ parameter of 

"HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Command Processor" key.

the solution was to export AutoRun parameter to .reg file, remove it in RegEdit, and to start Skype Installer again.
Then Skype update was quick and successful.

After import of exported before .reg file to Registry, updated Skype starts fine, so it seems like only Skype Installer has problems with doskey in CMD's AutoRun.
