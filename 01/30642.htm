This is actually the "pinch-zoom" feature which has been optimized for touch screen devices. If, like me, you find this feature problematic on an older laptop, here is the solution.

Click the Windows Icon in the lower left->Settings->Devices->Mouse & touchpad.

Click 'Settings'

You should now see a menu for your specific touchpad device. Scroll through the settings and look for "Pinch Zoom" and disable it. For Synaptics ClickPad you will find "Pinch Zoom" listed under 'Scrolling.' After clicking Pinch Zoom, uncheck the box 'Enable Pinch Zoom'.

Is 7% fragmented an acceptable level of hard drive - Microsoft Community

b52gunr say:

Is 7% fragmented an acceptable level of hard drive fragmentation?

How can one defrag to zero percent or  close to it?

a. user say:

7% is beautiful, even excessive.

In the first place, defragmenting to zero is neither desirable nor does it make sense, because your hard drive will start fragmenting immediately after you're done.

In the second place, while defragmenting might have been a good idea about 10 years ago, hard drive technology has progressed mightily since then and fragmentation is no longer the issue it used to be.

Finally, the amount of time you will spend trying to defrag your hard drive to some extremely low level is a hundred thousand times longer than the time you will save by doing so.

A good thing to do is to conduct a web search for: is it necessary to defragment a hard drive

Quite illuminating.

frankymacky say:

Backup cloning from 512 byte/sector HDD to 4KB/sector HDD

I'm trying to educate myself on cloning i.e. bootable sector-by-sector replication, not just creating an image file of the used disk space.  So far, I've bought a high performance HDD of the wrong size, trying to make sure that I replace it with an HDD with which my first cloning attempt will go without a hitch.

My laptop contains a 640GB Toshiba MK6465GSX HDD (http://storage.toshiba.eu/cms/en/hdd/multimedia/product_detail.jsp?productid=341).  I am considering getting 640GB WD Scorpio Blue HDD for the cloning.

One of the constraints I want to observe is that I should be able to pick it up in person from one of the retailers in my city.  However, they invariably carry the WD6400BPVT variant, which uses 4KB sectors ("Advance Format").  Nothing comes up in a search for "MKxx65GSX MKxx76GSX advanced-format" (without quotes), so I assume it has 512 byte sectors.

Is there a problem with cloning from a 512 bytes/sector HDD to an 4KB/sector HDD?
thomas ferguson say:

First, I will answer the question which you did not ask:

You can use the disk you have as a source and use the "advanced format" drive as destination to produce a bootable HD for your laptop. It does involve creating an image and using that to write the new drive. It is a type of logical file copy.

The answer to the question that you did ask:

By definition and in practice, you cannot do a bitwise copy and get a copy beween dissimilar disk types that is one-to-one and onto. Which is to say, the copy will not be a sector-by-sector match. The copy will function but there will be performance issues, which is to say that read and write speeds will suffer. And you cannot do a sector to sector copy since the sector sizes do not match.

To do a bitwise, sector-by-sector copy, you need drives that can set up with identical sectors.

I would be pleased to be proven incorrect on this. Please provide appropriate reference(s).

Tom Ferguson

Hello Shruthi_19,

connect to Skype for Business PowerShell, run:

Get-CsOnlineUser -Identity YourSipAddress |fl *conferen*

Note the ConferencingPolicy name, run

get-CsConferencingPolicy -Identity result from the last output.

If the EnableAppDesktopSharing value is None, you are not allowed to share desktop.
