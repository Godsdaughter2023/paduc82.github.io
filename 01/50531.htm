The IRQL_NOT_LESS_OR_EQUAL bug check indicates that Microsoft Windows or a kernel-mode driver accessed paged memory at DISPATCH_LEVEL or above. This bug check is issued if paged memory (or invalid memory) is accessed when the IRQL is too high.
