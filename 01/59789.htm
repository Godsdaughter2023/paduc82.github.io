mikhail grunko say:

Unfortunately I have not found a solution, but a good workaround:

 

1. Install a cloud service

2. Run the command prompt as administrator

3. Enter the command:

mklink /D "C:\Users\[user name]\Desktop\Local_Disk_D_Sym_Link" "D:\[cloud service folder]"

Remark. The first parameter is the name of a symbolic link (i.e. it can be chosen any name) and also the folder where the simlink will be created (in the example above a symbolic link with the name "Local_Disk_D_Sym_Link" will be created on your desktop; it will look like a normal shortcut). The second parameter is the folder to which your symbolic link will be placed.

4. Copy the created symbolic link to your cloud service folder

5. Syncing

 

Done!

 

A small problem:

Unfortunately symbolic links do not work as a file: after you made p. 4, your symbolic link is synchronized (as expected), but if you change something what it refers to, the changes will not be caught by your cloud service.

But if you edit something directly in your cloud service folder, the two changes will be synchronized: both cloud service folder and your symbolic link.
