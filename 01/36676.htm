tryx3 say:

1 Do check that Documents & settings is actually deleted rather than just hidden from view. Windows explorer, Organise, Folder & search options, View tab. Enable Show hidden folders and clear the Hide protected operating system files [a couple of lines below the hidden folders item in the list]. 

2 Command prompt - right-click on the command prompt entry in the All programs, Accessories list & select Run as Administrator.

3 System restore - This procedure returns system files to the state they were in at the time the restore point was created.  But I have never seen a report of anyone deleting the Documents & settings folder before so I don't know that restore would correct this particular problem.  This comment also applies to running sfc / scannow.

4 New admin account - When I created a new Admin account, all the existing programs were accessible and that's the behaviour I would expect in all cases.  You won't see any of the desktop shortcuts as they belong to each account but the programs should be in the Start button, All programs lists.  You can get access permission in the Admin account to access your existing user account & copy the desktop shortcuts from there if you wish but you should do a full backup first or your next user account backup routine will regard every single user data file as having been changed.  After altering permissions, you can then run [I think you need a command prompt with Run as Administrator again] attrib -a /s from your My documents folder [check the syntax using attrib  /?] to reset all user data file archive bits & hence stop them from all being backed up again.

5 Reinstaling the OS would solve the missing Docuemnts & settings folder.  I have seen threads in this forum about putting data files in a different partition but you'd have to search for that subject as I've never paid much attention to it.  Before doing this [and this is just speculation], try running then saving a file from your oldest pre-Win7 application to see if Win7 itself can recreate the Documents & settings folder when an old application calls for a configuration file or a save procedure.  If this works it would save you a lot of aggravation.  Alternatively, since the Documents & settings folder should only be called upon when using an old pre-Win7 application, if your oldest application can run & save correctly without it then don't bother doing any re-installation at all. 

Best of luck with all this - and once again just to avoid misunderstandings, I have never seen a report of the Documents & settings folder having been deleted so I can only give sensible advice without the benefit of direct experience.
