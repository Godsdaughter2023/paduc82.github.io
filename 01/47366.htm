tomás crespo say:

After a lot of researching I've found that the problem is in the permissions of %TEMP% folder.

C:\Users\username\AppData\Local\Temp

After giving Full Control to username all is working as in a normal machine, but I'm not sure what are the default permissions for that folder. I will check in a clean installation.

Edit:  After checking the permisions of %TEMP% folder in a clean installation I've watched that the permision are:

  • SYSTEM: Full control
  • CURRENTUSER: Full control
  • Admins: Full control

But the important thing is that these permissions are not stablished in the %TEMP% folder, they are inherit from the main user folder, from c:\users\username

So now, I set the permission to inherit and all is working in my three affected computers.

After auditing this folder in one computer I've realised that the software Adobe Acrobat 11 is who is changing the permissions of this folder, causing the "UAC not asking prompt" problem. I'm not totally sure about this, because I've not performed a deep investigation, it could be %TEMP% permissions were modified by other cause, but it looks was Acrobat.
