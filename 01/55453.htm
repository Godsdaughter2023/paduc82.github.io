My Workaround for Problems with Windows 10 Virtual Machine - Microsoft Community

_carl say:

My Workaround for Problems with Windows 10 Virtual Machine

  In the process of setting up a Windows 10 VM on Hyper-V installed on a Windows Server 2012 Standard Domain Controller, I’ve experienced numerous issues with connectivity and updates.   In the process of setting up a Windows 10 VM on Hyper-V installed on a Windows Server 2012 Standard Domain Controller, I’ve experienced numerous issues with connectivity and updates. For connectivity, the computer named “Win10” is clearly visible in DHCP. However, even from the domain controller itself, I can only ping it by IP address. Pinging by computer name returns “not found”.  

The Win 10 VM will not download updates from either my local WSUS server or Windows Update site. It freezes at any given percentage point on every attempt to download updates. Have gone through all the mess of fixing the update client on the VM, to include wiping the softwareupdates folder, resetting authorization, re-registering DLLs, and everything else I could find to “fix” the issue in these forums. None of them work.

For remoting in via RDP through the RD Gateway, when using the machine name sometimes I can, sometimes I can’t with a “computer not found. May not be on or a part of the network” error. However, if I use the IP address in place of the machine name, it’s no issue, occasionally requiring a 2nd attempt before it will successfully connect. So here’s what I did to get things finally working with a tolerable (for the moment) level of acceptance.

First, I created the VM without a network adapter and installed Windows 10 Pro without a license/activation key from the installation ISO attached as a virtual CD drive. 

Next, I created a Virtual HD, attached it to a fully functional Windows 7 VM, then went to the windows update catalog and downloaded KB3201845 to the Virtual HD. Then disconnected the Virtual HD from the Windows 7 VM.

Then I attached the Virtual HD to the Win10 VM and copied the KB3201845 download to the desktop before I disconnected the V-HD from the Win10 VM.

I then installed KB3201845 on the Windows 10 machine, already set up to download from my WSUS once I added a NIC (later). It took about 15 minutes to install. After installation and reboot, everything checked out so I shut down the Win10 VM, added a virtual NIC and restarted the VM.

Now, the Win10 VM is correctly reporting the WSUS and downloading/installing updates with no issues thus far. I can also ping it by name from anywhere on the network, and RDP through the RD Gateway has worked flawlessly thus far.

I will leave this sit for a day or so. If it continues to function problem free, then I’ll export this Win10 VM to its final destination and activate it with the license key.
