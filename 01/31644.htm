j w stuart say:

If XP was the first version of Windows you installed, on the first partition (here is the key, the first partition when viewed in XP's 'Disk Management') it is assigned the drive letter C:, the other two partitions can be assigned any drive letter (except A & B). On your computer It would appear as D: and E:.

 

Now if you where to install a second version of XP on the second of the 3 partitions, when booting to that version of XP everything would be seen as and installed to D:\Programs\appname and for some poorly written software D:\ whatever would fail to either install of run.

 

Back to Windows 7.

Windows 7 does not care which partition you install in on, for example the 10th partition on a 2GB hard drive, it will always boot to what appears to be the C: partition (which most of us assume is the first partition on the hard drive). In addition it will assign a new drive letter to the partition you installed XP on, so what was from XP's point of view the C: partition may now be an entirely different drive letter.  Remember that partition order (1st, 2nd and 3rd) do not automatically equate to C:, D: and E:.

While it is true that you will need at least one primary partition, additional Windows 7 installations can make use of logical partitions thereby giving you the potential for a nearly unlimited number of bootable versions of Windows.

The option in the Fonts control panel to "hide" fonts was added in Windows 7 and has effect in certain "common controls" built into Windows that applications can use.
