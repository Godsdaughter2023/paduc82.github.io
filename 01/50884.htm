Day-5 and this is what I’ve discovered:

The initial Release of Windows 10 Pro created by the MediaCreationToolx64 application is not entirely stable for all platforms, such as Alienware systems like mine.  Dell has confirmed this through a support call earlier yesterday (Aug-5).

This is really disappointing because I have been a contract software developer at Microsoft for 22 years, I know how software should install having written installations many times before, and Windows 10 has one of the worst user experiences; it reminds me of a time before DVDs and CDs when we had to use floppy drives.

For example, if using Windows Update, we have to download the entire update each time the installation fails.  The download time can be eliminated by using an ISO image... although to get that image you still have to download it once.  I have checked the MSDN subscription source and it looks like we can bypass the deficient MediaCreationToolx64 app and get the Windows 10 Enterprise ISO directly if the subscriber has rights.  There are small file differences between the two versions, mainly two involving the Install and Boot WIM, and a handful of other minor differences like an extra API stubout that is very tiny in size.  This might be worth investigating when time allows.

The other problem Microsoft must face are the misleading cryptic error codes that have relatively little useful information relating to the failure.  What value does WHEA_UNCORRECTABLE_ERROR or BAD_POOL_HEADER have on the BSOD when there is no supporting documentation?

Five days I spent trying to debug why my 18 month old hot rod won’t boot.  I have ran all the extended diagnostics (both Dell apps and built in BIOS) and the box has passed with flying colors.  Reading, researching, trying all sorts of ways to get the box to go, staying connected to the Internet vs. removing that connection; it made no difference.  Every device driver was reviewed and the latest sources were applied.  Nope, nada, no joy; the same errors in the $Windows.~WS\Sources\Panther\*logs.
