wunders say: You need to identify which service is taking all your CPU and disable it.  Try the following procedure...

  1. Download and run "Process Explorer" from Microsoft.  This is basically a task manager on Steroids:
    Process Explorer:  < http://technet.microsoft.com/en-us/sysinternals/bb896653.aspx >
  2. When you run Process Explorer, you will see the instance of "svchost" that is using up all of your CPU.  Hover your cursor over the text "Svchost.exe".  A Pop-up window will appear that will give you a list of the Services that are being executed under this instance of svchost.  Write these services down on a sheet of paper.
  3. Start up the Services window  (Start -> Run -> "services.msc").
  4. For each of the services you wrote down in step 2, find that service in the list, double-click on it, and then click "Stop" or "Pause".  If your CPU goes down to normal, then you have found the service giving you the problem.  If not, then restart or resume that service and move on to the next one.
  5. Once you find the service responsible, you will need to determine how to handle it.  If it is a 3rd party application, then you may need to uninstall / reinstall that application.
I have had prior experience that "Windows Management Instrumentation" often will peg my machine at 100% CPU.  Usually this is only for ~10-15 minutes or so and then it goes back to normal.

Let's try to create a new user profile using the Command Prompt.

  1. Right-click Start button and choose Command Prompt (Admin).
  2. Click Yes in the dialog box.
  3. On your Command Prompt, type the following commands:

    Net user Test /add
    Net localgroup administrators Test /add

  4. After that, exit Command Prompt.

Once you are done, click Start button and click your current user profile, then select your Test user profile.
