¡firedog say:

While there were no errors, the files were however corrupted. The message headers appear to be correct, but the body of each message contains exactly the same text within a file.


This is not evidence of corruption in the files concerned. It is evidence that the message database hasn't yet caught up with all the newly-imported messages.

The message list is taken from the message index. When you select a message in the list, the program will try to retrieve the message body from the underlying file. If the index is not yet complete, this retrieval may have unexpected results.

When you suddenly dump a load of messages into WLMail, e.g. by importing a whole store folder, an awful lot of stuff has to happen. Each message is an individual file: your anti-virus will want to intercept each file as it's written to disk, scan it and then return it to WLMail; the Windows Search indexer will want to read the file to record it in its database and possibly write properties to the file, and the message database will try to extract the message headers to record them along with other information about the file, including its location.

All this is bound to take time, so it's as well to create as peaceful an environment as possible for it to take place in. This might include the following measures:

  • Set WLMail to Work offline. This will prevent any interference from scheduled messaging operations.
  • Disconnect from the network and disable any real-time file scanning by security or privacy software.
  • Stop the indexing service to prevent interference from the Search scanners.
  • Leave the program - and preferably the computer - alone while WLMail digests all its new stuff, overnight for example. Keep an eye on progress by using Task Manager to see how busy wlmail.exe is.


OK, so it would have been good to know all that before you did your import. You may find that the database will catch up if you only give it peace and quiet to do so. When wlmail.exe has slowed down to tickover (perhaps 1% or 2% of CPU cycles), close down the program and leave it to do its post-close housekeeping, which could take a few minutes. Again, watch wlmail.exe in Task Manager. Don't shut down Windows or re-launch the program until wlmail.exe is no longer visible in Task Manager.
