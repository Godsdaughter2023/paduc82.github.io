lemp say:
I agree with ZigZag that 200 MB is too small to worry much about.

I suspect that the zip extraction process is creating a temporary file or files.  By default, CCleaner only removes temporary files that have not been accessed within 48 (or 24; the Piriform website gives both values) hours (you can change this in CCleaner's Advanced Settings). 

Try the following:

Extract the contents of a zip file
Open Windows Explorer
Go to the root of the C drive
Click in the Search box in the upper right
Add a filter for Date modified:  and when the calendar appears select today's date
Click in the Search box again and after the date, type a space followed by
     size:>150MB AND <250MB
You can adjust the values if you want, but don't try to get the range too narrow.  Don't forget the colon after size, type AND in all caps, and no spaces between the number and "MB".

For example, the filter in the search box might read
     datemodified:‎11/‎19/‎2013 size:>150MB AND <250MB

That search should find your newly-created file, although if it's a hidden and/or system file, you may have to go into Tools > Folder Options and make such files visible.  The problem is that there are many possible locations for temporary files, thus you will have to make the changes effective for all folders -- and then reset all folders back to normal.
