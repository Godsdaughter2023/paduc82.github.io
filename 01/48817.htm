a. user say:

Hi Adnan,

Try this solution:

1. Open Registry Editor(WIN + R, type regedit, hit Enter)
2. Browse to key HKEY_Local Machine > Software > Microsoft > Windows NT > CurrentVersion
3. Change EditionID to Home (double click EditionID, change value, click OK). In your case it should at the moment show Pro
4. Change ProductName to Windows 10 Home. In your case it should at the moment show Windows 10 Professional
5. Browse to key HKEY_Local Machine > Software > Wow6432Node > Microsoft > Windows NT > CurrentVersion, change the same two values as in steps 3 and 4 above
6. Close the Registry Editor
7. Insert the Windows 10 Homeinstallation media, do an in-place upgrade by launching the setup from desktop (not booting with Windows 10 Home install media)
8. When asked, enter your validWindows 10 Home product key and select what to keep(settings, personal files and apps, only personal files or nothing)
9. Windows 10 Home in-place upgrade will be done now.

This should work. Tell me if it does.

wunders say:

Your best chance is to restore the whole disk from an image-style backup, then log in as your old user and then back up the certificate.  Every time you install Windows, it picks random Secure ID (SID) numbers for the users which is one of the factors that contribute to the encryption of certificates. IMHO, one major drawback of Windows Encrypting File System (EFS) is that it automatically generates encryption keys and then manages them for you without even telling you what it is doing -- most people don't even know encryption certificates are involved.  It is critical to back up these certificates to cover exactly the situation that you are in now.  Microsoft does provide the following article to describe the importance of backing up certificates, but not many people know about it:

"Best practices for the Encrypting File System"
  < http://support.microsoft.com/kb/223316 >

Theoretically, you might be able to recover if you have your old registry files (contents of C:\Windows\System32\Config folder) plus your old user profile (C:\documents and settings\<username>\ntuser.dat) plus your old user password; but I've never seen this procedure published in a forum other than to recommend a pay call to Microsoft -- with no feedback to know if it succeeded or not.

Sorry,
   JW
