shilohbob say:

How much space File History uses is dependent on Four things:

1. The No. & type of files/folders that you choose to utilise File History; and

2. The frequency of saving copies; hourly or portions thereof / daily.;

3. How long you keep saved version: this can be set from 1 mth. to forever or until space is needed;

4. You can also set the size of the online cache; % of disk space.

Mine is set for: Daily / 10% / 6 mths (now changed to 2mths) ; and is using approx. 65Gb, to an external HDD.

This does include a large Music collection, which I considering removing.

Check out info at: Control Panel > File History > Advanced Settings'

So the size, as you can see, is not constant.

To determine which files could not be repaired by the System File Checker tool, follow these steps:

 

Method 1:

 NOTE: When SFC runs, it logs it's actions to the C:\WINDOWS\LOGS\CBS\CBS.LOG. You can find the specific SFC entries by searching for the [SR] tags in the log.

 For how, to see only the SFC scan details in the CBS.LOG:

5.      You can safely delete the sfcdetails.txt file afterwards if you like.

Hello, 

I had a similar problem. Thankfully, I was able to find the solution. It is a specific driver. If you are having the same problems please follow these steps: 

1) Go to Device Manager (Windows Key+X, select it) 
2) Under the "Mice and other pointing devices" list find "USB Touchscreen controller", or the driver that's not your mouse/touchpad. 
3) Right click to select "Properties", click the "Driver" tab, and click "Roll back". 
4) After confirming, this will remove the driver from that list but bring up a bunch of new ones under "Human Interface Devices", where the touchscreen stuff logically should live. 
5) Make sure that all the other drivers are updated EXCEPT "USB input device". This is the one that moves it all to the mouse drivers. 

After a restart the driver didn't automatically reinstall like they usually do. If it does, run the "Show or hide updates troubleshooter" found here.
