In theory it would suffice to disable the Windows Automatic Update service. In practice this is not enough because the service can get re-enabled by other mechanisms. You would need to create a scheduled task that invokes once every hour a script to keep the update service disabled. Creating such a task and the script itself requires good skills at a maintenance level.

frederik long say:

I'm a programmer by trade so I'm probably capable of setting up this task and the script. Not sure where I'd get started though. Any pointers?

You are a rare specimen in this forum . . .

  1. Save the script you see below in a suitable folder as DisableUpdateService.vbs.
  2. Create a scheduled task with these parameters:
    - Run it once every hour.
    - Run it under the System account. This is essential!
    - Use this command line: cscript.exe  c:\Tools\DisableUpdateService.vbs
  3. Open an elevated Command Prompt.
  4. Run this command: DisableUpdateService.vbs xxx. It should report the current state of the update service, and disable it.
  5. Manually enable the Windows Update service.
  6. Right-click the scheduled task, then left-click it to run it.
  7. Check the Windows Update service. It should be disabled.
  8. Enjoy the peace and quiet.

To keep your machine in good nick you should disable the scheduled task once every few months.

Addendum: I just found out that disabling the Windows Update Service also prevents the virus definitions from being updated. You need to judge for yourself if you're prepared to run this risk.

'------------------------------------------
'Stop & disable the Windows Update service.
'16.2.2018 FNL
'------------------------------------------
Set oWMIService = GetObject("winmgmts:{impersonationLevel=impersonate}!\\.\root\cimv2")
Set oService    = oWMIService.Get("Win32_Service.Name='wuauserv'")
if wscript.arguments.count > 0 then wscript.echo oService.StartMode

If lCase(oService.StartMode) <> "disabled" Then 
    oService.StopService()
    oService.ChangeStartMode("Disabled")
End If

bossdweeb say:

Short story... If you can't match what you have you're better off sticking with the one  2GB stick.

Even if you match the specifications there is no guarantee of stability. No RAM mfg will

guarantee stability unless you use a factory matched kit ( in practice most machines do manage

with using sticks that were not factory matched as long as the specs match, but not always).

Your motherboard is only compatible with DDR2, and if it were compatible with DDR3 as well,
(some motherboards are) you still  cannot mix and match different RAM  specifications.

Particularly the data rate (DDR)

- In addition to the data rate, the speed, latency and voltage also have to be the same.

I don't think running mis-matched RAM is will  harm your PC, but I'm not positive.

Using two different data rates may harm the memory controller.
It may not boot into Windows at all, and very likely will BSOD if it does.
At the very least it will  be unstable.

For more details visit the forums for the mfg of your RAM.
