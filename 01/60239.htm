marcelo meireles say:

My Win is 8.1 and the POS Service keeps gnawing 5-9% of my CPU.

First time I looked into it I was able to solve (for some hours) restarting a service, but I can't remember which one and It seems I can't duplicate this solution anymore. Searching further I've found a solution that's working for me whenever I detect the issue:

SOLUTION

In the prompt, as administrator, issue the command

winmgmt.exe /resetrepository

You probably get this response:

WMI repository reset failed
Error code:     0x8007041B
Facility:       Win32
Description:    A stop control has been sent to a service that other running ser
vices are dependent on.

Then you just reissue the same command until you get

WMI repository has been reset

WMI Provider Host goes to 0% CPU usage ;^)

---

I have created a batch file to automate the process with a shortcut to it:

Reset_WMI.bat

@echo off
winmgmt.exe /resetrepository
ping 8.8.8.8
winmgmt.exe /resetrepository

Obs: remember to configure the properties of the batch file to run as administrator. The arbitrary ping line is just to simulate a pause of about 5 sec so the first reset command successfully ends the service before the final command kicks in and effectively resets the service. That way you can fire and forget, so to speak.

Marcelo Meireles

You can do it like so:

  1. Remove the user's name from the access list.
  2. Include only administrators in the access list.
  3. Give the user a shortcut that runs this command:
    net use Q: \\MyPC\MyShare  PeterAdmin  *

"MyShare" is a share that points to the protected folder.

"PeterAdmin" is the name of an admin account for which the user knows the password.

To prevent the user from modifying the ACL you must edit the ACL in such a way that PeterAdmin cannot change the permissions. You must probably prevent him from taking ownership too.
