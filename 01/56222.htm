devzoo say:

Eureka! I've now got Windows 10 working. The solution turned out to be relatively simple--no boot disk or ISO image required.

I should start by saying that the one non-standard thing about my laptop is that I fiddled with the partitions a tiny bit. (I used EaseUS Partition Master to create a separate partition for my personal data.) This may have been the root cause of the 80070003 error.

The solution that worked for me, which I found on another thread in this forum, was to

Of course, before trying any solution like this, you should definitely backup all your data.

The hardest part was just getting into the Advanced Startup Options. In Windows 7 and earlier versions, it was super easy to get to the startup options. You just rebooted the computer and tapped the F8 key, and a menu would pop up. Turns out this is no longer easy to do in Windows 8.1, at least not on my Lenovo Edge 15. This article explains how to get into the Advanced Startup Options:

http://pcsupport.about.com/od/windows-8/a/open-advanced-startup-options-windows-8.htm

From there, I started the command prompt, then I did the following steps in Cifad's post here:

(These steps assume EFCI boot mode as opposed to legacy boot mode.)

diskpart
select disk 0
list volume

assign letter=Z:
exit
cd /d Z:\EFI\Microsoft\Boot\
attrib BCD -h -r -s
ren BCD BCD.old
bootrec /rebuildbcd

What's this all doing?

You use diskpart to find the secret volume where Windows stores the BCD and assign a drive letter to it. Then you can unhide the BCD file and rename it (to create a backup). Finally, you use bootrec to rebuild the BCD.

Then exit and reboot into Windows 8.1 normally. Go back into Windows Update. This time the upgrade to Windows 10 will work, or at least it did for me.
