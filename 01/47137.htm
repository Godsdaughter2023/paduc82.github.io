miqrogroove say:

I finally found the solution that works for me.

The setting to disable wake timers through a power settings profile is defective and allows wake timers to work when they should not.

The workaround is to manually remove all wake timers.

  1. Open the Task Scheduler.  You can find this in Settings by searching in the upper right corner for "schedule tasks".
  2. Expand the Task Scheduler Library folder in the left column.
  3. Expand the Microsoft folder.
  4. Expand the Windows folder.
  5. Click into every subfolder and every item in every subfolder and click on the Conditions tab for each task.
  6. On the conditions tab, if the "Wake the computer to run this task" option is checked, then this task is causing your computer to turn on automatically.  Make sure this option is unchecked for all tasks.

One task in particular that I know was active could be found under Task Scheduler Library > Microsoft > Windows > Media Center > mcupdate_scheduled.  The irony is that task does nothing but throw an error message every day at 7:25 AM: "The system cannot find the file specified. (0x80070002)"

For additional diagnostics, try the "powercfg /waketimers" command at an elevated command prompt.  You can start an elevated command prompt by pressing the windows key + X.
