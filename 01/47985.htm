The mail app is capable of reading .eml files like the ones Windows Live Mail produces. You should also be able to read them in File Explorer with the right layout in Details view. Unless you have Windows Live Mail installed, though, you won't be able to do much other than read them.

Your best bet therefore would be to re-install Windows Live Mail. You can download an archive copy of the full installer from the wayback machine using the links I've given below. Before doing this, though, you'd better remove your exported folders from the Backup folder, which should normally only contain a single folder called New containing a handful of log files, a copy of the message database index file Mail.MSMessageStore and a file called Mail.pat. Any other files and folders should be moved somewhere else, e.g. into a new folder in your Documents folder if you don't have an external drive handy for backups.

The store folder (in your case the one called C:\Users\Bill\AppData\Local\Microsoft\Windows Live Mail) is hidden by default for a very good reason: the files and folders it contains are required for the normal functioning of the program, and any direct manipulation of them - inserting new files or folders, deleting or moving files or folders - will interfere with that normal functioning, even to the point of stopping the program from working altogether. Any file operations should be conducted within the program itself.
