cmdrkeene say:

Your live mail is stored on your email server.  If your computer burns in a fire, your emails are still on the email Live server ready to use and download to any email client.  However if you want to backup the locally-stored emails (which may not be them all), it depends on what email program you're using.  Since Windows 7 has no email program built-in, let me first ask which email program you're using.

Anyway, to answer your Backup question:  yes, only files that are new or changed since the last backup get copied again.  If there's already an up-to-date copy because nothing's changed, it doesn't make a double, since that would just waste space on the backup.

Hi Brook,

This has been resolved now I spoke to 365 Support through our admin console.

After looking through the logs turns out “The credential is blocked “ was shown, this was fixed by running an Azure AD Powershell Script.

Set-ExecutionPolicy unrestricted

$Cred = Get-Credential

$Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri Import-PSSession $Session

Run these commands in the Azure Active Directory PowerShell Module:

Connect-MSOLService –Credential $cred

After connecting to MSol service please run the below cmdlts:

Set-MsolUser -UserPrincipalName *** Email address is removed for privacy *** -BlockCredential $true

Wait 1 minute and then re-enable the account again
Set-MsolUser -UserPrincipalName *** Email address is removed for privacy *** -BlockCredential $false

You need to make sure that the AzureAD Powershell is logged into by a 365 admin with full Admin permissions to 365.
