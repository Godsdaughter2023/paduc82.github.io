Error 80169cf9 occurs when you try to redeem an Xbox Live Gold code   ( support.xbox.com/.../error-code-lookup )

Problem:

You see the following error code and message when you try to redeem an Xbox Live Gold membership prepaid code or token:

80169cf9

The payment option can't be used for this purchase. Please select a different payment option or add a new one.

This error message occurs when you have an active membership but auto renewal is turned off, and the payment instrument that was used to purchase that membership has been removed from your account or is no longer associated with your membership.

Solution:

Because the payment instrument has been removed from your current membership, you will not be able to add a prepaid code or token for a membership while your current membership is still active. You must wait until your current membership has ended to redeem the token. Membership tokens do not expire. To find out when your membership will end, sign in to your Xbox account and tap or click Subscriptions to view the end dates for your current memberships and subscriptions.

thank you for the proposals. It turned out that a much simpler thing was working. Since it may help others, I'll describe the situation again:

And in fact - this time, chkdsk was able to repair the disk and made it available again immediately. I disconnected the USB, connected it to my regular computer, and everything's working again now.

Summary for me: Something is broken in the 1803 chkdsk, which is very bad news.

In audit mode, Windows Setup will implicitly enable the built-in Administrator account as the last action in the auditSystem configuration pass if the built-in Administrator is not already enabled. The first action in the auditUser configuration pass is to disable the built-in Administrator account. This enables you to run programs and applications as an Administrator. When you complete your customizations in audit mode and log out, the built-in Administrator account will be disabled. Unless you want to explicitly leave the built-in Administrator account enabled, there’s no need to re-enable the built-in Administrator account in audit mode.

 

Refer to these steps to enable/disable the built-in administrator account.

a. Press “Windows Key + Q” button from the keyboard.

b. Type “Command Prompt” without quotes in the search box.

c. On the left pane, right click on the “Command Prompt” option and select “Run as Administrator”.

d. Enter the following command to enable the hidden administrator account: net user administrator /active:yes

e. If you ever want to disable the account follow the same instructions, but run the following command: net user administrator /active:no
