jmh3143 say:
"It's not a true crash, in the sense that the Blue Screen was initiated only because the combination of video driver and video hardware was being unresponsive, and not because of any synchronous processing exception".

Since Vista, the "Timeout Detection and Recovery" (TDR) components of the OS video subsystem have been capable of doing some truly impressive things to try to recover from issues which would have caused earlier OS's like XP to crash.

As a last resort, the TDR subsystem sends the video driver a "please restart yourself now!" command and waits a few seconds.
If there's no response, the OS concludes that the video driver/hardware combo has truly collapsed in a heap, and it fires off that stop 0x116 BSOD.

If playing with video driver versions hasn't helped, make sure the box is not overheating.
 Try removing a side panel and aiming a big mains fan straight at the motherboard and GPU.
 Run it like that for a few hours or days - long enough to ascertain whether cooler temperatures make a difference.

If so, it might be as simple as dust buildup and subsequently inadequate cooling.

I would download cpu-z and gpu-z (both free) and keep an eye on the video temps

frederik long say: This is the result of the UAC (User Account Control) feature under Windows 8. It prevents programs from making changes to your installation behind your back. Here are your options:
  • Use Windows 8 compatible programs.
  • Tweak the incompatible programs so that they no longer modify files in system folders.
  • Turn off UAC (not recommended).
By the way - running a machine with a single admin account is a very risky thing. Just one little glitch would get you into diabolical trouble. You should have two fully tested and documented admin accounts.

    brian-- say: Hi eager to learn,


    I have a few things for you to try:

    First, see what color scheme you have set by:

    1.       Right-clicking an empty space on the Desktop, then click Personalize

    2.       Click the first category Window Color and Appearance

    3.       Now see if changing the “Color scheme:” will allow you to see a transparent background. If you have Windows Vista Aero available I would use that, but you may need to change to Windows Vista Basic if you have something selected like “Window Classic” or “Windows Standard”

    Second thing to try would be to see if you have Window Sidebar set to always be on top of other windows, which I have found to cause it to have a background. To check if this is currently selected you should be able to easily tell if every time you open a window part of the screen still show Windows Sidebar as opposed to the window covering the Windows Sidebar. If you feel that it is enabled to have Windows Sidebar on top of other windows, you can do the following to disable that feature:

    1.       Right-click the Windows Sidebar icon in System Tray located on the Taskbar, or you can right-click an empty space on the actual Windows Sidebar, then click Properties

    2.       Under “Arrangement” see if Sidebar is always on top of other windows is checked, if so uncheck it.

    3.       Click OK

    Third option would be to download a third party program that will allow you to adjust the background image. If you go to the link listed below you can see an example of a program that allegedly does this:
