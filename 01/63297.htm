This thread is becoming somewhat tangential in your wonderment of long numbers and somewhat offensive presumptions.

I’m not sure what you mean by ‘relatively narrow pipeline’, by the way.  The only pipeline I can think of is the SATA 6Gb/s ‘pipe’ leading to the fast helium filled HDD with a large cache we’ve been referring to.

Anyway, here’re my conclusions from testing with old internal and external drives over the past day or so.

BitLocker encrypting a drive which already has data is much, much slower than encrypting an empty drive and then transferring the data to it.  Both methods differ in time hugely, but have the same outcome – the data is encrypted.  This is because HDDs cannot read and write at the same time, unlike some SSDs.  They can swap between reading and writing very quickly (BL was seemingly reading and writing/encrypting the 4.5TB at around 15MB/s) and appear to do this simultaneously in Resource Monitor.

So, anyone who wants to encrypt a large amount of data on a HDD should format the target drive and then BitLocker encrypt the empty volume before transferring the data.  It can be up to 10x quicker:  instead of 15MB/s encrypting existing drive data, you can encrypt at 150MB/s by transferring the unencrypted data to the empty BitLocker enabled drive.  However, be aware that there'll be a large spike in CPU usage during the encryption transfer process.

So, encrypting 4.5TB of data already on the drive took about 8 days. If I would’ve encrypted the empty drive first then transferred the data to it, it would’ve taken less than one day.  Thanks again, @ IJustHelping.

    3) Click Browse to open the Open Backup File box.
    4). Click Browse and Select file to catalog dialog box is displayed.
    5) Select the backup file and click Open. (The name and path of the backup file are now displayed in the Open Backup File dialog box.)

    7) In the list on the left, expand the various backup files and select the drive or folder to view details of the backup in the right pane.
    8) Select the file(s) or folder(s) to be restored from the list on the right.

    11) Click the Close button in the Restore Progress dialog box.
