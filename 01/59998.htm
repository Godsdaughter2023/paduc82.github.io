wesernsberger say:

By doing a systematic search, was able to isolate the memory leak to the "Dell Wireless WLAN Tray Service".  Once I disabled it, the memory leak went away and my Windows 10 system is now running fine.  My big breakthrough was when I learned about clean boot.  See How to Perform a Clean Boot.  The leak went away when I did a clean boot so I knew my problem had to be one of my startup tasks or system services.  Fortunately, the clean boot instructions show how to do a boot with any desired subset of startup tasks and system services.  So first I rebooted with all the startup tasks disabled but all system services enabled.  There was still a memory leak, so I knew it was one of the system services.  So then I disabled half of the system services and rebooted.  You get the idea. For each test I would only run task manager after reboot (so I could see what was going on) but do nothing else on the computer.  I would record memory utilization every 15 minutes or so until it was clear whether a leak was still present.

I did a quick Internet search.  It looks like there is a history of problems with the "Dell Wireless WLAN Tray Service" including memory leaks.  I also saw comments that it is no longer needed -- that the wireless support built into Windows 10 handles wireless just fine.
