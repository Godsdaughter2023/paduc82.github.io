r--b say:

Had the same problem with my Windows 10 insider build.      Was able to fix it.     I think it may be related to permissions certain applications put on registry keys queried by the "set defaults by app"

To fix it, download process monitor (sysinternals aka microsoft)

1. Run process monitor and add process name -> SystemSettings.exe to the include list.  Uncheck monitoring of everything except registry.

2. Open settings->default apps

3. Clear any captures already on screen (ctrl-x in process explorer)

4. Click set defaults by app

5. Wait for it to crash, then pause/stop capture in processexplorer

scroll to the bottom of the capture, and read upwards and look for what registry keys it was trying to to access prior to it crashing (ignore the bunch of regkeyclosed at the bottom),   In my case the last application it was hitting in the registry was Nullsoft Winamp.

Uninstalling Winamp, then Set Defaults by app worked perfectly.

Good luck
