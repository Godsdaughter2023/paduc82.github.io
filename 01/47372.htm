I think what he is asking is if there was a wireless controller that was paired to the device that is paired to another console now. Either that or if there is a wireless controller that is paired to the console that might be causing the issue.

Simple way to figure this out, do you have two consoles? If not then A would not be an issue. If so, then it STILL should not be an issue for the same reason a Bluetooth can only be working with one phone at a time. Pairing can only be done with one device at a time. BUT... I have seen it happen with my sons controller once. He turned on his xbox with the controller and it also turned on MY xbox. We paired the controller with his xbox a second time and the issue was gone. I think it was just a fluke and it started to look for the original xbox, but then it went to his. It never happened again after the second pairing.

To test theory B, take the battery pack off of the controller (s) and see if it happens again. If it does not happen again, skip to the next paragraph. If it does, you may have another device (a remote control, phone, wireless doorbell, or something else that is sending out signal) that is interfering with the signal to the xbox. Now you have to see what other wireless devices you have in the area that may be causing the interference. Test each device by using it in front of the xbox and see if anything causes the issue.

If detaching the battery pack makes the issue go away, then try putting the battery pack back into ONE of the controllers. Wait and see what happens. If the issue returns, take the battery pack out of that controller and repeat the process with each controller you have making sure only ONE controller has a battery pack in at a time. Should one controller cause the issue but not any of the others, try swapping a different battery pack into the suspect controller. Test again. If it does not happen, it might be the battery pack. If it does happen again, it may be the controller. If it is the battery pack, just replace it. If it is the controller, you may have corrosion in the controller, a bad circuit, or a closed loop (etc.). In other words, the controller may need to be rebuilt or it may need a deep cleaning. I will let you decide how to handle that, but I would just buy a new controller.

Select the view.

Go to the drive that the zip is in and copy the file.

Again go to App Data.

Click Local.

Click Microsoft.

Put your curser on the page and right click then paste.

 

You should now have MSN and MSN.old in the list.

 

Open up your email and the old stored folders should be visible.

js75xdtyhe44 say:

Backing up history folder serves no good purpose. Because history folder is dynamically created every day, it has different index.dat file. This index.dat files is system locked up by the Windows Explorer (which can be ulocked with Unlocker_1.9). Restoring index files will not help you since the site you want to revisit will always create a new session cookie, or permanent cookie depending on what server requires.

Now, you could recover the names and links of sites you have visited from index.dat. But you have to use a trick. Here is the way how you would use the trick.

Since index.dat is locked by current explorer session, you use WinPE-based utilities as mentioned before (UBCD, BartPE, Live CD, Hiren Boot CD, etc...). This will allow you to access to the index.dat files, cookies, and even temporary files folder and its contents.

Then copy index.dat file(s) to a USB drive as index.dat.txt, and cookie files which starts with*** Email address is removed for privacy ***, *** Email address is removed for privacy ***, etc..

Using two utilities you can get the following items:
(1) URL Extractor: This will allow you to extract URLs  from renamed index.dat.txt file (links http://, ftp://) and save it as csv or txt file. At least it will allow you to obtain link values.

http://www.spadixbd.com/extracturl/index.htm

(2) Cookie Explorer: This will allow oyu to view cookies, and its values:

http://www.karenware.com/powertools/ptcookie.asp

But be mindful that these exercises are futile to get connection to a server since you need to establish a session cookie, and if your license (via cookie) has expired, then you cannot access server (a trick in olden days was used to reset server timing..but that is fixed now).

Another trick which may or may not work: Use a program called Locate32. With this program you can get everything under the temporary folder tree, even system locked index.dat. This way might be simpler, if it works. Plus, Locate32 is far superioir to Windows Search or Google Desktop, as you do not convey user data back to Microsoft or Google.

To avoid future problems like this one, you can install doPDF  program, andprint PDF files of the web page you are interested in. This way you will never have to resort to data mining.
http://www.dopdf.com/download.php

2) Select the option for C:/Windows

4) Type this command: BATCH spuninst.txt

Hit ENTER

5) Type this command: systemroot

Hit ENTER
