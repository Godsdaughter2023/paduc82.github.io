KMODE_EXCEPTION_NOT_HANDLED (epfwwfp.sys) - Microsoft Community

kidzoom84 say:

KMODE_EXCEPTION_NOT_HANDLED (epfwwfp.sys)

Upgrade Windows 7 Ultimate 64 bit to Windows 8 Pro.

Downloaded to usb. Ran install later without usb.

Install complete. Booted to Windows 8.

Manually performed updates. Then reboot failed. Windows 8 provided text on blue screen indicating problem and had a percentage counter.

Reboot failed multiple times and provided error KMODE_EXCEPTION_NOT_HANDLED (epfwwfp.sys)

 

System offered to boot Win 7 or Win 8. Running in Win 7 to access help.

 

How do I repair the install?

 

How do I handle KMODE_EXCEPTION_NOT_HANDLED (epfwwfp.sys)?

mikhail aurelien say: I just installed windows 8, and I too had this problem, and thanks to you i found a solution...
took me all night to solve it and it was so simple...

the file is indeed from ESET smart security, the problem was removing the file. here is how i did it

After windows had about 2 failed attempts to boot, it went into recovery mode, which also failed. now this kinda gets hazy to me from here but u'll get my point if u just play around with it, the key is to find "command prompt"

http://www.howtogeek.com/107511/how-to-boot-into-safe-mode-on-windows-8-the-easy-way/

This link makes it easier to find, "command prompt" is in step 4

When you open command prompt just type 

DEL /F /S /Q /A "C:\Windows\System32\drivers\epfwwfp.sys"

type it exactly as shown with quotes spaces and all. then press enter

it should say that the file was deleted.
exit command prompt and restart your pc, i should boot up with no problems now. 

i uninstalled ESET completely after, and the pc has been working fine ever since i haven't tried reinstalling it yet, so i can't say if the problem was just after updating or if its just not compatible with windows 8.

let me know if this also works for you
