afzal taher say:

Hi,

Yes. You can join Windows XP mode to an Active Directory or a Domain.

Follow these steps:
a. Make sure that the user's Active Directory account is not limited to one concurrent login.
b. Open the XP Mode virtual machine and select Tools -> Disable Integration Features.
c. In the running XP Mode virtual machine, open Start -> Run -> sysdm.cpl, rename the computer (Example - 'Hostname-XP', where 'Hostname' is the name of the physical machine)
and then join the XP Mode virtual machine to the domain.
d. In AD Users and Computers, move the virtual machine from the Computers container to the correct Organizational Unit.
e. If the user is supposed to be a local admin, log in with XPMUser (or a domain account with local admin rights), Start -> Run -> control userpasswords2, click Add, specify the user's
domain account, and put it in the Administrators group.
f. Have the user log into the XP Mode VM using his/her domain account instead of XPMUser.
g. Map a network drive to the user's home directory, if this didn't happen automatically via Group Policy when the VM was joined to the domain and the user logged in.
h. If the user is not supposed to have local admin privileges, disable the XPMUser account or reset its password.

hairyfool say:

Clean up both PCs by removing them from the HomeGroup on each PC.

Then join a HomeGroup on one PC. It will scan the network for any existing group then offer to create a new one and generate a password. Note or change the password and complete the HomeGroup setup defining which folders/devices are shared.

Then join a HomeGroup on the second PC. This time it should detect the existence of the group you created on the 1st computer so enter the password set up previously and set your shares.

Only one HomeGroup can be active on a network, computers cannot be on different networks to be in a group, the network type must be Home and the IPv6 protocol must be active. A computer cannot be on two HomeGroup.

michaelwhitlock jr. say: So, if I do not have IPv6 I can't use home group to connect to other PC's in my house???

hairyfool say:

Yes, it only requires that HomeGroup members have IPv6 not necessarily the router. This works because they have to be on the same network anyway so do not depend on any routing function.

Some people have disabled IPv6 because as far as general networking is concerned it isn't needed then find HomeGroup setup fails.

For Windows XP, it makes a difference whether "Simple File Sharing" is enabled or disabled.  Simple File Sharing is *always* enabled if you have Windows XP Home and it is the default setting in XP Pro.  With Simple File Sharing enabled, *all* users that network-connect to your machine are forced to authenticate to the machine as the "Guest" user -- which means that you cannot have any per-user authentications in Windows Home or in XP Pro without disabling Simple File Sharing.  This means that every network user only has file access to files that the Guest user has permissions to.

 

If you have XP Pro and disable Simple File Sharing, then you can have users authenticate to your local machine and give each his/her own set of file permissions.  If the user connects to a network machine that contains the same username and password as the user is currently logged-in on his local machine, then the authentication is automatic.

"How to disable simple file sharing and how to set permissions on a shared folder in Windows XP"
