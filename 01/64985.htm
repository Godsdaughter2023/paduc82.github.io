Sooner or later, all software becomes obsolete and unusable.  I think you've hit that particular wall with your old version of Money.

Note that the registry hack that "worked" for some users was for pre-release versions of Windows 10; this may or may not work on the current version of Windows 10.  Be sure to backup your registry before trying anything.

afzal taher say:

Hi,

When exactly do you get the second error?

This issue may occur if the "NT Service\DHCP" account does not have the necessary permissions for the following keys:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Dhcp
permission needed: Query value, Create value, Enumerate Subkeys, Notify, Read Control

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Dhcp\Configurations
permission needed: Full Control, Read

To fix this issue, you may try the following steps:

Method 1:

a. In Registry Editor, browse to the key for which you need to add permission.
b. Right click the key, and click Permissions.
c. Make sure Locations is selected to be the local computer.
d. In the "Enter the object names to select field, type "NT SERVICE\mpssvc". Then click “Check name.”
e. Click OK.
f. Then select the account which appears in the list, and add the appropriate permission for it.
g. When this is done, click OK.

Refer this KB article for more information:
http://support.microsoft.com/kb/943996

Important: This Method contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly.
Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if
a problem occurs. For more information about how to back up and restore the registry, click the following link:
http://windows.microsoft.com/en-US/windows7-vista/Back-up-the-registry

Method 2:

If the issue persists, then run the following commands in an elevated command prompt.
a. Click on start button.
b. Type cmd in the start search box, right click on the Command Prompt and select "Run as Administrator".
c. Type the following commands (One after the other) and press enter.
NET  LOCALGROUP   administrators  "NT Authority\Local Service"   /add

netsh winsock reset catalog

Check if it resolves the issue.

i don't know how to diagnose or test a windows share from within the windows PC itself. is there a utility for this or some manual method (like telnet) to see if the service is up and running or otherwise figure out what's wrong?

kapil t [msft] say:

 

There are few things that you could do.

1. On win 7 machine makes sure "server" service is started. (find it in services.msc).

2. run netstat -ano on win 7 machine to make sure it is listening on TCP port 445. This indicates that Win 7 can be used as SMB server.

3. from linux /Imac use any tool which can do telnet and telnet to win7 machine on port 445. If the connection gets through then you know that connectivty is fine and firewall is not blocking anything.

4. If telnet fails the easiest test is to turn of firewall profile on win7 machine. run the comand

c:\> netsh advfirewall set allprofiles state off

after this command try to do telnet and if telnet work, try to connect to the share.

to turn on the firewall profiles run the command

c:\> netsh advfirewall set allprofiles state on

If telnet works and share access does not then i would suggest you to use network monitor to take a network trace. This will help you inspect the packet exchange and see where is it failing.

You can download netmon from the location: http://www.microsoft.com/downloads/en/details.aspx?FamilyID=983b941d-06cb-4658-b7f6-3088333d062f&displaylang=en

once you have taken the traces when attempting to connect please share it with me probably on skydrive  and i can look into it to give you further suggestions

Hi Jimmy

If those are blue arrows facing each other, this is a bug in the latest Windows Update, your files are compressed

In File Explorer, right click your hard drive and choose properties

At the bottom of the General Tab, is the Compress Drive box ticked, if so Untick that option

If not, check the folders like Documents, Desktop . . .etc. and see if they are compressed, if that are, untick that option

Please Note, it can take a long time to decompress you drive, so only do this when you won't need your PC for a while . . .
