  7. Type the following commands to start the Background Intelligent Transfer Service and press Enter:

So I've just spent several, several hours figuring out how to fix this.  Thank you, everyone, for your tips and advice.


Getting into Safe Mode (from a hard boot) is the first challenge.  All you people who say "Just go into Windows Recovery and select the Safe Mode box" are morons.  If we can't get into Windows, we can't get into Windows Recovery.  The issue is HARD Booting into Safe Mode.


The solution above is the correct solution.  Power off completely (hold the power button for five seconds.  You'll know when the machine powers off.)  I tried a few various arm and finger positions to get the optimal contortion for these keystrokes.  While off, hold down the right shift key with your left hand.  Starting tapping F8 with your right hand, then use your left index finger to tap the power button while still tapping F8.  The trick is to register this combination of keystrokes BEFORE the Samsung Logo.  If you get to the spinning dots, you're too late.  Wait for it to boot up to the black screen/flashing cursor and try again.  If you get it to register, you'll get a "Please Wait" under the Samsung logo.  You can stop mashing F8 and go through the Advanced Settings / Troubleshoot / Restart Settings / Safe Mode (Option #4 or #5.)


Once in Safe Mode, go into Control Panel > Hardware and System > Device Manager > Display Adapters.


I really don't believe the Black Screen issue has anything to do with AMD Radeon.  But just in case:


You don't have to delete all your display adapters, just Disable them for now.  Click on each display adapter (Radeon and Intel), then the tab for Driver Details.  Click DISABLE near the bottom.  Confirm. 


Now you can reboot normally.


You'll come up using on-board graphics and a generic Microsoft Display Adapter from 2006.  That's fine.  Resolution sucks and everything is huge.  It takes longer for the screen and items to display and is choppy (you're using on-board graphics instead of your dedicated graphics card, so this is normal.)


To verify this, go to your desktop, right-click on the background, and choose Screen Resolution.  Then click Advanced Settings.  You should see that you're currently using the generic MS Display Adapter.  This is good for this point.


From here I tried Samsung's Software Update Utility, Windows Update, Intel, and everyone else.  Everyone believes that Intel Diver 10.3 something (released 10/3/2013 or thereabouts) is the best driver.  Intel's Identification Utility even thinks so.  Well, I didn't believe it.  So like others above and on other boards, I searched high and low to find an Intel driver for the HD 4000 series that works.  It took three tries, but the one from June 2013 works and works just fine.


Once you install the Intel driver, you'll have to boot up again.  BUT, this won't be the end of it!!!


Even after rebooting, I was still using on-board graphics.  I had to go into Device Manager, and could see that Intel Graphics was enabled.  So I checked again through Screen Resolutions > Advanced Settings and could see I was still using on-board.  So I disabled the Intel Display Adapter and immediately reenabled it.  (Also make sure your Radeon Adapater is enabled.)  This immediately prompted a "Hardware Settings have Changed" message with the option to restart.


I did.  And everything came up just fine.  I'm now using the Intel HD 4000 Display Adapter with a three month old driver and everything works and seems (so far) to work well.


If by chance you install a driver that doesn't work, you'll have to boot up into safe mode again (good luck) and disable the Intel Display Adapter, boot up again, find another driver, and try again.

6. When you are finished, log into "Sheldon". All your stuff should be there. When you are sure you have everything you need, you can delete the old-corrupted-account from Control Panel>User Accounts.

C. General Recommendations For Setting Up Users In Vista:

You absolutely do not want to have only one user account. Like XP and all other modern operating systems, Vista is a multi-user operating system with built-in system accounts such as Administrator, Default, All Users, and Guest. These accounts should be left alone as they are part of the operating system structure.

You particularly don't want only one user account with administrative privileges on Vista because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult. It isn't impossible to activate the built-in Administrator to rescue things, but it will require third-party tools and working outside the operating system.

The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. Running as a Standard user is best practice for security purposes and will help protect your computer from infection. After you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).
