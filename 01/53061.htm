¡firedog say:
Do you have any program running that could prevent unexpected changes taking place? This would include any sort of security suite or programs claiming to carry out system care.

Set Windows Live Mail to Work offline, then move all messages from the account folders to user-created folders under Storage folders. When you're sure that there are no messages in folders under the account that you want to keep, right-click on the account name in the folder pane and select Remove account.

Close down Windows Live Mail and wait a few minutes while the program does its housekeeping, indexing the messages you just moved. Open Task Manager (Ctrl-Shift-Escape) and wait until wlmail.exe no longer appears in the list on the Processes tab.

Re-launch Windows Live Mail and check that you're still working offline (the button on the Home tab of the ribbon should read Go online). Then press Ctrl-Shift-T to start the Add your email accounts wizard and add the account back again. To be on the safe side, before going online to test the account, right-click on its name in the folder pane and select Properties. Give it a name different from the one you just removed.

Go online and if necessary click Send/receive to initiate a download. If you used the default settings, the account folders should be downloaded and eventually populated with the messages present on the server.

a. user say:

Try going to the C:\Users folder and setting permissions (and ownership if absolutely necessary)  from there (since I think the Downloads folder is inheriting permissions from this folder as the username folder has no Properties button so it probably isn't a factor).  Before trying that try changing ownership on the documents folder and/or those sub-folders again and ignore the fact that it can't display the current owner. Click on Edit anyway and see if you are able to get in to make the changes.  Then in permissions, check Advanced to make sure you're listed with full control (and change or add  it if necessary) and the Everyone user is not listed with Deny access.  I have ful rights access (and ownership) of the Downloads folder on my system so I know it is possible.  Also first review the following information before attempting to change the Users folder.

Here are some articles on permissions and ownership that may help you in case you're missing a step or something:

To see your permissions, right click the folder/file, click on properties, and check out the security tab.  Check to see what permissions you have by clicking on your username (or user group).  Here are the types of permissions you can have: http://windows.microsoft.com/en-US/windows-vista/What-are-permissions.  You need to be an administrator or the owner to change permissions (and sometimes being an administrator or even an owner isn't enough - there are ways to block even them from access (though a smart administrator knows these ways and can get around them - but usually shouldn't because they were denied access usually for a very good reason).  Here's how to change folder permissions in Vista: http://www.online-tech-tips.com/windows-vista/set-file-folder-permissions-vista/.  To add take and grant full right permissions and ownership to the right click menu (which will make it faster to get in once it is set up), check out the following article: http://www.mydigitallife.info/2009/05/21/take-and-grant-full-control-permissions-and-ownership-in-windows-7-or-vista-right-click-menu/.

To fix this problem with the folders, take Owner tab.  Click edit and then click the name of the person you want to give ownership to (you may need to add it if it isn't there - or it may be yourself). If you want it to apply to sub-folders and files within that folder/drive, then check the box for replace owner on sub-containers and objects and click OK.  Back out and now there's a new owner for the folders/drive and files who can change permissions as needed.  Here's more information on taking ownership of a file or folder: http://www.vistax64.com/tutorials/67717-take-ownership-file.html.  To add take ownership to the right click menu (which will make it faster to get in once it is set up), check out the following article: http://www.howtogeek.com/howto/windows-vista/add-take-ownership-to-explorer-right-click-menu-in-vista/.

Good luck and I hope this helps!

frederik long say: Regardless of your password policy, you cannot afford to ignore the facts. From your reports I gather that they are as follows:
  • Using imaging techniques, you reset Windows to its condition of x months ago.
  • Restoring an old image will reset the machine to that old date, warts and all, including passwords.
  • You are no longer able to log on.

The inescapable conclusion is that the password you now use (whether it is blank or not) is not the same as the password you used x months ago (whether it was blank or not). I have used the imaging technique for many years - first with third-party products and more recently with the native Windows image creation tool. I have never seen an exception to the rule that an image restoration performs a full reset.

Instead of arguing about what happened, you should now do this to put your operation back on a stable basis:

  1. Reset the one of your admin passwords, using the complex process outlined below.
  2. Make sure you have two fully tested admin accounts.
  3. Keep a paper record of these admin accounts.
  4. Set a password for the Administrator account. Keeping it at a blank is an open invitation to hackers.
  5. Create a new image.
You can reset any password by going through the steps outlined below. The process is involved and requires some careful typing. Ask a computer-savvy friend to assist you if you lack experience in this area.
 1. Set your BIOS to use the DVD drive as the primary boot device.
 2. Boot with a Windows Repair CD.
 3. Press a key when the message indicating "Press any key to boot from CD or DVD …" appears.
 4. Select a language, a time and currency, and a keyboard or input method, and then click Next. You are now in Repair Mode.
 5. Select an Operating System, then click Next.
 6. Select the Command Prompt.
 7. Identify the correct drive letter for your Windows installation. It could be a drive other than C: but it won't be X:. You do it by typing the commands
    dir C:\Win* {Enter}
    dir D:\Win* {Enter}
    dir E:\Win* {Enter}
    etc.
    until you find the Windows folder.
    I will call that drive "Q:".
 8. Type these commands:
    cd /d Q:\windows\System32 {Enter}
    move  sethc.exe  \  {Enter}
    copy  cmd.exe  sethc.exe  {Enter}
 9. Reboot normally but hold down the Shift key permanently.
10. When challenged for a password, tap the Shift key five times. You will see a small black screen.
11. Type these commands:
    net user  {Enter}
    (To see a list of all accounts that exist on your machine)
    net user Administrator {Enter}
    (To see if the Administrator account is active)
12. Type these commands:
    net user "xxx" ppp  {Enter}
    (Replace xxx with the account for which you wish to set a new password, and ppp with the new password)
    net user Administrator /active:yes {Enter}
    (To activate the Administrator account if desired)
13. Log on under this account, using the new password.
14. Reboot the machine into Repair Mode as per Steps 1..6 above.
15. Type these commands:
    cd /d Q:\windows\System32 {Enter}
    copy \sethc.exe {Enter}
16. Reboot into Windows.
Instead using a Windows Repair CD in Step 2, you can use your Windows 7 Installation DVD. The steps are similar to the ones above.

vanessa yar say:

Hi Zandrae,

The slim white border that appears on every window when in high contrast mode is set by design. That is to distinguish the contents of each specific window from the main screen. To lessen the brightness of the white border, you can enable the Night Light feature which displays warmer colors. Just navigate to Settings > System > Display. Under Color, toggle it On. Just click the Night light settings option to adjust the color temperature according to your preference. 

If you want to have the ability to change its color, I suggest that you submit it as a suggestion through our Feedback Hub. This way our software engineers will be able to look into your suggestion to check if its feasible in the future updates. For the detailed instruction in sending your feedback, please check out Send feedback to Microsoft with the Feedback Hub app.
