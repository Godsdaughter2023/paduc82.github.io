andre da costa say:
CPU Not Compatible

To install Windows 10 on your PC, the processor (CPU) must support the following features: Physical Address Extension (PAE), NX, and SSE2. Most CPUs have support for these features, so if you receive this error, it is likely because the NX feature is not enabled on your system.

If your computer does support these extensions, you could still try using ISO media. In some instances, you might have to switch from your current architecture,  64 to 32 bit since these requirements mostly affect installation of 64 bit Windows 10:

How to: upgrade from previous versions of Windows using Windows 10 ISO file

See instructions: How to download official Windows 10 ISO files


hairyfool say:

There are some CPU requirements that are specific to 64bit introduced with Windows 8.1 so yes it is possible that compatibility failure in 64bit setup would pass in 32bit. It depends on what the system failed on in the 1st place.

"Processor: 1 GHz or faster processor or SoC (system on a chip). 64-bit versions of Windows 10 require a processor that supports CMPXCHG16b, PrefetchW, and LAHF/SAHF capabilities." (Paul Thurrott)

CoreInfo from Sysinternals will give a detailed list of the CPU capabilities

https://technet.microsoft.com/en-us/sysinternals/cc835722?f=255&MSPPError=-2147217396

Some are confused by the output. * means it is supported - means it's not. CMPXCHG16b is listed as CX16

spapakons say:

Hello!

I faced exactly the same problem today with a tablet HP Stream 7. Trying to install "regular" Windows 8.1 core, doesn't work. I had to use a working HP Stream 7 installation (thankfully I work in a computer store where we sell these), boot with Acronis 2017 Rescue Disk and create a full disk backup on an external USB hard disk. Then I could restore this backup to the internal storage of the non-working device and login into Windows. I can then change the product key to match the specific device and activate Windows again. To find the Windows key embedded in the Firmware you can use a utility such as ShowKey Plus.

If you don't have a working device to create the backup, an alternative is to install Windows 10 Home instead. No key is required, it is taken automatically from the Firmware and Windows activate online without any issue. In fact it is preferred to install Windows 10 for better compatibility, as recently I had had cases where Windows Update would go wrong and make Windows 8.1 installations non-working. Since you are installing on a tablet (usually UEFI boot only, not legacy BIOS boot supported), you have to create a USB Flash drive with Rufus from the corresponding Windows ISO. Before starting the process make sure in Rufus you select GPT for UEFI or it will not boot. Also it is a good idea to add all the necessary drivers in the USB Flash drive in advance, so you can install them later. Since touch will not be functional until you install drivers, you must use a USB hub and connect USB keyboard and mouse to control the device until you have installed all drivers.

I hope that helps!
