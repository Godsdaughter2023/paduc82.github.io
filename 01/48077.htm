The Media Creation Tool is region based, so you won't see some editions such as KN or N as options.

In most cases, you just need 'Windows 10' not "Windows 10 N' or "Windows 10 KN"

Windows 10 - contains Windows 10 Home or Pro

Windows 10 N - for users in Europe containing Windows 10 Home N & Pro N, lacks media features such as Windows Media Player.

Windows 10 KN - for users in South Korea containing Windows 10 Home KN & Pro KN, lacks media features such as Windows Media Player.

The thumbdrive needs to be formatted as FAT32 and use the GPT partitioning scheme. To do this, you need to use Rufus, a small tool you can download for free.

 

After you have installed Rufus:

  • Launch it
  • Select ISO Image
  • Point to the Windows 10 ISO file
  • Check off Create a bootable disk using
  • Select GPT partitioning for EUFI firmware as the Partition scheme
  • Choose FAT32 NOT NTFS as the File system
  • Make sure your USB thumbdrive in the Device list box
  • Click Start
  • Close when complete

First, note that you should be concerned with *all* programs that
start automatically, not just with those that go into the system tray.
Not all auto-starting programs manifest themselves by an icon in the
tray.
 
On each program you don't want to start automatically, check its
Options to see if it has the choice not to start (make sure you
actually choose the option not to run it, not just a "don't show icon"
option). Many can easily and best be stopped that way. If that doesn't
work, run MSCONFIG from the Start | Run line, and on the Startup tab,
uncheck the programs you don't want to start automatically.
 
However, if I were you, I wouldn't do this just for the purpose of
running the minimum number of programs. Despite what many people tell
you, you should be concerned, not with how *many* of these programs
you run, but *which*. Some of them can hurt performance severely, but
others have no effect on performance.
 
Don't just stop programs from running willy-nilly. What you should do
is determine what each program is, what its value is to you, and what
the cost in performance is of its running all the time. You can try
internet searches and ask about specifics here.
 
Once you have that information, you can make an intelligent informed
decision about what you want to keep and what you want to get rid of.

We know that Windows keeps backup copies of critical system files in this folder (assuming XP is installed on your C drive):

c:\windows\system32\dllcache

The advapi32.dll is one of the 3498 files that Windows File Protection (WFP) looks after so if you boot normally and rename the file, as soon as WFP recognizes that (instantly), WFP is going to swiftly and silently replace the now "missing" file with a copy from the c:\windows\system32\dllcache folder.  It will do that before you have a chance to copy in the file you downloaded and when it does you will not see anything on your screen but you will see a message like this in the Event Viewer System log:

Event Type:    Information
Event Source:    Windows File Protection
Event Category:    None
Event ID:    64002
Date:        2/6/2015
Time:        5:36:30 PM
User:        N/A
Description:
File replacement was attempted on the protected system file c:\windows\system32\advapi32.dll. This file was restored to the original version to maintain
system stability. The file version of the system file is 5.1.2600.5755.

Then if you were to copy some file you downloaded into the system32 folder, WFP will recognize it as (probably) not being a valid file and WFP will also replace it.

So you might think you are replacing the file, but WFP is going to keep copying the hopefully good backup copy over the top of it (but you won't see that happening).

That is not how you replace critical system files.

When you boot into Safe Mode, WFP is not running, so you could replace the file that way and WFP will not replace it since WFP doesn't run in Safe Mode, but what you probably did was replace the "good" file with some bogus file you downloaded from some unspecified source - and now your system will not boot.
