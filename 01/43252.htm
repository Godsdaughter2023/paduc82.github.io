I just got hit with this as well.  In my case, it happens when I rip with EAC while WMP is opened.  I assume WMP attempts to read file info while EAC is still creating the WMA file.

My QUICK FIX that avoids rebuilding the whole library and losing the play counts on other files, ratings of FLAC files, and any other data not stored in the music files themselves:

Short version: 

---------------

With WMP opened, send the files to recycle bin, wait a little, and undo the delete.

Detailed instructions:

-----------------------

1) In WMP, go to an album folder that has 0-bytes files

2) Right-click the file and select "Open file location"

3) In the Windows Explorer window that opened, select the files that are noted as 0 size in WMP

4) Select "Delete" (recycle).  Do NOT select "permanently delete"

5) In WMP, wait for the file to disappear

6) In Windows Explorer, do an 'undo' (type Ctrl-Z)

The file(s) will reappear in WMP with the proper size.

Hopefully, this will help some out there!

I should just like to emphasize do this by opening the File Location...if you delete from wmp or the explorer library - you'll screw up your wmp library! - All Thanks to Music_Fan - R
