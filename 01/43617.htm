    jacyintha say:

    I have the same DLC and a GoD version of Skyrim. I haven't had that happen to me at all. rather than a content issue, it sounds like the downloads are corrupted.

    ~

    One of the best ways to clear out corruption is the start with deleting it. Skyrim itself doesn't sound corrupted, so you shouldn't need to delete that. Just go into Games and Apps and remove the DLC.

    After that, go done to System Items (found in Storage) and delete the Marketplace System Data. This file is the Marketplace/store cache and could be where the corruption is residing.

    Next, go clear the System Cache (highlight your Storage device and press 'Y'). Please note: clearing the System Cache may require some game updates be redownloaded as it acts like the temporary storage for the console. It's usually a good idea to clear it out at least once every six months or so.

    For a bit more thoroughness, try reseating your hard drive. This is pretty simple, just turn your console off, remove the hard drive (if you have one), turn it on WITHOUT the hard drive, then turn it back off and replace the hard drive in/on the unit.

    ~

    That's about all you can do for corrupted content. If it's still having problems at that point, you might want to try following the same steps, but redownloading the DLC on a more stable network connection and not during peak hours.

    ~

    Good luck and happy gaming!

    mike hall... always say:

    First mistake.. adding a partition on a single drive system for the pagefile.

    A page file will only benefit from being moved if it is to another physical drive. The idea is that while one drive's heads are working with whatever you are doing, the other physical drive can pick up and put down data in the pagefile at the same time.

    Having it on a separate partition makes the one drive's heads work way harder than if the pagefile is in with the rest of the Windows files.

     

    Second mistake.. messing with other parts of the system when it was fairly obvious that the creation of the separate partition was the cause of the slowing down. Unfortortunately, you were working with the false belief that moving the pagefile was supposed to make a system faster.

     

    You should not take offernce at the above statement. It happens to ALL of us at some point in time. If you make chamges and the system bucks at them, reverse the changes before looking at other reasons for the bucking. If reversing the changes improves the situation, you know NOT to make the same changes again.

     

    As it is now, you have made changes over and above the original cause of the slowdown, and even reversing the pagefile change may not get back the performance that you had previously. 

     

    OK.. You need to at least put your pagefile back into C. You can leave your documents in the new parttition if you like, but they are no safer there than if you had left them in C. If the hard drive crashes out, you lose everything. To keep documents safe, back them up or copy them to an external drive. Do not MOVE them off C, because if the back up drive breaks, you have still lost all of them. A backup should be a mirror of what is on C.

     

    On a single drive laptop, I would put everything back as it was to make the restore function easier for the next time that you tinker around. BTW, tinkering is your third mistake.

     

    If you want something to mess with and learn on, get yourself a old desktop machine in which you can install multiple hard drives, multiple operating systems. Don't ever turn your production laptop machine into a test bed for tinkering.

    a. user say:

    That STOP error and the second parameter is 0xC0000034 indicates a mismatch of the interface to the hard drive.

    I can't say how that happened or what caused it...  sometimes a flat CMOS battery can cause some of the setting to change. 

    Windows gets started but then is having trouble communicating with the HDD.  If there is no complaint about a missing or corrupt file I don' think it indicates the hard drive has failed.

    Over time I have accumulated a list of I have a list of BIOS HDD interface types and adjustments that have worked for other systems of various manufacturers and models so it might take some trial and error and experimentation to find what works for your system.

    Here it is:

    Before making any adjustments, you should make a note of what the current BIOS settings are so you can change them back when you are finished.

    The adjustment varies so here are some examples I have accumulated:

    The adjustment is made in the BIOS and could be under Integrated Peripherals, SATA Device Configuration, SATA Mode or something similar.

    Make a note of what the current settings are before making any changes so you can change them back when you get done or if things get worse.

    After you make the changes, save them and see how things look.  You may have to try a few things to get it to behave.

    If you find the mode is set to RAID/SATA mode, change the mode to IDE.

    If you find the mode is set to SATA, change the mode to IDE.
        (Acer Aspire)

    If you find the mode is set to SATA, disable SATA mode.

    If you find the mode is set to AHCI, change the mode to ATA.

    If you find the mode is set to AHCI Emulation Mode, change the mode to IDE Emulation.

    If you find the mode is set to SATA, disable SATA mode and/or change the mode to ATA.

    If you find the mode is set to SATA Native Mode, disable SATA Native Mode.

    If you find the mode is set to RAID/Autodetect AHCI, change the mode to Combination.

    If you find the mode is set to RAID Auto/AHCI, change it to RAID Auto/ATA (this was a Dell XPS 420)

    On some system of unknown make and model, In the Integrated Peripherals menu, disable" On Chip IDE Channel 0

    On an Acer Aspire 4720Z the mode needed to be changed from "AHCI Mode" to "IDE Mode"

    The WWW says this works on some Dells:  "I had to set my SATA Operation to ATA instead of AHCI"

    After making the adjustments, ACCEPT and SAVE the changes, and restart the system and see if it will now boot on the installation or Recovery Console CD.

    2. Search for Troubleshooting, then click on Troubleshooting.

    3. On the left pane, click on View all and select Video Playback.

    4. Click on Next and follow the on-screen instruction to complete the process.

    Method 2: Change Battery Plan Settings

    1. Press Windows key + X, select Control Panel.

    2. Search for Power Options, then click on Power Options.

    3. Click on Balanced, then select Change plansettings.

    4. Click on Change advanced power settings. In the drop-down select High Performance and click Apply.

    5. Then in the list of options that show up expand Multimedia settings. Double-click on When playing video.

    6. Click on Plugged in and then select Optimize video quality.
