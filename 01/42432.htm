considerate_guy say:

Movie Maker cares about the aspect ratio of the picture, not the actual pixel size. And it uses that aspect ratio to figure out how to fit it into the project aspect ratio.

The aspect ratio is basically the width divided by the height.

Let's take a few examples:

1920 x 1080
--> 1920/1080 ≈ 1.7778 (this is a widescreen 16:9 aspect ratio)

1526 x 1080
--> 1526/1080 ≈ 1.413 (this is closer to a 3:2 aspect ratio)

1620 x 1080
--> 1620/1080 = 1.5 (this exactly a 3:2 aspect ratio)

1440 x 1080
--> 1440/1080 = 1.3333 (this is a fullscreen 4:3 aspect ratio)

Movie Maker has some special handling for pictures around a 3:2 aspect ratio. In a widescreen project, it will try to fill the full width (losing part of the top and bottom). For a full screen project, it will try to fill the full height (losing part of the left and right).

If you want to make this work, if you have any photos that are close to a 3:2 ratio, edit your photo in PS so that it has pixels on the left and right to make the dimensions exactly match 1920 x 1080.

When you create a scheduled task to run this command:

wscript.exe  c:\Scripts\Monitor.vbs

then the task will create a pop-up message for the nominated account. Outputs other than a pop-up are, of course, possible. Monitor.vbs looks like so:

'---------------------------------------------------
'Generate a pop-up when an event with ID 15 recorded.
'5.2.2009 FNL
'---------------------------------------------------
iEventID = "'15'"
sUser = "FLong"

Set oWshShell = CreateObject("WScript.Shell")
Set objWMIService = GetObject("winmgmts:{(Security)}\\.\root\cimv2")
Set colMonitoredEvents = objWMIService.ExecNotificationQuery _    
    ("Select * from __InstanceCreationEvent Where " _
    & "TargetInstance ISA 'Win32_NTLogEvent' " _
    & "and TargetInstance.EventCode = " & iEventID)

Do
    Set oLatestEvent = colMonitoredEvents.NextEvent
    oWshShell.Run "msg.exe """ & sUser & """ """ _
    & oLatestEvent.TargetInstance.EventCode & " " _
    & oLatestEvent.TargetInstance.Message & """", 0, False
Loop

You must run the task under the System account. When you adjust the code to your needs while testing, you must use the command taskkill.exe /f /im wscript.exe to force Windows to use the modified script. The trigger for the scheduled task should look like so:

Here is some modified code:

'---------------------------------------------------
'Write a log line when an event with ID 7036 recorded.
'3.6.2015 FNL
'---------------------------------------------------
iEventID = "'7036'"

set oFSO = CreateObject("Scripting.FileSystemObject")
Set objWMIService = GetObject("winmgmts:{(Security)}\\.\root\cimv2")
Set colMonitoredEvents = objWMIService.ExecNotificationQuery _    
    ("Select * from __InstanceCreationEvent Where " _
    & "TargetInstance ISA 'Win32_NTLogEvent' " _
    & "and TargetInstance.EventCode = " & iEventID)

Do
    Set oLatestEvent = colMonitoredEvents.NextEvent
    set oFile = oFSO.OpenTextfile("d:\Event.log", 8, true)
    oFile.write date() & " " & time() & " "
    oFile.WriteLine oLatestEvent.TargetInstance.EventCode & " " _
    & oLatestEvent.TargetInstance.Message
    oFile.close

    set oFile = nothing
Loop

It will add a log line to d:\Event.log each time event 7036 is registered. The script works perfectly when invoked like so under an elevated Command Prompt:

wscript.exe  d:\event.vbs

It then runs continuously in the background. When you modify the code you must kill wscript.exe, then re-issue the above command.

Hi ligazetom,

The error code, 0x80070032, means a corrupted file system. A ReFS drive does not support installation of any apps or programs. This file system is currently available only in Windows 10 Enterprise and Windows 10 Pro Workstations. I would suggest that you create a NTFS drive so you can change the default location for your applications/programs.

In line with this, we suggest submitting a feedback through the Feedback Hub app. This way, you can provide a suggestion or feedback to our developers to include ReFS again in the future updates. To use the Feedback Hub app, follow these steps:

  1. On a Windows 10 device, search for “Feedback Hub” in Cortana search, then launch the app.
  2. Navigate to Feedback in the left menu, then press + Add new feedback.
  3. Select the Problem, and share any details you think are relevant, and choose an appropriate category and subcategory.
  4. Important: If possible, reproduce the problem(s) after clicking Begin monitoring (or Start capture) near the end of the form; Stop monitoring when you’re done.
  5. Once you’ve completed the form and monitoring, click Submit.
  6. Click Continue using Feedback Hub.
  7. Click My feedback at the top, then find the item you submitted (it may take a while to appear).
  8. Open the item you submitted, then click Share.
  9. Click Copy link, then paste the link here in your response (it will look like https://aka.ms/<unique link>).

Should you have further queries, feel free to post it.
