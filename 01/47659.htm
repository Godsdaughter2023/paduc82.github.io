robertjwin say:

Amazing that no-one has managed to answer this properly so far (the MS employee's response was of course unsurprisingly useless).

The behavior as far as I can tell was copied by MS from Apple, and unless my memory is failing me pre-dates Win95 (if I remember I'll check Win3.11's File Manager next time I install the OS in a VM). At the very least (sorry to disappoint you ;) I know for sure that Windows Explorer has had it since Win95's debut.

The default double-click delay in Windows I think is still 500ms, although it can be easily changed:

So while the first click selects a file (obviously if it's not already selected), a second click on the same file within 500ms (or whatever the user has set) of the first is what makes the combined action qualify as a double-click. A second click outside the double-click threshold on a previously selected file is naturally thus treated as a second single-click and activates renaming or edit mode. This of course trips up many people but since it's a divisive issue I guess MS can't remove it completely.

Instead there are options for Win/File Explorer to use single-click to open items (as in a browser), and to display checkboxes for item selection. You can of course also rely on the ribbon (earlier versions of Explorer had the main menu on top) or the right-click context menu to rename, or (my favorite and the fastest method) simply highlight a file and press F2.

Hope that answers the question to your satisfaction. :)

(The answer to how to stop windows update: Disable the BITS service). 

1. Start+'R'. Type "services.msc"

2. In the list of services, locate "Background Intelligent Transfer Service". Right-click and select "Properties".

3. In the properties dialog, set the startup type to "Disabled".

Remember to set it back when you're done messing around. Barring catastrophic Windows Update cycles like this one, that require delaying updates for a few days, it's very bad to leave this disabled for any significant period of time, and it interferes with services other than Windows Update (some of which are critical for correct operation of servers in AD domains).

To do this you'll need to access a Command Prompt in repair mode using whichever method works here: https://www.tenforums.com/tutorials/2294-advanc.... You can also try the option to System Restore. If you don't have the password this may be limited to using the booted media.

Insert media, boot it by powering up PC while pressing the BIOS Boot Menu Key: https://www.sysnative.com/forums/hardware-tutor...

Choose the boot device as a UEFI device if offered. If the media won't boot you may need to enter BIOS/UEFI Setup (pressing key given in chart in link above) to turn off Fast Boot or Fast Startup first.

After it loads, on second screen, choose Repair Your Computer, then Advanced Troubleshoot Options to try System Restore, or if that fails the Command Prompt to enable the Built-in Admin account .

If this fails from the Troubleshoot Options try a Reset, saving files.

If this fails choose Install Now, then Custom Install, then at drive selection screen delete all partitions down to Unallocated Space to get it cleanest, click Next to let it create needed partitions and start install - this makes it foolproof.
