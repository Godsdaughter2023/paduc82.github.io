a. user say:

Hi,

 

It can be done on SSD's

 

1. Click on Start
2. Enter SystemPropertiesAdvanced.exe into the search field
3. In the fieldset Performance at the top click on Settings
4. Go to the tab Advanced and click on Change

5. Uncheck “Automatically manage paging file size for all drives”
6. Select your SSD drive, e.g. Volume E:
7. At the bottom select No paging file, click on Set

8. Confirm with OK

The paging file on your SSD is now disabled. Hopefully, this will increase the life span of your SSD.

mike hall... always say:

Windows, MacOS, Linux and UNIX operating systems all use pagefiles.

 

Windows requires a small page file of around 500mb on the boot drive such that dump files can be created and saved in the event of a crash, and Windows and other applications rely on a page file for normal operation. 

 

By leaving 500mb on the SSD and putting the bulk of the pagefile on the regular drive, reading and writing to the page file on the SSD will be vastly reduced, and Windows et al will still have a useful and usable page file..

How did I miss this one?

Error Number: 0x800C0133 = a corrupt Inbox.

Move any mail you wish to save to a local folder you create. Then delete the problem Inbox as follows.

Tools | Options | Maintenance | Store Folder will reveal the location of your Outlook Express files. Write the location down and navigate to it in Windows Explorer or, copy and paste it into Start | Run.

In WinXP, Win2K & Win2K3 the OE user files (DBX and WAB) are by default marked as hidden. To view these files in Windows Explorer, you must enable Show Hidden Files and Folders under Start | Control Panel | Folder Options | View.

With OE closed, find the Inbox.dbx and delete it. A new one will be created automatically when you open OE.

General precautions for Outlook Express:

Do not archive mail in the Inbox or Sent Items. Create your own user defined folders and move the messages you wish to save to them. Empty Deleted Items folder daily. Although dbx files have a theoretical capacity of 2GB, I recommend about a 300MB max for less chance of corruption.

After you are done, follow up by compacting your folders manually while working *offline* and do it often.

Click on Outlook Express at the top of the folder tree so no folders are open. Then: File | Work Offline (or double click Working Online in the Status Bar). File | Folder | Compact all folders. Don't touch anything until the compacting is completed.

Turn off e-mail scanning in your anti-virus program. It is a redundant layer of protection that eats up CPUs and causes a multitude of problems such as time-outs and account setting changes. Your up-to-date A/V program will continue to protect you sufficiently. For more, see:
http://www.oehelp.com/OETips.aspx#3

In Tools | Options | Maintenance: Uncheck Compact messages in background and leave it unchecked. {N/A if running XP/SP2}.

¡firedog say:

OK. I'm not sure why you're seeing the certificate error, but I wouldn't put it past Avast to be interfering in this respect.

  1. Please turn off Avast's Mail Shield - you don't need it and it causes problems with secure email connections. Then change your mail settings to match AT&T's recommended ones:
  2. Right-click on the account name in the folder pane and select Properties.
  3. On the Servers tab,
    1. Change the incoming mail server to inbound.att.net
    2. Change the outgoing mail server to outbound.att.net
    3. Ensure that the Email username is your complete SBCGlobal email address
  4. On the Advanced tab,
    1. Ensure that the server port for outgoing mail is 465
    2. Ensure that the server port for incoming mail is 995
    3. Ensure that This server requires a secure connection (SSL) is selected for both servers.
  5. Click Apply then OK.
  6.  Remove any unsent messages from the outbox (below the account folders in the folder pane), then close down Windows Live Mail and wait a few minutes while the program does its housekeeping and stores the new settings.
  7. Re-launch the program, then compose a new test message with a unique subject and send it to yourself. Post any new error message in your reply: you can click on a POP3/SMTP error message to select it, then use right-click to copy it and paste it into the reply box.
