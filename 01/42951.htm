runandwrite say: Well, I'm happy to say that I was able to come up with a workaround of sorts. While I by no means consider myself an expert, I thought I'd post my solution in case anyone else has the same problem.

STEP 1: Get a text file listing all of the files that need to be changed.

First, I found a Web site that told me how to use the command prompt to create a list of all of the path names in the directory I was trying to change:

1) Click the Windows orb, click All Programs, Accessories, Command Prompt.
2) Navigate to the directory for which you want to change permissions and ownership. (e.g. Type "c:\Users\Bob\Documents".) This should take you to the right directory.
3) Type the command "dir /on /b /s >list.txt"

That should create a text file called "list" in the directory of interest.

STEP 2: Create a command list to change file ownership.

The second step is to use a copy of the list to create a list of command lines to change the ownership of each file. To change a single file with the command prompt, you would type

takeown /f "full path of file"


For example:
takeown /f "C:\Windows\System32\en-US\winload.exe.mui"

Obviously, you don't want to have to type this over and over again. Fortunately, MS-Word will make it easy to change each line in your file list to include this information. Here's how you do this.

1) Create a copy of your list.txt file. You'll need the original again later on.
2) Open the file in MS-Word.
3) Click on "Replace."
3) For "Find what," enter ^p
4) For "Replace with, enter (including the quotation marks) "^ptakeown /f "
5) At this point you should almost have your command list. On the first line, enter takeown /f " before the file name. After the last line, delete the extra takeown /f "
6) Click on Save As > Other Formats and make sure "Plain Text" is selected. Choose "MS-DOS" as your text-file format. Be sure to check the box that "Allow character substitution," because this will ensure that your "Smart quotes" will be changed to straight quotes.
7) Close the file and open it as a text file. Hit CTRL-A to select all of the text and CTRL-C to copy it.
8) Use your mouse to paste all of the text into the command prompt. CTRL-V will not work. The command list will start executing line after line. Wait for it to finish -- this took about 10 minutes in my case.

STEP 3: Create a command list to change permissions.

Almost there. The command line to change permissions usually looks as follows:

icacls "full path of file" /grant (USER NAME):F

e.g.:

icacls "C:\Windows\System32\en-US\winload.exe.mui" /grant Bob:F

Once again, we'll use MS-Word to turn the file list into a command list.

1) As before, create a copy of your list.txt file, just in case you need it again for some reason.
2) Open the file in MS-Word.
3) Click on "Replace."
3) For "Find what," enter ^p
4) For "Replace with, enter (including the quotation marks) " /grant Bob:F^picacls "
6) In the above step, instead of "Bob" type the name of the user to who you wish to grant permission, or type "Users" to giver permission to everybody on your machine.
5) Clean it up -- enter icacls " in front of the first line and delete the extra icacls " at the end of the text.
6) As before, click on "Save" and choose "MS-DOS" as your text-file format. Be sure to check the box that allows it to change characters automatically, because this will ensure that your "Smart quotes" will be changed to straight quotes.
7) Close the file and open it as a text file. Hit CTRL-A to select all of the text and CTRL-C to copy it.
8) Use your mouse to paste all of the text into the command prompt. CTRL-V will not work. The command list will start executing line after line.

When it's done, all of the permissions should be changed! I did have a few exceptions - system files and things that did not want to be moved or deleted or moved, but this ended up being a dozen files or so instead of 4,000+, which is the number of files for which I changed permissions.
