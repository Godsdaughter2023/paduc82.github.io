tryx3 say:

On the opening screen of Windows backup, there is a link on the left Create a system image.  That is what you need to do [ThrashZone's reply was equally valid - he suggested using Macrium Reflect to make a system image instead].  A system image is an exact copy of the state of the drive[s] at the time the image is created [Windows, drivers, applications and all user data files on the drive - everything].

Whether you use Windows' own tool or Macrium Reflect, select both the partitions you mention.  By the way, the 10GB hidden partition is probably a computer maker's Restore to factory partition so look in your computer manual to find out how you would use that if the need arose.

Whether you use Windows' own tool or Macrium Reflect, you also need to make a boot disk so that you can be sure of being able to run the Restore system image procedure**.  On the opening screen of Windows backup, there is a link on the left Create a system repair disk.  Macrium Reflect will have an equivalent [probably called a boot disk or rescue disk or similar]. 

**  The Restore system image procedure using a system repair disk [this is a menu-driven procedure, there is really nothing to memorise]:-

  • You boot the computer from the system repair disk then
  • select your language & keyboard when it shows you these choices upon booting, then
  • connect the external drive containing your System image, then
  • select Restore your computer using a system image that you created earlier, then
  • choose the latest image [which it will show you the date-time of to avoid mistakes], then
  • have a pot of tea while it does the job [it will reboot into Windows on completion].

Macrium's procedure will be different but will achieve the same end.  It will almost certainly also be entirely menu-driven.

If you are restoring an image because your Windows had become corrupted then there is normally nothing further to do. 

If you have restored the image onto a replacement hard drive, you should then go to Control panel, [View by - Small icons, if necessary], Device manager.  Select the hard drive entries in the Disk drives section & in the IDE ATA / ATAPI controllers section and update them [i.e. update the drivers that control hard drive use].  If you have replaced the HDD with one of the same model & brand then this should all complete without incident.  If, however, the new hard drive is a different model & make, there might be problems [possibly with an entry for IRST in the controllers' section above that needed updating].  Personally, I use Acronis True Image system images as they are definitely better at coping with hardware changes.  Macrium system images are, as I understand it, also good at helping you cope with such changes but ThrashZone is better placed to comment on this.  When I changed my hard drive a few years ago, I found that the best test that the new system was fully working was then to run Windows update.  If it fails then search this forum or post a question explaining what the old & new HDDs were and asking for the advice of others who have changed similar HDDs.

1  Last night my computer became corrupted and hardly anything was working correctly

2  I tried a couple of things to fix it but got nowhere

3  I restored a system image from 2 Feb

4  My computer is now in the same state it was on 2 Feb, fully working - my antimalware application needed 42 days of updates, my TV application had to be retuned because channel frequencies had changed on 1 March so I lost the retuning work I did that day. 

- - - - - - - - - - - - - - - - - -

By the way, you might not need to transfer to a new HDD for several years.  I am still using HDDs that are 8, 6, 5, 1 years old in my computers.  In USB enclosures, I am still using HDDs that are 16, 11 years old.  I have seen lots of statements about HDDs being unreliable & prone to failure but that's a comparative statement not a damnation [things with lots of fast moving parts are not as reliable as solid state devices with no moving parts at all].  Personally, I have never had a failed HDD but I have had an HDD that was misdiagnosed as failed when in fact I could have saved my money because it was merely severely corrupted [it continues to work correctly today, 8 years after the alleged failure].

Your immediate benefit from making system images is that you have a quick & easy method of recovering from system corruption. 

- - - - - - - - - - - - - - - - - -

You do not have to do what follows, you can just carry on as you are.

You can also set up your computer to make system images easier to make & recover.  You can partition your HDD so that it behaves as two separate drives. 

Your OS & applications [everything that does something] stays on the main part of it but you move all your user files [Word files, Excel files, ...] onto the new "separate" drive. 

  • For example, I had a 500GB HDD and I partitioned it into 70GB & 430GB drives [C:\, D:\]. 
  • I installed Windows on C:\ and all applications went on the same drive [that's how Windows works, it knows it is on C:\ so it installs everything on the same drive - you don't have to go messing about at all]. 
  • I created a set of folders on D:\ for my own files - Documents, Favorites, Music, ...].  I then moved all my own files into these folders.
  • I then went to each of the Documents, Favorites, Music, ... folders within C:\Users\MyUserName, right-clicked, selected Properties, selected the Location tab and entered the new D:\ drive folder that I wanted the computer to use.  So WMP looks to D:\Music just as it had once looked to C:\Users\MyUserName\Music for files to show.  When I save a Favorite in IE, it is saved in D:\Favorites rather than C:\Users\MyUserName\Favorites.

The result of partitioning and "relocating user files" in this way is that my system itself is only using about 40GB so making a system image, or restoring a system image, only takes about 25 minutes because the image is also only about 40GB.  I have enough room on my external drive to keep several images** so I have several chances of restoring an old image to get around any system corruption problem that has developed such as last night's severe corruption.  [** Windows can cope with images being piled on top of each other, it just loses the older bits as it decides that they are no longer relevant.  I manage this myself instead.]

There are lots of threads in this forum with step by step instructions for doing the partitioning [or here] and for relocating user files.  All you have to do is keep reminding yourself is that each step is easy and it is just the unfamiliarity of the whole thing that makes it look daunting.

ThrashZone - I appreciate that Macrium Reflect might have advantages but some of your statements are incorrect.

  • Windows system images can be moved for storage from drive to drive or even to subfolders within drives. 
  • The WindowsImageBackup folder must be left whole with its contents unaltered [there is an exception but the general rule is to leave it all alone]. 
  • When you want to restore the system image, the WindowsImageBackup folder must either be where it was originally made or be moved to straight off the root of any drive except the Windows drive - so the image made in K:\WindowsImageBackup could be restored from K:\WindowsImageBackup, J:\WindowsImageBackup, D:\WindowsImageBackup [the drive I keep my data on] or even E:\WindowsImageBackup [an additional HDD in my computer].
  • Windows system images can be restored to drives that are larger than the system drive they were made from.
  • Windows system images cannot be restored to drives that are

    Please note also:-

    • Acronis True Image system images can be restored to drives that are smaller than the system drive they were made from.
    • Acronis True Image can work with Windows system images as well and can restore them to drives that are WD or Samsung-Seagate HDD is inside / connected both at the time the image is made and when recovery is required.

    • I agree that external HDDs are the best media for system images.

    • A system image made on a drive within the computer is barely more useful than a chocolate teapot.  If you drop the computer, there's no telling if the system image can be recovered.
    • The combination of a system image on an external HDD & the System repair disk [or Acronis / Macrium equivalent boot disk] is the best defence against system corruption - it can restore an image onto a completely corrupted computer even if that computer is so messed up that it cannot boot anymore.

GSM 7-bit Rules/Limits
160 characters for a single message charge.
153 characters for each part above 160. Ex. 153 x 2 = 306 = 2 message charges.

Unicode 16-bit Rules/Limits

70 characters for a single message charge.
67 characters for each part above 70. Ex. 67 x 2  = 134 = 2 message charges.
