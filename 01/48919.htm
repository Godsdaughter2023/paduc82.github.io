Homegroups are the Windows 7/8.x default local networking system for file and printer sharing on a local network. It is meant to be a simplified alternative system to setting up a conventional "Workgroup" on your LAN.


Homegroups are created automatically when a new network connection is detected or created on your computer. Possibly a nuisance but nothing scary in it.


As the name implies it is intended for use on private networks (LAN), not public ones (ie. internet).  Homegroups can only be used by Windows 7 and Windows 8.x computers. That makes them nearly useless for networking with computers that use different operating systems.

 

Each computer with a full version of Win7 (not Basic or Starter) can create its own Homegroup but these different Homegroups do not connect to each other. To have the Win7 computers connect to each other's shares they must all join the same Homegroup.


Windows by default generates a random password for each new Homegroup. You can change the password. No other computer on the network can join any  Homegroup without entering the current password for that specific Homegroup.


You still need to designate which files and folders will be shared on the network and provide the Homegroup password to the computers you want to have connected to it. You also control whether other computers have write permissions in your shared resources.


To get rid of an unwanted Homegroup every computer that has joined that Homegroup must leave it. This includes the computer that the Homegroup was created on. Once the Homegroup is gone you'll need to set up a conventional network Workgroup if you want to share files etc. with other computers on your local network.  If you've only got a single computer you need to make sure that file and printer sharing is not enabled on "Public" networks. That should be the default anyway but double checking never hurts. 


All the relevant settings can be accessed via the "Network and Sharing Center".

OR,


Your computer has a direct connection to the internet through your ISP and you have set that connection up as a Private network connection when it is actually a Public network connection.


You should only enable network discovery and sharing on Private (a home or work LAN connection isolated from your ISP) networks.


It seems that every so often your computer is discovering some of your ISP's other customers' computers that have similar network settings and also happen to have a Homegroup set up. In short, your computer is treating your ISP connection as your private LAN and is making LAN connections to other customers who have made the same mistake when setting up their network connections.

To address your points: Firstly, HomeGroup doesn't install itself, it is an integral part of Windows networking. It is always there in the background, available to be used on private networks only. You can either start up a HomeGroup on a Win7/8 computer (Windows starts doing this automatically when you have a new Private network connection) or you can join a Homegroup that was created on a different computer on the same local network that your computer is connected to.  If no one joins the HomeGroup it is pretty much just being there doing nothing and you can ignore it.


If you create a Homegroup on your own computer your computer is automatically joined to that Homegroup and it is always present. IOW you wouldn't be invited to join it. Other computers can only join it if you provide them the Homegroup password. No password, no sharing, no unwanted guests.


If you haven't created a Homegroup but you keep getting the option to join a Homegroup then the likeliest cause would be that you are intermittently connecting to other computers that DO have Homegroups enabled.  It is always possible to join a Homegroup on a private (Home) network so the invitation automatically pops up asking if you want to join. You cannot actually join those Homegroups unless you have their passwords.


The thing is it should not be possible to detect Homegroups that are not in your own local area network (ie. a home LAN) that your computer is connected to and Homegroups should not be able to exist at all on Public networks. The only way that a single computer connected directly to the internet could detect other people's HomeGroups would be if both your computer and those others have their internet connections configured as a Private network. If that network connection is changed to "Public" the Homegroup(s) should no longer appear.


As far as your concern about whether to let Windows manage Homegroup connections or to use user accounts and passwords, it really is a non-issue for anyone who is not sharing files and printers on a network. If you're not joined to other computers in a Homegroup then there are no connections to manage and if you aren't sharing files and printers over a network at all then there are no other users or their passwords needed to make network connections either. In your case it makes no real difference whichever option you pick.

Hi justinrpg7,

Generally, whenever your computer is scanned by Microsoft Windows Server for updates, it will only download those updates which are important and necessary for the computer, so that the computer is up to date. In order to check for the updates installed on the computer, you may go to Windows Update site and check the update history.

You may also check whether the Windows Update is automatically set to scan for updates every day.
Control Panel >> Windows Update >> Change settings >> Install updates automatically (recommended).
