voodoodreamer13 say:

Changing the active partition is likely to stop your computer from booting  PortableOperatingSystem to “0” from 1. is the answer here.

Yes. Even though these machines are "smart" they still "think" in binary where 1="yes" and 0="no". Without getting into the hex part of the registry, most simple "true/false" things still drop back to that basic binary function. If "PortableOperatingSystem" is set to "1", then it's essentially telling Windows that "Yes, this IS in fact running off a USB drive". If it is set to "0", then it's telling Windows, "No, this is not portable but is instead an internal hard drive".

In short, I agree. The answer is to set it to 0.

gokul t say:

Hi dan.clayton,

 

Before sharing a printer that is hosted by a computer running a 32-bit operating system with a 64-bit operating system, there are specific steps to be performed.

 

First, you need to know the name of the computer which hosts the printer (32-bit Windows Vista, in your case), and the name of the printer to be shared.

 

Do the following:

 

Sitting at the host computer (32-bit Windows Vista):

 

Click Start > Computer > right-click and select Properties > Computer Name tab > Change button.

 

Make a note of the name that you see here -- this is the computer name.

 

Now you need to get the printer name.

 

Start > Settings > Printers and Faxes > Right click on the printer to be shared > Properties > Sharing tab.

 

Make a note of the name that you see here -- this is the printer name.

 

Now, do the following on the 64-bit computer:

 

Start > Control Panel > Printers > Add printer > Add a local printer > Click Create a new port’ > Local Port > Next.

 

The port name will look like this: \\Computer name\printer name

 

Enter the names already retrieved (on the host computer), ensuring that you use the same format, then click OK.


Additionally, you may also refer the following Microsoft article for more information on networking home computers running different versions of Windows:

 

Networking home computers running different versions of Windows

http://windows.microsoft.com/en-us/windows7/Networking-home-computers-running-different-versions-of-Windows

 

Hope this helps!
