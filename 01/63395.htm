You will need access to a computer running Windows 7 with the same architecture (32/64) on which the update was successfully installed.

  1. Extremely recommended - Create a system restore point to rollback if something bad happens.
  2. On the broken computer, take ownership & give yourself write access to C:\Windows\servicing, including all files and sub-directories.
  3. cd /d C:\Windows\servicing\packages
  4. Backup files in case of failure -
  5. mkdir ..\backup
  6. move *KB4056894* ..\backup
  7. Copy the files from the working computer -
  8. copy \\working_computer\c$\Windows\servicing\packages\*KB4056894* .
  9. Note - If the computers are not on the same network you can use removable media to transfer the files instead.
  10. Download the offline KB4056894 package from MS Update Catalog.
  11. Install the update manually. The update should now install successfully.
  12. After the computer restarts, go to Windows Update and check for updates again - Windows Update will recognize that the update has already been installed and will remove it from the list.

Hope it works for you.

davidahart say: If you have recently reset Internet Explorer, than you need to re-enable the microsoft add-ons.

Resetting Internet Explorer disables all Internet Explorer add-ons, including key Microsoft add-ons used by Microsoft Media Player.  Re-enabling these allows the Media Player internet components, such as Album lookup, to function properly.

1.  Close Windows Media Player
2. Open Internet Explorer
3. Click the Gear icon i.e. Tools.
4. Scroll down to Internet options and click on it.
5. Click on the Programs tab.
6. Scroll down to Manage Add-ons and click on it
7. Scroll down to Currently Loaded Addon-ons under Show, click on the drop-down, select All add-ons
8. Scroll down and find the Add-ons under Microsoft Corporation.  Enable all of the add-ons under Microsoft Corporation that are disabled.

This should correct the problem.
