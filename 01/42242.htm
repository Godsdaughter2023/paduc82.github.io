olidagenais say:

This happened to me, too and it is indeed the most infuriating user experience.

Thankfully, the Windows 10 auto locks every 30 minutes even when system is not idle thread did have a solution for me: disable the lock screen slideshow.

It turns out this issue is documented! Windows spotlight on lock screen may cause devices to spontaneously sleep or lock says:

Because of a timing condition, the LockScreenContentServer.exe process could "leak" (fail to close) when the device is unlocked. In this situation, it will continue to send a monitor-off command to the OS at the configured screen timeout while it's running. If you are experiencing a problem like this, you can verify the root cause by checking whether any LockScreenContentServer.exe processes are still running after the device is unlocked. (To do this, start Task Manager, click More Details, click the Details tab, and then sort the Name column.)

...and killing the process running LockScreenContentServer.exe is how I stopped the cycle of "scheduled locks" on my system.

The knowledge base article also has this to say regarding resolution:

  • The issue is fixed in the Windows 10 Anniversary Update.
  • [if you can't get the fix] you can work around this issue by using a static image for the lock screen.

Hope that helps!
