ramesh.kumar say:

Hi Kristen,

 

Thank you for posting your query on Microsoft Forum.

 

Error Code 43 is a Device Manager error that means the following:

 

Windows has stopped this device because it has reported problems. (Code 43)

When you get this error with USB devices it’s because one of the drivers controlling the device notified the operating system that the device failed in some manner. This can be because of corruption in one of the loaded drivers and the driver cache just needs to be flushed. Unplugging the computer sometimes doesn’t resolve the problem because the battery still holds power. To fix this, unplug all of your USB devices, then turn off the PC and take out the battery (This is assuming that you’re on a laptop), let the PC set for about 5 minutes, then put the battery back in and restart the PC. Next, plug your USB devices back in one at a time and make sure that they work.

 

Try connecting the Ipod Nano to a different USB port and check if it works.

 

Also, run the following Microsoft Fixit to get the problem automatically diagnosed and fixed.

 

Diagnose and fix Windows USB problems automatically

http://support.microsoft.com/mats/Windows_USB_diagnostics

 

Additional Information:

 

You can also refer to the steps mentioned in the following Microsoft Article.

 

Error codes in Device Manager in Windows
http://support.microsoft.com/kb/310123/en-us

 

Code 43: Windows has stopped this device because it has reported problems

http://technet.microsoft.com/en-us/library/cc725873(v=WS.10).aspx

Close all open windows and click Start then type or paste this into the Search programs and files box,

%LOCALAPPDATA%

There should be a file called IconCache.db listed (e.g. IconCache  <Data Base File>). Rename it or delete it.

Restarting the computer will ask Windows 7 to rebuild the IconCache.db file.

What is system folder? - Microsoft Community

prudhvi ch say:

What is system folder?

why is the system folder empty(local disk :C>windows>system)......i was just tinkering and i noticed this strange problem :/ .......my os win 8 help will be appreciated :)
cmdrkeene say:

It is supposed to be empty.  The only reason it is even there is just in case some older or badly-built program tries to find it.  Basically, for compatibility reasons so old/bad programs don't crash.

There's a whole lot of this in Windows actually.  For example, years ago (back in 2006), the "Documents and Settings" folder was renamed to "users" because it's easier, doesn't have spaces, and other reasons. But some programs are built to look for "Documents and Settings" instead of using the correct method of detecting the program.  Those bad programs would crash if you installed them on another language because (duh) other languages don't spell the folder the same as in English.  So to try to keep those bad programs from crashing, there's all kinds of things like this.
