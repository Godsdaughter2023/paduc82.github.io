bossdweeb say:

What brand and model cpu graphics processor are you using, and where are you forcing how many cores are used ?

Games will normally use as many available cores as they are coded to use without any user input.

There are a few games that were coded for dual core that need fixes to the game configuration files when running more than

dual core cpu's as well (eg- Fallout 3) or they will crash, but most don't need extra cores to be forced.

You use the term Priority, but this is more commonly used for the option in TaskManager>Processes to force Windows to direct more system resources on the process in question, and does not directly affect the number of cores used by that process.

- Selecting the number of cores (processors) in msconfig is not recommended for general use. This option is intended for troubleshooting.

Generally, if you want to force a game to use four rather than one or two cores it's best done in the games .cfg or .ini files.

If you are forcing four cores here (from what I see Civ6 benefits from 4 cores but no improvement with 8), you may need to change

the file to Read Only to prevent the game from changing it back to the default, which commonly happens as the game is loading,

or shortly thereafter with games that don't normally make an allowance for modified configuration files.
