Symptom #2:

A variation of the first one.  System boots and loads the Desktop.  At some point in using the computer the screen goes black.  Again similar results as with Symptom #1.  Even if the system recovers from the Black Screen, it is sluggish for at least several minutes if not indefinitely.

Symptom #3:

System boots and runs fine.  This may be shortly after booting or even after running for along period of time.  User attempts to type in the Cortana Search and it won't type but mouse tracks normally.  It can take several seconds to even get a character to type, usually frustration sets in and reboot the computer.  Note the system may have worked perfectly fine for several minutes, but it is almost like the system goes into slow motion.  Note this has occurred on SSD systems, Latitude 5414 and Latitude 7480 systems.

Symptom #4:

System boots and runs fine:  But when user clicks on applications, the computer doesn't show any sign that the application was selected.  It can be two to up to five minutes later the application loads.  Again this occurs on both Conventional and SSD hard drives.

We do tend to keep the BIOS up to date, and on a system that had no symptoms, I upgraded the BIOS and immediately this system exhibited the same symptoms that have been randomly affecting other computers.  I downgraded the BIOS and the symptom cleared up immediately.

---------------------------------------------------------------------

Things that seem to reduce or even seem to fix the situation:

On systems with conventional hard drives (Symptoms 1-2) any one of the following changes seems to greatly reduce or seemly fix the issue.

1.  Remove Cylance from the computer

2.  Disable the SysMain service (SuperFetch)

3.  Replace the Convention Hard drive with an SSD drive

4.  Downgrade the BIOS from a current version to an older version

On systems with SSD drives, Both the Black Screen and the slow application load can still occur.  

And changing the BIOS may impact the symptoms on SSD based systems but even downgrading or upgrading the BIOS hasn't been a consistent fix for them.  But fear we are possibly reducing the security fixes needed to protect from Spectre/Meltdown in order to get usable systems.

Since I am dealing with several models, validating each issue has been challenging.  I can easily blame the BIOS update, Cylance Antivirus, even the type of hard drive and not know the real cause.
