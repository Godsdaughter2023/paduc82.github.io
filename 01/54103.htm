bill smithers say:

Hi,

The Windows 7 system image is for the Windows 7 machine, if you ever need to reinstall the operating system on the event of total system corruption or have a hard drive failure.

You should have done that System Image, complete PC backup, when 7 was running correctly to save yourself time doing a reinstall as it contained Windows updates, installed programs, settings, drivers, etc. etc.

That's what a system image is for.

@@@@@@@@@@@@@@@@@@@@@@

And there are other things to consider if you wanted to transfer a License, via a clean install of 7 to 10.

If the Laptop/Computer came with the Windows Operating System preinstalled (Dell, HP, Acer, etc) , that Product Key that came with the Laptop/Computer is for the preinstalled OEM License and is non-transferable.

The OEM License/Product Key (preinstalled Operating Systems on Dell, HP, etc machines) is tied to the first machine it is installed on and dies when that machine dies.

The above rule also applies to OEM System Builders Licenses.

Retail Operating System License, that you buy and install yourself, are allowed to be transferred to another machine.

But Retail Licenses can also only be installed on one machine at any particular time.

ken - say: Hi QuickHare

The USN journal is a fixed-size log that records all changes to NTFS 5.0-based partitions. Some applications use the USN journal to track changes that occur on the file system.

Deleting the change journal impacts the File Replication service (FRS) and the Indexing Service, because it would require these services to perform a complete (and time-consuming) scan of the volume. This in turn negatively impacts FRS SYSVOL replication and replication between DFS link alternates while the volume is being rescanned.

Hi, DaGreenOne

The problem is not so much with a script to disable it, as it's easy to run a shortcut on start-up from the Startup folder of the Start Menu to solve this. The problem is it resets the data and clears it out, not disabling the USN Journal.

Each time you write to a file, the USN Journal is updated with how the file changed. The amount of data stored per alteration is very small per file, but if you think that Windows alters many files in the course of a session, the user alters files and if you use any download manager (such as a file-sharer, the BBC iPlayer service [UK Only] or a download 'accelerator'), if you browse pages (which are saved to disk temporarily) or just a keen photographer with many digital snaps you wanted to touch up, then the number of entries becomes quite large.

Over time the records are so large in number that Chkdsk (or scanning a disk for errors) will show the USN Journal as erroneous, metafile data on the disk needs defragmenting and your computer has to spend time removing old entries (which it does only when you start the computer). It does state in the Knowledge Base that although the USN Journal has a maximum size set, Windows will exceed the maximum during a session. Older entries will be removed only on start-up as if anything goes wrong with the drive, a full backup of changes are stored.

The extra disk writes to the hard drive to update your file, the MFT and then the USN Journal reduces the performance for what Windows XP users saw as very little benefit. I still don't see much benefit for Windows Vista. Most back-up software (which Microsoft tend to assume uses the USN Journal to operate) mostly uses a comparison method based on a CRC check or takes a complete drive image rather than use the USN Journal, which may or may not exist on that Windows system. Remember the USN Journal is only for NTFS 5 partitions. Any NTFS 4, FAT32 or Unix partitions cannot have a USN Journal, meaning the back-up software wouldn't work as well. For this reason many back-up software developers do not rely on the USN Journal as Microsoft suggests.

So basically I'm finding a way to disable the USN Journal for good. However logging back in as my usual account (part of Administrators, but not the Administraor account itself) somehow turned it back on.

If it is turned off as expected, then if you need those other services - without having the INDEXING Feature on - you should be able to re-enable those services - FRS, RIS and Remote Storage from the Services.mmc.
