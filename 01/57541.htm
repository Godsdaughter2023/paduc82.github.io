Hello Borate,

 

There is no way for us to disable error 69 reporting from the Event Viewer. However, we can create a custom filter for the event logs and hide error 69 from the report. You may follow these steps, how to:

 

  1. Open Event Viewer. Under Windows logs, click System.
  2. On the Action tab, click Filter Current Log.
  3. You can now filter error 69 using Filter current log. On the screenshot provided, I filtered the error 69. I type in 1-69, and starts again with 70-100000 so it will not count in error 69. Once youâre done, click Okay.
  4. Click Save Filter to Custom View. Add the name and description then click OK.

 

Hiding or stopping Event Viewer from reporting Error 69 may not address the root cause of this issue. We would highly suggest that you run the System File Checker (SFC) and Deployment Image Servicing and Management  (DISM) scans to check for missing or corrupted system files, if there are any. To run the said scans, follow the steps below:

  1. Press Windows key + X
  2. Click Command Prompt (Admin)
  3. Type the following commands, one after the other completes the scan:
    sfc /scannow
    dism /online /cleanup-image /restorehealth

If the results of SFC or DISM scan would state that it found corrupted files and repaired those, you would need to restart your computer to apply the changes. Afterwards, check if you would still encounter the Error 69 from Event Viewer which you were getting.

Turns out the thing I was asking above (as expected) does indeed have a way to be controlled via the registry without having to go through the folder options and **** every other folder of that type that you may already have.

go to this registry key HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Streams\Defaults

here by default you'll have no entries on a fresh install or after a major update, but if you change any folder template, it will make a new entry here that will be used for any future folders of that type that you will open. I'm guessing windows wipes this clean after every major update and hence why the above mentioned problem kept re-appearing after every one of them.

Every entry here will be named something like "{5C4F28B5-F869-4E84-8E60-F11DB97C5CC7}" each representing a folder type (documents/photos/videos/etc.) being of type REG_BINARY and containing a long -almost unreadable- block of data in hex that I presume contains the actual view settings.

Sadly I can't provide you with the actual folder types {gibberish} codes and the content for every view type. You'll have to find that yourself. I suggest doing that by basically biting the bullet and using one last time the folder "options>view>apply to folders" method to actually generate the entries you need and then writing them down or exporting them for later use.
