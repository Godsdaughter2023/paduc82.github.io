A programer is a digital artist.

Just had this problem and this fixed it for me. I wanted to add a few notes in case it is useful to others having this issue... (Sorry, I can't tell if your blog entry addresses any of this as it is not in english...)

Getting to this folder can be a little tricky. Pasting "%APPDATA%\Microsoft\Windows\Recent\AutomaticDestinations" into the run box, which would normally work, won't in this case. Neither will pasting it into the target of a shortcut. Additionally, you won't find it if you try to normally browse to it as the "Recent" folder shows up as "Recent Items" and displays the special Recent Items folder in Explorer if you browse to it, rather than the true contents of "Recent". So you must get there by typing the full path (without the %appdata% variable) into an Explorer address bar or a run box. If you don't know the full path of %appdata% you can get there in 2 steps by typing %appdata% into an explorer address bar, then once you get there, adding " \Microsoft\Windows\Recent\AutomaticDestinations" to the end of the address shown in the address bar.

I didn't find it necessary to reboot. The changes were immediate. I only tested the ability to pin things to the jumplist. It's possible the recent files requires a reboot but I'm not convinced that that is necessary

It is possible to fix just Explorer (or any application) that does not have a working jumplist without losing any changes you made to jumplists of other applications. Here's how I did it:

Rather than delete the files in this folder, MOVE them to a temporary folder somewhere else. All the jumplists of all the applications will now be empty. Now pin something to the jumplist of the application you want to fix (in my case Windows Explorer). 1 new file should appear in the folder. You now know the name of the file controlling the jumplist of that application. DELETE this file from the temporary folder where you had moved everything. Now move the remaining files back to their original location. This restores the jumplists of all the other applications so you will only have to rebuild the one.

If you make regular backups of your system drive, you can also restore the files from this directory from a backup made before you lost the jumplist, in which case you won't have to necessarily reconstruct the list. In fact, if you use the above procedure to determine which file needs to be fixed, you should be able to restore just that one file. I have a Windows Home Server that does daily backups, so I restored the 1 file from yesterdays backup and everything is now working again as it was.

Incidentally, on my computer the file for Windows Explorer is "1b4dd67f29cb1962.automaticDestinations-ms". Not sure if this is the same on all systems, though...
