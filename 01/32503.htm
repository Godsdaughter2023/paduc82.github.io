myronh say: I've got most of it. Still looking into restricting access to command prompt and control panel. I think they can be done in policy editor but not sure right now. Everything else can be done without any real problems...to include no changes to the favorites. Here's how:

First, create your accounts. Admin account is, of course, the first account. Set the password on the account. Once both accounts are completed, log into the standard user account. This needs to be done to create the files and folders that are part of the account. You might also want to open the IE. When I tested (I didn’t take ownership yet by the administrator so I could do in the user account) I had to change the favorites folder permissions. This might be fixed if first you have opened IE.  Once in the account, log out and log back in as the administrator.

Now, to limit the standard user and protect the disk, take owner ship of the standard users folders. You do this by right clicking on the folder and select properties. From properties, click on the security tab, click advanced, and click the owner tab. Click edit, select your administrator account click OK on the pop-up window (it’s telling you that you have to exit and re-enter for the owner change to take effect).

Now it’s time to change the permissions on the folder. Again, right click the folder, select properties and the security tab. Click on Edit, select the user account, and turn off anything you want to turn off. Go into the user folder and right click AppData (it’s a hidden folder so you will have to unhide first. Do this in control panel, search “hidden” and click on show hidden files and folders link. Then just hit the radio button to show hidden files/folder) and allow full control. They need full control for the applications to work properly. Also, this would solve a potential E-mail client issue.
