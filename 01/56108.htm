bell john say:

How to jump from windows 7 to windows xp without restarting the pc, if 2 operating systems are installed in a pc.

Operating systems are huge and increasing technology applications. so, if we use both the windows 7 and windows xp sp2 at a time. how to switch from one operating system to the other.





elizabeth23 say:

Consider an OS like a kid and the computer hardware the kid's toys. If there is only one kid, then that kid can easily decide which toys the kid will use. If you introduce a second kid into the equation, you now need to have some "mechanism" to decide which toys which kid can play with. If you put one kid in the room with the toys and lock the other kid in a second room without the toys, then this is like the dual boot option. If you have a "dominate kid" (i.e. say a big brother) that basically gets to decide which toys that kid will play with and which toys the "subordinate kid" will play with, then that is like running a primary OS with a secondary OS running in a virtual machine. If you have two kids with no mechanism to decide who gets to play with which toy, then the kids fight over the toys.



The point is that you have to have SOMETHING that decides which OS gets access to what computer resources. This is the primary purpose of an OS...it allocates/grabs computer resources and decides which programs, etc get what resources. So, you have to have either only one OS or you have to have a dominate OS that decides this. The dual boot option is basically having one OS running at a time. Thus, that one OS decides which programs get the resources. In the case of a virtual machine, you will have a primary OS that ultimately "doles" out the resources and it is the virtual machine software that puts in requests for resources for the secondary OS to use. Think of the virtual machine as the ambassador/intermediary to the primary OS from the secondary OS.



The only other way would be for some very rudamentary OS to be running that then allows multiple OSs to run on top of it. This would be much like the virtual machine option, but the rudamentary OS would ONLY be about doling out resources. I am not aware of any such system for a desktop environment.

lemp say:

I am also not able to install the necessary Microsoft drivers to connect my Canon i9100 printer.  Can you direct me to the appropriate driver site?  I have a windows 7 64 bit operating system.

When Canon says that "Your printer must be installed via Microsoft Update before you download and use this module," that does not mean that you need to run Windows Update (or Microsoft Update). 

The advice from the Support Engineer is not correct:  Windows Update will not find a new driver for a device that is not yet installed.

A,  Normal Method

What the message means is that your computer needs to be connected to the Internet when you attempt to install the Canon i9100.  When you connect the printer, Windows should search for a driver.  When it can't find a driver locally, you should get a pop up message asking for permission to connect to Windows Update to search for a driver.  You should allow this.  Windows should then find the driver online, download it, and complete the installation.

If that doesn't work, you can obtain the driver in one of two ways.

B.  Manually through Windows Update.

  1. Disconnect the printer from the computer
  2. Click Start > Devices and Printer
  3. Click "Add a local printer"
  4. Click "Use an existing port" and leave the default of LPT1, then click Next
  5. Select Canon in the list of manufacturers
  6. Confirm that there is no entry for the i9100 in the list of printers and then click the "Windows Update" button
  7. Wait ... this may take quite a while ... when it's done, confirm that the i9100 is now listed
  8. Cancel out of the printer install wizard
  9. Connect the printer to a USB port and it should now install properly

C.  From the Windows Update Catalog

Although the above methods should work, you can also got to https://catalog.update.microsoft.com/v7/site/Home.aspx using Internet Explorer only and search for "Canon i9100".  You should get 5 hits.  Click on each of the drivers that are for Windows 7.  Look at the line that says "Architecture."  If you have Windows 7 32-bit, you need the x86 version.  If you have Windows 7 64-bit, you need the AMD64 version.  The procedure for installing a driver obtained this way is a bit complex.  If methods A or B don't work, post back for additional instructions.

SRX randomizes source ports on source NAT to counter DNS poisoning attacks (among other things). This confuses XBL.

"set security nat source port-randomization disable" doesn't help.
You need to create a source NAT pool containing your public IP address and set "port no-translation" in the pool.

This worked for me to get Open NAT on a SRX100H running Junos 12.1X44-D40.2.

Unless you have a static IP you'll probably want a SLAX script to update the config whenever your public IP changes.
