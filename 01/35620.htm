Can I perform a clean install using the Free upgrade?

No, it will require that you are running a previous qualifying version and start the upgrade from within the qualifying version. You can initiate a clean install after completing the Upgrade.

Please be aware that you cannot use the free upgrade offer to perform a clean install on first attempt. You must first upgrade from the qualifying version of Windows 7 or Windows 8/8.1 (whether you do it through Windows Update or using the .ISO file). Ensure the upgrade is completed successfully and then ensure that it is activated. You can then proceed to do a clean install by using recovery media or using the Reset function in Start > Settings > Update & Security > Recovery > Reset this PC (Get Started).

daavee say: You have a number of issues/questions!

Firstly, for best results, run chkdsk WITH THE /R SWITCH from Recovery Console.

Since you do not have a genuine XP installation CD, please refer to this post:

http://answers.microsoft.com/en-us/windows/forum/windows_xp-system/my-dimension-e521-wont-boot-up-windows/4a239481-4f64-4353-a049-69374beeeb9d?msgId=f177fb81-26f0-4412-9b61-5fb63847dc1c

Here are the specific instructions, courtesy of wunderkind ElderL:

You need to start by booting into the XP Recovery Console using a CD you can make (no XP media required) and run a chkdsk with error correction enough times until it runs clean and if that doesn't work, you can create a new boot.ini file (it is usually one of those two things).


Here's how:


Use the XP Recovery Console to verify the file system on your HDD and correct any problems and then try to boot your system - this may be all you need to do.  Or, you are welcome to just start trying things that might work.


Boot into the Windows Recovery Console using a bootable XP installation CD.

If you have no bootable XP media (or are not sure what you have) create a bootable XP Recovery Console CD and be sure what you have (no XP media required).


This is not the same as any recovery disks that might have come with a store bought system.


You can make a bootable Recovery Console CD by downloading an ISO file and burning it to a CD.


The bootable ISO image file you need to download is called:

rc.iso 


Download the ISO file from here:

http://www.thecomputerparamedic.com/?p=8


Use a new CD and this free and easy program to burn your ISO file and create your bootable CD:

http://www.imgburn.com/


When installing ImgBurn, DO NOT install the Ask toolbar.


Here are some instructions for ImgBurn:

http://forum.imgburn.com/index.php?showtopic=61


It would be a good idea to test your bootable CD on a computer that is working.

You may need to adjust the computer BIOS settings to use the CD ROM drive as the first boot device instead of the hard disk.  These adjustments are made before Windows tries to load.  If you miss it, you will have to reboot the system again.


When you boot on the CD, follow the prompts:


Press any key to boot from CD...

The Windows Setup... will proceed.

Press 'R' to enter the Recovery Console.

Select the installation you want to access (usually  1: C:\WINDOWS)

You may be asked to enter the Administrator password (usually empty).

You should be in the C:\WINDOWS folder.  This is the same as the 

C:\WINDOWS folder you see in explorer.


The Recovery Console allows basic file commands like: copy, rename, replace, delete, cd, chkdsk, fixboot, fixmbr, etc.


For a list of Recovery Console commands, enter help at the prompt or read about the XP Recovery Console here:

http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/recovery_console_cmds.mspx?mfr=true


A good idea before starting things is to first verify the integrity of your file system using the chkdsk command.


From the command prompt window run the chkdsk command on the drive where Windows is installed to try to repair any problems on the afflicted drive.

Running chkdsk is fine even if it doesn't find any problems.  It will not hurt anything to run it.


Assuming your boot drive is C, run the following command:

chkdsk C: /r


Let chkdsk finish and correct any problems it might find.  


It may take a long time for chkdsk to complete or it may appear to be 'stuck'.  Be patient. If the HDD light is still flashing, chkdsk is doing something.  Keep an eye on the percentage amount to be sure it is still making progress.  It may even appear to go backwards sometimes.


You should run chkdsk /r again until it finds no errors to correct.


Remove the CD and type 'exit' to leave the RC and restart the computer.


You do not have to adjust the BIOS again to boot on the HDD since the CD will not be present.

Regarding .Net frameworks...


They are OPTIONAL. They may be uninstalled. Uninstalling them won't hurt Windows. If there is an issue, by all means use that removal tool (that's what it's for!).


In the event any of your programs needs a framework now or in the future, then you may install THAT particular framework.


From this point forward, as a rule, stay away from ALL optional updates (especially "hardware" updates!!!) in Windows Update with the exception of Root Certificate updates.

how can I find my user name to home group so IO can access files on my - Microsoft Community

bm 995 say:

how can I find my user name to home group so IO can access files on my PC from my laptop win.7

How can I find my user name to my home group so that I can access files on my PC from my laptop - Windows7 PC Windows 10

hairyfool say:

Your user name is not relevant to HomeGroup Access across the network.

When you create a HomeGroup 2 additional users are created on each PC, all HomeGroups have the same 2 user names but each HomeGroup has its own password assigned ( the one you create when you set up the group in the 1st place). This prevents somebody setting up a HomeGroup on their PC then connecting to another group to gain access.

These 2 users are in turn placed in 2 Administrative groups and it is these Groups that define if the user has read only or read write access. Because the same user exists on all PCs in the HomeGroup with the same password rights are transferred.

These users do not exist as logon users visible in the Control Panel > Users tool. Instead in order to find these users you need to open the Computer Management utility. Under System Tools > Local Users and Groups you will find a number of Users that do not exist as logon accounts.

hairyfool say:

1st off 2 profiles are used because you can assign two different access levels to folders, Read Only and Read/Write.

Best practice for Windows networks is to assign rights to a group rather than to an individual user. That way you can give any number of users the same rights by placing them in a particular group. Not entirely relevant in a HomeGroup but in a domain where users come and go, change roles etc. it makes it easy to manage. So we have two groups and two users to place in those groups. You can see this mechanism in action by looking at the Security settings for one of the shared folders.

In a basic Workgroup environment with say 5 PCs with each having their own primary user that is 5 user accounts. For all users to be able to access files on every other PC from their own that means that not only does the user account have to exist on their own PC but it has to be created on all other PCs. This includes management so if a user changes his password on his PC then he has to go round each of the other PCs and change the password on those or else he will be declined access.

HomeGroup prevents this by creating an additional user which exists on all PCs and has the same password throughout the group because it is defined in HomeGroup setup. Although the primary user on one PC is declined by another PC it will accept the credentials of the HomeGroup user. Which of the two users is accepted depends on the required access.

So although the mechanism seems a little convoluted (and there is more to it) there is some logic to it.
