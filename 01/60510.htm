dominicp say:

I am not familiar with Google Apps for Business but understand that it keeps contacts in separate folders.

Outlook and Outlook.com have a single contact list but each entry can be assigned one or more Categories (Groups in Outlook.com). Outlook Categories and Outlook.com Groups will synchronise but the Group names need to be created in Outlook.com.

With Outlook.com and Google (and IMAP mail) the data is in the cloud and Outlook works with a local copy: an ost file stored at C:\Users\<username>\AppData\Local\Microsoft\Outlook. If this is damaged, you can close Outlook, delete the ost and Outlook will re-download the data next time it opens and recreate the ost file.

An Outlook pst is stored on your system and you have to ensure its preservation. Asking Outlook to move data between a pst and a cloud store will only work if the Outlook interface with the cloud allows it. Changes in Outlook's local copy of cloud data are synchronised at short intervals and can be forced at any time with Send/Receive All Folders (F9 is the shortcut).

You can work with cloud sources and copy or move cloud data to a local pst if you want.

You can run Outlook.com and Google in parallel before you decide which to prefer.

While Outlook 2010 will work with the cloud newer versions of Outlook are better integrated. You could try the Outlook 2016 Preview .

Hi Larissa,

I need more information about this issue so that we can assist you better.

1. Which version of Windows 7 Operating System are you using?

2. Do you get any error code or error message?

If you are using Professional and ultimate version of Windows Operating System, then you will be able to edit the Group Policy and Administrator tools.

You should disable automatic driver installation. To do this you don’t need to install any software at all, you can do this using Windows Driver Settings.

Properly disable automatic driver installation.
• Go to Start then Search type in gpedit.msc.
• Click the file to open the Local Group Policy Editor and show Windows who is in control!!
• You want to go here: Computer Configuration then Administrative Templates then System Device Installation. Click on the subfolder Device Installation on the left and on the right side you will see the possible restrictions.
• Right Click on Prevent Installation of Devices not described by other policy settings and edit this option, set it on ENABLED.
Reboot Windows and enjoy its inability to pollute your system with its standard driver, open gpedit.msc again and revert the change so you will be able to install your driver.
Now your Windows won't automatically download drivers.

As reference follow the link.

http://windows.microsoft.com/en-us/windows/turn-automatic-updating-on-off#turn-automatic-updating-on-off=windows-7
