This error message indicates that the VBScript engine is not currently registered on the machine where the install/uninstall is taking place.

 

You may try these steps:

The VBScript engine can be quickly registered by following these steps:

a. Click Start, type cmd, right click cmd and select Run as administrator

b. If you are using 32-bit version of Windows type:

cd %windir%\system32 into the Command Prompt and hit Enter.

c. If you are using 64-bit version of Windows:

cd %windir%\syswow64 and hit Enter.

regsvr32 vbscript.dll into the Command Prompt and hit Enter

regsvr32 jscript.dll into the Command Prompt and hit Enter

a. user say:

Hi Barbara,

 

I see that this is your first post in this forum, welcome to the Microsoft Community.

 

From what you have described, it appears that you don’t have sufficient permissions to save the file in the location or the file you are trying to save is protected and users are allowed only to read the files and not save them.

 

Let’s first check if the ZIP file you are trying to save is protected file. When you right click on the ZIP file, click on the option to open the file instead of saving it. If you are able to open the file successfully, then most probably the file is a protected file and the owner of the file has set permissions to the file so that the recipient of the file can only read the file, but not make any changes to the file or save that file.

 

If you are able to open the contents of the ZIP file, then you can try extracting the contents of the ZIP file to a location in the computer. Most of the ZIP programs has an ‘Extract to’ option. Click on this option and choose a location to save the file in your PC.

 

Sometimes, this error may be displayed if the user doesn’t have sufficient permissions to save a file in a particular location. In this case, when the save window prompts you to choose a location to save the file, choose a different location and check if you are able to save the file. If you get the same error message while trying to save in other locations as well, then you can try these steps:

 

a)      Log into an Administrator account and navigate to the folder where you are trying to save the file.

b)    Right-click the folder that you want to take ownership of, and then click Properties.

c)     Click the Security tab, and then click OK on the Security message, if one appears.

d)    Click Advanced, and then check against the owner field. If you user name is not listed, click on ‘Change owner’ link.

e)     In the Name list, click your user name, Administrator if you are logged in as Administrator, or click the Administrators group.

If you want to take ownership of the contents of that folder, click to select the Replace owner on subcontainers and objects check box.

f)      Click OK.

You may receive the following error message, where Folder is the name of the folder that you want to take ownership of:

 

You do not have permission to read the contents of directory Folder. Do you want to replace the directory permissions with permissions granting you Full Control? All permissions will be replaced if you press Yes.

 

g)      Click Yes.

 

Try these steps and let us know the results. We’d be happy to help if you need any further assistance.
