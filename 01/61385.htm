elrumbo say: Tripledz,

Don't worry about devices using the same IRQs or ports. The specifications of PCI, Firewire, USB PnP devices let this kind of devices to share IRQs. Now a days, all devices are PnP compliant which in other words means Plug and Play (not like in the old days which was Plug and Pray because there were so many legacy devices that were not PnP compliant). Unless, you are using many old devices (many not just a couple), you could potentially have a conflict. In the old good days of hardware, many devices had jumpers or DIP switches that needed to be set to avoid conflicts. Not anymore. PnP takes care of that for you.

javier_martlnez say:

Dear, use this trick.

Example: C:\tools\path_too_long\etc1\etc2\etc3\etc4...\etc.n...

1. In this example, "c:\tools\path_too_long" has an very very looong tree of folders under it. I want to delete all of that.

2. Lets create a new "empty" folder. This will be my "template". : c:\tools\myEmptyFolder. This folder has nothing inside it.

3. Open Command Prompt, and use robocopy /MIR  to make the trick. robocopy can handle long path and delete everything under the long path:

robocopy c:\tools\myEmptyFolder c:\tools\path_too_long  /E /Z /V /R:0 /W:0 /MIR

Result: After the process, c:\tools\path_too_long is totally empty. Robocopy forces the destination to be a "mirror" of the empty location "myEmptyFolder".

Robocopy is included from Windows Vista, so you don't need to download anything. It Works directly from command prompt (cmd).
