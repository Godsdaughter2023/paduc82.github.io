Every third reboot, I get the repair options (after Windows 'helpfully' tries to automatically diagnose the same problem for the 8th time today). So very slowly, I worked my way through the tools I had, not wanting to wipe the drive on account of the data I had stored on there. So Day 1 was mostly trying these things. Startup repair failed. I had no system image to load. I popped open the recovery command prompt and tried everything I could, from checking the disk (fine) to rebuilding the BCD (no change) and tried to repair the boot sector, which changed nothing. I could still open directories on the disk, and the files were all there and fine, so I couldn't fathom it not being a boot sector or Windows corruption issue. So I gave up on my files and reinstalled Windows manually (the 'Reset your PC' option also failed, some-frigging-how since it's just a Windows install right?), and got set up and logged in. (I wrongly assumed at this point the SSD was failing, and ordered a new one. Day 2 I did not try anything.)

Day 3, the new SSD gets here. Plug it in, install Windows 10, get exact same result after restarting after my first login. At this point I turned to my SATA cables. At this point we gave up for the night, mostly stumped on the issue.

sam_pw say: We ended up finding the issue was with the graphics card finally dying off. Whenever the drivers auto-installed, the boot loop would occur.
