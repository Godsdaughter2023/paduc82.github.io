cmdrkeene say:

Unfortunately I don't Kalle's issue itself is preventable.  If a program opens up a window, it gets to decide where to place itself on the screen as a coordinate of X,Y position.  A lot of programs will not expect part of the left edge to be used by the taskbar, so it will use a X value of 0 (so the window touches the left edge of the screen) or a low value (so the window's left side is close to the left edge, even if that happens to be "under" the taskbar).  If the window already exists, resizing the taskbar will nudge it over, but that only applies when you are resizing the taskbar, not when the window is opened.

But Chris's issue isn't supposed to occur.  If you have the taskbar set to auto-hide, it should disappear and allow you to reach and click the window zoom controls (not really called a sizer gadget, but now I know what you are talking about).  Usually when this happens, turning the auto-hide function off and back on will clear the state so the taskbar disappears out of the way and stops blocking that control.

ravelin say:

I had a similar problem, or rather two similar problems - I maintain 2008 (vcproj) and 2010 (vcxproj) project files for a single project, and frequently open either file in the appropriate version of Visual C++ Express.  Mine does not say "Version Selector" in the jump list or when dragging a file onto the button, so I guess it is different to your case.  (Perhaps because I pinned them while running rather than dragging a shortcut, or just because they're Express Editions?)  However, the vcproj and vcxproj files were associated with the Version Selector.

At some point I tried associating vcxproj files with VCExpress.exe (2010) via Open With -> Choose Default Program, but this didn't seem to help.  It also changed the icon to a default white page with a tiny VCExpress icon in the middle - I was not impressed.  The only way I found to revert it was to delete the following registry key:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.vcxproj\UserChoice

I then changed the default program(s) for vcxproj and vcproj files as follows:

  • Open Regedit
  • Find HKEY_CLASSES_ROOT\.vcxproj (or the appropriate key).  The default value for this key names another key which contains the actual settings for the file type.  In my case, it was VCExpress.Launcher.vcxproj.10.0 .
  • Find the key named in the previous step, under KEY_CLASSES_ROOT.
  • Open up the shell\Open\Command subkey.  You may want to export this key as a backup before making modifications .  (I didn't.)

To get to event viewer in Windows 8:
 
a) Press Windows Key + X Key, Click on Event Viewer.
b) In the left pane, expand out Windows Logs.
c) Click on Application log. Highlight the first event in the log and use your arrow keys to scroll down. Most of your event will be Information. You may see Yellow Warnings or Red Errors. If you see a red error, you can double click on it to bring it up and copy the contents to a document.
