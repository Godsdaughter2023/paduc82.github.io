john john __ say:

Oh boy...UDF packet writting, aka Drag & Drop!  Don't get rid of the Windows 98 hardware until you are sure that all of his files on CD can be read and retrieved on the Windows XP machine or else they may be lost or difficult to recover!  The problem with packet writing is that while in concept and theory it may appear to be very useful there is no certainty that files written in this manner will ever be guaranteed to be read in anything other than the very same CD/DVD drive and model as that on which they were written. This is a known problem with cds written with this writing method, in the old days it wasn't unusual to see posts on help groups from people seeking help because they could not recover their files.  You change your computer and get rid of the old one, you forget (if you even knew) about your "Drag and Drop" cds and the new computer that you got has a different make of cd drive and all of a sudden all your precious files that you so carefully copied to cd are lost.  Then you have to have recourse of recovery software like ISOBuster to try to recover the files!  It can lead to file loss and major aggravation!

Windows XP has the "Files and Settings Transfer Wizard" that you can use to transfer settings from some microsoft programs, there is not really anything else in the Windows set of tools to transfer the other stuff and forget about copying registry settings from the Windows 98 to Windows XP, while there is a semblance of similarity the two registry are significantly different.  If the transfer wizard cannot migrate some of the programs your user will have to fiddle and play with his applications to set them back to his liking.  The biggest problem is usually email but luckily the Files and Settings Transfer Wizard can migrate Outlook and if he is using Thunderbird there is a profile backup and restore tool available to migrate his profile and emails to the new computer.

These may be helpful:

Files and Settings Transfer Wizard

Using Files and Settings Transfer Wizard (Windows XP)

Step-by-Step Guide to Migrating Files and Settings

Files and Settings Are Not Transferred When You Use the Files and Settings Transfer Wizard

Hi Saber,

The issue you're currently experiencing regarding the computer unable to sleep can be due to the audio driver that is active due to an audio stream. It can be caused by a browser with a Flash content in it or an app that is running a process in the background. You can set the power management to ignore any audio stream so your computer can go to sleep. To do this, open an elevated command prompt and type powercfg -REQUESTSOVERRIDE DRIVER "Realtek High Definition Audio" SYSTEM. Once done, close the command prompt window, restart the computer and check if the issue persists.

Let me know if you'll be able to put your computer to sleep after running the command.
