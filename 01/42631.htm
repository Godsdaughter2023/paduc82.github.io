USB flash drive

ISO file

If you are using Windows 7, you can create the .ISO and burn it using the built in Disc Image utility. If you are running Windows XP or Windows Vista, you can using a third party burning tool such as ImgBurn or Roxio/Nero.

don cuthbert say:

UEFI is the most current mode to set in your BIOS, but a hard disk needs to be initialized as a GPT disk for it to work properly (which is a good thing to do).

If your prior qualifying OS is Windows 7, you cannot use full UEFI BIOS mode, but need some BIOS Legacy settings for Windows 7.

If your prior OS is working still, enter your BIOS configuration (usually by tapping a certain key during boot, such as F12 or F2 - depends on the computer).  First, set BIOS to factory defaults.  There is normally a single key to do this and then this change must be saved before exiting.

You have to upgrade from within Windows 7 or 8 before you can clean install Windows 10 for free.

See if your pre-Windows 10 OS (Windows 8 or 7) still runs OK.  If not, re-enter BIOS settings and disable any UEFI setting that is enabled and try again.  This should appear as a Legacy-mode, which should work for Windows 7 and will tend to make Windows 10 use the disk as an MBR-initialized disk, rather than giving you the errors you are seeing.  See if your current OS works now and if so, the Windows 10 upgrade should work.  I would back up any data that you have not backed up already at this point, as it will be best to completely wipe your hard disk to get it setup in the ideal way.

If you have Windows 8.1, you would be better off to set everything to UEFI BIOS mode, enable Secure Boot and reinstall Windows 8.1.  Activate Windows 8.1 with your Product Key.  This will initialize a wiped hard disk as GPT and create the partitions mentioned in your error message.  Then upgrade to Windows 10 using the Media created by the Media Creation Tool:

http://windows.microsoft.com/en-us/windows-10/media-creation-tool-install#

don cuthbert say:

You can instead try to ensure that UEFI BIOS mode is completely disabled to force a Legacy BIOS mode install (unless that is how it already is now).  If the computer is UEFI capable (but started out with Windows 7, as mine did) it might have had UEFI enabled, but with some Legacy compatibility features enabled in BIOS, as well as the MBR-initialized hard disk.  Maybe some Windows 8 shipped like that too?

You can't convert a drive to GPT unless it is empty (using Windows software).  I have done it by booting with a live Ubuntu disk and running GParted, but it's risky and can cause some problems getting it to work right, as it requires moving partitions containing your Windows and data and creating some new partitions (the EFI and MSR ones you mentioned).

The best way to convert to GPT would be to image your whole C: drive, wipe all the partitions, reinstall Windows 8.1 after changing BIOS mode to UEFI and then restore everything to the new C: drive partition.  That's what I did when I upgraded from Windows 7 to 8 and installed an SSD for my System drive.

You can create Windows 8.1 media (a DVD or ISO to use on a USB flash drive) through this link, so that part will not cost anything.  It can go on a USB flash drive of 4GB or larger.

http://windows.microsoft.com/en-US/windows-8/create-reset-refresh-media

If your computer shipped with Windows 8, you can retrieve your Windows 8 Product Key by opening an Administrator Command prompt and entering this line:

If you prefer to first prove that you can install Windows 8.1 and activate with your Product Key, without getting rid of what you have now, you could use Disk Management to shrink your C: drive by 20GB to make some un-partitioned space.  Then boot with the created Windows 8.1 USB flash drive or DVD and choose Custom install to install Windows to this empty space.  You can only have a max of 4 primary partitions on an MBR disk, so hopefully you do not already have this many.  Installing and activating another Windows 8.1 to this small partition will prove that the process works.  It will become the default, but you can switch this back to your main install under the boot tab in msconfig.

Is the original install error you reported from the automated Windows 10 upgrade from the Get Windows 10 (GWX) App?

Windows 8.1 and 10 will try to use your computer's optimal capabilities when it comes to UEFI BIOS mode and GPT disk initialization.  Because your BIOS is in a "mixed" mode (some Legacy compatibility settings are required for Windows 7) I think Windows 10 is wanting to set things up optimally after detecting the UEFI capability of your computer.  This led to the initial error you saw.  You might be best to cancel your Windows 10 upgrade until this is sorted out, as it takes up 4-5GB of disk space once Windows 10 is just sitting there waiting to install.

So the best course of action is still to reinstall Windows 8.1 with full UEFI settings and GPT disk partitioning, before attempting to upgrade to Windows 10.  It would require a backup of your data to do this and not lose your data, etc.
