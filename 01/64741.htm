Virtual Desktop Access - Microsoft Community

kepy say:

Virtual Desktop Access

Hello,

I have four questions: 1) Is it possible to have concurrent remote desktop sessions open to multiple virtual machines? 2) Is there a maximum to the number of concurrent desktop that can be open on one machine? 3) In order to open a remote desktop session on a physical workstation connecting to a virtual server on the same network, is it necessary to have a RDS/RDA CAL for the workstation? 4) If I want to access the virtual server from multiple physical workstations concurrently, do I need a RDA CAL for each workstation? BTW, the workstations are Windows 10 Professional and the virtual server is Server 2012 R2.

hairyfool say:

MSTSC.exe which is the Remote client application is just that, an application and as such there is no specific limit. Where a limit does arise is the number of remote devices that can connect to your computer limiting how the local services can be connected. When you set up a session by default you give the host access to your clipboard and printers.

Server has specific licensing requirements, virtual or otherwise makes no difference. 2 admin connections are allowed without CALs because they are not strictly Terminal Services clients, beyond that and a CAL is required for each client.

kepy say:

That's helpful. Thank you. I have one more question: I read the requirements for a RDA CAL and one said that it would only work with a Windows OS with a 'Volume' license. The Server 2012 R2 is on an OEM license, and the workstations are standard Dell models with Win 10 Pro OEM licenses. Thank you for your assistance.

hairyfool say:

I have never come across such a limitation and itself does not make sense. Businesses do not automatically use a Volume License on all of their client PCs as they all come with PC license anyway. I would be most peeved if I had to use one of a limited number of VLs in order to assign a RDA CAL to a computer that already has a valid usable license. The CAL allows a client to connect to a terminal server, that could even apply to a non Windows client as there is no software associated with the CAL itself. RDP works on iOS, OS-X and Android as well as Windows operating systems.

kepy say: Also, is it possible to open a remote desktop session without a Remote Desktop Services CAL if I have a Device CAL and the workstation and the server are on the same network?

hairyfool say: You would normally create a single Licensing Server to manage the TS CALs which is then responsible for all TS servers within the domain. This would then allow the same Client or Device (there are two licensing modes) to access multiple servers under the same license.
