"How to Refresh Windows 8 and Windows 8.1"

  • The PC boots into Windows RE (Recovery Environment).
  • Windows RE scans the hard drive for your data, settings, and apps, and puts them aside (on the same drive).
  • Windows RE installs a fresh copy of Windows.
  • Windows RE restores the data, settings, and apps it has set aside into the newly installed copy of Windows.
  • The PC restarts into the newly installed copy of Windows.

"How to Reset Windows 8 and Windows 8.1"

"When you reset your PC, the drive will be completely reset, and all your apps and files will be removed from it. Windows 8 will be reinstalled, and you'll need to change any settings and reinstall any apps that you want to get back after resetting. If you know if any files that you don't want to lose, then you should back those files up before starting the reset"

mowgreen say:

as for virus program only what comes with windows... for now and spybot


No AV comes with Vista, MSE was downloaded and installed from Windows Updates.
You posted that Spybot was installed.
Now you're stating that Spybot was installed after updating failed.
Only you know when it was actually installed, if TeaTimer and Immunize were Enabled, and if it was installed after the updating issue occurred, then it's not in play here.

As Taurarian has already posted, 0x80070005 signifies an Access Denied message. You can run a Fixit that will reset Windows Update components and reset the Permissions for 2 of the Services that are required to update, that can be downloaded and saved from this page - How do I reset Windows Update components?

Once the Fixit's download has completed, configure the system to Clean boot -
How to troubleshoot a problem by performing a clean boot in Windows Vista or in Windows 7

Restart to the Clean boot state and then run MicrosoftFixit50202.msi and choose it's Aggressive mode.
That mode resets the Permissions of the BITS service and the Windows Update service to the default security descriptor.
NOTE: It will also remove the updating history that is accessed via View update history.

Restart the system to the same Clean boot state and see if it can install all of the updates except one for now.
Whether it could or could not, undo the Clean boot steps and boot to normal Windows mode.

If the system could install updates in the Clean boot state, run Windows Update now to see if it can install the one remaining update in normal Windows mode.

If the system could not install updates in the Clean boot state or in normal Windows mode, please copy & paste at least the last 50 or so lines of the WindowsUpdate.log into your reply - How to read the Windowsupdate.log file
