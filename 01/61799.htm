iautorenewedu say:

Hi Folks,

There seems to be a wide causation of issues here, so I will give this a top down view:

1. Removing extra HDMI hops is a good idea. The longer the daisy chain the higher chance for a reduction of quality (audio and video)

    - For car enthusiasts, think of this like a driveshaft. The longer the driveshaft the lower the total amount of horsepower is transferred to the pumpkin.

    - There is no such thing as a perfect pass-through, even hubs and nice switches isolate the network path to layer data more efficiently. Thus a reduction of quality can be expected. This is normal as 2 additional hops (including an AVR).

2. 5.1 and 7.1 Dolby uncompressed audio format is supported, and does work very well.

3. Match the out/inputs. If the Xbox is at 1080p, the AVR is 1080p, and Cable box is 720p. That means that each device in the chain is going to attempt to upscale 1080p (Including the TV), thus reduction of quality of video and/or audio.

4. On this forum section I always ask for the following info, please help me by providing this information to know where to direct focus:

- TV provider

- Layout (cable box HDMI Xbox One HDMI TV)

- Make/Model (cable box and AVR if being used)

Cheers!

jennmar say:

This is actually a feature of Windows 10 to improve performance.  :) 

Here is what is happening: your computer only has so much memory (RAM) and when you have a lot of programs open and are running low on memory, data is then stored on a pagefile on your hard drive (which is much slower to access than RAM).  In Windows 10, they designed it so instead of writing out to the pagefile on disk, data can be compressed and retained in memory, so it can still be accessed quickly if needed.  All of that extra data that it is holding in memory is showing up under the System process. 

More info at:

http://lifehacker.com/why-is-windows-10s-system-process-using-so-much-ram-1725076206

and

http://superuser.com/questions/952141/windows-10-system-process-taking-massive-amounts-of-ram
