The Blue Screen of Death (BSOD) errors usually occur when a serious problem causes Windows to shut down or restart unexpectedly. There are multiple reasons why this type of errors occur on the computer. They usually happen when there are some driver issues on the computer. However, it might also be due to some other software/hardware reasons. The blue screen error provides a bug-check error code which is used to identify the root cause of the issue. A bug-check (BSOD) is a mechanism to protect your PC from damage when such critical errors occurs on it.

 

If the computer is in a restart loop and if you are unable to boot into it, then we need first disable the “Automatic Restart” option from Windows 8.1 Startup Settings. If you have the Windows 8.1 installation/recovery media, then you may follow these steps to enable this option.

Disable Automatic Restarting

 

a) Boot the computer from the Windows 8.1 installation/recovery media.

b) In the “Windows Setup” dialog box, set the Time and currency format, Keyboard or input method and the language to install and then click “Next”.

b) Click on “Repair your computer” from the lower left corner.

c) From the “Choose an option screen”, click on “Troubleshoot”.

d) Click “Advanced Options” in the “Troubleshoot” screen.

e) Click on “Windows Startup Settings”.

f) Click on “Restart”.

g) Upon restarting the computer choose the option “Disable automatic restart after failure”.

 

Now, you may try booting the computer normally from the hard disk and when you get the Blue Screen error, you should be able to find the STOP ERROR CODE on the Blue Screen. Please make a note of it and include that information in your reply.

 

You may also try performing an Automatic Repair of Windows 8.1 by booting from the installation/recovery media. Please follow these steps:

 

Perform an Automatic Repair of Windows 8.1 using the Installation Media

 

a) Boot the computer from the Windows 8.1 installation/recovery media.

b) In the “Windows Setup” dialog box that appears, set the Time and currency format, Keyboard or input method and the language to install and then click “Next”.

c) Click on “Repair your computer” from the lower left corner.

d) From the “Choose an option screen”, click on “Troubleshoot”.

e) Click “Advanced Options” in the “Troubleshoot” screen.

f) In the “Advanced Options” screen, click “Automatic Repair”.

g) Follow the on-screen instructions to complete the Automatic Repair.

 

If that doesn’t help, then we need to perform a deeper diagnosis using the dump files saved on the computer during this issue. This can be done using the Debugging Tools for Windows. funbit say:

The reason is found.. thanks to sysinternals forum.

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\FontSubstitutes\MS Shell Dlg
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\FontSubstitutes\MS Shell Dlg 2

The "MS Shell Dlg" attribute was set to "MS UI Gothic".. Changed it to "Tahoma" and now the font is ok (http://img682.imageshack.us/img682/731/fontsubstitutes.png)
