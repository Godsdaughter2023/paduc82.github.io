net stop wuauserv
cd %systemroot%
ren SoftwareDistribution SoftwareDistribution.old
net start wuauserv
4. Restart the computer and check the functionality.

The Windows Installer is Broken.


OK, the title is deliberately provocative.  I'm suggesting that the mountain of Installation problems reported from around the world indicate that it's not a support problem, but a design flaw that needs to be addressed by the developers.

  • As a former Product Manager and Release Manager for major software vendors, I can say that I would turn this back to the developers for resolution.
  • I have tried installations on multiple computers at our site and I can report that Windows 8.1 and Windows Server 2012 R2 will simply not install on computers with Pre-existing installations of Windows 7, or with any pre-existing BCD Store.
  • Symptoms include "Computer requires repair", hardware stop errors, and "winload.exe cannot be found or is corrupt".
  • I've watched the flurry of MVP responses in the fora, and have as yet not seen a single definitive solution.
  • The best advice is that we should use the Recovery Environment, BCDedit, or Bootrec.exe.  These are tools that should never need to be invoked for a simple installation of an OS.

I hate to be alarmist, but I've been working on these issues for a couple of months, and have not had a single success.  I am cancelling the migration to Windows 8.1 and Server 2012 in my own business, and recommending the same to all of my clients.

I invite everyone to contribute their experiences in this thread, and perhaps we can get a proper response from Microsoft.
