lemp say:

This line from the ipconfig /all output ...

   Autoconfiguration IPv4 Address ..................: 169.254.241.198(Preferred)

means that your network adapter is configured to obtain its IP address automatically from the DHCP server in the router (which is the default configuration) but that it has been unable to obtain such an address.  There are only a limited number of reasons why this might happen:

  1. The cable is bad (try using a different cable)
  2. The port on the router is bad (try connecting to a different router port)

    If one of your other computers is connected to the router via an Ethernet cable, you can eliminate 1 & 2 by disconnecting the end of the cable from the other computer and connecting it to the problem computer.  If still no joy, the problem is neither the cable nor the router port.

  3. The port on the new computer is bad.  This is harder to check out and there is only one LAN port on your mobo. When the cable is connected to the router and the computer, what do the LED lights at the top of the LAN jack on the computer do?  The top left LED is the activity indicator and the top right is the speed indicator.



  4. The router is configured to only assign a limited number of IP addresses and you've already connected that many devices.  This would be pretty unusual -- and you would know that you did it.  I'm not sure why you created a MAC address filter (this is usually used to block access rather than grant it).  If your router has the capability to reserve IP addresses for a specific MAC address you could try that.  I doubt, however, that the router is the cause of the problem.
  5. The LAN adapter on the mobo is bad or the driver is corrupted.  Try uninstalling and reinstalling the driver.

Given what you've reported so far, my guess is that it's either some odd behavior by one of the ASUS utilities or a corrupted driver for the Ethernet adapter. 

Have you looked at Device Manager to see the status of your Ethernet adapter?  I expect that it will not show anything wrong, given that the ipconfig /all command seems to have completed with no error.

I don't think you mentioned any antivirus or other security software (including third-party firewalls, "internet security suites" and the like).  If you have any such software installed, you might consider temporarily uninstalling it.  If you're concerned about possible malware attack while your security software is uninstalled, just disconnect your router from the Internet.  At this point, what you're trying to establish is the connection between the router and the computer; the connection between the router and the Internet is not relevant to that problem.

BTW, your custom SSID and network password are not relevant to an Ethernet connection to the router.  Those are only used for wifi.  When properly connected by Ethernet, your computer may (or may not) show the SSID, but it doesn't use that information in order to make the connection.

This bug has plagued every release of Windows Live Mail since v.2011 first saw the light of day in 2010, more than five years ago. One of the earliest reports was this one dated 27 October 2010. Another long-running thread on the subject is this one that has just passed its fifth birthday. Despite the cooperation of many affected users in submitting sample messages, log files, Fiddler traces and so on for analysis, no explanation has ever been forthcoming and none of the promised fixes has ever emerged.

The problem only occurs when the message being forwarded exhibits certain characteristics, including (a) having at some stage passed through an AOL mail server, (b) having at some stage passed through an Apple mail client, and (3) having at some stage passed through an Office application like Word.

There is no solution, so far - after so many years. Here are a few workarounds:
 

  • Use Forward > Forward as attachment (click the bottom half of the button to get the option) to forward the message exactly as received. This has the disadvantages that (a) the message isn't edited before sending, so all previous recipients may be listed at the top of the message. Some users find this uncomfortable; (b) it requires that the recipient has software capable of displaying .eml files. Many mail clients can do this, but probably not all.
     
  • Click on each image and select Save as to save the picture to e.g. the desktop. Then, in the Forward message window, delete each image and use Insert > Single photo to embed the ones you saved.
     
  • Forward the received message in webmail. I.e. if you received the message on your Comcast account, log in to your mailbox at
     
  • An unproven workaround involves setting Windows Live Mail 2012 to use Internet Explorer in its native mode instead of the default IE7. This certainly works for some types of received message on some systems.
