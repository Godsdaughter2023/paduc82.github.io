Additionally, if you have issues with RAM loss, after big Windows updates, you can do this:

Hit Win Key + R
Type in “Regedit” and then hit Enter.
Go to ‘HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management’ (you can copy paste this in the regedit search bar, just like win explorer)
Find ‘ClearPageFileAtShutDown’ and change its value to 1

Secondly, go to 'HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\Ndu' (you can copy paste this in the regedit search bar, just like win explorer)
Find 'Start' and set the value to 4.
Restart the computer

¡firedog say:
An email message is a simple text file, which means you can open it and edit it in a simple text editor like Notepad. There is no way of guaranteeing that a printout of an email message is 'genuine', because anyone could have changed it after it left the sender's outbox. A digital signature will ensure that the message as received by the addressee is identical to that sent, but even a digitally-signed message could be tampered with before printing.

How a message appears in a printout depends entirely on the program it's being printed from. The From: line of a message sent from Windows Live Mail is in the format "Name"<address>, where Name is constructed by the program from available data. The quotation marks will be used if Name contains a space. If no data are available (e.g. if there is no matching entry in the contacts list), then the From: line will take the form address<address>. This done so that mail programs that by default suppress the <address> part still have something to display and print.

In short, you can never be sure that an email message someone shows you - either as a file on a computer or as a printout - is a true representation of that message as it left the sender's computer.
