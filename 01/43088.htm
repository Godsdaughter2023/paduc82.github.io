  • Cleared system cache three time
  • Deleted console updates
  • Deleted Marketplace cache

cmdrkeene say:

That's a good question.  I don't think you can do it from the command line directly, because the process that is hosting the command itself would need to be elevated.

Example, if I have cmd.exe running but not elevated mode,  I don't think it can start an elevated session of calc.exe.

You can set your shortcut (the shortcut that opens cmd.exe command prompt) to always run "with highest privileges" or "always run as admin" (depending on your version).  Then anything you launch from it (including items launched from a batch file) will also have those high privileges.

In any case, creating an elevated process (of anything) will first show the UAC "do you wish to allow this program to run" dialog.

Of course, you can disable UAC.  Then every process you run will run as your user account, and have full privileges.  But that has the downside of weakening your security greatly. Even a bad website you visit could run code (which is running as admin now) and do anything without your knowledge or consent.  Although in some environments, this might be acceptable risk and required for what you need to do.

I have no idea why, but my company has configured our computers to disable UAC.  I'm one of a very few that have administrative rights to my own computer, but I have no UAC to mitigate that risk.  Seems odd to me, but I'm not in charge.
