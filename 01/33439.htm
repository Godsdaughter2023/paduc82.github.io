The symptoms and diagnosis you have provided is similar to what happened to Max Payne 2 in around 2012. After successful installation and many hours of use, the application suddenly would not launch after clicking on the icon. The user tries the usual diagnostics; clean the disk, troubleshoot comparability in windows, run application as an administrator, ... checked Data Execution Prevention ... nothing works ever after.

It appears only the disk based versions of these games have been affected, indicating that each program executable installation and .NET framework are uncorrupted. The disk based installations included anti-piracy software called DRM (Digital Rights Management), that was designed to protect the software under licence from being copied freely by naughty people.

However, it turns out that the DRM anti-piracy software, that was used to detect genuine versions, is now in conflict with the computer's operating system, and wont allow the user to launch the application. Well, Max Payne 2 discs used Direct 2 Drive DRM software, while Civ 3 uses SafeDisc (SECDRV.SYS) , so it follows that lots of softwares reliant on either Direct 2 Drive or SafeDisc would be showing the same application launch symptoms. 

There is no way of switching off the DRM feature as it is embedded in the program. Back in 2012 Max Payne user community suggested turning to piracy and using the nocd crack, although this is still illegal, in 2012, if you do not own the software, even if the software is more than five or ten years old. There was no claim ever made in public as to why a perfectly legitimate and fully operational installation of MaxPayne2 would stop working.

Many people have cited this problem with their secdrv.sys DRM based game applications. From Civilisation 3, SIMS 2, Command & Conquor, and so on. The game once worked, and now has stopped working from launch. All these problems have all occurred since 9th September 2015, which is the date Microsoft issued new updates for its Windows 7, Windows 8/8.1, and Windows 10 operating systems. It appears that Microsoft Security Update KB3083324 is depreciating the DRM based games without the users knowing about it, and causing confusion on the way. 

Microsoft has come clean about this by

  1. announcing the security flaw in 7 November 2007, and releasing a patch in the same year, for Windows 7 systems. (see Wikipedia article "SafeDrive").
  2. Providing a security bulletin MS15-097 in September 2015.
