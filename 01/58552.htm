gerry c j cornell say:

I would check the status of the Print Spooler Service.

Click on the Start button and select Control Panel, Administrative Tools, Services. Locate Print Spooler in the list. Check that it has started and that the StartUp type is Automatic.

I suspect that it has stopped and that just starting it will not solve the problem.

You may need to show hidden files. Type Folder Options in the search box above the Start button and select View, Advanced Settings and verify that the box before "Show hidden files and folders" is checked and "Hide protected operating system files" is unchecked. You need to scroll down to see the second item. You should also make certain that the box before "Hide extensions for known file types" is not checked.

Now access Windows Explorer and select c:\windows\system32\spool\printers. Normally this folder should be empty but there is a problem when files with “spl” and “shd” extensions stick there. Delete the files and restart your computer. This may resolve your immediate problem.

andym03 say: Actually, I just solved the issue with steps found in another forum. Hope this helps someone.

Steps below:

1. Click Start, type "Services.msc" (without the quotation marks) in the Search box and press Enter.

2. Double-click "Printer Spooler" in the Services list.

3. Click Stop and click OK.

4. Click Start, type "%WINDIR%\system32\spool\printers" in the Start Search box and press Enter, delete all files in this folder.

5. Click Start, type "Services.msc" (without the quotation marks) in the Start Search box and press Enter.

6. Double-click "Printer Spooler" in the Services list.

7. Click on Start. In the Startup Type list, make sure that "Automatic" is selected and click OK.

5) Posted by : artologik
The solution came to me when I could not recreate
the problem on a different laptop with Vista Home.
Comparing everything that I could, I noticed that the
%PATH% environment variable was different between
the 2 machines.

I guess WLPG relies on DLLs that must be found
on the system path; when not found, it must revert
back to "basic or incomplete" libraries that it finds
on default or user specific paths. The solution for
me was to ensure the overall system path pointed
to the proper folders in the proper order.

To solve, do this:

A. Right-click the COMPUTER icon on the
desktop and choose PROPERTIES

B. Choose ADVANCED SYSTEM SETTINGS

C. If prompted by UAC to authorize, go ahead
and ALLOW

D. Click ENVIRONMENT VARIABLES....

E. In the bottom half, under SYSTEM VARIABLES,
locate the PATH variable and highlight it

F. Click the EDIT button

G. Make sure that whatever is there starts with
%SystemRoot%\system32;%SystemRoot%;

H. Make sure a semi-colon separates  each entry.

I. Click OK three times

J. Restart computer

Good luck...

Under Windows 7 certain operations require an elevated status that you must invoke manually. The purpose of this measure is to make it harder for rogue programs to make changes behind your back.

To run a Command Prompt in elevated mode, do this:
1. Click Start.
2. Type the three letters cmd into the Search box.
3. Press Ctrl+Shift+Enter
4. Click "Run as Administrator".
