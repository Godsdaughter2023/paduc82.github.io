Hi

I'll just use one of the folders where you get the 'access denied' message to explain what you are seeing, since the same explanation applies to all of these 'folders' that have the Shortcut Arrow.

C:\Documents and Settings.

This folder name was deprecated and no longer used starting with Windows Vista. What you now see as the 'Documents and Settings' folder is actually a Junction or Soft Link, (also called Reparse Points) this is why you see the shortcut arrow on that folder.

The only reason for this Junction is for backward compatibility purposes. If an older program, designed for XP is installed and it is hard coded to look for the old 'Documents and Settings' folder, the 'Documents and Settings' Junction will redirect the program to the new location in Windows 7.

The new location is: C:\Users\username\AppData\Roaming.

A folder junction will have the permissions set as 'Everyone/Access Denied' to prevent the Junction from being used for any other purpose.

 

These Reparse Points are usually hidden unless you have seleted to Show Hidden Folders in the View Tab of the Folder and Search Options.

 

Important Note: You should not try to change the permissions or other attributes of these Reparse Points. Doing this can result in unexpected and unwanted behavior with certain programs that may be installed on the system.

hairyfool say:

Furthermore as to the "not support VirtualBox virtual machines" the EULA sect 2 subsect d clause (iv) has the following:

Use in a virtualized environment. This license allows you to install only one instance of the software for use on one device, whether that device is physical or virtual. If you want to use the software on more than one virtual device, you must obtain a separate license for each instance.

This EULA makes no distinction of the license type (home , pro, Ent) nor does it specify the emulation software. Elsewhere within Microsoft systems test VMs are provided for all major emulators.

Strictly speaking OEM licenses should not be on VMs so as Andre says for retail licenses the Virtual Hardware ID used by licensing can change anyway, you should still be able to activate.
