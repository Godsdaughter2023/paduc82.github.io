considerate_guy say:

Another thing to consider is the file type of the video you are using.  Some file types, though technically compatible with Movie Maker can have playback issues with trying to provide a real-time preview of your added transitions/effects/etc.

 

There are two things you could try:

  1. Ignore the issue, especially if the stalled playback is not evident in the final saved movie.  Movie Maker tries its best to give you a real-time preview, but it may have to pause momentarily if it is switching to a high-resolution file as it waits for it to load/cache off the disk, as it applies the video transitions or effects, etc.  This pausing should NOT be present in the final movie.
  2. Try switching to a different video file type.  Movie Maker prefers .wmv video, so you might try converting your original video using something like Freemake Video Converter (freemake.com).  This is especially true if you are dealing with an uncompressed raw video file from something like video game screen capture.  The conversion to .wmv can make it much easier to handle in Movie Maker.

cmdrkeene say:

This does not happen because Windows now tries to have less noise.  You are only warned when something dangerous or destructive is about to happen.  Moving a file to the Recycle Bin is not dangerous because you can always undo it.  If you were instead going to permanently remove a file (by pressing shift-delete or because it was on a network drive where there is no recycle bin), then you would still be warned.  This way, starting with Windows 8 and beyond, the only warnings you see are the ones that really matter.

If you want to turn back on the extra prompts for safe operations, you can right-click the Recycle Bin and choose properties.  There you will see the option to turn on the warnings.
