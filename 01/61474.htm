With triggering you don't need to specify the ip address of the Xbox and it opens/shuts the ports as they are needed/not needed.

With forwarding you need to specify the ip address of the Xbox and the ports are open all the time (nailed up) even if the Xbox isn't on.

Just two different ways of doing the same thing really. You need the same ports for both

In your initial post you state that you ran System Restore to restore connectivity but that did not resolve the issue. And, that the updates were still present after SR finished it's operations. That should not be happeningIF the restore point you used was created after the updates were installed.

You could determine which update may have caused this issue by manually downloading and saving both of them, manually installing one, then restart and see if the system loses connectivity. Then do the same for the second one.

KB2158563 is a Time Zone update and one would think that if you are not located in the geographical region that it was created for, then it would have absolutely no effect on connectivity.
As we've already mentioned, KB979538 updates two drivers to eliminate a race condition between them. It's specifically meant for systems that are experiencing Stop errors ( Blue Screen of Death, aka BSOD ). If your system is connected to the net via a USB device, then it's the most likely culprit for the loss of connectivity.
The update being marked as Important is not what you should base your decision on whether it should be installed or not.
Your decision should be based on whether the system is experiencing the issue as described in theKB article.

If you want to go the manual update installation route, both updates are available on the Microsoft Download Center.
Enter the KB numbers in the Search All Download Center field, then click the magnifying glass icon to start the search.
Be sure to download and save the updates for Vista and the architecture of the system, eg. x86 is for 32 bit Windows 7 systems, x64 is for 64 bit Windows 7 systems.
