Bug in GDIPlus API FontFamily.IsStyleAvailable - Microsoft Community

enguerrangillet say:

Bug in GDIPlus API FontFamily.IsStyleAvailable

Hi there,

ever since Windows 8 was released, there is a bug with the GDIPlus API FontFamily.IsStyleAvailable.

When calling FontFamily.IsStyleAvailable, to check if the font supports bold or italic, the method will always return true, even for fonts that do NOT support italic or bold style.
This issue was not present on Windows 7 but can be reproduced on Windows 8, 8.1 and 10.

Sample C# code reproducing the problem:
public void TestIsStyleAvailable()
{
var font = new FontFamily("Brush Script MT");
Assert(!font.IsStyleAvailable(FontStyle.Regular));
}
Regular isn't a style available on that font, yet IsStyleAvailable says it is.

This issue has been initially reported to the Visual Studio and DotNet team, but they mentioned this is an issue for the GDIPlus team and they informed the GDIPlus team.

See details here https://connect.microsoft.com/VisualStudio/feedback/details/785535/ and here https://connect.microsoft.com/VisualStudio/feedback/details/1693536.

This has been reported 3 years ago, and the bug is still present. Do you have an ETA on the fix for this? Any known workaround?

Thanks.

Enguerran

Activate Administrator Account in Vista

 

1. Go to your Start menu and in “Accessories” list, open “Command Prompt” by right-clicking on its icon and choosing “Run as Administrator”

2. When the Command Prompt window appears, enter the command net user administrator /active:yes

3. When done, log out from your current account.

4. The Built-in Administrator account should now be present on your log in screen.


that's for the built in Administrator account - if you're having issue with an account that you created, try the following.

Boot to the Windows Recovery Environment (WINRE) mode,


- To boot the computer in WinRE mode, insert the Vista CD into the CD tray, restart the computer.
- When prompted to "press any key to boot from CD.." press Enter.
- Select the Default language, click Next.
- Select "Repair your computer".
http://www.flickr.com/photos/freeed/363935713/in/set-72157594490796701/
- Select Command Prompt and then type in the following commands:

net user administrator /active:yes
net user {name of the user account} {new user password}

Note : Please type the User Name and the Password without the flower braces.

- Restart the computer normally.
