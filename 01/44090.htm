Hi

You cannot have a single file on the system unless it is contained in a folder?

When you create a Library, it is just a name for a collection of folders that contain files that are usually of a similar type.

The Libraries main function is to be able to view and work with files, in one central location, that are actually contained in multiple locations including, folders on the local drive, remote hard drives, removable hard drives, or network locations.

The Libraries do not duplicate folders or files, they are Virtual Folders. They simply mirror the contents of a physical folder. You can add very large folders to a Library and only use a small fraction of the space that the physical folder actually occupies. During beta testing we added folders that were 50GB+ and the extra amount of disk space used by the Library was less than 100KB.

You can add, delete, move, copy, edit, view, or perform any function to files from within the Library and the changes are made to the actual physical files, in the remote location. The sorting options in a Library are almost unlimited.

One of the key reasons that Libraries were created was because of the move by more and more users wanting to keep the amount of space on the system volume low for optimized operation, but still be able to easily access their data.
