Here's what I do now for manually backing up my PC data files.  First, I do NOT use the Win-10 Backup system.  That's turned off.  Second, my system is cumbersome, to say the least.  But it does work.

I power up the external drive, and then plug in the external drive's USB cable to my PC.  In File Explorer, I create a main folder with only the date (e.g., 2017-01-13).  Using a year-month-day system allows for chronological sorting.  [HINT, I also use that naming system for naming files.  It's very handy to be able to sort files using the date.  I'm working on a book of a historical nature, and sorting all the files by date makes it easy to work on the book decade by decade.]

If I need more room on the external drive I delete one of the older main folders (that is, I delete a previous backup).  It's good to buy an external drive with at least 5-7 times the space of all your data files.  In File Explorer, I highlight the data folders in my C: drive (i.e., the internal PC drive) that I want to copy.  I right-click and select "copy."  I highlight the external drive folder (2017-01-13), and then right-click and select "paste."

I make sure that my PC doesn't go to sleep during the 4-5 hour manual backup.  Using "Settings" (the gear icon in the main menu) and then "Personalization" and "Lock Screen" and finally "Screen Timeout Setting."  I set the PC sleep time to five hours.  You have to scroll down to the bottom, and then scroll down some more because the screen jumps back up (long story).

I leave the PC along while it manually backs up.  Near the end there will likely be some pop-up screens that I have to address.  The manual backup won't finish until those are all addressed.  It's best to use "do this for all."

HINT: I name all the main data folders (level #1) on my PC using an allowed special character (e.g., "@" or "#").  That way you can spot all your main (level-one) data folders in File Explorer quite easily (I have only about 7-8).  For example, #Project Alpha" or "Writing Projects."  It's best to keep main file-folder names somewhat short.  Spaces are allowed in folder names, so you don't have to use goofy underscores.  Spaces make it easier to read the folder names, in my humble opinion.

I leave the PC alone for the several hours it takes.  I can do basic work, but I avoid that.  HINT: You can pause the backup using the pause button on the small copying-files menu screen ("||").  If you have work to perform on your PC during a major backup you can pause the copying to prevent any PC slowness while you're doing your work.  The mini-screen turns yellow to remind you to restart the copying (using the solid right arrow ">").

I backup my main data folders every couple of weeks.  So, what about files created or saved during that period?  Good question.  I create a special main-level folder at the very top of the sort ("!NEW").  Every file I create or save after the manual backup is put into that folder.  Every day (or more often) I manually save that "!NEW" folder (it's much quicker to save that folder compared to a complete data-folder backup).  I prefer a memory stick for that save.  I use the same folder-naming system, but add "new" to the folder name (e.g., "2017-01-13 New") so I know it's not a complete backup.

I also make perform a manual data-folder save to a set of large memory sticks 2-3 times a year (e.g., 256 Gb sticks are now less than $50).  I give those to a family member to hold in case of fire or other issue with the external drive.  I also have two external drives.  I save to each one every other time.  Only after both external drives have a complete data-folder backup do I move all the files out of the "!NEW" folder.

So, that's my current backup system.  It's cumbersome, and it's quite imperfect.  It doesn't copy software, for example, which is a major shortcoming.  But, it does work for me.  I hope this helps other understand that there are alternatives to using Win-10 Backup.  Maybe Win-13 will have a better Backup app.  Thanks for reading my post.

thahaseena m say:

Hi,

·        Are you trying to copy the file from the system to another location (e.g. USB flash drive) or are you trying to copy the file from a different location to the system?

·        Were you able to copy the video files on the system or from the system before without any issues?

·        Apart from Video files are you able to copy other files on the system or elsewhere?

·        Do you remember making any change to the system?

·        What is the size of the file that you are trying to copy?

 

Step 1:

If you are try to copy the video file to a USB flash drive or any other external device, I would suggest you to check for the space available on that particular device. If there is not enough space left on the external device, then you might encounter this issue. I would suggest you to format the external device and then try to copy the video files and check.

Note:Before formatting the device I would suggest you to back up the data that is present in the external device as formatting will erase all the data.

Note:The external drive you are trying to copy should be NTFS, if yes then, it won’t allow huge file to be copied.

 

It might be that the video file that you are trying to copy is corrupted. I would suggest you to copy another video file and check or copy this video file in another computer and check.

 

Step 2:

It might be that some third party application is causing this issue.I would suggest you to do a clean boot and then try to copy the files on the system.

Follow the steps mentioned in the article given below to configure the system in clean boot.

How to troubleshoot a problem by performing a clean boot in Windows Vista or in Windows 7

http://support.microsoft.com/kb/929135     

Note: After you have finished installing, follow step 7 in the article to boot to normal startup.

 

Step 3:

I would suggest you to optimize windows and check. Follow the steps mentioned in the article given below.

OptimizeWindows 7 for better performance

http://windows.microsoft.com/en-us/windows7/Optimize-Windows-7-for-better-performance
