If your system has SATA drives and you try to boot on an XP installation CD that does not have SATA drivers incorporated (which it doesn't), when you boot into the Recovery Console, you will end up with a C:>\ prompt instead of a C:\WINDOWS prompt like you "should".

To get things to work, you need to get into your system BIOS and temporarily adjust the HDD interface mode to something the CD can understand.

Here are some examples I have accumulated:

Where to find the adjustment and how to change it depends on your system manufacturer and BIOS version so you may end up having to just look around for it.  If you know your system make and model, we can probably help you find a manual.

You need to locate and change the interface mode for your primary hard disk in the BIOS so that when the XP Setup loads, it will be able to locate the hard disk.

Before making any adjustments, you should make a note of what the current BIOS settings are so you can change them back when you are finished.

The adjustment varies so here are some examples I have accumulated:

The adjustment is made in the BIOS and could be under Integrated Peripherals, SATA Device Configuration, SATA Mode or something similar.

Make a note of what the current settings are before making any changes so you can change them back when you get done or if things get worse.

After you make the changes, save them and see how things look.  You may have to try a few things to get it to behave.

If you find the mode is set to RAID/SATA mode, change the mode to IDE.

If you find the mode is set to SATA, change the mode to IDE.

If you find the mode is set to SATA, disable SATA mode.

If you find the mode is set to AHCI, change the mode to ATA.

If you find the mode is set to AHCI Emulation Mode, change the mode to IDE Emulation.

If you find the mode is set to SATA, disable SATA mode and/or change the mode to ATA.

If you find the mode is set to SATA Native Mode, disable SATA Native Mode.

If you find the mode is set to RAID/Autodetect AHCI, change the mode to Combination.

If you find the mode is set to RAID Auto/AHCI, change it to RAID Auto/ATA (this was a Dell XPS 420)

rseiersen say:

I found an answer that seems to have worked for me...  I selected 32 bit install (was selecting 64) and also selected Advanced Feature (F8) and went with Safe Mode.  Much to my surprise, after files loaded (via the installer), the first Windows Installation dialog screen appeared and I was able to progress with the installation per normal procedures, including entering the Key that came along with my purchase (see below).  Note that during this entire process only thing connected to PC was keyboard, mouse, and HDMI monitor.

One obvious tip: as per normal Win installation, at some point it wants to restart the computer.  Don't forget to interrupt the process and point it back to the hard disk with it's new Win 10 files, and away from your thumb drive.  Or simply pull the thumb drive out when power cycles.

Prior to the above, I was having issue very similar to elsewhere in this thread... trying to install Win 10 on thumb flash drive purchased in full from local retailer.  Installing onto blank, reformatted hard disk.  (Reformatted in another PC running Win 10 and using included Disk Management tool.)  Drive is 100% blank, all in single partition.

Booting from the MS Win 10 thumb drive, DOS-like screen comes up asking to select 32 or 64 bit installation.  Choosing 64, same screen shows "Loading Files" progress, then repaints to standard Win 10 blue w/ Win logo and circular "working" dot motion.  Finally repaints to Win sad face and auto restart; includes note about Stop Code 0xc00021a.

Restarting and selecting Advanced Mode options for Safe Mod, Debugging Mode, and Disable Driver Signature Enforcement all yield same Stop Code result.
