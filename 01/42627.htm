a. user say:

If you are seeing a message like this in the Backup Report file:

Error returned while creating the volume shadow copy:Catastrophic failure
Aborting Backup.

And if you look in the Event Viewer Application log, you will see errors like this:

Event Type:    Error
Event Source:    NTBackup
Event Category:    None
Event ID:    8019
Description:
End Operation: Warnings or errors were encountered.
Consult the backup report for more details.

Event Type:    Error
Event Source:    VSS
Event Category:    None
Event ID:    5012
Description:
Volume Shadow Copy Service error: Shadow Copy shim called routine CoCreateInstance( CLSID_VSSCoordinator, IID_IVssShim) which failed with status 0x80070422 (converted to 0x8000ffff).

Event Type:    Error
Event Source:    VSS
Event Category:    None
Description:
Volume Shadow Copy Service error: Unexpected error calling routine CoCreateInstance.  hr = 0x80070422.

That usually means that the Volume Shadow Copy Service is either Disabled or otherwise afflicted.

To check that, click Start, Run and in the box enter:

services.msc

Click OK to open the Services applet and locate the Volume Shadow Copy Service, right click it, choose Properties, set the Startup type to Manual, then click Start to see if the VSS Service will start.

If you are using a Scheduled Task for your backup, test your task by choosing to run it manually to be sure it works.

Note that in spite of what the Microsoft engaged Support Engineer "experts" say, the Volume Shadow Copy Service does not have to be set to "Automatic" start.  There is no reason for the VSS Service to be starting and running all day and night with nothing to do wasting your CPU cycles.

Setting the VSS Service to Manual means it will start when it needs to start and stop itself when it is done.
