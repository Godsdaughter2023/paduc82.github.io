I had a similar problem after installing Windows 8 upgrade from Windows 7.  After much searching the Internet, I think I finally found my problem. While this may not be the same as your situation, I figure it may be worth a try. 

Go to Device Manager, select view, and on the dropdown, select  "show hidden devices", then look under "Network Adapters" to determine if there are adapters which you might not need and are causing your problem.  These were not all showing under the Control Panel Network and Sharing Center items.  I was particularly interested in two items, Microsoft Kernel Debug Network Adapter (which was disabled already) and Microsoft Wi-Fi Direct Virtual Adapter (which was enabled but not showing under Control Panel)  After researching the Internet, I found Microsoft Wi-Fi Direct Virtual Adapter was not being used and could be disabled with no problem, so I disabled it.  Since then, I have had no Error Code 1001 error in my Event Viewer.  Apparently this adapter was some way interfering with DHCP.....I'm not techie enough to know how or why.  But, I no longer have continual and frequent 1001 errors.  Hope this helps you or someone reading this thread.    Note:  I only disabled them - I did NOT uninstall them.

LOL! I know it doesn't make sense, but I know exactly how he feels - I actually picked up a 3TB so I wouldn't inevitably have to play Sophie's Choice deciding which ones to delete! :-)

On your question about file systems, the Xbox One uses an NTFS file system, which is the same as a Windows PC, however all hard drives have something called a Master Boot Record, which is what the system uses to recognize the drive on startup. In other words, you could format the external drive on the PC,  but it still wouldn't be recognized by the Xbox, because the MBR will be set to PC. There is a tool to manually switch the MBR without reformatting, but that's really only useful if you want to use the drive on both an Xbox and a PC. You're better off letting the Xbox do the formatting from scratch.

And to answer that question, yes, the Xbox is designed to recognize it and reformat it as soon as it gets detected - since the Shadow was acting weird, I bet that was the problem. (And if it turns out that 3 consecutive Shadows were unreliable, that will be the biggest possible testimonial against them! :-)

Hi Dorothy,

 

I see that this is your first post in this forum, welcome to the Microsoft Community.

 

The run32dll.exe is a Windows application that helps in using DLL files. The run32dll.exe might be used by various applications which might need associated DLL files to function. If you get an error message stating that this process is not running, then either the application file could be corrupt or some other third application might be conflicting with this service and preventing it from launching successfully. I would suggest you to try these steps and check:

 

The genuine version of rundll32.exe is saved in the following location: C:\ Windows\System32\rundll32.exe (where C:\ is the drive where windows is installed). I would suggest you to double click on the file from this location to try manually starting the application. If the file can’t be started, then it is possible that the file is corrupt. In this case, I would suggest you to run a System File check. System File checker (SFC) scan is done to check if there are any corrupted system files that could be causing this issue. Refer to this article:

 

http://support.microsoft.com/kb/929833/en-us

 

If the SFC scan didn’t reveal any corrupt files, then some other service could be conflicting with this service and causing the issue. In this case, I would suggest you to boot the computer in a clean boot state and then check if the issue persists. Booting the computer in a clean boot state can reduce software conflicts and help identify the root cause of the issue. You can refer to this article for more information:

 

http://support.microsoft.com/kb/929135

 

Note: After troubleshooting, reset the computer to start as usual. Refer to the step How to set the computer to start as usual after clean boot troubleshooting under more information.

 

If the rundll32.exe file is missing from the above said location, it is possible that the file has been deleted by any third party security programs that you have installed or any registry cleaners that you might have installed. In this case, I would suggest you to restore the computer to an earlier date prior to the issue and check. Refer to the section ‘restore your computer to an earlier point in time’ in this article:

 

http://windows.microsoft.com/en-us/windows-8/restore-refresh-reset-pc

 

Try these steps and let us know the results. We’d be happy to help if you need any further assistance.
