occasionallyfrustrated say:

I came up with a work around that resolved the issue for me of being required to manually uninstall a nonexistent Windows SharePoint Services before proceeding to upgrade from 1607 to 1703 or 1709. Before trying it, I strongly suggest backing up the computer first in case you need to revert.

  1. Login with an Administrator account.
  2. Create another Administrator account. For example, AdminTemp
  3. Using the IOS image for 1709 (created by the Microsoft Media Creation Tool), run setup.exe for the upgrade as AdminTemp.
    (Shift + Right-click to select run as AdminTemp.)
  4. Run regedit and navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft. Create a permission entry for AdminTemp and leave the regedit window open.
  5. Exactly when the compatibility check starts ("Checking if your computer is ready..."), explicitly disallow all access of AdminTemp to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft.
  6. Immediately after the installation passes that stage (if it is successful), explicitly allow all access of AdminTemp to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft when  the installation begins.

On my workstation, I ran into other issues about an hour later during the installation process that are still being worked on with MS Support, but the above did resolve the WSS issue.

a. user say:

Hi Idear,

Thank you for posting your query in Microsoft Community.

I understand your concern and will certainly help you.

Please provide the following information to understand the issue better.

1. Are you using any other Image backup software?

The timntr.sys file is a part of the Acronis True Image Software. True Image is a backup and disk imaging software created for Microsoft operating system by Acronics. The timntr.sys file allows True Image to restore the image which has been previously captured to another disk. It replicates the structure and contents of the original image to the new disk.
This file enables True Image to creates file backup and disk backup. This file is used by Acronis True Image software or Acronis TrueImage Backup Archive Explorer or Seagate DiscWizard Image Backup Archive Explorer or Maxtor MaxBlast Image Backup Archive Explorer.

The timntr.sys file is safe and should not be considered threat to your computer.

If you shut down timntr.sys, it will likely start again at a later time either after you restart your computer or after an application start. To stop timntr.sys, permanently you need to uninstall the application that runs this process which in this case is Acronis True Image, from your system.

I suggest you to run the Microsoft safety scanner and check if your computer is infected by malware.
The Microsoft Safety Scanner is a free downloadable security tool that provides on-demand scanning and helps remove viruses, spyware, and other malicious software. It works with your existing antivirus software.
