Print queue,

a. user say:

You cannot delete any queued print jobs while the Print Spooler Service is running.

To stop the Print Spooler Service, click the Start button/Orb and in the box enter:

services.msc

Press Enter to open the Services applet. 

Locate the Print Spooler Service, right click it and choose "Properties" and then click the "Stop" button to Stop the Print Spooler Service if it is running.

The Print Spooler Service Properties "Startup type" option is usually set to "Automatic" (now is a good time to check it).

If the "Startup type" is not set to "Automatic", set the "Startup type" to "Automatic" if that is your desire (now is the time to check it).

Do not try to Start or restart the Print Spooler Service just yet.  You need to remove any backed up print jobs first...

Next, delete all the spooled up and pending print jobs.  If they did not print before, you will have to send them to the printer again.

Using Windows Explorer, navigate to the following folder or copy/paste the following line into the Start button/orb dialog box and press the Enter key to open the folder (assuming Windows is installed on your C drive):

C:\WINDOWS\system32\spool\PRINTERS

Delete all the queued print jobs (select and delete all the files) in the folder so the folder is devoid of pending print jobs.  The folder should be empty when you are done.

Restart your system and look in the Services applet again to check that the Print Spooler Service restarts automatically on reboot (since the Startup type is set to Automatic).

After rebooting, check the Print Spooler Service again and the "Status" column for the Print Spooler should read "Started".

Resubmit any failed print jobs.

32 bit can only see a max of 4GB - usually around 3.5GB, but if your graphics card is using shared memory it will be even less. ﻿Check to see how you can reduce your graphics card shared memory and increase available RAM, usually it's done in the BIOS, or in your GPU settings.
