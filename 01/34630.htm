Your problem is Microsoft considers that set up two separate installations of Windows.

I, myself, have a Win 10 main host setup, a Virtual box Win 7 setup, and a Win 7 secondary dual-boot partition all on one hard drive. Each Windows  OS installed has its own license.

It doesn't matter If it is one computer with one hard drive running through one motherboard. Each Windows Install is considered a separate installation.

The same thing applies to MS Office. It sounds to me like you are using one license to run it both in Host and Guest. That would be seen as two different installations.

Sorry to break the news to you.

Unfortunately, both Windows and Office use hardware hash for activation purposes so they perceive real and virtual hardware as different computers and there are now ways to avoid this.

pauli o say:

When Windows 10 is first time in some computer activated then that computer hardware ID is stored to MS activation server and that ID is based mainly on mother board. Every time you do reinstall to that same computer Windows will be automatic activated.

If you do small changes to hardware (change hdd, display adapter, add memory) that doesn't affect activation, but new mother board = new computer and that required new license. In your case virtual machine is counted as new computer and it need own license.

Retail licenses can be transferred, but you have OEM license so it isn't possible to transfer that to VM.

    a)      Right click on start and select ‘RUN’.
