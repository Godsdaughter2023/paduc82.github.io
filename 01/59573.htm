¡firedog say:

That was quite a catalogue of errors. There is no point at all at guessing the settings for an email account; these are laid down by whoever hosts the servers you're using. The settings for one account will not necessarily work for a different account, and you certainly can't mix and match server ports and security settings at random as you seem to have done. The ones you need are allegedly given on this page: http://www.networksolutions.com/support/step-by-step-pop-setup-instructions/#3,  although it certainly looks as if the information might be seriously outdated.

The POP3 server looks to be called mail.weloop.com, (not pop.weloop.com as you have).

POP3 uses port 110 unsecured or port 995 with SSL. I've never come across any variation on this.

Network Solutions SMTP servers use port 2525 or 587. It's not clear whether either of these supports SSL, so that would have to be tested.
Almost all SMTP servers these days require authentication.

IMAP normally uses port 143 unsecured or port 993 with SSL. You can't connect to a POP3 server on port 143.

The Email username for a Network Solutions account is the full email address.

Some of those server responses look very suspicious; how can the server respond if the program wasn't even able to connect to it? It's the sort of response returned by security software, like an anti-virus program, that's interfering with the connection between Windows Live Mail and the mail server. Do you have anti-virus - e.g. Avast or AVG - that is scanning mail? If so, please try disabling the mail scanning part and then sending a new test message.

Let's see how you get on with the right settings.

What do I do if my system fails validation, but I am certain that I purchased/have a genuine copy of Windows?

When a copy of Windows fails validation, the user is directed to a customized Web page with details about what caused the failure and recommendations for how to fix the problem. This page contains a section with troubleshooting steps. One of these steps will let you check to see whether you can use the online Product Key Update Tool. If your computer came with a genuine product key, but Windows was improperly installed using an invalid product key, the Product Key Update Tool helps convert your computer to a genuine status without having to purchase a new copy of Windows. If this solution does not work, refer to the instructions above for what to do when your system fails validation.


I have heard a lot of people are unable to validate through WGA - what's going on? Is there a system glitch or an error in the validation service?

The WGA validation process is extremely reliable, but there are times when system or network errors prevent validation from working properly. Validation failure is usually caused by the use of a non-genuine Windows license. In many cases, customers don't know they have received a counterfeit copy. Unintentional victims of counterfeit are often surprised by the result and may question the validity of the service. However, WGA has been operating since July 2005 and has validated more than 500 million systems. Microsoft investigates all user inquiries about possible errors. When errors occur, we work with our customers to resolve them. Customers who suspect problems with the validation process should follow any on-screen instructions

Is there a 100% correlation between WGA validation failure and owning a non-genuine copy of Windows? Is it possible to pass WGA validation once and fail later?

It is possible for users to pass WGA validation once, and then fail later as a result of non-genuine software. Microsoft frequently discovers new keys that are being counterfeited and adds them to the WGA service. After new keys are added to the WGA service, a validation result could change.
