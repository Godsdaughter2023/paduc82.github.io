Hi Mustafa Vohra,

Aside from disabling CDPUserSvc in the registry and uninstalling the recent update, there are other troubleshooting steps that you can try. Follow the steps on how to resolve CDpusersvc issue:

Method 1: Isolate the process by running a few commands in Command Prompt:

Running this command will scan and fix issues with the Cdpusersvc process. For instructions, follow the steps below:

  1. On the Start Menu, type Command Prompt. Right-click the result, and then click Run as administrator.
  2. Type this command followed by hitting Enter:
    sc config cdpusersvc type=own

Method 2: Create a .bat file to stop the Cdpusersvc service.

  1. Open Notepad.
  2. Copy and paste these command lines on the Notepad:
    @ECHO OFF
    SC QUERY state= all>servicesdump.txt
    FINDSTR /L /C:”SERVICE_NAME: CDPUserSvc_” servicesdump.txt >CDPservice.txt
    FOR /F “usebackq tokens=2” %%i IN (CDPservice.txt) DO SET CDPUserSvc=%%i
    NET Stop “%CDPUserSvc%”
    SC  Delete “%CDPUserSvc%”
    DEL CDPservice.txt
    DEL servicesdump.txt

    (Note: Make sure to copy as well the space below the last command.)
  3. Select File and then click Save as.
  4. Set Save as type to All Files and enter fix.bat in File name field. Click Save to save changes.

Yes, as a matter of "best practice" it is recommended that people do not use an Administrator account for daily work, especially web surfing. That is because for programs, like malware, to get access to the core of Windows they need "Admin rights". The idea start 10-20 years ago when logging in as admin gave you "Full Administrator Access" all the time. So once you logged in admin, malware would automatically run as admin. BAD THING! So the general user security rule, "don't do daily work when logged in as admin". It is less of a concern now that UAC, User Access Control, is in place and developing. Now, even when logged in as Admin you have standard access, so you get UAC prompts for admin password.
