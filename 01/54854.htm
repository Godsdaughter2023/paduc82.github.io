Not documented, but i should enable some registry settings and run ps script. 

1) DG_Readiness_Tool_v3.5.ps1 -Capable

   [ IF NOT CAPABLE DONT CONTINUE]

2) backup all work files. this can be your last boot in case of error

3) dism /Online /Enable-Feature /FeatureName:Microsoft-Hyper-V-Hypervisor /all

4) Go to HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\DeviceGuard.

Add a new DWORD value named EnableVirtualizationBasedSecurity. Set the value of this registry setting to 1 to enable virtualization-based security and set it to 0 to disable it. Set to 1

  • Add a new DWORD value named RequirePlatformSecurityFeatures. Set the value of this registry setting to 1 to use Secure Boot only or set it to 3 to use Secure Boot and DMA protection. Set to 3

5) 

  • Go to HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\LSA.
  • Add a new DWORD value named LsaCfgFlags. Set the value of this registry setting to 1 to enable Windows Defender Credential Guard with UEFI lock, set it to 2 to enable Windows Defender Credential Guard without lock, and set it to 0 to disable it. Set to 2

6) DG_Readiness_Tool_v3.5.ps1 -Enable -AutoReboot

7) reboot 

8) run hyper-v!

This issue has been around for a long time, and it seems to have a very high incidence – judging from the fact that the main thread for this specific error message has over half a million views. Initially, the solution was to use Autoruns to remove the “startup entry” for the Background Container.dll malware (the malware itself has already been removed, but its removal causes an orphaned “startup entry” to trigger the “module could not be found” error message).

Once we were informed that the “startup entry” for BackgroundContainer.dll is actually a scheduled task, it was obvious that there was a simpler and safer solution that doesn’t require any downloads at all: Just delete the Background Container startup task in Task Scheduler:

http://answers.microsoft.com/en-us/protect/forum/protect_other-protect_scanning/error-message-on-start-up-windows-8/7f29c7ad-2d55-4f1c-8110-ddca3902f596

Check if graphics card in the device manager is properly installed. Try re-installing drivers/software available in the graphics card manufacturers website.

Just a few questions to get a clear picture of this issue:
1. What changes did you make in the registry?

2. What is the make and model of your computer? what is the model of the graphics card?
 
First check the functionality of the graphics cards:
1. Press the Windows + X key together.
2. Select Device Manager.
3. Expand the Display adapters section, you should see your graphics card.
4. Right click on it, and click "Properties." Under "Device Status," it should say "This device is working properly."

If you are not able to check the status/properties here, update the latest drivers for the graphics cards.
 

1. Press the Windows key and the X key together.
2. Select Device Manager.
3. Expand the Display adapters section.
4. Right-click the Intel/AMD graphics entry and select Update Driver Software.
