bitwiseken say: Here are my final conclusions about this issue...  and thanks to everyone who replied.

I ran chkdsk in read-only mode (while the OS is running) and it reported serious errors (MFT errors).

I then ran chkdsk /f at restart time and it fixes whatever it fixes.

I then run chkdsk in read-only mode again and it still reports the serious errors, as if nothing has been fixed yet.

This can continue over and over again, making it very frustrating and alarming.

THE SECRET IS: DO NOT TRUST WHAT CHKDSK IN READ-ONLY MODE TELLS YOU. IN FACT DO NOT EVEN RUN IT AT ALL. ONLY USE CHKDSK /F AND BELIEVE WHAT IT TELLS YOU!

Chkdsk /f runs before the operating system is active, so the file system is in a static state that chkdsk can deal with in an accurate way.

With chkdsk /f, you have to go see the results in the event viewer, and you may have to run it a few times in this mode to correct all problems, but believe what it tells you and never believe what chkdsk in read-only mode tells you. In fact, IMHO, MS should remove the ability to run it in read-only mode!

1. Turn off Scheduled defragmentation via Windows Orb (Start) > All Programs > Accessories > System Tools > Disk Defragmentation. However, every so often you need to check if any disks have become excessively fragmented, which can affect disk performance.
2. Check if your antivirus program is scanning at boot time. This can be run when you feel it needs to be run, not at every start up. Also, disable it for a while to determine if the real-time protection is the culprit.
3. You’ll get a slightly faster start up if you optimise the boot files and applications by running a special defragmentation from an elevated CMD prompt, i.e. click the Windows Orb (Start) > All Programs > Accessories and right-click Command Prompt, then ‘Run as Administrator’. 5. Although hard disk errors are rare, they can slow up the machine, so it‘s worth spending a bit of time checking. Click Computer > right-click the hard disk drive that you want to check > Properties > Tools Tab, and then, under Error-checking, click Check Now.
