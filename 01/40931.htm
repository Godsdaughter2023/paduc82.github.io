lemp say:

Pardon me for jumping in, but there's an inconsistency in your description.

In your first response, you provided the "line with the problem" and wrote the final parameter as

     /summaryoutputfile="C:\Users\vlt-oneclick-1\Desktop\Copy of Jeffs Weekly Pants Stability.xlsx"

There are spaces in the file name.

In your response when you provided the complete batch file, you wrote the final parameter as

     /summaryoutputfile="C:\Users\vlt-oneclick-1\Desktop\JeffWeeklyPantsStability.xlsx"

There are no spaces in the file name and "Copy of" isn't there.

If you are experiencing the problem with the parameter /summaryoutputfile value having spaces, it seems likely that the problem is not how your batch file is being processed but how the Visual Basic in your Excel macro is processing the parameters passed to it.

Does your macro attempt to create a new summary output file each time it runs or add to an existing file?  It would seem to be the latter, because otherwise you shouldn't get a "couldn't find" error.

As Frederik points out, because the argument for the /summaryoutput file parameter is enclosed in double-quotes, that entire string will be passed to your macro without any further processing.  If there are spaces in the file name, your macro's code is recognizing the spaces as delimiters and attempts to set the summary output file as C:\Users\vlt-oneclick-1\Desktop\Jeff (it's not at all clear why the macro keeps trying to find files for the other words in the parameter).

Try it with a simpler file name for the summaryoutputfile parameter.

It's probably not related to your issue, but in your invocation of Excel from the command line, you have the /r and /e switches followed by a number of parameters passed to your Excel macro. 

The /r switch is supposed to be followed by by a file name, and I don't know what behavior occurs without the file name.

 /r <file name>    Forces Excel to open the specified file in read-only mode.

https://support.microsoft.com/en-us/help/291288/description-of-the-startup-switches-for-excel
