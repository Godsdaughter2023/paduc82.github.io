Can no longer boot to Windows - BIOS no longer recognizes hard drive - Microsoft Community

j w stuart say:

Can no longer boot to Windows - BIOS no longer recognizes hard drive

Can no longer boot to Windows - BIOS no longer recognizes hard drive

If you already have a motherboard that uses the new P67/H67 chipset and an Intel 2000 series (Sandy Bridge) processor, the problem relates to connectivity between the SATA ports and hard drives. That link can degrade over time and, in a worst-case scenario, you’ll boot your machine to find attached storage simply isn’t identified at all.

None of your data is at risk—anything on the drive already can’t be affected by the link degrading and ultimately failing, after all.

Read the complete article here:
http://www.tomshardware.com/news/cougar-point-sandy-bridge-sata-error,12108.html

The Source of Intel's Cougar Point SATA Bug: http://www.anandtech.com/show/4143/the-source-of-intels-cougar-point-sata-bug

"The problem in the chipset was traced back to a transistor in the 3Gbps PLL clocking tree. The aforementioned transistor has a very thin gate oxide, which allows you to turn it on with a very low voltage. Unfortunately in this case Intel biased the transistor with too high of a voltage, resulting in higher than expected leakage current. Depending on the physical characteristics of the transistor the leakage current here can increase over time which can ultimately result in this failure on the 3Gbps ports. The fact that the 3Gbps and 6Gbps circuits have their own independent clocking trees is what ensures that this problem is limited to only ports 2 - 5 off the controller."

dalavi say:

Windows Photo Viewer colour problem after applying ICC from X-Rite i1 Display Pro

As is shown above in the screenshot, Windows Photo Viewer seem to be handling the colour in an absolutely wrong way, but everything except WPV  is presenting the correct colour.

I used X-Rite i1 Display Pro to create the ICC profile. and applied the profile as the default profile.

Can you guys help with this problem? 

Thx!

dalavi say:

Fine, Problem Solved.

It's not the problem with drivers or hardware error.

It's just that you need to use a L*a*b Look Up Table for your profile, don't use anything else. An XYZ LUT + matrix or swapped matrix may do the job, but they may cause Chrome and Firefox to interpret colour erroneously. A gamma curve profile will cause problems with WPV, most probably it also alters the gamma inside of WPV, due to the sequence WPV deals with gamma - the problem an LUT does not have. 

So the conclusion is: USE LAB LUTS!
