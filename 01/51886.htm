Hi DougBermingham:

Many Vista SP2 and Win 7 SP1 users are currently having the same problem. If you leave Windows Update running for several hours and don't see an error message you might find that "Checking for updates..." will eventually run to completion and report that updates are available for download.


If that doesn't help, please see the step-by-step instructions I posted on 16-Jun-2016 in m#l's thread Updates not working, it has been searching for updates for hours.

Those instructions are based on the how-to on Dalai's webpage Search for Windows Updates takes forever? - A possible solution at http://wu.krelay.de/en/ and take you through the process of finding any Windows kernel-mode driver (Win32K.sys) updates currently missing from your system, downloading the standalone update packages (.msu files) from the Microsoft Download Center, and pre-installing those missing updates (in order, from oldest to newest) before running Windows Update to install your remaining updates.  Most affected users find that this workaround decreases "Checking for updates..." from several hours to ~ 20 min.

If you were fully patched as of May 2016 there should only be one missing Win32K.sys update - KB3161664 - that you have to pre-install before running Windows Update again to install this month's remaining dozen or so June 2016 Patch Tuesday updates.

Your comment regarding the digicab file leads me to believe that you were trying to follow the instructions in KB949104 - How to update the Windows Update Agent to the latest version, but I don't believe the Easy Fix Tool there works with Vista machines.  The Download button prompts users to download a file called microsofteasyfix20179.mini.diagcab but the Microsoft Easy Fix FAQ <here> states that "diagcab solutions are for Windows 7 and later versions of Windows only. Windows Vista and Windows XP cannot run .diagcab solutions."

The Windows Update Agent (WUA) for Vista is located at C:\Windows\system32\wuaueng.dll and the latest available version of this file is v7.6.7600.256 (released June 2012). The WUA for Vista hasn't been updated in over 4 years so most Vista users already have this version of the WUA - unless they've recently rolled back their Vista OS to an earlier date.

trekdozer say: To start installing 64 bit requires restarting and booting from the 64 bit disk - have you tried this?

Installing Windows 64 bit with a retail disk:

A. Insert the install disk, restart and boot from the DVD

B. Click Install Now

C. At the setup screen, click Custom (Advanced)

D. Choose the partition to install Windows on

E. Continue with the install



If you can't boot from the DVD:

BIOS - How to Enter the BIOS and Change Boot Order To Boot From CD/DVD

The computer may need to be set to boot from the DVD drive.

Every computer is different, when you first turn on the computer it should tell you what button to push to enter the BIOS. It's usually one of the following and you must push it within the first 5 seconds:

F1 F2 F10 F12 ESC DEL

Once in the BIOS you have to find the menu with the boot order and change it so it boots from the CD/DVD drive first. Every computer is a little different for this process as well so I can't give exact directions.






Just some troubleshooting steps to see if the disk is bad or if the DVD drive is bad:
If you have another DVD drive available try using that drive in your computer instead.
If you have another computer available try the Windows 7 disk using that computer and see if it can read the disk, or if the disk can't be read and is bad.
Also, if you bought the retail version of Windows 7 try putting in the other bit type disk of Windows 7 disk and see if that works properly.
If you can't boot from the Windows 7 install disk, can your computer boot from other bootable disks?
If you're using the 64 bit disk - can your computer run 64 bit? Use the Upgrade Advisor to check:
http://windows.microsoft.com/en-US/windows/downloads/upgrade-advisor
