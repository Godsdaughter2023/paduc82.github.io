    Have been using a USB connected smart phone as a ICS sharing device for my home network for years. Since the update it is no longer possible to change the sharing setting option in the adapter settings. Attempting to mark as off causes the program to hang (only resolvable by ending the task). Attempting to mark as on causes an error message "Internet Connection Sharing is currently enabled for connection 'Ethernet_32772'. It will now be enabled for connection 'Ethernet 2' instead". Pressing OK causes a second pop-up "An error occurred while Internet Connection Sharing was being enabled. (null)". It never displays as ICS on.

    If after a system reboot the USB adapter shows the ICS option 'ticked', the Ethernet adapter (connected to my home network router) never updates the IP address to 192.168.137.1.

    Like other reported problems the ICS service is never started - it only displays as Starting...

    I've reported this not to ask for help - but to report as a bug with the update release 1709 - so that others wont waste as much time as I have trying to get ICS working. I'll report as fixed once Microsoft fix the bug.
