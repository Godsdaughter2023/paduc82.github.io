I'm sorry to hear you got caught in this update debacle, too.  I never did find (nor did anyone else ever suggest) a workable solution to this crash.  Once I retrieved my other (still-working) Windows 10 computer from my remote office, I did some further investigation with it, and discovered that my computers WILL boot from a flash drive, even though that option is not explicitly listed in the BIOS settings.  I therefore found a 4GB flash drive and used the working computer to create a new recovery drive, on which I included copies of all the system files.  I then booted the crashed computer from that recovery drive, and attempted all of the repair options that were provided, which completely failed.  The damage the FCU installation did to my existing Windows 10 Version 1703 was irreparable...I could find no way to undo the FCU installation, revert back to Version 1703, reactivate the files in the "Windows.old" folder on the crashed hard drive, or otherwise repair the damage.

Because the only suggestion short of a clean installation that Michelle Vil offered was to attempt a repair using an ISO, I assumed that was not worth trying because the recovery-drive option was a failure, and the recovery drive was customized to my exact computer, rather than just being a generic copy of the operating system installation files.  I therefore "gave up" at that point, removed the corrupt hard drive from the crashed computer, temporarily installed it in place of the DVD drive in my still-working computer, and cloned the still-working hard drive over the data on the corrupt one.  I then installed the cloned hard drive back into the crashed computer, which then (predictably) booted-up and worked perfectly fine thereafter.  There was some housekeeping to do, such as changing the computer name, hard-drive label, and share names back to the original values for the crashed computer, and tweaking a few e-mail settings, but that was hugely easier than doing a clean install and having to re-create all of my customized settings and reinstall of of my programs and data.

My main goal in making my original posting back on December 1st was to make sure that Microsoft was aware that the update had the potential to crash some well-maintained and fully up-to-date computers, and that the pre-installation verification procedure to check that the computer could successfully run the new version needed to be strengthened to insure that the update would NOT install on hardware incapable of running it.  While I make no claim to know this for certain, my suspicion is that the FCU includes graphics "improvements" to the operating system that my low-end video card simply cannot run, even though it IS capable of running the Version 1703 of Windows 10 without major problems.  If that suspicion is correct, the FCU should be programmed to check for such incompatibilities, and prevented from attempting to install itself if they are found, rather than allowed to simply crash people's computers.  I had a fairly straightforward recovery option because I maintain two identical computers for just such contingencies, but if your ONLY computer has been completely disabled by the FCU update, that's a big deal.

As a protective measure, after both of my computers were working properly again, I changed the advanced settings on my Windows Updates to set my branch readiness level to "current branch for business," and to defer feature updates for 365 days, all in an attempt to make sure the FCU will not try to install itself again anytime soon.  Perhaps, during the next year, Microsoft will "tweak" the FCU so that it will not crash my computers the next time it tries to install itself.

Just to put a period at the end of this experience, I have now been able to successfully install both the FCU and the April 2018 updates on my Dell Optiplex 755 computers, but only after researching which PCIe video cards would run DirectX 12, physically fit in my computers, provide the output ports I needed, and operate properly on the limited power available on my particular motherboards.  I initially bought two new NVidia GeForce GT-710 cards, and while they seemed to be compatible at first, further daily usage proved they were not...they created many problems with driver crashes and resets, and audio problems while surfing websites, even after performing clean, minimal driver installations.  When the driver crashes became unbearable (sometimes up to five per hour), I gave up on the GeForce cards and bought used AMD Radeon R5-240 cards, which required buying (and modifying) full-height brackets and DisplayPort-to-HDMI plug-in adapters.  The Radeon cards and their drivers work fine, never crash, and create no audio problems, so I then cautiously reset one of my computers to receive Windows 10 updates without any delay.  After a lengthy updating process, the first computer booted fine with both the FCU and April 2018 updates installed, and has been running fine ever since.  The second computer was also updated to fully current status without any problems.

That seems to be pretty conclusive proof that the hardware limitations of my original EAH-3450 video cards...and most likely their inability to run DirectX-12...was the cause of the original crash and boot-loop.  While it was not exactly quick or easy to find a DirectX-12 graphics card that would work with my older computer, once a suitable card was found and installed, the two most recent Windows 10 updates were trouble-free.  It would certainly make life easier for the customers if these updates could be programmed to check for hardware-compatibility problems BEFORE attempting to install themselves, and notify the customer of any necessary upgrades FIRST, rather than crashing a computer that only needs a fairly straightforward hardware upgrade.  Live and learn....

Hi, CharlieSmith4

My name is Phillip and I am an Independent Advisor. I hope it helps.

I would recommend you to check your Hard Disk HDD with chkdsk.
1) On your keyboard, press the Windows logo key and S key at the same time, then type cmd in the search box.
2) Right-click Command Prompt, then select Run as administrator.
3) Type chkdsk /f /r, then press Enter on your keyboard.
4) Press Y on your keyboard.
5) Press the Windows logo key and R at the same time, then type mdsched.exe.
6) Click Restart now and check for problems (recommended). Your PC will restart immediately.
7) Wait for the scans to finish.

Diagnose memory problems
Using these steps requires you to restart your computer, so make sure to save any files you may have open and close any running applications before proceeding. Here goes:
1.- Open Control Panel.
2.- Click on System and Security.
3.- Click on Administrative Tools.
4.- Double-click the Windows Memory Diagnostic shortcut. Quick tip: Alternatively, you can also use the Windows key + R keyboard shortcut, type mdsched.exe, and click OK to launch the tool.
5.- Click the Restart now and check problems option. (The tool also offers an option to schedule a test in the next reboot, if you can't afford to close your applications and restart immediately.)
Once your computer reboots, it will start in the Windows Memory Diagnostics environment, and the tests will kick in the Standard mode right away. In this mode, the tool will run all the scans available in the Basic mode, in addition to LRAND, Stride6 (cache enabled), CHCKR3, WMATS+, and WINVC tests.
You can simply let the Standard mode test your computer's memory, or you can also press the F1 key at any time to open the Windows Memory Diagnostic page to change the scan options.

Hope this helps. Keep us posted on the status for further assistance.

Insert the USB flash Drive

Boot into BIOS - on the Manufacturers logo screen at the start, you will need to repeatedly press the appropriate key to get into setup. That key varies from system to system, usually it is F2 or F10

Change the boot order, so that the flash drive is top of the boot order list

a. user say:

Hello Patrick,

Thank you for writing to us.

From the description provided, I understand that the sign in window either becomes responsive or just crashes out without any error message.

If there is any functionality issue with regards to the issue being discussed over here,

  • Press Windows key + X, select Command prompt (Admin) to bring up elevated Command prompt.

  • In Command prompt type sfc/scannow and press enter.

  • Restart the computer.

If no corrupted system files were found in the previous state, then I suggest you to try the following steps to employ the DISM command lines: 

  • Open command prompt by pressing Windows key + X and selecting “Command prompt admin”.

  • In command prompt, type the following commands and press Enter after each command:

                DISM.exe /Online /Cleanup-image /Scanhealth

    DISM.exe /Online /Cleanup-image /Cleanupimage

                DISM.exe /Online /Cleanup-image /Restorehealth

  • Close the command prompt and restart the PC and check if it works.

Hope the above information is helpful. If you need further information, feel fre to write to us and we would ber happy to help you.

Thank you.
