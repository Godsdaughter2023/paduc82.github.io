arminatorx say:

[quote user="Anaerin360"]I changed my Microsoft password to a nice, secure, 17-character password.[/quote]

No matter how long you make up your password, a password can eventually be leaked or otherwise be stolen.

If you REALLY want to make your account absolutely password proof, turn on Two Step Verification and set up the free authenticator app on your smartphone.

This has two advantages:

First, even if someone gets hold of your password somehow (e.g. a keyboard sniffer on a computer in an internet cafe during your holiday trip), without the additional code that is generated by your phone, your account can't be accessed. So an account hijacker would also have to physically steal your phone as well.

Without the phone, there is no access to your account.

However, since the Xbox 360 can't work with the two step verification code yet, you generate an additional App Password on the Two Step Verification site, and use THAT on the Xbox 360 instead of your regular password.

And since that app password can't be used for regular account access (only for special apps or devices like the Xbox 360), you are double secure, even with a "short" 16 character password.

So please use Two Step Verification in addition to your password to make your account "bulletproof".

aileen alf say:

Hi Pynd,

Since you continue to experience the issue after following the troubleshooting steps provided by seahawks65, I suggest that you adjust the page file size or virtual memory in Windows 10. Adjusting the page file size is known for regulating the disk usage and it dramatically improves Windows 10 performance.

Here's how:

  1. In the search box, type Adjust the appearance and performance of Windows and press Enter.
  2. Go to Advanced tab and click Change... button in the Virtual memory section.
  3. At the top, uncheck the box right next to the Automatically manage paging file size for all drives option.
  4. Select drive C: (or the drive where Windows 10 is installed).
  5. Select Custom size: by ticking its radio button, and change the Initial size (MB) and Maximum size (MB) to the Recommended value that you will find just below the Total paging file size for all drivers section.

This method will help improve the system performance of Windows 10, and will gradually decrease the freeze issues.

hairyfool say:

Although the operating system could affect run time due to the load it puts on the battery it cannot have any influence directly on the charge time. That is entirely dependent on the power supply and the controller built into the battery.

The reduced run time could be due to the battery not really being fully charged or the low battery warning triggering too early.

 A true charge time can only be established if the battery is really flat. To do that you need to override the battery control built into Windows. If the BIOS setup has a memory test or use a bootable hard disk test tool to run the system until it goes down. Then plug in the PSU and see how long it takes to charge without switching on.

hairyfool say:

It is not so much about fixing it, more trying to identify what your real problem is.

You can't really say how long a battery takes to charge unless you know how flat it is. If Windows is misreporting the battery condition which is possible then you need to get a reliable benchmark. Flat is about the most reliable one.

How long it takes to get there depends on how you can work the system which is why I use a disk tester, exercising the drive pulls a good current. Recharging from flat with computer off should take a few hours.
