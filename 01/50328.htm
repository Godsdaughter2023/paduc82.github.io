The mechanism for putting a website shortcut on your desktop is to right-click the desktop, choose "New > Shortcut" and paste in the website URL you want the icon to be for.

The favorites stored in the app's package data folder were never intended to be "user facing".  There's nothing wrong with you exploring those files, it's your computer after all, but they didn't design it thinking that anyone ever would. The design was to use the application's direct interface UI to manage it.

Documentation only exists for the designed use of the product (how it was intended to be used), so there's no documentation for the inner proprietary workings of how it achieves it under the hood.  As an anology: your car manual tells you about the features and functions to use the car, not how to rebuild the engine and put in custom parts.  Doesn't mean you can't do it, but it means manufacturer isn't obligated to support it.

Actually in this type of problem (0xc000007b) the cause maybe because of the “msvcp110.dll”  file located at C:\Windows\SysWOW64. Several applications are allowed to run by using this folder, but each application have their own separate version of the file with the same name.
