mrhon say:

FILE TYPE issue with Windows 7

In Windows XP you could modify the FILE TYPES to allow for additional parms to use a paticular definition to access a file.  We have a .DTV file that has parms, but a customer would like to just double click on the file and it knows how to open it... how can I do that with Windows 7 that doesn't have FILE TYPES like in Windows XP?

However, the file type and association commands (ftype and assoc) can be typed via a command prompt. Alternatively, registry changes can be made.

 

The ftype command associates a file type with an application. For example, if you had a file type called myaccs that had a program file named ledger.exe and used files with the .ldg extension, the manual assoc and ftype commands typed into a cmd window would be:

assoc .ldg=myaccs

assoc .acc=myaccs (i.e. more than one extension can be associated to a file type)

ftype myaccs= %SystemRoot%\Program Files\ledger.exe %1

 

Now, when you double-click on a file with the extension .ldg or .acc, Windows will recognize it and launch ledger.exe using the optionally typed parameter.

 

An easier to understand example of the above is a .txt file, which we all use in Notepad:

Typing assoc .txt will return txtfile (the file type)

Typing ftype txtfile will return %SystemRoot%\System32\notepad.exe %1 (the full path of the program and %1 representing a passed parameter).

 

Alternatively, look in the registry hive HKEY_CLASSES_ROOT for .txt and further down for txtfile.

When a Li-ion battery loses a certain percentage of it's charging capacity, you may see inconsistencies, such as the one you are describing in the charging circuit for the laptop.

I would recommend that you create a battery report to check the status of the battery. Perform the following.

powercfg -energy -output %userprofile%\desktop\energy-report.html

 

Press ENTER.

 

Wait until the command completes (usually about 2 minutes).

Design Capacity xxxxx
