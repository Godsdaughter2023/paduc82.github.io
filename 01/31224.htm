enterprised1 say: By default, the command prompt will only look in the directory you're currently in for the command you type.

For example, if you're using the xcopy command, it will only work from c:\windows\system32 since that's where it's located.  If you go to a different directory, say c:\, it technically won't work.

The way Windows gets around that is by using the PATH variable.  Anything you set as the PATH will also be checked by windows for the command you're trying to run.

If you use the following
PATH=c:\windows; c:\windows\system32; c:\program files,
Windows will check each of those directories for the xcopy.exe command.  If it finds xcopy.exe in any of those directories, it'll run the command regardless of what directory you're currently in.

Having said that, c:\windows\system32 should already be included in the PATH variable.

To check, go to Control Panel --> System --> Advanced System Settings
Click on Environment Variables
In the System Variables section, scroll down and click on the PATH variable
Click Edit
Make sure C:\Windows\System32 is somewhere in the path.

If it's not in the path, just type it in at the end and it should work.

It sounds as if you have a mapping problem with your IMAP folders. Review all the settings; here are some suggestions - you may find on experimentation that different settings work better:

  2. Map the local special folders to the IMAP folders on the server
    Right-click on the account name in the folder pane and select     Properties.
    On the IMAP tab, select     Store special folders on IMAP server,  then enter the server's names for the special folders in the boxes. If the folder for junk email at the server is called Spam, then enter Spam (not spam or SPAM) into the box alongside Junk email path and so on.   3. Make Deleted items folders work the same way for all accounts
    When viewing your message list, press Ctrl-Shift-O for Options.

The "email program associated" refers to an email program that is installed on the computer, like Office Outlook or Thunderbird.

[Webmail services like GMail, Outlook.com, Yahoo Mail, etc. are not email programs. They aren't installed on the computer. They're services that come from the internet. Windows Mail is not an email program.  It's an interface to webmail services.]

"There is no email program associated ..." appears if you - or someone else who has access to your user account - tried to send email directly from the computer at a time when no email program was installed or, if an email program was installed, it was not selected as the default email program. The reminder appears until you either send the email or cancel the email.

If you have never before tried to send an email directly from your computer ... someone else did.
