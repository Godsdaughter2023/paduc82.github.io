Other information that might help :

Control Panel - Power Options - Power Plans - Change plan Settings - check the Advanced
Settings for drain and usage settings (this is how much to be used and not how much or how
long to charge).

Actually what causes a lot of wear on a battery is draining it too low repeatedly. With today's
systems over-charging is not an issue.

Here are some tips to help and troubleshoot battery related issues.

Old battery? Unplug power to computer - remove battery and clean the contacts with a pencil
eraser (do not use this if your battery has thin slots - just clean the knife edges that fit into
them and be careful). Batteries do wear out if old, or it could be a problem with the computer.
Check with the support of the system maker and many of them have on-line forums.

Newer type Litium-Ion batteries usually last longer if you do not discharge them below 30%
however on a laptop that will not extend their life that much. Best is to use A/C adapter
whenever possible.

Yellow box on right side of page to download latest stable version.

The kernel power event ID 41 error occurs when the computer is shut down, or it restarts unexpectedly. When a computer that is running Windows starts, a check is performed to determine whether the computer was shut down cleanly. If the computer was not shut down cleanly, a Kernel Power Event 41 message is generated.

An event 41 is used to report that something unexpected happened that prevented Windows from shutting down correctly. There may be insufficient information to explicitly define what happened. To determine what may have happened and to identify a potential resolution, it is important to know what the computer was doing at the time just before the event occurred.

If event 41 is logged because power to the computer was interrupted, consider obtaining an uninterruptible power supply (UPS) such as a battery backup power supply. An underpowered or failing power supply may cause this behavior. For example, if you added RAM or additional devices or hard disks when this problem began, the power supply may cause the problem.

hairyfool say:

How did you do the "complete" backup? If you created a system image then that is exactly what it is. A complete image including Windows itself. To restore that you do not want Windows on the computer, it should be completely blanked.

As part of creating the system image you should had the option to make a Recovery drive as well. This contains a bootable stripped down version of Windows capable of then taking that system image and expanding it onto the blank drive (sometimes referred to as bare metal recovery). The only caveat is the new drive MUST be the same size or larger, even slightly smaller and it will not restore.

You can still make that Recovery drive, boot from it and clear down your disk to restore the system image. It is also possible to connect to the image and copy off files from it. What you cannot do is copy programs, they can only be Installed using the setup routine provided on the original media. Setup integrates programs into Windows itself which would not happen just by copying files.

If your complete backup is just a copy of all the files then your only option is to reinstall your programs
