computernerds say:

After months of researching, giving up and then taking another look at this shunimpl.dll problem I finally came up with a solution that seems to be working thus far. Here is how I fixed it on a Windows 7 x64bit OS.

 

Go to C:\Windows\syswow64 and locate shunimpl.dll and copy it to your desktop just in case you need it again. Then delete shunimpl.dll from the syswow64 folder. You will most likely get a message stating you do not have permissions to delete a system file. So you will need to take ownership of the file. Go to http://www.howtogeek.com/howto/windows-vista/add-take-ownership-to-explorer-right-click-menu-in-vista/ and add the context menu for taking complete ownership of a file. Right-click on shinimpl.dll in the syswow64 folder and select "Take Ownership." Now delete the file. Now try the program that was giving you the shunimpl.dll error and it should now go away. I received the shunimpl.dll error message every time I tried running a program in compatibility mode so test that as well.

 

I assume if you are having the same problem in a x86 OS you will have to delete the shunimpl.dll in the system32 folder.

 

I hope this helps someone else out there.

 

Good luck!
