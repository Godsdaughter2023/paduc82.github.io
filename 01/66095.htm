winston m say:

Hi Srinivasan,

Thank you for posting the query in Microsoft Community Forums.

This issue normally occurs when the services do not respond to the operation and cause different conflicts. It could also occur due to any third party programs.

Method 1:

You may try to all the services that run Internet Connection Sharing(ICS) and check. 

1. Click Start, Run, type services.msc, and click OK.
2. To enable a service, find it in the right pane and double-click its entry.
3. Be sure its Startup Type is set to Automatic and click OK.

ICS depends on four other services namely:

1. Base Filtering Engine  (default startup: Automatic)
2. Network Connections (default startup: Manual)
3. Remote Access Connection Manager  (default startup: Manual)
4. Windows Management Instrumentation  (default startup: Automatic)

See if you can start all of the above four services first, and set their startup types as given above.
Certain viruses/malware may wipe out Base Filtering Engine in an attempt to disrupt the functioning of Windows Firewall and other critical services. To know if BFE is running in your system, click Start, and type this command:


After verifying and/or configuring Automatic startup for each of these services, close the Services window and restart the computer before attempting again to enable ICS.

Method 2:

Error could be caused by your security software blocking Internet access/sharing. I suggest you disable all security software and check if the issue persists.


Note: Antivirus software can help protect your computer against viruses and other security threats. In most cases, you shouldn't disable your antivirus software. If you have to temporarily disable it to install other software, you should re-enable it as soon as you're done. If you're connected to the Internet or a network while your antivirus software is disabled, your computer is vulnerable to attacks.

Method 3:

If the issue still persists, you may perform SFC scan to check if there is any system corrupted files.

Use the System File Checker tool to repair missing or corrupted system files

http://support.microsoft.com/kb/929833/en-us

              fatal error 1603

Is usually a summary code which means "something's wrong".  See if there are any other diagnostics being produced by your install attempt.  You can use File Explorer Search for files which were changed in that interval.  Alternatively use ProcMon to trace it the next time you try the install.  Filter with Operation Is WriteFile to find out what files are being changed.  Some of them are likely to be log files containing supplementary clues about your problem symptom.
