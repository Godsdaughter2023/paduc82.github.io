robert aldwinckle on forums say:
have four HID-compliant device drivers.

Only 4?   <eg>


See if you can uniquely identify the one you want by a part of its description.   Then (perhaps) you could use wmic  to do what you want precisely.


E.g. starting with


wmic.exe  path  Win32_PnPEntity  where  (Description  like  "%touch%")  get  Description


I found that the one I wanted was the only one which would contain the word  Multitouch, so then I could do this to list its details:


wmic.exe  path  Win32_PnPEntity  where  (Description  like  "%Multitouch%")   get  /format:list


In fact, then it might be easier to do with the same kind of query using Powershell:


Get-WmiObject Win32_PnPEntity | Where-Object {$_.Description -Match "Multitouch"}


E.g. the "verb" is understood to be  Format-List.   ; )


So, now I have a way of listing things like the DeviceID  and  Status  for the thing I want to change.


Ideally, at this point there would be a way of simply specifying either a Call for wmic to execute or some other cmdlet for Powershell to pipe this device object to.   Unfortunately I haven't figured that out yet.  Nevertheless, now I can at least go into Device Manager and find something specific to try to disable, confident that I am going to pick the right thing (or at least that I am going to have a way of checking that I can undo a change to a device if I pick the wrong one <eg>).


In fact, in my case, the Multitouch device is not listed where we were guessing it would be.   I found it listed under  N-trig  DuoSense (R) Digitizers.   Then I used the Details tab to check that it really was the Device I wanted to disable.   Then I disabled it.


Consequently,  I now have no Touch on my screen--not just no Multi-touch--no Touch.   But my pen is still working.   If I also wanted to disable it too evidently I would have to find another driver to disable for it.


Also, redoing my queries both in wmic and Powershell finds that the Status has changed to Error.  StatusInfo is still blank but  ConfigManagerErrorCode=22.   When I go back into Device Manager for this device it shows a down arrow overlay of its icon and its Properties General tab says:  This device is disabled.  (Code 22)
