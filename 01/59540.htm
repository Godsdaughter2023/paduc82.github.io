freeespirit say:

I resolved the 0x8100001A problem only after I realized that my external backup drive letter didn't match the original backup volume that I had created.  I discovered this problem when I first ran sdutil.exe and it failed,  in the error message it told me my original backup volume letter.  So I then used the disk management utility -- windows 7 control panel->system&security->Admin Tools->computer management->storage->disk management to change the mount volume to match the original.  Within disk management: Right click on the backup volume and change the volume letter to match.

 

Once I did that I was able to successfully run sdutil.exe to create the MediaId.bin file as explained in your link.  Keep in mind you need to run "backup and restore" again and make sure to click on "select another backup to restore from" in order to force it to re-read the new MediaId.bin file.  Then you can proceed to restore files....good luck.
