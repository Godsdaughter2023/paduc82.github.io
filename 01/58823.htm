How to reinstall Windows 8

You have a couple options:

If you would like to reinstall Windows 8, which means removing apps and personal files and settings, you can use reset and refresh. This is equivalent to a clean install.

To do it, do the following:

· Press Windows key + i on your keyboard to show the Charms bar (If you are using a touchscreen: Touch the right edge of your screen and swipe your finger to the left)

· Click Settings

· Click Change PC settings

· Click General in the left column

· Under Remove everything and reinstall Windows, click Get started

Then follow the instructions that will be provided on the screen to reinstall Windows on your PC.

Option 2:

If you create a installation DVD or Bootable thumbdrive, you can use that to reinstall Windows. First, you will need to reinstall the qualifying license, then start the upgrade again to Windows 8. Qualifying license means you will either need to reinstall Windows XP, Vista or Windows 7.

Option 3:

A workaround for this situation is to create a system image of Windows 8 just after you have upgraded. So if your drive does crash, you can simply restore the image on a new hard disk or restore the image on the same drive if you just want to reinstall Windos 8.

If I understand your problem description correctly, I know what you mean. This has been a recurring problem with Windows Live Mail, which uses Internet Explorer functions for message display. I think it arises because each version of WLMail is designed to work with the current (at the time of development) versions of the IE libraries. When these change, with the introduction of a new version of IE or sometimes just a bi-monthly Cumulative Security Update, something changes in the way WLMail displays messages.

Since installing IE10, I see the behaviour you describe. It applies particularly to plain text messages, where the font settings defined in program options are set aside in favour of some elusive 'default' settings. However, it doesn't always happen, and I haven't been able to pin down the trigger. When messages start being displayed with, say 12pt Arial instead of my preferred 10pt Courier New, it may be enough to close down the program, wait a few minutes and then re-launch it. If the display hasn't corrected itself, repeat the close/re-launch cycle. Whether the display changes back or not may depend on other factors, though, like whether IE is running or whether WLMail conducts some extensive housekeeping on shut-down or start-up. I haven't tested the behaviour rigorously in all the different situations. Sometimes it takes me by surprise and I suddenly notice that the display is as it should be.

Sadly, there's little we can do about it. The IE developers clearly don't concern themselves with the possible effects their code changes might have on legacy programs like WLMail, and the most the WLMail developer(s) can do is adapt their code to the status quo. This will, of course, at worst only last until the next change in the IE DLLs.

Incidentally, you may find that the same phenomenon affects the display of plain text files in Windows Explorer's preview pane. With IE9 installed, I seem to remember that the font used was Times New Roman. Now with IE10 it seems (at the moment) to be Courier New on my machine.
