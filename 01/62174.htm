As a field tech I have been dealing with these failed 1709 updates quite a bit.

Resetting your components CAN fix the issue - but in nearly all the cases where I get a black screen during 1709 it has been due to multiple video cards...   

What's actually happening behind the scenes is that Windows is booting into a SafePE environment from which it can apply the updates ..    if that environment is detecting and using the onboard video - then you'll get a black screen.

How to fix this?   I've had success (albeit limited) in removing the extra video card or disabling the onboard in bios to get it to work ..    In other situations, I've actually installed a different video card prior to the update and its worked.

If you have access to a PE image (from ADK for example) .. you could test booting into that ...  you should get the same black screen if its a compatibility issue or using wrong video device issue...

There seems to be a lot of "voodoo" fixes for the 1709 update.  Ultimately .. something is causing each update to fail ... its not a single common cause ...  so therefore success varies with each fix ... and there are many different fixes are out there ...
