1. Possible Faulty RAM

A common problem which can result in the 0xc0000005 error during Windows 7 installation is faulty RAM. You should run a memory diagnostic test to make sure all your ram is stable:

Follow the below steps.

  1. Type Windows Memory Diagnostic in search bar and click it

  2. In the Windows Memory Diagnostic window, select the first option which is recommended. As a result, your system will restart right away and will check for problems.

  3. After the computer has restarted, you will see that the Windows Memory Diagnostic Tool will be checking for memory problems. This diagnosis might take several minutes. Once the process has been completed, the system will restart and will open in normal condition.

  4. If the error occurs again, simply replace your memory modules. And that's it. In this manner, you can easily resolve the blue screen error of memory management.

2. Bad Sectors or Corrupt Hard Drive

If you have tried formatting your PC’s hard drive, and your RAM passed the diagnostics, then you could have bad sectors on your hard drive.

Try getting to the Windows recovery console and perform a force of the chkdsk command (chkdsk C: /f).

Please quickly provide a way to disable tilt correction, or even tilt sensitivity altogether, to alleviate the issue for the time being.

EDIT (2018-08-30): This question is no longer relevant. The original issue with the tilt correction was fixed by Microsoft on August 8th, 2018.

https://answers.microsoft.com/en-us/surface/forum/surfpronew-surfdrivers/surface-pro-intermittent-pen-inaccuracy-when-hand/28ef8566-0003-4c68-a964-6c2562423eb0(Note: the term "parallax offset" was inappropriate - I should have written "tilt offset")

With the new Surface Pen introduced with the Surface Pro 2017, Microsoft engineers have introduced a new system to compensate the "parallax offset", that is the difference of location between the tip of the pen and where the ink appears on the screen.

Before, this compensation was done by manually calibrating the pen, i.e. asking the user to hold the pen and tap some reference points on the screen. The measured offsets were stored and then used to shift the location of the ink on the screen so that it appears as close as possible to the tip of the pen. It was not perfect but very acceptable. In fact, previous Surface Pro models came pre-calibrated, so that they would suit most people without ever having to calibrate the pen manually.

Now, the compensation is done automatically in real time. By measuring the tilt of the pen, thanks to the new tilt sensor, the expected offset between the tip of the pen and the pen signal is calculated and the corresponding compensation is applied, so that the ink is displayed right below the tip of the pen. This system is much more precise.

Unfortunately, the current implementation suffers from a bug or hardware issue that frequently and randomly turns off the automatic parallax correction whenever a hand is in contact with the screen, making the pen very difficult if not impossible to use. This issue has been reported here by more than 2500 people:

https://answers.microsoft.com/en-us/surface/forum/surfpronew-surfdrivers/surface-pro-intermittent-pen-inaccuracy-when-hand/28ef8566-0003-4c68-a964-6c2562423eb0

After 2 months, a Microsoft agent replied that the root cause of this issue was identified and that a solution would be provided in a future Windows update. It has been 10 months since that reply, and Microsoft has not provided any fix, nor any update on the situation. It appears that Microsoft is unable to fix this issue.

In this situation, most users would be much happier to simply disable that faulty automatic parallax correction. It sounds like a common-sense solution to disable a feature that causes trouble when that feature is not essential. In fact, the Surface Studio does not support this automatic parallax correction (since it does not support tilt), and the new Surface Pen just works fine on the Surface Studio.

So, my question is: how to disable the automatic parallax correction on the Surface Pro 2017?

I tried disabling each of the Human Interface Devices in the device manager but none of them disabled the automatic parallax correction. I would like Microsoft to provide detailed instructions on how to do it, or if it cannot be done manually, provide a software or patch that can do it.

(Note: this question is not about solving the issue discussed in "Surface Pro - intermittent pen inaccuracy when hand is on the screen". It is just about a way to disable the automatic parallax correction.)

ramesh srinivasan say:

Hi,

Try clearing the thumbnail cache. Use these steps:

  1. Open File Explorer, and click This PC
  2. Select the system drive (usually, C:\)
  3. From the Manage tab at the top, click "Cleanup". (This launches Disk Cleanup for C:\ drive.)
  4. Enable the check box for "Thumbnails", and uncheck all other items in the list.
  5. Click OK
  6. When Disk Cleanup prompt you with the message "Are you sure you want to permanently delete these files?", click the "Delete Files" button.
  7. Restart Windows and let Windows regenerate fresh set of thumbnail cache as required.

Hope that helps.

ramesh srinivasan say:

I just checked here (Windows 10 x64 1511/10586.164) and found that the problem is with Windows Media Player's Rich Preview Handler. If this handler is disabled, the Preview pane refreshes the image correctly, but you lose the ability to "play" that video from the Preview pane. I mean, the control bar won't appear under the preview image.

On the other hand, if the WMP Preview Handler is enabled (default), the preview image fails to update, but you can still play the video from the preview pane. Hope this gets fixed in future updates.

Here are the REG files to enable and disable the handler.

;-------------------------------------

;Disable WMP Rich Preview Handler

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\PreviewHandlers]
"{031EE060-67BC-460d-8847-E4A7C5E45A27}"=-

;-------------------------------------

;-------------------------------------

;Enable WMP Rich Preview Handler

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\PreviewHandlers]
"{031EE060-67BC-460d-8847-E4A7C5E45A27}"="Windows Media Player Rich Preview Handler"

;-------------------------------------
