lemp say:
Windows is working the way it is supposed to work.  When amount of memory being used by the system and the applications running exceeds the sum of the physical memory (RAM) and the current page file, Windows automatically increases the size of the page file (unless your hard drive is close to full).  The message that you are seeing is Windows reacting to a heavy load on your system -- it is not the result of misconfiguring the page file.

When the message says "look in Help," you should do that.  Click Start > Help and Support and type
           virtual memory
into the search box.  Look for the topic titled "Managing your computer's performance."

A good explanation of how Windows XP deals with "virtual memory" starts here --> http://www.theeldergeek.com/paging_file.htm   You should particularly read the part on "Sizing the page file"

If your system is slow, it is because it is overloaded, either with applications that you are intentionally running or with malware (you didn't give the basis for your belief that you're pretty sure that you don't have a malware problem).  Take a look at the processes listed in Task Manager.  Do you know what they all are?  Do you really need for all of them to be running?  Which ones are taking up the greatest percentage of CPU cycles?  Don't bother looking at the "memory" column -- that won't help you much in determining what's slowing things down.

If your system is like most, you will likely have many instances of svchost.exe running.  This is normal.  You can use Process Explorer to see what svchost is actually doing (details here http://www.bleepingcomputer.com/tutorials/list-services-running-under-svchost.exe-process/) but you can try this easy way first:

Open a Command Prompt window (Start > Run > cmd > OK)
In the black Command Prompt window type the following line and press Enter

        tasklist /svc /fi "imagename eq svchost.exe"

[That's a lower-case "eff" "eye" after the slash -- short for filter]

To work around this error, create a temporary Windows TEMP folder in a new location. Then install the product and restore the original TEMP folder.
Using File Explorer, create a new folder with a simple name in the root of your C:\ drive, such as C:\TESTTEMP. Edit the folder permissions to give full control to everyone:
a. Right-click the folder and select Properties. 
b. Choose the Security tab. 
c. Click Edit to change permissions. 
d. For each group or user name, select Allow for Full control. 
e. Click OK.
Using a text editor such as Notepad, create a text file.
Go to Control Panel > System.
Click Advanced system settings in the left menu and select the Advanced tab.
Click Environment Variables.
Under User variables, double-click TEMP, copy the TEMP variable path value, and paste it into the text file you created in step 2.
Replace the TEMP variable value with the path to the TEMP directory you created in step 1 (in this example, C:\TESTTEMP).
Repeat steps 6 and 7 for the TMP user variable.
Click OK to save your changes.
Try to install the software again.
When the software is installed successfully, return to Environment Variables to restore the default TEMP and TMP variable values using the strings you pasted into the text file.

change the boot sequence to boot from the DVD.

 

a)      Open up your bios by pressing, in most cases, f2 at the start up screen(What key is pressed depends on the manufacturer, but in most cases it is F2).
