bossdweeb say:

"So can I use a SSD with 250GB and my clone my 189GB on it?"

Yes, not a problem. You can either clone to a single partition, or create a second partition of  50GB or less.

It depends on how you like to organise things and what you plan on doing with the rest of the SSD space if you do partition the SSD.

Keep in mind that any partition you create on the SSD needs to have 15+% free space for optimal SSD performance

(for Trim and extra cells to replace dead cells that occur over time).

- One option would be to create a small second partition for your programs.

I prefer keeping C:\ as small as possible with only Windows, my User data (except music. Video too if I had any, as a large collection

of either can use a lot of storage space), and a few programs such as Browsers, Anti-Virus, stuff that gets put there automatically

by software or Windows, and anything that is similarly closely  linked to Windows. 

I have a small 10GB partition on my 120GB SSD for my software, but all music and games go on the large HDD.

There is no benefit to having game performance on an SSD, excepting for a very small increase in loading time.

General software can benefit though.

My reason for preferring a small C:\ is I make a clone of C:\ on an external HDD prior to each Win10 version update as a precaution

should the  update go wrong, and having  a small C:\ makes the process faster.

I haven't installed games in the default C:\ Program folders since 2005.

With a clone of your Windows you never have to worry about reinstalling your games as long as you don't reinstall Windows clean

at some point ( my Win10 is an 'upgrade' from Win7 installed in Jan 2010).

I also keep backups of my games on external HDD's as precaution against sudden HDD failure. This also helps prevent having

to reinstall games in that event.

·         All PCs are peers; no PC has control over another PC.

·         Each PC has a set of user accounts. To log on to any PC in the workgroup, you must have an account on that PC.

·         There are typically no more than twenty PCs.

·         A workgroup isn't protected by a password.

·         All PCs must be on the same local network or subnet.
