To move Event Viewer log files to another location on the hard disk, follow these steps:

a.    Click Start, and then type regedit, and then click OK.

d.    In the right pane, double-click  File.

e.    Type the complete path to the new location (including the log file name) in the Value data box, and then click OK.

g.    Click Exit on the Registry menu.

h.    Restart the computer.

cmdrkeene say:

There's no way to restore a deleted sticky notes normally. But you can try this. It only works if "System Restore" is turned on.

  1. Open the folder C:\Users\skeene\AppData\Roaming\microsoft


  2. Make a backup copy of this whole folder (copy it [but don't move it] to your desktop for example).  Then just leave it there, we won't be touching that desktop copy.


  3. Find the "Sticky Notes" folder in the Microsoft folder, right-click and choose Restore Previous Versions


  4. If there are previous versions, try restoring them, then re-start your sticky notes program (by right-click the taskbar button for them and choosing "close all windows") to see if the previous notes file has the note you need.

In the future I recommend using OneNote.  It's free, works on everything in the world*, and your notes are always backed up into your private account online.  Plus you can see your notes anytime from your private login, or your phone, tablet, TV, whatever.

To return to the normal NUMLOCK setting (of not being active), just click NUMLOCK again and it should return to the normal situation - it's like an on/off switch.  If you mean to where NUMLOCK doesn't work like above and functions normally when it is on, then that's a different issue because I'm not sure what caused the problem in the first place (or how abnormal it is since I don't know the type of keyboard or computer).  Do you have another keyboard?  Try plugging that in and see if it has the same problem.  If so, continue on.  If not, then there is something wrong with the keyboard (it might just be drivers or software so check before you go out and buy a new one).

a. user say:

That message indicates some (hopefully minor) corruption in the NT File System (NTFS).

The file system can get corrupted when the system loses power (like powering off while Windows is running), using the reset button while Windows is running, pulling the plug while Windows is running, a power failure while Windows is running, other ungraceful shutdowns, aborted restarts, etc.

Your system will never boot in any mode until you first fix the file system and you are going to have to boot on something in order to do that.

Once booted on something else from a Command Prompt window you need to run a chkdsk with error correction (chkdsk  /r) on the afflicted system and that will usually clear things up.

From another Windows 7 system of the same architecture (32 or 64 bit) from Control Panel, Backup you could make a System Repair disc and boot the afflicted system on that...  all you need to do is get to a Command Prompt window.

You could download a Windows 7 ISO of the same Windows Edition (Home, Pro, etc.) and architecture (32 or 64-bit) as the afflicted system, create a bootable CD/DVD or USB thumb drive and repair the system from that.

Click this link to download WIndows 7 ISOs

Making your own Windows 7 SP1 installation media is usually the best choice since you will then have something useful for any future problems.

Somehow or another you need to figure out a way to run the chkdsk with error correct (chkdsk  /r) on the afflicted system.
