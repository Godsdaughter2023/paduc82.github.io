Task Scheduler of Windows 10 has some new features *not* - Microsoft Community

microtiger say:

Task Scheduler of Windows 10 has some new features *not*

I've upgraded from Windows 7 to Windows 10 and found the Task Scheduler of Windows 10 to have some tricky behaviours, if not even Bugs.

1)

Task scheduler will no longer find executables which are located in user path.
->two workarounds:
1) executable must be put somewhere in machine path
2) add user paths to machine path

->>user path and machine path can be viewed/changed here : Control panel. System.Advanced Settings.Environment Variables.

Iin the upper half of the windows you will find user environment, the lower half shows machine environment.

2)

Task Scheduler does not accept spaces for expression to start ("hello 1.txt" won't work). Instead Task Scheduler tries to edit "hello.txt".
->workaround:

Set explicitly the name of the application to start: "notepad.exe" with parameter "hello 1.txt"

3)

Tasks exported with Windows 7 compatibility cannot be imported in Windows 7.

There will be a very stupid error message and tasks without names when importing these tasks in Windows 7.
->workaround: export the taks with Windows Vista compatibility

->> You will find the compatibility setting (dropdown menu) in the lower left when editing a Task.
