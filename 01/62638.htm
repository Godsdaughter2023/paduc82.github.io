lz. say:

Hi Steven,

I suspect your "~$..." file to have "non-standard" file attribute(s). When files have System and/or Hidden attributes they're not displayed - by default - by a DIR command. The ATTRIB command allows listing files attributes:

In the above example file ~$test.txt has Archived + Hidden + Read-only attributes. Running a DIR ~*.* command would not show that file - a DIR ~*.* /A:H would though.

If you want to list all files that have Hidden or System attribute run a DIR /A:H /A:S command. If you want to list all files like ~$... with Hidden or System attribute run a DIR ~$*.* /A:H /A:S command:

Note that running a DIR ~$*.* /A:HS command will show ~$... files that have Hidden and System attributes.

For more information regarding the DIR command parameters just enter DIR /? at the command prompt.

Hope this helps

considerate_guy say:

Movie Maker will sometimes have problems with certain audio files, especially high-bitrate .mp3 files and sometimes .m4a from iTunes.

Given that you are able to save your movie without the added music is a further clue that it is the music that is the problem. So I would follow skier-hughes suggestion that you convert the music to .wav or .wma format. Then try the steps (File > Save movie) again and it should work this time.

One other tip:

  • In the older versions of Movie Maker (through 2011), the default format for saving your movie was .wmv.
  • In Movie Maker 2012, the default has been changed to .mp4. If you truly need it in .wmv instead, then be sure to switch the output file type when saving your movie. Instead of "H.264/MPEG-4 (.mp4)" choose "Windows Media Video (.wmv)" in the lower right of the file name dialog.
