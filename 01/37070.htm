frederik long say:

Within the Command Prompt I've noticed that it is hellbent on defaulting to a mythical X: drive, I've added the basic C:\windows\system32 path via the Set Path command stuff. But the only thing that did was return errors when I attempted the Control Panel and devmgmt.msc commands whereas before the prompt would claim such commands could not be found.

If you saw drive X: then you were in Windows Repair Mode (WRE). This is a cut-down mode that gives you access to basic repair functions and not much else. You will not get the Control Panel and you will not get any snap-ins either. Drive X: is actually a virtual drive that disappears as soon as you reboot.

Assuming that your Windows system drive is C: while in WRE (which is often not the case!), you can restore an old version of the registry as follows. It might fix your problem.

7. Type these commands exactly:
    cd  /d  C:\windows\System32\config
    xcopy  *.*  C:\MyRegBack\
    cd  RegBack
    dir
8. Examine the dates of the Software, System and SAM files. Were the files created before or after your problem became apparent?
9. If they were made before the problem became apparent then you can type these commands:
    copy /y software  ..
    copy /y System  ..
    copy /y Sam ..
    (the two dots are part of each command)
10. Reboot normally. If things do not work out then you can backtrack by copying the original registry files from C:\MyRegBack to C:\Windows\System32\config.

And yes, Windows does have a habit of deleting restore points without warning. The wise man protects himself by creating an image of the system partition once every so often.
