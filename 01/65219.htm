SMB login failures due to Windows using invalid MicrosoftAccount - Microsoft Community

ajs_52 say:

SMB login failures due to Windows using invalid MicrosoftAccount username

On Window 8.1, and presumably Windows 8, using a Microsoft Account to login to Windows the default username when you make a network connection using SMB is in the form ‘MicrosoftAccount\<live_id_username>’ instead of ‘<computer_name>\<username>’ as previous versions of Windows.  Using the Microsoft Account form of username is likely to cause a login failure on any server.  

If the old format username and password is stored in the credential manager for the server concerned then this will be used in preference to the default.

Now my problem.

If you start a process with Windows Task Scheduler to run when the user is logged on with elevated privileges this process cannot access the credential manager.  This can be reproduced by creating a task in the task schedule to run the follow command with the ‘Run only when user logged on’ and the ‘Run with highest privileges’ options selected and triggered to run either at login or on a schedule.

CMD /C "cmdkey /list && pause"

This will not list any of the stored credentials.  I have reproduced this on Windows 8.1 and Windows 7, however this does not cause a issue on Windows 7, or Windows 8.1 when using a local account, as these configurations do not need to store and retrieve the valid username and password from the credential manager for the server.  However on Windows 8.1 using a Microsoft Account because it cannot get the valid username and password from the credential manager the invalid default Microsoft Account format username is used instead which will fail to login to the server.

If the test task is run with normal privileges it will successfully list the credentials.

If the user interactively starts an application that runs with elevated privileges, such as starting a console window as admin, then any subsequent run of the task that would normal fail will now successfully list the credentials.   I am guessing that this action is causing the credentials to be cached in memory which is not happening when the task scheduler creates the privileged task.

The task schedule configuration describe above is how the backup software I use starts so that it can perform background backups.  Because of this issue it fails to access password protected network shares.

Networking Windows 10 to Windows 7 via Cat6 Ethernet cable to router. - Microsoft Community

destinythielemier say:

Networking Windows 10 to Windows 7 via Cat6 Ethernet cable to router.

Okay, I have just recently purchased a dell inspiron with windows 10, my other work computer is a toshiba laptop (not sure if that matters),running windows 7. I need help because I HAVE to have the two networked through an Ethernet cable (cat6 crossover to be exact). and I have them both wired to my router. but for some reason it keeps kicking me off the internet and saying that the default gateway is unavailable?
don cuthbert say:

With 2 network adapters in one computer, you can share out the internet access that is connected to one adapter with other computers that are only networked to it via the second adapter.  This used to be done to share out a single dial-up modem or to share an Internet connection when only one computer was allowed to connect to the ISP through the modem.

I can't see any advantage to this sort of setup these days, as I expect almost every ISP expects multiple computers to connect from each residence.

So restoring what used to work for you might seem simplest, but since it is not working out that way, you might consider a change.

You mentioned that both computers are wired to the same router.  That should be done with a normal Ethernet cable directly between each PC and the router.

I presume that the router is connected to Internet access, but if your Internet has instead been plugged into the laptop instead of the router, move the Internet cable to the router also.  Possibly this might be a place to use the crossover cable.  If a crossover cable is required, the Ethernet port connected to the Internet will not light up when a normal cable is connected (there may be a separate LED somewhere that indicates connection).

Perhaps your router is included as part of your modem from your ISP and it just has multiple ports?  Then just plug each PC to an Ethernet port on it.

It may be that some or all of your computers were given static IP addresses, which might be part of your problems now.  It will be easiest to switch them all to Auto-configure.  In Windows 10 hold the Windows key and type the letter X (or right-click the Start button).  Then type W or choose Network Connections from the menu that pops up.  Find your Ethernet connector in that Window.  Right-click it to choose Properties and then find Internet Protocol and choose Properties to ensure it is set to Auto.
