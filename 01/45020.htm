steve winograd say:

When more than one network connection is available, Windows uses the one with the lowest metric value. By default, it automatically assigns a metric value based on the network connection's rated speed. For details, please see An explanation of the Automatic Metric feature for Internet Protocol routes .

To force Windows to use a specific network connection, assign a metric value to each one, giving the lowest value to the desired connection:

  1. Press WindowsKey+R , type ncpa.cpl  in the box, and click OK .
  2. Right click the desired connection.
  3. Click Properties > Internet Protocol Version 4 (TCP/IPv4) .
  4. Click Properties > Advanced .
  5. Un-check Automatic metric .
  6. Enter a number between 1 and 9999 for the Interface metric .

a. user say:

This is not new with Windows 10. Windows has always worked this way.

When a removable device is removed, its drive letter is thrown back into the pool of available drive letters; any other removable drive can use it.

Often that doesn't matter. For example: I have two USB drives that are assigned E and F, respectively. I can unplug those for days and then plug them in again and they will still be E and F, but that's because I don't plug any other drives into my USB ports.

Windows doesn't have the native ability to reserve drive letters, but there are a few things that you can do:

1- There may be third-party software that will do this (make sure it will work with Windows 10.)

2- Assign a high drive letter - like Z - to your device; windows usually assigns drive letters in alphabetical order

3- Use a mount point (also known as a junction point) instead of a drive letter; but now we're getting unnecessarily complicated.
