Error 0xc000007b when starting an application (Camera frame grabber)

A camera frame-grabber application fails to start and displays an "Application Error" pop-up window.  The error code is 0xc000007b.

The application has been working for two years on several hundred different systems.  On this one particular system, the application will not start.

The camera vendor provides an installer with a "repair" option.  The installer "repair" action fails when it cannot update or replace a codec file that is proprietary to the camera.  The codec file exists on the problem system and appears to be identical to copies on other working systems.

Any suggestions on where to look next for the cause of this problem?

michaelpettigrew say:

Installing a 3rd-party mixed-mode application package (32-bit and 64-bit executables) appears to have incorrectly updated C++ runtime libraries.  All 32-bit versions of the camera application failed after installing the 3rd-party application (which oddly enough never references the camera in any way).

Immediate correction was to revert to earlier system restore point.

Vendor produced new installation packages which separate their 32-bit and 64-bit applications, and do not attempt to upgrade C++ run-time libraries.

No further problems seen.
