a. user say:

The hiberfil.sys will be created every time you boot...  if you have Hibernate enabled in your Control Panel, Power Options and the file is "open" thinking that you might actually choose to hibernate at some point.  When you hibernate XP makes a note of it, and will write a sometimes large hiberfil.sys in the c:\ folder so that when XP fires up again and returns from hibernation, XP will read the hiberfil.sys file and return you to the situation you were in when you hibernated.

I use to be a chronic hibernator since it usually takes less time to return from hibernation than a cold boot.  Sometimes it can actually be slower than a cold boot though depending on how much RAM your system has.  The more RAM you have, the larger your hiberfil.sys will be, so sometimes it is faster to get running again with just a cold boot if you don't hibernate (26.03 seconds for me).

If hiberfil.sys is giving you a hard time (you would not want to copy it anyway) disable Hibernate in your Power Options and the file will immediately be deleted since it would no longer be needed.  That would solve that, but your if you are trying to copy the entire C drive, you will have other complaining files after that.

It is certainly possible to take an entire copy of the C drive if you want, but you are going to have to boot on something beside your hard disk if you want to do that so the XP files are not being used - then you can copy everything and anything.  You can get some free disk imaging software that will help you too.

It depends on what the problem is you are trying to resolve and what you wand to end up with when you are done (so far, unspecified).

I would work on figuring our why your XP is sticking and fix that instead.  XP should not be sticking.

mala_s say:

Hello Lee_2008,

Thank you for writing to Microsoft Community Forums.

I understand the inconvenience caused while trying to access the touch keyboard since Windows updates, kindly follow the below methods and see if it helps.

Method 1: Check with the Services.

  • Press Win + R keys to open Run command > type services.msc in it and click on OK to open Services window.

  • Narrow down to Touch Keyboard and Handwriting Panel Service > right click on it and select Properties.

  • In General tab, select the Startup type as Automatic or Automatic (Delayed Start).

  • Go to Dependencies tab in Properties, and check for the services that are below This service depends on the following system components and make sure that those services are running.

  • Also right click on the service to select Start if the service is stopped.

  • Restart your computer and see if it works.

Method 2: Update the touch feature driver.

  • Right click on the Start menu and select Device Manager.

  • Expand Human Interface Devices, right click on the Touch screen driver and select Uninstall Driver Software.

  • Reboot the PC > Once after the reboot, your PC will automatically pick the drivers and see if that helps.

If that doesn’t help, you may update the touch screen driver from manufacturer’s website.

Method 3: Registry Edit for touch keyboard.

  • Press Win + R keys and type regedit in the Run dialog box and click on Enter.

  • Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\Scaling.

  • On the right pane of the Scaling, look for the MonitorSize named registry String.

  • The incorrect Value data for the MonitorSize registry string is the root cause behind this issue.

  • Double click on the MonitorSize and edit the Value data to 22.5 and click on OK.  

Registry Disclaimer: Serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. To backup and restore registry key, refer to this

Hope it helps.
