Unable to delete files from an FTP site using Windows 10 File Explorer - Microsoft Community

yotvb say:

Unable to delete files from an FTP site using Windows 10 File Explorer

Since the previous thread is locked, I'm re-posting this question to see if anyone has a real fix for this problem, or if Microsoft is even aware of it. 

The original post is found here: 

https://answers.microsoft.com/en-us/windows/forum/windows_10-files/unable-to-delete-files-from-an-ftp-site-using/04f664d0-7028-4f68-a185-82cdb2fa2cbe?auth=1

The issue is this: after some time of using an FTP server, Windows stops deleting files when you tell it to. You can upload (copy-paste) to it, but you can't delete. I know this issue is present on the spring Creator's update (1703), as that's where I have users experiencing this problem. It's present on two different FTP servers, networks, and two different computers (those are just the ones who have told me). Based on the linked thread and the Feedback app, it appears to be an issue on other builds of Windows as well, and has been for several years.

The only workaround I've found is to create a new folder on the FTP site, in the same sub-folder where you're trying to delete files. After creating the folder, go ahead and delete it. Then try to delete nearby files. For some reason, simply the act of creating a new folder re-establishes whatever "stale" credentials were in place.

Move your Mail folder (a one time thing)

Windows Live Mail (WLM) sets itself up in a very inconvenient place.  Hard to find.  There is no reason not to move it to a more accessible location on your hard drive.  This is a one time thing.  Once you do it, you will never have to do it again.  Now, make backing up that folder a regular part of your regular backup procedure, in addition to the above steps

To move it:

  1. With WLM open, click on the little blue rectangle with the arrow in it on the top left of the WLM window
  2. Click on Options then Mail, the Advanced tab, Maintenance, Store folder
  3. Click the Change button.  A navigation list of your hard drive pops up.
  4. Now navigate to the C: line, click once on it.
  5. Click Make new folder
  6. Type your name followed by the word mail, click OK, Click OK
  7. Close all the open windows.
  8. Close WLM
  9. When you next open WLM, it will take longer because that is when it actually moves your mail folder

The reason you should do this is that in the future you will be following the back up procedure above, BUT also you will be backing up that Mail folder to your external hard drive as well.  The reason to do this is that if ever your system needs to be re-built, the re-builder can copy that Mail folder to the right place in your “new” system and all of your mail, contacts, calendars and whatever the way you have WLM set up, will be in effect in the new computer.

Once Windows is re-built and all is working, download WLM 2011.

The 2011 version works better than the 2012 one.  The main advantage of the 2011 version is that Groups do not work in 2012.  If you do not use groups, download WLM 2012 from the Microsoft site

  1. Start your newly installed WLM and ignore all messages. 
  2. In WLM, click the tiny arrow icon on the nearly top left.  Choose Options, Mail
  3. Click the Advanced tab, now Maintenance, Store folder
  4. Use your mouse to select the entire content (the location where your mail is installed in the new installation) shown in that window, Ctrl+C to copy it.
  5. Close WLM
  6. Navigate to the location you just copied by pasting (Ctrl+V) it into the text box above the start button
  7. Delete all the contents of this folder, but not the folder, which will now be empty.
  8. Copy the content (not the folder) of your backed-up WLM folder, and paste it into the folder you just deleted the contents of.
  9. Start WLM, and you will have everything back just as you had it before.  No importing, no having to go and reset everything back the way you like it.  You won’t even have to set up the accounts again.  This even has your passwords.
  10. Type indexing options into the text box above the start globe
  11. Click Modify and add your new Windows Live Mail folder.

You should now use the “move” procedure above on your new or newly re-built PC.

¡firedog say:

Wendy, I'm sorry not to have seen your question earlier. There is a dedicated forum at this site for questions about Windows Live Mail, and had you posted there, I would have reacted earlier. You've been given a lot of well-meaning advice, but we should first have asked some questions to find out what has gone wrong.

Messages don't usually just 'disappear'. Each one is stored as a separate file, and I don't know of any fault that would cause 'all' of them to be deleted since you say they were all classified in different folders. You don't say whether the folders have also disappeared, whether they were under an account heading or in Storage folders, whether more than one account is involved and so on.

The folders you see in Windows Live Mail are virtual folders bearing only a superficial resemblance to what you may see in Windows Explorer. There is a message database index that keeps track of which messages are in which folder, and your experience would indicate only that something's gone wrong with this index. If this is the case, I don't know what damage you may have done by following some of the advice you've been given about moving the store folder and reinstalling the program, neither of which would help with a database index corruption. There are several reasons why an index corruption might occur, and I would certainly not put hard drive failure high up the list of possibilities.

I don't know which bits of this advice you've so far followed. I can only suggest that you first check that your messages haven't just been moved.
 

  • Make sure that Storage folders is selected (highlighted) on the  
  • When viewing your message list, press Ctrl-Y to reveal the entire folder tree. Look for your missing messages, possibly several layers deep under Recovered items or inside another folder. 
 

If one or more folders called Recovered items exist, they won't be empty unless you emptied them yourself. You may have to drill down through their contents to find messages:

You'll see that the first level is a folder whose name is the timestamp of the event that corrupted the index. You may be able to spot the event on 26 or 27 September that made your messages 'disappear,' noting that the timestamps are American style, so the folders don't sort chronologically - you're looking for one whose name starts with 09-26- or 09-27- .

If you do find missing messages in the Recovered items folder(s), you can drag folders around within Storage folders. If you want to move messages to folders under an account above Storage folders, you can drag and drop them into their target folder. You can't drag a folder out of or into Storage folders.

That may be enough to recover your messages. Otherwise, you have in Windows 7 the Windows Explorer feature called Previous versions. Again, not knowing what actions you have already taken, it's impossible to know whether this is an option for you. To check, 

     
  1. Press Windows key + R to open a Run box. Type %localappdata% and press Enter. You should now see your user account's AppData\Local directory.

  2. In the right-hand pane, right-click on the folder called Microsoft and select Restore previous versions. Do you have a version dated shortly before this event? If so, we should be able to restore most of your messages. I'll give instructions after I've seen your reply if we have to go this route.
