After ChkDsk is done go back to the admin prompt and run following commands, one at a time to Repair The Installed Windows Image

this command checks whether the image has been flagged as corrupted by a failed process and whether the corruption can be repaired:

DISM /Online /Cleanup-Image /CheckHealth


this command scans the installed windows for component store corruption:

DISM /Online /Cleanup-Image /ScanHealth


this command performs Window repair operations automatically:

DISM /Online /Cleanup-Image /RestoreHealth


DISM creates a log file (%windir%/Logs/CBS/CBS.log) that captures any issues that the tool found or fixed. %windir% is the folder in which Windows is installed. For example, the %windir% folder is C:\Windows.

sirtra say:

So after a bit of trawling the forums with the mindset that my headset/mic was okay but something else was the cause of my non-working-mic or more specifically the inability to speak to people using it, given it used to work many moons ago and has been kept in mint condition, ive discovered a few things:

1) A non-intuitive but easy way to test the mic - open up 'party' and start a new party with jut yourself. The little circle should illuminate on/off according to whether there is speech/noise heard from the mic.

2) If the little circle next to your name is permanently illuminated, like mine was, chances are its not due to there constantly being speech/noise but rather a bug that many people have encountered. See forums.xbox.com/.../1904207.aspx

3) With some wiggling and/or putting pressure on the headset connector at the bottom of my controller whilst plugged in and turned on (specifically by pressing hard on the frame as if you're pressing the volume down button but not actually pressing the button to the point where there was a mm or so gap between connector and controller) this somehow managed to squash the bug as the circle suddenly became non-illuminated and the headset started working as expected, illuminates with noise and un-illuminates with silence.

Note: I say squashed the bug because i dont feel the problem was physical such as a dry solder joint, dicky cable or connector which only works in a certain position and similar which would be most peoples initial thought.

Prior to the fix I tried several unplug/replugs, turning controller off/on, moving cables about, covering the mic holes etc all made no difference - the circle would remain illuminated the moment i plugged the headset into the controller and irrespective of whether mute was on/off.

I have no idea how the wiggling/pressure helped other than it triggering some sort of re-detection or firmware reload or protection mechanism to kick on/off. Either way i dont care what it actually did but rather that it did somehow manage to reset/unblock whatever was causing my problem for wees now and the headset has been working perfectly since (further suggesting it wasn't a physical issue).

As tim the toolman taylor would sort of say, if its broke just push it harder / use more power ... it might just fix an electronical problem with a Microsoft product :P
