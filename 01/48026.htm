    Hi. Because you installed Japanese first, that does affect certain obscure aspects of font configuration that could affect some apps and older parts of the Windows user interface, and just might account for what you're seeing.

    Since you're evidently a Windows developer, then you might be familiar with a font setting that apps can use, "MS Shell Dlg". This is not a real font but rather is a font alias, and there's a registry entry that maps it into a real font. When you install any language other than Japanese, it gets mapped into the Microsoft Sans Serif font. But if you install using a Japanese install image, it gets mapped into MS UI Gothic.

    Assuming that your issue is related to this, then adding and selecting an English language pack won't change the behavior. However, I believe that applying language settings to system accounts will. If you don't need the welcome screen or system accounts to have Japanese settings, then you can try this:

    • Go to the legacy Control Panel. (Right-click on the Start button, then click on Control Panel.)
    • If your Control Panel is in the Category view mode, then click the link for "Change date, time, or number formats" under the "Clock, Language, and Region" heading. Or, if the view is either Large icons or Small icons, then click the "Region" icon.
    • In the Region dialog, click on the Administrative tab.
    • In the "Welcome screen and new user accounts" section, click the "Copy settings..." button.
    At this point, you should see this dialog (though perhaps with different settings shown):

    On your system, I expect the top section has English as the display language, but that the lower two sections show Japanese as the display language. If so...

    • Check the item highlighted. (You can also check "New user accounts" if you want any new local user accounts that might be added in the future to have English defaults.)
    • Click OK to copy the language settings from the current user account to the system account.

    Copying non-Japanese language settings to the system account should change the mapping for MS Shell Dlg to Microsoft Sans Serif, and hopefully that will fix the symptoms you were seeing.
