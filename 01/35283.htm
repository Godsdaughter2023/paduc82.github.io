The Device Manager Code 39 gives an error description of Windows cannot load the device driver for this hardware. The driver may be corrupted or missing.

Follow the below suggested methods and check if it helps to resolve the issue.

Method 1

Follow these steps to run the Windows in built troubleshooter to check if it helps to identify any issues or possibly fix it 

a)      Write Troubleshooting in the search box and hit enter.

b)      Click on Troubleshooting tab.

c)      Click on view all option on the upper left corner.

d)      Select the Hardware and Device option from the list.

e)      Click Next to run the Hardware and Device Troubleshooter.

Method 2

Next we need to start again, by searching for the most up to date Windows drivers through Device Manager settings.

Check for Windows update to look for latest drivers for devices installed in your computer.

Refer to the below Microsoft Help Article on Install and Update drivers in Windows 10, written by one of our MVPs, Andre da Costa.

http://answers.microsoft.com/en-us/insider/wiki/insider_wintp-insider_devices/how-to-install-and-update-drivers-in-windows-10/a97bbbd1-9973-4d66-9a5b-291300006293

  1. Right-click on the setup file and click on "Properties".

  2. Click on the ‘compatibility’ tab and check the box "Run this program in compatibility mode for” and select previous Operating System from the drop down.

why does system have bad shutdown will in hybernation status - Microsoft Community

leebrown43 say:

why does system have bad shutdown will in hybernation status

Computer shuts down incorrectly will not in use

a. user say:

Hi, LeeBrown43,

 

"The main issue with excessive hibernation is that the PC grows sluggish and non-responsive after a while."

The best thing to do at that point is to shut it off and allow it some “reflection time” to cool off the processor and save you energy.

http://www.brighthub.com/computing/hardware/articles/36359.aspx

 

1. Run CMD.EXE as administrator
2. Run the following command: bcdedit -enum all
Look for "Resume from Hibernate" in the output from the command above(example below):
Resume from Hibernate
---------------------
identifier {3d8d3081-33ac-11dc-9a41-806e6f6e6963}
device partition=C:
path \Windows\system32\winresume.exe
description

inherit {resumeloadersettings}
filedevice partition=C:
filepath \hiberfil.sys
pae Yes
debugoptionenabled No

3. Once you have found it, copy the value for identifier (in this example - {3d8d3081-33ac-11dc-9a41-806e6f6e6963})
4. Run the following command: bcdedit /deletevalue {3d8d3081-33ac-11dc-9a41-806e6f6e6963} inherit
5. Test hibernation

http://answers.microsoft.com/en-us/windows/forum/windows_7-hardware/unexpected-shutdown-after-hibernating-or-standby/d9fde351-abbf-46f0-a150-d6f8bfcb065d
