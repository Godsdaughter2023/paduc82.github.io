marvin ma msft say:

Hi Matt,

Skype for Business is a completely different product from regular Skype. The former of Skype for Business is Lync, which was renamed Skype for Business in 2015. For your reference, see Why do I see Skype for Business when I'm using Lync?. So it's normal that the same devices may have different behaviors between regular Skype and Skype for Business, because Skype for Business has its own certificated devices.

In this scenario, please ensure you use the latest version of Skype for Business. For your reference, see Office update history. Besides, you can try some common troubleshooting steps in Skype for Business client to verify if it helps. 

1. Sign out the user account in Skype for Business.

2. Delete the sign-in info and completely exit. If the progress is still running in the Task Manager, please shut it down.

3. Navigate to the following path in File Explorer: %userprofile%\appdata\local\Microsoft\Office\16.0\Lync.

4. Delete all the folders in Lync. 

5. Launch Skype for Business and check if the issue still exists.

And if the issue persists, the issue could be related with your device. You can provide the detailed information of your bluetooth device here so that we can help you check if it's certificated. Or, by yourself, you can check if your device is listed in the article USB audio and video devices or if it's provided by Microsoft partners, who participate in the Skype for Business Certification Program to test their products.

Regards,

Marvin

a. user say:

Really?  If the updates are the culprit then millions of systems all over the world would be crashing - and they're not.

If a system crashes after an update the cause is usually a system that has been set to install updates automatically (the riskiest possible setting) which allows some hardware driver updates to install  or some out of date third party software or driver that begins to malfunction after MS has issued a patch that plugs a security hole making the third party software incompatible (it still has the security hole).

The proper way to diagnose a BSoD is to collect fundamental system information along with crash dump files and use the Windows debugging tools (not something like WhoCrashed, BlueScreenView, OCR, etc.) to analyze things.

STOP errors of 116 and 117 are very common so here are some thoughts to get you started:

The problem (errors 0x00000116 and 0x00000117) is most likely a video driver, a thermal (heat) issue or both.

Video intensive operations can contribute to thermal related BSODs. 

Can you reproduce the problem anytime you want...  say by playing games, watching videos, etc.? 

Do this so we can see what you have:


What is your system make and model or is it something assembled from separately purchased components?


If the system is a Dell what is the Service Tag Number?

If the system is an HP what is the serial number?

If the system is an Toshiba what is the serial number?

If the system is an Lenovo what is the serial number?


Describe your antivirus/antimalware environment.

Click the Start button/orb in the lower-left corner of your screen. Or, press the Windows logo key on your keyboard.

In the box enter:

msinfo32

Select/click msinfo32.exe and the System Information dialog box will open.

Click on System Summary, click Edit, Select All (Ctrl-A), Copy (Ctrl-C) and then paste (Ctrl-V) the information back here in your next reply.

For video driver information, expand the Components, click Display, click Edit, Select All (Ctrl-A), Copy (Ctrl-C) and then paste (Ctrl-V) the information back here.

There will be some personal information (like System Name and User Name), and whatever appears to be private information to you, just delete it from the pasted information.

Here is a link from contributor Gerry with some instructions on how to upload a few of your most recent crash dump files for analysis:

http://www.gerryscomputertips.co.uk/MicrosoftCommunity1.htm

If you download the free version of Speccy you can see what your temperatures are and also monitor them while you are performing video intensive things that might cause a BSOD:

https://www.piriform.com/speccy

You can uninstall Speccy later if you don't like it.

When Speccy is running temperatures in green are good and temperatures in red would be bad (in this example everything is green=good and red=bad):

Remember:  Green=Good  Orange=A Concern  Red=Bad

If you minimize Speccy to the system tray the average temperature will still be displayed next to the system clock so you can keep an eye on it and if it gets into the red, you have a thermal problem:

See if you can reproduce the BSOD while Speccy is running and showing the temperature of things.

Keep an eye on the temperature while you are doing whatever you do that causes the BSOD.

Allen, did you flag that first response as the answer???

What you want to do is set a Music folder to your preferred view settings & then, from the View tab, select Options, found on the far right of the View Ribbon.

In the Folder Options dialog, select the View tab and click Apply to Folders.

This will give you your preferred view settings on any folder that has not already had its view saved --- which means any folder you've already viewed.  If you want to start with a 'clean slate', You can delete all saved views for all folders by deleting the following Registry key:

HKCU\Local Settings\Software\Microsoft\Windows\Shell\Bags

Best to export the key first so you can restore it if you don't like the results.  This will delete all saved views, but using the Apply to Folders setting on top-level folders of each type (Documents, Music, Pictures, etc.) before you start opening subfolders will set up consistent views fairly quickly.
