martin gerhold (original) say:

When I first experienced this, it was a folder with many photos in it, possibly under Windows 7.  I don't know if you can stop Windows from reading all that metadata that Locutus mentioned, but you may be able to stop Explorer from re-reading it when you change the sort order.  If you right-click the empty space to the right of the column labels, you will be able to add a new column.  I suggest you try 'date created' or 'date modified', as these seem to be read from the MFT data, rather than the file contents.  See if sorting on these is snappier, and if it is remove the existing 'date' column (I can't be sure what date you are getting by default).

Windows does remember the date info for a while, so don't be deceived if the generic 'date' seems less sluggish after some exercise (!).  However, its memory buffer is apparently limited: try viewing some other large folders, then come back to the one with the video files, and see how the sort speeds of different date columns compare.

I set my default view for all folders to be details, with 'date modified', and have no issues on very old hardware (but don't have many video files).

HTH, Martin

PS Also remove any surplus columns that might necessitate reading the metadata

cmdrkeene say:

No commands necessary, as long as you're using a local account on the computer, it will work just like all previous versions of Windows, where you can just remove the password and switch seamlessly.

If you've connected your account to an online account, then you can't remove the password.  This is sort of obvious, because it's impossible to have any online account (yahoo/google/Microsoft/live/outlook/msn/hotmail) without having a password for it.

To convert your account to a local PC account, open the PC Settings app, go to Accounts, and choose "disconnect".
