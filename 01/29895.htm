peer_sam say:

Hi Asad,

 

Thank you for writing to Microsoft Community Forums.

 

Miracast is a peer-to-peer wireless screen casting standard formed via Wi-Fi direct connection like Bluetooth. It is a protocol that transmits audio and video between the supported devices via Wi-Fi. It relies on Wi-Fi network available using 2.4 GHz and 5 GHz bands. The shared information is sent by the device via Wi-Fi through a Wi-Fi Direct connection to a receiver connected to the display device. The receiver then decodes the video signal and passes it to the TV display (or other display device). Miracast supports WPA2/WPA3/WPA2 or WPA3 WPA2 or WPA3WPA2-PSK encryption, so all you share is safe.

For additional information, please refer to the article: Miracast in Enterprise Environments

 

Like Bluetooth and Wi-Fi, once your project the PC screen to a Miracast enabled device, the connection should be established automatically for all future sessions.

You may find this article interesting.

Miracast over Infrastructure Connection Establishment Protocol

 

When you say that you’re planning to enable Miracast in your environment, I assume that it is a corporate network. We do have a dedicated support for systems which are part of a domain or corporate environment. The thing here is that you are mixing up 2 different installation processes with the wrong responsible parties. Microsoft and Windows don't have factory images, that is something that your computer manufacturer put on the harddrive. Microsoft has no involvment with that process so all the guy from MS could tell you was how to start up Windows own system image recovery. Windows gives you the option to create a System Image when you first install it, which you then have to create and store in some safe place.  A Windows system image is nothing like the factory recovery image on D: drive and Windows itself cannot do anything with the recovery partion on your D: drive.

Do you happen to have an HP computer? HP puts their System Recovery software on a D: drive partition. Their recovery systemONLY works with the original HP OEM operating system and software still installed on the computer or if theC: partition of the harddrive is completely empty, or even unallocated. It's an anti-piracy feature.  That's probably why the system recovery failed when you tried to do it. When you get the computer with its original operating system you can access the recovery partition 2 ways, either from Windows using a utility in one of the Program Files folders or when you boot the computer and press the required F-key (which is not F8).

The D: drive and software are dedicated for use by the manufacturer's System Recovery software and changing the operating system from Vista pretty much disables it. If you used a normal retail Windows 7 upgrade disc then the D: partition only might work to put Vista back on the computer if you are still able to boot from the D: partition.
