rajesh govind say:

 

Hi Gary,

 

The “Large” or “Wide” options while resizing an app tile is not available for all the apps tiles on the Start Screen. They are available only with certain apps. The App tiles shown on the Start screen can be either wide or square. Each app has a default tile. This is the tile that the user sees when they first install your app. The default tile logo image represents your app logo or other branding so users can clearly identify the app.

 

When the App developer defines the default tile logo image in the app manifest and Windows displays it until the tile is updated by receiving its first notification. The tile can revert back to the default image when it has no notifications to display; for example, when the current tile notification expires or the user turns off live tile updates for the app.

 

  • If only a square logo image is provided in an app's manifest file (package.appxmanifest), then that app's tile will always be square and you cannot resize it to large or wide. Here, you can only use square templates in tile notifications. This is sufficient for some scenarios.

 

  • If both a square and wide logo images are provided in the manifest, the app's tile will default to a wide tile when it is installed. Tile notifications sent to your app can use both wide templates and square templates.

 

Thus, it's up to the app developer to implement the optional large tile mode for the apps.

 

For more information refer to the section: “Default tile and tile size” from this MSDN article:

 

Creating a great tile experience (part 1)
