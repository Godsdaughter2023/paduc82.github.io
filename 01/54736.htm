Hello, AsifKhan1

A few thoughts to consider here ...

First, if you use Skype and have a Skype account, fraud reports can be filed directly with Skype Customer Service. However, note that once you flag your Skype account or payment method as having been used fraudulently, Skype will likely suspend or block your account to prevent further fraud, leaving your account disabled for use, as a result of the charge-back process to recover funds.

Second, if you do not use Skype, and do not use the payment method for anything related to Microsoft, then there is no need to contact Skype directly. Simply contact your payment method (e.g. credit card company, banking institution) to report the unauthorized charges.

Skype does not host a call center, so there is no phone number to share with you. Neither does Skype use a general or generic "contact us" email address.

Stop: 0x0000007B (0xF7C6E524, 0xC0000034, 0x00000000, 0x00000000

malkeleah say:

Most commonly caused by bad RAM so start by testing your memory. I like Memtest86+ fromwww.memtest.org . Obviously, you have to get the program from a working machine. You will download the .iso to make a bootable cd. You want the pre-compiled bootable ISO (.zip). Unzip the file you download by double-clicking on it and drag the contents out. You will now have a file called memtest86+-4/00.iso (the version number may be different). You can delete the .zip file now. Put in a CD-R disc and start a third-party burning program such as Roxio, Nero, or the freeImgBurn (unless you have Windows 7 which can burn .isos natively). You will need to burn the file as an image, not as data. Refer to your burning program's Help if you don't know how to do this.

Leave the CD-R in your optical drive and restart your computer. When you restart the computer you will see messages:

1. Possibly a message that says something like "Press F12 for temporary boot menu". If you have this message, press that function key. Use your arrow key to select the CD/DVD drive and the computer will boot from the Memtest86+ CD you made.

2. If you don't see a message about a boot menu you will need to go into the BIOS to change the boot order. This message will say something like "Press F2 to enter Setup". Press that function key and you will enter the BIOS. Find the section about boot and change the boot order to CD/DVD drive first, hard drive second. Save your changes and exit Setup. The computer will boot from the Memtest86+ CD you made.

The test will run immediately. You can remove the CD while the test is running.You will need third-party burning software to do this  Burn as an image, not as data. Then boot with the media you made (you may need to change the boot priority in your BIOS so that the optical drive is the first boot device). The test will run immediately. Let the test run for an hour or two - unless errors are seen immediately. If you get any errors, replace the RAM.

frederik long say: Have a look at these forums. The vast majority, perhaps 99%, deals with issues such as
  • Windows fails to boot
  • Some Windows functions no longer work properly
  • File corruption
  • Account problems
  • Virus infections
  • Hacker attacks
  • File system corruption
  • File loss
  • Backup/restore problems
A RAID array gives you no protection at all against any of those. RAID arrays are great for banks whose computers must survive a disk crash without interruption. Do you run a bank? You would be far better served with a scheme like the following:
  • Install a disk in an external USB case.
  • Back up your data files every week to this disk.
  • Use a flash disk for your data backups on alternate weeks.
  • Create an image of your Windows installation. Store it on the external disk.
  • Update the image twice a year and also before major changes.
  • Keep two or three versions of the image.
  • Burn a Windows Repair CD.
  • Boot the machine with this Repair CD to check whether it will detect the image you created.
  • Keep your backup media well away from your PC for most of the time.
  • Have two fully tested admin accounts and record their details on paper.
If you observe the above points then you are very unlikely to have any serious problems with your machine.
