It would have been better if you had provided the make and model of your computer as well as the version of Windows you have.

Because you asked your question in the XP section of Microsoft Community, I assume that you are still using Windows XP, despite the fact that it hasn't been supported since April 2014 and is increasingly vulnerable to online attacks. 

Have you at least installed Service Pack 3 and all critical updates up to April 8, 2014?  If you haven't that may well be a cause of your problem with installing .NET Framework 3.5.

Most printer manufacturers bundle a lot of mostly unnecessary software with their printers.  In your case, apparently some of this extra software requires .NET Framework 3.5.

You have a couple of options.

If you don't want/need the extra software, you often can download just the printer driver from the manufacturer's web site.  This usually is a much smaller download than the "full feature" software package and will not require .NET Framework.

If you do need the extra software, you'll have to determine why .NET Framework 3.5 is having a problem.

Start by going to Control Panel > Add or Remove Programs and see what "Microsoft .Net Framework xxx" versions you have.

If you do not have .NET Framework 3.5 or 3.5 sp 1 (and DO have XP sp 3 and subsequent updates) download the installer here --> https://www.microsoft.com/en-us/download/details.aspx?id=22

Temporarily turn off your antivirus program and run the downloaded installer.  This installer will access Windows Update and install the necessary components.  If for some reason this fails, you can try the stand-alone installer, but it's a rather large download -->  https://www.microsoft.com/en-us/download/details.aspx?id=25150

Don't forget to turn your a/v back on.

The stand-alone installer instructions say that after installing .Net Framework 3.5 sp1,  "you should immediately install the update KB959209 to address a set of known application compatibility issues."  I don't know if the bootstrap installer will automatically get that for you, so you should check the list of installed updates (or just try to install the patch).

If the .Net Framework installation goes OK, try installing your printer software again.  If it doesn't please describe what happens, including the complete text of any error messages.
