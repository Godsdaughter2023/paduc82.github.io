If you use the snap feature to snap a non-desktop app to the side of the screen, the desktop itself will get resized to a new physical pixel size, and the icons will move accordingly (so they aren't hidden off-screen and unavailable to you).  The effect is just like if the monitor resolution was changed (users who use Projectors/TV's, or other monitors frequently see this "icon move" problem, so you aren't alone).

Another option might be to pin your icons to your taskbar instead.

MAPI stands for Messaging Application Programming Interface and the MAPI32.DLL file is a Dynamic Link Library file relating to to how Windows communicates via the network and the Internet, most often tied to Microsoft Outlook.

This cryptic error message indicates a problem with the file due to corruption of the data or, more likely, that the original file was overwritten or substituted during a recent update or software install.

Microsoft has a utility called fixmapi.exe tucked away on your computer to address such problems and it's located in C:\Windows\System32. If you browse to that folder and double click the file it should clear this up in a jiff.

considerate_guy say:

I don't know what to say other than this behavior has been reported but is unlikely to get fixed.

What is wrong with Movie Maker

Movie Maker uses Windows DVD Maker to create a video DVD, if it is present on your version of Windows. So if you are using Windows Vista or Windows 7, you see the Burn to DVD option.

However, since Windows DVD Maker is no longer supported (i.e. isn't included in Windows 8 or 8.1) we are stuck with whatever state it was in back when Microsoft stopped updating it. In that state, Windows DVD Maker doesn't seem to get the correct widescreen flag from Movie Maker. It also can't handle .mp4 video directly since it expects .wmv instead.

Personally, I gave up on Windows DVD Maker when Microsoft stopped including it. I'm guessing that Microsoft decided that consumers no longer need to create a movie in this legacy format. In a sense they are right because the DVD is a standard-definition format with a maximum resolution of 480p (NTSC). It also uses the older MPEG-2 compression standard. I think they probably got tired of the licensing fee that had to be paid for every copy included in Windows and/or just saw it as not as useful as it used to be.

I usually just stream my media from a NAS to our connected devices in our home, say via Windows Media Server. If I need to share with family, I usually just give them a link to the video stored on YouTube or Vimeo. Not only does this allow for HD video, it means I don't have to send a physical item to share video.

tl;dr - There's an issue with Windows DVD Maker not respecting the widescreen flag sent by Movie Maker. Find another video DVD burning solution (e.g. Freemake Video Converter, DVD Flick, DVD Styler, etc.). Or find an alternate way to share and view videos (e.g. online sharing, media server sharing).
