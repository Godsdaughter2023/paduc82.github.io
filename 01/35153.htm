I'll have a guess. Sharing a store folder between Windows users won't really work, because the program is designed as a strictly per-user application. In particular, passwords are encrypted using a key based on the user profile, so switching user account will switch key and the password won't work.

It's quite possible - though a bit inefficient - to share an email account between different user instances of the program. What you can't achieve consistently is sharing folders under Storage folders. If you don't need this, then simply allow each user to have his own store folder.  



¡firedog say:

Let's clear up a potential source of confusion first. The Store folder is the main folder where the program stores most of its data - messages, the message database, mail account details, spelling dictionaries for extra languages and custom spelling dictionaries. Each account has two folders, one for settings and another for messages.

In addition to messages in the account folders, there may be messages in another folder called Storage folders. This is a folder with several functions, but the main one is as a repository for messages not in account folders.

I pointed out that it's not easy to share a Store folder between Windows user accounts, not least because of the problem with encryption. I've no idea how this encryption works.

It is quite possible to set up the same mail account in two instances of the program running in different Windows user accounts.

  • For IMAP and HTTP accounts, this means that both users will see the same messages in the same folders.
  • For POP3 accounts, they will both be able to download the same messages so long as the account is configured to leave a copy of messages on the server. Neither will be able to see messages sent by the other.
