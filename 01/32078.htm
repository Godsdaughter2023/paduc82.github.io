sandeep_km say:

Hello,

Thank you for writing to Microsoft Community Forums.

I understand your concern, if the workstations in your organization are networked, you likely depend on network server software to perform certain functions, such as file and print sharing. To access this server software legally, a Client Access License (CAL) may be required.

1.    Answering you first query, With the User CAL, you will have to purchase a CAL for every user who accesses the server to use services such as file storage or printing, regardless of the number of devices they use for that access. With a Device CAL, you purchase a CAL for every device that accesses your server, regardless of the number of users who use that device to access the server. Refer the article on Client Access Licenses and Management Licenses for more information.

2.    Answering your second query, a Client Access License (CAL) Suite is a single license that provides use rights that are equivalent to multiple licenses. There are two types of Microsoft Client Access Licenses Suites, Microsoft Core CAL Suite and Microsoft Enterprise CAL Suite, which includes all of the functionality of the Microsoft Core CAL Suite. I suggest you to refer the section under CAL Suits from the following article on Client Access License Suite licensing rights for more information.

3.    Answering your third query, under the per Core model, when the server software is running in the physical OSE, you must license all physical cores on the server. To determine the number of core licenses you need, count the total number of physical cores for each processor on the server, and then multiply that number by the appropriate core factor. You do not need to purchase additional CALs.

4.    Answering your fourth, as primary and secondary server are two independent server, you can purchase different Licenses for both the servers.

If you need more information on SQL and CAL licensing, I suggest you to contact Microsoft Licensing Service Center for more information.

To add a new remote shared printer using the native 'Epson Stylus Photo 1290 (M)' driver, this is what you do:-

From Control Panel, select Devices and Printers and select 'Add a printer'.

When Windows 10 starts looking for printers, Select 'The printer that I want isn't listed'.

Choose 'Add a local printer or network printer with manual settings'.

Select 'Use an existing port'. LPT1: (Printer Port).

Choose the native 'Epson Stylus Photo 1290 (M)' driver.

This will create a new local printer using the native driver.

The next step is to change the port settings to reference the remote shared printer...

Double click on the newly created printer in Devices and Printers followed by 'Customise your printer'.

Choose the 'Ports' Tab.

Select 'Add Port' and Choose 'Local Port'.

Click 'New Port...'

A dialogue box opens requesting the port name.

You need to enter the path to the remote shared printer including the remote hostname like this:-

e.g. "\\<hostname>\<shared printer name>"

Note: It may be possible to create the new port when first adding the printer without the need to add a local printer first followed by changing the port settings. This just happens to be the way I did it.
