lemp say:
Yes, but you must connect the Ethernet cable from the Win 7 computer's Ethernet NIC to one of the local network ports on the router (NOT to the WAN or Internet port, which must be left vacant).  Thus, for most routers (which have 4 local ports) you will be able to connect a maximum of 3 additional computers.

Furthermore, in order to be able to access the router's configuration utility from any of the local computers, you must be configure its LAN-facing port to be in the same subnet as that used by ICS while at the same time not conflicting with the ICS-assigned IP addresses.  I haven't looked at Win 7, but in XP, ICS forces the port on the "host" machine (the one connected to the Internet) to have an IP address of 192.168.0.1 and this can't be changed. 

ICS will allocate IP addresses to the other computers in the range 192.168.0.2 - 192.168.0.x.  I suggest setting the router's LAN-facing IP to 192.168.0.99 (and make a note of it and tape the note to the router).

See the following tutorial:  http://www.sevenforums.com/tutorials/76617-internet-connection-sharing-enable.html  (note that you almost certainly will not have to use a cross-over cable as indicated in Step 1 of the tutorial -- although such a cable will work -- because most modern home routers have auto-sensing Ethernet ports).

lemp say:
"Correct" means "it works."  If it doesn't work for you, then there is something wrong with your setup or your hardware.

You still don't seem to follow what I'm writing.  Perhaps I'm not being clear enough. 

Connect your Windows 7 computer to the Internet using the wireless adapter in that computer.

Now connect an Ethernet cable between the Windows 7 computer and ONE of your other computers (that is, don't use the router at this point). 

The idea is to see if you have ICS properly setup in the Windows 7 computer.  If you connect the two computers directly this way, then you probably will need a crossover cable (as described in the tutorial) unless both of the NICs involved have auto-MDI/MDIX capability.  Many older computers have NICs that don't have this capability, but almost all home
The Windows XP computer should have its NIC configured to obtain an IP address automatically (this is the default).  ICS should assign it a proper IP address (which you can check using ipconfig from a Command Prompt on the XP computer).  The IP address of the Windows XP computer should be in the same subnet as the ICS-assigned IP of the Windows 7 NIC -- which as you correctly pointed out is 192.168.137.x (x may be between 2 and 254 for the XP computer).

Once you have established that ICS is working, you can then substitute the router as the device directly connected to the Windows 7 computer.  As I wrote earlier, the wire from the Windows 7 computer should be connected to a LAN port on the router (because you don't want it to go through the NAT part of the router, which it would do if connected to the WAN/Internet port).

Note that when you have the router involved you don't HAVE to configure its LAN-facing IP address to 192.168.137.y, but unless you do so, you won't be able to communicate with the router's configuration utility from one of the connected computers without first manually configuring the IP address of the NIC in one of the computers (other than the Win 7 one) to be in whatever subnet the router uses (default for this depends on the manufacturer).

ferdz amp say:

Hi Jan,

This issue could be caused by the User Account Control settings being enabled. This problem occurs because UAC treats members of the Administrators group as standard users. Therefore, network shares that are mapped by logon scripts are shared with the standard user access token instead of with the full administrator access token. You can read more information about this and the suggested resolution in: Some Programs Cannot Access Network Locations When UAC Is Enabled. The documentation provided is tagged for Windows Server 2008 R2, though related settings and resolution is also applicable to Windows 10.
