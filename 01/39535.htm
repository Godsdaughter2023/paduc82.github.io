The installation of that software replace the XP MSGINA.dll with it's own RtlGina2.dll.

MSGINA.dll: Microsoft Graphical Identification and Authentication

Usually what people notice is the Netgear GINA DLL disables Fast User Switching and changed the way the logon process works.

If your system does not boot at all in any mode, perhaps the installation of the Netgear software miserably failed.

The installation installs the new RtlGina2.dll and then changes a value in the registry to tell XP to start using that file instead of the MSGINA.dll.  It looks like at least part of your installation went okay (the registry changing part), but maybe the RtlGina2.dll did not get installed at all or only partly.

If I were you, I would boot into the XP Recovery Console and copy the XP MSGINA.dll and call the copy RtlGina2.dll to try to "fool" XP into at least loading a DLL that works.  Then when XP tries to find the RtlGina2.dll  (like the registry is telling it) it will really be loading the copied and default XP MSGINA.dll.  Pretty sneaky huh?

matthew_omega say:

Hello EthanEzrahite,

 

I have to echo Jose's strategy here.

 

The gina <pronounced Jee-nah> is the logon prompt and certain softwares liked to replace it pre-Windows Vista.

 

There are a couple of ways to approach this:

 

  1. Boot to recovery console
  2. Attach the drive to another machine.

 

If you have an XP CD, Recovery Console CD, or are fortunate enough to have Recovery Console installed so that it shows up on the boot menu, we can swap out the gina files, boot into Windows XP, and change the gina reference in the winlogon key:

 

PART1:

Each command in bold should be followed by the <Enter> key

  1. Within Recovery console, navigate to C:\Windows\System32:
    cd\windows\system32
  2. Type the following to rename rtlgina2.dll <if it is there> :
    ren rtlgina2.dll rtlgina2.old
  3. If the file is not found, proceed to step 4.
  4. Type the following to copy msgina.dll to rtlgina2.dll:
    copy msgina.dll rtlgina2.dll

We will now create a backup of the registry's System and Software Hives:

  1. Change directory to the config folder:
    CD config
  2. Copy the system hive to system2:
    Copy system system2
  3. Copy software hive to software2:
    Copy software software2
  4. Exit the recovery console:
    EXIT

This should allow us to get into Windows and we have a good backup of the software and system hives in the event that we need to swap them back into place. 

 

PART2:

We now need to remove the reference to it in the registry:

<At this point, we no longer press <Enter> for everything in bold.

  1. Click Start and Type:
    regedit
  2. Press <Enter>
  3. Navigate to the following key:
    HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\WinLogon
  4. Delete the GinaDLL key on the right hand side

 

We should be able to boot with no problem now.

 

If we are connecting the disk to another machine, part1 is slightly different:

 

After attaching the disk to the other machine, perform the following:

 

  1. Open Windows Explorer.
  2. Navigate to the <assigned drive letter>:\windows\system32 folder.
  3. Check for Rtlgina2.dll- rename it if present to Rtlgina2.old.
  4. Copy msgina.dll to the desktop.
  5. Rename the desktop copy of msgina.dll to Rtlgina2.dll.
  6. Move the copy back to the <assigned drive letter>:\Windows\System32 folder

 

Creating the Hive backups:

 

  1. Open the config folder under <assigned drive letter>:\Windows\System32
  2. Copy the Software and System files to the desktop. <these will have no extension>
  3. Rename the Software and System files on the desktop to Software2 and System2.
  4. Move Software2 and System2 from the desktop to the <assigned drive letter>:\Windows\System32\Config folder.

 

At this point you may move the disk back into its proper machine and we should be able to boot and proceed with part2.

http://support.microsoft.com/kb/817142/en-us
