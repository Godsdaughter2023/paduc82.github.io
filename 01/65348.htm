If you start Windows and see a blank or black desktop with no icons, video adapter problems are the most common cause of this issue.

 

Try the following troubleshooting steps:

 

Method 1: Gain access to your computer and reinstall your current video driver

 

Uninstall your video driver and allow Windows to re-install it:

 

1. Click Start and type Device Manager in the Search box

2. Click Device Manager under Programs

3.Select the Video driver and uninstall it

4. Restart your computer to allow Windows to reinstall the driver

 

Note: Once the driver is reinstalled you may need to reboot your computer again to complete the installation. Additionally, you may also need to adjust the screen resolution to its previous setting.

 

 

Method 2: Enable low-resolution video (640x480), and then update your video adapter driver

 

If resolving registry issues does not work, perform the following steps to enable low-resolution video:

 

  1. Click the Start button, click the arrow next to Shut Down, and then click Enable low-resolution video (640×480), and then press Enter.
  2. Log on to your computer with a user account that has administrator rights.

 

If you can see your desktop, there's a problem with your video adapter. Try updating your video adapter driver. For more information,

 

Update a driver for hardware that isn't working properly

http://windows.microsoft.com/en-us/windows-vista/Update-a-driver-for-hardware-that-isnt-working-properly

 

Method 3: Start your computer in Safe Mode and disable your video adapter

 

If enabling low resolution and then updating your video adapter driver did not work, perform the following troubleshooting steps to disable your video adapter driver:

 

Start your computer in safe mode

http://windows.microsoft.com/en-US/windows-vista/Start-your-computer-in-safe-mode

 

Note: Safe mode starts Windows with a limited set of files and drivers. Startup programs don't run in safe mode, and only the basic drivers needed to start Windows are installed

Update a driver for hardware that isn't working properly

Method 5: If these steps did not solve your problem, try booting into Safe Mode again and follow the steps in System Restore to restore your computer to an earlier point in time.

Best Quality = original image file with no compression applied

OneDrive has never converted the file names, it just uses whatever the camera sends (or whatever file name you use). As principle it never modifies a user's files or names (which would make people angry if they set a name and suddenly OneDrive changed it).

zachd123 say:

I suspect there a number of unrelated concepts in this discussion, so let's separate them out:

* IIRC Live Tiles are just limited to what is displayed on screen.  When other tiles scroll into view those will be enlivened, but there's no point to updating them if they're offscreen.

* There's a finite amount of defined space available on the Lock Screen and thus a limited number of slots available for apps that want to run on the Lock Screen.  This is controlled in PC Settings::PC and Devices::Lock Screen.


I'm running ~13 major discrete applications right now (including compiling Windows) on my Windows 8.1 system without a hitch.  I would suspect that you are not having problems playing TV or ripping video: if you are, those failures would be wholly unrelated to the concern as regards Lock Screen slots and would be worthy of their own independent threads.


The clock you mention is an interesting and weird thing: are you referring to the Live Tiles or the Lock Screen here?  IIRC, Live Tiles clocks have often been implemented ... "intriguingly", flooding the Tiles Notification System with updates.  That should all be handled now via throttling, but Tile clocks are indeed an interesting scenario.  If you're using a(n additional) Lock Screen clock - why?  There should already be the built-in clock on the Lock Screen.


Anyways, I read your comments as stating (net relative to LS) that you wish you had more Lock Screen slots to work with.  The slots are established in regards to a physical layout surface (the Lock Screen), so there is indeed a limit there that would not be present if you were just trying to run applications without worrying about a clear layout.


I hope I've made this opaque area slightly clearer.  This is indeed a very interesting area, and I appreciate your interest here.  We'll see how it can be improved for the future. =]
