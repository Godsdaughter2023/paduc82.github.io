duy bui say:

I'm very sorry if my reply seems rude: You have absolutely no ideas what the problem is about, and that's understandable if you don't use the language.

Ever since the Vietnamese locale was introduced into Windows (Windows 2000 if memory serves), it has been using the Windows locale 1258 code page (aka "Composite Unicode").
In that code page, the visual tones (there are 5 of them) are depicted as an EXTRA character (&#768; in the example above).

However, the Vietnamese community has MUCH MUCH MORE WIDELY adopted another Unicode code page (aka "Predefined" Unicode) for the language, in which the visual tone and the vowel it modifies are combined into ONE character (&agrave; in the example above).

The fact that there are 2 code pages (whose difference can't be told from the displayed outcome) brings about quite a few problems:
1. Indexing & Searching
If a document was created and indexed in one code page, it cannot be accurately searched when the keyword is typed with the other code page.
2. Fonts
There are nearly a thousand fancy Unicode fonts that support the "predefined Unicode" code page.
3. Other uses of texts that require exact matching
