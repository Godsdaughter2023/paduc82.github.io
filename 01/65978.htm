¡firedog say:

I wonder where you found those instructions. The chances are that ownership of the store folder or perhaps only of the .MSMessageStore file has changed. Could you have done anything to cause this - perhaps running this command earlier? Or changing the location of the store folder?

In any case, this is how to regain access to the file:

  

  1. Find the name of the current Windows user:
    Open an administrator command window*     and type whoami. Write down the username displayed, e.g. JohnnyPC\Johnny
     
  2. Take ownership of the message store file:
    Type the black line (or better still, copy it, then in the command window right-click and select     Paste):
    takeown /f "%localappdata%\Microsoft\Windows Live Mail\Mail.MSMessageStore"
    You should see a message like this:
    SUCCESS: The file (or folder): "C:\Users\Johnny\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore" now owned by user "JohnnyPC\Johnny".
     
  3. Give yourself full control of this file:
    Type the black line (or better still, copy it, then in the command window right-click and select Paste):
    icacls "%localappdata%\Microsoft\Windows Live Mail\Mail.MSMessageStore" /grant "JohnnyPC\Johnny":f
      
    substituting your own user account from (1) for     JohnnyPC\Johnny.
    You should see a message like this:
    Processed file: C:\Users\My Windows Username\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore
    Successfully processed 1 files; Failed processing 0 files

  

If you then execute the esentutil command to defragment the store folder database index, you may find that you see error 0x800C0155,n. If this is the case, just follow steps 1-3 above again. 

  

* To open an administrator command window, press Windows key + R to open a Run box. Type cmd and press Ctrl-Shift-Enter (press and hold both Ctrl and Shift, then tap Enter). Give permission when you're asked for it.
