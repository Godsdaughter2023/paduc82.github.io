MS WINDOWS 10 INACCESSIBLE_BOOT_DEVICE or How to Fix that error:)

Hi to all,

After spending a lot of hours trying to fix the never ending loops of restart caused from INACCESSIBLE_BOOT_DEVICE error,  found a working solution and I want to share with you.

First of all, this solution it's not mine but from user "Dzomlija" published at 22 Jan 2011 in URL:

Startup Repair Infinite Loop Recovery  with title: How to Recover from an Infinitely Looping Startup Repair Loop without Reinstalling Windows.

The article was for Windows 7 but it works as is for Windows 10.

Let's go:

1. Use the installation disk from Windows 10 to access advanced startup options (an older Windows 8 or 7 disk it works too).

2. Choose Troubleshoot>Advanced options>Command Prompt

3.You need to find your system drive (where windows 10 installed).
Depending on how your system is setup, this could be either C: or D:.

I.Type C: and press <ENTER>.
II.Verify that this is your system drive by typing DIR and pressing <ENTER>. If you see the Program Files, Users and Windows folders, then you have found your system drive, and can continue to step 4 below.
III.If the drive is not your system drive, repeat steps I and II above, changing the drive letter to D, E or some other letter until the system drive is located.
4.When you have located your Windows system drive, type CD \windows\system32\config and press <ENTER>:
5.Type DIR and press <ENTER>, and verify that the following files and folders exist in the config folder:
-RegBack (which is a folder)
-DEFAULT
-SAM
-SECURITY
-SOFTWARE
-SYSTEM

6.Type MD mybackup and press <ENTER> to create a backup folder that you can use incase this procedure does not work as expected.

7.Type copy *.* mybackup and press <ENTER>.

If you are prompted to overwrite existing files, press A to allow all backups to be overwritten.

8.Now you need to check if you can use the automatic Windows backups to restore your registry:

I.Type CD RegBack and press <ENTER> to go to the RegBack folder.

II.Type DIR and press <ENTER> to view the contents of the folder. All the following files must exist:

-The DEFAULT, SAM and SECURITY files should each be about 262,000 bytes in size.
-The SOFTWARE file should be about 26,000,000 bytes.
-The SYSTEM file should be about 9,900,000 bytes.
-The file sizes presented here are approximate estimations, and may vary depending on your system. If any one of them are 0 bytes, then you should stop what you're doing now and seek an alternative method of recovering your system, because Windows cannot function with a 0-byte size registry hive.

III.If the hive files listed in RegBack are ok, then proceed to step 9 of the tutorial.

9.Type copy *.* .. and press <ENTER> to copy the backup hive files to \Windows\System32\config.

If you are prompted to overwrite existing files, press A to allow all file to be overwritten.

10.Type exit and press <ENTER> to close the command prompt.

11.Click the Turn off and the Reboot your computer.

I hope all goes well and your system boot normally:)

Open the required ports for Xbox LIVE on your router.

Thanks to ArminatorX for the tutorial!

Solution 2

Random Disconnects or Lag can be caused by a bug in the profile.

To make sure that the issue is not your profile, please try the following solution:

  1. Delete your profile (profile only, NOT profile and all items)
  2. Clear the System Cache twice
  3. Restart your console
  4. Re-download your profile
Solution 3
Make sure that all of your cables are plugged in properly and restart your Network Hardware.
This includes:
  • Router
  • Modem
  • Xbox 360
  • Wireless Adapter for Xbox 360

  1. Press the Guide button on your controller, select Settings, and select System Settings.

  3. Select Wired Network or the name of your wireless network when the network list is displayed.

  6. Turn off your console.
  7. Turn on your console.
If using wireless: You will receive a message to configure wireless settings. Follow the on-screen instructions to connect to your wireless network.
Solution 4
If you're still experiencing Lag issues or random Disconnects, please go ahead and create a new thread explaining your issue in full detail and include your answers to the bold questions listed at the bottom of This Thread.
