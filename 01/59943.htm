3TB HGST SATA drive not recognized by Windows 10 fresh install - - Microsoft Community

semichipz say:

3TB HGST SATA drive not recognized by Windows 10 fresh install - SOLVED !!!

Motherboard: Gigabyte Z68x-ud3p-b3 using UEFI BIOS U1F - BIOS set to AHCI, IRST disabled

Installed identical twin 3GB HDDs, HGST HDN724030ALE640

Fresh install of Windows 10 Version 1511 OS Build 10586.36

Symptoms:

Windows 10 installed successfully on first HDD (with zero additional software), second HDD is not yet formatted

Device manager shows both HDDs

Disk management shows first HDD only (as Disk 0); does not recognize second HDD (presumably Disk 1) but does show Windows 10 USB installation disk (as Disk 2)

Both drives are recognised in BIOS prior to loading the operating system

Attempts to solve this:

(1) Via Device manager - scan for hardware changes; disable/enable second HDD; uninstall second HDD (then forced restart when HDD is recognized in device manager but not in disk management - thus symptoms unchanged)

(2) Via command prompt - sfc /scannow and dism /restorehealth - no obvious problems found, reboot machine, no change in symptoms

This seems utterly puzzling to me. Any suggestions?

semichipz say:

SOLVED BY MYSELF!

Tried brute force across multiple attempts:

* Updated driver software by systematically ploughing through all items in device manager - Intel chipset, Intel Management Engine (note that this stays stubbornly at Version 11.0.0.1157 rather than changing to 11.0.0.1158 as recommended by Gigabyte and Intel), USB drivers, SM bus ... yada yada but no joy

* Switched SATA Driver from Intel chipset to Marvell chipset - no joy

* Switched which SATA ports the drives were connected to, in case this mattered - no change

* Varied BIOS settings to see whether this made much difference - quite a tortured process as one often needs to reboot several times following a change in settings. Gigabyte support appears pretty much non-existent on this level of detail.

* Cleared motherboard CMOS settings by removing battery - no joy

* Added Seagate drive, downloaded and ran Seagate DiskWizard - no joy, could not see additional drive

* Downloaded HGST's own WinDFT utility - could only find one drive

* Checked and replaced SATA cables to errant drive in case of bad connection of bad cable - remained stubbornly invisible to Disk Manager (but visible on Device Manager)

* Hit several mysterious error messages - "No more IRP stack locations" and "Microsoft storage spaces ... does not like this drive and has rejected it" ... still no joy

ROUTE TO FINAL SOULTION:

* Removed the missing hard drive, installed in an external drive caddy - still not recognised by main PC

* Tried the external caddy in a separate laptop running Windows 10 Pro - nope, still invisible

* These steps confirmed for me that the issue was generic to Windows 10 software rather than my desktop PC's BIOS, drivers or SATA cables, for example

FINAL SOLUTION:

* Ran a pair of Linux utilities from the excellent and free "Ultimate Boot CD"  - called GParted and PartedMagic (can also be run in other ways) - this was able to show the partition table structure in all disks on my system - The GPT partition table was as expected (four partitions) on my primary 3TB drive but was not as expected (only two partitions) on the secondary 3TB drive

* Deleted the two partitions on the "missing" 3TB so as to return the entire drive to "Unallocated" space

* Re-booted into Windows 10 Pro - now Disk Manager was finally able to recognize the "missing" 3TB drive ... then followed with what I had been trying to do in the first place ... initialize this drive, mirror the partitions with the primary 3TB drive etc.

DIAGNOSIS:

* Windows 10 installation (or repeated repair attempts) failed (for some reason) to successfully create a set of GPT partitions on the "missing" 3TB drive

* Virtually all the Windows based utilities which I tried did not recognize a badly partitioned disk and (even worse) did not flag up any error message about this

* Clearly Microsoft Community support is not up to speed on this issue, neither is Windows 10 currently able to recognize the problem

* This problem has occupied several weeks of my time. It is extremely disappointing that Microsoft support does not seem to have a systematic guidance routine to identify mis-partitioned drives

Good luck to all others who benefit from this advice. A few "Helpful" votes would be nice. Cheerio.
