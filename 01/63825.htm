Thanks for your reply. After reading the above I could understand that you are having problems with the path of your user profile as it is in Chinese, and not with the user name? Please correct me if I'm wrong. Please try the following to rename your user folder

Method [1]: Renaming a user account folder
  • Create a new local account with administrative privileges
  • Restart and select your new local account while logging in
  • Press the following keys at the same time to open the Run command: Win+R
  • Insert the following into the text box and click on OK: lusrmgr.msc
  • From the left pane, click on Users
  • From the right pane, right-click the user you are renaming and select Rename
  • Type the new name ($newname) and press Enter to save your changes where $newname indicates the new name you would like to set
  • Close the window
  • Press the following keys at the same time: Win+X
  • Select Command Prompt (Admin) from the list
  • Insert the following and press enter: REN C:\Users\$oldname $newname
    where $oldname indicates the current name of the user and $newname indicates the new name you would like to set
  • Press the following keys at the same time to open the Run command: Win+R
  • Insert the following into the text box and click on OK: regedit.exe
  • Navigate to the following key: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList\
  • From the left pane, click to expand ProfileList and locate the folder that is responsible for the user you are renaming (To detect which folder is responsible for the user, check the ProfileImagePath value from the right pane)
    - To see a screenshot, please click here
  • When you have indicated the correct SID for your user account, select its folder under ProfileList
  • From the right pane, right-click on ProfileImagePath and change it as follows: C:\Users\$newname where $newname indicates the new name you have set before
Additionally, you might want to replace all entries in the registry that matches your old full path of the user account with the new full path you have set.

IMPORTANT: Some applications may encounter problems while trying to access your old user full path. To avoid this, please try to add a Symbolic Link that would transparently redirect applications that access your old user full path to the new user full path

To do this, please open Command Prompt (Administrator) and insert the following:
mklink /d C:\Users\$newname C:\Users\$oldname

matt smith - segana say:

In all likelihood Surface support probably don't realise the end pins are like that.


They'll be most likely that way so that when plugged in, they're pushed in, thus completing the circuit and initiating the charging cycle and charging light.


If yours appear to have excessive play in them though, then that's probably not right. They should be centrally aligned (maybe a little lean to them) and should push down and spring back up when released.


That knowledge however should have no bearing on the fact your Surface doesn't fully charge. So out of interest, what does it actually charge too? I have heard others talk about their Surface 2 batteries not fully charging, so you're definitely not alone there.


2 things I'd point out/suggest:#


1. Not fully charging actually has its benefits as its been shown to extend the battery life. Some devices actually have an option to stop them fully charging (normally charge to 80%) in order to extend the battery life.


2. There's a new batch of updates available for the Surface 2 today (Patch Tuesday), so I'd recommend installing those and then as long as you feel the charger is safe, plug the Surface in and see if it fully charges after the updates.


Matt
