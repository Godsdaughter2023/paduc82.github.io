Event Viewer Reports

1. Normally when an error occurs on your computer looking in Event Viewer should be your starting point for finding a solution. Most system related errors are logged and getting an exact copy of the relevant report is important. Unfortunately understanding the reports is not easy and most computer users need help with their interpretation. I have more to say later on interpretation.

2. Event Viewer comprises three main Windows logs. These are Application, Security and System. For troubleshooting purposes System is by far the most important.

3. To access the System log select Start, Control Panel, Administrative Tools, Event Viewer, from the list in the left side of the window select Windows Logs and System. Place the cursor on System, right click and select Filter Current Log. Check the box before Error and click on OK and you see only Error reports. Click on the Date and Time Column Header to sort. You may need to click a second time to see the latest Report at the top.

4. A tip for posting copies of Error Reports! Run Event Viewer and double click on the error you want to copy. Click on the Copy button to place a copy on your Clipboard and close Event Viewer. Now start your message and paste into the body of the message. Make sure this is the first paste after exiting from Event Viewer.

5. There are three types of Report, being Information, Warning and Error reports. In most situations it is Error Reports that offer the best information but occasionally Warning Reports provide useful clues.

6. All reports have date and time stamps and when troubleshooting it is important to concentrate on more recent reports. Study reports since the point when the computer was last booted and then check whether a similar report appeared in the previous session. If errors do not repeat investigation as to why they occurred is wasted effort.

7. Within individual reports the more important information is Event ID and Source as these help when looking for help on the internet. The description is equally important and copying the exact text for use as the search criteria greatly helps getting better results when using Google. Do not paraphrase descriptions when asking others for help.

gerry c j cornell say:

PCTools Performance Toolkit (registry maintenance)

Registry cleaners provide negligible benefits with the potential to create system errors which are insoluble other than by a clean reinstallation of the operating system and all that entails.

Updating Windows commonly fails because the process is not allowed to complete. After installation with some, but not all, updates the system requires the computer to be restarted. During the shutdown the system is reconfigured. Users can become frustrated when the shutdown takes some time to complete and they intervene by pressing the Power button. When the computer is next booted or automatically restarted the process of reconfiguring the system is completed. Reconfiguration can take a time and prolong the boot process thereby providing a second situation where the user becomes frustrated and intervenes by pressing the Power button.

When the Power button is pressed the process of updating files is interrupted and there is the real risk of file corruption. You will then see problematic and prolonged shutdowns and boots until the underlying corrupted file corruption is resolved. Resolving this is not always easy.

It can help to understand that with automatic updating of Windows enabled the process occurs in the backgound. The user may therefore not be fully aware of what is going on. Automatic updating is supposed to ensure that all users update their version of Windows, which is clearly desirable. Unfortunately the lack of awareness of a user to what is going on generates a lot of problems resulting in failed updates. For this reason it can be better to set Windows to tell you about Updates so that you can manually download and install updates at a time when the computer is not being used for other activities.

The failure to load drivers is what is my immediate concern with regard to your computer. You did not answer all my questions. The one regarding whether you had booted to safe mode on 20 July was important with regard to the failure to load drivers. I am also conscious that you could have used your Registry Cleaner and that this could have damaged your system.

It is always sensible in this situation to run a check for malware

Boot the computer to safe mode with networking and download and install Malwarebytes (freeware version for home users only), update definitions and run in safe mode. Disable other security software whilst you are doing scans.
