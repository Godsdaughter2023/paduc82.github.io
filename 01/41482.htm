I don't have an answer for you, but I wanted to point out two things:


1.  Wanting to minimize the amount of memory Windows uses is a
counterproductive desire. Windows is designed to use all, or most, of
your memory, all the time, and that's good not bad. Free memory is
wasted memory. You paid for it all and shouldn't want to see any of it
wasted.


Windows works hard to find a use for all the memory you have all the
time. For example if your apps don't need some of it, it will use that
part for caching, then give it back when your apps later need it. In
this way Windows keeps all your memory working for you all the time.


2. "System Idle Process" is not any problem at all. "Idle" means "not in use." That's just the name for what the system is doing when it's not doing anything. It's there just to make the total add up to 100%

foxtwo say:

EFS with Smart Card in Uncached Key Mode

 

I have read the document at http://technet.microsoft.com/en-us/library/cc162818.aspx and I have a question about the mitigated risks using EFS with smart card in Uncached Key Mode.

Extracting from the Microsoft document regarding the mitigated risks using EFS with Smart Card: "Platform attacks (Windows Vista uncached key mode only). In cached key mode, a computer configured to use EFS with smart card key storage will maintain the EFS keys in memory, and a platform attack might succeed in recovering them. EFS used with uncached smart card key storage effectively mitigates this attack by requiring a direct attack against the smart card to recover the key material."

While i clearly understand that using EFS with smart card in uncached key mode is much more secure than using it in cached key mode, I don't understand why Microsoft says that in uncached key mode the platform attacks risks are mitigated. I mean, although it is true that the private key used to encrypt the feks is never exported in windows and all the decrypting operations of the FEKs happen inside the smart card microprocessor, it is also true (in my understanding) that, once decrypted by the smart card, the fek(s) are inside windows memory (I suppose in a protected memory area) for the reading and writing operations and a platform attack might succeed in recovering them.

The platform attacks risks using the uncached key mode are much smaller than the ones you may have using the cached key mode because in the cached key mode the platform attack could recover the derived key used to encrypt all the feks and it can use it to decrypt all the efs encrypted files while in the uncached key mode the platform attack can only recover the feks of the encrypted documents that are in memory while the user is using them for reading and writing.

So, although the risks in a platform attack scenario using EFS with smart card in the uncached key mode are very low, nevertheless they exist.

Can you confirm that I am right?

Thank a lot for reading

Michele
