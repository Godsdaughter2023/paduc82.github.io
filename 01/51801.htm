This sounds like it is a codec issue. Windows Explorer is building a preview image and rundll32 is crashing when using a codec that is buggy.

 

To find out what is causing your issue, you need to isolate a single file that causes the rundll crash and then try to find out what codec that file requires. To determine what codes was used with a specific file, play the file in the Player, if possible. While the file is playing, right-click the file in the library, and then click Properties. On the File tab, look at the Audio codec and Video codec sections.
