Rules for populating Date attribute of files (e.g., display and - Microsoft Community

j80 say:

Rules for populating Date attribute of files (e.g., display and sort on Date in Windows (File) Explorer) (image v. sound v. PDF, etc.) (NTFS v. FAT) (Date v. Modified or Created)

The rules populating the Date attribute for files in Windows 10 1809 are not clear and changed substantially between Windows7 and 10.  The Windows 10 rule appears to be approximately populate Date from Date Modified (or maybe the later of Date Modified and Date Created).   Date Last Saved may also be a factor.  Sometimes an internal attribute for some file types takes precedence when present: Date Taken from exif for many image files takes precedence for Date over Date Modified.

Note I have not found a remotely complete list of internal attribute with corresponding file type (e.g., Date Taken for .jpg)..

Note that copying a file versus moving a file has different effects on Date Created and Date Modified and also depends on the file system (NTFS v. FAT).  The only apparently complete rules for several file systems for Windows 7 disappeared.  Downloading a file can result in Date Created being later than Date Modified.

Note Windows 7 works partially opposite from Windows 10 1809: Win7 Date is the earlier of Date Created or Date Modified while Win10 Date is either later of created or modified OR just uses Date Modified. (My Win10 sample examples didn't distinguish between these cases.)

j80 say:

So what are the rules for populating Date attribute? 

Try setting up Windows (File) Explorer to show Type, Extension, Date, Date Created, Date Modified, Date Last Saved, Date Taken, and perhaps a few more in View Details mode.  Compare Date attribute values with other date based attributes values.  Be sure to look at a variety of file types and several folders if necessary. 

Focusing where Date Created <> Date Modified, I immediately see some files where Date = Date Modified (also = Date Last Saved).  Another file has Date = Date Taken.  A few (sub)folders have Date = Date Created.  Several files have Date = Date Created, but Date Modified is only 1 minute later (3 minutes in one case), so I wonder if this is a timing or duration artifact (e.g., for virus check or when a completed download temp file is copied to the final location) and Date Created is really effectively equal to Date Modified.  I don't see a situation where Date Modified is earlier than Date Created (although I have in the past) so I can't evaluate that case.
