Windows should not throw away error messages like that!  At some point in each of these processes, they tried to write something to TEMP, and got an error.  They should have saved that error and reported it back up the call stack, not thrown it out and reported a different error!

Throwing away error messages is usually due to code that catches errors.  This is because most coding languages will report the error correctly if they crash normally, but code that catches errors within a program will usually only know where the error was caught, losing the error (unless that specific error is checked for).  Catching errors is nearly always a bad idea that makes code more buggy, because the way coding languages implement it assume code errors other than those checked for in the catch don't occur.
