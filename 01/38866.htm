chriso747 say:

Hi TechNow,

As you know, FSX throws up different types of problems for which there could be multiple possible solutions.   Looking at your problem, it seems that your FSX crash is occurring when you are using the top Menu bar in-flight.   Does this situation occur at any other time?  For example, after a long flight or when loading a lot of detailed scenery?   Then this could indicate a Virtual Address Space (VAS) problem.

If it does happen only when using the Menu bar (and when using it very often), then generally an installation of a .dll into the FSX directory should fix the problem.   I have detailed this matter to others and have pasted my reply hereunder to you as well.  It is as follows:

"This is a fairly common error in FSX (there are many others that come up of course!).   The UIAutomationCore.dll that FSX tries to use when needed is found in the System 32 and SysWOW64 folders in your Windows directory.  However, FSX is not able to use that newer version of the .dll and so it crashes.   Do not remove any of these versions in the Windows directory as these .dlls are needed for other programs.  

To get around this error, one needs to download an earlier version of the .dll.   There are two versions of the .dll that can be used:   one is 6.0.5840.16386, and the other, 6.0.6001.18000

You can go to your preferred site to download .dlls, or else, you can try here:

For the 16386:   http://www.search-dll.com/dll-files/download/uiautomationcore.dll.html   (32 bit version only)

For the 18000:   http://dllworld.org/uiautomationcore-dll/   (32 and 64 bit versions)

Since FSX is a 32 bit application, select the .dll for that one.   I use the 16386 version.   After you download it, copy/paste it in the FSX Directory (not anywhere else).   FSX will search its own backyard for the .dll when it needs it before it searches elsewhere in Windows."  

Hope you have success with this.

Chris
