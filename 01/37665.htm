rohn007 say:
I don't see any provision to do that in Windows. Windows is designed from the ground up that file types are associated with only one program. So Windows can't image a situation where we would want to disassociate a file type from a program.

There are a few ways to reach that goal.

Uninstall the application that is currently assigned as default. The file type will be freed up. Then you can re-install it. As long as that file type is not normally associated with the program it will not be picked up again.

Do a Registry Hack. Find the specific key in the registry that the file type and program are linked, delete it.

Trick Windows. Change the file association to a new dummy "program". Delete the program. Specifically:

. * make a copy of any file on your desktop.
. * Change the filename to a dummy name, ie dummy.exe
. * Say yes to the warning about changing the file extension
. * Right click on a file of the type you want to disassociate
. * select "Open With"
. * select "Choose Default Program"
. * scroll down to "Brows for an app on this PC"
. * find your dummy.exe file

If you look in File explorer you'll see that it is now associated with dummy.exe
Now, delete dummy.exe from the desktop.
The file association is now broken, so it should now ask what program to use.

To perform a clean boot on a computer that is running Windows Vista:

 If you are prompted for an administrator password or for a confirmation, type the password, or click Continue.

5. Click OK.

6. When you are prompted, click Restart.

7. After the computer starts, check if the issue is fixed.
