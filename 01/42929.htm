All are of the PAGE_FAULT_IN_NONPAGED_AREA (50) bug check.

This indicates that invalid system memory has been referenced.

Bug check 0x50 usually occurs after the installation of faulty hardware or in the event of failure of installed hardware (usually related to defective RAM, be it main memory, L2 RAM cache, or video RAM).

Another common cause is the installation of a faulty system service.

Antivirus software can also trigger this error, as can a corrupted NTFS volume.

Throughout the dumps we are seeing a consistent igdpmd64.sys fault. Even more interesting, let's take a look at the call stack from one of the dumps:

Which should I download?

You can either download the pre-compiled ISO that you would burn to a CD and then boot from the CD, or you can download the auto-installer for the USB key. What this will do is format your USB drive, make it a bootable device, and then install the necessary files. Both do the same job, it's just up to you which you choose, or which you have available (whether it's CD or USB).

How Memtest works:

Memtest86 writes a series of test patterns to most memory addresses, reads back the data written, and compares it for errors.

The default pass does 9 different tests, varying in access patterns and test data. A tenth test, bit fade, is selectable from the menu. It writes all memory with zeroes, then sleeps for 90 minutes before checking to see if bits have changed (perhaps because of refresh problems). This is repeated with all ones for a total time of 3 hours per pass.

Many chipsets can report RAM speeds and timings via SPD (Serial Presence Detect) or EPP (Enhanced Performance Profiles), and some even support changing the expected memory speed. If the expected memory speed is overclocked, Memtest86 can test that memory performance is error-free with these faster settings.

Some hardware is able to report the "PAT status" (PAT: enabled or PAT: disabled). This is a reference to Intel Performance acceleration technology; there may be BIOS settings which affect this aspect of memory timing.

This information, if available to the program, can be displayed via a menu option.

Oh, I know that the device drivers offered via Windows Update are
provided to Microsoft by the divers manufacturers, but those drivers are
not necessarily the same as those available directly from the
manufacturers' web sites. I've seen numerous cases where the drivers
obtained via Windows Update, while providing proper basic device
functionality, lacked some of the more "advanced" features (i.e., the
ability to duplex print jobs, gamma correction of video, and/or sound
mixing) of the manufacturer-supplied drivers. This has been most
noticeable of printer, video and audio adapter drivers, from multiple
manufacturers. As for specific manufacturers, I've encountered this
condition with NVidia, Creative, ECS, Epson and Hewlett Packard, to name
those that most readily come to mind; though there seems to be no
uniformity across those vendors' product lines.

It's not so much that the vendors are trying to "short" the users of
Windows Update, I don't believe, but that the level of testing that
Microsoft rightly requires before distributing someone else's product
makes it much "simpler" for the hardware vendors if they provide a more
basic product to be tested. Also, the necessary delays inherent in any
worthwhile Microsoft compatibility testing process will quite often mean
that the drivers available via Windows Update are not the most recent
available.

Arguably, one could maintain that many of these advanced "features" are
not truly the function of the device drivers, but rather those of the
proprietary utilities and applications that the manufacturers provide to
more fully leverage the capabilities of the actual basic device drivers
form their web-sites, but not via Windows Update. To the end-user,
however, it will generally seem that the drivers available from the
manufacturer, which automatically come with these utilities and
applications when downloaded from the manufacturers' sites, are more
"powerful" than the drivers alone when downloaded/installed from Windows
Update.
