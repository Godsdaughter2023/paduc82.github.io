lemp say:

It sounds as if some program is running something in a Command Prompt window, but if you can't associate it with some program that you are running and can't even read what's on the Title Bar of the window as it flashes by, tracking it down will be a tedious process.

Can you recall what changed about your system shortly before this behavior started?  Did you install some software?  Update something?  Install some hardware?

What are your antivirus program and your other security software?

Does this behavior start essentially as soon as Windows starts?

Or does it only start after you've opened some program?

For a first approach, download and run Autoruns:  https://docs.microsoft.com/en-us/sysinternals/downloads/autoruns

When the scan completes ("Ready" at lower left corner), select the "Scheduled Tasks" tab.  Take a look through the list of tasks and if you don't recognize something, right-click and select "Search online." 

If you clear the box at the left end of a line in the "Scheduled Tasks" tab and restart the computer, that task will not run.  Thus, if you see something that appears suspicious and Search Online doesn't suggest that it's a necessary Windows action, clear the box, restart, and see if your "little black box" stops showing up.

You can return things to the way they were simply by running Autoruns and placing a check back in the box you cleared.

If looking in Scheduled Tasks in AutoRuns didn't solve the problem, the only other thing that I can suggest is a tedious process of trial and error.  This method is based on the assumption that the problem is caused by non-Microsoft software that starts when Windows starts.  Starting in "Clean Boot" disables all non-Microsoft services and all software other than necessary Windows components.

Configure the machine to start in "Clean Boot" by following the steps here:  https://support.logi.com/hc/en-001/articles/360023183914-Clean-boot-for-Windows-operating-systems#win7

If after the system restarts the problem still occurs, then the assumption above is not correct.  Restore the system to normal behavior following the second set of steps at the article linked above.

Hopefully, however, the problem will NOT occur after the computer restarts.  Now what you have to do is selectively re-enable the items on the "Startup" tab of Msconfig.  I suggest that you do this in groups of half of the listed items. 

That is, suppose that there are 10 items on the Startup tab. 

  • Re-enable the top 5 on the list and restart
  • If the problem recurs, then you know that the culprit is one of the 5 items you just re-enabled. 
  • If the problem does not recur, then you know that the culprit is one of the bottom 5 items.
  • Keep repeating this process until you've determined which startup item causes the problem.  Once  you've done that, you can decide what do to about it.
