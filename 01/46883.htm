Service won't stay "started" when I reboot. - Microsoft Community

cleocfi say:

Service won't stay "started" when I reboot.

I have a update service that updates automatically every other day or so.  In Services, the Startup Type is "Automatic".  I have to manually change the Status to "Started" every time I reboot.  Even when I reboot with the Status set as "Started", the Status is changed to nothing in the Status column.  I have been in touch with the service provider and they say that if the update won't happen if the Service is not "Started" and that it is not a problem with their program.  Can someone please tell me how to keep this service "Started" when I reboot?  Any help will be welcome! 
cmdrkeene say:

No services actually stay "started" in the status column when you reboot.  They all start off blank, and only start when requested to do so (by some other program asking them to please start, or from a user like yourself clicking the Start button or typing the start command).

Normally there would be a Scheduled Task in the Task Scheduler to start the service and make it check for an update.  For example, that's how things like Windows Update check for updates (and a great many other things, even non-Microsoft things like Apple's iTunes).

Since your program's updates fail when this service isn't running, and the program fails to ask the service to start, it sounds to me like this program you're using has a flaw that needs fixed.

However, you might (might!) be able to work-around the problem by setting a command to start the service when you login.  Here's how:

  1. Open the service's properties (double-click it) and note the service name at the top.
  2. Run the command shell:startup to open your Startup folder.  Right-click and choose "New > Shortcut".
  3. Use this command as your shortcut:

    net start (service name)

  4. Save the shortcut.  Optionally you can edit the shortcut to "run minimized" so you don't see it flash on the screen every morning.

Now when you logon, this shortcut will fire and start the service, the way the program is supposed to do in the background when needed.

zigzag3143 (ms -mvp) say:

The NTFS_FILE_SYSTEM bug check has a value of 0x00000024. This indicates a problem occurred in ntfs.sys, the driver file that allows the system to read and write to NTFS drives.


One possible cause of this bug check is disk corruption. Corruption in the NTFS file system or bad blocks (sectors) on the hard disk can induce this error. Corrupted SCSI and IDE drivers can also adversely affect the system's ability to read and write to disk, thus causing the error.

Another possible cause is depletion of nonpaged pool memory. If the nonpaged pool memory is completely depleted, this error can stop the system. However, during the indexing process, if the amount of available nonpaged pool memory is very low, another kernel-mode driver requiring nonpaged pool memory can also trigger this error.

jessen p say:

Hi,

Thank you for being part of Microsoft Community.

Windows Update error 80070643 can happen for several reasons. The most common cause is a problem with the .NET Framework installed on the computer.

Please follow the below steps to reset the Windows Updates Components:

  1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.
  2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

   3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

   4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

   5. Type Exit in the Command Prompt to close it.

Please feel free to revert if you face any issues with Windows in future.
