sam___ say:

Ben,

I realise this hasn't been answered in a few months, but I thought I'd fill this in other users of Storage Spaces. Basically people tend to think about Storage Space like a traditional RAID type set up. It doesn't quite work that way. Storage Space provide space using a factor of 2 so the 10TB / 2 = 5TB of usable space. A parity space consumes space using a factor of 1.5, so the 10TB / 1.5 = 6.66TB of space. Storage spaces is very flexible and adding an additional drive to the pool would move your current data around to make the most use of the additional space that you added. The reason you are seeing a small figure for your parity space is because your 3rd volume is only 2TB in size so it can only yield 3.63TB of space used. If you want to overcome this hurdle you could create 2 spaces on a single pool, a mirror and a parity. Let me know if this answers your original question. Thanks.

Sam

Hi JGT,

I'm not quite sure what you mean. I provided 2 types of resiliency that can be used on a 3 drive setup, but for clarity I will include the following table:

Storage Pool Size Resiliency Type Usable user space
1TB (using up to 5 drives) Stripe (equivalent to RAID 0) 1TB**
1TB (using up to 5 drives) Two-way mirror (equivalent to RAID 1+0 with bit rot protection*). Pool size / 2
 500GB**
1TB (using up to 5 drives) Parity (equivalent to RAID 5). Pool size / 1.5 666GB**
1TB (using 5+ drives) Three-way mirror (equivalent to RAID 6 with bit rot protection*). Pool size / 3 333GB**
1TB (using 5+ drives) Parity (equivalent to RAID 6 without bit rot protection). Pool size / 3 333GB**

*Note that in order to use the bit rot protection feature, the file system used by the Storage Space needs to be ReFS (Resilient File System).

**Note that the sizes don't include the overhead for file system meta data, etc. The actual usable space will be slightly lower.
