Oh I see, sorry I thought you just wanted to delete the history of that notification.  That particular one alerting you that a mapped network drive couldn't be connected during login, is not something you can turn off.

When you first map a network drive, there's a checkbox that says "Reconnect at Logon". When that box is checked, Windows will attempt to map the drive each time you boot up. So, it only makes sense for Windows to alert you when the drive can't be found, after telling Windows to do so.

Your other option is to uncheck the "Reconnect at Logon" box and just map the drive when you need it. Or add a desktop shortcut to the drive and use it on-demand instead of having Windows attempt (and fail) to connect during login.

At my office, I created a small batch file with two commands:

net use s: \\server\folder\subfolder

I put this on my desktop, and use it as a shortcut to my S: drive.  It handles the mapping (silently ignoring if it's already mapped) and opens the drive. But since it's not trying to connect during login, it doesn't alert me when it fails.

xbr dionysus say:

How to Find SSID (Network Name)

Login Information

 

  • Gateway: 192.168.0.1
  • Username: admin
  • Password: There is no default password

 

How to Find SSID (Network Name)

 

  1. Connect a computer to the router with an Ethernet cable.
  2. Open a web browser on a computer.
  3. Type the address 192.168.0.1 into the web address bar.
  4. This will take you to a login page.  For the Username type in admin and for the Password leave it blank then click on the Login button.
  5. Once logged in, click on the Wireless Setup on the left hand side.
  6. Click on the Manual Wireless Connection Setup.
  7. The current wireless network name will be displayed on this page in the Wireless Network Name box.
