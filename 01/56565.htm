If you plan to reinstall Windows 10 in the future, its probably best you reinstall Linux after doing so.

If you are considering reinstallation of Windows 10 with Linux setup as a dual boot, you need to carefully consider your method of reinstall for Windows 10.

Boot from install media for Windows 10

This option will remove your Linux boot manager (GRUB), which manages boot options for Linux and Windows 10 when you install it. If you boot from the Windows 10 install media then wipe the Windows 10 partition, you will need to readd a boot entry for the distribution of Linux you installed.

The easiest option I have found for doing this is EasyBCD.

Use Start Fresh in Windows 10 to perform a clean install:

https://www.groovypost.com/howto/perform-a-clean-install-of-windows-10-using-the-refresh-tool/

This option will let you perform a clean install from within Windows 10 (Anniversary or Creators Update) without disturbing your Linux installation or Boot Manager.

Another option is also Reset:

How to: Perform a clean install using Reset this PC in Windows 10

Perform a clean install by starting setup from the desktop

This is another option that will preserve your Linux boot settings and configurations. Choose the option, Keep Nothing during setup:

How to: Perform a clean install of Windows 10

Regardless of which option you choose, your Windows 10 activation settings will not be affected.

When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate. You will prompted to enter a product key a couple times through the installation, click I don't have a key and Do this later.

There is no need to know or get a new key, but if you have issues with Digital Licensing or the Activation Troubleshooter, you can utilize an existing Windows 7 or Windows 8/8.1 product key you already ownto reactivate if necessary. or use the reset function in Windows 10.

    The attempted repairs:

    Check speaker functionality. 
    I plugged an old Zen player into my speakers. They worked fine. Check your speaker volume to make sure they're turned up

    Right-click the speaker icon in the lower right corner.

    Restore to an Earlier Date (Courtesy of How-To Geek)
    http://www.howtogeek.com/howto/windows-vista/using-windows-vista-system-restore/
    This is probably the simplest fix and there's lots of info out there on how to restore but my restore points only went back for a week or so and none of the ones that worked ( some did some didn't) fixed the problem. 

    Sometime around this point, I bought a creative Sound Blaster sound card and installed it. It made no difference. No sound. I still Failed to play test Tone. To say I was annoyed doesn't begin to describe it. I started seriously thinking about buying Apple (shudder!). I probably had close to 40 hours of digging and failed attempts at this point but I started digging harder and found something I previously didn't know about, SFC /scannow (System file Checker). 

    sfc /scannow and SFC /Scannow error: Windows Resource Protection could not perform the requested operation.
    I don't remember (and now can't find) the posting that referenced this but another possibility was that I had corrupt Windows files (probably due to a virus). Since I'm meticulous about running an essentially enterprise level of security, this particular problem wasn't one that I gave any credit to until I was forced to. There are dozens of convoluted ways to run SFC /scannow but this was the most straightforward.
    Hit Start/All programs/Accessories
    Right-click on Command Prompt listing
    Select Run As Administrator
    Select Yes to allow.
    The command prompt screen will open.
    Type in sfc /scannow and enter.
    Exit command prompt screen by X-ing out when SFC completes. This can take some time.
    N.B. There are numerous postings that SFC often will not make all the repairs on the first pass and can benefit from repeated passes. Five repeats are usually mentioned as a maximum. In my particular case, SFC consistently advanced to 1% and gave me the error Windows Resource Protection could not perform the requested operation. An option that didn't work for me was to try running SFC in F8 Safe Mode. That gave me the identical error. At this point I was reasonably sure I had errors in my Windows system files but couldn't be sure since scan now would not run to completion.

    sfc /verifyonly
    Since scannow would not run/complete, I wanted to verify I had errors in my system files. By running sfc /verifyonly (using the same process described for scannow), I could obtain a list of corrupt files. Verifyonly will not make repairs but only indicate where files have integrity violations The list verified that I had corrupt files and so I began trying different methods to repair those files. I have over 100 GB of programs on my SSD and a clean install and subsequent software rebuild would take days and was/is essentially unfeasible.

    In reality, a list of the specific files The CBS.log Is only useful if you intend to replace the specific files from a secure copy of Windows. I instead opted to try a repair install from my upgrade Windows 7 installation disk. In short, repair install made no difference and did not fix the sound problem, repair the corrupt files or change the SFC /scannow failure. An In Place Upgrade was not an option available on my upgrade disk. I think it's only an option on the retail disc.

    Image restore
    I sporadically make a disk image with Acronis on a more or less monthly basis. Rather than attempt a clean install of Windows, I opted to restore an image from several months (Late August) prior. I began having problems in early October and wasn't sure whether my October image was clean. The good news is that I could restore only the image from the C drive (programs on SSD), leaving my data on my D drive intact. It took 20 min. for the restore to process and about 2 hours to reinstall all updates to everything from virus software to Windows to utilities. I had to remember to enable my BIOS onboard audio setting but after that. the sound and microphone all began working. SFC /scan out runs to 100%completion. My sound works now and my system is clean. 

    Moral of the story.
    I've compiled most of the fast and easy fixes I could find here. If they work for you great. This post should save you time trying to find them yourself. Making regular images and restoring should be an option that will save you time and aggravation. Windows "restore to an earlier date" is useful only in the short-term since Restore Points are often unfunctional beyond a few days. Good Luck!
