This isn't a corruption in the folders themselves or the messages they contain, but in the database index that tries to keep track of which messages are where, along with information about their status (read/unread, replied to etc.). From the path you quote, it seems that folders have been repeatedly imported from the same place as they were in originally, for whatever reason. A complicating factor in your case is that the path to these recovered/imported message files is getting longer and longer; eventually, Windows will say stop - very long file paths can't be resolved.

It looks as if many of those who revert to Windows 7 from Windows 10 experience a corruption of the store folder (the folder where messages, calendars and account settings are stored). Depending on how complicated your store folder is (many accounts, many calendars, many messages in many folders), it may be simplest just to create a new store folder and repopulate it:
 

  1. While viewing your message list in Windows Live Mail, press Ctrl-Shift-O for Options. On the Advanced tab, click Maintenance and then Store folder. Copy the path shown for the location of the store folder.
     
  2. Close down Windows Live Mail and wait a few minutes (the program will continue to run in the background while it does its housekeeping).
     
  3. Press Windows key + E to open an Explorer window, then paste or type the path you copied at (1) into the address bar and press Enter. This should open your store folder.
     
  4. Right-click on the folder name in the navigation pane and select  
  5. A repair should check that all the program files are present and correct and properly registered where necessary. To do this, close all running programs, then press Windows key + R, type Enter. In the window that opens, select Repair all Windows Essentials programs:

     

    When it's finished, reboot the computer.
      
  6. Launch Windows Live Mail and wait until the program settles down before doing anything. Then press Ctrl-Shift-T to start the Add your email accounts wizard and follow the prompts to add your email accounts again.
               
  7. Click once on Working online on the status bar and disconnect from the Internet to prevent interference from email downloads, then use File > Import messages (Alt-F I) and point the wizard at the Live Mail Backup folder. Don't interfere while the import is going on.
     
  8. Leave the program alone for some time (an hour or two, or even overnight) to allow it to re-index all your messages, then close it down and wait again for half an hour for the program to do its housekeeping.

I need detailed instructions to get the folders and their respective emails into the Windows Live Mail program under Storage Folders.

Please understand that Windows Live Mail is only creating a specific view of the messages in your store folder. There is only one set of folders and files on your hard disk; you see one view of them in Windows Explorer and a different view of the same files in Windows Live Mail. The one in Windows Live Mail is currently inaccurate, most probably because of the long filepath problem.

Please don't be tempted to move files around in Windows Explorer. Any move you make will be invisible to the database index, which can only act on changes made within the program. This will only make matters worse.

Disconnect from the Internet again, and disable your anti-virus unless you're using Microsoft Security Essentials (there's no danger while you're not connected). In Windows Live Mail, press Ctrl-Y to reveal what should be the entire folder tree. Can you see a Recipes folder anywhere? If so, select it so that it's highlighted in the folder pane. Now click and hold that folder and drag it upwards, then drop it on the topmost Storage folders heading. If there are too many folders below Storage folders for you to be able to see the topmost heading, while keeping the mouse button pressed point at the little upwards pointing triangle at the top of the folder pane's scrollbar. The pointer will probably change shape, but the folder list should scroll while you hover over the triangle. When you see the Storage folders heading immediately below the outbox, point at it and release the mouse button. Now don't do anything else for a long time while the program attempts to adjust to the change.

Let us know how you get on. If this doesn't bring the messages to light after the program has finished indexing, we'll try a completely different tack.

linkysys say:

Solution: inadvertent resizing of text and icons using touchpad on older laptops after upgrading to Windows 10

My upgrade from Windows 7 to Windows 10 went smoothly except for one persistent annoyance. When highlighting text or desktop icons with my touchpad, it frequently resizes the page arbitrarily. This is actually the "pinch-zoom" feature which has been optimized for touch screen devices. If, like me, you find this feature problematic on an older laptop, here is the solution.

  1. Click the Windows Icon in the lower left->Settings->Devices->Mouse & touchpad.
  2. Under Related Settings, click 'Additional mouse options'
  3. Click 'Settings'
  4. You should now see a menu for your specific touchpad device. Scroll through the settings and look for "Pinch Zoom" and disable it. For Synaptics ClickPad you will find "Pinch Zoom" listed under 'Scrolling.' After clicking Pinch Zoom, uncheck the box 'Enable Pinch Zoom'

vibroplex say:

Well, back to basics;  I'd start by clearing the system cache (it's just updates plus other temporary files, nothing that's at all important to anyone) by:

1) Tap the silver Xbox Guide button on the controller, go right 2 clicks to Settings, pick System Settings.

2) Select Storage (or Memory).

3) Move over any hard drive or memory unit (not the Cloud) & press (Y) on your controller & pick Clear System Cache.

4) When asked to confirm storage device maintenance, choose Yes.

Some NAT problems are subtle and don't appear in the Test Xbox Live Connection test. Most likely there's a port blocked... ports can be blocked in your router, modem, gateway or at your ISP. Since the Xbox 360 uses different ports than PCs & Macs use, you can't see the effect of a port blockage except on the Xbox.

 

A sequential power cycle is the first step in clearing a port blockage:

1) turn off the Xbox then

2) unplug the power from the WN2000RPT  

3) unplug the power from the NVG510.

4) Wait at least 30 secs then replug the NVG510 power.

5) Wait 'til the light on the front of the NVG510 which says ONLINE, READY, CABLE or DSL is on solid (not flashing), then plug in the WN2000RPT .

6) Wait until the WiFi light on the WN2000RPT to turn on the Xbox.

Some reasons for not being able to create a recovery drive in Windows 10, 8 or 7

You can create an either USB flash drive or optical disc based recovery drive on a Windows platform, which lets you easily access a failing Windows system that won't boot. 1_ The option 'back up system files to the recovery drive' requires at least 16GB USB flash drive.
2_ USB flash drive is with corruption problem that Windows can't format everything on the drive so as to create a recovery drive.
3_ Windows file system has corrupted.

4 highly recommended solutions for can't create recovery drive in Windows 10, 8 or 7
