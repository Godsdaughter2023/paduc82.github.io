Windows 10 Photos app generates thumbnails in - Microsoft Community

3lectr0 say:

Windows 10 Photos app generates thumbnails in %LocalAppData%\Microsoft\Windows\Explorer\thumbcache_*.db

Hello,

it is well known that thumbnail generation in Windows Explorer can be turned off by enabling:

"always show icons, never thumbnails".

This way no thumbnails will be stored in files like:

%LocalAppData%\Microsoft\Windows\Explorer\thumbcache_*.db

However, it seems that the Photos app in Windows 10 uses the same thumbcache_*.db files

to store its own thumbnails. And it does not even care about the Explorer setting above,

so it even generates thumbnails when the above setting is enabled!

This is a huge privacy issue as nobody expects that a Universal Windows Platforms (UWP) app like

the Photos app generates thumbnails in a legacy Explorer folder (rather than using a folder in

C:\Users\SomeUser\AppData\Local\Packages\* as for example the Contacts app does).

IMHO Microsoft should at least make this behavior visible and allow to turn thumbnail

generation off in the Photos app settings.

Best regards
