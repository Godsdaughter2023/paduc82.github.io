a_user_here say:

From my searching on it, it sounds like you are doing the right thing, but keep in mind why the feature is there in the first place.  It is there to switch to Hibernate mode if there is a chance of your system losing the current state (in RAM) because the battery is run down to zero.  If you put your computer to sleep it will draw a lot less power, but it will draw power.  And the current state of everything is stored in memory.  Meaning depending on the state of charge on your batteries, and how long your battery can supply it, sooner or later it, the battery will be drained and the system will lose what is in memory when it powers down.

Hibernate on the other hand saves the current state to disk, and restores it from disk at the cost of a bit slower startup time (but if you have an SSD for the system drive this can actually be very fast).  So not only will it not draw your power down to nothing, even if you were to do something like take out the battery the current state would be OK.

So that is a long winded way of saying make sure that you have not set "Hibernate after" to a value greater than how long your computer can supply power in sleep mode.

a_user_here say:

The option to wake from sleep so that it can hibernate has two settings.  One for when on batteries and one for when on the power outlet.  The default setting for the one when on power outlet is Never.  So in fact if it is set that way and you are using a power outlet then your computer shouldn't have woken up for the "Wake Source: S4 Doze to Hibernate" that it listed.  And you shouldn't even have changed the default 180 minutes one, because it wasn't for this purpose.

If you want to turn it back on it is just:

powercfg /HIBERNATE ON
