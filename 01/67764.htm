saltgrass say:

I am not sure what you are wanting to do but if you want to move or create additional backup images you can rename the WindowsImageBackup folder.  That way, if there is room on the drive, another one can be created or that one moved to another location.  But it must be put back to the original name for the image recovery utility to find it.

Keep in mind if the image is on a local drive it must be in the root directory of a partition.  If you move it to a Network location in can be in other places.

The version of the recovery/install media can also be very important in recovering images.  Exactly what all the build number changes does to this process I am not sure, but it you can find an image you know is there, it may be the wrong version to what you have installed.

From Win 10 you can mount one of the partition in the image.  Go to the partition the size of the c drive and right click and select Mount.  It will give an error but ignore it.  Then open Disk Management and assign a drive letter and you should be able to access the partition.   A word of warning, if you want to close the image go to the left end of the graphical view and select dismount or whatever word is used to do that.

Most newer systems have USB drivers built into the Bios.  On some, certain USB ports are disabled during boot to help speed that process.  You might check to see if that could be the case on your system.  I have an external Seagate Backup Plus and on earlier versions of the OS I had to go through the restore process until it had already looked for an image.  After that I plugged in the external drive and searched again and the image was found.  There was some type of problem with the system booting and that particular drive plugged in.
