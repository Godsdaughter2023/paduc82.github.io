What is the difference between %TEMP% folder and the TEMP - Microsoft Community

arushpcguy say:

What is the difference between %TEMP% folder and the TEMP folder in C:\Windows\TEMP ?

Hi, I just want to know the difference between the %TEMP% folder and the TEMP folder in C:\Windows\TEMP. What's the difference??

Original TItle:Windows Temporary Folders

a. user say:

Hi,

Thank you for posting your query on Microsoft Community.

I understand that you would like to know the difference between C:\ temp and %temp%. We will assist you.

Windows Temporary files are created by the operating system during the normal course of its running when there may not be enough memory allocated for the task.

Software which use large amounts of data like Graphics, Video or Media editing software also create temporary files.

Temporary files are also created for back up purposes by its respective programs. These temp files are left behind even after the task is over which leads to disk space usage.

The Temporary Files in Windows are typically found located in two locations:

  • %systemdrive%\Windows\Temp
  • %userprofile%\AppData\Local\Temp

The Temporary files created by the Windows operating system are usually stored in the %system%\Windows\Temp folderand this is where the system itself will put temporary files during installations. 

The temporary files created by the User when running any software is stored in his user profile at %userprofiles%\AppData\Local\. The other folder typically located at C:\Users\username\AppData\Local\Temp, is created for each User. This folder is hidden and you will have to first ‘un-hide’ System folders from the Folder Options.

Hope the information helps. Let us know if you need further assistance. We will be happy to help.

Try this:

If you are using your Windows 7 installation DVD to perform a System Recovery follow these instructions.

1) If you haven’t done so, change the BIOS so that the DVD drive is the first boot device.

2) When the error message is displayed, insert your original “Retail” Windows 7 installation DVD and press the ‘Enter’ key or just press the computer’s reset button to reboot your PC.

3) When prompted to “Choose an operating system to start, or press TAB to select a tool:” select the “Windows Setup” option and press the ‘Enter’ key.  You should now see a “Windows is loading files…” message, followed by the “Starting Windows” display.

4) After Windows starts (all you may initially see is a blue screen with only a mouse cursor and it may takes few minutes for the next screen to appear). Now verify or if necessary select the correct Language, Time and currency format and Keyboard input method are correct and click the ‘Next’ button.

Note: When using the Windows7 installation DVD, at the 'Install Now' prompt:
Do not pick the “Install Now” option; choose the 'Repair your computer' option located in the lower left of the display.

5) At the “System Recovery Options” window note that in the list of ‘Operating System’ column will be blank/empty. Choose the top option “Use recovery tools that can help fix problems starting Windows” and click ‘Next’.

6) Choose a recovery tool: Click the ‘Command Prompt’ option to open a command prompt window.

Note: For the Windows 7 Installation DVD the command prompt will be X:\Sources>

Enter the following the ‘Diskpart’ commands in the order listed below:

X:\Sources> Diskpart  
(Press ‘Enter’ and wait for the ‘DISKPART’ command line utility to start).

DISKPART>  list disk
(This should list the one and only remaining hard disk. Typically Disk 0)

DISKPART>  select disk 0
(Sets the focus to Disk 0)

DISKPART>  detail disk
(This will list the volumes/partitions on the drive. If you formatted the drive with more than 1 partition, typically “Volume 1” contains the Windows operating system. If you labeled the partitions with meaningful names the Windows partition should be easy to spot either by label name or partition size.)
Assuming volume #1 contains the Windows Operating System; execute the next command as shown below.

DISKPART>  select volume 1
(Sets the focus to Volume 1)

DISKPART>  active
(Sets volume 1 as the ‘Active’ partition)
This is a required step since the original drive which you removed, replaced or reformatted originally contained both the hidden ‘System’ partition and the Windows 7 partition was marked ‘Active’.

DISKPART>  exit
(Returns you back to the command prompt X:\Sources> )

At the command prompt enter the following 3 ‘bootrec.exe’ commands in the following order:

bootrec.exe  /fixmbr

bootrec.exe  /fixboot 

bootrec.exe  /RebuildBcd

Note: RebuildBcd will scan your hard drive for existing Windows installations and may take more than a few seconds.

If a Windows 7 installation is found you will be prompted to: “Add installation to boot list?”.
Type the letter Y (Yes) and press the ‘Enter’ key.

Close the command prompt window by typing:  exit.

7) Choose a recovery tool: Click the ‘Startup Repair’ option.
This may take some time, so wait until the ‘Searching’ and ‘Attempting Repairs’ process completes, at which point you will see two messages “Windows cannot repair this computer automatically” and in a separate and smaller window “Startup Repair cannot repair this computer automatically”.

These messages are the indirect result of the radical changes made by the removal or reformatting of your original/first Windows 7 drive. Ignore the “Send information about this problem” and “Don’t send” options.

Close the smaller message window by clicking on the X in the upper right corner of the Send/Don’t send options.

8) Next click on the ‘View advanced options for system recovery and support’ option.

From the main menu once again click ‘Startup Repair’. This time the repair process may only a few seconds, again ignore the “Send information about this problem”/ “Don’t send” options and close the window by clicking on the X in the upper right corner.
9) Click the ‘Finish’ button and ‘Restart’ button to reboot your computer.

Note: You will again boot and load Windows from the Windows 7 installation DVD. (Don’t forget to press any key on the keyboard to boot from the DVD).

After your computer reboots, the next pass at repairing your operating system (step #10) will backup the BCD store created in step 6 above and add the necessary entries in the BCD store required to finally boot to Windows.

10) At the ‘Language, Time and Keyboard’ prompt (this may take a while to appear) click ‘Next’.

Choose the 'Repair your computer' option located in the lower left of the display.

The System Recovery disc will start a scan ending with two messages “Windows found problems with your computer’s start options.” and “Do you want to apply repairs and restart your computer?”.

11) Click the ‘Repair and restart’ button and reboot once again to the Windows 7 System Repair environment. Don’t forget to press any key on the keyboard to boot from the DVD.

12) Again verify the Language, Time and currency format and Keyboard input method are correct and click the ‘Next’ button. In the next window after the search completes, select/click the ‘Repair you computer’ option.

13) A scan will begin and when the ‘System Recovery Options’ screen appears.

Verify that the “Use recovery tools that can help fix problems starting Windows” located in the upper left is selected.

Note that on your former Dual Boot system you should now see your only remaining version of Windows listed in the ‘Operating System’ column.

Click the ‘Next’ button.

14) Finally from the “Choose a recovery tool” options list select/click the ‘Startup Repair’ option. When Startup Repair prompts you to “Restart your computer to complete the repairs.”, click ‘Finish’ and let the computer reboot normally (do not boot from the repair DVD).

You should now see Windows 7 boot and load, enter your account password (if you created one) when prompted.

FileHistory is an good way to maintain a local backup (and helps maintain folder revision history), its integrated in Win 8/10:

Make sure your FileHistory backup is up-to-date (right click start, control panel, FileHistory-Turn it On). (it requires a usb external harddrive (Windows 8/10 will, automatically, create the FileHistory folder on the usb drive, when FileHistory is turned On);  or network harddrive can be used (make a 'FileHistory' folder on the Network Harddrive to connect the app to).... To make sure you don't run out of room on your external storage-- in FileHistory (go to Advanced Setting (on left of the FileHistory window) change the 'Keep saved versions:' to 'Until space is needed'

and/or See:

https://support.microsoft.com/en-us/help/17143/windows-10-back-up-your-files

Libraries are backed up in FileHistory-

If you keep things in non-standard places, make new Libraries or add the Folders to one of the Default Libraries

(To make a new Library--Right Click on Libraries- New-> Library (name it).  To add a folders to a Default Library or your custom Library - right clicking the folder you want to add to your library (choose'Include in Library')).

Check to see where quicken , printer scan-to folders, or any other user installed software is saving data.  (You can backup quicken/intuit program data to a OneDrive folder, but do not let database files (i.e quicken/quickbooks) reside in OneDrive;  although, they can be included in Libraries (and probably already are), they will not backup when open.  (some (other) databases can not be restored easily from raw data alone).

see:

https://support.microsoft.com/en-us/instantanswers/3fea0c19-0ae3-4c9c-a737-b228a06f8b57/show-libraries-in-file-explorer
