Programs often load their frame beneath all other open frames. - Microsoft Community

ryoga303 say:

Programs often load their frame beneath all other open frames.

So that I ensure we are communicating with the same understanding, I will define some terms before I pose my question:

  • Frame:  The bounded window within which a program displays its interactive user interface.  For example, when you run Windows Explorer, it opens a new frame onto your desktop which constitutes the user interface via which users interact with the Windows Explorer session.
  • Program Session:  A running instance of a program.  Some programs permit multiple sessions to run simultaneously (like Windows Explorer) while other programs do not under normal circumstances (like Windows Media Center).
  • Z-Order:  The visible stacking order of frames on the Windows desktop.  The stacking of frames can be altered by interactively selecting them (like clicking into some visible portion of the frame, selecting a frame from the Windows Task Bar, or using Alt+Tab to select a frame).
  • Background Frame:  Any frame which is either wholly or partially obscured by other frames that are above it in the z-order.
  • Foreground Frame:  Any frame which is not obscured by any other frames (except top-most frames).
  • Top-Most Frame:  Any frame for which its placement on the z-order is enforced, ensuring that the frame can never be obscured by any other frames that are not also top-most frames.
