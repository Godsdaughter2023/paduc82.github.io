charlie_wright say:

I had a similar issue.  After trying Windows 10 for a week I decided to go back to a previously saved Windows-7 image.  But I too was "stuck" with Windows Update telling me it was ready for the Windows 10 upgrade.  I could NOT hide the Windows 10 upgrade within Windows Update because Windows Update apparently thought I was already in the middle of an upgrade and insisted on a restart to continue.  But restarts had no effect one way or the other.

For anyone who might be interested, here's what I did.  Note that I'm running Windows 7.  If you're trying this from Windows 8.1 then your steps may vary somewhat.

WARNING: Edit the registry at your own risk.  I make no claim that the registry edits I specify below are "recommended," "correct," or "appropriate."  Only that they worked for me.

  1. I used regedit to delete the entire key:  HKLM/Software/Microsoft/Windows/CurrentVersion/WindowsUpdate/OSUpdate
  2. Restarted
  3. I don't remember, but believe that Windows Update continued to indicate that a reboot was required to install pending updates.
  4. Configured Windows Update to allow me to choose which updates would be installed rather than auto-install.
  5. Selected GWX (Get Windows 10) in the system tray and checked status of upgrade: Rather than downloaded and ready to install - where I had been stuck for days - Get Windows 10 now reported "Upgrade Reserved" but indicated no progress regarding download.  Finally!  At least _some_ progress.
  6. Used Get Windows 10 to cancel my Windows 10 reservation (GWX vanished from system tray about 5 minutes thereafter?)
  7. Used Control Panel > Programs and Features > View Installed Updates to uninstall KB3035583 (Get Windows 10), then restarted the PC
  8. Windows Update indicated: Restart to install important updates.  Restarted as requested, but the "Restart to install important updates" remained after restart.  So this restart may be unnecessary.
  9. Used regedit to examine HKLM/Software/Microsoft/Windows/CurrentVersion/WindowsUpadate/AutoUpdate
  10. Set OSUpgradePendingReboot to 0 (believe it had been = either 2 or 4, but don't remember for sure)
  11. Restarted.  Windows Update now says: Check for Updates.  Did so.  Windows Update announced "Restart to install important updates." 
  12. Among the updates was KB3035583 (which would re-install "Get Windows 10" and would start this whole nightmare over again)  Right-clicked and Hid that update.  Selected the two remaining updates.  Configured WU to auto-download and auto-install all important updates. 
  13. Installed them and Restarted one last time.
  14. All has been "normal" since then!  Windows 7 is up-to-date and Windows Update is working as expected.  And no more periodic nag from "Get Windows 10" to reserve your update

Good Luck!

a. user say:

Without a recovery disk and without having a fall-back account, this is going to be painful. The process to reset your password is involved and requires some careful typing.

Ask a computer-savvy friend to assist you if you lack experience in this area.
 1. Set your BIOS to use the DVD drive as the primary boot device.
 2. Boot with a Windows Repair CD. 
 3. Press a key when the message indicating "Press any key to boot from CD or DVD …" appears.
 4. Select a language, a time and currency, and a keyboard or input method, and then click Next. You are now in Repair Mode.
 5. Select an Operating System, then click Next.
 6. Select the Command Prompt.
 7. Identify the correct drive letter for your Windows installation. It could be a drive other than C: but it won't be X:. You do it by typing the commands
    dir C:\Win* {Enter}
    dir D:\Win* {Enter}
    dir E:\Win* {Enter}
    etc.
    until you find the Windows folder.
    I will call that drive "Q:".
 8. Type these commands:
    cd /d Q:\windows\System32 {Enter}
    move  sethc.exe  \  {Enter}
    copy  cmd.exe  sethc.exe  {Enter}
 9. Reboot normally but hold down the Shift key permanently.
10. When challenged for a password, tape the Shift key five times. You will see a small black screen.
11. Type these commands:
    net user  {Enter}
    (To see a list of all accounts that exist on your machine)
    net user Administrator {Enter}
    (To see if the Administrator account is active)
12. Type these commands:
    net user "xxx" ppp  {Enter}
    (Replace xxx with the account for which you wish to set a new password, and ppp with the new password)
    net user Administrator /active:yes {Enter}
    (To activate the Administrator account if desired)
13. Log on under this account, using the new password.
14. Reboot the machine into Repair Mode as per Steps 1..6 above.
15. Type these commands:
    cd /d Q:\windows\System32 {Enter}
    copy \sethc.exe {Enter}
16. Reboot into Windows.
Instead using a Windows Repair CD in Step 2, you can use your Windows 7 Installation DVD. The steps are similar to the ones above.

When you have fixed this then do this:

Do NOT use Windows 7 with only ONE user account. In Windows 7, the built-in Administrator account is disabled by default and although it is possible to enable it, that process may be beyond what you feel comfortable with.
WHEN, and not IF,you have problems accessing your ONE user account, you will have tremendous difficulty if you only have ONE use account.
 
You need to forget anything to do with XP, and create a User Account with Administrator privileges, (call it Admin or similar) and give it a password. Keep this account for emergency access and for elevation purposes. Then change your User account to a Standard User account.(There is absolutely no need to run as an administrator on a daily basis in Windows 7 - Windows 7 has the "right-click-run as administrator" function for when you need admin privileges). You will be much safer and better protected doing this.

I want to setup a limited user profile in Windows 7 that does the following after EVERY log-off or reboot:

  • Reset the background / theme
  • Reset desktop icons / shortcuts
  • Reset mapped network drives
  • Reset saved passwords
  • Delete stored data from last session
  • Delete newly installed applications / block application installation
  • Reset the print queue to empty - if it wasn't already
  • Reset the default printer
  • Reset Internet Explorer homepage & settings
  • Reset the screen resolution

The idea is to create a profile for the workstations in an admin office which functions very much like an internet café. They have a couple of shared workstations, a networked printer, and they store data on a file server and work on a local application server.

The current problem is that they constantly save their passwords, delete network drives and change the general configuration of the workstations (by accident), which makes it hard for the other users to work - as they do not recognize the look & feel, and now don't know where to do what.

I want to create a "shared" user profile where they can log in, and everything looks familiar. All settings are saved as defaults and no matter what you do, as soon as you log off or reboot, the settings return to default.

 

I would greatly appreciate any help.
frederik long say: Here are a couple of options:
  • Windows Steady State (http://www.microsoft.com/windows/products/winfamily/sharedaccess/default.mspx). The product is supported for Windows XP and Vista but perhaps it will work for Windows 7 too. Or maybe there is a Windows 7 equivalent somewhere.
  • Create a scheduled task that runs at boot time. It should execute this command:
    robocopy c:\Users\UserName.bak c:\Users\UserName  /s  /r:1  /w:1  /MIR  /IS /IT  /np 1> c:\Windows\temp\restore.log 2>&1
    Note this:
    - Before you start you must use the robocopy command in reverse so that it populates the folder C:\Users\UserName.bak.
    - The task must run under the System Account.
    - The task must run with the highest privileges.
    - Use the log file while testing the facility. Later on you can do without it.
    - The tool will not reset passwords. You must create a policy that prevents users from changing the password.
