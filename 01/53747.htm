As the article says, BootRec will either just try to add a Windows compatible installation into the BCD, or it will generate an entirely fresh BCD - depending on whether the BCD is in existence when the command is issued. So, method 11 in that article only works (to generate a new BCD) if the current BCD is indeed in C:\Boot (to be renamed)...

BCDedit /export C:\BCD_Backup
C:
CD boot
Attrib BCD -s -h -r
Ren C:\boot\bcd bcd.old
Bootrec /RebuildBcd

The BCDEdit & BootRec commands already know where the current BCD store is. No problem with those. But to rename/delete the current BCD, one must...

(a) Use BCDEdit & DiskPart to discover where the BCD is located.

     UEFI systems normally have it in an EFI system partition.

(b) If necessary, put a letter on the partition that holds the BCD using DiskPart.

(c) Use that letter & path in the REN command.

So... my Windows Boot Manager is located on Harddiskvolume2. That refers to partition 2 - not volume 2. The current BCD is also there. I would have to put a letter on partition 2 (or on volume 4) & use that letter & path in the rename command...

When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate.
