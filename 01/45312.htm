arminatorx say:

The Xbox One has a specially reserved storage space to keep a local copy of the cloud savegames stored online. 

That storage space is only used to keep a local copy of your savegames to speed up loading, or to store savegames temporarily when your console was not connected online. 

If you clear that save game storage, this storage is cleared. 

If the console was connected online, your savegames are already synced to the cloud, and will be re-downloaded the next time you start a game. 

If there are unsynchronized savegames on your console, because the console was offline, your savegame for that offline play will be lost.

    [image of Task Manager "not responding"]

    You had the right idea to get into Task Manager but you might have to go further into its File menu and start a  cmd.exe  window with it (elevated if desired).   "Not responding" often means that some of your manual input had to be discarded because no task was waiting to receive it.  But that doesn't mean it might not eventually receive it.   E.g. some other high priority task might be looping and stealing all the CPU cycles from the one that you want to use.   So, I would still be hopeful that you might get somewhere by pressing Alt-F  then waiting for a menu, then pressing Enter (to use the first menu item), then slowly typing cmd.exe and finally pressing Enter.  You might have to also make sure that you were still getting keyboard focus into that  Create new task  Open input field.


    Doing that would hopefully avoid the problem you apparently have with the MUF side of Windows (e.g. the corruption involving  Windows.UI.Immersive.dll).   However, you might also be able to get some clues by starting other Desktop programs in the same way, e.g. resmon.exe might work better where TM isn't and give you a clue about performance problems.   Starting cmd.exe  would make starting others easier than having to deal with Task Manager's new task dialog each time.

    job sam varkey say:

    Hi attaboys,

    Welcome to Microsoft Windows.

    The backup operation fails because Automated System Recovery (ASR) unnecessarily returns an error for noncritical disks. The expected behavior is that ASR returns an error when critical disks are not available. ASR then logs a Warning event for noncritical disks. However, ASR returns errors when both critical disks and noncritical disks are not available.

    Notes:
    Critical disks are disks that are required by the operating system, such as the disks that save the system volume and the boot volume.
    Noncritical disks are disks that are used to store user data or application data.

    I recommend you to try restoring the backup data in safe mode. To start the computer in safe mode, try the following steps.

    1. Remove all floppy disks, CDs, and DVDs from your computer, and then restart your computer.
    Click the Start button, click the arrow next to the Shut Down button, and then click Restart.
    2. Do one of the following:
    • If your computer has a single operating system installed, press and hold the F8 key as your computer restarts. You need to press F8 before the Windows logo appears. If the Windows logo appears, you'll need to try again by waiting until the Windows logon prompt appears, and then shutting down and restarting your computer.
    • If your computer has more than one operating system, use the arrow keys to highlight the operating system you want to start in safe mode, and then press F8.
    3. On the Advanced Boot Options screen, use the arrow keys to highlight the safe mode option you want, and then press Enter.
    4. Log on to your computer with a user account that has administrator rights.
    When your computer is in safe mode, you'll see the words Safe Mode in the corners of your monitor. To exit safe mode, restart your computer and let Windows start normally.

    For more information on safe mode, you may visit the link mentioned below:
    Start your computer in safe mode
    http://windows.microsoft.com/en-US/windows7/Start-your-computer-in-safe-mode

    If this doesn’t help, I will suggest you to follow the steps mentioned below and check if you come across the same problem:

    1. Open the Local Group Policy Editor.
    2. In the left pane, click on to expand User Configuration, Administrative Templates, Windows Components, Backup, and Client.
    3. In the right pane, right click on Turn off the ability to back up date files and click on Edit.
    4. To Enable Backup Data Files -
     Select (dot) either Not Configured or Disabled.
    5. Click on OK.
    6. Close the Local Group Policy Editor window.

    Hope this information is helpful.
