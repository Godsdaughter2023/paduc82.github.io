¡firedog say:

The result at the receiving end depends also on the Content-Transfer-Encoding. If it's Quoted-Printable or Base 64, you should get the result you want. If you use None (AKA 7bit) or Uuencode, then lines will be wrapped. QP is more economical, and for a plain text message, all it does is insert '=' as End-Of-Line markers if there are no non-ASCII characters. The EOL markers should be stripped out in the receiving client. B64 uses quite a bit more space and can occasionally have unintended consequences (e.g. Google Groups doesn't [or didn't] index B64-encoded posts).

You can't change the 132-character upper limit in the UI, but you can in the registry. However, SMTP doesn't allow lines longer than 1000 characters, so the maximum should be set to, say, 990 (0x3DE), to allow for quote characters at the beginning of the line and CR-LF at the end.
