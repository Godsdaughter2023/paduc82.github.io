saltgrass say:

The process has certain rules it follows which checks locations.  I have seen a File backup complain if a destination is not reliable (always available) or if the backup is on a location which will be backed up.  System Image backups may not work but that location should not be offered if the process does not feel it is adequate.

So, what path are you seeing when the system offers you a destination for the WindowsImageBackup folder?  If it is a Network location, is it on your local network and secure.  Maybe the location is not accessible if it was being accessed during an actual restore situation.

Yesterday, I got a new NVMe drive and made a System Image of the original drive, replaced the drive and restored the image and it worked great.  I had an external drive and not a network location. 

If I were going to use a network location, because of the speed of the process is limited over a network, I would make the image on a local drive and then move it to the network location.  Restoring the image from a network location would also be slow.

I don't install programs on a partition other than the OS partition.
