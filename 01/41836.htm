I found the answer from Asus Tech support. Windows OS no longer control the operation of the touch pad. For my Asus Ultrabook. thrav say: I had the same problem and it took a lot of searching, but I finally fixed it, and wanted to document it for everyone else that runs into this issue. The two methods suggested by the support engineer did not work for me.

Start by going to this website: http://support.asus.com/Download.aspx?SLanguage=en&m=Touchpad&os=30

...and installing the first result. You want the "ElanTech Touchpad Driver". Download all that stuff to your desktop, unzip the folder, and start the setup. When setup finishes, you will be asked to restart. After restart navigate to Computer > C: > Program Files > Elantech > Then double-click on "ASUS_UI" (application) in that folder.

The first first page when you open the application is labeled general gestures and has "single click" as the first section. Uncheck the box next to "Select". There is also a two finger option called "show pop-up menu" that you can disable, but it is much harder to trigger by accident.

You'll also probably want to check the box in the 4th tab that disables the trackpad when you have an external mouse plugged in.

Good luck everyone.

rajithr say:

Hi Calvin,

Thank you for posting your query in the Microsoft Community forum>

It is not recommended to run Disk Optimization on Solid State Drives(SSDs). It isn't particularly helpful as it can shorten the life of the drive, it also does not result in an improvement of it's performance.

Disk optimization means arranging data (of a storage device) in a manner, which enhances the performance of the device. Optimization places all the free  space of a storage device at one place. It works by joining all the lumps of  memory space

Defragmentation is the process of reducing  the fragmentation of a storage device.

This behavior is by design.

To work around this issue, use one of the following methods.

Method 1

Use the IP address of the remote server when you try to connect to the network share.

Method 2

Create a different Domain Name System (DNS) alias for the remote server, and then use this alias to connect to the network share.

After you use one of these methods, you can use different user credentials to connect to the network share. In this situation, the computer behaves as if it is connecting to a different server.
