You could try to understand the problem better by using PowerCfg to check just what power plan settings the system actually has set on it [as opposed to what power plan settings you have told the user interface to set on it].  In an elevated command prompt, enter

PowerCfg -qh >%UserProfile%\Desktop\CurrentPowerPlanDetailedSettings.txt

I paste the output into Excel to look through the very long output file. 

It might look daunting at first but is just a large number of repetitions of a simple pattern -

    Power Setting GUID: 245d8541-3943-4422-b025-13a784f679b7  (Power plan type)
      Possible Setting Index: 000
      Possible Setting Friendly Name: Power saver
      Possible Setting Index: 001
      Possible Setting Friendly Name: High performance
      Possible Setting Index: 002
      Possible Setting Friendly Name: Balanced
    Current AC Power Setting Index: 0x00000002
    Current DC Power Setting Index: 0x00000002

The initial GUID entries for each group are the reference number[s] for that item

The last two lines tell you what the settings are for AC & DC. 

The intervening lines tell you what units are used for the setting and, where relevant, what the range of possible settings is.  I chose a simple example above but the units could be %, seconds, Yes/No or many other things.
