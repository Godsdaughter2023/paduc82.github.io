cmdrkeene say:

Sub-Pixel positioning, required for cleartype, is not possible with the Remote Desktop Protocol.  Because of this, it is turned off when viewing a remote desktop.  Turning it on may cause an appearance change (probably more of a blurring or "bolding"), but is not the true ClearType.  Windows is hard-coded by design to disable ClearType anytime a remote client connects.
