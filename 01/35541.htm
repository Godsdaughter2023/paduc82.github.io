wunders say:

This is not a problem, it is a "feature".

 

Microsoft has designed it such that after any new user is created, the original administrator account then becomes hidden.  Windows will not delete the original administrator account, so you can be assured that it is still there.  If you desire to log in as the original administrator, press Alt-Ctrl-Del twice at the logon screen.  This will bring up a classic user/password login screen.  Simply type "Administrator" as the username (or whatever the original administrator name is) and then enter the appropriate password or leave password blank if no password.  This should allow you to log in as administrator.

 

You might also find that the administrator user shows up if you start in "Safe Mode" (repeatedly tap the F8 key while your system boots up and then select a Safe Mode option).

 

If you are trying to transfer the Administrator files and settings over to one of the more recently-added users, you can use the following procedure to do this.  "Settings" may not transfer, though as they are contained in the user registry and each user has a unique registry.

"How to copy data from a corrupt user profile to a new profile in Windows XP"

  < http://support.microsoft.com/kb/811151/en-us >

... just pretend that the original administor account is corrupted.
