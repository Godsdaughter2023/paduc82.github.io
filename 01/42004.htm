2. The Network Service account does not have the correct permissions.

Remote Activation

a. user say:

Regardless of how much you install, the maximum RAM that can be used by a 32-bit operating system is 4 GB. (The exact amount is 2 raised to the 32nd power.)

However, some of that is reserved for use by your hardware, which typically leaves about 3.2 or 3.4 GB available for use by the software. Some of that will be used by the operating system and by software you have running in the background. So, an amount of  2.47 GB available to you (an amount which can change from minute to minute) doesn't seem unreasonable.

To make more of that RAM available to you, exit software running in the background if you don't need to have it running.

The 64-bit version of an operating system can use a ridiculously high amount of RAM - 2 raised to the 64th power. For that, you need 64-bit hardware. I presume here that your computer has 32-bit hardware, otherwise why would 32-bit Windows be installed.

I tested this facility a little while ago because I thought it was a great idea. I am no longer so sure. Here is why:

  • You end up with two sets of user folders: One on drive C:, one on drive D:.
  • Your new files are indeed stored on drive D: but your library locations are not updated, which should answers your question.
  • The Windows backup tool ("File History") only backs up files that are included in Library folders. This means that it skips your user folders on drive D:.

Maybe I missed something here. If I did then I'd love to hear how to make this facility work reliably.
