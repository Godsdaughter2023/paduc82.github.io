steve winograd say:

The problem is that the router has assigned a Default Gateway address to the Local Area Connection.

The solution is to manually assign the IP address and subnet mask to the Local Area Connection.  Choose an address in the 192.168.0.x range, but outside of the router's DHCP address range.  For example, if the router's DHCP server assigns addresses 192.168.0.100 through 192.168.0.199, assign 192.168.0.200.
