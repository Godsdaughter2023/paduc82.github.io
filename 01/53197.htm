wungraileng vashi say:

Hi,

In response to your query, please be informed that it is not recommended to use a USB drive as a recovery drive as the size of the System Image could be bigger than the USB drive's available memory. Also, since the drive size is usually small, it tends to fill up quickly, especially when creating the System Images as it is scheduled to occur at a particular time.

However, I suggest you to try the steps below and check if the issue persists.

1. Create a directory/folder on your USB drive.

2. Right-click the directory and choose "Share with", "Specific people".
3. Choose your account from the list and click Share and Done.

4. In the "Create System Image" menu choose "On a network location".

5. Enter the URL/folder path of the shared folder in the USB drive for your share (For eg: This D:\sharedfolder)

6. Enter your username and password and click OK

Let us know the result, we will be glad to help if you need further assistance.

    a. user say:

    I did some research on this process and it says its a windows login manager but i don't know if thats real or not.

    Hello Jamessyy – Welcome to Microsoft Answers Community.

    Yes, it is a logon executable. However, Windows® boot-up
    after loading the kernel is this process:

    Time Flow:
    C:\NTLDR     -> C:\Windows\system32\ntoskrnl.exe -> C:\Windows\system32\smss.exe ->
    The smss.exe Starts -> C:\ Windows\system32\Csrss.exe
    The smss.exe Starts -> C:\Windows\system32\Winlogon.exe

    Then smss.exe will be waiting for Csrss.exe or Winlogon.exe to exit.
    However, none of these two exits, so smss.exe waits forever. But if
    you kill one of these two PIDs using Taskmgr.exe you’ll end-up with
    BlueScreen.  

    Most correct answer about Winlogon.exe is:
    Yes, the program is part of Windows® and, isn’t a malware or a virus.
