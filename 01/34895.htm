Tethering cell (Motorola W376 or 260) to PC (Windows - Microsoft Community

winnard say:

Tethering cell (Motorola W376 or 260) to PC (Windows 7) with USB cable for internet access while on road

Need detailed instruction on how to add Motorola Trac Phone W376g 0r W260g as modem to pc for internet access using USB cable from power on cell to USB on PC - found the following instruction on web but not detailed enough for latests drivers listed by Motorola - links provided below:

Motorola 4.2.0 Driver with MotoConnect* (for 32-bit versions of Windows®)
Motorola 4.2.0 Driver with MotoConnect* (for 64-bit versions of Windows®) 

Method A
 just switched off of a Smartphone with Alltel to a normal phone on Verizon. I chose the Motorolla Moto W755 mostly because it was free on sale. The phone has a standard mini-USB plug (yeay), so you can charge it with your machine (or any other 5 volt source). Motorola hasn't developed drivers for Windows 7 yet, but the drivers for this phone and others can be installed anyway.

1. Downloaded the latest drivers from here: http://direct.motorola.com/hellomoto...ng_drivers.asp
http://direct.motorola.com/hellomoto...ng_drivers.asp
a. Pick "Download Motorola USB Driver v3.7.0(for 32-bit versions of Windows®)" for 32-bit, or the one right underneath for 64-bit. These instructions will work for both.
2. Create a new Folder "C:\Driver"
3. Decompress the MSI-File (Start/Run CMD as Administrator):
4. In the directory of the MSI type: msiexec /a Handset_USB_Driver_x86_v3.7.0.msi /qn TARGETDIR="C:\Driver"
5. Plug in the phone. It will fail to install the drivers.
6. Open Device Manager, and choose to manually install the drivers on one of the unkown items.
7. Browse to the folder: C:\Driver\Common Files Folder\Motorola Shared\MotPCSDrivers and let it install. Do the same for the others.
8. Done! You can now install Motorola Phone Tools or whatever you want to do next.

Method B

using the Motorola W376g and have been able to charge it on a Vista machine using the before mentioned Motorola USB Driver v3.7. For some reason the Windows 7 PC doesn't recognize the same 3.7 drivers. I tinkered with the INI file and eventually got it to work. I'm sure I'm not the only one, so I thought I'd share the fix with anyone with a similar problem. Thanks.

Gungistoker


* Download & install the Motorola USB Drivers

* Search for file 'motccgp.inf' at C:\Windows\System32\DriverStore\FileRepository,
mine is in a folder named motccgp.inf_x86_neutral_181385012a0af6ec

* Open motccgp.inf with MS Notepad, scroll down to:

[Motorola]
&#xUS;BMOTCCGP% = MOTCCGP, USB\VID_22B8&PID_3804

* Copy the below code:

USB\VID_22B8&PID_4061&REV_0001
USB\VID_22B8&PID_4061

* Paste the code one line below &#xUS;BMOTCCGP% = MOTCCGP, USB\VID_22B8&PID_3804.

It should look like this:

&#xUS;BMOTCCGP% = MOTCCGP, USB\VID_22B8&PID_3804
&#xUS;BMOTCCGP% = MOTCCGP, USB\VID_22B8&PID_4061&REV_0001
&#xUS;BMOTCCGP% = MOTCCGP, USB\VID_22B8&PID_4061

* Exit & save motccgp.inf

* Open Device Manager, plug in your phone’s USB, quickly double-click the ‘Unknown’ device (mine disappears in 2 seconds)

* Click ‘Update Driver’ > Click ‘Browse my computer for driver software’> Click ‘Browse’

* Locate the folder that motccgp.inf resides in at C:\Windows\System32\DriverStore\FileRepository, click ‘Next’, the drivers should install

* Unplug your phone’s USB and then plug it back in

Note:

To attempt another Motorola model phone or a W376g that may have a different USB\VID code, try this:

* Open Device Manager, plug in your phone’s USB, quickly double-click the ‘Unknown’ device

* Click the ‘Details’ tab, under the “Property’ heading, click the menu

* Click ‘Hardware IDs, copy or write down the USB\VID codes for your phone

* Replace the USB\VID codes in the above steps with your USB\VID codes

Thanks, I'll post there. But I disagree, it is definitely something you can customize. There is no way that there is any hardware limitation that prevents you from tuning the joysticks. Any controller, be it from the original xbox, xbox 360, ps 1 etc, all have joysticks that are customizable. For a basic example:


- Let's say you have a joystick that is fixed to one axis (only goes left and right). The joystick is simply connected to a sensor that outputs an analog value e.g. between -1 and 1. So all the way left == -1, centered == 0, right == 1, somewhere between the center and right is a decimal value >0 and < 1. 

- Better illustration: ( -1 ------- -0.5 ------- 0 ------- 0.5 ------- 1 )

- When a controller sends this sensor value to the xbox, all the xbox does is apply some sort of pre-tuned profile (i.e. an equation) that translates that analog sensor value to the correspon

xxwytebladexx say:

The only way it would be hardware limitation is if this translation was somehow done in hardware via an ASIC for performance reasons, which I find extremely unlikely and illogical. 


Actually to clarify, even if there was some ASIC performing this calculation using a fixed algorithm, there is still always the option to further correct this via software. But the reason I say it isn't a hardware limitation is because:

a) Controllers are not sending sensor so fast that the xbox can't process it (no need for ASICs), neither does it need to send data that fast to get smooth motion control

b) Since applying a tuning curve to the sensor data would technically slow down the performance (bottleneck for processing is software instead of hardware), that would make the elite controllers "less performant" than standard controllers. But as mentioned in a) this probably isn't an issue.
