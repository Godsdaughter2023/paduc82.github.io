cmdrkeene say:
What specific program process is causing the high utilization?  If it always correlates to when you plug it in, it could be running as a scheduled task with A/C power as the trigger, such as a backup job or re-sync.  But check which program is causing it and we'll narrow it down.  If you don't see what's using the processor so strongly at first, try running Task Manager as admin so that it can show all processes.

caesarsr say:

Windows runs a set of scheduled tasks on a regular basis, such as scanning for malware or optimizing .NET applications.  These tasks are usually high-intensity tasks, so they're skipped or postponed while the system is on battery.  When you plug the device in, and the device is on, and has been idle a certain amount of time, these tasks will launch.  For example, in the picture you posted above, the Antimalware Service Executable is running a scan.

So, charging generates heat, and the system only performs some tasks when it's plugged in (to conserve battery).  So your heat load can get pretty high, and the back of the computer gets warm and the fan kicks in.
