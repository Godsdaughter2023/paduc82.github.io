http://support.microsoft.com/kb/950717

http://h10025.www1.hp.com/ewfrf/wc/softwareDownloadIndex?cc=uk&lc=en&dlc=en&softwareitem=pv-60484-2

    darth.cabs say:

    You can't separate Windows bootloader, what you can do is to have on GRUB two options, Linux and Windows, and when you select Windows, the bootloader of the most recent installed Windows will list all of your Windows installations. To be clearer. Imagine you have one Linux and two Windows 7. You can't have, in GRUB, three options, Linux, Windows 7, Windows 7. What you can have is 2 options in Grub, Linux and Windows, and then when you select Windows, you have two Windows 7 listed.

    Windows will always use the same bootloader to startup all the installed Windows'. This is why you need to install the older versions first and then the recent ones later.

     

    Grub:  Linux

              Windows   ->  Windows 7

                                  Windows 7

    arthlopes say:

    You can't have, in GRUB, three options, Linux, Windows 7, Windows 7. 

    Yes, you can. I just did that. I know this is 5 years late, but I had such a hard time trying to get this information on internet that I decided to share it.

    Windows 7 creates a 100Mb partition to use as MBR, and puts it's Bootloader there. When one installs a second Windows 7, it just updates the Bootloader. But you can create Bootloaders on other partitions too, so that you can boot Windows separately.

    You will need:

    - Some bootable tool to change the bootable partition. I used GParted Live-CD.

    - Windows 7 install DVD

    - Rescatux Live-CD (optional, it would be used only if something went wrong and you lost GRUB, but I strongly recommend)

    Just select one of the Windows partitions, mark it as bootable and restart, booting with the Windows 7 DVD.

    When you get to the "Install Now" screen, click on the option "Repair your computer". It will try some automatic repair, but wait and select "No". On the next screen, choose "Use recovery tools" and click "Next".

    Open the command prompt, run "bootrec.exe /scanos" and make sure both Windows appear. This command shows the Windows installs that are not on the current bootloader, so both (or more, if you have) should appear.

    If so, run "bootrec.exe /rebuildbcd". This command will create the bootloader on the boot partition and allows you to choose which installs it will manage. This is where the magic happens. So, choose only one. Write somewhere which option you chose on this partition, it may be useful later.

    After this, do all over again, but this time on the other Windows partition.

    Be warned: it's very easy to end up creating two bootloaders pointing to the same install. If this happens, redo all work on one of them, but this time choosing the other option. This is why it's good to write the chosen option before. But probably just choosing the first option on both partitions will be right.

    By the end, return to Linux (here I had to use Rescatux, but I believe it was because I used "bootrec.exe /fixmbr" and it overwrote GRUB on MBR - probably you won't need it).

    Mount the MBR partition and copy the bootmgr.exe file to the root folder of both Windows partitions.  

    After that, reboot (don't know why, but for me it only worked after I rebooted) to Linux again and use update-grub, or Grub Customizer. GRUB will now show the old "Windows 7 (loader)" option alongside the two direct Windows loaders.

    Hope this will be useful for someone.
