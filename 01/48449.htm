beefrwijker say:

IINM you used

PowerShell Get-AppxPackage -Name *Microsoft.windowscommunicationsapps* | Remove-AppxPackage

to uninstall Mail and Calendar. I think that this merely deregisters it, it's still there but you can't use it anymore as the appdata area for this app has been removed. The program folder will still be there. It must remain because this script will only deregister for this local user: You.

This is the one to reregister it for you:

PowerShell Get-AppxPackage -Name *Microsoft.windowscommunicationsapps* | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml" -Verbose}  > %homepath%\desktop\result.txt

Winkey-X, select Windows Command Prompt (admin)

PowerShell (powershell will take over the window in Admin mode)

Copy/paste:

Get-AppxPackage -Name *Microsoft.windowscommunicationsapps* | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml" -Verbose}

This will bring back Mail and Calendar app data back in your user area  i.e. %localappdata%

If you add -AllUsers it will do it for all the users. In which case Powershell needs to have admin privilege.

(Actually it needs Administrator rights because MS extra protected C:\Program Files\WindowsApps

which security I busted months ago and then forgot all about it :-( )

I think of it as the repair option that you can find in some regular Windows application Installers.
