Windows Photo Viewer - Manual Picture Rotation Compatability

Gentlemen,

If I manually rotate a portrait orientation picture 90 degrees using Windows File Explorer, Microsoft Paint.Net, or Nikon ViewNX 2 no problems with the correspondence between the Thumbnail display and the Picture display occur. The Thumbnail and Picture orientations always match.
If I manually rotate a portrait orientation picture 90 degrees using Windows Photo Viewer the Thumbnail and Picture orientation can be off by 90 degrees when this picture is viewed with any of the above mentioned programs. This unmatched Thumbnail and Picture orientation error cannot be rectified.

My conclusion is that the picture rotation method that Windows Photo Viewer uses is not compatible with the picture rotation methods that are used by Windows File Explorer, Microsoft Paint.NET, and Nikon ViewNX 2.

The picture file was from a Nikon Coolpix S6500 camera.
The OS was Windows 8 (64 bit) with a clean boot on a Samsung Series 7 Chronos PC.
All of the relevant programs were up-to-date.
The suggestions in What to do if you have problems with an app were reviewed and followed as appropriate.

Thank you, WH

Clear and Reset the Thumbnail Cache.

Windows 7: Thumbnail Cache - Clear and Reset
http://www.sevenforums.com/tutorials/10797-thumbnail-cache-clear-reset.html

occurring in the future.

Windows 7: Thumbnail Cache - Enable or Disable
http://www.sevenforums.com/tutorials/10794-thumbnail-cache-enable-disable.html

3) If you are having the same issue in Windows Photo Gallery...
you might try renaming the Photo Gallery proprietary database.

This will force Photo Gallery to create a new database the next
time it's launched.  Be aware you may lose some tags and
metadata and the gallery will have to be re-populated.

The database file is Pictures.pd6  (rename to : Pictures.OLD)

If the result is not satisfactory...delete your new .pd6 file and
rename the .OLD file back to .pd6.

The path is:

Windows 7 -
%userprofile%\AppData\Local\Microsoft\Windows Live Photo
Gallery\Pictures.pd6

This is a Hidden file so if you cannot find it...the following setting
may need to be changed.

Try the following steps in Windows 7:

Go to.....Start button / Control Panel / Appearance
and Personalization / Folder Options / View tab
Advanced settings / Show hidden files, folders, and
drives / OK

Windows 7 - Show hidden files
http://windows.microsoft.com/en-US/windows7/Show-hidden-files

williamherron say:
Subject: Windows Photo Viewer - Manual Picture Rotation - Part 2

Gentlemen,

The content of the answer link provided by Phil Reineman presented interesting information. A conclusion was presented that "some applications do not correctly update the photo rotation metadata." This is another way of stating my original question. It is not helpful to "civilian" users to discuss internal programming issues. Civilian users do not normally have access to internal programming information. Discussing internal program processes, real or inferred, is not helpful, and these processes cannot be demonstrated by civilian users.

The content of the answer links referred by Volunteer J were not helpful. The problem that I presented did not suggest that the issue was with any cache database corruption or size issue.

To clarify this issue, and perhaps to belabor the point, here is a step by step demonstration of a small part of the picture file rotation problem.

1. Using a Nikon S6500 digital camera, take a picture using a portrait orientation. The picture will appear to be correct if the picture is viewed when the picture rectangle is in the vertical orientation, long side up.
1a. For simplicity, I will call "horizontal" orientation to mean that the picture rectangle is in the horizontal orientation, short side up with a sideways picture presentation. I will call "vertical" orientation to mean that the picture rectangle is in the vertical orientation, long side up with a meaningful up and down picture presentation.
2. Remove the SD card from the camera and, using an SD Card Reader, copy the picture file to the desktop for further viewing and manipulation. The thumbnail for this picture file shows the picture in the horizontal orientation with the top of the scene to the right short side. Note that at this point the orientation of the picture image is not indicated or viewable without using a picture viewing or editing program.
2a. Open this picture file using Windows Photo Viewer. This picture image is presented with the same horizontal orientation as in the thumbnail image.
2b. Using Windows Photo Viewer, rotate the horizontally presented image once (90 degrees) to the left (CCW) so that the picture image will be presented in the vertical orientation with the top of the scene up. This is a more meaningful presentation of this image.
2c. Close Windows Photo Viewer. The thumbnail for this picture file is changed to a vertical orientation that matches how the picture image looked when it was viewed with Windows Photo Viewer.
2d. Open this rotated picture file using the Nikon ViewNX 2 program. Select the view mode "Image Viewer" where the picture image is presented above the filmstrip that shows thumbnail images for each picture. At this point, the picture image is presented in a vertical orientation and the thumbnail image is presented in a horizontal orientation. Note that is mismatch between the presentation of the picture image and the presentation of the thumbnail image cannot be fixed.
3. Still viewing pictures with Nikon ViewNX 2, select the original, not rotated picture file. Here the picture image is presented in a horizontal orientation and the thumbnail image is presented with a matching horizontal orientation.
3a. Rotate the horizontally presented image once (90 degrees) to the left (CCW) so that the picture image will be presented in the vertical orientation with the top of the scene up. The thumbnail image changes to a matching vertical orientation.
3b. Close the Nikon ViewNX 2 program. A "Confirm File Change" message is presented ("Some files have been edited. Do you want to cancel closing the application and save the changes?"). If you click on "No" the program will close. A rotation action will not be made. The thumbnail for this picture file will appear with a horizontal orientation as it was before using the Nikon ViewNX 2 program.
4. Open the Nikon ViewNX 2 program and select the original, not rotated picture file again. Here the picture image is presented in a horizontal orientation and the thumbnail image is presented with a matching horizontal orientation.
4a. Rotate the horizontally presented image once (90 degrees) to the left (CCW) so that the picture image will be presented in the vertical orientation with the top of the scene up. The thumbnail image changes to a matching vertical orientation.
4b. Close the Nikon ViewNX 2 program. A "Confirm File Change" message is presented. (the File menu, Save command is not available) If you click on "Yes" the Nikon ViewNX 2 program will remain open, the picture image will be incorrectly presented in a horizontal orientation with the top of the scene to the left. The thumbnail image changes to a matching incorrect horizontal orientation. This new orientation was not selected and is not sensible. [This is an error with the Nikon ViewNX 2 program.] Closing and then running the Nikon ViewNX 2 program returns the presentation to the correct view. The Nikon ViewNX 2 program has a few problems with presentation; however the saved file is saved in a meaningful way. [It is unfortunate that these minor problems with Nikon ViwNX 2 have confounded this discussion.]
4c.Please also note that if all that was done with the picture file was a rotation, the only change to the file will probably be a rotation, and the user can reasonably assume that this was the action that was taken. The user can choose to rotate or not to rotate. 4d. If the picture file was rotated and edited (e.g. the Brightness was changed), other things will happen and the user will become even more confused.
4e. Close the Nikon ViewNX 2 program. The thumbnail for this picture file appears with a vertical orientation which was the intended orientation.
5. Open the Nikon ViewNX 2 program and select the rotated picture image file. The rotated picture image appears with the correct vertical orientation along with a matching thumbnail image.
5a. Close the Nikon ViewNX 2 program.
6. Open the Nikon ViewNX 2 rotated picture file using the Windows Photo Viewer. The rotated picture image appears with the correct vertical orientation.

Summary:

The above examples do not present all of the picture rotation possibilities. Who knows how much trouble and confusion other rotation editing combinations can cause users?

As was stated before, Windows Photo Viewer and Nikon ViewNX 2 can produce inconsistent results with the correspondence between the picture image orientation and the thumbnail image orientation. If Nikon ViweNX 2 is used first to rotate an image, the two orientations will remain synchronized. However, if a user did not know my little trick about closing Nikon ViewNX 2 and then opening it again before proceeding with additional picture edits, the user would be significantly confused about how the Nikon ViewNX 2 program performs picture rotations. If the Windows Photo Viewer is used first to rotate an image, the two orientations will be different and cannot be changed to present matching orientations.

As a "civilian" user (non-Microsoft or non-Nikon professional) it is not appropriate for me to criticize or suggest programming decisions made by Microsoft or by Nikon. It is legitimate for me to operate programs and to note inconsistencies about how programs work. My original question asked "Can this picture file rotation compatibility issue be evaluated and resolved?" This question still stands unanswered. Clearly several parties will need to be involved with this evaluation and resolution.

Additional Discussion:

The roles of picture file Viewing programs and picture file Editing programs may be philosophically quite different. Although it is probably a popular feature, providing an editing function (i.e. picture rotation) within a file Viewing program may create problems that may be difficult to solve.

File and PC specifics:
The picture file was from a Nikon Coolpix S6500 camera.
The OS was Windows 8 (64 bit) with a clean boot on a Samsung Series 7 Chronos PC.
All of the relevant programs were up-to-date.

Thank you for sticking with this interesting issue, WH

 We do need the actual DMP file as it contains the only record of the sequence of events leading up to the crash, what drivers were loaded, and what was responsible.
You may be able to get the DMP files without crashing by booting into safe mode (F8) with networking.
If you are overclocking stop.  (chances are if you dont know if you are, you are not)
To enable us to assist you with your computer's BSOD symptoms, upload the contents of your "\Windows\Minidump" folder.

The procedure:
* Copy the contents of \Windows\Minidump to another (temporary) location somewhere on your machine.
* Zip up the copy.
* Attach the ZIP archive to your post using the "paperclip" (file attachments) button.
 *Please upload them to a file sharing service like Skydrive or"Rapidshare" and put a link to them in your reply.

To ensure minidumps are enabled:
* Go to Start, in the Search Box type: sysdm.cpl, press Enter.
* Under the Advanced tab, click on the Startup and Recovery Settings... button.
* Ensure that Automatically restart is unchecked.
* Under the Write Debugging Information header select Small memory dump (256 kB) in the dropdown box (the 256kb varies).
* Ensure that the Small Dump Directory is listed as %systemroot%\Minidump.
* OK your way out.
* Reboot if changes have been made.

Please also run MSinfo32 and upload the output as well.
To run MSinfo32 please go to start>run>MSinfo32
Go to "file" "save" and upload the saved file with the DMPS
(Instructions for XP, Vista, Win 7, Win 8)
