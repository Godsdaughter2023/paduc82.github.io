justanotheraccount say:

Well, this is a very aggravating problem, and solving it can be a major headache. I spent a good day troubleshooting and finally stumbled upon my particular solution.

In my case, I had installed Windows 7 on an old laptop that can take two SATA drives. The two drives came pre-configured as a Boot/OS disk and a secondary backup/data disk.

And now, the obscure reason Windows would only boot with a CD or USB install disk inserted--

The BIOS is hardcoded to look for the operating system bootloader only on the _SECOND_ harddisk, even though the OS always was (and is) actually installed on the _FIRST_ harddisk.  There was only an option to "Boot from Notebook Harddrive" in the BIOS and this meant to boot only from the second. It wouldn't even check the first.

SOLUTION: Use EasyBCD utility (Google it) to move the Windows 7 bootloader to the second drive.

Yes, this finally fixed it!  There's probably also a way to do this from the Windows Install disk command prompt/system repair, but I don't really know it.

Try moving the bootloader to another drive with EasyBCD if your situation was like mine--business PC, more than one drive, locked down BIOS, "operating system not found."
