Normally things look like this:

If you schedule a chkdsk with error correction (chkdsk  /r) to run on drive C on the next reboot the value gets changed to this:

If you were to reboot after that a chkdsk with error correction (chkdsk  /r) would run on volume C - because you scheduled it to do so.

To reset the value back to the default (removing the scheduled chkdsk  /r) you can enter this command from a Command Prompt window opened as Administrator:

chkntfs  /d
