brian walsh say:

I've been living with this problem for a long time on Windows 7 - I only see it when I've got my laptop connected to two external displays, not when the just using the laptop screen on it's own. 

I originally thought that it was a z order problem, but it's actually an "always on top" setting that is wrong, this thread seems to describe the problem perfectly, although I avoid using win+d, so there's another problem with the same symthoms:

https://superuser.com/questions/269415/show-desktop-sometimes-sets-a-window-to-always-on-top

so  I've been using both of the workarounds in that thread to avoid having to continuously restart. 

I have the same few applications running all the time - IE, Chrome, and Remote Desktop Connection manager (+ others, but the problem will appear with just those 3 running) . I suspect that it's some action in Chrome, IE or RDCMan that is messing up the windows.

Workaround 1: 

With the window that is shown in the foreground selected, press CTRL + ALT + ESC. This should remove the window's always on top setting and stop it from misbehaving.

this doesn't seem to work with the Skype for desktop window though.

Workaround 2: 

Sometimes a modal child dialog box gets hidden behind the main application window, typical example is a printer dialogue box. The main application can't be moved & doesn't respond, and the modal child window can't be seen. 

If you know where the child window is, then you can ALT+SPACE, then M followed by arrow keys to move the child window from behind the parent window... but this doesn't work if the main window is using the full desktop. For this,  using the UltraMon utility, I've assigned a hot key to run the following vbs from that thread: 

Set wnd = CreateObject("UltraMon.Window")

If wnd.GetForegroundWindow() = True Then
If wnd.AlwaysOnTop = True Then
wnd.AlwaysOnTop = False
Else
wnd.AlwaysOnTop = True
End If
wnd.ApplyChanges 0
End If

This will display whatever window should be in the foreground. It's far from a fix, but it's a usable workaround.

a. user say:

Hi,

The problem with folder file view automatically changing could be due to automatic folder type discovery feature on your computer.

We suggest that you try the registry-script steps provided below, it will reset the view settings of all folders and then it'll completely disable the automatic folder type discovery feature:

1. Type regedit in RUN or Start search box and press Enter. It'll open Registry Editor.
2. Now navigate to following key:

HKEY_CURRENT_USER\SOFTWARE\Classes\Local Settings\Software\Microsoft\Windows\Shell


3. Under "Shell" key, select "Bags" key and press "Delete" key. It'll ask for confirmation, click on Yes button.
4. Now right-click on "Shell" key and select "New -> Key" option. Set the name of new key as Bags
5. Now right-click on "Bags" key and select "New -> Key" option. Set the name of new key as AllFolders
6. Again right-click on "AllFolders" key and select "New -> Key" option. Now set the name of new key as Shell

So the final key path will be:

HKEY_CURRENT_USER\SOFTWARE\Classes\Local Settings\Software\Microsoft\Windows\Shell\Bags\AllFolders\Shell

7. Now click on this new "Shell" key and in right-side pane, create a new String value and give it name as FolderType and set its value to NotSpecified

Fixing_Folder_View_Type_Problem.jpg

8. Either restart your system or log off and now Windows will no longer forget or automatically change your customized folder view type settings.

Let us know if you need further assistance.
