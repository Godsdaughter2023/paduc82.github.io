displayed: load address of the driver, description, version, product name,
company that created the driver, and more.
http://www.nirsoft.net/utils/driverview.html


For Drivers check System Maker as fallbacks and Device Maker's which are the
most current. Control Panel - Device Manager - Display Adapter - write down
the make and complete model of your video adapter - double click - Driver's
tab - write down the version info. Now click Update Driver (this may not do
anything as MS is far behind certifying drivers) - then Right Click - Uninstall -
REBOOT this will refresh the driver stack.

Repeat that for Network - Network Card (NIC), Wifi, Sound, Mouse and
Keyboard if 3rd party with their own software and drivers and any other
major device drivers you have.

Now go to System Maker's site (Dell, HP, Toshiba as examples) (as rollback)
and then Device Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get
their latest versions. (Look for BIOS, Chipset and software updates at System
Maker's site while there.)

Download - SAVE - go to where you put them - Right Click - RUN AD ADMIN -
REBOOT After each installation.

installing actually shows up. This is because some drivers rollback before the
latest is installed (sound drivers particularly do this) so install a driver - reboot
- check to be sure it is installed and repeat as needed.

lemp say:

As I said, you can't use a password reset disk created on computer A to reset the password on computer B.  Consider that if you were able to do that, passwords would be useless.

If you have a Windows 7 installation DVD (not a computer manufacturer's "restore" disk), you can reset the password on your computer IF the account whose password your have forgotten is NOT named "Administrator."

Warning:  the circumstances you described -- you left the computer, it shut off by itself, and when you restarted it your password had changed -- are reasonably suspicious.  Your computer may be infected with malware, which may make recovery more difficult.  Or your computer may have suffered a power failure/power surge that corrupted some files on the hard drive, again making recovery more difficult.

Windows 7 has a built-in account named "Administrator" that has full control of the computer, including the ability to change or reset the passwords of other user accounts.  However, unlike Windows XP, in Windows 7 the built-in Administrator account is disabled by default.  Thus, in order to access this account, you must first enable it.  When you can't boot into Windows, you can enable the Administrator account by using the Windows 7 Recovery Environment.

First, however, you should attempt a few repairs using the Windows Repair Environment.

Start the Windows 7 Recovery Environment Command Prompt as explained here --> http://www.bleepingcomputer.com/tutorials/windows-7-recovery-environment-command-prompt/

You may be lucky and the automated startup repair process may fix things.  If not, and you get to Figure 8, run the bcdedit | find "osdevice" command described in the text following Fig. 8 and make a note of the drive letter.  For illustration purposes, assume that your Windows installation is on drive D: as shown in the article.

Type the following command and press Enter

     chkdsk D: /R

Remember to replace D: with whatever partition was shown when you ran the first command.

Chkdsk may take a long time to run, depending on the size of your hard drive and the number of files.  If it finds and fixes errors, remove the Windows 7 DVD from the computer and reboot normally.  If you're lucky, you'll be able to log in.

NOTE:  The following involves some fairly complicated steps and includes modifying the Windows Registry, which can be dangerous if not done carefully.  Not everyone will feel comfortable doing this.  If you trust your "tech savvy friend," you may want to have her do it for you ... or you may want to take your computer to a local independent computer repair shop.  Do not go to a chain store but do ask your friends for recommendations.

If when you reboot you still are locked out, restart the Recovery Environment Command Prompt again and start with Step 2 of Method II to enable the built in Administrator account -->

http://www.sevenforums.com/tutorials/102552-built-administrator-enable-winre.html

Once you've enabled the built in Administrator account, start the computer normally and log in to that account (by default, there is no password for the Administrator account).  Go to Control Panel > User Accounts and reset the password for your account.  Then disable Administrator again (see http://www.sevenforums.com/tutorials/507-built-administrator-account-enable-disable.html)
