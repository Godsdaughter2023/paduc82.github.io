ron sommer mvp 2009-2017 say:

This information comes from another forum by Bob Lucas:

This is in addition to the excellent advice from Magnus.

GMail has adopted a non-standard implementation of IMAP.  GMail
keeps all of a subscribers' messages in one composite  'folder' -
and applies a 'label' to each message.  The label indicates
whether a message should be treated as 'Inbox', Sent Mail',
'Spam', or 'Bin'.  GMail also has one additional label, namely
'All Mail'.

Because of this non-standard implementation, GMail does not
conform to some of the usual IMAP practices, when accessed using
a desktop mail client program (such as WLMail or Outlook
Express).

When you use WLMail with the GMail IMAP server to delete a
message, it merely removes the Label from the message on the
server.  However, WLMail does not use the terms 'Trash or Deleted
Items' and does not move messages physically to the 'Bin' on the
server.  The message disappears from view in IMAP, because WLMail
can no longer see it in any of the standard folders.

The OP should use a web browser to sign into his GMail account.
If he selects the 'All Mail' label, he will probably discover
that the deleted messages are still there.  Even if messages have
been labelled as 'Bin', they will remain in the 'Bin' for 30
days, before GMail deletes them permanently.

In either case, the OP can reinstate the 'Inbox' or 'Sent Mail'
labels (or allocate a different Label).  If they are in the
'Bin', he should also remove the 'Bin' label.  Next time he uses
WLMail to synchronize with his GMail account, the messages should
return to view.

brian tillman [outlook mvp 2007-2019] say:

This forum is for providing feedback about the Microsoft Answers web site, not for asking technical questions.

Any file ending in .reg is a registry import/export file.  If you double-click it, the keys it contains will be added to your registry.  No not add a .reg file to your registry unless you know what it does.  The file name cannot tell you that.  Open it in Notepad and look at what it does.  Apply it only if you know you need it to fix a problem.  .reg files don't magically appear on your PC.  Either you downloaded it, generated it from within regedit, or some software you ran or installed put it there.  If you don't know what it's for and your PC is running properly, don't use it.
