bornival say:

Setting Eudora as the default mail program

Hi,

OS: Win7

I still (for good reasons) sue Eudora as my main mail client.  Each time I installed a Windows update, Windows set Outlook as my client but this was noticed by Eudora which, upon starting, told me that it was no longer the default client.  I could easily change this by starting Eudora  as an administrator and then clicking on the option to make it my mail default client. 

BUT since the last Windows update, this does not work anymore and Eudora tells me that it was unable to modify the registry (even when starting as an administrator).

I tried to restore Eudora as default mail program by going to Start\Default Program wut Eudora does not appear there and I cannot change the option for MAPI (set to Outlook) to Eudora because this option is greyed...

How do I restore Eudora as my default mail client (mainly for the mailto protocol) ?

bornival say:

I have found my-self a solution that may be useful to others struggling with the same or similar problems.

1. At each (major) Windows update, and for reasons known to Microsoft only, the registry key HKEY_LOCAL_MACHINE\SOFTWARE\Classes\mailto\shell\open\command is modified and gives the command to outlook ... (although it was to Eudora before the update).

2. Eudora could not modify this key (and give back again the command to Eudora.exe) even when launched as administrator because this key is protected...  (I noticed that because I could not modify the key manually).

3. However, when I gave permission to the Administrators for modifying this key and restarted the computer, Eudora (launched as administrator) could modify the key which now reads: C:\PROGRA~2\Qualcomm\Eudora\Eudora.exe /m %1, whihc is what it should do...

So, problem solved but why did Microsoft protected this key during one if its latest update remains a mystery.
