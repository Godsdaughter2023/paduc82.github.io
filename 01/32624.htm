**Update**

I have been messing around with some stuff:

1. I ran a Prime95 test (for 30 minutes) to see if it was the processor. Prime95 ran flawlessly for all 30 minutes. Neither heat or the CPU seems to be malfunctioning.

2. When I go into NVIDIA Settings and change it from Intel HD Graphics to NVIDIA graphics as the global preference (and I set it to NVIDIA by default on the troublesome apps), the BSODs go away.

I'm convinced this is either my Intel HD Graphics going down the tube or the graphics driver being messed up (my desktop is an AMD based system which is perhaps why nothing is wrong on that one). For now, I guess I'll just make sure the Intel HD Graphics get used as little as possible but I would love the opinion of an expert to make sure it's not some silly, stupid little fix that I didn't see.

**Update 2**

I have managed to troubleshoot the issue on my own. My hardware seems to be okay and it appears to be a driver issue with the Intel HD 4600 graphics. 

To fix:

1. I rolled back the display driver (which effectively uninstalled the display driver).

2. I downloaded the Intel Graphics driver from my laptop's support site (straight from Lenovo). It was a much older version.

3. Installed the old graphics driver.

4. No more BSODs.

This is definitely a problem my machine had with the most recent Intel HD 4600 graphics driver. I guess I'll keep Windows from installing driver updates over Windows Update for a while until Intel can get their **** together. Since then, I've experienced no BSODs. 

The reason I didn't try this earlier is because I was stupid enough to try to download the drivers directly from Intel's site, which kept giving me "your computer doesn't support this driver" (screw you, Intel). The manual driver download was my last ditch effort before I returned the machine for a replacement. Good thing I didn't, because the machine is just fine.

**Update 3**

I wasn't able to roll back drivers and re-install the old ones again because, well, Windows 8.1 hates me apparently. I am now getting a "your system is incompatible with this thing that was clearly made for your machine" error. Refraining from taking a hammer to my laptop at this point was the hardest part of this entire troubleshooting process.

So I did some further researching and I guess it is the latest Intel HD Graphics 4600 driver that's causing the issue thanks to several forum threads I was able to find on Adobe's forums where people had the same issue as I do (as in, reporting it over the last week) Using the Adobe forums, I was able to find the following (very temporary) fix which is working for now:

1. Go into the Photoshop/Premiere Pro folder and remove sniffer.exe (keep it somewhere safe in case you have to put it back because this fix apparently doesn't work 100% of the time).

With the sniffer.exe removed, the driver isn't throwing fatal crashes anymore and everything is working fine. I've also found that forcing Adobe apps to use NVIDIA's GPU using the NVIDIA control panel works as well. 

Here's hoping Intel gets their heads out of their collective rear end and releases an update for this soon. Thankfully, my laptop has been restored back to working condition, but not without workarounds which is irritating. Hope this thread helps someone!

Missing and corrupt operating system files (like many DLL files) are arguably the biggest cause of major Windows issues. Considering that, plus the fact that SFC /scannowis completely automatic and very easy to use, the tool should usually be one of your first troubleshooting steps.
