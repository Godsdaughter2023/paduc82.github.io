a. user say:

Often times that error indicates a mismatch between the HDD interface settings in the BIOS and the HDD interface settings of the HDD.  The second parameter being (ending with) 00034 is another clue that might be the problem.

If you are lucky you may be able to get into the BIOS on your system of specified make and model and make an adjustment there.  After you make the changes, save them and see how things look.  You may have to do some experimentation to get it to behave.

You'll have to get into the BIOS, find the HDD interface section and before making any adjustments make a written note of the current settings so you can undo the adjustment later if needed.

Here are some examples I have accumulated for various systems:

If you find the mode is set to RAID/SATA mode, change the mode to IDE.

If you find the mode is set to SATA, change the mode to IDE.
    (Acer Aspire)

If you find the mode is set to SATA, disable SATA mode.

If you find the mode is set to AHCI, change the mode to ATA.

If you find the mode is set to AHCI Emulation Mode, change the mode to IDE Emulation.

If you find the mode is set to SATA, disable SATA mode and/or change the mode to ATA.

If you find the mode is set to SATA Native Mode, disable SATA Native Mode.

If you find the mode is set to RAID/Autodetect AHCI, change the mode to Combination.

If you find the mode is set to RAID Auto/AHCI, change it to RAID Auto/ATA (this was a Dell XPS 420)

On some system of unknown make and model, In the Integrated Peripherals menu, disable" On Chip IDE Channel 0

On an Acer Aspire 4720Z the mode needed to be changed from "AHCI Mode" to "IDE Mode"

The WWW says this works on some Dells:  "I had to set my SATA Operation to ATA instead of AHCI"

After making the adjustments, ACCEPT and SAVE the changes, and restart the system and see if it will now boot.

Hi MackenzieXQ,

 

NvMcTray.dll is the ‘NVIDIA System Tray process’ file that loads your default video profile upon startup.

 

I’d recommend you to disable the same from loading at startup.

 

Follow these steps:

 

a.     Click “Start.”

b.     Type “msconfig.exe” in the “Start” search box and hit “Enter.”

c.     Click the “Startup” tab.

d.     Uncheck “NvMcTray.dll” from the list.

e.     Click “Apply” and “OK”.

f.      Restart the computer.

 

Note: Ensure NOT to disable any of the Microsoft services.

 

Additionally, you may uninstall and reinstall the NVIDIA drivers to fix the startup issue.

 

Also see: Troubleshoot installing or uninstalling programs
