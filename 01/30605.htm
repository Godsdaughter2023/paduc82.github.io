a. user say:

When you see that message, it means that your XP volume has been marked "dirty" for some reason.

You can check the status of your dirty bit from a Command Prompt window.

To do that, click Start, Run and in the box enter:

cmd

Click OK to open a Command Prompt window.

For example, to query the dirty bit on drive C, type:

fsutil  dirty  query  C:

Sample output:
Volume C: is dirty
Volume C: is not dirty

Is your volume dirty or is it not dirty?

Your's is dirty.

A volume can be marked dirty from things like a power interruption, aborted restart, or improper shutdown?  (this includes plug pulling, power buttons, removing the battery, etc.).

Have any of these event occurred?

The only thing that can clear the dirty bit on a volume is chkdsk /r (or chkdsk /p).  You could tell XP not to check the dirty bit on a volume where the dirty bit just will not clear, but that is probably not a good idea since there is some kind of problem on the volume.

You will also not fix this problem by editing your registry or running any kind of chkntfs command (pay attention Microsoft engaged Support Engineer and Forum Moderator "experts") - that is not where the problem is.

Once a volume has been marked dirty, the only way to clear the dirty bit is to run "chkdsk /r" or "chkdsk /p" when the volume is not in use.  The only way a volume can be not in use is while XP is just starting to load or from the XP Recovery Console.

If XP will not run chkdsk properly when your system reboots or the dirty bit will not clear as it should, you can look in the Event Viewer for the chkdsk log and see what is going on if you are curious, but somehow you must get chkdsk /r  to run clean.

If using "normal" methods XP still won't clear the dirty bit, you should run your chkdsk /r  from the XP Recovery Console so you can watch it and see what does and run it again until it does work (this will usually do the trick). 

If your hard drive is failing, you may never be able to clear the dirty bit, in which case you can also tell XP to never check the dirty bit on the afflicted volume, but that will not fix your defrag issue.

Telling XP not to check the dirty bit on a volume would be risky behavior since there would still be some kind of problem with your hard disk.
