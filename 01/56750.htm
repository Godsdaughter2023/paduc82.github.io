This issue may have been caused by broken components of Windows Update, or due to a software conflict.

Meanwhile, to troubleshoot Windows Update issues, kindly refer to the steps below:

Method 1: Run the Windows Update troubleshooter.

Method 2: Scan for system file corruption.
System File Checker (SFC) is a utility in Windows that scans for corrupted Windows system files. If it finds a problem, it will try to replace the problematic system files from a cached version that’s on your PC. If SFC can’t replace the files, the Deployment Image Service and Management Tool (DISM) can check the corrupted files then download and replace the files through Windows Update. To do this, refer to the steps below:

  1. Press Windows key + X on your keyboard.
  2. Choose Command Prompt (Admin).
  3. Copy or type each command followed by hitting Enter.
    sfc /scannow
    dism /online /cleanup-image /restorehealth

Method 3: Perform a clean boot.
A clean boot will help eliminate software conflicts that occur when you install a program, update, or when you run a program in Windows 10. For instructions, you may refer to this link.

Note: After the troubleshooting, boot your computer in normal mode by checking the same link and look for How to reset the computer to start normally after clean boot troubleshooting section of the article.

Regarding your query with the device compatibility with Windows 10 Creators Update, you can consider contacting your device manufacturer for assistance regarding your query. You can also seek help from their support team in installing the most updated driver compatible for the Windows 10 1703 build.
