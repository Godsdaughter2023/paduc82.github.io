ronnie vernon say:

Hi jafinnie

There can be many reasons for this behavior when downloading files from an internet location.

The bottom line is that Windows 7 is attempting to protect itself from infections. The files are being Virtualized to a safe location on the hard drive.

Open the folder where you selected to save the files and look on the Toolbar for a  Compatibility Button. Click that button and the virtualized location will open. This will be somewhere under the  C:\Users\your name\AppData\Local\VirtualStore folder.

Right click each one of the files you downloaded and select Properties. Look at the bottom of the dialog and you should see an  Unblock  button. Click that button. You can then move the files to whatever folder you wish.

Reference:  Common file and registry virtualization issues in Windows Vista or in Windows 7

cmdrkeene say:
Sorry, it's actually not possible.  Those columns read data from inside the files themselves.  Different files can contain different kinds of info.  Pictures, for example, might show their length and width, while music files show a composer.  Folders have no such "meta data" attached to them.
