keith a. miller say:

Sounds like we're about the same age! :D

PowerShell will make this easy. The first command you want to use is 'Get-ChildItem', which can be abbreviated 'gci'. It's similar to the 'dir' command in the Command Prompt. So, open a PowerShell window & try the following commands one at a time:

gci

gci -force

gci desktop.ini -force

gci desktop.ini -force -recurse

gci desktop.ini -force -recurse -erroraction silentlycontinue

The first command lists the files & folders in the current directory. The second adds the '-force' parameter to include hidden files & folders. The third command limits the output to paths ending in 'desktop.ini'. The fourth command adds the '-recurse' parameter. If you execute this command from your userprofile folder, you'll see 'access denied' errors for various sub-directories of AppData, so we add '-erroraction silentlycontinue' to suppress those errors.

One of the great and fundamental concepts of PowerShell is the pipeline. The '|' character is the pipe character, which passes output from one command to the next. So let's take a look at the content of those files:

gci desktop.ini -force -recurse -erroraction silentlycontinue | get-content
