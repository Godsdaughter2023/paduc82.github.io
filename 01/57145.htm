    COMM Ports Change After Setting in Device Manager and Reboot - Microsoft Community

    aaronhamilton511 say:

    COMM Ports Change After Setting in Device Manager and Reboot

    I am having an issue with Windows 10 resetting my comports after the system reboots. I keep finding information labeled similar to my issue but they only talk about going into device management and setting the COMM Port that I have. I run 4 Arduino's set on COMM 6, 7, 8, and 9 but after I restart the computer the ports change COMM Position. I then have to unplug and plug the USB ports back in and the COMM assignments correct themselves.

    How can I keep the COMM Ports from having this issue?

     

    mala_s say:

    Hello AaronHamilton511,

    Thank you for writing to Microsoft Community Forum.

    When you restart the computer, communications port (COM port) settings revert to the default. This occurs even though you have changed the settings in Device Manager. For example: if you run a command prompt, you may notice that the default settings for com 1 are:

    Baud rate=1200; Parity=None; Data Bits=7; Stop Bits=1

    You may have a program that requires different settings, such as:

    Baud rate=9600; Parity=None; Data Bits=8; Stop Bits=1

    You can manually set com 1 to function at the settings you want by using this command:

    Mode Com1: 9600,n,8,1

    However, when you restart the system, you find that the setting reverts back to the default:

    Baud rate=1200; Parity=None; Data Bits=7; Stop Bits=1

    COM port settings for command functions are maintained only for the active Windows session. Custom settings are discarded at shutdown.

    To resolve this issue, create a startup task that sets the COM port to the settings that you want. The task can be set to run minimized with the close window on exit setting selected.

    A sample shortcut has this command line: C:\winnt\system32\mode.com com1: 9600,n,8,1.

    Hope it helps.

    Mala S
