    dir E:\Win*

    cd /d Q:\windows\System32\config
    xcopy *.* Q:\RegBack\
    cd RegBack
    dir
8. Examine the dates of the Software, System and SAM files. Were the files created before or after your problem became apparent?

    copy /y software  ..
    copy /y System  ..
    copy /y Sam ..

lemp say:
I can't give you a definitive answer because I haven't looked into Internet Connection Sharing (ICS) for Windows 7, but if it works the same way as ICS in XP, the following may help.

A number of things happen when you set up ICS.  The network adapter that is the interface to the devices that want to share the Internet connection is configured with the IP address 192.168.0.1.  
There's no documented or supported way in XP to change this IP address or the ICS subnet.  I assume
 -- but don't know -- that it's the same for Win 7.

In addition, a software "IP address allocator" (similar to a DHCP server) is set up to automatically 
assign IP addresses to the network adapters in the devices (in your case the Xboxes) using 
the 192.168.0.x subnet.

If your Nokia's Internet sharing feature also forces the Nokia to have an IP address of 192.168.0.1 
(and allocates an 192.168.0.x address to the wifi adapter in the notebook), you'll have a problem.

You may be able to do things manually, but there's no guarantee that the following will work.  
I assume that the Nokia connects to the laptop via wifi.

  • Start by setting up the Internet connection between the Nokia and the laptop.
  • Open a Command Prompt, type
                       ipconfig

            then press Enter.  Make a note of the IPv4 address and subnet mask assigned to the wifi adapter.
  • Open the Network and Sharing Center and then click "Change adapter settings"
  • Press the CTRL key and click the two adapters (the wifi adapter which is connected to the Nokia and the LAN adapter that will be connected the Xbox).
  • Right click and select "Bridge connections"
  • Run another ipconfig and make a note of the IPv4 address, subnet mask, Default Gateway, and DNS server addresses assigned to the bridged connection.  
  • Manually configure the Xbox to use an IP address in the same subnet as that assigned to the Bridge (i.e., 192.168.0.x, where x is less than 255 and different than the other assigned addresses; choose something such as 192.168.0.75).
  • Manually configure the Xbox with the remaining information:  the subnet mask, Default Gateway, and DNS server addresses assigned to the bridged connection. 
If this doesn't work, you may want to look into getting a wifi adapter for your Xbox: 
 https://www.google.com/#q=xbox+wifi
