quilnux say:

By default Automatic maintenance shouldn't be running when you are on battery. However, I have seen cases where it will start when you are on AC but will not pause/stop when you switch from AC to battery while it is already running.

 

Automatic maintenance does several things and each item it does may or may not run every day.

 

The OOB jobs include:

 

Defragment the hard drive (once a week)

check hard drives for integrity issues (not the same as CHKDSK. It just roams the file system looking for quick fix issues) (every 2 days)

Preforms SCRUM and TRIM on any SSDs as needed (I think once a week)

Checks Windows 8 Licensing (every week)

Checks for Windows Updates and downloads them (Only check and download, doesn't install. Once a day)

Runs built-in Anti-virus check (once a day, quick scan, once a week, full scan. Disabled/doesn't run if a different anti-virus solution is installed)

Runs performance checks (once a day I think)

 

Any 3rd party software package you install can also register to run jobs in the Automatic Maintenance Window. It's also important to understand that other maintenance "can" run outside of this window. Some maintenance gets run when the system is on AC and in idle mode.

 

My understanding is, this cannot be fully disabled. Doing so would be bad as some critical jobs won't run. This also includes renewing your Windows 8 license which must occur at least once a month.

So for all of you, who are like me, annoyed by this stupid feature here are the steps to take: 

1. Get to the control panel, for me I found the easiest way is to press windows key + e, this opens up windows explorer, click the "computer" tab at the top and you will see an option to open up the control panel

5. in the window on the right you will see the tasks that it has scheduled for you that are predefined by windows 8.

6. select all "ctrl-a" then delete

7. this will delete all of those tasks that Windows 8 runs for you, including the Automatic Maintenance.

quilnux say:

Well, the built-in AV does. So that would probably be the Windows 8 version of Security Essentials.

 

I got a friend that uses Kingsoft AV. After he installed it he deleted his automatic maintenance tasks and shortly after that the Kingsoft AV kept crashing when it would want to run (either manually or automatically). After he called Kingsoft TS and spent 3 hours on the phone they told him he needed to reinstall Windows. You can get Kingsoft from the Windows Store. I don't have a direct link for it.

 

Granted I don't think that Norton or McAfee will be using Automatic Maintenance. But there are other things besides AV that can use this. Personally, I feel that when you delete something it's a permanent thing. It's destructive and there is no way to return to a state where you have it. not without reinstalling. It is much safer to disable it instead. That way if you ever run into software that uses it, you can simply re-enable it.

 

Keep in mind that if you ever use an SSD, you cannot use the SCRUM or TRIM features without going thru AM. You cannot start it manually without AM because when you use the console to start this process manually it adds a task into the AM manual task job.

 

To me it's just better to disable rather then delete. You never know what software you are going to install in the future that will require the use of this feature. Since this is something new in Windows 8, software developers will probably implement it into their software moving forward. It may take a few more iterations of Windows before we see wide-spread use but it's better not to get into the habit of destroying new features until you know exactly what software is going to use it. Disabling is always the safe option.
