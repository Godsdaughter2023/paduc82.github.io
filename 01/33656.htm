andre.ziegler say:

Bug Check 0xC000021A: STATUS_SYSTEM_PROCESS_TERMINATED

The STATUS_SYSTEM_PROCESS_TERMINATED bug check has a value of 0xC000021A. This means that an error has occurred in a crucial user-mode subsystem.

http://msdn.microsoft.com/en-us/library/ff560177%28VS.85%29.aspx

Cause

This error occurs when a user-mode subsystem, such as WinLogon or the Client Server Run-Time Subsystem (CSRSS), has been fatally compromised and security can no longer be guaranteed. In response, the operating system switches to kernel mode. Microsoft Windows cannot run without WinLogon or CSRSS. Therefore, this is one of the few cases where the failure of a user-mode service can shut down the system.

Mismatched system files can also cause this error. This can occur if you have restored your hard disk from a backup. Some backup programs might skip restoring system files that they determine are in use.

Resolution

Resolving an error in a user-mode device driver, system service, or third-party application:Because bug check 0xC000021A occurs in a user-mode process, the most common culprits are third-party applications.If the error occurred after the installation of a new or updated device driver, system service, or third-party application, the new software should be removed or disabled.

If the error occurs during system startup, restart your computer, and press F8 at the character-based menu that displays the operating system choices. At the resulting WindowsAdvanced Options menu, choose the Last Known Good Configuration option. This option is most effective when only one driver or service is added at a time.

nirmal s say:

Hi AngelikaBraun,


1) Are you referring to a Word document?

 

 

Method –


Step 1-


You need to follow the steps mentioned below only if it was a Microsoft Word document-


a) Open up Microsoft Word. Do not click on a file to open the program. Instead, open the program by double-clicking on the program's icon. Allow the program to load for a couple of minutes before clicking on anything.
b) Look on the left side of the screen. If Microsoft Word recovered the file, a "Document Recovery" task pane will load.
c) Find the file's date. Double-click on the file in the "Document Recovery" pane. This will open the file in Microsoft Word.
d) Click on the "File Menu" and select "Save As."
e) Save the document as a ".doc" file type, and type a name for your document in the "File Name" box.

 

 

Step 2-


a) Look for backup copies of the file that may have been created by Word. Click "Start" on the Windows desktop, and then select "Search." Type "*.wbk" into the search box, hit "Enter" on your keyboard and look through the results for the file in question.
b) Use the Auto Recover function of Word. Open the Word program and select the "Open" function. Look for the file in question, and if you locate it, click on "Open and Repair."
c) Look for files that were saved by Auto Recover in another location. Click "Start" on the Windows desktop, and then select "Search." Type "*.ASD" into the search box, hit "Enter" on your keyboard and look through the results for the file in question.
d) Look for files that may have been saved as temporary files. Click "Start" on the Windows desktop, and then select "Search." Type "*.TMP" into the search box, hit "Enter" on your keyboard. Look through the results for the file in question.
e) Look for files that may have been moved to the Recycle Bin. On the Windows desktop, click on "Recycle Bin." Click "View" and select "Arrange Icons by." Select "Date Deleted" in order to quickly locate any files that may have been moved here on the date in question.

 

 

If none of the steps help fix your issue then you may use your favorite search engine to look for third party softwares over the Internet that can recover the files which were not saved.
