The error code 0x80070057 signifies that there are some damaged/corrupt System Files which is preventing the installation of the Update.   2. Click on the Command Prompt(Admin) option
  3. If there is a User Account Control prompt, click Yes or Continue.

  • DISM.exe /Online /Cleanup-image /Scanhealth
  • DISM.exe /Online /Cleanup-image /Restorehealth

Please note, depending on the size of the system, these commands could take long time to complete. Please let them run and finish their job.

Another thing you can try is to scan your system for general corruption or corrupt system files (which may include this file if it isn't malware).  In the process of repairing the corruption, it may repair whatever is causing MSE to freeze when it comes to that file.  It may not be able to repair all corruption without the Vista Installation Disk but some it can repair without it (and hopefully that will be the case here).

It it has happened with three well-known and respected anti-malware programs, then there is definitely something going on in your system.

There are a lot of errors in the Event Log, but I could find nothing that relates to this problem (though there may be something there - I'm only marginally familiar with analyzing Event Viewer reports and I look for patterns or repetitions or indications of bad or missing/corrupted files.  The message about the BIOS concerns me - but though it should be addressed, I don't believe it is the cause of this problem.  Nevertheless, the options we're left with will hopefully resolve all those errors as well as the error that is the subject of this thread.

It looks like we'll need to do a system repair/upgrade using the genuine Windows Vista Installation Disk (one you own or one you can borrow from ANYONE).  Here's the procedure: http://www.vistax64.com/tutorials/88236-repair-install-vista.html.  Although this will not affect your data, settings, or programs, you should still backup your data before starting just to be on the safe side. If the other procedures don't work, THIS one is almost certain to work.  You may have a lot of updates to re-install (including any service packs you had to remove).  If the version on the system came with SP1 or SP2 pre-installed and the disk is an earlier version, then you'll need to make a slipstream disk as follows: http://www.vistax64.com/tutorials/151606-vista-sp1-slipstream-installation-dvd.html.

If you don't have the disk and can't get your hands on one (it would be really best if you could), then I'm afraid the only option is a clean install.  You can use Knoppix http://www.knopper.net/knoppix/index-en.html with a good ISO copier like: http://isorecorder.alexfeinman.com/isorecorder.htm along with a blank CD.  This should give you enough access to the system (if you can't get in any other way but I don't think that's a problem for you) to backup your important data.  Once done, you can do a clean install either using the genuine Windows Vista Installation Disk or the Recovery Disk or the Recovery Partition (whatever process is dictated by your computer manufacturer – you may need to contact them for the procedure and perhaps to get recovery disks).  To do a clean install proceed as follows: http://www.winsupersite.com/showcase/winvista_install_03.asp (adapted as necessary by the procedures of your computer manufacturer).  Then you will need to re-install all your programs, reset all your preferences, reconfigure your network and email settings, restore your backed up data, run Windows Update with possibly nearly 100 updates pending,...

NOTE: Once you've completed the backup if you're doing a clean install and since you have nothing to lose, try deleting the mscoree.dll file.  If normal delete doesn't work, rry using one of the following free products to delete the file/folder.  Unlocker at: http://www.softpedia.com/get/System/System-Miscellaneous/Unlocker.shtml or File Assassin http://www.malwarebytes.org/fileassassin.php (with or without Force Delete: http://www.mydigitallife.info/2008/12/27/force-delete-cannot-delete-locked-or-in-used-files-or-folders-with-fileassassin/). These programs often work when normal Vista delete functions fail to operate properly.

You may need to revise the permissions and ownership to allow this deletion.  Here's how to do that.

To see your permissions, right click the folder/file, click on properties, and check out the security tab.  Check to see what permissions you have by clicking on your username (or user group).  Here are the types of permissions you can have: http://windows.microsoft.com/en-US/windows-vista/What-are-permissions.  You need to be an administrator or the owner to change permissions (and sometimes being an administrator or even an owner isn't enough - there are ways to block even them from access (though a smart administrator knows these ways and can get around them - but usually shouldn't because they were denied access usually for a very good reason).  Here's how to change folder permissions in Vista: http://www.online-tech-tips.com/windows-vista/set-file-folder-permissions-vista/.  To add take and grant full right permissions and ownership to the right click menu (which will make it faster to get in once it is set up), check out the following article: http://www.mydigitallife.info/2009/05/21/take-and-grant-full-control-permissions-and-ownership-in-windows-7-or-vista-right-click-menu/.

To fix this problem with the folders, take Owner tab.  Click edit and then click the name of the person you want to give ownership to (you may need to add it if it isn't there - or it may be yourself). If you want it to apply to sub-folders and files within that folder/drive, then check the box for replace owner on sub-containers and objects and click OK.  Back out and now there's a new owner for the folders/drive and files who can change permissions as needed.  You can now change them from read-only (even if the main folder says they still are read only - you can have access yourself as the owner).  You can keep them as read-only for other users and guests and even administrators (though they can take ownership themselves and gain access if they so desire and there really isn't much you can do to stop this except protect the folder with a password using a 3rd party product.  Here's more information on taking ownership of a file or folder: http://www.vistax64.com/tutorials/67717-take-ownership-file.html.  To add take ownership to the right click menu (which will make it faster to get in once it is set up), check out the following article: http://www.howtogeek.com/howto/windows-vista/add-take-ownership-to-explorer-right-click-menu-in-vista/.

I realize this is a lot of effort (and we're not sure it will work or that it will help or if it will cause even more problems), but if we can remove the file and the system still functions normally (and the anti-malware scans can complete properly) then that's probably better than a clean re-installation if that's the only remaining option and you've decided to do it.  But the choice is yours.

I hope this helps.

Good luck!
