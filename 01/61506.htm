bill smithers say:

Hi,

You can't install any type of RAM you wish to and you are governed by the Motherboards maximum capacity for installing RAM.

In other words, you may want to install 16GBS but your Motherboard's max. RAM capacity may be 12GBs, or it may be at the max now.

Either go to your Computer/Laptop Manufacturer's website > Support/Downloads Section > look up the exact details there.

Or it may be easier for you to download the Crucial Scanner which will tell you what is the matching RAM for your Motherboard and the max. amount of RAM your Motherboard can accommodate.

Crucial System Scanner

http://www.crucial.com/usa/en/systemscanner

@@@@@@@@@@@@@@@@@@@@@@@@@@@@@

And remember if you are using a 32bit operating system, installing more than 4GBs of RAM is a waste of time, as Windows can't use it.

It will use only about 3.25GBs out of the 4GBs anyway and the rest is reserved for hardware, etc

To use 8GBs of RAM, IF you can install 8GBs of RAM on that Motherboard, you will have to install a 64bit operating system.

Cheers.

david dickinson say:

When this error occurred in Windows 7, Microsoft published a hotfix for it. See KB2615701:

https://support.microsoft.com/en-us/kb/2615701

There does not appear to be a similar fix for Windows 10 available yet. However, There is a workaround for Windows 10, although it is not a solution to the problem that is causing the error.

When the message appears in Windows 10, the mouse is inactive and the keyboard appears to be unresponsive. You cannot click the "OK" button in the dialog box that displays the error message, and pressing <ENTER> or other keys does nothing. This is because the dialog box is not the active window and cannot receive inputs from the keyboard.

When the message appears on the screen, press ALT+TAB. You may see a change in the color of the dialog box's title bar indicating that it is now the active window. Press <ENTER>, and the dialog will close, allowing the logon process to begin.

This workaround succeeds on Windows 10 version 1511 (build 10586).
