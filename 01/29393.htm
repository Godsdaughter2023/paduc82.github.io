BCCode: D1     0x000000D1

Cause

A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high.

This bug check is usually caused by drivers that have used improper addresses.

If the first parameter has the same value as the fourth parameter, and the third parameter indicates an execute operation, this bug check was likely caused by a driver that was trying to execute code when the code itself was paged out. Possible causes for the page fault include the following:

  • The function was marked as pageable and was running at an elevated IRQL (which includes obtaining a lock).
  • The function call was made to a function in another driver, and that driver was unloaded.
  • The function was called by using a function pointer that was an invalid pointer.

gary vandermolen say:
192.168.1.1 is the local address of your router or modem.  In turn, your router or modem gets its DNS information from your ISP, unless you override it with a manual setting.
 
I don't have a Vista computer handy, but on my Windows 7 computer I would right-click on the connection icon in the system tray, select "Open Network and Sharing Center". Then click on "Change adapter settings". Right-click on the adapter being used (wireless or LAN), select Properties, then "Internet Protocol Version 4 (TCP/IPv4)", then Properties. Instead of "Obtain DNS server address automatically" click on "Use the following DNS server addresses". Then enter 8.8.8.8 and 8.8.4.4 in the two spaces, click OK, and reboot. Did that correct the problem?

The first thing to ask yourself before you install a software application is: Will this version of the software run properly on my operating system?

It's easy to find out. Just go the the software developer's website and see whether the version you want to install is supported on Windows 10 (in this case.)

If the version you want to install is not supported it may still work ... or it may not. It's often possible to install unsupported software on Windows 10 by using compatibility mode, but that's no guarantee that the application will work properly. It might or it might not. And here's the thing: If the unsupported software doesn't work properly, the software developer won't help you to get it working, if that's even possible.

So all things considered, it's best to use the current version of a software application.
