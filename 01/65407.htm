Over the last few days I've upgraded two computers to Windows 10 and in both cases have hit some problems on the way.

The first upgrade was of a four-year old Dell N5110 laptop from Windows 7.

The download and upgrade went smoothly. Windows 10 seemed to start but then suddenly the screen went black. No pointer, no nothing. It stayed like that for several hours. I forced the system to shut down and powered it up and the same thing happened. I plugged a monitor into the VGA port at the back and now I could see that Windows 10 was waiting for a reply to a prompt so that it could complete its configuration. After Windows 10 was fully installed and configured the laptop screen remained black. Eventually I used device manager to disable the Intel Graphics 3000 Display Adapter and shut the laptop down. When I restarted the laptop Windows 10 found Microsoft's basic display driver and the screen started to behave.

It seems that the Intel Graphics 3000 driver doesn't always work with Windows 10. I say 'always' because some contributors to forums claim to have had success.

Note that uninstalling the driver didn't work. Windows 10 reinstalled it and the screen went black again, but disabling it seems to have done the trick.

The second upgrade was of a Lenovo G510 laptop from Windows 8.1. The upgrade went very smoothly but there have been a few strange problems since.

The first concerns Google Chrome. Clicking on any of the various icons for Chrome achieved nothing. I decided to uninstall Chrome and reinstall it to see if that fixed things. Apart from a problem with Gmail which I need to look into everything seems fine now.

The second problem was that I was unable to save files. Somewhere along the way I had lost permission to write or update anything on my C drive - including anything in my Documents folder. Hi Dave,

Sorry to know that you are facing such an issue with Windows 10.

If you see the black screen only after sign-in and you can see a mouse cursor:

  1. While at the black screen, try using CTRL+SHIFT+ESC to start Task Manager.
  2. If it becomes visible, try finding LockAppHost.exe on the Details tab and ending the task.
    1. If this resolves the black screen, use corruption resolution steps (dism, then sfc), and run the all apps PowerShell command:PowerShell -ExecutionPolicy Unrestricted
      Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register
      "$($_.InstallLocation)\AppXManifest.xml"}
  3. If this does not work, use File, Run new task in Task Manager to start an Administrator CMD window. Use this to create a new administrator user with the following commands (changing “<password>” to something of the customer’s choosing):
    net user test <password> /add
    net localgroup Administrators test /add
  4. This will give you a test user to try, which will confirm if the issue is user or system-specific.

If you see the black screen or have a flickering screen before sign-in:

  1. Try connecting to different video outputs on the PC. Sometimes a change in driver can cause the video to be sent:
    • To a different Adapter: from integrated graphics to a discrete card or vice versa.
    • To a different output: from HDMI to DVI, DisplayPort to VGA, or any other combination.
  2. Try to boot into Safe Mode with Networking or Enable low-resolution video.

    • Advanced boot options, Safe Mode with Networking, or Low Resolution Video
    • Check Device Manager and look for Display Adapters
      • If you see 2 items under Display Adapters, disable to integrated video card (e.g. Intel HD 4000 or AMD Radeon HD 4200) and restart. (try vice versa if it does not work)
      • If you see 1 item under Display Adapters, rollback the driver or uninstall the display adapter in Device Manager, and enable the option to delete driver software.

Check the PC or display adapter manufacturer’s site for an updated driver.
