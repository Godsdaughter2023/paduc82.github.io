Microsoft does not provide any other disk optimizing tools apart from the one which is inbuilt in Windows Operating System. But, I would first suggest you to try the following methods and check if that resolves the issue.

 

Important: Using Third Party Software, including hardware drivers can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the use of Third Party Software can be solved. Using Third Party Software is at your own risk.

 

There are several reasons why Windows 8 “Optimize Drives” tool stops in between while analyzing the drive on the computer. This could be either due to some disk error, corrupted system files, virus infection or some third party software conflicts.

 

1)       Are you using an SSD drive or a normal hard disk?

2)       Does this happens only with the C Drive?

3)       When the analysis of the drive stops, what does it show under Current Status for the C drive?

To run “ChkDsk” utility using Command Prompt, you can follow these steps:

 

a)       Press “Windows Logo” + “X” keys on the keyboard and select “Command Prompt (Admin)” from the list.

b)       In the command prompt window, type “chkdsk /r” and press “Enter”.

c)       Answer “Y” when “chkdsk” asks you if you want to check the drive next time computer boots (restarts).

 

Important: While performing Error-checking or ChkDsk on the hard drive, if any bad sectors are found and when attempting to repair that sector, the data available on that sector might be lost.

 

After completing the check disk, check the result and let’s know the details. You may also check if this is caused by any corrupted system protected files on the computer.

 

Method 2: Run an SFC Scan

 

The sfc /scannow command scans all protected system files and replaces incorrect versions with correct Microsoft versions.

 

a)       Press “Windows Logo” + “X” keys on the keyboard and select “Command Prompt (Admin)” from that menu.

b)       On the command prompt window, type the following command, and then press “Enter”

 

sfc /scannow

 

c)       Follow the on-screen instructions and complete the scan.

 

Important: It may ask for windows installation DVD to fix and to enable SFC to make more than minor repairs. Some files saved on your computer might get deleted when fixing corrupted files.

Optimize your hard drive

lemp say:
You need to chill out.

Windows Powershell did not "install itself" nor did it get installed when you ran the as-yet unidentified FixIt routine.

Powershell is available as an optional Windows Update.  Optional updates are not installed by Automatic Update, so you must have gone to Windows Update (or Microsoft Update), selected the "Custom" button and then selected it.

Powershell can be uninstalled by going to Control Panel > Add or Remove Programs, selecting "Show Updates" and then selecting the proper update and clicking Remove.  The problem is that the update is not identified as "Powershell" in the list of Windows XP udates.  "Syed N" gave you only half the answer:  he didn't tell you that for Powershell 1 you need to uninstall the Windows update with the Knowledge Base article number KB926139 and for Powershell 2.0 (the most likely, since that update was offered recently) you need to uninstall the update identified as "Windows Management Framework Core" (or possibly, but less likely, KB968930).

I guess I'm not surprised that took 4 different MS tech support reps to know how to identify the update.  Disappointed, perhaps, but not surprised.

Finally, Powershell scripting is disabled by default.

The security settings built into Windows PowerShell include something called the “execution policy;” the execution policy determines how (or if) PowerShell runs scripts. By default, PowerShell’s execution policy is set to Restricted; that means that scripts - including those you write yourself - won’t run. Period.  http://technet.microsoft.com/en-us/library/ee176949.aspx

As for your IE8 problems, start another thread in the IE forum: http://answers.microsoft.com/en-us/ie/forum

Give it a meaningful subject
Identify the version of IE you have (as well as the version of Windows, including service pack)
Describe the issue in as much detail as you can.
Give the web address for the FixIt you used and describe the options you selected.

Your System Restore issues are yet another problem, almost certainly unrelated to the Powershell incident.  The most comprehensive help site for System Restore is http://bertk.mvps.org/

However, if you are getting "Unable to restore" messages, the best thing to do is probably to turn System Restore off, which will delete all of the current Restore Points, and then turn it back on. 

  • Right-click on My Computer
  • select Properties
  • select System Restore tab
  • check "Turn off System Restore on all drives"
  • OK

Uncheck the box and use the "Settings" button to turn SR back on for the C drive.  Set the amount of space used to be around 1 GB.
