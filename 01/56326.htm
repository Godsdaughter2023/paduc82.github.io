lemp say:

What is the make and model of the printer?

How is it showing that it is connected? 

What is the make and model of your wireless router?  Please look at the label on the back or bottom of the device and include any hardware version mentioned.

This issue happens fairly frequently with wireless printers, typically when the printer has been turned off for a while during which time other devices are connected to your local wireless network. 

Almost all (if not all) wireless printers are configured by default to automatically obtain an IP address from your router.  When you initially installed the printer, Windows discovered the printer at that IP address and configured the printing system to direct its output to that IP address.

If during the time that the printer was off you connected another wireless device (e.g., another computer or a smart phone), the router may have assigned the original IP address to the other device.  The router will then assign a new IP address to the printer.  The problem now is that Windows is still configured to print to the original IP address ... but the printer is no longer there.

The solution is to assign the printer a "static" or fixed IP address instead of permitting it to use its default of obtaining an IP address automatically.

To confirm the above diagnosis, read your printer manual and either print a configuration page or use the printer's LCD display (if it has one) to show its current IP address.  Write it down.

On the Windows 7 computer that doesn't print, open Devices and Printers. 

Select (click once) any printer. 

Click "Print server properties" on the tool bar at the top of the Devices and Printers window.

Select the "Ports" tab.

Scroll through the list of "Ports on this server" until you find the one for your printer.

Expand the "Port" column until you can see the entire IP address listed for your printer.

If the IP address listed here is different than the IP address reported by your printer, you'll need to configure the printer to use a static IP address.  If the IP address here is the same as that reported by the printer, you have a different problem.
