Hello groverofl,

Getting the error: Not a valid 32-bit application can have many causes:
I have listed some of them below.

File is corrupt, bad, or missing.
- If the hard disk drive is corrupted or bad it can cause working programs to fail either because not all the data can be read from the hard disk drive or because the program has become corrupted while on the drive. Run scandisk and defrag on the hard disk drive to check for any potential errors.
- If you're running the program from a shortcut on the computer, verify that the program is still on the computer. In some cases if the program is no longer installed on the computer you may get this error.

File is not designed for your version of Windows:
- If you're trying to run a program that is not designed for your version of Windows you may get this error.
- Although many older programs designed to work in older versions of Windows will work with new versions of Windows, unfortunately, not all programs will work.
- If the program is an older MS-DOS program or early Windows program you may get this error.
- If the program is designed for a 64-bit version of Windows and you're running it in a 32-bit version of Windows, it will not work and generate this error.

File is a virus, worm, or other malware file.
- This error can be generated by a file that is a virus, worm, trojan, or other type of malware file. Often, this will be caused because the virus scanner installed in the computer will not allow the file to be installed or run. Try scanning the file to verify it is not a virus and/or infected.
- If the file has been checked and is clean, it is still possible that the virus protection program and/or another program installed on the computer is causing issues during the install or execution of the program. Boot the computer into Safe Mode and try running the program; booting the computer into Safe Mode will make sure nothing is running in the background that could be causing this issue.

Hardware incompatibility.
 - If you're getting this error during the installation of a program it's also possible that the CD-ROM drive or the drive you're installing the program from is not compatible with Windows or has drivers that are not compatible with Windows.
- Download the latest drivers for your CD drive or other drive from the computer manufacturer or from the hardware manufacturer. (Microsoft does not make driver for other hardware)

 You will have to look over the issues to see if any of them apply in your case.

Hope this helps.

futurelight say:

I believe I found an actual solution to this problem, and not just a workaround. 

The core issue is a terrible default power management setting for the nVidia GPU. Here's how to fix it:

Video explanation: https://www.youtube.com/watch?v=y7QIBMEyiy4


Steps:

  1. Download and install GeForce Experience from here: http://www.geforce.com/geforce-experience/download
  2. Once installed, right click on your desktop and select "NVIDIA Control Panel"
  3. Click "Manage 3D Settings"
  4. Go to the "Program Settings" tab and select "Adobe Photoshop (Photoshop.exe)" from the dropdown list.
  5. Under the "Specify Settings" list, scroll down to find the "Power Management Mode" setting, and change it from "Adaptive" to "Prefer Maximum Performance."
  6. Restart Photoshop, and optionally restart your computer for good measure. 

"Adaptive performance" saves power, but it is not the ideal setting for Photoshop. I don't work for Microsoft or Adobe, but here's my theoretical explanation for why this default setting is terrible for Photoshop:

Unlike 3D games or video editing where the GPU is consistently active, Photoshop only accelerates certain tasks, so there is a delay in the time you start drawing and the time the system directs more power to the GPU. This delay slows apparent drawing speed and responsiveness. By setting the GPU to "Prefer Maximum Performance," the GPU always has the power necessary to process your brush strokes. 

You could set "Prefer Maximum Performance" as a global setting instead of Photoshop specific override, but this might negatively impact your battery life in times when you're just surfing the web. 

Hope this helps you! We stumbled through many incorrect solutions and depressing workarounds before discovering this one for ourselves.

lemp say:
This sort of problem is difficult to diagnose through an on-line forum.

Is the computer a laptop or desktop/tower?  I'm guessing that it's a desktop/tower.

If the computer has to be rebooted to turn back on (rather than striking a key on the keyboard, for example), it sounds as if it is completely turning off which then causes the display (which still has power) to go into power saving mode because it no longer senses a signal from the computer.

When the issue happens, are there ANY lights on the computer that remain lit?

Assuming that you have some variety of Windows XP, see THIS and report what the settings are for the following:
  • On the "Power Schemes" tab, the entries for each of the 4 items
  • On the "Hibernate' tab, is the box checked or not?
  • On the "Advanced" tab, the setting for what happens when the power button is pressed

There are several reasons why a computer might spontaneously shut down.  The most common, with either laptops or desktop/towers, is overheating.  When the computer is on, do you hear the fan?  Is the computer in a dusty environment or do you have pets that shed hair?


Another reason for random shutdown is a failing or defective power supply or a bad connection between the power supply and the motherboard.  What is the make and model of the computer and about how long have you had it before this problem started?  Has the computer been dropped or handled roughly?


Depending on your comfort level, here are some things to try:

  • Check the Event Logs (both System and Application) for clues.  To see the Event Logs, click Start then click Run, then type eventvwr in the "Open" box and click OK.  Start at the top and go down until just before the entry that shows the reboot following an unexpected shutdown.  If you want someone to help you interpret an event, double click it to open its Properties, then click the "Copy" icon (just below the up/down arrows) and Paste the data into your Reply in this forum.
  • Turn off the computer, unplug it from the power strip/wall outlet, open the case, and use a can of compressed air to blow out all the dust bunnies (you can buy these cans at office supply stores like Staples or OfficeMax or at computer/electronics stores). Do NOT use a vacuum cleaner.  If you live in a low humidity area, be sure to take static electricity precautions (touch the case before you work inside the computer and don't walk over a rug to get to it).
  • Install a hardware diagnostic utility like Speedfan or HWMonitor to see if your fans are operating properly and your system temperatures are within normal limits
  • Take the computer to a good local independent computer repair shop.  Don't go to a BigBoxStoreUSA or GeekSquad type of place.  For best results, ask your friends and acquaintances for their recommendations/experiences for computer repairs.

When accessing a folder on a remote machine then you must do this:

  1. Create a share on the remote machine, e.g. by running this command on the remote machine:
    net  share  DriveC=C:\
  2. On the local machine, use the standard syntax for connecting to the shared folder on the remote machine:
    net  use  Q:  \\192.168.0.118\Drivec
    or
    net  use  Q:  \\192.168.0.118\Drivec  /user:JohnDoe  SomePassword

Your syntax will never work:
net use Q: \\192.168.0.118\C:

It's best to use an identical Windows account / password on both machines.

It happens due to some 3rd party apps.

Due to which it was stucked on initializing.

"End process" "regsvr32.exe" or other processes with that same name and you installation will start.

The "regsvr32.exe" is a Microsoft process.
