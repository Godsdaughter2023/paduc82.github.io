basd_ca say:

I have upgraded two 8.1 EFI installs and two Windows 7 installs to Windows 10.  I came to this thread because on install #4, I had the problem described here.

Then I realized why.  I had set my computer BIOS to boot from USB drive before the hard drive, so when the Windows 10 install rebooted, it booted from the USB flash drive installer and therefore was attempting a new install instead of continuing the upgrade process I started from within Windows 7.

In any event, I have found that the fastest way to upgrade is to create a folder on the computer in Windows 7 and copy all of the folders and files from the USB flash drive installer into the new folder on the computer.   Then I remove the flash drive and run "setup" from the new folder on the computer that contains all of the install files.

I have found this procedure solves the problem of asking for a product key and also the upgrade runs much faster than from the flash drive.

(BTW, I believe you can only invoke the upgrade process if you run "setup" from a running licensed windows 7, 8 or 8.1 desktop.  I don't think the installer will check for a licensed windows version if the installer is started by booting the flash drive, so if you boot the flash drive, your only option is a new install, not an update one.  I could be wrong on this, but I think that is what I found after several tries of booting the flash drive installer.)
