dusty harper [msft] say:

The MPSSVC service host much more functionality than just windows firewall. For instance, Windows Service hardening which is a windows protection of system services. It also host network isolation which is a crucial part of the confidence model for Windows Store based applications. 3 party firewalls know this fact and instead of disabling the firewall service they coordinate through public APIs with Windows Firewall so that they can have ownership of the firewall policies of the computer.  Hence you do not have to do anything special once you install a 3 party security product

dusty harper [msft] say:

Thank you for your input.  A router by itself does not protect a user from malicious content.  In fact an edge firewall also has known flaws in this space as well (no protection against malicious content originating within the local subnet).  Microsoft believes in the Defense in Depth Strategy.  Windows Firewall is our solution to protect the host from malicious content  from both inside the network and without.  As always there is a trade off between security and convenience.  With that said, Microsoft does not recommend turning off the MPSSvc Service.  In fact, should you choose a 3rd party firewall solution, they do not turn off the MPSSvc service, but rather work with the Windows Firewall and programmatically disable the functions they wish to replace.
