bruce hagen ~ mvp say:

Exporting from OE only sends messages to Outlook or Exchange.

Transferring data from Outlook Express to Windows Live Mail:

For messages:

Copy the *ENTIRE* OE message store folder to a flash drive. (Folders.dbx must be included). Place this on the Desktop or other location on the machine using WLMail. Open WLMail and: File | Import | Messages | Microsoft Outlook Express 6 and point to where you saved it.

OE Message Store Location:

In OE: Tools | Options | Maintenance | Store Folder will reveal the location of your Outlook Express files. Write the location down and navigate to it in Windows Explorer or, copy and paste it into Start | Run.

In WinXP, Win2K & Win2K3, the OE user files (DBX and WAB) are by default marked as hidden. To view these files in Windows Explorer, you must enable Show Hidden Files and Folders under Start | Control Panel | Folder Options Icon | View, or in Windows Explorer | Tools | Folder Options | View. 

For Addresses:

Open the Address Book in OE and File | Export | Address Book (wab) and save it to the Desktop. Copy to a flash drive. Place this on the Desktop or other location on the machine using WLMail.

Open the Contacts list in WLMail, (Go | Contacts on the Menu Bar), and File | Import | Windows Address Book (wab) and point to where you saved it. 

Note: If you use a CD or DVD instead of a flash drive, after placing on the new machine you must remove the Read Only attribute in Properties before you import.

Bruce Hagen MS-MVP [Mail]

    hairyfool say:

    Re-activation due to hardware changes can be cumulative i.e. a series of changes can not require re-activation then on more even minor one can tip it over into needing it. This is quite logical or you could gradually migrate a system over to a completely new system by making one change at a time.

     

    Having said that I didn't think that a USB device was included in the hardware profile so shouldn't be that final straw to force a re-activation. This rather indicates that something else is going on besides the fitbit issue. The fact remains that you are in a grace period at the moment and without knowing why it has kicked in it is difficult to say what will happen when it expires.

     

    The licensing system analyses numerous files for tampering as these are known to be altered with various "hacks" to bypass registration. I am not suggesting this is what you have tried to do, there are several examples of other causes the best known being corruptions in the Intel Rapid Storage driver although these most don't give a grace period but go straight into "illegal" mode.

     

    Noel is the "star" on this subject so do as he suggests and let him look at the results and give some guidance on where your problem lies.

     

    As for the Product ID question, no this not what you need. What they want is your encrypted 5x5 character key. This is the value on the Yellow COA label somewhere on the PC if provided with the computer or in the box if a retail pack. If the Windows is OEM the MGADiag report will probably show the last 3 5 character block different to what is on the label due to the way in which system manufacturers pre license Windows. If you have upgraded to Windows Ultimate then it will be the key you were provided for the upgrade.

     

    I hope this at least partly answers your actual question and shows that somebody has read it.

    Brian

    It is relatively straight forward since you are using and will be using WLM.


    You will need to copy a folder from where it is (the old installation) to another spot where you know it will be (for example to your desktop)

    The folder is in c:\users\your user name\app data (if it is not there see the note below)\local\microsoft\

    At that point you will see a folder called oddly enough "windows live mail"  Copy it to your desktop

    If you cant see app data it is because it is invisible atm.  To make it visible you will need to go into your control panel>folder options>view>Change to show hidden folders (see snip)


    Once you have the mail safeguarded on your desktop (or where ever) drag the folder from the desktop to the location for the new WLM (again it should be C;\.... just like above).  Just make sure WLM is not running at the time.  What wont be imported is the passwords and rules.

    The accounts and data should now be in the new WLM when you open it.
