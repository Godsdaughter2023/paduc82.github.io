a. user say:

Be careful when deleting duplicate files.  Some files are INTENTIONALLY duplicated in Windows Vista very much unlike prior versions of Windows (including your personal data) using things called junction points: http://msdn.microsoft.com/en-us/library/bb968829(VS.85).aspx.  If you delete these duplicate files, some of your programs will cease to function (and some of your data will be lost and other files will refuse to open).  The file winsxs keeps track of duplicate system files and various versions of files throughout the system and if you delete these files, it will mess up the connections and could cause some programs to stop functioning or the system to crash.

What I'm saying is be very, very careful that you're deleting TRUE duplicate files of yours that are in C:\Users\<username>\Documents or ...\Music or ...\Pictures or ...\Videos.  Do NOT delete anything in the Documents and Settings Folder (stay away from that junction point).  Don't delete duplicate system files in different folders because they may be necessary for programs to work which are looking for the files in a specific location.  Also remember that downloaded music that you've paid for requires a key (that was put in the registry when you purchased the song) and if you have two copies make sure you don't delete the one for which the key applies (or save it on another machine or give it a slightly different name unless you're sure you want to delete it).

These programs are going to look for exact duplicates - not drafts of a final copy that you no longer need (it won't find them because they're different sizes, different dates, and different names).  Those will have to be done manually and i imagine that's the vast majority of the "duplicates" you have on your system (unless you did some copying and pasting of files to different folders until you lost track of how many copies you made and where they all were - then the programs will help.

Good luck and I hope this helps.

Your message does not conform to RFC2822 standard


There are any number of reasons why the server might return this error. A common one is a mistake in one of the addresses the message is being sent to. To check this, click Work offline and remove all unsent messages from the outbox (below the account folders in the folder pane). Then compose a new test message with a unique subject (like Test 1) and send it to yourself.

  • If the message is sent and received without problem, you know that the fault was with the message(s) that failed to send earlier. Check the addresses carefully for illegal characters like spaces, commas or periods at the beginning or end.
  • If the message is not sent, something else is wrong. Please click on the error message to select it, then use right-click to copy it and paste it verbatim into your reply here.
