xb1r apollo say:

Wireless Security

Login Information

 

  • Gateway: 192.168.1.1
  • Username: root
  • Password: 12345

 

 

How to Find Wireless Passphrase (WPA/WEP)

 

  1. Open a web browser.
  2. Log into the router by entering the default gateway (192.168.1.1) into the address bar of the internet browser.
  3. When prompted, enter in "root" for the user name and "12345" for the password.
  4. Select "Wireless" from the menu to the left.
  5. Select "Security" from the sub menu.
  6. If the "Network Authentication" is showing "802.1X" the passphrase will be based on the "Current Network Key". (By default it will be key 1, and the wireless passphrase will be "Network Key 1".)
  7. If the "Network Authentication" is showing "WPA-PSK", "WPA2-PSK", or "Mixed WPA2/WPA-PSK", the wireless passphrase will be labeled "WPA Pre-Shared Key".

 

 

Changing Wireless Security Type/ Disabling Wireless Security

 

  1. Open a web browser.
  2. Log into the router by entering the default gateway (192.168.1.1) into the address bar of the internet browser.
  3. When prompted, enter in "root" for the user name and "12345" for the password.
  4. Select "Wireless" from the menu to the left.
  5. Select "Security" from the sub menu.
  6. Select the desired security type from the "Network Authentication" drop down menu. (To disable security, select "Open" and then "Disabled" from the "WEP Encryption" drop down menu.)
  7. Select Save/Apply.

tryx3 say:

I make a full System Image each week [I alternate between two external drives so I have two chances of recovery on each one].  I move/delete the old images before making the new ones so that each one is pure.  I know that I can just make a new one and that it gets amalgamated with any old ones that are in place but I have never done a comparison to see if there is any benefit to just allowing them to lump together [perhaps an image is incremental so the updated image would be quicker & use up less disk space - I simply do not know]. 

 

To make a Windows System Image go to the starting screen of Windows Backup and look for the link on the left-hand side of the window.

 

Images are made in H:\WindowsImageBackup  or F:\WindowsImageBackup etc [you can use any external drive or even a partition on your hard drive but that would be at risk if the drive failed].  You can move them around between drives & even stock them up into libraries in other folders but the recovery procedure will only find them if they are back in a *:\WindowsImageBackup folder.  I keep older ones in H:\OldImages\20130427, H:\OldImages\20130413 etc then just delete older ones as the drive fills up.

 

If I need to restore a System image then I use this procedure [and this has always worked correctly - it never, for example, overwrites more than the system partition - but you will not have that worry if your data drive is a separate disk anyway]: 

Recovery method 12 - Windows Backup System Image restoration using Windows 7 Installation Disk          [PDF version]

If you already have a Windows 7 Installation Disk then you can use that.  If you don't then see Replacing pre-installed with regular for the procedure for making one.

 

 

I also make an Acronis True Image system image every five weeks.  This is merely a last fallback in case of complete hard drive failure rather than the system screwed up problem that has been my normal [if infrequent] experience of needing to restore.  An Acronis system image can cope with more hardware changes than a Windows Backup system image so this has more flexibility.  When I wanted to change my hard drive a couple of years ago, I bought a larger drive from a different manufacturer.  My system seemed to restore the image & then to work okay but Windows Update would not run and neither would my attempts to run driver updates for the hard drive controllers causing the problem.  I have since discovered that the Intel® Driver Update Utility could have sorted out this driver update for me and that this would have corrected the Windows Update problem.  The only reason that I do not use just Acronis True Image is that it had a problem with my single disk having both the system partition & the data partition on it.  You would not find that problem as you have separate disks rather than partitions.
