oljim2001 say:

After many tries at the solutions listed here I found a way to fix my problem. I continually got a failure basically saying the system could not create a shadow copy or that my drive had a failure. It seems with Windows 7 you have to configure some options that are not apparent to users.

1. go to control panel/system/System Protection and select your C: drive.

2. Select "configure" and make sure "Restore system settings and previous versions of files" is on.

3. Set Max Usage to at least 2%

4. Apply

5. Select your USB drive you will be using for backup and again Configure it for protection and set the file size to at least 18%.

6. apply

7. back in the control panel go to Administrative Tools, open Services.

8. Scroll down to Windows Backup and make sure it is set to Manual and not started.

9. Scroll to Volume Shadow and make sure it is also set to Manual and not started.

10. Go back to the control panel, select Backup and Restore.

11. Click "Change Settings" and select the drive you will be backing up to.

12 Run backup letting the system do it's thing and all should work.

  1. From the Start menu, type create a restore point.
  2. Select Create a restore point from the search results.
  3. Choose Create, and then follow the steps to create a restore point.

Once you have created a restore point, you can proceed with resetting the Windows Update components by following the steps below:

  1. Open a command prompt. To do this, use one of the following methods, depending on your version of Windows:
     
    • Windows 10: Press and hold (or right-click) the Windows logo button on the taskbar and then select Command Prompt (Admin). Select Yes when the User Account Control box appears.
    • Windows8.1 and Windows 8: From the Start screen, swipe in from the right side to display the charms, select Search, and then search for cmd. madeni k n say:

Hi,

 

When you get this message while trying to safely remove a USB drive, it is basically a lockdown of that USB device by some processes running in the computer. If we can find out which process is locking that device, we could also find out which Service is associated with this issue in the computer.

 

In order to check this, you can use a program called Process Explorer. You can download this program from the following link. Once downloaded, extract the contents of the downloaded zip to a folder on your desktop. Inside that folder, you will see a file by the name procexp which is the executable for Process Explorer. The download link is:

 

http://technet.microsoft.com/en-us/sysinternals/bb896653

 

First of all, plug in the USB drive to any of the USB ports. Then, once you have the extracted executable for Process Explorer, right click on it and then click on Run as Administrator. Please allow any User Account Control prompts that may come up. Once Process Explorer is running, please follow these steps to search for processes using the USB device at the moment:

 

1. Click Find, and then click Find Handle or DLL on the menu options on top.

2. Then in the pop-up box that will turn up next, type in the drive letter for the USB drive connected to the computer (for example G: or H:) at the moment and click Search.

3. It should then display all the process instances that is using that USB drive at the moment along with the corresponding Process IDs (PID).

 

Now, once you have the names of the processes using the USB drive at the moment, you can terminate them by right clicking on them and then clicking on Close Handle. Once that is done, you should be able to eject the USB drive safely.

 

I would ask you for the names of the processes locking the drive in your computer. That should give us an idea of what steps we may have to take next.

 

Moreover, for the other issue you have mentioned, wherein the computer does not automatically and correctly detect the printer when plugged into the computer, I would suggest you to check if the Plug and Play service in your computer is set to Automatic or not. Here is how you can do that:

 

1. Press the Windows and the R together on your keyboard to pull up the Run box.

2. Type in Services.msc in the Run box and hit Enter from the keyboard. Allow any UAC prompts that may come up next.

3. Once it opens up the Services window for you, scroll down to Plug and Play service and double click on it.

4. In the Properties box that will come up next, the Startup type should be set to Automatic and the Service Status should be Started. If not, please set them to these values.

5. Click Apply and OK, close all the windows in the computer and restart the computer once.

 

Once these steps has been performed, please check if the issue is still the same. Do let us know the results.

 

I hope these steps sets you off in the right direction in order to troubleshoot this issue. Do reply if you still need assistance, we will be happy to be of assistance.
