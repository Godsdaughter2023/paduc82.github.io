frederik long say:

What exactly is going on? Did the upgrade break something that the rollback couldn't restore?

This is probably a bug in the roll-back process. You can fix it like so:

  1. Logon under an admin account.
  2. Copy the problem task files from c:\Windows\System32\Tasks to a new folder such as  c:\MyTasks.
  3. Rename the files by giving them an .xml extension.
  4. Launch the Task Scheduler GUI.
  5. Delete the bad tasks.
  6. Use the Import function to the right to import the tasks from c:\MyTasks.
