You're correct.  Although Windows Update is very often the culprit when svchost.exe uses excess CPU cycles, the Windows Update service (wuauserv) is in a different "service group" than the one you show. 

You can dig a little deeper.

At a time when you are experiencing the high CPU usage, open Task Manager, find the instance of svchost.exe that's using the CPU, note the PID, and right-click > Go to service(s) to get to the screen you posted (so you can confirm it's the same collection of services).

Then go to the "Performance" tab and click on the "Resource Monitor" button.  Select the "CPU" tab and determine which of the services is using the CPU.  You can sort on the PID column or the CPU column by clicking in the header row.

That may give you (or us) some idea of what's going on.
