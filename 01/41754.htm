When you say that "none of the options work," what options have you tried?  Is there an option for setting the "Encryption type" to "TKIP/AES"?  If so, does that setting work?

This message appears to indicate that when you configured security in your DPC3941 modem/router, you configured something other than WPA2-Personal with AES.

  1. Connect a computer to the DPC3941 using an Ethernet cable
  2. Open a web browser (Internet Explorer, Firefox, Chrome, etc.)
  3. Type the following in the browser's address box and press Enter or click the arrow:  http://10.0.0.1
  4. When prompted, under admin for the User Name and password for the Password (unless you changed these values)
  5. In the configuration utility, go to Gateway > Connection > Wifi
  6. You can check the "security type" of your wifi connection here.  This is called "Security Mode" here:  http://media2.comcast.net/anon.comcastonline2/support/help/faqs/wireless_gateway/wg3/WG3_UserGuide_HOW5220.pdf

The "Security Modes" or "Security Types" available on your modem/router are:

  • Open
  • WEP 64 bits (5 Ascii characters or 10 Hex digits)
  • WEP 128 bits (13 Ascii characters or 26 Hex digits)
  • WPA-PSK (TKIP) (8 to 63 Ascii characters or 64 Hex digits)
  • WPA-PSK (AES) (8 to 63 Ascii characters or 64 Hex digits)
  • WPA2-PSK (TKIP) (8 to 63 Ascii characters or 64 Hex digits)
  • WPA2-PSK (AES) (8 to 63 Ascii characters or 64 Hex digits)
  • WPA2-PSK (TKIP/AES) (8 to 63 Ascii characters or 64 Hex digits)
  • WPAWPA2-PSK (TKIP/AES) (8 to 63 Ascii characters or 64 Hex digits)

Other than "Open" the rest of these options define different security standards and different methods of encryption.  WEP is an obsolete standard and you should not use it.  WPA is an early standard.  WPA2 is an improvement to WPA. 

The "PSK" refers to "Pre-Shared Key" which simply means that you have to know the key in order to connect.  WPA-PSK and WPA2-PSK are the same as WPA-Personal and WPA2-Personal, respectively

TKIP was an encryption method meant to improve the security of WEP.  It turned out that TKIP had vulnerabilities, so the industry moved to the AES encryption standard.

The options are listed above in order of increasing security.  Thus, if your equipment is capable of it -- and yours appears to be so -- you should use WPA2-PSK (AES).

The last two options in the list are intended to allow you to have a mix of wifi devices.  That is, if you have an old computer that had one of the early implementations of WPA2 that only used TKIP encryption and you have a newer computer that can use WPA2 with AES encryption, the option WPA2-PSK (TKIP/AES) would allow both of them to connect to the router.  The last option allows for even older devices that are only WPA capable.

All of the above is also explained here:  https://www.howtogeek.com/204697/wi-fi-security-should-you-use-wpa2-aes-wpa2-tkip-or-both/

Bottom line:  if you have only the one computer on which you have the problem, leave it set the way it is and make sure that the modem/router is configured the same way, that is:  WPA2-PSK (AES) (with a password comprising 8 to 63 Ascii characters).
