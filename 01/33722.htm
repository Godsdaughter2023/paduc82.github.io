I have not yet found a reliable fix - but there are a few points I think are being missed.

The majority of cases list the SrtTrail.txt report created by the Win10 Startup Repair which reports: 

Root cause found: 
---------------------------
Boot critical file c:\efi\microsoft\boot\resources\custom\bootres.dll is corrupt.

Repair action: File repair
Result: Failed. Error code =  0x57

The odd part of this error is the "custom" folder location - this is not part of the normal folder structure. The bootres.dll file normally resides in the "resources" folder with the BCD file in the folder above (boot)

What the error is reporting is that the bootres.dll file is missing (rather than corrupted)
On the systems I have checked the "Custom" folder does NOT exist - thus the bootres.dll cannot be present at this location and is declared "corrupt" by the Startup Repair utility

The bigger mystery is why the System thinks the file should be located in a "Custom" folder in the first place.  


I think the idea that (the thread starters) damaged Win10 system is EFI booting from the C: drive is wrong. The drive letter (in this case c:) varies from case to case but is (I think) an artifact (not a real drive letter).
On a standard Win10 EFI installation the folder path shown doesn't exist on the C: drive 
- there is no C:\EFI folder, but it does exist on the hidden EFI boot partition (the FAT32 100-260MB hidden system partition) 

If you examine the Volume and Partition info using DISKPART and check the BCD boot file   
- you find that in almost all cases the system is booting from the hidden EFI System partition 
- and not from the drive letter state in the error message.

You can list the basic Boot Configuration Data (BCD) using the BcdEdit command line utility 
either from within a working system or from the Recovery Environment (RE) 

On a normal Win10 EFI boot system the Windows Boot Manager section shows: 

Windows Boot Manager
--------------------

This folder is where the BCD file also resides alongside the BOOTMGFW.EFI file 
The normal Bootres.dll file resides in the Resources sub-folder (never "Resources\custom")

The final confusion is the partition/volume that is being used for the boot
The BCD device line shows "partition=\Device\HarddiskVolume2"
Which should be pointing to the hidden EFI system partition (or the Windows volume, C:)
If you use DiskPart to list Volumes the number is wrong
However if you instead list Partitions it correctly points to the EFI partition  
Microsoft appear to be using Partition numbers not Volume numbers for the \Device\HarddiskVolume2
