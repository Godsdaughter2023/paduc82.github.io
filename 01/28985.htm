usasma say:
Just FYI - sometimes lower level drivers (such as AMD Overdrive or Gigabyte EasyTune) can cause systems to act as if there is a hardware problem.  So, we're still not certain if this is hardware or software - so please continue to run the diagnostics suggested previously.

AMD OverDrive (AODDriver2.sys) is either a stand-alone application, or a component of the AMD VISION Engine Control Center.  This driver is known to cause BSOD's on some Win7 systems.  If doesn't appear to be installed on your system, then it may be a part of the Gigabyte EasyTune6 program (another one that's known to cause BSOD's on Win7 systems).
*******
Please uninstall all AMD/ATI video stuff from Control Panel...Programs...Uninstall a program
Then, download and install a fresh copy of the ATI drivers from http://www.ati.com (in the upper right corner of the page)
- Select the "Custom" installation option and don't let it install AMD OverDrive or the VISION Engine Control Center (if present)
Then use the ATI uninstall manager to uninstall the Catalyst Control Center (CCC).  This is a separate issue and isn't required for the AMD OverDrive removal.

If the device (AODDriver or AODDriver4.01) remains a problem in perfmon /report, open Device Manager, select the "View" item.
Then select "Show hidden devices" and scroll down to the Non-Plug and Play Drivers section.
Locate the AODDriver entry, right click on it and select "Uninstall".  Reboot for changes to take affect.
*******
Sometimes the driver remains and continues to cause BSOD's.  If this is the case for you, post back and we'll give further instructions for safely removing it.

If overclocking, please stop. Remove the overclock and return the system to stock/standard values while we're troubleshooting. Once the system is stable again, feel free to resume the overclocking.

Also, please uninstall the Gigabyte Easy Saver - mobo power utility driver and the Gigabyte EasyTune6 program (it may contain the AMD Overdrive program).  Both are known to cause BSOD's on some Win7 systems.

I am suspicious of the Applecharger program also, but have no firm evidence for it.  Please just keep an eye on it for now.

If that doesn't stop the BSOD's, please run Driver Verifier according to these directions:  http;//www.carrona.org/verifier.html

Unfortunately none of the Home versions of Windows are supported on Hyper-V.

 

Hyper-V Overview:

Windows 7 support - Ultimate, Enterprise, and Professional editions (32-bit and 64-bit). Upgrade the integration services after you set up the operating system in the virtual machine.

 

Windows Vista support - with Service Pack 2 (SP2). Business, Enterprise, and Ultimate, including N and KN editions. Install the integration services after you set up the operating system in the virtual machine.

Hyper-V:

http://en.wikipedia.org/wiki/Hyper-V#cite_note-hyperv2012technetoverview-31

 

No support for home editions of Windows.
The management tools for Hyper-V are not compatible with Windows Vista Home Basic or Windows Vista Home Premium. This is also the case for Windows 7 home editions (Home Premium, Home Basic, and Starter).

techlady-48 say:

I found my own answer for  this and want to share it.  The computer had a 2011 Acronis True Image on it and I had decided it was no longer needed (since Win 7 Backup and Restore is now an acceptable alternative).  So I had uninstalled the Acronis software - and then started getting no response when I selected "Backup and Resore" on Win 7.  It turns out that there was a now-fixed bug in the 2011 Acronis software that didn't reset the registry properly when it was uninstalled.  It removed the Acronis from the registry but didn't reset the registry to point to Backup and Restore.  So when I tried to do the Backup and Restore, there was nothing in the registry to make it execute. 

I found a fix on the Acronis support site that fixed my problem completely - Look for 14741: Acronis True Image Home 2011: DIsabling "Integrate True Image into Windows" Does Not Work".

Thanks for trying to help me!  I spent hours trying to find this and was relieved to find the solution on the Acronis website.
