¡firedog say:

That's the way IMAP works - all messages are left on the server, and the folders in Windows Live Mail mirror what's on the server. If you delete a message from the inbox in WLMail, it will also be deleted from the server at the next sync.

If you need to keep messages but you've run out of space at the server, move the messages you want to delete from the server to a folder under Storage folders. Just select the messages in the inbox in WLMail, right-click and select Move to folder. You can create a new folder under Storage folders in the same manoeuvre.

considerate_guy say:

Project files (.mswmm files from versions 2.x and 6.0, or .wlmp from versions 2009-2012) are NOT movies. They do not contain actual media.

The project files are more like the save state of your editing session and once you are happy with the edits in Movie Maker, you should finish the process by rendering a final copy of your movie in .wmv or .mp4 format.

Your .mswmm files came from an older version of Movie Maker. They should be readable in Windows Movie Maker 6.0.

If you are able to open the project file in WMM 6.0 (assuming all the corresponding media files are also on the harddisk in their original locations) you should be able to create an actual movie using File > Publish Movie...

The default output in WMM 6.0 is .wmv which can then be viewed in your favorite media player.

yangzheng_msft6300 say:

Hi Danny,

I understand that you want to use different domain as suffix for Skype for Business sign-in address and email address. However, this difference between Skype for Business sign-in address (SIP address) and email address (default SMTP address) can cause some issue when using Skype for Business client and Outlook client.
Different SIP address and default SMTP address could lead to Skype for Business or Outlook client sign in issue. Also it could lead to the issue that you can’t create Skype for Business meeting in Outlook client and you may not be able to attend Skype for Business online meeting.

So based on above, it is not suggested to set Skype for Business sign-in address (SIP address) different from email address (default SMTP address).

Yang

datwater say: Solved my own problem.  

Permissions on:

C:\ProgramData\Microsoft\WLANSVC and sub folders had been messed up.

I made sure local administrators could read and write on that folder and propagated the permissions down in the folder.

After that Windows 7 started saving network profiles like it should.
