keith a. miller say:

Interesting. I had been able to get them to show by creating the two values under HKCU\...\NewStartPanel and setting their data to 0.

I did find another way to make them display.

Look in the registry under:
HKEY_CURRENT_USER\SOFTWARE\Classes\CLSID

This key will have a number of CLSID subkeys. The Libraries key, {031E4825-7B94-4dc3-B131-E946B44C8DD5}

should already exist. Check for a value named System.HideOnDesktop in the right-hand pane.  Again, it's a DWORD type.  A value of 1 should hide the folder, a value of 0 display it.  If it doesn't exist, create it & set it to 1.

For the Homegroup folder, I did not have an entry for

{B4FB3F98-C1EA-428d-A78A-D1F5659CBA93} and had to create it, but I've never set up a Homegroup on this machine.  Once I created it & the System.HideOnDesktop value, 0 forced it to display, so create if you need to & set the value to 1.

Hope this does the trick.

Keith
