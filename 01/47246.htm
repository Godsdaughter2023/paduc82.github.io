It is not a bug. It depends on many factors.

The Frequency is measured in MHz and higher numbers potentially indicate faster access to the information stored in memory. This is a key factor if your graphics card shares your RAM. Latency describes the delay between a request and execution of the task, meaning lower numbers are better.

Together, Frequency and Latency affect the speed of your RAM.

magikseven say:

It is actually a bug occurring primarily with many AMD boards. CPU-Z and bios will report accurate speeds (CPU-Z always will report half speed and thats correct) yet the task manager which usually reports full speed is reporting only half speed also. 

Its a confirmed windows bug that is caused by the smbios info not being read right. It does not impact performance or memory allocation and microsoft reccommend using another software to measure speed.

There are several causes on why the results of a saved movie has audio but no video, please check the following:

1. Ensure you have installed the latest version of Movie Maker:
http://windows.microsoft.com/en-US/windows-live/movie-maker-get-started
• For Windows Vista - The current 2011 version: 15.4.3555.0308
• For Windows 7 / 8 - The current 2012 version: 16.4.3505.912
• Windows Live Movie Maker is not supported on Windows XP

2. Try making movies of a shorter duration - If you have a large movie project, splitting the video into smaller projects may help. Open the whole project, split them to multiple parts. Save them as a movie individually. After saving each part as a movie, import and then combine the saved movies in Movie Maker.

3. Check that the files you are using are of a supported format– Some file formats are not supported by Movie Maker. A list of supported formats can be found here: http://windows.microsoft.com/en-US/windows-live/movie-maker-file-types-faq

4. Ensure that you have the proper codecs to play your videos and audio files - Audio andVideo require codecs to play the file and it’s important to have the right codecs installed or updated in your PC.

• Try running the Microsoft FixIt tool for codecs: http://support.microsoft.com/mats/video_freezes_or_crashes/

• Media info will be able to identify the codec required for your audio and video files, and will provide a link to the website for the required codec. You can get it at http://mediainfo.sourceforge.net/en
a. Open Mediainfo, click File > Open
b. Select the movie file you cannot play with video, then click Open
c. Click View tab then select Sheet to view all the required codecs for this video

d. Next to drop down codec bar is the Web button that when clicked will direct you to site where update to date codec is available to download.
Note: If can't find the codec you need you may convert the video to a supported format (eg.wmv) and then add it to your Windows Live Movie Maker project.

5. Ensure that your video card drivers are up to date- Movie Maker makes extensive use of the video card GPU (graphics processing unit). The display driver affects the reliability, performance, and compatibility of the video cardwith various software.

To determine the version and video card driver date, run the DirectX Diagnostic Tool (dxdiag):

DirectX Diagnostic Tool
• Click on Start button and type “dxdiag” in the Search programs and files box and press Enter key on the keyboard.
• Click on the Display tab and you will see the driver details including the driver version.
• If your video card driver seems outdated please go to the video card driver manufacturer and see if a newer version is available.
