Hello,

Thank you for posting your query on Microsoft Community forum.

As per the issue description you are getting some error message at each start up which is related to FDResPub service.

Function Discovery Resource Publication (FDResPub). This service publishes this computer and resources attached to this computer so they can be discovered over the network. If this service is stopped, network resources will no longer be published and they will not be discovered by other computers on the network. 

The Function Discovery Resource Publication service runs as NT AUTHORITY\LocalService in a shared process of svchost.exe. Other services might run in the same process. If Function Discovery Resource Publication fails to start, the error is logged. Windows 8 startup proceeds, but a message box is displayed informing you that the FDResPub service has failed to start.

Dependencies

Function Discovery Resource Publication won't start, if the following services are stopped or disabled:

  • HTTP Service
  • Remote Procedure Call (RPC)

If Function Discovery Resource Publication is stopped, the HomeGroup Provider service fails to start and initialize.

You must be logged on as an administrator to perform these steps. Try to check these services are running or not.

  1. Press Windows key + R and type services.msc in the search field or run window and press ENTER.

  2. If you're prompted for an administrator password or confirmation, type the password or provide confirmation.

  3. Find the Function Discovery Resource Publication service, right-click it, and then click Start and make sure that start type should be Automatic.

  4. If Start type is not set as Automatic click on properties and then change the start type to Automatic.

jk_0141 say:

Brother 7420 built in driver

I have read an attempted the proposed "fix" for installing the driver for my Brother 7420 printer after upgrading to Windows 10.  

The Brother driver site does not have a driver to download - it refers to built in drivers in Windows 10 that should allow the printer to work.  Instructions call for disconnecting printer, then reconnecting, and the drivers should load.

The printer is discovered and shows on the Control Panel (Devices and Printers) - but it does not work.  Nothing comes out of the printer.  The print queue does not show pending jobs - so from Win 10 perspective - it worked, but still nothing.

I have uninstalled printer, I have run the updates, I am now at a loss as to where to go next.  Any help is appreciated.

JK 

a. user say:

Hi,

Thank you for posting your query in Microsoft Community. Appreciate your interest in using Windows 10.

I suggest you to follow the methods below to fix the issue:

Method 1: Run the Printer Troubleshooter.

  1. Type Troubleshooter in search box and click Enter.
  2. On the top left corner click on “View all”.
  3. Select Printer from the list and follow the prompts for printer troubleshooting.

Method 2: Update the printer drivers.

  1. Press “Windows key + R” and type “decmgmt.msc”
  2. Expand the printer drivers.
  3. Click on “Properties” and select the “driver” tab.
  4. Click on “Update driver” button and click “OK”.

Hope the information is helpful. Feel free to write to us for any other assistance with Windows, we’ll be glad to assist you.
