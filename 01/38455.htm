joe men say:

Hello,

There are three factors that affect FPS when you play games:

  • CPU - A general rule of thumb for this is the higher the frame rate, the higher the CPU requirement. For the Windows Media Encoder to keep up with the data that is being sent to it from the capture card, a fast CPU is required. To keep synchronized with the audio stream, the Encoder will start dropping video frames to keep up with the data input. When frames are dropped the Current FPS statistic is lower than the Expected FPS statistic on the monitor panel. Note that some capture cards are only able to capture at 15 FPS, while others are able to capture at the full 30 FPS. Consult your card's manufacturer for specific details about your card.
  • Content Type -As stated previously, high-motion content requires more resources than low-motion video. Therefore, it is more important to properly select a CPU to handle the task, and to choose the right profile and key frame rate. If the encoding process cannot keep up with the content, it starts dropping frames. One exception to this is static content. If the content is a stationary object (such as a wall or an empty room), the frame rate may be lower because the Encoder does not need to capture any deltas.
  • Selected Profile - Each profile has frame size and frame rate determined. If you select a profile that the Encoder cannot keep up with, then select a lower quality profile. Multiple Bitrate profiles can also affect the Encoder performance. If the Encoder starts dropping frames, you may need to remove a stream from the profile, or adjust the frame size.

We recommend that you get in touch with the support pages of the game that you are playing for assistance on how you can achieve your desired FPS.

How to temporarily prevent a Windows or driver update from reinstalling in Windows 10

- On Line games often to have lag issues. Sometimes due to too high 'ping' (you are too far away) and other times

because of deficient game servers for the number of players on at the time.

One On Line game at the moment that has some serious lag is UbiSofts  Steep  (on the Aus server). They force players to play this game on line, regardless of if the user wants to.  Then they cheapskate on servers.

The game runs smooth until too many other players start showing up on the region of the map you are skiing on.

Too many player and the game gets freeze frame length lags.

Pushing for too much eye candy can cause inconsistency in performance, depending on the game in question.

Stutter can be hard to pin down as there are a lot of contributing factors. Including, I believe, software/driver conflicts, eg- Skype

and other tools/programs you may have open while gaming. There are heaps of articles/forum thread on the subject.

I believe it pretty much boils down to getting the right balance between monitor and game frame rates by whatever means

works for you.

Using a monitor with a 120/144Hz frame rate is a big help too.

With some games performance is affected more by a particular graphics setting which may not have been as well

optimised as the other settings. I have one game that has a number of individual graphics options, and all but one

could be set to High/Ultra - Dynamic lights. Big fps hit using this option over medium.

Another game went from smooth to slide show if AA was boosted  from 2X > 4X.

- Trying to force just a bit too much eye-candy can cause 'random'  lag. More so with games that have a walking

and driving component. Driving at speed will use more VRAM and will cause a game that was otherwise smooth

to lag or go jerky.

Even without having a driving component, games like Fallout 4 will overwhelm 3GB VRAM at times such as large

battles, if you force all high settings.

- To  V-sync or not to V-syn.... Different games are affected differently, but disabling V-sync can help with stutter.

Of course if your game is running over 100fps on a 60Hz monitor you may get 'tearing', so sometimes you have to

choose the one that bothers you the least.

Stutter seems to be more noticeable for some people, and would be more of an issue with those who spin and turn

quickly and frequently. Particularly while running forward. What would be motion blur becomes stutter instead.

The same happens when I select my monitors (so called) Blur Reduction feature. Blur reduces, bad choppy effect is introduced.

BCCode: 3b    0x0000003b

3b is usually a graphic's driver (display adapter/video driver), other possibilities
include but are not limited to antivirus/antispyware/security programs and even
memory (mapping).

Try removing and reseating all cards, memory, and cables (both ends when possible) - actually remove and replace - do not just snug (on a PC) and clean out the dust bunnies and vents. On a laptop about all that can be done is reseat the memory and clean the
vents. (Of course  remove ALL power prior to opening the case.)

REBOOT this will refresh the driver stack.

if 3rd party with their own software and drivers and any other major device drivers
you have.

Download - SAVE - go to where you put them - Right Click - RUN AD ADMIN -

installing actually shows up. This is because some drivers rollback before the
