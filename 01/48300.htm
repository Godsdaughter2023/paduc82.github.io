considerate_guy say:

Here's a little bit of history:

Windows ME --> Windows Movie Maker 1.0 (used .mswmm files)

Windows XP --> Windows Movie Maker 1.1, 2.0, 2.1 (used .mswmm files)

Windows Vista --> Windows Movie Maker 2.6, 6.0 (used .mswmm files)

Windows 7, 8, 8.1, 10 --> Windows "Live" Movie Maker 2009 to 2011 and Movie Maker 2012 (uses .wlmp files)

WMM 2.6 can open .mswmm project files from WMM 1.0 to 2.6

WMM 6.0 can open .mswmm project files from WMM 1.0 to 6.0

WLMM and MM2012 can only open .mswmm project files from WMM 6.0

So the best bet is to install Windows Movie Maker 2.6 or 6.0.

How to install Windows Movie Maker 6.0 on Windows 7, 8, 8.1 or 10

As you have noticed, you may find that your collection doesn't build correctly. That's because WMM 2.x and WMM 6.0 used the same collection files, but treated them differently. So you may just have to wipe out the collection as you did. That being said, the collection isn't necessary to open a project file.

The next complication is going to be file types. The "invalid pointer" error usually indicates you have some MPEG-2 files in your project. Microsoft only provided the MPEG-2 codec in Windows Vista and Windows 7. They did not carry it forward to Windows 8, 8.1 or 10. I suspect this is why you are having problems.

I'd start by loading WMM 6.0 and then seeing if you can open the project files. If not, please come back and provide a screenshot of the exact error message. You can add it using the "Insert image' button when you are typing a post here.

Update:

One further clarification. The .mswmm files are *project* files. They are used when you need to make further edits. Is that really what you need to do? Are you trying to edit the videos from 2006 again? Didn't you save actual movies (.wmv files) after making your edits? If so, I would just find *those* files and you don't even need any version of Movie Maker. The saved movies can be played back on any PC even if it doesn't have Movie Maker.
