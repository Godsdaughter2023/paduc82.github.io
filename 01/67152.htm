matthew_omega say:

Hello Stephen,

 

I took note of the fact that you do not have a boot.ini file.

 

Windows XP is capable of booting without a boot.ini file as long as the boot drive is on multi(0)disk(0)rdisk(0)partition(1)\WINDOWS.

 

If the machine has a floppy drive, it may prove beneficial to create a bootup floppy to try the steps mentioned in the article you found- that way you may alter the boot.ini file without fear of being unable to boot suddenly.  You will need to create a new boot.ini file using the following article:

http://support.microsoft.com/kb/289022

 

You should be able to use the Sample boot.ini text with some degree of success- there is always a certain amount of trial and error with this. 

 

That being done, you can create the boot floppy that will allow us to experiment with the boot.ini file:

 

1.     Format a floppy disk on another Windows XP machine with the same service pack.

2.     Copy the NTLDR, NTDETECT.COM, and BOOT.INI files to the disk.
These will be located at C:\ <or whichever drive Windows is installed to> and will most likely be hidden.

3.     You may then use this floppy to attempt a boot to the afflicted machine.
Note that you will need to work with the boot path in the boot.ini until it works.

You may also create this floppy from the your Windows XP machine before any problems arise and have it available as a "just-in-case measure."

Let us know how it goes.

 

Best regards,

Matthew_Ha
