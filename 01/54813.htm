This continues to be a problem, all right, but let me share something I discovered that, to me, makes no sense at all, but somehow worked.  By the way, this seems to be a situation here, not only with Vista, but with at least one computer I have running Windows 7.

The subject computer(s) got stuck in that situation where the Windows Update icon appeared in the Notification Area, but when clicked-on, the WU pane said that everything was up-to-date and no new updates were available.  At this point, clicking on Check for Updates brought up the 'Searching' bar, which would simply run and run with no results.  Closing Windows Update through Task Manager would stop the search and remove the icon from the Notification Area, but within minutes the icon would pop up again.  This said to me that the computer did get some sort of message that updates were available; either that, or it simply times-out into that mode of operation monthly or so.

At this point the scenario can go either of two ways.  The computer can continue to Search for Updates, or the Updates pane can say, "X Important Updates Available" or whatever, but even then they never seem to download.  A 'Downloading' bar comes up, but the pane tells you that 0% are downloaded, even after a couple of days' running.

HOWEVER, under either one condition or another (Searching for Updates -or- Downloading Updates), at some point a little exclamation mark appears next to Shut Down in the Start menu.  Hovering over it, a message appears, saying that updates will be installed at shutdown.  Keep in mind that at this same time, Windows is still either 'Searching' or 'Downloading' with no corresponding activity indicated.

Once that exclamation mark appears, if you do shut down, sure enough updates are installed.  The usual warning after logging-off (Installing Updates - Don't Unplug Your Computer), and when next you boot (Finishing the Update Process 1 of 3, XX%), come onto the screen.  And when Windows finishes restarting, sure enough, updates are shown to have been installed 'Successfully.'  In fact, at that point there may be a couple of additional updates shown as ready to install, and they do indeed install properly, just like back in the old days.

This "phantom update" has happened to me twice now on my Vista system, in September and in October, and on one of my Win7 machines just in October.

malik yasirnawaz say:
Hi,

To terminate this BitLocker recovery loop, BitLocker will first need to be suspended from within WinRE. To do so, follow these steps:

- On the page where you are asked to enter the recovery key, choose the Skip this drive link at the bottom. You should be presented with a menu that will let you get to a command prompt (The sequence is Advanced options -> Troubleshoot -> Advanced options -> Command prompt)
- Once you have a command prompt, use the following command to check the BitLocker status of the C: drive:
manage-bde -status c:
- If the status is returned as locked, you'll need to use the following command to unlock it using your recovery password:
manage-bde -unlock c: -rp <your 48-digit recovery password>
- Once the drive is unlocked you'll need to use the following command to suspend protection:
manage-bde -protectors -disable c:
- Exit and reboot.
- The system should now successfully boot Windows. Once there, use the BitLocker control panel to resume BitLocker protection.

There's nothing built-in that will change the volume level when you restart, so it should stay where you put it unless there's some other program running that is causing it.  I would try this.  Set your volume level to something you want, like 25%, then:

1)  Close all programs, then Press Ctrl+Shift+Escape

2)  Go to the Startup tab

3)  Disable everything listed

4)  Right-click the Start button and choose Run

5)  Type shutdown /r and click OK

This type of shutdown avoids any issues with fast-startup (which relies on hibernation functionality) and should start up.
