Homegroup with XP and 7 - Microsoft Community

ingat say:

Homegroup with XP and 7

My desktop runs Windows XP 2000, this is where my printer is physically connected.  I now have a laptop with Windows 7.  Is there anyway to share my printer with my laptop wirelessly?

malkeleah say: Yes, but you will not use Homegroup. Homegroup only works when all the computers on the Local Area Network are running Windows 7.

Here are general network troubleshooting steps. Not everything may be applicable to your situation, so just take the bits that are. It may look daunting, but if you follow the steps at the links and suggestions below systematically and calmly, you will have no difficulty in setting up your sharing.

Problems sharing files between computers on a network are generally caused by 1) a misconfigured firewall or overlooked firewall (including a stateful firewall in a VPN); or 2) inadvertently running two firewalls such as the built-in Windows Firewall and a third-party firewall; and/or 3) not having identical user accounts and passwords on all Workgroup machines; 4) trying to create shares where the operating system does not permit it.

In Windows 7, go to Control Panel>All Control Panel Items>Network and Sharing Center. Click on "Change advanced sharing settings". You don't want to use Homegroup unless you have all Windows 7 machines. If you do and you want to use Homegroup, see Windows 7's Help & Support. Otherwise, in the Advanced Sharing:

Turn ON network discovery
Turn ON file and printer sharing
Turn ON sharing in the Public folder sharing section
Turn ON password protected sharing

In XP, run the Network Setup Wizard or simply check the box for File/Printer Sharing in the Windows Firewall.

A. Configure firewalls on all machines to allow the Local Area Network (LAN) traffic as trusted. With Windows Firewall, turning on Window's File and Printer Sharing as above will take care of this for you. If you aren't running a third-party firewall or have an antivirus/security program with its own firewall component, then you're fine.  With third-party firewalls, I usually configure the LAN allowance with an IP range. Ex. would be 192.168.1.0-192.168.1.254. Obviously you would substitute your correct subnet. Refer to any third party security program's Help or user forums for how to properly configure its firewall. Do not run more than one firewall.

B. For ease of organization, put all computers in the same Workgroup. This is done from the System applet in Control Panel, Computer Name tab.

C. Create matching user accounts and passwords on all machines. You do not need to be logged into the same account on all machines and the passwords assigned to each user account can be different; the accounts/passwords just need to exist and match on all machines.
Start>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).

XP - Configure Windows to Automatically Login (MVP Ramesh) - http://windowsxp.mvps.org/Autologon.htm

D. If one or more of the computers on your network is XP Pro or Media Center, turn off Simple File Sharing (Folder Options>View tab).

E. Create shares as desired. In Windows 7 I usually share out the user's Desktop and the Public directory.

F. After you have file sharing working (and have tested this by exchanging a file between all machines), if you want to share a printer connected locally to one of your computers, share it out from that machine. Then go to the printer mftr.'s website and download the latest drivers for the correct operating system(s). Install them on the target machine(s). The printer should be seen during the installation routine. If it is not, install the drivers and then use the Add Printer Wizard. In some instances, certain printers need to be installed as Local printers but that is outside of this response.

How to create Recovery disk for windows 10 - Microsoft Community

suryakantnayak say:

How to create Recovery disk for windows 10

Hii Panel;

Last week i upgrade window7 to window 10 , so now what i want to know is that how to create a complete recovery disk (using Pen-drive or hard disk) for window 10. So that in future if any problem occur i can make format to currently install window10 and reinstall windows10 from that recovery disk .

Please tell me step by step process for creating recovery disk and also tell me how to again reinstall window 10 form that recovery disk ?

shenan stanley say:

Depends on what your purpose is.

  

A recovery drive is a tool you can use to boot your system and perform certain tasks (including restoration of a system image backup) to try and repair a problem with an existing Windows system.

  

A system image is a full backup of the system as it is when you create said image and can be used to restore that system to that point in time in a variety of situations (like you replacing the drive in the system with a new one for whatever reason OR you upgrade to a new OS and decide you do not like it, etc.)

  

The Windows 10 installation media you can get via the Media Creation Tool (creating a bootable DVD or USB stick) is the full installation media for Windows 10 and can be used to install Windows 10 cleanly or upgrade a previous OS in-place as well as a recovery drive.

  

If you just want a bootable media that would not be able to reinstall Windows in case of an issue, but could be used to repair issues if you have the technical knowledge-how or can find the detailed instructions to follow (and have the ability to follow them) - you just create the recovery drive.
  

If you want to make sure your system is fully backed up as-is so that if something goes wrong you could use something like a bootable recovery drive or Windows 10 installation media as a recovery drive you could restore that system image to get back to the point in time for your system when you made said system image - then you want to make a system image.  I suggest most people make system images periodically to add value to their regular personal file backups AND that they make a system image backup right before any major changes to their system (like upgrading from Windows 7 to Windows 10 OR installing a whole new office suite product, etc.)

  

If you want a bit of flexibility and the ability to install Windows 10 without an Internet connection at any point in time - cleanly, without any prior OS even - then you will want to get the Windows 10 installation media created and stored away.

  

I suggested you do ALL of them.  You create a recovery drive for those times when you might need to repair something but do not want to risk an accidental installation of an OS over your otherwise working system.  You create a system image so you have a way to restore to said point in time as a whole without data loss and leaving everything configured as it was hen you made the image.  You get the Windows 10 installation media (DVD or USB stick is your choice) so you have a way to cleanly install if desired, upgrade other systems if desired, use it as a recovery media if desired, etc.

  

You almost seem to be confusing yourself as you learn the options you have.  I think you may want to sit down and figure out your end goal is.  Is it to expand your C drive and continue using the system as is?  Is it to have the best protection for your system as possible so you can do minimal work to get that system back to working order in case of a problem and/or be able to restore your system as-is to a new drive in the same system?  Is it to backup only your personal data and clean install Windows 10 on that system once you replace your drive(s)?  Is it just to have a way to access the system in case it will not boot (allowing you to get data off and possibly repair and/or restore from backups)?

  

How to do all that has been fairly well pointed out in the links in this conversation so far ... But which you want to accomplish has been left unclear and changing with each response.
