Hi Maui,

You can use Windows Media Player to play digital media content that is streamed from the Internet. To ensure that a server running Windows Media Services can establish the best connection to the Player, the Player is configured to receive automatic streams from any of the following streaming protocols (depending on the network environment): User Datagram Protocol (UDP), Transmission Control Protocol (TCP), and Hypertext Transfer Protocol (HTTP). UDP and TCP are basic networking protocols that are used in combination with the Real Time Streaming Protocol (RTSP) to manage the high-level exchange of data.

The Player can also receive a multicast stream.

We require the following detail from you to assist you further.

1) Do you get any error when the connection to Webinar fails?
2) Do you have any third party security software installed on your computer?

If you're on a public network, such as at a coffee shop or an airport, Windows Firewall might block UDP and multicast streams from reaching the Player. To resolve this problem, open Windows Firewall, click Allow a program or feature through Windows Firewall, and then select the Windows Media Player check box. If this setting isn't available, it might be controlled by Group Policy.

To change protocol settings for the Player

a) Press Windows Key + Q Key and type Windows Media Player.
b) If the Player is currently open and you’re in Now Playing mode, click the Switch to Library button in the upper-right corner of the Player.
c) Click Organize, and then click Options.
d) Click the Network tab, and then make your changes.


Note: If you clear the TCP check box, or if your firewall blocks TCP streams, and the Player is configured to use the RTSP protocol, the Player will try to receive the streams by using the UDP protocol. If you also clear the UDP check box, or if your firewall blocks streams that use these protocols, and the Player is configured to use the RTSP protocol, the Player will try to receive the streams by using the HTTP protocol.


To change proxy settings for the Player


You might connect to the Internet through a proxy server, a computer that functions as an intermediary between a web browser and the Internet. To change your proxy server settings, do the following:

Open the Network Tab as mentioned in the section “To change protocol settings for the Player” , in Streaming proxy settings, click the protocol you want to configure, and then click Configure. Change any of the following settings:

a) AutoDetect proxy settings. Select this option if you want the Player to detect the proxy settings or automatic configuration settings automatically.

b) Use proxy settings of the web browser. Select this option (available with the HTTP protocol only) if you want the Player to use the proxy settings of your default web browser.

c) Do not use a proxy server. If you select this option, the Player won't try to communicate with a proxy server when connecting to the Internet. Typically, this means that the Player doesn't receive streaming content from the Internet.

What is the System Update Readiness Tool?

System resources, such as file data, registry data, and even in-memory data, can develop inconsistencies during the lifetime of the operating system. These inconsistencies might be caused by various hardware failures or might be caused by software issues. In some cases, these inconsistencies can affect the Windows servicing store, and they can cause software updates not to work. The System Update Readiness Tool tries to resolve these inconsistencies.

Back to the top
What does the System Update Readiness Tool do?

The System Update Readiness Tool verifies the integrity of the following resources:

  • Files that are located under the following directories:
    • %SYSTEMROOT%\Servicing\Packages
    • %SYSTEMROOT%\WinSxS\Manifests
  • Registry data that is located under the following registry subkeys:
    • HKEY_LOCAL_MACHINE\Components
    • HKEY_LOCAL_MACHINE\Schema
    • HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Component Based Servicing

En ce qui concerne ton code d'erreur :

 

Code et message d'erreur

Le code d'erreur 80151909 est précédé du message suivant :

 

Impossible de télécharger les profils Xbox Live pour le moment. Réessayez ultérieurement.

Problème

Si ce message et ce code d'erreur s'affichent, cela signifie qu'une tentative de téléchargement de profil a échoué, mais que le problème est peut-être temporaire.

 

Solutions

Solution 1 : redémarrer la console Xbox 360 et retélécharger votre profil

D'abord, redémarrez la console. Réessayez ensuite de télécharger votre profil. Voici comment procéder :

Télécharger votre profil Xbox Live

  1. Appuyez sur la touche Xbox Guide de la manette.
                                        
  1. Sélectionnez Télécharger le profil. 

    Remarque Si l'option Télécharger le profil ne s'affiche pas, cela signifie probablement que vous êtes connecté à votre profil. Appuyez sur la touche X pour vous déconnecter.

  2. Entrez le compte Microsoft (anciennement appelé identifiant Windows Live ID) associé à votre gamertag Xbox Live. Si vous ne connaissez pas votre mot de passe ou compte Microsoft, consultez Questions concernant un compte Microsoft ou un gamertag.

    Remarque Si votre compte est un compte enfant Xbox Live, vous devez utiliser le compte Microsoft du parent pour télécharger le profil.

  3. Sélectionnez le périphérique de stockage de votre profil (par exemple, le disque dur). 

    Le téléchargement de votre profil commence. Lorsque le téléchargement est terminé, vous pouvez choisir de demander un mot de passe de compte Microsoft pour la connexion à votre profil.

    Option Mémoriser le mot de passe

                                        

L'option Mémoriser le mot de passe est désactivée par défaut, ce qui signifie que vous êtes invité à entrer votre mot de passe Microsoft à chaque fois que vous vous connectez à votre profil Xbox Live. Laissez l'option Mémoriser le mot de passe désactivée si vous voulez être invité à entrer votre mot de passe lorsque vous vous connectez à votre profil sur cette console.

Vous devriez désormais pouvoir vous connecter à la console en utilisant votre gamertag Xbox Live.

 

 

Solution 2 : contacter le support technique Xbox

Si la solution précédente ne résout pas le problème, contactez le support technique Xbox.

Add-ons are apps that browsers use to interact with web content like videos and games.   • Temporary Internet Files
  • Cookies
  • History

f) Click Close, and then click OK to close the Internet Options dialog box.

Note : This will delete the web history of the browser.

UEFI IS the replacement for BIOS in all modern PC's.

greg carmack - windows mvp 2010-2020 say:
Hi Erika. I'm Greg, an installation specialist and 9 year Windows MVP, here to help you.

If your PC comes with the modern UEFI replacement for BIOS then you'd want to install it to UEFI to have the full features including greatly enhanced Security and should not have to install to old Legacy or CSM mode that is only to allow other Legacy drives to boot on the same PC.

Are you creating the media only using Media Creation Tool so we know for sure it's UEFI capable, and then are you booting the media only as a UEFI device?

If so in UEFI Setup reset to Optimal Defaults, save & Exit. GO back to check if UEFI is enabled, CSM disabled, Secure boot disabled during install if necessary, Windows Boot Manager set first in Boot Priority?

Then boot the media as a UEFI device using only the HP or F9 ESC BIOS Boot menu key as the PC powers on.

Make sure all other drives and peripherals are unnplugged.

Choose Custom install, delete all partitions down to Unallocated Space, then click Next to auto-create and -format partitions to change formatting to GPT if necessary. This makes it foolproof.

Once you've tried all of this if it still has problems post back the exact failure and I'll help you troubleshoot further.
