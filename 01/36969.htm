petersteier say: Dear Microsoft,

I think that there is now enough description in this thread so that a possible software solution is almost obvious to every experienced hardware programmer. Please forward this message literally to the pen driver programmers.

* normal operation: the signal from the tilt sensor is used to calculate a tilt correction offset, proportional to the tilt and in the opposite direction. The tilt correction offset is added to the position reported by the position sensor, to arrive at the effective drawing location.

* for some reasons, the tilt sensor sometimes does not respond properly due to electrical interference; this happens if the device is touched too much or too little, if it is hot, or if the battery is low. Whatsoever, does not matter. This cannot be solved by software. However, the wrong signal should be easily detectable, it might be a error code on the I2C bus, or a reading which is precisely 0, or just the sudden jump in tilt, which cannot be physical.

* the driver programmer decided to handle this case as if the pen tilt was really 0. Thus, the effective drawing location jumps by the present tilt correction offset. This is the software bug.

* the driver should rather ignore the invalid tilt signal and continue to use the last valid tilt correction offset. Thus there will be no jump, and the user will not recognize the sporadic malfunction. Problem solved.

BTW:
* the hoover cursor does not use the tilt correction. Thus, under normal operation, the cursor jumps when the tip touches. I do not know whether it is technically possible to measure tilt if the tip does not touch, but also this jumping would be significantly reduced if the last tilt correction offset before the pen is lifted is used.

Mapped drive is a network term to assign Drive letter to a shared Drive or a Shared Folder.

Most of the Drive that you mentioned might not be an Existing Network Shared Drives, and were not mapped on other computers so they do not appear in the list.

Let use an example.

You assigned the letter G to your CD/DVD Burner. That means that on the computer that the physical Drive is installed on there is a Drive G.

The drive is Not mapped to anything, it is just Drive G: on that computer.

If you configure the Drive to be a shared with its correct permission and security, then the drive should appear on the Network, but it is still not a mapped drive.

At this point you can Map the Drive on another computer by assigning it a Drive letter on the other computer. It does not have to be G you can map it on the other computer with any available letter above C.

http://windows.microsoft.com/en-US/windows7/Create-a-shortcut-to-map-a-network-drive

The process of UnMapping a drive is similar.

Let summarize.

On computer A there is physical drive G. This drive can be Mapped on computer B as Drive Z, and On computer C as drive X.

The drive letter assignment it always pertaining only to the computer that the Drive was mapped on it is Not affecting the assignment of the same Drive on other computers.
