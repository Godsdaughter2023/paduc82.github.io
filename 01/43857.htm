malacath say:

If it worked for a while with UPnP then it could just be due to the IP addresses changing.

Some routers really do need you to use a static ip with UPnP to stop the NAT changing.

The static IP should be set in the router address reservation or DHCP reservation options though and not in the console itself. The IP settings in the console should be left as automatic.

Using DMZ and port forwarding will likely make the problem worse with most routers
