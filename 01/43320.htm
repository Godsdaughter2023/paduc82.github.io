santosh y say:

Hi e115dl7,

It is safe to install Windows Updates while programs are running. If one of the applications being updated are open (for instance, Internet Explorer), the files will be replaced once the system is restarted. This is why sometimes after you install updates and reboot you will see “Windows is configuring updates”. This is commonly the operating system replacing the files that were active..

Since you previously had Windows 10 installed and activated on that PC during the installation process skip the steps which ask for a product key and select the option 'I am Reinstalling Windows 10 on this PC', and activation will not be an issue, your PC will have a digital entitlement stored on the Microsoft Activation Servers

https://www.microsoft.com/en-gb/software-downlo...

If you have problems booting from a boot disc, you may have UEFI BIOS:
Insert the Bootable Installation Media, then go into your BIOS and make the following changes:
1. Disable Secure Boot
2. Enable Legacy Boot
3. If Available enable CSM
4. If Required enable USB Boot
5. Move the device with the bootable disc to the top of the boot order
6. Save BIOS changes, restart your System and it should boot from the Installation Media

Yes, the steps are valid for Windows 10 as well, however you can also refer to the section “To set up assigned access in PC settings” in article: Set up a single-app kiosk

 

As you mentioned that you are receiving: "Something went wrong” error message. This can be if your current Administrator account is corrupted. I would request you to enable hidden administrator account and then try to setup Assigned Access. Here is how you can Enable Hidden administrator:

  1. Open Command Prompt (search CMD on start menu) and run it as administrator.

  2. Then type: net user administrator /active:yes

  3. Close the Command Prompt.

  4. Then Log off from this user account sign in the account which says Administrator.

  5. Try to setup Assigned access mode.
