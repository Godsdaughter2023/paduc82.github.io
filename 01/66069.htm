How will Windows 10 handle the EoL of tomorrow’s - Microsoft Community

georgeogden say:

How will Windows 10 handle the EoL of tomorrow’s "legacy" hardware?

In the past, it’s been abundantly clear when a hardware component reached its EoL – the OEM ceased to produce drivers for the next version of the OS. From the earliest days of DOS through Windows 1, 2, 3, 95, 98, ME, XP, Vista and 8 there’s been a “clean” break when the decision to upgrade to the next version of the OS was almost entirely dependent on the availability of drivers from one or more OEMs. (A similar consideration applied to the availability of 3rd party applications for the new OS.)

Over time, while the drivers were and remain available as downloads direct from the OEM, the delivery of drivers has become integrated with the OS updates – a development welcomed by most users (there are a few exceptions) as this has made the maintenance of a PC simpler and further lessened the technical knowledge required to do so – that’s assuming the user knew that it was necessary in the first place…. most don’t.

Clearly an OEM cannot maintain driver compatibility with future releases of the OS in perpetuity; it would be far too costly, and, legacy hardware would not deliver the performance and features to support the functionality available in the new OS and other applications.

However, that “clean” break, the “step change” in the OS, has now ended with Windows 10, an OS provided using the “Software as a Service” (SaaS) model. Following the initial release of Windows 10 there’s been the Anniversary update and then the Creators update.  In addition, between each of these major releases, there have been interim updates delivering functional enhancements, important security updates, bug fixes, various others such as the MSRT and updated hardware drivers.

There are several questions regarding the impact of drivers when they reach EoL; here’s a few to which I have yet to find an answer…. you may have few more…..

  1. What will happen when changes in a future Windows 10 update renders a hardware component redundant? i.e. “legacy” and no longer accompanied by an updated driver?
  2. Will it simply fail and roll back? Probably but….
  3. How will the user be informed (or be able to discover) that driver support for a hardware component has been discontinued?
  4. Will the user be able to continue using the previous release/build? Almost certainly but….
  5. Will the Windows update process cease attempts to update to a build that’s failed? Possibly but after 7 failed (KB4020102) updates to 15063.332 before (KB4022725) succeeded to update to 15063.413 I’m not so sure….
  6. Will all new feature and functionality enhancements cease to be provided?
  7. How long will that release/build be maintained?
  8. Will that release/build continue to receive security updates?
  9. Will apps installed from the Store continue to function? Possibly but if even if they do….
  10. Will “legacy” versions continue to function and for how long? If not…
  11. Will users receive compensation for “paid” apps that no longer function? 

Slàinte Mhath!
