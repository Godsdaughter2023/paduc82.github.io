Important Information If You Have Received a STOP Code

If you have received a blue screen error, or stop code, the computer has shut down abruptly to protect itself from data loss. If you have experience with computers and want to try to recover from this error, follow the steps provided in the Microsoft article Resolving STOP (Blue Screen) Errors in Windows.

  1. Download and install updates and device drivers for your computer from Windows Update.
  2. Scan your computer for computer viruses.
  3. Check your hard disk for errors.

NTFS_FILE_SYSTEM Parameters

The following parameters are displayed on the blue screen.

Parameter Description

1

Specifies source file and line number information. The high 16 bits (the first four hexadecimal digits after the "0x") identify the source file by its identifier number. The low 16 bits identify the source line in the file where the bug check occurred.

2

If NtfsExceptionFilter is on the stack, this parameter specifies the address of the exception record.

3

If NtfsExceptionFilter is on the stack, this parameter specifies the address of the context record.

4

Cause

One possible cause of this bug check is disk corruption. Corruption in the NTFS file system or bad blocks (sectors) on the hard disk can induce this error. Corrupted SCSI and IDE drivers can also adversely affect the system's ability to read and write to disk, thus causing the error.

Another possible cause is depletion of nonpaged pool memory. If the nonpaged pool memory is completely depleted, this error can stop the system. However, during the indexing process, if the amount of available nonpaged pool memory is very low, another kernel-mode driver requiring nonpaged pool memory can also trigger this error.

Resolving the Problem

To debug this problem: Use the .cxr (Display Context Record) command with Parameter 3, and then use the kb (Display Stack Trace) command.

To resolve a disk corruption problem: Check Event Viewer for error messages from SCSI and FASTFAT (System Log) or Autochk (Application Log) that might help pinpoint the device or driver that is causing the error. Try disabling any virus scanners, backup programs, or disk defragmenter tools that continually monitor the system. You should also run hardware diagnostics supplied by the system manufacturer. For details on these procedures, see the owner's manual for your computer. Run Chkdsk /f /r to detect and resolve any file system structural corruption. You must restart the system before the disk scan begins on a system partition.

To resolve a nonpaged pool memory depletion problem: Either add new physical memory to the computer (thus increasing the quantity of nonpaged pool memory available to the kernel), or reduce the number of files on the Services for Macintosh (SFM) volume.

I have 2 user accounts in Vista using Windows Mail. The email accounts are completely separate from each other. I just setup a new Windows 10 computer. I had planned to import my other Vista account, Vista 2, into WLM but my concern is that the folders in that account will also end up in the same Storage folder, which would be messy. Is that what will happen, or will there be a separate folder (Storage Folder 2)?

Is there any way to move the folders from the storage folder into the actual account folders? I tried by dragging and had no success. I could move the folders around in the Storage Folder, but that's not what I want.  Some of the folders in the Storage folders are active and I'd like to be able to add new incoming/sent emails to them.

If I am unable to move the folders in the Storage folder, would I be better to setup another "user" in Win 10 and have a separate WLM in that for my Vista 2?

¡firedog say:

When you import folders into Windows Live Mail, they will be put into Imported folder under Storage folders. You should have no trouble distinguishing between the two imports, because they will retain their original folder structure.

You can't move folders out of Storage folders, but you can move messages to an existing folder - newly-created if necessary - under an account. However, you may find it just as convenient to keep (some of) the folders under Storage folders*. You can drag a folder with its sub-folders out of Imported folder and drop it on the Storage folders heading to move it to the root of storage folders. When Imported folder is empty, you can delete it.

It's as well to take it gently when conducting major file operations like importing lots of messages or moving them around. The quieter the environment you create for the program to work in, the greater the chances of completing the exercise without major upsets. I would Work offline, disconnect from the network and disable anti-virus before doing this to avoid unnecessary interference. Once the import is complete, give Windows Live Mail plenty of time to index the newly-imported messages before doing anything else in the program. When it's settled down completely, close down the program and wait ten minutes before rebooting the computer and re-enabling your anti-virus.

* Unlike in some other programs, removing an account in Windows Live Mail permanently deletes all the account folders and their contents. Keeping important messages in Storage folders makes it less likely that data would be lost if an account is removed.
