Welcome to Microsoft Answers!

Just-In-Time debugging is a feature that launches the Visual Studio debugger automatically when a program, running outside Visual Studio, encounters a fatal error. Just-In-Time debugging allows you to examine the error before the application is terminated by the operating system. The Visual Studio debugger does not need to be running when the error occurs.

To enable/disable Just-In-Time debugging

   1. On the Tools menu, click Options.
   2. In the Options dialog box, select the Debugging folder.
   3. In the Debugging folder, select the Just-In-Time page.
   4. In the Enable Just-In-Time debugging of these types of code box, select or clear the relevant program types: Managed, Native, or Script.

To disable Just-In-Time debugging, once it has been enabled, you must be running with Administrator privileges. Enabling Just-In-Time debugging sets a registry key, and Administrator privileges are required to change that key.

  5. Click OK.

¡firedog say:

This error has affected a number of users who have upgraded to Windows 10. They have discovered that they have to go to their account security page and sign in with the affected account. The account needs verifying, e.g. by submitting a security code sent to the phone number registered on the account or to one of its alternative email addresses. After verification, server error 3204 no longer occurs.

 

It's a good idea when visiting the account security page to ensure that at least two verification channels are registered, e.g. a mobile phone number and an alternative email address, preferably on a different domain like Gmail. This will make it possible to recover the account should you ever lose access to it.
