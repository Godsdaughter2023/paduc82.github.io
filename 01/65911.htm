Since the introduction of Windows Update as part of Windows, Microsoft has delivered updates as way to keep Windows updated and secure. This has always been the way the company keeps commercial versions of Windows on the market current.

With Windows 10, Microsoft is making significant changes to this model by delivering new methods of servicing Windows. Taking into account we now live in a mobile, constantly connected world, where threats happen all the time. Part of the solution to this is to keep users current by providing the latest updates as soon as they are available in addition to delivering features as soon as they ready too.

  • Security Updates, Features and Fixes are automatically applied.
  • There is no option to delay or customize these updates.
  • Windows 10 Home
  • CBB includes the requirements of the Current Branch, but also provides the option of customizing when and which Security Updates, Features and Fixes are applied, similar to how Windows Update works today in current versions of Windows. 
  • Updates cannot be deferred indefinitely.
  • Windows Updates can be managed using enterprise management tools such as Windows Update for Business and/ WSUS.
  • Windows 10 Pro
  • Windows 10 Enterprise
  • Windows 10 Education
  • Only available to volume license customers running Windows 10 Enterprise.
  • Flexible options for managing Windows Updates.
  • Windows Updates can be customized to only accept Security Fixes.
  • Windows Updates can be managed using enterprise management tools such as Windows Update for Business and/ WSUS.
  • Windows 10 Enterprise

In previous Windows releases, when you installed an upgrade version of Windows, if you wanted to reinstall the upgrade version, you had to first reinstall the qualifying version from which you upgraded, reactivate it, then upgrade again and reactivate again.


With Windows 10, this is no longer the case. Once you have upgraded to Windows 10 and activate, you can simply create a bootable copy on DVD or USB and reinstall just Windows 10 without the need to reinstall the qualifying version and it will reactivate automatically.

Currently at work we are trying to deploy Windows 7 on our product. We have been using pcAnywhere with an older version of Windows to gain dial up access to our machines on the the field. Many of our machines are not plugged into any network on the field, if they require support the costumer would plug the phone line into the modem, they would start up pcAnywhere as a Host and we would dial in using pcAnywhere in remote access mode. Then we have full control of the PC.

I tried to replicate this using windows which worked but I ran into some problems. I created a new incoming connection using the modem, dialed into the PC and then started up Remote Desktop.

The only way I could create a "New Incoming Connection" was to plug the NIC into a router, the router had no internet access, it seems that the Routing and Remote Access service starts up only when the NIC detects a LAN present (Microsoft says this is by design0. After this creating a New Incoming Connection is no problem. However once you disconnect the NIC from the router and restart Windows 7, the new Incoming Connection I created is no longer under "Network Connections" only the NIC is there. If the NIC is still connected to the router when you restart the New Incoming Connection I created is still under "Network Connections" along side the NIC.

Is there any way to prevent windows 7 from removing the New Incoming Connection if there is no LAN detected? I understand one way of doing so is to always have the NIC plugged into a LAN, but this is not possible all the time, especially where broadband is not available and the only way to access the machine would be via dial up. Any help or insight would be appreciated.

 

jdhere say:

So I found a way around the problem of having to be connected

to a Network in order to setup a New Incoming Connection and preventing

it from being removed after restart, installing the Microsoft Loopback Adapter

seems to have done the trick.

http://support.microsoft.com/kb/839013
