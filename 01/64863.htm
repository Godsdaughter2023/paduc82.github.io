jeeped say:
From what you've stated above, I believe the file association for LNK files has been misdirected to MyGame.exe. I've seen this happen when upon opening a program through a shortcut (e.g. .LNK file), you are asked what program to open it with. After selecting MyGame.exe (or some other program) via the Select a program from a list of installed programs, and checking the Always use the selected program to open this type of file (circled in red below), the document or program referenced by the original shortcut opens in MyGame.exe as it should.

The trouble is that now all shortcuts (e.g. .LNK files) on the desktop or throughout Explorer will want to be opened by MyGame.exe and use the MyGame.exe icon to display the shortcut regardless of what the shortcut was originally intentioned to open. This is known as a file association and default icon. The same condition could occur if you performed those 'Open With' actions on a .JPG or even an .EXE, but the .LNK explanation is far more likely for items on the desktop.


The good news is that a small free utility has been written to undo the .LNK file association that was created. There is also a registry hack to accomplish this, but I recommend the unAssoc utility for its ease of use. I used to send people to another site to download this utility, but that site has become confusing with all the ads and false download links. I've put a copy on my own SkyDrive here.

                                    

Just download, then use the utility to unassociate the .LNK files for the User or globally Delete the file type and let Windows rebuild a new one to defaults.

Solution for Windows Modules Installer Worker constant disk/CPU - Microsoft Community

assion say:

Solution for Windows Modules Installer Worker constant disk/CPU overuse.

I'd like to submit a solution that helped me with this problem:

WMIW uses CPU and disk resources sometimes hogging 100% disk activity causing other programs to hang. I had a very similar issue. What i found was that the WMIW process somehow just kept trying and failing to properly install one of the updates and got into some endless loop retrying all the time.

  • Open Windows update - Check for updates.
  • If several are found see if some have "Failed to install because another update must be installed first" if so, there should be one that is sitting at "X% downloaded" and doesn't change. 
  • Search for that update online and download it from a trusted site.
  • Kill the WMIW process in the Task Manager (right click->End task; Important, otherwise the manual install will hang too.) and run the update file.
  • Restart when prompted.

In my case the failing update was "KB4054517" but it might be different for you. After I did the above, WMIW finished installing the remaining updates in a few minutes and shut itself down. Haven't had a problem since.
