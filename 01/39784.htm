a. user say:

The PerfStringBackup.ini file is just a long text file with information that is used to display stuff in the Performance Monitor tool (perfmon.exe).

Assuming XP is installed on your C drive, the file should be here:

c:\windows\system32

If yours is missing, there is a copy on my SkyDrive you can download.

The Performance Monitor will not improve your performance.  It may help you spot bottlenecks in your system from which you can make adjustments to improve performance, but I would really be desperate to use it for performance issues, but then I don't nave much experience using it either.  It is just a monitoring tool - you can't adjust anything with it.

I uploaded a copy of the XP Pro SP3 file you need to my SkyDrive (everybody has a SkyDrive for sharing files).

You can download it and when you do, put a copy of the file in these folders (assuming Windows is installed on your C drive):

c:\windows\system32

Here is the link to my SkyDrive and you can look for the file you need there:

https://skydrive.live.com/?cid=6a7e789cab1d6f39&id=6A7E789CAB1D6F39!311

When you see the files available for download, you may not see the file extension (.exe, .dll, .cpl, .sys, .zip, etc.), but when you download them they will have the correct extension.

When you download the file (especially if you use Internet Explorer), when you get a chance to save the file, your browser may not save the file with an extension (like .exe, .dll, .cpl, .sys, .zip, etc.) so you will have to manually add the proper extension to the file when you download the file before you save it.

You can download the file with no extension and then rename the file to add the proper extension.  You don't want to try to use a file called PerfStringBackup when the file name really needs to be called PerfStringBackup.ini (if the downloaded file has no extension you will have to change the name of the file to add the proper extension to get it to work.

You then need to put the downloaded files in the correct folders on your system.

Then to load up the Performance Monitor counters from the PerfStringBackup.ini file, click Start, Run and in the box enter:

lodctr  /R:PerfStringBackup.ini

Click OK, the screen will flash and you'll be all set to add counters into Performance Monitor.

How to Remove Windows Update Ghost Folder - Microsoft Community

wmcole say:

How to Remove Windows Update Ghost Folder

After a major Windows Update that created a "Windows.old" folder, a "Windows.old(1)" folder appeared on the C:\ root.  Attempts to delete it yield:

"Could not find this item.  This is no longer located in C:\. Verify the item's location an try again."

The folder contains eight 0-sized sub-folders, all of which I can access properties, take ownership, change permissions UNTIL I get to the last file in the chain, "C:\Windows.old(1)\Users\<MyUserName>\AppData\Local\Packages\Microsoft.Windows.Cortana_cw5n1h2txyewy\LocalState".  On this last folder, the Properties Security tab says "The requested security information is either unavailable or can't be displayed."

  1. chkdsk C:\ /R does not help.
  2. FileAssasin does not help (it can't deal with ghost directories, only files).
  3. Copying "C:\Windows.old\Users\<MyUserName>\AppData\Local\Packages\Microsoft.Windows.Cortana_cw5n1h2txyewy\LocalState" to the corresponding Windows.old(1) location allows me to delete the copied version (which apparently can't replace the ghost "LocalState") but nothing else in the Windows.old(1) directory chain.
  4. Windows.old(1) does not appear in the registry (according to regedit).

This appears to be an error in the MFT or whatever NTFS mechanism is used to store filename : vector correspondence. 

So, how do I fix the index to eliminate the false entry for a non-existent directory chain?

wmcole say:

SOLVED (finally)!

First, I looked at Properties->Security tab for each folder starting at Windows.old(1). Either at "Users" or "...Users\<MyUserName> I spotted one of those long strings like is found in the user section of the registry with a "?" next to it.  AFAIK, that means "unknown group or user name".  So I deleted it from permissions at that sub-folder level.  The deleted security principle did not appear anywhere else in the chain down to "LocalSate..".  But as usual, "LocalState.." showed "X The requested security information is either unavailable or can't be displayed." in the Properties->Security tab.

File Explorer would not allow me to open the "LocalState.." folder.  Just for grins I tried to open it in a third-party file explorer which showed the folder as empty.  I then copied a small .jpg file to LocalState..  It seemed to succeed.

In Windows File Explorer I found that now, inside C:\Windows.old(1)\...Microsoft.Windows.Cortana_cw5n1h2txyewy\ there appeared two folders, "LocalState" and "LocalState..", both of which I could open, but both of which were empty.  So I selected Windows.old(1) in File Explorer and right-click delete - Voila!  It went away.  After a reboot it is still gone so it appears the problem is solved.
