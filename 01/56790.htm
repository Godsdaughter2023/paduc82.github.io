the larch say:

Windows 7 Search was invented in such a way that it is practically two different search tools from the point of view of people using it.  The simplified (dumbed down perhaps?) method is the imprecise usage based on an indexing system meant to involve the Library setup for people who don't necessarily care which actual folders their files get stored to, letting Windows 7 take care of it and accepting default locations that don't really matter if all you care about is accessing files through the Libary setup.

The complicated (arcane perhaps?) method is the precise usage based on something called Advanced Query Syntax.  The syntax works with indexed and non-indexed folders, but there are some peculiar differences in usability that makes usage in non-indexed folders more versatile (specifically, searching for phrases in file content).

Anyway, for your task in particular, you can try entering:

name:~=notepad

The use of "name" ensures that the searching will not also look in file content (as it would by default in indexed folders), and the use of ~= means "match anywhere" regardless of being at the start of words or within words.

the larch say:

I've never read of a way (and I don't see a way in the settings) to make "match anywhere" the default.  As for symbol searching such as the percent signs, over a few months I used a lot of time trying a lot of different syntax and found a lot of different kinds of results that depend on the syntax and on the symbols being searched.  The general advice is to search with syntax name:~= followed by the text within quotation marks, such as:

name:~="big&small"

Now, if you ever want to use wildcard characters (asterisk * for any number of any characters, and question mark ? for one of any character) while also searching for symbols, it becomes necessary to use tilde ~ without the equals sign with name:, and also necessary to otherwise type the entire file name being searched or at least put asterisks at the start and end of the search text, as in:

name:~big&s?all.txt

or for instance

name:~*ig&s?all*

As for Windows 8 being able to have "match anywhere" set to default, I have not used Windows 8 -- anybody else can report?
