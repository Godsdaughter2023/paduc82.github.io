gregtrounson say:

Fixed it.

I encountered a PC with the same problem, and none of the usual recommended repair steps had any effect (netsh int reset et al, Windows 10 Network Reset tool).  In my case the DHCP client service was using 100% of one CPU core, and setting a static IP address didn't help.  System Restore had no intact restore points to work with, and the 1709 update couldn't be rolled back.

In the end I found this fix that worked for me, suggesting a problem with the Base Filtering Engine:

Launch regedit, and delete the entries in:
HKLM\System\CurrentControlSet\Services\BFE\Paramters\Policy\BootTime\Filter and
HKLM\System\CurrentControlSet\Services\BFE\Paramters\Policy\Persitent\Filter

As suggested in:

https://answers.microsoft.com/en-us/windows/forum/windows_7-networking/ping-general-failure/fbd61186-29ad-4f03-a3fc-d8b82b666190

After a reboot both network interfaces were able to get IP addresses and have worked fine ever since.
