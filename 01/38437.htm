a. user say:

I'm sure it has a reason - it is most likely some background task like your security software checking a file, or some software checking for updates, or some scheduled task that was setup by the system or an installed program that you don't even know exists, or some type of monitoring activity that is enabled (perhaps part of the Windows Reporting System), or event viewer recording some new event (an error or merely information), or any number of things.  If it occurs for a long time, then it may be downloading or scanning for Windows Updates or security software definition updates, or it could actually be doing a scheduled defragmentation that exists in addition to the one you have setup yourself (but don't realize exists).

Try Process Explorer http://technet.microsoft.com/en-us/sysinternals/bb896653.aspx and Process Monitorhttp://technet.microsoft.com/en-us/sysinternals/bb896645.aspx to identify what is occurring on your system.  You can configure the columns to show different types of activity and thus narrow down your search for the cause.  It will provide more about processes than programs, so you may need to do a bit of work to determine which program is causing which process to be working - but with all the information you will have, that should be possible.

Another alternative (not as detailed but possible just as effective and, if it works, certain to find the cause) is to try a clean boot http://support.microsoft.com/kb/929135.  If the problem/activity goes away then it's just a matter of tracking down the culprit causing the problem/activity.   Follow the procedures in the article.  Once found, delete, remove, deactivate, or uninstall it - or leave it alone now that you know what it is and realize it's OK that it is happening.  Once done be sure to reset Vista back to normal status as explained in the procedures. If the problem occurs in clean mode then just restore the system to normal status and reboot - this solution is not going to work.

You can also go to Start / Search Box and type Task Scheduler and double-click on the icon that appears and check every task that is scheduled on the system and when they occur (some are designed to start during idle time and can be configured to pause if the system becomes active (you start using it) or continue until done despite what you're doing- it depends entirely on how it is setup).  I recently checked mine thoroughly and was stunned at the number of scheduled tasks I had no idea were there - there were dozens or more (and while I have done some cleaning up there, I still haven't identified what all of them are or what all of them do and continue to work on it in my spare time as I want to be certain I know exactly what I'm doing before I do it so I don't disable a task that I actually want running - I leave things as they are until I fully understand what they do and can decide if it is necessary or not - or change the parameters or frequency to something more to my liking).  I suggest you adopt a similar philosophy when dealing with whatever is causing the hard drive activity - meaning, be SURE you know what you are stopping or deactivating and the consequences before you do anything. 

You don't have the type of "problem" - if it is a problem - where you need to resolve it quickly because your system isn't functioning or something equally catastrophic.  You have time to research it and do it right.  You may discover that you agree with everything that is happening and that no changes at all are needed - or you may find many changes you want to make.  I don't know because I don't know the causes.  Just take your time and be careful and know what you're doing before you do it.  It's better here to be safe than fast.

I hope this helps.

Good luck!
