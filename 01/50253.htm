frederik long say:

I have been playing with this kind of execution with my friends machines if am able to at least SMB access.

 

Is this a vulnerability or a normal execution??

This is perfectly normal. Console commands are executed by using this strategy:
  1. Check if the command is internal to the Command Processor. Examples: CD, del, copy.
  2. Check if the command exists in the current folder.
  3. Check each folder listed in the %path% for the existence of the command.
  4. Check this registry location:
    HKLM\Software\Microsoft\Windows\CurrentVersion\App Paths\
Access rights are NOT affected by placing commands into a user's profile folder. They are independent of the location of the command.

a. user say:

It sometimes will combine separate instances of IE in the taskbar into one instance if there are too many other programs open at the same time.  If you click on the combined listing, you should see all the open windows. None should disappear from the listing and CERTAINLY should it should not close down any window.  I frankly don't even know how to limit the number of open windows IE can have at any one time - though there probably is some feature to control this (but I've never seen it).  I do not believe this is being caused by Vista but rather by Internet Explorer.

There IS a limit of being able to download from only two internet sites simultaneously (I'm not sure how the liimit makes itself known - I suspect an error message of some sort but it is remotely possible that it actually closes one of the instances to conform to the limit (though I really doubt that sincerely).  This ONLY applies with attempting multiple downloads - not with just browsing with multiple windows.

Here are two links for possibly fixing this issue with Internet Explorer.  I should point out that the second option is fairly radical and you lose your history and some other things (it does give you a warning message about what you are about to lose before you actually start the process so you can decide if you want to proceed - I think you may have little other choice if you want this fixed but the decision is yours - you cana just go to the IE Forum posted below and see if they have any simpler options to suggest).
