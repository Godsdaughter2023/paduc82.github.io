a. user say:

Error messages like that which end in 0002 mean - file not found

Error messages like that which end in 0003 mean - path not found

Error messages like that which end in 0005 mean - permissions, security or access denied issues

Did you check to see if that folder actually exists on your system:

C:\Windows\System32\config\systemprofile\AppData\Local\Microsoft\Windows\Burn\Burn

It could have been a temporary folder that existed when you created your backup set and has since been deleted.

If that folder/path no longer exists you will need to refresh your backup set or make a new backup set that doesn't include that folder/path.

Those kinds of error message that end in 0002, 0003, 0005 are telling you something and once you know what they mean it is easier to figure out the problem.

They go back to the good old days of DOS:

01  Invalid function number
02  File not found
03  Path not found
04  Too many open files (no handles left)
05  Access denied
