rickcp say:

Error 80070643 (or simply 0x643) that shows up when using the Microsoft/Windows Update system, as the result of an unsuccessful install of an update (or updates that may be re-offered) for any version of the .NET Framework installation on your system, can happen for several reasons. The most common cause is a problem due to system corruption on your computer. This occurs for various reasons, including but not limited to other applications overwriting of .NET files or corrupted hard disk sectors. It's a WU generic error code that means that setup failed, but it doesn’t provide any more detailed information about why it failed or how to fix the failure. When a .NET Framework update fails to install with a 0x643 error code, it means that there is something wrong with the version of the .NET Framework that is installed on the system which the update is supposed to be fixing.

  To troubleshoot this problem, you must fix MSI software update registration corruption issues (Method Nº1), or uninstall and reinstall multiple versions of the .NET Framework installation (Method Nº2). Both methods are described in Microsoft Knowledge Base article KB976982 "Error codes “0x80070643” or “0x643” occur when you install the .NET Framework updates" > http://support.microsoft.com/kb/976982/en-us >

Please note this KB has just been reviewed May 3, 2012. See [1]

  You can try Method 1 first and see if KB951847 now installs correctly. If the problem is solved and you have the updates successfuly installed, you are done!  If the issue is not resolved, you would need to go on Method 2 and troubleshoot your whole .NET Framework installation (All versions).

  To repair your .NET Framework installation, you must uninstall individual or multiple versions of the .NET Framework by using the .NET Framework Cleanup Tool, and then reinstall the components previously uninstalled. As the Cleanup Tool will delete shared files and registry keys used by other versions of the .NET Framework, you will need to perform a repair/ re-install for all other versions (or whatever the version(s) you need) of the .NET Framework that are on your computer or they will not work correctly afterwards.

System file check (SFC) is a utility built into the Operating System that will check for system file corruption.
