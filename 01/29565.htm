    lemp say:

    Back in the day, when I used that technique to enable printing to a USB-connected printer from an environment that didn't "know about" USB ports, I used the /B switch on the copy command to ensure that the PCL commands embedded in the output from the print spooler got passed to the printer correctly.  I don't know if that was required, but you could try and see if it works for you:

       copy /b test.prn LPT1

    Another technique that was used to print to a USB printer from a program that only printed to LPT1: is to pool the port the printer actually is on with the LPT1: port.  http://www.pctips3000.com/how-to-enable-printer-pooling-on-windows-7/

    See, generally, https://web.archive.org/web/20161123161133/http://members.shaw.ca/bsanders/printfromdos.htm

    For additional thoughts, see http://www.columbia.edu/~em36/wpdos/usbprint.html

    Interestingly enough, this article says that the net use method only works for XP (which is when I used it) and not for later versions of Windows.  I have no personal experience using this method on Windows 7.
