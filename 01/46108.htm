wunders say: It would help to know what change you attempted in Regedit.
It would also help to know whether you have XP Home or XP Pro.

In any event, there is a Group Policy that can be set that prohibits access to the control panel.  The description of this setting is as follows...
<Quote>
Disables all Control Panel programs.  This setting prevents Control.exe, the program file for Control Panel, from starting. As a result, users cannot start Control Panel or run any Control Panel items.  This setting also removes Control Panel from the Start menu. (To open Control Panel, click Start, point to Settings, and then click Control Panel.) This setting also removes the Control Panel folder from Windows Explorer.  If users try to select a Control Panel item from the Properties item on a context menu, a message appears explaining that a setting prevents the action.  Also, see the Remove Display in Control Panel and Remove programs on Settings menu settings.
</Quote>

If you think this might be your problem...
If you have Windows XP Pro, start the Group Policy Editor:
  (Start -> Run -> gpedit.msc)
Then look in the path:
  Administrative Templates -> Control Panel
for a setting labeled "Prohibit access to the control Panel" and change it as desired.

Otherwise, you will need to use Regedit.  Go to the following path:
  HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer
and look for the value:  "NoControlPanel".
If you see it there, delete it.

If you have received a blue screen error, or stop code, the computer has shut down abruptly to protect itself from data loss.
