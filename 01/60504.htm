In order to DDoS you, the person needs to know your IP address. And they can only get that address, if they are in the same game lobby or party chat or other direct peer-to-peer connection (for example using chat software on your PC or whatever else the hacker coaxes you to contact him). 

So make sure you block and mute the hacker, so they can't contact you, and Xbox Live will *try* to not match you with that person in a lobby. 

After you blocked the hacker from contacting you, you need to change your IP address. This is done by unplugging your modem/router gateway for about a minute. After your modem re-connects to your Internet Provider, you should have gotten a new IP address. 

If the hacker was DDoSing you at that time, the attack will still go to the old IP address, while you are then using a new address that isn't DDoSed. 

Just make sure you are not accepting party invites or anything that can reveal your new IP to the hacker. Then you'll be safe from DDoS attacks from that person.

Also contact your Internet Provider, so they can log the DDoS attack, and they can then press legal charges against the hacker for abusing their system with a DDoS attack.

xircal say:

Hi Byron,

Sorry to hear that the problem has erupted again.

This could point to a problem with the hard disk. To check, please do the following.

  1. Right click the Start button and go to Command Prompt (Admin).
  2. Type (or copy/paste): Chkdsk /f and hit Enter.
  3. You'll get a message that Chkdsk can't run because volume is in use by another process and whether you want to schedule a diags check on the next restart. Type the letter Y and then either restart, or finish what you're doing and then restart.

Chkdsk will check the drive for errors after which the machine will boot to the desktop. Do the following next:

  1. Right click the Start button and go to Event Viewer.
  2. In the left hand pane, open the tree to Windows Logs and then click Application to highlight it.
  3. Next, right click the highlight and select "Filter current log".
  4. In the menu which opens, change "Anytime" at the top to a time which reflects something a little closer to the time you ran Chkdsk.
  5. In the Event Sources dropdown, scroll down and checkmark "Wininit". I've included a screenshot for you to make it easier to spot (it's a long list).
  6. Next, click OK which should show the log file with "Wininit" under the column called "Source".
  7. Right click that log file and go to Copy --> Copy Details as Text. This will copy the contents to the Windows Clipboard.
  8. Finally, come back  here, right click in the message box and select Paste.

At this stage, you'll only be running a quick diags check which shouldn't take more than 20 mins. There is another switch which can be added to the command which is /r  That causes Chkdsk to check the drive for bad sectors and try to repair them. However, it can take several hours and at this particular stage, the intention is only to find out what the cause of the problem might be.

If you want to retain a copy of the Wininit log yourself, open Notepad and then right click and then paste or hit Ctrl+V. You can either do that first, or after you've posted it here.

Be aware though that if you copy something else, it replaces the log file which means you'll have to open Event Viewer again to copy it once again.
