rwauldron say:

If these issues started since October 2012,  you will find the culprits causing these BSODs are 2 updates whereas Microsoft is pushing the WHQL Compliance and "enhanced" compatibility with Windows 8 and Server 2012.   If they started before then, your system is set to receive drivers from AutoUpdates and is receiving incompatible WHQL drivers.

 

It's a good idea to create "Universal" drivers that are "Microsoft Compliant" but a BAD Idea to push them thru without testing different platforms, devices, and software.  I guess that's our job.

By the way, you will also find that these updates also affect Office  and other security hardware & software products which will hang because the IRQ allocations are colliding with the Kernel mode drivers.

 

The 2 are the Windows User Mode Driver Update Version 1.11 (KB2685813), and the Windows Kernel Mode Drivers Update version 1.11 (KB2685813). 

 

These two updates are reaching into your systems during automatic updates and replacing the correct vendor-supplied drivers with Microsoft "Compliant" drivers for both device & software.  Several key collisions are with symantec software, smartcard readers, display drivers, etc.

Recommended fix: Uninstall the two updates!  First take note as to when they were initially installed.

Easiest method if you have not installed any new software since then is to us system restore an return your system state to the date prior to these updates being installed.

 

If you do not have restore points configured, then do a straight uninstall of the drivers/updates. 

Then run windows update and when they pop up again, right-click on each item to hide them.  Afterward you may have to reinstall the drivers from the vendor that were received with the system.

In addition to hiding the updates, make sure to disable the downloading of device drivers so yours are not replaced again.

Hope this helps,

spiritx say:

Hi,

The correct answers is "sort of" - the Experience rating does not use complex read or write scenarios
to test that of a RAID 0 setup however that setup is not optimized in a single user environment.

RAID 0 performance increase is mainly for servers where there can be multiple random reads and writes
at the same time. So for a home user during most times the performance of a RAID 0 setup would only
be slightly faster than for a single high speed drive (assuming all drives were fairly high speed). Keep in
mind that average performance does not mean there can't be bursts of efficiency if the conditions for
multiple read and/or writes exists even temporarily.

Use Passmark site to compare the performance of your RAID 0 versus that of the same type drives
set up solo - run the Software and compare to same/similar drives.

Passmark - Benchmarks - Hard drives
http://www.harddrivebenchmark.net/common_drives.html

Hope this helps.

Click Ransomware protection
