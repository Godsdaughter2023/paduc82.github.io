A default setting is one that will be used automatically next time.  In other words, if you want whatever action you're taking to be remembered and become the standard way the item is done, set it as the default.  If you are taking a one time action that you may or may not want to always be repeated, don't set it as the default.

nikolai filipov say: In reply to deleted message

BossDweebe, sorry but your answer is clearly wrong. 32-bit programs run just fine on 64-bit OS, it is 16-bit ones that do not.
To original poster, the problem is 16-bit, not 32-bit.

The problem here is 32-bit games with an old, outdated 16-bit installer (usually InstallShield). You can't run a 16-bit installer on Win 64x, thus you can't install the game. If you try to run a Windows 3.x game (which means it is also 16-bit), it will also fail.
On the other hand, if you have Windows 7 32-bit, both 16-bit installer and 16-bit Windows 3.x games should work fine.

Another good virtual machine program is VMware. It has several versions, paid and free.

A short Google search produces a 32-bit installer patch for NFS III, look for it, it might fix your problem.

May the source be with you...
