ariehome say: The culprit has been found.
The problem is that remote computers are not seen with file explorer in the network or cannot be accessed (e.g. with \\compname). After quite some time the computer is still invisible, but can often be DFS accessed with \\compname.

It is caused by Kaspersky Internet Security component Network Attack Blocker on the remote computer.
The new 2016 (16.0.0.614 d) version has several new security features that improve security but also have some side effects. Pausing Kaspersky protection on the remote computer solves the problem.  The problem did not occur in the previous Kaspersky 15 version witn Windows 7, but due to Windows 10 compatibility I had to switch to Kaspersky 16. 
Unfortunately I did not find a setting in Kaspersky 16 that removes the problem.
