wunders say: This would be expected behavior.

When you start the computer or logon to the computer, Windows itself must access and start many programs and processes.  For Windows to find these programs and processes, it must access the C:\Windows directory, the C:\Program Files directory (many startup programs reside here including McAfee), the C:\Documents and Settings directory (to find your profile) and many other directories.  As it accesses these directories, the "Last Accessed" date is dutifully updated to reflect the current date/time.

As you might expect, constantly updating these fields in the NTFS file system has an impact on disk performance.  If you desire, you can disable Windows from updating the "Last Accessed" field for directories/folders and gain back a bit of performance.  To do this, start a Command Prompt window (Start -> Run -> "cmd") and enter the command:
    fsutil  behavior  set  disablelastaccess 1

There is a nice paragraph explaining the details here:
"FSUtil Behavior"
  < http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/fsutil_behavior.mspx?mfr=true >
