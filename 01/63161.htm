taidara say:

Never heard of an end user NAT device that doesn't have DHCP server installed, or at least the ability to enable this feature.

According to the Cox documentation for the Ubee DDW365 DHCP should be listed under Basic settings and DHCP.

Do not confuse DHCP on your router's two interfaces, as one of these is actually facing the public WAN (obtained from your ISP), and the other facing your local network, or internal network, which distribute IP information to any device on this network that request this information.

If you use an older firmware version for your gateway, and at the same time, configure your console with static IP address that are within the dynamically range of the DHCP servers pool of addresses, then you'll have to force a release for this specific address before you assign this to a host machine on your network (e.g. Xbox).

Whenever you assign a machine on your network with static IP settings, make sure you assign this address outside of the DHCP servers pool of dynamically assigned addresses to avoid IP conflicts. Or, you can use DHCP reservation, or in your case, DHCP statically.

I would also recommend to check that Ethernet Cable of yours, to make sure it's not faulty.

      OKI C5100 driver download and installation instructions here. (The - Microsoft Community

      cwm9cwm9cwm9cwm9 say:

      OKI C5100 driver download and installation instructions here. (The C5150 driver is fully compatible w/ C5100 & Windows 7 x64, but vista driver is not. Instructions inside.)

      The Vista 64 driver isn't all that compatible, but the C5150N driver works flawlessly.  (The Vista driver will sometimes clear the screen almost like you've locked the console with a message about how the driver is trying to communicate with you.  It works, kind of, but it's a PITA.)

      Download and install the C5100 Windows x64 (or x86) driver from Okidata.  Then, when you are installing the printer (or doing a driver update from the driver manager), you have to override windows by selecting the "browse my computer for driver software" option, then, "let me pick from a list of devices on my computer," and finally, "have disk..."  Browse to the folder you installed the driver to, and when prompted to select the right printer, choose the C5150 driver.

      Alternatively, if the printer is already installed, you can right click on it and select 'Printer Properties" (Be careful -- there is also a 'properties' and they are not the same thing.)  Switch to the advanced tab at the top, and press 'New Driver..." and click "have disk...".

      There's no difference between the C5150 and C5100 driver that I can tell, except that the C5100 driver is fully compatible with Windows 7 x64.  It flawlessly works with my C5100.

      Be warned if you disconnect the printer and attach it to a different USB port, Windows will reinstall what it thinks is the correct (and wrong) C5100 driver -- you will have to change it again.

      I'm sure someone with the right knowledge could modify the .inf file for us, but it's not that hard to override windows.
