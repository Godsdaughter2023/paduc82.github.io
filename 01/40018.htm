shinmila h say:

Hi kdonn,

 

Windows search should find the file.

 

Let’s see what’s causing this problem.

 

Before doing a search in Windows 7, make sure that you have set your Search Options for the What to search, How to search, and When searching non-indexed locations settings.

 

Step 1: Restore search to default options.

 

You can check this information from here.

 

1.     Go to start and type Folder Options.

2.     Select Folder Options.

3.     Go to Search Tab.

4.     Check the settings.

5.     To restore the default search options, click on Restore Defaults button.

 

Step 2: Check if the location on which the file is stored is indexed.

 

 

1.     Go to start and type indexing options in the start menu.

2.     Double click on the Indexing Options icon.

3.     Click on the Modify button.

4.     If You Don't See All Locations on Your Computer in the List, click on the Show all locations button.

5.     To Add a Folder Location to be Indexed:

a.     Continue to click on the arrows to the right of the check box to expand the folder tree until you reach the location of the folder you want to add, and select it.

b.     Check the selected folder's box.

c.     Repeat the procedure till you add the locations desired.

6.     When finished, click on OK.

 

Check if search works properly after indexing.

 

Step 3: Rebuild the index.

 

It could be possible that the index may not have finshed updating yet wherein you will need to manually rebuild the index to force it to update now.

 

Here’s how:

 

1.     Go to start and type indexing options in the start menu.

2.     Double click on the Indexing Options icon.

3.     Click on the Advanced button.

4.     Click on the Rebuild button.

5.     Click on OK.

 

After rebuilding index, check if the file shows up upon search.

calculator error(s)

I have noticed another bug in the Windows 10 built in calculator.

It only appears in the standard calculator. I know others have reported errors on square root for several years.

How can we trust any of their software if something like a simple calculation doesn't work.

The built-in standard calculator does not take operator precedence into account.

Here is the example of the standard calculator bug:  9000 - 9000 *0.23 = 0.   ***The result should be 6930 not 0.

Therefore it is obvious that the standard calculator is taking 9000-9000 and then taking that result and multiplying by 0.23 giving a bogus result.

Using the scientific calculator:  9000 - 9000 *0.23 = 6930  which is the correct result. (Note: parenthesis were not used in either case "actually there are no parenthesis in the standard calculator for unknown reasons).
