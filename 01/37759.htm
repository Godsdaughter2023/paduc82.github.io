lemp say:
The only approved methods to remove Restore Points are:
  1. Turn System Restore off and then back on; this will delete ALL Restore Points
  2. Run Disk Cleanup, click the "More Options" tab, and click the "Cleanup" button in the "System Restore" section at the bottom to remove all but the most recent restore point

By default in Windows XP, System Restore allocates 12% of your partition for restore points.  If you have a modern large hard drive with a single partition, this allocation is too much.  You should adjust it to about 1 GB.  See http://bertk.mvps.org/html/diskspace.html


On the other hand, a single Restore Point should not be 1.7 GB (especially if you are going to allocate only 1 GB for all Restore Points!).  I'm aware of two things that can cause the creation of extra-large Restore Points (I'm sure there are other causes as well):


Zone Alarm version 6.5 caused this problem.  Because that's a very old version of ZA, it's not likely to be your cause.


Sometimes, the Windows Management Instrumentation (WMI) repository can become corrupted and quite large.  Because this is monitored by SR, your Restore Points will be quite large.


Look in the %systemroot%\System32\Wbem\Repository\FS directory  (%systemroot% is usually C:\Windows) and see if you have a large objects.data file located there.

 See if WMI logging is turned on, and if so, turn it off:  http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/wmi_turn_error_logging_on_off.mspx?mfr=true


Rebuild the WMI Repository --> http://windowsxp.mvps.org/repairwmi.htm

What is BitGuard? - Microsoft Community

susanbolles say:

What is BitGuard?

I found a program on my computer titled: BitGuard.  I have searched the internet for information and cannot find anything.  Does anyone know what this software is?

Many thanks for your help!
roggen say:
BitGuard, by PerformerSoft LLC, is a software component which register the bitguard.dll file under the AppInit_DLLs in the Windows Registry. Thus bitguard.dll will be loaded by each Microsoft Windows-based program that is running on the computer.

BitGuard also register its bitguard.exe file as a Windows Service. This means that bitguard.exe will start before users log on to the machine.

When BitGuard is running, it intercepts and blocks attempts to change some settings in the web browser, such as the home page, search settings and changes to the web browser's toolbar area. BitGuard pops up an alert window where the user is informed about the blocking.

If you have PerformerSoft's BitGuard on your system, you will typically see two "BitGuard.exe" processes running in the Windows Task Manager.

If you want to remove BitGuard, you can do so from the Windows Control Panel. If that failed, here's an alternative approach to remove it:

http://www.youtube.com/watch?v=vh2IYUMv5Qo
