Hi Meebers,

 

 

You may try to refresh the drive letter assigning and then try to rescan the drives that are connect to the computer and check if drive letter ‘H’ for the new zip drive.

To refresh drive letter, file system, and volume information

  1. Open Computer Management (Local).
  2. In the console tree, clickComputer Management (Local), click Storage, and then clickDisk Management.

Now check if drive letter ‘H’ is available to assign.

- removed all Recycle Bins on all disks & rebooted. New Recycle Bins were created, but it didn't helped.

- changed ownership of Recycle Bins and all files underneath (with the command takeown). System complained about corrupted Recycle Bins, emptied them and restored their previous state. This didn't help with the primary problem.

- set maximum size to disk size on all Recycle Bins. This had no effect on the primary problem.

- I also checked tab Security for folders $RECYCLE.BIN on all drives and set "Full control" for Users. SYSTEM and Administrators already had "Full comtrol". This also didn't help.


What is strange (and new) is when I open the Recycle Bin, files are always viewed as "Tiles" (which I didn't set). If I set it back to "Details", the change is not permanent - next time I open Recycle Bin, files are again showed as "Tiles".


What can I do to be able to delete (all) files via Recycle Bin?

 

mrihtar say:

Thanks Ravinath for the answer, but none of the methods did help. There was no malware and even in Safe mode I couldn't delete files to Recycle Bin. Also, the original Administrator account was not able to delete them to Recycle Bin.

 

But I think I found the culprit:

This problem apparently showed only in Downloads folder, to which I had many junctions (to be easily accessible). Because of lack of space on disk C: I remembered I moved it via Properties -> Location to a disk with more space. In place of old Downloads folder in C:\Users\<user> I put a junction pointing to the new location.

Summary:

Delete file (SHFileOperation?) is not able to use Recycle Bin, if file is accessed via 2 (or more?) junction points.
