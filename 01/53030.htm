Not listing all the fixes you have already tried is an invitation to read the same suggestions again.

It can also be helpful if the system specifications, including brand and model graphics processor are listed.

- Crashing/freezing/ lost signal are very unlikely to be DX related. Heat, inadequate power supply ( desktop PC ),

and software conflicts are more common suspects with these symptoms.


- DX12 does not come into it. This is a DX11 exclusive game (exclusive in that it does not have DX9/10 options).

Games use specific DX files, and games that don't use DX12 are not affected by that file.

- Regarding  XAudio2_7.dll . This is a DirectX file, and  _7  is one of the more 'recent' version of this file.

- One possibility for an XAudio  error is not having installed a suitable Win10 version of your audio driver

(from the mfg of your machines motherboard, or mfg of the machine, if using a laptop. )

- If there is actually a problem with the DX file, and if a Windows install has never had DirextX updated this

Hi,

 

Welcome to Microsoft community and thank you for posting the question. I will surely help you to find a resolution regarding the issue.

 

1.       Have you tried connecting the Hard drive to a different computer?

2.       What happens when you plug in the Hard drive?

 

It appears that the hardware is not detected on your computer. You should check if the External Hard disk shows up in the Device Manager after plugin to the 3.0 USB port.

 

1. Click on start and type Devmgmt.msc in the search field.

2. Check if the External Hard disk shows up there.

If the External Hard disk shows up in the Device Manager, check the status of the device if you find any Yellow bangs or exclamations next to the device.

 

 

Method 1:

 

Run the hardware troubleshooter and see if it helps.

 

Open the Hardware and Devices troubleshooter

http://windows.microsoft.com/en-IN/windows7/Open-the-Hardware-and-Devices-troubleshooter

 

 

Method 2: Unplug the external Hard Disk Drive

 

a. Click start, type "cmd" in the start search option.

 

b. At the command prompt, type the following lines, pressing ENTER after each line

 

set devmgr_show_nonpresent_devices=1
cd\ %SystemRoot% \System32
start devmgmt.msc

 

 

c. Device manager will open

d. Click show hidden devices under View...

e. Expand all the nods under device manager

g. There would be many greyed out entries in device manager uninstall all the greyed out entries under all the nods and then restart the computer and plug the Hard Disk Drive to the computer and check if it works.

 

 

Method 3: Remove and reinstall all USB controllers

 

 

To remove and reinstall all USB controllers, follow these steps:

a. Click Start, click Run, type sysdm.cpl in the Open box, and then click OK.

b. Click the Hardware tab.

c. Click the Device Manager button.

d. Expand Universal Serial Bus controllers.

e. Right-click every device under the Universal Serial Bus controllers node, and then click Uninstall to remove them one at a time.

f. Restart the computer, and then reinstall the USB controllers.

g. Plug in the removable USB storage device, and then test to make sure that the issue is resolved.

 

 

If the issue still persists, I would suggest you to contact Toshiba support for further assistance. You may check if the Hard drive is configured to work with 3.0 USB port.

 

 

Refer:

http://www.toshiba.com/taec/support/support_main.jsp

 

 

Hope this information helps. Please reply back with the status so that we could help you further.
