borderrider say:

On the various message boards the instructions for doing robocopy were vague, and even wrong.  What follows applies to Windows 10 and Office 2016, and may work with other MS operating systems.  I am copy/pasting below what I did and have saved in a Word doc for future reference:

MOVING FOLDERS OR FILES FROM DVD TO “C” DRIVE

 

  1. Using File Explorer determine Source and Destination file paths (shown in path window).
  2. With mouse “Copy Address As Text”, then Paste Source and Destination file paths in Word Doc.  The last item in a path is a folder, not a single file.  I never could get a single file to copy with this method. 

Source: “E:\My Documents\Excel\EOG\E.TEX.WORKOVERS”

Destination: “C:\Users\Jim\Documents\stuff”

  1. Combine the Robocopy and file paths, as in sample below:

 

Robocopy “E:\My Documents\Excel\EOG\E.TEX.WORKOVERS”

 

               Note: /E = Copy Subfolders, including Empty Subfolders.  This captures all file & folder info. 

 

  1. In “Ask me anything” type “Notepad”, and enter.
  2. Copy/paste the Robocopy command from Word into Notepad.  (Notepad “cleans up” hidden junk that exists in Word).
  3. Right click on Start button, open “Command Prompt”.  Go to DOS command prompt.
  4. Copy/paste this Robocopy command from Notepad into the cursor, and Enter. 
  5. The files should now be in the desired “C” folder (stuff), with correct time stamps, After listing all files copied, at the bottom of the DOS display it will summarize the number of folders and files copied, and provide the total MB copied. 
  6. Right click on file and choose “Properties”.  Uncheck the “Read Only” box and click OK.  Now the file can be altered Without having to save as a New File. 

OR

  1. Highlight all files in folder, Right click on one file and chose “Properties”.  Uncheck the “Read Only” box and click OK. 

 

 

CAUTION:

When a file is on the C Drive, must drag/drop to preserve the Create and (Last) Modified dates.  A Copy/Paste of a file will LOSE the Create Date, and re-name it as “Modified”.  Ridiculous!

 

 

 

The Following Commands were used to transfer files from a DVD in the "E" drive to the "C" drive, and serve as examples.  And yes, there are spaces after "robocopy", source path, and destination path:

 

Robocopy “E:\My Documents\AutoCad” “C:\Users\Jim\Documents\My Documents\AutoCad” /E

 

Robocopy “E:\My Documents\Excel” “C:\Users\Jim\Documents\My Documents\Excel” /E

 

Robocopy “E:\My Documents\Word” “C:\Users\Jim\Documents\My Documents\Word” /E

Lisa,

Sometimes a weak CMOS battery can cause various issues including slow, locking as well as other unexplained performance issues.

I have yet to see a laptop whose battery is easy to replace . . .

Here is a simple test to check if the battery needs replacing:
  1. Boot the machine into Safe Mode.
  2. Set the date and time.
  3. Shut the machine down.
  4. Remove the bit battery and disconnect the machine from the mains.
  5. Wait at lease 24 hours.
  6. Boot the machine back into Safe Mode.
  7. Check the date and the time.
If the date/time is incorrect then you are probably dealing with a flat motherboard battery. You probably need to get the machine serviced to have it replaced unless you have good skills in disassembling electronic components.
