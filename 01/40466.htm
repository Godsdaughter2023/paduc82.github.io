wolfy762 say:

I actually was able to fix mine....

Essentially I needed to use the “metered connection” setting as a screen to prevent auto install of drivers/bloatware/malware (or any other unwanted update) and once the updates where trapped use wushowhide.diagcab to hide them.

1. Clean install of windows 10 with USB 3.0 16 drive/or restored from disk image

2. Did not set up wireless at first boot options

3. Changed >System Properties>Hardware>Device Installation Settings: No, let me Choose what to do/Never install driver software from Windows Update but checked the get device info

4. Changed >Control Panel\Network and Internet\Network and Sharing Center\Advanced sharing settings>Private profile>turn on network discovery UNCHECKED Turn on automatic setup of network connected devices.

5. Changed: Settings >Update & security>Windows Update>Advanced Options>Choose how updates are installed to: Notify to schedule restart

6. Allowed Wireless to connect

7. Changed: Settings >WI-FI>Advanced Options Set as Metered connection to ON

8. Joined the homegroup

9. Downloaded wushowhide.diagcab

10. Selected: Settings >Update & security>Windows Update> Check for Updates

11. Warnings for drivers and bloat come up and complains about the Metered connection

12. Ran wushowhide.diagcab >Next>Hide Updates: selected the updates/software/bloatware/malware I didn’t want.

13. Logout/Log back in or restart (Windows Update would not give me a retry option until I did)

14. Selected: Settings >Update & security>Windows Update> Retry

15. Selected: Settings >Update & security>Windows Update> Details (to verify the whole list)

-Update gave me a list minus the **** I hid

16. Selected: Settings >Update & security>Windows Update> Download (Good updates/security definitions etc)

Specifically for the Printers on the Network that are shared, I was having a problem with two sets of the same printer wanting to install. My solution was use the WSD or LAN search in the printer install software (ONLY-had IPV4,IPV6,WSD as options, all 3 checked - and may not be relevant for your printer). Apparently the WSD is an automatic port created when the private network profile has “Turn on automatic setup of network connected devices” checked. I was able to stop this WSD printer version from showing up by unchecking it or using a public non sharing profile.

Don’t let anyone tell you that step 3 fixes anything by itself, it seems to only have an effect on the printers and devices area but makes the devices visible for windows update to go install whatever it thinks is good.

Hope this helps

paul.zimmerman1993 say: Successfully printed and scanned with SCX-4100 in Windows 8 (after many failed attempts)

Steps:
1. Loaded the Universal print driver for SCX-4100 and Universal scan driver for SCX-4200 (YES, 4200) from Samsung support website.
2. From Win 8 "Search", select "settings" (not Apps) and type in "Device Manager" and select it from the list. 
3. Look for Imaging devices.Double click on Samsung Universal Scan Driver. Mine shows Driver version 1.2.5.0 (5/15/2010). If there is no Imaging device listed, reload the scan driver.
4. From the "SCX-4200" support download page, select "Software", instead of Drivers.
5. Download and install SmarThru office 2 (Software) (ver.V2.10.00.07)
6. When I installed SmartThru Office, it put a 4 button icon in my system tray. 
7. Click on the scanner icon. My 4100 appeared in the select scanner window. I was able to prescan and scan and even save as a PDF. 
8. Don't bother with the The SmartThru Office icon that appears on your desktop. There are no scanner settings there.
9. Get rid of SmartThru4.

** Note to Samsung support: Put these drivers on the SCX-4100 driver page.

Good luck!!!

Paul

alaor miguel say: I also had this problem. I saw at a website a general solution for hash drivers problem in windows 8 and worked for me.

Do it at your own risk.

1. Open the SPLV1.INF file (it can be found at c:\temp\scx-4100\printer\gdi\vista or vista_64)

2. There is a code line: CatalogFile = SPLV1.cat

3. Change it like this: ;CatalogFile = SPLV1.cat          (add a ; in the begining)

4. Restart windows and disable the file signature. You can see how in this video: http://www.youtube.com/watch?v=NM1MN8QZhnk

5. Run again the driver instalation

6. Answer YES when Windows prompts you about installing this driver

7. Ready!

Let me know if it worked for you.

Yes you can go to 64 bit. You must do a clean install (Custom install)
