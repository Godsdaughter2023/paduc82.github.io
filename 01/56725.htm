svchost.exe is a generic process that, as the name suggests, hosts services.  You'll have to first determine which service is the one that's using the resources and then figure out why.

You can start with the built-in tools.  In Task Manager, use the View menu to add a column to display PID.  Then open a Command Prompt window and type

tasklist /fi "imagename eq svchost.exe" /svc

and press Enter.  This will display information about all of the svchost.exe processes and list which services go with which PID.

Unless you see something pretty unusual in the list of services, however, you'll have to dig deeper.  For that, get the free program Process Explorer from Microsoft.  Process Explorer is a powerful tool, and using it is beyond what can be put in a simple forum answer.  Start by reading some of the following:

Search 1

Search 2

You can also try a more brute force approach.  Once you figure out the list of services in the instance of svchost.exe that is using the resources, you can stop them one by one [Start > Run > services.msc > OK] until the resource use drops.  Be aware, however, that randomly stopping processes may cause your computer to behave strangely.  Also note that some services are dependent on other services -- you can't stop the top level service without also stopping all of its dependent services.  For a great deal information on services, see Black Viper's website and wiki .

lemp say:

I wrote the material below the line of asterisks before I re-read your original post.  A very likely cause of your problem is a conflict between Norton and Avira.  You shouldnever have two anti-virus applications active at the same time, and there are those who go so far as to say that you should never have two anti-virus applications installed.  Although I think the latter is a bit extreme, some a/v applications -- such as Norton -- are virtually impossible to turn off completely.  Moreover, several a/v applications -- I know this includes McAfee and I suspect Norton as well -- have "anti-tamper" technology that will prevent you from stopping them using Windows tools like services.msc.

If I were you, I'd remove Norton using the appropriate Norton Removal Tool , although you could optionally remove Avira instead.

After you preferably remove (or at the very least disable from real-time monitoring) one of your a/v applications, reboot and see if your svchost issue remains.  If so, read on.

****************************************************************

I'm not expert in the use of Process Explorer or its related tools, including Process Monitor, so I can't give you step-by-step cookbook type instructions. That said, there are a few basic things you should know before you start trying to dig into your problem.

Services should be controlled by running services.msc from the Start > Run box. When you run this, you'll see a list of all of the services available. Selecting a service will give you a very brief description of what it does, and you can also see whether it's running or not and what the "startup type" is set to.  The startup type determines what happens when your computer is rebooted:  "automatic" starts the service automatically; "manual" starts the service if and when the service is required; "disabled" prevents the service from starting.

Read Black Viper's website for introductory information.  Black Viper's wiki explains what each service does. Be sure to read the section for XP.  A similar source of information is The Elder Geek'sServices Guide .

The first thing to do is to go through the list of 29 services and confirm that each of them is, in fact, a genuine Windows service and not malware.  Clever malware may run as a service just because it then doesn't appear in the list of processes displayed in Task Manager.

Note The Elder Geek's advice: "Changing services settings can be just as dangerous as editing the registry. Have current -tested - backups of your system/data and understand how to restore them if something goes terribly wrong."  If you have a disk imaging/cloning application such as Acronis True Image, Ghost, or Casper, I strongly suggest that you make an image before you start experimenting.
