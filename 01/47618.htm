Skype does not have a built-in spell checker.  It interfaces with the spell checker that comes with Windows 8/10.   Older OSes do not come with a spell checker and 3rd party software would have to be used to inject support into Skype.  To use the spell checking option in Windows 8/10 be sure the following are enabled or configured accordingly.

1.  The setting within Skype as seen below:


2.  The setting in Windows 8/10.

3.  The primay/top-most language in Windows 8/10 is set to the lanuage you want the spell checker to proof against.  In the image below, this would be English.  It may be possible, now or in the future, this dynamically changes.  This is to ensure you do not run into problems, if it does not.

4.  Make sure compatiblty mode is not set to a previous OS that has no spell checker support.  The spell checker may still work in that mode, however, you will not be able to enable or disable the spell checking option within Skype (as seen in point #1).  Additionally other problems may arise.

5.  Make sure a space exists after the word, otherwise it will not know it is a word to be spell checked.

If you are still experiencing problems this may point to:

1.  An underlying issue with Skype. 

    a.  Uninstall and reinstall Skype

    b.  Reset your local configuration and settings.  You can do this by renaming the Skype folder located in %appdata% to something else like SkypeOld.  This way a new Skype and profile folder will be created  after you launch and sign into Skype.

2.  A driver issue causing Skype not to function properly.  Update your audio, video and camera drivers.

3.  3rd party software causing Skype not to function properly.  This can be antivirus restricting access to the spell checker or other software causing Skype not to function properly.
4.  An underlying disk/OS issue.  Running a check disk or system file checker might resolve the issue.

Hello Graham,

Thank you for posting your query in Microsoft Community.

I appreciate all the efforts you took to fix the issue. We are glad to help you ahead in fixing the issue.

The backup may still fail, and you may receive the following error code: 0x8007045D.

These conditions may occur after you run chkdsk on the backup source and on the destination locations. In this case, you may be experiencing a rare issue where backup fails, and you receive the 0x8007045D error. This rare issue can occur when the backup fails because the last cluster on the volume has gone bad. You should be aware that chkdsk.exe does not handle this situation.

You can shrink or extend the volume by several megabytes to move the last cluster of the volume to a different area. Then, you can run chkdsk again.

Shrink a Basic Volume

http://technet.microsoft.com/en-us/library/cc731894.aspx

And to perform check disk in Windows, follow the steps mentioned below and see if that helps:

a) Click Start, type cmd, right click on cmd and click Run as Administrator.

b) On the prompt, type chkdsk /f /r and hit enter.

c) Try to backup and check the result.

For more information on "Check a drive for errors" check this link:

http://support.microsoft.com/kb/2854570

Important: While performing chkdsk on the hard drive if any bad sectors are found on the hard drive when chkdsk tries to repair that sector if any

Also, run System File Checker (SFC) because, if a Windows Resource Protection (WRP) file is missing or is corrupted, Windows may not behave as expected. Run System File Checker (SFC) scan to fix any corrupt system files. To do this, follow the steps mentioned in the link below:

Use the System File Checker tool to repair missing or corrupted system files

http://support.microsoft.com/kb/929833

Hope this helps. Otherwise, feel free to write us back for further assistance and we will be glad to help you ahead.
