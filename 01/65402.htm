Hi

This is by design and is implemented to prevent a very large number of files being accidentally opened and possibly running the system out of resouces and causing a crash.

If you have a powerful system, with a lot of RAM, there is a fix for the problem that involves editing the registry.

Warning: Serious problems might occur if you modify the registry incorrectly by using Registry Editor or by using another method. These problems might require that you reinstall the operating system. Microsoft cannot guarantee that these problems can be solved. Modify the registry at your own risk.See   How to back up the registry

 

Press the Windows Logo Key+R to bring up the Run dialog box.

In the Open line type: regedit.exe and press OK.

In the Registry, navigate to:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer

With the Explorer key selected, in the right side window, right click and select

New/DWORD (32-bit) Value.

Name the new value   MultipleInvokePromptMinimum (Case sensitive and no spaces)

Right click the new value and select Modify.

In the Base section, select Decimal.

In the Value Data box, type the number you wish to use. (16 will make the number of files selected unlimited)

Click OK.

Exit the registry editor. 

You will need to log off / log on to see the change.
