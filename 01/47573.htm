lemp say:

1.  You are not seeing the Content.IE5 folder because it has the "System" attribute in addition to the "Hidden" attribute.  If you want to see it in Windows Explorer, you have to go to Tools > Folder Options > View tab and UNcheck the box to "Hide protected operating system files (Recommended)" and then click OK.

2.  Or you could see it by following Frederik Long's advice.  You forgot to put the "dot" (.) between "Content" and "IE5"

3.  Normally, Internet Explorer does not save files there.  To change the default location for saving files:

  • Open Internet Explorer
  • Click the "Tools" menu and select "View downloads."  If you don't have the menu bar visible, press the ALT key to see the menu bar (which includes "Tools").  Or find "View downloads" after clicking the "gear" icon at the upper right of an IE window.  Or just press CTRL+J
  • In the "View downloads" window, click the "Options" link at the lower left
  • Use the "Browse" button to select the folder where you want downloads to be saved.  Click on the desired folder and click the "Select folder" button.
  • OK your way out

4.  When downloading files in IE, you can click the arrow on the "Save" button to select "Save as."  This will allow you to save a download to a location other than the default location.

How to open old emails in Windows 8 - Microsoft Community

vernknoop say:

How to open old emails in Windows 8

How do I open old emails in Windows 8?
trevor j lawrence say:

By default the included Mail program only keeps the most recent 14 days worth of mail.  You can, however, change this via the app's settings.  With the app open, pull out the Charm bar and hit Settings.  Choose "Accounts" and select the account in question (if you have more than one).  There is a drop-down labelled "Download mail from" with choices "Last 3 days", "Last 7 days", "Last 2 weeks", "Last month" and "Any Time".  The latter choice will download all mail off your server and keep a local cache of it.  I use that choice, but it depends on how big your mail store is and how much space you have on your machine.

 

Note also that if you use certain mail service providers the Search capability in the mail app will search not just what is cached locally, but also the whole folder on the server.  So that if you only keep 2 weeks worth of mail cached locally, you can still search for (and read) older mail.

The basic premise seems to be that the IE9 installer that is online in Windows Update and even the IE9 consumer site is bad. So if we try to pull the installer from these online sources after April 10, we would always get a bad copy, whether it's from Windows Update or from a manual installation. You would know you have a bad installation if the version number in the "About Internet Explorer" dialog does not match the version number stored in the iexplore.exe executable. A good installation would have the version number displayed for both as 9.0.8112.16421. On my other computer with an old installation that updated without problems, this is indeed the version number being displayed.

Based on this observation and taking a cue from Spectrum7glr's post about installations prior to April 10 working perfectly, I tried to locate an installer that would have binaries before April 10. Luckily, I was able to find one from Microsoft's Download Center.  Here's a link to the version for Windows 7 and Windows Server 2008 R2, 64-bit:

http://www.microsoft.com/download/en/details.aspx?id=23332

Instructions:

  1. Uninstall IE9, if you already have it installed, and restart the computer.
  2. Download the installer.
  3. Disconnect from the internet. This is important because the installer attempts to download the latest, bad binaries from Windows Update.
  4. Let the install finish, and restart.
  5. Reconnect to the internet and check Windows Update. If all went well, you should be able to see the Cumulative Update for IE9, instead of IE9 itself.

Again, the key is to install without any internet connection, to prevent the installer from fetching the latest and bad binaries.

This kind of error is almost always caused by file system corruption.  Whenever you see it, your first action would be to make sure that any files that you consider important are safely backed up on another disk somewhere.  Your second action would be to perform a complete disk check (chkdsk) with the /R or "Scan for and attempt recovery" option.  As a warning, this will be performed on a reboot before your computer fully loads Windows, can't be interrupted, and can take upwards of a couple of hours to perform depending on hard disk size and amount and type of corruption.  The procedure is here:

"How to perform disk error checking in Windows XP"
  < http://support.microsoft.com/kb/315265 >

If you continue to see this error after a complete disk check, this would likely be indicative of a failing hard drive -- so either replace the drive or be careful about committing only copies of important files to this disk.
