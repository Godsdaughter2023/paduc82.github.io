rogeroriginals say:

Technically speaking, they both are the same thing. Disk defragmentation or optimisation is a set of processes that are used to increase the efficiency of hard disks.

Hard disks store data in form of "blocks", which are written and retrieved by use of "pin-heads", also called as 'read/write heads'. Imagine that you're walking on a uniformly tiled sidewalk; The sidewalk is your HDD, tiles are "blocks" and you are the "pin-head"

  • In a brand new sidewalk, if you want to fill it with data, you will have to walk over every consecutive empty tiles, starting with the first tile, until all the necessary data has been written. This is how HDD's write data when used for the first time.
  • You want to delete some data? You'll have to walk over only those tiles containing the data which needs to be erased until it's done. This creates empty tiles between tiles filled with data, which can be later walked on if you want to fill those with new full ones in the process. That is how HDD's fill blocks-they don't move on to the next ones until all before them are filled, including those empty blocks located between filled blocks.

As we can see, one single file data can be scattered over many empty blocks, especially for big files. This makes it slower for HDD to read data when pockets of free blocks exist between filled-blocks, as it has to check all the blocks until the entire data is obtained.

Defragmentation is the process of rearranging data in such a way that the empty blocks are moved to one side and data-filled blocks to another side. The data-filled blocks are further grouped so that same files' data are grouped together, making it easy and faster for HDD's to read them and retrieve data. Because we are improving the speed of HDD's by defragmenting them, we are technically optimising them.

andre da costa say:

If you knew the answer, why bother asking for assistance then?

Because, the way I interpreted your original post, you said:

I am using virtualbox on a dual boot system.

I assumed that meant, you are running Windows XP and Linux Mint in a dual boot configuration.

I assumed from this that you installed Windows 7 Home Premium upgrade in Virtual Box on Linux Mint and thinking because you have a Windows License (XP) installed, you think you qualify to use it elseware in the system even if its a separate partition. The point is, if the Windows XP License is installed on another partition, you can't use it to qualify for the upgrade in Virtual Machine on another partition, because the license is already "in use".

You would be out of compliance that way, but I am sure no one is gonna come knocking on your door for such a loop hole.

UPGRADES. To use upgrade software, you must first be licensed for the software that is eligible for the upgrade. Upon upgrade, this agreement takes the place of the agreement for the software you upgraded from. After you upgrade, you may no longer use the software you upgraded from.

So, if you installed the XP just to qualify for the Windows 7 Upgrade, you would have to remove it from the physical disk partition its already installed on.

z3braman say:

That seems like a somewhat lazy answer.

 

I would suggest a better answer is to address the present need.  I have a legal license for windows 7 home premium (3 licenses.  Presently, two are in use).  What do I need to do to transfer the license to the virtual machine?

Answer.  Since the license is being blocked on the basis that this is a clean install you can call in and explain to MS the issue.  To use the license there must be a legitimate qualifying installation to upgrade.  Since the hardware had such an installation prior to the hard installation on the computer the machine met that qualification.  In order to use those circumstances for the virtual machine the hard installation must go in order to use the virtual installation.  In effect, this would be identical to getting a new hard drive and transferring the existing license to it (of course the virtual machine has other unique identifiers such as the virtual mac address that triggers bonus points on the calcualtion for registering a significant change in hardware).

Another solution would be to install a legal copy of xp on the virtualbox and use the third license to upgrade that leaving the hard installation intact.
