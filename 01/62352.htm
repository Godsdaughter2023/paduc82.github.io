Hi all,

 

because I just definitely solved nearly same issue (main difference is minor indeed) I experienced while trying to start Books Online for SQL Server 2012 (from SQL Server 2012 SP1 Express) after my last Windows 7 Ultimate SP1 reboot see below for more details on a known solution.

 

First of all I already knew that Microsoft Help Viewer 1.1 (both HelpLibAgent.exe and HlpViewer.exe are v1.0.40219.1) was working fine because I've used it until this morning.

Secondly because I knew that my main changes were minorly related to IPv4 and Windows Firewall components (if interested I installed KB2675785 QFE so for more details just look there) but also IPv6 (from Control Panel-Network and Internet-Network Connections -> Advanced (menu) -> Advanced Settings I simply moved it higher then IPv4) I wasn't expecting to see nearly same error reported by David_LS image.

    netsh http add urlacl url=http://[::1]:[port]/help/ sddl=D:(A;;GX;;;WD)

    netsh http add urlacl url=http://[::1]:47873/help/ user=\Everyone listen=yes delegate=no sddl=D:(A;;GX;;;WD)
