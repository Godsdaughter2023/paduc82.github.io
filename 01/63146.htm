Rundll32:

If you’ve been around Windows for any amount of time, you’ve seen the zillions of *.dll (Dynamic Link Library) files in every application folder, which are used to store common pieces of application logic that can be accessed from multiple applications.

Since there’s no way to directly launch a DLL file, the rundll32.exe application is simply used to launch functionality stored in shared .dll files. This executable is a valid part of Windows, and normally shouldn’t be a threat.

Hello Teddy,

You can try to perform these steps on how to turn off autofill. Please follow the steps below:

  1. Press the Windows key + R > type inetcpl.cpl > press Enter.
  2. On the Advanced tab > uncheck the Use inline AutoComplete in File Explorer and Run Dialog then click OK.

Alternatively, you can also modify the Registry to disable autofill in Windows Explorer:

  1. Press the Windows key + R > type regedit > press Enter.
  2. Go to the path: HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\AutoComplete
  3. Right-click on Append Complete > then select String Value from the context menu.
  4. Set the Value data to "No" > click OK.

jeffs33 say:

Limited CPU clock speed on battery power

I have just noticed that when on battery, my Surface Pro 4 runs at 1.17 GHz maximum.  If I plug in the power cord, it immediately jumps to the expected maximum of 2.5 GHz or more.  I am seeing this when processing video files with Movie Maker, which is CPU intensive work.

I haven't noticed this before, and wonder if this is a new feature of the Anniversary Build to try to extend battery life.  Is there any way of controlling this CPU constraint, which may of may not be helpful - depending on what you want.

jeffs33 say:

I found out what was going on.  For some reason, the maximum processor state was set to 50%.  This parameter isn't visible normally, and it needs a registry setting to have it show in advanced power settings.  I have set this up to 100% and things work as I expect now.

Just as a note, in looking at this carefully, on power the clock rate is 2.99 GHz, whereas on battery it drops to the stated maximum of 2.5 GHz.  So, a bonus for being plugged in of nearly 20%?
