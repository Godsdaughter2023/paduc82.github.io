hairyfool say:

Boot from the recovery Media but use the repair option and get to the command prompt Window

At the command prompt run Diskpart, this is a text based utility akin to Disk Manager.

Use List Disk to enumerate installed drives. In fact this tool is both powerful and unforgiving so will do whatever you tell it. If possible it is better to only have the drive you want to work on installed then there can be no mistake.

Select Disk n forces the focus onto the particular drive.

Clean will erase the boot sector completely and return it to as supplied by the factory. It resets the WHOLE DISK so be absolutely certain this is what you want.

Partitioning under Linux seems to configure the drive in such a way that the normal erase under Disk Manager cannot remove it completely so we have to revert to DiskPart. I reiterate, it does what you tell it and has none of the "are you sure" protections of Disk Manager.

For further guidance on DiskPart read this article here:

https://technet.microsoft.com/en-us/library/bb490893.aspx?f=255&MSPPError=-2147217396
