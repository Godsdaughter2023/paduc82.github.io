HP Smart Install was a feature that HP included in the firmware of some of its printer models, including the LaserJet Pro 400 Printer M401, LaserJet Pro 400 MFP M425, LaserJet Pro M435 Multifunction Printer, and LaserJet Pro 400 color Printer M451. 

When you connected one of these printers to an Internet-connected Windows computer, the printer "phoned home" to HP, selected the appropriate driver, and installed the printer.  It's not clear to me whether the driver was downloaded from HP or whether it was somehow embedded in the printer's firmware.  See http://www.hp.com/hpinfo/newsroom/press_kits/2010/plugandprint/pdf/Smart_Install_FAQ.pdf

At some point -- and I don't know when -- HP discontinued this program and removed the servers to which the printers were attempting to connect.

You should do several things.

First, try to disable Smart Install.  If you have the original CD that came with the printer, there allegedly is a SIUninstall  (or SIUninstall64) utility that supposedly has the ability to disable Smart Install.  I've also seen a post that asserts that Smart Install can be disabled directly from the printer's front panel controls:


  • Click OK for the menu
  • Push the right arrow 3 times to get to the service menu
  • Then Push the left arrow (I think 3 times) to get to the Smart Install options
  • Then choose off.

Whether or not you can disable Smart Install, you need to install the driver the old-fashioned way, using the correct download from HP.com.  The name "LaserJet 400" seems to be a bit too generic, so I can't point you to the exact HP web site, but you should be able to find it with not too much difficulty.  Here, for example, is the download page for the "LaserJet Pro 400 color Printer M451 series" -- https://support.hp.com/ro-en/drivers/selfservice/hp-laserjet-pro-400-color-printer-m451-series/4346082

Here's my usual first step in clearing out bad printer installs, but given what you've done so far, it may or may not help.

1.  Disconnect the printer from the computer

2.  Right-click the icon for the LaserJet 400 in Devices and Printers (if it's still there) and select "Remove device"

3.  Select any other item in the "Printers and Faxes" section and then click the link in the upper tool bar for "Print Server properties."  Select the "Drivers" tab

4.  Select all HP drivers related to the LJ400 and click "Remove."   When asked, choose to remove both driver and driver package.

5.  Reboot and attempt to reinstall the printer using the downloaded installer file.

If that doesn't work, you'll have bring out the heavy artillery. 

Microsoft used to have an automated "Fixit" utility that dealt with printer problems.  This is described here:  https://support.microsoft.com/en-us/help/2793718/you-experience-print-spooler-error-messages-after-installing-or-upgrad  Note that there are two "modes."  I suspect that you'll need the "full" mode, but this uninstalls all printers from the computer and requires a reboot.

In its zeal to move everyone to Windows 10, Microsoft "retired" the Fixit utility described above (as you'll discover if you click on one of the links in that article).  Fortunately, the developers of the utility preserved the guts as a Powershell script that you can download and run.

  1. Disconnect the printer from the computer if connected by USB
  2. Go to https://github.com/jdickson289/Print-Reset-Tool/ and download print-reset.ps1 and make a note of where you save the file
  3. Click Start, type PowerShell into the Search box, right-click on the link to Windows PowerShell, select "Run as administrator" and respond as needed to the UAC prompt 
  4. Type the following command into the PowerShell window and press Enter
    Set-ExecutionPolicy Unrestricted
  5. In response to the "security risks" question, type Y and press Enter
  6. Type the following command into the PowerShell window and press Enter.  In typing the command, use the full path of the location where you saved the print-reset.ps1 file.  In the example shown below, print-reset.ps1 was saved in C:\Temp
      C:\Temp\print-reset.ps1 -full

    You can try using -light first if you like.  You won't have reboot after doing that.  However, as I suggested above, you're probably going to need -full to restore everything that's been messed up

  7. Type R to run the script
  8. After the script completes, restart the computer
  9. Run the installation package and follow the on-screen connections to connect the first printer
  10. Go to Devices and Printers and confirm (a) that the printer is installed and set as the Default Printer and (b) right-click on the icon, select Printer properties, and click "Print Test Page"

This is what the PowerShell window should look like as you proceed through steps 3 through 7 (I didn't enter R because I didn't want to reset my own printing system).  Don't forget to use the actual location where you saved the print-reset.ps1 file rather than C:\Temp as shown below.
