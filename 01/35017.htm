Have a look at The Windows 7 startup process is slow when you create many restore points

This issue occurs because the boot plan for the ReadyBoot feature exceeds the size limit of 512 kilobytes (KB). Each restore point creates a snapshot of Windows that Volsnap.sys must validate during the startup process. When you create many restore points, the boot plan for the ReadyBoot feature eventually exceeds the size limit of 512 KB, and cannot be stored. Therefore, startup I/O operations are not precached, and the startup process is slow....

bossdweeb say:

Different versions of C++ should not be look at as 'updates'.  C++ versions are not cumulative.

The software that uses C++ libraries is very often coded to search for a specific C++ version, so while your

machine may no longer require a specific C++ because you have uninstalled a software that accessed it,

and you may never have a problem resulting from uninstalling that version, you may also be hit one day with an obscure error message that neglects to mention C++ 2008 (x.xx), or one of it's component .dll files.

Then you're potentially up for

Some (a lot ?) of software that may have be developed years ago may still use a C++ version from '08 or even '05 with

newer updates versions of that software.

The good news is, that while they look like a lot when viewing Programs and Features, but they use only a bit over 100MB HDD/SSD space. Other than that they are benign in that they do not run as executables, and use cpu or RAM

until accessed by the relevant software.

- More good news . Reinstalling the software that needs version X will also generally reinstall the required version

of C++.  Best to just leave them regardless.
