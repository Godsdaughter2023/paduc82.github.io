robinblackdiamond say: This might be related to NTFS permissions.  So the permissions that the Windows file system assigns to each file object (files, folders, etc).  So you might be able to connect to the Share based on your Share permissions, but the next layer is the NTFS permissions.  If you're looking at folder properties it will be located under the Security tab.  If your notebook doesn't connect using an account that has permissions (either a local account to the system your connecting TO or a domain account that has valid permissions on the system your connecting TO) it will block the access to the file/folder.

If you're environment is a Windows Domain and your notebook isn't a member of the domain (AND you're logging on using a Domain Account) I would suggest joining it to the domain and using a domain account that has been given both SHARE and NTFS permissions to the folder you're trying to reach.

One thing you can do to test access is trying to reach the same share from the system that is sharing the folder also.  See if that works using the account you're using from the notebook.  If that works, then you know there might be something on the notebook or network that is blocking your access.  Most likely it's related to permissions as you're getting Access Denied.  That means that you're reaching the resource, but the permission is denied.  If you weren't able to reach it you would get an error like, "The network path was not found."

Good luck!
