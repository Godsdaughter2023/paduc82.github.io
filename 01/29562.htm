I had installed Linux before, so I think this had affected the EFI partition even after reinstalled windows 8... Some after many researchs on internet I found some useful information to fix this.

First, you need to: 

Go into the Advanced options and run the Command Prompt.
Type DISKPART."

Now you need to remove your EFI partition:
type: list disk
(find your principal device and select it)
select disk "X" (number of you device, probably 0)

Then: list volume
Find your EFI Partition(FAt32)"X" and then remove:
delete volume "X" override ( replace the "X" with the correct number)

Now, create another:

> create partition efi size=260

> format quick fs=fat32 label="System"

> create partition msr size = 128

> list volume

> select volume X (replace the X with the number of new partition)

> assign letter=b:

> exit

>bootrec /fixboot

>bcdboot c:\Windows /s b: /f ALL

 

bcdedit /set {default} hypervisorlaunchtype Auto
bcdedit /set {default} nx OptIn

Now, restart your pc and try to upgrade again.
