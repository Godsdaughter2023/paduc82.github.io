For an answer more specific to your particular error, you would have to ask the printer manufacturer's tech support.

However, before you start doing things, please answer the following questions:

What is the make and specific model of the printer?

What edition of Windows do you have, including 32-bit or 64-bit?

Is this "network printer" on your own local area network or does the printer belong to an organization with which you're associated, e.g., your employer or your school?

Is the printer directly connected to the network router (wifi, Ethernet, or USB) or is it connected to a computer that is connected to the network?

Is there a reason that you have installed a PCL 6 driver?  Although PCL 6 is newer and "more powerful" than PCL 5, it is also prone to more problems, at least anecdotally.  PCL 6 is also called PCL XL, so you won't get a PCL XL error with a PCL 5 driver.

greg carmack - windows mvp 2010-2020 say:
hi Blurp. I'm Greg, an installation specialist and 9 year Windows MVP, here to help you.

How was Windows 10 originally Activated on the PC? Did it come preinstalled from Factory, was it Upgraded with free or paid Upgrade, or was it a retail copy and license?

For Upgrade when you first upgraded and it activated it assigned a permanent Digital License to the PC in Microsoft Activation Servers so you can reinstall any time and in any way you want as long as you reinstall the licensed version. It should reactivate soon after going online.

If you change hardware follow the steps here: https://support.microsoft.com/en-us/help/20530/.... As long as the original license that was upgraded was not factory or builder's OEM it should allow it to move. If you sign in with the MS Account that is linked to the Digital License and run the Troubleshooter it should automate this, and even if OEM make an exception if all else is correct.

Factory installs have the Product Key embedded in BIOS or UEFI firmware and should activate themselves, but you can check for the key by installing ProducKey and look for OEM-BIOS key: http://www.nirsoft.net/utils/produkey-x64.zip

Retail keys must be reinserted for reinstalls during install or in Settings. There is no need to deactivate the key on the old install first. The Activation Troubleshoot will only ask which was the old hardware as a reference. It is merely a convenience. A retail key will always be honored for reactivation on different hardware, even if you need to ccontact Microsoft Support via one of these methods: https://partner.support.services.microsoft.com/...

nathaniel mon say:

Hi,

Running some networking commands can help diagnose and troubleshoot common connections problems. Kindly try these steps to run the networking commands in a command prompt window:

  1. In the search box on the taskbar, type Command prompt, press and hold (or right-click) Command prompt, and then select Run as administrator > Yes.
  2. At the command prompt, run the following commands in the listed order, and then check to see if that fixes your connection problem:
  • Type netsh winsock reset and press Enter.
  • Type netsh int ip reset and press Enter.
  • Type ipconfig /release and press Enter.
  • Type ipconfig /renew and press Enter.
  • Type ipconfig /flushdns and press Enter.

Let us know whenever you need further assistance.
