naman r say:

Hi,

This issue occurs because of one of the following conditions:

  • The %SYSTEMROOT%\System32\Wbem\ WLDCore.dll file is missing or damaged.
  • The Path environment variable does in Windows does not point to the %SYSTEMROOT%\System32\Wbem folder.

You may try SFC scan which would fix the corrupted system files.

SFC /scannow is a super-useful command you can use in any Windows version. When the SFC (System File Checker) command is used with the /scannow switch, the tool will scan all of the important Windows files on your computer and replace them as necessary.

Missing and corrupt operating system files (like many DLL files) are arguably the biggest cause of major Windows issues. Considering that, plus the fact that SFC /scannnow is completely automatic and very easy to use, the tool should usually be one of your first troubleshooting steps.

 

a)      Click Start,

b)      Click  on All Programs

c)       Select Accessories

d)      Right-click on Command Prompt, and then click Run as    administrator.

If you are prompted for an administrator password or for a confirmation, type the password, or    click Allow

Type the following command, and then press ENTER:  sfc /scannow 

[The sfc /scannow command scans all protected system files and replaces incorrect versions with correct    Microsoft versions.]

e)      A message will appear stating that 'The system scan will begin'. Be patient because the scan may take some time

 

f)       If any files require a replace SFC will replace them. You may be asked to insert your Win7 DVD for this process    to continue

 

g)      If everything is okay you should, after the scan, see the following message "Windows resource protection did not    find any integrity violations"

 

h)      After the scan has completed, close the command prompt window, restart the computer and check.

 

Check the below mentioned link for further guidance on SFC scan –

 

How to use the System File Checker tool to troubleshoot missing or corrupted system files on Windows Vista or on Windows 7

cmdrkeene say:

Does Task Manager show that your CPU is being heavily used?  Anything more than 5% sustained will cause heat to build and the fans to turn on.

But if you aren't busy doing a lot of stuff, your Task Manager should show the CPU being used 2% or maybe even less (example below), and the system should cool off and lower the fan speed once the temperature goes down.

If your CPU is using more than 1 or 2% when you aren't actually doing stuff, then find out what program is hogging it and causing all the heat to build up.  Even if you have dozens or hundreds of programs running, your CPU should not actually be busy more than 1% of the time unless you're actually doing stuff (calculating finances, playing games, videos, music, etc.).
