steve cochran say:
Well, you have 8 different startup items and one of them is starting Winmail, so you can selectively test which is doing that.  Windows does NOT start WinMail by itself.
 
There is a setting in the registry that is incremented each time Winmail is closed, so when that reaches 100, then you get the prompt to compact the database (not compress).  If you use WMUtil, it will not clear that counter.  You have to click "OK" to let it compact itself and then it will reset the registry key to 0 until the next 100 closings occur.
 
The value is "Compact Check Count" under this key:
 
HKEY_CURRENT_USER\Software\Microsoft\Windows Mail
 
steve

sanmartin say:

Well, just to clarify some things:

  • If a "System Idle Process" is showing 80 % CPU usage at TaskManager, that means just the opposite of what you're suspecting (high CPU impact by running tasks) - it means the CPU uses 80 % of processor time for "waiting for instructions" (= idle, doing nothing = not causing any fan activity at all). This may happen when a CPU request is sent by some thread (thus initiating the "waiting" status) but the thread suddenly hangs and doesn't send any further instructions.
  • A RAM usage of 2.82 GB indicates the same - apparently the game is loaded into memory but doesn't launch (or doesn't show you a "launch screen" - "continue game? new game? load game? options?", etc.) because one or more subprocesses aren't started by the main process.
  • Lastly, "svchost.exe" is a superprocess (it manages all kinds of Windows services - from desktop display, sidebar gadgets, to power management, etc. as a "service host") which should not be touched. or you'll multiply your troubles. Normally, it only launches and schedules other threads which by their own may or may not cause high CPU impact (using ambitious Windows themes and gadgets are sure candidates for this); "Svchost.exe" by itself rarely will show big CPU power consumtion. It only will get up to 25 % CPU usage when the scenario fits into the above observations: some schedule is interrupted, some service should be launched but is hindered by something, so the whole thing falls into an endless loop which can and will be power consuming = the computer is struggling against itself like in a case of flu.

Possible causes for this behavior are "computer flu" (viruses), registry corruption (often induced by "optimization software" or "registry cleaners" - only useful for people who know what the suggested changes mean, those programs should never be allowed to change things automatically) or errors during the game installation. Recommendation: Uninstall and reinstall the game, preferredly at another location than the first time (to avoid mismatches with not fully deleted traces of the first install). Do not install any game into the default location (the "Program Files" or "Program Files (x86)" folder on your C: drive; this is a system protected space, and games don't respect system protection, thus confronting you with all kinds of hangs, crashes, etc.).
