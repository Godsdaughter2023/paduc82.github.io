As no one mentioned this earlier and I've had no problems with credential manager…  Did any of you specify the full authentication path?  If either computer involved in an authentication attempt is not a domain member, you must target the user you’re attempting to utilize.

If you’re on a domain member attempting to cache a non-domain computer, use <non-domain-computer-name>\<user name> (COMPUTERNAME\USERNAME).

If your attempting to cache domain credentials on a non-domain member, you can target the specific server or a wildcard (fs0.domain.local vs *.domain.local) with the user name DOMAIN\USER or user@domain-dns (In my example would be *** Email address is removed for privacy ***).

Having not used the Credential Manager with Web Servers yet, the following is a guess based on how IIS normally works.  It will probably depend on the sites configuration.  It may require you target the IIS Computer Name, the IIS Domain Name, or it nothing except the user name.  If IIS is fully configured, it probably won’t want more than a straight user name.

user@domain-dns only works on variants of Windows post XP/2003r2, and older applications will not understand it.

a. user say:

1. Are you meant to dictate the punctuation?
Some people will assume you should and others assume you don't - surely that will affect how well the speech is interpreted and the speech recognition programme is trained.

2. If you make a mistake, should you start again or repeat the word correctly having said it incorrectly? - Or will repeating the word be taken as your pronounciation of the next word?

Yes, dictate punctuation. Windows Speech Recognition first listens for the sound of words and tries to make a good guess at what you said. It then compares the word it thinks you may have said to the words before and the words after for context clues. Therefore, definitely use punctuation to help with this context process and speak in phrases for best accuracy. Think of the sentence, "They're going to park their car over there. Context is how speech recognition knows which there/their/they're to use.

You should correct the mistake in order for the system to learn and not make the mistake the next time. As a matter of fact you should make corrections in context by saying, select the two were three words the mistake is within.

Marty

Yes, I've trained the software. Thanks for the clarification of the question. You do not have to dictate punctuation when reading the training selections. I asked this question of the Microsoft Speech Recognition team back in 2006 and they said it does not matter if you do or don't.

As for making corrections when training, it is not important. The training is listening to how you pronounce phonemes, the 47 or so individual sounds in the English language. There are multiple instances of each phoneme in the training so you need not worry about a slip of the tongue. There is an add on toolkit for WSR that has 3 features, among many others, for fine tuning training. These features include reading a linguistic selection to improve phoneme recognition, a tool that allows adding lists of words, and a tool to read Word or text documents to add words in the context/style you dictate. The rules of this forum do not allow mention of this inexpensive/useful toolkit so you will have to use Bing or Google to find it.

When I wrote:

"You should correct the mistake in order for the system to learn and not make the mistake the next time. As a matter of fact you should make corrections in context by saying, select the two were three words the mistake is within."

I was referring to correcting mistakes while dictating. The software learns from your correcting mistakes and best when mistakes are corrected within a phrase.
