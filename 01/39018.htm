How does Multimedia Class Scheduler function?

Hello,

          I have noticed in the Services list a service called Multimedia Class Scheduler and see that it is set to run automatically. However, I am not sure whether it runs as whenever I go to check it it is listed as stopped. I would like to know if there is a way to check to see whether it has run or not.

Thanks,

            Ash 

a. user say:

Hi Ashley,

Thank you for posting your query to us.

The Multimedia Class Scheduler service (MMCSS) enables multimedia applications to ensure that their time-sensitive processing receives prioritized access to CPU resources. This service enables multimedia applications to utilize as much of the CPU as possible without denying CPU resources to lower-priority applications.


This service is required only for the fact that Windows Audio service lists this as a dependency. Without the Windows Audio service, no audio will be played on the system. This service is started upon system boot, but shortly after (~ 8 minutes), it stops. This service will start again if an audio is played.


It is a Notification that will start with all the programs that uses Windows Audio. For an example if you open Windows Media player, then the Multimedia class scheduler will also be started because it uses the windows audio.

The BAD_POOL_HEADER bug check has a value of 0x00000019. This indicates that a pool header is corrupt.

A pool header issue is a problem with Windows memory allocation. Device driver issues are probably the most common, but this can have diverse causes including bad sectors.
