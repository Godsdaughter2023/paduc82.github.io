ron sommer mvp 2009-2017 say:

I suggest that the messages be copied from one account folder to another account folder.

 

¡firedog say:
I considered that, Ron: I first wrote 'copied or moved'. However, on reflection, bearing in mind the increasing number of problems with the database that we've been seeing, I thought that the 'move' option was more likely to proceed without hitches. 'Move' only involves changing the location field in the index, while 'copy' requires a whole new set of files to be created and indexed (and virus -scanned). On balance, I came to the conclusion that 'less is better'. The old account is scheduled for removal, anyway.

If the 'move' is conducted while working offline, I think the probability of data loss resulting from untimely synchronizations is minimized. A 'belt and braces' procedure would be to copy first to a folder under storage folders, then copy from there to the new account.

We are both suggesting ways to safely move the messages.

 

'Move' only involves changing the location field in the index

This is true for WLMail, but when an account synchronization occurs, the 'moved' emails will be removed from the server.

 

Add the 'move emails' account first.

After the folders and emails have synchronized, right click on the account name in the folder pane, Properties, uncheck Include this account when receiving mail or synchronizing.

Now if something happens, the emails are still on the 'move emails' server.

 

One thing that will help the database is to close WLMail several minutes before turning off or putting the computer to sleep.
