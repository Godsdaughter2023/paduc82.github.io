Can't Completely Remove HID Non-User Input Data Filter (KB - Microsoft Community

gts-nj say:

Can't Completely Remove HID Non-User Input Data Filter (KB 911895) Driver

  A customers Dell Windows 8.1 laptop had multiple problems that I found were caused by the "HID Non-User Input Data Filter (KB 911895)" device  which is shown in Device Manger under Human Interface Devices.  (This was an optional MS update intended to fix a screen saver issue when a wireless mouse is used.)

  I have uninstalled the KB, hidden the update, and turned off recommended updates.   I also uninstalled the device in Device Manger.  However, it is reinstalled every time I remove and reinsert the mouse dongle.  I have also tried disabling the device.  That doesn't work either as an ongoing solution.  It re-enables itself on every reboot or removal/insertion of the dongle.

  I need to find some way to completely remove the device from the system.  Would appreciate any suggestions.

gts-nj say:

I resolved this some time ago, but am now updating the forum to close the issue and to possibly help others who might run into a similar problem.

Firstly, the "HID Non-User Input Data Filter (KB 911895)" was reportedly an optional Winows update to address a problem whereby a wireless mouse may sometimes prevent a screen saver activation.  

 This ‘filter’ may cause serious conflicts on some hardware.  In my customers case (on a Dell laptop) it prevented typing certain letters in the user password on Windows startup.  Once in Windows it, intermittently caused keys to issue incorrect character codes and various other problems.  All other drivers were fully up to date and the computer had no other problems.

By looking at advanced device driver properties and the registry I determined that the installation package for this driver was oem81.inf.  I completely removed that package from the driver store using pnputil (http://technet.microsoft.com/en-us/library/cc730875.aspx).   The computer then worked properly.  After a few reboots and repeatedly removing and inserting the mouse transmitter for testing the device came back (and NOT through Windows update) and the same problems!  I found that Microsoft is aggressively pushing this out to Windows 8 machines as an automatic driver update, if a wireless mouse is present, with no notice (and no information anywhere about it)!   In the end I had to completely disable automatic driver installation in Advanced System properties (as explained here http://www.microcentertech.com/tech_center/DB/read_article.php?faqid=./HowTos/HOW5006661B.htm) and again remove the package with pnputil.   (In general I consider disabling automatic driver installation to be a good policy. Unfortunately, MS defaults to allowing it and most consumer users will be unaware of the setting altogether.

No doubt there could be a finger pointing war between Dell and Microsoft and Logitech as to why this device crippled the laptop, but my customer and I are satisfied just to have the machine working reliably once again.
