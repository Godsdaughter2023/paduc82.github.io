Cause

A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high.

This bug check is usually caused by drivers that have used improper addresses.

If the first parameter has the same value as the fourth parameter, and the third parameter indicates an execute operation, this bug check was likely caused by a driver that was trying to execute code when the code itself was paged out. Possible causes for the page fault include the following:

  • The function was marked as pageable and was running at an elevated IRQL (which includes obtaining a lock).
  • The function call was made to a function in another driver, and that driver was unloaded.
  • The function was called by using a function pointer that was an invalid pointer.

The SAFE_OS phase of an upgeade to Windows 10 is where the new software is downloaded and applied to your computer.

0x80070490 refers to a group of errors, all of which mean that your existing Windows installation is corrupted, typically from unwise configuration changes, therefore the upgrade to Windows 10 cannot be completed.

You can try some of the analytical tools like SFC and DISM to detect where the corruption occurs, but it's difficult to interpret the results they provide and they may not help you anyway.

The best way to proceed in this situation is not to try to upgrade to Windows 10 but to do a clean install instead. A clean install is far superior to an upgrade anyway.

Don't think of installing Windows 10 unless you know for a fact that your computer's hardware is supported on Windows 10, or if it isn't still supported (mine isn't) then the experience of other users of your computer has shown that it's compatible nonetheless. Installing an operating system on incompatible hardware is a losing proposition.
