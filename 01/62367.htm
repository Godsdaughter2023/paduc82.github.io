When downgrading from Win8 to Win7, the installer stated my files would be moved to a Windows.old folder, thus I didn't think about checking my recycle bin just in case.

Turns out I had quite a few valuable website backups in there. So now, I go to the recycle bin and it's empty. Then, I opened the windows.old folder and looked at the hidden $recycle.bin folder. When I opened it it had a Recycle Bin shortcut. I clicked it and it opened up that other blank recycle bin.

So I right clicked on the old bin, and it showed that there's over 16,000 files and 6,000 folders (1.22 GB total) yet it shows "empty". I know all my files are still there, and to verify this I copied that recycle bin to my documents and watched as it said the name of every old file I had, including the .tar backups!

How can I access these files? Are there any programs/secrets to doing it? It's ridiculous that it won't let me see them.
frederik long say: Here you go:
  1. Log on under an admin account.
  2. Click the Start globe.
  3. Type the three letters cmd into the Search box.
  4. Press Ctrl+Shift+Enter
  5. Click "Run as Administrator".
  6. Type the following commands and press Enter after each:
    taskkill  /f  /im  explorer.exe
    (your desktop will go black)
    taskkill
    (your desktop will re-appear)
  7. Use Explorer to navigate to the old bin
  8. Start wading through your files.
Since the files have computer-assigned names, it will take some time to examine all 16,000 of them. You can ignore the small ones - they contain only folder/file names.

As an alternative you could search the web for a utility to get a GUI view of the old bin. You might also want to review your backup policy. Having to trawl through the recycle bin of an upgraded/downgraded installation seems to be pushing things a little.
