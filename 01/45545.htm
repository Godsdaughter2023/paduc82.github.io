¡firedog say:

It immediately said the backup was completed successfully.

Failure of the export function on Windows 8.1 seems to be a result of installing security update KB3126593. Because of the security implications, uninstalling the update to correct the fault can't be recommended. The fault has earlier been shown to affect Windows Live Mail 2012, but now it seems that Windows Live Mail 2011 is also affected. The mitigation steps in the KB article (add a trusted path to the Safe Load List) apparently do not help.

Unless updates are released for either KB3126593 or Windows Live Mail to correct this fault, the export mechanism will have to be regarded as broken and other methods used instead:

  • Move the store folder to one of the locations backed up by File History, e.g. into the Documents folder.
     
  • A simple 'sledgehammer' approach would be to copy the entire store folder to a backup location or include it in the set for a scheduled backup.
     
  • For a manageable number of folders, dragging messages from the Windows Live Mail message list to an appropriately-named backup folder may be more efficient, but this can't be automated.

To find and/or change the location of the store folder, in Windows Live Mail press Ctrl-Shift-O for Options when viewing the message list. On the Advanced tab, click Maintenance and then Store folder.

Because this fault is caused by changes to Windows 8.1, no amount of repairing or re-installing the program will make any difference.

Windows Live Mail 2011 has not been supported on versions of Windows later than Vista since 2012 and may stop working with no notice. Anyone still using an obsolete version should consider upgrading to Windows Live Mail 2012.

rajesh govind say:

 

Hi Shahid,

 

When using a USB to parallel connector cable for an older printer, the computer must have new drivers installed. Insert the CD that came with the USB to parallel printer cable to install the needed drivers on the system. When a USB to parallel cable is connected, the computer will use the USB port to send information to the printer. Here, if you have already installed the printer driver software, then when you turn on the Printer, the computer should recognize the printer and will automatically configure the printer port. If this does not happen, then you try to uninstall the existing printer drivers and then add the printer again to the computer using the "Add a Printer" option from the "Devices and Printers" option in the “Control Panel”. This will install the generic Windows driver for the printer. To do this, you may follow these steps.

 

Uninstall the existing printer drivers from the computer.

 

a)       Press “Windows Logo” + “X” keys on the keyboard.

b)       Click on “Device Manager” from that list.

c)       Search for the printer device from the device list, right click on it and then select “Uninstall”. On the un-installation window, if you have an option: “Delete the driver software for this device” you may select that and then remove the corrupted drivers from the computer.

d)       Follow the on-screen instructions to complete it and the restart the computer if prompted.

 

After the restart, you may refer to the below given article to install the printer:

 

Install a printer

 

Hope this is helpful.
