FAT32 maximum number of files in a folder
dax1792 say:

The maximum size for a FAT32 directory is 65536 32bit entries. A file will require at least 2 entries depending on the length of the filename. Long filenames are stored in 16bit UNICODE characters and only thirteen characters are stored per directory entry.

 

Large directories should be avoided for performance reasons. For example, adding a file requires all existing entries to be scanned to avoid duplicates.

 

The FAT32 specification can be downloaded here:

http://msdn.microsoft.com/en-us/windows/hardware/gg463080.aspx

David MacLaren,

You are responding to a thread that is over 2 years old.  You should start your own question for best results.

On the other hand, please read the links provided by Nass a bit more carefully.

Serscan.sys should be in C:\Windows\system32\drivers if it is needed. 

A backup copy should be in C:\Windows\system32\dllcache (a normally-hidden folder).

A compressed copy should be in C:\Windows\Driver Cache\i386\driver.cab as explained in the HP article linked by Nass.  If it's there, use the expand command in a Command Prompt window (or WinZip, as the HP article suggests) to get it out.  That reference is to the C:\Windows directory on your hard drive, not to the Windows CD. 

You may also have a C:\i386 folder on your hard drive.  In that case, you'll find a a copy in C:\i386\driver.cab.

Finally, although you don't seem to have the issue addressed by the KB article linked by Nass, you;ll notice that the hotfix appears to include two copies of serscan.sys (one for sp2 and one for sp3. If you request the hotfix, you should be able to extract the correct copy for your system.

I have no need of serscan.sys, so there is no copy in my C:\Windows\system 32\drivers directory.  The copy in dllcache looks like the original version, pre-service pack 1 (5.1.2600.0). I suspect the compressed copies are as well.  I imaging that Windows Update didn't bother giving me updated versions of this file over the years because I don't use it.

If you decide to use one of these old versions, I suggest that you go to Windows Update shortly after you install the printer and make sure that you get the most recent version (if Windows Update decides that it's needed).

davem121 say:
Hi Michael, you should proceed as follows:

Remove the old Hard Drive
Plug the SSD into the SATA port the old drive was on
Go into your BIOS and make sure SATA mode is set to AHCI
Boot from the installation USB and install Windows 10 onto the SSD

When the installation completes and you arrive at the Desktop, shut your system down
Connect the old HDD to another SATA port

Start your system again

Ideally you should copy off the files you need to keep from the old HDD, format it and copy the files back onto it

If you cannot do that,

Run Disk Cleanup on the old hard drive
let it scan and delete everything - it should see the old Windows OEM installation and remove it
