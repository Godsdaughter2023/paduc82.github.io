a. user say:

The System Idle Process is the amount of "free" time the CPU has so the closer it is to 100% the better things are for your system.

If you look in Task Manager the higher the "Percentage of time the processor is idle" the better (it is a good thing):

It is also normal to see several svchost.exe Processes running at the same time as underneath each of them are important Windows Services.  You can see 11 in that screenshot but that is not a problem since I know that each one is hosting some important Services underneath and the amount of CPU they use is almost always going to be zero zero.

It is abnormal to see one or more svchost.exe Processes that is jumping to or constantly consuming a large amount of CPU time.  If you see that then there might be a problem to be solved.  In spite of what you might read on the Internet about such abnormal behavior from a svchost.exe process it is wrong to think that the svchost.exe process has a "memory leak" - what it really means is that something else might be wrong.  There is no memory leak in the svchost.exe Process.

With Task Manger open to the Processes tab and the "Show processes from all users" checked click the CPU column header to sort highest to lowest and watch it for a while to see if you observe any process that jumps to the top of the list.

Post additional screen shots if you observe somethings other than System Idle that might he hogging your CPU resources.
