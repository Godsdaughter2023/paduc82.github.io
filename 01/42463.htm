lemp say:

Often, Windows error messages appear in a small window with the name of the underlying program shown in the title bar of the window.  What is shown for these errors?  Does the second error appear to be from the same source as the first?

I've seen some reports that suggest that the first error, at least, is caused by NUSB3MON.EXE which is the executable file for Renesas Electronics USB monitor that checks to see if any devices are connected to your computer's USB 3.0 ports.  This seems like an unnecessary function, so you might take a look in Programs and Features and if you see a Renesas program there, uninstall it.

Alternatively, download Autoruns:  https://docs.microsoft.com/en-us/sysinternals/downloads/autoruns

After Autoruns completes its scan, select the "Logon" tab and see if there is an entry (or perhaps two identical entries) for either a Renesas or an NEC USB monitor.  If there are two identical entries, delete one of them. 

If you want to be slightly less affirmative, simply uncheck the box at the left end of any item that you want not to start when Windows starts.  If you change your mind about disabling a start-up item, just run Autoruns again and check the box for the item.
