(1)Netbook having Intel GMA 3150 Graphics Card, not supporting the game "Rise of Nations" in Windows 7.

(2)Notebook Wide Screen Issue for "Rise of nations".

 

 

===============================================================

 

 

Reason:-

======

1) Graphics card issue. [of course it is].

2) Display resolution Issue. [Of Course. As the Minimum Resolution for playing "Rise of nation" is 1024X768].

 

 

===============================================================

 

 

Solution:-

=======

[Solution to Issue 1]:- (1)Netbook having Intel GMA 3150 Graphics Card, not supporting the game "Rise of Nations" in Windows 7.


Well, as the game "Rise of nations" having 1024X768 Minimum resolution. So, the Windows 7 should be supporting a resolution Physically or Virtually equal or more than 1024X768. So, what we will do is: we will change the registry setting in Windows 7 to avail the higher resolutions like: [1024X768] and [1152X864]. But, Virtually.

 

 

Way to do: Go to Start: Run: [Regedit]: HKEY_LOCAL_MACHINE\SYSTEM

 

Now, Search for the "Value Name": [Display1_DownScalingSupported]

 

And Once found, Change the value Data to "1". By default, it will be "0".

 

Now, apply the new value. Restart your machine. And check the "Reslution". You can find two new Resolutions:

[1024X768] and [1152X864]

 

Set to [1024X768] and start "Rise of Nations". It will run smoothly. If not, then feel free to mail me at "*** Email address is removed for privacy ***" or contact me via Facebook [Facebook Name: "Biswa Dip" or, using "*** Email address is removed for privacy ***"]

 

For me the registry location was:

HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Control\Class\{4D36E968-E325-11CE-BFC1-08002BE10318}\0000

 

For you it will be different (Registry Entry Location).

 

 

===============================================================

 

 

[Solution to Issue 2]:- (2)Notebook Wide Screen Issue for "Rise of nations".


Wide Screen issue happens because the "Rise of nations" does not support your current resolution properly. So do as below:

 

After you register a Player name when first time playing "Rise of nations", it creates user setting folder inside:-

"C:\Users\your_username\AppData\Roaming\Microsoft Games\"

 

 

Edit this file:-

"C:\Users\your_username\AppData\Roaming\Microsoft Games\Rise of Nations\rise2.ini"

Or,

find the "rise2.ini" inside the "Rise of Nations" Installation Directory. Change wherever you find it or in the both locations if available.

 

Like Below:-

===========================

Windowed Width=1024

Windowed Height=764

AllowAllResolutions=1

===========================

 

Change the above Settings, change AllowAllResolutions=1 [by deafult it's "0"] and change "Windowed Width/Height" as per your need.

ethan b. say:
Hi emjov,

It seems you've hit a random occurrence where an improper shut down (or restart in this case) prompted a disk consistency check (called chkdsk) to be scheduled at the next boot up. It will not delete your data, but it will perform a disk consistency check to find errors and even fix them when Windows is not running (or is about to be booted). If you restart, a disk consistency check will be run in order to find said errors, and fix them whenever necessary.

In layman terms, because you shut down the computer (presumably by holding down the power button until it turns off, and then back on?) abruptly, the computer then schedules a disk check session to see if anything was wrong with your hard drive or solid-state drive.

wunders say:

Making some assumptions here:

  • You had an XP Pro machine that you use Windows Encrypting File System (EFS) to encrypt some data
  • For some reason you had to re-install Windows and you restored the encrypted files from a backup.

If this is the case, the key to decrypt these files was managed by Windows as a Certificate in your Certificate store of your original installation. These certificates are, in turn, encrypted to your Secure ID, your password, and who knows what else.  This information would be lost on a new XP install.  The easy way to restore access to your files is to restore your certificate from the backup that you dutifully performed according to the paragraph "Why you must back up your certificates" in the article:

"Best practices for the Encrypting File System"
  <http://support.microsoft.com/kb/223316/en-us >

If you don't have a copy of your certificate, then you must restore your old system from an image-style backup, login, and then extract your certificate.  Otherwise, your data is history.  EFS is quite secure and there are no back doors.  Because it is not made clear the importance of backing up the certificate, EFS has affectionately been referred to as the "Delayed Trash Can".

HTH,
  JW
