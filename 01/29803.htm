My confusion is that in the past I've never seen user/group names resembling this.  Has Microsoft come up with a new naming convention?

-r

Jon

hairyfool say:

The naming convention adopted by Microsoft of ComputerName\UserName (or DomainName\UserName) has been in place since the advent of Windows. The full naming convention was only required when eliminating ambiguity. In a workgroup there could be two users with the same name but not on the same computer

Users with Admin level privileges appear in two different Groups (Users and Administrators) The 1st example the user jb is getting the rights because he is a member of the Administrators Group and in the second because he is logging on as a User.

And yes it is essentially garbage because it is the result of a random number generator on the basis that the likelihood of two computers on the same network getting the same name is virtually nil. The name is generated before network connectivity is on so cannot be verified as exclusive.
