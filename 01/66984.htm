Could be a corrupted update.  You have set up Windows Updates to disallow the graphics driver

KB 2973337, or the six most recent 'Security' updates, and have not had

problems with those games that use UE3. 

However, if the problem is only with the one game (or UE3) there may be a conflict between

other software that have been updated, and the game.

eg - "If I don't have chrome running it seems to not happen "

My main experience with persistent black screen crashes has pretty much always been due to a mod, or conflicts with other software. Browser addons are one possibility.

Multi-tasking is always going to introduce more chance for conflict, especially when there is

a large on line component.

- Had you made any 'tweaks' to settings in BIOS or Windows around the time the crashes started ?

- Audio drivers can also conflict and lead to black screen with, or without audio.
If you are not using HDMI for audio don't install that part (or any part you don't use)

of the Nvidia driver package. There may also be a conflict between your on-board audio and the ASUS card.

Speaking of ASUS. ASUS Smart Doctor has been known to cause a few problems including Nvidia

driver issues, but I don't recall if  they  led to Black Screen crashes, and have not heard much about

SD problems this year..

- People have problems with Flash Player versions from time to time as well.

- Have you run from - Clean Boot - to troubleshoot for conflicts (it sometimes helps ;] ) ?

- Which driver version did you settle on ? Is this still a 'clean' install from after re-installing

Windows ? Whatever drivers you were using before the problem started may be the best

to use, at least until the issue is resolved.
