aj don say:

Hello,

We appreciate your insight regarding the auto eject function of the external media, if the detach button is pressed. To shed light on this matter, the function to safely remove the media prior to ejecting it physically from the port, is an additional feature to avoid data loss. Data loss can be caused by abruptly interrupting the communication between an external drive and the main machine. If there is no active communication between the two devices, such as a data transfer, then safe removal is not necessary. However, there are instances of passive communication, a good example would be accessing an executable file or playing a media file that is not saved in the Surface Book 2. This particular task, requires the Windows machine to create a cache of the said data into its local memory to seamlessly perform the desired operation and if by chance, the continuity between both devices was unexpectedly broken, it could cause file corruption. Thus, we recommend the safety removal of the external media.

We really appreciate your insight. Hence, we encourage you to utilize the Feedback Hub. Your suggestion will greatly improve the experience of Surface Book users such as yourself.
