TIA

 

 

I have resolved this problem, it's a defect / bug with Teamviewer at least through version 8.0.18051.  This is strictly a Teamviewer problem and Remote Desktop (RDP) correctly handles character sets.

 

Problem:

- I was maintaining the machine via Teamviewer

- Teamviewer is sending the my local keyboard character set and ignoring the keyboard set on the remote

 

Resolution

- you need to have the subject language keyboard installed on the local machine, and switch to that keyboard using the language toolbar in order to use the correct character set on the remote

- also, the keyboard language selection on the remote is irrelevant / is ignored by Teamviewer

 

Notes:

- you do NOT need to have win7 Ultimate or Enterprise in order to use a foreign language keyboard

- you do NOT need to set the ‘Region and Language’ snap in to the foreign language / country on the Format, Location, or Admin / Unicode localization tabs

- you do NOT need to download or install a Language Pack, MUI or LIP

- If you are using a right to left language like Hebrew, you would need to set up the right justification in the subject app

 

Language Options in Windows 7:

I’ll outline these as they are vague to the end user as they are currently presented here

 

Keyboard only

To add a foreign language keyboard in order to be able to write foreign character sets / respond to foreign language emails / use foreign languages in apps: (note, this has changed somewhat in win8 but is similar)

-          Start “Region and Language” from control panel or by clicking the start button then typing “region” and selecting it from the search

-          On the ‘Keyboards and Languages’ tab, click ‘Change Keyboards’

-          Click ‘add’ then select your language from the list, then drill down to the keyboards section and check the appropriate language

-          Click OK 3 times to get out and the language bar will be in the lower right Taskbar, click the language name to select different languages

 

Partial Windows UI

Some languages have partial language packs to render the windows UI in the subject language, these are called LIPs and are downloadable for all versions of windows 7 here:

http://windows.microsoft.com/en-us/windows/language-packs#lptabs=win7

 

Full Windows UI

If your language requires Ultimate or Enterprise, you need to either

A-     download the full Multilanguage User Interface (MUI) from windows update and must have Ultimate or Enterprise or use Anytime Upgrade to upgrade to them

B-      buy a localized version of windows from the subject geography

From Jose KB#2417 (not enough Boot Options):

If there is a problem booting, XP is configured to automatically try to boot again and you can get stuck in a loop of just being unable to get past the boot options screen or none of the boot options you choose will work.

Sometimes when XP has a problem starting or crashes and tries to start again, it will give you a "short" menu of boot options and none of them will seem to be the right ones to get your system going again.  You've tried them all!

It may look something like this:

safe mode
safe mode with command prompt
safe mode with networking
last known good configuration
start normally

The options resemble the XP Advanced Boot Options menu, but the one option you need (Disable automatic restart on system failure) is not offered because XP has gone too far along in the boot process and offers you a limited number of boot options.

If that is the case, you must invoke the Advanced Boot Options menu yourself until you do see the option:

Disable automatic restart on system failure

When you do get to the correct XP Advanced Options Boot menu you want to see, it has options on it like these:



Safe Mode
Safe Mode with Networking
Safe Mode with Command Prompt

Enable Boot Logging
Enable VGA mode
Last Known Good Configuration (your most recent settings that worked)
Directory Services Restore Mode (Windows domain controllers only)
Debugging Mode
Disable automatic restart on system failure

Start Windows Normally
Reboot
Return to OS Choices Menu



What you need to choose from that menu is the option:

Disable automatic restart on system failure

Then if XP fails to boot normally, you will see an error screen with information and clues regarding the problem and then you can decide what to do next.

If you do not see the Disable automatic restart on system failure option, you need to reset your system and start tapping the F8 key on the keyboard until you do see the Disable automatic restart on system failure option. 

If you miss the F8 window of opportunity, you need to try again and start tapping the F8 key with more urgency (sooner and more frequently) until you do see Disable automatic restart on system failure, then select it.

You need to keep trying the F8 menu until you do see Disable automatic restart on system failure option, and select it.

a. user say:

Hello,

In Windows 8, Microsoft has redesigned chkdsk utility – the tool for detecting and fixing disk corruption. In Windows 8, Microsoft introduced a file system called ReFS, which does not require an offline chkdsk to repair corruptions – as it follows a different model for resiliency and hence does not need to run the traditional chkdsk utility.

The disk is periodically checked for file system errors, bad sectors, lost clusters, etc, during Action Center or under the Drive properties in File Explorer. If potential errors are found, you will be informed about it. You can continue to use the computer, while the scan is carried out in the background. If errors are found, you may be prompted via a notification to restart your computer.

If you wish to nevertheless manually run a scan, you can do so. Earlier you had to schedule Disk Error Checking for the system drive and for drives which had files or processes or folders opened. In Windows 8, error checking  starts right away, even on the system drive – and it longer needs to be scheduled at start-up. Only if some errors are found, will you have to restart to let Windows 8 fix the errors.

To begin the scan, right-click on the Drive which you wish to check and select Properties. Next, click on Tools tab and under Error checking, click on the Check button.  This option will check the drive for file system errors.

If the system detects that there are errors, you will be asked to check the disk. If no errors are found, you will see a message – You don’t need to scan this drive. You can, nevertheless, choose to check the drive. Click on Scan drive to do so.

The scanning starts. I found that the process ran quite fast and the scanning was over in less than 5 minutes.

On completion, Windows 8 will display a message. If no errors are found it will say so.

If errors are found, you will see the following message:

Restart your computer to repair file system. You can restart right away or schedule the error fixing on next restart.

When I clicked on Show Details, the Event Viewer sprang to life, showing me the relevant log.

In Windows 8, Microsoft has made the Disk Error detection and correction of file system errors less intrusive, so that users can carry on working at their computers without worrying about such error.

a. user say:

Even when you power off your computer, there is still a tiny amount of electricity in the system. People sometimes call it 'flea power.' It was designed so network administrators can wake your computer to perform maintenance, and Microsoft uses it to wake your computer in order to restart it after installing an update.

After the update is installed your computer will remain awake. Some time later it will either Sleep or Hibernate, depending on your settings in Control Panel > Power Options.

If instead you want to shut down your computer after the update installs, you can do it but it's a little tricky. You have to guess how long after the restart your computer needs to stay awake in order to finish installing the update. Then you have to schedule a Shutdown command with Task Scheduler.

It's easier just to let your computer Sleep or Hibernate.
