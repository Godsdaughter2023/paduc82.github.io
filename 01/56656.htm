For the past 5 years Microsoft made it quite easy for users of the Windows 7 operating system to obtain reinstall media. If you lost your retail installation disc or recovery media; either it was damaged or faulty; you could download a copy of the edition you have a license for from an affiliate website named Digital River. Most recently, Microsoft ended availability of reinstall media which you could download as a .ISO file from Digital River, which is a digital replica of a optical disc.

Recently I wanted to change boot disks.  I got two huge capacity drives.  The BIOS would not boot and hung when it tried to compute the size of the new high capacity disks.  I had to upgrade the BIOS so that it would not hang.  This was complicated due to the fact that the mainboard manufacturer's latest BIOS still didn't make it work--I had to integrate the SATA BIOS from the SATA manufacturer into the mainboard's BIOS and remove the older module for the SATA controller.  It worked, and I kept on experimenting.

I found that I could backup my entire 80 Gbyte EIDE WinXP system boot drive using NTbackup.exe that can be found either as an add-on on the install CD for WinXP home, or it is installed by WinXP Pro to be within the Accessories.

I deleted all of my browser cache so that those crappy files would not take up time and space in the backup, then I removed the system booting hard disk and placed it in a second computer system so that none of the files to be backed up would be busy / in-use / unavailable for backup, then used NTbackup.exe to save the entire disk of everything to one of the newly formatted large capacity drives into one huge 70+ Gbyte backup file.  Surprisingly this took a short amount of time to do.

I formatted the other blank new disk, then marked the partition as active (system bootable).  Once that was done, I restored all of the backup files to that empty disk.  When I placed that restored version new disk as the only disk to boot, the system booted up fine on a SATA rather than EIDE controller--Once WinXP tried to log me in, it put up a message saying that "Your system appears to be substantially different than your original install configuration--you will need to reactivate Windows"

I unplugged the network cable, then played around a bit to see if things were broken, but they seemed to work fine.  The backup / restore test worked.  Now I could test other things.  I downloaded Ubuntu Linux Server 9.10 boot DVD, burned that, and reformatted that SATA drive with it, replacing the restored WinXP system (that wanted me to Activate it through Microsoft).

So it looks like as long as I perform a complete backup like this of the system boot hard drive, I can screw around with all of the settings without fear of messing things up because a restore will wipe things back to when they might have been working better (so I can fearlessly learn life's lessons of hard knocks when the new configurations turn out really badly).  Right now, I've got about 3 or 4 backup files of about 70 Gbytes in size sitting on one of the new drives in case I screw up somehow.

This means for your situation that you will probably have to re-Activate your copy of Windows should it appear to Microsoft's software that the system configuration's "signature" has significantly changed.

  11. Close all tabs and go to Administrative tool.
  12. Open component services.
  13. Click Computer, click my computer, then click DCOM.
  14. Look for the corresponding service that appears on the error viewer.
  15. Right click on it then click properties.
  16. Click security tab then click Add User. Add SYSTEM then apply.
  17. Tick the Activate local box.

The VIDEO_TDR_ ERROR bug check has a value of 0x00000116. This indicates that an attempt
