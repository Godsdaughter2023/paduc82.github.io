Hi Morrie,

It's always good to check for any Error messages, either at the FSX screen or at Event Viewer (Admin events) to find out more about the error you receive.   This will help identify the type of problem you're facing.

However, since FSX first "Stopped Responding", you have tried a Repair and several Uninstalls/Reinstalls and finally got the .dll error.   You're correct, that that .dll is found in FSX's directory because it's part of the FSX file composition, but since you're still getting the error, it could be that your System Registry may have some old FSX entries that are preventing FSX from installing correctly.

Suggest that you once again uninstall FSX completely.  That is, not only via the Uninstall option in the Disk or from Uninstall Program in Control Panel, but also do a search of Windows to find remnants that are usually scattered in various folders and Registry (type FSX or Flight Simulator X in your Drive's Search box).  All these need to be removed so that FSX can do a clean install.  If you use a Registry cleaner (e.g. Advanced System Care:  free), then that usually finds a number of redundant registry entries for removal.   But the warning that goes out is that you need to use these types of cleaners judiciously to prevent further problems down the line.  Don't use it if you're unsure.

Once FSX has been completely removed, reboot your computer.   Then go ahead and install FSX.  It's suggested that FSX be installed to the root of your drive (e.g. C:\FSX, rather than to the default Program Files (xxx)).   You may also need to temporarily turn off any antivirus program you have during the install.

Hopefully, the above will help you.   There is also a Microsoft program that fixes Registry issues, though I've never used it, so can't vouch for it.   It is found here:  

https://support.microsoft.com/en-gb/help/17588/fix-problems-that-block-programs-from-being-installed-or-removed

Hi AveryDavis,

Thank you for posting to Microsoft Community.

Is this an external hard drive or an internal hard drive into which the Windows 10 is installed in?

I can understand how important the hard drive is for you. If the hard drive was encrypted with a BitLocker, it can only be opened with the BitLocker key. This is because the hard drive will be encrypted with respect to the BitLocker and cannot be accessed on a different system without the key.

If you have not set the BitLocker key, the' Secure Boot' settings may be toggled to 'Disabled', which may be causing this issue. You can check that from the steps below.

  1. Power the device 'Off' and then back 'On'.
  2. You should be able to find a message on the boot screen just before the manufacturer logo appears. You may need to press the ‘F1’, ‘F2’, or ‘Delete’ button, this is usually indicated on the boot screen.
  3. Enter the BIOS settings, look for a tab called 'BOOT' and select that page.
  4. Depending on the BIOS manufacturer, this page could be 'BOOT',  'ADVANCED', 'STARTUP', etc.
  5. Look for a setting for the 'UEFI Secure Boot'. Each BIOS could have a different name for this. These are the keywords to look for: 'UEFI', 'Secure Boot', 'Legacy Boot'.
  6. Toggle the 'Secure Boot' setting to 'Enabled' and attempt to boot the machine.

However, if the BitLocker is set and the password is not known, you can refer this article on ‘Find my BitLocker recovery key’. If you do not have the BitLocker password or the recovery key, the only option would be to proceed with formatting the hard drive to access it and if Windows was installed on this drive, a reinstallation of Windows.

Note: Formatting the hard drive will delete all its contents.

Hope it helps.

frederik long say:
I have lot of data in C:\ that is not backed up.

Unfortunately it appears that you belong to the large class of PC users who choose not to back up important data until disaster strikes. Your data is not lost but recovery is a lot more painful than using an external backup disk. Here are a couple of options:

  • If this is a desktop PC: Connect the hard disk to some other desktop PC, then retrieve your data.
  • If this is a laptop: Remove the hard disk then place it into a 2.5" USB disk case. You can now connect this case to some other PC and retrieve your data.
  • Boot the machine with an Ubuntu boot CD and use its GUI to save your files to an external storage device.

When finished, get yourself a 2.5" USB hard disk and use it for your regular (e.g. weekly) backups. You might also store an image of your new installation on it, to save yourself the trouble of rebuilding Windows in case you suffer another accident.
