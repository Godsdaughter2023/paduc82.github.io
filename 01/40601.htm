Just want to say I was able to get this fixed. Here is what I found:

There was a similar issue with Windows 7 after an update. The recommended fix was to go into the registry.

http://answers.microsoft.com/en-us/windows/forum/all/windows-7-default-font-changed-unreadable/b8f8181b-711f-4c3d-9b69-21ce79529b7e

Since I found what the font was that it was using I just went into the registry and found the font that was causing the problem and removed the key. I didn't do the recommended steps just deleted the key. This worked for me. 

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\fonts

Works like a charm now. Hope this helps some if anyone else comes across this issue.

    I have .NET framework 4.5 installed, but windows keeps asking me to - Microsoft Community

    mongusdamango say:

    I have .NET framework 4.5 installed, but windows keeps asking me to install .NET framework 3.5 and below?

    When i try to install .NET framework 3.5, I get the error code  0x800F0906 0x800F0906

    My internet is working fine. And the fixes specific to that error code have not been useful for me.

    My question is do I even need .NET 3.5 if I already have 4.5 and if I don't need it is there any way to disable windows asking me for it every time I try to install certain programs.
    rajesh govind say:

     

    Hi Mongusda,

     

    The .NET Framework 4.5 in Windows 8 does not include the 3.5 version and is not automatically installed with Windows 8.  Thus, when you try to install an app that requires the .NET Framework 3.5 on Windows 8 or later operating system, it gives you error message. Here, the app's setup program might install the .NET Framework 3.5 on your computer as a part of the installation process (that is, by installing the .NET Framework 3.5 on demand) or prompts you to install/enable it manually from Control Panel. Both options require an Internet connection.

     

    In general, you should not uninstall any versions of the .NET Framework from your computer, because an app you use may depend on a specific version and may break if that version is removed. Multiple versions of the .NET Framework can be loaded on a single computer at the same time. This means that you do not have to uninstall previous versions in order to install a later version. Please refer to this MSDN article for more information: Installing the .NET Framework 3.5 on Windows 8 or 8.1.

     

    As long as you do not install/use any app that requires the .NET Framework 3.5, you do not need this Windows component on your computer. The error code 0x800F0906, that you get while installing this Windows component occurs because the computer is unable to download the required files from Windows Update. This might be due to some issues with your Internet Connection or some settings/features that might be blocking the connection.

     

    To check if your Internet Connection is working properly, you can try connecting to the Microsoft Update website. If you are able to connect to the Microsoft Update website without any issues, then there could be something else on the computer that is blocking the Internet Connection from downloading the .NET Framework 3.5 component.

     

    Hope this is helpful.
