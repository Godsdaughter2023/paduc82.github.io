a. user say:

"I am becoming concerned that as Window 10 continues to be updated and features added to the operating system, that these obligatory future updates will eventually mean that computers which run fine now, become incompatible with the OS installed on them - rendering them obsolete ..."

That is absolutely correct, and it's true of all operating systems: past, present and future.

As an operating system is updated, hardware manufacturers and software developers have a decision to make: Are they going to update their drivers and applications to keep up with the changes in the operating system? That's what we mean when we say that a hardware manufacturer or software developer supports an operating system.

Supporting an operating system ain't easy or cheap for a hardware manufacturer or software developer. It means they have to test their products on the updated OS to see if the product still works properly, and if it won't work properly they have to update their driver (for hardware) or their application (for software.) Of course, they have to test the updated driver or software, which may need to be re-coded and re-tested several times before it works properly.

Then the documentation for the driver or software has to be updated, to take into account the changes made, and technical support staff have to be taught how to work with the revised product.

For this reason, hardware manufacturers and software developers will only support a given version of their products for so long, after which they drop support. This usually happens when the revenue stream for that product starts to dry up. (You knew there was a money angle, right?)

As far as Microsoft is concerned, they would be really happy if every hardware device and software application in existence worked perfectly on Windows 10. That's why they publish technical details of updates in advance, so that hardware manufacturers and software developers are able to update their products. Regardless, manufacturers and developers will abandon a product when the money train pulls out, no matter what Microsoft does.

This is the way it's always been. The only difference between now and years ago is that now the pace of change in technology is much faster than it was years ago. If Microsoft (and Apple, and Google, etc.) don't keep up, they'll be dinosaurs.

david.chase say:

Click on start and type regedit in the search box. In Registry Editor navigate to HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Policies \ Explorer then right-click and create a new DWORD value.

Name it ConfirmFileDelete and give it a value of “1”. You’ll need to log off and back on again for the setting to go into effect.

Note: Please follow the steps precisely.

pratiksha devadiga say:

Hi,

As per your concern to delete the scheduled task, try the below steps and check if it helps.

1. Open the task scheduler by pressing Windows + R keys, and type taskschd.msc

2. Click on the task scheduler(Local) in the left pane.

3. Now under the Active tasks, select the task you have created.

4. Double click on the task.

5. It will navigate you to the created task. Now, right click on the task, and click on delete option, and confirm it.
