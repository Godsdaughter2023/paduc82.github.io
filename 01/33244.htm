¡firedog say:

"Do not allow attachment to be saved or opened that could potentially be a virus".  

The setting on the Security tab of Windows Live Mail's Do not allow attachments that could potentially be a virus to be saved or opened dates from the early years of Outlook Express when malware delivery methods were just as primitive as the measures taken to avoid them. Now that both virus protection and the bad guys have moved on in the intervening 15 years or so, the setting should be regarded as an anachronism.

If you're careful about not opening attachments you're not expecting, your real-time anti-virus will protect you from any of the dangers that this setting would guard against. In other words, this option need not be selected on any OS later than Windows XP SP1. It's wise to run Windows Live Mail in the Restricted security zone, but running it in the Internet zone will not normally be problematical for careful users.

That said, I seem not to be able to change that setting. It might require a reboot, which I can't do at the moment. However, if it's active, you shouldn't be able to save the attachment first and then open it.

I'm fairly certain that this is an Office problem. If you try to open an attachment in Windows Live Mail, the attachment will be written as a file to the Temporary Internet Files folder, which is inherently suspect as far as Office is concerned. This should mean, though, that the file will open in Protected View. If it's not doing that, then Office is to blame ...
