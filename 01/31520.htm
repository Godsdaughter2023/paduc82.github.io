kenneth gar say:

Hi Joshua,

If your Surface device won't charge and the battery icon doesn't show a power plug on the taskbar, even when the power supply is connected, and the power connector light is on. This means that your Surface does not detect the PSU and is also not charging the battery. Try to change the power connector orientation:

1. Remove the power connector from your Surface, turn it over, and plug it back in. Make sure the connection is secure and the power connector light is on.
2. Wait 10 minutes, and check to see if your Surface is charging.

If changing the power connector orientation doesn't resolve the issue, try to remove the Surface battery driver:

1. Plug in your Surface.
2. Select the search box in the taskbar, enter device manager, and then select Device Manager from the list of results.
3. Select the arrow next to the Batteries category.
4. Double-tap or double-click Microsoft Surface ACPI-Compliant Control Method Battery, select the Driver tab, and select Uninstall > OK.
(Or you can right-click Microsoft Surface ACPI-Compliant Control Method Battery and select Uninstall.)
5. Select the computer at the top of all the categories, select the Action menu, and then select Scan for hardware changes.
6. Leave your Surface plugged in.

After you remove the battery driver, install Surface and Windows updates via Start button, and then go to Settings > Update & security > Windows Update, and select Check for updates. If Windows Update says your device is up to date, you have all the updates that are currently available.  

If removing the battery driver and installing the update don't fix the issue, we recommend that you perform a two-button shutdown reset: 

1. Press and hold the power button until the screen turns off (about 10 seconds), then release the power button.

2. Press and release the power button to turn your Surface back on. You should see the Microsoft logo.

If you have additional questions, or if the issue persists after the troubleshooting steps above, please let me know by replying to this thread. This way, I'll get notified and respond to you as soon as I can.

Regards.

Hi,

 

The Error Code: 0xC1900101-0x 20017 is caused by a driver bug check while booting the computer during the Windows 8.1 installation process.

 

This usually happens due to the out dated drivers on the computer at the time of the update installation. I request you to check if all the updates available via Windows Update are installed on the computer. I also request you to update all the device drivers on the computer to its latest version and check if that helps. It would be great if you can provide more information regarding this issue:

 

1)       What is the make and model of the computer?

2)       Do you have any external devices (except the mouse and the keyboard) connected to the computer?

 

You may try the following methods and check if that helps.

 

Method 1: Disconnect all the unnecessary external devices connected to the computer (except the mouse and the keyboard)

 

Please disconnect all the unnecessary external devices connected to the computer (except the mouse and the keyboard) and then check if you still face this issue while installing Windows 8.1. This is to check if the issue is caused by any external devices connected to the computer.

 

You may also try installing the driver updates for all the devices installed on the computer and check if that resolves the issue. You may try the next methods and let’s know the result.

 

Method 2: Update the Device Drivers on the Computer.

 

Step 1: Check the Windows updates for any latest driver updates.

 

Check in Windows updates if any driver updates are available. If so, install the same and check if it helps.

 

a)       Press “Windows Logo” + “W” keys from the keyboard.

b)       Type “Updates” in the search bar and click on “Install optional updates” from the left pane.

c)       From the “Windows Update” window, click on “Check for Updates” on the top left corner, to receive a fresh list of updates and then install all the updates listed there.

 

For any issues while installing Windows Updates, you may also refer to this link:

 

Troubleshoot problems with installing updates

 

If no driver updates are available via Windows Update, then you may install them from the Computer Manufacturer’s website.

 

Step 2: Update Driver Software from the Manufacturer’s website:

 

Go to the computer manufacturer’s website and search for the latest Windows 8 drivers available for all the devices on the computer and then install them by following the instructions given in the website.

 

If all the device drivers installed on the computer are of the latest version, then check if you have any third party desktop program installed on the computer. It has been found that certain third party desktop programs (Eg: certain programs from IOBit or Intel Desktop Utilities including Extreme Tuning software etc.) conflicts with the Windows 8.1 installation causing this errors. Thus, you may uninstall all the third party desktop programs/utilities from the computer and then try installing Windows 8.1 again. To uninstall the third party desktop programs, you may follow the next method.

 

Method 3: Uninstall the Third Party Desktop Programs

 

You may uninstall all the third party desktop programs from the “Programs and Features” and then try installing Windows 8.1 update. Please follow these steps:

 

a)       Press “Windows Logo” + “X” keys on the keyboard and select “Programs and Features” from that menu.

b)       Search for the third party desktop program on the computer, click on it and select “Uninstall”.

c)       Follow the on-screen instructions and restart the computer if prompted.

 

After installing Windows 8.1 update, if required, you may reinstall these programs.
