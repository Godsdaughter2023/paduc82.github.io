rohn007 say:
This is a long stretch for me. But I think that if you look into it you will see that you also have to make changes for the <CTL> combinations.

For example, say the "y" key is numbered "25" in the system. When you press the key at position 25 it originally converted that to "y". But now you have told it to change it to "z". Fine, but I think you'll find that the combination of <CTL> and Key25 has it's own mapping, of <CTL><Y>. So, you (probably) have to change that combination also.

This page tells how to customize keyboards using the tool:
http://littletinyfish.com/create-a-new-keyboard...

One of the things it says on that page is:
<snip>
The Keyboard Layout Creator lets you map keys to Ctrl, Ctrl+Shift, Ctrl+Alt, and Ctrl+Alt+Shift among a few other functions.
</snip>

If you look on the left side of the Keyboard Layout tool you'll see check boxes for the Modifier combinations. You would have to check the CTL box and change the current "y" definition also.
