Does this persist when you boot the computer to a clean state?

How to configure Windows XP to start in a "clean boot" state

Also, make sure that the machine is virus/pest free as pests can cause all kinds of problems or unexpected behaviour.

The first bugcheck error is easy to figure out because the driver (BHDrvx86.sys) is identified in the error message:

PAGE_FAULT_IN_NONPAGED_AREA
Bug Check Code    : 0x10000050
Caused By Driver  : BHDrvx86.sys

This is an 0x50 error (PAGE_FAULT_IN_NONPAGED_AREA) and it is almost invariably always caused by bad device drivers or bad filter drivers, in your case it's caused by BHDrvx86.sys and it appears to be a file system filter driver, the kind of filter driver used by AV software, this was already pointed out to you in another post.

As for your other error messages, in particular the 0x7f  (UNEXPECTED_KERNEL_MODE_TRAP), if they persist after you get rid of the defective AV filter driver or when you run the computer in a clean boot state for an extended period of time then I would strongly suspect that you have a hardware problem, bad RAM being a common cause.  If the RAM tests fine then this can be hard to troubleshoot, it is often caused by a bad CPU or motherboard but you really need a competent hardware person to pinpoint this or else you may just end up on a costly "fishing" expedition.
