Why not start with your boot time - you say it is slow.  Well how slow is it?  Unless you measure it you can only guess.  If you measure it and make an adjustment, you can measure it again to see if things get better or worse.

Measure it, make an adjustment, measure it again.  There will be no guessing or trying things that might work sometimes maybe - you will know.


You can download a good free tool called Boot Log XP (bootlogxp) and measure and see graphically your entire startup process from end to end.  Then you will know exactly how long it takes and how long each individual startup process takes.

Only when you know how long things take and what the slow spots are, can you make informed decisions about what to do about them.  On the other hand, you could just start trying experiments and guessing.  That may work too eventually, but it will take longer to figure things out.

You can get Boot Log XP from the following site:

http://www.greatis.com/

Before running Boot Log XP, set your system up to automatically login so there is no human factor involved and test the automatic login part first by rebooting to be sure it is working properly and you get logged in automatically to the desktop.

When you run Boot Log XP, you will need to restart your system so it can measure things.  Wait patiently for it to completely finish and you will get a graphical report in a Boot Diagram and then you can see how long each startup process on your system takes and the total time.

In the chart, you would be looking for long horizontal lines that show you how long each startup process takes.

You can also generate and export a report that you can save, share it, print it, etc.  If you can't figure it out or have questions, export the short text file report to a text file and copy and paste the contents of just the short text file report back here for analysis and suggestions.  Just send the short text report, not the long text report.  The short text report.

Then when you start to make adjustments, you can run the Boot Log XP tool again and see what effects your adjustments have on your startup time.

a. user say:

I remember when you were thinking about moving your paging file to a separate drive and in the bad old days of slower hardware, you might see a small performance improvement doing that but you would have to run benchmarks to measure things to see if it is worth it or not - but there is also a consequence.

Here is a MS article about that and explains some of the reasons you might want to do that and reasons you might not want to do that:

http://support.microsoft.com/kb/314482

Since you have IDE drives you can check your HDD interface settings in Device Manager to be sure they are set up optimally.

Here's how:

If you are using IDE drives, use Device Manager to verify the Transfer Mode of the IDE channels is set to something like DMA if available (depends on your hardware) and not the slower PIO mode.
 
Remember:  PIO is the slowest, DMA is the fastest.

This is easy to check and generally easy to fix and the mode would not have changed by itself,so if it has changed to PIO, change it to DMA and then figure out why it changed and fix it.

To launch the Device Manager console, click Start, Run and in the box enter:

devmgmt.msc

Click OK.

Expand the IDE/ATA controller section to see your IDE channels. Right click each one, choose Properties, and for each channel that has an Advanced Settings tab, determine the Transfer Mode. There are usually 4 channels to check in a desktop, maybe fewer for laptops so just check them all.

The fastest selection will always be some kind of DMA selection (usually: DMA, DMA5, DMA if available, UDMA, etc.).  If it is PIO, change it to DMA.
 
If you are not sure about what you see post back for help and advice.
 
If you make any adjustments, reboot and check the settings again to make sure any changes "stick".
 
What may have caused the change?
 
http://support.microsoft.com/kb/817472

Sometimes the changes don't "stick" or won't change and for that, there are these articles:

http://winhlp.com/node/10        lame hard disks that won't go back to DMA

Here is another one:

http://www.technize.com/is-your-hard-disk-cddvd-drives-too-slow-while-copying/

If you make a change, unless you have already run something like XP Boot Log to get some timings, you won't know for sure how the adjustment changed things...

Is there any other way to fix this problem so that I will not lose everything?  And how do I prevent this issue from happening again ?

It looks like I've just fixed the problem.....thanks to my I.T. Guru (Tony)

Step 1, download windows 10 using the link + instructions above.

Step 2, boot laptop/PC using USB stick

Step 3, let stick boot up and follow instructions until you see repair windows install, go through options and select command prompt.

Step 4, When you are at the command prompt run these options :-
Bootrec /fixMBR
bootrec /fixBoot
bootrec /rebuildBCD

once done type exit.

Step 5, reboot  as normal ......... and hopefully you'll be back in windows 10 with no lost of data.

Step 6, Enjoy ........
