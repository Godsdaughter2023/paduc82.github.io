Microsoft .NET Framework NGEN v4.0.30319_X86 (startup type: Automatic - Delayed Start)

sailfish say:

Problem Fixed!

In review of the C:\Windows\Microsoft.NET folder I took note of the fact that the Frameworkv3.0 & v3.5 folders were missing. Also, I noted that Win7 would not allow .Net 3.x to be installed outside of the Turn Windows Features On and Off (TWFOO ) window, i.e., every time I attempted to install it from a downloaded 3.x install file, it always stopped and gave the message that it must be installed using theTWFOO window which, in turn, kept terminating with the encryptic and useless message, "an error has occured. not all features were successfully changed ".

So, I considered the possibility that the TWFOO problem might be related to the fact that the 3.x folders were missing from the C:\Windows\Microsoft.NET folder. Thus, I copied theC:\Windows\Microsoft.NET\Framework folder in its entirety from a vanilla Win7 laptop machine to a flash drive, renamed the desktop Framework folder to FrameworkOLD and pasted the flash drive Framework folder in its place. I then brought up the TWFOO window, checked the Microsoft .NET Framework 3.5.1 box, clickedOK and, Voila! it installed without a hitch and programs that now rely on it are able to install.

Post Problem Review:

The TWFOO install process seems to require the existence of the v3.x folders and, quite possibly, some of the files in them before it will proceed with re-installing the package. I believe this is a bug in the Win7 system that could be resolved in one of two ways. One, if one downloads the install package from the Microsoft site, it should not have the TWFOO process inhibit the installation. Two, fix theTWFOO installation such that it doesn't refuse to reinstall a package simply because some folders are not present.
