Your path is larger than the DOS4G extender was designed to handle.   It has a limit of 250 characters (some sources indicate 260).    Edit the batch file the starts the application and simply add the line:

SET PATH=

(with nothing after the equal sign).   If you application does not use a batch file then create one with the SET PATH= statement followed by the name of your application (or you may be able to modify the properties of your icon, etc. to shorten your path which in more modern systems typically is significantly longer than 260 especially as additional applications are installed on a system).

You may have to add a path or two to SET PATH= if your application needs to be able to find support in for example the C:\WINDOWS\SYSTEM32 or other folders.    Starting the application from the CMD prompt will help in troubleshooting  errors.

This issue is also unrelated to operating system in use.  Once the path environment variable exceeds the maximum, this DOS4G extender error will occur.
