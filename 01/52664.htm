greg [msft] say:

Hi Malcolm,

1. In order to support screen off audio playback via low power audio (LPA), the application needs to be a native UWP Store app. For example, Groove or Pandora will support this.

2. There is a new feature called Adaptive Hibernate, where the device will learn your usage behavior. When it determines that you typically are not going to use the device again, instead of sleeping it will go straight to hibernate. This helps save battery power. When it determines you will be using the device again, a few minutes prior it will move from hibernate to sleep.

This setting can be changed at Settings > Power & Sleep > Save me power when my device knows I’m away. If this setting is Off, then Surface Laptop will only move from sleep to hibernation after a cumulative 6 hour period sleep session has elapsed (or) if a 5% battery drain occurred while in the sleep state – whichever happens first.

3. It will not automatically wake if it is hibernating. In some cases, like when you are using the device during a time that you normally don't, you may need to open the lid and press the power button.

Hope this helps,

Greg

Hi Robbin,

The Automatic Repair loop that you are experiencing can occur due to several reasons such as corrupted/missing system files, bad sectors on your hard disk, etc. We recommend that you boot your computer in safe mode so that you can troubleshoot the issue. To do so, kindly follow these steps:

  1. Start your system.
  2. After your computer shows its original equipment manufacturer's logo, perform a hard shut down by pressing and holding the power button until your computer shuts down.
  3. Repeat steps 1 and 2.
  4. Start your system again. This time, Windows will boot into the Windows 10 Repair Environment.
  5. Select Troubleshoot.
  6. Click on Advanced Options.
  7. Select Startup Settings, then click restart.
  8. When your computer restarts, press the corresponding key for safe mode to boot your computer in safe mode.

Once you're able to boot into safe mode, kindly run the System File Checker tool and the Deployment Image Servicing and Management tool. The System File Checker tool and the Deployment Image Servicing and Management tool check if there are corrupted system files on your computer. If the System File Checker tool detects incorrect, corrupted, or damaged system files, it replaces them with the correct file versions.   1. Launch the File Explorer.
  2. On the left panel, click on This PC.
  3. Right-click on your local disk (C:, D:, E:, etc.), then click on Properties.
  4. Select the Tools tab, then click on Check.
  5. Click on Scan drive. If errors are found, click on Scan and repair drive, then follow the on-screen instructions to repair your disk. Once done, check if the issue persists.

If Windows 7/8/10 came preinstalled with the computer, that is called an OEM license. For an OEM license, the license dies as soon as Motherboard dies. So essentially it is non-transferable. If that is the case you need to get a new license.
