To change these sizes, you would now use the options under here:

cmdrkeene say:
Those options don't exist anymore because they aren't compatible with the new DPI scaling to handle high resolution monitors (what Apple likes to trademark as "Retina" screens).  To a lesser extent it isn't compatible with the new .NET/Windows Run Time frameworks, so any application built in the past several years doesn't properly respect those setting anyway (driving tons of user complaints). So to put it in short, it's old code from the original NT codebase getting dug out.

tamas pataki say: Does that mean there's nothing I can now do to restore my settings to a legible and comfortable (for me) level?

cmdrkeene say:

You should still be able to accomplish the change you want by using a combination of icon sizes, system interface scaling, or both.  And font size can still be adjusted in nearly any area tool.

I'd set the DPI scaling how you like to get it close (however large you normally would have set the fonts/menu sizes using the legacy control panel), and then fine-tune your adjustments by the Ctrl-Scroll method on the icons (there's 256 sizes of icons to choose from, and fonts have no limit) in your programs like email/web browser/etc.

And unlike the old settings, these will be respected across any monitor (high definition or not, TVs, projectors, tablets, large displays, tablets and more) and for all programs.
