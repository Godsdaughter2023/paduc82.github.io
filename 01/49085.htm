How to enter command into command line program from Task Scheduler or - Microsoft Community

caleb_j say:

How to enter command into command line program from Task Scheduler or batch file

I was wondering if there is a way to enter a command into an already open command line program (bukkit server in my case, could be anything I presume) from a batch file or Task Scheduler. I can't find anything with info on how to do this.

An example: I open a command prompt and log into something like mysql. Now every hour I want Task Scheduler to enter a command into that command prompt.

Is this possible with Task Scheduler or a batch file? (Note: I just want to know if this is possible. My examples are hypothetical.)
cmdrkeene say:

Possible yes, although I don't think it'd be very advised.  I'm sure the better option is to have the batch file handle the login portion too. This way if a failure occurs, the task scheduler can handle it (close task if it runs too long, has a problem, etch).  You can also implement error codes in your batch file.  For example I have a real-world example where I use a simple batch file to login to a server and pull down HTTP data and save as a text file (CSV) every 10 minutes.  The files can be quite large and if the result is anything other than "success", my batch file will try again, for up to 8 minutes, before giving up and sending me an email about the failure (which has never happened outside of testing the functionality).

But for your "possible" question. Yes.  You could write a windows shell script (or really any script, like VBS) and have it send keystrokes to the command prompt window.  Save that script and when it executes it would perform those keystrokes.

Somewhat related: this type of activity (sending keystrokes) is what the UAC security system is designed to prevent.  You'll notice that when the UAC administrator permission dialog shows up, the desktop of your computer is hidden.  What's really happening is that you're being switched to another desktop (not unlike fast user switching), so that any running program on your account can't click "accept" without your knowledge.
