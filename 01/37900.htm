tryx3 say:

Dava,

1  Boot into Safe mode.  Try using Snipping tool to find out if the problem persists.

2.1  If the problem does not occur in Safe mode then it is caused by something outside Windows so investigate your applications.

2.2  If the problem persists in Safe mode then it might be a fault within your user account so check that by creating a new one & seeing if the problem is evident in that new user account.

2.3  If the problem persists in Safe mode & is not caused by a user account fault then Windows is at fault.  Try a Repair install [also called an InPlace upgrade]. 

  • It should fix Windows problems [but not user account problems]. 
  • I have never seen any untoward side-effects from this procedure and it is a lot quicker & easier than a real reinstallation. 
  • You will need to check your Windows settings afterwards but your applications will not be affected.
  • You need to make an installation disk using http://windows.microsoft.com/en-us/windows-10/media-creation-tool-install
  • To run the procedure, you boot normally [you do not boot from the installation disk].  Then insert the installation disk, browse to it in File explorer and run its Setup.exe file.  There are a few options to confirm then it just gets on with it while you go away and have a pot of tea.
  • If you have a third-party anti-malware application then it could be worth uninstalling it [either before you start or afterwards], getting their latest version and installing that afterwards.

Since the Snipping tool & Chrome work okay in Safe mode, you can conclude that the core of Windows itself is not responsible for the problem.

Clean boot procedures take quite some time.

  • By the way, the term guest user has a particular meaning that I do not think was your intention.  You can just refer to having made an additional local user account and giving it Admin-level privileges.
  • In general, all installed applications are available to all users so you should not lose access to any of them in a new user account.  There are some exceptions - some applications ask you during installation whether they are to be made available to all users or just the current user.
  • Any Admin account can give itself permission to access the files of any other user account so you would be able to copy across the files belonging to your original account.
  • Whether or not you link a user account to an MSAccount or keep it as a local account is up to you.  MSAccount-linkage allows syncing settings across devices and Windows 10 is designed to facilitate access to online MSAccount capabilities such as email.  But it is up to you.
