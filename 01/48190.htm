a. user say:

When a Windows Live ID is not used for a long period of time, it is deleted. Problem is, an Xbox LIVE gamertag linked to the deleted Windows Live ID still thinks it is linked to a Windows Live ID. Even if a new Windows Live ID is created with the same email address, it is not in return linked back to the Xbox LIVE gamertag. This makes it impossible to recover the gamertag or do anything with the gamertag that would require the Windows Live ID, including switching it to a new Windows Live ID.

For me personally, this problem is bad because I had assets in my Xbox LIVE gamertag that I can't get to now, but they're clearly visible when viewing my account from another account, so it's still there.

Using Live Messenger integration on the Xbox 360 console (even if you have no friends set up) will force the Windows Live ID (WLID) sign-in to remain active.

Using the WLID on a website (such as xbox.com, billing.microsoft.com) will also reset the expire timer

Only signing into Xbox Live itself on the console doesn’t reset the WLID timer; it’s the second login of Messenger that is counted.

If you are in the situation where the WLID has expired:

1) Do not perform an Account Recovery – No matter what level of troubleshooting is ever asked of you, by anyone, as doing this will strip access to the Xbox Live account because the Windows Live ID checks will fail.

2) Do not attempt to switch Windows Live IDs through the console – This will also fail with weird error messages

3) Do not re-create the Windows Live ID – Even though it will have the same member name (email address), it is NOT the same Windows Live ID, and Xbox Live will become even more confused, potentially breaking any account related to the new Windows Live ID that has the same name as the one that is expired.

Saying that, you can get your email recovered here:

support.xbox.com/.../gamer-profile.aspx

1. The same message every time:   "DELIVERY STATUS NOTIFICATION FAILURE: ...

That isn't a Windows Live Mail error message, it's an email from a mail server to tell you that your message couldn't be delivered. So you message) you receive.

When a message bounces like this, there is usually a Non-Delivery Report (NDR) which explains why the message was undeliverable. This may be in an attachment to the Delivery Status Notification (DSN) that you received, or you may need to examine the source of the notification. You are looking for lines like this:
 

       <email address>
Reporting-MTA: dns;mailserver1.domain.com
Received-From-MTA: dns;mailserver2
Arrival-Date: Tue, 22 Mar 2016 14:21:03 -0700
Final-Recipient: rfc822;  <email address>
Action: failed
Status: 5.5.0
Diagnostic-Code: smtp;550  <email address> unknown user account


The Diagnostic-Code will often explain the difficulty.


If the NDR is in an attachment (often with the extension .dat), you can open it in Notepad. If there is no attachment, select the DSN in the message list and press Ctrl-F3. If you can't make sense of the NDR, paste it into your reply here.

You can read a bit more about this here: How do I interpret the delivery failure message that I have received?

I don’t want you to confuse file type with extension. Here is an example where a new program is being ‘manually’ associated with specific extensions and a program.

 

EXPLANATION OF ASSOCIATIONS AND FILE TYPES

The assoc command associates extensions to file types. The ftype command associates a file type with an application. For example, if you had a file type called myaccs that had a program file named ledger.exe and used files with the .ldg extension, the manual assoc and ftype commands would be created as follows. Press the Windows Logo key+X (or right-click or press and hold the bottom left corner)  and choose Command Prompt/Windows PowerShell (Admin). Type each of the following followed by the Enter key:

 

assoc .ldg=myaccs

assoc .acc=myaccs (i.e. more than one extension can be associated to a file type)

ftype myaccs= %SystemRoot%\Program Files\ledger.exe %1

 

Now, when you double-click or double-tap on a file with the extension .ldg or .acc, Windows will recognize it and launch ledger.exe.

 

REMOVE AN ASSOCIATION (which I believe you want to do)

To remove the association of .acc in the above example, simply type assoc .acc=  and press Enter. If you remove the wrong extension, it is reversible by typing assoc .acc=myaccs (as per above).
