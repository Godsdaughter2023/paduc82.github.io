Window sizing is a Windows function, not part of the program. The program will however remember program and message window sizes and positions.

... there is not option to restore down, only minimize or full screen! 

If the Restore button isn't active, the window isn't maximized. Use the control menu (Alt-Spacebar) to move and resize the window using the arrow keys. For example,

  • Press Alt-Spacebar S to activate Size.
    • Press Down arrow ↓ once to select the bottom border;
    • Press Up arrow ↑ three or four times to reduce the height of the window by 30 or 40 pixels;
    • Press Enter to confirm.
  • Press Alt-Spacebar M to activate Move.
    • Press Down arrow ↓ to move the window down 10 pixels.

Similar commands resize the window sideways. Holding Control while pressing an arrow key restricts the resizing manœuvre to one pixel per press for accuracy.

For message windows, close all but one before resizing: the program will remember the size and position of the last one closed.

Once you have the program window at the size and position you want, close the program and wait a few minutes for it to finish its housekeeping. Now re-launch it - does the window open at the size and position it was before?

¡firedog say:

Your protocol here doesn't work. Alt-Spacebar-S gets me to a selection box  

I know it's a bit fiddly, but if you follow the steps precisely, it should work. For example,

  • Alt-Spacebar opens the window control menu.
  • Pressing S then selects the Size function. The menu disappears and you're left with a 4-way-arrow-pointer.  
  • One press of an arrow key then selects the edge to be moved. The pointer changes to a two-way arrow at the edge selected.
  • Subsequent presses of the arrow keys in the same plane (left-right or up-down) move that edge by 10 pixels per press in the direction of the arrow pressed. This naturally increases/decreases the size of the window.
  • Holding Control pressed while using the arrow keys to resize the window restricts the adjustment to one pixel per press.
  • When you're finished adjusting in one plane, press Enter to confirm. Then start again from the beginning to adjust the other plane.
  • To Move the window, press M instead of S.
  • In this mode, pressing an arrow key has immediate effect. One press will move the window 10 pixels in the dir
  • To move less than 10 pixels in one direction, press an arrow key once, then hold Control and press the opposing arrow once to move the window back one pixel.

I may be sticking my neck out here, but these manœuvres work in both Windows 7 and Windows 10, so I assume it's the same in Windows 8. They have been the same since Windows 2.0, which will soon be 30 years old.

I had the same/similar problem: Lenovo Thinkpad T440S wouldn't recognize a SSD in a external enclosure through any of the USB ports. Other computers had no problems recognizing and using that SSD in the same enclosure through their USB ports.

Solution:

In the BIOS settings (press F1 while booting, make sure FnLck is ON otherwise function key won't be recognized), go to:

BIOS->Config->USB->USB 3.0 Mode

change to "Disabled"

Now the USB ports will run in USB 2.0 Mode.

Now Windows 8.1 had no problem "seeing" the external drive, in Disk Manager, File Explorer, etc.. I was able to clone the factory internal HD to the external SSD, then replace the internal HD with the SSD, and boot up & run with zero problems.

Something isn't quite setup right with the current, most updated USB 3.0 drivers for the T440S (as of Sept 6, 2014).  I gave up trying to figure out why and/or find the fabled drivers that would work.
