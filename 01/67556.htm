Windows 10 does not defrag an SSD because an SSD should not be defragged.

moek3405 say: Thanks for the respose Buildog, but my Question is why are my physical Hard Drives defined as SSD in W-10?

a. user say:

Ah yes! Technical terminology is just as difficult to understand as the technology itself. Especially when the Marketing Department gets involved. But I digress.

Until recently there was only one type of hard drive, so there was no reason to call it anything else but a 'hard drive' or a HDD (hard disk drive.) Recently a new type of hard drive came on the market called a solid state drive (SSD.) So now there are two types of hard drive: a 'hard drive' and a solid state drive.

Both types of hard drives can be defragmented, but a SSD needs it much less and the activities of a defragmentation would reduce its useful life. For that reason, we don't defragment SSDs and neither does Windows.

laserbeak43 say:

Windows 10 in VMWare Player?

I want to migrate to Linux, but I'm aware of the fact that some apps just wont work there. How do I keep my license intact if I run Win10 from VMWare Player?

ralf_g say:

You can't unless your previous OS was a full retail version. If your previous OS was an OEM version (came with the computer) then the Win10 license is tied to the previous OS licence which is tied directly to the computer hardware it was installed on. A VM counts as an entirely different computer.

If however your previous OS (say Windows 7 for example) was a full retail copy and not an upgrade version itself, then you can install it on any single computer. Once it is removed from your computer you could install the Win7 onto a VMWare virtual machine instead and then upgrade that to Windows 10 (before July 29).

Alternately, there is also the possibility of running Insider Preview editions of Win10 in a VM.

Ralf_G is right on the money. Just keep in mind that the Insider builds are intended for testing only, so at some time in the future they'll stop working. That could be a short time or a long time.

Microsoft Windows Update on Windows 7 searches forever. (SOLVED) with negative replies.

Microsoft Windows Update searches for updates forever on Windows. When reinstalling a clean copy of the Windows 7 operating system, I notice a lot of users are having this issue with Windows Update searching forever and not finding none what-so-ever.

PHASE 1:  Use a copy of Windows 10 ISO on a DVD or USB device that contains the bootable image. PHASE 2: If you open Windows Update in the Control Panel and it searches forever while seeing the Windows Update icon in the taskbar, it is not on same level with the background status of its update. Use the Windows Update Troubleshooter by navigating: http://windows.microsoft.com/en-us/windows/troubleshoot-problems-installing-updates There are at least two fixes with Windows Update when running Microsoft Fix It; do not run as administrator

PHASE 3 (Recommended but not required): After completing the second phase, consider creating a backup system image drive.
