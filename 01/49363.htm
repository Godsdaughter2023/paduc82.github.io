I did schedule a chkdsk /r on the system drive for the next restart and maybe that did the trick?  Personally I tink I have just been on an extended 'turn-it-off turn-it-on' journey we are all too familar with.

This is what I did and this time it worked, and I think the only difference was that a different Microsoft cummuative release was finally loaded.

First thing to say - make sure you have a backup.

  1. Start PC in Safe Mode - this seemed to stop my PC restarting, but does not allow you to check all the drivers are the right version or check what your Microsoft update history is (at least for me anyway)
  2. Restore a suitable backup from a time before you think the problem was introduced
  3. Start again in Safe Mode to prevent a 'Critical Process Died' restart
  4. Edit gpedit and change 'Configure Automatic Updates' to option 5 'Allow local Admin to decide'. To edit this navigate to Windows Components>Windows Updates>Configure Automatic Updates
  5. Restart in 'normal' mode
  6. Hold breath and hope system stays up
  7. Backup if stable
  8. Apply updates that are pending
  9. Restart and hope system is stable

You could then repeat loading the backup from 7 if the updates cause instability, waiting until there are different Microsoft Cummulative Updates to install - at least you will have a stable system to use even if not all latest security patches and drivers loaded.

Obviously also worth searching, or raising an issue, on this forum to see if others have the same issue and found a problem.

I hope this helps someone in the future.
