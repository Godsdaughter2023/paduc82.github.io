mark john fernandez say:

What's the difference Recovery Drive and Media Creation Tool

Can somebody tell me the difference between Recovery Drive and Media Creation Tool?
sher amo say:

Hi Mark,

Recovery Drive is a bootable USB drive that gives you access to the same troubleshooting tools as a system repair disc, but also allows you to reinstall Windows if it comes to that. To achieve this, the recovery drive actually copies the system files necessary for reinstallation from your current PC. To create one, you can check this article.

While the Windows Media Creation Tool will allow you to download the Windows 10 ISO directly from Microsoft, without having a product key. You can use it to clean install or Upgrade to Windows 10. It can be used to download the following editions of Windows 10 – Windows 10 Home, Windows 10 Home N, Windows 10 Home Single Language, Windows 10 Pro and Windows Pro N. To create one, you can check this article.

Hello,

One in the first batch of patches that is applied on a fresh Windows 10x64 FCU install is not dealing with the 8700K CPU (at stock settings) properly. The maximum multiplier should be 47x for a 4700 MHz speed/one core.

- after the patches are applied on the fresh OS and the PC rebooted, the maximum multiplier is now 46x. No core will ever go above 46x, regardless of workloads and time observed. Before patches you could immediately see 47x.

There is no other modification to the PC other than the patches you see in the image. I can't be sure which one exactly is the culprit.

Kindly do not suggest running SFC and DISM or other such generic stuff. This is a bug that's easy to reproduce and happens on default settings for a fresh OS installation. There are no modifications or overclocking of any sort in the BIOS and the CPU runs at all default clocks and settings. You can see the maximum 46X in HWinfo screenshot below. 

Again, if I install the OS fresh and don't apply any patch, the CPU runs as expected.

Many thanks to TerkildHolst who had the solution for Windows 7!  It seems that Google Chrome causes this problem.  To fix, copy the following bold text.  Save it to a text file.  Change the text file name to ftp.reg.  Double click the file and voila!


Windows Registry Editor Version 5.00

[HKEY_CLASSES_ROOT\ftp]
@="URL:File Transfer Protocol"
"AppUserModelID"="Microsoft.InternetExplorer.Default"
"EditFlags"=dword:00000002
"FriendlyTypeName"="@C:\\Windows\\system32\\ieframe.dll,-905"
"ShellFolder"="{63da6ec0-2e98-11cf-8d82-444553540000}"
"Source Filter"="{E436EBB6-524F-11CE-9F53-0020AF0BA770}"

Problems that prevent you from accessing files and folders after you upgrade to a new version of Windows can occur for one or more of the following reasons:

  • The folder ownership has changed.

  • The files are being stored in a Windows.old folder from your previous system.

 

If you recently upgraded your computer to Windows 10 from an earlier version of Windows, some of your account information may have changed. Therefore, you may no longer have ownership of some files and folders. You might be able to resolve this issue by restoring your ownership of the files and folders.

  • Open File Explorer (press Windows logo + E), navigate to This PC > Local Disk (C:) > Users >Select your User profile.

  • Right click on your User profile and select Properties.

  • Click the Security tab, under Group or user names menu, select your user name and click on Edit.

  • Click on Full control check box under Permissions for authenticated users and click on Apply and OK.

  • Select Advanced under Security tab.

  • Select your User profile under Permission entries and check on Edit, customize the permissions level and click OK.

  • Check the option Replace all child object permissions entries with inheritable permission entries from this object.

  • Click on Apply and OK to save changes and check if the issue is resolved.

Hope this helps.
