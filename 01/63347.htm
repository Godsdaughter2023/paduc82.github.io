This forum does not charge by the letter, nor does it charge extra for punctuation.  Please try to write in complete sentences.

Merely connecting your external drive to the computer will not re-direct the importation of files from the CD.  You still don't have enough space on your internal drive -- at least that's what seems to be the case from the incomplete information you provided.

Complete error messages are necessary to accurately diagnose problems.

There are a number of things you can do to -- temporarily -- free up disk space.  The only real solution, however, is to get a larger internal hard drive.   You can move data to your external drive, but if it is data that you use frequently, the slower access times may annoy you.  Therefore, I don't recommend that you move your My Documents folder in its entirety to the external drive, although you certainly could do that by following the linked instructions. You also could move your iTunes folder there (http://support.apple.com/kb/HT1364)

  • You can run Disk Cleanup (Start > Run > cleanmgr > OK)
  • You can reduce the size allocated to System Restore to about 1 GB (right-click My Computer > Properties > System Restore >Settings)
  • You can turn off Hibernation (if you don't use it) (right-click empty spot on desktop > Properties > Screen Saver > Power > Hibernate)
  • You can disable Disk Indexing, which will also speed up your computer a bit (http://lifehacker.com/031440/turn-off-indexing-and-speed-up-windows-xp)
  • You can reduce the size of your Internet browser cache (varies depending on which browser you use)
  • You can remove most of the $NTUninstallKBxxxxxx$ folders following the directions here:  http://windowsxp.mvps.org/Hotfix_backup.htm  (read the caveats in the grey box first)


But none of the above will really gain you significant amounts of space for very long.  The only long term solution is a bigger hard drive.

In the meantime, download and run JDiskReport, which will graphically show you what files are taking up the most space and thus suggest what files you should move to your external drive.  If you want to move an application, you'll have to uninstall it and then reinstall it specifying your external drive as the install location.  You can't simply move the files.

Bssdk.dll is a BsSDK belonging to ivt BsSDK from IVT. Non-system processes like bssdk.dll originate from software you installed on your system.

 

BTTray.exe is the tray bar process for Bluetooth products. It gives you easy access to the configurations and settings for the Bluetooth products. It is not an essential process but is required for the correct functioning of the program.

 

You can try to reinstall Bluetooth software and check. Follow these steps.

a. Press Windows key + Q, type Control Panel and then click on the Uninstall a program link listed under the Programs list.

b. Scroll down until you see the Bluetooth software and uninstall it.

c. Now, press on Windows key + R, type Device Manager and press OK.

d. Expand the Bluetooth option, then right-click on the Bluetooth device listed. Select Update Driver Software"

 

Click Search automatically for updated driver software. Windows will search online for the correct Bluetooth drivers that work with your device. Once it is done, restart your system.

 

Follow the steps and let us know the results. If you have more questions, please post, we will be happy to help you.

If, as you say, your old disk is still fully functional, and boots, I would suggest imaging the disk with something like Acronis True Image Home. I believe you can get a free trial version that will do the job. Store the resulting .tib file in an external drive.

 

According to Acronis, the clone function is disabled in the trial version. Sorry.

 

You can create a bootable recovery disk (CD) that will allow restoring from a .tib file to your main disk. Make the backup file, change disks, use the recovery disk and the .tib in the external drive to build a new boot disk. This one will not require re-activation, nor will a disk cloned with Clonezilla. Both will be identical copies of your original disk.

 

Would suggest restoring "AS IS" then adjust the partitions later. Allowing proportional restoration will result in a huge "System Reserved" partition, which you don't want.

 

There are also free utilities such as Clonezilla that will do the same job, but IMHO, not as easily or well.

 

If you want to clone the disk, use Clonezilla. You will need a USB to SATA adapter ($18US) to fire up the second disk. Clonezilla will work fine as long as the new disk is equal or larger than the original. Just to be perfectly clear, Clonezilla is a Linux application that should be run from a CD or from a USB stick. Unetbootin can make the USB drive for you. There is also a version called Clonezilla Live, that includes  the GParted partition editor along with several other utilities.
