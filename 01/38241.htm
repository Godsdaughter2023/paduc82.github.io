KB2858725 (.NET Framework 4.5.1) would not have been installed via Automatic Updates, even with the default Automatic configuration.

 

Note: .NET Framework updates & upgrades are notorious for taking an excessively long time to fully-install. During the process, your computer's constantly "checking in" with the update servers. Many things can interfere with this communication resulting in an even longer installation process.

 

If you used a third-party download manager [1] to download the installer for an update, your computer would still enter into this back-and-forth communication with the update servers. The only difference would be the size of the download: The Windows Update installer would be much smaller.
