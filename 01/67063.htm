Microsoft's view on purchasing Windows 10 OEM. Allowed or not?

As the title says, what is Microsoft's opinion on users purchasing Windows 10 OEM keys for use in their system builds. As per the Windows 10 licensing guide found under the Device Partner Center section (here: https://dpcenter.microsoft.com/en/Windows/License/dp-Windows-10-Licensing-Guide)

  1. The full version FPP (fully packaged product) is the sole retail offering in the retail channel. This addresses the greatest number of consumer scenarios (upgrading from previous versions of Windows, installing Windows on a Mac, virtual machine or building a PC from scratch).
  2. The DIY(personal use rights) addendum is removed from COEM (Commercial Original Equipment Manufacturer) product. Microsoft will carry the OEM EULA only and COEM product continues to be for exclusive use by System Builders, which is the product’s historical intention.

However, OEM licenses continue to be made available in the retail channel through legitimate resellers like Newegg, Amazon, BestBuy, B&H. Microsoft also does not define who qualifies as a "system builder" anywhere within that document or elsewhere on their site (as far as I can find). lemp say:
This is a technical support forum, primarily staffed by volunteers with no official connection to Microsoft. As a lawyer, I can tell you that any legal advice you get from a tech support person, no matter how technically skilled they are, is not something you want to rely upon.

That said, I can say that Microsoft has been remarkably inconsistent with how it treats OEM System Builder products. See, for example, this 2009 article by Ed Bott: http://www.zdnet.com/article/is-it-ok-to-use-oe...

Microsoft relented for Windows 8, but for Windows 7, 8.1, and 10, system builder software does not permit personal use, and is intended only for preinstallation on customer systems that will be sold to end users. See https://dpcenter.microsoft.com/en/Windows/Licen...

I have never, however, seen a volume requirement. That is, if you buy one OEM System Builder version of Windows 10, preinstall it on a system, and then sell that system to an unrelated end user, that seems to me to fit perfectly within the terms of Microsoft's license, even if you never do it again.

That retailers continue to sell OEM System Builder licenses is not relevant. As far as the retailers are concerned, anyone who buys such a product is supposed to abide by the shrink wrap license that comes with it.

Unlike activation, the software itself is not self-policing. That is, you can buy an OEM System Builder edition of Windows 10, install it on a machine without using the OEM Preinstallation Kit (OPK), and use it yourself ... and the software won't know that you aren't an "end user who is not the system builder." Is that "acceptable"? That's not a question I can answer for you.
