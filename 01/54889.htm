A workaround is to set the AppReadiness service to a startup type of "disabled", but this can break Windows Updates preventing you from being able to download a permanent fix from MS.

AppXDeployment-Server\Operational log has errors 401 and 404 with error code 0x8067000C

0x8067000C refers to "STATEREPOSITORY_E_BUSY_TIMEOUT_EXCEEDED"

Used ProcessExplorer to search handles and found to PIDs that are accessing this file, the StateRepository service and the AppXSvc service.

So it looks like the attempt to repair the modern shell is stuck because the StateRepository service doesn't have exclusive access to the database and the db is getting locked by another process. AppXSvc and StateRepository can't seem to get along.

I suspect this is also the case during normal startup - appreadiness hangs because appx and state repository can't read the state repository database.

davehrot say:

I found a fix that worked for me.

 

Finding a Solution for Me that related to DisplayLink USB 3.0 Video Drivers

I only got this error after purchasing a Dell S2340T touch monitor, which comes with DisplayLink drivers for single cable USB 3.0 video operation.

 

I didn't realise this was related and worked with Microsoft support over several calls to Xbox Live and MS Tech Support, but finally found out from DisplayLink Forum that many users were having this issue and getting the 0xc00d11d1 error in Video App in Windows 8 when trying to view rented videos.    As of 6/3/2013 they have not got a supported fix for the issue (something relates to pixel display issues in the video app itself).   The latest release of these drivers are 7.1 M1 1/2/2013 which still contains the bug causing issues with Video App (protected content, eg. rented video play back).

 

Note DisplayLink drivers are not specific to the Dell S2340T, but used for many single USB3.0 cable video adaptors. 

 

 

Solution / Work around:

Move to a 2 cable solution eg. HMDI + USB, and uninstall all displaylink software from Programs.  Restart PC.   I changed the display to run in HDMI and Green Mode to avoid it forcing automatic install of drivers when the display is detected by Windows 8.

 

Problem Solved ! I will monitor DisplayLink driver updates over the next few months and see if a fix comes out for it.
