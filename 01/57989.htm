lemp say:

When a computer comes with Windows pre-installed, the license from Microsoft is only valid for that computer.  That is, it can't be transferred to a "new" computer. 

Microsoft uses a secret algorithm that assigns various values to different hardware components of a system to determine if the system is "new" compared to the system on which Windows was originally installed.  If the "score" changes "too much," you get the "need to reactivate" message.

The one hardware component that Microsoft explicitly says can't be changed is the motherboard (unless it's a direct replacement of the same make/model).  Other than that, the actual algorithm that Microsoft uses is not disclosed, but from what I've read, the weighting of components is as follows, from highest to lowest:

  1. Motherboard (and CPU)
  2. Hard drive
  3. Network interface card (NIC)
  4. Graphics card
  5. RAM

So, if you had previously changed a graphics card, or added RAM or put in a new hard drive, when you disabled the NIC, Windows decided that you had a new computer and de-activated.

lemp say:

It has long been observed that changes to hardware other than a motherboard may trigger a "non-genuine" warning.  That's not to say that a system can't be re-activated (with the same license key) after such changes; it can, although sometimes it takes a conversation with a human being at Microsoft rather than the automated system.

Not surprisingly, Microsoft is not very explicit in explaining exactly what hardware changes may trigger the warning.  For example, Microsoft says is that "significant differences between the current hardware and the hardware that was in use [in] the original installation ... require product re-activation because the installation now appears to have been installed on a new (additional) computer."  https://support.microsoft.com/en-us/kb/305356

On the other hand, if your system came from one of the large "direct OEMs" (e.g., Dell, HP, Toshiba, etc.), activation is almost certainly accomplished by code embedded in the BIOS chip of the motherboard.  This is called System Locked Pre-installation (SLP).  In this case, Microsoft has said that "Every single piece of hardware could be changed on a PC with SLP and no reactivation would be required — even the motherboard could be replaced as long as the replacement motherboard was original equipment manufactured by the OEM and retained the proper BIOS."  https://technet.microsoft.com/en-us/library/bb457054.aspx?f=255&MSPPError=-2147217396

The list I cited has been repeated on many "tech" web sites.  For example:

http://www.makeuseof.com/tag/how-to-make-your-windows-7-pc-genuine-again-after-a-hardware-upgrade/

http://searchitchannel.techtarget.com/feature/How-Windows-7-hardware-upgrades-affect-licensing

To the extent that the list is not based on actual experience, I suspect it originates in that same TechNet article, which has the following to say on the subject (emphasis added):

At each login, Windows XP checks to see that it is running on the same or similar hardware that it was activated on. If it detects that the hardware is "substantially different", reactivation is required. This check is performed after the SLP BIOS check discussed above, if the SLP BIOS check fails. This means that if your PC is pre-activated in the factory using the SLP pre-activation method, all the components in the PC could be swapped, including the motherboard, so long as the replacement motherboard was genuine and from the OEM with the proper BIOS. As noted above, installations of Windows XP made using volume licensing media and volume license product keys (VLKs) will not have any hardware component checking.

Microsoft defines "substantially different" hardware differently for PCs that are configured to be dockable. Additionally, the network adapter is given a superior "weighting." If the PC is not dockable and a network adapter exists and is not changed, 6 or more of the other above values would have to change before reactivation was required. If a network adapter existed but is changed or never existed at all, 4 or more changes (including the changed network adapter if it previously existed) will result in a requirement to reactivate.

The "above values" referenced in that paragraph are (note that these are for the types of hardware that existed when XP was the latest version of Windows; to the extent that Microsoft uses a similar system today, I suspect that the list will have change a bit): 

Display Adapter
SCSI Adapter
IDE Adapter
Network Adapter MAC Address
RAM Amount Range (i.e. 0-64mb, 64-128mb, etc)
Processor Type
Processor Serial Number
Hard Drive Device
Hard Drive Volume Serial Number
CD—ROM / CD-RW / DVD-ROM
