tianjiwu say:

Hi Patrick,

In modern 64bit Windows, System32 is where 64bit DLLs are stored, and syswow64 (stands for sys Win-on-Win64) is where 32bit DLLs are stored. This is quite confusing. The reason is that a few applications always look for their DLLs in System32. To maintain compatibility, windows actually maps syswow64 onto System32 when loading 32bit applications. 

As for opencl.dll, windows has its own "official" version, which sfc uses. However, every graphics driver seems to come with their own version of opencl.dll. So whenever you install Intel or NVidia driver, your original opencl.dll gets replaced.

This is certainly a bad design, and MS should be solving the conflict. But generally speaking, this is not a critical problem, and the vendor-specific opencl.dll is supposed to work better than the official MS version. 

This theory is confirm by a senior MS engineer somewhere on this forum. Sorry I can not find the thread at this moment. But not all MS employees know this problem. MS supports may be taught that "sfc corrupted files" == "corrupted system", which is not the case for opencl.dll. They will insists on restoring a corruption-free system, which in this case means a corrupted graphic driver. 

There are a number of reports about fixing sfc errors and the system goes much more smoothly. The reason is either the vendor graphics driver is buggy at that moment, or sfc fixed other issues (file permissions, etc.) that makes the difference.

Hi Patrick, you should view this post in the Windows forum:

The System File Checker (sfc.exe) may incorrectly report that opencl.dll is corrupt in Windows 10

This issue may occur in Windows 10 Version 1511. When you install Windows, it installs a version of opencl.dll. However hardware drivers may also install their own version of opencl.dll, and if the driver version is different than the one installed by Windows, SFC may report this as file corruption and may not replace the file.

When this happens, the opencl.dll file is not actually corrupt. It is not necessary that you take any action – the report that opencl.dll is corrupt can be safely ignored.

Microsoft is aware of this problem with Windows 10 Version 1511 and is working on a resolution.

Hope this helps,

Greg
