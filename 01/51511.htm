How to make a DVD photo slideshow - Microsoft Community

phillipnichols say:

How to make a DVD photo slideshow

The output options I see are for other devices but I want to be able to burn it to a dvd. Is that not an option in Windows 10? Also what happened to Windows DVD Maker? Is it not available in windows 10?

considerate_guy say:

You can always put the resulting .mp4 video file onto a DVD disk, but in doing so, you are creating a data DVD. That will only be readable in devices that understand that format (e.g. another PC, a Blu-ray player, etc.)

But if your goal is to make a video DVD which is playable in a standalone DVD player, that won't work. Microsoft dropped support for creating video DVDs after Windows 7. Previously you could do this with Windows DVD Maker but that is not present in Windows 8 or later, nor can it be downloaded or installed on the later versions of Windows.

It's similar to what Apple did in dropping support for iDVD in their iLife suite several years back. Partly it's a licensing issue because including the MPEG-2 video codec with every copy of Windows incurs a cost. It's also a technology issue because DVDs are a legacy format. DVD players can't be used to play more than standard-definition (480p NTSC) video which is not ideal for multi-megapixel photos and HD videos.

If you still need to use this legacy format, you can look into 3rd-party alternatives like DVD Flick or DVD Styler. Or your PC might have been bundled with software to do it.

Otherwise, you might want to look at more modern options for handling HD video and the like.

lemp say:
As a preliminary comment, it's usually a good practice to configure a wireless printer with a static IP address.  I don't know if you've done that or not, but 192.168.1.73 looks far enough away from the usual range of addresses assigned by routers so that it probably is a static IP address.  If not, however, that doesn't matter for the following steps.

Confirm that your computer is connected to your router.
Open a Command Prompt window (Start > Run > cmd > OK), type the following and press Enter
        ping 192.168.1.73
Do you get 4 "request timed out" or do you get 4 replies?

Open a web browser (e.g., Internet Explorer, Firefox, etc.)
Type 192.168.1.73 in the address location and press Enter
Do you get the printer's web server showing status and other information?

If the ping requests timed out and you did NOT get to the printer's web page,
There is a problem communicating with the printer.  Please do the following for diagnosis:
Open a Command Prompt window (Start > Run > cmd > OK)
Type the following in the black Command Prompt window and press Enter after each line
     ipconfig /all > %userprofile%\Desktop\ipinfo.txt
     notepad %userprofile%\Desktop\ipinfo.txt
     exit
Click in the opened Notepad window and Edit > Select All (or press CTRL+A) and Edit > Copy (or press CTRL+C)
Click in your reply here and Paste (CTRL+V) the data from the Notepad file.
Close Notepad and delete ipinfo.txt from your Desktop

If the ping requests produced 4 replies and you DID get the printer's web page,
Open Devices and Printers and click Add a Printer

Normally, if you click Add a Network, Wireless, or Bluetooth Printer, after a few moments, your printer's IP address should show up here, assuming that your computer is connected to your router.  If it does, then just follow the rest of the wizard.

You get another chance if you click "The Printer I Want Isn't Listed."  You can select the radio button to "Add a printer using a TCP/IP address or hostname" and enter 192.168.1.73

Still problems, post back with a description of what happens.

lemp say:
Congratulations, and thanks for the feedback.

Sometimes I'm good, but this time I was just lucky.  Most of what I had you do was diagnostic in nature, to see if you could "see" your printer on the network. 

Using the "Add a Printer" wizard isn't much different that what the installation CD should have done automatically, so I'm surprised that it worked. 

As I mentioned, it's a good idea to configure wireless printers to have a static IP address rather than the usual default of "obtain an IP address automatically."  The reason for this is if you turn the printer off and -- while the printer is off -- you connect another device (computer, printer, smartphone, etc.) to your network, your router may well automatically assign the new device the IP address that previously had been assigned to the printer. 

If this happens, the next time you connect the printer, it may get a new (different) IP address from the router.  But the Windows printing system doesn't know about that, so all print jobs get sent to the old IP address -- where there is no longer a printer -- and seem to disappear into the void. 

Some HP wireless printer drivers have a setting that supposedly deals with this situation without the need for setting a static IP address, but I've seen too many people with this problem to rely on that.
