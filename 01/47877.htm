malkeleah say:

A. Every link from a Google search on that error indicates a problem with the optical drive. Is your drive SATA or IDE? If it is IDE, then check its DMA mode by going to Control Panel>Device Manager and expanding the IDE ATA/ATAPI Controllers category. Double-click on the entries to find your optical drive and then look on the Advanced tab. The drive should be in some flavor of DMA mode and not PIO. If it is in PIO mode, follow the instructions below.

Start>Search box>type: regedit
When regedit appears above, right-click it and choose "Run as administrator". Provide the password/OK to UAC to get an elevated instance of regedit. Navigate to:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Class\{4D36E96A-E325-11CE-BFC1-08002BE10318}

It has subkeys like 0000, 0001, 0002, etc. Normally 0001 is the primary IDE channel, 0002 the secondary, but other numbers can occur under certain circumstances. Check the DriverDesc value to see which one it is.

Delete MasterIdDataChecksum or SlaveIdDataChecksum, depending on whether the device in question is attached as master or slave, but it can't actually hurt to delete both. Reboot. The drive DMA capabilities will be redetected.

Open Device Manager again and check whether the device is now actually using DMA mode. If so, you're done. If the drive is still in PIO Mode then see B. below.

B. If your optical drive is SATA, then make sure you have the latest chipset (motherboard) drivers. I notice that you said "all of them installed" in reference to the drivers. That's ambiguous to me. If you actually went to Toshiba's website and downloaded/installed those drivers, OK. But if you let Win7 automatically install the drivers and don't have any yellow Bang marks next to devices, that's not OK. In that case you should go to Toshiba's site for your specific model laptop and install the Win7 chipset drivers from there.

Hello HariKrishna,

Thank you for the update on the issue.

There is a possibility that may be not all the services that run Internet Connection Sharing(ICS) are enabled. I'd suggest you to start from the beginning and make sure the below dependency services are enabled.

3. Be sure its Startup Type is set to Automatic and click OK.
 
All the following services must be thus configured in order to use ICS:
 
Application Layer Gateway Service
Network Connections
Network Location Awareness (NLA)
Plug And Play
Remote Access Auto Connection Manager
Remote Access Connection Manager
Remote Procedure Call (RPC)
Telephony
 

After verifying and/or configuring Automatic startup for each of these services, close the Services window and restart the computer before attempting again to enable ICS.

1. Press Windows + R, input AppWiz.cpl, click OK.
2. Select Windows Essentials 2012/Windows Live Essentials, click Uninstall/Change -or- Remove.
3. Click Remove one or more Windows Essentials programs.

4. Tick Photo Gallery.

5. Click Uninstall.

6. Download Photo Gallery and install.

 

Furthermore, in case you have other folders that you want or need to include to Photo Gallery try the following below.

 

1. On Photo Gallery, press ALT + F.

2. Click Include folder.

3. A pop-up window showing folders included. Click Add.

4. Browse for the folder and click Include folder.

 

Thanks!
