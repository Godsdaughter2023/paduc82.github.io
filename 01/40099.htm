wsmierciak say:

So I figured out why mine didn't work and that was a Pain.

https://docs.microsoft.com/en-us/windows-hardware/design/device-experiences/powercfg-command-line-options is the primary site for all the commands.  

It turned out, that all my Power Scheme names were either deleted or corrupt.  Because of this, I would get the "Not Available" error message.  Or if I tried to create a new Power Scheme, I got the "File cannot be located" error.

here is what i did.

open Admin Command prompt:

Type powercfg -restoredefaultschemes <enter>

Powercfg.exe -L (This should show you 3 power schemes.  the one with the asterix next to it is the currently selected scheme in use)

powercfg.exe /changename <Copy/Paste first GUID> "Primary Power"  (you must surround the name in quotes.  You can change the name in quotes to whatever you like...this was just my way to see if this worked)
powercfg.exe /changename <Copy/Paste second GUID> "Secondary Power"

powercfg.exe /changename <Copy/Paste third GUID> "Tirtiary Power"

Once that's completed, I was able to open the Control panel, and open power options, and view all my power plans.  From here, I was able to create new plans, edit existing ones, etc. 

I really hope this helps many of you out there with this issue that I had.

I am so sorry to hear that you are having these problems with your external hard drive. Preventing data loss in this situation honestly is tricky. My best recommendation would be to run a CHKDSK from the command prompt. Here are some steps for you:

1. Press Windows Key + X button to bring up the power users menu.
2. In the power users menu, select the Command Prompt (Admin) option.
3. Click Yes when you are presented with a UAC window requesting permission to launch the Command Prompt as Administrator.
4. In the new Command Prompt window, type chkdsk E: /f /r /x. Here you should replace E with your drive letter.

This option will run chkdsk to check and repair disk errors in Windows 10. The "/f" parameter tells CHKDSK to fix any errors it finds; "/r" tells it to locate the bad sectors on the drive and recover readable information; "/x" forces the drive to dismount before the process starts.

I cannot guarantee this will work or that you wont loose any data but its worth a shot, before reformatting.

Please let me know if this does help at all!

When I press the sleep button  ~ Turn Off the Display.

When I close the Lid  ~ Sleep.

Power Button ... change to 'Hibernate' for both options
