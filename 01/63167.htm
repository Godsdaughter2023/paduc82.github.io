a. user say:

Hi Stephen,

 

Thank you for getting back to us.

 

I understand that you're unable (well, sporadically) to receive emails from your account and that you are getting bounce back message prior to this issue. Let us see how I can help.

 

Let's interpret the bounce message that you are getting.

 

Traditional Bounce Code 550 > User’s mailbox was unavailable (such as not found)

Enhanced Bounce Code 5.7.0 > Other or undefined security status

 

The domain's message authentication policy > Email nowadays is easy to spoof. Users can't tell a real message from a fake one, and large mailbox providers have to make very difficult (and frequently incorrect) choices about which messages to deliver and which ones might harm users.

 

There is an organization (Microsoft is one of its contributors) helping email senders and receivers work together to better secure emails, protecting users and brands from painfully costly abuse. They are responsible for evaluating whether to trust the message for further handling, such as delivery.

 

There are emails that are protected by SPF and/or DKIM (DomainKeys Identified Mail), and tells a receiver what to do if neither of those authentication methods passes - such as junk or reject the message limiting or eliminating the user's exposure to potentially fraudulent & harmful messages. Technically DKIM provides a method for validating a domain name's identity that is associated with a message through cryptographic authentication.

 

Therefore, it provides a way for the email receiver to report back to the sender about messages that pass and/or fail the evaluation. In your case, some emails did not pass the evaluation.

 

What you can do is to encourage the sender to send a support request regarding Junk Mail Reporting Partner Program. If, in case, you've already done that, what we can do is to wait and hope that they will take the necessary actions to help resolve this issue.

 

I hope this somehow elucidate the matter.

 

Should you have other questions, clarifications or any other concerns, please update this thread and we will be happy to further assist you.

Warning: Microsoft does not recommend interrupting the CHKDSK process. Microsoft does not guarantee the integrity of the disk if the CHKDSK program is interrupted. For safety you can back up your data before using the check disk command.

 1. Click Start.

 3. Type at the command prompt CHKDSK /P and press ENTER to scan the disk.
 4. Then type CHKDSK C: /F /R and press ENTER to repair corrupt sectors, if there are any.
