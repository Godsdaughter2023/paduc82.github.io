Hi Don

The tradition of 'hating' any new version of the Windows OS started with Windows XP. This was the first version that was released after the DOJ Anti-Trust lawsuits against Microsoft and the hate Microsoft phenomenon was in full swing.

XP did not even start to become popular until sometime after the XP Service Pack1 was released.

Remembering Windows XP's early days:

This revolution started in late 2009, about a year before the first iPad was available, when the worldwide Mobile computer shipments first exceeded the standard Desktop PC shipments.

The XP Tablet PC Edition was mainly designed for Digitizer Pen Input. The Touch input was very primitive. It was a superset of XP and was initially only available as an OEM Pre-installed option. It was not included in the retail version of XP until the XP Service Pack 2 was released.

entegy say:

The issue was Microsoft trying to shoehorn a mouse/keyboard interface into a touch interface. It wasn't pretty and the tablets of the time simply didn't have decent power to run well. User interface elements were small and designed for precision. Hidden commands everywhere in the OS and apps assumed you have a keyboard always available and a mouse always available. Heck, mouse-hover events continue to be an issue to this day.

We had a few XP-based tablets at my previous job. They ended up being quickly abandoned because they were frustrating to use and there was very little software adapted to a touch interface.

The iPads are a combination of good marketing, and a touch-first interface, which makes them much more pleasant to use than XP Tablet edition.

With Windows 8, we have a reversal. Although I think a touch interface is more usuable with a mouse than a mouse interface is with touch, we have the opposite complaints: The Windows 8 Store, with touch first apps, is great on a tablet, but the Desktop is clunky with touch. But I think that's why the Surface Pro and similar Windows 8 tablets are gaining some traction: The touch first interface is there which is pleasant to use, but if you absolutely need Desktop software on your tablet, you can.

The flop of XP tablet edition wasn't irrational hate, it was simply a combination of pre-mature technology and lack of touch-first software. Luckily, all the good stuff continues to be carried forward, such as the excellent handwriting recognition. How my computer can decipher my chicken scratch is nothing short of magic.

wunders say:

You probably do not have rights to this folder.

Just because you are the owner or are administrator, permissions can still be set so that you have no access rights to a folder.

It would help to know whether you have XP Home or XP Pro and, if XP Pro, whether you have Simple File Sharing enabled or not...  however the following should work for whatever you have.

 

To see the current permissions on the folder, bring up a command prompt window

    Start -> Run -> "cmd"

and enter the command:

    cacls "C:\documents and settings\username\Desktop"

the following command should give you full permission unless there is a "deny" entry (you may need to be an administrator user to make it succeed):

    cacls "C:\documents and settings\username\Desktop" /T /E /G youruser:F

where "youruser" is the username you want to grant full (F) privilege to.

 

For more info on the cacls command, type

   cacls /?

 

HTH,

   JW
