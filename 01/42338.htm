Windows has a security feature which enables each user account to be designated as either Administrator or Standard (formerly 'Limited') User.  Standard User accounts are not allowed to install software without providing credentials (user name and password) for an Administrator account.  Standard accounts are intended for children or anyone else who you don't want to mess-up the computer. 

In Windows 10, right click the Start button and the select (left click) Control Panel.  In Control Panel find User Accounts.  In User Accounts you should see your user account and it should say (by the icon) whether you are an Administrator of a Standard user.  If it says Standard then report back because you need to do something about this.  If it says Administrator then your account does indeed have Administrator privileges.

Windows7 cannot obtain an IP address from certain routers or from certain non-Microsoft DHCP servers
 
To resolve this issue, disable the DHCP BROADCAST flag in Windows7. To do this, follow these steps:
1. Click Start, type regedit in the Start Search box, and then click regedit in the Programs list.
2. If you are prompted for an administrator password or for confirmation, type your password, or click Continue.
3. Locate and then click the following registry subkey:
4. HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\Interfaces\{GUID}
5. In this registry path, click the (GUID) subkey that corresponds to the network adapter that is connected to the network.
6. On the Edit menu, point to New, and then click DWORD (32-bit) Value.
7. In the New Value #1 box, type DhcpConnEnableBcastFlagToggle, and then press ENTER.
8. Right-click DhcpConnEnableBcastFlagToggle, and then click Modify.
9. In the Value data box, type 1, and then click OK.
10. Close Registry Editor.
 
By setting this registry key to 1, Windows7 will first try to obtain an IP address by using the BROADCAST flag in DHCP Discover packets. If that fails, it will try to obtain an IP address without using the BROADCAST flag in DHCP Discover packets.

 I am using a brand new Windows 7 Professional 64-bit installation disk. fabianriofrio say: Thanks for your quick reply.

I tried both of your suggestions, without any luck.

After reading some more posts regarding this error, it ocurred to me to disconnect my recently mounted secondary disk drive that I bought specifically for this installation (to use as Slave).

Neither my DVD drive nor my secondary HDD had jumpers. I specifically looked online on my HDD manufacturer's web site for Slave jumper configuration, but found none. After simply mounting the drive "as-is", it worked.

Check the security pages and see if the key is displayed on plain text. If the key is not displayed in plain text you could create a new encryption key making sure all of your wireless clients are updated with the new key...
