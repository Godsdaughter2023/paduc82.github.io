If there are no Administrator accounts on the system, you can boot to Safe Mode to access the Built-in Administrator account.  From this fail-safe account, you can create a new user account as an admin to use outside of safe mode..

  1. Turn off the computer.
  2. Turn it on, and immediately hold F8 until you see options.
  3. On the next screen, press the Number 4 Key to choose Safe Mode.

Now that you are in safe mode, you can use the built-in backup Administrator account to perform you necessary recovery. The built in Administrator account should not be used for any other purpose, or it won't be available in situations like this when you need it.

  1. At the login screen, select the Administrator account.
  2. After Safe Mode starts, open Control Panel / User Accounts.
  3. Select create account and create another account (be sure to make it an admin!).

When you're done, restart the computer normally.

This is a crazy setup. Having Guest as an admin account is a serious security risk. Here is what you must do:

Might try the following steps -


First, check your computer's IP configurations by following commands:


ipconfig /all
or
netsh int ip show config
or
wmic nicconfig get DefaultIPGateway, IPAddress, IPSubnet, MACAddress


Second, check if your default gateway can be gotten through ping:


for /f "delims={}" %g in ('wmic nicconfig get DefaultIPGateway ^| findstr [0-9]') do ping %~g


Third,
verify whether any of your two computers have the same IP (If yes, will conflict);
verify whether the IP addresses of your computers and your default gateway all belong to the same network segment.


Fourth,
gave up using 'obtain an IP address automatically', and try to use static fixed IP address.
For example, sets your network adapter (Name of NIC) "Ethernet", makes its IP as 192.168.1.10, mask as 255.255.255.0, and gateway as 192.168.1.1, just as below:
netsh int ip set address "Ethernet" static 192.168.1.10 255.255.255.0 192.168.1.1
For example, adds Google public DNS static IP addresses as preferred and alternate DNS server:
netsh int ip add dnsservers "Ethernet" 8.8.8.8 & netsh int ip add dnsservers "Ethernet" 8.8.4.4 index=2


Good luck.

arya s asok say: Hi,
Thank you for posting in Microsoft community!
You've reached the right forum for your question. I would suggest you to follow the troubleshooting steps to check if it helps.
The System Volume information folder keeps the restore point entries. Usually it is not recommended to delete the entries from the System volume information.

Try the steps to remove the files to get more drive space.
a. In the dialog box, select the drive where Windows 7 is installed (usually C: ). The drive will be scanned.
b. Select the Clean Up System files button. Select the C: drive again.
c. You will now see a new tab called More Options. Click this tab and in the System Restore and Shadow Copies section, click the Cleanup Button. Click OK and click Delete Files.
d. This will delete all but the most recent System Restore Point.
You should repeat this routine about every 2 weeks to keep those files cleaned up.
