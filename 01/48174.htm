gerbil say:

No sound via HDMI output when connecting laptop to TV

Original title: HDMI Laptop to TV - No Sound

Up until a few weeks ago it worked well:  I simply connected the HDMI cable between my laptop and TV and got both the audio and video on the TV (After selecting the right AV channel on the TV).  Now I only get the video.

I have looked through other questions on this forum but have not found anything that works.  I am reluctant to change too much, as it worked before.

In my "Playback Devices" I see two devices: "Speakers" (Default) and "SPDIF Interface".   

Laptop: Packard Bell Easy Note TJ65 Pentium Dual-Core CPU

Windows 7 Version 6.1 (Build 7600)

TV: Panasonic Plasma TX-PS42S10B (HD Ready)

Any ideas, anybody?  I feel that as I have made very few changes, the fix must be simple.  I haven't tried a different cable yet, but I'm pretty sure it's OK.

 
gerbil say:

BINGO!!

(So I had to go on to JustAnswers.com & pay £28, but I got the answer straight away ... wish I had gone earlier!)

Happy to share, though, for anyone struggling with this annoying problem...

I went on to Device Manager (Start Button > right click on Computer and selectManage) and uninstalled the device calledIntel(R) High Definition Audio HDMI.  Restarted the computer.  Connected TV with HDMI cable.  Right clicked on the little speaker symbol & slected Playback Devices.  Panasonic TV shows as a device.  Right clicked on it & set as default.  I got my sound back & everything works fine. 

It seems the driver had been updated somewhere along the line and was no longer compatible.  Following this process makes the computer put back its own driver.  All this time I was looking for a newer driver, and what it wanted was the old one!

For the record, the device in Device Manager is now called High Definition Audio Device. No mention of HDMI!

By the way, if you right click on the desktop you get a wizard to set up the TV interface.

Slow xp (professional) startup logon and logoff shutdown - Microsoft Community

a.v.n. say:

Slow xp (professional) startup logon and logoff shutdown

On an otherwise normal system, after some time of use parts of the windows system build up extra data stored in various files. I was concerned with the past items list and icons cache/data for the tray becoming too large because it was causing problems. In trying to fix the mixed up and or mismatched past items icons/descriptions in right click tray, customize notifications menu, I fixed the problem using a microsoft fix it solution and stumbled upon related fixes that ended up speeding up my logon/logoff times.


1: To fix the past items for the tray icons in right click tray, customize notification properties menu, run this microsoft fix it:
http://support.microsoft.com/kb/283084/en-us

To do this manually, the procedure is described in web link. However, I did not test the manual method completely because I had deleted the PastIconsStream data by creating a new entry and without deleting the IconStreams data on my own hoping this would work before I found the fix it page, and after restarting windows this left all the past items listed, but with folder icons in place of their actual icons. After that, I found the web page and decided to run the fix it tool for this problem and it solved it.


2: The reduced logon/logoff times occurred after applying this item, and the related first item, and the third item:
 
link: http://support.microsoft.com/kb/132668/en-us ; although microsoft suggested to Change the data value from its original value to 2000, it was already at 2048 and I changed it to 4096. Default is 500 on microsoft.com, but I had first run this microsoft fix it: http://support.microsoft.com/mats/windows_file_and_folder_diag/ and it may have changed this to 2048. I did not check first because I did not know to. Another microsoft fix with a different name but the same four manual menu items that I ran after the first one that also verified the first fix worked was found here: http://support.microsoft.com/kb/813711/en-us 


3: What I did besides the preceding info to make windows desktop load quicker on startup logon and on logoff shutdown and help windows explorer run faster* (logon went from 10-12 blocks to 4-5 blocks** in task manager's performance tab that shows cpu usage history): 

* My machine didn't have any other loading problems/delays since I installed windows xp pro, so the reason for so many blocks is that I set all services to automatic in administrative tools (two services rereturn to manual and two I set to manual) to see how the computer would run, which adds some time to the logon screen processing time (takes about 3-4 minutes total (1-2 otherwise), but I usually wait for the processor light to become idle anyway before logging on because I think there are some processes that sometimes occur during/after that time like possibly the two .NET services (Optimization and NGEN), which needs to be set back to manual from disabled after each time it runs (few times a year)) Although I would prefer all on automatic, because I believe that windows should by default automatically handle startup and stop states for all services as start, trigger start, stop, and trigger stop, most users do not need to set all services to automatic (export list to reverse changes easily) because windows xp is not fully configured to stop services properly when not in use as I believe it should. But if users at least start .NET services if needed plus any other important dependencies, and change nonnecessary items to manual or disabled, this could increase overall application performance. Check the services help menu or this these links http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/sys_srv_default_settings.mspx?mfr=true and http://technet.microsoft.com/en-us/library/dd349799(v=ws.10).aspx for suggested default startup state lists and expainations..These or the basic computer manufacturers default setup (depending on harware/software configuration settings, i.e. things that are enabled or not enabled change the default services startup type and start/stop states) might show the logon time reduced to 1-3 blocks. There are some service managers on the web that offer useful preconfigurations like gaming, networking, etc. I am using a free utility in Advanced Uninstaller Pro (this software contains other useful utilities) that can allow me to switch between saved profiles. Another useful suite towards these ends is Free Advanced System Care 6, which allows you to save and revert any changes. However, for my uses, I prefer the default computer manufacture basic setup and the default services setups with some minor changes. Other common things I did to improve performance include security sweeps from antivirus/antimalware programs, disk cleanup, careful registry cleaning and defragmenting, defragmenting the hard drive, and defragmenting the page file with system internals suite's pagedfrg from microsoft's website, which has further performance information on slow logons/logoffs and startup/shutdown problems including: enabling defrag boot optimize function in registry editor at HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Dfrg\BootOptimizeFunction by setting the enable value to Y. See: http://technet.microsoft.com/en-us/library/cc784391(v=WS.10).aspx , and also running the User Profile Hive Cleanup Service from http://www.microsoft.com/en-us/download/details.aspx?id=6676 for faster log offs. Also, I noticed in task manager (wuauclt.exe) that microsoft update cycles memory for about 20 to 40 minutes, so I switched back to windows update, and turned off updating in microsoft programs like silverlight, which can be done manually now on windows update, and this helped speed things up. After that I ran a microsoft fix it found at http://support.microsoft.com/mats/windows_update/. There are other common fix its that can be run from your machine, but fix it center beta is currently discontinued, so you have to go to http://support.microsoft.com/fixit/. Other performance related fixes are found here too.

** However, the shorter startup time that lasted for about four startups disappears some time after defragmenting the hard drive and goes back to 10-12 blocks. So, I defragmented more than one time in a row, after about 2 or 3 times the process sped up and I got much more ordered pattern of blue bars by compaction that was closer to a solid blue block by around 10 times. This helped along with other things and I was back to 4-5 Blocks for CPU usage history. I also noticed that on normal update speed, task manager CPU usage history progresses at one forward click per second. However, this was not true when I had installed compatible but generic intel drivers from intel.com instead of the recommended ones from the computer manufacturer. philrosenberg say:

Hi Anyone that cares.

I finally got this working, so here is a step by step guide to restoring backups of your system from an image, but putting bootable drives in a different order. I'm posting it just in case it is useful to anyone else trying to solve the some problem. Here goes:

Most of my woes were because I was swapping the order of the partitions. As I reported, when I left the order the same everything was okay, but I didn’t want Windows 8, which is the OS I use most as the last partition on the slowest bit of my hard drive. I think the main problem was that when booting up, Windows allocated drive letters in the same order as previously. So for example in windows 8 the fourth partition was allocated as C:, which happened to be the Data partition. This meant that all the paths hard coded into the registry as C: couldn’t be found and windows pretty much failed to boot. Similarly for 32 bit Windows 7, the C: ended up being the 64 bit Windows 8 partition, which obviously didn’t work well either.

So how do we fix it. Well importantly the answer is in http://paulski.com/zpages.php?id=2102. The id and drive letter for each partition can be found in the registry under HKEY_LOCAL_MACHINE/SYSTEM/MountedDevices. Look for Keys \DosDevices\C: and similar. To change which drive letter each partition gets, right click the key and select Rename then change the drive letter. How do you know which partition is which? The first four bytes of the value are the id of the drive, the second 8 bytes are little endian representation of the start of the partition, for example my first two partitions have values

80 00 00 00 00 00 50 06 00 00 00 00
80 00 00 00 00 00 10 00 19 00 00 00

So the first partition is on drive id 80 00 00 00 and starts at position 00 00 00 00 06 50, the second is on drive id 80 00 00 00 (the same drive) and starts at position 00 00 00 19 00 10. Using this information you should be able to id your drives. In order to edit the registry without booting windows there are instructions here http://4sysops.com/archives/regedit-as-offline-registry-editor/.

The second issue is the boot manager – it too probably expects the partitions to be in the same order. We can edit this with the bcdedit command. Instructions for this are here http://www.sevenforums.com/tutorials/2676-bcdedit-how-use.html.

So here is a recipe that should work. Fingers crossed. My backup uses Macrium Reflect free, which uses a Linux or Win PE recovery disk. Other image backup software should work too, but you will need access to a command prompt without booting into Windows so you will need a Windows 7 or 8 installation media or a Win PE recovery disk of some sort – if nothing else then download the free Macrium Reflect and create a Win PE disk from that.

  1. Perform your restore putting the partitions in the locations you want and resizing them as needed. I used the Reflect Linux recovery disk for this because it allows you to resize partitions and the Win PE one doesn’t seem to allow this. But make sure the Boot Manager partition (the 100 MB System Reserved partition) is in the same location as before – presumably first.
  2. Boot into your Windows install or Win PE recovery disk.
  3. Start a command prompt (on the install media I think this is under repair your computer->troubleshoot->advanced options, on an existing windows install right click cmd.exe and run as administrator) and find which drive has been allocated which letter by changing drive and using dir, Don’t worry, the drive letters assigned in your recovery disk bear no resemblance to that when you boot.
  4. Still from the command prompt run bcdedit. You will see all your startup entries, each will have an identifier in curly braces. Change the device and osdevice properties as needed to match the drive letters you just found. The command is bcdedit /set {id} device partition=C: and bcdedit /set {id} osdevice partition=C:
  5. Now enter regedit at the command prompt and click HKEY_LOCAL_MACHINE, then click load Hive. Open the first Windows partition’s system database which should be at %windir%\System32\Config\system, give it a name that you will remember like WINDOWS7_SYSTEM. Navigate to mountedDevices and rename the DOSDevices entries so that the rearranged partitions have the correct letter. You can identify the partition order by looking at the last 8 bytes of the value which is the little endian position of the beginning of the partition. Also presumably the order matches the order from the original system, so if you know your system was C:, D:, E: and you have swapped C: and D: then presumably swap C: and D: here.
  6. Repeat step 5 for the remaining Windows Partitions
Reboot and all should be well. If not, then there is no point asking me questions because you now know everything I do, but I wish you good luck.
