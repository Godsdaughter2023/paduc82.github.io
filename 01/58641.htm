Windows does not pay attention to the Read only attribute on folders - you can't make a folder read only but you can make the contents of the folder read only.

redscarab say:

OK I think I figured it out.

When I first deployed the workstations, I disabled the UAC because it annoyed many users who were using a whole bunch of legacy software at the time. Since those are now gone, I turned the UAC back on but it removed the inherit permissions from parent folder on certain folders. All folders on mapped drives were not affected. Only folders residing on the local drive were affected such as D:\Projects or C:\Temp. In order to see this, you need to drill all the way down into the Advanced Permissions menus which is why I didn't initially see any permissions problem.

Also on some computers, turning the UAC back on removed the username object permission from the user profile folder. The user profile folder should have three object permissions: SYSTEM, Administrators and Username permissions - the last one being removed entirely which affects email apps.

Turning the UAC back to disabled does NOT remedy the situation. The folders are left in their improper state. Manually restoring the proper permissions on all affected folders turns the blue read only square back to the normal grey checkmark after a reboot and the system behaves normally again.

The following is generally considered the Activation fix for those trying to activate FSX on Windows 10.  This applies to both FSX and the Acceleration pack:

1. Close FSX and run Registry Editor (press Windows key + R and type   regedit    and     Ok)

2. Expand the following registry key:            

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Class\{4d36e967-e325-11ce-bfc1-08002be10318}      

3. After you find the folder, double-click Upperfilters on the right side.

4. Delete any value there and type in   PartMgr  (note, this is case sensitive;  type it as shown here) and click OK.  

5. Close Registry Editor and reboot Computer.

6. Start FSX and check whether the Activation box comes up.

As always, the warning needs to go out, to be careful whilst moving about the System Registry.  Back it up if unsure.

a. user say:

Go to Command prompt.

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.  We may be able to replace the corrupted fules without needing to re-install the system (which is equivalent to a clean boot) as long as there aren't too many - and that's unfortunately very likely if it's saying the volume is corrupt.  In fact, you may need a new hard drive.

While in Command Prompt, type chkdsk /f /r and enter and let it run.  It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a cause.  I'd bet it finds some corrupt sectors.


Can you get into safe made (by clicking continuously on the F8 button while booting and then selecting safe mode with networking)?  If so, then please post any repeating error messages from the Event Viewer concerning startup (Start / Control Panel / Administrative Tools / Event Viewer).  Here's how to use Event Viewer: http://www.bleepingcomputer.com/forums/topic40108.html.


Go to Device Manager by going to start / search box and type device manager and enter and then double-click on the program icon that appears.  Check each device for a red x, yellow ! or white ?.  These identify devices with problems (probably drivers, but also conflicts or something else).  Click on each for further details and troubleshooting tips.  If you need to get drivers, do so from the computer vendor or device manufacturer (NOT from Microsoft Updates).  In fact, you should turn off automatic driver updates from Windows Update as follows: http://www.addictivetips.com/windows-tips/how-to-disable-automatic-driver-installation-in-windows-vista/. Proceed as follows: http://pcsupport.about.com/od/driverssupport/ht/driverdlmfgr.htm.  Once you have the drivers, you can install them through Device Manager as follows: http://www.vistax64.com/tutorials/193584-device-manager-install-driver.html. 

To test your hard drive, check the manufacturer and then get the diagnostic utility from: http://www.techsupportforum.com/hardware-support/hard-drive-support/302602-hard-drive-diagnostic-utilities.html.  If it fails the test, replace it.  If it passes the test, then there's probably nothing wrong with it.

You can also test your RAM at http://www.memtest.org/.  If it fails, test each chip individually to find those that are bad and replace them.  If it passes, RAM is probably OK.


If it comes to a clean boot, you can use Knoppix http://www.knopper.net/knoppix/index-en.html with a good ISO copier like: http://isorecorder.alexfeinman.com/isorecorder.htm along with a blank CD.  This should give you enough access to the system (if you can't get in any other way like with safe mode) to backup your important data.  Once done, you can do a clean install either using the genuine Windows Vista Installation Disk or the Recovery Disk with the Recovery Partition or the OEM Reinstallation Disk (whatever process is dictated by your computer manufacturer).  Make sure you backup your important data first because you will lose everything on the partition during this process (unless the OEM installation disk allows for repairs/upgrades which you should try first if available).  To do a clean install proceed as follows: http://www.winsupersite.com/showcase/winvista_install_03.asp (adapted as necessary by the procedures of your computer manufacturer).  Then you will need to re-install all your programs, reset all your preferences, reconfigure your network and email settings, restore your backed up data, run Windows Update with possibly over 100 updates pending,...

Good luck and I hope this helps - but you may need to buy a new hard drive and you may need to do a clean re-installation (we'll see from these procedures).
