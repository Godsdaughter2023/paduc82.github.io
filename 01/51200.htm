    UniDrv (Universal Printer Driver or Unidriver) is a GDI-based Microsoft Windows universal printer driver and architecture for non-PostScript printers. It is used to simplify driver development of non-PostScript printers (also called Winprinters) for printer manufacturers. Unidrv allows the creation of a printer-specific minidriver in the form of a GPD (Generic Printer Description) file, similar to a PPD file, which is much simpler than kernel mode driver development.

    OK, so I fixed it. The issue was that between dual-booting with Linux and some other shenanigans, the system didn't boot from the "System Reserved Partition" at any point in time because it just wasn't active. That seems to be really bad news to some updates, and Windows Update doesn't seem to have any meaningful diagnostics when that happens. Boo.

    The fix was as follows:

    • Make the System Reserved Partition active again (Windows onboard "Disk Management", just note that the system likely won't boot afterwards because that partition is not all well.)
    • Boot from a Windows install medium, navigate through advanced troubleshooting until a command line pops up
    • Use "bootrec /FixMbr" and "bootrec /FixBoot" to make things presentable to BIOS again (thus killing dual-boot)

    At that point, all updates installed without a hitch.

    As an added bonus I converted the system disk to GPT so the sucker now boots both OSs without clobbering each other's bootloaders. (the rEFInd Boot Manager is gold!)

    andre da costa say:

    I would certainly decrypt the drive first and uninstall the TrueCrypt software.

    When Microsoft provides pre-release versions of Windows for testing, they usually use the most complete edition, case in point, Windows 7 Ultimate, Vista Ultimate were the sku's (editions) used for evaluation. The Ultimate edition includes BitLocker Drive Encryption. Also, an encryption program such as TrueCrypt works at the kernel level, so there might be compatibility issues/

     

    Please note that Windows 8 will not include in place upgrade (otherwise known as upgrade setup) support in the first pre-release builds. Microsoft President for Windows and Windows Live made this announcement on the Building WIndows 8 blog:

    Media Center will not be part of the first pre-release builds. Some other features/capabilities will not be in the first pre-release builds including: Windows 7 games, DVD Creator, upgrade setup, Dot Net 3.5 (Note there are perhaps a couple of other relatively low profile items but just wanted to hit the major ones here). These are engineering decisions as well as business decisions.

    http://blogs.msdn.com/b/b8/archive/2011/09/02/reflecting-on-our-first-conversations-part-2.aspx

     

    So, you will either have to install on a partition or in a virtual machine if you don't want to wipe out your existing Windows 7 installation or backup your Windows 7 installation before attempting the upgrade.
