¡firedog say:

My accounting software requires a MAPI compliant email client ... Windows Live 32 bit version is compliant ...

You will read in lots of places that email from Sage 50 will only work with Office Outlook, not any other client. Windows Live Mail 2012 is a 32-bit application, and I'm not sure that Sage won't be able to use it for email.

I also heard that Outlook 2013 is NOT MAPI compliant?

This is presumably a misconception. Sage 50 is a 32-bit application, so it will only work with a 32-bit mail client. If you try and use it with 64-bit Outlook 2013, a misleading MAPI error message will be displayed. It seems that the click-to-run versions of OL2013 don't work with Sage either.

I don't have Sage, so I can't test this. However, this procedure has worked in the past for programs that it is claimed won't work with Windows Live Mail. You didn't tell us which operating system you have on your new computer; these instructions are for 64-bit Windows 7*:

  1. Close down all running programs and set a system restore point.
  2. Press Windows key + R, type regedit into the box and press Enter. Give UAC permission to open Registry Editor.
  3. In the left-hand pane, navigate to
    HKEY_LOCAL_MACHINE\SOFTWARE\Clients\Mail
    and click on Mail.
  4. In the right-hand pane, double-click on  (Default) to open the Edit String dialogue. In the Value data box, type Windows Live Mail then click OK.
  5. Expand the Mail key and then click on Windows Live Mail. In the Name column, do you see values called DLLPath and DLLPathEx?

    If not, add them:

    $
    Right-click in the right-hand pane and select New > String value
    Type DLLPath to replace the highlighted New Value #1 and press Enter.
    Double-click on the new value to open the the Edit String dialogue. In the Value data box, type
    C:\Program Files (x86)\Windows Live\Mail\smapi.dll
    then click OK.
    Start again from $, but this time type DLLPathEx as the new value name.

  6. Press F5, then close Registry Editor and reboot the computer.
  7. Launch Sage and see whether you can configure the email function to use Windows Live Mail.

*  For a 32-bit system, the only difference is the name of the program files directory: C:\Program Files\ instead of C:\Program Files (x86)\ in step 5. The procedure will be more-or-less the same in W8.1

a. user say:

Probably not the best idea...

If you are using/relying on the built in NT Backup to backup your system, that is primitive technology (obviously - since it will ask you for a floppy when it is done).  If you have a disaster (even a minor one), you are going to have a big problem trying to recover your stuff if you don't have a floppy drive - and maybe even if you do have a floppy drive you are very likely still to a big problem if you need to restore your stuff.

If you want to make a backup, an image or some kind of clone of your system, it makes sense to use modern technology - and that technology is not built in to XP.

There are lots of popular options for third party XP backup and imaging software(s).

A popular one folks seem to like is from Acronis.

With that (free 30 day trial) you backup, you can make a bootable CD or bootable USB (that's modern, huh?) and use that in the event of a disaster.

They used to have a totally free version - maybe not anymore - just the 30 day trial, then you have to buy it, but it is well respected.

It is so much more robust than the XP backup, it is laughable.

There are many other options if you do a little Googling...

Here are some products that forum user LemP suggested might be appealing.  use Google to find them and read about their features, and capabilities.  Some have free versions or you can purchase them to get more features:

If you have a Western Digital drive, you can get a version of Acronis True Image free from WD:

http://support.wdc.com/product/downloaddetail.asp?swid=119

Or you can obtain cloning software separately (this is not a comprehensive list and is no particular order; some are free, others not):

Acronis True Image Home
Easeus ToDo Backup
Paragon Drive Copy  or Partition Manager Personal
Casper
Norton Ghost

Some disk cloning software ideas for you to Google:

XXClone
Marcium Reflect Free
DriveImageXML
Acronis True Image Home 11
Easeus ToDo Backup
Paragon Drive Copy  or Partition Manager Personal
Casper
Norton Ghost

Whatever method you choose, it is prudent to test your method from end to end to be sure it works.

You can create a temporary folder somewhere on your system and copy a bunch of files into it, perform your backup, then delete the temporary folder and be sure that you can recover it from your backup.

The day you need your backup/restore process to work is not the day to find out it doesn't work.
