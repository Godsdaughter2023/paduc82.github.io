tryx3 say:

Jim,

If you still have this problem then I can help.

1  First of all, you need to check which accounts exist on your computer - Open a command prompt & enter  wmic useraccount list full  For each block of results [each user account], check these lines

1.1  Name=  that is the username

1.2  FullName=  local user accounts have no entry, an entry here means it is an MSAccount-linked user account

1.3  Disabled=  either True [disabled] or False [enabled].  This is also shown in the line Status=  OK means the account is enabled, Degraded means disabled

[[You could also use something like wmic useraccount list full >D:\Desktop\UserAccountPropertiesList.txt to send the results to a file that you can keep for future reference]]

2  Note that you should find the accounts with usernames Administrator & Guest disabled.  You might also find one called defaultuser0 and that should also be disabled.  You do not want to use those accounts anyway. 

[[Note that referring to the account with username Administrator is not the same as referring to an account as being an Admin account.  The account with username Administrator is a special built-in recovery tool for particular conditions.]]

3  To find out which accounts are Standard users and which are Admin users, enter this in a command prompt for each username in turn net user <Username>

3.1  In the results, look for the line Local Group Memberships. 

3.2  If an account is a member of the Administrators group then it is an Admin account.  If not, then it is a Standard user account.

4  Now you know all the accounts'

  • usernames,
  • local or MSAccount-linked,
  • enabled / disabled status,
  • Standard user / Admin user status

   so you can say what remaining problems you have i.e. which results are not what you want.
