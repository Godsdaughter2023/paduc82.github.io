ken blake say:
On Sat, 26 Jun 2010 14:29:53 +0000, J W Stuart wrote:
 
 
> While your computer does have 4GB (Decimal value = 4GB, Binary value = 4096) your grahics chip and other hardware devices make use of some of the 4GB of memory, leaving you with less than 4GB (typicall about 3.2GB) available for Windows.
 
 
 
Sorry, that's not quite correct. The hardware does *not* make use of
any of the memory. It makes use of the *address space*. If it made use
of memory, and you had 2GB, you would not be able to use all 2GB.
 
Here's my standard post on this subject:
 
All 32-bit client versions of Windows (not just Vista/XP/7) have a
address space (64-bit versions can use much more). That's the
theoretical upper limit beyond which you can not go.
 
But you can't use the entire address space. Even though you have a
4GB address space, you can only use *around* 3.1GB of RAM. That's
because some of that space is used by hardware and is not available to
the operating system and applications. The amount you can
use varies, depending on what hardware you have installed, but can
range from as little as 2GB to as much as 3.5GB. It's usually around
3.1GB.
 
Note that the hardware is using the address *space*, not the actual
RAM itself. If you have a greater amount of RAM, the rest of the RAM
goes unused because there is no address space to map it to.
