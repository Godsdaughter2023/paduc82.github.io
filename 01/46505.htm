It looks as if you're talking at cross-purposes.

  • You cannot sync between Windows Live Mail (a mail client program running on your computer) and a mobile device.
  • You can sync between {Windows Live Hotmail or outlook.com} (two names for the same web-based service commonly accessed via a browser. They have different user interfaces) and Windows Live Mail.
  • You can sync between {Windows Live Hotmail or outlook.com} and a mobile device.
  • You can sync between {Windows Live Hotmail or outlook.com} and Microsoft Office Outlook (another mail client program running on your computer).
  • The email address you use for a {Windows Live Hotmail or outlook.com} account can be @hotmail.com, @live.com or @outlook.com, along with many other possibilities.

    auggy say:

    A handy application to read minidump files is BlueScreenView:

    http://www.nirsoft.net/utils/blue_screen_view.html

    However, you may find using WinDbg of the Debugging Tools for Windows gives you more complete information.

    You can download the Debugging Tools for Windows from the following link:

    http://www.microsoft.com/whdc/devtools/debugging/default.mspx

    There is a tutorial in the following link on setting up WinDbg, such as setting the symbol path and utilizing the !analyze -v command:

    http://www.sevenforums.com/crash-lockup-debug-how/26584-2-intermediate-configuring-debugging-tools.html

    Basically, install the Debugging Tools for Windows, open WinDbg, > File > Symbol File Path and then paste the following under "Symbol path":

    SRV*c:\symbols*http://msdl.microsoft.com/download/symbols

    then click "OK"

    Then close WinDbg and answer "Yes" to the "Save information for workspace" question.

    Now open WinDbg and drag a minidump file onto the WinDbg GUI or WinDbg > File >  Open Crash Dump > then browse to and select a minidump file.

    To get more information on the crash, type the following at the kd> prompt, and press "Enter":

    !analyze -v

    To get a list of drivers loaded at the time of the crash run the lmtn command at the KD> prompt.

    Hi Rajeev,

    Since automatic configuration fails but manually configuration works (you are in the Skype for Business Hybrid environment), my understanding is that the DNS records haven't been set up correctly. To be more accurate, the issue might be related to the DNS (internal SRV records) can't sign into the Front End or Pool. For more information, you can refer to https://technet.microsoft.com/en-us/library/dn951397.aspx?f=255&MSPPError=-2147217396 (DNS requirements for automatic client sign-in). 

    And, you can perform a test of the Lync (Skype for Business) server remote connectivity test via https://testconnectivity.microsoft.com/. Choose "SFB/Lync"> "Skype for Business Server Remote Connectivity Test". After you get the result, you can self-analyze it or provide the complete information in the Private Message (PM) so that we can help check it.
