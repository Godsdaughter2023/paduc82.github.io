TCP and set supplemental are referring to Network settings.

The error message” Tcpip Event ID 4230” is mainly caused due to the Peer to Peer network connectivity issue.

Are you facing any issue while accessing the network?

Please follow the below steps to find if the issue is with the PNP network.

1. At the command prompt, type
Netstat –no
2. Find the process with a large number of open connections that are not yet established.
These connections are indicated by the TCP state SYN_SENT in the State column of the Active Connections information.
3. Note the process identification number (PID) of the process in the PID column.
4. Press CTRL+ALT+DELETE and then click Task Manager.
5. On the Processes tab, select the processes with the matching PID, and then click End Process.

If you need to select the option to view the PID for processes, on the View menu, click Select Columns, select the PID (Process Identifier) check box, and then click OK.

    vkapartzianis say:

    As far as I can tell (i.e. that's what I did) you have to re-register all counters by hand. Using "lodctr /q" will list all available counters. You'll then have to locate and manually register each one of them. Fortunately, almost all of them are in %windir%\INF, you can try and find the counter's folder using "dir %windir%\INF /a:d".

    So, for example, to re-register the [usbhub] counter:

    cd %windir%\INF\usbhub

    The different subfolders you'll see, using "dir *.ini /s /b" are probably for different system languages. Use either 0000 or your preferred language.

    I found a page that describes the above in more detail for you, see Troubleshooting Windows Server: Corrupt or broken performance counters 

    vkapartzianis say:

    To summarize "lodctr /R" will, I believe, fix all the "default" counters. Then, you have to locate and re-register all "extra" counters from various other programs and services installed on your computer.

    This two-step procedure will work every time. If it doesn't, for a specific counter, then the related files are corrupted. Just disable the counter using "lodctr /D". Chances are, that won't affect anything else besides performance measurements for the related program or service.
