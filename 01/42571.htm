thorstweezers say:

Add-On (resource pack) doesn't work in Minecraft Realms

I'm playing Minecraft PE on Windows10.

I've done the following:

1 - Create a resource pack add-on (modified the creeper resource and texture using BlockBench ... zipped it up and renamed to .mcpack ... placed the .mcpack file in the correct location based on Windows10 instructions here: https://minecraft.net/en-us/addons/)

2 - Create a New World (Creative ... "Require players to accept resource packs to join") and applied the resource pack.

3 - Played the new world.  The creepers were all red and had 3 heads (which is what I changed in BlockBench)

4 - In Minecraft PE for Windows10, I went to edit my Realm.  I then clicked "Replace World" and replaced my Realm world with the one I just created (using these instructions "How do I use Add-Ons on Realms" - https://minecraft.net/en-us/addons/faq/ ).

5 - When I try to play my Realm (from Windows10 or from iPad), the creepers all look normal.  It's not downloading (or applying) my resource pack.

What am I doing wrong?

thorstweezers say:
I figured out the problem.

When creating a Minecraft resource pack, you have to create 2 new UUIDs and put them in the manifest.json file.
I did this but I used Visual Studio to create the UUIDs (by using the "Create GUID" tool).  Visual Studio creates UUIDs with capital letters (ABCDEF).  Minecraft wouldn't download/install the resource pack because of this.  Everything worked fine once I used a different tool to create the UUIDs - a tool that uses lower case letters (abcdef): https://www.uuidgenerator.net/version4

Regardless of the similarity with your issue - fps drop - for the most part, most peoples problems will

Which graphics driver version are you using, or are you allowing Windows Updates to install graphics drivers ?

-  Win10 will automatically install it's choice of drivers when you reboot after uninstalling your current graphics

driver. To prevent this if you want to select the driver version and install yourself, you need to block Windows Updates from installing graphics drivers.

You can use the 'Show/hide' tool in the article below to block automatic driver updates...

How to temporarily prevent a Windows or driver update from reinstalling in Windows 10

- Most times when an error message points at the graphics driver it is only because the graphics driver

was the 'last man standing' before it crashed at the end of an event triggered by something else.

With the exception of a corrupt install, the driver version is very rarely the source of system wide issues.

There are occasions specific issues with some games that get fixed with a driver update, but across the board

fps drops are highly unlikely do to a driver version.

- Uninstalling graphics drivers is straightforward. Generally all you need to do is uninstall the graphics driver via Programs and Features and reboot.

Then install the driver you have selected from the web site for the mfg of your graphics processor

(for AMD or Nvidia), or the mfg of your motherboard or PC ( Intel or AMD 'integrated' graphics ).
