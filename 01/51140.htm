I know this is an old thread but the 50 minute thing sounds like the same problem I had a few years ago with WHS 2011. The problem comes when the network lease runs out. WHS 2011 freaks out when its lease runs out and instead of just renewing it and carrying on, it fails. (GJ Microsoft....)

You have to set your DHCP server to renew your lease after 12 hours or more, to allow for very large restores to have enough time to complete. Techies can do it pretty easy, for non techies you should get your router's make (Netgear, DLink, etc) then google this:

Netgear set dhcp lease time

Replace Netgear with your router's name. Set the lease time for more than 12 hours (43200 seconds). Then restart your computer to be restored and try the process again. WHS has multiple other issues that need to be addressed and can cause failed restores too, but this is the big one for the 50 minute restore problem. Hope that helps someone.

To run these networking commands in a command prompt window
  1. In the search box on the taskbar, type Command prompt, press and hold (or right-click) Command prompt, and then select Run as administrator > Yes.
  2. At the command prompt, run the following commands in the listed order, and then check to see if that fixes your connection problem:
    • Type netsh winsock reset and press Enter.
    • Type netsh int ip reset and press Enter.
    • Type ipconfig /release and press Enter.
    • Type ipconfig /renew and press Enter.
    • Type ipconfig /flushdns and press Enter.

davem121 say:
Hi Dean

In Version 1803 of Windows 10, Microsoft suddenly and without explanation removed the Homegroup functionality from Windows 10


On the Windows 10 device, make the following changes:

Click your Start Button, type Services and hit Enter

When the Services app opens, scroll down to find the following services, then double click each one in turn, set their startup type to Automatic and start the service if necessary

1. Function Discovery Provider Host

2. Function Discovery Resource Publication

3. SSDP Discovery

4. UPnP Device Host

Save that and close the Services app



If you get a Network Credentials popup, you should be able to turn off that requirement . . .

Open Control Panel

Go to Network and Sharing Center

Select Change advanced sharing settings in the left pane

Expand the network you want to disable password protected sharing on by clicking the down arrow on the right of the profile

Select 'Turn off password protected sharing' and then click Save changes.
