¡firedog say:

The file name is a property of the file, not part of its content. So indexing content as well as properties won't help in this case (and I can't imagine it's much use for image files anyway, which are largely binary).

Search will trawl expensively through properties and content looking for a search query string, so it's often better to limit the search scope. It may well be that including file contents in the search, you are excluding the search for the dot ( . ) in the string, because it's usually uninteresting in a text search.

You're looking for files whose filenames share particular attributes. The query to use, then is 

file:*a.jpg

(file is an approved abbreviation for the property known as filename.) This should find any file whose name contains the string a.jpg somewhere in it, not necessarily at the beginning. On my computer, this is what I got in a folder with lots of files:

Any ideas on how to fix this?

weltonsouza say:

I just found the solution to your problem. I edited the channel table in registry to match brazil frequencies. I would attach the reg file here if I can, but below you will find a step by step on how to do it.

 

Step 1: Create an excel file, start the first column with 14 and the second with 473143000. On the next line increase the first cell by 1 and the second by 6000000 and keep increasing until you reach 69 on the first column or 803143000 on the second.

Your excel will have something like that:

14 - 473143000
15 - 479143000 (previous cell plus 6000000)
16 - 485143000 (previous cell plus 6000000)
...

69 - 803143000

Step2: On the next cell convert those values to HEX (use DEC2HEX function).

Your excel will look like this:

14 - 473143000 - 1C3396D8
15 - 479143000 - 1C8F2458
16 - 485143000 - 1CEAB1D8
...
69 - 803143000 - 2FDEFD58

Step 3: Open regedit and fing this key: HKEY_LOCAL_MACHINE\Software\Wow6432Node\Microsoft\TV System Services\TVAutoTune\TS55-0. (WPM uses 32bis by default)

55 is the area code for Brasil and 0 means over the air, so this ONLY WORKS FOR BRAZIL.

 

Now all you have to do is open each key starting from 14 and enter the HEX value from excel. You should also create an reg file to simplify the process if you know how to do it, but the manual step will work too.

 

I hope this helps you or someone else with the same problem.

Additional information:

Computers, which are built by large manufactures that come with Windows Pre-Installed, come with two (2) Product Keys:

 

A)   OEM SLP: This key comes pre-installed in Windows, when it comes from the Factory. This key is geared to work with the special instructions found only on that Manufacturer's computer hardware. So when Windows was installed using the OEM SLP key (at the factory) Windows 7 looks at the motherboard and sees the special instructions and Self-Activates. (that's why you did not need to Activate your computer after you brought it home)

 

B)    COA SLP: This is the Product key that you see on the sticker on the side (or bottom,[ or inside the battery compartment]) of your computer. It is a valid product key, but should only be used in limited situations (such as if the OEM SLP key stops self-activating for whatever reason). The key must be activated by Phone. (Note: All manufacturers that use the OEM SLP system are required by contract to include a Certificate of Authenticity (COA) sticker, that has a COA SLP key, on the computer)

Hello Cliff,

Thank you for posting your query in Microsoft Community.

Do you have Windows 8.1 installation media?

I would suggest you to disconnect all the unnecessary devices from the computer (except the mouse and the keyboard) and then try booting the computer. If you are still not able to boot the computer, then try to do the automatic repair.

Method 1: Perform Automatic Repair

Automatic Repair checks for the errors which is preventing Windows from booting up and will fix it. If this does not happen, then when you may need to run the Automatic Repair by booting the computer to Windows 8.1 installation/recovery disc and then it will fix the issue if the issue with any startup files required for Windows 8.1 to boot up.

If you do not have Windows 8 installation DVD, you may have to create one, boot the computer from it and then perform an Automatic Repair of Windows 8. To create installation media, you may refer to the article given below:

Create installation media for Windows 8.1

After creating the disc, boot the computer from the disc and then try the following method to perform an Automatic Repair of Windows 8.1.

a) Contact the computer manufacturer to know on how to change the boot order of the computer. You will have to change the boot device to CD/DVD drive and then boot from the Windows 8.1 installation disk.

b) In the “Windows Setup” dialog box that appears, set the Time and currency format, Keyboard or input method and the language to install and then click “Next”.

c) Click on “Repair your computer” from the lower left corner.

d) From the “Choose an option screen”, click on “Troubleshoot”.

e) Click “Advanced Options” in the “Troubleshoot” screen.

f) In the “Advanced Options” screen, click “Automatic Repair”.

g) Follow the on-screen instructions to complete the Automatic Repair.

If the issue still exists, try the next method.

Method 2: Perform a System restore

To perform a System Restore, follow these instructions and check:

a) Boot from the Windows 8 installation disk.

b) In the “Windows Setup” dialog box that appears, set the Time and currency format, Keyboard or input method and the language to install and then click “Next”.

c) Click on “Repair your computer” from the lower left corner.

d) From the “Choose an option screen”, click on “Troubleshoot”.

e) Click “Advanced Options” in the “Troubleshoot” screen.

f) In the “Advanced Options” screen, select “System Restore”.

g) Follow the on screen instructions to restore the computer to an earlier state when the computer was working fine.

You should be able to boot the computer now.

Important: When you use System Restore to restore the computer to a previous state, programs and updates that you installed after that date are removed.
