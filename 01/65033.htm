a. user say:

Go to Start / Control Panel / Folder Options / View and uncheck the box for hide extensions of known file types.  You should also check the box for show system and hidden files while you're at it.  This should show you the extensions of all files if that's the problem.  You can then rename the files so the extensions are associated with a programs that will open and play the files (you may need to do some testing first with a few programs to find one that works and an extension that works if they've been changed).  I know of no way to do this for all the files at once - they will have to be renamed individually (yes - I realize the number of files involved but I know of no program in Vista or even a 3rd party program that will change the extensions for a group of files all at once - you can check for yourself at www.tucows.com or www.download.com).

The desktop shortcuts no longer work because either the file type of the source file has changed and the shortcut can't open the file or because the file has been moved and the shortcut can't find the path.  You should be able to check the path by finding the files (whatever they are now called) and seeing if they are in the same path directories as they were before this occurred.  I think it's because the source file no longer functions properly because of the extension (or because of corruption).

Recuva is for recovering lost or deleted files - not files that are still on the system (and you won't solve your problem by deleting them and then using Recuva to retrieve them because it sounds like the problem is with the extensions and Recuva is not going to be able to fix that (it's not a part of its program)).

There's no listing of ALL the possible messages which could be generated by chkdsk that I know of.  Here's an article on how chkdsk works in XP and some of the messages it generates - but nowhere near all and nothing sounding like the one you received (and it essentially works the same in Vista): http://www.isd518.net/District/TechDir/Help/Chkdsk.htm.

Backing up the drive to your internal hard drive isn't going to solve the problem - just move it.  The "damage" to the files is already done so a backup now is not going to solve anything.  The fact that the drive is setup in FAT32 format is not the issue (and probably wasn't the issue in the first place either) and changing it to NTFS now is not going to resolve the problem (though you may want to do that after all of this has been resolved).

I don't want to tell you to cut your losses - that's a rather drastic step and there may still be other options (though I don't know them myself).  You could take the drive to a data recovery expert (NOT Geek Squad or any of those other big store centers) who may be able to recover some, most, or all of the files.  It will be expensive and only you can decide if it's worth the cost.  These people usually charge a set fee and do not offer a guarantee of recovery - so even if they recover nothing you still have to pay.  The decision is yours.

If none of this works, repost your question in the Pictures and Videos Forum at: http://social.answers.microsoft.com/Forums/en-US/vistapictures/threads where people who specialize in video-related problems may be able to assist you with your problem since it involves video files.
