considerate_guy say:

@saardvir,

Unfortunately your project file has become corrupted and won't open. The most common result is that the .wlmp file contains video clips that are listed with a negative duration.

First make a copy of your .wlmp (project) file and only work in the copy. Open that file in Notepad and search through for any VideoClip blocks. Check the inTime and outTime settings. I think you'll find several places where the endTime is set to 0 or the endTime is set before the startTime.

If you edit the endTime values to be larger than the startTime and save that project file copy, you should be able to at least open it again. You'll have to then reset the clips to have the desired end time, but at least that is better than losing all 6 hours of work.

In the future, I suggest saving your project with different names as you go along (perhaps with a timestamp in the name like "My movie 5-14 at 0846". If you get in that habit, then if one of them gets corrupted, you could go back to the version before and at least only lose that amount of work, but not all. And also get in the habit of saving frequently.

I'd also recommend trying a test save of your project as a movie as you go along, just to make sure there aren't going to be issues in saving at the end.
