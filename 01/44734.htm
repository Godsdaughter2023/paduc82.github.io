a. user say:

You need to boot to an Installation or Recovery disk to access the Recovery Environment Options.  You will not be able to do a system repair/upgrade unless you are able to boot normally even if you have the correct disk.  You can't even do this option from Safe Mode.

Do you know when this problem began?  Try a System Restore to a point in timeBEFORE the problem began.  Here's the procedure:http://www.howtogeek.com/howto/windows-vista/using-windows-vista-system-restore/.  Be sure to check the box to show more than 5 days of restore points.  If the first attempt fails, then try an earlier point or two.  NOTE: You will have to re-install any software and updates you installed between now and the restore point, but you can use Windows Update for the updates.  Use the recovery disk if the system prompt doesn’t work. The recovery disk works a bit different from the above procedures but if you follow the prompts from the System Restore menu option with the above information you should be able to restore with no problems.

If the System Restore doesn't work, do a Startup Repair by booting to the genuine Windows Vista Installation Disk (or one you can borrow from ANYONE) or from a Recovery Disk.  Here's the procedure: http://www.bleepingcomputer.com/tutorials/tutorial148.html.  To boot to the CD you may need to change the BIOS to make the CD-drive first in the boot sequence.  To do that, wait for the screen that tells you the F key to push to access the boot menu or boot setup.  Push it quickly. Make the changes, save your work, and exit.  Put the CD in the drive and reboot.  When prompted, push any key to boot from the CD.

If you don't have either disk, you can make a bootable Recovery Disk using http://neosmart.net/blog/2008/windows-vista-recovery-disc-download/ along with burning software like: http://www.snapfiles.com/get/active-isoburner.html and, of course, a blank CD (perhaps made on a different computer).

If that doesn't work, try to boot into safe mode (repeatedly click the F8 key while booting and go to safe mode with networking – or do so from the command prompt on the disk).  Then let's check some of your system files:

Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator (you can skip this step if using the disk).

If using the disk, cd to C:\Windows\System32. Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.  Try to post any corrupted files here so we can see if they can be repaired with good copies from the installation disk (unless there are too many).

While in Command Prompt, type chkdsk /f /r and enter and let it run.  It will want to schedule itself to run at the next restart.   Answer yes and then reboot to run the program.  It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a potential cause.

I hope this helps at least identify the cause and maybe even solve the problem.  If not, post back with results.

Good luck!

a. user say:

That message hints that you are accepting and attempting to install hardware (video, audio, network, etc.) driver updates offered by Windows Update and that is never a good idea.

Had you simply performed a Google search on the message you might read about the cause:

This issue occurs if the following scenarios are true:

  • Windows Update or Microsoft Update tries to install a driver a second time for an additional piece of identical hardware such as a graphics adapter.
  • Windows Update or Microsoft Update tries to update a currently installed driver to an unsuitable version of that driver.

Microsoft does not know what the best drivers are for your system and you should only get them from the hardware manufacturers WWW site - and you should only update hardware drivers when you know it will fix a specific issue and not ever because Microsoft Windows Update thinks your system needs one.

Many is the topic/question here with a subject line like "Windows Update broke my computer!" and many times it is because some hardware driver update has slipped into the mix - and it is the wrong hardware driver update.

Here is what might be too much information and how you might stop, prevent and undo the hardware affliction caused by accepting a driver update from Windows Update.

If the message looks something like this:

If you see and can recognize those hardware driver updates offered you should right click and hide them:

Pretty soon you might have quite a few:

Windows has a way built in to not allow these kinds of updates and I'm not 100% sure it works as advertised but this is a safe adjustment that may prevent a whoops event in the future and how you might undo them if they happen:

Windows 7 has built in ways to disable driver updates from Windows Update that you can read about here:

https://support.microsoft.com/en-us/kb/2500967

I'm not sure if you disable from Control Panel if you still need to or will it make any difference to also disable in Group Policy.

If your Windows version does not support Group Policy Editor you can still enable the Policy via a registry import:

Open notepad to create a new text file by clicking Start, Run and in the box enter:

notepad

Copy and paste the following lines of bold text into the new notepad file:

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\DeviceInstall]

[HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\DeviceInstall\Restrictions]
"DenyUnspecified"=dword:00000001

Save the new text file with a .reg extension to your desktop or a folder with a name you can remember, something like:

disablehwupdates.reg

After saving the file, close notepad.

Right click the new registry import file you created and choose Merge...  to import the file.

Respond in the affirmative to the question... "Are you sure you want to add the information to the registry?"

You should then see a message that the information was successfully entered into the registry.

Alternatively, you could right click the file and choose 'Open With...  Registry Editor' and then import the file.


The Policy adjustment in the registry looks like this:

You should reboot at this point to make sure the updates "stick".

I don't have any driver updates from MS so I don't know how you can look in your Installed Updates to spot any driver updates which may have been installed.

What might be good to know is how to undo a hardware driver update that may have slipped in and afflicted something like video, audio, network.

Two things to know are that Windows always makes a Restore Point before installing updates so you could use that to undo all the updates.

Or you could use Device Manager to roll back any specific driver update like this example from an audio driver update that was afflicted:

It is generally not a good idea to accept hardware driver updates (network, audio, video) offered by Windows Update and if you ever see them it is best to hide them so you never see them again.  Only get hardware driver updates from your hardware manufacturers WWW site.

Just because Microsoft offers you a driver update does not mean you have an out of date or afflicted driver and it certainly doesn't mean you have to install the driver updates MS offers - it is generally best NOT to install driver updates offered by Microsoft.

One rule of thumb is "if it ain't broke don't fix it" so if you are not having a problem with video, audio or network don't accept or install those driver updates offered from Microsoft.  If you are having a problem like that you should only get driver updates from the hardware manufacturer.

Sometimes a driver update will afflict the system in such a way that it just won't boot properly and you might not even be able to get to the desktop to undo the changes.  In that case you need to start continuously tapping the F8 key on your keyboard before Windows starts to load to get to the Advanced Boot Option menu and from there choose Last Known Good Configuration:

LKGC is a copy of the system configuration that was saved the last time the system booted properly.  If that gets the system going again you con then undo or uninstall any adjustments or changes you made from the last update.

I have never seen a situation where MS would send a driver update, have it installed and there not be an option to roll back the driver but maybe it could happen.

In Device Manager make a note and write down what driver version is installed now:

If you think Windows Update is the problem you can look in Control Panel, Windows Update, View Update History (sort by Date Installed) to see what updates were installed yesterday and uninstall any that look related to audio, restart and see how things sound.

Windows Update always makes a Restore Point (the description will say Windows Update) before installing updates so you could restore your system to a date and time prior to the installation of the updates and after a restart see how things sound (check the driver version again to see if it is the same or has it changed).

If the problem was caused by Windows Update, that should fix it, right?

Then you can go back to Windows Update and manually check for updates and see what things are offered and if there is an audio driver update, choose to hide it so it won't be installed.
