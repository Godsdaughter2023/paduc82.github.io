suman_b say:

Hi Kasey,

Thank you for writing to Microsoft Community Forums.

We understand that this caused you inconvenience.

I suggest you to try the following methods and check:

Method 1: Modify Registry.

  1. Right-click Start , then select Run.

  2. Enter regedit in the Open: box and select OK.

  3. Locate and then click the following registry subkey: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control.

  4. In the left pane, right click Control key and select New -> Key.

  5. Name the new sub-key so created as StorageDevicePolicies.

  6. In the right pane of this sub-key i.e. StorageDevicePolicies, right click and select New -> DWORD Value.

  7. Name the newly created DWORD as WriteProtect.

  8. Sometimes DWORD might be found under the sub-key and the DWORD has a Value set to Double-click on the DWORD to modify its Value data.

  9. Change the Value data to 0 from 1.

Method 2: Diskpart command.

  1. In the search box on the taskbar, type Command Prompt, right click on Command Prompt and then select as administrator.

  2. Type Diskpart in Command prompt and press enter.

  3. Type list disk on the next prompt and hit enter (Search for the disk number under the heading Disk ### for the disk that you want to switch off write protection.).

  4. Enter Select disk followed by the disk number and hit enter.

  5. Type attributes disk clear readonly and press enter

  6. Close the command prompt and reboot the PC to take effect

    Suggest you perform Restore from a system restore point.

    How to back up and restore the registry in Windows

john john __ say:

In that case you need to use a different cloning utility as the one you are using now cannot clone the entire disk.  A disk image is an image of the entire disk including the disk structure, it will be everything on the disk, your entire Windows installation with all installed programs and updates at the time the image is taken.  You can clone a disk or take an image and both will be the same, the difference is that a clone is a sector by sector copy of the whole disk including unused sectors whereas an image does not include unused sectors as these contain no data.  In a disaster situation a clone is ready to use when needed, you just have to remove the failed disk and install the clone and boot to the cloned disk.  You can even have the clone already installed in a drive bay and when disaster strikes you can just change the boot order in the BIOS and boot to the clone, to ensure that the clone is not inadvertently written to, erased or infected it can be disabled in the BIOS when not in use.  On the other hand an image is just a file, it has to be restored to a disk then the disk can be booted, restoring the image takes a bit more time than switching to a clone but otherwise images and clones are the same thing, they contain all the information on the disk.  Regardless of which method you use you still have to keep the clone or the image up to date with changes to your Windows installation and user files.  Terabyte's Image for Windows can do images or sector by sector copies of the whole disk.

Casper by Future Systems Solutions (not free) is an easy to use utility to clone and maintain the clone up to date, it can also do images.  There are others out there, many of them freeware, just know that some are better than others so do your research before you chose your solution, select a well proven and well respected product, you don't need a disaster recovery solution that blows up in your face when you need it most!

John

PS.  When you set up a computer it is good practice to keep an image of the clean new updated Windows installation with all your preferred programs installed, the image doesn't need a whole disk to itself and it can be kept in different locations.  In case of a disaster this pristine image of the new installation is often better than an up to date clone as an up to date clone contains accumulated detritus, undocumented or unknown changes made since the installation and may contain hidden or unwanted pests so it may not be the best recovery solution.  With images you can also keep several images in time, not so easily done with clones, lets say you always keep an image one month older than the the installation then if a problem arises that you have difficulty repairing you can always restore the one month old image where you know the system was running properly.  Note that you should also keep regular backups of your user files in a separate location so that they can be restored when needed.
