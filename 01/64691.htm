hal hostetler - mvp-outlook say:

The Null device is a one way ticket to the Twilight Zone...think of it as a waste basket or a doorway to nowhere.  The "device" is a handy place to send things you don't want to appear on the screen or anywhere else.  It's not something you need concern yourself with, as you'll never likely use it, it's mostly a convenience and test facility for programs and programmers.  As a test, you can open a command window and copy a file to the Null device:

Copy (filename).txt Null

The files should copy. If they do, the device is present and functional, the Copy command sent copies to it and these copies vanished into thin air, just like they should.

1. Use the Hardware & Devices Troubleshooter

2. Take actions related to Display Driver
Windows Key+X > Click Device Manager > Expand Display Adapter > Right click on entry below >

A. Rollback the Driver
Click Properties > Go to Driver tab > If there is rollback option, click it

B. Uninstall the Driver
Uninstall > Reboot the computer > The driver will be installed automatically when you restart

C. Update the driver
Click Update Driver Software > Click Browse my computer for driver software > Click Let my pick from a list of device drivers on my computer > Choose the driver > Click Next to update > If this does not work, select Search automatically for the updated driver in place of Browse my computer for driver software > Restart your computer

hairyfool say:

PXE is a mechanism to allow a computer to boot from a network image of the operating system, it was common with "thin client" diskless workstations and is also used to load computers in the 1st place from a network.

The BIOS contains a list of devices to BOOT from in a specific order, Optical, USB and Hard Disk are normal but PXE boot can be one of them. What has happened is for some reason PXE boot has been enabled in the BIOS and is selected before the Hard Disk.

You need to go into BIOS setup mode, how you do this depends on the computer as unfortunately does the menus offered.

Look for the network interface settings, could come under integrated devices or similar. Disable network boot.

If you can't find that look for the Boot Order and rearrange it so that PXE or Network Boot comes after the hard disk (last).

Save the changes and reboot.

if you want a program to be displayed on all virtual desktops - 

with the application open, click the task view icon (or windows key+tab)

to move an open application to a different virtual desktop-

with the application open, click the task view icon (or windows key+tab)

click the centre of the app (not the titlebar) and drag it to the other virtual desktop,

you then need to switch to the other desktop to drag it to the desired monitor.
