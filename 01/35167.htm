Not sure what a "complete restore" is but it looks like from activity in other forums that might mean a restore of the system to the as shipped from the factory condition using a built in recovery partition that might be available on some store bought computers.

That is just about the absolute last resort in problem solving since you will have to download, reinstall all the driver updates, Windows updates, third party software, etc. since your system came out of the box.  Then restore a backup if you have one and hope it all works.  Too much like work.

Using a recovery partition or reinstalling Windows is not "fixing" any problem.  It is giving up.

I don't know why you thought reinstalling C++ would help, the advice from the Microsoft engaged Support Engineer "expert" to "try" things like disabling your video driver does not make sense (notice the words "try" all over the place).  And I am sure not reading through 13 pages of things to try when looking for a solution.

Unless you want to waste a lot of time and get frustrated, you don't need to try things - you need to do things.

If your system is up to date with MS updates and if Explorer is crashing, the problem is often a third party Explorer shell extension.  Sometimes things you install or download will add a shell extension to Explorer and can cause it to crash.  You may not even know that a new shell extension has been installed.

To find out, use ShellExView from NirSoft and you will be able to see all your Explorer shell extensions and then you can start disabling the third party non Microsoft extensions one by one (or all at once) and see if the problem goes away.  If it does, you can start enabling them again one at a time until the problem returns and the last extension you enable is likely the culprit.  Once you figure out which it is you can visit the WWW page of that software and see if there is an update for it.

After running ShellExView you can widen the columns to better see things and then click the Product Name or Company Name column header to sort the extensions and easily see which are from Microsoft and which are not (they will be in kind of a pink color).

Then you can disable them all or one at a time as you isolate the afflicted extension.  Note that the effect is immediate and there is no need to restart the system or login again.  And disabling a shell extension extension does not uninstall it - it is just disabled and you can simply enable it again when you are ready.

DIY Windows 8 Start Screen Tiles – Pin to Start Screen

Technical Level : Basic

Summary

The Windows 8 user interface depends on “Tiles” as the first line of access to the contents of the computer. At a high level, you can consider tiles as the rough equivalent of Windows Shortcuts in previous versions of Windows.  For example, here is a screen clip of Word Shortcuts on the Window 8 desktop

Here are Tiles (Shorcuts) to the the same programs on the Windows 8 Start Screen

In both cases they are just static “shortcuts”, not new active Windows 8 Tiles.

Tiles go beyond simply providing a graphical link to the underlying file. Tiles provide the option of displaying active content from the app on the Start Screen. For example, the “Live Tile” for a weather app can show weather “highlights” that are periodically updated, as shown below.

Up until Windows 8, you could easily create “shortcuts” on the desktop and Start Menu. Apparently it is no longer that easy to create “shortcuts” on the Start Screen. Here is how to do it on the legacy desktop:

  • Right click on desktop
  • Select new
  • Select Shortcut
  • Define the location of the user file or program executable file
  • Click on Next
  • Define the name to display on shortcut
  • Click Finish

Or you could create a shortcut by right clicking and dragging a program or user file from the File Explorer to a new location on the desktop, then releasing the mouse button and selecting “create shortcut”.  This process is documented with pictures in this tip: http://www.7tutorials.com/how-create-shortcuts

True Windows 8 Tiles can come in 2 sizes, the squares and rectangles which are 2 squares “wide”. The program creator has the option of deciding what size of Tile they create. They also have the option of creating both sizes to give the user the option True Windows 8 Tiles can come in 2 sizes, the squares and rectangles which are 2 squares “wide”. The program creator has the option of deciding what size of Tile they create. They also have the option of creating both sizes to give the user the option of how much space they wish to dedicate to an application. Active tiles will not display exactly the same content in the 2 tiles.
Details

Why Create or Tweak Start Screen Tiles

I can think of at least 3 situations where you want to work with the Start Screen tiles:

  1. 1.    You want to create a tile for a “legacy” desktop program
  2. 2.    You want to create a tile for a user file,
  3. 3.    Rename one of the default tiles.

Point 1: Desktop Programs

Unlike MS Office, most 3rd party “Desktop” program installers do not include Code to create Win 8 Tiles. For example, if you install a Portable application, you have to manually create the shortcut on the desktop and the Start Screen.

Point 2: User Files

On the desktop it is trivially easy to create a shortcut to a file you want fast access to.  It takes a little more effort to create a tile to file, but it can be done

Point 3: Rename Tiles

Given the HUGE icons in tiles, thank you very much, but I don’t need to see the full name “Microsoft Office 2013 Word” on the tile, renaming the tile to “Word 2013” gets the message across very clearly.

Create Tiles for Installed Desktop Programs - “Pin to Start”

“Legacy” desktop programs do not create “Active Tiles” on the start screen. The installer program does not include the special code required to generate fully functional Windows 8 Active Tiles.

After you install a legacy / desktop program, the first thing to do is scroll to the far right side of the Start Screen. If you are lucky, all of the old Start Menu shortcuts will appear there. You will probably want to hide some (or most), and move the rest to a more appropriate location on the Start Screen.

If the shortcuts to the application are not on the Start Screen you next want to look in the “All Apps” view of the Start Screen. There are several ways of doing that. You want to display the task bar at the bottom of the screen by doing one of the following actions:

  • Use <CTL><TAB> shortcut key combo
  • Right click on the Start Screen
  • Right click on any existing Tile
  • Hit the “Menu” key on the keyboard

You then want to click on the “All Apps” icon at the far right end of the task bar.

This will display the tiles for all of the programs installed on the computer.

First are the Tiles for new style Windows Apps, then you will see all of the icons for legacy programs and finally many of the built in Windows utilities. These are all of the programs and folders you would normally see in the Start Menu. In The Start Menu you would see folders, now you see low key group names above tiles for programs in that folder. Just like the Start Screen, this screen scrolls from off screen on the right to left.

Sure, you can still create a shortcut to them on the old desktop, and/or you can pin them to the task bar (if you have enough space). But, there does not appear to be an easy way to put them on the Start Screen.

To create a static Tile for a program you have to find the folder that the program has been installed to. You can navigate to it in Windows Explorer. Typically that would be in one of these 2 locations:

C:\Program Files

C:\Program Files (x86)  (for 32 bit apps installed on 64 bit Windows)

Right click on the program file and select “Pin to Start”. The new tile will be displayed at the far right end of all tiles currently on the Start Screen. You can click and drag it to relocate it.

Create Tiles for User Files

If you try to pin a user file, such as a Word document or Excel spreadsheet, you will not be offered the ”Pin to Start” option.  Instead you will see options to “Create Shortcut”, or “Send to / Desktop (create shortcut)”. But nothing to do with the Start Screen.

To get a document tile on the Start Screen you have to “think outside of the box” and combine a couple of “old school” techniques. Many (all?) applications allow you to start the program and automatically load a specified file at the same time. In DOS you used command line parameters. In previous versions of Windows you used the “Target” parameter in the shortcut properties.  Either way, you provide both the executable program name and the file name. This technique still works for Windows 8 Start Screen Tiles. I prefer to “fully qualify”, include the full filename and path, to make it clear where they are located.  Either way, you provide both the executable program name and the file name. This technique still works for Windows 8 Start Screen Tiles. I prefer to “fully qualify”, include the full filename and path, to make it clear where they are located. This is especially important if you have multiple versions of a program installed or multiple files with the same or similar names in various folders. For example, I have 4 versions of Office, so simply specifying WINWORD.EXE would start the current default application, not necessarily the version I wanted to start.

There are 2 ways of approaching this problem.  I find it easier to have 2 File Explorer windows open at the same time.  In one, navigate to the program file, in the other navigate to the user file. In one window you create the base shortcut, in the other you extract the full path and file name.

  • right click on the program file
  • create a shortcut on the desktop for the program
  • right click on the new program shortcut
  • select properties to display the shortcut properties dialog
  • copy the path and file name into Target field after the program name. You will have something that looks like this:

"C:\Program Files (x86)\Microsoft Office\Office14\WINWORD.EXE" "F:\Public\Documents\@@Answers dot com.DOCX "

  • You may also want to the “Start in” location to the document folder.
  • Click on OK to save the changes.

You now have a shortcut on the old style desktop that will start Word 2010 and load the file “@@Answers dot com.docx”.

The final step is to right click on this desktop shortcut and select “Pin to Start”. You now have a new TILE placed at the far right end of the existing tiles.  You can drag and drop it to place it in a more appropriate location.

Create Tiles and Taskbar Shortcuts with Parameters

It is simple enough, working from the desktop side of the OS.  In Windows Explorer

  1. Locate the program or data file
  2. Right click on it
  3. Select “Send To > Desktop (create shortcut)
  4. Right click on the resulting shortcut
  5. Select the Properties option
  6. In the Properties dialog, click on the Shortcut tab
  7. In the Target field, you can add the desired parameters.
    NOTE: you may have to add quotes around the file name if there are spaces anywhere in the name or path
  8. Click on OK or Apply to save changes
  9. Right click on the modified shortcut
  10. Select “Pin to Start” or “Pin to Taskbar”

After you have pinned it, you optionally may delete the shortcut

Delete a Live Tile

Live tiles constantly update summary data displays. Usually the data is retrieved from the internet. If you want to cut down on internet bandwidth use, you can selectively turn off “live tiles”

Right click on the live tile and select  “Turn Live Tile Off.

Rename Tiles on the Start Screen

Tile names, and previously Shortcut names, are often long and cumbersome, ie Microsoft Office 2010 PowerPoint etc.  Well, I KNOW that PowerPoint comes from Microsoft and if I put it in an “Office 2010” Group on the desktop I don’t need that as part of the shortcut/tile name. Often the repetitive stuff is so long the relevant part is wrapped off screen. So I go from a Tile name of “Microsoft Office 2010 PowerPoint” to “PowerPoint 2010” which will probably all be displayed all the time.

Although it is not obvious, the Windows 8 Start Screen still relies on the “old” Start Menu folders. For example the underlying shortcuts for Office applications are located in:

C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Microsoft Office

And

C:\Users\username\AppData\Roaming\Microsoft\Windows\Start Menu\Programs

Any folder inside of these folders is displayed as a Group on the All Apps screen. And all shortcuts in these folders appear as app tiles on the Start Screen.

If you right click on a tile, a command bar is displayed on the bottom of the screen. One of the options is “Open File Location”. 

That will open Windows Explorer in the appropriate location, in the start menu or the program files location.

Now you can rename the existing shortcuts by single clicking to select the shortcut then using the <F2> (rename) shortcut key or right clicking and selecting the “rename”. You are now in edit mode on the name and change it as desired. When you are finished, hit <enter>. You will then see the UAC prompts requesting the Admin Password for the change

Use “Group” to Organize Tiles on the Start Screen

By default, tiles can be organized into groups that use multiples of 2 columns of tiles, ie 2, 4, 6, etc columns.  The tiles automatically flow within the group.  I like to control where the tiles are placed within a group and have not been able to find a way of turning the auto flow off. The work around I use is to go with no more than 2 columns in each group.  This next screen capture has 2 groups in it, the first one is unnamed, the second one is called “Office”:

Note: I have already customized 3 of the Office application tile names

You create new groups simply by slowly sliding a tile until you see a “ghostly” thick white vertical bar. Release the tile and a new group will be formed. Simple.

Name Groups

Mouse down to the bottom of the screen until you see a horizontal scroll bar. At the far right end of the scroll bar you will see a small minus sign (indicated by the Yellow arrow).

When you click on it the start screen will zoom “out”, making all of the tiles small.

Right click on a group, for example you can see a check mark in the upper right corner of the first one and the task bar displays with the Name Group command. You can give the group a name or change the existing name.

Build an “Applications” folder for quick program launching from the Desktop

There's an even quicker way to access all of your apps from the Desktop, create an “Applications” that directly access all of them.

  1. 1.    Copy the full text on the next line:

Applications.{4234d49b-0245-4df3-b780-3893943456e1}

  1. 2.    Right click on the desktop
  2. 3.    Select “Create a new Folder”
  3. 4.    Paste the copied text into the folder name

NOTE: The “name” before the period and braces, ie “Applications” is simply descriptive text. It is the “Registry Key” in the braces that enables the access (this is also true for the “God Mode” folder). So you can change it to anything you want. For example, I changed mine to “All Apps” (remind you of anything, say on the start screen).

On the Desktop and in File Explorer, the folder will be called Applications. Double-click it to see a list of all your applications, including Windows 8 Store apps, traditional Desktop applications and many system applet such as Control Panel. To run an app, double-click it. So, yes, this means you can now start “Metro” apps from the Desktop.

When you are in the “Application” (or “All Apps”) folder, as any other folder there are several views (see Tile view):


Applications Folder View Options and Tile View

The Tiles View (shown above) is somewhat limited. I prefer the “Details” View because the names are sortable (click on the column title “Name”).


Applications folder, Details View

There's still one problem, though: The folder doesn't show up on the Start screen. It's simple to put it there, though. Right-click it on the Desktop or in File Explorer and select "Pin to Start." It's now pinned to the Start screen, though it might not be immediately visible there.

To find it, scroll all the way over to the right, and it'll be there. Click it, and the folder opens with all your apps. If you like, you can move it to a more prominent location on the Start screen by dragging it to the left.

Wordpad is not designed to open PDF files - that's why the content looks like garbled code.

To open PDF files you need a PDF reader program. The most widely used is the free Adobe Reader* program.

If you don't have such a program installed then install one and it should then automatically open PDF files for you when required.

If you already have a PDF reader program then you may need to associate the PDF file type with that program as follows:

1. Click on Start->Default programs->Associate a file type or protocol with a program.

2. Scroll down the list of file types until you see .PDF then select it.

3. Click on Change program button (top-right).

4. Select your PDF reader prgram such as Adobe Reader.

5. Tick the box at the bottom for Always use.... 

6. Hit OK.

 

* To install Adobe Reader:

1. Go to http://get.adobe.com/uk/reader/

2. Untick the optional Google Toolbar if you don't want it then download and install Adobe Reader.

cody c. say: MikeJ2009,

The information below has been edited for Windows 7 from this thread:

There are 2 locations for the desktop in Windows 7.

C:\Users\Public\Desktop is the location for All Users. Any shortcut icon in this folder will appear on every user desktop.

C:\Users\user name\Desktop is the location for each individual user. Any shortcut icon in this folder will only appear on that individual user desktop.

To get the result you want, if a program installation places an icon in the “All users” desktop folder, you can simply move that icon from that folder to your individual desktop folder. i.e You can remove a shortcut from an individual user's desktop on either OS by moving the icon from the All Users (on Vista, Public) Desktop directory into the Desktop directory(s) for the user(s) you want.
