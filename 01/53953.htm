It took about 6 times, each time taking *a long time*, before enough files were repaired for the Usn journal to be read. So, one day, after nearly two weeks, the hard disk was finally visible again, complete with volume label and intact data.

In short: keep running chkdsk and as long as it says it is fixing a few errors (albeit failing on many others), don't give up.

worldtraveler58 say:

Sure there is.  What probably has happened is that he interrupted a file burning session at one time, leaving files ready to be burned to a DVD/CD.  In previous versions of Windows, Windows Explorer would show a directory for the drive even if there were no files in it and no media in the drive.  With Windows 7 for some reason it (usually) won't until you put a blank DVD/CD in the drive. 

 

When you put a blank DVD in the drive a pop-up will give you the choice of burning files to disk or burning a DVD video disk.  A CD gives similar choices.  A directory for the drive will open in Windows Explorer, and you can drag and drop files to it to be burned. As soon as there are files in it a button of sorts will appear in the tool bar of Windows Explorer that says "Burn to Disk."  Click on that to complete the process.

 

The solution to his problem is easy.  Put a blank DVD or CD in the drive and select "Burn files to disk" from the pop-up.  When the directory opens for that drive in Windows Explorer, delete all the files from it.  You can then remove the DVD/CD and save it for later use - it won't be written.  You shouldn't get that baloon any more.
