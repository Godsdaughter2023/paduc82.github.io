Where did the IDs come from, XBox?

a. user say:

Every time a network connects to your NIC, Windows assigns it a name if the network's owner didn't already give their network a name. Without a name, Windows calls them Network 1, Network 2, etc. If the same network disconnects and then re-connects, it gets a new name, so the same network could be called Network 2, Network 13, Network 34. They're just names. There's nothing to delete.

mapped drive tries to connect before wifi connects - Microsoft Community

richhee say:

mapped drive tries to connect before wifi connects

i have a mapped network drive but when windows starts it tries to connect before my wifi connects please help have searched everywhere but everybody talks about batch files and i no nothing about that
frederik long say:

As you guessed, you can solve your problem with a simple batch file. It could look like this:

@echo off
net use * /del /yes
net use /persistent:no

:Loop
ping SomePC | find /i "bytes=" && goto Connected
ping LocalHost -n 3 > nul
goto Loop

:Connected
ping LocalHost -n 3 > nul
net use R: \\SomePC\SomeShare
net use S: \\SomePC\OtherShare

The two lines at the top starting with "net" are required for the first run only. Later on you could delete them.

The first "ping" command checks if SomePC responds. You must replace "SomePC" with the real name of the PC hosting the share. If the PC responds then we continue with the line starting with ":Connected". Otherwise we do a ping to LocalHost, which causes the batch file to pause for a few seconds.

When connected we wait a few more seconds to ensure that the network is really up and running. We then map a couple of shares. You must, of course, use real names for the shares!

Where to from here? Well, here is the recipe:
1. Mark and copy the above code.
2. Click Start / Run, then type the three letters cmd and press Enter. You are now in a Command Prompt.
3. Type the following commands and press Enter after each:
    cd  /d  "%UserProfile%\Start Menu\Programs\Startup"
    notepad netlogon.bat
    (allow the file to be created)
4. Past the code into this notepad session.
5. Save and close the file.
6. Type the word netlogon and press Enter. This will execute the program.
7. Any error messages? If yes, fix them!
8. Reopen the file with notepad and delete the first two "net" lines if you wish.
9. Reboot your PC and check if the drives are mapped successfully. If not then you need to go through Steps 2 . . 5 again and make whatever adjustments are necessary.

If you run Windows 7 then the instruction in Step 3 must read:
    cd  /d  "%UserProfile%\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup"

seth nowai say:

Good news, I fixed my problem.

I actually did all those steps and nothing worked. Then I decided to fully commit and do fresh install. Once again, not working. Then I once again started playing in device manager and nothing...

And then I decided to do something random. And part of random stuff was to go to Network Connections, TCP/IPv4 and manually configure it. And there you go everything went back to life and I am online now.

In case someone else has this issues:

- hit search, magnifying glass

- search for "ncpa.cpl"

- right click your connection, in my case that was "Local Area Connection 2"

- choose Properties

- in list, select "Internet Protocol Version 4(TCP/IPv4)" and hit Properties below

- choose "Use following IP address".

- fill in following settings:

IP address: 192.168.1.100

Subnet mask: 255.255.255.0

Default gateway: 192.168.1.1

- Windows will also require you to set DNS server. You can get those from ISP or just use Google DNS. In latter case set:

Prefered DNS server: 8.8.8.8

Alternative DNS server: 8.8.4.4

- hit ok, then close and close everything up, you are done.

At least for me, above worked. Modem configuration page popped up about 10 seconds after. It also seems to persist if I unplug USB modem.

andre da costa say:

You can make the text and other items, such as icons, on your screen easier to see by making them larger. You do this by increasing the dots per inch (DPI) scale. You can also decrease the DPI scale to make text and other items on your screen smaller, so that more information fits on the screen.

Change the size of the text on your screen

  1. Open Personalization by clicking the Start button , clicking Control Panel, clicking Appearance and Personalization, and then clicking Personalization.

  2. In the left pane, click Adjust font size (DPI).  If you are prompted for an administrator password or confirmation, type the password or provide confirmation.

  3. In the DPI Scaling dialog box, do one of the following:

    • To increase the size of text and other items on the screen, click Larger scale (120 DPI)–make text more readable, and then click OK.

    • To decrease the size of text and other items on the screen, click Default scale (96 DPI)–fit more information, and then click OK.

    To see the changes, close all of your programs, and then restart Windows.

Hide all
To create a custom DPI scale

  1. Open Personalization by clicking the Start button , clicking Control Panel, clicking Appearance and Personalization, and then clicking Personalization.

  2. In the left pane, click Adjust font size (DPI).  If you are prompted for an administrator password or confirmation, type the password or provide confirmation.

  3. In the DPI Scaling dialog box, click Custom DPI.

    Customize the size of the text that displays

  4. In the Scale to this percentage of normal size list, enter the percentage you want, and then click OK.

    To see the changes, close all of your programs, and then restart Windows.

To make text and on-screen items clearer in programs that aren't designed for high DPI

If you set the DPI higher than 96, and you are running Windows Aero (the premium visual experience of Windows Vista), the text and other items on the screen might appear blurry in some programs that are not designed for high–DPI display in this version of Windows. You can avoid this issue by using Windows XP–style DPI scaling for these programs.

Some programs display better using Windows XP style of DPI scaling

  1. Open Personalization by clicking the Start button , clicking Control Panel, clicking Appearance and Personalization, and then clicking Personalization.

  2. In the left pane, click Adjust font size (DPI).  If you are prompted for an administrator password or confirmation, type the password or provide confirmation.

  3. In the DPI Scaling dialog box, click Custom DPI.

  4. Select the Use Windows XP style DPI scaling check box, and then click OK.

The solution I found was to go into the Group Policy Editor.  Click on Computer Configuration -> Administrative Templates.  Then right-click on Administrative Templates and select Filter Options.  Under Filter Options, click on enable keyword filters.  In Filter for word(s), type in clipboard.  Then press OK.  Expand Administrative Templates and click on All Settings.  In the right hand pane, you will see "Allow Clipboard History" and "Allow Clipboard synchronization across devices".  Click on those items and enable them.  You must then sign-off and log back in.  This worked for me.
