Hi Dilteg. I'm Greg, an installation specialist and 9 year Windows MVP, here to help you.

You can delete the 450mb Recovery partition which only includes the onboard repair console that pops up if Windows wont' start (which was on C in older Windows editions). This can be replaced by creating a bootable Recover Drive or Installation media which have the exact same tools:
http://windows.microsoft.com/en-us/windows-10/c...
http://windows.microsoft.com/en-us/windows-10/m...

Then you can move the data off of E and F, delete them as well and extend C over them to gather all the space into C.

But the best recovery plan is to save externally a System Image backup of C so that if it ever becomes irreparable you'd boot Rescue media to apply the saved image in 20 minutes and be back up and running. If you do this then you might want to separate out the User folders and other data storage to a separate data partition to keep the System Image smaller for imaging storage purposes. Then you'd restore Windows and Apps on C and the data is safe, current and waiting on the data partition, backed up also to the external drive where you should store the image.

Here's the best imaging app and how to use it: https://www.howtogeek.com/howto/7363/macrium-re...

Here's how to move your User folders (Documents, Pictures, etc) off of C onto the Data partition: http://www.dummies.com/computers/operating-syst...

To do this best, I'd estimate the size needed for your data partition, then extend C into the space you free up leaving enough space after it to create and format a data partition there, or you could just leave F if it is big enough and extend C up to it.

If you don't want to delete the Recovery partition then you could still move the User folders to one of the data partitions to free up space on C, store the image and other files on the other data partition.

Plenty for you to think over and ask back any questions.

I hope this helps. Feel free to ask back any questions and let us know how it goes. I will keep working with you until it's resolved.

1. Open Network Diagnostics by right-clicking the network icon in the notification area, and then clicking Diagnose and repair.
