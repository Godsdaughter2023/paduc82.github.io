Hello ncbpoppa,

 

I'm afraid I have to disagree with Satish SV. The difficulty arises in that Microsoft use the term 'devices' more than the word PC. As far as I am aware a dual boot is actually two separate devices even though they are on the same machine. In Microsoft's terminology that basically means you are using the same operating system on 'two' machine, not one, even though you will only be using one machine at a time.

 

As an example I run Windows XP, Vista, and Windows 7 on virtual machine software. Each and every operating system has its own product key. That product key is unique to the VM and not used on any other machine. If I wanted to use, say, a copy of Windows 7 on another PC then I would buy another copy. I would not try installing the copy I already have on another machine without first removing it from my VM software.

 

Doing what you are proposing will also flag a message upon attempted activation which would say ' product key already in use', thus activation would be refused. In my opinion you would be better off simply scrapping the old drive which is failing and then installing the operating system onto the new drive. If you have an OEM copy (one that came pre-installed with the PC) then that copy is locked to that machine and you would need to replace the hard drive with same make, specs etc., as the one you are removing.

john_barnett say:

Hello ncbpoppa,

 

An OEM (Original Equipment Manufacturer) copy of Windows is supplied to the PC manufacturer at a highly discounted price. Because of this Microsoft do not support the product. If you have any problems you have to sort them out with the PC Manufacturer. Also, as a result of this arrangement, the OEM copy is tied to the first machine it is installed on. In other words you cannot install it onto another machine. Also, if anything goes wrong with the OEM machine the replaced part has to be the same make, model etc as the original otherwise the product key will not work. This particularly relates to motherboards, and, I think, graphic cards. I'm not too sure about hard drives, but if you have replaced the hard drive before on this same machine without any issue then maybe the specs option doesn't apply. By replacing the hard drive you will find that you will probably find that you will need to re-activate your copy of Windows. This will probably need to be done via telephone activation.

 

A full retail boxed version is a different beast. You would buy this yourself from a computer store and the install the operating system yourself on your PC. With a full retail version, should your PC die, you can simply install the retail version onto another computer without having to worry about the end user licence agreement. I personally never buy upgrades I always buy a retail version simply because I have more control.

 

Upgrades, of course are different again in the sense that you must already have a qualifying version of Windows on your PC before you can use the upgrade. With previous incarnations of Windows you could actually do a clean install and then when asked for a qualifying copy of Windows input the product key of say Windows Vista or Windows 7. With Windows 8 you cannot do a clean install unless you buy a copy of the System builder version of Windows 8. If you have an upgrade copy and you wanted to do a clean install you would need to first install, say Windows Vista or windows 7, then upgrade that operating system using your Windows 8 upgrade disc.

 

This explanation has probably confused matters even more but, as the End User Licence agreement is a legally binding document one has to adhere to its rules and regulations.

To add to John's excellent advice here is what you can replace/not replace in Hardware.

 

here is what Microsoft has to say about replacing Motherboards in preinstalled OEM Computers (eMachines, Gateway, Dell, etc):

 

http://oem.microsoft.com/script/contentpage.aspx?pageid=552846#faq1

Q. Can a PC with an OEM Windows operating system have its motherboard upgraded and keep the same license? What if it was replaced because it was defective?

A. Generally, an end user can upgrade or replace all of the hardware components on a computer—except the motherboard—and still retain the license for the original Microsoft OEM operating system software. If the motherboard is upgraded or replaced for reasons other than a defect, then a new computer has been created. Microsoft OEM operating system software cannot be transferred to the new computer, and the license of new operating system software is required. If the motherboard is replaced because it is defective, you do not need to acquire a new operating system license for the PC as long as the replacement motherboard is the same make/model or the same manufacturer's replacement/equivalent, as defined by the manufacturer's warranty.

What Is the Windows.old Folder?

This folder is created when you upgrade from one version of Windows to another, starting with Windows Vista. The Windows.old folder contains all the files and data from your previous Windows installation. You can use it to restore your system to the old version of Windows if you don’t like the new version. If you’re looking for a specific file that wasn’t properly copied to your new Windows installation, you also could dig into the Windows.old folder and find it.

Essentially, the Windows.old folder just contains the old Windows system. From the Windows system files to your installed programs and each user account’s settings and files, it’s all here. The new version of Windows just keeps it around in case you’d like to go back to that older version of Windows or in case you need to dig in and find a file.

But, don’t wait too long—Windows will automatically delete the Windows.old folder to free up space after a month.

Csrss.exe is a vital component of the Microsoft windows operating system. It stands for Microsoft Client/Server Runtime Server Subsystem. This process is responsible for dealing with the majority of the interface of windows. Console windows and threads are managed by this process.

 

The csrss.exe file is a useful component of windows, however various things can happen which will result in a csrss.exe error.

 

By default csrss.exe might use 700k to 800k of memory. This can be seen usingtask manager.
