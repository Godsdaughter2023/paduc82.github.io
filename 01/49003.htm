Hard drive's firmware can mark bad sectors.  It can also allocate reserved sectors as replacements.  In fact, my hard drive's firmware has already marked the problem sector as bad, and just waiting to replace it.  I know this because the SMART attribute "current pending sector" is 1, meaning 1 sector pending reallocation.  I believe this is what you want to achieve by running a manufacturer's utility.

Since low-level manufacturer utility or firmware only operate on the disk level, not the file system level, they will not modify NTFS and let Windows know there is a bad sector on the disk.  (If you know a low-level utility that can work on NTFS, do let me know.)  This is where chkdsk comes in.  If chkdsk detects a used cluster has a bad sector, it will try to recover data, allocate a new cluster to replace the bad, and mark the bad cluster unusable.

robertjwin say:

Here's how I understand it. The drive's controller can mark bad sectors and also remap them without the knowledge of the OS running on it. If during the remap all the data isn't copied properly to the spare sector then of course chkdsk /f will detect file system (FS) corruption and fix it using the backup file table or error correction or whatever. chkdsk /r can also hit upon a bad sector that the firmware hasn't detected yet, and this will make it add to the OS' own bad cluster table that tells the file system not to use that cluster in future. The job of marking and swapping the bad sector on disk is still the controller's.

The Current Pending Sector count (counter 197) is just the number of sectors currently marked as unstable (not marked as bad like you wrote) and which might be remapped on the next failed write attempt (see here for a more detailed description). The counter is decremented once the sectors have been successfully remapped. So it seems you have an unstable sector that hasn't been remapped yet by the controller, but IMO running the manufacturer's utility should hopefully stress it enough that it should get marked and remapped so that the Pending Sector count falls back to 0. Once that is done either chkdsk /f will detect no FS issues due to a successful copy of the data and you'll be good to go, or it will detect an issue and attempt to repair the FS.

Now perhaps running chkdsk /r again will also cause the unstable sector to error out and get marked/remapped, but if you are going to test the drive anyway I recommend the manufacturer's utility over chkdsk (that you've run once as it is). Your choice though.

a. user say:

Bad sectors are rare nowadays, but they happen. The operating system will mark the sector as bad and won't use it. That's all. Nothing to sweat about.

If it's a logical error, the disk manufacturer's own diagnostic software can usually repair the bad sector. If it's a physical error there isn't anything you can do about it. In either case, the OS will mark the sector as bad and not use it. Nothing to worry about.

On the other hand, if you're getting a lot of bad sectors then it's time for a warranty replacement.

lemp say:

When you checked the status of your mic, did you look both in Device Manager and in Control Panel > Sound?

You can get to Control Panel > Sound quicker by just right-clicking on the Volume icon (speaker) in the Notification Area next to the time display.

On the "Recording Devices" tab of the Control Panel Sound applet (this is where you saw that your mic was set to be the default device), select the mic and click the "Properties" button.

On the General tab of the Microphone Properties, you should see that the device is enabled.  You can check the "Properties" button on that page to see if there is a driver issue, but that should have been apparent in Device Manager if a problem existed.

Select the "Levels" tab.  Is the microphone level at 100% (slider all the way to the right)?  Is the microphone muted (button next to volume slider shows a red )?

You can also run the audio recording troubleshooter.  Click Start and type audio in the Search box.  This also may reveal another control program (e.g., the Realtek audio manager) that you can check to see if there is a setting that is overriding the Windows setting.

Q: How can I restore my sound card settings?
