Restore points have always been temporary - they have a limited life, after which they are deleted by the operating system. This is intentional. In addition, following a feature update, all previously existing restore points are deleted. This too is intentional.

jason c beck say: None of these are relevant to the question I asked.

So here is what happened. Unlike Desktop apps, which generally stay put after you open them, with Modern Apps, you open them, and then move them to the monitor you want. Well then you click the Start Button and are taken to the Start Screen. You can either have the task bar and Start Button on all monitors or just the primary one. But either way, clicking the Start Screen tends to take you away from your modern apps. If you press the Windows key to go back, whatever monitor you opened the Start Screen is the one that the Modern App opens on. It shouldn't do that. It should stay on whatever monitor you moved it to, and never move from that monitor unless you drag it to another monitor.

jason c beck say: This wasn't exactly what I was seeing the other day, but here is a simple enough example:
  • Open any modern app (it will open on whatever monitor you have the Start Screen on.
  • Move it to any other monitor.
  • Close that app.
  • Click to the Start Screen and open the modern app (it will open on whatever monitor you have the Start Screen on.)
Desired behavior
  • When you re-open a modern app that you've previously placed on a specific monitor, it should be on that monitor, in the same configuration as before. If it's using a split-screen, it should use the same split-screen. If it's full-screen, it should still be full-screen, on the last monitor where you used the app.
Even if I have the Mail app open on a secondary monitor, split screen or full, and click the Mail icon from the Start Screen, it opens on the primary monitor.

(Your solution of changing the primary monitor still means all apps open on the primary monitor. A different one, and not the one where you moved the Modern app. The correct implementation is for applications to remember where you had them open, and put them back there. Most desktop applications get this right.)
