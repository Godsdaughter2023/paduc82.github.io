Hi Daniel,

Thank you for writing to Microsoft Community forum.

A file with 000 file extension is most likely an Indexing Service Data file used to store file locations so that Windows can perform file searches. A 000 file can also be a DoubleSpace Compressed (DriveSpace) file which is a compression utility used on the old MS-DOS OS. These file extensions are also utilized by a data format as part of the Windows CE installation. Still, a few applications append the .000 extension to a file for data backup or archive part files.

The files seen on the screenshot are not compressed files, instead contain separate components of a file. Any 000 files found as part of a backup set or archive, especially with other numbered extensions like 001, 002, etc., are meant to be used together and combined,  and possible uncompressed, with whatever backup software or archiving utility that created them.  We need to get all those numbered extensions together, combine them with whatever did the splitting, to access the real file. For these files to work, they must be joined back together.

There are a few third party tools available which can combine these extensions. You may use your favorite search engine to search for a tool that can split files or put .000, .001, etc., files into the original state. Install a tool that supports this, combine the files and check if you can access.

If the backup created was a complete system image back up, then I would suggest you to refer this article to refer the section 'Restore' and check if it is helpful to you.

Note:
