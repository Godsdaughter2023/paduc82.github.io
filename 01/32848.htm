Hello Aunt,

Thank you for writing to Microsoft Vista Community Forums.

Note in Windows Vista the sound recorder is not limited to any time limit. You can record to any time length.

If you are using earlier versions of Windows like Windows XP, Windows 98, the time is limited to 60 seconds to recording.

However, you may increase the Sound Recorder time using the following procedures. 

  1. Click Start, Programs, Accessories, Entertainment, and then click on Sound Recorder.
  2. Click the record button and let the recording run for 60 seconds.
  3. After Recorder has stopped, click File, and then Save As, name the file BLANK.WAV.
  4. Increase the recording size. From the Edit menu choose Insert File, select BLANK.WAV and then click Open.
  5. This increases the maximum recording time of Sound Recorder by 60 seconds. Repeat step 4 once for each additional minute desired. For example, for a 5-minute recording time, insert BLANK.WAV four times.

Access the link below to know more about how to Increase the Maximum Recording Time in the Sound Recorder Utility:
http://support.microsoft.com/kb/82215

I have used Windows Update to install this particular update.

So, I have made the following, and the issue didn't (yet? hope not) reproduce.

I have set up a "clean boot" in advance (disable all the services in msconfig + disabled LifeExp startup in Process Manager), and then tried to install this update using Windows Update yet again. After the reboot the issue didn't reproduce, so I started to enable the services one by one.

The list of services was as follows:

  • Intel Content Protection HECI Service
  • Intel HD Graphics Control Panel Service
  • Intel Capability Licensing Service Interface
  • Intel Capability Licensing Service TCP IP Interface
  • Intel Dynamic Application Loader Host Interface Service
  • Intel Management and Security Application Local
  • NVIDIA Display Driver Service

After enabling "Intel Content Protection HECI Service", the issue didn't reproduce. I found out that this service is responsible for some Blu-Ray etc. content playback using integrated Intel video, so I decided to remove this service (cmd.exe -> sc delete).

Then I enabled "Intel HD Graphics Control Panel Service" and reboot. The issue didn't reproduce. Though I decided to disable integrated video in UEFI (BIOS) Setup, and also removed this service.

I enabled all the services and returned to the "normal boot", and still couldn't reproduce the issue. Though I found out that all the other Intel service are related to Intel AMT technology that is definitely not needed on the home computer, so I removed all the other Intel services. I also found out that NVIDIA service makes it easier to switch between adapters, and since I have just one NVIDIA video adapter, I decided to disable this service (it was set to auto start).

So now I don't (yet) have this issue, though regarding that it first appeared accidentally (and made my previous SSD die), I can't be 100% sure it won't be back.
