administative events log - Microsoft Community

richardkeierleber say:

administative events log

I have over 19,500 administrative events logged since I purchased my HP Pavillion Windows 8 computer.  How can I clear the event log?

ronnie vernon say:

I have over 19,500 administrative events logged since I purchased my HP Pavillion Windows 8 computer.  How can I clear the event log?


Hi


The Administrative Events Log is a compilation of events that include the Administrative Level Events from most of the other Winsows Logs in the event viewer. This is why there are usually a large number of events in that log.

 

You cannot Clear the Administrative Events Log, since it collects those events from so many other logs.

 

You can clear any of the individual Windows Events Logs, which will remove those events from the Administrative Events Log.

 

When you selet an Administrative Event, look at the General Tab in the lower section and it will show  Log Name: <path to the log where the event was generated>.

 

You can use the left side Navigation Pane to go that log. Right Click the log and select the Clear Log option. In the popup window, select Clear.

This will clear that log and remove those events from the Administrative Events Log.

 

Before you consider doing this, remember that the event logs can be important for a support tech who may be trying to track down a gnarly problem.

 

Let us know if you have more questions.

If you took your 80GB drive and made it a slave, that implies (to me) that you had some kind of master HDD set up in the system too, and was XP installed on the master drive.

Now you have a master with XP installed and running and an 80GB drive as the slave, right?

If you then use XP and quick format the 80GB slave drive, that wipes it out.  Microsoft says:

If you choose the Quick format option, format removes files from the partition, but does not scan the disk for bad sectors. Only use this option if your hard disk has been previously formatted and you are sure that your hard disk is not damaged.

You cannot undo a quick format.

If you then want to install XP on the 80GB drive, you need to unhook your master drive and configure the clean 80GB drive as the master, boot on your genuine bootable XP installation CD and then install XP on the clean 80GB drive that is now the master.

Do you have a genuine bootable XP installation CD?

You can't leave the clean 80GB drive in as the slave and expect to boot on it if XP has not been installed on it yet.  Of course there will be no NTLDR file on it since you just quick formatted it.
