ken blake say: In reply to deleted message

On Wed, 13 Oct 2010 12:20:38 +0000, BossDweebe wrote:

I've never done an 'upgrade' so I don't know what happened to your files, but when you install Windows you will be offered an option to  reformat and partition the HDD. This wipes  the HDD and makes for the most stable Windows installation. A fresh install is more trouble but doing an 'upgrade' (as in installing over the old)  is not the best option as you are building on an old foundation.

Doing an upgrade that changes the "bitness" of Windows (from 32-bit to
64-bit, or vice-versa) is not possible. A clean installation is the
only way to get from one to the other.

However, regarding upgrading versus clean installing in general,
although many people will tell you that formatting and installing
cleanly is the best way to go, I disagree. Unlike with previous
versions of Windows, when doing an upgrade was often a mistake, an
upgrade to Windows XP or later replaces almost everything, and usually
works very well. The only real exception is when you are having
problems; in that situation, an upgrade often worsens problems, rather
than solving them.

My recommendation is to at least try the upgrade, since it's much
easier than a clean installation. You can always change your mind and
reinstall cleanly if problems develop.

However, don't assume that doing an upgrade relieves you of the need
to backup your data, etc. before beginning. Before starting to
upgrade, it's always prudent to recognize that things like a sudden
power loss can occur in the middle of it and cause the loss of
everything. For that reason you should make sure you have backups and
anything else you need to reinstall if the worst happens.

rcwhite say:

﻿Hi, Matthew.

You made the right choice, Custom.  But you apparently chose to NOT reformat the partition where you installed Win7 x64.  A reformat is not strictly necessary for a Custom install, but it is almost always a very good idea.

A reformat would have wiped out 32-bit Win7, along with ALL your program and data files on that partition.  Since ALL your applications will need to be re-installed in 64-bit Win7 anyhow, so that their Setup processes can write their entries into the 64-bit Win7 Registry, it is pointless to keep their files.  Just be sure that you have their CDs or other media to re-install them.

In 32-bit Windows, all applications are normally installed into C:\Program Files.  But 64-bit Windows uses that folder only for 64-bit applications! It puts all 32-bit apps into a new folder called C:\Program Files (x86); the (x86) refers to the x86 family of CPUs from Intel - 8086, 80286, 80486, etc. Windows x64 uses this system to associate 32-bit and 64-bit apps with the proper dlls, drivers and other support files.

Your preserved data files can still be used after the apps are re-installed; you'll just have to run the apps and browse to the locations of the data. But all the reinstallations will have scrambled the disk locations, making it less efficient.  It's much better to use WET (Windows Easy Transfer - included with every Windows package) to save your tweaks and data to a safe location (USB flash drive, external hard disk, separate partition, etc.), then let WET restore them after your new installation is up and running.

At this stage, Matthew, I would save all the data in a location that will be accessible later, then re-install Win7 x64 from scratch, INCLUDING REFORMATTING of the partition.

They don't download automatically.
