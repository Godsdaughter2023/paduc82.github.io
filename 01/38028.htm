Anytime the Windows Update agent is turned on, Windows Update will take care of updating itself.  Windows Update is considered “on” when anything other than “Never check for updates” or “Turn off Automatic Updates” is selected.  What this means is that if you’ve selected “install updates automatically,” the update to the Windows Update client will install automatically.  You may also notice that Windows Update will also automatically download and install updates necessary for the Windows Update client to function properly if you’ve chosen a different setting, like setting 2 (“Automatically Download Updates”) or setting 3 (“Automatically Check for Updates”).  [emphasis mine]  This is done in order to ensure that a system will continue to notify the user about available updates. 
 
The Windows Update client does not download or install infrastructure updates if the user has selected setting 4 (“Off”).  In other words, if a [Windows 7 or] Windows Vista user selects “Never check for updates,” (or if a Windows XP user selects “Turn off Automatic Updates”), the user will not receive Windows Update client infrastructure updates unless they choose to manually install it from Windows Update.

jo mari gianguintu say:

That's easy. If you were able to change it to Open before and upon waking up, you find it changed again to moderate. It only means one thing. You have a dynamic IP so no matter how many times you do port forwarding on your router, it will change back every single time you reset your router.

First step, set a static local IP for your Xbox One.

1. Scroll left from Home to open the guide.

2. Select Settings.

3. Select All Settings.

4. Select Network.

5. Select Network settings. Select Advanced Settings.

6. Select IP settings.

7. Select Manual.

8. Enter the unique IP address that you made up, and then select Enter.

9. Enter your computer’s Subnet mask, and then select Enter.

10. Enter your computer’s default Gateway, and then select Enter.

11. Select DNS Settings.

12. Select Manual.

13. Enter the Primary DNS setting that you found on the computer, and then select Enter.

14. Enter the Secondary DNS setting that you found on the computer (if there was one) and then select Enter.

ONLY after setting a static IP on your console, do you open ports on your router or gateway.

Now, just on the off chance that it's still not working,

Just, enable perimeter network (also known as DMZ) functionality on your router

               Note It is critical that you use a static IP address on the console when you set up a perimeter network. To set a static IP address, use your router’s DHCP reservation function, when available. If this is not available, you will have to set up manual IP settings on your Xbox. You will only be able to place one Xbox console into the perimeter network. Therefore, if you have multiple consoles, this solution will only resolve the issue for one Xbox. (the static IP referred here is not the Static WAN IP from ISP, it is the local static IP)

               - manually configure console IP settings

               - access router's administration panel to enable DMZ and provide the local static IP address that was assigned for the console

               - in layman, we need to tell the router to open the ports for the console, the static IP will act as its identifier

And yeah. these steps are on Xbox Support's official web support link: support.xbox.com/.../nat-error-solution

I just find that support link lacking so I added specific details for people who are not so techie.
