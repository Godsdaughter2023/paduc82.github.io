a. user say:

Hi,

Thank you for posting your query on Microsoft Community. Let me assist you.

I suggest you to follow the below steps to export a certificate with a private key

1. Open the Certificates console for the user, computer, or service you want to manage.

2. In the console pane, select the certificate store and container holding the certificate that you want to export.

3. In the details pane, click the certificate you want to export.

4. On the Action menu, point to All Tasks, and then click Export.

5. In the Certificate Export Wizard, click Yes, export the private key. (This option will appear only if the private key is marked as exportable and you have access to the private key.)

6. Under Export File Format, do one or all of the following, and then click Next.

  •  To include all certificates in the certification path, select the Include all certificates in the certification path if possible check box.
  • To enable strong protection, select the Enable strong protection (requires IE 5.0, NT 4.0 SP4 or above) check box.
  • To delete the private key if the export is successful, select the Delete the private key if the export is successful check box.

7.In Password, type a password to encrypt the private key you are exporting. In Confirm password, type the same password again, and then click Next.

8. In File name, type a file name and path for the PKCS #12 file that will store the exported certificate and private key, click Next, and then click Finish.

Hope this information is helpful. Please do let us know if you need further assistance, we’ll be glad to assist you.

So main question is: How do I rename the sub-folder named 'Larry'?  Thanks very much to all who may try or be able to help with this!

hairyfool say:

This is your personal user folder and takes its name from your username.

 

When you create a user on the computer you set a user name, when you then log on as that user the 1st time it builds your profile copying the contents of the Default user profile files into the new folder based on the username. This folder is tied to your security Id and cannot be modified as everything about you points to this folder or subfolders.

 

So you create a user called Fred which just creates the security identity for him. You log on as Fred so the folder structure for Fred is created (which is why this can take a few seconds). You then want to change the username to Freddie which you can do but the folder for Freddie will not be changed and remain Fred.

 

The only way to remove a folder from Documents and Settings is to delete the user that owns it. Someone may have a "trick" to make a change but this is how Windows user profiles work.

cmdrkeene say:

Can you press Ctrl+Shift+Escape together and in the list, see which process is using the most CPU and Disk activity?  You can click the column heading up top until the highest ones are listed first to make it easy to identify.

The processes that are using the most of these could be causing the bottleneck and slow performance, especially if there isn't a lot of Memory available, because then it has to "swap" out to the swap file on the drive, leading to even more bottlenecks.

If the programs using the most resources aren't items you're using, it's likely one of the following:

  • Virus scanner running the "first time scan" after installing your new upgrade
  • Search indexing system building up a new index after the upgrade
  • OneDrive/GoogleDrive/OtherOnlineCloudStorageThings re-scanning for any possible changes after the upgrade
  • Windows "deferred tasks" as Gabel Aul calls them (Microsoft General Manager). These are post-install cleanup tasks and other optimizations (you may be familiar with that LONG wait after upgrading android where it re-optimizes all the apps).

If any of these are busy doing their work, it'll really impact performance. And if more than one happens to be going at once, it really basically drags you down to a crawl.  I've honestly advised some users to just set their power options to not go to sleep mode, and leave it on overnight to finish all the things like that.

But...

But that's of course not always the case.  If you don't think these above are what's impacting you, or you see other processes causing resource utilization (using CPU/Disk/Memory) that aren't these things, let me know and we'll see if we can identify them and what's causing the freeze.

PS. I'd definitely advise checking for updates from Windows Update.

In every install I've done so far, things are quiet and cool once these finally finish but it can take a full day of powered-up idle time, especially for that index crawler.  And if it doesn't finish, it's likely hung up on some XML file (iTunes tends to cause this), but let me know and we can dig into that, it's actually easy to fix by just telling the indexer to stop looking inside XML files.

cmdrkeene say:

You can check that indexing status (type "index" into the search box and it should be the first result). If it says "indexing complete" at the top, then you can rule out it as a culprit. If it's not complete but not making progress, it could be stuck parsing a file.

I'd recommend turning off all those background things you can, but I fully realize many (maybe all in your case) provide a service that you want to be present. I myself disabled chrome from continuing to run in the background and from pre-downloading pages I might visit, but I have team viewer, two flavors of OneDrive (business and consumer), two flavors of Skype (business and consumer), and even the MS Band fitness tracker's sync utility that is doing nothing.  Full Office 2016 OneNote and Outlook are running, as is tabs in Edge, Groove, and an active Team Viewer session as I type this. Also a wireless mouse's dongle is in the USB slot right now. Keyboard backlight at the brightest setting).

(Sorry, I didn't really start that paragraph intending to become a discussion about me, but I realize it might help to at least explain what I observe on my SP3).

Right now I'm at 18% and have an estimated 1:20 remaining (I'm on battery power since leaving for work this morning). The fan is off right now, and system is using about 377.6mb.

So, I consider myself in the "all systems normal" category.

One thing you should know is that Windows 10 has a new memory compression system that keeps data in RAM more. It avoids the slow process of paging the data out to virtual memory on the hard drive, which is hundreds of times slower than ram even on an SSD hard drive). It compresses the data in-memory and keeps it in that system process, and when needed expands.  You can see how much memory is actually in use Resource Monitor and only count the green section as "used" RAM.

I generally trust the memory manger to do right for me and it usually does, especially if I avoid massive hogs like Chrome.  Don't get me wrong, I have it installed, I depend on it as a web developer and use it (love the profiles function, I couldn't live without it). But I don't let it spiral out of control (search online for chrome and memory usage)... but I digress.

Honestly even a sustained CPU load of 10% would be enough to cause battery drain, probably enough heat to run the fan. It just shouldn't be that high all the time unless something is busy working. Mine is hovering at about 5 or 6% right now with all this stuff "running".

Aside from all that, if you've reached a point that you're positive it's not software inflicted you can run "powercfg /sleepstudy" and "powercfg /?" to learn about other tests. The sleep study can tell you if your device is sleeping fully or if specific hardware components aren't fully powering off. There is also an energy report that will identify what processes are using power, down to the milliwatt hour of energy they draw. Both reports output a nice HTML page graphics/charts report that opens in your browser, so you don't need to read or interpret the command line to use them on the quest to energy efficiency.

I hope this helps or at least informs a bit.  There's threads on these forums of solo many Surface owners talking about high heat or performance, but in the many many Surface Pro 3's I've been around, it's always been my experience that it's just got something running causing it and once identified and killed, solves the problem. Sometimes it takes booting into safe mode and adding back one thing at a time until you find what task wasn't playing nice.
