The reason that the install fails activation is that your machine was originally shipped with Vista, and the SLIC table in the BIOS is therefore for Vista.

An OEM install of Windows 7 requires a SLIC table for Win7 - it will NOT self-activate with teh SLIC table from Vista. (no matter what Andre tries to say)

The proper Upgrade disk came with its own Key. What you have is NOT an upgrade disk - it is the Recovery disk for a Win7 computer.

a_user_here say:

First off the group policy editor you are talking about is not in the Home version, only the Pro version.

There is no "Home Premium" version for Windows 10 so you got Home version in the update.

Next when you do an update from your old version of Windows to the new one, what is happening is that Windows installs a fresh copy of Windows, and then after that is done it transfers what it feels are the "user/program settings" from the registry in the old version of Windows to the new registry that it created.

So the answer is no, there is no way to have it leave your group policy changes in place.
