neilcameron1215 say:

This happens when you are using any application (or process related to an application) where the SB is using the nVidia dGPU as the graphics processor, because the dGPU is in the keyboard. So when you try and Detach it is saying, "hey, but I'm using the video card you are about to remove!".

You either have to close such applications and/or process before you detach - a royal pain.

Or go into the nVidia 3D settings - in the NVidia Control Panel - and change the Global Settings graphics card default ('Preferred Graphics Processor') - either to Auto-Select (which will work pretty well in practice) or to Integrated Graphics.

If you do the latter, you can still go into the Program Settings tab and change the default to the NVidia dGPU for any specific application you think would benefit from it. BUT if you are running that application you will get the same message on a Detach request, and will have to close it down first.

What you are seeing is intended as worlds created on individual devices are created locally and accessible when the creator is logged in on that device, but they are not sustained online and therefore not accessible if the creator is offline on the device that created the world.

Hi Dave Brougham,

The message is part of UAC (User Account Control), and is designed to inform you that a program requires elevated administrative) privileges to run.

An unidentified program is one that doesn't have a valid digital signature from its publisher to ensure that the program is what it claims to be. This doesn't necessarily indicate danger, as many older, legitimate programs lack signatures. However, you should use extra caution and only allow this program to run if you obtained it from a trusted source, such as the original CD or a publisher's website.  

If the program is from a legitimate source, then you may better check the manufacturer for updates where the application is aware of the security protocols in Windows Vista.
