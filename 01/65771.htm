tim bailen say:

Here's the actual fix I believe you are looking for. (I realize this is 5 years later and probably doesn't matter to you any more, but your post here is the top google result for "windows 7 default font size registry", so I'm mentioning it for the sake of other people finding this thread.)

This Microsoft post explains what's happening in detail. In short, your program is using bitmapped fonts instead of TrueType fonts and the text size setting (100% / 125% / 150%) doesn't affect those. The bitmapped fonts are set at the time of OS installation based on the default text size setting.

The post offers a registry solution which tells Windows to use the 100% sized versions of the bitmapped fonts (such as MS Serif, MS Sans Serif, and Courier) instead of the 125% sized versions.

The 125% versions of the fonts end with the letter "F", and the 100% sized versions end with the letter "E", so just import this .reg file to tell Windows to use the 100% sized versions, and then reboot so Windows picks up the change:

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Fonts]
"MS Sans Serif 8,10,12,14,18,24"="SSERIFE.FON"
"MS Serif 8,10,12,14,18,24"="SERIFE.FON"
"Courier 10,12,15"="COURE.FON"

Cheers,

Tim
