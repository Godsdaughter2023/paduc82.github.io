greg carmack - mvp 2010-2020 say:

That's why I asked to see a screenshot of Disk Mgmt, because nothing was mentioned about it being a UEFI install to GPT disk which does not Support 32 bit. 

To run 32 bit you'll need to use another hard drive, enable CSM or Legacy Mode in the UEFI firmware, turn off Secure Boot, boot the media as a non-UEFI device to install to it's own hard drive while deleting all partitions to clear the formatting.  Now you can have a mix of UEFI/.GPT and Legacy/MBR disks which should cooperate. 

The only way to install both 32 and 64 bit on the same hard drive is to enable CSM or Legacy BIOS, delete all partitions during install and do not boot the installation media for 64 bit as a UEFI device.  

A partition manager cannot  convert GPT to MBR format and have the OS remain bootable because UEFI uses special boot partitions which do not match MBR at all, so it will not be able to boot except as a fluke.

Windows 10 Maps App may fail to launch on some of the Tablets with SD cards

We would like to share a known issue and its workaround for the community. Please let us know if you have questions/concerns or if the workaround does not work for you.

Affected Apps Windows 10 Maps App and all other apps with Windows Maps
Affected devices Tablets with SD Card
Issue If you store Offline Maps on SD card storage on Windows 10 Tablets then the Windows 10 Maps App (and all other apps with Windows Maps) may fail to launch
Workaround While we are working on getting the fix out as soon as possible. Here is a workaround for you start using your Maps again. Update the offline maps storage settings to system drive (not the SD card)
Start>Offline maps settings>Storage location Select system drive from the drop down.

Update: The issue should be resolved with the latest Windows 10 update (Build 10586.35). After getting that update you should be able to put offline maps on an SD card without causing issues to the Maps app. Do let us know if you still have any issues.

Update 1/8/16: Based on our analytics and customer reports, we can now verify that the fix in build 10586.36 of the Maps app addresses this issue in about 80% of the devices originally affected. We are still seeing the problem on the remaining 20% of affected devices, and we continue to actively investigate its root cause. For those of you still seeing this problem, we recommend changing the storage location of your offline maps back to your C: drive for now (in Settings > System > Offline maps).  We apologize for the inconvenience, and can assure you that we have experts working on this as a top priority.

a. user say:

[quote user="Niruin"]

Just been into town and got my hands on Mass Effect 1 & 2, Just wondering should I put Myself & My Squad on Auto-Leveling, or just My Squad, or just jump in the deep end and learn?[/quote]Jump in at the deep end. The game will try and create balanced characters and while it might seem like a good idea to have a few points ini everything you are best off creating characters that compliment each other.

For example if you are running an Engineer Shepard and putting points into Decryption then it is a waste of time levelling that power up to a large degree in the squad-mates. 

 

[quote user="Niruin"]Also, what difficulty should I first start out on? I've been thinking Casual, be'coz I read somwhere that all the found weapons & levels can be use'd on the harder difficulty levels, is this true?[/quote]Personally I'd go for Normal for the first time because that'll give you a good level of balance and challenge. A note for later on though is that when going for the difficulty level achievements you can't change it once you've started playing, so as soon as the game starts set it at what you want and never touch it.

In Mass Effect 1 you can chose to start a new game and pick Select Existing ID to bring your Shepard and everything they have into the new run.
