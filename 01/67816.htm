net user administrator /active:no [enter]

Exit the command prompt.


You absolutely do not want to have only one user account. Like XP, Vista and all other modern operating systems, Windows 7 is a multi-user operating system with built-in system accounts such as Administrator, Default, and Guest. These accounts should be left alone as they are part of the operating system structure.

You particularly don't want only one user account with administrative privileges in Windows 7 because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult.  It isn't impossible to activate the built-in Administrator to rescue things, but it may be more than you want to do. Better not to put yourself into a bad situation to begin with.

The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. Running as a Standard user is best practice for security purposes and will help protect your computer from infection. After you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

If you want to go directly to the Desktop and skip the Welcome Screen with the icons of user accounts, you can do this:

Start Orb>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).

You can enable the built-in Administrator account from the command line by booting the Vista/Windows 7 installation DVD, by booting into the System Repair option on the diagnostic menu often given by OEMs, by booting using the recovery DVD .iso provided by NeoSmart, by booting with third-party rescue CDs which shall remain nameless ;-), or by booting with media made for the specific target operating system using the Windows Diagnostic and Recovery Toolkit (DaRT). This last is part of MDOP and not available to "regular people".

The point of enabling the built-in Administrator is so you can then log into it and then do whatever you have to do to fix the original problem. If it's that the end user only made one administrative user account and forgot the password (most common problem), you can change that account's password from the command line once you've enabled the built-in Administrator as I wrote above (and hence my "General Recommendations for Setting Up Users"). If the problem is that the end user somehow disabled his only administrative account or changed the account type to Standard without having another administrative account or it has become a Standard account because of malware, then you just want to enable the built-in Administrator, log in, do the fixes. Another situation that's very common with Vista (Windows 7 seems to be more stable in this regard but Time Will Tell) is that the end user only made one administrative user account, is using it for daily work, and that user account has become corrupted. So one can enable the built-in Administrator, log in, retrieve data from the corrupted account, scan for malware, make a new user account, etc.

In the very rare case (and I've never come across this in the wild) where an end user actually enabled the built-in Administrator, used it for his regular account, and then disabled it as you did, I would use "other methods". wink, wink. Since it is extremely unlikely that anyone posting here (not counting you!) would ever do this In Real Life(tm), I think we don't need to worry about it and can continue giving the "boot with your Windows DVD.... etc." information since doing that is completely native to Windows and not using any third-party tools.

keith a. miller say:

Only in a backwards sort of way. You can assign display names via desktop.ini, but the sort order is imposed by the actual file name --- not the display name.

Take a look at the Desktop.ini ifles found in the subfolders under:

C:\Users\UserName\AppData\Local\Microsoft\Windows\WinX

These control the Display name of the items in the WinKey+X menu, but the items sort on the menu based on the numerical prefix of the actual filenames, which can be seen via a 'dir' command in a Command Prompt or PowerShell window opened to the same folder.
