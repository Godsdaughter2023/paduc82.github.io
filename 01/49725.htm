xircal say:

You might have accidentally hit a comma instead of a full stop i.e. *,pdf.  If you subsequently use the cached search string, the error repeats itself.

Try doing it this way.

  1. Open File Explorer and click the drive you want to search PDF files on.
  2. Click once in the search field to get Search options at the top, click "Other Properties" and then click "File extension" from the menu options. See screenshot.
  3. In the Search field on the right, you'll see that appear as "fileextension:" Add "pdf" (without quotes) and without a space in between. All PDF files should now  appear instantly.

xircal say:

A thought just occurred to me.

You might have File Extensions hidden which is why Windows can't find them. To check, do the following.

  1. Open File Explorer, click "View" at the top and then click the Options button which appears on the right.
  2. In the Options menu, click the "View" tab.
  3. Next, click the little white circle where it says "Show hidden files, folders and drives" which will turn it black.
  4. Underneath that, remove the checkmark from "Hide file extensions for known file types".
  5. Click Apply/OK to exit.

Now do a search for PDF files.

patrick barker say: Hi John,




1. Navigate to the %systemroot%\Minidump folder.

2. Copy any and all DMP files in the Minidump folder to your Desktop and then zip up these files.

3. Upload the zip containing the .DMP files to Onedrive or a hosting site of your choice and paste in your reply. Prefered sites: Onedrive, Mediafire, Dropbox, etc. Nothing with wait-timers.

4 (optional): The type of .DMP files located in the Minidump folder are known as Small Memory Dumps. In %systemroot% there will be what is known as a Kernel-Dump (if your system is set to generate). It is labeled MEMORY.DMP. The difference between Small Memory Dumps and Kernel-Dumps in the simplest definition is a Kernel-Dump contains much more information at the time of the crash, therefore allowing further debugging of your issue. If your upload speed permits it, and you aren't going against any strict bandwidth and/or usage caps, etc, the Kernel-Dump is the best choice. Do note that Kernel-Dumps are much larger in size due to containing much more info, which is why I mentioned upload speed, etc.


Upload photos and files to Onedrive.

Please note that any "cleaner" programs such as TuneUp Utilities, CCleaner, etc, by default will delete .DMP files upon use.


1. Start > type %systemroot% which should show the Windows folder, click on it. Once inside that folder, ensure there is a Minidump folder created. If not, CTRL-SHIFT-N to make a New Folder and name it Minidump.

2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

4. Double check that the WERS is ENABLED:

Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. Also ensure that under Startup Type it is set to Automatic rather than Manual. You can do this by right clicking it, selecting properties, and under General selecting startup type to 'Automatic', and then click Apply.

Choose to keep your files and apps, just your files or to keep nothing

lemp say:

It sounds to me much more likely that the fault is in your TG782T than in Windows.

Just as an aside, a "normal" router will work with many more than 4 Ethernet connections (although most only have a max of 4 physical Ethernet jacks) and many more than 5 wireless connections.  So it is likely that your ISP has "customized" your device in some way that limits the number of connections -- and it's probably this "customization" that's giving you your current problems.  It's going to be a little difficult doing this via an Internet forum (rather than sitting in front of your computer), but it should be possible.

Please clarify the following sentence: "Disconnecting this computer and trying a new computer still has the same problem as the Gateway must still have this IP setting for this computer in it."  What is "this IP setting"?

In Local Area Connections > Internet Protocols (TCP/IP), how is the "working" computer configured?  Normally, it would be set to "Obtain an IP address automatically."  What was the "alternative setting" that your ISP wanted you to try?

The first thing to do is to get some information about your working computer.  Please do the following:

1. Click Start, then type the three letters cmd into the Run box and click OK.
2. Type this command and press Enter:
   ipconfig /all > "%UserProfile%\Desktop\ip.txt" {Enter}
3. Open the file IP.txt on your desktop
4. Copy and paste the text into your response.

One of the lines in IP.txt will be something like the following:

      Default Gateway . . . . . . . . . : 192.168.1.254

That number is the IP Address of your router.  Open a web browser (Firefox or Internet Explorer) and enter the IP address of your default gateway in the address box.

At this point, it's possible that you'll be asked for a user name and password.  If you didn't set one, use admin as the username and leave the box for the password blank.  If that doesn't work, there is a way around this, but it involves more phone calls with your ISP.

Unfortunately, the only manual for the TG782 that I have been able to find is woefully incomplete. http://www.thomsonbroadbandpartner.com/getfile.php?id=6978

After you log in, you should see a page with a menu on the left side (see p. 26 of the above pdf from thomsonbroadbandpartner).  Click the entry for "Home Network."  Unfortunately, the manual doesn't show the screen that appears when you enter this part of the configuration utility.  What you are looking for are the settings for the router's DHCP server, which assigns IP addresses to the computers connected to the router (doesn't matter if they're connected by wire or wireless).

Because the manual for your TG782T doesn't depict this, take a look at this, which is a portion of the configuration utility for a Linksys WRT54G router:http://cid-18b259158254905b.office.live.com/self.aspx/.Public/router%20setup.jpg

Under Network Setup, the Local IP Address is set to 192.168.1.1.  On your router, the default for this entry is 192.168.1.254.  Just under that are entries for "DHCP Server," "Starting IP Address," and "Maximum number of DHCP users."  These are the settings you want to look for.  The two most crucial settings, as far as your current problem is concerned are (1) the DHCP server should be "Enabled" and (2) the "Maximum number of DHCP users" should be equal to or greater than the total number of computers (wired and wireless) that you intend to connect.

Sorry I can't give you more specific instructions.  Usually, a router's user guide will show all of the available screens in the router's configuration utility ... but not this one.
