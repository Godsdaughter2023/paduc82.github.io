weatherdave say:

Hate to resurrect an old topic, but we've been seeing this on our new Win10 1703 Enterprise roll-out.  Firewall is on (apparently it can prevent installation if off).  Running as a standard user, I cannot install the fonts; right click doesn't install, nor does drag-n-drop to fonts folder.

However, when I switch over to an elevated account (switch user), who has local admin rights, I CAN install the fonts by both methods mentioned above.  In our case it was caused by a Group Policy:

REG:  HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows NT\MitigationOptions:MitigationOptions_FontBocking
GPO:  Computer Configuration\Policies\Administrative Templates\System\Mitigation Options\Untrusted Font Blocking

Briefly: An Aaron Margosis article from July, 2017 stated in regards to this setting:

"Only administrators can put files into the Fonts directory, so this setting keeps standard user programs from using GDI to handle fonts downloaded through web pages, embedded in Office or PDF documents, or downloaded by users."

Pre Win10 there was a privilege escalation issue with font handling via GDI, and this setting was a mitigation.  However, in Microsoft's 1703 baseline (same Aaron article), they suggest this be left as "Unconfigured" (basically Off) as it's no longer an issue.

Apologies for the long winded reply, but hope this help.
