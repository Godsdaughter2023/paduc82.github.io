Workaround #1 (Recommended by Microsoft):

Disable "Receive segment coalescing" (Rsc) on your wireless adapter

Tripp Parks [MSFT], provided an excellent workaround, that is recommended by Microsoft, and appears to resolve the issues. Thanks Tripp! And Tyler [MSFT] has provided a tool that automates it. Thanks Tyler!

Steps to disable Rsc using Microsoft's automated tool:

1) Download the ".diagcab" file located here:

https://aka.ms/diag_cssemerg11005

2) Run it, and let it see if it can determine the problem and fix it.

3) Re-test your wireless internet connection.

If you'd rather do it manually, then I have those steps below.

Steps to disable Rsc manually:

1) Open an Admin Command Prompt window:
Start > All Apps > Windows System -> Right-click "Command Prompt" > More > Run as administrator
... Note: If you want this available on your Start's Right-click menu, you can go to Settings > Personalization > Taskbar > Replace Command Prompt > Set to Off

2) Get your network adapter name:
powershell Get-NetAdapter

3) Note your adapter's name (mine is: Wi-Fi)

4) Get your network adapter "Receive segment coalescing" (Rsc) settings:
powershell Get-NetAdapterRsc
... Mine listed a table that had 1 row, for adapter name Wi-Fi, that had IPv4Enabled True, and IPv6Enabled True

5) Disable Rsc for your network adapter:
powershell Disable-NetAdapterRsc -Name ADAPTER_NAME_FROM_BEFORE
... Be sure to replace ADAPTER_NAME_FROM_BEFORE with whatever your network adapter name is, without quotes

6) Verify Rsc is disabled for ipv4 and ipv6, on your network adapter:
powershell Get-NetAdapterRsc
... Should now show IPv4Enabled False, and IPv6Enabled False, for your network adapter

7) Re-test your wireless internet connection.

Workaround #2:
Change MTU to 1400 (from Default, which may be 1500)

PB.Anton noticed the following, which I confirmed -- Thanks PB.Anton!

17.15.0.5 driver: MTU=1400: WORKS

17.15.0.5 driver: MTU=1500: WORKS

19.50.1.5 driver: MTU=1400: WORKS

19.50.1.5 driver: MTU=1500: DOES NOT WORK

To VIEW your MTU:
Right Click Start > Command Prompt (Admin) > Type or right-click-paste the following command:

netsh interface ipv4 show subinterfaces

.... make a note of your wireless one, it might be called "Wi-Fi" or "Wireless Network Connection"

To CHANGE your MTU to 1400:

Right Click Start > Command Prompt (Admin) > Type or right-click-paste the following command:

netsh interface ipv4 set subinterface "YOUR_WIRELESS_CONNECTION_NAME" mtu=1400 store=persistent

.... and use the VIEW command to make sure the change was accepted. Then test this workaround - Good luck!

Revert to driver 17.15.0.5 (2/22/2015)

Get driver 17.15.0.5 (2/22/2015):

- Open Device Manager

- Let it reload the Wi-Fi network adapter

- Keep Uninstalling, until it reloads the same version it had last reloaded.

...

- If you have 17.15.0.5, great! Read further down this post, to prevent Windows Update from changing it!

- If you do not yet have 17.15.0.5, try this:

- Right-click, Update driver

- Browse my computer for driver software

- Let me pick from a list of available drivers on my computer

- If 17.15.0.5 is available, choose it.

- If you have 17.15.0.5, great! Read further down this post, to prevent Windows Update from changing it!

18.32.1.2 (12/30/2015): Has problems

https://support.microsoft.com/en-us/help/3073930/how-to-temporarily-prevent-a-driver-update-from-reinstalling-in-windows-10

- Settings > Update & security > Windows Update > Check for Updates

- Verify the driver does not get updated

lemp say:

Windows Update (or its closely-related site Microsoft Update) will not give you any Windows XP updates until you are already running service pack 3 for Windows XP.


Starting with no service packs (sometimes called Windows XP Gold), there have been Service Pack 1, 1a, 2, and 3.  There are some prerequisites for the later service packs, so the first thing you need to do is to find out what service pack you now have.  RIght-click "My Computer" and select "Properties."  You will see the "System Properties" display, which will show which edition (Home, Pro, Media Center) and which service pack (none, 1, 2, 3) is installed.


Service Pack 3 requires that either Service Pack 1 (or 1a) or 2 be installed first.  Thus, if your System Properties shows no service pack installation, you first need to download and install Service Pack 2 and then download and install Service Pack 3.


You can download sp2 here -->

http://www.microsoft.com/download/en/details.aspx?displaylang=en&id=28


You can download sp3 here -->

http://www.microsoft.com/download/en/details.aspx?displaylang=en&id=24


Comments at both of those locations say

This installation package is intended for IT professionals and developers downloading and installing on multiple computers on a network. If you're updating just one computer, please visit Windows Update at http://update.microsoft.com.

Ignore that statement.  At the time that Windows Update was working for pre-sp3 Windows XP, that comment was intended to make the update process quicker and easier for people "updating just one computer."  It's no longer applicable.

Before you install Service Pack 3, read this --> http://support.microsoft.com/kb/950717

If you have an HP Pavilion, Presario or Media Center and has an AMD processor (you can determine this from the "Computer" section of the System Properties display that you looked at earlier) you need to install this patch before installing SP3…

http://h10025.www1.hp.com/ewfrf/wc/softwareDownloadIndex?cc=uk&lc=en&dlc=en&softwareitem=pv-60484-2

Before you actually start the installation process, be sure to temporarily disable your antivirus program.  Don't forget to turn it back on when you're done.  Reboot twice after the installation completes.

You also need to install Internet Explorer 8.  Go here --> http://www.microsoft.com/en-us/download/internet-explorer-8-details.aspx  Keep answering the questions until you get to the download page for IE 8.  Although you can download this along with sp 3, do not install IE 8 until AFTER you have already installed sp 3 and rebooted twice.

At this point, you can go back to Windows Update and it should scan your system and tell you about the many (many) post-sp3 updates you need to install.
