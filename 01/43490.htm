jaymo1978 say:

This sounds like a potential "cascading router" issue. Any time you have more than one network device between your device and the internet, it's possible for the devices to interfere with each other and cause a port conflict, resulting in a strict NAT. Even in a situation where it works great one minute, that can change at any time.

Do you happen to have the base IP address of the modem on your antenna? For example, 10.0.0.1, or 192.168.1.1? If you can get that from your friend, you'd be able to type that address into a web browser and access the firmware page for the modem itself.

From there, it can get tricky, but many modems/routers have something called bridge mode. This essentially passes the connection directly through the modem and allows the router to handle all security protocols and connection processing. If you already have port forwarding set up on your Netgear, placing the modem in bridge mode would allow that port forwarding to operate unhindered. If there is no bridge mode, there may be a way to set the port forwarding on the modem, specific to the IP addess of the router, but I'm not certain.

Theoretically it would work, because it would provide 2 layers of identical port forwarding, one from the modem to the router, and one from the router to your device.... Still, bridge mode would be an easier, more reliable solution.
