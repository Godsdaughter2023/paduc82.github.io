eric oberg say:

The problem is when choosing an icon by browsing icon files, if the icon file has a single image, there is no numeric value appended for the icon file in the registry. This value must be 0 for a single image icon. This ",0" must be appended by the Microsoft icon selection process. This problem also existed in Vista as well. If you browse to an icon with multiple images available, the image number is appended, the 0 image as well (IE %SystemRoot%\System32\moricons.dll,0). This is definitely a bug, as the Recycle Bin icon image real time change looks for the image number, where the initial display does not care.

When selecting an icon using the Personalization > Change Desktop Icons tool, an entry of

C:\Windows\Resources\Themes\Aurora\Icons\BIN Empty.ico

is stored in the empty field in key

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\CLSID\{645FF040-5081-101B-9F08-00AA002F954E}\DefaultIcon

The icon will not refresh if trash is added or emptied. If you simply add the ",0" to the end of the line, the refresh works fine, as below.

C:\Windows\Resources\Themes\Aurora\Icons\BIN Empty.ico,0

Please, Microsoft, add the "'0" to a single image icon!
