ken blake - microsoft mvp say:
I'm using Vista and I purchased a external hard drive to save my drive space.  I want to move Flight Simulator-X from C drive to my F drive.  How can I do it so I can save drive space?


 

You can not move an installed program from one drive to another. That's because there are many references to the program's location within the registry and elsewhere in \Windows. If you try to do this, the program won't work. What you need to do is uninstall the program, then reinstall it where you want it.

 

Note that there *are* several third-party programs that purport to find all those references and change them for you. My experience with them is that, at best, they *sometimes* work, so I recommend against them.

jeeped say:

I've culled together some steps from varying methods and put together these step-by-step instructions for retaining user settings for new Wordpad documents in Windows 7.

Step One - Create a Wordpad 'template'

  1. Start Wordpad and without typing anything, make any and all changes to Paragraph layout, Tabs and and/or Font style and size that you want to base new Wordpad documents on.
  2. When all formatting changes are completed, press <enter> once or twice.
  3. Press CTRL+S (e.g. Save) and save as Defaults.rtf in the folder you most commonly use to store your Wordpad documents. For the sake of example, these instructions will use the Documents library.
  4. Exit Wordpad.
  5. In Explorer, locate your new Defaults.rtf file and right click it, then select Send to, Desktop (create shortcut).
  6. Right click the Defaults.rtf file again and select Properties. On the General tab, check Read Only. Optionally, check Hidden. Click OK to save your changes.

Step Two - Modify the Shortcut to your Wordpad 'template'

  1. Back at the Desktop, rename the new shortcut as you like.
  2. Right click the new shortcut and change the Shortcut tab's Target: to,
    "%ProgramFiles%\Windows NT\Accessories\wordpad.exe" "%USERPROFILE%\Documents\Defaults.rtf"
  3. Change the Shortcut tab's Start in: to,
    %USERPROFILE%\Documents\
  4. <optional> If you want to use a different icon, click the Change Icon button and locate your preferred icon.
  5. Click OK to save your new Properties settings.

Step Three - (optional) Create Menu Items for your Wordpad 'template'

  1. Right click the new desktop shortcut and select from Pin to Taskbar and/or Pin to Start Menu.
  2. To create a Start Menu item, click Start and type (or paste in),
    %APPDATA%\Microsoft\Windows\Start Menu\Programs
    ... and hit <enter>.
  3. Copy the desktop shortcut to this folder and/or the Accessories subfolder. 

Your menu and desktop should now be populated with shortcuts to your new Wordpad with custom format settings.

Two things to note: You will have to use Save As instead of Save on new documents that have not already been saved, and navigating past the original <enter> that was inserted into the 'template' document will revert the font to defaults (e.g. Calibri, 11pt).

¡firedog say:

1.  Do I really need to re-install WLM on the new computer? 

Yes. Installation involves much more than just copying files.

2. ... how do I (a) get and install WlM and (2) how to I move my data (several email accounts and many storage folders) to the new computer?

Assuming that you had the latest version of Windows Live Mail installed on the old machine, you should be able to carry out the move fairly painlessly. You will need to know your account passwords, and you will have to recreate any message rules.

Locate the store folder in your backup set. This is by default

C:\Users\{user}\AppData\Local\Microsoft\Windows Live Mail

where {user} is the Windows user account of the user involved.

Copy this folder in its entirety to the equivalent location on the new machine - {user} may of course be different. Make sure that the new user account has ownership of the folder.

Download the installer for Windows Essentials from this page: How do I install Windows Essentials while offline?, then run it to install only the programs you want. The installer should find the store folder and take it into use.
