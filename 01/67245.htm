... there is a problem with the synching between talktalk and windows live, which talktalk confirmed to me two weeks ago stating that Microsoft had changed the settings on a number of occasions.

This sounds like Talktalk trying to shuffle off the blame for the problems that so many of their customers have when trying to access their mail account using a mail client like Windows Live Mail. There have been absolutely no changes in the way Windows Live Mail connects to an IMAP or POP3 server since at least 2012. There have been some changes, however, in how Windows deals with connections between mail clients and servers, but the ones I know of involve security, so I don't think they have anything to do with your unsecured connection over port 143.

To troubleshoot the problem would involve logging the transactions between client and server, then having an expert scrutinize the log to see where the connection is breaking down. I don't know of any such expert. You could try generating a log and then posting it here in case the support staff can help. Post back if you need instructions for doing this.

In the meantime, I wonder if you've screwed the server timeout setting up to its maximum just in case the server is responding too slowly to Windows Live Mail's commands. I don't expect this to make any difference, but you never know and it won't hurt to try. Right-click on the account name in the folder pane and select Properties. On the Advanced tab, set the server timeout to its maximum (5 minutes).

Hi,

 

For resolving issues with KB2863239 first try running the .NET Repair tool from here and then retry the update installation again.

 

-          The tool creates a log file (FixDotNet*.cab) on the desktop.

 

If this does not fix your problem then:

1.     Attempt a repair for the .NET 2.0 product from Control Panel and then try installing the update again.

 

2.     If even product repair does not fix your problem then share the log file FixDotNet*.cab by sending it to: netfxsetup_at_microsoft_dot_com (Alias: netfxsetup).
