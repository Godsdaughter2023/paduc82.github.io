abdul raqeeb kola say:

Hi,

 

For which app are you installing the certificates?

 

Installing self-signed certificates for your Domain’s Certification Authority

In order for self-signed certificates to work, the administrator needs to provide a certificate file that will be installed on client machines that are to be connected to the server that uses self-signed certificates. The certificates need to be installed to the trusted root certificate authority store for either the current user (doesn't require admin rights but needs to be completed for each user on the machine) or the local machine (requires admin rights and needs to be done only once for a machine).

Important: The following section contains steps that require administrative privileges to perform. Serious problems might occur if these steps are performed incorrectly. Please make sure that you follow these steps carefully.

 

Certificates can be installed using the following means:

 

Command Line: Running the following from an admin elevated command prompt will install the certificate to the trusted root certificate authorities for all users using that machine.

 

certutil.exe -f -addstore root <name_of_certificatefile>.cer

User Interface:

 

1. Double click the certificate file provided by the administrator. This will open a certificate dialog.

2. From the Certificate dialog, click the “Install Certificate” button located on the general tab. This will open a Certificate Import Wizard Window.

3. Select the option to install the certificate for only the current user or for the Local Machine.

4. Select “Place all certificates in the following store”.

5. Click the browse button to open the store selection dialog.

6. Select “Trusted Root Certification Authorities”

7. Once the store is selected, click Ok. This will return you to Certificate Import Wizard dialog and will display the certificate store and certificate to be installed into that store.

8. Click finish to install the certificate.

lemp say:

For most home wireless routers, if the IP address shown by running ipconfig isn't of the form 192.168.1.x or 192.168.0.x, you're not really connected to the router.  There are exceptions, of course, so please do the following while you presumably are connected by either Ethernet or Wi-Fi:

  1. Open a Command Prompt window (Start > Run > cmd.exe > OK)
  2. In the black Command Prompt window, type the following and press Enter after each line            
           ipconfig /all > "%userprofile%\Desktop\ipinfo.txt"
               exit
  3. Copy the file "ipinfo.txt" from the Desktop of your laptop to a USB flash drive; bring that flash drive to a computer that can connect to the Internet; open the file. Then follow the remaining steps using the other computer.
  4. In the open Notepad window, click Edit > Select All (or press CTRL+A) then click Edit > Copy (or press Ctrl+C)
  5. In the body of your Question, Right-click > Paste (or press CTRL+V) the data from the Notepad window
  6. Close Notepad and delete ipinfo.txt from your Desktop

As an alternative to steps 3-6, if you can access files on your Windows Desktop while booted into Ubuntu, open ipinfo.txt, copy the contents, and paste the data into your Reply here.

However, because you have the same issue whether you are connected using Ethernet or wi-fi, I suspect that the connection itself (which will be shown by the ipconfig results) is not the problem.  Assuming that you have at least XP service pack 2 (and preferably service pack 3), do the following:

Open a Command Prompt window (Start > Run > cmd.exe > OK)

In the black Command Prompt window, type the following lines and press Enter after each line

     netsh int ip reset c:\resetlog.txt

     netsh winsock reset catalog

     exit

Reboot the computer

Note: Programs that access or monitor the Internet such as antivirus, firewall, or proxy clients may be negatively affected when you run the netsh winsock reset command. If you have a program that no longer functions correctly after you use this resolution, reinstall the program to restore functionality.

lemp say:

It doesn't matter in this case (because you called attention to it) but generally, it's better not to edit posts after the fact but instead add new information in a new Reply.


From the ipconfig information you supplied, it appears that you ran the command while the Ethernet cable was disconnected, although I would have expected it to report "Media disconnected" if that was the case.


If the Ethernet cable was connected between the computer and the router when you ran the ipconfig command, then something is blocking communication between your computer and the router.  The IP address of 169.254.25.3 is a so-called APIPA address, which is automatically assigned when your network adapter is configured to obtain an IP address automatically via DHCP, but is not able to communicate with a DHCP server.


The problem may be caused by

  • a firewall on your computer (what antivirus and other security software is installed?)
  • a restriction configured on the router, such as a MAC address filter, time of day filter, or the like
  • a hardware problem with the cable itself, the Ethernet adapter on your computer, or the Ethernet port on your router.  You can test this by swapping in a different cable and connecting it to a different port on the router (assuming that your router has more than one Ethernet ports). 

With respect to the wireless connection, it appears as if you have configured the wireless network adapter to use a static IP address.  Although it is possible to do things this way, you would have also had to configure IP addresses for the Default Gateway and DNS servers. 


  • Go to Control Panel > Network Connections
  • Right-click on the icon for your wireless adapter and select "Properties"
  • In the list under "This connection uses the following items," double-click the line for "Internet Protocol (TCP/IP)"
  • On the "General" tab, make sure that the radio buttons are selected for "Obtain an IP address automatically" and "Obtain DNS server address automatically" as shown below

  • Click the "Advanced" button
  • On the "WINS" tab, select the radio button to "Enable NetBIOS over TCP/IP"
  • On the "Options" tab, if "TCP/IP Filtering" is shown, select it, click the "Properties" button, and make sure that the box to Enable TCP/IP Filtering is NOT checked.
  • OK your way out
  • Test your wi-fi connection.  If it doesn't work immediately, reboot and test again.

I'm not familiar with ADB Broadband and I couldn't find anything online about the P.DG A4001A.  I did, however, find the P.DG A4001N.  That device is wireless b/g/n capable and if

(a) your wireless router is also wireless-n capable and

(b) is somehow configured to be in wireless-n only mode (not likely to be the default),
