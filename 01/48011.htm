ken blake say:

If you have correctly set the BIOS's boot order to CD/DVD first, and the computer doesn't boot from it, your problem has nothing to do with the CD/DVD drive's driver. Also running in safe mode or running SFC can't possibly help you.

The reason that drivers, safe mode, and SFC can't help you is the same for all three: booting (whether from the hard drive or a CD or DVD) takes place before Windows loads. If it won't boot from a bootable CD or DVD, the problem has nothing to do with Windows; it's a hardware issue. I don't know what's wrong, but one possibility is that the CD/DVD drive has died.

Whatever's wrong can but be repaired, but whether it's economical to do so is another matter. It depends on what's wrong. If it's the CD/DVD drive, they are inexpensive, so replacing it should not be expensive. Fortunately, it doesn't sound like there's a problem with the hard drive, so even if you should buy a new computer, you should be able to remove the hard drive from the old computer, put it in an external case, and transfer its data to the new computer.

Let me add one other point: You say "I have a lot of information that I need to get off it if I have to buy another one." That suggests that you do not regularly back up your data to external media. If that's the case, you've been playing with fire. It is always possible that a hard drive crash, user error, nearby lightning strike, virus attack, even theft of the computer, can cause the loss of everything on your drive. As has often been said, it's not a matter of whether you will have such a problem, but when.
