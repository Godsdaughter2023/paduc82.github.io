robert aldwinckle on forums say:

I've downloaded procmon but How do I use it to find out the error?

You have to do it by inference.  E.g. you will be able to see writes being done to the file(s) that you see messages in that you want more information about, so clearly any conditions which occurred which caused those writes to be done happened before that, probably just before that, e.g. hopefully expressed in terms of a previous registry or file access which failed, such as with an Access Denied.  Etc.  

You can filter on "writes" in general by using  Category Is Write.   The length of the write and timestamp can be a clue when the length of the message you are interested in is distinct in a pattern of writes to that file and its timestamp is in the range of your message but before the write.

 

Good luck

Robert
