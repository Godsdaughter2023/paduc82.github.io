Not to be a jerk but...

Those are some ideas, but there are lots of reasons why they will not work.

First of all, there is no error 'sfc_os.dll file' missing' so Sahihyan needs to provide more information and the correct error message.

Second of all, the sfc_os.dll is one of the 3498 files that Windows File Protection looks after so it should never be missing.  It takes extra effort to even delete it on purpose.  Try renaming or deleting yours and Windows File Protection will quickly and silently replace it unless WFP is also afflicted.

So if it really is missing, it was probably quarantined by the unspecified antivirus/antimalware application on the system of unspecified make and model with unspecified XP version and Service Pack

Next, you can't run sfc /scannow in Safe Mode and if you were to actually try it yourself, you will get an error like this:

Windows File Protection could not initiate a scan of protected system files.
The specific error code is 0x000006ba [The RPC server is unavailable.].

You cannot start the RPC Server in Safe Mode either.

You could however run sfc /scanonce to initiate a /scannow on the next reboot:

http://support.microsoft.com/kb/310747/en

But that will only work if you have a genuine bootable XP installation CD that is the same Service Pack as your installed Service Pack (most people don't).  And it still will not replace a missing sfc_os.dll file in the system32 folder since that is not what sfc does, that is not how it works and that is not what it is for (try it).

If the sfc_os.dll file is really missing, no matter what mode you use to boot you will see this:

winlogon.exe - Unable to Locate Component
This application has failed to start because sfc_os.dll was not found.  Re-installing the application may fix this problem.

Click OK and things should look okay if nothing else is wrong.  You should still get a desktop and be able to maneuver unless something there is some other problem.

If the sfc_os.dll file is corrupt, you will see this:

winlogon.exe - Bad Image
The application or DLL C:\windows\system32\sfc_os.dll is not a valid Windows image.  Please check this against your installation diskette.

Then when you do get past that, sfc /scannow will not work either because it will not run without the sfc_os.dll file and if you were to actually try that you will see this:

sfc.exe - Unable to Locate Component
This application has failed to start because sfc_os.dll was not found.  Re-installing the application may fix this problem.

Window File Protection is also broken if that file is missing.

Let me assure you with 100% certainty that running sfc /scannow will not 'detect that sfc_os.dll is missing and replace it'.  That is not what sfc /scannow does, that is not how it works and that is not what it is for.  That is what a lot of people think it does, but I can give you a test scenario you can go through to see what sfc /scannow really does and what it doesn't do.

Why not boot your system in Safe Mode (so Windows File Protection is not running) or the Recovery Console and rename your sfc_os.dll to something you can remember:

From the system32 folder:

rename sfc_os.dll   sfc_os.xxx

Now the file is "missing" (but you can rename it back later).  Since Windows File Protection does not run in Safe Mode, the file will not be automatically replaced.

If you were to boot normally and rename the file, Windows File Protection would quickly and silently replace it and put a message like this in the Event Viewer System log:

Event Type:    Information
Event Source:    Windows File Protection
Event Category:    None
Event ID:    64002
Description:
File replacement was attempted on the protected system file c:\windows\system32\sfc_os.dll. This file was restored to the original version to maintain system stability. The file version of the system file is 5.1.2600.5512.

That is Windows File Protection in action.

When your file is "missing" because you renamed it in Safe Mode or the Recovery Console, Reboot and see how your ideas about Safe Mode and other things work out.

C:\windows\system32\dllcache
