So I needed a way to restart this device upon wake from sleep.  Google posts suggested using DEVCON.EXE from the Windows Driver Kit (WDK) for the device restart.  So I downloaded and installed the WDK and then wrote a 4-line DOS script (cmd.exe) to perform the device restart and rescan.  I needed something to run the script on system "wake" and Google posts suggested using a scheduled task with trigger on event ID "1".  I checked my event log and this event did occur on each re-wake.  So I created a scheduled task to run the DOS script that was triggered on event ID 1 from source "Power-TroubleShooter".  

So far, my fingerprint scanner has worked every time the laptop wakes from sleep (fingers crossed tightly).

Here's the DOS script in egis_reset.bat file on my desktop (replace <username> with your username if you're going to try using it):

del c:\users\<username>\desktop\egis_reset.txt
echo Egis reset batch starting %Date% -- %Time% > c:\users\<username>\desktop\egis_reset.txt
devcon restart *pid_0603 >> c:\users\<username>\desktop\egis_reset.txt
devcon rescan >> c:\users\<username>\desktop\egis_reset.txt

If your fingerprint scanning device isn't an EgisTec 603, you'll have to use DEVCON.EXE to determine the hardware ID of your device and insert it into the above script in place of my "*pid_0603".

a. user say:

Hello Peter,

 

I realize that you have few queries for the effects of changing paging file (pagefile.sys) on your Windows 8.1 computer.

 

PageFile.sys is your "virtual memory" or "swap space". It is where things from your RAM go when they are not currently being used. You should never delete that file. You may, with some caution, change the Windows system settings that dictate the size of that file.

 

I’ll answer your queries in series of the questions asked.

 

  1. The pagefile is designed to store modified data that has not been recently accessed. This makes more RAM available for more important purposes, such as for active applications and file caching.
    Disabling the pagefile unbalances the entire virtual memory system. Modified data must then remain in RAM no matter how infrequently it is accessed.

  2. There is no such utility to modify these changes.

  3. We don't recommend doing so because of the possible drop in performance. There are a couple of things you can do though. First of all you can move the file to another drive. For example, if you're trying to make space on an SSD, move pagefile.sys to a hard drive if you have one.

 

Hope this information helps. If you need any further assistance or information on this issue, reply back to this post. I will be glad to help you.

 

Sarandeep S - Microsoft Support.

 How to activate Windows 7 manually (activate by phone)

6) Explain your problem clearly to the support person.
