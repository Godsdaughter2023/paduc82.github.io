a. user say:

Hello,

 

Thank you for posting on Microsoft Community. We regret the inconvenience caused and will assist you in resolving the issue.

 

Before assisting further, I would like to ask you few questions:

  1. What are the issue you are facing with?

  2. What do you mean by “I had this happen earlier”, is similar issue occurring again?

  3. If yes, what was the issue you were facing with before and now?

 

To delete a corrupted user profile, you have to enable built-in administrator.

 

Enable built-in administrator by following the steps below, create a new user profile, transfer all the data from the corrupted user to the newly created user profile and check if the issue is resolved.

 

Step 1: Enable built-in administrator account.

 

  1. Open elevated command prompt by entering Safe mode with Command prompt. To go to safe mode, use the power button on the sign-in screen and press Shift + Restart. This will take you to the recovery boot menu. Click Troubleshoot> Advanced options> Startup settings. You then see a list where you can choose Safe Mode with Command prompt.

  2. Type the following command: net user administrator /active:yes

  3. Login into the built in administrator user profile and create a new user profile with administrator privileges.

  4. Delete the corrupted user account from the built in administrator.

  5. Then lock your screen by pressing Windows key + L, and sign-in at the Administrator account.

  6. net user administrator /active:no

 

Step 2: To create a new user account follow the steps below:

 

  1. Open Elevated command prompt by pressing Windows logo + X keys on the keyboard and select Command prompt (Admin).

  2. Type net user NewAccount password /add and click Enter.

  3. Type net localgroup Administrators NewAccount /add and click Enter.

  4. Log off from the system and login to the new user account.

 

Write to us with the answers to the above questions and the status of the issue on the same post for further assistance. Your reply is most important for us to ensure we assist you accordingly.

 

Thank you.

chiefnerd say:

Our team ran into this exact same problem with Windows 10 1709 and Check Point FDE. See Check Point SK122633 for details.

The problem is tied to Windows Fast Boot that forces the kernel into hibernation mode instead of power off mode to save time when booting (aka HiberBoot). This can be fixed by disabling Fast Boot.

See https://www.howtogeek.com/243901/the-pros-and-cons-of-windows-10s-fast-startup-mode/ for manual disable instructions.

This can also be scripted via GPO.

considerate_guy say:

If you are talking about a file that is pinned to the start menu or task bar, you just unpin it there.  You right-click on the application icon to bring up the list of associated files.  Next to that you should see a pin icon.  If it is pinned you can toggle it to unpinned.

 

On the other hand, if you are talking about the "Recent Projects" list inside of Movie Maker, that is stored in the registry.  There is no facility within Movie Maker to remove items from the Recent Projects list.  If you are comfortable editing the registry, you could find the corresponding items and remove them.  So if you want to continue, let me know and I'll look up the registry key.

considerate_guy say: The list of recent project files is stored in the registry under the following key.
HKEY_CURRENT_USER\Software\Microsoft\Windows Live\Movie Maker\Recent

If you are comfortable with editing the registry, you can delete the contents under that key. But if you have any doubts about what you are doing, you should just leave the items in the list.
