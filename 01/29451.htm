Method I

a. Click On start; In the Start search, type Services.msc and Hit Enter.
b. Right click on Windows Installer and click on Start ( Only if the Service has stopped).
 
Restart the computer and try to uninstall Java TM from the Programs and Feature Wizard in the Control Panel.

Method II
Unregister and re-register the Windows Installer
1. Click Start, click Run, type MSIEXEC /UNREGISTER, and then click OK. Even if you do this correctly, it may look like nothing occurs.
2. Click Start, click Run, type MSIEXEC /REGSERVER, and then click OK. Even if you do this correctly, it may look like nothing occurs, or you may briefly see an hourglass. After you run this command, the operation is complete.
3. Now try uninstalling Java TM again.

Method III
I would suggest you to update your Windows Installer Package with the latest version of Windows Installer available in the link below.

Windows Installer 4.5 Redistributable

1. Click on Start; in the start search type Services.msc and press Enter.
2. Right click on Remote Procedural Call and click on Properties.
3. Click on the Log on tab and highlight This account.
4. Type Local Service. Click OK and then Click Apply

Now check if you’re able to uninstall Java TM from Windows Vista.

Method IV

Windows 7 - how to stop application foreground focus stealing - Microsoft Community

andreyrac say:

Windows 7 - how to stop application foreground focus stealing

Ever try to load up an application in the background and start typing somewhere?  Ever get extremely annoyed by all the focus stealing and end up with a jumble of words showing up in different applications by accident?  Well I have a fix, that's well known, but the searches I have found on this matter yielded poor results.

Steps for remove foreground focus stealing:

1. Go to 'Run..." or just start typing "regedit" in the start menu.  However you get there, open regedit.exe.

2. On the left pane, collapse all the folders and look for root folder "HKEY_CURRENT_USER" and expand it.  Then expand the "Control Panel" folder within.

3. Click on the "Desktop" folder and you should see bunch of fields in the right pane.

4. Double-Click on "ForegroundLockTimeout" and set "Value data:" to 0. (Hex or Dec should make a difference)

And viola, your gripes and pains are relieved (until the next quirk gets on your nerves anyways).

Hope this is useful and please tell me if this doesn't work for you.

joem_50 say:

The default for this setting should be 200000 which equates to about 3.5min.  This setting tells the OS to wait this amount of time before allowing a background window which made a foreground request to move to the foreground.  When this is set, the icon should flash if a request is made, but the window should remain in the background for the specified time. 

If this value is set to '0'. then any background window requesting to move to the foreground will not be delayed.  In your case, setting this to 0 adds to the issue.  I would set this to 900000 and see it that helps.
