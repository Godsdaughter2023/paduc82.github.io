If I were you (you did state that you do "like" (enjoy) having Win 10 Pro active on the device), even that the query in and of itself is of valid concern, I would be happy to the point of not looking to "look a Gift Horse in the Mouth", so to speak.

To check the assigned license assigned to the device in question, (could be hit or miss but worth trying);

https://support.microsoft.com/en-us/help/10749/windows-product-key

From a command prompt;

C:\Windows\System32>slmgr /dlv
Run the above command from a CLI prompt and you'll get feedback from Windows 10 which will tell you if you have a retail license or an OEM license
C:\Windows\System32>slmgr /xpr
Run this command to find out if you are permanently licensed

Basically, with an OEM license, you change out the motherboard you lose your OEM license & must purchase a new one. But a retail license will survive even a motherboard swap/move to a newer machine, etc. If this was not true then Microsoft would have no need to differentiate between the two--retail,OEM--& explains why retail costs more than OEM, etc.

Or from a PowerShell environment (recommended if your device is UEFI enabled);

If you want to find the UEFI-embedded key and write it down, you can do so quite simply. Just open the Start menu, type “powershell”, and run the Powershell application that comes up.

Then, enter the following command and press Enter:

(Get-WmiObject -query 'select * from SoftwareLicensingService').OA3xOriginalProductKey

You should then be rewarded with your embedded license key. (Hint: write it down and store it in a safe place).

A 3rd party app is/can also be a viable way to retrieve a Prod Key (at your own risk, of course).

IMPORTANT:  The following procedure permanently deletes all files and folders which are already in the Recycle Bin (from all user accounts in the system) and cannot be recovered.

1.Open an elevated Command Prompt window. To open an elevated Command Prompt, click Start, click All Programs, click Accessories, right-click Command Prompt, and then click Run as administrator.

rd /s /q C:\$Recycle.bin

(This clears out the $Recycle.bin folder from the C:\ partition. You may have to do this for each hard disk partition in your system.)

Type exit to close the Command Prompt window.

1. Open an elevated command prompt. To do this, click Start, click All Programs, click Accessories, right-click Command Prompt, and then click Run as administrator. If you are prompted for an administrator password or for a confirmation, type the password, or click Allow.
2. Type the following command, and then press ENTER:

sfc /scannow

The sfc /scannow command scans all protected system files and replaces incorrect versions with correct Microsoft versions.

I have not recommended that Blue Screen View  be used to debug dump files. This is what I have said.

 

2.10. The interpretation of a dump report is extremely difficult and many users are not able to master the techniques and tools needed to undertake the task. To diagnose what is going on you often need to look at a number of dump files. For those who wish to try debugging the next link offers helpful advice here.

2.11. Microsoft has provided a list of Bug Check Codes with information as to their meaning. Many BSOD errors are driver related and a memory dump file contains details of all drivers loaded when the system failure occurred. If you are not using the Windows Debugging tool you can use Blue Screen View to get a list of drivers in the dump file. You can use Driver View to get a complete list of installed drivers and their versions. These details help when checking for driver updates. Go to the computer or motherboard manufacturer’s web site for free updated drivers before looking elsewhere.

Source: http://www.gerryscomputertips.co.uk/syserrors3.htm

 

The Windows Debugging Tool is the best way to determine the cause of a BSOD error but to use it effectively requires acquired skills that many users do not have. Acquiring those skills takes time to learn and considerable effort.

 

Blue Screen View, used in conjunction with Driver View, enables users to identify third party drivers appearing in dump files and their versions, so that updated drivers may be found. Updating drivers usually helps the system run better and the skills required to do this are far less demanding than those required to use the Windows Debugging Tool. Updating drivers is something all users can do and it can eliminate blue screen errors. If that approach does not work, you can provide dump files in this and other forums where more experienced users can interpret the dump files for you.
