a. user say:

Hi Jordan,
Thank you for reaching out to Microsoft Community Forums.
 
I would like to tell you that "winlogon.exe" runs in the background. Winlogon is a part of the Windows Login subsystem, and is necessary for user authorization and Windows activation checks. Do not delete it, it is a valid windows system file. If we delete the file we will not be able to login to the computer.
 
Taskhost is not a virus or malware, it is actually just a process that runs in the background. Windows uses taskhost.exe as the host for all DLL based services that are run. The process is very similar to svchost.exe.
 
Taskeng is responsible for keeping track of tasks set to run at a time predetermined by the user, and to invoke them when necessary.
 
Hope this answers your query.

a. user say:

Hi Jordan,

Thank you for the reply.

 

The Winlogon.exe shown in the Task Manager is the background process which is displayed notifying that the particular user is logged in and it's location is C:\WINDOWS\system32\winlogon.exe

If it is from any other location then it could be an infection.

Don't try to delete or rename C:\WINDOWS\system32\winlogon.exe.

Based on the issue description I understand that you are facing intermittent BSOD issue on your computer.
I can understand how frustrating it could be when computer behaves unusual.

The Blue Screen errors usually occur when a serious problem causes Windows to shut down or restart unexpectedly. There are multiple reasons why this type of errors occurs on the computer. They usually happen when there are some driver issues on the computer. However, it might also be due to some other software/hardware reasons. The blue screen error provides a bug-check error code which is used to identify the root cause of the issue. A bug-check (BSOD) is a mechanism to protect your PC from damage when such critical errors occur on it.

The KERNEL_SECURITY_CHECK_FAILURE bug check has a value of 0x00000139. This bug check indicates that the kernel has detected the corruption of a critical data structure. For more information refer to this article:

Bug Check 0x139 KERNEL_SECURITY_CHECK_FAILURE
http://msdn.microsoft.com/en-in/library/windows/hardware/jj569891(v=vs.85).aspx

You may also refer to the article below:

Resolving Blue Screen errors in Windows

http://windows.microsoft.com/en-us/windows-8/resolve-windows-blue-screen-errors
