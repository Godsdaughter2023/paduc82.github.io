LESSON LEARNED


In Windows 7, all administrator accounts (except for the built-in Administrator account that is disabled by default) run as standard users and implement User Account Control ("Do you want to allow the following program to make changes to this computer?") for elevation of privileges. You can right-click on a program and choose Run as Administrator to run with elevated privileges.


The solution would seem to be to change ownership of C to yourself and grant yourself full permissions so that you can do anything. After all, you own the computer, right?  However, Windows does not like when you alter the ownership or permissions of the drive or drive partition on which it is installed – normally the C: drive, or whichever drive or partition is listed as "System, Boot, Page File, Active, Crash Dump, Primary Partition" in Administrative Tools – Computer Management – Disk Management.


Do not mess with ownership or permissions of C drive. When you right-click within the drive's root folder, you can only add a folder from the context menu. (Right-click – New – Folder.) This is normal. To do anything more, you need elevation of privileges. You can copy files in from somewhere else, but you can't do much directly on C:.


If you mess with the ownership or permissions of C, or whatever the boot and system drive is, you can create a situation (like I did), where you can boot the computer fine, but there are no icons on the Desktop, you can't access the C drive, and can't open any Control Panels or programs on C. You can also cause more subtle problems such as the illusion of multiple nested folders because of junction point issues. For example, when you look into your file structure, you can have what looks like dozens of Application Data folders.



Now, back to the original problem with chkdsk/f. I did some searching and found the following to be the best work-around to this flaw in Windows 7. This work-around will put the Desktop back on C:, and will backup C:\Desktop to "D:\Desktop backup" continuously in the background. When you run chkdsk/f on D:, you will no longer have any instability or freeze-up issues if you concurrently do something else on D: such as opening a document.


If you have the Desktop in D:\Desktop and haven't done so already, open D: and right-click on the folder Desktop within D:.  Click on the Location tab and change the location to C:\Desktop.

Afterwards, create the batch file, script, and scheduled task to backup C:\Desktop as follows:

Start – Run – cmd

Copy con "Desktop backup.bat"

Robocopy C:\Desktop "D:\Desktop backup" /MIR /MOT:5 /RH:0002-2356

You can change the times as desired.  I'm using 12:02 am to 11:56 pm, checking at 5 min intervals.
Hit F6 key or Ctrl-Z to generate ^Z and click Enter to save the file.

Windows reports "1 file copied".
