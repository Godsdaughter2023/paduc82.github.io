I had the same problem, and for three days I tried everything listed here and on every other site I could find.  Finally (finally!) I found the problem and solution:

https://www.askwoody.com/2018/report-of-the-win10-1709-cumulative-update-kb-4074588-disabling-usb-devices/

antoneth ber say:

Service Set Identifier or SSID is the name of a wireless network. You need to know the SSID of your network if you want to let other devices connect to it. To find the SSID of your PC, please follow these steps:

  1. Open Control Panel.
  2. Under Network and Internet, click View network status and tasks.
  3. Click Change adapter settings.
  4. Double-click Wi-Fi.
  5. Click Wireless Properties and look for SSID.

cmdrkeene say:

Hey Michael, the vote link should have been at the top of that.

Libraries are functionally like you describe.  What they really are is "search definitions".  In essence, a library is a list of locations and search criteria (it's nearly identical to if you ran a search and then clicked "save this search" as a shortcut to repeat it later).

If you open up "%appdata%\Microsoft\windows\Libraries" (no quotes), you can see the actual library definition files.  They are like shortcuts though, so even if you have Explorer set to show file extentions, you won't see that they really end with ".library-ms" as the file names.  You used to be able to open them up in Notepad (drag it into an already-open Notepad window), to read the definition of the search terms.  But in recent versions that's not as easy to do.

Their legacy is really from what Vista had as "search folders", sometimes referred to as "stacks".  It was awesome (I sincerely miss it) but they were so confusing to the general public that they were completely abandoned and Libraries got introduced in Windows 7.

Libraries themselves aren't totally deprecated, but they've been really de-emphasized in recent editions and not shown in the Explorer pane by default.  I wouldn't be COMPLETELY surprised if they get removed entirely in the future, but for now a great many programs use them as a reference of which folders you want to show up in your Music or Pictures apps, for example, so it's likely they'll stick around for that anyway.  (File history also uses them to decide which files to backup).
