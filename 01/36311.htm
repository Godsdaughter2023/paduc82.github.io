tim de baets say:

WMP automatically picks up any embedded album art, and also uses it as cover art for the album of the song. This is by design, and can't be changed.

The album art that is displayed in WMP's library, is stored as hidden JPG files in the same folder as the music files they belong to. You may need to set Windows Explorer to show hidden files and folders, and to show protected operating files, before you can see these JPG files.

tim de baets say:

WMP doesn't keep separate album art for each song, there's just one album art JPG for each album. The format of these JPG filenames is AlbumArt_{GUID}_Large/Small.jpg , where  GUID is a unique value that is internally stored in WMP's library, and used to track albums. You can also have Folder.jpg files, if such a file exists in a folder, then it will be used as album art for all music files in that folder.

I'm pretty sure that files named  cover1.jpg won't be recognized by WMP. Are you trying to set individual album art for songs that were ripped with WMP? Have you already 'broken up' the album, so the individual songs aren't grouped anymore as a single album by WMP?

Tim De Baets

WMP keeps a cache of the album art in  C:\Users\<username>\AppData\Local\Microsoft\Media Player\Art Cache\LocalMLS . You can delete all files in that folder to clear the cache.
