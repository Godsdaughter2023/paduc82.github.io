Alternate Solution to Install Error 0x80300001 - Microsoft Community

redxii say:

Alternate Solution to Install Error 0x80300001

I have an all SATA configuration, it shouldn't be too uncommon for all hard drives and all optical drives to be SATA. My mobo has a Intel ICH7R controller currently in AHCI mode. When loading the driver using Load Driver, it causes the drive letter of my BD-ROM to be reassigned. Googling around the only other solution is because someone forgot to remove the driver disk, pulling extra drives out, erasing the entire disk and other destructive options like that. This drive letter reassignment didn't occur with a IDE DVD-ROM.

 

To view the list of volumes and assigned letters I opened the Command Prompt using Shift+F10, and activated diskpart with the 'diskpart' command. Here is a list of volumes before loading the driver, using the 'list volume' command:

 

X:\Sources>diskpart

Microsoft DiskPart version 6.1.7601
Copyright (C) 1999-2008 Microsoft Corporation.
On computer: MININT-D7HI7S9

DISKPART> list volume

  Volume ###  Ltr  Label        Fs     Type        Size     Status     Info
  ----------  ---  -----------  -----  ----------  -------  ---------  --------
  Volume 0     G   GRMCULXFRER  UDF     DVD-ROM     4482 MB  Healthy
  Volume 1     C   SevenD       NTFS   Partition    100 GB  Healthy
  Volume 2     E   Data         NTFS   Partition    200 GB  Healthy
  Volume 3     F   Media        NTFS   Partition    165 GB  Healthy
  Volume 4     D   Games        NTFS   Partition    465 GB  Healthy

 

After loading the driver, the list looked like this

 

DISKPART> list volume

  Volume ###  Ltr  Label        Fs     Type        Size     Status     Info
  ----------  ---  -----------  -----  ----------  -------  ---------  --------
  Volume 0     D   GRMCULXFRER  UDF    DVD-ROM     4482 MB  Healthy
  Volume 1     C   SevenD       NTFS   Partition    100 GB  Healthy
  Volume 2     E   Data         NTFS   Partition    200 GB  Healthy
  Volume 3     F   Media        NTFS   Partition    165 GB  Healthy
  Volume 4     G   Games        NTFS   Partition    465 GB  Healthy

 

G: and D: got switched, it can no longer find the installation files and produces 0x80300001. The setup environment is expecting the sources to be on G:. So I had to use diskpart to reassign the letters:

 

  • First assign volume 4 letter H so I can free up G. diskpart can't assign letters in use.

 

DISKPART> select volume 4

Volume 4 is the selected volume.

DISKPART> assign letter=H

DiskPart successfully assigned the drive letter or mount point.

 

  • Then select the BD-ROM and change the drive letter back to G:

 

DISKPART> select volume 0

Volume 0 is the selected volume.

DISKPART> assign letter=G

DiskPart successfully assigned the drive letter or mount point.

 

  • And reassign volume 4 back to letter D:

 

DISKPART> select volume 4

Volume 4 is the selected volume.

DISKPART> assign letter=D

DiskPart successfully assigned the drive letter or mount point.

 

Now the drive letters are back where they should be:

DISKPART> list volume

  Volume ###  Ltr  Label        Fs     Type        Size     Status     Info
  ----------  ---  -----------  -----  ----------  -------  ---------  --------
  Volume 0     G   GRMCULXFRER  UDF    DVD-ROM     4482 MB  Healthy
  Volume 1     C   SevenD       NTFS   Partition    100 GB  Healthy
  Volume 2     E   Data         NTFS   Partition    200 GB  Healthy
  Volume 3     F   Media        NTFS   Partition    165 GB  Healthy
* Volume 4     D   Games        NTFS   Partition    465 GB  Healthy

 

I hope this helps someone, or if I was unclear about something. Your drives will obviously be different, do a 'list volume' before loading the driver so you know what the DVD/BD-ROM should be assigned when it gets reassigned due to loading the driver.
