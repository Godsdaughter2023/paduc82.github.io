¡firedog say:

Three points about image sizing:

  1. In signatures, it's a good idea to use both CSS and HTML specifications for size. Some clients will honour only one of these, others will prioritize the one over the other. If they're both the same, this won't matter and all clients will produce the same effect. You have given only HTML specs: width=80 height=80. In the style=" ... rule, add width: 80px; height: 80px;
     
  2. You'll get most consistent results if the image you include in your signature is already at the size you want it to be. Image processing software will almost always produce a better result than rendering engines used by mail clients. So, if you want the image in your signature to be 80 x 80 px, use an image editing program to resize it. MS Paint will usually produce a satisfactory result, especially if you save the image as PNG rather than JPG.
     
  3. Whenever you insert an image into a Windows Live Mail message, it will adopt the default formatting options. Select the image and use the options on the Format tab to give it the required size, border, alignment and margins, then click Set to default. The settings you applied will then be used for all inserted images until you change them and Set to default again.

I'm not sure how well your border-radius commands will work. It depends so much on the rendering engine used by the recipient's mail client what the effect will be, so you can't rely on it. Keep it simple ...

The same applies to non-standard fonts. Although the signature doesn't use it, you have specified the Nyala font for the container. Some clients may well spend time looking for this font, and then for a substitute when it doesn't find it. This probably won't affect the end result as far as the recipient is concerned, but it will delay the display of the message unnecessarily.

To test your hard drive, check the manufacturer and then get the diagnostic utility from:http://www.techsupportforum.com/hardware-support/hard-drive-support/302602-hard-drive-diagnostic-utilities.html (and/or get one from your hard drive or computer manufacturer if they have one available – it wouldn’t hurt to try both).  If it fails the test, replace it.  If it passes the test, then there's probably nothing wrong with it.

You can also test your RAM at http://www.memtest.org/.  If it fails, test each chip individually to find those that are bad and replace them.  If it passes, RAM is probably OK.

 

Hopefully one of these procedures will resolve or at least identify the cause of your problem.  If not, post back with results from what you tried and copies of some of the scans and we'll try something else.  Post back in any case so we'll know how it turned out.

andy rathbone say:
Can any USB flash drive work on the Surface RT?
I've used my Surface RT with a wide variety of USB flash drives from many manufacturers. They all work fine.

The only problem you may face is if a flash drive includes software for encryption or backing up your computer. The bundled software won't work.

But if you're planning on using the flash drive only for storage, you'll be fine.
