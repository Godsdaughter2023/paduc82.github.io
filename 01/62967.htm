ronnie vernon say:

Hi JF

 

The Event ID 7000 is recorded when a service fails to start because of missing or corrupted files associated with the service or another software program/driver is preventing the service from starting properly.

 

The SessionLauncher service is associated with Roxio Software products. This could be a compatibility problem with that software or the SessionLauncher service.exe file may be corrupted.

 

If you are experiencing problems with this software, I recommend that you check with Roxio Support or post this question on the Roxio Support Forums and ask about any known issues with this error message.

 

Roxio Community:

http://forums.support.roxio.com/

 

More information.

 

If the Roxio software is working properly, this event could be a simple timing or other issue that does not effect the software. Event Errors can be misleading because they record every small detail of how software interacts with the computer.

 


Regards

devin lanei say:

I have successfully installed Photo Story 3 on Windows 8 Enterprise x64, and the Windows 8.1 Enterprise x64 Preview.  I have also discovered exactly what breaks it, and I would guess that this is what broke Photo Story 3 for people on Windows 7: LegitCheckControl.dll

 

If you download Photo Story 3 from the Microsoft website via the alternate method the application will install and work just fine.  If however you choose to install the ActiveX Windows Genuine Advantage version, it installs the LegitCheckControl.dll, a few registry settings, and will result in the following errors.

 

 

Installer error:

"Photo Story 3 for Windows cannot be installed on this version of Windows. Check system requirements for Photo Story 3 for Windows before trying to install the program."

 

 

Application launch error:

"Photo Story 3 for Windows cannot run on this version of Windows.  Check the system requirements for Photo Story 3 for Windows before trying to run it."

 

 

While the problem is going on, you can install Photo Story 3 on systems that are damaged by the ActiveX components through the command line with the following example:

 

msiexec.exe /i Pstory.msi /passive SKIPOSCHECK=1 NOLEGITCHECK=1

 

To fix both issues:

As the ActiveX installer does not include an uninstaller, the best solution is to rename the WGA DLL via PowerShell or Command Line as Administrator so that Photo Story can no longer find it, and so you can enable it again in the future without issue.

 

Windows x86:

PS: "Rename-Item $env:windir\System32\LegitCheckControl.DLL LegitCheckControl.Disabled.DLL"

CMD: "rename %windir%\System32\LegitCheckControl.DLL LegitCheckControl.Disabled.DLL"

 

Windows x64:

PS: "Rename-Item $env:windir\SysWOW64\LegitCheckControl.DLL LegitCheckControl.Disabled.DLL"

CMD: "rename %windir%\SysWOW64\LegitCheckControl.DLL LegitCheckControl.Disabled.DLL"
