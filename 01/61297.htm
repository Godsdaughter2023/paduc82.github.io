Based on your ipconfig data, it appears that you are connected to the network:  your wireless network adapter has been assigned an IP address by the network router's DHCP server and you have a default gateway address.  Neither of these would appear as they do if your computer was not communicating with the network.

While you're waiting, open a command prompt and do some connectivity testing without a browser (save the results and show them to your network administrator):

type the following commands and press enter after each one:

     ping google.com

     ping 172.20.10.1

"Passing" results should look something like the following:

Ping statistics for 172.217.6.238:
    Packets: Sent = 4, Received = 4, Lost = 0 (0% loss),
Approximate round trip times in milli-seconds:
    Minimum = 18ms, Maximum = 19ms, Average = 18ms

If you have 0% loss from both ping commands, your computer is connected to the Internet with no issues.  The problem is related to your browser and should have been helped by the winsock reset command.  This could be the result of a misconfigured firewall or security application on your computer or corrupted browser settings. 


If you have 100% loss from both ping commands, you are being blocked from the Internet despite having received an IP address from the router.
