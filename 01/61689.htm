Why is 32-bit even still an option? - Microsoft Community

sdcondon say:

Why is 32-bit even still an option?

This is more of a marketing opinion than anything else, but these days, everything that is capable of running still-supported versions of Windows has a 64-bit-capable CPU. Further, the limitations of a 32-bit Windows are really unacceptable, and will become even more so as more and more programs become more and more RAM-Hungry. I don't believe Microsoft should offer a 32-bit option at all.

The situation with my computer at work is a prime testament to this. It runs Windows 7 32-bit. It has 4 GB of RAM, which I could possibly upgrade, If I were not affected by registry access restrictions (I'm aware PAE exists, and aware it can be enabled). This 4GB gets eaten up in a matter of minutes with all the stuff my job has me using, causing the system to run incredibly slow.

Microsoft is used to cutting support for clingers, as evidenced by the cutting of support for the Windows 9x series, and, even more recently, the cutting of support for Windows XP. Microsoft should offer some way to upgrade from 32-bit to 64-bit (To my knowledge, no method to do so exists currently), and stop offering 32-bit altogether, and kill it off with Windows 10.

a_user_here say:

Just because you don't use something doesn't mean it doesn't have a purpose.

So you are using memory hogging programs and probably are one of those people that never closes a program, just have to have them all running at the same time.  That's fine, but that doesn't mean that the whole world is like that.  I have three machines.  One is a server with 10 GB, one is a desktop with 4 GB, and the last is a laptop with 3 GB.  All perform just fine.  It is either you machine has problems or that your use case needs the memory.

It really seems that for some reason that because you have not been able to change from 32 bit to 64 bit, that you believe that if somehow you convince Microsoft to drop support for 32 bit they will then magically provide you with that option.  Well in fact going from 32 bit to 64 bit has always been possible provided your machine will support it (most do) and you have the right install media, and that you realize that it can't be an "upgrade" it has to be a clean install and then restore your data files/settings kind of operations.

For your Windows 7 machine the main sticking point there is most likely your the lack of the install media/license.  In the case of Windows 10 Microsoft has provided both the 32 and 64 bit media, so you can in fact take your machine from 32 bit to 64 bit if you want to.

http://answers.microsoft.com/en-us/windows/wiki/windows_10-win_upgrade/how-to-switch-from-32-bit-versions-of-windows-to/40d90c4e-8845-4bcb-bb2f-aeab312caf74

I know that it really wouldn't concern you, but there is in fact a few other considerations of why someone would want to stay with 32-bit.

1) If they are like me and need a machine to support very old programs that are 16 bit.  A 16 bit program can't be run on a 64 bit operating system.

2) Some old machines can't upgrade their memory past 4 GB.  In this same area, a 64-bit program/operating system tends to take between 1.25 and 1.5 times more memory than a 32 bit one does.  So right off the bat if you have a 32 bit operating system with 4 GB you will want to get to about 6 GB just to have the same amount of memory available.
