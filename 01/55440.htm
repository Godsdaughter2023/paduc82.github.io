anatoliy tishin say: I finally figured it out myself... I am not sure which of two actions helped, but here is what I did:

1. I made a default setwallpaper command under various of graphics file extensions pointing to Windows Photo Viewer CLSID (before they were pointing to another CLSID, which I believe was Office Fax Viewer -  {ff609cc7-d34d-4049-a1aa-2293517ffcc6}). Here is how I did it:

   - In RegEdit I searched for "wallpaper" and under HKCR\SystemFileAssociations it found several extensions (.bmp, .jpg, etc.) where I swapped the value of a key "HKEY_CLASSES_ROOT\SystemFileAssociations\.bmp\Shell\setdesktopwallpaper\Command\DelegateExecute" from {ff609cc7-d34d-4049-a1aa-2293517ffcc6} to {FFE2A43C-56B9-4bf5-9A79-CC6D4285608A}

   - Then it found similar keys under HKEY_LOCAL_MACHINE - their values were already set to  {FFE2A43C-56B9-4bf5-9A79-CC6D4285608A}, but it won't hurt to double-check


2. Then I removed the register key "HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\Control Panel\Desktop\Wallpaper". This key had empty value - so I assumed there is no much use and then shortly after I did that I got my wallpaper from selected theme.

Hi Akash,

 

Thank you for the update.

 

The reason is very simple, as the error says, the Internet Protocol (IP) address conflict is there, it means that there are more than one devices, modems or routers or laptops which exist at the same IP address as is your computer.

Method 1:

Follow these steps in order and check if it helps in fixing the issue:

a. Turn off everything on the network, all the machines and any other network device like network printers and NAS drives.

b. Turn off the router and modem.

c. Turn on the modem, wait for a steady data light.

d. Turn on the router, wait two minutes.

e. Turn on the network devices and computers, one at a time.

 

Method 2:

You may refer to the article to reset TCP\IP settings and check if it helps.

How to reset Internet Protocol (TCP/IP)

 

Method 3:

You may also follow these steps and check if the issue persists.

a. Click on Start button.

b. Type ‘cmd.exe’ in Start search and press Enter.

c. Right click on the “cmd” and run as administrator.

d. Type the following commands and press Enter.

ipconfig /release

ipconfig /renew

e. Close the command prompt.

If you have others un-check the selection boxes to those items. Next, select the TCP/IP line and click the properties button. Setup the connection to use "Obtain an IP address automatically (DHCP) and "Obtain DNS server address automatically". This way your wifi switch/router will define what IP address the PC receives because it sounds like you manually configured the properties. Question: is your wifi switch/router configured to distribute IP addresses (meaning it's acting as a DHCP server)? If not, my solution will not work for you.
