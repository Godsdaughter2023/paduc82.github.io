Step 1: Each computer on a network must have a unique IP address. To fix this problem, run the Network troubleshooter.

Step 2: Resetting everything(computers/routers/ect) will force all systems to request a new IP if they do not have a manual IP setup

Select Network and Internet
Select Network and Sharing Center
select Local Area Network
In the Window that opens select Properties
On the next page select Internet Protocol Version 4(TCP/IPv4) and select Properties.
Ensure that Obtain an IP Address automatically is selected. Same goes for DNS
Select Alternative Configuration and make sure it is set to Automatic Private Address
Ok out of all those Windows and see if things are better.

Driver Verifier:


What is Driver Verifier?



Driver Verifier is included in Windows 7, Windows Server 2008 R2, Windows Vista, Windows Server 2008, Windows 2000, Windows XP, and Windows Server 2003 to promote stability and reliability; you can use this tool to troubleshoot driver issues. Windows kernel-mode components can cause system corruption or system failures as a result of an improperly written driver, such as an earlier version of a Windows Driver Model (WDM) driver.



Essentially, if there's a 3rd party driver believed to be at issue, enabling Driver Verifier will help flush out the rogue driver by flagging it and causing your system to BSOD.



Before enabling Driver Verifier, it is recommended to create a System Restore Point:



Vista - START | type rstrui - create a restore point

Important information regarding Driver Verifier:



- If Driver Verifier finds a violation, the system will BSOD.



- After enabling Driver Verifier and restarting the system, depending on the culprit, if for

example the driver is on start-up, you may not be able to get back into normal Windows

because Driver Verifier will flag it, and as stated above, that will cause / force a BSOD.



If this happens, do [b]not[/b] panic, do the following:



- Boot into Safe Mode by repeatedly tapping the F8 key during boot-up.



- Once in Safe Mode - Start > type "system restore" without the quotes.



- Choose the restore point you created earlier.



If you did not set up a restore point, do not worry, you can still disable Driver Verifier to get

back into normal Windows:



- Start > Search > type "cmd" without the quotes.



- To turn off Driver Verifier, type in cmd "verifier /reset" without the quotes.



- Restart and boot into normal Windows.

How Memtest works:



Memtest86 writes a series of test patterns to most memory addresses, reads back the data written, and compares it for errors.



The default pass does 9 different tests, varying in access patterns and test data. A tenth test, bit fade, is selectable from the menu. It writes all memory with zeroes, then sleeps for 90 minutes before checking to see if bits have changed (perhaps because of refresh problems). This is repeated with all ones for a total time of 3 hours per pass.



Many chipsets can report RAM speeds and timings via SPD (Serial Presence Detect) or EPP (Enhanced Performance Profiles), and some even support changing the expected memory speed. If the expected memory speed is overclocked, Memtest86 can test that memory performance is error-free with these faster settings.



Some hardware is able to report the "PAT status" (PAT: enabled or PAT: disabled). This is a reference to Intel Performance acceleration technology; there may be BIOS settings which affect this aspect of memory timing.



This information, if available to the program, can be displayed via a menu option.

¡firedog say:

Server Response: -ERR [SYS/TEMP] problem retrieving message.
...
Windows Live Mail Error ID: 0x800420CD

This is a temporary system error at the server, because there is probably a message in the mailbox at the server that Windows Live Mail can't or won't download for some reason - it's too big, it contains malicious code or it's just plain dodgy.

To clear the blockage, log in to the account in webmail and start by emptying the folders for spam and deleted items. Then create a new folder or folders and move all of the messages out of the inbox and into the new folder. When the inbox is empty, select the account from the dropdown list on the bottom half of the send/receive button in Windows Live Mail. No new email messages should eventually appear in the status bar.

Back in webmail, move half of the messages that haven't yet been downloaded by Windows Live Mail back into the inbox and again try send/receive for this account in Windows Live Mail. If the fault recurs, we now know which half of the messages the bad one is in. Repeat the process, taking half of the messages each time until the dodgy message can be identified and dealt with in webmail.

In future, don't use the webmail inbox as a storage folder. Move any messages for keeping on the server into archive folders, leaving only a few days' worth of messages in the inbox at any one time. Empty the spam and deleted folders regularly.

allen z. msft support say:

Hi PW,

I understand that you’d like to do the migration without setting up a hybrid deployment. The main steps you listed are correct. And there’s no other special configuration that you will need.

Additionally, this is the order I often suggest my customers:
1. Disable the on-premises accounts by running the Disable-CsUser command. Normally, this command will also remove all those related attributes from the on-premises AD.
2. Run Directory Sync to synchronize the user accounts to Office 365.
3. Assign the Skype for Business online licenses to the synchronized user accounts.
4. Change the related DNS records, such as SRV and CNAME, to Office 365.

Feel free to let us know if anything is still unclear.

Thanks,
Allen

young yang msft support say:

Hi PW,

There is no problem.

Previously, users’ Lync service is hosted in on-premise Lync server. DirSync service can detect this. So, Skype for Business (SFB) Online service will not be provisioned for the users, even if SFB Online license is assigned to the users.

Now, you disable on-premise Lync account and remove on-premise Lync attributes. With the SFB Online license, SFB Online service will be provisioned for the users.

Thanks,

Young
