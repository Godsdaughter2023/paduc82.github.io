entegy say:

The language packs in the Store are for 1803 only. This is mentioned in the system requirements on the Store page (Windows build 17074 or higher).

1709 must use the Settings app's Time & Language category. Add a language to the list, then click "Options" for that language and wait to see a download button for that language. It can take several minutes to search Windows Update for the pack. I also leave the Settings app open when downloading a language pack so I can see its progress.

Once it's done, there's a tool in Windows called lpksetup that can show you what language packs are installed. Right-click the Start button and choose Run. Type lpksetup and press Enter. In the tool, click "Uninstall language packs" and you'll be presented with a list of installed language packs. 

There is one major thing that can block the Settings app from finding language packs. If your computer is a work computer connected to a WSUS server for updates, your admin must add the language packs to the WSUS for your computer to download them. If your admin has not added them to the WSUS server, then you have to find and manually install a language pack using lpksetup. Microsoft has an ISO with all the language packs available for admins on their various sites, but I don't see this kind of download for end users. This is likely why they were added to the Store for 1803.

shekhar s say: Hi Modularity,
 
Are you using a static or dynamic IP address?
 
Method 1. Run Network Troubleshooter
Run the netwotk diagnostic tool, find the steps mentioned in the article below:
http://windows.microsoft.com/en-US/windows7/Using-the-Network-troubleshooter-in-Windows-7

Method 2. DHCP Broadcast
You may be experiencing this problem due to the router’s IP possibly being assigned as 192.168.0.*
The router IP should be set to 192.168.(1-10)*.1
*Pick a number from 1 through 10
Computer would be picking this up as an irregular IP, thus the “unidentified network.”
Additionally, the router may not support the DHCP BROADCAST flag so we can modify the registry in order to prevent Windows 7 from using the DHCP BROADCAST flag.
 
From KB 928233: http://support.microsoft.com/default.aspx/kb/928233  (This article stands good for Windows 7 as well)
Note: Modifying REGISTRY settings incorrectly can cause serious problems that may prevent your computer from booting properly.
Microsoft cannot guarantee that any problems resulting from the configuring of REGISTRY settings can be solved. Modifications of these settings are at your own risk.
    1.       Open the Start Menu
    2.       Type regedit in the Start Search area and press Enter
    3.       Click on Continue in the UAC prompt
    4.       Locate and then click the following registry subkey:
           HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\Interfaces\{GUID}
           In this registry path, click the {GUID} subkey that corresponds to the network adapter that is connected to the network
    5.       In the right pane, locate DhcpConnForceBroadcastFlag
    6.       Right-click DhcpConnForceBroadcastFlag, and then click Modify
    7.       In the Value data box, type 0, and then click OK
    8.     Close Registry Editor and restart your computer to apply the changes
Note A data value of 0 disables this registry entry. You can use this registry entry to prevent Windows Vista from using the DHCP BROADCAST flag. After you set this registry entry, Windows Vista never uses the DHCP BROADCAST flag.

Method 3. Use Network troubleshooter event logs to solve network problems.
You can see the Network troubleshooter event logs in Event Viewer.
The event logs are saved as system events in the Windows Logs folder with an Event ID of 6100. Provide us the log to help you better.
1. Open Event Viewer by clicking the Start button  , clicking Control Panel, clicking System and Security, clicking Administrative Tools, and then double-clicking Event Viewer.    If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
The events should be filtered so that only events from the Network troubleshooter are shown.
2. To open an event, double-click it.
http://windows.microsoft.com/en-US/windows7/Use-Network-troubleshooter-event-logs-to-solve-network-problems

  • Try disabling the firewall temporarily and check. If the firewall is the problem, you'll have to configure it to allow access to "trusted zone" addresses. Note that some firewalls must be completely uninstalled to stop them from affecting your networking. Note: You are advised to enable firewall as soon as  you finish with the troubleshooting.

a. user say:

Hi,

Thank you for posting your query on Microsoft Community.

This command "user passwords2" is used to open User Control Window.

Did you check the box "Users must enter a user name and password to use this computer" ?

If so, please go back to the User Control Windows and uncheck this option.

  

If it doesn't help then I would suggest you run the built in administrator:

  1. On the Search bar type “cmd” and click ctrl + shift + Enter to open the Elevated Command Prompt. (If prompted for a password, click Allow).
  2. In Command Prompt type "net user administrator /active:yes"
  3. Sign out from this system and then sign in to administrator account.
  4. Open Windows key + R to go to the run Command and type "user passwords2". Click Enter.
  5. Uncheck the box "Users must enter a user name and password to use this computer".

Hope this helps. Please respond if you have further related queries.
