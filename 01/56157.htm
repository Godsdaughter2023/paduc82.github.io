a. user say:

Hello,

Welcome to Microsoft Community. I will certainly help you with the issue that you are facing.

I would like to ask you:

  • Are you using HomeGroup?
  • Is the system connected to a domain?

A network drive is a shared storage area associated with a local drive letter made available from one host to other hosts on a computer network, so that a piece of information on a computer can be remotely accessed. Drive mapping is used to locate directories, files or objects, and programs or apps, and all data on various mapped drives have certain permissions set and the user need the particular security authorizations to access which are also intervened by the third party firewalls from the security software installed on the systems.

So, I would suggest you to disable the anti-virus temporarily, installed on your system and map the desktop's drive on the network from your Windows XP computer.

Step 1: Disable the security software on both the computers

Disable Antivirus

If you have installed any third party security software like antivirus or firewall, I would suggest you disable them temporarily and check if it works.

 

Important Note:Antivirus software can help protect your computer against viruses and other security threats. In most cases, you shouldn't disable your antivirus software. If you have to temporarily disable it to install other software, you should re-enable it as soon as you're done. If you're connected to the Internet or a network while your antivirus software is disabled, your computer is vulnerable to attacks.

Step 2: Check your Windows XP Desktop's IP address

  1. On your Windows XP Desktop, press Windows + R.
  2. Type cmd and hit enter.
  3. Type ipconfig and hit ENTER.
  4. In the information displayed, note the IPv4 Address.

Step 3: Map the drive in Windows 8.1 computer from the IP address

  1. Press Windows + E to launch 'This PC'. Click on Map network drive from the options given under the address bar.
  2. In the Drive box, click the drive letter you want to use.
  3. In the Folder box, type \\server\share, where you can type the IP address of the drive noted in Step 1. For example, if the IP address is 192.168.6.xxx, you can type \\192.168.6.xxx.
  4. Click Finish.

Check if you are able to access the desktop's files. Please post back your results for further assistance.
