a. user say:

You will waste your time checking the behavior by booting the computer in a clean boot state.  That has nothing to do with this problem.

The lsdelete program is installed on your system by Lavasoft/Ad-Aware software.  

If that software detects any files on the system that it thinks needs to be deleted after a virus/malware scan, it adds an entry to the startup portion of XP so the lsdelete program will run to delete the files while XP is starting and before XP is completely loaded.

This is because some files cannot be deleted one XP has finished loading and is up and running, so the lsdelete program will delete any files that need deleting before XP loads up all the way (ls delete = LavaSoft delete).

Usually what happens for this issue is if you have uninstalled LavaSoft without using their special Lavasoft uninstaller, it may have left the entry in the registry or perhaps your LavaSoft installation is broken or otherwise afflicted, but you can get rid of the entry that wants to run the lsdelete program.

Before making any changes to your registry by hand or with third party tools, be sure to make a backup of the registry
first. There is no undo or quit without saving option in regedit.

Here is a link to a popular registry backup tool:

http://www.snapfiles.com/get/erunt.html

Click Start, Run and in the box enter:

regedit.exe

Click OK to open the registry editor and navigate to the following key:

HKEY_LOCAL_MACHINE\SYSTEM\CURRENTCONTROLSET\CONTROL\Session Manager

Locate the BootExecute entry on the right hand side of the screen.

The default value is for BootExecute is:

autocheck autochk *

Yours probably looks something like this:

autocheck lsdelete autocheck autochk *

Double click the BootExecute value and remove the reference to lsdelete so the BootExecute value looks like this when you are done:

autocheck autochk *

Click OK to save the changes, then click File, Exit to close the registry editor and reboot the system.

If you still think you want to use LavaSoft products, go to their WWW site and download their latest version.

FYI:  The default entry for BootExecute is the check XP makes when it starts to see if there are any drives/volumes have the "dirty" bit set on them.  If a volume has the dirty bit set (usually from a power failure), that autocheck autochk * line will cause a chkdsk /r to be run on the dirty volumes.

Running chkntfs /d will not fix this problem (that is not what chkntfs /d is supposed to do) as a matter of fact, it will make things worse.
