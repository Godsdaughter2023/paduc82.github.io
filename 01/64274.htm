Hi Alex,

The error message "Ethernet doesn't have a valid IP configuration" is usually occurring because the Dynamic Host Configuration Protocol Server is unable to get a valid IP Address from your Network Interface Card. To further troubleshoot your concern, we suggest disabling and enabling your Network Interface Card. Follow these steps to do so:

  1. Press Windows key + R.
  2. On the Run box, type ncpa.cpl and then hit the Enter key.
  3. Right-click  on the Network Interface Card.
  4. Select Disable and then Enable it after a few minutes.
  5. Wait until it successfully receives an IP address.
  6. Restart the PC and then check if the same issue persists.

If the issue persists type the following commands in Command Prompt:

  • ipconfig /release
  • ipconfig /flushdns
  • ipconfig /renew

If further assistance is needed, we are here to help.

It's just a VGA Splitter?  A true plain VGA splitter?

The OS would have no effect on that.  A VGA Splitter just takes one signal and duplicates it (internally) to two screens.  You should be able to take a VGA Splitter and connect it to any system with its input type of output and split the single output signal to two different screens.  Mirror.

carlo cesar olano say:

Workaround for WIndows 8 update stuck at 12-13% (AMD CHIPSET Drivers)

This issue is caused by driver conflict between Windows 8 and AMD chipset Driver

The main goal of this Troubleshooting is to force install the Windows 8 update before reinstalling the AMD Chipset drivers

Try this step by step:

Step 1: If the computer is configuring windows update @12% or 13%, you need to interrupt it by powering it off(hold down the power button till it shut down completely<no lights at all>) Restart your computer and check if will still configure the windows update. If it does, interrupt it again by holding the power button till it shut down completely

Diagram: "configuring windows update at 12-13%> TURN OFF >reboot> Configuring windows update> Turnoff

Step 2: Restart again and After two failed attempts, Windows automatically opens the ''automatic repair" menu and it will revert the windows update.  Next screen will give you Keyboard layout option, just choose the US and hit next to bring you over to the next screen which will give you 3 options; CONTINUE, TROUBLESHOOT, RESTART. Just choose Choose the TROUBLESHOOT.

Here are the option that you might see under advance option:

*System Restore

*System Image Recovery

*Refresh your PC

*Automatic Repair

*Start Up settings

Step 3: Click the Startup Settings, and it should prompt you to "Restart to change Windows option such as:

*enable debugiging

*enable low video resolution

*enable boot logging

 *Enable safe mode and blah blah blah.

Just click the "restart" on the right bottom side corner to boot it on Startup Setting Screen

Step 4: Once you are on the Startup Settings screen. You need to select "Enable Safe Mode with Networking (F5)" and the computer should load the "Windows 8 Safe with Networking"

 

 

Step 5: Let the computer reboot to safe mode. Once your reached the desktop screen using safe mode, just restart the computer again under the setting option.

Diagram:

Safe mode Desktop> Restart> Windows Normal Screen

Note:  It's useless to uninstall the drivers in device manager because "windows update" is not available in Safe mode screen. So just restart the computer to normal windows screen

Step 6: Once you are on the "Normal Start screen"(notsafemode) Pull up your control panel and open the device manager. Look for Sound video and game controllers expand it and uninstall IDT HD audio and AMD, or whatever driver you have under Sound video and game controlers.  Also, uninstall all drivers under "Display Adapter" Especially if it's an AMD Graphics driver.

Step 7: After uninstalling the drivers, Go back to control panel and open "Windows update" then Click "check for updates" on the left side. After checking updates, download and apply all important updates. Then Restart

Note: If you don't see "Check for updates" under Windows update. You might be using a local account or the computer is on safe mode. You need to switch to Microsoft account before doing updates or load the normal desktop screen

Step 8: Once computer restarted and reached the "Startscreen", Pull up HPSA and run HPSA live update. Then go back to Device manager, Check if the Sound driver and display driver has been installed automatically after restart

Step 9:  If above TS will trigger again "configuring windows update and stuck at 12-13%. Just Repeat step 1 to 5 and Set the windows update to never install. Inform customer that this will temporary fix the problem and we need to wait for microsoft and AMD to release a permanent fix
