Hi Koham

You're correct that the JumpList behavior has changed within the last few months.

I think these changes have been an effort to try and stabilize the JumpLists. There have been many issues where the JumpLists would become corrupted and most of these issues seem to have been resolved since we have seen the number of problems posted here decline, significantly.

The core problem with what you are trying to do is that you want the JumpList to behave like a normal Toolbar, where you can pin any shortcut and have it retain the shortcut name. A normal toolbar is simply a standard folder, where you can place any type of file shortcut.

This just does not work with the JumpLists. They are much more complex than a traditional toolbar because they also integrate the Recent files, Frequent files and Tasks for certain programs. The data for the JumpLists are stored in raw code -ms files.

If this is important to you, you can create a new folder somewhere in your User folders and name it Windows Explorer. Right click the Taskbar and select Toolbars / New Toolbar and point it to the new Windows Explorer folder.

This will give you a Taskbar toolbar named Window Explorer where you can pin any shortcut.
