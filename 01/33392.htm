lemp say:

Although there may be some way to identify the cause of the installation failure using the "Add a Printer" wizard, if you are concerned that Brother's software package is very bloated (which seems to be the case for most printer manufacturers) why not try some of Brother's stripped-down packages?

There's a "Printer Driver & Scanner Driver for Local Connection" which is only 45 MB and a "Add Printer Wizard Driver" which also is about 45 MB.

Brother describes the first as "only includ[ing] the printer and scanner (WIA and/or TWAIN) drivers, optimized for USB or Parallel interface" and the second as "only includes the printer drivers and is for users who are familiar with installation using the Add Printer Wizard in Windows."

The links above are for the Windows 7 x64 versions, but you can find the other versions here: http://support.brother.com/g/b/downloadtop.aspx?c=us&lang=en&prod=mfcj485dw_us_as

I have no idea what Brother means by "optimized for USB or Parallel interface," but I fully expect that once that driver is installed, you should at least be able to print via wifi, if that's what you want, but I don't know about scanning.

lemp say:

P.S.:  When you used the Add a Printer wizard, I assume that the driver had to be downloaded via Windows Update.  At the least, the MFC-J485DW is not in the list of Brother printers that appears in the list of drivers on my Windows 7 system.

There are, however, drivers for this printer (for Windows 7, 8, 8.1, or 10) available through the Microsoft Update Catalog, so if using one of the smaller downloads from Brother doesn't work, you might consider downloading the driver appropriate for your system.  There's a bit of jumping through hoops required for installing a driver obtained from the Update Catalog.  If you need help, just ask.  http://www.catalog.update.microsoft.com/Search.aspx?q=mfc-j485dw

Hi ,

 

1.    Was it working fine earlier?

2.    Is the issue specific to USB modem?

Follow the below mentioned methods:

 

 

Method 1:

 

 

This behavior can occur if the incorrect driver is installed for the modem.

 

To resolve this issue, remove the modem, and then restart the computer. When the computer restarts, Windows detects the modem and attempts to install the correct driver for it. To do this, follow these steps:

a. Log on to the computer as administrator or as a member of the Administrators group.

b. Click Start and then clickControl Panel.

c. Double-clickPhone and Modem Options.

d. Click the Modems tab.

e. In the ‘The following modems are installed’ list, click the modem that you want to remove, and then click Remove.

f. Click Yes in response to the "Are you sure you want to remove the selected modem(s) from your system?" message.

g. Restart the computer.

h. TheFound New Hardware Wizard starts. Follow the instructions in the wizard to reinstall the modem.

Also refer to the below link:
http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/mode_test_modem.mspx?mfr=true

 

Method 2:


I would suggest you to do a SFC (System file checker) scan on the system and check whether the issue is resolved. Follow the steps mentioned in the article given below.
Description of Windows XP and Windows Server 2003 System File Checker (Sfc.exe)
http://support.microsoft.com/kb/310747


Note:

Windows XP installation disc may be required for SFC.

Try connecting the device in different USB port, if that does not work; check with different computer.
