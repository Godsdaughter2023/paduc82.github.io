rustyrotary say:

After lots of troubleshooting, i've solved it.
Here's how:

1st- I've used a registry entry, thanks to sevenforums.com, that reset the WinHTTP Web Proxy Auto-Discovery Service registry values to it's default values from a Windows 7 build.


2nd- Opened cmd as an Administrator to see if SFC (using the sfc /scannow command) detected any anomalies (which in my case it didn't).

3rd- Restarted my PC in "advanced mode" (Settings >Update & Security >Recovery >Advanced Startup, and clicking the Restart Now button).

4th- Clicked on Troubleshoot, and next, Startup Repair.
Because i've got my Windows 10 install on an SSD, it took around 25min to complete.

5th- After that, i checked back on the Services panel to see if it wasn't grayed out anymore, and it wasn't!
Then i just set it to "Automatic" on the drop down menu, and rebooted.

6th- Then after the reboot my audio drivers loaded up (on the previous boot, they didn't), then i went to the Services panel and started the WinHTTP Web Proxy Auto-Discovery Service and the They both started without any errors.
