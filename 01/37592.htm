techfreak_ say:

Usually the only cause of this is 3rd party updater software. Sometimes this updater software is a feature of your antivirus suite. I suggest you attempt to locate such software and disable or uninstall it.

An example of what occurs:

1. You have Skype 7.x installed.

2. The third party updater software locates Skype 8.x.

3. The third party updater software attempts to install Skype 8.x.

4. The third party updater software or the Skype 8.x installer uninstalls Skype 7.x.

5. If the install is successful, you will have Skype 8.x present on your system.
6. If the install is unsuccessful, you will have no Skype present on your system (except possibly the Skype for Windows 10 App).
7. You reinstall Skype 7.x and you are back to step 1.


The inability to save and retain custom ringtone settings is a known bug in the the later Skype 7.x releases. Due to the focus being on the newer clients, I highly doubt they will release a fix for this bug.

lemp say:

You appear to have purchased a no-name wifi adapter that is said to use a chip from chip manufacturer Realtek.  However, as near as I can tell, Realtek doesn't supply a driver for this chip for download to consumers.

Presumably, whoever the original purchaser bought it from did supply a driver and the seller has uploaded a copy of that driver to his "cloud" storage space at onedrive.com (onedrive is a cloud storage space hosted by Microsoft; you have one of these as well, which you can sign into using the same username -- HOTROD_1975 -- and password you use to sign into this forum).

Yes, you need some sort of unzip program to extract the driver from the compressed file you downloaded (you downloaded the same file at least twice, which is why one copy has the (1) at the end of the name).  If you don't want to pay for the WinRAR program, get 7-zip, which is free:  http://7-zip.org/

Before you use the driver, however, be sure to scan it with a very good antivirus program.  It is probably not likely that someone would go to the trouble of selling you a piece of hardware in order to get you to install malware, but one never knows.

Hi Adrian,

The error code 0x800710d2 appears when the device failed to send any resource data. To immediately resolve the error, we suggest removing the PIN and then adding a new one. To do so, please follow the steps below:

  1. Log in to your computer using your password.
  2. Press Windows key + i to open Settings.
  3. Under Settings, select Accounts.
  4. Under Accounts, locate the PIN section and then click the Remove button.
  5. You will be asked "Are you sure you want to remove PIN?"
  6. Click Remove to confirm.
  7. Enter your account password, then click OK to successfully remove the PIN.

Note: Once you're done, please proceed in adding a new PIN.

Feel free to post back with the result for further assistance.
