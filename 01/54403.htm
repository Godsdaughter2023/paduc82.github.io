To add some additional background, in looking at forums this happens on Windows 10 with Nvidia and Intel HDMI audio as well. From my own personal experience with my home theater PC (with an AMD Graphics Card) that I upgraded to Windows 10, the only reliable work around for me was to create a batch file that would do the following:

1. Disable the AMD High Definition Audio Device

2. Wait for user input ('pause' command)

3. Re-enable the AMD High Definition Audio Device.

This allows you to avoid having to reboot but doesn't address the cause of the problem (which I haven't been able to find yet either). Step 2 was necessary for me because without it, the batch file would fail to re-enable (associated services likely were still stopping).

Using this method also requires the installation of Windows Device Console (Devcon.exe) which can be downloaded from Microsoft. There's a nice explanation here: http://ccm.net/faq/1886-enable-disable-a-device-from-the-command-line
