thebobinator say: When windows XP copies files, it just does it.  Meaning it Renames the NTFS header and calls it a day.

When Windows Vista\7\8 copies files, it calculates the size of what's being moved and plans out the operation before doing it.  There's also file and folder metadata to move.

The advantage to the Windows XP way is speed.  However if you run into a problem; e.g. NTFS permissions, you implode.  The Job stops and your files are strewn in-between directories.

The advantage to the Windows 7 way is reliability, but it can take forever to move 1 freekin' file depending on the mood windows is in.

The solution is to open up the command prompt and use robocopy (Win7 has a copy in the system32 dir) or the move command.

¡firedog say:
My live mail is sorted alphabetically by last name,

It's not clear what you mean by this statement.

  • There is an option in the contacts window to Sort (names) by > Last name - is this what you mean?
  • Or that when you open/switch to the contacts window, you see a list sorted by last name?
  • Or that when you click To... or start typing in the To... box in a message compose window, you see a list (a 'contact picker' list) sorted by last name?

Only contacts with an email address will appear in the 'picker' list. The list consists of Names in alphabetical order. For contacts with both a First Name and a Last Name registered, the list will be sorted according to the preference given at Sort by in the Contacts window. Contacts with more than one email address will appear once for each address, with the address in brackets after the Name.

Name is a special value constructed by WLMail from contact data fields. It will consist of the first match found when scanning data fields in the following order:

  1. Nickname
  2. First Name + Last Name (in the order specified at Sort by in the Contacts window)
  3. First Name or Last Name
  4. Company
  5. E-mail address
As you start typing in the To... box, WLMail will present a list of matching entries in your contacts list. You can start typing a contact's Nickname, First Name, Last Name, Company, email address or any non-trivial word in the Notes field for that contact. So, if you type bil, for example, the list that appears will include
  • everyone whose Nickname starts with Bil, e.g. Billyboy
  • everyone whose First Name starts with Bil, e.g. Bilbo Baggins
  • everyone whose Last Name starts with Bil, e.g. Jim Bilko
  • everyone whose Company name starts with Bil, e.g. Billingsgate Market
  • everyone with an email address starting with bil or with bil straight after a period or the '@' symbol, e.g. 'bilbo at hotmail dot com' or 'jim at billingsgate dot com' or 'jim.bilko at hotmail dot com'
  • every category whose name includes a word starting with bil

If this doesn't address your problem, please post your question in the forum dedicated to contacts in Windows Live Mail.

ASUS Notebook keeps Sleeping/Hibernating (SOLVED) - Microsoft Community

alan111s say:

ASUS Notebook keeps Sleeping/Hibernating (SOLVED) Kernel-Power Event ID 42 Sleep Reason: Application API.

Hi all ASUS Owners and Windows 10 Users

My Notebook is an ASUS N53SV, running Win 10 Pro and the BIOS is on the latest version.

I have been experiencing a very annoying problem. Despite the Power Options being set to 'Put the computer to sleep'='Never', the machine keeps Sleeping/Hibernating even when I'm using it.

In Event Viewer, System Log I get a Kernel-Power Event ID 42 Sleep Reason: Application API. It's worse when the machine is busy and happens every few minutes. This has plagued my machine for a long time, even when it ran Windows 8. I don't think it was an issue under Windows 7.

I have tried every possible fix. I reset the Power Plans. I even took the machine apart to clean the fans etc. in case it was Overheating. I even made a bootable USB running Win 7 and ran the CPU on 100% all night using Prime95 to make sure it wasn't a hardware issue. (Edit: I later found that Cleaning the fans wasn't enough. I replaced the Thermal Paste which brought the temperatures down by 20 degreesC (was 95 degreesC under load, now 75). The basic issue was that Win 7 let the machine run HOT but Windows 10 doesn't. Since replacing the Thermal Paste, I've had no more random Hibernations)


I also removed ASUS Power4Gear (Power4Gear is not needed for Win 10) and upgraded ASUS ATK to Windows 10 x64 Version 1.0.0041. Also, I tried removing ASUS ATK altogether which did stop the random hibernations but the machine would then crash under load (NO BSOD, just shutdown) . So I also tried older ATKs but not all function keys worked. So I went back to 10x64 Version 1.0.0041.

In the end, I discovered that I had set the PC to use Integrated Graphics all the time. My notebook has Intel HD 3000 which is not officially supported for Win 10. So I changed the setting to use NVidia GT540M all the time and it has since completed intensive tasks without hibernating.  I use the Notebook mainly over Remote Desktop or via Teamviewer and this is definitely stopping it 'Sleeping/Hibernating'  ;)

Secondly, I often use Wake on LAN to get the PC to start remotely from Hibernate. There is a hidden Sleep setting called 'System Unattended Sleep Timeout' that was causing the Notebook to hibernate again 2 minutes after I woke it up. I set this to 0 minutes to disable that.

I know there are many posts about this kind of issue. So I started a seperate thread so searches would find this post more easily.

Hope this helps. Please reply if this works for you.

Each Windows Phone is created with an activation code as part of the manufacturing process. In rare cases, a new activation code is required to complete the Microsoft account sign-in process. This is why you're asked to enter a new activation code when signing in with your Microsoft account for the first time.
