considerate_guy say:

Because you have an NTSC camcorder, when it isn't actually playing/recording anything, it sends data down the FireWire that tells the other end that it is NTSC (matching the native format of the camcorder). 

Since that's usually what the camcorder expects to play, this behavior is fine most of the time.  The capture software prepares for NTSC video, starts up the playback and gets what it expects.

However, in your case you are playing PAL video.  Since the capture software was primed to receive NTSC video, it gets confused by the PAL video and you end up with blocky, unreadable video instead.

I believe you are in luck though.  The Sony camcorder is smart enough to detect PAL video and play it correctly.  This is evidenced by the fact you can see it on the LCD screen.  So the only thing you need to do is to somehow trick your camcorder into telling the capture software to get ready for PAL video instead of NTSC video.

I did some research on other forums and it appears this is possible.  The suggestion I read was to put the camcorder in play/pause mode before starting the capture.  If it works, the capture routine should see the format of the video as PAL instead of NTSC and hopefully capture it correctly.

I obviously haven't tried this trick since I don't have the requisite hardware, but I would love to know if it works.

Method 2:

a. Right click on the folder and click on “Properties”.

b. Go to the tab “Security”.

c. Click on the “Advanced” button at the bottom.

d. Click on “Change Permission”.

e. Select your user account and check the option “Include inheritable permissions from this object’s parent”.

f. Click on apply to remove the lock icon.

wunders say:

My experience is that most of these "extra fields" do not exist as fields in the directory (folder) itself.  Rather, these fields exist in the target file itself. Each file type has its own established file format and except for the file types that belong to Microsoft, Microsoft cannot control or arbitrarily change the format of a given file type.

Take a ".jpg" file for example.  The format of this file type allows for a lot of fields such as comment, tag, date taken, aperture, camera manufacturer, and on and on.  If you go to the properties on a .jpg file, you will see a lot of these and will be able to change them.  After changing one of these fields, you will find that the file itself has been edited to contain this information (as can be verified by checksumming the file and coming up with a different checksum).  If you display one of these fields in Explorer and have 10000 ".jpg" files in that folder, then Explorer must open every one of these 10000 files to extract the data in the corresponding field(s) you chose to display.  Might take a while.

Compare that to the same photo saved as a ".bmp" file.  The ".bmp" format does not have provisions for any of these fields, so you will not be able to view, save, change, or sort by them.  So the changeable fields are directly related to the file type that is being displayed.

Microsoft has apparently surveyed a lot of file types and made a compilation of the editable fields in each file type and put the union of selected fields into what can be displayed and changed.   This is apparently what you see in the "More..." option.   So, in summary, the file type determines which fields are available for editing and viewing.

HTH,
