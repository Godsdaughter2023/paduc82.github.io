    I get an iaStorAVC.sys failure with RAID M.2's - Microsoft Community

    id2tall say:

    I get an iaStorAVC.sys failure with RAID M.2's

    I have read the previous post about this failure but my situation is a bit different.

    I have Z390 motherboard with 2 NVMe drives in RAID 1. I know know I need the drivers installed for Windows to boot, but I never get far enough to add the drivers. Here is the problem:

    I have a Windows 10 USB Boot drive.

    I set my BIOS per the Manufacture, save and exit.

    Windows begins to boot by showing the spinning dots.

    After a couple of minutes I get a BSOD with the iaStorAVC.sys fail.

    I never get to the install screen that would allow me to add drivers.

    I know the USB is good, because if I un-RAID the drives and try to boot off of the USB it works fine!

    How can I get Windows to install?

    id2tall say:

    I finally figured out this issue on my own.

    On a separate system, I had to copy off the boot.wim file from my bootable thumb drive and then Dismount it. Then I had to add the iRST drivers to the file and remount it. Once I put the new boot.wim file on the thumb drive, Window booted and installed perfect.

    Windows needs to update their 1809 download to include the latest iRST drivers or a lot of people who RAID their boot drives are going to be stuck!
