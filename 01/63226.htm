While I can't answer your questions directly, if you are ever in a spot where DISM is not able to repair WinSxS, you can perform a repair install, which keeps all files and programs you have installed.  

  • To perform a repair install, run Windows setup from within Windows and upgrade your install to the same version of Windows that's currently installed.
    • A repair install essentially replaces all system files within the Windows directory.  All installed programs remain untouched.

jw0914 say:

A repair install replaces all system files contained within the Windows directory, and a handful of directories within Program Files. A repair install should always be the last option, as it does rewrite certain registry entries related to the user profile hives.  This means certain settings and customizations will be reset to default, however it will rarely fix a problem that's resulting from manual registry edits/deletions since only certain keys are overwritten and reset to default.

A repair install will remove any updates not already injected into the install.wim/install.esd, as it overwrites the WinSxS folder and all files/directories contained therein.  

  • The WinSxS folder (C:\Windows\WinSxS) contains a backup (in the form of cab files) of all system files within the Windows directory.  This is what DISM's /cleanup-image argument checks against the Windows Update servers to ensure parity.
  • SFC compares all system files to the original backups in the WinSxS folder and if a parity mismatch is found (hashes don't match), SFC deletes the file and replaces it with the original located in the WinSxS folder.

This is why it's recommended to use the media creation tool to create ISOs and not rely on old ISOs, since the media creation tool's ISO image either contains all updates up until the past month or the past patch Tuesday.  If you want to use an old ISO, you can inject all past updates into the install.wim/install.esd using DISM, as, every month, Microsoft releases an ISO containing all updates from the previous month.  There are also several programs that one can use to auto download and auto inject updates.

Hi

Wire or Wireless does not matter for this purpose.

The Drive has to be set as Shared Device, and then other computers that are on the same network (wire or Wireless) can access it.

Here how.

Win7 when configured on peer-to-peer Network has three types of Sharing configurations.

HomeGroup Network = Works only between Win 7 computers. This type of configuration makes it very easy to Entry Level Users to start Network sharing.

 Work Network = Basically similar to the previous methods of sharing that let you control what, how, and to whom folders would be shared with.

 Public Sharing = Public Network (like Internet cafe) to reduce security risks.

 For best results log to each computer's System Screen and set all the computers to be on a Network with the same name while each computer has its own unique name.

http://www.ezlan.net/Win7/net_name.jpg


Make sure that the Software Firewall on each computer allows free local traffic. If you use 3rd party Firewall On, Vista/XP Native Firewall should be Off, and the active Firewall has to adjusted to your Network IP numbers on what is some time called the Trusted Zone (consult your 3rd Party Firewall instructions

General example,  http://www.ezlan.net/faq#trusted

Please Note that some 3rd party Software Firewalls keep blocking aspects of Local Traffic even it they are turned Off (disabled). If possible configure the Firewall correctly, or totally uninstall it to allow clean flow of local network traffic. If the 3rd party software is uninstalled or disables, make sure that Windows native Firewall is On .
