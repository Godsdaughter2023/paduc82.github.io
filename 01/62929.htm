rdhw say:

This issue is nothing to do with Office needing repair. The issue is more accurately described like this:

After the Windows 10 1803 update, no application (not just Office) is capable of printing any document that references OpenType fonts with CFF outlines, if the printer driver is a "Type 4" printer driver.

This issue was not one of those fixed in the June 12th update for v1803, and remains live at the time of writing.

Note that not all OpenType fonts have CFF outlines: some have TrueType outlines, and they are not affected by this issue. Also unaffected are TrueType fonts (.ttf), such as most of the Microsoft ones. It is not easy for an end-user to know whether their OpenType fonts (.otf) have CFF outlines or TrueType outlines, but almost all OpenType fonts from Adobe (for instance) have CFF outlines. The forums at Adobe show that Adobe are very well aware of the issue in v1803.

The issue (it is claimed) does not affect "Type 3" printer drivers. Again, it is quite tricky (but not impossible) for end-users to discover whether their printer drivers are "Type 3" or "Type 4".

I was fortunate that my HP laser printer was supplied (by HP) with alternate "Type 3" and "Type 4" drivers, so I was able to switch drivers to verify that I was unable to print documents having CFF outline fonts with the "Type 4" driver, but was able to print the same document with the alternate "Type 3" driver for the same printer. Nothing was changed except for the selected printer driver in the Printer Properties->Advanced.

The issue also affects some (but not all) virtual printers.  For instance, the "Microsoft Print to PDF" virtual printer is Type 4, and fails to generate PDFs from any application if CFF-outline fonts are involved. However, if you have Adobe Acrobat installed, then the "Adobe PDF" virtual printer is Type 3, and will succeed in producing valid PDFs, even from applications using OpenType fonts with CFF outlines.

So, this offers a work-around for users affected by this issue and unable to find a Type 3 driver for their printer:

- acquire a PDF-generating virtual printer of the Type 3 variety: Adobe Acrobat provides one, and there are other free offerings on the web (take care, as usual, that you do not download malware-infested fakes);

- use that virtual printer to generate PDFs from your applications, and ensure that the option is selected to embed all fonts in the PDF;

- using any applications that can open PDFs, then print the PDF to the physical printer, even if the printer driver is Type 4.

This works because, once the CFF-outline fonts are embedded in a PDF file, they no longer trigger the bug in the printer driver that is triggered by the original CFF-outline fonts on disk.
