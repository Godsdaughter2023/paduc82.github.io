¡firedog say:

That's an unusual error - I don't think I've ever come across it in Windows Live Mail. The problem arises in Windows, so my suggestions would be to reset Winsock, TCP/IP, DNS and SSL and see if that helps. It won't hurt!

Here's a little batch file that will do this for you. Copy the text in the box, then paste it into a blank Notepad document. Save it as netwkfix.bat somewhere where you can easily find it again. Close down all running programs, then double-click on the file you just saved to run it.

:: Check for permissions
>nul 2>&1 "%SYSTEMROOT%\system32\cacls.exe" "%SYSTEMROOT%\system32\config\system"

if '%errorlevel%' NEQ '0' (
    echo Requesting administrative privileges...
    goto UACPrompt
) else ( goto gotAdmin )

    echo Set UAC = CreateObject^("Shell.Application"^) > "%temp%\getadmin.vbs"
    set params = %*:"=""
    echo UAC.ShellExecute "cmd.exe", "/c %~s0 %params%", "", "runas", 1 >> "%temp%\getadmin.vbs"

    del "%temp%\getadmin.vbs"
    exit /B

    pushd "%CD%"
    CD /D "%~dp0"
:: --------------------------------------

:: Clear the SSL cache
rundll32 "C:\Windows\system32\WININET.dll",DispatchAPICall 3

:: Clear the DNS cache
ipconfig /flushdns

:: Renew DNS client registration and refresh DHCP leases:
ipconfig /registerdns

:: Reset the TCP/IP stack to default
netsh int ip reset reset.txt

:: Reset WINSOCK entries to default
netsh winsock reset catalog

@ECHO Remedies applied. Please restart the computer.
