marilyn-- say:  Hello drolston,

 

Thank you for posting to the Microsoft Answers Forum.

 

The Blue Screen error you are see 1000007f is UNEXPECTED_KERNEL_MODE_TRAP_M. This means a trap occurred in kernel mode, and it's a trap of a kind that the kernel isn't allowed to have/catch (bound trap) or that is always instant death (double fault). The first number in the bug check parameters is the number of the trap (8 = double fault, etc).

 

Most of the time bug check code 7F is caused by hardware error. Usually it is because of faulty ram. Run memtest to stress test the ram. You could have corrupt or mismatched memory that is causing problems.

 

When Windows crashes with blue screen, it writes a system event 1001/1003 and a minidump to the folder \windows\minidump. Check system event 1001 and 1003 and see if it has the content of the blue screen.

 

To get to the event viewer:

Go to Start Button, right click on Computer, select Manage.

If you get the User Account Control Dialog box, click Continue.

On the Computer Management Console, in the left pane, double click on Event Viewer. This will bring up a Overview and Summary of the event logs. Check for the event id's 1001 and 1003. You can verify if you are getting the same parameters for

the Blue Screen.

 

There is another quick way to bring up the Event Viewer. (This may not work in Vista Home Basic or Vista Home Premium)

Go to Start Button.

In the Start Search box, type in "eventvwr.msc".

If you get the User Account Control Dialog box, click Continue.

This will bring up the Event Viewer Overview and Summary logs.

 

Please report back with the results of the memory test and if you need further assistance to help with troubleshooting.
