If you have a computer with an OEM pre-installed version of Windows XP, you almost certainly have a folder named C:\i386 that includes the versions of the necessary files (in compressed form) that were valid for the service pack level of the original installation.  Although you could probably expand those files and/or updated versions in c:\windows\servicepackfiles\i386, your best bet would be to download the service pack 3 versions.

I have just fxsapi.dll and fxscfgwz.dll in my Skydrive --> http://sdrv.ms/11zIqFJ but I suggest that you download the entire collection (XPFax.zip) from the Skydrive of frequent contributor Jose Ibarra (aka ElderL) --> https://skydrive.live.com/?cid=6a7e789cab1d6f39&id=6A7E789CAB1D6F39!311

If you download the zip file, just unzip it (right-click and "Extract all")into a folder and then point the Add Windows Components wizard to that folder during the install.

NOTE:  if you look at the contents of the zip file or the contents of C:\i386, you'll find that the file names all end with an underscore (e.g., fxscfgwz.dl_).  This means that the files are compressed and need to be uncompressed (and renamed) using the EXPAND command in an Command Prompt window.  http://support.microsoft.com/kb/888017

Is the printer connecting directly to the computer by wire or wireless or is it a networked printer?

 

Print problems –Microsoft Fixit Tool
http://support.microsoft.com/mats/printing_problems/en-us

 

1. Click Start, type “services.msc” and hit Enter.
2. Scroll down to Print spooler.
3. Right-click Print spooler, then select Properties.
4. Select the Log On tab, and uncheck the Allow service to interact with desktop setting.

 (Microsoft recommends that you do not change the Allow service to interact with desktop setting. If you allow the service to interact with the desktop, any information that the service displays on the desktop will also be displayed on an interactive user's desktop. A malicious user could then take control of the service or attack it from the interactive desktop).

5. Once you have uncheck this setting, select the General tab and click "Start" (the Print Spooler should be able to start without popping up the Error 1053 again).

mike hall... always say:
Set the mic on the headset as the default mic for the system..

Right click on the sound icon in the notification area.. select 'Recording Devices'

Set the mic which you want as the default

If your headset is the type than connects via regular jacks, set the sound card as the default.

If you are using a USB headset, you can only set this as default for as long as it is connected. The system will look for another mic (and sound outlet) if you disconnect the headset.
