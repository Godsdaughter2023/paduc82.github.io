wurlitzer1015 say: Well finally found the cause of the problem, there are quite a few threads around the web with this problem, none with a satisfactory conclusion so I thought I'd post this for a solution that should help quite a few...

At the crux of the problem is WORKGROUPS and how it inter-acts with other other non-windows devices on the WORKGROUP network.  Samba (smb) is used to provide the connectivity between devices on the network but windows machines become very confused if those non-windows devices are configured to act as the "master" and any windows PC has the "Computer Browser" service running. This results in the problem I was having with no network devices showing up, although they could be contacted via IP address or name.  It was all down to a conflict that Windows couldn't resolve in this scenario and resulted in no network devices being returned by the "Computer Browser" service.

In this situation the following actions resulted in the network showing the devices in explorer, either of the two solutions below works, choose whichever suits you best.

1. Go to each windows PC and STOP "Computer Browser" under CONTROL PANEL-->ADMINISTRATIVE TOOLS-->SERVICES and also "disable" it from it's properties. Don't just change it to manual, WORKGROUPS will start it.  You must do this on ALL windows PCs in the WORKGROUP.

2. Alternatively, go to each non-windows device (in my case linux boxes and a Mac), login as root using ssh or telnet and navigate to /etc/samba/ , edit smb.conf and change the following lines from "yes" to "no"

preferred master = no
domain master = no
local master = no

Probably also best to check the windows machines and only have one running "Computer Browser" although multiple PCs running the service do seem to be resolved OK by WORKGROUPS.

So, as simple as that, would be nice if Microsoft came out of their shell once in a while and realise that other devices besides windows machines run on WORKGROUPS and didn't get so confused when many try an assert themselves as the preferred master...
