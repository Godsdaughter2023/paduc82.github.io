This opens the "Network connections" portion of the Control Panel.  Right-click on the icon for your Ethernet adapter and select "Properties."  Please list all of the items shown in the box under "This connection uses the following items."  Sorry, I know of no easy way to copy/paste information from this list.

The lights mean that there is a connection between the router and the computer. 

The solid green is what I would expect.  This indicates a 100 Megabit/sec (in theory) connection.  Although the network adapter in your computer is a "gigabit" device (meaning it is theoretically capable of 1000 Megabits/second), the router's Ethernet capability is 100 Megabits/second, so that's the maximum theoretical speed of the connection.

The blinking green light is also normal.  That indicates brief periods of network activity as the computer and router attempt to communicate with each other.

So if we assume that the hardware is OK, the question is:  "why is the router's DCHP server failing to allocate an IP address to the computer"?

With an Ethernet connection, there's no issue of encryption problems; unlike wifi, Ethernet traffic is not encrypted.

There are only three possibilities that I can think of: 

  1. There's a firewall blocking communication between the computer and the router
  2. There is a filter configured in the router that is blocking the connection to your computer
  3. The router's DCHP server is broken

I wrote a fair amount of complicated directions below in an attempt to figure out which of the above 3 possibilities might be the issue ... and then realized that none of it is really likely to solve your problem.  If it's understandable, by all means go ahead and see if you discover anything that is blocking your access.  I'm afraid, however, that what you'll have to do in the end is to take your computer AND your router (but not the Huawei modem) to an independent computer repair shop.  I advise against going to a chain store that sell computers and advertises that it also repairs them.  Ask your friends and fellow computer users for recommendations for a knowledgeable local repair shop.

Sorry I couldn't be more help.

********************************************************************************************************


I think we can easily eliminate 3, because you said that your phones connect via wifi through the router to the Internet and a laptop connected via Ethernet.  This requires that the router assign each of these devices an IP address, and thus the router's DCHP server must be functional. 

Possibility number 1 would seem to be the most obvious, but you said that AVG is not active.  Nevertheless, which particular AVG product do you have?  You also said that you restored your computer to its factory settings.  Typically, this will also install any free trial software that may have come with the computer.  Open Control Panel > Programs and Features.  Look through the list carefully.  Is any security software listed there?  Is there anything there that you don't recognize?

It's highly unlikely that the Windows firewall would block things this way, but you can check this:

  • First disconnect the cable between your modem and router.  This will ensure that you don't connect to the Internet
  • Open Control Panel and type "firewall" (without quotes) in the upper right search box
  • Click "Windows Firewall"
  • Click the link to "Turn Windows Firewall On or Off"
  • Click the two radio buttons to turn off the firewall and then click OK
  • Open a Command Prompt window and run an ipconfig /all command
  • Did the IPv4 address change from 169.254.x.y?  (probably not)
  • Go back and turn the firewall on for both types of network (the default is the lower checkbox in each case -- to notify you when a new program is blocked).  Don't forget to click OK.
  • Reconnect the cable between the modem and the router

If there is no security software active and Windows Firewall is not blocking the connection, the only thing left is some configuration in the router that is blocking the connection to your computer.  This seems unlikely because you said that you had no issues initially.  However, you can check this through the router's web-based configuration utility.  You'll need to use a computer that can connect to the router, such as the laptop you mentioned earlier.  First use that computer to get the manual for your router here --> http://www.dlink.com/-/media/Consumer_Products/DIR/DIR%20810L/Manual/DIR_810L_RevA_Manual_EN.pdf

Log on to the router by opening a browser (e.g., Internet Explorer, Firefox, Chrome) on a computer connected to the router and type either of the following as the address you want to go to:

      192.168.0.1

      dlinkrouter.local

Enter Admin as the user and leave the password blank (unless you changed these default values). See manual page 20.  You should see the main menu as follows.  Click "Network settings"

Find the "DHCP Server Settings" box.  See manual p. 55.  "Enable DHCP server" must be checked.  "DHCP Address Range" should include enough numbers for the maximum number of devices that might be connected simultaneously.  For example, for your computer, two phones, a laptop, and a spare, you'd want at least 100 through 104.  More is OK.  The default of 100 through 199 is fine.

Normally, the "DHCP Reservations" section (manual p. 57) should be empty unless you have printers connected to your network.  If so, you would want to reserve an IP address for each printer so that the printer always gets the same address, even if it has been powered off for a long time.

Somewhere in the "Network settings" section you should find the "Network filters" section.  The manual isn't clear on how you get there, but the section itself is described starting on page 82.  Unless you have a good reason otherwise, make sure that "MAC Filtering" is OFF and that "Access Control" (page 83) is NOT enabled.

a. user say:

Hi,

 

If the apps and settings that you have setup on Gmail account is only for that Microsoft account/user account and not for others, then, removing the account will remove the apps and settings as well.

 

To tranfer Windows or user account/Gmail account files to the Outlook user profile, follow the steps:

a) Press Windows and X together and select Windows Explorer.

 b) Click on View tab and click on Options at the top right corner.

 c) Click the View tab, and then click Show hidden files, folders, and drives.

 d) Uncheck the box which states Hide protected operating system files.

e) Click Yes to confirm, and then click OK.

f) Locate the C:\Users\Old_Username folder, where C is the drive that Windows is installed on, and Old Username is the name of the profile you want to copy files from.

g) Select all of the files and folders in this folder, except the following files:

•Ntuser.dat

 •Ntuser.dat.log

 •Ntuser.ini

 h) Right click and Select Copy.

 i) Locate the C:\Users\New_Username folder, where C is the drive that Windows is installed on, and New Username is the name of the new user profile you created.

 j) Right click inside the folder and select Paste.

 k) Restart your PC and check if you are able to log in to default profile.

 

You can download and install same apps from App store for the Outlook account. Refer to the link:

http://windows.microsoft.com/en-in/windows-8/apps-windows-store-tutorial
