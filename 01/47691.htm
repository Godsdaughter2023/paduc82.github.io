Unknown device in Device Manager after reset - Solution - Microsoft Community

7thsense say:

Unknown device in Device Manager after reset - Solution

Technical Level : Expert

Summary
When you reset a Surface Pro 2 to factory settings, an unknown device (that can't be repaired) will appear in the device manager. Here is a solution that will fix this issue.
Details

Step 1:

Reset your Surface Pro 2 back to factory settings (Change PC Settings > Update and Recovery > Recovery > Remove everything and reinstall Windows).

Step 2:

Login to Windows with your local- or Microsoft account. Note! Do not apply any Windows Updates yet.

Step 3:

Download the latest Surface Pro 2 Driver Pack from Microsoft and extract to a folder on your local hard drive:
http://www.microsoft.com/en-us/download/details.aspx?id=38826

Step 4:

Create a text file in the top level root folder of the extracted driver pack with the following script. Make sure to rename this file with a .PS1 extension and not .TXT.


$ScriptPath = Split-Path -parent $MyInvocation.MyCommand.Definition
$files = get-childitem -path $Scriptpath -recurse -filter *.inf
foreach ($file in $files)
{
    Write-host "Injecting driver $file"
    pnputil -i -a $file.FullName
}

Step 5:

Open Powershell as admin (right click and choose to Run as administrator).

Step 6:

Run the command:   set-executionpolicy UNRESTRICTED and choose Yes to confirm.  This will allow you to run the unsigned powershell script you created.  If you want to lock this back down later you can run set-executionpolicy RESTRICTED after you run the script in step 8.

Step 7:

In the admin powershell window navigate to C:\FOLDERWITHDRIVERUPDATES\ by using the CD command.

Step 8:

Run ./scriptyoucreated.ps1  make sure to have the ./ before the name of the script file. All drivers from the pack will then be installed.

Step 9:

When it is done, reboot your Surface Pro 2 and perform all Windows Updates.

That's all. You are done and the unknown device should be gone.
