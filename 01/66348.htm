Mohammed, I have 3 questions or recommendations.

1. What was the error code in your trial 3 - Store uninstallation and re-installation? Can you show the result of Get-AppxLog -ActivityId xxxxx that PowerShell will tell you to execute to see more details of the error? When you try the Store installation, I believe that PowerShell should be run as not administrator, if your account is not an administrator, because apps are installed for each user account.

2. Can you check the dependencies of your Store App and compare with the files that are installed in your device?

The dependencies of Store App of an account is checked by this PowerShell command-let. Again take care that you run PowerShell as not administrator, if your account is a standard user, because the dependencies could be different among users.

(Get-AppxPackage *WindowsStore).Dependencies.PackageFullName

Those pacakges usually must exist in the C:\Program Files\WindowsApps folder. The contents of the folder can be checked by PowerShell ls command like the following. In this time, the PowerShell must be run as administrator to obtain the privilege to access that folder.

ls 'C:\Program Files\WindowsApps\*WindowsStore*'

3. Since the store failed even in a new account in your case, I'd also recommend to check the PackageUserInformation of Store app by the following PowerShell (run as administrator) command-let.

(Get-AppxPackage -AllUsers *WindowsStore).PackageUserInformation.UserSecurityId

In case that the information contains an already-deleted user (shown with username as sid), I experienced that the provisioned package failed to update apps. As far as I know, in this case, the workaround of this issue is removal of the provisioned package or Windows recovery installation.
