¡firedog say:
On occasions, when I drag file from inbox and drop in a storage folder, the transferred file overwrites the content of the other files in the folder but the subject title remains correct.


This may be what appears to happen, but it doesn't happen in fact.

When you 'move' a message from one folder to another in WLMail, all that happens initially is that the message database records a new location for the message. The message file itself may not be moved immediately, depending on how busy the program is. When you select a folder to view, the program displays in the message list all the entries from its database that have that folder recorded as their location. If you have the reading pane visible, or you open one of those messages, the program will dig into the message store to retrieve the contents of the message from its .eml file and display them.

The problem you see occurs when the indexing process is incomplete. The program can't find the message content and shows either a blank message body or, in some circumstances, the body of some other message. It's not clear why this sometimes happens, but the most likely causes are
(a) the program is too busy doing other things like polling for new messages on the mail server, or
(b) security software (e.g. anti-virus) is slowing up the process by insisting on intercepting the files involved and scanning them before it allows them to be written to disk.

To prevent it from happening, or at least make the problem less frequent,

  • Click Work offline before moving messages around. This will minimize problems caused by (a).
  • Don't move or copy messages from Quick views. Find them in their own folders first.
  • Allow the program time to settle down after moving multiple messages around.
  • If you see symptoms of incomplete indexing that don't go away within a few minutes, close the program down. WLMail will then do its housekeeping after the program interface has disappeared. Allow it time (five or ten minutes) to finish this before closing down Windows or re-launching the program.
  • In your security software, create an exception to exclude the message index file from scans. This is only an index, so it's unlikely to be able to harbour anything nasty. The file to exclude is Mail.MSMessageStore and it lives in the store folder, by default at
    C:\Users\{user}\AppData\Local\Microsoft\Windows Live Mail 
    This file can grow to be very large (mine is over a gigabyte), so scanning it every time a minor change takes place is an unnecessary strain.

As a matter of interest, which anti-virus are you using? Do you have any other software running that might be interfering? This would include any sort of 'optimizer' or 'tune-up' or 'system care' program that purports to do things better than Windows itself.
