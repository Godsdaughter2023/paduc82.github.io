WLMP stands for Windows Live Movie Project. The file contains the information of the media files used in the project. That media files are defined by a codec used to encode them. If that codec is not supported either by Windows Movie Maker or by Windows Media Player (or any other media player), the video/audio/both does not work correctly. To solve the issue, either convert the output file or the original media file to a more known format, like H.264 or WMV and then try. There are a variety of free media converters available like FormatFactory, Handbrake, etc.

frederik long say:

This is going to be a little painful . . .

1. Set your BIOS so that it uses the DVD drive as its primary boot device.
2. Boot the machine with your Windows 7 Repair CD.
3. Press a key if prompted to boot from the DVD.
4. Select "Safe Mode Command Prompt" when prompted.
5. Select Repair when prompted.
6. Select System Restore when prompted.
7. Set Windows back to a point before you entered the password that you have since forgotten.
8. When finished, remove the CD before rebooting the system.

In Step 2 you could also use your Windows 7 installation DVD. The subsequent steps are similar to the ones above.

It would also be a good idea to plan ahead and create a second account, with admin privileges, that you could use if you have a problem with the first account. Provided, of course, that you keep its details recorded in a safe place.
