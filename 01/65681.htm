timothyjmcgowan say:

If you have more than one program window open, and if they're not full-screen, it is sometimes helpful to know at a glance which window is the one with focus.

By "focus," I mean the activated window, the one at the top of the Z order, the one in the foreground, the one that's going to accept keystrokes from the keyboard.

Normally, that window will be the one in front, but if your windowed programs are not overlapping, you can't tell which one is set to receive keyboard keystrokes until you start typing. Very occasionally, a window will appear on screen in front of the window that is set to receive keystrokes. (This can still happen even when your program windows are all full-screen, by the way.)

In earlier versions of Windows, since Windows 2.0 or 3.0, the user could tell which window had focus by looking at the title bars of the windows. The window with focus would have a unique color; all other windows would share a different color.

If you are in the habit of always clicking a program window with your mouse before typing, or if you only ever use one program at a time, you'll never see the issue. But those who use Windows as it was designed, to run multiple programs at once, find it necessary to know which window has focus. Microsoft did us no favors by removing the only visual cue. We need to have that back.

wunders say:

First of all, I assume that you have XP Pro and not XP Home.

This behavior is usually set by group policy.  If you have administrator privilege, you can start up the Group Policy Editor (Start -> Run -> "gpedit.msc") and navigate to:

  User Configuration -> Administrative Templates -> Control Panel

and on the right side, double-click on "Prohibit access to the Control Panel" and change it to "No".  If you find that it is grayed-out and/or you can't change it, that would indicate that there is a Domain Policy in effect that will not permit this operation.

 

You *might* be able to affect this behavior by using Regedit to delete the following registry value:

  HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer value=NoControlPanel

but if it is set by Domain Policy, the effects may be temporary at best.  The usual warnings about using Regedit apply...  make sure you have a backup of your registry.

 

HTH,

  JW
