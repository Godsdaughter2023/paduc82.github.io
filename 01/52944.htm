I've been suffering from the same problem since November: "There was an error while creating a new profile for user. Profile exists but no profile path could be found.: Cannot create a file when that file already exists."

In fact I think my computer had been silently downloading the update every day, failing to install it, rolling back - and then trying again the next day. Thankfully I have no monthly broadband limit!

However the good news is I have finally found a fix - for me anyway.

Here's what I did. I don't know which of these steps fixed it. All I know is that #4 on its own wasn't sufficient:

  1. Checked all the entries in HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList were correct.
    1. In particular, I checked there were entries for all the users on the machine.
    2. There were also 3 "magic" users (S-1-5-18, S-1-5-19 and S-1-5-20).
    3. I checked that all the ProfileImagePath entries were valid paths. Note you need to be running with administrator privileges to check the "magic" users. And because you can't run Windows Explorer with admin privileges, the best way is to open a command prompt window "As Administrator", and use the trusty "dir" command.
    4. These all looked ok, so nothing to fix here.
  2. Deleted the spurious "Default.migrated" sub-folder from my "C:\Users" folder. Actually I wonder whether that folder was causing the "Cannot create a file when that file already exists" error? If you have one of these, I believe it's safe to delete, but don't take my word for it!
  3. Download the fix for KB4057144 from https://catalog.update.microsoft.com/v7/site/DownloadInformation.aspx. There were 2 possible versions there for x86. I (arbitrarily!) chose the larger one.
  4. Stop the "Windows Update" service and renamed c:\Windows\SoftwareDistribution to SoftwareDistribution-old
  5. Unplugged my network cable so Windows Update couldn't start downloading anything while I was manually installing the update.
  6. Started the "Windows Update" service again.
  7. Ran the fix for KB4057144 which I had downloaded earlier.
  8. My computer rebooted a couple of times during the update.

And instead of the update saying it had a problem and rolling back, it actually worked :-) Happy me!

I'm now updating to "Build 1709" using Windows Update. If that fails in a similar way, I'll try the above again.
