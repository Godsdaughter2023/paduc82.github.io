ajay k say:

Hi granniewilsey,

 

This problem occurs because the backup process detects that system drive is dirty.

Windows Vista Backup always includes the system drive as part of backup. If the system drive is dirty, the backup process skips the system drive, and no snapshot of the system drive is created. Therefore, the backup cannot locate a snapshot of the system drive later, and the error occurs.

 

To resolve this issue, run the disk check to repair the error on the system drive. To do this, follow the steps mentioned in the article below:

 

Error message when you try to perform a system backup operation on a Windows Vista-based computer: "Catastrophic failure (0x8000FFFF)"

http://support.microsoft.com/kb/973558


Note: CHKDSK would try to recover bad sectors on the hard disk, during this process there are chances of DATA loss from that particular sector.

The reason you can't find gpedit.msc is because that utility is not available in Windows 7 Home Premium; it's in Windows 7 Professional.

Windows versions since Windows 2000 (including XP, Vista, and 7) have been based on Windows NT, which is why there are references to Windows NT in the registry.  Although you are smart to hesitate before making registry edits that you are not sure of, many Windows 7 registry entries are indeed in a "Windows NT" key.
