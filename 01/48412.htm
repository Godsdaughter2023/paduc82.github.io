mike hall... always say:

It has never been a good idea to get hardware drivers from Windows Update unless specifically told to do so by the hardware device manufacturer.

 

The trouble with Windows Update re hardware is that it appears to collect hardware info in much the same way as driver finder utilities do.

 

"Ooh look, a SoundBlaster device. We have a driver for them" 

 

Unfortunately, it may not be the driver for your specific SoundBlaster product, maybe for a different hardware revision, and so it invariably fails or causes grief at every reboot.

 

The secret when upgrading an OS is not to take the silly minimum recommendations seriously. The biggest issue when upgrading is not the speed of the processor or RAM as long as it is more than 2gb. Driver support is the big issue.

 

As long as a low powered computer gets FULL driver support for ALL of the hardware, it may not be close to a Chess challenge, but it will perform as the maker originally intended.

 

So, to your issues..

 

Run with the Windows 8 video driver which installs by default. You can roll back to it by going into Control Panel > Device Manager > Display > and right clicking when there.

If you want to see if NVidia have a better driver, go to NVidia driver support..

http://www.nvidia.com/Download/index.aspx?lang=en-us

 

Sound is a tougher issue because it always has been. Still, you should get drivers directly from Creative..

http://support.creative.com/

If Creative do not have Windows 8 drivers for your soundcard, contact Creative and complain.

I suspect that re-installing Windows will not help you. The problem is probably caused by older applications whose installation routine cannot handle numbers above 2 TBytes. They get confused and report the ridiculous number you see. Here are a couple of ways go work around the issue:

  • Create a folder C:\LegacyApps\App1, or
  • Shrink your E: partition by one GByte, then create a new partition F: of one GByte. Create the folder F:\LegacyApps\App1 in it.

In each case you then create a symbolic link in E:\Program Files to the selected folder on drive C: or F:, e.g. E:\Program Files\App1 -> F:\LegacyApps\App1. Your legacy application will visible on drive E: but the installation program will see the free space of drive F: which should be well within its allowable range.
