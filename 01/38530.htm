After spending 2 hours with Microsoft online chat, we were not able to find a solution. Seeing their approach though made me try my own, and I have a fix.

1. Create a new Microsoft account.

2. As an administrator on your computer, go to START > Accounts > Family & other users tab. Click the "Add someone else to this PC" button. Insert the new Microsoft email address, and finish the new user creation process.

3. On the same "Family & others" window, click the new user at the bottom and set them as an administrator.

4. Go to START > Your Account name (very top of menu) > Sign out. Now login as the new user.

5. Open File Explorer and navigate to C:\Users. Rename your original account folder (the one for your old Microsoft account) and add .bak to the end of it.

6. Open the registry by press Windows + R and then typing regedit

7. Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList.

8. Move through each registry subfolder until you find the one whose ProfileImagePath key maps to the folder you just renamed. This is the profile entry in the registry for your messed up account.

9. Delete the entire subfolder whose SID (e.g., S-1-S-21-1234567890) matches your old Microsoft account/username. DO NOT DELETE ANY OTHER FOLDERS OR KEYS.

10. Reboot

11. Log back in as the new created user again. Go to START > Accounts > Family & other users tab. Click your old account at the bottom and select remove. Removing this account will not remove your C:\Users\{my user}.bak folder (my .bak folder stayed, but if you'd like to back it up you're welcome to).

12. Click the "Add someone else to this PC" button. Insert your OLD Microsoft email address, and finish the new user creation process. Set this account to be an administrator too.

13. Go to START > Your Account name (very top of menu) > Sign out. Now login as the new user that relates to your OLD account.

14. Click the Cortana bar and go through the initial setup. It should now function properly.

15. To get all of your files back, open File Explorer and go to C:\Users\{old name}.bak. Select all of the folders and copy them (I didn't do AppData [hidden] for fear it might affect Cortana) . Now go to C:\Users\{new functioning account} and paste those folders inside. Say yes to replace everything. All of your folders/files should appear now. You will need to reconfigure your START menu, OneDrive, email, etc.

16. Also, make sure to go back and delete the account that you made using the new Microsoft account.

Hope this works for you guys!

UPDATE: I did not test this to see if it was persistent after a reboot. Unfortunately even the new user migration doesn't fix the issue.

There's no setting for it.  But Windows is different than other operating systems that use window managers. It's up to the program itself to keep track of and decide where it should draw itself on the screen. So some programs may be more consistent than others, depending on how much time and thought the builder put into programming it and considering such scenarios.  If they didn't consider it, the behavior is essentially random because there's no rules or instructions for it to follow.

a. user say:

Hi,

Thank you for posting your query in Microsoft Community. I understand the inconvenience caused to you and will assist you in solving the issue.

I suggest you to reset the Windows Update Components manually by following the steps provided below:

1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

5. Type Exit in the Command Prompt to close it and then restart the computer.

6. After restarting the computer, now retry to install Windows Updates.
