congo harrison say:

Here is a solution that I use.

I have given up on trying to install windows to a RAID-0 array with 64k clusters,

I don't think it's possible using normal Win10 install media, because the installer

detects 64k clusters and reports an error, halting the installation.

To prep - copy the relevant raid driver to the install media usb drive, or a separate

usb drive, you may need that for the Windows installer to see the raid array, and you

load it during setup when you see the option to do that.

Once the raid drivers are loaded, go ahead and install windows normally.

I would go ahead and install all your drivers, clearing out any flags in device manager.

Next, I use third party partitioning software to change the cluster size on the array

to 64k, or use any method (DOS ?) to achieve this. What ever method or software

you use, it cannot be destructive, or you lose your windows install. I use EaseUS

Partition Master after trying and failing with several other types of software.

I also do this before filling the drive with data, and the operation is quick and easy.

I believe EaseUS Partition Master will "lock" the drive and complete the operation

when the disk is "offline", but I prefer to have EaseUS Partition Master installed on

another Windows drive, and then operate on the new raid array from that system,

that way I have a windows environment to operate from and I can oversee the

conversion "live" at my desktop, and I know the Windows Raid array is offline since

I booted from the other drive.

The above may sound complex in explanation, but it is really quite simple, and it's

the ONLY solution I've come up with after years of researching this problem.
