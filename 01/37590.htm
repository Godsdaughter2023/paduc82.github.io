Hi IrishBug

I would not recommend turning System Restore off on the System Drive.

If you turn System Restore off, you also lose the Previous Versions shadow copy capabilities. There are many other components, such as Windows Update, and other installers that depend on the capability to create a Restore Point before installing which may cause problems.

Actually your 128GB SSD is much larger than what most users allocate for the System Volume. You should not have any disk space problems with a drive that large.

You can easily control the disk space that System Restore uses. I would recommend setting it at around 3GB, which is usually enough space for a couple of restore points. Here is a link to a short tutorial that describes how to do this.

How to Change the Disk Space Allotted to System Restore in Windows 7

You also want to run the Disk Cleanup on a regular basis to clean up temporary files and keep the system volume as small as possible. Delete files using Disk Cleanup

Also, have you moved all of your personal data to the D: drive? This can easily free up a ton of space on a typical system. All of the standard user folders, such as My Documents, My Pictures, My Videos, My Music, etc, can all be easily moved to your D: drive.

Simply create a folder on the D: drive for each one of these folders. Right click the folder and select the Location Tab. Click the Move Button and select the folder you created on the d: drive. Any folder with a Location Tab is safe to move.

Another location that usually takes up a huge amount of space is the installed Email program. Most of these applications allow you to easily move the Store folder to a remote location.

Let us know if you need more help.

Thank You for using Windows 7

sylvainlafleur say:

SOLUTION FOUND!!!!!

Hey guys, after spending over a month with Asus suppport (note that I helped them more fixing their own problems than they helped me, which is not at all) and trying to find a way throu forums, I finally found a solution that I can reproduce.

So first of all, after installing Windows 10 from the media creation kit, I downloaded both the driver packages from Asus site for windows 10 (SOCPackage_Intel_Win10_32_VER003), AND for Windows 8.1 (SOCPackage_Intel_Win81_32_VER112) since the camera was working fine Under win 8.1, and after upgrading to Windows 10. I suspected there was some "left behind" piece of software that made all the difference.

From the driver bundle from Windows 10, go into EACH of the folders and find the .inf files (configuration files). Right-click on them ans select install from the context menu. Do that for every folder EXCEPT for the camera. And this is really important.

Once done, reboot. In the device manager, you should now see that all of these yellow warnings are gone save 2 (the front and back cameras). You will also notice that Under the "video input devices" (or Something like that, I have the french version), you will now see that Windows now recognizes the intel signal processor.

Now go into the win8.1 bundle, and go in the camera folders. Yes, all 3 of them. You'll see that there are more than only one config file there. Install EVERYONE of them as you did previously (right-click). There are 8 in total.

Reboot. Now in the device manager, you'll see both camera, and everything looks fine. BUT it is not yet. Notice that the names are in lower cases... Now do the same with the Windows 10 bundle and reboot.

See the names changed to upper case? Now there is one more thing to do. You need to reinstall the Windows camera app as it seems to lock the camera in use at boot.

To do that, use the PowerShell in admin mose as seen on this thread:

http://www.howtogeek.com/224798/how-to-uninstall-windows-10s-built-in-apps-and-how-to-reinstall-them/

Reboot (very important; on my machine, not rebooting here causes the entire process to fail and you have to do it all over again. So it may do the same to you).

Now re-install the app from MS store (https://www.microsoft.com/en-us/store/apps/windows-camera/9wzdncrfjbbg)

Reboot, ENJOY!!!! (don't forget to make a system image at this point...).

Reboot.
