icestorm iii say:

The 360 does not require HDCP for gaming. It's only required for DVD or HD DVD playback. This is something of a moot point as the Dell 2209WA supports HDCP.

 

HDMI to DVI may be an issue if the device on the other end has issues with errant signals. This can happen if the cable wired audio to DVI pins (it shouldn't, but anything's possible).

 

Dell's documentation on the 2209WA indicates it'll accept all the "normal" HDMI signals - 480i/480p/576i/576p/720p/1080i/1080p. As a regular display it'll also accept all resolutions the 360 can output.

 

Seems like the cable could be an issue. Best to try another, or an HDMI cable with an HDMI to DVI adapter.

¡firedog say:

I am having an intermittent problem where ... I get two copies of every email it downloads


I've read through this thread. This is a unique problem that I don't recall having come across before. Just to clarify, and in no particular order,

  • When you say 'I get two copies', does this mean that two identical messages are delivered simultaneously to your inbox? You can check whether they are in fact identical by examining the message headers: select one of the duplicates in the message list and press Ctrl-F3. Note the Date: and Message-ID: headers. Do the same for the other copy of the same message. Are the headers exactly the same?
  • Can you associate any other event with the time this behaviour starts and stops, for example switching on or off one the other device that's accessing the same mailbox? It could be that your smartphone is sending some command to the server in respect of a message it's downloading that makes WLMail think this is a new message.
  • Do you have any software installed that intercepts the mail stream before releasing it to WLMail? This could be an anti-virus or anti-spam program. There is no need to have anti-virus scanning mail - it adds no extra security if you're otherwise careful, and it can cause problems. Some anti-spam software is known to interfere with the proper working of Windows Live Mail, so it would be wise to disable any such program for testing purposes.
  • Are you reading your mail in the account inbox, or do you use one or other of the Quick views? 
  • You leave mail on the server. Does this mean that your server inbox contains a lot (e.g. thousands) of messages? If so, try moving all but the latest few into user-created folders in webmail. Overloaded inboxes can cause problems with POP3 download. While you're in webmail, clear out the deleted items and junk folders.
  • The program log will reveal details of the POP3 traffic between client and server. I see that you have submitted the log to the support staff for examination. Did anything come out of this?
  • How is the computer connected to the Internet? Network Interface Cards can occasionally interfere with traffic between client and server, so if you have an alternative NIC, try that for a bit. E.g. if you're normally on Wi-Fi, try a wired connection instead.


Troubleshooting is tedious, but necessary if you're to get to the bottom of the problem.
