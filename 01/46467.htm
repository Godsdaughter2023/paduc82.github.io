Uninstall/reinstall/reboot? You've got to be kidding. This problem has been here for more than a year. 

MacOS prompts for a password to allow the installation of a "helper application." That is a MacOS thing, not a Skype problem - yet.

Where SKype's failure comes in is actually running the update. Even though you authorize it to do so, it fails. Later, you'll receive the same prompt, in a loop. Uninstalling and reinstalling Skype makes the problem go away... until the next update; then the loop will resume. While Susan S does offer a workaround, I don't consider it a solution. A solution would involve Microsoft listening to the thousands of MacOS+Skype users who have complained about this for over a year, and writing code accordingly - the same way other companies' devs seem to be able to do.

Until then, here's my alternate workaround:

(0) When the "...add a new helper" dialog box appears, don't click anything in it. You might want to drag it by its title bar to the side a little, as you'll need to find it again in step 2.

(1) Click the main Skype window, and from its "Skype" menu, choose "Check for updates." This makes a dialog box titled "About" appear, with a spinner, and causes my CPU to get hot - !?!?!

(2) Now go back to the original "...add a new helper" dialog box, and click "Cancel." You'll get an "Update failed" warning in the other ("About") dialog box.

(3) In the now-failed "About" dialog box, click "Retry." You get "Downloading update...", and then you'll get the exact same "...add a new helper" dialog box again.

(4) This time, authorize the installation with an admin username&password. You'll end up with the "About..." dialog box showing you that your version of Skype is up to date. Close the "About" dialog box. DONE.

Again, that's only a way to work around Microsoft's obstinate problem with auto-updating Skype. But I think it's better than uninstall/reinstall/reboot!

Hi stephenfullard,


You can follow the steps below to reinstall the Microsoft Teredo Tunneling Adapter:

  1. Go to Device Manager by pressing the Windows key + R on your keyboard.
  2. On the run command box, type devmgmt.msc and then hit Enter.
  3. Expand Network Adapter.
  4. Click the Action tab and then select Add legacy hardware. Click Next.

  5. Under Add a Hardware window, select Search for and install the hardware automatically (Recommended).

  6. Choose Network adapters from the list of Common hardware type. Click Next.

  7. On the left pane, select Microsoft and then select Microsoft Teredo Tunneling Adapter on the right pane. Click Next.

  8. Click Next to start the installation. Click Finish.
  9. Go back to Device Manager, click the Scan for hardware changes. Check if the Microsoft Teredo Tunneling Adapter is now listed in Device Manager.
