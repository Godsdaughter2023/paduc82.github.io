TabTip32 Windows 7 64 bit grabs focus - Microsoft Community

slshafer say:

TabTip32 Windows 7 64 bit grabs focus

I have two Fujitsu Lifebook laptops running 64 bit Windows, and both have recently developed a peculiar problem with TabTip32. After logging in, the system almost immediately becomes unusable because the foreground application loses focus. You can open an application, or a command window, but it becomes impossible to type because the focus is lost within a second.

I have tracked the problem down to tabtip32.exe, located in c:\program files (x86)\Common Files\microsoft shared\ink\tabtip32. Tabtip32 is called every few milliseconds. When tabtip32 is called, the foreground task loses focus. You can see it being called every few milliseconds by opening up the task manager.

It is a little concerning that "microsoft shared" and "ink" both show up as entirely lower case names. This is inconsistent with Microsoft's policy of capitalizing the first letter of folder names. Every directory in Common Files other than "microsoft shared" has the first letter capitalized. Similarly, every directory in "microsoft shared" has a capitalized first letter except for "ink". Odd....

The problem is easy to fix: 1) take ownership of tabtip32.exe (the default owner is the TrustedInstaller, and 2) change the name to something else. The instant the name is changed, the problem goes away (other than the event log, which reflects errors because Windows can't find TabTip32). Unfortunately, Windows Update occasionally reinstalls tabtip32.exe. When that happens, both of my computers become unusable until I repeat the process described above.

The behavior is consistent with malware. However, I have Sophos and Malwarebytes installed on each computer as active processes. I also installed Adaware as a backup scanning program. I run Adaware every week or so. Both computers shows up as clean with all of these programs. It seems unlikely that tabtip32.exe has been replaced by malware missed by three respected virus detection programs. Also, malware is supposed to operate invisibly. Malware that makes your computer inoperable would not readily spread.

I am quite puzzled that this problem has not been reported. I have two nearly identical Fujitsu computers (T731 and T734), both running Windows 7 64 bit, and both with exactly the same problem. Based on that, I would expect this would be widely reported. It isn't.

This could be caused by registry problems on both machines. I've used the registry cleaner from Lavasoft (a vendor I trust), but that has not had any influence on the bizarre behavior of tabtip32.

I would appreciate suggestions on what is causing this, and how it can be permanently addressed.

Thank you,

Steve Shafer

In Windows 7, the Tabtip32.exe file is the Tablet Input Panel and Touch Keyboard component.

lemp say:
The best thing to do would be to completely wipe the hard drive using a "secure delete" utility such as Darik's Boot and Nuke.  If you wanted to be generous, you could then re-install Windows XP and include the install CD with the computer.

More paranoid individuals would advocate removing the hard drive entirely, selling the computer without a hard drive, and either keeping or physically destroying the hard drive as desired.

If you have no Windows XP install CD and you insist on selling the computer with an operating system, the next best thing would be to use the computer manufacturer's system to restore the computer to its "fresh from the factory" state.  Usually, this is done by using a hidden partition on the hard drive.

There are "file shredder" applications available (you can search for them), but the problem with deleting individual files and folders -- even if the delete is done with a tool that reasonably prevents data retrieval -- is that you will never know if you managed to root out all of your personal information.  In particular, you probably will be unable to safely assume that the Windows registry has been completely "sanitized."
