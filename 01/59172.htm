ad-man-gamer say:

(4K Private, 16k Page Table, 20K Total) Memory leaking in Windows

Well this is odd.

There seems to be a bug in the Ndu service in windows 8.1.

I realised that my computer was losing memory over time. Memory that the system was not registering in task manager or performance monitor.

After taking a look under the hood using RamMap, it turned out that the page table became huge as it was filled with 16K fragments from processes that have been long gone, and windows services where spamming the page table consistently.

After looking around, there were a few people that reported having the same problem, and said that disabling Ndu in regedit by setting the start value to 4 fixed the problem. So I gave it a go, and what do you know. Fixed!

There is still the odd 20K entry, but they're no longer spamming the memory like before.

So has anyone else had this problem? Is there a memory bug in the Ndu service? It sure does seem that way given the drastic improvement after disabling.

I'm also surprised that windows can't auto detect and clean these entry's out, as they have a very unique fingerprint (4K Private, 16k Page Table, 20K Total).

(Note: This 20K leak also occurs on windows 7. I helped a friend a couple of days ago with a similar problem and had to use Windows Performance toolkit to find the culprit. I managed to minimise the 20K memory spam but some still remained.

Still, Windows should auto detect and purge these from ram when it occurs.
