Well it seems strange that you went from 3.47 to 2.99 when your video memory is at 512 MB.  3.47 MB is just about I would expect with 512 MB of video RAM.  It sounds like something else has mapped some memory.  That could happen for instance when running a virtual machine, but it should only be true while it is running.

If you look in the System Information -> Hardware Resources -> Memory, it gives the memory ranges the hardware is using.  You would be looking for a pretty big range (512 MB).  The addresses are in hex if you bring up the calculator, select the "bar" menu and switch to Programmer -> Hex, you can put in the addresses and do the math.  And click on DEC to see the size in decimal.

Oh course you could change the memory use for the video card, but that probably would only be about 256 MB if it is like mind.  Since 256 MB is the min it would allow.  You might be able to change this while running if the control panel item for you video card allows it, but in my case I had to change it in the BIOS.  I can't give you instructions on exactly how to change it in the BIOS because it is different for different processors/motherboards.  You usually press Del or F2 to get into the BIOS at boot time, and in my case I had to change a video setting that picked the size automatically (1 GB!) to manual and then I choose the smallest size because I don't play any games or such that would make use of even the 256 MB let alone 1 GB!

a_user_here say:

I think you are confusing "usable memory" with memory available on the Task Manager.

And hopefully ChengJ8603 isn't doing the same.

I you right click on the Start button, and select System.  It will give two numbers for the RAM.

One is the amount installed, and in this case that should be 4 GB.  The other is the "usable" RAM.

This is physical memory that the system can use.  And as I described above that is going to be less than the 4 GB because that at least some of that memory is being shared with the video, and it sounds like something else is taking another 512 MB for something.

In any case the usable RAM is what the Task Manager uses for the max (second number), and the first number is what is currently being used.  Which is of course includes everything that the system (including apps, system, file cache, ...) is using.

What's more in the case of the checking on the apps sizes in the settings you are confusing the space they take on disk opposed to the space they are taking in RAM.  You would want to look in the Task Manager at any running process to see what memory they are taking at any given time.

EDIT maybe you are not confusing the sizes in the settings and are just using this section as a place to uninstall apps that you are not using.  If so that is fine, but in truth as long as you have set them not to run then that is all that is needed as far as memory is concerned.  Also you probably would be better off using Settings -> Privacy -> Background apps, and turn them off there.  Depending on your needs.
