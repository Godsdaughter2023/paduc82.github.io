wunders say: One difference between the normal host name resolving sequence (like with your ping) and using "nslookup" is that when you do the ping, the following name-resolving sequence occurs:
   1) it checks to see if it is your machine
   2) it searches the "Hosts" file  (usually located in C:\Windows\System32\Drivers\Etc\Hosts)
   3) it goes out to the DNS server.
   4) it checks NetBIOS name resolutions (Broadcasts, WINS, etc.)

Using "nslookup", it forces an immediate query directly to the DNS server, skipping steps 1 and 2 above.

So the logical culprit would be a problem with your Hosts file.
A default Hosts file contains only one line that does not begin with a "#" character and that line is:
   127.0.0.1   localhost

If your hosts file contains more than that, then try resetting it:
"How do I reset the hosts file back to the default?"
  < http://support.microsoft.com/kb/972034 >

There are some malwares out there that use the Hosts file to hijack connections.

HTH,
  JW

Windows 10 error 0xc004f014 is an activation error that indicates your computer is unable to contact Microsoft Key Management Service (KMS) to validate your copy of Windows. To resolve an issue with Windows 10 activation error code 0xc004f014, follow the steps below:

  1. Open a CMD window as an administrator.
  2. Type or paste "slui 4".
  3. Select your country and call the number for the automated activation system.
  4. Enter your valid Windows key and copy the Installation ID that system provides.
  5. Let Windows activate.

Give the system a few hours to activate before doing anything else. If it doesn’t work, you may contact our Answer Desk Support team through phone or chat.
