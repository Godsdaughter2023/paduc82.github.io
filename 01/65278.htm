a. user say:

If you have a genuine Vista Installation Disk (and not an OEM disk or a Recovery disk from Dell), then I can help you. If not, you need to contact Dell as it is their recovery process and not Microsoft's and we have no infomration about how it works.

Assuming you're using a genuine Vista Installation Dist, the installation has obviously failed at this point so you should cold stop the system and reboot to begin the process again.  Before doing so I would clean the disk gently with a tissue running from center to edge to remove any fingerprints or smudges that may be causing the problem.

When given the option to format, choose a full format rather than a quick format (just to make sure that has nothing to do with the problem).

Then try the re-iinstallation again and hope it works this time.  If it won't re-install from the disk, then try to find another genuine Vista Installation Disk of the same bit-size to borrow and try.  If not, then push F12 immediatly after pushing the startup button to access system diagnotics (at least that's how it works on my Dell - use whatever process works for yours if you have the diagnostics program installed) and run every option to see if it finds any problems with your system (hopefully not).

If not, then you should contact Dell Technical Support (if out of warranty a phone call may cost money but I believe the chat option is free even after the warranty expires and see if they have any suggetions to offer.

I hope this helps.

Good luck!

Hi...

It wasn't removing the main battery that fixed the problem, but the smaller CMOS battery...the round one like a watch battery.

If your PC is outside its warranty, then open it up, find the CMOS battery and remove it. (Should be something on the internet about how to do it).

Then you need to 'get rid of any extra power' by holding the power-on button for about 30 secs. (At least that's how I did it on mine).

BUT don't ever let the computer go into sleep mode again. I've set my computer to never go into sleep mode (use settings to adjust this). Often, if the computer is unattended for more than 20 mins, it goes into sleep mode. I set mine to just blank the display.

lemp say:

I don't know why this is happening (network communication problems?), but you can make a batch job to implement the process described at the link in Yasshiel Zhang's post.

@echo off
net stop spooler > nul
del /Q "%windir%\system32\spool\PRINTERS\*.*"
net start spooler > nul
exit

If the workstations have the Fax service running (probably not), the above has to be modified slightly:

@echo off
net stop spooler /y > nul
del /Q "%windir%\system32\spool\PRINTERS\*.*"
net start spooler > nul
net start fax > nul
exit

If you create a shortcut and set the shortcut to "Run as administrator" ("Advanced" button in Shortcut properties) and "Run minimized," the only thing you have to do after double-clicking the shortcut is respond to the UAC prompt.

lemp say:
Merely stopping and restarting the spooler won't necessarily get rid of corrupted print jobs in the queue.  The "queue" is the files in "%windir%\system32\spool\PRINTERS\*.*".  On the other hand, you can't delete those files if the print spooler is running.
