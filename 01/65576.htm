plugin_container.exe has encounterder a problem.

a. user say:

plugin-container.exe is a Firefox program - not an XP program.

That means you don't have an XP problem, you have a Firefox problem.

When Firefox loads and if it then loads a plugin, the plugin gets launched as a separate executable process so that if the plugin has problems and crashes, your whole Firefox browser won't crash - just the plugin will crash.

If it just started happening and you can remember any recently installed plugins, you might temporarily switch that plugin to "Never Activate" to see if you can isolate the crashing to a particular plugin, or you may need to just start deactivating them one at a time until the problem goes away and the last plugin you deactivated would be the suspect.

I have all my plugins set to "Ask to Activate" and you might check to see if the plugin manufacturer has an update for you:

You can read more about it here:

https://support.mozilla.org/en-US/kb/What%20is%20plugin-container

a. user say:

Hello,

Thank you for posting your query on Microsoft community.

As per stated, I understand that you are facing issues while copying some files into folders.

The basic problem is that under standard Windows file naming the total name cannot exceed 259 characters. Unfortunately this also includes the path to the file as well so "C:\Folder A\Folder B\.....\Folder X\Filename.ext is limited.

However if you share a folder C:\some long path\folder m\ as Share A then on the PC using the share as say drive M:\ knows nothing of the "long path". It then applies its limit of 259 characters to the M: drive.

But NTFS itself allows up to 256 characters per element and can be up to 30,000 characters so a write at share level can produce a path which Windows itself cannot support.

I would like to know are you getting this message while transferring any particular file types or for all files.

I would suggest you to run the fixit and follow the steps in below mentioned link.

https://support.microsoft.com/en-us/kb/2891362

You may also try the following steps as a workaround and see if it helps you.

  • Rename the file so that it has a shorter name.
  • Rename one or more folders that contain the file so that they have shorter names.
  • Move the file to a folder that has a shorter path name.

bill smithers say:

You can not move the Windows 8.1 license to another computer as it came preinstalled.

If the Laptop/Computer came with the Windows Operating System preinstalled (Dell, HP, Acer, etc) , that Product Key that came with the Laptop/Computer is for the preinstalled OEM License and is non-transferable.

The OEM License/Product Key (preinstalled Operating Systems on Dell, HP, etc machines) is tied to the first machine it is installed on and dies when that machine dies.

The above rule also applies to OEM System Builders Licenses.

Retail Operating System License, that you buy and install yourself, are allowed to be transferred to another machine.

But Retail Licenses can also only be installed on one machine at any particular time.
