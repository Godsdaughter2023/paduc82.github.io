The PSE Organizer likely uses system routines to do the movement of files, but based on system performance I would guess that it is a logical movement of the file that alters links / pointers in the file system rather than actually reading and rewriting the file and then deleting the original copy.  After the file is "move" to a new folder / directory the "Archive" attribute is set, but no other attributes, especially date attributes, appear to be changed.  I also experimented with File Explorer and noticed identical results which I would attribute to PSE Organizer and File Explorer using the same system routines.

Here's where a problem arises.  Windows File History appears to look at one or more file attributes in order to decide if a file needs to be copied to the File History location, however it is not using the "Archive" attribute. From what I've read in several places, most backup programs use some algorithm based on date information and possibly other file attributes to decide if a file needs to be backed up. The "Archive" attribute seems to be a legacy structure brought from the creation of NTFS and Windows NT and no longer is really used, but has not been removed.  For Windows File History it seems the decision to back up a file depends on at least one of the dates associated with the file, most like the Date Modified or something similar.  That date would need to be later than the date the last backup was run.  Since no date attributes are changed, the files that I've moved using PSE Organizer are not detected by Windows File History and consequently are not copied to the History archive.  The only thing that seems to trigger File History to copy a file is to actually make a modification to the file contents and resave it. This is a very tedious and time consuming task and really makes using File History unworkable.  This appears to me to be an error or logical oversight in the design of the algorithm in File History that decides when a file needs to be copied.

Hi Barrie,

 

Microsoft Secure Boot is a component that relies on the UEFI (Unified Extensible Firmware Interface) specification to help prevent malicious software and unauthorized operating systems from booting. It's safe to start Windows 8 without Secure Boot enabled.

How To Find Wireless Passphrase (WPA/WEP)

  1. Login to the router by entering the default gateway IP address.
  2. The router will then prompt you to enter the user name and password.the default user name and password are: "admin", "admin".
  3. From the menu on the left select WLAN.
  4. Your passphrase will be listed as WPA Shared Key or Key 1.

 

Changing Wireless Security Type/ Disabling Wireless Security

  1. Login to the router by entering the default gateway IP address.
  2. The router will then prompt you to enter the user name and password.the default user name and password are: "admin", "admin".
  3. From the menu on the left select WLAN.
  4. Use the Security Mode drop down to choose your desired security
  5. If WEP is selected, select the WEP Key Length (either 64-bit or 128-bit).  The password will need to be 10 characters in length (if 64-bit) or 26 characters in length (if 128-bit) and can only use 0-9 and A-F.
  6. If WPA or WPA2 is selected use the Encryption drop down to select your desired cipher type (TKIP preferred).  The password can be between 8-63 characters, is case sensitive and can use any character on your keyboard.
  7. Select Apply.
